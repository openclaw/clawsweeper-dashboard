## What This Changes

The issue reports gateway event-loop saturation attributed to accumulating atomic-write temp files and slow repeated `node.list` polling with no nodes paired.

## Best Possible Solution

Capture a focused profile on the affected state directory, then add the narrow measured fix: likely bounded orphan-temp cleanup for known state files and, only if profiling confirms it, a faster `node.list` no-pairing/cache path.
