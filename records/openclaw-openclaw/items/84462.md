## Summary

This should stay open: current Telegram mention-gating still has a source-visible path where `/command@TargetBot` is not treated as an explicit address unless the command is already a recognized OpenClaw control command, so an arbitrary targeted bot command can still be dropped as `no-mention`.

## What This Changes

The issue requests that Telegram group ingress treat `bot_command` entities like `/command@TargetBot` as an explicit mention for the addressed bot account.

## Best Possible Solution

Teach the Telegram ingress mention detector to recognize `bot_command` entities addressed to the current bot username as explicit mentions before both normal body handling and the pre-download media skip path, with regression coverage for current-bot, other-bot, known-command, and arbitrary-command cases.

## Reproduction Assessment

Yes, by source inspection: a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no matching text pattern leaves `wasMentioned` false and reaches the `no-mention` skip path. I did not execute a live Telegram run in this read-only pass.

## Solution Assessment

Yes, the maintainable fix is narrow: extend Telegram's explicit-address detection for current-bot `bot_command` entities rather than weakening group mention gates or broadening control-command bypasses.
