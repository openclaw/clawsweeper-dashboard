## Summary

Keep this open: current main still ignores Telegram `bot_command` entities in `hasBotMention`, so `/command@TargetBot` can remain unmentioned for the group gate, and https://github.com/openclaw/openclaw/pull/84471 is the open closing PR that should be reviewed or replaced before this issue closes.

## What This Changes

Treat Telegram `/command@TargetBot` `bot_command` entities as explicit mentions for the addressed bot in mention-required groups.

## Best Possible Solution

Land one shared Telegram helper/gate fix with regression coverage and redacted live Telegram proof for `/command@TargetBot` and `/command@OtherBot`, then close this issue from the merged PR.

## Reproduction Assessment

Yes. By source inspection, a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no matching OpenClaw control command leaves `wasMentioned` false and reaches the `no-mention` skip path.

## Solution Assessment

Yes for the implementation direction: extend Telegram's shared explicit-address detection for current-bot `bot_command` entities rather than weakening mention gates. The issue should remain open while the linked fix PR is reviewed, proved, or replaced.
