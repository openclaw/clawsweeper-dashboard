## Summary

This should stay open: current `main` still has the source-visible Telegram mention-gate gap, and the issue is already paired with an open closing PR that should remain the implementation path.

## What This Changes

The issue reports that Telegram group messages addressed as `/command@TargetBot` are dropped as `no-mention` when `requireMention` is enabled.

## Best Possible Solution

Keep the issue open until the active linked PR lands or is replaced, with coverage for targeted and other-bot commands plus redacted live Telegram proof before merge.

## Reproduction Assessment

Yes. By source inspection, a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no configured mention-pattern match leaves `wasMentioned` false and can reach the `no-mention` skip path.

## Solution Assessment

Yes. The narrow maintainable solution is to teach the shared Telegram mention helper to recognize `bot_command` entities addressed to the current bot, rather than weakening group mention gates or adding a separate routing bypass.
