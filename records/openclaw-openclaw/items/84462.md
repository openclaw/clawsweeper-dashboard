## Summary

This should stay open: current main still has the Telegram source path that drops arbitrary `/command@TargetBot` messages under `requireMention: true`, and an open linked PR is already the active fix path.

## What This Changes

The issue reports that Telegram group messages addressed as `/command@TargetBot` are not treated as explicit mentions for the target bot and can be skipped as `no-mention`.

## Best Possible Solution

Review and land the open linked PR, or an equivalent narrow patch, so Telegram `bot_command` entities addressed to the current bot set the explicit mention signal in both normal body handling and pre-download media gating with regression coverage.

## Reproduction Assessment

Yes, by source inspection: a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no matching mention pattern leaves `wasMentioned` false and can reach the `no-mention` skip path. I did not execute a live Telegram run in this read-only pass.

## Solution Assessment

Yes, the best fix is narrow: extend Telegram explicit-address detection to recognize current-bot `bot_command` entity slices rather than weakening group mention gates or treating all bot commands as allowed mentions.
