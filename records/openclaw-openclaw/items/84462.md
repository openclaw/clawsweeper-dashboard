## Summary

This should stay open: current main still has a source-visible Telegram path where `/command@TargetBot` is not recognized as an explicit mention, and two open fix PRs already reference this issue with closing syntax, so the issue should remain open until one lands.

## What This Changes

The issue reports Telegram group messages addressed as `/command@TargetBot` being dropped by the mention gate when `requireMention` is enabled.

## Best Possible Solution

Review and land one of the linked fix PRs after it updates Telegram bot-command mention detection for both normal body handling and pre-download media gating, with regression coverage for current-bot and other-bot command entities.

## Reproduction Assessment

Yes, by source inspection: a message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no configured mention-pattern match leaves `wasMentioned` false and can hit the `no-mention` skip path. I did not execute a live Telegram run in this read-only pass.

## Solution Assessment

Yes, the best fix is narrow: teach Telegram's explicit-address detector to recognize current-bot `bot_command` entities instead of weakening group mention gates or relying on control-command bypasses.
