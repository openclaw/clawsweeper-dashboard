## Summary

Keep open: current `main` still has the source-visible Telegram mention-gate bug, and the linked fix at https://github.com/openclaw/openclaw/pull/84471 is still open rather than merged.

## What This Changes

The issue requests treating Telegram `/command@TargetBot` `bot_command` entities as explicit mentions for the targeted bot when group `requireMention` gating is enabled.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84471 or an equivalent narrow helper fix with regression coverage for targeted, non-targeted, and existing mention cases.

## Reproduction Assessment

Yes, by source inspection: a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no matching mention pattern leaves `wasMentioned` false and can reach the `no-mention` skip path.

## Solution Assessment

Yes, the narrow maintainable fix is to recognize `bot_command` entities addressed to the current bot username in the shared Telegram mention helper, without weakening group mention gates.
