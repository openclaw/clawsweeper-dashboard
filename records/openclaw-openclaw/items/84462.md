## Summary

This should stay open: current main still has the source-visible Telegram mention-gate bug, and the issue is now paired with an open implementation PR that uses closing syntax but has not merged yet.

## What This Changes

Telegram group messages addressed as `/command@TargetBot` should pass the require-mention gate for the target bot instead of being dropped as unmentioned.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84471 after regression coverage and redacted live Telegram proof, then close this issue as implemented.

## Reproduction Assessment

Yes. By source inspection, a Telegram group message with text `/ping@TargetBot`, a `bot_command` entity, `requireMention: true`, and no separate `mention` entity leaves `hasBotMention()` false and can reach the `no-mention` skip path.

## Solution Assessment

Yes. The narrow maintainable fix is to teach the shared Telegram mention helper to recognize current-bot `bot_command` entities, so normal body handling and the pre-download media gate stay aligned.
