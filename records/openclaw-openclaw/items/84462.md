## Summary

This should stay open: current main still ignores Telegram `bot_command` entities in the mention detector, and the open closing PR at https://github.com/openclaw/openclaw/pull/84471 is the right implementation candidate to review before closing this issue.

## What This Changes

Treat Telegram `/command@TargetBot` bot-command entities as explicit mentions for the addressed bot in mention-gated groups.

## Best Possible Solution

Review and land a single focused Telegram helper fix that recognizes current-bot `bot_command` entities before both mention-gate paths, with regression coverage and live Telegram proof for target-bot and other-bot cases.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `/ping@TargetBot` with a `bot_command` entity, `requireMention: true`, and no matching text pattern leaves `wasMentioned` false and reaches the `no-mention` skip path. I did not run a live Telegram credentialed repro.

## Solution Assessment

Yes, the best code shape is to extend the shared Telegram explicit-address detector for current-bot `bot_command` entities, not to relax mention gates or broaden the control-command bypass.
