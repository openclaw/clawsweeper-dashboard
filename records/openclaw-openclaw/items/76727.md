## Summary

Keep open: the PR addresses the reported Chat Completions failure, but the current diff is overbroad and would also suppress `gpt-5.4-mini` reasoning on the supported Azure Responses transport.

## What This Changes

The PR makes `gpt-5.4-mini` reasoning-effort resolution API-aware, adds resolver and Chat Completions payload regressions, and records the Telegram/OpenAI fix in the changelog.

## Best Possible Solution

Scope the `gpt-5.4-mini` suppression to Chat Completions only and add a regression proving Responses transports, including Azure, still keep reasoning support.

## Reproduction Assessment

Yes, source-level reproduction is clear: current main can emit Chat Completions `reasoning_effort` for `gpt-5.4-mini`, and the PR's added branch would make Azure Responses resolve no reasoning efforts.

## Solution Assessment

No. The transport-specific direction is right, but the current implementation suppresses every non-listed API instead of only `openai-completions`; preserving all Responses transports is the narrower fix.
