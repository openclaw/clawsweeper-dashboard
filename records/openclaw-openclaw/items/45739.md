## Summary

Keep open. Current main still lacks this PR's intended ACP parent relay fallback from gateway state, and the PR remains a useful implementation candidate, but the proposed patch has blocking relay race/duplication issues that need maintainer review and a rebase before merge.

## What This Changes

The PR adds `chat.history` and `agent.wait` fallback polling to the ACP parent stream relay, regression tests for missing local relay events, and a changelog entry.

## Best Possible Solution

Keep the PR open and rework the relay fallback on top of current main, preferably using the existing `src/agents/run-wait.ts` helpers for scoped history/wait reads, preserving current commentary/task-progress behavior, and adding regression coverage for stale transcript persistence, whitespace-only local events, baseline races, and history-first/local-second duplication.

## Reproduction Assessment

Yes. The linked report and PR body provide concrete ACP `sessions_spawn({ runtime: "acp", streamTo: "parent" })` steps where the parent gets only start/stall while child history contains assistant output; this review did not run a live ACP harness, but current-main source inspection confirms there is no gateway history/wait recovery path.

## Solution Assessment

No, not as-is. The fallback direction is plausible and targets the right relay file, but the patch should be rebased and fixed for lifecycle/history timing, local/history deduplication, baseline races, and changelog attribution before it is a maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing the lifecycle path:** `src/agents/acp-spawn-parent-stream.ts:481`
  - body: `emitAcpLifecycleEnd` currently runs before `persistAcpTurnTranscript`, so this single forced `chat.history` read can still be stale. If local assistant events are missing, the relay then emits completion and disposes, permanently losing the child reply this fallback is meant to recover.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:451`
  - body: A whitespace-only assistant event sets `sawLocalAssistantEvent` before `recordAssistantOutput` rejects it, which suppresses history backfill even though no output was emitted. Gate the flag on a true return from `recordAssistantOutput`.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:306`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data. Recheck `historyBaselineReady` after the await before writing the baseline.
  - confidence: 0.85
- **[P2] Deduplicate local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:459`
  - body: If the history fallback relays the full assistant text just before delayed local assistant deltas arrive, this path records and emits the overlapping local text again. Add a guard or regression test for history-first/local-second delivery before merging.
  - confidence: 0.78
- **[P3] Add required changelog attribution:** `CHANGELOG.md:67`
  - body: The added changelog entry lacks the required `Thanks @...` credit. OpenClaw changelog policy requires every added entry to include an attribution, so credit the reporter or contributor on this line.
  - confidence: 0.9
