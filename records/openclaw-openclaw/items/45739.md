## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing lifecycle end:** `src/agents/acp-spawn-parent-stream.ts:481-482`
  - body: The local `lifecycle:end` path performs one forced history read and then disposes. ACP lifecycle completion can arrive before transcript persistence, so when local assistant events are missing this can still permanently miss the child output the fallback is meant to recover.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: `sawLocalAssistantEvent` is set before `recordAssistantOutput` confirms there is non-empty text. A whitespace-only assistant event can therefore suppress history backfill even though the parent saw no progress output.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:304-309`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data, causing duplicate relay output later.
  - confidence: 0.86
- **[P2] Deduplicate delayed local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:459`
  - body: If history fallback emits the full assistant text before delayed local assistant deltas arrive, the local path can append overlapping content again. Add a guard or regression case for history-first, local-second delivery.
  - confidence: 0.78
