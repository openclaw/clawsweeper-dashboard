## Summary

Keep open: the PR targets a real ACP parent-relay gap, but current main still has only the local `onAgentEvent` relay path for live parent streaming while this branch has concrete timing and deduplication blockers before it is safe to merge.

## What This Changes

The PR adds gateway `chat.history` and `agent.wait` fallback polling to the ACP parent stream relay, new relay regression tests, and a changelog entry.

## Best Possible Solution

Rework the fallback on current main so ACP parent relays can recover missing child output and terminal state through shared gateway wait/history helpers while preserving existing local-event semantics, task completion delivery, and commentary suppression.

## Reproduction Assessment

Yes from source inspection and the linked reports. The current-main relay path is `sessions_spawn({ runtime: "acp", streamTo: "parent" })` through `startAcpSpawnParentStreamRelay`, and that relay still depends on local `onAgentEvent` delivery for live output/completion.

## Solution Assessment

No, not as-is. Gateway-backed recovery is a plausible narrow fix, but this branch needs current-main rework plus fixes for lifecycle/history timing, local/history deduplication, and baseline races before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing lifecycle:** `src/agents/acp-spawn-parent-stream.ts:481-482`
  - body: The local lifecycle-end path performs one forced history read and then disposes. ACP terminal lifecycle can arrive before the transcript is persisted, so when local assistant events are missing this can still permanently miss the child output the fallback is intended to recover.
  - confidence: 0.9
- **[P2] Gate the local-output flag on relayed text:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: `sawLocalAssistantEvent` is set before checking whether the assistant event has relayable text. A whitespace-only or empty local assistant event can suppress history backfill even though the parent received no child output.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:304-309`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then a slower priming read overwrites `lastHistoryAssistant` with stale data and makes the next poll relay duplicate output.
  - confidence: 0.86
- **[P2] Deduplicate local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:343-459`
  - body: If history fallback emits the full assistant text and delayed local assistant deltas arrive afterward, the local path can append overlapping content again. Add a guard or regression test for history-first, local-second delivery.
  - confidence: 0.78
- **[P3] Add human changelog attribution:** `CHANGELOG.md:67`
  - body: This is a user-facing bug fix, and the changelog entry lacks the required human `Thanks @...` credit. Credit the reporter or contributor on the same single line before merge.
  - confidence: 0.9
