## Summary

This PR should stay open for now: it targets a real ACP parent-streaming gap, but current main has since changed the completion path and this branch still has concrete timing/deduplication bugs before it is mergeable.

## What This Changes

The branch adds gateway `chat.history` and `agent.wait` fallback polling to the ACP parent stream relay, new relay regression tests, and a changelog entry.

## Best Possible Solution

Rebase or replace the branch with a current-main implementation that keeps existing ACP spawn semantics, uses the shared run-wait/history helpers where possible, and adds focused regression coverage for missing local events plus mixed local/history races.

## Reproduction Assessment

Yes from source inspection and linked reports, though I did not run a live ACP smoke in this read-only pass. The reported path is `sessions_spawn({ runtime: "acp", streamTo: "parent" })`, and current main still lacks gateway history backfill in the live parent relay.

## Solution Assessment

No, not as-is. Gateway-backed recovery is a plausible fix direction, but this branch needs current-main rework and fixes for lifecycle/history timing, local/history deduplication, baseline races, and changelog policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing the lifecycle path:** `src/agents/acp-spawn-parent-stream.ts:482`
  - body: The lifecycle-end path performs one forced history read and then disposes. ACP command execution can emit terminal lifecycle before the transcript is persisted, so this can still permanently miss the child assistant output in the missing-local-event case the PR is meant to recover.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:453`
  - body: `sawLocalAssistantEvent` is set before validating that the assistant event contains relayable text. A whitespace-only local event can suppress history backfill even though the parent received no progress output.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:306-310`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data, causing a later duplicate relay.
  - confidence: 0.86
- **[P2] Deduplicate delayed local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: If history fallback emits the full assistant text just before delayed local assistant deltas arrive, the local path can emit overlapping content again. Add a guard or regression test for history-first, local-second delivery.
  - confidence: 0.78
- **[P3] Add required changelog attribution:** `CHANGELOG.md:67`
  - body: The user-facing changelog entry lacks the required human `Thanks @...` credit. Credit the reporter or contributor on the same single line before merge.
  - confidence: 0.9
