## Summary

Keep open. The PR targets a real ACP `streamTo:"parent"` relay gap and current `main` still does not implement this branch's live assistant-history backfill, but the patch is not merge-ready because it is stale against the current task-registry delivery model and has concrete race/deduplication bugs in the new fallback state machine.

## What This Changes

The PR adds gateway `chat.history` and `agent.wait` fallback polling to the ACP parent stream relay, regression tests for missing local relay events, and a changelog entry.

## Best Possible Solution

Rework the live-output fallback on current main only if maintainers still want explicit `streamTo:"parent"` live progress, using shared run-wait/history helpers, task-registry state where appropriate, strict local/history deduplication, and focused regression tests.

## Reproduction Assessment

Yes. #45205 and this PR describe a concrete ACP `sessions_spawn({ runtime: "acp", streamTo: "parent" })` path where the parent receives only start/stall while child history contains assistant output; current-main source inspection confirms the dedicated relay still lacks a gateway history backfill path.

## Solution Assessment

No, not as-is. Gateway-backed recovery is a plausible direction for live output, but this branch needs rebase and correctness fixes for lifecycle/history timing, local/history deduplication, baseline races, and changelog policy before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing the lifecycle path:** `src/agents/acp-spawn-parent-stream.ts:482`
  - body: The lifecycle-end path performs one forced history read and then disposes. ACP command execution can emit terminal lifecycle before the transcript is persisted, so this can still permanently miss the child assistant output in the missing-local-event case the PR is meant to recover.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:453`
  - body: `sawLocalAssistantEvent` is set before validating that the assistant event contains relayable text. A whitespace-only local event can suppress history backfill even though the parent received no progress output.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:306-310`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data, causing a later duplicate relay.
  - confidence: 0.86
- **[P2] Deduplicate delayed local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: If history fallback emits the full assistant text just before delayed local assistant deltas arrive, the local path can emit overlapping content again. Add a guard or regression test for history-first, local-second delivery.
  - confidence: 0.78
- **[P3] Add required changelog attribution:** `CHANGELOG.md:67`
  - body: The user-facing changelog entry lacks the required human `Thanks @...` credit. Credit the reporter or contributor on the same line before merge.
  - confidence: 0.9
