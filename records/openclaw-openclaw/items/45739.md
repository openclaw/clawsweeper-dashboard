## Summary

Keep open. Current main has addressed the canonical ACP terminal-completion cluster, but it does not implement this PR's intended live parent-relay backfill from `chat.history`/`agent.wait`; the PR branch also has blocking relay race and deduplication issues that make it unsafe to merge as-is.

## What This Changes

The PR adds gateway `chat.history` and `agent.wait` fallback polling to the ACP parent stream relay, regression tests for missing local relay events, and a changelog entry.

## Best Possible Solution

A maintainer should decide whether live assistant-output backfill is still desired after the current task-completion fixes; if it is, rework the relay on current main with run-scoped fallback state, existing run-wait/history helpers, strict deduplication between local deltas and history snapshots, and regression tests for stale persistence and mixed delivery ordering.

## Reproduction Assessment

Yes. #45205 and this PR describe a concrete ACP `sessions_spawn({ runtime: "acp", streamTo: "parent" })` path where the parent receives only start/stall while child history contains assistant output; current-main source inspection confirms the dedicated relay still lacks the gateway backfill path proposed here.

## Solution Assessment

No, not as-is. The direction is plausible for live-output recovery, but the branch needs rebase and correctness fixes for lifecycle/history timing, local/history deduplication, baseline races, and changelog attribution before it is a maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing the lifecycle path:** `src/agents/acp-spawn-parent-stream.ts:482`
  - body: The forced history read runs after lifecycle end, but current ACP execution emits lifecycle end before transcript persistence. If local assistant events are missing, this single read can be stale and the relay then completes and disposes before the child reply appears in history.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:453`
  - body: `sawLocalAssistantEvent` is set before `recordAssistantOutput` rejects empty or whitespace text. A whitespace-only local event can therefore suppress all history backfill even though no progress was emitted to the parent.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:306-310`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data, causing a later duplicate relay.
  - confidence: 0.85
- **[P2] Deduplicate delayed local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: If history fallback emits the full assistant text just before delayed local assistant deltas arrive, the local path can emit overlapping content again. Add a guard or regression test for history-first, local-second delivery.
  - confidence: 0.78
- **[P3] Add required changelog attribution:** `CHANGELOG.md:67`
  - body: The new user-facing changelog entry lacks the required `Thanks @...` credit. Credit the reporter or contributor on that line before merge.
  - confidence: 0.9
