## Summary

Keep open: the PR targets a real ACP parent-relay failure, but current main has since changed the completion path, the branch is now conflicting, and the proposed fallback still has concrete race/deduplication bugs before it can merge.

## What This Changes

The PR adds ACP parent stream relay fallback polling that reads child assistant output from gateway `chat.history`, probes child terminal state with `agent.wait`, adds regression tests, and updates the changelog.

## Best Possible Solution

Rebase or replace the branch with a current-main patch that preserves ACP task-completion routing, uses the shared run-wait/history helpers, and adds regression coverage for missing local events plus mixed local/history races.

## Reproduction Assessment

Yes for the PR blockers from source inspection: the branch code shows the baseline race, one-shot lifecycle history read, and local/history deduplication gaps. I did not run a live ACP smoke in this read-only pass, and current main has partially changed the original terminal-completion path.

## Solution Assessment

No, not as-is. Gateway-backed recovery is a plausible fix direction, but this implementation should be rebased onto current main and repaired around shared helper contracts and timing races.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry history before completing the lifecycle path:** `src/agents/acp-spawn-parent-stream.ts:481-482`
  - body: The lifecycle-end path performs a single forced history read and then disposes. ACP lifecycle completion can arrive before transcript persistence, so this can still permanently miss child assistant output in the missing-local-event case this PR is meant to recover.
  - confidence: 0.9
- **[P2] Set the local-output flag only after relaying text:** `src/agents/acp-spawn-parent-stream.ts:453-459`
  - body: `sawLocalAssistantEvent` is set before validating that the assistant event contains relayable text. A whitespace-only local event can suppress history backfill even though the parent received no progress output.
  - confidence: 0.88
- **[P2] Recheck the history baseline after awaiting:** `src/agents/acp-spawn-parent-stream.ts:304-309`
  - body: `primeHistoryBaseline` can race the first fallback poll: the poll may initialize and relay the real reply, then the slower priming read overwrites `lastHistoryAssistant` with stale data, causing a later duplicate relay.
  - confidence: 0.86
- **[P2] Deduplicate delayed local deltas after history backfill:** `src/agents/acp-spawn-parent-stream.ts:459`
  - body: If history fallback emits the full assistant text just before delayed local assistant deltas arrive, the local path can emit overlapping content again. Add a guard or regression test for history-first, local-second delivery.
  - confidence: 0.78
