Full review comments:

- **[P1] Remove the private Doctor context import:** `src/commands/doctor-wsl.ts:24`
  - body: This import fails on current main because `DoctorHealthFlowContext` is a local, non-exported type in `src/flows/doctor-health-contributions.ts`; CI already reports TS2459, so the PR cannot typecheck until this entry point stops depending on that private flow type or the type is intentionally exported.
  - confidence: 0.98
- **[P2] Warn when partial WSL2 configs omit memory:** `src/commands/doctor-wsl.ts:337-339`
  - body: When `.wslconfig` has a `[wsl2]` section but no `memory` key, this branch skips the visible-memory fallback because `diag.wslconfig` is non-null. A low-memory WSL2 instance with only `swap` or `processors` configured would get no memory warning even though there is no explicit memory cap to inspect.
  - confidence: 0.92
- **[P2] Compare visible memory before rounding:** `src/commands/doctor-wsl.ts:355-356`
  - body: The missing-config path rounds visible memory to whole GB before testing `< 4`, so 3.5-3.99GB rounds to `4` and suppresses the warning. Compare bytes or MB against the 4GB threshold and round only the display value.
  - confidence: 0.9
- **[P2] Hide WSL2 resource limits in WSL1 summaries:** `src/commands/doctor-wsl.ts:379-383`
  - body: The summary appends `.wslconfig` memory and processor values even when `diag.isWSL2` is false. Because those resource caps do not control WSL1, WSL1 users can see misleading active-limit context; gate these summary fields behind `diag.isWSL2`.
  - confidence: 0.86
