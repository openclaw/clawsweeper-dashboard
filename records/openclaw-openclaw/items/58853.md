- **[P2] Warn when partial WSL2 configs omit memory:** `src/commands/doctor-wsl.ts:336-338`
  - body: When `[wsl2]` exists but omits `memory`, `diag.wslconfig` is truthy so this branch never reaches the visible-memory fallback. A low-memory WSL2 VM with a partial config such as `swap=...` will get no memory warning; run the fallback when the memory cap is absent and add a regression test.

  - body: Rounding `wslVisibleMemoryBytes` before checking `< 4` turns values like 3.6 GiB into `4`, suppressing the low-memory note. Compare bytes or MB against 4 GiB first, then round only the displayed value.
  - confidence: 0.93

  - body: The summary appends `.wslconfig` memory/processors regardless of `diag.isWSL2`, but those settings are WSL2-only. If WSL1 can read a Windows-side `.wslconfig`, Doctor will report inactive limits as current environment facts; gate these fields on WSL2.
  - confidence: 0.86
