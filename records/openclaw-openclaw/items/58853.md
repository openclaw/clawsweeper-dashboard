Full review comments:

- **[P1] Remove the private Doctor context import:** `src/commands/doctor-wsl.ts:24`
  - body: `DoctorHealthFlowContext` is declared as an internal type in `doctor-health-contributions.ts` and is not exported on current main, so this type-only import will fail validation for the PR head. The WSL note function does not use the context, so drop the import and parameter or use a local public shape instead.
  - confidence: 0.94
- **[P2] Warn when partial WSL2 configs omit memory:** `src/commands/doctor-wsl.ts:337-339`
  - body: When `.wslconfig` has a `[wsl2]` section but no `memory` value, `diag.wslconfig` is truthy and this branch skips the visible-memory fallback. A low-memory WSL2 VM with only `swap` or `processors` configured can therefore get no warning, even though no explicit memory cap is set.
  - confidence: 0.9
- **[P2] Compare visible memory before rounding:** `src/commands/doctor-wsl.ts:355-356`
  - body: This rounds visible WSL memory to whole GB before checking `< 4`, so limits such as 3.75GB round to 4 and suppress the low-memory note. Compare bytes or MB against the threshold, then round only for display.
  - confidence: 0.88
- **[P3] Hide WSL2-only limits from WSL1 summaries:** `src/commands/doctor-wsl.ts:379-383`
  - body: The diagnostic checks correctly skip `.wslconfig` resource advice for WSL1, but the summary still prints `memory limit` and `processors` whenever a Windows-side `[wsl2]` config is readable. Gate these summary fields on `diag.isWSL2` so WSL1 users do not see limits that do not apply.
  - confidence: 0.78
