Full review comments:

- **[P1] Fix the private Doctor context import:** `src/commands/doctor-wsl.ts:24`
  - body: `DoctorHealthFlowContext` is declared inside `doctor-health-contributions.ts` but is not exported on current main, so this import fails TS2459 in the PR's type/build checks. Since the parameter is unused, remove this dependency or use an intentional public/minimal type instead of importing a private flow type.
  - confidence: 0.97
- **[P2] Warn when partial WSL2 configs omit memory:** `src/commands/doctor-wsl.ts:337-339`
  - body: When `.wslconfig` has a `[wsl2]` section but omits `memory`, this branch treats the config as present and never falls back to `wslVisibleMemoryBytes`. A low-memory WSL2 setup with only `swap` or `processors` configured can therefore get no memory warning; run the visible-memory fallback when the memory limit is absent.
  - confidence: 0.9
- **[P2] Compare visible memory before rounding:** `src/commands/doctor-wsl.ts:355-356`
  - body: This rounds WSL-visible memory to whole GB before applying the `< 4` threshold, so allocations from 3.5GB up to just under 4GB round to `4` and skip the warning. Compare the byte or MB value against the threshold, then round only for display.
  - confidence: 0.9
- **[P3] Hide WSL2-only limits from WSL1 summaries:** `src/commands/doctor-wsl.ts:379-383`
  - body: The resource fields are appended whenever `.wslconfig` is parsed, even if `diag.isWSL2` is false. Because those limits do not control WSL1, gate the memory and processor summary fields on WSL2 so WSL1 users do not see misleading active limits.
  - confidence: 0.77
