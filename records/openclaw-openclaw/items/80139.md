## Summary

Keep this PR open because current main still has the cron execution/delivery status ambiguity and this branch is a useful candidate for https://github.com/openclaw/openclaw/issues/49190, but it is not merge-ready: it expands the public cron status contract, targets stale protocol/run-log surfaces, and has a concrete QA wait bug. ## What This Changes

The PR adds a first-class cron `warning` run status for successful isolated-agent execution with failed final delivery, updating cron runtime state, run logs, Gateway schema, plugin hook types, CLI/UI/docs, QA helper waits, i18n, and tests.

## Best Possible Solution

Maintain one approved cron partial-success contract: either update this branch on current main with `warning` across the current protocol, SQLite, UI, plugin-hook, docs, and QA surfaces, or defer to a narrower `ok` plus `deliveryStatus` path if maintainers choose that model.

## Reproduction Assessment

Yes. Current main is source-reproducible: cron status is only `ok | error | skipped`, delivery-target/direct delivery failures can return `status: "error"`, and scheduler persistence/backoff consumes that returned status.

## Solution Assessment

No, not as submitted. A first-class `warning` status is a plausible broad solution, but the PR must be retargeted to current Gateway/SQLite surfaces, fix the QA predicate, remove the changelog edit, and get maintainer approval for the public status contract.
