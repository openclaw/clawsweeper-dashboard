## Summary

Keep open. Current `main` still has the reported Active Memory chat-type classifier gap: Telegram DM topic sessions use `agent:main:main:thread:{chatId}:{topicId}`, but `resolveChatType` only treats the unthreaded three-part `agent:<agent>:main` shape as `direct`, so the `allowedChatTypes` gate can still skip before recall or transcript persistence.

## What This Changes

Active Memory should treat Telegram DM topic session keys like `agent:main:main:thread:{chatId}:{topicId}` as direct-message sessions instead of silently skipping them.

## Best Possible Solution

Normalize the generic `:thread:` suffix inside Active Memory chat-type classification, then add a focused regression test proving `allowedChatTypes: ["direct"]` runs for the Telegram default-main DM topic key while leaving Telegram session-key generation and core routing unchanged.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current Telegram tests prove the exact session-key shape, and the current Active Memory classifier can be traced to `undefined` for that key before the hook returns early.

## Solution Assessment

Yes. Reusing `parseThreadSessionSuffix` before `resolveChatType` is narrower and safer than falling back from every unknown provider/channel key to `direct`, which could enable Active Memory on future non-direct session shapes.
