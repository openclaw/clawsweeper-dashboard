## Summary

Current main and shipped releases already handle the Bonjour `CIAO PROBING CANCELLED` crash path by keeping ciao rejection handling active across advertiser restarts and by suppressing/recovering from ciao cancellations.

## What This Changes

The issue reports a Windows gateway crash after Bonjour gets stuck probing and emits an unhandled `CIAO PROBING CANCELLED` rejection.

## Best Possible Solution

Keep the shipped Bonjour handler/recovery behavior and ask for a new report only if the crash still occurs on a current release with fresh Windows logs.

## Reproduction Assessment

Yes for the affected behavior: the issue logs, ciao source, and current regression tests establish the old unhandled cancellation path. I did not reproduce a current-main Windows crash because current main contains the fix.

## Solution Assessment

Yes. The current implementation keeps ciao rejection handling scoped to the advertiser lifetime, recovers on ciao cancellation, and cleans handlers only on final shutdown, which is the narrow maintainable fix.
