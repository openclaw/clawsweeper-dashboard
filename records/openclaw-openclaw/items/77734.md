## What This Changes

The issue reports the Windows Gateway exiting every few minutes in OpenClaw 2026.4.24 when the bundled Bonjour/ciao advertiser gets stuck probing and emits an unhandled `CIAO PROBING CANCELLED` rejection.

## Best Possible Solution

Keep the shipped Bonjour hardening; affected users should upgrade to v2026.4.25 or newer, preferably the latest release, and disable Bonjour when LAN mDNS is not needed.

## Reproduction Assessment

No current-main failing repro was established. The report includes concrete 2026.4.24 logs, and current source plus regression tests cover the formerly crashing `CIAO PROBING CANCELLED` path.

## Solution Assessment

Yes. Handling known ciao failures inside the Bonjour plugin, bounding advertiser restarts, and making Windows Bonjour opt-in is the narrow maintainable fix without moving dependency-specific policy into Gateway core.
