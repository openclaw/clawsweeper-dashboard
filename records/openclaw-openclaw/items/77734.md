## Summary

Current main and released builds after the reported 2026.4.24 version already handle the Bonjour `CIAO PROBING CANCELLED` path without crashing the Gateway, with source, tests, docs, and changelog coverage.

## What This Changes

The issue requests that Windows Gateway startup keep running when the bundled Bonjour advertiser gets stuck in probing and emits `CIAO PROBING CANCELLED`.

## Best Possible Solution

Keep the shipped Bonjour-owner recovery path and have affected 2026.4.24 users upgrade to a released build containing the fix; use `OPENCLAW_DISABLE_BONJOUR=1` or `discovery.mdns.mode: "off"` when LAN mDNS is not needed.

## Reproduction Assessment

No current-main failing repro remains. The report has a concrete 2026.4.24 Windows crash path and logs, while current main has source and regression-test coverage for the exact `CIAO PROBING CANCELLED` condition.

## Solution Assessment

Yes. Handling ciao-specific failures inside the Bonjour plugin and disabling only the local advertiser after repeated stuck-state recovery is the narrowest maintainable fix because it preserves the Gateway and avoids adding core crash policy.
