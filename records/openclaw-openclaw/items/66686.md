## Summary

Keep open. Current main still contains broad mandatory memory_search wording in the memory-core prompt and tool metadata, while the live-state guidance is generic and does not explicitly exempt current configuration, service, or time checks.

## What This Changes

The issue asks OpenClaw to avoid mandatory memory_search for current system-state verification questions and verify live state directly instead.

## Best Possible Solution

Refine the memory-core prompt and memory_search descriptions to target historical recall, explicitly route current config/service/time/version/filesystem/package-state questions to live verification first, and add focused prompt regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main still emits mandatory memory_search guidance from both the prompt section and tool descriptions. I did not run a live provider replay of the original conversation.

## Solution Assessment
