## Summary

Keep open. Current main still emits broad mandatory memory-recall guidance from memory-core prompt/tool metadata, while the generic live-check guidance does not explicitly route current config, service, clock, version, file, process, or package-state questions away from memory search first.

## What This Changes

The issue asks OpenClaw to distinguish historical memory recall questions from current system-state verification questions so `memory_search` is not prompted first for live status/config/time checks.

## Best Possible Solution

Refine memory-core prompt and `memory_search` descriptions so historical recall stays explicit, while current configuration, service health, clock/time, version, filesystem, process, and package-state questions are directed to live verification first.

## Reproduction Assessment

Yes. Source inspection on current main shows the same mandatory memory-search wording in both prompt and tool metadata, so the confusing instruction path is reproducible without relying on a particular live model run.

## Solution Assessment

Yes. A narrow wording change plus regression tests is the best path; a new config option, plugin API, or broader prompt rewrite is not needed for the reported behavior.
