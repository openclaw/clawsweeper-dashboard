## Summary

Keep open. Current main still contains broad mandatory `memory_search` wording in the memory-core prompt and both tool descriptions, while the live-check guidance remains generic and does not explicitly carve out current configuration, service, or time checks.

## What This Changes

The issue asks OpenClaw to distinguish historical memory recall from current system-state verification so questions like configured agents, running services, or current time go to live verification first.

## Best Possible Solution

Refine memory-core prompt and `memory_search` descriptions to target historical recall, explicitly route current config/service/time/version/filesystem/package-state questions to live verification first, and add focused prompt regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main still emits mandatory `memory_search` guidance from the prompt section and both tool descriptions, though I did not capture a live model trajectory.

## Solution Assessment

Yes. A narrow prompt/tool wording change plus regression tests is the best first fix; broader prompt rewrites, new config, or plugin API changes are unnecessary for the reported behavior.
