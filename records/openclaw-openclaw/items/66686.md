## Summary

Keep open. Current main still contains broad mandatory `memory_search` wording in both the memory-core prompt and tool descriptions, and the existing live-check guidance is generic rather than an explicit exception for current configuration, service, or time checks.

## What This Changes

The issue asks to narrow memory recall guidance so current system-state and configuration questions use live verification instead of mandatory memory search.

## Best Possible Solution

Refine memory-core prompt and `memory_search` descriptions to target historical recall, explicitly send current config/service/time/version/filesystem/package-state questions to live verification first, and add focused prompt regression coverage.
