## Summary

Keep open: this PR targets a valid proxy regression, but it is not merge-ready because the new dispatcher wrapper would send ALL_PROXY-only deployments directly instead of through the configured proxy, and the env-proxy timeout rebuild path still drops the wrapper. The contributor also has not shown after-fix real gateway or qqbot/COS behavior proof.

## What This Changes

The PR wraps OpenClaw’s global Undici EnvHttpProxyAgent with a dispatcher proxy that bypasses the proxy for requests matched by OpenClaw’s enhanced proxy-bypass matcher.

## Best Possible Solution

Use a global-dispatcher-specific proxy predicate or factory that preserves HTTP_PROXY, HTTPS_PROXY, and ALL_PROXY option parity, applies enhanced proxy-bypass matching only for true bypass matches, and is reused by bootstrap, timeout rebuild, and reset paths with dispatch tests and redacted real runtime proof.

## Reproduction Assessment

Yes. With only ALL_PROXY set, current main installs proxy options, but the PR's shouldUseEnvHttpProxyForUrl() check returns false and dispatches direct; the linked issue also provides user logs for the original qqbot COS failure.

## Solution Assessment

No. Wrapping the global EnvHttpProxyAgent is a plausible fix shape, but this implementation reuses a provider helper with different ALL_PROXY semantics and does not preserve the wrapper through existing env-proxy rebuild paths.
