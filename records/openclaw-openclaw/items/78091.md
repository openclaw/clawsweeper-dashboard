## Summary

Keep open: current main still creates a random OpenAI-compatible session when requests lack `x-openclaw-session-key` or OpenAI `user`, and the linked closing pull request is still open rather than merged.

## What This Changes

Make OpenAI-compatible Chat Completions reuse a stable Open-WebUI chat session instead of generating a random session per message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78124, or a replacement that reads Open-WebUI's stable forwarded chat id, documents precedence, adds regression coverage, and includes real multi-turn proof.

## Reproduction Assessment

Yes for the OpenClaw side: source inspection shows a Chat Completions request without `x-openclaw-session-key` or `user` uses `randomUUID()`. I did not run a live Open-WebUI container, but upstream source shows the stable chat id can be forwarded as `X-OpenWebUI-Chat-Id` when enabled.

## Solution Assessment

Yes in direction: an explicit client-supplied chat/session id is the maintainable solution, not a global default session or IP/user-agent hash. The linked PR is the right review path, but it still needs the actual Open-WebUI header contract, precedence agreement, docs, and real behavior proof.
