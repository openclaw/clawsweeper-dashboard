## Summary

Keep open: current main still routes TUI final and fallback streamed assistant output through `resolveFinalAssistantText` without applying the existing GLM-5/DeepSeek special-token sanitizer, and the only related fix PR is closed unmerged.

## What This Changes

Apply the existing model-special-token stripping behavior to the main TUI chat output path so GLM-5/DeepSeek `<|...|>` tokens do not appear in visible assistant replies.

## Best Possible Solution

Wire the existing shared assistant-visible/model-token sanitizer into the TUI final and streamed-fallback output boundary, preserve the current error-message path, and add focused regressions for ASCII tokens, full-width tokens, streamed fallback finalization, and code-region preservation.
