## Summary

Keep open for ordinary maintainer merge review: current main still has the unbounded MiniMax OAuth success-body read, and this PR narrowly routes that read through the existing bounded provider JSON helper with sufficient terminal proof and no blocking findings.

## What This Changes

The PR changes `extensions/minimax/oauth.ts` so the MiniMax OAuth authorization-code success response uses `readProviderJsonResponse` instead of direct `response.json()`.

## Best Possible Solution

Merge the OAuth-only helper reuse after maintainer approval; keep the broader provider JSON hardening work tracked by its own focused PRs.

## Reproduction Assessment

Yes. Current main still uses direct `response.json()` on the MiniMax OAuth authorization-code success path, and the PR body provides terminal loopback proof that the bounded helper rejects and cancels oversized streams while the old path buffers past the cap.

## Solution Assessment

Yes. Reusing the existing plugin-SDK `readProviderJsonResponse` helper is the narrowest maintainable fix because the shared helper already owns the cap, cancellation behavior, and malformed JSON wrapping; a MiniMax-local reader would duplicate that contract.
