Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve the TTS provider JSON reader:** `extensions/minimax/tts.ts:112-115`
  - body: Current main already reads MiniMax TTS success JSON through `readProviderJsonResponse`, which defaults to a 16 MiB provider JSON cap and wraps malformed JSON. This replacement caps the whole TTS response at 1 MiB even though the response contains `data.audio` as a hex payload before returning `Buffer.from(hexAudio, "hex")`, so normal speech output above roughly 512 KiB will fail before synthesis.
  - confidence: 0.9
- **[P1] Size the image JSON cap for inline images:** `extensions/minimax/image-generation-provider.ts:169-172`
  - body: This provider requests `response_format: "base64"`, supports up to nine images, and decodes `data.image_base64` from the same JSON body. A fixed 1 MiB cap on the entire success response will reject ordinary inline image results; this should use the image-sized cap path or be left to https://github.com/openclaw/openclaw/pull/96495.
  - confidence: 0.89
- **[P2] Keep the new test lint-clean:** `extensions/minimax/video-generation-provider.test.ts:256`
  - body: The PR head currently fails the extension lint shard on this one-line `if` because repo lint requires braces after conditions. Add braces around the early return before merge.
