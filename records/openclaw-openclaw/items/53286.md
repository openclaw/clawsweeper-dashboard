## Summary

Keep open. Current `main` still suppresses display-cost resolution behind auth-mode checks in both `/status` and the per-response usage footer, so explicit Bedrock `aws-sdk` model pricing remains token-only. The linked fix PR #53303 is closed unmerged, and #29100 tracks broader Bedrock discovery pricing rather than this explicit-cost display bug.

## What This Changes

Request that `/status` and `/usage full` estimate costs for providers such as Amazon Bedrock using `aws-sdk` auth when model cost fields are explicitly configured.

## Best Possible Solution

Implement a narrow display-layer fix that resolves explicit nonzero configured model pricing before auth filtering, shows USD estimates for Bedrock `aws-sdk` when an estimate can be computed, leaves #29100’s auto-discovery pricing overlay separate, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes. Configure `amazon-bedrock` with `auth: "aws-sdk"`, nonzero `models[].cost`, and response usage; current source shows `resolveModelAuthMode` returns `aws-sdk`, then both display paths keep `showCost` false before pricing is read.

## Solution Assessment

Yes. The narrow maintainable solution is to decouple display eligibility from auth mode while still requiring explicit usable pricing; it should not change AWS SDK auth semantics or broaden Bedrock model discovery.
