## Summary

Keep open. Current `origin/main` still suppresses the two user-visible cost display paths behind auth-mode checks, so an explicitly priced Amazon Bedrock provider using `aws-sdk` auth remains token-only; the linked fix PR is closed unmerged, and the related Bedrock discovery issue covers a broader, separate pricing overlay.

## What This Changes

Show estimated USD cost in `/status` and `/usage full` for providers such as Amazon Bedrock when explicit nonzero `models.providers.*.models[].cost` pricing is configured, regardless of auth method.

## Best Possible Solution

Implement a narrow display-layer fix that resolves explicit configured pricing before auth filtering, shows USD only when a usable estimate can be computed, keeps missing-price and subscription/OAuth token-only behavior intact, and leaves #29100’s auto-discovery pricing overlay as separate work.

## Reproduction Assessment

Yes. Configure `amazon-bedrock` with `auth: "aws-sdk"`, nonzero explicit model `cost`, and response usage; current source resolves that provider to `aws-sdk`, then both `/status` and `/usage full` keep `showCost` false before pricing is used.

## Solution Assessment

Yes. The maintainable fix is to decouple display eligibility from auth mode while still requiring explicit usable pricing; it should not change AWS SDK authentication, provider discovery, or remote pricing bootstrap behavior.
