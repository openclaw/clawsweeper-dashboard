## Best Possible Solution

Land a focused replacement PR that shows estimated cost when explicit nonzero pricing exists for the active provider/model, with status and response-usage tests plus docs updates, while leaving Bedrock discovery cost overlays to the separate open issue.

## Solution Assessment

Yes for a focused replacement. The narrow maintainable fix is to resolve explicit pricing separately from auth-mode display policy and show cost only when usable nonzero pricing exists, not to revive the broad closed PR or add Bedrock discovery overlays here.
