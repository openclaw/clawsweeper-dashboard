## Summary

Keep open. Current main still suppresses estimated USD display for aws-sdk providers before explicit model pricing is read, and the focused fix has not landed; the previous implementation PR was closed unmerged after maintainer feedback about unrelated churn, while #29100 tracks a broader auto-discovery pricing overlay rather than this display-layer bug.

## What This Changes

Show estimated USD costs in `/status` and `/usage full` whenever a model has explicit nonzero `models.providers.*.models[].cost` pricing, regardless of provider auth mode such as Bedrock `aws-sdk`.

## Best Possible Solution

Implement a narrow display-layer fix: resolve explicit model pricing before auth filtering, show USD estimates when usable nonzero pricing exists for Bedrock aws-sdk and other non-API-key providers, preserve token-only output for missing or all-zero pricing, and keep the broader Bedrock auto-discovery pricing overlay in #29100 separate.

## Reproduction Assessment

Yes. A static current-main path is enough: configure `amazon-bedrock` with `auth: "aws-sdk"`, nonzero `models[].cost`, and response usage; `resolveModelAuthMode` returns `aws-sdk`, then both `/status` and `/usage full` keep `showCost` false before pricing is resolved.

## Solution Assessment

Yes. Decoupling display eligibility from auth mode while still requiring usable pricing is the narrowest maintainable fix; auto-discovered Bedrock pricing metadata should remain a separate product/API change tracked by #29100.
