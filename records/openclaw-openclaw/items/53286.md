## Summary

Keep open. Current main still gates `/status` and `/usage full` cost display on API-key-style auth before configured model pricing can affect the visible output, so explicit Amazon Bedrock `aws-sdk` pricing remains token-only. The linked source PR #53303 is closed unmerged, and #29100 tracks a broader Bedrock discovery pricing overlay rather than this explicit-provider display gap.

## What This Changes

Show estimated USD costs in `/status` and `/usage full` for Amazon Bedrock `aws-sdk` providers when explicit model `cost` config is present.

## Best Possible Solution

Add narrow display eligibility for explicit, usable configured pricing on Bedrock `aws-sdk`, preserve OAuth/token subscription cost hiding, and leave #29100's auto-discovery pricing overlay as separate work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `amazon-bedrock` with `auth: "aws-sdk"`, nonzero `models[].cost`, and response usage; both visible display paths keep `showCost` false before the cost lookup can run.

## Solution Assessment

Yes for the narrow Bedrock `aws-sdk` explicit-pricing display change. The broader “all auth methods” wording is not the best default because current docs intentionally hide OAuth/token dollar costs.
