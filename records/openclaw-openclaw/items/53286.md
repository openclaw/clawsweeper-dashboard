## Best Possible Solution

Land a focused replacement PR that displays cost when explicit nonzero pricing exists for the active provider/model regardless of auth mode, with status and reply-usage tests plus token-use docs updates, while leaving Bedrock discovery pricing overlays to https://github.com/openclaw/openclaw/issues/29100.

## Solution Assessment

Yes for a focused replacement. The narrow maintainable fix is to resolve explicit pricing separately from auth-mode display policy and show cost only when usable nonzero pricing exists, not to revive the broad closed PR or add Bedrock discovery overlays here.
