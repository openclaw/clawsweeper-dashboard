## Summary

Keep this issue open: current main and the latest release still route task and Task Flow persistence through `node:sqlite`, and the linked fallback PR was closed unmerged after build and JSON-store safety concerns.

## What This Changes

The issue requests graceful Gateway task persistence when `node:sqlite` is missing in a Homebrew Node runtime.

## Best Possible Solution

Choose and implement a supported runtime-safe persistence strategy for both registries, preserving SQLite where available and adding a hardened documented fallback or supported dependency where it is not.

## Reproduction Assessment

Yes for a source repro: run a Node runtime where `require("node:sqlite")` throws, then load/upsert task registry state or run task maintenance. Current main still reaches `requireNodeSqlite()` through both task registry and Task Flow default stores; the local Node runtime has `node:sqlite`, so I did not reproduce the Homebrew-specific runtime locally.

## Solution Assessment

Unclear. The suggested `better-sqlite3` replacement or JSON fallback could solve the crash, but the closed fallback PR shows why maintainers should first choose the backend, packaging, durability, locking, validation, and path-safety contract.
