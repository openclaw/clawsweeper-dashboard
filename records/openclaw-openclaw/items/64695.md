## Summary

Keep this open: current main and the latest shipped release still route both task registry and Task Flow persistence through `node:sqlite` without an availability probe or fallback, and the linked JSON fallback PR closed unmerged after build and durability/security concerns.

## What This Changes

The issue reports gateway crashes during task maintenance on Homebrew Node.js builds missing `node:sqlite`, and asks for a runtime-safe persistence fallback or replacement backend.

## Best Possible Solution

Add a maintainer-approved runtime-safe storage strategy or explicit unsupported-runtime gate for both registries, with durable semantics, hardening, tests, and public docs aligned.

## Reproduction Assessment

Yes for source-level reproduction: current default task and Task Flow stores call `requireNodeSqlite()` on load/upsert paths, so a runtime where `require('node:sqlite')` fails will still hit the reported failure path. I did not run the exact Homebrew Node bottle from the report.

## Solution Assessment

Unclear. The requested fallback direction is valid, but the closed JSON fallback PR shows the backend choice needs maintainer design on locking, filesystem safety, durability, and packaging before implementation.
