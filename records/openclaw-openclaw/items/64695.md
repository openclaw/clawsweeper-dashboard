## Summary

Keep this open: current main and the latest release still route task and Task Flow persistence through `node:sqlite`, while the only linked fallback PR was closed unmerged and raised durability/security concerns.

## What This Changes

Ensure Gateway task registry and Task Flow persistence do not crash on supported installs whose Node runtime lacks `node:sqlite`, such as the reported Homebrew Node build.

## Best Possible Solution

Add a maintainer-approved storage selection layer for both task and Task Flow registries that keeps SQLite where supported and uses a hardened, documented fallback or supported dependency when `node:sqlite` is unavailable.

## Reproduction Assessment

Yes, as a source repro: run a Node runtime where `require("node:sqlite")` throws, then trigger task restore/upsert or the maintenance sweep path. Current main still reaches `requireNodeSqlite()` from those paths, though I did not run the exact Homebrew Node v23.3.0 build here.

## Solution Assessment

Unclear. The suggested `better-sqlite3` replacement or JSON fallback could address the crash, but the closed fallback PR showed why maintainers need to choose the backend, packaging, durability, and security contract before implementation.
