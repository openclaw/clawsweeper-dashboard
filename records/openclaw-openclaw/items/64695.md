## Summary

Keep this open: current main and v2026.5.12 still depend on `node:sqlite` for task and Task Flow persistence, so runtimes missing that builtin can still fail restore/upsert durability paths even though scheduled sweep errors are now contained.

## What This Changes

The issue asks OpenClaw to avoid gateway crashes when task registry maintenance runs on a Node runtime without the `node:sqlite` builtin.

## Best Possible Solution

Add a maintainer-approved runtime storage strategy that either probes `node:sqlite` and uses a hardened durable fallback where unavailable, or fails early with documented unsupported-runtime guidance before the gateway enters periodic maintenance.

## Reproduction Assessment

Yes for source reproduction: current main's default task and Task Flow stores still call `requireNodeSqlite()` on restore and persistence paths, so the reported runtime missing that builtin would throw. I did not execute the exact Homebrew Node bottle from the issue.

## Solution Assessment

Unclear. The requested runtime-safe persistence direction is valid, but choosing between a native dependency, JSON fallback, or an explicit unsupported-runtime gate needs maintainer judgment on packaging, locking, and durability.
