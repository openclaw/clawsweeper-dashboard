The PR carries Discord reply kind through delivery sanitization, preserves explicit Discord tool-progress payloads, forwards queued follow-up progress/tool-result events, refreshes a Swift gateway model field, and adds focused regression tests.

## Best Possible Solution

Keep the PR direction, but make queued follow-up progress use the same verbose-overrides-preview-suppression policy as direct dispatch and add the missing queued suppress-plus-verbose regression case.

## Reproduction Assessment

Yes, with source-level confidence. Current main defines the verbose-overrides-preview-suppression contract, Discord draft preview can set suppression while wiring progress callbacks, and the PR body supplies redacted live Discord canary output for the after-fix behavior.

## Solution Assessment

No. The overall approach is narrow and maintainable, but the queued follow-up gates currently duplicate the suppression policy incorrectly and still hide verbose progress when draft preview suppression is set.
