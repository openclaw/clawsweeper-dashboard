## Summary

Keep this PR open: it targets a real Discord verbose-progress regression and the overall direction is useful, but the latest head still suppresses queued follow-up progress whenever Discord preview suppression is set, even when verbose mode is on.

## What This Changes

The PR preserves Discord tool-progress payloads through delivery sanitization, forwards queued follow-up progress/tool-result events, and adds focused regression tests.

## Best Possible Solution

Keep the PR direction, but make queued follow-up progress mirror the direct dispatch verbose/suppression policy and add a regression for preview suppression with verbose enabled.

## Reproduction Assessment

Yes, source-level: current main strips Discord tool-looking payload text and queued follow-ups only track compaction, while the PR body supplies live Discord canary output for the intended fixed behavior. The remaining review finding is also source-reproducible with a queued follow-up using suppressDefaultToolProgressMessages: true and verboseLevel: "on".

## Solution Assessment

No, not yet. The sanitizer and queued-forwarding approach is narrow, but the queued gates need to reuse the direct verbose-overrides-preview-suppression semantics instead of treating preview suppression as a hard block.
