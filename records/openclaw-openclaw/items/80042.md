## Summary

Keep open: the PR is a real, narrow Discord verbose-progress fix, but the latest head still over-suppresses queued follow-up progress when Discord draft preview sets progress suppression while verbose mode is enabled.

## What This Changes

## Best Possible Solution

Keep the PR direction, but make queued follow-up progress share the direct-dispatch verbose-overrides-preview-suppression policy and add the missing queued regression case while preserving compaction accounting.
