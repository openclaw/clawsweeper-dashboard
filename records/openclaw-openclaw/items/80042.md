## Summary

Keep open: the PR is a plausible fix for the Discord verbose progress regression, but the latest head still suppresses queued follow-up progress whenever preview suppression is set, even when verbose mode is enabled.

## What This Changes

The PR preserves Discord tool-progress payloads through delivery sanitization, forwards queued follow-up progress/tool-result events, refreshes a Swift gateway model field, and adds focused regression tests.

## Best Possible Solution

Queued follow-up progress should reuse the direct dispatch policy: verbose progress overrides preview suppression, verbose-off remains suppressed, and compaction accounting still runs.

## Reproduction Assessment

Yes, with source-level confidence. PR head lines 467-490 show the remaining queued suppression path, and current main tests define that verbose mode should override preview suppression.

## Solution Assessment

No. The PR direction is narrow, but the queued follow-up gates should match the direct-dispatch suppression helper and include a regression for suppressDefaultToolProgressMessages with verboseLevel on.
