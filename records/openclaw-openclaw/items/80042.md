## Summary

## What This Changes

## Best Possible Solution

Keep the sanitizer and queued-forwarding direction, but align queued follow-up suppression with direct dispatch and add the verbose-on plus preview-suppression regression before merge.

Yes. Current main shows the missing queued forwarding and sanitizer behavior from source, and the PR head still has a source-reproducible queued suppression gap when preview suppression is true and verbose mode is on.

## Solution Assessment

No, not yet. The implementation is narrow and mostly in the right owner boundary, but queued follow-up gates need to mirror direct dispatch's verbose-overrides-preview-suppression semantics.
