## Summary

Keep open: this PR addresses a real Discord verbose-progress regression linked from https://github.com/openclaw/openclaw/issues/78365, but the current patch is not merge-ready because queued follow-up progress forwarding bypasses the existing verbose/suppression gates and the external contributor has not provided real Discord behavior proof.

## What This Changes

The PR threads reply kind into Discord delivery sanitization, relaxes message-tool-only progress delivery, and adds queued follow-up progress forwarding plus regression tests.

## Best Possible Solution

Keep the PR open, gate queued follow-up progress forwarding with the same verbose and suppressDefaultToolProgressMessages policy while preserving compaction accounting, then require real Discord proof before merge.

## Reproduction Assessment

Yes from source for the review finding: the PR forwards queued onAgentEvent progress callbacks even though current tool handlers emit those callbacks outside shouldEmitToolResult. I did not run a live Discord repro because this is a read-only review and pnpm is unavailable in the checkout.

## Solution Assessment

No. The direction is right, but the queued follow-up helper should apply the same verbose/suppression gates before forwarding user-visible progress events, while still counting compaction events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate queued progress forwarding by verbose policy:** `src/auto-reply/reply/followup-runner.ts:477`
  - body: The queued follow-up runner now forwards every embedded `onAgentEvent` to channel progress callbacks, but those events are emitted even when `shouldEmitToolResult()` is false. In `message_tool_only` follow-ups with `verboseLevel: "off"` or `suppressDefaultToolProgressMessages`, this can still drive Discord tool/item/command progress output that should remain suppressed; gate this forwarding with the same verbose/suppression policy while keeping compaction accounting active.
  - confidence: 0.9
