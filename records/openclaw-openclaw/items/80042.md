## Summary

Keep open: this targets a real Discord verbose-progress regression and includes after-fix live output, but the latest head still suppresses queued verbose progress when draft preview sets progress suppression even though current direct-dispatch behavior lets verbose mode override that suppression.

## What This Changes

The PR preserves Discord tool-progress payloads through reply sanitization, forwards queued follow-up progress/tool-result events, passes tool progress detail into follow-up runs, refreshes a Swift gateway model field, and adds regression tests.

## Best Possible Solution

Keep the PR direction, but make queued follow-up progress share the direct-dispatch verbose-overrides-preview-suppression policy and add the missing queued regression while preserving compaction accounting.

## Reproduction Assessment

Yes, with source-level confidence rather than a local live run. Current main and existing tests show the expected verbose override policy, and the PR body supplies redacted live Discord canary output for the after-fix behavior.

## Solution Assessment

No. The overall approach is narrow, but the queued follow-up gates duplicate the wrong suppression semantics and should match the established direct-dispatch policy.
