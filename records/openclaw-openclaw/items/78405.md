## Summary

Keep open: current main still suppresses normal final text in group/channel `message_tool` mode with only a verbose pre-run note, while recent Discord, WhatsApp, and Telegram reports show the remaining operator-visible gap is real and cross-channel.

## What This Changes

The issue requests better observability when `messages.groupChat.visibleReplies: "message_tool"` suppresses a normal text final because the agent did not call the `message` tool.

## Best Possible Solution

Add a scoped, deduped diagnostic for suppressed non-empty final text in `message_tool_only` mode, while preserving current delivery semantics and leaving fallback/config changes for separate maintainer policy work.

## Reproduction Assessment

Yes, at source level: current main resolves group/channel turns to `message_tool_only`, sets `suppressDelivery`, and tests assert no final dispatcher call for normal text. I did not execute a live channel repro in this read-only review.

## Solution Assessment

Yes for the diagnostic subset: a terminal suppressed-final warning is the narrowest maintainable improvement and avoids changing private-by-default delivery. Auto-fallback and per-agent override ideas need separate policy decisions.
