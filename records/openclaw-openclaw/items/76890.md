## Summary

Keep open: current main still has a source-backed path from memory-core session-start warming into full session transcript indexing on the Gateway process, and the linked fix PR #76978 is still open with closing syntax.

## What This Changes

The issue reports gateway event-loop starvation during memory-core session-start indexing for large session corpora.

## Best Possible Solution

Land or revise #76978 so memory-core session sync yields, defers, targets, or offloads heavy transcript indexing while preserving existing session source semantics and adding focused regression coverage.

## Reproduction Assessment

No live reproduction was run. Source inspection gives a clear current-main path from session-start memory warming into full session-file enumeration and indexing on the Gateway process without a session-loop macrotask yield.

## Solution Assessment

Yes: the narrow maintainable fix is inside memory-core session sync responsiveness, not a new config option. A blanket master-toggle gate needs care because current behavior intentionally allows some session ingestion paths even when memorySearch is disabled.
