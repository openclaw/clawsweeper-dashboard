## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76890. Current main still lets MemoryIndexManager.warmSession() schedule sync({ reason: "session-start" }); runSync() can enter needsFullReindex and runSafeReindex(), and shouldSyncSessionsForReindex() currently allows full session reindexes before the session-start/watch skip, so syncSessionFiles() can enumerate and index the whole session corpus on the Gateway process. Keep the fix inside memory-core/session sync and nearby Gateway startup/memory tests. Do not add a new user-facing setting, do not change provider defaults, and do not move plugin-owned behavior into generic Gateway code. Preserve explicit/manual/boot full reindex behavior, but make session-start work nonblocking and avoid full session-corpus indexing on the event loop; also ensure memorySearch.enabled=false prevents this path. Add focused regression tests and a user-facing CHANGELOG.md fix entry.
