## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76890. The observable bug is that opening a brand-new session can trigger memory-core session-start work that blocks the Gateway event loop while session memory performs full-corpus indexing. Keep the fix inside memory-core/session sync unless a generic core seam is strictly necessary. Ensure `agents.defaults.memorySearch.enabled=false` and plugin-slot disablement boundaries are respected, avoid full session-corpus reindex from `warmSession()`/`reason: "session-start"` on the main Gateway event loop, and add focused regression coverage showing session-start warmup returns promptly or defers/yields heavy indexing. Do not add a new user-facing config option or move memory-core policy into core. Add a user-facing changelog entry if runtime behavior changes.
