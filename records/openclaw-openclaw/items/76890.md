## Summary

Keep open: current main still has a source-backed path from memory-core session-start warming into full transcript indexing without a session-loop yield, and the linked implementation PR #76978 is open and unmerged.

## What This Changes

memory-core session-start indexing is reported to block the gateway event loop during large session transcript sync work.

## Best Possible Solution

Review and land or revise #76978 so memory-core session sync yields, defers, targets, or offloads heavy transcript indexing while preserving session source semantics and adding focused regression coverage.

## Reproduction Assessment

No live reproduction was run. Source inspection gives a clear current-main path from session-start memory warming into full session-file enumeration and indexing on the gateway process without a session-loop macrotask yield.

## Solution Assessment

Yes: the narrow maintainable fix belongs in memory-core session sync responsiveness, matching the direction of #76978. A new config option or disabling session indexing would change behavior instead of fixing the blocking path.
