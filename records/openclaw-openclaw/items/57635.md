## Summary

Keep open: this is a valid Mattermost upgrade-regression fix direction, but the patch misses the generated bundled channel metadata path used by packaged validation and still lacks real CLI/gateway proof, so it should not merge as-is.

## What This Changes

The PR adds Mattermost schema, type, changelog, and regression-test changes to accept legacy `channelOverrides`, `sessionPolicy`, `attachments`, and aligned `groups` config keys.

## Best Possible Solution

Refresh the generated bundled channel metadata, add full config-validation coverage for the legacy Mattermost keys, then land after redacted `openclaw status`, gateway-start, or equivalent legacy-config proof is supplied.

## Reproduction Assessment

Yes. Source inspection shows current main's strict Mattermost schema and generated channel metadata omit `channelOverrides`, `sessionPolicy`, and `attachments` while `validateConfigObjectWithPlugins` validates channels through generated JSON schema, so a legacy config with those keys follows the reported rejection path.

## Solution Assessment

No, not yet. Explicitly whitelisting the known legacy fields in the Mattermost plugin schema is the right approach, but this PR also needs generated metadata/full validation coverage and real behavior proof before merge.
