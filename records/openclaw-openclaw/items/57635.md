## Summary

Keep this PR open: it targets a real Mattermost upgrade regression, but current main only partially supersedes it, the branch is now conflicting, after-fix real behavior proof is missing, and the patch accepts group config fields the runtime does not enforce.

## What This Changes

The PR expands the strict Mattermost config schema, types, and schema tests to accept legacy top-level and account-scoped keys including `groups`, `channelOverrides`, `sessionPolicy`, and `attachments`.

## Best Possible Solution

Rebase onto current main and replace broad passthrough schema acceptance with a narrow, tested Mattermost compatibility path that only accepts fields the runtime enforces or migrates/removes legacy no-op keys via doctor.

## Reproduction Assessment

Yes, source-reproducible: current main's strict Mattermost schema still lacks `channelOverrides`, `sessionPolicy`, and top-level `attachments`, so configs containing those legacy keys will be rejected. I did not execute the test suite because `pnpm` is unavailable in this read-only checkout.

## Solution Assessment

No. The PR should not merge as written because it broadens runtime validation for group fields that Mattermost does not enforce; a safer path is a rebase plus a narrow schema or doctor repair with real upgrade proof.
