## Summary

Keep open: the reported Mattermost config regression is valid and the schema direction is right, but the patch still misses the generated bundled channel metadata used by packaged config validation and only supplies test/typecheck proof.

## What This Changes

The PR adds Mattermost schema, type, regression-test, and changelog changes to accept legacy `channelOverrides`, `sessionPolicy`, `attachments`, and aligned `groups` config keys.

## Best Possible Solution

Regenerate the bundled channel config metadata from the updated Mattermost schema, add full `validateConfigObjectWithPlugins` coverage for the legacy keys, and land after redacted CLI or gateway proof is supplied.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a legacy Mattermost config with `channelOverrides`, `sessionPolicy`, or `attachments` reaches `validateConfigObjectWithPlugins`, whose generated Mattermost JSON schema still lacks those keys on current main.

## Solution Assessment

No, not yet. Explicitly whitelisting the known legacy fields in the Mattermost plugin schema is the right narrow fix, but the PR must also update generated metadata and prove the full config-validation path.
