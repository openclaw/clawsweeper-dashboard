## Summary

Keep open: the patch addresses a real prompt-visible provider-key leak, but it currently makes an upgrade-sensitive auth change by stripping models.json-only plaintext credentials, and several active Layer 1 PRs overlap this same fix.

## What This Changes

The PR removes plaintext apiKey persistence from generated models.json by dropping merge-preserved keys, adding a final provider redaction pass, skipping plaintext auth-profile projection, and updating focused agents tests.

## Best Possible Solution

Land one canonical Layer 1 implementation that strips prompt-visible plaintext while either migrating or preserving models.json-only credentials until a documented, tested fail-closed upgrade path exists.

## Reproduction Assessment

Yes. Source inspection on current main shows merge-mode preservation of plaintext apiKey values and direct serialization into models.json; runtime auth also accepts models.json apiKey as a credential source.

## Solution Assessment

No, not quite. The final redaction boundary is the right shape, but unconditional stripping needs upgrade-safe credential handling or an explicit maintainer decision that stale models.json-only auth should fail closed.
