## Summary

Keep open. Current main and the latest release still preserve some plaintext provider `apiKey` values into generated `models.json`, while this focused PR is a plausible Layer 1 fix that now needs rebase, duplicate-branch selection, and upgrade-auth policy review before merge.

## What This Changes

## Best Possible Solution

Rebase one canonical implementation onto current main, strip prompt-visible plaintext from every generated model-catalog serialization path, preserve documented non-secret markers, and prove both fresh-install and stale-upgrade auth behavior.

## Solution Assessment
