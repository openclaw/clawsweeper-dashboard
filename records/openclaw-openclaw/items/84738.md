## Summary

Keep open: current main and the latest release still preserve prompt-visible plaintext provider keys in generated model-catalog paths, and this PR is a meaningful candidate, but it is currently conflicted and its unconditional stripping can break custom provider model rows or stale upgrade auth.

## What This Changes

The PR removes merge-mode plaintext `apiKey` preservation, strips plaintext provider keys before generated `models.json` serialization, skips plaintext auth-profile key projection, and adds focused agents tests.

## Best Possible Solution

Land one rebased canonical fix that sanitizes every prompt-visible generated catalog output while preserving non-secret markers or migrating plaintext credentials so custom/provider auth still loads after upgrade.
