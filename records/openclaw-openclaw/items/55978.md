## Summary

Keep open: current main still has the hardcoded Cron form sticky offset, and this PR is a narrow, proof-backed fix for that mismatch. It still needs maintainer UX judgment because an open alternative removes sticky behavior entirely, and the branch needs update work before merge.

## What This Changes

The PR changes the Cron workspace form CSS from a hardcoded `74px` sticky offset to `top: 0` with a `--shell-topbar-height` max-height calculation, plus a CSS assertion and changelog entry.

## Best Possible Solution

Choose the intended desktop Cron editor behavior; if the editor should stay pinned, rebase and land this variable-backed sticky fix, otherwise close it in favor of the normal-flow alternative.
