## Summary

Keep this PR open: current main still has the stale Cron form sticky offset, and the PR is a narrow, proof-backed fix for that mismatch. It should not be closed as cleanup, but it needs maintainer UX direction against the open normal-flow alternative and a branch refresh before merge.

## What This Changes

This PR replaces the Cron workspace form's hardcoded sticky offset with `top: 0`, uses `--shell-topbar-height` in its `max-height`, and adds a CSS assertion test plus changelog entry.

## Best Possible Solution

Keep a single chosen desktop Cron editor layout: land this CSS-variable sticky fix if the editor should remain pinned, or land the normal-flow alternative and close this branch after that decision.
