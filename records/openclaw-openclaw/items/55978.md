## Summary

This PR should stay open because current main still has the reported Cron form sticky-offset mismatch and the proposed patch is a narrow CSS/test/changelog fix with no blocking correctness finding. The remaining decision is a maintainer UX choice between preserving the sticky editor with this corrected offset and using the open normal-flow alternative.

## What This Changes

The branch changes the Cron workspace form CSS from a hardcoded `74px` sticky offset to `top: 0` with `--shell-topbar-height` in `max-height`, adds a CSS regression assertion, and adds a changelog entry.

## Best Possible Solution

Choose the intended desktop Cron editor interaction, then either land this variable-backed sticky fix or close it in favor of the normal-flow panel PR if the pinned editor is no longer desired.

## Reproduction Assessment

Yes, at source-and-proof level: current main still has a `74px` sticky offset while the shell topbar variable is `52px`, and the PR includes before/after recordings for `/cron`. I did not run a fresh live browser reproduction in this read-only review.

## Solution Assessment

Yes for the reported offset mismatch: `top: 0` relative to the content scroller and `--shell-topbar-height` in the height calculation is the narrowest sticky-editor fix. The broader sticky versus normal-flow UX choice remains unresolved because https://github.com/openclaw/openclaw/pull/47582 targets the same panel differently.
