## Summary

Keep open: current main still has the stale Cron form sticky offset, and this PR is a focused CSS/test/changelog fix with real recording proof; the remaining blocker is maintainer UX choice between keeping the editor sticky with this offset fix or taking the open normal-flow alternative.

## What This Changes

This PR changes the Cron workspace form CSS from a hardcoded 74px sticky offset to `top: 0` with `--shell-topbar-height` in `max-height`, adds a CSS assertion, and adds a changelog entry.

## Best Possible Solution

Choose the intended desktop Cron editor interaction, then land this variable-backed sticky fix if the editor should remain pinned or close it in favor of the normal-flow PR if sticky behavior is no longer wanted.

## Reproduction Assessment

Yes, at source-and-proof level: current main still has `top: 74px` while the shell topbar contract is `--shell-topbar-height: 52px`, and the PR includes before/after `/cron` recordings. I did not run a fresh live browser session in this read-only review.

## Solution Assessment

Yes for the reported offset mismatch: `top: 0` relative to the content scroller plus `--shell-topbar-height` in `max-height` is the narrow sticky-editor fix. The broader UX direction remains a maintainer choice because https://github.com/openclaw/openclaw/pull/47582 targets the same panel by removing sticky behavior entirely.
