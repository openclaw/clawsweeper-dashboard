## Best Possible Solution

Confirm the desktop Cron editor layout contract, then land this sticky-offset variable fix if the editor should remain sticky, or supersede it with the normal-flow alternative if that is the intended UX.

## Reproduction Assessment

## Solution Assessment

Yes if maintainers want to preserve sticky behavior: using `--shell-topbar-height` removes a duplicated stale layout constant with a small CSS change. It is product-dependent because https://github.com/openclaw/openclaw/pull/47582 proposes normal document flow instead.
