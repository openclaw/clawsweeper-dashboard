This should stay open as a plausible, narrow implementation PR. ## What This Changes

## Best Possible Solution

Maintainers should define the desktop Cron form layout contract, then land either this sticky-offset fix or the normal-flow alternative with equivalent regression coverage.

## Solution Assessment

Yes for preserving sticky behavior: using `--shell-topbar-height` removes a stale duplicated layout constant with a very small CSS change. It is unclear at the product level because https://github.com/openclaw/openclaw/pull/47582 is a plausible alternative if maintainers want the panel in normal document flow instead.
