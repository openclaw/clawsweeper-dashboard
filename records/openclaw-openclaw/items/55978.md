## Summary

Keep this PR open. Current main still has the hardcoded Cron workspace form sticky offset, and this PR is a narrow CSS/test/changelog fix for that remaining UI bug; related #47582 is an open alternative UX direction, not proof that this PR is obsolete.

## What This Changes

This PR updates the Cron workspace form CSS to use `top: 0` and `var(--shell-topbar-height)` for sticky sizing, adds a CSS regression test, and records an Unreleased changelog fix.

## Best Possible Solution

Choose the intended Cron desktop layout explicitly: land this PR if the form should remain sticky but aligned to the shell layout variable, or supersede it with #47582 if the form should return to normal document flow. Either path should keep a focused regression guard for the chosen CSS contract.
