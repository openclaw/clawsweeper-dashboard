## Summary

Keep open: the hardening remains useful and is not on current main, but the proposed patch cannot merge as-is because its security cap is code-unit based and real behavior proof is absent.

## What This Changes

Adds config `$include` NUL-byte rejection, a 4096 path-length cap, focused include tests, and gateway config docs updates.

## Best Possible Solution

Land a rebased narrow hardening patch that rejects NUL before resolution, enforces a documented fewer-than-4096 UTF-8-byte cap on raw and normalized paths, preserves allowed-root guarded reads, and adds ASCII plus multibyte boundary coverage.

## Reproduction Assessment

Yes for the PR defect: the diff shows `.length` checks in `src/config/includes.ts`, and a Node probe proves 2048 CJK characters are 6144 UTF-8 bytes while only 2048 code units. This is source-level PR review proof rather than a current-main bug reproduction.

## Solution Assessment

No. The hardening direction is appropriate, but the implementation should measure byte length, clarify the exclusive byte boundary, add boundary and multibyte tests, and rebase over current allowed-root logic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Measure include path limits in bytes:** `src/config/includes.ts:198-211`
  - body: The new guards compare `includePath.length` and `normalized.length`, which count UTF-16 code units. Because the docs tie the cap to PATH_MAX-style path length, multibyte UTF-8 paths can exceed 4096 bytes while passing validation; use byte length for both checks and add multibyte coverage.
  - confidence: 0.93
- **[P3] Clarify the exclusive path length boundary:** `src/config/includes.ts:199-213`
  - body: The guards reject values at `>= MAX_INCLUDE_PATH_LENGTH`, but the errors say `maximum length (4096 characters)`, which reads as if 4096 characters are allowed. After switching to byte length, phrase the limit as fewer than 4096 bytes and add acceptance-side boundary coverage.
  - confidence: 0.82
