## Summary

Keep open: current main has useful adjacent mitigations for sessions.list and compaction timeouts, but the reported root problem, large compaction work starving cheap Gateway RPCs until the 10s wrapper timeout fires, is not proven fixed or fully owned by a narrower merged change.

## What This Changes

The issue reports gateway RPC calls such as sessions.list, sessions.send, and sessions_history timing out when large transcript compaction stalls the Node event loop.

## Best Possible Solution

Keep this open until a profile-backed fix proves cheap Gateway RPCs remain responsive during large compaction, ideally with focused regression coverage around preflight compaction plus concurrent sessions.list/send/history calls.

## Reproduction Assessment

No, not as a high-confidence live current-main repro. The source path is clear enough to be source-reproducible: current main can run large preflight compaction in the Gateway process while the gateway call wrapper still has a 10s default timeout.

## Solution Assessment

Unclear. The current sessions.list async/lightweight path and compaction timeout wrapper are useful partial mitigations, but they do not settle whether the maintainable fix is yielding, isolation, queue admission, or timeout/backpressure policy.
