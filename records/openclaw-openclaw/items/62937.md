## Summary

Keep open. This PR targets a real sandboxing feature gap, but the branch is draft, not implemented on current main, has source-review blockers in the proposed profile/test code, lacks real macOS behavior proof, and is paired with an open broader sandbox issue.

## What This Changes

The PR adds a macOS `sandbox-exec` profile option to `exec`, runtime wrapping logic, a Seatbelt wrapper module, and profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed sandbox backend or maintainer-owned sandbox capability under `agents.defaults.sandbox`, with docs, regression tests, and redacted macOS live proof while keeping broader architecture tracked by https://github.com/openclaw/openclaw/issues/58730 and https://github.com/openclaw/openclaw/issues/12505.

## Reproduction Assessment

Yes for the PR defects from source review: the diff shows an allow-by-default profile and a test import of a non-exported helper. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected `sandboxProfile` argument is not the safest boundary for host command isolation; the existing sandbox backend/config contract is the narrower maintainable integration point.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:130-134`
  - body: The default profile starts from `(allow default)` and then adds path-specific file allow rules, so `sandboxProfile: "default"` would not enforce the advertised filesystem restriction. Start from a deny/default-denied file policy and allow only the intended paths.
  - confidence: 0.86
- **[P2] Export or stop importing the profile helper:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString`, but the helper added in the runtime file is local and not exported. Test collection or typechecking will fail until the helper is exported or the test exercises a public API.
  - confidence: 0.96
- **[P2] Emit a flat enum schema for sandboxProfile:** `src/agents/bash-tools.exec-runtime.ts:190`
  - body: The new tool argument uses a literal union, which serializes as an `anyOf`-style schema for providers that reject those shapes. Rebase onto the current schema module and use the repo's flat string enum helper for this option.
  - confidence: 0.78
