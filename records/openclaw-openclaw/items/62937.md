## Summary

Keep open. Current main does not implement the proposed macOS Seatbelt backend or `sandboxProfile` option, and the PR is still a draft, unmergeable, and tied to the open #58730 feature request; the proposed patch also has concrete correctness and security-boundary issues before it could be considered.

## What This Changes

The PR adds a macOS `sandboxProfile` exec parameter, wraps Darwin host exec calls with `/usr/bin/sandbox-exec`, adds a Seatbelt helper module, and adds profile-generation tests.

## Best Possible Solution

If native macOS Seatbelt support is accepted, implement it as a maintainer-configured sandbox backend or backend capability under `agents.defaults.sandbox`, keep the default fail-closed and not model-selectable, update schema/docs, and add regression plus macOS live validation for network and filesystem boundaries.

## Reproduction Assessment

Yes for the PR defects: the diff itself shows the default profile starts with `(allow default)`, and the new test imports a helper that the same diff does not export. Runtime macOS boundary validation was not run because this review is read-only.

## Solution Assessment

No. A per-call `sandboxProfile` chosen by the tool call is not the narrowest maintainable security boundary; a configured sandbox backend/capability owned by maintainer config fits the existing architecture better.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:130-135`
  - body: The default profile begins with `(allow default)` and never denies file operations before adding path-specific allow rules, so `sandboxProfile: "default"` would not enforce the advertised read/write restriction. A sandbox profile used as a security boundary needs to deny by default and then explicitly allow the required paths.
  - confidence: 0.83
- **[P2] Export the profile helper used by the new test:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString` from `bash-tools.exec-runtime.js`, but the helper added in the runtime file is local. Test collection/typecheck will fail until the helper is exported or the test exercises a public API.
  - confidence: 0.96
