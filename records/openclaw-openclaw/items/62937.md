## Summary

Keep open: this draft PR is still a security-sensitive sandbox design change, with blocking correctness issues in the proposed profile and test import, no real macOS behavior proof, and an open paired architecture issue.

## What This Changes

The PR adds a macOS `sandbox-exec` profile option to the exec tool, plus Seatbelt wrapper code and profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed sandbox backend or backend capability under `agents.defaults.sandbox`, document the contract, and add unit plus macOS live boundary proof while keeping https://github.com/openclaw/openclaw/issues/58730 as the broader architecture tracker.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR defects: the diff shows the allow-by-default profile and the test import of a non-exported helper; I did not establish live macOS runtime proof in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. A model-selected per-call `sandboxProfile` is not the narrowest maintainable security boundary; the existing sandbox backend/config contract is the safer integration point.
