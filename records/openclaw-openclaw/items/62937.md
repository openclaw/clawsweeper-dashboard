## Summary

Keep open. The PR still proposes a security-sensitive macOS host-exec sandbox change, but the current diff is draft/dirty, lacks real macOS behavior proof, and has concrete correctness and security blockers around fail-open profiles and caller-selected sandbox strength.

## What This Changes

Adds a `sandboxProfile` parameter to `exec`, wraps Darwin local commands with `sandbox-exec`, adds Seatbelt profile helpers, and adds profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed, config-owned sandbox backend or capability under `agents.defaults.sandbox`, with docs, regression coverage, and redacted macOS live proof.

## Reproduction Assessment

Yes for the PR defects from source review: the diff shows an allow-by-default restricted profile, a non-exported helper import, an `anyOf`-style literal union schema, and unused wrapper locals under core no-unused settings. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected per-call `sandboxProfile`, especially with a permissive option, is not the safest boundary for host command isolation; the existing config-owned sandbox backend contract is the narrower maintainable integration point.

Overall confidence: 0.91

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:133-136`
  - body: The advertised restricted profile starts with `(allow default)`, so the later path-specific file allow rules do not confine filesystem access. Start from a denied/default-denied file policy and allow only the intended paths.
  - confidence: 0.86
- **[P2] Keep sandbox strength out of tool-call parameters:** `src/agents/bash-tools.exec.ts:1364`
  - body: Exposing `sandboxProfile` to the model, including `permissive`, makes sandbox strength caller-selected instead of enforced by operator-owned sandbox config. If this is a security boundary, route it through the sandbox backend/config contract instead.
  - confidence: 0.78
- **[P2] Export or stop importing the profile helper:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString`, but the helper is declared as a local function in `bash-tools.exec-runtime.ts`. Test collection or typechecking will fail until the helper is exported or the test exercises a public API.
  - confidence: 0.96
- **[P2] Emit a flat enum schema for sandboxProfile:** `src/agents/bash-tools.exec-runtime.ts:193`
  - body: The new tool argument uses a literal union, which serializes as an `anyOf`-style schema for providers that reject those shapes. Rebase onto the current schema module and use the repo's flat string enum helper if this option remains.
  - confidence: 0.78
- **[P2] Make the added wrapper file typecheck-clean:** `src/agents/sandbox-exec-wrapper.ts:168-194`
  - body: `sandbox-exec-wrapper.ts` assigns profile content values that are never used and leaves `options` unused; core tsgo enables `noUnusedLocals` and `noUnusedParameters`, so adding this file will fail typechecking unless the dead code is removed or wired correctly.
