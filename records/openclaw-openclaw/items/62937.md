## Summary

Keep this PR open but not merge-ready: it is a draft security-boundary change with missing real macOS runtime proof, blocking source-level safety issues, and an unresolved fit question against the existing operator-owned sandbox backend contract.

## What This Changes

The PR adds a model-visible macOS `sandboxProfile` option for `exec`, wraps host commands with `sandbox-exec` on Darwin, and adds a Seatbelt wrapper module plus profile tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as an operator-owned sandbox backend or capability under `agents.defaults.sandbox`, fail closed by default, document the config contract, and prove it with redacted macOS runtime output.

## Reproduction Assessment

Yes for the blocking PR defects from source review: the diff shows an allow-by-default Seatbelt profile, a non-exported helper imported by the new test, a provider-sensitive union schema shape, and typecheck-cleanliness issues. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected per-call `sandboxProfile`, especially with `permissive`, is not the narrowest maintainable security boundary; the current sandbox backend/config contract is the safer integration point.
