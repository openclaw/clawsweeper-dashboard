## Summary

Keep open. The PR targets a real open exec-isolation gap, but it is a draft and the patch is not ready to merge because the advertised default profile is allow-by-default, the new test imports a non-exported helper, and the security boundary belongs in maintainer-owned sandbox configuration rather than a model-selectable per-call option.

## What This Changes

The PR adds a macOS `sandboxProfile` exec parameter, wraps Darwin host exec calls with `/usr/bin/sandbox-exec`, adds a Seatbelt helper module, and adds profile-generation tests.

## Best Possible Solution

If macOS Seatbelt support is accepted, implement it as a maintainer-configured sandbox backend or backend capability under `agents.defaults.sandbox`, fail closed, keep sandbox strength out of model-selected tool parameters, document it, and add unit plus macOS live boundary tests.

## Reproduction Assessment

Yes for the PR defects: the patch shows the default profile starts with `(allow default)`, and the new test imports a helper that the same patch does not export. Runtime macOS boundary validation is still missing.

## Solution Assessment

No. A per-call `sandboxProfile` selected by the tool call is not the narrowest maintainable security boundary; the existing configured sandbox backend contract is the safer integration point.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:130-135`
  - body: The default profile begins with `(allow default)` and never denies file operations before adding path-specific allow rules, so `sandboxProfile: "default"` would not enforce the advertised filesystem restriction. A sandbox profile used as a boundary needs to deny by default and then explicitly allow the required paths.
  - confidence: 0.83
- **[P2] Export the profile helper used by the new test:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString` from `bash-tools.exec-runtime.js`, but the helper added in the runtime file is local. Test collection or typecheck will fail until the helper is exported or the test exercises a public API.
  - confidence: 0.96
