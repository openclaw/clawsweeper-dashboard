  - body: The advertised `default` sandbox says file access is restricted to `/tmp` and the workspace, but this profile starts with `(allow default)` and never denies file operations before adding the read/write allow regexes.   - confidence: 0.9

  - body: This test imports `getSandboxProfileString`, but the PR declares that helper as a local non-exported function in `bash-tools.exec-runtime.ts`. The new test file will fail at module load/compile time before it can exercise the profile behavior.
  - confidence: 0.96
- **[P3] Remove the broken unused wrapper API:** `src/agents/sandbox-exec-wrapper.ts:168-198`
  - body: `buildSandboxExecArgv` loads `profileContent` but never uses it, returns `-f -` without wiring stdin for custom/default inline profiles, and for named profiles points at state-dir files that this PR never creates. Since the exec runtime does not call this module, this adds a misleading second sandbox path instead of one working implementation.
  - confidence: 0.82
