## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Remove the parallel sandboxProfile contract:** `src/agents/bash-tools.exec-runtime.ts:192-199`
  - body: Current main already defines exec placement through `host=auto|sandbox|gateway|node` and `agents.defaults.sandbox.backend`. Adding a separate per-call `sandboxProfile` knob creates a competing sandbox contract that is silently ignored off macOS and bypasses the backend seam, so agents can get conflicting isolation behavior. Please move this behind the existing sandbox backend/config contract or keep it in the tracker until that contract is designed.
  - confidence: 0.91
- **[P2] Export or stop importing the helper:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString`, but the helper added in `bash-tools.exec-runtime.ts` is a local function, not an export. That means the test/type build fails before it can exercise any sandbox behavior. Either export a deliberately public helper or test the observable exec behavior instead.
  - confidence: 0.97
- **[P2] Build the profile from the actual workdir:** `src/agents/bash-tools.exec-runtime.ts:135-136`
  - body: The default profile only allows `/Users/<name>/openclaw-workspace/...`, but exec workdirs come from config, tool args, or the gateway cwd and current docs describe configurable OpenClaw workspaces. On a normal configured workspace this profile would deny the file access it advertises, so the profile needs to be generated from the resolved workdir and proven on macOS.
  - confidence: 0.9
- **[P2] Delete or wire the wrapper as the single implementation:** `src/agents/sandbox-exec-wrapper.ts:164-199`
  - body: `sandbox-exec-wrapper.ts` is never imported by the runtime change, and `buildSandboxExecArgv` loads profile content but then returns `-f -` without supplying stdin, or returns a state-dir profile path that this PR never creates. Keeping this dead second implementation around is misleading for a security boundary; either remove it or make it the single tested execution path.
  - confidence: 0.88
