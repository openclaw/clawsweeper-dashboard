## Summary

Keep open, but do not merge as-is: this is a recent draft for an open security feature, and the diff has source-visible correctness/security blockers plus no real macOS behavior proof.

## What This Changes

The PR adds a `sandboxProfile` exec parameter, wraps Darwin host exec calls with `/usr/bin/sandbox-exec`, and adds Seatbelt helper code plus profile-generation tests.

## Best Possible Solution

If macOS Seatbelt support is accepted, implement it as a maintainer-configured sandbox backend or backend capability under `agents.defaults.sandbox`, fail closed, document it, and add unit plus macOS live boundary tests.

## Reproduction Assessment

Yes for the PR defects: the diff shows the allow-by-default profile and the test import of a non-exported helper. I did not establish live macOS runtime boundary proof in this read-only review.

## Solution Assessment

No. A per-call `sandboxProfile` chosen through the exec tool is not the narrowest maintainable security boundary; the existing sandbox backend contract is the safer integration point.
