## Summary

The PR should stay open because current main does not implement the proposed macOS Seatbelt exec wrapper, and the branch still has security-sensitive design and correctness blockers from the prior ClawSweeper review. The linked broader sandboxing issue remains open, so this PR is an implementation candidate that needs maintainer/security direction rather than cleanup closure.

## What This Changes

The PR adds a macOS `sandbox-exec` profile option to the exec tool, runtime wrapping logic, a Seatbelt wrapper module, and profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed sandbox backend or maintainer-owned sandbox capability under the existing `agents.defaults.sandbox` contract, with docs, regression tests, and macOS live proof while keeping the broader sandbox architecture discussion on the linked canonical issue.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR defects from source review: the diff shows an allow-by-default profile and a test import of a non-exported helper, though no live macOS sandbox run was performed in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. A model-selected `sandboxProfile` argument is not the safest boundary for host command isolation; the existing sandbox backend/config contract is the narrower maintainable integration point.
