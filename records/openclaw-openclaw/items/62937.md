## Summary

Keep open for maintainer/security review: the PR targets a real sandboxing gap, but the current patch conflicts with the existing sandbox backend contract, has source-visible security/type defects, and lacks macOS runtime proof.

## What This Changes

The branch adds a macOS `sandbox-exec` profile option to `exec`, runtime wrapping, a Seatbelt wrapper module, and profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as an operator-owned sandbox backend or backend capability under `agents.defaults.sandbox`, fail closed by default, document the config contract, and prove it with redacted macOS runtime output.

## Reproduction Assessment

Yes for the blocking PR defects from source review: the diff shows an allow-by-default Seatbelt profile, a non-exported helper imported by the new test, provider-sensitive union schema shape, and typecheck-cleanliness issues. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected per-call `sandboxProfile`, especially with `permissive`, is not the narrowest maintainable security boundary; the current sandbox backend/config contract is the safer integration point.

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:133-136`
  - body: The advertised restricted profile starts with `(allow default)`, so the later path-specific file allow rules do not confine filesystem access. Start from a denied/default-denied file policy and allow only the intended paths.
  - confidence: 0.86
- **[P2] Keep sandbox strength out of tool-call parameters:** `src/agents/bash-tools.exec.ts:1364`
  - body: Exposing `sandboxProfile` to the model, including `permissive`, makes the proposed sandbox strength caller-selected instead of enforced by operator-owned sandbox config. If this is a security boundary, route it through the sandbox backend/config contract.
  - confidence: 0.8
- **[P2] Export or stop importing the profile helper:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString`, but the helper is declared as a local function in `bash-tools.exec-runtime.ts`. Test typechecking or collection will fail until the helper is exported or the test exercises a public API.
  - confidence: 0.96
- **[P2] Emit a flat enum schema for sandboxProfile:** `src/agents/bash-tools.exec-runtime.ts:189-199`
  - body: The new tool argument uses a literal union, which serializes as an `anyOf`-style schema for providers that reject those shapes. Rebase onto the current schema module and use the repo's flat string enum helper if this option remains.
  - confidence: 0.78
- **[P2] Remove or wire the unused wrapper implementation:** `src/agents/sandbox-exec-wrapper.ts:168-194`
  - body: `sandbox-exec-wrapper.ts` assigns profile content values that are never used and leaves the wrapper disconnected from the runtime path; core tsgo has `noUnusedLocals` enabled, so this file needs to be removed or wired through a typecheck-clean public API.
  - confidence: 0.88
