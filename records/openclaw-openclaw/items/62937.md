## Summary

Keep open, but not merge-ready: the PR changes host command isolation without macOS runtime proof, exposes sandbox strength as a model-selected tool argument, and conflicts with the current sandbox backend/config contract.

## What This Changes

The PR adds a macOS `sandbox-exec` `sandboxProfile` option to `exec`, wraps Darwin host exec with inline Seatbelt profiles, and adds profile tests plus a standalone wrapper module.

## Best Possible Solution

If macOS Seatbelt support is accepted, implement it as a registered sandbox backend or operator-owned `agents.defaults.sandbox` config path, fail closed, document the contract, and prove fresh and upgrade behavior on macOS.

## Reproduction Assessment

Yes for the blocking PR defects from source review: the diff shows an allow-by-default Seatbelt profile, a caller-selected sandbox profile, a non-exported helper imported by the test, and stale schema integration. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected per-call `sandboxProfile`, especially with `permissive`, is not the narrowest maintainable security boundary; the current sandbox backend/config contract is the safer integration point.
