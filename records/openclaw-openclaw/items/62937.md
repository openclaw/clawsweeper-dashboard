## Summary

Keep open. This is a draft security-sensitive feature PR paired with an open issue it would close, and the patch still has concrete correctness, security-design, typecheck, and real-behavior-proof blockers.

## What This Changes

The PR adds a macOS `sandbox-exec` profile option to the exec tool/runtime, a Seatbelt wrapper helper, and profile-generation tests.

## Best Possible Solution

If macOS Seatbelt support is accepted, implement it as a maintainer-configured sandbox backend or backend capability under `agents.defaults.sandbox`, fail closed, document it, and add unit plus macOS live boundary tests.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the allow-by-default profile, the non-exported test import, and noUnused typecheck blockers. I did not establish live macOS sandbox behavior proof.

## Solution Assessment

No. A model-selected per-call `sandboxProfile` is not the narrowest maintainable security boundary; the existing sandbox backend/config contract is the safer integration point.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:133-136`
  - body: The default profile starts from `(allow default)` and only adds file allow rules, so `sandboxProfile: "default"` would not enforce the advertised filesystem restriction. A sandbox boundary needs to deny by default or explicitly deny file operations before allowing required paths.
  - confidence: 0.86
- **[P2] Export or stop importing the private profile helper:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The test imports `getSandboxProfileString`, but the helper is declared as a local function in the runtime file. Test collection or typechecking will fail until the test uses a public API or the helper is exported intentionally.
  - confidence: 0.96
- **[P2] Make the wrapper pass noUnused checks:** `src/agents/sandbox-exec-wrapper.ts:168`
  - body: This production file assigns `profileContent` and later `profileContent2` without reading them, and `tsconfig.core.json` enables noUnused checks for `src`. The PR should either use the loaded profile content or remove the dead locals before core typecheck can pass.
  - confidence: 0.88
