## Summary

Keep open: the PR is a recent external draft with a linked open sandbox/security tracker, missing real macOS behavior proof, and blocking source-review issues in the proposed sandbox boundary, schema, and typecheck surface.

## What This Changes

Adds a macOS `sandbox-exec` profile parameter to `exec`, wraps local Darwin commands when set, adds Seatbelt profile helpers, and adds profile-generation tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed config-owned sandbox backend or capability under `agents.defaults.sandbox`, with docs, regression tests, and redacted macOS live proof.

## Reproduction Assessment

Yes for the PR defects from source review: the diff shows an allow-by-default restricted profile, a non-exported helper imported by the new test, an `anyOf`-style literal union schema, and unused core wrapper locals under `noUnusedLocals`. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A model-selected per-call `sandboxProfile`, especially with a permissive option, is not the safest boundary for host command isolation; the existing sandbox backend/config contract is the narrower maintainable integration point.
