## Summary

Keep open. The PR is not mergeable as-is: it is draft/conflicting, adds security-sensitive sandbox behavior through a model-selected tool parameter instead of the current config-owned sandbox backend, has source-visible correctness defects, and lacks real macOS behavior proof.

## What This Changes

The PR adds a macOS `sandboxProfile` argument to `exec`, wraps Darwin local commands with `/usr/bin/sandbox-exec`, and adds Seatbelt profile helper code plus profile tests.

## Best Possible Solution

If Seatbelt support is accepted, implement it as a fail-closed config-owned sandbox backend or sandbox capability under `agents.defaults.sandbox`, with docs, regression tests, and redacted macOS live proof while keeping the broader architecture tracked in the linked sandbox issues.

## Reproduction Assessment

Yes for the PR defects from source review: the diff shows an allow-by-default profile, a test import of a non-exported helper, and an `anyOf`-style literal union schema. I did not run a live macOS sandbox canary in this read-only review.

## Solution Assessment

No. A per-call `sandboxProfile` parameter, especially with a permissive option, is not the safest boundary for host command isolation; the existing sandbox backend/config contract is the narrower maintainable integration point.
