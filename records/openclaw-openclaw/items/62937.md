- **[P1] Make the default profile deny by default:** `src/agents/bash-tools.exec-runtime.ts:130`
  - body: The PR advertises the default profile as limiting file reads/writes to `/tmp` and the workspace, but the profile starts with `(allow default)` and only denies network before adding allow rules. That gives users a false filesystem-isolation boundary; make the profile deny by default, then allow only the required read/write paths and prove it on macOS.
  - confidence: 0.93
- **[P2] Export the helper before importing it in tests:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString` from `bash-tools.exec-runtime.js`, but the PR defines that function as a non-exported local helper. The added test file will fail to compile until the helper is exported or the test is moved to a public/runtime seam.
  - confidence: 0.96
- **[P2] Preserve normal host command preparation:** `src/agents/bash-tools.exec-runtime.ts:704`
  - body: The sandbox branch builds argv directly from `execCommand` and returns before the normal host path applies PATH prepends and shell-snapshot wrapping. A sandboxed host command should wrap the already-prepared shell command so `tools.exec.pathPrepend` and shell startup behavior stay consistent with regular gateway exec.
  - confidence: 0.82
