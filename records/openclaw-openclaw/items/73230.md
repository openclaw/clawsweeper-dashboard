## Summary

Keep open. Current main still treats cron delivery `accountId` as a static explicit identity and does not provide an `auto`/`active` selector, single-active-account fallback, or stale-account warning/recovery path for rotating channel plugin accounts.

## What This Changes

The issue asks cron delivery to support dynamic or recoverable `delivery.accountId` resolution for channel plugins whose account IDs change across login or Gateway restart, such as `openclaw-weixin`.

## Best Possible Solution

Implement a conservative, channel-agnostic delivery-time recovery path for cron announce delivery: keep valid explicit account IDs, recover only when the stored account no longer resolves and exactly one eligible active/configured account exists for the same channel, and otherwise fail with an actionable diagnostic. Cover the multi-account ambiguity behavior in tests and document the resulting cron account-resolution contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure an isolated cron announce job with a stale `delivery.accountId`; current main threads that string through `resolveCronDeliveryPlan`, `resolveDeliveryTarget`, and `deliverOutboundPayloads` without validating or recovering it.

## Solution Assessment

Yes, the best fix direction is a narrow generic delivery-time stale-account recovery with strict ambiguity handling. That is safer than mutating job configs on Gateway startup or adding a WeChat-specific branch in core.

Current cron announce delivery persists an explicit `delivery.accountId` and passes it unchanged; rotating channel plugins such as `openclaw-weixin` can generate a new account ID after login or Gateway restart, leaving cron jobs pointed at a stale account. Implement a generic, channel-agnostic delivery-time recovery path. Preserve valid explicit account IDs and existing account-boundary protections. Do not add WeChat-specific core code, do not mutate cron job configs on startup, and do not weaken the account-spoof guard from #69940.
