## Summary

Keep open: the patch still leaves Feishu missing-content events unsafe before the new guards run, and the external PR only provides unit-test proof rather than real Feishu runtime proof.

## What This Changes

The PR guards Feishu empty message-content handling in sequential-key and bot dispatch code and adds a sequential-key regression test.

## Best Possible Solution

Normalize inbound Feishu message content to a string at the parser or receive boundary, then cover missing-content mention events through sequential key, debounce, and dispatch paths.

## Reproduction Assessment

Yes, at source level: a Feishu text event with runtime-missing `message.content` and mention metadata can reach parser and downstream `.replace()` or `.trim()` sites that assume a string.

## Solution Assessment

No. Optional chaining at two downstream consumers is not the narrowest maintainable fix because parser and receive-boundary first-use paths still need one consistent normalization point.
