## Summary

Keep open: the patch still leaves Feishu missing-content events unsafe before the new downstream guards run, and the external PR only includes unit-test output rather than real Feishu behavior proof.

## What This Changes

The PR adds optional chaining around Feishu parsed-content trim checks in `sequential-key.ts` and `bot.ts`, plus a sequential-key regression test for events without `message.content`.

## Best Possible Solution

Normalize inbound Feishu message content to a string at the parser or receive boundary, then cover missing-content mention events through sequential-key, debounce, and dispatch paths.

## Reproduction Assessment

Yes, at source level: construct a Feishu text event with missing `message.content` and mention metadata, then exercise `parseFeishuMessageEvent()` or `getFeishuSequentialKey()`. The parser can reach `normalizeMentions()` and dispatch can reach `ctx.content.replace(...)` before the PR's new guards.

## Solution Assessment

No. Guarding two downstream trim sites is not the narrowest maintainable fix because parser, receive/debounce, and dispatch first-use paths still need one consistent content normalization boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Normalize content before mention parsing:** `extensions/feishu/src/sequential-key.ts:17`
  - body: The optional chain here runs after `parseFeishuMessageEvent()` has already parsed content and normalized mentions. For a runtime event with missing content and mention metadata, `normalizeMentions()` can still call `.replace()` on a non-string value before this guard is reached, so the reported mention-only crash is not fully fixed.
  - confidence: 0.92
- **[P2] Normalize before the handler preview reads content:** `extensions/feishu/src/bot.ts:919`
  - body: The empty-message guard still runs after `ctx.content.replace(...)` builds the log preview. If parsed content is missing, dispatch can still throw before this guard can skip the empty message; normalize `ctx.content` before its first use, preferably at the parser boundary.
  - confidence: 0.9
