## Summary

Keep open. The PR addresses two leaf trim sites, but source inspection shows Feishu missing-content events can still fail before those guards in mention normalization, debounce text resolution, and handler preview logging.

## What This Changes

The PR guards two Feishu message-content trim sites and adds a sequential-key regression test for missing message content.

## Best Possible Solution

Normalize Feishu inbound content to a string at the parser or receive boundary, then cover missing-content events with mentions through sequential key, debounce, and dispatch, plus an Unreleased changelog fix entry.

## Reproduction Assessment

Yes, at source level: construct a Feishu text event with missing `message.content` and mention metadata, then call `parseFeishuMessageEvent()` or `getFeishuSequentialKey()`. The parser can hit `normalizeMentions()` before the PR's optional chain runs.

## Solution Assessment

No. The maintainable fix is to normalize content once at the Feishu parser/receive boundary; optional chaining at two downstream trim sites leaves other first-use paths unsafe.
