## Summary

Keep open: the PR moves the Feishu missing-content handling in the right direction, but the receive payload parser still rejects missing or empty content before the new fallback, debounce guard, or empty-message skip can run.

## What This Changes

The PR normalizes missing Feishu message content in `parseFeishuMessageEvent`, guards preview/debounce string operations, adds a sequential-key regression test, and adds a changelog entry.

## Best Possible Solution

Accept missing Feishu content at the receive/parser boundary, preserve mention gating and empty-message skipping, and add receive-handler regression coverage plus after-fix runtime proof.

## Reproduction Assessment

Yes, at source level: construct a Feishu text event with missing `message.content` and mention metadata, then exercise the receive handler or sequential-key path. I did not run a live Feishu replay.

## Solution Assessment

No. Defaulting in `parseFeishuMessageEvent()` and guarding string methods are useful, but the receive payload parser still requires truthy content and needs to be fixed with receive-handler coverage.
