## Summary

Keep open: current main and v2026.6.10 still map unresolved workspace-external touched files to missing entries and the UI renders that flag as the missing badge. The linked closing PR remains open, so this issue should stay as the canonical report until a fix lands.

## What This Changes

The issue asks for the Session Workspace Rail to stop showing successfully read workspace-external files as red missing entries.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95695 or an equivalent narrow fix that hides or distinguishes workspace-external entries while preserving true missing status for workspace-internal files.

## Reproduction Assessment

Yes, at source level: transcript read tool calls can collect absolute or parent-relative paths outside the workspace, current main converts those unresolved paths into missing entries, and the UI renders the missing badge. I did not run a live Control UI reproduction in this read-only review.

## Solution Assessment

Yes, the best current path is the linked gateway fix or an equivalent change in sessions.files.list. If maintainers decide that the gateway response must remain a complete touched-file inventory, the safer alternative is a UI-only filter or neutral status in the rail.
