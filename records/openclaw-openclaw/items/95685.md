## Summary

Keep open: current main and v2026.6.9 still convert transcript-touched paths outside the session workspace into `missing: true`, and the Control UI renders that as the Missing badge. I found no duplicate or open fix that owns this specific external-file UX bug.

## What This Changes

The issue asks for the Session Workspace Rail to stop labeling transcript-read files outside the configured workspace as missing.

## Best Possible Solution

Keep the safe workspace-root preview boundary, but change list semantics so outside-workspace transcript references are either omitted from the rail or shown with a neutral non-workspace status rather than a missing-file badge.

## Reproduction Assessment

Yes at source level: a transcript `read` tool call with an absolute path outside the resolved workspace makes `resolveTouchedFilePath` return undefined, so `toSessionFileEntry` returns `missing: true`. I did not run the live macOS/Control UI flow.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. The safest direction is to avoid reading outside the workspace while distinguishing external references from genuinely missing workspace files, or to filter external references out of the workspace rail.
