## Summary

Keep open. Current main and v2026.6.10 still convert workspace-external transcript-touched files into `missing: true` entries that the UI renders as the missing badge; https://github.com/openclaw/openclaw/pull/95695 is the open linked candidate fix and has not merged.

## What This Changes

The issue requests that the Session Workspace Rail stop showing successfully read files outside the configured workspace as red missing entries.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95695 or an equivalent narrow Gateway fix that keeps workspace-external transcript files out of the rail while preserving true missing status for workspace-internal files.

## Reproduction Assessment

Yes, at source level. A transcript-touched absolute path outside the workspace resolves to no workspace path, `toSessionFileEntry` returns `missing: true`, and the UI renders that flag as the missing badge.

## Solution Assessment

Yes. The best path is to fix the `sessions.files.list`/workspace-rail boundary so external reads are not shown as missing, and closing this issue before the linked PR merges would be premature.
