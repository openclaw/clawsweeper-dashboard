## What This Changes

The issue reports that Session Workspace Rail labels files outside the configured workspace as missing even when the agent read them successfully.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95695 or an equivalent narrow gateway fix that keeps external files out of the workspace rail list while preserving true missing status for workspace-internal files.

## Reproduction Assessment

Yes, at source level: transcript read tool calls can collect absolute external paths, current main maps those paths into `missing: true`, and the UI renders that flag as the missing badge. I did not run a live UI reproduction in this read-only review.

## Solution Assessment

Yes, the best current path is to review and land the linked narrow fix or an equivalent change in `sessions.files.list`. Closing the issue before that PR merges would be premature because current main still exposes the reported behavior.
