## Summary

Close as implemented: current main and shipped releases provide the supported Gateway service wrapper path for operator-controlled Node startup, including a locally compiled Node or Node flags, while raw NODE_OPTIONS remains intentionally blocked by the host-environment security policy.

The issue asks for a safe way to keep Gateway running on non-AVX2 Linux hardware, originally via service-level NODE_OPTIONS/--jitless or a custom Node startup path.

Yes. The wrapper path is the narrow maintainable solution because it supports custom Node startup without reopening broad NODE_OPTIONS inheritance across managed services.
