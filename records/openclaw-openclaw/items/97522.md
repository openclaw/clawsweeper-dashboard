## What This Changes

The issue reports that `openclaw doctor --fix` can migrate a legacy `<provider>:default` OAuth profile onto an existing second OAuth profile, collapsing `auth.profiles` and wiping profile metadata/order.

## Best Possible Solution

Make `repairOAuthProfileIdMismatch` non-destructive when the suggested destination id already exists as a distinct configured profile, preserve config-owned metadata/order, and add regression coverage for the Anthropic doctor path.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current code still selects the sole non-legacy OAuth profile and overwrites that destination while doctor auto-applies fixes under `--yes`. I did not run the full CLI command in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow repair change in `repairOAuthProfileIdMismatch`, plus tests for the two-profile collision and the doctor/Anthropic trigger. Adding a new option or broad migration mode is not necessary for the reported destructive overwrite.
