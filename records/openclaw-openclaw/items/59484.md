## Summary

Keep open: this is a plausible bug-fix direction, but the submitted patch is not merge-ready because it rethrows every `channels status --probe` gateway-call failure, including typed unreachable-gateway transport failures that the shipped CLI contract still sends through config-only fallback. The PR also still lacks after-fix real CLI proof and is currently reported as conflicting.

## What This Changes

The PR adds an unconditional probe-mode rethrow in `channelsStatusCommand` and a focused command-flow regression test for a mocked probe failure.

## Best Possible Solution

Rethrow only non-transport `channels.status` request/RPC failures in probe mode, preserve config-only fallback for unreachable gateway transport errors, and prove both branches with focused tests plus redacted CLI output.

## Reproduction Assessment

Yes, at source level: current main catches all `callGateway` failures and the PR’s mocked rejection shows the reported false fallback path. There is not yet real CLI proof from a live setup.

## Solution Assessment

No. The maintainable fix is to separate post-connect request/RPC failures from typed transport failures, not to rethrow every probe-mode gateway-call error.
