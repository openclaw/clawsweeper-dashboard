## Summary

Close as mostly implemented: current main and the latest release already handle per-account probe failures as partial gateway status data, while this stale branch’s remaining broad rethrow is conflicting and compatibility-risky.

## What This Changes

The PR changes `channels status --probe` so probe-mode gateway-call failures are rethrown instead of using the config-only fallback, with one regression test.

## Best Possible Solution

Close this stale branch and keep the shipped gateway-side partial-status behavior together with the documented config-only fallback for actual reachability failures.

## Reproduction Assessment

No current-main failure reproduces from source: the gateway handler now returns partial status payloads when live probes throw or time out, and current tests cover that path. I did not run a live CLI repro because this review was read-only.

## Solution Assessment

No; this PR is no longer the best way to solve the issue because the unconditional probe rethrow is broader than the stated failure and removes the documented fallback for unreachable gateways. The shipped gateway-side partial-status handling is the narrower maintainable solution.
