## Summary

Keep open: the PR is a focused bug-fix attempt, but the current patch rethrows every probe-mode gateway call failure and would bypass the documented config-only fallback for unreachable-gateway transport failures. It also still lacks contributor-supplied after-fix real behavior proof.

## What This Changes

The PR adds an early rethrow for `channels status --probe` gateway-call failures and a regression test for a plain probe error not falling back to config-only status.

## Best Possible Solution

Update the PR to distinguish transport reachability failures from request/probe failures, add tests for both `GatewayTransportError` fallback and non-transport rethrow, then attach redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence harness path: mock `callGateway` with a plain Error for the intended rethrow behavior and with a `GatewayTransportError` for the fallback-preservation regression; tests were not run because this review is read-only.

## Solution Assessment

No. Rethrowing every probe-mode `callGateway` failure is broader than the after-gateway scope; the narrower maintainable fix is to keep typed transport errors on the documented fallback path and rethrow only non-transport request/probe failures.
