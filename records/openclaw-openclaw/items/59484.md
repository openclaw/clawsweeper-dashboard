## Summary

## What This Changes

The PR rethrows `channels status --probe` gateway-call failures before config-only fallback and adds a mocked command-flow regression test.

## Best Possible Solution

Rethrow only non-transport request or RPC failures from `channels status --probe`, preserve `GatewayTransportError` on the config-only fallback path, and cover both branches with focused tests plus redacted CLI proof.

## Reproduction Assessment

Yes, at source level. Current main catches all `callGateway` failures, while `GatewayClient.request` can reject after handshake and `callGateway` can also reject typed transport failures for unreachable gateway paths.

## Solution Assessment

No. The PR should not rethrow every probe-mode failure; the narrow maintainable fix is to rethrow non-transport request/RPC failures while preserving unreachable-gateway fallback.
