## Summary

Keep open: the bridge direction is useful, but the patch still misses at least one Codex app-server startup family and the external proof is only targeted tests, not the claimed live Gateway/Cron behavior.

## What This Changes

The PR adds stdio Codex app-server Gateway auth env injection from resolved parent config SecretRefs, SDK exports/docs, tests, and a changelog entry.

## Best Possible Solution

Thread the parent runtime config through every stdio Codex app-server startup path that can spawn OpenClaw CLI subprocesses, keep the bridge stdio-only with explicit override behavior covered, and add redacted real runtime proof.

## Reproduction Assessment

Unclear for a live setup: source inspection shows current main isolates Codex `HOME` without Gateway env bridging, but I did not establish a live Gateway/Cron failure. The PR's tests model the failure path only.

## Solution Assessment

No, not yet: the helper is a reasonable place for the fix, but callers must consistently pass the parent config and the PR still needs real behavior proof before merge.
