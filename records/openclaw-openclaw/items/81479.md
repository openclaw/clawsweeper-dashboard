## Summary

Keep open: the bridge is useful and the supplied proof is sufficient, but the patch still misses bound Codex conversation startup paths that do not pass parent config into the shared app-server client, so SecretRef-backed Gateway auth can still be absent.

## What This Changes

This PR adds Codex stdio app-server Gateway auth env injection for SecretRef-backed local Gateway credentials, plus SDK exports, docs, changelog, and focused bridge/spawn tests.

## Best Possible Solution

Keep the stdio-only, parent-resolved injection design, then thread runtime config through every Codex app-server launch path that can run OpenClaw CLI subprocesses and cover the bound conversation paths with focused tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure file-backed `gateway.auth.*`, launch a bound Codex stdio app-server path with isolated `HOME`, then run a child OpenClaw CLI command. The PR bridge can resolve the SecretRef only when parent config reaches it, and the bound conversation callers still omit that config.

## Solution Assessment

No, not yet: the bridge helper and Gateway resolver choice are reasonable, but the fix must consistently pass parent runtime config through the remaining Codex shared-client callers before it solves the reported behavior.
