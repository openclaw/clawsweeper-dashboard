## Summary

Keep open: the underlying bug is plausible, but the current PR is not merge-ready because source inspection and a later human changes-requested review show the fix is incomplete across Codex app-server startup paths and needs maintainer judgment on the secret-env, Gateway-context, and public SDK boundaries.

## What This Changes

The PR injects resolved local Gateway auth into Codex stdio app-server environments, exports Gateway auth helpers through the plugin SDK, and updates focused tests, docs, and changelog text.

## Best Possible Solution

Keep the stdio bridge only if every Codex app-server startup/control path gets the parent config and maintainers explicitly approve or narrow the Gateway credential, connection-context, and SDK surface.

## Reproduction Assessment

Yes. Source inspection shows current main isolates Codex stdio app-server HOME, the PR bridge only injects Gateway auth when config is supplied, and several bound conversation/control callers still omit that config.

## Solution Assessment

No, not yet. The parent-resolved stdio bridge is a reasonable direction, but this PR must cover all app-server launch paths and settle the Gateway context and SDK boundary before it is the narrow maintainable fix.
