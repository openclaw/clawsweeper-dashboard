## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor default URL-fetch settings in gateway evidence:** `extensions/policy/src/policy-state.ts:834-835`
  - body: `gateway.http.endpoints.responses.files` and `responses.images` both allow URL inputs by default when the Responses endpoint is enabled, but this evidence path only records URL-fetch exposure when `allowUrl` is explicitly `true`. A policy with `gateway.http.requireUrlAllowlists` can therefore pass even though runtime still accepts arbitrary URL inputs unless allowlists are configured. Materialize the effective defaults, or reuse the existing limit resolvers, before deciding whether a URL-fetch input needs an allowlist finding.
  - confidence: 0.93
- **[P2] Keep policy check read-only:** `extensions/policy/src/cli.ts:146`
  - body: This path calls `readConfigFileSnapshot()` without the new `{ observe: false }` option. Config snapshot reads still observe by default and can write `logs/config-health.json`, so `openclaw policy check` mutates state even though the command and docs describe it as read-only. Use the same observe-free read that `doctor --lint` uses.
  - confidence: 0.9
