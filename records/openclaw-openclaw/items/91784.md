## Summary

Keep open for maintainer review: the patch looks like the right narrow route-boundary fix and the supplied terminal proof is sufficient, but merging intentionally changes current and shipped realtime WebSocket routing compatibility.

## What This Changes

This PR changes the voice-call realtime WebSocket upgrade classifier to accept only the configured stream path or slash-delimited child paths, and adds bound-server regression tests for sibling rejection and root child routing.

## Best Possible Solution

Land this clean route-boundary fix if maintainers accept rejecting accidental same-prefix aliases, then close https://github.com/openclaw/openclaw/issues/79918 and supersede https://github.com/openclaw/openclaw/pull/79919.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.6 shows `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` matches a configured `/voice/stream/realtime` path; no runtime test was run because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the compatibility break. The upgrade classifier is the right layer because generated stream URLs use the configured stream path plus a slash-delimited token child, and rejecting siblings before RealtimeCallHandler dispatch is the narrow owner-boundary fix.
