## What This Changes

The PR changes voice-call realtime WebSocket upgrade matching from raw prefix routing to exact-or-slash-child path routing and adds sibling/root route-boundary coverage.

## Best Possible Solution

Land one focused exact-or-slash-child realtime route-boundary fix with sibling-prefix rejection and root-path coverage, then close the canonical issue after maintainers accept the compatibility change.

## Reproduction Assessment

## Solution Assessment

Yes, this is the right layer: the upgrade dispatch predicate owns whether a request reaches `RealtimeCallHandler`, and exact-or-slash-child matching preserves generated token children while rejecting sibling prefixes. The safer alternative only applies if maintainers decide sibling-prefix aliases are a shipped compatibility contract.
