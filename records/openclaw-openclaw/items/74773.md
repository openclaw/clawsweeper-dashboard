## Summary

Keep open. The PR targets the right Twitch-owned schema problem, but it only changes the source Zod schema while packaged config validation is seeded from committed generated channel metadata. The PR changes the Twitch simplified and multi-account config schemas from `z.intersection()` composition to `.merge()` composition in `extensions/twitch/src/config-schema.ts`.

Update this PR or a narrow replacement so the Twitch source schema, generated bundled channel metadata, regression tests, and changelog entry all move together, then land it as the implementation for #60041 without changing Twitch runtime, auth, token refresh, setup, or access-control behavior.

## Solution Assessment

No. The `.merge()` source change is a plausible owner-local fix, but this PR is incomplete because packaged `openclaw config validate` uses generated bundled channel metadata that the PR does not update.
