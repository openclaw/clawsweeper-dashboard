## Summary

Keep open for maintainer review: current main still lacks the shared-memory capability, but this branch adds a new config/storage/data-access surface with unresolved correctness and security findings plus missing real behavior proof.

## What This Changes

The PR adds a `_shared` memory store with `memorySearch.sharedPaths`, shared-memory CLI flags, shared-result merging, reserved `_shared` agent validation, and tests for shared indexing/exclusion behavior.

## Best Possible Solution

Rework the shared-memory design against the current `packages/memory-host-sdk` and `extensions/memory-core` boundary, preserve existing `extraPaths` semantics, validate shared paths strictly, document the data-access contract, and provide redacted real CLI proof.

## Reproduction Assessment

Yes for the review findings: the final-head source shows the config remap, blank shared-path handling, duplicate-weight precedence, file-path mkdir behavior, and status DB path mismatch. This is not a current-main bug reproduction because the PR adds a new shared-memory feature.

## Solution Assessment

No. The capability may be useful, but this branch should not merge as-is; the maintainable path is a current-boundary rework with strict validation, docs, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep relative extraPaths workspace-relative:** `src/agents/memory-search.ts:242-263`
  - body: The PR remaps default `extraPaths` into `sharedPaths` and resolves agent `extraPaths` with `resolveUserPath`, so relative entries that current main resolves against each agent workspace can point at the process cwd or disappear from local indexing. Existing configs like `../team-docs` can silently stop indexing the intended files.
  - confidence: 0.88
- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank `sharedPaths` entries are accepted and can become `""` in `settings.sharedPaths`. The shared read gate later checks `absPath.startsWith(`${sp.path}${path.sep}`)`, so an empty path matches broad absolute Markdown paths on POSIX and expands reads beyond configured directories.
  - confidence: 0.87
- **[P2] Apply override weights after defaults:** `src/memory/manager.ts:444-447`
  - body: Shared result weighting stops on the first matching path. When defaults and an agent override contain the same path with different weights, the default match wins and the per-agent weight override has no effect.
  - confidence: 0.82
- **[P2] Avoid mkdir for file sharedPaths:** `src/memory/manager.ts:250-252`
  - body: The constructor eagerly calls `ensureDir` for every shared path, but shared paths are otherwise treated like directories or individual Markdown files. Configuring a not-yet-created `policy.md` creates a directory at that file path, blocking the intended file from being created and indexed.
  - confidence: 0.74
- **[P2] Report counts from the actual shared DB:** `src/memory/manager.ts:884-893`
  - body: Shared status reports and reads counts from `this.settings.sharedStorePath`, but the `_shared` manager can persist to `settings.store.path`. With a custom store path, `memory status` can report the wrong DB and stale zero counts after shared indexing succeeds.
  - confidence: 0.78
