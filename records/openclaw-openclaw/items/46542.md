## Summary

Keep open. The feature is not implemented on current main, and the PR still needs maintainer product/security review because it expands cross-agent memory reads while patching old core memory paths instead of the active `memory-core` plugin runtime.

## What This Changes

This branch adds `memorySearch.sharedPaths`, an internal `_shared` memory index, shared memory CLI flags, shared-result tagging, reserved `_shared` agent validation, and related tests.

## Best Possible Solution

Rework shared document recall as plugin-owned `memory-core` or SDK shared-corpus support with strict canonical path validation, explicit default/agent precedence, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes for review purposes: the PR diff has deterministic source-level review paths for the blank shared path, relative `extraPaths`, weighting, status-path, and file-path `mkdir` findings. I did not run live indexing because this was a read-only review.

## Solution Assessment

No. The direction is plausible, but this patch is not the best current solution because it modifies stale core memory surfaces and leaves concrete path/security bugs; the safer route is a plugin-owned design with explicit access rules.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject empty shared paths before allowing reads:** `src/config/zod-schema.agent-runtime.ts:601`
  - body: `sharedPaths` accepts blank strings/paths, and the later shared read check treats an empty resolved path as a broad prefix match. Trim and reject empty shared path entries before resolving or using them for read authorization.
  - confidence: 0.91
- **[P2] Resolve agent extraPaths relative to the workspace:** `src/agents/memory-search.ts:254-258`
  - body: This maps per-agent `extraPaths` through `resolveUserPath`, so relative entries are resolved from the process cwd instead of the agent workspace. Existing indexing resolves these through `normalizeExtraMemoryPaths(workspaceDir, ...)`, so current memory files can be silently dropped or retargeted.
  - confidence: 0.88
- **[P2] Apply override weights when shared paths repeat:** `src/memory/manager.ts:428-432`
  - body: The shared result weighting loop stops at the first matching path. Because merged config keeps defaults before agent overrides, an agent-specific weight for the same path is ignored.
  - confidence: 0.86
- **[P2] Report status from the actual shared store path:** `src/memory/manager.ts:881-884`
  - body: Shared status reads `this.settings.sharedStorePath`, but the `_shared` manager can persist to its resolved `settings.store.path`. With a custom store path, status can report the wrong DB and stale zero counts after indexing succeeds.
  - confidence: 0.84
- **[P2] Avoid mkdir on file shared paths:** `src/memory/manager.ts:252-254`
  - body: The constructor eagerly calls `ensureDir` for every shared path even though shared paths are treated like `extraPaths` and may be individual Markdown files. A configured file path such as `policy.md` can be created as a directory and block indexing the intended file.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The patch adds a new cross-agent memory read surface and currently allows a blank shared path to broaden reads beyond explicitly configured shared directories.

  - body: Because the schema accepts empty shared path entries and the read check matches path prefixes, a blank config value can authorize arbitrary absolute Markdown paths instead of only explicit shared directories.
