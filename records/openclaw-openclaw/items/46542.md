## Summary

## What This Changes

The PR adds a `_shared` memory namespace, `memorySearch.sharedPaths`, reserved `_shared` agent validation, shared-memory CLI flags, and tests for shared-path indexing/exclusion.

## Best Possible Solution

Rework shared memory against the current memory-core/memory-host-sdk boundary, preserve existing `extraPaths` semantics, validate shared paths before use, document the config/security contract, and add real behavior proof for shared indexing/search.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve workspace-relative default extraPaths:** `src/agents/memory-search.ts:241-263`
  - body: This remaps `agents.defaults.memorySearch.extraPaths` through `resolveUserPath` before putting them in the shared store and drops defaults from local `extraPaths`. Existing relative defaults like `../team-docs` are currently resolved per agent workspace by `normalizeExtraMemoryPaths(workspaceDir, ...)`; after this patch they resolve against the process cwd and can stop indexing for every agent.
  - confidence: 0.88
- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-232`
  - body: The schema allows `sharedPaths: [""]`, and this maps to a shared path of `""`. Later `readFile` checks `absPath.startsWith(`${sp.path}${path.sep}`)`, so on POSIX an empty shared path matches every absolute `.md` path and broadens readable files beyond the configured shared directories.
  - confidence: 0.87
- **[P2] Apply agent sharedPath overrides last:** `src/memory/manager.ts:444-448`
  - body: Config merges default shared paths before per-agent overrides, but weighting stops at the first matching path. If an agent overrides the same shared path with a different weight, the default match wins, so the documented per-agent weighting silently has no effect.
  - confidence: 0.82
- **[P2] Avoid creating directories for file sharedPaths:** `src/memory/manager.ts:248-252`
  - body: The constructor calls `ensureDir` for every shared path, but those paths are later fed into the same file-or-directory indexing flow as `extraPaths`. A not-yet-created `policy.md` shared path becomes a directory named `policy.md`, preventing the intended file from being created and indexed without manual cleanup.
  - confidence: 0.74

## Security Review
