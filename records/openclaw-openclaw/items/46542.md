## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve default extraPaths during upgrade:** `src/agents/memory-search.ts:232-242`
  - body: This block migrates `agents.defaults.memorySearch.extraPaths` into the shared store and later builds agent-local `extraPaths` only from overrides. Existing users who rely on default-level extra paths will silently move from per-agent indexes to `_shared`, changing storage, search visibility, and upgrade behavior despite the PR saying no migration is needed.
  - confidence: 0.9
- **[P1] Keep relative extraPaths workspace-scoped:** `src/agents/memory-search.ts:257`
  - body: Mapping override `extraPaths` through `resolveUserPath()` here resolves relative entries against the process working directory before the memory normalizer can resolve them against the agent workspace. Configs like `extraPaths: ["../team-docs"]` can therefore index the wrong path or miss documents that work on current main.
  - confidence: 0.89
- **[P1] Reject blank sharedPaths before matching reads:** `src/memory/manager.ts:731`
  - body: `sharedPaths` entries are accepted without trim/filter validation, and the read check later treats each entry as an absolute prefix. A blank entry can make `absPath.startsWith(path.sep)` true for absolute Markdown paths on POSIX, turning a config typo into broad file-read access outside the intended shared directories.
  - confidence: 0.86
- **[P2] Avoid creating configured file paths as directories:** `src/memory/manager.ts:249`
  - body: The constructor calls `ensureDir(sp.path)` for every configured shared path, but shared paths are otherwise treated like `extraPaths` and can be individual `.md` files. A not-yet-created file path such as `~/shared/policy.md` will be created as a directory, blocking the intended file until manual cleanup.
  - confidence: 0.84
