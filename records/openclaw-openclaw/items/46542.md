## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve default extraPaths during upgrade:** `src/agents/memory-search.ts:234-245`
  - body: Current main tests and docs keep `agents.defaults.memorySearch.extraPaths` in each agent's memory scope, but this branch silently moves those defaults into the shared store and removes them from normal agent extra paths. That changes existing users' index topology and QMD collection scope during upgrade, so keep existing behavior by default and add an explicit tested migration only if maintainers choose it.
  - confidence: 0.9
- **[P1] Keep relative extraPaths workspace-relative:** `src/agents/memory-search.ts:259-263`
  - body: This resolves per-agent `extraPaths` with `resolveUserPath()` before the workspace-aware normalizer sees them. Relative configs such as `../team-docs` used to resolve against the agent workspace; after this change they resolve against the process cwd and can silently index the wrong location.
  - confidence: 0.86
- **[P1] Reject blank sharedPaths before read checks:** `src/memory/manager.ts:731-734`
  - body: A blank shared path reaches the shared read boundary, where `absPath.startsWith(`${sp.path}${path.sep}`)` becomes a `/` prefix check on POSIX. A config typo like `sharedPaths: [""]` can therefore allow reads from any absolute Markdown path; trim and reject blank shared paths before storing them.
  - confidence: 0.92
- **[P2] Avoid creating directories for file shared paths:** `src/memory/manager.ts:248-252`
  - body: `sharedPaths` are later handled like extra paths that may be directories or individual Markdown files, but the constructor calls `ensureDir()` for every configured path. If a user configures a not-yet-created file such as `policy.md`, this creates a directory at that file path and blocks the intended file from being indexed.
  - confidence: 0.83
- **[P2] Read shared status from the actual shared DB:** `src/memory/manager.ts:891-895`
  - body: Status reads counts from `settings.sharedStorePath`, but the `_shared` manager can persist to its resolved `memorySearch.store.path` when users customize store paths or `{agentId}` templates. In that setup, `memory status` can show the wrong DB path and stale zero counts after indexing succeeds.
  - confidence: 0.78
