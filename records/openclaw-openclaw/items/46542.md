## Summary

Keep this PR open, but it is not mergeable as-is. Current main still lacks `sharedPaths`, `_shared.sqlite`, and memory `--shared` support, and related duplicate reports were closed to this PR as the canonical shared-memory work; however the branch predates the current memory plugin/host-SDK layout, has concrete path-scope bugs, and still needs real behavior proof.

## What This Changes

The PR adds a shared `_shared` memory namespace, `memorySearch.sharedPaths`, shared memory CLI flags, shared-origin search results, reserved `_shared` agent validation, and tests for shared-path indexing/exclusion.

## Best Possible Solution

Rework the shared-memory concept against the current memory-core and memory-host-SDK boundaries, preserve `extraPaths` semantics, validate shared paths strictly, document the data-scope contract, and require real shared indexing/search proof before merge.

## Reproduction Assessment

Yes for the review findings: the default `extraPaths` remap, blank `sharedPaths` access check, duplicate-weight precedence, and file-path directory creation are source-reproducible from the PR diff. This is otherwise a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The requested capability may be useful, but this stale implementation is not the best path because the memory architecture has moved and the config/security contract needs to be designed in the current plugin/host-SDK layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve workspace-relative default extraPaths:** `src/agents/memory-search.ts:241-263`
  - body: This remaps `agents.defaults.memorySearch.extraPaths` through `resolveUserPath()` and removes defaults from local `extraPaths`. Existing relative defaults such as `../team-docs` are currently resolved per agent workspace by `normalizeExtraMemoryPaths(workspaceDir, ...)`; after this patch they resolve against the process cwd and can stop indexing for every agent.
  - confidence: 0.88
- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-232`
  - body: The schema accepts blank `sharedPaths` entries, and the merge path resolves them before filtering. An empty shared path then matches absolute Markdown paths in the shared read gate, broadening readable files beyond the configured directories.
  - confidence: 0.87
- **[P2] Apply agent sharedPath overrides last:** `src/memory/manager.ts:444-448`
  - body: Shared result weighting stops at the first matching shared path. Because defaults are merged before per-agent overrides, an agent-specific weight for the same path is ignored and the documented per-agent weighting silently has no effect.
  - confidence: 0.82
- **[P2] Avoid creating directories for file sharedPaths:** `src/memory/manager.ts:248-252`
  - body: The constructor eagerly calls `ensureDir()` for every shared path, but shared paths are otherwise treated like file-or-directory `extraPaths`. A configured not-yet-created `policy.md` shared path becomes a directory named `policy.md`, blocking the intended file from being created and indexed without manual cleanup.
  - confidence: 0.74
