## Summary

Keep open. Current main still lacks shared cross-agent memory, but this PR is conflicting/stale against the current `extensions/memory-core` layout and still has blocking correctness, security, CI, and real-behavior-proof gaps.

## What This Changes

The PR adds `memorySearch.sharedPaths`, a reserved `_shared` memory store, shared memory CLI flags, shared-result merging, and tests for reserved IDs and excluded shared paths.

## Best Possible Solution

Rework the feature into the current `extensions/memory-core` and `packages/memory-host-sdk` contracts, preserve or explicitly document `extraPaths` semantics, validate shared path scope strictly, add docs/tests, and provide redacted live proof.

## Reproduction Assessment

Yes for the PR review defects: PR-head source shows blank shared-path acceptance, default `extraPaths` migration, eager directory creation for shared paths, first-match weighting, and a hard-coded shared status path. This is not a current-main bug reproduction because current main does not implement the shared memory feature.

## Solution Assessment

No. A shared store may be useful, but this branch is not the best merge path until it is rebased to the current memory-core/host-SDK layout, fixes path-scope bugs, preserves or intentionally changes existing config semantics, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths flow into `settings.sharedPaths` as an empty path. The shared read gate later checks `${sp.path}${path.sep}`, so an empty path becomes `/` on POSIX and can authorize broad absolute `.md` reads instead of only configured shared roots.
  - confidence: 0.88
- **[P1] Preserve default extraPaths semantics:** `src/agents/memory-search.ts:234-263`
  - body: This moves `agents.defaults.memorySearch.extraPaths` into the shared store and drops defaults from the agent-local `extraPaths` list. Current code, docs, and tests treat defaults as per-agent paths merged with overrides, so existing configs can silently index/search different locations.
  - confidence: 0.87
- **[P2] Avoid mkdir on shared file paths:** `src/memory/manager.ts:250-251`
  - body: Shared paths are documented and handled like `extraPaths`, which can be individual markdown files, but the constructor calls `ensureDir(sp.path)` for every entry. A configured `policy.md` path can be created as a directory and block the intended file from being indexed.
  - confidence: 0.82
- **[P2] Apply shared path weight overrides last:** `src/memory/manager.ts:444-447`
  - body: Per-agent `sharedPaths` overrides are meant to adjust weights, but the loop stops at the first matching path. When defaults and an agent override contain the same path, the default weight wins and the agent-specific override is ignored.
  - confidence: 0.8
- **[P2] Use the resolved shared store path for status:** `src/agents/memory-search.ts:265`
  - body: `sharedStorePath` is hard-coded to the default `_shared.sqlite` path, while the `_shared` manager can use a custom `memorySearch.store.path` template. With a custom store path, shared status reads and reports the wrong database.
  - confidence: 0.78
