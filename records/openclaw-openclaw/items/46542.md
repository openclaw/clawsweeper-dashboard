## Summary

Keep open: the shared-memory idea is useful and prior review discussion already led to fixes for the `--shared` CLI path, but the final PR head still has correctness/security defects, lacks inspectable real behavior proof, and is now conflicting with the current memory-core/host-SDK layout.

## What This Changes

The PR adds `memorySearch.sharedPaths`, a reserved `_shared` memory store, shared-memory CLI flags, merged shared-origin results, and tests for the new config and file-exclusion behavior.

## Best Possible Solution

Rework shared memory on the current memory-core and memory-host SDK boundaries with strict shared-path validation, preserved `extraPaths` compatibility, a documented data-access contract, and live after-fix proof.

## Reproduction Assessment

Yes for the PR review defects: the final-head source shows the blank shared-path acceptance, first-match weighting, eager directory creation, and stale implementation paths. This is not a current-main bug reproduction because the PR proposes a new feature.

## Solution Assessment

No. The capability may be useful, but this branch should not merge until it is rebased to current `extensions/memory-core` and `packages/memory-host-sdk`, fixes the path-scope bugs, preserves existing config behavior, and adds real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve workspace-relative default extraPaths:** `src/agents/memory-search.ts:241-263`
  - body: This remaps `agents.defaults.memorySearch.extraPaths` into `sharedPaths` and resolves them with `resolveUserPath`. Current main keeps defaults in `extraPaths` and resolves relative entries with `normalizeExtraMemoryPaths(workspaceDir, ...)`, so configs like `../team-docs` can silently point at the process cwd instead of each agent workspace.
  - confidence: 0.88
- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths are retained as an empty shared root. The later shared read check treats `${sp.path}${path.sep}` as `/` on POSIX, so a config typo can allow reads of arbitrary absolute `.md` paths.
  - confidence: 0.87
- **[P2] Apply agent sharedPath weight overrides deterministically:** `src/memory/manager.ts:444-447`
  - body: Merged shared paths are ordered defaults first, then overrides, but weighting stops at the first matching path. If an agent overrides the same shared path’s weight, the default weight wins and the per-agent override has no effect.
  - confidence: 0.82
- **[P2] Avoid creating configured shared files as directories:** `src/memory/manager.ts:250-251`
  - body: The constructor calls `ensureDir` for every configured shared path. Shared paths can be individual Markdown files through the config and soft migration, so a missing `policy.md` is created as a directory and then cannot be indexed as the intended file.
  - confidence: 0.74
- **[P3] Emit the extraPaths migration warning once:** `src/agents/memory-search.ts:236-240`
  - body: The deprecation warning runs every time memory config is resolved. Users with default `extraPaths` can get repeated warnings during searches; guard the migration warning once per process or per config value.
  - confidence: 0.75
