## Summary

Keep open. Current main does not implement shared memory, and this PR is not ready to merge because it changes cross-agent read scope, targets an older memory layout, has source-proven path-scope bugs, and lacks inspectable real behavior proof.

## What This Changes

The PR adds `memorySearch.sharedPaths`, a `_shared` SQLite memory store, shared memory CLI flags, shared-result merging, reserved `_shared` agent validation, and focused tests.

## Best Possible Solution

Rework the feature against the current memory-host SDK and memory-core plugin surfaces with strict path validation, explicit config/docs, regression tests, security review, and redacted live proof.

## Reproduction Assessment

Yes for the PR review defects: PR-head source shows blank shared-path acceptance, default `extraPaths` migration, eager directory creation for configured shared paths, first-match weighting, and hard-coded shared status paths. This is not a current-main bug reproduction because current main does not implement a shared `_shared` memory store.

## Solution Assessment

No. A shared memory store may be useful, but this branch is not the best merge path until it is rebased to the current memory plugin/host SDK layout, preserves existing `extraPaths` semantics or intentionally changes them with docs, fixes path-scope bugs, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths are mapped into `settings.sharedPaths`. Because `resolveUserPath("")` returns an empty string and the read gate later checks `${sp.path}${path.sep}`, a config typo can authorize broad absolute-path markdown reads instead of only the intended shared directory.
  - confidence: 0.88
- **[P1] Keep default extraPaths agent-scoped:** `src/agents/memory-search.ts:234-263`
  - body: This soft-migrates `agents.defaults.memorySearch.extraPaths` into the shared store and drops default paths from `extraPaths`, changing the documented/current behavior where defaults and per-agent overrides are merged for each agent. Existing configs can silently index different locations and lose per-agent behavior.
  - confidence: 0.87
- **[P2] Avoid mkdir on shared file paths:** `src/memory/manager.ts:250-251`
  - body: Shared paths are treated like extra paths and may be individual markdown files, but the constructor calls `ensureDir(sp.path)` for every configured entry. A configured `policy.md` path can be created as a directory and then block the intended file from being indexed.
  - confidence: 0.8
- **[P2] Apply shared path overrides last:** `src/memory/manager.ts:444-447`
  - body: The weighting loop stops at the first matching shared path. When defaults and an agent override contain the same path with different weights, the default entry wins and the agent-specific weight override is ignored.
  - confidence: 0.82
- **[P2] Use the actual shared store path for status:** `src/agents/memory-search.ts:265`
  - body: `sharedStorePath` is hard-coded to the default `_shared.sqlite` path, while the `_shared` manager can use a custom `memorySearch.store.path` template. With a custom store path, shared status reports counts from the wrong database.
  - confidence: 0.78
