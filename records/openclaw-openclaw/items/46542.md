  - body: The PR moves `agents.defaults.memorySearch.extraPaths` into `sharedPaths` and then builds per-agent `extraPaths` only from overrides.   - body: Current main registers the memory CLI/runtime through `extensions/memory-core` and `packages/memory-host-sdk`, while this PR adds the shared manager to `src/memory/manager.ts`, a stale path that no longer owns the active memory runtime. After rebase this would either conflict or leave the real plugin-backed command path unchanged.
  - confidence: 0.88
- **[P2] Reject blank shared path entries before resolving them:** `src/agents/memory-search.ts:225-230`
  - body: The new `sharedPaths` mapping accepts strings and object paths without trimming/filtering blanks before they are later used in shared read-prefix checks. A blank entry can turn the shared read check into a broad path match, widening access beyond the configured shared directories.
  - confidence: 0.86
- **[P2] Avoid creating directories for configured shared file paths:** `src/memory/manager.ts:248-251`
  - body: The constructor calls `ensureDir` for every configured shared path, but shared paths are treated like extra paths and can be individual Markdown files. Configuring a not-yet-created file such as `policy.md` would create a directory at that path and block the intended file from being indexed.
  - confidence: 0.82
