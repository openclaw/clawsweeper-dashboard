## Summary

Keep open. The requested capability is still absent from current main, but this PR is not mergeable as-is because current memory is plugin-owned and the diff targets older core memory paths while leaving concrete path/security bugs.

## What This Changes

The PR adds a core `_shared` memory index, `memorySearch.sharedPaths`, shared memory CLI flags, result origin tagging, and reserved `_shared` agent validation.

## Best Possible Solution

Rework the feature in the active `memory-core` plugin and host SDK surfaces with explicit opt-in config, strict canonical shared path validation, deterministic default/agent override precedence, docs, and focused builtin/QMD tests.

## Reproduction Assessment

Yes for review purposes: current-main source and docs show no `sharedPaths`, `_shared`, or `--shared` behavior, and the PR diff has deterministic code paths for the review findings. I did not run live indexing because this was a read-only review.

## Solution Assessment

No. The direction may be useful, but this PR is not currently the best solution because it patches old core memory files instead of the active plugin-owned runtime and leaves concrete path/security bugs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject empty shared paths before allowing reads:** `src/config/zod-schema.agent-runtime.ts:601`
  - body: `sharedPaths` accepts blank strings/paths, and the later shared read check treats an empty resolved path as a broad prefix match. Trim and reject empty shared path entries before resolving or using them for read authorization.
  - confidence: 0.91
- **[P2] Resolve agent extraPaths relative to the workspace:** `src/agents/memory-search.ts:254-258`
  - body: This maps per-agent `extraPaths` through `resolveUserPath`, so relative entries are resolved from the process cwd instead of the agent workspace. Existing indexing resolves these through `normalizeExtraMemoryPaths(workspaceDir, ...)`, so this can silently drop or retarget current memory files.
  - confidence: 0.88
- **[P2] Apply override weights when shared paths repeat:** `src/memory/manager.ts:428-432`
  - body: The shared result weighting loop stops at the first matching path. Because merged config keeps defaults before agent overrides, an agent-specific weight for the same path is ignored.
  - confidence: 0.86
- **[P2] Report status from the actual shared store path:** `src/memory/manager.ts:881-884`
  - body: Shared status reads `this.settings.sharedStorePath`, but the `_shared` manager can persist to its resolved `settings.store.path`. With a custom store path, status can report the wrong DB and stale zero counts after indexing succeeds.
  - confidence: 0.84
- **[P2] Avoid mkdir on file shared paths:** `src/memory/manager.ts:252-254`
  - body: The constructor eagerly calls `ensureDir` for every shared path even though shared paths are treated like `extraPaths` and may be individual Markdown files. A future file path such as `policy.md` can be created as a directory and block indexing the intended file.
  - confidence: 0.8
