## Summary

Keep open. Current main still does not implement the PR's `sharedPaths`, `_shared` store, or `--shared` memory CLI behavior, so this is not obsolete; however, the proposed patch is not merge-ready because it targets stale core memory paths instead of the active `memory-core` plugin surface and introduces concrete cross-agent path/read-scope bugs.

## What This Changes

The PR adds a `_shared` memory index, `memorySearch.sharedPaths`, shared-result merging and origin tagging, reserved `_shared` agent validation, and `--shared` flags for memory status, index, and search.

## Best Possible Solution

Rework the feature against the active memory plugin architecture, either in `memory-core` or a dedicated ClawHub memory plugin, with strict non-empty canonical shared path validation, deterministic default/agent override precedence, correct custom store-path status, docs/changelog coverage, and focused builtin/QMD tests.

## Reproduction Assessment

Yes for review purposes: current-main source inspection shows the requested `sharedPaths`, `_shared`, and `--shared` behavior is absent, and the proposed diff has deterministic path/config cases for the review findings. I did not run live indexing because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this PR is not the best way to solve it because it patches old core memory surfaces instead of the active plugin-owned runtime and leaves concrete correctness and security defects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject empty shared paths before allowing reads:** `src/config/zod-schema.agent-runtime.ts:601-607`
  - body: `sharedPaths` accepts empty or whitespace-only entries, which can resolve into a broad path prefix for the new shared read check. Trim and reject blank shared paths before resolving or matching them so a config typo cannot widen `memory_get` access outside the intended shared directories.
  - confidence: 0.91
- **[P2] Resolve agent extraPaths relative to each workspace:** `src/agents/memory-search.ts:262`
  - body: The PR resolves per-agent `extraPaths` with user-path resolution before indexing, which makes relative entries resolve from the process cwd instead of the agent workspace. Preserve the existing `normalizeExtraMemoryPaths(workspaceDir, ...)` behavior so configs like `../team-docs` keep pointing at the same files.
  - confidence: 0.88
- **[P2] Apply override weights when shared paths repeat:** `src/memory/manager.ts:428`
  - body: Merged shared paths keep defaults before agent overrides, but the score weighting loop matches the first path. If an agent overrides the weight for a default shared path, the default weight wins and the per-agent override silently does nothing.
  - confidence: 0.86
- **[P2] Report status from the actual shared store path:** `src/memory/manager.ts:881`
  - body: Shared status reads `settings.sharedStorePath`, while the `_shared` manager can persist to its resolved `settings.store.path`. With a custom store path, `memory status` can report the wrong database and stale zero counts after indexing succeeds.
  - confidence: 0.84
- **[P2] Avoid mkdir on file shared paths:** `src/memory/manager.ts:252`
  - body: The constructor calls `ensureDir` for every shared path, but shared paths are otherwise treated like `extraPaths` and may be individual Markdown files. A configured file path can be created as a directory, blocking the intended file from being indexed.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The proposed diff introduces a cross-agent read surface with a concrete blank-path validation gap.

Concerns:

- **[medium] Blank shared path can broaden memory reads:** `src/config/zod-schema.agent-runtime.ts:601`
  - body: Because the schema accepts empty shared paths and the shared read check matches path prefixes, an empty path can become an overly broad absolute-path prefix on POSIX and allow reads outside explicitly configured shared directories.
  - confidence: 0.9
