## Summary

Keep open. The shared-memory idea is still the canonical thread for cross-agent memory sharing, but the current branch is not mergeable as-is: it predates the current memory-core/host SDK layout, changes the existing `extraPaths` contract, has source-visible path-scope bugs, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a shared `_shared` memory store with `memorySearch.sharedPaths`, shared-result merging, reserved-agent validation, and `openclaw memory ... --shared` CLI support.

## Best Possible Solution

Rebase or replace the feature in the current `extensions/memory-core` and `packages/memory-host-sdk` architecture, preserve or explicitly redesign the `extraPaths` contract, strictly validate shared-path scope, document the new config/CLI surface, and add redacted live proof for shared index/status/search.

## Reproduction Assessment

Yes for the PR review defects: PR-head source shows blank shared-path acceptance, default `extraPaths` migration, eager directory creation for configured shared paths, first-match weighting, and hard-coded shared status paths. This is not a current-main bug reproduction because current main does not implement a shared `_shared` memory store.

## Solution Assessment

No. The requested capability may be valuable, but this branch is not the best merge path until it is reworked against the current memory plugin/host SDK layout, fixes the path-scope bugs, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths are accepted and passed through `resolveUserPath`, which returns an empty path. The later shared read-prefix check can then treat that empty path as a broad absolute-path prefix, widening Markdown read access beyond configured shared directories.
  - confidence: 0.88
- **[P1] Preserve default extraPaths semantics:** `src/agents/memory-search.ts:234-263`
  - body: This migrates `agents.defaults.memorySearch.extraPaths` into the shared store and removes defaults from agent-local `extraPaths`. Current main and QMD tests define default plus per-agent `extraPaths` as merged per-agent indexing scope, so existing configs can silently search/index a different set of files.
  - confidence: 0.87
- **[P2] Skip mkdir for shared file paths:** `src/memory/manager.ts:250-251`
  - body: Shared paths are treated like `extraPaths` and may be individual Markdown files, but the constructor calls `ensureDir` for every configured shared path. A configured `policy.md` path can be created as a directory and block the intended file from being indexed.
  - confidence: 0.82
- **[P2] Apply shared path overrides last:** `src/memory/manager.ts:444-447`
  - body: The weighting loop stops at the first matching shared path. When defaults and an agent override include the same path with different weights, the default entry wins and the agent-specific weight override is ignored.
  - confidence: 0.8
- **[P2] Use the actual shared store path for status:** `src/memory/manager.ts:893-907`
  - body: Shared status reads and reports `settings.sharedStorePath`, but the `_shared` manager can persist to `settings.store.path` via a custom `memorySearch.store.path` template. With a custom store path, status can show the wrong database and stale counts.
  - confidence: 0.78
