## What This Changes

The PR adds memorySearch.sharedPaths, a reserved _shared memory store, shared memory CLI flags, shared result tagging and weighting, and focused tests for cross-agent document sharing.

## Best Possible Solution

Rework the feature into the current memory-core and memory-host SDK layout, preserve or explicitly migrate the extraPaths contract, strictly validate shared path scope, document the new config and CLI surface, and attach redacted live proof.

## Reproduction Assessment

Yes for the PR review defects: PR-head source shows the path validation, extraPaths migration, shared path directory creation, weighting, and status-path issues. This is not a current-main bug reproduction because current main does not implement a shared _shared memory store.

## Solution Assessment

No. The shared-memory direction may be useful, but this branch is not the best merge path until it is rebased to current memory-core architecture, preserves existing contracts, fixes path-scope bugs, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default extraPaths in agent-local indexing:** `src/agents/memory-search.ts:234-263`
  - body: This moves agents.defaults.memorySearch.extraPaths into the shared store and builds agent-local extraPaths only from overrides. Current main and the docs merge defaults plus per-agent paths for each agent, so existing configs would silently change index/read scope.
  - confidence: 0.88
- **[P1] Resolve agent extraPaths relative to workspace:** `src/agents/memory-search.ts:259-263`
  - body: Mapping override extraPaths through resolveUserPath resolves relative entries before normalizeExtraMemoryPaths() can resolve them against the agent workspace. Existing configs such as ../team-docs can point at the process cwd instead and miss files.
  - confidence: 0.84
- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths are accepted and mapped into settings.sharedPaths. The shared read gate later treats each shared path as a prefix, so an empty entry can widen Markdown reads outside intended shared directories.
  - confidence: 0.88
- **[P2] Avoid creating configured shared files as directories:** `src/memory/manager.ts:248-252`
  - body: Shared paths inherit from extraPaths, which supports individual Markdown files, but the constructor calls ensureDir(sp.path) for every configured entry. A shared policy.md path can be created as a directory and block the intended file.
  - confidence: 0.81
- **[P2] Apply agent sharedPath weight overrides after defaults:** `src/memory/manager.ts:444-447`
  - body: The weighting loop stops at the first matching shared path. When defaults and an agent override contain the same path with different weights, the default entry wins and the agent-specific weight override is ignored.
  - confidence: 0.82
- **[P2] Read shared status from the actual store path:** `src/agents/memory-search.ts:265`
  - body: The resolver hard-codes sharedStorePath to _shared.sqlite even though the _shared manager can use a custom memorySearch.store.path template. With a custom store path, shared status can report the wrong database and stale counts.
  - confidence: 0.78
