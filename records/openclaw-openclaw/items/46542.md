## Summary

Keep open: the shared-memory feature is not implemented on current main, and this PR still needs a rebase plus blocking correctness, security, and real-behavior-proof work before it is mergeable.

## What This Changes

The PR adds `memorySearch.sharedPaths`, a reserved `_shared` memory store, shared result merging, `--shared` memory CLI paths, and related config/memory tests.

## Best Possible Solution

Rework the feature against the current memory-core plugin and host SDK contracts, preserve or explicitly migrate `extraPaths`, add strict shared-path validation/docs, and provide redacted live proof for shared index/status/search.

## Reproduction Assessment

Yes for the PR review defects: PR-head source shows blank shared-path acceptance, default `extraPaths` migration, eager directory creation for configured shared paths, first-match weighting, and hard-coded shared status paths. This is not a current-main bug reproduction because current main does not implement a shared `_shared` memory store.

## Solution Assessment

No. The concept may be useful, but this branch is not the best merge path until it is rebased to the current memory plugin/host SDK layout, preserves or intentionally migrates `extraPaths`, fixes path-scope bugs, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject blank sharedPaths before resolving:** `src/agents/memory-search.ts:228-231`
  - body: Blank string or object paths are accepted and mapped through `resolveUserPath`, which returns an empty string. The shared read gate later treats that empty path as a broad prefix, so a config typo can authorize Markdown reads outside the intended shared directories.
  - confidence: 0.88
- **[P1] Preserve default extraPaths semantics:** `src/agents/memory-search.ts:234-263`
  - body: This soft-migrates `agents.defaults.memorySearch.extraPaths` into the shared store and removes defaults from agent-local `extraPaths`. Current main merges default and per-agent paths for each agent, so existing configs can silently index/search a different scope.
  - confidence: 0.87
- **[P2] Avoid mkdir on shared file paths:** `src/memory/manager.ts:250-251`
  - body: Shared paths are treated like extra paths and may be individual Markdown files, but the constructor calls `ensureDir(sp.path)` for every configured entry. A configured `policy.md` path can be created as a directory and then block the intended file from being indexed.
  - confidence: 0.81
- **[P2] Apply shared path overrides last:** `src/memory/manager.ts:444-447`
  - body: The weighting loop stops at the first matching `sharedPaths` entry. When defaults and an agent override contain the same path with different weights, the default entry wins and the agent-specific override weight is ignored.
  - confidence: 0.82
- **[P2] Use the actual shared store path for status:** `src/memory/manager.ts:891-894`
  - body: Shared status reads and reports `settings.sharedStorePath`, while the `_shared` manager can use a custom `memorySearch.store.path` template. With a custom store path, status can show the wrong database and stale counts.
  - confidence: 0.78
