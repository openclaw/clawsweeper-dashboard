## Summary

Keep open, but this PR is not mergeable as-is: it targets the pre-plugin memory layout and also adds a shared-path read surface that needs tighter validation before maintainer review.

## What This Changes

The PR adds a shared `_shared` memory store, `memorySearch.sharedPaths`, `--shared` memory CLI flags, reserved `_shared` agent-id validation, and shared-path indexing tests.

## Best Possible Solution

Port the idea through the current memory-core plugin and memory-host SDK boundaries, with strict path validation, preserved workspace-relative `extraPaths` behavior, docs/changelog updates, and clear guidance on when QMD extra collections are the preferred path.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: the diff modifies old core memory files that are absent from current main, and the proposed schema/read check allows blank shared paths to widen reads.

## Solution Assessment

No. The feature direction may be useful, but this implementation is not the best current solution because it bypasses the current plugin-owned memory architecture and needs a tighter data-access contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Port shared memory to the memory-core plugin:** `src/cli/memory-cli.ts:628-648`
  - body: This PR adds the new `--shared` CLI path under the old `src/cli/memory-cli.ts` surface, but current main registers memory commands from `extensions/memory-core/src/cli.ts` and runs them through `extensions/memory-core/src/cli.runtime.ts`. As written, the feature is stale against main and the new flags/manager wiring will not merge or ship without being ported to the plugin-owned memory surfaces.
  - confidence: 0.93
- **[P1] Reject blank shared paths before enabling reads:** `src/config/zod-schema.agent-runtime.ts:593-603`
  - body: `sharedPaths` accepts raw strings and object paths without trimming or requiring a non-empty value. A blank entry resolves to an empty path, and the new shared `readFile` check treats `${sp.path}${path.sep}` as the root prefix, which can allow reads of any absolute Markdown path instead of only configured shared directories. Trim/filter or reject blank paths before storing them.
  - confidence: 0.88
- **[P2] Preserve workspace-relative extraPaths semantics:** `src/agents/memory-search.ts:220-261`
  - body: The merge logic now resolves default `extraPaths` with `resolveUserPath` and removes defaults from `extraPaths`, but current docs and memory-host code treat `extraPaths` as workspace-relative when not absolute and merge defaults with per-agent overrides. Relative default paths such as `../team-docs` would be indexed from the process cwd/shared store instead of each agent workspace, changing existing behavior despite the PR's backward-compatibility claim.
  - confidence: 0.82
