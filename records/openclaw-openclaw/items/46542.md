## What This Changes

The PR adds an opt-in `_shared` memory store with `sharedPaths` config, shared memory CLI flags, shared-result tagging, reserved `_shared` agent validation, and tests around schema and file-list exclusion.

## Best Possible Solution

Rework the feature on the current memory-core and memory-host-sdk architecture with strict shared-path validation, preserved `extraPaths` compatibility, plugin-owned CLI/status integration, docs/schema updates, and targeted regression tests before reconsidering merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve workspace-relative extraPaths:** `src/agents/memory-search.ts:262-263`
  - body: Existing `extraPaths` entries are normalized against each agent workspace by `normalizeExtraMemoryPaths`, and docs show values like `../team-docs`. Resolving per-agent `extraPaths` here with `resolveUserPath` makes relative paths point at the process cwd instead, so existing memory files can be silently missed.
  - confidence: 0.9
- **[P1] Reject empty sharedPaths before granting reads:** `src/config/zod-schema.agent-runtime.ts:599-601`
  - body: The schema accepts empty or whitespace shared path strings, which `mergeConfig` stores as an empty path. In `readFile`, `${sp.path}${path.sep}` then becomes `/` on POSIX, so any absolute `.md` path can satisfy the shared-path check; trim and reject empty paths before they reach the manager.
  - confidence: 0.88
- **[P2] Avoid creating directories for shared file paths:** `src/memory/manager.ts:251`
  - body: Shared paths are later passed to `listMemoryFiles` as extra paths, which supports individual `.md` files as well as directories. Calling `ensureDir(sp.path)` here creates a directory at a not-yet-created file path such as `~/policy.md`, preventing the intended file from being created and indexed.
  - confidence: 0.82
- **[P2] Apply override weights after defaults:** `src/memory/manager.ts:444-447`
  - body: When the same shared path appears in defaults and in an agent override with a different weight, the merged list has the default first and this loop stops at that first match. Per-agent weight overrides therefore never take effect; dedupe with override precedence or scan matches in reverse order.
  - confidence: 0.84
- **[P2] Report counts from the actual shared store path:** `src/memory/manager.ts:884`
  - body: The shared manager writes to its resolved `settings.store.path`, which can be customized with `memorySearch.store.path`, but shared status reports and reads from the fixed `sharedStorePath`. With a custom store path, `memory status` can show the wrong DB and stale zero counts after successful indexing.
  - confidence: 0.82
- **[P3] Emit the extraPaths migration warning once:** `src/agents/memory-search.ts:237-240`
  - body: `mergeConfig` runs for repeated config/status/search resolution, so users with `agents.defaults.memorySearch.extraPaths` would get the deprecation warning repeatedly during normal memory operations. Add a process-local once guard or emit it from a less frequently called migration/doctor path.
  - confidence: 0.75

## Security Review

Status: needs_attention

Summary: The diff expands memory read scope and currently lets an empty shared path turn into broad host Markdown read access.

Concerns:

- **[medium] Empty sharedPaths can allow broad Markdown reads:** `src/config/zod-schema.agent-runtime.ts:599`
  - body: `sharedPaths` accepts blank strings, and the shared read guard checks `absPath.startsWith(`${sp.path}${path.sep}`)`. With an empty path on POSIX, that prefix is `/`, so a misconfigured shared path can authorize reads of arbitrary absolute `.md` files outside the intended directories.
  - confidence: 0.88
