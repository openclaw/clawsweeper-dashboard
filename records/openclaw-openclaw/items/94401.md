The PR imports the shared transcript-only OpenClaw assistant predicate into session-memory transcript extraction and adds a regression test for delivery-mirror and gateway-injected rows.

## Best Possible Solution

Make session-memory skip only redundant mirror or injected rows with a matched raw assistant-text duplicate while preserving standalone visible mirror and nonduplicate injected replies, then land one canonical fix for https://github.com/openclaw/openclaw/issues/92563.

## Reproduction Assessment

Yes, at source level. Current getRecentSessionContent reads every user/assistant text block, so a raw assistant text row followed by a delivery-mirror text row produces duplicate assistant lines; I did not run a live provider flow in this read-only review.

## Solution Assessment

No. Filtering by the broad transcript-only helper is a plausible mitigation for the duplicate case, but it is not the best fix because it also removes visible mirror-only and nonduplicate gateway-injected assistant rows that other current history surfaces preserve.
