## Summary

Keep open, but the PR is not merge-ready: it adds a new memory embedding config feature through stale `src/memory` files, while current main routes active memory embeddings through the memory-host SDK/provider adapters and documents an omit-and-normalize compatibility direction for OpenAI-compatible providers.

## What This Changes

The branch adds optional `memorySearch.remote.encodingFormat` config, sends `encoding_format` on old remote embedding direct and batch requests, and adds base64 decode helpers for old memory embedding responses.

## Best Possible Solution

Implement this only if maintainers want a provider-specific encoding knob; otherwise keep the documented omit-and-normalize policy and route OpenAI-compatible endpoint work through the provider/plugin path, including https://github.com/openclaw/openclaw/pull/84998 if that stack lands.
