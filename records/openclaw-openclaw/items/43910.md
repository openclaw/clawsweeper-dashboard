## Summary

Keep open: the requested configurable embedding encoding feature is not implemented on current main, and this PR is not currently mergeable because it targets retired `src/memory` paths while the active config and provider paths live elsewhere.

## What This Changes

The PR adds an optional `memorySearch.remote.encodingFormat` setting and threads `float`/`base64` encoding through remote embedding requests and base64 response parsing in the old `src/memory` implementation.

## Best Possible Solution

Rebase onto current main and implement the option through the active schema/types/help, memory-search config merge, memory-host SDK request/parser helpers, and provider direct/batch adapters with omit-by-default compatibility, focused tests, docs, and redacted real-provider proof.

## Reproduction Assessment

Yes for the review blockers: current-main source inspection shows the option is absent from active config, request, and parser paths. The original request is a new configurable capability rather than a broken existing-behavior bug reproduction.

## Solution Assessment

No. The configurable direction may be useful, but this patch updates stale `src/memory` files and misses the active memory-host SDK, provider adapters, and config merge path.
