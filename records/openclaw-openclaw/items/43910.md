## Summary

Keep open: the requested configurable embedding encoding may be useful, but this branch is stale against current `main`, targets removed `src/memory` embedding paths, lacks real provider proof, and needs maintainer API/product judgment before a replacement implementation.

## What This Changes

The PR adds an optional `memorySearch.remote.encodingFormat` config key and attempts to pass/decode `encoding_format` for remote embedding direct and batch requests.

## Best Possible Solution

Expose `encodingFormat` only if maintainers want the generic knob, implemented through the active schema, resolver, memory-host SDK contracts, provider adapters, direct and batch parsers, docs, focused tests, and real-provider proof.

## Reproduction Assessment

Yes for the PR blockers: source inspection on current `main` shows the active config/provider paths omit `encodingFormat`, while the PR diff edits removed or stale `src/memory` embedding files. The requested capability itself is a new config feature, not a broken existing-behavior bug reproduction.

## Solution Assessment

No. The direction may be useful, but this patch is not the maintainable implementation because it misses the active memory-host/provider surfaces, does not preserve the option through resolved config, and lacks real behavior proof.
