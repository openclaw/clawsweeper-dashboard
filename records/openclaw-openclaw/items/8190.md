## Summary

Keep open. Current main still exposes only single-session/current-session reset and cleanup maintenance paths; it does not provide a global, context-preserving auth refresh/reset, and the related auth-order, auth-migration, and auth-epoch items cover narrower problems rather than superseding this request.

## What This Changes

The issue requests a global, context-preserving session auth reset/refresh for auth backend or API key switches across active channels.

## Best Possible Solution

Add an additive admin-scoped session auth refresh capability for all or selected active sessions that invalidates auth-backed runtime/CLI bindings without transcript rollover, preserves session ids/context where feasible, and documents failure semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior against an existing contract. Source inspection gives high confidence that current main only has single-session/current-session reset paths and no global context-preserving auth-refresh action.

## Solution Assessment

Unclear as proposed. Looping existing sessions.reset is not the best solution because it hard-rolls session ids and transcripts; the maintainable direction is a purpose-built admin-scoped auth refresh lifecycle.
