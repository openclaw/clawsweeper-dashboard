## Summary

Current main still lacks a gateway-wide, auth-only session refresh: the public RPC requires one session key, hard reset rolls session identity and transcripts, and soft reset preserves context but operates on only the addressed session. The related auth-migration and Windows provider-switch issues cover adjacent persisted-profile and concrete onboarding failures, not this cross-session runtime lifecycle capability, so this issue remains necessary and needs a maintainer product/security decision.

## What This Changes

Add a global, context-preserving refresh of authentication state for active sessions after provider, account, or credential changes.

## Best Possible Solution

Add an additive operator-admin Gateway capability that enumerates canonical agent session stores, invalidates only auth-derived runtime and CLI bindings by default, preserves session IDs/transcripts/context and explicit user overrides, and defines deterministic in-flight-run and partial-failure behavior; coordinate persisted profile migration separately with https://github.com/openclaw/openclaw/issues/61709.

## Reproduction Assessment

Yes at source level: current main has only a required-key sessions.reset RPC and a per-session soft reset, with no global auth-only refresh path. A live multi-channel run was not performed, but the missing API and lifecycle operation are directly verifiable in source.

## Solution Assessment

No, a global hard reset or transcript export/restore is not the best solution because it changes session identity and introduces unnecessary state migration. The narrow maintainable design is auth-derived state invalidation that preserves canonical session data and explicit choices.

## Risks / Open Questions

- A global refresh must distinguish auto-selected auth from explicit user-pinned session overrides; clearing both would silently overwrite persisted user preference.
- The operation must define whether in-flight runs finish with their existing credentials, are aborted, or block refresh, otherwise one logical workflow may straddle credential identities.
- Enumerating and mutating all agent session stores is an operator-admin and sensitive credential-lifecycle action that needs explicit authorization, audit/result reporting, and partial-failure semantics.
