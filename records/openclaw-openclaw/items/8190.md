## Summary

Keep open. Current main still exposes keyed hard reset and per-session soft cleanup only, while the requested all-active-session auth refresh with context preservation remains a new Gateway/session lifecycle feature; #61709 is related auth-store migration work, not a substitute for this session reset surface.

## What This Changes

Add a global auth-switch reset or refresh that invalidates auth caches for all active sessions while preserving conversation context.

## Best Possible Solution

Define an additive Gateway/session capability that enumerates active sessions and invalidates auth-backed runtime or CLI bindings while preserving session ids, transcripts, and conversation context where each runtime supports it.

## Reproduction Assessment

Yes for source-inspection verification of the missing feature, not as a runtime bug reproduction. Current main exposes keyed sessions.reset plus per-session soft reset cleanup, and the reset service performs a hard rollover.

## Solution Assessment

Unclear until maintainers settle the protocol and preservation contract. A new additive refresh/reset-all API is likely appropriate, but applying the existing hard reset globally is not the best solution.
