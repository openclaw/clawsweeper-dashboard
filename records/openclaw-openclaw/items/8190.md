## Summary

Keep open. Current main still has single-session reset only, auth profile selection remains session-pinned, and the related auth-switching issues do not replace this cross-session, context-preserving auth refresh request.

## What This Changes

The issue requests a global auth/session refresh or reset that updates all active channel sessions after auth changes while preserving conversation context.

## Best Possible Solution

Define an additive admin-scoped Gateway/session capability that refreshes auth-backed runtime bindings across selected active sessions while preserving session ids, transcripts, and context where possible.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs provide a high-confidence check: auth profiles are pinned per session, and only single-session reset/soft-reset surfaces exist today.

## Solution Assessment

Unclear as proposed. A gateway action such as sessions.refreshAuth may be the right direction, but the best solution needs owner-approved protocol, security, and context-preservation semantics before implementation.
