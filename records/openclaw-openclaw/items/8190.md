## Summary

Keep open: current main still lacks a global, context-preserving all-session auth refresh. Existing reset paths are single-session scoped, current reset behavior rolls session identity/transcripts, and the related auth-switching issues do not replace this product/security-sensitive lifecycle request.

## What This Changes

Request a global gateway/session action that refreshes or resets auth-backed session state across all active channels after an auth switch while preserving conversation context.

## Best Possible Solution

Define an additive admin-scoped session auth refresh capability that can target selected active sessions, refresh auth-backed runtime bindings, preserve session ids/transcripts/context where possible, and document the security and failure semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: only single-session reset and current-session soft-reset surfaces exist, and no global auth-refresh method is registered.

## Solution Assessment

Unclear as proposed. A sessions.refreshAuth-style capability is plausible, but the maintainable solution needs maintainer-approved protocol, security, scope, and context-preservation semantics first.
