## Summary

Current main still does not provide a global, context-preserving auth refresh/reset for active sessions; the existing reset surfaces remain single-session/current-session scoped or hard-roll session state, and the related auth migration work does not cover active channel session refresh.

## What This Changes

Request a gateway-level command or protocol action that refreshes or resets auth for all active sessions while preserving conversation context after switching auth backends or API keys.

## Best Possible Solution

Define an additive operator-admin session auth refresh capability that can target all or selected active sessions, refresh credential-backed runtime bindings, preserve session ids/transcripts/context where possible, and document security and failure semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: only single-session reset/current-session soft reset and per-run auth/session invalidation surfaces exist, not a global context-preserving auth refresh.

## Solution Assessment

Unclear as proposed. A sessions.refreshAuth-style capability is plausible, but the maintainable solution needs maintainer-approved protocol, security, target-scope, locking, and context-preservation semantics first.
