## Summary

Keep open. Current main still lacks a global all-session auth refresh/reset, and the existing reset paths are single-session scoped with hard session rollover rather than the requested auth-only context-preserving refresh.

## What This Changes

The issue requests a Gateway-level action or command to refresh/reset auth across all active sessions after an auth switch while preserving conversation context.

## Best Possible Solution

Keep this as the canonical core Gateway/session lifecycle tracker for an additive all-session auth refresh API that invalidates auth-backed runtime/CLI bindings while preserving session ids, transcripts, and context where feasible.

## Reproduction Assessment

Yes for source-inspection verification, not as a runtime bug reproduction. Current main exposes only single-session reset and per-session soft CLI binding cleanup, with no global auth-refresh endpoint or all-session reset action.

## Solution Assessment

Unclear until maintainers settle the API contract. Reusing the existing hard reset globally is not the best solution because it creates new session ids and archives transcripts; the safer direction is an auth-only refresh with explicit all-session scope.
