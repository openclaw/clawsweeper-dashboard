## Summary

Keep open. Current main still lacks a global, context-preserving all-session auth refresh/reset; the existing pieces are per-session reset, per-session soft reset, and a narrower CLI auth-epoch invalidation path.

## What This Changes

The issue requests a global, context-preserving auth refresh or session reset for all active channel sessions after auth/backend changes.

## Best Possible Solution

Add an additive admin-scoped Gateway/session capability that refreshes auth-related runtime/session bindings across selected or all active sessions without reusing the hard transcript-rolling reset path, with protocol, channel/session-store, tests, and docs coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main exposes only per-session reset/soft-reset plus narrower CLI auth-epoch invalidation, not the requested global auth refresh.

## Solution Assessment

Unclear as proposed. The requested direction is valid, but the safest implementation should be an additive auth-refresh/session-lifecycle capability rather than looping the current hard `sessions.reset` over every session.
