## Summary

Keep open. Current main still has per-request CLI auth-epoch invalidation and single-session reset paths, but no global, context-preserving auth refresh/reset action across active channel sessions.

## What This Changes

Request an operator action that refreshes or resets all active sessions after auth/backend changes while preserving conversation context.

## Best Possible Solution

Add an additive admin-scoped session/auth refresh capability that targets all or selected active sessions, invalidates auth-backed runtime and CLI bindings without hard transcript rollover, preserves session ids/context where feasible, and documents active-run and failure semantics.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection of current main gives high confidence that reset surfaces are still single-session/current-session scoped and no global auth-refresh method exists.

## Solution Assessment

Unclear as proposed. Reusing existing sessions.reset globally would hard-roll sessions, so the maintainable path is a purpose-built admin-scoped auth-refresh lifecycle that reuses current auth-epoch/runtime-routing checks where appropriate.
