## Summary

Keep open. Current main still exposes keyed hard reset and per-session soft cleanup only, while the requested all-session auth refresh with context preservation remains a new Gateway/session lifecycle feature; related auth-switching items do not supersede this request.

## What This Changes

Add a global, context-preserving session auth refresh/reset for switching auth backends or API keys across active channel sessions.

## Best Possible Solution

Define an additive Gateway/session API that enumerates active sessions and invalidates auth-backed runtime and CLI bindings while preserving session ids, transcripts, and conversation context wherever supported.

## Reproduction Assessment

Yes for source-inspection verification of the missing feature, not as a runtime bug reproduction. Current main exposes only keyed `sessions.reset` plus per-session soft reset cleanup, and the reset service performs a hard rollover.

## Solution Assessment

Unclear until maintainers settle the API contract. Reusing the existing hard reset globally is not the best solution because it creates new session ids and archives transcripts rather than doing an auth-only refresh.
