## Summary

Keep open: current main still lacks a global, context-preserving all-session auth refresh/reset. Existing reset paths are single-session scoped or hard rollovers, and the related auth-order and auth-migration issues do not supersede this product/security-sensitive lifecycle request.

## What This Changes

Request an admin/global session auth refresh or reset so active sessions pick up changed provider credentials without losing conversation context.

## Best Possible Solution

Define an additive operator-admin session auth refresh capability that can target selected active sessions, refresh auth-backed runtime bindings, preserve session ids/transcripts/context where possible, and document security and failure semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: only single-session reset and current-session soft-reset surfaces exist, and no global auth-refresh method is registered.

## Solution Assessment

Unclear as proposed. A sessions.refreshAuth-style capability is plausible, but the maintainable solution needs maintainer-approved protocol, security, scope, and context-preservation semantics first.
