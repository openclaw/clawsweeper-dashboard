## Summary

Keep open. Current main still has per-session reset and current-session soft reset, but no global, context-preserving auth refresh/reset for all active sessions; the related auth migration issue tracks a different host-wide auth-store problem.

## What This Changes

Request a global auth/session reset or refresh so provider/API-key switches apply across active channel sessions without losing conversation context.

## Best Possible Solution

Add an additive admin-scoped Gateway/session capability that refreshes auth-backed runtime and CLI bindings across selected or all active sessions while preserving session ids, transcripts, and context where feasible.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence verification path: only per-session reset and current-session soft reset exist, with no global context-preserving auth refresh surface.

## Solution Assessment

Unclear as proposed. The maintainable direction is likely a new auth-refresh/session-lifecycle capability, not looping the current hard sessions.reset implementation across every active session.
