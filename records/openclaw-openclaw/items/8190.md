## Summary

Keep open. Current main and v2026.5.22 still expose only per-session reset, per-chat soft reset, and SecretRef reload; the related auth migration and no-model switch items do not replace an admin-scoped, context-preserving active-session auth refresh.

## What This Changes

The issue requests a global auth/session refresh or reset path that updates active sessions after auth/backend changes while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active sessions, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where safe, reports affected sessions, and documents coordination with auth migration and secrets.reload.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.22 expose per-session reset, current-session soft reset, and SecretRef reload rather than global active-session auth refresh.

## Solution Assessment

Yes for the direction, but not as a broad loop over the current hard reset service. The maintainable solution should add explicit admin-scoped auth-refresh semantics that preserve session identity and transcripts while invalidating auth-backed bindings.
