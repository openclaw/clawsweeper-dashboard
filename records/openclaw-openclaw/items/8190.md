## Summary

Keep open. Current main and v2026.5.22 still expose per-session reset, current-session soft reset, and SecretRef reload, but not an admin-scoped all-active-session auth refresh that preserves context; the related auth-migration issue is adjacent rather than a replacement.

## What This Changes

The issue requests a global session auth reset or refresh that updates all active sessions after auth/provider changes while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active sessions, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where safe, reports affected sessions, and documents coordination with auth migration and secrets.reload.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, tests, and latest-release inspection give high confidence that OpenClaw exposes per-session reset and current-session soft reset rather than global active-session auth refresh.

## Solution Assessment

Yes for the direction, but not as a broad loop over the current hard reset service. The maintainable solution should add explicit admin-scoped auth-refresh semantics that preserve session identity and transcripts while invalidating auth-backed bindings.
