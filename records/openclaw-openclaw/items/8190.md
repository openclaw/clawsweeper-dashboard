## Summary

Keep open: current main still lacks a global, context-preserving all-session auth refresh/reset. Existing paths either reset one addressed session, hard-roll session identity/transcripts, or only invalidate stored CLI session reuse on the next run; the related auth switching and auth migration issues do not supersede this active-session lifecycle request.

## What This Changes

Request a global auth-switch reset/refresh action that updates active sessions to current credentials while preserving conversation context across channels.

## Best Possible Solution

Define an additive operator-admin session auth refresh capability that can target selected active sessions, refresh auth-backed runtime bindings, preserve session ids/transcripts/context where possible, and document security and failure semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: only single-session reset/current-session soft-reset and per-run CLI auth-epoch invalidation exist, not a global context-preserving auth refresh.

## Solution Assessment

Unclear as proposed. A sessions.refreshAuth-style capability is plausible, but the maintainable solution needs maintainer-approved protocol, security, target-scope, locking, and context-preservation semantics first.
