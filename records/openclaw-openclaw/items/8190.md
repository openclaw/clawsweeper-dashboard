## Summary

Keep open: current main still exposes only single-session reset/soft-reset behavior, session auth selection remains session-pinned, and the requested cross-session auth refresh needs product and security design rather than cleanup closure.

## What This Changes

Request a global Gateway/session reset or auth-refresh action that invalidates cached credentials for all active sessions while preserving conversation context.

## Best Possible Solution

Define an additive admin-scoped Gateway/session capability that refreshes auth-backed runtime bindings across selected active sessions while preserving session ids, transcripts, and context where possible.

## Reproduction Assessment

Not applicable as a feature request. Source and docs give a high-confidence current-main check: auth profiles are pinned per session, and only single-session reset/soft-reset surfaces exist today.

## Solution Assessment

Unclear as proposed. A gateway action such as sessions.refreshAuth may be the right direction, but the best solution needs maintainer-approved protocol, security, and context-preservation semantics first.
