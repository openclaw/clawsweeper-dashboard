## Summary

Keep open: current main still lacks a global, context-preserving all-session auth refresh. Existing reset paths are single-session scoped, the hard reset rolls session identity/transcripts, and related auth-order work does not supersede this product/security-sensitive lifecycle request.

## What This Changes

Request a global session auth refresh/reset for auth backend or profile changes while preserving active conversation context across channels.

## Best Possible Solution

Define an additive admin-scoped Gateway/session capability that refreshes auth-backed runtime bindings across selected active sessions while preserving session ids, transcripts, and context where possible.
