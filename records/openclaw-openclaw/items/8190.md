## Summary

Keep open. Current main still has per-session hard reset, current-session soft reset, CLI auth invalidation, and SecretRef reload paths, but no admin-scoped all-session auth refresh that preserves active session context; the related auth-migration issue is adjacent rather than a substitute.

## What This Changes

The issue requests a global session auth reset or refresh action that updates all active channel sessions after an auth switch while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active session stores, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where possible, and documents coordination with host-wide auth migration.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: Gateway exposes single-session hard reset, current-session soft reset, CLI auth invalidation, and SecretRef reload paths, but no global active-session auth refresh surface.

## Solution Assessment

Yes for the direction, but it should be solved as an additive admin-scoped auth refresh rather than by looping the existing hard reset service over every session. The current hard reset service rolls session ids and transcripts, which does not match the requested context-preserving behavior.
