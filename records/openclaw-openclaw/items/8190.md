## Summary

Keep open. Current main still only supports single-session reset/soft-reset paths, and the related auth-profile migration issue does not cover global active-session auth refresh with context preservation.

## What This Changes

The issue requests a gateway-level way to refresh or reset all active sessions after auth/provider changes while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed Gateway/session capability that refreshes auth-backed runtime and CLI bindings across selected or all active sessions while preserving session ids, transcripts, and context wherever the channel/runtime contract allows.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: only single-session reset and session-local soft reset are implemented, with no global auth-refresh/reset method.

## Solution Assessment

Unclear until maintainer design review. The additive Gateway/session capability direction is plausible, but the maintainable solution needs explicit auth/security semantics and context-preservation rules before implementation.
