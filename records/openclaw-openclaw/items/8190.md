## Summary

Keep open. Current main and v2026.6.5 still expose per-session hard reset and current-session soft reset, but no global, context-preserving active-session auth refresh for provider or credential switches.

## What This Changes

Request a global gateway/session auth-refresh or reset action that updates cached credentials across active sessions while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active sessions, invalidates auth/runtime bindings, preserves session ids/transcripts/context where safe, reports affected sessions, and coordinates with auth migration/provider-switch work.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, tests, and latest-release inspection give high confidence that OpenClaw exposes per-session reset and current-session soft reset rather than global active-session auth refresh.

## Solution Assessment

Yes for the direction, but not as a broad loop over the current hard reset service. The maintainable solution should add explicit admin-scoped auth-refresh semantics that preserve session identity and transcripts while invalidating auth-backed bindings.
