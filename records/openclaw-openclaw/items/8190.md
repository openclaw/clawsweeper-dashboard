## Summary

Keep open. Current main and the latest release still have per-session reset, current-session soft reset, and SecretRef reload support, but no global active-session auth refresh that preserves context across channels; https://github.com/openclaw/openclaw/issues/61709 and https://github.com/openclaw/openclaw/issues/7533 are related but do not supersede this lifecycle request.

## What This Changes

The issue requests a global auth/session reset capability that refreshes active sessions after provider or API-key changes while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active sessions, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where safe, reports affected sessions, and documents coordination with host-wide auth migration.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.20 expose per-session reset, current-session soft reset, and SecretRef reload rather than global active-session auth refresh.

## Solution Assessment

Yes for the direction, but not as a broad loop over the current hard reset service. The maintainable solution should add explicit admin-scoped auth-refresh semantics that preserve session identity and transcripts while invalidating auth-backed bindings.
