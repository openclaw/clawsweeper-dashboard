## Summary

Keep open. Current main still lacks a global, context-preserving auth refresh/reset for active sessions; only single-session Gateway reset and current-session soft reset exist, and the related auth-migration/auth-epoch issues cover narrower problems.

## What This Changes

Request an operator-visible global action to invalidate auth caches for all active channel sessions after auth switching while preserving conversation context.

## Best Possible Solution

Add an additive admin-scoped session auth refresh capability for all or selected active sessions that invalidates auth-backed runtime/CLI bindings without hard transcript rollover, preserves session ids and context where feasible, and documents failure semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior against an existing contract. Source inspection gives high confidence that current main only exposes single-session/current-session reset paths and no global context-preserving auth-refresh action.

## Solution Assessment

Unclear as proposed. Looping `sessions.reset` is not the best fix because it hard-rolls session ids/transcripts; the maintainable direction is a purpose-built admin-scoped auth refresh lifecycle.
