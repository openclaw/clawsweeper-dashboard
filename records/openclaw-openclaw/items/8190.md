## Summary

Keep open. Current main still has only keyed per-session reset and per-session soft CLI binding cleanup, while the requested all-session auth refresh with context preservation remains a new Gateway/session lifecycle feature that needs maintainer API design.

## What This Changes

Add a global session auth refresh/reset action for auth backend or API-key switches that invalidates cached auth across active sessions while preserving conversation context.

## Best Possible Solution

Define an additive Gateway/session API that enumerates active sessions and invalidates auth-backed runtime and CLI bindings while preserving session ids, transcripts, and conversation context wherever supported.

## Reproduction Assessment

Yes for source-inspection verification of the missing feature, not as a runtime bug reproduction. Current main documents per-session auth stickiness and exposes only keyed sessions.reset plus per-session soft reset cleanup.

## Solution Assessment

Unclear until maintainers settle the API contract. Reusing the existing hard reset globally is not the best solution because it creates new session ids and archives transcripts rather than doing an auth-only refresh.
