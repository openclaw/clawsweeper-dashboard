## Summary

Keep open. Current main still lacks a global, context-preserving all-session auth refresh/reset; the implemented surfaces are single-session reset, current-session soft reset, and narrower CLI session reuse invalidation.

## What This Changes

The issue requests a Gateway/global action that refreshes all active sessions onto current auth after an auth/backend switch while preserving conversation context.

## Best Possible Solution

Add an additive admin-scoped Gateway/session capability that refreshes auth-backed runtime/session bindings across selected or all active sessions while preserving session ids/transcripts/context where feasible, with protocol, session-store, channel, tests, and docs coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main exposes per-session reset/soft reset plus per-binding CLI auth invalidation, but no global context-preserving auth refresh path.

## Solution Assessment

Unclear as proposed. The product direction is valid, but the maintainable fix should be an additive auth-refresh/session-lifecycle capability rather than looping the current hard `sessions.reset` over every active session.
