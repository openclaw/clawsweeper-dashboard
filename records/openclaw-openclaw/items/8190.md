## Summary

Keep open. Current main still has per-session hard reset, current-session soft reset, and SecretRef reload support, but no admin-scoped global active-session auth refresh that preserves context across channels.

## What This Changes

The issue requests a global session auth refresh/reset for provider or API-key switches that invalidates cached auth across active sessions while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped sessions.refreshAuth-style capability that enumerates active sessions, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where safe, reports affected sessions, and documents coordination with host-wide auth migration.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check that existing surfaces are per-session reset, current-session chat reset, and SecretRef reload helpers rather than global active-session auth refresh.

## Solution Assessment

Yes for the direction, but the maintainable solution should be an additive admin-scoped auth refresh rather than a broad loop over the current hard reset service. The current reset path rolls session identity and transcripts, which conflicts with the requested context-preserving behavior.
