## Summary

Keep open. Current main still has single-session/current-session reset paths and per-request CLI auth invalidation, but not a global, context-preserving auth refresh/reset action across active sessions.

## What This Changes

Request a gateway-level action or command to refresh all active sessions after auth/backend changes while preserving conversation context.

## Best Possible Solution

Add an additive admin-scoped session auth refresh capability that targets all or selected active sessions, clears auth-backed runtime/CLI bindings without hard transcript rollover, preserves session ids/context where feasible, and documents failure semantics.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection gives high confidence that current reset surfaces are single-session/current-session scoped and there is no global auth-refresh method.

## Solution Assessment

Unclear as proposed. Reusing the existing sessions.reset path globally would hard-roll session ids and transcripts, so the maintainable solution is a purpose-built admin auth-refresh lifecycle that reuses current auth-epoch checks where appropriate.
