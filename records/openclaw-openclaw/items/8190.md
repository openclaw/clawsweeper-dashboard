## Summary

Keep open: current main still has single-session hard reset plus per-session auth stickiness, while this issue asks for a new global, context-preserving auth refresh/reset across active sessions.

## What This Changes

Request to add a global auth-switch/session refresh action that invalidates active session auth caches while preserving conversation context.

## Best Possible Solution

Add an owner-reviewed, admin-scoped Gateway/session capability that enumerates active sessions, invalidates auth-backed runtime and CLI bindings, preserves session ids/transcripts/context where possible, and documents how it coordinates with host-wide auth profile migration.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: the Gateway protocol and handler still operate on one session key and documented auth profile stickiness remains per-session.

## Solution Assessment

Unclear until maintainer/security design. A global auth refresh is plausible, but it should be additive, admin-scoped, and context-preserving rather than broad use of the hard reset service.
