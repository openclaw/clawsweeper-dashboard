The issue asks OpenClaw to document whether QMD is practical as a local memory backend on CPU-only VPS hosts, including hardware guidance, embedding settings, scheduling safety, and possible warnings/defaults.

## Best Possible Solution

Add a maintainer-approved CPU-only QMD section to the QMD memory docs, separating shipped BM25-safe defaults from semantic embedding requirements and linking any upstream QMD guidance; track warning/default changes separately if maintainers choose them.
