## Summary

Keep open: current main already has the DeepSeek V4 profile in the full provider runtime, but the lightweight provider-policy artifact used for lazy profile/session resolution does not expose that same profile, so the reported missing `xhigh|max` path remains actionable.

## What This Changes

The issue asks OpenClaw to expose `/think xhigh` and `/think max` for DeepSeek V4 models and map both to DeepSeek `reasoning_effort: "max"`.

## Best Possible Solution

Add a DeepSeek-owned lightweight `resolveThinkingProfile` export that reuses the V4 model-id/profile logic, then cover lazy policy resolution and session thinking levels with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve DeepSeek V4 thinking through the lazy provider-policy surface, where `provider-policy-api.ts` lacks `resolveThinkingProfile`, so `xhigh` and `max` are absent even though docs and the full runtime define them.

## Solution Assessment

Yes. The narrowest maintainable fix is provider-owned: expose the same DeepSeek V4 thinking profile from the lightweight policy artifact instead of hardcoding DeepSeek behavior in core session or thinking logic.
