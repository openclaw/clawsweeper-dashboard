## Summary

Keep open. Current main has adjacent safety pieces, including tool allow/deny profiles, `update_plan`, strict-agentic plan-stall handling, ACP runtime-mode pass-through, and generic plugin host hooks, but it still lacks first-class Ask, Plan, and Modify modes with OpenClaw-enforced read-only behavior, action preview, and an explicit transition into mutation.

## What This Changes

Request first-class Ask, Plan, and Modify workflow modes that separate read-only discussion, action preview, and mutation.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces Ask at the tool-policy boundary, records Plan previews without mutation, and enables Modify only through an explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives a high-confidence verification path that first-class Ask, Plan, and Modify modes are not implemented.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but maintainers need to choose the core/runtime/client/plugin contract before a narrow implementation path is safe.
