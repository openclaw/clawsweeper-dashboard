## Summary

Keep open. Current main and v2026.6.10 still have adjacent sandbox, approval, read-only profile, side-question, dry-run, and structured-plan primitives, but not first-class Ask, Plan, and Modify workflow modes with enforced read-only questioning, approvable previews, and an explicit transition to mutation.

## What This Changes

The issue requests first-class OpenClaw Ask, Plan, and Modify modes so users can separate read-only exploration, plan review, and mutating execution.

## Best Possible Solution

Define a core authority-state contract for Ask, Plan, and Modify, reusing sandbox/tool-policy/update_plan/Codex primitives where they fit and coordinating with the global dry-run issue.

## Reproduction Assessment

Yes, source-level: current main exposes adjacent read-only, sandbox, approval, side-question, dry-run, and planning pieces but no first-class Ask/Plan/Modify workflow mode. This is a feature-gap reproduction rather than a failing bug reproduction.

Unclear. The direction is aligned, but the best implementation needs maintainer product and security design because the authority model crosses runtime enforcement, CLI/Gateway/UI/API shape, plugin hooks, and existing dry-run work.
