## Summary

Keep open. Current main has related building blocks, but it still does not provide first-class Ask, Plan, and Modify workflow modes with OpenClaw-enforced read-only behavior, plan approval, and an explicit transition to mutation.

## What This Changes

The issue requests three user-facing workflow modes: Ask for read-only exploration, Plan for explicit preview and approval, and Modify for executing approved changes.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at the tool-policy/runtime boundary, records Plan previews and approvals without mutation, and enables Modify only through an explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, and targeted searches give high-confidence verification that first-class Ask, Plan, and Modify modes are not implemented on current main.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but the maintainable fix needs a maintainer-approved runtime, client, ACP, plugin, and security contract rather than a narrow code patch.
