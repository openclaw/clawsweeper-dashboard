## Summary

Keep open. Current main has related safety building blocks, but it still does not provide first-class Ask, Plan, and Modify modes with OpenClaw-enforced read-only behavior, plan approval, and an explicit transition to mutation.

## What This Changes

The issue requests first-class Ask, Plan, and Modify workflow modes that separate read-only discussion, explicit planning, and approved mutation.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at the tool-policy/runtime boundary, records Plan previews and approvals without mutation, and enables Modify only through explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live GitHub context, and targeted searches give a high-confidence verification path that first-class Ask, Plan, and Modify modes are not implemented on current main.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but the maintainable fix needs a maintainer-approved runtime/client/plugin contract rather than a narrow code patch.
