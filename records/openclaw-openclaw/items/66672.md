## Summary

Keep open. Current main has overlapping safety primitives, including tool profiles, sandboxing, exec approvals, and `update_plan`, and the prior ClawSweeper comment reached the same keep-open conclusion. It still does not provide first-class Ask, Plan, and Modify workflow modes with enforced read-only operation, preview/validation, and explicit transition into execution.

## What This Changes

Requests first-class Ask, Plan, and Modify workflow modes that separate read-only discussion, action preview/approval, and mutating execution.

## Best Possible Solution

Define a core workflow-mode contract: Ask should enforce read-only/no-mutation at the tool-policy boundary, Plan should capture proposed file/action changes and block mutation until explicit approval, and Modify should enable existing write/exec-capable paths under current sandbox, approval, and tool-policy rules. The work should include UI/session controls plus runtime tests proving enforcement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source and docs inspection provides a high-confidence verification path that the requested first-class modes are not implemented.

## Solution Assessment

Unclear. The requested direction is aligned with the safety goal, but the maintainable solution needs core product/runtime design that reuses existing policy and approval seams instead of adding a parallel permission system.
