## Summary

Keep open. Current main has related building blocks such as `update_plan`, strict-agentic planning-only retry handling, ACP runtime-mode forwarding, and generic plugin host hooks, but it still does not provide first-class Ask, Plan, and Modify modes with OpenClaw-enforced read-only behavior, plan approval, and an explicit transition to mutation. Related open work at https://github.com/openclaw/openclaw/issues/71736, https://github.com/openclaw/openclaw/issues/80081, and https://github.com/openclaw/openclaw/issues/41418 covers important slices but does not fully replace this broader workflow-mode request.

## What This Changes

The issue requests first-class Ask, Plan, and Modify workflow modes that separate read-only discussion, explicit planning, and mutation.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at the tool-policy/runtime boundary, records Plan previews and approvals without mutation, and enables Modify only through an explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live GitHub context, and targeted searches give a high-confidence verification path that first-class Ask, Plan, and Modify modes are not implemented on current main.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but the maintainable fix needs a maintainer-approved runtime, client, ACP, and plugin contract rather than a narrow code patch.
