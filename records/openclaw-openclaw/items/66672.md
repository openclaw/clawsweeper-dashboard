## Summary

Keep open. Current main and the latest release have adjacent read-only, sandbox, approval, and structured-plan pieces, but they do not implement first-class Ask, Plan, and Modify workflow modes with enforced read-only behavior, approvable previews, and an explicit transition to mutation.

## What This Changes

The issue asks OpenClaw to add explicit Ask, Plan, and Modify modes so users can separate read-only exploration, planned preview, and mutation.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at runtime/tool-policy boundaries, records approvable Plan previews, and enables Modify only through an explicit user transition across supported runtimes and clients.

## Reproduction Assessment

Yes for source-level verification of the feature gap: current main and v2026.6.8 have adjacent controls but no first-class Ask/Plan/Modify workflow contract. This is not a failing bug reproduction.

## Solution Assessment

Unclear. The requested separation matches OpenClaw's safety direction, but the best implementation needs a product and security contract across runtime policy, clients, ACP/plugin host hooks, and approval semantics rather than a narrow patch.
