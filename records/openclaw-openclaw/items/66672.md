## Summary

Keep open. Current main has useful safety building blocks, including tool allow/deny profiles, sandbox workspace access, exec approvals, `update_plan`, and strict-agentic no-stall handling, but it still lacks first-class Ask, Plan, and Modify workflow modes with enforced read-only operation, action preview, and explicit transition into mutation.

## What This Changes

Request first-class Ask, Plan, and Modify workflow modes that separate read-only conversation, action planning, and mutation.

## Best Possible Solution

Define a core workflow-mode contract that enforces Ask at the tool-policy boundary, records Plan previews without mutation, and enables Modify only through an explicit user transition while preserving sandbox and approval controls.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main provides a high-confidence verification path that the requested first-class modes are not implemented.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but maintainers need to choose the runtime/session/client contract before a narrow implementation path is safe.
