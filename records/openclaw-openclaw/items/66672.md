## Summary

Keep open. Current main has read-only tool/sandbox profiles, exec approvals, the `update_plan` tracking tool, and strict-agentic plan-only retry handling, but it still does not provide first-class Ask, Plan, and Modify workflow modes with enforced read-only operation, action preview, and explicit transition into mutation.

## What This Changes

The issue asks OpenClaw to add first-class Ask, Plan, and Modify modes that separate read-only conversation, action preview, and mutation.

## Best Possible Solution

Define a core workflow-mode contract that enforces Ask at the tool-policy boundary, records Plan previews without mutation, and enables Modify only through explicit user transition while preserving current sandbox and approval rules.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main provides a high-confidence verification path that the requested first-class modes are not implemented.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but maintainers need to choose the runtime/session/client contract before a narrow code path can be safely queued.
