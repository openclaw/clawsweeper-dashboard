## Summary

Keep open: current main and v2026.6.10 still have adjacent planning, tool-policy, approval, sandbox, and plugin host-hook primitives, but not enforceable Ask/Plan/Modify workflow modes with a clear transition from read-only inquiry to approved mutation.

## What This Changes

The issue requests first-class Ask, Plan, and Modify modes that separate read-only interaction, previewable planning, and actual execution.

## Best Possible Solution

Define a first-class workflow-mode authority contract that reuses existing planning, tool-policy, approval, sandbox, and plugin-host primitives where they fit, then implement the chosen CLI/Gateway/UI/plugin surfaces together.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.10 shows adjacent primitives but no first-class Ask/Plan/Modify mode contract that gates tool execution and transitions from preview to mutation.
