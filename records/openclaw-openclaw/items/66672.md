## Summary

Keep open. Current main has tool profiles, sandboxing, exec approvals, thinking/fast directives, ACP controls, and strict-agentic plan-only retry behavior, but it still lacks first-class Ask, Plan, and Modify workflow modes with enforced read-only operation, action preview, and explicit transition into mutation. The prior ClawSweeper comment reached the same keep-open result and the later timeline only mentions likely maintainers; it does not show a merged implementation or maintainer close signal.

## What This Changes

Request first-class Ask, Plan, and Modify modes that separate read-only exploration, explicit action preview, and actual execution.

## Best Possible Solution

Define a core workflow-mode contract: Ask enforces read-only/no-mutation at the tool-policy boundary, Plan captures proposed actions and affected files while blocking mutation until approval, and Modify enables current write/exec-capable paths under existing sandbox and approval rules. The implementation should include Control UI/chat/ACP affordances plus runtime tests proving mode enforcement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives a high-confidence verification path that the requested first-class modes are not implemented.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but the maintainable fix needs product/runtime design across sessions, tool policy, approvals, sandboxing, and clients before code should be queued.
