## Summary

Keep open. Current main and v2026.6.8 have adjacent read-only, sandbox, approval, and structured-plan primitives, but they do not provide first-class Ask, Plan, and Modify workflow modes with enforced read-only questioning, approvable previews, and an explicit transition to mutation.

## What This Changes

Request first-class Ask, Plan, and Modify workflow modes that separate read-only questioning, approvable action previews, and mutation.

## Best Possible Solution

Design a core authority-state contract for read-only Ask, approvable Plan previews, and explicit Modify transitions, reusing existing sandbox/tool-policy/dry-run primitives where they fit.

## Reproduction Assessment

Yes for source-level verification: current main exposes adjacent read-only, sandbox, approval, and planning pieces but no first-class Ask/Plan/Modify workflow mode. This is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear. The requested direction is aligned, but the best implementation needs maintainer product and security design because the authority model crosses runtime enforcement, CLI/UI/API shape, plugin hooks, and existing dry-run work.
