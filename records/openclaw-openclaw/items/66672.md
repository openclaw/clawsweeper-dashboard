## Summary

Keep open. Current main has related safety pieces, including tool policy, exec approvals, ACP runtime-mode pass-through, update_plan, and generic workflow plugin host hooks, but it still does not provide first-class Ask, Plan, and Modify modes with OpenClaw-enforced read-only behavior, plan approval, and explicit mutation transition.

## What This Changes

Add first-class Ask, Plan, and Modify workflow modes that separate read-only interaction, action preview/approval, and mutation execution.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at the tool-policy boundary, records Plan previews/approvals without mutation, and enables Modify only through explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, GitHub discussion, and targeted current-main searches give a high-confidence verification path that first-class Ask, Plan, and Modify modes are not implemented.

## Solution Assessment

Unclear. The requested separation matches the safety goal, but the maintainable fix needs a maintainer-approved runtime/client/plugin contract rather than a narrow code patch.
