## Summary

Keep open. Current main and the latest release have adjacent pieces, but not first-class Ask, Plan, and Modify modes with OpenClaw-enforced read-only behavior, approvable plan previews, and an explicit transition to mutation.

## What This Changes

Request to add safe workflow modes that separate read-only questions, plan preview/approval, and mutating execution.

## Best Possible Solution

Define a maintainer-approved workflow-mode contract that enforces read-only Ask at the runtime/tool-policy boundary, records Plan previews and approval state, and enables Modify only through an explicit user transition across supported clients and runtimes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release, and related-issue checks give high-confidence verification that first-class Ask, Plan, and Modify modes are not implemented today.

## Solution Assessment

Unclear. The requested separation matches OpenClaw's safety direction, but the maintainable solution needs a product and security contract across runtime policy, clients, ACP/plugin host hooks, and approval semantics rather than a narrow patch.
