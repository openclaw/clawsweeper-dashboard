## What This Changes

The PR guards the Gateway `plugins.uiDescriptors` projection so unreadable plugin Control UI descriptor rows are skipped while healthy rows remain, and adds a contract regression.

## Best Possible Solution

Land one focused descriptor-projection guard after maintainers pick the canonical PR, undraft it, and complete the required CI or changed-lane proof.

## Reproduction Assessment

Yes. On current main, `plugins.uiDescriptors` maps `entry.descriptor` for every registry row without per-row isolation; a test registry with a throwing descriptor getter would abort before returning healthy rows.

## Solution Assessment

Yes for the code shape: the Gateway projection boundary is the narrowest maintainable place to isolate a corrupted descriptor row because the registry already validates normal registrations. The remaining solution choice is which same-author PR should be the canonical landing branch.
