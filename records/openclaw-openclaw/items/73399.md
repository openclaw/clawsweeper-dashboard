## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The new DM lookup can return `members[0]` whenever Feishu returns exactly one member but that member_id does not match the inbound sender. Since this value becomes `SenderName` and prefixes the agent-visible message body, a partial page or permission-filtered response can attribute a user's message to the wrong person; only accept an exact member_id match or page until the sender is found before falling back to contact lookup.
  - confidence: 0.91
- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The added `directMemberNameCache` stores account/chat/sender-derived keys with only an expiry timestamp and no max size or expired-entry pruning, while the sender cache key is widened to account scope. In a long-running Feishu gateway, distinct chats or senders can leave expired entries resident indefinitely, so this needs a bounded TTL/LRU helper or explicit pruning before merge.
  - confidence: 0.88
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: By accepting raw API response objects as permission errors, this patch makes the new chatMembers path feed `msg/message` text into URL extraction, and `buildFeishuAgentBody` later places that `grantUrl` in a system message for the agent to relay. Validate the URL host/path against Feishu/Lark grant domains or omit it rather than trusting arbitrary remote error text.
  - confidence: 0.86
