Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The new DM lookup falls back to `members[0]` whenever the chatMembers response has one item, even if that item’s `member_id` does not equal the sender id. If Feishu returns a partial or mismatched member list, the incoming message can be labeled as the wrong person; only accept an exact sender match and otherwise fall back to contact/no name.
  - confidence: 0.88
- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds another process-wide Map and widens the existing sender cache keys to account/chat/sender-derived values, but there is no max size and expired entries are not pruned unless the same key is touched. A long-running Feishu gateway can accumulate unbounded entries from distinct users/chats, so this needs the same bounded eviction shape used by the group-name cache.
  - confidence: 0.86
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: This broadens permission-error extraction to raw response objects and still accepts the first `https://.../app/...` URL from remote error text before it is surfaced in an agent-visible system message. Allowlist Feishu/Lark hosts or omit the URL so an upstream error body cannot present an arbitrary grant link as official guidance.
  - confidence: 0.78
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:15`
  - body: OpenClaw release generation owns CHANGELOG.md, and normal PRs should keep release-note context in the PR body or squash message. Please drop this direct changelog line before merge.
