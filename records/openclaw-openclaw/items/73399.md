Overall correctness: patch is incorrect

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The new DM resolver accepts `members[0]` whenever the response has one item, even if that member's id does not match the sender id. Since the local chatMembers wrapper is page-shaped and returns `has_more`/`page_token`, an incomplete or mismatched response can label an incoming message as the wrong person; only accept an exact sender match and otherwise fall back.
  - confidence: 0.88
- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: This adds another process-wide Map and widens sender cache keys to account/chat/sender-derived values without a max size or general pruning path. Long-running Feishu gateways can accumulate entries from distinct users and chats, so mirror the bounded group-name cache pattern or another capped TTL cache.
  - confidence: 0.9
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: The widened permission-error path accepts any raw object with the Feishu permission code and extracts the first `https://.../app/...` URL, which is later shown in agent-visible permission guidance. Only surface allowlisted Feishu/Lark hosts and format the value safely before putting it in the agent body.
  - confidence: 0.82
