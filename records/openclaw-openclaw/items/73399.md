Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The fallback accepts the only returned chat member even when its `member_id` does not match the incoming sender. That can attribute a DM turn to the wrong person in transcript and agent context; fall back to contact lookup or no name unless the member id matches.
  - confidence: 0.9
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache is process-wide and keyed by account/chat/sender-derived values, while the widened sender cache also grows by account. Long-running Feishu gateways need a hard cap and stale pruning before event-derived keys can accumulate without bound.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: The parser accepts the first `https://.../app/...` URL from remote error text and `buildFeishuAgentBody` presents it as a permission grant URL. Validate Feishu/Lark hosts before surfacing the link, or omit the URL when it is not trusted.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:15`
  - body: Normal PRs should not edit `CHANGELOG.md` directly because release generation owns that file. Keep this release-note context in the PR body or squash message instead.
  - confidence: 0.87
