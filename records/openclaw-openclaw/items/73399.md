## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: In `resolveFeishuDirectMemberName`, a p2p/private lookup accepts the sole returned chat member even when its `member_id` does not equal the sender id. If Feishu returns a partial page or permission-scoped one-member response, the agent transcript can attribute the message to the wrong person; only an exact or paginated match should win before contact fallback.
  - confidence: 0.88
- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds `directMemberNameCache` and widens `senderNameCache` keys to account/chat/sender ids, but neither Map has max-size eviction and expired entries are only checked for the requested key. A long-running gateway receiving many distinct Feishu users/chats can grow process memory without bound; use a capped TTL cache or bounded prune path.
  - confidence: 0.86
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: `extractPermissionError` accepts any `https://.../app/...` URL from remote error text and `buildFeishuAgentBody` later injects that URL into an agent-visible System note. The new direct-member path forwards permission errors, so malicious or compromised upstream error text can present an attacker URL; allowlist Feishu/Lark hosts or omit the link.
  - confidence: 0.82
