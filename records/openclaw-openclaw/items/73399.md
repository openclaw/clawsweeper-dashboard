Full review comments:

- **[P1] Require the member lookup to match the sender:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When the member list has exactly one entry, this fallback accepts that entry even if its `member_id` is not the inbound `senderId`. That value becomes `SenderName`/agent transcript attribution, so an incomplete, paged, or permission-filtered Feishu response can label the message as the wrong person; only use an exact `member_id` match or page until the sender is found, then fall back.
  - confidence: 0.9
- **[P2] Bound the sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache is keyed by account, chat, and sender but has no max size or prune path, and the branch's sender cache no longer removes expired entries on read. Since these keys come from inbound Feishu events, a long-running process can accumulate stale entries across many DMs/chats; use the bounded group-name cache pattern or an equivalent capped TTL helper.
  - confidence: 0.86
- **[P2] Validate grant URLs before exposing them to the agent:** `extensions/feishu/src/bot-sender-name.ts:52-60`
  - body: The PR broadens permission-error extraction to raw response objects, but extracted URLs are still any `https://.../app/...` host and are later presented as permission grant URLs in a system message. Allowlist official Feishu/Lark hosts or omit the URL so an unexpected error body cannot steer the user toward a non-Feishu link.
  - confidence: 0.78
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:15`
  - body: Repository policy says `CHANGELOG.md` is release-generated and normal PRs should carry release-note context in the PR body or squash message. Keeping this manual entry makes the already-conflicting branch more likely to drift during release generation, so drop the file change from this PR.
