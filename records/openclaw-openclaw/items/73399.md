- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:142-143`
  - body: The fallback accepts `members[0]` whenever the API returns one item, even if that item's `member_id` does not match the event sender. A partial page, namespace mismatch, or bot/member-only response would label the sender as the wrong human in agent context and persisted session metadata; only accept an exact match or paginate until the sender is found before using a member name.
  - confidence: 0.9
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds a direct-member cache and widens sender-cache keys to account/chat/sender-derived values, but expired entries for old keys are never pruned and there is no maximum size. A long-running Feishu process can retain unbounded entries from many chats or senders, so this needs a shared bounded TTL eviction path and focused tests before merge.
  - confidence: 0.88

  - body: `extractPermissionError` now also trusts raw response objects and still accepts any `https://.../app/...` URL before `buildFeishuAgentBody` places it in an agent-visible system message. Restrict surfaced grant links to trusted Feishu/Lark hosts, or omit the URL when it cannot be validated.
  - confidence: 0.82
