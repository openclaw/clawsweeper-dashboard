  - body: `resolveFeishuDirectMemberName` accepts the only returned `chatMembers` item when its `member_id` does not match the incoming sender. A partial, permission-scoped, or otherwise unexpected response can attribute a DM to the wrong person; only accept an exact match or page until the sender is found before falling back to contact lookup.
  - confidence: 0.92
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache and widened sender-name cache write account/chat/sender-derived keys, but the PR has no max size and does not delete expired entries on stale reads. In a long-running Feishu process, many unique senders or DMs can grow these maps without bound; use a bounded TTL helper and cover stale pruning for both caches.
  - confidence: 0.9
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: The widened parser now accepts raw error objects and extracts any `https://.../app/...` URL, which is later presented as a Feishu permission grant URL in agent-visible system guidance. Restrict grant URLs to trusted Feishu/Lark hosts or omit the URL when it cannot be validated.
  - confidence: 0.88
- **[P3] Remove the direct changelog edit:** `CHANGELOG.md:15`
  - body: `CHANGELOG.md` is release-owned in this repo, but this PR adds a normal unreleased entry directly. Move the release-note context into the PR body or eventual squash/commit message and leave changelog generation to the release flow.
  - confidence: 0.95
