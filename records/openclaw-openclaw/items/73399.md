  - body: `resolveFeishuDirectMemberName` uses the only returned member even when its `member_id` does not match `senderId`. Because that name becomes the speaker in `BodyForAgent`, an incomplete, paged, or filtered chatMembers response can attribute a DM to the wrong human; fall back to contact lookup or page until an exact sender match is found.
  - confidence: 0.87

  - body: The new `directMemberNameCache` is process-wide, keyed by inbound account/chat/sender identifiers, and never prunes expired or excess entries.   - confidence: 0.9

  - body: The PR broadens permission-error parsing to raw SDK response objects, but the extracted URL is still later embedded in a System message without host validation. Validate `open.feishu.cn`/`open.larksuite.com` grant URLs or omit the URL so an untrusted error payload cannot inject a misleading admin link into the agent prompt.
