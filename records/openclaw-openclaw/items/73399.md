Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When no `member_id` matches `senderId`, the new DM path falls back to the only returned member. The SDK response is paginated and exposes `page_token`/`has_more`, so a single returned member is not proof it is the sender; only accept an exact match or page until one is found before falling back to contact lookup.
  - confidence: 0.91
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds `directMemberNameCache` and widens sender-name keys by account/chat/sender, but neither map has max-size eviction and expired entries are only removed for the current key. Long-running Feishu gateways can retain unbounded event-derived entries; reuse a bounded TTL eviction pattern like `groupNameCache`.
  - confidence: 0.88

  - body: The widened extractor now treats plain response objects as permission errors and parses any `https://.../app/...` URL, which is later inserted into the agent System message. Allowlist Feishu/Lark grant hosts and sanitize or omit untrusted URLs before surfacing permission guidance.
  - confidence: 0.84
