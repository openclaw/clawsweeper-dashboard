## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: This fallback accepts the only returned member even when its `member_id` does not match the inbound sender. The Feishu SDK exposes paginated `chatMembers.get` results, so a partial or namespace-mismatched page can put the wrong human name into agent context and session metadata; only exact matches or a paginated search should beat the contact fallback.
  - confidence: 0.91
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds `directMemberNameCache` and widens the existing sender cache to account-derived keys, but neither cache has a max size and PR-head cache reads no longer delete stale entries. A long-running Feishu process can accumulate account/chat/sender keys indefinitely, so both caches need bounded eviction and stale pruning with focused tests.
  - confidence: 0.9
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: `extractPermissionError` now accepts raw API response objects and extracts the first `https://.../app/...` URL from remote error text, which is later inserted into agent-visible permission guidance. Before merge, only trusted Feishu/Lark hosts should be surfaced, with unsafe or malformed URLs omitted.
  - confidence: 0.86
