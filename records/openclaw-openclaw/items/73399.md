## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: `chatMembers.get` is a paginated list API, and this fallback accepts the only returned member even when its `member_id` does not match the sender. In a DM/private sender-name path, that can label the wrong human as the speaker; return a name only for an exact `member_id` match, or page until the sender is found before falling back.
  - confidence: 0.9
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache and widened sender cache are keyed by event-derived account/chat/sender identifiers, but entries are never globally capped or pruned when old unique keys are not read again. A long-running Feishu gateway can accumulate unbounded cache entries, so keep current expiry cleanup and add a hard cap or bounded prune path for both caches.
  - confidence: 0.86
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: The extractor accepts any `https://.../app/...` URL from remote error text or a raw error object, and that value is later embedded in agent-visible system guidance. Allowlist the expected Feishu/Lark hosts and sanitize or omit invalid URLs before surfacing them to the agent/user.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:13`
  - body: Repository policy makes `CHANGELOG.md` release-owned, so normal PRs should carry release-note context in the PR body or squash message instead of editing the changelog directly. Drop this line and keep the behavior/credit in the PR body.
  - confidence: 0.92
