- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The DM path accepts the sole returned chat member even when its `member_id` does not match `senderId`. If `chatMembers` returns a partial page or a different single member, OpenClaw will label the incoming message with the wrong human name and skip the contact fallback; require an exact ID match or page until the sender is found before returning a name.
  - confidence: 0.92
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: This adds `directMemberNameCache` and moves `senderNameCache` to account/chat/sender-derived keys, but entries are never capped and expired entries are not deleted on stale reads. In a long-running Feishu process, distinct DM chats and senders can grow these Maps indefinitely; keep current-main expiry deletion and add a hard bound/eviction for both caches.
  - confidence: 0.9
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: By accepting raw response objects and extracting the first `https://.../app/...` URL, this path can carry a non-Feishu URL from an API/error payload into `buildFeishuAgentBody`'s System guidance. Allowlist Feishu/Lark hosts or drop the URL before surfacing it to the agent/user.
  - confidence: 0.86
