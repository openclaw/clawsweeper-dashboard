## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When `chatMembers.get` returns one member whose `member_id` does not match `senderId`, this fallback still uses that member's display name. A partial, filtered, or unexpected p2p/private response can therefore attribute the incoming message to the wrong person; only accept an exact `member_id` match before falling back to the contact path.
  - confidence: 0.91
- **[P1] Bound the direct-member sender-name cache:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new `directMemberNameCache` is process-wide and keyed by account/chat/sender data, but it has no max size or active expired-entry pruning. High-cardinality Feishu traffic can grow gateway memory over time, so use a bounded/pruned TTL cache before adding this hot-path lookup.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: The PR now lets direct-member permission failures reach the agent, but the grant URL is parsed from remote error text and later inserted into a system message. Validate the URL host/path or omit it so unexpected Feishu/proxy error text cannot present an attacker-controlled permission link as trusted guidance.
  - confidence: 0.84
