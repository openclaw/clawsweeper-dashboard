## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require exact direct-member sender matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: `chatMembers.get` is a paged member-list API, but this fallback uses the only returned member even when its `member_id` does not match the message sender. If Feishu returns a partial or permission-scoped one-item page, the agent and session metadata can attribute the message to the wrong person; only use an exact sender match or page until the sender is found before falling back.
  - confidence: 0.86
- **[P1] Bound and prune Feishu name caches:** `extensions/feishu/src/bot-sender-name.ts:149`
  - body: Current main deletes expired sender-name cache entries before retrying, but this PR adds `directMemberNameCache` and rewrites the sender cache path so expired entries stay in process forever and neither map has a size cap. A long-running gateway that sees many account/chat/sender combinations can grow memory without bound, so reuse a bounded TTL helper and delete stale entries on access.
  - confidence: 0.91
- **[P1] Validate permission URLs before agent hints:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: This change lets raw Feishu response/error objects feed `extractPermissionError`, and the resulting regex-captured URL is later inserted into a system hint for the agent. Before surfacing that value, restrict it to official Feishu/Lark hosts or omit it so a crafted upstream error cannot turn into an agent-visible phishing or prompt-formatting hint.
  - confidence: 0.82
