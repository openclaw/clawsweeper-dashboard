## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The fallback accepts `members[0]` whenever the member list has one item even if its `member_id` does not match the inbound sender. If Feishu returns a partial or unexpected list, the agent transcript can attribute the message to the wrong person; only an exact match or a paginated lookup that finds the sender should be trusted.
  - confidence: 0.89
- **[P1] Bound the sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache and widened sender cache are process-wide maps keyed by account/chat/sender data, but expired entries are not deleted on stale reads and there is no maximum size. A busy or hostile Feishu workspace can create unbounded cache growth in a long-running process.
  - confidence: 0.87
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: The parser now accepts raw Feishu response objects and still extracts the first `https://.../app/...` string as a grant URL. Because that value is later inserted into agent-visible permission guidance, it should be allowlisted to Feishu/Lark hosts or suppressed instead of trusting arbitrary error text.
  - confidence: 0.8
- **[P3] Remove the normal PR changelog edit:** `CHANGELOG.md:12`
  - body: `CHANGELOG.md` is release-owned in this repo, and normal PRs should carry release-note context in the PR body or commit message instead. Please drop this entry from the branch.
  - confidence: 0.95
