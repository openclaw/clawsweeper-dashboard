  - body: When `chatMembers.get` returns one member whose `member_id` does not match `senderId`, this fallback still uses that member's display name.   - body: The new `directMemberNameCache` is process-wide and keyed by account/chat/sender data, but it has no max size or active expired-entry pruning. High-cardinality Feishu traffic can grow gateway memory over time, so use a bounded/pruned TTL cache before adding this hot-path lookup.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: The PR now lets direct-member permission failures reach the agent, but the grant URL is parsed from remote error text and later inserted into a system message. Validate the URL host/path or omit it so unexpected Feishu/proxy error text cannot present an attacker-controlled permission link as trusted guidance.
