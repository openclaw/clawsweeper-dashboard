Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The fallback to `members[0]` means a p2p/private lookup can attribute the inbound sender to a different human whenever Feishu returns one member that does not match `senderId`. Only accept an exact `member_id` match, then fall back to the contact path or no name.
  - confidence: 0.89
- **[P1] Bound the sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache and widened sender cache are process-wide maps keyed by account/chat/sender data, but there is no max size and expired entries are not deleted when reads miss the TTL. A long-running bot can grow memory with unique chats or senders, so add bounded eviction/pruning like the Feishu group-name cache.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: Permission errors now reach the agent from the DM fallback path, but `extractPermissionError` accepts any `https://.../app/...` URL from remote error text. Validate Feishu/Lark hosts before surfacing a grant URL in agent-visible system text.
  - confidence: 0.84
