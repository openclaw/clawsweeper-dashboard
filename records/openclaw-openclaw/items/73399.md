- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: `chatMembers.get` is queried for the sender, but if no returned `member_id` matches, the PR accepts the sole returned member anyway. A partial, paginated, or permission-scoped response could label the incoming DM as the wrong person; fall back to contact lookup or no name unless the member id matches the sender.
  - confidence: 0.9
- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache stores account/chat/sender-derived keys with a TTL but no maximum size, active pruning, or negative-hit bound. A long-running gateway that sees many distinct DMs can grow this map without bound, so add bounded eviction/pruning consistent with the Feishu probe cache pattern.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: The new DM fallback can pass Feishu permission-error text through `extractPermissionError`, which parses any `https://.../app/...` URL and later puts it in agent-facing system text. Validate the host against expected Feishu/Lark domains or omit the URL before surfacing it.
  - confidence: 0.82
