- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds a second process-wide Map and widens sender-name cache keys to account/chat/sender identifiers, but neither cache has a maximum size or sweep for expired entries. In a long-running Feishu gateway, distinct event-derived identifiers can grow these Maps until memory pressure or restart.
  - confidence: 0.91
- **[P2] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When chatMembers returns no member_id matching the sender, this fallback accepts the sole returned member anyway. A partial page or permission-scoped response can therefore attribute a DM to the wrong human; fall back to contact/no name unless the sender is matched or pagination finds it.
  - confidence: 0.9
- **[P1] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: This extracts the first https URL ending in /app/... from remote error text and later presents it as a permission grant URL. Restrict grant URLs to trusted Feishu/Lark hosts and escape or omit invalid values before surfacing them in agent-visible guidance.
  - confidence: 0.87
