- **[P1] Bound sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The PR adds `directMemberNameCache` and widens sender cache keys to account/chat/sender identifiers, but the maps have no max size and expired entries are not pruned on stale reads. A long-running Feishu gateway can grow memory with every distinct chat/sender, so add bounded eviction like the existing group-name cache path.
  - confidence: 0.88
- **[P2] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The new DM lookup accepts the lone returned `chatMembers` entry even when its `member_id` does not match the event sender. If Feishu returns a partial or mismatched member list, the agent transcript can attribute the message to the wrong person; fall back to contact lookup unless the member id matches.
  - confidence: 0.82
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: The new direct-member permission path parses raw API result/error text and later surfaces `grantUrl` in agent-facing system guidance, but any `https://.../app/...` host is accepted. Allowlist Feishu/Lark app hosts or omit untrusted URLs before putting them in the agent prompt.
  - confidence: 0.86
