Full review comments:

- **[P2] Require an exact member-id match for DM names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When no returned member_id matches the inbound sender, this still uses the only returned member's name. If the list is partial or the id type differs, the agent transcript can attribute a DM to the wrong person; only accept an exact match and fall back otherwise.
  - confidence: 0.89
- **[P1] Bound the direct-member name cache:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: directMemberNameCache is process-wide and keyed by account, chat, and sender, but it has no max size or global cleanup. A long-running gateway that sees many distinct DMs can retain entries indefinitely, so this should use bounded TTL eviction like the group-name and chat-type caches.
  - confidence: 0.86
- **[P1] Validate grant URLs before accepting raw permission responses:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: By treating raw response objects as permission errors and accepting any https URL under /app/, this can put an attacker-controlled link into the agent-visible permission guidance. Only surface known Feishu/Lark grant hosts, or omit the URL when it cannot be validated.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:13`
  - body: CHANGELOG.md is release-generated in this repository, so normal PRs should carry release-note context in the PR body or squash message instead of editing the changelog directly.
