Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: `chatMembers.get` is used to identify the DM sender, but this fallback accepts the only returned member even when its `member_id` does not match the incoming sender id. If Feishu returns a partial or permission-scoped member list, OpenClaw can attribute a message to the wrong human name in the agent transcript; fall through to contact lookup or the id fallback unless the member id matches.
  - confidence: 0.91
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new direct-member cache and the widened sender-name cache are process-wide maps keyed by account/chat/sender values with TTL timestamps but no hard size cap or broad pruning. A long-running Feishu process that sees many unique senders or chats can retain expired entries indefinitely; use the bounded eviction pattern already present for Feishu group-name caching or an equivalent cap.
  - confidence: 0.9
- **[P2] Validate Feishu grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: The permission parser now also accepts raw error objects and extracts the first `https://.../app/...` URL from remote error text, then the bot can send that URL as permission guidance. Validate the host against official Feishu/Lark grant domains, or omit the link, so an upstream/proxy-shaped error cannot present an attacker-controlled app URL to the user or agent.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:15`
  - body: This normal PR adds a `CHANGELOG.md` entry directly, but repository policy makes the changelog release-generated. Keep the release-note context in the PR body or eventual squash message instead so release generation remains the single source of truth.
