- **[P2] Require an exact member-id match for DM names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When no returned member_id matches the inbound sender, this still uses the only returned member's name. If the list is partial or the id type differs, the agent transcript can attribute a DM to the wrong person; only accept an exact match and fall back otherwise.
  - confidence: 0.89
- **[P1] Bound and prune sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new directMemberNameCache is process-wide and keyed by account, chat, and sender, and the changed sender-name cache path no longer mirrors current-main expiry deletion. A long-running gateway that sees many distinct DMs can retain entries indefinitely, so this should use bounded TTL eviction like the group-name cache.
  - confidence: 0.86
- **[P1] Validate grant URLs before surfacing permission errors:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: This parser now accepts raw API response objects and message fields, then passes any matching https URL through to agent-visible System guidance. Validate the URL host/path against official Feishu/Lark domains before surfacing it, or omit the link when it is not trusted.
  - confidence: 0.84
