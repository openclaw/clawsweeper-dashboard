- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The direct-member lookup falls back to `members[0]` whenever the API returns one item, even if that member ID does not match the inbound sender. If Feishu returns a partial page, a different ID type, or a permission-filtered result, OpenClaw can attribute a message to the wrong human; only accept an exact `member_id` match or continue to the contact fallback.
  - confidence: 0.86
- **[P1] Bound the sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: This PR adds `directMemberNameCache` and widens sender cache keys to account/chat/sender-derived values, but the maps have no max size and do not prune stale entries. A long-running Feishu bot can retain unbounded entries for unique chats or senders, so preserve current-main expiry deletion and add a cap or prune path for both caches.
  - confidence: 0.88
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:61-65`
  - body: `extractPermissionError` now accepts raw API result objects and extracts the first `https://.../app/...` URL, which the bot later inserts into an agent-visible System message. Validate allowed Feishu/Lark hosts or omit the URL so a crafted upstream error cannot present an attacker-controlled grant link.
  - confidence: 0.82
