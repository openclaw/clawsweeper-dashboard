Overall correctness: patch is incorrect

Full review comments:

- **[P1] Require exact DM member matches:** `extensions/feishu/src/bot-sender-name.ts:142-143`
  - body: In the new DM path, if Feishu returns one `chatMembers` item whose `member_id` does not equal `senderId`, this code still uses that item’s name. That can write the wrong human name into `SenderName` and `BodyForAgent`; require an exact match or page until the sender is found, then fall back to contact lookup.
  - confidence: 0.88
- **[P1] Bound the sender-name caches:** `extensions/feishu/src/bot-sender-name.ts:149`
  - body: This new cache records an expiry timestamp but never deletes stale entries or caps total entries; the same branch changes sender-name cache keys without preserving current-main stale deletion. Because keys include Feishu sender/chat IDs, a long-running bot can accumulate unbounded entries; prune or cap both caches.
  - confidence: 0.84
- **[P2] Validate permission URLs before surfacing them:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: By falling back to the raw object, any unwrapped Feishu-style response with `code: 99991672` can contribute a `https://.../app/...` URL that is later put into agent system text. Validate the grant URL host/path against official Feishu/Lark domains or avoid surfacing raw URLs before using this broader extractor.
  - confidence: 0.78
