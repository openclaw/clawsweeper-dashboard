Full review comments:

- **[P1] Require exact member matching for DM sender names:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: The fallback accepts the only returned chat member even when its `member_id` does not match `senderId`. If `chatMembers.get` returns a partial or permission-scoped result, this can put another user's display name into the transcript; accept an exact match or keep falling back.
  - confidence: 0.92
- **[P1] Bound the direct-member sender-name cache:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: `directMemberNameCache` is a process-wide map keyed by account, chat, and sender, but entries are never swept after expiry and there is no size cap. High-cardinality Feishu traffic can grow gateway memory over time; mirror the bounded/pruned cache pattern used by current group-name caching.
  - confidence: 0.9
- **[P2] Validate grant URLs before agent guidance:** `extensions/feishu/src/bot-sender-name.ts:52`
  - body: This change lets raw API response objects become permission errors, and the resulting `grantUrl` is later inserted into agent-facing system text. Validate official Feishu/Lark hosts or omit the URL so a crafted error cannot surface an arbitrary admin link.
  - confidence: 0.86
