  - body: `chatMembers.get` is queried for the sender, but if no `member_id` matches, the PR accepts the sole returned member anyway.   - body: The PR broadens permission-error parsing to raw API response objects and carries DM fallback permission errors into agent-facing text, but the parsed URL host is not validated before being presented as the permission grant URL. Only surface known Feishu/Lark grant hosts, or omit the URL.
