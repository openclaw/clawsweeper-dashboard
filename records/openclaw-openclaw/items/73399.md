## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Require an exact sender match before using the member name:** `extensions/feishu/src/bot-sender-name.ts:141-143`
  - body: When `chatMembers.get` returns exactly one item but that item's `member_id` does not match the inbound sender, this fallback still uses that name. That can attribute a DM to the wrong person; please drop the single-item fallback and let the contact lookup handle misses.
  - confidence: 0.88
- **[P2] Bound the direct-member sender-name cache:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: `directMemberNameCache` is process-wide and keys entries by account, chat, and sender, but expired entries are never pruned and there is no max size. A long-running gateway that sees many distinct direct chats can grow this map indefinitely, so this new cache needs the same kind of cap/eviction used by the group-name cache.
  - confidence: 0.82
- **[P2] Validate Feishu grant URLs before surfacing them:** `extensions/feishu/src/bot-sender-name.ts:52-65`
  - body: This broadens permission-error parsing to raw API response objects and accepts any `https://.../app/...` URL from the error text; that value is later placed into a system message for the agent/user. Please only keep official Feishu/Lark Open Platform hosts, or omit the URL when the host is unexpected.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff adds a new DM lookup/cache path and propagates permission URLs into agent-visible system text without sufficient attribution, cache-bound, or host-validation safeguards.

Concerns:

- **[medium] Potential DM sender-name misattribution:** `extensions/feishu/src/bot-sender-name.ts:141`
  - body: The new direct-member fallback can use a single non-matching chat member as the sender name, which is an integrity issue for transcripts and agent context.
  - confidence: 0.88
- **[medium] Unbounded direct-member name cache:** `extensions/feishu/src/bot-sender-name.ts:31`
  - body: The new process-wide direct-member cache stores account/chat/sender entries without max size or expired-entry pruning, allowing avoidable memory growth in long-running gateways.
  - confidence: 0.82
- **[medium] Unvalidated permission grant URL:** `extensions/feishu/src/bot-sender-name.ts:61`
  - body: A URL parsed from Feishu error text is accepted without host validation and later shown in agent-visible system guidance; unexpected hosts should be dropped or restricted to official Feishu/Lark domains.
  - confidence: 0.76
