## What This Changes

The issue asks OpenClaw to refresh heartbeat and cron `Current time` context on every run instead of reusing the timestamp from an earlier prompt.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent idempotent refresh in `appendCronStyleCurrentTimeLine` with regression coverage for helper-emitted `Current time` / `Reference UTC` blocks.
