## Summary

Keep open. Current main and the latest `v2026.5.3` release still preserve stale `Current time:` text in the shared helper, and open PR #75025 already uses closing syntax for the narrow fix.

## What This Changes

Heartbeat and cron prompt text should refresh any existing `Current time:` line on each run instead of preserving the first injected timestamp.

## Best Possible Solution

Land PR #75025 if its implementation and checks are acceptable; otherwise apply an equivalent helper-level refresh with focused regression coverage.

## Reproduction Assessment

Yes. A helper-level call with text containing an older line-bound `Current time:` value plus a newer `nowMs` will return the stale text unchanged, and heartbeat routes each run through that helper.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and reset prompt paths already pass a fresh timestamp there.
