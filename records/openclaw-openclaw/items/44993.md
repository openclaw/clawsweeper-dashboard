## What This Changes

The issue reports that heartbeat and main-session cron prompts keep an older `Current time:` line instead of refreshing it on each run.

## Best Possible Solution

Land or update https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent helper-level refresh with regression coverage for the current two-line `Current time`/`Reference UTC` format.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session reset callers already pass fresh timestamps there; the open PR or an equivalent helper refresh is the right direction.
