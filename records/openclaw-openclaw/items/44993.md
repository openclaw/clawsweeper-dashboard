## Summary

Keep open. Current main still returns a prompt unchanged when it already contains a line with `Current time:`, heartbeat still sends each run through that helper, and the latest release tag still has the same guard. The provided GitHub context shows PR #75025 is open, unmerged, and explicitly paired with this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for heartbeat and cron prompt timestamp injection to refresh the `Current time:` line on each run instead of preserving an older value.

## Best Possible Solution

Land PR #75025 if its implementation and CI are acceptable; otherwise apply the same narrow shared-helper fix so existing line-bound `Current time:` markers are refreshed or collapsed while empty-input behavior is preserved, with focused regression coverage for append, refresh, and duplicate-line cases.

## Reproduction Assessment

Yes. Pass text containing an older line-bound `Current time:` value plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat still routes run prompts through that helper.

## Solution Assessment

Yes. The narrow maintainable fix is at the shared helper boundary because heartbeat, main-session cron delivery, and session reset prompts already pass fresh timestamps into that helper; fixing only one caller would leave shared consumers inconsistent.
