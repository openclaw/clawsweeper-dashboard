## Summary

Keep open because current main and the latest release still preserve already-injected `Current time:` text unchanged, and an open closing PR now owns the narrow helper/test fix path.

## What This Changes

Heartbeat and cron prompt formatting should refresh the injected `Current time:` timestamp on each run instead of preserving a stale line from an earlier run.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent idempotent refresh in the shared helper with regression coverage for the current two-line timestamp format.

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session-reset callers already pass fresh timestamps there, and the open PR follows that direction.
