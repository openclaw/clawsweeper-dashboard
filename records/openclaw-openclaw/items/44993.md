## Summary

Keep open. Current main and the latest release still preserve stale prompt text when it already contains `Current time:`, and an open linked PR owns the narrow helper/test fix path.

## What This Changes

Heartbeat and cron prompt current-time lines should refresh on each run instead of preserving an older injected timestamp.

## Best Possible Solution

Land or update https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent helper-level refresh with regression coverage for the current two-line timestamp format.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session reset callers already pass fresh timestamps there, and the open PR follows that direction.
