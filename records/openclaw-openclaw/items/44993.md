## Summary

Current main and the latest stable release still preserve a previously generated current-time block instead of refreshing it, while the clean open closing PR remains the active fix path and still needs current-head behavior proof.

## What This Changes

Refresh the heartbeat/cron-injected current-time block on every run instead of preserving an older timestamp.

## Best Possible Solution

Land the focused linked PR after it proves the current two-line generated block is refreshed and deduplicated while user-authored text remains unchanged, then close this issue as implemented.

## Reproduction Assessment

Yes. Generate a block with `appendCronStyleCurrentTimeLine`, then call the helper again with that output and a later `nowMs`; current main returns the first block unchanged because of the `base.includes("Current time:")` guard.

## Solution Assessment

Yes at the shared-helper boundary. Heartbeat and reset callers already provide fresh timestamps, so the narrow maintainable repair is to recognize and refresh only helper-generated blocks rather than changing each caller.
