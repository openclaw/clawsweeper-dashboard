## Summary

Keep open. Current main and the latest release still preserve stale `Current time:` text in the shared helper, and an open PR with closing syntax already covers the narrow fix, so this issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue reports that heartbeat/cron prompts can carry an old `Current time:` timestamp because repeated runs do not refresh an existing timestamp line.

## Best Possible Solution

Review and land PR #75025 if its implementation and checks are acceptable; otherwise apply the same narrow helper-level refresh with focused regression coverage.

## Reproduction Assessment

Yes. Pass text containing an older line-bound `Current time:` value plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat routes each run through that helper.

## Solution Assessment

Yes. The best fix direction is the shared helper boundary because heartbeat, main-session cron delivery, and session-reset prompts already pass fresh timestamps there; fixing only a caller would leave shared consumers inconsistent.
