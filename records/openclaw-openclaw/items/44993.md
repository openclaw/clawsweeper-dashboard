## Summary

Keep open: current main and the latest release still preserve an already-injected `Current time:` line unchanged, and the open closing PR is the active fix path.

## What This Changes

Heartbeat and cron current-time injection should refresh on each recurring run instead of preserving an earlier timestamp.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent idempotent refresh in `appendCronStyleCurrentTimeLine` with regression coverage for helper-emitted `Current time` and `Reference UTC` lines.

## Reproduction Assessment

Yes, at source level. Calling `appendCronStyleCurrentTimeLine` with text that already contains an older helper-emitted `Current time:` line and a newer `nowMs` follows the current early return and preserves the stale text unchanged.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session-reset callers already pass fresh timestamps there, and the open closing PR follows that direction.
