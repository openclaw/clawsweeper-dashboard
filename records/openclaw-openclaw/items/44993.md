## Summary

Keep open. Current main and the latest release still return stale text from the shared current-time helper, and an open closing PR already owns the narrow helper/test fix path.

## What This Changes

Heartbeat and cron prompts should refresh the helper-injected `Current time` timestamp on each run instead of preserving an older line.

## Best Possible Solution

Land or update https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent helper-level refresh with regression coverage for the current two-line `Current time` / `Reference UTC` format.

## Reproduction Assessment

Yes. A helper-level call with text containing an older helper-emitted `Current time:` line plus a newer `nowMs` follows current main's early return and preserves the stale text unchanged; this is source-level proof rather than a live gateway run.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session reset callers already pass fresh timestamps there; the open PR or an equivalent helper refresh is the right direction.
