## What This Changes

The issue asks OpenClaw to refresh the heartbeat/cron `Current time` line on every run instead of preserving an old injected timestamp.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75025, or apply an equivalent idempotent refresh in the shared helper with regression coverage for the current two-line timestamp format.

Yes. The shared helper is the narrowest maintainable fix point because heartbeat and session-reset callers already pass fresh timestamps there, and the open PR follows that direction.
