## What This Changes

The PR wraps Anthropic and Gemini native PDF provider calls in strict SSRF-guarded fetches, moves Gemini API-key usage to `x-goog-api-key`, and adds guarded-fetch release cleanup.

## Best Possible Solution

Add a provider-transport-aware guarded native-PDF fetch path that blocks SSRF while preserving proxy/TLS dispatch policy, provider timeouts, Gemini header auth, base-URL normalization, audit context, regression tests, and redacted real-provider proof.

## Solution Assessment

No. The security direction is right, but direct strict guarded fetch is not the best fix because native PDF provider traffic needs to preserve the same provider transport policy and timeout behavior as other model traffic.
