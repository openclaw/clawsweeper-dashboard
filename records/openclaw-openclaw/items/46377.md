## Summary

Keep this PR open, but it is not mergeable as written: current main still has the native PDF SSRF exposure for Anthropic and Gemini, while the PR's direct strict-guarded fetch path can bypass provider transport proxy policy and the contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR wraps Anthropic and Gemini native PDF provider fetches in strict SSRF-guarded fetches, moves Gemini API key auth to a header, and adds guarded-fetch release cleanup.

## Best Possible Solution

Add a provider-transport-aware guarded fetch path for native PDF analysis that preserves SSRF blocking, provider proxy/dispatcher policy, native PDF timeouts, Gemini header auth, audit context, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main passes model.baseUrl into the native PDF helpers, and those helpers still perform raw provider fetches against derived URLs; I did not run live provider credentials.

## Solution Assessment

No. The security direction is right, but direct strict guarded fetch is not the best fix because native PDF provider traffic needs the same provider transport policy as the rest of model traffic.
