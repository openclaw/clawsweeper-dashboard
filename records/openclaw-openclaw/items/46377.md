## Summary

Keep open. The underlying security problem is still real on current main, and the PR is directionally useful, but it is stale against current provider transport behavior and is not safe to merge as-is because it drops timeout/normalization behavior and does not preserve provider egress policy wiring.

## What This Changes

The branch modifies `src/agents/tools/pdf-native-providers.ts` to wrap Anthropic and Gemini native PDF calls in the SSRF guard, move Gemini auth from the URL query string to `x-goog-api-key`, and release guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that guards the normalized Anthropic/Gemini native PDF endpoints, sends Gemini credentials in `x-goog-api-key`, preserves the 120s timeout, carries provider request transport policy where applicable, adds audit context, and adds focused tests for private-network blocking, header placement, timeout propagation, and Gemini transport normalization.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: both native PDF provider functions still build request destinations from configurable base URLs, and Gemini still embeds the API key in the URL query string.

## Solution Assessment

No. The security direction is right, but this PR is not the best current patch because it is stale against main and does not preserve timeout, Gemini transport normalization, or provider-aware egress policy behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve provider egress policy:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: Calling `withStrictGuardedFetchMode` directly gives native PDF calls a guarded direct path without the provider request dispatcher policy, TLS/proxy overrides, or private-network opt-in used by model-provider transports. Rework this through the provider-aware guarded fetch path or pass equivalent policy so native PDF requests do not diverge from configured provider egress controls.
  - confidence: 0.82
- **[P2] Preserve the native PDF timeout:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: The new guarded fetch options do not pass `timeoutMs` or a signal. Current main bounds both native PDF provider requests at 120 seconds, so rebasing this patch as-is would make hung provider calls unbounded again. Please thread the existing native PDF timeout into the guarded fetch call.
  - confidence: 0.88
- **[P2] Preserve Gemini transport normalization:** `src/agents/tools/pdf-native-providers.ts:154-157`
  - body: The PR builds the Gemini URL directly from `params.baseUrl`, but current main first runs the Google transport through `normalizeProviderTransportWithPlugin`. A rebase should guard the already-normalized transport URL so plugin-owned base URL and request-policy behavior stays intact.
  - confidence: 0.84
