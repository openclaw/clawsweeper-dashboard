## What This Changes

The PR changes `src/agents/tools/pdf-native-providers.ts` to wrap Anthropic and Gemini native PDF requests in strict SSRF-guarded fetches, send Gemini auth via `x-goog-api-key`, and release guarded-fetch resources.

## Best Possible Solution

Native PDF analysis should use a provider-aware guarded fetch helper that preserves SSRF blocking, provider proxy/dispatcher policy, native PDF timeouts, Gemini header auth, audit context, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main passes configured provider `baseUrl` into raw native PDF fetches, and the PR's direct strict-guard path can be compared against provider transport tests that preserve env-proxy mode.

## Solution Assessment

No. The security direction is right, but direct strict guarded fetch is not the best fix because native PDF provider traffic needs to preserve the same provider transport policy as other model traffic.
