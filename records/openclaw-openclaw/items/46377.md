## Summary

Keep this PR open but do not merge it as-is: the reported security problem is real, but the patch routes native PDF provider calls through direct strict guarded fetches instead of the provider transport path, which can regress proxy-only provider setups and controlled egress behavior.

## What This Changes

The PR wraps Anthropic and Gemini native PDF provider calls in strict SSRF-guarded fetches, moves Gemini API key usage to `x-goog-api-key`, and adds guarded-fetch release cleanup.

## Best Possible Solution

Introduce a provider-transport-aware guarded fetch path for native PDF analysis that preserves SSRF blocking, provider proxy/dispatcher policy, native PDF timeouts, Gemini header auth, audit context, and focused regression coverage.
