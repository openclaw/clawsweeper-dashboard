## Summary

Keep open: the underlying native-PDF SSRF gap remains on current main, but this stale security patch is not safe to merge as-is because it bypasses provider request policy and predates current timeout/Gemini transport fixes.

## What This Changes

The branch wraps Anthropic and Gemini native PDF fetches in the strict SSRF guard, moves Gemini auth from the URL query string to `x-goog-api-key`, and releases guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that guards the normalized Anthropic/Gemini native PDF endpoints, keeps Gemini credentials in `x-goog-api-key`, preserves the 120s timeout and provider egress policy, adds audit context, and covers SSRF/transport behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: Anthropic and Gemini native PDF calls still use global `fetch` against configurable provider URLs, while only Gemini's URL-secret leakage has already been fixed.

## Solution Assessment

No. The security direction is right, but this PR is not the best current patch because it must be rebased and reworked to preserve timeout, Gemini transport normalization, and provider-aware egress controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve provider egress policy:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: Calling `withStrictGuardedFetchMode` directly gives native PDF calls a strict guarded path without the provider request dispatcher policy, proxy/TLS overrides, or private-network opt-in used by model-provider transports. Rework this through the provider-aware guarded fetch path or pass equivalent policy so the security fix does not diverge from configured provider egress controls.
  - confidence: 0.84
- **[P2] Preserve the native PDF timeout:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: The guarded fetch options do not pass `timeoutMs` or a signal. Current main bounds both native PDF provider requests at 120 seconds, so rebasing this patch as-is would make hung provider calls unbounded again.
  - confidence: 0.88
- **[P2] Preserve Gemini transport normalization:** `src/agents/tools/pdf-native-providers.ts:154-157`
  - body: The PR builds the Gemini URL directly from `params.baseUrl`, but current main first runs the Google transport through `normalizeProviderTransportWithPlugin`. A rebase should guard the already-normalized transport URL so plugin-owned base URL behavior stays intact.
  - confidence: 0.85
