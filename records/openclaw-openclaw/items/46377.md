## Summary

Keep open. The underlying SSRF issue remains valid on current main, but this stale PR is not mergeable as-is because it bypasses provider-aware egress policy and would drop current timeout and Gemini transport-normalization behavior.

## What This Changes

The PR wraps native Anthropic and Gemini PDF requests in `fetchWithSsrFGuard`, moves Gemini authentication to `x-goog-api-key`, and releases guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that guards normalized Anthropic and Gemini native PDF endpoints, keeps Gemini credentials in `x-goog-api-key`, preserves the 120s timeout and provider egress policy, adds audit context, and covers SSRF/transport behavior with focused tests.

## Reproduction Assessment

Yes. Static reproduction on current main is high-confidence: both native PDF helpers still call global `fetch` against configurable provider URLs, while Gemini URL-secret leakage has already been fixed.

## Solution Assessment

No. The security direction is right, but this PR is not the best current patch because it must be rebased and reworked to preserve provider policy, timeout, and Gemini transport normalization.
