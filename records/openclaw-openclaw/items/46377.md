## Summary

This PR should stay open because the SSRF hardening remains useful on current main, but the patch bypasses existing provider proxy/dispatcher policy and still lacks after-fix real behavior proof.

## What This Changes

The PR wraps Anthropic and Gemini native PDF provider POSTs in strict SSRF-guarded fetches, moves Gemini auth from a query parameter to `x-goog-api-key`, and adds guarded-fetch release cleanup.

## Best Possible Solution

Add a provider-transport-aware guarded-fetch path for native PDF analysis that keeps SSRF blocking, Gemini header auth, audit context, timeout/proxy/dispatcher behavior, and focused regression coverage aligned with the rest of model-provider traffic.

## Reproduction Assessment

Yes, at source level. Current main still passes `model.baseUrl` into the native Anthropic/Gemini PDF helpers and those helpers perform raw `fetch` calls against derived URLs; I did not run live provider credentials.

## Solution Assessment

No. The security direction is right, but direct `withStrictGuardedFetchMode` is not the best fix because provider traffic needs to preserve the existing trusted env-proxy and dispatcher policy.

## Review Findings
