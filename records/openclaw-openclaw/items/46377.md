## Summary

Keep open: the security problem is still real, but the branch is stale against current provider transport behavior and would regress provider egress policy, timeout handling, and Gemini normalization if merged as-is.

## What This Changes

The PR wraps Anthropic and Gemini native PDF requests in strict SSRF guarded fetch, moves Gemini authentication to `x-goog-api-key`, and releases guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that guards the normalized Anthropic/Gemini native PDF endpoints, keeps Gemini credentials in `x-goog-api-key`, preserves provider egress policy and the 120s timeout, adds audit context, and covers SSRF/transport behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: Anthropic and Gemini native PDF calls still use global `fetch` against configurable provider URLs, while only Gemini's URL-secret leakage has already been fixed.

## Solution Assessment

No. The security direction is right, but this PR is not the best current patch because it must be rebased and reworked to preserve timeout, Gemini transport normalization, and provider-aware egress controls.
