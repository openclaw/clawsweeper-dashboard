## Summary

Keep this PR open, but it is not merge-ready: it addresses a real Feishu message-loss bug, while the current patch mixes in an unrelated agent thinking/session change and keys the Feishu bypass to an externally sourced message-id prefix.

## What This Changes

The branch makes Feishu synthetic card-action messages bypass `requireMention`, adds a Feishu broadcast regression test, and also changes pi-embedded-runner thinking-signature cleanup and tests.

## Best Possible Solution

Land a narrow Feishu-only fix that uses an internal card-action signal, preserves normal `requireMention` for ordinary messages, includes focused regression coverage, and has redacted live Feishu group card-click proof; move the thinking fix to its own PR.

## Solution Assessment

No as-is. The Feishu fix direction is plausible, but the patch should use an internal synthetic card-action marker and should not bundle unrelated thinking/session changes.
