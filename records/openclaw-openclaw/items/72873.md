## Summary

Keep open. Current main still has no Telegram `edited_message` runtime handler, and open PR #73104 is already the active closing implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Telegram should process debounced `edited_message` updates so streaming bot edits are not dropped after the initial partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: add an `edited_message` handler that reuses the inbound message path, coalesces rapid edits by chat and message id, avoids bot-authored echoes, and locks the behavior with focused registration, debounce, dedupe, and dispatch tests. The active path is to review and land #73104 or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes. A synthetic Telegram `edited_message` update is enough to reproduce the source-level gap because current main can request that update type but has no registered handler to pass it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and dedupe coverage is the maintainable boundary; because #73104 already uses closing syntax for this issue, the issue should remain open until that PR lands or is replaced.
