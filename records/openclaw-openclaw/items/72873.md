## Summary

Keep open. Current main still requests and keys Telegram edited-message updates, but it does not register an `edited_message` runtime handler that dispatches edited content into the inbound pipeline; the active open PR #73104 already references this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Process Telegram `edited_message` updates, especially streaming bot edits, with debounce and dedupe so the final edited content reaches the inbound Telegram pipeline.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: register an `edited_message` handler that reuses the existing inbound message path, coalesces rapid edits by chat/message id, avoids bot-authored echo loops, and locks the behavior with focused registration, debounce, dedupe, and dispatch tests. The active path is to review and land #73104 or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main asks for the update type and can key it, but no registered handler passes it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and focused tests is the maintainable solution; because #73104 already uses closing syntax for this issue, the issue should stay open until that PR lands or is replaced.
