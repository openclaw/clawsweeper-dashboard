## Summary

Keep open. Current main still lacks an inbound Telegram `edited_message` handler, and open PR #73104 already references this issue with closing syntax, so the issue should remain open until that fix lands or is replaced.

## What This Changes

Process Telegram `edited_message` updates so streaming or edited Telegram messages reach the inbound pipeline after debounce/deduplication instead of leaving OpenClaw with only the initial partial text.

## Best Possible Solution

Land or replace #73104 with a Telegram-plugin fix that registers an edit-aware inbound handler, debounces by chat/message id, avoids bot echo loops, handles dedupe semantics, and adds focused tests.

## Reproduction Assessment

Yes. A synthetic grammY-style `edited_message` update is enough to reproduce the source-level gap because current main subscribes/keys edited updates but never registers a handler that dispatches them into inbound processing.

## Solution Assessment

Yes. A narrow Telegram-owned handler with debounce and edit-aware dedupe is the maintainable direction; the active path is to review and land #73104 or an equivalent focused fix.
