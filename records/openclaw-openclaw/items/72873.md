## Summary

Keep open. Current main still has Telegram edited updates in allow/keying paths but no runtime `edited_message` handler dispatching edited content, and open PR #73104 already references this issue with closing syntax.

## What This Changes

Process Telegram `edited_message` inbound updates so streaming edits are coalesced and dispatched instead of leaving the agent with the first partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: register an `edited_message` handler that reuses the existing inbound path, coalesces rapid edits by chat/message id, avoids bot-authored echo loops, and lands through #73104 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main can name/key edited updates, but no registered handler passes them into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and focused tests is the maintainable path; because #73104 already uses closing syntax, this issue should stay open until that PR lands or is replaced.
