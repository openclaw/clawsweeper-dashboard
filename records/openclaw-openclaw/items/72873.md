## Summary

Close as implemented on current main. The Telegram plugin now records `edited_message` and `edited_channel_post` updates into the observed-message cache, and the merged regression covers the reported streaming-bot case where the first draft text is replaced by the final edited text in later agent context.

## What This Changes

Handle Telegram edited message updates so streamed bot replies no longer appear truncated in agent context.

## Best Possible Solution

Keep the current Telegram-owned cache update and treat broader edit-to-reprocess behavior as a separate channel product feature if maintainers want that later.

## Reproduction Assessment

Yes for the original bug: the merged regression recreates the partial `K` bot reply followed by a Telegram edit to the final text. On current main, that path no longer reproduces the truncation because the edited message replaces the cached context entry.

## Solution Assessment

Yes. Updating the observed-message cache is the narrowest maintainable fix for the reported truncated context and avoids flooding agent turns from rapid edit streams; a general edit-to-reprocess mode would be a separate feature decision.
