## Summary

Keep open. Current main still allows and keys Telegram edited-message updates but only dispatches inbound `message` and `channel_post` handlers, and open PR #73104 already references this issue with closing syntax, so the issue should remain open until that fix lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates so edited or streaming Telegram messages are debounced and dispatched as inbound content instead of leaving the agent with the initial partial message.

## Best Possible Solution

Land or replace #73104 with a Telegram-owned fix that registers an edit-aware inbound handler, debounces by chat/message id, avoids bot echo loops, and adds focused regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main subscribes/keys `edited_message` updates but has no `bot.on("edited_message")` dispatch path, so a synthetic grammY-style edited update would not reach inbound processing.

## Solution Assessment

Yes. A narrow Telegram plugin handler with debounce and edit-aware dedupe is the maintainable direction; because #73104 is already open with closing syntax, the issue should stay open until that PR or an equivalent fix lands.
