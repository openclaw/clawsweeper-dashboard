## Summary

Keep open. Current main still requests and keys Telegram edited-message updates, but it does not register a Telegram `edited_message` runtime handler; open PR #73104 is the active unmerged implementation candidate and uses closing syntax for this issue.

## What This Changes

Handle Telegram `edited_message` updates so rapidly edited streaming bot messages are coalesced and dispatched with their final content.

## Best Possible Solution

Keep the fix in the Telegram plugin: register an `edited_message` handler that reuses the existing inbound path, debounces by chat/message id, avoids bot-authored echo loops, and lands with focused registration, coalescing, dedupe, and dispatch tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main can request and key the update type, but no registered handler sends it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin change is the maintainable path, and the best next step is to review and land #73104 or replace it with an equivalent focused fix.
