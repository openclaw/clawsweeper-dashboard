## Summary

Current main now handles Telegram edit updates for the reported streamed-bot truncation path by recording edited messages back into the reply-context cache, with regression coverage proving later agent context sees the final edited text instead of the first draft.

## What This Changes

The issue requests Telegram `edited_message` handling so streamed bot replies do not remain truncated at the initial partial message.

## Best Possible Solution

Keep the merged cache-based Telegram edit handling and track any broader edit-triggered reprocessing behavior separately by channel.

## Reproduction Assessment

Yes for the original failure: the merged regression test models a bot reply that starts as `K`, receives an `editedMessage` with the same Telegram message id, and then proves later context contains the final edited text. Current main no longer reproduces the truncation.

## Solution Assessment

Yes. Updating the Telegram reply-context cache for edit updates solves the observed streamed-bot truncation without dispatching a new agent turn for every edit, which is narrower and avoids edit-spam behavior.
