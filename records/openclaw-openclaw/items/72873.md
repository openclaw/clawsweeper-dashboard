## Summary

Keep open. Current main still drops Telegram edited-message updates before they reach the inbound pipeline, and the active open PR #73104 already references this issue with closing syntax, so this issue should remain open until that implementation lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates with debounce and dedupe so streaming bot edits are processed as final inbound content.

## Best Possible Solution

Keep the fix in the Telegram plugin: register an edit-aware inbound handler, coalesce rapid edits by chat/message id, avoid bot-authored echo loops, and cover dispatch plus dedupe behavior with focused tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main has update-key plumbing but no registered handler that dispatches edited content into the inbound path.

## Solution Assessment

Yes. The narrow Telegram-owned handler with debounce and edit-aware dedupe is the maintainable direction; the active path is to review and land #73104 or replace it with an equivalent focused fix.
