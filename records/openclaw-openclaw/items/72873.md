## Summary

Keep open. Current main still allows and keys Telegram edited updates but only registers inbound handlers for `message` and `channel_post`, and open PR #73104 references this issue with closing syntax, so this issue should remain open until that fix lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates so edited or streaming Telegram messages are debounced and dispatched as final inbound content.

## Best Possible Solution

Land or replace #73104 with a Telegram-owned fix that registers an edit-aware inbound handler, debounces by chat/message id, avoids bot echo loops, and adds focused regression coverage.

## Solution Assessment

Yes. A narrow Telegram-plugin handler with debounce and edit-aware dedupe is the maintainable direction; the active path is to review and land #73104 or an equivalent focused fix.
