## Summary

Keep open. Current main still requests and keys Telegram edited-message updates, but the Telegram runtime only registers inbound handlers for normal messages and channel posts, so `edited_message` updates have no dispatch path. Open PR #73104 is already the active closing implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

Add Telegram `edited_message` inbound handling so edited or streaming bot messages are debounced and processed instead of being dropped after the initial message.

## Best Possible Solution

Keep the fix in the Telegram plugin: register an `edited_message` handler that reuses the existing inbound message path, coalesces rapid edits by chat/message id, avoids bot-authored echo loops, and locks the behavior with focused registration, debounce, dedupe, and dispatch tests. The active path is to review and land #73104 or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes. A synthetic Telegram `edited_message` update is enough to reproduce the source-level gap because current main requests that update type but has no registered handler to pass it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and dedupe coverage is the maintainable boundary; because #73104 already uses closing syntax for this issue, the issue should remain open until that PR lands or is replaced.
