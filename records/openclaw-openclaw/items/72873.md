## Summary

Current main still lacks a Telegram inbound `edited_message` handler even though edited updates are allowed/keyed, and an open unmerged PR already references this issue with closing semantics, so this issue should remain open until that fix lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates so streamed bot edits are debounced and delivered as inbound content instead of leaving OpenClaw with the first partial message.

## Best Possible Solution

Land or replace #73104 with a Telegram-plugin fix that registers an edit-aware inbound handler, debounces by chat/message id, avoids bot echo loops, handles dedupe semantics, and adds focused tests.

## Reproduction Assessment

Yes. A synthetic grammY-style `edited_message` update is enough to reproduce the source-level gap because current main allows/keys edited updates but never registers a handler that dispatches them into inbound processing.

## Solution Assessment

Yes. A narrow Telegram-owned handler with debounce and edit-aware dedupe is the maintainable direction; the active path is to review and land #73104 or an equivalent focused fix.
