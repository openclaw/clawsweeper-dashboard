## Summary

Keep open. Current main still subscribes to Telegram edited-message update types but has no runtime handler that dispatches `edited_message` into the inbound pipeline, and open PR #73104 is already the paired implementation candidate using closing syntax.

## What This Changes

Add Telegram `edited_message` handling so edited or streaming Telegram messages are processed after debouncing instead of leaving the agent with only the initial partial message.

## Best Possible Solution

Keep the fix in the Telegram plugin: register an `edited_message` handler that reuses the inbound message path, coalesces rapid edits by chat and message id, avoids self-authored echoes, and adds focused registration, debounce, dedupe, and dispatch tests.

## Reproduction Assessment

Yes. A synthetic Telegram update with `update.edited_message` is enough to reproduce the source-level gap because current main requests that update type but has no registered runtime handler that feeds it into inbound processing.

## Solution Assessment

Yes. The narrow Telegram plugin handler with debounce and dedupe coverage is the maintainable fix boundary; because PR #73104 is already the open closing implementation candidate, this issue should remain open until that PR lands or is replaced.
