## Summary

Keep open. Current main still requests and keys Telegram edited-message updates, but it does not register a runtime `edited_message` handler to dispatch edited content into the inbound pipeline. #73104 is already the active open closing PR for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates so streaming bot edits are debounced/coalesced and dispatched as inbound content instead of leaving the agent with only the initial partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: add an `edited_message` handler that reuses the existing inbound message path, coalesces rapid edits by chat/message id, avoids bot echo loops, and lands with focused registration, debounce, dedupe, and dispatch tests.

## Reproduction Assessment

Yes. A synthetic Telegram `edited_message` update is enough to reproduce the gap at source level because current main requests and keys the update type but has no handler that passes it to inbound processing.

## Solution Assessment

Yes. The narrow Telegram plugin direction described by #73104 is the maintainable fix; this issue should stay open until that PR lands or is replaced by an equivalent patch.
