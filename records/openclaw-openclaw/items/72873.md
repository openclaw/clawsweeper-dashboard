## Summary

Current main now handles the reported Telegram streaming-edit symptom by caching `edited_message` and `edited_channel_post` updates so later agent context uses the final edited text instead of the first draft; the merged maintainer PR intentionally chose cache update without spawning another agent turn.

## What This Changes

The issue asks Telegram inbound handling to account for `edited_message` updates so streamed bot edits do not leave OpenClaw with only the initial partial message.

## Best Possible Solution

Keep the merged Telegram cache-update behavior for streamed bot replies; any future opt-in re-dispatch-on-edit behavior should be tracked as a separate product decision.

## Reproduction Assessment

Yes. The source-level reproduction is the regression case from the merged fix: an initial bot message with a partial body is followed by an `edited_message` update for the same Telegram message id, and later agent context must contain the edited final text.

## Solution Assessment

Yes. The merged cache update is the narrow Telegram-owned fix for the reported truncated streaming-bot context, while avoiding a new debounce-triggered agent turn that would be a broader behavior choice.
