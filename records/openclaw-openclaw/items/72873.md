## Summary

Keep open. Current main still requests Telegram edited-message updates and can key them for sequencing/dedupe, but it has no runtime `edited_message` handler to dispatch edited content into the inbound pipeline. Open PR #73104 is already the active closing implementation candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Handle Telegram `edited_message` updates so streaming edits from other bots are coalesced and processed as inbound content instead of leaving only the initial partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: add an `edited_message` handler that reuses the inbound message path, coalesces rapid edits by chat/message id, avoids bot-authored echo loops, and locks the behavior with focused registration, debounce, dedupe, and dispatch tests. The active path is to review and land #73104 or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes. A synthetic Telegram `edited_message` update is enough to reproduce the source-level gap because current main asks for the update type but has no handler that passes it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and dedupe coverage is the maintainable solution; because #73104 already uses closing syntax for this issue, this issue should remain open until that PR lands or is replaced.
