## Summary

Keep open: current main has static tool filtering and Codex dynamic-tool excludes, but it does not expose a user-facing lazy schema materialization flow for first-party OpenClaw tools.

## What This Changes

Expose user-editable controls to keep a small core of first-party OpenClaw MCP/dynamic tools eager and lazy-load or rediscover the remaining schemas on demand.

## Best Possible Solution

Design a documented lazy schema contract that composes with existing tool policy, uses provider-specific defer support only where safe, and keeps eager fallback behavior elsewhere.

## Reproduction Assessment

Not applicable; this is a feature request for new config/runtime behavior, not broken existing behavior. Source inspection confirms current main has static filtering but no lazy first-party schema discovery path.

## Solution Assessment
