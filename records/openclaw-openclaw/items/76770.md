## Summary

Keep open: current main has static tool profiles, allow/deny policy, and Codex dynamic-tool exclusion controls, but it does not expose a defer-loading flag, lazy discovery index, or first-party OpenClaw tool schema materialization flow.

## What This Changes

Request a user-editable config option to gate or defer first-party OpenClaw MCP/dynamic tool schemas and load non-core tool schemas only when needed.

## Best Possible Solution

Design a documented lazy schema/index contract that composes with existing tool policy, uses provider-specific defer support where available, and preserves eager fallback behavior elsewhere.

## Reproduction Assessment

Not applicable; this is a feature request for new config/runtime behavior, not broken existing behavior. Source inspection confirms current main has static filtering and Codex excludes, but no lazy first-party schema discovery path.

## Solution Assessment

Unclear; both requested shapes are plausible, but the best implementation needs maintainer design across OpenClaw tool policy, Codex deferLoading support, plugin tools, and eager-provider fallback behavior.
