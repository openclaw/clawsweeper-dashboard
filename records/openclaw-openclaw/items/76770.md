## Summary

Keep open: current main still has static tool policy and Codex static dynamic-tool exclusions, but no user-facing lazy/defer-loading contract for first-party OpenClaw tool schemas. Because this would add config and cross-runtime behavior, it needs maintainer design rather than closure or an automated bug fix.

## What This Changes

The issue requests a user-editable flag or per-tool policy to defer loading first-party OpenClaw MCP tool schemas.

## Best Possible Solution

Define and document a lazy schema contract that composes with existing tool policy and uses provider/Codex defer support only where the runtime can fall back safely.
