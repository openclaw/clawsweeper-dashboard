## Summary

Keep open: current main already has documented `tools.allow`/`tools.deny`/profile filtering that can reduce MCP tool exposure, but it does not expose the requested `defer_loading` or `tools_index` style lazy schema materialization. This is a core runtime/config product decision rather than a ClawHub-only plugin request.

## What This Changes

Expose user-editable gating or deferred loading for first-party OpenClaw MCP tool schemas so long-running agents can avoid paying every schema on each turn.

## Best Possible Solution

Define a backwards-compatible core design that preserves existing tool policy semantics, then add lazy schema discovery only if maintainers choose that protocol/runtime direction.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection does confirm the requested `defer_loading`/`tools_index` surface is absent on current main.

## Solution Assessment

Unclear: Option A overlaps existing `tools.allow`/`tools.deny`, while Option B adds new runtime/protocol semantics. A maintainer design decision should come before an implementation PR.
