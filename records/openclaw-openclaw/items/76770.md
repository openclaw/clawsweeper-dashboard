## Summary

Keep open: current main has static tool reduction controls and Codex dynamic-tool exclusion controls, but it does not expose a user-facing defer-loading flag, lazy discovery index, or first-party OpenClaw tool schema materialization flow.

## What This Changes

The issue requests a user-editable way to defer or gate first-party OpenClaw MCP tool schemas so long-running sessions avoid paying every schema cost on every turn.

## Best Possible Solution

Add a documented lazy schema contract that composes with existing profile and allow/deny policy, uses provider-specific defer support only where safe, and keeps eager fallback behavior elsewhere.

## Reproduction Assessment

Not applicable; this is a feature request for new config/runtime behavior, not broken existing behavior. Source inspection confirms current main has static filtering and Codex excludes, but no lazy first-party schema discovery path.

## Solution Assessment

Unclear; the requested directions are plausible, but the best implementation needs maintainer design across OpenClaw tool policy, Codex deferLoading support, bundled/plugin tools, and eager-provider fallback behavior.
