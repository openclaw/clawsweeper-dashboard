## Summary

Keep open: current main has static profiles, allow/deny policy, and Codex-native static excludes, but it does not expose a user-facing defer-loading flag or a lazy schema discovery/materialization contract for first-party OpenClaw/MCP tools.

## What This Changes

The issue requests a user-configurable defer-loading or per-tool gating mode for first-party OpenClaw MCP tool schemas to reduce per-turn prompt schema cost.

## Best Possible Solution

Design and document a lazy schema contract that composes with existing profile and allow/deny policy, uses provider-specific defer support only where safe, and preserves eager fallback behavior elsewhere.

## Reproduction Assessment

Not applicable; this is a feature request for new config/runtime behavior, not a broken existing contract. Source inspection confirms current main has static filtering but no lazy first-party schema discovery path.

## Solution Assessment

Unclear; the requested direction is plausible, but the best implementation needs maintainer design across OpenClaw tool policy, MCP loopback tools/list behavior, Codex deferLoading, and provider fallback behavior.
