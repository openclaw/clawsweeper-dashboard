## Summary

Keep open: current main only supports static tool profiles/allow/deny and Codex dynamic-tool exclusion, while this asks for new lazy first-party OpenClaw tool schema loading or rediscovery behavior.

## What This Changes

Expose user-editable config that keeps first-party OpenClaw MCP/dynamic tool schemas small and lazy-loads or disables non-core tools.

## Best Possible Solution

Design a documented lazy schema contract that composes with current tool policy, uses provider-specific deferral only where safe, and preserves eager fallback elsewhere.
