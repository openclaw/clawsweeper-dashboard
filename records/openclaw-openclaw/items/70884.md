## Summary

Keep this PR open, but it should not merge as-is: the trusted-proxy docs wording contradicts current WebSocket behavior, and the branch now carries unrelated provider-request runtime/config changes that need their own feature/security/protocol review.

## What This Changes

The PR adds trusted-proxy WebSocket scope-clearing documentation and also adds per-agent/per-cron provider request override plumbing across agents, config, and cron code.

## Best Possible Solution

Split or rebase this into a focused docs correction that distinguishes missing-device WebSocket scope clearing from paired trusted-proxy WebSocket scope bounding, and handle provider-request headers in https://github.com/openclaw/openclaw/pull/79990 with protocol/security/docs coverage.

## Reproduction Assessment

Yes. The blockers are source-reproducible on current main: the WebSocket handler/test prove the HTTP-only docs note is inaccurate, and the cron TypeBox protocol schema omits the new providers field while rejecting additional payload properties.

## Solution Assessment

No. The docs fix is the right general direction, but the wording must match the existing WebSocket header behavior and the unrelated provider-header feature should be reviewed through the canonical feature PR instead of this docs branch.
