## Summary

Keep this PR open: the docs-only direction is useful, but the current wording conflicts with current main after the trusted-proxy Control UI scope fix and needs revision before merge.

## What This Changes

The PR adds trusted-proxy and gateway protocol documentation about WebSocket scope clearing, `x-openclaw-scopes`, troubleshooting, and `dangerouslyDisableDeviceAuth` guidance.

## Best Possible Solution

Keep this as a focused docs correction, but state that successful trusted-proxy Control UI WebSockets can preserve or bound scopes with x-openclaw-scopes, and reserve scope-clearing plus break-glass guidance for the paths that actually clear scopes.

## Reproduction Assessment

Yes. Source inspection and current tests show the review findings: the WebSocket handler reads x-openclaw-scopes for trusted-proxy Control UI upgrades, tests assert bounded WebSocket scopes, and the clearing predicate excludes trustedProxyAuthOk.

## Solution Assessment

No. A docs-only fix is the right shape, but the current wording overcorrects; the maintainable solution is to narrow the docs to the missing-device cases that actually clear scopes while preserving the current trusted-proxy WebSocket header behavior.
