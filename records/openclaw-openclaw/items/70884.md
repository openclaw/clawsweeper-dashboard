## Summary

Keep this PR open: the docs gap is real, but the proposed wording still conflicts with current Gateway WebSocket auth behavior and security guidance.

## What This Changes

The PR adds Gateway trusted-proxy and protocol documentation for device-less WebSocket scope clearing, `x-openclaw-scopes`, missing-scope troubleshooting, and `dangerouslyDisableDeviceAuth`.

## Best Possible Solution

Revise the docs to distinguish paired, fresh, and device-less trusted-proxy Control UI WebSockets, WebSocket scope-header bounding, and narrow break-glass guidance.

## Reproduction Assessment

Yes for the review findings: current source and tests show the WebSocket header is read and paired sessions are bounded by it, while device-less trusted-proxy sessions clear scopes. This is a source-verifiable docs mismatch, not a runtime bug reproduction.

## Solution Assessment

No. A docs-only correction is the right surface, but this patch should align with the current WebSocket contract and narrower missing-device exceptions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Describe WebSocket scope headers accurately:** `docs/gateway/trusted-proxy-auth.md:323-325`
  - body: The added note says `x-openclaw-scopes` applies to HTTP endpoints only, but current main reads that header from trusted-proxy Control UI WebSocket upgrades and tests assert paired sessions are bounded to the proxy-declared scope set. Please document paired WebSocket scope bounding separately from device-less scope clearing.
  - confidence: 0.95
- **[P2] Narrow missing-device scope guidance:** `docs/gateway/protocol.md:719-725`
  - body: This says token, password, and trusted-proxy Control UI connections without device identity are allowed and then lose scopes. Current policy rejects most missing-device Control UI token/password paths unless narrow local `allowInsecureAuth` or break-glass bypass applies, so this should describe the trusted-proxy allowed-empty-scope case separately.
  - confidence: 0.9
