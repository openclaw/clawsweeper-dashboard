## Summary

Keep open: the branch still documents trusted-proxy WebSocket scope behavior in a way that conflicts with current main, and the external PR needs revised proof before merge.

## What This Changes

The PR adds trusted-proxy and Gateway protocol docs notes about missing-device WebSocket scope clearing, `x-openclaw-scopes`, troubleshooting, and `dangerouslyDisableDeviceAuth`.

## Best Possible Solution

Revise the docs to distinguish successful trusted-proxy Control UI WebSocket scope preservation/bounding from the remaining missing-device scope-clearing paths, and keep break-glass guidance narrowly scoped.

## Reproduction Assessment

Yes. Source inspection and existing tests on current main show the relevant behavior: WebSocket upgrades read `x-openclaw-scopes`, successful trusted-proxy Control UI auth skips the clearing path, and tests lock in proxy-scope bounding.

## Solution Assessment

No. A docs-only correction is the right surface, but this wording overcorrects; the safer fix is to document the current trusted-proxy WebSocket contract and narrow the scope-clearing guidance.
