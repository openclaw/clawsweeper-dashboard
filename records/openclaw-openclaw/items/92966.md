## Summary

Keep this PR open, but do not merge it as-is: the reported duplicate memory output is source-reproducible, while the patch deduplicates by text alone and can drop valid repeated assistant lines outside the thinking-stripped child case.

## What This Changes

The PR adds a last-assistant-text guard in `getRecentSessionContent()` and session-memory regression tests to collapse consecutive assistant transcript rows with identical extracted text.

## Best Possible Solution

Use one canonical narrow fix that filters only the cleaned child of a raw thinking assistant entry, or a storage-layer fix if maintainers choose that, with regression coverage for legitimate repeated assistant messages and real `/new` or `/reset` proof.

## Reproduction Assessment

Yes at source level: current main reads both raw thinking+text and stripped text-only assistant transcript rows through the same text extractor, producing duplicate `assistant:` memory lines. A live thinking-enabled provider run was not established in this read-only review.

## Solution Assessment

No. Deduplicating by adjacent assistant text is a plausible mitigation, but the maintainable fix should target the actual parent-linked cleaned duplicate shape or an explicit storage-layer decision.
