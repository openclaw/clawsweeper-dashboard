## What This Changes

The PR adds a session-memory transcript dedupe guard for consecutive assistant messages with identical extracted text and adds regression tests for duplicate and non-consecutive repeated assistant text.

## Best Possible Solution

Use one canonical narrow fix that skips only the parent-linked cleaned child of a raw thinking assistant row, or an explicit storage-layer change if maintainers choose that, with regression coverage and redacted real `/new` or `/reset` proof.

## Reproduction Assessment

Yes, at source level: current main reads both raw thinking+text and stripped text-only assistant transcript rows through the same text extractor, producing duplicate `assistant:` memory lines. A live thinking-enabled provider run was not established in this read-only review.

## Solution Assessment

No. Deduplicating by adjacent assistant text is a plausible mitigation, but the maintainable fix should target the actual parent-linked cleaned duplicate shape or an explicit storage-layer decision.
