## Summary

Keep open: current main still has the shared-client race, while the latest PR head now preserves the injected client factory seam and has no blocking code-review findings. Exact-head CI was still partly queued during review.

## What This Changes

This PR adds an isolated Codex app-server client factory, routes run attempts to it when `agents.defaults.maxConcurrent > 1`, closes isolated clients on exit paths, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the isolated-client fix after exact-head CI completes, keeping the behavior tied to the existing concurrency setting and leaving spawned-run-specific isolation to the already related follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main allows default main concurrency of 4, but a second Codex app-server run with a different agent directory clears and closes the shared client used by the first run.

## Solution Assessment

Yes. Routing concurrent runs to per-run isolated app-server clients is the narrowest maintainable fix for this race, and the latest head preserves the existing test injection seam while keeping maxConcurrent 1 on the shared path.
