## Summary

The central shipped regression is already corrected on current main: Slack now deep-merges account-level streaming settings over channel defaults, so the reported account `mode: off` suppresses the partial delivery path. The fix landed after v2026.6.5 and is not yet in a tagged release.

## What This Changes

Restore Slack account-level streaming precedence so `mode: off` overrides the channel default and prevents partial snapshots from being delivered.

## Best Possible Solution

Ship the current account-level streaming merge in the next release, and track a separate partial-mode lifecycle issue only if duplicates persist when streaming is intentionally enabled.

## Reproduction Assessment

Yes. The supplied channel `partial` plus account `off` configuration maps directly to the account merge surface, and current main contains an explicit correction for that precedence.

## Solution Assessment

Yes. Correcting canonical Slack account resolution is narrower and more maintainable than changing agent output or adding a delivery fallback; any remaining intentionally enabled partial-stream behavior should be isolated separately.
