## Summary

This is a credible released Slack delivery bug that remains necessary to investigate. The v2026.6.5 account resolver already contains account-over-channel streaming merging and dispatch consumes `account.config`, but partial mode still intentionally rotates the draft at assistant boundaries; a focused full-path regression test is needed to identify where the reported account policy or draft identity is lost.

## What This Changes

Fix Slack multi-account delivery so an account-level `streaming.mode="off"` overrides channel-level partial streaming and one agent reply is not emitted as many progressive-prefix messages.

## Best Possible Solution

Exercise the real multi-account inbound path, ensure the selected account's resolved streaming policy remains authoritative, and preserve one editable Slack preview where Codex commentary or tool boundaries belong to the same logical reply.

## Reproduction Assessment

Yes at the source level: configure channel mode `partial`, active account mode `off`, and drive a long Codex turn with commentary or tool boundaries while observing Slack sends. This read-only review did not execute a live current-main Slack reproduction.

## Solution Assessment

No, the referenced progress-only fix is not sufficient. The narrow maintainable solution is a full-path account-aware regression test followed by a repair where the resolved policy or draft message identity is actually lost.
