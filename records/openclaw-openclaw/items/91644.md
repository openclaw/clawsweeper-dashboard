## Summary

Keep open because the new audio-speech and TTS model-list paths let unknown-agent errors escape as server 500s, unlike sibling OpenAI-compatible handlers; the feature direction and proof are otherwise useful, but the additive config/plugin API remains compatibility-sensitive.

## What This Changes

The PR adds an opt-in OpenAI-compatible `/v1/audio/speech` gateway endpoint, lists compatible TTS providers under `/v1/models`, adds a speech-provider capability flag, and updates providers, tests, and docs.

## Best Possible Solution

Handle unknown-agent resolution with the same OpenAI JSON 400 behavior as sibling handlers, clarify the docs scope around speech versus agent-run/session behavior, and land only after maintainers accept the additive config/plugin API.

## Solution Assessment

Mostly yes: reusing the TTS runtime, provider registry, shared auth helpers, and provider capability metadata is the right owner-boundary shape, but error parity and doc scoping need repair before this is the best fix.
