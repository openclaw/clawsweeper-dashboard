Overall correctness: patch is incorrect

Full review comments:

- **[P2] Do not advertise providers that ignore speech overrides:** `src/gateway/audio-speech-http.ts:176-185`
  - body: The endpoint lists every configured speech provider as `tts/<provider>`, then sends the same `{ voice, speed, responseFormat }` override shape to all of them. Existing providers such as ElevenLabs read `voiceId` and `outputFormat` instead, so a request like `model: "tts/elevenlabs"` with a `voice` can succeed with the configured default voice while `/v1/models` advertised the provider as OpenAI-compatible. Either translate/validate per provider or exclude providers that cannot honor the documented fields.
  - confidence: 0.86
