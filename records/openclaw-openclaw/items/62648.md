## Summary

Keep open: the diagnostic gap is still real, but this PR should not merge as-is because it moves the property name into the shared schema-validator message instead of rendering the existing structured field at config diagnostic consumers. It also still lacks contributor-provided real behavior proof.

## What This Changes

The PR changes `formatAjvErrors` to append AJV `additionalProperties` property names to the shared validation message and adds one schema-validator unit test.

## Best Possible Solution

Render `error.additionalProperty` exactly once in config diagnostic consumers, especially plugin config validation, while keeping `JsonSchemaValidationError.message` as the base AJV message and adding focused regression coverage.

## Reproduction Assessment

Yes, via source inspection rather than execution. Plugin config validation receives structured `additionalProperty` from `validateJsonSchemaValue`, but the plugin config diagnostic renderer still formats only `error.message`.

## Solution Assessment

No. The safer maintainable fix is to render the structured `additionalProperty` field in the remaining config consumer instead of changing the shared message string that other consumers already decorate.
