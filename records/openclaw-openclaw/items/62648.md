## What This Changes

## Best Possible Solution

Render `additionalProperty` exactly once in the config diagnostic consumers that need it, keep the shared schema-validator message as the base AJV message, and add regression coverage for plugin config plus the existing bundled-channel formatter.

## Reproduction Assessment

Yes, via source inspection rather than command execution. Current main exposes `error.additionalProperty`, raw channel diagnostics render it, and plugin config diagnostics still format only `error.message`.

## Solution Assessment

No. The safer maintainable fix is to render the structured field in the remaining config consumers instead of changing the shared message string that other consumers already decorate.
