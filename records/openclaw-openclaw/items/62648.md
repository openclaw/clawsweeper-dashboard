## Summary

Keep open: the diagnostic goal is useful, but this patch changes the shared schema-validator message instead of rendering the structured `additionalProperty` field at the config consumers, which can duplicate the key on an existing raw-channel path. The external PR also still lacks after-fix real behavior proof for the CLI/config diagnostic path.

## What This Changes

This PR appends AJV `additionalProperties` keys to `JsonSchemaValidationError.message` and adds a schema-validator unit test for that message.

## Best Possible Solution

Render `additionalProperty` exactly once in the config diagnostic consumers that need it, keep the shared schema-validator message as the base AJV message, and cover both plugin config diagnostics and the raw-channel no-duplication case.

## Reproduction Assessment

Yes, via source inspection. Current main exposes `error.additionalProperty`, raw-channel diagnostics already render it, and plugin config diagnostics still format only `error.message`.

## Solution Assessment

No. The requested user-facing diagnostic is worth keeping, but the safer implementation is consumer-level rendering of the structured field rather than mutating the shared base message.
