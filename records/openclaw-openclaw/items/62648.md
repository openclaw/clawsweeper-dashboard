## Summary

Keep open: the diagnostic goal is valid, but this PR should not merge as written because current main now exposes the AJV additional property structurally, so mutating the shared message can duplicate the property name while the plugin config renderer still needs a targeted fix.

## What This Changes

The PR appends AJV additionalProperties property names to the shared schema-validator message string and adds a schema-validator unit test for that message.

## Best Possible Solution

Render `error.additionalProperty` exactly once in config diagnostic consumers, especially plugin config validation, while keeping `JsonSchemaValidationError.message` as the base AJV message.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: plugin config validation receives structured `additionalProperty` from `validateJsonSchemaValue`, but `src/config/validation.ts` currently renders only `error.message` for plugin config diagnostics.

## Solution Assessment

No. The narrower maintainable fix is to consume the structured `additionalProperty` field in the remaining diagnostic renderer instead of appending it inside the shared schema-validator message.
