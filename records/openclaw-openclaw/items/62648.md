## Summary

Keep open: the diagnostic improvement is useful, but this patch puts the property name into the shared schema-validator message, which can duplicate existing consumer rendering and still lacks real terminal/runtime proof.

## What This Changes

The PR appends AJV `additionalProperties` property names to `JsonSchemaValidationError.message` and adds a schema-validator unit test for that message text.

## Best Possible Solution

Render `additionalProperty` exactly once in the config diagnostic consumers that need it, keep the shared schema-validator message as the base AJV message, and cover plugin config output plus the existing no-duplication path.

## Reproduction Assessment

Yes, via source inspection. Current main exposes `error.additionalProperty`, raw-channel diagnostics already render it, and plugin config diagnostics still format only `error.message`; I did not run tests because this review is read-only.

## Solution Assessment

No. The user-facing diagnostic is worth keeping, but the safer fix is consumer-level rendering of the structured field rather than mutating the shared base AJV message.
