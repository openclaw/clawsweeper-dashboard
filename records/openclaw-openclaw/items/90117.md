## What This Changes

The PR skips the synchronous zero-hit sync-and-retry path for QMD-backed `memory_search`, preserves the existing retry for other memory backends, and adds a regression test.

## Best Possible Solution

Land one clean, proof-backed backend-aware guard that keeps QMD update work out of the interactive `memory_search` zero-hit path while preserving non-QMD retry behavior.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 still await forced sync after zero `memory_search` hits, and QMD sync awaits update work. The linked issue also includes live QMD evidence, but I did not run an independent live QMD timing repro in this read-only pass.

## Solution Assessment

Yes as a fix shape, but not merge-ready on this branch: the backend guard is the narrow owner-local mitigation, while async or debounced QMD refresh can remain a follow-up if maintainers want freshness repair without blocking turns.
