## Summary

Keep open: current main still has the false-positive empty-tool allowlist path for Lobster-driven llm-task runs, and #74139 is an open unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue asks for Lobster-driven llm-task runs to ignore inherited tool allowlists when the embedded JSON-only model call intentionally disables tools.

## Best Possible Solution

Review and land #74139 or an equivalent focused fix that distinguishes inherited config allowlists from runtime tool constraints, with regression coverage for llm-task and generic no-tool runs.

## Reproduction Assessment

Yes. The report gives exact config and Lobster invocation steps, and source inspection on current main shows llm-task sets disableTools, the embedded runner clears toolsRaw, and the guard still errors when inherited allowlist sources remain.

## Solution Assessment

Yes with a caveat. The narrow maintainable fix belongs in the empty allowlist guard or embedded-runner policy path, but it should preserve the runtime toolsAllow fail-closed contract already covered by tests.
