## Summary

Keep open: current main still contains the reported false-positive path, and #74139 is an open unmerged PR that explicitly closes this issue.

## What This Changes

The issue reports that Lobster workflows invoking `llm-task` fail because inherited tool allowlists are still treated as an error when the embedded model run intentionally disables tools.

## Best Possible Solution

Review and land #74139 or an equivalent focused fix that distinguishes intentional no-tool embedded/model runs from invalid empty allowlist configurations.

## Reproduction Assessment

Yes. The report gives exact config and invocation steps, a release user confirmed the same failure on 2026.4.29, and current-main source still matches the failing path.

## Solution Assessment

Yes with a caveat. The narrow fix belongs in the allowlist guard or embedded-runner policy path, while preserving the runtime `toolsAllow` fail-closed contract already covered by tests.
