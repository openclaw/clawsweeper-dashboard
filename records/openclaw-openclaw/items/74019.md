## What This Changes

The issue requests that Lobster workflows invoking llm-task avoid failing the explicit tool allowlist guard when llm-task intentionally disables tools for its embedded JSON-only model call.

## Best Possible Solution

Review and land #74139 or an equivalent focused fix that distinguishes intentional no-tool embedded runs from real empty runtime tool constraints, with regression coverage for both cases.

## Reproduction Assessment

Yes. The report gives exact config and invocation steps, and current source inspection shows llm-task sets disableTools, the runner clears callable tools, and the guard still errors when inherited allowlist sources remain.

## Solution Assessment

Unclear for the exact proposed PR until reviewed. The narrow direction is right, but the safest implementation should preserve the runtime toolsAllow fail-closed contract already covered by tests.
