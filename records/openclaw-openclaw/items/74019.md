## What This Changes

Allow Lobster workflows to invoke llm-task's JSON-only embedded model call when inherited tools.alsoAllow includes lobster/llm-task without triggering the empty allowlist failure.

## Best Possible Solution

Review and land #74139 or an equivalent focused fix that distinguishes inherited config allowlists from caller-supplied runtime constraints for intentional no-tool llm-task/modelRun calls, with regression coverage.

## Reproduction Assessment

Yes. The report gives concrete config, Lobster, and /tools/invoke steps, and static current-main inspection confirms the same path from llm-task through disableTools, inherited allowlist collection, empty tools, and the guard error.

## Solution Assessment

Yes for the requested direction, but current main is not fixed. The safest implementation should skip the empty guard only for intentional no-tool runs caused by inherited policy, while preserving runtime toolsAllow failure semantics.
