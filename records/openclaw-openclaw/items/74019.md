## Summary

Keep open. Current main still has the reported failure path, and #74139 is an open, unmerged PR with closing syntax for this issue, so the issue should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue requests that Lobster workflows invoking `llm-task` should run JSON-only embedded model calls without inherited `tools.alsoAllow` entries triggering the empty explicit allowlist guard.

## Best Possible Solution

Land #74139 or an equivalent focused fix that lets intentional no-tool `llm-task` embedded runs ignore inherited config allowlists while preserving fail-closed behavior for explicit runtime tool requests, with focused coverage in `src/agents/tool-allowlist-guard.test.ts`.

## Reproduction Assessment

Yes. The issue provides concrete config, Lobster, and `/tools/invoke` steps, and current main statically confirms the same path from `llm-task` through disabled tools, collected allowlists, and the empty-callable-tools guard.

## Solution Assessment

Yes for the requested direction. The narrow maintainable fix is to distinguish inherited config allowlists from caller-supplied runtime tool constraints when tools are intentionally disabled, rather than removing the guard for ordinary tool-enabled runs.
