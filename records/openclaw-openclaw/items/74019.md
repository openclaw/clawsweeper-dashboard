## Summary

Keep open: current main still has the reported false-positive empty-tool allowlist path, and #74139 is an open unmerged implementation PR with closing syntax for the same Lobster/`llm-task` bug.

## What This Changes

Report requests that Lobster workflows invoking `llm-task` run JSON-only embedded model calls without failing on inherited `tools.alsoAllow` entries.

## Best Possible Solution

Review and land #74139 or an equivalent focused fix that skips the empty allowlist guard only for intentional no-tool runs caused by inherited config policy, with a regression test for `llm-task`/`disableTools` plus inherited `tools.alsoAllow`.

## Reproduction Assessment

Yes. The report gives concrete config, Lobster, and `/tools/invoke` steps, and static inspection of current main confirms the same path from `llm-task` through disabled tools, inherited allowlist-source collection, and the empty-callable-tools guard.

## Solution Assessment

Yes for the requested direction, but current main is not fixed. The narrow maintainable solution is to distinguish inherited config allowlists from caller-supplied runtime tool constraints when tools are intentionally disabled.
