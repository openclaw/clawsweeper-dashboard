The issue reports a regression where google-gemini-cli OAuth turns became consistently slow after upgrading to OpenClaw 2026.3.23-2, especially in fresh Telegram sessions with minimal prompts.

## Best Possible Solution

Use live macOS proof to decide whether to document/diagnose upstream Gemini CLI Keychain fallback latency or add a plugin-owned, security-reviewed credential path that avoids repeated slow credential retrieval without changing auth precedence unexpectedly.
