## What This Changes

The issue reports a regression where `google-gemini-cli` OAuth turns became consistently slow after OpenClaw 2026.3.23-2, apparently from per-turn credential/keychain lookup latency on macOS.

## Best Possible Solution

Keep this open until maintainers decide whether OpenClaw should safely prepare selected Gemini OAuth credentials for the external CLI process or diagnose/document the upstream keychain delay, backed by live macOS timing proof.

## Reproduction Assessment

No, not as a high-confidence live current-main reproduction. Source inspection shows current main still invokes the local `gemini` CLI and lacks provider-owned execution preparation for this path, but the exact macOS keychain delay needs fresh redacted timing proof.

## Solution Assessment

Unclear. A Google-plugin `prepareExecution` bridge is a plausible maintainable layer, but maintainers need to decide whether OpenClaw should stage selected OAuth credentials for an external CLI or treat this as upstream Gemini CLI/keychain behavior.
