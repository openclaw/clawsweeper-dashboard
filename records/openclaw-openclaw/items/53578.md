## What This Changes

The issue reports a regression where `google-gemini-cli` OAuth turns became consistently slow after upgrading, especially on fresh Telegram sessions with minimal prompts.

## Best Possible Solution

Profile an affected macOS OAuth host end to end, then land a narrow credential-sync, diagnostics, or documentation fix only if the timing shows an OpenClaw-owned delay.

## Reproduction Assessment

## Solution Assessment

Unclear. The credential-sync direction is plausible, but a code fix should be based on timing that separates OpenClaw setup, process spawn, session reuse, auth-profile sync, and upstream Gemini CLI Keychain/FileKeychain initialization.
