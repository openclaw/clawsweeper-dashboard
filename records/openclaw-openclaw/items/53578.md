## Summary

Current main fixes the reported google-gemini-cli OAuth credential path by staging the selected OpenClaw profile into an isolated Gemini CLI home and clearing ambient keychain/auth environment; this is not in the latest release yet.

## What This Changes

The issue reports a regression where google-gemini-cli OAuth turns became consistently slow after OpenClaw 2026.3.23-2, with diagnostics pointing to repeated keychain/FileKeychain credential lookup.

## Best Possible Solution

Keep the current main credential-staging bridge and let the next release carry it to users.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main failure; the report gives concrete macOS keychain/FileKeychain diagnostics, and source/history now show the credential path is bypassed by profile-scoped Gemini CLI staging.

## Solution Assessment

Is this the best way to solve the issue? Yes; the fix lives at the Google-owned CLI prepare-execution boundary, avoids a public credential-forwarding API, and follows Gemini CLI's own env/file contracts.
