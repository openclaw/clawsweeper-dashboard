## Summary

Current main implements the Gemini CLI OAuth credential-staging bridge that addresses the reported per-turn keychain/FileKeychain delay path; the fix is current-main-only and is not in `v2026.6.8`.

## What This Changes

The issue asks OpenClaw to avoid a fixed per-turn slowdown when running `google-gemini-cli` OAuth turns after upgrading to `2026.3.23-2`.

## Best Possible Solution

Keep the current main credential-staging bridge and let the next release carry it to users; users on releases through `v2026.6.8` need a newer main build or later release for this fix.

## Reproduction Assessment

No, not as a current-main failure. The reported release-era path is source-reproducible because `v2026.6.8` lacks the Gemini CLI credential-staging bridge, while current main now stages selected OAuth credentials before execution.

## Solution Assessment

Yes. The current fix sits at the Google plugin CLI `prepareExecution` boundary, uses Gemini CLI's own env/file contracts, and avoids adding a public config or plugin-SDK surface for this bundled-provider bridge.
