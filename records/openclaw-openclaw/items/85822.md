## Summary

## What This Changes

The issue asks OpenClaw to stop or diagnose a repeatable silent Discord reply delay after `embedded run done`, tied to post-run auth-profile/OAuth bookkeeping on the lane.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.9` both await auth-profile success bookkeeping after `embedded run done`, and the issue comments provide production Discord timing for the same gap. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment
