## Summary

Keep open. The PR targets a real Feishu threading gap, but the patch is not mergeable as written: it adds duplicate `MessageThreadId` and `ReplyToId` keys, and the later existing guarded `MessageThreadId` assignment still wins. Current main still only forwards Feishu `MessageThreadId` for topic-session contexts, and the overlapping open Feishu threading work in #60848 should be reconciled before choosing the final policy.

The PR adds Feishu `ctx.rootId` and `ctx.parentId` to the inbound context payload as `MessageThreadId` and `ReplyToId` in `extensions/feishu/src/bot.ts`.

## Best Possible Solution
