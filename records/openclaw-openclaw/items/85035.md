Full review comments:

- **[P1] Restrict hard reflex gates before exposing them:** `src/plugins/registry.ts:2378-2408`
  - body: `registerReflexGates` is wired into the public plugin API and accepts gates from any plugin record with only shape checks. That lets an external plugin hard-deny or rewrite tool calls, outbound messages, persistence, queues, and subagent spawning before existing hook policy boundaries, so this needs removal or an explicit trusted gate before merge.
  - confidence: 0.92
- **[P1] Put queue hooks under prompt-content policy:** `src/plugins/hook-types.ts:89-90`
  - body: `queue_before_enqueue` exposes and can rewrite `event.prompt`, but adding it only to the general hook list leaves it outside the prompt-injection policy enforced by `hooks.allowPromptInjection=false`. A plugin with prompt injection disabled can still register this prompt-rewriting hook, so classify it with the prompt-content guard and add coverage.
  - confidence: 0.9
- **[P2] Send the after hook when a follow-up is blocked:** `src/auto-reply/reply/agent-runner.ts:1331-1336`
  - body: The normal follow-up path returns as soon as `queue_before_enqueue` blocks, before calling `runQueueAfterEnqueueHooks`. Embedded steering emits `queue_after_enqueue` with `enqueued: false` for the same blocked outcome, so normal follow-up blocks become invisible to queue observers.
  - confidence: 0.88
