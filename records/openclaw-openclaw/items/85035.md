## Summary

## What This Changes

The PR adds `queue_before_enqueue` and `queue_after_enqueue` plugin hooks for follow-up queueing and embedded steering, plus SDK docs, baseline hashes, and focused hook/runner tests.

## Best Possible Solution

Keep the queue hook API only if blocked, rewritten, skipped, and enqueued decisions share one consistent after-event contract across follow-up and embedded steering, with focused regression coverage.

## Reproduction Assessment

Yes. By source inspection of the PR diff, the follow-up block branch returns before `runQueueAfterEnqueueHooks`, while the embedded blocked branch emits `queue_after_enqueue` with `enqueued:false`.

## Solution Assessment

Not yet. The queue hook direction is maintainable, but the implementation should make after-event semantics consistent across queue paths before maintainers decide on the public API.
