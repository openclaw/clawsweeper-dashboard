Full review comments:

- **[P1] Restrict hard reflex gates before exposing them:** `src/plugins/registry.ts:2378`
  - body: This registrar accepts gates from any plugin record and immediately stores them; `buildPluginApi` also exposes `registerReflexGates` to every plugin API. That lets an external plugin hard-deny or rewrite tool calls, outbound messages, persistence, and subagent spawning before the existing hook policy checks, so this needs a trusted/explicit policy gate or removal before merge.
  - confidence: 0.91
- **[P1] Put queue hooks under prompt-content policy:** `src/plugins/hook-types.ts:89`
  - body: `queue_before_enqueue` exposes and can rewrite `event.prompt`, but it is added only to the general hook list and is still absent from `PROMPT_INJECTION_HOOK_NAMES`. A plugin configured with `hooks.allowPromptInjection=false` can still register this prompt-rewriting queue hook, so add it to the prompt-content guard and cover the blocked registration case.
  - confidence: 0.9
- **[P2] Send the after hook when a follow-up is blocked:** `src/auto-reply/reply/agent-runner.ts:1331-1336`
  - body: The core follow-up path returns as soon as `queue_before_enqueue` blocks, before calling `runQueueAfterEnqueueHooks`. Embedded steering now emits `queue_after_enqueue` with `enqueued:false` on the same block path, so ordinary follow-up blocks become invisible to observers and OpenCOAT cannot record the applied queue decision.
  - confidence: 0.88
