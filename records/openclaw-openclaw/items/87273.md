## Summary

Keep open: this is an active, focused bug-fix PR with unique diagnostic behavior that current main does not already provide. Current main already removed OAuth-only Realtime voice support through the merged API-key-auth fix, but this branch still adds the missing safe handling for Azure/proxy-looking keys and upstream invalid-key failures, with targeted tests and contributor proof.

## What This Changes

The PR adds OpenAI Realtime voice auth diagnostics for Azure/proxy-looking credentials, preserves custom/Azure endpoint routing, documents the direct-key requirement and emergency override, and expands provider regression tests.

## Best Possible Solution

Land a narrow provider-owned diagnostic once maintainers accept the key-shape heuristic, documented override, and no-OAuth Realtime auth baseline, while keeping first-class Azure Foundry Realtime support in the separate canonical feature issue.

## Reproduction Assessment

Yes, at source level: current main sends available API-key-shaped values through the OpenAI Realtime client-secret path, while this branch proves the obvious proxy/Azure key classes fail before that request. I did not run a live OpenAI or Azure Realtime call in this read-only review.

## Solution Assessment

Yes, with maintainer sign-off: provider-local validation and error rewriting are the narrow owner-boundary fix, and the PR preserves custom/Azure endpoint routes. The remaining judgment is whether the local key-shape heuristic plus env override is the preferred compatibility policy.
