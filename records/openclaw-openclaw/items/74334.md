## Summary

Keep open. Current `main` still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-normalized live windows and silently returns zero when no selected candidate survives rehydration.

## What This Changes

The issue requests a memory-core fix so `memory promote --apply` can rehydrate and apply daily-ingestion candidates whose stored snippets use heading and list normalization.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and log one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current `main` still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; an in-memory probe of the reported shape produces no relocation match.

## Solution Assessment

Yes. Aligning the relocation comparison with the daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates would restore counts but weaken the documented live daily-note re-read guard.
