## Summary

Keep open. Current main still has the reported memory-core normalization mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-collapsed live windows and can silently drop every selected candidate during apply.

## What This Changes

The issue requests a fix for memory-core promotion apply so candidates selected from daily-ingestion snippets rehydrate against live daily notes instead of being silently skipped.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale guard, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a read-only probe of the reported shape produces zero relocation matches.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates restores counts but weakens the documented live daily-note re-read guard.
