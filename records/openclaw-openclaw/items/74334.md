## Summary

Keep open. Current main still has the memory-core mismatch: daily ingestion records heading-prefixed/list-joined snippets, while promotion relocation compares only whitespace-collapsed live windows and silently reports zero applied candidates when every selected candidate fails rehydration.

## What This Changes

The issue requests fixing memory-core promotion rehydration so daily-dreaming snippets can be relocated and applied instead of silently producing zero promotions.

## Best Possible Solution

Memory-core should share or equivalently mirror the daily-ingestion snippet normalizer in promotion relocation, keep live daily files as the source of truth, add a heading-plus-list regression test, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily note with a meaningful Markdown heading and bullet list produces a stored `Heading: item; item` chunk that cannot match the current whitespace-only live-window comparison, and the issue provides dry-run/apply output from an affected host.

## Solution Assessment

Yes for aligning the normalizers and adding all-dropped observability; no for the workaround fallback, because returning the stored candidate would bypass the documented live-source stale-snippet guard.
