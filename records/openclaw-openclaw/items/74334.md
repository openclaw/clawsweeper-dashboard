## Summary

Keep open. Current main still has the reported memory-core apply-path mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-collapsed live windows and drops candidates that cannot be relocated.

## What This Changes

The issue asks memory-core promotion apply to rehydrate daily-ingested snippets with compatible normalization and warn when all selected candidates fail relocation.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, preserve live-source stale checks, add a heading-plus-list apply regression, and emit one clear diagnostic when every selected candidate drops.

## Reproduction Assessment

Yes. A source-level probe using the reported heading/list shape returns no match under current normalization and comparison logic, and current apply code returns `applied: 0` when `rehydratedSelected` is empty.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but weaken the documented live daily-note re-read guard.

## Repair Work Prompt

Fix the memory-core short-term promotion apply bug reported in https://github.com/openclaw/openclaw/issues/74334. Current daily ingestion stores snippets with heading prefixes, stripped list markers, semicolon-joined list chunks, and 280-character truncation, while `relocateCandidateRange` compares only whitespace-collapsed live windows, causing `openclaw memory promote --apply` to drop qualified daily-ingested candidates. Implement the fix in memory-core, preferably by sharing or faithfully mirroring the daily-ingestion normalization for relocation comparison while preserving the live-source stale/deleted-snippet guard. Do not blindly fall back to stored candidates as the main fix. Add a regression where a daily note section like `### Cache Eviction Policy` plus list bullets is ingested/ranked and then successfully applied from the live note. Add a single clear diagnostic when all selected candidates fail rehydration. Coordinate with related PR #74051 because it touches adjacent promotion span/gate behavior, but keep this PR focused on the normalizer mismatch. Add a user-facing CHANGELOG.md fix entry crediting the human reporter if appropriate.
