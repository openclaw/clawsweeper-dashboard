## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-joined snippets, while promotion rehydration compares only whitespace-collapsed raw source windows and silently drops candidates when relocation fails.

## What This Changes

The issue reports that memory-core daily recall snippets are stored with rich markdown normalization, while promotion rehydration compares only whitespace-normalized live snippets, causing `openclaw memory promote --apply` to drop qualified candidates silently.

## Best Possible Solution

Memory-core should make daily-ingestion stored snippets and promotion relocation comparison compatible through a shared or equivalent normalizer, keep live daily files as the promotion authority, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A source-level repro is clear: store a daily chunk as `Heading: item; item`, then compare it against live markdown containing `## Heading` plus raw `- item` lines; current main's equality/includes checks cannot match those normalized forms.

## Solution Assessment

Yes. Aligning the normalizers and adding focused observability is the narrow maintainable solution; falling back to stored candidates would make `applied` non-zero but bypass the documented live-source stale-snippet guard.
