## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion relocation compares only whitespace-collapsed raw source windows and silently drops candidates that cannot be rehydrated.

## What This Changes

The issue requests fixing memory-core short-term promotion apply rehydration so daily-ingested heading/list snippets relocate correctly and all-candidate rehydration failure emits a warning.

## Best Possible Solution

Memory-core should make daily-ingestion stored snippets and promotion relocation comparison compatible through a shared or equivalent normalizer, preserve live daily files as the promotion authority, and log one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily chunk stored as `Heading: item; item` can be compared against live markdown lines containing `## Heading` plus raw `- item` bullets; current main's equality/includes checks cannot match those normalized forms, and the issue includes dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding a focused warning is the narrow maintainable solution; falling back to the stored candidate would make `applied` non-zero but bypass the documented live-source stale-snippet guard.
