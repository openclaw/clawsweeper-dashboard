## Summary

Keep open. Current main still has the reported memory-core normalizer mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-collapsed live windows and silently drops candidates that fail rehydration.

## What This Changes

The issue reports that memory-core promotion apply drops selected dreaming candidates because stored daily snippets and relocation window snippets use incompatible normalization.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale guard, add a heading/list apply regression, and emit one clear warning when all selected candidates fail rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores rich daily snippets and compares them against whitespace-only live windows; a read-only probe of the reported shape produces no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates would restore counts but weaken the documented live daily-note re-read guard.
