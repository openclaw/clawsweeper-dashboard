## Summary

Keep open. Current main and the latest shipped v2026.5.3 tag still compare rich daily-ingestion snippets against whitespace-only live windows, then silently drop selected candidates when relocation fails.

## What This Changes

Fix memory-core promotion apply rehydration so stored daily snippets with heading/list normalization can be relocated against live daily notes before writing to MEMORY.md.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, keep the live-source stale guard, add a heading/list apply regression, and warn when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a read-only probe of that exact shape produces no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; the reported stored-candidate fallback is useful mitigation but weakens the documented live re-read guard.
