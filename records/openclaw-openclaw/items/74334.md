## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion rehydration relocates with whitespace-only normalization and silently drops every selected candidate that cannot be relocated.

## What This Changes

Issue requests fixing memory-core promotion apply rehydration when stored daily snippets use richer Markdown normalization than live-window relocation comparison.

## Best Possible Solution

Memory-core should use a shared or equivalent normalizer between daily-ingestion snippets and promotion relocation, preserve live daily files as the source of truth for apply, add a focused regression test for heading-prefixed list chunks, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. Source inspection and a static snippet comparison show that `Heading: item; item` daily chunks cannot match live Markdown windows with the current whitespace-only comparison; the issue also provides dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding all-dropped observability is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but bypass the documented live-source stale-snippet guard.
