## Summary

Keep open. Current main still has the memory-core normalizer mismatch: daily ingestion stores heading-prefixed/list-joined snippets, while promotion relocation compares with whitespace-only normalization and silently returns zero applied promotions when every selected candidate fails rehydration.

## What This Changes

The issue requests fixing memory-core promotion apply so candidates stored by daily ingestion can be rehydrated when their snippets use richer Markdown normalization than the live-window relocation comparison.

## Best Possible Solution

Memory-core should share or equivalently mirror the daily-ingestion snippet normalizer in promotion relocation, keep live daily files as the source of truth, add a heading-plus-list regression test, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily chunk stored as `Heading: item; item` from Markdown heading plus bullet lines cannot match the current whitespace-only live-window comparison, and the issue provides dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding all-dropped observability is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but bypass the documented live-source stale-snippet guard.
