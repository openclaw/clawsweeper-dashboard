## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-collapsed live windows and drops every candidate that cannot be relocated.

## What This Changes

Fix `memory promote --apply` so daily-ingested heading/list snippets can rehydrate from live daily notes instead of being silently skipped.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear diagnostic when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. The reported heading-plus-list daily note shape fails a read-only current-main probe of `normalizeSnippet`, `compareCandidateWindow`, and `relocateCandidateRange`, which returns null before apply can write anything.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but weaken the documented live daily-note re-read guard.
