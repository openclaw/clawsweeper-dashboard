## Summary

Keep open. Current main still has the mismatched memory-core normalization and silent rehydration drop path, and the latest reporter follow-up says the same pattern is still present in the v2026.5.7 release artifact.

## What This Changes

The issue reports that memory promotion apply rehydration drops qualified candidates because stored daily snippets and live relocation windows use incompatible normalization.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, preserve the live-source stale guard, add heading-plus-list apply regression coverage, and emit a clear warning or audit signal when all selected candidates fail rehydration.

## Reproduction Assessment

Yes, at source level: current main's exact relocation logic returns null for a heading-prefixed daily-ingestion chunk compared with the corresponding raw markdown list window. I did not run the full CLI against the reporter's recall store.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; the reporter's stored-candidate fallback is useful mitigation but weakens the documented live re-read guard.
