## Summary

Keep open. Current `main` and the latest inspected release still contain the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion rehydration searches with whitespace-only normalization and can silently drop every selected candidate.

## What This Changes

The issue asks memory-core promotion `--apply` to rehydrate daily-ingestion candidates with compatible snippet normalization instead of silently applying zero qualified promotions.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale guard, add a heading-plus-list apply regression, and emit one clear warning when all selected candidates fail rehydration.

## Reproduction Assessment

Yes, at source level. Current `main` still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a read-only probe of the reported shape returns no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates restores counts but weakens the documented live daily-note re-read guard.
