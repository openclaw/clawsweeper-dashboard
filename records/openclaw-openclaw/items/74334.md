## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-normalized snippets, while promotion relocation compares candidate snippets against live source windows with whitespace-only equality/containment and silently returns zero applied candidates when all rehydrations fail. The related open PR #74051 targets #71976 recall-gate/span behavior and does not supersede this normalizer bug.

## What This Changes

The issue requests that memory-core promotion rehydration handle daily-ingested stored snippets consistently with the daily-ingestion normalizer and warn when selected candidates are all dropped during apply.

## Best Possible Solution

Memory-core should use a shared or equivalent rich normalization path for daily-ingestion snippets and promotion relocation comparison, keep live daily files as the source of truth for apply, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily note with a non-generic markdown heading and bullet list is stored as a `Heading: item; item` snippet, but current apply rehydration compares it against whitespace-collapsed raw source windows such as `- item - item`, so the equality/containment checks cannot match; the issue also includes affected-host dry-run/apply output.

## Solution Assessment

Yes. Aligning the normalizers and adding focused all-dropped observability is the narrow maintainable fix; the stored-candidate fallback should not ship because it bypasses the live-source rehydration contract documented for promotion.
