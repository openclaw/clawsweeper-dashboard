## Summary

Keep open. Current main still has the reported memory-core normalization mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while apply-time relocation compares only whitespace-collapsed live windows and drops selected candidates that fail relocation.

## What This Changes

Fix memory-core promotion apply so daily-ingestion snippets with heading/list normalization can be relocated and promoted instead of silently yielding zero applied candidates.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, keep live-source validation, add a heading/list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; the reported snippet shape produces no relocation match under the current compare logic.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; the local stored-candidate fallback is useful evidence but would weaken the documented live re-read guard if used as the structural fix.
