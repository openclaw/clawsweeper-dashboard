The issue asks for memory-core promotion apply to rehydrate stored daily-ingestion snippets reliably instead of silently dropping qualified candidates when stored and live snippet normalization differ.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, preserve live-source stale guarding, add a heading/list apply regression, and warn when every selected candidate fails rehydration.
