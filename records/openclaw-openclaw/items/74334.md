## Summary

Keep open. Current `main` still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion rehydration relocates with whitespace-only window comparison and silently returns zero when every selected candidate drops.

## What This Changes

The issue requests fixing memory-core short-term promotion apply rehydration so daily-ingestion snippets can be relocated and promoted instead of being silently dropped.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, keep the live-source stale-snippet guard, add a heading-plus-list apply regression, and surface one clear diagnostic when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. The reported heading-plus-list daily note shape fails a read-only check against current `normalizeSnippet` plus equality/includes matching, and the source path then drops all selected candidates before apply writes anything.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but weaken the documented live daily-note re-read guard.
