## Summary

Keep open. Current main and the latest shipped tag still contain the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion rehydration compares only whitespace-collapsed live windows and silently drops candidates that cannot be relocated.

## What This Changes

The issue reports that memory-core `memory promote --apply` drops qualified daily-dreaming candidates because stored daily snippets and live relocation windows use incompatible normalization.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation comparison, preserve the live-source stale guard, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a no-write probe of the reported shape produces zero relocation matches.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates would restore counts but weaken the documented live daily-note stale guard.
