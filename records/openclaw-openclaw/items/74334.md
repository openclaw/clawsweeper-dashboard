## Summary

Keep open. Current main still has the mismatched daily-ingestion versus promotion-relocation snippet normalization, and the latest shipped v2026.5.7 source still carries the same relocation/drop behavior reported by the user.

## What This Changes

The issue reports that memory-core dreaming promotion drops qualified daily-ingestion candidates because stored heading/list-normalized snippets cannot be relocated against whitespace-only live-file windows.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, preserve the live-source stale/deleted guard, add a heading-plus-list apply regression, and warn when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A source-level current-main probe of the relocation helper shape returns `relocated: null` for a daily-ingestion heading/list snippet that should match the live markdown window; the reporter also provided live v2026.5.7 apply evidence.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; the stored-candidate fallback is useful as mitigation but weakens the documented live re-read guard.
