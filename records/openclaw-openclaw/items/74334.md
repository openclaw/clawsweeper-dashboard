## Summary

Keep open. Current main still has the memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion relocation compares only whitespace-collapsed live windows and silently drops candidates that cannot be rehydrated.

## What This Changes

Fix memory dreaming promotion so `--apply` can rehydrate daily-ingestion candidates whose stored snippets use richer markdown normalization.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, keep live daily files as the source of truth, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The source-level reproduction of a meaningful heading plus bullet-list daily chunk yields no relocation match under current main, and the reporter provided dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding all-dropped observability is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but bypass the documented live-source stale-snippet guard.

## Repair Work Prompt

Fix openclaw/openclaw#74334 in memory-core. The observable bug is that `openclaw memory promote --apply` can drop all selected daily-ingestion candidates because stored snippets use daily chunk normalization (`Heading: ...`, list markers stripped, list items joined with `; `, truncated) while `relocateCandidateRange` compares only whitespace-collapsed live file windows. Keep the live daily note as the source of truth; do not use the reporter's stored-candidate fallback as the main fix. Prefer a shared or equivalent normalizer for daily-ingestion snippets and relocation comparison, and add one clear warning when selected candidates exist but every candidate fails rehydration. Add focused regression coverage for a daily note with a meaningful markdown heading and bullet list whose stored candidate starts at the body lines. Do not change unrelated memory ranking policy or provider/channel surfaces. Add a user-facing `CHANGELOG.md` fix entry. Validate with targeted memory-core tests, targeted formatter check, and `pnpm check:changed` in Testbox before handoff.
