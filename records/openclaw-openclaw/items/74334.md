## Summary

Keep open. Current main still has the mismatched daily-ingestion versus relocation normalization path, still drops candidates when relocation returns null, and the reporter’s v2026.5.3 follow-up says the shipped artifact still carries the same failing apply behavior.

## What This Changes

Fix memory-core short-term promotion so apply rehydrates heading/list-normalized daily snippets instead of silently dropping qualified candidates.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization in promotion relocation, preserve the live daily-note stale guard, add a heading/list apply regression, and warn when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a focused read-only probe of that shape returns no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; the stored-candidate fallback is useful as mitigation but weakens the documented live re-read guard.

Current main stores daily-ingestion snippets with heading prefixes, list-marker stripping, semicolon-joined list chunks, and truncation, but promotion relocation compares them against whitespace-only live windows and drops candidates when relocation returns null. Implement the fix in the owner plugin, preferably by sharing or faithfully mirroring daily-ingestion snippet normalization for relocation comparison while preserving the live daily-note stale-source guard. Do not replace the fix with a blind fallback that trusts stale stored snippets unless a maintainer explicitly approves that policy. Add a user-facing CHANGELOG.md fix entry.
