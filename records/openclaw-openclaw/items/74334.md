## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion relocation compares only whitespace-collapsed source windows and then silently returns zero applied candidates when every selected candidate fails rehydration. Adjacent PR #74051 is related to recall gates and relocation width, but the provided context and current source show it does not align these normalizers.

## What This Changes

Fix memory-core short-term promotion so dreaming-ingested snippets use compatible normalization during live-source rehydration and no longer silently apply zero candidates.

## Best Possible Solution

Memory-core should make the daily-ingestion stored snippet form and promotion relocation comparison compatible through one shared normalization/reconstruction contract, preserve live daily files as the promotion authority, add regression coverage for heading-prefixed/list-joined daily chunks through apply, and emit one clear warning when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. The issue includes an end-to-end dry-run/apply reproduction from an affected host, and the current source can be reproduced with a heading-prefixed/list-joined stored snippet that fails all three relocation comparison checks against the raw markdown window.

## Solution Assessment

Yes. Aligning the normalizers or comparison contract is the narrow maintainable fix; falling back to the stored candidate would make apply non-zero but would bypass the current live-file stale-snippet protection documented for promotion.
