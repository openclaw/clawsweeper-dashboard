## Summary

Keep open. Current main still has the reported memory-core promotion mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while apply-time relocation compares only whitespace-collapsed live windows and drops every candidate that cannot be rehydrated.

## What This Changes

Fix memory-core promotion apply so daily-ingested heading/list-normalized snippets rehydrate from live daily notes instead of being silently dropped.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear diagnostic when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. The reported heading-plus-list shape fails a read-only comparison probe against current main, and the source path then drops null relocations before any write.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero while weakening the documented live daily-note re-read guard.
