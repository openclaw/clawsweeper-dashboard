## Summary

Keep open. Current main still has the reported memory-core apply-path mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares against whitespace-only live windows and silently drops candidates that cannot be relocated.

## What This Changes

Fix memory-core promotion apply so daily-ingested heading/list snippets can rehydrate from live daily notes instead of being silently dropped.

## Best Possible Solution

Memory-core should share or faithfully mirror the daily-ingestion snippet normalizer during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear diagnostic when all selected candidates fail rehydration.

## Reproduction Assessment

Yes, as a source-level current-main reproduction: the reported heading-plus-list shape fails the same normalize/equality/includes comparison, and the source then drops candidates before writing. I did not run a writable end-to-end apply flow in this read-only review.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; using the stored candidate as a fallback may restore counts but weakens the documented live daily-note reread guard.
