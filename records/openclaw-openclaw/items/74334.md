## Summary

Keep open. Current `main` still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares against whitespace-only live windows and drops every selected candidate that cannot be relocated.

## What This Changes

Fix memory-core promotion apply so daily-ingestion snippets normalized with headings and list chunks can be rehydrated from live daily notes, with a diagnostic when selected candidates all fail relocation.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The issue's heading-plus-list daily note shape fails a read-only check against current `normalizeSnippet` plus equality/includes matching, and the source path then drops all selected candidates before apply writes anything.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but weaken the documented live daily-note re-read guard.
