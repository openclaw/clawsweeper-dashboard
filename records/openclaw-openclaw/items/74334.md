## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion relocation compares candidates against live source windows using only whitespace normalization and silently returns zero applied promotions when all rehydrations fail. The related open PR #74051 targets #71976 recall gates and relocation span behavior, but it does not supersede this normalizer mismatch.

## What This Changes

Fix memory-core promotion apply so daily-ingestion candidates with heading-prefixed/list-joined stored snippets can rehydrate against live daily notes instead of silently applying zero promotions.

## Best Possible Solution

Memory-core should use a shared or equivalent rich normalizer for daily-ingestion snippets and promotion relocation comparison, keep live daily files as the source of truth for apply, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily chunk stored as `Heading: first item; second item` from markdown `## Heading` plus raw bullet lines cannot match current main's whitespace-only live-window comparison; the issue also includes dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding a focused all-dropped warning is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but bypass the documented live-source stale-snippet guard.
