## Summary

Keep open. Current main still has the reported memory-core normalizer mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion rehydration compares stored candidates against live file windows with whitespace-only normalization and silently returns zero applied promotions when all selected candidates fail relocation. The related open PR #74051 targets #71976 recall gates and relocation span behavior, but it does not supersede this normalizer mismatch.

## What This Changes

The issue requests fixing memory-core promotion rehydration so stored daily-ingestion snippets normalize compatibly with live-source relocation and selected candidates are not silently dropped during `memory promote --apply`.

## Best Possible Solution

Memory-core should use a shared or equivalent rich normalizer for daily-ingestion snippets and promotion relocation comparison, keep live daily files as the source of truth for apply, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. A daily chunk stored as `Heading: first item; second item` from Markdown heading plus raw bullet lines cannot match current main's whitespace-only live-window comparison; the issue also includes dry-run/apply output from an affected host.

## Solution Assessment

Yes. Aligning the normalizers and adding a focused all-dropped warning is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but bypass the documented live-source stale-snippet guard.
