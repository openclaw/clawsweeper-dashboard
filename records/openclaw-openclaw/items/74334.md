## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-joined snippets, while promotion relocation compares only whitespace-collapsed live windows and drops all selected candidates when rehydration fails.

## What This Changes

The issue requests fixing memory-core promotion apply so daily-ingestion snippets can be rehydrated and promoted instead of silently returning zero applied candidates.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during live-source relocation, add a heading-plus-list apply regression test, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The issue's markdown heading/list example fails the same whitespace-only equality/includes checks in a read-only Node probe, and current `applyShortTermPromotions` returns zero when every selected candidate fails rehydration.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would avoid zero counts but weaken stale-source protection.
