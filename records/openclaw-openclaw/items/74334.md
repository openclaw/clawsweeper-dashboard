## Summary

Keep open. Current main still has the reported memory-core normalization mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-normalized live windows and silently drops every selected candidate that fails rehydration.

## What This Changes

The issue asks for `memory promote --apply` to rehydrate daily-ingestion promotion candidates reliably when stored snippets were normalized differently from the live daily note window.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live daily-note re-read guard, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main stores daily snippets as heading-prefixed/list-joined chunks and compares them against whitespace-only live windows; a read-only probe of the reported heading-plus-bullets shape produced no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; blindly falling back to stored candidates would restore counts but weaken the documented live-source stale-snippet guard.
