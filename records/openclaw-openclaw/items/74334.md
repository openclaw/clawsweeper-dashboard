## Summary

Keep open. Current `main` still has the reported memory-core apply-path mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-normalized live windows and silently drops candidates that cannot be rehydrated.

## What This Changes

Fix memory-core promotion so daily-ingestion snippets rehydrate during `memory promote --apply` instead of being silently dropped.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, keep the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current `main` still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a read-only probe of the reported shape produces no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates would restore counts but weaken the documented live daily-note re-read guard.
