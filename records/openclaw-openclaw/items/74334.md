## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74334 in memory-core. Current `applyShortTermPromotions` rehydrates selected daily-ingestion candidates through `relocateCandidateRange`, but daily ingestion stores `buildDailyChunkSnippet` output with stripped headings/list markers, heading prefixes, `; ` list joins, and truncation while relocation only whitespace-normalizes live ranges; selected candidates can silently drop when relocation returns null. Implement a narrow fix that makes relocation compare against the same daily-ingestion normalized form or factors a shared helper, preserving the live-source stale/deleted guard. Do not blindly trust stored snippets as the primary fix. Add regression coverage for a meaningful markdown heading plus list bullets where apply appends a candidate from the live daily note. Add one warning/audit/log signal when every selected candidate fails rehydration. Check overlapping PR #74051/#71976 before editing. Add a CHANGELOG.md entry because this is user-facing.
