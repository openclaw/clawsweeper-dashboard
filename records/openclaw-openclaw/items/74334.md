## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-normalized live windows and drops every selected candidate that cannot be rehydrated.

## What This Changes

The issue asks for memory-core promotion apply to rehydrate daily-ingestion snippets successfully when stored snippets use richer heading/list normalization than the live relocation comparison.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live daily-note stale-snippet guard, add a heading-plus-list apply regression, and log one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; an in-memory probe of the reported shape produces no relocation match.

## Solution Assessment

Yes. Aligning the relocation comparison with the daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates would restore counts but weaken the documented live daily-note re-read guard.
