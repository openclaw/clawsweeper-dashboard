## Summary

Keep open. Current `main` still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares them against whitespace-only live windows and drops candidates when relocation returns null.

## What This Changes

The issue requests a fix for memory-core promotion apply runs silently dropping qualified daily-memory candidates when stored snippet normalization does not match live-source relocation normalization.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one clear diagnostic when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. Current source contains the exact normalizer mismatch, and a read-only probe of the reported heading-plus-list shape fails the same equality/includes comparison before the apply path drops null rehydrations.

## Solution Assessment

Yes. Aligning or sharing the normalizers is the narrow maintainable fix; falling back to stored candidates would restore counts but weaken the documented live daily-note re-read guard.
