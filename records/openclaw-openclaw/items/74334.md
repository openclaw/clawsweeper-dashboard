## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-normalized live windows and silently drops candidates that fail rehydration.

## What This Changes

The issue reports that `memory promote --apply` drops qualified dreaming candidates because stored daily-ingestion snippets and live relocation snippets use incompatible normalization.

## Best Possible Solution

Align relocation matching with daily-ingestion snippet normalization, preserve live-source stale/deleted protection, add a heading-plus-list apply regression, and surface a clear diagnostic when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; the reported shape fails the current comparison logic.

## Solution Assessment

Yes. Sharing or faithfully mirroring daily-ingestion normalization in relocation is the narrow maintainable fix; blindly falling back to stored candidates would restore counts but weaken the documented live re-read guard.
