## Summary

Keep open. Current main still has the source-level mismatch between daily-ingestion snippets and promotion rehydration, and the related open pull request at https://github.com/openclaw/openclaw/pull/74051 changes relocation span/tie-breaking but does not align the stored-snippet and live-window normalizers.

## What This Changes

The issue requests a memory-core fix so dreaming promotion apply can rehydrate daily-ingestion candidates whose stored snippets were heading/list normalized before writing to the recall store.

## Best Possible Solution

Align promotion relocation with daily-ingestion snippet normalization, preserve live-file stale/deleted protection, add a heading-plus-list apply regression, and emit one actionable warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main stores heading-prefixed, semicolon-joined daily chunks but compares them to whitespace-collapsed raw Markdown windows. A read-only probe of the current relocation logic returned `relocated: null` for that exact shape; I did not run a write-based CLI fixture.

## Solution Assessment

No, this should not be closed or handled by the stored-candidate fallback alone. The narrow maintainable fix is to share or faithfully mirror daily-ingestion normalization in relocation while keeping the live-source guard and focused regression coverage.
