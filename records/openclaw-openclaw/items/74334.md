## Summary

Keep open. Current main still has the memory-core mismatch described in the report: daily ingestion stores heading-prefixed, list-joined snippets, but promotion relocation compares against live daily-note windows using whitespace-only normalization and silently returns zero applied promotions when every selected candidate fails rehydration. The adjacent open PR #74051 targets #71976 recall gates and relocation span behavior, but the provided context does not show it closing or superseding this normalizer bug.

## What This Changes

The issue requests fixing memory-core promotion rehydration so daily-ingestion snippets and live-source relocation use compatible normalization and all selected candidates are not silently dropped during `memory promote --apply`.

## Best Possible Solution

Memory-core should compare daily-ingestion candidates and live daily-note windows through a shared or equivalent rich normalizer, keep live file rehydration as the source of truth for stale/deleted snippets, and emit one clear warning when a non-empty selected set is entirely dropped during rehydration.

## Reproduction Assessment

Yes. A daily note with a meaningful markdown heading followed by bullets is stored as `Heading: item; item` with a body-only range, while current main only whitespace-collapses live windows, so the stored candidate cannot match the source window shape. The issue also includes dry-run/apply output from an affected v2026.4.24 host.

## Solution Assessment

Yes, the suggested normalizer alignment plus an all-dropped warning is the narrow maintainable direction. Falling back to the stored candidate would make counts non-zero but would bypass the documented live-source stale-snippet guard.
