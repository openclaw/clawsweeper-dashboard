## Summary

Keep open. Current `main` still has the memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares against whitespace-only live windows and drops candidates that fail rehydration.

## What This Changes

Fix memory-core promotion apply so daily-ingestion snippets normalized with headings and list chunks can be rehydrated from live daily notes, with a diagnostic when selected candidates all fail relocation.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale-snippet guard, add a heading-plus-list apply regression, and emit one bounded all-candidates-failed diagnostic.

## Reproduction Assessment

Yes. Source inspection plus a read-only Node probe reproduce the reported heading-plus-list mismatch against the current comparison shape; I did not run the full CLI apply path in this read-only review.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; the local workaround of returning the stored candidate would restore counts but weakens the documented live-source re-read guard.
