## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-joined snippets, while promotion relocation compares only whitespace-collapsed live source windows and silently drops candidates that cannot be rehydrated.

## What This Changes

Fix memory-core promotion so daily-ingestion stored snippets can be rehydrated against live daily notes, and warn when all selected candidates fail relocation.

## Best Possible Solution

Memory-core should use a shared or equivalent normalization path for daily-ingestion stored snippets and promotion relocation, keep live files as the promotion authority, add regression coverage for headed list chunks, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The repo currently has enough source-level evidence to reproduce the mismatch with a headed markdown list stored as `Heading: item; item` and compared against raw live `## Heading`/`- item` lines; the issue also includes affected-host dry-run/apply output.

## Solution Assessment

Yes. Aligning the normalizers and adding a focused all-dropped warning is the narrow maintainable fix; falling back to the stored candidate should not be the primary solution because it bypasses live-source revalidation.
