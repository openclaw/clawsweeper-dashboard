## Summary

Keep open. Current main still has the memory-core mismatch: daily ingestion stores heading-prefixed/list-joined snippets, while promotion relocation compares them to whitespace-only live windows and still drops all selected candidates with `applied: 0` when none rehydrate.

## What This Changes

Fix memory-core promotion apply so ranked short-term recall candidates stored by daily dreaming ingestion can be rehydrated from live daily notes instead of silently producing zero applied promotions.

## Best Possible Solution

Memory-core should share or faithfully mirror daily-ingestion snippet normalization during live-source relocation, preserve the stale-source guard, add a heading-plus-list apply regression test, and emit one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The issue's heading-plus-bullet daily note shape can be checked directly against current `normalizeSnippet` plus equality/includes matching, and it fails before `applyShortTermPromotions` silently returns zero when all selected candidates drop.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; falling back to stored candidates would make counts non-zero but weaken the documented live daily-note re-read guard.
