## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion relocation compares only whitespace-collapsed live windows and silently drops selected candidates that fail rehydration.

## What This Changes

Make `memory promote --apply` rehydrate daily-ingested, heading-prefixed/list-normalized promotion candidates instead of silently applying zero.

## Best Possible Solution

Share or faithfully mirror daily-ingestion snippet normalization during relocation, preserve the live-source stale guard, add a heading-plus-list apply regression, and log one clear warning when every selected candidate fails rehydration.

## Reproduction Assessment

Yes, at source level. Current main still stores heading/list-normalized daily snippets and compares them against whitespace-only live windows; a read-only probe of the reported shape produces no relocation match.

## Solution Assessment

Yes. Aligning relocation comparison with daily-ingestion normalization is the narrow maintainable fix; falling back blindly to stored candidates restores counts but weakens the documented live daily-note re-read guard.
