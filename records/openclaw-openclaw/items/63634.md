## Summary

This PR still needs changes before merge: the central typeless MCP schema fix is useful and well evidenced, but the current patch rebuilds SDK tool metadata inside the pagination loop, which can drop call-time guards for earlier accepted pages while those tools remain exposed.

## What This Changes

The PR changes session bundle MCP tool cataloging to use an SDK-derived relaxed tools/list schema, normalize missing root inputSchema.type, skip explicit non-object roots, call SDK metadata caching, and add runtime tests.

## Best Possible Solution

- **[P1] Cache MCP metadata after collecting all pages:** `src/agents/agent-bundle-mcp-runtime.ts:327`
  - body: `cacheToolMetadata()` clears all cached output validators and task-tool sets each time it runs in the pinned SDK, but this call runs once per `tools/list` page while the final catalog concatenates every page.
