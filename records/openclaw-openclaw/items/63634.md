## Summary

Keep open: this is a meaningful fix for a live bundle MCP cataloging regression, but the current patch can drop SDK call-time output-schema and task guards for cataloged tools after pagination or skipped invalid entries.

## Best Possible Solution

Keep the single-response typeless-schema compatibility path, but rebuild SDK metadata cache once from the surviving accepted tools across all pages and add a paginated skipped-tool regression before owner acceptance.

## Solution Assessment

No as-is. The single-response SDK-derived schema is the right direction, but metadata caching needs to happen after pagination and filtering over surviving tools so skipped invalid entries cannot drop callTool guards.
