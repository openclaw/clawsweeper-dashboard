## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve SDK tool metadata after lenient listing:** `src/agents/pi-bundle-mcp-runtime.ts:205-208`
  - body: SDK 1.29.0 `Client.listTools()` does more than validate the response: after `request()` it calls `cacheToolMetadata(result.tools)`, which preloads output-schema validators and required task-tool metadata used later by `callTool()`. This raw `client.request()` path never performs that caching, so MCP tools that define `outputSchema` can stop enforcing structured output and tools marked as task-required can be called through the wrong API. Please cache the normalized SDK-compatible tool metadata after lenient listing before returning the catalog.
  - confidence: 0.91
