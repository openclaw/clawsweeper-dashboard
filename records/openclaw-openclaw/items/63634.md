## Summary

Keep open. The reported bug is valid and the PR is aimed at the right bundled MCP catalog path, but current PR head still bypasses SDK `listTools()` metadata caching and the contributor proof is test/check output only, so it should not merge yet.

## What This Changes

The PR changes bundled MCP tool discovery to parse `tools/list` with a lenient schema, normalize missing root `inputSchema.type`, skip explicit non-object schemas, update tests, and add a changelog entry.

## Best Possible Solution

Keep the lenient `tools/list` parser and malformed-root filtering, but preserve SDK tool metadata caching for accepted normalized tools and require redacted live MCP/provider proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main calls SDK `client.listTools()` for bundled MCP cataloging, and SDK v1.29.0 requires root `inputSchema.type` to be literal `object` while the linked Playwright/Puppeteer report shows the matching failure.

## Solution Assessment

No. The lenient listing direction is right, but raw `client.request()` bypasses the SDK `listTools()` metadata cache that `callTool()` uses for required task tools and `outputSchema` validation.
