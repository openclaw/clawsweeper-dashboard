## Summary

Keep open: the PR targets a real source-reproducible MCP regression, but the current diff still bypasses SDK tool metadata caching and the supplied proof is only local tests/checks, so it needs revision plus real behavior proof before merge.

## What This Changes

This PR changes bundled MCP tool listing to parse typeless MCP `inputSchema` objects leniently, normalize accepted catalog tools, skip explicit non-object root schemas, add regression tests, and update the changelog.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, but preserve SDK `listTools()` metadata behavior and require redacted live/Testbox MCP provider proof before merge.

## Reproduction Assessment

Yes, from source and dependency contracts. Current main calls the SDK `Client.listTools()` path while `@modelcontextprotocol/sdk@1.29.0` requires `inputSchema.type` to be literal `object`, matching the Playwright/Puppeteer MCP failure described in https://github.com/openclaw/openclaw/issues/63602.

## Solution Assessment

No, not as-is. The lenient parser and bad-schema filtering are the right narrow direction, but the raw request path must also preserve the SDK metadata cache used by later tool calls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve SDK tool metadata after lenient listing:** `src/agents/pi-bundle-mcp-runtime.ts:205-208`
