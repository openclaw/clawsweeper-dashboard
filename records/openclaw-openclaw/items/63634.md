## Summary

Keep open: the MCP regression is valid and the PR is aimed at the right runtime path, but latest head still bypasses SDK `listTools()` metadata caching and has only test/check proof rather than a real MCP/provider run.

## What This Changes

The PR changes bundle MCP tool listing to parse typeless input schemas with a lenient `tools/list` schema, normalize and filter catalog tools, add regression coverage, and add a changelog entry.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, add an equivalent metadata-cache preservation path for lenient-listed tools, and verify it with redacted real MCP provider proof.

## Reproduction Assessment

Yes, source inspection plus the SDK contract gives a high-confidence path: current main calls `Client.listTools()`, and SDK v1.29.0 rejects tools whose root `inputSchema.type` is missing. This matches the linked Playwright/Puppeteer MCP loading failure.

## Solution Assessment

No, not as-is. The lenient parser and bad-root filtering are the right narrow direction, but raw `client.request()` must preserve the SDK metadata side effect that current `listTools()` provides.
