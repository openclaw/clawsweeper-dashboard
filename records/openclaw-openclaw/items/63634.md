## Summary

Keep open: this is a plausible fix for a real bundle MCP cataloging regression, but the current PR still bypasses MCP SDK metadata caching and only supplies test-output proof rather than a real MCP/provider run.

## What This Changes

The PR changes bundle MCP tool cataloging to leniently parse `tools/list`, backfill missing root `inputSchema.type`, skip explicit non-object root schemas, add focused tests, and update the changelog.

## Best Possible Solution

Keep the lenient cataloging and malformed-tool filtering approach, but restore SDK-equivalent metadata behavior before `callTool()` and require redacted real MCP/provider proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main calls SDK `Client.listTools()`, and SDK v1.29.0 requires root `inputSchema.type` to be literal `object`. I did not run a live MCP server in this read-only review.

## Solution Assessment

No, not as written. The lenient schema and malformed-tool filtering are the right direction, but the raw request path must preserve SDK metadata behavior before this is the best fix.
