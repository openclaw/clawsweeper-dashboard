## Summary

Keep open: the PR fixes the typeless MCP schema direction, but current head still bypasses SDK tool metadata caching and only provides test/mock proof, so it is not ready to merge.

## What This Changes

The PR changes bundled MCP tool discovery to parse `tools/list` with a lenient schema, normalize/filter root `inputSchema`, update tool counts/tests, and add a changelog entry.

## Best Possible Solution

Keep the lenient MCP listing and malformed-root filtering, but preserve SDK metadata caching for accepted normalized tools before `callTool()` and add real MCP server proof.

## Reproduction Assessment

Yes, from source inspection. Current main calls SDK `client.listTools()` and SDK 1.29.0 rejects typeless root `inputSchema`, while the PR regression is source-reproducible because raw `client.request()` skips `cacheToolMetadata()` before later `callTool()` use.

## Solution Assessment

No. The narrow maintainable fix is lenient `tools/list` parsing plus normalization/filtering while still invoking or reproducing SDK metadata caching for accepted tools.
