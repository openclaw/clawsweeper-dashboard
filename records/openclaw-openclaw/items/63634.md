## Summary

Keep open. The reported MCP listing bug is valid and the PR fixes much of the catalog path, but the current patch still bypasses MCP SDK `listTools()` metadata caching and the supplied proof is local tests/checks only, so it is not merge-ready.

## What This Changes

The PR replaces bundle MCP tool listing with a lenient `tools/list` parser, normalizes typeless `inputSchema` values, filters explicit non-object schemas, adds tests, and updates the changelog.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, but preserve SDK-equivalent tool metadata caching and require redacted live or Testbox MCP/provider proof before merge.

## Reproduction Assessment

Yes, from source and dependency contracts. Current `main` uses SDK `Client.listTools()`, while SDK v1.29.0 validates root `inputSchema.type` as literal `object`, matching the linked MCP server failure mode.

## Solution Assessment

No, not as-is. The lenient parser and malformed-root filtering are the right narrow direction, but the raw request path must also preserve SDK `listTools()` metadata behavior.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve SDK tool metadata after lenient listing:** `src/agents/pi-bundle-mcp-runtime.ts:205-208`
