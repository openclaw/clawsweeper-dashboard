The PR changes bundled MCP client tool cataloging to accept tools whose root inputSchema omits type, normalize accepted schemas to object roots, skip explicit non-object roots, invoke SDK tool-metadata caching, and add regression tests.

## Best Possible Solution

Land a rebased bundled-MCP consuming-path fix that accepts typeless object roots while caching SDK metadata once for the final accepted paginated catalog and preserving current main's session concurrency/lifecycle changes.

## Reproduction Assessment

## Solution Assessment
