## Summary

Keep open: this is a focused fix for a source-reproducible bundled MCP schema failure, but the current patch still bypasses SDK tool-metadata caching and only provides test/check output rather than real MCP/provider behavior proof.

## What This Changes

The PR changes bundled MCP tool discovery to use lenient tools/list parsing, normalize and filter root input schemas, update catalog counts/tests, and add a changelog entry.

## Best Possible Solution

Keep the lenient tools/list approach and malformed-root filtering, but preserve SDK tool metadata caching for accepted normalized tools and require redacted live MCP/provider proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main calls SDK client.listTools(), the pinned SDK requires root inputSchema.type to be literal object, and the linked report shows Playwright/Puppeteer MCP servers failing with that validation error.

## Solution Assessment

No. The direction is right, but the implementation must keep the SDK listTools metadata side effect or an equivalent cacheToolMetadata call for accepted tools before it is safe to merge.
