## Summary

Keep open. The consuming-path fix is still useful and current main still uses the strict MCP SDK list path, but this branch remains blocked because paginated `tools/list` responses can drop SDK call-time validators for earlier accepted tools while those tools stay in the final catalog.

## What This Changes

The PR replaces bundled MCP tool cataloging with a relaxed SDK-derived schema, backfills missing root object types, filters explicit non-object roots, preserves SDK metadata caching, and adds focused runtime tests.

## Best Possible Solution

Keep the typeless-root compatibility fix, rebuild SDK metadata once after pagination for the final accepted tool set, add a paginated output-schema regression, then get bundle MCP owner review or fold the fix into https://github.com/openclaw/openclaw/pull/89686.
