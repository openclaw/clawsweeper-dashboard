Full review comments:

- **[P1] Cache MCP metadata after collecting all pages:** `src/agents/agent-bundle-mcp-runtime.ts:327`
  - body: `cacheToolMetadata()` clears the SDK's cached validators and task-tool sets every time it runs, but this call runs once per `tools/list` page while `tools` keeps every previous page. If an accepted typeless tool with an `outputSchema` appears on an earlier page and a later page follows, the final cache contains only the later page, so `callTool()` can run without the SDK output-schema/task guard this PR is trying to preserve. Accumulate the validated tools and rebuild the cache once after pagination, with a regression covering an output-schema tool before `nextCursor`.
  - confidence: 0.9
