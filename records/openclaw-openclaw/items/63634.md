## Summary

Keep open: this targets a real, source-reproducible MCP loading bug, but the branch still bypasses SDK tool-metadata caching and has only test/check output instead of real MCP/provider behavior proof.

## What This Changes

The PR replaces bundle MCP `client.listTools()` with a lenient `tools/list` request, normalizes typeless input schemas, skips explicit non-object root schemas, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the lenient listing and malformed-root filtering direction, but preserve SDK-equivalent metadata caching for accepted tools and require redacted live or Testbox MCP/provider proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main calls SDK `Client.listTools()`, and SDK 1.29.0 requires root `inputSchema.type === "object"`, matching the linked MCP loading failure for typeless tools. I did not run a live MCP server in this read-only review.

## Solution Assessment

No, not as written: the lenient parser and malformed-root filtering are the right narrow direction, but raw `client.request()` must preserve SDK `listTools()` metadata caching and the PR still needs real MCP/provider proof.
