## Summary

Keep this PR open: it targets a real bundle MCP consuming-path regression, but the current patch still clears the SDK metadata cache once per tools/list page, so earlier paginated tools can lose output-schema validation and required-task guards before callTool(). The branch is also currently conflicting, and the overlapping maintainer draft needs owner direction rather than cleanup closure.

## What This Changes

This PR changes bundle MCP client tool listing to accept root-typeless inputSchema values with an SDK-derived relaxed schema, skip explicit non-object roots, invoke SDK metadata caching, and add focused regression tests.

## Best Possible Solution

Land one reconciled bundle MCP catalog fix that accepts typeless object roots while caching the final accepted paginated tool set and preserving current-main runtime behavior.

## Solution Assessment
