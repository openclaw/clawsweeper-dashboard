## Summary

Keep open. The PR targets a real bundled MCP discovery failure that current main still has, but the latest head bypasses SDK metadata caching that `Client.callTool()` relies on for output-schema validation and required task-tool guarding.

## What This Changes

The PR changes bundled MCP tool discovery to use a lenient `tools/list` schema, backfill missing root object types, skip explicit non-object schemas, add regression tests, and add a changelog entry.

## Best Possible Solution

Land a reworked version that keeps lenient MCP tool listing and malformed-root isolation while preserving the SDK `listTools()` metadata semantics before any later `callTool()` invocation.

## Reproduction Assessment

Yes. Current main can be reproduced with a mock or Playwright MCP `tools/list` response whose root `inputSchema.type` is missing, and the PR regression is reproducible with a listed tool that declares `outputSchema` or required task execution.

## Solution Assessment

No. The proposed direction is close, but replacing `listTools()` with raw `request()` is not complete unless it also caches or reimplements the SDK metadata behavior that `callTool()` depends on.
