## What This Changes

The branch replaces bundle MCP `client.listTools()` cataloging with a lenient `tools/list` request, normalizes typeless input schemas, skips explicit non-object root schemas, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, but preserve the SDK metadata behavior for output schemas and task-required tools, then require redacted real MCP/provider proof before merge.

## Reproduction Assessment

Yes, from source and dependency contracts. Current main uses SDK `Client.listTools()`, and the pinned SDK requires root `inputSchema.type` to be literal `object`, matching the linked Playwright/Puppeteer MCP failure path.

## Solution Assessment

No, not as-is. The lenient parser and non-object filtering are the right narrow direction, but the implementation must preserve the SDK `listTools()` metadata behavior before merge.
