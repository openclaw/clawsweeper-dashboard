## Summary

Keep open: current main and the latest release still use strict MCP SDK tool listing on the bundled MCP consuming path, so the reported bug remains, but this PR is not merge-ready because the relaxed paginated path can clear SDK call-time guards for earlier accepted tools.

The branch changes bundled MCP client tool cataloging to use an SDK-derived relaxed tools/list schema, backfill missing root inputSchema.type, filter explicit non-object roots, call SDK metadata caching, and add regression tests.
