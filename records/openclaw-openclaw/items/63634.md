## Summary

This PR should stay open: the bug fix direction is useful, but the current patch is not merge-ready because the lenient raw `tools/list` path bypasses SDK metadata caching, and the supplied proof is only local test/type/check output.

## What This Changes

The PR changes bundle MCP tool listing to accept typeless MCP input schemas, normalize accepted schemas, skip explicit non-object root schemas, add regression tests, and add a changelog entry.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, but preserve SDK-equivalent metadata caching and require redacted live or Testbox MCP/provider proof before merge.

## Reproduction Assessment

Yes, from source and dependency contracts: current main calls SDK `Client.listTools()`, and SDK 1.29.0 rejects tool schemas whose root `inputSchema.type` is missing. I did not run a live MCP server in this read-only review, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No, not as written. The lenient schema approach is the right narrow direction, but it needs SDK-equivalent metadata caching before it is the best maintainable fix.
