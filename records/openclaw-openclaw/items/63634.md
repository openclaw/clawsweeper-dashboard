## Summary

Keep this PR open: the reported MCP loading bug is real and the patch is moving in the right direction, but current head still bypasses SDK `listTools()` metadata caching and only provides unit/check output rather than real MCP/provider behavior proof.

## What This Changes

The PR replaces bundle MCP tool listing with a lenient `tools/list` schema, normalizes typeless input schemas, skips explicit non-object root schemas, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the lenient typeless-schema catalog fix and malformed-root filtering, but preserve SDK tool metadata caching and require redacted live or Testbox MCP/provider proof before merge.

## Reproduction Assessment

Yes, from source and dependency contracts. Current main uses SDK `Client.listTools()`, while the pinned SDK requires root `inputSchema.type` to be literal `object`, matching the linked Playwright/Puppeteer MCP failure.

## Solution Assessment

No, not as-is. The lenient parser and non-object filtering are the right narrow direction, but the raw request path must also preserve SDK `listTools()` metadata behavior.
