## Best Possible Solution

Land a maintainer-reviewed sandbox-browser VNC change that preserves observer auth/security semantics and proves UTF-8 paste plus existing noVNC access modes in Docker.

## Solution Assessment

Unclear: replacing x11vnc with TigerVNC is plausible, but the closed attempt showed security-sensitive auth and localhost-binding pitfalls, so maintainers should choose the exact VNC replacement shape before automation proceeds.
