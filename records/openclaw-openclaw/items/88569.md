## What This Changes

Request support for opt-in Telegram Guest Mode so allowed users can @-mention a Telegram bot from chats where it is not a member and receive a one-shot reply via `answerGuestQuery`.

## Best Possible Solution

Review and land or deliberately reject https://github.com/openclaw/openclaw/pull/83632 as the narrow opt-in Guest Mode path, while leaving broader bot-to-bot loop-safety and peer-routing policy in https://github.com/openclaw/openclaw/issues/79077.

## Reproduction Assessment

Yes as a source-level feature gap, not as a broken existing-behavior bug: current main requests the update type through grammY defaults but has no `guest_message` handler, no Guest Mode config, and no `answerGuestQuery` delivery path. A live end-to-end reproduction would require Telegram Guest Mode credentials and BotFather setup.

## Solution Assessment

Yes for the implementation direction: the Telegram plugin is the right owner boundary and `answerGuestQuery` is the required API. The exact account-level authorization, route binding, and isolated-session contract should be settled in the active PR review rather than by opening a duplicate fix PR.
