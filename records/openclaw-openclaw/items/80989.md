## Summary

Keep open. Current main still has only config-level `streaming.preview.toolProgress`, no `/progress` command, and no session-scoped tool-progress override; the closed implementation attempt at https://github.com/openclaw/openclaw/pull/81009 did not land. The broader streaming-command request at https://github.com/openclaw/openclaw/issues/74077 is related, but it does not fully supersede this narrower request to keep text preview streaming while toggling only tool-progress lines.

## What This Changes

The issue requests a session-scoped `/progress` command to show or hide tool-call progress in streaming previews without editing config or restarting the Gateway.

## Best Possible Solution

Define the command scope first, then add a session-scoped tool-progress override that layers over channel config and is honored consistently by supported preview-rendering paths, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested `/progress` command and session-scoped tool-progress override.
