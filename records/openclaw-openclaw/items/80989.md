## Summary

Keep open. Current main has the channel config and Telegram preview plumbing for tool-progress visibility, but it does not have a `/progress` command, a command handler, or a session-level tool-progress override; this is a new feature/API decision rather than stale cleanup.

## What This Changes

Request to add a `/progress` chat/native command that toggles tool-progress visibility per session without editing config or restarting the Gateway.

## Best Possible Solution

Define a shared session-level tool-progress visibility override, expose it through `/progress` and Telegram native buttons, and have preview-streaming-capable channels layer that override over the existing channel config resolver.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested command/session override while the documented control remains config-only.

## Solution Assessment

Unclear until maintainers choose the override scope and how it relates to the broader `/stream` command. A shared resolver-backed session override is safer than a Telegram-only toggle or a command that rewrites gateway config.
