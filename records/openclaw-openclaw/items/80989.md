## Summary

Keep open. Current main still only supports config-level `streaming.preview.toolProgress`; there is no `/progress` command or session-scoped tool-progress override. The closest canonical issue covers broader `/stream` mode switching but does not fully cover this exact tool-progress visibility toggle, and the linked implementation PR is closed unmerged.

## What This Changes

Request a session-scoped `/progress` command that shows, enables, or disables tool-call progress in streaming previews without editing config or restarting the Gateway.

## Best Possible Solution

Fold the tool-progress visibility toggle into the maintainer-approved streaming command design, or explicitly scope a Telegram `/progress` command, then add a session override that every exposed surface reports and honors consistently.
