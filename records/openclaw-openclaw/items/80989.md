## Summary

Keep open. Current main and the latest shipped tag still only expose `streaming.preview.toolProgress` as config-driven behavior; there is no `/progress` command, session-scoped tool-progress override, or Telegram dispatch layer that reads such an override. The closed implementation attempt at https://github.com/openclaw/openclaw/pull/81009 did not merge, and the broader open streaming-mode request at https://github.com/openclaw/openclaw/issues/74077 is adjacent rather than a complete duplicate of this tool-progress toggle.

## What This Changes

Requests a session-scoped `/progress [on|off]` command to show or hide tool-call progress in streaming previews without editing config or restarting the Gateway.

## Best Possible Solution

Define the command scope first, then add a session-scoped tool-progress override that layers over channel config and is honored by each exposed preview-rendering surface with docs and focused coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested `/progress` command and session override.

## Solution Assessment

Unclear. The requested UX is plausible, but the maintainable implementation depends on a maintainer decision about shared versus Telegram-only command scope and its relationship to https://github.com/openclaw/openclaw/issues/74077.
