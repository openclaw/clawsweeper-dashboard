## Summary

Keep open. Current main still has config-level tool-progress controls but no `/progress` command or session-scoped override, the exact implementation PR was closed unmerged, and the broader streaming-command issue is adjacent rather than a proven replacement for this narrower tool-progress toggle.

## What This Changes

The issue requests a session-scoped `/progress` command to show, enable, or disable tool-call progress lines in streaming previews without editing config or restarting the Gateway.

## Best Possible Solution

Define the command scope and precedence first, then add a session-scoped tool-progress override that every exposed surface either honors consistently or rejects clearly, with docs, regression coverage, and live Telegram proof if Telegram visible behavior changes.
