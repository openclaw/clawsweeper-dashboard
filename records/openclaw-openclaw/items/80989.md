## Summary

Keep open. This is a recent feature request for a new session-scoped command; current main already has config-level Telegram/tool-progress controls and session-scoped command machinery, but it does not yet expose a `/progress` override, and adding one is a product/API decision rather than conservative cleanup.

## What This Changes

Add a `/progress` native/chat command that shows or toggles streaming preview tool-call progress per conversation session.

## Best Possible Solution

If accepted, add a session-scoped tool-progress visibility override in the shared command/session path and have Telegram preview rendering honor it, with docs and Telegram-focused coverage while keeping leak fixes tracked separately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main has config-level `streaming.preview.toolProgress` handling but no `/progress` command or session override.

## Solution Assessment

Unclear. The requested UX is plausible, but maintainers should first decide whether the command belongs in the shared slash-command catalog or only in Telegram native command handling, and how it should interact with `/verbose` and the linked leak bugs.
