## Summary

Keep open: current main still has config-level `streaming.preview.toolProgress` controls but no `/progress` command or session-scoped tool-progress override, and the broader open streaming-command request plus the closed unmerged implementation PR do not terminally resolve this narrower UX request.

## What This Changes

Add a `/progress` native/inline command that shows or toggles per-session streaming-preview tool-progress visibility without editing config or restarting the Gateway.

## Best Possible Solution

If accepted, define the command scope first, then add a session-scoped tool-progress visibility override that every exposed runtime surface honors, with docs and Telegram-focused coverage while keeping leak/default-policy bugs separate.

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main only supports config-level `streaming.preview.toolProgress` and has no `/progress` command or session override.

Unclear. The requested UX is plausible, but maintainers should first decide whether the command is Telegram-only, shared across draft/progress channels, or folded into the broader preview-streaming command design.
