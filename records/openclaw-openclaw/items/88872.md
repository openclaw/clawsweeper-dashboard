## Summary

Keep this PR open: current main and the latest release still lack first-class child-agent attribution for spawned subagent task rows, while this branch is a focused writer-side fix with sufficient proof and no blocking review findings.

## What This Changes

The branch passes the resolved subagent target agent id through subagent run registration, lets task creation infer agent_id from childSessionKey, and adds regression coverage for registry filtering and SQLite persistence.

## Best Possible Solution

Land the narrow attribution fix after maintainer review, preserving parent owner/requester fields while storing the executing child agent in task_runs.agent_id for new spawned subagent rows.

## Reproduction Assessment

Yes, at source level. Current main builds childSessionKey from targetAgentId but registers the task without targetAgentId, and createTaskRecord falls back to owner/requester ids; I did not run a write-producing current-main repro in this read-only review.

## Solution Assessment

Yes. Passing targetAgentId through the existing subagent registration seam plus childSessionKey inference in task creation fixes the writer side; plugin reader parsing should remain a workaround, not the canonical persisted attribution path.
