## Summary

Keep open: this PR is still relevant because current main does not implement `agents.defaults.skipOptionalBootstrapFiles`, but the current patch is not merge-ready. It only threads the setting through one bootstrap entrypoint, leaves config contract/docs/generated schema surfaces stale, accepts unsupported filenames silently, and has an identified legacy-bootstrap regression around skipped existing `USER.md`/`IDENTITY.md` files.

## What This Changes

The PR adds an `agents.defaults.skipOptionalBootstrapFiles` setting, threads it into the agent-command workspace bootstrap path, and gates creation of optional workspace bootstrap files in `ensureAgentWorkspace`.

## Best Possible Solution

Keep the PR open and update it into a complete config-contract change: validate the option as the four supported optional filenames, apply it consistently at every workspace-bootstrap entrypoint, preserve current legacy setup detection semantics, regenerate config/help/docs surfaces, add focused tests, and include the required user-facing changelog/docs updates.

## Solution Assessment
