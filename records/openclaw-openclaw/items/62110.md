## Summary

Keep open: the branch addresses the earlier config-contract feedback, but it still leaves a correctness gap in sandbox workspace seeding. Seeded optional bootstrap files are copied before the new skip list is enforced, so sandboxed sessions can recreate files the config asks OpenClaw to omit.

## What This Changes

The PR adds `agents.defaults.skipOptionalBootstrapFiles` across workspace bootstrap, config schema/types, generated config metadata, docs, callers, changelog, and focused tests.

## Best Possible Solution

Apply the optional-bootstrap skip policy at every writer of workspace bootstrap files, including sandbox seed copies, while preserving enum validation, required-file behavior, generated config metadata, docs, and the existing changelog entry.

## Reproduction Assessment

Yes. On the PR head, seed a workspace with `SOUL.md` or `USER.md`, then call `ensureSandboxWorkspace(sandbox, seed, false, ["SOUL.md", "USER.md"])`; the seed loop copies those files before the skip list reaches `ensureAgentWorkspace`.

## Solution Assessment

No for the PR as currently written. The direct workspace and schema approach is maintainable, but the sandbox seed loop must filter skipped optional filenames before writing them into the sandbox.
