## Summary

Keep open: current main does not implement this config option, and the PR is useful, but the latest head still has a blocking behavior gap. Seeded sandbox workspaces can copy optional bootstrap files before the new skip list is applied, so the PR does not consistently honor its own skip contract.

## What This Changes

The PR adds `agents.defaults.skipOptionalBootstrapFiles` to config schema/types/docs/generated metadata, threads it through workspace bootstrap call sites, adds tests and a changelog entry, and includes incidental Brave package, Telegram RTT driver, and plugin-loader edits.

## Best Possible Solution

Honor the optional-bootstrap skip policy at every writer of workspace bootstrap files, including sandbox seed copies, while preserving enum validation, required-file behavior, generated config metadata, docs, and the changelog entry.

## Reproduction Assessment

Yes. On the PR head, seed a workspace with `SOUL.md` or `USER.md`, then call `ensureSandboxWorkspace(sandbox, seed, false, ["SOUL.md", "USER.md"])`; the seed loop copies those files before the skip list reaches `ensureAgentWorkspace`.

## Solution Assessment

No for the PR as currently written. The direct workspace/config approach is maintainable, but the sandbox seed loop must filter skipped optional filenames before writing into the sandbox.
