## What This Changes

This PR adds a Control UI token-resolution helper, wires the Overview WebSocket URL input to preserve or reload gateway-scoped session tokens, extends node/browser UI regression coverage, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent scoped Control UI fix that preserves tokens for the same normalized gateway endpoint, reloads or clears tokens when switching endpoints, keeps regression coverage, and preserves changelog attribution for the canonical report and source PR.

## Reproduction Assessment

Yes. On current main, enter a Gateway Token in Overview > Gateway Access, then edit the WebSocket URL to a non-identical same-normalized value such as adding a trailing slash; the handler at ui/src/ui/views/overview.ts:271 clears the token immediately.

## Solution Assessment

Yes. The normalized-scope helper is the narrowest maintainable solution because it avoids carrying a gateway-scoped credential to a different endpoint while preserving harmless same-endpoint edits.
