## Summary

Keep open. Current main and the latest tagged release still clear the Gateway Token on any non-identical WebSocket URL edit, while this PR is a narrow Control UI fix that preserves tokens for same-normalized endpoints and reloads or clears endpoint-scoped tokens when the endpoint actually changes.

## What This Changes

This PR adds a Control UI storage helper for Gateway Access token resolution during WebSocket URL edits, wires Overview to use it, extends UI regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent scoped Control UI fix as the canonical resolution for #41545, keeping normalized-scope token preservation, different-endpoint reload-or-clear behavior, regression tests, and changelog credit.

## Reproduction Assessment

Yes. Enter a Gateway Token in Overview > Gateway Access, then edit the WebSocket URL to a harmless non-identical value such as adding a trailing slash; current main clears the token through the handler in ui/src/ui/views/overview.ts.

## Solution Assessment

Yes. The PR's normalized-scope helper is the narrowest maintainable fix because it preserves credentials for the same endpoint while avoiding credential carryover to a different gateway endpoint.
