## Summary

Keep open. Current main still clears the Gateway Token on harmless same-endpoint WebSocket URL edits, and the PR is a narrow implementation that matches the existing normalized gateway-scope storage contract with focused regression coverage.

## What This Changes

The PR adds Control UI logic, tests, and a changelog entry to preserve Gateway Access tokens across same-normalized WebSocket URL edits while reloading or clearing tokens when the normalized endpoint changes.

## Best Possible Solution

Land a Control UI fix that preserves tokens only within the same normalized gateway scope and uses scoped session storage when switching endpoints.

## Reproduction Assessment

Yes. On current main, enter a Gateway Token in Overview > Gateway Access, then edit the WebSocket URL to an equivalent non-identical value such as adding a trailing slash; the handler clears the token because the trimmed strings differ.

## Solution Assessment

Yes. The normalized-scope resolver is narrower and safer than preserving unconditionally because it avoids carrying a gateway-scoped credential to a different endpoint.
