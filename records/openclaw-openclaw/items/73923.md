## Summary

Keep open. Current main and the latest release still clear the Gateway Token on same-normalized WebSocket URL edits, while this PR’s scoped resolver matches the existing Control UI token-storage contract and I found no blocking correctness or security findings.

## What This Changes

The PR adds a storage helper, Overview input wiring, regression tests, and a changelog entry to preserve Gateway Access tokens for same-normalized WebSocket URL edits while clearing or reloading scoped tokens on endpoint changes.

## Best Possible Solution

Land or replace this with the same scoped Control UI fix: preserve tokens only when the normalized gateway scope is unchanged, and load or clear the token when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the Overview input handler clears the token for any non-identical trimmed URL, and the linked bug report gives a concrete frontend-only path using a trailing-slash or URL edit.

## Solution Assessment

Yes. The PR’s normalized-scope resolver is the narrow maintainable fix because it preserves harmless same-endpoint edits without carrying a gateway-scoped credential to an unrelated endpoint.
