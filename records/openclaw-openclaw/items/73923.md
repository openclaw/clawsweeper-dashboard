## Summary

Keep open: current main still has the token-clearing path, and the proposed patch aligns with the existing gateway-scoped token storage contract with no concrete correctness or security finding found.

## What This Changes

The PR adds Control UI token-resolution logic, Overview wiring, regression tests, and a changelog entry for preserving Gateway Access tokens across same-normalized WebSocket URL edits.

## Best Possible Solution

The desired end state is a Control UI fix that preserves tokens only within the same normalized gateway scope and uses scoped session storage when changing endpoints.

## Reproduction Assessment

Yes. On current main, enter a Gateway Token in Overview > Gateway Access, then edit the WebSocket URL to an equivalent non-identical value such as adding a trailing slash; the handler clears the token because the trimmed strings differ.

## Solution Assessment

Yes. The proposed normalized-scope resolver is narrower and safer than preserving unconditionally because it avoids carrying a gateway-scoped credential to a different endpoint.
