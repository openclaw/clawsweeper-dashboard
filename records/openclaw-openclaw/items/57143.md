## Summary

Keep open. Current main still contains the reported hardcoded `/data/.clawdbot` references in the bundled `gh-issues` skill, and the related open closing PR #57146 should remain the implementation candidate until it lands or is replaced.

## What This Changes

Replace legacy `/data/.clawdbot` config, claims, and cursor paths in `skills/gh-issues/SKILL.md` with the standard OpenClaw state/config path pattern.

## Best Possible Solution

Land PR #57146 or an equivalent narrow patch that removes every `/data/.clawdbot` reference, uses the existing env-aware state/config path pattern, and extends the env-path guidance test to prevent regression.

## Reproduction Assessment

Yes. A static `rg` against current main reproduces the bug by finding the legacy config, claims, cursor, and sub-agent prompt paths while the config helpers and docs define standard installs under `~/.openclaw` or env overrides.

## Solution Assessment

Yes, with one refinement: the maintainable fix should follow the existing `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` pattern rather than replacing every state reference with a bare `~/.openclaw` literal.
