## Summary

Keep open. Current main still contains the reported hardcoded `/data/.clawdbot` paths in the bundled `gh-issues` skill, and open PR #57146 is already the closing implementation candidate that should land or be replaced before this issue closes.

## What This Changes

The issue requests replacing legacy `/data/.clawdbot` references in the bundled `gh-issues` skill with OpenClaw’s standard state/config paths.

## Best Possible Solution

Land #57146 or an equivalent narrow patch that removes every `/data/.clawdbot` reference, follows the existing `OPENCLAW_CONFIG_PATH`/`OPENCLAW_STATE_DIR` pattern, and extends the env-path guidance test to prevent regression.

## Reproduction Assessment

Yes. A static `rg` or `nl` inspection against current main reproduces the issue by finding the legacy token, claims, cursor, and sub-agent prompt paths while core config helpers and docs define standard state/config locations under `~/.openclaw` with env overrides.

## Solution Assessment

Yes, the narrowest maintainable fix is in the bundled `gh-issues` skill plus its path-guidance test. The only refinement is to use env-aware active config/state expressions rather than a blanket bare `~/.openclaw` replacement.
