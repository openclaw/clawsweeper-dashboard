## Summary

Keep open. Current main still contains the reported hardcoded `/data/.clawdbot` references in the bundled `gh-issues` skill, and open PR #57146 already references this issue with closing syntax, so it should remain the implementation candidate until it lands or is replaced.

## What This Changes

Replace the bundled `gh-issues` skill's legacy `/data/.clawdbot` config, claims, and cursor paths with the standard OpenClaw state/config path pattern.

## Best Possible Solution

Land PR #57146 or an equivalent narrow patch that removes every `/data/.clawdbot` reference and follows the existing `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` pattern with a regression check.

## Reproduction Assessment

Yes. A static `rg` against current main reproduces the issue by finding the legacy token, claims, and cursor paths while the config helper, tests, and docs define standard state/config locations under `~/.openclaw` or env overrides.

## Solution Assessment

Yes, with one refinement. The narrow maintainable fix is not just a literal `~/.openclaw` replacement everywhere, but use of the existing env-aware state/config path pattern for mutable state and token lookup.
