Keep open. Current main still builds exec, process, and node-host tool results from raw stdout/stderr-derived strings before a local exec/process redaction boundary, and https://github.com/openclaw/openclaw/pull/81185 is the open closing PR that should carry the fix through review and merge.

The issue requests redacting secret-shaped stdout/stderr from exec, process, and node-host tool results before those payloads reach agent-visible tool result content/details.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main still returns stdout/stderr-derived strings through foreground exec, process poll/log, and node-host exec result content/details, and the reporter confirmed a fake-canary leak on OpenClaw 2026.5.7.

## Solution Assessment

Yes, directionally. Applying the existing tool-payload redaction helper at the exec/process/node-host result assembly boundary is the narrow maintainable fix; hard-blocking specific files would be separate defense-in-depth and would miss generic secret-bearing stdout.
