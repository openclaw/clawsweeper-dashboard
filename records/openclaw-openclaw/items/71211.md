## Summary

Keep open. Current main and the latest release still show stdout/stderr-derived exec, process, and node-host output being assembled into tool-result text/details before a local redaction boundary, and an open linked PR with closing syntax is already the implementation path.

## What This Changes

The issue reports that exec/process/node-host tool results return stdout/stderr-derived content to the agent without first redacting secret-shaped output.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81185 if maintainer/security review and required checks agree with its shared tool-result redaction boundary, then close this issue as fixed by that merge.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.7` show exec/process/node-host output copied into returned tool-result content/details before a local redaction call, and the reporter confirmed a fake canary leak on the latest release.

## Solution Assessment

Yes, directionally. Applying the existing tool-payload redaction helper at the exec/process/node-host result assembly boundary is the narrowest maintainable fix; hard-blocking specific secret-file reads would be a separate defense-in-depth choice and would miss generic secret-bearing stdout.
