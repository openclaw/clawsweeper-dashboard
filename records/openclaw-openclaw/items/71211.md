- `P1`: The issue reports a current security-sensitive agent tool-result leak that can expose credentials in real exec workflows.
- `impact:security`: The affected behavior can put tokens, env files, passwords, and other sensitive command output into agent/tool context.

## Summary

Keep open: current main still returns stdout/stderr-derived exec, process, and node-host output through tool-result content/details before an exec-owned redaction boundary, and the open closing PR is the right path to resolve it.

## What This Changes

Redact secret-bearing stdout/stderr before exec, background process, and node-host tool results are returned to the agent.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81185 or an equivalent focused replacement that redacts exec/process/node-host returned content and details at the tool-result boundary, then close this issue after merge proof.

## Reproduction Assessment

Yes, at source level. A command that prints a fake secret-shaped env assignment flows through foreground exec, process poll/log, and node-host result assembly without a local redaction call on current main.

## Solution Assessment

Yes, directionally. Applying the existing tool-payload redaction helper at the exec/process/node-host result assembly boundary is the narrow maintainable fix; this issue should stay open until the linked PR or replacement lands.
