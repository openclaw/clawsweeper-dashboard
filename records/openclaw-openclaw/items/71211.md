## Summary

Keep open. Current main still returns exec/process/node-host stdout and stderr-derived payloads before an exec-owned redaction boundary, and the open linked PR is the right fix path but has not merged.

## What This Changes

The issue reports that exec/process-style tool results can return secret-bearing stdout or stderr to the agent without redaction.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81185 or a narrow equivalent that redacts exec, process, node-host, and live-update tool-result payloads before agent delivery, then close this issue after merge proof.

## Reproduction Assessment

Yes, at source level. Current main still routes exec/process stdout, stderr, aggregated output, and live-update tails into returned tool results or update payloads without local redaction; the reporter also confirmed a fake-canary leak on OpenClaw 2026.5.7.

## Solution Assessment

Yes, directionally. Applying the existing tool-payload redaction helper at the exec/process/node-host result and update boundaries is the narrow maintainable fix; hard-blocking specific files would be defense-in-depth and would miss generic secret-bearing stdout.
