## Summary

Keep open. Current main still assembles exec, process, and node-host stdout/stderr-derived output into returned tool-result content/details before a local redaction boundary, and an open linked PR with closing syntax already owns the implementation path.

## What This Changes

The issue asks for exec/process/node-host command output returned to agents to be redacted before secrets can enter tool-result context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81185 if maintainer/security review and required checks agree with its shared tool-result redaction boundary, then close this issue as fixed by that merge.

## Reproduction Assessment

Yes, at source level. Current main still assembles foreground exec, process poll/log, and node-host stdout/stderr/error strings into returned tool-result content/details before a local redaction pass; the reporter also confirmed a fake canary leak on release v2026.5.7.

## Solution Assessment

Yes, directionally. Applying the existing tool-payload redaction helper at the exec/process/node-host result assembly boundary is the narrowest maintainable fix; hard-blocking specific secret-file reads would be separate defense-in-depth and would miss generic secret-bearing stdout.
