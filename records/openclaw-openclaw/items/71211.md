## Summary

Keep open. Current main still constructs raw exec/process/node-host tool-result content and details from stdout/stderr before an exec-owned redaction boundary; downstream UI/transcript redaction and related merged redaction PRs help adjacent surfaces but do not fully resolve this issue.

## What This Changes

The issue requests redaction or blocking before exec stdout/stderr is delivered as tool-result content.

## Best Possible Solution

Add a shared exec/tool-result redaction boundary before stdout/stderr-derived content and structured details are handed to agent runtimes, while preserving existing UI, transcript, logging, and diagnostic redaction as defense in depth.

## Reproduction Assessment

Yes, at source level. On current main, command output in outcome.aggregated/stdout/stderr/errorText is copied into exec, process, and node-host AgentToolResult content or details before any local exec-owned redaction call.

## Solution Assessment

Yes, directionally. Redacting stdout/stderr-derived tool-result content before delivery is the narrowest durable fix; hard-blocking secret-file reads may be useful defense in depth but would miss generic secret-bearing command output.
