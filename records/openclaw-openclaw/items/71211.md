## Summary

Keep open. Current main and the latest stable tag still assemble exec, process, node-host, and live-update output from raw stdout/stderr-derived strings before an exec-owned redaction boundary; the open closing PR should carry the fix through review or be replaced by an equivalent narrow patch.

## What This Changes

The issue reports that exec/process/node-host command output can return secret-shaped stdout/stderr to the agent without redaction.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81185 or an equivalent focused fix that redacts foreground exec, process poll/log/list, node-host exec, and live update payloads at their result assembly boundaries before closing this issue.

## Reproduction Assessment

Yes, at source level. A command that prints a fake secret-shaped value would flow through current foreground exec, process, node-host, or live-update builders before local redaction; I did not run a live command in this read-only review.

## Solution Assessment

Yes, directionally. Redacting returned tool-result and live-update payloads at the exec/process/node-host assembly boundaries is the narrow maintainable fix; file-read blocking would be defense-in-depth but would not cover generic secret-bearing stdout.
