## Summary

Keep open: current main still trusts runtime conversation bindings that point at missing session targets and cleanup can prune missing-transcript rows without clearing matching bindings, while live GitHub shows open fix PRs covering the remaining work. The stuck-session recovery half appears partly addressed on main, but the binding hygiene bug is still source-reproducible.

## What This Changes

The issue asks gateway session resume to clear or rebind orphaned LCM/current-conversation bindings and to recover stuck sessions instead of queuing indefinitely.

## Best Possible Solution

Land a generic binding hygiene fix that validates persisted target sessions before route rewrite and unbinds keys removed by cleanup, with focused regression coverage; keep the stuck-session path covered by existing recovery tests unless live evidence shows another gap.

## Reproduction Assessment

Yes, source-reproducible: a persisted current-conversation binding can point at a missing transcript, and current main still resolves/touches it before route rewrite while cleanup removes missing rows without unbinding. I did not run a live Discord/LCM setup in this read-only review.

## Solution Assessment

Yes, generic binding reconciliation and cleanup unbinding are the narrowest maintainable fix; restoring LCM history into a new session would be plugin-specific and is not required to stop routing to orphaned targets.
