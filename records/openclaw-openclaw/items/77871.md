## Summary

Keep open: current main still has the stale runtime conversation-binding path and cleanup-binding hygiene gap, while the stuck-session recovery half is partly addressed. There are open fix PRs that specifically cover this issue, so the right cleanup is to review and land or close those PRs rather than close the issue early.

## What This Changes

The issue requests gateway session-resume recovery when current-conversation bindings point to missing session transcripts, plus stuck-session escalation instead of log-only diagnostics.

## Best Possible Solution

Land one generic binding hygiene fix that validates persisted runtime binding targets before route rewrite and unbinds keys removed by session cleanup, with focused regression coverage and dry-run cleanup kept read-only.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the stale-binding class: bind a conversation to a session key, remove the backing transcript/store row through missing-transcript cleanup, then resolve the conversation route and it still rewrites to the stale target.

## Solution Assessment

Yes. Generic binding validation plus cleanup unbinding is the narrowest maintainable fix; restoring LCM history into a new session would be optional and more plugin-specific than needed to stop phantom routing.
