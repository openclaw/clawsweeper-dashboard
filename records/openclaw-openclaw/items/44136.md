## Summary

Keep open. The Copilot base URL precedence problem is still real on current main, but this PR is not merge-ready: current main routes the active flow through provider-runtime auth, while the branch edits the older runner file and still has lazy-loading and profile-rotation correctness issues.

## What This Changes

The PR branch adds conditional Copilot token-derived baseUrl assignment in `src/agents/pi-embedded-runner/run.ts` during initial auth and token refresh so explicit user baseUrl values are not overwritten.

## Best Possible Solution

Port the precedence rule into the current provider-runtime auth controller: preserve explicit custom GitHub Copilot baseUrl values, apply token-derived Business/Enterprise endpoints when the endpoint is absent/default or replacing a prior token-derived endpoint, keep runtime and effective models synchronized, preserve lazy runtime imports, and cover initial auth, refresh, context-window-capped models, and profile rotation.

## Reproduction Assessment

Yes. A focused unit reproduction can mock GitHub Copilot `prepareRuntimeAuth` to return a token-derived `baseUrl` while the runtime model already has a custom `baseUrl`; current `auth-controller.ts` will overwrite it unconditionally.

## Solution Assessment

No. The intended behavior is correct, but this branch is not the best implementation now because the active code moved to `createEmbeddedRunAuthController` and the diff adds an eager token-helper import; the safer fix is a narrow current-main port with explicit token-derived versus user-configured endpoint tracking.

  - body: This top-level import pulls the Copilot token helper into every embedded-runner startup even though this provider path is otherwise loaded lazily. Keep the default endpoint lookup behind the existing async provider-runtime boundary instead of making `run.ts` eagerly import it.
  - confidence: 0.9

  - body: The guard treats any non-default `model.baseUrl` as user-configured, including a value written by a previous Copilot token exchange. After rotating between Copilot profiles, the next token can keep the prior profile's endpoint and route requests to the wrong host.
  - confidence: 0.88
