## Summary

Keep open: the requested precedence rule is still valid, but this PR is stale against the current auth-controller seam, has a lazy-import regression, mishandles profile rotation, and still lacks after-fix real behavior proof.

## What This Changes

The PR patches the old embedded-runner Copilot token flow to avoid overriding a configured `baseUrl` with the token-derived endpoint unless the model has no base URL or still has the default Copilot URL.

## Best Possible Solution

Implement the precedence rule in the current auth-controller/provider-runtime boundary with explicit configured-versus-token-derived base URL state, preserving custom endpoints while updating token-derived endpoints on initial auth, refresh, effective models, and profile rotation.

## Reproduction Assessment

Yes. Source inspection on current main shows GitHub Copilot `prepareRuntimeAuth` returns a token-derived `baseUrl`, and the auth controller unconditionally applies it over runtime/effective models despite docs defining explicit `models.providers.<id>.baseUrl` as the override path.

## Solution Assessment

No. The intended precedence rule is right, but this branch patches stale inline `run.ts` code and uses `model.baseUrl` as the state marker, which cannot safely handle profile rotation after a token-derived endpoint has already been written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep the Copilot token helper lazy:** `src/agents/pi-embedded-runner/run.ts:40`
  - body: This top-level import defeats the existing lazy boundary for the Copilot token helper, while the same runtime surface is still loaded dynamically for Copilot-only paths. Keep the default endpoint lookup behind the lazy/runtime-auth path so embedded-runner startup does not eagerly evaluate Copilot runtime code.
  - confidence: 0.9
- **[P2] Track token-derived base URLs across profile rotation:** `src/agents/pi-embedded-runner/run.ts:635-639`
  - body: The new guard treats any non-default `model.baseUrl` as user-configured after the first token exchange. When auth profile rotation switches between mixed individual/business accounts, the old token-derived endpoint can be kept for the new token and route requests to the wrong Copilot host.
  - confidence: 0.88
