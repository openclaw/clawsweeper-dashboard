## What This Changes

The PR conditionally applies token-derived GitHub Copilot `baseUrl` values during embedded-runner initial auth and token refresh while trying to preserve a custom configured `baseUrl`.

## Best Possible Solution

Implement configured-vs-token-derived Copilot `baseUrl` precedence in `src/agents/pi-embedded-runner/run/auth-controller.ts` or the provider-runtime seam with explicit origin tracking, focused refresh/profile-rotation tests, and redacted Business or Enterprise proof.

## Reproduction Assessment

Yes, from source inspection. A focused test can mock configured Copilot baseUrl plus two token-derived profile endpoints and show the PR's `model.baseUrl` guard preserving stale runtime host state instead of cleanly separating configured and runtime origins.

## Solution Assessment

No. The precedence rule is valid, but this branch applies it in the old inline runner path and should be rebuilt on the current auth-controller/provider-runtime design with explicit configured-vs-runtime baseUrl tracking.
