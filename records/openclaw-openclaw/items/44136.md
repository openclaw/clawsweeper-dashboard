## Summary

Keep open. The requested precedence rule is still missing from the active current-main runtime-auth path, but this PR is not merge-ready because it targets the old direct Copilot token flow and introduces a lazy-loading regression.

## What This Changes

The PR branch adds conditional Copilot token-derived `baseUrl` assignment in `src/agents/pi-embedded-runner/run.ts` during initial auth and refresh, updating both `model` and `effectiveModel` only when the existing base URL is absent or default.

## Best Possible Solution

Port the precedence rule into the current provider-runtime auth flow: preserve an explicit custom GitHub Copilot `baseUrl`, apply token-derived Business/Enterprise endpoints when the endpoint is absent/default or already token-derived, keep runtime and effective models synchronized, and cover initial auth, refresh, context-window-capped models, and profile rotation.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is available by mocking GitHub Copilot `prepareRuntimeAuth` to return a token-derived `baseUrl` while the runtime model already has a custom `baseUrl`; current `auth-controller.ts` will overwrite it unconditionally.

## Solution Assessment

No. The intended behavior is right, but this PR is not the best implementation now because the active code moved to provider runtime auth and the patch adds an eager token-helper import; a narrow current-main repair should preserve lazy loading and handle profile rotation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

  - body: This top-level import pulls the Copilot token helper into every embedded-runner startup even though the runner relies on lazy runtime loading for this provider path. Keep the token/default endpoint lookup behind the existing async provider-runtime boundary instead of making `run.ts` eagerly import it.
  - confidence: 0.9
- **[P2] Reapply token base URL when profiles rotate:** `src/agents/pi-embedded-runner/run.ts:486-488`

After rotating from one Copilot profile to another, the second profile can keep the first profile's token-derived endpoint and route requests to the wrong host.
  - confidence: 0.88
