## Summary

Keep this PR open: the reported Copilot baseUrl precedence bug is still present on current main, but this branch is stale and not merge-ready because it edits the older inline runner flow and introduces review-blocking runtime-auth issues.

## What This Changes

The branch adds conditional Copilot token-derived `baseUrl` assignment in the embedded runner so explicit user-configured `baseUrl` values are not overwritten during initial auth and refresh.

## Best Possible Solution

Port the precedence rule into the current `createEmbeddedRunAuthController`/GitHub Copilot runtime-auth boundary, preserving explicit custom endpoints while keeping token-derived endpoints synchronized for initial auth, refresh, context-capped effective models, and profile rotation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock GitHub Copilot `prepareRuntimeAuth` to return a token-derived `baseUrl` while the runtime model starts with a custom `baseUrl`, then `initializeAuthProfile()` currently overwrites both runtime and effective models through `auth-controller.ts`.

## Solution Assessment

No. The intended behavior is right, but this branch is not the best current implementation because it patches stale inline runner code, adds an eager Copilot helper import, and does not safely track token-derived endpoints across profile rotation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep the Copilot token helper lazy:** `src/agents/pi-embedded-runner/run.ts:40`
  - body: This top-level import pulls the Copilot token helper into every embedded-runner startup even though the Copilot path is otherwise behind provider runtime loading. Keep the endpoint/default lookup behind the existing plugin/runtime auth boundary so unrelated providers do not eagerly evaluate Copilot token code.
  - confidence: 0.9
- **[P2] Track token-derived base URLs across profile rotation:** `src/agents/pi-embedded-runner/run.ts:486-488`
  - body: The guard treats any non-default `model.baseUrl` as user-configured, including a value written by a previous Copilot token exchange. After rotating between Copilot profiles, the next token can keep the prior profile's endpoint and route requests to the wrong host.
  - confidence: 0.88
