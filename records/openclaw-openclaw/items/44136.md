## Summary

Keep open: current main still has the Copilot baseUrl precedence gap, but this PR should not merge as-is because it edits the stale inline runner flow, adds an eager Copilot token import, can retain stale token-derived hosts across profile rotation, and still lacks contributor real-behavior proof.

## What This Changes

The PR changes GitHub Copilot runtime auth so token-derived baseUrl values only replace missing or default model baseUrls, preserving a user-configured provider baseUrl.

## Best Possible Solution

Implement the precedence rule in the current auth-controller/provider-runtime boundary, preserving explicit provider baseUrls while still refreshing token-derived Copilot endpoints across auth refresh and profile changes.

## Reproduction Assessment

Yes, by source inspection. Configure a GitHub Copilot provider baseUrl: current main's Copilot `prepareRuntimeAuth` returns a token-derived baseUrl, and the auth controller applies it to runtime and effective models without an explicit-config guard.

## Solution Assessment

No. The precedence rule is the right direction, but this branch edits stale inline runner logic, adds an eager import across a lazy boundary, and uses mutable `model.baseUrl` as the user-config marker instead of fixing the current auth-controller seam.
