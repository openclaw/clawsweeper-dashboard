## Summary

Keep open: the PR targets a real Copilot baseUrl precedence bug on current main, but the branch is stale against the current auth-controller seam, has blocking correctness findings, and lacks external real behavior proof.

## What This Changes

The PR adds a top-level Copilot default-base-URL import and conditional token-derived baseUrl assignments in `src/agents/pi-embedded-runner/run.ts` so non-default configured endpoints are not overwritten.

## Best Possible Solution

Implement the precedence rule in the current auth-controller and GitHub Copilot runtime-auth boundary, preserving explicit custom endpoints while keeping token-derived endpoints synchronized for initial auth, refresh, effective models, and profile rotation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: GitHub Copilot `prepareRuntimeAuth` returns a token-derived `baseUrl`, and `applyPreparedRuntimeRequestOverrides()` unconditionally writes it over the runtime and effective models even if the model started with a configured custom endpoint.

## Solution Assessment

No. The intended precedence rule is right, but this branch patches stale inline `run.ts` auth code and relies on mutated `model.baseUrl` as the state marker; the maintainable fix belongs in the current auth-controller/runtime-auth seam with explicit configured-versus-token-derived tracking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep the Copilot token helper lazy:** `src/agents/pi-embedded-runner/run.ts:40`
  - body: This adds a top-level import from `../../providers/github-copilot-token.js` while the same module is still dynamically loaded for Copilot-only paths. That defeats the lazy boundary on every embedded-runner start; keep the default endpoint lookup behind the existing lazy/runtime-auth path.
  - confidence: 0.9
- **[P2] Track token-derived base URLs across profile rotation:** `src/agents/pi-embedded-runner/run.ts:637-641`
  - body: After the first token exchange this writes `model.baseUrl` to a token-derived endpoint, so the next profile switch can treat that prior endpoint as user-configured and keep it for the new token. Mixed individual/business profiles can then route the new token to the wrong Copilot host.
  - confidence: 0.88
