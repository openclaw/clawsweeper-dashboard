## Summary

Keep open: this PR targets a real current-main Copilot endpoint precedence bug, but the branch is stale and not merge-ready because current main now applies runtime auth through `createEmbeddedRunAuthController` and the branch still carries review blockers.

## What This Changes

The PR branch adds conditional Copilot token-derived `baseUrl` assignment in `src/agents/pi-embedded-runner/run.ts` during initial auth and token refresh so explicit user `baseUrl` values are not overwritten.

## Best Possible Solution

Port the precedence rule into the current runtime auth controller: preserve explicit custom GitHub Copilot baseUrl values, apply token-derived endpoints only when appropriate, keep runtime and effective models synchronized, preserve lazy runtime boundaries, and cover initial auth, refresh, context-window-capped models, and profile rotation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock GitHub Copilot `prepareRuntimeAuth` to return a token-derived `baseUrl` while the runtime model already has a custom `baseUrl`, then `initializeAuthProfile()` will currently overwrite it through `auth-controller.ts`.

## Solution Assessment

No. The intended behavior is correct, but this branch is not the best current implementation because the active code moved into `createEmbeddedRunAuthController` and the branch introduces a lazy-loading boundary regression.
