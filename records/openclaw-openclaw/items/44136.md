## What This Changes

The PR conditionally applies Copilot token-derived baseUrl in the embedded runner auth and refresh paths while preserving non-default user-configured baseUrl values.

## Best Possible Solution

Implement configured-vs-token-derived Copilot baseUrl precedence in the current auth-controller/provider-runtime path with explicit origin tracking, regression coverage, and redacted Business or Enterprise proof.

## Reproduction Assessment

No live high-confidence reproduction was established; source inspection gives a credible reproduction path because current main applies preparedAuth.baseUrl over the resolved model baseUrl while docs/tests preserve configured provider baseUrl before runtime auth. I did not run a live Copilot Business account.

## Solution Assessment

No, not as currently patched. The precedence rule is valid, but the fix should be ported to auth-controller/provider-runtime and avoid using model.baseUrl as both an explicit config marker and mutable runtime token state.
