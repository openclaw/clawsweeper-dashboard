## Summary

Keep open: source confirms the nested approval metadata bug, but this PR is not merge-ready because it expands a parser still used on generic command output and the supplied proof is code diff inspection rather than real after-fix behavior.

## What This Changes

The branch replaces exec approval result regex parsing with balanced metadata parsing, changes the gateway timeout allowlist reason to colon-separated text, strips parentheses from node denial reasons, adds parser and followup tests, and adds a changelog entry.

## Best Possible Solution

Land the nested-metadata fix only with approval-result parsing limited to trusted exec lifecycle or structured `execDetails` data, plus redacted runtime proof that the denied followup branch is used.

## Reproduction Assessment

Yes, at source level: current main emits nested `approval-timeout (allowlist-miss)` metadata and the current `[^)]*` parser rejects that denied payload. I did not execute tests or a live gateway flow because this was a read-only review.

## Solution Assessment

No. Balanced parsing addresses the reported bug, but the proposed parser should be gated to trusted approval lifecycle data or replaced with structured fields before it is safe to merge.
