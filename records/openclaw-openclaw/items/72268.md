## Summary

Keep this PR open: it targets a real exec approval followup bug, but the latest head still broadens approval-wrapper parsing while that parser is reachable from raw command output, and the supplied proof is code-diff inspection rather than after-fix runtime behavior.

## What This Changes

The PR replaces exec approval regex parsing with balanced metadata parsing, changes one timeout denial reason to colon-separated text, strips parentheses from node denial reasons, and adds parser/followup tests plus a changelog entry.

## Best Possible Solution

## Solution Assessment

No. Balanced parsing addresses the reported symptom, but the proposed parser should be gated to trusted approval lifecycle data or replaced with structured fields before merge.
