## Summary

Keep open: the PR targets a real current-main exec approval followup bug, but it still needs contributor real behavior proof and a trusted-source boundary for approval-result parsing before merge.

## What This Changes

The PR replaces denied/finished exec approval result regex parsing with balanced-parentheses parsing, adds nested-metadata parser and followup tests, and updates the changelog.

## Best Possible Solution

Land nested metadata parsing together with a trusted-source or structured approval-result gate, spoofed-output regression coverage, and redacted after-fix terminal/log or Telegram proof.

## Reproduction Assessment

Yes. Current main source plus a read-only regex probe show nested denied and finished metadata fails parsing, while gateway code emits `approval-timeout (allowlist-miss)` and followup routing keys off denied parsing.

## Solution Assessment

No. Balanced parsing is the right parser direction, but the mergeable fix should also gate approval-result parsing to trusted lifecycle data or structured fields before accepting the broader wrapper grammar from generic output.
