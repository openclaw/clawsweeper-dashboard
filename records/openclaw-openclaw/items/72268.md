## Summary

Keep open: the nested-metadata bug is real on current main, but the latest PR still broadens approval-result parsing without a trusted-source gate and lacks after-fix real behavior proof.

## What This Changes

The PR replaces denied/finished exec approval result regex parsing with balanced-parentheses parsing, adds nested-metadata parser and followup tests, and updates the changelog.

## Best Possible Solution

Land nested metadata parsing together with a trusted-source or structured approval-result gate and spoofed-output regression coverage, then require redacted terminal/log or Telegram proof.

## Reproduction Assessment

Yes. Current main source plus a read-only regex probe show nested denied/finished metadata fails parsing, while gateway code emits `approval-timeout (allowlist-miss)` and followup routing keys off denied parsing.

## Solution Assessment

No. Balanced parsing is the right parser direction, but the complete fix should also gate approval-result parsing to trusted lifecycle data or structured approval fields before broadening accepted wrappers.
