## Summary

Keep open: this fixes a real current-main exec approval parsing bug, but the PR is not merge-ready because it widens approval-wrapper parsing while generic command output is still parsed as approval status, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR replaces regex parsing of exec approval result metadata with a balanced-parentheses parser, adds nested-metadata regression tests, and adds a changelog entry.

## Best Possible Solution

Land balanced metadata parsing only with a trusted-source or structured approval-result path, regression coverage for nested metadata and spoofed command output, and redacted real async/Telegram approval proof.

## Reproduction Assessment

Yes. Source inspection plus a read-only regex check show current main rejects nested denied/finished metadata, and the gateway currently emits `approval-timeout (allowlist-miss)` for the affected denial path.

## Solution Assessment

No. The balanced parser is the right direction for the reported bug, but widening parser acceptance is not the best mergeable solution until approval parsing is scoped to structured or otherwise trusted exec lifecycle data.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:88`
