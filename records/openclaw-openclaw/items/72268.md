## What This Changes

The PR replaces exec approval denied/finished regex parsing with balanced-parentheses parsing, adds nested-metadata parser and followup tests, and adds a changelog entry.

## Best Possible Solution

Land nested metadata parsing with a trusted approval-result source gate and spoofed-output regression coverage, then require redacted real Telegram or terminal proof of the nested-denial flow.

## Reproduction Assessment

Yes. Current main source plus the read-only Node regex check show nested denied/finished metadata fails parsing, while the gateway emits `approval-timeout (allowlist-miss)` and followup routing keys off denied parsing.

## Solution Assessment

No. Balanced parsing is the right parser fix, but it is not the best complete merge shape until approval-status parsing is limited to trusted lifecycle data and after-fix behavior proof is posted.
