## Summary

Keep open. Current main still has the nested metadata parsing bug and this PR targets the right parser path, but it is not merge-ready because it lacks real after-fix behavior proof and widens spoofable approval-status parsing from generic exec output.

## What This Changes

The PR replaces regex-based exec approval result parsing with a balanced-parentheses parser, adds nested metadata parser/followup tests, and adds a changelog entry.

## Best Possible Solution

Land nested metadata support together with a trusted-source or structured approval-result gate, plus regression coverage for spoofed command output and nested denial followups.

## Reproduction Assessment

Yes. Source inspection plus a read-only regex check show current main accepts flat denial metadata but rejects nested denied/finished metadata, while the gateway emits the nested `approval-timeout (allowlist-miss)` reason.

## Solution Assessment

No. The balanced parser is the right direction, but this PR should also limit approval-status parsing to trusted lifecycle data and provide real terminal/log or Telegram proof before merge.
