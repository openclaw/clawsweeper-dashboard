## Summary

Keep open. The PR targets a real current-main bug in async exec approval followups, but the latest diff still broadens approval-wrapper parsing while that parser is also applied to generic command output, and it still carries unrelated generated SDK hash churn plus no changelog entry.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with balanced metadata parsing, adds nested-metadata parser and followup tests, and changes the generated plugin SDK API hash file.

## Best Possible Solution

Land a narrowed version that keeps the balanced metadata parser and nested denied/finished regression coverage, but first scopes approval-result parsing to trusted exec lifecycle data or a structured source guard, removes unrelated generated SDK hash changes, and adds the required changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the regex rejects nested approval metadata, while the gateway emits `approval-timeout (allowlist-miss)` and followup routing depends on the denied parser result.

## Solution Assessment

No, not as currently proposed. The balanced parser is the right narrow direction for the nested metadata bug, but the merge-ready solution should address the trusted-source boundary before broadening accepted approval-wrapper text.
