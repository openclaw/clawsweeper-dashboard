## Summary

## What This Changes

The PR replaces regex-based exec approval result parsing with balanced metadata parsing, changes one denial reason format, strips parentheses from node denial reasons, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a scoped parser and producer fix only after approval-result parsing is gated to trusted lifecycle or structured exec data, with focused tests and redacted real followup proof.

## Reproduction Assessment

Yes, at source level: current main emits `approval-timeout (allowlist-miss)` and the current regex parser closes at the first `)`, so it misses the denied branch. I did not run tests or a live gateway flow because this was a read-only review.

## Solution Assessment

No. Balanced parsing addresses the symptom, but the expanded grammar should be limited to trusted lifecycle or structured exec data before merge because the parser is still applied to raw command output.
