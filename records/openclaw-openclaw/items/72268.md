## Summary

Keep open: the nested-metadata bug is real and this PR targets the right parser, but the current diff is not merge-ready because it widens approval-result text parsing while generic command output is still parsed as approval status, and it still carries unrelated generated SDK hash churn plus no changelog entry.

## What This Changes

This PR replaces exec approval denied/finished regex parsing with a balanced metadata parser and adds nested-metadata parser and followup tests, while also changing the generated plugin SDK API hash file.

## Best Possible Solution

Land a narrowed version that keeps the balanced nested-metadata parser and regression tests, scopes approval-result classification to trusted or structured exec lifecycle data, drops generated SDK hash churn, and adds a single-line changelog Fixes entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the regex rejects nested approval metadata, the gateway emits `approval-timeout (allowlist-miss)`, and followup routing depends on the denied parser result.

## Solution Assessment

No. The balanced parser is the right targeted parse fix, but the merge-ready solution should first keep approval resolution on trusted or structured exec lifecycle data and remove unrelated generated hash churn.
