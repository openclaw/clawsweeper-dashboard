## Summary

This PR targets a real current-main parser bug, but it should stay open because the patch widens approval-wrapper parsing while approval events are still derived from generic exec output, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch replaces regex-based exec approval result parsing with a balanced-parentheses parser, adds nested-metadata parser/followup tests, and adds a changelog entry.

## Best Possible Solution

Land nested metadata support with a trusted-source or structured approval-result gate, plus regression coverage for spoofed command output and nested denial followups.

## Reproduction Assessment

Yes. Source inspection plus a focused Node regex check show current main accepts flat denied metadata but rejects nested denied/finished metadata, while gateway code emits `approval-timeout (allowlist-miss)`; I did not run a live async approval flow.

## Solution Assessment

No. The balanced parser is the right direction, but the PR should also limit approval-status parsing to structured or trusted lifecycle data before widening accepted strings.
