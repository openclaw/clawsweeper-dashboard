## Summary

Keep open. The nested-parentheses parser fix addresses a real async exec followup bug, but the latest diff still widens a shared text parser that is also used to emit approval events from command output, so it needs approval-event trust-boundary review before merge.

## What This Changes

The PR replaces denied/finished exec approval regex parsing with balanced-parentheses metadata parsing, adds nested-metadata followup/parser tests, and adds a changelog fix entry.

## Best Possible Solution

Land a narrowed version that keeps the balanced parser, regression tests, and changelog entry, but derives approval resolution only from structured/trusted exec lifecycle data or a clearly gated trusted followup source.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the regex rejects nested denied/finished metadata, the gateway emits `approval-timeout (allowlist-miss)`, and followup routing depends on the denied parser result.

## Solution Assessment

No. The balanced parser is the right narrow parser fix, but the merge-ready solution should scope approval status parsing to trusted lifecycle data before widening the accepted wrapper grammar.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:87`
  - body: `parseExecApprovalResultText()` is also applied to `execDetails.aggregated` or generic tool result text before approval events are emitted. With the broader balanced parser, command output such as `Exec denied (... (nested) ...): ...` can be classified as a resolved approval denial, so gate this parsing to structured/trusted exec lifecycle data before widening the wrapper grammar.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff broadens approval-wrapper parsing while another handler still parses command output into approval events.

Concerns:

- **[medium] Spoofable exec approval status from command output:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1085`
  - body: The embedded tool handler parses output derived from `execDetails.aggregated` or generic tool result text and emits a resolved approval event when it parses as denied. Accepting nested metadata makes more command-produced strings look like trusted approval wrappers unless this path is gated to structured lifecycle data.
  - confidence: 0.84
