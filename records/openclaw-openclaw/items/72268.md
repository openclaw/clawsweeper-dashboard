## Summary

Keep open. The nested-parentheses parser bug is real on current main, but this PR is not merge-ready because it widens a shared approval-result text parser while an existing caller still derives approval-resolution events from generic exec output.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with balanced-parentheses metadata parsing, adds nested metadata parser and followup tests, and adds a changelog fix entry.

## Best Possible Solution

Land a narrowed version that keeps the balanced parser and regression coverage, but only emits approval-resolution events from trusted exec lifecycle data or a structured approval status source.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the regex rejects nested denied/finished metadata, the gateway emits `approval-timeout (allowlist-miss)`, and followup routing branches on `isExecDeniedResultText()`.

## Solution Assessment

No. Balanced parsing is the right narrow fix for nested metadata, but widening the shared parser before constraining the generic-output approval caller is not the safest merge-ready solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:86`

With this broader parser, a command that prints `Exec denied (...
