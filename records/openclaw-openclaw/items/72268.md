## Summary

Keep open. The parser fix addresses a real current-main bug, but the branch broadens approval-result parsing before approval events are gated to trusted exec lifecycle data and still lacks real after-fix behavior proof.

## What This Changes

The PR replaces exec approval result regex parsing with a balanced-parentheses parser, adds nested-metadata parser/followup tests, and adds a changelog entry.

## Best Possible Solution

Land nested metadata parsing together with a trusted exec lifecycle or structured approval-result gate and spoofed-output regression coverage.

## Reproduction Assessment

Yes. Source inspection and a read-only Node regex check show current main accepts flat denial metadata but rejects nested denied/finished metadata, while the gateway emits `approval-timeout (allowlist-miss)` and followup routing keys off denied parsing.

## Solution Assessment

No. The balanced parser is the right direction for the parser bug, but it should not merge until approval-status parsing is limited to structured or otherwise trusted lifecycle data instead of generic command output.
