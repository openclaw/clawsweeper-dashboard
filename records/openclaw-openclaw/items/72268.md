## Summary

Keep this PR open: the nested-parentheses exec approval bug is real on current main, but the patch is not merge-ready because it broadens approval-wrapper parsing while generic exec output can still be parsed into approval events, and the PR has no after-fix real behavior proof.

## What This Changes

The PR replaces denied/finished exec approval regex parsing with balanced-parentheses parsing, adds parser and followup tests for nested metadata, and adds a changelog entry.

## Best Possible Solution

Land nested metadata parsing with a trusted approval-result source gate and spoofed-output regression coverage, then require redacted real Telegram or terminal proof of the nested-denial flow.

## Reproduction Assessment

Yes. Current main source and the read-only regex check show nested denied/finished metadata fails parsing, while the gateway emits `approval-timeout (allowlist-miss)` and followup routing keys off denied parsing.

## Solution Assessment

No. Balanced parsing is the right parser fix, but it is not the best complete merge shape until approval-status parsing is limited to trusted lifecycle data and after-fix behavior proof is posted.
