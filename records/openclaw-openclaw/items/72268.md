## Summary

Keep open. The PR addresses a real current-main nested-metadata parsing bug and has added the required changelog entry, but it still broadens approval-result parsing while the embedded exec handler applies that parser to command output before emitting approval events.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with balanced-parentheses metadata parsing and adds nested-metadata followup tests plus a changelog fix entry.

## Best Possible Solution

Land a narrowed version that keeps the balanced parser, regression tests, and changelog entry, but only derives approval resolution from trusted exec lifecycle data or a structured approval status source.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the regex rejects nested denied/finished metadata, the gateway emits `approval-timeout (allowlist-miss)`, and followup routing depends on the denied parser result.

## Solution Assessment

No. The balanced parser is the right narrow fix for nested metadata, but widening a shared text parser before scoping approval status parsing to trusted lifecycle data is not the safest merge-ready solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:86`
  - body: `parseExecApprovalResultText()` is also applied to `execDetails.aggregated` or generic tool result text before emitting approval events. With this broader parser, a command that prints `Exec denied (... (nested) ...): ...` can be surfaced as a resolved approval denial, so gate this parsing to structured/trusted exec lifecycle data before widening the accepted wrapper grammar.

## Security Review

Status: needs_attention

Summary: The diff still broadens parsing across an approval-event trust boundary without constraining the caller to trusted exec lifecycle data.

Concerns:

- **[medium] Spoofable exec approval status from command output:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1085`

Accepting nested metadata makes more command-produced strings look like trusted approval wrappers unless the caller is gated to structured lifecycle data.
