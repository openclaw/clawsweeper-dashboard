## Summary

Keep open. The nested-parentheses parsing bug is real and the PR has useful focused tests, but the latest diff still broadens a shared approval text parser that is also applied to generic command output before the approval-event trust boundary is constrained.

## What This Changes

The PR replaces denied/finished exec approval regex parsing with balanced-parentheses metadata parsing, adds nested-metadata followup tests, and records a changelog fix.

## Best Possible Solution

Land the balanced metadata parser together with a trust-boundary fix that derives approval resolution only from structured or otherwise trusted exec lifecycle data.

## Reproduction Assessment

Yes. Source inspection plus a read-only regex check against current main shows nested `Exec denied` and `Exec finished` metadata fail the existing regex, and the gateway emits the nested denial reason `approval-timeout (allowlist-miss)`.

## Solution Assessment

No. The balanced parser is the right narrow parser change, but broadening it before scoping approval status parsing to trusted lifecycle data is not the safest merge-ready solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:88`
  - body: `parseExecApprovalResultText()` is also applied to generic command output before emitting approval events. With this broader parser, a command that prints `Exec denied (... (nested) ...): ...` can be surfaced as a resolved approval denial, so gate parsing to structured or trusted exec lifecycle data before widening the accepted wrapper grammar.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff broadens approval-wrapper parsing while an existing caller still applies that parser to generic tool output.

Concerns:

- **[medium] Spoofable exec approval status from command output:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1060`
  - body: `pi-embedded-subscribe.handlers.tools.ts` parses arbitrary string output with `parseExecApprovalResultText()` and emits an approval event when it parses as denied. Accepting nested metadata makes more command-produced strings look like trusted approval wrappers unless the caller is constrained to structured or trusted exec lifecycle data.
  - confidence: 0.86
