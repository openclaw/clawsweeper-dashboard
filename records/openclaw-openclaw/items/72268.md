## Summary

Keep open. The PR targets a real current-main exec approval parsing bug, but it is not merge-ready because the widened parser is still used on generic exec output and there is no after-fix real behavior proof.

## What This Changes

The branch replaces regex-based exec approval result parsing with a balanced-parentheses parser, adds nested-metadata parser/followup tests, and adds a changelog entry.

## Best Possible Solution

Land nested metadata support together with a trusted-source or structured approval-result gate, plus regression coverage for spoofed command output and nested denial followups.

## Reproduction Assessment

Yes. Source inspection plus a Node regex check against current main show the parser accepts flat denial metadata but rejects nested denied/finished metadata, and current gateway code emits the nested `approval-timeout (allowlist-miss)` reason.

## Solution Assessment

No. The balanced parser is the right direction, but this PR should not merge until approval-status parsing is limited to structured or otherwise trusted lifecycle data instead of generic command output.
