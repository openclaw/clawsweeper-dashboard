## Summary

Keep open. The nested-parentheses parser fixes a real current-main misclassification, but the latest diff still widens approval-wrapper parsing while an existing event path treats generic exec output as trusted approval status.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with balanced-parentheses metadata parsing, adds nested-metadata parser and followup tests, and adds a changelog entry.

## Best Possible Solution

Land balanced metadata support together with a trust-boundary change that derives approval resolution only from structured or otherwise trusted exec lifecycle data, plus spoofed-output regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main emits `approval-timeout (allowlist-miss)` from the gateway and the current parser regex rejects that nested metadata, which sends followups down the generic completion path.

## Solution Assessment

No. The balanced parser is the right narrow parser change, but it is not merge-ready until approval status parsing is scoped away from generic command output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope approval parsing to trusted exec results:** `src/agents/exec-approval-result.ts:88`
  - body: `parseExecApprovalResultText()` is widened to accept nested metadata, but the handler still runs it on exec command output before emitting approval events. A command can print `Exec denied (... (nested) ...): ...` and be surfaced as a resolved approval denial, so derive approval status from structured/trusted lifecycle data before accepting the broader wrapper grammar.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff broadens exec approval wrapper parsing while current event code still parses generic exec output as a trusted approval resolution.

Concerns:

- **[medium] Spoofable exec approval status from command output:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1060`
  - body: `pi-embedded-subscribe.handlers.tools.ts` parses arbitrary exec output with `parseExecApprovalResultText()` and emits an approval event when it parses as denied. Accepting nested metadata increases the set of command-produced strings that can masquerade as trusted approval wrappers.
  - confidence: 0.86
