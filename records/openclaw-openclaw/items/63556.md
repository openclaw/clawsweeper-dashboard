## Summary

Keep open. Current main still lacks the proposed high-level `sessions_send` target-mode guidance in both the model-facing description and public session-tool docs, while the runtime already rejects calls that provide both `sessionKey` and `label`; the PR is a narrow documentation/tool-description clarification that matches current behavior.

## What This Changes

The PR adds `sessions_send` target-selection guidance to `src/agents/tool-description-presets.ts` and `docs/concepts/session-tool.md`, clarifying that callers should use either `sessionKey` or `label` and prefer `sessionKey` when the exact target is known.

## Best Possible Solution

Land this narrow clarification as-is if maintainers are comfortable with separate PRs, or fold it into #63339 so all `sessions_send` targeting guidance is updated consistently in one review.
