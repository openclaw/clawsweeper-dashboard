## Summary

Keep open. Current main still has runtime/in-memory tool-result pairing repair and a Codex-specific durable missing-output repair, but it does not provide the requested provider-generic append-only disk repair for missing tail tool results or an append-only orphan-user recovery marker.

## What This Changes

The issue requests first-party append-only reopen compensation for unresolved tail tool calls and orphan-user tails in persisted session transcripts.

## Best Possible Solution

Add a first-party append-only tail repair before prompt build that synthesizes error toolResults for unresolved tail-only calls and appends one idempotent orphan-user recovery marker without tool re-execution or older-row rewrites.

## Reproduction Assessment

Yes, at source level. Current main keeps generic trailing assistant toolCall entries unrepaired on disk, while orphan-user handling still returns removeLeaf=true and rewinds the active leaf through branch/resetLeaf.

## Solution Assessment

Yes. A bounded append-only disk/reopen repair is narrower than relying on broad replay sanitization because it makes persisted recovery explicit without re-executing tools or rewriting older history.
