## Summary

Keep open. Current main has partial runtime and code-mode repairs, but the durable session-file repair still excludes provider-generic tail tool calls and orphan-user handling still removes or rewinds the leaf instead of appending an explicit recovery marker.

## What This Changes

The issue requests append-only, idempotent reopen repair for unresolved tail tool calls and persisted orphan-user tails.

## Best Possible Solution

Add a narrow durable repair pass before prompt build that appends missing synthetic error tool results for unresolved tail-only calls and one idempotent orphan-user recovery marker without re-executing tools or rewriting older rows.

## Reproduction Assessment

Yes, at source level. Current tests preserve a generic trailing assistant `toolCall` without a `toolResult`, and the orphan-user merge path returns `removeLeaf: true` before `branch` or `resetLeaf` removes the active leaf.

## Solution Assessment

Yes. The requested tail-only, append-only durable repair is the narrowest maintainable direction; using the broad replay sanitizer directly as a write repair would be riskier because it can move or drop non-tail history.
