Keep open. Current main still only durably inserts missing tool results for the OpenAI Codex code-mode path, while generic trailing assistant tool calls are explicitly preserved and orphan-user handling still relies on prompt-build merge/removal rather than append-only recovery rows.

## What This Changes

The issue requests first-party durable reopen compensation for unresolved tail tool calls and orphan user tails in session transcripts.

## Best Possible Solution

Add a durable, tail-only reopen repair before prompt build that appends missing synthetic error toolResult rows and one explicit orphan-user recovery marker without re-executing tools or deleting prior transcript rows.

## Solution Assessment
