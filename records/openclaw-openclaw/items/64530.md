## Summary

Keep open. Current main has useful partial repairs for code-mode persisted results and model-context replay, but it still lacks the requested durable, provider-generic tail repair and explicit orphan-user recovery marker.

## What This Changes

The issue asks OpenClaw to add durable, append-only reopen repair for unresolved tail tool calls and orphan user tails, with idempotent upstream tests.

## Best Possible Solution

Add a narrow durable reopen repair pass that appends missing synthetic error `toolResult` rows for unresolved tail tool calls and appends one explicit orphan-user recovery marker without deleting prior rows or re-executing tools.

## Reproduction Assessment

Yes. Current source and tests show the provider-generic trailing assistant `toolCall` case remains byte-preserved on disk, and orphan-user handling still removes or rewinds the leaf during prompt build rather than appending a durable marker.

## Solution Assessment

Yes. The requested tail-only, append-only durable repair is the narrowest maintainable direction; reusing the broad replay sanitizer directly would be riskier because it can move or drop non-tail history.
