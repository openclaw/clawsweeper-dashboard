## Summary

Keep open. Current main still only writes durable missing-tool-result repairs for OpenAI Codex code-mode entries, while provider-generic trailing assistant tool calls are explicitly preserved and orphan-user tails are still rewound through branch/resetLeaf instead of getting an append-only recovery marker.

## What This Changes

Request first-party append-only reopen repair for unresolved tail tool calls and orphan user tails in session transcripts.

## Best Possible Solution

Add a first-party append-only session-file/reopen repair for tail-only missing toolResults plus an idempotent orphan-user recovery marker, with regression coverage and no tool re-execution.

## Reproduction Assessment

Yes, at source level. Current tests preserve a generic trailing assistant toolCall without a toolResult unchanged, and orphan-user handling still returns removeLeaf=true before branch/resetLeaf rewinds the active leaf.

## Solution Assessment

Yes. A bounded pre-load/tail repair that appends synthetic error toolResults and one idempotent orphan-user marker is narrower than applying the broad replay sanitizer as a durable rewrite because it avoids moving or deleting older history.
