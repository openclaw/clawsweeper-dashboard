## Summary

Keep this issue open. Current main still has useful in-memory replay repair and live pending-tool guards, but it does not add the requested durable reopen/startup seam that appends idempotent recovery rows for unresolved tail tool calls while preserving existing transcript rows. The related open PR #74687 is adjacent restart-recovery work, not a replacement for this transcript-file repair.

## What This Changes

Add first-party reopen/startup transcript compensation that append-only repairs unresolved tail tool calls and explicitly preserves orphan user tails.

## Best Possible Solution

Add a narrow runtime/session recovery pass before prompt build that inspects only the active tail, appends parentId-valid synthetic error toolResult rows for eligible unresolved tool calls, appends one explicit assistant recovery marker for a persisted orphan-user tail, and leaves replay sanitization as a model-context fallback.

## Reproduction Assessment

Yes. A high-confidence static reproducer is a session JSONL tail with an assistant toolCall stopReason `toolUse` and no matching toolResult, then reopening that session; current source inspection shows no durable append-only repair before prompt build. I did not execute it because this review is read-only.

## Solution Assessment

No. The current in-memory replay sanitizer and live guard are necessary but not the best complete solution for this issue because they do not make the persisted transcript source of truth healthy after a restart. The safer fix is a tail-only, append-only SessionManager repair seam before orphan-user handling.

## Repair Work Prompt

Implement the official source-level fix for https://github.com/openclaw/openclaw/issues/64530. Add a narrow startup/reopen transcript recovery seam for embedded agent sessions that runs before prompt build and before current orphan-user prompt compensation. It must inspect only the active transcript tail, preserve existing rows, append parentId-valid synthetic error toolResult rows for eligible unresolved assistant tool calls, append one explicit assistant recovery marker for a persisted orphan user tail, be idempotent on repeated reopen, and never re-execute tools. Coordinate with the existing live session-tool-result guard and replay sanitizer so recovery rows are not duplicated. Preserve existing policy that errored/aborted assistant turns should not get synthetic tool results unless the current code already treats that case as eligible. Do not do deep historical transcript rewrites, raw JSONL appends that bypass SessionManager parentId semantics, provider-specific hacks, dependency upgrades, or broad compaction/restart refactors. Add focused tests for missing tail toolResult, partial parallel tool results, repeated reopen no-op, healthy transcript no-op, orphan-user recovery marker append, and repair ordering before orphan-user handling. Add a user-facing CHANGELOG.md fix entry with Thanks @glfruit if the behavior changes shipped transcript recovery.
