## Summary

Keep open: the PR targets a real Gateway/Control UI regression, but current main moved active-minute filtering into the shared entry-selection path, so the branch’s later row-filter exemption would not keep stale main sessions visible after rebase.

## What This Changes

The branch adds a Gateway primary main-session exemption and regression tests so configured main agent sessions survive activeMinutes filtering while other stale session rows stay hidden.

## Best Possible Solution

Move the primary main-session exemption into filterSessionEntries or selectSessionEntries, preserve custom mainKey behavior, cover sync and async sessions.list, and add redacted runtime proof for the dropdown or Gateway RPC path.

## Reproduction Assessment

Yes, at source level. A stale agent:main:main row plus a fresh heartbeat row and positive activeMinutes is removed by current main's filterSessionEntries before either sync or async sessions.list rows are built.

## Solution Assessment

No. The helper semantics are useful, but the exemption must be applied before row construction in filterSessionEntries or selectSessionEntries; this branch's later row-filter hook is obsolete on current main.
