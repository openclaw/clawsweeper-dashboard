## Summary

Keep open: this is a real Gateway session-list regression fix, but the PR is stale against current main and applies the main-session exemption after the filter point that now feeds both sync and async `sessions.list`; contributor real behavior proof is also still missing.

## What This Changes

The PR adds a Gateway helper and regression tests to keep configured agent main sessions in `sessions.list` when `activeMinutes` would otherwise hide stale rows.

## Best Possible Solution

Move the exemption into shared entry selection so sync and async `sessions.list` preserve configured primary main sessions while stale non-main sessions still filter out, with custom `mainKey` coverage and real Gateway or Control UI proof.

## Reproduction Assessment

Yes. A stale `agent:main:main` row plus a fresh heartbeat row and positive `activeMinutes` will be filtered by current main in `filterSessionEntries` before row construction.

## Solution Assessment

No. The helper semantics are useful, but the PR applies the exemption after the current shared entry filter, so the maintainable fix needs to move earlier in selection.
