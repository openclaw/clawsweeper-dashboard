## Summary

Keep open: the linked Gateway session-list bug is still source-reproducible on current main, but this PR needs changes because it applies the exemption at an obsolete row-filter point instead of the current shared entry filter, and contributor real behavior proof is missing.

## What This Changes

The PR adds a Gateway session-list exemption and regression tests so stale top-level agent main sessions remain visible when `activeMinutes` is set.

## Best Possible Solution

Move the primary main-session exemption into the shared entry-selection path, preserve custom `mainKey` semantics, and cover both direct and RPC `sessions.list` behavior.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main`, a fresher heartbeat/main row, and `sessions.list` with positive `activeMinutes` will drop the stale main entry in `filterSessionEntries` before rows are built.

## Solution Assessment

No. The intended predicate and custom-`mainKey` direction are sound, but the PR applies them to an obsolete row-filter location instead of the current shared entry filter used by sync and async listing.
