## Summary

Keep open: the linked bug is still valid on current main, but this PR is stale against the shared Gateway entry-filter path and would not fix async/RPC session listing as written; it also still needs real behavior proof before merge.

## What This Changes

The PR adds Gateway session-list logic and tests so configured primary agent main sessions remain visible when active-session filtering is enabled.

## Best Possible Solution

Move the primary main-session exemption into filterSessionEntries, preserve custom mainKey semantics, cover sync and async sessions.list behavior, then add redacted live Gateway or Control UI proof.

## Reproduction Assessment

Yes, at source level. A stale agent:main:main row plus a fresh heartbeat row and positive activeMinutes will be removed by filterSessionEntries before either sync or async session rows are built on current main.

## Solution Assessment

No. The intended predicate is reasonable after the custom-mainKey fix, but this PR applies it to an obsolete row-level filter instead of the shared entry-selection path used by sync and Gateway RPC listing.
