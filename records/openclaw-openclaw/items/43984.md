## Summary

Keep open: the PR targets a real Gateway session-list regression, but the patch is currently incorrect against main because it applies the primary-main exemption in the old sync row-filter location while current main filters entries in shared selection before both sync and async rows are built. It also still needs contributor-supplied real behavior proof before merge.

## What This Changes

The PR adds a primary agent-session-key helper and regression tests so top-level agent main sessions bypass `activeMinutes` filtering while stale non-main sessions remain hidden.

## Best Possible Solution

Move the primary-main exemption into shared Gateway session-entry selection, cover default and custom `mainKey` for sync and async `sessions.list`, then require redacted real Gateway or Control UI proof before merge.

## Reproduction Assessment

Yes, source-level. A store with a stale `agent:main:main` row plus a fresh `agent:heartbeat:main` row and positive `activeMinutes` is filtered in current `filterSessionEntries` before the Control UI receives rows.

## Solution Assessment

No. The helper semantics and custom `mainKey` coverage are useful, but the exemption needs to live in `filterSessionEntries` or shared selection so both sync and async `sessions.list` preserve primary main sessions before row construction.
