## Summary

Keep open: this PR targets a valid linked Gateway session-list bug, but the branch is now outdated and the patch applies the primary-main exemption after current main has already filtered entries out. It also still lacks contributor after-fix real behavior proof.

## What This Changes

The PR adds a primary-agent main-session exemption and regression tests so stale main sessions survive `activeMinutes` filtering while other stale sessions remain hidden.

## Best Possible Solution

Move the primary main-session exemption into `filterSessionEntries` or `selectSessionEntries`, preserve custom `mainKey` behavior, cover sync and async `sessions.list`, and add redacted runtime proof for the dropdown or Gateway RPC path.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` row plus a fresh heartbeat row and positive `activeMinutes` will be removed by current main's `filterSessionEntries` before either sync or async session rows are built.

## Solution Assessment

No. The helper semantics are the right direction after the custom-`mainKey` fix, but the exemption must run in the shared entry-selection path rather than the obsolete later row-array filter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

- **[P2] Move the exemption into entry filtering:** `src/gateway/session-utils.ts:870-873`
