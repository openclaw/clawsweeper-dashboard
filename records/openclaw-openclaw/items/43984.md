## Summary

Keep open: this targets a real stale-main-session bug, but the PR applies the exemption after current main's shared entry filter has already removed stale rows, so it will not fix the sync or Gateway RPC listing path as written. The contributor also has not provided after-fix real behavior proof.

## What This Changes

The PR adds Gateway session-list logic and regression tests intended to keep configured main agent sessions visible when `activeMinutes` filters stale `sessions.list` results.

## Best Possible Solution

Move the primary main-session exemption into `filterSessionEntries`, preserve custom `mainKey` semantics, and cover both direct and RPC `sessions.list` behavior before asking for redacted live Gateway or Control UI proof.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` entry plus a fresh heartbeat entry and positive `activeMinutes` will be filtered out in `filterSessionEntries` before row construction on current `origin/main`.

## Solution Assessment

No. The intended predicate is reasonable, but this PR applies it to an obsolete row-level filter instead of the current shared entry-selection path used by sync and async listing.
