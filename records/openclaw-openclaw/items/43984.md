## Summary

Keep open: the linked regression is still valid, but this PR is currently conflicting and targets an older row-filtering path that current main no longer uses for the public session list. It also still lacks contributor real behavior proof before merge.

## What This Changes

The PR adds Gateway session-list filtering logic and regression tests so configured main agent sessions remain visible when `activeMinutes` hides other stale sessions.

## Best Possible Solution

Rebase or replace the patch so the primary main-session exemption lives in `filterSessionEntries` or `selectSessionEntries`, preserves custom `mainKey` behavior, and covers both sync and async `sessions.list`.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` row plus a fresh heartbeat row and positive `activeMinutes` is filtered out by current main before Gateway rows or dropdown options are built.

## Solution Assessment

No. The helper semantics and tests are useful, but the current PR applies the exemption too late in an obsolete row-filter location; the maintainable fix belongs in the shared entry-selection filter.
