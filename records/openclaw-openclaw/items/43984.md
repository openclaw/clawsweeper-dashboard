## Summary

Keep open. The underlying regression is still present on current main and this PR is the active implementation path for #43737, but the branch is stale: current main moved `activeMinutes` filtering to the raw `entries` list, while the PR still applies the exemption to the old row-level `sessions` filter.

## What This Changes

The PR adds a primary-agent-session helper, exempts primary agent main sessions from `activeMinutes` filtering, and adds default/custom `mainKey` regression tests.

## Best Possible Solution

Land a rebased gateway fix that applies the primary main-session exemption during the current `entries` active cutoff in `listSessionsFromStore`, before `limit` and row construction, while preserving the custom `mainKey` guard and the default/custom regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with a stale `agent:main:main`, a fresh `agent:heartbeat:main`, and `sessions.list` called with `activeMinutes`, current main drops the stale main entry at the `entries` cutoff before the Control UI dropdown receives rows.

## Solution Assessment

No. The intended behavior and tests are narrow and useful, but this PR applies the exemption to a stale row-level filter shape; the safer fix is to move the same predicate to current main's pre-row `entries` filter.
