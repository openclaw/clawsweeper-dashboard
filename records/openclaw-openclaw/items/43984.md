## What This Changes

The PR adds Gateway session-list filtering logic and tests so configured agent main sessions stay visible when `sessions.list` uses `activeMinutes`.

## Best Possible Solution

Rebase or replace the patch so the primary main-session exemption lives in shared entry filtering, preserves custom `mainKey` behavior, covers sync and async `sessions.list`, and includes real Gateway or Control UI proof.

## Reproduction Assessment

Yes, at source level. Current main filters entries by `updatedAt` inside `filterSessionEntries` before both sync and async session rows are built, so a stale `agent:main:main` row is removed when `activeMinutes` is set.

## Solution Assessment

No. The helper semantics and tests are useful, but the PR applies the exemption in the old row-filter location; the narrow maintainable fix belongs in `filterSessionEntries` or `selectSessionEntries`.
