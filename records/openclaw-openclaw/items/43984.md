## Summary

## What This Changes

The PR adds a primary-agent-session helper to keep configured main agent sessions in `sessions.list` results under `activeMinutes` filtering, with default and custom `mainKey` regression tests.

## Best Possible Solution

Rebase or replace this PR so Gateway applies the primary-main-session exemption in `filterAndSortSessionEntries` before sort, limit, and row hydration, with regression tests and a single-line changelog fix entry.

Yes. A high-confidence source reproduction is a store with stale `agent:main:main`, fresh `agent:heartbeat:main`, and `sessions.list` using a positive `activeMinutes`; current `main` drops the stale main entry at the entry cutoff before rows are built.

## Solution Assessment

No. The intended predicate and custom-`mainKey` test direction are sound, but the PR applies them to an obsolete row-filter shape; the maintainable fix belongs in the current entry-level filter shared by sync and async listing.
