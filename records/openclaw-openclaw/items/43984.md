## Summary

Keep open: the underlying main-session visibility bug is still present on current main, but this branch is not mergeable as-is because it applies the exemption after current main's shared entry filter has already removed stale sessions.

## What This Changes

The branch adds Gateway session-list logic and tests intended to keep configured main agent sessions visible when `activeMinutes` filters stale `sessions.list` results.

## Best Possible Solution

Move the primary main-session exemption into the shared Gateway entry-selection filter, preserve custom `mainKey` semantics, and cover both direct and RPC `sessions.list` behavior with focused regression tests and real UI or Gateway proof.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` entry plus a fresh heartbeat entry and positive `activeMinutes` will be filtered in `filterSessionEntries` before row construction.

## Solution Assessment

No. The intended predicate is reasonable, but this PR applies it to an obsolete row-level filter instead of the current shared entry-selection path used by sync and async listing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

- **[P2] Move the exemption into entry filtering:** `src/gateway/session-utils.ts:870-873`

Rebase the predicate into the shared entry filter or stale main sessions will still disappear through both sync and RPC listing.
