## Summary

Keep open: the linked regression is still source-reproducible on current main, but this PR is stale against the current Gateway session-listing path and needs rebase/rework plus contributor real behavior proof before merge.

## What This Changes

The PR adds a Gateway helper and regression tests to keep configured agent main sessions in `sessions.list` when `activeMinutes` filters stale sessions.

## Best Possible Solution

Put the exemption in shared entry selection so sync and async `sessions.list` preserve configured main sessions while stale non-main rows still filter out, with custom `mainKey` coverage and real Gateway or Control UI proof.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` row plus a fresher heartbeat row and positive `activeMinutes` will be filtered in `filterSessionEntries` before row construction on current main.

## Solution Assessment

No. The helper semantics and custom `mainKey` test are useful, but the PR applies the exemption after the current shared entry filter, so the maintainable fix needs to move earlier in selection.
