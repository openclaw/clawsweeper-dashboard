## Summary

Keep open: the linked regression is still present on current main, and this PR is useful but currently applies the exemption in the old row-filtering location, so it needs rework before merge.

## What This Changes

The PR adds Gateway session-list filtering logic and regression tests so configured agent main sessions remain visible when `activeMinutes` filters stale sessions.

## Best Possible Solution

Move the primary-main exemption into shared entry filtering, cover default and custom `mainKey` for sync and async `sessions.list`, and require real Gateway or Control UI proof.

## Reproduction Assessment

Yes, source-level. A stale `agent:main:main` entry plus a fresh heartbeat entry with positive `activeMinutes` is filtered by current main before rows or the dropdown can see the main session.

## Solution Assessment

No. The helper semantics are useful, but the exemption must move into `filterSessionEntries` or shared selection so both sync and async `sessions.list` preserve primary main sessions.
