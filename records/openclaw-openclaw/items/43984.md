## Summary

Keep open: the bug remains source-reproducible on current main and this PR is still the linked implementation candidate, but the patch is incorrect against current main because the exemption targets the old row-filter location and the contributor has not provided real behavior proof.

## What This Changes

The PR adds Gateway session-list filtering logic and regression tests so configured agent main sessions remain visible when `activeMinutes` filters stale sessions.

## Best Possible Solution

Move the primary-main exemption into shared Gateway session-entry selection, cover default and custom `mainKey` for sync and async `sessions.list`, and require real Gateway or Control UI proof before merge.

## Reproduction Assessment

Yes, source-level. In current main, a stale `agent:main:main` row plus a fresh `agent:heartbeat:main` row with positive `activeMinutes` loses the main entry in `filterSessionEntries` before the Control UI dropdown receives rows.

## Solution Assessment

No. The helper semantics are useful and the custom `mainKey` edge case was addressed, but the exemption needs to live in `filterSessionEntries` or shared selection so both sync and async `sessions.list` preserve primary main sessions.

## Review Findings
