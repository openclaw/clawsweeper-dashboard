## Summary

Keep open: this PR targets a real source-reproducible Gateway bug, but the patch applies the exemption after session rows are built while current main filters entries earlier, and contributor real behavior proof is still missing.

## What This Changes

The PR adds a Gateway session-list exemption so configured primary agent main sessions stay visible when activeMinutes filters other stale sessions, with regression tests for default and custom mainKey cases.

## Best Possible Solution

Move the primary main-session exemption into filterSessionEntries or the shared selectSessionEntries path, preserve custom mainKey semantics, cover sync and async sessions.list behavior, and add redacted live Gateway or Control UI proof.

## Reproduction Assessment

Yes, at source level. A stale agent:main:main row plus a fresh heartbeat row and positive activeMinutes will be removed by filterSessionEntries before either sync or async session rows are built on current main.

## Solution Assessment

No. The predicate is the right direction after the custom-mainKey fix, but the patch applies it to a later row-array filter instead of the shared entry filter that now controls sync and Gateway RPC listing.
