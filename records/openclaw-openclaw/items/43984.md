## Summary

Keep this PR open: it targets a real open regression in #43737, but the branch is stale against current main and applies the active-session exemption to an older row-level filter shape instead of the current entry-level cutoff.

## What This Changes

The PR adds a primary-agent-session helper, exempts primary agent main sessions from `activeMinutes` filtering, and adds default/custom `mainKey` regression tests in gateway session utilities.

## Best Possible Solution

Rebase or replace this PR so the Gateway applies a primary-main-session exemption inside `filterAndSortSessionEntries` before sort, limit, and row hydration, with default/custom `mainKey` tests and a `CHANGELOG.md` Fixes entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a store containing stale `agent:main:main`, fresh `agent:heartbeat:main`, and `sessions.list` with a positive `activeMinutes`; current main drops the stale main entry at the entry cutoff before the Control UI can list it.

## Solution Assessment

No. The intended predicate and regression tests are the right direction after the custom `mainKey` fix, but the current PR applies them to the pre-refactor row filter; the maintainable fix is to move the predicate to the current entry-level filter shared by sync and async listing.
