## Summary

Keep open: the PR adds a useful new ContextEngine capability that is not on current main, but it is not merge-ready because the external real-behavior proof is still test-only and the diff has actionable contract/validation issues.

## What This Changes

This PR adds optional `currentTokenCount` support to `ContextEngine.assemble`, threads estimates through PI and Codex assemble paths, re-exports `estimatePrePromptTokens`, and updates docs/tests.

## Best Possible Solution

Merge a single additive token-count contract only after maintainer API approval, live long-session proof, sanitized-message token semantics, and Plugin SDK API baseline alignment.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: this PR adds a new public ContextEngine capability. The motivating downstream overflow is described but not reproduced in a live OpenClaw run here.

## Solution Assessment

Unclear pending maintainer API judgment and live proof. The approach is plausible, but the current diff should first align the token count with the sanitized assemble input and update the public SDK API baseline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Count the sanitized assemble messages:** `src/agents/harness/context-engine-lifecycle.ts:79-82`
  - body: `assembleHarnessContextEngine` strips `openclaw.runtime-context` custom messages before invoking the engine, but the new `currentTokenCount` is computed by callers over the original history and then passed through. In sessions with hidden runtime-context messages, engines receive a smaller `messages` array than the one counted, so `remaining = tokenBudget - currentTokenCount` is too low and can wrongly suppress `systemPromptAddition`; compute the estimate after the same strip or make the wrapper own the sanitized count.
  - confidence: 0.83
- **[P2] Update the Plugin SDK API baseline:** `src/plugin-sdk/agent-harness-runtime.ts:184`
  - body: Re-exporting `estimatePrePromptTokens` adds a public symbol to `openclaw/plugin-sdk/agent-harness-runtime`, but this PR does not update `docs/.generated/plugin-sdk-api-baseline.sha256`. The SDK API baseline check is the tracked drift guard for public SDK exports, so regenerate the baseline hash with this new export included.
  - confidence: 0.9
