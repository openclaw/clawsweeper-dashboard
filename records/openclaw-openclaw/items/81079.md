## Summary

Keep open: this is a valid feature/API proposal that is not on current main, but the patch undercounts prompt-visible current-turn context and the external contributor proof is still test-only.

## What This Changes

The branch adds optional `currentTokenCount` support to `ContextEngine.assemble`, wires estimates through PI and Codex harness paths, re-exports the estimator, and updates docs/tests.

## Best Possible Solution

Expose one additive token-headroom value computed from the same known prompt/system/messages that the runtime will submit or precheck, document the exact exclusions, and merge only after live long-session proof is added.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: this PR adds a new public ContextEngine capability. The downstream overflow scenario is described, but the PR body says live provider verification is still pending.

## Solution Assessment

No, not yet: the direction is plausible, but the implementation currently omits prompt-visible current-turn context from the new estimate on PI and Codex paths. The API should also receive maintainer approval and live proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Include current turn context in the PI estimate:** `src/agents/pi-embedded-runner/run/attempt.ts:2553`
  - body: `currentTokenCount` is computed from raw `params.prompt`, but the run later submits `buildCurrentTurnPrompt({ context: params.currentTurnContext, ... })`. When current-turn context carries sizable runtime text, engines will overestimate remaining headroom and can still oversize `systemPromptAddition`, which defeats the new guard this PR is adding.
  - confidence: 0.84
- **[P2] Include current turn context in the Codex estimate:** `extensions/codex/src/app-server/run-attempt.ts:628`
  - body: The Codex path estimates tokens from `params.prompt` and only afterward calls `prependCurrentTurnContext`. That prompt-visible context is part of the model input, so context engines on this harness can receive a `currentTokenCount` that is too low and inject more system prompt than actually fits.
  - confidence: 0.83
