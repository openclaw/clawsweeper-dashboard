## What This Changes

ACP thread-bound Discord follow-up replies are generated by the Claude ACP runtime but are not posted back to the bound Discord thread.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/81341 so direct follow-up turns from a spawned ACP session's own bound Discord thread deliver only when the saved delivery context matches the current thread, account, and target.

## Reproduction Assessment

Yes, at source level. Current main still suppresses parent-owned ACP child user delivery before `dispatch-acp-delivery` can route visible output, and the issue plus PR proof show the runtime generated a reply while Discord received none.
