## Summary

Keep open. Current main and the latest release still route Feishu Ogg/Opus voice media to `uploadFileFeishu` without a `duration`, and the related open PR targets the right bug but remains unmerged and has review blockers.

## What This Changes

The issue reports that Feishu voice messages sent through the `message` tool with an Opus media file arrive without a visible duration in the Feishu UI.

## Best Possible Solution

Land a narrow Feishu plugin fix that computes a finite millisecond duration from the actual uploaded native or transcoded audio/video bytes and passes it through `uploadFileFeishu` while preserving existing routing and metadata behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path from message-tool `media` / `asVoice` to `sendMediaFeishu`, where Ogg/Opus audio reaches `uploadFileFeishu` without `duration`; I did not perform a live Feishu UI send.

## Solution Assessment

Yes for a narrow Feishu media fix, but not by closing this issue now. The existing open PR targets the bug but is broader than necessary and has review blockers, so a refreshed or replacement patch is the safer implementation path.
