## Summary

Keep open. Current main and the latest release still route Feishu Ogg/Opus voice media through `uploadFileFeishu` without computing or passing `duration`, while the overlapping duration PRs were closed unmerged and this remains the canonical narrow bug.

## What This Changes

Feishu voice messages sent through the message tool should include upload duration metadata so the Feishu UI shows the audio length.

## Best Possible Solution

Land a narrow Feishu plugin fix that computes a finite millisecond duration for final native or transcoded audio/video bytes and passes it to `uploadFileFeishu`, with regression coverage that preserves existing media routing and fallback behavior.

## Reproduction Assessment

Yes, source-level. The message tool reads `media` plus `asVoice`, reaches `sendMediaFeishu`, routes Ogg/Opus as Feishu audio, and uploads without the duration field that the Feishu upload contract supports.

## Solution Assessment

Yes. The maintainable fix is a narrow Feishu media change that derives duration from the final uploaded bytes and threads it through the existing `uploadFileFeishu` parameter, not a broad routing rewrite or synchronous shell-string probe.
