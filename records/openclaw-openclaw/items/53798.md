Keep open. Current main still sends Feishu Ogg/Opus voice media through the audio upload path without deriving or passing `duration`, and the related duration PRs are closed unmerged.

## What This Changes

The issue requests that Feishu voice messages sent through the `message` tool include audio duration metadata so Feishu displays the normal time indicator.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives finite millisecond duration from the final uploaded voice media and passes it through the existing `uploadFileFeishu.duration` field with focused regression coverage.

## Reproduction Assessment

Yes at source level: the message action reaches `sendMediaFeishu`, Ogg/Opus routes as `msgType: "audio"`, and the final upload call omits `duration`. I did not run a live Feishu client reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to compute duration inside the Feishu plugin after final media preparation and pass it into the existing upload helper, without changing core message-tool contracts.
