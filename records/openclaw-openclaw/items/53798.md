## Summary

Keep open. Current main and v2026.5.22 still send Feishu Ogg/Opus voice media through `uploadFileFeishu` without computing or passing `duration`, while the SDK contract and OpenClaw upload helper already support that field and the overlapping duration PRs were closed unmerged.

## What This Changes

The issue requests that Feishu voice messages sent via the message tool include audio duration metadata so the Feishu UI shows the normal duration indicator.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives finite millisecond duration from the final uploaded audio/video bytes and passes it through the existing `uploadFileFeishu` field with focused regression coverage.

## Reproduction Assessment

Yes, at source level. The message action reads `media` and optional `asVoice`, Feishu routes Ogg/Opus as `msg_type: "audio"`, and the final `uploadFileFeishu` call currently omits `duration`.

## Solution Assessment

Yes. The narrow maintainable fix is to compute duration after `prepareFeishuVoiceMedia` and pass it into the existing helper, while preserving current routing and download metadata behavior.
