## Summary

Keep open. Current main and latest release still have the Feishu upload API sink for `duration`, but `sendMediaFeishu` does not compute or pass it for native Ogg/Opus voice uploads; the related fix PR was closed unmerged as superseded by this canonical issue.

## What This Changes

The issue asks Feishu message-tool voice sends to include audio duration metadata so Feishu displays the duration indicator.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives a finite millisecond duration from the actual uploaded native or transcoded audio/video bytes and passes it through `uploadFileFeishu` while preserving current routing, metadata, and fallback behavior.

## Reproduction Assessment

Yes, source-level rather than live. The message action reads `media` and `asVoice`, reaches `sendMediaFeishu`, routes native Ogg/Opus as Feishu audio, and uploads without `duration`.

## Solution Assessment

Yes, a narrow Feishu media fix is the best path. The closed related PR was broader than needed, so a replacement should preserve the current router and add focused duration propagation tests.
