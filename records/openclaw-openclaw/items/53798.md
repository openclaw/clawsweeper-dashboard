## Summary

Keep open. Current main and the latest release still route Feishu Ogg/Opus voice media through `uploadFileFeishu` without computing or passing `duration`, while the upload helper and upstream SDK contract support the field and the overlapping duration PRs are closed unmerged.

## What This Changes

Requested fix for Feishu message-tool voice media to include audio duration metadata so the Feishu chat UI displays playback length.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives a finite millisecond duration from the final uploaded audio/video bytes and passes it through the existing `uploadFileFeishu` field with fallback coverage.

## Reproduction Assessment

Yes, source-level rather than live UI: the message action reads `media` and `asVoice`, calls Feishu outbound `sendMedia`, routes Ogg/Opus as audio, then invokes `uploadFileFeishu` without `duration`.

## Solution Assessment

Yes. The maintainable fix is to compute duration after `prepareFeishuVoiceMedia` and pass it into the existing helper; avoid a broad routing rewrite or a stale pre-transcode duration.
