## Summary

Keep open. Current main still has the Feishu voice-message path described by the reporter: `media` plus `asVoice` can reach native Ogg/Opus audio delivery, and `uploadFileFeishu` can forward a duration, but `sendMediaFeishu` never computes or passes one. The related PR #50399 targets this bug but is open and unmerged, so it is not implementation evidence.

## What This Changes

The issue requests that Feishu voice messages sent through the message tool with Opus/Ogg media include an audio duration so the Feishu UI shows the usual time label.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin with a narrow media-path change: derive a finite millisecond duration for native Ogg/Opus uploads and successful audioAsVoice transcode outputs, pass it through the Feishu upload/send payloads where the API expects it, and add focused regression tests for native voice media and transcode voice media.
