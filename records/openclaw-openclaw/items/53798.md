## Summary

Keep open. Current main still has the Feishu voice-media path reported here, and the outbound upload call still does not compute or pass `duration`; related PR #50399 targets this bug but remains open and unmerged, so it is not implementation evidence.

## What This Changes

Feishu voice or Opus media sent through the `message` tool should include audio duration metadata so the Feishu UI shows the normal voice-message duration.

## Best Possible Solution

Fix this in the bundled Feishu plugin with a narrow media-path change: compute a finite millisecond duration for native Ogg/Opus voice uploads and successful `audioAsVoice` transcode outputs, pass it to `uploadFileFeishu`, and add focused regression tests proving `client.im.file.create` receives `duration` on those voice paths.

## Reproduction Assessment

Yes. An instrumented `sendMediaFeishu` call with `.opus` media follows the reported path and currently reaches `client.im.file.create` without `duration`; a live Feishu UI check would confirm the visible duration symptom.

## Solution Assessment

Yes, the right boundary is the Feishu plugin media send path, not core message-tool behavior. The safer fix should preserve current media routing and local-root handling while adding duration calculation and tests rather than broadening core channel contracts.
