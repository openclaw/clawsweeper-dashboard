## Summary

Keep open. Current main and the latest release still route Feishu Opus/Ogg voice media to an audio upload without passing `duration`, while Feishu's upload contract says omitted duration prevents concrete duration display; an open related PR targets the bug but remains unmerged and not landable as-is.

## What This Changes

Feishu voice messages sent through the message tool should include audio duration metadata so the Feishu UI displays the playback length.

## Best Possible Solution

Land a narrow current-main Feishu media fix, or a replacement for the related PR, that computes a finite millisecond duration from the actual uploaded native or transcoded audio/video bytes and passes it through `uploadFileFeishu` without weakening existing routing or metadata behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path from message-tool `media` / `asVoice` to `sendMediaFeishu`, where Opus/Ogg audio reaches `uploadFileFeishu` without a `duration` field; I did not perform a live Feishu UI send.

## Solution Assessment

Yes for the narrow Feishu plugin fix, but not by closing this issue now. The existing open PR is not the best merge shape as-is, so maintainers should land a smaller current-main patch or replace that branch.
