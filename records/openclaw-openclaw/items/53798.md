## Summary

Keep open. Current main and the latest release still route Feishu Ogg/Opus voice media through the audio upload path without deriving or passing `duration`, while the related duration PRs are closed unmerged and the duplicate issue was closed in favor of this canonical report.

## What This Changes

The issue requests that Feishu voice messages sent via the message tool include audio duration metadata so the Feishu UI shows the usual duration indicator.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives a finite millisecond duration from the final uploaded audio/video media and passes it through the existing `uploadFileFeishu.duration` field with focused regression coverage.

## Reproduction Assessment

Yes at source level: the message action reaches `sendMediaFeishu`, Ogg/Opus routes as `msgType: "audio"`, and the final upload call omits `duration`. I did not perform a live Feishu client send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to compute duration inside the Feishu plugin after final media preparation and pass it into the existing upload helper, without changing core message-tool contracts or broadening the audio routing policy.
