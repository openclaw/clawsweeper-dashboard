## Summary

Keep open. Current main and the latest release still route the reported Feishu Opus voice send through `sendMediaFeishu` without computing or passing `duration`; the overlapping fix PR was closed unmerged in favor of this canonical issue.

## What This Changes

The issue reports that Feishu voice messages sent through the message tool deliver as audio but omit the duration shown by the Feishu client.

## Best Possible Solution

Land a narrow Feishu plugin fix that derives a finite millisecond duration from the final uploaded media bytes and passes it through `uploadFileFeishu`, while preserving current routing, metadata, and fallback behavior.

## Reproduction Assessment

Yes, source-level rather than live. The message action reads `media` plus `asVoice`, reaches `sendMediaFeishu`, routes Opus/Ogg as Feishu audio, and uploads without the duration that Feishu expects for display.

## Solution Assessment

Yes. A narrow Feishu media fix that computes duration from the final bytes and passes the existing `duration` field is the maintainable path; the prior PR's broader routing and synchronous probe changes were not the best branch shape.
