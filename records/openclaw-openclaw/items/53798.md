## Summary

Keep open. Current main still has the reported Feishu voice-media path: `.opus` / `.ogg` routes to a Feishu audio message, but the upload call omits `duration`; related PR #50399 targets this bug but remains open and unmerged.

## What This Changes

The issue asks Feishu voice media sent through the message tool to include audio duration metadata so Feishu shows the clip length.

## Best Possible Solution

Review and land or replace #50399 with a narrow Feishu plugin fix that computes a finite millisecond duration for native and transcoded voice uploads and asserts `client.im.file.create` receives it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path from `message` tool `media` / `asVoice` to Feishu `sendMediaFeishu`, where `.opus` / `.ogg` routes to audio upload without any `duration` field; I did not perform a live UI send.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the bundled Feishu plugin media path by calculating and passing duration, without changing the core message-tool contract or adding a new config knob.
