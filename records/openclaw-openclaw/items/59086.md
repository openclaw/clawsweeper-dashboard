## Summary

Keep open. Current main has not implemented the central structured live-reasoning chat path, but this external PR is not merge-ready because it lacks real behavior proof, is conflicting, and introduces a reasoning visibility/security regression plus UI formatting/rendering bugs.

## What This Changes

The branch adds Gateway chat deltas for streamed thinking events and Control UI structured streaming/rendering for interleaved thinking, text, and tool blocks.

## Best Possible Solution

Land one coordinated, rebased implementation that preserves visibility and capability gates, carries structured thinking/text through Gateway chat and Control UI rendering, and includes redacted live behavior proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path for the PR-head defects: thinkingLevel="off" would no longer block streamed thinking, hidden thinking-only stream messages can suppress the loading indicator, and double Markdown markers are stripped incorrectly.

## Solution Assessment

No, not as-is. The direction is useful, but the branch needs a rebase that keeps the current visibility gate, fixes the UI/Markdown defects, supplies real behavior proof, and coordinates with the active reasoning-stream work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:53`
  - body: Current main gates streamed reasoning on canShowReasoning so thinkingLevel="off" suppresses thinking. The PR head sets streamReasoning from reasoningMode alone, so rebasing this over main would emit thinking events for sessions where reasoning visibility is disabled.
  - confidence: 0.9
- **[P2] Keep hidden reasoning streams as a loading indicator:** `ui/src/ui/views/chat.ts:1552-1554`
  - body: This creates a stream item whenever streamMessage exists. If that message only has thinking blocks and showReasoning is false, renderGroupedMessage suppresses the content and the reading-indicator fallback is skipped, leaving an empty assistant row until visible text arrives.
  - confidence: 0.9
- **[P3] Avoid stripping double Markdown markers:** `ui/src/ui/chat/message-extract.ts:127-130`
  - body: The single-marker checks treat **bold** and __text__ as single-wrapped lines, slice one marker from each side, and then rewrap them, corrupting intentionally formatted reasoning. Exclude double-marker prefixes and suffixes before slicing.
  - confidence: 0.83
