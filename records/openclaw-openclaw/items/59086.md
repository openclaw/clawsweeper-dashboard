## Summary

Keep open: current main still does not carry live reasoning blocks through the Gateway chat event path into Control UI state/rendering, so the central behavior is not implemented. The branch is also not merge-ready as-is because it introduces at least one user-visible streaming regression, and the overlapping #68982 discussion points toward maintainer consolidation rather than a clear superseding fix.

## What This Changes

The PR streams assistant thinking events through Gateway chat payloads, adds structured Control UI streaming state/rendering for interleaved thinking/text/tool blocks, reorders local agent-event callbacks before global fan-out, and adds focused agent, gateway, and UI tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through Gateway chat delta/final payloads, Control UI stream state, item building, and streaming rendering while preserving `/reasoning` visibility semantics and hidden-reasoning reading indicators.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: at PR head, a reasoning-only `streamMessage` with `showThinking` off creates a stream item at `ui/src/ui/views/chat.ts:1554` but renders no visible bubble, and current main confirms the underlying live path is still text-only.

## Solution Assessment

No, not as-is. The PR covers more of the end-to-end path than the smaller overlapping UI PR, but it needs rebasing, hidden-reasoning rendering fixes, and consolidation with the current Gateway/UI stream-state work before merge.
