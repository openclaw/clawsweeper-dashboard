## Summary

Keep open for maintainer review. The PR addresses a real gap in live reasoning/chat rendering, but it is external-authored without after-fix real behavior proof, has a user-visible hidden-reasoning streaming regression, and overlaps several active reasoning-stream branches.

## What This Changes

The PR routes streamed assistant thinking through Gateway chat events and adds structured Control UI rendering for interleaved thinking, text, and tool stream blocks.

## Best Possible Solution

Land one rebased, consolidated implementation that carries structured thinking/text payloads through agent events, Gateway chat deltas/finals, Control UI state, and rendering while preserving reasoning visibility gates and hidden-reasoning loading behavior.

## Reproduction Assessment

Yes, by source inspection. Current main’s live chat path is text-only, and the PR head has a clear hidden-reasoning path where a thinking-only streamMessage suppresses the reading indicator when Show Thinking is off.

## Solution Assessment

No, not as-is. The direction is maintainable, but this branch needs the hidden-reasoning loading fix, the Markdown normalizer fix, real behavior proof, and reconciliation with active reasoning-stream PRs before merge.
