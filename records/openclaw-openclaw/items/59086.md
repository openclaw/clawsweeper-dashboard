## Summary

Keep open: current main still does not carry reasoning/thinking blocks end to end through live Gateway chat and Control UI streaming, so this PR is not obsolete. It is not ready to merge as-is because the external branch lacks real behavior proof, has a hidden-reasoning empty-row regression, and overlaps active reasoning-stream work.

## What This Changes

The branch routes reasoning/thinking stream events through agent and Gateway chat deltas and adds structured Control UI stream state/rendering for interleaved thinking, text, and tool blocks.

## Best Possible Solution

Land one rebased, coordinated implementation that carries structured thinking/text payloads through agent events, Gateway chat deltas/finals, Control UI state, stream segmentation, and rendering while preserving reasoning visibility gates and hidden-reasoning loading behavior.

## Reproduction Assessment

Yes, by source inspection. Current main’s live Control UI chat path is text-only, and the PR head has a clear hidden-reasoning path where a thinking-only streamMessage suppresses the reading indicator when Show Thinking is off.

## Solution Assessment

No, not as-is. The direction is maintainable, but the branch needs the hidden-reasoning loading fix, the Markdown normalizer fix, real behavior proof, and reconciliation with active reasoning-stream PRs before merge.
