## Summary

Keep open: current main still has text-only live chat streaming, while this PR has a user-visible hidden-reasoning streaming regression and lacks after-fix real behavior proof. The remaining work overlaps active follow-ups at https://github.com/openclaw/openclaw/pull/68982 and https://github.com/openclaw/openclaw/issues/42276, so the best next step is consolidation rather than cleanup closure.

## What This Changes

The PR forwards assistant reasoning events through Gateway chat and Control UI streaming state so interleaved thinking, text, and tool blocks can render in order.

## Best Possible Solution

Land one rebased, consolidated implementation that carries structured thinking/text payloads through agent events, Gateway chat deltas/finals, Control UI state, and rendering while preserving `/reasoning` visibility semantics and hidden-reasoning loading behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main emits and stores live chat as text only, and the PR branch can render an empty assistant stream row when a reasoning-only stream message is hidden by the Show Thinking setting.

## Solution Assessment

No, not as-is. The direction is maintainable, but the branch needs the hidden-reasoning loading fix, the Markdown normalizer fix, real behavior proof, and reconciliation with the overlapping active reasoning-stream PRs before it is the best merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep the reading indicator when reasoning is hidden:** `ui/src/ui/views/chat.ts:1549-1559`
  - body: This branch adds a stream item whenever `props.streamMessage` exists. If that payload only contains thinking blocks and `showReasoning` is false, `renderStreamingGroup` suppresses the content, so the reading-indicator fallback is skipped and an empty assistant row renders until visible text arrives.
  - confidence: 0.93
- **[P3] Avoid stripping double Markdown markers:** `ui/src/ui/chat/message-extract.ts:127-132`
  - body: The single-marker checks only test the first and last character, so `**bold**` and `__text__` lose one marker on each side before the reasoning formatter re-wraps them. Exclude double-marker prefixes and suffixes before slicing.
  - confidence: 0.82
