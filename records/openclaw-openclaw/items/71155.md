## Summary

Keep this PR open. Current main still drops the stability-bundle error message and stack, and maintainer discussion says this PR is the preferred base over #71352, but the patch is not merge-ready because it expands the shared import/export sanitizer path and can leak raw imported error text into shareable support exports, while also missing parts of its stated safety contract.

## What This Changes

This PR extends diagnostic stability-bundle error metadata with message, stack, and cause fields plus a sanitizer, and updates stability-bundle tests to expect persisted error messages.

## Best Possible Solution

Revise this PR in place so generated startup/restart failure bundles capture useful message, stack, and bounded single-level cause metadata, while imported bundles and support exports continue to enforce the documented diagnostics privacy model. The sanitizer should explicitly strip NUL bytes, redact local paths, cap message/stack/cause text, and have focused tests for generated and imported bundle paths.

## Reproduction Assessment

Yes. Current main has a focused test path that writes a startup-failure bundle from an Error and still asserts the raw error message is absent, and the source shows the serializer only preserves name/code.

## Solution Assessment

No. This PR is the right base, but the implementation should not merge until write-time capture is separated from import/export sanitization, cause remains structured and bounded, and the advertised NUL/length safety contract has coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Redact imported bundle messages before exporting:** `src/logging/diagnostic-stability-bundle.ts:144-155`
  - body: `readSafeErrorMetadata` is also used while parsing imported stability bundles for `gateway stability --bundle ... --export`. Adding `message`, `stack`, and `cause` here lets arbitrary imported error text flow into shareable support zips, despite the docs and tests requiring imported diagnostics to omit payload-like error text. Split generated-bundle capture from imported-bundle sanitization, or redact these fields before export.
  - confidence: 0.88
- **[P2] Preserve cause as bounded metadata:** `src/logging/diagnostic-stability-bundle.ts:152-155`
  - body: `String(cause)` collapses an `Error.cause` into text and drops the useful safe fields the PR promises, such as cause name, message, and code. Serialize one bounded cause object without recursion so the bundle keeps actionable cause details without unbounded growth.
  - confidence: 0.82
- **[P2] Strip NUL bytes from bundle text:** `src/logging/diagnostic-stability-bundle.ts:133-136`
  - body: The sanitizer removes ANSI escapes and truncates text, but it does not remove embedded NUL bytes even though the PR describes NUL stripping as part of the safety contract. Add an escaped `\x00` replacement and coverage for message, stack, and cause text.
  - confidence: 0.76
