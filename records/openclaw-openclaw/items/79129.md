## Summary

Keep open: current main still routes plain `status --json` through channel-presence metadata discovery, and the report includes current-release live timing and strace evidence. The related reports cover adjacent plugin metadata hot paths, but not this CLI JSON repro specifically.

## What This Changes

The issue requests that `openclaw status --json --timeout 1000` avoid repeated plugin metadata and manifest discovery and complete in bounded time on 2026.5.7.

## Best Possible Solution

Keep the JSON status path on a single prepared metadata view or a cheaper configured-channel predicate, and only perform broad manifest discovery for `--all`, deep diagnostics, or paths that truly need manifest-owned channel details.

## Reproduction Assessment

Yes at source level: `status --json` reaches `scanStatusJsonFast()`, which calls channel-presence policy code that loads plugin manifest records when no prepared records are supplied. I did not rerun the reporter's timing benchmark in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to remove or amortize manifest discovery from plain JSON status while preserving installed-index freshness. A broad persistent metadata cache would be the riskier solution.

## Repair Work Prompt

Fix the CLI performance bug described in https://github.com/openclaw/openclaw/issues/79129: plain `openclaw status --json --timeout 1000` should not repeatedly or broadly discover bundled plugin manifests before returning a small gateway/status JSON payload. Current main routes through `src/commands/status-json.ts` -> `src/commands/status.scan.fast-json.ts`, where `hasConfiguredChannelsForReadOnlyScope()` can load installed channel manifest records through `src/plugins/channel-presence-policy.ts` and `src/plugins/plugin-registry-contributions.ts`. Keep the fix narrow: prefer passing/reusing one prepared metadata or manifest view inside the status scan, or use a cheaper configured-channel predicate for plain JSON status; preserve installed-index freshness and do not add a persistent global metadata cache. Add focused regression coverage showing plain status JSON does not invoke broad manifest/registry loaders unnecessarily while preserving configured-channel and env-only channel detection semantics. Check related reports https://github.com/openclaw/openclaw/issues/77348, https://github.com/openclaw/openclaw/issues/78100, and https://github.com/openclaw/openclaw/issues/78461 before implementation to avoid duplicating broader gateway/plugin-metadata work. Add a user-facing CHANGELOG.md fix entry if behavior changes.
