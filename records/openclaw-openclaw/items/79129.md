The observable bug is that lean `openclaw status --json --timeout 1000` on v2026.5.7 can spend tens of seconds repeatedly scanning plugin metadata/manifests even though it should stay on a fast read-only path. Keep the fix narrow: make the JSON status path avoid duplicate plugin metadata/manifest registry construction while preserving SecretRef resolution and channel presence semantics.
