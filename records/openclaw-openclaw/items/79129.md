## Summary

Keep open. Current main still routes lean `status --json` through a configured-channel check that can build installed channel manifest records from the plugin registry before channel rows are skipped, so the 2026.5.7 report is not covered by the related gateway-side fixes.

## What This Changes

Avoid repeated plugin registry and manifest discovery in `openclaw status --json --timeout 1000` so JSON status completes in bounded time on current installs.

## Best Possible Solution

Keep this issue open and make the lean JSON status path avoid manifest-registry construction unless `--all` or a truly manifest-backed channel answer is required.
