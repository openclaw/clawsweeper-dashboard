## Summary

Keep open, but not merge-ready: the constrained bridge serializers do not currently expose or prove `structuredClone`, so object and array tool arguments can be converted to strings and dropped; the PR also lacks real behavior and benchmark proof.

## What This Changes

The PR replaces JSON round-trip projection with `structuredClone()` in the Tool Search code-mode child and QuickJS code-mode worker serializers.

## Best Possible Solution

Keep bridge serializers JSON-safe, or explicitly load and prove `structuredClone` support in both constrained runtimes with object/array bridge coverage and timing/RSS evidence.

## Reproduction Assessment

Yes for the PR-introduced Tool Search failure path: a focused Node VM check matching `vm.createContext(Object.create(null))` shows `structuredClone` is undefined and array args become a non-array string. The QuickJS path is source/dependency-proven because OpenClaw does not load the structured-clone extension.

## Solution Assessment

No. A direct replacement is not the best fix unless both constrained runtimes are made to expose `structuredClone` while preserving the existing JSON-safe bridge contract and proving the claimed performance gain.
