## Summary

Keep open. The patch is not merge-ready because the changed bridge serializers run inside guest contexts where the JSON-safe projection is part of the delivery contract, and replacing it with `structuredClone` can turn object/array bridge payloads into strings.

## What This Changes

The PR replaces JSON round-trip cloning with `structuredClone` in code-mode worker serialization and tool-search result normalization.

## Best Possible Solution

Keep the bridge serializers JSON-safe, or introduce a replacement proven to work inside both Node VM and QuickJS-WASI with focused bridge regression tests and benchmark proof.

## Reproduction Assessment

Yes for the PR-introduced bug: source inspection plus a Node VM check show `structuredClone` is unavailable in the tool-search child context, which makes the patched fallback stringify object/array bridge payloads. I did not run the full test suite because this review is read-only.

## Solution Assessment

No. Replacing JSON projection with `structuredClone` is not the best fix here because the bridge contract is JSON-safe payload delivery across constrained guest runtimes; any optimization needs runtime proof and regression coverage first.
