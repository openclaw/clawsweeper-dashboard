## What This Changes

The PR replaces two JSON.parse(JSON.stringify(...)) projections in code-mode and tool-search bridge helpers with structuredClone.

## Best Possible Solution

Keep a JSON-safe projection helper for these bridge paths; only optimize it if the replacement preserves the existing JSON/IPC contract and includes focused regression tests plus benchmark proof.

## Reproduction Assessment

Yes. Source inspection plus a local node:vm probe show the patched tool-search helper would not have structuredClone available in the sandbox, and a Node probe shows structuredClone preserves values the surrounding JSON encoders cannot carry unchanged.

## Solution Assessment

No. These helpers are JSON-safe bridge projectors rather than general deep-clone utilities, so the best fix is to preserve that contract and benchmark any safer optimization separately.
