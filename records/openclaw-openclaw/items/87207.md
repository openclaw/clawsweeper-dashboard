## Summary

Keep open. Current main and the latest release still expose `agent_end` as a narrow messages/status/duration payload, no public plugin `turn_end` hook exists, and the related implementation attempts were closed unmerged after maintainer feedback that this needs a generic lifecycle-owned contract.

## What This Changes

Request to expose per-turn tool-call outcomes, tool durations, and completion reason through `agent_end` or a new public plugin `turn_end` hook.

## Best Possible Solution

Keep this issue open as the canonical design request and define a small additive turn-outcome hook contract with privacy gating, normalized tool outcomes, completion reason mapping, parity across supported harnesses, tests, and docs.

## Reproduction Assessment

Yes, by source inspection. Current main and `v2026.6.8` show the public `agent_end` payload lacks the requested fields and no public plugin `turn_end` hook exists.

## Solution Assessment

Unclear until maintainer API design happens. The requested boundary is plausible, but maintainers still need to choose `agent_end` enrichment versus a new `turn_end` hook and define privacy plus completion-reason normalization.
