## Summary

Keep open. Current main still exposes `agent_end` as final messages plus success/error/duration, while adjacent `after_tool_call` and `model_call_ended` hooks do not provide the requested per-turn aggregate tool outcomes or normalized completion reason.

## What This Changes

The issue requests richer per-turn plugin hook payload data for tool-call outcomes, tool durations, and model completion reason on `agent_end` or a new `turn_end` hook.

## Best Possible Solution

Define a small additive turn-outcome contract for plugins, backed by Pi and Codex runtime data, with SDK types, focused contract tests, and docs that steer plugins away from parsing provider-shaped message blocks.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection gives a high-confidence current-state check that `agent_end` does not expose the requested aggregate tool outcomes or completion reason.

## Solution Assessment

Unclear until maintainer API design happens. An additive `agent_end` extension preserves existing subscribers, while a separate `turn_end` hook may be cleaner if maintainers want a new typed turn-outcome contract.
