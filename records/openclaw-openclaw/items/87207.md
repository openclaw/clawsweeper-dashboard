## Summary

Keep open. Current main and v2026.6.8 still expose `agent_end` as messages plus success/error/duration only; adjacent hooks expose partial signals, but not the requested per-turn aggregate tool outcomes and completion reason.

## What This Changes

The issue requests richer `agent_end` fields or a new `turn_end` hook exposing per-turn tool calls, tool results, tool durations, and completion reason for plugins.

## Best Possible Solution

Keep this issue open as the canonical request and design a small additive plugin SDK turn-outcome contract with runner/Codex/provider mapping, privacy rules, types, tests, and docs.

## Reproduction Assessment

Yes for source-level current-state verification, not as a broken-behavior runtime bug. Current main and v2026.6.8 show the public `agent_end` payload lacks the requested fields and no public plugin `turn_end` hook exists.

## Solution Assessment

Unclear until maintainer API design happens. The requested boundary is plausible, but maintainers still need to choose `agent_end` enrichment versus a new `turn_end` hook and define privacy plus completion-reason normalization.
