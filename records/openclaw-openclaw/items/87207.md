## Summary

Keep open. Current main and the latest release still expose `agent_end` as messages plus success/error/duration only; adjacent per-call hooks and internal turn events do not provide the requested public plugin SDK per-turn aggregate tool outcomes or completion reason.

## What This Changes

Request to enrich `agent_end` or add a new public `turn_end` plugin hook carrying typed per-turn tool calls, results, durations, and completion reason.

## Best Possible Solution

Keep this open as the canonical hook API request and design a small additive turn-outcome contract with privacy rules, provider/Codex normalization, SDK types, docs, and focused hook dispatch tests.

## Reproduction Assessment

Yes for source-level current-state verification, not as a broken-behavior runtime bug. Current main and v2026.6.8 show the public `agent_end` payload lacks the requested fields and no public plugin `turn_end` hook exists.

## Solution Assessment

Unclear until maintainer API design happens. The requested boundary is plausible, but maintainers still need to choose `agent_end` enrichment versus a new `turn_end` hook and define privacy plus completion-reason normalization.
