## Summary

Keep open: current main and v2026.6.6 have useful memory search, promotion, REM backfill, provenance-rich Memory Wiki ingestion, and LanceDB store/forget pieces, but not the requested unified remember/recall/review/edit/forget operator lifecycle. The Engram Core comment is useful external context, and the closed retrieval-scoring sibling covers part of recall explanations, but the core schema/provenance/write/review decisions are still open.

## What This Changes

The issue requests a minimal memory command and skill surface for remember, recall, review, edit, and forget, plus ingestion helpers with provenance and match explanations.

## Best Possible Solution

Resolve the canonical memory schema, provenance model, write resolver, and lifecycle semantics first, then expose a small plugin-owned operator surface with core routing only where the memory slot needs it.

## Reproduction Assessment

Not applicable: this issue requests a new unified memory operator and ingestion surface rather than reporting a failure of an existing documented command contract. Source review shows partial current surfaces but no single failing current-main reproduction path.

## Solution Assessment

No as a standalone CLI-first change: implementing the listed commands before the canonical schema, provenance, resolver, review queue, and forget semantics are selected would be the wrong order. The maintainable path is to resolve the open Memory MVP design issues and then expose the small operator surface through the selected memory plugin boundary.
