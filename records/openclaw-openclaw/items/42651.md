## Summary

Keep open: current main still lacks the requested default remember/recall/review/edit/forget operator contract, default write/review/edit/forget lifecycle, and structured recall match explanations. The Engram Core comment and bundled memory-wiki/LanceDB surfaces are useful adjacent implementations, but they do not settle the default OpenClaw v1 memory contract or supersede the open companion memory MVP issues.

## What This Changes

Request to add v1 memory ingestion helpers plus an operator-friendly CLI/skill surface for remember, recall, review, edit, and forget with provenance and match explanations.

## Best Possible Solution

Define one v1 default memory operator contract, then implement the selected CLI/skill/tool surface with conservative write/review/edit/forget semantics, provenance-preserving storage, structured match explanations, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The missing surface is high-confidence from source and docs inspection because current default memory exposes search/get plus promotion/dreaming commands rather than the requested operator verbs.

## Solution Assessment

Unclear until maintainers settle the default memory contract. Adding commands directly is plausible, but the maintainable path is to resolve write, review, edit, forget, provenance, and conflict semantics before exposing public operator verbs.
