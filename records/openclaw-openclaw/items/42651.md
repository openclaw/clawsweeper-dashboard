The remaining work depends on memory write/review product decisions tracked by the related open Memory MVP issues.

## What This Changes

Request a minimal Memory MVP operator surface with ingestion helpers plus remember, recall, review, edit, and forget commands that preserve provenance and match explanations.

## Best Possible Solution

Settle the durable write pipeline in https://github.com/openclaw/openclaw/issues/42648 and review/edit/forget lifecycle in https://github.com/openclaw/openclaw/issues/42650, then add a small memory-plugin-owned CLI/skill surface with provenance, match explanations, revision support, and non-destructive forget behavior.
