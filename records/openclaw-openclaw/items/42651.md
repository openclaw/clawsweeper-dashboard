## Summary

Keep open. Current main has substantial adjacent memory work, including search, promotion, dreaming backfill, memory-wiki, and an alternate LanceDB plugin, but it still does not provide the requested default remember/recall/review/edit/forget operator surface or structured recall match explanations. The linked Engram implementation is useful external context, but the remaining OpenClaw work depends on unresolved default write, review, edit, forget, and conflict semantics tracked by related open memory MVP items.

## What This Changes

Expose memory through operator-friendly remember, recall, review, edit, and forget CLI/skill commands with ingestion helpers, provenance, and match explanations.

## Best Possible Solution

Keep this open until maintainers define the v1 memory operator contract. The durable end state should add one default, documented path for remember/recall/review/edit/forget, map it to provenance-preserving write and retrieval flows, include structured match explanations, and coordinate non-destructive review/edit/forget behavior with the open write-path and lifecycle issues.

## Reproduction Assessment

Yes for the missing-surface claim: static inspection of current main shows the default CLI, tools, docs, and host SDK result type do not expose the requested operator verbs or match explanation contract. No live runtime reproduction is needed for this feature gap.

## Solution Assessment

Unclear at implementation level. The requested operator surface is directionally useful, but the best fix is maintainer contract design first because write, review, edit, forget, and conflict semantics remain open in related MVP issues.
