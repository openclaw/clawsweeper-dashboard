## Summary

Keep open: current main and the latest release still provide memory search/get, indexing, promotion, REM backfill, and backend-specific LanceDB store/forget tools, but not the requested shared operator MVP surface for remember/recall/review/edit/forget with durable schema, provenance, and lifecycle semantics. Engram Core, smart-memory, and the closed memory_add PR are useful design context, but they do not settle OpenClaw's active memory slot or provide a merge-ready core/plugin implementation.

## What This Changes

The issue requests ingestion helpers plus a minimal operator-facing CLI/skill surface for durable memory commands with provenance and match explanations.

## Best Possible Solution

Settle the active memory plugin contract, durable schema, provenance, write resolver, and lifecycle semantics first, then expose one small plugin-owned CLI/skill/operator facade that reuses those contracts without backend-specific duplication.
