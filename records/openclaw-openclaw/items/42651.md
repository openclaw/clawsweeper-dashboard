## Summary

Keep open: current main has substantial adjacent memory ingestion, promotion, and review work, plus the Engram Core external implementation noted in discussion, but the default OpenClaw memory surface still lacks the requested small remember/recall/review/edit/forget operator contract and structured recall match explanations. This is feature/product/API work rather than a stale cleanup close.

## What This Changes

Expose memory ingestion and lifecycle operations through a small CLI/skill surface with provenance and match explanations.

## Best Possible Solution

Not applicable as a bug reproduction; this is a feature request. The missing-surface claim is source-verifiable because current main registers only the existing memory CLI and read/search tool surface described above.

## Solution Assessment

Unclear until maintainers settle the default memory contract. Adding commands directly now is plausible, but the safer path is to resolve the write/review/forget/conflict semantics in the related memory MVP work before exposing public operator verbs.
