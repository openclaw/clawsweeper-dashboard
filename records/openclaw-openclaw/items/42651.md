## Summary

Keep open for maintainer product sequencing. Current main and the latest shipped release provide memory indexing/search, promotion, REM backfill, Memory Wiki, and a separate LanceDB memory plugin, but they still do not expose the requested default `remember`/`recall`/`review`/`edit`/`forget` operator contract or the durable review/edit/forget lifecycle behind it; the Engram Core comment is useful external implementation input rather than an OpenClaw core implementation.

## What This Changes

Request to expose a Memory MVP operator surface with ingestion helpers and `remember`, `recall`, `review`, `edit`, and `forget` commands that preserve provenance and match explanations.

## Best Possible Solution

Define the v1 memory operator contract after the write-pipeline and review/forget lifecycle are settled, then expose one small CLI/tool/skill surface that preserves provenance and explains recall matches.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, related issue context, and release-tag inspection give a high-confidence current-main and v2026.5.12 state check that the requested default verbs and durable lifecycle are absent.

## Solution Assessment

Unclear until maintainers settle the Memory MVP contract. Adding commands directly may be part of the answer, but the maintainable path is to resolve write, review/edit/forget, provenance, ingestion, and retention semantics before exposing the operator surface.
