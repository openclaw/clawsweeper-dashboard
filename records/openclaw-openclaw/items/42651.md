## Summary

Keep open for maintainer memory-product sequencing. Current main and the latest release still expose memory search, indexing, promotion, REM backfill, Memory Wiki, and an optional LanceDB memory plugin, but not the requested default `remember`/`recall`/`review`/`edit`/`forget` operator contract or the durable lifecycle needed behind it.

## What This Changes

Expose Memory MVP ingestion helpers plus `remember`, `recall`, `review`, `edit`, and `forget` CLI/skill commands with provenance-rich recall and first-class review/edit/forget flows.

## Best Possible Solution

Settle the write pipeline in https://github.com/openclaw/openclaw/issues/42648 and the review/edit/forget lifecycle in https://github.com/openclaw/openclaw/issues/42650, then add one small default operator CLI/skill surface with provenance, match explanations, revisions, and non-destructive forget semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, tests, release-tag inspection, and issue discussion give high confidence that the requested default lifecycle verbs and durable record semantics are absent today.

## Solution Assessment

Unclear until maintainers settle the Memory MVP contract. The existing memory-core, Memory Wiki, LanceDB, and Engram-style surfaces are useful inputs, but the maintainable core path needs a canonical write/review/forget lifecycle before adding default operator commands.
