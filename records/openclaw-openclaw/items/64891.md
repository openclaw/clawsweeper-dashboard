## Summary

Keep open. Current main and the latest release still let Bedrock memory auto-selection call the AWS SDK default credential chain, which can fall through to IMDS; the prior targeted PR was closed unmerged because its unconditional IMDS-disable approach risked breaking real EC2 instance-role setups.

## What This Changes

The issue reports a regression where Bedrock memory credential auto-detection repeatedly probes AWS IMDS in sandboxed NemoClaw/OpenShell startup environments after upgrading to OpenClaw 2026.4.9.

## Best Possible Solution

Implement a passive, no-network Bedrock credential eligibility check for auto-selection/status paths while preserving the full AWS SDK default credential chain for explicit Bedrock runtime and documented IMDS opt-in use.

## Reproduction Assessment

Yes at source level. The reporter supplied concrete OpenShell logs, and current main plus the pinned AWS SDK source shows Bedrock auto-selection can still reach IMDS through `defaultProvider()` when no earlier credentials resolve.

## Solution Assessment

Yes. The maintainable fix is a narrow passive eligibility probe for auto-selection/status, not the closed PR's global IMDS-disable approach that could break EC2 instance-role users.
