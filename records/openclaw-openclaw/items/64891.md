## What This Changes

The issue reports a regression where Amazon Bedrock memory credential auto-detection can make repeated AWS IMDS requests in an NVIDIA OpenShell/NemoClaw sandbox after startup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90694, or an equivalent Bedrock provider-local fix that suppresses no-signal IMDS probes while preserving explicit AWS default-chain Bedrock credentials.

## Reproduction Assessment

Yes, at source level. Current main can fall through from Bedrock memory credential detection into the AWS SDK IMDS provider, and the issue includes sandbox logs matching the SDK's PUT token and GET credentials paths.

## Solution Assessment

Yes for the current direction, but it is not complete yet. The open linked PR targets the right Bedrock plugin boundary; this issue should remain open until that PR or an equivalent fix merges.
