## Summary

Keep this issue open: current main and v2026.6.10 still call the AWS SDK default credential chain from Bedrock memory credential detection, which can fall through to IMDS, and the open linked PR is the active candidate rather than merged evidence.

## What This Changes

The issue requests stopping Amazon Bedrock memory credential auto-detection from repeatedly probing 169.254.169.254 during sandboxed startup when no AWS credentials are configured.

## Best Possible Solution

Review, revise, and land https://github.com/openclaw/openclaw/pull/90694 or an equivalent Bedrock-local fix that prevents no-signal IMDS probing while preserving explicit Bedrock AWS SDK credential flows.

## Reproduction Assessment

Yes, at source level. Current main reaches the AWS SDK default credential chain from Bedrock memory setup, and the pinned SDK source shows that chain can make the reporter's IMDS PUT/GET requests.

## Solution Assessment

Unclear until maintainer compatibility sign-off. The linked PR is the focused implementation path, but the final shape depends on accepting the boundary between no-signal auto-detection and explicit EC2/ECS/default-chain Bedrock credential use.
