## Summary

Keep this issue open: current main and the latest release still let Bedrock memory credential detection call the AWS SDK default credential chain, which can fall through to IMDS, and the open linked fix PR is the active implementation path rather than a reason to close the report.

## What This Changes

The issue asks OpenClaw to stop or sharply reduce repeated startup requests to AWS IMDS at 169.254.169.254 from Bedrock credential auto-detection in sandboxed non-AWS environments.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/90694 after maintainers accept the explicit Bedrock credential boundary and, ideally, confirm both no-signal sandbox startup and explicit EC2/ECS/default-chain Bedrock behavior.
