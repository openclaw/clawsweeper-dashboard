## Summary

Keep open. Current main still puts Bedrock memory auto-selection on a path that calls `hasAwsCredentials()`, and that helper still delegates to the AWS SDK default credential chain that can fall through to IMDS; the closed direct fix was not merged and the merged adjacent gate still calls the same probe.

## What This Changes

Startup Bedrock memory auto-selection should avoid repeated AWS IMDS requests to `169.254.169.254` in sandboxed non-AWS environments.

## Best Possible Solution

Separate passive Bedrock memory eligibility detection from actual Bedrock runtime authentication so startup auto-selection never probes IMDS, while explicit Bedrock provider usage continues to use the AWS SDK default chain.

## Reproduction Assessment

Yes. The live NemoClaw/OpenShell reproduction is detailed, and current source plus the pinned AWS SDK source gives a high-confidence static reproduction path from memory auto-selection to IMDS token and credential requests.

## Solution Assessment

No current solution is in place. The safest direction is a focused no-network/passive credential detection path for auto-selection, not a broad change to Bedrock runtime AWS SDK authentication.
