## Summary

Keep this issue open: current `main` and latest release still invoke the AWS SDK default credential chain from Bedrock memory credential auto-detection, and the linked open PR is the active fix path rather than a reason to close the report.

## What This Changes

The issue reports a regression where Amazon Bedrock memory credential auto-detection repeatedly probes AWS IMDS in sandboxed non-AWS startup environments.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/90694 lands, or a narrower replacement preserves explicit Bedrock runtime auth while preventing unconfigured auto-detection from probing IMDS.

## Reproduction Assessment

Yes, at source level: current Bedrock memory auto-selection calls `hasAwsCredentials()`, and that helper can invoke the AWS SDK default chain that falls through to IMDS. I did not rerun the full NVIDIA NemoClaw sandbox scenario locally.

## Solution Assessment

Yes for issue routing: the best path is a Bedrock-plugin fix that skips IMDS for unconfigured auto-detection while preserving explicit/default-chain runtime auth. The open linked PR appears to be the active implementation candidate and needs maintainer compatibility review rather than a new issue repair job.
