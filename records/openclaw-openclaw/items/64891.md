## Summary

Keep open. Current main still routes Bedrock memory auto-selection through `hasAwsCredentials()`, and that helper still invokes the AWS SDK default credential chain with no passive/no-IMDS detection path, cache, or backoff. The direct fix PR is closed unmerged, and the merged adapter gate still calls the same credential probe that can reach IMDS.

## What This Changes

The issue requests stopping repeated AWS IMDS requests during startup credential detection in sandboxed non-AWS environments.

## Best Possible Solution

Add a focused Amazon Bedrock plugin fix that separates passive memory-embedding eligibility detection from actual Bedrock runtime authentication, preventing startup auto-selection from repeatedly contacting IMDS in non-AWS sandboxes while preserving explicit Bedrock usage and real AWS SDK default-chain flows. Lock it with unit tests for IMDS suppression, `AWS_EC2_METADATA_DISABLED`, env restoration or no env mutation, repeated negative probes, and profile/SSO/role-compatible paths.

Yes. ## Repair Work Prompt

Fix the Amazon Bedrock memory embedding auto-selection IMDS probe regression tracked by https://github.com/openclaw/openclaw/issues/64891. Current `extensions/amazon-bedrock/embedding-provider.ts` uses `hasAwsCredentials()` for passive eligibility and calls the AWS SDK `defaultProvider()`, which can probe `169.254.169.254`; `extensions/amazon-bedrock/memory-embedding-adapter.ts` calls that helper during auto-selection. Implement a focused plugin-local fix so passive/default memory auto-selection does not repeatedly contact IMDS in credential-less sandboxed environments, while explicit Bedrock usage and legitimate AWS SDK runtime auth via env keys, `AWS_PROFILE`, SSO/shared config, web identity, ECS, EKS, or EC2 instance roles continue to work. Use #64944 as prior context but avoid unconditional IMDS disabling that would break instance-role users; #69708 is useful context for preserving EC2 IMDS auth. Add or update tests for no-IMDS passive detection or bounded/coalesced probes, `AWS_EC2_METADATA_DISABLED`, env restoration if env is touched, repeated negative probes, static credentials, profile/SSO-compatible paths, and adapter auto-selection skip behavior. Keep the fix in the Amazon Bedrock plugin unless a generic seam is truly required. Add a user-facing `CHANGELOG.md` fix entry with reporter/contributor credit, and do not change dependency versions or unrelated auth-profile behavior.
