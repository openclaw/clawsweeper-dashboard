## Summary

Keep this issue open: current `main` and the latest release still have the Bedrock credential auto-detection path that can fall through to AWS IMDS, and the remaining implementation is already owned by the open closing PR at https://github.com/openclaw/openclaw/pull/90694.

## What This Changes

The issue reports repeated AWS IMDS requests during OpenClaw startup in a sandboxed NemoClaw environment after upgrading from 2026.3.11 to 2026.4.9.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/90694; after that PR is resolved, close this issue with the merged commit or the maintainer decision.

## Reproduction Assessment

Yes. Current main calls the AWS SDK default credential chain when no static or Bedrock bearer env credentials are present, and the locked dependency source shows that chain reaches EC2 IMDS unless metadata is disabled.

## Solution Assessment

Yes. The open linked PR is the best current implementation path because it stays in the Amazon Bedrock plugin, targets the no-signal auto-detection path, and preserves explicit Bedrock credential-chain use for maintainer review.
