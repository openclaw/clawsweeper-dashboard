## Summary

Keep open. Current main still lets Bedrock memory auto-selection call `hasAwsCredentials()`, and that helper still invokes the AWS SDK default credential chain that falls through to IMDS; the merged auto-select gate fixed an adjacent failure mode, while the targeted no-IMDS PR was closed unmerged.

## What This Changes

Stop Bedrock memory credential auto-detection from generating repeated AWS IMDS requests to `169.254.169.254` during sandboxed non-AWS startups.

## Best Possible Solution

Add a passive, no-network Bedrock credential eligibility path for memory auto-selection/status surfaces while preserving the full AWS SDK default credential chain for explicit Bedrock runtime use.

## Reproduction Assessment

Yes at source level. The reporter supplied concrete sandbox logs, and current source plus the pinned AWS SDK source shows the Bedrock auto-selection path can still reach IMDS, though I did not live-run the NemoClaw setup.

## Solution Assessment

Yes, keeping this open for a narrow fix is the best path. The merged auto-selection gate is useful but still probes IMDS, while the prior globally-disable-IMDS approach needs a safer replacement that does not break explicit EC2 instance-role auth.
