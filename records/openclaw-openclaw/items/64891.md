## Summary

Keep open. Current `main` still lets Bedrock memory auto-selection call `hasAwsCredentials()`, and that helper still invokes the AWS SDK default credential chain that can fall through to IMDS; the direct no-IMDS fix was closed unmerged and the merged adjacent gate did not remove the metadata probe.

## What This Changes

Avoid repeated AWS IMDS HTTP requests during startup in sandboxed non-AWS environments when Bedrock memory auto-detection runs without AWS credentials.

## Best Possible Solution

Add a passive/no-network credential detection path for Bedrock memory auto-selection and status surfaces while preserving the full AWS SDK default credential chain for explicit Bedrock runtime usage.

## Reproduction Assessment

Yes. The reporter provided detailed NemoClaw/OpenShell logs, and current `main` plus the pinned AWS SDK source gives a high-confidence source path from memory auto-selection to IMDS token and credential requests.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is to separate passive Bedrock eligibility detection from actual Bedrock runtime authentication, not to disable the AWS default credential chain globally.
