## Summary

## What This Changes

Investigate and fix account-dependent native Codex startup context inflation when multiple OAuth accounts use the same OpenClaw agent on one machine.

## Best Possible Solution

Use one stable, privacy-safe Codex state namespace per effective auth profile only after confirming the namespace owns the causal data, preserve or deliberately migrate existing single-profile state, and prove both fresh-session normalization and expected resume behavior with a real two-account setup.
