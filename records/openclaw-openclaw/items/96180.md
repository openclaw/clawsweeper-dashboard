## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Filter config-scoped Codex CLI auth for custom OpenAI providers:** `src/commands/models/list.status-command.ts:455-458`
  - body: This still copies provider IDs from `externalCliDiscoveryForConfigStatus` into the scoped auth-store load. In the real helper, every key under `models.providers` is added to provider scope, so a config with `models.providers.openai.baseUrl: "https://proxy.example.test/v1"` still scopes Codex CLI discovery to `openai` and can make `models status --check` report a custom endpoint as usable from OpenAI/Codex OAuth; the new test masks this by mocking discovery to `mode: "none"`.
  - confidence: 0.92
- **[P1] Do not widen workspace-write network access by default:** `extensions/codex/src/app-server/config.ts:910`
  - body: This changes every Codex app-server `workspace-write` turn from network denied to network enabled. Codex exposes `networkAccess` as an explicit sandbox field, and OpenClaw docs describe network proxying as an opt-in sandbox contract change; existing guardian/workspace-write users should not get outbound network unless a maintainer-approved config or active sandbox egress policy asks for it.
  - confidence: 0.88
- **[P2] Carry file-tab allowance into later tab guards:** `extensions/browser/src/browser/routes/tabs.ts:241-242`
  - body: The PR now lets `/tabs/open` create `file:///...txt` tabs, but the same route module still lists and focuses tabs by calling `assertBrowserNavigationResultAllowed` without `allowLocalFileNavigation`. Because that result guard now revalidates `file:` URLs, the newly opened local report tab will be redacted from `/tabs` and rejected by `/tabs/focus` under strict SSRF policy.
  - confidence: 0.9
- **[P2] Pass file permission through the CDP fallback result check:** `extensions/browser/src/browser/server-context.tab-ops.ts:350-360`
  - body: The CDP fallback path can still open a tab through `/json/new` after the new pre-check allows a local file URL, but its final `assertBrowserNavigationResultAllowed` call is not given the same local-file allowance. If `Target.createTarget` fails and the HTTP fallback succeeds, the final file URL is rejected by the new result guard instead of returning the opened tab.
  - confidence: 0.78
