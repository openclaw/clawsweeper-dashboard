## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Limit serviceName-derived hosts to Serve mode:** `src/gateway/server-tailscale.ts:59-60`
  - body: Here `serviceName` is used for URL logging regardless of `tailscaleMode`. Since `server.impl.ts` passes the config value through for every mode, `gateway.tailscale: { mode: "funnel", serviceName: "svc:openclaw" }` will still run Funnel but log `funnel enabled for svc:openclaw` with a Service-derived host, even though the docs and Tailscale contract make this a Serve-only option. Restrict the effective service name to Serve or reject it outside Serve, and cover the Funnel case.
  - confidence: 0.92
- **[P2] Avoid deriving Service URLs from IP fallbacks:** `src/gateway/server-tailscale.ts:13-14`
  - body: `serviceHostnameFromTailnetHost` assumes the value from `getTailnetHostname()` is a DNS name, but that helper falls back to a Tailscale IP when `Self.DNSName` is absent. With `serviceName` set, an IP like `100.64.0.8` would be logged as `openclaw.64.0.8`, which is not a valid Service URL. Use a DNS-only suffix for Service URL logging or fall back to a generic enabled log when no DNS suffix is available.
  - confidence: 0.88
