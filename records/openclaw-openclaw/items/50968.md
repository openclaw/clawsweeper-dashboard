## Summary

Keep open. The bug remains valid on current main, but this PR is stale and not merge-ready because its LiteLLM changes target command files that no longer exist on main and the probe can still treat partial token metadata as authoritative.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probes to Copilot Proxy and legacy LiteLLM onboarding so discovered token limits populate `contextWindow` and `maxTokens` with fallback defaults.

## Best Possible Solution

Port model-info discovery into the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, only apply complete authoritative token metadata, and cover first-run plus re-auth fallback behavior with focused plugin tests.

## Reproduction Assessment

Yes. Configure LiteLLM or Copilot Proxy with a `/model/info` response advertising a larger window, run onboarding on current main, and the generated model config still comes from the static 128000/8192 builders.

## Solution Assessment

No. The direction is right, but this branch is not the best current solution because the LiteLLM changes attach to deleted command files and mark incomplete metadata as discovered; a plugin-owned port with a complete-metadata guard is safer.
