## Summary

## What This Changes

The PR adds onboarding-time LiteLLM-compatible `/model/info` probes so Copilot Proxy and LiteLLM provider config can use reported context and output-token limits instead of fixed 128k/8192 defaults.

## Best Possible Solution

Port model-info probing into the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, require complete token metadata before patching persisted limits, preserve stored/manual limits on fallback, and add focused tests plus redacted live proxy proof.

## Reproduction Assessment

Yes, source-level: current main still assigns static 128000/8192 limits in Copilot Proxy and active LiteLLM onboarding, so a proxy reporting larger `/model/info` limits would still be configured with smaller values. I did not run a live proxy in this read-only review.

## Solution Assessment

No. Querying LiteLLM-compatible model info is the right direction, but this branch must be ported to the active plugin files and must avoid treating partial token metadata as authoritative before it is the best fix.
