## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during Copilot Proxy and LiteLLM onboarding so model token limits can be configured from proxy metadata instead of static defaults.

## Best Possible Solution

Port the fix into `extensions/litellm` and `extensions/copilot-proxy`, map complete authoritative token metadata to `contextWindow` and `maxTokens`, preserve existing/manual limits on fallback or partial metadata, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-level. Configure LiteLLM or Copilot Proxy so `/v1/model/info` advertises a larger context window, then current main's active onboarding paths still write 128000/8192 from static constants.

## Solution Assessment

No. Querying `/model/info` is the right direction, but this branch needs to move the LiteLLM changes into the current plugin-owned files and avoid treating incomplete token metadata as authoritative before merge.
