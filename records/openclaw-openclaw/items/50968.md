## Summary

Keep open: the reported problem still exists on current main, but this PR is not ready because the LiteLLM portion targets command-owned files that no longer exist on main, it can overwrite existing limits when token metadata is incomplete, and inspectable real behavior proof is still missing.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during Copilot Proxy and LiteLLM onboarding so configured model token limits can come from proxy metadata instead of static defaults.

## Best Possible Solution

Implement provider-local model-info discovery in extensions/litellm and extensions/copilot-proxy, map complete authoritative token metadata to contextWindow and maxTokens, preserve existing/manual limits on fallback or partial metadata, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-level. Configure LiteLLM or Copilot Proxy so `/v1/model/info` advertises a larger context window, then current main’s active onboarding paths still write 128000/8192 from static constants.

## Solution Assessment

No. Querying `/model/info` is the right direction, but this branch needs to move LiteLLM changes into the current plugin-owned files and avoid treating incomplete token metadata as authoritative before merge.
