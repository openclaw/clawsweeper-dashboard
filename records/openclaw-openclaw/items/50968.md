## Summary

Keep open. The reported onboarding bug still exists on current main and this PR is useful source work, but the branch is stale against the plugin-owned LiteLLM onboarding path and still has a partial-metadata downgrade bug, so it should not merge as-is.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probes during Copilot Proxy and LiteLLM onboarding to populate model `contextWindow` and `maxTokens`, with API-key/base-URL handling and fallback defaults.

## Best Possible Solution

Port the model-info discovery into the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, preserve saved limits on failed or partial probes, and cover first-run plus re-auth behavior with focused plugin tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure LiteLLM or Copilot Proxy with a `/v1/model/info` response advertising a larger window, run current onboarding, and observe that current main still writes 128000/8192 from the plugin model builders.

## Solution Assessment

No. The direction is right, but this branch is not the best current solution because the LiteLLM changes attach to deleted command files and mark incomplete metadata as discovered; the safer fix is a current-main plugin-owned port with a complete-metadata guard.
