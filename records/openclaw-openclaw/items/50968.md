## Summary

Keep this PR open: the underlying bug is still present on current main, but the branch is not merge-ready because the LiteLLM half targets command-owned files that current main no longer uses and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds `/model/info` probes to Copilot Proxy onboarding and legacy LiteLLM command onboarding so model limits can be populated from proxy-reported token metadata.

## Best Possible Solution

Port model-info discovery into `extensions/litellm` and `extensions/copilot-proxy`, require complete authoritative token metadata before patching existing limits, preserve fallback/manual values, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-level. Configure LiteLLM or Copilot Proxy so `/model/info` advertises a larger context window, then current main’s onboarding builders still write 128000/8192 from static constants in `extensions/litellm/onboard.ts` and `extensions/copilot-proxy/index.ts`.

## Solution Assessment

No. Querying `/model/info` is the right direction, but this branch needs to move LiteLLM changes to the current plugin-owned files and avoid treating incomplete token metadata as authoritative before merge.
