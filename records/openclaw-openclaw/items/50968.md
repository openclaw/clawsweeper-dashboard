## Summary

Keep open: current main still has the reported static model-limit behavior, but this PR is not merge-ready because the LiteLLM half targets deleted command-owned files, incomplete token metadata can still be treated as authoritative, and external real behavior proof is missing.

## What This Changes

The PR adds LiteLLM-compatible `/v1/model/info` probes for Copilot Proxy and legacy command-owned LiteLLM onboarding so configured `contextWindow` and `maxTokens` can use proxy-reported token limits.

## Best Possible Solution

Port model-info discovery into `extensions/litellm` and `extensions/copilot-proxy`, only apply authoritative token metadata, preserve existing tuned limits on fallback or incomplete responses, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-level. Configure LiteLLM or Copilot Proxy so `/model/info` advertises a larger context window, then current main's onboarding builders still write 128000/8192 from static constants in `extensions/litellm/onboard.ts` and `extensions/copilot-proxy/index.ts`.

## Solution Assessment

No. Querying model-info is the right direction, but this branch needs to move LiteLLM changes to the current plugin-owned files and avoid treating incomplete token metadata as authoritative before merge.
