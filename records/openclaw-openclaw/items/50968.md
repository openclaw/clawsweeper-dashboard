## Summary

Keep open: current main and v2026.5.7 still hardcode the LiteLLM and Copilot Proxy token limits, and this PR is useful but not mergeable as-is because the LiteLLM half targets deleted command-owned files and partial metadata can still overwrite tuned values. The PR body lists local verification, but it does not include inspectable after-fix output, logs, screenshots, or artifacts.

## What This Changes

The branch adds LiteLLM-compatible `/model/info` probes to Copilot Proxy onboarding and legacy command-owned LiteLLM onboarding so token limits can come from `max_input_tokens` and `max_output_tokens` instead of static defaults.

## Best Possible Solution

Port model-info discovery into `extensions/litellm` and `extensions/copilot-proxy`, require complete authoritative token metadata before patching existing config, and add focused regression coverage for larger-context models plus fallback behavior.

## Reproduction Assessment

Yes, source-level. Configure LiteLLM or Copilot Proxy with `/model/info` advertising a larger context window, run current onboarding, and the current plugin builders still write 128000/8192 from static constants.

## Solution Assessment

No. The direction is right, but this PR is not the best current solution until the LiteLLM work is ported to the plugin-owned path and partial metadata is treated as non-authoritative for patching existing limits.
