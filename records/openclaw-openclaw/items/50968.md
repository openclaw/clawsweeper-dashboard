## Summary

Keep open: current main still has the reported hardcoded model limits, but this PR is not mergeable as-is because the LiteLLM changes target command-owned files that no longer exist on main, partial token metadata can still be treated as authoritative, and inspectable real behavior proof is missing.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probes to Copilot Proxy and the old command-owned LiteLLM onboarding path so generated model definitions can use proxy-reported context and output-token limits.

## Best Possible Solution

Port model-info probing into the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, accept only complete token metadata as authoritative, preserve existing/manual limits on fallback, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-level. Current main's Copilot Proxy and LiteLLM onboarding still write 128000/8192 from static constants, so a proxy reporting larger `/v1/model/info` limits would still be configured with the smaller values; I did not run a live proxy.

## Solution Assessment

No. Querying model-info is the right direction, but this branch targets deleted LiteLLM command files and treats partial token metadata as discovered; the safer solution is to port the logic to the plugin-owned onboarding paths with stricter metadata handling.
