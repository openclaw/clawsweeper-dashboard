## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during Copilot Proxy and LiteLLM onboarding so generated model config uses proxy-reported token limits instead of static defaults.

## Best Possible Solution

Port probing into active `extensions/litellm` and `extensions/copilot-proxy` onboarding, only trust complete token metadata, preserve saved/manual limits on fallback, and add focused tests plus redacted live proxy proof.

## Reproduction Assessment

Yes, source-level: current main still writes static 128000/8192 limits in both active Copilot Proxy and LiteLLM onboarding, so a proxy reporting larger limits would still be configured too small. I did not run a live proxy in this read-only review.

## Solution Assessment

No. Querying LiteLLM-compatible model info is the right direction, but the branch must be ported to the active LiteLLM plugin files and must not treat partial token metadata as authoritative before it is the best fix.
