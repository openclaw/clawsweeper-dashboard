## Summary

Keep open: current main still hardcodes 128k/8192 for LiteLLM and Copilot Proxy, so the underlying bug remains, but this PR is not merge-ready because the LiteLLM half targets a deleted command-owned path, incomplete metadata can be treated as discovered, and inspectable real behavior proof is missing.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during Copilot Proxy and LiteLLM onboarding so generated model config can use reported token limits instead of static defaults.

## Best Possible Solution

Port model-info probing to the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, only treat complete token metadata as authoritative, preserve existing/manual limits on fallback or partial metadata, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-level: configure LiteLLM or Copilot Proxy so `/v1/model/info` reports larger limits, then current main's active onboarding paths still write 128000/8192 from static constants. I did not run a live proxy in this read-only review.

## Solution Assessment

No. Querying model-info is the right direction, but this branch must be ported to `extensions/litellm` and should not mark partial token metadata as discovered before merge.
