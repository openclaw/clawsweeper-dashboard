## Summary

Keep this PR open because the reported onboarding problem is still real on current main, but the branch is not merge-ready: it conflicts with main after LiteLLM onboarding moved into the plugin, still has a metadata-safety bug for partial `/model/info` responses, and has no inspectable real behavior proof.

## What This Changes

The PR adds onboarding-time LiteLLM-compatible `/model/info` probes for Copilot Proxy and LiteLLM so configured model definitions can use proxy-reported context and output-token limits instead of fixed 128k/8192 defaults.

## Best Possible Solution

Port model-info discovery into the active Copilot Proxy and LiteLLM plugin onboarding paths, only patch persisted limits when the proxy returns trustworthy numeric metadata for the fields being changed, and add focused tests plus redacted live proxy proof.

## Reproduction Assessment

Yes, source-level. Current main still builds Copilot Proxy and LiteLLM model definitions with static 128000/8192 limits, so a LiteLLM-compatible proxy reporting larger limits would still be configured too small; I did not run a live proxy in this read-only review.

## Solution Assessment

No, not yet. The direction is right, but the PR must be ported to the active plugin-owned LiteLLM onboarding files and must avoid treating fallback values as discovered metadata before it is the maintainable fix.
