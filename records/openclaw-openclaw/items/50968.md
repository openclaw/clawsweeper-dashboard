## Summary

Keep this PR open: the underlying LiteLLM/Copilot Proxy onboarding bug still exists on current main, but this branch is stale against the plugin-owned onboarding paths and still has one unsafe partial-metadata edge case.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probes during LiteLLM and Copilot Proxy onboarding so discovered `max_input_tokens` and `max_output_tokens` can populate `contextWindow` and `maxTokens` instead of static defaults.

## Best Possible Solution

Port model-info discovery into the current plugin-owned LiteLLM and Copilot Proxy onboarding paths, preserve saved limits on failed or partial probes, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level path is to configure LiteLLM or Copilot Proxy with `/model/info` advertising a larger window, run current onboarding, and observe current main still writes the 128000/8192 defaults from the plugin builders.

## Solution Assessment

No. The direction is right, but this PR is not the best current solution because its LiteLLM changes target deleted command files and its discovery result can treat defaulted partial metadata as authoritative.
