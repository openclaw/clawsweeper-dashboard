## Repair Work Prompt

Implement a docs-only convention for workspace capability descriptors for https://github.com/openclaw/openclaw/issues/76074. Define `<workspace>/capabilities/*.md` and optional `<workspace>/capabilities/index.md` as durable local capability descriptors, with fields for purpose, supported requests, required tools/commands/queues/runtimes, safety and approval boundaries, verification steps, fallback behavior, and what to inspect before saying a task is not possible. Cross-link from the agent workspace and system prompt/bootstrap docs, and mention related surfaces such as skills, `before_prompt_build`, Task Flow, ACP runtimes, and `bootstrap-extra-files`. Do not add a gateway wrapper, daemon, router, auth/session store, or runtime native discovery in this PR. Be explicit that current `bootstrap-extra-files` only loads recognized bootstrap basenames, so descriptor surfacing should happen by referencing the descriptors from `AGENTS.md` or through an explicit plugin/hook until a separate native-discovery follow-up is approved. Keep docs frontmatter/read_when valid and avoid changelog unless maintainers require one for the new user-facing docs page.
