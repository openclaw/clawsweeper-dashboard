- **[P1] Write a schema-valid My Browser profile:** `apps/macos/Sources/OpenClaw/ConfigStore.swift:178`
  - body: `config.set` validates the browser object against a strict schema, but this toggle writes an unknown `browser.priorDefaultProfile` key and creates `profiles.mybrowser` without the required `color`. Toggling My Browser from macOS can therefore fail to persist or reload instead of enabling the new mode.
  - confidence: 0.9
- **[P1] Authenticate the extension bridge socket:** `extensions/browser/src/browser/extension-bridge-server.ts:268`
  - body: `background-utils.js` derives a relay HMAC for `/extension`, but the bridge accepts any `/extension` WebSocket and reaches `handleExtension` without checking that token. A local client could send `agent.request` frames through the node-host emitter or CDP relay without proving it is the configured extension.
  - confidence: 0.91
- **[P1] Keep node event emission off the broad SDK surface:** `src/plugin-sdk/gateway-runtime.ts:36-40`
  - body: `plugin-sdk/gateway-runtime` is a public plugin-facing export, and this exposes both node-event emission and process-global emitter registration. The browser bridge already receives an injected callback, so keep this internal or add a narrow documented contract with maintainer approval and compatibility tests.
  - confidence: 0.85
- **[P2] Forward the browser copilot prompt to the agent run:** `extensions/browser/src/browser/server-lifecycle.ts:50`
  - body: This adds `extraSystemPrompt` to the emitted `agent.request` payload, but `server-node-events` does not read or pass that field when dispatching the node-originated run. The advertised browser-copilot persona will be silently dropped unless the node-event contract and tests are updated or the prompt is injected at a supported call site.
  - confidence: 0.9
- **[P2] Do not send a fake loopback gateway token:** `extensions/chrome-extension/sidepanel.js:181`
  - body: When no gateway token is configured, the side panel substitutes `test-token-for-container` for loopback gateways and sends it as `auth.token`. Token-auth local gateways treat a wrong token as a mismatch and record a failure, so this can fail or rate-limit instead of prompting for the real credential.
  - confidence: 0.86
