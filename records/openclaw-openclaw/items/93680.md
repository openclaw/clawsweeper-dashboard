- **[P1] Write a schema-valid My Browser profile:** `apps/macos/Sources/OpenClaw/ConfigStore.swift:175-178`
  - body: The macOS toggle writes `browser.priorDefaultProfile` and `profiles.mybrowser = { driver: "extension" }`, but the PR-head browser schema is strict and still requires each profile to include `color`. `config.set` validates the submitted config, so enabling My Browser can fail instead of persisting the mode.
  - confidence: 0.92
- **[P1] Authenticate the extension bridge socket:** `extensions/browser/src/browser/extension-bridge-server.ts:268`
  - body: `background-utils.js` derives a relay HMAC for `/extension`, but this upgrade path accepts any `/extension` WebSocket and reaches `handleExtension` without validating that token or an equivalent capability. A local client can impersonate the extension path and originate node-attributed turns or CDP frames.
  - confidence: 0.91
- **[P1] Keep node event emission off the broad SDK surface:** `src/plugin-sdk/gateway-runtime.ts:36-40`
  - body: `plugin-sdk/gateway-runtime` is a public package export, but this exposes both node-event emission and process-global emitter registration to plugins. Keep registration internal or introduce a narrow documented contract with maintainer approval and compatibility tests.
  - confidence: 0.84
- **[P2] Do not send a fake loopback gateway token:** `extensions/chrome-extension/sidepanel.js:189`
  - body: When no gateway token is configured, the side panel substitutes `test-token-for-container` for loopback gateways and sends it as `auth.token`. Token-auth local gateways can see this as a token mismatch instead of prompting for the real credential.
  - confidence: 0.87
- **[P2] Forward the browser copilot prompt through node events:** `extensions/browser/src/browser/server-lifecycle.ts:50`
  - body: This adds `extraSystemPrompt` to the emitted `agent.request` payload, but the `server-node-events` parser only reads the AgentDeepLink fields and dispatches without that option.   - confidence: 0.9
