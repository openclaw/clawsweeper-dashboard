  - confidence: 0.91

  - body: `plugin-sdk/gateway-runtime` is a public plugin-facing export, but this adds a host-internal primitive that can originate node-attributed gateway events from any plugin running in the node-host process. Keep this behind an internal/injected bridge callback or add an explicit approved SDK contract with compatibility tests.
  - confidence: 0.86

  - body: The bridge adds `extraSystemPrompt` to the emitted `agent.request`, but `server-node-events.ts` parses only the existing AgentDeepLink fields and dispatches without forwarding it. The advertised browser-copilot persona will be silently dropped unless the node-event contract and tests are updated or the prompt is injected at a supported call site.
  - confidence: 0.9

  - body: `plugin-sdk/gateway-runtime` is a public plugin-facing export, so adding `emitNodeGatewayEvent` creates a node-attributed event capability for plugins without an approved SDK contract. Keep this host-internal or add the full API, permission, compatibility, and test story.
  - confidence: 0.82

  - confidence: 0.91
