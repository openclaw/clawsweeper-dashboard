## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the legacy extension-driver migration:** `src/config/zod-schema.ts:671`
  - body: The latest release treats `browser.profiles.*.driver: "extension"` as a legacy Chrome-extension relay value and doctors it to `existing-session`. Adding the same literal as a new My Browser mode while removing that migration can make existing upgraded configs run a different driver instead of being repaired; use a distinct current value or keep a tested migration path.
  - confidence: 0.9
- **[P1] Write a schema-valid My Browser profile:** `apps/macos/Sources/OpenClaw/ConfigStore.swift:178`
  - body: The macOS toggle saves `profiles.mybrowser = { driver: "extension" }`, but the PR-head browser profile schema still requires `color`. Because `ConfigStore.save` writes through `config.set`, enabling My Browser can fail validation instead of persisting the mode.
  - confidence: 0.92
- **[P1] Authenticate the extension bridge socket:** `extensions/browser/src/browser/extension-bridge-server.ts:268`
  - body: `background-utils.js` derives a relay HMAC for `/extension`, but this upgrade path accepts any `/extension` WebSocket and reaches `handleExtension` without validating that token. That socket can send `agent.request` frames into the node-host emitter, so a local client can impersonate the extension path and originate node-attributed turns or CDP frames.
  - confidence: 0.92
- **[P1] Keep node event emission off the broad SDK surface:** `src/plugin-sdk/gateway-runtime.ts:36-40`
  - body: `plugin-sdk/gateway-runtime` is a public package export, but this exposes node-event emission and process-global emitter registration to plugins. The browser bridge already has an injected callback; keep registration internal or introduce a narrow documented contract with maintainer approval and compatibility tests.
  - confidence: 0.84
- **[P2] Forward the browser copilot prompt into node turns:** `extensions/browser/src/browser/server-lifecycle.ts:48-50`
  - body: This adds `extraSystemPrompt` to the emitted `agent.request` payload, but the `server-node-events` parser and dispatch path do not read or forward that field. The advertised browser-copilot persona will be silently dropped unless the node-event contract and tests are updated or the prompt is injected at a supported call site.
  - confidence: 0.9
- **[P2] Do not substitute placeholder gateway tokens:** `extensions/chrome-extension/sidepanel.js:189`
  - body: When no gateway token is configured, the side panel substitutes `test-token-for-container` for loopback gateways and sends it as `auth.token`. Token-auth local gateways can treat this as a mismatch and fail instead of prompting for the real credential or omitting auth only on a truly unauthenticated loopback.
  - confidence: 0.86
