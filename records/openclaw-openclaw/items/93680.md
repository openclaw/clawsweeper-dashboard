Full review comments:

- **[P2] Re-enable the panel on fail-closed turns:** `extensions/chrome-extension/sidepanel.js:631`
  - body: When `nodeTurn` is unavailable and direct fallback is disallowed, `deliverTurn` adds the fail-closed system message and returns. `sendMessage` disabled `sendBtn` and `msgInput` before calling it, and this path does not throw or receive a final chat event, so the side panel stays stuck disabled after a recoverable node outage. Re-enable the controls before this return or make the path throw a handled error.
  - confidence: 0.93
- **[P1] Re-register the emitter after reconnect:** `src/node-host/runner.ts:313`
  - body: This `onClose` clears the process-wide node gateway-event emitter, but the runner only registers it once before the initial connection. The gateway client reconnects by calling `start()` again and exposes `onHelloOk` for successful handshakes, so after a transient gateway close the node can reconnect while the bridge still has no usable emitter, causing node-routed side-panel turns to fail until the node process restarts. Reinstall the emitter on each successful connection or pair clearing with reconnect-time registration.
  - confidence: 0.9
