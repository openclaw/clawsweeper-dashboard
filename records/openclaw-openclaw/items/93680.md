- **[P1] Authenticate the extension bridge socket:** `extensions/browser/src/browser/extension-bridge-server.ts:268`
  - body: `background-utils.js` derives a relay HMAC for `/extension`, but the bridge upgrade path accepts any `/extension` WebSocket and reaches `handleExtension` without validating that token or an equivalent origin/capability. Because that socket can send `agent.request` frames into the node-host emitter, a local client can impersonate the extension path and originate node-attributed turns or CDP frames.
  - confidence: 0.91

  - body: When no gateway token is configured, the side panel substitutes `test-token-for-container` for loopback gateways and sends it as `auth.token`. Token-auth local gateways can then fail or rate-limit on a wrong credential instead of prompting for the real credential.
  - confidence: 0.87
- **[P1] Keep node event emission off the broad SDK surface:** `src/plugin-sdk/gateway-runtime.ts:36-40`
  - body: `plugin-sdk/gateway-runtime` is a public package export, but this exposes both node-event emission and process-global emitter registration to plugins. The browser bridge already has an injected `onAgentRequest` callback, so keep registration internal or introduce a narrow documented contract with maintainer approval and compatibility tests.
  - confidence: 0.84
- **[P2] Forward the browser copilot prompt to the agent run:** `extensions/browser/src/browser/server-lifecycle.ts:50`
  - body: This adds `extraSystemPrompt` to the emitted `agent.request` payload, but `server-node-events` does not read or forward that field into `AgentCommandOpts`.
