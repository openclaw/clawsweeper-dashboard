Fix Feishu plugin tool availability for account-scoped configs. The observable bug is that `channels.feishu.accounts.<id>.appId/appSecret` is supported by Feishu config/setup and runtime account resolution, but Feishu tool metadata only gates `feishu_doc`, `feishu_drive`, `feishu_perm`, `feishu_wiki`, and related tools on top-level `channels.feishu.appId/appSecret`, so account-only setups can load the plugin while exposing no model-visible Feishu tools. Preserve plugin/core boundaries: prefer a generic manifest availability shape or a Feishu-owned metadata/helper path that supports dynamic account maps without loading broad plugin runtime; do not add a DM-only injection bypass, do not make tools visible with no configured credentials, and keep existing tool allow/deny/profile behavior intact. Add regression coverage for top-level credentials, account-scoped credentials, and no credentials. Treat https://github.com/openclaw/openclaw/issues/84095 and https://github.com/openclaw/openclaw/issues/84241 as one cluster; do not rely on https://github.com/openclaw/openclaw/pull/84133 because it is release hardening rather than the runtime availability fix.
