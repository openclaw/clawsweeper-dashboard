## Summary

Keep open: current main and the v2026.5.18 npm package already include Feishu `contracts.tools` in `openclaw.plugin.json`, so the latest comment is likely not the remaining root cause; the still-valid source-backed gap is that account-scoped Feishu credentials do not satisfy the current top-level-only tool availability metadata.

## What This Changes

Feishu plugin tools should appear in the agent callable tool list for configured Feishu DM accounts using the full tool profile.

## Best Possible Solution

Make Feishu plugin tool availability account-aware so both top-level and named-account credentials expose declared Feishu tools while still hiding tools when no usable credentials exist.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for account-scoped Feishu credentials without top-level `channels.feishu.appId/appSecret`: the manifest availability check reads only the top-level required paths and filters the tools before they reach the agent list.

## Solution Assessment

Yes, with one correction: the best fix is account-aware availability metadata/resolution, not adding `contracts.tools` to `package.json`, because current main and the v2026.5.18 tarball already ship those contracts in `openclaw.plugin.json`.
