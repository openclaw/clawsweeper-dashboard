## Summary

Keep open. Current main still lacks the central dispatcher stall guard and memory session-delta retry behavior, while the PR branch has active blocking review findings, broad bundled runtime changes, and no real after-fix behavior proof.

## What This Changes

This PR adds dispatcher fail-closed handling for plan/control stalls, memory-core session-delta retry and search degradation, plus related subagent, diagnostics, queue, timeout, compaction, and tests.

## Best Possible Solution

Split and rework the useful pieces into focused patches: dispatcher stall fail-close with refreshed timers and listener cleanup, memory-core session-delta retry with real backoff, memory search fallback, and separately reviewed diagnostic/subagent/queue/compaction changes.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head has deterministic static paths for false stall aborts, leaked abort listeners, skipped retry backoff, empty memory fallback, orphan re-announcement, stale diagnostic reconciliation, and wrong main-lane attribution. The original runtime freeze/outage was not live-reproduced in this read-only pass.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refresh the plan-stall timer on each plan update:** `src/auto-reply/reply/dispatch-from-config.ts:335-336`
  - body: `armControlPlaneStallTimer` returns while a timer is already set, so a run that keeps emitting plan updates for more than 45 seconds before concrete output is aborted even though control-plane progress is still active. Refresh or rearm the deadline on each qualifying update.
  - confidence: 0.91
- **[P2] Clean up forwarded abort listeners on early exits:** `src/auto-reply/reply/dispatch-from-config.ts:318-320`
  - body: The forwarded abort listener is installed before plugin-bound early-return paths, but cleanup only runs in the later resolver `finally`. Handled, declined, or errored plugin-bound dispatches can leak listeners on long-lived upstream abort signals.
  - confidence: 0.87
- **[P2] Preserve completed cleanup state for orphaned runs:** `src/agents/subagent-registry-helpers.ts:235-236`
  - body: Clearing `cleanupCompletedAt` reopens runs that were already announced and cleaned up on a prior boot. They can be retried and announced again as fresh orphan errors, overwriting terminal state and duplicating parent-facing failure messages.
  - confidence: 0.85
- **[P2] Limit stale reconciliation to tracked run sources:** `src/logging/diagnostic.ts:406`
  - body: The heartbeat flips any `processing` session to `idle` when it is missing from the embedded/reply registries, but some dispatch paths are not registered there. Active plugin inbound turns can be reclassified as idle mid-flight, suppressing stuck-session diagnostics and skewing telemetry.
  - confidence: 0.86
- **[P2] Resolve the true session before fail-closing main lane:** `src/process/command-queue.ts:74-76`
  - body: Mapping the shared `main` lane to `agent:main:main` marks the wrong session idle when a failover error belongs to another run using that lane. The real session remains stuck while diagnostics point at the main session.
  - confidence: 0.86
- **[P2] Fall back to keyword results when embedding lookup fails:** `extensions/memory-core/src/memory/manager.ts:399-403`
  - body: When embedding fails, the patch leaves `queryVec` empty but vector-first mode still returns vector-only results when hybrid/FTS checks are false. Transient embedding outages can silently produce empty memory searches instead of keyword fallback or a visible error.
  - confidence: 0.84
- **[P2] Enforce retry backoff when a debounce timer exists:** `extensions/memory-core/src/memory/manager-sync-ops.ts:488-490`
  - body: The fetch-failure retry path returns whenever `sessionWatchTimer` exists, but active transcript traffic keeps that normal 5s debounce timer populated. During an outage this skips the intended 60s backoff and continues retrying at debounce cadence.
  - confidence: 0.88
