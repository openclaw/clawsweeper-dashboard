## Summary

Keep open. Current main has related strict-agentic plan-only retry/blocking behavior, but the broader dispatcher stall guard and memory session-delta retry/backoff are still absent; the PR head is also not safe to merge because several deterministic P1/P2 correctness findings remain across the proposed runtime and memory changes.

## What This Changes

This PR adds dispatcher fail-closed handling after plan/control progress stalls, memory session-delta fetch-failure retry/backoff and search degradation, plus related diagnostic, subagent, timeout, compaction, and test updates.

## Best Possible Solution

Split and rework the useful pieces: one focused dispatcher patch that fails closed after true control-plane stalls while refreshing timers and cleaning up abort listeners, one memory-core patch that preserves pending session-delta work and enforces real retry backoff after fetch failures, and one explicit memory search fallback patch. The diagnostic, subagent orphan, command-lane, timeout, and compaction changes should be justified and reviewed separately before merge.

## Reproduction Assessment

Yes. The PR defects have deterministic static reproduction paths on the PR head: the stall guard returns while a timer already exists, the retry path returns when sessionWatchTimer is populated, and the embedding-failure branch can fall through to vector-only results in vector-first mode; current main also visibly lacks the requested dispatcher and memory retry mechanisms.

## Solution Assessment

No. The requested direction is useful, but this PR is not the best merge shape because it bundles multiple behavior changes and introduces concrete regressions; the safer solution is smaller, separately validated patches with the current review findings fixed first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refresh the plan-stall timer on each plan update:** `src/auto-reply/reply/dispatch-from-config.ts:335-336`
  - body: `armControlPlaneStallTimer` returns while `controlPlaneStallTimer` is already set, so a run that keeps emitting plan updates for more than 45 seconds before concrete output is aborted even though progress is still active. Refresh or rearm the deadline on each qualifying update.
  - confidence: 0.91
- **[P2] Clean up forwarded abort listeners on early exits:** `src/auto-reply/reply/dispatch-from-config.ts:318-320`
  - body: The forwarded abort listener is installed before plugin-bound early-return paths, but cleanup only runs in the later resolver `finally`. Handled, declined, or errored plugin-bound dispatches can leak listeners on long-lived upstream abort signals.
  - confidence: 0.87
- **[P2] Enforce retry backoff when a debounce timer exists:** `extensions/memory-core/src/memory/manager-sync-ops.ts:488-490`
  - body: The fetch-failure retry path returns whenever `sessionWatchTimer` exists, but active transcript traffic keeps that normal 5s debounce timer populated. During an outage this skips the intended 60s backoff and continues retrying at debounce cadence.
  - confidence: 0.88
- **[P2] Fall back to keyword results when embedding lookup fails:** `extensions/memory-core/src/memory/manager.ts:399-403`
  - body: When embedding fails, the patch leaves `queryVec` empty but vector-first mode still returns vector-only results when hybrid/FTS checks are false. That makes transient embedding outages silently produce empty memory searches instead of keyword fallback or a visible error.
  - confidence: 0.84
- **[P2] Preserve completed cleanup state for orphaned runs:** `src/agents/subagent-registry-helpers.ts:235-236`
  - body: Clearing `cleanupCompletedAt` reopens runs that were already announced and cleaned up on a prior boot. They can be retried and announced again as fresh orphan errors, overwriting terminal state and duplicating parent-facing failure messages.
  - confidence: 0.85
- **[P2] Limit stale reconciliation to tracked run sources:** `src/logging/diagnostic.ts:406`
  - body: The heartbeat flips any `processing` session to `idle` when it is missing from the embedded/reply registries, but some dispatch paths do not register there. Active plugin inbound turns can be reclassified as idle mid-flight, suppressing stuck-session diagnostics and skewing state telemetry.
  - confidence: 0.86
- **[P2] Resolve the true session before fail-closing main lane:** `src/process/command-queue.ts:74-75`
  - body: Mapping the shared `main` lane to `agent:main:main` marks the wrong session idle when a failover error belongs to another run using that lane. The real session remains stuck while diagnostics point at the main session.
  - confidence: 0.86
