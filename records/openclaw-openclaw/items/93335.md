## Summary

Keep open: current main still maps a below-range stored `off` request on a no-off profile to the highest non-off level, and this PR is a narrow shared-helper fix, but external contributor proof remains mock-only so it still needs real behavior proof or a maintainer proof override before merge.

## What This Changes

The branch changes the shared thinking-level fallback to choose the lowest supported non-off level when the request is below every supported level and adds a no-off regression test.

## Best Possible Solution

Land this shared helper fix after redacted real no-off session/provider proof or a maintainer proof override, while leaving broader adaptive and persistence semantics to their existing related items.

## Reproduction Assessment

Yes. Source inspection shows current main sorts levels descending and then returns the first non-off fallback for below-range requests, so a no-off `low`/`medium`/`high` profile maps requested `off` to `high`; I did not run a live provider/model switch in this read-only review.

## Solution Assessment

Yes for the code fix. The shared helper is the narrowest maintainable fix point because durable model switching, gateway patching, reply execution, and agent execution all use it; caller-local fixes would duplicate fallback policy.
