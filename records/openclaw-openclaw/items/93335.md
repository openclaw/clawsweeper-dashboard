## Summary

Keep open. The patch is the right narrow code shape and current main/latest release still have the old below-range fallback, but this external PR still has only mocked/unit-test proof rather than real CLI/provider/session evidence or a maintainer proof override.

## What This Changes

The branch changes `resolveSupportedThinkingLevelFromProfile` to choose the lowest non-off fallback for below-range requests and adds a regression test for no-off profiles.

## Best Possible Solution

Land this shared helper fix after redacted real no-off session/provider proof or a maintainer proof override, ideally before or together with the related Fireworks catalog PR.

## Reproduction Assessment

Yes, source-reproducible. Current main sorts supported levels descending and then returns the first non-off fallback for below-range requests, so a no-off `low|medium|high` profile maps `off` to `high`; I did not run a live provider/model switch in this read-only review.

## Solution Assessment

Yes for the code shape. The shared helper is the narrowest maintainable fix point because directive persistence, gateway patching, and turn-local execution all delegate unsupported thinking-level remaps there; caller-local fixes would duplicate policy.
