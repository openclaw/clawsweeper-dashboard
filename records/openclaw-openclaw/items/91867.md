## What This Changes

The issue requests a durable channel.message design invariant that preserves one logical text-plus-media payload until the channel plugin/backend chooses the physical send shape.

## Best Possible Solution

Have maintainers choose the durable channel.message contract, then implement a typed channel/plugin path that preserves logical text-plus-media payloads while retaining delivery receipts, pinning, hooks, and fallback behavior under focused tests.
