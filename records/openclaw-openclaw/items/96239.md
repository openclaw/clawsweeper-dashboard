## Summary

Keep open: current source still has a plausible path where a successful assistant reply plus a retained mutating `lastToolError` becomes an additional user-visible tool-warning final, and the related Feishu fix in https://github.com/openclaw/openclaw/pull/94428 does not fully settle the reported false `lastToolError` trigger or the second observed run.

## What This Changes

The issue reports that successful Feishu agent turns can surface a `⚠️ 🛠️` tool-warning final above the actual reply when `lastToolError` is populated from exec output.

## Best Possible Solution

Preserve warnings for real unresolved mutating failures, but clear or suppress false/recovered exec warnings and make Feishu present any necessary warning after or within the final answer rather than as the visually leading card.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The source path is clear for a retained mutating `lastToolError` to produce an extra warning final after an assistant reply, but the exact `configuration file` trigger and the second run still need trajectory proof.

## Solution Assessment

Unclear as stated: suppressing every uncertain mutation warning is a product/safety decision. The best repair should be narrower: fix false exec error/mutation classification and Feishu ordering while preserving warnings for real unresolved side effects.
