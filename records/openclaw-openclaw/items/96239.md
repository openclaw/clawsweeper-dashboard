## Summary

Keep open: current `main` still has a source path where a retained mutating `lastToolError` appends an extra user-visible warning payload after an assistant reply, and the merged Feishu ordering fix does not prove the reported false `configuration file` trigger or second Feishu run is fixed.

## What This Changes

The issue reports successful Feishu/Gateway agent turns producing a warning card before the actual final reply when a retained tool warning is surfaced as an extra final payload.

## Best Possible Solution

Preserve warnings for real unresolved mutating failures, but clear or suppress false/recovered exec warnings and make Feishu present any necessary warning after or within the final answer rather than as the visually leading card.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The source path is clear for a retained mutating `lastToolError` to produce an extra warning final after an assistant reply, but the exact `configuration file` trigger and second run still need trajectory proof.

## Solution Assessment

Unclear as stated: suppressing every uncertain mutation warning is a product/safety decision. The best repair should be narrower: fix false exec error/mutation classification and Feishu ordering while preserving warnings for real unresolved side effects.
