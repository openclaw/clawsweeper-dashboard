## Summary

Keep open: current main improves the Feishu streaming-card case, but it still preserves user-visible warning payloads after successful assistant replies for unresolved mutating tool errors, and the reported false `lastToolError` trigger is not proven fixed.

## What This Changes

The issue reports a Feishu/gateway turn where a successful assistant reply is visually displaced by a leading `⚠️ 🛠️` tool-warning card after exec stdout is retained as `lastToolError`.

## Best Possible Solution

Preserve warnings for real unresolved mutating failures, but fix the false exec error/mutation classification and ensure Feishu presents any necessary warning after or inside the final answer rather than as a visually leading separate reply.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review, but current source and tests show the core two-payload shape for successful assistant text plus unresolved mutating tool errors. The specific `configuration file` stdout trigger and the second observed run still need trajectory or live proof.

## Solution Assessment

Unclear as stated: suppressing every uncertain mutation warning would be too broad. The best repair should be narrower: fix false exec error/mutation classification and Feishu ordering while preserving warnings for real unresolved side effects.
