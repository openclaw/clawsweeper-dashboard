## Summary

Keep open: current main and the latest release include a Feishu mitigation that preserves answer text when a later error final arrives, but the retained `lastToolError`/mutating-warning path can still append a warning after a successful assistant reply and the reported false `configuration file` trigger is not proven fixed.

## What This Changes

The issue asks OpenClaw to stop misclassifying successful read-only exec output as a mutating tool error and to prevent Feishu warning cards from visually pre-empting the real final reply.

## Best Possible Solution

Keep real warnings for unresolved side effects, but clear or suppress false/recovered exec errors and render any necessary Feishu warning after or within the final answer rather than as the leading user-visible card.

## Reproduction Assessment

No high-confidence controlled reproduction was established in this read-only review. The source path is clear for a retained mutating `lastToolError` to produce an extra warning final after a successful reply, but the exact `configuration file` false trigger and the second run still need trajectory proof.

## Solution Assessment

Unclear as stated: the Feishu fix in https://github.com/openclaw/openclaw/pull/94428 is a good mitigation for answer preservation, but it does not settle whether recovered or false exec warnings should be suppressed globally. The best fix should target the false error/mutation classification while preserving warnings for real unresolved side effects.
