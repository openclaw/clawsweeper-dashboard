- `impact:session-state`: The reported failure is loss of active chat run identity after reconnect or tab return.

## Summary

Keep open: current main and v2026.5.19 still clear the Dashboard chat run identity on Gateway hello and still lack an active-run recovery field in the hello snapshot; the linked fixing PR remains open and conflicting, so this issue is still the canonical tracker.

## What This Changes

The issue reports Gateway Dashboard chat controls reverting from Stop/Queue to New session/Send after returning to chat during an active run, allowing an operator to interrupt or replace the active run.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/41067 with a current-main additive active-run recovery contract that keeps Stop, Queue, abort, and send behavior on one source of truth.

## Reproduction Assessment

Yes, at source level. Seed chatRunId/chatStream and process a Gateway hello or same-session return path on current main: the UI clears local run identity while the snapshot has no active-run recovery field and send queueing still keys off chatRunId.

## Solution Assessment

Yes for keeping this issue open. The maintainable fix is an additive Gateway active-run snapshot plus Control UI reconciliation, and the linked open PR is the current implementation candidate rather than a reason to close early.
