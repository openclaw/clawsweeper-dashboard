## Summary

Keep open. Current main still has the reported failure path: reconnect/hello clears the Dashboard chat run marker, the hello snapshot has no active chat-run recovery data, and the composer still switches Stop/Queue from `chatRunId`. The issue is also paired with open, unmerged PR #41067 using closing syntax, so this should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Gateway Dashboard chat should preserve the active run busy state and continue showing Stop/Queue after returning to a chat session during an active run.

## Best Possible Solution

Keep this issue open and resolve it through PR #41067 or a narrower replacement that adds an additive active chat-run snapshot to Gateway hello, populates it from server active run state, restores matching UI `chatRunId`/busy controls on reconnect or tab return, and updates focused Gateway/UI coverage.
