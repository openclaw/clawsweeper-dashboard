## Summary

Keep open: current main recognizes `mcp.servers`, but root `defaultModel` and `agents.list[].description` are still source-reproducible as strict unknown-key pruning and need a maintainer config-contract decision before a safe fix.

## What This Changes

The issue asks for `openclaw doctor --fix` to preserve `mcp.servers`, root `defaultModel`, and `agents.list[].description` instead of stripping them as unknown config fields.

## Best Possible Solution

Keep supported `mcp.servers` and strict unknown-key cleanup, then decide whether root `defaultModel` or agent descriptions should be explicitly accepted or migrated with focused regression coverage.

## Reproduction Assessment

Yes, from source for the remaining fields. `stripUnknownConfigKeys` deletes Zod `unrecognized_keys`, and current schemas do not define root `defaultModel` or `agents.list[].description`, while `mcp.servers` is already recognized.

## Solution Assessment

No, not as broad unknown-key preservation. A safer solution is a maintainer-approved schema or migration change for specific fields that should be valid config while preserving doctor’s documented cleanup of real unknown keys.
