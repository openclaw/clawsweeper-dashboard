## Summary

Keep open: current main supports the MCP path, but root `defaultModel` and agent `description` remain outside the strict config contract, so closing would skip a real maintainer decision about whether those fields should be supported, migrated, or documented as unsupported.

## What This Changes

The issue asks `openclaw doctor --fix` to preserve `mcp.servers`, root `defaultModel`, and `agents.list[].description` instead of pruning them as unknown config.

## Best Possible Solution

Keep `mcp.servers` as the supported MCP contract, keep root unknown-key cleanup strict, and make an explicit maintainer decision for any `defaultModel` or agent-description schema/migration/docs change.

## Reproduction Assessment

Yes, source-reproducible for the remaining fields: `stripUnknownConfigKeys` removes Zod `unrecognized_keys`, and current schemas still omit root `defaultModel` and agent `description`; `mcp.servers` is already supported.

## Solution Assessment

No, not as a blanket preserve-all-unknown-fields fix. The safer path is to keep strict cleanup and only add or migrate maintainer-approved config fields.
