## Summary

Keep open: current main already recognizes `mcp.servers`, but source inspection shows `defaultModel` and `agents.list[].description` are still outside the strict config schemas, so this is not fully resolved or safely closeable.

## What This Changes

Requests that `doctor --fix` preserve user config fields such as `mcp.servers`, root `defaultModel`, and `agents.list[].description` instead of deleting them as unknown keys.

## Best Possible Solution

Keep `mcp.servers` as the supported MCP path, preserve documented unknown-key cleanup, and have maintainers decide whether `defaultModel` and agent descriptions should be explicit supported fields or migrated/documented alternatives.

## Reproduction Assessment

Yes, partially from source. Current main should preserve `mcp.servers`, but `stripUnknownConfigKeys` will remove root `defaultModel` and `agents.list[].description` because those fields are absent from strict schemas.

## Solution Assessment

No for preserving all unknown root keys; that would disable documented cleanup. The safer direction is a narrow maintainer-approved schema or migration change for any fields that should be valid config.
