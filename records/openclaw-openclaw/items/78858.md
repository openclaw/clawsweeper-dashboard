## Summary

Keep open: current main already recognizes `mcp.servers`, but the remaining `defaultModel` and `agents.list[].description` behavior is source-reproducible as strict unknown-key pruning and needs maintainer config-contract judgment before a fix is safe.

## What This Changes

The issue asks `doctor --fix` to preserve `mcp`, top-level `defaultModel`, and agent `description` fields instead of pruning them as unknown config.

## Best Possible Solution

Keep supported `mcp.servers` and strict unknown-key cleanup, then have maintainers decide whether to add or migrate explicit support for root `defaultModel` and agent descriptions with regression coverage.

## Reproduction Assessment

Yes, from source. `stripUnknownConfigKeys` deletes Zod `unrecognized_keys`; current schemas recognize `mcp` but do not define root `defaultModel` or `agents.list[].description`, so those fields would be pruned.

## Solution Assessment

No, not as broad unknown-key preservation. Preserving arbitrary unknown root keys would contradict documented cleanup; the safer path is a narrow maintainer-approved schema or migration change for fields that should be valid config.
