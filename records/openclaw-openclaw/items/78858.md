## Summary

Keep open: current main and the latest release already recognize `mcp.servers`, but root `defaultModel` and `agents.list[].description` remain outside the strict config schema while `doctor --fix` intentionally drops unknown keys. The remaining work needs an explicit maintainer config-contract decision rather than cleanup closure.

## What This Changes

The issue asks for `openclaw doctor --fix` to preserve MCP config, root `defaultModel`, and agent description fields instead of removing them as unknown config.

## Best Possible Solution

Keep `mcp.servers` as the supported MCP contract, keep strict unknown-key cleanup, and make a maintainer decision to either add/migrate documented support for `defaultModel` and agent descriptions or document them as unsupported.

## Reproduction Assessment

Yes for the remaining fields by source inspection: `stripUnknownConfigKeys` deletes Zod `unrecognized_keys`, and the current schemas omit root `defaultModel` and `agents.list[].description`. No for MCP removal on current main, because `mcp.servers` is now a recognized config path.

## Solution Assessment

No for a blanket preserve-all-unknown-fields fix; that would disable documented root unknown-key cleanup. The maintainable path is an explicit schema/migration/docs decision for only approved legacy or supported config fields.
