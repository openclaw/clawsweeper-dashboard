## Summary

This mixed report should close because the `mcp.servers` part is not reproducible on current main, root `defaultModel` preservation conflicts with the documented unknown-key cleanup contract, and the only remaining useful `agents.list[].description` request is already tracked by the open agent-description issue and PR.

## What This Changes

The issue asks for `openclaw doctor --fix` to preserve `mcp`, root `defaultModel`, and agent description fields instead of pruning them as unknown config.

## Best Possible Solution

Keep the documented unknown-key cleanup behavior, use `agents.defaults.model` for model defaults, and route agent-description support through https://github.com/openclaw/openclaw/issues/7490 and https://github.com/openclaw/openclaw/pull/75225.

## Reproduction Assessment

Yes at source level for unsupported root `defaultModel` and `agents.list[].description` being pruned: the strict schemas omit those keys and `stripUnknownConfigKeys` deletes Zod `unrecognized_keys`. The `mcp.servers` part is not reproducible because current main already includes top-level `mcp` and permissive server entries.

## Solution Assessment

No for the broad request to preserve all unrecognized fields, because that would change documented doctor policy. The maintainable path is to keep `mcp.servers`, use `agents.defaults.model`, and resolve agent descriptions through the canonical open feature path.
