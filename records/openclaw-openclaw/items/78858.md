## Summary

Keep open. Current main already recognizes `mcp`, but the source still shows `doctor --fix` will prune root `defaultModel` and `agents.list[].description`; because those fields are not current documented config contracts and unknown-key pruning is documented behavior, this needs maintainer config-contract judgment rather than cleanup closure.

## What This Changes

The issue asks for `openclaw doctor --fix` to preserve user config fields such as MCP settings, root `defaultModel`, and agent descriptions instead of dropping them as unknown keys.

## Best Possible Solution

Maintainers should decide whether root `defaultModel` and `agents.list[].description` are supported or legacy config; if yes, add explicit schema/types/docs or a targeted doctor migration with regression coverage while keeping root unknown-key cleanup strict.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence path for `defaultModel` and `agents.list[].description` being removed by strict-schema unknown-key cleanup, but current main recognizes `mcp` and I did not establish a live MCP-removal path.

## Solution Assessment

No for the broad root-catchall approach from the related closed PR. The safer solution is a maintainer-approved, explicit config contract or targeted legacy migration that preserves only supported fields.
