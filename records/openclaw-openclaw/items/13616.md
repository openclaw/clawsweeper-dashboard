Keep open: current main and the latest release still provide local backup archive creation/verification plus provider-based migration, but not the requested native restore/import/list/cloud/encryption/scheduled-backup lifecycle or component-aware cron/session restore semantics. The thread links useful workarounds such as `openclaw-workspace-sync` and Keep My Claw, plus open restore/encrypted-backup PRs, but they do not close the first-party destructive restore and session-history design gap.

## What This Changes

Request to add first-party backup, restore, export, import, and list commands for config, cron jobs, session history, scheduled/cloud/encrypted backups, and disaster recovery.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.
