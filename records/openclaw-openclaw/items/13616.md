## Summary

Keep open: current main and the latest release still provide local backup archive creation and verification only, while native restore/import/list/cloud/encryption/scheduled backup and component-aware cron/session restore remain unresolved. Community plugin/skill workarounds cover useful optional cloud backup scope, but they do not replace the missing first-party restore contract and session-history semantics.

## What This Changes

Requested a native OpenClaw backup/restore utility for config, cron jobs, workspace state, cloud snapshots, and session history.

## Best Possible Solution

Define a versioned, security-reviewed native restore contract for archives created by `openclaw backup create`, with dry-run validation, safe extraction, rollback, explicit component and secret policies, while leaving cloud scheduling, provider storage, and optional encryption automation to plugins or ClawHub.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs give high-confidence proof that the native restore/import/list/cloud workflow is absent on current main and in `v2026.6.6`.

## Solution Assessment

No, not as the full proposed core scope. The maintainable path is a narrow first-party local restore contract with security-reviewed state handling, while optional cloud sync, scheduling, retention, and provider-specific storage should stay in plugin or ClawHub surfaces unless maintainers decide otherwise.
