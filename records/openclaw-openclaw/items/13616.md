## Summary

Keep open. Current main has local backup create/verify and adjacent migration/profile work, but it still lacks the requested native restore/import/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore semantics; the direct restore PR remains open with security-sensitive review blockers.

## What This Changes

Request a first-party backup/restore utility for OpenClaw config, cron jobs, session history, cloud storage, encryption, scheduling, and disaster recovery workflows.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella, settle a hardened restore/archive safety model first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewable follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, changelog, live issue context, and related PR checks give high-confidence proof that create/verify exists while the requested restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.

## Solution Assessment

No, not as one monolithic autonomous implementation. The maintainable path is to settle archive compatibility, restore safety, secret policy, and component semantics first, then review smaller slices such as https://github.com/openclaw/openclaw/pull/59280 and later cron/session/cloud work.
