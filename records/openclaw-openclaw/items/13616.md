Community tools mentioned in the thread cover useful backup/sync workflows today, but they do not close the first-party session-history and destructive-restore design gap.

## What This Changes

Request to add built-in backup/restore commands for config, cron jobs, session history, cloud/encryption/scheduling, and component-specific export/import.

Keep this as the native umbrella for a maintainer-approved first-party backup lifecycle built on create/verify, with safe restore, dry-run validation, component contracts for cron and sessions, a secret policy, and a clear split between core recovery and optional plugin/cloud features.
