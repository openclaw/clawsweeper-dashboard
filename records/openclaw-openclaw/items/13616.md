## Summary

Keep open. Current main and v2026.5.22 still expose only first-party local backup create/verify, while native restore/import/list/cloud/encryption/scheduling and component-aware cron/session-history semantics remain unresolved; community options like openclaw-workspace-sync and keepmyclaw cover useful parts but do not complete the native core request.

## What This Changes

Request a built-in backup/restore lifecycle for OpenClaw config, cron jobs, session history, workspace files, optional cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native backup-lifecycle umbrella, first settling safe restore/import semantics for local archives and component modeling, then deciding which cloud, encryption, scheduling, and portability layers belong in core versus plugins.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source and docs inspection provide high-confidence proof that current main exposes local create/verify only and lacks the requested restore/import/list/cloud and component-aware cron/session workflows.

## Solution Assessment

Unclear as a single broad implementation. The maintainable path is to settle the restore/import safety model and archive contract first, then split optional cloud, encryption, scheduling, and portability layers into reviewed core or plugin work as appropriate.
