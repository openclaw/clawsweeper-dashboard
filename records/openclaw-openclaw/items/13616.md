## Summary

Keep open: current main and the latest release still provide first-party local backup creation and verification only, while native restore/import/export/list, cloud/encryption/scheduled backups, and component-aware cron/session restore remain unresolved. Community plugin/skill workarounds are useful context, but the prior maintainer-side comment kept the native umbrella open and current source still shows a product/security design gap.

## What This Changes

The issue requests first-party backup/export/import/restore/list commands for config, cron jobs, workspaces, session history, scheduled/cloud/encrypted backups, and disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella until maintainers define a safe first-party restore/import contract, split optional cloud sync/encryption pieces to plugin or ClawHub surfaces where appropriate, and land reviewed docs/tests for the chosen scope.

## Reproduction Assessment

Not applicable: this is a feature request for new first-party backup/restore workflows, not a broken existing contract. Source, docs, tests, and the latest release give high-confidence proof that current OpenClaw still lacks native restore/import/list/cloud and component-aware cron/session restore behavior.

## Solution Assessment

Unclear: the native recovery problem is valid, but the proposed all-in-one CLI/config/cloud/encryption scope is too broad and security-sensitive for an automated fix. A safer path is a maintainer-approved core restore contract, with optional cloud sync/encryption routed through plugin or ClawHub surfaces when core APIs are sufficient.
