## Summary

Keep open. Current main and v2026.5.12 still provide local backup archive create/verify only, while native restore/import/export/list/cloud/encryption/scheduled backups and component-aware cron/session restore remain unresolved; community tools like openclaw-workspace-sync and Keep My Claw are useful workarounds but do not complete the first-party lifecycle.

## What This Changes

Add a first-party backup/export/import/restore/list workflow for config, cron jobs, workspace/state, session history, and cloud/encrypted/scheduled backups.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and related GitHub state give high-confidence proof that create/verify exists while the requested restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.

## Solution Assessment

No, not as one monolithic autonomous implementation. The maintainable route is to define archive compatibility, restore safety, secret policy, and component semantics first, then review smaller slices such as https://github.com/openclaw/openclaw/pull/59280 and the related backup design items.
