## Summary

Keep open. Current main and v2026.5.12 still provide local backup archive create/verify only, while the requested first-party restore/import/export/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore remain unresolved; community tools such as openclaw-workspace-sync and Keep My Claw are useful workarounds but do not complete the native security-sensitive lifecycle.

## What This Changes

Request for built-in backup/restore commands and configuration covering OpenClaw config, cron jobs, workspace files, session history, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and related GitHub state give high-confidence proof that create/verify exists while the requested restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.

## Solution Assessment

No, not as one monolithic autonomous implementation.
