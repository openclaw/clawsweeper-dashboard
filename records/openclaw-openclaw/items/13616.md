## Summary

Current main provides first-party local archive creation and verification, but it still does not provide the central native restore/import lifecycle. Community tools cover much of the optional cloud workflow, while multiple open narrower items track migration, archive design, per-agent restore, and transcript preservation; this umbrella remains useful for a maintainer product and security decision.

## What This Changes

Requests a built-in backup and restore lifecycle covering configuration, cron jobs, workspace data, sessions, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Define a versioned, security-reviewed native restore contract for archives created by `openclaw backup create`, including dry-run validation, safe extraction, atomic rollback, explicit component and secret policies, while leaving cloud storage, scheduling, retention, and provider-specific encryption to plugins or ClawHub.

## Reproduction Assessment

Not applicable: this is a feature request, and current source provides high-confidence proof that native restore/import is absent rather than demonstrating a failure of an existing contract.

## Solution Assessment

No: implementing the entire proposal as one core subsystem is not the best path; the maintainable direction is a safe local archive restore contract with optional cloud automation owned by plugins or ClawHub.
