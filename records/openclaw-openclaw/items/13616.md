## Summary

Keep open: current main and the latest release still provide local backup archive creation and verification, but the native restore/import/list/cloud/encryption lifecycle and component-aware cron/session restore semantics are not complete. Community plugin/skill work and open PRs cover useful slices, but they do not close the first-party destructive restore and session-history design gap.

## What This Changes

The issue requests first-party backup and restore commands for configuration, cron jobs, session history, cloud storage, encryption, scheduled backups, and migration workflows.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.
