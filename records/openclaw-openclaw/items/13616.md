## Summary

Keep open: current main and v2026.6.6 still have first-party backup creation and verification only, while native restore/import/list/cloud/encryption/scheduled backup and component-aware cron/session restore remain unresolved. Community sync/backup tools and the open encrypted snapshot PR are useful context, but they do not close the native security-sensitive umbrella.

## What This Changes

Request to add native backup/restore tooling for OpenClaw config, cron jobs, session history, cloud storage, encryption, and scheduled backups.

## Best Possible Solution

Keep this as the native first-party backup lifecycle umbrella, scope the core restore/import contract around safe archive verification, extraction, component modeling for cron and sessions, secret policy, overwrite/rollback rules, and route optional cloud/scheduled integrations through plugins or ClawHub unless core APIs are missing.
