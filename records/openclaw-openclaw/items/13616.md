## Summary

Keep open: current main and the latest release provide local backup archive creation and verification, but the requested native restore/import/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore semantics remain unresolved. Community plugin/skill work covers useful backup and sync workflows today, but it does not close the first-party session-history and destructive native restore design gap.

## What This Changes

Request for native backup and restore utilities covering configuration, cron jobs, workspace/session history, cloud storage, encryption, scheduling, and migration workflows.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, latest-release inspection, and related GitHub state give high-confidence proof that create/verify exists while restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.

## Solution Assessment

No, not as one monolithic autonomous implementation.
