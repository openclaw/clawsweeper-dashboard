## Summary

Keep open: current main and v2026.5.12 still provide only local backup archive creation and verification, while native restore/import/export/list/cloud/encryption/scheduled backup plus component-aware cron/session restore remain unresolved and security-sensitive.

## What This Changes

Add a first-party backup and restore lifecycle for OpenClaw config, cron jobs, workspace files, skills, session history, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.
