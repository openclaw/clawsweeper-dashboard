## Summary

Keep this issue open. Current main provides first-party local archive creation and verification, but no general restore/import lifecycle; community tools cover useful cloud and workspace cases, while native cron/session recovery and safe core-state restoration remain unresolved. The scope is too broad and security-sensitive for automated implementation until maintainers separate the core restore contract from optional storage and scheduling integrations.

## What This Changes

Requests a first-party backup and restore lifecycle covering configuration, cron jobs, workspaces, session history, encryption, scheduling, and cloud storage.

## Best Possible Solution

Define a narrow, reviewed core contract for verifying and restoring local OpenClaw archives with dry-run, explicit confirmation, rollback, safe extraction, and component-aware state handling; keep cloud-provider storage and managed scheduling plugin-owned unless a missing core extension seam is demonstrated.

Not applicable: this is a feature request rather than broken existing behavior. Source and docs provide high-confidence confirmation that the requested general native restore lifecycle is not currently supported.

## Solution Assessment

No, not as one monolithic core feature. The maintainable solution is a security-reviewed local restore primitive with explicit component semantics, while optional cloud storage, retention, and scheduled synchronization remain plugin-owned where current extension surfaces suffice.
