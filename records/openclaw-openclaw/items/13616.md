## Summary

Keep open. Current main has first-party local backup archive creation and verification, plus migration pre-apply backups, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics; prior maintainer-side discussion also explicitly kept this native umbrella open despite useful community workarounds.

## What This Changes

Request a native backup/restore utility for OpenClaw config, cron jobs, session history, workspace state, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella: build on shipped create/verify archive support, define a safe restore/import contract with dry-run validation and explicit component modeling, then split cloud/encryption, scheduling, cron merge, session-history export/import, and docs into reviewed follow-ups while documenting community tools as optional workarounds.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is high-confidence because source, docs, tests, changelog, and issue discussion all show create/verify only with no native restore/import/list/cloud/encryption/scheduled-backup lifecycle.

## Solution Assessment

Unclear as a single implementation because the requested workflow is broad and security-sensitive. The maintainable direction is to settle archive format and restore safety first, then land smaller reviewed pieces for restore, cloud/encryption, scheduling, cron merge, and session-history semantics.
