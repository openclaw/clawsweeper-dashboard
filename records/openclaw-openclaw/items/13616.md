## Summary

Keep open. Current main and the latest release provide first-party local backup archive creation and verification only; the requested native restore/import/export/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore semantics remain unresolved, and maintainer-side discussion already kept this native umbrella open despite useful community workarounds.

## What This Changes

Request a built-in backup and restore utility for OpenClaw config, cron jobs, sessions, workspaces, optional cloud storage, encryption, retention, and scheduled disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella: build on shipped create/verify archives, define a safe restore/import contract with component modeling for cron and sessions, then split cloud/encryption, scheduling, and docs into reviewed follow-ups while documenting community tools as optional workarounds.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is high-confidence because source, docs, tests, release-tag inspection, and issue discussion all show create/verify plus migration pre-backup only, with no native restore/import/list/cloud/encryption/scheduled-backup lifecycle.

## Solution Assessment

Unclear as one monolithic implementation because the requested workflow is broad and security-sensitive. The maintainable path is to settle archive format, restore safety, and secret policy first, then land smaller reviewed pieces for restore, cron merge, session-history semantics, cloud/encryption, and scheduling.
