## Summary

Keep open: current main and v2026.5.19 still provide local backup creation and verification only, while the requested native restore/import/export/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore remain unresolved and security-sensitive.

## What This Changes

The issue requests built-in backup and restore utilities for OpenClaw config, cron jobs, workspace state, and session history, including cloud storage, encryption, scheduling, and selective import/export.

## Best Possible Solution

Keep this as the canonical native backup lifecycle umbrella, settle archive/restore safety and secret policy first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and related GitHub state give high-confidence proof that create/verify exists while restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.
