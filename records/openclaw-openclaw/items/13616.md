## Summary

Keep open: current main and v2026.6.6 still provide first-party backup archive create/verify, not the requested native restore/import/export/list/cloud/encryption/scheduled workflow or component-aware cron/session restore semantics. Community workspace-sync and Keep My Claw links in the discussion cover useful backup/sync use cases today, but they do not close the remaining first-party session-history and destructive-restore contract.

## What This Changes

Request a first-party backup/restore utility for OpenClaw config, cron jobs, session history, cloud/encrypted scheduled backups, migration, and disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella until maintainers define and land a reviewed first-party restore/import contract, with community tools documented as optional workarounds rather than replacements.

Not applicable: this is a feature request for a broader first-party backup/restore lifecycle, not a broken existing behavior report. Source, docs, tests, and the latest release give a high-confidence feature-gap check because only create/verify exists today.
