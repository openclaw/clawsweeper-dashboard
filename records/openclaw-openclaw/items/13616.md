## Summary

Keep open: current main and v2026.6.8 still provide first-party backup archive create/verify only, while native restore/import/export/list, scheduled/cloud/encrypted backup workflow, and component-aware cron/session restore semantics remain unresolved. The workspace-sync plugin and Keep My Claw skill linked in the discussion are useful workarounds, but they do not close the first-party session-history and destructive-restore contract.

## What This Changes

Request to add native backup/restore utilities for OpenClaw config, cron jobs, workspace files, and session history.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella until maintainers define and land a reviewed first-party restore/import contract, with community tools documented as optional workarounds rather than replacements.
