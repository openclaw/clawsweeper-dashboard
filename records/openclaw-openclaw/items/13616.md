## Summary

Keep open. Current main and the latest release still provide local backup create/verify only; native restore/import/list/cloud/encryption/scheduled-backup workflows and component-aware cron/session restore semantics remain unshipped, while the linked community plugin and skill are useful workarounds rather than core coverage.

## What This Changes

The issue requests a built-in backup and restore lifecycle for OpenClaw config, cron jobs, session history, and related local state.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella, build a reviewed restore/import foundation on the existing create/verify archive primitives, and split cloud, encryption, scheduling, cron merge, session-history handling, and docs into explicit follow-ups with a clear safety model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, related PRs, and the v2026.5.7 release show create/verify support exists while restore/import/list/cloud/encryption/scheduled lifecycle behavior remains absent.

## Solution Assessment

No, not as one monolithic autonomous implementation. The maintainable path is to settle archive compatibility, restore safety, secret policy, and component semantics first, then review smaller slices such as the open restore-command PR and later cron/session/cloud work.
