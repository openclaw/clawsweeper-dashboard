## Summary

Keep open: current `main` and v2026.6.6 still provide local backup archive creation and verification only, while native restore/import/list/cloud/encryption/scheduled backup and component-aware cron/session restore semantics remain unresolved. Community options such as `openclaw-workspace-sync` and `keepmyclaw` are useful workarounds, but they do not replace first-party session-history and destructive restore policy.

## What This Changes

The issue requests a built-in backup/restore utility for OpenClaw config, cron jobs, workspace files, session history, cloud storage, encryption, and scheduled backups.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on existing create/verify archives, with explicit restore safety, cron/session modeling, secret policy, cloud/encryption choices, and docs that point to community tools as optional workarounds.

## Reproduction Assessment

Not applicable: this is a feature request for a native lifecycle, not a broken existing behavior report. Source and docs provide a high-confidence feature-gap check because current `main` still lacks restore/import/list/cloud/encryption/scheduled backup commands.

## Solution Assessment

Unclear: native backup/restore is a plausible core direction, but the exact restore contract, archive format, secret handling, cron/session model, and cloud/encryption scope need maintainer product and security decisions first.
