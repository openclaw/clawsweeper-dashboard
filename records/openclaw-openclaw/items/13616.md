## Summary

Keep open. Current main and the latest shipped backup surface provide local `backup create` and `backup verify`, plus recent live-file hardening, but they still do not provide the requested native restore/import/export/list/cloud/encryption/scheduled lifecycle or component-aware cron/session restore semantics. Community options such as openclaw-workspace-sync, Keep My Claw, and OpenClaw Snapshot are useful workarounds and prior art, while the direct restore/snapshot/profile PRs remain open or closed rather than merged as a complete core solution.

## What This Changes

Request a native backup/restore lifecycle for OpenClaw config, cron jobs, session history, cloud/encryption/scheduling, and component-aware import/export.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella, settle a hardened restore/archive safety model first, then split restore, cron/session component semantics, cloud/encryption/scheduling, profile portability, and per-agent clone into reviewable follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, current-main CLI registration, and related GitHub checks give high-confidence proof that create/verify exists while the requested restore/import/list/cloud/encryption/scheduled lifecycle and cron/session semantics remain absent.

## Solution Assessment

No, not as one monolithic autonomous implementation. The maintainable path is to settle archive compatibility, restore safety, secret policy, and component semantics first, then review smaller slices such as https://github.com/openclaw/openclaw/pull/59280, https://github.com/openclaw/openclaw/pull/44111, and later cron/session/cloud work.
