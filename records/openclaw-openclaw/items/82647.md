## Summary

Keep open: this is a focused Control UI feature with sufficient screenshot proof, but the latest diff still puts ancillary quota loading on the chat refresh path and adds hardcoded quota copy outside the i18n catalog. The protected maintainer/product review can decide the final UI shape after those branch-local fixes.

## What This Changes

The branch adds a provider quota Usage card to Overview, a Chat header quota pill, shared quota-window formatting, refresh wiring, stale chat-run recovery, CSS, and focused UI tests.

## Best Possible Solution

Land the quota surfaces on the existing `models.authStatus.usage.windows` contract after decoupling quota refresh from Chat refresh completion and moving new visible copy into the Control UI i18n catalog.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Current-main source shows quota only in the Model Auth hint, and the inspected Crabbox screenshots show the proposed Overview card and Chat header pill.

## Solution Assessment

No as-is: reusing `models.authStatus.usage.windows` is the right boundary, but the branch should not await ancillary quota status before completing Chat refresh, and new visible quota copy should be localized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep quota status off the chat refresh path:** `ui/src/ui/app-settings.ts:409`
  - body: The Chat branch starts `loadModelAuthStatusState` for ancillary quota data, but then awaits it before scheduling scroll and finishing the tab refresh. Since `models.authStatus` can spend up to 3500 ms in provider usage enrichment, a slow auth-status call now delays the Chat refresh path even after `refreshChat` has returned. Fire the quota refresh in the background or wait for it only after the chat refresh completion work is done.
  - confidence: 0.88
- **[P3] Localize the chat quota pill copy:** `ui/src/ui/chat/session-controls.ts:122-154`
  - body: The new Chat quota pill builds visible and accessible text with hardcoded English fragments such as `resets`, `% left`, `Provider usage`, and `Usage`, while nearby controls use `t(...)`. Please add or reuse Control UI i18n catalog keys and regenerate locale outputs instead of shipping mixed-language header copy.
  - confidence: 0.82
- **[P3] Localize the overview reset hint:** `ui/src/ui/views/overview-cards.ts:65`
  - body: The Overview quota card localizes the label and percent-left value, but the reset fragment is still hardcoded as English `reset ...`. Put that hint through the Control UI i18n catalog with the other overview card strings so translated locales do not show mixed-language card text.
  - confidence: 0.76
