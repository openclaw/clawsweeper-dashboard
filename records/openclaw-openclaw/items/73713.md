## Summary

Keep open. Current main still routes `infer embedding create` provider failures through a default CLI catch that prints only `String(err)`, and the shared remote embedding HTTP helper still lets pre-response guarded-fetch failures escape without Voyage/provider context; the live Node 24/Voyage transport failure still needs credential-backed confirmation.

## What This Changes

The issue asks OpenClaw to audit the Voyage embedding CLI path for Node 24 fetch/Undici dispatcher mixing and to preserve the underlying fetch cause when provider embedding requests fail.

## Best Possible Solution

Keep the guarded-fetch and SSRF design, then make the CLI default error path and shared remote embedding transport cause-aware with redacted provider context and stable nested codes when available.

## Reproduction Assessment

Yes for the diagnostic gap: source inspection shows a caused fetch failure from `embedBatch` reaches `runCommandWithRuntime` and is printed with `String(err)`, and a Node check confirms that drops `UND_ERR_*` cause details. The exact live Voyage transport failure still needs credential-backed Node 24 confirmation.

## Solution Assessment

Yes for a narrow diagnostics repair. Reusing the existing redacted cause-aware formatter and wrapping pre-response remote embedding transport errors is more maintainable than adding a Voyage-only fetch path or changing dispatcher policy without live proof.
