## Summary

Keep open. Current main still has a source-reproducible diagnostics bug: remote embedding failures from the documented CLI path can propagate to the generic command wrapper and be printed with `String(err)`, which drops nested fetch causes such as undici error codes.

## What This Changes

The issue asks OpenClaw to make Voyage embedding CLI failures on Node 24 preserve the underlying fetch cause and to audit the Voyage embedding path for global-fetch/runtime-dispatcher mixing.

## Best Possible Solution

Use the existing redacted cause-aware formatter at the CLI boundary and wrap pre-response remote embedding transport failures with provider context while preserving the current SSRF/runtime-dispatcher design.

## Reproduction Assessment

Yes for the diagnostic gap: current source shows caused embedding fetch failures can reach the default CLI catch, and a focused Node proof shows `String(err)` drops the nested undici cause. The exact credential-backed Voyage transport failure was not live-reproduced in this review.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing redacted cause-aware formatter in `runCommandWithRuntime` and preserve provider context around shared remote embedding transport failures, not to add a Voyage-only fetch path or new config.
