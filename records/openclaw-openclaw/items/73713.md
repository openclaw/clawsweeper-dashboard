## Summary

Keep open. Current main still has the reported diagnostics gap: embedding provider failures can reach the default CLI catch path and be printed as `String(err)`, while pre-response memory remote transport failures do not get provider-context wrapping.

## What This Changes

The issue asks OpenClaw to audit the Voyage embedding CLI path under Node 24 and preserve underlying fetch causes such as `UND_ERR_*` instead of showing only `TypeError: fetch failed`.

## Best Possible Solution

Reuse the existing redacted cause-aware formatter at the CLI boundary and wrap remote embedding transport failures with provider context while preserving the shared guarded-fetch/SSRF dispatcher design.

## Reproduction Assessment

Yes for the diagnostics gap: source inspection shows a caused provider fetch failure reaches the default CLI catch path, and Node 24 `String(err)` drops the nested `UND_ERR_*` cause. The exact live Voyage request failure still needs credential-backed verification.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `formatErrorMessage` for default CLI errors and add shared provider-context wrapping for pre-response remote embedding HTTP failures, not to add a Voyage-only fetch implementation.
