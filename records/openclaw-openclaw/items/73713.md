Keep open. Current main still has the actionable diagnostic gap: `openclaw infer embedding create` uses a default CLI catch path that prints `String(error)`, and the shared remote embedding HTTP helper can let pre-response guarded-fetch failures escape without Voyage/provider context.

## What This Changes

The issue asks for the embedding CLI and Voyage remote embedding path to succeed on Node 24 or preserve the underlying fetch cause instead of printing only `TypeError: fetch failed`.

## Best Possible Solution

Keep the guarded-fetch design, then make the embedding CLI and shared remote embedding HTTP boundary cause-aware with redacted provider context and stable nested codes when available.

Yes. The narrow maintainable fix is to use the existing redacted cause-aware formatter in the CLI default catch path and wrap pre-response remote embedding transport errors while preserving the original cause/code.
