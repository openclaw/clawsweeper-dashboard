## Summary

Keep open. Current main still has the actionable diagnostic gap: caused embedding fetch failures can leave `infer embedding create` through the default CLI catch as only `TypeError: fetch failed`, and pre-response remote embedding transport errors still do not get provider context.

## What This Changes

The issue asks OpenClaw to audit the Voyage embedding CLI path for Node 24 fetch/Undici dispatcher mixing and preserve the underlying fetch cause for provider embedding failures.

## Best Possible Solution

Keep the guarded-fetch and SSRF design, then make the default CLI error path and shared remote embedding transport cause-aware with redacted provider context and stable nested codes when available.

## Reproduction Assessment

Yes for the diagnostic gap: source inspection shows a caused fetch failure from `embedBatch` reaches `runCommandWithRuntime` and is printed with `String(err)`, and a Node check confirms that drops `UND_ERR_*` cause details. The exact live Voyage transport failure still needs credential-backed Node 24 confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing redacted cause-aware formatter in the CLI default catch path and wrap pre-response remote embedding transport errors while preserving the original cause/code.
