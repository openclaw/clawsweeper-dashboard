## Summary

Keep open. Current main still has the source-reproducible diagnostic gap described in the prior ClawSweeper review: `infer embedding create` can let embedding provider fetch failures reach the default CLI catch path, where nested fetch causes are printed as `String(err)` and lose `error.cause` details.

## What This Changes

The issue asks for the Voyage embedding CLI path to succeed on Node 24 when the same request works directly, and to preserve the underlying fetch cause when provider embedding requests fail.

## Best Possible Solution

Reuse the existing redacted cause-aware formatter at the CLI boundary and wrap pre-response remote embedding transport failures with provider context while keeping the guarded-fetch/SSRF dispatcher design intact.

## Reproduction Assessment

Yes for the diagnostic gap: source inspection shows a caused fetch failure from `embedBatch` reaches the default command catch and is printed with `String(err)`. The exact live Voyage transport failure still needs Node 24 and credential-backed confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is cause-aware CLI error formatting plus shared embedding transport wrapping; changing Voyage-specific fetch behavior first would be broader than the source-backed problem requires.
