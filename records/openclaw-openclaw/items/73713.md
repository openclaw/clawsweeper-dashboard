Keep open. Current main still source-reproduces the diagnostic gap: embedding provider fetch failures can reach the CLI default catch path and be printed with `String(err)`, while pre-response remote embedding transport errors still lack provider-context wrapping.

## What This Changes

The issue asks OpenClaw to make `openclaw infer embedding create --provider voyage` work on Node 24 with valid Voyage credentials and preserve the underlying fetch cause when it fails.

## Best Possible Solution

Reuse the existing redacted cause-aware formatter at the CLI boundary and wrap pre-response remote embedding fetch failures with provider context, while leaving the SSRF/runtime-fetch dispatcher policy unchanged unless live Node 24 proof shows a separate transport bug.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse existing cause-aware, redacted error formatting and add shared provider-context wrapping for pre-response embedding HTTP failures rather than adding a Voyage-specific fetch path or configuration knob.
