## Summary

Keep open. Current main still has a source-reproducible diagnostics bug: a Voyage embedding transport error can propagate through the documented `infer embedding create` command and be printed with `String(err)`, while shared remote embedding transport only prefixes HTTP response failures, not pre-response fetch failures.

## What This Changes

The issue asks OpenClaw to audit and fix the Voyage `infer embedding create` path so valid embedding requests succeed and fetch failures expose their nested `error.cause` / `UND_ERR_*` details.

## Best Possible Solution

Embedding CLI failures should use the existing redacted cause-chain formatter, and shared remote embedding transport should wrap pre-response fetch failures with provider context while preserving the guarded-fetch/SSRF dispatcher design.

## Reproduction Assessment

Yes for the swallowed-cause failure path: source inspection shows a caused `embedBatch` failure reaches the default CLI catch and is printed with `String(err)`. The exact live Voyage/Node 24 transport failure still needs credential-backed confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing redacted cause-aware formatter at the CLI boundary and add shared provider-prefixed wrapping for pre-response embedding HTTP failures, rather than changing Voyage or the SSRF dispatcher policy first.
