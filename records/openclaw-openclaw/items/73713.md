Keep open. Current main still supports the reported embedding CLI path, but provider fetch failures pass through the generic CLI wrapper where `String(err)` drops nested fetch causes and undici codes, so the diagnostic half of the report remains valid.

## What This Changes

The issue requests an audit/fix for `openclaw infer embedding create --provider voyage` on Node 24 and cause-preserving diagnostics for embedding provider fetch failures.

## Best Possible Solution

Use the existing redacted cause-aware formatter at the CLI boundary, extend it to include useful nested codes if needed, and wrap shared remote embedding transport failures with provider/request context while preserving the guarded-fetch design.
