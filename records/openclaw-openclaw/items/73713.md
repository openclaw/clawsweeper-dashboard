## Summary

Keep open. Current main appears to have the dispatcher-routing protection for guarded fetches, but the documented embedding CLI path still lets Voyage remote embedding failures reach the generic CLI wrapper where `String(err)` drops nested fetch causes and undici codes.

## What This Changes

The issue requests that `openclaw infer embedding create` succeed on the Voyage path when guarded/direct fetch succeeds, and preserve nested fetch causes when it fails.

## Best Possible Solution

Use the existing redacted error formatter at the CLI boundary, extend it if needed to include nested error codes, and wrap shared remote embedding transport failures with provider context while preserving the current guarded-fetch design.

## Reproduction Assessment

Yes, for the diagnostic gap at source level: current main routes `embedding create` provider failures through `runCommandWithRuntime`, whose default catch still uses `String(err)`, and a focused Node proof shows that drops the nested undici cause. The live Voyage credential path itself was not rerun.

## Solution Assessment

Yes. The narrow maintainable fix is to make the CLI and shared embedding transport preserve redacted cause/code context, not to add a Voyage-only fetch workaround or a new configuration knob.
