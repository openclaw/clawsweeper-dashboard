## Summary

Keep open: current main and v2026.6.6 still expose only three static OpenRouter image model shortcuts, while OpenRouter's live catalog still has additional image-output models. The exact `microsoft/mai-image-2.5` example is no longer in the live OpenRouter catalog, but the broader provider catalog/support-policy request remains valid and is not superseded by another merged PR.

## What This Changes

The issue asks OpenClaw to support additional OpenRouter image-generation models, either by curating more shortcuts or by dynamically detecting image-capable OpenRouter models.

## Best Possible Solution

Keep the work provider-local and choose a maintained OpenRouter image catalog strategy, with docs and tests proving model listing, request payloads, and response extraction for newly advertised models.

## Reproduction Assessment

Yes for the source-level catalog/listing gap: current main and v2026.6.6 expose only three static OpenRouter image model shortcuts. The exact `microsoft/mai-image-2.5` example is not currently present in OpenRouter's live model API, and I did not run a paid live generation call.

## Solution Assessment

Unclear as an implementation path: adding individual ids is mechanically small, but the best long-term fix needs a maintainer choice between curated shortcuts and dynamic OpenRouter image-output discovery with stable capability mapping.
