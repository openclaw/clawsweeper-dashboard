## Summary

Keep open: current main and v2026.6.8 still expose only a static three-model OpenRouter image shortcut list, while OpenRouter's live image catalog includes the reporter's example. Explicit OpenRouter model overrides are already source-routable, so the remaining work is a provider catalog/support-policy decision rather than a strict bug fix.

## What This Changes

Request support for additional OpenRouter image-generation models, either by curated additions or dynamic OpenRouter catalog discovery.

## Best Possible Solution

Keep this provider-local and choose a maintained OpenRouter image catalog strategy: curated shortcuts with live proof, or auth-gated dynamic image-output discovery with conservative per-model capability mapping, docs, and focused tests.

## Reproduction Assessment

Yes for the source-level catalog gap: current main exposes three static OpenRouter image model shortcuts, while OpenRouter's live image-output catalog includes many more models, including the reporter's example. I did not run a paid generation request for that exact model.

## Solution Assessment

Unclear until maintainers choose the policy. Adding a single ID is mechanically small, but the more maintainable fix may be provider-local dynamic discovery with conservative capability mapping.
