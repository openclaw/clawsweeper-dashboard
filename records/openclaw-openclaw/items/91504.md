## Summary

Keep open: current main and v2026.6.9 still expose only three static OpenRouter image shortcuts and no image live catalog. The reporter’s example is present in OpenRouter’s current image-filtered model API, while explicit OpenRouter model overrides are already source-routable, so the remaining work is a provider catalog/support-policy decision rather than a strict bug fix.

## What This Changes

The issue requests broader OpenRouter image generation model support, either by adding curated model shortcuts or by dynamically discovering image-capable OpenRouter models.

## Best Possible Solution

Keep this provider-local and choose one supported OpenRouter image catalog strategy: curated shortcuts with live proof, or auth-gated dynamic image-output discovery with conservative capability mapping, docs, and focused tests.

## Reproduction Assessment

Yes for the source-level catalog gap: current main exposes three static OpenRouter image model shortcuts, while OpenRouter’s current image-filtered models endpoint includes many more models including microsoft/mai-image-2.5. I did not run a paid generation request for that exact model.

## Solution Assessment

Unclear until maintainers choose the policy. Adding one id is mechanically small, but the maintainable solution may be provider-local dynamic discovery with conservative capability mapping and live validation.
