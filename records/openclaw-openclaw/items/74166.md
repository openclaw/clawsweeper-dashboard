## Summary

Keep open: current main still hardcodes the Gateway chat-delta throttle at 150 ms and lacks the requested env/config resolver, but this is a new Gateway operator tuning contract that needs maintainer product/API approval.

## What This Changes

Request to make the Gateway chat-delta broadcast throttle configurable via `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the 150 ms default.

## Best Possible Solution

If accepted, add a Gateway-owned throttle resolver with default 150 ms, documented invalid/zero/bounds semantics, focused unit and agent-event coverage, and docs/changelog for the chosen operator surface.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-behavior check: `emitChatDelta` uses a fixed 150 ms threshold and no requested env/config resolver exists.

## Solution Assessment

Unclear pending maintainer product/API review. The proposed env var is narrow and default-preserving, but Gateway precedent includes documented config plus env precedence, so maintainers should choose the supported surface and bounds before implementation.
