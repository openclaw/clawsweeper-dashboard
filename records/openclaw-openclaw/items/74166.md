## What This Changes

Request to make the Gateway chat-delta broadcast throttle configurable via `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the current 150 ms default.

## Best Possible Solution

If accepted, add a Gateway-owned throttle resolver with the 150 ms default, clear invalid/zero semantics, focused regression coverage, and docs/changelog for the chosen operator surface.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current-behavior check: `emitChatDelta` still uses a fixed 150 ms threshold and the requested resolver/env var is absent.

## Solution Assessment

Unclear pending maintainer product/API review. The proposed env resolver is narrow and default-preserving, but existing Gateway timing precedent suggests maintainers should decide whether this should also be a documented config field.
