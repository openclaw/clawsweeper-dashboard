Keep open. Current main and the latest release still hardcode the Gateway chat-delta broadcast throttle at 150 ms with no resolver for the requested env var, and accepting a new core Gateway operator setting needs maintainer product/API judgment.

If accepted, add a Gateway-owned throttle resolver with explicit invalid, negative, empty, and `0` semantics, preserve the 150 ms default, and update focused tests plus operator-facing docs/changelog as maintainers prefer.
