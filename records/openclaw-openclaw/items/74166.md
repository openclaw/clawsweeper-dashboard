## Summary

Keep open. Current main and the latest release still hardcode the Gateway chat-delta broadcast throttle at 150 ms, and the requested env/config knob is a new core Gateway operator setting that needs maintainer product/API acceptance.

## What This Changes

Add an operator-controlled Gateway chat-delta broadcast throttle via `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the 150 ms default.

## Best Possible Solution

If accepted, add a Gateway-owned throttle resolver with explicit unset, empty, invalid, negative, fractional, and `0` semantics, keep the 150 ms default, and update focused tests plus docs/changelog according to the chosen supported surface.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection does confirm the current behavior: `emitChatDelta` uses a fixed 150 ms threshold, and tests cover suppression inside that window.

## Solution Assessment

Unclear pending maintainer product/API review. The proposed env resolver is narrow and default-preserving, but maintainers should first choose whether this belongs as an env-only operator knob, a config schema field, or no supported core setting.
