## Summary

Keep open. Current main and v2026.5.2 still hardcode the Gateway chat-delta broadcast throttle at 150 ms and have no `OPENCLAW_CHAT_DELTA_THROTTLE_MS` resolver; because this is a new core Gateway operator setting and the matching PR #74152 is closed/unmerged, it needs maintainer product/API acceptance. https://github.com/openclaw/openclaw/issues/74166

## What This Changes

Add an operator-configurable Gateway chat-delta broadcast throttle via `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the existing 150 ms default.

## Best Possible Solution

If accepted, add a Gateway-owned throttle setting with explicit invalid, negative, empty, and `0` semantics, preserve the 150 ms default, and update focused tests plus operator-facing docs/changelog as maintainers prefer.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection verifies the current fixed 150 ms guard and the existing test paths for sub-150 ms suppression and flushing.

## Solution Assessment

Unclear pending maintainer product/API review. The env-var proposal is narrow and default-preserving, but it creates a supported core Gateway setting and maintainers should choose the supported surface before implementation.
