The PR adds a shared effective Pi compaction reserve resolver, switches preflight compaction and memory-flush gates to use it, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land this patch or an equivalent refreshed core change after resolving the dirty head, keeping effective reserve resolution centralized so Pi settings, preflight compaction, and memory flush share the same reserve semantics.

Yes, after rebase or replacement. Centralizing effective reserve calculation in pi-settings and threading that value into both gates is the narrow maintainable fix, and it avoids changing the public MemoryFlushPlan plugin surface.
