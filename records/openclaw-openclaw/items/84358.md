## What This Changes

This PR adds an `authProfilesRepaired` flag to doctor repair sequencing and uses it to best-effort call `models.authStatus` with `refresh: true` after OAuth profile repairs.

## Best Possible Solution

Keep this PR open until contributor or maintainer proof shows `doctor --fix` repairing a legacy or stale OAuth profile while the gateway is running and the gateway auth status refreshes without a restart.

## Reproduction Assessment

Yes by source inspection, but not by a live run in this review. Current main has a 60-second gateway auth-status cache and doctor mutates auth profile files without refreshing the separate gateway process.

## Solution Assessment

Mostly yes: using the existing `models.authStatus` `refresh: true` path is the narrowest cross-process fix direction. The merge blocker is proof and validation, not a clear code-direction problem.
