The branch adds an auth-profile repair flag to doctor sequencing, calls `models.authStatus` with `refresh: true` after OAuth profile repairs, and adds flag tests.

## Best Possible Solution

Refresh the gateway runtime auth snapshot and invalidate the status cache after doctor auth-profile repairs, then prove both the runtime-snapshot and status-cache paths with tests plus redacted real command output.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: start a gateway with runtime auth-store snapshots, populate the `models.authStatus` cache, rewrite auth profile files through doctor repair, then call `models.authStatus` with `refresh: true`. The handler bypasses only its result cache while `ensureAuthProfileStore` can still return the active runtime snapshot.

## Solution Assessment

No. Calling `models.authStatus` with `refresh: true` is aimed at the daemon, but it is too narrow because it does not refresh active secrets/runtime auth snapshots before rebuilding status.
