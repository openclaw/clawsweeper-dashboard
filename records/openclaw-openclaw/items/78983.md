## Summary

Keep open: current main has fixed the flat/no-distinction part for spawned subagents by nesting them under parents, but the agent-switch preference still scans every target-agent row by recency and can auto-select a newer subagent session. The exact always-reset-to-main request conflicts with current tested behavior for recent dashboard sessions, so the narrow remaining bug is to exclude subagent/internal runtime rows from automatic agent-switch selection.

## What This Changes

The issue asks the Control UI to hide or distinguish sub-agent sessions in the session picker and to avoid carrying or selecting sub-agent sessions when switching agents.

## Best Possible Solution

Keep the landed visual nesting behavior, then change agent-switch preference so subagent/internal runtime rows are never auto-selected while normal recent dashboard sessions still work as documented.

## Reproduction Assessment

Yes, from source inspection: seed `sessionsResult.sessions` with a newer target-agent `agent:<id>:subagent:*` row and `resolvePreferredSessionForAgent` will return it because it only checks agent ownership and `updatedAt`. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Partly. The best narrow fix is not an unconditional reset to main; it should preserve current recent-dashboard selection while excluding subagent/internal runtime rows from automatic agent-switch preference.
