## Summary

Keep open: current main still lets subagent session rows appear as normal Control UI session-picker options, and agent switching can choose a target agent's newest subagent row. The related open PR only addresses visual grouping, while the requested main-session reset conflicts with current most-recent-dashboard behavior.

## What This Changes

The issue asks the Control UI to reset agent switches to the selected agent's main session and keep subagent sessions from appearing as undifferentiated session-picker choices.

## Best Possible Solution

Update Control UI session selection so agent changes never choose subagent sessions by default, then decide whether normal picker options should hide subagents or group them distinctly with regression coverage.

## Reproduction Assessment

Yes, from source inspection: a target agent with a newer `agent:<id>:subagent:*` row can be selected by `resolvePreferredSessionForAgent`, and `resolveSessionOptionGroups` includes subagent rows in the picker. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Unclear until maintainer UI policy is confirmed. Excluding subagents from automatic agent-switch selection is narrow, but hide-versus-group presentation and always-main versus most-recent-dashboard behavior overlap current tests and the related open PR.
