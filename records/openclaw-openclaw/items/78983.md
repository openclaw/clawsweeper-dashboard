## Summary

Keep open: current main fixed the visual grouping part via the merged subagent nesting PR, but agent switching can still select a newer subagent row because the preferred-session resolver does not filter subagent sessions.

## What This Changes

The issue asks the Control UI to avoid indistinguishable subagent session picker entries and to stop carrying users into a subagent session when switching agents.

## Best Possible Solution

Preserve the current recent-dashboard selection contract, but exclude subagent/internal rows from automatic agent-switch selection and keep the merged visual grouping behavior.

## Reproduction Assessment

Yes, source-level: current main's agent switch calls `resolvePreferredSessionForAgent`, and that helper can return a newer `agent:<id>:subagent:*` row because it only filters by target agent and `updatedAt`. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Yes for a narrow fix: skip subagent/internal rows when choosing the automatic session after an agent change, while leaving explicit subagent selection and the existing recent-dashboard behavior intact.
