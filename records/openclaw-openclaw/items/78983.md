## Summary

Keep open: current main still lets subagent rows flow into the Control UI chat session picker and the agent switcher can choose the most recently updated target-agent row, including a subagent. The open related pull request only covers visual grouping, not the reset-to-main/default-selection part.

## What This Changes

The issue requests that Control UI agent switching reset to the selected agent's main session and that subagent sessions be hidden or clearly separated in the session picker.

## Best Possible Solution

Update Control UI session selection so agent changes never land on subagent sessions by default, and settle the picker presentation by either hiding subagents from normal options or grouping them distinctly with regression coverage.

## Reproduction Assessment

Yes from source inspection: a target agent with a newer `agent:<id>:subagent:*` row can be selected by `resolvePreferredSessionForAgent`, and `resolveSessionOptionGroups` includes subagent rows in the picker. I did not run a live UI scenario in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed: excluding subagents from automatic agent-switch selection is narrow, but hide-versus-group presentation and always-main versus most-recent-dashboard behavior overlap existing tests and https://github.com/openclaw/openclaw/pull/78623.
