## Summary

Keep open: current main and the latest release still leave the generic child supervisor no-input path on inherited stdin, and the EBADF retry only disables detach rather than changing stdio. A newer open PR targets the service-managed stdin default, but it is not merged and does not fully cover the originally requested fallback or MCP-startup scope.

## What This Changes

The issue requests hardening child-process spawning under high gateway file-descriptor pressure by avoiding inherited stdin for service-managed gateways and adding a safer EBADF retry path.

## Best Possible Solution

Review and land a narrow supervisor-spawn hardening fix, or a replacement for the open related PR, that preserves no-input EOF semantics, adds a stdio-changing EBADF fallback where needed, and explicitly covers or splits MCP stdio startup.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The current source still has the inherited-stdin default and no stdio-changing EBADF fallback that match the reported mitigation gap.

## Solution Assessment

Partly: avoiding inherited stdin for service-managed no-input children is a maintainable direction, but the complete fix should preserve EOF semantics and address or split the MCP direct-spawn path. Current main is not fixed, and the open related PR is only a partial landing path until reviewed.
