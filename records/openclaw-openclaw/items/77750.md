## Summary

Keep open: current main and v2026.6.10 still default omitted child stdin to inherited stdin and only retry EBADF by dropping detachment. Host exec is partly mitigated through explicit pipe-closed stdin, but the linked open PR is unmerged and does not settle the requested fallback or MCP sibling-spawn scope.

## What This Changes

The issue requests hardening gateway child-process spawns so high file descriptor pressure or service-managed stdin does not make exec and helper startup fail with EBADF.

## Best Possible Solution

Review, prove, and land or replace https://github.com/openclaw/openclaw/pull/87527, then explicitly decide whether the same narrow hardening should add a closed-pipe EBADF fallback and MCP direct-spawn coverage.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection confirms current main still has inherited omitted stdin plus a no-detach-only EBADF fallback, matching the reported mitigation gap.

## Solution Assessment

Yes for keeping it open. The best current path is to finish the open candidate PR or replace it with a focused supervisor hardening change that preserves EOF semantics and covers the remaining fallback scope.
