## Summary

Keep open: current main still preserves the generic child-adapter inherited-stdin default and only retries EBADF by disabling detach, so the requested spawn hardening is not implemented. Some related FD-pressure sources and host exec callers are mitigated, but the linked fixing PR was closed unmerged and no canonical replacement owns this exact adapter fallback work.

## What This Changes

The issue asks OpenClaw to prevent gateway child-process spawn EBADF under high file-descriptor pressure by changing supervisor stdin defaults and adding a safer EBADF fallback.

## Best Possible Solution

Land a focused child-adapter hardening that preserves explicit interactive stdin behavior, uses EOF-safe no-input semantics for service/non-TTY spawns, and adds an EBADF retry that switches both detach and stdio to a safe shape.

## Reproduction Assessment

Yes for the source-level invariant: current main still has the inherited-stdin default and no stdio-changing EBADF fallback in createChildAdapter. No high-confidence live macOS high-FD reproduction was run in this read-only review.

## Solution Assessment

Yes, a narrow supervisor child-adapter repair is the best remaining path, but it should not copy the closed PR's open-pipe default because that can change EOF behavior for no-input commands.
