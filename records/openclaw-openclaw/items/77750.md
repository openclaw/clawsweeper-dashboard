## Summary

Keep open: current main and the latest release still leave the generic child supervisor no-input path on inherited stdin, and the EBADF retry only disables detach rather than switching stdio. Host exec and the memory watcher root cause have been partially mitigated elsewhere, but the requested supervisor spawn hardening remains unlanded and the linked fixing PR is closed unmerged.

## What This Changes

The issue asks for gateway child-process spawning to survive high file-descriptor pressure by avoiding inherited stdin in service/non-TTY no-input spawns and adding a safer EBADF retry.

## Best Possible Solution

Land a narrow supervisor-spawn hardening fix that preserves interactive stdin and no-input EOF, adds a stdio-changing EBADF fallback, and explicitly handles or splits the MCP direct-spawn scope.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The issue body provides concrete launchd/macOS logs, and current source still has the inherited-stdin plus no-stdio-changing-fallback path that matches the reported mitigation gap.

## Solution Assessment

Mostly yes, but the closed PR's open-pipe approach is not the best final shape. The maintainable fix should use EOF-preserving pipe-closed behavior for no-input service spawns and add fallback coverage without changing interactive stdin semantics.
