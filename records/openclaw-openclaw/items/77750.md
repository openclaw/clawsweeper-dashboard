## Summary

Keep open: current main still has the reported source path, and the open closing PR #77751 should be reviewed and merged or closed before this issue is resolved.

## What This Changes

The issue reports gateway child process spawns failing with `EBADF` when a service-managed gateway has a very large inherited file descriptor table.

## Best Possible Solution

Review and land PR #77751, or an equivalent narrow supervisor fix that avoids inherited stdin for noninteractive service spawns and adds regression coverage without changing interactive TTY behavior.

## Reproduction Assessment

Yes, at source level: current main still defaults unset child-adapter stdin to `inherit`, and at least one supervisor call path can leave `stdinMode` unset. This review did not live-reproduce the macOS high-fd `EBADF` failure.

## Solution Assessment

Yes, the supervisor child adapter is the narrow maintainable boundary because several higher-level runners funnel through it. The linked PR still needs normal review and merge before this issue can close.
