## What This Changes

Dynamic TTS tool media should auto-deliver trusted generated audio in Discord and Telegram message-tool-only channel contexts without exposing ordinary final text.

## Best Possible Solution

Land a scoped media-only suppression bypass for trusted local dynamic TTS media, with regression coverage that proves audio is delivered while ordinary final text stays private.

## Reproduction Assessment

Yes, source-reproducible: current main queues trusted TTS media, flushes it through block/final reply paths, and suppresses those paths in `message_tool_only` contexts before provider send. The issue also includes fresh Discord and Telegram traces with valid staged Opus media and no outbound send.

## Solution Assessment

No, current main is not the best solution because it does not satisfy the TTS auto-delivery contract. The narrow maintainable fix is a trusted-media-only bypass that strips ordinary text before delivery.
