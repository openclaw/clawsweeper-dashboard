## What This Changes

The issue reports that successful dynamic TTS tool results stage trusted Opus media but are not delivered to Discord or Telegram when the originating channel uses `message_tool_only` source-reply policy.

## Best Possible Solution

Route only trusted dynamic TTS media through a media-only source-suppression bypass while keeping ordinary final text private in message-tool-only channel turns.

## Reproduction Assessment

Yes, source-reproducible: the issue includes Discord and Telegram traces, and current main still queues trusted TTS media before suppressing block/final delivery in message_tool_only paths.

## Solution Assessment

No, current main still does not satisfy the existing TTS auto-delivery contract. The narrow fix is to mark only trusted dynamic TTS media for media-only delivery despite source-reply suppression without making normal final text visible.
