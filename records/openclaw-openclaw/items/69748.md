## Summary

Keep open. Current main still lacks the proposed Discord `bypass`/`mention`/`deny` access-list model, `inherit` channel cascade, and `perm:*` resolution, while narrower per-user mention-policy issues were already closed into this broader canonical design discussion.

## What This Changes

Request a Discord config redesign replacing `requireMention` and flat sender allowlists with bypass, mention, and deny access lists that support users, roles, permissions, and channel-level cascading overrides.

## Best Possible Solution

Keep this as an open Discord plugin design task: define the public access-list schema, preserve intentional wildcard/channel behavior, add migration/docs/tests, and include a focused security review of precedence and access widening.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config redesign. Source inspection gives high confidence that current main lacks the proposed fields and still gates Discord guild senders with `users`/`roles` plus one `requireMention` boolean.

## Solution Assessment

Unclear until maintainers approve the public config names, defaults, precedence, permission shortcuts, and migration path. The amended proposal is plausible, but it changes security-sensitive Discord ingress semantics rather than being a narrow repair.
