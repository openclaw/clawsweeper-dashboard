## Summary

Keep open. Current main still exposes Discord guild/channel ingress through `requireMention`, `users`, and `roles`, then applies a single mention gate after sender allowlist checks; the updated proposal coherently folds in open #73387 but remains unimplemented and security-sensitive.

## What This Changes

Replace Discord `requireMention`/`users` access configuration with deny-first `bypass`/`mention`/`deny` lists, role and permission references, and groupPolicy-aware per-channel inheritance.

## Best Possible Solution

Keep this as an open Discord plugin design task: add a maintainer-approved access resolver and config schema, preserve legacy fields through warning/doctor migration, coordinate #73387 channel inheritance, and update runtime, docs, tests, and security audit together.

## Reproduction Assessment

Yes for the current-main gap. Source, docs, and tests show the schema and runtime still use `requireMention` plus `users`/`roles`, and configured channel misses are still denied instead of inheriting guild defaults under `groupPolicy: "open"`.

## Solution Assessment

Unclear. The proposed shape addresses the reported all-or-nothing problem, but the access-control migration and groupPolicy-aware inheritance need maintainer/security approval before this becomes the narrowest maintainable implementation.
