## Summary

Keep open. Current main still implements Discord ingress with `requireMention`, `users`, and `roles`; the requested `bypass`/`mention`/`deny` access-list model, `perm:*` shortcuts, and wildcard-plus-specific channel cascade are not implemented, and the latest discussion keeps this as a maintainer-approved Discord access-control redesign rather than a stale duplicate.

## What This Changes

The issue requests replacing Discord `requireMention` and flat user allowlists with layered `bypass`, `mention`, and `deny` access lists that support users, roles, permission shortcuts, and per-channel cascade overrides.

## Best Possible Solution

Keep this as an open Discord plugin design task: approve the public access-list semantics, preserve intentional wildcard/channel behavior, add typed schema/runtime/docs/tests, and include focused security review for migration and precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config redesign. Source inspection gives high confidence that current main lacks the proposed fields and still gates with `users`/`roles` plus one `requireMention` boolean.

## Solution Assessment

Unclear until maintainers approve the config names, defaults, precedence, permission shortcuts, and migration path. The amended proposal is plausible, but it changes security-sensitive Discord ingress semantics rather than being a narrow repair.
