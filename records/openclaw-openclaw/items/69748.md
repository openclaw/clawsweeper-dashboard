## Summary

Keep open. Current main still uses Discord `users`/`roles` sender allowlists plus a single `requireMention` boolean, while this issue now asks for a public deny/bypass/mention access-list redesign with role/permission entries and wildcard-specific channel cascade semantics that remain unimplemented and require maintainer product/security review.

## What This Changes

The issue requests replacing Discord `requireMention` and flat sender allowlists with deny/bypass/mention access lists, role and permission entries, migration behavior, and per-channel cascade overrides.

## Best Possible Solution

Keep this as an open Discord plugin design task and implement it only after maintainers approve the access-list schema, deny precedence, wildcard cascade rules, migration warnings, docs, and focused security tests.
