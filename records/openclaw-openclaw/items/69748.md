## Summary

Keep open. Current main still uses the legacy Discord `requireMention` plus `users`/`roles` model and wildcard fallback replacement, so the amended access-control proposal remains unimplemented and needs maintainer/security design review.

## What This Changes

Request replacing Discord guild/channel `requireMention` and `users` access controls with `bypass`/`mention`/`deny` lists, role/permission entries, and a channel cascade that preserves wildcard allowlist semantics.

## Best Possible Solution

Add a maintainer-approved Discord access resolver in the Discord plugin with schema, docs, doctor migration, and regression coverage for legacy compatibility, deny precedence, role/permission resolution, wildcard cascade, and channel allowlist behavior.

## Reproduction Assessment

Not applicable as a feature/config redesign. Source inspection confirms current main cannot express `bypass`/`mention`/`deny` or the proposed channel cascade because it only accepts `requireMention`, `users`, and `roles`, with wildcard as fallback rather than a merge layer.

## Solution Assessment

Unclear. The amended proposal addresses the all-or-nothing access problem while preserving existing wildcard semantics, but the final schema, precedence, and legacy migration path need maintainer/security approval before this is the best maintainable fix.
