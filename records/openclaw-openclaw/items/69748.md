## Summary

Keep open: current main and v2026.6.8 still use Discord `requireMention` plus `users`/`roles`, and they lack the requested `bypass`/`mention`/`deny`, deny-first, permission shortcut, and channel-cascade semantics.

## What This Changes

The issue requests replacing Discord guild/channel `requireMention` and `users` access with bypass/mention/deny access lists, role/permission entries, and per-channel cascade overrides.

## Best Possible Solution

Approve one canonical Discord plugin access-control contract, then implement it in schema, runtime, docs, tests, and doctor migration with deny-first behavior covered.

## Reproduction Assessment

Yes, at source level: current config, schema, and runtime cannot express owner bypass plus everyone-else mention access or deny-first sender rules. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent contract; the proposal is coherent, but the safest path is a single Discord-owned schema plus doctor migration rather than long-lived runtime compatibility for old fields.
