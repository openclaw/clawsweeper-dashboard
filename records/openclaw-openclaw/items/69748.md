## Summary

Keep open. Current main still models Discord guild/channel ingress with `requireMention`, `users`, and `roles`, and channel wildcard matching remains fallback/replacement rather than the proposed guild + `*` + specific cascade; this is a real Discord access-control feature/design task, not implemented cleanup.

## What This Changes

The issue requests a Discord guild/channel access-control redesign replacing `requireMention` and flat `users` lists with deny-first `bypass`/`mention`/`deny` access lists, role/permission entries, legacy migration, and wildcard/channel cascade semantics.

## Best Possible Solution

Keep this as an open Discord plugin design task: add a maintainer-approved access resolver and config schema, preserve current wildcard allowlist semantics, provide legacy warnings/doctor migration, and land docs plus security-focused regression tests together.

## Solution Assessment

Unclear. The amended cascade/access-list model addresses the reported limitation while preserving the wildcard allowlist model, but it changes Discord ingress authorization and needs maintainer/security design before it is the narrowest maintainable implementation.
