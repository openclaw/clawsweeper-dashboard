## Summary

Keep open: current main and v2026.6.10 still use Discord `requireMention` plus `users`/`roles` allowlists, while the requested `bypass`/`mention`/`deny` model, permission shortcuts, channel cascade, and migration path remain unimplemented and need maintainer product/security review.

## What This Changes

The issue requests replacing Discord `requireMention`/`users` authorization with `bypass`/`mention`/`deny` access lists, role/permission entries, migration behavior, and per-channel cascade overrides.

## Best Possible Solution

Keep this as the canonical Discord access-control redesign until maintainers approve the schema, precedence, doctor migration, and security posture, then implement it in the Discord plugin with focused docs and tests.

## Reproduction Assessment

Yes, at source level. Current code can choose only one `requireMention` boolean plus sender `users`/`roles` allowlists, so it cannot represent an owner bypassing mentions while everyone else can trigger only by mentioning the bot.

## Solution Assessment

Unclear as a final design. The direction addresses the real gap, but maintainers need to choose the permanent schema, precedence, permission shortcut contract, and doctor migration rather than accepting an ad hoc runtime compatibility layer.
