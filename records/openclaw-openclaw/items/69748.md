Keep open. This is still the canonical remaining Discord access-control redesign: current main keeps `requireMention`, `users`, and `roles`, while narrower per-sender mention issues were closed into this issue and the channel-override thread was closed around the existing wildcard model.

## Best Possible Solution

Keep this as an open Discord plugin design task: define the public access-list semantics, preserve the wildcard-channel model intentionally, add one typed resolver and schema, provide legacy migration or doctor guidance, and cover runtime, docs, tests, and security audit together.
