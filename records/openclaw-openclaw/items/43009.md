## Summary

Keep open: current main and the latest release still have the source-reachable TUI footer path that renders a missing token total with a known context window as `tokens ?/<ctx>`, and an open draft pull request already references this issue with closing syntax.

## What This Changes

The issue asks the TUI footer to display a real context-token total such as `0/200k (0%)` instead of `?/200k` when the context window is known.
