## Summary

Keep open: current main still has a TUI footer path where a known context window and missing session total render as `tokens ?/<ctx>`. The prior formatter-only PR was closed unmerged because guessing zero would erase the intentional unknown-vs-zero distinction, so the remaining fix belongs in session usage propagation.

## What This Changes

The issue requests that the TUI status/footer display actual context token usage instead of `?/200k` when the gateway/status data has a concrete value.

## Best Possible Solution

Propagate a concrete fresh or transcript-derived `totalTokens` to the TUI when usage is known, while preserving `?/<ctx>` for genuinely stale or missing totals.

## Reproduction Assessment

Yes for a source-level reproduction: current main still formats a null total plus known context as `tokens ?/200k`, and TUI refresh can hold context defaults while no row total is present. I did not run a live TUI/Gateway session in this read-only pass.

## Solution Assessment

Yes. The maintainable fix is at the session-data/TUI contract boundary; changing the formatter to coerce all unknown totals to zero would contradict the existing freshness contract and tests.
