## Summary

Keep open: current main still has a source-reachable TUI footer path where a known context window plus missing session total renders as `tokens ?/<ctx>`, and the closed formatter-only PR is not the right fix because current tests preserve unknown totals as distinct from zero.

## What This Changes

The issue reports that the TUI footer renders context token usage as `?/200k` even when gateway status has concrete token data.

## Best Possible Solution

Propagate a concrete fresh or transcript-derived `totalTokens` to the TUI when usage is known, while preserving `?/<ctx>` for genuinely stale or missing totals.

## Reproduction Assessment

Yes, at source level: `formatTokens(null, 200000)` still renders the reported `tokens ?/200k` shape, and the TUI refresh path can retain context defaults while no row total is present. I did not run a live TUI/Gateway session.

## Solution Assessment

Yes: the narrow maintainable fix is at the session-data/TUI contract boundary so a real total is supplied when one exists. Coercing unknown totals to zero in the formatter would underreport nonzero unknown usage and conflicts with current freshness tests.
