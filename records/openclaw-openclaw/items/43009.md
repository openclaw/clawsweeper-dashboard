## Summary

Keep this open: current main still has the reported TUI footer path where a known context window and missing total render as `tokens ?/<ctx>`, and the only related implementation PR is closed and unmerged.

## What This Changes

The issue requests that the TUI footer display concrete context token usage instead of `?/200k` when the session status path has a known zero value.

## Best Possible Solution

Feed the TUI footer a concrete fresh total only when session/gateway data knows it, preserve unknown stale totals, and add focused coverage for both `0/ctx (0%)` and `?/ctx`.

## Reproduction Assessment

Yes. Source inspection on current main shows `formatTokens(null, 200000)` renders `tokens ?/200k`, and the footer passes nullable `sessionInfo.totalTokens` with a known `contextTokens` value into that formatter.

## Solution Assessment

Yes. The narrow maintainable repair is at the session-data/TUI contract boundary; the closed formatter-only PR is not the best fix because it would convert all unknown totals with known context into zero.
