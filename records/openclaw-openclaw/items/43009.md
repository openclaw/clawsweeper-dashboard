## Summary

Keep open: current main still has a source-reachable TUI footer path where a known context window and missing or filtered session total renders as `tokens ?/<ctx>`. The related formatter-only PR is closed unmerged because the durable fix belongs in session usage propagation, not in guessing zero for unknown totals.

## What This Changes

The issue requests that the TUI footer show actual context token usage instead of `?/200k` when the gateway has real usage data.

## Best Possible Solution

Propagate a concrete fresh or transcript-derived `totalTokens` to the TUI when usage is known, while preserving `?/<ctx>` for genuinely stale or missing totals.

## Reproduction Assessment

Yes for a source-level reproduction: current main still formats a null total plus known context as `tokens ?/200k`, and the TUI refresh path can retain context defaults while no row total is present. I did not run a live TUI/Gateway session in this read-only review.

## Solution Assessment

Yes: the maintainable fix is at the session-data/TUI contract boundary, so the TUI receives a real total when one exists. Changing the formatter to coerce unknown totals to zero would conflict with the current freshness contract and regression tests.
