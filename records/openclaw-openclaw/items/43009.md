## Summary

Keep open: current main and the latest release still have the TUI footer path that renders a missing total with a known context window as `tokens ?/<ctx>`. The newer merged context-budget work affects `/status` and gateway session rows, not the TUI footer, and https://github.com/openclaw/openclaw/pull/82737 remains the open closing PR for the narrow TUI fix.

## What This Changes

The issue requests that the TUI footer show a numeric context token value such as `0/200k` or `1.2k/200k` instead of `?/200k` when the context window is known.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82737, or an equivalent narrow TUI footer fix that preserves stale-token semantics, then close this issue after merge.

## Reproduction Assessment

Yes, at source level: current main passes a missing total and known context into `formatTokens`, which renders `tokens ?/<ctx>`. I did not run a live interactive TUI/Gateway session in this read-only review.

## Solution Assessment
