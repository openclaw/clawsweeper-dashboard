## Label Justifications

- `impact:session-state`: The footer inaccurately represents the active session's context-usage state when usable session context data exists elsewhere.

## Summary

Keep open: current main and the latest release still render the TUI footer from exact `totalTokens` plus `contextTokens` only, while the gateway/status layer already has freshness and estimated context-budget data that the TUI does not carry. The closed unmerged zero-token PR was useful context, but it does not supersede the freshness-aware TUI repair.

## What This Changes

The issue requests that the OpenClaw TUI footer show actual context token usage instead of `tokens ?/<context>` when the session has usable context data.

## Best Possible Solution

Carry `totalTokensFresh` and `contextBudgetStatus` through the TUI session metadata, render fresh exact totals when available, render estimated context usage with explicit estimated notation when exact usage is unavailable, and keep `?` only for genuinely unknown context usage.

## Reproduction Assessment

Yes. Current main can hold `contextTokens` without a fresh exact `totalTokens`, and the TUI footer deterministically renders that state through `formatTokens` as `tokens ?/<context>`.

## Solution Assessment

Yes. The best fix is freshness-aware TUI rendering that reuses existing gateway/status semantics, not a zero-default display that invents exact usage when the value is stale or estimated.
