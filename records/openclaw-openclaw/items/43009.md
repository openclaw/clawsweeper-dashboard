## Summary

Keep this open. Current main still has the reported TUI footer path where a known context window and missing/unknown total render as `tokens ?/<ctx>`, while the safe fix needs to preserve the distinction between a fresh zero and an unknown total.

## What This Changes

The issue requests that the TUI status bar show concrete context token usage instead of `tokens ?/200k` when the gateway/session status has usable token data.

## Best Possible Solution

Feed the TUI footer a concrete fresh total only when session/gateway data knows it, preserve unknown stale totals, and add focused coverage for both `0/ctx (0%)` and `?/ctx`.

## Reproduction Assessment

Yes. Source inspection on current main shows `formatTokens(null, 200000)` renders `tokens ?/200k`, and the footer passes nullable `sessionInfo.totalTokens` with a known context window into that formatter.

## Solution Assessment

Yes for a narrow session-data/TUI contract repair. The closed formatter-only PR is not the best solution because it would convert all unknown totals with known context into zero.
