## Summary

Keep open: current `main` and the latest release still have the TUI footer path that renders a missing total with a known context window as `tokens ?/<ctx>`. The merged context-budget work helps `/status` and gateway session rows, but the linked open fix PR still owns the narrow TUI footer repair.

## What This Changes

The issue asks for the TUI footer to show a numeric context-token value such as `0/200k` or `1.2k/200k` instead of `?/200k` when session context information is available.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82737, or an equivalent narrow TUI footer fix that preserves stale-token semantics, then close this issue after merge.
