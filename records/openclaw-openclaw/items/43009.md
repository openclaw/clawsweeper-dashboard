## Summary

Keep this open. Current main still has the reported TUI footer path where a known context window and missing total render as `tokens ?/<ctx>`, and the only related implementation PR is closed/unmerged with a review note against the unsafe null-to-zero shortcut.

## What This Changes

The issue requests that the TUI status bar show concrete context token usage instead of `tokens ?/<ctx>` when usage is known through session/status data.

## Best Possible Solution

Align the TUI footer with session/status semantics by supplying a concrete fresh total when known, preserving unknown totals otherwise, and adding focused coverage for both `?/ctx` and `0/ctx (0%)` cases.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence static reproduction: `formatTokens(null, 200000)` renders `tokens ?/200k`, and the footer passes nullable `sessionInfo` token fields directly into that formatter.

## Solution Assessment

No, the closed formatter-only PR is not the best way to solve the issue because it converts all unknown totals with known context into zero. The safer fix is to make session/list/status data supply a concrete fresh zero or derived total only when that value is known, then keep unknown values visibly unknown.
