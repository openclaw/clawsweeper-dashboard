## Summary

Keep open: current main still has a source-reachable TUI footer path where a known context window plus missing session total renders as `tokens ?/<ctx>`, and the closed formatter-only pull request is not the right fix because the session freshness contract intentionally preserves unknown totals.

## What This Changes

The issue reports that the TUI footer shows `tokens ?/200k` instead of a concrete token count even when session status has usage data.

## Best Possible Solution

Propagate a concrete fresh or transcript-derived `totalTokens` to the TUI when usage is known, while preserving `?/<ctx>` for genuinely stale or missing totals.

## Reproduction Assessment

Yes, at source level: `formatTokens(null, 200000)` still renders the reported `tokens ?/200k` shape, and the TUI refresh path can keep a known context from row/defaults without assigning a total. I did not run a live TUI/Gateway session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is at the session-data/TUI contract boundary so a real total is supplied when one exists. Coercing unknown totals to zero in the formatter would underreport nonzero unknown usage and conflicts with current freshness tests.
