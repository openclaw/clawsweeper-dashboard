## Summary

Keep open: current main and v2026.6.8 still classify the reported Gemini `current quota` plus `/rate-limits` 429 as long-window, and both linked fix PRs were closed unmerged after maintainers rejected the config-only and URL-only approaches.

## What This Changes

The issue asks OpenClaw to retry Gemini TPM/RPM 429s as short-window same-model rate limits and originally proposed making the same-model retry budget configurable.

## Best Possible Solution

Keep this issue open for a maintainer-approved, provider-scoped Gemini classifier fix with positive TPM/RPM and negative RPD coverage, while routing any public retry-budget config design to the broader retry-policy tracker.

## Reproduction Assessment

Yes, at source level: current main matches `current quota` as long-window and has no `rate-limits` short-window token, so the reported Gemini string never reaches same-model retry. I did not run a live Gemini quota-exhaustion test.

## Solution Assessment

No, not as the original combined request or the closed URL-only PR shape. The narrow classifier belongs in `assistant-failover.ts`, but it needs provider-scoped or structured evidence that distinguishes minute windows from daily quotas; the config knob needs a separate product contract.
