## Summary

Keep open: current main and `v2026.6.8` still classify the reported Gemini `current quota` plus `/rate-limits` 429 text as long-window, and the linked implementation attempts were closed unmerged. The remaining work needs maintainer/provider-specific classifier direction rather than cleanup closure.

## What This Changes

This issue asks OpenClaw to route Gemini TPM/RPM 429 quota messages into the bounded same-model retry path after the original configurable retry-count proposal was found insufficient.

## Best Possible Solution

Keep this issue open for a maintainer-approved, provider-scoped classifier fix backed by verified short-window and long-window Gemini payloads, with any retry-count config handled separately only if accepted as public policy.

## Reproduction Assessment

Yes at source level: current main classifies `current quota` as long-window unless an RPM/TPM/per-minute token or short Retry-After is present, and the reported `/rate-limits` URL is not recognized. I did not run a live Gemini quota-exhaustion test in this read-only review.

## Solution Assessment

No, neither rejected path is the best solution: increasing the retry count does not reach the classifier path, and a generic `/rate-limits` regex was rejected as too broad. The maintainable fix is a provider-scoped or structured classifier boundary with positive and negative regression coverage.
