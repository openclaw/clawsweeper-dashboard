## What This Changes

Expose the hardcoded same-model rate-limit retry cap as a configurable OpenClaw setting for longer provider quota recovery windows.

## Best Possible Solution

Choose one canonical retry contract: either add a bounded `auth.cooldowns` same-model retry budget with schema/help/docs/tests, or handle longer Gemini quota windows through existing cooldown/fallback policy without a new config key.

## Reproduction Assessment

Yes for the source-level claim: current main has a fixed three-retry same-model budget and no matching config key. This is still a feature/config request rather than a broken existing contract with a live failing reproduction.

## Solution Assessment

Unclear. A configurable retry cap is plausible, but the best maintainable solution may be a narrower retry-window policy adjustment or one canonical cooldown setting rather than another independent retry knob.
