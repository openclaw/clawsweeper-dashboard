## Summary

Keep open: current main and the latest release still route the exact reported Gemini `current quota` plus `/rate-limits` 429 away from the same-model retry path, and both related fix PRs were closed unmerged with maintainer comments saying the issue remains valid.

## What This Changes

The issue requests that Gemini TPM/RPM 429 quota errors trigger bounded same-model retry instead of immediate provider/profile fallback, with a possible configurable retry budget.

## Best Possible Solution

Add a provider-scoped classifier fix that retries only confirmed Gemini minute-scale TPM/RPM failures while preserving daily/RPD fallback behavior, and handle any retry-budget config through an explicit maintainer retry-policy decision.

## Reproduction Assessment

Yes, at source level. The exact reported Gemini text includes `current quota` and the shared `/rate-limits` URL, while current main lacks `rate-limits` as a short-window token and therefore returns `null` before same-model retry.

## Solution Assessment

Unclear as originally proposed. The classifier layer is the right narrow bug-fix surface, but the retry-count config request should remain a separate maintainer decision unless existing retry policy cannot solve it.
