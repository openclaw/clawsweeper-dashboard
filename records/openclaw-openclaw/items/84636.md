## Summary

Keep this PR open: the prior Markdown round-trip concern appears addressed, but the new recent-snapshot reader bypasses the memory SDK's existing symlink-safe file-read contract, so merging as-is could let continuity consumers read host files outside the workspace.

## What This Changes

The PR adds local continuity snapshot parsing, rendering, manifest sorting/formatting, material-change detection, tests, and runtime-files exports in the memory host SDK.

## Best Possible Solution

Keep the local document/manifest slice, but make recent continuity reads use the same fs-safe regular-file and symlink rejection contract as existing memory file reads, with regression coverage for symlinked latest and snapshots paths.

## Reproduction Assessment

Yes for the review finding by source inspection: `readRecentContinuitySnapshot()` uses direct `fs.readFile`, `fs.readdir`, and `fs.stat` on recent continuity paths, while current memory reads and listing use fs-safe/lstat guards. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No as-is. The local helper layer is a reasonable boundary, but the reader should reuse existing fs-safe regular-file semantics before this runtime-files contract is exposed.
