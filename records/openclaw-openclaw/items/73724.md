## Summary

Keep open: the PR targets a valid local loopback status/probe false-negative that current main does not fully cover, but the latest head still has narrow review blockers before automerge.

## What This Changes

The PR adds a shared local loopback `status` RPC fallback for CLI status/probe paths, regression tests, a guarded loopback-only callGateway override, and a changelog entry.

## Best Possible Solution

Keep the bounded local `status` RPC fallback, but place the shared helper in a neutral support module, preserve daemon config context in the fallback call, and let exact-head CI/automerge gate the merge.

## Reproduction Assessment

Yes. The provided #48360/#73535 reports describe local token-auth loopback probes timing out or reporting unknown capability while direct `status` or `--require-rpc` succeeds, and current main still returns raw probe failures on daemon/gateway-status probe paths.

## Solution Assessment

No, not yet. The fallback design is the right narrow direction, but the current PR head appears to violate the core support boundary and misses the daemon config context in the fallback RPC.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Move the fallback helper out of the commands tree:** `src/cli/daemon-cli/probe.ts:1`
  - body: This new static import makes daemon CLI support code depend on a `src/commands/...` helper, and the exact-head `checks-node-core-support-boundary` shard is failing. Move the helper to a neutral shared/gateway support module that both daemon CLI and command code can import without crossing that boundary.
  - confidence: 0.86
- **[P2] Forward daemon config into the fallback RPC:** `src/cli/daemon-cli/probe.ts:85-96`
  - body: The `--require-rpc` branch passes `opts.config` into `callGateway`, but the new fallback branch does not. When daemon status is inspecting a non-default config, this fallback can load ambient config and reject or use the wrong auth context, recreating a false negative; pass `opts.config` here as well.
  - confidence: 0.82
