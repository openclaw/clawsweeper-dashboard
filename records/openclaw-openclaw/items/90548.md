## Summary

Keep open: current main and the latest release still run Unix listener diagnostics through per-port `lsof`, and the only focused mitigation, https://github.com/openclaw/openclaw/pull/90622, is still open and explicitly leaves restart-health polling out of scope.

## What This Changes

The issue asks OpenClaw to reduce macOS gateway port-health `lsof` fan-out because repeated per-port polling may overload LaunchServices and contribute to WindowServer watchdog teardown.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90622 if maintainers accept the partial status-path mitigation, then keep this issue as the canonical follow-up to remove or tightly bound `lsof` in restart-health polling with macOS-focused proof.

## Reproduction Assessment

Yes for the source-level issue: current main routes Unix `inspectPortUsage(port)` through per-port `lsof`, and daemon status/restart-health call that path. No for the full watchdog reboot outcome; that remains based on the reporter's macOS logs.

## Solution Assessment

No, the issue is not solved yet. The best fix direction is to remove repeated health-loop `lsof` launches or batch them at the shared inspection boundary while preserving bounded diagnostics, with the open mitigation PR treated as partial rather than closing proof.
