## Summary

This PR should stay open because the CSS-only patch does not complete the linked WebChat overlap fix: current grouped chat rendering can emit action buttons without adding the `has-copy` class that the changed padding rules still depend on.

## What This Changes

The PR changes grouped chat CSS so action padding applies on hover and the always-visible action-button rule only applies under `(hover: none) and (pointer: coarse)`.

## Best Possible Solution

Make the action-state contract explicit in the renderer or CSS selector, add focused regression coverage for hover and coarse-pointer cases, and include the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `hasActions` can render `.chat-bubble-actions`, while `bubbleClasses` never includes `has-copy`, so the PR's padding selectors can miss real action-bearing bubbles.

## Solution Assessment

No. The media-query narrowing is plausible, but the patch is not the best complete fix until the rendered DOM and CSS selector agree on which bubbles have actions.
