## Summary

Keep this PR open for changes rather than cleanup-close: current main has already fixed the renderer/class overlap path, but this PR still proposes unique hover/touch CSS behavior and its media query does not reliably exclude hybrid devices as described.

## What This Changes

The PR changes grouped chat CSS so action-button padding applies on hover except for coarse no-hover inputs, and scopes always-visible action buttons from `hover: none` to `hover: none` plus `pointer: coarse`.

## Best Possible Solution

Keep the merged renderer fix, revise this CSS patch so always-visible actions apply only when no hover-capable input is available, and prove the updated behavior in Control UI on desktop, hybrid, and touch paths.

## Reproduction Assessment

Yes at source level: current main still makes chat bubble actions always visible under `@media (hover: none)`, while the proposed selector still keys off primary pointer state. I did not run a live hybrid or touch-device browser reproduction in this read-only review.

## Solution Assessment

No. The PR improves one selector, but it does not fully match the stated hybrid-device goal or include real behavior proof against the now-updated main branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Gate touch-only actions on any-hover:** `ui/src/styles/chat/grouped.css:282`
  - body: The new `@media (hover: none) and (pointer: coarse)` rule still keys off the primary pointer, so hybrid devices whose primary input is coarse/no-hover can keep matching the always-visible action rule even when a mouse or trackpad can hover. Since the PR’s stated goal is to exclude hybrid hover-capable devices, gate this on no hover-capable input, such as adding an `any-hover: none` condition, or provide device proof showing the target hybrids report `pointer: fine`.
  - confidence: 0.78
