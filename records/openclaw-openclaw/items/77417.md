## Summary

Keep this PR open for revision rather than cleanup close: the renderer/class overlap fix is already on main, but this CSS media-query change remains unproven and still does not reliably exclude hover-capable hybrid devices.

## What This Changes

The PR changes grouped chat CSS so action padding applies only on hover for hover-capable bubbles and always-visible actions/padding are limited to `(hover: none) and (pointer: coarse)` devices.

## Best Possible Solution

Keep the current renderer fix, revise the CSS to gate always-visible actions on no hover-capable input, and add real Control UI/WebChat proof on desktop, hybrid, and touch before merge.

## Reproduction Assessment

Yes at source level, but not as a live device reproduction: current main still reveals actions under `@media (hover: none)`, and the PR replacement still uses primary-pointer media features. I did not verify the reported hybrid/touch behavior in a real browser/device setup.

## Solution Assessment

No. The PR improves padding timing, but `(hover: none) and (pointer: coarse)` is still not the narrowest reliable gate for excluding hover-capable hybrids; an `any-hover: none` style gate plus real device proof is safer.
