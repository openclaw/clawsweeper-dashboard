Keep this PR open: it is aimed at a real WebChat layout bug, but the CSS-only patch is incomplete because it still relies on `.chat-bubble.has-copy` while current grouped message rendering emits action buttons without that class, and the required changelog entry is missing.

## What This Changes

The PR changes grouped chat CSS so action-bubble padding applies on hover and the hoverless action-button rule only applies to coarse pointers.

## Best Possible Solution

No. The media-query direction is plausible, but a CSS-only change is not the narrow complete fix until action-bearing bubbles either emit the class being styled or the CSS targets the actual rendered DOM.
