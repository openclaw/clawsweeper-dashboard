## Summary

Keep this PR open, but it needs changes before merge: the CSS-only patch still depends on `.chat-bubble.has-copy`, while current grouped chat rendering emits action buttons without adding that class, and the user-facing fix is missing a changelog entry.

The PR changes grouped chat CSS so action-button padding applies only on hover for hover-capable devices and scopes always-visible action buttons to coarse-pointer hoverless devices.

## Best Possible Solution

Align the rendered DOM with the action-state CSS contract, preserve the intended hover/coarse-pointer behavior, add focused regression coverage, and record the user-facing fix in `CHANGELOG.md`.
