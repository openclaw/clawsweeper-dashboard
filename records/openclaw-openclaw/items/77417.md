## Summary

Keep open: the PR's CSS hover/coarse-pointer adjustment is incomplete because current main still renders chat action buttons without the `.has-copy` class that the changed selectors depend on, matching the linked issue's prior diagnosis.

## What This Changes

The PR changes Control UI grouped chat CSS so `.chat-bubble.has-copy` padding applies only on hover, scopes the hoverless action-button rule to coarse pointers, and restores unconditional padding inside that coarse-pointer media query.

## Best Possible Solution

Make the action-state contract explicit in the actual renderer or selector, then apply the hover/coarse-pointer padding behavior to matching bubbles with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows action buttons can render from `hasActions` while the bubble never receives `has-copy`, and the PR only changes CSS selectors that still depend on that missing class.

## Solution Assessment

No. The PR may improve the media-query policy, but it is not the best complete fix until the renderer emits `has-copy` for action-bearing bubbles or the CSS targets the actual rendered DOM.
