## Summary

Keep this PR open, but not merge-ready: it still targets a real WebChat hover/touch behavior after the renderer overlap fix landed on main, yet the CSS selector still relies on primary-pointer state and the external contributor has not supplied real behavior proof.

## What This Changes

The PR changes grouped chat CSS so action-button padding applies on hover on hover-capable bubbles and always-visible action buttons apply under `(hover: none) and (pointer: coarse)`.

## Best Possible Solution

Keep the merged renderer fix, revise this CSS change to gate always-visible actions on no hover-capable input, and provide real desktop, hybrid, and touch proof before merge.

## Reproduction Assessment

Yes at source level, but not as a live device reproduction: current main still uses `@media (hover: none)` for always-visible actions, and the PR replacement still keys off primary pointer state. I did not establish a real hybrid/touch browser reproduction in this read-only review.

## Solution Assessment

No. The PR improves one selector, but `(hover: none) and (pointer: coarse)` can still match hybrid devices whose primary pointer is coarse/no-hover; the safer fix is to require no hover-capable input, such as with `any-hover: none`, and prove it on target devices.
