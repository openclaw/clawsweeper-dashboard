## Summary

Keep open: current main still forces the plugin-sdk root alias off Jiti's native path on Windows, and the PR is a narrow follow-up to the Windows native-loader work from https://github.com/openclaw/openclaw/pull/74173. The patch looks directionally correct, but this external PR still lacks after-fix real behavior proof from a Windows run.

## What This Changes

The PR removes the `win32` override in `src/plugin-sdk/root-alias.cjs` so `tryNative` is passed through, and updates the root-alias contract test to expect native loading on Windows.

## Best Possible Solution

Land the narrow root-alias fix with the contract assertion and real Windows runtime proof that plugin-sdk dist loads use native require without path or alias regressions.

## Reproduction Assessment

Yes, at source level: current main's `getModuleLoader(true)` is forced to `false` on `win32`, and the contract harness has a Windows dist case asserting that behavior. I did not execute a Windows runtime in this read-only review.

## Solution Assessment

Yes for the code direction: removing the root-alias-only Windows override is the narrowest fix and matches the adjacent native-loader policy. It is not merge-ready until the external PR supplies real after-fix Windows behavior proof.
