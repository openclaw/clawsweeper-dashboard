## What This Changes

The PR removes the Windows `tryNative` override in `src/plugin-sdk/root-alias.cjs` and deletes the old Windows contract case that expected native loading to be disabled.

## Best Possible Solution

Land the narrow root-alias change after it preserves an explicit Windows contract assertion and includes real Windows runtime proof that plugin-sdk dist loads use the native path without alias regressions.

## Solution Assessment

Yes for the code direction: removing the root-alias-only Windows override is the narrowest fix and matches the adjacent native-loader policy. The final PR should keep explicit win32 contract coverage and add real Windows after-fix proof before merge.
