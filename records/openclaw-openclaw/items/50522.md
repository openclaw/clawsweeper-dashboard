Keep open. The Discord thread-starter bug is valid and this PR is directionally useful, but the branch is dirty against current main and still patches the old inline `processDiscordMessage` payload path while current main builds the inbound context in `buildDiscordMessageProcessContext`. The PR adds an effective Discord thread-session timestamp lookup, uses it to omit `ThreadStarterBody` on existing thread sessions, adds a Discord process regression test, and records the fix in the changelog.
