## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Fail when targets appear after planning:** `src/infra/backup-restore.ts:128-130`
  - body: `restoreSingleAsset` rechecks `conflictNow`, but when a target now exists and `--force` is false it only leaves `snapshotPath` undefined and continues into extraction/copy. That violates the command's overwrite-safety contract for targets created between `planRestore` and execution; throw before writing when `conflictNow && !force`.
  - confidence: 0.92
- **[P2] Keep rollback active after the snapshot rename:** `src/infra/backup-restore.ts:132-136`
  - body: The existing target is renamed before the staging directory is created and before the rollback `try` block starts. If `fs.mkdtemp` fails after the rename, the user's active state remains moved to `.pre-restore-*`; allocate staging first or include the rename in the rollback-protected block.
  - confidence: 0.83
