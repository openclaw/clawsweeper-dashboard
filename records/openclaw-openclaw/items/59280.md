## Summary

Keep open: current main and the latest release still only expose backup create/verify, but this PR is not merge-ready because latest-head source inspection shows overwrite/rollback safety gaps, no attached real behavior proof, and no restore docs or trust-model coverage. The native archive restore design also overlaps the open encrypted snapshot restore work at https://github.com/openclaw/openclaw/pull/44111.

## What This Changes

This PR adds a native `openclaw backup restore <archive>` command with dry-run, force, JSON output, archive path decoding, shared manifest reading, restore execution helpers, and restore tests.

## Best Possible Solution

Harden native archive restore with execution-time overwrite refusal, rollback-safe state replacement, public docs and trust-model guidance, then make an explicit maintainer choice on whether it lands separately from the encrypted snapshot restore flow.

## Reproduction Assessment

Yes for the PR findings: latest-head source inspection shows the no-force execution-time overwrite path and the forced directory rollback failure path. This is not a current-main bug reproduction because main has no native `backup restore` command.

## Solution Assessment

No: the proposed direction is plausible, but this patch is not yet the best safe implementation. The safer path is to fail closed on execution-time conflicts, restore through a rollback-safe staging/swap model, document the trust model, and decide how it relates to the encrypted snapshot restore PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject raced-in targets without --force:** `src/infra/backup-restore.ts:130-132`
  - body: `restoreSingleAsset` recomputes `conflictNow`, but when `--force` is false it only leaves `snapshotPath` undefined and then continues into extraction/copy/merge. If another process creates the restore target after planning, this overwrites live state without the explicit force flag, violating the command's safety contract.
  - confidence: 0.93
- **[P1] Make forced directory rollback remove partial targets:** `src/infra/backup-restore.ts:181-182`
  - body: On a forced directory restore, the code recreates `asset.restorePath` before merging. If merge fails after that, `fs.rename(snapshotPath, asset.restorePath)` cannot replace the existing partial directory and the error is swallowed, leaving partial restored state active instead of rolling back the original state.
  - confidence: 0.87
- **[P3] Document restore before exposing the command:** `src/cli/program/register.backup.ts:94-95`
  - body: This command points users at the backup docs, but that page still only explains create/verify and has no restore, `--force`, same-platform, or trusted-archive guidance. A destructive restore command needs public operator guidance before it ships.
  - confidence: 0.82
