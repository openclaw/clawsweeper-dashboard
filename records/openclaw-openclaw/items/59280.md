## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject raced-in targets without --force:** `src/infra/backup-restore.ts:130-132`
  - body: `conflictNow` is only used to make a snapshot when `--force` is set. If the plan saw no conflict but another process creates the restore target before execution, this code continues into `mkdir`/`copyFile`/merge and overwrites that path even though the user did not pass `--force`. Throw before extracting when `conflictNow && !force`.
  - confidence: 0.93
- **[P1] Keep current hardlink validation in the shared reader:** `src/commands/backup-verify.ts:169-178`
  - body: This refactor turns archive entries into plain strings and drops the current `entry.type`/`entry.linkpath` validation path. Current main rejects hardlink targets outside the archive root or missing from the archive; this branch would let `backup verify` and restore planning accept archives that the shipped verifier now rejects. Preserve `ArchiveEntry` metadata and call the hardlink-target check from the shared reader.
  - confidence: 0.9
- **[P3] Document restore before exposing the command:** `src/cli/program/register.backup.ts:94-116`
  - body: The CLI registers a new destructive restore command, but `docs/cli/backup.md` remains create/verify-only. Users need documented `--dry-run`, `--force` snapshot behavior, platform limitations, and recovery expectations before this command ships.
  - confidence: 0.82
