## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Fail when a target appears after planning:** `src/infra/backup-restore.ts:128-130`
  - body: `conflictNow` is recomputed here, but when `force` is false the result is ignored. If a restore target is created after `planRestore` and before execution, the command proceeds to extract and copy into that path without `--force` or a `.pre-restore-*` snapshot, violating the advertised refusal to overwrite unless forced.
  - confidence: 0.87
- **[P2] Keep rollback active after the snapshot rename:** `src/infra/backup-restore.ts:132-138`
  - body: The forced-restore rename happens before staging allocation and before the `try` block. If `fs.mkdtemp` fails after the rename, the rollback handler is skipped and the user's active state remains moved away from its expected path; allocate staging first or put the rename inside the guarded block.
  - confidence: 0.78
