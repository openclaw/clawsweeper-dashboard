## Summary

Keep this PR open, but it is not merge-ready: the proposed future-version guard is useful, yet it runs after a write-capable preflight path and the external PR still lacks real CLI behavior proof.

## What This Changes

The PR adds future-version checks to doctor config repair, gateway-token generation, downstream doctor health writes, and the generic doctor --fix hint.

## Best Possible Solution

Run the future-version decision before any write-capable doctor preflight or health repair, downgrade the prompter for every detected future skew unless the explicit override is set, and prove the real CLI path with redacted output.

## Reproduction Assessment

Yes from source inspection: the PR calls write-capable doctor preflight before resolving the future-config block, and current main shows both recovery paths write the config file. I did not execute the CLI because this review was required to keep the checkout read-only.

## Solution Assessment

No: the direction is right, but the guard needs to run before any repair-capable preflight write and the read-only prompter should apply to any detected future skew.
