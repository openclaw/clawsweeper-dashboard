Keep open: the compatibility direction is useful, but the patch gates doctor repairs too late, so repair-capable preflight and early doctor steps can still run before the newer-config refusal; the draft branch is also conflicting and lacks real behavior proof.

## What This Changes

The branch adds a future-version guard to doctor config repair and health write paths so newer-written configs suppress repair hints and should block doctor repair or gateway-token writes unless the older-binary override is set.

## Best Possible Solution

Rebase and move the future-config refusal to the first repair-capable boundary so --fix, --yes, and gateway-token generation stay read-only before preflight and early doctor repairs, then add focused regression coverage plus redacted real doctor output.
