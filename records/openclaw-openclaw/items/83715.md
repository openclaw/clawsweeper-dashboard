## Summary

Keep open: the branch addresses a real doctor/config downgrade safety gap, but the proposed guard still runs after write-capable preflight recovery and does not make every future-skew interactive doctor path read-only. The PR also still lacks contributor real-behavior proof for an older doctor inspecting and refusing repair against a newer-written config.

## What This Changes

The branch adds a future-version guard to doctor repair, yes-mode, and gateway-token paths, suppresses doctor repair hints during version skew, and adds focused doctor config and health tests.

## Best Possible Solution

Move future-version detection before every write-capable doctor preflight or health repair, make future-skew doctor read-only unless the explicit downgrade override is set, and add redacted real CLI proof before merge.
