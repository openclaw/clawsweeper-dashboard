## What This Changes

The PR adds a future-version skew guard to `openclaw doctor` repair/write paths so older binaries skip doctor repairs and suppress repair hints for newer-written configs.

## Best Possible Solution

Rebase the branch and move the future-version decision ahead of every mutating doctor preflight, config write, and health repair path while keeping read-only inspection usable and the explicit downgrade override intact.

## Reproduction Assessment

Yes. By source inspection, the PR invokes mutating doctor preflight with repair intent before resolving the future-version block, and preflight can write recovered config or run state migrations.

## Solution Assessment

No. The owner-boundary direction is right, but the guard must happen before mutating preflight work rather than only before later doctor repair sequencing and final writes.
