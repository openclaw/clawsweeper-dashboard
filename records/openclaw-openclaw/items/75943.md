## Summary

Keep open. The latest PR head fixes the previously flagged Discord default-account spawn gate, but it still resolves the new default thread-spawn context against the raw requester account id, which can ignore account-scoped context isolation; the protected maintainer label plus the security-sensitive policy regression block merge.

## What This Changes

The PR consolidates split thread-bound subagent and ACP spawn toggles into `threadBindings.spawnSessions`, adds thread-bound spawn context defaults, updates parent-fork policy, and refreshes related docs, schemas, metadata, changelog, and tests.

## Best Possible Solution

Resolve the effective channel account before every shared thread-spawn policy lookup, including default context and capability/schema guidance, then add omitted-account regression coverage for Discord `defaultAccount` plus `defaultSpawnContext: "isolated"`.

## Reproduction Assessment

Yes. A focused source-level reproduction is `spawnSubagentDirect({ thread: true })` from a Discord context with no `agentAccountId`, `channels.discord.defaultAccount: "work"`, and `accounts.work.threadBindings.defaultSpawnContext: "isolated"`; the PR resolves context against literal `default` and chooses `fork`.

## Solution Assessment

No. The consolidation direction is reasonable, but this implementation should make effective account resolution part of the shared policy path or resolve it before calling the shared resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve spawn context against the effective account:** `src/agents/subagent-spawn.ts:568`
  - body: The default context lookup still passes the raw requester account id into `resolveThreadBindingSpawnPolicy`. When Discord uses `channels.discord.defaultAccount` and the requester omits an account id, the resolver checks literal `default`, so `accounts.work.threadBindings.defaultSpawnContext: "isolated"` is ignored and the child can fork the parent transcript. Resolve the effective channel account before choosing context, matching the binding hook.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR can bypass account-scoped thread-spawn context isolation when a channel default account is used and the requester account id is omitted.

Concerns:

- **[medium] Account-scoped context isolation can be bypassed:** `src/agents/subagent-spawn.ts:568`
  - body: The new `defaultSpawnContext` policy is evaluated before Discord default-account resolution, so an account that sets `defaultSpawnContext: "isolated"` can still receive a forked child transcript when `agentAccountId` is omitted.
  - confidence: 0.82
