## Summary

Keep open. This PR has the protected `maintainer` label and the diff appears to regress Discord default-account policy handling for thread-bound subagent spawns.

## What This Changes

The PR consolidates split thread-bound subagent and ACP spawn flags into `threadBindings.spawnSessions`, changes default spawn/context behavior, and updates doctor migration, schemas, docs, generated metadata, and tests.

## Best Possible Solution

Keep the consolidation, but make thread-binding spawn policy preserve channel `defaultAccount` resolution before applying account overrides and add a Discord regression test for omitted requester account ids.

## Reproduction Assessment

Yes. A focused reproduction is `handleDiscordSubagentSpawning` with `requester.channel: "discord"`, no requester account id, `channels.discord.defaultAccount: "work"`, and `accounts.work.threadBindings.spawnSessions: false`; current main checks `work`, while the PR would check `default`.

## Solution Assessment

No. The overall simplification is reasonable, but the implementation should preserve default-account-aware policy resolution before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve default Discord account policy:** `extensions/discord/src/subagent-hooks.ts:97`
  - body: The new shared policy is called with `event.requester?.accountId`, but when that is omitted it normalizes to `default` instead of `channels.discord.defaultAccount`. Resolve the effective account before checking policy, or teach the shared policy about channel `defaultAccount`.
