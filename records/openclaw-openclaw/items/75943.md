## Summary

Keep open. The PR has the protected `maintainer` label and the latest head still has a P2 security-sensitive account-policy gap: the thread-bound subagent context-mode lookup uses the raw requester account id before Discord `defaultAccount` resolution.

## What This Changes

The PR replaces split thread-bound subagent/ACP spawn settings with `threadBindings.spawnSessions`, adds default spawn context and parent-fork fallback behavior, migrates legacy config, updates docs/schemas/tests, and drops the root `@homebridge/ciao` dependency.

## Best Possible Solution

Resolve the effective channel account before every thread-spawn policy lookup, including `defaultSpawnContext` and tool/prompt capability decisions, then add omitted-account Discord coverage for `defaultAccount` plus `defaultSpawnContext: "isolated"`.

## Reproduction Assessment

Yes. A focused source-level reproduction is `sessions_spawn({ thread: true })` from Discord with no `agentAccountId`, `channels.discord.defaultAccount: "work"`, and `accounts.work.threadBindings.defaultSpawnContext: "isolated"`; the PR-head lookup checks the omitted/raw account and chooses the default `fork` context.

## Solution Assessment

No. The consolidation direction is reasonable, but the implementation should make effective-account resolution part of the shared thread-spawn policy path or resolve it before calling the shared resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve spawn context against the effective account:** `src/agents/subagent-spawn.ts:568`
  - body: `resolveSubagentContextMode` reads thread-spawn policy with the raw requester account id before Discord resolves `channels.discord.defaultAccount`. When the requester omits `agentAccountId` and the configured default account sets `defaultSpawnContext: "isolated"`, this lookup misses the account policy and leaves the child in `fork` mode, bypassing the account isolation setting. Resolve the effective account before this policy lookup or move default-account resolution into the shared resolver.
  - confidence: 0.87
