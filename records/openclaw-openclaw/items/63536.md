## Summary

Keep open: the PR targets a plausible gateway session bug, but the patch as written regresses existing freshest-duplicate session selection and lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway session loading to return an exact requested or canonical store entry before falling back to the freshest updatedAt match.

## Best Possible Solution

Scope exact-key preference to explicit non-canonical requested keys, preserve freshest duplicate selection for canonical logical keys, and add regression coverage for archived session history.

## Reproduction Assessment

Yes. The PR regression is source-reproducible from the diff plus existing tests that seed stale canonical and fresher case-variant rows; the original archived-key behavior still needs a focused regression test.

## Solution Assessment

No. The exact-first idea is close, but the current diff is too broad because canonical exact hits must not outrank fresher logical duplicates.
