## Summary

Keep open. The session-history bug being targeted is plausible, but this PR is not mergeable as written because the exact-entry shortcut can return stale canonical rows and it still lacks after-fix real behavior proof.

## What This Changes

The branch changes `loadSessionEntry` to return an exact requested or canonical session-store row before falling back to the freshest `updatedAt` match.

## Best Possible Solution

Scope exact-key preference to explicit non-canonical archived keys, preserve freshest duplicate selection for canonical logical keys, update any history transport that needs the same semantics, and add regression coverage plus real proof.

## Reproduction Assessment

Yes for the PR regression: the diff plus existing current-main tests show stale canonical rows would be returned ahead of fresher logical duplicates. The original archived-key bug remains plausible but is not yet proven with a focused test or real setup artifact.

## Solution Assessment

No. The exact-first direction is close for explicit archived keys, but the current implementation is too broad because canonical exact hits must not outrank fresher duplicate rows.

## Review Findings
