## Summary

Keep open. Current main still lacks diagnostic progress tracking and repeated stuck-warning backoff, and this PR is the right general direction, but the latest head still drops mark-only progress callbacks when source reply delivery is suppressed, so default group/message-tool-only turns can still emit false stuck-session warnings during long tool-start/item/command/compaction activity before a tool result or block arrives.

## What This Changes

The PR adds diagnostic session progress/backoff state, wires reply and ACP runtime progress callbacks, updates focused tests, generated config help/docs, and the changelog.

## Best Possible Solution

Separate diagnostic progress marking from user-visible source delivery suppression, keep typing keepalive excluded from progress, preserve the ACP diagnostics.enabled gate, then land after the focused dispatch/logging tests and CI are green with the existing docs and changelog entry intact.

## Reproduction Assessment

Yes. Current main can be reproduced with a processing diagnostic session, fake timers past stuckSessionWarnMs, and activity represented only by reply/ACP progress; main still warns from age alone. The PR finding can be reproduced by dispatching a message_tool_only turn whose resolver emits onToolStart/onItemEvent/onCommandOutput before any tool result or block and observing that markDiagnosticSessionProgress is not called.

## Solution Assessment

No, not as currently written. The diagnostic state/backoff layer and ACP gate are the right narrow solution, but dispatch progress marking must survive source delivery suppression so private/message-tool-only turns do not keep the old false-stuck behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve progress marks when source delivery is suppressed:** `src/auto-reply/reply/dispatch-from-config.ts:1155-1156`
  - body: `wrapProgressCallback` returns `undefined` for every wrapped progress callback whenever the turn is `message_tool_only`. In that common group/channel mode, agent-runner only reports tool-start, item, command-output, and compaction activity through these callbacks, so a long active command can still age past `stuckSessionWarnMs` until a tool result or block arrives and emit the false `session.stuck` warning this PR is meant to prevent. Please split diagnostic marking from user-visible callback forwarding so suppressed delivery still gets mark-only progress callbacks.
  - confidence: 0.88
