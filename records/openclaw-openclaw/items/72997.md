## Summary

Keep open. Current main has internal per-channel stop/start plumbing for config, secrets, and health-monitor reload paths, but it does not expose the requested manual `openclaw channels reload` command or `channels.reload` gateway action; the existing `channels.start` RPC only starts a stopped runtime and does not force a reconnect for an already-running channel.

## What This Changes

Request a targeted operator command or gateway action to disconnect and reconnect only a named channel provider without restarting the Gateway or interrupting other channels or sessions.

## Best Possible Solution

Add a narrow admin-scoped `channels.reload` Gateway method and a matching command under the existing `openclaw channels` namespace, reusing the current plugin-agnostic `stopChannel` and `startChannel` lifecycle primitives while leaving config watcher debounce work in #74122 separate.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the CLI and Gateway method list have no reload surface, and `channels.start` cannot reconnect an already-running provider because the channel manager returns when the account task already exists.

## Solution Assessment

Yes. A small core control-plane addition is the maintainable fix because channel lifecycle is already Gateway-owned and plugin-agnostic; implementing this as provider-specific plugin behavior would duplicate existing lifecycle seams.

## Repair Work Prompt

Implement a narrow manual channel reload surface for https://github.com/openclaw/openclaw/issues/72997. Observable gap: current main has config/secrets/health-monitor channel stop/start paths, but no public `channels.reload` Gateway method and no `openclaw channels reload` command; `channels.start` only starts stopped runtimes and returns early for already-running channel tasks. Add an admin-scoped Gateway RPC such as `channels.reload` with params `{ channel: string, accountId?: string }` that validates the channel, stops the selected channel/account, resets restart attempts if appropriate, starts it again, and returns `{ channel, accountId, reloaded/started }` or a clear error. Add CLI wiring under the existing plural namespace, preferably `openclaw channels reload --channel <name> [--account <id>] [--json]`, plus any positional shorthand only if it fits existing Commander patterns. Update protocol schema/types, method list, method scopes, tests, docs/cli/channels.md, and CHANGELOG.md with reporter credit `Thanks @axonrelaybot`. Do not solve #74122 watcher debounce/scoped file reload in this PR and do not change existing config hot reload semantics except to share a helper if that keeps the reload implementation small.
