## Summary

Keep open. Current main still has lower-level channel stop/start lifecycle and automatic config hot-reload paths, but it does not expose the requested one-step manual channel reload CLI command or Gateway RPC.

## What This Changes

Add a targeted channel reload CLI/RPC that disconnects and reconnects only the named channel without restarting the Gateway or interrupting other sessions.

## Best Possible Solution

Add an additive, admin-scoped `channels.reload` RPC plus `openclaw channels reload --channel <name> [--account <id>]`, implemented through the existing generic stop/start lifecycle with tests, docs, and changelog.

## Reproduction Assessment

Yes. A source and docs check against current main shows no CLI command, Gateway method, admin scope, or protocol schema for reload while stop/start and automatic config reload seams exist.

## Solution Assessment

Yes. The narrow maintainable path is a generic core RPC/CLI wrapper over the existing channel lifecycle; a plugin-only wrapper or full Gateway restart would not provide the requested operator surface.

Reuse the existing generic channel lifecycle (`stopChannelAccount` then `startChannelAccount` or an equivalent shared helper) and preserve session/Gateway process state; do not add provider-specific core policy, do not solve watcher debounce/scoped-file reload work from #74122, and do not require a full Gateway restart.
