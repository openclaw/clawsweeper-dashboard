## Summary

Keep open. Current main has automatic per-channel hot reloads and admin-scoped `channels.start`/`channels.stop` RPCs, but it does not provide the requested targeted `openclaw channels reload` command or Gateway/tool `channel.reload` action.

## What This Changes

Request a targeted channel reload command or Gateway action that restarts one named channel without restarting the gateway or interrupting other channels.

## Best Possible Solution

Add an explicit admin-scoped channel reload surface, if approved, that reuses existing channel lifecycle primitives and returns status without weakening hot-reload safety or account-scoped semantics.
