## Summary

Keep open. Current main does not implement the requested multi-workspace agent config, and the PR is still an open feature with merge-blocking correctness issues in non-reply execution paths and root-path handling.

## What This Changes

The PR adds an `agents.list[].multipleWorkspaces` config field, resolves those agents to a state-dir composite workspace, populates repo symlinks during reply setup, updates workspace lookup/listing, and extends agent config types/schema.

## Best Possible Solution

Keep the feature PR as an implementation candidate only if maintainers want this core config surface; before merge, centralize or propagate multi-workspace setup through every agent execution path, reject or safely name invalid roots, and add focused tests plus docs/changelog coverage.

## Reproduction Assessment

Yes. Static source inspection gives a high-confidence reproduction path: configure `multipleWorkspaces`, then run CLI or isolated cron agent execution on a fresh setup; those paths resolve to the composite directory but do not pass the workspace list needed to populate symlinks.

## Solution Assessment

No. The symlink-composite approach may be acceptable, but this patch is not mergeable until setup is consistent across execution paths, root workspace entries are handled deliberately, and the user-facing config change is documented and changelogged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Populate composite workspaces in every setup path:** `src/agents/agent-scope.ts:295-296`
  - body: The new resolver sends multi-workspace agents to `workspace-composite-<id>`, but only the reply path passes `multipleWorkspaces` into `ensureAgentWorkspace`. CLI and isolated cron runs still call setup without the list, so fresh executions create or use an empty composite workspace and cannot see the configured repos.
  - confidence: 0.9
- **[P2] Reject root workspace paths before deriving link names:** `src/agents/workspace.ts:732`
  - body: For a configured filesystem root, `path.basename()` is empty and this branch assigns that empty string as the link name. `path.join(compositeDir, "")` points at the composite root, so setup cannot create a usable repo link and may target the composite directory itself.
  - confidence: 0.84

  - body: This is a user-facing feature because it adds a public `agents.list[].multipleWorkspaces` config key, but the PR does not touch `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.92
