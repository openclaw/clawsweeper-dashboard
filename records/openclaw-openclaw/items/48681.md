## Summary

Keep open: current main still has only the single-workspace contract, while this PR remains a plausible feature proposal but has blocking PR-head correctness issues, no inspectable real behavior proof, and needs maintainer product approval for a new core config/symlink workspace model.

## What This Changes

The PR adds an optional `agents.list[].multipleWorkspaces` config field, resolves those agents to a symlink-backed composite workspace directory, and extends workspace listing and reverse lookup.

## Best Possible Solution

If maintainers want this feature, land a first-class multi-workspace contract against current main that consistently populates composite links across reply, CLI, cron, reverse lookup, docs, and validation edge cases.

## Reproduction Assessment

Yes. Static PR-head inspection gives a high-confidence path: configure `multipleWorkspaces`, run a non-reply path such as agent command or isolated cron, and it resolves to the composite directory without passing the workspace list needed to populate symlinks.

## Solution Assessment

No. The symlink-composite approach may be viable, but this branch does not yet cover all workspace setup paths or root-path validation and needs current-main docs/tests plus maintainer approval for the new config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass multi-workspace setup into every workspace flow:** `src/agents/agent-scope.ts:291-295`
  - body: `resolveAgentWorkspaceDir` sends every multi-workspace agent to `workspace-composite-<id>`, but only the reply path forwards `multipleWorkspaces` to `ensureAgentWorkspace`. Agent command and isolated cron setup still call workspace setup without that list, so those flows can run in an empty composite directory and lose access to the configured repos.
  - confidence: 0.92
- **[P2] Reject root workspace entries before planning links:** `src/agents/workspace.ts:733`
  - body: For a filesystem root path, `path.basename(ws)` is empty and this branch uses that empty value as the link name. `path.join(compositeDir, "")` targets the composite root itself, so the root workspace is never exposed as a usable repo entry; reject roots with a clear config error or define a stable explicit name.
  - confidence: 0.86
