## Summary

Keep open. Current main does not implement multi-workspace agents, and this PR still needs maintainer direction because it introduces a symlink-based filesystem contract that conflicts with other open multi-root proposals and has source-visible runtime gaps.

## What This Changes

The PR adds `agents.list[].multipleWorkspaces`, resolves those agents to a generated composite workspace directory, and symlinks configured repositories into that directory.

## Best Possible Solution

Choose one maintainer-approved multi-root filesystem model, then wire it through config/schema, every runtime setup caller, path guards, sandbox/media/tool handling, docs, tests, and real multi-repo proof before merging.

## Solution Assessment

No, not in its current merge shape. The symlink-composite approach may be useful, but the safer path is one maintainer-approved multi-root contract with complete runtime propagation, security-boundary coverage, docs, tests, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass multi-workspaces through every setup flow:** `src/auto-reply/reply/get-reply.ts:174`
  - body: `resolveAgentWorkspaceDir()` makes every agent with `multipleWorkspaces` run from the composite directory, but this patch only passes the workspace list in the reply flow. Agent command and isolated cron still call `ensureAgentWorkspace` without `multipleWorkspaces`, so those executions can start in an empty composite workspace and lose access to the configured repos.
  - confidence: 0.9
- **[P2] Reject root paths before naming symlinks:** `src/agents/workspace.ts:733`
  - body: When a configured workspace is a filesystem root such as `/` or a Windows drive root, `path.basename(ws)` is empty and this branch uses the empty string as the link name. That makes `path.join(compositeDir, linkName)` address the composite directory itself instead of creating a child symlink, so the root workspace is never exposed correctly.
  - confidence: 0.88
