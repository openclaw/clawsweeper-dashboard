## Summary

Keep open: current main does not implement multi-workspace agents, and the PR is a new core config/filesystem feature with blocking source-review findings plus missing real behavior proof.

## What This Changes

The PR adds an optional `agents.list[].multipleWorkspaces` config field and creates a symlink-backed composite workspace for those agents.

## Best Possible Solution

If maintainers want this feature, land a first-class multi-workspace contract that consistently handles reply, CLI, cron, reverse lookup, docs, validation edge cases, and sandbox expectations on current main.

## Reproduction Assessment

Yes. Static PR-head inspection shows that `multipleWorkspaces` redirects workspace resolution to a composite directory while command and isolated cron setup do not pass the workspace list needed to populate that directory.

## Solution Assessment

No. The symlink-composite approach may be viable, but this branch does not yet cover all workspace setup paths or root-path validation and still needs maintainer approval for the new config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass multi-workspace setup to every workspace caller:** `src/agents/agent-scope.ts:291-295`
  - body: `resolveAgentWorkspaceDir` now sends multi-workspace agents to `workspace-composite-<id>`, but only the reply path passes `multipleWorkspaces` into `ensureAgentWorkspace`. Agent command and isolated cron setup still call workspace setup without that list, so those flows can run in an empty composite directory and lose access to the configured repos.
  - confidence: 0.92
- **[P2] Reject root paths before deriving symlink names:** `src/agents/workspace.ts:733`
  - body: For a filesystem root entry, `path.basename(ws)` is empty and this branch uses that empty value as the link name. `path.join(compositeDir, "")` targets the composite root itself, so the root workspace is never exposed as a usable repo entry; reject roots with a clear config error or define a stable explicit name.
  - confidence: 0.86
