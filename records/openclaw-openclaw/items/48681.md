## Summary

Keep open: current main still has only the single-workspace agent contract, and this PR is a real feature that is not implemented on main. The branch still has blocking runtime gaps, lacks inspectable real behavior proof, and needs maintainer product review before merge.

## What This Changes

The PR adds `agents.list[].multipleWorkspaces`, routes those agents through a symlink-backed composite workspace directory, and extends workspace reverse lookup to include the configured physical workspaces.

## Best Possible Solution

If maintainers want this feature, land a first-class multi-workspace config contract that consistently populates composite workspaces across reply, CLI, cron, reverse lookup, docs, and edge-case validation.

## Reproduction Assessment

Yes. Static inspection of PR head shows `resolveAgentWorkspaceDir` redirects multi-workspace agents to a composite directory while CLI and isolated cron calls do not pass `multipleWorkspaces`, so those paths use an unpopulated composite workspace.

## Solution Assessment

No. The symlink-composite approach may be viable, but this branch is not mergeable until all workspace setup paths are populated, root paths are rejected or deliberately named, docs/config contract work is aligned, and real behavior proof is added.
