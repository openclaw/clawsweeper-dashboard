## Summary

Keep open. The latest head addresses the earlier stale-cache finding, but the new hook can mutate prompt-loaded workspace templates while bypassing `allowPromptInjection=false`, so merge should wait for policy coverage and a regression test.

## What This Changes

The PR adds a typed `substitute_template` plugin hook and threads it through workspace template loading/caching, with docs, changelog, API hash, and workspace tests.

## Best Possible Solution

Keep the hook and per-runner cache, but classify `substitute_template` under the prompt-injection policy or add an equivalent explicit gate, with a loader-policy regression and docs update.

## Reproduction Assessment

Yes. Configure a plugin with `hooks.allowPromptInjection=false`, register `substitute_template`, and load plugins: current policy checks only `PROMPT_INJECTION_HOOK_NAMES`, so the new hook would still register and seed prompt-loaded workspace files.

## Solution Assessment

No. The hook API and cache boundary are narrow, but the implementation is not the best merge path until it honors the existing prompt-injection safety contract.
