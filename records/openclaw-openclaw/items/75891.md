## Summary

Keep open; the PR is useful but latest head still has a filename-only workspace template cache that can bypass the new hook after a prior unhooked template load.

## What This Changes

This PR adds a typed `substitute_template` plugin hook, threads it into workspace template loading and bootstrap completion checks, updates hook docs/changelog/API baseline, and adds workspace tests.

## Best Possible Solution

Keep the typed load-time hook, but cache raw template reads separately from hook-substituted output or key substituted entries by effective hook runner, then add a same-process no-hook-then-hook regression.

## Reproduction Assessment

Yes. In one process, load or ensure workspace templates without a `substitute_template` hook to populate `workspaceTemplateCache`, then ensure a fresh workspace with a hook runner; the PR returns the cached filename entry before dispatching the hook.

## Solution Assessment

No. The hook API is a narrow maintainable seam, but this implementation is not reliable until cache state accounts for hook participation or only caches raw template reads.
