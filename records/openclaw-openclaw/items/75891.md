## Summary

Keep open. The PR adds a relevant plugin hook that current main lacks, but the latest patch still has a blocking cache correctness issue: template content is cached only by filename, so a previous non-hook load can prevent a later hook-enabled workspace seed from invoking `substitute_template`.

## What This Changes

The PR adds a typed `substitute_template` plugin hook, wires workspace template loading through it, updates hook docs and SDK baseline metadata, adds workspace tests, and records a changelog entry.

## Best Possible Solution

Keep the load-time typed hook, but make template caching respect hook participation or runner-specific substitutions and add a same-process default-load-then-hook-load regression test.

## Reproduction Assessment

Yes. On the PR head, warm `workspaceTemplateCache` by loading a template without a `substitute_template` hook, then ensure a fresh workspace with a hook runner in the same process; the filename-only cache can return before the hook runs.

## Solution Assessment

No, not as patched. The typed hook is the right narrow seam, but the cache must not let default or differently substituted template content suppress active hooks.
