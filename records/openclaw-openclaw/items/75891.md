## Summary

Keep open. The hook direction is useful and current main does not provide it, but the patch is not mergeable because template caching is still keyed only by filename, so prior loads can bypass later substitution in the same process.

## What This Changes

The PR adds a typed `substitute_template` plugin hook, invokes it while loading workspace templates, documents the hook, refreshes SDK baseline hashes, and adds workspace seeding tests.

## Best Possible Solution

Keep the load-time typed hook, but make template caching respect hook participation or runner identity and add a regression test for default-load followed by hook-enabled load in one process.

## Reproduction Assessment

Yes. On the PR head, load a workspace template once without a `substitute_template` hook, then ensure a fresh workspace with a hook runner in the same process; the filename-only cache returns before the hook can run.

## Solution Assessment

No, not as patched. A typed load-time hook is the narrow maintainable seam, but the cache must not let default or differently substituted template content suppress active hooks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Key template cache by hook state:** `src/agents/workspace.ts:110-112`
