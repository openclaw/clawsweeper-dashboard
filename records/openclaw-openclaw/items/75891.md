## Summary

Keep open. Current main does not implement this hook, and the PR is directionally useful, but latest head still has a cache-boundary bug: `loadTemplate` returns the filename-only cached template before checking the supplied `substitute_template` hook runner.

## What This Changes

The PR adds a typed `substitute_template` plugin hook for workspace template content, threads it into workspace seeding/bootstrap checks, and updates docs, changelog, API hash, and workspace tests.

## Best Possible Solution

Keep the typed hook, but cache raw template reads separately from hook-transformed output or key transformed entries by effective hook runner, then add a no-hook-then-hook same-process regression.

## Reproduction Assessment

Yes. In one process, load or ensure workspace templates without a `substitute_template` hook to populate the cache, then ensure a fresh workspace with a hook runner; the PR returns the cached template before dispatching the hook.

## Solution Assessment

No. The hook API is a narrow maintainable seam, but this implementation is not reliable until cache state accounts for hook participation or only caches raw template reads.
