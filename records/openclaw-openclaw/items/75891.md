## Summary

Keep open. The typed hook is useful and current main does not already implement it, but the PR still has a filename-only module cache that can skip the hook or leak substituted template content across later workspace ensures in the same process.

## What This Changes

The PR adds a typed `substitute_template` plugin hook, invokes it during workspace template loading/seeding, updates docs/changelog/SDK baseline hashes, and adds focused workspace tests.

## Best Possible Solution

Keep the typed load-time hook, but separate raw template caching from hook-substituted output or key substituted entries by effective hook participation/runner, then cover both cache orders in `src/agents/workspace.test.ts`.

## Reproduction Assessment

Yes. On the PR head, load a workspace template once without a `substitute_template` hook and then ensure a fresh workspace with a hook runner in the same process; the filename cache returns before the hook can run.

## Solution Assessment

No. A typed `substitute_template` hook is the narrow maintainable seam, but the current cache design makes the implementation unreliable when hook state differs across calls.
