## Summary

Keep open. The PR adds the right public hook shape, but the latest head still uses a filename-only workspace template cache before considering hook participation, so plugin substitutions can be skipped or cross-contaminate between hook runners in a long-lived process.

## What This Changes

The PR adds a typed `substitute_template` plugin hook, threads it through workspace template seeding, updates hook docs/changelog/SDK baseline hashes, and adds focused workspace tests.

## Best Possible Solution

Keep the load-time typed hook, but cache raw template reads separately from hook-substituted output or key substituted entries by effective hook runner, then add a regression for default-load followed by hook-enabled load in one process.

## Reproduction Assessment

Yes. On the PR head, load a workspace template once without `substitute_template` hooks, then ensure a fresh workspace with a hook runner in the same process; `loadTemplate` returns the filename cache before the hook can run.

## Solution Assessment

No. A typed `substitute_template` hook is the narrow maintainable seam, but the current filename-only cache makes the implementation unreliable when hook state or active plugins change.
