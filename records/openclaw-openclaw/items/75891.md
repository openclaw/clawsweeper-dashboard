## Summary

Keep open. Current main still lacks the requested hook, and the PR head adds the right public seam but has blocking cache and typecheck regressions before it can merge.

## What This Changes

Adds a typed `substitute_template` plugin hook, routes workspace template loading through it, documents it, refreshes the SDK baseline hash, and adds workspace seeding tests.

## Best Possible Solution

Keep the typed load-time hook, but isolate template caching by substitution context or runner identity, normalize nullable global hook runners, and align tests/docs with the final cache contract.

## Reproduction Assessment

Yes. On the PR head, load a workspace template without a hook, then ensure a fresh workspace with a `substitute_template` runner in the same process; the filename-only cache returns before the hook can run. Public CI also reproduces the type error at `src/agents/workspace.ts:441`.

## Solution Assessment

No, not as patched. A typed load-time hook is the narrow maintainable seam, but the implementation must fix cache isolation and the nullable global runner type before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Key template cache by substitution context:** `src/agents/workspace.ts:110-112`
  - body: `workspaceTemplateCache` is still keyed only by template name and is checked before the hook runner is consulted. If a template was first loaded without `substitute_template`, a later hook-enabled workspace ensure in the same process reuses the default content and never calls `runSubstituteTemplate`, so enabled plugins silently fail to customize newly seeded files.
  - confidence: 0.92
- **[P2] Normalize the global hook runner before passing it:** `src/agents/workspace.ts:441`
  - body: `getGlobalHookRunner()` returns `HookRunner | null`, but `reconcileWorkspaceBootstrapCompletionState` accepts `TemplateSubstitutionHookRunner | undefined`; the exact PR head is failing prod/test typechecks on this line. Convert `null` to `undefined` here or widen the parameter type consistently.
  - confidence: 0.96
