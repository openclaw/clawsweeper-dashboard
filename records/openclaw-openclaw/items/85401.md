## Summary

Keep open: current main and the latest release still turn blocked final-reply media into generic `⚠️ Media failed.` text rather than the requested sanitized, actionable local-media-policy error for WhatsApp document/PDF attachments.

## What This Changes

The issue reports that WhatsApp final assistant `MEDIA:` directives for PDF/document attachments can fail delivery and should not expose raw local paths or collapse blocked-path failures to an unhelpful generic message.

## Best Possible Solution

Keep the issue open for maintainer/security approval of the sanitized policy-error wording, then implement it in the shared reply-media normalizer or WhatsApp media fallback with regression tests for blocked absolute PDF paths.

## Solution Assessment

No for closing: current main strips/directs `MEDIA:` better than the original raw-text leak, but it does not provide the requested sanitized actionable allowed-roots rejection for blocked absolute host paths.
