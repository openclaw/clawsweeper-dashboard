- `impact:security`: The issue is about SSRF enforcement and redirect-hop inspection for browser navigation.

## Summary

Keep open. Current main and the latest release still route local-managed browser profiles away from persistent Playwright, so strict SSRF HTTP(S) opens fail closed before redirect-hop inspection; this remains valid security-sensitive product work.

## What This Changes

The issue requests Playwright navigation interception for local-managed browser profiles so strict browser SSRF can inspect redirect hops like remote CDP.

## Best Possible Solution

Implement a targeted browser-plugin path that uses the existing Playwright navigation guard for local-managed profiles when strict SSRF requires redirect-hop inspection, while preserving existing-session behavior and raw CDP paths for safe non-strict or bootstrap cases.

## Reproduction Assessment

Yes. Source inspection shows a local-managed profile with browser.ssrfPolicy.dangerouslyAllowPrivateNetwork false skips Playwright and throws the strict redirect-inspection error for HTTP(S) opens before redirect hops can be inspected.

## Solution Assessment

Unclear pending maintainer/security review. A strict-only local-managed Playwright path looks narrower than always enabling Playwright or adding a new config knob, but the rollout affects browser lifecycle and compatibility.
