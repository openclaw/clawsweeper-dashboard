## Summary

Keep open. Current main still lacks the requested local-managed Playwright navigation interception path: local-managed profiles do not select persistent Playwright, so strict HTTP(S) navigation still fails closed before raw CDP tab creation can inspect redirect hops.

## What This Changes

The issue requests enabling Playwright-backed navigation interception for local-managed browser profiles so strict browser SSRF policy can inspect redirect hops.

## Best Possible Solution

Implement a targeted browser-plugin change that routes local-managed strict HTTP(S) open-tab navigation through the existing Playwright `createPageViaPlaywright` and `gotoPageWithNavigationGuard` path, while preserving Chrome MCP existing-session behavior and raw CDP behavior for safe cases such as `about:blank` or non-strict policy. Add regression coverage for local-managed strict IP-literal navigation, blocked private redirects, and current remote-CDP behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with a local-managed profile and `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false`, HTTP(S) open-tab navigation skips the Playwright branch and then throws the strict redirect-inspection error before raw CDP tab creation.

## Solution Assessment

Unclear pending maintainer/security design review. Reusing the existing Playwright guard for local-managed strict navigation is the narrowest apparent implementation, but the exact rollout should be chosen deliberately because this changes a security enforcement path.
