## Summary

Keep open. Current main and the latest release still route local-managed browser profiles away from persistent Playwright, so strict HTTP(S) opens fail closed before redirect-hop inspection; this is valid security-sensitive browser-plugin feature work that needs maintainer/security rollout judgment.

## What This Changes

Request enabling Playwright-backed navigation interception for local-managed browser profiles so strict SSRF policy can inspect redirect hops like remote CDP mode.

## Best Possible Solution

Implement a targeted browser-plugin path that uses the existing Playwright navigation guard for local-managed profiles when strict SSRF requires redirect-hop inspection, while preserving existing-session behavior and raw CDP paths for safe cases such as about:blank or non-strict navigation.

## Reproduction Assessment

Yes at source level, but not by a live browser run. Local-managed profiles return usesPersistentPlaywright false, openTab only uses Playwright behind that capability, and strict HTTP(S) URLs throw the redirect-inspection error when dangerouslyAllowPrivateNetwork is false.

## Solution Assessment

Unclear pending maintainer/security review. Reusing createPageViaPlaywright and gotoPageWithNavigationGuard for strict local-managed opens looks like the narrowest technical direction, but always-on, strict-only, or configurable rollout choices affect local CDP lifecycle and performance.
