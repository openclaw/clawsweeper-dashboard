## Summary

Keep open. Current main still keeps local-managed browser profiles on the raw CDP tab-open path, so strict SSRF HTTP(S) opens that require redirect-hop inspection fail before the existing Playwright guard can run.

## What This Changes

Request Playwright-backed redirect-hop inspection for local-managed browser open/navigation so strict SSRF policy can inspect redirect hops like remote CDP.

## Best Possible Solution

Use the existing Playwright navigation guard for local-managed strict SSRF tab opens while preserving existing-session behavior and safe raw-CDP fallbacks, with focused regression coverage for IP-literal opens and private redirect blocking.

## Reproduction Assessment

Yes, from source inspection. A local-managed profile with `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false` and an HTTP(S) `/tabs/open` request skips Playwright because `usesPersistentPlaywright` is false, then throws the strict redirect-inspection error before raw CDP can create the tab.

## Solution Assessment

Unclear pending maintainer/security review. Reusing the existing Playwright guard for strict local-managed opens looks like the narrowest maintainable direction, but maintainers should choose the exact rollout and coordinate it with #64929.
