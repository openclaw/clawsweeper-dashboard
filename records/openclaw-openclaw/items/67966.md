## Summary

Keep open. Current main still disables persistent Playwright for local-managed browser profiles, so strict SSRF HTTP(S) opens fail closed before redirect-hop inspection; the rollout needs security/product judgment rather than cleanup closure.

## What This Changes

Enable Playwright-backed navigation interception for local-managed browser profiles so strict SSRF policy can inspect redirect hops.

## Best Possible Solution

Add a targeted browser-plugin path that uses the existing Playwright navigation guard for local-managed profiles when strict SSRF requires redirect inspection, while preserving existing-session behavior and covered raw-CDP paths.

## Reproduction Assessment

Yes. Source inspection on current main shows a local-managed profile with `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false` skips the Playwright path and throws the strict redirect-inspection error for HTTP(S) opens before redirect hops can be inspected.

## Solution Assessment

Unclear pending maintainer/security review. Reusing `createPageViaPlaywright` and `gotoPageWithNavigationGuard` for strict local-managed opens looks like the narrowest technical direction, but the rollout scope has lifecycle, compatibility, and performance tradeoffs.
