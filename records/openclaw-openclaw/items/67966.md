## Summary

Keep open. Current main and the latest tagged release still route local-managed browser profiles away from persistent Playwright, so explicit strict SSRF HTTP(S) opens fail closed before redirect-hop inspection; this is valid security-sensitive feature work that needs maintainer/security rollout judgment.

## What This Changes

The issue requests Playwright-backed navigation interception for local-managed browser profiles so strict SSRF policy can inspect redirect hops.

## Best Possible Solution

Implement a targeted browser-plugin path that uses the existing Playwright navigation guard for local-managed profiles when strict SSRF requires redirect-hop inspection, while preserving existing-session behavior and raw CDP paths for safe cases such as about:blank or non-strict navigation.

## Reproduction Assessment

Yes. Source inspection on current main shows a local-managed profile with browser.ssrfPolicy.dangerouslyAllowPrivateNetwork false skips the Playwright path and throws the strict redirect-inspection error for HTTP(S) opens before redirect hops can be inspected.

## Solution Assessment

Unclear pending maintainer/security review. Reusing createPageViaPlaywright and gotoPageWithNavigationGuard for strict local-managed opens looks like the narrowest technical direction, but always-on, strict-only, or configurable rollout choices affect local CDP lifecycle and performance.
