## Summary

Keep open. Current main still leaves local-managed browser profiles on the raw CDP path; explicit strict SSRF HTTP(S) opens skip the Playwright route guard and throw before redirect-hop inspection, and the rollout needs maintainer/security judgment.

## What This Changes

Request local-managed browser profiles to use Playwright navigation interception so strict SSRF policy can inspect redirect hops like remote CDP profiles.

## Best Possible Solution

Implement a targeted browser-plugin path that uses Playwright for strict local-managed opens while preserving existing-session behavior and safe raw CDP paths, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a local-managed profile with `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false` skips the Playwright path and throws the strict redirect-inspection error for HTTP(S) opens before redirect hops can be inspected.

## Solution Assessment

Unclear pending maintainer/security review. Reusing `createPageViaPlaywright` and `gotoPageWithNavigationGuard` for strict local-managed opens looks like the narrowest direction, but the exact rollout has lifecycle and performance tradeoffs.
