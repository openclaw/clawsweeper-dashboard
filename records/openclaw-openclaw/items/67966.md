## Summary

Keep open. Current main still lacks local-managed Playwright navigation interception; strict HTTP(S) opens in local-managed mode still skip the Playwright guard and fail before redirect-hop inspection, and the rollout needs maintainer/security judgment because it changes SSRF enforcement plus local-managed Playwright/CDP lifecycle behavior.

## What This Changes

The issue requests Playwright-backed navigation interception for local-managed browser mode so strict browser SSRF policy can inspect redirect hops.

## Best Possible Solution

Implement a targeted browser-plugin path that uses the existing Playwright navigation guard for strict local-managed opens, preserves existing-session behavior and safe raw-CDP fallbacks, and adds regression coverage for local-managed strict-mode IP-literal navigation plus private redirect blocking.

## Reproduction Assessment

Yes, source-reproducible on current main: a local-managed profile with `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false` takes the non-Playwright `openTab` path and throws the strict redirect-inspection error for HTTP(S) URLs before raw CDP creation can inspect redirect hops.

## Solution Assessment

Unclear pending maintainer/security review. Reusing `createPageViaPlaywright` and `gotoPageWithNavigationGuard` for strict local-managed opens looks like the narrowest maintainable direction, but the exact rollout should be coordinated with the local-managed lifecycle/performance work.
