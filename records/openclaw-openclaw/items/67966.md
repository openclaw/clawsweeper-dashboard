## Summary

Keep open. Current main and the latest release still route local-managed browser profiles away from persistent Playwright, so strict SSRF HTTP(S) opens fail closed before redirect-hop inspection can run.

## What This Changes

Request local-managed browser profiles to use Playwright navigation interception for strict SSRF redirect-hop inspection, matching the remote CDP path.

## Best Possible Solution

Add targeted browser-plugin support for Playwright-backed navigation guard on local-managed strict SSRF opens, preserving existing-session and non-strict raw CDP behavior with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows local-managed profiles return usesPersistentPlaywright false, so strict HTTP(S) open hits the redirect-inspection guard and throws before a redirect-inspecting path can run.

## Solution Assessment

Unclear pending maintainer/security review. Reusing createPageViaPlaywright and gotoPageWithNavigationGuard for strict local-managed opens looks like the narrowest technical direction, but always-on versus strict-only rollout changes lifecycle and performance behavior.
