## Summary

Keep open. Current main still leaves local-managed browser profiles on the raw CDP tab-open path, while strict SSRF mode requires inspectable redirect hops and fails closed before raw CDP navigation can run. The requested direction is valid, but the exact rollout is security-sensitive because it changes local browser lifecycle, Playwright attachment, and SSRF enforcement behavior.

## What This Changes

The issue requests Playwright-backed navigation interception for local-managed browser profiles so strict SSRF policy can inspect redirect hops before navigation.

## Best Possible Solution

Implement a browser plugin change that routes local-managed profiles through the existing Playwright navigation guard when strict SSRF policy requires redirect-hop inspection, while preserving Chrome MCP existing-session behavior and safe raw CDP fallbacks such as non-strict navigation and `about:blank`. Add focused tests for local-managed strict-mode IP-literal navigation, blocked private redirects, and unchanged remote/existing-session behavior.

## Reproduction Assessment

Yes. Configure the local-managed `openclaw` profile with `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork=false` and open an HTTP(S) URL; the source trace shows `openTab` skips Playwright for local-managed profiles, `requiresInspectableBrowserNavigationRedirectsForUrl` returns true, and the strict redirect-inspection error is thrown before raw CDP navigation.

## Solution Assessment

Unclear as written. The direction of using the existing Playwright guarded path is maintainable, but always enabling Playwright for all local-managed navigation may be broader than necessary; a strict-mode-only or explicitly configured rollout is the safer design to review first.
