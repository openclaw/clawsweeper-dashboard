- `impact:security`: The issue is about browser SSRF enforcement and whether local browser navigation can inspect redirects before reaching private/internal hosts.

## Summary

Keep open. Current main and the latest release still route local-managed browser profiles away from persistent Playwright, so explicit strict SSRF HTTP(S) opens fail before redirect-hop inspection can run; the remaining work is valid but security-sensitive and needs maintainer rollout judgment.

## What This Changes

Enable local-managed browser profiles to use Playwright-backed navigation interception so strict SSRF policy can inspect redirect hops.

## Best Possible Solution

Implement a targeted browser-plugin path that uses the existing Playwright navigation guard for local-managed profiles when explicit strict SSRF requires redirect-hop inspection, while preserving existing-session behavior and raw CDP paths for safe non-strict or about:blank cases.

## Reproduction Assessment

Yes at source level, but not by a live browser run. Local-managed profiles return usesPersistentPlaywright false, openTab only uses Playwright behind that capability, and explicit strict HTTP(S) URLs throw the redirect-inspection error before raw CDP tab creation.

## Solution Assessment

Unclear pending maintainer/security review. Reusing createPageViaPlaywright and gotoPageWithNavigationGuard for strict local-managed opens looks like the narrowest technical path, but always-on, strict-only, or configurable rollout choices affect local CDP lifecycle and compatibility.
