## Summary

Keep open. Current main still sends local-managed open-tab requests through the raw CDP path, so strict SSRF HTTP(S) opens that need redirect-hop inspection fail before Playwright can inspect redirects.

## What This Changes

Request Playwright-backed redirect-hop inspection for local-managed browser open/navigation so strict SSRF policy can allow safe targets and block private redirects consistently.

## Best Possible Solution

Implement a targeted browser plugin change that uses the existing Playwright guard for local-managed open-tab navigation when strict SSRF requires redirect inspection, preserves existing-session behavior and safe raw-CDP fallbacks, and adds focused regression tests.

## Reproduction Assessment

Yes. With a local-managed profile, `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false`, and an HTTP(S) `/tabs/open` request to an IP-literal URL, the source path skips Playwright and throws the strict redirect-inspection error before raw CDP tab creation.

## Solution Assessment

Unclear pending maintainer/security review. Reusing the existing Playwright guard for strict local-managed open-tab navigation looks like the narrowest maintainable fix, but the exact rollout should account for lifecycle and performance work in #64929.
