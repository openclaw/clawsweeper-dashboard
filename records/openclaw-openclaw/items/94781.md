## Summary

Keep open: the PR targets a real Copilot registration crash, but the current patch is too broad because it can silently drop durable Copilot SDK session storage instead of only avoiding metadata-mode runtime access.

## What This Changes

The PR changes Copilot plugin registration to make `runtime.state.openSyncKeyedStore` optional and omit `sessionStore` when that API is absent.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves durable session-store creation in full runtime, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata loader can pass `runtime: {}` while the Copilot entrypoint dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No: optional chaining is a plausible mitigation, but it is too broad because it also hides a missing full-runtime state API and drops durable session persistence. The safer fix is a registration-mode boundary plus full-runtime session-store coverage.
