## Summary

Keep open: the patch is a plausible Copilot bug fix and is not obsolete on current main, but it lacks after-fix real behavior proof, has no missing-state regression coverage in this branch, and needs maintainer consolidation with the competing open draft before merge.

## What This Changes

The PR makes Copilot plugin registration guard `api.runtime.state?.openSyncKeyedStore` and omit `sessionStore` when the runtime state API is unavailable.

## Best Possible Solution

Land one canonical Copilot fix that preserves durable state when available, explicitly chooses the missing-state fallback behavior, adds missing-state regression coverage, and includes redacted installed-runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection shows current main and v2026.6.8 throw if `api.runtime.state` is absent, and the linked issue reports that exact shipped failure, but I did not run a real installed host that omits the state API.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the plugin-local fallback is a reasonable mitigation because the harness already treats `sessionStore` as optional, but maintainers should choose it over enforcing the runtime state contract and should consolidate the duplicate fix branches.
