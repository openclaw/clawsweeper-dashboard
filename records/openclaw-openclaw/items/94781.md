## What This Changes

The PR changes the Copilot plugin entrypoint to optional-chain `api.runtime.state.openSyncKeyedStore` and register the harness without `sessionStore` when that API is missing.

## Best Possible Solution

Land one canonical Copilot fix that skips non-full registration before runtime state access, preserves durable sessions in full runtime, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata loader can pass `runtime: {}` while the Copilot entrypoint dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The plugin-local optional fallback is a plausible mitigation, but the safer maintainable fix is to avoid runtime-only Copilot registration outside full mode while keeping full-mode state-store creation loud.
