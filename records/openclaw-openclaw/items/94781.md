## Summary

Keep open: the PR targets a real current-main and v2026.6.8 Copilot registration failure in no-state registration capture, but it still needs reporter-equivalent runtime proof and focused coverage before merge.

## What This Changes

The PR makes Copilot plugin registration tolerate a missing runtime state API by omitting the SDK session store when `openSyncKeyedStore` is unavailable.

## Best Possible Solution

Land one canonical Copilot fix that preserves durable state-backed sessions in full runtime registration and covers no-state registration capture without failing plugin load.

## Reproduction Assessment

Yes, source-level: captured plugin registration constructs an API with `runtime: {}`, and current Copilot registration immediately dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the installed reporter-equivalent CLI path in this read-only review.

## Solution Assessment

Mostly: the plugin-local guard is a narrow fix because full runtime registration still uses the durable store while no-state capture can register without state. The best merge shape also adds focused regression coverage and real CLI proof.
