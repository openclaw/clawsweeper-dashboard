## Summary

## What This Changes

This PR changes `extensions/copilot/index.ts` to call `openSyncKeyedStore` only when `runtime.state` exposes it and otherwise register the Copilot harness without `sessionStore`.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only harness registration outside `full` mode, keeps full-runtime state storage mandatory, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes from source inspection: current main's CLI metadata path passes `runtime: {}` while Copilot dereferences `api.runtime.state.openSyncKeyedStore`, and the shipped-version report matches that path. I did not run the installed reporter-equivalent CLI path in this read-only review.

## Solution Assessment

No. The optional state-shape fallback is a plausible mitigation, but the narrower maintainable fix is to skip Copilot runtime-only registration outside `full` mode and keep full-runtime state-store failures loud.
