## What This Changes

The PR changes `extensions/copilot/index.ts` so Copilot registration only opens the durable SDK session store when `api.runtime.state.openSyncKeyedStore` exists, otherwise registering the harness without `sessionStore`.

## Best Possible Solution

Land one narrow Copilot fix that preserves durable bindings when state exists, tolerates a missing state surface during registration, includes missing-state regression coverage, and has redacted real `openclaw plugins list` or `openclaw status` proof.

## Reproduction Assessment

## Solution Assessment

Yes as an acceptable mitigation, not merge-ready as submitted: the harness already supports missing persistence and the PR preserves the durable path when the API exists. The best mergeable version should add the missing-state test and real behavior proof.
