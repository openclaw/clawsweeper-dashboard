## What This Changes

The PR changes `extensions/copilot/index.ts` to guard `api.runtime.state?.openSyncKeyedStore` and omit the Copilot `sessionStore` when the host runtime lacks that API.

## Best Possible Solution

Land one canonical Copilot fix that returns before runtime-only registration outside `full`, keeps durable session-store creation loud in full runtime, adds focused missing-state/full-mode coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata loader can pass `runtime: {}` while the Copilot entrypoint dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. Optional chaining is a plausible mitigation, but it is too broad because it also hides a missing full-runtime state API and drops durable session persistence; a registration-mode guard is the safer owner-boundary fix.
