## Summary

Keep open: current `main` and the latest release still contain the Copilot missing-state dereference, but this PR is not merge-ready because its fallback also affects full runtime and can silently drop durable SDK session persistence.

## What This Changes

The PR changes Copilot plugin registration to skip creating a `sessionStore` when `api.runtime.state.openSyncKeyedStore` is absent.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-runtime `sdk-sessions` persistence, adds focused regression coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current `main` passes `runtime: {} as PluginRuntime` during CLI metadata registration while Copilot dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The optional fallback is a plausible mitigation, but the safer fix is to avoid runtime-only Copilot registration outside full mode while keeping full-mode state-store creation loud.
