## Summary

Current main already fixes the linked Copilot registration failure at the host/runtime boundary by exposing `api.runtime.state.openSyncKeyedStore` to trusted plugins; this PR adds an unproven fallback for the pre-fix release state rather than a still-needed main change.

## What This Changes

This PR wraps the Copilot plugin's `api.runtime.state.openSyncKeyedStore` call in optional chaining and omits `sessionStore` when the state API is missing.

## Best Possible Solution

Keep current main's runtime-state API as the canonical fix and close this duplicate fallback PR; ship or backport the host-runtime fix for affected release users.

## Reproduction Assessment

Yes. Source inspection reproduces the reported release failure: `v2026.6.8` lacks `runtime.state`, while the shipped Copilot entrypoint dereferences `api.runtime.state.openSyncKeyedStore` during registration.

## Solution Assessment

No for this PR as a landing path. Providing the runtime state API on current main is the cleaner owner-boundary fix; a fallback would need explicit tested compatibility proof if maintainers still want it.
