## Summary

Keep open: the linked Copilot registration bug is real, but this branch is not merge-ready because the fallback also applies in full runtime, can drop durable Copilot SDK session bindings, and still lacks reporter-equivalent CLI proof.

## What This Changes

The PR makes Copilot registration tolerate a missing `api.runtime.state.openSyncKeyedStore` by omitting `sessionStore` when that runtime API is unavailable.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-runtime `sdk-sessions` creation, adds focused coverage, and supplies redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata path can pass `runtime: {}` while Copilot dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. Optionalizing the state-store lookup is a plausible mitigation, but it is too broad because it also hides a missing full-runtime state API and drops durable session persistence; a registration-mode boundary is safer.
