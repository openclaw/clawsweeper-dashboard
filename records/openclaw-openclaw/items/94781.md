## Summary

Keep open: the PR targets a real source-reproducible Copilot CLI metadata crash, but the patch is not merge-ready because the optional runtime-state fallback also applies in full registration and would let Copilot run without durable SDK session storage.

## What This Changes

The PR optional-chains Copilot runtime state-store creation, logs a warning when it is missing, and registers the harness without `sessionStore` when unavailable.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-mode `sdk-sessions` persistence, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata path passes `runtime: {} as PluginRuntime` while Copilot dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The optional fallback is a plausible mitigation, but the safer fix is to avoid runtime-only Copilot registration outside full mode while preserving full-mode state-store creation and durable session persistence.
