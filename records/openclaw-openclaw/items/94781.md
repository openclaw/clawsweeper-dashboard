## Summary

## What This Changes

The PR changes `extensions/copilot/index.ts` to tolerate missing `runtime.state.openSyncKeyedStore` and register Copilot without a durable `sessionStore` when that API is absent.

## Best Possible Solution

Land one canonical Copilot fix that skips non-full registration before touching runtime state, preserves durable sessions in full runtime, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes from source inspection: current main registers Copilot in a CLI metadata path with an empty runtime while the entrypoint dereferences `runtime.state`. I did not run the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The narrow owner-boundary fix is to avoid Copilot full-runtime registration work in cli-metadata/non-full modes while keeping full-mode durable state as required.
