## Summary

Keep open: this is a focused candidate for the Copilot registration bug, but the patch is not merge-ready because the optional fallback can also mask a broken full-runtime state API and run Copilot without durable SDK session bindings; the supplied proof is still synthetic rather than reporter-equivalent installed CLI or gateway proof.

## What This Changes

The branch optional-chains Copilot's plugin runtime state-store opener during registration and omits the session store when `openSyncKeyedStore` is unavailable.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-mode `sdk-sessions` persistence, adds focused coverage, and includes redacted installed CLI or gateway proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata loader can pass `runtime: {}` while the Copilot entrypoint dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No: optional chaining is a plausible mitigation, but it is too broad because it also hides a missing full-runtime state API and drops durable session persistence. The safer fix is a registration-mode boundary plus full-runtime session-store coverage.
