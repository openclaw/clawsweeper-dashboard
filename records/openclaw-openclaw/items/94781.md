The PR changes `extensions/copilot/index.ts` so Copilot only opens the durable `sdk-sessions` store when `api.runtime.state.openSyncKeyedStore` exists, otherwise registering the harness without a `sessionStore`.

## Best Possible Solution

Land one canonical Copilot fix that keeps full-runtime durable session storage, handles metadata or missing-state registration without crashing, adds focused regression coverage, and includes redacted installed CLI proof.

## Solution Assessment

Unclear: the plugin-local fallback likely fixes the reported crash, but the best maintainable fix should be mode-aware and should preserve durable session storage wherever the full runtime state API is available.
