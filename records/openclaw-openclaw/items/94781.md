## Summary

Keep open: the reported Copilot registration crash is source-reproducible and still present on current main/latest release, but this PR is not merge-ready because it changes a full-runtime missing-state contract into a non-persistent Copilot session fallback and still lacks reporter-equivalent CLI proof.

## What This Changes

The PR changes `extensions/copilot/index.ts` to optional-chain `api.runtime.state?.openSyncKeyedStore`, warn when it is absent, and register Copilot without a session store when the state API is unavailable.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-mode `sdk-sessions` persistence, adds focused metadata/full-mode coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata path passes `runtime: {} as PluginRuntime` while Copilot dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The optional fallback is a plausible mitigation, but the safer fix is to avoid runtime-only Copilot registration outside full mode while preserving full-mode state-store creation and durable session persistence.
