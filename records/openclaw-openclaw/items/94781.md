## Summary

Keep this PR open, but the current branch is not merge-ready: it fixes the source-level metadata crash path by tolerating missing runtime state, but it also lets full Copilot runtime registration continue without durable SDK session persistence and still lacks reporter-equivalent CLI proof.

## What This Changes

This PR makes Copilot registration optional-chain `api.runtime.state.openSyncKeyedStore`, logs a warning when it is absent, and registers the harness without a `sessionStore`.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves full-runtime `sdk-sessions` persistence, adds focused coverage, and includes redacted installed CLI proof.

## Reproduction Assessment

Yes, at source level: current main's CLI metadata path passes `runtime: {} as PluginRuntime` while Copilot dereferences `api.runtime.state.openSyncKeyedStore`. I did not execute the reporter-equivalent installed CLI path in this read-only review.

## Solution Assessment

No. The optional fallback is a plausible mitigation, but the safer fix is to avoid runtime-only Copilot registration outside full mode while keeping full-mode state-store creation loud.
