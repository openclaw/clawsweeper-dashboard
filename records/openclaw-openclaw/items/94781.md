## What This Changes

The PR makes the Copilot plugin skip durable session-store setup when `api.runtime.state?.openSyncKeyedStore` is unavailable and registers the harness without `sessionStore`.

## Best Possible Solution

Land one canonical Copilot fix that preserves the durable state-backed path, adds missing-state regression coverage, and includes redacted real installed-runtime proof; consolidate this PR with https://github.com/openclaw/openclaw/pull/94525 before merge.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Unclear: the plugin-local fallback is a reasonable mitigation because the harness already treats `sessionStore` as optional, but maintainers should confirm fail-open registration is preferred over fixing the host runtime to always provide `state`.
