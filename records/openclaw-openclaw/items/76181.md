## Summary

Keep open. This PR is the right narrower direction for the Ollama idle-timeout regression, but the proposed local URL classifier misses an already documented 127/8 loopback Ollama path and the user-facing fix still needs a changelog entry before merge.

## What This Changes

The PR passes the resolved model base URL into LLM idle-timeout resolution and disables the fallback 120-second idle watchdog for local/private/.local provider URLs, with focused resolver tests.

## Best Possible Solution

Land the provider-aware idle-timeout policy, but make the local endpoint classifier match existing Ollama loopback/local semantics and document the user-facing fix in the active changelog.

## Reproduction Assessment

Yes for the review finding: the PR helper checks only exact `127.0.0.1`, while current docs/tests show `http://127.0.0.2:11434` is a local loopback Ollama provider. The original long-running Ollama regression has a concrete report, but I did not run a live model reproduction here.

## Solution Assessment

No, not as submitted. The local-provider fallback is the maintainable direction, but it should at least cover documented 127/8 loopback providers and include the required release-note entry before merge.
