## Summary

Keep this PR open: the runtime fix matches the linked Ollama regression at source level, but the user-facing Ollama docs still describe the old native `num_ctx` behavior and the PR has a protected maintainer label.

## What This Changes

The PR restores catalog-derived `num_ctx` for native Ollama `/api/chat`, skips the default idle watchdog for local provider base URLs, passes model `baseUrl` into timeout resolution, and updates related tests and changelog.

## Best Possible Solution

Keep the runtime fix, align the Ollama docs and release note with the new native precedence, then land after maintainer approval and normal checks.

## Reproduction Assessment

Yes, at source level. Current main drops catalog windows from native Ollama `options.num_ctx`, and the linked regression is consistent with context truncation rather than the idle watchdog alone.

## Solution Assessment

No, not complete as-is. The runtime direction is narrow, but the docs still tell users that native Ollama leaves `num_ctx` unset unless explicitly configured.
