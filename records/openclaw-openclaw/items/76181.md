## Summary

Keep open: the PR targets a real current-main local Ollama idle-timeout regression and has moved in the right direction, but the new local URL classifier can misclassify numeric-looking hostnames and the user-facing fix still needs a changelog entry before merge.

## What This Changes

The PR passes model `baseUrl` into LLM idle-timeout resolution and disables the fallback 120-second idle watchdog for local/private/.local provider URLs, with resolver tests.

## Best Possible Solution

Land the provider-aware idle-timeout policy after tightening host classification to strict IP literals and adding the required active changelog fix entry.

## Reproduction Assessment

Yes for the review findings: the classifier bug is reproducible with the PR helper logic on numeric-looking hostnames, and the missing changelog is visible from the PR file list. The original Ollama regression has a concrete report and current-main code path, but I did not run a live slow model.

## Solution Assessment

No, not as submitted. Disabling the default watchdog for local provider URLs is the narrow maintainable direction, but the local detector should not use partial numeric parsing and the user-facing fix needs release-note coverage.
