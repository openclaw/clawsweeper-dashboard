## Summary

Keep open: current main and latest release still compute status-only memory index identity from `settings.model` or the adapter default instead of `local.modelPath`, and the mergeable linked fix at https://github.com/openclaw/openclaw/pull/91010 already owns the repair.

## What This Changes

Local GGUF memory indexes rebuilt with `memorySearch.provider: "local"` and a custom `local.modelPath` can still be reported as `Dirty: yes` by `openclaw memory status`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/91010 or an equivalent narrow status-identity fix, then close this issue after the fix reaches main; keep the separate vector-store status symptom tracked at https://github.com/openclaw/openclaw/issues/92102.
