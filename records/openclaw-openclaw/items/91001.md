## What This Changes

The issue reports that local GGUF memory indexes remain reported as `Dirty: yes` after rebuild when `memorySearch.provider` is `local` and `local.modelPath` is configured.

## Best Possible Solution

Land one narrow memory-core fix that mirrors the local provider model identity in the status-only path, with regression coverage for `local.modelPath` and current proof that rebuilt local indexes report clean.

## Reproduction Assessment

Yes, at source level: current main’s status-only identity path does not use `memorySearch.local.modelPath`, while the local llama.cpp provider records that path as the indexed model. I did not run a live GGUF rebuild in this read-only review.

## Solution Assessment

Yes, the best fix is to make the status-only identity mirror use the same local modelPath-based identity that indexing persists, without changing config shape or local embedding runtime behavior. The open canonical PR path already matches that boundary.
