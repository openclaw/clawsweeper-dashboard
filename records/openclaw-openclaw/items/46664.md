## Summary

Keep open. Current main still has only narrow, mostly process-local mobile queue paths, while the requested durable offline text/voice outbox needs product, mobile storage, gateway replay, privacy, and OS background-behavior decisions.

## What This Changes

The issue requests a durable iOS/Android offline command outbox that captures text and voice intents while disconnected and syncs them to the gateway on reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship one durable encrypted storage and reconnect-replay slice for mobile text/voice intents before broader queue-management UI.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable: this is a feature request, and current source/tests show the missing durable offline outbox rather than a broken existing contract.

## Solution Assessment

Is this the best way to solve the issue? No: the full cross-platform SQLite, background sync, and queue-management proposal is useful directionally, but the maintainable path is a staged contract for payload shape, captured time, idempotency, storage privacy, and mobile OS limits.
