## Summary

Keep open. The request is a valid mobile feature, and current main/latest release only have narrow in-memory Android voice and iOS watch-reply queues, not a durable cross-platform text/voice outbox with persistence, replay semantics, and privacy controls.

## What This Changes

Add a durable offline command outbox for iOS/Android mobile text and voice intents that captures disconnected user commands and replays them to the gateway on reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then add durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.

Not applicable: this is a feature request, not a broken existing contract. Source inspection shows partial in-memory queue behavior and the missing durable mobile outbox rather than a current-main bug reproduction path.

## Solution Assessment

No for the all-at-once SQLite/background-sync/queue-management proposal as written. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on existing narrow mobile queue patterns.
