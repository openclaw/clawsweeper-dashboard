## Summary

Keep open. Current main still has only narrow, process-local mobile queue paths, while the requested durable cross-platform text/voice outbox needs product, API, storage, privacy, and mobile OS behavior decisions.

## What This Changes

Add a durable offline command outbox to iOS/Android that captures text/voice intents while disconnected and replays them to the gateway on reconnect.

## Best Possible Solution

Define a staged mobile outbox contract, then ship one durable encrypted storage and reconnect-replay slice before broader queue-management UI.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows no existing durable offline text/voice outbox contract to reproduce as broken behavior.

## Solution Assessment

No. The full SQLite/background-sync/queue-management proposal is useful directionally, but the maintainable path is a staged contract around payload shape, capturedAt, idempotency, storage privacy, and mobile OS limits.
