## Summary

Keep open. Current main has useful partial mobile queue primitives, but it does not provide the requested crash-persistent, cross-platform offline command outbox for arbitrary text/voice intents with review/sync status and replay metadata.

## What This Changes

Request an iOS/Android offline command queue that captures user intents while disconnected, persists them locally, and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Keep this issue open and narrow it into a staged mobile outbox design: define a shared queued-intent payload with idempotency and capture time, persist the existing Android/iOS narrow queues first, then expand to arbitrary text/voice commands and queue management UI after owner review.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence gap check: Android text chat is disabled when gateway health is false, iOS PTT returns offline, and the existing queues are narrow in-memory paths rather than a persistent general outbox.

## Solution Assessment

No for a one-shot implementation of the full proposal. The safer path is a staged design that starts with payload/delivery semantics and persistence for the existing mobile queue paths before adding cross-platform queue management and background sync UX.
