## Summary

Keep open. Current main and the latest release have only narrow in-memory mobile queue paths plus gateway-to-node pending work, not the requested durable cross-platform text/voice offline command outbox; the remaining work needs product, replay, storage, and privacy scoping.

## What This Changes

Add a durable offline command queue to the iOS and Android node apps that stores disconnected text/voice intents and syncs them to the gateway after reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.
