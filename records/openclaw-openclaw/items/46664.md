## Summary

Keep open. Current main still has only narrow process-local mobile queue paths, and the live discussion correctly narrows the remaining work to a durable cross-platform outbox contract with storage, replay, privacy, and mobile OS behavior decisions.

## What This Changes

Requests a durable offline command outbox in the iOS and Android node apps to capture text/voice intents offline and sync them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship a durable encrypted storage and reconnect-replay slice for text/voice intents before broader queue-management UI.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable: this is a feature request, and source inspection shows the missing durable outbox rather than a broken existing contract.

## Solution Assessment

Is this the best way to solve the issue? No: the full SQLite/background-sync/queue-management proposal is useful directionally, but the maintainable path is a staged payload, storage, privacy, and replay contract.
