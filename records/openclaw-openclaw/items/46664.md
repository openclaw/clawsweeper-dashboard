## Summary

Keep open. Current main still only has narrow, mostly in-memory mobile queue paths, while the requested durable cross-platform text/voice command outbox needs product, API, storage, privacy, and mobile OS behavior decisions.

## What This Changes

The issue requests an iOS/Android offline command queue that stores text and voice user intents locally and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first, then implement durable encrypted local storage and reconnect replay for one narrow text/voice path before broad queue-management UI.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection shows partial in-memory mobile queues rather than a broken existing offline outbox contract.

## Solution Assessment

No. The full SQLite/background-sync/queue-management proposal is useful directionally, but the maintainable path is a staged design around payload shape, persistence, replay guarantees, privacy, and mobile OS limits.
