## Summary

Keep open. Current main has useful offline-adjacent mobile pieces, including a persisted iOS Watch chat queue and Android voice reconnect queue, but it still does not provide the requested durable, general iOS/Android phone command outbox for text and voice intents.

## What This Changes

Requested an offline mobile command queue that stores iOS/Android user intents while disconnected and syncs them to the gateway after reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then add encrypted durable storage and idempotent reconnect replay for text and voice before expanding review/edit/reorder and richer background-sync UX.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Current-main source inspection gives high-confidence proof that the remaining gap is a missing durable, general mobile outbox rather than a reproducible regression.

## Solution Assessment

No for implementing the full SQLite, background-sync, and queue-management proposal in one sweep. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on the existing iOS Watch queue and Android voice queue deliberately.
