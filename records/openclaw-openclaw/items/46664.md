## Summary

Keep open: current main has partial in-memory mobile queues, but the requested persistent cross-platform offline command outbox, general text/voice capture, queue management, and replay semantics are still missing and need maintainer product design.

## What This Changes

The issue requests a persistent offline command queue for iOS and Android that captures mobile user intents while disconnected and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first: persist queued intents on-device with stable IDs and captured-time metadata, reuse gateway chat.send idempotency for replay, then extend Android/iOS voice and text surfaces with queue status, user controls, and regression coverage.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main has only narrow in-memory queues plus explicit offline rejection paths rather than a broken documented offline outbox.

## Solution Assessment

No, this should not be treated as one narrow repair PR. The best direction is a staged owner-reviewed feature that first nails the mobile queued-intent payload, persistence, replay guarantees, and UX boundaries before broadening every command surface.
