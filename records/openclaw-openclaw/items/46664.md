## Summary

Keep open. Current main has narrow in-memory mobile queue patterns, but it still does not provide the requested durable cross-platform offline outbox for arbitrary iOS/Android text and voice intents; the remaining work needs product, replay, storage, and privacy scoping.

## What This Changes

The issue requests a durable iOS/Android mobile offline command queue that captures text or voice intents while disconnected and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.
