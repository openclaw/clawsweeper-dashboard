## Summary

Keep open. Current main has partial mobile queue patterns, but it does not provide the requested persistent, general offline command outbox for iOS/Android text and voice intents with queue management and replay semantics.

## What This Changes

Add an offline command queue to the iOS/Android node apps so user voice/text intents can be captured while disconnected and synced to the gateway after reconnect.

## Best Possible Solution

Define a staged mobile outbox: persist the existing Android voice queue first, add a shared queued-intent payload with captured time and idempotency metadata, then expand to text input and queue-management UI with focused docs and tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the requested general persistent offline command outbox is not current behavior.

## Solution Assessment

No, not as one large cross-platform feature. The maintainable path is a staged outbox design that builds on the existing Android voice queue, iOS reconnect hooks, and gateway idempotency contracts.
