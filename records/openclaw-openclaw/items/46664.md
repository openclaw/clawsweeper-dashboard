## Summary

Keep open. Current main still has only narrow offline-adjacent mobile queues: Android voice queues text in memory, iOS watch replies queue in memory, typed chat stays gateway-gated, and the gateway pending-work API is gateway-to-node rather than a durable mobile-originated outbox.

## What This Changes

The issue requests a durable offline mobile command outbox for iOS and Android that captures user intents while disconnected and syncs them later.

## Best Possible Solution

Define a staged mobile outbox contract first, then add durable encrypted local storage and idempotent reconnect replay for text/voice intents before expanding to review/edit/reorder and richer background-sync UX.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Current-main source inspection gives high-confidence proof that existing offline-adjacent queues are narrow and in-memory rather than a durable general mobile outbox.

## Solution Assessment

No for implementing the full SQLite/background-sync/queue-management proposal in one sweep. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on existing Android voice, iOS watch/reachability, and gateway pending-work/idempotency primitives deliberately.
