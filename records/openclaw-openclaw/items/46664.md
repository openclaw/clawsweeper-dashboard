## Summary

Keep open. Current main still has narrow, process-local mobile queue paths, but the requested durable offline text/voice outbox remains a product/API/privacy feature rather than an implemented core behavior or stale low-signal report.

## What This Changes

The issue requests a durable offline command queue for the iOS and Android mobile node apps that stores text or voice intents locally and replays them to the gateway after reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship one durable encrypted storage and reconnect-replay slice for mobile text/voice intents before broader queue-management UI.

## Reproduction Assessment

Not applicable: this is a feature request, and current source/tests show partial offline queue concepts rather than a broken existing contract. The missing behavior is a durable general mobile outbox across text/voice commands and app restarts.

## Solution Assessment

No: the full cross-platform SQLite, background sync, and queue-management proposal is useful directionally, but the maintainable path is a staged contract for payload shape, captured time, idempotency, storage privacy, and mobile OS limits.
