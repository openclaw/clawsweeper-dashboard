## Summary

Keep open. Current main has partial, narrow mobile queuing, but the requested persistent cross-platform command outbox for arbitrary text/voice intents is still missing and needs product/API/privacy design.

## What This Changes

Add a persistent iOS/Android offline command queue that captures text or voice intents while disconnected and syncs them to the gateway later.

## Best Possible Solution

Define a staged mobile outbox contract first, then implement durable encrypted local storage and reconnect replay for one narrow text/voice path before adding queue-management UI.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows only partial in-memory mobile queue patterns rather than a broken existing contract.

## Solution Assessment

No, the full requested SQLite/background-sync/queue-management feature is too broad as a single automatic repair. The maintainable path is a staged design around payload shape, persistence, replay guarantees, privacy, and mobile OS background limits.
