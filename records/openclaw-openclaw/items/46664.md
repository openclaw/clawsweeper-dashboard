## Summary

Keep open. Current main still has narrow mobile queue patterns, but not the requested durable cross-platform offline command outbox for arbitrary iOS/Android text and voice intents; the remaining work needs product, replay, storage, privacy, and mobile background-behavior decisions.

## What This Changes

Add a durable offline command queue for the iOS and Android node apps that captures text/voice intents while disconnected and syncs them to the gateway on reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.

No for the all-at-once SQLite/background-sync/queue-management proposal as written. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on existing narrow mobile queue patterns.
