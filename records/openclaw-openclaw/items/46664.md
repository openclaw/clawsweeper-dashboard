## Summary

Keep open. Current main and the latest release still only provide narrow offline-adjacent pieces: Android voice has a process-local queue, iOS has scoped watch and gateway-to-node pending paths, and neither platform has the requested durable cross-platform mobile command outbox with crash persistence, captured-time metadata, replay acknowledgements, privacy controls, and queue-management UX.

## What This Changes

Add a durable iOS/Android mobile offline outbox that captures text and voice commands while disconnected and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first, then add durable encrypted local storage and idempotent reconnect replay for text/voice intents before expanding to review/edit/reorder, Wi-Fi-only, and richer background sync UX.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection gives high-confidence current-state proof that existing queues are narrow and not a durable general mobile command outbox.

## Solution Assessment

No for implementing the full SQLite/background-sync/queue-management proposal in one sweep. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on existing Android voice, iOS watch/reachability, and gateway idempotency primitives deliberately.
