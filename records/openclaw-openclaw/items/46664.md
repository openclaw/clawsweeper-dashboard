## Summary

Keep open. Current main and v2026.5.19 still only provide narrow in-memory mobile queue paths plus gateway-to-node pending work, not the requested durable cross-platform text/voice mobile outbox; the remaining work needs product, replay, storage, and privacy scoping.

## What This Changes

Request a durable offline command queue for iOS and Android that stores text/voice intents locally and syncs them to the gateway when connectivity returns.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.
