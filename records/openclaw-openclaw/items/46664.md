## Summary

Keep open. Current main still has only narrow in-memory mobile queue paths, while the requested durable cross-platform text/voice command outbox remains a product/API/privacy design feature rather than a completed bug fix.

## What This Changes

Add a persistent offline command queue to the iOS and Android mobile node apps, with local storage, reconnect replay, status feedback, voice/text support, and queue management.

## Best Possible Solution

Define a staged mobile outbox contract first, then implement durable encrypted local storage and reconnect replay for one narrow text/voice path before adding broad queue-management UI.

## Solution Assessment

No. The full SQLite/background-sync/queue-management proposal is too broad for an automatic repair; the maintainable path is a staged design around payload shape, persistence, replay guarantees, privacy, and mobile OS limits.
