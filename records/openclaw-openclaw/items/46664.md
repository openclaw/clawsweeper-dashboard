## Summary

Keep open. Current main still has narrow in-process mobile queues and gateway-side node pending primitives, but it does not implement the requested durable cross-platform mobile outbox for offline text and voice intents; the remaining work needs product, API, replay, and privacy scoping.

## What This Changes

The issue requests a durable offline command queue for the iOS and Android mobile node apps so text or voice intents captured while disconnected can replay to the gateway after reconnect.

## Best Possible Solution

Define a staged mobile outbox contract first, then ship durable encrypted storage and reconnect replay for text/voice intents before broader queue-management UI.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection shows partial in-memory queue behavior today and the missing durable mobile outbox rather than a current-main bug reproduction path.

## Solution Assessment

No for the full all-at-once SQLite/background-sync/queue-management shape. The maintainable path is a staged payload, persistence, privacy, and replay contract that builds on the existing narrow queue patterns.
