## Summary

Keep open: the patch is a sound bounded-reader fix with direct catalog-path coverage, while current main still has the unbounded success read. The only blocker is maintainer choice because https://github.com/openclaw/openclaw/pull/96500 overlaps the same call site but does not include this PR's direct regression test.

## What This Changes

The PR replaces the OpenRouter video model catalog success-path `response.json()` call with `readProviderJsonResponse` and adds a focused oversized-stream cancellation test.

## Best Possible Solution

Land one bounded-reader implementation for this call site, preferably preserving this PR's direct catalog-path regression coverage if maintainers choose https://github.com/openclaw/openclaw/pull/96500 as the runtime patch.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path because `fetchOpenRouterVideoModels` still reads the successful catalog response with `response.json()`, and the PR body includes after-fix output showing the bounded reader rejects and cancels an oversized stream.

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` from the provider HTTP SDK is the narrowest maintainable fix because it stays inside the plugin SDK boundary and shares the existing cap, cancellation, and malformed JSON behavior; the remaining question is which overlapping PR should land.
