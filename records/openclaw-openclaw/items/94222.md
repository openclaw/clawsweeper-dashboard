## Summary

Keep open: current main and v2026.6.8 still have bounded but eager recursive skill discovery, prompt caps, and snapshot reuse, but no INDEX/lazy resolver or supported depth/exclusion contract for very large nested skill bundles.

## What This Changes

The issue requests indexed, lazy, or configurable skill discovery so very large nested skill bundles do not bloat prompt context or trigger repeated truncation warnings.

## Best Possible Solution

Maintainers should choose one canonical large-bundle skill discovery contract, likely indexed or lazy disclosure, then update loader behavior, prompt-budget behavior, docs, and regression coverage without adding overlapping fallback stacks.

## Reproduction Assessment

Yes for the nested truncation path by source and existing test coverage: current main still emits the reported warning and caps nested children. No live 700-skill profile was run, so the exact CPU and token impact remains unmeasured here.

## Solution Assessment

Unclear as implementation: INDEX or lazy discovery is plausible and probably cleaner than broad max-depth/exclusion knobs, but maintainers need to choose the public skills discovery contract before implementation.
