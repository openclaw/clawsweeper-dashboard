## Summary

Keep open: current main and v2026.6.8 still have bounded nested skill discovery that can truncate large grouped directories, and they do not provide an INDEX/lazy discovery contract or max-depth/exclude controls; existing prompt caps and snapshot reuse make this a product/API design issue rather than a strict bug-fix lane.

## What This Changes

The issue requests a large-skill-directory discovery mode, such as INDEX-based lazy loading or max-depth/exclusion controls, to reduce nested skill traversal and prompt catalog overhead.

## Best Possible Solution

Choose one canonical large-skill-bundle discovery contract that keeps traversal bounded, preserves prompt-budget limits and snapshot reuse, and documents the public skills/config behavior.

## Reproduction Assessment

Yes for the nested-directory truncation warning by source inspection and existing tests. No for the full 700+ skill live performance claim because this read-only pass did not run that large catalog scenario.

## Solution Assessment

Unclear: INDEX/lazy loading is plausible, but current OpenClaw already has skill caps, compact prompt fallback, on-demand SKILL.md reads, and snapshot reuse, so maintainers should pick the permanent discovery contract first.
