## Summary

Keep open: current main still has bounded but eager nested skill discovery that warns and truncates large grouping directories, and OpenClaw does not yet have the requested INDEX/lazy-load or max-depth/exclusion contract for very large skill bundles. Existing caps and session snapshot reuse reduce the blast radius, but they do not solve the central discovery and prompt-budget problem.

## What This Changes

The issue requests indexed or lazy skill discovery, or max-depth/exclusion controls, so very large nested skill bundles do not repeatedly consume discovery and prompt budget.

## Best Possible Solution

Choose one canonical large-bundle skill discovery contract, likely indexed or lazy disclosure, then update the loader, prompt-budget behavior, docs, and regression tests without adding broad fallback stacks.

## Reproduction Assessment

Yes for the nested truncation path by source and existing test coverage: current main still emits the reported warning and caps nested children. The exact 700-skill live CPU/context cost has not been profiled in this pass.

## Solution Assessment

Unclear as implementation: INDEX/lazy discovery is plausible and cleaner than another broad config knob, but maintainers should choose the permanent contract because max-depth and exclusion settings would expand public config and may hide skills rather than solve selection.
