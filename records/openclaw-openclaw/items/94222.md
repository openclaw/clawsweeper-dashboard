## Summary

Keep open: current main and v2026.6.8 still use bounded eager nested skill discovery that warns and truncates large grouped directories, and there is still no INDEX/lazy resolver, smart loading mode, or depth/exclusion contract for very large skill bundles. Related smart-index and skill-graph issues overlap, but this issue preserves a concrete nested-directory truncation path and large-bundle report that is not strictly superseded.

## What This Changes

The issue requests indexed or lazy discovery for very large skill directories so large skill bundles do not bloat prompts or repeatedly hit nested discovery truncation.

## Best Possible Solution

Choose one canonical large-bundle skill discovery contract, likely indexed or progressive disclosure, then update loader behavior, prompt semantics, docs, and regression tests while cross-linking the smart-index and graph trackers.

## Reproduction Assessment

Yes for a source-level reproduction of the nested truncation path: current main still emits the reported nested warning and the existing test covers capped nested discovery. No live 754-skill profile was run in this read-only review, so the exact CPU and token-cost magnitude remains unproven.

## Solution Assessment

Unclear. INDEX-based lazy loading is plausible, but maintainers should choose the durable contract across smart indexing, dependency graphs, MCP indexing, and config tradeoffs before implementation.
