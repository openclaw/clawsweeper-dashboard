Keep open: this is a concrete Lobster workflow bug report, but current source review does not prove the condition-plus-stdin bypass is fixed or reproducible on main. ## What This Changes

The issue reports that a Lobster workflow step with `condition:` still runs when its `stdin:` references an upstream conditional step that was skipped.

## Best Possible Solution

Add a focused embedded-Lobster regression for a false-conditioned consumer whose `stdin` references a skipped conditional producer, then repair the precise runtime path or document a confirmed limitation.
