The issue asks for Lobster workflow `condition:` to gate a step even when that step reads `stdin:` from a skipped conditional upstream step.

## Best Possible Solution

Add a focused embedded-Lobster regression for a skipped conditional producer feeding `stdin` into a false-conditioned consumer, then either bump/fix `@clawdbot/lobster` or document the confirmed runtime limitation.

## Solution Assessment

Unclear.
