## Summary

Keep open: the latest commit fixes the prior same-path options poisoning case, but this PR still lets cache hits bypass fs-safe validation for root-bounded and regular JSON reads. Because this is a protected maintainer PR with concrete security-boundary findings, it needs normal author or maintainer repair before merge.

## What This Changes

The branch adds an opt-in process-level stat-validated cache for sync JSON reads, routes selected JSON helpers through it, and adds cache option coverage in infra tests.

## Best Possible Solution

Keep the performance win only if cache hits preserve the full fs-safe contract, including root-boundary inputs, symlink checks, hardlink checks, and file identity fields, or limit caching to a narrower caller-owned snapshot layer.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection. A cached allowed root read can be reused for an escaping relativePath with the same resolved path, and a cached regular read can survive hardlink or symlink state changes because the hit check uses fs.statSync and only compares mtime, size, and inode.

## Solution Assessment

No. The latest options-key fix is in the right direction, but the best fix must preserve fs-safe validation on cache hits or move the optimization to a caller-owned snapshot layer.
