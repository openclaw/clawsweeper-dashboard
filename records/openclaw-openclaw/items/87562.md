## Summary

Keep open. The PR is aimed at the right OpenRouter boundary and adds focused mocked coverage, but direct `result()` consumers still bypass reconciliation, live provider proof is absent, and the synchronous final-message lookup remains a merge risk.

## What This Changes

The branch adds an OpenRouter cost-reconciliation helper and tests, then wraps OpenRouter streams so final streamed messages can raise `usage.cost.total` from `/generation` `total_cost`.

## Best Possible Solution

Land a version that reconciles exactly once for both iterator and direct `result()` paths, includes redacted live OpenRouter `/generation` proof with latency, and has maintainer signoff on the final-message delay and overwrite policy.

## Reproduction Assessment

Source-reproducible, but not live-reproduced in this review. Current `main` stores streamed OpenAI-compatible usage directly, and the linked report provides concrete OpenRouter `/generation` mismatches, but this review did not run a real provider call.

## Solution Assessment

No, not fully. The iterator path is a plausible fix for the main agent loop, but the wrapper must also reconcile direct `result()` consumers and needs live endpoint proof before this is the safest maintainable solution.
