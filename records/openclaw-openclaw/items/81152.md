## Summary

Keep open: this PR is a plausible fix direction for the Codex app-server idle-timeout bug, but the latest head still counts selected request families as attempt progress before active-turn validation, and the external proof is still fake-client test output rather than real app-server behavior.

## What This Changes

The PR changes the Codex app-server run-attempt watchdog from a fixed post-start timeout to a progress-aware idle timeout with regression coverage for progressing turns and unrelated app-server traffic.

## Best Possible Solution

Keep the progress-aware watchdog, but move request-response attempt progress updates after active thread/turn validation and require redacted real Codex app-server proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main accepts a turn and then enforces a fixed post-start timeout while current-turn notifications are scoped separately. I did not run a live Codex app-server session because this review is read-only.

## Solution Assessment

No, not yet: the watchdog direction is narrow and maintainable, but request families are still counted as attempt progress before their helpers confirm active-turn scope. The safer fix is to set attempt progress only from validated current-turn notifications and validated current-turn request responses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate attempt progress on validated turn-scoped requests:** `extensions/codex/src/app-server/run-attempt.ts:1368`
  - body: This sets `requestCountsAsTurnActivity = true` before the elicitation helper validates the active `threadId`/`turnId`; the user-input and approval branches do the same. Because the `finally` block updates `turnAttemptLastProgressAt` whenever that flag is true, a mismatched request using one of these method names can keep an idle accepted turn alive. Set the flag only after the request is confirmed current-turn, or have the helpers return a validated progress marker.
  - confidence: 0.89
