## Summary

Keep open: the PR targets a real source-reproducible Codex app-server liveness bug, but the head patch is still not merge-ready because some request families can reset attempt progress before active thread/turn validation succeeds. The external proof gate also remains mock-only: the PR body supplies fake-client Vitest output and explicitly says no live OAuth/frontier Codex app-server run was tested.

## What This Changes

The PR replaces the Codex app-server post-turn/start absolute timeout with a progress-aware attempt idle watchdog and adds fake-client regressions for current-turn progress plus non-turn request traffic.

## Best Possible Solution

Keep the progress-aware watchdog direction, but count request-response progress only after validated active thread/turn scope and require real after-fix Codex app-server proof or maintainer override before merge.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: accept a Codex turn, emit current-turn notifications past params.timeoutMs without turn/completed, and the fixed post-start attempt timer still aborts independently of that progress.

## Solution Assessment

No, not as written. The progress-aware idle watchdog is the right narrow owner-boundary fix, but request-response attempt progress still needs to be derived from successful current-turn validation rather than method name alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate attempt progress after turn-scope validation:** `extensions/codex/src/app-server/run-attempt.ts:1365`
  - body: This still marks elicitation, user-input, and approval request methods as turn progress before their handlers confirm the params belong to the active thread and turn. Those helpers return undefined for missing or mismatched scope, but the finally block still records attemptProgress: true, so an unrelated request with one of these method names can keep a dead accepted turn alive. Move the progress flag after validation succeeds or have the helpers return a scoped result the finally block can trust.
  - confidence: 0.86
