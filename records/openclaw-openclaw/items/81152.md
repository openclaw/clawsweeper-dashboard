## Summary

Keep open: the progress-aware watchdog is a useful fix direction, but the PR is not merge-ready because some turn-scoped request branches still mark attempt progress before confirming the request belongs to the active turn, and the external real-behavior proof is still mock-only.

## What This Changes

The PR changes the Codex app-server attempt watchdog so current-turn progress can extend the timeout while non-turn traffic cannot.

## Best Possible Solution

Keep the progress-aware attempt watchdog, rebase it over current main’s Codex watchdog changes, gate request-response progress on confirmed current-turn scope for every request family, and add redacted live Codex app-server proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main still has the fixed post-start attempt timeout, and the PR-head bug is visible where selected request branches set the progress flag before helper-level current-turn validation. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet: the watchdog approach is the right narrow fix, but the PR still needs request-scope gating and real after-fix behavior proof before it is the maintainable solution.
