## Summary

Keep open: the latest head still marks some special app-server request methods as attempt progress before their helpers prove the request belongs to the active turn, and the external proof remains fake-client only.

## What This Changes

The branch replaces the Codex app-server fixed post-start attempt timeout with a progress-aware idle watchdog and adds fake-client regressions for current-turn progress plus non-turn request traffic.

## Best Possible Solution

Keep the progress-aware watchdog approach, but move attempt-progress marking behind current-turn validation for every request family, add the mismatched-method regression, and require redacted live app-server proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main accepts turn/start and then enforces a fixed post-start timeout while current-turn notifications are tracked separately. I did not run a live OAuth/frontier Codex app-server session in this read-only review.

## Solution Assessment

No, not yet: the progress-aware watchdog is the right direction, but request-response progress must be derived from successful current-turn validation rather than method names. The safer fix is to share or move the validation before setting attemptProgress.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate attempt progress on validated turn requests:** `extensions/codex/src/app-server/run-attempt.ts:1368`
  - body: This sets `requestCountsAsTurnActivity` for `mcpServer/elicitation/request` before `handleCodexAppServerElicitationRequest` confirms the params belong to the active thread and turn; the same pattern is used for user-input and approval requests. Those helpers return `undefined` for missing or mismatched scope, but the `finally` block still records `attemptProgress: true`, so repeated unrelated requests with these method names can keep an idle accepted turn alive. Set the progress flag only after the request is confirmed to match the active turn, and add a regression for a mismatched same-method request.
  - confidence: 0.88
