## Summary

Keep open: the PR targets a real source-reproducible Codex app-server timeout problem, but the head patch still lets some app-server request families reset attempt progress before they are proven to belong to the active turn, and the external real-behavior proof is still mock-only.

## What This Changes

The PR replaces the fixed post-turn/start Codex app-server attempt timeout with a progress-aware idle watchdog and adds fake-client regressions for progressing turns and non-turn request traffic.

## Best Possible Solution

Keep the progress-aware watchdog direction, but only mark request-response attempt progress after current-turn validation succeeds for every request family, add mismatched-scope regressions, and require redacted live Codex app-server proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main accepts a turn and then enforces a fixed post-start timeout while current-turn notifications update separate activity state. I did not run a live Codex app-server session in this read-only review.

## Solution Assessment

No, not yet: the watchdog approach is the right narrow fix, but request-response progress is still marked before active thread/turn validation for some request families. The safer fix is to move or derive attemptProgress from successful current-turn validation and add the matching regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate request progress on current-turn validation:** `extensions/codex/src/app-server/run-attempt.ts:1368`
  - body: This sets `requestCountsAsTurnActivity = true` for elicitation, user-input, and approval requests before the helpers verify that params match the active thread/turn. Those helpers return `undefined` for missing or mismatched scope, but the `finally` block still records `attemptProgress: true`, so a stray request with one of these method names can keep a dead accepted turn alive. Move the progress flag to after successful current-turn validation.
  - confidence: 0.86
