## Summary

Keep this feature request open. Current main has adjacent Web Push, `nodes notify`, `tasks notify`, `system event`, and `chat.inject` paths, but it still lacks a first-class shell command plus typed admin-scoped Gateway/UI broadcast that renders visible in-app notifications to connected Control UI sessions without an agent turn.

## What This Changes

Requested a shell-facing `openclaw notify <message>` command that broadcasts visible notifications to connected Control UI sessions without routing through an agent turn.

## Best Possible Solution

Define a deliberate core notification contract: a CLI entry point, typed admin-scoped Gateway RPC and broadcast event, Control UI rendering, and docs/tests that explain how it differs from Web Push, `chat.inject`, and `system event`.
