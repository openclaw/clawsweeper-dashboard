## Summary

Keep this issue open. Current main still lacks the requested shell-facing `openclaw notify <message>` command, typed admin-scoped Gateway notification RPC/event, and Control UI visible-notification handler; the adjacent Web Push, `chat.inject`, `system-event`, sessions broadcast, and Notifications-tab work do not cover this no-agent-turn UI broadcast path.

## What This Changes

Request a shell-facing `openclaw notify <message>` command that sends a visible in-app notification to all connected Control UI sessions via an admin-scoped Gateway broadcast without starting an agent turn.

## Best Possible Solution

Define an intentional core notification contract: an admin-scoped Gateway RPC plus allowlisted broadcast event, a shell entry point, Control UI rendering, and docs/tests for authorization, delivery semantics, abuse controls, and its relationship to Web Push, `chat.inject`, and `system-event`.

## Solution Assessment
