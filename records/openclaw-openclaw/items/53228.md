## Summary

Keep this feature request open. Current main still has adjacent Web Push, node-local notification, task notification policy, `system event`, and `chat.inject` paths, but it still lacks a shell-facing `openclaw notify <message>` command plus a typed admin-scoped Gateway/UI notification broadcast that renders a visible in-app bubble to connected Control UI sessions without an agent turn.

## What This Changes

Request an `openclaw notify <message>` style CLI path that pushes visible notifications to all connected UI sessions through an admin-scoped broadcast without entering the agent queue.

## Best Possible Solution

Define a deliberate core notification contract: CLI entry point, typed admin-scoped Gateway RPC and broadcast event, Control UI rendering, and docs/tests covering auth, delivery, and overlap with Web Push, `chat.inject`, and `system event`.
