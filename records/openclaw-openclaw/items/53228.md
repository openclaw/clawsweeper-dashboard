## Summary

Keep open. Current main and the latest release still have adjacent Web Push, node-local notification, task notify policy, `system event`, and `chat.inject` paths, but not the requested shell-facing `openclaw notify <message>` plus typed admin-scoped Gateway/UI notification contract for connected Control UI sessions.

## What This Changes

Add an `openclaw notify <message>` CLI command that pushes a visible notification bubble to connected Control UI sessions through an admin-scoped Gateway broadcast without starting an agent turn.

## Best Possible Solution

Define a deliberate core notification contract: a typed admin-scoped Gateway RPC/event, a top-level CLI entry point, explicit broadcast guards, Control UI rendering, docs, and tests, while coordinating the toast/UI layer with https://github.com/openclaw/openclaw/issues/20237.
