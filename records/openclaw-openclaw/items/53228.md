## Summary

Keep this feature request open. Current main has adjacent Web Push, node-local notification, task notification policy, `system event`, and `chat.inject` surfaces, but it still lacks a first-class `openclaw notify <message>` command plus a typed admin-scoped Gateway event/RPC and Control UI handler for visible connected-session notifications.

## What This Changes

The issue requests a shell-facing `openclaw notify <message>` command that broadcasts visible in-app notifications to connected UI sessions without routing through an agent turn.

## Best Possible Solution

Define a deliberate core notification contract: CLI entry point, typed admin-scoped Gateway RPC/event, broadcast guard, Control UI rendering, and docs/tests explaining how it differs from Web Push, `chat.inject`, and `system event`.
