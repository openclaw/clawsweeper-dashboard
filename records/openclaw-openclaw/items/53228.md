## Summary

Keep this feature request open. Current main still has adjacent Web Push, `chat.inject`, `system event`, task/node notification commands, and notification-settings UI pieces, but no shell-facing `openclaw notify <message>` command or typed admin-scoped Gateway/UI notification broadcast path that reaches connected Control UI sessions without an agent turn.

## What This Changes

Request a new `openclaw notify <message>` shell command that sends visible notifications to connected UI sessions through an admin-scoped Gateway broadcast without routing through the agent queue.

## Best Possible Solution

Define a core notification contract only after maintainer API/UX review: a typed admin-scoped Gateway RPC and allowlisted broadcast event, a CLI entry point, Control UI rendering, and docs/tests for auth, delivery semantics, rate limiting or abuse controls, and how it differs from Web Push, `chat.inject`, and `system event`.
