## Summary

Keep this issue open. Current main still lacks the requested shell-facing `openclaw notify <message>` command, typed admin-scoped Gateway notification RPC/event, and Control UI visible notification handling; the related Web Push, WebUI notification, sessions broadcast, and Notifications-tab items are adjacent but do not cover this no-agent-turn UI broadcast path.

## What This Changes

Add an `openclaw notify <message>` CLI path that broadcasts visible notifications to connected UI sessions without creating an agent turn.

## Best Possible Solution

Define an intentional notification contract: an admin-scoped Gateway RPC plus allowlisted broadcast event, a shell entry point such as `openclaw notify <message>`, Control UI rendering, and docs/tests for authorization, delivery semantics, abuse controls, and its relationship to Web Push and existing chat/system-event surfaces.
