## Summary

Keep this issue open. Current main still has adjacent notification paths, including Web Push, node-local notifications, system events, and chat injection, but it does not provide a first-class shell-facing `openclaw notify <message>` command, a typed admin-scoped Gateway notification RPC/event, or Control UI rendering for visible connected-session notification bubbles. The related #20237, #47486, #74715, and merged #44590 do not fully supersede this request.

## What This Changes

Add an `openclaw notify <message>` shell command that broadcasts visible notification bubbles to all connected UI sessions through an admin-scoped Gateway path without starting an agent turn.

## Best Possible Solution

Define an intentional core notification contract: an admin-scoped Gateway RPC plus allowlisted broadcast event, a shell entry point such as `openclaw notify <message>`, Control UI rendering for connected-session notifications, and docs/tests that spell out authorization, delivery semantics, and its relationship to the existing chat, system-event, Web Push, and session-broadcast surfaces.
