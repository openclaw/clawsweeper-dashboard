## Summary

Keep open. Current main still has adjacent Web Push subscription/test plumbing and a Notifications settings tab, but it lacks the requested shell-facing `openclaw notify <message>` command, typed admin-scoped Gateway notification RPC/event, and Control UI handler for visible connected-session notifications.

## What This Changes

Requested feature: add `openclaw notify <message>` or an equivalent shell-facing command that pushes visible notifications to connected Control UI sessions via an admin-scoped broadcast path.

## Best Possible Solution

Define one intentional core notification contract: an additive CLI entry point, typed admin-scoped Gateway RPC and broadcast event, explicit broadcast guards, Control UI rendering, tests, and docs that explain how it differs from Web Push, `chat.inject`, and `system event`.
