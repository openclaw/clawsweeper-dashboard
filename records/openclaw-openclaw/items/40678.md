## Summary

Keep open: current main still has no opt-in TUI observe or mirror mode, and this remains the canonical tracker after broader duplicates were closed in favor of it. The narrower same-session live-tail problem is already tracked by https://github.com/openclaw/openclaw/issues/38829, https://github.com/openclaw/openclaw/issues/45388, and https://github.com/openclaw/openclaw/pull/73402, but the broader cross-channel visibility feature still needs maintainer product review.

## What This Changes

Requests an opt-in TUI observer or mirror mode that can show iMessage, Telegram, cron, and sub-agent activity across channels without reverting isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace the separate same-session live transcript subscription fix, then design an explicit opt-in TUI observer mode with clear identity, authorization, privacy, read-only, and delivery boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe or mirror mode. Source inspection gives high confidence current main lacks that mode, while the narrower same-session live-tail symptom is source-reproducible and tracked separately.

## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, privacy, and delivery boundary for cross-channel observation. The narrow same-session live-tail symptom should use the existing sessions.messages.subscribe and session.message gateway contract rather than widening generic chat fanout.
