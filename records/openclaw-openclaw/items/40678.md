## Summary

Keep open: current main still lacks an opt-in TUI observe or mirror mode, and the narrower same-session live-tail problem is tracked separately by https://github.com/openclaw/openclaw/issues/45388 and https://github.com/openclaw/openclaw/pull/73402. ## What This Changes

Request an opt-in TUI observer or mirror mode that shows iMessage, Telegram, cron, and other channel activity without reverting isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace the scoped same-session live-tail fix, then design an explicit opt-in TUI observer mode with clear identity, authorization, privacy, read-only, and delivery boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe or mirror mode. ## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, privacy, and delivery boundary for cross-channel observation. The same-session live-tail symptom should use the existing sessions.messages.subscribe and session.message gateway contract rather than broadening generic chat fanout.
