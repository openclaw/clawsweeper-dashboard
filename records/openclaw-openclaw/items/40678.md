## Summary

Keep open: current main and v2026.6.10 still lack an opt-in TUI observe/mirror mode, and the open related PR only targets the narrower active-session live-tail bug. This remains a product and security decision because cross-channel visibility can expose transcripts outside the selected session.

## What This Changes

Request an opt-in TUI observe/mirror mode that shows activity from iMessage, Telegram, cron/sub-agent, and other channel sessions without reverting isolated TUI `/new` sessions.

## Best Possible Solution

Keep this as the canonical product/security design issue for default-off TUI observe/mirror behavior, while reviewing the narrow active-session live-tail PR separately and preserving isolated `/new` sessions by default.

## Reproduction Assessment

Yes, at source level for the narrower live-tail gap: Gateway emits `session.message` and current TUI does not dispatch it. The broader observe/mirror behavior is a feature request rather than a runnable current-main bug reproduction.

## Solution Assessment

Unclear for the full feature until maintainers choose the visibility policy. The narrow active-session fix belongs in the linked live-tail PR, while cross-channel observe/mirror needs a default-off, authorization-aware design.
