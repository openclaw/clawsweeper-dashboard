## Summary

Keep open: current main and v2026.6.10 still do not provide an opt-in TUI observe/mirror mode, and the remaining all-channel visibility behavior needs explicit product and security direction rather than cleanup closure.

## What This Changes

Request an opt-in TUI mode or setting that can observe or mirror iMessage, Telegram, cron, and other channel activity routed to the same agent without reverting isolated TUI sessions.

## Best Possible Solution

Define a default-off read-only observe or mirror policy for TUI visibility with explicit authorization boundaries, and handle the narrower active-session live-tail bug through the linked focused PR.

## Reproduction Assessment

Yes for the narrower live-tail overlap at source level: Gateway emits session.message, while current TUI subscribes only to broad session events and does not dispatch session.message. The broader all-channel observe/mirror request is a feature request, not a runnable current-main bug reproduction.

## Solution Assessment

Unclear as a full feature: active-session live tailing should consume the existing session.message path, but all-channel observe/mirror needs a maintainer-approved visibility and security policy before adding a flag or config surface.
