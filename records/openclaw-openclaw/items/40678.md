## Summary

Keep open: current main and v2026.6.8 still do not provide an opt-in TUI observe/mirror mode, and the remaining broad behavior needs product and security direction rather than cleanup closure.

## What This Changes

Request an opt-in TUI mode, flag, or config setting to observe or mirror activity from iMessage, Telegram, cron, and other channel sessions without reverting per-TUI session isolation.

## Best Possible Solution

Define an explicit, default-off observe/mirror policy for TUI visibility, keep active-session live tailing on the existing session.message path, and require security review before any cross-channel transcript fanout ships.

## Reproduction Assessment

Yes, at source level: current main exposes Gateway session.message subscriptions, but the TUI does not subscribe to or dispatch that event and has no observe/mirror option. I did not run a live terminal reproduction in this read-only review.

## Solution Assessment

Unclear as a full feature: the narrow active-session live-tail fix should use sessions.messages.subscribe, but full cross-channel observe/mirror behavior needs maintainer product and security policy first.
