## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode, and the TUI is not wired to the Gateway session-event subscription surface that already exists for live transcript updates.

## What This Changes

Request to add opt-in TUI visibility for iMessage, Telegram, and other channel activity while preserving TUI session isolation.

## Best Possible Solution

Keep `/new` isolation, add an explicit TUI session-event subscription or observer mode, and gate any visible cross-channel delivery mirroring behind an opt-in privacy policy.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-reproducible but not live-reproduced: current main has no TUI observe/mirror option and does not subscribe to `session.message`, while the May 4 Telegram/TUI comment gives a concrete manual verification path.

## Solution Assessment

Is this the best way to solve the issue? The opt-in observe direction is safer than reverting `/new` isolation, but maintainers should first choose the read-only UX, CLI/config surface, and visible cross-channel mirroring policy.
