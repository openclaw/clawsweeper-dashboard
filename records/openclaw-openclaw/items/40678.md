## Summary

Keep open: current main and v2026.6.6 still do not provide an opt-in TUI observe/mirror mode, and the TUI still does not consume live `session.message` transcript events. The narrower active-session live-update bug is already tracked by open issues and an open maintainer PR, while the broader cross-channel observer mode needs an explicit product and security boundary.

## What This Changes

Requested an opt-in TUI mode or setting that mirrors or observes cross-channel activity from iMessage, Telegram, cron, and TUI sessions without reverting TUI session isolation.

## Best Possible Solution

Keep this as the canonical product tracker for a permission-aware TUI observe mode, and resolve active-session live updates through the existing session-scoped subscription PR or an equivalent replacement.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 have no observe/mirror flag or config, and the TUI does not consume `session.message` transcript events. I did not run a live multi-channel Gateway/TUI reproduction in this read-only review.

## Solution Assessment

Unclear for the broad request: active-session live updates have an existing narrow subscription path, but all-channel observation needs a maintainer decision on identity, authorization, read-only versus interactive behavior, and default-off UX.
