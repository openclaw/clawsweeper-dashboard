## Summary

Keep open: current main still lacks an opt-in cross-channel TUI observe/mirror mode, and the narrower same-session live-tail symptom is tracked by https://github.com/openclaw/openclaw/issues/45388 and https://github.com/openclaw/openclaw/pull/73402 rather than shipped.

## What This Changes

Request an opt-in TUI mode or setting that mirrors or observes activity from iMessage, Telegram, and other channels routed to the same agent.

## Best Possible Solution

Keep isolated TUI sessions, land or replace https://github.com/openclaw/openclaw/pull/73402 for same-session live transcript updates, then design an explicit opt-in cross-channel observer mode with clear identity and privacy limits.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe/mirror mode. Source and docs inspection give high confidence that current main lacks that mode, while the narrower same-session live-tail symptom is source-reproducible and tracked separately.

## Solution Assessment

Unclear for the broad request until maintainers choose the UX, identity, authorization, and privacy boundary. The narrow same-session live-tail symptom should use the existing `sessions.messages.subscribe` and `session.message` gateway contract rather than widening generic chat fanout.
