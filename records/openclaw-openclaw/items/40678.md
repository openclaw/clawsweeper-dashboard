## Summary

Keep open: current main and v2026.6.8 still do not provide an opt-in TUI observe/mirror mode, and the TUI still lacks the active-session `session.message` subscription/handler needed for the narrower live-tail part. The narrower live transcript bug is covered by related open issues, but the broader cross-channel visibility policy remains unresolved.

## What This Changes

Request an opt-in TUI mode or setting that mirrors or observes iMessage, Telegram, cron, and other channel activity for the same agent without reverting TUI session isolation.

## Best Possible Solution

Define an explicit opt-in observe/mirror policy for TUI visibility, fix active-session live tailing through the existing Gateway `session.message` primitive, and require security review before any cross-channel transcript fanout ships.

## Reproduction Assessment

Yes, at source level for the narrow live-tail gap: Gateway emits scoped `session.message` transcript updates, while current main TUI does not subscribe to or dispatch that event. The broader observe/mirror request is a missing feature rather than a runnable failing test path.

## Solution Assessment

Unclear for the full feature: the existing Gateway `session.message` path is the best narrow primitive for active-session live updates, but cross-channel observe/mirror behavior needs a maintainer product and security decision before implementation.
