## Summary

Current main still limits the TUI to one selected session, so the requested opt-in cross-channel observer remains necessary. The related exact-session live-streaming defect is narrower and does not replace this canonical product request; maintainers must define the observer’s privacy, authorization, scope, and interaction semantics before implementation.

## What This Changes

Add an opt-in TUI observer view for activity across channel and agent sessions while preserving isolated interactive sessions.

## Best Possible Solution

Design a read-only observer with explicit agent and session filters using existing operator-read authorization, while keeping interactive TUI turns and outbound channel delivery bound to one selected session.

## Reproduction Assessment

Yes for the central feature gap: current-main source clearly filters the TUI to one selected session and provides no multi-session observer view. The narrower same-session live-update workflow is separately tracked at https://github.com/openclaw/openclaw/issues/45388.

## Solution Assessment

Unclear. Read-only observation is the safest direction, but a global mirror flag or interactive mode is not the best solution until maintainers define agent scope, authorization, redaction, history depth, and delivery semantics.
