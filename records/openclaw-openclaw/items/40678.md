## Summary

Keep open: current main and the latest release still do not provide an opt-in TUI observe/mirror mode, and the existing narrow live-session fix candidate only addresses active-session transcript updates, not the broader cross-channel visibility policy.

## What This Changes

The issue requests an opt-in TUI observe or mirror mode that shows iMessage, Telegram, cron, and other channel activity while preserving isolated TUI sessions.

## Best Possible Solution

Define an explicit, opt-in observe/mirror policy for TUI visibility, keep the narrow active-session live-tail fix on the existing Gateway session.message path, and require security review before any cross-channel transcript fanout ships.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose Gateway session.message subscriptions, but the TUI has no subscription API and does not route session.message events. I did not run a live terminal reproduction in this read-only review.

## Solution Assessment

Unclear as a full feature: using Gateway session.message is the best narrow fix for active-session live tailing, but full cross-channel observe/mirror behavior needs a maintainer product and security decision before implementation.
