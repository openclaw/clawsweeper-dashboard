## Summary

Keep open: current main still lacks an opt-in cross-channel TUI observe or mirror mode, while the narrower same-session live-tail symptom is already tracked by an open issue and maintainer PR rather than shipped.

## What This Changes

Request an opt-in TUI observe or mirror mode that shows iMessage, Telegram, and other channel activity without undoing isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace https://github.com/openclaw/openclaw/pull/73402 for same-session live transcript updates, then design an explicit opt-in observer mode with clear identity and privacy limits.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe/mirror mode. Source inspection gives high confidence current main lacks that mode, while the narrower same-session live-tail symptom is source-reproducible and tracked separately.

## Solution Assessment

Unclear for the broad request until maintainers choose the UX, identity, authorization, and privacy boundary. The narrow same-session live-tail symptom should use the existing `sessions.messages.subscribe` and `session.message` gateway contract rather than widening generic chat fanout.
