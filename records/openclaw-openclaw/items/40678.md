## Summary

Keep open: current main and the latest release still do not provide an opt-in TUI observe or mirror mode for cross-channel activity. The active related work only covers the narrower same-session live transcript gap, so this broader feature needs explicit product and security boundaries before implementation.

## What This Changes

The issue requests an opt-in TUI mode or setting to observe or mirror iMessage, Telegram, cron, and other channel activity for the same agent without reverting isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace the separate same-session live transcript subscription fix, then design an explicit opt-in TUI observer mode with clear identity, authorization, privacy, read-only, and delivery boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe or mirror mode. Source and release checks give high confidence current main lacks that mode, while the narrower same-session live-tail symptom is tracked separately.

## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, privacy, and delivery boundary for cross-channel observation. The narrow same-session live-tail issue should use the existing sessions.messages.subscribe and session.message contract rather than broadening generic chat fanout.
