## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode and still does not consume the Gateway live transcript event surface in the TUI, while related reports and an open PR show the narrower same-session live-update bug is still active work rather than shipped behavior.

## What This Changes

The issue requests an opt-in TUI observe or mirror mode for activity from iMessage, Telegram, and other channels routed to the same agent.

## Best Possible Solution

Preserve /new isolation, land or replace the narrow TUI live-transcript subscription fix, and separately design an explicit opt-in observe/mirror policy with clear identity and privacy limits.

## Reproduction Assessment

No live reproduction was run. Source inspection gives a high-confidence current-main path: TUI has no observe/mirror option and no session.message subscription or handler, while the May 4 comment provides a concrete Telegram/TUI same-session repro.

## Solution Assessment

Unclear for the broad request until maintainers choose the UX and privacy boundary. The narrow same-session live-update piece should reuse the existing sessions.messages.subscribe/session.message contract, while broader channel mirroring needs explicit opt-in policy.
