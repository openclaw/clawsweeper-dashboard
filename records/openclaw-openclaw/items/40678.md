## Summary

Keep open: current main still lacks an opt-in TUI cross-channel observe or mirror mode, and the active related fix path only covers the narrower same-session live transcript gap. The request fits OpenClaw’s channel and CLI ergonomics direction, but it needs an explicit product/security boundary before implementation.

## What This Changes

Request an opt-in TUI flag, config option, or observe mode that shows activity from iMessage, Telegram, cron jobs, and other channels without reverting isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace the separate same-session live transcript subscription fix, then design an explicit opt-in TUI observer mode with clear identity, authorization, privacy, read-only, and delivery boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe or mirror mode. Source and docs checks give high confidence current main lacks that mode, while the narrower same-session live-tail symptom is tracked separately.

## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, privacy, and delivery boundary for cross-channel observation. The narrow same-session live-tail issue should use the existing sessions.messages.subscribe and session.message contract rather than broadening generic chat fanout.
