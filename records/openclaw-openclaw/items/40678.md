## Summary

Keep open: current main still has a session-scoped TUI with isolated `/new` sessions and no opt-in observe/mirror mode. The broader #41548 is related and open, but it is a same-author paired feature request, so this needs maintainer canonicalization rather than an automatic cleanup close.

## What This Changes

The issue requests an opt-in TUI observe/mirror mode that can show activity from other channels such as iMessage and Telegram without undoing per-TUI session isolation.

## Best Possible Solution

Keep per-TUI `/new` isolation, then design an opt-in read-only TUI activity feed backed by Gateway session subscriptions or a new typed observer feed if the existing protocol is not enough.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that the requested observe/mirror mode is absent because TUI options do not expose it and chat events are filtered to the active session.

## Solution Assessment

Yes in direction: an opt-in observer mode is a better path than reverting `/new` isolation. The exact read-only versus interactive UX and permission model still need maintainer design before implementation.
