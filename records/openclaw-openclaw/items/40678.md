## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode, and the newer same-session Telegram repro points to a live-sync gap rather than something already implemented or safely closeable.

## What This Changes

Add an opt-in TUI mode or config that observes cross-channel activity and keeps explicitly opened channel sessions visibly in sync without undoing `/new` session isolation.

## Best Possible Solution

Keep `/new` isolation, then design an opt-in read-only TUI observer path that subscribes to Gateway session message updates, with a separate explicit policy for visible cross-channel outbound mirroring.

## Reproduction Assessment

Source-reproducible, but not live-reproduced here: current main has no observe/mirror option, the TUI does not subscribe to `session.message`, and session switching only reloads history. The 2026.5.3 Telegram/TUI comment gives a concrete live path for maintainer verification.

## Solution Assessment

Yes in direction: an opt-in observer mode is safer than reverting `/new` isolation. The exact read-only UX, config surface, and cross-channel send-mirroring policy still need maintainer design before implementation.
