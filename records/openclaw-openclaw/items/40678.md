## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode, and the narrower same-session live transcript gap is already tracked by open follow-up work rather than shipped behavior.

## What This Changes

Request an explicit TUI mode or setting that can observe or mirror activity from iMessage, Telegram, and other channel sessions without undoing isolated `/new` TUI sessions.

## Best Possible Solution

Preserve isolated `/new` sessions, land or replace the scoped live transcript subscription fix, then design an explicit opt-in observe/mirror policy with clear identity and privacy limits.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe/mirror mode. Source inspection gives high confidence that current main still lacks that mode, and the narrower same-session live transcript symptom is covered by open related work.

## Solution Assessment

Unclear for the broad request until maintainers choose the UX, identity, and privacy boundary. The narrow same-session live-update part should reuse the existing `sessions.messages.subscribe` and `session.message` gateway contract rather than widening generic chat fanout.
