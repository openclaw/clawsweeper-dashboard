## Summary

Keep open: current main still lacks the requested opt-in cross-channel TUI observe or mirror mode, and the remaining broad behavior needs a product decision even though the narrower same-session live-tail gap is already tracked separately.

## What This Changes

The issue requests an opt-in TUI mode or setting that can show iMessage, Telegram, cron, and other channel activity for the same agent without undoing isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace https://github.com/openclaw/openclaw/pull/73402 for same-session live transcript updates, then design an explicit opt-in observer mode with clear identity, authorization, and privacy limits.

## Reproduction Assessment

## Solution Assessment

Unclear for the broad request until maintainers choose the UX, identity, authorization, and privacy boundary. The narrow same-session live-tail symptom should use the existing `sessions.messages.subscribe` and `session.message` gateway contract rather than widening generic chat fanout.
