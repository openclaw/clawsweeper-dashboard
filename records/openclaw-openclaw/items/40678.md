## Summary

Keep open: current main still lacks an opt-in cross-channel TUI observe/mirror mode, and the same-session live transcript gap is already tracked by a narrower open issue and PR rather than shipped behavior.

## What This Changes

The issue requests an opt-in TUI observe or mirror mode so a terminal view can show iMessage, Telegram, cron, and other channel activity while preserving isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace the scoped same-session transcript subscription fix, then design an explicit opt-in cross-channel observer mode with clear identity and privacy limits.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe/mirror mode. Current source inspection gives high confidence that main still lacks that mode.

## Solution Assessment

Unclear for the broad request until maintainers choose the UX, identity, authorization, and privacy boundary. The narrow same-session live-tail symptom should reuse the existing `sessions.messages.subscribe` and `session.message` gateway contract rather than widening generic chat fanout.
