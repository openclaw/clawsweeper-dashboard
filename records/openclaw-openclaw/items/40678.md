## Summary

Keep open: current main and the latest release still do not provide an opt-in TUI observe or mirror mode for cross-channel activity. The related open work only covers the narrower same-session live transcript gap, while this feature still needs explicit product and security boundaries.

## What This Changes

Add an opt-in `openclaw tui` observer or mirror mode that can show activity from iMessage, Telegram, cron, and other sessions for the same agent without reverting TUI session isolation.

## Best Possible Solution

Keep isolated TUI sessions by default, land or replace the narrower same-session live transcript fix, then design an explicit opt-in TUI observer mode with clear identity, authorization, read-only, privacy, and delivery boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in TUI observe or mirror mode. Source, docs, and release checks give high confidence that current main and `v2026.5.22` lack that mode, while same-session live-tail is tracked separately.

## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, privacy, and delivery boundary for cross-channel observation. The narrower same-session live-tail issue should use the existing `sessions.messages.subscribe` and `session.message` contract rather than broadening generic chat fanout.
