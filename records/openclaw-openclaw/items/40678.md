## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode, and the TUI is not wired to the Gateway `session.message` subscription surface that already powers Control UI same-session refresh. The related WebChat mirroring issue was closed as implemented, but that implementation does not cover the terminal UI path requested here.

## What This Changes

Requested opt-in TUI observe or mirror mode for activity from other channels such as iMessage and Telegram while preserving isolated `/new` TUI sessions.

## Best Possible Solution

Keep `/new` isolation, add TUI support for live transcript/session-event updates, and gate any broader observe or visible channel mirroring behavior behind an explicit maintainer-approved opt-in policy.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection plus the May 4 Telegram/TUI report gives a clear current-main reproduction path: TUI has no observe/mirror option and does not consume `session.message` events even though the Gateway emits them.

## Solution Assessment

Unclear as a single implementation until maintainers choose the UX and privacy boundary. The requested opt-in observe direction is safer than reverting `/new` isolation, but same-session live refresh may be a narrower first fix than full bidirectional channel mirroring.
