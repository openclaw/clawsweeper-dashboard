## Summary

Keep open: current main still has no opt-in TUI observe or mirror mode, and this remains the specific canonical tracker after the broader duplicate was closed in favor of it. The narrower same-session live-tail symptom is separately tracked by https://github.com/openclaw/openclaw/issues/45388 and https://github.com/openclaw/openclaw/pull/73402, but the broader cross-channel visibility feature still needs maintainer product review.

## What This Changes

Request an opt-in TUI observe or mirror mode that shows iMessage, Telegram, cron, sub-agent, and other channel activity without reverting isolated TUI sessions.

## Best Possible Solution

Keep isolated TUI sessions, land or replace https://github.com/openclaw/openclaw/pull/73402 for same-session live transcript updates, then design an explicit opt-in TUI observer mode with clear identity, authorization, privacy, and delivery limits.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the central request is a new opt-in observe or mirror mode. Source inspection gives high confidence current main lacks that mode, while the narrower same-session live-tail symptom is source-reproducible and tracked separately.

## Solution Assessment

Unclear until maintainers choose the UX, identity, authorization, and privacy boundary for cross-channel observation. The narrow same-session live-tail symptom should use the existing sessions.messages.subscribe and session.message gateway contract rather than widening generic chat fanout.
