## Summary

Keep open: current main still lacks an opt-in TUI observe/mirror mode, and the fresh v2026.5.3 same-session report maps to a real source-level gap because the TUI only consumes live `chat` events, not transcript `session.message` events. The related #41548 is closed, so it is not an open canonical tracker for the remaining work.

## What This Changes

Add an opt-in TUI mode or setting that shows cross-channel activity for the same agent without reverting isolated `/new` TUI sessions.

## Best Possible Solution

Keep `/new` session isolation, add TUI support for active-session transcript updates, and design any all-channel observer mode as an explicit opt-in UX with clear read-only or delivery behavior.

## Reproduction Assessment

Partly yes: source inspection shows the requested observe/mirror mode is absent, and the newer same-session live-sync report maps to TUI not subscribing to or handling `session.message`. I did not run a live channel setup in this read-only review.

## Solution Assessment

Yes in direction: an opt-in observer path is safer than reverting `/new` isolation. The narrower active-session live reload could be implemented with existing gateway transcript subscriptions, but all-channel mirroring still needs UX and privacy design.
