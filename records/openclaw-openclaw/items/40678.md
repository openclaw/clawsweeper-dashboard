## Summary

Keep open: current main and v2026.6.5 still do not provide an opt-in TUI observe/mirror mode, and the narrow active-session live-update bug is already tracked by a separate open issue and conflicting maintainer PR. ## What This Changes

Request an opt-in TUI mode or setting that lets a terminal observe or mirror activity from iMessage, Telegram, cron, subagents, and other same-agent channels without reverting isolated TUI sessions.

## Best Possible Solution

Keep this as the canonical product tracker for an explicit, permission-aware TUI observe mode, while resolving active-session live updates through the existing linked TUI subscription work first.
