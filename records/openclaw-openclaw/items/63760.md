## Summary

Keep open. Current main still has no announce-specific model or light-context policy: completion announces are sent through the requester session's normal gateway `agent` run, and the only documented subagent config in this area is spawned-child model selection plus announce timeout.

## What This Changes

The issue requests announce-specific controls to reduce subagent and ACP completion announcement cost by using a cheaper model, lighter context, or both.

## Best Possible Solution

Define an explicit announce-cost policy, then add a documented config/API contract that preserves requester delivery semantics while allowing a cheaper model or bounded context where maintainers decide it is safe.

## Reproduction Assessment

Yes for source-level behavior: current main routes direct completion announces through the requester session's gateway `agent` path without announce-specific provider/model or bootstrap-context overrides. I did not run a live token-cost reproduction.

## Solution Assessment

Unclear. The proposed `announce.model` and `announce.lightContext` options are plausible, but maintainers should first decide how far announce runs may diverge from the requester agent's model, context, and documented voice semantics.
