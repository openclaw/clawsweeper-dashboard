## Summary

Keep open. Current main still routes subagent and ACP completion announce turns through the normal requester-session `agent` path without an announce-specific model or lightweight/no-history context policy, while the existing cheaper-model and `lightContext` controls apply to spawned child runs, plugin runs, or heartbeat runs instead of the relay turn.

## What This Changes

Request adds cost controls for subagent/ACP completion announce turns so they can use a cheaper model and/or lightweight context instead of the parent session's full model and history.

## Best Possible Solution

Add a schema-backed announce policy under subagent defaults and per-agent inheritance, then wire direct, queued, and nested announce calls to use authorized model overrides and optional lightweight/no-history context without changing delivery routing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to follow any completion announce path to the gateway `agent` call and observe that no model or context override is passed; the gateway and runner then inherit the requester session model and loaded history.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix when implemented through existing config, model-selection, and authorization machinery; hardcoding a cheaper model or bypassing the announce agent turn would be less safe.
