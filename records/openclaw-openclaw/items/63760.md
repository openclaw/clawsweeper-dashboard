## Summary

Keep open. Current main still routes completion announces through the requester session's normal gateway `agent` run without an announce-specific model or lightweight bootstrap-context policy, so the reported cost-control gap remains; solving it requires maintainer API/config and delivery-semantics judgment.

## What This Changes

The issue requests lower-cost completion announces for subagent and ACP runs by avoiding parent-model cold-cache embedded runs.

## Best Possible Solution

Define a documented announce-cost policy that preserves requester delivery semantics while allowing a maintainer-approved cheaper model and/or bounded context for completion announces.
