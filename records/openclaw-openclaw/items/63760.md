## Summary

Keep open: current main has announce timeout, routing, and transcript-hygiene work, but it does not provide an announce-specific model override or lightweight announce context policy. Related open reports cover adjacent model-selection and context-quality bugs, but they do not supersede this cost/config request.

## What This Changes

Request cheaper and/or lighter subagent/ACP completion announce runs, such as an `announce.model` override and an `announce.lightContext` mode.

## Best Possible Solution

Define a typed announce delivery policy that can choose an economical model and bounded context while preserving requester voice, routing, auth/profile rules, and the existing channel-model bug tracked separately.

## Reproduction Assessment

Yes for the current source behavior: direct announce delivery enters Gateway `agent` without model or light-context overrides, and config only exposes `announceTimeoutMs`. I did not run live billing or cache telemetry proof.

## Solution Assessment

Unclear until maintainers choose policy. The proposed knobs are plausible, but a safe implementation needs to reconcile cheaper model selection with existing channel-model precedence and the bounded-context needs discussed in https://github.com/openclaw/openclaw/issues/74286.
