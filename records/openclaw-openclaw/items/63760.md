## Summary

Keep open. Current main still sends subagent announce relays as ordinary gateway `agent` calls without announce-specific provider/model or lightweight-context policy, while the config schema only exposes child-run subagent defaults plus announce timeout.

## What This Changes

Request an announce-specific model/light-context policy so subagent/ACP completion relays avoid waking the parent model with full cold context.

## Best Possible Solution

Add an additive, schema-backed announce policy that reuses existing gateway model and bootstrap-context override fields for all completion relay paths without changing child-run defaults.

## Reproduction Assessment

Yes, at source level. The current announce call sites can be traced to gateway `agent` requests that omit model/context overrides, and the strict config schema has no announce-specific controls.

## Solution Assessment

Yes. An explicit announce policy wired through the existing gateway override contract is narrower and more maintainable than hardcoding a cheaper model, disabling announces, or changing the default child-run model.
