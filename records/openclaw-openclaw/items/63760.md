## Summary

Keep open. Current main still lacks announce-specific model or no-history/light-context controls for subagent and ACP completion relay turns; the existing cheaper-model and light-context knobs apply to spawned child runs, plugin subagent calls, or heartbeat paths rather than the requester-session announce wake.

## What This Changes

Requests configurable cheaper-model and lightweight/no-history context for subagent and ACP completion announce relays to reduce cold-cache token cost.

## Best Possible Solution

Add a schema-backed announce policy for completion relays that reuses authorized model selection and explicit context-reduction semantics while preserving existing delivery routing and transcript behavior.

## Reproduction Assessment

Yes, at source level. Following queued, direct, and nested completion announce paths on current main shows normal requester/child gateway `agent` calls with no announce-specific model or context override.

## Solution Assessment

Yes on direction, but the exact config contract needs maintainer review. A first-class announce policy is safer than hardcoding a cheap model, disabling completion delivery, or reusing child-run `lightContext` as if it removed requester transcript history.
