## Summary

Keep open. Current main still routes completion announces through the requester session's normal `agent` run without an announce-specific model or lightweight context policy, so the reported cost-control gap remains real but requires maintainer API/product judgment.

## What This Changes

Request configurable cheaper or lighter completion announce runs for subagent and ACP completions to avoid parent-model cold-cache cost.

## Best Possible Solution

Define a documented announce-cost policy that preserves requester delivery semantics while allowing a maintainer-approved cheaper model or bounded context for completion announces.

## Reproduction Assessment

Yes for source-level behavior: current main routes direct completion announces through the requester session's normal gateway `agent` path without announce-specific provider/model or bootstrap-context overrides. I did not run a live token-cost reproduction.

## Solution Assessment

Unclear. The proposed `announce.model` and `announce.lightContext` options are plausible, but maintainers should decide how far announce runs may diverge from the requester agent's model, context, and documented voice semantics.
