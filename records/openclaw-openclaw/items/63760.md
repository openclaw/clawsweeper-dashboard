## Summary

Keep open. Current main still lacks announce-specific model or context controls for subagent/ACP completion relay turns; the existing cheaper-model and light-context controls apply to spawned child runs, heartbeat, or cron paths rather than the requester-session announce turn.

## What This Changes

Request an announce-run model override and lightweight/no-history context option for subagent and ACP completion relay turns to reduce token cost.

## Best Possible Solution

Add a schema-backed announce policy for subagent/ACP completion relays, then wire direct, queued, and nested announce calls to use authorized model overrides and optional lightweight/no-history context without changing delivery routing.

## Reproduction Assessment

Yes, at source level. Following any completion announce path shows a normal requester-session `agent` call with no model or context override, so the runner uses the session model/default context behavior.

## Solution Assessment

Yes. The maintainable direction is to add an explicit announce policy through existing config, model-selection, and authorization plumbing; hardcoding a cheaper model or bypassing the announce turn would be less safe.
