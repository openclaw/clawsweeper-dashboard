## Summary

Keep open. Current main still lacks announce-specific model or lightweight-context controls; the matching code paths invoke normal requester-session agent turns and the config only exposes child-run model/context controls plus announce timeout.

## What This Changes

Add announce-specific model and lightweight-context controls for subagent/ACP completion relay turns to reduce token cost.

## Best Possible Solution

Add an additive, schema-backed announce policy and wire all subagent/ACP completion relay paths through existing authorized model and context controls without changing child-run defaults.

## Reproduction Assessment

Yes, at source level. Direct, queued, and descendant announce paths all call gateway `agent` without provider/model or context override fields, while the only existing light-context setting applies to the spawned child run.

## Solution Assessment

Yes. The maintainable direction is an explicit announce policy that uses the existing gateway model/context override contract; hardcoding a cheaper model or disabling announcements would be narrower operationally but not a good product fix.
