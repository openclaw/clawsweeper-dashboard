## Summary

Keep open. Current main still has no announce-specific cheaper model or lightweight-context control; the existing `sessions_spawn.lightContext` only affects spawned native child runs, while completion announces still hand off through the requester agent path and need a maintainer-approved config/API decision.

## What This Changes

Request an announce-specific cheaper model and/or lightweight context for subagent and ACP completion announce runs.

## Best Possible Solution

Define a documented, backward-compatible announce-cost policy that preserves current delivery semantics by default while allowing an opt-in cheaper model and/or bounded context for completion announces.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main lacks announce-specific provider/model and lightweight bootstrap-context controls.

## Solution Assessment

Unclear until maintainers settle the public policy. The proposed `announce.model` and `announce.lightContext` controls are plausible, but the safest solution must define how far announce runs may diverge from requester model, context, and documented parent rewrite semantics.
