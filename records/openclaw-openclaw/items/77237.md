## Summary

Keep open: current main still uses the old managed npm rollback path, while this PR adds a meaningful snapshot-based rollback for blocked plugin installs. ## What This Changes

The PR snapshots and restores managed npm plugin project package files, node_modules, and staged npm-pack archive state when npm-backed plugin install validation rejects a candidate.

## Best Possible Solution

Land a focused rollback fix only after refreshed proof covers the exact current head, including staged npm-pack rollback, and maintainers accept or narrow the full-project snapshot tradeoff.

## Solution Assessment

Unclear as the final best fix: the snapshot approach is an acceptable conservative atomic rollback, but maintainers still need to accept its full node_modules I/O cost or ask for a narrower rollback design.
