## What This Changes

The branch snapshots and restores the managed npm plugin root around npm-backed install failures, adds blocked-install/update regressions, and records the fix in the changelog.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm candidates restore the previous managed npm root instead of uninstalling a working plugin.

## Reproduction Assessment

Yes. Source inspection of current main shows npm mutates the managed root before metadata/security validation and then rolls back by uninstalling/removing state; the PR body also reports the added blocked-update regression failing on main and passing on the branch.

## Solution Assessment

Yes. Snapshotting the managed root before npm mutation is the narrow maintainable fix because npm may change the target package, manifest, lockfile, and hoisted dependencies before OpenClaw can reject the candidate.
