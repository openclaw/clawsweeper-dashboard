## Summary

Keep open for maintainer review and merge. Current main still uses destructive npm-root rollback after npm has mutated the managed plugin root, while this PR adds a focused snapshot restore with regression coverage, a changelog entry, and no blocking review findings. Review target: https://github.com/openclaw/openclaw/pull/77237

## What This Changes

The PR snapshots and restores the managed npm plugin root during rejected installs or updates, expands npm install rollback regressions, and adds a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm installs restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main mutates the managed npm root before validation and rolls back by uninstall/removal instead of restoring prior package metadata and hoisted dependencies. I did not run a live failing scenario because this was a read-only review.

## Solution Assessment

Yes. Snapshotting package.json, package-lock.json, and node_modules before mutation is the narrow reliable fix because npm can change hoisted shared dependencies and lockfile entries in ways that cannot be inferred after rejection.
