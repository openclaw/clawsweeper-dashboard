## Summary

Keep open for normal review/merge: current main still has destructive npm-root rollback, and this PR’s snapshot restore plus amended regressions directly address the source-reproducible install/update bug without a blocking finding.

## What This Changes

The PR adds snapshot-based rollback for the managed npm plugin root, regression tests for blocked npm installs/updates, and a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm install/update candidates restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main mutates package.json and node_modules before validation, then rolls back by uninstall/removal rather than restoring prior root state. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. Snapshotting package.json, package-lock.json, and node_modules before npm mutates the root is the narrow maintainable fix because reconstructing hoisted dependency state after rejection would depend on npm and registry behavior.
