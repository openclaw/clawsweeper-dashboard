## Summary

Keep open for normal maintainer review: current main still has the destructive npm-root rollback path, while the PR adds a focused snapshot rollback with regression coverage and no blocking review findings found.

## What This Changes

The branch snapshots the managed npm plugin root before npm-backed installs and restores package.json, package-lock.json, and node_modules when install or validation rejects a candidate.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm install or update candidates restore the exact previously trusted managed-root state.

## Reproduction Assessment

Yes, source-reproducible: current main updates the managed npm manifest and runs npm before metadata and security validation, then rolls back by uninstall/removal rather than restoring the prior root. I did not run a live install scenario during this read-only review.

## Solution Assessment

Yes. Snapshotting package.json, package-lock.json, and node_modules before npm mutates the managed root is the narrow maintainable fix because hoisted dependency and lockfile changes cannot be reliably inferred after rejection.
