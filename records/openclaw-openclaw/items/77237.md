## Summary

Keep open for maintainer review: the patch appears correct and well-proven, but full managed-root node_modules snapshot/restore on failure paths is an availability and I/O tradeoff maintainers should explicitly accept before merge.

## What This Changes

The PR adds managed npm root snapshot/restore around plugin install/update rollback, preserves managed-root npm config scoping, adds regression coverage, and records a changelog entry.

## Best Possible Solution

Merge the exact snapshot rollback if maintainers accept the availability tradeoff; otherwise require a lighter rollback design that still atomically restores package metadata, lockfile, plugin files, and hoisted dependencies.

## Reproduction Assessment

Yes. Source inspection shows current main mutates the managed npm root before post-install validation and then rolls back by uninstalling/removing instead of restoring the previous root; I did not execute the live script in this read-only pass.

## Solution Assessment

Yes, with a maintainer tradeoff. Exact snapshot restore is the most reliable narrow fix for npm mutations across metadata, lockfile, plugin files, and hoisted dependencies; a lighter design is only preferable if maintainers reject the I/O cost.
