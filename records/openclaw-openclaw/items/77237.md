## Summary

Keep open: the rollback idea is valuable, but the current PR head has blocking installer control-flow and type-contract regressions. The posted terminal proof references an older head than the live PR, so proof should be refreshed after the branch is repaired.

## What This Changes

The PR snapshots and restores the managed npm plugin root when npm-backed validation rejects an install or update, with regression coverage and a changelog entry.

## Best Possible Solution

Keep the snapshot/restore strategy, but wrap the original unconditional npm install, metadata verification, security scan, and rollback calls in a `try/finally` without moving them behind peer-link repair, then either remove or separately justify the unrelated provider/session changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a normal managed npm install or update with no package-local OpenClaw peer repair needed will take the false branch at `installedPackageNeedsOpenClawPeerLinkRepair` and skip npm install entirely in the PR head; the original current-main rollback bug is also supported by the PR body's failing regression check.

## Solution Assessment

No. Snapshot rollback is the right direction, but this patch is not the best current solution because it breaks the ordinary installer control flow and no longer type-checks one rollback call.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Restore the npm install flow outside the peer-link check:** `src/plugins/install.ts:610`
  - body: After the snapshot is created, the PR head only runs `upsertManagedNpmRootDependency`, `npm install`, metadata verification, and the installed-package scan inside `if (installedPackageNeedsOpenClawPeerLinkRepair(installRoot))`. For a normal fresh install or update, that predicate is false, so `installPluginFromManagedNpmRoot` falls through after cleanup and resolves without installing anything or returning an `InstallPluginResult`.
  - confidence: 0.97

  - body: This new rollback call does not match `rollbackManagedNpmPluginInstall`: it passes `dependencySpec` and `managedOverrides`, but omits required `targetDir`, `timeoutMs`, and `logger`. Strict TypeScript should reject the branch before the rollback path can run.
  - confidence: 0.96
