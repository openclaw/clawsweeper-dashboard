## Summary

## What This Changes

The PR snapshots the managed npm plugin root before npm-backed installs, restores package metadata and node_modules when late validation rejects a candidate, preserves managed-root npm config scope, and adds installer regression coverage plus a changelog entry.

## Best Possible Solution

Merge the exact snapshot rollback if owners accept the disk and I/O cost; otherwise require a lighter rollback design that still atomically restores package metadata, lockfile state, node_modules, and hoisted dependencies.

## Reproduction Assessment

Yes. Source inspection shows current main rolls back late validation failures by uninstalling/removing the managed package after npm has already mutated the root, and the PR adds a focused blocked-update regression for that path; I did not execute the failing path in this read-only review.

## Solution Assessment

Yes, with the stated tradeoff. A pre-mutation snapshot is the narrowest reliable way shown here to restore npm-mutated package metadata, lockfile state, installed package files, and hoisted dependencies without depending on another npm or registry operation during rollback.
