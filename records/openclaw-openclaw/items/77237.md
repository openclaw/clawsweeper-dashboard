## Summary

Keep open: current main still uses destructive npm install rollback, while this PR adds the focused snapshot restore path, regression coverage, and changelog entry with no blocking review findings found.

## What This Changes

The PR snapshots the managed npm plugin root before npm-backed install/update mutation and restores package.json, package-lock.json, and node_modules on rejected candidates, with npm rollback regressions and a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after normal maintainer review so rejected npm plugin installs restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main mutates the managed npm root before validation and rolls back by uninstall/removal instead of restoring prior metadata and hoisted dependencies. The PR adds a focused blocked-update regression that exercises the same failure mode.

## Solution Assessment

Yes. Snapshotting and restoring the pre-install managed npm root is the narrow maintainable fix because npm can mutate package metadata, lockfile entries, plugin files, and hoisted dependencies in ways that are not reliably inferable after rejection.
