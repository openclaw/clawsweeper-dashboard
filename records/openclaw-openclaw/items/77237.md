## Summary

Keep this PR open: the reported rollback gap is still present on current main, the patch is focused and has no blocking code-review finding, but merge still needs green required checks and an explicit maintainer call on the full managed-root snapshot cost.

## What This Changes

The PR snapshots and restores managed npm plugin root state on blocked installs or updates, then adds regression coverage and a changelog entry for that rollback behavior.

## Best Possible Solution

Land the scoped snapshot-based rollback after required checks pass and maintainers accept the temporary disk and I/O cost as the price of exact managed npm root restoration.

## Solution Assessment

Yes, with the disk/I/O tradeoff called out. Restoring the exact previous managed npm root is the narrowest reliable solution when npm may mutate package files, lockfiles, plugin directories, hidden lock state, and hoisted dependencies before validation rejects the candidate.
