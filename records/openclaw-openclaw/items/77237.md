## What This Changes

The PR snapshots and restores the managed npm root around rejected plugin installs or updates, updates installer regression tests, and adds a changelog entry.

## Best Possible Solution

## Solution Assessment

Yes. Capturing the exact local root before npm mutation and restoring it on later validation failure is the narrow maintainable fix; reconstructing from npm state during an error path would be less reliable.
