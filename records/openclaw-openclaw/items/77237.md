## Summary

Keep open for normal maintainer review/merge: current head appears to address the earlier ClawSweeper blockers, the rollback bug is source-reproducible on main, and I found no blocking correctness finding in the repaired diff. The main remaining merge decision is whether maintainers accept the temporary disk/I/O cost of exact managed-root snapshots during npm install/update failure paths.

## What This Changes

The PR snapshots the managed npm plugin root and restores `package.json`, `package-lock.json`, and `node_modules` when a later validation step rejects an npm-backed plugin install or update.

## Best Possible Solution

Merge the exact snapshot rollback design if maintainers accept the failure-path I/O cost, because preserving the previous local root is more reliable than reconstructing npm mutations after validation rejects a candidate.

## Reproduction Assessment

Yes. Source inspection shows current main mutates the managed npm root before late validation and then rolls back with uninstall/remove rather than an exact previous-root restore; the PR adds a focused test for the blocked-update path.

## Solution Assessment

Yes. The snapshot approach is the most maintainable fix for exact rollback because npm can mutate package metadata, lockfiles, the target package, and hoisted dependencies in one install; a lighter reconstruction would depend on inferring npm's side effects.
