## What This Changes

The PR snapshots and restores the managed npm plugin root when validation rejects an npm-backed plugin install or update after npm has mutated the root.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review and green checks, ideally trimming unrelated formatting churn if maintainers want a smaller branch.

## Reproduction Assessment

Yes, from source inspection and the PR's reported focused regression: current main mutates the managed npm root before validation and rollback removes the target instead of restoring the previous root. I did not run the test because this review is read-only, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes. Snapshotting the managed npm root before npm mutation is the narrow maintainable fix because npm can change the plugin package, manifest, lockfile, and hoisted dependencies before OpenClaw rejects the candidate.
