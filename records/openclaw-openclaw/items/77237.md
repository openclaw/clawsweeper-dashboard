## Summary

Keep open: current main still uses destructive rollback after npm mutates the managed plugin root, and the branch addresses that path with focused code, regression coverage, changelog, and sufficient terminal proof.

## What This Changes

This PR adds snapshot-based rollback for managed npm plugin installs, updates npm install rollback regressions, and adds a changelog entry.

## Best Possible Solution

Land the snapshot rollback after maintainer review and green required checks so rejected npm candidates leave the previous managed npm root intact.

## Reproduction Assessment

Yes. Source inspection shows current main mutates the managed npm root before metadata/security validation and then uses destructive rollback; the PR body also reports the new regression failing on main and passing on the branch.

## Solution Assessment

Yes. Snapshotting the root before npm mutation is the narrow maintainable fix because npm can alter the target package, manifest, lockfile, and hoisted dependencies before OpenClaw rejects the candidate.
