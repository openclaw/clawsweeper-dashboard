## What This Changes

The PR snapshots and restores the managed npm plugin root around npm-backed installs/updates and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm candidates restore the exact previous managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main writes the managed npm root dependency and runs npm before metadata/security validation, then rollback removes/uninstalls rather than restoring the previous root. I did not run the failing scenario because this review was read-only.

## Solution Assessment

Yes. Snapshotting before npm mutation is the narrow maintainable fix because npm may alter package.json, package-lock.json, the plugin directory, and hoisted dependencies in ways that cannot be reliably reconstructed after rejection.
