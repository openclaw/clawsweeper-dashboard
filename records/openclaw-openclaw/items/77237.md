## Summary

Keep open for maintainer review: the PR addresses a source-reproducible rollback bug in current main, includes focused regression coverage and a changelog entry, and I did not find a blocking correctness or security issue.

## What This Changes

The branch adds snapshot/restore rollback for managed npm plugin root state during npm installs, extends npm install regressions, and adds a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm install/update candidates restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main mutates the managed npm root before validation and rolls back by uninstall/removal instead of restoring prior metadata, lockfile, and hoisted dependencies. I did not run a live install scenario during this read-only review, but the code path and regression test are clear.

## Solution Assessment

Yes. Snapshotting package.json, package-lock.json, and node_modules before npm mutates the root is the narrow maintainable fix because reconstructing the prior hoisted tree after rejection would depend on npm and registry behavior.
