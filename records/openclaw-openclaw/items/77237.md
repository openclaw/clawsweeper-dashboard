## Summary

Keep this PR open for normal maintainer review and merge handling. Current main still has a managed npm rollback gap, the PR is a focused installer fix with sufficient real behavior proof, and I did not find a concrete blocking correctness issue in the diff.

## What This Changes

The PR snapshots `package.json`, `package-lock.json`, and `node_modules` before managed npm plugin installs and restores that snapshot when later validation blocks the candidate, with regression coverage and a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after required CI and maintainer gates pass, keeping the change scoped to managed npm install/update rollback paths.

## Reproduction Assessment

Yes, source-level. Current main lets npm mutate the managed root before later validation and then rolls back with uninstall/removal rather than restoring the prior manifest, lockfile, and hoisted dependency state.

## Solution Assessment

Yes. Capturing the previous local root before npm mutation and restoring it on later validation failure is the narrow maintainable fix; reconstructing from npm state during an error path would be less reliable.
