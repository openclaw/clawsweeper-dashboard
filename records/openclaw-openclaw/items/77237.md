## Summary

Keep open for maintainer merge review. The PR appears to fix the managed npm rollback bug with focused coverage and current-head terminal proof; the remaining decision is whether exact root snapshot/restore is an acceptable failure-path I/O tradeoff.

## What This Changes

The PR snapshots the managed npm plugin root before mutation and restores prior package metadata, lockfile, and node_modules state when validation rejects an npm-backed install or update.

## Best Possible Solution

Merge the exact snapshot rollback design if maintainers accept the failure-path I/O cost, because it preserves npm-mutated root state more reliably than trying to reconstruct package metadata, lockfile entries, plugin files, and hoisted dependency changes after rejection.

## Reproduction Assessment

Yes. Source inspection shows current main mutates the managed npm root before late validation and then uses uninstall/remove rollback rather than restoring the prior root image; I did not run local repro commands in this read-only pass.

## Solution Assessment

Yes, assuming maintainers accept the I/O tradeoff. The snapshot approach is the narrowest maintainable fix for exact rollback because npm can mutate metadata, lockfiles, target package files, and hoisted dependencies in one install.
