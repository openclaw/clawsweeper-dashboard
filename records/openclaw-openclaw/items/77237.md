## Summary

Keep open for maintainer review: current main still rolls back rejected npm plugin installs destructively, while this PR adds a focused snapshot restore with regression coverage and no blocking review findings.

## What This Changes

The PR snapshots and restores the managed npm plugin root during blocked installs/updates, updates npm-spec rollback tests, and adds a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm install/update candidates restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main mutates package.json and node_modules before validation, then rolls back by uninstall/removal instead of restoring the previous root. I did not run tests because the checkout had to remain read-only.

## Solution Assessment

Yes. Snapshotting the root before npm mutates it is the narrow maintainable fix because reconstructing hoisted dependency state after rejection would depend on npm and registry behavior.
