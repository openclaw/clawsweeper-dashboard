## What This Changes

The PR snapshots managed npm package files and `node_modules` before plugin install/update mutation, restores them when later validation blocks the candidate, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review and settled checks, keeping the implementation confined to managed npm install/update failure paths.

## Reproduction Assessment

Yes, source-level rather than locally executed. Current main lets npm mutate `package.json`, `package-lock.json`, and `node_modules` before later validation/security failures call a rollback path that lacks a full root snapshot.

## Solution Assessment

Yes. Capturing the previous managed npm root before mutation and restoring it on post-install rejection is the narrow maintainable fix; inferring npm's package, lockfile, and hoist mutations after failure would be more fragile.
