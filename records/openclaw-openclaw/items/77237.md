## What This Changes

The PR snapshots the managed npm plugin root before npm install mutations, restores package.json, package-lock.json, and node_modules on rejected installs or updates, and adds npm rollback regressions plus a changelog entry.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Snapshotting the exact managed npm root before mutation is the narrowest maintainable fix because npm can alter package metadata, lockfiles, and hoisted dependencies in ways that are hard to infer after rejection.
