## Summary

Keep this PR open for maintainer review: the patch appears to fix a real managed npm rollback bug with sufficient terminal proof and focused regression coverage, but merging it still requires accepting the full managed-root snapshot disk/I/O tradeoff.

## What This Changes

The PR adds managed npm root snapshot/restore rollback for blocked plugin installs or updates, preserves managed-root npm config scope, updates installer regression tests, and adds a changelog entry.

## Best Possible Solution

Merge the exact snapshot rollback if maintainers accept the failure-path availability tradeoff; otherwise require a lighter rollback design that still atomically restores package metadata, lockfile, plugin files, and hoisted dependencies.

## Reproduction Assessment

Yes. Source inspection of current main shows npm mutates the managed root and rollback then uninstalls/removes the target instead of restoring the previous root; the PR body also supplies after-fix terminal proof for the same blocked-update path.

## Solution Assessment

Yes. Exact snapshot restore is the narrowest reliable fix for npm mutations across package metadata, lockfile entries, plugin files, and hoisted dependencies; a lighter design is only preferable if maintainers reject the disk/I/O tradeoff.
