## Summary

Keep open: current main still uses the destructive managed-npm rollback path, and this PR adds a conservative snapshot/restore fix with solid terminal proof. The remaining merge question is maintainer ownership of the full `node_modules` snapshot disk/I/O tradeoff before landing.

## What This Changes

The PR snapshots the managed npm plugin root before npm install, restores `package.json`, `package-lock.json`, and `node_modules` after blocked validation, preserves managed-root npm config scope, and adds installer regression coverage plus a changelog entry.

## Best Possible Solution

Land the snapshot/restore rollback only after maintainers accept the managed-root I/O cost, with the direct changelog edit removed and the installer regression proof preserved.

## Reproduction Assessment

## Solution Assessment

The remaining question is whether maintainers accept the full `node_modules` snapshot cost, and the release-owned changelog edit should be removed.
