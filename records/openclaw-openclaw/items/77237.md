## Summary

Keep open: this is a real managed npm rollback fix with sufficient terminal proof, but the patch currently snapshots the root before existing stale `openclaw` peer cleanup and then skips that cleanup when restoring from the snapshot.

## What This Changes

The PR adds managed npm root snapshot/restore rollback around blocked plugin installs, preserves managed-root npm config scoping, adds installer regression tests, and updates the changelog.

## Best Possible Solution

Capture the rollback snapshot after the pre-install stale-peer repair, or rerun the repair/relink step after snapshot restore, while preserving exact rollback for package metadata, lockfile, plugin files, and hoisted dependencies.

## Reproduction Assessment

Yes. Source inspection shows current main mutates the managed npm root before post-install validation and then rolls back destructively; the PR body also supplies after-fix terminal proof for the central blocked-update path.

## Solution Assessment

Not quite. Exact snapshot restore is a strong fix direction, but the snapshot must not capture stale `openclaw` peer state that current main repairs before install.
