## What This Changes

The PR snapshots and restores the managed npm plugin root around npm-backed installs, updates npm-spec rollback tests and changelog, and also includes unrelated oc-path formatting-only changes.

## Best Possible Solution

Keep the snapshot-based rollback in the managed npm installer, trim unrelated formatting churn if maintainers want a narrower branch, and merge only after required checks are green.

## Reproduction Assessment

Yes. Current main's source shows a blocked post-install validation path calls rollback that removes the target package and managed dependency without preserving the previous root; the PR body also supplies a targeted current-main failure check for the blocked update scenario.

## Solution Assessment

Yes. Snapshotting the managed npm root before npm mutates it is the narrowest robust fix for preserving hoisted dependencies, package.json, and package-lock.json after a rejected candidate; reconstructing the old tree through npm would depend on registry/npm behavior during an error path.
