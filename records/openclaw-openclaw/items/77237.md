## What This Changes

The PR adds managed npm root snapshot/restore rollback around npm plugin installs, preserves managed-root npm config scope, and expands installer regression tests plus changelog coverage.

## Best Possible Solution

Merge the exact snapshot rollback if maintainers accept the availability tradeoff; otherwise require a lighter design that still atomically restores package metadata, lockfile, plugin files, and hoisted dependencies.

## Reproduction Assessment

## Solution Assessment

Yes: exact snapshot restore is the narrowest reliable way to undo npm mutations across package metadata, lockfile, plugin files, and hoisted dependencies. A lighter rollback is a maintainer tradeoff only if the full-root I/O cost is unacceptable.
