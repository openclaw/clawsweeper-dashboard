## What This Changes

The PR snapshots and restores the managed npm plugin root around blocked npm-backed installs, preserves managed-root npm config scope, updates rollback regression tests, and adds a changelog entry.

## Best Possible Solution

Merge the exact snapshot rollback if maintainers accept the temporary disk and I/O cost; otherwise require a lighter rollback design that still restores package metadata and hoisted dependencies.

## Reproduction Assessment

Yes. The source path is clear: install a prior managed npm plugin, attempt a blocked update after npm mutates the root, and current main's rollback can remove or rewrite the active root instead of restoring it.

## Solution Assessment

Yes: exact snapshot restore is the narrowest reliable way to undo npm mutations across package metadata, lockfile, plugin files, and hoisted dependencies. A lighter rollback is a maintainer tradeoff only if the full-root I/O cost is unacceptable.
