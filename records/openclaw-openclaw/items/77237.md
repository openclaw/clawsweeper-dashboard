## What This Changes

The PR adds snapshot/restore rollback for managed npm plugin installs, regression coverage, and a changelog entry.

## Best Possible Solution

Land the snapshot-based rollback after maintainer review so rejected npm install/update candidates restore the exact previously trusted managed root state.

## Reproduction Assessment

Yes, source-reproducible: current main mutates package.json and node_modules before validation, then rolls back by uninstall/removal instead of restoring the previous root. I did not run a live reproduction because this review had to keep the checkout read-only.

## Solution Assessment

Yes. Snapshotting the managed npm root before npm mutates it is the narrow maintainable fix because reconstructing hoisted dependency and lockfile state after rejection would depend on npm and registry behavior.
