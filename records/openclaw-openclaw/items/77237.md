## Summary

## What This Changes

The PR snapshots and restores managed npm root package metadata and `node_modules` around npm-backed plugin installs, preserves managed-root npm config scope, adds rollback regression tests, and adds a changelog entry.

## Best Possible Solution

Merge the exact snapshot rollback design if maintainers accept the disk and I/O tradeoff; otherwise require a lighter rollback design that still preserves package metadata and hoisted dependencies.

## Reproduction Assessment

Yes. The source path is clear: install a prior managed npm plugin, attempt a blocked update after npm mutates the root, and current main’s rollback can remove or rewrite the active root instead of restoring it.

## Solution Assessment

Yes, if maintainers accept the cost. Exact snapshot restore is the most reliable way to undo package metadata and hoisted dependency mutations; reconstructing the tree would be lighter but more brittle.
