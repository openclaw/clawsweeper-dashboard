## Summary

Keep open: current main still has a source-backed fresh-session status path where `/new` clears usage totals and `/status` renders the missing context numerator as `?`, while explicit zero already formats correctly.

## What This Changes

The issue requests that `/status` render zero context usage as `0/1.0m` instead of `?/1.0m` for a brand-new session.

## Best Possible Solution

Normalize fresh no-usage session status inputs to an explicit fresh zero and add regression coverage, while preserving `?` for stale or genuinely unknown context usage.

## Reproduction Assessment

Yes at source level: a `/new`-fresh session can have no `totalTokens`, and the shared status formatter renders nullish totals as `?/1.0m`. I did not run a live installed `/new` plus `/status` flow in this read-only pass.

## Solution Assessment

Yes. The best fix is narrow normalization or status handling for fresh zero-usage sessions, not a global formatter fallback, because stale cumulative totals intentionally stay unknown.
