## Summary

Keep open: current main still has a source-visible path where paused, abandoned, or otherwise incomplete subagent lifecycle ends are surfaced as `ok`, allowing the parent handoff to say `completed successfully` with progress text.

## What This Changes

The issue asks OpenClaw to mark interrupted or incomplete subagent runs as failed, timed out, or unknown instead of reporting them as completed successfully.

## Best Possible Solution

Honor runtime completion facts in `agent.wait` and subagent outcome mapping so non-yielded paused, abandoned, blocked, replay-invalid, aborted, or timed-out terminal states cannot be announced as success.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live reproduction was run, but current source provides a credible path: lifecycle `end` can carry paused or abandoned liveness metadata that is later mapped to `ok`.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix is to carry and honor existing runtime liveness, replay, timeout, abort, and yield facts rather than parsing progress-like result text.
