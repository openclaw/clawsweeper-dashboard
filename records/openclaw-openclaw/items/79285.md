Current main should stay open: the same source-level path still exists where a non-yielded paused or abandoned subagent lifecycle end can reach `agent.wait` as `ok`, then be announced as `completed successfully` with the latest assistant progress text.

## What This Changes

The issue reports that interrupted subagent work can be surfaced to the parent as a successful completion with incomplete progress text.

## Best Possible Solution

Carry authoritative runtime completion facts through `agent.wait` and subagent outcome mapping so non-yielded paused, abandoned, replay-invalid, aborted, or timed-out terminal states cannot become successful completion announcements.

## Reproduction Assessment

No live reproduction was run, so this is not a high-confidence runtime reproduction. Current main is source-reproducible: lifecycle metadata can represent incomplete work while the downstream wait and announce mapping still treats `ok` as successful completion.

## Solution Assessment

Yes: the best fix is to honor existing runtime liveness, replay, abort, timeout, and yield metadata rather than guessing from progress-like result text. The repair should preserve yielded `sessions_yield` pauses while making non-yielded incomplete endings fail, time out, or surface as incomplete.
