## What This Changes

Report that subagent completion events can announce `completed successfully` with mid-task output when the child task is still incomplete.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/74448 as the canonical tracker and fix the runtime lifecycle contract so compaction/retry liveness cannot produce a successful parent completion before the resumed child task really ends.

## Reproduction Assessment

Yes, source-level. The production reports match the current code path where lifecycle `end` becomes `ok`, subagent completion freezes current child output, and `ok` is announced as `completed successfully`; no live long-running compaction run was executed in this read-only pass.

## Solution Assessment

Yes. The best cleanup is to close this duplicate and keep the compaction-specific fix focused in the open canonical issue; the implementation should honor runtime liveness/resume facts rather than parse progress-like result text.
