Keep open: current main still has a source-visible path where incomplete subagent lifecycle facts can be mapped to `ok`, then announced to the parent as `completed successfully`. The existing runtime metadata is the right signal to honor; the issue should not be closed as stale, duplicate, or implemented.

The issue reports that interrupted or incomplete subagent work can be surfaced to the parent as a successful completion with only progress text.

## Best Possible Solution

Honor existing runtime completion facts in `agent.wait`, subagent registry, and announce mapping so non-yielded abandoned, blocked, replay-invalid, aborted, or timed-out terminal states cannot be labeled as successful completion.
