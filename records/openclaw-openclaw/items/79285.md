## Summary

Keep open: current main still has a source-visible path where incomplete non-yielded subagent lifecycle endings are carried as `ok` and then announced as `completed successfully`. I did not find a linked closing PR or duplicate that owns the remaining fix.

## What This Changes

The issue requests that interrupted or incomplete subagent sessions not be announced to parent sessions as completed successfully.

## Best Possible Solution

Honor existing lifecycle liveness, replay, abort, and yield facts in `agent.wait` and subagent outcome mapping so non-yielded incomplete terminal states cannot become success.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-reproducible: emit a lifecycle `end` with `livenessState: "abandoned"` or replay-invalid incomplete state and no `aborted`/`yielded`, and current main maps it to `agent.wait` status `ok` and then `completed successfully`.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix is to carry and honor runtime completion facts, not parse progress-looking model text heuristics.
