## Summary

Keep open. Current main still limits agent lifecycle events to start/end/error while usage is persisted and available through other session surfaces; https://github.com/openclaw/openclaw/pull/51317 is closed unmerged, and the related open observability proposals do not settle this lifecycle-stream contract.

## What This Changes

Request to expose per-run token usage, provider/model, duration, and last-call usage to passive WebSocket subscribers through a lifecycle usage event.

## Best Possible Solution

Choose one additive, documented Gateway observability contract and cover all agent run paths with focused protocol, runtime, and WebSocket tests before landing it.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing contract. Source and docs checks show usage is persisted and session-visible, but no lifecycle usage event exists on current main.

## Solution Assessment

Unclear until maintainer API review. A passive usage event is plausible, but maintainers should choose between a lifecycle usage phase, enriched terminal payloads, or a separate documented status event before implementation.
