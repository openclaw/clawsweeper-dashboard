## Summary

Keep open. Current main persists and exposes usage through session metadata, usage RPCs, and session transcript events, but it still does not emit the requested passive WebSocket `lifecycle` usage event; the linked implementation PR at https://github.com/openclaw/openclaw/pull/51317 is closed unmerged, and the Paperclip incident comment is useful real-world support for the feature.

## What This Changes

Request to add an additive WebSocket lifecycle usage event exposing per-run token counts, model/provider, duration, and last-call usage after agent runs complete.

## Best Possible Solution

Keep this open until maintainers choose and document an additive Gateway usage-event contract, then land focused implementation and tests across the run paths that produce `agentMeta.usage`.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing contract. Source inspection gives high confidence that current main persists usage and exposes session usage surfaces but does not emit a `phase: "usage"` lifecycle event.

## Solution Assessment

Unclear until maintainer protocol review. A passive usage event is plausible, but the maintainable fix depends on whether usage belongs in a new lifecycle phase, terminal payload metadata, or another documented event family.
