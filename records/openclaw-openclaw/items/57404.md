## Summary

Keep open. Current main and the latest release still expose run usage through session/RPC surfaces rather than the requested passive WebSocket lifecycle usage event, and the related implementation PRs are closed unmerged or still under separate API review.

## What This Changes

Expose accumulated per-run token, cost, provider, model, duration, and last-call usage metadata to passive WebSocket lifecycle subscribers after agent runs complete.

## Best Possible Solution

Keep this as the focused tracker until maintainers choose an additive Gateway/SDK usage-event contract, then implement it across all run paths that produce agentMeta.usage with docs and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main emits lifecycle terminal events and persists usage metadata, but does not emit the requested lifecycle usage event.

## Solution Assessment

Unclear until maintainer protocol review. A passive usage event is plausible, but maintainers should choose whether usage belongs in a lifecycle phase, a separate run-status event, agent.wait metadata, or another SDK-normalized event family.
