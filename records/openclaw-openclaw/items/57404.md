## Summary

Keep open. Current main and the latest release still persist usage metadata and expose adjacent session snapshots, but they do not provide the requested passive lifecycle `phase: "usage"` WebSocket event; the closed unmerged implementation at https://github.com/openclaw/openclaw/pull/51317 and the Paperclip incident remain useful input for a maintainer protocol decision.

## What This Changes

The issue requests an additive WebSocket lifecycle event carrying per-run token usage, last-call usage, provider/model identifiers, and duration for passive observers.

## Best Possible Solution

Keep this as the focused tracker until maintainers choose and document an additive Gateway/SDK usage-event contract, then land a bounded implementation with regression coverage for each run path that produces `agentMeta.usage`.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main documents and emits lifecycle `start`/`end`/`error` while usage is persisted through session metadata instead.

## Solution Assessment

Unclear until maintainer protocol review. A passive usage event is plausible, but maintainers still need to choose whether usage belongs in a new lifecycle phase, terminal lifecycle payload metadata, SDK mapping, or another event family.
