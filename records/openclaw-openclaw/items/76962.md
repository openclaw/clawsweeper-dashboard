## Summary

Keep open: current main still has a source-visible path where a subagent run can be marked timed out while detached exec-approval completion work remains able to resume or directly deliver late output outside the parent session.

## What This Changes

The issue reports that a timed-out subagent can leave a physical Claude CLI process and pending exec-approval work alive, later sending raw follow-up output directly to the user transport.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as explicit subagent kill, cancel or expire child-owned exec approval continuations, and suppress any late child-session follow-up once the owning subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical Claude CLI process. Source inspection does show the current-main chain: subagent timeout is recorded, detached exec approval completion can continue, and failed session follow-up can fall back to direct external delivery.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the stronger abort and queue cleanup path. The maintainable direction is a narrow lifecycle fix plus a child-owned late-followup guard, with maintainer approval for the exact fail-closed behavior.
