## Summary

Keep open: current main still has a source-level path where timed-out subagent work can be marked terminal while delayed exec-approval followup code may later deliver directly to the external transport after session resume fails.

## What This Changes

The issue asks OpenClaw to kill timed-out subagent `claude -p` runs, cancel or ignore their pending exec approvals, and prevent late child output from bypassing the parent agent.

## Best Possible Solution

Treat this as a core runtime bug: subagent timeout should abort the owning child run/process, cancel or ignore pending approval followups tied to that child, and suppress late child output once the parent/subagent state is terminal.

## Reproduction Assessment

No, not for the full live process-table reproduction. Source inspection does show a credible current-main path through subagent timeout bookkeeping plus delayed exec-approval direct fallback.

## Solution Assessment

No, current main is not the best solution because late exec followups can still direct-send after session resume failure. The narrower maintainable fix is to bind timeout/cancellation state to child runs and gate followup delivery on that state.
