## Summary

Keep open: current main still has a source-level path where subagent timeout is recorded as terminal without reusing the stronger subagent kill cleanup, and exec-approval followup can still fall back to direct external delivery after session resume failure.

## What This Changes

The issue reports that a timed-out subagent can leave a live Claude CLI run whose later exec-approval output bypasses the parent agent and reaches the user transport directly.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then suppress or keep internal any child-owned exec-approval followup after the subagent run is terminal.

## Reproduction Assessment

No, not as a live high-confidence reproduction of the physical Claude CLI zombie. Source inspection does show the current-main timeout/followup path that can mark a subagent terminal while leaving direct fallback delivery available.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The safer direction is a narrow owner-session cleanup plus child-terminal followup suppression, preserving legitimate direct delivery outside subagents.
