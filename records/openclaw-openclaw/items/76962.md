## Summary

Keep open: current main still has a source-level path where subagent timeout is recorded before the child run is actively aborted, while successful exec-approval followups can still fall back to direct external delivery after session followup failure.

## What This Changes

The issue requests that timed-out subagents kill their physical Claude CLI process, cancel pending exec approvals, and suppress any late child output from reaching the user transport.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then suppress child-owned exec-approval followups once the owning subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction of the physical zombie process was established here. Source inspection does show the timeout/followup path: timeout is recorded without the kill cleanup helper, and successful exec followups can still fall back to direct external delivery.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The safer repair is narrow: abort the child run at timeout, clear its queues, and drop stale child-owned followups after terminal timeout.
