## Summary

Keep open: current main still has a source-level path where a subagent wait timeout records terminal status without reusing the stronger subagent abort and queue-clear cleanup, while exec-approval followups can still fall back to direct user delivery after a failed session followup.

## What This Changes

The issue reports that timed-out Claude CLI subagents can keep running after `runTimeoutSeconds` and later deliver exec-approval output directly to the user transport.

## Best Possible Solution

Bind subagent timeout to child-run abort and queue cleanup, then suppress or keep internal any late exec-approval followups owned by terminal subagent sessions while preserving legitimate direct fallback for non-session diagnostics.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie process. Source inspection does show the current-main timeout and exec-approval followup paths that can mark the child terminal while leaving direct fallback delivery available.

## Solution Assessment

No, current main does not appear to be the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The safer fix is narrow but policy-sensitive: cancel the owning child run and suppress late child-owned followups after terminal timeout.
