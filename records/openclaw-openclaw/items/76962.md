## Summary

Keep open: current main still shows the reported timeout cleanup asymmetry and direct exec-approval fallback path, so this is not safe to close as stale, duplicate, or implemented.

## What This Changes

The issue reports that a timed-out subagent can leave a Claude CLI child running and later emit child-owned exec output directly to the user transport.

## Best Possible Solution

Bind subagent timeouts to the same abort and queue cleanup used by subagent kill, then suppress child-owned exec followups after the owning subagent is terminal.

## Reproduction Assessment

No, not as a live current-main reproduction. Source inspection does show a credible path: subagent timeout is recorded without kill-path cleanup, while exec-approval followup can still fall back to external direct delivery after session resume failure.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse child abort and queue cleanup. The requested direction is narrow, but the exact late-output suppression semantics should be maintainer-approved because they affect user transport routing.
