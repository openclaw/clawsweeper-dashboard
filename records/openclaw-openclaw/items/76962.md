## Summary

Keep open: current main has stronger CLI process timeout handling than the report's OpenClaw 2026.5.2 observation, but source inspection still shows subagent timeout completion does not reuse the subagent kill/queue-clear path and exec-approval followups can still direct-send after session resume failure.

## What This Changes

Request that timed-out subagents kill underlying Claude CLI runs, cancel pending exec approvals, and prevent late subagent output from bypassing parent delivery.

## Best Possible Solution

Bind subagent timeout terminalization to child abort and session queue cleanup, then suppress or keep internal any exec-approval followup owned by a terminal subagent unless a live parent-mediated resume succeeds.

## Reproduction Assessment

No live high-confidence way to reproduce the full zombie Claude process was established. Source inspection does show the current-main timeout/followup paths that can mark a subagent timed out without the kill cleanup path while preserving direct exec-approval fallback delivery.

## Solution Assessment

No, current main is not the best solution because timeout completion and manual kill have different cleanup semantics. The narrow maintainable fix is to share abort/queue-clear behavior and gate late subagent-owned followups after terminal timeout.
