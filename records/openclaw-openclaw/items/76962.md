## Summary

Keep open: current main still has a source-level path where exec-approval followup work can outlive a timed-out subagent and fall back to direct channel delivery after session resume fails. Current main has better agent-timeout abort plumbing for CLI runs, but the detached approval/followup path is not bound to the subagent lifecycle.

## What This Changes

The issue reports that timed-out subagents can leave child work alive and later emit exec-approval output directly to the user transport outside the parent agent.

## Best Possible Solution

Bind exec approval waits and followups to the owning run/session lifecycle: cancel or ignore late approval completions once a subagent is terminal, suppress direct fallback for terminal subagent sessions, and add regression coverage for successful late completions.

## Reproduction Assessment

Yes at source level: a subagent timeout can be recorded while detached exec-approval followup work remains capable of direct delivery after a failed session followup. I did not establish a live `claude -p` process-table reproduction.

## Solution Assessment

No, current main is not the best solution because agent-timeout abort plumbing does not cancel or suppress detached exec-approval followups. The narrow fix is lifecycle-bound cancellation/suppression around approval waits and direct fallback.
