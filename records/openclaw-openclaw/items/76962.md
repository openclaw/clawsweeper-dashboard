## Summary

Keep open: current main still has a source-visible exec approval follow-up path that can send directly to user transport after session resume failure, while the related timeout-attribution PR was closed unmerged and explicitly left physical CLI child cleanup, approval cancellation, and late direct-output suppression unresolved.

## What This Changes

The issue reports that timed-out subagents can leave a live Claude CLI child whose late exec-approval output is delivered directly to user transport, bypassing the parent agent.

## Best Possible Solution

Implement one hard-timeout invariant: timed-out subagents abort the active CLI child, clear pending exec approval/follow-up work, and suppress late child output from direct user transport unless maintainers explicitly define a safe fallback.

## Reproduction Assessment

No live reproduction was run, but the issue gives a concrete delayed approval path and current main still contains the session-resume-failure direct fallback that can deliver after a timed-out subagent.

## Solution Assessment

A prefix-only or timeout-attribution-only fix is not the best solution; the fix needs to join subagent hard-timeout cleanup, approval queue cancellation, and late-output suppression at the owner boundaries.
