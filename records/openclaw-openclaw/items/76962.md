## Summary

Keep open: current main still shows a source-level path where a subagent timeout is recorded as terminal without reusing the stronger subagent kill cleanup, and exec-approval followup can still fall back to direct external delivery after session resume failure.

## What This Changes

The issue reports that timed-out Claude CLI subagents can keep running and later emit exec-approval output directly to the user transport instead of through the parent agent.

## Best Possible Solution

Bind subagent timeout to child-run cancellation and child queue cleanup, then suppress child-owned late followups once the owning subagent or parent session is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie `claude -p` process. Source inspection does show the current-main timeout/followup chain that can mark the subagent timed out while approval waits and direct fallback delivery remain available.

## Solution Assessment

No, current main is not the best solution because the timeout path records completion without the cleanup used by explicit subagent kill. The safer direction is a narrow runtime fix that cancels the child run, clears child approval/followup queues, and drops late child-owned output after terminal timeout.
