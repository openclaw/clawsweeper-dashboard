## Summary

Keep open: current main still has a source-level path where subagent timeout is recorded without immediately using the existing child abort and queue cleanup path, while exec-approval follow-up can still fall back to external direct delivery after a failed session follow-up.

## What This Changes

The issue reports that timed-out subagents can leave the underlying Claude CLI process alive and later deliver raw follow-up output directly to the user transport.

## Best Possible Solution

Bind subagent timeout to child-run abort and queue cleanup, then suppress or internalize late exec-approval follow-ups once the owning subagent is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only pass. Source inspection does show the current timeout/follow-up path needed for the reported routing leak, while the physical Claude CLI zombie duration still needs live proof.

## Solution Assessment

No, current main is not the best way to solve the issue because timeout completion does not immediately reuse the existing abort and queue-clear cleanup path. The safer fix is narrow but policy-sensitive: cancel the owning child run and prevent late child-owned follow-ups from reaching external transport after terminal timeout.
