## Summary

Keep open: current main still shows a source-level path where subagent timeout is recorded without reusing the child abort/queue-clear cleanup path, and exec-approval followup can still fall back to direct external delivery after session resume failure.

## What This Changes

The issue asks OpenClaw to abort timed-out subagent processes, cancel their pending exec approvals, and prevent late child output from bypassing the parent agent.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then suppress or keep internal any child-owned exec-approval followup that arrives after the subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical `claude -p` zombie in this read-only review. Source inspection does show the current-main timeout waiter and direct followup fallback path that match the reported routing failure shape.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path, and successful late followups can still fall back to direct delivery. The narrow maintainable direction is to cancel the owning child run and suppress late child-owned output after terminal timeout while preserving ordinary-session fallback behavior.
