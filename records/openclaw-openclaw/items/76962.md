## Summary

Keep open: current main still has a source-level path where a timed-out subagent can be marked terminal while exec-approval followup code may later fall back to direct channel delivery instead of routing through the parent/session.

## What This Changes

The issue reports that timed-out claude-cli subagents can keep running, complete delayed exec-approval work, and emit late output directly to the user transport.

## Best Possible Solution

Treat this as a core runtime bug: timed-out subagents should abort the owning child run/process, cancel or ignore pending approval followups for that child, and never direct-send late child output after parent/session completion.

## Reproduction Assessment

Yes, at source level: the exec-approval followup path can direct-send after a failed session resume, and the subagent timeout waiter records timeout without cancelling approval work. I did not establish a live claude-cli zombie process reproduction, so this is not a high-confidence live reproduction.

## Solution Assessment

No, current main is not the best solution because direct fallback remains available for late subagent exec completions. The narrower fix is to bind timeout cancellation to the child run/process and suppress or discard followups once the owning subagent is terminal.
