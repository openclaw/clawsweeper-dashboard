## Summary

Keep open: current main still records a subagent wait timeout without reusing the abort-and-queue-clear path, while the exec approval followup path can still deliver late child-session output directly after session resume failure.

## What This Changes

The issue reports that timed-out subagents can leave a live Claude CLI process whose late approval output is delivered directly to the user transport.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then suppress or keep internal any late child-owned exec followups once the owning subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical Claude CLI process. Source inspection does show the current-main timeout/followup path that can mark a subagent terminal while direct fallback delivery remains available.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the stronger abort and queue cleanup path. The safer direction is a narrow lifecycle fix plus a child-owned late-followup guard, with maintainer approval for the exact fail-closed semantics.
