## Summary

Keep open: current main still has a source-visible chain where a subagent timeout is recorded as terminal without using the explicit kill/queue-cleanup path, while detached exec-approval completion can still attempt session follow-up or direct external delivery later.

## What This Changes

The issue asks OpenClaw to hard-abort timed-out subagent processes, cancel their pending exec approvals, and suppress late child output after the parent has finished.

## Best Possible Solution

Bind subagent timeout to explicit abort and queue-clear semantics, expire child-owned exec approval continuations, and make late child-session follow-up fail closed once the owning subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical Claude CLI process. Source inspection does show the current-main chain: timeout records a terminal subagent outcome, detached exec approval completion can continue, and failed follow-up can fall back to direct delivery.

## Solution Assessment

Yes for the maintainer direction, but the exact fail-closed routing policy needs owner/security review. The narrow maintainable fix is lifecycle cleanup parity with explicit kill plus a late-follow-up guard for terminal child runs.
