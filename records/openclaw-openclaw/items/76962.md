## Summary

Keep open: current main still records subagent `agent.wait` timeouts as terminal completion without invoking the existing abort and queue-clear cleanup path, while exec-approval followups still have a direct external fallback after session resume failure.

## What This Changes

The issue reports that timed-out subagents can leave a child Claude CLI run alive and later deliver exec-approval output directly to the user transport.

## Best Possible Solution

Bind subagent timeout to child runtime abort and session queue clearing, then suppress late child-owned exec followups once the subagent run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie Claude CLI process. Source inspection on current main does show the timeout path can mark the child run terminal without invoking the manual abort/queue-clear cleanup path, and tests only cover timeout recording.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the stronger cleanup semantics already used by manual subagent kill. The safer direction is a narrow runtime fix that aborts the owning child run and suppresses late child-owned followups after terminal timeout.
