## Summary

Keep open: current main still has a source-level path where subagent timeout is recorded without using the existing subagent kill cleanup path, while exec-approval followup can still fall back to direct user delivery after session resume failure.

## What This Changes

The issue asks OpenClaw to kill timed-out subagent CLI processes, cancel their pending exec approvals, and suppress late child output after the parent has finished.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then make exec-approval followups drop or stay internal once the owning subagent is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie process. Source inspection does show the current-main timeout/followup path that can mark the child terminal while leaving direct fallback delivery available.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The safer fix is narrow: cancel the owning child run and suppress late child-owned followups after terminal timeout.
