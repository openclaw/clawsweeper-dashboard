## Summary

Keep open: current main still records the subagent timeout as a terminal outcome without invoking the existing child abort and queue-clear path, while exec-approval followups can still fall back to direct user delivery after a failed session followup.

## What This Changes

The issue reports that subagent run timeouts can leave an underlying Claude CLI child alive long enough for delayed exec-approval output to bypass the parent agent and reach the user transport.

## Best Possible Solution

Timeout completion should abort the child run through the same runtime and queue cleanup used by manual subagent kill, then suppress child-owned exec followups once the subagent is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical Claude CLI process. Source inspection does show the current-main path where `agent.wait` timeout completes the subagent record without abort/queue cleanup and later exec followups can still direct-send.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The narrow maintainable fix is to cancel the owning child run and suppress late child-owned followups after terminal timeout.
