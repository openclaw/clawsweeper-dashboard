## Summary

Keep open: current main still has a source-level path where a subagent wait timeout is recorded as terminal completion without the stronger abort and queue cleanup used by explicit subagent kill, while exec-approval followup can still fall back to direct user delivery after failed session resume.

## What This Changes

The issue reports that timed-out subagents can leave an underlying Claude CLI process alive and later leak late exec-approval output directly to the user transport.

## Best Possible Solution

Bind subagent timeout to the same run abort and queue-clear semantics as explicit subagent kill, then suppress or keep internal any late child-owned exec followups after the child run is terminal.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie process. Source inspection does show a current-main path where timeout records terminal subagent completion without explicit kill cleanup while direct followup fallback remains reachable.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the existing child abort and queue cleanup path. The safer repair is narrow: cancel the owning child run and suppress late child-owned followups once the child is terminal.
