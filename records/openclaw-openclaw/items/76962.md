## Summary

Keep open: current main still leaves a source-visible path where subagent timeout is recorded while detached exec-approval follow-up work can later resume or directly deliver to an external route. The related open issues cover adjacent fallback-prefix and source-like payload problems, but they do not replace the subagent timeout cleanup and late-child-output lifecycle work here.

## What This Changes

Ensure timed-out subagents kill underlying Claude CLI work, cancel pending exec approvals, and suppress late child output from reaching the user transport.

## Best Possible Solution

Bind subagent timeout to explicit abort and queue-clear semantics, expire child-owned exec approval continuations, and make late child-session follow-up after terminal timeout fail closed.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical Claude CLI zombie path. Source inspection does show the current-main chain: subagent timeout can be recorded, detached exec approval completion can continue, and failed follow-up can fall back to external direct delivery.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the stronger abort and queue cleanup path. The maintainable direction is a narrow lifecycle fix plus a late-child-follow-up guard, with maintainer approval for exact fail-closed behavior.
