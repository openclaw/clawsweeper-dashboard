## Summary

Keep open: current main and the latest release still contain the exec-approval resume-failure direct fallback path, and current timeout handling only projects terminal subagent state rather than proving child-process kill, approval cancellation, or late-output suppression.

## What This Changes

Ensure timed-out subagents cannot leave live Claude CLI processes, pending exec approvals, or late direct user-transport output after parent finalization.

## Best Possible Solution

Enforce the hard-timeout invariant end to end: abort and drain the timed-out child CLI process, cancel its owned pending exec approvals, and suppress late child output from direct user delivery unless a maintainer-approved safe fallback applies.

Partially yes: the direct fallback and prefix are source-reproducible on current main, but the full physical `claude -p` zombie timing still needs live Claude/systemd proof.

No current-main solution is sufficient; the landed timeout projection work is useful but does not prove process kill, approval cancellation, or late-output suppression.
