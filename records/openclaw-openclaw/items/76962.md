## What This Changes

Issue #76962 reports that a hard-timed-out subagent can leave a live Claude CLI child whose later exec-approval output bypasses the parent agent and reaches the user transport directly.

## Best Possible Solution

Implement one core invariant for hard-timed-out subagents: abort or reap the child run, cancel or drain pending exec-approval/follow-up work, and suppress any later child output from direct user transport after the parent has finalized.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the fail-closed policy; the best fix should connect hard-timeout abort, approval cancellation, and late-output suppression instead of only removing the visible fallback prefix.
