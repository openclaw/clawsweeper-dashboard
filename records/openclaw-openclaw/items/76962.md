## Summary

Keep open: current main still has a source-level path where subagent timeout records a terminal outcome without the stronger abort/queue cleanup used by kill/steer, and exec-approval followups can still fall back to direct external delivery after a failed session followup.

## What This Changes

The issue asks OpenClaw to kill timed-out subagent Claude CLI processes, cancel their pending exec approvals, and prevent late subagent output from bypassing the parent session.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clearing cleanup used by kill/steer, then keep late exec-approval followups for terminal subagent sessions internal or drop them with regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was established for the physical zombie process. Source inspection does show the timeout/followup path on current main: timeout completion records a terminal outcome while direct exec-approval fallback remains available.

## Solution Assessment

No, current main is not the best way to solve the issue because timeout completion does not reuse the existing abort and queue cleanup path. The safer direction is a narrow runtime fix plus tests for late approved exec output from a timed-out subagent.
