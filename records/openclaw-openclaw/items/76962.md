## Summary

Keep open: current main has partial mitigations for exec approval follow-up delivery, but it still has a source-visible direct transport fallback after session resume failure and the automatic subagent hard-timeout path does not show the same abort-and-queue-clear cleanup used by explicit subagent kill. The related open issue only tracks the narrower prefix/retry symptom, and the related timeout-attribution PR closed unmerged while explicitly leaving physical process cleanup, approval cancellation, and late-output suppression unresolved.

## What This Changes

The issue requests hard-timeout cleanup for subagents so timed-out child processes, pending exec approvals, and late direct transport output are stopped instead of bypassing the parent agent.

## Best Possible Solution

Keep this open until the automatic subagent hard-timeout path aborts the active child run, clears owned approval/follow-up queues, and suppresses late direct fallback after the parent-controlled run is terminal.

## Reproduction Assessment

No live repro was run here, but current source still shows the direct fallback path and a timeout completion path that does not visibly abort the active child run or clear owned approval waits.

## Solution Assessment

No: current main is only a partial mitigation; the best fix should connect timeout cleanup, approval cancellation, and late-output suppression instead of only removing or narrowing the visible fallback prefix.
