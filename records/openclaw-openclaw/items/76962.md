## Summary

Keep open: current main still has a source-level path where a subagent run is marked timed out without using the stronger subagent kill cleanup, and exec-approval followup can still fall back to direct external delivery after session resume failure.

## What This Changes

The issue asks OpenClaw to kill timed-out subagent Claude CLI runs, cancel their pending exec approvals, and suppress late child output from direct user delivery.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then gate exec-approval followups so terminal child sessions cannot emit to external transports.

## Reproduction Assessment

No, there is not a high-confidence live reproduction from this review. Source inspection does show the current-main timeout/followup path that can mark a child terminal while leaving direct fallback delivery available.

## Solution Assessment

No, current main is not the best way to solve the issue because timeout completion does not reuse the child abort and queue cleanup path. The safer fix is a maintainer-reviewed, narrow runtime-routing change for timed-out child sessions.
