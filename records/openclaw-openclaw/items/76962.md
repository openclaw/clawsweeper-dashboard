## Summary

Keep open: current main still has a source-level path where a subagent timeout is recorded as terminal without using the stronger subagent kill/queue cleanup path, while late exec-approval followup can still fall back to direct user delivery after session resume failure.

## What This Changes

The issue asks OpenClaw to kill timed-out Claude CLI subagents, cancel their pending exec approvals, and suppress late child output after the parent has finished.

## Best Possible Solution

Bind subagent timeout to the same abort and queue-clear semantics as subagent kill, then suppress or keep internal any exec-approval followups owned by a terminal timed-out child session.

## Reproduction Assessment

No, not as a full live reproduction of the physical zombie process. Source inspection does show the current-main timeout/followup path that can mark a child run terminal while late direct delivery remains available after resume failure.

## Solution Assessment

No, current main is not the best solution because timeout completion does not reuse the stronger child abort and queue cleanup path. The safer fix is narrow: cancel the owning child run and suppress late child-owned followups after terminal timeout.
