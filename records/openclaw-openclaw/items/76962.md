## Summary

Keep open: current main still has the direct exec approval follow-up fallback that can send to user transport after session resume failure, while the landed timeout-projection work does not address physical CLI child cleanup, pending approval cancellation, or late subagent output suppression.

## What This Changes

## Best Possible Solution

Enforce one hard-timeout invariant: timed-out subagents abort the active CLI child, clear owned pending exec approval work, and suppress late child output from direct user transport unless a maintainer-approved safe fallback applies.

## Solution Assessment

A prefix-only or terminal-projection-only fix is not enough; the best fix needs to join subagent hard-timeout cleanup, approval queue ownership, and late-output suppression at the owning runtime boundaries.
