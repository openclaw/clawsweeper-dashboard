## Summary

Keep this PR open: the updated branch fixes the earlier watcher reconfiguration regression and now includes plausible live timing logs, but the new module-level resolvedSkills cache is unbounded and can retain full SKILL.md bodies across stale snapshot versions in a long-lived gateway.

## What This Changes

The PR adds a warm resolvedSkills cache for ensureSkillSnapshot, short-circuits repeated skills watcher scans, and adds focused unit coverage for cache hits and watcher reuse/reconfiguration.

## Best Possible Solution

Keep the warm-path optimization, but bound or replace stale resolvedSkills cache entries so old full Skill[] snapshots cannot accumulate indefinitely.

## Reproduction Assessment

Yes for the review finding: source inspection shows every cache miss stores a full resolvedSkills array in a module-level Map, and there is no production eviction path when snapshotVersion advances. I did not execute a heap-growth run in this read-only pass.

## Solution Assessment

No: the performance direction is reasonable, and the watcher reconfiguration fix is now present, but the resolvedSkills cache needs a small bounded invalidation strategy before this is the safest maintainable solution.
