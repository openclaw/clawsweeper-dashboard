## Summary

Keep open: the performance direction and live timing proof are credible, but PR head still has blocking correctness issues in the new cache and logging behavior.

## What This Changes

The PR caches `resolvedSkills` across turns, skips repeated skills watcher path resolution when config roots and debounce are unchanged, and adds focused tests for watcher reuse and cache hits.

## Best Possible Solution

Keep the warm-path optimization, but make the cache bounded or current-version-aware and remove or gate timing logs behind existing debug logging.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows an unbounded module-level Map and unconditional `console.log` calls in `ensureSkillSnapshot`. I did not execute heap-growth or runtime log-volume proof in this read-only pass.

## Solution Assessment

No: the optimization is pointed at the right hot path and the watcher reconfiguration fix is present, but the cache needs bounded invalidation and the perf logs need to be removed or debug-gated before this is the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound the resolvedSkills cache:** `src/auto-reply/reply/session-updates.ts:36`
  - body: This module-level Map retains every full `resolvedSkills` array by workspace/version/filter and is only cleared in tests. Because `resolvedSkills` carries parsed SKILL.md bodies and `snapshotVersion` changes on skill invalidation, a long-lived gateway can accumulate stale snapshots indefinitely; keep only current entries or add a small TTL/LRU bound.
  - confidence: 0.87
- **[P2] Gate the perf logging:** `src/auto-reply/reply/session-updates.ts:188-190`
  - body: `ensureSkillSnapshot` runs on normal reply dispatches, but the PR prints `[perf]` timing lines unconditionally. That will spam gateway stdout and add work on the hot path for every user, so remove these logs or guard them behind the existing debug/verbose timing mechanism.
  - confidence: 0.9
