## What This Changes

The PR adds a module-level `resolvedSkills` warm cache in `ensureSkillSnapshot`, short-circuits `ensureSkillsWatcher` when a watcher exists, and adds focused unit tests.

## Best Possible Solution

Keep the performance optimization only after preserving watcher reconfiguration semantics and adding redacted terminal/log/artifact proof for the warm-start path.

## Solution Assessment

No: the resolved-skills cache direction is plausible, but the watcher fast path should only skip work after proving the existing watcher still matches the current target set and debounce value.
