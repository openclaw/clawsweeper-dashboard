## Summary

Keep open: the cache direction is reasonable, but the proposed key still misses runtime eligibility inputs, so a warm gateway turn can reuse stale resolved skills after environment or local binary availability changes.

## What This Changes

The PR adds a bounded in-memory cache for hydrated `resolvedSkills` in `ensureSkillSnapshot`, keyed by workspace, snapshot version, skill filter, session agent, remote eligibility, and a redacted config fingerprint, with focused tests and a changelog entry.

## Best Possible Solution

Keep the cache only after reuse follows all skill eligibility inputs, or bypass caching for env/bin-sensitive snapshots, then add redacted live gateway/runtime proof of warm-turn reuse and invalidation.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an internal performance PR. The review finding is source-reproducible: current skill eligibility reads env and local binary state while the PR cache key excludes those inputs.

## Solution Assessment

No: the scoped cache is a plausible optimization, but the current key is not complete enough to preserve existing hydration behavior. The safer fix is to include a redacted runtime eligibility boundary or avoid cache hits for skills whose eligibility depends on env or local binaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Key resolvedSkills cache by runtime eligibility:** `src/auto-reply/reply/session-updates.ts:277-285`
  - body: `shouldIncludeSkill` also depends on `process.env` and local binary/PATH state through `hasEnv` and `hasBinary`, but this key only includes workspace/version/filter/agent/remote/config. A warm turn can therefore reuse cached `resolvedSkills` after an env var is added or removed, or after PATH changes make a required binary available or unavailable, whereas current main rebuilds and re-evaluates that eligibility on hydration. Include those runtime inputs in the boundary or skip caching for env/bin-sensitive snapshots.
