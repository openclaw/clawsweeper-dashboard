## Summary

Keep open: the optimization is useful, but the current patch still has source-proven cache compatibility regressions and the supplied real proof does not exercise the new module-level cache path.

## What This Changes

The branch adds watcher reuse and a bounded module-level `resolvedSkills` cache for `ensureSkillSnapshot`, with tests for reuse, reconfiguration, and invalidation.

## Best Possible Solution

Keep the watcher optimization only if it preserves plugin-derived watch roots, and either key module-level `resolvedSkills` reuse by the full redacted eligibility context or remove that cache until it has regression coverage and real runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection. Seed the module cache while a config-gated skill such as Discord is eligible, then remove `channels.discord.token`; the PR cache key is unchanged even though current-main eligibility should change.

## Solution Assessment

No. Caching this hot path is reasonable, but the current implementation is not the safest maintainable boundary because skill inclusion and plugin watch roots depend on more than the fields currently checked.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Key resolvedSkills cache by all eligibility inputs:** `src/auto-reply/reply/session-updates.ts:207-214`
  - body: The cache key fingerprints only `cfg.skills`, but `buildWorkspaceSkillSnapshot` filters skills through `requires.config` against the full config. After caching with `channels.discord.token` set, removing it reuses cached `resolvedSkills` and keeps that skill eligible; include a redacted fingerprint of the full effective eligibility context or avoid this cache.
  - confidence: 0.86
- **[P2] Recompute plugin watch roots before reusing the watcher:** `src/agents/skills/refresh.ts:141-146`
  - body: This early return compares only config watch paths and debounce, but the existing target resolution also includes plugin skill dirs from plugin activation config. A live config change that enables or disables plugin skills will now keep the old watcher and generated plugin-skill links; include plugin-derived roots in the compatibility check or fall back to the full target comparison.
  - confidence: 0.8
