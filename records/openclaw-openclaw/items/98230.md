## Summary

Keep open: the index fallback is the right direction for the Telegram picker bug, but this PR can still select the wrong model when an index callback goes stale while the index remains in range, and the PR only provides test proof for a visible Telegram callback change.

## What This Changes

The PR adds a Telegram model-picker callback variant that falls back from name-based callbacks to provider/index callbacks for model names that exceed Telegram callback-data limits.

## Best Possible Solution

Keep the length-safe fallback approach, but bind index callbacks to a stable model identity or opaque snapshot key before applying the selection, then add redacted real Telegram proof.

## Reproduction Assessment

Yes. Current main is source-reproducible: an over-limit compact mdl_sel payload makes buildModelSelectionCallbackData return null, and buildModelsKeyboard skips that model row while the header counts the full model list.

## Solution Assessment

No, not yet. The index fallback is the right maintainable shape, but it needs an identity check or opaque snapshot key so stale callbacks cannot select a different current model.
