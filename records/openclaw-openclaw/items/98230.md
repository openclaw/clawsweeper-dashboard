## Best Possible Solution

Keep the length-safe callback direction, but bind index callbacks to a stable model identity or opaque snapshot key and require redacted real Telegram proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: an over-limit compact `mdl_sel/{model}` payload makes `buildModelSelectionCallbackData` return `null`, and `buildModelsKeyboard` skips that row while the handler reports the full model count.

## Solution Assessment

No, not yet. The index fallback is the right maintainable shape, but it needs an identity check or opaque snapshot key so stale callbacks cannot select a different current model.
