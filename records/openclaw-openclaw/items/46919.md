The issue requests a config-controlled way to disable token URL, HTTPS, and related Control UI trust checks for trusted tailnet/private deployments.

## Best Possible Solution

Maintainers should either keep the current token/password/trusted-proxy/Tailscale identity boundaries or explicitly design a named dangerous private-ingress/tailnet policy with audit warnings, docs, startup validation, and regression coverage.
