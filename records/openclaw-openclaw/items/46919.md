## Summary

Keep open. Current main and the latest release still append dashboard auth tokens when token auth is active, the requested `urlToken`/`tokenURL`/trusted-tailnet bypass config surface is absent, and the existing tokenless paths are intentionally narrower than the requested blanket auth and device-identity bypass.

## What This Changes

The issue requests a config-controlled way to disable tokenized dashboard URLs and relax related HTTPS, pairing, device identity, and Tailnet trust checks.

## Best Possible Solution

Make an explicit Gateway security/product decision; if accepted, design a narrow private-ingress mode with schema, docs, audit warnings, origin/proxy semantics, and focused tests instead of one broad token, HTTPS, pairing, and device-identity bypass flag.
