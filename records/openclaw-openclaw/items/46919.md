Keep open. Current main still lacks the requested URL-token disable switch, and the remaining request would intentionally change Gateway admin authentication, device identity, pairing, HTTPS, and tailnet trust policy.

## What This Changes

The issue requests a config flag to disable tokenized dashboard URLs and broadly trust tailnet/control UI access without tokens, pairing, HTTPS, or device identity.

## Best Possible Solution

Make an explicit Gateway security/product decision; if accepted, design a narrow private-ingress mode with schema, docs, audit warnings, origin/proxy semantics, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the requested keys are absent and the existing tokenless/trusted paths are narrower than the requested blanket bypass.

## Solution Assessment

No/unclear. A single flag spanning dashboard URL fragments, HTTP auth, WebSocket auth, pairing/device identity, HTTPS, and bind policy is not the safest maintainable design without secops/product approval.
