## Best Possible Solution

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows no `urlToken`/`tokenURL` switch and confirms existing tokenless/trusted paths are narrower than the requested blanket bypass.

## Solution Assessment

No/unclear. A blanket one-flag bypass across dashboard tokenization, HTTP auth, WebSocket auth, pairing/device identity, and HTTPS/bind policy is not the safest maintainable design; a secops-reviewed private-ingress design or explicit rejection is safer.
