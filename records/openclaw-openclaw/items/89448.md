## Summary

The PR is still necessary and appears correct: current main and the latest release still return the multiplied cron duration without checking the result for finiteness, while the PR adds the missing parser guard and focused regression coverage.

## What This Changes

The PR adds a finite-result check to `src/cli/cron-cli/shared.ts` `parseDurationMs` and adds tests for valid, invalid, and overflow duration inputs.

## Best Possible Solution

Land the narrow parser guard with the regression tests, then let the linked bug close through the merged PR.

## Reproduction Assessment

Yes. Source inspection shows a long pure-digit duration can match the cron parser, parse to a finite mantissa, overflow after multiplying by the day factor, and remain truthy for the `--every` caller; I did not run the repro because this review was read-only.

## Solution Assessment

Yes. Rejecting the non-finite computed milliseconds in the cron-specific parser is the narrowest maintainable fix; switching to the broader shared duration parser would change more parsing semantics than this bug requires.
