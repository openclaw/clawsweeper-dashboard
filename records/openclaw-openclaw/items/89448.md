## What This Changes

Adds a finite-result guard to the cron CLI duration parser and regression coverage for overflow, valid, and invalid duration inputs.

## Best Possible Solution

Land the narrow parser guard with the regression tests, then close the linked issue after the merge.

## Reproduction Assessment

Yes. Source inspection plus a focused Node arithmetic check show that a regex-accepted pure-digit duration can produce a finite mantissa but overflow to `Infinity` after multiplying by the day factor.

## Solution Assessment

Yes. The guard belongs in the cron CLI parser used by the affected options; relying on gateway validation would leave poorer CLI behavior, and switching to the broader general duration parser would change more semantics than this bug requires.
