## Summary

Keep open for merge review: current main and the latest release still have the cron duration overflow path, while this PR's narrow guard and regression tests address it with no blocking findings.

## What This Changes

This PR updates the cron CLI duration parser to return null when a finite token overflows during unit multiplication, and adds coverage for valid, invalid, and overflow duration strings.

## Best Possible Solution

Land the narrow parser guard with the regression coverage, then let the linked issue close from the merge.

## Reproduction Assessment

Yes. Source inspection shows a long pure-digit duration can match the cron parser, parse to a finite mantissa, overflow after multiplying by the day factor, and remain truthy for the --every caller.

## Solution Assessment

Yes. Rejecting the non-finite computed milliseconds in the cron-specific parser is the narrowest fix; switching to the broader shared duration parser would change more parsing semantics than this bug requires.
