## Summary

Keep open; this is a focused bug-fix PR and current main still lacks the finite-result check in the cron-local duration parser. The proposed guard matches existing caller behavior that treats `null` as an invalid duration, and I found no blocking correctness or security issue.

## What This Changes

The PR adds a finite-result guard to cron CLI `parseDurationMs` and extends shared cron CLI tests for valid, invalid, and overflow durations.

## Best Possible Solution

Land the focused guard and regression tests after normal maintainer and CI checks; leave broader duration-parser unification to a separate intentional cleanup.

## Reproduction Assessment

Yes. Current main source shows a high-confidence reproduction path: pass a long pure-digit day duration that parses to a finite mantissa but overflows after multiplication, and the current function returns the floored non-finite product.

## Solution Assessment

Yes. The finite-result guard is the narrowest compatible fix because replacing the cron-local parser with the broader shared CLI duration parser would also change accepted cron duration grammar and safe-integer behavior beyond this bug.
