  - body: After checking out the PR branch, this passes `prompt-file: .github/codex/prompts/ci-fix-agent.md` to the pinned Codex action.   - confidence: 0.9

  - body: The enforcement step only records that some change exists, then the commit step stages everything with `git add -A`.   - confidence: 0.93
- **[P2] Resolve failed PRs with the configured app token:** `.github/workflows/clawsweeper-dispatch.yml:310-312`
  - body: This step exports the ClawSweeper app token as `GH_TOKEN`, but the PR lookup overrides it with `${GITHUB_TOKEN}`, which this step never sets. That can make the lookup unauthenticated or use the wrong token, so the dispatched payload can carry `pr_number: 0` instead of the failed PR.
  - confidence: 0.88
- **[P2] Create an app token before cross-repo dispatch:** `.github/workflows/ci-fix-agent.yml:215-217`
  - body: The new CI Fix Agent checks for `CLAWSWEEPER_APP_PRIVATE_KEY` but never creates a ClawSweeper app token before calling `repos/openclaw/clawsweeper/dispatches`. The repository `GITHUB_TOKEN` is scoped to this repo, so this best-effort notification path can silently fail even when the app key exists.
  - confidence: 0.82
