Full review comments:

- **[P1] Load the repair prompt from trusted main inputs:** `.github/workflows/ci-fix-agent.yml:296`
  - body: After checking out the PR branch, the workflow reads `prompt-file: .github/codex/prompts/ci-fix-agent.md` from that same branch while passing an OpenAI key. A same-repo branch can rewrite the prompt that controls the privileged repair agent, so keep the prompt/control-plane input on trusted main or otherwise pin it before checkout.
  - confidence: 0.83
- **[P1] Enforce the promised patch boundary before pushing:** `.github/workflows/ci-fix-agent.yml:338`
  - body: The enforcement step only checks whether any diff or untracked file exists, then the commit step stages everything with `git add -A`. That means Codex can still add workflows, security policy, generated files, or unrelated paths despite the prompt limits; reject forbidden paths/statuses before committing, as the existing docs and test-performance agents do.
  - confidence: 0.91
- **[P2] Resolve CI failure PRs with an exported token:** `.github/workflows/clawsweeper-dispatch.yml:310-312`
  - body: This step exports the ClawSweeper app token as `GH_TOKEN`, but the PR lookup overrides the CLI with `${GITHUB_TOKEN}`, which the step never defines. On workflow_run, that falls back to `0`, so ClawSweeper receives a CI-failure payload that is not tied to the failed PR.
  - confidence: 0.88
- **[P3] Scope the retry budget to the target PR:** `.github/workflows/ci-fix-agent.yml:91-109`
  - body: The cadence gate counts all recent `ci-fix-agent` runs without filtering by PR, branch, or head SHA. Three unrelated failed PRs in six hours can suppress repairs for every other PR, so key the limit to the target PR or head SHA.
  - confidence: 0.8
