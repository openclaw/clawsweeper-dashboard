  - body: The only enforcement step records whether any change exists, then the commit step stages everything with `git add -A`. Prompt text cannot enforce the promised fix-only scope; reject untracked files, forbidden statuses, and forbidden paths before any commit or push.
  - confidence: 0.93

  - body: This step checks for `CLAWSWEEPER_APP_PRIVATE_KEY` but never creates or exports the ClawSweeper app token before calling `repos/openclaw/clawsweeper/dispatches`. The repository token is not enough for that cross-repo dispatch, so the notification path can silently fail even when the app key exists.
  - confidence: 0.86
- **[P2] Count CI repair cadence per PR:** `.github/workflows/ci-fix-agent.yml:108`
  - body: `Gate daily cadence per PR` fetches recent workflow runs but never filters them by `PR_NUMBER` or head SHA, so three unrelated CI-fix runs in six hours can disable repair for every other failing PR. Filter the cadence window to the resolved PR or head SHA before skipping.
  - confidence: 0.82

  - body: This step exports the ClawSweeper app token as `GH_TOKEN`, then overrides the PR lookup with `${GITHUB_TOKEN}`, which is not set in the step environment. That can make the lookup unauthenticated or tokenless and dispatch `pr_number: 0` instead of the failed PR.
  - confidence: 0.88
