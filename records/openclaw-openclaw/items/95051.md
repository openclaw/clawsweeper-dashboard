## Summary

Keep open: the PR is on the right Telegram-only fix path, but the newly allowed reasoning payloads can still pass through shared TTS before Telegram strips `isReasoning`, so TTS-enabled users could get spoken Thinking output.

## What This Changes

The PR lets Telegram durable `isReasoning` block and final replies through shared dispatch, converts them to Telegram reasoning-lane text, strips the marker before durable send, and adds regression tests.

## Best Possible Solution

Land the Telegram-only durable reasoning fix after reasoning payloads bypass both final and all-mode block TTS, with focused regression coverage and Telegram-visible proof before merge.

## Reproduction Assessment

Yes, source-level: current main suppresses both block and final `isReasoning` payloads before Telegram delivery, and the linked production report gives concrete `/reasoning on` steps. I did not run a live Telegram reasoning-model session in this read-only review.

## Solution Assessment

No as written: the Telegram-only pass-through and marker stripping are the right owner boundary, but reasoning payloads must bypass shared TTS before this is the best fix. The safer repair is to keep `isReasoning` out of final and all-mode block TTS while preserving the new Telegram delivery path.
