## Summary

Keep open: this is a valid candidate fix for the Telegram durable-reasoning message-loss bug, but current head still lets Telegram `isReasoning` final payloads enter shared final TTS before Telegram strips the marker, so TTS-enabled sessions can receive spoken Thinking content.

## What This Changes

The PR lets Telegram durable `isReasoning` replies pass shared dispatch, converts them into Telegram reasoning-lane text, strips the marker before send, and adds shared-dispatch plus Telegram regression tests.

## Best Possible Solution

Land a Telegram-only durable reasoning fix that bypasses TTS for reasoning payloads, preserves generic-channel suppression, keeps focused block/final regression coverage, and captures Telegram-visible proof before merge.

## Reproduction Assessment

Yes, source-level: current main suppresses both block and final `isReasoning` payloads before Telegram delivery, and the linked production report provides concrete `/reasoning on` steps. I did not run a live Telegram reasoning-model session in this read-only review.

## Solution Assessment

No as written: the Telegram-only pass-through and marker stripping are the right owner boundary, but reasoning payloads must also bypass shared TTS before this is the best fix. The safer repair is to keep `isReasoning` out of TTS for final and all-mode block paths while preserving the new Telegram delivery path.
