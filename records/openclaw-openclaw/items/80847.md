## What This Changes

The issue reports that plugin tools declared in `contracts.tools` can work initially but later fail in sub-agent sessions with `plugin tool runtime missing`.

## Best Possible Solution

Keep the current cached descriptor dispatch fix and direct 2026.5.7 users to a build containing `v2026.5.9-beta.1` or newer.

## Reproduction Assessment

Yes for the affected 2026.5.7 behavior: the canonical lossless-claw report and PR proof exercise the same multi-tool unnamed-factory cache path. Current main contains the dispatch fix, so this should not reproduce there.

## Solution Assessment

Yes. Dispatching cached descriptor tools by the actual returned runtime tool name is narrower than changing sub-agent registry reuse and matches the manifest contract boundary.
