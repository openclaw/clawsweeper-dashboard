## Summary

Keep open: this is a fresh, concrete regression report for an existing plugin-tool contract, and current main still has the registry path that can reload a standalone tool registry without pinning it before the cached descriptor wrapper throws `plugin tool runtime missing`. A follow-up comment confirms the path on current main, and the related open registry-state issue is adjacent rather than a complete duplicate.

## What This Changes

The issue reports that `contracts.tools` plugin tools work initially but later fail in sub-agent sessions after archival or respawn with `plugin tool runtime missing`.

## Best Possible Solution

Add a focused regression for cached plugin-tool execution after sub-agent archival or registry replacement, then keep the live gateway registry or pinned equivalent resolvable for the gateway lifetime without broad plugin loading.

## Reproduction Assessment

Yes, at source/report level. The issue provides a concrete gateway and sub-agent archival sequence, a follow-up confirms current main, and current code still shows the forced standalone load path ending at the reported throw.

## Solution Assessment

Unclear until a focused regression exists. Reusing or pinning the gateway registry is likely the right direction, but the fix must preserve existing explicit-subagent scoping and avoid loading unrelated plugins.
