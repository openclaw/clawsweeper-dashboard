## Summary

Keep open: this is an active focused bug-fix PR for a source-visible LINE delivery failure that is still present on current main, and I did not find a blocking correctness or security issue in the proposed patch.

## What This Changes

The PR stops cron-tool from inferring LINE delivery targets from lowercased session keys, adds a LINE outbound recipient guard, and adds focused cron-tool and LINE send tests.

## Best Possible Solution

Land this narrow cron routing and LINE outbound-boundary guard with the included regression coverage, then handle stricter LINE ID validation or per-peer source repair separately if maintainers want it.

## Reproduction Assessment

Yes for source-level reproduction: current main lowercases LINE peer IDs in session keys and cron-tool can copy that fragment into `delivery.to` when current delivery context is unavailable. I did not run a live LINE group reproduction in this read-only review.

## Solution Assessment

Yes; refusing unrecoverable LINE session-key fallback and adding a channel-owned permanent-error guard is a narrow maintainable fix. Broader strict LINE ID validation can stay separate because it would touch fixture and compatibility behavior beyond this bug.
