## Summary

Keep open for maintainer review: the PR is a narrow, source-backed fix for the Ollama non-reasoning `think` rejection path, with sufficient after-fix proof and no blocking correctness findings found in the diff.

## What This Changes

The PR adds a capability-aware guard that omits truthy native Ollama `think` for `reasoning: false` models and adds regression tests for configured and runtime thinking paths.

## Best Possible Solution

Land this narrow guard or an equivalent focused patch after maintainer review, while preserving top-level Ollama `think` for supported models instead of moving native thinking into `options.think`.

## Reproduction Assessment

Yes, by source inspection: current main can put truthy top-level native Ollama `think` on requests for models represented as `reasoning: false`, and upstream Ollama rejects truthy `req.Think` for models without thinking capability. I did not run a live Ollama daemon in this read-only review.

## Solution Assessment

Yes, the PR is the narrow maintainable fix because it preserves upstream-compatible top-level `think` for reasoning-capable or unknown models, preserves explicit `think:false`, and only suppresses the known-invalid truthy case.
