## Summary

This is an active, focused fix for a current Ollama request-shaping bug, not a cleanup-close candidate. I found no blocking code or security findings; the remaining path is normal maintainer review and CI.

## What This Changes

The PR adds a capability-aware guard that omits truthy native Ollama `think` values for models marked `reasoning: false` and adds regression tests for configured and runtime thinking.

## Best Possible Solution

Land the narrow guard after normal maintainer CI/review, preserving top-level `think` for reasoning-capable or unknown native Ollama models and `think:false` for explicit off.

## Reproduction Assessment

Yes, by source inspection: current main can send truthy top-level native Ollama `think` from configured params or runtime thinking, while upstream Ollama rejects truthy `req.Think` for models without thinking capability. I did not run a live Ollama daemon in this read-only review.

## Solution Assessment

Yes, the PR’s capability-aware guard is the narrow maintainable fix because moving `think` into `options` would contradict both OpenClaw docs and upstream Ollama’s typed top-level chat contract.
