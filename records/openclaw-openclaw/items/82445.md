## Summary

Keep open: the patch looks like a focused, correct fix for a source-reproducible Ollama request-shaping bug, but this external PR still lacks required after-fix proof from a real Ollama setup.

## What This Changes

The PR adds a capability-aware guard so truthy native Ollama `think` is skipped for models marked `reasoning: false`, while preserving `think: false` and adding request-body regression tests.

## Best Possible Solution

Land the capability-aware guard after the contributor adds real Ollama proof showing a non-reasoning model no longer returns the top-level truthy `think` 400, while preserving the documented top-level native `think` contract.

## Reproduction Assessment

Yes, by source inspection: current main can place truthy configured or runtime Ollama `think` on the top-level `/api/chat` request, and upstream Ollama rejects truthy `req.Think` for models without the thinking capability. I did not establish a live Ollama repro in this read-only review.

## Solution Assessment

Yes, the PR’s capability-aware skip is the narrowest maintainable fix I can point to: it avoids the upstream 400 for known non-reasoning models without moving native thinking control to undocumented `options.think`. The remaining blocker is real behavior proof, not a concrete code finding.
