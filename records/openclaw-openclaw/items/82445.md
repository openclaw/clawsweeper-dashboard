## Summary

Keep open: the patch is a focused source-level fix for the Ollama non-reasoning `think` failure and I found no blocking code-review bug, but this external PR still lacks after-fix proof from a real Ollama setup.

## What This Changes

The PR guards native Ollama `think` forwarding so truthy values are skipped for models explicitly marked `reasoning: false`, while preserving top-level `think` for reasoning-capable or unknown models and `think: false`.

## Best Possible Solution

Land a capability-aware guard that preserves upstream-compatible top-level `think` for reasoning-capable native Ollama models, skips truthy values for `reasoning: false`, keeps `think: false`, and proves the real non-reasoning Ollama path no longer returns 400.

## Reproduction Assessment

Yes, by source inspection: current main can place truthy configured or runtime Ollama `think` on the top-level `/api/chat` request, and upstream Ollama rejects truthy `req.Think` for models without the thinking capability. I did not run a live Ollama repro in this read-only review.

## Solution Assessment

Yes, the PR’s capability-aware skip is a narrower and more maintainable fix than moving all thinking control to undocumented `options.think`, because current OpenClaw and upstream Ollama both define native thinking as top-level `think`. The remaining blocker is proof, not a code-shape issue I can point to in the diff.
