## Best Possible Solution

Land browser screenshot vision only after the vision result keeps raw screenshots out of automatic channel delivery unless a maintainer-approved delivery policy explicitly allows it, with the unused import removed and tests/docs aligned.

## Reproduction Assessment

## Solution Assessment

No. The core idea is maintainable, but the current patch should first stop the vision text path from advertising the raw screenshot as deliverable media and remove the unused schema import.
