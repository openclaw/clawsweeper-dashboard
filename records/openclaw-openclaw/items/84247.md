## Summary

Keep open: this is a useful opt-in browser screenshot vision feature, but the current patch is not merge-ready because untrusted vision text can still contain line-start `MEDIA:` directives that current trusted browser tool-result extraction can deliver to channels.

## What This Changes

The PR adds opt-in `browser.models` screenshot vision for text-only main models, plus browser config/schema/docs/tests and auth-profile forwarding for media understanding.

## Best Possible Solution

Land only after browser vision text cannot create deliverable media, the opt-in provider egress contract stays documented, and current-head real proof shows both the text-only screenshot flow and directive-shaped text not attaching media.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR returns unescaped wrapped vision text, while current main extracts line-start `MEDIA:` from trusted browser text results. I did not run a live forged-provider scenario in this read-only review.

## Solution Assessment

No. The feature direction is useful, but the current patch should neutralize directive-shaped text or update extraction to ignore external-content-wrapped blocks before treating the vision output as a trusted browser result.
