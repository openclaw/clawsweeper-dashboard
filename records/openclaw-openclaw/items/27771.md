## Summary

Keep open. Current main still lacks Signal `configPath` support, and this useful PR is not merge-ready because it wires the feature through obsolete `src/signal` files instead of the current bundled Signal plugin; the author has also invited maintainer takeover.

## What This Changes

This PR adds optional Signal `configPath` config/docs/schema/help/type wiring and legacy runtime/tests intended to pass a trimmed `--config` argument to `signal-cli` autostart.

## Best Possible Solution

Port optional top-level and per-account Signal `configPath` support end to end through the current bundled Signal plugin and config/docs metadata, then supersede this source PR after a replacement lands.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Signal schema, typed config, account resolution, monitor options, and daemon argv builder all lack `configPath`, so a `channels.signal.configPath` config cannot validate or reach `signal-cli`.

## Solution Assessment

No. The feature direction is sound and upstream supports `--config`, but the current PR should be replaced or ported to `extensions/signal` plus the current generated config/docs surfaces rather than merged as-is.
