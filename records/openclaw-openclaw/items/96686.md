## Summary

## What This Changes

The PR changes Runtime prompt rendering to use a stable base session key for run-scoped isolated cron runs and adds parser/runtime-line tests.

## Best Possible Solution

Keep the Runtime render-boundary fix, but constrain run-scope stripping to the established agent-scoped cron-run key shape and cover non-cron terminal `:run:<id>` keys before merge.

## Reproduction Assessment

Yes at source level. Current main constructs a `:run:` isolated-cron session key, passes it into embedded execution, and renders it in the Runtime prompt line; I did not run a live metered-provider billing comparison.

## Solution Assessment

No. Stabilizing prompt-facing cron identity is the right layer, but this implementation should be anchored to the actual cron-run session-key contract instead of stripping any terminal `:run:<id>` suffix.
