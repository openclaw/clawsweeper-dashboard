The PR changes Runtime prompt rendering to show a stable base session key for isolated cron runs and adds tests for the new run-scope parser and runtime-line cache stability.

## Best Possible Solution

## Solution Assessment

No. Stabilizing prompt-facing cron identity is the right layer, but this implementation should be anchored to the actual cron-run session-key contract instead of stripping any terminal `:run:<id>` suffix.
