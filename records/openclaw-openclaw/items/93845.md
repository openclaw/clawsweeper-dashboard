## What This Changes

## Best Possible Solution

Preserve newer-visible stale suppression while fixing truncation, likely through a delivery-phase or lazy fence boundary that prevents both data loss and out-of-order replay.

## Reproduction Assessment

Yes, at source level. Current main has eager fence creation and unconditional newer-visible cancellation, while the PR's changed tests demonstrate the opposite risky ordering: newer output followed by older output or fallback.

## Solution Assessment

No. Guarding on activeGenerations is not the best fix because it tracks whole dispatch lifetime rather than visible-delivery freshness; the fix needs delivery-ordering state that prevents truncation without allowing stale output after a newer visible reply.
