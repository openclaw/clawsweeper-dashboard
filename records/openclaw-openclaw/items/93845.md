## What This Changes

## Best Possible Solution

Preserve newer-visible stale suppression while fixing truncation, likely with delivery-phase or lazy foreground-fence state that prevents both data loss and out-of-order replay.

## Reproduction Assessment

Yes, at source level. Current main has the eager fence and unconditional newer-visible cancellation, while this PR's changed tests demonstrate the opposite risky ordering: newer output followed by older output or fallback.

## Solution Assessment

No. Guarding on activeGenerations is not the best fix because it tracks whole dispatch lifetime rather than visible-delivery freshness; the fix needs a delivery-ordering state that prevents truncation without allowing stale output after a newer visible reply.
