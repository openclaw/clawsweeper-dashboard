Keeping this open because current `main` still does not contain the requested `openai-compatible` memory embedding provider, and https://github.com/openclaw/openclaw/pull/80479 is an open same-author implementation PR with closing syntax for this issue. ## What This Changes

Add a bundled `openai-compatible` memory embedding provider for self-hosted OpenAI-compatible HTTP embedding servers such as llama.cpp, Ollama `/v1`, vLLM, TGI, LocalAI, and llamafile.
