Request a bundled `openai-compatible` memory embedding provider for self-hosted OpenAI-compatible HTTP embedding servers such as llama.cpp, Ollama `/v1`, vLLM, TGI, LocalAI, and llamafile.

## Best Possible Solution

Resolve the request through the linked implementation PR: merge a bundled provider with aligned docs/tests if maintainers accept the scope, or route it to an external plugin/ClawHub path while preserving the current workaround docs.
