## Summary

Keep open: this is useful Control UI title work, but the current merge result still lets `sessions.changed` rows drop the channel metadata that the new title safety filter depends on, so generated contact or conversation names can still reach `document.title`.

## What This Changes

## Best Possible Solution

Preserve `origin` and `chatType` through the `sessions.changed` UI row projection with regression coverage, then merge only after maintainers choose the dynamic-session-title and static configured-title precedence policy.

No for merge as-is. The session-title resolver is a plausible implementation, but the best fix must preserve event metadata first and then align with the static-title precedence decision.
