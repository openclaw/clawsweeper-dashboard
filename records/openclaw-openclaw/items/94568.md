## Best Possible Solution

Land the narrow CLI filter with CLI docs, focused regression coverage for default and opt-in archived output, and redacted real behavior proof from a real `openclaw workboard list` run.

## Reproduction Assessment

Yes. Source inspection shows current `main` and `v2026.6.8` CLI list all store cards and filter only by status, while `workboard_list` already filters archived cards unless `includeArchived` is true; I did not run a live CLI because this review is read-only.

## Solution Assessment

Yes, with follow-up. Filtering in the terminal CLI list path is the narrow owner-boundary fix because `WorkboardStore.list` remains an all-cards primitive and sibling list surfaces own archived-card visibility; the user-visible flag needs docs and regression proof before merge.
