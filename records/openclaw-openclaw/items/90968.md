## Summary

Keep open: the startup replay suppression is useful, but this PR still needs current-head real Claude ACP proof and has blocking compatibility defects around Claude model routing and cached/update replay behavior.

## What This Changes

The PR changes ACP session initialization, runtime-handle caching, runtime-control replay, and `sessions_spawn` so startup model/thinking/cwd options are not resent as ACP config controls and Claude ACP model/thinking options are stripped, with regression tests.

## Best Possible Solution

Keep the startup replay suppression, but use one canonical Claude ACP policy that preserves valid Claude model ids, strips or remaps only unsupported controls across spawn/init/resume/cache/update paths, and proves the latest head with real Claude ACP output.

## Reproduction Assessment

Yes from source, though I did not run a live Claude ACP session in this read-only review. Current main can replay model/thinking as ACP config controls, and related live evidence shows non-Claude model/thinking controls are rejected by Claude ACP.

## Solution Assessment

No. The applied-options delta cache is the right direction for replay suppression, but blanket core stripping is too broad and should be replaced with a canonical Claude ACP policy that preserves valid model ids consistently.
