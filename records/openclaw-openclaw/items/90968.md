Keep open: the PR addresses a real ACP runtime-control problem, but the current branch is not merge-ready because it blanket-strips valid Claude ACP model choices, leaves replay/update paths inconsistent, and lacks current-head real behavior proof after the force push.

## What This Changes

The branch changes ACP startup/resume/runtime-control caching so startup options are not replayed as config controls and Claude ACP model/thinking options are stripped from managed spawn/init/resume paths.

## Best Possible Solution

Keep the replay-delta cache direction, but use one canonical Claude ACP runtime-option policy that preserves valid Claude model ids, filters unsupported non-Claude controls consistently across spawn/init/resume/update/replay, and adds current-head live proof.

## Reproduction Assessment

Yes from source and dependency inspection. Current main can replay model/thinking as ACP config controls, claude-agent-acp rejects unknown or invalid config controls, and the PR head visibly drops valid Claude model options instead of preserving the supported model path.

## Solution Assessment

No. The applied-runtime-options delta cache is a useful part of the fix, but blanket core stripping is too broad; the safer solution is a canonical Claude ACP policy that preserves valid model ids and handles unsupported controls consistently.
