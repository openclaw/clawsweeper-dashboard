## Summary

Keep open. Current `main` still has the duplicate top-level `web` entries in the WhatsApp configuration example, and the existing config type expects those fields to coexist under a single `web` object.

## What This Changes

The issue requests merging the WhatsApp docs example into one top-level `web` object so copy-pasted JSON5 preserves both `web.whatsapp` and general web-channel settings.

## Best Possible Solution

Update the WhatsApp docs example to use one top-level `web` object containing both the general web-channel settings and the nested `whatsapp` timeout settings.

## Reproduction Assessment

Yes. Current `main` visibly reproduces the docs issue at `docs/gateway/config-channels.md:99` and `docs/gateway/config-channels.md:121`, where the same JSON5 object declares `web` twice.

## Solution Assessment

Yes. A narrow docs-only merge of the two `web` objects is the maintainable fix; the runtime config type already supports all of these fields together.
