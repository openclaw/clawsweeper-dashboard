## Summary

Keep open: the cancellation problem is source-provable on current main, but this external PR is incomplete, targets stale memory-host provider files after the provider-plugin move, and lacks real after-fix behavior proof.

## What This Changes

The branch adds optional AbortSignal plumbing to old memory-host OpenAI, Gemini, and Voyage batch helpers plus the shared batch execution params.

## Best Possible Solution

Add an optional AbortSignal contract through MemoryEmbeddingBatchOptions, pass it from memory-core into provider runtimes, update the current extension batch helpers to cancel sleeps and HTTP requests with listener cleanup, and add focused OpenAI, Gemini, and Voyage coverage.

## Reproduction Assessment

Yes, by source inspection. Current main has no signal field in MemoryEmbeddingBatchOptions, memory-core does not forward one, and the OpenAI/Gemini/Voyage provider batch loops wait on uncancelable timers.

## Solution Assessment

No. The direction is right, but this PR is not the best merge path because it changes stale memory-host provider files and does not fully wire cancellation through the current production contract and provider-owned implementations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread the signal from the production batch API:** `packages/memory-host-sdk/src/host/batch-runner.ts:10`
  - body: Adding signal only to EmbeddingBatchExecutionParams does not make disconnect cancellation reachable. Current main's MemoryEmbeddingBatchOptions and memory-core batchEmbed call path still have no signal to provide, so normal OpenAI/Gemini/Voyage batch runs continue with params.signal undefined.
  - confidence: 0.91
- **[P2] Clean up abort listeners after polling sleeps:** `packages/memory-host-sdk/src/host/batch-openai.ts:191-200`
  - body: Each polling sleep registers an abort listener but never removes it when the timer resolves normally, and already-aborted signals are not rejected before sleeping. Long polls can retain stale listeners and still miss cancellation; the same pattern appears in the Gemini and Voyage additions.
  - confidence: 0.9
- **[P2] Pass the signal into Gemini status polling:** `packages/memory-host-sdk/src/host/batch-gemini.ts:160`
  - body: fetchGeminiBatchStatus gains a signal parameter, but the status request still is not wired to use it consistently. Aborts during an in-flight Gemini status poll can continue waiting on the network call instead of stopping promptly.
  - confidence: 0.84
