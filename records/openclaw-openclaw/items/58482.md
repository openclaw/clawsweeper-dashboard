The PR adds optional AbortSignal parameters through memory-host batch runner/provider helpers and forwards them into polling sleeps and some HTTP requests for OpenAI, Gemini, and Voyage batch embeddings.

## Best Possible Solution

Retarget cancellation into `MemoryEmbeddingBatchOptions`, the memory-core batch timeout path, provider adapters, and the live provider-plugin batch files using an abort-safe shared helper plus focused contract/tests and redacted real-run proof.

## Reproduction Assessment

Yes, at source level. Current main still has no AbortSignal in `MemoryEmbeddingBatchOptions` or the `batchEmbed` call, and live provider batch polling sleeps/fetches are not wired to cancellation; I did not run a live disconnect scenario.

## Solution Assessment

No. The direction is right, but this branch edits stale package-local provider files and uses an incomplete abortable sleep pattern; the maintainable fix belongs in the current memory batch contract and provider-plugin batch runtimes.
