## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Thread the signal from the public batch contract:** `packages/memory-host-sdk/src/host/batch-runner.ts:10`
  - body: Adding `signal` only to this execution params type is not enough to make cancellation reachable in production. The current memory batch contract used by plugin adapters still has no signal field, so callers cannot pass a disconnect signal into the OpenAI/Voyage/Gemini batch runners and polling will continue with `params.signal` undefined.
  - confidence: 0.87
- **[P2] Remove abort listeners after polling sleeps resolve:** `packages/memory-host-sdk/src/host/batch-openai.ts:197-202`
  - body: Each polling iteration registers an abort listener on the same signal but never removes it when the timer fires normally. Long-running batches will retain one listener per poll and a later abort can invoke stale handlers, so the port should use a shared helper that checks `signal.aborted` first and removes the listener in the non-abort path across all providers.
  - confidence: 0.9
