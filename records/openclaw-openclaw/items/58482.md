  - body: Adding `signal` here does not make disconnect cancellation reachable from production memory batches.   - confidence: 0.89

  - body: The new polling sleep registers an abort listener on every interval but never removes it when the timer resolves normally, and it does not reject immediately when the signal was already aborted. Long polls can retain one listener per interval and still miss cancellation if the disconnect happened before this sleep started; use a shared helper that checks `signal.aborted` and removes the listener in `finally`.
  - confidence: 0.86
