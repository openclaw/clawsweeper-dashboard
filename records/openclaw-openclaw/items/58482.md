## Summary

Keep open: the cancellation problem is still present on current main, but this PR should not merge as-is because it targets stale memory-host provider files, does not wire cancellation through the current production memory batch contract, and still needs real behavior proof.

## What This Changes

The branch adds optional AbortSignal plumbing to the memory-host SDK batch runner and legacy OpenAI, Gemini, and Voyage batch helpers.

## Best Possible Solution

Add an optional AbortSignal through MemoryEmbeddingBatchOptions, pass it from memory-core into current provider runtimes, update OpenAI/Gemini/Voyage batch sleeps and HTTP calls with listener cleanup, and cover the paths with focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main has no signal field in MemoryEmbeddingBatchOptions, memory-core does not forward one, and OpenAI/Gemini/Voyage batch polling waits on uncancelable timers.

## Solution Assessment

No. The cancellation direction is right, but this branch targets stale provider files and misses the current production contract; the safer fix should land against the current extensions/*/embedding-batch.ts implementations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread the signal from the production batch API:** `packages/memory-host-sdk/src/host/batch-runner.ts:10`
  - body: Adding signal here does not make disconnect cancellation reachable. Current main's MemoryEmbeddingBatchOptions and memory-core batchEmbed call path still have no signal to provide, and the provider batch implementations now live under extensions/*/embedding-batch.ts, so normal OpenAI/Gemini/Voyage batch runs continue without cancellation.
  - confidence: 0.91
- **[P2] Clean up abort listeners after polling sleeps:** `packages/memory-host-sdk/src/host/batch-openai.ts:192-201`
  - body: Each new polling sleep registers an abort listener but never removes it when the timer resolves normally, and already-aborted signals are not rejected before sleeping. Long polls can retain stale listeners and still miss cancellation; use an abortable sleep that checks signal.aborted and removes the listener on the non-abort path.
  - confidence: 0.9
- **[P2] Pass the signal into Gemini status polling:** `packages/memory-host-sdk/src/host/batch-gemini.ts:160`
  - body: fetchGeminiBatchStatus gains a signal parameter, but waitForGeminiBatch does not pass params.signal into the status request. If a disconnect happens during an in-flight Gemini status poll, the worker can still wait on the network call instead of stopping promptly.
  - confidence: 0.84
