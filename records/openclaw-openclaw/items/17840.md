## Summary

Keep open. Current main still lacks an opt-in reaction-triggered wake/turn contract; bundled reaction paths remain passive system-event notifications, and the recent discussion is explicitly waiting on maintainer API direction.

## What This Changes

Request a default-off mechanism that lets accepted reaction events wake the agent for a turn, either by per-message opt-in or channel/config scope.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing reaction filters accept the event, with schema, docs, generated metadata, and focused tests updated consistently.

## Reproduction Assessment

Yes for the current passive-behavior gap by source inspection: accepted reactions under existing `reactionNotifications` paths enqueue system events but do not call `requestHeartbeat` or start a turn. I did not run a live channel event, and the requested wake behavior is a new opt-in feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API. The requested opt-in direction is reasonable, but the maintainable fix should settle shared versus channel-first semantics before wiring heartbeat wakes into bundled reaction handlers.
