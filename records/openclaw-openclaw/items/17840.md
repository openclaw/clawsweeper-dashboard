## Summary

Keep open. Current main and v2026.5.20 still treat accepted reaction events as passive queued system events, while the opt-in reaction wake contract is only present in open PR work and still needs a maintainer product/API decision.

## What This Changes

Request a default-off config or per-message mechanism that lets accepted channel reaction events wake the agent for interactive workflows.

## Best Possible Solution

Adopt or supersede the open policy PRs with one default-off shared reaction wake contract that wakes only authorized, accepted reactions after successful enqueue and updates schemas, docs, generated metadata, and focused channel tests.

## Reproduction Assessment

Yes, by source inspection: accepted Discord, Telegram, Slack, and Signal reaction handlers enqueue system events while the heartbeat wake API remains separate on current main. This proves the requested opt-in behavior is absent, though it is a feature/API gap rather than a broken documented default.

## Solution Assessment

Unclear until maintainers choose the public contract. The shared default-off notification wake direction in https://github.com/openclaw/openclaw/pull/83611 looks maintainable, but it still needs review, replacement, or explicit product approval before this issue can close.
