## Summary

Keep open. Current main still lacks a default-off reaction-triggered wake/turn contract; reaction-capable channel handlers enqueue accepted reactions as passive system events, and the newest substantive discussion is asking maintainers to choose a shared cross-channel API versus a Discord-first shape.

## What This Changes

Request a default-off opt-in mechanism so accepted emoji reactions can wake an agent turn for lightweight confirm/cancel, polling, and workflow actions.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing channel access and reaction notification filters accept the event, with schema metadata, docs, and focused channel tests updated together.

## Reproduction Assessment

Yes, by source inspection: accepted Discord, Slack, Telegram, Signal, and Matrix reaction paths enqueue system events, while no opt-in trigger config or reaction-path heartbeat call exists on current main. I did not run a live channel event, and the requested behavior is a new opt-in feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is reasonable, but the maintainable fix should settle shared versus channel-first semantics before wiring requestHeartbeat into bundled reaction handlers.
