## Summary

Keep open. Current main still lacks an opt-in reaction-triggered wake/turn contract; bundled reaction handlers enqueue accepted reactions as passive system events, and the latest discussion is waiting on maintainer direction for a shared versus channel-first API.

## What This Changes

Request a default-off opt-in mechanism for reaction events to start agent turns instead of only being queued as system events.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing reaction filters accept the event, with schemas, generated metadata, public channel docs, and focused tests updated consistently.

## Reproduction Assessment

Yes for the current passive-behavior gap by source inspection: accepted reactions under existing reactionNotifications paths enqueue system events but do not call requestHeartbeat or start a turn. I did not run a live channel event, and the requested wake behavior is a new opt-in feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is reasonable, but the maintainable fix should settle shared versus channel-first semantics before wiring heartbeat wakes into bundled reaction handlers.
