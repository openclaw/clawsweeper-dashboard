## Summary

Keep open. Current main still lacks an opt-in reaction wake contract, and the remaining work is a cross-channel product/API feature rather than a stale or already-implemented cleanup item.

## What This Changes

The issue requests an opt-in mechanism for accepted reaction events to wake the agent and start an agent turn.

## Best Possible Solution

Add an additive, default-off reaction turn trigger contract in shared config/metadata or the message API, then wire accepted bundled-channel reactions to the heartbeat wake primitive only after existing access and notification filters pass.

## Reproduction Assessment

Yes. A high-confidence source-level path is to enable an existing reactionNotifications mode and deliver an accepted reaction: inspected handlers enqueue a system event but do not request a heartbeat wake.

## Solution Assessment

Unclear. The default-off opt-in direction is sound, but maintainers still need to choose the public contract; unconditional wake-on-reaction would not be the safest maintainable fix.
