## Summary

Keep open. Current main still lacks a default-off reaction-triggered wake/turn contract; accepted reactions are still queued as passive system events, and the current discussion is waiting on maintainer API scope rather than cleanup.

## What This Changes

Request an opt-in mechanism that lets accepted reaction events wake an agent turn, either per message or through channel/config-level reaction trigger scope.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing reaction filters accept the event, with schemas, generated metadata, public channel docs, and focused tests updated consistently.

## Reproduction Assessment

Yes for the current passive-behavior gap by source inspection: accepted reactions under existing reactionNotifications paths enqueue system events but do not call requestHeartbeat or start a turn. This is a new opt-in feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is reasonable, but the maintainable fix should settle shared versus channel-first semantics before wiring heartbeat wakes into bundled reaction handlers.
