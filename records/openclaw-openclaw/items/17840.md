## Summary

Keep open. Current main still lacks a documented opt-in reaction wake/turn contract, and the open related pull requests only cover narrower Telegram slices while the hook issue tracks a different plugin API surface.

## What This Changes

Request an opt-in mechanism, such as a config mode or per-message flag, that lets accepted reaction events trigger an agent turn instead of remaining passive system events.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing reaction filters accept the event, with schemas, generated metadata, public channel docs, and focused tests updated consistently.

## Reproduction Assessment

Yes by source inspection for the current passive-behavior gap: accepted reaction handlers enqueue system events and do not request a heartbeat or start a turn. This is a feature/API request rather than a broken existing public contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is reasonable, but the maintainable solution should settle shared versus channel-specific semantics before wiring heartbeat wakes into bundled reaction handlers.
