## Summary

Keep open. Current main still lacks an opt-in reaction-triggered wake/turn contract, and the open related PRs/issues cover narrower Telegram or Signal slices rather than superseding this cross-channel product/API request.

## What This Changes

Request an opt-in per-message, config, or channel mechanism that lets accepted reaction events wake an agent turn across reaction-capable channels.

## Best Possible Solution

Define a default-off reaction turn trigger contract, then wake only the routed session after existing reaction filters accept the event, with schemas, generated metadata, public channel docs, and focused tests updated consistently.

## Reproduction Assessment

Yes for the current passive-behavior gap by source inspection: accepted reactions under existing reactionNotifications paths enqueue system events but do not call requestHeartbeat or start a turn. The requested opt-in trigger contract is new feature work rather than a broken existing public contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is reasonable, but the maintainable solution should settle shared versus channel-specific semantics before wiring heartbeat wakes into bundled reaction handlers.
