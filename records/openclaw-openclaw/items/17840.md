## Summary

Keep open. Current main still has notification-only reaction handling, no opt-in wake/turn contract, and the related open Telegram/Discord work is narrower rather than a shipped cross-channel solution.

## What This Changes

Requests a default-off opt-in mechanism that lets accepted channel reaction events wake the routed agent session instead of remaining passive system events.

## Best Possible Solution

Define a default-off shared reaction-trigger contract, then wake only the routed session after existing reaction filters accept the event, with schema, generated metadata, public docs, and focused channel tests updated consistently.

## Reproduction Assessment

Yes, by source inspection: accepted reaction handlers enqueue system events and the wake API remains separate on current main. This proves the requested opt-in behavior is absent, though the item is a feature/API request rather than a broken existing public contract.

## Solution Assessment

Unclear until maintainers choose the public API. The opt-in direction is coherent, but the maintainable fix should settle shared versus channel-specific semantics before wiring reaction handlers to heartbeat wakes.
