## Summary

Keep open: current main and v2026.6.6 do not implement the requested policy-backed same-thread approval evidence path, and the earlier same-title sibling is not a safer canonical because its body is truncated while this report preserves the full safety/config proposal.

## What This Changes

Request Skill Workshop apply to accept and record channel-scoped same-thread approval evidence for chat-originated proposals.

## Best Possible Solution

Design a generic approval-evidence contract around trusted turn-source binding and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the existing pending approval default.

## Reproduction Assessment

Yes at source level: current main stores only minimal proposal origin and requires plugin approval for lifecycle apply, with no channel-scoped approval evidence fields or tool parameters. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose the approval authority, storage/redaction model, and whether this belongs in generic plugin approval plumbing or Skill Workshop-specific policy.
