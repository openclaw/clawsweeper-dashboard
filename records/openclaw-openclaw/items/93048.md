## Summary

Keep open: current main and v2026.6.6 still require the existing plugin approval route for Skill Workshop lifecycle actions and do not provide the requested same-thread, auditable chat approval evidence path.

## What This Changes

The issue requests a policy-backed approval evidence path so chat-originated Skill Workshop proposals can be applied from trusted same-conversation approval messages.

## Best Possible Solution

Define a generic trusted turn-source approval-evidence contract with redacted audit metadata, then let Skill Workshop apply consume it behind explicit policy while preserving fail-closed pending approval defaults.

## Reproduction Assessment

Yes, at source level. Current main requires plugin approval for Skill Workshop apply under the default pending policy and has no stored or accepted same-thread chat approval evidence path; I did not live-reproduce the private chat workflow.

## Solution Assessment

Unclear as an implementation direction. The request is plausible, but maintainers need to decide whether the durable contract belongs in generic gateway/plugin approval plumbing or Skill Workshop-specific policy before a fix is safe.
