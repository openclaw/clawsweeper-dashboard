- `impact:security`: The request changes who may authorize applying generated skill content and how that approval is audited.

## Summary

Keep open: current main and v2026.6.6 still lack the requested policy-backed same-thread approval evidence path, and the remaining work is a security/product decision rather than conservative cleanup.

## What This Changes

The issue requests an auditable chat-originated approval evidence path for `skill_workshop action=apply` when the user has approved the proposal in the originating conversation.

## Best Possible Solution

Design a generic, auditable approval-evidence contract around trusted turn-source binding and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the pending approval default.

## Reproduction Assessment

Yes at source level: current main stores only minimal proposal origin and requires plugin approval for lifecycle apply, with no channel-scoped approval evidence fields, config, or tool parameters. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose the approval authority, storage/redaction model, replay rules, and whether this belongs in generic plugin approval plumbing or Skill Workshop policy.
