## Summary

Keep open: current main and the latest release still do not implement the requested policy-backed, same-thread chat approval evidence path for Skill Workshop apply. Related open work may fix route delivery and timeout symptoms, but this issue remains a maintainer product/security decision about approval authority, replay scope, redacted audit metadata, and whether the contract belongs in generic plugin approval plumbing or Skill Workshop policy.

## What This Changes

The issue requests an explicit, auditable chat-originated approval-evidence path so Skill Workshop apply can proceed when the user approved in the originating conversation but no separate plugin approval route is available.

## Best Possible Solution

Define a generic approval-evidence contract for chat-originated lifecycle actions, then wire Skill Workshop apply to consume it without weakening the existing pending approval default.

## Reproduction Assessment

Yes at source level: current main requires plugin approval for Skill Workshop lifecycle actions, maps no-route approval to the reported error, and has no proposal metadata or config for chat-originated approval evidence. I did not run a live private chat workflow, so this is not marked reproduced.

## Solution Assessment

Unclear: the requested direction is plausible, but a Skill Workshop-only config shape is not clearly the best fix until maintainers choose the generic approval-evidence, authority, replay, and audit model.
