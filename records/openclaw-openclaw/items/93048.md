## Summary

Keep open: current main and v2026.6.6 still route Skill Workshop lifecycle apply through the plugin approval prompt path and do not implement the requested policy-backed same-thread approval evidence/audit contract.

## What This Changes

The issue requests a policy-backed channel-scoped approval evidence path for chat-originated Skill Workshop apply actions.

## Best Possible Solution

Design a generic approval-evidence contract around trusted turn-source binding and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the existing pending approval default.

## Reproduction Assessment

Yes at source level: current main requires plugin approval for Skill Workshop lifecycle actions unless `approvalPolicy` is `auto`, and the apply path cannot accept or persist chat approval evidence. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose the approval authority, storage/redaction model, replay window, and whether this belongs in generic gateway/plugin approval plumbing or Skill Workshop-specific policy.
