- `impact:security`: The request changes who or what may authorize applying generated skill content and how that approval is audited.

## Summary

Keep open: current main and v2026.6.6 still do not implement a policy-backed same-thread/chat approval-evidence path, and the reporter’s follow-up makes this a maintainer product/security decision rather than a duplicate of the lower-level approval-route bugs.

## What This Changes

The issue requests a first-class, auditable channel-scoped approval-evidence path so chat-originated Skill Workshop apply actions can use trusted same-thread owner/admin approval instead of a separate plugin approval card.

## Best Possible Solution

Design a generic approval-evidence contract around trusted turn-source binding, replay limits, authority checks, and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the pending approval default.

## Reproduction Assessment

Yes at source level: current main stores only minimal proposal origin, exposes only pending/auto approval policy, and applies proposals without approval-evidence fields. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: channel-scoped approval evidence is plausible, but maintainers need to choose the approval authority, replay/redaction model, and whether this belongs in generic plugin approval plumbing or Skill Workshop-specific policy.
