## Label Justifications

- `P2`: This is a meaningful Skill Workshop approval improvement with bounded surface, but it needs security/product design rather than emergency handling.
- `impact:security`: The request changes who may authorize applying generated skill content and how that approval is audited.
- `impact:message-loss`: The reported failure happens when the approval route or card cannot be used from the originating chat thread.

## Summary

Keep open: current main and the latest release still lack the requested policy-backed same-thread approval evidence path; the related open work covers approval-route delivery and timeout symptoms, not the broader auditable approval contract requested here.

## What This Changes

The issue requests a first-class, channel-scoped approval-evidence path so an authorized same-thread chat approval can satisfy Skill Workshop apply for chat-originated proposals.

## Best Possible Solution

Design a generic approval-evidence contract around trusted turn-source binding, authorized principals, replay limits, and redacted audit metadata, then let Skill Workshop apply consume it without weakening the default pending approval posture.

## Reproduction Assessment

Yes at source level: current main has minimal proposal origin metadata, pending/auto approval policy, and plugin approval routing, but no same-thread approval evidence fields or apply-time evidence consumption. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose whether the permanent fix belongs in generic gateway/plugin approval plumbing, Skill Workshop-specific policy, or both. A direct text-to-apply bypass would not be acceptable without the authority, replay, and audit design.
