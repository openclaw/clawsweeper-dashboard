## Summary

Keep open: current main and v2026.6.6 still lack the requested policy-backed same-thread approval evidence path, and the open related PRs address lower-level approval routing symptoms rather than this generic approval-evidence contract.

## What This Changes

Request for Skill Workshop apply to accept audited channel-scoped approval evidence from the originating chat thread instead of requiring a separate approval card.

## Best Possible Solution

Define a generic approval-evidence contract around trusted turn-source binding and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the default pending approval gate.

## Reproduction Assessment

Yes, at source level: current main stores only minimal proposal origin, requires plugin approval for lifecycle apply, and has no channel-scoped approval evidence fields or policy path. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose the approval authority, storage/redaction model, replay window, and whether the permanent owner is generic plugin approval plumbing or Skill Workshop policy.
