- `P2`: This is a meaningful Skill Workshop approval improvement with bounded surface, but it needs security/product design rather than emergency handling.
- `impact:security`: The request changes who may authorize applying generated skill content and how that authorization is audited.
- `impact:message-loss`: The reported failure mode is that approval in the originating chat thread cannot be routed to or consumed by the apply path.

## Summary

Keep open: current main and v2026.6.6 still lack a policy-backed path for Skill Workshop to consume same-thread/channel-scoped approval evidence, and the related open PRs only address route delivery, timeouts, or managed-skill targets rather than the requested approval-evidence contract.

## What This Changes

Skill Workshop apply is requested to accept auditable same-thread/channel-scoped approval evidence for chat-originated proposals.

## Best Possible Solution

Define a generic approval-evidence contract for trusted turn-source binding and redacted audit metadata, then wire Skill Workshop apply to consume it without weakening the existing pending approval default.

## Reproduction Assessment

Yes, at source level: current main records only minimal proposal origin and requires plugin approval for lifecycle apply, with no channel-scoped approval-evidence fields, policy, or apply parameters. I did not reproduce the private chat workflow live.

## Solution Assessment

Unclear as requested: the direction is plausible, but maintainers need to choose approval authority, replay binding, redaction/storage, and whether the contract belongs in generic plugin approval plumbing or Skill Workshop-specific policy.
