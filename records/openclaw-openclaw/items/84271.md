## What This Changes

The issue reports that `message.send` can successfully deliver to Slack but return a generic post-processing middleware error to the model, obscuring delivery success.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84292 or an equivalent narrow fallback that preserves safe delivery identifiers for successful sends while keeping other middleware failures fail-closed.

## Reproduction Assessment

Yes from source, not by executing a live Slack run: current main and v2026.6.8 replace thrown/invalid middleware outcomes with the generic middleware error result after a tool result exists.

## Solution Assessment

Yes, the linked narrow side-effect-safe fallback or equivalent is the best direction; broad pass-through of rejected middleware output would weaken the middleware safety contract.
