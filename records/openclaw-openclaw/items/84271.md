## What This Changes

The issue requests that `message.send` preserve successful Slack delivery results when tool-result middleware fails after the send.

## Best Possible Solution

Preserve a safe minimal success result for successful `message.send` side effects while keeping middleware fail-closed behavior for unsafe or unknown tool results.

## Solution Assessment

Yes, the maintainable fix is a narrow side-effect-safe fallback for `message.send`, not broad pass-through of rejected middleware output. The existing middleware safety contract should remain the default for other tools.
