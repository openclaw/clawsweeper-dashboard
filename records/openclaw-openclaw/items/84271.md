- `impact:message-loss`: A successful Slack send can lose its delivery receipt/status in the model-visible result and look like a failed send.

## Summary

Keep open: current main and v2026.6.8 still replace thrown or invalid post-processing middleware results with the generic middleware error after tool execution, and the linked implementation PR is open rather than merged.

## What This Changes

The issue requests preserving successful Slack `message.send` delivery identifiers when tool-result middleware fails after the message has already been delivered.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84292 or an equivalent narrow fallback that preserves safe delivery identifiers for successful sends while keeping unrelated middleware failures fail-closed.

## Reproduction Assessment

Yes from source, not by executing a live Slack run: current main and v2026.6.8 run the tool before middleware and return the generic middleware error result if post-processing throws or returns an invalid candidate.

## Solution Assessment

Yes, the linked narrow side-effect-safe fallback or equivalent is the best direction; broad pass-through of rejected middleware output would weaken the middleware safety contract.
