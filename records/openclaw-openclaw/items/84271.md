The issue asks OpenClaw to preserve successful `message.send` delivery identifiers when tool-result post-processing fails after Slack accepted the message.

## Best Possible Solution

## Solution Assessment

Yes. The narrow maintainable fix is to preserve a safe minimal delivered `message.send` result while keeping generic fail-closed middleware behavior for unknown or unsafe tool results.
