## Summary

Keep open: current `main` still has the post-delivery failure path, and the linked implementation PR is open and unmerged, so this issue should stay canonical until that fix or a replacement lands.

## What This Changes

The issue reports that a successful Slack `message.send` can be returned to the model as a generic tool-result middleware error after delivery.

## Best Possible Solution

Land a narrow fix in https://github.com/openclaw/openclaw/pull/84292 or a replacement that preserves a safe minimal delivered-send result while keeping generic middleware failures fail-closed for other tools.

## Reproduction Assessment

Yes. Source inspection on current `main` gives a high-confidence path: Slack delivery identifiers are produced before Codex tool-result middleware, and thrown or invalid middleware still replaces the result with the generic middleware error.

## Solution Assessment

Yes for the requested direction. The maintainable solution is a narrow delivered-message fallback that preserves only safe delivery identifiers and warnings, not a broad pass-through of middleware-rejected raw output.
