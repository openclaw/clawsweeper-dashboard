## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76529. In Telegram direct preview streaming, a short-lived preview can be created before visible intermediate/tool-step replies, and finalizing that old preview with editMessageText can leave the final answer above later bubbles. Add a focused regression in `extensions/telegram/src/bot-message-dispatch.test.ts` or `extensions/telegram/src/lane-delivery.test.ts` that simulates an active answer preview, visible intermediate delivery after it, then final delivery; expect the final conclusion to be delivered after the intermediate messages rather than by editing the older preview. Prefer a narrow Telegram-owned fix that tracks visible delivery after the active preview and routes the final through fresh send plus preview cleanup, similar to the existing long-lived preview path. Do not change shared dispatcher ordering, Telegram access/config, provider replay logic, or add a new config option. Add a user-facing changelog entry and update `docs/concepts/streaming.md` if the preview-final policy text changes.
