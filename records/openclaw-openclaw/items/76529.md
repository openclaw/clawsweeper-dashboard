## Summary

Keep open. Current main still has a source-level Telegram preview-finalization path where a short-lived preview can be edited into the final answer after a visible non-final delivery, and #76725 is an open closing PR rather than a merged fix.

## What This Changes

Telegram direct replies can display a final answer above later visible tool or block bubbles when streamed fragments are delivered quickly.

## Best Possible Solution

Review and land or reject #76725; the fix should keep Telegram final answers visually after visible non-preview tool/block bubbles without changing shared dispatcher ordering.

## Reproduction Assessment

Yes, at source level: current main can deliver a visible block/error bubble and then finalize the answer by editing the older preview. I did not establish a live Telegram run in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is a Telegram preview-finalization fix like #76725, because shared reply dispatch already serializes tool, block, and final sends.
