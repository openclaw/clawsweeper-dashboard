## Summary

Keep open. Current main still exposes reasoning visibility and thinking-effort controls only; I found no config key, protocol/session field, Control UI selector, docs entry, or runtime prompt/subscription path for choosing the language of visible reasoning output.

## What This Changes

The issue requests a per-agent or global option such as `agents.defaults.reasoningLanguage` so visible reasoning can be shown in a chosen language while leaving the final response language unchanged.

## Best Possible Solution

Add a product-approved reasoning-language preference only after deciding scope: config schema/types/docs, optional session or UI override, runtime/provider plumbing, and tests proving final answer language stays independent while reasoning visibility remains opt-in.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs/UI search against current main, which shows no accepted reasoning-language field or runtime language plumbing.

## Solution Assessment

Unclear. The proposed `agents.defaults.reasoningLanguage` shape is plausible, but the best maintainable solution needs maintainer decisions on per-agent versus per-session scope, provider limitations, UI placement, and safe reasoning exposure policy.
