## Summary

Close as ClawHub/plugin scope. Current main keeps `/reasoning` as a visibility-only core control, while existing plugin config plus `before_prompt_build` system-context hooks are sufficient for optional reasoning-language prompt guidance without adding a new OpenClaw core setting.

## What This Changes

The issue requests a config option such as `agents.defaults.reasoningLanguage` so visible reasoning can be emitted in a chosen language while final response language remains independent.

## Best Possible Solution

Keep core `/reasoning` as the visibility control and let a ClawHub/code plugin provide configurable reasoning-language prompt guidance through the existing plugin config and prompt-build hook surfaces; revisit core only if a concrete missing plugin API appears.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs/UI search showing no core reasoning-language field, paired with plugin API inspection showing an external implementation path.

## Solution Assessment

Yes for repository cleanup. ClawHub/plugin scope is the narrowest maintainable direction because it avoids provider-variable prompt shaping in core while using existing config and `before_prompt_build` seams.
