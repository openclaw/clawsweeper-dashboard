## Summary

Keep open: current main still has the MiniMax Anthropic-compatible usage propagation gap, and open PR #75136 is already the active closing implementation candidate. The source path matches the report: MiniMax uses the Anthropic stream transport, that transport only copies native Anthropic usage fields today, and downstream Usage UI totals can only display the zero assistant usage snapshot it receives.

## What This Changes

Fix MiniMax Anthropic-compatible usage propagation so the Control UI Usage page shows token totals instead of only message counts.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow transport-boundary fix that normalizes MiniMax-compatible usage before assistant snapshots and session persistence. The Control UI should keep consuming stored session usage rather than infer token totals from provider identity or message count.

## Reproduction Assessment

Yes. The issue provides exact setup details and raw nonzero provider usage, and current main can be reproduced with a deterministic Anthropic SSE fixture containing prompt_tokens/completion_tokens or message_stop usage that the transport currently ignores.

## Solution Assessment

Yes. Normalizing compatible-provider usage at the Anthropic stream transport boundary is the narrow maintainable solution; a Control UI-only workaround would leave session logs, status summaries, and cost totals wrong.
