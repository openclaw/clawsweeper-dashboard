## Summary

Keep open: current main still drops MiniMax Anthropic-compatible token usage in the Anthropic stream transport, and PR #75136 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue reports that MiniMax Anthropic-compatible sessions persist zero token usage, leaving the Control UI Usage page with message counts but no token totals.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow fix that normalizes compatible-provider stream usage in the Anthropic transport before session persistence.

## Reproduction Assessment

Yes, source-reproducible. MiniMax uses the Anthropic stream transport, and current main ignores the reported OpenAI-style usage aliases before `lastAssistant.usage` and session totals can be populated.

## Solution Assessment

Yes. Fixing the Anthropic stream transport to apply shared usage normalization is the narrow maintainable solution; a Control UI-only change would leave transcript/session accounting wrong.
