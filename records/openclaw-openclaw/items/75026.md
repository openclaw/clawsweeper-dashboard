## Summary

Keep open: current main still has the MiniMax Anthropic-compatible usage propagation gap, and open PR #75136 is already the active implementation candidate with closing syntax for this report.

## What This Changes

MiniMax Anthropic-compatible calls should persist provider token usage so the Control UI Usage page shows token totals, not only message counts.

## Best Possible Solution

Review, rebase if needed, and land PR #75136 or an equivalent narrow transport-boundary fix that normalizes MiniMax-compatible usage before assistant snapshots and session persistence. The Usage page should continue consuming stored session usage rather than inferring tokens from provider identity or message counts.

## Reproduction Assessment

Yes. The report gives a concrete MiniMax CN Anthropic-compatible setup, and current main can be reproduced with a deterministic Anthropic SSE fixture carrying `prompt_tokens`/`completion_tokens` or `message_stop` usage that the transport currently ignores.

## Solution Assessment

Yes. Fixing usage normalization at the Anthropic stream transport boundary is the narrow maintainable solution; a Control UI-only workaround would leave session logs, status summaries, and cost totals wrong.
