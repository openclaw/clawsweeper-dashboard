## Summary

Current main still appears to drop MiniMax Anthropic-compatible usage before assistant snapshots and session persistence, and open PR #75136 is the active closing implementation candidate. This issue should stay open until that PR, or an equivalent narrow fix, lands.

## What This Changes

MiniMax Anthropic-compatible runs should persist token usage so the Control UI Usage page shows nonzero token totals instead of only message counts.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow transport-boundary fix that normalizes compatible-provider usage from message_start, message_delta, and message_stop before assistant snapshots and session persistence. The Control UI should continue reading stored session usage rather than infer MiniMax-specific totals later.

## Reproduction Assessment

Yes. The issue includes concrete setup and raw nonzero MiniMax usage, and current main can be reproduced with an Anthropic SSE fixture containing MiniMax-style prompt_tokens/completion_tokens or message_stop usage that the transport currently ignores.

## Solution Assessment

Yes. Fixing the Anthropic stream transport is the narrow maintainable solution; a Control UI-only workaround would leave session logs, status summaries, compaction inputs, and cost totals wrong.
