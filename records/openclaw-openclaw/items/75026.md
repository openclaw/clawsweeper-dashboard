## Summary

Keep open: current main still loses MiniMax Anthropic-compatible token usage before session persistence, and PR #75136 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue reports that MiniMax Anthropic-compatible runs persist zero token usage in Control UI usage summaries while message counts are recorded.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow fix that normalizes compatible-provider stream usage in the Anthropic transport before session persistence.

## Reproduction Assessment

Yes, source-reproducible. The MiniMax Anthropic-compatible provider path is documented and current stream handlers ignore the reported `prompt_tokens`/`completion_tokens` aliases, which leaves `totalTokens` at zero before the Usage page reads persisted session totals.

## Solution Assessment

Yes. Fixing the Anthropic stream transport to apply shared usage normalization is the narrow maintainable solution; a Control UI-only change would leave transcript/session accounting wrong.
