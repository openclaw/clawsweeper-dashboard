Keep open: current main still drops MiniMax Anthropic-compatible usage aliases before session persistence, and open PR #75136 is the active closing implementation candidate.

## What This Changes

MiniMax Anthropic-compatible runs should persist nonzero token usage so the Control UI Usage page shows token counts as well as message counts.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow transport-boundary fix that normalizes compatible-provider usage before assistant snapshots and session persistence.

## Reproduction Assessment

Yes. The report includes concrete MiniMax CN Anthropic endpoint setup and raw nonzero usage, and current main can reproduce the drop with an Anthropic SSE fixture that emits `prompt_tokens`/`completion_tokens` or stop-time usage.

## Solution Assessment

Yes. Fixing `src/agents/anthropic-transport-stream.ts` is the narrow maintainable solution; a Control UI-only workaround would leave persisted session usage and downstream summaries wrong.
