## Summary

Keep open: current main still drops MiniMax Anthropic-compatible token usage before persistence, and #75136 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to persist and display MiniMax Anthropic-compatible token usage in the Control UI Usage page instead of recording zero tokens while message counts work.

## Best Possible Solution

Review and land #75136, or an equivalent narrow transport fix that normalizes compatible-provider stream usage before session persistence.

## Reproduction Assessment

Yes. This is source-reproducible because current MiniMax uses the Anthropic stream transport, while that transport ignores the reported alias fields and stop-event usage before persisted session totals are computed.

## Solution Assessment

Yes. The narrow maintainable fix is in Anthropic transport usage normalization before session/session-cost aggregation; a Control UI-only change would leave transcript accounting wrong.
