## Summary

Keep open: current main still drops MiniMax Anthropic-compatible token usage before session persistence, and PR #75136 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to persist and display MiniMax Anthropic-compatible token usage instead of showing zero tokens while message counts increase.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow transport fix that normalizes compatible-provider stream usage before session persistence.

## Reproduction Assessment

Yes. The current MiniMax Anthropic-compatible provider path reaches stream handlers that ignore the reported `prompt_tokens`/`completion_tokens` aliases, leaving persisted assistant/session totals at zero.

## Solution Assessment

Yes. Fixing the Anthropic stream transport to use shared usage normalization is the narrow maintainable solution; a Control UI-only change would leave transcript and session accounting wrong.
