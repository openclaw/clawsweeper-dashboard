## Summary

Keep open: current main still has the MiniMax Anthropic-compatible usage propagation gap, and open PR #75136 remains the active unmerged closing candidate. The report is concrete and matches the current source path: MiniMax text providers use the Anthropic stream transport, that transport only carries native Anthropic usage fields today, and downstream session/Control UI usage totals depend on the assistant usage snapshot it receives.

## What This Changes

The issue asks OpenClaw to persist MiniMax Anthropic-compatible token usage so the Control UI Usage page shows token totals instead of zero while message counts continue to work.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow fix in the Anthropic stream transport that normalizes compatible-provider usage aliases before assistant snapshots and session persistence. The Control UI should continue reading stored session usage instead of inferring token totals from provider identity or message counts.

## Reproduction Assessment

Yes. The issue gives exact setup details and raw nonzero MiniMax usage, and current main can be reproduced with a deterministic Anthropic SSE fixture containing prompt_tokens/completion_tokens or compatible message_stop usage that the transport currently ignores.

## Solution Assessment

Yes. Normalizing MiniMax-compatible usage at the Anthropic transport boundary is the narrow maintainable solution; a Control UI-only workaround would leave session logs, status summaries, and cost totals wrong.
