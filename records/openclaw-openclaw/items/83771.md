## What This Changes

This PR adds an optional `OPENCLAW_DOCKER_PIP_PACKAGES` Docker build argument, installs `python3-pip` when needed, runs `pip3 install`, and passes/persists the value through `scripts/docker/setup.sh`.

## Best Possible Solution

Keep the default image unchanged, but align the opt-in pip hook with the repository’s accepted image-build naming/docs/test pattern before merge.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new opt-in Docker build capability, not broken existing behavior. The useful verification path is a local image build with the build arg and an import check inside the resulting container.
