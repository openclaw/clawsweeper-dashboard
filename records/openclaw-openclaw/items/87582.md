## Summary

Keep this PR open: the central append fix is needed, but the current head can still persist an invalid signature when a thinking block starts with a signature and later receives `signature_delta` events.

## What This Changes

The PR changes the embedded Anthropic transport stream to append `signature_delta` chunks, updates transport-stream assertions, and adds a live proof script for thinking-signature replay.

## Best Possible Solution

Keep the focused parser fix, but make signature deltas replace any seeded start signature on the first delta for that block and append only subsequent delta chunks, with tests for both start-only and multi-delta streams.

## Reproduction Assessment

Yes. Current main's source path clearly overwrites repeated signature_delta chunks, and the PR body supplies terminal proof for the patched empty-start multi-delta path.

## Solution Assessment

No. Appending signature_delta chunks is the right direction, but the current implementation should not append them to a pre-seeded start signature because that can persist a replay-invalid combined value.
