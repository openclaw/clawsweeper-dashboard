This PR adds a win32 platform branch and test seam to `resolvePreferredOpenClawTmpDir`, uses a Windows path joiner for injected win32 fallback paths, adds resolver regression tests, and adjusts a WhatsApp media test that mocks `process.platform`.

## Best Possible Solution

Land the resolver-level win32 skip with the regression tests, keep the non-Windows POSIX temp behavior unchanged, and add one credited Unreleased Fixes changelog line for the Windows temp-root correction.

## Reproduction Assessment

Yes. Current main can reproduce through the resolver's injected `lstatSync` and `accessSync` seams: when `/tmp/openclaw` is reported available, `resolvePreferredOpenClawTmpDir` returns it before fallback, while `resolveGatewayLockDir` independently builds from `os.tmpdir()`.

## Solution Assessment

Yes, with the changelog added. The resolver is the narrowest maintainable boundary because logs, TTS/media temp files, plugin helpers, and media roots share it; changing every consumer separately would be broader and easier to miss.
