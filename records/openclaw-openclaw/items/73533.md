## Summary

Keep open. Current main still has the POSIX-first temp-dir behavior, and this PR is the narrow implementation candidate for the Windows split-temp bug, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes `resolvePreferredOpenClawTmpDir` to skip the POSIX `/tmp/openclaw` preference on win32, adds platform-injected regression tests, and adjusts a WhatsApp media test that mocks win32.

## Best Possible Solution

Land the resolver-level Windows skip with its regression tests, preserve non-Windows POSIX temp behavior, and add a credited Unreleased Fixes changelog line before merge.

## Reproduction Assessment

Yes. Current main can be reproduced through the resolver's injected `lstatSync`/`accessSync` seams: when `/tmp/openclaw` is reported available, it returns that path before the fallback while the gateway lock helper independently uses `os.tmpdir()`.

## Solution Assessment

Yes, with the changelog added. The shared resolver is the narrow maintainable boundary because logs, media/temp helpers, and plugin-facing temp APIs use it; changing each consumer separately would be broader and easier to miss.
