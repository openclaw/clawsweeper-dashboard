## What This Changes

The PR adds a win32 platform seam and fallback branch to the temp-dir resolver, uses a Windows joiner for injected win32 fallback paths, adds resolver regression tests, and adjusts a WhatsApp media test that mocks win32.

## Best Possible Solution

Land the shared resolver-level Windows fallback with its regression tests, preserve non-Windows `/tmp/openclaw` behavior, and add the single-line Unreleased Fixes changelog entry before merge.

Yes. The current-main bug is reproducible through the resolver's injected `lstatSync` and `accessSync` seams by reporting `/tmp/openclaw` as available; current main then returns that path while the gateway lock helper independently uses `os.tmpdir()`.

## Solution Assessment

Yes, with the changelog added. The resolver is the narrow maintainable boundary because logs, temp downloads, TTS/media helpers, and plugin-facing temp APIs share it; changing individual consumers would be broader and easier to miss.
