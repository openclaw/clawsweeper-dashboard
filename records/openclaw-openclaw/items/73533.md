## Best Possible Solution

Land the resolver-level Windows skip with its regression tests, preserve non-Windows POSIX temp behavior, and add a credited Unreleased Fixes changelog line before merge.

Yes. Current main can be reproduced through the resolver's injected `lstatSync` and `accessSync` seams: when `/tmp/openclaw` is available it returns that path, while the gateway lock helper independently uses `os.tmpdir()`.

## Solution Assessment

Yes, with the changelog added. The resolver is the narrow maintainable boundary because logs, temp downloads, media, TTS, and plugin-facing temp helpers share it; fixing individual consumers would be broader and easier to miss.
