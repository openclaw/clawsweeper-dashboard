## Summary

Keep open: current main and the latest release still have a Codex app-server path that can serialize OpenClaw runtime/workspace prompt context into `turn/start.input`, while the linked fix remains an open draft PR with conflicts rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to keep per-turn Codex runtime context out of native Codex user history so repeated turns do not accumulate prior runtime-context blocks in later model input.

## Best Possible Solution

Land or replace the linked fix with a conflict-resolved change that leaves only the real user request in `turn/start.input`, sends current-turn OpenClaw context through Codex transient/additional-context semantics, and proves multi-turn replay plus compaction do not retain historical wrappers.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the current app-server code can still build the runtime-context wrapper, prepend it to the prompt, and serialize that text as a normal Codex user input item. I did not run a live multi-turn Codex app-server replay in this read-only review.

## Solution Assessment

Yes for the issue direction: using Codex's transient `additional_context` path, or an equivalent non-history turn-scoped path, is the narrowest maintainable fix. Current main is only partially mitigated, and the linked PR still needs conflict resolution and validation before this issue can close.
