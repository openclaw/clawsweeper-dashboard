## Summary

Keep open: current main still sends the decorated OpenClaw runtime-context block as ordinary Codex `turn/start` user input, and the reporter's structured live evidence shows that exact marker accumulating in model-visible user history and compaction inputs.

## What This Changes

The issue reports that Codex app-server prepends per-turn OpenClaw runtime context to native `turn/start` user input, causing repeated historical copies in later `response.create` and compaction requests.

## Best Possible Solution

Keep current-turn reference context available through a transient Codex app-server path or sanitize replayed decorated user items so persisted history contains only the real request, with multi-turn and compaction regression coverage.

## Reproduction Assessment

Yes for source-plus-live-log reproduction: current main decorates Codex `turn/start` input as a text user item, and the reporter's structured logs show the same marker accumulating in `role=user` `response.create` and compact inputs. I did not run a local native Codex session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to keep the runtime context current-turn-only or strip only historical decorated user items while preserving the request. Moving user-editable workspace or memory context wholesale into developer instructions is not the right boundary.
