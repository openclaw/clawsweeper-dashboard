## Summary

Keep open: current main still decorates Codex app-server turns with OpenClaw runtime context and sends the decorated text as normal `turn/start` user input, while existing tests and prompt snapshots encode that behavior. The related items are broader and still open; this report is a sharper current-main Codex history growth bug worth fixing rather than closing.

## What This Changes

The issue asks for Codex app-server runtime context to remain current-turn-only instead of being persisted and replayed as normal Codex user history.

## Best Possible Solution

Keep current-turn reference context available to Codex while ensuring only the real user request is persisted or replayed, with regression coverage for multi-turn native threads and compaction/replay.

## Reproduction Assessment

Partially: there is a high-confidence source path showing the decorated context is sent as normal Codex user input, but I did not run a live multi-turn Codex session to count `response.create` copies. A focused mocked native-thread test or live app-server trace should reproduce the growth.

## Solution Assessment

Yes: the maintainable fix is to make this context current-turn-only or sanitize historical decorated user messages while preserving the real request. Promoting all workspace/memory context to developer instructions would be a weaker and riskier solution.
