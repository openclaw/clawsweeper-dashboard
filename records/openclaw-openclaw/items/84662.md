## Summary

Keep open: current main still prepends the per-turn OpenClaw runtime-context block into normal Codex `turn/start` text input, and the reporter's structured live logs show those decorated user items accumulating in model-visible `response.create` and compact inputs.

## What This Changes

The issue asks OpenClaw to stop persisting Codex app-server per-turn runtime context as native user history across later turns.

## Best Possible Solution

Keep current-turn reference context available through a transient Codex app-server path, or strip only historical decorated user input before replay/compaction while preserving the real request.

## Reproduction Assessment

Yes. Source inspection on current main shows the marker is prepended into `turn/start` input as a normal text user item, and the reporter's structured logs show historical `role=user` `response.create` and compact items accumulating.

## Solution Assessment

Yes. The narrow fix is to make this context current-turn-only or sanitize historical decorated user text while preserving the request after `Current user request:`; moving user-editable context wholesale to developer instructions is not the right authority boundary.
