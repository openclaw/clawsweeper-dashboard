## Summary

Keep open: current main still wraps per-turn Codex runtime context into normal `turn/start` user input, and the added structured live evidence shows those decorated `role=user` items accumulating in `response.create` and compact inputs. The related reports are broader and still open, so this remains a narrow Codex session-state bug rather than cleanup-close material.

## What This Changes

The issue asks OpenClaw to keep Codex per-turn runtime context out of persisted native user history while preserving it for the active model call.

## Best Possible Solution

Keep current-turn reference context available through a transient Codex app-server path or sanitize replayed decorated user items so persisted history contains only the real request, with multi-turn and compaction regression coverage.

## Reproduction Assessment

Yes, source-plus-live-log reproducible: current main decorates Codex `turn/start` input as a text user item, and the reporter's structured logs show the same marker accumulating in `role=user` `response.create` and compact inputs. I did not run a local native Codex session in this read-only review.

## Solution Assessment

Yes: the best fix is to keep the runtime context current-turn-only or strip only historical decorated user items while preserving the request. Moving user-editable workspace or memory context wholesale into developer instructions is not the best path.
