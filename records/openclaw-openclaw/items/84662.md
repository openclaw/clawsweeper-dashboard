## Summary

Keep open: current main and the latest stable release still serialize the OpenClaw runtime-context wrapper into Codex turn input, and https://github.com/openclaw/openclaw/pull/94609 is an open draft/conflicting candidate fix rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to keep per-turn Codex runtime context available to the active model call without persisting it as ordinary native Codex user history.

## Best Possible Solution

Land a refreshed, mergeable fix that routes runtime context through a bounded current-turn Codex context seam or equivalent sanitizer, with regression coverage for multi-turn replay and compaction.

## Reproduction Assessment

Yes. Current main still builds the reported wrapper, passes it as `turn/start.input`, and serializes it as a text user input item; the reporter also supplied structured live logs showing the same marker accumulating as model-visible user history.

## Solution Assessment

Yes as a repair direction, but not yet implemented on main. Keeping untrusted runtime context out of the real user input via Codex `additionalContext` or an equivalent current-turn seam is a narrower fix than promoting user-editable workspace context to developer instructions.
