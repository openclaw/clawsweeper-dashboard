The issue asks OpenClaw to stop Codex app-server per-turn runtime context from being persisted and replayed as native Codex user history.

## Best Possible Solution

## Solution Assessment

Yes: the maintainable fix is to keep the runtime context current-turn-only or sanitize historical decorated user items while preserving the real request. Moving user-editable workspace or memory context wholesale into developer instructions is not the right path.
