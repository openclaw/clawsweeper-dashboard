## Summary

Keep open: current main and the latest release still route Feishu streaming-card DMs through raw reply/root metadata, and the matching closing PR is still open rather than merged.

## What This Changes

The issue reports that Feishu direct-message streaming replies are sent with reply metadata, showing a visible Reply-to label instead of top-level DM delivery.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix that filters streaming-card reply/root metadata for ordinary DMs while preserving group/topic and direct-thread behavior, then close this issue from the merged commit.

## Reproduction Assessment

Yes. Source inspection shows ordinary DM callers can set `skipReplyToInMessages`, but current main still passes raw reply/root metadata into `streaming.start()`, which selects reply or root-create delivery when those fields are present.

## Solution Assessment

Yes. Reusing the existing filtered reply/root decision in the streaming-card start path is the narrow maintainable fix; disabling `blockStreaming` or adding another config knob would be broader than the bug requires.
