## Summary

Keep open: current main and v2026.6.8 still pass the raw `replyToMessageId` into Feishu streaming-card startup, so ordinary DM streaming cards can still select reply mode. Two open closing PRs are candidate fixes, so this issue should remain the canonical tracking item until one lands.

## What This Changes

The issue asks Feishu direct-message streaming-card replies to omit `replyToMessageId` so they send as top-level DM messages without a visible Reply-to label.

## Best Possible Solution

Review and land one canonical linked PR that reuses the existing DM-aware `sendReplyToMessageId` for `streaming.start()` while preserving direct-thread and group/topic reply routing.

## Reproduction Assessment

Yes. From source, ordinary Feishu DMs set `skipReplyToInMessages`, but current main still passes the raw `replyToMessageId` into streaming-card start, which selects reply mode.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the already-computed `sendReplyToMessageId` into `streaming.start()` rather than adding config or disabling Feishu streaming globally.
