## Summary

Keep open: current main and the latest release still route ordinary Feishu DM streaming cards through the raw reply target, and three linked open PRs are already candidate fixes rather than merged implementation evidence.

## What This Changes

The issue asks Feishu DM streaming-card replies to omit reply metadata for ordinary direct chats while preserving intentional group/topic and P2P-thread replies.

## Best Possible Solution

Review and land one narrow linked fix, or an equivalent maintainer-owned patch, that passes sendReplyToMessageId into streaming.start and adds regression coverage for ordinary DM, group/topic, and P2P-thread cases.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set skipReplyToInMessages, but the streaming-card start path still passes the original replyToMessageId and therefore selects the reply API. I did not run a live Feishu DM in this read-only review.

## Solution Assessment

Yes, the best fix is narrow: reuse the already-computed sendReplyToMessageId for streaming-card start and cover ordinary DM plus preserved thread/group cases. Disabling streaming or adding a new config knob would be a workaround rather than the clean owner-boundary fix.
