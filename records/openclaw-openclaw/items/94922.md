## Summary

Keep open: current main and the latest stable release still pass raw `replyToMessageId` into Feishu streaming-card startup, while three open linked PRs are candidate fixes for the same issue.

## What This Changes

Feishu direct-message streaming-card replies should omit reply metadata so ordinary DMs are sent as top-level messages without a visible Reply-to label.

## Best Possible Solution

Land one canonical narrow dispatcher fix that passes the existing DM-aware reply target into `streaming.start` while preserving group/topic and P2P direct-thread routing, then close this issue from the merged PR.

## Reproduction Assessment

Yes, from source inspection. Ordinary Feishu DMs set `skipReplyToInMessages`, but current main still passes raw `replyToMessageId` into streaming-card startup, which selects reply mode.

## Solution Assessment

Yes. Reusing the already-computed `sendReplyToMessageId` for streaming startup is the narrow owner-boundary fix; disabling streaming or adding config would be broader than the bug requires.
