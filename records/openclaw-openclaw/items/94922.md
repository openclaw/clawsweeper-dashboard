## Summary

Keep open: current main still lets ordinary Feishu DM streaming cards use the original reply target even though non-streaming DM replies already omit it, so the reported Reply-to label remains source-reproducible and has no canonical merged fix.

## What This Changes

The issue asks OpenClaw's bundled Feishu plugin to send ordinary direct-message replies as top-level messages instead of reply-anchored messages when streaming cards are enabled.

## Best Possible Solution

Use the existing `skipReplyToInMessages` decision for streaming-card starts too, while preserving explicit/threaded reply metadata for Feishu P2P threads and group topics.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set `skipReplyToInMessages`, but `startStreaming()` still passes the original `replyToMessageId` to the streaming-card session. I did not run a live Feishu DM reproduction in this read-only review.

## Solution Assessment

Yes. The narrow fix is to route the already-computed visible-message reply target into streaming-card start, rather than adding config or disabling Feishu reply behavior globally.
