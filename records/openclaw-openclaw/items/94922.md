## Summary

Keep open: current main and the latest release still pass the raw Feishu reply target into streaming-card startup, and three linked candidate PRs are open but unmerged, so this issue should remain the canonical tracker until one fix lands.

## What This Changes

Feishu DM streaming-card replies should omit `replyToMessageId` so direct chats render as top-level messages without Reply-to labels.

## Best Possible Solution

Land one canonical narrow dispatcher fix that passes the existing DM-aware reply target into `streaming.start()` while preserving group/topic and P2P direct-thread reply metadata, then close this issue from the merged commit.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set `skipReplyToInMessages`, but current main still passes the raw `replyToMessageId` into streaming-card start, which selects reply mode. I did not run a live Feishu DM reproduction in this read-only review.

## Solution Assessment

Yes. Reusing the already-computed `sendReplyToMessageId` for `streaming.start()` is narrower than adding config, disabling streaming, or changing Feishu reply policy globally.
