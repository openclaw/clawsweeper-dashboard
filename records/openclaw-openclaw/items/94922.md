## Summary

Keep open: current main still has a source-reproducible Feishu DM streaming-card path that bypasses the existing direct-message skip-reply rule and can send cards through Feishu reply mode.

## What This Changes

The issue reports that Feishu direct-message replies are sent with `replyToMessageId` through the streaming card path, producing an unwanted reply label instead of a top-level DM response.

## Best Possible Solution

Use the existing `skipReplyToInMessages` decision for streaming-card starts too, while preserving explicit/threaded reply metadata for Feishu P2P threads and group topics.

## Reproduction Assessment

Yes from source: ordinary Feishu DMs set `skipReplyToInMessages`, but `startStreaming()` still passes the original `replyToMessageId` to the streaming-card session. I did not run a live Feishu DM reproduction in this read-only review.

## Solution Assessment

Yes. The best fix is a narrow dispatcher/streaming-card repair that uses the already-computed visible-message reply target, rather than adding config or disabling Feishu reply behavior globally.
