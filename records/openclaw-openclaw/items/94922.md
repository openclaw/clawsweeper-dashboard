## Summary

Keep open: current main still passes the raw Feishu reply target into streaming-card startup for direct-message replies, and three open PRs currently reference this issue as candidate fixes.

## What This Changes

The issue asks for Feishu direct-message streaming-card replies to be sent without reply metadata so 1:1 chats do not show a visible Reply-to label.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix that passes the existing DM-aware reply target into streaming.start() while preserving group/topic and P2P direct-thread reply metadata.

## Reproduction Assessment

Yes. Source inspection shows ordinary Feishu DMs request reply metadata suppression, but the streaming-card path still passes the raw replyToMessageId and therefore selects reply mode; I did not run a live Feishu DM reproduction.

## Solution Assessment

Yes. Reusing the already-computed sendReplyToMessageId for streaming.start() is narrower than adding config, disabling streaming, or changing Feishu reply policy globally.
