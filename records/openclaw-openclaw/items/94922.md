## Summary

Keep open: current main and `v2026.6.9` still pass raw Feishu reply metadata into streaming-card startup for ordinary DMs, while https://github.com/openclaw/openclaw/pull/95236 is the open candidate fix that should be reviewed before this issue closes.

## What This Changes

The issue requests that Feishu direct-message streaming-card replies be sent as top-level messages without reply metadata, avoiding the visible reply label in 1:1 chats.

## Best Possible Solution

Merge an equivalent narrow Feishu dispatcher fix that filters streaming-card reply and root metadata for ordinary DMs while preserving group/topic and direct-message thread replies.

## Reproduction Assessment

Yes. Source inspection shows ordinary Feishu DMs set `skipReplyToInMessages`, but current main passes raw `replyToMessageId` and `rootId` into streaming-card startup, which selects reply or root-create mode; I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Yes. Reusing the already-computed filtered reply/root metadata in streaming-card startup is the narrow maintainable fix; disabling streaming or adding a new config workaround would be broader than the bug requires.
