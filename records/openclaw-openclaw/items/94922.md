## Summary

Keep open: current main and v2026.6.10 still route Feishu streaming-card DMs through raw reply/root metadata, and the open linked PR is the active candidate fix that should land or be closed before this issue is resolved.

## What This Changes

The issue reports that Feishu direct-message streaming-card replies are sent with reply metadata, causing visible Reply-to labels instead of top-level DM messages.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix that filters streaming-card reply/root metadata through the existing DM-aware decision while preserving group/topic and P2P direct-thread routing, then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-level: ordinary Feishu DMs set skipReplyToInMessages, while current main and v2026.6.10 pass raw reply/root IDs to streaming-card startup where they select reply or root_create delivery. I did not run live Feishu delivery in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse the already-computed DM-aware reply/root decision in the streaming-card start path; disabling streaming or adding a new config knob would be broader than the bug requires.
