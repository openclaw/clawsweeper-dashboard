## Summary

Keep open: current main still passes the raw Feishu reply target into streaming-card startup, and three linked fix PRs are open but unmerged, so the issue should remain the canonical tracker until one fix lands.

## What This Changes

Feishu direct-message streaming-card replies should omit reply metadata so ordinary DM responses do not render as quoted replies.

## Best Possible Solution

Land one narrow dispatcher fix that uses the existing DM-aware reply target for streaming-card starts while preserving Feishu P2P direct-thread and group/topic reply metadata, then close this issue from the merged commit.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set `skipReplyToInMessages`, but current main still passes the original `replyToMessageId` into streaming-card start. I did not run a live Feishu DM reproduction in this read-only review.

## Solution Assessment

Yes. Reusing the already-computed `sendReplyToMessageId` for `streaming.start()` is narrower than adding config, disabling streaming, or changing Feishu reply policy globally; the remaining task is to review and land one linked PR.
