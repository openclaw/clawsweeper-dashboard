## Summary

Keep open: current main and v2026.6.9 still pass raw Feishu reply/root metadata into streaming-card startup for DMs, and https://github.com/openclaw/openclaw/pull/95236 is the active open candidate fix.

## What This Changes

Feishu DM streaming-card replies should be sent as top-level DM messages without visible Reply-to metadata.

## Best Possible Solution

Land one canonical Feishu dispatcher fix that filters replyToMessageId and rootId for ordinary DM streaming cards while preserving group/topic and P2P thread routing.

## Reproduction Assessment

Yes, at source level: ordinary Feishu DMs set skipReplyToInMessages, but current main passes raw reply/root IDs into streaming-card startup, where those fields select reply or root_create delivery. I did not run live Feishu credentials in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the already-computed DM-aware reply/root targets in streaming.start; disabling blockStreaming or adding a new config knob would be broader than needed.
