## Summary

Keep open: current main and the latest release still pass the raw Feishu reply target into streaming-card startup, and three linked candidate PRs are open but unmerged, so this issue remains the canonical tracker until one fix lands.

## What This Changes

The issue reports that Feishu direct-message streaming card replies use reply mode, causing a visible Reply-to label and slower delivery in DMs.

## Best Possible Solution

Land one canonical narrow dispatcher fix that passes the existing DM-aware reply target into streaming.start while preserving group/topic and P2P direct-thread reply metadata, then close this issue from the merged commit.

## Reproduction Assessment

Yes. Source inspection on current main shows ordinary Feishu DMs set skipReplyToInMessages, but streaming-card start still receives raw replyToMessageId and therefore selects reply mode.

## Solution Assessment

Yes. Reusing the already-computed sendReplyToMessageId for streaming.start is the narrowest maintainable fix; disabling streaming or adding config would be broader than the bug requires.
