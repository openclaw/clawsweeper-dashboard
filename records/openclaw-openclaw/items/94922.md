## Summary

Keep open: current main and the latest release still pass the raw Feishu `replyToMessageId` into streaming-card starts for ordinary DMs, and the two linked PRs are open candidate fixes rather than merged implementation.

## What This Changes

The issue reports that Feishu direct-message streaming/card replies carry `replyToMessageId`, producing visible Reply-to labels instead of top-level DM messages.

## Best Possible Solution

Land one linked fix or an equivalent narrow Feishu plugin change that passes the guarded visible-message reply target into `streaming.start`, with regression coverage for ordinary DMs, P2P threads, and group/topic replies.

## Reproduction Assessment

Yes. From source, ordinary Feishu P2P dispatch sets `skipReplyToInMessages`, but streaming-card start still receives the raw `replyToMessageId`, and the streaming session uses any reply target to select reply mode.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing guarded `sendReplyToMessageId` decision for streaming-card starts, rather than adding config or disabling Feishu streaming globally.
