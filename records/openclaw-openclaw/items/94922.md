## Summary

Keep open: current main and latest release still pass raw Feishu reply/root metadata into streaming-card startup, and https://github.com/openclaw/openclaw/pull/95236 is an open closing candidate rather than a merged fix.

## What This Changes

The issue asks for Feishu direct-message streaming-card replies to be sent as top-level messages without visible reply-mode metadata.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix that filters both streaming-card reply and root routing fields through the existing DM skip decision, then close this issue from the merged commit.

## Reproduction Assessment

Yes, source-level: ordinary Feishu DMs set `skipReplyToInMessages`, while current main still passes raw reply/root metadata into streaming-card startup where it selects reply or root_create delivery. I did not run live Feishu credentials in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse the existing DM-aware dispatcher decision for streaming-card `replyToMessageId` and `rootId`; disabling `blockStreaming` or adding a new config option would be broader than the bug requires.
