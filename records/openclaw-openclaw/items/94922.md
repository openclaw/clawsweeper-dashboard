Keep open: current main and latest release still pass raw Feishu streaming-card reply/root metadata for direct messages, and the issue is already paired with an open candidate fix PR.

## What This Changes

The issue reports that Feishu direct-message streaming replies are sent with reply metadata, causing visible Reply-to labels instead of top-level DM messages.

## Best Possible Solution

Review and land one canonical Feishu dispatcher fix that filters streaming-card reply/root metadata for ordinary DMs while preserving group/topic and direct-thread behavior.

## Reproduction Assessment

Yes, source-level: ordinary Feishu DMs pass the skip flag, but current main and v2026.6.10 still pass raw reply/root IDs into streaming-card startup where they select reply or root_create delivery. I did not run live Feishu credentials in this read-only review.

## Solution Assessment

Yes. Reusing the already-computed DM-aware reply target, plus filtering root metadata for ordinary DMs, is the narrowest maintainable fix; disabling `blockStreaming` is only a workaround.
