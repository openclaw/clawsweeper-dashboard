## Summary

Keep open: current main and the latest release still route Feishu streaming-card DMs through the raw reply target, and three open linked PRs are candidate fixes that should be reviewed before this issue closes.

## What This Changes

The issue requests that Feishu direct-message streaming-card replies omit reply metadata so DMs render as top-level messages while group and thread replies keep reply routing.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix that passes the existing DM-aware reply target into streaming.start while preserving group/topic and P2P direct-thread behavior, then close this issue from the merged commit.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set skipReplyToInMessages, but current main passes raw replyToMessageId into streaming-card startup, which selects reply mode. I did not run live Feishu delivery in this read-only review.

## Solution Assessment

Yes. Reusing the already-computed sendReplyToMessageId for streaming.start is the narrow maintainable fix; disabling blockStreaming or adding a new config knob would be broader than the bug requires.
