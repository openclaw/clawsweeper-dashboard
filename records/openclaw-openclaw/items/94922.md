## Summary

Keep open: current main and the latest release still route Feishu DM streaming-card starts through the raw reply target even though the dispatcher already has a DM-safe target for other send paths. Two open PRs are candidate fixes, so this issue should wait for one reviewed fix to land.

## What This Changes

The issue requests that Feishu direct-message streaming-card replies be sent as top-level messages without reply metadata, avoiding visible Reply-to labels in 1:1 chats.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix after proof or maintainer override confirms ordinary DM streaming cards lose the Reply-to label while P2P direct threads and group/topic replies remain threaded.

## Reproduction Assessment

Yes, from source: ordinary Feishu DMs set `skipReplyToInMessages`, but current main still passes raw `replyToMessageId` into streaming-card start, which selects Feishu reply mode. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Yes. Reusing the existing DM-aware `sendReplyToMessageId` decision for `streaming.start()` is narrower than adding config, disabling streaming, or changing Feishu thread policy globally.
