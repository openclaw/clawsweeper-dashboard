## Summary

Keep open: current main and v2026.6.10 still send Feishu streaming-card DMs with raw reply/root metadata, and the linked fix is still an open PR rather than merged implementation.

## What This Changes

Feishu direct-message streaming-card replies should avoid reply metadata so 1:1 chats do not show a Reply-to label.

## Best Possible Solution

Land one canonical narrow Feishu dispatcher fix, preferably https://github.com/openclaw/openclaw/pull/95236 after proof/test gaps are resolved, then close this issue from the merged commit.

## Reproduction Assessment

Yes. Source inspection shows ordinary Feishu DMs set the skip flag, but current main still passes raw reply/root metadata into streaming-card startup where it selects reply or root_create delivery.

## Solution Assessment

Yes. Reusing the existing DM-aware reply target, and filtering rootId consistently, is the narrow maintainable fix; disabling blockStreaming is only a workaround.
