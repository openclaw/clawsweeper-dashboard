## What This Changes

The PR changes media-understanding auto image-model selection so configured `agents.defaults.imageModel` is skipped when the active model appears vision-capable, with related vision-skip tests.

## Best Possible Solution

Fix the inbound `runCapability` native-vision guard by treating only explicit `tools.media.image.models` as an override and letting catalog support cover MiniMax, while preserving or explicitly versioning exported resolver behavior for plugin callers.

## Reproduction Assessment

Yes, source-level: current main bypasses native-vision skip when `agents.defaults.imageModel` is present, and the linked discussion also identifies the MiniMax hard exclusion on stock main. I did not run a live Feishu/Gateway reproduction in this read-only review.

## Solution Assessment

No. The runner guard direction is right for non-MiniMax models, but the patch leaves MiniMax catalog-supported vision models excluded and changes the exported auto-image resolver instead of keeping the skip decision local or adding an explicit new mode.
