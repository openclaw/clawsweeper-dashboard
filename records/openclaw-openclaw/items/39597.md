## Summary

Keep open. Current main still has the requested product gap: Gateway clients already send and store structured client metadata that can distinguish native macOS from browser Control UI, but the agent-facing runtime prompt only exposes channel and capability fields, while chat.send maps the client id/displayName only into sender metadata rather than a trusted runtime contract.

## What This Changes

Request to expose WebChat client type, such as native Mac app versus browser or mobile web, in agent runtime context.

## Best Possible Solution

Add a narrow trusted Gateway client-context contract in core using already structured connect.client metadata, expose only sanitized low-entropy fields such as client id, mode, platform, and possibly device family in deterministic runtime prompt fields, and document the privacy-safe contract for WebChat and Control UI.

## Reproduction Assessment

Yes. A source-level reproduction is clear: connect as openclaw-control-ui or openclaw-macos, send chat.send, and inspect the generated runtime prompt; current code can store those client fields but buildRuntimeLine has no client fields to render.

## Solution Assessment

Yes, the best direction is to reuse the existing connect.client protocol metadata rather than adding an X-Client-Type header or config flag. The safer implementation is a small sanitized runtime-context bridge with prompt and Gateway chat coverage.

## Repair Work Prompt

Implement trusted WebChat client-type runtime context for openclaw/openclaw. Current main already receives structured `connect.client` metadata, but agent runtime prompts only expose `channel=`/`capabilities=`. Add a narrow core path that sanitizes selected client fields from `connect.client` for WebChat/Control UI chat sends, carries them through chat.send and queued/follow-up run params into `RuntimeInfoInput`, and renders deterministic runtime fields such as `client=openclaw-macos`, `client_mode=ui`, and `client_platform=macos` or `web`. Do not expose raw `userAgent`, `displayName`, `instanceId`, or `modelIdentifier` unless a maintainer explicitly approves. Keep field ordering deterministic for prompt-cache stability. Add focused prompt tests and Gateway chat coverage showing browser Control UI and macOS client metadata reach the runtime line. Update `docs/web/webchat.md` and/or `docs/web/control-ui.md` for the privacy-safe contract. Because this is user-facing behavior, add a single-line CHANGELOG.md entry under the active version with `Thanks @chenlaiyi`. Do not change unrelated channel/plugin behavior.
