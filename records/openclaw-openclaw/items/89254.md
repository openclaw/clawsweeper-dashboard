## Summary

Keep open: current main and the latest release still preserve the explicit wildcard requirement, while the reporter’s follow-up shows Matrix dispatch, replies, and E2EE work once `dm.allowFrom` is set. The remaining work is a security/product decision about diagnostics and policy wording, not cleanup closure.

## What This Changes

The issue asks OpenClaw to resolve Matrix DMs configured with `dm.policy="open"` and empty `dm.allowFrom` being dropped silently before agent dispatch.

## Best Possible Solution

Keep the explicit wildcard access boundary, then make the Matrix open-DM misconfiguration self-diagnosing through redacted bounded diagnostics and clearer Matrix-specific docs/status guidance.

## Reproduction Assessment

Yes, at source level. Current main routes Matrix DMs through the shared ingress gate, and `dm.policy="open"` with no wildcard or sender allowlist resolves to `dm_policy_not_allowlisted` before dispatch.

## Solution Assessment

Unclear until maintainers decide the policy shape. The safest direction is to preserve explicit wildcard authorization and make the invalid or surprising Matrix config self-diagnosing rather than making open DMs implicitly public.
