## What This Changes

The issue asks OpenClaw to resolve Matrix DM no-dispatch/no-reply symptoms that were later traced to `dm.policy: "open"` blocking senders when `dm.allowFrom` lacks an explicit wildcard.

## Best Possible Solution

Preserve the secure wildcard contract unless maintainers explicitly choose a different access model, and make Matrix DM denials self-diagnosing through clear docs, doctor/setup guidance, or operator-visible runtime diagnostics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `dm.policy: "open"` with empty `dm.allowFrom` reaches the shared sender gate and returns `dm_policy_not_allowlisted`, matching the reporter's follow-up trace.

## Solution Assessment

Unclear. Widening `open` is not automatically the best fix because it changes a security boundary; the safer likely direction is maintainer-approved diagnostics and Matrix-specific wording while preserving explicit wildcard public access.
