## Summary

Keep open: the cleanup goal is still useful on current main, but this PR cannot merge as-is because it targets the removed `src/discord` layout, lacks inspectable real Discord deletion proof, and omits the archive confinement guard used by existing session cleanup.

## What This Changes

The PR adds a Discord channel-delete listener, registration/export changes, tests, and a changelog entry under `src/discord` to remove matching channel sessions and archive transcripts.

## Best Possible Solution

Port the cleanup to `extensions/discord/src/monitor` using the current internal listener contract, exact session-store mutation, store-confined transcript archival/cleanup, focused plugin tests, and redacted real Discord deletion proof.

## Reproduction Assessment

Yes, source-reproducible: current main still records non-DM Discord messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. I did not run a live Discord deletion in this read-only review.

## Solution Assessment

No. The cleanup direction is maintainable, but this branch is not the right implementation because it targets removed `src/discord` files and archives transcripts without the existing store-directory confinement guard.
