## Summary

Keep open: the Discord cleanup behavior is still missing on current main, but this PR is not mergeable as-is because it targets the removed `src/discord` layout, lacks inspectable real behavior proof, and misses the existing transcript archive path confinement guard.

## What This Changes

The PR adds a Discord `CHANNEL_DELETE` listener under old `src/discord` monitor paths, registration/export updates, tests, and a changelog entry to delete channel-keyed sessions and archive transcripts.

## Best Possible Solution

Port the behavior to `extensions/discord/src/monitor` using the current internal Discord listener contract, exact deleted-channel session matching, store-confined transcript archival/cleanup, focused plugin tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, source-reproducible: current main records non-DM Discord messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. I did not run a live Discord channel deletion in this read-only review.

## Solution Assessment

No. The cleanup direction is useful, but this branch is not the best implementation because it targets removed `src/discord` paths and omits the archive confinement guard that existing session deletion uses.
