## Summary

Keep open: the reported Discord session-cleanup gap is still present on current main, but this PR is not mergeable as-is because it targets removed `src/discord` paths, lacks inspectable real behavior proof, and misses the existing transcript archive confinement guard.

## What This Changes

The PR adds a Discord channel-delete listener, registration/export changes, tests, and a changelog entry to remove matching channel sessions and archive transcripts when a channel is deleted.

## Best Possible Solution

Port the cleanup to `extensions/discord/src/monitor` using the current listener contract, exact deleted-channel session matching, store-confined transcript archival/cleanup, focused plugin tests, and redacted real Discord deletion proof.

## Reproduction Assessment

Yes, source-reproducible: current main still records non-DM Discord messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. I did not run a live Discord deletion in this read-only review.

## Solution Assessment

No. The cleanup direction is useful, but this branch is not the best implementation because it targets removed `src/discord` paths and archives transcripts without the existing store-directory confinement guard.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Port the listener into the active Discord plugin:** `src/discord/monitor/provider.ts:614`
  - body: The active Discord monitor is registered from `extensions/discord/src/monitor/provider.startup.ts`, and current main has no `src/discord` implementation tree. Registering the channel-delete listener here will not wire cleanup into the bundled Discord plugin after the channel migration.
