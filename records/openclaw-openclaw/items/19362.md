## Summary

Keep open: the deleted-channel cleanup is still source-reproducible on current main, but this PR cannot merge as-is because it targets removed `src/discord` paths, lacks inspectable real behavior proof, and still misses the transcript archive confinement used by existing cleanup code.

## What This Changes

The PR adds a Discord channel-delete listener, registration/export changes, tests, and a changelog entry to delete and archive channel-keyed sessions when a Discord channel is deleted.

## Best Possible Solution

Port the behavior to `extensions/discord/src/monitor` with an active internal channel-delete listener contract, exact channel-session matching, store-confined transcript archival and cleanup, focused plugin tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, source-reproducible: current main records guild messages under `discord:channel:<id>` and has no active channel-delete cleanup listener. I did not run a live Discord channel deletion in this read-only review.

## Solution Assessment

No. The cleanup direction is useful, but this branch is not the best implementation because it targets removed core paths and should preserve the existing store-confined archive safety contract.
