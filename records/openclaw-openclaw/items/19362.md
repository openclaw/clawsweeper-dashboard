Keep open: the underlying Discord session cleanup request is still valid on current main, but this PR cannot merge as-is because it implements the listener under the removed `src/discord` layout while the active bundled Discord plugin runs from `extensions/discord`.

The PR adds a Discord channel-delete listener, legacy monitor export/registration changes, tests, and a changelog entry to delete and archive stored Discord channel sessions when a channel is removed.

## Best Possible Solution

Refresh or replace the work as an `extensions/discord` plugin change: add a typed channel-delete listener in the active internal listener layer, register it during monitor startup, remove exact deleted-channel session entries through plugin-safe session-store/runtime helpers, preserve transcript archive and cleanup semantics, and cover it with focused Discord plugin tests plus user-facing changelog credit.
