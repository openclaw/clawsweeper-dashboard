## Summary

This PR should stay open but cannot merge as-is: the intended deleted-channel cleanup is still missing from current main, and the PR targets the removed `src/discord` layout instead of the active Discord plugin under `extensions/discord`.

## What This Changes

The PR adds a Discord `CHANNEL_DELETE` listener, registration, tests, and a changelog entry to remove and archive session-store entries for deleted Discord channels.

## Best Possible Solution

The active Discord plugin should own a channel-delete listener that removes exact `discord:channel:<channelId>` session entries, preserves transcript archive/cleanup semantics, and has focused plugin tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main persists guild-channel sessions but registers no `CHANNEL_DELETE` listener in the active Discord plugin. A live repro would create a Discord channel, send a message, delete the channel, and observe the session-store entry remain.

## Solution Assessment

No, this PR is not the best fix as-is because it modifies the removed `src/discord` layout. The maintainable solution is to port the listener, registration, and tests into `extensions/discord/src/monitor` with plugin-safe runtime APIs.
