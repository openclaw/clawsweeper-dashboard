## Summary

Keep this PR open: the requested Discord cleanup is still useful and source-reproducible, but the branch cannot merge as-is because it targets removed `src/discord` paths, lacks inspectable real behavior proof, and needs safer transcript archival.

## What This Changes

The PR adds a Discord `CHANNEL_DELETE` listener, registration/export changes, tests, and a changelog entry to remove channel-keyed sessions and archive transcripts when Discord channels are deleted.

## Best Possible Solution

Port the behavior to `extensions/discord/src/monitor` with exact deleted-channel session matching, plugin-safe session-store/runtime APIs, store-confined transcript archival and cleanup, focused tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, from source inspection: current main records guild messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. A live repro would send a guild message, delete that channel, and inspect that the session entry remains.

## Solution Assessment

No. The requested behavior is useful, but this branch is not the best implementation because it targets removed core paths and should be ported to the active Discord plugin while preserving existing archive safety semantics.
