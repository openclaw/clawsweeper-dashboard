## Summary

Keep open: the requested Discord cleanup is still useful and current main still lacks a channel-delete session cleanup listener, but this PR cannot merge as written because it targets removed `src/discord` paths and still needs real behavior proof plus a safer transcript archival path.

## What This Changes

The PR adds a Discord `CHANNEL_DELETE` listener, registration/export changes, tests, and a changelog entry under legacy `src/discord` paths to delete channel-keyed sessions and archive transcripts.

## Best Possible Solution

Port the behavior to `extensions/discord/src/monitor` with exact deleted-channel session matching, plugin-safe session-store/runtime APIs, store-confined transcript archival/cleanup, focused tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, from source inspection: current main records guild messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. A live repro would send a guild message, delete that channel, and inspect that the session entry remains.

## Solution Assessment

No. The requested behavior is useful, but this branch is not the best implementation because it targets removed paths and should be ported to the active Discord plugin while preserving existing archive safety semantics.
