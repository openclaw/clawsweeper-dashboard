## Summary

Keep this PR open: the requested Discord cleanup is still useful and source-reproducible on current main, but the branch cannot land because it targets removed `src/discord` paths, lacks inspectable real behavior proof, and still has a transcript-archival safety issue.

## What This Changes

The PR adds a Discord channel-delete listener, listener registration/export changes, tests, and a changelog entry to delete channel-keyed sessions and archive transcripts after a Discord channel is deleted.

## Best Possible Solution

Port the behavior to the active Discord plugin with a plugin-local channel-delete listener, exact deleted-channel session matching, store-confined transcript archival/cleanup, focused tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, from source inspection: current main records guild messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. A live repro would send a guild message, delete that channel, and inspect that the session entry remains.

## Solution Assessment

No. The requested behavior is useful, but this branch is not the best implementation because it targets removed paths and should be ported to the active Discord plugin while preserving existing archive safety semantics.
