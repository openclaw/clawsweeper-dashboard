## Summary

Keep open. Current main still lacks Discord channel-delete session cleanup, but this PR is not mergeable as-is because it targets the removed `src/discord` implementation path, lacks inspectable real Discord deletion proof, and adds an automatic transcript archive path without the existing store-directory confinement guard.

## What This Changes

The branch adds a Discord channel-delete listener under `src/discord`, exports and registers it, adds focused tests, and adds a changelog note for deleting channel-keyed sessions and archiving transcripts.

## Best Possible Solution

Port the listener into `extensions/discord/src/monitor` using the current internal listener contract, exact deleted-channel key matching, store-locked deletion metadata, store-confined transcript archival and cleanup, focused tests, and redacted real Discord deletion proof.

## Reproduction Assessment

Yes. Source inspection shows current main still records non-DM Discord sessions under `discord:channel:<id>` and registers no channel-delete cleanup listener; I did not run a live Discord deletion in this read-only review.

## Solution Assessment

No. The cleanup direction is reasonable, but this branch targets the removed `src/discord` layout and needs to preserve the existing transcript archive confinement contract in the active plugin path.
