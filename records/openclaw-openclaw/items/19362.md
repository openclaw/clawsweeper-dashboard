The PR adds a Discord channel-delete listener, registration/export changes, tests, and a changelog entry in the old `src/discord` monitor to delete and archive channel-keyed sessions.

## Best Possible Solution

Yes. Source inspection gives a high-confidence path: current main records guild sessions from `discord:channel:<id>` and registers no channel-delete cleanup listener, so a live repro would send a Discord guild message, delete the channel, and inspect that the session entry remains.

## Solution Assessment
