## Summary

Keep open: the requested Discord cleanup is still useful, but this PR targets the removed `src/discord` monitor layout, lacks inspectable after-fix behavior proof, and current `main` still has no active Discord channel-delete session cleanup.

## What This Changes

The branch adds a Discord `CHANNEL_DELETE` listener, registration/export changes, tests, and a changelog entry in the old `src/discord` monitor to delete and archive channel-keyed sessions.

## Best Possible Solution

Port the behavior to the active `extensions/discord` monitor with an internal channel-delete listener, exact channel-key matching, plugin-safe session-store APIs, restricted transcript archival/cleanup, focused tests, and real Discord proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current `main` persists guild sessions from `discord:channel:<id>` and registers no active channel-delete cleanup listener. A live repro would send a Discord guild message, delete the channel, and inspect that the session-store entry remains.

## Solution Assessment

No. The requested behavior is useful, but this PR is not the best implementation because it targets removed `src/discord` paths and should be ported to the active plugin with existing archive safety semantics.
