## Summary

Keep open: the requested Discord cleanup still appears valid on current main, but this PR cannot merge as-is because it targets removed `src/discord` paths, lacks inspectable real behavior proof, and omits the current archive path safety guard.

## What This Changes

The PR adds a Discord channel-delete listener, listener registration/export changes, focused tests, and a changelog entry to remove channel-keyed sessions and archive transcripts when Discord channels are deleted.

## Best Possible Solution

Port the behavior to `extensions/discord/src/monitor` with the current internal listener contract, exact deleted-channel session matching, store-confined transcript archival and cleanup, focused plugin tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, source-reproducible: current main records guild messages under `discord:channel:<id>` and registers no channel-delete cleanup listener. A live repro would send a Discord guild message, delete that channel, and inspect that the session entry remains; I did not run live Discord in this read-only review.

## Solution Assessment

No. The cleanup direction is useful, but this branch is not the best implementation because it targets removed core paths and should be ported to the active Discord plugin while preserving archive safety semantics.
