## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Enforce a hard status-probe timeout:** `src/acp/control-plane/manager.core.ts:1282`
  - body: Under pressure this awaits `cached.runtime.getStatus` directly and only aborts via an optional signal. Since `AcpRuntime.getStatus` is not required to honor `signal`, an adapter that hangs can stall every new session attempt at capacity; race this probe against a real timeout.
  - confidence: 0.87
- **[P2] Recheck capacity after oneshot eviction:** `src/acp/control-plane/manager.core.ts:1105`
  - body: `evictDormantOneshotRuntimesUnderPressure` may already free a slot, but the code still probes stale persistent runtimes before checking `runtimeCache.size()`. A single new session can evict an unrelated idle persistent runtime even though capacity is already available.
  - confidence: 0.84
- **[P2] Recheck pressure before clearing stale runtimes:** `src/acp/control-plane/manager.core.ts:1297`
  - body: While this candidate waits in `actorQueue.run`, another close or eviction can bring the cache below the limit. This still clears and closes the candidate, causing unnecessary context loss; revalidate that the manager is still at capacity before clearing.
  - confidence: 0.8
- **[P2] Recompute blind idle age inside the actor lock:** `src/acp/control-plane/manager.core.ts:1288`
  - body: The no-`getStatus` fallback uses `candidate.idleMs` from the snapshot taken before the actor lock. If another actor-scoped operation touched the session first, this can evict a no-status backend using stale idle age; recompute current idle time before treating it as stale.
  - confidence: 0.78
