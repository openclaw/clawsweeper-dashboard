## Summary

Keep open. Current main still has the ACP manager counting problem, and this PR is a plausible fix direction, but the current head still has correctness blockers and lacks required real behavior proof before merge.

## What This Changes

The branch makes ACP session-limit enforcement async, releases cached oneshot handles after startup identity reconciliation, evicts non-running oneshot/stale runtimes under pressure, and adds ACP manager regression tests.

## Best Possible Solution

Land a bounded ACP manager fix that releases non-running oneshot handles and reclaims stale cached runtimes only while still at capacity, with focused regression tests and redacted live ACP proof.

## Reproduction Assessment

Yes, source-level: current main counts runtimeCache.size() in enforceConcurrentSessionLimit, while getSessionStatus and startup identity reconciliation can leave oneshot handles cached. I did not run a live ACPX reproduction in this read-only pass.

## Solution Assessment

No. The fix direction is appropriate, but this PR needs a hard status-probe timeout, capacity/idleness rechecks, and real after-fix proof before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Enforce a hard timeout for status probes:** `src/acp/control-plane/manager.core.ts:1282`
  - body: This starts a 5s AbortController timer but awaits getStatus directly. Since the runtime contract only accepts an optional signal and wrappers delegate to adapters, an adapter that ignores abort can hang session initialization under capacity pressure; race the probe against a timer.
  - confidence: 0.87
- **[P2] Skip stale probing once oneshot eviction frees capacity:** `src/acp/control-plane/manager.core.ts:1105`
  - body: After dormant oneshot eviction, this always enters stale probing even if the cache is already below maxConcurrentSessions. One new session can evict an unrelated idle persistent runtime and lose context; recheck the cache size before stale probing.
  - confidence: 0.82
- **[P2] Recompute blind idle age inside the actor lock:** `src/acp/control-plane/manager.core.ts:1288`
  - body: The no-getStatus fallback uses candidate.idleMs from the initial snapshot after waiting on actorQueue. A queued operation can touch the cache first, so re-read lastTouchedAt or current idle age inside the lock before blind eviction.
  - confidence: 0.78
