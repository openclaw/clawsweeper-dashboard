## Summary

Keep open: current main still lacks the ACP pressure-eviction path, but this PR head has concrete correctness blockers and no after-fix real behavior proof beyond unit tests.

## What This Changes

The PR changes the ACP session manager to close cached oneshot handles after startup identity reconciliation, evict dormant oneshot and stale runtimes under concurrent-session pressure, and add ACP manager regression tests.

## Best Possible Solution

Land a bounded ACP manager fix that releases non-running oneshot handles and reclaims stale cached runtimes only while still at capacity, with focused regression tests and redacted live ACP proof; keep reset/process cleanup in https://github.com/openclaw/openclaw/issues/68916.

## Reproduction Assessment

Yes, source-level: current main counts runtimeCache.size() in enforceConcurrentSessionLimit, while status and startup reconciliation paths can leave oneshot handles cached. I did not run a live ACPX reproduction in this read-only review.

## Solution Assessment

No. The fix direction is appropriate, but this branch needs a hard status-probe timeout plus capacity and idleness rechecks before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Enforce a real timeout for status probes:** `src/acp/control-plane/manager.core.ts:1282`
  - body: This starts a 5s AbortController timer but awaits getStatus directly. Since AcpRuntime.getStatus only receives an optional signal, an adapter that ignores abort can hang session initialization under capacity pressure; race the probe against a timer so the manager always progresses.
  - confidence: 0.87
- **[P2] Recheck capacity after oneshot eviction:** `src/acp/control-plane/manager.core.ts:1105`
  - body: After dormant oneshot eviction, this always enters stale probing even if the cache is already below maxConcurrentSessions. That can evict an unrelated idle persistent runtime for a single new session; skip stale probing once the oneshot phase has relieved pressure.
  - confidence: 0.82
- **[P2] Recompute idle age inside the actor lock:** `src/acp/control-plane/manager.core.ts:1288`
  - body: The no-getStatus fallback uses candidate.idleMs from the initial snapshot after waiting on actorQueue. A queued operation can touch the cache first, so re-read lastTouchedAt or current idle age inside the lock before blind eviction.
  - confidence: 0.78
