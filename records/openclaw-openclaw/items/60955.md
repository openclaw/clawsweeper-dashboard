## Summary

Keep open. Current main and v2026.5.18 still lack a guard for first-person execution-intent shell fences with no same-turn tool activity, and the focused fix PR was closed unmerged while the broader simulated-tool policy remains separate.

## What This Changes

This issue reports that Telegram direct/main sessions can deliver first-person exec-looking shell-fence replies even when the session turn has zero real tool calls or tool results.

## Best Possible Solution

Add a channel-agnostic embedded visible-reply guard that replaces or blocks strong first-person execution-intent shell-fence replies when the current assistant turn lacks real tool lifecycle evidence, with explicit bypass coverage for real tool-backed turns.

## Reproduction Assessment

Yes, at source level. A no-tool assistant message with first-person install/check/run text plus a fenced shell command can still flow through the current visible reply paths on main and v2026.5.18 without a same-turn tool-activity guard.

## Solution Assessment

Yes. A narrow embedded visible-output guard is the maintainable fix because it addresses the Telegram/direct truthfulness bug without changing exec policy, provider defaults, or the broader strict-agentic contract.
