## Review Findings

Full review comments:

- **[P1] Handle auth-profile watcher errors:** `src/agents/auth-profiles-watcher.ts:22-26`
  - body: This new chokidar watcher has no `error` listener. Chokidar-style watchers can emit `error`, and Node throws an unhandled EventEmitter `error`, so a watch failure such as ENOSPC or a bad auth-profile path can take down the gateway. Add the same log-and-close handling used by the existing config and skills watchers.
  - confidence: 0.9
- **[P2] Keep the auth-profile watcher stoppable:** `src/gateway/server-startup-post-attach.ts:1029-1032`
  - body: `watchAuthProfilesForChanges` returns a handle, but startup discards it. That leaves server close and hot reload with no way to stop or replace the watcher, so restarts/tests can leak polling watchers and keep watching stale initial agent paths. Store the handle in gateway runtime state or another cleanup path and close it with the other gateway watchers.
  - confidence: 0.86
