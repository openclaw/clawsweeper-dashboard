## Summary

Keep open for maintainer review: the PR targets a real provider-auth pre-warm performance problem, but the patch removes the auth freshness backstop and drops agentDir-scoped picker auth without an equivalent agentId path.

## What This Changes

This PR changes prepared provider auth from a single 10-second default-agent cache to per-agent warmed state keyed by agentId, removes agentDir from the auth checker API/callers, and updates focused auth tests.

## Best Possible Solution

Keep the per-agent pre-warm direction only after preserving a tested freshness/invalidation path and threading agentId through picker flows or retaining an agentDir-compatible lookup.

## Reproduction Assessment

Yes, source-reproducible. Current main has TTL fallback coverage and agentDir-scoped picker/auth store paths, while the PR diff removes those protections; I did not run tests because this was a read-only review.

## Solution Assessment

No. The per-agent pre-warm direction is useful, but the proposed implementation should preserve auth freshness and agent-scoped credential lookup before it is the narrow maintainable fix.
