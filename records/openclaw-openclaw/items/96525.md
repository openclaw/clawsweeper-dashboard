## Label Justifications

## Summary

Keep open: current main and v2026.6.10 still omit result-level fallback classification in isolated cron embedded runs, so the reported fallback gap remains until a fix merges. The open, proof-positive candidate fix is https://github.com/openclaw/openclaw/pull/96529, so this issue should remain the canonical report while that PR is reviewed.

## What This Changes

The issue requests that isolated cron embedded agentTurn runs apply the configured model fallback chain when the primary returns fallback-safe reasoning-only, empty, or incomplete result-level failures.

## Best Possible Solution

Land the scoped classifier-only fix from https://github.com/openclaw/openclaw/pull/96529, then close this issue as implemented once that commit reaches main.

## Reproduction Assessment

Yes at source level. Current main still omits the classifier hooks, and the fallback helper only advances returned result-level failures when a classifier returns a classification, matching the provided failing repro shape.

## Solution Assessment

Yes. Reusing the existing embedded result classifier and exhausted-result merge helper in the cron embedded path is the narrowest maintainable fix, with CLI providers explicitly excluded as shown in https://github.com/openclaw/openclaw/pull/96529.
