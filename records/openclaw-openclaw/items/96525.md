## Summary

Keep open: current main still has the reported isolated-cron result-level fallback gap, and the latest release has the same behavior. An open focused PR now appears to be the active fix candidate, so this issue should remain open until that PR lands or maintainers choose another path.

## What This Changes

The issue reports that isolated cron agentTurn jobs omit embedded result classification in the model fallback loop, so reasoning-only, empty, or fallback-safe incomplete primary results can end as cron errors instead of trying configured fallback models.

## Best Possible Solution

Review and land the focused fix in https://github.com/openclaw/openclaw/pull/96529 if maintainers accept the documented fallback/provider-routing behavior, then close this issue after the merge.

## Reproduction Assessment

Yes, source-reproducible. Current main omits result classification in the cron fallback call, while runWithModelFallback only advances returned result-level failures when a classifier is supplied; the issue also provides a focused failing repro shape.

## Solution Assessment

Yes. Reusing the existing embedded result classifier and exhausted-result merge helper in the cron embedded path is the narrowest maintainable fix, and it matches the already-wired interactive and auto-reply callers.
