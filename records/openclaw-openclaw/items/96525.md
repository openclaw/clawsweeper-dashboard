## Summary

Keep open: current main and v2026.6.10 still lack cron result-classifier wiring, and the central fix is currently represented by an open mergeable PR rather than a merged implementation.

## What This Changes

The issue reports that isolated cron embedded `agentTurn` fallback chains do not advance on returned result-level failures such as reasoning-only, empty, or fallback-safe incomplete turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96529, or an equivalent narrow change that wires the existing embedded classifier and merge helper into cron with focused regression coverage.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the gap: cron omits `classifyResult`, while `runWithModelFallback` only advances on returned result-level failures when a classifier is provided.

## Solution Assessment

Yes. The best fix is the narrow owner-boundary change already proposed by the open PR: pass the existing embedded result classifier and exhausted-result merge helper into the cron embedded fallback call while leaving CLI terminal handling scoped out.
