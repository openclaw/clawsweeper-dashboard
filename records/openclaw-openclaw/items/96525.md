## Summary

Keep open: current main still lacks cron-specific result classification in the isolated embedded fallback path, so returned fallback-safe failures can be accepted as the primary result and later reported as a cron error instead of advancing to the configured fallback model.

## What This Changes

This issue reports that isolated cron `agentTurn` runs do not apply configured model fallbacks to embedded result-level failures such as reasoning-only, empty-visible, or `incomplete_turn` results.

## Best Possible Solution

Reuse the existing embedded result fallback classifier and exhausted-result merge helper in the cron embedded fallback path, then add a focused regression test for returned fallback-safe incomplete/reasoning-only results advancing to the configured fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: cron passes no classifier into `runWithModelFallback`, so a returned fallback-safe incomplete result is accepted as the first candidate result rather than advancing to the fallback model.

## Solution Assessment

Yes. Wiring the existing embedded result classifier and exhausted-result merge helper into the cron embedded fallback path is the narrowest maintainable fix because it reuses the established interactive and auto-reply fallback contract.
