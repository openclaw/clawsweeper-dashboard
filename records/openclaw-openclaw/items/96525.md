## Summary

Keep open: current main still has the cron embedded fallback gap, and the useful fix has not merged. The issue is now paired with several open fix PRs, with https://github.com/openclaw/openclaw/pull/96529 looking like the most direct candidate because it targets this exact classifier/merge-helper omission and has sufficient proof.

## What This Changes

The issue reports that isolated cron embedded `agentTurn` runs do not invoke configured fallback models when the primary returns reasoning-only, empty, or fallback-safe `incomplete_turn` results.

## Best Possible Solution

Land one narrow cron fix that wires the existing embedded result classifier and exhausted-result merge helper into the embedded cron `runWithModelFallback` path, with regression coverage proving fallback engagement and CLI-provider scope.

## Reproduction Assessment

Yes. Source inspection shows cron omits the classifier while `runWithModelFallback` only advances returned result-level failures when a classifier is supplied; I did not run the ad hoc repro because this review is read-only and cannot add the temporary test file.

## Solution Assessment

Yes. Reusing the existing embedded classifier and exhaustion merge helper in the cron embedded path is the narrowest maintainable fix because the interactive and auto-reply paths already use that contract for the same returned-result failure class.
