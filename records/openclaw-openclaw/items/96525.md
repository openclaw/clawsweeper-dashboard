## Summary

Keep open: current main and v2026.6.10 still omit cron-side result classification for embedded model fallback, so this bug is not implemented yet. The issue should stay canonical while maintainers review the viable linked fix PR.

## What This Changes

The issue reports that isolated cron agentTurn jobs skip configured fallback models when embedded providers return reasoning-only, empty, or incomplete result-level failures.

## Best Possible Solution

Land or replace the viable linked cron fallback PR, then close this canonical issue with the merge commit and release/main provenance.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: cron omits `classifyResult`, and the shared fallback loop only advances on returned result-level failures when a classifier is supplied.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing embedded result classifier and exhaustion merge helper into the cron embedded fallback call, matching interactive and auto-reply while leaving CLI cron behavior to its own terminal classification.
