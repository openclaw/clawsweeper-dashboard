## Label Justifications

- `impact:message-loss`: The reported path drops the useful cron answer when a fallback-safe failed primary result is accepted as the final result.

## Summary

Keep open: current main and latest release still omit result-level fallback classification in the isolated cron fallback call, so fallback-safe incomplete/reasoning-only embedded results can stop on the primary model. The open candidate fix at https://github.com/openclaw/openclaw/pull/96529 now owns the remaining work and has real-behavior proof, so this issue should close only after that PR, or an equivalent fix, lands.

## What This Changes

The issue requests that isolated cron embedded agentTurn runs engage configured model fallbacks when the primary returns reasoning-only, empty, or fallback-safe incomplete_turn results.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96529 or an equivalent patch that wires the existing embedded result classifier and exhaustion merge helper into the cron embedded fallback path, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Current main omits classifyResult and mergeExhaustedResult in the cron fallback call, while runWithModelFallback only advances returned result-level failures when a classifier is supplied.

## Solution Assessment

Yes. Reusing the existing embedded result classifier and exhausted-result merge helper in the cron embedded path is the narrowest maintainable fix and matches the interactive/auto-reply fallback contract.
