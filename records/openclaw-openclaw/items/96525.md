## Summary

Keep open: current main still lacks embedded result classification in the isolated cron fallback call, so the reported result-level failure path remains source-reproducible until an open fix PR lands.

## What This Changes

The issue reports that isolated cron agentTurn runs do not engage configured model fallbacks when an embedded primary returns reasoning-only, empty, or fallback-safe incomplete results.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/96529 as the canonical embedded-only classifier/merge-helper fix, with adjacent stream-stall and abort work tracked separately.

## Reproduction Assessment

Yes, at source level. Current main omits the classifier at the cron fallback call, while runWithModelFallback requires classifier output to advance returned result-level failures.

## Solution Assessment

Yes. Reusing the existing embedded result classifier and exhausted-result merge helper in the cron embedded path is the narrowest maintainable fix because the interactive and auto-reply paths already use that contract.
