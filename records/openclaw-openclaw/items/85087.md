## Summary

Keep open. Current main has a boolean fast-mode contract across session state, config, UI/TUI controls, provider extra params, and Codex app-server service-tier handling; the requested `auto` cutoff would be a new cross-surface policy/config feature rather than cleanup for an already implemented behavior.

## What This Changes

The issue requests `fast: auto` / `fastMode: "auto"` with a configurable cutoff, per-call fast-tier resolution, provider/app-server integration, and progress events when long turns stop using fast tier.

## Best Possible Solution

Add a backward-compatible fast-mode policy resolver that preserves existing booleans, introduces an explicit `auto` state plus cutoff seconds, resolves the effective fast tier at each supported model-call boundary, and documents/tests Codex app-server’s turn-boundary limitation.
