## Summary

Keep open. Current `main` has a boolean-only fast-mode contract across session state, config, UI/TUI controls, provider params, and Codex app-server service-tier forwarding; the requested `auto` cutoff is a new cross-surface policy feature that needs maintainer API/product direction.

## What This Changes

Request automatic fast-mode cutoff via `fast: auto` / `fastMode: "auto"`, configurable cutoff seconds, per-call provider re-resolution, progress events, and Codex app-server documentation.

## Best Possible Solution

Add a backward-compatible fast-mode policy resolver that preserves existing boolean behavior, introduces an explicit `auto` state plus cutoff seconds, resolves the effective fast tier at each supported call boundary, and documents/tests Codex app-server’s turn-boundary limitation.

## Reproduction Assessment

Not applicable: this is a feature request, and current docs/schema define only `on`, `off`, and default boolean fast-mode behavior rather than an existing `auto` contract to reproduce as broken.

## Solution Assessment

Unclear without maintainer API/product approval; the requested direction is plausible, but the maintainable path is a shared policy resolver and compatibility plan rather than provider-specific cutoff checks scattered through call sites.
