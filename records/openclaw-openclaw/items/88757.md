- `impact:session-state`: The issue is about drift between delivered outbound messages and the agent’s later session context.

## Summary

Keep open: current main has shipped transcript/UI mirroring for delivered outbound messages, but the agent replay path explicitly treats those mirrors as transcript-only and drops them before provider input, so the central “agent remembers the proactive message” problem is not implemented.

## What This Changes

The issue requests that proactive outbound messages, including message-tool sends, scheduled reports, and reminders, become visible in the agent session context so later user replies retain conversational continuity.

## Best Possible Solution

Design a session-awareness path that gives the next relevant agent turn durable knowledge of proactive deliveries without replaying transcript-only mirror records as provider assistant output.

## Reproduction Assessment

Yes, source-reproducible: current code writes proactive delivery mirrors to transcript/history but then filters delivery-mirror entries before provider replay, which matches the reported “delivered but not remembered” failure mode.

## Solution Assessment

Unclear: the requested outcome is valid, but the best fix is not to blindly replay mirror entries because current source documents duplicate-content and provider turn-ordering risks.
