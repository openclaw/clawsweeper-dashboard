## Summary

Keep open. The reported bug is still present on current main, but this PR should not merge as-is because it raw-copies model/provider overrides without the current source-aware reset policy and it still lacks after-fix real behavior proof.

## What This Changes

The PR moves model/provider override preservation outside the explicit reset guard for stale daily rollover sessions and adds a fake-timer regression test.

## Best Possible Solution

Preserve stale-rollover user and legacy selections through `resolveResetPreservedSelection` or an equivalent shared helper, clear auto-sourced model/auth state, and cover user, legacy, and auto-sourced stale rollovers.

## Reproduction Assessment

Yes, by source inspection. A stale stored session with `resetTriggered=false` enters the new-session branch, skips `resolveResetPreservedSelection`, and has no `baseEntry` fallback for model/provider/source fields.

## Solution Assessment

No. The PR points at the right rollover branch, but the maintainable fix should reuse the shared source-aware preservation policy instead of copying raw model/provider fields.
