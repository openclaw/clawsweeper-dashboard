## Summary

Keep open: the reported daily rollover bug is still present on current main, but this PR predates the current source-aware model override policy and should not merge unchanged.

## What This Changes

The PR moves model/provider override preservation outside the explicit reset guard in `initSessionState` and adds a daily freshness rollover regression test.

## Best Possible Solution

Preserve user-driven and legacy source-less model/provider selections during implicit stale rollover by reusing the existing source-aware reset selection policy, while continuing to clear auto-sourced fallback pins.

## Reproduction Assessment

Yes. A current-main fixture with system time after the 4 AM boundary and a stored session from before that boundary reaches `isNewSession=true` and `resetTriggered=false`, then drops the previous model/provider fields because preservation is reset-gated.

## Solution Assessment

No. The requested direction is right, but the PR's implementation is not the best current-main fix because it copies raw override fields instead of applying the source-aware user/legacy/auto policy.
