## Summary

Keep this PR open: the reported stale-rollover bug is still source-reproducible on current main, but the submitted patch is not merge-ready because it bypasses the newer source-aware reset-selection policy and still lacks after-fix real behavior proof.

## What This Changes

The PR moves raw `modelOverride` and `providerOverride` preservation outside the explicit reset guard and adds fake-timer coverage for a stale daily reset.

## Best Possible Solution

Preserve stale-rollover user and legacy selections through `resolveResetPreservedSelection` or an equivalent shared helper, clear auto-sourced model/auth state, and cover user, legacy, and auto-sourced stale rollovers.

Yes, by source inspection. A stale stored session with `resetTriggered=false` enters the new-session branch, skips the source-aware preserved-selection helper, and has no `baseEntry` fallback for model/provider/source fields.

## Solution Assessment

No. The PR targets the right rollover branch, but the maintainable fix should reuse the shared source-aware preservation policy instead of copying raw model/provider fields.
