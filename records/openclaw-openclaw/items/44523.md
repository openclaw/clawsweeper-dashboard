## Summary

## What This Changes

The PR moves model/provider override preservation out of the explicit reset guard in `initSessionState` and adds a fake-timer daily freshness reset regression test.

## Best Possible Solution

Reuse `resolveResetPreservedSelection` or a shared equivalent for stale rollovers, preserve user/legacy selections with source metadata, clear auto-sourced fallback/auth state, and add focused coverage plus redacted real rollover proof.

## Reproduction Assessment

Yes, by source inspection. A stale entry with `freshEntry=false` and `resetTriggered=false` goes into the new-session branch, does not call the preservation helper, and has no `baseEntry` fallback for model/provider/source fields.

## Solution Assessment

No. The PR targets the right rollover branch, but the maintainable fix should reuse the source-aware preservation policy instead of copying raw model/provider fields.
