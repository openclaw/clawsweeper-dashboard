## Summary

Keep open: the reported stale-rollover bug is valid, but this PR predates the current source-aware model selection policy and would preserve or reclassify auto-sourced fallback pins if rebased unchanged.

## What This Changes

The PR raw-copies `modelOverride` and `providerOverride` into new stale-rollover sessions and adds a daily freshness regression test.

## Best Possible Solution

Apply source-aware preservation to implicit stale rollover so user and legacy model selections survive with source metadata while auto fallback pins clear, with regression coverage for user, legacy, and auto-sourced entries.

## Reproduction Assessment

Yes by source inspection: a stored stale session updated before the daily boundary reaches the new-session branch with `resetTriggered=false`, skips the only current selection-preservation block, and has no `baseEntry` fallback. I did not execute tests because this review is read-only.

## Solution Assessment

No. The requested direction is right, but the maintainable fix should reuse `resolveResetPreservedSelection` or equivalent source-aware logic instead of raw-copying override fields.
