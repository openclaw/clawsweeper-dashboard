Overall correctness: patch is incorrect

Full review comments:

- **[P2] Stop fetch-all when the cursor does not advance:** `extensions/feishu/src/drive.ts:379-380`
  - body: With `all:true`, the loop continues whenever `has_more` is true, even if the response omits `next_page_token` or repeats the prior cursor. The next iteration then sends the same params again and can duplicate the same page while burning through up to 100 API calls, so continue only when there is a fresh cursor or remove the fetch-all mode.
  - confidence: 0.84

  - body: The schema/runtime now accept `info.folder_token`, but the bundled drive skill still says info searches the root directory and shows no `folder_token` example. Agents using the skill can miss the scoped lookup path, so update that section.
  - confidence: 0.86
