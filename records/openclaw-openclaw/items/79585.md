## Summary

Keep open: current main still has a concrete runtime/auth mismatch because Gemini CLI OAuth setup writes an ignored whole-agent runtime while execution selection only honors provider/model runtime policy.

## What This Changes

The issue asks OpenClaw to make Gemini CLI OAuth apply and report consistently for canonical `google/*` model refs when `GEMINI_API_KEY` is also configured.

## Best Possible Solution

Store Gemini CLI OAuth setup as provider/model-scoped runtime policy for canonical Google models and make status distinguish selected Gemini CLI OAuth runtime auth from normal Google API-key auth.

## Reproduction Assessment

Yes, source-reproducible: current setup writes `agents.defaults.agentRuntime`, while current runtime selection ignores that field and status still reports canonical `google` auth from env. I did not perform a live OAuth run with real Google credentials.

## Solution Assessment

Yes, the narrow maintainable fix is in the Google plugin setup and models-status/runtime reporting path. It should not forward or print OAuth secrets, and it should avoid changing normal `google` API-key behavior.

## Repair Work Prompt

Fix the reported Google Gemini CLI OAuth runtime/status mismatch. Current `openclaw models auth login --provider google-gemini-cli --set-default` stores a `google-gemini-cli` OAuth profile but the provider config patch writes legacy `agents.defaults.agentRuntime`, which current runtime selection ignores. Update the Google plugin auth result so the canonical `google/gemini-...` default gets provider/model-scoped `agentRuntime.id: "google-gemini-cli"` without using `agents.defaults.agentRuntime`; make `models status` clearly report the selected runtime auth path when canonical `google/*` uses the Gemini CLI runtime alongside `GEMINI_API_KEY`; add focused regression coverage. Do not forward, print, or broaden OAuth token handling, and do not change normal Google API-key behavior for the default Google runtime. Add a changelog entry because this is user-visible auth/status behavior.
