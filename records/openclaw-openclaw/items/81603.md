## Summary

Keep open: this is a narrow ACP bug-fix PR with sufficient real OpenCode proof and green CI, but the current diff still adds duplicate Codex SDK subpath documentation rows that should be removed before landing.

## What This Changes

The PR makes ACP timeout config hints best-effort when adapters reject them, adds positive and negative ACP manager regressions, and adds Codex SDK subpath documentation rows.

## Best Possible Solution

Land the manager-side timeout tolerance after removing the duplicate SDK subpath doc rows, then let the linked OpenCode issue close when the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main can emit a timeout `set_config_option` before `runTurn`, and that awaited call fails the turn if an OpenCode-like adapter rejects it. I did not run a live current-main OpenCode repro in this read-only review, but the PR body supplies after-fix live terminal proof.

## Solution Assessment

Mostly yes: treating only `timeout` and `timeout_seconds` as best-effort adapter hints preserves strict failures for model/thinking/permission config while relying on OpenClaw's local turn timeout. The docs hunk is not the best form because those Codex helper subpaths are already documented on main.
