## What This Changes

The PR changes ACP pre-turn runtime-control handling to ignore rejected optional `timeout`/`timeout_seconds` config hints while adding manager regression tests that required config rejections still fail.

## Best Possible Solution

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: tolerate only optional timeout config rejection while preserving strict failures for required runtime controls. The remaining blocker is proof quality, not an obvious code-design mismatch.
