Request an agent-callable Slack capability for sending messages with real `{channel, ts}` receipts, reading channel messages, and listing channels from sandboxed Codex/OpenClaw agent sessions.

## Best Possible Solution

Keep Slack send/read on the shared `message` tool, document and prove the receipt path, and decide whether to expose Slack channel listing as a `channel-list` message action rather than adding a parallel Slack-only tool.
