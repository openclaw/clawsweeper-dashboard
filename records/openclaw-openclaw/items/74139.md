## Summary

Keep open. The reported llm-task failure is real on current main, but this PR should not merge as-is because its blanket disableTools bypass also suppresses the runtime toolsAllow fail-closed contract protected by current tests.

## What This Changes

The PR changes the empty explicit tool allowlist guard to return null whenever disableTools is true and updates the colocated guard test to expect no error for that case.

## Best Possible Solution

Land a narrowed guard fix that allows disabled llm-task runs to ignore inherited config/profile allowlists while keeping runtime toolsAllow conflicts erroring, and coordinate generic modelRun behavior with #73031/#74810.

## Reproduction Assessment

Yes. A direct guard call with config-derived sources, no callable tools, toolsEnabled true, and disableTools true reproduces the current false-positive; the same direct call with runtime toolsAllow demonstrates the regression introduced by this PR.

## Solution Assessment

No. The fix direction is valid, but returning null for every disableTools run is broader than needed; the maintainable fix should distinguish inherited policy from explicit runtime toolsAllow or carry a narrower no-tool call-shape flag.
