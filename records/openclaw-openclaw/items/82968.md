## Summary

This should stay open: current main and the latest shipped release already stamp many agent inputs and document `session_status` as the time source, but there is still no dedicated clock/now tool and the `session_status` path currently wires uptime without a current wall-clock line.

## What This Changes

Requests an explicit agent-facing wall-clock time authority instead of relying on uptime or `exec date` workarounds.

## Best Possible Solution

Have maintainers bless one explicit clock contract, then implement it consistently as either a read-only `session_status` timestamp source or a dedicated `clock.now` tool with UTC, local time, timezone, and tests/docs aligned.

## Reproduction Assessment

No high-confidence live prompt reproduction was established. Source inspection does show the remaining gap: there is no dedicated clock/now tool, and the documented `session_status` route currently receives uptime without a wall-clock `timeLine`.

## Solution Assessment

Unclear. A native now tool matches the request, but current OpenClaw docs and prompt guidance already point to `session_status` and message timestamps, so maintainers should choose that contract before implementation.
