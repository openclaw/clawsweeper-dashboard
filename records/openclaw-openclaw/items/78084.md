## Summary

Keep open. Current main still lacks dynamic wildcard tool contracts, but this PR is not merge-ready because the proposed concrete allowlist filtering can expose same-prefix tools that were not explicitly requested, the external PR proof is still unit/check output only, and the public manifest/security-boundary change needs maintainer acceptance.

## What This Changes

The PR adds terminal `prefix_*` matching for plugin `contracts.tools` across registration validation, runtime plugin selection, explicit tool allowlist resolution, tests, and manifest docs.

## Best Possible Solution

No. Bounded wildcard ownership is plausible, but this implementation is not the best fix until concrete allowlists remain narrow and maintainers accept the public plugin contract/security semantics.
