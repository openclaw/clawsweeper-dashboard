## Summary

Keep open: the user-facing direction is valid, but this draft still relies on brittle marker-text detection, has a stale test import, lacks real TUI proof, and overlaps the broader yielded parent-session progress contract.

## What This Changes

This PR adds an `awaiting follow-up` TUI busy state, detects a `sessions_yield` marker in final/history text, and adds focused TUI tests for live and history-restore behavior.

## Best Possible Solution

Drive the TUI wait indicator from structured yielded/liveness or paused-session state, and coordinate it with the broader parent-session progress contract.

## Reproduction Assessment

Yes, at source level. Current main finalizes non-error TUI chat finals to `idle` while `sessions_yield` persists hidden context and lifecycle/wait metadata that the TUI does not map to a busy state.

## Solution Assessment

No. A visible busy state is the right direction, but this patch scans finalized display text instead of existing structured yielded/liveness or hidden session state, so it can render or restore incorrectly.
