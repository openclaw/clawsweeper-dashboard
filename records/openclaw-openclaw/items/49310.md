## Summary

Keep open. The PR targets a real TUI status gap, but the proposed marker-text detection is likely ineffective because the production sessions_yield marker is persisted as hidden custom data while the TUI receives visible chat/lifecycle projections.

## What This Changes

The PR adds an `awaiting follow-up` TUI busy state keyed off a literal `sessions_yield` context marker in final assistant text and restored history, with focused TUI tests.

## Best Possible Solution

Keep the `awaiting follow-up` busy state, but drive it from structured sessions_yield lifecycle or session metadata instead of matching hidden context text.

## Reproduction Assessment

Yes, source-level. Current main emits `yielded: true` on sessions_yield lifecycle end, while the TUI lifecycle handler unconditionally sets the activity status back to `idle`.

## Solution Assessment

No. The proposed patch matches a literal marker that production stores as hidden custom context, so the narrow maintainable fix is to consume the structured `yielded` signal or expose equivalent structured history state.
