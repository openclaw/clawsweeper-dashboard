## Summary

Keep open. The reported TUI busy-state gap looks valid, but this draft patch is not merge-ready because it keys off final assistant text while current `sessions_yield` state is persisted as hidden custom context plus structured yielded metadata, and the contributor has not provided real TUI behavior proof.

## What This Changes

The PR adds an `awaiting follow-up` TUI activity state, detects a `sessions_yield` marker in final output, restores that state from history, and adds focused TUI tests.

## Best Possible Solution

Preserve the busy TUI state from structured yielded/liveness data or the actual hidden custom yield context, with a shared helper and live proof that the footer stays busy until the follow-up event arrives.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current main can drop the TUI to idle after a final event, but the exact `sessions_yield` event and history shape still needs a real TUI/gateway proof path.

## Solution Assessment

No. The PR’s user-visible goal is right, but the implementation should use structured yielded/liveness metadata or the persisted custom yield context instead of matching duplicated assistant-text marker strings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use yielded state instead of final text markers:** `src/tui/tui-event-handlers.ts:265-266`
  - body: The `sessions_yield` marker is persisted as hidden custom context and terminal metadata carries `yielded: true`/`livenessState: "paused"`; the final assistant payload can be empty or silent. This condition can miss the actual yield and still finalize the active run as `idle`, leaving the TUI idle for the wait this PR is trying to cover.
  - confidence: 0.86
- **[P2] Restore from the custom yield context message:** `src/tui/tui-session-actions.ts:367`
  - body: History restore only assigns `lastAssistantText` inside the assistant branch, but the yield marker is stored as a hidden custom `openclaw.sessions_yield` message. Reloading a paused session can therefore skip the marker and clear `awaiting follow-up` back to `idle`; inspect the custom context or a structured paused/yielded field instead.
  - confidence: 0.89
