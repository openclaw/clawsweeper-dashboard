## Summary

Keep open. The PR targets a real TUI `sessions_yield` idle-state gap, but the patch likely keys off the wrong signal: current runner code persists the marker as hidden custom state while TUI live finals and history restore consume assistant text, so real yielded turns may still fall back to idle. The PR is also draft and lacks required after-fix real behavior proof.

## What This Changes

The PR adds an `awaiting follow-up` TUI activity state based on a literal `sessions_yield` context marker in final assistant output and restored session history.

## Best Possible Solution

Expose or consume an explicit yielded/follow-up state from the runner or gateway session metadata, then update TUI live events and history restore with tests against the real event/history shapes.

## Reproduction Assessment

Yes, source-reproducible. Current main finalizes successful TUI chat finals to `idle`, while `sessions_yield` persists its follow-up marker as hidden custom state that the TUI history loop currently ignores; no live TUI run was executed in this read-only review.

## Solution Assessment

No. The proposed assistant-text marker check is not the narrowest reliable fix because the real runner path persists the marker as hidden custom state; the safer fix is an explicit yielded/session-pause signal consumed by both live and history TUI code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Key the busy state off the real yield signal:** `src/tui/tui-event-handlers.ts:265-266`
  - body: `sessions_yield` does not put this marker in the assistant final text in the real runner path: the marker is persisted after abort as a hidden custom message (`role: custom`, `display: false`), while chat finals are built from buffered assistant text and history restore only records assistant messages. As a result real yielded turns can still finalize to idle and reload from history without entering `awaiting follow-up`. Please consume an explicit yielded/session-pause signal or the real custom/tool-result shape instead of this assistant-text marker.
