## Summary

Keep this draft PR open. Current main still lacks an `awaiting follow-up` busy state for TUI `sessions_yield` waits, but the proposed patch is not yet the best implementation because it keys live and restore behavior off a literal final-message marker while current main exposes the durable signal through typed lifecycle metadata and a hidden custom session context message.

## What This Changes

The draft PR adds an `awaiting follow-up` TUI busy activity, detects a `sessions_yield` follow-up marker in final/history output, updates the status renderer, and adds focused TUI event/history tests.

## Best Possible Solution

Revise the focused TUI fix so live status derives follow-up waits from stable lifecycle data such as `yielded: true` and `livenessState: "paused"`, history restore detects the actual persisted `openclaw.sessions_yield` context or a shared helper rather than duplicating a literal assistant-text marker, and the state is cleared on continuation, unrelated session switches, and stale restores.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a TUI run whose terminal lifecycle event carries `yielded: true` and `livenessState: "paused"`, or a restored history containing the hidden `openclaw.sessions_yield` context; current main settles those paths to `idle` or ignores them during history restore.

## Solution Assessment

No. The PR touches the right TUI surfaces, but detecting a duplicated literal marker in final assistant text is weaker than using the typed lifecycle signal for live events and the hidden custom context or shared helper for restore behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Drive follow-up waits from lifecycle metadata:** `src/tui/tui-event-handlers.ts:265-266`
  - body: The new detection only looks for a literal marker in `finalText`, but real `sessions_yield` runs set `yielded: true` and `livenessState: "paused"` on the terminal lifecycle event, while the synthetic aborted response can produce no final marker text. This can leave the TUI going back to idle for the actual live yield path.
  - confidence: 0.86
- **[P2] Restore waits from the persisted yield context:** `src/tui/tui-session-actions.ts:341`
  - body: History restore records only assistant text, but the durable `sessions_yield` marker is persisted as a hidden custom message with custom type `openclaw.sessions_yield`. Reloading real history can therefore miss the pending follow-up state even though the marker exists in the session.
  - confidence: 0.84
