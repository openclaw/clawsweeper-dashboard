## Repair Work Prompt

Implement a narrow diagnostic for https://github.com/openclaw/openclaw/issues/79091. Current main emits a terse future-config warning from `src/config/io.ts`, while `src/gateway/probe.ts` already captures the running gateway `server.version` and `gateway status` prints service command data but drops the gateway version. Add user-facing guidance for stale PATH/global CLI wrappers when the CLI version is older than `meta.lastTouchedVersion` and/or the probed gateway version. Prefer a shared helper used by the config warning and `openclaw gateway status`/doctor where appropriate; do not add config options, change destructive downgrade guards, or broaden install/update policy. Include regression coverage for the warning/hint and status/probe version propagation, update docs if output changes, and add a single-line `CHANGELOG.md` entry because this is user-facing diagnostics work.
