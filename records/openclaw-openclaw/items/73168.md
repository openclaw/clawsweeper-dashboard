Full review comments:

- **[P1] Clear proxy credentials with setup auth:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:475-481`
  - body: This PR adds instance-scoped bearer/basic credentials, but `clearGatewaySetupAuth()` still removes only the existing token/bootstrap/password keys. A setup reset, QR re-scan, or auth reset can leave proxy credentials stored and later sent to a different gateway, so the new keys and StateFlows need to be cleared in the same lifecycle path and covered by reset tests.
  - confidence: 0.9
- **[P2] Avoid replaying WebView requests without bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:141-143`
  - body: The interceptor recreates every same-origin request with `HttpURLConnection`, but `WebResourceRequest` exposes method, headers, and URL only, not the request body. Any canvas POST/PUT/fetch with a body will be forwarded without that body behind an authenticating proxy, so this path should avoid intercepting body-bearing methods or use a body-preserving design.
  - confidence: 0.88
- **[P1] Gate the full TLS verification bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: `disableTlsVerification` returns before certificate validation and also installs an accepting hostname verifier, so the persisted Android setting is a full trust-all TLS mode rather than only disabling fingerprint checks. This needs removal, narrowing, or explicit maintainer-approved security UX before merge.
  - confidence: 0.86

## Risks / Open Questions
