- **[P1] Clear proxy credentials during setup-auth reset:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-472`
  - body: The PR adds instance-scoped bearer/basic credentials that reconnect loads later, but `clearGatewaySetupAuth()` still clears only the existing Gateway token/bootstrap/password keys.   - confidence: 0.88
- **[P1] Remove or explicitly gate the persisted TLS bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: Returning here accepts any server certificate, and the same PR also installs a permissive hostname verifier for this mode. Because the toggle is persisted from Android UI, this is broader than a fingerprint-reset helper and needs removal, a much narrower dev-only design, or explicit maintainer security approval before merge.
  - confidence: 0.9
- **[P2] Avoid replaying WebView requests without request bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:140-142`
  - body: This interceptor rebuilds same-origin WebView requests with `HttpURLConnection` using only URL, method, and headers. Android's `WebResourceRequest` exposes no body accessor, so body-bearing canvas fetch/XHR calls can be converted into empty-body requests behind an authenticating proxy; constrain this path or use a supported request layer.
  - confidence: 0.84
