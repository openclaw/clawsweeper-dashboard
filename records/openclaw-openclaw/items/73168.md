Full review comments:

- **[P1] Clear proxy credentials with setup auth:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-472`
  - body: The PR adds instance-scoped `gateway.bearerToken`, `gateway.basicAuthUser`, and `gateway.basicAuthPassword` storage, but `clearGatewaySetupAuth()` still removes only the old token/bootstrap/password keys. After a reset or setup change, those proxy credentials can survive and be reused on reconnect, so add removals, StateFlow resets, and regression coverage for the new keys.
  - confidence: 0.9
- **[P2] Avoid replaying WebView requests without bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:140-143`
  - body: `shouldInterceptRequest` now intercepts every same-origin HTTP request with proxy auth and replays it through `HttpURLConnection`, but the replay only copies method and headers. Body-bearing canvas `fetch` or XHR calls will be forwarded without their body, so limit this path to safe requests or use a mechanism that preserves request bodies.
  - confidence: 0.84
- **[P1] Gate the full TLS verification bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: When `disableTlsVerification` is true, this returns before certificate validation and the same config also installs a permissive hostname verifier. The PR describes disabling fingerprint checks, but this is a persisted full TLS validation bypass, so it needs maintainer security approval or a narrower dev-only design before merge.
  - confidence: 0.91
