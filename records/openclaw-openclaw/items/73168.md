## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Clear proxy credentials during setup-auth reset:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-472`
  - body: The PR adds instance-scoped bearer/basic credential keys that are loaded on reconnect, but `clearGatewaySetupAuth()` still clears only the existing Gateway token/bootstrap/password keys. Scanning a new setup code or resetting setup auth can therefore reuse stale proxy credentials against a later endpoint; clear the new keys and StateFlows with the rest of setup auth.
  - confidence: 0.88
- **[P1] Remove or gate the persisted TLS validation bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: Returning here accepts any server certificate, and this mode also installs a permissive hostname verifier. Because the Android UI persists the setting across restarts, this is broader than a one-time fingerprint recovery helper and needs removal, a much narrower dev-only design, or explicit maintainer security approval before merge.
  - confidence: 0.9
- **[P2] Avoid replaying WebView requests without request bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:140-142`
  - body: This interceptor reconstructs same-origin WebView requests through `HttpURLConnection` using only URL, method, and headers, but `WebResourceRequest` does not expose the original request body. Body-bearing canvas fetch/XHR calls can be converted into empty-body requests behind an authenticating proxy, so constrain this to safe methods or use a request layer that preserves bodies. ([developer.android.com](https://developer.android.com/reference/android/webkit/WebResourceRequest))
  - confidence: 0.84
