## Summary

Keep open: this is an active Android feature PR paired with an open proxy-auth feature issue, but the current patch is not mergeable because proxy credentials can be forwarded outside the exact gateway origin and the TLS bypass needs explicit security/product review.

## What This Changes

The branch adds Android manual/onboarding bearer/basic reverse-proxy auth fields, persists them, forwards Authorization headers through gateway/canvas requests, adds a TLS verification bypass setting, and adds Android tests.

## Best Possible Solution

Land a revised Android implementation that sends proxy Authorization only to the exact configured gateway/canvas origin and separates or explicitly approves the TLS-bypass policy.

## Reproduction Assessment

Yes for the PR blockers: source inspection on PR head shows gateway-provided canvas URLs can receive proxy Authorization headers, and the interceptor scopes credentials by host:port instead of full origin. A live Android run is still needed for contributor behavior proof, but not to establish these code-path failures.

## Solution Assessment

No. The feature direction is reasonable, but the current patch is not the best mergeable solution because credential forwarding needs exact-origin scoping and the TLS bypass needs separate maintainer/security approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope canvas load auth to the gateway origin:** `apps/android/app/src/main/java/ai/openclaw/app/node/CanvasController.kt:166`
  - body: `canvas.navigate` accepts URLs supplied by the gateway, but this path sends proxy `Authorization` headers to every top-level `loadUrl` target.   - confidence: 0.94
- **[P2] Compare canvas request auth by full origin:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:132-134`
  - body: The interceptor compares only `host:port` against `remoteAddress`, so `http://host:port` and `https://host:port` both qualify for proxy Authorization. Include the scheme in the trusted-origin comparison before adding credentials.
  - confidence: 0.86
