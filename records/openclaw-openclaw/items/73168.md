Full review comments:

- **[P1] Clear proxy credentials during setup-auth reset:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-472`
  - body: These new instance-scoped proxy credentials are loaded on later reconnects, but `clearGatewaySetupAuth()` below still removes only the existing Gateway token/bootstrap/password keys. Scanning a new setup code or resetting setup auth can therefore reuse stale bearer/basic proxy credentials against a later endpoint; clear these keys and StateFlows with the rest of setup auth.
  - confidence: 0.88
- **[P1] Remove or gate the persisted TLS validation bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: Returning here accepts any server certificate, and the same PR also installs a permissive hostname verifier for this mode. Because the toggle is persisted from the Android UI, this is broader than a fingerprint-reset helper and needs removal, a much narrower dev-only design, or explicit maintainer security approval before merge.
  - confidence: 0.9
- **[P2] Avoid replaying WebView requests without request bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:140-142`
  - body: This interceptor rebuilds same-origin WebView requests with `HttpURLConnection` using only URL, method, and headers, but Android's `WebResourceRequest` does not expose the original request body and `HttpURLConnection` requires an output stream to upload one. Body-bearing canvas fetch/XHR calls can be converted into empty-body requests behind an authenticating proxy, so constrain this path or use a supported request layer. ([developer.android.com](https://developer.android.com/reference/android/webkit/WebResourceRequest?utm_source=openai))
  - confidence: 0.84
