## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve request bodies when proxying WebView requests:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:142-150`
  - body: This interceptor reissues every same-origin WebView request with `HttpURLConnection`, but it only copies the method and headers. `WebResourceRequest` exposes no request body, so POST/PUT canvas or API calls will be replayed with an empty body once proxy auth is configured. Use an approach that does not intercept body-bearing requests, or a real proxy/client path that preserves the original request semantics.
  - confidence: 0.83
- **[P1] Do not persist a full TLS trust bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74-98`
  - body: With `disableTlsVerification` set, certificate validation returns immediately and the hostname verifier accepts every host, while the UI stores this setting across restarts. Combined with the new bearer/basic credentials, that can keep sending proxy secrets over unauthenticated TLS. Remove this from the feature or make it an explicitly approved, non-default development mode with active warning behavior.
  - confidence: 0.88
- **[P2] Clear stored proxy credentials on auth reset:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:436-472`
  - body: The PR adds instance-scoped bearer/basic credential keys, but the setup-auth reset path still removes only the existing gateway token/bootstrap/password keys. Resetting or changing setup codes can leave old proxy credentials available for a later connection, so the new keys and StateFlows should be cleared with the rest of gateway setup auth.
  - confidence: 0.8
