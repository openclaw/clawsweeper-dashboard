## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve WebView request bodies before replaying requests:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:141-143`
  - body: `shouldInterceptRequest` builds a new `HttpURLConnection` with the original method and headers, but it never forwards the original request body. Android's `WebResourceRequest` API exposes no body, so same-origin POST/PUT canvas or API calls will reach the gateway empty or broken whenever proxy auth headers are enabled.
  - confidence: 0.91
- **[P1] Do not persist a full TLS trust bypass:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: When `disableTlsVerification` is enabled, server certificate checks return immediately and the hostname verifier accepts every host. Persisting that toggle in the manual connection flow weakens the TLS boundary for the newly stored proxy credentials; remove it or replace it with an explicitly approved, tightly scoped security design.
  - confidence: 0.92
- **[P2] Clear proxy credentials with setup auth reset:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-446`
  - body: The new bearer/basic credentials are saved under instance-scoped secure-pref keys, but the reset path still removes only the existing token/bootstrap/password keys. A reset or new setup-code flow can leave old proxy credentials available for the next connection, so the new keys and StateFlows need to be cleared with the rest of setup auth.
  - confidence: 0.86
