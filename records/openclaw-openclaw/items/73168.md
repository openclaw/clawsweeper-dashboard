- **[P1] Guard the TLS bypass before persisting it:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewayTls.kt:74`
  - body: `disableTlsVerification` returns before both fingerprint and system certificate checks, and the hostname verifier is also made permissive. Because the manual-connect UI persists this setting and describes it as skipping fingerprint checks, users can accidentally carry a full TLS trust bypass across restarts; make this an explicit transient/dev-only path with strong warning, or keep pinned/default validation as the persisted behavior.
  - confidence: 0.88
- **[P2] Do not replay WebView methods without request bodies:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:142-143`
  - body: `shouldInterceptRequest` can see the method and headers, but not the request body. Setting `conn.requestMethod = request.method` forwards same-origin POST/PUT requests with the injected Authorization header but without their body, so authenticated canvas XHR/form calls can be turned into empty requests; limit this interceptor to bodyless methods or move header injection to a path that preserves bodies.
  - confidence: 0.82
