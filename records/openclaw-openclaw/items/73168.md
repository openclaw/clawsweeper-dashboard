## Review Findings

Full review comments:

- **[P1] Clear proxy auth when setup auth resets:** `apps/android/app/src/main/java/ai/openclaw/app/SecurePrefs.kt:442-446`
  - body: This PR introduces new instance-scoped proxy credentials, but `clearGatewaySetupAuth()` still removes only the existing gateway token, bootstrap token, and password keys. `pairNewGateway()` and reset flows rely on that clear path, so an old reverse-proxy `Authorization` value can survive a gateway reset and be sent to a later endpoint. Add the new keys and StateFlows to the reset path and extend the existing reset tests.
  - confidence: 0.89
- **[P2] Avoid lossy WebView request replay:** `apps/android/app/src/main/java/ai/openclaw/app/ui/CanvasScreen.kt:141-143`
  - body: The new interceptor rebuilds same-origin WebView requests with `HttpURLConnection`, but `WebResourceRequest` gives this callback the URL, method, and headers, not the request body. Any protected canvas/API request with a body that is handled here will reach the gateway/proxy through a different client without its original payload, so keep auth on a faithful WebView path or limit interception to safe bodyless resources with explicit behavior for the rest.
  - confidence: 0.84
