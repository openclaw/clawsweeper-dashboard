## Summary

Close: the report is credible, but the stale Weixin context-token/account-selection behavior is owned by Tencent's external `@tencent-weixin/openclaw-weixin` plugin, while current OpenClaw core only maps and forwards the requested account/channel delivery data.

## What This Changes

Report asks OpenClaw to keep `openclaw-weixin` personal WeChat replies delivered after the conversation goes cold and the plugin's context token appears stale.

## Best Possible Solution

Fix stale context-token refresh and cold-session account selection in Tencent's external Weixin plugin, then update OpenClaw's official external catalog only after a fixed plugin release exists.

## Reproduction Assessment

No high-confidence current-main reproduction is available in this checkout. The report gives a plausible live path, and source inspection shows the relevant context-token/account-selection logic is in the external Tencent plugin, but proving it requires a live personal WeChat account and plugin runtime.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the external Weixin plugin that owns context-token storage and send semantics. OpenClaw core should only change if the plugin maintainers identify a missing SDK/API seam.
