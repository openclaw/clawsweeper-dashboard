Keep open. Current main still requires the Gateway service environment to provide BWS_ACCESS_TOKEN for the bundled Bitwarden exec resolver, and the report describes a plausible released upgrade failure that has not been superseded or fixed.

## What This Changes

The issue reports that v2026.6.6 gateways using BWS-backed SecretRefs can fail after cache expiry when their LaunchAgent environment lacks BWS_ACCESS_TOKEN.

## Best Possible Solution

Keep this open for a maintainer decision on whether to add upgrade-safe diagnostics/install/doctor guidance or restore a bounded cache-compatible path for BWS-backed exec SecretRefs.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the bundled BWS exec resolver exits when BWS_ACCESS_TOKEN is absent, and gateway startup activation fails closed when required SecretRefs cannot resolve.

## Solution Assessment

Unclear. A docs-only answer matches current docs, but the upgrade from a previously working LaunchAgent setup may need maintainer policy before choosing fail-fast strictness, doctor/install repair, or cache-compatible behavior.
