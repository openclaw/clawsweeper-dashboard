## Summary

Keep open. Current main and the latest release still require `BWS_ACCESS_TOKEN` to be present in the Gateway service environment for the bundled BWS exec resolver, and the report describes a plausible released upgrade failure that has not been fixed or superseded.

## What This Changes

The issue reports that OpenClaw v2026.6.6 gateways using Bitwarden Secrets can fail after the BWS cache expires when the LaunchAgent environment lacks `BWS_ACCESS_TOKEN`.

## Best Possible Solution

A maintainer should decide whether the intended outcome is stricter documented service-env requirements or an upgrade-safe doctor/install/reload path for existing BWS-backed LaunchAgents.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection supports the missing-token failure path because the bundled BWS resolver exits without `BWS_ACCESS_TOKEN` and gateway startup fails closed when active SecretRefs cannot resolve.

## Solution Assessment

Unclear. A docs-only answer matches current docs, but the reported upgrade from a previously working LaunchAgent setup needs maintainer policy before choosing fail-fast strictness, doctor/install repair, or cache-compatible behavior.
