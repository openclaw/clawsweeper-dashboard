## Summary

Keep open. Current main still requires the BWS exec resolver to receive `BWS_ACCESS_TOKEN` from the gateway service environment, and neither current source nor the latest release proves an upgrade-safe repair for LaunchAgents that were installed without that variable.

## What This Changes

The issue asks OpenClaw to address a v2026.6.6 BWS SecretRef upgrade failure for macOS LaunchAgent gateways missing `BWS_ACCESS_TOKEN`, either by preserving cache fallback across upgrade or clearly documenting/repairing the new environment requirement.

## Best Possible Solution

A maintainer should choose the intended upgrade policy, then either add explicit service/doctor/release guidance that catches missing BWS_ACCESS_TOKEN before expiry or implement a bounded migration/compat path for BWS-backed exec SecretRefs.

## Reproduction Assessment

No, not with high-confidence live proof. Source inspection does establish the missing-token failure path: the BWS resolver requires BWS_ACCESS_TOKEN, exec SecretRefs only pass configured environment keys, and startup fails closed when active SecretRefs cannot resolve.

## Solution Assessment

Unclear. Current docs support strict fail-fast behavior, but the reported upgrade break still needs a maintainer decision before choosing docs-only, doctor/service repair, or cache-compatible behavior as the best fix.
