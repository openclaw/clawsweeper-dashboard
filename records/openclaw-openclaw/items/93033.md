## Summary

Keep open. Current main still requires the Gateway service environment to provide BWS_ACCESS_TOKEN for BWS exec SecretRefs, and neither current main nor the latest release proves an upgrade-safe cache fallback or release/doctor migration for LaunchAgent installs that were created without that token.

## What This Changes

The issue asks OpenClaw to address a v2026.6.6 BWS SecretRef upgrade regression for macOS LaunchAgent gateways missing BWS_ACCESS_TOKEN, either by documenting/enforcing the service-env requirement or preserving cache-compatible behavior.

## Best Possible Solution

A maintainer should choose one explicit upgrade policy: strict fail-fast with doctor/install/release guidance for BWS service env, or a bounded cache-compatible path with security proof.

## Reproduction Assessment

No high-confidence live reproduction of the four-hour cache-expiry path was run. Source inspection does show that current BWS resolution requires BWS_ACCESS_TOKEN in the Gateway service env and fails closed when active SecretRefs are unavailable.

## Solution Assessment

Unclear. The current strict implementation is coherent, but this report is about shipped upgrade behavior, so the best solution depends on maintainer policy between fail-fast docs/doctor repair and a bounded cache-compatible path.
