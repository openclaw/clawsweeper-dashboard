## Summary

Keep open. Current main and v2026.6.9 still require `BWS_ACCESS_TOKEN` in the Gateway service environment for the bundled BWS resolver, and the merged Bitwarden follow-up only addressed `BWS_SERVER_URL`, not the reported LaunchAgent missing-token upgrade path.

## What This Changes

The issue reports that BWS SecretRef resolution in OpenClaw v2026.6.6 fails for LaunchAgent-managed gateways that lack `BWS_ACCESS_TOKEN` after the prior BWS cache expires.

## Best Possible Solution

Keep this open for a maintainer decision on upgrade-safe Gateway service-env diagnostics or repair, with security review for any cache fallback or broader resolver environment inheritance.

## Reproduction Assessment

No, not end to end: no live macOS LaunchAgent plus BWS cache-expiry reproduction was run. Source inspection does show the central failure path: the BWS resolver requires `BWS_ACCESS_TOKEN`, exec providers only receive explicit env, and startup fails closed for unresolved required SecretRefs.

## Solution Assessment

Unclear: the narrowest maintainable fix depends on whether maintainers want strict documented service-env requirements, doctor/install repair, or a bounded cache-compatible BWS path. A broad parent-env inheritance change is not proven to be the best fix for this specific missing LaunchAgent token case.
