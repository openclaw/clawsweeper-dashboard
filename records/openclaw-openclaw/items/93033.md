## Summary

Keep open. Current main and the latest release still require `BWS_ACCESS_TOKEN` to be available to the gateway service for the bundled BWS exec resolver, and the adjacent BWS fix only covered `BWS_SERVER_URL`, not this missing-token upgrade path.

## What This Changes

The issue reports a v2026.6.6 upgrade regression where BWS-backed SecretRefs in LaunchAgent-managed gateways fail after cache expiry unless `BWS_ACCESS_TOKEN` is present in the service environment.

## Best Possible Solution

A maintainer should decide whether OpenClaw wants strict upgrade documentation, doctor/install repair for service environment drift, or a bounded cache-compatible BWS resolver path.

## Reproduction Assessment

Yes for the core source path: an exec BWS resolver without `BWS_ACCESS_TOKEN` exits and startup fails when an active SecretRef cannot resolve. No live macOS four-hour cache-expiry reproduction was run in this read-only review.

## Solution Assessment

Unclear. Current docs support the strict contract, but the released upgrade from a previously working LaunchAgent setup needs maintainer policy before choosing fail-fast strictness, doctor/install repair, or cache-compatible behavior.
