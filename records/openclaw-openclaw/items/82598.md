## Summary

Keep open: this fixes a real CLI SecretRef gap and includes terminal proof, but the patch resolves web-search secrets before honoring the CLI provider override, which can require or fetch an unrelated configured provider key and break explicit provider runs.

## What This Changes

The branch adds web-search command SecretRef resolution, a web-search-only target selector, and tests for the CLI web-search call path and target set.

## Best Possible Solution

Honor the same provider selection that runWebSearch will execute when choosing command secret targets, and keep regression coverage for both the default SecretRef path and a --provider override path.

## Reproduction Assessment

Yes, from source and PR evidence: current main passes raw config into runWebSearch, and the PR shows the unresolved Exa file SecretRef error before the patch. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The fix should resolve the web-search SecretRef for the provider the CLI will actually execute, including --provider, rather than resolving every configured search credential before dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scope secret resolution to the selected provider:** `src/cli/capability-cli.ts:1534`
  - body: When the CLI caller passes `--provider`, `runWebSearch` treats that provider as explicit, but this new preflight resolves the configured web-search SecretRefs before `providerId` is applied. A config with Exa as default and a file-backed SecretRef can now fail `openclaw infer web search --provider duckduckgo ...` when Exa's gateway-backed key is unavailable, even though DuckDuckGo does not need that key. Please make the command-secret scope honor `params.provider` and cover that override path.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or CI execution, but it broadens CLI access to web-search secrets beyond the provider being executed.

Concerns:

- **[medium] Overbroad web-search secret resolution:** `src/cli/capability-cli.ts:1534`
  - body: The command asks the resolver for the whole web-search credential target set before applying `params.provider`, which can retrieve or require unrelated provider API keys for a single-provider CLI call. Scope resolution to the selected provider path where possible.
  - confidence: 0.78
