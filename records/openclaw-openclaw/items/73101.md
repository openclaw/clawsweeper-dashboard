## Summary

Keep open for normal maintainer review: current main still lacks Slack configured ACP binding support, while this PR implements it inside the Slack plugin boundary with tests, docs, live proof, and no blocking correctness or security findings from this review.

## What This Changes

The PR adds Slack configured ACP binding compilation/matching, configured-route fallback with readiness/drop handling, stable thread session keys for configured ACP sessions, tests, docs, and a changelog entry.

## Best Possible Solution

Merge this Slack-owned implementation after normal maintainer approval and branch policy gates, keeping the route/session behavior covered by the new tests and documented on the ACP agents page.

## Reproduction Assessment

Yes, at source level. Current main has the configured-binding compiler and sibling channel adapters, but Slack lacks the required `bindings` adapter and its inbound route never invokes the configured-binding resolver.

## Solution Assessment

Yes. Adding the adapter and configured-route readiness check inside the Slack plugin follows the established Discord, Telegram, Feishu, Matrix, iMessage, and LINE pattern without adding core Slack-specific policy.
