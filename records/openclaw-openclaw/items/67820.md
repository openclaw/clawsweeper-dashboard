## Best Possible Solution

Rebase and narrow this work onto current main so active QR reuse happens only for still-pending logins, runtime warnings are preserved when linked state is unknown, and the current QR-rotation and gateway stop/start behavior remain intact.

## Reproduction Assessment

## Solution Assessment

No. The requested behavior is valid, but the branch should be rebased onto current main and the active-QR helper should return a cached QR only while the login is still pending.
