  - confidence: 0.9

  - body: Current main only stops a running channel up front for forced login, or after a non-forced call actually takes over with a QR.   - confidence: 0.88
- **[P1] Filter finished active logins before returning a QR:** `extensions/whatsapp/src/login-qr.ts:84-92`
  - body: `readActiveQrResult` returns any fresh `qrDataUrl` without checking whether that login has already connected or failed. Because the gateway calls this before preflight, a second start after a scan or failure can return a stale or consumed QR instead of the real terminal state.
  - confidence: 0.86
