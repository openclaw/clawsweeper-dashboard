## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve QR rotation while reusing active QR:** `extensions/whatsapp/src/login-qr.ts:236-238`
  - body: Current main keeps streaming QR state: `waitForWebLogin` accepts `currentQrDataUrl` and returns a refreshed `qrDataUrl` when WhatsApp rotates the code. This branch ignores later `onQr` emissions once `pendingQr` is set and removes the wait payload, so clients can keep showing a stale QR after rotation. Rebase the existing-QR hook on top of the current QR refresh path instead of replacing it.
  - confidence: 0.9
- **[P1] Preserve delayed channel stop on web login start:** `src/gateway/server-methods/web.ts:109-110`
  - body: Current main only stops an already-running WhatsApp channel up front for forced login, or after a non-forced call actually takes over with a QR. This branch stops the channel before login whenever existing/preflight return null, so a non-QR failure can interrupt a running account that main would have left running or restored. Keep the delayed-stop logic when adding the existing/preflight checks.
  - confidence: 0.88
- **[P1] Filter finished active logins before returning a QR:** `extensions/whatsapp/src/login-qr.ts:84-92`
  - body: `readActiveQrResult` returns any fresh `qrDataUrl` without checking `existing.connected` or `existing.error`. Because the gateway calls this before preflight, a second start after the QR has been scanned or the login failed can return a stale or consumed QR instead of the linked or failed state. Clear or ignore terminal active logins before reusing the QR.
  - confidence: 0.86
- **[P2] Keep auth-stabilizing status warnings:** `extensions/whatsapp/src/status-issues.ts:61`
  - body: The status rewrite removes `statusState` from the snapshot reader and deletes the unstable branch. A snapshot with `statusState: "unstable"` now falls through instead of reporting `Auth state is still stabilizing`, so the PR can hide the auth barrier warning it needs to preserve. Keep the unstable warning and then allow runtime warnings when linked is unknown.
  - confidence: 0.84
