## Summary

Keep open. Current main still has the Telegram multi-account resolver mismatch, and the linked implementation PR is closed without merging, so this remains a narrow Telegram plugin bug rather than cleanup-close material.

## What This Changes

The issue asks Telegram multi-account group/topic `systemPrompt`, `skills`, and topic config to resolve consistently with `requireMention` when the config is placed at `channels.telegram.groups`.

## Best Possible Solution

Keep the #30673 routing guard in `mergeTelegramAccountConfig`, but add a Telegram-owned group context lookup that falls back from account-scoped groups to channel-level groups for prompt, skill, and topic resolution only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure two Telegram accounts plus channel-level `channels.telegram.groups`, and the merged account config omits `groups` while `resolveTelegramGroupRequireMention` still sees the channel-level block.

## Solution Assessment

Yes. The narrowest maintainable fix is to align the prompt/skill/topic resolver with the existing account-level then channel-level mention fallback, without changing account routing or merged config inheritance semantics.
