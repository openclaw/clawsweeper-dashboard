Keep open. Current main still has the Telegram multi-account resolver mismatch: prompt/skills/topic resolution reads only merged account groups, while mention policy still falls back to channel-level groups; the linked fix PR is closed unmerged and no newer canonical fix was found.

## Repair Work Prompt

Fix the Telegram multi-account group/topic prompt config mismatch reported in https://github.com/openclaw/openclaw/issues/64609. In multi-account setups, channel-level `channels.telegram.groups` currently still affects `requireMention` through `group-policy.ts`, but `systemPrompt`, `skills`, and topic settings are ignored because `resolveTelegramGroupConfig` reads only merged account groups from `resolveTelegramAccount`, which intentionally drops channel-level groups for routing safety. Implement a Telegram-owned prompt/skill/topic lookup that prefers `channels.telegram.accounts.<accountId>.groups` and falls back to `channels.telegram.groups` when no account-scoped group config exists, without changing `mergeTelegramAccountConfig` routing semantics or reintroducing channel-level group inheritance into account delivery. Add regression coverage for multi-account channel-level group/topic `systemPrompt` and precedence where account-scoped groups win. Do not add a new config option or change multi-account membership/routing behavior from the earlier group leak fix. Add a `CHANGELOG.md` fix entry if this becomes a user-facing fix PR.
