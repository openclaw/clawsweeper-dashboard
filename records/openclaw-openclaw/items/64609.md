## Summary

Keep open. Current main still has the Telegram multi-account resolver mismatch: `requireMention` falls back from account groups to channel-level groups, but group/topic prompt and skill lookup still reads only the merged account config, where channel-level groups are intentionally removed for multi-account setups.

## What This Changes

The issue asks OpenClaw to make Telegram multi-account group and topic `systemPrompt`/`skills` resolution consistent with the existing `requireMention` fallback or warn when channel-level group config is ignored.

## Best Possible Solution

Add Telegram-owned prompt, skill, and topic lookup that falls back from account-scoped groups to channel-level groups while keeping multi-account routing and account config merging fail-safe, with regression coverage for fallback and account override precedence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure two Telegram accounts plus channel-level `channels.telegram.groups`, and prompt/topic resolution sees no groups while mention policy still sees the same channel-level block.

## Solution Assessment

Yes. The narrow maintainable fix is to align prompt/skill/topic lookup with the existing account-level then channel-level fallback while preserving the multi-account routing guard from the earlier message-loss fix.
