## Summary

Keep open. Current main still has the Telegram multi-account resolver mismatch: prompt, skills, and topic prompt resolution only reads merged account groups, while mention policy still falls back to channel-level groups; the related fix PR was closed unmerged and no open replacement was found.

## What This Changes

Report that Telegram multi-account setups ignore channel-level group/topic systemPrompt and skills while requireMention from the same groups block still applies.

## Best Possible Solution

Add a Telegram-owned prompt, skill, and topic lookup that falls back from account-scoped groups to channel-level groups without changing merged account routing or access-control semantics.

## Solution Assessment

Yes. The narrow maintainable fix is to align prompt/skill/topic lookup with the existing account-level then channel-level fallback while preserving the multi-account routing guard from the earlier leak fix.
