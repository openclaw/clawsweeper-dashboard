## Summary

Keep open. Current main still resolves Telegram group/topic prompts and skills only from the merged account snapshot, while mention gating still falls back to channel-level groups; the linked fix PR is closed and unmerged.

## What This Changes

The issue asks for Telegram multi-account group/topic `systemPrompt`, `skills`, and topic settings to resolve consistently with `requireMention`, or for users to be warned when channel-level groups are unsupported in multi-account mode.

## Best Possible Solution

Implement a Telegram-owned group context fallback for prompt, skill, and topic lookup: use account-scoped groups first, then channel-level groups when the account has no groups, while leaving account routing and merged account config isolation unchanged. Add a regression test for two Telegram accounts with channel-level group/topic `systemPrompt` and `skills`; update docs or doctor warnings only if maintainers intentionally choose not to support that config shape.

## Reproduction Assessment

Yes. The current source gives a high-confidence static reproduction: two configured Telegram accounts plus channel-level `groups` makes the merged account config omit `groups`, so `resolveTelegramGroupConfig` returns no prompt/topic config while `resolveTelegramGroupRequireMention` still falls back to the same channel-level block.

## Solution Assessment

Yes. The narrow maintainable fix is a Telegram prompt/context resolver fallback, not a broad change to `mergeTelegramAccountConfig`, because the latter owns the multi-account routing guard from the earlier message-loss bug.
