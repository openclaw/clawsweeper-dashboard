## Summary

Keep open. Current main still has the Telegram multi-account resolver mismatch: prompt/skill/topic resolution only reads merged account groups, while the mention policy still falls back to channel-level groups, and the linked fix PR is closed unmerged.

## What This Changes

The issue requests consistent Telegram multi-account group/topic config resolution so channel-level `systemPrompt`, `skills`, and `topics` are honored like `requireMention` without reintroducing cross-account routing inheritance.

## Best Possible Solution

Add a Telegram-owned prompt/skill/topic lookup that falls back from account-scoped groups to channel-level groups without changing merged account config or account routing semantics, then cover the multi-account regression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two Telegram accounts plus channel-level `channels.telegram.groups`, and the prompt resolver sees no groups while the mention resolver still sees the same channel-level block.

## Solution Assessment

Yes. The narrow maintainable fix is to align prompt/skill/topic lookup with the existing account-level then channel-level fallback while keeping access control and account routing scoped to account config.
