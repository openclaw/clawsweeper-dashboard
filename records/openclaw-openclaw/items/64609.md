## Summary

Keep open. Current main still has the Telegram multi-account resolver mismatch: merged account config drops channel-level groups, while the mention-policy path still falls back to them, and the linked implementation PR is closed unmerged.

## What This Changes

The issue asks Telegram multi-account setups to apply channel-level group/topic `systemPrompt`, `skills`, and topic config consistently with `requireMention` without reintroducing cross-account group routing.

## Best Possible Solution

Add a Telegram-owned group context lookup that falls back from account-scoped groups to channel-level groups for prompt, skill, and topic resolution only, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two Telegram accounts plus channel-level `channels.telegram.groups`, and `resolveTelegramAccount(...).config.groups` is empty while `resolveTelegramGroupRequireMention` still sees the same channel-level block.

## Solution Assessment

Yes. The narrow maintainable fix is to align the prompt/skill/topic resolver with the existing account-level then channel-level mention fallback, without changing account routing or merged config inheritance semantics.
