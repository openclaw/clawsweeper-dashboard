## Summary

Keep open. Current `main` still has the Telegram multi-account resolver mismatch: prompt/skill/topic resolution reads only the merged account groups, while mention policy still falls back to channel-level groups, and the linked fix PR is closed unmerged.

## What This Changes

Telegram multi-account channel-level group/topic `systemPrompt`, `skills`, and `topics` config is reported as ignored while `requireMention` from the same group block still applies.

## Best Possible Solution

Add a Telegram-owned prompt/skill/topic lookup that falls back from account-scoped groups to channel-level groups without changing merged account routing or access-control semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two Telegram accounts plus channel-level `channels.telegram.groups`, and prompt/topic resolution sees no groups while mention policy still sees the same channel-level block.

## Solution Assessment

Yes. The narrow maintainable fix is to align prompt/skill/topic lookup with the existing account-level then channel-level fallback while keeping account routing and access-control fields scoped to account config.
