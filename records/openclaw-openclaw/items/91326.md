Keep open for normal PR review: the fix targets a real Control UI bug and is mostly in the right layer, but the external PR still has only mock/unit proof and the new soft-reset detector should match the gateway parser's whitespace semantics before merge.

## What This Changes

The PR updates the Control UI reset slash-command dispatch to forward `/reset` arguments and adds regression tests for typed and button-triggered soft reset behavior.

## Best Possible Solution

Preserve the full `/reset <args>` body at the Control UI dispatch boundary, classify soft reset with the same whitespace boundaries as the gateway parser, keep focused regression coverage, and require redacted real Control UI/gateway proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on current main, `parseSlashCommand` preserves `args`, but `dispatchSlashCommand` sends a hardcoded `/reset`. I did not execute tests because this review was required to keep the checkout read-only.

## Solution Assessment

Mostly: preserving args in the Control UI dispatcher is the right owner-boundary fix. The implementation should use gateway-equivalent whitespace matching for soft reset and still needs real behavior proof before merge.
