## Summary

Keep open. Current main intentionally detects unreferenced transcript files and can archive or prune them, but it does not provide a safe recovery path to re-add orphaned transcript files as normal sessions; the same author also has an open bug about recurring orphan creation at https://github.com/openclaw/openclaw/issues/76681, so this feature should stay available for maintainer product review.

## What This Changes

Request a way to add orphaned session transcript files back into OpenClaw’s session store so they appear as normal sessions instead of being archived.

## Best Possible Solution

Design an explicit preview-first recovery workflow that reconstructs enough metadata from a transcript, chooses a safe session key and agent, and keeps doctor’s orphan warning intact while the root-cause bug is investigated.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows the requested recovery path is absent, while the linked bug still lacks a high-confidence current-main sequence that creates new orphan files.

## Solution Assessment

Unclear. Re-adding files blindly is not yet the best confirmed solution; maintainers should define preview, metadata, ownership, and reconciliation semantics before implementation.
