## Summary

Keep open: current main only detects and archives orphan transcript files, while the requested adoption/recovery flow is a new feature that still needs maintainer design around session keys, metadata reconstruction, and safety.

## What This Changes

The issue requests a way to re-add orphaned transcript files to OpenClaw so they appear as normal sessions instead of only being reported or archived by doctor.

## Best Possible Solution

Design an explicit doctor or sessions restore flow that previews orphan transcript candidates, safely infers or asks for session keys/agent metadata, and refuses adoption when provenance is ambiguous.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main can detect/archive orphan transcripts but has no recovery/adoption flow to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract. Re-adding files directly to sessions.json may be useful, but the safer design needs an explicit preview/restore command with metadata and privacy safeguards rather than automatic doctor adoption.
