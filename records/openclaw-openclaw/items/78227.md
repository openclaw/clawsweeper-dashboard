## What This Changes

The PR removes `isConnecting === true` from Discord gateway startup's socket-started check and flips the proxy regression test to expect base registration when only that flag is set.

## Solution Assessment

No. The linked readiness bug was resolved through the merged heartbeat scheduler fix, and this PR's stale-flag path is not sufficient because the real base gateway will return before creating a WebSocket while `isConnecting` remains true.
