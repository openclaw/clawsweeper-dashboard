## Best Possible Solution

Close this PR as superseded by https://github.com/openclaw/openclaw/pull/92639.

## Reproduction Assessment

Yes. Source inspection on current main shows raw Windows backslashes in `memory.qmd.command` are consumed by `splitShellArgs` before the resolved command reaches QMD spawn resolution; I did not run a live Windows QMD setup.

## Solution Assessment

Yes, mostly. Preserving Windows absolute command paths before POSIX-style shell parsing is the right boundary, but this PR still needs focused regression tests and real Windows/OpenClaw proof before it is the best merge-ready fix.
