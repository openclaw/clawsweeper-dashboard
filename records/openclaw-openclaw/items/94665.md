## Summary

Keep this PR open: the patch is small and aimed at the right QMD command-normalization boundary, but it still needs redacted Windows/OpenClaw behavior proof and focused regression coverage before merge.

## What This Changes

The PR changes `memory.qmd.command` resolution so drive-letter and UNC-style Windows absolute command paths bypass `splitShellArgs` and keep their backslashes.

## Best Possible Solution

Land a focused parser fix with regression tests for drive-letter and UNC Windows absolute commands plus redacted Windows QMD command-resolution proof, then close the linked issue.

## Reproduction Assessment

Yes. Source inspection on current main shows `memory.qmd.command` still flows through `splitShellArgs`, whose unquoted backslash handling strips Windows separators before QMD spawn resolution.

## Solution Assessment

Mostly yes. Preserving Windows absolute commands before POSIX-style parsing is the right boundary, but the PR should add focused regression tests and real Windows/OpenClaw proof before merge.
