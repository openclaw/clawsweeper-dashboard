## Summary

Keep open: the patch targets the right QMD config-parser boundary, but it is not merge-ready because the new UNC check fails lint, there is no focused regression coverage for the new parser branch, and the supplied proof is Linux logic output rather than a real Windows/OpenClaw QMD run.

## What This Changes

The PR changes QMD backend command parsing so Windows drive-letter and UNC absolute command strings bypass POSIX-style shell argument splitting.

## Best Possible Solution

Land a focused parser fix after replacing the lint-failing UNC regex, adding drive-letter and UNC backend-config regression tests, and supplying redacted Windows QMD command-resolution proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `memory.qmd.command` is parsed by `splitShellArgs`, and that parser consumes unquoted backslashes before the value reaches QMD spawn resolution.

## Solution Assessment

Mostly yes, but not merge-ready. Preserving Windows absolute command paths before POSIX-style parsing is the right boundary; this PR still needs the lint fix, focused regression tests, and real Windows/OpenClaw proof.
