## Summary

Current main still has the reported Telegram album notification gap: recoverable media-group fetch failures are logged and skipped, then the remaining media are sent to the agent without a Telegram warning. The related gateway batching request in #39343 is broader and does not supersede this narrow plugin bug.

## What This Changes

The issue requests that Telegram media groups warn the user when one or more album images cannot be downloaded instead of silently forwarding only the successful images.

## Best Possible Solution

Add a Telegram plugin fix that preserves partial album success, counts recoverable exceptions and null media results during `processMediaGroup`, and sends one concise warning message describing how many album items were skipped.

## Reproduction Assessment

Yes. The existing channel-post media-group test already simulates an album with one failed download; the current source path shows the failed item is skipped and only the remaining media are processed, with no user-facing warning.

## Solution Assessment

Yes. A narrow Telegram plugin change in `processMediaGroup` is the maintainable fix; this does not need a core gateway batching change or a new plugin API.
