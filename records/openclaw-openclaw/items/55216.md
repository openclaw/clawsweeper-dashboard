## Summary

Keep open. Current main still skips recoverable Telegram media-group download failures and forwards the reduced album without a user-visible Telegram warning; the media-group flush timer is now configurable, but that does not address the reported silent image-loss notification gap.

## What This Changes

The issue asks the Telegram plugin to notify users when one or more items in a media group fail to download or resolve to null instead of silently sending fewer images to the agent.

## Best Possible Solution

Add a narrow Telegram-plugin media-group warning that preserves partial-success delivery, counts skipped or null album items, and sends one concise Telegram notice tied to the album’s primary message.

## Reproduction Assessment

Yes, at source level. The current media-group loop catches recoverable item failures, continues with the remaining media, and has no bot.api.sendMessage warning path; the existing channel-post test already simulates this partial-success case.

## Solution Assessment

Yes for the core issue: a narrow Telegram plugin fix is the best path. Gateway batching, broad retry policy, and download-integrity changes should stay separate unless independently reproduced.
