## Summary

Keep open. Current main still has the central silent-loss path: Telegram media groups forward the successfully resolved items after recoverable per-item failures or null media resolution, but the album handler does not send a user-visible warning for skipped items.

## What This Changes

The issue requests a Telegram media-group warning when one or more album photos fail to download or resolve and are skipped.

## Best Possible Solution

Add a narrow Telegram plugin fix that counts recoverable and null skipped media-group items, preserves partial-success forwarding, and sends one concise warning tied to the album’s primary message.

## Reproduction Assessment

Yes, at source level. Current main catches recoverable per-item media failures in processMediaGroup, continues with remaining media, silently ignores null media, and has no album warning send path.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram media-group loop or a small helper it calls; gateway batching, new config, and partial-file integrity work should stay separate unless independently reproduced.
