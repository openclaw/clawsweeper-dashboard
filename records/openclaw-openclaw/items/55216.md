## Summary

Keep open. Current main still logs and skips recoverable Telegram media-group download failures, then forwards the remaining album media without a user-visible Telegram warning; the broader gateway batching issue does not cover this notification gap.

## What This Changes

The issue requests that Telegram album/media-group processing warn the user when one or more photos fail to download instead of silently forwarding only the successful images.

## Best Possible Solution

Implement the warning inside the Telegram plugin’s media-group handler: count recoverable exceptions and null media results, keep successful media flowing to processMessage, and send one Telegram warning that states how many album items could not be fetched.

## Reproduction Assessment

Yes. The main issue is reproducible from source and existing tests: the current channel-post media-group test simulates a second-photo fetch failure and the handler continues with the remaining media, while no code path sends a Telegram warning for that skipped item.

## Solution Assessment

Yes. A narrow Telegram-plugin fix is the best path; gateway-wide batching, timer configurability, or download-integrity changes would broaden the scope beyond the proven notification bug.
