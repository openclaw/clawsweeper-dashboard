## Summary

Keep open. Current main still forwards a reduced Telegram media group after recoverable per-item download failures or null media resolution, and there is no user-visible album warning path for the skipped items.

## What This Changes

The issue asks OpenClaw to notify Telegram users when one or more images in an album/media group could not be downloaded and were skipped.

## Best Possible Solution

Add a narrow Telegram plugin fix that counts skipped or null media-group items, preserves partial-success forwarding, and sends one concise, deduped warning tied to the album’s primary message.

## Reproduction Assessment

Yes, at source level. The current media-group loop catches recoverable item failures, continues with the remaining media, silently ignores null media, and has no bot.api.sendMessage warning path for the album.

## Solution Assessment

Yes for the central bug: the maintainable fix belongs in the Telegram media-group loop with a shared/deduped warning helper. Gateway batching, new config, and partial-file integrity checks should stay separate unless independently reproduced.
