## Summary

Keep open. Current main still has the central silent-loss path: Telegram media groups keep forwarding successfully resolved album media after recoverable per-item failures or null media resolution, but the album handler does not send a user-visible warning for skipped items. Related merged PRs fixed whole-group drops, getFile retry/null fallback, and caption-placeholder visibility, but not this skipped-item notification path.

## What This Changes

Request that Telegram albums warn users when one or more photos fail to download instead of silently forwarding fewer images.

## Best Possible Solution

Add a narrow Telegram plugin fix that counts skipped album items from recoverable failures and null media returns, preserves partial-success forwarding, and sends one concise warning tied to the album’s primary message; handle partial-file integrity separately if reproduced.

## Reproduction Assessment

Yes, at source level. Current main catches recoverable per-item media failures in processMediaGroup, continues with remaining media, silently ignores null media, and has no album warning send path.

## Solution Assessment

Yes for the warning gap. The narrow maintainable fix belongs in the Telegram media-group loop or a small helper it calls; gateway batching, new config, and partial-file integrity work should stay separate unless independently reproduced.
