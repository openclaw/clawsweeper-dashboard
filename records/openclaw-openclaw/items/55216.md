## Summary

Keep open. Current main still skips recoverable Telegram media-group download failures and null media results while forwarding the reduced album, so the user-visible warning gap remains.

## What This Changes

The issue asks Telegram albums to notify users when one or more photos fail to download instead of silently forwarding fewer images.

## Best Possible Solution

Add a narrow Telegram-plugin media-group warning that counts skipped or null album items, preserves partial-success delivery, and sends one deduped notice tied to the album’s primary message.

## Reproduction Assessment

Yes, at source level. The current media-group loop catches recoverable item failures, continues with the remaining media, and has no bot.api.sendMessage warning path; the existing album test already simulates this partial-success case.

## Solution Assessment

Yes for the central bug: a narrow Telegram plugin fix is the best path. Gateway batching, broad retry policy, and deeper download-integrity changes should stay separate unless independently reproduced.
