## Summary

Keep open. The PR targets a real current-main memory indexing bug and has useful live proof, but it is not merge-ready because structured embedding inputs can still carry unsanitized text parts to providers and the head is currently conflicting.

## What This Changes

The PR updates memory chunking and embedding retry wrappers to preserve UTF-16 surrogate pairs, sanitize embedding text, and add focused unit/regression tests.

## Best Possible Solution

Land a repaired current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes for the underlying bug. Current main has a high-confidence source reproduction: the coarse chunker slice can split a flower emoji into lone surrogate halves at maxChars=40 before any guard runs.

## Solution Assessment

Mostly yes. The chunker fix plus embed-boundary sanitizer is the narrow maintainable direction, but the PR should also sanitize structured text parts and be brought onto current main.
