## Summary

Keep open. The bug is still present on current main, and this PR has the right core direction, but it is not merge-ready because the sanitizer misses remote batch and structured text-part embedding paths and the PR is currently dirty against main.

## What This Changes

The PR changes memory chunking to avoid coarse UTF-16 surrogate splits, adds a lone-surrogate sanitizer around embedding inputs, and adds focused memory regression tests.

## Best Possible Solution

Land a refreshed patch that keeps the coarse split guard and applies one sanitizer to every provider-bound text field across inline, structured, and remote batch embedding paths.

## Reproduction Assessment

Yes, at source level. Current main's coarse loop can split 39 ASCII characters plus a flower emoji at a 40-code-unit boundary into chunks ending and starting with lone surrogate halves.

## Solution Assessment

No, not as currently patched. The chunker guard is the right root-cause fix, but the sanitizer must also cover remote batch chunks and structured text parts before the fix is complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize remote batch chunks before dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259`
  - body: The new sanitizer only runs in `embedBatchWithRetry`. Remote batch mode still sends `missingChunks` directly to `providerRuntime.batchEmbed`, and the OpenAI/Voyage/Gemini adapters serialize `chunk.text` or `chunk.embeddingInput` into provider requests, so a lone surrogate can still fail the indexing run whenever remote batch is enabled.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:295`
  - body: `sanitizedInputs` replaces only `input.text`. Gemini request construction uses `parts[].text` when parts are present, so structured memory inputs can bypass the sanitizer and hit the same provider-side Unicode failure.
  - confidence: 0.9
