## Summary

Keep open: current main still has the surrogate-splitting memory bug and this PR is useful, but the patch misses provider runtime batch chunks and text parts inside structured embedding inputs, so it needs a focused repair before merge.

## What This Changes

The PR adds a surrogate-aware coarse split in memory chunking, an embedding-input sanitizer for lone UTF-16 surrogates, and focused regression tests.

## Best Possible Solution

Land a rebased or replacement memory patch that guards coarse chunk boundaries and sanitizes every provider-bound text field before inline, structured, and batch embedding calls, with regression coverage.

## Reproduction Assessment

Yes at source level. Current main's coarse loop can split 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate chunks before embedding.

## Solution Assessment

No, not yet. The chunker guard and sanitizer direction are right, but the PR must also sanitize provider runtime batch chunks and structured text parts to fully close the embedding boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize batch chunks before provider runtime dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259`
  - body: The new sanitizer only runs in the inline retry wrappers, but remote batch indexing still passes missingChunks directly to providerRuntime.batchEmbed. OpenAI, Voyage, and Gemini batch adapters serialize chunk.text or chunk.embeddingInput into provider request bodies, so batch mode can still send a lone surrogate and fail the reindex.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:295`
  - body: sanitizedInputs replaces input.text only. For multimodal chunks, input.parts can include a text part, and Gemini serializes part.text when parts are present, so structured embedding inputs can still bypass the sanitizer with the same lone-surrogate failure.
  - confidence: 0.9
