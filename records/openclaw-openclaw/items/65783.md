## Summary

The PR fixes a valid memory-indexing bug directionally, but it still leaves structured embedding text parts unsanitized, so Gemini or other structured providers can receive the same invalid UTF-16 payload through `parts[].text`.

## What This Changes

The PR updates memory chunking, embedding input handling, and focused tests to preserve surrogate pairs and replace unpaired surrogates before embedding calls.

## Best Possible Solution

Land a current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured `parts[].text`, while leaving retry and checkpointing policy to the existing follow-up issues.

## Reproduction Assessment

Yes. Source inspection shows current main still slices a mostly-ASCII line at `maxChars=40`, so `39` ASCII characters followed by an emoji puts the coarse boundary between the high and low surrogate before any guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and embed-boundary sanitizer are the right shape, but the PR must also sanitize structured text parts before passing them to providers such as Gemini.
