## Summary

Keep open. The PR is a useful fix for a source-reproducible memory indexing bug, but the latest head still leaves structured embedding text parts unsanitized before provider calls, so the patch is not ready to merge as-is.

## What This Changes

The PR updates memory chunking and embedding retry wrappers to preserve UTF-16 surrogate pairs, sanitize embedding text, and add focused unit/regression tests.

## Best Possible Solution

Land a repaired current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured text-part embedding inputs before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: the coarse chunker slice can split 39 ASCII characters plus a flower emoji into lone high and low surrogate chunks at maxChars=40 before the inner guard runs.

## Solution Assessment

Mostly yes. The chunker boundary guard plus embed-boundary sanitizer is the narrow maintainable direction, but the PR should also sanitize structured text parts before provider calls and be reconciled with current main.
