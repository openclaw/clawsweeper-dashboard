## Summary

Keep open: the bug is real on current main and the PR is close, but it still leaves a provider-visible structured text path unsanitized, so it needs a narrow repair before merge.

## What This Changes

The PR changes memory chunking to avoid UTF-16 surrogate-pair splits, adds embedding input sanitization, and adds focused tests.

## Best Possible Solution

Land a repaired patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding input before provider serialization, while keeping retry/checkpointing work in separate reports.

## Reproduction Assessment

Yes, at source level. Current main's coarse split can place `\uD83C` at the end of one chunk and `\uDF38` at the start of the next for the reported mostly-Latin emoji boundary, and the embedding retry wrappers currently forward unsanitized text.

## Solution Assessment

No, not as submitted. The chunker fix and embed-boundary sanitizer are the right direction, but structured `parts[].text` must also be sanitized before providers serialize the request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
