## Summary

Keep open. The PR targets a real current-main memory indexing bug, but the submitted patch still leaves structured embedding text parts unsanitized, so it should not merge as-is and it is not obsolete on main.

## What This Changes

The PR changes memory chunking to avoid coarse UTF-16 surrogate splits, adds embed-boundary surrogate sanitization, and adds focused tests.

## Best Possible Solution

Land a current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured `parts[].text`, while leaving retry/checkpointing policy to existing follow-ups.

## Reproduction Assessment

Yes. Source inspection plus a small read-only Node probe show the current coarse split can place the boundary between `\uD83C` and `\uDF38` for `39` ASCII characters followed by an emoji at `maxChars=40`.

## Solution Assessment

No, not as submitted. The chunker fix and top-level embed sanitizer are the right direction, but the final patch also needs to sanitize structured text parts before providers such as Gemini serialize them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
