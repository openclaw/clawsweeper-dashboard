## Summary

Keep open: this PR targets a real current-main memory indexing bug and includes useful live proof, but the patch is still incomplete because structured embedding text parts can bypass the sanitizer.

## What This Changes

The PR changes memory chunking to avoid UTF-16 surrogate-pair splits, adds an embedding-input sanitizer, and adds focused regression tests.

## Best Possible Solution

Land a repaired current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider calls.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: with maxChars=40, 39 ASCII characters followed by an emoji makes the coarse slice boundary land between the high and low surrogate before the fine guard runs.

## Solution Assessment

Mostly yes. The chunker plus embed-boundary sanitizer is the narrow maintainable fix, but the PR should also sanitize text parts inside structured inputs and be brought onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87
