## Summary

This PR should stay open for a narrow repair: the reported memory-indexing bug is valid, but the current PR head still leaves structured embedding text parts unsanitized for providers that serialize `parts` instead of the sanitized top-level text.

## What This Changes

The PR changes memory chunking to avoid splitting UTF-16 surrogate pairs, adds lone-surrogate sanitization at embedding entry points, and adds focused memory regression tests.

## Best Possible Solution

Land the chunker boundary fix and sanitize every text-bearing embedding field, including structured `parts[].text`, while leaving broader retry and checkpointing behavior to the existing follow-up issues.

## Reproduction Assessment

Yes. Source inspection plus a focused Node probe show the current coarse chunk boundary can split an emoji between `\uD83C` and `\uDF38`, and the paired issue includes live before/after indexing evidence.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured text parts must also be sanitized before providers such as Gemini serialize them.
