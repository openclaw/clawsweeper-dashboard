## Summary

Keep open. The PR fixes the central chunker boundary bug, but the proposed sanitizer still misses provider runtime batch chunks and structured text parts, so unpaired surrogates can still reach embedding providers.

## What This Changes

The PR adds a surrogate-pair guard to memory chunking, a lone-surrogate sanitizer at embedding retry wrappers, and focused regression tests.

## Best Possible Solution

Land a rebased or replacement patch that guards coarse chunk boundaries and sanitizes every provider-bound text field before inline, structured, and remote batch embedding calls, with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's coarse loop can split 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate chunks before embedding.

## Solution Assessment

No, not yet. The chunker guard and top-level sanitizer are the right direction, but the PR must also cover provider runtime batch chunks and structured text parts before it fully fixes the embedding boundary.
