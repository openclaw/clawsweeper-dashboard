## Summary

Keep open. The PR addresses a valid memory-indexing bug that is still present on current main, but the submitted sanitizer only rewrites top-level structured input text and misses `parts[].text`, so Gemini-style structured embedding requests can still send lone surrogates.

## What This Changes

The PR changes memory chunking and embedding retry code to preserve UTF-16 surrogate pairs, sanitize embedding text, and add focused regression tests.

## Best Possible Solution

Land a current-main patch that fixes the coarse chunk boundary and sanitizes every text-bearing embedding field before provider calls, while leaving retry and checkpointing policy to the existing follow-up issues.

## Reproduction Assessment

Yes. Source inspection plus a focused Node probe show the current coarse split can place a chunk boundary between `\uD83C` and `\uDF38` for `39` ASCII characters followed by an emoji at `maxChars=40`.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but the PR must also sanitize structured `parts[].text` before providers such as Gemini serialize those fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
