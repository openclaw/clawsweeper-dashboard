## Summary

Keep open: current main still has the surrogate-splitting and unsanitized embedding problem, and this PR is useful but still misses remote batch and structured text-part paths.

## What This Changes

The PR updates memory chunking and embedding wrappers to avoid splitting surrogate pairs and replace unpaired surrogates before embedding, with focused regression tests.

## Best Possible Solution

Land a current-main repair that keeps the chunker boundary guard and applies one shared sanitizer to every embedding-provider text boundary, including structured parts and remote batch payloads, with regression coverage.

## Reproduction Assessment

Yes, at source level. A read-only Node probe matching current main's coarse loop with maxChars=40 and 39 ASCII characters plus a flower emoji produced one chunk ending in high surrogate d83c and the next starting with low surrogate df38.

## Solution Assessment

No, not yet. The PR's chunker guard and top-level embed sanitizer are the right direction, but it misses remote batchEmbed and structured input part text before all provider JSON paths are protected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize remote batch chunks before dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259-269`
  - body: The new sanitizer only runs in the inline retry wrappers, but remote batch indexing bypasses those wrappers and sends `missingChunks` directly to `providerRuntime.batchEmbed`. OpenAI/Voyage serialize `chunk.text` and Gemini serializes `chunk.embeddingInput`, so a lone surrogate can still break remote batch reindex; sanitize cloned chunks or their embedding inputs before dispatch.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-306`
  - body: The structured path sanitizes only `input.text`, but providers such as Gemini serialize `input.parts` when present. A lone surrogate in a text part can still reach the provider payload, so sanitize text parts as well and add structured-input coverage.
  - confidence: 0.9
