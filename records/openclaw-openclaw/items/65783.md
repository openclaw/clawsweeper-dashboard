## Summary

Keep open: the reported memory indexing failure is source-reproducible on current main, and the PR fixes the core chunking/top-level text paths, but it still leaves structured `parts[].text` unsanitized for providers that serialize parts.

## What This Changes

The PR changes memory chunking to avoid coarse UTF-16 surrogate splits, adds embed-boundary surrogate sanitization, and adds focused tests.

## Best Possible Solution

Land a current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured `parts[].text`, while leaving retry and checkpointing policy to the existing follow-up issues.

## Reproduction Assessment

Yes. Source inspection shows current main can split a mostly-ASCII line at `maxChars=40`, so `39` ASCII characters followed by an emoji puts the coarse boundary between the high and low surrogate before any guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and top-level embed-boundary sanitizer are the right shape, but the PR must also sanitize structured text parts before passing them to providers such as Gemini.

## Review Findings
