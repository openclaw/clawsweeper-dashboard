## Summary

Keep open: the reported memory indexing bug is still present on current main and this PR is aimed at the right code paths, but the proposed embed-boundary sanitizer still misses structured text parts that some providers serialize instead of the top-level text.

## What This Changes

The PR changes memory chunking to avoid splitting UTF-16 surrogate pairs, adds an embed-boundary lone-surrogate sanitizer, and adds focused regression tests.

## Best Possible Solution

Land a current-main repair that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider calls.

## Reproduction Assessment

Yes. Source inspection plus a read-only boundary probe show current main can split `39` ASCII characters plus an emoji between `\uD83C` and `\uDF38` before the fine-split guard runs.

## Solution Assessment

Mostly yes. The two-layer chunker plus embed-boundary sanitizer is the right narrow fix, but the PR is incomplete until structured `parts[].text` values are sanitized and covered by a regression test.
