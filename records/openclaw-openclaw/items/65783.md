## Summary

Keep open. Current main still has the reported memory chunker and embedding-sanitization bug, and this PR is a useful targeted fix, but it still leaves structured embedding `parts[].text` unsanitized on the Gemini/multimodal path.

## What This Changes

The PR updates memory chunking to avoid splitting UTF-16 surrogate pairs, adds lone-surrogate sanitization before embedding calls, and adds focused chunker and sanitizer tests.

## Best Possible Solution

Land a repaired version that fixes the coarse chunk boundary and sanitizes every text-bearing embedding input, including structured `parts[].text`, while leaving broader retry/checkpointing behavior to separate issues.

## Reproduction Assessment

Yes, at source level. Current main's coarse loop can slice between `\uD83C` and `\uDF38` for the reported 39-ASCII-plus-emoji boundary, and the embedding wrappers currently forward unsanitized text to providers.

## Solution Assessment

No, not as submitted. The chunker fix and embed-boundary sanitizer are the right direction, but the structured embedding path must also sanitize text parts before providers serialize them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
