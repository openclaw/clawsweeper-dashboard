## Summary

Keep open. The PR targets a real, source-reproducible memory indexing bug with sufficient live before/after proof, but the latest patch still leaves structured embedding text parts unsanitized before providers serialize them.

## What This Changes

This PR updates memory chunking to avoid coarse UTF-16 surrogate-pair splits, adds lone-surrogate sanitization at embedding boundaries, and adds focused regression tests.

## Best Possible Solution

Land this after sanitizing structured text parts as well as top-level text, with focused coverage; keep broader retry and checkpointing behavior in the existing follow-up items.

## Reproduction Assessment

Yes, at source level. Current main can cut `39` ASCII characters plus an emoji at a `maxChars=40` coarse boundary between `\uD83C` and `\uDF38`, and the PR body includes live before/after indexing output from a patched installed runtime.

## Solution Assessment

Mostly yes, but not complete. The chunker fix plus embed-boundary sanitizer is the right narrow shape, but the sanitizer also needs to rewrite text parts inside structured embedding inputs before this is merge-ready.

## Review Findings
