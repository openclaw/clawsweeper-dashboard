## Summary

Keep open. The PR addresses a real memory-indexing bug and has credible live before/after proof, but it is not mergeable as submitted because structured embedding inputs can still carry lone surrogates through `parts[].text` to providers.

## What This Changes

The PR updates memory chunking to avoid splitting UTF-16 surrogate pairs, adds an embedding-input sanitizer, and adds focused chunker and sanitizer tests.

## Best Possible Solution

Land a repaired patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider serialization, while leaving broader retry/checkpointing behavior to separate reports.

## Reproduction Assessment

Yes, at source level. Current main's coarse split can place `\uD83C` at the end of one chunk and `\uDF38` at the start of the next for the reported 39-ASCII-plus-emoji boundary, and the embedding wrappers currently forward unsanitized text.

## Solution Assessment

No, not as submitted. The chunker fix and embed-boundary sanitizer are the right direction, but the sanitizer must also rewrite structured text parts before providers serialize them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
