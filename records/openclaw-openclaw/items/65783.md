## Summary

Keep open. Current main still has the reported memory indexing bug, and this useful PR is not merge-ready because two embedding dispatch paths can still send unsanitized text to providers.

## What This Changes

The PR adds a coarse chunker surrogate-pair guard, an embed-boundary lone-surrogate sanitizer, and focused memory indexing tests.

## Best Possible Solution

Land a repaired current-main patch that guards coarse chunk boundaries and sanitizes all embedding text across inline, structured, and batchEmbed paths with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's coarse split can bisect 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate chunks before embedding.

## Solution Assessment

No, not yet. The chunker guard and top-level sanitizer are the right direction, but the PR must also sanitize structured text parts and provider runtime batchEmbed payloads before it fully fixes the embedding boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

  - confidence: 0.91

  - confidence: 0.9
