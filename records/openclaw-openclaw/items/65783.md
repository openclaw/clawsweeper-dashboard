## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize source-wide batch requests too:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:393-414`
  - body: The new sanitizer is only added to the inline `embedBatch*` helpers, but source-wide batch mode bypasses those helpers and provider runtimes such as OpenAI serialize `chunk.text` directly. A preexisting lone surrogate in memory content can still reach the batch upload path and stall indexing, so the sanitizer needs to run before `batchEmbed` as well.
  - confidence: 0.89
- **[P1] Sanitize structured text parts too:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:440-462`
  - body: `embedBatchInputsWithRetry` only rewrites `input.text`, but Gemini builds the actual request body from `input.parts[].text` whenever parts exist. A multimodal chunk with a lone surrogate in its text part can still reach the provider and fail the indexing run, so the sanitizer needs to cover each text part before `embedBatchInputs`.
  - confidence: 0.9
