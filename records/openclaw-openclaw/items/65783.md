## Summary

Keep open. The PR addresses a valid memory indexing bug and has useful live proof, but the proposed sanitizer still misses structured embedding text parts, so a lone surrogate can still reach providers such as Gemini when `input.parts` is present.

## What This Changes

The PR changes memory chunking to avoid coarse UTF-16 surrogate-pair splits and adds embed-boundary lone-surrogate sanitization plus focused tests.

## Best Possible Solution

Land a current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured `parts[].text`, while leaving retry and checkpointing policy to the existing follow-up issues.

## Reproduction Assessment

Yes, at source level. With `maxChars=40`, `39` ASCII characters followed by `\uD83C\uDF38` makes current main slice between the high and low surrogate before any guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured embedding inputs must also sanitize `parts[].text` before provider calls.
