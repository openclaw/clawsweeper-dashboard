## Summary

Keep open. The PR addresses a real memory indexing bug, but the current branch still leaves structured embedding text parts unsanitized, so Gemini-style structured providers can still receive lone surrogates when `parts` are present.

## What This Changes

The PR changes memory chunking to avoid splitting UTF-16 surrogate pairs, adds embedding-input surrogate sanitization, and adds focused regression/unit tests.

## Best Possible Solution

Land a repaired current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's coarse chunker slice can split `39` ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate halves before the existing fine guard runs.

## Solution Assessment

Mostly yes. The chunker boundary guard plus embed-boundary sanitizer is the narrow maintainable direction, but the PR should also sanitize structured text parts and be brought onto current main.
