## Summary

Keep this PR open. Current main still has the coarse memory chunk split that can bisect UTF-16 surrogate pairs and still sends raw embedding text to providers, so the reported bug remains real; the PR is the active implementation candidate, but it is stale against current main and needs one structured-input sanitizer gap fixed before merge.

## What This Changes

The PR updates memory chunking to preserve surrogate pairs, adds an unpaired-surrogate sanitizer at memory embedding entry points, and adds focused regression tests.

## Best Possible Solution

Rebase or replace the PR with the same narrow product shape: fix the active package chunker’s coarse boundary, sanitize every text-bearing embedding input field before provider calls, keep the SDK/source mirror aligned with current wrapper behavior, and land focused regression tests plus the changed gate.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with `tokens: 10`, `maxChars` is 40, so `"a".repeat(39) + emoji` puts the high surrogate at index 39 and current `line.slice(start, start + maxChars)` emits a chunk ending in that lone high surrogate before the fine-split guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and text-only sanitizer are the right narrow direction, but the branch needs a current-main rebase and should sanitize structured text parts as well as `input.text` before it is merge-ready.
