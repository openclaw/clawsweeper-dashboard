## Summary

Keep open. Current main still has the coarse UTF-16 split bug and still sends unsanitized embedding inputs, so the PR is useful source work, but it should not merge as submitted because structured text parts can bypass the sanitizer and the user-facing fix needs the required changelog entry.

## What This Changes

The PR updates memory chunking, memory embedding input handling, and regression tests to preserve UTF-16 surrogate pairs and replace unpaired surrogates before embedding provider calls.

## Best Possible Solution

Land a current-main patch that fixes the package chunker’s coarse surrogate boundary, sanitizes every text-bearing embedding input including structured `parts`, adds focused regression coverage for text-only and structured provider paths, and leaves the broader retry/checkpointing work to #56815 and #58255.

## Reproduction Assessment

Yes. The reproduction is deterministic at source level: with `tokens: 10`, `maxChars` is 40, and `"a".repeat(39) + "\uD83C\uDF38"` makes current main slice between the high and low surrogate before the fine-split guard runs.

## Solution Assessment

No, not as submitted. The direction is the narrow maintainable fix, but the branch must sanitize structured text parts as well as top-level text, add the required changelog entry, and be ported to current main.
