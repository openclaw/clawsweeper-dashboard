## Summary

Keep open. The memory indexing bug is valid and the chunker fix targets the right runtime path, but the PR's structured embedding sanitizer only rewrites `input.text` and can still pass an unsanitized text part to providers that serialize `EmbeddingInput.parts`.

## What This Changes

The PR changes memory chunking to avoid splitting UTF-16 surrogate pairs, sanitizes embedding inputs, and adds regression tests for both paths.

## Best Possible Solution

Land the PR after the sanitizer covers both `input.text` and structured text parts, preserving the package chunker fix and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows the coarse chunker loop can cut at `start + maxChars` without checking surrogate boundaries, and the PR's 39-ASCII-plus-emoji regression is a high-confidence focused reproduction path.

## Solution Assessment

Mostly yes. The chunker change is the narrow root-cause fix and it patches the runtime-exported package path, but the embed-boundary sanitizer should also rewrite text parts inside structured inputs before this is the best complete fix.
