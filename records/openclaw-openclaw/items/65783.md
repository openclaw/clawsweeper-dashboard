## Summary

Keep open. The PR targets a real current-main memory indexing bug, but the branch is incomplete because provider batch jobs and structured text parts can still carry lone surrogates to embedding providers.

## What This Changes

The branch adds a chunkMarkdown surrogate-boundary guard, an embedding-input sanitizer, and focused tests for memory chunking and sanitizer behavior.

## Best Possible Solution

Land a repaired current-main patch that guards coarse chunk boundaries and sanitizes every embedding text surface, including inline batches, structured text parts, and provider batch payloads, with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's coarse split can bisect 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate chunks before embedding.

## Solution Assessment

No, not yet. The chunker guard and top-level sanitizer are the right direction, but the PR must also sanitize structured text parts and provider runtime batchEmbed payloads before it fully fixes the embedding boundary.

  - confidence: 0.9
