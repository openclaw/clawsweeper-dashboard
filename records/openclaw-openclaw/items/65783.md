## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize batchEmbed chunks before dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259-260`
  - body: The new sanitizer only runs in the inline retry wrappers, but remote batch indexing sends missingChunks directly to providerRuntime.batchEmbed. OpenAI and Voyage batch adapters serialize chunk.text into batch payloads, so a lone surrogate can still reach the provider and fail the reindex.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
  - body: The structured path sanitizes only input.text, but providers such as Gemini serialize input.parts when present. A lone surrogate in a text part can still be sent to the provider, so sanitize text parts as well and add structured-input coverage.
  - confidence: 0.9
