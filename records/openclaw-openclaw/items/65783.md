## Summary

Keep open. The bug is valid and the PR is pointed at the right surface, but the patch is not merge-ready because structured embedding text parts still bypass the new sanitizer and the branch needs a current-main repair pass.

## What This Changes

The PR updates memory chunking, embedding input handling, and tests to keep UTF-16 surrogate pairs intact and replace unpaired surrogates before embedding calls.

## Best Possible Solution

Land a current-main patch that fixes the coarse chunk boundary, sanitizes every text-bearing embedding field including structured `parts[].text`, and leaves retry/checkpointing behavior to the existing follow-up reports.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. With `maxChars=40`, `39` ASCII characters followed by the flower emoji puts the current coarse slice boundary between `\uD83C` and `\uDF38` before any surrogate guard runs.

## Solution Assessment

No, not as submitted. The chunker fix direction is right, but the sanitizer only covers top-level structured input text and should also rewrite text-bearing `parts[]` entries before provider calls.

## Review Findings

Overall correctness: patch is incorrect
