## Summary

Keep open: the reported memory indexing failure is source-reproducible on current main, and this PR contains useful central fixes, but the proposed sanitizer still misses remote batch and structured text-part provider paths that can carry the same invalid surrogate payload.

## What This Changes

The PR updates memory chunking and embedding code/tests to avoid splitting UTF-16 surrogate pairs and replace unpaired surrogates before embedding.

## Best Possible Solution

Land a repaired current-main patch that guards coarse chunk boundaries and sanitizes every provider-bound embedding text path, with focused regression coverage for inline, remote batch, and structured inputs.

## Reproduction Assessment

Yes, at source level. Current main's coarse split can bisect 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate chunks before embedding.

## Solution Assessment

No, not yet. The chunker guard and top-level sanitizer are the right direction, but the PR must also sanitize providerRuntime.batchEmbed chunks and structured text parts before it fully fixes the embedding boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize batchEmbed chunks before dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259-260`
  - body: The new sanitizer only runs in the inline retry wrappers, but remote batch indexing sends missingChunks directly to providerRuntime.batchEmbed. Batch adapters serialize chunk.text, so a lone surrogate can still reach the provider and fail the reindex.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
  - body: This maps `stripUnpairedSurrogates` over only `input.text`; if `input.parts` contains a text part, providers such as Gemini serialize `part.text` and bypass the sanitizer. Sanitize text parts as well so structured inputs cannot keep the same failure mode.
  - confidence: 0.9
