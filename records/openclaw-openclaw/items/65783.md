## Summary

Keep open. The PR targets a real memory indexing bug and includes useful live proof, but the current branch is stale and the sanitizer coverage still misses provider-bound remote batch chunks and structured text parts.

## What This Changes

The PR changes memory chunking to avoid UTF-16 surrogate-pair splits, adds an unpaired-surrogate sanitizer near embedding calls, and adds focused memory tests.

## Best Possible Solution

Land a refreshed patch that fixes the coarse chunk boundary and sanitizes every provider-bound text field before inline, structured, and remote batch embedding calls.

## Reproduction Assessment

Yes, at source level. Current main's coarse split at maxChars=40 with 39 ASCII characters plus a flower emoji yields a chunk ending with high surrogate d83c and the next chunk starting with low surrogate df38.

## Solution Assessment

No, not yet. The chunker guard and inline sanitizer are the right direction, but the PR must be refreshed onto the current memory-host layout and must cover remote batch chunks plus structured text parts before it fully fixes the embedding boundary.
