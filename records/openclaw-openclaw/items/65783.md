## Summary

Keep open: current main still exposes the reported memory indexing failure and this PR is a plausible narrow fix, but the submitted sanitizer still lets structured text parts bypass sanitization before provider calls.

## What This Changes

The PR updates memory chunking and embedding retry paths/tests to preserve UTF-16 surrogate pairs and replace unpaired surrogates before embeddings.

## Best Possible Solution

Land a current-main patch that fixes the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured `parts[].text`, while leaving retry and checkpointing policy to the existing follow-up issues.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with `tokens=10`, `maxChars=40`, and `39` ASCII characters before `🌸`, current main slices between `\uD83C` and `\uDF38`.

## Solution Assessment

No, not as submitted. The fix direction is right, but the PR only sanitizes top-level structured input text and still lets `parts[].text` reach providers unchanged.

## Review Findings

Overall confidence: 0.86
