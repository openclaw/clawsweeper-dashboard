## Summary

Keep open. Current main still has the memory-indexing surrogate bug and this PR is the right narrow direction, but the proposed sanitizer misses structured text parts and provider runtime batch jobs, so it needs a focused repair before merge.

## What This Changes

The PR changes memory chunking, embedding retry wrappers, and focused tests to preserve surrogate pairs and replace unpaired surrogates before embedding.

## Best Possible Solution

Land a repaired current-main patch that guards coarse chunk boundaries and sanitizes all text sent to embedding providers across inline, structured, and batchEmbed paths, with regression coverage.

## Reproduction Assessment

Yes, at source level. A read-only Node probe matching current main's coarse loop with maxChars=40 and 39 ASCII characters plus a flower emoji produced a chunk ending in high surrogate d83c and the next starting in low surrogate df38.

## Solution Assessment

No, not yet. The chunker guard and top-level sanitizer are the right direction, but the PR must also sanitize structured text parts and provider runtime batchEmbed payloads before it fully fixes the embedding boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize batchEmbed chunks before dispatch:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:259-260`
  - body: The new sanitizer only runs in the inline retry wrappers, but when remote batch indexing is enabled `embedChunksWithBatch` sends `missingChunks` directly to `providerRuntime.batchEmbed`. OpenAI, Gemini, and Voyage batch adapters serialize `chunk.text` or `chunk.embeddingInput`, so a lone surrogate can still reach batch JSONL and fail the reindex; sanitize the chunks/embedding inputs before dispatching batchEmbed or inside the batch adapters.
  - confidence: 0.91
- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-306`
  - body: The structured path sanitizes only `input.text`, but providers such as Gemini serialize `input.parts` when present. A lone surrogate in a text part can still be sent to the provider and reproduce the embedding failure, so sanitize text parts as well and add structured-input coverage.
  - confidence: 0.9
