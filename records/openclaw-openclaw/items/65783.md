## Summary

This PR should stay open: it fixes a real current-main memory indexing bug, but the branch still misses provider-visible structured text parts, so a lone surrogate can reach embedding providers through `parts[].text`.

## What This Changes

The PR updates memory chunking to avoid splitting UTF-16 surrogate pairs, adds an embedding-input lone-surrogate sanitizer, and adds focused chunker and sanitizer tests.

## Best Possible Solution

Land the chunker boundary fix plus sanitizer coverage for every text-bearing embedding field, including structured `parts[].text`, while leaving broader retry and checkpointing behavior to the linked follow-ups.

## Reproduction Assessment

Yes. Source inspection plus a focused current-main Node probe show the reported coarse split can bisect the flower emoji surrogate pair, and the paired report provides live before/after indexing evidence.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but the patch must also sanitize structured text parts before providers such as Gemini serialize them.
