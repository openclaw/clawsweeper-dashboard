## Summary

Keep open. The PR addresses a real memory indexing bug and includes sufficient live proof, but the proposed structured-input sanitizer still leaves `parts[].text` unsanitized on the provider-visible Gemini path.

## What This Changes

The PR updates memory chunking to avoid UTF-16 surrogate splits, sanitizes embedding text before provider calls, and adds focused regression tests.

## Best Possible Solution

Land the chunker boundary fix and sanitize every text-bearing embedding field, including structured `parts[].text`, with focused regression coverage and no retry/checkpointing expansion in this PR.

## Reproduction Assessment

Yes. Source inspection shows the current coarse chunk boundary can split an emoji surrogate pair with `maxChars=40` and `39` ASCII characters before the existing guard runs, and the paired report provides live before/after indexing evidence.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured text parts must also be sanitized before providers such as Gemini serialize them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
