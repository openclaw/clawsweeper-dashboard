## Summary

Keep open: the bug is valid and the patch is useful, but it still leaves provider-bound text unsanitized in remote batch and structured part paths, so it should be repaired/refreshed before merge.

## What This Changes

The PR updates memory chunking to avoid coarse UTF-16 surrogate splits, adds a lone-surrogate sanitizer around embedding calls, and adds focused regression tests.

## Best Possible Solution

Land a refreshed patch that fixes the coarse chunk boundary and applies one sanitizer to every provider-bound text field across inline, structured, and remote batch embedding paths.

## Reproduction Assessment

Yes, at source level. Current main's fixed-step coarse loop can split 39 ASCII characters plus a flower emoji at maxChars=40 into chunks ending and starting with lone surrogate halves.

## Solution Assessment

No, not as currently patched. The chunker guard is the right direction, but the sanitizer must also cover remote batch chunks and structured text parts before the fix is complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

  - confidence: 0.9
