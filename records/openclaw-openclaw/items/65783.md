## Summary

Keep open. The underlying memory-indexing bug is still present on current main, and this PR is a useful fix path, but the patch is incomplete for structured embedding inputs and the head is currently conflicting.

## What This Changes

The PR updates memory chunking to avoid coarse UTF-16 surrogate-pair splits, adds embed-boundary surrogate sanitization, and adds focused chunker/sanitizer tests.

## Best Possible Solution

Land a repaired current-main patch that guards the coarse chunk boundary and sanitizes top-level plus structured embedding text before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's coarse loop can split 39 ASCII characters plus a flower emoji at maxChars=40 into lone high and low surrogate halves before embedding.

## Solution Assessment

Mostly yes. The chunker guard and embed-boundary sanitizer are the right narrow fix, but the PR must also sanitize structured text parts before embedBatchInputs and resolve the current merge conflict.
