## Summary

Keep open. The reported bug is valid and the PR has the right main direction plus live proof, but the submitted sanitizer still misses structured text parts, leaving a provider path that can send lone surrogates unchanged.

## What This Changes

The PR changes memory chunking and memory embedding retry paths to preserve UTF-16 surrogate pairs, sanitize lone surrogates before provider calls, and add focused regression tests.

## Best Possible Solution

Land a current-main repair that fixes the coarse chunk boundary and sanitizes every text-bearing embedding field before provider calls, leaving broader retry and checkpointing policy to existing follow-up issues.

## Reproduction Assessment

Yes. Source inspection plus a focused Node probe show current main can split an emoji between `\uD83C` and `\uDF38` at the coarse chunk boundary.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured `parts[].text` must also be sanitized before providers such as Gemini serialize those fields.
