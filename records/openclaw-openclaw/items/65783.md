## Summary

Keep open. This is a plausible P1 memory-indexing fix with live after-fix proof, but the branch is currently conflicting and the sanitizer misses structured text parts used by providers such as Gemini.

## What This Changes

The branch changes memory chunking, embedding sanitization, and unit coverage so lone UTF-16 surrogates do not reach memory embedding providers.

## Best Possible Solution

Rebase onto current main, preserve the timeout/signal embedding wrapper, sanitize top-level text and every structured text part before provider calls, and keep focused regression tests for the chunk boundary and embed sanitizer.
