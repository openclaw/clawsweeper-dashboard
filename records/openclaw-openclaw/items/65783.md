## Summary

Keep open. The PR targets a real memory indexing bug and fixes the coarse chunk boundary, but it still leaves structured embedding `parts[].text` unsanitized on providers that serialize parts, so it needs a narrow repair before merge.

## What This Changes

The branch updates memory chunking to avoid splitting UTF-16 surrogate pairs, adds an embed-input lone-surrogate sanitizer, and adds focused regression tests.

## Best Possible Solution

Land the fix after the embed-boundary sanitizer rewrites both top-level text and structured text parts, keeping the chunker regression and leaving broader retry/checkpointing issues to their existing threads.

## Reproduction Assessment

Yes. Source inspection plus a read-only boundary probe show current main can split `39` ASCII characters plus an emoji between `\uD83C` and `\uDF38` before the fine-split guard runs.

## Solution Assessment

Mostly yes. The PR uses the right two-layer fix, but it is not complete until structured `parts[].text` values are sanitized and covered by a focused regression test.
