## Summary

Keep open: this PR targets a real current-main memory indexing bug, but the submitted sanitizer still misses provider-visible structured text parts, so it should not merge as-is.

## What This Changes

The PR updates memory chunking to avoid splitting UTF-16 surrogate pairs, adds an embedding-input lone-surrogate sanitizer, and adds focused chunker and sanitizer tests.

## Best Possible Solution

Land a current-main repair that preserves surrogate pairs at coarse chunk boundaries and sanitizes every text-bearing embedding field, including structured parts[].text, while leaving broader retry and checkpointing work to the linked follow-ups.

## Reproduction Assessment

Yes. Source inspection plus a focused Node probe show current main can split the flower emoji between \uD83C and \uDF38 at the coarse boundary, and the paired issue includes live before/after indexing evidence.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured text parts must also be sanitized before providers such as Gemini serialize them.
