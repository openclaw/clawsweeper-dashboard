## Summary

Keep open. The PR targets a real memory-indexing bug on current main and includes useful live before/after proof, but the submitted sanitizer still misses structured text parts that Gemini serializes, so it needs a narrow repair before merge.

## What This Changes

The PR changes memory chunking to avoid UTF-16 surrogate-pair splits, adds an embedding-input sanitizer, and adds focused chunker and sanitizer tests.

## Best Possible Solution

Land a repaired patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding input, including structured parts, before provider serialization; keep broader retry and checkpointing work in the linked follow-ups.

## Reproduction Assessment

Yes, at source level. Current main slices `39` ASCII characters plus the flower emoji at `maxChars=40` into separate high and low surrogate halves before any guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but structured `EmbeddingInput.parts` text must also be sanitized before providers serialize requests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
