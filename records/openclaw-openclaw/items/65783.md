## Summary

Keep open. The PR targets a valid current-main memory indexing bug and includes useful live proof, but it is not ready as submitted because structured embedding `parts[].text` can still bypass the new sanitizer and the branch is currently conflicting.

## What This Changes

The PR updates memory chunking, embedding input sanitization, and tests to keep UTF-16 surrogate pairs intact and replace unpaired surrogates before embedding calls.

## Best Possible Solution

Land a current-main patch that fixes the coarse chunk boundary, sanitizes every text-bearing embedding field including structured `parts[].text`, and keeps retry/checkpointing behavior in separate follow-up work.

## Reproduction Assessment

Yes, at source level. With `tokens=10`, `maxChars` is 40, and `39` ASCII characters followed by the flower emoji puts the current coarse slice boundary between `\uD83C` and `\uDF38`.

## Solution Assessment

No, not as submitted. The fix direction is right, but structured text parts still bypass the sanitizer and the branch needs a current-main port before it is the narrow maintainable solution.
