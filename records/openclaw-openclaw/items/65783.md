## Summary

Keep open. The PR targets a real current-main memory indexing bug and its chunker fix is directionally sound, but the submitted sanitizer still misses structured embedding text parts that providers can serialize.

## What This Changes

The PR changes memory chunking to avoid surrogate-pair coarse splits, sanitizes embedding text inputs, and adds focused memory-host and memory-core tests.

## Best Possible Solution

Land a current-main repair that fixes the coarse chunk boundary and sanitizes every text-bearing embedding field before provider calls, leaving broader retry and checkpointing work to existing follow-up issues.

## Reproduction Assessment

Yes. Current-main source plus a focused Node probe reproduce the coarse split between `\uD83C` and `\uDF38` when `maxChars=40` and the emoji begins at index 39.

## Solution Assessment

No, not as submitted. The chunker fix and top-level embed sanitizer are the right direction, but the final patch must also sanitize structured `parts[].text` before providers such as Gemini serialize those fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
