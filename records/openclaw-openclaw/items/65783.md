## Summary

Keep open. The PR targets a real current-main memory indexing bug, but its structured embedding path still sanitizes only `input.text` while providers can serialize `input.parts`, and the PR is currently merge-conflicting.

## What This Changes

The PR changes memory chunking to avoid UTF-16 surrogate splits, adds embedding text sanitization, and adds focused regression tests.

## Best Possible Solution

Land a repaired current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes both top-level and structured embedding text before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: with `maxChars=40`, `39` ASCII characters followed by the flower emoji makes the coarse slice boundary land between `\uD83C` and `\uDF38` before any guard runs.

## Solution Assessment

Mostly yes. The chunker fix plus embed-boundary sanitizer is the narrow maintainable direction, but the PR should also sanitize text parts inside structured inputs and be brought onto current main.
