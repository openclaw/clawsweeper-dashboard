## Summary

Keep open. The PR targets a real current-main memory indexing bug and includes useful live proof, but it still misses structured `parts[].text` sanitization that Gemini serializes, so it should not merge as submitted.

## What This Changes

The PR changes memory chunking to avoid surrogate-pair splits, adds a lone-surrogate embedding sanitizer with tests, and applies it to memory embedding retry paths.

## Best Possible Solution

Land a current-main patch that preserves surrogate pairs at the coarse chunk boundary and sanitizes every text-bearing embedding field, including structured parts, before provider calls.

## Reproduction Assessment

Yes, at source level. Current main plus a read-only Node probe show that `maxChars=40` with `39` ASCII characters followed by the flower emoji splits between `\uD83C` and `\uDF38` before any guard runs.

## Solution Assessment

No, not as submitted. The chunker fix and top-level sanitizer are the right direction, but the PR must also sanitize structured `parts[].text` before providers such as Gemini serialize those fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Sanitize structured text parts before embedding:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:293-296`
