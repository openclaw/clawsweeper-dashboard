## Summary

Keep open. Current main still uses the destructive whole-target sync this PR tries to replace, but the proposed incremental sync can leave stale or no-longer-authorized sandbox skill content behind, so it is not merge-ready as written.

## What This Changes

The PR changes `syncSkillsToWorkspace` to stop deleting the whole target skills directory, build a per-skill manifest, skip copies by newest mtime, and add regression tests for non-destructive syncing.

## Best Possible Solution

Keep the non-destructive intent, but make the sync own and prune only OpenClaw-created target dirs, preserve confirmed user-created dirs, handle source deletions by tree replacement or a content manifest, and fold in failed-copy cleanup from #72254.

## Reproduction Assessment

Yes. Sync an allowed skill once, then remove it from the effective allowlist or delete the newest file from its source tree and sync again; the proposed branch can leave the old skill directory or deleted file in the target.

## Solution Assessment

No. Avoiding the whole-target wipe is directionally right, but preserving every non-manifest directory and relying on newest mtime is not the narrowest maintainable sync contract for sandbox-visible skills.

  - body: Replacing the target wipe with `mkdir` leaves directories outside the current manifest untouched.
