## Summary

Keep open: current main still has the destructive sandbox skill sync, but this branch is not mergeable as-is because it can retain stale/no-longer-eligible skill content and still lacks required real behavior proof.

## What This Changes

This PR changes `syncSkillsToWorkspace` from whole-target removal to incremental per-skill copying with newest-mtime skips, debug counts, and non-destructive sync regression tests.

## Best Possible Solution

Implement an owned-manifest sync that preserves unknown user-created target content, prunes OpenClaw-owned stale skills, detects file-set changes, and preserves the failed-copy cleanup guarantee from https://github.com/openclaw/openclaw/pull/72254.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main removes the target skills directory before copying, and this PR branch can leave stale directories or deleted source files in the target after eligibility or tree changes.

## Solution Assessment

No. Avoiding the whole-target wipe is the right direction, but preserving every non-manifest directory and using newest mtime as the freshness contract is not the narrowest maintainable fix.
