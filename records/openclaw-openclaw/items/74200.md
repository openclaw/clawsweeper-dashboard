- **[P1] Prune previously synced skills that leave the manifest:** `src/agents/skills/workspace.ts:1015`
  - body: The new loop only visits the current source manifest, so a skill that was previously synced but later removed, disabled, or filtered out is never removed from the target workspace.   - confidence: 0.84
- **[P2] Do not use newest mtime as the copy oracle:** `src/agents/skills/workspace.ts:1018-1020`
  - body: If a source skill deletes its newest file, the destination mtime remains newer than the source and this branch skips the copy. That leaves deleted instructions or helper files in the sandbox copy until some unrelated source file gets a newer timestamp.
  - confidence: 0.91
