## Summary

Keep this PR open, but do not merge it as-is: current main still has the destructive whole-target wipe, while the proposed incremental sync can preserve no-longer-eligible skills and deleted source files in sandbox targets.

## What This Changes

The PR changes `syncSkillsToWorkspace` from full target-directory removal to per-skill incremental copying with mtime-based skips and adds sync regression tests.

## Best Possible Solution

Implement an owned-manifest sync that preserves unknown user-created target content while pruning OpenClaw-owned stale skills, detecting full tree changes, and coordinating the failed-copy cleanup from https://github.com/openclaw/openclaw/pull/72254.

## Reproduction Assessment

Yes, by source inspection of the proposed branch. Sync a skill once, then remove it from the effective allowlist or delete the newest source file; the next sync can leave the old directory or deleted file in the target.

## Solution Assessment

No. Avoiding the whole-target wipe is the right direction, but preserving every non-manifest directory and using newest mtime as the freshness contract is not the narrowest maintainable fix.
