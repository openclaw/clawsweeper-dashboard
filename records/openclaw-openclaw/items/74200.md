## Summary

Keep open. Current main still has the destructive sandbox skill sync path, so the PR is aimed at a real bug, but the proposed implementation can leave stale owned skills and deleted source files available in sandbox targets; related #72254 is still open and should be coordinated before merge.

## What This Changes

The PR replaces wholesale sandbox skill directory deletion with per-skill incremental copy/mtime skips and adds regression tests for preserved, skipped, and updated synced skills.

## Best Possible Solution

Keep the non-destructive intent, but implement an OpenClaw-owned manifest/full-tree sync that preserves unknown user-created target content while pruning stale owned sync outputs and coordinating failure cleanup with #72254.

## Reproduction Assessment

Yes, by source inspection of the proposed patch. Sync a skill once, then remove it from the effective allowlist or delete the newest source file; the PR can leave the prior directory or deleted file in the sandbox target.

## Solution Assessment

No. Avoiding the whole-target wipe is the right direction, but preserving every non-manifest directory and using newest mtime as the freshness contract is not the narrowest maintainable fix.
