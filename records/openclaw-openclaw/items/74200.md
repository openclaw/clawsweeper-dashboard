## What This Changes

The PR changes skill workspace syncing from full target reset to incremental per-skill copying with a newest-mtime skip path and regression tests for preserving target-only skill directories.

## Best Possible Solution

Rework the current src/skills/loading/workspace.ts path so sync avoids deleting user-owned skill data while tracking and removing stale OpenClaw-synced directories, handles deletions without mtime-only shortcuts, cleans partial copy failures, and includes real upgrade/user-data proof.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 still remove target skill children before copying, and the PR-introduced stale-skill paths are reproducible from the diff when a skill is removed or when a source file deletion lowers the newest source mtime.

## Solution Assessment

No. The non-destructive direction is the right bug boundary, but this implementation is not the best fix because it conflates user-created target dirs with stale synced dirs and uses an mtime shortcut that misses deletions.

## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Track stale synced dirs before preserving target children:** `src/agents/skills/workspace.ts:968-977`
  - body: This keeps every target directory that is not in the current source manifest. Because the target workspace later loads skills from `<target>/skills`, a skill that was removed from the source, disabled by allowlist, or made ineligible can remain visible and readable in later sandbox runs. Keep the root non-destructive, but distinguish user-owned target dirs from old OpenClaw-synced dirs and clean the stale synced ones.
  - confidence: 0.87
- **[P2] Replace mtime-only skipping with deletion-aware sync:** `src/agents/skills/workspace.ts:1017-1019`
  - body: The `destMtime >= srcMtime` shortcut misses deletions: if the newest source file is deleted, the destination still has that file's newer mtime and the copy is skipped, leaving deleted files in the sandbox indefinitely. Use a manifest/hash comparison or replace the manifest-owned destination when the source entry is present.
  - confidence: 0.9
