## Summary

Keep open. The PR addresses a real destructive sync path on current main, but the proposed incremental sync is not merge-ready because it can retain no-longer-eligible sandbox skills and stale deleted files.

## What This Changes

The PR changes `syncSkillsToWorkspace` to stop deleting the entire target `skills` directory, adds per-skill mtime-based copying with debug counts, and adds regression tests for preserved and updated skills.

## Best Possible Solution

Keep the data-loss fix intent, but implement an owned-manifest sync that preserves true user-created target content while pruning stale OpenClaw-owned skills and using full-tree replacement or comparison semantics.

## Reproduction Assessment

Yes. Sync a skill once, then remove it from the effective allowlist or delete a newer source file and sync again; the proposed branch can leave the old skill directory or deleted file in the target.

## Solution Assessment

No. Avoiding the whole-directory wipe is directionally right, but preserving every non-manifest directory and relying on newest mtime is not the safest maintainable sync contract for sandbox skills.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prune stale owned skills when the manifest changes:** `src/agents/skills/workspace.ts:968-976`
  - body: Replacing the target wipe with `mkdir` leaves directories outside the current manifest untouched. That means a sandbox can keep a previously synced skill after allowlists, config gates, plugin enablement, or remote eligibility remove it, so the sync needs an owned-manifest cleanup path instead of preserving all prior target directories.
  - confidence: 0.88
- **[P2] Avoid newest-mtime skips for tree deletions:** `src/agents/skills/workspace.ts:1017-1019`
  - body: The `destMtime >= srcMtime` shortcut misses source deletions: if a newer source file is deleted, the destination still has that file and a newer mtime, so the next sync skips and the stale file remains indefinitely. Use a content/tree manifest or recopy owned skill directories when tree shape changes.
  - confidence: 0.92
