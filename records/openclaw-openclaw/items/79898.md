## Summary

Current main still has the reported git-ref handling bug, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author PR with closing syntax. This issue should stay open until that PR lands or is closed.

## What This Changes

The issue asks for explicit `git:<repo>@<ref>` plugin refs to be treated only as refs, rejecting option-like values instead of letting Git parse them as checkout flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent narrow fix that separates Git options from the requested ref and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current source still parses `@--ignore-skip-worktree-bits` as a ref and passes it to `git checkout --detach <ref>`, and local Git help confirms the sample value is a `checkout` option.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable solution: pass the requested value after a Git option separator using a ref-checkout command and lock the argv boundary with a regression test.
