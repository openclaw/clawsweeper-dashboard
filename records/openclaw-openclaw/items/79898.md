## Summary

Keep open: current main still contains the git-ref checkout bug, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author fix PR with closing syntax for this issue.

## What This Changes

The issue reports that explicit `git:` plugin refs beginning with option-like strings can be parsed by `git checkout` as flags, allowing install to continue from the default branch.

## Best Possible Solution

Git plugin install should pass explicit refs through a ref-only Git command boundary and keep regression coverage for option-like selectors before the linked issue is closed by a merged fix.

## Reproduction Assessment

Yes, at source/dependency level: current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`, and a scratch Git 2.53.0 repo reproduced checkout accepting that option-like value. I did not run a full OpenClaw CLI install because `pnpm` is unavailable in this environment.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: use `git switch --detach -- <ref>` at the single checkout argv boundary and cover option-like refs in `src/plugins/git-install.test.ts`. This issue should stay open while the linked PR is still open.
