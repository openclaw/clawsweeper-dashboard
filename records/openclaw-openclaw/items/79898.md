## Summary

Current main still contains the git-ref handling bug, and the same author has an open closing PR at https://github.com/openclaw/openclaw/pull/79901, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue asks for git plugin installs to treat explicit `@<ref>` values only as refs so option-like values fail instead of being parsed as `git checkout` flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901, or an equivalent focused fix, then let this issue close as implemented.

## Reproduction Assessment

Yes. Current main parses the option-like selector as `parsed.ref` and passes it to `git checkout --detach <ref>`, while Git 2.53.0 documents `--ignore-skip-worktree-bits` as a checkout option.

## Solution Assessment

Yes. The linked PR's `git switch --detach -- <ref>` direction is the narrow maintainable fix because it separates Git options from the requested ref and keeps the documented git install contract intact.
