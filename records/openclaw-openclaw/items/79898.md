## Summary

Current main still has the reported git-ref handling bug, and an open same-author fix PR already references this issue as a closing issue, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for git plugin install refs to be treated only as refs so option-like values fail instead of being parsed as `git checkout` flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent narrow fix that passes the requested ref after a Git option separator and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction path: `git:<repo>@--ignore-skip-worktree-bits` is parsed as `parsed.ref` and then passed to `git checkout --detach <ref>` in Git's option position.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: use a Git command shape that separates command options from the requested ref and lock the argv boundary with a regression test.
