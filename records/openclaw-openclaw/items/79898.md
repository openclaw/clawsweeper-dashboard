## Summary

Keep open: current main still passes explicit git plugin refs to `git checkout --detach <ref>`, matching the reported option-parsing bug, and an open linked PR is already the active fix candidate.

## What This Changes

Ensure `git:` plugin install refs that begin with `-` are treated as refs or rejected instead of being parsed as checkout options.

## Best Possible Solution

Land the narrow git checkout argv fix with regression coverage, then close this issue once current main rejects option-like refs before plugin install metadata is persisted.

## Reproduction Assessment

Yes. Source inspection on current main shows the user-supplied ref is passed directly after `git checkout --detach`, and the issue includes concrete Git output showing `--ignore-skip-worktree-bits` can succeed without validating a ref.

## Solution Assessment

Yes. The narrow maintainable fix is to separate Git options from the requested ref in the checkout command and add a regression test for option-like refs; the linked open PR follows that direction.
