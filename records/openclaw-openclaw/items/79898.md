## Summary

Current main still has the reported git-ref handling bug, and an open same-author PR already closes this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

Ensure explicit refs in `git:` plugin installs are treated only as refs, so option-like values fail instead of being parsed as `git checkout` flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent focused fix that separates Git options from the requested ref and keeps regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and passes it to `git checkout --detach <ref>`; the issue body and prior review also show the Git command behavior that makes this succeed incorrectly.

## Solution Assessment

Yes. The linked PR direction, `git switch --detach -- <ref>` plus focused regression coverage, is the narrow maintainable fix; the issue should stay open because that PR is still open.
