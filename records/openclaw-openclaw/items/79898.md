## Summary

Keep open: current main still has the reported git-ref handling bug, and https://github.com/openclaw/openclaw/pull/79901 is still an open same-author fix PR with closing syntax for this issue.

## What This Changes

The issue reports that `git:` plugin installs can treat an explicit option-like ref as a Git checkout flag and continue from the default branch while recording the supplied ref.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent fix that passes the requested ref after a Git option separator and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes, there is a high-confidence source-level path: current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and passes it to `git checkout --detach <ref>`. A scratch Git 2.53.0 check confirmed that checkout form exits successfully while `git switch --detach -- <ref>` rejects the invalid ref.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: change the single checkout argv boundary to treat the supplied value only as a ref and add unit coverage. The issue should remain open until that PR or an equivalent fix is merged.
