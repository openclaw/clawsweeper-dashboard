## Summary

Current main still has the reported git-ref handling bug, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author fix PR with closing syntax, so this issue should stay open until that PR lands or is closed.

## What This Changes

Git plugin installs should treat requested git refs only as refs and reject option-like values instead of allowing `git checkout` to parse them as flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent fix that passes the requested ref after a Git option separator and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes, we have a high-confidence source-level reproduction path: current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`. An isolated Git 2.53.0 check also confirmed that this checkout form exits 0 while the proposed separated ref form rejects the invalid ref.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable solution: use a Git command shape that separates options from the requested ref and lock it with a unit regression test. The issue itself should remain open because the fixing PR is still open.
