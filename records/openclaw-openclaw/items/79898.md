## Summary

Current main still has the reported git-ref handling bug, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author fix PR with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that explicit `git:` plugin refs be treated only as refs so option-like values fail checkout instead of being parsed as Git checkout flags.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent focused change that separates Git options from the requested ref and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`; the issue and linked PR include Git/CLI output showing the unsafe checkout behavior and the proposed rejected-ref behavior.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the single Git argv boundary to `git switch --detach -- <ref>` and adds a regression test; the issue should stay open while that PR is still open.
