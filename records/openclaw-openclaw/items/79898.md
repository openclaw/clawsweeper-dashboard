## Summary

Keep open: current main still passes explicit git plugin refs to `git checkout --detach <ref>`, and an open same-author PR already references this issue with closing syntax.

## What This Changes

Explicit git plugin refs that look like Git options should be treated only as refs and fail before install if invalid.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79901 or an equivalent focused fix that checks out refs with an explicit ref boundary and keeps regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and passes it directly to `git checkout --detach <ref>`, matching the issue's Git reproduction and the documented install path.

## Solution Assessment

Yes. The linked PR direction, `git switch --detach -- <ref>` plus a unit regression test, is the narrow maintainable fix for this checkout argv boundary.
