## Summary

Keep open: current main still has the unsafe git checkout argv, the latest release carries the same behavior, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author fix PR with closing syntax.

## What This Changes

The issue reports that explicit `git:<repo>@<ref>` plugin installs can pass leading-dash refs to `git checkout --detach <ref>`, letting Git treat the requested ref as a flag and continue on the wrong commit.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901, or an equivalent single fix, so explicit git plugin refs are passed after a Git option separator and covered by regression tests.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`; the issue and linked PR include terminal evidence that Git accepts the checkout form and rejects `git switch --detach -- <ref>`.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: change only the checkout argv boundary to `git switch --detach -- <ref>` and add targeted regression coverage; this issue should stay open while that PR is open.
