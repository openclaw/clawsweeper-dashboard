## Summary

Current main still has the reported git ref handling bug, and https://github.com/openclaw/openclaw/pull/79901 is an open same-author fix PR with closing syntax, so this issue should stay open until that PR lands or is closed.

## What This Changes

Ensure explicit `git:` plugin refs are treated only as refs so option-like selectors cannot be parsed as checkout flags during plugin install.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent focused fix that separates Git options from the requested ref and keeps regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`, and a scratch Git 2.53.0 repo reproduced checkout exiting successfully while `git switch --detach -- <ref>` rejects the invalid ref.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes only the Git checkout argv boundary and adds regression coverage; this issue should remain open while that PR is still open.
