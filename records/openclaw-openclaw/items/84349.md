## Summary

Keep open: current main still does not give configured custom `anthropic-messages` providers the Claude provider thinking profile, and the Claude 4.6 `max` effort question remains a real contract/design gap rather than cleanup-close material.

## What This Changes

Custom `anthropic-messages` providers should expose Claude adaptive/max thinking profiles, including `max` for Claude Opus/Sonnet 4.6 where supported.

## Best Possible Solution

Add provider-boundary-safe Claude thinking profile resolution for configured Anthropic-compatible providers, align 4.6 `max` support after maintainer contract approval, and cover configured proxy cases with tests/docs.

## Reproduction Assessment

Yes. Source inspection shows a custom provider id with no `resolveThinkingProfile` hook reaches the base thinking profile, so `adaptive` and `max` are not valid profile levels even when the model uses the Anthropic Messages transport.

## Solution Assessment

No, not as suggested. A direct Claude fallback inside generic thinking resolution is likely the wrong ownership boundary; maintainers should choose whether to infer from existing API/model metadata or add a typed provider/profile capability.
