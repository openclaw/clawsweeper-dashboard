Full review comments:

- **[P2] Move legacy reads out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the `try` whose `catch` falls through to the docx path. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real legacy-doc error is swallowed and retried as docx with the legacy token, masking the actionable failure.
  - confidence: 0.91
- **[P2] Keep raw legacy reads independent of rich content:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency before any plaintext can be returned. A rich-content API failure would discard a successful raw-text read, which is the central value of this feature.
  - confidence: 0.83
- **[P3] Update legacy token and scope guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This new branch accepts `/docs/:token`, but the schema and bundled Feishu doc skill still tell callers to use `/docx/` tokens and list only docx permissions. Agents and operators will not know how to invoke or permission the new path.
  - confidence: 0.78
