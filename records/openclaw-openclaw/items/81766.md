## Summary

Keep open: the feature belongs in the Feishu plugin, but this PR still lacks real behavior proof and has blocking source-visible bugs in the legacy read fallback and raw-content path.

## What This Changes

The PR adds legacy Feishu `/docs/:token` read support via doc v2 metadata detection, REST helpers/types, and `readDoc` routing changes.

## Best Possible Solution

Land the Feishu-plugin-owned legacy read path only after old-doc failures propagate clearly, raw text degrades gracefully when rich content fails, legacy token/scope guidance and focused tests are added, and redacted live proof is posted.

## Reproduction Assessment

Yes for the review findings: source inspection at PR head shows the old-doc read is inside the fallback catch and raw text depends on a combined `/raw_content` plus `/content` `Promise.all`. I did not establish a live Feishu reproduction for the feature itself.

## Solution Assessment

No. The Feishu plugin boundary is the right place, but this patch is not the best final solution until the fallback, best-effort rich-content handling, caller guidance, tests, and real proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move legacy reads out of the fallback catch:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the same `try` whose `catch` falls through to the docx path. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real old-doc error is swallowed and retried as docx with the legacy token, hiding the actual failure.
  - confidence: 0.91
- **[P2] Make rich-content fetch best-effort:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but this `Promise.all` makes `/content` a hard dependency. A rich-content failure currently discards raw text that was already readable, so the block metadata fetch should be best-effort when raw content succeeds.
  - confidence: 0.84
- **[P3] Document legacy doc tokens and scopes:** `extensions/feishu/src/docx.ts:856`
  - body: This new branch makes `read` accept legacy `/docs/:token` documents, but the schema and bundled skill still direct callers to `/docx/` token extraction and docx-only scopes. Add legacy token and permission guidance so operators and agents can use the feature.
  - confidence: 0.78
