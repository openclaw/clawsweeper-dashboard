## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid blocking docx reads on legacy probing:** `extensions/feishu/src/docx.ts:853`
  - body: Every `read` now awaits `fetchOldDocMeta` before the existing docx `Promise.all`, and the legacy helper passes a fixed 30,000 ms timeout through `Client.request`. If the legacy endpoint is slow or unreachable, normal `/docx/` reads can stall before current behavior starts and configured Feishu timeouts are overridden.
  - confidence: 0.9
- **[P2] Keep old-doc errors out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the `try` whose `catch` falls through to the docx path. If metadata identifies a legacy doc but either old-doc content request fails, the real old-doc failure is swallowed and retried as docx with the legacy token, masking the cause and likely failing incorrectly.
  - confidence: 0.9
- **[P2] Keep raw old-doc reads working when rich content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns text from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. A rich-content failure discards raw text that was already readable, so the rich block payload should be best-effort when raw text succeeds.
  - confidence: 0.82
