## Summary

Keep open: current main still only supports the typed docx read path, but this PR is not merge-ready because PR head still catches legacy read failures as if they were metadata-detection failures, makes plain-text legacy reads depend on a second rich-content endpoint, and lacks real Feishu proof or regression coverage.

## What This Changes

The branch adds a Feishu old-doc reader using generic Lark client REST calls and routes `feishu_doc` reads through legacy meta detection before the existing docx API path.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after the metadata probe is bounded, old-doc read errors propagate clearly, raw text remains available when rich parsing fails, schema/skill guidance is updated, and real legacy-document proof is supplied.

## Reproduction Assessment

## Solution Assessment

No. The Feishu plugin boundary is the right place, but the current patch is not the best final shape until metadata detection is isolated, rich block parsing is best-effort, the hot-path preflight is bounded, and user-facing token/scope guidance is updated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` still runs inside the `try` whose `catch` falls through to the docx path. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real old-doc failure is swallowed and retried as docx with the legacy token, masking the cause and likely returning a misleading error. Only the metadata probe should be caught.
  - confidence: 0.91
- **[P2] Return raw content when rich parsing fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. If rich-content JSON is unavailable while raw text succeeds, `read` fails instead of returning the plain text it already fetched; make the rich block payload best-effort and keep raw text available.
  - confidence: 0.82
- **[P2] Do not gate docx reads on a long legacy probe:** `extensions/feishu/src/docx.ts:853`
  - body: This new preflight runs before the existing docx `Promise.all`, while the generic old-doc request helper allows a 30s timeout. A slow or unavailable legacy meta endpoint can now stall normal docx reads before fallback; bound the probe tightly or avoid probing tokens known to be docx.
  - confidence: 0.8
- **[P3] Document legacy tokens and scopes:** `extensions/feishu/src/docx.ts:856`
  - body: This line introduces support for legacy `/docs/:token` reads, but the schema and bundled skill still tell agents to use `/docx/XXX` tokens and only list docx scopes. Update the caller-facing guidance so users know how to pass old-document tokens and grant the required legacy doc permissions.
  - confidence: 0.79
