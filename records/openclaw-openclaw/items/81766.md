## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move legacy reads out of the metadata fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the same `try` that is meant to catch metadata lookup failures. If metadata succeeds but the legacy `/content` or `/raw_content` call fails, the real old-doc error is swallowed and the code retries docx with the legacy token, masking the actionable failure.
  - confidence: 0.91
- **[P2] Keep raw legacy reads independent of rich content:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. A rich-content API or scope failure would discard a successful raw-text read, which is the central value of this feature.
  - confidence: 0.84
- **[P3] Update legacy token and scope guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This exposes `/docs/:token` read support, but the schema and bundled skill still tell callers to use only `/docx/` tokens and list only docx permissions. Agents and operators will not know how to invoke or authorize the new path.
  - confidence: 0.79
