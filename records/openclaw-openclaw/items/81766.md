Full review comments:

- **[P1] Avoid blocking docx reads on legacy probing:** `extensions/feishu/src/docx.ts:853-856`
  - body: Every `read` now awaits `fetchOldDocMeta` before the existing docx `Promise.all`. If the legacy endpoint is slow, unavailable, or permission-throttled, normal `/docx/` reads can stall before current behavior starts, so the legacy probe needs to be targeted or bounded without changing the established docx path.
  - confidence: 0.9
- **[P2] Preserve configured Feishu timeouts for old-doc calls:** `extensions/feishu/src/old-doc.ts:53`
  - body: `requestOldDocApi` passes a fixed 30 second timeout into `Client.request`. The SDK forwards that option to the configured HTTP instance, so old-doc probes and reads ignore the user's configured Feishu timeout instead of preserving the existing client-level timeout contract.
  - confidence: 0.86
- **[P2] Keep old-doc errors out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the `try` whose `catch` falls through to the docx path. If metadata identifies a legacy doc but either old-doc content request fails, the real old-doc failure is swallowed and retried as docx with the legacy token.
  - confidence: 0.91
- **[P2] Keep raw old-doc reads working when rich content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns text from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. A rich-content failure discards raw text that was already readable, so the rich block payload should be best-effort when raw text succeeds.
  - confidence: 0.82
