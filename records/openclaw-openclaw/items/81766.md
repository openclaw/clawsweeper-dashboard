## Summary

Keep this PR open: current main still only reads modern Docx documents, and the earlier same-title PR was closed unmerged, so the legacy Feishu read capability is not implemented. The branch is useful but not merge-ready because it still swallows legacy read failures behind the docx fallback, makes plaintext legacy reads depend on the richer content endpoint, and has no real old-document proof.

## What This Changes

The PR adds a Feishu legacy `/docs/:token` read path that detects doc v2 metadata, reads old docs through doc v2 REST endpoints, and routes upgraded docs to the existing docx API.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc failures propagate clearly, raw plaintext can survive rich-content failures, legacy token/scope guidance is updated, and a redacted live old-document read is attached.

## Reproduction Assessment

Yes for the review findings via source inspection: PR head can detect `is_upgraded: false`, then a `/content` or `/raw_content` failure from `readOldDoc` is caught by the metadata fallback and retried as docx with the legacy token. I did not reproduce the real Feishu old-document flow because no live token/proof was provided.

## Solution Assessment

No. The Feishu plugin is the right owner boundary, but this is not the best final shape until the metadata fallback only covers metadata detection, rich-content parsing is best-effort for plaintext reads, and caller-facing legacy token/scope guidance plus live proof are present.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move legacy reads out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the same `try` that is supposed to catch only metadata lookup failures. If metadata succeeds but the legacy `/content` or `/raw_content` read fails, the real old-doc error is swallowed and the code retries the docx API with the legacy token, which masks the actionable failure. Keep only `fetchOldDocMeta` in the fallback catch, then call `readOldDoc` after the catch decision.
  - confidence: 0.91
- **[P2] Keep raw legacy reads independent of rich content:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency before any plaintext can be returned. A rich-content API failure or scope mismatch would discard a successful raw-text read, which is the central value of this feature. Fetch raw content as the required path and make rich content best-effort for title/block stats/hints.
  - confidence: 0.83
- **[P3] Update legacy token and scope guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This line exposes `/docs/:token` support, but the schema and bundled skill still tell callers to use only `/docx/XXX` tokens and list only docx permissions. Agents and operators will not know how to invoke or permission the new path. Update the read parameter description and Feishu doc skill with `/docs/` extraction plus the required legacy doc permissions.
  - confidence: 0.78
