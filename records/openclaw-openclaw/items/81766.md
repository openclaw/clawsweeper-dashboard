## What This Changes

The PR adds Feishu legacy `/docs/:token` read support by probing doc v2 metadata, reading old-doc raw/rich content through `client.request()`, and routing upgraded documents to the docx path.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after the meta probe is bounded, old-doc read failures are not swallowed, raw text remains usable when rich parsing fails, and docs/tests/live proof cover both `/docs/:token` and normal docx reads.

## Solution Assessment

No. The Feishu plugin boundary is the right place, but the current patch is not the best final solution until error handling, raw-content resilience, guidance, tests, and live proof are added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not catch old-doc read failures as meta misses:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` still runs inside the metadata `try`. If metadata succeeds for a legacy document but `/content` or `/raw_content` fails, the catch hides the real old-doc error and falls through to docx with the legacy token, producing a misleading second failure. Keep only `fetchOldDocMeta` in the catch, then call `readOldDoc` after the metadata decision.
  - confidence: 0.92
- **[P2] Keep raw legacy reads working when rich content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. If rich block content fails while raw text succeeds, this new read path throws instead of returning the plain text it already fetched. Fetch rich content best-effort and still return raw content with reduced block metadata.
  - confidence: 0.82
- **[P3] Update legacy token and permission guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This exposes `read` for legacy `/docs/:token` documents, but the existing tool schema and bundled skill still tell callers to extract `/docx/` tokens and list only docx permissions. Add the legacy token extraction and old-doc permission guidance so agents and operators can actually use the new path.
  - confidence: 0.79
