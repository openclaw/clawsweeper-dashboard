Full review comments:

- **[P2] Move legacy reads out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` is awaited inside the same `try` that is supposed to catch metadata lookup failures. If metadata succeeds but `/content` or `/raw_content` fails, the real old-doc error is swallowed and retried as docx with the legacy token, masking the actionable failure.
  - confidence: 0.91
- **[P2] Keep raw legacy reads independent of rich content:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` comes from `/raw_content`, but `Promise.all` makes `/content` a hard dependency first. A rich-content API failure or scope mismatch would discard a successful raw-text read, which is the central value of this feature.
  - confidence: 0.84
- **[P3] Document the legacy token and scope contract:** `extensions/feishu/src/docx.ts:856`
  - body: The new path supports `/docs/:token`, but the schema and bundled skill still tell callers to extract only `/docx/` tokens and list only docx permissions. Agents and operators will not know how to invoke or permission the legacy read path.
  - confidence: 0.78
