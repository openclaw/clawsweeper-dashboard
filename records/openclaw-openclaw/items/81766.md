## Summary

Keep open: this is useful Feishu plugin work, but the PR still needs after-fix real behavior proof and has blocking source-inspection findings in the legacy read path.

## What This Changes

This PR adds Feishu legacy `/docs/:token` read support by introducing doc v2 REST helpers/types and routing `feishu_doc` reads through old-doc metadata detection before the existing docx APIs.

## Best Possible Solution

Keep the legacy read path in the Feishu plugin, but fix error handling, keep raw reads resilient when rich content is unavailable, update caller guidance, and land with focused tests plus redacted live proof.

## Reproduction Assessment

Yes, for the review findings via PR-head source inspection: legacy detection can enter `readOldDoc` inside the fallback catch, and old-doc raw text depends on `/content` in one `Promise.all`. I did not establish live Feishu reproduction for the feature itself.

## Solution Assessment

No. The Feishu plugin boundary is appropriate, but the patch should narrow the fallback catch, make rich content best-effort for plain reads, and update guidance before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` still runs inside the `try` whose `catch` falls back to docx. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real old-doc error is swallowed and retried as docx with the legacy token, producing a misleading failure. Only the metadata lookup should be inside this fallback catch.
  - confidence: 0.91
- **[P2] Keep rich content optional for raw old-doc reads:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns plain text from `/raw_content`, but the `Promise.all` makes `/content` a hard dependency. A rich-content failure currently discards raw text that was already readable; fetch or handle rich block content best-effort so plain legacy reads still work when only block metadata fails.
  - confidence: 0.83
- **[P3] Document legacy document tokens and scopes:** `extensions/feishu/src/docx.ts:856`
  - body: This exposes `read` for legacy `/docs/:token` documents, but the tool schema and bundled skill still tell callers to extract `/docx/` tokens and list only docx scopes. Add legacy token extraction and old-doc permission guidance so agents and operators can use the new path.
  - confidence: 0.78
