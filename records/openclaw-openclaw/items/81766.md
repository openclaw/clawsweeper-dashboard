## Summary

Keep this PR open: the Feishu plugin is the right boundary for legacy document reads, but the current patch still has blocking error-handling issues and lacks after-fix real behavior proof for an actual legacy `/docs/:token` read.

## What This Changes

The PR adds Feishu `feishu_doc` read support for legacy `/docs/:token` documents by detecting doc v2 metadata and adding old-doc REST helper/types.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc read failures propagate clearly, raw text still returns when only rich block content fails, caller guidance covers `/docs/:token`, focused tests are restored, and redacted live proof is provided.

## Reproduction Assessment

Yes, for the review findings via source inspection: PR head awaits `readOldDoc` inside the meta fallback catch and makes raw text depend on `/content` in one `Promise.all`. I did not establish live Feishu reproduction for the feature itself.

## Solution Assessment

No. The Feishu plugin boundary is appropriate, but the patch is not ready until fallback handling is narrowed, rich content is best-effort for raw reads, and legacy token/permission guidance is updated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move old-doc reads out of the meta catch:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` still runs inside the `try` whose `catch` falls back to docx. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real old-doc error is swallowed and retried as docx with the legacy token, producing a misleading failure. Only the metadata lookup should be inside this fallback catch.
  - confidence: 0.91
- **[P2] Make rich old-doc content best-effort:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns plain text from `/raw_content`, but the `Promise.all` makes `/content` a hard dependency. A rich-content failure currently discards raw text that was already readable; fetch or handle rich block content best-effort so plain legacy reads still work when only block metadata fails.
  - confidence: 0.83
- **[P3] Update legacy document guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This exposes `read` for legacy `/docs/:token` documents, but the tool schema and bundled skill still tell callers to extract `/docx/` tokens and list only docx scopes. Add legacy token extraction and old-doc permission guidance so agents and operators can use the new path.
  - confidence: 0.78
