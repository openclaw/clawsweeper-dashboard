Keep open. Current main still lacks a parent-death guard for launcher respawn children, and the issue is already paired with an open closing PR that targets that gap.

## What This Changes

The issue asks OpenClaw to prevent launcher respawn children from outliving `openclaw.mjs` and keeping gateway ports bound during supervised restarts.

## Best Possible Solution

Review, refresh if needed, and land https://github.com/openclaw/openclaw/pull/79571 or an equivalent focused parent-death guard with its regression coverage.

## Solution Assessment

Yes. Fixing the launcher child lifecycle with a parent-death guard is narrower and more maintainable than broadening systemd `fuser` cleanup policy, and the linked PR follows that direction.
