## Summary

Keep open: current main and the latest release still lack a child-side parent-death guard for launcher respawns; the closed fixing PR is unmerged, and systemd/stale-port cleanup only mitigates part of the supervisor path.

## What This Changes

The issue reports that the gateway launcher can leave a spawned child process alive after parent termination, causing systemd restarts to fail with EADDRINUSE.

## Best Possible Solution

Land an equivalent parent-death guard in the launcher/shared respawn supervision path, with SIGKILL/crash-style regression coverage and Linux systemd restart proof; keep stale-port cleanup as secondary defense.

## Reproduction Assessment

No live current-main systemd reproduction was run in this read-only review. Source inspection shows the parent-side cleanup depends on signal handlers, while the reported parent SIGKILL/crash path cannot run those handlers.

## Solution Assessment

Yes. A parent-death guard in the launcher/shared respawn path is the narrow maintainable fix; systemd `KillMode` and stale-port cleanup are useful mitigations but do not solve manually launched or different-owner orphan children.
