## Summary

Keep open. Current main still lacks a parent-death guard for respawned launcher children, and the already-linked open PR at https://github.com/openclaw/openclaw/pull/79571 is the active implementation candidate that uses closing syntax for this issue.

## What This Changes

Ensure gateway launcher respawn children terminate when the wrapper dies so systemd restarts do not fail on a still-bound gateway port.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79571 if it remains correct, then close this issue as implemented by that merged fix.

## Reproduction Assessment

Yes at source level, but not through a live systemd/root-owned cycle in this read-only review. Current main forwards catchable signals from the wrapper but has no parent-death guard, matching the reported SIGKILL/crash orphan path.

## Solution Assessment

Yes. A focused parent-PID/death guard in the launcher and respawn bridge is narrower than relying on root-owned `fuser` hooks and directly addresses the child lifecycle failure.
