## Summary

Keep open. Current main still lacks a parent-death guard for respawned launcher children, and an open linked pull request is already the implementation path, so this issue should close only after that pull request lands.

## What This Changes

The issue reports that a gateway child spawned by `openclaw.mjs gateway` can survive wrapper termination, keep the gateway port bound, and block systemd restarts with `EADDRINUSE`.

## Best Possible Solution

Land a focused parent-death guard for respawned launcher children after reviewing https://github.com/openclaw/openclaw/pull/79571, while leaving the broader custom-supervisor SIGUSR1 behavior to its separate open thread.

## Reproduction Assessment

No, not for the exact live systemd/root-owned setup in this read-only review. Source inspection on current main gives a clear reproduction path for the remaining gap: respawn children receive forwarded catchable signals, but there is no guard for wrapper SIGKILL or crash.

## Solution Assessment

Yes. Fixing the launcher respawn lifecycle with a parent-death guard is the narrow maintainable direction; root-level `fuser` service hooks are broader defensive cleanup and do not solve the wrapper child lifecycle itself.
