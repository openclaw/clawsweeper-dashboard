## Summary

Keep open. Current main still lacks a parent-death guard for launcher respawn children, and https://github.com/openclaw/openclaw/pull/79571 is still open with closing syntax for this issue.

## What This Changes

The issue reports that `openclaw.mjs gateway` respawn children can outlive the wrapper after parent death, keep the gateway port bound, and make systemd restarts fail with `EADDRINUSE`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79571, or an equivalent focused parent-death guard with regression coverage, then close this issue as implemented after merge.

## Reproduction Assessment

No live, high-confidence Ubuntu systemd reproduction was run in this read-only pass. Source inspection does show the current-main gap: catchable signals are forwarded, but SIGKILL or wrapper crash cannot run that cleanup path.

## Solution Assessment

Yes. Fixing the launcher child lifecycle with a parent-death guard is narrower and more maintainable than broadening systemd `fuser` cleanup policy, and the linked PR follows that direction.
