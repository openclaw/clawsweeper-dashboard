## Summary

Keep open. Current main still lacks a parent-death guard for respawned launcher children, and the live linked PR with closing syntax should remain the implementation path until it merges or is closed.

## What This Changes

The issue reports that `openclaw.mjs gateway` can leave its respawned child alive after parent termination, keeping the gateway port bound and breaking supervisor restarts.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/79571; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes, at source level: current main forwards catchable signals but has no parent-death guard, so a parent SIGKILL/crash cannot run the cleanup path. I did not run the live Ubuntu systemd/root-owned reproduction in this read-only review.

## Solution Assessment

Yes, the linked parent-death guard is the narrowest maintainable direction. A root `fuser` service hook would be broader operational policy and would not fix the wrapper lifecycle itself.
