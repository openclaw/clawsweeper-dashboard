## Summary

Keep open: current main and v2026.6.6 still cover only catchable signal forwarding plus service-mode stale-port cleanup, not the requested parent-death guard for wrapper SIGKILL/crash/root-owned orphan cases. The focused fixing PR was closed unmerged, and the related systemd report was closed into this canonical issue rather than replacing it.

## What This Changes

The issue requests that `openclaw.mjs gateway` prevent its respawned gateway child from outliving the wrapper and blocking supervised restarts with `EADDRINUSE`.

## Best Possible Solution

Land an equivalent parent-death guard in the launcher/shared respawn supervision path, with regression coverage for SIGKILL/crash-style wrapper death and systemd cleanup kept as secondary defense.

## Reproduction Assessment

No live systemd reproduction was run in this read-only review. The source-level path is clear for wrapper death that bypasses signal handlers because current respawn supervision only reacts when the parent receives a catchable signal.

## Solution Assessment

Yes. A parent-death guard in the launcher/shared respawn path is the maintainable fix; the systemd `fuser` or stale-port cleanup paths are defensive mitigations and do not solve manually launched or out-of-cgroup orphan children.
