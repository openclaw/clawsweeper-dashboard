## Summary

Keep open. Current main already has root-execution protection, systemd control-group cleanup, and SIGTERM forwarding for launcher respawns, but the remaining parent-death case described here is not fully closed because the wrapper has no guard for an uncatchable parent crash or SIGKILL leaving its respawned child alive.

## What This Changes

Harden the gateway launcher so a respawned child cannot outlive the wrapper and keep the gateway port bound across service restarts.

## Best Possible Solution

Add a parent-death guard for launcher respawn children, keep the current root guard and systemd cgroup behavior, and cover SIGKILL/parent-crash plus existing SIGTERM behavior with focused tests.

## Reproduction Assessment

Unclear for the exact systemd/root setup without a live host run. Source inspection gives a clear current-main path for the remaining parent-death gap: the launcher forwards catchable signals but does not arrange for the child to exit when the wrapper disappears without running a signal handler.

## Solution Assessment

Yes, fixing the wrapper lifecycle is the best direction. The current root guard and systemd cgroup settings are useful defenses, but a root `fuser` workaround would be broader and still would not solve parent-death behavior in the launcher itself.
