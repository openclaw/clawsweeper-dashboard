## Summary

Keep open. Current main still only forwards catchable wrapper signals, and the issue is now paired with an open closing PR that specifically implements the parent-death guard rather than leaving this for cleanup closure.

## What This Changes

The issue reports that launcher-spawned gateway children can survive wrapper termination and keep the gateway port bound during systemd restarts.

## Best Possible Solution

Review and land or close the linked parent-death guard PR so respawned launcher children cannot outlive their wrapper; keep systemd root cleanup policy as a separate maintainer choice.

## Reproduction Assessment

Not as a high-confidence live reproduction in this review. The current-main source path is clear: respawn parents only forward catchable signals and do not guard children against parent SIGKILL or crash.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path. A launcher child lifecycle guard addresses the root cause more directly than broadening systemd `fuser` cleanup hooks.
