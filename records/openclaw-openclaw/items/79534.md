## Summary

Keep open: the report is fresh and detailed, and while current main already has SIGTERM forwarding, force-kill fallback, systemd `KillMode=control-group`, and root-execution guards, source inspection still shows no parent-death guarantee for wrapper crash/SIGKILL and no duplicate or fix PR.

## What This Changes

The issue asks OpenClaw to prevent the `openclaw.mjs gateway` respawn child from outliving its wrapper and blocking systemd restarts on the gateway port.

## Best Possible Solution

Make respawn lifecycle deterministic under supervisor shutdown, preferably by extending the shared respawn or child-process bridge path with a Linux-safe parent-death mechanism and documenting any system-service-only fallback.

## Reproduction Assessment

Unclear overall. The current SIGTERM path is covered by source and tests, but source inspection shows a remaining parent-death gap for wrapper crash or SIGKILL because no PDEATHSIG or child-side parent watchdog is installed.

## Solution Assessment

Unclear. The `+ fuser` workaround is useful operator evidence but broadens root process-kill behavior, so maintainers should choose whether core parent-death handling or system-service documentation is the supported fix.
