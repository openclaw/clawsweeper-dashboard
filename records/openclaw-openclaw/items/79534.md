## Summary

Keep open because current main still lacks the parent-death guard for respawned launcher children, and an open closing PR already targets this exact lifecycle bug. The issue should stay open until https://github.com/openclaw/openclaw/pull/79571 lands or is replaced.

## What This Changes

The issue reports that `openclaw.mjs gateway` respawn children can survive wrapper termination and keep gateway ports bound, causing systemd restarts to fail with `EADDRINUSE`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79571 if checks and maintainer review stay clean; otherwise replace it with the same focused parent-death guard in the launcher, compile-cache, and CLI respawn surfaces.

## Reproduction Assessment

Yes, at source level. Current main forwards catchable signals from the wrapper to respawn children but has no parent-death guard, so wrapper SIGKILL/crash cannot trigger cleanup.

## Solution Assessment

Yes. A parent-death guard in the respawn lifecycle is narrower and more maintainable than relying on root `fuser` service hooks, and the open closing PR follows that direction.
