## Summary

Keep open. Current main still lacks a parent-death guard for respawned launcher children, and the issue is already paired with an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

Ensure the gateway launcher wrapper's respawned child cannot outlive the wrapper and keep the gateway port bound after parent termination.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79571 or an equivalent parent-death guard, while leaving the broader custom-supervisor SIGUSR1 behavior to https://github.com/openclaw/openclaw/issues/65668.

## Reproduction Assessment

Yes, at source level. Current main forwards catchable signals but has no parent-death guard, so wrapper SIGKILL/crash cannot trigger child cleanup; I did not run the live systemd/root-owned cycle here.

## Solution Assessment

Yes. A focused parent-death guard in the launcher/respawn child lifecycle is narrower than a root `fuser` service hook and directly addresses the child-survives-parent failure mode.
