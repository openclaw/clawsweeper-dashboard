## Summary

Keep open. Current main still lacks a Discord `slashCommandDeploy` config surface and persisted cross-restart command fingerprinting, and #75961 is an open PR that explicitly targets this issue, so the issue should remain open until that PR is reviewed and either landed or replaced.

## What This Changes

Expose Discord slash-command deploy lifecycle controls so unchanged restarts can skip deploys and operators can disable startup deployment.

## Best Possible Solution
