## Summary

Keep open: the PR addresses a real source-reproducible `/compact` reply bug, but it also forwards the model context-window capacity as live compaction usage, which can make manual compaction report and persist bogus `tokensBefore` values.

## What This Changes

The branch updates `src/auto-reply/reply/commands-compact.ts` to clamp manual `/compact` labels, estimate stale session totals from JSONL file size, and pass a token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless a real live usage snapshot is available, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts exist, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.
