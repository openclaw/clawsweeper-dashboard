## Summary

Keep open. The PR addresses a real `/compact` UX bug and has sufficient copied live Telegram output, but the current branch has a blocking correctness issue: it forwards the model context window as `currentTokenCount`, which the compactor treats as live prompt usage for returned and persisted before-counts.

## What This Changes

The PR updates `src/auto-reply/reply/commands-compact.ts` to suppress non-reducing `/compact` arrows, estimate stale totals from the session file, and pass a derived token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, but only supplies `currentTokenCount` from a trustworthy live usage snapshot or omits it.

## Reproduction Assessment

Yes, at source level. Current main still renders the before/after label whenever both counts are present, and the PR body includes copied Telegram `/compact` output for the small-session case.

## Solution Assessment

No, not as written. The label and stale-total fallback are the right direction, but passing `contextTokens` as `currentTokenCount` violates the compactor contract; the safer fix is to omit that field unless real live usage is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compactor:** `src/auto-reply/reply/commands-compact.ts:178`
  - body: `observedContextTokens` is derived from the model context window, but `compactEmbeddedPiSession.currentTokenCount` is caller-observed live prompt usage and is reused for token-after estimation plus persisted/returned `tokensBefore`. Passing the window size can make manual `/compact` report or store a bogus before count, so omit this field unless the command has an actual usage snapshot.
  - confidence: 0.94
