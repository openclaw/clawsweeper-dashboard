## Summary

Keep open: the reported `/compact` label problem is valid on current main, but this PR still has a correctness blocker because it passes context-window capacity as live `currentTokenCount`, which can corrupt the reported and persisted pre-compaction token count.

## What This Changes

The PR changes manual `/compact` reply formatting and token accounting in `src/auto-reply/reply/commands-compact.ts` to suppress non-reducing labels and estimate stale context totals from the session JSONL file.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless manual `/compact` has actual live usage, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after compact label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total display fallback are the right direction, but passing `contextTokens` as `currentTokenCount` violates the compactor contract; omit the field unless a real live usage snapshot is available.
