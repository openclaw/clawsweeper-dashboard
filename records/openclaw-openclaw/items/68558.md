## Summary

Keep this PR open: the user-visible `/compact` formatter bug is still present on current main and the latest release, but the submitted branch is stale and its token-source changes would mix context budgets and rough file-size estimates into usage/session state.

## What This Changes

The branch changes `src/auto-reply/reply/commands-compact.ts` to clamp `/compact` labels, forward `currentTokenCount` from context-token fields, and estimate fallback totals from JSONL file size.

## Best Possible Solution

Port only the non-decreasing-arrow formatter guard onto current main, add focused tests, and keep usage tokens, context budgets, and stale-token fallback decisions on separate proven paths.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` still render a `tokensBefore -> tokensAfter` label whenever both fields are non-null. A compact result like `tokensBefore: 0` and `tokensAfter: 36` therefore still has the reported misleading output path.

## Solution Assessment

No, not as submitted. The formatter clamp is useful, but the best fix is to port that guard onto current main without treating context-window budgets or rough file-size estimates as canonical usage.

- **[P1] Pass live usage tokens, not context windows:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens` or persisted `contextTokens`, which are context-window budget values, but `currentTokenCount` is caller-observed live prompt-token usage. Passing a 272k/1M window here can make manual `/compact` report or checkpoint a huge `tokensBefore` value for a small session instead of actual usage.
  - confidence: 0.93
- **[P2] Do not let file size override fresh token totals:** `src/auto-reply/reply/commands-compact.ts:224`
  - body: `Math.max(storedTotal, fileEstimate)` lets a rough JSONL bytes/4 estimate replace a fresh stored token total whenever transcript JSON is large, especially with tool results. Use this only when the stored total is missing or known stale, or keep the canonical stored/session result so the context summary does not overstate usage.
  - confidence: 0.86
