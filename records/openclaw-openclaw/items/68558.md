## Summary

Keep open: the PR targets a real `/compact` UX bug, but the patch still has a blocking correctness issue because it forwards model context-window limits as live compaction usage tokens.

## What This Changes

The branch changes `commands-compact.ts` to adjust `/compact` labels, estimate stale session totals from JSONL file size, and forward a computed token count to embedded compaction.

## Best Possible Solution

Land a corrected patch that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless true live prompt usage is available, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current `main` still renders the unconditional `before → after` label path, and the PR body includes copied before/after Telegram output for the small-session case.

## Solution Assessment

No. The label guard and stale-total fallback are in the right owner module, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract and should be removed unless a real usage snapshot is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` is derived from `params.contextTokens` or `targetSessionEntry.contextTokens`, which are model context-window limits, but `currentTokenCount` is caller-observed live prompt usage and is returned as `tokensBefore`/used for token-after sanity checks. Passing a window size can make `/compact` report the window as the before count and skew diagnostics; omit it unless real usage is available.
  - confidence: 0.94
