## Summary

Keep open: this PR addresses a real current-main `/compact` reply bug, but the branch still has a blocking contract regression because it forwards model context-window size as `currentTokenCount`. The useful label guard and stale-total fallback should be kept, but the token forward needs repair before merge.

## What This Changes

The PR modifies `src/auto-reply/reply/commands-compact.ts` to change manual `/compact` reply labels, pass a token count into compaction, and estimate stale session totals from the JSONL file size.

## Best Possible Solution

Land a corrected patch that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless true live prompt usage is available, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main can be exercised with a focused command test that mocks `compactEmbeddedPiSession` to return `tokensBefore <= tokensAfter`, which will render the misleading arrow today; the PR body also includes copied before/after Telegram output.

## Solution Assessment

No. The label guard and stale-total fallback are in the right owner module, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract because that field expects live prompt usage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens` or `targetSessionEntry.contextTokens`, which are model context-window limits. `compactEmbeddedPiSession.currentTokenCount` is caller-observed live prompt usage and is reused for `tokensBefore` and token-after estimation, so this can make manual `/compact` report the window size as the before count and skew compaction diagnostics. Omit this field unless the handler has a real usage snapshot.
  - confidence: 0.94
