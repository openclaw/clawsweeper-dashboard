## Summary

Keep open: the reported `/compact` reply problem is real, but this PR still forwards a context-window capacity as `currentTokenCount`, which the compactor treats as observed live prompt usage and can return or persist as `tokensBefore`.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to forward a derived token count, guard non-reducing compact labels, and estimate stale context totals from session file size.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, only forwards real live usage tokens when available, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after compact label whenever both counts exist, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata; omit it unless manual `/compact` has a real live usage snapshot.
