## Summary

This focused bug-fix PR should stay open, but the current patch still has a blocking correctness issue: it passes the model context-window value as `currentTokenCount`, which the compactor treats as observed live prompt usage. That can replace the original misleading small-session label with window-sized token accounting in manual `/compact` output and checkpoints.

## What This Changes

The PR changes manual `/compact` reply formatting and context-summary accounting in `src/auto-reply/reply/commands-compact.ts` by suppressing non-reducing before/after labels, forwarding a token count into compaction, and estimating stale totals from the session JSONL file.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless manual `/compact` has actual live usage, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders a before/after compact label whenever both token counts are present, and the PR body provides copied Telegram before/after output for a small session with stale stored totals.

## Solution Assessment

No, not as written. The label-rendering guard and stale-total fallback are the right direction, but forwarding context-window capacity as `currentTokenCount` violates the compactor usage contract and should be removed or replaced with real live usage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only observed usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens` or the session entry, which are used as the model context-window denominator, not live prompt usage. Passing that value as `currentTokenCount` lets the compactor use a capacity value as `tokensBefore` and persist it in checkpoint metadata, so manual `/compact` can report or store window-sized counts instead of the actual session size.
