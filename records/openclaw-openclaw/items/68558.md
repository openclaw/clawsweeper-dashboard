## Summary

Keep this PR open, but it still needs changes before merge: it fixes a real `/compact` reply-label problem, but the current patch forwards a context-window budget as live prompt usage and can create new misleading compaction counts.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to adjust manual `/compact` labels, pass a token count into compaction, and estimate stale session totals from the session JSONL file size.

## Best Possible Solution

Land a corrected version that keeps the non-decreasing-label guard and stale-total display fallback, but omits `currentTokenCount` unless the handler has a real live prompt-usage snapshot.

## Reproduction Assessment

Yes, at source level. Current `main` unconditionally renders the compacted arrow when both values exist, and the PR body provides copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No. The label guard and stale-total fallback belong in this command handler, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract and should be removed or replaced with actual usage data.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
