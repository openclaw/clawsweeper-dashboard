## Summary

Keep open: the reported `/compact` reply bug is real, but this PR is not merge-ready because it passes model context-window metadata as `currentTokenCount`, which the compactor treats as live prompt usage and can persist or return as `tokensBefore`.

## What This Changes

The branch changes `src/auto-reply/reply/commands-compact.ts` to guard misleading compact labels, pass an observed token value into compaction, and estimate stale totals from the session JSONL file.

## Best Possible Solution

Land a corrected version that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real prompt-usage data is available, and preserves current-main `resolveManualCompactContextTokenBudget` behavior.

## Reproduction Assessment

Yes, at source level. Current main unconditionally renders the before/after arrow when both counts exist, and the PR body plus a production comment provide Telegram-visible examples of misleading compact context output.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but forwarding context-window metadata as live prompt usage is not the maintainable fix.
