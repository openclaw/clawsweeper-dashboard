The PR changes manual `/compact` reply formatting in `src/auto-reply/reply/commands-compact.ts`, adds a session-file token estimate fallback, and forwards a token count into manual compaction.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real live usage is available, preserves current-main context-budget resolution, and adds focused regression tests.

## Reproduction Assessment

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata violates the compactor contract; omit it unless real prompt usage is available.
