## Summary

Keep open: the misleading non-decreasing `/compact` label is still present on current main and v2026.6.10, but this PR is not merge-ready because it mixes context-window budgets into live token usage and adds a broad file-size estimate that can override token state.

## What This Changes

The PR changes `/compact` reply formatting, passes a token count into manual compaction, and adds a JSONL file-size estimate fallback for context usage in `src/auto-reply/reply/commands-compact.ts`.

## Best Possible Solution

Port the non-decreasing label guard onto current main while keeping context budget and observed usage tokens separate; only add a stale-token fallback if maintainers define a narrow tested contract for stale totals.

## Reproduction Assessment

Yes. Current main and v2026.6.10 still format `tokensBefore -> tokensAfter` whenever both values are non-null, so a compact result like 0 to 36 remains source-reproducible without a live channel.

## Solution Assessment

No, not as submitted. The formatter guard is the right narrow fix, but forwarding context-window values as live usage and letting JSONL byte estimates override stored totals is not the safest implementation path.
