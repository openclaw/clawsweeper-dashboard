## Summary

Keep open. The reported `/compact` UX bug is real and the branch contains useful label/context-summary fixes, but it still has a blocking compactor-contract bug because it passes model context-window limits as observed live usage tokens.

## What This Changes

The PR changes manual `/compact` reply formatting, forwards a token count into embedded compaction, and estimates stale session totals from the JSONL file size.

## Best Possible Solution

Land a corrected compact-command patch that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless true live prompt usage is available, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current `main` still formats `Compacted (before → after)` whenever both values exist, so a focused command test that mocks `tokensBefore <= tokensAfter` reproduces the misleading output path.

## Solution Assessment

No. The label guard and stale-total fallback are in the right owner module, but forwarding `contextTokens` as `currentTokenCount` is not the narrow correct fix because that field expects live prompt usage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` is derived from `params.contextTokens` or `targetSessionEntry.contextTokens`, which are model context-window limits, but `currentTokenCount` is defined as caller-observed live prompt tokens and is returned as `tokensBefore`. Passing a window size can make `/compact` report the model window as the before count and skew diagnostics; omit this field unless a real usage snapshot is available.
  - confidence: 0.94
