## Summary

Keep open: the visible `/compact` bug is valid and current `main` still has the misleading label/stale-total path, but this PR also passes context-window budgets as live prompt token usage, which would produce bogus compaction diagnostics.

## What This Changes

The PR changes manual `/compact` reply formatting, adds a JSONL file-size fallback for stale token totals, and forwards a token count into the compactor.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real prompt usage is available, preserves current-main context-budget resolution, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current `main` still formats the arrow whenever both token counts are present and still uses the stale stored-total path for the context summary, while the PR body includes copied Telegram before/after output for the small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata violates the compactor contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass live usage tokens, not context windows:** `src/auto-reply/reply/commands-compact.ts:128-134`
