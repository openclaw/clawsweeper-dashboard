## Summary

Keep open: the `/compact` UX bug is real and the label/fallback parts are directionally useful, but this PR passes model context-window metadata as live prompt usage, which can create new bogus `tokensBefore` output and checkpoint data.

## What This Changes

The PR changes manual `/compact` reply formatting to avoid non-decreasing before/after arrows, adds a JSONL file-size token estimate fallback, and forwards a token count into compaction.

## Best Possible Solution

Land a corrected version that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real live usage is available, adds focused coverage, and preserves current-main context-budget resolution.

## Reproduction Assessment

Yes, at source level. Current main formats the misleading before/after arrow whenever both token counts exist, and the PR body includes copied Telegram before/after output for the small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but forwarding context-window metadata as live prompt usage is not the narrow maintainable fix.
