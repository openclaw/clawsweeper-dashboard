## Summary

Keep this PR open: current main still has the misleading `/compact` label path and the PR contains useful UX fixes, but the patch is not mergeable as written because it passes model context-window metadata as live `currentTokenCount`.

## What This Changes

The PR changes `/compact` reply formatting to suppress non-reducing before/after arrows, estimate stale context totals from the session JSONL file, and forward a token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, but only supplies `currentTokenCount` from a trustworthy live usage snapshot or omits it.

## Reproduction Assessment

Yes, at source level. Current main still renders the before/after compact label whenever both counts exist, and the PR body includes copied Telegram output for the small-session case.

## Solution Assessment

No, not as written. The label and fallback direction is appropriate, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract; omit that field unless real live usage is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compactor:** `src/auto-reply/reply/commands-compact.ts:178`
  - body: `observedContextTokens` comes from model context-window metadata, but `compactEmbeddedPiSession.currentTokenCount` is treated as observed live prompt usage and reused for token-after estimation plus persisted/returned `tokensBefore`. Passing the window size can make manual `/compact` store or report a bogus before count near the model limit, so omit this field unless the command has a real usage snapshot.
