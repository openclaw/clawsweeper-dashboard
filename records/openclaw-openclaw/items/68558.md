## Summary

Keep open: the reported `/compact` UX bug is valid, but this PR still passes context-window capacity as live prompt usage, which would create new bogus `tokensBefore` values in manual compaction output and checkpoints.

## What This Changes

The PR changes the manual `/compact` reply path to guard non-reducing compact labels, estimate stale context totals from the session file, and forward a token count to the embedded compactor.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless real live prompt usage is available, and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both counts exist, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.

## Review Findings
