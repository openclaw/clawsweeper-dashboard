## Summary

Keep open: current main still has the misleading `/compact` label path, but this PR should not merge as written because it forwards a context-window limit as `currentTokenCount`, which the compactor treats as observed prompt usage and can persist as `tokensBefore`.

## What This Changes

This PR changes manual `/compact` reply formatting, adds a JSONL file-size fallback for stale totals, and forwards a token count into manual compaction.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real prompt-usage measurement is available, preserves current-main context-budget resolution, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders `Compacted (before → after)` whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata is not a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stop forwarding context-window tokens:** `src/auto-reply/reply/commands-compact.ts:129-131`
