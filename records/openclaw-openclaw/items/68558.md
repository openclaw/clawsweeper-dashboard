## Summary

Keep open: this PR targets a real `/compact` reply bug, but the current branch still passes context-window metadata as `currentTokenCount`, which can make compaction diagnostics and persisted `tokensBefore` wrong.

## What This Changes

This PR changes manual `/compact` reply formatting, adds a JSONL file-size estimate for stale session totals, and forwards a token count into manual compaction.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real live usage is available, rebases onto current main, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still renders the before/after arrow without checking that `before > after`, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The formatting guard and stale-total fallback are the right direction, but passing context-window metadata as observed live usage is not a safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stop passing context-window tokens as observed usage:** `src/auto-reply/reply/commands-compact.ts:169`
