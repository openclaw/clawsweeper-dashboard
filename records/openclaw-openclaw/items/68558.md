## Summary

Keep open: the underlying `/compact` display bug is source-reproducible on current main, but this PR still forwards context-window metadata as live compaction usage, which can make manual compaction report and persist bogus token counts.

## What This Changes

The PR changes manual `/compact` reply formatting, adds a JSONL-size fallback for stale context totals, and forwards a token count into the compactor.

## Best Possible Solution

Land a corrected `/compact` reply fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real live usage snapshot exists, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.
