## Summary

Keep open: current main still has the misleading manual `/compact` label path, and this PR contains useful user-facing work, but the patch is not mergeable as written because it passes context-window metadata as `currentTokenCount` and is now conflicting with current main.

## What This Changes

The PR changes manual `/compact` reply formatting, adds a JSONL file-size fallback for stale session totals, and forwards a token count into manual compaction.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real live usage measurement is available, preserves current-main context-budget resolution, and adds focused regression tests.

## Reproduction Assessment

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata is not a maintainable fix.
