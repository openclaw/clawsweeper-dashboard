## Summary

Keep this PR open: it addresses a real `/compact` reply bug and has useful label/fallback work, but the current branch still passes context-window capacity as `currentTokenCount`, which violates the compactor contract and can create bogus `tokensBefore` values.

## What This Changes

The PR changes manual `/compact` reply formatting, adds a session-file size fallback for stale totals, and forwards a token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real prompt-usage count is available, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94
