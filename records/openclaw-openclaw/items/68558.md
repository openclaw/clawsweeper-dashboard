## Summary

Keep open: this is a useful, source-reproducible bug fix, but the PR is not mergeable as written because it forwards a context-window limit as the compactor’s live usage token count.

## What This Changes

This PR updates manual `/compact` reply formatting to suppress non-reducing before/after arrows, add a session-file token estimate for stale totals, and forward a derived token count into compaction.

## Best Possible Solution

Land a corrected compact-command change that keeps the label guard and stale-total fallback, but supplies `currentTokenCount` only from a trustworthy live usage snapshot.

## Reproduction Assessment

Yes, at source level. Current main still has the unconditional before/after label path, and the PR body includes copied Telegram `/compact` before/after output for the small-session case.

## Solution Assessment

No, not as written. The reply-label and stale-total fallback are the right direction, but passing `contextTokens` as `currentTokenCount` violates the compactor’s live-usage contract.
