## Summary

Keep open: current main and the latest release still have the misleading `/compact` label path, but this branch is conflicting with current main and its extra token-accounting changes can misreport session usage. The useful part is the non-decreasing label guard; it should be ported narrowly with focused tests.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to guard non-decreasing compaction labels, pass context-token fields as `currentTokenCount`, and estimate fallback usage from JSONL file size.

## Best Possible Solution

Port the `before > after` formatter guard onto current main, preserve current compaction runtime/session contracts, and add focused tests for non-decreasing labels and token-source selection.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` still render a `tokensBefore -> tokensAfter` label whenever both fields are non-null. A compact result like `tokensBefore: 0` and `tokensAfter: 36` therefore still has the reported misleading output path.

## Solution Assessment

No, not as submitted. The formatter clamp is useful, but the best fix is to port that guard onto current main without treating context-window budgets or rough file-size estimates as canonical usage.
