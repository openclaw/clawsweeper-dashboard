## Summary

Keep this PR open because it addresses a real `/compact` reply bug, but the current branch still has a blocking contract issue: it forwards a context-window budget as `currentTokenCount`, which the compactor treats as observed live prompt usage.

## What This Changes

The PR changes manual `/compact` reply formatting, adds a session JSONL file-size token estimate for stale totals, and forwards a token count into compaction.

## Best Possible Solution

Land a corrected patch that preserves current-main context-budget resolution, guards non-shrinking labels, estimates stale totals from the session file, and does not pass `currentTokenCount` unless true usage is available.

## Reproduction Assessment

Yes, at source level. Current main builds the misleading arrow whenever both token counts exist, and the PR body includes copied Telegram before/after `/compact` output for a small session.

## Solution Assessment

No, not as written. The label guard and file-size fallback are the right direction, but forwarding context-window metadata as `currentTokenCount` is not the narrow correct fix.
