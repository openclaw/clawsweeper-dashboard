## Summary

Keep open: the PR addresses a real misleading `/compact` reply, but it is not mergeable as written because it passes model context-window capacity as `currentTokenCount`, which the compactor treats as live prompt usage and can persist or report as `tokensBefore`.

## What This Changes

The PR updates `src/auto-reply/reply/commands-compact.ts` to suppress non-reducing compact labels, estimate stale context totals from the session JSONL file, and pass a token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless manual `/compact` has actual live usage, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both counts are present, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but passing `contextTokens` as `currentTokenCount` violates the compactor contract; omit the field unless a real live usage snapshot is available.
