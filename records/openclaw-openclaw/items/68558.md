## Summary

Keep this PR open, but it needs changes before merge because one part of the patch feeds model context-window metadata into a compactor field that is explicitly live prompt usage.

## What This Changes

The PR changes manual `/compact` reply formatting to suppress non-reducing before/after labels, estimate stale totals from the session file, and pass a token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless actual live usage is available, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current `main` still renders the before/after label whenever both counts are present, and the PR body includes copied Telegram output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` should not be populated from `contextTokens`; omit it unless manual `/compact` has an actual live usage snapshot.

## Review Findings

Overall correctness: patch is incorrect
