## Summary

Keep open: the formatter bug is still real on current main and the latest release, but this branch is not merge-ready because it mixes context-window budgets into live usage tokens and adds a rough transcript-byte fallback that can misreport context usage.

## What This Changes

The PR changes the manual `/compact` reply path to suppress non-decreasing before/after labels, forward a token count into compaction, and display a JSONL file-size fallback when stored totals look stale.

## Best Possible Solution

Port only the non-decreasing `/compact` label guard onto current main, keep context budget and observed usage tokens separate, and leave broader stale Codex accounting to https://github.com/openclaw/openclaw/issues/66263 unless a narrow tested fallback is approved.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` still render the before/after arrow whenever both compaction token fields are non-null, so a compact result like `0 → 36` remains source-reproducible.

## Solution Assessment

No, not as submitted. The non-decreasing label guard is the right narrow fix, but using context windows as live usage and letting byte estimates override totals is not the safest implementation.
