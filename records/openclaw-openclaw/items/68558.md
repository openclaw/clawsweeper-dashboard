## Best Possible Solution

Land a corrected version that keeps the reply-label guard and stale-total fallback, preserves current-main context-budget resolution, and only passes `currentTokenCount` when real live prompt usage is available.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after compaction arrow whenever both token values are present, and the PR body includes copied Telegram before/after output for the small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but forwarding context-window metadata as live prompt usage is not the narrow maintainable fix.
