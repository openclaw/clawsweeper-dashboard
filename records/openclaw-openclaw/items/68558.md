## Summary

Keep open: current main still has the visible `/compact` label bug, but this PR is not mergeable as written because it passes context-window metadata as `currentTokenCount`, which the compactor treats as observed prompt usage.

## What This Changes

This PR changes manual `/compact` reply formatting to hide non-shrinking token arrows, estimate stale session totals from JSONL file size, and forward a token count into compaction.

## Best Possible Solution

Land a corrected current-main patch that guards the label, estimates stale totals from the session file, omits `currentTokenCount` unless true usage is available, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main still builds `Compacted (before → after)` whenever both counts exist, and the PR body includes copied Telegram before/after output for a small session.

## Solution Assessment

No, not as written. The label guard and stale-token fallback are the right direction, but forwarding context-window metadata as `currentTokenCount` is not the narrow correct fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:166`

  - confidence: 0.94
