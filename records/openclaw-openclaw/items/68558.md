## Summary

Keep open: the useful `/compact` UX fix is still needed, but this PR should not merge as written because it passes a model context-window value into a compactor parameter that is defined as live prompt usage.

## What This Changes

The branch changes `src/auto-reply/reply/commands-compact.ts` to suppress non-reducing `/compact` labels, estimate stale totals from the session file, and forward a derived token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless actual live usage is available, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current main still renders the before/after label whenever both counts are present, and the PR body includes copied Telegram output for the small-session case.

## Solution Assessment

No, not as written. The label and stale-total fallback are the right direction, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract; the safer fix is to omit that field unless real live usage is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compactor:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` is derived from the model context window, but `compactEmbeddedPiSession.currentTokenCount` is caller-observed live prompt usage and is reused for token-after estimation plus persisted/returned `tokensBefore`. Passing the window size can make manual `/compact` report or store bogus before counts, so omit this field unless the command has an actual usage snapshot.
  - confidence: 0.94
