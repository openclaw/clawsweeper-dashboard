## Summary

Keep open: the visible `/compact` reply fix is useful, but the branch still forwards model context-window metadata as `currentTokenCount`, which violates the compactor contract and can report or persist bogus `tokensBefore` values.

## What This Changes

This PR changes manual `/compact` reply formatting to suppress non-reducing before/after labels, estimate stale context totals from session-file size, and forward a token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real live prompt-usage count is available, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.

## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Omit context-window tokens from currentTokenCount:** `src/auto-reply/reply/commands-compact.ts:169`
