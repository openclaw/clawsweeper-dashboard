## Summary

Keep open: current main still has the misleading small-session `/compact` formatting path, but this PR is not safe to merge as written because it forwards context-window budgets as `currentTokenCount`, which the compactor treats as live prompt usage.

## What This Changes

The PR changes manual `/compact` replies to guard the compaction label, estimate stale session totals from the JSONL file, and pass a token count into the compactor.

## Best Possible Solution

Land a corrected patch that guards the compaction label, estimates stale totals from the session file, preserves current context-budget resolution, and only passes `currentTokenCount` when real live usage is available.

## Reproduction Assessment

Yes, at source level. Current main formats the arrow whenever both counts exist, and the PR body includes copied Telegram before/after output for the small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata violates the compactor contract; omit that field unless a real usage measurement is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass live usage tokens, not context windows:** `src/auto-reply/reply/commands-compact.ts:128-134`
  - body: `observedContextTokens` comes from `params.contextTokens` or `targetSessionEntry.contextTokens`, then gets forwarded as `currentTokenCount`. The compactor treats that field as caller-observed live prompt usage and can persist or return it as `tokensBefore`, so manual `/compact` can still report bogus before counts. Omit this field unless real usage is available.
