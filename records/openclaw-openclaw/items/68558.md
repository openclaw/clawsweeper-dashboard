The PR changes manual `/compact` reply formatting, adds a session-file token estimate fallback, and forwards a token count into compaction.

## Best Possible Solution

Land a corrected version that keeps the label guard and stale-total fallback, removes context-window `currentTokenCount` forwarding unless real usage is available, and preserves current-main context-budget resolution.

## Reproduction Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but forwarding context-window metadata as live prompt usage is not the narrow maintainable fix.
