## Summary

Keep open: current main still has the misleading `/compact` label and stale-total path, and this PR fixes useful user-visible output, but the branch incorrectly forwards a model context-window value as observed compaction token usage.

## What This Changes

The branch modifies `src/auto-reply/reply/commands-compact.ts` to guard `/compact` compaction labels, estimate stale session totals from JSONL file size, and forward a derived token count to the compactor.

## Best Possible Solution

Land a corrected `/compact` patch that keeps the label guard and stale-total fallback, passes `currentTokenCount` only from actual usage snapshots, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current main still has the unconditional label path and no stale-total file estimate, and the PR body includes copied before/after Telegram output for the small-session case.

## Solution Assessment

No, not as written. The UI label and stale-total fixes belong in the command module, but the token forward should be removed unless a real live usage count is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` is derived from `params.contextTokens` or `targetSessionEntry.contextTokens`, which are model context-window limits, but `compactEmbeddedPiSession.currentTokenCount` is live prompt usage and is reused as `tokensBefore` plus checkpoint and after-token diagnostic input. Passing the window size can make `/compact` report bogus before counts and skew compaction diagnostics; omit this field unless a real usage snapshot is available.
  - confidence: 0.94
