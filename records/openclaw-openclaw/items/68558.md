## Summary

Keep open: current main and the latest release still have the misleading `/compact` before/after formatter, but this PR is not merge-ready because it mixes context-window budgets into usage-token state and can overstate context usage with a rough file-size estimate.

## What This Changes

The PR adds a manual `/compact` label guard, passes a derived token count into compaction, and estimates stale context usage from the session JSONL file.

## Best Possible Solution

Port only the formatter-safe guard onto current main, keep context budget and live usage as separate inputs, and leave broader stale/unknown usage estimation to a separately proven session-state fix.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 still render a `tokensBefore → tokensAfter` label whenever both values are non-null, so small-session values like `0` and `36` can produce the reported misleading output.

## Solution Assessment

No, not as submitted. The formatter clamp is the useful part, but the best fix is to port that guard onto current main without treating context-window budgets or rough file-size estimates as canonical usage.

- **[P1] Pass usage tokens, not context windows:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens` or persisted `contextTokens`, which are context-window budget values, but `currentTokenCount` is the live prompt-token usage contract. Passing a 272k/1M window here can make manual `/compact` report or checkpoint a huge `tokensBefore` value for a small session instead of actual session usage.
  - confidence: 0.92
- **[P2] Do not let file size override fresh token totals:** `src/auto-reply/reply/commands-compact.ts:224`
  - body: `Math.max(storedTotal, fileEstimate)` lets a rough JSONL bytes/4 estimate replace a fresh stored token total whenever transcript JSON is large, especially with tool results. Use this only when the stored total is missing or known stale, or keep the canonical stored/session result so the context summary does not overstate usage.
  - confidence: 0.84
