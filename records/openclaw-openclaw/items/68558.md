## Summary

Keep open: this is a useful `/compact` reply bug fix, but the PR is not mergeable as written because it forwards context-window metadata as live `currentTokenCount`, which can make the compactor report or persist bogus `tokensBefore` values. The PR body includes enough copied live Telegram output for real behavior proof; the remaining action is a narrow branch repair.

## What This Changes

The PR changes manual `/compact` reply formatting to suppress non-reducing before/after labels, estimate stale context totals from session-file size, and pass an observed token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real live prompt-usage count is available, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.

## Review Findings

Overall correctness: patch is incorrect
