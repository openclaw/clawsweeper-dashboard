## Summary

Keep this PR open, but do not merge it as-is. The reported `/compact` label bug is real on current main, but the patch forwards the model context window as live prompt-token usage and still needs the required changelog entry.

## What This Changes

The branch changes `src/auto-reply/reply/commands-compact.ts` to adjust manual `/compact` labels, pass a token count into compaction, and estimate stale session totals from the JSONL file size.

## Best Possible Solution

Land a corrected patch that keeps the non-decreasing-label guard and stale-total display fallback, omits `currentTokenCount` unless true live prompt usage is available, adds focused tests, and adds a changelog bullet.

## Reproduction Assessment

Yes. A focused unit test can mock `compactEmbeddedPiSession` to return `tokensBefore <= tokensAfter` and assert the manual `/compact` reply line; the PR body also provides a concrete Telegram stale-total reproduction.

## Solution Assessment

No. The label and stale-total fixes are in the right owner module, but using `contextTokens` for `currentTokenCount` violates the compactor contract; omit that field unless the handler has real observed prompt-token usage.
