## Summary

Keep this PR open because the reported `/compact` output bug is source-reproducible on current main and the branch contains useful formatting/fallback work, but the proposed `currentTokenCount` forwarding is incorrect and should block merge until repaired.

## What This Changes

The PR changes manual `/compact` reply labeling, adds a JSONL file-size fallback for stale session token totals, and forwards derived token metadata into manual compaction.

## Best Possible Solution

Land a corrected `/compact` fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless real prompt usage is available, preserves current context-budget resolution, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still renders the arrow whenever both token counts are present and still builds the context summary from result or stored totals only, while the PR body includes copied Telegram before/after output for the small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but deriving `currentTokenCount` from context-window metadata violates the compactor contract and can create bogus before counts.

## Review Findings
