## Summary

Keep open: the reported `/compact` reply bug is valid and the PR contains useful UI work, but the patch should not merge as written because it passes context-window metadata as live prompt token usage.

## What This Changes

The PR changes manual `/compact` reply formatting to suppress non-reducing before/after labels, estimate stale context totals from JSONL file size, and forward a token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless a real live usage snapshot is available, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram before/after output for the stale-total small-session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compactor:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens` or the session entry's `contextTokens`, which are model/window caps. `compactEmbeddedPiSession.currentTokenCount` is live prompt usage and feeds hook metrics, checkpoint `tokensBefore`, returned output, and tokens-after sanity checks, so this can make manual `/compact` report and persist bogus counts. Omit the field unless manual `/compact` has an actual live usage snapshot.
  - confidence: 0.94
