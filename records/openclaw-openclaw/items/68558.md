## Summary

Keep open; this is a real `/compact` UX bug, but the PR currently forwards a model context-window value as compactor live usage, which can make diagnostics and returned token counts misleading in a different way.

## What This Changes

The branch changes `/compact` reply formatting in `src/auto-reply/reply/commands-compact.ts`, adds a JSONL file-size token fallback, and forwards a derived token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command patch that keeps the label guard and stale-total fallback, but supplies `currentTokenCount` only from a real live usage snapshot.

## Reproduction Assessment

Yes, at source level. Current main still has the unconditional before/after label path, and the PR body includes copied before/after Telegram `/compact` output for the small-session case.

## Solution Assessment

No, not as written. The UI label and stale-total fixes are the right direction, but passing a context-window value as `currentTokenCount` violates the compactor contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to compaction:** `src/auto-reply/reply/commands-compact.ts:169`
