## Summary

This should stay open: the PR targets a real `/compact` reply bug, but the current patch forwards the model context window as observed usage, which can create a new misleading compaction label and bad compaction metadata.

## What This Changes

The PR changes manual `/compact` auto-reply formatting to avoid misleading before/after labels on small sessions and to estimate stale session totals from the JSONL file size.

## Best Possible Solution

Keep the label clamp, but only pass `currentTokenCount` when the manual command has a real observed prompt-token/session-usage value; otherwise omit it and cover the small-session label cases with focused tests.

## Reproduction Assessment

Yes, by source inspection. Current `main` unconditionally renders `before → after` when both compaction token fields exist, and the PR body gives a concrete Telegram small-session example with stale `totalTokens`.

## Solution Assessment

No. The label-rendering direction is reasonable, but forwarding `params.contextTokens` as `currentTokenCount` violates the compactor contract; the safer fix is to omit that field or pass a real usage estimate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass usage tokens, not the context window:** `src/auto-reply/reply/commands-compact.ts:125-132`
  - body: `params.contextTokens` is the model context window, but `currentTokenCount` is later reused as the compactor's observed `tokensBefore` value. On a 1m-token model this patch can turn a normal manual compaction into `Compacted (1.0m → 321)` and persist the inflated before count, so omit this field here or pass an actual observed session usage estimate.
  - confidence: 0.92
