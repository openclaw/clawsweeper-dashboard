## Summary

Keep open: the reported `/compact` reply bug is still present on current `main`, and the useful label/fallback parts of the branch should be preserved, but the current patch violates the compactor token-count contract by forwarding a context-window value as live usage.

## What This Changes

The PR changes `/compact` reply formatting to suppress non-reducing before/after labels, estimate stale totals from the session file, and pass a token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless actual live usage is available, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current `main` still renders the before/after label whenever both token counts are present, and the PR body includes copied Telegram output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` should not be populated from `contextTokens`; omit it unless manual `/compact` has an actual live usage snapshot.
