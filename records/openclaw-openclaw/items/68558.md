## Summary

Keep open: the reported `/compact` reply bug is real and the useful label/fallback direction is clear, but the current branch still passes a context-window limit as `currentTokenCount`, which violates the compactor contract and can create bogus before-token diagnostics/checkpoints.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to guard `/compact` before/after labels, add a JSONL file-size fallback for stale totals, and pass a derived token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command patch that keeps the label guard and stale-total fallback, but supplies `currentTokenCount` only from a real live usage snapshot.

## Reproduction Assessment

Yes, at source level. Current `main` still has the unconditional before/after label path, and the PR body includes copied Telegram `/compact` before/after output for the small-session case.

## Solution Assessment

No, not as written. The UI label and stale-total fixes are the right direction, but passing `contextTokens` as `currentTokenCount` is not the narrowest maintainable fix because it violates the compactorâ€™s live-usage contract.
