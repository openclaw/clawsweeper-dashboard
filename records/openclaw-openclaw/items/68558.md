## Summary

Keep open: the user-facing bug is valid, but the branch still passes context-window capacity as live usage, which would make compaction diagnostics and checkpoints misleading in a different way.

## What This Changes

The PR updates the manual `/compact` reply path to avoid non-reducing compact labels, estimate stale context totals from the session file, and forward a derived token count into compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total display fallback, omits `currentTokenCount` unless a real live usage snapshot is available, and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Current main renders the before/after label whenever both counts exist, and the PR body includes copied Telegram before/after output for the small stale-total session case.

## Solution Assessment

No, not as written. The label guard and stale-total fallback are the right direction, but `currentTokenCount` must not be populated from context-window metadata.
