## Summary

Keep open: the PR addresses a real `/compact` reply bug, but the branch forwards model context-window metadata as live prompt usage, which can create bogus compaction counts.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to suppress non-reducing `/compact` labels, estimate stale totals from the session file, and pass a derived token count into manual compaction.

## Best Possible Solution

Land a corrected compact-command fix that keeps the label guard and stale-total fallback, omits `currentTokenCount` unless actual live usage is available, and adds focused tests.

## Reproduction Assessment

Yes, at source level. Current main still renders the before/after label whenever both counts are present, and the PR body includes copied Telegram output for the small-session case.

## Solution Assessment

No, not as written. The label and stale-total fallback are the right direction, but forwarding `contextTokens` as `currentTokenCount` violates the compactor contract; omit that field unless the command has real live usage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass only live usage tokens to the compactor:** `src/auto-reply/reply/commands-compact.ts:169`

Passing the window size can make manual `/compact` report or store bogus before counts, so omit this field unless the command has an actual usage snapshot.
