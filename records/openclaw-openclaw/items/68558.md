## Summary

Keep this PR open because it contains a useful `/compact` label fix, but the submitted branch is not merge-ready: it passes the model context window as observed current usage and can overwrite fresh token totals with a rough file-size estimate.

## What This Changes

The branch changes `src/auto-reply/reply/commands-compact.ts` to pass a derived context value as `currentTokenCount`, suppress `before → after` labels unless before is greater than after, and estimate context usage from the session JSONL file when stored totals look low.

## Best Possible Solution

Rebase on current main and port only the safe presentation fix plus focused tests, while preserving current-main context-budget resolution and avoiding context-window-as-usage or fresh-total override behavior.

## Reproduction Assessment

Yes, at source level: a compact result with `tokensBefore` less than or equal to `tokensAfter` reaches a formatter that can render the misleading arrow. The PR body also includes copied Telegram before/after output, but I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No, this is not the best fix as submitted. The useful label guard should be ported, but the `currentTokenCount` and JSONL fallback changes need to be removed or narrowed so the command keeps accurate session metrics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass real usage tokens to currentTokenCount:** `src/auto-reply/reply/commands-compact.ts:169`
  - body: `observedContextTokens` comes from `params.contextTokens`/`targetSessionEntry.contextTokens`, which is the model context window, not the current prompt usage. The compactor treats `currentTokenCount` as observed usage and reuses it as `tokensBefore`, so this would make `/compact` report and persist window-sized pre-compaction counts on normal sessions. Pass a real observed usage count or omit this field.
  - confidence: 0.92
- **[P2] Do not override fresh totals with file size estimates:** `src/auto-reply/reply/commands-compact.ts:223-225`
  - body: `fallbackTotal` uses `Math.max(storedTotal, fileEstimate)` whenever both numbers exist, even if `storedTotal` is fresh. Because the JSONL file includes metadata and tool payload overhead, this can replace an accurate small token count with a rough bytes/4 estimate and make the context summary misleading. Use the estimate only when the stored total is absent or explicitly stale.
  - confidence: 0.81
