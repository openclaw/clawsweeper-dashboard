## Best Possible Solution

Key the retry off the structured `PAIRING_REQUIRED` connect detail for a loopback explicit-auth attempt that failed before hello, preserve current `transportValidated` reachability behavior, and add redacted real loopback probe proof.

## Reproduction Assessment

No live reproduction was run in this read-only review. The source path is clear enough to review: current probes can attach cached device identity while explicit auth is present, and GatewayClient can surface structured `PAIRING_REQUIRED` details on pre-hello connect failures.

## Solution Assessment

No, not as currently written. Retrying in `probeGateway` is the right narrow layer for this CLI diagnostic edge case, but the predicate should use structured details and explicit pre-hello state instead of `connectLatencyMs` or derived capability.
