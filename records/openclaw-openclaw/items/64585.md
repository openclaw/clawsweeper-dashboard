## Summary

Keep open. Current main has lock-marker restart recovery and live diagnostic recovery, but it still does not implement the requested startup scanner that selects ordinary persisted chat sessions by transcript tail and replays them through `sessions.send`; channel-backed replay safety also remains an open policy question.

## What This Changes

Add bounded gateway startup recovery that detects stuck persisted chat sessions by transcript tail and replays eligible sessions through the existing `sessions.send` pipeline.

## Best Possible Solution

Define the restart-time replay policy for local versus channel-backed sessions, then add a bounded scanner only for approved routes with focused detection and replay coverage.

## Reproduction Assessment

Yes, at source level: create a persisted `running` main session without `abortedLastRun` and `recoverRestartAbortedMainSessions` will skip it before reading the transcript. Current tests already assert that ordinary running sessions without the marker do not call the gateway.

## Solution Assessment

Unclear. A bounded scanner may be appropriate for some sessions, but the requested `sessions.send` replay for channel-backed conversations needs maintainer policy because related work proposes avoiding hidden channel restart replay.
