## Summary

Keep open. Current main has delivery queue recovery, diagnostic stale-lane release, and lock-marked main-session restart recovery, but it still lacks the requested startup scan of persisted chat session stores by transcript tail and does not replay eligible sessions through `sessions.send`.

## What This Changes

Add bounded gateway startup recovery that scans persisted chat session stores for unanswered transcript tails and replays eligible chat sessions through `sessions.send`.

## Best Possible Solution

Add a `src/gateway` startup recovery module that runs after channel sidecars are ready, scans persisted chat sessions newest-first, skips internal/non-chat sessions, selects safe unanswered transcript-tail candidates, replays through `sessions.send` with idempotency and a cap, and logs a scanned/candidates/replayed/failed summary.

## Reproduction Assessment

Yes. A source-level reproduction is available: create a persisted ordinary `running` session with a user-tail transcript and no `abortedLastRun` marker; current `recoverRestartAbortedMainSessions` skips it, and the existing test at `src/agents/main-session-restart-recovery.test.ts:160` locks in that skip behavior.

## Solution Assessment

Yes. The requested bounded startup scanner using the existing `sessions.send` pipeline is the narrow maintainable direction, provided it keeps the issue's chat-route filters, idempotency, and cap safeguards and coordinates with the channel-backed replay concern in #74687.

## Repair Work Prompt

Implement startup recovery for stuck gateway chat sessions from https://github.com/openclaw/openclaw/issues/64585. Add a bounded module under `src/gateway/` that runs after request context and channel sidecars are ready, scans persisted session stores newest-first, detects safe unanswered chat transcript tails (last meaningful message is user, or assistant has no visible text and no tool call), skips `global`, `unknown`, `cron`, ACP, subagent, and internal/non-chat sessions, caps replay per startup, and logs scanned/candidates/replayed/failed. Replay eligible sessions through the existing `sessions.send` handler/context pipeline with an idempotency key; do not broaden the existing `agent` hidden-resume path or silently replay channel-backed restart-aborted sessions covered by #74687 without an explicit safe filter. Add focused unit tests for detection/selection and a startup wiring test. Add a single-line `CHANGELOG.md` fix entry under the active version unless maintainers classify it as internal-only.
