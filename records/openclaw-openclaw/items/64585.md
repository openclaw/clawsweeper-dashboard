## Summary

Keep open. Current main has adjacent delivery recovery, diagnostic stale-lane release, and lock-marked main-session restart recovery, but it still does not implement this issue's startup scan of persisted chat session stores by transcript tail or replay eligible conversations through `sessions.send`.

## What This Changes

Add gateway startup recovery that scans persisted chat session transcript tails after restart and replays safe unanswered chat sessions.

## Best Possible Solution

Add a bounded startup recovery module only after the channel-backed replay policy is settled: scan chat session stores newest-first, select unanswered or empty assistant-tail candidates, skip internal/non-chat sessions, replay through `sessions.send` after sidecars/context are ready, and cover selection/replay safety with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a persisted ordinary `running` chat session without `abortedLastRun` and it is skipped before transcript inspection; a marked assistant-tail session is rejected as not resumable.

## Solution Assessment

Unclear. A bounded scanner using the existing `sessions.send` pipeline is plausible, but automatic replay for channel-backed sessions overlaps with open safety work in #74687 and should be a maintainer policy decision before coding.
