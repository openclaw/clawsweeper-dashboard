## Summary

Keep open. Current main has outbound delivery recovery, restart-aborted recovery from stale transcript locks, and live diagnostic stuck-session recovery, but it still does not implement the requested startup scanner that selects ordinary persisted sessions by transcript tail and replays them through `sessions.send`; the external channel replay policy is also still unsettled.

## What This Changes

Add gateway startup recovery that scans persisted session stores and replays eligible stuck chat sessions after restart.

## Best Possible Solution

Define the restart-time replay policy for local versus external channel-backed sessions, then add a bounded startup scanner only for approved routes with focused transcript-tail detection and replay coverage.

## Reproduction Assessment

Yes, at source level: create a persisted `running` main session without `abortedLastRun` and `recoverRestartAbortedMainSessions` skips it before transcript inspection. Current tests also assert that ordinary running sessions without the marker do not call the gateway.

## Solution Assessment

Unclear. A bounded scanner may be appropriate for some sessions, but the requested `sessions.send` replay for channel-backed conversations needs maintainer policy because related work argues hidden channel restart replay should be avoided.
