## Summary

Keep open. Current main has adjacent delivery recovery, diagnostic lane release, and lock-marked restart-aborted main-session recovery, but it still lacks the requested startup scan of persisted chat session stores by transcript tail and replay through `sessions.send`.

## What This Changes

Add gateway startup recovery that scans persisted chat session stores for unanswered transcript tails after restart and replays safe candidates through `sessions.send`.

## Best Possible Solution

Add a bounded gateway startup scanner after sidecars/channels are ready, select safe unanswered chat transcript tails newest-first, replay through `sessions.send` with idempotency and summary logging, and keep diagnostic lane release plus lock-marked restart recovery as separate safeguards.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a persisted ordinary `running` session without `abortedLastRun` is skipped by `recoverStore`, and the existing unit test at `src/agents/main-session-restart-recovery.test.ts:160` asserts that behavior.

## Solution Assessment

Yes, with the issue's safety constraints. A focused startup scanner using the existing `sessions.send` pipeline is narrower than broad diagnostic aborts or expanding the hidden `agent` restart-replay path.
