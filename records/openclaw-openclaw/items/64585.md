## Summary

Keep open. Current main has marker-based restart recovery and the newer forced-restart marking fix, but it still does not implement the requested markerless startup scanner that selects ordinary persisted chat sessions by transcript tail and replays them through `sessions.send`; channel-backed replay policy also remains unsettled.

## What This Changes

The issue requests gateway startup recovery that scans persisted chat sessions after restart and replays stuck sessions whose transcript tail indicates an unanswered turn.

## Best Possible Solution

Define the restart-time replay policy for local versus channel-backed sessions, then add a bounded startup scanner only for approved chat routes with focused detection and replay coverage.

## Reproduction Assessment

Yes, at source level: a persisted `running` main session without `abortedLastRun` is skipped before transcript inspection, and the current test suite asserts no gateway call. I did not run a live restart scenario in this read-only review.

## Solution Assessment

Unclear. A bounded scanner may be appropriate, but replaying channel-backed conversations through startup recovery needs an explicit maintainer policy because related work favors avoiding hidden channel replay.
