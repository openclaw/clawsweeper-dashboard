## Summary

Keep open. Current main has adjacent delivery recovery, diagnostic stuck-lane handling, and restart-aborted lock recovery, but it still lacks the requested startup scan of persisted chat sessions by transcript tail and replay through `sessions.send`; related PR #74687 is an adjacent safety change, not an implementation of this request.

## What This Changes

Requests a bounded gateway startup recovery pass that scans persisted chat session transcripts for unanswered tails and replays eligible chat sessions through `sessions.send`.

## Best Possible Solution

Add a bounded gateway startup recovery module that runs after request context and channels are ready, scans persisted chat session stores newest-first, selects safe unanswered transcript-tail candidates, skips internal/non-chat sessions, and replays through `sessions.send` with idempotency and summary logging.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a persisted ordinary `running` session without `abortedLastRun` is skipped, and the existing unit test asserts that no gateway call is made for that case.

## Solution Assessment

Yes, with the issue's safety constraints. A focused startup scanner using the existing `sessions.send` pipeline is the narrow maintainable solution; broad diagnostic abort behavior or hidden synthetic `agent` replay would be riskier.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/64585 by adding bounded gateway startup recovery for stuck ordinary chat sessions. Implement a small module under `src/gateway/` that runs after request context and channel sidecars are ready, scans persisted agent session stores newest-first, selects at most a small per-startup cap of eligible chat sessions, and replays by invoking the existing `sessions.send` handler/context path. Eligibility should cover an unanswered user tail and an assistant tail with no visible text and no tool call, while skipping `global`, `unknown`, cron, ACP, subagent/internal, and non-chat sessions. Preserve or reuse existing session-store/transcript helpers such as `resolveAgentSessionDirs`, `readSessionMessagesAsync`, `extractAssistantVisibleText`, and session-key classifiers. Add idempotency and summary logging for scanned/candidates/replayed/failed. Do not broaden hidden `agent` replay or diagnostic active-run abort behavior, and coordinate with the safety boundary raised by PR #74687 by keeping channel-backed hidden restart-aborted recovery separate from this explicit `sessions.send` startup path. Add focused unit tests for detection, ordering/cap, skip rules, and startup integration. Add a user-facing changelog entry because this is a stability fix.
