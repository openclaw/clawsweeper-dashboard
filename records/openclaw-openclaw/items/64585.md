## Summary

Keep open. Current main still has lock-marker restart recovery and live diagnostic recovery, but it does not implement the requested startup scanner that selects ordinary persisted chat sessions by transcript tail and replays them through `sessions.send`; channel-backed replay policy also remains unsettled.

## What This Changes

Request adds gateway startup recovery that scans persisted chat-session transcript tails after restart and replays selected stuck sessions through `sessions.send`.

## Best Possible Solution

Define the restart-time replay policy for local versus channel-backed sessions, then add a bounded startup scanner only for approved chat routes with focused detection and replay coverage.

## Reproduction Assessment

Yes, at source level. Seed a persisted `running` main session without `abortedLastRun`; current `recoverRestartAbortedMainSessions` skips it before reading the transcript, and the colocated test asserts no gateway call.

## Solution Assessment

Unclear. A bounded scanner may be appropriate for local sessions, but unconditional `sessions.send` replay for channel-backed conversations conflicts with https://github.com/openclaw/openclaw/pull/74687 and needs maintainer policy first.
