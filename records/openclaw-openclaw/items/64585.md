## Summary

Keep open. Current main has adjacent lock-marked main-session restart recovery, restart-continuation delivery recovery, and diagnostic stale-lane release, but it still does not implement the requested startup scan of chat session stores by transcript tail or replay eligible conversations through `sessions.send`. The existing tests explicitly lock in the current gap for ordinary running sessions without `abortedLastRun`.

## What This Changes

The issue requests a gateway startup recovery pass that scans chat session stores for unanswered transcript tails and safely replays eligible chat sessions through `sessions.send` after restart.

## Best Possible Solution

Add a bounded gateway startup recovery module that runs after request context and channel sidecars are ready, scans persisted chat session stores newest-first, selects safe unanswered transcript-tail candidates, skips internal/non-chat sessions, and replays through `sessions.send` with idempotency and summary logging while leaving diagnostic stale-lane release and restart-aborted main-session recovery as separate safeguards.
