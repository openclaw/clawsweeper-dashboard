## Summary

Keep open. Current main and v2026.5.7 still only recover lock-marked main sessions and outbound delivery queues; they do not scan ordinary persisted chat sessions by transcript tail or replay eligible conversations through `sessions.send`.

## What This Changes

The issue requests bounded gateway startup recovery that scans persisted chat sessions by transcript tail and replays eligible stuck conversations after restart.

## Best Possible Solution

Add a bounded startup recovery module only after the channel-backed replay policy is settled, with newest-first selection, internal/non-chat skips, `sessions.send` replay, and focused selection/replay tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a persisted ordinary `running` chat session without `abortedLastRun` is skipped before transcript inspection, and a marked empty-assistant-tail session is failed as not resumable rather than replayed.

## Solution Assessment

Unclear. A bounded scanner through `sessions.send` is plausible, but automatic replay for channel-backed sessions overlaps with open safety work and needs maintainer policy before coding.
