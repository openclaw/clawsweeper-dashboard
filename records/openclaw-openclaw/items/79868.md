## Summary

Keep open. Current `main` still has a source-reproducible silent path: an auto-persisted fallback provider/model can be reused as the first candidate on a later embedded run, so the primary-failure fallback logging path is not entered even though operators see a fallback provider in trajectory output.

## What This Changes

The issue asks OpenClaw to emit gateway fallback telemetry when an embedded run uses a fallback provider instead of the configured primary model.

## Best Possible Solution

Keep auto fallback persistence, but emit explicit telemetry when an embedded run reuses an auto fallback selection that differs from the configured primary.

## Reproduction Assessment

Yes for a source-level path: create an embedded session with `modelOverrideSource: "auto"` pointing at a configured fallback, then dispatch a later turn; current source will start from that fallback and skip the failed-primary log path. The exact Slack/Codex transient outage trigger still needs live capture.

## Solution Assessment

Yes. The narrow maintainable fix is telemetry at the reused auto fallback-selection boundary or equivalent fallback-step emission, while preserving user-selected and legacy model override semantics.
