## Summary

Keep open. Current main improved the no-visible-reply fallback case, but the reported visible-success case can still bypass the `model fallback decision` log when a persisted auto fallback is the effective starting model.

## What This Changes

The issue asks OpenClaw to emit gateway-visible fallback telemetry whenever an embedded run uses a fallback provider instead of the configured primary.

## Best Possible Solution

Add bounded structured telemetry for reused auto fallback selections or effective-provider mismatches, without changing fallback routing, user model override semantics, or the no-visible-reply behavior added by the merged adjacent PR.

## Reproduction Assessment

Yes, at source level. A session already pinned to an auto fallback can produce a visible reply with zero current-run fallback attempts, so the existing failed-candidate and later-candidate-succeeded log branches do not fire.

## Solution Assessment

No, current main is not enough. The merged adjacent PR handles silent fallback output, but the narrow maintainable fix here is explicit sanitized telemetry for reused auto fallback state or an equivalent gateway-visible fallback decision record.

## Repair Work Prompt

Fix the gateway-log observability gap reported in https://github.com/openclaw/openclaw/issues/79868. Current main already handles no-visible-reply fallback via https://github.com/openclaw/openclaw/pull/80917, so do not change fallback routing, model selection, user `/model` override semantics, or user-visible fallback notices. Add a bounded, sanitized structured telemetry path so an embedded auto-reply run that is using an auto-persisted fallback instead of the configured origin model emits a gateway-visible fallback decision or equivalent once at the appropriate runtime boundary, including selected/origin model, active fallback model, run/session identifiers, and a safe reason such as persisted auto fallback or prior fallback state. Avoid raw provider error leakage and avoid noisy duplicate warnings for unchanged state unless the design intentionally emits per-run operator telemetry. Add regression coverage for a persisted `modelOverrideSource: "auto"` session with a visible successful fallback reply and for user-selected overrides not being mislabeled as fallback.
