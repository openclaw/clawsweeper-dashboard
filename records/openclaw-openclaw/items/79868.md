## Summary

Keep open. Current main still has a source-reproducible silent telemetry path: an auto-persisted fallback selection can be reused as the embedded run's starting model, so the normal failed-primary `model fallback decision` path never runs.

## What This Changes

The issue asks for gateway telemetry whenever an embedded run answers from a configured fallback provider instead of the configured primary, especially when an auto fallback selection is reused.

## Best Possible Solution

Keep fallback persistence, but add explicit telemetry for reused auto fallback selections when they bypass the configured primary, preserving strict behavior for user-selected session models.

## Reproduction Assessment

Yes for a source-level path: a session entry with `modelOverrideSource: "auto"` pointing at a configured fallback becomes candidate 1, so a successful embedded run emits no failed-primary or later-candidate `model fallback decision`. The exact Slack/Codex outage trigger still needs live capture.

## Solution Assessment

Yes for the direction, though no patch is proposed here. The narrow maintainable fix is telemetry at auto fallback reuse or equivalent fallback-step emission, not changing candidate ordering or user override semantics.
