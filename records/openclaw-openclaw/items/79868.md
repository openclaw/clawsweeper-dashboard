## Summary

Keep open: the report describes a real observability gap in the embedded model-fallback path, and current source/docs show fallback transitions are supposed to be reconstructable from structured fallback telemetry, but I did not find a merged fix or canonical duplicate.

## What This Changes

The issue requests gateway log telemetry whenever an embedded run uses a fallback model instead of the configured primary model.

## Best Possible Solution

Keep sticky fallback behavior, but emit an explicit structured gateway diagnostic when an auto fallback override or fallback candidate selection makes the active provider differ from the configured primary.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows a plausible current-main path where an auto-persisted fallback override can route later embedded turns to the fallback without the primary-failure `model fallback decision` path running.

## Solution Assessment

Yes, the narrow maintainable fix is to add observability to the existing fallback selection/auto-override path, not to change fallback routing or add a new config knob.
