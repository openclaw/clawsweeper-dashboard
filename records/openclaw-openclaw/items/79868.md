## Summary

Keep open. Current main can reuse an auto-persisted fallback model as the first embedded-run candidate, so a successful Anthropic dispatch can bypass the failed-primary `model fallback decision` path while still matching the reported trajectory/log mismatch.

## What This Changes

The issue asks OpenClaw to log a model fallback decision when an embedded run uses a fallback provider instead of the configured primary.

## Best Possible Solution

Add explicit telemetry for reused auto fallback selections that differ from the configured primary, while preserving fallback persistence and strict user override semantics.

## Reproduction Assessment

Yes, for a source-level path: a session entry with `modelOverrideSource: "auto"` pointing at the configured fallback starts the fallback runner on that provider, and first-candidate success skips the current fallback decision log. The exact live Slack/Codex outage trigger was not reproduced here.

## Solution Assessment

Yes. The narrow maintainable fix is telemetry at the auto fallback-reuse boundary, or equivalent fallback-step emission, rather than changing model selection order or user override policy.
