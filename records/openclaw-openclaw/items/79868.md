## Summary

Keep open. Current main still has a source-level path where an auto-sourced fallback model can be reused as the next embedded run's first candidate, so the primary is bypassed without the failed-candidate fallback log that operators expect.

## What This Changes

The issue asks OpenClaw to emit gateway fallback telemetry whenever an embedded run uses a fallback provider instead of the configured primary.

## Best Possible Solution

Add explicit telemetry for reused auto fallback selections, or an equivalent fallback-step event, while preserving fallback persistence and strict user override semantics.

## Reproduction Assessment

Yes for a source-level path: a session entry with `modelOverrideSource: "auto"` set to the configured fallback is selected as the next run's provider/model, then succeeds as candidate 1 without a failed-primary fallback decision. The exact Slack/Codex outage trigger still needs live capture.

## Solution Assessment

Yes. The narrow maintainable fix is telemetry at the auto fallback reuse boundary or fallback-step emission path, not a broad change to model routing or user override policy.
