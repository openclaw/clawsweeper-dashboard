## Summary

Keep this issue open: current main and the latest release still compile exec approval `argPattern` values directly, and the linked fixing PR is open but unmerged.

## What This Changes

The issue requests rejecting ReDoS-prone exec approval `argPattern` regexes before `matchArgPattern` compiles and evaluates them.

## Best Possible Solution

Keep this issue as the canonical tracker until https://github.com/openclaw/openclaw/pull/82950 or an equivalent narrow guard lands with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a hand-authored `argPattern` such as `(a+)+$` reaches `new RegExp(argPattern).test(argsString)` in `matchArgPattern`; I did not execute the hang case because this was a read-only review.

## Solution Assessment

Yes, a narrow guard that rejects nested repetition before compiling the original pattern is the maintainable fix shape. The issue should remain open until the linked PR or an equivalent current-main fix lands.
