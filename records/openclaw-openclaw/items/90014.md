## Summary

Keep this issue open: current main and the latest release still compile exec approval `argPattern` values directly, and the linked closing PR is open and waiting on review follow-up.

## What This Changes

The issue requests rejecting nested-repetition exec approval `argPattern` regexes before matching so hand-authored allowlist patterns cannot trigger ReDoS.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82950 after its review blockers are resolved, or merge an equivalent narrow guard that rejects nested repetition before compiling the original `argPattern` with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a hand-authored `argPattern` reaches `new RegExp(argPattern).test(argsString)` in `matchArgPattern`; I did not execute a hang case in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reject nested repetition at `matchArgPattern` before compiling the original pattern; the linked PR is the right ownership path once its review blockers are resolved.
