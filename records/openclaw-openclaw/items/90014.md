## Summary

Keep this issue open: current main and v2026.6.8 still compile exec approval `argPattern` values through `new RegExp`, and https://github.com/openclaw/openclaw/pull/82950 is the open clean candidate fix rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to reject ReDoS-prone exec approval `argPattern` regexes before compiling them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82950, or an equivalent narrow guard that rejects nested repetition before compiling the original `argPattern` with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows docs-supported stored `entry.argPattern` values reach `new RegExp(argPattern).test(...)` through `matchAllowlist`; I did not execute a deliberate hang case in this read-only review.

## Solution Assessment

Yes. Guarding with `hasNestedRepetition` immediately before compiling the original `argPattern` is the narrowest maintainable fix, and the linked PR is the right review path.
