## Summary

Keep this issue open: current main and the latest release still compile stored exec approval `argPattern` regexes directly, and the open linked PR is the active fix path.

## What This Changes

The issue asks OpenClaw to reject unsafe nested-repetition exec approval `argPattern` regexes before compilation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82950, or an equivalent narrow guard that rejects nested repetition before compiling the original `argPattern` with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows stored `entry.argPattern` reaches `new RegExp(argPattern).test(...)` through `matchAllowlist`; I did not run a deliberate hang case in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reject unsafe nested repetition at `matchArgPattern` before compiling the original pattern; the linked open PR is the right review path.
