## Summary

Keep open. Current main does not yet render the Fluid bridge result shape, and the PR is aimed at the right existing canvas-preview path, but the branch needs revision before merge because the new safe-ref guard still accepts dot-only refs that URL normalization moves out of the intended document route, and the user-facing Control UI change is missing a changelog entry.

## What This Changes

The PR extends `src/chat/canvas-render.ts` to coerce Fluid bridge `ack`/`executed` regenerate embed payloads into assistant-message canvas previews and adds tool-card extraction tests for accepted and rejected hand-offs.

## Best Possible Solution

Land this after tightening Fluid bridge ref validation to mirror the canvas document-id constraints, adding negative coverage for `.` and `..` refs, and adding one Unreleased changelog bullet credited to the contributor.

## Reproduction Assessment

Yes. A proposed bridge payload with `ref: ".."` and a matching `[embed ref=".." /]` passes the new regex, emits `/__openclaw__/canvas/documents/../index.html`, and URL parsing normalizes that outside `/documents/`.

## Solution Assessment

No. Reusing the existing assistant-message canvas preview path is the right narrow design, but the proposed validation needs a stricter document-id check before this is the best implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject dot-only bridge refs:** `src/chat/canvas-render.ts:17`
  - body: The new safe-ref regex still accepts `.` and `..`. Those values survive `encodeURIComponent`, so `ref: ".."` produces `/__openclaw__/canvas/documents/../index.html`, which the browser URL parser normalizes to `/__openclaw__/canvas/index.html` before the iframe route. Please reject dot-only refs, or mirror the canvas document-id validator, and add a negative test for this case.
  - confidence: 0.9

  - body: This changes user-visible Control UI behavior: Fluid bridge tool output will render as an inline canvas preview instead of raw JSON.
