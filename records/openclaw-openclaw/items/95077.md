Keep open: current main and the latest release still have the local TUI fallthrough, and this PR is the active narrow fix candidate, but the external-contributor proof is still unit-test and grep output rather than a real local TUI run.

## What This Changes

The PR adds `/status` and `/compact` cases to the TUI command switch so local mode shows an unsupported-command system message while gateway mode still forwards the raw command.

## Best Possible Solution

Local TUI should intercept advertised `/status` and `/compact` before embedded model ingress, with proof or coverage showing those commands produce the unsupported local-mode response and no model request.

## Reproduction Assessment

Yes, from source: slash input enters `handleCommand`, `/status` and `/compact` are advertised shared commands, and current main falls through to `sendMessage(raw)` into embedded agent ingress. I did not run the local PTY repro in this read-only review.

## Solution Assessment

Yes for the narrow reported bug: an explicit unsupported local-mode response is one of the issue's acceptable fixes. A broader shared command-boundary repair could cover more unsupported commands, but this PR's remaining blocker is proof and optional regression coverage.
