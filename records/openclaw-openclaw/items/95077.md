## Summary

Keep open: this PR targets the right local TUI fallthrough path, but it is not merge-ready because colon-form shared commands still miss the new cases and the supplied proof is unit/static output rather than real local TUI behavior proof.

## What This Changes

The branch adds `status` and `compact` cases to the TUI command switch so local mode emits an unsupported-command system message and gateway mode still sends the raw command.

## Best Possible Solution

Normalize TUI slash parsing to the shared command contract, block `/status` and `/compact` including colon forms from local model sends, and add focused handler or local PTY proof before closing the linked issue.

## Reproduction Assessment

Yes, source-reproducible: current main routes slash input through `handleCommand`, advertises `/status` and `/compact`, and the command switch falls through to `sendMessage(raw)` into `EmbeddedTuiBackend.sendChat`. I did not run artifact-producing PTY tests in this read-only review.

## Solution Assessment

No as submitted: explicit cases are a plausible narrow fix for plain `/status` and `/compact`, but the shared colon-form command contract still bypasses them and proof is unit/static only.
