## What This Changes

The PR adds bare `/status` and `/compact` cases to TUI command dispatch so local mode emits an unsupported-command system message and gateway mode still forwards the command.

## Best Possible Solution

Normalize local TUI command parsing against the shared slash-command contract, block supported local forms before model delivery, add focused coverage, and require redacted live local TUI proof before merge.

## Reproduction Assessment

Yes, from source: slash input enters `handleCommand`, current main lacks `status` and `compact` cases, and the default `sendMessage(raw)` path reaches `agentCommandFromIngress`. I did not run artifact-producing PTY tests in this read-only pass.

## Solution Assessment

No: explicit cases are a plausible narrow mitigation, but they do not catch colon-form shared command syntax such as `/compact: focus on decisions`. A parser/shared-normalization fix with focused tests is the safer solution.
