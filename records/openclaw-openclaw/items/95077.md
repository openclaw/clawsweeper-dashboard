## Summary

Keep open: the PR is a useful partial mitigation for the local TUI slash-command fallthrough, but it still misses colon-form shared commands and the supplied proof is unit-test/static output rather than a real local TUI run.

## What This Changes

The branch adds `status` and `compact` cases to `src/tui/tui-command-handlers.ts` so local mode prints an unsupported-command system message while non-local mode forwards the raw command.

## Best Possible Solution

Normalize local TUI command parsing against the shared slash-command contract, cover bare and colon-form `/status` and `/compact`, and require redacted real local TUI proof before merge.

## Reproduction Assessment

Yes from source. Slash input routes to `handleCommand`, `parseCommand` splits only on whitespace, unhandled names call `sendMessage(raw)`, and local `sendChat` forwards text into `agentCommandFromIngress`; I did not run artifact-producing PTY tests in this read-only pass.

## Solution Assessment

No. Explicit switch cases are a plausible partial mitigation, but the best fix should reuse or mirror shared normalization so colon-form command text cannot continue into the model path.
