## Summary

Keep open: current main still passes bundled MCP input schemas through without resolving nested local `$ref` or the Notion `parent` `oneOf` shape, so the reported page-creation failure remains a valid source-proven bug.

## What This Changes

The issue requests that OpenClaw's MCP tool bridge resolve local `$ref` and `oneOf` parameter schemas so Notion page creation accepts valid `parent` objects.

## Best Possible Solution

Add generic bundled-MCP schema normalization that resolves local `$defs`/`$ref` and makes nested object unions provider-safe without introducing a Notion-specific path or changing runtime tool arguments.

## Reproduction Assessment

No live high-confidence reproduction was run, but yes, there is a high-confidence source reproduction path: Notion 2.2.1 emits a required `parent` `$ref` to a `oneOf`, and current main exposes that raw schema through bundled MCP tools.

## Solution Assessment

Yes, the best fix direction is a generic bundled-MCP schema normalization step with a Notion-like regression fixture, not a Notion-only workaround.
