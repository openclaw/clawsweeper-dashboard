Keep open: current main still has a clear bundled-MCP schema gap for Notion-style local `$ref` plus `oneOf` parameter schemas, and I did not find a canonical merged/open fix for this specific problem.

## What This Changes

## Best Possible Solution

Add a generic bundled-MCP schema normalization step that resolves local `$defs`/`$ref` and preserves or safely flattens object `oneOf`/`anyOf` shapes, with a Notion-like regression fixture.

## Solution Assessment

Yes. The narrow maintainable solution is generic bundled-MCP/tool-schema normalization rather than a Notion-specific special case, while preserving the original tool-call arguments sent to the MCP server.
