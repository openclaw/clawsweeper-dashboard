## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` remains undefined, so both the default and final resolver calls use global defaults instead of the configured default agent. Configs with a default-agent model override will get explain output that does not match normal models/session behavior.
  - confidence: 0.9
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode the resolver entry is built only from stored session fields, but the payload later reports the positional model and `--provider` values as applied overrides. `models explain <model> --provider <p> --session <key>` can therefore claim overrides were used while resolving the unchanged persisted session.
  - confidence: 0.9
- **[P2] Keep the models CLI on the lazy-loading path:** `src/cli/models-cli.ts:13`
  - body: This top-level import pulls the models command barrel into `src/cli/models-cli.ts`, but current main lazy-loads the models runtime and command implementations inside actions and has removed `src/commands/models.ts`. Rebase the command onto that pattern so help/startup do not eagerly load session/gateway/config code.
  - confidence: 0.88
- **[P3] Document the models explain command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing CLI command and options, but the models CLI docs do not describe `models explain`, `--provider`, `--agent`, `--session`, or `--json`, leaving the supported diagnostic semantics unspecified.
  - confidence: 0.78
