## Summary

Keep open: the diagnostic command is a plausible core CLI feature, but this PR is not mergeable as-is because it can report misleading model resolution, regresses the current lazy models CLI boundary, and still lacks after-fix real CLI proof.

## What This Changes

The PR adds an `openclaw models explain` CLI command with provider, agent, session, JSON/text diagnostics and focused model-resolution tests.

## Best Possible Solution

Land a rebased diagnostic command that preserves lazy command loading, defines session override behavior, documents the CLI surface, and includes focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run no-agent explain under a config whose default agent has its own model override, or run `--session` with positional/provider overrides and compare the reported inputs with the resolver input.

## Solution Assessment

No. The diagnostic belongs in the core models CLI, but this branch should be revised to align with current lazy CLI loading and to settle default-agent/session semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the configured default agent before explaining:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` can remain undefined before calling the session resolver, so the output uses global defaults instead of the configured default agent. A default agent with a per-agent model override will get misleading `Default resolved` and `Final resolved` output compared with runtime session behavior.
  - confidence: 0.9
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode, the resolver input is built only from the stored session entry, but the payload still reports positional `model` and `--provider` values as explicit overrides. `models explain <model> --provider <p> --session <key>` can therefore claim an override was applied while resolving the unchanged persisted session entry.
  - confidence: 0.9
- **[P2] Preserve lazy models command loading:** `src/cli/models-cli.ts:13`
  - body: Current main lazy-loads models command implementations inside each action, but this patch imports `modelsExplainCommand` from the old top-level command barrel. Rebase the new command onto the existing lazy pattern so registering `openclaw models` does not eagerly load session/gateway/config code.
  - confidence: 0.86
- **[P3] Document the new models explain command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing CLI command and options, but the models CLI docs do not describe `models explain`, `--provider`, `--agent`, `--session`, or `--json`, leaving the supported diagnostic semantics unspecified.
  - confidence: 0.78
