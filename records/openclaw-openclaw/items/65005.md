## Summary

Keep open. Current main still does not ship `openclaw models explain`, but the latest PR diff has correctness issues in default-agent and persisted-session resolution and needs a rebase onto the current lazy-loaded models CLI structure before it can merge.

## What This Changes

The PR adds an `openclaw models explain` CLI command with text/JSON diagnostics, provider/agent/session inputs, command wiring, and focused resolution tests.

## Best Possible Solution

Keep this as a core CLI diagnostic only if maintainers want the surface, then rebase it onto the lazy models CLI, align it with default-agent/session semantics, and add docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction exists with a config whose configured default agent has its own model override and `models explain` is run without `--agent`; a second reproduction is combining `--session` with positional/provider overrides and observing that the payload reports overrides that were not used for resolution.

## Solution Assessment

No. The command direction is reasonable, but this patch is not the best current implementation until it matches runtime default-agent/session resolution and the current CLI module boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63`
  - body: When `--agent` is omitted, `agentId` can stay undefined, so `resolveSessionModelRef` uses global defaults instead of the configured default agent. That makes `Default resolved` and `Final resolved` disagree with real bare/main session routing when the default agent has its own model override.
  - confidence: 0.88
- **[P2] Apply or reject CLI overrides in session explain mode:** `src/commands/models/explain.ts:50-54`
  - body: In the `--session` branch, `entry` is built only from persisted session fields, but the payload later reports `opts.provider` and positional `model` as applied. `models explain <model> --provider <p> --session <key>` can therefore show overrides that were not used to compute `resolved`.
  - confidence: 0.9
- **[P3] Document the new user-facing command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a new user-facing `openclaw models explain` command, but the patch does not update the models CLI docs or `CHANGELOG.md`. The release notes and CLI reference should cover the new command and flags before merge.
  - confidence: 0.78
