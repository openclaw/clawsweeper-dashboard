## Summary

Keep open. Current main does not ship or document `openclaw models explain`, so the PR is not obsolete, but the latest patch still has correctness issues in default-agent and persisted-session resolution and is stale against the current lazy-loaded models CLI structure.

## What This Changes

The PR adds an `openclaw models explain` command with text/JSON diagnostics, optional provider/agent/session inputs, command wiring, and focused tests for provider/model resolution output.

## Best Possible Solution

Keep the command only if maintainers want this core CLI diagnostic, but rebase it onto the current lazy-loaded models CLI, resolve the configured default agent before no-agent explanations, define and enforce `--session` override semantics, then add docs, changelog, and regression coverage for default-agent and persisted-session cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: use a config whose configured default agent has its own model override and run explain without `--agent`, or combine `--session` with positional/provider overrides and observe that the PR reports overrides that were not used for resolution.

## Solution Assessment

No. The command is a reasonable diagnostic surface, but this patch is not the best implementation until it matches current session/default-agent semantics and the current lazy models CLI architecture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:62`
  - body: When `--agent` is omitted, `agentId` can stay undefined, so `resolveSessionModelRef` uses global defaults instead of the configured default agent. That makes `Default resolved` and `Final resolved` disagree with real bare/main session routing when the default agent has its own model.
  - confidence: 0.88
- **[P2] Apply or reject CLI overrides in session explain mode:** `src/commands/models/explain.ts:50-54`
  - body: In the `--session` branch, `entry` is built only from persisted session fields, but the payload later reports `opts.provider` and positional `model` as applied. `models explain <model> --provider <p> --session <key>` can therefore show overrides that were not used to compute `resolved`.
  - confidence: 0.9
- **[P3] Document the new user-facing command:** `src/cli/models-cli.ts:114-120`
  - body: This adds a new user-facing `openclaw models explain` command, but the patch does not update the models CLI docs or changelog. Users would not have a discoverable reference for the new flags, and the user-facing feature should be represented in release notes.
  - confidence: 0.78
