## What This Changes

Adds an `openclaw models explain` command with provider, agent, session, JSON/text diagnostics and focused model-resolution tests.

## Best Possible Solution

Rebase the command onto the lazy models CLI runtime, define and enforce default-agent and session override semantics, document the CLI surface, and require redacted real CLI proof before merge.

Yes. ## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep models explain on the lazy command path:** `src/cli/models-cli.ts:13`
  - body: Current main imports models command implementations only inside action handlers and has a test that `models --help` does not load `models-cli.runtime.js`. This top-level command import would eager-load the models command/session/config path during help/startup, regressing that boundary.
  - confidence: 0.9
- **[P2] Resolve the default agent before explaining models:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` can stay undefined, so defaults/final resolution use global configured defaults instead of the configured default agent used by existing models status/session behavior. Configs with a default-agent model override can get misleading explain output.
  - confidence: 0.88
- **[P2] Reject or apply session-mode CLI overrides:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode the resolver input is built from the stored session entry, but the payload can still report positional model and `--provider` values as applied overrides. Either reject those flags with `--session` or include them in the resolver input so the explanation stays internally consistent.
  - confidence: 0.88
