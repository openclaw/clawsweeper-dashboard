## Summary

Keep this PR open: the feature is not implemented on current main, but the branch still has source-reproducible correctness issues in default-agent and session override handling, plus no real CLI behavior proof from the contributor.

## What This Changes

The PR adds an `openclaw models explain` CLI command with provider, agent, session, JSON, and text diagnostics for model/provider resolution.

## Best Possible Solution

Refresh the command onto current main's lazy models CLI structure, align defaults with runtime/default-agent resolution, explicitly define session override behavior, document the surface, and add focused tests plus real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run without `--agent` under a config whose default agent has a model override, or run `--session` together with positional/provider overrides and compare the reported inputs to the resolver input.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best mergeable solution until it follows current main's CLI boundary and resolves the default-agent and session override semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` stays undefined, so the defaults and final resolution use global configured defaults instead of the configured default agent. A default agent with its own model override will therefore get misleading explain output.
  - confidence: 0.88
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode, the resolver input is built only from persisted session fields, but the payload reports positional model and `--provider` values as applied. That can claim an override was used while resolving the unchanged session entry.
  - confidence: 0.9
- **[P3] Document the new models explain command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing `openclaw models explain` command, but the models CLI docs do not describe the command or its `--provider`, `--agent`, `--session`, and `--json` semantics.
  - confidence: 0.78
