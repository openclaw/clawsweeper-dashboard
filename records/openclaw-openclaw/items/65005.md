## Summary

Keep open: current main still lacks `openclaw models explain`, but this PR is not mergeable as-is because it conflicts with the current lazy models CLI structure, has source-reproducible diagnostic inaccuracies, and still lacks contributor real behavior proof.

## What This Changes

This PR adds an `openclaw models explain` CLI command with provider, agent, session, JSON/text diagnostics and focused model-resolution tests.

## Best Possible Solution

Land a rebased diagnostic command that preserves lazy command loading, resolves default-agent behavior consistently, defines session override semantics, documents the CLI surface, and includes focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths for the review findings: no-agent explain leaves `agentId` undefined while current models status/session behavior resolves a default agent, and `--session` mode reports CLI overrides that are not part of the resolver input.

## Solution Assessment

No. The diagnostic command is useful, but this branch is not the best mergeable form until it is rebased onto the current lazy CLI layout and the default-agent/session semantics are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` remains undefined, so both default and final resolution use global defaults instead of the configured default agent. Configs with a default-agent model override will get explain output that does not match normal models/session behavior.
  - confidence: 0.9
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode the resolver entry is built only from stored session fields, but the payload later reports positional model and `--provider` values as applied overrides. Either reject those flags with `--session` or include them in the resolver input so the explanation is internally consistent.
  - confidence: 0.9
- **[P2] Keep the models CLI on the lazy-loading path:** `src/cli/models-cli.ts:13`
  - body: This top-level import pulls the models command barrel into `src/cli/models-cli.ts`, but current main lazy-loads the models runtime and command implementations inside actions and has removed `src/commands/models.ts`. Rebase the command onto that pattern so help/startup do not eagerly load session, gateway, and config code.
  - confidence: 0.88
- **[P3] Document the models explain command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing CLI command and options, but the models CLI docs do not describe `models explain`, `--provider`, `--agent`, `--session`, or `--json`, leaving the supported diagnostic semantics unspecified.
  - confidence: 0.78
