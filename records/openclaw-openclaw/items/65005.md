- **[P2] Keep the models CLI on the lazy path:** `src/cli/models-cli.ts:13`
  - body: Current main imports `models-cli.runtime.js` only inside command actions and tests that `models --help` does not load it. This top-level command barrel import would eager-load models command/session/config code and conflicts with the current removal of `src/commands/models.ts`.

  - body: When `--agent` is omitted, `agentId` can stay undefined, so defaults and final resolution use global configured defaults instead of the configured default agent used by existing models status/session behavior. Configs with a default-agent model override can get misleading explain output.
  - confidence: 0.88

  - body: In `--session` mode the resolver input is built from the stored session entry, but the payload can report positional model and `--provider` values as applied overrides. Either reject those flags with `--session` or include them in the resolver input so the explanation stays internally consistent.
  - confidence: 0.88
