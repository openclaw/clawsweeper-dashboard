## Summary

Keep open: current main does not have this diagnostic command, but the PR is not mergeable as-is because it conflicts with the current lazy models CLI boundary, has unresolved default-agent/session override semantics, and lacks required real CLI proof.

## What This Changes

This PR adds an `openclaw models explain` CLI command with provider, agent, session, JSON/text diagnostics and focused model-resolution tests.

## Best Possible Solution

Rebase the command onto the current lazy models CLI runtime, align omitted-agent and session override semantics with existing resolution behavior, document the CLI surface, and add redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths for the review findings: the PR imports through an eager/removed command barrel, leaves omitted-agent resolution out of the default-agent path, and reports CLI overrides in session mode without using them in the resolver input.

## Solution Assessment

No. The diagnostic command is useful, but this branch is not the best mergeable form until it preserves current lazy loading and the default-agent/session semantics are defined.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep the models CLI on the lazy path:** `src/cli/models-cli.ts:13`
  - body: Current main imports `models-cli.runtime.js` only inside command actions and tests that `models --help` does not load it. This top-level command import would eager-load models command/session/config code and conflicts with the current removal of the `src/commands/models.ts` barrel.
  - confidence: 0.9
- **[P2] Resolve the default agent before explaining models:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` can stay undefined, so defaults/final resolution use global configured defaults instead of the configured default agent used by the existing models status/session behavior. Configs with a default-agent model override can get misleading explain output.
  - confidence: 0.88
- **[P2] Reject or apply overrides in session explain mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode the resolver input is built from the stored session entry, but the payload can report positional model and `--provider` values as applied overrides. Either reject those flags with `--session` or include them in the resolver input so the explanation stays internally consistent.
  - confidence: 0.88
- **[P3] Document the new models explain surface:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing CLI command and options, but the models CLI docs do not describe `models explain`, `--provider`, `--agent`, `--session`, or `--json`, leaving the supported diagnostic semantics unspecified.
  - confidence: 0.78
