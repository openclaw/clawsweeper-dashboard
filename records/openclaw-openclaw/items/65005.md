## What This Changes

The PR adds `openclaw models explain` with provider, agent, session, JSON/text diagnostics and tests for model/provider resolution.

## Best Possible Solution

Land a rebased diagnostic command that preserves current lazy CLI boundaries, resolves the default agent consistently, defines session override behavior, documents the surface, and includes focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run no-agent explain under a config whose default agent has its own model override, or run `--session` with positional/provider overrides and compare the reported inputs to the resolver input.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best mergeable solution until it follows current main's CLI structure and settles default-agent plus session override semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63-64`
  - body: When `--agent` is omitted, `agentId` remains undefined, so defaults and final resolution use global model defaults instead of the configured default agent. A default agent with its own model override will get misleading `Default resolved` and `Final resolved` output.
  - confidence: 0.9
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-55`
  - body: In `--session` mode, the resolver entry is built only from the stored session fields, but the payload later reports positional `model` and `--provider` values as applied. That can claim an override was used while resolving the unchanged session entry.
  - confidence: 0.9
- **[P2] Preserve lazy models command loading:** `src/cli/models-cli.ts:13`
  - body: Current main lazy-loads each models command implementation from the action, but this branch adds `modelsExplainCommand` to the old top-level barrel import. Rebase this onto the current lazy layout so registering `openclaw models` does not eagerly load the new session/gateway/config code or revive the removed barrel.
  - confidence: 0.82
- **[P3] Document the new models explain surface:** `src/cli/models-cli.ts:118-123`
  - body: This adds a user-facing CLI command and options, but the models CLI docs do not describe `models explain`, `--provider`, `--agent`, `--session`, or `--json`, leaving users without the supported semantics for the new diagnostic.
  - confidence: 0.78
