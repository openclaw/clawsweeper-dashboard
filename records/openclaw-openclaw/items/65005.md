## Summary

Keep open: current main does not implement the requested diagnostic command, and the PR still needs correctness fixes plus contributor real behavior proof before merge.

## What This Changes

The PR adds an `openclaw models explain` CLI command with text/JSON diagnostics for provider/model resolution, including optional provider, agent, and session inputs.

## Best Possible Solution

Add the command only after maintainer agreement on the CLI UX, using the current lazy-import models CLI pattern, resolving the configured default agent consistently, and either rejecting or actually applying explicit overrides in session explain mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: use a config where the configured default agent has its own model override and run the proposed command without `--agent`, or combine `--session` with positional/provider overrides and compare reported inputs to the resolver input.

## Solution Assessment

No. The command direction is useful, but this patch is not the best current implementation until it matches runtime default-agent/session semantics and current CLI module boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:63`
  - body: When `--agent` is omitted, `agentId` stays undefined, so `resolveSessionModelRef` uses global defaults rather than the configured default agent. Configs with a default-agent model override will get misleading `Default resolved` and `Final resolved` output.
  - confidence: 0.88
- **[P2] Apply or reject CLI overrides in session mode:** `src/commands/models/explain.ts:50-54`
  - body: In the `--session` branch, the resolver input is built only from persisted session fields, but the payload later reports positional model and `--provider` values as applied. That can claim overrides were used while `resolved` came from the unchanged session entry.
  - confidence: 0.9
- **[P3] Document the new models explain command:** `src/cli/models-cli.ts:118-123`
  - body: This adds a new user-facing `openclaw models explain` command, but the models CLI docs do not describe the command or its `--provider`, `--agent`, `--session`, and `--json` semantics.
  - confidence: 0.78
