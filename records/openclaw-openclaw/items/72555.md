## Summary

Keep open. Current main still accepts identical `oldText`/`newText` edit replacements and delegates them to the wrapped edit tool; loop detection remains opt-in/default-off, and the related open cross-tool cascade work does not close this narrow edit-safety gap.

## What This Changes

Request to reject no-op edit replacements before execution and add clearer repeated tool-error circuit-breaker behavior.

## Best Possible Solution

Add canonical edit validation that rejects identical replacements before the upstream edit tool runs, and leave broader cross-tool retry policy to #75923/#75924.

## Reproduction Assessment

Yes, source-level. Wrap a dummy edit tool with `REQUIRED_PARAM_GROUPS.edit` and call it with `edits: [{ oldText: "same", newText: "same" }]`; current validation accepts the payload and reaches the base tool.

## Solution Assessment

Yes for the narrow repair. Rejecting no-op edit replacements in OpenClaw's edit parameter validation is the smallest maintainable fix; the broader circuit-breaker behavior should stay with #75923/#75924.
