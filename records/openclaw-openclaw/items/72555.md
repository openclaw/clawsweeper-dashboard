## Summary

Keep open. Current main still lacks the narrow identical `oldText`/`newText` edit pre-validation, and the related open loop-detection work only covers an adjacent cross-tool error-cascade guard.

## What This Changes

The issue asks OpenClaw to reject no-op edit replacements before execution and add tool-error retry/circuit-breaker behavior for repeated non-retryable failures.

## Best Possible Solution

Add OpenClaw-owned validation that rejects identical edit replacements with regression coverage, while leaving broader retry/circuit-breaker policy to #75923/#75924.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: wrap a dummy edit tool with `REQUIRED_PARAM_GROUPS.edit` and call it with `edits: [{ oldText: "same", newText: "same" }]`; current validation accepts the payload and reaches the base tool.

## Solution Assessment

Yes for the narrow edit-safety slice: rejecting no-op edit replacements in OpenClaw's edit parameter validation is the smallest maintainable repair. The broader circuit-breaker policy should stay with the existing loop-detection cluster rather than be folded into this fix.

## Repair Work Prompt

Fix the edit safety pre-validation slice of https://github.com/openclaw/openclaw/issues/72555 only. Add OpenClaw-owned validation that rejects any edit replacement where `oldText` and `newText` are identical before the wrapped `createEditTool` executes, with a clear non-retryable validation message such as `oldText and newText are identical. No change will be made.` Preserve deletion edits where `newText` is an empty string and preserve existing structured-payload rejection behavior. Do not implement broad per-session retry budgets, default loop-detection changes, provider prompt changes, or new circuit-breaker config; #75923/#75924 track adjacent loop-detection policy. Add regression coverage for the validator and at least one composed edit-tool path, add a user-facing CHANGELOG entry, and keep the change scoped to agent edit validation/tests/docs needed for that behavior.
