## Summary

Keep open: current main and the latest release still use Discord's process-local pending-history map, so a fresh Discord guild-channel session after reset/restart cannot recover prior same-channel context. The issue remains valid but needs maintainer product/privacy direction before an implementation path is safe.

## What This Changes

The issue requests Discord same-channel session recovery so a fresh session after idle/reset/restart can inject bounded recent channel history for ambiguous follow-up messages.

## Best Possible Solution

Define privacy-aware same-channel recovery for fresh Discord channel sessions, preferably with a durable bounded source, fresh-session-only injection, clear diagnostics, and Discord guild-channel proof.

## Reproduction Assessment

Yes, at source level. Current main creates Discord channel history from a provider-local map and builds prompt context only from that map, so a fresh provider/gateway session cannot recover prior same-channel history from durable state.

## Solution Assessment

Unclear as a final product path. Bounded recovery is plausible, but maintainers need to choose the privacy default, context source, config shape, and diagnostics before implementation.
