## Summary

Keep open: current main and shipped v2026.6.8 still only provide process-local pending Discord channel history, so same-channel reset recovery remains missing and needs maintainer product/privacy direction before implementation.

## What This Changes

Discord channel sessions are reported to lose recent same-channel conversational context after reset or restart, causing ambiguous follow-ups to resolve against only the newest event.

## Best Possible Solution

Define privacy-aware same-channel recovery for fresh Discord channel sessions, preferably reusing existing transcript or history primitives before adding config, with diagnostics and regression plus live Discord proof.

## Reproduction Assessment

Yes, at source level. Current main creates Discord prompt history from a provider-local pending map, so a fresh provider/gateway process or reset cannot recover prior same-channel context from that buffer.

## Solution Assessment

Unclear as a final product path. Bounded recovery is plausible, but maintainers need to choose whether this becomes a default, an opt-in setting, or a transcript/plugin-backed recovery path.
