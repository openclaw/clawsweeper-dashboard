## Summary

Keep open: current main still has only process-local pending Discord guild history, and the docs define group history as pending-only context that excludes messages already in the session transcript, so reset/restart same-channel recovery remains missing and product/privacy-sensitive.

## What This Changes

Add Discord guild/channel session recovery that injects bounded recent same-channel context after idle, manual, or gateway reset.

## Best Possible Solution

Maintainers should decide the privacy/default policy, then implement a bounded same-channel recovery path at the Discord/channel session boundary with diagnostics and regression coverage.

## Reproduction Assessment

Yes. The issue has live repro details, and current source shows Discord guild history is only a process-local pending/skipped buffer, so a fresh provider/session path can lack prior same-channel transcript context.

## Solution Assessment

Unclear. Bounded same-channel recovery is the likely direction, but default versus opt-in behavior and the durable source of recovered context need maintainer privacy/product judgment.
