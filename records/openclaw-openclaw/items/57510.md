Adds a blocking `pre_route` plugin hook before reply dispatch, route/session/account/agent overrides, timeout fail-open handling, docs/changelog, and dispatch/hook regression tests.

## Best Possible Solution

Rebase the branch on current main, define `pre_route` in `src/plugins/hook-types.ts`, thread route overrides through the current dispatch/ACP/reply path without losing `claimInboundDedupe`, update canonical docs and changelog credit, then rerun focused dispatch/hook tests plus the changed gate.

## Reproduction Assessment

Yes. Register a `pre_route` handler returning `handled: true` with a route override, send an inbound message, and assert same-turn routing plus duplicate and timeout behavior; current main has no such hook, so the requested behavior cannot occur there.

## Solution Assessment

No as-is. A blocking pre-dispatch hook is a plausible narrow core seam, but this patch must be rebased and aligned with current hook contracts, reply option types, and dedupe claims before it is the maintainable solution.
