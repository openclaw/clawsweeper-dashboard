## Summary

Keep open: current main still has no `pre_route` hook, and this PR is a real but stale public plugin API change with blocking rebase/API issues plus missing real behavior proof.

## What This Changes

The PR adds a blocking `pre_route` plugin hook before reply dispatch, route/session/account/agent overrides, timeout fail-open behavior, docs, changelog text, and dispatch/hook regression tests.

## Best Possible Solution

Define `pre_route` in the current hook contract, preserve in-flight dedupe, thread route overrides through dispatch, ACP, and reply execution consistently, document trusted-plugin semantics, and require focused tests plus redacted live proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR adds a new public plugin routing capability. Source inspection confirms current main has no `pre_route` hook, so the requested behavior cannot occur through an existing contract.

## Solution Assessment

No as-is. A blocking pre-routing hook may be a reasonable core seam, but this patch needs rebase work, ACP/reply consistency fixes, maintainer API review, and real behavior proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move pre_route into hook-types:** `src/plugins/types.ts:1806`
  - body: Current main's active hook-name list and handler map live in `src/plugins/hook-types.ts`, while `src/plugins/types.ts` re-exports that contract. Adding `pre_route` here leaves registry validation and hook lookup unaware of the new hook after rebase.
  - confidence: 0.9
- **[P2] Add agentIdOverride to the canonical options type:** `src/auto-reply/types.ts:78`
  - body: Current main imports `GetReplyOptions` from `src/auto-reply/get-reply-options.types.ts`; `src/auto-reply/types.ts` is only a re-export barrel. Adding `agentIdOverride` here does not update the option type used by `getReplyFromConfig` or dispatch call sites after rebase.
  - confidence: 0.88
- **[P2] Preserve the in-flight dedupe claim:** `src/auto-reply/reply/dispatch-from-config.ts:221`
  - body: Current dispatch uses `claimInboundDedupe` so concurrent duplicates are marked `inflight` until the turn commits or releases. This older `shouldSkipDuplicateInbound` guard runs before `pre_route` without taking that claim, so concurrent duplicate messages can both execute `pre_route` and dispatch side effects.
  - confidence: 0.86
- **[P2] Thread agent overrides through ACP dispatch:** `src/auto-reply/reply/dispatch-from-config.ts:743-749`
  - body: `pre_route` can set `routeOverride.agentId`, but ACP dispatch is attempted before the reply resolver receives `agentIdOverride` and the ACP call gets only `sessionKey`. ACP-bound turns whose session key does not encode the override agent can still resolve and run on the wrong agent path.
  - confidence: 0.82
