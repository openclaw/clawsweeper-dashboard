## Summary

Keep open: the routing hook is a plausible feature and the discussion shows user interest, but this branch is stale/conflicting against current main and still has correctness, API-review, security-boundary, and real-proof blockers.

## What This Changes

The PR adds a blocking `pre_route` plugin hook before reply dispatch with route/session/account/agent overrides, timeout fail-open behavior, docs, changelog text, and dispatch/hook regression tests.

## Best Possible Solution

Define the pre-routing contract in the current hook surface, preserve in-flight dedupe, thread overrides consistently through dispatch, ACP, and reply execution, document trusted-plugin semantics, and land only after focused tests plus redacted live proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR proposes a new public plugin routing capability. Source inspection confirms current main has no `pre_route` hook, so the requested behavior cannot occur through an existing contract.

## Solution Assessment

No as-is. A blocking pre-routing hook may be reasonable, but this patch must be rebased to current hook contracts, preserve in-flight dedupe, handle ACP agent overrides consistently, and receive API/security review plus real proof.

  - body: Current main's active hook-name list and handler map live in `src/plugins/hook-types.ts`, while `src/plugins/types.ts` re-exports that contract. Adding `pre_route` here leaves registry validation and hook lookup unaware of the new hook after rebase.

  - confidence: 0.88

  - body: This early `shouldSkipDuplicateInbound` check does not take the current `claimInboundDedupe` in-flight claim. Concurrent duplicate inbound messages can both execute `pre_route` and its side effects before either turn commits.
  - confidence: 0.9

  - body: `pre_route` can set `routeOverride.agentId`, but ACP dispatch runs before the normal reply resolver and receives only `sessionKey`. ACP-bound turns whose session key does not encode the override agent can still resolve or run on the wrong path.
  - confidence: 0.82
