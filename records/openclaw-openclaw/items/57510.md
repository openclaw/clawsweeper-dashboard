## What This Changes

Adds a blocking `pre_route` plugin hook before reply dispatch, docs/changelog text, route override plumbing, agent override handling, timeout behavior, and hook/dispatch tests.

## Best Possible Solution

Define `pre_route` in the current `hook-types` contract, preserve current dedupe and ACP/reply routing semantics, document trusted-plugin scope, then land with focused tests and redacted live Gateway/plugin proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR proposes a new public plugin routing capability. Source inspection on current main shows no `pre_route` hook contract, so this is not a failing existing API.

## Solution Assessment

No as-is. A blocking pre-routing hook may be viable, but this branch is stale against current hook/reply option contracts, misses in-flight dedupe and ACP routing handling, and needs API/security review plus live proof.
