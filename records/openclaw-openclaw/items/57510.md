## What This Changes

The branch adds a blocking `pre_route` plugin hook before reply dispatch, route/session/account/agent overrides, timeout fail-open handling, docs/changelog entries, and dispatch/hook regression tests.

## Best Possible Solution

Define `pre_route` in the current `src/plugins/hook-types.ts` contract, thread route overrides through dispatch, ACP, and reply execution while preserving `claimInboundDedupe`, document the trusted-plugin semantics, and verify with focused tests plus redacted live proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR adds a new public plugin routing capability. Source inspection confirms current main has no `pre_route` hook, so the requested behavior cannot occur through an existing contract.

## Solution Assessment

No as-is. A blocking pre-routing hook may be a reasonable core seam, but this patch must be rebased onto the current hook/options/dedupe contracts and needs maintainer API review before it is the maintainable solution.
