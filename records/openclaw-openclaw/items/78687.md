## What This Changes

The PR adds optional `sessionKey` support to wake protocol and `openclaw system event`, threads it through CronService and gateway cron routing, updates Swift/docs/changelog, and adds regression tests plus small extension test typing cleanups.

## Best Possible Solution

Land the external wake/sessionKey surface after aligning the docs with the implemented unknown-agent fallback behavior, and coordinate final issue closure with the companion internal routing PR at https://github.com/openclaw/openclaw/pull/80214.

## Reproduction Assessment

Yes. Source inspection on current main shows the wake schema lacks a typed `sessionKey` and the CLI sends only `{ mode, text }`; the PR body also supplies after-fix live gateway logs for targeted default and non-default agent routing.

## Solution Assessment

Mostly yes. The additive protocol field, CLI flag, CronService threading, and gateway adapter target derivation are the narrow maintainable direction, but the fallback wording must match the code/tests before merge.
