## Summary

Keep open. The PR addresses a real current-main wake/session-targeting gap with sufficient live proof and green CI, but it is an active external implementation touching cron wake routing and has one small docs mismatch plus a recent human request for extra maintainer scrutiny before merge.

## What This Changes

The PR adds optional `sessionKey` support to the wake protocol, `openclaw system event --session-key`, cron wake/enqueue routing, Swift protocol models, docs, changelog, and targeted tests.

## Best Possible Solution

Land the additive wake/session targeting surface after correcting the fallback documentation and getting maintainer approval for the cron routing semantics, while keeping the cron-run remap work tracked in https://github.com/openclaw/openclaw/pull/50818.

## Reproduction Assessment

Yes, source-reproducible: current main's schema, gateway handler, CLI, and cron wake path only carry `mode` and `text`, so external callers cannot provide a typed target session. The PR body also supplies after-fix live gateway output for targeted system events.

## Solution Assessment

Mostly yes: an additive typed field plus CLI flag threaded through existing cron/session-key routing is the narrow maintainable direction for this slice. The docs need to match the implemented unknown-agent fallback before merge, and maintainers should review the broader cron blast radius.
