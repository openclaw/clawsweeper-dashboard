Keep open. Current main still lacks the typed wake `sessionKey` and CLI targeting surface, and this active PR supplies a useful implementation with real gateway proof; I found one non-blocking docs/runtime mismatch that should be cleaned up before maintainer merge review finishes.

The PR adds optional wake `sessionKey` support, threads it through CronService and gateway cron routing, adds `openclaw system event --session-key`, updates Swift protocol models/docs/changelog, and adds focused tests.

## Best Possible Solution

Land the additive wake protocol and CLI targeting surface after maintainer/CI review, align the fallback documentation with the implemented unknown-agent behavior, and keep the internal cron-run remap tracked in https://github.com/openclaw/openclaw/pull/80214.

Yes for source-level reproduction: current main has no typed wake `sessionKey`, no CLI flag, and `wake()` enqueues without a session target. The PR body also supplies after-fix live gateway output for targeted system-event routing.
