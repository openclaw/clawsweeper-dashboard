## Summary

Keep open for normal maintainer merge handling. The patch addresses a real Chat Completions tool-protocol gap, has sufficient live behavior proof, and I did not find a blocking correctness or security issue.

## What This Changes

The branch updates the gateway Chat Completions adapter, docs, changelog, and tests to accept client function tools/tool_choice, return tool_calls, support role: tool follow-ups, and reject unsupported forced modes.

## Best Possible Solution

Land the focused gateway contract once normal CI/merge gates are green, preserving forced tool_choice rejection until the agent runtime can enforce it correctly.

## Reproduction Assessment

Yes, source-level. Current main's /v1/chat/completions adapter does not parse tools/tool_choice or forward clientTools, and it always emits stop-shaped Chat responses instead of tool_calls.

## Solution Assessment

Yes. Reusing the existing clientTools and pendingToolCalls gateway seam is the narrow maintainable path, and explicitly rejecting forced tool_choice modes is safer than claiming enforcement that does not exist yet.
