This PR updates the Gateway OpenAI-compatible Chat Completions adapter, tests, docs, and changelog to support function tools, tool_choice parsing, tool_calls responses, streaming delta.tool_calls, and role:tool follow-up turns while rejecting unsupported forced modes.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main has no Chat Completions tools/tool_choice request path, no clientTools forwarding, and stop-shaped responses where OpenAI-compatible tool-call clients need tool_calls output.

## Solution Assessment

Yes. Reusing the existing clientTools and pendingToolCalls seams is the narrow maintainable path, and rejecting forced tool_choice modes is safer than silently pretending runtime enforcement exists.
