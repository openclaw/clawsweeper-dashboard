## Summary

Keep open. Current main still lacks the Chat Completions tool protocol, while this PR appears to implement the requested compatibility path coherently with focused tests and sufficient live proof; the remaining action is normal maintainer and CI review, not cleanup closure.

## What This Changes

The PR updates the OpenAI-compatible Chat Completions gateway adapter, tests, docs, and changelog to support function tools, `tool_choice`, `tool_calls`, and `role: "tool"` follow-up turns while rejecting unsupported forced modes.

## Best Possible Solution

Land the focused gateway contract after normal CI and maintainer merge gates, preserving explicit forced-mode rejection until the agent runtime can enforce those modes correctly.

## Reproduction Assessment

Yes, source-level. At the provided current-main SHA, `/v1/chat/completions` has no request fields or command path for client tools and always emits stop-shaped Chat responses instead of OpenAI-compatible tool-call responses.

## Solution Assessment

Yes. Reusing the existing `clientTools` and `pendingToolCalls` gateway seams is the narrow maintainable path, and rejecting forced `tool_choice` modes is safer than pretending runtime enforcement exists.
