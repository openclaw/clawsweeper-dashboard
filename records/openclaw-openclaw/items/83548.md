## Summary

Keep open: current main has enough evidence that Discord parent-channel sends should use `target`, not `channel`, but the reported failure mode is still a real message-tool ergonomics/docs gap rather than a safely closeable duplicate or shipped fix.

## What This Changes

The issue requests support, a clearer error, or documentation for Discord thread-bound sessions trying to use the message tool to post to a parent channel.

## Best Possible Solution

Keep `channel` as provider selection, make `target` the documented way to send to a Discord parent channel from a thread, and add a focused error/schema/doc hint for channel-id-shaped `channel` misuse.

## Reproduction Assessment

No high-confidence live Discord reproduction was established. Source inspection does show the relevant path: `channel` is provider selection, `target` is the recipient, and an invalid provider-shaped `channel` can produce the reported `Unknown channel` style failure when no usable fallback exists.

## Solution Assessment

Unclear until a maintainer chooses the policy. The narrowest maintainable direction appears to be keeping `target` as the parent-channel recipient and improving the tool/schema/error/docs rather than adding a second recipient meaning to `channel`.
