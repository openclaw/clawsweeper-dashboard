## Summary

Keep this open. The report is fresh, detailed, externally reported, and current main still shows a plausible Telegram recovery gap in the default isolated polling ingress path rather than a clearly shipped or superseded fix.

## What This Changes

Investigate and harden Telegram polling recovery so stale-socket restarts cannot leave inbound processing wedged while gateway and channel health appear green.

## Best Possible Solution

Add isolated-ingress liveness recovery using the existing polling stall threshold, so a silent worker or missing getUpdates progress restarts the Telegram polling worker/bot/transport and surfaces unhealthy status promptly.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection does show a plausible current-main reproduction path: the default isolated ingress can await a silent worker indefinitely while the legacy polling path has an explicit getUpdates stall watchdog.

## Solution Assessment

Yes, the narrowest maintainable fix is to port the existing Telegram polling liveness semantics into the default isolated ingress path rather than adding a new config knob or broadly changing gateway health thresholds.
