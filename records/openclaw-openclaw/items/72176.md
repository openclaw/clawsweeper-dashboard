Keep open: current main still has the missing-stable-message-id inbound dedupe gap, and the focused fix remains an open PR rather than merged code. The report is backed by concrete session-history details plus a later confirmation that the behavior still reproduced in v2026.5.2.

## What This Changes

The issue reports intermittent duplicate user-message delivery after upgrading to 2026.4.24, causing duplicate assistant replies across WebChat and WeChat.

## Best Possible Solution

Land or replace the focused idless inbound retry fix, keep the WebChat attachment race separate, and close this issue only after the equivalent fix reaches main or a release.

## Solution Assessment

Yes, the narrow maintainable path is to add a deterministic idless inbound dedupe fallback in the shared auto-reply layer while keeping WebChat-specific races in their own gateway fix.
