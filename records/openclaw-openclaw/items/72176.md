## Summary

Keep open: current main still leaves source-level gaps for regenerated or missing inbound IDs and for user transcript/session-history idempotency, and a fresh comment reports the problem still occurs in v2026.5.2.

## What This Changes

Prevent intermittent duplicate user-message and assistant-reply delivery across WebChat, WeChat, and other channels after 2026.4.24.

## Best Possible Solution

Add one generic Gateway/auto-reply idempotency path that covers missing or regenerated inbound IDs, carries stable duplicate metadata into user transcript/session-history updates, and preserves legitimate repeated messages with targeted regressions.

## Reproduction Assessment

Yes, by source inspection rather than a live run: missing `MessageSid` produces an invalid inbound claim that is not skipped, and duplicate inline user transcript updates receive fresh sequence numbers.

## Solution Assessment

No current-main solution fully solves this report. #72605 is useful for same-key WebChat races, but the broader all-channel path still needs generic missing/regenerated-id fallback plus user transcript idempotency.
