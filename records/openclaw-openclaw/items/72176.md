## What This Changes

The issue reports intermittent duplicate user-message and assistant-reply transcript entries after upgrading to 2026.4.24, seen across WebChat and WeChat with identical timestamps but different sequence numbers.

## Best Possible Solution

Add a generic Gateway/auto-reply idempotency path for missing or regenerated inbound IDs, propagate stable duplicate metadata into transcript/session-history updates, and preserve legitimate repeated messages with focused regression tests.

## Reproduction Assessment

## Solution Assessment
