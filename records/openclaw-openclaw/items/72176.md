## Summary

Keep open: current main still has a source-backed all-channel duplicate path when inbound ids are missing or regenerated, and the related WebChat work only covers same-id retries.

## What This Changes

The issue asks OpenClaw to stop intermittent duplicate user and assistant deliveries introduced in 2026.4.24 across WebChat, WeChat, and other channels.

## Best Possible Solution

Implement a generic Gateway/auto-reply idempotency path for missing or regenerated inbound ids, propagate stable duplicate metadata into user transcript updates, and keep same-key WebChat race handling coordinated with #72605.

## Reproduction Assessment

Yes. The issue provides concrete sessions_history duplicate pairs, and current main has a source-level path where missing MessageSid yields an invalid dedupe claim, user transcript updates lack stable ids, and SSE assigns fresh seq numbers.

## Solution Assessment

No current-main solution fully covers this report. #72605 is useful for same-key WebChat retries, but this issue needs an all-channel fallback for missing/regenerated ids plus transcript idempotency coverage.
