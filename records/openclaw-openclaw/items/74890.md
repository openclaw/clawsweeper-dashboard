## Summary

Keep this issue open. Current `main` still has a macOS-owned config write path that can commit a root config with `gateway.mode` removed before treating that loss as an audit-only anomaly, while the TypeScript gateway writer and browser Control UI path already have stronger guards.

## What This Changes

The issue asks for Dashboard/macOS config writes to preserve or reject loss of required config envelope fields such as `meta` and `gateway.mode` instead of relying on gateway auto-restore after a bad snapshot is written.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh full config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current `main` writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the macOS direct-writer/fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash/pre-commit guards for the same class of clobber.
