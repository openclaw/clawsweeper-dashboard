## Summary

Keep this open. Current main still lets the macOS direct writer and local fallback persist a root config payload that has lost `gateway.mode`; the TypeScript gateway writer already rejects that class before commit, and open PR #75815 is the active fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue reports that macOS app/dashboard config writes can replace the live config with snapshots missing mandatory envelope fields such as `meta` or `gateway.mode`, causing gateway restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side fix that preserves existing gateway envelope/auth fields and rejects destructive `gateway.mode` removal before replacing `openclaw.json`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrow maintainable fix is at the macOS direct-writer/fallback boundary, matching the existing gateway-side pre-commit guard model; a broader migration to patch-based app writes can remain follow-up work.
