## Summary

Keep this open. Current main still has macOS direct and local-fallback config write paths that can persist a root payload missing `gateway.mode` before only auditing the anomaly, while the TypeScript gateway writer already rejects that class before commit. Open PR #75815 is the active fix candidate and should be reviewed or landed rather than closing this issue ahead of the merge.

## What This Changes

Prevent Dashboard/Control UI/macOS app config writes from stripping required config envelope fields such as `meta` and `gateway.mode`, causing last-good restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side fix that makes direct/local config saves preserve existing gateway-owned fields, reject destructive `gateway.mode` removal before replacing `openclaw.json`, surface rejected saves to callers, and cover `OpenClawConfigFile.saveDict`, `ConfigStore.save`, `AppState.syncGatewayConfigNow`, and visible config UI call sites with focused Swift regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The best near-term fix is the narrow macOS writer/fallback guard already proposed in #75815, while keeping the existing TypeScript gateway pre-commit rejection model; a broader migration to patch-only app writes can remain follow-up work.
