## Summary

Keep this issue open. Current main has a partial macOS fallback-preservation fix, but the direct macOS config writer still persists the replacement file before treating `gateway-mode-removed` as an audit-only anomaly, and open PR #75815 remains the active fix candidate for rejecting that destructive path.

## What This Changes

Make Dashboard/macOS app config writes preserve `meta`, `gateway.mode`, and other root envelope fields and reject destructive partial writes before replacing `openclaw.json`.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side guard that preserves mandatory envelope fields and rejects `gateway.mode` removal before any direct file replacement.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call the direct Swift writer with a root payload whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes, the active PR direction is the narrowest maintainable fix. Current main’s fallback merge helps one path, but the remaining direct-writer boundary should reject destructive writes pre-commit instead of relying on gateway auto-restore.
