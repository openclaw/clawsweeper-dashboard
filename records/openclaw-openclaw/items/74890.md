## Summary

Keep this issue open. Current main still has a macOS-owned config write path that can replace the live config with a root payload missing `gateway.mode`; that loss is classified only after the atomic write, while the TypeScript gateway writer and browser Control UI path already have stronger pre-commit/hash guards.

## What This Changes

The issue asks for Dashboard/Control UI and macOS app config writes to preserve mandatory root envelope fields such as `meta` and `gateway.mode` instead of committing partial snapshots that trigger last-good restore churn.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh full config or reject/preserve mandatory envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and `ConfigStore.save` local fallback behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the macOS direct-writer and fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash/pre-commit guards for the same clobber class.
