## Summary

Keep this issue open. Current main has a partial macOS fallback-preservation fix for `gateway.auth` and unrelated keys, but the direct Swift config writer still writes the replacement file before treating `gateway-mode-removed` as an audit-only anomaly; open PR #75815 remains the active fix candidate for rejecting that destructive path.

## What This Changes

The issue asks for macOS/dashboard config writes to preserve mandatory config envelope fields such as `meta` and `gateway.mode` instead of relying on gateway auto-restore after clobbers.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side guard that preserves mandatory envelope fields and rejects `gateway.mode` removal before any direct file replacement.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call the direct Swift `saveDict` path with a root payload whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes, the active PR direction is the narrowest maintainable fix. Current main’s fallback merge helps one path, but the remaining direct-writer boundary should reject destructive writes pre-commit instead of relying on gateway auto-restore.
