## Summary

Keep this open. Current main still writes the macOS direct-config payload before treating `gateway-mode-removed` as an audit anomaly, and open PR #75815 is already the focused fix candidate for rejecting that destructive path.

## What This Changes

Prevent Dashboard/macOS app config writes from dropping required root envelope fields such as `meta` and `gateway.mode` and causing restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side pre-commit guard that preserves required envelope fields and rejects destructive `gateway.mode` removal before replacing `openclaw.json`.

## Reproduction Assessment

Yes. Start with an existing config containing `gateway.mode: "local"`, then call the direct Swift `OpenClawConfigFile.saveDict` path with a root payload whose `gateway` object lacks `mode`; current main writes that payload and only audits `gateway-mode-removed` afterward.

## Solution Assessment

Yes. The active PR direction is the narrowest maintainable fix because the TypeScript writer already rejects this class pre-commit, while current macOS fallback merging covers only one path.
