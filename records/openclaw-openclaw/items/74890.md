## Summary

Keep this open. Current main still lets the macOS direct config writer replace the live file before treating `gateway-mode-removed` as an audit anomaly, and open PR #75815 is the focused fix candidate for rejecting that path.

## What This Changes

The issue reports that macOS app/dashboard config writes can replace `openclaw.json` with snapshots missing `meta` or `gateway.mode`, causing last-good restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side pre-commit guard that preserves required envelope fields and rejects destructive `gateway.mode` removal before replacing `openclaw.json`.

## Reproduction Assessment

Yes. Start with an existing config containing `gateway.mode: "local"`, then call the direct Swift `OpenClawConfigFile.saveDict` path with a root payload whose `gateway` object lacks `mode`; current main writes that payload and only audits `gateway-mode-removed` afterward.

## Solution Assessment

Yes. The active PR direction is the narrowest maintainable fix because it brings the macOS direct writer closer to the TypeScript writer’s reject-before-commit behavior without requiring a broad config architecture migration first.
