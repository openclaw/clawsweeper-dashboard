## Summary

Keep this open. Current main still has a macOS-owned direct config write path that can replace the live config with a root missing `gateway.mode`; the anomaly is logged after the atomic write instead of being rejected or merged before commit, while the TypeScript gateway writer and browser Control UI path already have stronger pre-commit guards.

## What This Changes

The issue requests that Dashboard/Control UI and macOS app config writes preserve mandatory root envelope fields such as `meta` and `gateway.mode` instead of committing partial snapshots that trigger last-good restore churn.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh full config or reject/preserve mandatory envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the macOS direct-writer/fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash and pre-commit rejection for this class of clobber.
