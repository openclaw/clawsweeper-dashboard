## Summary

Keep this open. Current main still has a macOS-owned direct config write path that can replace the live config before `gateway.mode` removal is treated as suspicious, and the local `ConfigStore.save` fallback still reuses that writer after a gateway write failure.

## What This Changes

The issue asks Dashboard/Control UI/macOS config writes to preserve mandatory config envelope fields such as `meta` and `gateway.mode` and reject partial root snapshots before replacing `openclaw.json`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh full config or reject/preserve mandatory envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for direct saves and local fallback behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable repair is in the macOS direct-writer and `ConfigStore.save` fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash/pre-commit guards for this clobber class.
