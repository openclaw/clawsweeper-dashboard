## Summary

Keep this open. Current main still has the macOS direct config writer and local fallback path that can replace an existing config with a supplied root missing `gateway.mode`; the Swift code audits that loss only after the atomic write, while the gateway/browser write path rejects the same clobber class before committing.

## What This Changes

Prevent Dashboard/macOS app config writes from stripping required config envelope fields such as `meta` and `gateway.mode`, and add regression coverage for those write paths.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh full config or reject/preserve mandatory envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict`, `ConfigStore.save`, and any direct app-owned gateway sync path that can write the root config.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or `ConfigStore.save` local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the macOS direct-writer and fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash and pre-commit clobber guards for this failure class.
