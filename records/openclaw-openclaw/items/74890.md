## Summary

Keep this open. Current main still has the macOS direct config writer and local fallback path that can replace the live config with a root payload missing `gateway.mode`; the TypeScript gateway path already rejects that class before commit, and open PR #75815 is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

Prevent Dashboard/macOS app config writes from stripping mandatory config envelope fields such as `meta` and `gateway.mode`, and route those writes through preservation, validation, or pre-commit rejection instead of relying on auto-restore.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side fix that makes direct/local config saves preserve existing `gateway.auth` and mandatory envelope fields, reject destructive `gateway.mode` removal before replacing `openclaw.json`, and cover `OpenClawConfigFile.saveDict`, `ConfigStore.save`, and app sync call sites with focused Swift regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable path is to fix the macOS direct-writer/fallback boundary and keep the gateway writer’s existing pre-commit guard model; a broader migration to patch-based app writes can remain follow-up work after the destructive fallback is blocked.
