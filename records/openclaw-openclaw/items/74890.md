## Summary

Keep this issue open. Current `main` still has a macOS-owned direct config writer and local fallback path that can commit a root config after `gateway.mode` disappears, while only logging/auditing the anomaly; the browser Control UI and TypeScript gateway writer already reject the same clobber class before replacing the live config.

## What This Changes

Prevent Dashboard/macOS app config writes from stripping required root config envelope fields such as `meta` and `gateway.mode`, which can trigger repeated auto-restore churn.

## Best Possible Solution

MacOS local/direct config saves should share the gateway writer’s safety contract: merge against a fresh full config or reject before replacing `openclaw.json` whenever an existing config has required envelope fields and the proposed payload drops them, with Swift regression tests proving `meta`, `gateway.mode`, auth, reload, and unrelated root fields survive.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current `main` writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable repair is in the macOS direct-writer/fallback boundary, reusing the same pre-commit preservation or rejection semantics already present in the TypeScript writer instead of adding startup defaults or relying on auto-restore.
