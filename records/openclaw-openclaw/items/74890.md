## Summary

Keep this open: current main still allows the macOS direct config writer and local fallback path to persist a payload missing `gateway.mode`, while the linked PR is only an open fix candidate and not merged into main.

## What This Changes

The issue asks for macOS app/dashboard config writes to preserve mandatory config envelope fields such as `meta` and `gateway.mode` instead of triggering gateway restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side fix that rejects destructive `gateway.mode` removal before replacing `openclaw.json`, preserves unrelated gateway fields, and covers the direct writer plus local fallback with focused Swift tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` local fallback with a root whose `gateway` object lacks `mode`; current main writes the payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrow maintainable solution is to harden the macOS direct-writer/fallback boundary while leaving the existing TypeScript gateway pre-commit rejection model intact; a full patch-based dashboard migration can remain a follow-up.
