## Summary

Keep this open. Current main still has a macOS-owned direct config write path that can replace the live config with a payload missing `gateway.mode`; the anomaly is logged after the atomic write, while the gateway/browser Control UI path has a base-hash and pre-commit rejection guard.

## What This Changes

The issue asks OpenClaw to prevent Dashboard/macOS app config saves from stripping required root config envelope fields such as `meta` and `gateway.mode`.

## Best Possible Solution

Direct/local macOS saves should preserve or reject loss of mandatory envelope fields before replacing `openclaw.json`, preferably by sharing the gateway writer’s pre-commit safety contract or adding an equivalent Swift read/merge/validate/write guard with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: "local"`, then call the Swift direct writer or local `ConfigStore.save` fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable repair is in the macOS direct-writer/local-fallback boundary, because the browser Control UI and TypeScript gateway writer already enforce the stronger base-hash and pre-commit rejection path.
