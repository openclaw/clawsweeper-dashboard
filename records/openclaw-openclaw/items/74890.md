## Summary

Keep this open because current main still lets the macOS direct config writer replace the live config before treating `gateway-mode-removed` as an audit anomaly, while PR #75815 is the active focused fix candidate.

## What This Changes

The issue asks for macOS/dashboard config writes to preserve mandatory config envelope fields and reject destructive partial snapshots before replacing `openclaw.json`.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side pre-commit guard that preserves required gateway fields and rejects destructive direct writes before replacing `openclaw.json`.

## Reproduction Assessment

Yes. Seed an existing config with `gateway.mode: "local"`, then pass a direct Swift `OpenClawConfigFile.saveDict` payload whose `gateway` object lacks `mode`; current main writes the payload before recording `gateway-mode-removed` as suspicious.

## Solution Assessment

Yes. A focused guard in the macOS direct writer is the narrowest maintainable fix for this report because direct Swift callers bypass the TypeScript writer that already rejects this class of clobber.
