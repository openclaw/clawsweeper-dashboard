## Summary

Keep this open. Current main still has a macOS direct config-write path that commits the replacement file before treating `gateway-mode-removed` as an audit-only anomaly, and PR #75815 is the active focused fix candidate for the same writer boundary.

## What This Changes

Prevent dashboard/macOS app config writes from stripping mandatory `meta` and `gateway.mode` fields and causing last-good auto-restore churn.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side guard that preserves mandatory config envelope fields and rejects destructive `gateway.mode` removal before any file replacement.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call the direct Swift writer with a root payload whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrow maintainable fix is to reject or merge-preserve mandatory envelope fields in the macOS writer before commit; relying on gateway auto-restore happens too late for this failure mode.
