## Summary

Keep this open. Current main still lets the macOS direct config writer commit a payload missing `gateway.mode` and only records `gateway-mode-removed` after the write; the adjacent #75631 fix preserved auth/fallback keys but did not close this mode-removal path, and open PR #75815 is the focused fix candidate.

## What This Changes

The issue asks for Dashboard/macOS app config writes to preserve required envelope fields such as `meta` and `gateway.mode` and reject destructive partial snapshots before replacing the live config.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side pre-commit guard that rejects destructive `gateway.mode` removal, preserves required envelope fields, records rejected payloads, and makes direct-save callers handle rejection.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to start with an existing config containing `gateway.mode: "local"`, then call the direct Swift `OpenClawConfigFile.saveDict` path with a root payload whose `gateway` object omits `mode`; current main writes first and only audits `gateway-mode-removed` afterward.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the macOS direct writer before atomic replacement and make callers notice rejection, matching the TypeScript config writer's existing `.rejected.*` behavior; a broader migration to `config.patch` can remain follow-up work.
