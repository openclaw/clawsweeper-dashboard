## Summary

Keep this open. Current main still has a macOS direct config writer and local fallback path that can replace `openclaw.json` with a root payload missing `gateway.mode`; the loss is detected only after the atomic write, while the TypeScript gateway/browser path already has pre-commit hash and clobber guards.

## What This Changes

Prevent Dashboard/macOS app config writes from stripping required config envelope fields such as `meta` and `gateway.mode`, and add regression coverage for those write paths.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh full config or reject/preserve mandatory envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and `ConfigStore.save` local fallback behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the macOS direct-writer and fallback boundary, because the browser Control UI and TypeScript gateway writer already use base-hash and pre-commit guards for this clobber class.
