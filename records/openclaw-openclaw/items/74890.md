## Summary

Keep this open. Current main has a partial macOS config preservation fix, but `OpenClawConfigFile.saveDict` still writes the supplied root before classifying `gateway-mode-removed`, and open PR #75815 remains the focused fix candidate for the remaining pre-commit rejection/hardening work.

## What This Changes

The issue asks for Dashboard/Control UI/macOS config writes to preserve required config envelope fields such as `meta` and `gateway.mode` and reject partial clobbers before replacing the live config.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side hardening that rejects destructive `gateway.mode` removal before replace, preserves required envelope/auth fields on fallback writes, and covers the direct writer and app save call sites with focused Swift tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction remains: start from a config with `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes for the linked fix direction, but not for closing on the current partial main implementation. The narrow maintainable solution is to bring the macOS direct writer/fallback boundary up to the gateway writer’s pre-commit rejection behavior.
