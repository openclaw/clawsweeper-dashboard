## Reproduction Assessment

Yes. The issue is source-reproducible from current main: a non-greedy fenced-code regex is used to skip code segments, and the rewrite loop rewrites any text after that premature match.

## Solution Assessment

Yes at the issue level. The best fix is a narrow Markdown-aware scanner in the Discord mention rewriter plus focused regression tests, not a config option or broad mention-policy redesign.
