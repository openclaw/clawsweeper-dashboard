## Summary

Close as implemented: current main replaces the Discord mention code-region regex with a scanner that preserves backtick code segments before mention rewriting, and the merged replacement PR is already included in the v2026.6.9-beta.1 prerelease.

## What This Changes

The issue reports that Discord mention alias rewriting can convert an @handle inside a fenced code block when the code body contains a triple-backtick literal.

## Best Possible Solution

Keep the current shared Discord mention scanner on main and let the normal release train carry it forward; the remaining open source PR can be handled separately as superseded.

## Reproduction Assessment

Yes for the original bug: the issue includes a focused source repro and the old release tag shows the regex path that could stop at an inline triple-backtick literal. Current main no longer reproduces the failure because it uses a scanner that skips fenced code before rewriting mentions.

## Solution Assessment

Yes. The current fix is in the shared Discord mention helper used by normal sends, structured sends, webhooks, and preview/final gating, which is the narrow maintainable owner boundary for this behavior.
