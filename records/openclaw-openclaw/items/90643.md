## Summary

Keep open: current main and latest release still use the regex-based Discord code-region skip that can expose later fenced-code text to mention rewriting, and an open linked PR already owns the narrow fix path.

## What This Changes

The issue reports that Discord mention alias rewriting can turn an `@handle` inside a fenced Markdown code block into a targeted Discord mention when the code body contains a triple-backtick literal.

## Best Possible Solution

Review and land the linked Discord mention-scanner PR, or replace it with an equivalent narrow fix in `extensions/discord/src/mentions.ts` plus focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows the non-greedy code-segment regex can stop at an inline triple-backtick literal, then `rewritePlainTextMentions` processes the remainder of the fenced body; I did not run the repro command in this read-only checkout.

## Solution Assessment

Yes. The best solution is a local Markdown-aware code-span/code-fence scanner in `extensions/discord/src/mentions.ts` with regressions for embedded backtick runs, because normal sends, structured sends, webhooks, and preview/final logic all call this helper.
