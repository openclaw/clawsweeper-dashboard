## What This Changes

The issue reports that Discord outbound mention alias rewriting can convert an `@handle` inside a fenced Markdown code block when the code body contains a triple-backtick literal.

## Best Possible Solution

Keep the current main scanner fix and close this issue as implemented, while treating the still-open source PR as superseded by the merged replacement.

## Reproduction Assessment

Yes. The old release code used a non-greedy code-segment regex that could terminate at a triple-backtick literal inside a fence; current main no longer has that path and includes a regression test for the case.

## Solution Assessment

Yes. The best fix is the current owner-local Discord mention scanner plus regression coverage, because every affected outbound Discord path already funnels through `rewriteDiscordKnownMentions`.
