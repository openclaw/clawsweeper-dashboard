## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Merge missing plugin install records on import:** `src/infra/profile-portability.ts:1188-1190`
  - body: `importProfileArchive` skips the entire `plugin-installs` asset whenever the target `plugins/installs.json` already exists. A target machine with any existing plugin index will import none of the portable install records from the profile archive, even records for new plugin IDs, so the documented non-destructive fill-missing portability behavior loses plugin installs. Read existing records, merge only missing IDs, then refresh the index.
  - confidence: 0.83
- **[P3] Register profile docs in CLI navigation:** `docs/cli/profile.md:1-6`
  - body: The PR adds a dedicated `docs/cli/profile.md` page and links CLI help to `/cli/profile`, but it does not add `cli/profile` to `docs/docs.json` or the CLI index command table/tree. That leaves the new command page orphaned in the published CLI reference navigation.
  - confidence: 0.9
- **[P3] Use dry-run wording for config import summaries:** `src/infra/profile-portability.ts:1276`
  - body: `importProfileArchive` correctly skips `replaceConfigFile` when `--dry-run` is set, but the text summary still prints `Config paths applied`. In a dry run this reports completed config writes that did not happen; use dry-run-specific wording such as `would apply` or split projected and completed fields.
  - confidence: 0.86
