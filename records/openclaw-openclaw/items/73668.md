## Summary

Keep open. Current main still lacks this profile export/import surface, and the latest PR head is not mergeable because the privacy sanitizer can still export path-like arbitrary config keys into the archive; docs/changelog discoverability also need cleanup.

## What This Changes

The PR adds `openclaw profile export/import`, a `.openclaw-profile.tar.gz` archive implementation for sanitized config, workspace markdown, and plugin install metadata, plus CLI, command, infra tests, and docs.

## Best Possible Solution

If core profile portability is accepted, land it as a documented privacy-safe archive format that strips secret and host-local path material from every exported config projection, keeps plugin install records as a portable projection, preserves non-destructive import behavior, updates CLI discoverability and changelog, and includes regression tests for the sanitizer gaps.

## Reproduction Assessment

Yes. A unit-level `buildSafeProfileConfig` input with `plugins.entries.demo.config.sourcePath`, `mcp.servers.demo.localPath`, or `tools.foo.configPath` will pass the current key filter because the normalized keys are neither exact local-path keys nor secret suffixes.

## Solution Assessment

No. The feature direction is plausible, but this PR is not the best mergeable solution until path-like arbitrary config keys are stripped or otherwise privacy-filtered, with regression coverage and the required docs/changelog follow-through.
