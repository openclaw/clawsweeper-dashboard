## What This Changes

The issue reports that `secrets apply` removes `auth-profiles.<provider>:default.keyRef` when applying a `models.providers.<provider>.apiKey` target with auth-profile scrubbing enabled.

## Best Possible Solution

Review and land one of the linked narrow fix PRs, or close them and make the equivalent source/test change that preserves canonical auth-profile SecretRefs during provider-target scrubbing.

## Reproduction Assessment

Yes. Source inspection shows a `models.providers.<id>.apiKey` target enters the provider scrub set, and the scrub loop deletes the matching auth profile's ref field; the existing apply test also asserts that destructive post-state.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve SecretRef-backed auth-profile refs during provider-target scrubbing, or refuse the apply as already shadowed; adding another option would leave the destructive default path intact.
