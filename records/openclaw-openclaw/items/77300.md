## What This Changes

`secrets apply` should preserve an existing `auth-profiles.<provider>:default.keyRef` when applying a `models.providers.<provider>.apiKey` SecretRef plan with auth-profile scrubbing enabled.

## Best Possible Solution

Review and land the linked PR or an equivalent narrow patch that preserves authoritative auth-profile SecretRefs and updates the existing regression test expectation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a `models.providers.<id>.apiKey` target enters provider scrubbing, and the matching auth profile's `keyRef` is deleted by the scrub loop.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve existing auth-profile SecretRef objects, or refuse a shadowed no-op with a clear message, so apply behavior matches the audit precedence contract.
