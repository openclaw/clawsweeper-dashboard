## What This Changes

Preserve `auth-profiles.<provider>:default.keyRef` when applying a `models.providers.<provider>.apiKey` secrets plan with provider auth-profile scrubbing enabled.

## Best Possible Solution

Land one focused fix that preserves existing auth-profile SecretRef fields during provider API-key scrub, updates the regression test to lock that behavior in, and resolves the duplicate open fix PR.

## Reproduction Assessment

Yes. Source inspection on current main shows `models.providers.<id>.apiKey` targets enter provider scrubbing and the scrub loop deletes the matching auth profile's `keyRef`; the current test suite also asserts that destructive post-state.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve SecretRef objects in auth-profile ref fields while still scrubbing plaintext values, with regression coverage around the reported plan shape.
