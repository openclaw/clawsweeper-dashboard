## Summary

The PR passes the session-memory hook model override into LLM slug generation and adds regression tests for provider-qualified, model-only, and default fallback cases.

## Best Possible Solution

Type-narrow the hook config value, resolve the override through the existing alias-aware model-selection contract, add handler/resolver coverage, and document or explicitly decide the `session-memory.model` config surface before merge.

## Reproduction Assessment

Yes. Source inspection shows current main reads the session-memory hook config but does not forward the hook-level model, and the slug generator then resolves only the agent default model.

## Solution Assessment

No. The PR is a plausible start, but the best fix should narrow the hook value and use the existing alias-aware model-selection contract instead of raw splitModelRef passthrough.
