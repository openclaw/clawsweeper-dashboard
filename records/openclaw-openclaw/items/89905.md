## Summary

## What This Changes

The PR passes the session-memory hook `model` value into LLM slug generation and adds regression tests for provider-qualified, bare-model, and default fallback cases.

## Best Possible Solution

Support the session-memory model override through a narrowed hook-config value, resolve it with the existing alias/catalog-aware model-selection contract, document or validate the activated config behavior, and land with real hook-path proof.

## Reproduction Assessment

Yes, source-reproducible: current main reads session-memory hook config for `messages` and `llmSlug`, but does not pass `hookConfig.model` to `generateSlugViaLLM`, and the generator always calls `resolveDefaultModelForAgent`. I did not run a live `/reset` hook path.

## Solution Assessment

No: passing the raw hook value through `splitModelRef` is not the best fix because it skips existing alias/provider inference and type narrowing. The safer path is to normalize and resolve the hook override through the same model-selection contract used by other hook model surfaces.
