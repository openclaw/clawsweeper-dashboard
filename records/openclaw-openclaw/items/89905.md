## Summary

Keep open: the PR targets a real source-reproducible bug, but the patch is not merge-ready because it forwards a provider-qualified hook model as only `model`, leaving the embedded runner to default `provider` to OpenAI. The supplied proof is also a standalone logic simulation, not a real OpenClaw hook run.

## What This Changes

The PR forwards the session-memory hook’s configured `model` value into LLM slug generation instead of always resolving the agent default model.

## Best Possible Solution

Resolve the hook-level model string through the existing provider/model and alias resolver, then pass both `provider` and `model` into `runEmbeddedAgent` while preserving the default-model fallback.

## Reproduction Assessment

Yes from source: current main reads the session-memory hook config but drops the `model` field before slug generation. The PR’s replacement path is also source-reproducible as incorrect for provider-qualified refs because the runner defaults an unset provider to OpenAI.

## Solution Assessment

No; forwarding a raw hook model string is too narrow for the documented provider/model or alias contract. The safer fix is to resolve the hook override into a provider/model pair before calling the embedded runner.
