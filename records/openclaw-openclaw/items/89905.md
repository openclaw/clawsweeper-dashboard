## Summary

Keep open: the linked bug is valid, but this patch still bypasses OpenClaw's configured model-resolution path for bare or alias hook model refs, forwards an unnarrowed hook config value, and lacks real `/new` or `/reset` hook-path proof.

## What This Changes

This PR passes the session-memory hook's configured model into `generateSlugViaLLM`, splits provider-qualified refs, and adds slug-generator unit coverage for override and fallback cases.

## Best Possible Solution

Honor the session-memory hook model by narrowing the config value, resolving it through the existing configured model-selection alias/catalog/provider path, preserving default fallback when absent, and proving a real `/new` or `/reset` slug run.

## Reproduction Assessment

Yes, from source inspection: current main resolves the session-memory hook config but never forwards `model`, and `generateSlugViaLLM` always resolves the agent default. I did not run a live `/new` or `/reset` flow because this was a read-only review.

## Solution Assessment

No. Passing the hook value is the right direction, but using `splitModelRef` and an unnarrowed config value is not the best fix because existing model-selection helpers already handle aliases, provider inference, and configured refs.
