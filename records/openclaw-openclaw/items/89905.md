## Summary

Keep open: the current PR still has blocking correctness and proof gaps because it forwards an un-narrowed hook config value and parses the override with `splitModelRef`, which bypasses OpenClaw's alias-aware model-selection path.

## What This Changes

The PR forwards `hooks.internal.entries.session-memory.model` into LLM slug generation and adds regression tests for provider-qualified, bare-model, and default fallback cases.

## Best Possible Solution

Resolve the session-memory override through the canonical alias-aware model-selection API after narrowing the hook config value, document or doctor-check the new supported config behavior, and land only with real hook-path proof.

## Reproduction Assessment

Yes. Current main reads the session-memory hook config but does not pass `hookConfig.model` into slug generation, and the generator resolves only the agent default model.

## Solution Assessment

No. The proposed path is plausible for slash-qualified strings, but the best fix should narrow the hook config value and use the existing alias-aware model-selection contract rather than `splitModelRef`.
