## Summary

Keep open: this PR targets a valid session-memory hook model-routing bug, but the current patch is not merge-ready because the hook override is not type-narrowed, bare/alias model overrides bypass the existing config-aware resolver, and the supplied proof does not exercise the real session-memory hook path.

## What This Changes

The PR passes `hooks.internal.entries.session-memory.model` into slug generation, parses provider-qualified model refs with `splitModelRef`, and adds slug-generator unit tests for override and fallback behavior.

## Best Possible Solution

Resolve the hook model through the existing config/alias-aware model-selection path, narrow the open-ended hook config value to a string, preserve default fallback behavior, and require redacted real hook-run proof before merge.

## Reproduction Assessment

Yes from source inspection: current main reads the session-memory hook config but never forwards `model`, and the slug generator always resolves the agent default model. I did not run a live `/new` or `/reset` hook repro in this read-only review.

## Solution Assessment

No: passing the model from the handler is the right surface, but `splitModelRef` is too narrow for OpenClaw config semantics. The safer fix is to reuse the existing config/alias-aware model resolver and narrow the hook config value before forwarding it.
