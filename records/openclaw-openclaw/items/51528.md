## Summary

Keep this PR open: the UX is plausible, but the branch is not mergeable as-is because it applies ranking and persistence to an inactive Control UI helper path while current main renders and switches the chat model picker through the newer chat select-state/session-controls modules. The external PR also lacks after-fix real behavior proof, so it needs contributor or human follow-up before normal merge review.

## What This Changes

The PR adds browser-local recent chat-model storage, ranks recent and custom-provider model options in the Gateway UI picker, persists successful selections, and adds related UI tests.

## Best Possible Solution

Port the UX into ui/src/ui/chat-model-select-state.ts and ui/src/ui/chat/session-controls.ts, keep Default pinned, rank resolvable recents before true custom-provider entries, preserve catalog order for the rest, and persist only successful non-default canonical selections best-effort.

## Reproduction Assessment

Yes, for the PR defects by source inspection. Current main renders and switches through ui/src/ui/chat/session-controls.ts and ui/src/ui/chat-model-select-state.ts, while this PR changes ui/src/ui/app-render.helpers.ts and does not update that active path.

## Solution Assessment

No, not as-is. The feature should move to the active picker modules, avoid recording Default as an explicit recent model, and classify bundled provider IDs before custom sorting.
