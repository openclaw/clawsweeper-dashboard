## Summary

Keep open: the UX goal is plausible, but this branch is not mergeable as reviewed because it applies the picker behavior to the old helper path while current main renders and switches through the split chat modules, and the external PR still lacks required real behavior proof.

## What This Changes

The PR adds browser-local recent chat-model storage, custom-provider ranking, picker option reordering, and tests for the Gateway UI chat model picker.

## Best Possible Solution

Port the UX into ui/src/ui/chat-model-select-state.ts and ui/src/ui/chat/session-controls.ts, keep Default pinned, rank resolvable recents before true custom-provider entries, preserve catalog order for the rest, and persist only successful non-default canonical selections best-effort.

## Reproduction Assessment

Yes, for the PR defects by source inspection. Current main renders and switches the picker through ui/src/ui/chat/session-controls.ts and ui/src/ui/chat-model-select-state.ts, while this PR changes ui/src/ui/app-render.helpers.ts and does not update that active path.

## Solution Assessment

No, not as-is. The intended UX is narrow, but the implementation should move to the active picker modules, avoid recording Default as an explicit recent model, and classify bundled provider IDs before custom sorting.
