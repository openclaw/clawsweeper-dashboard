## Summary

Keep this PR open because the requested recent-first/custom-provider-first model picker behavior still is not implemented on current main, and the PR remains useful but stale against the current Control UI split. Earlier review feedback about best-effort localStorage handling and canonical recent refs appears to have been addressed in the PR discussion, but the active picker and switch paths have since moved.

## What This Changes

The PR adds browser-local recent chat model storage, ranks recent and custom-provider models ahead of other picker options, persists successful selections, and adds UI unit coverage.

## Best Possible Solution

Port the useful behavior into the current Control UI model picker architecture: keep the Default option pinned, rank available recent selections first, rank truly custom-provider models before remaining built-ins, ignore stale recents, persist only successful non-default selections using canonical resolved refs, and keep storage access best-effort.
