## Summary

Keep open: the feature is still useful, but the PR is not mergeable as-is because it targets a stale picker implementation path, misses current bundled provider ids, and has only test/CI evidence rather than real after-fix behavior proof.

## What This Changes

The branch adds browser-local MRU chat model ordering, custom-provider prioritization, and successful-selection persistence for the gateway/control UI model picker with focused tests.

## Best Possible Solution

Port the UX into the active Control UI modules: keep Default pinned, rank resolvable recents first, then true custom-provider entries, then catalog order, and persist only successful non-default canonical selections best-effort with current-path tests.

## Reproduction Assessment

Yes for the PR defects by source inspection. Current main renders and switches the model picker through ui/src/ui/chat/session-controls.ts and ui/src/ui/chat-model-select-state.ts, while this PR edits the older app-render.helpers.ts path and adds no active-path MRU persistence.

## Solution Assessment

No, not as-is. The intended UX is narrow, but the implementation should move to ui/src/ui/chat-model-select-state.ts and ui/src/ui/chat/session-controls.ts, skip Default writes, and classify bundled providers before custom sorting.
