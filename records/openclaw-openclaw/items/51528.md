## Summary

Keep open: the requested picker UX is not implemented on current main, but this branch is not mergeable as-is because it updates the older app-render helper path while the live picker now runs through the chat select-state and session-controls modules, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds browser-local recent chat-model storage, custom-provider model bucketing, option reordering, and tests for the Control UI chat model picker.

## Best Possible Solution

Port the UX into `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, keeping Default pinned, ranking resolvable recents before true custom-provider entries, preserving catalog order for the rest, and persisting only successful non-default canonical selections best-effort.

## Reproduction Assessment

Yes, for the PR defects by source inspection. Current main renders and switches the picker through `ui/src/ui/chat/session-controls.ts` and `ui/src/ui/chat-model-select-state.ts`, while this PR changes `ui/src/ui/app-render.helpers.ts` and does not update the active path.

## Solution Assessment

No, not as-is. The intended UX is narrow, but the implementation should move to the active picker modules, avoid recording Default as an explicit recent model, and classify bundled providers before custom sorting.
