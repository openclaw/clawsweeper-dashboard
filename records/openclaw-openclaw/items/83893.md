Keep open: current main and the latest release still read Commander's runtime-only `rawArgs` field through a cast, and Commander 14.0.3 typings do not expose that property. The useful prior discussion now has an open fixing PR, so this issue should stay open until https://github.com/openclaw/openclaw/pull/84767 lands or is rejected.

## What This Changes

Make OpenClaw's CLI action reparse path stop silently depending on Commander's non-public `rawArgs` field, or make that fallback dependency explicit and tested.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/84767 as the implementation candidate, preserving lazy-command fallback semantics and keeping machine-readable CLI output quiet.

## Reproduction Assessment

Yes for the source-level concern: current main still casts `Command` to read `rawArgs`, and Commander 14.0.3 typings do not expose that member. No high-confidence user-visible CLI failure has been reproduced.

## Solution Assessment

Yes, a narrow cleanup or diagnostic at the existing reparse seam is the right shape. The linked PR is a plausible fix path, but maintainers should confirm any warning cannot break JSON or other scripted CLI output.
