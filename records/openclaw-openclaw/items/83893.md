## Summary

Keep open as a narrow P3 cleanup: current main still reaches into Commander's runtime-only `rawArgs` field through a cast, and Commander 14.0.3 typings do not expose that property. Existing fallback tests reduce the impact, so this is maintainability work rather than a reproduced user-visible CLI bug.

## What This Changes

The issue requests replacing or hardening the private Commander `rawArgs` access used during lazy CLI command reparsing.

## Best Possible Solution

Encapsulate or remove the private Commander `rawArgs` dependency while preserving lazy-command reparse and quiet JSON-output behavior.

## Reproduction Assessment

Yes for the source-level concern: current main still casts `Command` to read `rawArgs`, and Commander 14.0.3 typings do not expose that member. No high-confidence user-visible CLI failure has been reproduced.

## Solution Assessment

Yes, a narrow cleanup is the right direction, but it should not add noisy fallback warnings that can break JSON-producing CLI paths. Prefer an explicit local adapter or OpenClaw-owned argv capture that keeps current parse semantics.
