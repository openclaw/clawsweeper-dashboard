## Summary

This PR is still necessary because current main still lets cron wildcard tool construction reach the shared profile-policy filter, which removes shell tools under the messaging profile. The patch looks mechanically sound and scoped to cron, but it intentionally changes a security/compatibility boundary, so the remaining action is maintainer acceptance rather than automated repair.

## What This Changes

The PR changes `createOpenClawCodingTools` so cron runs with `runtimeToolAllowlist: ["*"]` skip static/provider profile policy filtering for local tool visibility and background derivation, with regression coverage for cron and non-cron callers.

## Best Possible Solution

Land the scoped cron-only wildcard precedence rule if maintainers agree that explicit cron `toolsAllow: ["*"]` should override static/provider profile defaults; otherwise require a narrower explicit opt-in policy instead of broad wildcard precedence.

## Solution Assessment
