## Summary

## What This Changes

The branch modifies `src/agents/agent-tools.ts` and adds regression tests so cron-triggered wildcard runtime allowlists skip static/provider profile filtering while non-cron callers keep profile caps.

## Best Possible Solution

Land the scoped cron-only wildcard precedence rule if maintainers accept explicit cron `toolsAllow: ["*"]` as an operator request for all local tools; otherwise require a narrower opt-in that preserves profiles as hard caps by default.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance. `createOpenClawCodingTools` is the narrow owner for both final profile filtering and `allowBackground` derivation, but the chosen precedence rule changes the security boundary for cron wildcard allowlists.
