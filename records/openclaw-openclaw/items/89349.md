## What This Changes

This PR changes `createOpenClawCodingTools` so a runtime `*` allowlist skips profile/provider-profile policy filters and adds a focused regression test for `exec` with the `messaging` profile.

## Best Possible Solution

Apply one consistent runtime-wildcard policy before all profile-derived decisions, keep non-profile deny/sandbox/sender restrictions intact, and prove the behavior with focused tests plus a real cron or isolated runtime run.
