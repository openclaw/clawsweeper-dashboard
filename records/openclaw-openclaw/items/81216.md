Keep open: the revised CI-refresh proposal is not implemented on current main. The repository still builds CLI runtime picker data from plugin/catalog entries and static allowlists, and adding scheduled provider API refreshes plus maintainer secrets needs maintainer product and security direction.

## What This Changes

Request adding scheduled CI automation that refreshes CLI runtime model catalogs for Claude CLI, Codex CLI, and Gemini CLI using maintainer-controlled provider API keys and opens automated PRs.
