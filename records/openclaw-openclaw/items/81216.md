## Summary

Keep open: current main still has no live CLI model enumeration hook, and the related picker visibility thread is separate and still open rather than a merged implementation. This needs maintainer API/product direction, not conservative cleanup.

## What This Changes

The issue requests replacing static catalog rows for CLI-backed runtimes with live model enumeration for `/models`, including inline fetch failures and removal of bundled CLI runtime catalog entries.

## Best Possible Solution

Define a provider/plugin-owned live catalog or typed CLI-backend enumeration seam with explicit auth, timeout, cache, and inline-error behavior, then wire `/models` through that seam without putting vendor API policy directly in the picker.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested live CLI enumeration path is absent: `/models` reads the catalog, while CLI backend contracts expose execution/config hooks rather than model listing.

## Solution Assessment

Unclear until maintainers choose the ownership boundary. The direction is plausible, but the safer implementation is likely a plugin-owned live catalog or typed CLI-backend catalog hook rather than provider REST calls embedded in `commands-models.ts`.
