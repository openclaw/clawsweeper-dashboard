## What This Changes

The branch adds LINE-plugin HEAD Content-Length prechecks for outbound image, video, audio, and preview URLs, with focused tests for cap enforcement and same-URL probe deduping.

## Best Possible Solution

Land the LINE-local guarded precheck if maintainers accept the compatibility and latency tradeoffs, and track generic quick-reply inline media coverage separately if full outbound coverage is required.

## Reproduction Assessment

Yes from source inspection. Current main validates outbound LINE media URLs but does not precheck remote Content-Length before push/reply calls, so over-cap media can still be handed to LINE and fail outside OpenClaw's returned result.

## Solution Assessment

Mostly yes. A plugin-local guarded HEAD precheck is narrow and uses the existing SSRF guard; maintainers still need to accept the fail-closed and latency tradeoffs or ask for adapter probe deduping first.
