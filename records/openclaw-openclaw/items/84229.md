## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Cap omitted resolver previews:** `extensions/line/src/outbound-media.ts:203-207`
  - body: When `resolveLineOutboundMedia` returns an image without `previewImageUrl`, `buildLineMediaMessageObject` later sends `previewImageUrl: resolved.previewImageUrl ?? resolved.mediaUrl`. This branch only applies the 10 MiB image cap, so a 5 MiB image in the inline batch or quick-reply path can still reach LINE as a 1 MiB-capped preview instead of failing locally. Treat absent image previews as the same URL and apply the preview cap.
  - confidence: 0.92
- **[P2] Cap omitted pushImageMessage previews:** `extensions/line/src/send.ts:406-410`
  - body: `createImageMessage` fills an omitted preview with the original URL, but this condition only uses the stricter preview cap when `previewImageUrl` is truthy. A caller can pass a 5 MiB image with no preview, have it prechecked only against the 10 MiB image cap, and still send it as the 1 MiB-capped preview. Treat `undefined` as the shared-URL case here.
  - confidence: 0.94
