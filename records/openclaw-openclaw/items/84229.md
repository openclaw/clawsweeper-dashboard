## Summary

Keep this PR open: the main approach is useful and the live proof is strong, but the latest patch still misses two implicit image-preview paths where OpenClaw later sends the original URL as `previewImageUrl` and therefore must enforce LINE's 1 MiB preview cap.

The PR adds LINE outbound media `HEAD` size prechecks, cap constants, URL redaction, soft-fail logging, and focused tests for image, video, audio, and preview media sends.

## Best Possible Solution

Keep the role-aware size precheck, but make every image path that will send an implicit preview evaluate the shared URL against the preview cap and add regression coverage for those exact paths.
