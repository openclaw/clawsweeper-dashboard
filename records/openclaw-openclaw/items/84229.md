## Summary

Keep this PR open for normal maintainer review: current main has URL/SSRF validation for outbound LINE media but no size precheck, and the PR adds a focused plugin-local guard with no blocking code findings. The remaining concerns are merge tradeoffs around extra HEAD probes, fail-closed sends when an origin reports over-cap size, and a disclosed quick-reply inline coverage gap.

## What This Changes

The PR adds guarded HEAD Content-Length prechecks for outbound LINE image, video, audio, and preview media in the LINE plugin send helpers, plus regression tests for cap enforcement and same-URL probe deduping.

## Best Possible Solution

Land the plugin-local precheck once maintainers accept the probe and fail-closed tradeoffs, and track generic quick-reply inline media coverage separately if full outbound coverage is required.
