## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Cap implicit resolver previews:** `extensions/line/src/outbound-media.ts:196-198`
  - body: When `resolveLineOutboundMedia` returns an image without `previewImageUrl`, `buildLineMediaMessageObject` later sends `previewImageUrl: resolved.previewImageUrl ?? resolved.mediaUrl`. This branch only applies the 10 MiB image cap, so a 5 MiB image in the inline batch/quick-reply path still reaches LINE and can be rejected instead of failing locally. Treat absent image previews as the same URL and apply the preview cap.
  - confidence: 0.91
- **[P2] Cap omitted pushImageMessage previews:** `extensions/line/src/send.ts:403-405`
  - body: `createImageMessage` fills an omitted preview with the original URL, but this condition only uses the stricter preview cap when `previewImageUrl` is truthy. A caller can pass a 5 MiB image with no preview, have it prechecked only against the 10 MiB image cap, and still send it as the 1 MiB-capped preview. Treat `undefined` as the shared-URL case here.
  - confidence: 0.93
