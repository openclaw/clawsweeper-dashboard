## Summary

Keep open. Current main still lacks first-class outgoing Telegram custom emoji support for messages, captions, edits/previews, and the shared message tool; the related items are useful context but do not implement the requested message/caption behavior.

## What This Changes

Request support for Telegram Bot API custom emojis in outgoing messages, captions, replies, and agent/tool-facing formatting.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then implement validated Telegram support across send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused docs and regression tests.

Yes. A high-confidence source-level reproduction exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode.

## Solution Assessment
