## Summary

Keep open. Current main and v2026.6.5 preserve raw Telegram `<tg-emoji>` HTML, but first-class custom emoji support is still missing from the shared message tool, Markdown/entity representation, and Telegram send/caption/edit/preview paths.

## What This Changes

Request support for Telegram custom emojis in bot messages, captions, and replies, including custom emoji identifiers and send/display behavior.

## Best Possible Solution

Define a stable custom-emoji/entity contract, then wire validated Telegram support through message-tool sends, captions, replies/quotes, edits/previews, streaming, docs, and focused regression tests while preserving current HTML fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, upstream API, current-main, latest-release, and related-item checks give high confidence that first-class outgoing custom emoji/entity support remains incomplete.

## Solution Assessment

Yes directionally, but not as a narrow automatic repair. Raw `<tg-emoji>` HTML passthrough is useful partial support; the maintainable solution needs an explicit public representation and entity validation across send, caption, reply, edit, and streaming paths.
