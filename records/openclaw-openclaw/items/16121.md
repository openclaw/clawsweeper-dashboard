## Summary

Keep open. Release v2026.6.5 and current main preserve valid raw Telegram `<tg-emoji>` HTML, but the central first-class capability remains missing: agents cannot express custom emoji through the typed message tool or Markdown IR, and Telegram send, caption, edit, and streaming paths have no structured custom-emoji entity contract. The closed entity-formatting PR is useful prior art but was not merged and had delivery-safety concerns, so it does not supersede this issue.

## What This Changes

Add first-class Telegram custom emoji handling for outgoing messages, captions, and replies.

## Best Possible Solution

Define a typed custom-emoji presentation value that keeps raw Telegram offsets private, then map it in the Telegram plugin across text, captions, replies, edits, and streaming while reusing existing chunking and parse-error fallback behavior.

## Reproduction Assessment

Yes, at source level with high confidence: raw HTML passthrough exists, but the typed Markdown IR, message-tool schema, Telegram send options, and streaming edit path contain no custom-emoji entity representation. A live failing bot run was not needed to establish that capability gap.

## Solution Assessment

No existing proposal is yet the best complete fix. The closed entity-builder PR bypassed important delivery safeguards, while an allowlist-only change does not make the capability available to agents; maintainers should first select the typed owner boundary and then integrate it with existing delivery machinery.

## Risks / Open Questions

- The permanent representation is unresolved: a Telegram-only raw entity array would expose transport-private offsets to agents, while a shared IR span changes a portable presentation contract.
- Entity offsets use Telegram's UTF-16 semantics and must remain correct after chunking, caption limits, edits, retries, and streaming updates; a parallel send path could turn currently deliverable messages into hard failures.
- Telegram eligibility constraints for sending custom emoji need explicit validation and a predictable fallback so unsupported bots or emoji identifiers do not suppress the whole message.
