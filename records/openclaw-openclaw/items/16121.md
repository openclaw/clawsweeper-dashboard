## Summary

Keep open. Current main still lacks first-class outgoing Telegram custom emoji support for message text, captions, edits/previews, and the shared message tool; the related reaction PR and closed entity-support discussions are useful context but do not ship the requested message/caption behavior.

## What This Changes

Request support for Telegram Bot API custom emoji in outgoing Clawdbot messages, captions, replies, and places where standard emoji are currently usable.

## Best Possible Solution

Define a small stable custom-emoji/entity representation, then implement validated Telegram support across send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused docs and regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode.

## Solution Assessment

Unclear. The requested support is valid, but raw <tg-emoji> passthrough alone would not cover message-tool input, captions, edits/previews, chunking, fallback, or UTF-16 entity validation.
