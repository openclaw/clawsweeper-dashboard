## Summary

Keep open. Current main and the latest release have partial raw `<tg-emoji>` HTML preservation, but the central custom-emoji entity path is still missing from the shared message tool, Telegram outbound payload handling, send/edit/caption delivery, and regression coverage.

## What This Changes

Add first-class Telegram custom emoji support for messages, captions, and replies, including identifiers and Bot API formatting/entity behavior.

## Best Possible Solution

Choose the API representation first, then implement validated Telegram custom emoji entities through the shared message tool, Telegram payload/channelData handling, text/edit/caption delivery, and focused tests while preserving existing raw HTML fallback behavior.

## Reproduction Assessment

Yes from source, but not by a live Telegram send in this read-only review. The message tool has no entity/custom-emoji schema, Telegram send options do not accept entity arrays, and the adapter only forwards buttons/quote text from Telegram channel data.

## Solution Assessment

Unclear until maintainers choose the API shape. Preserving raw `<tg-emoji>` HTML is useful but not the best complete solution for the reported message/caption/reply entity path; a typed representation with delivery validation is safer.
