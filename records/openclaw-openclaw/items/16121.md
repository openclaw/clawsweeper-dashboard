## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages, captions, edits/previews, and the shared message tool, and the related reaction-only pull request does not cover this issue's message/caption/reply scope.

## What This Changes

The issue requests Telegram custom emoji support in messages, captions, and replies, including custom emoji identifiers and rendering/sending behavior.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then implement validated Telegram support across send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused docs and regression tests.
