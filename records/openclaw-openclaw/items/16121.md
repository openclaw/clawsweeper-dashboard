## Summary

Keep open. Current main still lacks first-class outgoing Telegram custom emoji support for messages, captions, edits/previews, and the shared message tool, while the related reaction-only pull request and closed entity issue do not cover this issue's message/caption/reply scope.

## What This Changes

Add Telegram custom emoji support for messages, captions, and replies, including custom emoji identifiers and a usable agent-facing send surface.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then implement validated Telegram support across send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused docs and regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse mode.

## Solution Assessment

Unclear. The requested support is valid, but raw tg-emoji passthrough alone would not cover message-tool input, captions, edits/previews, chunking, fallback, or UTF-16 entity validation.
