## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages, captions, replies, edits/previews, and the shared message tool; #13905 was closed unmerged, #40848 is closed but the broader behavior remains absent, and #47562 is an open reaction-focused PR that does not cover the message/caption path.

## What This Changes

Add Telegram custom emoji support for messages, captions, replies, identifiers, and rendering according to the current Bot API formatting surfaces.

## Best Possible Solution

Define a small public custom-emoji/entity representation, then implement validated Telegram support through send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level missing-capability check exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/delivery paths still rely on HTML parse_mode.

## Solution Assessment

Unclear. A raw tg-emoji passthrough would only cover one HTML path; the maintainable solution needs a chosen representation that works across message-tool input, captions, edits, draft streaming, chunking, and UTF-16 entity validation.
