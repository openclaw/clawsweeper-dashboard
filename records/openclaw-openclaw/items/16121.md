## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages, captions, replies, edits/previews, and the shared message tool; the related closed issue and unmerged PR are useful prior art but do not implement the remaining product behavior.

## What This Changes

Add Telegram custom emoji support for messages, captions, and replies, including custom emoji identifiers and rendering/sending behavior.

## Best Possible Solution

Define a small public custom-emoji/entity representation, then implement validated Telegram support through send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode.

## Solution Assessment

Unclear. A raw <tg-emoji> passthrough would only cover one HTML path and would not solve message-tool input, captions, edits, draft streaming, chunking, or UTF-16 entity validation.
