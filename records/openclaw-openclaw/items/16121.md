## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages, captions, edits, draft previews, and the shared message tool; #40848 was closed as the duplicate of this tracker, while open PR #47562 only targets custom emoji reactions.

## What This Changes

Request to add Telegram custom emoji handling for messages, captions, replies, identifiers, and outgoing display according to the Telegram Bot API formatting/entity contract.

## Best Possible Solution

Define a small public custom-emoji/entity representation, then implement validated Telegram support through send, caption, reply, edit, delivery, draft streaming, and the shared message tool, with docs and focused regression coverage that preserves current HTML fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode.

## Solution Assessment

Unclear. The feature direction is valid, but a raw <tg-emoji> passthrough alone would not cover message-tool input, captions, edits, draft streaming, chunking, or UTF-16 entity offset validation.
