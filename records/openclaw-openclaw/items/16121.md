## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages/captions/replies and the shared message tool, while the related open pull request is reaction-only and the closed entity issue points back here as the canonical tracker.

## What This Changes

Add Telegram custom emoji support for messages, captions, and replies, including custom emoji identifiers and a way for agents to send them.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then wire validated Telegram support through send, caption, reply, edit, delivery, draft streaming, and the shared message tool with docs and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested custom-emoji/entity send surface.

## Solution Assessment

Unclear. The requested direction is valid, but raw tg-emoji passthrough alone would not cover message-tool input, captions, edits/previews, chunking, fallback, or UTF-16 entity validation.
