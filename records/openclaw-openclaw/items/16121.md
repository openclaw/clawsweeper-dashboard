## Summary

Keep open. Current main and v2026.6.5 now preserve supported Telegram HTML tags including `<tg-emoji emoji-id="...">`, but the broader request still lacks a first-class custom emoji contract for the shared message tool, Markdown IR, outgoing entities/caption entities, and reply/caption/edit paths.

## What This Changes

Request support for Telegram custom emoji identifiers in outgoing messages, captions, and replies according to the Telegram Bot API formatting contract.

## Best Possible Solution

Define a maintainer-approved Telegram custom emoji representation, then implement it in the bundled Telegram plugin and shared message tool while preserving the supported-HTML fallback across messages, captions, replies, edits, docs, and focused tests.

## Reproduction Assessment

Yes at the source level. Current main and v2026.6.5 show the raw supported-HTML path is partially present, while the message tool, Markdown IR, and Telegram send option surfaces still lack a first-class custom emoji/entity contract.

## Solution Assessment

No, the existing allowlist-only path is not the complete solution. It is useful partial support, but the best path needs a maintainer-selected representation and consistent wiring across text, captions, replies, edits, and fallback behavior.
