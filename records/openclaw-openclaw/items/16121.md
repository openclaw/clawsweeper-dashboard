## Summary

Keep open. Current main still lacks first-class outgoing Telegram custom emoji support for messages, captions, replies, edits, and the shared message tool; the related open PR is reaction-only, and the related entity issue was closed back to this issue as the canonical tracker.

## What This Changes

Request support for Telegram custom emoji in OpenClaw Telegram messages, captions, and replies according to Telegram Bot API custom emoji formatting and entity semantics.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then wire validated Telegram support through send, captions, replies, edits/previews, delivery/streaming, and the shared message tool with focused docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and dependency inspection on current main gives high confidence that the requested custom-emoji/entity send surface is still absent.

## Solution Assessment

Unclear. The requested capability is valid, but raw `<tg-emoji>` passthrough alone would not cover message-tool input, captions, replies, edits/previews, chunking, fallback, or entity validation.
