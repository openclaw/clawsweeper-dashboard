## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for outgoing messages, captions, edits/previews, and the shared message tool, and recent maintainer/Codex discussion explicitly kept this as the canonical tracker rather than closing it behind reaction-only or parse-mode related work.

## What This Changes

Request support for Telegram custom emojis in messages, captions, replies, and agent-facing Telegram send paths.

## Best Possible Solution

Define a stable custom-emoji/entity representation, then implement validated Telegram support across send, caption, reply, edit, delivery, draft streaming, and the shared message tool with focused docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows no custom-emoji IR span, no outgoing entities/caption_entities send surface, and Telegram send/edit/delivery paths still rely on HTML parse mode.

## Solution Assessment

Unclear. The requested support is valid, but raw tg-emoji passthrough alone would not cover message-tool input, captions, edits/previews, chunking, fallback, or UTF-16 entity validation.
