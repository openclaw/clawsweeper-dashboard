Full review comments:

- **[P1] Keep operator system events actionable:** `src/gateway/server-methods/system.ts:154`
  - body: This handler backs the operator-admin `system-event` method, and the CLI docs promise those events are injected as `System:` prompt lines. Adding `quarantineInPrompt: true` here moves authorized admin text under the untrusted metadata header, so workflows like `openclaw system event --text "Check for urgent follow-ups" --mode now` stop being actionable unless maintainers deliberately redefine that contract.
  - confidence: 0.91
- **[P1] Classify plugin producers before defaulting to trusted:** `src/auto-reply/reply/session-system-events.ts:143`
  - body: The new drain logic treats every unflagged event as actionable, but bundled channel producers such as Slack and Microsoft Teams still enqueue user-controlled event text through the plugin runtime without the new flag. That leaves the prompt-injection surface open for those system events after this PR; update the producer classification or make external plugin events safe by default.
  - confidence: 0.89
- **[P1] Settle the plugin-facing quarantine contract:** `src/infra/system-events.ts:47`
  - body: `enqueueSystemEvent` is exposed through the plugin runtime, so adding `quarantineInPrompt` creates a plugin-visible trust option. Without documented semantics and version-skew behavior, external plugins cannot know which events should become non-actionable or what happens on older hosts that ignore the option.
  - confidence: 0.86
