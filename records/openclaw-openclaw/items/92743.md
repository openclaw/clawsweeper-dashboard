## Summary

Close as external ownership: the reported hotfix targets `openclaw-hy-memory`, which is not in this source tree, while current OpenClaw already exposes the plugin hook data and non-blocking session-end behavior needed for that plugin to fix itself.

## What This Changes

The issue asks HY Memory AutoRecall to query only the current user request instead of the full OpenClaw prompt envelope, and to avoid blocking session completion during session-end capture.

## Best Possible Solution

Keep OpenClaw core unchanged and patch/publish the HY Memory plugin against the existing hook contract; reopen or file a narrower issue only if that plugin proves it needs a new SDK field.

## Reproduction Assessment

No; there is no high-confidence current-main reproduction path inside this repository because the named `openclaw-hy-memory/dist/index.js` file is not present. The issue logs are useful evidence for the external plugin owner.

## Solution Assessment

No; changing OpenClaw core is not the best fix when the current hook contract already supplies `prompt` and `messages`. The maintainable fix is for HY Memory to derive a bounded query in its plugin code, matching the bundled memory plugin pattern.
