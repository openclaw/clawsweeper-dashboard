## Summary

Keep open: the PR addresses a real, source-reproducible message-loss bug, but the new session-key parser misparses supported multi-account DM session keys and the supplied proof does not exercise the patched production path.

## What This Changes

This PR adds a guarded session-key-derived fallback in `src/agents/tools/message-tool.ts` so stale webchat current-channel context can be replaced with non-webchat delivery context for schema, description, secret scope, and runtime tool context.

## Best Possible Solution

Keep the narrow session-key fallback direction, but parse canonical session-key shapes explicitly, preserve the real peer target for per-account keys, and add regression coverage for stale webchat context across normal and per-account DM sessions.

## Reproduction Assessment

Yes, at source level: current main still passes raw stale `currentChannelProvider: webchat` into the message tool runtime context, and the linked issue review traces how that can select the internal UI sink before channel fallback. I did not run a live Signal or Telegram gateway reproduction in this read-only pass.

## Solution Assessment

No, not in its current form: the guarded fallback is the right narrow direction, but the parser must follow the documented session-key grammar instead of taking the first non-marker segment as the peer. The safer fix is to parse `channel[:account]:direct:<peer>` and related supported shapes with focused tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Parse account-scoped session keys before inferring the target:** `src/agents/tools/message-tool.ts:829`
  - body: For a supported key like `agent:main:telegram:account-a:direct:6812765697`, this loop returns `account-a` as `currentChannelId` because it takes the first non-marker segment after the channel. In the stale-webchat case the patched fallback would then send to the account id instead of the DM peer, so multi-account direct sessions remain broken or misrouted.
  - confidence: 0.92
