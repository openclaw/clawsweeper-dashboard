Keep open: current main and the latest release still allow untimestamped transcript usage to enter bounded dashboard/session totals, and an open linked closing pull request is already the active fix path.

## What This Changes

The issue reports that the Usage dashboard's selected-day model call totals show historical cumulative usage instead of usage for the selected date after upgrading to v2026.5.28.

## Best Possible Solution

Land or replace the linked backend usage-range fix so bounded dashboard usage excludes untimestamped historical rows while all-time summaries remain intact.

## Reproduction Assessment

Yes, at source level: current main still applies bounded range filtering only when a usage row has a timestamp, so untimestamped rows can enter selected-day summaries. I did not run a live WSL2 dashboard profile reproduction in this read-only review.

## Solution Assessment

Yes: the durable fix belongs in the shared backend/cache aggregation path rather than UI-only filtering, and the open linked pull request is the current candidate for that path.
