The issue asks for Dashboard Usage daily model-call totals to respect the selected date range instead of showing historical cumulative usage after the v2026.5.28 upgrade.

## Best Possible Solution

Land or replace the linked backend usage-range fix so bounded dashboard/session usage excludes untimestamped historical rows while all-time summaries remain intact.

## Solution Assessment
