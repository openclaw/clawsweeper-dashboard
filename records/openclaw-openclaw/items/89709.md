## Summary

Keep open: current main and v2026.6.9 still allow untimestamped usage rows to enter bounded usage summaries, and https://github.com/openclaw/openclaw/pull/89745 is the active but unmerged fix candidate.

## What This Changes

The issue asks for Dashboard Usage selected-date model calls to show only usage from the selected day instead of historical cumulative totals after the v2026.5.28 upgrade.

## Best Possible Solution

Land or replace the backend usage-range fix so bounded dashboard ranges exclude untimestamped historical rows across single, direct, and batched summary paths while preserving all-time summaries.

## Reproduction Assessment

Yes. Source inspection shows current main filters out-of-range usage only when a row has a timestamp, so untimestamped rows can still contribute to selected-day totals.

## Solution Assessment

Yes. The durable fix belongs in the shared backend/cache aggregation path rather than UI-only filtering; the linked PR is the active candidate but still needs to cover the full summary surface before the issue can close.
