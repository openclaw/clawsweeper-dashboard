## What This Changes

Adds an `openclaw sessions tail` subcommand and tests for rendering redacted session trajectory JSONL progress with session selection, tail count, follow, store, agent, and all-agent options.

## Best Possible Solution

Land a narrow sessions progress reader after preserving follow-mode event continuity, keeping the conservative preview policy, and getting maintainer confirmation on the CLI surface.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR renders the initial tail before initializing follow offsets from the current file size, so an event appended during that handoff is skipped. I did not run tests or the command because this review is read-only.

## Solution Assessment

No: the feature direction is plausible, but the follow handoff should carry forward the consumed byte offset or otherwise drain the gap before polling. Maintainer command-shape and redaction approval should remain separate from the mechanical follow fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve events appended while follow starts:** `src/commands/sessions-tail.ts:286`
  - body: `sessions tail --follow` renders the initial tail first, then `followSelections` snapshots `statFileSize(...)` as the starting offset. If the trajectory file receives a line between those two steps, that line is neither in the already-rendered tail nor in future follow reads, so the operator view silently drops progress. Please initialize follow from the byte offset that was actually consumed by the initial render, or otherwise drain the gap before entering the interval loop.
  - confidence: 0.86
