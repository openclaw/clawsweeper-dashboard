## Summary

Keep this PR open for normal review: it targets a real Control UI context-window problem, but the branch still has blocking correctness issues and no after-fix real behavior proof, so a cleanup close is not justified.

## What This Changes

This PR populates the gateway model-catalog context-window cache, adds default context-token fallbacks in session row construction, and adds gateway catalog cache tests.

## Best Possible Solution

Rebase and rework the PR so session rows get exact context tokens from persisted runtime metadata or an exact catalog lookup without default guesses, cache-key collisions, or stale global test state.

## Reproduction Assessment

Yes, source-level. Current main shows the Control UI reads activeSession.contextTokens for the meter, while the async sessions.list path can omit derived contextTokens in lightweight rows unless the session entry already persisted them.

## Solution Assessment

No. Catalog-backed resolution may be viable, but this PR's current implementation guesses 200k for unknown windows, mixes raw and qualified cache keys, and does not align cleanly with current lightweight session-list behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid guessing 200k for uncached session rows:** `src/gateway/session-utils.ts:1358`
  - body: When the cache has no exact window, this fallback makes discovery-only models report DEFAULT_CONTEXT_TOKENS instead of unknown. Models with smaller or larger windows will display a confident but wrong context percentage before an exact catalog value is available.
  - confidence: 0.9
- **[P2] Keep slash model IDs out of the qualified cache keyspace:** `src/gateway/server-model-catalog.ts:33-40`
  - body: Writing raw entry.id values and synthetic provider/id values into the same cache lets raw IDs like google/gemini-2.5-pro collide with native provider-qualified lookups. Because the merge keeps the minimum, this can underreport native provider context windows.
  - confidence: 0.87
- **[P2] Clear context-window cache when resetting catalog state:** `src/gateway/server-model-catalog.ts:46`
  - body: loadGatewayModelCatalog now mutates MODEL_CONTEXT_TOKEN_CACHE, but the test reset helper still resets only the model-catalog cache. Tests that reset the catalog can leak discovered windows across cases and become order-dependent.
  - confidence: 0.84
