## Summary

Keep this PR open, but do not merge it as-is: the bug is plausible and source-reproducible, yet the current branch is conflicting, lacks real Anthropic/Control UI proof, and introduces three concrete correctness issues in the session context-token path.

## What This Changes

The PR populates gateway model-catalog context windows into the shared context-token cache, adds default context-token fallbacks in session row construction, and adds gateway catalog-cache tests.

## Best Possible Solution

Rework the PR so session rows receive exact context tokens from persisted runtime metadata or an exact catalog lookup without default guesses, cache-key collisions, or stale global test state, then add redacted real Anthropic Control UI proof.

## Reproduction Assessment

Yes, source-level. Current main shows the UI consumes activeSession.contextTokens for the meter and the sessions.list lightweight path can omit derived contextTokens unless the stored entry already has them; no live Anthropic reproduction was run in this read-only review.

## Solution Assessment

No. Catalog-backed resolution may be a viable fix direction, but this patch guesses a default window, mixes cache key semantics, and misses the reset path for its new global side effect.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid guessing 200k for uncached session rows:** `src/gateway/session-utils.ts:1358`
  - body: When the cache has no exact window, this fallback makes discovery-only models report DEFAULT_CONTEXT_TOKENS instead of unknown. Models with smaller or larger windows will display a confident but wrong context percentage before an exact catalog value is available.
  - confidence: 0.9
- **[P2] Keep slash model IDs out of the qualified cache keyspace:** `src/gateway/server-model-catalog.ts:33-40`
  - body: Writing raw entry.id values and synthetic provider/id values into the same cache lets raw IDs like google/gemini-2.5-pro collide with native provider-qualified lookups. Because the merge keeps the minimum, this can underreport native provider context windows.
  - confidence: 0.87
- **[P2] Clear context-window cache when resetting catalog state:** `src/gateway/server-model-catalog.ts:46`
  - body: loadGatewayModelCatalog now mutates MODEL_CONTEXT_TOKEN_CACHE, but the test reset helper still resets only the model-catalog cache. Tests that reset the catalog can leak discovered windows across cases and become order-dependent.
  - confidence: 0.84
