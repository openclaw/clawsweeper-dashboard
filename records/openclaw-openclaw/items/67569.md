## Summary

Keep open: the report and discussion describe a real Control UI context-meter gap, but this PR still has correctness blockers and only test/check proof rather than after-fix real behavior proof.

## What This Changes

The PR populates the gateway context-window cache from model catalog entries, adds provider-qualified cache keys, adds default context-token fallbacks in session row builders, and adds gateway catalog cache tests.

## Best Possible Solution

Rework against current main so `sessions.list` supplies exact context tokens from persisted runtime metadata or an exact catalog lookup without default guesses, shared-key collisions, or stale global cache state.

## Reproduction Assessment

Yes, source-level. The Control UI chat takes `activeSession.contextTokens` from `sessions.list`, and current async list rows can omit derived context tokens when the session entry lacks a persisted value.

## Solution Assessment

No. Catalog-backed resolution is a plausible direction, but this patch guesses 200k for unknown windows, collides raw slash model IDs with qualified keys, and does not account for the current lightweight session-list path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid guessing 200k for uncached session rows:** `src/gateway/session-utils.ts:1355-1360`
  - body: When the catalog/cache has no exact window, this fallback makes every discovery-only model report a 200k limit. Models with smaller or larger windows will show a confident but wrong percentage instead of leaving the context window unknown until it can be resolved.
  - confidence: 0.9
- **[P2] Keep slash model IDs out of the qualified cache keyspace:** `src/gateway/server-model-catalog.ts:31-38`
  - body: Writing raw `entry.id` and synthetic `provider/id` keys into the same cache lets a raw catalog ID such as `google/gemini-2.5-pro` collide with the native Google qualified key. Because the cache merge keeps the minimum, this can underreport native provider context windows in session rows.
  - confidence: 0.87
- **[P2] Clear context-window cache when resetting catalog state:** `src/gateway/server-model-catalog.ts:40`
  - body: `loadGatewayModelCatalog` gains a global side effect by writing `MODEL_CONTEXT_TOKEN_CACHE`, but the catalog reset helper still resets only model-catalog state. Gateway tests that reset the catalog can leak discovered context windows across cases and become order-dependent.
  - confidence: 0.84
