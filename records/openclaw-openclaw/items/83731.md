## Summary

Keep open: current main and the latest release still persist automatic fallback model pins, no requested config flag exists, and the linked PRs are partial or alternative policy work rather than a shipped resolution.

## What This Changes

The issue requests a `gateway.routing.persist_auto_fallback` config flag so automatic fallback selection can be used for one turn without persisting `modelOverrideSource: "auto"` in session state.

## Best Possible Solution

If maintainers want this operator policy, add a typed config option with schema/help/docs and tests that keep current stickiness as the default while making automatic fallback persistence opt-out; otherwise make the existing probe/reset/doctor recovery policy explicit and use the related PRs for narrower cleanup.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run in this read-only review. Source and shipped-release inspection do reproduce the core condition: auto fallback selections are persisted with `modelOverrideSource: "auto"`, the primary probe is throttled, and no no-persist config key exists.

## Solution Assessment

Unclear: the requested flag is plausible and clean for operators who want primary-or-explicit behavior, but it is a new routing policy knob. Maintainers should decide whether this is preferable to the open TTL approach at https://github.com/openclaw/openclaw/pull/79678 or the narrower cleanup work at https://github.com/openclaw/openclaw/pull/84476.
