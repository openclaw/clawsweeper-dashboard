## Summary

Keep open: current main still persists auto fallback overrides, has no supported `gateway.routing` config key for this policy, and the linked open PRs are partial or alternative policy work rather than a merged replacement.

## What This Changes

Request to add `gateway.routing.persist_auto_fallback` so automatic fallback can be used for the current turn without persisting auto model override state.

## Best Possible Solution

If maintainers accept this operator policy, add a typed config option with default-compatible behavior plus schema, help, docs, and regression tests; otherwise pick the TTL or ephemeral-cleanup direction and document that no-persist fallback is not supported.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection and tests do show the core precondition: auto fallback selections persist with `modelOverrideSource: "auto"` and are reused until reset, validation, doctor cleanup, or a successful primary probe clears them.

## Solution Assessment

Unclear: the requested flag is plausible and matches the operator use case, but it is a new routing policy knob rather than the only maintainable fix. Maintainers should choose between opt-out persistence, TTL policy in https://github.com/openclaw/openclaw/pull/79678, and the narrower cleanup path in https://github.com/openclaw/openclaw/pull/84476.
