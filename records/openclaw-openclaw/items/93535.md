## What This Changes

The PR changes `src/agents/embedded-agent-runner/run.ts` so cron-triggered embedded runs emit `before_agent_reply` before checking whether a `before_agent_reply` hook is registered.

## Best Possible Solution

Repair the runner/watchdog phase contract so no-hook cron progress avoids the false abort while true startup stalls still use the pre-execution guard, then add focused coverage and redacted real cron proof.

## Reproduction Assessment

Yes, from source inspection. Current main gates `before_agent_reply` on hook registration, the cron watchdog clears only on execution-stage progress or `firstModelCallStarted`, and the linked issue describes the no-hook isolated cron abort path.

## Solution Assessment

No. The runner-seam change is plausible, but as written it clears the watchdog before the no-hook setup path is protected again; the safer fix must preserve true-stall detection or move the behavior into the watchdog contract.
