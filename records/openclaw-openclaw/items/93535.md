## What This Changes

The PR changes cron-triggered embedded agent execution to emit `before_agent_reply` before checking whether a `before_agent_reply` hook is registered.

## Best Possible Solution

Emit `before_agent_reply` for cron no-hook runs while preserving the existing setup-phase re-arm contract, then add focused regression coverage and real cron isolated-agent proof before merge.

## Reproduction Assessment

Yes from source. `executeJobCoreWithTimeout` feeds isolated cron phases into `createCronAgentWatchdog`, current `runEmbeddedAgent` skips `before_agent_reply` when no hook is registered, and the watchdog only clears the pre-execution timer on execution-stage phases or `firstModelCallStarted`.

## Solution Assessment

No. The runner-seam fix is plausible, but the current diff leaves the follow-up `runtime_plugins` re-arm signal inside the hook-present branch and does not prove the no-hook cron path with a focused regression or real behavior proof.
