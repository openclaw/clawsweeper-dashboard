## Summary

## What This Changes

The PR changes `src/agents/embedded-agent-runner/run.ts` so cron-triggered embedded runs emit `before_agent_reply` even when no `before_agent_reply` hook is registered.

## Best Possible Solution

Implement one canonical cron runner/watchdog phase contract that prevents no-hook false aborts while preserving the 60-second guard for true startup stalls, then add focused regression coverage and redacted real cron proof.

## Reproduction Assessment

Yes. Source inspection shows current main gates `before_agent_reply` on hook registration, while the cron watchdog only clears its pre-execution timer on execution-stage progress or `firstModelCallStarted`.

## Solution Assessment

No. The PR is a plausible mitigation for the no-hook symptom, but the best fix must preserve true startup-stall detection instead of clearing the guard before later startup progress is emitted.
