## Summary

Keep open: this is a plausible core feature, but it is not obsolete and the current branch is not mergeable as-is. The open related discussions still have unsettled contract/security questions, and the patch diverges from current skill allowlist, PI package, read-path, and provider-stream contracts.

## What This Changes

The branch adds SKILL.md `model`/`modelProfile` metadata parsing, a skill-to-model map, read-tool and stream wrappers, cached SkillEntry rebuilding, and model-router tests.

## Best Possible Solution

Settle the public skill-routing contract first, then land or replace this branch with an implementation that preserves allowlists, uses current provider/model selection helpers, normalizes paths, documents consent and precedence, and includes focused tests plus runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection. Current main threads `agentId` through skill loading, uses `@earendil-works/pi-*`, recognizes `file_path`, and prepares streams from the original provider/model while this PR diverges from those contracts.

## Solution Assessment

No, not as currently proposed. The direction is useful, but the implementation needs maintainer-approved security/precedence semantics plus a rebase onto current allowlist, provider-routing, and path-normalization contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve agent skill allowlists when loading entries:** `src/agents/pi-embedded-runner/skills-runtime.ts:59`
  - body: Current main passes `agentId` into `resolveEmbeddedRunSkillEntries()` and on to `loadWorkspaceSkillEntries()`. This branch's helper has no `agentId` parameter and calls `loadWorkspaceSkillEntries(params.workspaceDir, { config })`, so per-agent skill allowlists are not applied to prompt entries or model-routing metadata.
  - confidence: 0.93
- **[P1] Rebase imports onto the current PI package scope:** `src/agents/skills/model-router.ts:2-4`
  - body: Current `package.json` declares `@earendil-works/pi-*`, and current agent code imports that scope. The new router and tests import `@mariozechner/pi-*`, leaving undeclared modules on current main and breaking the TypeScript build.
  - confidence: 0.9
- **[P2] Handle file_path read arguments:** `src/agents/skills/model-router.ts:164-171`
  - body: The wrapper runs before the underlying read tool can normalize aliases, but existing OpenClaw read callers use the `file_path` alias. Reads emitted as `{ file_path: ... }` leave `rawFilePath` empty here, so the skill model override never activates.
  - confidence: 0.86
- **[P2] Expand home-relative skill paths before lookup:** `src/agents/skills/model-router.ts:177-178`
  - body: Skill prompt locations can be compacted to `~/.../SKILL.md`, while `skillModelMap` is keyed by absolute paths. `path.resolve('~/...')` does not expand the user's home directory, so home-installed skills can miss the override map and silently use the session model.
  - confidence: 0.87
- **[P2] Resolve a stream for the routed provider:** `src/agents/skills/model-router.ts:210`
  - body: The wrapper substitutes only the `Model` argument. The stream function, auth lookup, transport, extra params, and provider transforms were already prepared for the original `params.model`, so routing a skill to another provider can still use the original provider path and fail or send misleading requests.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention: metadata-driven model/provider selection from untrusted skills requires explicit consent, allowlist preservation, and provider/auth-safe routing before merge.
