## Summary

Keep open: current main still lacks a per-skill model-routing contract, but this PR is not mergeable as proposed because the branch is stale and the implementation has provider-routing, agent-scope, consent/allowlist, and proof blockers.

## What This Changes

The PR adds SKILL.md `model` and `modelProfile` metadata, builds a per-skill model map, wraps the read tool and stream function for model overrides, reconstructs cached skill entries, and adds router unit tests.

## Best Possible Solution

Settle the public per-skill routing contract first, then rebase or replace this PR with an early provider-aware, allowlist-safe implementation, docs, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes, for the PR review findings via source inspection: current main and the PR diff show the stale imports, omitted agentId, late model substitution, and path-lookup gaps. The underlying request is a feature, not a failing current-main bug.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the patch routes by swapping a late model argument and lacks maintainer-approved consent, precedence, fallback, and allowlist semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current PI package scope:** `src/agents/skills/model-router.ts:2-4`
  - body: Current main depends on `@earendil-works/pi-*`, but the new router imports `@mariozechner/pi-*`. A rebased branch will fail to resolve these imports until they use the current package scope.
  - confidence: 0.95
- **[P1] Preserve agent-scoped skill loading:** `src/agents/pi-embedded-runner/skills-runtime.ts:59`
  - body: Current main passes `agentId` into `loadWorkspaceSkillEntries` so per-agent skill filters shape runtime entries. This branch omits that parameter, so the routing map can include model metadata from skills outside the active agent's allowlist.
  - confidence: 0.9
- **[P2] Route through the selected provider runtime:** `src/agents/skills/model-router.ts:210`
  - body: The router substitutes only the `Model` argument after stream setup, auth lookup, transport, provider transforms, and extra params have already been prepared for the original model. Cross-provider skill routing can therefore use the wrong transport/auth path or fail despite logging an override.
  - confidence: 0.88
- **[P2] Normalize read paths before routing lookup:** `src/agents/skills/model-router.ts:164-178`
  - body: The read interceptor runs before the underlying path handling but ignores the `file_path` alias and resolves `~/...` relative to cwd. Skill reads using current prompt/read conventions can miss the map and never activate the override.
  - confidence: 0.86
