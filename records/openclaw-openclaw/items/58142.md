## Summary

Keep this PR open. Current main still lacks per-skill model metadata and runtime model switching, and the related feature issue remains open; however, the PR is not merge-ready because the branch drops current-main agent skill allowlist plumbing and introduces provider/model switching from untrusted skill metadata without a settled security and precedence contract.

## What This Changes

The PR adds `model` and `modelProfile` skill metadata parsing, a skill-aware read-tool and stream-function router, cached-snapshot SkillEntry reconstruction, and focused model-router tests.

## Best Possible Solution

Define the skill model-routing contract first, then land a rebased implementation that preserves agent skill allowlists and snapshot semantics, applies routing at a well-defined skill boundary, documents privacy/cost/precedence behavior, and covers parser, cached-snapshot, allowlist, fallback, and end-to-end routing cases.

## Reproduction Assessment

Yes. The current-main gap is reproducible by inspecting the skill metadata type/parser and absence of model-router code; the PR regression is reproducible by comparing current main's `agentId` threading with the PR head's `skills-runtime.ts`.

## Solution Assessment

No, not as currently proposed. The implementation is a useful candidate, but it needs the allowlist regression fixed and a maintainer-approved routing/security contract before it is the narrowest maintainable solution.
