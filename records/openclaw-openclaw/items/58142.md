## Summary

Keep open: the feature is plausible and has a live canonical issue, but this PR is not mergeable against current main due to stale dependency imports, stale skill-loading plumbing, provider/auth routing problems, and unresolved consent/precedence policy.

## What This Changes

The branch adds SKILL.md `model`/`modelProfile` metadata parsing, builds a skill-to-model map, wraps the read tool and stream function to switch models during a run, and adds router unit tests.

## Best Possible Solution

Settle the skill-routing contract first, then land or replace this branch with an implementation that uses current skill allowlists, current provider/model resolution, explicit consent and precedence rules, public docs, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection. Current main uses `@earendil-works/pi-*`, threads `agentId` through skill loading, and prepares stream/auth state from `params.model`, while the PR diverges from those contracts.

## Solution Assessment

No, not as currently proposed. The direction is useful, but the implementation needs a rebase plus maintainer-approved consent, precedence, fallback, and provider-routing semantics before it is the maintainable fix.

  - body: Current main declares and imports `@earendil-works/pi-*`, but the new router and tests import `@mariozechner/pi-*`. Those modules are not declared in current `package.json`, so a rebased branch will fail to build until these imports use the current scope.

This branch's changed load path calls `loadWorkspaceSkillEntries(params.workspaceDir, { config })`, which would expose prompt/model metadata outside the active agent's allowlist after rebase.
  - confidence: 0.9
- **[P2] Resolve a stream for the routed provider:** `src/agents/skills/model-router.ts:211`
  - body: The router substitutes only the `Model` argument after run setup has already prepared the stream function, auth provider id, extra params, and provider transforms for `params.model`. Routing a skill to another provider, which the PR explicitly targets, can therefore use the wrong transport/auth path or fail despite logging a model override.
  - confidence: 0.87
