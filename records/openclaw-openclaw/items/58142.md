## Summary

Keep this PR open, but it is not merge-ready: the feature is still useful and has active linked demand, while the branch is stale against current main and introduces routing, allowlist, path-normalization, and consent/security issues that need maintainer review.

## What This Changes

The branch adds SKILL.md `model`/`modelProfile` metadata parsing, read/stream wrappers for skill-selected models, cached-snapshot SkillEntry rebuilding, and router tests.

## Best Possible Solution

Define the public skill-routing contract first, then land a rebased implementation that preserves allowlists, uses current provider/model selection helpers, normalizes skill paths, documents consent and precedence, and includes focused tests plus real runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection. Current main threads `agentId` through skill loading, compacts home skill paths, recognizes `file_path`, and prepares provider streams from the original model, while PR head diverges from those contracts.

## Solution Assessment

No, not as currently proposed. The direction is useful, but the implementation needs a maintainer-approved security/precedence contract plus rebase, allowlist preservation, provider-aware routing, and path normalization before it is the narrowest maintainable solution.

  - body: The wrapper runs before the underlying read tool gets a chance to normalize arguments, and current embedded tooling already treats `file_path` as a read path alias. Calls using that alias leave `rawFilePath` empty here, so the skill model override never activates.
  - confidence: 0.84

  - confidence: 0.89
- **[P2] Resolve a stream for the routed provider:** `src/agents/skills/model-router.ts:210`
  - body: The wrapper substitutes only the `Model` argument, but the active stream and auth wrapper were already prepared for the original `params.model`.   - confidence: 0.86
