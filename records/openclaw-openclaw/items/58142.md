## Summary

Keep open. Current main still lacks per-skill model routing, and this PR remains a plausible implementation candidate, but it needs a rebase plus correctness, security, and product-contract fixes before merge.

## What This Changes

The PR adds SKILL.md `model`/`modelProfile` metadata parsing, runtime read/stream wrappers for skill-selected models, cached-snapshot SkillEntry reconstruction, and router tests.

## Best Possible Solution

Define the public per-skill routing contract first, then land a rebased implementation that preserves allowlists, uses current provider/model selection helpers, normalizes skill paths consistently, documents consent and precedence, and includes focused tests plus real runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection. Current main threads `sessionAgentId` through skill loading and compacts home skill paths, while PR head drops the agent ID, compares raw `~/...` reads with `path.resolve()`, and wraps a stream selected for the original provider.

## Solution Assessment

No, not as currently proposed. The feature direction is useful, but the PR needs rebase work, allowlist preservation, provider-aware routing, home-path normalization, and a maintainer-approved security/precedence contract before it is the narrowest maintainable solution.
