## Summary

Keep this PR open, but do not merge as-is. The feature is still useful and tied to open canonical requests, but the branch is stale against current main, has blocking runtime/build issues, needs a maintainer-approved model-routing security contract, and needs stronger real behavior proof.

## What This Changes

The PR adds SKILL.md `model`/`modelProfile` metadata parsing, cached-snapshot skill entry reconstruction, a read-tool detector, a stream wrapper for per-skill model routing, and router unit tests.

## Best Possible Solution

Define the public per-skill routing contract first, then land a rebased implementation that preserves agent skill filters, resolves provider/auth/transport for the routed model, normalizes prompt paths consistently, documents precedence and consent, and includes real runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection. Current main threads `agentId` through skill loading, compacts prompt skill paths, and resolves streams/auth for the session model, while the PR branch omits or bypasses those current-main contracts.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but this implementation needs rebase work, provider-aware stream resolution, path normalization, allowlist preservation, and a maintainer-approved security/precedence contract before it is the narrowest maintainable solution.
