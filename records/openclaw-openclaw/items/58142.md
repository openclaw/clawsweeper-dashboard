## Summary

Keep open. Current main still lacks per-skill model routing, and this PR is a useful implementation candidate, but the PR head regresses agent skill allowlists and needs a settled public/security contract before merge.

## What This Changes

The PR adds `model` and `modelProfile` SKILL.md metadata parsing, a skill read-tool detector, a stream-function model router, cached-snapshot SkillEntry reconstruction, and focused router tests.

## Best Possible Solution

Define the skill model-routing contract first, then land a rebased implementation that preserves agent skill allowlists and snapshot semantics, documents privacy/cost/precedence behavior, and covers parser, cached-snapshot, fallback, allowlist, and routing cases.

## Reproduction Assessment

Yes. The current-main feature gap is reproducible by inspecting the skill metadata type/parser and absence of model-router code; the PR regression is reproducible by comparing current main's `agentId` threading with the PR diff's `skills-runtime.ts`.

## Solution Assessment

No, not as currently proposed. The approach is directionally useful, but it needs the allowlist regression fixed, a changelog entry, and maintainer-approved routing/security semantics before it is the narrowest maintainable solution.

## Security Review

Status: needs_attention

Summary: The diff introduces metadata-driven provider/model switching for untrusted SKILL.md content without a settled permission, precedence, or audit policy.

Concerns:

- **[medium] Gate skill-declared model routing:** `src/agents/skills/model-router.ts:210`
  - body: `wrapStreamFnSkillModelRouter` substitutes `ctx.activeModel` for later model calls once a skill read sets it. A third-party or workspace skill could silently move prompt content to another configured provider or a more expensive model unless OpenClaw defines explicit precedence, consent, and observability for this public metadata contract.
  - confidence: 0.78
