## What This Changes

The issue asks for secret-safe warning logs and branch-specific reasons when hook gateway requests return non-2xx validation responses.

## Best Possible Solution

Add centralized, secret-safe hook rejection diagnostics in the hook request handler, with focused regression tests for body-read, direct-hook, and mapped-hook rejects.

## Solution Assessment

Yes. Handler-level rejection logging with stable reason codes is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
