## Summary

Keep open. Current main and the latest release still return several hook rejection responses without a warning journal entry, and the only related fix PR was closed unmerged.

## What This Changes

The issue requests warning-level gateway journal entries and more specific rejection reasons for hook POST requests that return non-2xx validation errors.

## Best Possible Solution

Add a small, secret-safe hook rejection logging helper in the gateway hook request handler and cover the main rejection branches with focused regression tests.

## Reproduction Assessment

Yes. The malformed JSON path reaches `readJsonBody`, immediately sends a 400/408/413 response, and returns without `logHooks.warn`; adjacent tests already exercise the response status paths but not journal diagnostics.

## Solution Assessment

Yes. Handler-level rejection diagnostics with tests is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected ingress requests.
