## Summary

Keep open. Current main still returns hook body-read and validation 4xx responses without warn-level hook rejection diagnostics, v2026.5.18 has the same behavior, and the only linked fix PR was closed unmerged.

## What This Changes

The issue requests warn-level, branch-specific gateway journal diagnostics for hook requests that return non-2xx validation responses.

## Best Possible Solution

Add centralized, secret-safe hook rejection logging in the request handler with stable branch-specific reason codes and focused regression coverage for body-read, direct-hook, mapped-hook, and policy rejects.

## Reproduction Assessment

Yes, at source level. The malformed JSON path reaches readJsonBody, maps to 400 through sendJson, and returns before any hook warning; adjacent direct and mapped validation branches follow the same no-log pattern.

## Solution Assessment

Yes. Handler-level rejection logging with sanitized, branch-specific reason codes is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
