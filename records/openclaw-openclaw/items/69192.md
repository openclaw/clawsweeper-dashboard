## Summary

Keep open. Current main and v2026.5.18 still return hook body-read, direct validation, and mapped validation 4xx responses before any warn-level hook rejection diagnostic, and the only linked fix PR is closed unmerged.

## What This Changes

The issue requests warn-level, secret-safe diagnostics for every non-2xx hook gateway request, with stable rejection reason metadata and less ambiguous mapped-message rejection branches.

## Best Possible Solution

Add centralized, secret-safe hook rejection diagnostics in the hook request handler with stable reason codes and regression coverage for body-read, direct hook, and mapped hook reject branches.

## Reproduction Assessment

Yes, at source level. Malformed JSON returns through the body-read 400 path before any hook warning, and direct plus mapped validation branches do the same for their 400 responses.

## Solution Assessment

Yes. Handler-level rejection logging with sanitized, branch-specific reason codes is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
