## Summary

## What This Changes

The issue asks Gateway hooks to log sanitized warn-level diagnostics for non-2xx hook requests, with stable rejection reasons and less ambiguous mapped-message errors.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/86607 for the focused JSON rejection logging, then decide whether any remaining non-2xx hook branches need a follow-up issue.

## Reproduction Assessment

Yes. Source inspection shows authenticated invalid JSON and mapped-validation branches return JSON 4xx responses without a matching warn log; I did not run the curl repro because this review is read-only.

## Solution Assessment

Yes for the focused JSON rejection gap. A small handler/test change is the narrow maintainable fix, while broader every-non-2xx coverage should be a maintainer scope decision after the linked PR is reviewed.
