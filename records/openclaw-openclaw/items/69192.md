## Summary

Keep open. Current main and the latest release still return hook body-read, direct validation, and mapped validation 4xx responses without warn-level hook diagnostics, and the only linked fix PR was closed unmerged with no canonical replacement.

## What This Changes

The issue requests warn-level, branch-specific gateway journal diagnostics for hook requests that return non-2xx, especially invalid JSON and mapped validation failures.

## Best Possible Solution

Add centralized, secret-safe hook rejection diagnostics in the hook request handler with stable reason codes and regression coverage for body-read, direct hook, and mapped hook reject branches.

## Reproduction Assessment

Yes, at source level. Malformed JSON, direct validation, and mapped validation all return 4xx before any logHooks.warn call, and existing hook status tests can be extended around those branches.

## Solution Assessment

Yes. Handler-level rejection logging with sanitized, branch-specific reason codes is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
