## Summary

Keep open. Current main and v2026.5.18 still return hook body-read, direct validation, and mapped validation 4xx responses without warn-level hook diagnostics; the only linked fix PR was closed unmerged and no canonical replacement surfaced.

## What This Changes

The issue asks for warn-level, branch-specific diagnostics whenever the hook gateway rejects a webhook request with a non-2xx response.

## Best Possible Solution

Add centralized, secret-safe hook rejection diagnostics in the hook request handler with stable reason codes and regression coverage for body-read, direct hook, and mapped hook reject branches.

## Reproduction Assessment

Yes, at source level. Malformed JSON, direct validation, and mapped validation return 4xx before any logHooks.warn call, and existing hook status tests can be extended around those branches.

## Solution Assessment

Yes. Handler-level rejection logging with sanitized, branch-specific reason codes is the narrow maintainable fix; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
