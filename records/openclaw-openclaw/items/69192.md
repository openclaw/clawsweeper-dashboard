## Summary

Keep open. Current main still returns hook validation failures without warning-level rejection diagnostics, and the only linked fix PR was closed unmerged with no merged replacement found.

## What This Changes

The issue requests warning-level, branch-specific diagnostics for hook gateway non-2xx validation responses.

## Best Possible Solution

Add centralized, secret-safe hook rejection diagnostics in the hook request handler with stable internal reason codes and focused regression coverage for body-read, direct-hook, and mapped-hook rejects.

## Reproduction Assessment

Yes, at source level. The malformed JSON path reaches readJsonBody, maps to a 400 through sendJson, and returns before any hook warning; adjacent direct and mapped validation branches show the same no-log pattern.

## Solution Assessment

Yes. Handler-level rejection logging with sanitized, branch-specific reason codes is the narrow maintainable solution; docs or plugin changes would not restore server-side visibility for rejected hook ingress requests.
