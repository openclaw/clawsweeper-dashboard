## What This Changes

The issue asks OpenClaw to stop Claude CLI active-tool heartbeat events from masking a wedged native tool that never emits the matching `tool_result`.

## Best Possible Solution

Land or rework the linked fix so wedged missing-result tools become visible to stuck-session recovery while valid silent Claude-native tools remain protected from false aborts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Claude `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and current diagnostic activity records that as fresh progress.

## Solution Assessment

Unclear for the exact repair shape. The issue is valid, but the linked PR needs maintainer review because making every heartbeat passive can conflict with the shipped protection for valid silent Claude-native tools.
