## Summary

Keep open: current main and the latest release still let Claude live active-tool heartbeats refresh diagnostic progress, and the linked closing PR is open but not yet a safe merged fix path.

## What This Changes

The issue reports that Claude CLI active-tool heartbeat progress keeps diagnostic `lastProgress` fresh when a native tool never emits a matching `tool_result`, preventing stuck-session recovery.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/96198 with a fix that lets wedged Claude tools age into recovery while preserving the valid long-running-tool protection from https://github.com/openclaw/openclaw/pull/87546.

## Reproduction Assessment

Yes at source level. A Claude live `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and current diagnostic activity records every `run.progress` as fresh progress.

## Solution Assessment

Unclear for the open fix path. The issue is valid, but the linked PR's blanket passive-heartbeat approach needs maintainer review because it can undercut the prior shipped long-running-tool invariant.
