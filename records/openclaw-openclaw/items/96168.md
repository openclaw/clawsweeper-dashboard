## Summary

Keep open: current main still lets Claude live active-tool heartbeats refresh diagnostic activity, and the open closing PR is the implementation path but is not merged and currently has unresolved proof/rework concerns.

## What This Changes

The issue asks OpenClaw to stop synthetic Claude CLI active-tool heartbeat progress from masking wedged subprocesses in stuck-session detection.

## Best Possible Solution

Preserve heartbeat observability and freshness for valid silent Claude tools, then add a narrower wedged-tool detector or timeout path that distinguishes missing `tool_result` hangs from legitimate long-running silent tools.

## Reproduction Assessment

Yes by source inspection. A Claude live `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and current diagnostic activity records that heartbeat as fresh progress.

## Solution Assessment

No final fix is merged yet. The current linked PR addresses the symptom at the diagnostic boundary, but it must avoid regressing the shipped valid-silent-tool heartbeat invariant before it is the best solution.
