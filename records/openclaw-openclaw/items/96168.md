## Summary

Keep open: current main and the latest release still treat Claude live active-tool heartbeats as real diagnostic progress, and the open closing PR https://github.com/openclaw/openclaw/pull/96198 is the active fix path but has not merged.

## What This Changes

The issue reports that Claude CLI live active-tool heartbeats can refresh diagnostic progress indefinitely when a tool_use never receives a matching tool_result, delaying stuck-session recovery.

## Best Possible Solution

Review and land or replace the linked PR with a fix that lets truly wedged Claude tools age and recover without regressing valid long-running Claude-native tools.

## Reproduction Assessment

Yes, source-level. Current main emits cli_live:tool_running while activeTools remains populated, diagnostic progress refreshes lastProgressAt for every run.progress reason, and the reporter supplied live fleet logs for the wedged Claude CLI case.

## Solution Assessment

Yes for keeping this issue open. The issue is a valid bug, but the current linked PR still needs maintainer review because blanket passive heartbeat handling may conflict with the prior long-running-tool protection.
