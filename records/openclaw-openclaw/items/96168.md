## What This Changes

The issue requests a fix for Claude CLI live-session heartbeats masking wedged native tools that never emit a matching `tool_result`.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/96198 or a replacement lands a fix that detects missing-`tool_result` wedges without reintroducing false aborts for valid silent Claude tools.

## Reproduction Assessment

Yes, source-reproducible with high confidence: a Claude `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and diagnostic activity records that as fresh progress.

## Solution Assessment

No final fix has landed. The best fix should distinguish wedged missing-result tools from valid silent long-running Claude tools rather than making all heartbeat activity passive unconditionally.
