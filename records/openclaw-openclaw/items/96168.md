## Summary

Keep open: current main and the latest release still treat Claude live active-tool heartbeats as real diagnostic progress, and the linked candidate fix has not merged. The issue remains necessary until https://github.com/openclaw/openclaw/pull/96198 is landed, reworked, or replaced.

## What This Changes

The issue reports that Claude CLI active-tool heartbeat events can keep stuck-session diagnostics fresh when a native tool never emits a matching tool result.

## Best Possible Solution

Keep heartbeat observability and preserve valid silent-tool liveness, then add a narrower missing-result or wedged-tool detector with regression coverage and live Claude CLI proof.

## Reproduction Assessment

Yes by source inspection. A Claude `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and current diagnostic activity records that as fresh progress.

## Solution Assessment

Unclear for the currently linked PR. The issue is valid, but the best fix must distinguish wedged missing-result tools from valid silent Claude-native tools rather than replacing one liveness bug with the inverse one.
