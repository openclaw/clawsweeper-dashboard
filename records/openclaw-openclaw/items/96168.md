## What This Changes

The issue reports that Claude CLI active-tool heartbeat progress can keep stuck-session activity fresh when a native tool never emits a matching `tool_result`.

## Best Possible Solution

Review, rework if needed, and land https://github.com/openclaw/openclaw/pull/96198 or an equivalent fix that distinguishes wedged missing-result tools from valid silent Claude native tools.

## Reproduction Assessment

Yes at source level: current main keeps active tools in a Map until a matching `tool_result`, emits `cli_live:tool_running` while the Map stays non-empty, and records every run progress event as fresh session activity. I did not run a live wedged subprocess in this read-only review.

## Solution Assessment

No final fix is on main yet. The best path is to review and land a narrow fix that keeps heartbeat observability without letting synthetic ticks mask genuinely wedged tools or regress valid silent-tool runs.
