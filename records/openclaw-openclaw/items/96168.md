## Summary

Keep open: current main and the latest release still treat Claude live active-tool heartbeats as fresh diagnostic progress, and the open linked PR is the candidate fix but has not merged.

## What This Changes

The issue reports that Claude CLI live-session active-tool heartbeat progress can mask wedged subprocesses and prevent stuck-session recovery from firing on time.

## Best Possible Solution

Resolve the linked PR so wedged Claude tools can age and recover without losing the shipped heartbeat protection for legitimate quiet long-running Claude-native tools.

## Reproduction Assessment

Yes, at source level: a Claude live `tool_use` without a matching `tool_result` leaves `activeTools` populated, the heartbeat emits `cli_live:tool_running`, and current diagnostic activity records that as fresh progress. I did not run a live Claude CLI wedge in this read-only review.

## Solution Assessment

No final best fix yet. The open linked PR is the right implementation vehicle, but its current review says the fix must reconcile the wedge with the prior valid long-running tool heartbeat invariant.
