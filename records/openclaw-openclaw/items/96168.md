## Summary

Keep open: current main and v2026.6.10 still treat Claude live active-tool heartbeat events as fresh diagnostic progress, and the linked closing PR is open but not merged.

## What This Changes

The issue reports that Claude CLI active-tool heartbeat progress refreshes diagnostic session activity while a native tool is wedged, preventing stuck-session detection from aging the run normally.

## Best Possible Solution

Review and land a narrowed fix in https://github.com/openclaw/openclaw/pull/96198, or a replacement, that lets wedged Claude CLI tools age while preserving valid quiet-tool heartbeat protection.

## Reproduction Assessment

Yes. The source path is high-confidence: current main leaves `activeTools` populated without a matching `tool_result`, emits `cli_live:tool_running`, and records that heartbeat as fresh diagnostic progress; I did not run a live Claude CLI repro in this read-only pass.

## Solution Assessment

Unclear for the current linked PR as written. The best fix should distinguish synthetic heartbeat observability from real progress while preserving the prior protection for legitimate quiet long-running Claude tools.
