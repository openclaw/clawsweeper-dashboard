## Summary

Keep open: this PR fixes the real missing-id write bug in the right CLI surface, but the current guard still rejects manifest-backed plugin contribution aliases because it uses the static config normalizer before exact registry-id matching.

## What This Changes

The PR adds pre-write registry validation for `plugins enable` and `plugins disable`, focused policy tests for missing and selected compatibility ids, and a changelog fix entry.

## Best Possible Solution

Use the registry metadata alias-aware normalizer to canonicalize policy command input before the missing-id check, while keeping the no-write behavior for truly unknown ids.

## Reproduction Assessment

Yes, source-reproducible. Current main shows both policy commands mutate plugin config before discovered-plugin validation; I did not execute the CLI because `pnpm` is unavailable in this checkout.

## Solution Assessment

No, not yet. The validation boundary is right, but the implementation should use the registry-backed alias contract rather than the narrower static fallback normalizer.
