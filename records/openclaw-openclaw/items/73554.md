## Summary

Keep open: the bug is real and the PR is directionally right, but the latest head still rejects manifest-backed registry aliases and lacks the required changelog entry, so it is not ready to merge.

## What This Changes

The PR adds pre-write discovered-plugin checks to `plugins enable` and `plugins disable`, with policy tests for missing ids and selected legacy alias normalization.

## Best Possible Solution

Use the registry-backed alias normalizer from the discovered plugin metadata to resolve CLI input to a canonical plugin id before mutation, then add the missing Unreleased fix entry.

## Reproduction Assessment

Yes. Current main source shows deterministic pre-validation config writes for missing ids, and the PR regression is reproducible from source by using a manifest contribution alias such as `byteplus-plan` when `report.plugins` contains only the canonical `byteplus` id.

## Solution Assessment

No, not yet. Pre-write validation is the right boundary, but the implementation must preserve registry-backed aliases rather than only hard-coded legacy aliases, and the user-visible fix needs a changelog entry.
