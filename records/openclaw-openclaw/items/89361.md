## Summary

Keep open. Current main still uses a model-visible skill catalog with prompt budget and compact/truncation safeguards, but it does not implement category-aware skill search, trigger metadata, or a scenario-driven smart loading mode; the overlapping progressive-disclosure PR is open but not a safe canonical close target because it is dirty, proof-blocked, and does not cover the full category/frontmatter proposal.

## What This Changes

The issue proposes a category-aware smart skill index with `skill_search`, new skill frontmatter metadata, scenario-triggered loading, and an opt-in `skills.loading` mode to reduce skill catalog prompt cost.

## Best Possible Solution
