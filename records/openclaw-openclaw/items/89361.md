## Summary

Keep open. Current main and v2026.6.6 still use a flat prompt-visible skill catalog with compact/truncation fallback, while the requested category metadata, local runtime `skill_search`/`skill_view`, and opt-in smart-loading config remain unimplemented; https://github.com/openclaw/openclaw/issues/74100 overlaps on token reduction but is graph-specific, and https://github.com/openclaw/openclaw/pull/75871 was closed unmerged.

## What This Changes

Request category-aware, scenario-triggered smart skill indexing that replaces always injecting the full skill catalog with opt-in metadata, search, and lazy SKILL.md loading.

## Best Possible Solution

Maintainers should choose one canonical progressive skill-disclosure contract that reconciles this issue with graph/runtime-discovery work, then implement opt-in metadata, search, prompt, docs, and compatibility tests around that decision.
