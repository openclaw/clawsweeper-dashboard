## Summary

Keep open. Current main and v2026.6.6 still expose a flat prompt-visible `<available_skills>` catalog with compact/truncation fallbacks, but they do not implement category/tags/triggers metadata, `skill_search`/`skill_view`, or an opt-in smart-loading config; https://github.com/openclaw/openclaw/issues/74100 overlaps on token reduction but does not fully supersede this category-aware search design, and https://github.com/openclaw/openclaw/pull/75871 was closed unmerged.

## What This Changes

Request category-aware smart skill indexing that replaces always-visible skill catalog descriptions with an opt-in `skill_search`/lazy-loading prompt mode and new skill frontmatter metadata.

## Best Possible Solution

Choose one canonical progressive skill-disclosure design that settles metadata, search semantics, prompt shape, config rollout, and compatibility tests, then link graph-specific dependency loading to the related open issue.

## Solution Assessment
