## Summary

Keep open. Current main already has `argsMenu: "auto"` for `/reasoning`, but `/verbose` still has choices without a menu and regression coverage currently asserts that no-arg `/verbose` stays a status-style dispatch, so the remaining work needs a small maintainer UX decision rather than cleanup closure.

## What This Changes

The issue requests automatic argument menus for `/verbose` and `/reasoning` so supported chat surfaces can render option buttons instead of plain text choices.

## Best Possible Solution

Decide whether `/verbose` should keep a plain no-arg status reply or become a status-bearing button menu, then align any Feishu card-menu work with the existing canonical follow-up at https://github.com/openclaw/openclaw/issues/55686.

## Reproduction Assessment

Yes for the `/verbose` half by source inspection: the command has choices but no `argsMenu`, and the resolver returns null when `argsMenu` is absent. The `/reasoning` half does not reproduce on current main because the registry already sets `argsMenu: "auto"`.

## Solution Assessment

No, the proposed two-line registry edit is not clearly the best complete fix. `/reasoning` is already updated, `/verbose` conflicts with existing no-arg status coverage, and Feishu needs channel-specific card/menu handling or should remain tracked by the related Feishu issue.
