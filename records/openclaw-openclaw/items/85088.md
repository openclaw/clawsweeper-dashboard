## Summary

Keep open. Current main already has `argsMenu: "auto"` for `/reasoning`, but `/verbose` still has choices without a menu and current tests explicitly preserve no-arg `/verbose` as a status-style reply, so the remaining work needs a small UX decision instead of cleanup closure.

## What This Changes

Request to show inline option buttons for `/verbose` and `/reasoning` choices on native chat surfaces.

## Best Possible Solution

Decide whether `/verbose` should keep a plain no-arg status reply or become a status-bearing button menu, and keep Feishu card-menu work aligned with https://github.com/openclaw/openclaw/issues/55686.

## Reproduction Assessment

Yes for the `/verbose` half by source inspection: the command has choices but no `argsMenu`, and the resolver returns null when `argsMenu` is absent. The `/reasoning` half does not reproduce on current main because the registry already sets `argsMenu: "auto"`.

## Solution Assessment

No, the proposed two-line registry edit is not clearly the best complete fix. `/reasoning` is already updated, `/verbose` conflicts with existing no-arg status coverage, and Feishu needs channel-specific card/menu handling or should remain tracked by the related Feishu issue.
