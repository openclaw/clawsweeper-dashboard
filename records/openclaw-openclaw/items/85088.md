## Summary

Keep open. Current main and the latest release already have `argsMenu: "auto"` for `/reasoning`, but `/verbose` still has choices without an argument menu, and the active implementation candidate at https://github.com/openclaw/openclaw/pull/85159 explicitly closes this issue but is still open.

## What This Changes

The issue requests inline argument menus for `/verbose` and `/reasoning` so users can tap command choices instead of typing them.

## Best Possible Solution

Review, repair, and land or close https://github.com/openclaw/openclaw/pull/85159 as the active fix path, preserving no-arg current-level status across affected native menu consumers and leaving Feishu-specific card buttons to https://github.com/openclaw/openclaw/issues/55686.

## Reproduction Assessment

Yes for the remaining `/verbose` half by source inspection: it has choices but no `argsMenu`, and the resolver returns no menu when `argsMenu` is absent. The `/reasoning` half does not reproduce on current main because the registry already sets `argsMenu: "auto"`.

## Solution Assessment

No, the issue's proposed two-line registry edit is not the best complete solution by itself. The safer path is the linked PR or an equivalent change that preserves the documented no-arg current-level status across every native menu consumer and keeps Feishu card behavior separate.
