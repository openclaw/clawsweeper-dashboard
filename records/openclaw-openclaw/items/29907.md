## Summary

Keep open: current main still only supports `systemEvent` and `agentTurn` cron payloads, while the requested deterministic exec/Lobster runner with conditional agent handoff is a new scheduler and security design.

## What This Changes

Request a new cron payload kind for direct shell or Lobster pipeline execution that evaluates output conditions and only starts an agent turn when the condition matches.

## Best Possible Solution

Keep this open as a product/security design item, likely folding the generic direct-exec scheduler work into #18160 while specifying the conditional Lobster-to-agent handoff here.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection against current main shows the requested payload kinds and `condition`/`onCondition` fields are absent rather than a broken existing contract.

## Solution Assessment

Unclear. A no-LLM cron guard path addresses the reported token-cost problem, but maintainers still need to choose the safe core shape for direct exec, Lobster execution, condition evaluation, approvals, and agent handoff.
