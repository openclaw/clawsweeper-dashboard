## Summary

Keep this open: current main still only supports cron `systemEvent` and `agentTurn` payloads, while this request asks for a new deterministic exec/Lobster cron execution mode with conditional agent handoff that is not covered by existing behavior.

## What This Changes

The issue requests a new cron mode that runs a shell command or Lobster pipeline directly, evaluates a deterministic condition, and starts an agent turn only when that condition passes.

## Best Possible Solution

Keep this open as a product/security design item unless maintainers explicitly fold the conditional Lobster trigger scope into the older direct-exec cron issue; the end state should be an additive cron payload contract with documented safety policy and tests across gateway schema, CLI, runtime, delivery, and docs.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection shows the requested cron payload kinds and condition fields are absent from the cron contract.

## Solution Assessment

Unclear. Direct deterministic cron execution is a plausible solution to the token-cost problem, but the proposed core shape is not yet clearly the narrowest maintainable design because it crosses scheduler, command execution, Lobster, delivery, and security policy boundaries.
