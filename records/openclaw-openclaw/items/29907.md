## Summary

Keep open. Current main still only accepts `systemEvent` and `agentTurn` cron payloads, while the requested deterministic `exec`/Lobster runner with `condition` and `onCondition` handoff is missing and needs maintainer product/security design.

## What This Changes

Request adds a deterministic cron kind (`exec` or `lobster`) that runs a command or Lobster pipeline, evaluates a condition, and only then triggers an agent turn.

## Best Possible Solution

Land a maintainer-approved additive cron executor contract, in core or as a plugin seam, with deterministic exec/Lobster execution, bounded condition evaluation, run records, failure delivery, and explicit conditional agent-turn handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source inspection is high confidence that the requested payloads cannot be created, validated, or executed today.

## Solution Assessment

Unclear. The deterministic precondition-before-agent goal is sound, but the raw `exec`/`lobster` shape and condition DSL need maintainer product and security design; an additive executor contract is safer than a one-off cron branch.
