## Summary

Keep open: current main still has only systemEvent and agentTurn cron payloads, and this request adds conditional deterministic exec/Lobster behavior that is broader than the open direct-exec tracker #18160. The closed implementation PR #51276 did not land, and the remaining work needs owner and secops design before a repair PR is safe.

## What This Changes

Request to add deterministic cron payloads that run shell commands or Lobster pipelines, evaluate conditions, and only start an agent turn when a condition matches.

## Best Possible Solution

Keep this open as a conditional deterministic-cron follow-up while maintainers decide whether to layer it on top of #18160's base exec payload or design a broader exec/Lobster condition payload together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main lacks exec/lobster cron payloads and condition/onCondition dispatch rather than having a broken existing contract.

## Solution Assessment

Unclear as specified. Deterministic cron execution is the right area, but the combined exec/Lobster/condition/onCondition shape needs maintainer and secops decisions before it is the best maintainable design.
