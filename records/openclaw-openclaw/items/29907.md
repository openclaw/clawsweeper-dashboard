## Summary

Keep open. Current main still has no cron payload kind for direct shell or Lobster execution, no condition/onCondition contract, and no scheduler branch that can run a deterministic pipeline before deciding whether to start an agent turn. The related #18160 remains the narrower direct-exec tracker, while this issue covers the broader conditional pipeline workflow.

## What This Changes

The issue requests a new cron execution kind for direct shell or Lobster pipeline runs that evaluates deterministic output conditions and only starts an agent turn when the condition passes.

## Best Possible Solution

Add a maintainer-approved cron executor design that either introduces additive core payloads for exec and Lobster with bounded condition evaluation plus conditional agent-turn handoff, or defines a plugin/ClawHub executor API seam first. The result should preserve existing systemEvent and agentTurn behavior while sharing cron observability, run history, delivery, failure alert, timeout, and security policy semantics.

## Risks / Open Questions

- A native scheduled exec or Lobster path would broaden unattended local command/workflow execution, so maintainer design needs explicit approval policy, sandboxing, allowlists, timeouts, output caps, run-log redaction, templating, and delivery semantics.
- The request overlaps #18160 but is not fully covered by it; closing this into direct exec alone would lose the Lobster pipeline and conditional agent-turn handoff requirements.
- The current plugin cron surface can observe and manage gateway-owned cron jobs, but it does not appear sufficient to implement a safe external cron executor without a core payload change or new plugin executor API seam.
