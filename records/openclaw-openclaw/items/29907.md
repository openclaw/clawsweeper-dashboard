## Summary

Keep open. Current main still supports only `systemEvent` and `agentTurn` cron payloads, and the requested deterministic `exec`/`lobster` executor plus `condition`/`onCondition` handoff is not implemented in the core type, protocol schema, runtime executor, CLI, UI, or docs. The related #18160 tracks the narrower direct-exec subset, but it does not cover the broader Lobster pipeline and conditional agent-turn workflow requested here.

## What This Changes

Request to add deterministic cron payloads such as `exec` or `lobster` that run shell commands or Lobster pipelines, evaluate output conditions, and only start an agent turn when the condition passes.

## Best Possible Solution

Design a maintainer-approved additive cron executor contract that can support deterministic `exec` and Lobster payloads, bounded condition evaluation, durable run records, delivery/failure semantics, and conditional `agentTurn` handoff. If the long-term answer is plugin-owned execution, add the missing scheduler executor API seam first rather than relying on today’s observe-only cron plugin hooks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification is high-confidence: source and protocol schemas show the requested payloads cannot be created or executed today.

## Solution Assessment

Unclear. The deterministic precondition-before-agent goal is sound, but the exact `exec`/`lobster` payload shape and condition DSL need maintainer product and security design; the safer path is an additive, versioned executor contract rather than a narrow one-off branch in cron runtime.
