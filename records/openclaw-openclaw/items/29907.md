## Summary

Keep open: current main still lacks deterministic cron execution with conditional agent handoff, and the remaining design crosses scheduler, Lobster, delivery, config, and host-command security boundaries.

## What This Changes

Request a new cron exec/Lobster payload that runs commands or pipelines without an LLM, evaluates output conditions, and starts an agent turn only when the condition passes.

## Best Possible Solution

Keep this open as a product/security design item, or explicitly split/fold the direct exec part into #18160 while preserving the conditional Lobster handoff scope.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows current main lacks the requested exec/lobster cron payload kinds and condition fields rather than failing an existing contract.

## Solution Assessment

Unclear. A deterministic no-LLM cron path addresses the token-cost problem, but maintainers still need to decide the safest core shape for generic exec, Lobster pipelines, condition evaluation, and agent handoff.
