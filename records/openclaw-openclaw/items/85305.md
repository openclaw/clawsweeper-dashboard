## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Handle missing package-lock before reading installed deps:** `src/plugins/install.ts:709-712`
  - body: This new pre-install read runs before `upsertManagedNpmRootDependency` and before npm install creates `package-lock.json`. `readManagedNpmRootInstalledDependency` uses strict `readJson` on that lockfile, so a fresh managed npm root can throw before a normal npm plugin install starts. Make this read missing-lock-safe, or only run it when an update lock entry actually exists, and add fresh-install coverage.
  - confidence: 0.93
- **[P2] Do not return mixed npm resolution metadata:** `src/plugins/install.ts:920-925`
  - body: When the guard preserves the newer package, this object still spreads the older incoming resolution and only replaces `version`/`integrity`. Install records persist `resolvedSpec` and `shasum`, so a preserved 0.11.2 install can still record the downgraded target's spec and shasum. Build the returned resolution from the kept artifact, or clear/fetch fields that cannot be proven, and assert those fields in the integration test.
  - confidence: 0.88
