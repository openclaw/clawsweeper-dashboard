## Summary

Keep open: current main has the narrower hotfix from https://github.com/openclaw/openclaw/pull/81570, but this active draft still carries broader freshness work and currently has a source-level bug where fast memo hits bypass the broader plugin-file freshness verifier.

This PR adds a broad process-local memo layer for loadPluginMetadataSnapshot, memo-key/fingerprint helpers, lifecycle invalidation, and plugin registry/memo tests for persisted registry freshness.

## Best Possible Solution

Rebase onto the landed slim memo, preserve the broad freshness contract, and make fast-hit cache reuse either include plugin-file freshness or fall through to the full verifier before merge.

## Reproduction Assessment

Yes, at source/proof level. The PR body gives terminal timing proof from a real Linux/WSL2 beta6 install, and the current review finding follows directly from the PR fast-hit branch returning before the full freshness key can run.

## Solution Assessment

No, not yet. The process memo is plausible and the slim hotfix is already on main, but this broader PR should preserve same-process plugin-file freshness by extending fast-key coverage or falling through to the full verifier.
