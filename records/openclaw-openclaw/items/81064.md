## Summary

Keep open. Current main still has the non-memoized snapshot loader, while this PR adds a plausible fingerprint-keyed process memo with sufficient terminal proof and no blocking finding from read-only review; because the fingerprint surface is broad and the PR is new, it should go through normal maintainer and CI validation.

## What This Changes

The PR adds a process-local, fingerprint-keyed memo for plugin metadata snapshot loading, expands discovery fingerprinting, clears the memo on metadata lifecycle changes, and adds memo invalidation and aliasing tests.

## Best Possible Solution

Land the process-scoped memo after required CI confirms the broad invalidation surface, keeping the cache inside plugin metadata control-plane ownership.

## Reproduction Assessment

I did not rerun the 104-plugin scenario in this read-only checkout.

## Solution Assessment

Likely yes: a caller-input plus discovery-fingerprint memo is narrower than a time-based or config-only cache and the PR clears it on lifecycle state changes. The remaining gate is broad CI and maintainer review, not an identified patch defect.
