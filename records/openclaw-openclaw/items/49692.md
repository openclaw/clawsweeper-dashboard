## Summary

Keep open. Current main has friendlier restart copy and a new explicit safe-restart path, but command enqueues are still rejected during the gateway drain window before they can enter the preserved queue, so the requested automatic replay behavior remains unresolved; the overlapping #46303 and #57425 tracks need maintainer selection rather than cleanup closure.

## What This Changes

Request friendlier channel-facing gateway restart errors and automatic FIFO replay of inbound messages rejected during the gateway drain window.

## Best Possible Solution

Keep the issue open until OpenClaw either lands a native FIFO queue-during-drain path for in-process restarts or explicitly scopes that work into the broader graceful restart recovery track, while preserving the friendly fallback copy for non-replayable failures.

## Reproduction Assessment

Yes. The source-level path is `markGatewayDraining()` followed by `enqueueCommand()`, which current main still tests and implements as a `GatewayDrainingError` rejection before any queue entry is preserved.

## Solution Assessment

Unclear as a final product path. Queueing during the SIGUSR1 drain window is likely the narrowest in-process fix, but durable replay and session recovery should stay under maintainer review because they change restart durability and replay semantics.
