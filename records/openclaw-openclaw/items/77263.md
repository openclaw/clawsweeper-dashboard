## Summary

Keep open: current main still applies a POSIX group/other mode-bit check to the native hook relay bridge directory on every platform, and the later issue comment correctly calls out that a simple Windows skip would weaken a security guard rather than fully fix it.

## What This Changes

The issue asks OpenClaw to stop blocking Windows Codex native hook relay startup with a POSIX-only bridge directory permission check.

## Best Possible Solution

Keep the Unix owner/private-mode checks, add Windows-specific ACL or ownership validation for the bridge directory and registry files, and cover the Codex startup path with focused relay tests plus Windows validation.

## Reproduction Assessment

Yes for source reproduction: current main throws the exact reported error if the post-`chmodSync(0o700)` mode still has group/other bits, and Node’s Windows chmod contract makes that POSIX check unsuitable. I did not establish a live Windows gateway failure in this read-only pass.

## Solution Assessment

No complete fix exists yet; the reporter’s `win32` skip is directionally useful for availability but not the best durable solution because it removes validation from a security-sensitive relay directory. The best fix is an explicit Windows ACL branch with equivalent safety guarantees.
