## Summary

Keep open. Current main still preserves the short explicit isolated cron `agentTurn` timeout behavior and does not add a cold-start warning or prominent docs guidance; open PR #74893 already references this issue with closing syntax and is the active implementation candidate.

## What This Changes

The issue asks for a warning, safer default timeout, or stronger documentation for isolated `agentTurn` cron jobs that omit `lightContext` and `toolsAllow` and can spend short timeout budgets during bootstrap.

## Best Possible Solution

Review PR #74893 and land or replace it with a narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while preserving explicit timeout semantics unless the broader timeout issue is addressed separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated `agentTurn` cron job with a short explicit `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the configured timer before later workspace/runtime bootstrap work completes.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning and docs guardrail looks narrower than silently increasing or changing explicit timeout behavior, but maintainers still need to choose the threshold and user-facing surface.
