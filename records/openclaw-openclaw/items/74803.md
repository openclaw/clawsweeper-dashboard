## Summary

Keep open. Current main still preserves the short explicit isolated cron `agentTurn` timeout behavior and does not add the requested creation/update warning or prominent cold-start guidance; PR #74893 remains the active open closing implementation candidate.

## What This Changes

The issue asks for a warning, safer default, or prominent docs guidance for isolated `agentTurn` cron jobs with short `timeoutSeconds` and no `lightContext` or `toolsAllow`.

## Best Possible Solution

Review and land PR #74893, or replace it with an equivalent narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while keeping broader hard-timeout semantics coordinated with #74681.

## Reproduction Assessment
