## Summary

Keep open. Current main still preserves explicit short isolated cron `agentTurn` timeout behavior without a cold-start warning, and open PR #74893 already references this issue with closing syntax, so the issue should remain open until that PR is reviewed and either landed or closed.

## What This Changes

The issue asks for a warning, safer default timeout, or stronger docs guidance for short isolated `agentTurn` cron jobs without `lightContext` or `toolsAllow` that spend their timeout budget during cold bootstrap.

## Best Possible Solution

Review PR #74893 and land or replace it with a narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while preserving explicit timeout semantics unless the broader timeout issue is handled separately.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: create an isolated `agentTurn` cron job with a short explicit `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the configured timer before later workspace/runtime bootstrap finishes.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning and docs guardrail looks narrower than silently increasing or changing explicit timeout behavior, but maintainers still need to choose the threshold and user-facing surface.
