## Summary

Keep open. Current main still applies short explicit isolated cron agentTurn timeouts directly and does not add the requested creation/update warning or prominent cold-start guidance; PR #74893 is open, unmerged, and references this issue with closing syntax, so the issue should stay open until that implementation lands or is replaced.

## What This Changes

Warn, retune, or document isolated cron agentTurn jobs that use a short explicit timeout without lightContext or toolsAllow, because cold-start bootstrap can consume the whole budget before useful work begins.

## Best Possible Solution

Land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist, while keeping any broader hard-timeout semantic change coordinated with #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated agentTurn cron job with a short explicit timeoutSeconds, no lightContext, and no toolsAllow; current main starts the configured timeout before later embedded-run workspace/context bootstrap completes.

## Solution Assessment

Unclear. A targeted warning and docs guardrail is likely narrower than silently changing explicit timeout behavior, but PR #74893 still needs maintainer review for the exact threshold and user-facing surface.
