## Summary

Keep open. Current main still lacks the requested creation/update warning or prominent cold-start timeout guidance, and PR #74893 is an open, unmerged implementation candidate that explicitly closes this issue.

## What This Changes

Request guardrails for isolated `agentTurn` cron jobs whose short explicit timeouts can be consumed by full bootstrap when `lightContext` and `toolsAllow` are not set.

## Best Possible Solution

Review and land PR #74893, or an equivalent narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout behavior to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the configured timer before later embedded-run workspace/plugin/bootstrap work completes.

## Solution Assessment

Unclear. A targeted warning and docs guardrail appears narrower than changing explicit timeout semantics, but the exact threshold and UX should be settled through PR #74893 or a replacement.
