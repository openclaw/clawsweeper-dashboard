## Summary

Keep open. Current main still preserves short explicit isolated `agentTurn` timeouts, starts the cron wall-time timer as soon as embedded execution begins before runtime/plugin/tool/bootstrap preparation, and exposes `lightContext`/`toolsAllow` only as optional mitigations; open PR #74893 is the active closing candidate.

## What This Changes

The issue requests a warning, safer default, or clearer docs for isolated `agentTurn` cron jobs whose cold-start bootstrap can consume short timeout budgets when `lightContext` and `toolsAllow` are omitted.

## Best Possible Solution

Add or land a narrow guardrail for this risky cron configuration, preferably alongside docs guidance, while leaving the broader hard wall-time enforcement tracked by #74681 separate.

## Reproduction Assessment

Yes, at source level. Create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the cron budget before embedded cold-start prep and has no add/update/docs warning for that combination.

## Solution Assessment

Unclear. A targeted warning plus documentation looks narrower than silently changing explicit timeout behavior, but PR #74893 should be reviewed for whether runtime warning timing and wording satisfy the requested guardrail.
