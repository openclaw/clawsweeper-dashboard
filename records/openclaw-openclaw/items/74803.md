## Summary

Keep open. Current main still has the guardrail gap: short explicit isolated `agentTurn` timeouts are preserved, the timeout can include embedded runner startup before the first tool call, and the CLI/docs do not warn users to combine short jobs with `lightContext` or `toolsAllow`; open PR #74893 is the active closing implementation candidate.

## What This Changes

Request a warning, safer default, or prominent docs guidance for short isolated `agentTurn` cron jobs that omit `lightContext` and `toolsAllow`.

## Best Possible Solution

Review and land PR #74893, or an equivalent narrow warning/docs guardrail, while keeping the broader hard wall-time enforcement discussion in #74681.

## Reproduction Assessment

Yes, at source level. Create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main preserves that timeout, starts it as embedded execution begins before startup stages finish, and has no add/edit warning.

## Solution Assessment

Unclear. A targeted warning plus docs appears narrower than silently changing explicit timeout semantics, but the warning threshold and wording should be settled through PR #74893 or maintainer review.
