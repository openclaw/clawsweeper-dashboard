## Summary

Keep open. Current main still lacks a warning, default-timeout change, or prominent cold-start guidance for this cron shape, and open PR #74893 remains the active unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests a warning, timeout-policy adjustment, or documentation for short isolated cron agentTurn jobs that omit lightContext and toolsAllow because cold-start bootstrap can consume the whole timeout budget.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout semantics to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated agentTurn cron job with a short explicit timeoutSeconds, no lightContext, and no toolsAllow; current main starts the configured timeout before later embedded-run workspace/plugin/bootstrap work completes.

## Solution Assessment

Unclear. A targeted warning or docs guardrail appears narrower than silently changing explicit timeout behavior, but the exact warning threshold and user-facing surface should be settled through PR #74893 or a replacement.
