## Summary

Keep open. Current main still lacks the requested creation/update warning or prominent cold-start guidance, explicit isolated cron agentTurn timeouts still apply as configured, and open PR #74893 remains the active closing implementation candidate.

## What This Changes

Requests a guardrail for isolated cron agentTurn jobs with short timeouts and full context/tool access so cold-start bootstrap does not silently consume the entire timeout budget.

## Best Possible Solution

Review and land PR #74893, or replace it with an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist while preserving explicit timeout semantics unless #74681 deliberately changes them.

## Reproduction Assessment

Yes. ## Solution Assessment
