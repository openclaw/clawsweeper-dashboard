## Summary

Keep open. Current main still lacks the requested creation/update warning or prominent cold-start timeout guidance, and PR #74893 is open, unmerged, and explicitly linked as the active implementation candidate.

## What This Changes

Request a warning, safer default, or prominent docs for short isolated cron agentTurn jobs that omit lightContext and toolsAllow and can spend their timeout budget on bootstrap.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout semantics to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated agentTurn cron job with a short positive timeoutSeconds, no lightContext, and no toolsAllow; current main applies that exact timeout and starts the watchdog before later embedded runner startup work completes.

## Solution Assessment

Unclear. A targeted warning plus docs guardrail appears narrower than silently changing explicit timeout behavior, but the warning threshold and surface should be settled through PR #74893 or a replacement.
