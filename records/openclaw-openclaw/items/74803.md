## Summary

Keep open. Current main still lacks the requested warning or prominent cold-start timeout guidance, and PR #74893 is open, unmerged, and explicitly linked as the active implementation candidate.

## What This Changes

The issue requests a warning, timeout-policy adjustment, or documentation for short isolated cron agentTurn jobs that omit lightContext and toolsAllow because cold-start bootstrap can consume the whole timeout budget.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout behavior to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated agentTurn cron job with a short explicit timeoutSeconds, no lightContext, and no toolsAllow; current main starts the configured timeout before later embedded-run workspace/plugin/bootstrap work completes.

## Solution Assessment

Unclear. A targeted warning and docs guardrail appears narrower than silently changing explicit timeout behavior, but the exact warning threshold and user-facing surface still need maintainer review through PR #74893 or a replacement.
