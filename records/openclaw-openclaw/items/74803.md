## Summary

Keep open. Current main still lacks the requested warning, default-timeout adjustment, or prominent cold-start documentation, and the open closing PR is an implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests a guardrail for short isolated cron agentTurn jobs without lightContext or toolsAllow because cold-start bootstrap can consume the timeout before useful work starts.

## Best Possible Solution

Land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron agentTurn jobs without lightweight context or a tool allowlist, while coordinating any broader hard-timeout semantic change with #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated agentTurn cron job with a short explicit timeoutSeconds, no lightContext, and no toolsAllow; current main starts the configured timeout before later embedded-run workspace/context bootstrap completes.

## Solution Assessment

Unclear. A targeted warning and docs guardrail appears narrower than silently changing explicit timeout behavior, but the exact threshold and user-facing surface should be settled through review of PR #74893 or a replacement.
