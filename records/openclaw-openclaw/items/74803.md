## Summary

Keep open. Current main still lacks the requested creation/update warning or prominent cold-start timeout guidance, and PR #74893 is open as the linked implementation candidate.

## What This Changes

The issue asks for a guardrail or documentation so short isolated cron `agentTurn` jobs without `lightContext` or `toolsAllow` do not spend their timeout budget on bootstrap before useful work.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout semantics to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main applies the exact timeout and starts it before later embedded-run startup and bootstrap work completes.

## Solution Assessment

Unclear. A targeted warning plus docs guardrail appears narrower than silently changing explicit timeout behavior, but PR #74893 or a replacement should settle the threshold and user-facing surface.
