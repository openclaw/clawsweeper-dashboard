## Summary

Keep open. Current main still lacks the requested guardrail for short isolated cron agentTurn jobs, and the issue is already paired with an open implementation PR; the fresh field report also suggests a broader pre-first-model-call stall may remain outside that narrow warning fix.

## What This Changes

The issue asks OpenClaw to warn, document, or adjust timeout behavior when isolated cron agentTurn jobs can spend their configured budget before useful work begins.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74893 for the narrow warning/docs guardrail, while routing the broader hard-wall-time or pre-first-model-call stall behavior through https://github.com/openclaw/openclaw/issues/74681 or a distinct follow-up if needed.

## Reproduction Assessment

Yes, at source level. A short isolated agentTurn job with positive timeoutSeconds, no lightContext, and no toolsAllow preserves the explicit timeout, has optional mitigations, and currently gets no creation/update warning or prominent cron-docs warning.

## Solution Assessment

Unclear. A targeted warning/docs change is the narrowest candidate for the original report, but the newer 2026.5.7 data suggests a separate stall point may need instrumentation or split timeout semantics rather than only the existing workaround advice.
