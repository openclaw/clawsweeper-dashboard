## Summary

Keep open. Current main still honors the short explicit isolated cron `agentTurn` timeout and only exposes `lightContext`/`toolsAllow` as optional mitigations; PR #74893 remains the open closing implementation candidate, so this issue should wait for that PR or an equivalent fix.

## What This Changes

The issue asks for a warning, safer default timeout, or prominent docs guidance for isolated `agentTurn` cron jobs with short `timeoutSeconds` and no `lightContext` or `toolsAllow`.

## Best Possible Solution

Review and land PR #74893, or replace it with an equivalent narrow guardrail that warns or documents short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while leaving broader hard-timeout semantics to #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated `agentTurn` cron job with a short explicit `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the configured timeout before later embedded-runner bootstrap stages complete.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning and docs guardrail appears narrower than silently changing explicit timeout behavior, but maintainers still need to settle the threshold and user-facing surface.
