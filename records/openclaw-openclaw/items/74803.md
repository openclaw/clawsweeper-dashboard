## Summary

Keep open. Current main still has the reported guardrail gap for short isolated cron agentTurn jobs, and open PR #74893 is already the linked closing implementation candidate for this issue: https://github.com/openclaw/openclaw/issues/74803.

## What This Changes

Request a guardrail for isolated cron agentTurn jobs with short explicit timeouts and no lightContext/toolsAllow so cold-start bootstrap does not consume the run budget.

## Best Possible Solution

Review and land PR #74893, or an equivalent narrow warning/docs guardrail, while keeping hard wall-time enforcement tracked separately in #74681.

## Reproduction Assessment

Yes, at source level. A current-main isolated agentTurn cron job with a short positive timeoutSeconds, no lightContext, and no toolsAllow preserves that short budget while leaving full bootstrap/tool surface enabled and emits no creation/update warning.

## Solution Assessment

Unclear. A targeted warning plus docs guidance appears narrower than changing explicit timeout semantics, but the threshold and wording should be settled through PR #74893 or maintainer review.
