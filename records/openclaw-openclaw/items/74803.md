## Summary

Keep open. Current main still applies short explicit isolated `agentTurn` cron timeouts without a targeted cold-start guardrail, and PR #74893 is open and explicitly linked as the active implementation candidate.

## What This Changes

Add a warning, safer timeout guidance, or prominent docs so short isolated `agentTurn` cron jobs without `lightContext` or `toolsAllow` do not silently spend their budget on cold-start bootstrap.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow guardrail for short explicit isolated cron `agentTurn` jobs, while leaving broader hard-timeout semantics tracked in #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the exact timer before later embedded startup/bootstrap work finishes.

## Solution Assessment

Unclear. A targeted warning plus docs guardrail appears narrower than changing explicit timeout semantics, but the threshold and user-facing surface should be settled through PR #74893 or a replacement.
