## Summary

Keep open. Current `main` still has the reported guardrail gap: short explicit isolated `agentTurn` timeouts are preserved, the mitigation flags are wired but optional, and there is no cold-start warning or prominent docs note; PR #74893 is the active open implementation candidate. https://github.com/openclaw/openclaw/issues/74803

## What This Changes

The issue requests a warning, default timeout adjustment, or prominent documentation for isolated cron `agentTurn` jobs that omit `lightContext` and `toolsAllow` and can spend a short timeout budget on bootstrap.

## Best Possible Solution

Review and land PR #74893, or an equivalent narrow warning/docs guardrail, while leaving broader hard wall-time enforcement to #74681.

## Reproduction Assessment

Yes, at source level. Create an isolated `agentTurn` cron job with a short positive `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current `main` preserves that budget and has no creation/update warning for the risky combination.

## Solution Assessment

Unclear. A targeted warning plus docs guidance appears narrower than silently changing explicit timeout behavior, but PR #74893 should settle the threshold and user-facing wording.
