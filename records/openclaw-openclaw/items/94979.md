## Summary

Keep open. ## What This Changes

The issue reports Kimi `web_search` returning Moonshot 401 invalid authentication for a key that works against the Moonshot `.cn` chat completions endpoint.

## Best Possible Solution

Confirm the failing host and full redacted config first; if `.cn` chat config is present but not reused by Kimi search, make a narrow Moonshot plugin fix with focused tests and docs alignment.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows a plausible host-mismatch path when a `.cn` key is provided only as bare Kimi web-search API key and no Kimi or Moonshot base URL is configured.

## Solution Assessment
