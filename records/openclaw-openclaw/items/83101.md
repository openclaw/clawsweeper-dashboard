## Summary

Keep open: the latest PR improved docs/proof and is mergeable, but it still has a concrete Telegram forum-topic targeting bug and a public skipped-message visibility boundary that needs maintainer approval before merge.

## What This Changes

Adds a public `inbound_observed` plugin hook with types, mapper, runner, docs, tests, and Telegram emission for `requireMention`-skipped group messages.

## Best Possible Solution

Preserve Telegram topic targets in the new observed event, keep the hook read-only/no-session/no-reply, and merge only after explicit maintainer approval of the skipped-message plugin visibility contract.

## Reproduction Assessment

Yes. Source inspection shows the PR emits `inbound_observed` before the skip return, but the helper recomputes the conversation target as the parent chat while current Telegram routing already carries a topic-aware `originatingTo`.

## Solution Assessment

No, not yet. A dedicated read-only hook is a maintainable direction, but this implementation needs the topic-target repair and maintainer approval for the new public privacy boundary before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve topic-scoped conversation IDs in skipped events:** `extensions/telegram/src/bot-message-context.body.ts:82`
  - body: When a forum-topic message is skipped by `requireMention`, this helper always sets `conversationId` and `metadata.originatingTo` to `telegram:<chatId>`, dropping the `:topic:<threadId>` suffix used elsewhere for Telegram inbound routing. Archive or audit plugins consuming `inbound_observed` will merge skipped messages from different topics under one conversation, so pass the existing topic-aware origin target into this helper and cover the new hook path with a forum-topic skipped-message test.
  - confidence: 0.87
