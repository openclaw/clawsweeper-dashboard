## Summary

Keep open: this is a focused implementation for the linked skipped-message archive problem, but it adds a new public Plugin SDK hook with privacy/API implications and still has a small docs catalog gap before merge.

## What This Changes

The branch adds a read-only `inbound_observed` plugin hook, maps its payload, emits it for Telegram `requireMention`-skipped group messages, and adds docs plus focused tests.

## Best Possible Solution

Land a documented, read-only observational hook after maintainer approval, keeping Telegram emission narrow and updating the hook catalog so the new public surface is discoverable.

## Reproduction Assessment

Yes, by source inspection: current main reaches the Telegram mention-skip return without a public typed plugin hook, while the PR inserts `inbound_observed` before that return. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

Mostly yes: a dedicated read-only hook is safer than moving `message_received` earlier and changing existing semantics. The remaining concerns are maintainer approval of the public privacy/API boundary and the small docs catalog update.
