## Summary

Keep open: the PR is a focused implementation of the missing passive inbound hook, but it adds a public Plugin SDK surface with privacy-sensitive pre-dispatch group-message visibility. The previous docs issue is still partly actionable because the detailed hook section was updated, but the top-level hook catalog still omits the new hook.

## What This Changes

The PR adds a read-only `inbound_observed` plugin hook, maps its payload, emits it for Telegram `requireMention`-skipped group messages, and adds docs plus focused tests.

## Best Possible Solution

Land a privacy-reviewed, read-only observational hook with the docs catalog completed, preserving existing `message_received` dispatch semantics and treating additional channels as follow-up scope if maintainers want them.

## Reproduction Assessment

Yes. By source inspection, current main reaches the Telegram mention-skip return without a typed public plugin hook, while the PR inserts `inbound_observed` before that return.

## Solution Assessment

Mostly yes. A dedicated read-only hook is safer than moving `message_received` earlier and changing existing semantics, but the public API/privacy boundary needs maintainer approval and the top-level docs catalog should list the new hook.
