## Summary

Keep open: the PR is a focused implementation for the linked skipped-message observability problem, but it adds a new public plugin hook and expands plugin-visible Telegram group content, so maintainer API/privacy approval is still needed. ## What This Changes

The PR adds a typed `inbound_observed` plugin hook, maps its event payload, emits it for Telegram `requireMention`-skipped group messages, and updates docs plus focused tests.

## Best Possible Solution

Land a documented, read-only observational hook after maintainer approval, keep Telegram emission narrow, preserve existing `message_received` dispatch semantics, and add the missing top-catalog docs entry.
