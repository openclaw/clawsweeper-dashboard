## Summary

Keep open: this is a plausible focused implementation for the linked archive-plugin issue, but it needs changes before merge because the Telegram emission bypasses the existing skipped-message ingest opt-in and the new public hook contract is undocumented. The contributor's redacted live output is enough for the real-behavior-proof gate.

## What This Changes

The PR adds a public `inbound_observed` plugin hook, maps and runs it through the hook infrastructure, and emits it from Telegram when `requireMention` skips a group message.

## Best Possible Solution

Expose the observation-only hook behind an explicit operator consent model, document its privacy and payload contract, and either keep the broad linked issue open for non-Telegram channels or split follow-up channel work deliberately.

## Reproduction Assessment

Yes, source-level: current main has no public `inbound_observed` hook, and Telegram's no-mention requireMention branch returns before the normal public message hook path. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The dedicated read-only hook is a reasonable shape, but it should respect an explicit ingest/observation opt-in and ship with public Plugin SDK documentation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Respect the ingest opt-in before observing skipped groups:** `extensions/telegram/src/bot-message-context.body.ts:417`
  - body: The existing Telegram no-mention branch only emits passive `message:received` when the resolved topic/group/default `ingest` setting is true. This new call runs before that gate and only checks whether any plugin registered `inbound_observed`, so installing a hook plugin would expose requireMention-skipped group content even for Telegram groups where passive ingest is disabled. Reuse the existing ingest decision or add an explicit equivalent opt-in before firing the hook.
  - confidence: 0.88
- **[P3] Document the new public hook contract:** `src/plugins/hook-types.ts:82`
  - body: Adding `inbound_observed` to `PluginHookName` makes it a public SDK hook, but the plugin hook docs still list only the existing message hooks and do not define when pre-filter skipped content is exposed, which payload fields are stable, or that the hook is observation-only. Please update the hook reference and privacy guidance with the API change.
  - confidence: 0.83
