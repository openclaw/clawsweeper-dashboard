The PR adds WhatsApp `replyToOfflineMessages` and `offlineMessageMaxAgeSeconds` config, resolves those values per account, and passes them into append-upsert handling.

## Best Possible Solution

Merge the source-branch repair or apply an equivalent patch that keeps the WhatsApp opt-in behavior extension-owned, enforces finite timestamp age gating, updates generated config/schema/docs/changelog surfaces, and adds focused regression coverage.
