## Summary

Keep open. Current main already has a 60-second recent-append grace for WhatsApp catch-up messages, but this PR still proposes a useful configurable extension and is not merge-ready because the new config is not accepted by the strict WhatsApp schema and the opt-in age check fails open for invalid timestamps.

## What This Changes

This PR adds WhatsApp config fields to opt into processing `messages.upsert` append catch-up messages up to a configurable age and passes those options into the inbound monitor.

## Best Possible Solution

Land a focused WhatsApp plugin change that preserves current default append handling, adds a schema-validated and documented opt-in catch-up window, fails closed on unknown timestamps, and covers root/account config plus age filtering in tests.

## Reproduction Assessment

Yes for the review blockers: the schema issue is reproducible by parsing the PR's example keys through the strict `WhatsAppConfigSchema`, and the timestamp issue follows directly from the opt-in branch converting invalid timestamps to `0` then skipping the age check.

## Solution Assessment

No. The opt-in direction is reasonable, but the current patch is not the best form until it wires the validation schema and public docs/changelog, adds focused tests, and treats missing or non-finite append timestamps as stale.
