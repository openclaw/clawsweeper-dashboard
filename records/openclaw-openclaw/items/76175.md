## What This Changes

The branch adds WhatsApp `replyToOfflineMessages` and `offlineMessageMaxAgeSeconds` config plumbing and changes append upsert handling to optionally process offline messages after reconnect.

## Best Possible Solution

Keep the feature WhatsApp-owned and opt-in, then tighten timestamp handling, publish the new config through validation/generated/docs surfaces, update typed fixtures, add focused opt-in tests, and land after the normal maintainer review.

## Reproduction Assessment

Yes for the patch defects: the diff shows the non-finite timestamp bypass, strict schema surfaces missing the new keys, and existing typed fixtures missing newly required fields. The broader live reconnect complaint has supporting reporter evidence but was not re-run here.

## Solution Assessment

No, not as submitted. The WhatsApp plugin is the right boundary for this opt-in behavior, but the implementation needs finite timestamp enforcement, generated config/schema/docs/changelog updates, and focused tests before merge.
