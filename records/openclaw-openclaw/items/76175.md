## What This Changes

The PR adds WhatsApp `replyToOfflineMessages` and `offlineMessageMaxAgeSeconds` options, resolves them per account, and passes them into append-upsert handling after reconnect.

## Best Possible Solution

Fold the linked branch fixes or equivalent changes into this PR: validate timestamps fail-closed, publish the new config through schema/generated/docs surfaces, add focused append tests, and credit the prior #21719 work appropriately.

Yes for the patch defects: the existing WhatsApp monitor harness can exercise append upserts with stale and non-finite timestamps, and source inspection shows the strict config schemas would reject the newly documented keys. I did not need live WhatsApp reconnect proof to reproduce these PR-level blockers.

## Solution Assessment

No, not as submitted. The WhatsApp plugin is the right place for the opt-in behavior, but the patch must fail closed on invalid timestamps and update generated validation/docs surfaces before it is a maintainable solution.
