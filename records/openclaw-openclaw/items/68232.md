## Summary

Keep open: current main and v2026.6.8 still route broad Telegram config edits into a Telegram channel restart, and the issue already has an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that editing Telegram channel config such as `channels.telegram.historyLimit` causes a hot-reload channel restart that discards Telegram in-memory context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90586, or an equivalent narrow fix that keeps runtime-only Telegram config paths from restarting the channel while preserving restart-backed behavior for transport-affecting paths.

## Reproduction Assessment

Yes, at source level: current main's broad `channels.telegram` reload prefix flows through the gateway planner into `restartChannels`, and the handler then stops and starts Telegram. I did not run a live Telegram bot reproduction in this read-only review.

## Solution Assessment

Yes as a direction: a narrow Telegram reload-classification fix is the right path, but this issue should stay open until the linked closing PR is merged or replaced by an equivalent fix.
