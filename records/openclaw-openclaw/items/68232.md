## Summary

Keep open: current main and the latest release still route broad Telegram config edits into a Telegram channel restart, and the issue already has an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks for runtime-only Telegram config edits such as `historyLimit` or `dmPolicy` to avoid destructive channel restarts that discard in-memory Telegram context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90586 or an equivalent narrow fix that keeps runtime-only Telegram config paths as dynamic/no-op reloads while preserving restart-backed behavior for transport-affecting paths.
