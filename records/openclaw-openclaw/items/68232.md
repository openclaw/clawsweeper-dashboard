## Summary

## What This Changes

The issue asks for Telegram config hot reloads to avoid restarting the channel and discarding in-memory group context for runtime-only edits such as `historyLimit`.

## Best Possible Solution

Repair and land the open Telegram reload PR so runtime-only Telegram config changes are hot no-ops while transport-affecting changes remain restart-backed, then close this issue after merge.

## Solution Assessment
