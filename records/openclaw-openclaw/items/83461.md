Keep open: current main and the latest release still contain the source-visible self-hosted discovery path that can drop marker-backed credentials before `/models` discovery, and an open closing PR already owns the candidate fix.

## What This Changes

The issue reports that provider catalog/self-hosted discovery can send unauthenticated `GET /models` requests when resolved provider credentials are non-secret marker values.

## Best Possible Solution

Review and either land or reject the open closing PR; if rejected, keep this issue as the canonical auth-provider discovery bug and require a narrower credential-aware fix.

## Reproduction Assessment

Yes, at source level: current main can carry a truthy marker `apiKey`, derive no `discoveryApiKey`, and then call `/models` discovery without Authorization. I did not run a live LiteLLM, vLLM, or SGLang endpoint in this read-only review.

## Solution Assessment

Unclear until the active closing PR is resolved. The fix belongs in the provider discovery credential path, but the final shape must avoid marker text as bearer auth while preserving concrete env-resolved keys and authless local behavior.
