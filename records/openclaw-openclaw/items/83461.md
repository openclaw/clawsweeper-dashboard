## What This Changes

The issue reports unauthenticated self-hosted provider catalog `/models` discovery when a non-secret API-key marker resolves to no runnable discovery key.

## Best Possible Solution

Review the open fix PR and land a credential-aware provider discovery fix that avoids unauthenticated marker-backed probes while preserving env, SecretRef, OAuth, SDK, and authless local-provider semantics.

## Reproduction Assessment

Yes, at source level. Current main can carry a truthy non-secret marker as `apiKey`, drop it from `discoveryApiKey`, and then issue `/models` discovery without an Authorization header.

## Solution Assessment

Unclear until the open PR is reviewed. The fix belongs in the provider discovery credential path, but it must preserve marker-backed catalog/auth contracts and prove the exact unauthenticated request is gone.
