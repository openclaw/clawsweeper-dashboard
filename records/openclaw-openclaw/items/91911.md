## Summary

Keep this PR open, but do not merge the current head: the earlier retry-budget lifetime defect is fixed, while the short-window classifier still contradicts the stated quota policy and the external contribution has no after-fix real behavior proof.

## What This Changes

Adds three bounded 10s/20s/30s assistant-stage retries on the same provider, model, and auth profile for messages classified as short-window rate limits before profile rotation or model fallback.

## Best Possible Solution

Restrict long-window quota messages to the existing escalation path unless they contain an actual RPM, TPM, requests-per-minute, tokens-per-minute, or per-minute signal, then prove both live short-window recovery and immediate quota escalation before merge.
