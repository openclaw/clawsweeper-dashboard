## Summary

This PR is a useful narrow mitigation, but it is not ready to close the linked cron overflow because the latest head only admits empty-history lightweight runs with zero active tools and the current cron path can still pass default or forced tool allowlists into the same precheck.

## What This Changes

The PR threads lightweight context, prompt-image count, and tool-count signals into embedded pre-prompt compaction, then raises the small-window prompt budget only for empty-history lightweight runs with no images or tools.

## Best Possible Solution

Cover the actual isolated cron runtime shape, including default or small active tool schemas, with current-head real proof; otherwise land only as an explicit no-tool mitigation and keep the broader linked issue open.

## Reproduction Assessment

Yes. Source inspection shows the remaining path: current cron jobs can carry default toolsAllow, isolated cron forwards that allowlist, and the PR's precheck excludes any nonzero tool count from the higher prompt budget.

## Solution Assessment

No. The current patch is a plausible no-tool mitigation, but the best closing fix needs to budget active tool-schema pressure or explicitly narrow the PR so the broader cron overflow report stays open.
