## Summary

Keep open: current main still has the relevant embedded usage-accounting and custom-provider streaming-usage policy gaps, and the linked candidate fix is open but not merged. The issue remains useful as the canonical place for the broader hosted custom-provider default decision.

## What This Changes

The issue reports zero token usage in `--local --json`, session files, trajectory logs, and sessions list for custom `openai-completions` providers whose upstream stream returns usage.

## Best Possible Solution

Review and either land or reject the narrowed embedded usage-preservation PR, then decide whether generic hosted custom providers should request streaming usage by default or remain behind explicit compatibility opt-in and clearer docs/diagnostics.
