Keep open: the branch still contains potentially useful restart-health timeout work that is not on current main, but it is stale against current gateway status and probe-auth behavior and should not merge as-is.

## What This Changes

The PR switches daemon gateway status/restart probing to lightweight health/connect probes, adds explicit-auth and secure-URL checks for overrides, increases and bounds local restart probe timeouts, and refactors explicit-auth helpers.

## Best Possible Solution

Rebase and narrow the branch so default status/restart checks stay lightweight, `--require-rpc` still proves a real `status` RPC, current non-mutating cached-token probe identity policy is preserved, and the bounded local restart timeout has focused tests plus real behavior proof.

## Solution Assessment

No. The health/connect-probe direction is plausible, but this branch is not the best merge shape until it is rebased to preserve current read-proof semantics and non-mutating probe identity policy.
