## Summary

Keep open. Current main has absorbed parts of this area, including lightweight daemon status probing and local restart-probe auth separation, but this PR is stale against current gateway status/restart code and would regress current `--require-rpc` read-probe semantics while still needing security reconciliation around probe identity and transport policy.

## What This Changes

The PR changes daemon gateway status and restart-health probing to use a health-oriented probe path, enforce explicit auth for daemon URL overrides, bound local restart probe timeouts, and extract explicit-auth helpers.

## Best Possible Solution

Rebase or replace this with a narrower patch that preserves current `--require-rpc` status-read proof and non-mutating/cached probe identity behavior while adding only the still-needed strict override-auth and bounded restart-handshake changes.

## Reproduction Assessment

Yes. Source and test inspection gives a high-confidence path: current main tests `--require-rpc` as a real status RPC, while the PR patch routes daemon status through the health probe path without that branch.

## Solution Assessment

No, not in the current shape. The direction is plausible, but the branch needs a rebase and security reconciliation before it is the narrowest maintainable fix.
