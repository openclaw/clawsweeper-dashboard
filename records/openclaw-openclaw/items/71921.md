## Summary

Keep open. Current main still preserves stale explicit Bedrock model rows without `input`, missing `input` still disables image handling, and open PR #72404 specifically references this issue as the active implementation candidate.

## What This Changes

The issue requests Bedrock-hosted Claude 4.x model metadata or merge behavior to preserve `input: ["text", "image"]` so image attachments are accepted.

## Best Possible Solution

Review and land PR #72404 or a provider-owned replacement that refreshes or inherits missing Bedrock `input` capability metadata for stale explicit/static rows after discovery succeeds, while preserving intentional explicit text-only opt-outs and covering foundation, regional, and global Claude ids.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an explicit `models.providers.amazon-bedrock.models` entry for a Claude 4.x id with no `input` and no matching implicit id: current main preserves that row unchanged, then `modelSupportsImages()` returns false for missing `input`.

## Solution Assessment

No for the original registry-only framing. Updating built-in entries would not fix already-persisted stale catalogs; the maintainable solution should repair merge or refresh behavior so provider discovery metadata can fill missing capability fields without overriding intentional explicit settings.
