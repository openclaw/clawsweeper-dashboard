## Summary

Keep open. Current main still preserves stale explicit Bedrock catalog rows without `input`, and the image capability checks still treat missing `input` as not image-capable. Open PR #72404 specifically references this issue as its implementation candidate, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Request to mark Bedrock-hosted Claude Opus/Sonnet 4.x models as image-capable and repair stale persisted Bedrock catalog rows that lost `input` metadata after discovery failed.

## Best Possible Solution

Review and land PR #72404 or a provider-owned replacement that refreshes or inherits missing Bedrock `input` capability metadata for stale explicit/static rows after discovery succeeds, while preserving intentional explicit text-only overrides.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a non-empty `models.providers.amazon-bedrock.models` list containing a Claude 4.x row with no `input`; current merge code preserves that row and current image gates reject missing `input`.

## Solution Assessment

No for the original registry-only framing. Updating only built-in registry entries would not repair already-persisted static catalogs; the maintainable fix should address stale explicit catalog metadata or refresh behavior after discovery succeeds.
