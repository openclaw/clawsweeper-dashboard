## Summary

Keep open. Current main still preserves stale explicit Bedrock model rows that omit `input`, and those rows are still treated as text-only; open PR #72404 remains the unmerged implementation candidate with closing syntax.

## What This Changes

The issue asks OpenClaw to mark Bedrock-hosted Claude Opus/Sonnet 4.x models as image-capable in generated or persisted model catalogs.

## Best Possible Solution

Review and land PR #72404 or an equivalent provider-owned merge/catalog refresh fix that restores missing image capability metadata for stale Bedrock Claude rows while preserving explicit text-only opt-outs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an explicit `models.providers.amazon-bedrock.models` Claude 4.x row with no `input` and no same-id implicit row; current main preserves it and `modelSupportsImages()` returns false.

## Solution Assessment

No for the original registry-only request. Updating generated registry entries alone would not repair already-persisted static catalogs, so the better fix is merge or refresh behavior for missing capability metadata while honoring explicit overrides.
