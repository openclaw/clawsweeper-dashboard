## Summary

Keep open: current main and the latest release still preserve explicit-only Bedrock model rows without `input`, so stale Bedrock Claude catalogs can still make OpenClaw reject image attachments before provider dispatch. The linked fix candidate is open and merge-conflicting, so this issue should remain the canonical tracker until that PR is repaired, landed, or replaced.

## What This Changes

This issue asks OpenClaw to mark Bedrock Claude Opus/Sonnet 4.x model entries as image-capable so image attachments are not stripped.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/72404 is repaired and landed, or replace it with a focused fix that restores Bedrock vision metadata while preserving explicit user overrides.
