## Summary

Keep open. Current main still has the source-level stale-catalog path that can leave Bedrock Claude image-capable models without `input`, and the linked closing implementation is still open rather than merged.

## What This Changes

Bedrock Claude model metadata can miss image input capability, causing OpenClaw to reject image attachments as unsupported.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72404 or an equivalent stale-catalog merge/discovery repair that restores Bedrock Claude image metadata while preserving explicit text-only overrides.

Yes. At source level, an explicit Bedrock Claude row without `input` and without a same-ID implicit discovery row remains unchanged, then `modelSupportsImages()` treats missing input as not image-capable.

## Solution Assessment

Yes for triage: keeping this issue open behind the unmerged closing PR is the right outcome. A static registry-only edit would be weaker because already-persisted stale catalogs can continue to miss image metadata.
