Keep open. Current main still has a source-reproducible stale explicit-catalog path where Bedrock Claude rows without `input` can remain text-only, and the open closing implementation candidate at https://github.com/openclaw/openclaw/pull/72404 is not merged.

## What This Changes

The issue requests that Bedrock Claude Opus/Sonnet 4.x model entries retain `input: ["text", "image"]` so image attachments are not blocked as unsupported.

## Best Possible Solution

Review, revise, and land the linked implementation or an equivalent merge/discovery repair that restores missing image metadata for stale Bedrock Claude rows while preserving explicit text-only overrides.
