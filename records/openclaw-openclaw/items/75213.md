## Summary

Keep open. Current main still refreshes the Codex app-server terminal idle activity before request/notification traffic is scoped to the active thread and turn, while this PR is a narrow implementation with matching regression coverage and no blocking review finding from this pass.

## What This Changes

The PR scopes Codex app-server run-attempt activity tracking to current-thread/turn notifications and requests, and adds regression tests for unrelated shared-client traffic not extending the silent-turn timeout.

## Best Possible Solution

Land this scoped activity-tracking fix after maintainer review, preserving shared app-server fan-out while ensuring only the active Codex thread/turn can refresh the silent-turn timeout.

## Reproduction Assessment

Yes. On current main, a fake-timer run can start a silent Codex app-server turn, deliver a shared-client notification or server request for another thread just before the terminal idle deadline, and observe the idle timer being refreshed before the handler rejects unrelated traffic.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it moves idle activity updates behind the same current-thread/turn checks already used by the projector/request payloads and adds focused regression coverage without changing the shared-client contract.
