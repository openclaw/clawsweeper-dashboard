## Summary

Keep open. Current main still has the unscoped Codex app-server activity updates that can let unrelated shared-client traffic refresh a silent turn's idle deadline, while this PR is a focused fix with matching regression coverage and no concrete blocking review finding.

## What This Changes

This PR scopes Codex app-server run-attempt activity tracking to current-thread/turn notifications and current-turn server requests, then adds regression tests for unrelated shared-client traffic during the silent-turn timeout.

## Best Possible Solution

Land this scoped-activity fix after maintainer review and current-main merge gating, preserving shared app-server fan-out while ensuring only the active Codex thread/turn can refresh the silent-turn timeout.

## Reproduction Assessment

Yes. On current main, start a Codex app-server turn with a short terminal-idle timeout, then deliver another thread's notification or server request before the deadline; the handler touches activity before rejecting unrelated traffic, delaying the timeout.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix I found: keep the shared client contract intact, scope idle activity updates before timer refresh, and cover the cross-turn notification/request cases with focused tests.
