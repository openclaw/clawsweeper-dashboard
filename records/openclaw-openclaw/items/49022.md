- **[P1] Use the current pi-tui package:** `src/tui/components/tool-execution.ts:1`
  - body: Current main depends on and imports `@earendil-works/pi-tui`, but this PR still imports from `@mariozechner/pi-tui` and does not add that package. After rebasing onto current main this will fail to resolve or reintroduce the stale package surface, so update the imports to the current dependency before merge.
  - confidence: 0.93
- **[P2] Constrain MEDIA reads to trusted image files:** `src/tui/components/inline-image.ts:54-55`
  - body: `readMediaImageAsBase64()` accepts any absolute image-looking path from tool output after normalization, extension, symlink, and size checks. Tool text is not itself proof that OpenClaw produced that file, so a malicious or mistaken result can make the TUI read and render arbitrary local images; gate this to a trusted media root or another explicit producer contract.
  - confidence: 0.84
- **[P2] Release rendered fingerprints when images are cleared:** `src/tui/components/tool-execution.ts:95`
  - body: The shared `globalRendered` set is only added to, while `setResult()`, `setPartialResult()`, `clearTools()`, and pruning can detach or remove the component that owns the cached image. A later retry or updated result with the same path/fingerprint will then skip image creation and can show only stripped text or nothing, so make the dedupe ownership reversible or scope it to one processed result.
  - confidence: 0.91
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:45`
  - body: OpenClaw's changelog is release-generated, and normal PRs should carry user-visible release-note context in the PR body or commit message instead. This PR already has that context in the body, so the direct `CHANGELOG.md` entry should be removed.
