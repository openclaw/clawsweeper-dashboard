## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict MEDIA file reads to trusted image outputs:** `src/tui/components/inline-image.ts:44`
  - body: Tool result text can contain MEDIA directives, but this accepts any absolute path and later reads it if the extension and size checks pass. A malicious or compromised result can make the TUI render local host images outside OpenClaw-generated media; gate file reads to trusted media attachments/roots or use the supplied image block instead.
  - confidence: 0.82
- **[P2] Release dedupe keys when image results detach:** `src/tui/components/tool-execution.ts:96-103`
  - body: The shared Set lives on ChatLog, and keys added here are never removed when setResult replaces cached images, setPartialResult clears them, or old components are pruned. After a retry or later tool result with the same path/fingerprint, the image can be skipped even though the current result should render it.
  - confidence: 0.88
- **[P2] Rebase pi-tui imports to the current package:** `src/tui/components/inline-image.ts:7`
  - body: Current main depends on @earendil-works/pi-tui, but this added module imports @mariozechner/pi-tui. The PR is dirty against main, and resolving it without updating the new imports leaves the inline-image code unable to compile against current dependencies.
  - confidence: 0.86
