- **[P1] Restrict MEDIA image reads to trusted artifacts:** `src/tui/components/tool-execution.ts:97`
  - body: `splitMediaFromOutput()` accepts legacy MEDIA directives from tool text, and this call then reads the referenced absolute path in the host TUI process. A tool/model-controlled string can make the terminal read and render local host images outside the gateway or media-artifact trust boundary, so this needs trusted media ownership or structured image blocks before reading files.
  - confidence: 0.89
- **[P2] Keep image dedupe scoped to live cached images:** `src/tui/components/tool-execution.ts:96-97`
  - body: The shared `globalRendered` set is append-only for the ChatLog lifetime, while `setPartialResult()`, `setResult()`, and pruning can detach or replace the image components that originally owned those keys. A later retry or updated result with the same path/fingerprint can be skipped even when no live image remains, so the dedupe entry needs to be released with the cached image lifecycle or scoped to one processed result.
  - confidence: 0.87
