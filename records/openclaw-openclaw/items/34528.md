The issue requests stripping Feishu synthetic reaction suffixes from message IDs before using them in Feishu API calls.

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that preserves internal synthetic reaction IDs while sending only base open message IDs to Feishu.
