## Summary

Keep open: current main still has the Feishu reaction-ID leak, and the issue is paired with the open canonical fix PR at https://github.com/openclaw/openclaw/pull/77653, so it should close only after that PR lands or is replaced.

## What This Changes

The issue asks Feishu to strip synthetic `:reaction:<emoji>:<uuid>` suffixes from message IDs before using them in Feishu API calls.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that preserves synthetic internal IDs while sending only base open message IDs to Feishu endpoints.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu reaction events produce `om_*:reaction:<emoji>:<uuid>` IDs and the reply/typing/reaction API paths pass raw IDs into `message_id`; the reporter's Feishu 400/code 99992354 log matches that path.

## Solution Assessment

Yes in direction. The narrow maintainable fix is Feishu API-boundary normalization, not removing internal synthetic reaction identity; the open linked PR is the canonical implementation path but still needs proof/conflict resolution.
