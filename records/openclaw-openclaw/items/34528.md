## Summary

Keep open: current main and the latest shipped tag still allow Feishu synthetic reaction IDs to reach Feishu `message_id` API fields, and the in-repo closing PR is still open rather than merged.

## What This Changes

The issue asks Feishu to strip OpenClaw's synthetic `:reaction:<emoji>:<uuid>` suffix before using message IDs in Feishu API calls.

## Best Possible Solution

Land one Feishu-plugin API-boundary normalizer that preserves synthetic reaction IDs for internal dedupe/session identity while sending only base open message IDs to Feishu.

## Reproduction Assessment

Yes. Current main constructs `om_*:reaction:<emoji>:<uuid>` IDs, can use them as reply targets, and forwards raw IDs into Feishu `message_id` API fields, matching the reported 400/code 99992354 failure.

## Solution Assessment

Yes in direction. Feishu-local API-boundary normalization is the narrow maintainable fix; the linked open PR targets that path but should land or be replaced before this issue closes.
