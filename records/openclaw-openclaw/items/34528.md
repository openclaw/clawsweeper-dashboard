## Summary

Keep open: current main and the latest release still synthesize Feishu reaction IDs with a `:reaction:` suffix and pass raw IDs into Feishu `message_id` API fields, while the canonical OpenClaw fix PR is open and unmerged.

## What This Changes

The issue asks OpenClaw to strip Feishu synthetic reaction suffixes from message IDs before using them in Feishu API calls.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that keeps synthetic reaction IDs internal and sends only base open message IDs to Feishu.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu reaction events create suffixed synthetic IDs and API-bound paths pass raw message IDs into Feishu `message_id` fields, matching the reported 400/code 99992354 failure.

## Solution Assessment

Yes in direction. The maintainable fix is Feishu API-boundary normalization that preserves internal synthetic reaction identity, and the open linked PR is the current canonical implementation path.
