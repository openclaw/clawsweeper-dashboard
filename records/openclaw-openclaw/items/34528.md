- `impact:message-loss`: A suffixed reaction message ID can make Feishu API calls fail instead of delivering the expected reply, typing indicator, reaction, media, or related channel action.

## Summary

Keep open: current main and the latest release still create Feishu synthetic reaction IDs with a `:reaction:` suffix and pass raw IDs into Feishu `message_id` API fields, while https://github.com/openclaw/openclaw/pull/77653 is the open unmerged canonical fix path.

## What This Changes

Strip Feishu synthetic reaction suffixes from API-bound message IDs while preserving internal reaction event identity.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that keeps synthetic reaction IDs internal and sends only base open message IDs to Feishu.

## Reproduction Assessment

Yes. Source inspection on current main shows reaction events create suffixed synthetic IDs and Feishu API-bound paths pass raw IDs into `message_id` fields, matching the reported 400/code 99992354 failure.

## Solution Assessment

Yes in direction. The maintainable fix is Feishu API-boundary normalization that preserves synthetic reaction identity internally, and the open linked PR is the current canonical implementation path.
