## Summary

Keep open: current main and the latest release still allow Feishu reaction synthetic IDs to flow into API-facing message_id parameters, and https://github.com/openclaw/openclaw/pull/77653 is the open same-repo closing fix candidate.

## What This Changes

The issue requests stripping Feishu reaction synthetic suffixes from API-facing message_id values while preserving synthetic IDs for internal reaction turns.

## Best Possible Solution

Land or replace the linked Feishu normalization PR with a focused fix that normalizes only API-facing Open Message IDs and preserves synthetic IDs for dedupe and session state.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: reaction events create suffixed synthetic message IDs, the bot falls back to that ID as the reply target, and Feishu API callers pass the value into message_id path parameters. I did not run a live Feishu reaction setup in this read-only review.

## Solution Assessment

Yes, normalizing only API-facing Feishu Open Message IDs while preserving synthetic internal IDs is the narrow maintainable fix shape. Closing this issue now would be premature because current main is still affected and the linked implementation PR has not landed.
