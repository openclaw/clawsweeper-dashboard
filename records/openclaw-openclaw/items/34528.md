## Summary

Keep open: current main and the latest release still construct suffixed Feishu reaction IDs and pass raw IDs into Feishu API `message_id` fields; https://github.com/openclaw/openclaw/pull/77653 is the open closing implementation candidate, so this issue should close only after that PR lands or is replaced.

## What This Changes

Strip Feishu synthetic `:reaction:<emoji>:<uuid>` suffixes before using message IDs in Feishu API calls.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that preserves synthetic internal IDs while sending only base open message IDs to Feishu endpoints.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu reaction events produce `om_*:reaction:<emoji>:<uuid>` IDs and reply/typing/reaction API paths pass raw IDs into Feishu `message_id` fields, matching the reporter's Feishu 400/code 99992354 log.

## Solution Assessment

Yes in direction. The narrow maintainable solution is Feishu API-boundary normalization, not removing internal synthetic reaction identity; the open linked PR is the canonical implementation path but still needs merge readiness work.
