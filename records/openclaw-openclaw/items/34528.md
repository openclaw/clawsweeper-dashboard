## Summary

Current main and the latest release still allow a synthetic Feishu reaction ID to become an API-bound reply target. This issue should remain open while https://github.com/openclaw/openclaw/pull/77653 is rebased, given live Feishu proof, and either landed or replaced.

- @yfge — high confidence; owns the current open fix candidate.

## What This Changes

Fix Feishu API calls that receive synthetic reaction-suffixed message IDs instead of base Open Message IDs.

## Best Possible Solution

Land a rebased, live-proven canonical normalizer at every external Feishu message-ID boundary while retaining synthetic IDs solely for internal event uniqueness, deduplication, and session identity.

## Reproduction Assessment

Yes at source level: current main creates a reaction-suffixed event ID and can select it as the API-bound reply target. The reporter's Feishu 400 log matches that path, although this read-only review did not run a live credentialed reproduction.

## Solution Assessment

Yes. Normalizing at external Feishu API boundaries is the narrowest maintainable solution because it prevents invalid API parameters without weakening the synthetic IDs used for internal event and session identity.
