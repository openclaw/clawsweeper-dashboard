## Summary

Keep open: current main and the latest release still synthesize Feishu reaction message IDs with `:reaction:` suffixes and pass raw IDs into Feishu `message_id` API fields. The remaining in-repo fix path is the open https://github.com/openclaw/openclaw/pull/77653, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks Feishu API call sites to strip synthetic `:reaction:<emoji>:<uuid>` suffixes from message IDs before sending them to Feishu endpoints.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local API-boundary normalizer that preserves synthetic reaction IDs internally while sending only base open message IDs to Feishu endpoints.

## Reproduction Assessment

Yes. Source inspection shows current main constructs `om_*:reaction:<emoji>:<uuid>` IDs and passes reply/reaction/typing message IDs directly to Feishu `message_id` path fields; the reporter's 400/code 99992354 log matches that path.

## Solution Assessment

Yes in direction. The narrow maintainable fix is Feishu API-boundary normalization, not changing internal synthetic reaction identity; the existing open PR or a replacement should carry it.
