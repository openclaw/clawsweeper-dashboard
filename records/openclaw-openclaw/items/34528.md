## Summary

Keep open: this Feishu bug is still present on current main, and there is an open closing PR for the same failure mode that should land or be replaced before this issue closes.

## What This Changes

Feishu reaction events can produce synthetic `message_id` values with `:reaction:<emoji>:<uuid>` suffixes that are later sent to Feishu APIs expecting the base open message ID.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with a Feishu-local normalizer that preserves synthetic internal reaction IDs while sending only base open message IDs to Feishu API endpoints.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu reaction events create suffixed synthetic IDs and API paths pass raw message IDs into Feishu `message_id` fields, matching the reported 400/code 99992354 failure.

## Solution Assessment

Yes in direction. The maintainable fix is Feishu API-boundary normalization that keeps internal synthetic reaction identity intact, and the linked open PR is the current canonical implementation path.
