- `P2`: This is a concrete Feishu channel bug with limited blast radius, but reaction-triggered replies and related channel actions can fail for real users.
- `impact:message-loss`: A suffixed reaction message ID can make Feishu API calls fail instead of delivering the expected reply, typing indicator, reaction, media, or related channel action.

## Summary

Keep open: current main and the latest release still create Feishu synthetic reaction message IDs with a `:reaction:` suffix and pass raw IDs into Feishu `message_id` API fields, while the canonical fix PR remains open and unmerged.

## What This Changes

Strip Feishu synthetic `:reaction:<emoji>:<uuid>` suffixes from API-bound message IDs so reaction-triggered replies and actions use the base open message ID.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77653 with Feishu-local API-boundary normalization that preserves synthetic reaction IDs internally and strips the suffix before Feishu `message_id` API fields.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu reaction events create suffixed synthetic IDs and API-bound paths pass raw message IDs into Feishu `message_id` fields, matching the reported 400/code 99992354 failure.

## Solution Assessment

Yes in direction. The maintainable fix is Feishu API-boundary normalization that keeps synthetic reaction identity internal, and the open linked PR is the current canonical implementation path.
