## Summary

Keep open because current main still produces Feishu synthetic reaction message IDs with the reported suffix and still passes raw message IDs into Feishu API path parameters; open PR #39558 is the active closing fix candidate.

## What This Changes

Strip Feishu `:reaction:<emoji>:<uuid>` suffixes from `message_id` values before using them in Feishu API calls.

## Best Possible Solution

Land, rebase, or replace #39558 with a Feishu plugin fix that normalizes only API-boundary `message_id` values while preserving synthetic reaction IDs for internal routing, dedupe, and session keys.

## Reproduction Assessment

Yes. The report gives concrete Feishu reaction steps, and current-main source inspection shows a suffixed reaction ID can flow from `resolveReactionSyntheticEvent` through `ctx.messageId` into Feishu `message_id` API path parameters.

## Solution Assessment

Yes. Stripping the synthetic suffix at Feishu API boundaries is the narrow maintainable fix; stripping it at the producer would risk breaking internal dedupe and session routing that intentionally use the synthetic ID.
