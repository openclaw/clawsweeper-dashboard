## Summary

Keep open: current main and the latest shipped tag still generate Feishu synthetic reaction message IDs and can forward them unchanged into Feishu API `message_id` fields. The canonical in-repo fix candidate is https://github.com/openclaw/openclaw/pull/77653, which uses closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to strip Feishu synthetic `:reaction:<emoji>:<uuid>` suffixes from message IDs before using those IDs in Feishu API calls.

## Best Possible Solution

Land one Feishu-plugin API-boundary normalization fix, preserving synthetic reaction IDs for internal dedupe/session identity while sending only base open message IDs to Feishu.

Yes, source-reproducible. Current main constructs `om_*:reaction:<emoji>:<uuid>` IDs, can select them as reply targets, and forwards them unchanged to Feishu SDK `message_id` fields, matching the reporter's 400/code 99992354 log.
