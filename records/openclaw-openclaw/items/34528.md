## Summary

Keep open: current main and v2026.6.6 still preserve Feishu reaction-suffixed synthetic IDs and can route them to Feishu API-bound reply paths; https://github.com/openclaw/openclaw/pull/77653 is the open canonical fix candidate but has not merged.

## What This Changes

The issue asks Feishu to strip OpenClaw's synthetic `:reaction:<emoji>:<uuid>` suffix from message IDs before using them in Feishu API calls.

## Best Possible Solution

Land a Feishu-plugin normalizer at Feishu API boundaries via the canonical linked PR, preserving synthetic reaction IDs internally for dedupe and session identity.

## Reproduction Assessment

Yes, at source level: current main creates reaction-suffixed Feishu event IDs and can use that same ID as the default reply target sent to Feishu APIs. I did not run a live credentialed Feishu reaction flow in this read-only review.

## Solution Assessment

Yes for the fix direction: normalizing only at Feishu API boundaries is narrower than removing synthetic internal IDs. The current open PR still needs conflict resolution and real Feishu proof before it can resolve this issue.
