## Summary

Keep open: the PR is aimed at the right Feishu filename bug, but it still misses the supported inbound message-payload fallback path and lacks after-fix proof from a real Feishu setup.

## What This Changes

The PR renames Feishu's conservative Latin-1 UTF-8 filename recovery helper, applies it to download-response JSON filename metadata, and updates Feishu media tests.

## Best Possible Solution

Extend the Feishu-local recovery to both download metadata and the message-payload fallback before saveMediaBuffer, then add redacted real Feishu proof and focused media/bot coverage.

## Reproduction Assessment

Yes by source inspection: current main has JSON file_name paths that bypass the recovery helper, and media.test pins one mojibake output. The linked issue also provides a concrete Feishu upload path and terminal evidence, but I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No. Reusing the helper is the right direction, but applying it only in download metadata does not cover the existing mediaKeys.fileName fallback used when download metadata omits the name.
