## Summary

Keep open: the PR is a useful Feishu bug-fix direction, but it still misses the message-payload filename fallback and the supplied proof does not show a real patched Feishu/OpenClaw receive or saved-file result.

## What This Changes

The PR renames Feishu's Latin-1 filename recovery helper, applies it to JSON-derived download metadata, and updates media tests for CJK recovery plus Latin-1 preservation.

## Best Possible Solution

Normalize Feishu inbound filenames at the final saved-name boundary, or cover both download metadata and payload fallback, with focused regression coverage and redacted real Feishu/OpenClaw proof before merge.

## Reproduction Assessment

Yes by source inspection: current main returns JSON response file_name metadata unchanged and has a focused test that pins the mojibake result, with a visible fallback path through bot-content.ts. I did not run a credentialed live Feishu reproduction in this read-only review.

## Solution Assessment

No, not yet. Reusing the existing conservative helper is the right direction, but applying it only to download metadata misses the message-payload fallback used when Feishu omits filename metadata from the download response.
