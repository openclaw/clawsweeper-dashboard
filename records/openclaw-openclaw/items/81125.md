## Summary

Keep this PR open: it addresses a real Feishu filename bug, but the patch still misses the message-payload fallback path and the supplied proof does not show an observed after-fix Feishu/OpenClaw runtime result.

## What This Changes

The PR renames Feishu's Latin-1 filename recovery helper, applies it to JSON-derived download metadata, and updates focused media tests for CJK recovery plus Latin-1 preservation.

## Best Possible Solution

Normalize Feishu inbound filenames at the final saved-name boundary, or land the broader shared decoder if preferred, with coverage for both download metadata and payload fallback paths.

## Reproduction Assessment

Yes by source inspection: current main has a JSON file_name path that bypasses the recovery helper, and the Feishu media test pins the mojibake result. I did not run a live Feishu reproduction or tests in this read-only review.

## Solution Assessment

No, not yet. Reusing the existing conservative helper is the right direction, but applying it only inside download metadata misses the message-payload fallback used when download metadata omits a filename.
