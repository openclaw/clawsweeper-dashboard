## Summary

Keep this PR open: the Feishu filename recovery direction is right, but the branch still misses the post embedded-media fallback path and the supplied proof does not show patched OpenClaw saving an inbound Feishu file after the fix.

## What This Changes

The PR exports Feishu’s conservative filename recovery helper, applies it to JSON download metadata and direct media payload fallback filenames, and updates focused Feishu media/bot tests.

## Best Possible Solution

Centralize the conservative Feishu filename recovery at every JSON-derived inbound filename boundary, including post embedded media fallbacks, then merge after redacted real behavior proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes by source inspection: current main returns raw JSON `file_name` values and the existing Feishu media test asserts the mojibake result. I did not run live Feishu in this read-only review.

## Solution Assessment

No as-is: reusing the existing helper is the right code direction, but applying it only to direct media parsing leaves post embedded-media fallback filenames raw. The safer narrow fix is to recover all JSON-derived fallback names before they reach media storage.
