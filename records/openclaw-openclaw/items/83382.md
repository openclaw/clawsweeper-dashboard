## Summary

Keep this maintainer draft open: the Chrome MCP redaction direction looks useful, but the proxy-env lease change is not merge-ready because it can overwrite or leak user `NO_PROXY`/`no_proxy` state instead of keeping the scoped bypass per casing.

## What This Changes

This PR updates browser/CDP proxy-bypass handling, Chrome MCP attach/timeout diagnostic redaction, related regression tests, and the changelog.

## Best Possible Solution

Keep the diagnostic redaction hardening, but revise the CDP proxy lease to compute, apply, and restore `NO_PROXY` and `no_proxy` independently, with regression tests for divergent casing values and one-casing deletion.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `NO_PROXY` to a loopback-covered value and `no_proxy` to a different lower-only bypass, then enter `withNoProxyForLocalhost`; the PR mirrors the upper value into the lower casing during the lease.

## Solution Assessment

No. The redaction approach is appropriate, but the proxy-bypass fix needs per-casing lease state instead of a shared applied value and all-or-nothing restore.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep NO_PROXY casings independent:** `extensions/browser/src/browser/cdp-proxy-bypass.ts:60-61`
  - body: When `NO_PROXY` already covers loopback but `no_proxy` has different user bypass entries, this new all-casings check acquires the lease while the existing apply step still builds one `applied` value from `NO_PROXY || no_proxy` and writes it to both variables. That drops lowercase-only bypass entries for the duration of the CDP call; append loopback to each casing's own original value instead.
  - confidence: 0.86
- **[P2] Restore untouched proxy vars per casing:** `extensions/browser/src/browser/cdp-proxy-bypass.ts:122`
  - body: The new all-or-nothing `untouched` check preserves a concurrent deletion of `no_proxy`, but it also leaves an untouched `NO_PROXY` at the temporary appended value after the scoped call. Restore each casing whose current value still equals its applied value, while preserving only the casing that was actually changed externally.
  - confidence: 0.84
