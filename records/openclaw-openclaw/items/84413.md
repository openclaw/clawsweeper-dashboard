The issue reports unreaped Codex Chrome DevTools MCP sidecars accumulating under the gateway and driving service memory growth.

## Best Possible Solution

Define and test an explicit Codex app-server MCP lifecycle that preserves native resume while bounding, unsubscribing, pooling, or reaping Chrome DevTools MCP sidecars with live gateway sidecar-count proof.
