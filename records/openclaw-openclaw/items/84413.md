## Summary

## What This Changes

Report that repeated Codex app-server embedded runs leave unreaped `chrome-devtools-mcp` sidecars under the gateway service cgroup, growing memory until restart.

## Best Possible Solution

Define and test an explicit Codex app-server thread/MCP lifecycle that preserves native resume while bounding or reaping Chrome DevTools MCP sidecars.
