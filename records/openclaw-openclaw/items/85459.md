## Summary

Keep open: this protected, maintainer-labeled PR addresses a real Control UI first-use regression, but the diff reopens browser-origin silent pairing before the acknowledged shared-auth-generation guard for minted browser device tokens exists.

## What This Changes

The PR changes gateway handshake pairing policy to silently pair fresh Control UI WebChat browser-origin clients with valid shared auth again, updates gateway auth regression tests, and rewrites the changelog note for https://github.com/openclaw/openclaw/pull/81289.

## Best Possible Solution

Preserve the local Control UI first-use UX only after browser-origin auto-minted device tokens are bound to shared-auth generation, or keep browser-origin fresh devices on explicit pairing until maintainers accept a narrower guard.

## Reproduction Assessment

Yes. From source inspection of the PR diff and current call site, the patched helper would allow Control UI WebChat browser-origin `not-paired` handshakes into the silent auto-approval path, and current tests show device-token sessions survive shared-token rotation.

## Solution Assessment

No. Restoring silent browser pairing before generation-bound browser device tokens exists is not the safest fix; the maintainable path is to add that guard or keep explicit browser pairing until maintainers accept the tradeoff.
