## Summary

Keep open. This is an active draft security-boundary PR with contributor-provided live proof, but the proposed approval-boundary fallback appears incomplete for native file-change approvals because it does not pass deriveable file-target facts to path-based before_tool_call policy hooks.

## What This Changes

The PR updates the Codex app-server harness, native hook relay config, approval bridge, SDK runtime exports, tests, and docs so implicit Codex-native shell/file approvals run OpenClaw before_tool_call policy.

## Best Possible Solution

Revise the fallback so command and file approval paths deliver equivalent policy facts to before_tool_call, then land with focused regression tests and live Codex app-server proof.

## Reproduction Assessment

No independent high-confidence current-main live reproduction was established in this read-only pass. The linked issue and updated PR body provide a concrete installed Codex reproduction/proof path, and source inspection confirms current main lacks the new approval-boundary policy check.

## Solution Assessment

No. The command approval fallback is plausible, but the file-change branch should pass deriveable target paths before the patch fully satisfies the stated shell/file native policy boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Pass file targets to before_tool_call fallback:** `extensions/codex/src/app-server/approval-bridge.ts:337`
  - body: The file-change approval fallback is mapped to `apply_patch`, but the params only carry Codex's `changes` array. Convert `changes` into deriveable path facts or extend the policy derivation for this request shape.
