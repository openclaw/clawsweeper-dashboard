## Summary

Keep this PR open for maintainer review: current main already implements the main Codex attempt policy relay through https://github.com/openclaw/openclaw/pull/82496, but this branch still carries meaningful `/btw` side-conversation relay coverage that is not on main. The branch is not merge-ready as-is because its hook config reintroduces `matcher: null`, which current main explicitly removed to keep Codex trusted-hook hashes stable.

## What This Changes

The PR enables Codex native hook relay for public attempts and side conversations, promotes implicit YOLO approval to `untrusted` when OpenClaw pre-tool policy is active, and updates related tests, docs, and SDK metadata.

## Best Possible Solution

Rebase or replace the branch on current main, keep the matcher-free trusted hook-state implementation from https://github.com/openclaw/openclaw/pull/82496, and add the side-conversation relay coverage only after focused tests and live Codex proof pass.

## Reproduction Assessment

Yes for the original bypass path from the linked report and PR body, though the main-attempt path is already fixed on current main. The remaining `/btw` gap is source-reproducible because current main's side-question fork does not register or inject native hook relay config.

## Solution Assessment

No, not as-is. The side-question coverage is the right narrow direction, but the branch should be rebased onto current main and must not reintroduce `matcher: null` in the trusted hook config.
