## Summary

Keep this issue open. Current main has Copilot vision-capable catalog entries and explicit image-model paths, but `github-copilot` is still not declared as a media-understanding provider for automatic image fallback.

## What This Changes

Request to register `github-copilot` as an image-capable media-understanding provider so automated image understanding can use Copilot vision models.

## Best Possible Solution

Add Copilot media-understanding manifest metadata and coverage after maintainers choose its default model, priority, and live-validation bar; otherwise document explicit Copilot image-model configuration as the supported path.
