## What This Changes

The issue requests registering `github-copilot` as an eligible media-understanding image provider for automated image-description/OCR-style workflows.

## Best Possible Solution

If maintainers approve this provider behavior, add Copilot-owned `mediaUnderstandingProviders` manifest metadata/defaults plus focused registry/routing tests and live Copilot image proof; otherwise leave the explicit `agents.defaults.imageModel` and config-derived image-provider paths as the supported workaround.
