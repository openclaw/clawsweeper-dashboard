## Summary

Keep this issue open. Current main has improved Copilot model metadata by publishing a static catalog with image-capable model entries, but it still does not register `github-copilot` as a media-understanding provider, the plugin runtime does not call `registerMediaUnderstandingProvider`, and the media-understanding docs still omit Copilot from the image provider matrix.

## What This Changes

Register GitHub Copilot as an image-capable media-understanding provider so automated image understanding can use Copilot vision models.

## Best Possible Solution

Keep the fix plugin-owned: add a GitHub Copilot media-understanding provider using the existing generic image runtime helpers, declare the manifest contract and metadata, cover active-model and registry behavior with tests, and document explicit `agents.defaults.imageModel` use plus any intentionally chosen fallback behavior.

## Reproduction Assessment

Yes for the missing registration: current-main source inspection shows no Copilot media-understanding contract or runtime registration, and the registry only auto-registers explicit config providers with image-capable model entries. Live Copilot image success is a separate validation step, not needed to confirm the missing registration.

## Solution Assessment

The requested direction is reasonable, but the safest implementation is a narrow bundled GitHub Copilot plugin change rather than a core registry special case. Automatic fallback priority should be conservative or omitted until maintainers choose it deliberately.
