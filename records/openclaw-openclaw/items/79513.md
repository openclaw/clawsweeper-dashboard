## What This Changes

This PR replaces the Codex app-server shared singleton with a key-indexed pool, updates pooled clear/close behavior, adds regression coverage for alternate agent dirs and auth-token changes, and adds a changelog entry.

## Best Possible Solution

Land the keyed shared-client pool with the regression test, then track idle eviction separately only if real deployments show unbounded key churn.

## Reproduction Assessment

Yes. A focused source reproduction is to call getSharedCodexAppServerClient with one agentDir, then another, then the first again; current main includes agentDir in the key and clears the singleton when that key changes.

## Solution Assessment

Yes. Pooling by the existing compatibility key is the narrow maintainable fix because it preserves per-agent HOME/CODEX_HOME/auth isolation while stopping unrelated agents from evicting each other.
