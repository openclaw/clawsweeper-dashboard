## Summary

Keep open: the PR fixes the reported cross-agent singleton eviction, but the branch still regresses the existing auth/env rotation cleanup by retaining stale secret-keyed clients.

## What This Changes

The PR branch replaces the Codex shared app-server singleton with a map keyed by start/auth/agent options, adds agent-dir pooling test coverage, and adds a changelog entry.

## Best Possible Solution

Keep agentDir in the isolation key, but split or evict incompatible auth/env/start-option generations so per-agent warm clients coexist without retaining retired secret-bearing clients.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main includes agentDir in the shared-client key and clears the singleton on key mismatch, and the PR body adds terminal after-fix proof for the intended agent-dir reuse.

## Solution Assessment

Is this the best way to solve the issue? Not yet. A keyed pool is the right direction for agentDir isolation, but this implementation pools every full start-options key and loses the old auth/env rotation cleanup contract.

## Review Findings
