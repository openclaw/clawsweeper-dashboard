## Summary

Keep this PR open, but it still needs changes before merge: the keyed pool fixes the cross-agent eviction path, while the current branch keeps stale auth/env-keyed clients alive after token or start-option rotation.

## What This Changes

The branch replaces the Codex shared app-server singleton with a Map keyed by start/auth/agent compatibility, adds an agent-dir regression test, changes token-rotation expectations, and adds a changelog entry.

## Best Possible Solution

Keep the per-agent keyed warm-client pool, but evict and close stale entries when auth tokens, env values, or other start options rotate within the same agent scope.

## Reproduction Assessment

Yes. Source inspection shows current main deterministically clears the only shared client when agentDir changes, and the PR body supplies after-fix terminal proof for the intended per-agent reuse path; I did not run a live multi-agent Codex/OAuth setup.

## Solution Assessment

No. Pooling by the full compatibility key fixes cross-agent eviction, but the branch also pools retired auth/env/start-option keys indefinitely; a narrower fix should keep agentDir isolation while closing stale secret/start generations.
