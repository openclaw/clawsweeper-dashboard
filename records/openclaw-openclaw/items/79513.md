## Summary

Keep open: the PR fixes the source-reproducible Codex cross-agent eviction bug and has sufficient terminal proof, but the current diff also keeps retired auth/env-keyed app-server clients alive after token or start-option rotation, regressing the merged auth-rotation cleanup.

## What This Changes

This PR replaces the Codex shared app-server singleton with a Map keyed by compatible start/auth/agent options, adds shared-client regression coverage, and adds a changelog entry.

## Best Possible Solution

Keep the keyed per-agent pool, but evict and close stale auth/env/start-option generations so retired credentials are torn down while separate agentDir clients can stay warm.

## Reproduction Assessment

Yes. Source inspection shows current main includes agentDir in the key and clears the singleton on key mismatch, and the PR body demonstrates after-fix agent-dir reuse with a real shared-client process shim; I did not run a live OAuth/Codex overlap locally.

## Solution Assessment

No. Pooling by the full key fixes agentDir eviction, but it is broader than the maintainable fix because secret and env rotations should still close retired clients.
