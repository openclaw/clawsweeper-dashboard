## Summary

Keep this PR open: the linked Codex app-server eviction bug is still present on current main, and the patch appears to be the narrow keyed-pool fix with no blocking review findings.

## What This Changes

The PR changes the Codex app-server shared client from a single global slot to a key-indexed pool, adds regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the keyed shared-client pool with regression coverage, keep agentDir/auth/start options in the compatibility key, and leave dynamic fingerprint or idle-TTL tuning as separate follow-up work if needed.

## Reproduction Assessment

Yes. Current main deterministically computes different keys for different agentDir values and then clears the single global shared client on a key mismatch; a focused source-level reproduction is start agent A, start agent B, then request agent A again and observe a new start instead of warm reuse.

## Solution Assessment

Yes. A keyed pool is the narrow maintainable fix because it preserves per-agent HOME/CODEX_HOME/auth isolation while preventing unrelated compatible keys from evicting each other.
