## What This Changes

The branch remaps cron-run session keys to the heartbeat-drained queue for async exec, ACP stream, gateway node-event, and CLI watchdog events, with mainKey/scope threading, tests, and a changelog entry.

## Best Possible Solution

Land the enqueue-side cron-run remap if maintainers accept the slice, while leaving the broader external wake and CLI session-key contract to https://github.com/openclaw/openclaw/pull/78687 and keeping https://github.com/openclaw/openclaw/issues/52305 open until that remaining scope is resolved.

## Reproduction Assessment

Yes, source-level. Current main enqueues events under the provided cron-run key while heartbeat preflight peeks the resolved agent-main or global queue, so completion events can be stranded without this remap.

## Solution Assessment

Yes for this slice. Centralizing cron-run key remapping and threading mainKey/session scope through the affected enqueue and wake paths is the narrow maintainable fix, with the external wake-protocol work left separate.
