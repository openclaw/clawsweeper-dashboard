## Best Possible Solution

Yes, source-level. Current main enqueues system events under the original cron-run key while heartbeat preflight peeks only the resolved heartbeat session queue, so cron-run events can be stranded outside the queue the heartbeat drains.

## Solution Assessment

Yes for the narrowed fix direction, but not merge-ready as submitted. Centralizing cron-run key remapping and threading mainKey/session scope through the affected enqueue and wake paths is the narrow maintainable solution; the PR still needs the required changelog entry.
