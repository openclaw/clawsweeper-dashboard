## What This Changes

This PR remaps cron-run session keys to the agent-main or global heartbeat queue across exec, ACP relay, gateway node-event, and CLI watchdog event paths, threads session mainKey/scope through those callers, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this narrowed internal cron-run remap if maintainer review and exact-head checks remain clean, while leaving the external wake protocol and CLI session-key contract to the separate open companion PR.

## Reproduction Assessment

## Solution Assessment

Yes for this slice. Centralizing cron-run remapping and threading mainKey/session scope through the affected enqueue and wake callers is the narrow maintainable fix; the broader external wake/CLI contract is correctly separated.
