## Summary

Keep open: the PR targets the right shared thinking resolver/provider boundary for https://github.com/openclaw/openclaw/issues/84880, but it is not merge-ready because the PR has only test proof and its OpenAI prefix guard would let `gpt-5.3-chat-latest` override a current catalog `reasoning: false` row.

## What This Changes

The branch adds `overridesCatalogReasoning` to provider thinking profiles, uses it in shared thinking resolution, opts OpenAI/Codex profiles into the override, and updates docs, API baseline, and regression tests.

## Best Possible Solution

Consolidate on a single shared resolver fix that lets authoritative OpenAI/Codex profiles override stale catalog false only for reasoning-capable models, preserves off-only chat aliases, documents the public flag, and includes runtime proof.

## Reproduction Assessment

Yes at source level for the patch defect: current main declares `gpt-5.3-chat-latest` as `reasoning: false`, while this PR's `gpt-5.3` prefix and override-before-catalog resolver would accept non-off levels. The linked runtime bug is also source-reproducible from the resolver order, but I did not run a live subagent in this read-only review.

## Solution Assessment

No. The shared resolver override direction is sound, but this PR should exclude and test `gpt-5.3-chat-latest`, add real runtime proof, and be reconciled with the older open fixing PR before merge.
