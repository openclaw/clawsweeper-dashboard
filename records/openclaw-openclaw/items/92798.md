## Summary

Keep open, but the patch is not merge-ready: it changes MiniMax from the documented normalized `X% left` display to a formatter-only `% used` exception while current parser tests still treat MiniMax count fields as remaining quota.

## What This Changes

The PR adds MiniMax-specific usage formatting so MiniMax windows print `window.usedPercent` as `% used` while other providers continue to show `% left`, plus a formatter test for compact, summary, and report output.

## Best Possible Solution

Preserve the shared `X% left` formatter contract unless live MiniMax evidence proves the parser normalization is wrong; if so, fix `src/infra/provider-usage.fetch.minimax.ts` with focused parser tests instead of branching the shared formatter.

## Reproduction Assessment

Yes from source inspection: current main inverts `usedPercent` to `% left`, and MiniMax parser tests normalize remaining-count payloads into consumed `usedPercent`. I did not run a live MiniMax account flow.

## Solution Assessment

No. A formatter-level MiniMax exception is not the best fix while docs and parser tests define a normalized `% left` contract; provider-contract proof should drive any parser change instead.
