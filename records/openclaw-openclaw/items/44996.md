  - confidence: 0.9

  - confidence: 0.9
- **[P2] Port the bind validator to the current runner contract:** `src/cli/gateway-cli/run.ts:215-222`
  - body: Current main validates bind modes through the active `VALID_BIND_MODES` set and effective bind union. Updating this older validator block will not keep `openclaw gateway run --bind netbird` valid after rebase unless the current set, union, and help text are updated.
  - confidence: 0.87

  - body: Current main uses `src/wizard/setup.gateway-config.ts`, `setup.ts`, and `setup.types.ts`; the `onboarding.*` files changed here are no longer the active setup surface. After rebase, users would not see the NetBird option in the wizard they actually run.
  - confidence: 0.9

  - body: The PR advertises Control UI links at `https://<netbird-ip>:<port>`, but the default origin seeding only switches localhost and 127.0.0.1 to HTTPS. Browser origin checks exact-match the scheme and host, so the default NetBird UI path can still reject unless users manually add the NetBird origin.
  - confidence: 0.88
