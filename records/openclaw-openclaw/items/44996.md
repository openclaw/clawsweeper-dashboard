  - confidence: 0.88

  - body: Current main builds local gateway URLs through `src/gateway/connection-details.ts`, where TLS still depends only on explicit `gateway.tls.enabled`. Updating the older inline helper in `call.ts` leaves rebased local clients using `ws://` instead of NetBird auto-TLS.
  - confidence: 0.9

  - body: The PR switches daemon probes to `wss://` for NetBird, but current main only loads the local TLS runtime and fingerprint when `gateway.tls.enabled` is explicit. `openclaw daemon status --probe` can reject the generated self-signed cert unless users manually enable TLS.
  - confidence: 0.87

  - confidence: 0.9

  - body: Current main uses `src/wizard/setup.gateway-config.ts`, `src/wizard/setup.ts`, and localized setup labels for gateway choices. Adding the option only to the older onboarding files will not show NetBird in the active setup flow after rebase.
  - confidence: 0.88
- **[P2] Move Control UI links to the active helper:** `src/commands/onboard-helpers.ts:459-493`
  - body: Current main re-exports Control UI link resolution from `src/gateway/control-ui-links.ts`; `src/commands/onboard-helpers.ts` no longer owns the inline resolver. Updating the removed helper will not make dashboard/status/setup links use the NetBird host or HTTPS after rebase.
  - confidence: 0.88
- **[P3] Document the public bind mode:** `src/config/types.gateway.ts:3`
  - body: This adds a new public `gateway.bind` value, but the public gateway and CLI docs still list only the existing bind modes. Users following docs would not see how to configure or reason about the new NetBird mode.
