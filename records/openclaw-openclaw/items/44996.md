## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Seed the NetBird UI origin:** `src/config/gateway-control-ui-origins.ts:35-38`
  - body: The PR makes NetBird dashboard links use `https://<netbird-ip>:<port>`, but the seeded `gateway.controlUi.allowedOrigins` for `bind: "netbird"` only contains localhost/127.0.0.1 with an HTTPS scheme. `checkBrowserOrigin` compares the full Origin and non-local NetBird clients do not get the loopback fallback, so a default NetBird Control UI still rejects its websocket with `origin not allowed` until the user manually adds the NetBird IP origin.
  - confidence: 0.9
- **[P2] Pass the auto-TLS fingerprint to status probes:** `src/cli/daemon-cli/status.gather.ts:197-199`
  - body: This builds a `wss://` status probe URL whenever `bindMode === "netbird"`, but the later local TLS-runtime load remains gated on `daemonCfg.gateway?.tls?.enabled === true`. In the default auto-TLS NetBird config, the probe has a non-loopback `wss://<netbird-ip>` URL without a pinned fingerprint, and `GatewayClient` rejects non-loopback secure URLs that lack `tlsFingerprint`, so `openclaw gateway status --probe` can report a healthy gateway as unreachable.
  - confidence: 0.87
