## Summary

Keep open for maintainer review. Current main still lacks NetBird bind support and this PR addresses a real requested feature, but the patch has correctness blockers and is stale against moved gateway/setup surfaces.

## What This Changes

The PR adds a new `gateway.bind: "netbird"` mode with NetBird interface detection, auto-enabled self-signed TLS, CLI/onboarding/status/schema updates, tests, and a changelog entry.

## Best Possible Solution

Rebase onto current main and land a maintainer-approved NetBird bind design that detects supported NetBird interfaces cross-platform, seeds exact HTTPS Control UI origins for the advertised VPN URL, updates current setup/control-ui helpers/docs/tests, and keeps existing Tailscale behavior unchanged.

## Reproduction Assessment

Yes for the review findings by source inspection: upstream NetBird uses `utun100` on darwin while the PR scans only `wt*`, and OpenClaw's origin checker requires exact origins while the PR seeds only localhost origins. I did not run tests because this review was read-only and `pnpm` is unavailable in the checkout.

## Solution Assessment

No. The feature direction may be reasonable, but this patch is not the best current solution until it is rebased onto the live helpers and fixes cross-platform NetBird detection plus exact Control UI origin seeding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Seed the advertised NetBird origin:** `src/config/gateway-control-ui-origins.ts:35-38`
  - body: For `bind="netbird"`, the PR advertises `https://<netbird-ip>:port`, but the default origins added here include only localhost and 127.0.0.1. `checkBrowserOrigin` matches the full origin, so a fresh NetBird setup still rejects the Control UI WebSocket unless users manually edit `allowedOrigins`, which is the workflow this feature is meant to remove.
  - confidence: 0.93
- **[P2] Include NetBird's Darwin interface name:** `src/infra/netbird.ts:8`
  - body: NetBird's upstream source uses `WgInterfaceDefault = "utun100"` on darwin, while this detector only accepts interface names starting with `wt`. On macOS NetBird hosts the new bind mode falls back to loopback, so the VPN Control UI path does not work there; add the darwin case and regression coverage before enabling the mode.
  - confidence: 0.91
