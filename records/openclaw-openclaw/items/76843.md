## Summary

Keep open: the Docker migration docs address a real discoverability gap, but the branch should not merge while it tracks a machine-specific Docker overlay that current main deliberately generates and ignores.

## What This Changes

The PR adds Docker migration/setup documentation to README.md and docs/install/docker.md and adds a tracked docker-compose.extra.yml overlay for a local absolute-path migration.

## Best Possible Solution

Land a docs-only update that documents host-install-to-Docker migration with generic paths and leaves docker-compose.extra.yml generated and untracked.

## Reproduction Assessment

Not applicable for a docs PR. Source inspection and the PR diff are enough: current main supports generated extra compose files and persistence env vars, while current docs lack a dedicated migration section.

## Solution Assessment

No, not as currently written. The Docker guide is the right place for the migration workflow, but the committed local overlay should be removed and represented only as generic OPENCLAW_EXTRA_MOUNTS guidance.
