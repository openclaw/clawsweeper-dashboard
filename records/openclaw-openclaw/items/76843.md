## Best Possible Solution

Land this as a docs-only migration update with generic `OPENCLAW_EXTRA_MOUNTS` guidance and leave `docker-compose.extra.yml` generated and untracked.

Not applicable for a docs PR. Source inspection of current main plus the PR diff is enough: main already supports generated extra compose files, while current docs do not have a dedicated host-install-to-Docker migration section.

No, not as currently proposed. The migration documentation belongs in the Docker docs, but committing the local overlay is not the narrow maintainable solution because current main already has generated-overlay support.
