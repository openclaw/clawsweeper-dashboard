## Summary

Keep this PR open. The Discord hardening is scoped to the right plugin surface, but the latest PR head still stale-drops normal standard writes after 60 seconds, so it is not merge-ready for the automerge lane.

## What This Changes

The PR changes the bundled Discord plugin to add REST request lanes, gateway send/reconnect validation, command metadata comparison normalization, one-off component wait helpers, targeted tests, and a changelog entry.

## Best Possible Solution

Keep the Discord-local hardening, but constrain stale dropping to background requests so standard sends, edits, and deletes remain queued until sent or rejected by Discord.

## Reproduction Assessment

Yes. Use one REST worker, block the first request, enqueue a standard webhook or channel-message write, advance time past 60000 ms, then release the worker and the PR-head scheduler will reject the write as stale.

## Solution Assessment

No. The PR is scoped to the right Discord plugin surface, but default stale dropping on the standard lane is broader than the stated background-work policy and should be removed or made opt-in for standard writes.
