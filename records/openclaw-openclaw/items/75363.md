## Summary

Keep this PR open: the latest head is not implemented on main and still has a Discord REST lane regression where normal message writes can be stale-dropped; the earlier option-localization review concern is addressed by the current head.

## What This Changes

The PR updates the bundled Discord plugin's REST scheduler, gateway send validation, interaction/component helpers, command reconciliation, message-action component forwarding, tests, and changelog.

## Best Possible Solution

Keep the Discord-local hardening, but make stale dropping apply only to obsolete background work so user-visible writes stay queued until sent or fail by Discord response.

## Reproduction Assessment

Yes. A source-level reproduction is clear: block the single REST worker, enqueue a standard webhook or channel-message write, advance time past 60000 ms, then release the worker and observe the scheduler rejecting the write as stale.

## Solution Assessment

No. The PR is scoped to the right Discord plugin surface, but the stale-drop policy should not apply to normal standard writes.
