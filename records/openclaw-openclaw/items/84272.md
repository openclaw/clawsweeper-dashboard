## Summary

Keep open: current main still emits the retry-limit/expiry give-up warning without the saved delivery failure, while nearby code already captures a phase-aware `lastAnnounceDeliveryError` and macOS LaunchAgent docs confirm stderr is suppressed. The related subagent announce-loss threads are broader behavioral work, not a canonical replacement for this narrow gateway-log diagnostic fix.

## What This Changes

The issue asks for failed subagent completion retry-limit and expiry warnings to include the last underlying announcement delivery error in `gateway.log`.

## Best Possible Solution

Add an observability-only fix that logs the existing formatted delivery error through the normal gateway log path for retry-limit/expiry give-up and direct-attempt failures, without changing retry policy, delivery ordering, channel behavior, cleanup, or hook semantics.

## Reproduction Assessment

Yes at source level: set `lastAnnounceDeliveryError` on a subagent run that reaches retry-limit or expiry and the current give-up warning ignores it. I did not run tests because this review is read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is to reuse the existing formatted delivery error and log it through `defaultRuntime.log`, with regression coverage for retry-limit/expiry and direct-attempt failure logging.
