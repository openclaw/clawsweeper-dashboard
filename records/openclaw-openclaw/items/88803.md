## Summary

Keep this PR open: it addresses the linked `sessions_send`/Feishu lost-reply bug, but the current patch drops `deliveryContext.threadId` when inheriting external routes, so threaded external sessions can still be misdelivered.

## What This Changes

The PR changes reply routing so `sessions_send` inter-session webchat handoffs can inherit a target session's persisted external delivery context and adds focused route/dispatch tests.

## Best Possible Solution

Carry the complete persisted external delivery target, including `threadId`, only for inherited `sessions_send` external routes, while preserving the existing stale-thread guard for ordinary webchat and non-thread sessions.

## Reproduction Assessment

Yes. The linked report has concrete Feishu logs, and current main's source path shows non-system webchat turns ignore persisted external delivery context before dispatch suppresses internal webchat routing.

## Solution Assessment

No. The route-inheritance boundary is plausible, but it is not the best complete fix until the inherited route carries `threadId` and proves threaded channels are not routed to roots.
