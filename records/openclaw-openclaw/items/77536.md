## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep official diagnostics installs trusted:** `src/plugins/services.ts:24-29`
  - body: This change removes `trustedOfficialInstall` propagation and makes diagnostics services receive `internalDiagnostics` only when bundled. On current main, official installed diagnostics exporters such as npm-installed `@openclaw/diagnostics-prometheus` are deliberately marked trusted and granted the internal diagnostics event bus; after this patch they lose that context and metrics emission regresses.
  - confidence: 0.93
- **[P2] Do not block valid supervised gateway restarts:** `src/infra/process-respawn.ts:77-83`
  - body: The new guard treats service environment markers as proof that the current process is an unsafe child, but the managed gateway process itself also has those markers. `restartGatewayProcessWithFreshPid()` is used by the gateway run loop for normal SIGUSR1/restart handling; under systemd or launchd this would return `failed` instead of `supervised`, preventing the expected clean exit that lets the supervisor restart the gateway.
  - confidence: 0.89
