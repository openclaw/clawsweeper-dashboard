## Summary

Keep open: the PR fixes the right regression, but current head still lets an unset known provider env marker such as VLLM_API_KEY fall through as configured auth, so self-hosted wildcard discovery can run when the documented env opt-in is absent.

## What This Changes

The PR changes configured-provider auth resolution so explicit env SecretRefs resolve through the environment while bare all-caps apiKey strings remain literal provider keys, with focused auth-provenance coverage.

## Best Possible Solution

Keep the SecretRef-versus-literal split, but return undefined for missing known provider env markers and add a regression test for an absent VLLM_API_KEY or SGLANG_API_KEY marker.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure models.providers.vllm.apiKey as "VLLM_API_KEY" with no VLLM_API_KEY in env, and PR head falls through to a nonempty marker result instead of unavailable auth.

## Solution Assessment

No, not as currently written. The narrow maintainable fix is to return undefined after a missing known-marker env lookup while preserving explicit ${ENV} SecretRefs and bare all-caps literal keys.
