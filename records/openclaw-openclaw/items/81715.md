## Summary

Keep open: current main and the latest release still run the 30-minute synchronous TRUNCATE checkpoint, and an open linked fix PR references this issue with closing syntax, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests eliminating 30-minute synchronous SQLite TRUNCATE WAL checkpoints from the gateway event loop across SQLite-backed stores.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82366 if it remains acceptable; otherwise keep the fix in `src/infra/sqlite-wal.ts` by making periodic maintenance avoid blocking TRUNCATE while preserving controlled manual or shutdown cleanup.

## Reproduction Assessment

No for a full live reproduction: I did not run a 30-minute gateway with a large WAL and capture an 8-12s delay. Yes at source level: current main and the latest release schedule synchronous `TRUNCATE` checkpoints on the event loop, and the related production logs match that cadence.

## Solution Assessment

Yes for the current path: the linked hybrid PASSIVE-periodic/TRUNCATE-manual PR is a narrow maintainable fix with focused tests and real behavior proof. This issue should remain open until that PR is merged or rejected.
