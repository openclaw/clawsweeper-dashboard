## Summary

Keep open: current main and latest release still schedule a synchronous 30-minute SQLite WAL TRUNCATE checkpoint on the event loop, and an open fixing PR already references this issue with closing syntax.

## What This Changes

Avoid running default periodic SQLite WAL TRUNCATE checkpoints synchronously on the gateway event loop while preserving bounded WAL cleanup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82366 or an equivalent hybrid fix, then close this issue after merge and follow up only if idle stalls continue.

## Reproduction Assessment

## Solution Assessment

Yes: the linked hybrid PR is the narrowest maintainable direction because it moves the periodic default to PASSIVE while keeping TRUNCATE for explicit and close-time cleanup.
