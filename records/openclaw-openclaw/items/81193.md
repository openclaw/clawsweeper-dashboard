## Summary

Keep open: current main still documents and implements Active Memory as expecting `memory_recall` from `memory-lancedb`, and the reporter gives concrete v2026.5.7 logs where the embedded sub-agent resolves that allowlist but finds no registered tool. The broader external npm-plugin registry issue is related, but it does not fully supersede this report because this one shows `memory-lancedb` initialized and CLI commands working while only the embedded sub-agent tool surface is missing.

## What This Changes

The issue requests that npm-installed `@openclaw/memory-lancedb` expose its memory tools to Active Memory's embedded sub-agent when it owns the memory slot.

## Best Possible Solution

Add a packaged/npm `memory-lancedb` Active Memory regression proof, then fix the plugin-tool discovery or runtime registry path so selected external memory-slot tools materialize for embedded runs without broad-loading unrelated plugins.

## Reproduction Assessment

No high-confidence live current-main reproduction yet. The reporter provides detailed v2026.5.7 gateway logs and current source shows the embedded run uses `memory_recall` and handles the exact no-registered-tools error, but this review did not run npm-installed `memory-lancedb` against current main.

## Solution Assessment

Unclear until the packaged path is reproduced. The best fix should target the plugin-tool registry/materialization branch that drops the selected external memory plugin, not change the Active Memory allowlist or eagerly load unrelated plugins.
