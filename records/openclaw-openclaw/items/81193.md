## Summary

Keep open: current docs and source define `memory-lancedb` as an Active Memory recall backend via `memory_recall`, and the report gives concrete shipped-version logs showing that the npm-installed plugin loaded but the embedded recall run still found no callable memory tools. I did not find a merged fix or exact canonical duplicate; the closest open PR targets an adjacent embedded direct-lane allowlist bug and explicitly does not cover automatic npm plugin path discovery.

## What This Changes

The issue reports that npm-installed `@openclaw/memory-lancedb` tools are not materialized for Active Memory's embedded recall sub-agent when the memory slot selects `memory-lancedb`.

## Best Possible Solution

Add a packaged/npm regression for Active Memory with `memory-lancedb` selected, then fix external memory plugin tool discovery so `memory_recall` materializes for the embedded recall run while preserving manifest-first plugin loading and the empty-allowlist guard.
