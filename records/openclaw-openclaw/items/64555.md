## Summary

Keep this open. The issue includes credible reproduction steps and logs, the discussion already produced a focused but unmerged fix path, and current `main` plus the latest release still expose the implicated WhatsApp auth-dir resolution patterns.

## What This Changes

The issue requests fixing WhatsApp credential resolution so `--profile` gateways store and read Baileys auth state under the active profile's `$OPENCLAW_STATE_DIR/credentials/whatsapp/<account>` tree instead of the default profile.

## Best Possible Solution

Keep this issue open and land or recreate a focused WhatsApp fix that lazy-resolves the default web auth dir, threads explicit env/state-dir resolution through account discovery and default/legacy auth-dir helpers, and adds a regression test proving the default Baileys auth path stays under `OPENCLAW_STATE_DIR` for named profiles. Doctor and docs guidance for `OPENCLAW_OAUTH_DIR` can follow after the credential isolation path is corrected.

## Reproduction Assessment

Yes. The issue gives concrete profile-gateway steps and logs, and current source inspection still shows the exact path-resolution patterns that can place WhatsApp credentials outside the active profile boundary.

## Solution Assessment

Yes. A plugin-owned path-resolution fix with explicit env/state threading and a narrow regression test is the smallest maintainable correction; a docs-only workaround or broad core migration would leave the credential isolation bug in place.

## Security Review

Status: needs_attention

Summary: This issue reports a concrete credential/session isolation risk, and current main still contains the implicated WhatsApp path-resolution patterns.

Concerns:

- **[high] WhatsApp default auth dir can be pinned outside the active profile:** `extensions/whatsapp/src/auth-store.ts:39`
  - body: `WA_WEB_AUTH_DIR` is computed at module evaluation from `resolveOAuthDir()`, so an import before the intended profile env is authoritative can pin the default Baileys credentials directory to the main profile.
  - confidence: 0.88
- **[medium] Account auth discovery is not explicitly bound to profile state:** `extensions/whatsapp/src/accounts.ts:53`
  - body: WhatsApp account discovery and default/legacy auth-dir helpers resolve OAuth state from ambient process env instead of a passed env or state dir, which can drift from the multiple-gateway isolation contract.
  - confidence: 0.84
