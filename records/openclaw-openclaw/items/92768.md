## Summary

Keep open: current main and v2026.6.6 still leave `openclaw-macos` outside the same-family platform-version refresh path, and the open linked fix PR owns the implementation path before this issue should close.

## What This Changes

Allow paired macOS app reconnects to refresh macOS platform-version metadata without forcing re-pairing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89702, or an equivalent gateway fix, with focused metadata-pinning coverage and acceptable macOS reconnect proof.

## Reproduction Assessment

Yes, by source inspection: current main compares `openclaw-macos` platform strings literally, so `macOS 26.5.0` to `macOS 26.5.1` reaches the `metadata-upgrade` pairing path. I did not run a live macOS reconnect in this read-only review.

## Solution Assessment

Yes: extending the existing native platform-version normalization in the gateway is the narrowest maintainable fix. Extending request TTLs or changing the app prompt would leave the stale metadata mismatch in place.
