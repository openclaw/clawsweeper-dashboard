## Summary

Keep open: current main and v2026.6.6 still leave `openclaw-macos` out of the same-family platform-version refresh path, and the issue is paired with an open closing PR that should be reviewed or replaced before the issue closes.

## What This Changes

The issue reports that macOS app OS-version-only platform metadata changes are treated as mismatches and force re-pairing instead of silently refreshing paired metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89702, or an equivalent gateway fix, with focused metadata-pinning coverage and acceptable macOS reconnect proof.

## Reproduction Assessment

Yes by source inspection: current main sends versioned macOS platform strings and only normalizes iOS/Android same-family OS refreshes, so macOS point updates compare unequal and hit metadata-upgrade re-pairing. I did not run a live macOS reconnect.

## Solution Assessment

Yes: extending the existing same-family native platform refresh path to `openclaw-macos` in gateway metadata pinning is the narrow maintainable fix, while keeping device-family, public-key, role, and scope changes approval-bound.
