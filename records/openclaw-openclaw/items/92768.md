## Label Justifications

## Summary

Keep open: current main and the latest release still omit `openclaw-macos` from same-family platform-version refresh normalization, and the linked open PR is the right candidate fix path rather than a reason to close this issue now.

## What This Changes

The issue requests that macOS app OS-version-only platform metadata changes reconnect without forcing device re-pairing, matching the existing iOS and Android refresh behavior.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/89702 or an equivalent gateway fix lands with focused metadata-pinning coverage and accepted macOS reconnect proof.

## Reproduction Assessment

Yes by source inspection: a paired `openclaw-macos` client with a version-only `macos` platform string change does not hit the iOS/Android refresh normalization, so `platformMismatch` reaches metadata-upgrade pairing. I did not run a live macOS reconnect in this read-only review.

## Solution Assessment

Yes: extending the existing platform-version refresh decision to macOS at the metadata-pinning resolver is the narrow server-side fix. Changing only request TTLs or approval prompts would leave the stale metadata mismatch intact.
