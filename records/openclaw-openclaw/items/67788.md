## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P2] Expand help detection for all help flags:** `openclaw.mjs:166-168`
  - body: The launcher only disables lazy subcommands when there are more than four argv entries and a literal `--help`. That misses `openclaw models --help` and `-h` aliases such as `openclaw models auth -h`, so those paths can still render lazy placeholder help instead of the real nested command tree.
  - confidence: 0.9
- **[P2] Remove stale stop guidance from lock failures:** `src/cli/gateway-cli/run.ts:591-595`
  - body: This appends restart/status hints after the existing GatewayLockError message, which still tells supervised users to run `openclaw gateway stop`. A lock failure would now print both stop and restart guidance for the same supervised case, so keep one consistent restart/status recommendation.
  - confidence: 0.94
- **[P3] Avoid duplicating unreachable auth diagnostics:** `src/commands/gateway-status/output.ts:241-245`
  - body: `buildGatewayStatusWarnings` already promotes auth diagnostics for unreachable probes into the top-level Warning block. The new unconditional `Auth hint` prints the first same diagnostic again for failed targets, so gate it to reachable probes or skip diagnostics already shown in warnings.
  - confidence: 0.87
