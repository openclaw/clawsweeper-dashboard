## Summary

Keep open: current main and the latest release still have the source-level OpenRouter model-id mismatch, and an open closing PR now owns the implementation path but is not merged.

## What This Changes

The issue reports that OpenRouter-routed Anthropic model refs keep the internal `openrouter/` prefix in the OpenAI-compatible request payload, producing upstream 400s.

## Best Possible Solution

Land a narrowed OpenRouter-owned normalization that sends catalog slugs such as `anthropic/...` to OpenRouter, preserves valid slugs like `openrouter/auto`, adds regression coverage, and closes this issue after the PR merges.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` send the resolved OpenRouter `model.id` into the OpenAI-compatible payload unchanged, while the official OpenRouter catalog uses `anthropic/claude-sonnet-4.6` for the reported model.

## Solution Assessment

Yes on direction, but the current open PR is not yet the best final shape. Provider-owned normalization is the right layer, and it must be narrow enough to preserve valid OpenRouter slugs such as `openrouter/auto`.
