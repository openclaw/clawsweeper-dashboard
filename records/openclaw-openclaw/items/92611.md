## Summary

Close as implemented: current main and the v2026.6.6 release already normalize `openrouter/anthropic/...` before the OpenRouter request model reaches transport, and the linked broad OpenRouter hook patch was closed because it would break valid `openrouter/auto` routing.

## What This Changes

The issue reports that OpenRouter Anthropic refs such as `openrouter/anthropic/...` reach the OpenRouter API with the OpenClaw routing prefix still attached.

## Best Possible Solution

Keep the v2026.6.6 model-selection normalization; reopen only with a redacted current-version request payload or gateway log showing a path that still bypasses it.

## Reproduction Assessment

No. Current main and the v2026.6.6 release tree resolve the reported `openrouter/anthropic/...` ref to provider `openrouter` with model `anthropic/...` before probe/runtime execution.

## Solution Assessment

Yes. The best fix is the existing shared runtime model-selection normalization; stripping every `openrouter/` prefix inside the OpenRouter resolved-model hook would regress valid `openrouter/auto` behavior.
