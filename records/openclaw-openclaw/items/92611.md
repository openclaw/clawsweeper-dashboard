## Summary

Keep open: current main and the latest release still leave the OpenRouter resolved model ID unchanged before the OpenAI-compatible request payload, and the two open closing PRs have not landed yet.

## What This Changes

The issue reports that OpenRouter Anthropic model refs such as `openrouter/anthropic/claude-sonnet-4.6` are sent to the OpenRouter API with the internal `openrouter/` routing prefix.

## Best Possible Solution

Land a narrowed OpenRouter-owned normalization that converts duplicated routing refs like `openrouter/anthropic/...` before transport, preserves real OpenRouter API slugs such as `openrouter/auto`, and includes regression plus redacted live OpenRouter proof.

## Reproduction Assessment

Yes, at source level: current main leaves the OpenRouter runtime model ID unchanged and the OpenAI-compatible transport sends `model.id` as the request model. I did not run an authenticated live completion, but the unauthenticated OpenRouter catalog confirms the expected and unexpected slugs.

## Solution Assessment

Partly. The OpenRouter provider hook is the right owner boundary, but the implementation must not blindly strip every `openrouter/` prefix because `openrouter/auto` is a real OpenRouter slug.
