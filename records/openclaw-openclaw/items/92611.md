## Summary

Keep open: current main still preserves the `openrouter/` prefix through the OpenRouter resolved-model path, and this issue has open closing PRs that should be reviewed before the issue closes.

## What This Changes

The issue requests that OpenRouter Anthropic model refs keep `openrouter/...` for OpenClaw routing but send the OpenRouter API-facing model id without the `openrouter/` prefix.

## Best Possible Solution

Land a focused OpenRouter-owned fix that normalizes the resolved runtime model id before the OpenAI-compatible payload is built, preserves user-facing `openrouter/...` config refs, and adds regression coverage for prefixed and unprefixed OpenRouter model ids.

## Reproduction Assessment

Yes, source-level: current main keeps the OpenRouter runtime model id unchanged and the OpenAI-compatible request builder sends `model.id`. I did not run a live OpenRouter request in this read-only review.

## Solution Assessment

Yes: the best fix is provider-owned normalization in the OpenRouter resolved-model/request path, using the existing helper, rather than adding an OpenRouter special case to the shared OpenAI-compatible runtime.
