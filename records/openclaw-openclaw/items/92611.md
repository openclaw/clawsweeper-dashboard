## Summary

Keep open: current main still lacks OpenRouter model-id normalization before the OpenAI-compatible request payload, and the open closing PR has not merged yet. The issue remains necessary until a narrowed OpenRouter fix lands and proves both Anthropic refs and existing `openrouter/auto` behavior.

## What This Changes

The issue reports that OpenRouter Anthropic model refs keep the OpenClaw `openrouter/` routing prefix when sent to the OpenRouter API, causing 400 errors for `openrouter/anthropic/...` models.

## Best Possible Solution

Land a narrowed OpenRouter-owned normalization that strips `openrouter/` only for nested provider refs before transport, preserves valid OpenRouter API slugs such as `openrouter/auto`, and adds regression plus redacted live probe or completion proof.

## Reproduction Assessment

Yes, at source level. Current main passes `model.id` into the OpenAI-compatible request payload, and the OpenRouter provider hook does not rewrite `openrouter/anthropic/...` to the OpenRouter catalog slug before dispatch.

## Solution Assessment

Yes, provider-owned normalization in the OpenRouter path is the narrow maintainable direction. The active PR needs revision so it preserves valid OpenRouter slugs such as `openrouter/auto` while fixing nested Anthropic refs.
