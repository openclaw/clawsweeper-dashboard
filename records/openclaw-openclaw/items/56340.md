## Summary

Keep open: the Codex WebSocket regression portion is already covered on current main, but current main does not include the proposed soft/hard `before_tool_call` contract or the live `.profile` fallback changes, and the PR branch still has two concrete P2 correctness defects before it is mergeable.

## What This Changes

The PR narrows OpenAI WebSocket transport selection, adds soft/hard `before_tool_call` block semantics with approval handling, adjusts live test `.profile` loading, and updates related tests/docs.

## Best Possible Solution

Refresh the branch against current main, preserve the endpoint-aware WebSocket guard already on main, add `blockMode` to the current hook type and runner/consumer paths, clear approval state whenever a soft block is cleared, make profile loading continue until profile-derived variables or static parsing succeed, and update the public plugin docs plus focused regression tests.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Clear approval state when clearing soft blocks:** `src/plugins/hooks.ts:723-728`
  - body: When a lower-priority hook returns `{ block: false }`, this branch clears the soft block fields but the returned result still preserves `acc.requireApproval`. The consumer then prompts for approval even though the soft block was cleared, so the clear operation is incomplete.
  - confidence: 0.79
- **[P2] Require profile vars before stopping fallbacks:** `test/test-env.ts:92-97`
  - body: This loop returns after applying any new entry from the shell's full `env -0` output. If the first shell fails to source `.profile` but emits an unrelated absent variable, later shell candidates and static parsing are skipped, so profile-only live credentials can still be missed.
  - confidence: 0.74
