## Summary

Keep open: the PR is useful, but it still needs changes before merge because archive unlinking can bypass warn-mode no-mutation semantics, and the new docs misstate the reset archive retention default.

## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates docs, changelog, and tests.

## Best Possible Solution

Land archived-file cleanup after the apply path honors the resolved enforce mode and the docs match the current retention contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `session.maintenance.mode` to `warn`, place an old `.jsonl.deleted.<ts>` file beside the store, then run non-dry-run cleanup without `--enforce`; the PR still reaches `planArchivedSessionFileCleanup({ dryRun: false })`.

## Solution Assessment

No. The implementation direction is maintainable, but archived unlinking must be gated on the resolved enforce mode and the reset-retention docs should reflect that unset retention defaults to `pruneAfter`.
