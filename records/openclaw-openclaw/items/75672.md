## Summary

Keep open: the PR addresses a real sessions-cleanup gap and is mostly on the right path, but the latest diff still lets non-enforced warn-mode cleanup delete archived files and the new docs misstate the reset-archive default.

## What This Changes

The PR wires archived transcript and orphan checkpoint cleanup into `openclaw sessions cleanup`, adds an `archivedFiles` summary surface, and updates docs, changelog, and regression tests.

## Best Possible Solution

Land the archived cleanup after the apply path preserves warn-mode no-mutation semantics and the reset-retention docs match the existing default behavior.

## Reproduction Assessment

Yes. Source inspection of the PR diff gives a high-confidence path: configure `session.maintenance.mode` to `warn`, place an old `.jsonl.deleted.<ts>` file beside the store, then run non-dry-run cleanup without `--enforce`; the PR path reaches `planArchivedSessionFileCleanup({ dryRun: false })`.

## Solution Assessment

No. The implementation direction is maintainable, but archived unlinking must be gated on the resolved enforce mode and the docs should reflect that unset `resetArchiveRetention` defaults to `pruneAfter`.
