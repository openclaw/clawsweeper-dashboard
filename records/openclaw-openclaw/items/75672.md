## Summary

Keep open: the PR is useful, but the current branch is not merge-ready because it has a TypeScript binding error, weakens destructive cleanup semantics, can misreport apply-mode archive removals, has docs default drift, and still lacks real CLI behavior proof.

## What This Changes

The PR adds archived deleted/reset transcript and orphan checkpoint cleanup/reporting to `openclaw sessions cleanup`, plus docs, changelog, and focused tests.

## Best Possible Solution

Integrate archive cleanup reporting with the existing session maintenance and writer contracts so dry-run previews and apply summaries are accurate, warn mode remains non-destructive, checkpoint deletion keeps the age guard, and docs use public config keys/defaults.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproduction paths: `previewStoreCleanup` references an undefined `maintenance` binding, non-dry-run warn mode reaches archive unlinking, and fresh unindexed checkpoints can be queued without the current mtime cutoff.

## Solution Assessment

No. The cleanup/reporting direction is useful, but this implementation duplicates existing maintenance after mutation and weakens warn-mode, retention, and reporting contracts; extending the existing cleanup path is safer.

  - body: `previewStoreCleanup` has no `maintenance` binding, but the new planner call reads `maintenance.pruneAfterMs` and `maintenance.resetArchiveRetentionMs`, so the branch will fail TypeScript. Use `params.maintenance` here.
  - confidence: 0.98

  - body: A normal non-dry-run cleanup can still run in `warn` mode, but this new call passes `dryRun: false` for every non-dry-run invocation. That lets warn-mode cleanup delete archived files; keep this planner in preview mode unless the resolved mode is `enforce`.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes any checkpoint whose session id is absent from the index, regardless of mtime. Current unreferenced-artifact cleanup keeps the `olderThanMs` cutoff, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.91
- **[P2] Report archived removals before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:679-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner executes. The new `archivedFiles` summary can therefore report zero removals even though the same command removed files; collect counts from the existing helper or plan before applying.
  - confidence: 0.87
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
