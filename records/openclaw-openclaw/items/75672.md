## Summary

Keep this PR open. Current main still lacks the requested archived-file cleanup reporting and orphan-checkpoint cleanup in `openclaw sessions cleanup`, but the PR has two narrow pre-merge issues: enforce-mode JSON can underreport deleted/reset archive removals, and the new docs describe the retention config incorrectly.

## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup planning to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates tests, docs, and the changelog.

## Best Possible Solution

Land a corrected version that reuses the existing archive parsing, accounts for deleted/reset archive removals before store maintenance consumes them, cleans orphan checkpoints against the post-cleanup index, and documents the public retention knobs accurately.

## Reproduction Assessment

Yes. Place old `.jsonl.deleted.*`, `.jsonl.reset.*`, and orphan `.checkpoint.*.jsonl` files beside a `sessions.json`; current main dry-run does not report archived files, while the PR planner reports them. The enforce-summary mismatch is reproducible from source order: `updateSessionStore` can remove deleted/reset archives before the PR builds `archivedFiles`.

## Solution Assessment

No, not yet. The helper boundary is mostly right, but the enforce summary must not lose deleted/reset counts to existing store maintenance, and the docs need to match `session.maintenance.pruneAfter` plus the reset-retention default semantics.

  - confidence: 0.91
