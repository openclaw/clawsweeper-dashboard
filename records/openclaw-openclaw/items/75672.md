## Summary

Keep this PR open. The feature is still useful and current main does not provide the requested archived-file cleanup surface, but the force-pushed head still has a dry-run/apply mismatch for checkpoints orphaned by the same cleanup run and still lacks required user-facing docs/changelog coverage.

## What This Changes

The PR adds archived transcript cleanup planning to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and adds tests for deleted/reset archives and orphan checkpoints.

## Best Possible Solution

Land a revised version that plans archived-file cleanup from the post-cleanup referenced session IDs, documents the new output, adds the changelog entry, and leaves the broader async transcript performance work tracked separately in #75656.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a session entry that dry-run will prune, cap, or disk-budget evict plus a matching checkpoint file; this PR's dry-run reads the pre-cleanup store from disk while enforce runs after the store update.

## Solution Assessment

No. The implementation boundary is right, but the planner should accept the simulated post-cleanup session IDs so dry-run text/JSON match what `--enforce` will remove.
