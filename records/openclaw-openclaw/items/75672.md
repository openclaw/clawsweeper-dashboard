## Summary

Keep this PR open. Current main still lacks first-class cleanup for archived session transcript files, but this branch should not merge as-is because the new file-deletion path can bypass warn mode, dry-run can misreport checkpoint cleanup, and required release coverage is missing.

## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and adds focused planner tests.

## Best Possible Solution

Keep the focused CLI implementation, but gate archive deletion through the same maintenance mode semantics as existing cleanup, make dry-run use the simulated post-cleanup state and planned file removals, then add release/docs coverage before landing it for #75658.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with maintenance mode `warn`, create an old archive file and run non-dry-run cleanup without `--enforce`; the PR calls the archive planner with `dryRun: false` anyway. A second reproduction is a dry-run store entry that would be pruned or capped plus a matching checkpoint file, which the planner still treats as referenced because it reloads the current on-disk store.

## Solution Assessment

No, not in the current form. The implementation boundary is right, but the planner needs to share the cleanup command's simulated/enforced state and maintenance-mode contract before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Respect warn mode before unlinking archives:** `src/commands/sessions-cleanup.ts:581-586`
  - body: `sessions cleanup` can run in maintenance `warn` mode unless the user passes `--enforce`, and existing store maintenance only warns in that mode. This new call runs `planArchivedSessionFileCleanup` with `dryRun: false` for every non-dry-run invocation, so a bare cleanup can delete `.deleted`, `.reset`, and checkpoint files even when the rest of maintenance would not mutate. Gate this on enforce mode or make the planner dry-run in warn mode.
  - confidence: 0.9
- **[P2] Report checkpoints orphaned by dry-run cleanup:** `src/commands/sessions-cleanup.ts:369-374`
  - body: The dry-run path mutates `previewStore`, but then `planArchivedSessionFileCleanup` reloads the current store from disk. Checkpoints for sessions that this same cleanup would prune, cap, or evict are still counted as referenced in dry-run, while `--enforce` runs after the store update and removes them, so text and JSON under-report removals.
  - confidence: 0.87
- **[P2] Avoid double-counting disk-budget file removals:** `src/commands/sessions-cleanup.ts:369-374`
  - body: When disk-budget cleanup is configured, the dry-run `enforceSessionDiskBudget` call can already simulate removing archive or checkpoint files. The archive planner then scans the unchanged directory and reports the same files again, while enforce mode runs the archive planner after disk-budget deletion. Pass planned removed paths through or run the archive plan against the simulated file set.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/sessions-cleanup.ts:88`
  - body: This is a user-facing `sessions cleanup` feature with new text and JSON output, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires release-note coverage for user-facing `feat`/`fix` changes so operators can discover the new cleanup behavior.
  - confidence: 0.9
