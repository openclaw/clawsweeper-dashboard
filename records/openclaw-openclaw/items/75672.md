## Summary

Keep open: current main still lacks the requested archived-file cleanup, but this PR head can delete archived transcripts during warn-mode cleanup without `--enforce`, which violates the documented no-mutation contract. The changelog entry also needs a small contributor-credit fix before merge.

## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates CLI dry-run output, docs, changelog, and regression tests.

## Best Possible Solution

Land a corrected version that keeps the archive planner, summary, docs, and tests, but only unlinks archived files when the resolved cleanup mode is enforce and credits the contributor in the changelog.

Yes. From source inspection, configure `session.maintenance.mode` to `warn`, place an old `.jsonl.deleted.<ts>` file beside a store, and run non-dry-run cleanup without `--enforce`; PR head reaches `planArchivedSessionFileCleanup({ dryRun: false })`.

## Solution Assessment

No. The implementation direction is maintainable, but archive unlinking must be gated on the resolved enforce mode so warn-mode cleanup remains observational unless `--enforce` is passed.
