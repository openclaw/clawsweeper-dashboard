  - body: `mode` can still be `warn` when cleanup runs without `--enforce`, but this new call unlinks archive files with `dryRun: false` for every non-dry-run invocation. That makes ordinary warn-mode cleanup mutate disk unexpectedly; only apply archive deletion when the resolved mode is enforcing.

  - confidence: 0.9
- **[P2] Collect archive counts before store maintenance deletes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner executes. The new `archivedFiles` summary can therefore report zero removals even though the same command already removed those files; plan/report before applying or have the existing helper return counts.
  - confidence: 0.84
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
