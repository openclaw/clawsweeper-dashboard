## Summary

Keep open: the PR still allows archived transcript files to be removed during a non-dry-run cleanup when session maintenance resolves to `warn`, bypassing the existing no-mutation contract unless `--enforce` is used.

## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates CLI dry-run output, docs, changelog, and regression tests.

## Best Possible Solution

Land a corrected version that keeps the archive cleanup planner, summary, docs, and tests, but only unlinks archived files when the resolved cleanup mode is `enforce`.

## Reproduction Assessment

Yes. From source inspection, configure `session.maintenance.mode` to `warn`, place an old `.jsonl.deleted.<ts>` file beside a store, and run non-dry-run cleanup without `--enforce`; the PR path reaches `planArchivedSessionFileCleanup({ dryRun: false })` and would unlink it.

## Solution Assessment

No. The implementation direction is maintainable, but unlinking must be gated on the resolved enforce mode so warn-mode cleanup remains observational unless `--enforce` is passed.
