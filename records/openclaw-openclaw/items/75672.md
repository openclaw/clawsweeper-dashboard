## Best Possible Solution

Land after correcting the archived-files docs to use `session.maintenance.pruneAfter` and to state that reset archive retention defaults to `pruneAfter` unless set to `false`.

Yes, with the docs correction. Co-locating the planner with `runSessionsCleanup` after the controlled-writer refactor is the narrow maintainable boundary, and reusing the existing archive timestamp parser keeps the filesystem matching conservative.
