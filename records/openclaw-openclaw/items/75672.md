## Summary

Keep this PR open. It targets a real `sessions cleanup` gap from #75658, but the current diff still has accounting/parity bugs in the new archived-file summary and is missing the required user-facing changelog and CLI docs updates.

## What This Changes

The PR adds a `sessions cleanup` archive-file planner and tests for pruning old `.deleted`/`.reset` transcript archives plus orphan compaction checkpoints in the selected session-store directory.

## Best Possible Solution

Land a revised PR that computes archive cleanup from the same post-cleanup session state used by preview/enforce, reports files removed by existing store maintenance accurately, and updates the CLI docs and changelog.

## Reproduction Assessment

Yes. Create a store entry that dry-run will prune or cap plus a matching checkpoint file; the PR dry-run keeps that checkpoint counted as referenced from the pre-cleanup store, while enforce computes after mutation and can remove it.

## Solution Assessment

No. The implementation boundary is right, but the planner needs to share post-maintenance state and removal accounting with the existing store cleanup path before the new JSON/text output is reliable.
