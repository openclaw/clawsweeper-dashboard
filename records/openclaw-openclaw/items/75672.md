  - body: `mode` can still be `warn` when cleanup runs without `--enforce`, but this new call always executes the archive planner with `dryRun: false`. That lets warn-mode cleanup delete local archive files, so either skip archive deletion in warn mode or keep the planner in preview mode there.

  - confidence: 0.9
- **[P2] Plan archives before existing maintenance deletes them:** `src/config/sessions/cleanup-service.ts:679-684`
  - body: In enforce mode, `updateSessionStore` can already remove deleted/reset archives before this planner runs. The new `archivedFiles` summary can therefore report zero removals even though the command removed files, so collect counts before applying or return counts from the existing helper.
  - confidence: 0.86
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
