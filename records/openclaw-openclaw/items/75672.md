## Summary

Keep open: the feature is useful and current main does not yet expose archived-file cleanup in the session cleanup summary, but this PR has a blocking warn-mode deletion bug.

## What This Changes

This PR adds archived transcript file cleanup planning/removal to `sessions cleanup`, exposes an `archivedFiles` summary, and updates dry-run text, docs, changelog, and regression tests.

## Best Possible Solution

Land a corrected version that keeps the controlled-writer integration and dry-run parity, but gates physical archived-file deletion on effective enforce semantics.

## Reproduction Assessment

Yes, at source level. With `session.maintenance.mode: "warn"` and cleanup invoked without `--enforce`, current code passes `mode: "warn"` into `updateSessionStore`, but the PR then calls the archived-file planner with `dryRun: false` anyway.

## Solution Assessment

No. The direction is right, but the apply path must preserve warn-mode as preview/report-only and only unlink archived files when the effective mode is `enforce`.
