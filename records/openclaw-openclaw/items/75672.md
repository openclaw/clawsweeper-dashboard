## Best Possible Solution

Fold archive cleanup reporting into the existing session maintenance path so it uses `params.maintenance`, preserves warn/enforce and mtime-retention guards, records deleted/reset counts before mutation or from the existing helper, and documents only public config keys/defaults.

## Reproduction Assessment

Yes. Source inspection of PR head shows the undefined `maintenance` reference, non-dry-run warn mode reaching `dryRun: false` archive deletion, and orphan checkpoint removal without the current-main mtime cutoff; no runtime test was run because this was a read-only review.

## Solution Assessment
