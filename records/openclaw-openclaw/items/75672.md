The PR adds archived transcript and checkpoint file cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates CLI output, docs, changelog, and tests.

## Best Possible Solution

No. The implementation direction is maintainable, but it needs to gate archive unlinking on enforce-mode semantics and align the docs with the existing reset-retention resolver before merge.
