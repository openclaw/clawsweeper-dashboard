Full review comments:

- **[P1] Use the scoped maintenance config:** `src/config/sessions/cleanup-service.ts:524-525`
  - body: `previewStoreCleanup` does not have a `maintenance` binding in scope, so this call should use `params.maintenance`. As written, the PR head should fail typechecking before `sessions cleanup --dry-run` can build.
  - confidence: 0.98
- **[P2] Gate archive unlinking on enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can still be `warn` when cleanup runs without `--enforce`, but this new call passes `dryRun: false` for every non-dry-run invocation. That lets warn-mode cleanup delete archive files, so only execute archive deletion when the resolved mode is enforcing or keep this planner in preview mode for warn.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:336-350`
  - body: This branch removes any checkpoint whose session id is absent from the index, regardless of file mtime. Current unreferenced-artifact cleanup keeps an `olderThanMs` cutoff, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.9
- **[P2] Report archives before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:679-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner executes. The new `archivedFiles` summary can therefore report zero removals even though the same command removed files; collect counts from the existing helper or plan before applying.
  - confidence: 0.86
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
