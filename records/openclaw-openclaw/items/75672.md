- **[P1] Use the preview maintenance parameter:** `src/config/sessions/cleanup-service.ts:535-536`
  - body: `previewStoreCleanup` has no local `maintenance` binding, but this call reads `maintenance.pruneAfterMs` and `maintenance.resetArchiveRetentionMs`. Use `params.maintenance` here; as written the PR head fails type/build validation before cleanup can run.
  - confidence: 0.97
- **[P2] Keep warn-mode cleanup non-destructive:** `src/config/sessions/cleanup-service.ts:691-695`
  - body: When cleanup runs without `--enforce` and maintenance mode is `warn`, existing cleanup skips destructive artifact deletion, but this new planner still receives `dryRun: false`. Gate archive deletion on enforce mode or keep it preview-only in warn mode.
  - confidence: 0.94
- **[P2] Preserve the checkpoint age cutoff:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes orphan checkpoints solely because the session id is absent from the index. Current unreferenced-artifact cleanup also requires the file mtime to be older than the retention cutoff, which reduces races with fresh or in-flight checkpoint files.
  - confidence: 0.9
- **[P2] Count archives before existing maintenance removes them:** `src/config/sessions/cleanup-service.ts:691-695`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this post-update planner executes. That can make `archivedFiles` report zero even when the command removed archive files; collect counts from the existing helper or plan before mutation.
  - confidence: 0.88
- **[P3] Document public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to the prune window; only `false` disables reset archive cleanup.
  - confidence: 0.9
