## Summary

Keep open: the PR is a useful implementation direction for archived session cleanup, but the current diff still has destructive cleanup semantics that bypass warn mode, checkpoint age guards, and accurate apply reporting; it also still lacks contributor-supplied real CLI proof.

## What This Changes

The PR adds archived session transcript cleanup planning/reporting to `openclaw sessions cleanup`, updates dry-run output, docs/changelog, and adds archived-file tests.

## Best Possible Solution

Fold archive-file planning into the existing session maintenance cleanup contract so preview/apply share pre-mutation counts, warn mode stays non-destructive, checkpoint cleanup keeps the mtime/current-store guard, and docs use the public config keys.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the non-dry-run path calls the new archive planner with `dryRun: false` after store cleanup, and the checkpoint branch removes unindexed checkpoints without the current-main mtime guard; I did not run a write-producing CLI repro in this read-only review.

## Solution Assessment

No. The cleanup/reporting direction is useful, but this patch duplicates deletion after the writer and bypasses established warn-mode, age-guard, and reporting contracts; integrating counts with the existing helpers is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate archive cleanup on enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can still be `warn` for non-dry-run cleanup without `--enforce`, but this new call always passes `dryRun: false`. That makes warn-mode cleanup delete archived files even though current cleanup keeps unreferenced artifact removal non-destructive, so keep this planner in preview mode unless the resolved mode is enforcing.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes every checkpoint whose session id is absent from the index, regardless of mtime. Current unreferenced-artifact cleanup requires the file to be older than `pruneAfterMs`, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.9
- **[P2] Report archived removals before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner scans the directory. The new `archivedFiles` summary can therefore report zero removals and zero bytes even though the command removed files, so collect the counts before mutation or from the existing helper.
  - confidence: 0.88
- **[P3] Use the public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
