## Summary

Keep this PR open: current main still lacks first-class archive cleanup in `openclaw sessions cleanup`, but the branch still under-reports dry-run orphan checkpoint removals and is missing required release/docs coverage.

## What This Changes

The PR adds an archived session-file cleanup planner, an `archivedFiles` cleanup summary, and focused tests for deleted/reset archives and orphan checkpoints.

## Best Possible Solution

Land a revised PR that plans archived-file cleanup from the post-cleanup store state, updates changelog and CLI docs for the new output, and leaves the broader async transcript performance work tracked separately in #75656.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a session entry that dry-run will prune or cap plus a matching checkpoint file; the current PR's dry-run counts that checkpoint as referenced while `--enforce` plans after the store update and removes it.

## Solution Assessment

No. The implementation boundary is right, but the planner should consume the simulated post-cleanup session IDs for dry-run parity and the user-facing CLI output needs changelog/docs coverage.
