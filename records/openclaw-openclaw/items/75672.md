## Summary

This PR should stay open because the cleanup/reporting direction is useful for the linked sessions cleanup issue, but the current patch still has blocking cleanup semantics and lacks contributor-supplied real CLI proof.

## What This Changes

The PR adds archived deleted/reset transcript and orphan checkpoint cleanup planning/reporting to `openclaw sessions cleanup`, plus docs, changelog, and focused tests.

## Best Possible Solution

Extend the existing session maintenance cleanup contracts so preview/apply share counts before mutation, warn mode stays non-destructive, and checkpoint cleanup uses the established age-guarded artifact sweep.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows non-dry-run warn mode reaches `planArchivedSessionFileCleanup` with `dryRun: false`, and the checkpoint branch removes unindexed checkpoints without the current-main mtime cutoff; this read-only review did not run a write-producing CLI repro.

## Solution Assessment

No. The cleanup/reporting direction is useful, but this patch duplicates cleanup after the writer and bypasses established warn-mode, age-guard, and reporting contracts; folding the counts into the existing helpers is safer.

## Review Findings

Full review comments:

- **[P2] Gate archive cleanup on enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can still be `warn` for non-dry-run cleanup without `--enforce`, but this new call always passes `dryRun: false`. That makes warn-mode cleanup delete archived files even though the existing writer and unreferenced-artifact paths stay non-destructive, so keep this planner in preview mode unless the resolved mode is enforcing.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes every checkpoint whose session id is absent from the index, regardless of mtime. Current unreferenced-artifact cleanup requires the file to be older than `pruneAfterMs`, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.9
- **[P2] Report archived removals before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner scans the directory. The new `archivedFiles` summary can therefore report zero removals and zero bytes even though the command removed files, so collect the counts before mutation or from the existing helper.
  - confidence: 0.87
- **[P3] Use the public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
