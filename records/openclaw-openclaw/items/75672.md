## Summary

Keep this PR open: current main still lacks the requested archived-file cleanup, and the PR is directionally useful, but it should not merge until orphan checkpoint detection protects persisted checkpoint paths instead of assuming the checkpoint filename stem equals the live session id.

## What This Changes

The PR wires archived transcript cleanup into `openclaw sessions cleanup`, adds an `archivedFiles` summary, updates CLI/docs/changelog text, and adds regression coverage for deleted/reset/orphan checkpoint files.

## Best Possible Solution

Land the archived-file cleanup after the planner uses canonical referenced checkpoint paths, ideally by sharing the disk-budget reference logic or matching its contract, while keeping the deleted/reset retention behavior and public summary field.

## Reproduction Assessment

Yes at source level, though I did not run a live cleanup: current main computes only index and disk-budget cleanup, so old `.deleted`/`.reset` archives and non-budget orphan checkpoints are not surfaced in the cleanup summary. A focused fixture with aged archive files and a checkpoint in the store directory would reproduce the gap.

## Solution Assessment

No, not yet. The cleanup-service integration is the right direction, but the orphan-checkpoint logic should protect exact persisted checkpoint paths rather than session-id filename stems before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve referenced checkpoint files by path:** `src/config/sessions/cleanup-service.ts:269`
  - body: Checkpoint snapshots are named from the transcript file stem, not necessarily from `entry.sessionId`, and the store preserves the live reference in `compactionCheckpoints[].preCompaction.sessionFile`. With a custom or legacy `sessionFile` whose basename differs from the session id, this check treats a still-referenced checkpoint as orphaned and `--enforce` can delete the recovery snapshot. Please build the referenced set from canonical checkpoint paths, matching the existing disk-budget cleanup contract.
