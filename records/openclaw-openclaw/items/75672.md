## Summary

Keep open: the PR targets a real remaining gap in archive-file cleanup reporting, but the current patch still has destructive cleanup correctness issues and lacks contributor-supplied real CLI behavior proof.

## What This Changes

The PR adds archived transcript cleanup planning/reporting to `openclaw sessions cleanup`, exposes an `archivedFiles` summary field, updates CLI dry-run output/docs/changelog, and adds focused tests.

## Best Possible Solution

Integrate archive cleanup reporting with the existing maintenance path so counts are planned before mutation, warn mode remains non-destructive, checkpoint cleanup keeps the mtime safety guard, docs use public config keys/defaults, and the contributor supplies real CLI proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows non-dry-run warn mode reaches `planArchivedSessionFileCleanup` with `dryRun: false`, and the checkpoint branch removes unindexed checkpoints without the current-main mtime cutoff.

## Solution Assessment

No. The cleanup/reporting direction is useful, but the patch duplicates archive sweeping after existing maintenance and weakens established warn-mode, retention, and reporting contracts; it should collect or extend existing maintenance results instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate archive unlinking on enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can still be `warn` when cleanup runs without `--enforce`, but this new call passes `dryRun: false` for every non-dry-run invocation. That makes warn-mode cleanup delete archived files; keep this planner in preview mode unless the resolved mode is enforcing.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes any checkpoint whose session id is absent from the index, regardless of mtime. Current unreferenced-artifact cleanup keeps the `file.mtimeMs <= Date.now() - olderThanMs` cutoff, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.9
- **[P2] Report archived removals before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner executes. The new `archivedFiles` summary can therefore report zero removals even though the same command removed files; collect counts from the existing helper or plan before applying.
  - confidence: 0.86
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
