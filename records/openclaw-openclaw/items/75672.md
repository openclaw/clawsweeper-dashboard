## What This Changes

The PR adds archived transcript and orphan checkpoint cleanup to `openclaw sessions cleanup`, exposes an `archivedFiles` summary, and updates docs, changelog, and regression tests.

## Best Possible Solution

Repair the PR so archive cleanup preserves exact checkpoint paths referenced by store metadata and the public docs match the config schema/defaults, then land it to resolve #75658.

## Reproduction Assessment

Yes, source-level. A live entry whose `compactionCheckpoints[].preCompaction.sessionFile` points at a checkpoint file with a legacy/custom basename remains referenced by current metadata semantics, but the PR planner can classify it as orphaned because it checks only the filename-derived session id.

## Solution Assessment

No, not yet. Co-locating cleanup with `runSessionsCleanup` is maintainable, but checkpoint pruning should follow the existing exact-path reference contract and the docs should use public config names/defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve referenced checkpoint paths:** `src/config/sessions/cleanup-service.ts:269-271`
  - body: This keeps checkpoints only when the filename prefix matches a live `sessionId`. Existing metadata stores the authoritative `compactionCheckpoints[].preCompaction.sessionFile` path, and checkpoint names are derived from the transcript basename, so `cleanup --enforce` can delete a still-referenced checkpoint for sessions with legacy or custom transcript basenames. Build the preserved set from exact checkpoint paths instead.
  - confidence: 0.9
- **[P3] Correct archived-retention docs:** `docs/cli/sessions.md:105-106`
  - body: The new docs name the internal `session.maintenance.pruneAfterMs` field and say reset archives are left in place unless configured. The public key is `session.maintenance.pruneAfter`, and `resetArchiveRetention` defaults to that prune window unless explicitly set to `false`, so this points operators at the wrong config behavior.
  - confidence: 0.95
