## Summary

Keep this PR open. ## Best Possible Solution

Keep the PR open, fix dry-run/apply parity for checkpoints orphaned by the same cleanup run, add the required changelog and any CLI docs update for the new JSON/text summary, then land it as the focused implementation for #75658 after targeted tests and the changed gate pass.

Yes. A source-level reproduction is clear: create a store entry that dry-run will prune or cap plus a matching checkpoint file, then `--dry-run` will keep that checkpoint counted as referenced while `--enforce` runs the planner after the index update and removes it.
