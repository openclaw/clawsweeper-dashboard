## Summary

Keep open: the cleanup direction is useful, but the current PR still has blocking safety/reporting issues and only test/check output as proof. Current main partially covers session artifact cleanup, while the linked issue remains open for the missing archive dry-run/reporting behavior.

## What This Changes

The PR adds archived transcript cleanup planning and reporting to `openclaw sessions cleanup`, documents the `archivedFiles` JSON field, and adds archive-file cleanup tests.

## Best Possible Solution

Integrate archive planning/reporting with the existing store maintenance cleanup so dry-run and enforce share warn-mode, retention, age-guard, and JSON reporting semantics, then provide redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the blockers: non-dry cleanup invokes archive deletion regardless of warn/enforce mode, scans after existing store archive cleanup can already delete files, and checkpoint deletion lacks the current mtime cutoff.

## Solution Assessment

No. The feature should be integrated with the existing maintenance/archive and unreferenced-artifact cleanup paths so one code path owns deletion, retention, and reporting semantics.

- **[P2] Keep archive deletion behind enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can resolve to `warn` when cleanup runs without `--enforce`, and current cleanup avoids destructive artifact pruning in that mode.   - confidence: 0.93
- **[P2] Report archives before the writer removes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `updateSessionStore` already invokes archive retention cleanup during enforced maintenance, so by the time this planner scans, old `.deleted` and `.reset` files may already be gone. Applied JSON can then report zero removed files/bytes for archives that were actually deleted; move the reporting into the existing maintenance cleanup or plan before applying.
  - confidence: 0.88
- **[P2] Preserve the checkpoint age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: The existing unreferenced-artifact sweep only removes orphan checkpoint files after the `pruneAfter` mtime cutoff. This branch removes any checkpoint whose session id is absent from the index immediately, which can delete fresh or in-flight checkpoints; reuse the existing sweep semantics or apply the same age guard.
  - confidence: 0.89
- **[P3] Use public retention keys in archived-file docs:** `docs/cli/sessions.md:125-126`
  - body: The docs use the internal `pruneAfterMs` name and say unset reset retention leaves files in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that prune window while only `false` disables reset archive cleanup.
  - confidence: 0.91
