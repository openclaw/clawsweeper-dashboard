## Summary

Keep open: the PR addresses a real sessions cleanup gap, but the current head still has blocking safety issues around warn-mode deletion and checkpoint retention, and it still lacks real behavior proof from an actual cleanup run.

## What This Changes

The PR adds archive-file cleanup planning and reporting to `openclaw sessions cleanup`, updates docs/changelog output, and adds tests for deleted, reset, and orphan checkpoint files.

## Best Possible Solution

Land a rebased, scoped cleanup implementation that respects enforce semantics, preserves existing age/reference safety guards, documents public config keys and defaults, and includes redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: PR head runs archive deletion with `dryRun: false` regardless of warn/enforce mode, and checkpoint cleanup queues removal solely from missing index membership.

## Solution Assessment

No. The direction is useful, but the patch should reuse or match the existing cleanup retention and unreferenced-artifact semantics instead of adding a more aggressive parallel deletion path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate archive cleanup on enforce mode:** `src/config/sessions/cleanup-service.ts:675-680`
  - body: `mode` can be `warn` when cleanup runs without `--enforce`, and current cleanup deliberately avoids destructive artifact pruning in that mode. This new call still unlinks archived files with `dryRun: false`, so warn-mode cleanup can mutate disk unexpectedly; skip archive deletion unless the resolved mode is enforcing.
  - confidence: 0.93
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:335-349`
  - body: This branch removes any checkpoint whose session id is absent from the index immediately. Existing unreferenced-artifact cleanup only removes compaction checkpoints after the `pruneAfter` mtime cutoff, so fresh or in-flight checkpoint files can be deleted here; preserve the existing age guard or delegate checkpoint cleanup to that path.
  - confidence: 0.88
- **[P3] Correct archived retention docs:** `docs/cli/sessions.md:125-126`
  - body: The new docs use the internal `pruneAfterMs` name and say reset archives are left alone when retention is unset. The public config key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that prune window; only `false` disables reset archive cleanup.
  - confidence: 0.9
