## Summary

Keep open: current main still lacks archived-file cleanup and this PR is the right narrow implementation path, but the newly added CLI docs need a small retention-config correction before merge.

## What This Changes

The PR wires archived transcript cleanup into `openclaw sessions cleanup`, adds `archivedFiles` summary output, documents it, updates the changelog, and adds archived-file planner tests.

## Best Possible Solution

Land after correcting the archived-files docs to use public `session.maintenance.pruneAfter` wording and to state that reset archive retention defaults to `pruneAfter` unless set to `false`.

## Reproduction Assessment

Yes. The original issue can be reproduced at source level by placing old `.jsonl.deleted.*`, `.jsonl.reset.*`, and orphan `.checkpoint.*.jsonl` files beside a `sessions.json`; current main never reports them, while this PR plans them.

## Solution Assessment

Yes, with one docs fix. The code boundary is narrow and reuses existing archive parsing/session-store maintenance semantics; the safer alternative is only to correct the operator-facing retention wording before merge.
