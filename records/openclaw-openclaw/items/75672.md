## Summary

Keep open: the PR targets a real sessions cleanup gap and the author rebased it onto the controlled-writer refactor, but the current diff still has two merge-blocking cleanup safety regressions plus a docs mismatch.

## What This Changes

The PR adds archived transcript and checkpoint cleanup planning to `sessions cleanup`, exposes an `archivedFiles` text/JSON summary, and updates docs, changelog, and regression tests.

## Best Possible Solution

Land this feature after archived cleanup obeys warn/enforce semantics, preserves checkpoints by exact referenced paths, and documents the public retention keys and defaults.

## Reproduction Assessment

Yes, source-level. With `session.maintenance.mode: "warn"` and no `--enforce`, the PR still reaches archive deletion with `dryRun: false`; with a live checkpoint whose filename basename differs from `sessionId`, the PR treats the referenced file as orphaned.

## Solution Assessment

No, not yet. Co-locating archived-file cleanup with `runSessionsCleanup` is the right boundary, but deletion must be gated by enforce semantics and checkpoint liveness should use exact `preCompaction.sessionFile` references rather than filename prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Respect warn mode before deleting archives:** `src/config/sessions/cleanup-service.ts:481-487`
  - body: This always runs archived-file cleanup with `dryRun: false` after `updateSessionStore`, even when the resolved cleanup mode is `warn` and `--enforce` was not passed. Current cleanup treats warn mode as non-mutating, so a default `openclaw sessions cleanup` can now unlink archived files despite warn mode.
  - confidence: 0.94
- **[P2] Preserve referenced checkpoint paths:** `src/config/sessions/cleanup-service.ts:262-276`
  - body: This keeps checkpoints only when the filename prefix matches a live `sessionId`. Checkpoint filenames are derived from the transcript basename, while the authoritative live reference is `compactionCheckpoints[].preCompaction.sessionFile`, so a live session with a custom/topic/legacy transcript basename can lose its still-referenced snapshot during cleanup.
  - confidence: 0.92
- **[P3] Correct the archived-retention docs:** `docs/cli/sessions.md:105-106`
  - body: These docs use the internal `session.maintenance.pruneAfterMs` name and say reset archives are left in place unless configured. The public key is `session.maintenance.pruneAfter`, and `resetArchiveRetention` defaults to that prune window unless explicitly set to `false`.
  - confidence: 0.95
