## Summary

Keep open: this PR targets a real sessions cleanup gap, but the current patch still has destructive cleanup correctness issues and lacks contributor-supplied real CLI proof. Current main partially handles unreferenced artifacts, while this PR adds archive-file planning/reporting that still needs repair before merge.

## What This Changes

The PR adds archived transcript cleanup planning/reporting to `openclaw sessions cleanup`, adds an `archivedFiles` cleanup summary field, updates docs/changelog, and adds focused archived-file tests.

## Best Possible Solution

Integrate archive cleanup reporting with the existing maintenance helpers so dry-run and JSON can preview deleted/reset archive removals, apply-mode counts are collected before mutation, warn mode remains non-destructive, checkpoint cleanup keeps the age guard, and docs use public config keys/defaults.

## Reproduction Assessment

Yes. Source inspection of the PR head shows non-dry-run warn mode reaches the unlink planner, and a fresh unindexed checkpoint reaches deletion without the age cutoff used by current main; no write-producing local repro was run because this review is read-only.

## Solution Assessment

No. The requested cleanup/reporting direction is useful, but this implementation duplicates archive sweeps after existing maintenance and weakens established warn-mode, retention, and reporting contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate archive unlinking on enforce mode:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: `mode` can still be `warn` when cleanup runs without `--enforce`, but this call passes `dryRun: false` for every non-dry-run invocation. That makes warn-mode cleanup delete archived files, so keep the planner in preview mode unless the resolved mode is enforcing.
  - confidence: 0.94
- **[P2] Keep checkpoint cleanup behind the age guard:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: This branch removes any checkpoint whose session id is absent from the index, regardless of mtime. Current unreferenced-artifact cleanup keeps an age cutoff, so fresh or in-flight checkpoints can be removed here; reuse that sweep or apply the same cutoff.
  - confidence: 0.9
- **[P2] Report archived removals before maintenance deletes them:** `src/config/sessions/cleanup-service.ts:680-685`
  - body: In enforce mode, `updateSessionStore` already runs deleted/reset archive cleanup before this planner executes. The new `archivedFiles` summary can therefore report zero removals even though the same command removed files; collect counts from the existing helper or plan before applying.
  - confidence: 0.86
- **[P3] Use public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: The docs expose the internal `pruneAfterMs` name and say unset reset retention leaves reset archives in place. The public key is `session.maintenance.pruneAfter`, and unset `resetArchiveRetention` defaults to that window; only `false` disables reset archive cleanup.
  - confidence: 0.9
