## Summary

Keep open: current main and v2026.6.8 still do not provide the requested default-off Slack placeholder posted before agent work inside the Slack ACK window. Existing Slack draft/progress streaming can post and edit a message, but it starts from partial/progress callbacks or work evidence rather than an unconditional pre-dispatch placeholder.

## What This Changes

Request a default-off Slack option that posts a visible Thinking placeholder before agent dispatch, then edits it with the final reply or deletes it on no-reply outcomes.

## Best Possible Solution

Add a maintainer-approved, default-off Slack placeholder mode inside the Slack plugin that reuses the existing draft/finalization path and ships with schema, docs, focused tests, and live Slack timing proof.

## Reproduction Assessment

Yes at source level, not as a bug run: current main and v2026.6.8 have no Slack placeholder config or pre-dispatch send path. Existing progress and draft paths wait for partial/progress callbacks or work evidence.

## Solution Assessment

Unclear until maintainer product review. The likely best implementation should reuse the existing Slack draft stream and finalization helpers instead of adding a parallel placeholder stack, while reconciling overlap with https://github.com/openclaw/openclaw/issues/8285.
