## What This Changes

Add an opt-in Slack speculative placeholder message posted immediately during inbound handling, then edit it into the final reply or delete it on NO_REPLY.

## Best Possible Solution

Add a maintainer-approved, default-off Slack placeholder mode that starts the existing draft stream at inbound acceptance or reply start, then reuses current final-edit and clear paths with real Slack timing proof.
