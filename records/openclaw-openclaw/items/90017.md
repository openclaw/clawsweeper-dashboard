Existing Slack draft/progress streaming can post and edit messages, but it starts from partial/progress callbacks rather than an unconditional pre-dispatch placeholder, so this remains a product/config decision.

## What This Changes

Requests a Slack placeholder config that posts a visible “Thinking…” message immediately after inbound Slack receipt, then edits or deletes it when the turn finishes.

## Best Possible Solution

Add a maintainer-approved, default-off Slack placeholder mode that reuses the existing draft stream/finalization path and ships with schema, docs, focused tests, and live Slack timing proof.
