## Summary

Keep open: current main and latest release still do not provide the requested default-off Slack placeholder posted before agent work inside the Slack ACK window. Existing Slack draft/progress streaming can post and edit a message, but it starts from partial/progress callbacks or work evidence rather than an unconditional pre-dispatch placeholder.

## What This Changes

Request an opt-in Slack speculative placeholder message that appears before agent processing and is later updated with the final reply or removed on no reply.

## Best Possible Solution

Add a maintainer-approved, default-off Slack placeholder mode inside the Slack plugin that reuses the existing draft/finalization path and ships with schema, docs, focused tests, and live Slack timing proof.

## Reproduction Assessment

Yes at source level, not as a live bug run: current main exposes adjacent Slack ack reactions, typing reactions, and draft/progress streaming, but no placeholder config or pre-dispatch send path.

## Solution Assessment

Unclear until maintainer product review. The likely best implementation should reuse the existing Slack draft stream and finalization helpers instead of adding a parallel placeholder stack, while reconciling overlap with https://github.com/openclaw/openclaw/issues/8285.
