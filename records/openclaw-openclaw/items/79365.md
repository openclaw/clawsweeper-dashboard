## What This Changes

The issue requests discarding a resumed `claude-cli` session id after the first resume-watchdog FailoverError so subsequent attempts start cold.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79386 or a reconciled replacement that clears the stored resumed `claude-cli` binding on the poisoned timeout path, with focused regression coverage and redacted runtime proof.

Yes. A high-confidence source repro is to seed a stored `claude-cli` binding with a present transcript, have the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only clears `session_expired` or missing transcripts.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if scoped to resumed `claude-cli` timeout poison signals; lifting watchdog caps or broad model fallback would not break reuse of the same corrupted transcript.
