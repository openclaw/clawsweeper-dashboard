## Summary

Keep this issue open: current main still only recovers resumed CLI sessions for `session_expired` or missing transcripts, while a resumed `claude-cli` watchdog timeout can still rethrow without discarding the poisoned binding; the linked fix PR is open and should be reviewed rather than closing the issue early.

## What This Changes

Discard a poisoned `claude-cli` resume session id after a resumed no-output watchdog `FailoverError` so the next attempt starts cold instead of reusing the same corrupted transcript.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79386 or an equivalent narrow fix that clears the stored resumed `claude-cli` binding and retries cold only for the poisoned resume-timeout path, with regression coverage and real behavior proof.

## Reproduction Assessment

Yes. A high-confidence source repro is to seed a stored `claude-cli` binding, have the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only handles `session_expired`, leaving the timeout path unhandled.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if scoped to resumed `claude-cli` timeout poison signals. Lifting watchdog caps or broad model fallback would not break reuse of the same corrupted transcript.
