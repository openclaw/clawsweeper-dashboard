## Summary

Keep this issue open: current main still only clears and retries reused CLI sessions for `session_expired` or missing transcripts, while the reported resumed `claude-cli` watchdog timeout path still falls through with the poisoned binding intact; an open closing PR should be reviewed instead of closing the issue early.

## What This Changes

The issue requests clearing a reused `claude-cli` resume session id after a resumed no-output `FailoverError` so later attempts start cold instead of looping on poisoned state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79386 or an equivalent narrow fix that clears the stored resumed `claude-cli` binding and starts cold for the poisoned resume-timeout path, with regression coverage and real behavior proof.

## Reproduction Assessment

Yes. A high-confidence source repro is to seed a stored `claude-cli` binding, have the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only handles `session_expired` recovery.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if scoped to resumed `claude-cli` timeout poison signals; lifting watchdog caps or adding broad model fallback would not stop reuse of the same corrupted transcript.
