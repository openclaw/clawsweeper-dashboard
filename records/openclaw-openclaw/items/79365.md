## Summary

Keep this issue open: current main still handles resumed CLI cleanup only for `session_expired` or missing transcripts, while the reported `claude-cli` no-output watchdog path throws `FailoverError` with `reason: "timeout"` and can leave the poisoned session binding reusable. The open closing PR at https://github.com/openclaw/openclaw/pull/79386 should be reviewed or replaced rather than closing this issue early.

## What This Changes

Request to discard a poisoned resumed `claude-cli` session id after the first watchdog `FailoverError` so the next attempt starts cold.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79386, or a narrow replacement, after it clears the stored resumed `claude-cli` binding reliably and includes focused regression coverage plus accepted runtime proof.

## Reproduction Assessment

Yes. A high-confidence source path is to seed a stored `claude-cli` binding with a present transcript, have the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only clears `session_expired`.

## Solution Assessment

Yes, the requested behavioral direction is the narrow maintainable fix for this cascade. The implementation should clear the persisted binding before or independently from any cold retry so the poisoned id cannot survive a retry failure.
