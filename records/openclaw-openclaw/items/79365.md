## Summary

Keep this issue open: current main still handles resumed CLI recovery for `session_expired` and missing transcripts, but a resumed `claude-cli` watchdog `FailoverError` with reason `timeout` still falls through without clearing the poisoned binding. The directly closing PR is open and should be reviewed rather than closing the issue early.

## What This Changes

The issue requests clearing the persisted `claude-cli` resume id after the first resumed-session watchdog `FailoverError` so the next attempt starts cold.

## Best Possible Solution

Land one canonical narrow fix that clears the resumed `claude-cli` binding and starts cold after a poisoned timeout, with focused regression coverage and proof for the real recovery path.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed a stored `claude-cli` binding, make the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only special-cases `session_expired`.

## Solution Assessment

Yes. Clearing the poisoned resume id for a resumed `claude-cli` timeout is the narrow maintainable direction; increasing watchdog caps or broad model fallback would not prevent reuse of the same corrupted transcript.
