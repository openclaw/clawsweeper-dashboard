## Summary

Keep this issue open: current main still retries/clears resumed CLI sessions only for `session_expired` and missing transcripts, while a resumed `claude-cli` watchdog timeout still falls through without discarding the poisoned binding. The open closing PR at https://github.com/openclaw/openclaw/pull/79386 is the right implementation path to review rather than closing this issue early.

## What This Changes

Discard a persisted `claude-cli` resume session id after a resumed watchdog `FailoverError` so the next attempt starts cold instead of reusing poisoned state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79386 or an equivalent narrow fix that clears the stored resumed `claude-cli` binding and retries cold only for the poisoned resume-timeout path, with regression coverage and real behavior proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed a stored `claude-cli` binding with a present transcript, have the resumed attempt throw `FailoverError` reason `timeout`, and observe that current main only handles `session_expired`.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if scoped to resumed `claude-cli` timeout poison signals. Raising watchdog caps or broad model fallback would not stop reuse of the same corrupted transcript.
