## Summary

Keep open. Current main still serves a Control UI CSP that omits `https://tweakcn.com` while the documented Appearance importer performs a direct browser fetch to the tweakcn registry, so the reported failure remains valid.

## What This Changes

Allow the Control UI tweakcn theme importer to fetch tweakcn registry theme JSON without being blocked by the Control UI CSP.

## Best Possible Solution

Pick the approved egress boundary, then add either a host-specific tweakcn `connect-src` allowance or a Gateway import proxy with URL validation, redirect rejection, size limits, schema checks, regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main path: the Control UI CSP omits tweakcn while the Appearance importer performs a browser fetch to `https://tweakcn.com/r/themes/<id>`.

## Solution Assessment

Unclear until maintainers choose the egress boundary. A host-specific CSP allowance is the smallest fix for the current direct-fetch design; a Gateway proxy is safer if third-party browser egress should stay blocked.
