## Summary

Keep open. Current main and latest release still omit `https://tweakcn.com` from the Control UI CSP while the documented Appearance importer fetches tweakcn directly; open PR #71886 is the relevant repair path but has not landed.

## What This Changes

The issue asks for tweakcn theme import to work instead of failing because the Control UI CSP blocks direct registry fetches.

## Best Possible Solution

Land a rebased #71886-style fix that preserves `https://api.openai.com`, adds exact `https://tweakcn.com`, updates CSP assertions, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main path: the Control UI CSP omits tweakcn while the Appearance importer performs a browser fetch to `https://tweakcn.com/r/themes/<id>`.

## Solution Assessment

Yes in direction, but not via the stale PR branch as-is. A host-specific CSP allowance is the narrowest fix for the documented direct-browser import design; a proxy would only be needed if maintainers reject third-party browser egress.
