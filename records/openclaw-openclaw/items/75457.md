## Summary

Keep open. Current main still has the reported CSP/import mismatch, and there is already an open fix PR (#71886) for the same behavior, so the right cleanup path is to review and land that PR or an equivalent fix rather than close this issue early.

## What This Changes

The issue reports that Control UI tweakcn theme imports fail because the browser-direct fetch to `https://tweakcn.com/r/themes/<id>` is blocked by Control UI's `connect-src` CSP.

## Best Possible Solution

Land PR #71886 or an equivalent narrow fix that aligns the served Control UI CSP with the documented browser-direct tweakcn importer, including a regression assertion and the required changelog entry.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. On current main, a served Control UI page applies a `connect-src` policy that excludes `https://tweakcn.com`, while the Appearance importer calls browser `fetch()` for `https://tweakcn.com/r/themes/<id>`, so a valid theme import is blocked by browser CSP.

## Solution Assessment

Is this the best way to solve the issue? Yes, for the current documented architecture. Adding one exact HTTPS origin to `connect-src` is the narrowest maintainable fix for the browser-direct importer; a Gateway proxy is the safer alternative only if maintainers reject direct third-party browser fetches.
