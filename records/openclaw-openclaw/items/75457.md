## Summary

Keep open. Current main still serves a Control UI CSP that blocks `https://tweakcn.com` while the documented Appearance importer performs a direct browser fetch to the tweakcn registry, so the reported failure remains valid.

## What This Changes

The issue asks Control UI tweakcn theme imports to work for valid tweakcn share links despite the current `connect-src` restriction.

## Best Possible Solution

Approve one egress boundary, then implement the narrow fix with matching CSP/proxy tests, tweakcn URL and payload validation, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main path: the Control UI CSP omits tweakcn while the Appearance importer performs a browser fetch to `https://tweakcn.com/r/themes/<id>`.

## Solution Assessment

Unclear until maintainers choose the egress boundary. A host-specific CSP allowance is the smallest fix for the existing direct-fetch design; a Gateway proxy is safer if third-party browser egress should remain blocked.
