## Summary

Keep open. Current main still serves a Control UI CSP that excludes tweakcn while the Appearance importer performs a direct browser fetch to tweakcn; the related open fix PR is not merged and still needs rebase/adjustment before this can close.

## What This Changes

The issue reports that importing a tweakcn Control UI theme fails because the browser fetch to `https://tweakcn.com/r/themes/<id>` is blocked by the served `connect-src` CSP.

## Best Possible Solution

Land or replace PR #71886 with a current-main patch that allows both `https://api.openai.com` and `https://tweakcn.com` in `connect-src`, keeps exact CSP assertions, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: the served CSP excludes tweakcn, the Appearance importer calls browser `fetch()` for the tweakcn registry URL, and the UI displays the resulting fetch error.

## Solution Assessment

Yes for the current browser-fetch design. A host-specific `https://tweakcn.com` `connect-src` addition is the narrowest fix, provided it preserves existing allowed origins and does not expand to wildcard HTTPS.
