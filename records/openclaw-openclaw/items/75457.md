## Summary

Keep open. Current main and the latest v2026.5.2 release still serve a Control UI CSP that omits `https://tweakcn.com`, while the documented Appearance importer directly fetches `https://tweakcn.com/r/themes/<id>`, so the reported failure remains valid.

## What This Changes

The issue asks for tweakcn theme import to work despite the Control UI CSP currently blocking direct registry fetches.

## Best Possible Solution

Maintainers should choose the approved egress boundary, then add either a host-specific tweakcn `connect-src` allowance or a Gateway import proxy with validation, regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main path: the Control UI CSP omits tweakcn while the Appearance importer performs a browser fetch to `https://tweakcn.com/r/themes/<id>`.

## Solution Assessment

Unclear until maintainers choose the egress boundary. A host-specific CSP allowance is the smallest fix for the existing direct-fetch design; a Gateway proxy is safer if third-party browser egress should remain blocked.
