## Summary

Keep open. Current main still serves a Control UI CSP that excludes tweakcn while the Appearance importer performs a direct browser fetch to the tweakcn registry, and the related duplicate report is already closed back to this issue rather than superseding it.

## What This Changes

Allow the Control UI tweakcn theme importer to fetch `https://tweakcn.com/r/themes/<id>` under the served CSP.

## Best Possible Solution

Pick the approved egress boundary, then add either a host-specific tweakcn `connect-src` allowance or a Gateway import proxy with URL validation, redirect rejection, size limits, schema checks, regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: Control UI receives a CSP without `https://tweakcn.com`, while the Appearance importer calls browser `fetch()` for `https://tweakcn.com/r/themes/<id>`.

## Solution Assessment

Unclear until maintainers choose the egress boundary. A host-specific CSP allowance is the smallest fix for the current direct-fetch design; a Gateway proxy is safer if third-party browser egress should stay blocked.
