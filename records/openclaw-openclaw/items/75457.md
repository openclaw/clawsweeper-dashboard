## Summary

Keep open. Current main still serves the Control UI with a `connect-src` policy that excludes tweakcn while the Appearance importer still performs a direct browser fetch to the tweakcn registry; the remaining fix requires a maintainer decision on the Control UI egress boundary.

## What This Changes

The issue asks for tweakcn theme import in the Control UI to work instead of failing under the current CSP restriction.

## Best Possible Solution

Pick the approved architecture, then add either a host-specific tweakcn `connect-src` allowance or a Gateway proxy while preserving URL validation, redirect rejection, timeout, payload size limits, schema checks, regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: the served Control UI CSP excludes `https://tweakcn.com`, and the Appearance importer calls browser `fetch()` for the tweakcn registry URL.

## Solution Assessment

Unclear until maintainers choose the egress boundary. A host-specific CSP allowance is the smallest change for the current direct-fetch design, while a Gateway proxy is safer if third-party browser fetches should remain blocked.
