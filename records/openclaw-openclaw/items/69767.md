## Summary

Keep open. Current main still has a source-backed path that turns missing subagent output into `(no output)` and sends it as the requester-facing completion result, while the active hardening PR remains open and unmerged.

## What This Changes

The issue asks OpenClaw to defer async/subagent completion delivery until a usable result payload or safe fallback exists.

## Best Possible Solution

Land the existing hardening direction or a narrower successor that persists a user-safe completion payload, retries/defer delivery while no payload exists, and replaces the `(no output)` expectation with fail-closed regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main has a focused test path where child history contains only user text, delivery still succeeds, and the requester prompt contains `(no output)`, with reporter logs showing the same class of missing-payload completion in OpenClaw 2026.4.15.

## Solution Assessment

No. Current main is not the right solution because it promotes a placeholder as a completion result; the maintainable fix is to defer/retry or persist a real user-safe payload before delivery.
