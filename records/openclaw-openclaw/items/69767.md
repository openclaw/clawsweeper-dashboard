## Summary

Keep open. Current main still can deliver `(no output)` as the requester-facing subagent completion result when no child assistant/tool payload is available, and the focused regression coverage still expects that behavior while #68464 remains open and unmerged.

## What This Changes

The issue requests fail-closed async/subagent completion delivery so requester sessions do not receive completion events before a usable result payload exists.

## Best Possible Solution

Land #68464 or a narrower successor that persists a user-safe completion payload or retryable delivery state, defers requester delivery until real child output or a sanitized fallback exists, and changes the `(no output)` expectation into fail-closed regression coverage.

## Reproduction Assessment

Yes. Current main has a focused test path where child history contains only user text, completion delivery still succeeds, and the requester prompt contains `(no output)`; the issue discussion also includes production reproduction details from OpenClaw 2026.4.15 on Control UI and Telegram.

## Solution Assessment

No. The current main behavior is not the best solution because it promotes a placeholder as the completion result; the narrower maintainable fix is to defer/retry or persist a user-safe payload before delivery.
