## Summary

Keep open. Current main still promotes missing subagent output into a requester-facing completion event with `(no output)`, and the focused regression test still asserts that behavior. The issue discussion also has a production reproduction on OpenClaw 2026.4.15, while PR #68464 remains the active unmerged hardening candidate.

## What This Changes

The issue asks OpenClaw to defer async/subagent completion announcements until a usable result payload, structured payload, or sanitized fallback exists.

## Best Possible Solution

Land PR #68464 or a narrower successor that persists a user-safe completion payload or retryable delivery state, defers completion delivery until real child output or a sanitized fallback exists, and changes the current `(no output)` expectation into a fail-closed regression test.

## Reproduction Assessment

Yes. Current main has a focused test path where child history contains only user text, `expectsCompletionMessage` is true, and the announce still succeeds with `(no output)`; the issue discussion also includes production reproduction details from Telegram and Control UI.

## Solution Assessment

No. The current implementation is not the best way to solve the issue because it promotes a placeholder as the completion result; the narrower maintainable fix is to defer/retry or persist a user-safe payload before delivery.
