## Summary

Keep open. Current main still has a source-backed path that turns missing subagent output into `(no output)` and sends it as the requester-facing completion result, while the main hardening PR for persisted user-safe payloads remains open.

## What This Changes

The issue reports a race where async/subagent completion can be surfaced to the requester before any usable result payload exists.

## Best Possible Solution

Land the existing hardening direction or a narrower successor that persists a user-safe completion payload, defers or retries delivery while no payload exists, and replaces the `(no output)` expectation with fail-closed regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main has a focused test path where child history contains only user text, delivery still succeeds, and the requester prompt contains `(no output)`. I did not run a live Telegram/OpenRouter reproduction in this read-only review.

## Solution Assessment

No, current main is not the best solution because it promotes a placeholder as a completion result. The maintainable fix is to fail closed by deferring/retrying or persisting a real user-safe payload before announcing completion.
