## Summary

Keep this PR open: the Mattermost thread-file direction addresses a real source-level gap, but the submitted diff still has merge-blocking correctness issues against current `main` and lacks inspectable after-fix Mattermost proof.

## What This Changes

The PR hydrates incomplete inbound Mattermost posts, collects file IDs from earlier thread posts, dedupes them, and routes them through the existing inbound media resolver with new resource tests.

## Best Possible Solution

Land a rebased Mattermost-owned fix that preserves debounced text, uses the current resource test contract and mocks, and proves in a real Mattermost thread that root or earlier reply attachments are included in inbound media.

## Reproduction Assessment

Yes for source reproduction: current `main` computes thread context but still resolves inbound media from only `post.file_ids`, so files on a thread root or earlier reply are omitted. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

No, not as submitted. The thread-file lookup is the right boundary, but the hydration merge must preserve synthetic debounced text and the tests/mocks must match the current `saveRemoteMedia` resource API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve debounced text during hydration:** `extensions/mattermost/src/mattermost/monitor.ts:1248`
  - body: When multiple Mattermost posts are debounced, `onFlush` synthesizes one post with `message: combinedText` and `file_ids: []`. This hydration path spreads the fetched REST post over that synthetic post, so the last post's REST `message` overwrites `combinedText` and earlier debounced messages are lost before routing.
  - confidence: 0.9
- **[P2] Update thread-file tests for saveRemoteMedia:** `extensions/mattermost/src/mattermost/monitor-resources.test.ts:152-153`
  - body: Current `main` removed the `fetchRemoteMedia`/`saveMediaBuffer` resource contract in favor of `saveRemoteMedia`. The new thread-file tests still pass the removed fields and omit `saveRemoteMedia`, so they will fail after rebasing or merging with current `main`.
  - confidence: 0.93
- **[P2] Add the thread-file helper to monitor test mocks:** `extensions/mattermost/src/mattermost/monitor.ts:1512-1516`
  - body: The inbound monitor tests mock `createMattermostMonitorResources` without `resolveMattermostThreadFileIds`. The new call runs for normal inbound posts before dispatch, so those suites need the mock helper or they will call `undefined` when the monitor path is exercised.
  - confidence: 0.88
