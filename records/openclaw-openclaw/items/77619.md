## Summary

Keep open: current main still ignores Mattermost thread/root attachments for inbound media, but this PR is not mergeable as submitted because hydration can clobber debounced combined text, the branch needs current-main test API repairs, the inbound monitor mock is missing the new helper, and real behavior proof is insufficient.

## What This Changes

The PR adds Mattermost inbound post hydration, thread file ID collection, file ID deduplication, and resource tests so thread/root attachments can flow through the existing inbound media downloader.

## Best Possible Solution

Preserve debounced combined message text while hydrating only the needed REST fields, update the new resource tests to `saveRemoteMedia`, add the thread-file helper to monitor test mocks, and require redacted live Mattermost proof before merge.

## Reproduction Assessment

Yes for source reproduction: current main computes thread context but resolves inbound media from only `post.file_ids`, so root or earlier reply attachments are ignored. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

No, not as submitted. The runtime direction is narrow and uses the right Mattermost thread API shape, but hydration must not overwrite synthesized debounced text and the tests/mocks must be rebased to current main before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve debounced text during hydration:** `extensions/mattermost/src/mattermost/monitor.ts:1248`
  - body: When multiple Mattermost posts are debounced, `onFlush` synthesizes one post with `message: combinedText` and `file_ids: []`. This new hydration path runs before dispatch and spreads the fetched REST post over that synthetic post, so the last post's REST `message` overwrites `combinedText` and earlier debounced messages are lost.
  - confidence: 0.9
- **[P2] Update thread-file tests for saveRemoteMedia:** `extensions/mattermost/src/mattermost/monitor-resources.test.ts:152-153`
  - body: Current main removed the `fetchRemoteMedia`/`saveMediaBuffer` resource contract in favor of `saveRemoteMedia`. The new thread-file tests still pass the removed fields and omit `saveRemoteMedia`, so they will fail after rebasing or merging with current main.
  - confidence: 0.93
- **[P2] Add the thread-file helper to monitor test mocks:** `extensions/mattermost/src/mattermost/monitor.ts:1515-1519`
  - body: `monitor.inbound-system-event.test.ts` mocks `createMattermostMonitorResources` without `resolveMattermostThreadFileIds`. The new call runs for normal inbound posts before dispatch, so that suite will call `undefined` unless the mock returns an empty list or the path is covered there.
  - confidence: 0.91
