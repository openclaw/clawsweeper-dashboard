## Summary

Keep open. Current main still has the Mattermost inbound thread-file gap, but this PR is not correct as submitted because post hydration can overwrite debounced text, the new resource tests use the pre-`saveRemoteMedia` contract, and the existing monitor test mock is not updated for the new helper.

## What This Changes

The PR changes Mattermost inbound handling to hydrate incomplete WebSocket posts, collect earlier thread file IDs, dedupe media IDs, and add monitor resource tests.

## Best Possible Solution

Keep the thread-file fix direction, but preserve debounced text during hydration, rebase the resource tests and monitor mocks to current main, and require redacted live Mattermost proof before merge.

## Reproduction Assessment

Yes for source reproduction: current main computes thread context but still calls `resolveMattermostMedia(post.file_ids)`, so files attached to the thread root or earlier replies are not included. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

No, not as submitted. The owner boundary is reasonable, but the hydration step must avoid overwriting synthesized debounced text and the tests/mocks must be brought up to the current helper API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve debounced text during hydration:** `extensions/mattermost/src/mattermost/monitor.ts:1248`
  - body: When multiple Mattermost posts are debounced, `onFlush` synthesizes one post with `message: combinedText` and `file_ids: []`. This hydration path spreads the fetched REST post over that synthetic post, so the last post's REST `message` overwrites `combinedText` and earlier debounced messages are lost.
  - confidence: 0.9
- **[P2] Update thread-file tests for saveRemoteMedia:** `extensions/mattermost/src/mattermost/monitor-resources.test.ts:152-153`
  - body: Current main removed the `fetchRemoteMedia`/`saveMediaBuffer` resource contract in favor of `saveRemoteMedia`. The new thread-file tests still pass the removed fields and omit `saveRemoteMedia`, so they will fail after rebasing or merging with current main.
  - confidence: 0.93
- **[P2] Add the thread-file helper to monitor test mocks:** `extensions/mattermost/src/mattermost/monitor.ts:1512-1516`
  - body: `monitor.inbound-system-event.test.ts` mocks `createMattermostMonitorResources` without `resolveMattermostThreadFileIds`. The new call runs for normal inbound posts before dispatch, so that suite will call `undefined` unless the mock returns an empty list or the path is covered there.
  - confidence: 0.91
