## Summary

Keep open: the branch addresses a real current-main Mattermost inbound media gap, but it is not merge-ready because hydration can drop debounced text, tests are stale against the current resource contract, a monitor test mock needs the new helper, and real behavior proof is not inspectable.

## What This Changes

The PR hydrates incomplete inbound Mattermost posts, gathers earlier thread file IDs, dedupes them with current file IDs, and routes the combined IDs through inbound media resolution.

## Best Possible Solution

Land a rebased Mattermost-plugin fix that preserves debounced text, updates resource tests and monitor mocks to current contracts, and includes redacted live Mattermost proof that root or earlier thread attachments reach inbound media.

## Reproduction Assessment

Yes for source reproduction: current main accepts optional or empty `file_ids` and routes inbound media from only the triggering post, so a thread reply whose attachment IDs live on the root or earlier replies produces no media. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

No, not as submitted. The thread-file lookup is the right plugin-local boundary, but the branch must preserve synthetic debounced text and update its tests/mocks for the current resource API before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve debounced text during hydration:** `extensions/mattermost/src/mattermost/monitor.ts:1248`
  - body: When multiple Mattermost posts are debounced, `onFlush` synthesizes one post with `message: combinedText` and `file_ids: []`. This hydration path spreads the fetched REST post over that synthetic post, so the last post's REST `message` overwrites `combinedText` and earlier debounced messages are lost before routing.
  - confidence: 0.9
- **[P2] Update resource tests to use saveRemoteMedia:** `extensions/mattermost/src/mattermost/monitor-resources.test.ts:152-153`
  - body: Current main removed the `fetchRemoteMedia`/`saveMediaBuffer` resource contract in favor of `saveRemoteMedia`. The new thread-file tests still pass the removed fields and omit `saveRemoteMedia`, so type/check gates will fail after rebasing or merging with current main.
  - confidence: 0.93
- **[P2] Mock resolveMattermostThreadFileIds in monitor tests:** `extensions/mattermost/src/mattermost/monitor.ts:1496-1501`
  - body: The inbound monitor tests mock `createMattermostMonitorResources` without `resolveMattermostThreadFileIds`. The new call runs for normal inbound posts before dispatch, so those suites need the mock helper or they will call `undefined` when the path is exercised.
  - confidence: 0.88
