## Summary

Keep open: the PR targets a real ACPX Codex duplicate-override bug, but the proposed skip-on-existing-key behavior can discard explicit runtime model/thinking overrides and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds Codex ACP command-key detection so ACPX skips appending duplicate `-c model` and `-c model_reasoning_effort` arguments, with focused runtime tests.

## Best Possible Solution

Keep the fix extension-local: when an explicit runtime override is supplied, replace or remove same-key configured Codex `-c` entries before appending the runtime value; preserve configured defaults when no runtime override exists, with focused tests and a changelog line.

## Reproduction Assessment

Yes. The linked bug gives a concrete ACPX Codex configuration path, current main shows the unconditional append path, and the PR's own partial-overlap test statically reproduces the remaining precedence regression.

## Solution Assessment

No. Skipping existing keys avoids identical duplicates, but it is not the best fix because it lets configured command defaults override explicit `sessions_spawn` runtime requests; rewriting same-key flags or failing clearly on conflicts is safer.
