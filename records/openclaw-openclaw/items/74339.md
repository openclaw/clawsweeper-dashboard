## Summary

Keep open. The PR targets a real ACPX duplicate-append bug on current main, but the patch lets configured Codex `-c` defaults override explicit runtime `model`/`thinking` requests, lacks after-fix real behavior proof, and is currently conflict-marked.

## What This Changes

The PR adds ACPX Codex command parsing that skips appending `-c model` and `-c model_reasoning_effort` overrides when the configured command already contains the same config key, with focused runtime tests.

## Best Possible Solution

Keep the fix extension-local by normalizing Codex ACP `-c` args so same-key duplicates are removed, configured defaults are preserved when no runtime override is supplied, and explicit runtime overrides win or fail clearly with regression coverage and real ACPX proof.

## Reproduction Assessment

Yes, by source inspection. Current main forwards `sessions_spawn.model` and `thinking` into ACPX Codex startup config and appends duplicate `-c` keys when the configured command already contains them; the PR diff also shows the remaining precedence regression.

## Solution Assessment

No. Skipping already-present keys prevents identical duplicates, but it is not the best fix because it can silently drop an explicit runtime override in favor of a configured command default.
