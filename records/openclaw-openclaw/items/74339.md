## Summary

Keep open: the PR targets a real ACPX Codex duplicate `-c` bug and current main still appends runtime `model`/`model_reasoning_effort` blindly, but the branch should not merge as-is because its skip-if-present rule lets configured command defaults override explicit `sessions_spawn` runtime values and it lacks the required changelog entry.

## What This Changes

The PR adds an ACPX `commandHasConfigKey` helper, changes Codex ACP startup override appending to skip already-present `-c` config keys, and adds runtime tests for duplicate and partial-overlap command cases.

## Best Possible Solution

Keep the repair extension-local, but normalize the Codex ACP command so same-key `-c` entries are removed or replaced when runtime `model`/`thinking` overrides are present, preserving explicit runtime precedence while preventing duplicate keys. Cover identical duplicates, conflicting values, partial overlap, quoted args, wrapper commands, and the no-runtime-override case, then add an Unreleased Fixes changelog line.

## Reproduction Assessment

Yes. The linked bug report gives concrete ACPX Codex steps, and the PR's own partial-overlap test statically demonstrates the remaining regression by dropping a runtime `model: "gpt-5.5"` when the configured command already has `-c model=gpt-5.4`.

## Solution Assessment

No. Skipping already-present keys fixes identical duplicate arguments, but it is not the best fix because it silently gives configured command values precedence over explicit runtime overrides; replacing/removing same-key entries or failing clearly on conflict is safer.
