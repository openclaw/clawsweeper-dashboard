## Summary

Keep open. The underlying ACPX bug is valid, but this PR should not merge as-is because it drops explicit runtime override precedence, lacks after-fix real behavior proof, and is currently conflicting with main.

## What This Changes

The PR adds ACPX Codex command parsing and tests to skip appending duplicate `-c` config keys when a configured command already contains those keys.

## Best Possible Solution

Keep the fix extension-local by normalizing Codex ACP `-c` args so same-key duplicates are removed, configured defaults remain when no runtime override is supplied, and explicit runtime overrides win or fail clearly with regression coverage and real ACPX proof.

## Reproduction Assessment

Yes, by source inspection. Current main forwards `sessions_spawn.model` and `thinking` into ACPX Codex startup config while configured Codex commands can already carry the same `-c` keys, so duplicate same-key args can be launched.

## Solution Assessment

No. Skipping already-present keys handles identical duplicates, but it is not the best fix because it can silently discard a different explicit runtime override in favor of a configured command default.
