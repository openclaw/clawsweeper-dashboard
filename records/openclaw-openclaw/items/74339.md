## Summary

Keep open: the linked report and prior review comments point to a real ACPX duplicate-override bug, and current main still has the blind append path. The branch is useful but should not merge until it preserves explicit runtime override precedence and adds the required changelog entry.

## What This Changes

The PR adds ACPX command-key detection and tests so Codex ACP runtime model/reasoning `-c` flags are not appended when the configured command already contains those keys.

## Best Possible Solution

Keep the fix extension-local: explicit runtime model/thinking overrides should replace same-key configured Codex `-c` flags, configured defaults should remain when no runtime override exists, and tests/changelog should document that behavior.

## Reproduction Assessment

Yes. The linked issue gives concrete ACPX Codex configuration steps, current main shows the duplicate append path, and the PR's partial-overlap test statically reproduces the remaining precedence regression.

## Solution Assessment

No. Skipping already-present keys prevents identical duplicates, but it is not the best fix because it lets configured command defaults override explicit `sessions_spawn` runtime requests; replacing/removing same-key flags or failing clearly on conflict is safer.
