## Summary

Keep open: the PR targets a real ACPX Codex duplicate-override bug linked to #74305, but the patch would silently let configured command defaults override explicit `sessions_spawn` model/thinking requests and it is missing the required changelog entry.

## What This Changes

The PR adds ACPX Codex command parsing to skip appending runtime `-c model` and `-c model_reasoning_effort` flags when the configured command already contains those keys, plus runtime tests.

## Best Possible Solution

Keep the fix extension-local: when an explicit runtime override is present, replace or remove same-key configured Codex `-c` entries before appending the runtime value; preserve configured defaults only when no runtime override is supplied, with focused tests and a changelog line.

## Reproduction Assessment

Yes. Current main shows the duplicate append path, the linked report gives a concrete ACPX Codex configuration, and the PR's partial-overlap test statically reproduces the remaining runtime-precedence regression.

## Solution Assessment

No. Skipping already-present keys prevents identical duplicates, but it is not the best fix because it lets configured command defaults override explicit `sessions_spawn` runtime requests; rewriting same-key flags or failing clearly on conflicts is safer.
