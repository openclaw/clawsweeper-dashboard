## Summary

Keep open: the PR targets a real, source-reproducible ACPX/Codex duplicate-arg bug, but the current diff should not merge because it silently drops documented explicit runtime overrides when configured same-key `-c` args differ and it still lacks contributor-supplied real ACPX/Codex proof.

## What This Changes

The PR changes ACPX Codex command construction to skip duplicate Codex ACP `-c` config keys and adds focused runtime tests.

## Best Possible Solution

Keep the fix ACPX-local, but parse and replace conflicting Codex ACP `-c` model/thinking args when an explicit runtime override is present, with regression tests for identical, missing, and conflicting values plus a redacted real ACPX/Codex smoke.

## Reproduction Assessment

Yes, source-reproducible: current main appends Codex ACP model/thinking startup `-c` args to a command that may already contain same-key `-c` values, and the linked reporter confirmed the workflow works after removing configured overrides. I did not run a live ACP smoke in this read-only review.

## Solution Assessment

No. Skipping already-present keys avoids identical duplicates, but it is not the best fix because it silently discards a different explicit runtime override instead of replacing the configured value or failing clearly on conflict.
