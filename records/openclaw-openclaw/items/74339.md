## Summary

Keep open: the PR targets a real ACPX Codex duplicate `-c` override bug, but the proposed skip-if-present logic is not mergeable because it silently lets configured command defaults override explicit `sessions_spawn` runtime model/thinking values, and it is missing the required user-facing changelog entry.

## What This Changes

The PR adds `commandHasConfigKey()` to ACPX runtime command normalization and tests duplicate, partial-overlap, wrapper, helper, and no-runtime-override Codex ACP command cases.

## Best Possible Solution

Keep the repair extension-local: normalize the configured Codex ACP command by replacing or removing same-key `-c` entries when an explicit runtime model/thinking override is present, preserve configured defaults when no runtime override is supplied, cover quoted/wrapper/conflict cases in tests, and add the active-version changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete operator steps, and the PR's own partial-overlap test statically reproduces the remaining regression by showing a runtime `model: "gpt-5.5"` request being dropped when the configured command already has `-c model=gpt-5.4`.

## Solution Assessment

No. Skipping already-present config keys avoids identical duplicate args, but it is not the best fix because it changes the documented precedence for explicit runtime overrides; replacing same-key command args or failing clearly on conflicting values is safer.
