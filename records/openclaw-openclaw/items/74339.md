## Summary

Keep open. Current main still has the duplicate append path, but this PR is not merge-ready because it can silently drop a conflicting explicit runtime override and the external contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR changes ACPX Codex command handling to skip appending `-c` config overrides when the command string already contains the same config key, with focused runtime tests.

## Best Possible Solution

Keep the fix ACPX-local by normalizing Codex ACP `-c` args so configured defaults are preserved only when no runtime override is supplied, while explicit runtime overrides replace same-key configured args or fail clearly with regression coverage and real ACPX proof.

## Reproduction Assessment

Yes, by source inspection rather than a live ACP run. Current main appends runtime Codex model/thinking startup config to a command that can already contain the same configured `-c` keys, matching the linked issue's reported workaround.

## Solution Assessment

No. Skipping already-present keys handles identical duplicates, but it is not the best fix because it can silently discard a different explicit runtime override in favor of a configured command default.
