## Summary

Keep open: the PR targets a real current-main ACPX duplicate `-c` append bug, but the patch is not mergeable because it can discard explicit `sessions_spawn` runtime overrides in favor of configured command defaults, and it is missing the required changelog entry.

## What This Changes

The PR adds ACPX Codex command-key detection to skip duplicate `-c model` and `-c model_reasoning_effort` appends, plus focused runtime tests.

## Best Possible Solution

Keep the fix extension-local: replace or remove same-key configured Codex `-c` entries when an explicit runtime override is present, preserve configured defaults when no runtime override is supplied, and add regression tests plus a changelog line.

## Reproduction Assessment

Yes, by source inspection. Current main unconditionally appends duplicate Codex `-c` config args, and the PR's partial-overlap test demonstrates the remaining precedence regression without needing a live ACP run.

## Solution Assessment

No. Skipping already-present keys prevents identical duplicates, but it is not the best fix because it can let configured command defaults override explicit runtime `model`/`thinking` requests.
