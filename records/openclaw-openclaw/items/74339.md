## Summary

Keep open: current main still has the duplicate Codex ACP `-c` append path, but this PR is not merge-ready because it drops conflicting explicit runtime overrides and lacks the required changelog entry.

## What This Changes

The PR adds ACPX Codex command parsing to skip appending duplicate `-c model` and `-c model_reasoning_effort` flags, with focused runtime tests for configured and wrapper command variants.

## Best Possible Solution

Keep the fix ACPX-local: normalize same-key Codex startup config so explicit runtime model/thinking wins while configured defaults remain intact, then cover it with focused tests and a changelog line.

## Reproduction Assessment

Yes. #74305 gives a concrete ACPX Codex configuration path, current main shows the duplicate append path, and the PR's own partial-overlap expectation statically reproduces the remaining precedence regression.

## Solution Assessment

No. Skipping existing keys avoids identical duplicates, but it is not the best fix because a configured command default can silently override an explicit `sessions_spawn` runtime request.
