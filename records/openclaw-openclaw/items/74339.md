## Summary

Keep open: the PR targets a real ACPX Codex duplicate `-c` override problem linked to #74305, but the proposed skip-when-present logic would silently drop explicit runtime model/thinking overrides when the configured command has different values. It also lacks the required user-facing changelog entry.

## What This Changes

The PR adds a `commandHasConfigKey()` helper to ACPX Codex command normalization and tests that skip appending duplicate `-c model` and `-c model_reasoning_effort` flags.

## Best Possible Solution

Keep the fix extension-local: when an explicit runtime override is present, replace or remove same-key configured Codex `-c` entries before appending the runtime value; when no runtime override is supplied, preserve configured defaults, with focused tests and a changelog line.

## Reproduction Assessment

Yes. The linked report gives a concrete ACPX Codex configuration path, current main shows the duplicate append path, and the PR's partial-overlap test statically reproduces the remaining precedence regression.

## Solution Assessment

No. Skipping already-present keys prevents identical duplicates, but it is not the best fix because it lets configured command defaults override explicit `sessions_spawn` runtime requests; rewriting same-key flags or failing clearly on conflicts is safer.
