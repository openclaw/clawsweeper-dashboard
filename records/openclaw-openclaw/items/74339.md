Keep this PR open: it targets a real ACPX Codex duplicate-argument bug, but the current diff is not merge-ready because it drops explicit runtime `sessions_spawn` model/thinking overrides whenever the configured Codex command already has the same `-c` key, and it lacks the required changelog entry.

## What This Changes

The PR adds ACPX Codex command-key detection to skip duplicate `-c model` and `-c model_reasoning_effort` appends, with focused runtime tests.

## Best Possible Solution

Keep the fix extension-local: remove or replace same-key configured Codex `-c` entries only when an explicit runtime override is supplied, preserve configured defaults otherwise, and cover both conflict and identical-value cases in tests plus changelog.
