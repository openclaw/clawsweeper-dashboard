## Summary

Keep open: the duplicate-arg bug is valid, but this patch can drop explicit runtime model/thinking requests when configured Codex command defaults already contain the same `-c` keys, and it still needs a user-facing changelog entry.

## What This Changes

The PR adds ACPX Codex command-key detection to skip appending duplicate `-c model` and `-c model_reasoning_effort` args and adds runtime tests for configured Codex ACP commands.

## Best Possible Solution

Keep the fix extension-local: normalize configured Codex `-c` entries so explicit runtime overrides win when present, configured defaults remain when no runtime override is supplied, duplicate keys are not passed to Codex ACP, and the user-facing fix is documented in the active changelog.

## Reproduction Assessment

Yes, by source inspection: current main appends Codex ACP `-c` config args whenever a runtime override exists, and the PR diff includes a partial-overlap case that preserves a conflicting configured model. I did not run a live ACP task in this read-only review.

## Solution Assessment

No. Skipping existing keys prevents duplicate identical args, but it is not the best fix because it can let configured command defaults override explicit runtime `model`/`thinking` requests; replacing/removing same-key configured args or failing clearly on conflict is safer.
