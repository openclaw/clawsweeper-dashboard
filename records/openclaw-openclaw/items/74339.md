## Summary

Keep open. The PR targets a real ACPX Codex bug still present on current main, but the proposed skip-existing-key logic can discard a different explicit runtime override, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes ACPX Codex command handling to skip appending duplicate Codex ACP `-c` config keys and adds focused runtime tests.

## Best Possible Solution

Keep the fix ACPX-local by normalizing Codex ACP `-c` args so configured defaults are preserved only when no runtime override is supplied, while explicit runtime overrides replace same-key configured args or fail clearly with regression coverage and real ACPX proof.

## Reproduction Assessment

Yes, by source inspection rather than a live ACP run. Current main appends Codex model/thinking startup config to a command that can already contain configured same-key `-c` args, and the linked reporter confirmed the workflow works when those configured args are removed.

## Solution Assessment

No. Skipping already-present keys handles identical duplicates, but it is not the best fix because it silently discards a different explicit runtime override despite the documented `sessions_spawn.model` override contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
