## What This Changes

The PR changes the Signal daemon stop contract to await child-process exit with SIGKILL escalation, awaits monitor teardown, and adds daemon/monitor regression tests.

## Best Possible Solution

Land this PR or an equivalent Signal-owned lifecycle fix, then close the linked Signal race issue with the fixing commit and release provenance.

## Reproduction Assessment

Yes, source-level: enable Signal auto-start and trigger a channels.signal hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns before the child exits.

## Solution Assessment

Yes. Making Signal daemon stop async/idempotent, waiting on the existing exit promise, adding bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
