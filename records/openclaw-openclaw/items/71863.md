## Summary

Keep open: this PR is a focused, current fix for the still-present Signal daemon restart race; source inspection, PR proof, independent verification, and exact-head proof checks support merge after maintainer review, with no blocking code findings.

## What This Changes

The PR makes Signal daemon shutdown asynchronous, waits for child exit, escalates to SIGKILL after a timeout, awaits monitor teardown, and adds daemon/monitor regression coverage.

## Best Possible Solution

Land this Signal-owned lifecycle fix after maintainer review, then let the merge close https://github.com/openclaw/openclaw/issues/22676.

## Reproduction Assessment

Yes, source-reproducible. On current main, Signal auto-start plus a `channels.signal` hot reload can stop then start the channel while daemon stop only sends SIGTERM and returns; the PR thread also includes independent unpatched-main test-failure evidence.

## Solution Assessment

Yes. Making the Signal daemon handle async and idempotent, awaiting the existing exit promise, adding bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this Signal-owned race.
