## Summary

Keep open: current main still has the Signal daemon shutdown race, while this PR is a focused implementation with sufficient after-fix runtime proof, green checks, and no blocking review findings.

## What This Changes

This PR makes the Signal daemon stop contract asynchronous, awaits daemon shutdown during monitor teardown, adds bounded SIGKILL escalation, and adds daemon/monitor regression tests.

## Best Possible Solution

Land this focused Signal lifecycle fix, or an equivalent Signal-owned fix, then close the linked daemon race issue after merge.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a channels.signal hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns before child exit.

## Solution Assessment

Yes. Making the Signal daemon handle async/idempotent, waiting for process exit, bounding shutdown with SIGKILL, and awaiting monitor teardown is the narrow maintainable fix for this race.
