## Summary

Keep open: current main and v2026.5.18 still have the synchronous Signal daemon stop path, while this PR is the active focused fix for the linked Signal restart race and has sufficient lifecycle proof with no blocking review findings.

## What This Changes

The PR changes Signal daemon shutdown to return an awaited promise with idempotent stop handling, SIGKILL escalation, awaited monitor teardown, and focused daemon/monitor regression tests.

## Best Possible Solution

Land this PR or an equivalent Signal-owned lifecycle fix, then close the linked Signal race issue with the fixing commit and release provenance.

## Reproduction Assessment

Yes, source-level: with Signal autoStart enabled, a channels.signal hot reload aborts the channel and can immediately restart it while current main only sends SIGTERM and returns before signal-cli exits.

## Solution Assessment

Yes. Making Signal daemon stop async/idempotent, waiting on the existing exit promise with bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
