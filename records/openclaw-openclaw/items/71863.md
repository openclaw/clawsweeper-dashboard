## Summary

Keep open: current main still has the Signal daemon restart race, and this PR is a focused, well-supported fix with sufficient real behavior proof, green relevant checks, and no blocking code or security findings.

## What This Changes

The PR changes the Signal plugin daemon stop contract to return a Promise, waits for signal-cli exit with SIGKILL escalation, awaits monitor teardown on abort/restart, and adds daemon/monitor regression tests.

## Best Possible Solution

Land this Signal-owned lifecycle fix with its regression tests, then let the merge close https://github.com/openclaw/openclaw/issues/22676.

## Reproduction Assessment

Yes, source-level. Current main’s Signal stop returns void and monitor cleanup does not await it while hot reload stops then starts the channel; the PR thread also includes independent unpatched-main test-failure evidence.

## Solution Assessment

Yes. Making the Signal daemon handle await exit, share the in-flight stop, escalate to SIGKILL after a bounded timeout, and await monitor teardown is the narrow owner-boundary fix; broader iMessage/Gmail cleanup should stay separate.
