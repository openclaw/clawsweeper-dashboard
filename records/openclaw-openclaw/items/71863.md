- `P1`: The PR addresses a confirmed Signal restart race that can leave orphaned daemon processes and break active channel sends for real users.

## Summary

Keep open: current main and the latest release still have the fire-and-forget Signal daemon stop path, while this PR is the focused open fix for the linked Signal restart race and has no blocking review findings.

## What This Changes

The PR changes Signal daemon shutdown to return an awaited promise, adds SIGKILL escalation, awaits monitor teardown, and adds daemon/monitor regression coverage.

## Best Possible Solution

Land this PR or an equivalent Signal-owned lifecycle fix, then close the linked race issue with the fixing commit and release provenance.

## Reproduction Assessment

Yes, source-level: with Signal autoStart enabled, channel hot reload aborts the monitor and immediately restarts it while current main only sends SIGTERM and returns before signal-cli exits. The PR’s added tests also demonstrate the old stop contract and missing SIGKILL escalation.

## Solution Assessment

Yes. Making Signal daemon stop async/idempotent, waiting on the existing exit promise with bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
