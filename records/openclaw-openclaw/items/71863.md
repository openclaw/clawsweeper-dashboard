## What This Changes

The PR changes the Signal plugin daemon handle to stop asynchronously with SIGTERM/SIGKILL escalation, awaits monitor teardown, and adds daemon/monitor regression tests.

## Best Possible Solution

Land this focused Signal-owned lifecycle fix, or an equivalent fix, and then close https://github.com/openclaw/openclaw/issues/22676 after merge.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a channels.signal hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns before the child exits.

## Solution Assessment

Yes. Making Signal daemon stop async/idempotent, waiting on the existing exit promise, bounding SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
