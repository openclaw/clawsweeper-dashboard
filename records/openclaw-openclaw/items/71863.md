## Label Justifications

- `P1`: The PR addresses a confirmed Signal restart race that can leave orphaned daemon processes and break active channel sends for real users.
- `merge-risk: 🚨 availability`: The patch changes Signal child-process shutdown timing and adds forced SIGKILL escalation, which affects restart availability paths not fully covered by CI.

## Summary

Keep open: this PR is an active, focused fix for a confirmed Signal daemon restart race that is still present on current main. The patch has sufficient lifecycle proof and no blocking correctness or security findings, with the remaining decision being maintainer acceptance of the scoped proof gap and the 1500ms SIGKILL fallback.

## What This Changes

The PR makes Signal daemon shutdown asynchronous and idempotent, awaits monitor teardown, adds bounded SIGKILL escalation, and adds daemon/monitor regression tests.

## Best Possible Solution

Land this PR or an equivalent Signal-owned lifecycle fix, then close https://github.com/openclaw/openclaw/issues/22676 with the fixing commit and release provenance.

## Reproduction Assessment

Yes, at source level. Current main hot reload can stop and restart Signal while monitor cleanup only sends SIGTERM and returns before signal-cli exits, and the linked issue plus independent verification describe the orphaned-process failure mode.

## Solution Assessment

Yes. Making Signal daemon stop async and idempotent, waiting on the existing exit promise with bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
