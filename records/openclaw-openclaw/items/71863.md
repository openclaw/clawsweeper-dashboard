## Summary

Keep open: current main still has the Signal daemon shutdown race, and this PR targets the right Signal-owned lifecycle path, but it still needs after-fix real behavior proof and a changelog entry before merge.

## What This Changes

The PR makes the Signal daemon stop operation await process exit with SIGTERM/SIGKILL shutdown, awaits monitor cleanup, and adds daemon/monitor regression tests.

## Best Possible Solution

Land the Signal-owned lifecycle fix after the contributor adds real restart proof and a single-line changelog entry; keep https://github.com/openclaw/openclaw/issues/22676 open until the fix merges.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `channels.signal.autoStart=true` plus a Signal config reload: current main's reload path waits for channel stop, but Signal monitor cleanup resolves before the old daemon process exits; independent verification also showed the PR's new tests fail on unpatched main.

## Solution Assessment

Yes for the code direction. Making the Signal daemon handle awaitable and awaiting it during monitor shutdown is the narrow maintainable fix, but merge should wait for real behavior proof and changelog coverage.
