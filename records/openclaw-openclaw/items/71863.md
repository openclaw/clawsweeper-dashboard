## What This Changes

The PR makes Signal daemon shutdown return an awaited promise with bounded SIGKILL escalation and adds daemon/monitor regression coverage for abort and restart cleanup.

## Best Possible Solution

Land this PR or an equivalent Signal-owned lifecycle fix, then close https://github.com/openclaw/openclaw/issues/22676 with the fixing commit and release provenance.

## Reproduction Assessment

Yes, source-level. Current main aborts and restarts Signal hot reload while Signal cleanup only sends SIGTERM and returns before signal-cli exit, and the linked issue plus independent verification describe the orphaned-process failure mode.

## Solution Assessment

Yes. Making Signal daemon stop async and idempotent, waiting on the existing exit promise, adding bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix for this race.
