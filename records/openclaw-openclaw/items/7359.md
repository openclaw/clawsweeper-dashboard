## Summary

Keep open. Current main still has the Slack cross-session blind spot: DM/main sessions and channel sessions remain separate, outbound channel sends are mirrored only into the destination channel session, and default session-tool visibility does not expose that sibling session back to a DM turn.

## What This Changes

The issue requests Slack self-activity recall so an agent answering in DMs can remember its own outbound messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall mechanism keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send. The mirror write lands in the channel session, while default session tools cannot see that sibling session.

## Solution Assessment

No current implementation solves it, and broad transcript merging would not be the best fix. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not widening default session visibility globally.
