## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions in separate buckets, mirrors outbound channel sends into the destination channel session only, and default session-history visibility does not bridge that sibling channel session back into a DM/main turn. The closed Slack DM-history and native IM mirror-routing fixes are adjacent, but they do not implement recall of the agent’s own outbound channel activity from DMs.

## What This Changes

The issue requests Slack DM/main sessions to be able to recall the agent’s own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full DM and channel transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, from source inspection. A Slack channel send from a DM/main session is mirrored to the destination channel session, while default session-history visibility from the DM/main session only allows the current or spawned session tree.

## Solution Assessment

Unclear as a product decision. The narrow maintainable direction is scoped self-activity recall, not a broad cross-session transcript merge or a generic tools.sessions.visibility=all workaround.
