## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions as separate visibility buckets, and message-tool sends mirror only into the destination session; the remaining change needs a bounded Slack self-activity design rather than cleanup closure.

## What This Changes

The issue requests that a Slack DM/main-session agent be able to recall its own outbound Slack channel messages and related channel conversation context.

## Best Possible Solution

Add a bounded Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id, without merging full DM and channel transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes, source-level. Current main shows Slack channel sends mirror to the destination channel session, while the requester DM/main session is separate and default tree visibility excludes that sibling; no live Slack workspace run was performed.

## Solution Assessment

No for the broad options in the issue. Unified Slack sessions or general same-agent visibility would over-share; a purpose-built, authorization-aware self-activity recall path is the safer maintainable direction.
