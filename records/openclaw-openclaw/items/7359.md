## Summary

Keep open. Current main still keeps Slack DM/main sessions separate from channel sessions, mirrors outbound sends only into the destination session, and default session-tool visibility does not expose that sibling channel transcript back to a DM.

## What This Changes

The issue requests Slack agents to recall their own outbound channel messages from DM sessions.

## Best Possible Solution

Add a scoped Slack self-activity recall mechanism keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send. The mirror write lands in the channel session, while default session tools cannot see that sibling session.

## Solution Assessment

No current implementation solves it. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not merging full DM/channel transcripts or widening default session visibility globally.
