## Summary

Keep open. Current main still separates Slack DM/main and channel sessions, mirrors outbound Slack sends only into the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM.

## What This Changes

The issue requests Slack DM sessions to recall the agent’s own outbound Slack channel messages and related channel activity.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent’s own outbound channel activity to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send; the outbound mirror lands in the channel session and default session tools cannot see that sibling channel session. I did not run a live Slack workspace repro.

## Solution Assessment

No current implementation solves it. The best path is a scoped self-activity ledger or Slack-specific history bridge, not merging full DM and channel transcripts or widening default session visibility globally.
