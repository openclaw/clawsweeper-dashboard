## Summary

Keep open. Current main still preserves the Slack DM/channel session split and mirrors outbound channel sends only into the destination session, so the reported cross-session self-activity blind spot remains and needs a privacy/product decision before implementation.

## What This Changes

The issue asks for Slack DM sessions to recall the agent's own outbound Slack channel messages so later DM turns can reference channel activity.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id or timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored only into the destination channel session, and default sessions_history tree visibility blocks that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be privacy-sensitive. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
