## Summary

Keep open. Current main still has the Slack DM-to-channel self-activity blind spot: Slack DM/main and channel sessions are separate, outbound sends are mirrored only into the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM turn.

## What This Changes

Request Slack agents to recall their own outbound channel messages and channel conversations when later asked from a DM session.

## Best Possible Solution

Add a Slack-scoped self-activity recall path keyed by agent, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel messages without widening default sibling transcript visibility.

## Reproduction Assessment

Yes by source inspection: Slack DM/main and channel sends resolve to different session keys, outbound mirrors write only to the destination route, and default session-tool visibility is tree. I did not run a live Slack workspace reproduction.

## Solution Assessment

Unclear until maintainers choose the privacy boundary. A narrow self-activity ledger or scoped history bridge is safer than unified Slack sessions or globally changing session visibility to agent/all.
