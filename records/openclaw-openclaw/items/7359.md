## Summary

Keep open. Current main still does not give a Slack DM/main session automatic access to the agent's own outbound Slack channel messages: outbound sends are mirrored to the destination channel session, while default session-tool visibility keeps same-agent sibling channel sessions hidden. Recent Slack DM mirror canonicalization and cron destination mirroring are adjacent fixes, not a self-activity recall mechanism.

## What This Changes

The issue requests Slack self-activity recall so an agent asked in a DM can reference messages it previously sent to Slack channels.

## Best Possible Solution

Add a Slack-scoped self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel messages without merging full DM and channel transcripts or broadening default sibling-session visibility.

## Reproduction Assessment

Yes for the current behavior by source inspection: Slack DM/main and channel sends resolve to different session keys, outbound mirrors write to the destination route, and default session-tool visibility is tree. I did not run a live Slack workspace reproduction.

## Solution Assessment

Unclear until maintainers choose the privacy boundary. A narrow self-activity ledger or scoped history bridge is safer than unified Slack sessions or setting session visibility to agent/all globally.
