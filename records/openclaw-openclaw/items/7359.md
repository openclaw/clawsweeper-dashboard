## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound channel sends only into the destination session, and keeps sibling session history hidden by default; the remaining fix needs product and privacy review rather than cleanup closure.

## What This Changes

The issue requests Slack DM sessions to recall the agent's own outbound messages sent to Slack channels so follow-up DM turns can reference that activity.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack workspace/account, destination channel/thread, and Slack message id or timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored into the channel destination session, and default sessions_history tree visibility rejects that sibling session from the DM/main requester.

## Solution Assessment

No for the broad unified-session proposals in the issue. The maintainable direction is a narrow Slack self-activity recall mechanism that exposes only the agent's own outbound channel activity.
