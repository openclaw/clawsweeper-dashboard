## Summary

Keep open. Current main still leaves the Slack DM/channel self-activity gap in place: outbound channel sends are mirrored to the destination Slack session, while default session-history visibility does not expose that sibling channel session back to a DM/main requester.

## What This Changes

The issue requests a way for Slack DM sessions to let the agent recall its own outbound Slack channel messages and maintain coherent cross-context activity awareness.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack workspace/account, destination channel/thread, and Slack message id or timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored into the channel destination session, and default sessions_history tree visibility rejects that sibling session from the DM/main requester.

## Solution Assessment

No for the broad proposals in the issue. Unified Slack sessions or generic cross-session history would be privacy-sensitive; the maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
