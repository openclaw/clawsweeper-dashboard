## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors message-tool sends only into the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM, so the requested self-activity recall remains unimplemented and security-sensitive.

## What This Changes

Request a Slack-aware way for DM sessions to recall the agent's own outbound channel messages and follow up coherently across Slack contexts.

## Best Possible Solution

Add a bounded Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound messages to authorized DM requesters without merging full transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes, at source level. Current main routes Slack DMs and Slack channels into separate session keys, mirrors outbound sends to the destination session, and default tree session visibility denies the sibling channel session; I did not run a live Slack workspace.

## Solution Assessment

No for the broad unified-session options in the issue. A narrow, authorization-aware self-activity recall path is safer than merging full Slack DM and channel transcripts or globally widening session visibility.
