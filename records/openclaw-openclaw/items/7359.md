## Summary

Keep open. Current main still lacks automatic Slack self-activity recall from DM/main sessions: adjacent fixes now mirror outbound sends into the destination session and hydrate Slack DMs, but default session visibility keeps sibling channel sessions hidden and the remaining change needs a privacy/product decision.

## What This Changes

Request that Slack DM sessions let an agent recall and reference its own outbound messages sent to Slack channels.

## Best Possible Solution

Add a Slack-scoped self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel messages without widening default sibling transcript access.

## Reproduction Assessment

Yes for the current behavior by source inspection: Slack DM/main and channel sends resolve to different session keys, outbound mirrors write to the destination route, and default session-tool visibility is tree. I did not run a live Slack workspace reproduction.

## Solution Assessment

Unclear until maintainers choose the privacy boundary. A narrow self-activity ledger or scoped history bridge is safer than unified Slack sessions or globally setting session visibility to agent/all.
