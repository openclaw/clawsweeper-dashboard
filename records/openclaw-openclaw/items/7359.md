## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound channel sends only into the destination channel session, and default session-tool visibility does not expose that sibling session back to a DM turn; the closed Slack DM-history issue is adjacent, not a replacement for this self-activity bridge.

## What This Changes

Request Slack DM sessions to recall the agent's own outbound Slack channel messages sent from DM context.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp that exposes only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace. Send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send; the route/mirror/visibility code shows the channel send lands in a sibling channel session hidden under default tree visibility.

## Solution Assessment

No current implementation solves it, and broad transcript merging would not be the best fix. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not widening default session visibility globally.
