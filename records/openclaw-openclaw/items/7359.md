## Summary

Keep open. Current main still has the Slack self-activity blind spot: DM/main sessions and channel sessions are separate, outbound message-tool sends are mirrored to the destination session, and default session-tool visibility does not expose sibling channel sessions back to the DM.

## What This Changes

The issue requests cross-session visibility so a Slack DM can recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel activity to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send; the outbound mirror lands in the channel session and default session tools cannot see that sibling session. I did not run a live Slack workspace repro.

## Solution Assessment

No current implementation solves it. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not merging full DM/channel transcripts or widening default session visibility globally.
