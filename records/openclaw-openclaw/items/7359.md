## Summary

Keep open. Current main still has the Slack self-activity blind spot described here: DM/main sessions and Slack channel sessions are separate, outbound message-tool sends are mirrored to the destination channel session only, and default session-tool visibility does not expose that sibling channel session back to a DM turn. Recent issue discussion also kept this open after distinguishing the closed Slack DM-history issue as adjacent, not canonical.

## What This Changes

The issue requests that a Slack agent in a DM/main session be able to recall its own outbound messages and follow-up context from Slack channel sessions.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to DM/main sessions without merging full sibling transcripts.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace. A Slack channel send from a DM/main session resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

No current implementation solves the issue, and broad session unification would not be the best fix. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not widening default session visibility globally.
