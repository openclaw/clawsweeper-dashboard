## Summary

Keep open. Current main still has the Slack self-activity blind spot: DM/main sessions and channel sessions remain distinct, outbound channel sends are mirrored only to the destination channel session, and default session-tool visibility does not expose that sibling channel session back to a DM turn.

## What This Changes

The issue asks OpenClaw to let a Slack DM/main-session agent recall its own outbound Slack channel messages and related channel conversations.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp that exposes only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace. A Slack channel send from a DM/main session resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

No current implementation solves it, and broad session unification would not be the best fix. The maintainable direction is a scoped self-activity ledger or Slack-specific history bridge, not widening default session visibility globally.
