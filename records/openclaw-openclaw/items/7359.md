## Summary

Current main still keeps Slack DM/main and channel activity in separate sessions, mirrors outbound message-tool sends only into the destination session, and does not expose sibling channel sessions through default session-tool visibility; the issue remains valid but needs a privacy and product-boundary decision before implementation.

## What This Changes

Provide a privacy-scoped way for an agent operating in a Slack DM to recall its own outbound messages and conversations in Slack channels.

## Best Possible Solution

Add a narrow self-activity ledger or scoped recall API for the agent's own outbound Slack messages, keyed by agent, Slack account/workspace, destination channel or thread, and Slack message identifier, without merging full transcripts or widening general sibling-session access.

## Reproduction Assessment

Yes at the source level: Slack DM/main and channel destinations resolve to separate sessions, outbound sends mirror only to the destination session, and default tree visibility excludes sibling sessions. A live Slack run was not performed in this read-only review.

## Solution Assessment

No final implementation direction is approved yet; a scoped self-activity record is safer than unified sessions or general cross-session transcript access because it solves recall without exposing unrelated conversation content.
