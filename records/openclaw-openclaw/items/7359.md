## Summary

Keep open. Current main still keeps Slack DM/main sessions separate from Slack channel sessions, mirrors outbound channel sends only into the destination session, and default session-history visibility does not expose sibling channel sessions back to a DM turn; this remains a product/privacy design item rather than a ready autonomous fix.

## What This Changes

The issue requests Slack DM sessions to recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism keyed by agent, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection is high-confidence: a DM/main turn that sends to a Slack channel gets a destination channel outbound route, the send mirror writes only to that destination session, and default `sessions_history` tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as a product decision. The maintainable direction is scoped self-activity recall, not a broad transcript merge or a generic `tools.sessions.visibility=all` workaround.
