## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions in separate buckets, mirrors outbound channel sends into the destination channel session only, and default session-history visibility does not bridge sibling channel sessions back into a DM turn. Adjacent fixes for Slack DM history and native IM mirror routing do not cover recalling the agent's own outbound channel activity from a DM, so this remains a product/privacy design item rather than an auto-fix bug.

## What This Changes

The issue asks for Slack DM sessions to let an agent recall its own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full DM and channel transcripts or widening general session visibility.

## Reproduction Assessment

Yes, from source inspection. A Slack channel send from a DM/main session is mirrored to the destination channel session, while default session-history visibility from the DM/main session only allows the current or spawned session tree.

## Solution Assessment

Unclear as a product decision. The narrow maintainable direction is a scoped self-activity recall surface, not a broad cross-session transcript merge or a generic visibility=all workaround.
