## Summary

Keep open. Current main still has the Slack cross-session blind spot: Slack DM/main sessions and channel sessions remain distinct, outbound channel sends are mirrored only to the destination channel session, and default session-history visibility does not expose that sibling session back to a DM turn. This remains source-reproducible product work that needs a privacy-aware maintainer decision before implementation.

## What This Changes

Add Slack self-activity recall so an agent in a DM/main session can see its own outbound channel messages and follow up coherently across Slack contexts.

## Best Possible Solution

Add a narrow Slack self-activity ledger or scoped history bridge keyed by agent, Slack team/account, destination channel/thread, and Slack message id/timestamp without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a DM/main turn that sends to a Slack channel resolves a destination channel route, mirrors only that destination session, and default sessions_history tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as phrased because a unified Slack session or broad cross-session history bridge would be risky. The maintainable solution is a scoped self-activity recall mechanism for the agent's own outbound Slack messages.
