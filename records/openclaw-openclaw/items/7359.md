## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound channel sends into the destination session, and keeps sibling channel sessions hidden under default session-tool visibility; the closed Slack DM-history work is adjacent but does not solve this self-activity recall path.

## What This Changes

Request Slack DM/main sessions be able to recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp that exposes only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace. A Slack channel send from a DM/main session resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

No current implementation solves it, and broad session unification would not be the best fix. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not widening default session visibility globally.
