## Summary

Keep open. Current main still has the Slack self-activity blind spot: Slack DM/main and channel sessions remain separate, outbound channel sends mirror to the destination channel session, and default session-history visibility blocks a DM/main requester from reading that sibling channel session. The adjacent fixes for https://github.com/openclaw/openclaw/issues/64427 and https://github.com/openclaw/openclaw/issues/80091 help Slack DM history and native IM mirror routing, but they do not expose the agent's own channel sends back to a DM turn.

## What This Changes

The issue asks for Slack agents in DM sessions to recall their own messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id or timestamp, exposing only the agent's own outbound channel messages without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored only into the destination channel session, and default sessions_history tree visibility blocks that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be privacy-sensitive. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
