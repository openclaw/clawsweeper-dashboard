## Summary

Keep open. Current main still has the Slack self-activity blind spot: outbound Slack channel sends from a DM/main turn are mirrored to the destination channel session, and default session-tool visibility does not expose that sibling session back to the DM. The adjacent DM-history fix for #64427 is shipped, but it only backfills fresh 1:1 DM history and does not provide cross-session recall of the agent's own channel messages.

## What This Changes

Request Slack agents be able to recall their own outbound channel messages when later asked from a DM session.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent, Slack workspace/account, destination channel/thread, and Slack message timestamp, while preserving existing session isolation for other users' transcript content.

## Reproduction Assessment

Yes, from current-main source inspection: send a Slack `message` tool call from a DM/main session to a channel target, and the outbound route/mirror path records the assistant message in the channel session while default `sessions_history` tree visibility blocks that sibling session from the DM. I did not run a live Slack send because this was a read-only review.

## Solution Assessment

No current implementation is not the best solution yet. Broadly using `tools.sessions.visibility=agent` or merging Slack sessions would overexpose context; the maintainable direction is a scoped self-activity recall path for the agent's own outbound Slack messages.
