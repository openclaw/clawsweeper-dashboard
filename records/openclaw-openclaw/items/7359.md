## Summary

Current main still has the reported Slack cross-session blind spot. Slack message-tool sends are persisted against the destination session, while default session-tool visibility prevents a DM session from reading an unrelated channel session; the closed DM-history issue addresses session initialization, not recall of channel sends from a DM. The remaining work needs an explicit product and privacy decision, so this should stay open for maintainer design rather than autonomous repair.

## What This Changes

Add scoped recall of an agent’s own outbound Slack channel messages when queried from a separate Slack DM session.

## Best Possible Solution

Add one canonical, permission-scoped record or query for agent-authored Slack sends keyed by agent, Slack account/team, destination, thread, and native message identifier, while preserving normal transcript isolation and excluding other participants’ cross-session content.

## Reproduction Assessment

Yes. Current source provides a high-confidence reproduction path: from a Slack DM/main session, send a message to a channel through the message tool, then attempt to recall that send from the DM under default tree session visibility; the destination session is a sibling and remains inaccessible.

## Solution Assessment

Yes in direction, but not yet as an autonomous implementation. A scoped self-activity record is safer and narrower than unified Slack sessions or general sibling-session history access, while storage ownership, retention, and authorization still require maintainer judgment.
