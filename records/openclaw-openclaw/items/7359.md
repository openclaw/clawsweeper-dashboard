## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions as separate buckets, mirrors outbound message-tool sends only to the destination session, and leaves default session tools unable to see that sibling channel session from a DM turn.

## What This Changes

Request that Slack agents be able to recall their own outbound channel messages when later asked from a DM session.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, without merging full sibling transcripts or widening default session visibility globally.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace: a Slack channel send from a DM/main session resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

Yes, keeping this as a scoped Slack self-activity design item is the maintainable path; broad session unification or global visibility widening would be too large and riskier for privacy.
