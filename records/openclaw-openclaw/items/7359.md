## Summary

Keep open. Current main still has the reported Slack self-activity blind spot: Slack DM/main sessions and channel sessions remain separate, outbound Slack sends mirror only into the destination session, and default session-history visibility does not expose that sibling channel session back to a DM/main turn. The adjacent Slack DM-history issue was fixed, but that does not make this issue redundant because this issue is about recalling the agent's own outbound channel activity from another session context.

## What This Changes

The issue asks OpenClaw to let a Slack DM/main session recall the agent's own outbound messages and related channel activity from Slack channel sessions.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack workspace/account, destination channel/thread, and Slack message id or timestamp, exposing only the agent's own outbound channel messages without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored into the channel destination session, and default sessions_history tree visibility rejects that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be privacy-sensitive. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
