## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions separate, mirrors outbound channel sends only into the destination channel session, and default session-tool visibility does not expose that sibling channel session back to a DM turn; this needs a privacy-aware product decision before implementation.

## What This Changes

The issue requests a way for Slack DM sessions to recall the agent's own outbound Slack channel messages and preserve coherent cross-context activity memory.

## Best Possible Solution

Add a narrow Slack self-activity ledger or scoped history bridge keyed by agent, Slack team/account, destination channel/thread, and Slack message id/timestamp without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a DM/main turn that sends to a Slack channel resolves a destination channel route, mirrors only that destination session, and default sessions_history tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because a unified Slack session or broad cross-session history bridge would be risky. The maintainable direction is a scoped self-activity recall mechanism for the agent's own outbound Slack messages.
