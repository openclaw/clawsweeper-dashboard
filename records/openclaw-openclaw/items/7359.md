## Summary

Keep open. Current main and v2026.6.5 still preserve the reported Slack blind spot: Slack DM/main sessions and channel sessions remain separate, outbound message-tool sends mirror into the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM turn.

## What This Changes

The issue asks for Slack DM sessions to recall the agent's own outbound Slack channel messages and related channel activity.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel messages to the right DM/main sessions without merging full channel transcripts or broadening general session visibility.

## Reproduction Assessment

Yes at source level: current main deterministically stores Slack channel sends in the destination channel session while default session-tool visibility hides that sibling session from the DM/main session. I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

Unclear as a final product shape. A narrow self-activity ledger or scoped session-history bridge is safer than merging Slack DMs and channels into one transcript or setting broad session visibility by default.
