## Summary

Keep open. Current main and v2026.6.5 still keep Slack DM/main sessions separate from Slack channel sessions, and the fixed Slack DM-history path only backfills the current DM, not the agent's own outbound channel sends.

## What This Changes

Requests that a Slack DM/main session can recall the agent's own messages sent to Slack channels and follow up coherently across contexts.

## Best Possible Solution

Add a scoped Slack self-activity recall ledger or bridge keyed by agent, Slack team/account, destination channel/thread, and Slack message timestamp, exposing only the agent's own outbound channel messages without merging full transcripts or widening default session visibility.

## Reproduction Assessment

Yes, at source level. Current main routes Slack DMs to main/direct sessions and channels to per-channel keys, mirrors outbound sends only to the destination session, and defaults session-history visibility to tree, so a DM turn cannot see a sibling channel session by default.

## Solution Assessment

Unclear as final product design. A narrow own-outbound activity ledger is safer than unified Slack sessions or broad sibling visibility because it preserves existing privacy boundaries while solving the reported blind spot.
