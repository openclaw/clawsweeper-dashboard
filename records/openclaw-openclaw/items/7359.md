## Summary

Current main still has the reported default Slack blind spot: outbound channel sends are recorded in the destination session, while DM/main sessions default to tree-scoped session visibility. Release v2026.6.5 adds an explicit same-agent visibility option, but it is broader than self-message recall and warns that it may expose other users’ sessions, so the remaining work needs a maintainer product and privacy decision rather than cleanup closure.

## What This Changes

Add a safe way for Slack DM sessions to recall the agent’s own outbound messages sent to channel sessions.

## Best Possible Solution

Keep full same-agent transcript access opt-in, and add a narrowly authorized SQLite-backed activity projection or query containing only the agent’s own outbound message, Slack account/team, destination channel/thread, and message id/timestamp after maintainers define its privacy and retention contract.

## Reproduction Assessment

Yes at source level. A Slack DM/main session and channel use distinct keys, the outbound send is mirrored only to the destination session, and default `tree` visibility prevents the DM session from reading that sibling transcript.

## Solution Assessment

No, neither unified Slack sessions nor broad same-agent transcript access is the best default solution. A self-authored outbound activity surface is narrower, safer, and better aligned with existing session isolation.
