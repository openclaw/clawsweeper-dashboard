## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound channel sends only into the destination channel session, and does not expose that sibling channel session through default DM session-history visibility.

## What This Changes

The issue asks for Slack agents operating from a DM/main session to recall their own outbound messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored only into the destination channel session, and default sessions_history tree visibility blocks that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be privacy-sensitive. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
