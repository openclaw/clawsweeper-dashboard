## Summary

Keep open. Current main still isolates Slack DM/main sessions from channel sessions, mirrors outbound message-tool sends only into the destination session, and blocks sibling session history by default; the closed DM-history fix is adjacent but does not solve channel-send recall from a DM.

## What This Changes

Request that Slack DM sessions can recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall mechanism keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send; the mirror write lands in the channel session and default session tools cannot see that sibling session. I did not run a live Slack workspace repro.

## Solution Assessment

No current implementation solves it. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not merging full DM/channel transcripts or widening default session visibility globally.
