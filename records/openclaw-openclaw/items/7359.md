## Summary

Current main still has the reported blind spot: Slack sends are mirrored only into the destination session, while default session-tool visibility excludes unrelated sibling channel sessions from a DM. The issue needs maintainer product and privacy design rather than closure or autonomous repair.

## What This Changes

Add a privacy-safe way for an agent in a Slack DM to recall its own outbound messages in Slack channels.

## Best Possible Solution

Keep ordinary session isolation intact and add a queryable self-outbound activity record backed by canonical SQLite storage, scoped by agent, Slack account/workspace, authorized destination, message identifier, and bounded retention.

## Reproduction Assessment

Yes, at source level. Current main routes Slack sends to destination sessions, mirrors only into that destination transcript, and defaults session-history access to the requester plus spawned descendants, so a DM cannot inspect an unrelated channel session.

## Solution Assessment

No, unified Slack sessions or broad sibling-session visibility are not the best solution. A narrow self-authored activity ledger or query preserves existing privacy boundaries while addressing the reported recall gap.
