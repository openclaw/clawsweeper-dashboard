## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions in separate session buckets, mirrors outbound channel sends only into the destination session, and does not expose that sibling channel session back to a DM turn under default session-tool visibility.

## What This Changes

The issue requests Slack DM sessions to recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity ledger or scoped history bridge keyed by agent, Slack team/account, destination channel/thread, and Slack message id/timestamp without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a DM/main turn that sends to a Slack channel resolves a destination channel route, mirrors only that destination session, and default sessions_history tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be risky. The maintainable direction is scoped self-activity recall for the agent's own outbound Slack messages.
