## Summary

Keep open. Current main and the latest release still keep Slack DM/main sessions separate from Slack channel sessions, mirror message-tool sends only into the destination session, and restrict default session-tool visibility, so the requested Slack self-activity recall remains unimplemented and needs product/security design.

## What This Changes

The issue requests Slack self-activity recall so an agent asked in a DM can see messages it previously sent to Slack channels.

## Best Possible Solution

Add a bounded Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound messages to authorized DM requesters without merging full transcripts or widening general session visibility.

## Reproduction Assessment

Yes, at source level. Current main routes Slack DMs and Slack channels into separate session keys, mirrors outbound sends to the destination session, and default tree session visibility denies the sibling channel session.

## Solution Assessment

No for the broad unified-session options proposed in the issue. A narrow, authorization-aware self-activity recall path is safer than merging full Slack DM and channel transcripts or globally widening session visibility.
