## Summary

Keep open. Current main still has the Slack cross-session self-activity blind spot: DM/main sessions and Slack channel sessions remain separate, outbound message-tool sends are mirrored to the destination session, and default session-tool visibility does not expose that sibling channel session back to a DM turn.

## What This Changes

The issue asks for Slack agents in DM sessions to recall their own messages sent to Slack channels and follow up on those channel conversations coherently.

## Best Possible Solution

Add a scoped Slack self-activity recall path that exposes only the agent's own outbound channel messages to its DM/main session, keyed by agent, Slack account or team, channel/thread, and Slack message id or timestamp.

## Reproduction Assessment

Yes. A source-level reproduction is clear: from a Slack DM/main turn, send a message to a Slack channel; current main mirrors the assistant message to the destination channel session, and the DM/main requester cannot see that sibling session through default session tools.

## Solution Assessment

Unclear as proposed. A unified Slack session or full cross-session history merge is too broad; the maintainable direction is a narrow self-authored outbound activity ledger or equivalent visibility bridge with strict Slack metadata scoping.
