## Summary

Current main still has the reported blind spot: Slack outbound messages are recorded in destination sessions, while a DM session cannot read unrelated channel sessions under the safe default visibility policy. The separate DM-history fix does not solve this case, and no canonical replacement or implementation was found; the issue needs maintainer product and privacy design before implementation.

## What This Changes

Add a privacy-scoped way for a Slack DM session to recall the agent’s own outbound messages sent into separate Slack channel sessions.

## Best Possible Solution

Add a narrow queryable ledger for the agent’s own outbound Slack activity, keyed by agent, Slack workspace/account, destination channel or thread, and Slack message identity, without merging transcripts or granting general sibling-session access.

## Reproduction Assessment

Yes at source level. A Slack channel send is mirrored into the destination session, while a DM/main requester defaults to tree-scoped session visibility and cannot inspect that unrelated channel session.

## Solution Assessment

No to unified Slack sessions or broad same-agent visibility. A scoped self-authored outbound activity interface is the narrowest solution that preserves existing session and privacy boundaries.
