## Summary

Keep open. Current main still separates Slack DM/main and channel sessions, mirrors outbound channel sends into the destination channel session only, and keeps default session-history visibility scoped to the requester tree; the closed Slack DM-history and thread-context issues are adjacent but do not implement DM recall of the agent's own outbound channel activity.

## What This Changes

The issue requests Slack DM sessions to let the agent recall its own outbound channel messages and related channel activity.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full DM and channel transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection is high-confidence: a DM/main turn that sends to a Slack channel gets a destination channel outbound route, the plugin send mirror writes only to that destination session, and default `sessions_history` tree visibility blocks sibling sessions from the DM/main requester.

## Solution Assessment

Unclear as a product decision. The narrow maintainable direction is scoped self-activity recall, not a broad transcript merge or a generic `tools.sessions.visibility=all` workaround.
