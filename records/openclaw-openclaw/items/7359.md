## Summary

Keep open: current main and release v2026.6.5 still isolate Slack DM and channel sessions, record outbound sends only in the destination session, and restrict default session-tool access, so the reported self-activity blind spot remains; the eventual design needs maintainer privacy and product judgment.

## What This Changes

The issue requests scoped recall of an agent’s own Slack channel messages from a DM session.

## Best Possible Solution

Add a filtered, durable self-activity ledger or query keyed by agent, Slack account/team, destination channel or thread, and Slack message identifier, exposing only the agent’s own outbound activity while preserving separate session transcripts and existing visibility rules.

## Reproduction Assessment

Yes, at source level: Slack uses separate destination session keys, outbound sends are mirrored only into that destination, and default session visibility does not expose sibling channel sessions to a DM. This read-only review did not run a live Slack scenario.

## Solution Assessment

No, the proposed unified-session or general cross-session-history options are too broad. A filtered self-activity surface is the narrowest maintainable design and avoids weakening general transcript isolation.
