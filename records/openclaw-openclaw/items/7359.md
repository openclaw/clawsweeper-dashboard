## Summary

Keep open. Current main still leaves Slack DM/main sessions and Slack channel sessions as separate session buckets, mirrors outbound channel sends only into the destination channel session, and default session-tool visibility still blocks a DM/main requester from reading that sibling channel session.

## What This Changes

The issue asks OpenClaw to let a Slack DM-session agent recall its own outbound Slack channel messages and follow up coherently across DM and channel contexts.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored only into the destination channel session, and default sessions_history tree visibility blocks that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be too risky. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
