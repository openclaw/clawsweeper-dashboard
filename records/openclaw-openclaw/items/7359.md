## Summary

Keep open. Current main still separates Slack DM/main and channel sessions, mirrors outbound Slack channel sends into only the destination channel session, and default session-tool visibility does not expose that sibling channel session back to a DM turn; the closed Slack DM-history work at https://github.com/openclaw/openclaw/issues/64427 is adjacent, not a fix for this self-activity gap.

## What This Changes

Request that a Slack agent operating from a DM can recall its own outbound messages sent to Slack channels without manual user reminders.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, without merging full sibling transcripts or widening default session visibility globally.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace: a DM/main turn that sends to a Slack channel resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

Yes, keeping this open as a scoped Slack self-activity design item is the maintainable path; broad session unification or global visibility widening would be larger and riskier for privacy.
