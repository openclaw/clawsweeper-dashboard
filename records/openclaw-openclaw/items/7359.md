## Summary

Keep open. Current main still mirrors Slack message-tool sends only into the destination channel session, while DM/main sessions remain separate and default session-tool visibility hides sibling channel sessions; the remaining work needs a Slack-specific privacy and authorization design.

## What This Changes

The issue requests Slack DM sessions to expose the agent's own outbound channel messages so the agent can answer questions about prior channel activity.

## Best Possible Solution

Add a bounded Slack self-activity recall mechanism, such as an outbound activity ledger or scoped bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id, without merging full DM and channel transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes. Current-main source shows Slack channel sends mirror to the destination channel session, while the requester DM/main session is a separate session and default tree visibility excludes that sibling; no live Slack workspace run was performed in this read-only review.

## Solution Assessment

No for the broad options in the issue. Unified Slack sessions or general same-agent visibility would over-share; a purpose-built, authorization-aware self-activity recall path is the safer maintainable direction.
