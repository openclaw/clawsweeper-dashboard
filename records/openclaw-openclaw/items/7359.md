## Summary

Keep open. Current main and v2026.6.5 still isolate Slack DM/main sessions from Slack channel sessions, and the existing Slack DM-history fix does not expose the agent's own outbound channel sends back to a DM turn.

## What This Changes

The issue requests Slack DM/main sessions to recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall ledger or bridge keyed by agent, Slack team/account, destination channel/thread, and Slack message timestamp, exposing only the agent's own outbound channel messages without merging full transcripts or widening general sibling-session visibility.

## Reproduction Assessment

Yes, at source level. Current main routes Slack DMs to main/direct sessions and channels to per-channel keys, mirrors outbound sends only to the destination session, and defaults session-history visibility to tree, so a DM turn cannot see a sibling channel session by default.

## Solution Assessment

Unclear as a final product design. A narrow own-outbound activity ledger is safer than unified Slack sessions or broad sibling visibility because it preserves existing privacy boundaries while solving the reported blind spot.
