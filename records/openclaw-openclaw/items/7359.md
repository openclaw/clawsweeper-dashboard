## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions in separate buckets, mirrors outbound channel sends only into the destination session, and does not expose that sibling channel session back to a DM turn under default session-tool visibility. The live issue discussion also explicitly distinguishes the closed DM-history issue as adjacent rather than canonical for this outbound-channel self-activity gap.

## What This Changes

The issue requests that Slack DM sessions be able to recall the agent's own outbound messages sent to Slack channels so the agent can answer follow-up questions about its channel activity.

## Best Possible Solution

Add a narrow Slack self-activity recall path, such as an outbound ledger or scoped history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a DM/main turn sends to a Slack channel, resolves a destination channel session, mirrors only that destination session, and default sessions_history tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be too risky. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
