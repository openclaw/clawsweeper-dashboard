## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors message-tool sends only to the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM; the remaining work needs bounded product/security design rather than cleanup closure.

## What This Changes

Expose the agent's own outbound Slack channel messages to DM-session context so it can recall and follow up on prior channel sends.

## Best Possible Solution

Add a bounded Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound messages to authorized DM requesters without merging full transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes, source-level. Current main routes Slack DMs and Slack channels into separate session keys, mirrors outbound sends to the destination session, and default tree session visibility denies the sibling channel session; I did not run a live Slack workspace.

## Solution Assessment

No for the broad options in the issue. Unified Slack sessions or general same-agent visibility would over-share; a purpose-built, authorization-aware self-activity recall path is the safer maintainable direction.
