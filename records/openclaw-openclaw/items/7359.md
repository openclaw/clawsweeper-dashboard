## Summary

Keep open. Current main still has the reported Slack cross-session blind spot: Slack DM/main sessions and channel sessions are distinct, outbound message-tool sends are mirrored only into the destination channel session, and default session-tool visibility does not expose sibling channel sessions back to a DM turn. The April 26 maintainer review in the discussion also kept this open and treated the open DM-history report #64427 as adjacent rather than canonical.

## What This Changes

The issue requests that a Slack agent be able to recall and reference its own outbound channel messages when later asked from a DM session.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.

## Reproduction Assessment

Yes. The issue has a high-confidence source-level reproduction path: from a Slack DM/main turn, send to a Slack channel with the message tool; current code routes and mirrors the outbound assistant message into the destination channel session, while the DM/main requester cannot see that sibling session under default session-tool visibility.

## Solution Assessment

Unclear as proposed. The broad options in the report could overexpose cross-session history; the narrowest maintainable direction is a Slack self-activity recall path limited to the agent's own outbound messages and explicit Slack destination metadata.
