## Summary

Keep open. Current main still stores Slack DM/main sessions and Slack channel sessions separately, mirrors outbound Slack channel sends only into the destination channel session, and does not expose that sibling session through default session-tool visibility.

## What This Changes

The issue requests a Slack DM/main session to recall the agent's own outbound Slack channel messages and follow up on channel conversations.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, without merging full sibling transcripts or widening default session visibility globally.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace. A Slack channel send from a DM/main session resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

Yes, keeping this as a scoped Slack self-activity design item is the maintainable path. Broad session unification or global visibility widening would be larger and riskier for privacy.
