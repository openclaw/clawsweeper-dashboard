## Summary

Keep open. Current main still has the Slack self-activity blind spot: Slack DM/main sessions and channel sessions remain separate, outbound channel sends are mirrored only into the destination channel session, and default session-tool visibility does not expose that sibling session back to a DM turn. Recent DM-history and outbound-mirroring fixes solve adjacent cases, but not this channel self-activity request.

## What This Changes

The issue requests Slack DM sessions to recall the agent's own outbound Slack channel messages so the agent can answer later DM questions about channel activity.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id/timestamp, without merging full sibling transcripts or widening default session visibility globally.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace: a DM/main turn that sends to a Slack channel resolves and mirrors into the destination channel session, while default tree visibility hides that sibling session from the DM/main turn.

## Solution Assessment

Unclear until maintainers choose the privacy boundary. A narrow self-activity ledger or scoped session-history bridge is safer than unified Slack sessions or global sibling-session visibility.
