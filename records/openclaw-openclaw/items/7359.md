## Summary

Keep open. Current main still has the Slack cross-session blind spot: DM/main and Slack channel sessions are separate, outbound message-tool sends are mirrored into the destination channel session only, and default session-tool visibility does not expose sibling channel sessions back to the DM turn.

## What This Changes

Request a Slack self-activity view so an agent in a DM can recall its own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped bridge for the agent's own channel sends, without merging full DM and channel transcripts.

## Reproduction Assessment

Yes from source inspection, though not from a live Slack run in this read-only review. The current code path shows separate DM/main and channel sessions, destination-only outbound mirroring, and default session-tool visibility that blocks sibling channel history from a DM requester.

## Solution Assessment

No current implementation is sufficient, and there is no proposed patch to review. The maintainable direction is a scoped Slack self-activity surface rather than globally relaxing session visibility or unifying all Slack sessions.
