## Summary

Keep open. Current main and the latest release still separate Slack DM/main and channel sessions, mirror outbound sends only into the destination session, and restrict ordinary session-history access; the requested self-activity recall remains unimplemented and needs an explicit privacy/product contract.

## What This Changes

Add a way for Slack DM sessions to recall the agent’s own outbound channel messages and follow up coherently across contexts.

## Best Possible Solution

Keep Slack sessions isolated and add a bounded, authorization-aware record and query for the agent’s own outbound activity, keyed by agent, Slack account/team, destination, thread, and message identifier without exposing full sibling transcripts.

## Reproduction Assessment

Yes, from source with high confidence: outbound channel sends are mirrored into the destination session, DM and channel keys remain distinct, and the DM requester cannot ordinarily read that sibling session under the visibility guard.

## Solution Assessment

No, broad cross-session history or one unified Slack session is not the best solution because it expands disclosure and couples unrelated conversations; a least-privilege self-authored activity surface is safer.
