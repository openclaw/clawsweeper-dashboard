## Summary

Keep open. Current main and the latest release still isolate Slack DM/main sessions from Slack channel sessions, mirror outbound sends only into the destination session, and keep default session-tool visibility scoped to the current tree; the remaining work needs explicit Slack privacy and authorization design.

## What This Changes

Requests a Slack mechanism that lets an agent recall its own outbound channel messages from a DM session.

## Best Possible Solution

Add a bounded, authorization-aware Slack self-activity recall path for the agent's own outbound messages, without merging full transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes. Current-main source shows Slack channel sends are mirrored to the destination session, while the default tree visibility excludes that sibling session from a DM/main requester; I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

No. Broad unified Slack sessions or global visibility would be too wide; the maintainable solution is a narrow self-activity query or ledger with Slack authorization and retention semantics.
