## Summary

Keep open. Current main still has the Slack self-activity blind spot: Slack DM/main sessions and channel sessions remain separate, outbound message-tool sends mirror to the destination channel session, and default session-history visibility does not expose that sibling session back to the DM. The closed DM-history fix at https://github.com/openclaw/openclaw/issues/64427 is adjacent but does not solve channel-send recall from a DM.

## What This Changes

The issue requests that Slack DM sessions be able to recall the agent's own outbound messages sent to Slack channels and follow up across Slack contexts.

## Best Possible Solution

Add a scoped Slack self-activity recall mechanism keyed by agent, account/team, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to DM/main sessions.

## Reproduction Assessment

Yes, from source inspection: send a Slack message from a DM/main session to a channel, then ask from the DM/main session about that send; the mirror write lands in the channel session and default session tools cannot see that sibling session. I did not run a live Slack workspace repro.

## Solution Assessment

No current implementation solves it. The maintainable path is a scoped self-activity ledger or Slack-specific history bridge, not merging full DM/channel transcripts or widening default session visibility globally.
