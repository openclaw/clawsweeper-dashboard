## Summary

Keep open. Current main still separates Slack DM/main and channel sessions, mirrors outbound message-tool sends only into the destination session, and default session tools do not expose that sibling channel session back to a DM; the closed DM-history work in #64427 is adjacent but narrower.

## What This Changes

Request Slack agents to recall their own outbound channel messages and channel conversations when asked from a DM session.

## Best Possible Solution

Add a scoped Slack self-activity recall path, such as an outbound activity ledger keyed by agent, Slack team/account, channel/thread, and Slack message timestamp, without merging full transcripts or broadening default session visibility.

## Reproduction Assessment

Yes. A source-level reproduction is clear: send from a Slack DM/main turn to a Slack channel, then the assistant message is mirrored to the destination channel session while default session tools hide that sibling session from the DM/main requester.

## Solution Assessment

Unclear as proposed. A unified Slack session or full cross-session history merge is too broad; the maintainable direction is a narrow self-authored outbound activity ledger or equivalent scoped visibility bridge.
