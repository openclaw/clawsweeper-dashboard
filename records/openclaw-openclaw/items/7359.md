## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors message-tool channel sends only into the destination channel session, and default session-tool visibility does not expose that sibling channel session back to a DM turn.

## What This Changes

The issue asks for Slack DM sessions to recall the agent's own outbound Slack channel messages and related channel conversations.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a DM/main turn sends to a Slack channel, resolves a destination channel session, mirrors only that destination session, and default `sessions_history` tree visibility blocks the sibling channel session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be too risky. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
