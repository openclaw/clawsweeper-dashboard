## Summary

Current main still has the Slack self-activity blind spot: Slack DM/main and channel sessions remain separate, outbound channel sends are mirrored only to the destination session, and default session-history visibility blocks a DM/main requester from reading the sibling channel session. The closed Slack DM history issue and the broader shared-sessions feature do not cover this narrower recall of the agent's own outbound channel messages.

## What This Changes

Request a Slack self-activity recall path so an agent in a DM can know about messages it previously sent to Slack channels.

## Best Possible Solution

Add a scoped Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id or timestamp, exposing only the agent's own outbound channel messages without merging full transcripts or widening generic session visibility.

## Reproduction Assessment

Yes, by source inspection. A DM/main turn can send to a Slack channel, the send is mirrored into the channel destination session, and default sessions_history tree visibility rejects that sibling session from the DM/main requester.

## Solution Assessment

Unclear as originally proposed because unified Slack sessions or broad cross-session history would be privacy-sensitive. The maintainable direction is scoped recall of the agent's own outbound Slack channel activity only.
