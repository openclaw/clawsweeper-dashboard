## Summary

Keep open. The source and release checks show OpenClaw still routes Slack DM/main and Slack channel conversations into separate sessions, mirrors outbound Slack sends only into the destination session, and keeps sibling session access blocked by default, so the requested Slack self-visibility behavior remains missing and needs product/security design rather than closure.

## What This Changes

The issue requests a way for an agent in a Slack DM session to recall its own messages previously sent in Slack channel sessions.

## Best Possible Solution

Design a bounded Slack self-activity recall path keyed by agent, Slack team/account, destination channel/thread, and Slack message id or timestamp, exposing only the agent's own authorized outbound messages without merging full transcripts or widening general session visibility.

## Reproduction Assessment

Yes, source inspection is enough to reproduce the missing behavior: Slack channel sends are routed and mirrored into channel sessions, while DM sessions keep separate keys and default session-tool visibility blocks sibling access.

## Solution Assessment

No single code-path patch is sufficient yet; the best fix needs a maintainer-approved product and security design for narrow self-activity recall instead of broad transcript sharing.
