## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions as separate visibility buckets, and outbound Slack message-tool sends mirror into the destination session only; the remaining work is a bounded Slack self-activity design, not conservative cleanup.

## What This Changes

The issue requests Slack cross-session self-activity visibility so an agent in a DM can recall channel messages it previously sent.

## Best Possible Solution

Add a bounded Slack self-activity recall path keyed by agent, Slack account/team, destination channel/thread, and Slack message id, without merging full DM and channel transcripts or relaxing general session visibility.

## Reproduction Assessment

Yes, source-level. Current main shows Slack channel sends mirror to the destination channel session, while the requester DM/main session is separate and default tree visibility excludes that sibling; no live Slack workspace run was performed in this read-only review.

## Solution Assessment

No for the broad options in the issue. Unified Slack sessions or general same-agent visibility would over-share; a purpose-built, authorization-aware self-activity recall path is the safer maintainable direction.
