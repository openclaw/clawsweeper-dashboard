## Summary

Keep open. Current main still applies the recognized-bootstrap basename whitelist and workspace-root guarded read to configured `bootstrap-extra-files.paths`, while diagnostics are only debug-logged; however, relaxing those checks would change the documented bootstrap trust boundary and needs maintainer product/security review rather than automatic cleanup.

## What This Changes

The issue requests that explicitly configured `bootstrap-extra-files.paths` bypass the bootstrap basename whitelist, allow absolute paths outside the workspace, and surface skipped-path diagnostics.

## Best Possible Solution

Define the intended trust contract first, then either keep the documented workspace/basename restrictions with visible skip diagnostics or deliberately support explicit arbitrary files with tests, docs, size limits, and security review.

## Reproduction Assessment

Yes from source inspection: configured paths flow into `loadExtraBootstrapFilesWithDiagnostics`, where non-whitelisted basenames are rejected and reads stay rooted at the workspace. I did not run a live bootstrap because this review is read-only.

## Solution Assessment

Unclear. Bypassing the whitelist and workspace boundary would address the reporter’s deployment, but it changes the documented security contract; a maintainer-approved contract plus diagnostics, docs, and regression tests is the safer path.
