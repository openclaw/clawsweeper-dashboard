## Summary

Keep open: current main still has a source-reproducible path where enforce-mode session maintenance can delete non-protected subagent rows with `status: "running"`, and the directly related fix PR is closed unmerged with no replacement found.

## What This Changes

The issue asks session maintenance to preserve active or running subagent sessions during enforce-mode age, count, and disk-budget pruning.

## Best Possible Solution

Land a focused core fix that derives active/running subagent child session keys once and protects them consistently across save-time stale pruning, count capping, explicit load-time maintenance, disk-budget eviction, CLI cleanup, tests, docs, and changelog.

## Reproduction Assessment

Yes, from source inspection: an old or over-limit non-protected subagent entry with `status: "running"` can be removed when it is not the caller's single `activeSessionKey`. I did not run a live throughput reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable direction is to share a protected active-session-key set across the existing maintenance paths, using the persisted-plus-memory subagent run snapshot seam where registry state is needed.
