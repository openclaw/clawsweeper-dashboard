## Summary

Close: current main protects active and pending subagent child session keys from enforce-mode session pruning, capping, explicit cleanup/load maintenance, and disk-budget eviction, and the same source is present in the v2026.5.12 release.

## What This Changes

The issue asks for session maintenance to skip active/running subagent sessions instead of pruning them by age, count, or disk pressure.

## Best Possible Solution

Keep the provider-based preservation shipped in v2026.5.12; broader per-pattern retention policy remains separate from this active-session safety fix.

## Reproduction Assessment

Yes for the original affected behavior: enforce mode with saturated maxEntries or disk budget and a running subagent child session used to be source-reproducible. On current main and v2026.5.12, that path no longer reproduces because active and pending child keys are collected into preserveKeys before maintenance deletes rows.

## Solution Assessment

Yes. The merged provider seam keeps subagent lifecycle knowledge in the agents layer while the generic session-maintenance code consumes preserveKeys consistently, which is narrower and more maintainable than adding a new retention config or channel-specific exception.
