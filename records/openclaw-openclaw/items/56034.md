## Summary

Keep open: current main still has a source-reproducible path where enforce-mode session maintenance can prune non-protected persisted sessions with `status: "running"`, and related PR #59932 is still open rather than merged.

## What This Changes

The issue asks session maintenance to skip active/running sessions during age, count, and disk-budget pruning.

## Best Possible Solution

Land a focused core fix that protects all active/running session keys across save-time stale pruning, count capping, load-time maintenance, and disk-budget eviction, with regression coverage and docs updated for the active-running exception.

## Reproduction Assessment

Yes, from source inspection: an enforce-mode store with an old or over-limit non-protected subagent row marked `status: "running"` can be deleted when it is not the caller `activeSessionKey`. I did not run a live reproduction because this review was read-only and `pnpm` was unavailable.

## Solution Assessment

Yes, the narrow maintainable fix is to derive and apply a shared protected-session-key set for active/running sessions across every eviction path. PR #59932 should be reviewed for full coverage or replaced by an equivalent focused patch before closing this issue.
