## What This Changes

The issue reports a regression where a Qwen3.5 4B Ollama model that could use OpenClaw tools in v2026.5.7 no longer receives or uses tools in v2026.5.12.

## Best Possible Solution

Capture the reported model's `/api/show` capabilities and OpenClaw `/api/chat` payload on current main, then either fix the capability reconciliation or document a supported explicit override without weakening the fallback for models that truly lack tool support.
