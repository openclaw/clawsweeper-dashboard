## What This Changes

The issue reports that a custom Ollama Qwen3.5 4B GGUF model could use tools in OpenClaw v2026.5.7 but stopped using tools in v2026.5.12.

## Best Possible Solution

Capture live /api/show capabilities and /api/chat payloads for the reported model across v2026.5.7, v2026.5.12, and current main, then restore the compatible tool path without weakening the existing protection for Ollama models that truly do not support tools.

No high-confidence live reproduction yet. ## Solution Assessment

Unclear. A blind change that ignores Ollama capabilities would be risky because current docs and tests intentionally disable tools for models that omit tool support; the safer path is live proof followed by a narrow compatibility or documentation decision.
