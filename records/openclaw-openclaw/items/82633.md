Keep open: this is a fresh Ollama/Qwen regression with a plausible current-main source path, and a second user has now reported adjacent Qwen 3.5 behavior after upgrading. ## What This Changes

The issue reports that a custom Ollama Qwen3.5 4B model could call OpenClaw tools in v2026.5.7 but no longer appears to receive or use tools in v2026.5.12.

## Best Possible Solution

Capture live Ollama `/api/show` capabilities and the OpenClaw native `/api/chat` payload for the reported model, then make a narrow compatibility or documentation change that preserves the guard for models that truly lack tool support.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection shows a plausible current-main path where Ollama capabilities that omit `tools` set `compat.supportsTools=false` and cause the runner to remove OpenClaw tools, but the reported model's real `/api/show` and `/api/chat` payloads are missing.

## Solution Assessment

Unclear. A blind change that ignores Ollama capabilities is not the best fix because current tests and docs intentionally honor upstream tool capability metadata; the safer path is live proof followed by a narrow compatibility rule, explicit override, or documentation update.
