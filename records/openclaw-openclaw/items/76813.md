## Summary

Keep this PR open: it targets a real Anthropic bundled-policy bug and the direction is right, but the patch inlines a divergent Claude profile and loses the Claude 4.6 adaptive default contract before merge.

## What This Changes

The PR adds an Anthropic provider-policy `resolveThinkingProfile` export with tests for Opus 4.7, Sonnet 4.6, `claude-cli`, and non-Anthropic providers.

## Best Possible Solution

Keep the PR open and replace the inline Anthropic profile table with the shared Claude resolver so bundled Anthropic, runtime Anthropic, and proxied Claude providers stay aligned.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the Anthropic bundled `resolveThinkingProfile` hook while the fallback resolver calls it, and the linked issue provides a concrete 2026.5.2 published-bundle probe where Opus 4.7 `max` downgrades to `high`.

## Solution Assessment

No. Exporting the bundled hook is the right solution, but this PR should delegate to `resolveClaudeThinkingProfile` instead of inlining a profile that uses the wrong default field/value.
