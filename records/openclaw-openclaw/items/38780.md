## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require explicit delegate-drain capability:** `src/agents/openclaw-tools.ts:346-347`
  - body: `continue_delegate` is exposed whenever continuation is enabled unless `drainsContinuationDelegateQueue` is explicitly false. Several call paths fail to thread that flag, so missing metadata defaults open and can let runs enqueue delegates even when no reliable post-turn drain is guaranteed. Gate this tool on explicit drain capability and thread the flag from drain-capable callers.
  - confidence: 0.87
- **[P2] Check TaskFlow finish before returning delegates:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98-101`
  - body: `finishFlow` returns `applied: false` on revision conflicts or missing flows, but this consume path ignores the result and still returns the delegate as consumed. If finalization fails, the flow can remain queued and be dispatched again later.
  - confidence: 0.84
- **[P2] Check cancel status updates before leaving flows queued:** `src/auto-reply/continuation-delegate-store-taskflow.ts:134-141`
  - body: After `requestFlowCancel` succeeds, the follow-up transition to `cancelled` also returns a result, but this code ignores it. A revision conflict can leave a cancel-requested flow in `queued`, allowing later consume passes to dispatch work that should have been cancelled.
  - confidence: 0.82
- **[P2] Remove the dynamic import warning suppression:** `tsdown.config.ts:62-66`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` hides the repo's build guard around a runtime loading boundary that this PR changes. Keep the warning visible and fix the static/dynamic import shape instead.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces autonomous scheduling and delegation paths with concrete fail-open drain, replay, cancellation, and build-boundary concerns.

Concerns:

- **[medium] Continuation delegation is exposed without reliable drain proof:** `src/agents/openclaw-tools.ts:346`
  - body: The `continue_delegate` tool is exposed by default when the drain flag is absent, which can broaden autonomous scheduling into run paths that have not proven they consume queued delegate work.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay consumed or cancelled work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores `applied: false` from finish and cancel status updates, so conflict cases can leave queued records dispatchable after they were logically consumed or cancelled.
  - confidence: 0.81
- **[low] Build warning suppression weakens runtime-boundary review:** `tsdown.config.ts:62`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` is not a secret leak, but it removes a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
