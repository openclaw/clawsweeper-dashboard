## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Forward the delegate drain flag into embedded tools:** `src/agents/pi-embedded-runner/run/attempt.ts:689-690`
  - body: `runEmbeddedPiAgent` accepts `drainsContinuationDelegateQueue`, but this tool construction only passes continuation work and compaction callbacks into `createOpenClawCodingTools`. Because `createOpenClawTools` exposes `continue_delegate` when the drain flag is omitted, embedded runs can advertise a scheduling tool whose queued delegates are not drained by that path.
  - confidence: 0.87
- **[P2] Check TaskFlow finish results before returning delegates:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98-101`
  - body: `finishFlow` reports revision conflicts through `applied: false`, but this consume path ignores the result and still returns the delegate. If finalization fails, the flow can remain queued, so a later consume can dispatch the same delegate again.
  - confidence: 0.84
- **[P2] Check cancel status updates before leaving flows queued:** `src/auto-reply/continuation-delegate-store-taskflow.ts:134-141`
  - body: After `requestFlowCancel` succeeds, the status transition to `cancelled` also returns a result, but this code ignores it. A concurrent revision change can leave a cancel-requested flow with status `queued`, and later consume passes can still dispatch work that should have been cancelled.
  - confidence: 0.82
- **[P2] Remove the dynamic import warning suppression:** `tsdown.config.ts:63-66`
  - body: The build config now suppresses `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts`. This hides the lazy/static import boundary warning the repo normally treats as a guard while the PR changes runtime lazy-loading paths around continuation and embedded runs.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces autonomous scheduling and delegation paths while weakening delegate-drain, cancel, and build-boundary controls.

Concerns:

- **[medium] Continuation delegation is exposed outside draining runs:** `src/agents/openclaw-tools.ts:347`
  - body: `continue_delegate` can be exposed when continuation is enabled and the drain flag is omitted, while embedded tool construction does not forward the drain flag. For an opt-in autonomous background-work feature, this broadens the tool surface beyond the expected scheduler and drain pipeline.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay cancelled or consumed work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores `applied: false` results from finish and cancel status updates. In conflict cases, queued records can remain dispatchable after they were logically consumed or cancelled, weakening operator cancellation and autonomous-work bounds.
  - confidence: 0.81
- **[low] Build warning suppression weakens lazy-boundary review:** `tsdown.config.ts:63`
  - body: The diff suppresses `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts`. This is not a dependency or secret leak, but it weakens a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
