  - body: `runEmbeddedAttempt` passes the new continuation callbacks into `createOpenClawCodingTools` but does not pass `drainsContinuationDelegateQueue`.   - body: `finishFlow` signals revision conflicts with `applied: false`, but this consume path ignores the result and still returns the delegate. If finalization fails, the flow can remain queued and a later consume can dispatch the same delegate again.
  - confidence: 0.84

  - body: After `requestFlowCancel` succeeds, the follow-up transition to `cancelled` also returns a result, but this code ignores it. A revision conflict can leave a cancel-requested flow with status `queued`, so later consume passes can still dispatch work that should have been cancelled.
  - confidence: 0.82
- **[P2] Remove the dynamic import warning suppression:** `tsdown.config.ts:63-66`
  - body: The diff suppresses `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts`. That hides the build guard the repo uses for lazy/static import boundaries while this PR changes embedded runtime and continuation loading paths.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces autonomous scheduling and delegation paths with concrete control-bound, replay, cancellation, and build-boundary concerns.

Concerns:

- **[medium] Delegate tools are outside reliable drain control:** `src/agents/pi-embedded-runner/run/attempt.ts:689`
  - body: The embedded tool path does not forward the delegate-drain capability consistently, which can expose or hide autonomous delegation independently of whether queued delegate work will be drained.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay consumed or cancelled work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores `applied: false` results from finish and cancel status updates. In conflict cases, queued records can remain dispatchable after they were logically consumed or cancelled.
  - confidence: 0.81
- **[low] Build warning suppression weakens runtime-boundary review:** `tsdown.config.ts:63`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` is not a secret or dependency leak, but it weakens a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
