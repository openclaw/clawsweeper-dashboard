## Summary

Keep open: current main does not implement this continuation surface, and the latest PR head still has concrete blockers in delegate dispatch, cleanup, and continuation opt-in enforcement. This was a source review; validation commands were not run because `pnpm` is unavailable in this runner.

## What This Changes

The PR adds opt-in continuation tools, context-pressure and compaction hooks, TaskFlow-backed delegate queues, status/protocol/docs updates, and continuation-focused tests for autonomous agent turn continuation.

## Best Possible Solution

Keep this PR in review, remove the stale inline delegate drain in favor of one TaskFlow-aware dispatcher, require explicit continuation enablement before pressure events, avoid releasing queued delegates in cleanup without dispatch, then rebase and run focused continuation tests plus build/changed-gate validation before maintainer product and security approval.

## Reproduction Assessment

Yes for the review findings: PR-head source shows the delegate rows are consumed and finished before the dispatcher can see them, cleanup can release rows without dispatch, and context-pressure events ignore the explicit enabled flag. End-user reproduction is not applicable because this is a new feature PR.

## Solution Assessment

No. The feature direction may be useful, but the current patch is not the best mergeable form while durable delegate accounting is split across two paths and disabled continuation configs can still inject continuation-specific model guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Route tool delegates through the single dispatcher:** `src/auto-reply/reply/agent-runner.ts:2557`
  - body: This inline `consumePendingDelegates()` drains and finishes TaskFlow rows before the later `dispatchToolDelegates()` call can see them. That bypasses the new dispatcher's failure marking and hedge behavior, so rejected or failed inline spawns can be lost after the flow was already marked released.
  - confidence: 0.9
- **[P2] Gate context-pressure checks on continuation enablement:** `src/auto-reply/reply/agent-runner.ts:1434-1436`
  - body: `continuationFeatureEnabled` is computed above, but this block runs without checking it. A config with `continuation.enabled: false` and a retained threshold can still inject `[system:context-pressure]` guidance that tells the model to use `CONTINUE_DELEGATE`, violating the opt-in contract.
  - confidence: 0.86
- **[P2] Do not release queued delegates during cleanup:** `src/auto-reply/reply/agent-runner.ts:2938`
  - body: The `finally` cleanup calls `consumePendingDelegates(sessionKey)` and ignores the returned delegates. Because consuming a matured TaskFlow row calls `finishFlow`, any turn that errors after a `continue_delegate` tool call can silently drop the queued work while recording it as released.
  - confidence: 0.82
