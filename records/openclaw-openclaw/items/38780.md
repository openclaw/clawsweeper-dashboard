## Summary

Keep this PR open: current main does not implement the proposed self-elected continuation tools, and the latest inspected PR head still has merge-blocking correctness issues in the new continuation/delegation control path.

## What This Changes

Adds an opt-in continuation runtime with `continue_work`, `continue_delegate`, `request_compaction`, context-pressure events, TaskFlow-backed delegate queues, gateway/client/status/docs updates, diagnostics, and tests.

## Best Possible Solution

Keep the feature in review until continuation tools are exposed only on drain-capable paths, tool delegates use one TaskFlow-aware dispatcher, pressure events honor `continuation.enabled`, the branch is cleanly mergeable, and maintainers approve the autonomy model.

## Reproduction Assessment

Yes for the PR findings: source inspection of PR head `5397a00a` shows the non-draining tool exposure, context-pressure checks outside the enabled guard, and a duplicate inline delegate drain before the dispatcher. This is a new feature PR rather than an existing end-user bug reproduction.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable form while the branch is unmergeable and the new autonomy surface still has correctness and security defects in its control path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require an explicit delegate drain capability:** `src/agents/openclaw-tools.ts:613-618`
  - body: This exposes `continue_delegate` whenever the drain flag is omitted, not only when the current run actually consumes the delegate queue. Direct agent/command paths can therefore report `scheduled` after enqueuing TaskFlow work that no post-run scheduler drains.
  - confidence: 0.86
- **[P2] Route tool delegates through the single dispatcher:** `src/auto-reply/reply/agent-runner.ts:2556-2558`
  - body: This inline `consumePendingDelegates()` drains pending rows before the later `dispatchToolDelegates()` call can see them. That bypasses the new dispatcher’s failure accounting and hedge handling, so failed inline spawns can be lost or misreported.
  - confidence: 0.87
- **[P2] Gate context-pressure events on continuation enablement:** `src/auto-reply/reply/agent-runner.ts:1430-1436`
  - body: This block runs whenever a session exists and a pressure threshold is configured, even when `agents.defaults.continuation.enabled` is false. Disabled deployments can still enqueue model-visible continuation guidance, violating the opt-in contract.
  - confidence: 0.84
