## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require delegate drains before exposing continue_delegate:** `src/agents/openclaw-tools.ts:658-663`
  - body: This gate exposes `continue_delegate` whenever `drainsContinuationDelegateQueue` is omitted. Direct agent/command paths forward that optional flag unchanged and do not run the auto-reply post-turn delegate drain, so those runs can report `scheduled` after enqueueing TaskFlow work that has no guaranteed consumer. Require an explicit drain-capable path, or pass `false` from non-draining callers.
  - confidence: 0.86
- **[P2] Gate pressure checks on continuation.enabled:** `src/auto-reply/reply/agent-runner.ts:1463-1475`
  - body: This block runs before checking `continuationFeatureEnabled`, so a config with `continuation.enabled: false` but a retained `contextPressureThreshold` still enqueues model-visible context-pressure guidance. That violates the opt-in contract and can prompt agents to use continuation behavior while the feature is disabled.
  - confidence: 0.84
- **[P2] Honor zero-delay continue_work requests:** `src/auto-reply/reply/agent-runner.ts:2513-2514`
  - body: The `continue_work` tool normalizes omitted delay to `0` and returns `delaySeconds: 0`, but this clamp applies `minDelayMs` anyway. With the default minimum, every supposedly immediate tool request waits about five seconds, so the tool's observable scheduling contract is wrong.
  - confidence: 0.82
