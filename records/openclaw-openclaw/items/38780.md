## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Forward the delegate drain flag into embedded tools:** `src/agents/pi-embedded-runner/run/attempt.ts:689-690`
  - body: `RunEmbeddedPiAgentParams` has `drainsContinuationDelegateQueue`, and `createOpenClawTools` uses that flag to gate `continue_delegate`, but this embedded path forwards only the continuation callbacks. Because an omitted flag is treated as allowed, explicitly non-draining embedded runs can still expose `continue_delegate` and queue work that no consumer is expected to drain.
  - confidence: 0.87
- **[P2] Check TaskFlow finish results before returning delegates:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98-101`
  - body: `finishFlow` signals revision conflicts with `applied: false`, but this consume path ignores the result and still returns the delegate. If finalization fails, the flow can remain queued and a later consume can dispatch the same delegate again.
  - confidence: 0.84
- **[P2] Check cancel status updates before leaving flows queued:** `src/auto-reply/continuation-delegate-store-taskflow.ts:134-141`
  - body: After `requestFlowCancel` succeeds, the follow-up status transition to `cancelled` also returns an update result, but this code ignores it. A revision conflict can leave a cancel-requested flow with status `queued`, so later consume passes can still dispatch work that should have been cancelled.
  - confidence: 0.82
- **[P2] Remove the dynamic import warning suppression:** `tsdown.config.ts:63-66`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` hides the lazy/static import boundary warning the repo uses as a build guard while this PR changes embedded runtime and continuation loading paths. Keep the warning visible and fix the import boundary instead.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces autonomous scheduling and delegation paths with concrete control-bound and replay concerns.

Concerns:

- **[medium] Continuation delegation can be exposed without reliable drain control:** `src/agents/pi-embedded-runner/run/attempt.ts:689`
  - body: The embedded tool path does not forward the delegate-drain capability into tool construction, while the downstream gate treats an omitted flag as allowed. That can broaden autonomous delegation beyond runs that are expected to consume queued work.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay cancelled or consumed work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores `applied: false` results from finish and cancel status updates. In conflict cases, queued records can remain dispatchable after they were logically consumed or cancelled.
  - confidence: 0.81
- **[low] Build warning suppression weakens lazy-boundary review:** `tsdown.config.ts:63`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` is not a secret or dependency leak, but it weakens a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
