## Summary

Keep open. Current main has adjacent restart-continuation and sub-agent/task-flow substrate, but it does not implement this PR's self-elected continuation tools; the PR head still has concrete correctness and security-sensitive control-flow blockers before it can merge.

## What This Changes

The branch adds opt-in agent continuation tools, response-token fallback parsing, context-pressure and compaction hooks, continuation/delegate scheduling with optional TaskFlow storage, gateway/client/status/docs updates, and continuation-focused tests.

## Best Possible Solution

Keep the PR open and make it mergeable by rebasing onto current main, preserving the existing restart/session/sub-agent substrate, and tightening the new continuation control plane so delegation is only exposed on drain-capable runs and TaskFlow-backed delegate consumption/cancellation fails closed on revision conflicts.

## Reproduction Assessment

Not applicable as a user bug reproduction. Yes for the review findings: they are reproducible by static inspection of PR head 90db3699ccf3b6c7973dd3fdd9d489c8b507ff3f against current main's TaskFlow update contract and tool-construction paths.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best mergeable solution while delegate scheduling can be exposed without a drain pipeline, TaskFlow conflicts can replay consumed or cancelled delegates, and a build-boundary warning is suppressed instead of resolved.

## Security Review

Status: needs_attention

Summary: The diff introduces autonomous scheduling and delegation paths with concrete control-bound and cancellation concerns.

Concerns:

- **[medium] Continuation delegation is exposed outside draining runs:** `src/agents/openclaw-tools.ts:347`
  - body: `continue_delegate` can be exposed when continuation is enabled and the drain flag is omitted, while embedded tool construction does not forward the drain flag. For autonomous background work, that broadens the scheduling surface beyond the expected drain pipeline.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay cancelled or consumed work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores `applied: false` results from finish and cancel status updates. In conflict cases, queued records can remain dispatchable after they were logically consumed or cancelled, weakening operator cancellation and autonomous-work bounds.
  - confidence: 0.81
- **[low] Build warning suppression weakens lazy-boundary review:** `tsdown.config.ts:63`
  - body: Suppressing `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts` is not a dependency or secret leak, but it weakens a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
