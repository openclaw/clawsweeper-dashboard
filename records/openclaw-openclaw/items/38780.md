## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require an explicit delegate-drain capability:** `src/agents/openclaw-tools.ts:614-618`
  - body: This exposes `continue_delegate` whenever the drain flag is omitted, not only when the current run actually consumes the delegate queue. Direct agent/command paths can therefore return `scheduled` after enqueuing TaskFlow work that no post-run scheduler drains, leaving work orphaned or replayed on a later unrelated turn.
  - confidence: 0.86
- **[P2] Route tool delegates through the single dispatcher:** `src/auto-reply/reply/agent-runner.ts:2557`
  - body: This inline `consumePendingDelegates()` drains pending TaskFlow rows before the later `dispatchToolDelegates()` call can see them. That bypasses the new dispatcher’s failure accounting and hedge handling, so failed inline spawns can be lost or misreported after the flow was already consumed.
  - confidence: 0.87
- **[P2] Gate context-pressure events on continuation enablement:** `src/auto-reply/reply/agent-runner.ts:1434-1436`
  - body: This block runs whenever a threshold is configured, even when `agents.defaults.continuation.enabled` is false. Disabled deployments can still enqueue model-visible `[system:context-pressure]` guidance mentioning `CONTINUE_DELEGATE`, which violates the opt-in contract for this autonomy surface.
  - confidence: 0.84
