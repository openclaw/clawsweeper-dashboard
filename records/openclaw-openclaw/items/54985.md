## Summary

Keep this PR open but do not merge it as-is: current main does not yet implement the requested header surface, and the proposed patch still has blocking privacy and metadata-correctness issues that overlap with #69463 and #22021.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and per-turn context headers to embedded LLM requests, propagates fallback state through model-fallback callers, and exposes `isFallback` on plugin LLM output events.

## Best Possible Solution

Adopt a maintainer-approved observability design that scopes context headers to trusted gateway/proxy targets, derives fallback state from fallback candidate selection, maintains a per-run model-call counter across retries, and coordinates the public contract with #69463 and #22021.

## Reproduction Assessment

Yes. The blockers are reproducible by static diff tracing: the patch unconditionally merges new headers into model headers, candidate skips can make executed-callback fallback counters wrong, and the per-turn counter resets inside each embedded attempt.

## Solution Assessment

No. The direction is useful, but this patch is not the safest implementation until the header trust boundary and metadata semantics are explicit; #69463's opt-in configuration direction is a safer base to reconcile with this work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new `gatewayContextHeaders` are merged into `params.model.headers`, so ordinary provider transports can receive stable OpenClaw session and attempt identifiers. Make this opt-in or restrict it to trusted gateway/proxy targets before exposing those correlation IDs.
  - confidence: 0.86
- **[P2] Use fallback runner options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: The PR computes candidate-level fallback state in `runWithModelFallback`, but this path still forwards `params.isFallbackRetry`, which only counts executed callbacks. If the primary candidate is skipped before invocation, the first executed fallback is labeled primary in the new header and hook metadata.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: `llmTurnCount` is local to one `runEmbeddedAttempt` and is seeded from persisted assistant messages. Retry paths can restart before an assistant message is written, so a second HTTP request in the same run can reuse the same `{runId}-turn-{index}` value.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch forwards stable OpenClaw session/run identifiers to outbound model-provider headers without an opt-in trusted-gateway boundary.
