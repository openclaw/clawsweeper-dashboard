## Summary

Keep open for maintainer review. Current main does not implement these headers, but this PR is not mergeable as-is because it exports stable identifiers to outbound model requests without a trusted or opt-in boundary, has source-reproducible fallback and turn metadata bugs, lacks contributor real-behavior proof, and overlaps with https://github.com/openclaw/openclaw/pull/69463 and https://github.com/openclaw/openclaw/issues/22021.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and per-turn context headers to embedded LLM requests and exposes fallback status on plugin LLM output events.

## Best Possible Solution

Adopt one maintainer-approved, privacy-safe observability contract that is opt-in or trusted-target scoped, derives fallback state from selected candidates, uses per-run unique model-call identifiers, and is reconciled with the related model/header work before adding docs, tests, and changelog coverage.

## Reproduction Assessment

Yes. The blocking PR issues are source-reproducible from the PR diff and current main: the new headers merge into model headers, fallback candidates can be skipped before callback execution, and the turn counter is local to one embedded attempt.

## Solution Assessment

No. The direction is useful, but this patch is not the best solution until maintainers define the privacy/API contract and the implementation fixes fallback and model-call identity semantics.

  - confidence: 0.82
