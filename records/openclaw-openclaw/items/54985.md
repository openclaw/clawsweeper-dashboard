## Summary

Keep this PR open, but it is not merge-ready: the request-correlation direction is useful, yet the current patch sends stable identifiers to model providers by default, can reuse turn IDs across retries, can mislabel skipped-primary fallback attempts, and still lacks real behavior proof.

## What This Changes

The branch adds outbound LLM request context headers, per-call turn headers, fallback metadata propagation, and optional plugin `llm_output.isFallback` metadata across embedded agent fallback paths.

## Best Possible Solution

Define one documented opt-in or provider-owned transport identity contract that bounds what identifiers leave the process, keeps model-call IDs stable across retries, and coordinates with the open tracing/model-metadata work.
