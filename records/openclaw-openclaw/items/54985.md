## Summary

Keep open. The feature is useful, but the current PR still needs a maintainer privacy/API decision, fixes for fallback and request-id semantics, and contributor-supplied real behavior proof before merge.

## What This Changes

The PR adds outbound LLM request context headers for session/run/fallback/turn tracking and exposes `isFallback` on LLM output plugin hook events.

## Best Possible Solution

Define one documented, privacy-safe LLM request-correlation contract that is opt-in or trusted-target scoped, uses bounded identifiers, carries fallback state from actual model selection, and assigns unique per-model-call IDs across retries.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR. The blocking PR issues are source-reproducible from the diff and current fallback/retry paths.

## Solution Assessment

No. The request-correlation direction is useful, but this implementation should first establish the privacy/API boundary and fix fallback plus per-request identity semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate context headers to trusted targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2182-2194`
  - body: The new gateway context headers are merged into model headers before session creation, so stable session and run identifiers can be sent to ordinary provider transports. Gate this behind an explicit opt-in or trusted gateway target, or send bounded non-sensitive identifiers before crossing the provider boundary.
  - confidence: 0.9
- **[P2] Use the fallback runner state for agent-command metadata:** `src/agents/agent-command.ts:523`
  - body: This passes `isFallback` from the callback-count-derived `isFallbackRetry`, but candidates can be skipped before the callback runs. Use the fallback runner's `runOptions?.isFallback` so the first executed fallback is not labeled as primary in headers or hook events.
  - confidence: 0.87
- **[P2] Keep model-call IDs unique across retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2460-2469`
  - body: The turn counter is scoped to one embedded attempt and seeds IDs from persisted assistant messages, so retries before an assistant message is written can reuse the same `X-OpenClaw-Turn-Id`. Carry a run-scoped request counter or diagnostic call id across retries.
  - confidence: 0.82
