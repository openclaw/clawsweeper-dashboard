## What This Changes

This PR adds OpenClaw session, attempt, fallback, turn-index, and turn-id headers to outbound LLM requests and exposes fallback status on LLM output hook events.

## Best Possible Solution

Define one documented opt-in or trusted-target LLM request-correlation contract with bounded identifiers, run-stable model-call IDs, and fallback state propagated from the actual fallback runner, coordinated with the open tracing and actual-model metadata work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a current-main bug, but the blocking PR defects are source-reproducible from the diff and current retry/fallback paths.

## Solution Assessment

Is this the best way to solve the issue? No. The request-correlation direction is useful, but the implementation should first establish an opt-in privacy boundary, use retry-stable call IDs, carry fallback state from the actual fallback runner, and rebase onto current main.
