## Summary

Keep open, but do not merge as-is. The PR targets a real observability need, yet the diff hard-codes internal context headers onto outbound model requests and still has source-reproducible fallback and turn-ID correctness problems; the related #69463 is a safer opt-in direction for part of the same tracing problem.

## What This Changes

The PR adds outbound LLM request headers for session, run, fallback, and turn context, and exposes fallback status on LLM output plugin events.

## Best Possible Solution

Converge on an opt-in, configurable observability-header contract, then add any accepted turn/fallback metadata with run-scoped counters, correct fallback state, tests, docs, and changelog coverage.

## Reproduction Assessment

Yes for the review blockers by source inspection. Current model-fallback code can skip candidates before invoking the run callback, and the PR's turn counter is scoped inside a retryable runEmbeddedAttempt.

## Solution Assessment

No. Hard-coding provider-visible headers and per-attempt turn counters is not the narrowest maintainable solution; an opt-in wrapper with run-scoped turn identity and fallback state from ModelFallbackRunOptions is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate context headers behind an explicit opt-in:** `src/agents/pi-embedded-runner/run/attempt.ts:2184-2194`
  - body: This merges session/run/fallback IDs into model.headers for every embedded run. Those headers are forwarded by the provider transports, so users who did not enable gateway tracing will send internal identifiers to all model providers; make this opt-in/configurable and document the privacy boundary.
  - confidence: 0.9
- **[P2] Propagate the actual fallback state into CLI attempts:** `src/agents/agent-command.ts:523`
  - body: This still derives isFallback from the number of run callback invocations. runWithModelFallback can skip a cooldowned primary before invoking run, so the first executed fallback is labeled primary even though model-fallback now passes options.isFallback; use that option for the new header/hook metadata.
  - confidence: 0.92
- **[P2] Keep turn IDs unique across retry attempts:** `src/agents/pi-embedded-runner/run/attempt.ts:2463-2469`
  - body: llmTurnCount resets for each runEmbeddedAttempt and the baseline is the assistant-message count. Retry paths can recreate runEmbeddedAttempt under the same runId before any assistant message is persisted, so multiple provider requests can emit the same X-OpenClaw-Turn-Id.
  - confidence: 0.88
