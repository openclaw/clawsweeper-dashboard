## Summary

Keep open for maintainer review. The feature direction is useful, but this patch should not merge as-is because it exports stable OpenClaw identifiers to provider requests without an opt-in/trusted-target contract, still has fallback and retry identity correctness problems, and lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds outbound X-OpenClaw session/run/fallback/turn headers to embedded LLM requests, propagates fallback state through model fallback callers, and extends the plugin LLM output event.

## Best Possible Solution

Define one privacy-safe LLM request-correlation contract that is opt-in or trusted-target scoped, uses bounded identifiers, derives fallback state from selected candidates, and assigns unique model-call IDs across retries.

## Reproduction Assessment

Yes for the blocking PR issues: source inspection shows the new headers merge into model headers, fallback candidates can be skipped before callback execution, and the proposed turn counter is scoped to one embedded attempt. This is a feature PR, so there is no current-main bug to reproduce.

## Solution Assessment

No. The requested observability direction is useful, but this patch is not the best solution until maintainers define the privacy/API contract and the implementation fixes fallback and per-request identity semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Gate context headers to trusted targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The gatewayContextHeaders are merged into model headers for every embedded model request, so external provider transports can receive stable OpenClaw session and run identifiers without a trusted gateway or explicit opt-in. Gate this behind a trusted target or operator opt-in before crossing the provider boundary.
  - confidence: 0.88
- **[P2] Propagate fallback options to agent attempts:** `src/agents/agent-command.ts:520`
  - body: This forwards isFallback from the callback invocation count, but runWithModelFallback can skip the primary candidate before calling run. In that case runOptions.isFallback is true while fallbackAttemptIndex is still 0, so the first executed fallback is labeled primary in the header and hook metadata.
  - confidence: 0.87
- **[P2] Keep turn IDs unique across retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: llmTurnCount resets for each runEmbeddedAttempt and the ID is seeded from persisted assistant messages. Retry paths that restart before an assistant message is written can send another HTTP request with the same runId and turn id, conflating gateway traces or billing rows.
  - confidence: 0.82
