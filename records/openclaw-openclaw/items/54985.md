## Summary

Keep this PR open but do not merge it as-is: current main still does not implement the requested LLM request header surface, and the proposed patch has unresolved privacy and metadata-correctness blockers that overlap active observability work.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and per-turn LLM request headers, propagates fallback state through embedded runner call paths, and exposes fallback status on plugin LLM output events.

## Best Possible Solution

Adopt one privacy-safe observability contract that scopes custom request headers to trusted or explicitly enabled targets, uses non-sensitive run/model-call identifiers, derives fallback state from selected candidates, keeps turn IDs unique across retries, and documents/tests the public API.

## Reproduction Assessment

Yes. The blockers are reproducible by static diff tracing: the patch unconditionally merges stable context headers into model headers, candidate skips can mislabel fallback status, and the per-turn counter can reset across embedded retries before an assistant message is persisted.

## Solution Assessment

No. The direction is useful, but this patch is not the safest implementation until the header trust boundary and metadata semantics are explicit; the opt-in direction in #69463 is a safer base to reconcile with this work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new `gatewayContextHeaders` are merged into `params.model.headers`, so ordinary provider transports can receive stable OpenClaw session and attempt identifiers. Make this opt-in or restrict it to trusted gateway/proxy targets before exposing those correlation IDs.
  - confidence: 0.86
- **[P2] Use fallback runner options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: This path still forwards `params.isFallbackRetry`, which only counts executed callbacks. If the primary candidate is skipped before invocation, the first executed fallback is labeled primary in the new header and hook metadata.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: `llmTurnCount` is local to one `runEmbeddedAttempt` and is seeded from persisted assistant messages. Retry paths can restart before an assistant message is written, so a second HTTP request in the same run can reuse the same `{runId}-turn-{index}` value.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch forwards stable OpenClaw session/run identifiers to outbound model-provider headers without an opt-in trusted-gateway boundary.
