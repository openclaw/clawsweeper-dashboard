## Summary

Keep open: current main does not implement the requested gateway context headers, and this PR still has blocking correctness and security issues around unscoped identifier headers, fallback semantics, and retry turn IDs. The observability contract should be reconciled with #69463 and #22021 before merge.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and turn context headers to embedded LLM requests and exposes fallback status on plugin LLM output events.

## Best Possible Solution

Adopt a maintainer-approved observability design that is opt-in or trusted-target scoped, derives fallback state from selected fallback candidates, maintains per-run unique model-call IDs across retries, and aligns with the related model visibility work.

## Reproduction Assessment

Yes. The blockers are source-reproducible from the provided PR diff and current main: new headers are merged into model headers, fallback status can ignore skipped candidates, and the turn counter is local to a single embedded attempt while retry loops can repeat before persisting an assistant message.

## Solution Assessment

No. The direction is useful, but this patch is not the safest implementation because the header trust boundary and metadata semantics are not explicit; #69463's opt-in direction is the safer base to reconcile with this work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new gatewayContextHeaders are merged into params.model.headers, so ordinary provider transports can receive stable OpenClaw session and attempt identifiers. Make this opt-in or restrict it to trusted gateway/proxy targets before exposing those correlation IDs.
  - confidence: 0.86
- **[P2] Use fallback options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: The model-fallback runner can skip a primary candidate before invoking the callback, but this path forwards params.isFallbackRetry instead of runOptions?.isFallback. In that case the first executed fallback request is labeled primary in the new header and hook metadata.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: llmTurnCount is local to one runEmbeddedAttempt and is seeded from persisted assistant messages. Retry paths can restart before an assistant message is written, so a second HTTP request in the same run can reuse the same turn id.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete privacy/supply-chain-adjacent concern by forwarding stable OpenClaw identifiers to outbound provider requests without an explicit trusted boundary.
