## Summary

Keep open: the PR targets a valid current-main reply bug, but the patch still has blocking source-level issues in the one-shot model path and is not ready to merge.

## What This Changes

The branch preserves mixed-message `/model` as a one-shot directive, threads a per-message auth profile into reply runs, updates slash-command docs, and adds regression coverage.

## Best Possible Solution

Keep the one-shot `/model <provider/model> <message>` semantics, but implement them through an explicitly non-persistent model/auth-profile selection path that suppresses stored session overrides and cannot write session state.

## Reproduction Assessment

Yes. Source inspection on current main shows the parser extracts `/model`, then `resolveReplyDirectives()` clears the model directive when normal message text remains, so the run stays on the session/default model. The PR-specific blockers are also source-reproducible by seeding stored model/auth-profile overrides and sending a cross-provider one-shot `/model` message.

## Solution Assessment

No. The requested behavior is the right fix direction, but this patch recomputes through the persistent session-selection helper and also calls `resolveContextTokens` with missing required fields. A dedicated non-persistent recompute path, plus regression coverage for preexisting session overrides, is safer.
