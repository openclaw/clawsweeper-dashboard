## Summary

Keep open: the reported mixed-message `/model` behavior is real and this PR targets the right reply/directive path, but the patch is not merge-ready because it can mutate persisted session/auth state, can compute defaults from the wrong model, lacks real behavior proof, and is currently conflicting.

## What This Changes

The branch preserves mixed-message `/model` as one-shot directive state, applies that model during reply preparation, threads a per-message auth profile into run preparation, updates slash-command docs, and adds directive regression tests.

## Best Possible Solution

Carry the one-shot provider/model/auth-profile as request-local reply state through the run, bypass persisted session override cleanup during that turn, add regression coverage for stored model/profile overrides, and document/prove the behavior with real after-fix output.

## Reproduction Assessment

Yes at source level. Current main strips mixed `/model ... <message>` text and clears the model directive before model selection, while the PR diff shows the attempted one-shot path and its state-isolation defects.

## Solution Assessment

No. The target behavior is maintainable, but this implementation rebuilds model selection through live persisted session state instead of keeping the one-shot selection fully request-local.
