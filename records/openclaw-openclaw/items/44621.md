## Summary

Keep open. Current main still strips mixed-message `/model ... <message>` without applying the selected model, but this PR is dirty, lacks real behavior proof, and introduces blocking session/auth-provider state hazards.

## What This Changes

The PR preserves mixed-message `/model` selections as one-shot reply state, threads per-message auth-profile selection into runs, updates slash-command docs, and adds regression coverage.

## Best Possible Solution

Rebase or replace the branch with request-local one-shot model/auth-profile state that uses current one-turn isolation primitives, preserves stored session defaults, adds stored-override regressions, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main parses and strips mixed `/model ... <message>`, then `clearInlineDirectives()` removes the model directive before reply model selection can apply it; I did not run a live chat reproduction in this read-only review.

## Solution Assessment

No. The direction is right, but the proposed patch rebuilds model selection through live stored session state after the one-shot switch, so stored model and auth-profile state can affect or be mutated by a non-persistent override.
