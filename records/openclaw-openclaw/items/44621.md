## What This Changes

The PR adds one-shot mixed-message `/model <provider/model> <body>` handling, per-message auth-profile threading, slash-command docs, and regression tests for non-persistent model selection.

## Best Possible Solution

Rebase or replace the branch with request-local one-shot model/auth-profile state using current one-turn isolation primitives, preserving stored session defaults and adding stored-override regression coverage.

## Reproduction Assessment

Yes, at source level. Current main clears mixed `/model ... <message>` directive state once normal body text remains, so model selection never sees the requested provider/model for the current run.

## Solution Assessment

No. The behavior target is right, but this branch should use request-local one-turn state instead of rebuilding model selection through persisted session state after the one-shot switch.
