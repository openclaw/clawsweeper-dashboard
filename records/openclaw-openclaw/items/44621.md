## Summary

Keep open: this PR targets a real current-main `/model` bug, but the proposed diff still has blocking session-state regressions and lacks required after-fix real behavior proof.

## What This Changes

The PR preserves mixed-message `/model` directives as one-shot provider/model/auth-profile state, threads them through reply execution, updates slash-command docs, and adds regression tests.

## Best Possible Solution

Keep the one-shot `/model` behavior, but carry the selected model/auth profile as non-persistent per-message runtime state and cover sessions with existing model and auth-profile overrides.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main strips mixed `/model ... <message>` text, then clears `hasModelDirective`, so model selection runs with the existing/default model.

## Solution Assessment

No. The direction is right, but this patch rebuilds model selection against live session state after the one-shot switch, which can mutate persistent auth-profile state and derive defaults from the wrong model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve session auth profiles during one-shot refresh:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: This refresh runs after the one-shot `/model` changes `provider`/`model`, but it still passes the live session entry and store. `createModelSelectionState` clears incompatible `authProfileOverride` values, so a one-shot switch to another provider can permanently delete the session's normal auth profile even though the path is meant to be non-persistent.
  - confidence: 0.86
- **[P2] Compute defaults from the one-shot model:** `src/auto-reply/reply/get-reply-directives.ts:478-480`
  - body: The refreshed model state is created with `hasModelDirective: false` while still receiving the session override inputs. If the session already has a stored `/model` override, that stored model can be reapplied, so thinking/reasoning defaults come from the session model instead of the one-shot model used for the run.
  - confidence: 0.9
