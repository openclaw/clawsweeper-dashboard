## Summary

Keep open: current main and the latest release still lack the requested one-shot inline `/model` behavior, and this PR is the canonical implementation candidate, but the branch is conflicting, lacks real behavior proof, and has session/auth correctness blockers.

## What This Changes

The PR preserves mixed-message `/model <provider/model> <message>` as a non-persistent one-shot reply override, threads optional per-message auth-profile state, and updates slash-command docs and regression tests.

## Best Possible Solution

Rebase or replace the branch with isolated one-shot model/auth-profile run state that skips stored-session reads and writes, preserves follow-up session defaults, adds stored-override regressions, and includes redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows mixed `/model ... <message>` is parsed and stripped, then `clearInlineDirectives()` removes the model directive before model selection can apply it; I did not run a live chat reproduction in this read-only review.

## Solution Assessment

No. The direction is right, but the proposed patch rebuilds model selection through live stored session state after the one-shot switch, so stored model and auth-profile state can affect or be mutated by a non-persistent override.
