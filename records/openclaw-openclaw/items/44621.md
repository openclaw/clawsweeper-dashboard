## Summary

Keep open. The underlying mixed-message `/model` bug is still present on current main, and this PR takes the right product direction, but the proposed patch is not merge-ready because its one-shot recompute path can reapply stored session model state and mutate stored auth-profile state.

## What This Changes

The PR preserves mixed-message `/model` as one-shot directive state, applies the selected provider/model/auth profile to the current reply run, updates slash-command docs, and adds regression coverage for non-persistent follow-up behavior.

## Best Possible Solution

Keep the one-shot `/model <provider/model> <message>` semantics, but implement the runtime switch through an explicitly non-persistent selection path that suppresses stored model/auth-profile overrides and cannot write session state. Add regressions for sessions that already have a persistent model override and persistent auth-profile override, including a cross-provider one-shot turn followed by a normal follow-up turn.

## Reproduction Assessment

Yes. On current main, an authorized mixed message like `/model openai/gpt-4.1-mini please sync now` is parsed and stripped, but `resolveReplyDirectives()` clears the model directive before `createModelSelectionState()`, so the run stays on the current/default model. The PR regressions are reproducible by first seeding a stored session model or auth-profile override, then sending a cross-provider one-shot `/model` message.

## Solution Assessment

No. The requested behavior is the right direction, but this patch reuses a persistent session-selection helper for one-shot recomputation, which can reapply stored model overrides and clear stored auth profiles. A dedicated non-persistent recompute path or explicit suppress-stored-overrides parameter would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve session auth profile during one-shot recompute:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: After a mixed-message `/model` changes `provider`/`model`, this rebuild passes the live `sessionEntry` and `sessionStore` back into `createModelSelectionState`. That helper clears `sessionEntry.authProfileOverride` when the stored profile belongs to the old provider, so a one-shot cross-provider message can permanently delete the user's persistent auth-profile override even though this syntax is meant to be non-persistent.
  - confidence: 0.88
- **[P2] Compute defaults from the one-shot model:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: This recompute uses `hasModelDirective: false` while still passing session state, so `createModelSelectionState` can reapply a stored session model override before `resolveDefaultThinkingLevel()` and `resolveDefaultReasoningLevel()` run. The agent run uses the one-shot provider/model, but thinking or reasoning defaults can come from the old session model; suppress stored overrides for this one-shot refresh or avoid the persistent session path.
  - confidence: 0.9
