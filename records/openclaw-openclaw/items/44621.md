## Summary

Keep this PR open, but it should not merge as-is: current main still lacks the requested one-shot mixed-message `/model` behavior, while the PR head has blocking correctness defects and no after-fix real behavior proof.

## What This Changes

The branch threads a one-shot `/model <provider/model> <message>` directive through reply parsing, model selection, auth-profile propagation, docs, and regression coverage.

## Best Possible Solution

Implement one-shot `/model` through an explicit non-persistent selection path that strips directive text, applies provider/model/auth profile only to the current run, recomputes context and defaults from that selected model, and leaves stored session overrides untouched.

## Reproduction Assessment

Yes. Current main is source-reproducible: it parses `/model`, then clears mixed-message directive state before model selection, so the run stays on the current session/default model while directive text is stripped.

## Solution Assessment

No. The requested direction is right, but this patch calls `resolveContextTokens` without required inputs and recomputes through persistent session-selection state; a dedicated non-persistent path is safer.
