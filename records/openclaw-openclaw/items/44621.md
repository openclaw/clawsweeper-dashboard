## Summary

Keep open: current main still clears mixed-message `/model` before model selection, and this PR is not merge-ready because the branch can touch persistent session state during a one-shot refresh and lacks real behavior proof.

## What This Changes

The PR adds one-shot mixed-message `/model <provider/model> <message>` handling through reply directives, auth-profile propagation, docs, and regression tests.

## Best Possible Solution

Carry the selected model and auth profile as isolated, non-persistent per-message runtime state, recompute defaults from that state, and cover existing session model/auth-profile overrides with regressions.

## Reproduction Assessment

Yes. Source inspection on current main shows mixed `/model ... <message>` is parsed and stripped, then the directive state is cleared before model selection can apply it.

## Solution Assessment

No. The direction is right, but the current PR refreshes model selection through live session storage after the one-shot switch, so it needs branch fixes before it is the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve session auth profiles during one-shot refresh:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: After switching provider/model for the one-shot run, this refresh still passes `sessionEntry`, `sessionStore`, and `sessionKey` into `createModelSelectionState`. That helper clears incompatible `authProfileOverride`, so a mixed-message one-shot can permanently delete the session's stored auth profile even though the selection is supposed to be non-persistent.
  - confidence: 0.86
- **[P2] Compute defaults from the one-shot model:** `src/auto-reply/reply/get-reply-directives.ts:478-480`
  - body: The refreshed model state is created with `hasModelDirective: false` while still receiving session override inputs. If a stored `/model` override exists, thinking/reasoning defaults can come from the session model instead of the one-shot model used for this run.
  - confidence: 0.9
