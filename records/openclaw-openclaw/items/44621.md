## Summary

Keep open: current main still drops the mixed-message `/model` directive before model selection, and this PR is not ready because the patch has blocking session-state bugs plus no after-fix real behavior proof.

## What This Changes

The PR preserves mixed-message `/model` directives as per-message model/profile state, forwards that state into reply runs, updates slash-command docs, and adds directive regression tests.

## Best Possible Solution

Implement one-shot `/model <provider/model> <message>` through an explicitly non-persistent selection path that suppresses stored overrides, preserves persistent auth-profile state, recomputes context/defaults against the one-shot model, and adds regression coverage for preexisting session overrides.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model` is parsed and stripped, then directive state is cleared when normal message text remains, so the run stays on the session/default model.

## Solution Assessment

No. The requested direction is right, but this patch recomputes through persistent session-selection state and calls `resolveContextTokens` without required fields; a dedicated non-persistent path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass cfg and provider when recomputing context tokens:** `src/auto-reply/reply/get-reply-directives-apply.ts:274-277`
  - body: The new one-shot block calls `resolveContextTokens({ agentCfg, model })`, but current main's helper requires `cfg`, `agentCfg`, `provider`, and `model`. This is a typecheck blocker and cannot recompute context tokens for the selected provider.
  - confidence: 0.95
- **[P1] Preserve stored auth profiles during one-shot recompute:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: After the one-shot provider/model switch, this rebuild passes the live `sessionEntry` and `sessionStore` into `createModelSelectionState()`. That helper clears stored auth profiles on provider mismatch, so a cross-provider one-shot `/model` turn can permanently delete a persistent auth-profile override.
  - confidence: 0.88
- **[P2] Compute defaults from the one-shot model:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: This recompute uses `hasModelDirective: false` while still passing session state, so `createModelSelectionState()` can reapply a stored session model override before thinking/reasoning defaults are resolved. The run can use the one-shot model while defaults come from the old session model.
  - confidence: 0.9
