## Summary

Keep open: the PR targets a real current-main gap, but it is not merge-ready because it is conflicted, lacks real behavior proof, and introduces session/auth-provider isolation bugs in the one-shot model path.

## What This Changes

The PR preserves mixed-message `/model` as a one-shot provider/model/auth-profile override, threads it into reply preparation without persisting session overrides, updates slash-command docs, and adds regression tests.

## Best Possible Solution

Keep this as the canonical discussion for now, but require a rebase and a request-local one-shot model/auth path that never reads or mutates persisted session overrides, plus redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main strips mixed `/model ... <message>` and then clears model directive state before model selection can apply it; I did not run a live repro in this read-only review.

## Solution Assessment

No. The target behavior is right, but this implementation rebuilds one-shot model state through persisted session state; the safer fix is a rebased request-local model/auth path with regression coverage for existing stored overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep one-shot model refresh out of session storage:** `src/auto-reply/reply/get-reply-directives.ts:469-470`
  - body: This rebuild runs after a one-shot `/model` switch but passes live `sessionEntry` and `sessionStore` into `createModelSelectionState`. That helper can clear `sessionEntry.authProfileOverride` when the one-shot provider differs, so a per-message override can permanently delete the user's persisted auth profile.
  - confidence: 0.88
- **[P2] Suppress stored overrides when recomputing one-shot defaults:** `src/auto-reply/reply/get-reply-directives.ts:478`
  - body: The second model-state rebuild uses `hasModelDirective: false`, so a stored session model override can replace the one-shot provider/model inside `modelState`. The run may use the requested model while thinking/reasoning defaults are computed from the old session model.
  - confidence: 0.86
