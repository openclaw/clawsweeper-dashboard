## Summary

Keep open. Current `main` still drops mixed-message `/model ... <message>` before model selection, so the PR targets a real bug, but this branch is not merge-ready because its one-shot refresh can read or mutate persisted session/auth state, the branch is conflicting, and contributor real behavior proof is still missing.

## What This Changes

The PR adds non-persistent one-shot handling for mixed-message `/model <provider/model> <message>` replies, threads per-message auth-profile state into reply runs, updates slash-command docs, and adds regression tests.

## Best Possible Solution

Rebase or replace the branch with request-local one-shot model/auth-profile state that uses current one-turn isolation primitives, preserves stored session defaults, adds stored-override regressions, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current `main` parses mixed `/model ... <message>` and strips the directive text, then `clearInlineDirectives()` removes the model directive before model selection can apply it; I did not run a live chat reproduction in this read-only review.

## Solution Assessment

No. The direction is right, but the proposed patch rebuilds model selection through live stored session state after the one-shot switch, so stored model and auth-profile state can affect or be mutated by a non-persistent override.
