Full review comments:

- **[P1] Preserve stored auth profiles during one-shot refresh:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: After applying the one-shot model, this rebuild passes the live `sessionEntry`, `sessionStore`, and `storePath` back into `createModelSelectionState` with `hasModelDirective: false` and without a one-turn or skip-stored flag. That callee validates stored auth profiles against the temporary provider and can clear an incompatible `authProfileOverride`, so a single mixed-message `/model` can permanently remove a user's stored auth-profile setting.
  - confidence: 0.91
- **[P2] Compute defaults from the one-shot model:** `src/auto-reply/reply/get-reply-directives.ts:478-480`
  - body: The same rebuild can re-apply a stored session model override inside `modelState` because stored overrides are not skipped. The run uses the one-shot provider/model from `applyResult`, but `resolvedThinkLevelWithDefault` and default reasoning are then resolved from `modelState`, so defaults can come from the old session model instead of the selected one-shot model.
  - confidence: 0.88
