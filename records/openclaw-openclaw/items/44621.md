## Summary

Keep open: current main still has the source-reproducible mixed-message `/model` bug, and this PR remains the canonical implementation path, but the branch has blocking correctness issues plus missing real behavior proof before merge.

## What This Changes

The PR adds one-shot inline `/model <provider/model> <message>` handling across reply directive parsing, model selection, auth-profile propagation, docs, and regression coverage.

## Best Possible Solution

Implement a dedicated non-persistent `/model ... <body>` selection path that strips the directive, applies provider/model/auth profile only to the current run, recomputes context/defaults from that selected runtime, and preserves stored session overrides.

## Reproduction Assessment

Yes. Current main is source-reproducible: it parses `/model`, then clears mixed-message directive state before model selection, so the run stays on the current session/default model while directive text is stripped.

## Solution Assessment

No. The requested direction is right, but this patch calls the context helper with missing inputs and recomputes through persistent session-selection state; a dedicated non-persistent selection path is safer.

  - confidence: 0.9

  - confidence: 0.9
