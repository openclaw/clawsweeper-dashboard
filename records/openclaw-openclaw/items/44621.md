## Summary

Keep open: current main still has the mixed-message `/model` bug and this PR is the canonical linked fix, but the branch has blocking correctness issues and lacks external real behavior proof before merge.

## What This Changes

The PR adds one-shot mixed-message `/model <provider/model> <message>` handling across reply directives, per-message auth-profile propagation, slash-command docs, and regression tests.

## Best Possible Solution

Implement one-shot `/model` through an explicitly non-persistent selection path that strips directive text, suppresses stored model/auth-profile mutation, recomputes defaults and context from the selected provider/model, and covers preexisting session override cases.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model` is parsed and stripped, then mixed-message directive state is cleared before model selection, so the run stays on the session/default model.

## Solution Assessment

No. The requested direction is right, but this patch recomputes through persistent session-selection state and calls `resolveContextTokens` without required fields; a dedicated non-persistent path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass all context-token inputs for the one-shot model:** `src/auto-reply/reply/get-reply-directives-apply.ts:274-277`
  - body: The new one-shot block calls `resolveContextTokens({ agentCfg, model })`, but the helper requires `cfg`, `agentCfg`, `provider`, and `model`. This is a typecheck blocker and cannot resolve provider-specific context tokens for the selected model.
  - confidence: 0.95
- **[P1] Avoid mutating session auth profiles during one-shot recompute:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: The post-switch `createModelSelectionState()` call is given the live `sessionEntry`, `sessionStore`, and `sessionKey`. For a cross-provider one-shot model, that helper can clear the persistent session auth-profile override while validating it against the temporary provider, violating the non-persistent one-shot contract.
  - confidence: 0.9
- **[P2] Suppress stored model overrides for one-shot defaults:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: The same rebuild uses `hasModelDirective: false` while still passing session state, so a stored session model override can replace the rebuilt `modelState` before default thinking/reasoning are resolved. The run can use the one-shot model while defaults come from the old session model.
  - confidence: 0.9
