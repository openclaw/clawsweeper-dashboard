## Summary

Keep open: current main still drops mixed-message `/model` before model selection, but this PR head has concrete correctness blockers around context-token recomputation and non-persistent session/auth-profile handling.

## What This Changes

The PR preserves mixed-message `/model` selection as per-message state, threads an optional auth-profile override into reply execution, updates slash-command docs, and adds regression coverage.

## Best Possible Solution

Keep the one-shot `/model <provider/model> <message>` behavior, but implement it through an explicitly non-persistent selection path that recomputes context/defaults from the effective model without passing mutable session state.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model` is parsed and stripped, then `resolveReplyDirectives()` clears the model directive when normal text remains, leaving the run on the session/default model.

## Solution Assessment

No. The PR heads in the right direction, but the current patch calls `resolveContextTokens` with missing required fields and reuses the persistent model-selection helper in a way that can reapply stored model overrides or delete stored auth-profile overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass full context-token inputs:** `src/auto-reply/reply/get-reply-directives-apply.ts:274-277`
  - body: The one-shot branch calls `resolveContextTokens({ agentCfg, model })`, but the helper requires `cfg`, `agentCfg`, `provider`, and `model`. This will fail typechecking and would also drop the selected provider from context-window lookup.
  - confidence: 0.95
- **[P1] Keep one-shot recompute from clearing auth profiles:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: After the one-shot provider/model switch, this rebuild passes the live `sessionEntry`, `sessionStore`, and `storePath` into `createModelSelectionState()`. That helper clears `sessionEntry.authProfileOverride` on provider mismatch, so a cross-provider one-shot can permanently remove the user's persistent auth profile.
  - confidence: 0.9
- **[P2] Suppress stored model overrides during one-shot recompute:** `src/auto-reply/reply/get-reply-directives.ts:466-480`
  - body: This recompute uses `hasModelDirective: false` while still passing session state, so `createModelSelectionState()` can reapply a stored session model override before thinking/reasoning defaults are resolved. The run can use the one-shot model while defaults come from the old session model.
  - confidence: 0.9

  - body: This is a user-facing reply behavior fix with docs changes, but the PR does not update `CHANGELOG.md`. Add a single-line Unreleased `### Fixes` entry before merge.
