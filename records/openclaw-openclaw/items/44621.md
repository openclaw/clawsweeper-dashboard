## Summary

Keep this PR open because current main still has the inline `/model` one-shot bug and this is the canonical fix path, but the submitted patch has blocking session-state defects, merge conflicts, and no real behavior proof beyond tests.

## What This Changes

The branch preserves mixed-message `/model` as a one-shot directive, threads per-message model/auth-profile selection through reply preparation, updates slash-command docs, and adds regression tests.

## Best Possible Solution

Rebase the branch onto current main and implement the one-shot model path so temporary model/default calculation cannot read or mutate persistent session model or auth-profile overrides.

## Reproduction Assessment

Yes, source inspection gives a high-confidence current-main reproduction path: mixed-message `/model ... <body>` is parsed and stripped, then `clearInlineDirectives()` drops the model directive before model selection runs. I did not execute the path because this review is read-only and tests would write artifacts.

## Solution Assessment

No, not as submitted. The direction is right, but the patch rebuilds model state through persistent session objects, which can mutate auth-profile overrides and compute defaults from a stored session model instead of the one-shot model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not mutate session auth profiles during one-shot refresh:** `src/auto-reply/reply/get-reply-directives.ts:470`
  - body: This one-shot refresh passes the live session entry/store back into `createModelSelectionState()`. When the temporary model uses a different provider, that helper can clear `sessionEntry.authProfileOverride` in storage, so `/model other-provider/model <body>` permanently deletes the session's normal auth profile even though the feature is documented as non-persistent.
  - confidence: 0.93
- **[P2] Suppress stored overrides when recomputing one-shot defaults:** `src/auto-reply/reply/get-reply-directives.ts:486`
  - body: The rebuilt `modelState` is created with `hasModelDirective: false` while still receiving the session store, so a previous persistent `/model` override is reapplied internally. The agent run keeps the one-shot provider/model, but thinking and reasoning defaults can be derived from the old session model; rebuild the defaults from the one-shot selection without consulting stored overrides and add coverage for that case.
  - confidence: 0.91
