## Summary

Close as ClawHub: this is an optional browser/provider integration that current native plugin surfaces can host outside core, and the repo docs explicitly route optional plugin promotion and discovery through ClawHub. The issue does not identify a missing generic OpenClaw SDK seam that needs to remain tracked in the core repository.

## What This Changes

The issue proposes a new web-voice provider plugin for ChatGPT web, Gemini web, Grok web, Perplexity web, claude.ai web, Mac app surfaces, shared workstation leasing, and selector-version management.

## Best Possible Solution

Build this as an external native plugin published through ClawHub, using the existing provider/realtime-voice/browser/tool/service SDK surfaces. If implementation exposes a missing generic capability, open a narrower SDK issue for that seam rather than keeping this broad integration RFC open in core.

## Reproduction Assessment

Not applicable: this is an RFC/feature request, not a bug report. The high-confidence check is source and docs inspection showing that current main already has the plugin distribution and capability contracts needed for an external implementation.

## Solution Assessment

Yes. The ClawHub/plugin path is the narrowest maintainable solution because the requested behavior is an optional, high-power integration and the current core contracts already expose provider, realtime voice, browser, service, CLI, state, and locking hooks.

## Security Review

Status: needs_attention

Summary: The proposal itself is security-sensitive because it would automate logged-in browser sessions with AppleScript/JavaScript injection; no patch was reviewed.

Concerns:

- **[medium] Keep logged-in browser injection opt-in:**
  - body: Driving a user's existing Chrome session and injecting JavaScript can expose account state, cookies, and site actions. Any implementation should remain an explicit opt-in plugin with ClawHub provenance/scanning and clear operator consent, not bundled default core behavior.
  - confidence: 0.86
