## Summary

Close as ClawHub: the request is for an optional, high-power browser/provider integration that current native plugin and provider surfaces can host outside core. Current main documents the ClawHub distribution path and exposes the needed provider, realtime voice, browser-session, service, CLI, HTTP/RPC, and locking helper seams; the issue does not identify a missing generic OpenClaw SDK capability that needs to stay tracked in core.

## What This Changes

Requests a versioned `@openclaw/plugin-web-voices` provider plugin that wraps ChatGPT/Gemini/Grok/Perplexity/Claude web and Mac surfaces behind stable web-voice contracts plus shared browser lease and selector-registry infrastructure.

## Best Possible Solution

Build this as an external native plugin distributed through ClawHub, using the existing provider/realtime voice/browser/service/CLI and SDK helper surfaces. If implementation proves that a generic capability is missing, open a narrower SDK issue for that seam rather than keeping this broad integration RFC in core.

## Reproduction Assessment

Not applicable: this is an RFC/feature request, not a bug report. The high-confidence check is source and docs inspection showing that current main already has the plugin distribution and capability contracts needed for an external implementation.

## Solution Assessment

Yes. The ClawHub/plugin path is the narrowest maintainable direction because the requested behavior is optional and security-sensitive, while current core already exposes the relevant extension contracts without adding bundled vendor-specific web-driver code.
