## What This Changes

The issue reports a shipped regression where the selected lossless-claw context engine is invoked during read-only plugin registration, quarantined, and replaced by the legacy engine.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96357, or an equivalent narrow fix that keeps read-only discovery metadata-only while allowing full runtime activation to replace it and resolve the selected context engine.

## Reproduction Assessment

Yes, at source level: current main registers context-engine factories during read-only discovery-capable modes and `resolveContextEngine()` invokes and quarantines selected factory failures. I did not run a live macOS gateway reproduction with lossless-claw.

## Solution Assessment

Yes, the best direction is the narrow lifecycle separation in the linked PR: keep plugin SDK/config behavior unchanged, treat read-only discovery registrations as metadata-only, and let runtime activation provide the factory used for real resolution.
