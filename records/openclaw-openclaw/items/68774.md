## What This Changes

The branch broadens memory-core dreaming contamination detection, adds rehydration-time rejection for candidates inside managed dreaming fences, and adds short-term promotion tests for inline metadata snippets and fenced ranges.

## Best Possible Solution

Land a narrow memory-core fix that blocks staged dreaming artifacts at both detection and rehydration time, after correcting the fence-overlap helper so marker lines and nested inline markers are treated as fenced ranges.

## Reproduction Assessment

Yes. Current main can be inspected at the exact detector and rehydration paths, and the PR helper issue can be reproduced with a unit case where the requested range points to a line inside an outer fence that also contains an inline `openclaw:dreaming:*:start` marker.

## Solution Assessment

No, not as currently written. The direction is the right narrow fix, but the new overlap helper should check the requested line range before consuming marker lines or otherwise classify marker lines themselves as fenced.
