## Summary

Keep open: current main still has the reported memory-core leak, and this PR is on the right boundary but leaves a fence-overlap edge case where marker-line ranges can still pass through.

## What This Changes

This PR broadens memory-core dreaming contamination detection, rejects rehydrated promotion candidates that overlap dreaming fences, adds short-term promotion tests, and adds a changelog entry.

## Best Possible Solution

Land the focused memory-core guard after updating the fence helper to treat marker lines and same-line marker content as fenced, while keeping broader distillation or promotion-target changes in separate items.

## Reproduction Assessment

Yes, source-reproducible. Current main misses inline metadata before `Candidate:` because the detector requires the marker at the post-prefix start, and the apply path rehydrates live daily-note ranges without a dreaming-fence check.

## Solution Assessment

Mostly yes, but not quite in the current patch. The requested boundary is narrow and maintainable, but the new fence helper should count start/end marker lines and same-line marker content as overlapping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Count dreaming fence marker lines as fenced:** `extensions/memory-core/src/short-term-promotion.ts:1510-1512`
  - body: The new helper sets `insideFence` and immediately `continue`s on the start marker before checking whether the current line falls inside the candidate range. A candidate relocated to the marker line itself, or to a line that contains both the marker and scratch content, returns `false` and can still be appended if it does not match the five-signal text detector. Check the overlap for marker lines too, or otherwise treat any ranged line containing a dreaming start/end marker as fenced.
  - confidence: 0.87
