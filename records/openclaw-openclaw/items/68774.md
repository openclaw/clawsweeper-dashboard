## Summary

Keep open: current main still has the memory-core dreaming promotion leak, and this PR is the active focused fix. Source review found one concrete edge case in the new fence helper that can still let a marker-line dreaming artifact through before merge.

## What This Changes

The PR broadens memory-core dreaming contamination detection, rejects rehydrated ranges inside `openclaw:dreaming` fences, adds focused regression tests, and records a changelog entry.

## Best Possible Solution

Land this focused memory-core guard after treating fence marker lines as overlapping, while leaving broader distillation, search-index, or configurable promotion-target proposals to their separate tracking items.

## Reproduction Assessment

Yes from source inspection. Current main misses inline metadata before `Candidate:` and has no rehydration-time dreaming-fence check; the PR-specific fence-marker edge is also source-reproducible by pointing a candidate range at a line containing an `openclaw:dreaming:*:start` marker.

## Solution Assessment

Not quite yet. The PR fixes the right boundary with no new config or API surface, but the new fence helper should also reject start/end marker lines so malformed inline managed blocks cannot bypass the guard.
