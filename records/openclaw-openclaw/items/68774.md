## Summary

Keep this PR open: it targets a real memory-core bug that is still present on current main, but the new fence-overlap helper has a blocking false-negative for marker-line and nested-marker ranges. The inline metadata detector remains useful, and the rehydration guard should be fixed or reconciled with https://github.com/openclaw/openclaw/pull/80702 before merge.

## What This Changes

The PR broadens memory-core dreaming contamination detection and adds a rehydration-time dreaming-fence overlap guard with focused regression tests.

## Best Possible Solution

Keep the inline-metadata detector, then fix the rehydration guard to count marker lines as overlapping or fold it into the managed-block redaction approach from https://github.com/openclaw/openclaw/pull/80702 before landing.

## Reproduction Assessment

Yes. Current main's strict post-prefix lead check and unguarded rehydration path are visible in source, and the PR thread includes production evidence of staged dreaming rows leaking into MEMORY.md.

## Solution Assessment

No, not as currently written. The detector broadening is a narrow maintainable fix, but the new fence helper must treat marker lines and nested marker lines as overlapping or use the stronger managed-block redaction path before merge.

Full review comments:

- **[P2] Count fence marker lines as overlapping:** `extensions/memory-core/src/short-term-promotion.ts:1520-1522`
  - body: When the relocated range is the dreaming marker line itself, or a nested marker line inside an outer dreaming block, this branch sets `insideFence` and `continue`s before checking whether that line is inside `safeStart..safeEnd`. Please count marker lines in the selected range as overlapping and add a regression for the one-line or nested-marker case.
