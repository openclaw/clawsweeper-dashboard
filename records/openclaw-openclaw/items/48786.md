## What This Changes

Resolve Feishu quoted/replied message context so fetched `@_user_N` mention placeholders render as normalized mention tags or readable names.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-only patch that models fetched mentions, adapts the SDK shape, normalizes quoted/root/thread content, and adds focused `send.test.ts` coverage.

## Solution Assessment

Yes, the maintainable fix belongs in the Feishu fetched-message parser, reusing the existing normalization semantics after adapting the SDK response shape. The current linked PR should be narrowed because the core bug does not require unrelated gateway, command, package, or security changes.
