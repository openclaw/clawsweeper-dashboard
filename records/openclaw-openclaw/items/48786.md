## Summary

Keep open. Current main still ignores fetched Feishu `mentions` metadata when parsing quoted/root/thread message content, and an open closing PR targets the same bug, so this should remain open until that PR or a narrow equivalent lands.

## What This Changes

Resolve Feishu fetched quoted/replied message content so raw `@_user_N` placeholders are normalized into readable `<at>` mention tags using fetched mention metadata.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu plugin-only patch that models fetched `mentions`, adapts the SDK mention shape, normalizes fetched quoted/root/thread content, and adds focused `extensions/feishu/src/send.test.ts` regression coverage.

## Reproduction Assessment

Yes. A source-level repro is to mock Feishu `im.message.get` or thread-list response with `body.content` containing `@_user_1` plus item-level `mentions`; current `parseFeishuMessageItem()` ignores that metadata and feeds raw content into quoted/thread context.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the Feishu fetched-message parser, reusing the existing mention-normalization behavior through a fetched-shape adapter and focused regression tests; the current broad PR should be narrowed or replaced before landing.
