Keep open. Resolve Feishu mention placeholders in fetched quoted/replied message context so mentions render as readable tags instead of raw `@_user_N` text.

## Best Possible Solution

Land a narrow Feishu-only fix in the fetched-message parser path that adapts `im.message.get/list` mention entries, covers quoted and thread-history tests, and includes real Feishu quoted-message proof.

## Solution Assessment

Yes, the best fix is a narrow Feishu plugin repair in the shared fetched-message parser path. It should reuse the existing normalization behavior through a correct SDK-shape adapter rather than reviving the broad closed PR branch.
