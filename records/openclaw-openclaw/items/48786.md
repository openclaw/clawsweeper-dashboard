## Summary

Keep open. Current main and the latest release still parse Feishu fetched quoted/thread message content without using fetched-message mention metadata, and the issue is paired with an open closing PR that has not landed.

## What This Changes

Feishu quoted/replied message context should resolve raw `@_user_N` placeholders from fetched message bodies into normalized mention tags or readable mention text.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-plugin-only patch that models fetched mentions, adapts the SDK shape, normalizes quoted/root/thread content, and adds focused `extensions/feishu/src/send.test.ts` regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level fetched `mentions`; current main parses and returns the raw placeholder content without using that metadata.

## Solution Assessment

Yes, the maintainable fix belongs in the Feishu fetched-message parser, reusing existing mention-normalization semantics after adapting the fetched SDK response shape. The linked PR should be narrowed because the core bug does not require unrelated gateway, command, package, or security changes.
