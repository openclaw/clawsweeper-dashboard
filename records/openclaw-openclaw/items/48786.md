## Summary

Keep open. Current main still ignores Feishu fetched-message mention metadata for quoted/root/thread context, and an open closing PR already targets this bug, so the issue should stay open until that PR or an equivalent narrow fix is landed or closed.

## What This Changes

Resolve Feishu quoted/replied message content so fetched mention placeholders are rendered as readable mention tags instead of raw @_user_N keys.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-plugin-only patch that adapts fetched {key,id,id_type,name} mentions before normalizing quoted/root/thread content and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu im.message.get or im.message.list with body.content containing @_user_1 plus item-level mentions; current parseFeishuMessageItem ignores that metadata and returns parsed raw content.

## Solution Assessment

Mostly yes. The maintainable fix belongs in the Feishu fetched-message parser, but it should adapt the fetched SDK mention shape before reusing the existing normalizeMentions behavior and should avoid unrelated core changes.
