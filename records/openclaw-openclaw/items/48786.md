## Summary

Keep open. Current main still ignores Feishu fetched-message mention metadata for quoted/root/thread context, and an open implementation PR already targets the same bug.

## What This Changes

Resolve Feishu fetched quoted/replied message mention placeholders such as `@_user_N` into readable `<at user_id="...">name</at>` tags.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976 or land an equivalent Feishu-plugin-only patch that models fetched `mentions`, adapts the fetched shape, normalizes quoted/root/thread content, and adds focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current `parseFeishuMessageItem()` ignores that metadata and returns parsed raw content.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu fetched-message parser, reusing the existing mention-normalization behavior through a fetched-shape adapter and focused tests.
