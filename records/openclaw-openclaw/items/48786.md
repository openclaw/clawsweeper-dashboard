## Summary

Keep open. Current main and the latest release still parse fetched Feishu quoted/thread message content without using fetched-message mention metadata, and the only same-repo fix PR was closed unmerged while the remaining fork PR is open but conflicting.

## What This Changes

The issue asks Feishu quoted/replied message context to resolve raw `@_user_N` placeholders into readable `<at user_id="...">name</at>` mention tags like the live inbound path already does.

## Best Possible Solution

Land or sponsor a narrow Feishu-plugin-only fix that models fetched-message `mentions`, adapts them to the existing normalization helper, normalizes quoted/root/thread fetched content, and adds focused `send.test.ts` coverage.

## Reproduction Assessment

Yes, at source level: current main parses fetched Feishu quoted/thread message content without reading `item.mentions`, while callers inject that fetched content as quoted or thread context. I did not run a live Feishu chat reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Feishu parser repair that reuses the existing mention-normalization behavior after adapting the fetched SDK mention shape. The closed broad same-repo PR is not the right landing vehicle.
