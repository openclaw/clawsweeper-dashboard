## What This Changes

Resolve Feishu fetched quoted/replied message mentions so raw `@_user_N` placeholders become readable `<at user_id="...">Name</at>` context instead of leaking into agent-visible quoted content.

## Best Possible Solution

Review and narrow the open linked PR, or land an equivalent Feishu-plugin-only patch that models fetched-message `mentions`, adapts the flat SDK mention shape, normalizes quoted/root/thread content, and adds focused regression coverage.

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level fetched `mentions`; current main parses and returns the raw placeholder without using that metadata.

## Solution Assessment

Yes for the issue's requested boundary: the fix belongs in the Feishu fetched-message parser and should reuse existing mention-normalization semantics after adapting the fetched SDK response shape. The open linked PR should be narrowed because the core bug does not require unrelated gateway, command, package, or security changes.
