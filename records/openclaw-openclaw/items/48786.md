## What This Changes

Resolve Feishu quoted/replied fetched message content so raw `@_user_N` mention placeholders are rendered as readable mention tags or names.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-plugin-only fix that models fetched mentions, normalizes quoted/root/thread fetched content, and adds focused regression coverage.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu fetched-message parser, reusing existing mention-normalization semantics after adapting the fetched SDK response shape.
