## Summary

Keep open. Current main still ignores Feishu fetched-message `mentions` metadata when parsing quoted/root/thread content, and an open closing PR targets the same bug.

## What This Changes

Resolve Feishu `@_user_N` placeholders in fetched quoted/replied message content using the API response's mention metadata.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-plugin-only patch that models fetched `mentions`, adapts the fetched `{id, id_type}` shape, normalizes quoted/root/thread content, and adds focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current `parseFeishuMessageItem()` ignores that metadata and returns parsed raw content.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu fetched-message parser, reusing existing mention-normalization behavior through a fetched-shape adapter rather than blindly casting the API response.
