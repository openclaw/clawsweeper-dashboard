## Summary

Keep open. Current main still drops Feishu fetched-message `mentions` metadata before quoted/thread context is rendered, and an open implementation PR with closing syntax already targets the same bug.

## What This Changes

The issue asks for Feishu quoted/replied message content to resolve raw `@_user_N` placeholders using fetched-message mention metadata.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976 or land an equivalent Feishu-plugin-only patch that models fetched `mentions`, adapts the fetched SDK shape, normalizes quoted/root/thread content, and adds focused `send.test.ts` regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current main ignores that metadata in `parseFeishuMessageItem()`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu fetched-message parser with a small adapter for fetched mention shape, rather than broad unrelated gateway or command changes.
