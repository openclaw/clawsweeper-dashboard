## Summary

Keep open. Current main and the latest release still parse Feishu fetched quoted/thread message content without using fetched `mentions`, and https://github.com/openclaw/openclaw/pull/48976 is still the open implementation candidate.

## What This Changes

The issue requests resolving raw Feishu `@_user_N` placeholders in fetched quoted/replied message content before that content reaches agent context.

## Best Possible Solution

Review and narrow https://github.com/openclaw/openclaw/pull/48976, or land an equivalent Feishu-plugin-only fix that adapts fetched-message mention metadata, normalizes quoted/root/thread content, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current main parses and returns the placeholder without using that metadata.

## Solution Assessment

Yes for the issue boundary. The fix belongs in the Feishu fetched-message parser and should reuse existing mention-normalization semantics after adapting the fetched SDK response shape; the linked PR should be reviewed or narrowed instead of queueing duplicate work.
