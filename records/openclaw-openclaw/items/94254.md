## What This Changes

The issue reports that Apple app-specific-password redaction over-masks ordinary four-part kebab-case identifiers in generic text/error fields and persists the corrupted transcript content.

## Best Possible Solution

Land one canonical redaction fix with focused regression coverage, preserving ordinary generic-field kebab identifiers while retaining fail-closed masking for explicit Apple/iCloud/app-specific-password credential fields.

## Solution Assessment

Yes. The right fix boundary is a narrow heuristic change in `src/logging/redact.ts` with focused redaction regression coverage; the open linked PRs are candidate implementations, so a new repair lane would duplicate active work.
