## Summary

## What This Changes

The PR adds a `probe` boolean to the doctor gateway memory-status call, forwards `doctor --deep` into that probe, and suppresses the local-provider warning when the gateway memory probe was intentionally skipped.

## Best Possible Solution

Land one canonical fix with local skipped-probe regression coverage, redacted real doctor CLI proof, and an explicit maintainer decision on whether `doctor --deep` should force the live embedding probe in the same change.

## Reproduction Assessment

Yes. From current-main source, a local provider with `gatewayMemoryProbe: { checked: false, ready: false, skipped: true }` reaches the local warning branch, while the sibling key-optional branch treats the same skipped state as neutral.

## Solution Assessment

Unclear as the best merge package. The skipped-probe guard is the right narrow fix for the false warning, but this branch also adds untested `doctor --deep` live-probe forwarding that maintainers may want to split or prove separately.
