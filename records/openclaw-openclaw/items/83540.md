## Summary

Keep open: this is a focused status correctness fix with supplied terminal proof and green PR checks, but the patch drops an existing transcript context-estimate fallback for telemetry-missing local sessions and needs a narrow repair before automerge.

## What This Changes

Adds a synchronous latest recent transcript usage reader, switches status Context calculation to it, re-exports it, and adds a gateway regression test comparing aggregate versus latest usage.

## Best Possible Solution

Use a latest/current prompt snapshot for Context while preserving status' existing local transcript estimate fallback and leaving aggregate readers available for session-list totals.

## Reproduction Assessment

Yes, at source level: current main routes status Context fallback through aggregate transcript usage, and the PR body shows a terminal probe with aggregate 234% versus latest-only 2%. The introduced regression is also source-reproducible because current tests document telemetry-missing transcript estimates that the new latest reader does not preserve.

## Solution Assessment

No: the latest-turn boundary is the right direction, but this implementation should also preserve the existing transcript-estimate fallback for telemetry-missing local runs. A narrow helper/test adjustment is safer than merging the latest-only reader as-is.
