## What This Changes

The PR adds a `trajectory.flushTimeoutMs` config type, an `OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS` fallback, and a timer in the trajectory recorder that calls `writer.flush()` after inactivity.

## Best Possible Solution

Align the PR with #75839 by adding a documented, schema-backed cleanup timeout for `pi-trajectory-flush`, keeping broader `sessions.list` work separate, and proving the configured behavior in a real run.

## Solution Assessment

No. The branch should wire a validated config/env value into the cleanup timeout path, add schema/docs/changelog and tests, and include real behavior proof rather than adding an unvalidated recorder-level timer.
