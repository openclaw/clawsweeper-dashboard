## Summary

Keep open. The PR targets a real Control UI formatting gap on current main and is not superseded, but the head still fails typechecking because `null` formatted output can be stored into `ToolStreamEntry.output`, and contributor real behavior proof is absent.

## What This Changes

Formats structured Control UI tool-result text/content and synthesized stream outputs as JSON strings while adding focused node tests.

## Best Possible Solution

Keep the structured formatting direction, coalesce `null` formatter results back to `undefined` while preserving empty strings, then merge only after real Control UI proof and targeted validation are present.

## Reproduction Assessment

Yes, at source level. Current main ignores object-shaped tool-result text/content in `extractToolText`, and the PR head's CI reproduces the null assignment type failure in `app-tool-stream.ts`.

## Solution Assessment

No, not yet. The formatting approach is narrow, but `formatToolOutput` returns `null` and the PR stores it into a `string | undefined` field; normalize `null` before storing output.
