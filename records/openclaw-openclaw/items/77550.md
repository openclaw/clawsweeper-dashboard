Full review comments:

- **[P1] Keep null out of ToolStreamEntry.output:** `ui/src/ui/app-tool-stream.ts:521`
  - body: `formatToolOutput` returns `null` for nullish tool results, but this guard now preserves every non-`undefined` value and assigns it into `ToolStreamEntry.output?: string`. The head already fails prod/test type checks with TS2322/TS18048 around this path; normalize with a nullish check such as `output ?? undefined` and keep `entry` narrowed before rebuilding the message.
  - confidence: 0.93
- **[P2] Reuse the stored stream output:** `ui/src/ui/app-tool-stream.ts:185`
  - body: `entry.output` is already produced by `formatToolOutput(data.result)` or `formatToolOutput(data.partialResult)` before it reaches `buildToolStreamMessage`. Calling `formatToolOutput(entry.output)` again can truncate an already formatted large output a second time and report the wrong length, so keep the explicit empty-string preservation but pass the stored string through unchanged.
  - confidence: 0.86
