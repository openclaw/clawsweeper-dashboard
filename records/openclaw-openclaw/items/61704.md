## Summary

Keep open. The Gmail log-clarity idea is useful, but this branch is not mergeable as proposed: it marks real watcher startup failures as expected, adds an unfinished docs page, reintroduces a zh-CN redirect that current main deliberately removed, and lacks contributor-supplied real behavior proof.

## What This Changes

The PR appends external-webhook guidance to Gmail watcher non-start warnings, updates the lifecycle test, and adds an exec-approval limitations docs page with docs navigation/i18n changes.

## Best Possible Solution

Refresh this as a narrow Gmail watcher log change that only clarifies expected missing internal-watcher prerequisites, uses configurable endpoint wording, drops unrelated docs/redirect changes, and includes redacted after-fix proof.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: a mocked watcher result such as `gog binary not found` would receive the new `this is expected` suffix, and the added docs page ends inside an open bash fence. The original linked Gmail delivery regression is not reproduced by this PR because the PR only changes log/docs wording.

## Solution Assessment

No. Clarifying `gmail topic required` for external webhook setups is reasonable, but the maintainable fix is conditional wording plus a focused test, not a global warning suffix bundled with unrelated docs and redirect changes.
