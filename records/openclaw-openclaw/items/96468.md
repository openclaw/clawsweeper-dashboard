## Best Possible Solution

Merge this narrow resolver and regression-test change after maintainer acknowledgement that empty legacy arrays mean no explicit inline-button setting, while `capabilities.inlineButtons: "off"` remains the supported opt-out.

Yes. The resolver is the narrowest maintainable layer because prompt capabilities, explicit sends, and callback handling all consume the same inline-button scope, and the PR preserves explicit opt-out behavior.
