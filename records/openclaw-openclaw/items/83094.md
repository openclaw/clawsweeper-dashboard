Keep the implementation narrow to the Phase 1 transport acknowledgement contract, fix pre-acceptance error semantics, make the docs discoverable, and require real handoff proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Return rejected handoffs when dispatch fails:** `src/agents/tools/sessions-send-tool.ts:538`
  - body: The handoff is built as `accepted` before `startAgentRun` succeeds, and that same object is passed into `startAgentRun`. If the gateway `agent` call throws, the tool returns `status: "error"` with `handoff.status: "accepted"` and only the queued/created ledger entry, which violates the PR's own rejected-before-target-runtime contract. Build or update the acknowledgement after start succeeds, or return a rejected/failed handoff for this path.
  - confidence: 0.9
- **[P3] Link the new handoff page into docs:** `docs/concepts/a2a-handoff-phase1.md:1`
  - body: This adds a public concept page, but it is not added to `docs/docs.json` or linked from the existing session/A2A docs. As written, users reviewing `sessions_send` handoff behavior have to know the direct URL, so the docs part of the change is not discoverable. Add it to the relevant docs nav section or link it from the existing session tool and ACP agent pages.
  - confidence: 0.78
