## Summary

Keep this PR open: current main still lacks the proposed `sessions_send` handoff acknowledgement contract, but the draft patch has a source-provable error-path ack mismatch and still needs maintainer contract approval plus contributor real behavior proof before merge.

## What This Changes

This draft PR adds a Phase 1 `sessions_send` A2A handoff contract with structured acknowledgements, a local JSONL ledger helper, control-token outcome recording, docs, and focused tests.

## Best Possible Solution

Keep the primitive narrow, correct start-failure ack and ledger semantics, and land only after maintainer acceptance plus redacted real `sessions_send` proof.

## Reproduction Assessment

Yes for the review finding by source inspection: PR head builds an accepted handoff before `startAgentRun`, passes it to the `agent` call, and the catch path returns it when startup throws. I did not run a live reproduction in this read-only review.

## Solution Assessment

No as currently patched. The implementation should return a non-accepted handoff and record a terminal event when target startup fails, then proceed only after maintainer contract approval and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Return a non-accepted handoff when agent start fails:** `src/agents/tools/sessions-send-tool.ts:186`
  - body: `handoff` is built as `accepted` before either `startAgentRun` call, and this catch path spreads that same object into the error result when `callGateway({ method: "agent" })` throws. A target run that was never accepted can therefore return `status: "error"` with `handoff.status: "accepted"`, while the ledger only has the earlier queued `created` event. Build a rejected or failed handoff and record a terminal event before returning the start error.
  - confidence: 0.88
