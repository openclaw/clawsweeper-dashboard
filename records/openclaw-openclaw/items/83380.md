## Summary

Current main contains the Codex runtime routing and OAuth bootstrap fix for the reported `profile=-`/unauthenticated Responses path, and the fix shipped after the reported beta.4 build in `v2026.5.16-beta.5`.

## What This Changes

The issue reports that Codex-runtime model dispatch fails to bind a registered OpenAI Codex OAuth profile and falls back to an unauthenticated OpenAI Responses request.

## Best Possible Solution

Keep the merged OpenAI Codex routing/auth-bootstrap implementation and direct affected beta.4 or stable users to `v2026.5.16-beta.5` or newer until the next stable release carries it.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection: the corrected path now resolves and forwards Codex OAuth profiles. The reporter's beta.4 logs are concrete pre-fix evidence, but I did not run a live OAuth dispatch here.

## Solution Assessment

Yes. The merged fix is the narrow runtime-owner solution: route explicit Codex OpenAI runs to `openai-codex` and bootstrap the matching OAuth profile into the Codex harness instead of falling back to unauthenticated public OpenAI Responses.
