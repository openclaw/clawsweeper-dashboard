## Summary

Keep open. The auth and MCP hardening direction is plausible, but this PR is not merge-ready because the external-PR real behavior proof is insufficient and the config-schema addition advertises a GPT brevity-guard toggle that is not wired by this PR on current main.

## What This Changes

The PR marks unrecoverable OAuth refresh failures as permanent auth failures, adds optional MCP server degradation/cooldown catalog status, and adds an `agents.defaults.gptChatBrevityGuard` config schema/type entry.

## Best Possible Solution

Land the auth/MCP hardening only with real after-fix proof, and either move the GPT brevity-guard config contract into the paired runtime-toggle PR or include the runtime wiring here so the config does what the schema says.

## Reproduction Assessment

Yes, source-level only: current main lacks the permanent-auth marking, waits through MCP connection timeouts without a degraded cooldown, and rejects the new agent-defaults key. I did not execute a live reproduction because this review is read-only.

## Solution Assessment

No. The auth/MCP approach looks narrow, but the config-schema addition should be wired to runtime behavior or kept with the paired config-toggle PR before this can be the right merge unit.
