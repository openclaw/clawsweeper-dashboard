## Summary

Keep open: current main still lacks `openclaw cron runs <id>`, and this PR remains the canonical implementation path, but it needs contributor real behavior proof and a refresh that preserves current `--run-id` behavior before merge.

## What This Changes

The PR changes `openclaw cron runs` to accept an optional positional job id, keep `--id`, reject mismatched ids, and add focused CLI parsing tests.

## Best Possible Solution

Land a refreshed CLI/docs/test patch that supports both positional and `--id` forms without regressing `--run-id` filtering, backed by redacted real CLI proof.

## Reproduction Assessment

Yes. Current main source registers `cron runs` with required `--id` and no positional `.argument`, so `openclaw cron runs job-123` has no parsing path; I did not run the CLI in this read-only review.

## Solution Assessment

Yes with amendments. A CLI positional alias is a narrow maintainable direction, but the branch should preserve current `--run-id`, keep validation local, update docs, and include real CLI proof.
