## Summary

Keep open: current main still requires `--id` for `cron runs`, and the linked bug was closed in favor of this PR, but the external-contributor proof gate and stale branch refresh still need human handling before merge.

## What This Changes

The PR adds a positional `openclaw cron runs <jobId>` alias, preserves `--id`, rejects conflicting ids, and adds focused cron CLI parsing tests.

## Best Possible Solution

Land a refreshed version of the positional alias that preserves `--run-id`, validates ids before the gateway catch path, updates cron docs, and includes redacted real CLI proof.

## Reproduction Assessment

Yes from source inspection: current main registers `cron runs` with required `--id` and no positional argument, while gateway/protocol already accept a job selector. I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes with small amendments. A CLI-level positional alias is the narrow maintainable direction, but the branch should keep validation outside the gateway error catch, retain current `--run-id` support, and document the new invocation form.
