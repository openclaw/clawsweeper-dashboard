## Summary

Keep this PR open: current main still lacks the positional `cron runs <id>` alias, but the branch is not merge-ready because it lacks real behavior proof and predates current main's `--run-id` filtering/docs surface.

## What This Changes

The PR changes cron CLI parsing and tests so `openclaw cron runs` accepts an optional positional job id alongside `--id` and rejects mismatched ids.

## Best Possible Solution

Land a refreshed CLI/docs/test patch that supports both positional and `--id` forms, preserves `--run-id`, keeps validation local, and includes redacted real CLI proof.

## Reproduction Assessment

Yes, from source inspection. Current main registers `cron runs` with required `--id` and no positional `.argument`, so `openclaw cron runs job-123` has no CLI parsing path for the requested alias; I did not run the CLI in this read-only review.

## Solution Assessment

Yes with amendments. The positional alias is a narrow maintainable CLI improvement, but the PR should preserve current `--run-id`, keep Commander validation out of the gateway catch path, update docs, and add real behavior proof.
