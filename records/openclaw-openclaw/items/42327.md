## Summary

Keep open: current main still requires `--id` for `openclaw cron runs`, and this PR is the active canonical fix path for the linked closed bug. It still needs contributor real CLI proof before merge, and the diff has small P3 documentation/validation polish items.

## What This Changes

The branch changes `openclaw cron runs` to accept an optional positional job id alongside `--id`, reject conflicting ids, and add three cron CLI parsing tests.

## Best Possible Solution

Land a refreshed version of the CLI alias with real CLI proof, local id validation outside the gateway catch path, and cron docs that show the positional and `--id` forms.

## Reproduction Assessment

Yes. From source, current main registers `cron runs` with required `--id` and no positional argument, while the gateway/protocol already accept a job id selector; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes with small amendments. A CLI-level alias is the narrow maintainable direction because the gateway already accepts the selector, but validation should stay local and the public cron docs should describe the new invocation form.
