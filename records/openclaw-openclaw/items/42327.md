## Summary

Keep open: current main still only supports `openclaw cron runs --id <id>`, and this PR is the canonical useful fix for the linked cron-runs positional-id work. It still needs contributor real behavior proof before merge, and the diff has two small P3 follow-ups around validation placement and docs discoverability.

## What This Changes

The PR adds an optional positional job id for `openclaw cron runs`, keeps `--id`, rejects conflicting ids, and adds three cron CLI parsing tests.

## Best Possible Solution

Land the CLI alias after contributor real CLI proof, with local id validation outside the gateway catch path and cron docs updated for the new form.

## Reproduction Assessment

Yes, from source: current main registers `cron runs` with required `--id` and no positional argument, while the gateway/protocol already accept a job id selector. I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes with small amendments. A CLI-level alias is the narrow maintainable direction because the gateway already accepts the selector; validation should stay local and the public docs should describe the new invocation form.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P3] Keep Commander validation out of the gateway catch:** `src/cli/cron-cli/register.cron-simple.ts:77-90`
  - body: The new `command.error()` checks run inside the gateway `try`. Commander 14 writes the error and exits or throws via `_exit`; under `exitOverride`, that throw is caught by `handleCronCliError`, which can reprint it as a runtime error. Move the id validation before the gateway `try` or use one local validation path.
  - confidence: 0.8
- **[P3] Document the positional cron runs form:** `src/cli/cron-cli/register.cron-simple.ts:67-68`
  - body: This adds `openclaw cron runs <id>`, but the cron docs still show only `openclaw cron runs --id <job-id>`. Update the public cron docs when the alias lands so users can discover the new form.
  - confidence: 0.76
