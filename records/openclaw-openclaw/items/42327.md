## What This Changes

The branch changes `openclaw cron runs` parsing and tests to accept an optional positional job id, keep `--id`, and reject mismatched positional and flag ids.

## Best Possible Solution

Land a refreshed CLI/docs/test patch that supports both positional and `--id` forms, preserves current run-id and wait-mode behavior, and includes redacted real CLI output.

## Solution Assessment

Yes with amendments. The positional alias is a narrow maintainable CLI improvement, but the PR must be refreshed onto current main, preserve `--run-id`, update docs, and add real behavior proof.
