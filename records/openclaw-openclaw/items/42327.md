## Summary

Keep open: current main still exposes `openclaw cron runs` as flag-only, and this PR is the active narrow fix for the linked cron-runs positional-ID bug. It should not merge yet because external real behavior proof is still mock/CI-only, and two small P3 cleanup items remain.

## What This Changes

The PR changes `openclaw cron runs` to accept an optional positional job id, preserve `--id`, reject conflicting ids, and adds three cron CLI parsing tests.

## Best Possible Solution

Land a CLI-only positional alias that preserves `--id`, keeps gateway/protocol behavior unchanged, documents the new form, and validates id presence/conflicts through one clean command-validation path.

## Reproduction Assessment

Yes from source: current main registers `cron runs` with required `--id` and no positional argument, while adjacent cron commands use positional job ids and the gateway already accepts an id selector. I did not execute the CLI because this review was read-only.

## Solution Assessment

Yes with small amendments: the PR’s CLI-only alias is the narrow maintainable direction, but validation errors should stay out of the gateway catch path and the public cron docs should mention the new form.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P3] Keep Commander validation out of the gateway catch:** `src/cli/cron-cli/register.cron-simple.ts:77-88`
  - body: `command.error()` throws when the production root program uses `exitOverride`; because these new validation failures run inside the action `try`, they can be caught and re-logged through `handleCronCliError` instead of staying as plain Commander usage errors. Move the id presence/conflict checks before the gateway `try` or use one local validation path.
  - confidence: 0.8
- **[P3] Document the positional cron runs form:** `src/cli/cron-cli/register.cron-simple.ts:67-68`
  - body: This adds a user-facing CLI invocation, but the public cron docs still show only `openclaw cron runs --id <job-id>`. Update the cron docs so the alias is discoverable after merge.
  - confidence: 0.76
