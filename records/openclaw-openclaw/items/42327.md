## What This Changes

The PR adds an optional positional job id to `openclaw cron runs`, preserves `--id`, rejects conflicting ids, and adds cron CLI parsing coverage.

## Best Possible Solution

Land a CLI-only positional alias that preserves `--id`, reports missing/conflicting ids through one clean validation path, updates user docs and changelog, and leaves gateway/protocol behavior unchanged.

## Reproduction Assessment

Yes. Source inspection of current main shows `cron runs` still requires `--id` and has no positional argument, while the adjacent cron commands already use positional job ids.

## Solution Assessment

Yes, with small amendments. A CLI-only alias is the narrow maintainable solution because `cron.runs` already accepts an id selector; the PR should clean up validation, docs, changelog, and coverage before merge.
