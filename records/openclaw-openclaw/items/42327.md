## Summary

Keep open: current main still does not accept a positional job id for `cron runs`, and the linked bug report was closed in favor of this PR. The branch is not merge-ready because it predates newer `--run-id`/`--wait` behavior, has no after-fix real CLI proof, and needs a refresh before normal review.

## What This Changes

The PR changes `cron runs` to accept an optional positional job id alongside `--id`, rejects conflicting ids, and adds CLI parsing tests.

## Best Possible Solution

Refresh and land a positional `cron runs <job-id>` alias that preserves `--id`, `--run-id`, wait-mode polling, and public cron docs.

## Reproduction Assessment

Yes, from source inspection. Current main registers `cron runs` with required `--id` and no positional argument, while the gateway and schema already accept job-id selectors.

## Solution Assessment

No, not as-is. The alias is a reasonable narrow direction, but the branch must preserve current `--run-id`/`--wait` behavior, keep validation local, update docs, and include real CLI proof.
