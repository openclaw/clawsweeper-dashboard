## Summary

Keep this PR open: current main still requires `--id` for `openclaw cron runs`, and this PR is the active implementation candidate for the closed linked report. The branch is small and mostly correct, but it still needs contributor real-behavior proof plus a couple of low-risk cleanup/doc amendments before merge.

## What This Changes

The PR changes `openclaw cron runs` to accept an optional positional job id alongside `--id`, reject conflicting values, and adds cron CLI parsing tests.

## Best Possible Solution

Land a CLI-only positional alias that preserves `--id`, keeps gateway/protocol behavior unchanged, documents the new form, and routes id validation through one clean command-validation path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current main registers `cron runs` with required `--id` and no positional argument, while this PR adds the missing CLI alias.

## Solution Assessment

Is this the best way to solve the issue? Yes with small amendments: a CLI-only alias is the narrow maintainable change because `cron.runs` already accepts an id selector, but the PR should clean validation flow and docs before merge.
