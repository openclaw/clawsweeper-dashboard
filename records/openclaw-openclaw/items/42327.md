## Summary

Keep open: current main and the latest release still do not implement the positional `openclaw cron runs <id>` form, and the related report was closed because this PR is the canonical remaining fix path. The branch is not ready to merge as-is because it lacks external real behavior proof and predates newer `--run-id` cron history behavior on main.

## What This Changes

The PR changes `openclaw cron runs` to accept a positional job id in addition to `--id`, rejects mismatched positional and flag ids, and adds CLI tests for the new parsing behavior.

## Best Possible Solution

Land a refreshed CLI/docs/test patch that supports both positional and `--id` forms, preserves current `--run-id` and wait-mode behavior, and includes redacted after-fix CLI output from a real setup.

## Reproduction Assessment

Yes, from source inspection: current main and `v2026.5.12` register `cron runs` with required `--id` and no positional argument. I did not run the CLI in this read-only review.

## Solution Assessment

Yes with amendments. The positional alias is a narrow maintainable CLI improvement, but the patch needs a current-main refresh that preserves `--run-id`, docs coverage, and contributor real behavior proof.
