## Summary

Keep open: current main still documents xAI realtime voice as not exposed and does not register an xAI realtime voice provider for Voice Call. The open xAI realtime PR at https://github.com/openclaw/openclaw/pull/73032 is adjacent, but its body explicitly excludes the provider-plugin surface used by Voice Call, so it does not supersede this request.

## What This Changes

Request native xAI Grok Voice Agent support for Voice Call realtime, including xAI-specific session setup, barge-in behavior, and possible realtime phone-control tools.

## Best Possible Solution

Add a provider-owned xAI realtime voice bridge through the xAI plugin and existing realtime voice provider seam, then decide separately whether Voice Call should expose built-in realtime phone-control tool handlers.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks an xAI realtime voice provider for Voice Call while already supporting xAI streaming transcription and other xAI media surfaces.

## Solution Assessment

Yes, keeping this open for maintainer direction is the best current outcome. A dedicated xAI provider-plugin implementation fits the owner boundary better than branching on xAI hosts inside the OpenAI bridge, and the phone-control tools should be reviewed as a separate Voice Call behavior decision.
