## Summary

Keep open: current main explicitly documents xAI realtime voice as not exposed, and adding it would be a new provider capability rather than stale cleanup or a bug fix. The existing realtime voice SDK seam makes the feature feasible, but first-class bundled support needs maintainer/provider ownership review.

## What This Changes

Request native xAI Grok Voice Agent realtime voice support for Voice Call, including xAI-specific session handling and optional realtime phone-control tool handlers.

## Best Possible Solution

Add a provider-owned xAI realtime voice bridge through the xAI plugin and existing SDK registration seam, with docs, tests, and redacted live xAI/Twilio proof; decide separately whether realtime DTMF/hangup handlers belong in the same Voice Call change.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection gives high confidence that current main lacks an xAI realtime voice provider while already supporting other xAI audio surfaces.

## Solution Assessment

Yes for the report's Option B direction: a dedicated xAI realtime voice provider matches the plugin/provider boundary better than branching on xAI hosts inside the OpenAI bridge. The DTMF and hangup realtime tools should be reviewed as a second Voice Call feature scope.
