## Summary

Keep open. Current main has the registry and Voice Call seams for realtime voice providers, but xAI/Grok is still not registered for realtime voice; related xAI realtime work covers other surfaces and does not obsolete this Voice Call request.

## What This Changes

Request native xAI Grok Voice Agent support for Voice Call realtime, including provider selection, xAI session schema/barge-in handling, and DTMF/end-call realtime tool support.

## Best Possible Solution

Add a maintainer-approved xAI `RealtimeVoiceProviderPlugin` on the xAI-owned provider surface, with focused Voice Call docs/tests and live telephony proof while keeping Voice Call vendor-agnostic.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main cannot select `realtime.provider: "xai"` because xAI registers no realtime voice provider.

## Solution Assessment

Yes for the dedicated-provider direction, but not as a host branch inside the OpenAI bridge. The maintainable path is provider-owned xAI realtime voice registration plus a separate maintainer decision on Voice Call realtime tool exposure.
