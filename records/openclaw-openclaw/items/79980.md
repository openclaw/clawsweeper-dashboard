## Summary

Keep open: current main still documents xAI realtime voice as unavailable and the bundled xAI plugin does not register a Voice Call realtime provider. The existing SDK seam makes the request feasible, but adding bundled xAI realtime voice plus realtime DTMF/hangup tool handling is new provider/product work, not conservative cleanup.

## What This Changes

Request native xAI Grok Voice Agent support for Voice Call realtime, including provider-specific session schema/barge-in behavior and realtime DTMF/hangup tool handlers.

## Best Possible Solution

Add xAI realtime voice through the xAI plugin using the existing realtime voice provider seam, update manifest/docs/tests, and review realtime DTMF/hangup handlers as a separate Voice Call tool scope with redacted live Twilio/xAI proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection verifies current main lacks xAI realtimeVoiceProviders registration while Voice Call only documents Google and OpenAI realtime voice providers.

## Solution Assessment

Yes for the report's dedicated provider direction, but not for branching on xAI hosts inside the OpenAI bridge. The maintainable path is provider-owned xAI realtime voice plus a separate decision on Voice Call realtime phone-control tools.
