Keep open. Current main still passes the raw Active Memory `timeoutMs` into the embedded runner while the hook and outer watchdog use `timeoutMs + setupGraceTimeoutMs`, and the provided discussion has an open, unmerged PR specifically targeting that handoff. ## What This Changes

The issue reports Active Memory timing out with zero summary output and blocking Telegram/direct chat turns across v2026.4.25-v2026.4.29, with multiple providers and later hook-layer/prep-overhead observations.
