## Summary

Keep this PR open, but it is not merge-ready: the Feishu session activation override is in the right area, yet the patch can trap a group in mention mode by dropping an unmentioned `/activation always` before the command handler runs.

## What This Changes

The PR adds Feishu session-activation override logic and tests so persisted `groupActivation` can override static `requireMention` during group admission.

## Best Possible Solution

Keep Feishu session activation precedence, but make the first group admission gate command-aware so authorized `/activation always` can recover from mention mode, then add live Feishu proof before merge.

## Reproduction Assessment

Yes, at source level. Current main writes `groupActivation` in the shared command handler but Feishu's admission gate reads only config; the PR defect is also source-reproducible because the new override can set `requireMention` before command facts are available.

## Solution Assessment

No. Reading session activation in Feishu admission is the right owner boundary, but the implementation must preserve the command path for switching back to `always` and avoid relying on mock-only proof.
