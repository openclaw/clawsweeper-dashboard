## Summary

Keep open: this PR is the active candidate for the Feishu activation bug, but it is not merge-ready because the new admission override can trap groups in mention mode, adds an uncached session-store read on the group-message hot path, and still lacks real Feishu behavior proof.

## What This Changes

The PR adds a Feishu session `groupActivation` lookup before group admission, applies it to the effective `requireMention` decision, and adds Feishu activation override tests.

## Best Possible Solution

Keep the Feishu session-activation precedence, but make the admission gate command-aware, use cached or prepared session state instead of forced uncached reads, align the docs, and require redacted real Feishu proof before merge.

## Reproduction Assessment

Yes, source-level. Current main writes `groupActivation` through the shared `/activation` handler, while Feishu group admission still derives its hard mention gate from config before later session state can affect behavior.

## Solution Assessment

No. Reading session activation in Feishu admission is the right boundary, but this implementation must preserve `/activation always` recovery and avoid an uncached hot-path session read before it is the best fix.
