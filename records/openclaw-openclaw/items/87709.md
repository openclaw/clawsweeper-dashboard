## Summary

Keep open. The PR targets a valid Feishu group-activation regression, but the patch still reads activation from the pre-binding route for configured/bound group conversations and adds an uncached synchronous session-store read on the group-message hot path; the external PR proof is also source/mock-only.

## What This Changes

The PR adds Feishu session activation lookup so persisted `/activation mention|always` can override config-derived group mention gating, with new Feishu regression tests.

## Best Possible Solution

Use the exact final Feishu route/session key that dispatch records for activation lookup, rely on the existing session-store cache, and then verify the behavior with live or otherwise real Feishu proof before merge.

## Reproduction Assessment

Yes, at source level. Current Feishu group admission resolves `requireMention` from config before dispatch, while `/activation` persists `groupActivation` on the session entry that the gate did not read.

## Solution Assessment

No, not as patched. The Feishu-plugin boundary is the right place, but the lookup needs to use the final dispatch route/session key and the cached session-store path.
