## Summary

Keep open. The PR targets a valid Feishu activation regression, but the current patch can block the unmentioned command that should switch a group back out of mention mode, adds an uncached session-store read on the Feishu group-message hot path, and still lacks real Feishu behavior proof.

## What This Changes

Adds Feishu session `groupActivation` lookup before group admission, applies it to the effective `requireMention` decision, and adds Feishu activation override tests.

## Best Possible Solution

Keep the Feishu session-activation precedence, but make the first admission gate command-aware, avoid forced uncached session reads, align the docs, and require redacted real Feishu proof before merge.

## Reproduction Assessment

Yes, source-level. Current main writes `groupActivation` through the shared `/activation` handler, while Feishu group admission still derives the hard mention gate from config before later session state can affect behavior.

## Solution Assessment

No. Reading session activation in Feishu admission is the right boundary, but this implementation must preserve the `/activation always` recovery path and avoid an uncached hot-path session read before it is the best fix.
