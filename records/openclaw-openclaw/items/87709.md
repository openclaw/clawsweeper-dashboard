## Summary

Keep open. The PR targets a real Feishu activation bug, but the current patch is not merge-ready because it can trap a group in mention mode by dropping an unmentioned authorized `/activation always` before the command handler runs, and the external PR still has only source/mock proof.

## What This Changes

The branch adds Feishu session-activation override logic and regression tests so group admission can use persisted `/activation mention|always` state over static `requireMention` config.

## Best Possible Solution

Keep the Feishu session-activation override, but make the first group admission gate command-aware so authorized activation commands can recover from mention mode, then add redacted live Feishu proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because `/activation mention` writes `groupActivation` while Feishu group admission never reads it; the PR-specific defect is also source-reproducible because the new override can require mentions before command facts are available.

## Solution Assessment

No. Reading session activation in Feishu admission is the right owner boundary, but the implementation must preserve the command path for switching back to `always` and cannot clear the external proof gate with source tests alone.
