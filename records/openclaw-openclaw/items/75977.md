## Summary

Keep this PR open for the existing exact-head automerge path; the diff is coherent for the transcript-read performance goal and I did not find a blocking correctness or security issue.

## What This Changes

This PR makes async transcript history callers choose explicit full or recent reads, adds bounded recent usage/token estimation paths and shared in-flight transcript index builds, and updates tests, docs, and changelog.

## Best Possible Solution

Land this bounded-read performance change through the exact-head automerge/CI gate, keeping full transcript scans only on surfaces that explicitly require arbitrary history.

## Reproduction Assessment

Yes. The code-level reproduction is a large transcript hitting current main chat history, recovery, or token-check paths that call readSessionMessagesAsync and therefore materialize the full async transcript index.

## Solution Assessment

Yes. Explicit full/recent modes plus in-flight index sharing are a narrow maintainable fix because they bound hot readers while preserving annotated full scans where the behavior requires them.
