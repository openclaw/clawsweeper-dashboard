## Summary

Keep open. Current `main` has lower-level `sessions.abort`, `sessions.delete`, cleanup, and SDK cancel primitives, but it still lacks the public `openclaw sessions stop/kill/pause/resume` commands, Sessions UI lifecycle buttons, and quota/resource-limit behavior requested here.

## What This Changes

The issue requests public session lifecycle management commands, Sessions UI controls, and resource-limit controls for OpenClaw sessions.

## Best Possible Solution

Define a maintainer-approved lifecycle contract first, then split implementation into narrow tracks: public stop/abort command and UI action, direct delete/clear CLI review, hard-kill semantics, pause/resume, and quotas/resource alerts.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current `origin/main` shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` commands and no Sessions UI lifecycle callbacks, although lower-level abort/delete primitives exist.

## Solution Assessment

Unclear for the full requested scope. Keeping this open is appropriate, but the maintainable path is to start with a scoped stop/abort contract and route deletion, hard-kill, pause/resume, and quota policy as separate decisions or follow-ups.
