## Summary

Keep open. Current main has useful lower-level session primitives (`sessions.abort`, `sessions.delete`, cleanup, SDK `Run.cancel()`, and a scoped HTTP kill helper), but it still does not provide the requested public `openclaw sessions stop/kill/pause/resume` commands, Sessions UI lifecycle buttons, or quota/resource-limit enforcement.

## What This Changes

Request public session lifecycle management through CLI commands, Web Console controls, pause/resume support, and resource limits.

## Best Possible Solution

Define an additive lifecycle contract first, then split implementation into scoped tracks for stop/abort CLI and UI actions, hard-kill/delete semantics, pause/resume behavior, and quota/resource alerts.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` subcommands and no Sessions UI lifecycle controls, although lower-level abort/delete primitives exist.

## Solution Assessment

Unclear for the full requested bundle. Keeping this open is appropriate, but the maintainable path is a maintainer-approved contract followed by narrower PRs rather than one broad lifecycle/quota implementation.
