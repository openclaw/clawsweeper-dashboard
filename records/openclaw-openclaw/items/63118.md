## Summary

Keep open. Current main and v2026.6.6 have useful session listing, cleanup, trajectory export, usage reporting, `sessions.abort`, `sessions.delete`, and chat-scoped Stop behavior, but they still do not provide the requested public `openclaw sessions stop/kill/pause/resume` lifecycle commands, Sessions-page stop/pause/resume controls, or resource-limit/quota contract.

## What This Changes

Request to add first-class session lifecycle commands, Web Console controls, and resource/quota management for OpenClaw sessions.

## Best Possible Solution

Keep this open as an umbrella lifecycle-management request, design the public stop/abort lifecycle contract first, then split hard kill, pause/resume, Sessions UI controls, and resource/quota enforcement into owner-reviewed follow-ups.

## Reproduction Assessment

Yes for the feature gap: source and release-tag inspection show no public `openclaw sessions stop`, `kill`, `pause`, or `resume` command and no Sessions-page lifecycle callbacks. I did not run a live CLI repro because this was a read-only cleanup review and the command registrations are explicit in source.

## Solution Assessment

No for the full all-in-one requested scope. The maintainable path is staged design: start with a narrow public stop/abort contract, then separate owner-approved work for hard kill, pause/resume, UI controls, authorization, and quota policy.
