## Summary

Keep open: current main and v2026.6.8 expose session list/cleanup/tail/export, sessions.abort/delete, chat-scoped Stop, and partial Sessions UI deletion/settings, but not the requested public stop/kill/pause/resume CLI or Sessions-page lifecycle and quota contract.

Request public session lifecycle management through CLI and Web Console controls for stop, kill, pause, resume, resource limits, and high-consumption alerts.

## Best Possible Solution

Keep this open as the broad lifecycle-management umbrella, have maintainers define the public stop/kill/pause/resume and resource-limit contract, then split focused CLI/Gateway, Sessions UI, hard-kill/exec-control, and quota-enforcement work.
