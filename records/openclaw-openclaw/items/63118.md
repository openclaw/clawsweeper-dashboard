## Summary

Keep open: current main and v2026.6.10 cover pieces of session management with abort, delete, cleanup, compact, and chat/UI stop paths, but they still do not provide the requested public stop/kill/pause/resume CLI, Sessions-page lifecycle controls, or resource-limit contract.

## What This Changes

The issue requests first-class session lifecycle management commands, UI controls, and resource limits for OpenClaw sessions.

## Best Possible Solution

Keep this as the broad session lifecycle umbrella, define the public Gateway/CLI/UI/security contract, then split implementation into focused abort/kill, pause/resume, Sessions UI, and quota/resource-limit slices.

## Reproduction Assessment

Yes, for the source-level missing-command claim: current main registers list, cleanup, tail, export-trajectory, and compact CLI paths plus Gateway abort/delete primitives, but not stop, kill, pause, or resume session commands. I did not run a live long-running agent repro because this review was read-only and the remaining quota/UI pieces are feature scope.

## Solution Assessment

No, not as one all-in-one implementation. The maintainable solution is staged work after maintainers define lifecycle semantics, authorization, process-control behavior, resource policy, and UI ownership.
