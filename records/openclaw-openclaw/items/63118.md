## Summary

Keep open. Current main and the latest release still expose only session listing, cleanup, and export in the public CLI; Gateway has lower-level abort/delete/kill primitives, but the requested public stop/kill/pause/resume commands, Sessions UI lifecycle controls, and quota enforcement are not implemented. The open abort CLI PR covers only a first slice of the request.

## What This Changes

Add public session lifecycle management commands, Control UI actions, and resource limits for stopping, killing, pausing, resuming, and capping OpenClaw sessions.

## Best Possible Solution

Treat this as the lifecycle-management umbrella, review the open abort CLI work as a scoped first slice, and split hard-kill child-process control, UI orchestration, pause/resume, and quota/resource alerts into separately designed follow-ups.

## Reproduction Assessment

Yes for the feature gap. Current-main source and release-tag inspection show no public `openclaw sessions stop`, `kill`, `pause`, `resume`, or `abort` command, while Gateway only exposes lower-level abort/delete/kill primitives.

## Solution Assessment

Unclear for the full requested scope. A scoped abort/stop command is a maintainable first step, but force-kill behavior, pause/resume state, UI controls, and quota enforcement need maintainer design and authorization decisions before implementation.
