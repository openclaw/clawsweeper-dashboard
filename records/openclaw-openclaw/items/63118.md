## Summary

Keep open. Current main has lower-level `sessions.abort`, `sessions.delete`, `sessions.reset`, cleanup, usage, and HTTP kill plumbing, but it still lacks the requested public `openclaw sessions stop|kill|pause|resume` commands, Sessions-page lifecycle controls, and complete quota/resource-limit product contract.

## What This Changes

The issue requests public session lifecycle commands plus UI and resource controls for stopping, killing, pausing, resuming, cleaning up, and limiting OpenClaw sessions.

## Best Possible Solution

Keep this as the umbrella lifecycle-management request, first design the public stop/abort contract around existing `sessions.abort`, then split hard kill, pause/resume, Sessions UI controls, and quota/alert enforcement into staged follow-ups with explicit authorization and config semantics.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` commands and no Sessions-page lifecycle callbacks, while lower-level abort/delete/kill primitives do exist.

## Solution Assessment

Unclear for the all-in-one requested scope. Keeping the issue open is appropriate, but the maintainable fix path is staged design starting with a narrow public stop/abort contract rather than implementing every lifecycle and quota feature at once.
