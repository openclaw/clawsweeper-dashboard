## Summary

Keep open. Current main still has lower-level abort paths and a subagent-only HTTP kill endpoint, but it does not provide the public CLI or Sessions UI lifecycle controls requested here; the related open PR handles deletion cleanup only.

## What This Changes

Request public session lifecycle management for stopping, killing, pausing, resuming sessions, adding UI controls, and enforcing resource limits.

## Best Possible Solution

Define a maintainer-approved lifecycle contract, start with a scoped stop command and Sessions-page stop action backed by `sessions.abort`, then split pause/resume and quotas into narrower follow-ups.

## Reproduction Assessment

Yes. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` commands and no Sessions-page lifecycle callbacks, while `sessions.abort` and subagent kill remain narrower primitives.

## Solution Assessment

Unclear for the full requested scope. Keeping this open is the right triage result, but the safer implementation path is a narrow stop/abort contract first, not one broad PR that also adds pause/resume and resource quotas.
