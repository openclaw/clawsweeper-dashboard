## Summary

Keep open. Current main has useful lower-level primitives such as `sessions.abort`, `sessions.delete`, chat Stop handling, cleanup, and session usage display, but it still does not provide the requested public `openclaw sessions stop|kill|pause|resume` commands, Sessions-page lifecycle buttons, or resource-limit/quota contract.

## What This Changes

The issue requests public session lifecycle management through CLI commands, Web Console controls, cleanup/list filters, pause/resume behavior, and resource limits.

## Best Possible Solution

Keep this as an umbrella lifecycle-management request, design a public contract around existing `sessions.abort`, then split hard kill, pause/resume, Sessions UI controls, and quota enforcement into staged follow-ups.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` command and no Sessions-page lifecycle callbacks, while lower-level abort/delete primitives do exist.

## Solution Assessment

Unclear for the full all-in-one requested scope. The maintainable path is a staged design starting with a narrow public stop/abort contract, then separate owner-approved work for hard kill, pause/resume, UI controls, and quota policy.
