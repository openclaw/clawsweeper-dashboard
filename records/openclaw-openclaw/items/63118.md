## Summary

Keep open: current main and v2026.6.10 still only cover pieces of the request. OpenClaw has session list/cleanup/tail/export/compact, Gateway abort/reset/delete/compact, chat/TUI stop-abort behavior, and Sessions UI deletion/settings, but it still lacks the requested public stop/kill/pause/resume CLI, Sessions-page lifecycle controls, and resource-limit enforcement contract.

## What This Changes

Request public session lifecycle management commands, Web Console controls, and resource limits for running OpenClaw sessions.

## Best Possible Solution

Define a maintainer-approved session lifecycle contract first, then split implementation into focused CLI/Gateway, Sessions UI, session-scoped process-control, and resource/quota enforcement slices.

## Reproduction Assessment

Yes, at source level for the missing-command claim: current main registers list/cleanup/tail/export-trajectory/compact and Gateway abort/delete primitives, but not stop, kill, pause, or resume session commands. The quota and Web Console controls are feature scope rather than a failing existing behavior contract.

## Solution Assessment

No as one all-in implementation: the request should be decomposed after maintainers define lifecycle semantics, authorization, and safety behavior. The maintainable path is staged work that reuses existing abort/delete/session-list primitives without making them imply hard-kill or pause semantics they do not currently guarantee.
