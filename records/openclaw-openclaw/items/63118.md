## Summary

Keep open. Current main has lower-level session abort/delete/reset primitives, cleanup/export CLI support, and Sessions UI delete/status metadata, but the requested public stop/kill/pause/resume commands, UI lifecycle controls, and quota/resource-limit enforcement are still not implemented.

## What This Changes

The issue requests public session lifecycle controls for stopping, killing, pausing, resuming, cleaning up, filtering, and quota-managing OpenClaw sessions from the CLI and web console.

## Best Possible Solution

Use this as the umbrella lifecycle-management request, review the open abort CLI work as a first narrow slice, and split hard-kill child-process control, UI orchestration controls, pause/resume, and quota/resource alerts into separately designed follow-ups.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` command and no Sessions UI lifecycle callbacks, while lower-level Gateway abort/delete primitives do exist.

## Solution Assessment

Unclear for the full requested scope. Keeping the issue open is appropriate, but the maintainable path is staged design and review, starting with a scoped abort/stop contract rather than implementing every lifecycle and quota feature at once.
