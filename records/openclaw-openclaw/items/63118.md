## Summary

Keep open. Current main has useful lower-level session primitives, including `sessions.abort`, `sessions.delete`, cleanup, chat Stop controls, and scoped HTTP kill plumbing, but it still does not provide the requested public `openclaw sessions stop|kill|pause|resume` commands, Sessions-page lifecycle controls, or quota/resource-limit product contract.

## What This Changes

Request public session lifecycle management, including CLI stop/kill/pause/resume commands, Sessions UI controls, active/filtering support, and resource/cost controls.

## Best Possible Solution

Keep this as an umbrella lifecycle-management request, design a public contract around existing `sessions.abort`, then split hard kill, pause/resume, Sessions UI controls, and quota enforcement into staged follow-ups.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows no public `openclaw sessions stop`, `kill`, `pause`, or `resume` commands and no Sessions-page lifecycle callbacks, while lower-level abort/delete/kill primitives do exist.

## Solution Assessment

Unclear for the all-in-one requested scope. The maintainable path is a staged design starting with a narrow stop/abort contract, then separate maintainer-approved work for hard kill, pause/resume, UI controls, and quotas.
