## Summary

Keep open: current main and the latest release still only cover pieces of the request. `sessions.abort`, `sessions.delete`, cleanup/list/tail/export, chat-scoped `/stop`, and partial Sessions UI deletion/settings do not provide the requested public stop/kill/pause/resume CLI, Sessions-page lifecycle controls, or resource-limit contract.

## What This Changes

The issue asks for public session lifecycle controls such as stop/kill/pause/resume commands, Web Console session controls, cleanup/list filters, and resource-limit enforcement.

## Best Possible Solution

Keep this as the broad session lifecycle-management umbrella, have maintainers define the public stop/kill/pause/resume and resource-limit contract, then split focused CLI/Gateway, Sessions UI, hard-kill/exec-control, and quota-enforcement work.

## Reproduction Assessment

Yes for the missing-command claim: current source registers list/cleanup/tail/export-trajectory and Gateway abort/delete primitives, but not stop, kill, pause, or resume session commands. The quota and Web Console controls are feature scope rather than a failing existing behavior contract.

## Solution Assessment

No as a single all-in-one implementation: the request should be decomposed after maintainers define lifecycle semantics, authorization, and safety behavior. The closest maintainable path is staged work that reuses existing abort/delete/session-list primitives without making them imply hard-kill or pause semantics they do not currently guarantee.
