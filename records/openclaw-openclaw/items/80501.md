Keep open: current main has a source-backed Mattermost final-delivery gap and this is not covered by the related wake/thread-routing items. The issue is fresh, externally reported, and describes behavior that conflicts with the Mattermost streaming docs' promise that final replies are not lost.

The issue asks Mattermost group/channel/thread completions to always produce a visible reply or surface an explicit delivery failure.

## Best Possible Solution

Keep the issue open and add Mattermost final-delivery accounting/fallback so non-reasoning final replies either produce a visible post in the originating thread/channel or raise an explicit delivery failure.

## Solution Assessment
