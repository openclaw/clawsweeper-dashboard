## Summary

Keep open: current main still has a plausible Mattermost last-mile delivery gap where the agent can produce a final answer, the Mattermost send can fail, and the monitor only logs the dispatcher error instead of surfacing a visible channel/thread failure or failing the turn. The related open Mattermost items cover wake detection and thread/root routing, not this successful-wake final-delivery failure.

## What This Changes

The issue reports that Mattermost mention-triggered group/thread runs can produce a final answer without posting any visible channel or thread reply.

## Best Possible Solution

Keep this issue open and fix the Mattermost monitor delivery path so failed final sends cannot be counted as a successful private completion without either a visible thread/channel signal or an explicit failed-turn/error outcome.
