## Summary

Keep open: the requested per-label dry-run summary is useful and not on current main, but this PR has two concrete merge blockers: raw session labels are rendered into terminal output and an unrelated sort sweep introduces an argument-spread RangeError risk.

## What This Changes

The PR adds a text-mode `sessions cleanup --dry-run` per-label keep/remove/total summary with docs, changelog, and tests, while also converting many unrelated in-place `sort()` calls across the repo.

## Best Possible Solution

Land a narrow sessions-cleanup renderer/test/docs change that sanitizes labels, preserves JSON output and cleanup policy, and removes the unrelated sort sweep from this PR.

## Reproduction Assessment

Not applicable for the requested feature itself; this is new CLI output rather than broken documented behavior. The PR blockers are high-confidence from PR-head source inspection, and the spread-splice failure was reproduced with a direct Node probe.

## Solution Assessment

No: the per-label summary is the right direction, but this branch is not the best mergeable version. It should sanitize untrusted labels, add regression coverage for that case, and drop the unrelated `sort()` rewrite sweep.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Sanitize labels before rendering the summary:** `src/commands/sessions-cleanup.ts:154`
  - body: Stored session labels are only trimmed and length-checked before persistence, so a label containing newlines, ANSI, or other control bytes will be printed directly in the new dry-run table. Sanitize labels with the existing terminal-safe helper before grouping/rendering and add coverage for that case.
  - confidence: 0.92
- **[P2] Avoid spreading full directory listings into splice:** `extensions/file-transfer/src/node-host/dir-list.ts:164-168`
  - body: This rewrite spreads every listed directory entry into `splice` before pagination. Large directories can exceed V8's argument limit and throw `RangeError: Maximum call stack size exceeded`; keep the original in-place sort or assign a sorted array without spreading.
  - confidence: 0.86
