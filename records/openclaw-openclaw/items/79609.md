## Summary

Keep open: the feature is useful and has supplied live-output proof, but current PR head still renders stored session labels directly into terminal output and the branch now includes a broad unrelated sort rewrite.

## What This Changes

The PR adds a text-mode `sessions cleanup --dry-run` per-label keep/remove/total summary, updates sessions docs/changelog/tests, and rewrites many unrelated `sort()` calls to `toSorted()`.

## Best Possible Solution

Keep the label summary feature, sanitize display labels with `sanitizeTerminalText`, add malicious-label coverage, and scope the branch to sessions cleanup/docs/changelog unless a separate refactor PR owns the sort sweep.

## Reproduction Assessment

Not applicable for the feature request itself; the blocking review finding is source-reproducible from PR head. Stored labels flow from `beforeStore[row.key]?.label` to `row.label.padEnd(...)`, while label parsing does not remove terminal control characters.

## Solution Assessment

No: the requested table shape is the right narrow feature, but the merge-ready solution should sanitize untrusted label text and remove unrelated sweep churn.
