## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only Plugin SDK hooks, existing prompt hooks cannot replace the user prompt, history, or assistant output, and related #74483 explicitly closed unmerged without implementing writable prompt mutation.

## What This Changes

The issue requests writable Plugin SDK LLM input/output hooks so trusted plugins can rewrite prompt, system prompt, history, and assistant output for privacy transformations.

## Best Possible Solution

Design an additive trusted Plugin SDK mutation contract for pre-provider input and post-output content, with explicit permissions, deterministic sequencing, docs/API baselines, and focused contract/runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current main: source, docs, and tests show `llm_input` and `llm_output` are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The safest direction is a permissioned sequential mutation seam rather than making today’s fire-and-forget observation hooks silently accept return values.
