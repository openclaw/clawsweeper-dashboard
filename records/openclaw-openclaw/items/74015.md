## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only hooks, the hook runner discards returned values, and the embedded/CLI/Codex call sites fire the hooks without any path to apply rewritten prompt, history, or assistant output. This is a real Plugin SDK gap rather than a ClawHub-only plugin request because the missing piece is the core SDK mutation contract.

## What This Changes

Request support for trusted plugins to return modified user prompts, system prompts, history messages, and assistant output from existing LLM lifecycle hooks.

## Best Possible Solution

Design an additive, versioned Plugin SDK contract for trusted pre-provider input mutation and post-output mutation, or a deliberate evolution of the LLM hooks, with explicit raw-content permissions, deterministic handler ordering, failure behavior, docs/API baseline updates, and coverage across embedded, CLI, and Codex harness paths.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: source and docs show observation-only LLM hooks with no writable result contract or call-site consumption.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but silently making the current fire-and-forget observation hooks mutable may be less maintainable than a designed permissioned mutation seam.

## Security Review

Status: needs_attention

Summary: The requested SDK capability is security-sensitive because plugins would be able to alter raw prompts, history, and assistant text around model calls.

Concerns:

- **[medium] Define raw-content mutation trust gates:** `src/plugins/hook-types.ts:820`
  - body: A writable `llm_input` or `llm_output` contract would let plugins alter sensitive conversation content before or after provider calls. The design should define explicit permissions, deterministic ordering, auditability, timeout, and failure semantics rather than reusing today's observation-only raw-content access unchanged.
  - confidence: 0.86
- **[medium] Avoid parallel fire-and-forget mutation semantics:** `src/plugins/hooks.ts:461`
  - body: Current LLM lifecycle hooks run through the parallel void hook path. A mutation API needs sequential merge/application rules so multiple trusted plugins cannot race or produce nondeterministic prompt/output rewrites.
  - confidence: 0.82
