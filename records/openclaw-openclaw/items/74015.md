## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only Plugin SDK hooks, while existing prompt hooks can only add context or system-prompt text; writable prompt, history, and assistant-output mutation remains a core SDK design gap, and related #74483 explicitly says it did not implement this work.

## What This Changes

Allow existing Plugin SDK LLM input/output hooks to return prompt, system prompt, history, or assistant-output mutations for privacy plugins such as PII desensitization and resensitization.

## Best Possible Solution

Design an additive trusted Plugin SDK mutation contract for pre-provider input and post-output content, with explicit config permissions, deterministic sequencing, runtime-specific limits, docs/API baselines, and focused contract/runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current main: source, docs, and tests show `llm_input` and `llm_output` are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The safest direction is a permissioned sequential mutation seam rather than making today’s fire-and-forget observation hooks silently accept return values.
