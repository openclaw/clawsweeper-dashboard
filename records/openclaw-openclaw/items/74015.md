## Summary

Keep open. Current main still treats `llm_input` and `llm_output` as observation-only hooks, and the existing prompt hooks cannot rewrite the original user prompt, history, or assistant output; this remains a core Plugin SDK design gap rather than ClawHub-only work.

## What This Changes

The request asks for Plugin SDK LLM input/output hooks to optionally return replacements for prompt, system prompt, history, and assistant text so privacy plugins can desensitize and resensitize content around provider calls.

## Best Possible Solution

Define an additive, permissioned SDK contract for trusted pre-provider input mutation and post-output mutation, with deterministic ordering and failure semantics, then update API baselines, docs, config gates, and focused runtime/contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence: current source, docs, and tests show LLM input/output hooks are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The requested capability is valid, but a deliberately permissioned sequential mutation seam is safer than simply changing today’s fire-and-forget observation hooks to accept return values.
