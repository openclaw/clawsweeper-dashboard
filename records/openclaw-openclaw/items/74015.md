## Summary

Keep open. Current main still leaves `llm_input` and `llm_output` as observation-only hooks, and the newer plugin text-transform surface is only a static replacement mechanism rather than writable per-turn hook mutation for prompt, history, or assistant output.

## What This Changes

Request to let Plugin SDK LLM input/output hooks return modified prompt, system prompt, history messages, and assistant text for pre-provider sanitization and post-output restoration.

## Best Possible Solution

Define an additive trusted SDK mutation seam for pre-provider input and post-output content, with explicit permissions, deterministic ordering, failure semantics, docs, API baselines, and focused runtime/contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current source and docs: LLM hooks return void, prompt hooks cannot replace prompt/history, and returned LLM hook values are never applied.

## Solution Assessment

Unclear because there is no patch yet. A safer direction is a permissioned sequential mutation contract, likely aligned with existing text transforms where useful, rather than silently changing today’s fire-and-forget observation hooks to accept arbitrary return values.
