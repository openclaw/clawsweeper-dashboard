## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as void observation hooks and does not provide a way for plugins to replace the raw prompt, history, or assistant text; this is a real Plugin SDK gap, but it needs an API/security design decision before implementation.

## What This Changes

The issue requests making existing Plugin SDK LLM input/output hooks return optional prompt, system prompt, history, and assistant text replacements for privacy-preserving content transformation.

## Best Possible Solution

Design an additive, permissioned Plugin SDK mutation seam for pre-provider input and post-output content, with deterministic sequencing, failure behavior, docs, API baselines, and focused Pi/CLI/Codex harness coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The gap is high-confidence from current source and docs: LLM hooks are void-only observation hooks, and existing prompt hooks do not replace raw prompt or history.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the safer maintainable solution is an explicit permissioned mutation contract rather than silently making today’s fire-and-forget observation hooks writable.
