## Summary

Keep open. Current main still keeps `llm_input` and `llm_output` as observation-only hooks, while the existing prompt hooks can add context or system text but cannot replace the raw user prompt, history, or assistant output requested here.

## What This Changes

Request writable Plugin SDK hooks that can rewrite user prompt, system prompt, history, and assistant output for privacy/desensitization workflows.

## Best Possible Solution

Define an additive, permissioned, sequential mutation contract for trusted pre-provider input and post-output transformation, then update SDK baselines, docs, config gates, and focused runtime/contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current source, tests, and docs: returned values from `llm_input`/`llm_output` are not typed or applied, and prompt hooks cannot replace prompt/history.

## Solution Assessment

Unclear because there is no patch yet. The capability is valid, but a purpose-built trusted mutation seam with deterministic ordering is safer than changing today’s fire-and-forget observation hooks in place.
