## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only Plugin SDK hooks: their public handler types return `void`, the runner sends them through the parallel void path, and embedded/Codex harness call sites fire them without awaiting or applying returned data. This is not a ClawHub-only request because the core Plugin SDK lacks the writable raw-prompt/output seam needed for a privacy plugin to implement the proposed behavior.

## What This Changes

The issue requests a writable Plugin SDK hook path for modifying raw user prompts, system prompts, history messages, and assistant output around LLM calls.

## Best Possible Solution

Define an additive, permissioned Plugin SDK contract for trusted pre-provider input mutation and post-output mutation, with deterministic handler ordering, explicit raw-content permissions, docs, API baseline updates, and coverage for embedded, CLI/harness, and Codex paths.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main verification path is high confidence: source and docs show the LLM hooks are observation-only and no call site consumes writable prompt/output results.

## Solution Assessment

Unclear because there is no patch yet. The requested capability is valid, but silently making the existing fire-and-forget observation hooks mutable is probably less maintainable than a deliberately permissioned mutation seam.
