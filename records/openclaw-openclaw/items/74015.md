## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only hooks, and the existing prompt hooks can add context or replace system prompt text but cannot rewrite the raw prompt, history, or returned assistant text.

## What This Changes

Request to let Plugin SDK LLM input/output hooks return prompt, history, system prompt, and assistant-output mutations for privacy and compliance plugins.

## Best Possible Solution

Design an additive, permissioned Plugin SDK mutation contract for pre-provider input and post-output content, with deterministic sequencing, explicit trust gates, docs, API baselines, and focused harness coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from source and docs: current LLM hooks are void observation hooks, and prompt-build hooks cannot replace raw prompt/history or assistant output.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction is valid, but the maintainable solution should be an explicit trusted mutation contract rather than silently making today's fire-and-forget observation hooks writable.
