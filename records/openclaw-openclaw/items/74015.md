## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only Plugin SDK hooks, and the available prompt hooks cannot rewrite the raw user prompt, history, or assistant output. The related #74483 context explicitly says it does not implement writable prompt mutation, so this remains a real core SDK gap rather than a ClawHub-only plugin request.

## What This Changes

The issue requests making existing `llm_input` and `llm_output` Plugin SDK hooks optionally return modified prompt, system prompt, history, and assistant output values for local desensitization and resensitization.

## Best Possible Solution

Keep this open until maintainers define an additive, permissioned SDK contract for trusted pre-provider input mutation and post-output mutation, update docs and API baselines, and cover embedded, CLI/harness, and Codex execution paths so privacy plugins can live outside core.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The verification path is high-confidence: current main's type contract, hook runner, docs, and embedded/Codex/CLI call sites show `llm_input` and `llm_output` are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The requested capability is valid, but mutating today's fire-and-forget hooks in place is likely less maintainable than a deliberately permissioned additive mutation seam.
