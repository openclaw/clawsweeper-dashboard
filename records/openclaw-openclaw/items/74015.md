## Summary

Keep open. Current main still treats `llm_input` and `llm_output` as observation-only Plugin SDK hooks, while the existing prompt hooks cannot replace the raw user prompt, session history, or assistant output; this remains a real core SDK design gap, not a ClawHub-only plugin request.

## What This Changes

The issue requests writable Plugin SDK LLM input/output hooks so plugins can modify user prompts, system prompts, history, and assistant output around provider calls.

## Best Possible Solution

Define an additive, permissioned SDK mutation seam for trusted pre-provider input rewriting and post-output rewriting, with deterministic ordering, failure behavior, docs, API baselines, and focused runtime/contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence: current source, docs, and tests show LLM input/output hooks are observation-only and existing prompt hooks cannot replace prompt, history, or assistant output.

## Solution Assessment

Unclear because there is no patch yet. The requested capability is valid, but a deliberately permissioned sequential mutation seam is safer than changing today’s fire-and-forget observation hooks to silently accept return values.
