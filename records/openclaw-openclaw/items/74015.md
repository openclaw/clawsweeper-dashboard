## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only hooks, while the open overlapping PR #20802 is broader than this issue and does not settle the requested history-mutation/security contract well enough to make this issue safe to close as a duplicate.

## What This Changes

Allow Plugin SDK LLM input/output hooks to return mutations for prompt, system prompt, history, and assistant text so plugins can redact before provider calls and rehydrate after responses.

## Best Possible Solution

Define and land an additive, permissioned LLM mutation seam that covers prompt, system prompt, history, and assistant output consistently across Pi and Codex paths, with explicit sequencing, audit, failure handling, docs, API baselines, and contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current main: source, docs, and tests still show LLM hooks as observation-only and returned values are not applied.

## Solution Assessment

Unclear because there is no accepted patch yet. The request is valid, but the maintainable route is a reviewed trusted mutation contract or a narrowed replacement for #20802 rather than silently making the existing observation hooks writable without new trust semantics.
