## Summary

Keep open. Current main still treats `llm_input` and `llm_output` as observation-only hooks, the prompt-build hooks only add or replace context/system-prompt text, and related #74483 was closed unmerged while explicitly saying it did not implement writable prompt mutation.

## What This Changes

The issue requests writable Plugin SDK LLM input/output hooks that can rewrite user prompt, system prompt, history, and assistant text for privacy transformations.

## Best Possible Solution

Design an additive, permissioned Plugin SDK mutation contract for pre-provider input and post-output content, with deterministic ordering, failure semantics, docs/API baselines, and focused PI/Codex/CLI contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current source/docs/tests: LLM hooks are typed and run as void observation hooks, and prompt hooks cannot replace prompt text, history, or assistant output.

## Solution Assessment

Unclear because there is no patch yet. The safest direction is a trusted sequential mutation seam with explicit policy, not making today’s fire-and-forget observation hooks silently accept return values.
