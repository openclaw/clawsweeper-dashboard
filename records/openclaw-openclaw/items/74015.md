## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only hooks, and the existing prompt hooks can add context or replace system prompt text but cannot rewrite the raw user prompt, history, or assistant output. This remains a core Plugin SDK design gap rather than ClawHub-only work because the requested privacy plugin cannot be built cleanly without a new or expanded SDK seam.

## What This Changes

The issue requests writable Plugin SDK LLM input/output hooks so plugins can transform prompts, system prompts, history, and assistant text around provider calls.

## Best Possible Solution

Keep this open until maintainers define an additive, permissioned SDK contract for trusted pre-provider input mutation and post-output mutation, then update API baselines, config/schema/help, docs, and focused contract/runtime tests so the privacy plugin can live outside core.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The verification path is high-confidence: current source, docs, and tests show LLM input/output hooks are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The requested capability is valid, but changing today's fire-and-forget hooks in place is likely less maintainable than a deliberately permissioned, sequential mutation seam with explicit ordering and failure semantics.
