## Summary

Close as duplicate/superseded. Current main still lacks writable `llm_input`/`llm_output` results, but open PR #61485 already tracks the same focused mutable-hook API and is the better place for maintainer/security review; older broad PR #20802 is still open but its discussion points toward #61485 as the narrower path.

## What This Changes

Request to let existing Plugin SDK `llm_input` and `llm_output` hooks return mutation results so plugins can rewrite prompts/history and assistant output for PII redaction and rehydration.

## Best Possible Solution

Concentrate the implementation discussion on #61485 or a maintainer-approved successor: an additive, permissioned mutable `llm_input`/`llm_output` contract with deterministic ordering, failure semantics, docs/API baseline updates, and coverage for embedded, CLI/harness, and Codex paths.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main verification path is high confidence: source and docs show the LLM hooks are observation-only and call sites do not await or apply returned prompt/output data.

## Solution Assessment

Yes for cleanup: the best repository hygiene outcome is to close this duplicate feature request and route discussion to the existing focused PR #61485. The code solution itself is not complete yet because current main is unchanged and the open PR still needs maintainer/security review.
