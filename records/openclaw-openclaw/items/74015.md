## Summary

Keep open. Current main still treats `llm_input` and `llm_output` as observation-only hooks and discards handler return values; `before_prompt_build` can add context or system-prompt text but cannot rewrite the raw prompt/history, and related #74483 explicitly did not implement this work.

## What This Changes

The issue requests writable Plugin SDK LLM input/output hooks so trusted plugins can modify prompts, system prompts, history, and assistant text around provider calls.

## Best Possible Solution

Design an additive trusted Plugin SDK mutation seam for pre-provider input and post-output content, with explicit permission/config, deterministic sequencing, failure semantics, docs/API baselines, and focused Pi, CLI, and Codex harness tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from current source and docs: LLM hooks are void-only observation hooks and existing prompt hooks do not replace raw prompt or history.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the maintainable solution should be an explicit permissioned mutation contract rather than silently making today’s fire-and-forget observation hooks writable.
