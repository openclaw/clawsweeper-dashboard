## Summary

Keep open. Current main still treats `llm_input` and `llm_output` as observation-only void hooks, and the existing prompt-build hooks cannot replace the original prompt, history, or assistant output; this remains a real core Plugin SDK design gap with security implications.

## What This Changes

Request support for writable LLM input/output plugin hook results so trusted privacy plugins can sanitize prompts, history, and assistant text around provider calls.

## Best Possible Solution

Define an additive, permissioned SDK mutation seam for trusted pre-provider input and post-output transformation, then update runtime application points, API baselines, docs, config gates, and focused contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence: current source, docs, and tests show LLM input/output hooks are observation-only and returned values are never applied.

## Solution Assessment

Unclear because there is no patch yet. The safest direction is a deliberate sequential mutation contract with trust gates, not silently making today’s fire-and-forget observation hooks mutable.
