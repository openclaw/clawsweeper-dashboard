## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only fire-and-forget hooks, existing prompt hooks cannot replace the raw prompt/history or post-process assistant text, and the related assembly-hook PR is still open and does not fully settle this SDK/API design.

## What This Changes

Request to let Plugin SDK hooks rewrite raw prompt, system prompt, history, and assistant output for local desensitization and resensitization before and after provider calls.

## Best Possible Solution

Design an additive trusted Plugin SDK mutation seam for pre-provider input and post-output content, with explicit policy, deterministic sequencing, cloned/validated payloads, docs/API baselines, and focused Pi, CLI, and Codex harness coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from source and docs: current LLM hooks are void-only observations and prompt hooks cannot replace raw prompt/history/output.

## Solution Assessment

Unclear because there is no patch on this issue. The requested direction is plausible, but the maintainable solution should be a guarded mutation contract rather than silently making today’s observation hooks writable.
