## Summary

Keep open. Current main still exposes `llm_input` and `llm_output` as observation-only hooks, while `before_prompt_build` and legacy `before_agent_start` cannot replace the raw prompt, history, or assistant text; related #74483 explicitly left this work out. The remaining work is a valid but security-sensitive Plugin SDK API design, not conservative cleanup.

## What This Changes

Request writable Plugin SDK LLM input/output hook results so plugins can replace prompt, system prompt, history messages, and assistant texts for sanitization and resensitization.

## Best Possible Solution

Design an additive trusted Plugin SDK mutation seam for pre-provider input and post-output content, using existing or explicitly updated hook policy gates, deterministic ordering, failure semantics, docs/API baselines, and focused Pi, CLI, and Codex harness tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from source and docs: current LLM hooks are void-only observations, and existing prompt hooks cannot replace raw prompt, history, or assistant output.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the maintainable solution should be an explicit permissioned mutation contract rather than silently accepting return values from today’s fire-and-forget observation hooks.
