## Summary

Keep open: the PR is a plausible runtime stability fix, but it is not mergeable as-is because the Gmail watcher backoff resets before an immediately crashing child proves it is healthy, the external PR still lacks real after-fix runtime proof, and part of the diff is stale against current main.

## What This Changes

The PR wraps subagent output polling errors, adds best-effort config warning logs, adds Gmail watcher restart backoff, clears a legacy media-server cleanup interval, and deletes the i18n registry test.

## Best Possible Solution

Carry forward the useful fixes in a smaller rebased patch that resets Gmail backoff only after a healthy uptime window, preserves i18n coverage, drops the removed media-host path, adds focused tests, and includes redacted real runtime proof.

## Reproduction Assessment

Yes for the blocking PR defect: a Node child_process probe showed an immediately exiting child emits spawn before exit, so the PR's spawn-time backoff reset is reproducibly wrong. The remaining runtime gaps are source-reproducible on current main rather than fully exercised end to end.

## Solution Assessment

No. The direction is useful, but this patch does not fix persistent Gmail crash loops and includes stale or unrelated changes; a narrower rebased patch is the maintainable path.
