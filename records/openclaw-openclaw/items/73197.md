## What This Changes

The PR adds subagent output read-error suppression, best-effort config warning logs, Gmail watcher restart backoff, media cleanup interval clearing, and deletes the UI i18n registry test.

## Best Possible Solution

Carry forward the useful fixes in a smaller rebased patch that resets Gmail backoff only after a healthy uptime window, preserves i18n coverage, drops the obsolete media host edit, adds focused tests, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the blocking PR defect: the PR resets restartBackoffMs on child spawn, and a Node child_process probe showed an immediately exiting child emits spawn before exit. The remaining underlying runtime gaps are source-reproducible on current main rather than fully exercised end to end.

## Solution Assessment
