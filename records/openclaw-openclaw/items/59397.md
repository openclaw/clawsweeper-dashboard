## Summary

Keep open: current main still has the nodes CLI timeout validation gap, but this PR still has a blocking user-visible error-message regression and lacks after-fix real behavior proof from the contributor.

## What This Changes

The branch tightens shared CLI timeout parsing to decimal strings, routes nodes CLI RPC timeouts through that helper, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the narrow nodes CLI validation fix, but reject partial/scientific timeout strings through the shared helper while preserving invalidTimeout() wording, then add redacted terminal proof that invalid nodes --timeout input fails before the gateway call.

## Reproduction Assessment

Yes, by source inspection. Current main forwards raw Number(opts.timeout ?? 10_000) from the nodes CLI runtime, and callGateway falls back when that value is non-finite, so invalid CLI timeout input can bypass validation.

## Solution Assessment

Mostly yes. Reusing the shared timeout parser in the nodes runtime is the narrow maintainable fix, but the branch should preserve the current shared invalid-timeout error wording and provide real after-fix CLI proof.
