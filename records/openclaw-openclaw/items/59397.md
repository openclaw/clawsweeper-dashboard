## Summary

Keep open: this PR targets a real nodes CLI timeout-validation gap that still exists on current main, but the current branch is not mergeable as-is because it is based on an older shared parser shape, changes the invalid timeout error contract, and still lacks after-fix real behavior proof.

## What This Changes

The PR routes the nodes CLI runtime timeout through the shared parser, tightens timeout parsing to reject partial/non-decimal strings, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Rebase the narrow nodes-runtime parser change on current main, preserve the current `invalidTimeout()` wording, keep the focused parser/nodes tests, and add redacted real CLI behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main forwards `Number(opts.timeout ?? 10_000)` from the nodes CLI runtime, and `callGateway` treats non-finite timeout values as unset, so invalid timeout input can still bypass CLI validation.

## Solution Assessment

Mostly yes. Reusing the shared parser in the nodes runtime is the narrow maintainable fix, but the branch should preserve current main's shared invalid-timeout error contract instead of adding a separate lower-case error path.
