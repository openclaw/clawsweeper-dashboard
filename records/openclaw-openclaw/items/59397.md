## Summary

Keep open: the underlying nodes CLI timeout bug is valid, but this PR is not merge-ready because it drops the current shared timeout error wording and still lacks contributor-provided real behavior proof.

## What This Changes

The PR tightens shared CLI timeout parsing to decimal strings, routes nodes CLI RPC timeouts through that helper, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the narrow nodes runtime validation fix, preserve the current invalidTimeout() wording in the shared helper, and add redacted real CLI proof that invalid nodes --timeout input fails before a gateway call.

## Reproduction Assessment

Yes, by source inspection. Current main forwards raw Number(opts.timeout ?? 10_000) from the nodes CLI runtime, and callGateway falls back when that value is non-finite, so invalid CLI timeout input can bypass validation.

## Solution Assessment

Mostly yes. Reusing the shared timeout parser in the nodes runtime is the narrow maintainable fix, but the branch should preserve the current shared invalid-timeout error wording and provide real after-fix CLI proof.
