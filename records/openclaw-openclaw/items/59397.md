## What This Changes

The branch routes nodes CLI RPC timeout values through the shared timeout parser, tightens timeout parsing to decimal strings, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land a narrow nodes runtime validation fix after preserving the shared error helper contract, resolving the merge conflict, deciding the strict-decimal compatibility tradeoff, and adding redacted CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main forwards raw `Number(opts.timeout ?? 10_000)` from the nodes CLI runtime, and the gateway call falls back when the value is non-finite, so invalid timeout input can bypass CLI validation.

## Solution Assessment

Mostly yes. Reusing the shared parser in the nodes runtime is the right narrow direction, but the branch should preserve the shared invalid-timeout error wording and needs a maintainer call on strict decimal compatibility.
