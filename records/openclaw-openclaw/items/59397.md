## Summary

Keep this PR open: the underlying nodes CLI timeout bug is still present on current main, but the PR head still needs a small parser error-contract fix and real behavior proof before merge.

## What This Changes

The PR applies shared timeout parsing to the nodes CLI gateway transport path, rejects partial/non-decimal timeout strings, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Rebase the narrow nodes-runtime/shared-parser change on current main, keep the current invalidTimeout wording, reject whole-string invalid timeout values, and add redacted real CLI behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main still uses raw Number(opts.timeout ?? 10_000) in the nodes CLI runtime, and gateway timeout resolution masks non-finite values, so invalid nodes CLI timeout handling can be traced without a live gateway run.

## Solution Assessment

Mostly yes. Reusing the shared parser in the nodes runtime is the narrow maintainable fix, but the PR should preserve the current shared invalidTimeout() error contract for the newly rejected string forms.
