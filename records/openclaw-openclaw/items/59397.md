## Summary

Keep this PR open. Current main and the latest release still leave the nodes CLI timeout unvalidated, but the source branch is stale against the runtime split and its helper path would accept partial timeout strings, so it needs a narrow repair before merge.

## What This Changes

The PR switches the nodes CLI gateway timeout path to `parseTimeoutMsWithFallback(..., { invalidType: "error" })` and adds a regression test for `--timeout 0`.

## Best Possible Solution

Land a narrow update against current main that validates whole positive integer millisecond strings in the nodes CLI runtime path, adds focused shared-helper and nodes CLI tests for invalid and valid inputs, and records the user-visible fix in the active changelog.

## Solution Assessment
