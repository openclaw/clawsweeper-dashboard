## Summary

Keep open. Current main still has the nodes CLI timeout validation gap and this PR is the right narrow direction, but the head is merge-conflicting, lacks after-fix real behavior proof, and needs one current-main error-message fix during rebase.

## What This Changes

The PR tightens shared CLI timeout parsing, applies the parser to the nodes CLI runtime transport timeout, adds focused parser/nodes tests, and adds a changelog entry.

## Best Possible Solution

Rebase this PR on current main, keep the strict whole-decimal parser and nodes-runtime validation, preserve the current shared timeout error wording, add redacted real CLI output proof, and rerun the targeted parser/nodes tests plus the changed gate.

## Reproduction Assessment

Yes. Source inspection shows the nodes runtime still uses raw `Number(...)` timeout coercion and the shared helper still uses `parseInt`, so invalid or partial values can be traced without a live gateway run.

## Solution Assessment

Mostly yes. The shared-parser approach is the narrow maintainable fix, but the rebased patch should use the current `invalidTimeout(value)` path for non-decimal strings instead of adding a second error format.
