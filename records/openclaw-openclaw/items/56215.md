## Summary

Keep open. Current main and v2026.6.8 have some Gateway WebSocket cleanup hardening, but they do not prove the reported Windows CLOSE_WAIT/FIN_WAIT_2 accumulation and crashes are fixed; the WebSocket handler still only sends pings and does not track pong liveness or terminate missed-pong sockets.

## What This Changes

The issue reports Gateway WebSocket connections accumulating in CLOSE_WAIT/FIN_WAIT_2 on Windows until the Gateway crashes.

## Best Possible Solution

Keep this issue open until a Windows Gateway run on current main or latest release proves zombie sockets no longer accumulate, or land a focused heartbeat/termination or native-crash repair with live Windows netstat and crash-report proof.
