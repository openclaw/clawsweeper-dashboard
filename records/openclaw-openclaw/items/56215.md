## Summary

Keep open. ## What This Changes

The issue reports Gateway WebSocket socket lifecycle leaks on Windows, with CLOSE_WAIT/FIN_WAIT_2 connections accumulating until Gateway crashes.

## Best Possible Solution

Keep this open for a maintainer-chosen Gateway reliability fix that proves normal runtime half-open WebSocket cleanup on current Windows builds, while treating the related Windows crash report as overlapping evidence rather than a duplicate.

## Reproduction Assessment

No high-confidence live reproduction was established against current main. Source inspection does show the normal Gateway WebSocket path lacks missed-pong termination, keeping the reported half-open socket leak plausible.

## Solution Assessment

Unclear. Adding a ws-style missed-pong termination path is a plausible narrow fix for the socket lifecycle gap, but Windows live proof or a fatal report should confirm whether it also addresses the observed crashes.
