## Summary

The requested pre-spend refusal capability already exists through the shipped `before_agent_run` plugin hook, which receives the assembled prompt and session context, accepts a block verdict with an optional user-facing message, and skips model submission and downstream output hooks.

## What This Changes

Requests an additive plugin hook verdict that can cancel a model turn before upstream dispatch and return a user-visible response.

## Best Possible Solution

Use the shipped `before_agent_run` block decision for pre-spend refusal and its optional user-facing replacement message rather than introducing a second cancellation path around model resolution.

## Reproduction Assessment

Yes at source and contract level. The shipped runner path and focused test show that a `before_agent_run` block skips model submission and suppresses `llm_output`; the merged implementation also includes live gateway proof.

## Solution Assessment

Yes. The existing `before_agent_run` gate is better positioned than overloading `before_model_resolve` because it sees the final prompt, loaded messages, and system prompt immediately before submission while keeping cost policy plugin-owned.
