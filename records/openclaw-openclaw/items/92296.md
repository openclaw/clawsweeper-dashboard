## Summary

Current main already provides the requested pre-spend refusal path through `before_agent_run`: plugins can inspect prepared messages, block before model execution, and supply a user-visible refusal. This implementation landed shortly before the issue was opened and is not yet included in the latest release.

## What This Changes

The issue requests an additive plugin hook verdict that can cancel a model turn before provider dispatch and return a user-visible reply.

## Best Possible Solution

Keep `before_agent_run` as the single canonical pre-dispatch refusal mechanism and document it for cost-policy plugins; open a narrower follow-up only if a concrete cost-estimation input is still unavailable.

## Reproduction Assessment

No. At current main, focused source and regression tests show `before_agent_run` can block after messages are prepared and before `llm_input` or model execution.

## Solution Assessment

No. Extending `before_model_resolve` or adding another model-call hook would duplicate the existing input gate; `before_agent_run` is the narrower canonical solution.
