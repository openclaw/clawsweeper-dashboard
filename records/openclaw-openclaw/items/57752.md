## Summary

The issue requests compatibility for Kimi/Moonshot models that failed on v2026.3.28 with an Anthropic-compatible streaming event-order error.

Keep the shipped boundary-aware Anthropic-compatible transport as the canonical fix. If Kimi or Moonshot still fails on a current release, it should be handled as a new provider-specific regression with fresh logs, selected model ref, endpoint family, and live reproduction details rather than keeping this v2026.3.28 SDK stream-order report open.
