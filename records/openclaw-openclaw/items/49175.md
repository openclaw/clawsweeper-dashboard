## Summary

Keep open: current main and v2026.6.6 still build all-scope `cron.runs` job-name enrichment from the unfiltered cron job list, and the only live closing PR is open but conflicting/unmerged.

## What This Changes

Scope `cron.runs(scope="all")` job-name enrichment so callers do not see cron job names from other agents or sessions.

## Best Possible Solution

Land or replace the linked fix PR with a current-main-compatible patch that builds all-scope `jobNameById` from the scoped paginated job list and includes focused regression coverage.

## Reproduction Assessment

Yes, at source level. A `cron.runs` call with `scope: "all"` reaches the handler branch that enriches run rows from unfiltered `context.cron.list()`, so jobs from other agents can contribute names to returned history entries.

## Solution Assessment

Yes. The narrow maintainable fix is to keep all-scope run-log scanning semantics but build the job-name map through the existing scoped paginated job listing path.
