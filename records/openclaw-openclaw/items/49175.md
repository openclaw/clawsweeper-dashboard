## Summary

Keep open: current `main` and the latest release still build the all-scope cron run-history job-name map from an unfiltered cron job list, while an open unmerged PR already targets this issue.

## What This Changes

The issue asks for `cron.runs(scope: "all")` to avoid exposing cron job names from other agent sessions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80146, or an equivalent narrow fix, so all-scope job-name enrichment is built from scoped `listPage()` results with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: create jobs for multiple agents, call `cron.runs` with `scope: "all"`, and current `main` builds the returned/searchable job-name map from an unfiltered cron list.

## Solution Assessment

Yes. Reusing the existing `listPage()` agent filter for name enrichment is the narrow maintainable fix for the named leak while leaving the broader all-scope run-log model as an explicit maintainer boundary.
