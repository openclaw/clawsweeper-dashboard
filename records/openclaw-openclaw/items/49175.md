## Summary

Keep open: current `main` and the latest release still build the all-scope `cron.runs` job-name map from the unfiltered cron job list. The linked closing PR at https://github.com/openclaw/openclaw/pull/80146 is still open and conflicting, so this issue should remain open until that fix is rebased, landed, or replaced.

## What This Changes

The issue requests that `cron.runs` with `scope: "all"` stop enriching run-log entries with cron job names from other agents or sessions.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80146 with a current-main-compatible patch that builds all-scope `jobNameById` from the scoped paginated job list while preserving the existing run-log scan semantics.

## Reproduction Assessment

Yes, at source level. Seed jobs for different agents and call `cron.runs` with `scope: "all"`; current `main` builds the response name map from unfiltered `context.cron.list()` before stamping `jobName` onto run-log entries.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing agent-filtered `listPage` path for name enrichment while keeping all-scope run-log scanning unchanged.
