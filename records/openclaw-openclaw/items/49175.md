## Summary

Keep open: current main and the latest release still build the `cron.runs(scope: "all")` job-name map from the unfiltered cron list, and the open closing PR has not merged yet.

## What This Changes

The issue reports that `cron.runs(scope: "all")` enriches run-history rows with cron job names from other agents or sessions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80146, or an equivalent narrow fix that builds all-scope job-name enrichment from the scoped `listPage()` path with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `cron.runs(scope: "all")` still builds the job-name map from an unfiltered cron list and the all-run log reader exposes that map in search text and returned `jobName` fields.

## Solution Assessment

Yes. Reusing the existing scoped `listPage()` contract for name enrichment is the narrow maintainable fix for the reported leak while preserving the explicitly separate all-scope run-log behavior.
