## Summary

Keep this issue open: current main and v2026.6.8 still build all-scope `cron.runs` job-name enrichment from an unfiltered cron job list, and the linked fixing PR remains open and unmerged.

## What This Changes

The issue requests that `cron.runs` with `scope: "all"` only enrich returned run-log rows with cron job names visible to the calling agent/session.

## Best Possible Solution

Keep this issue as the canonical tracker until the linked PR is rebased, reviewed, and landed, or replaced by a current-main fix that scopes all-scope job-name enrichment and proves non-visible names are omitted.

## Reproduction Assessment

Yes, at source level: current main builds the all-scope `cron.runs` name map from unfiltered `context.cron.list`, and the run-log reader stamps names from that map onto returned entries. I did not run a live repro because this review was read-only.

## Solution Assessment

Yes for the narrow direction: reusing the filtered paginated job listing path for job-name enrichment is the maintainable fix. The best current path is to review, rebase, and land the linked PR or replace it with an equivalent current-main patch.
