## Summary

Keep open. Current main still has source-level gaps in compaction retry delivery and active-turn resume semantics, and newer related reports were closed back into this issue as the canonical compaction-resume tracker rather than proving it fixed.

## What This Changes

The issue reports that auto-compaction during an active turn can interrupt or resume incorrectly and leave the user without a completed visible answer.

## Best Possible Solution

Define one shared compaction-resume contract that preserves active turn identity, originating delivery context, retry-produced output flushing, and pending user-facing reply obligations across auto-compaction and internal continuations.

## Reproduction Assessment

No, not as a high-confidence live current-main reproduction. The issue has credible release-era logs and current source still exposes the retry/wait/delivery and runtime-continuation gaps, but I did not run a live high-context channel reproduction.

## Solution Assessment

No; the suggested single-path fixes are too narrow for the observed issue family. The maintainable direction is a shared active-turn and delivery-resume contract, with narrower delivery flush work tracked in https://github.com/openclaw/openclaw/issues/47335 and retry identity work in https://github.com/openclaw/openclaw/issues/66126.
