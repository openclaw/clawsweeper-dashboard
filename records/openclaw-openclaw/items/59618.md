## Summary

Keep open. Current main has partial compaction-recovery mitigations, but it still does not prove a shared active-turn resume and delivery contract for the reported mid-turn abandonment family; narrower delivery and retry-identity variants remain open in https://github.com/openclaw/openclaw/issues/47335 and https://github.com/openclaw/openclaw/issues/66126.

## What This Changes

The issue reports auto-compaction interrupting an in-flight agent turn so the final user-visible answer is abandoned or not delivered after compaction.

## Best Possible Solution

Define one shared compaction-resume contract that preserves active turn identity, originating delivery context, retry-produced output flushing, and pending user-facing reply obligations across auto-compaction and internal continuations.

## Reproduction Assessment

No, not as a high-confidence live current-main reproduction. The reporter logs are credible and current source still exposes the retry/wait/delivery and continuation-state gaps, but this review did not run a long-context channel repro.

## Solution Assessment

No; the issue body's single-path suggestions are too narrow for the observed family. The maintainable fix is a shared active-turn and delivery-resume contract, with narrower delivery flushing and retry identity work coordinated through the linked open reports.
