## Summary

Keep open. The fresh shipped-version reproduction plus current source still point to an unresolved compaction-resume/delivery failure family: current main has partial continuation safeguards, but not a durable reply obligation or proven delivery barrier for post-compaction tool-loop output.

## What This Changes

Automatic compaction during an active turn should resume the interrupted work and deliver the final user-visible reply instead of leaving the session silent or replacing the turn with an internal continuation.

## Best Possible Solution

Land a regression-backed compaction-resume fix that preserves active turn identity, records when a visible reply is still owed, and proves retry-produced tool events plus final output reach the active channel.

## Reproduction Assessment

No, not as a live current-main reproduction in this read-only pass. The issue has credible 2026.5.7 logs and current source still exposes the retry/delivery gaps, but the full high-context channel scenario was not executed here.

## Solution Assessment

Unclear. The current partial continuation retry is not enough for the reported tool-loop and delivery-disappearance cases; maintainers should choose a shared turn-state and delivery-barrier contract rather than a one-off resume patch.
