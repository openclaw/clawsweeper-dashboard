## Summary

Current main now covers part of the proposal through ClawHub trust-envelope verification and operator-owned install policy, but it still lacks a standardized skill capability declaration, install-time permission consent, and skill-scoped runtime authorization. Keep this issue open for explicit product/security consolidation with the narrower manifest RFC and the broader enforcement tracker; the latest maintainer discussion also expressly chose continued follow-up rather than cleanup closure.

## What This Changes

Define a standardized skill permission manifest with install-time permission review, auditing, trust scoring, signing, and attestation support.

## Best Possible Solution

Keep the existing ClawHub trust envelope and operator install-policy seams, adopt one versioned skills capability vocabulary, then stage inspection, consent, persisted grants, runtime enforcement, and attestation as separately reviewed security milestones.

## Reproduction Assessment

Not applicable: this requests a new skill security, consent, and trust contract rather than reporting failure of an established behavior.

## Solution Assessment

The defense-in-depth direction is sound, but the proposed all-in-one scope is not the best implementation unit; maintainers should approve a canonical declaration vocabulary first and separately define consent, enforcement, and cryptographic trust semantics.
