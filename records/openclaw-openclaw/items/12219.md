## Summary

Keep open. Current main and v2026.5.22 still rely on SKILL.md/frontmatter skills, ClawHub scan visibility, and generic install/tool-call hooks, but do not implement the requested standardized skill.yaml permission manifest, install-time consent, skills audit/trust grades, signing, attestation, or ClawHub trust badges.

## What This Changes

Request a standardized skill.yaml permission manifest for skills, with install-time review, audit/trust scoring, signing or attestation, and ClawHub trust-badge integration.

## Best Possible Solution

Design a maintainer-owned, versioned skill permission or capability manifest that integrates with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.
