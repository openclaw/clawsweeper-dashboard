## Summary

Keep open. Current main and v2026.5.19 still have SKILL.md/frontmatter skills, ClawHub install paths, scan visibility, and generic install/tool-call hooks, but not the requested standardized `skill.yaml` permission manifest, install-time consent/audit flow, trust grades, signing, or attestation; the external `@ellistevo/openclaw-secure`, shell-security-ultimate ClawHub skill, and globalcaos/tinkerclaw references are useful prior art rather than a core implementation.

## What This Changes

The issue requests a standardized `skill.yaml` permission manifest with install-time review, audit/trust scoring, signing, and attestation for skills.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission/capability manifest integrated with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.
