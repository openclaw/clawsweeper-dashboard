## Summary

Keep open. Current main and the latest shipped release still expose SKILL.md/frontmatter metadata, ClawHub scan visibility, and generic install/tool-call hooks, but not the requested standardized skill permission manifest, install-time consent, audit/trust reporting, signing, or attestation contract; the live thread also has maintainer keep-open context and external prior art rather than a settled duplicate.

## What This Changes

The issue requests a standardized `skill.yaml` permission manifest for skills, including install-time permission review, audit/trust scoring, signing, and attestation.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission/capability manifest integrated with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.
