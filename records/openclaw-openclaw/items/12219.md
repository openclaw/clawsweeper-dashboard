## Summary

Keep open. Current main and v2026.6.5 have SKILL.md/frontmatter skills, ClawHub verification/scanning, install policy hooks, and generic tool/install hooks, but not the requested skill.yaml permission manifest, install-time consent review, skills audit/trust grading, signing, or attestation standard.

## What This Changes

Request a standardized skill.yaml manifest for skills to declare permissions, with install-time review, audit/trust scoring, signing, and attestation integration.

## Best Possible Solution

Design a versioned, skills-first permission/capability manifest with ClawHub and direct-install visibility, consent/audit/trust integration, signing or attestation expectations, and a clear relationship to before_install and before_tool_call hooks.

## Reproduction Assessment

Not applicable: this is a feature/security design proposal, not a bug with a current expected behavior. Source and docs inspection do confirm the requested manifest, consent, audit, signing, and attestation surfaces are missing.

## Solution Assessment

Yes directionally, but not as a single automated implementation. The maintainable path is staged maintainer product/security design, starting with a public schema and diagnostics before consent, enforcement, trust grading, signing, or attestation behavior.
