## Summary

Keep open. Current main still lacks the requested standardized skill permission manifest, install-time consent review, audit/trust reporting, signing, and attestation contract; the issue discussion also has useful external prior art rather than a merged replacement.

## What This Changes

Request a standardized `skill.yaml` permission manifest with install-time review, audit/trust reporting, signing, and attestation for skills.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission manifest that integrates with SKILL.md skills, ClawHub install/update review, CLI audit/inspection, signing/attestation expectations, and documented runtime-enforcement boundaries.

## Reproduction Assessment

Not applicable: this is a feature/security design request, not a regression. Static inspection on current main gives a high-confidence verification path that the requested manifest, consent, audit/trust, signing, and attestation surfaces are absent.
