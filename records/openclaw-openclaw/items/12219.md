## Summary

Keep open. Current main and v2026.6.5 still provide SKILL.md/frontmatter skills, ClawHub verification/scanning, generic install policy, and before_install/before_tool_call hooks, but not the requested skill.yaml permission manifest, install-time consent review, skills audit/trust grades, signing, or attestation standard.

## What This Changes

The issue requests a standardized skill.yaml permission manifest with install-time review, audit/trust reporting, signing, and attestation support for skills.

## Best Possible Solution

Design a versioned, skills-first permission/capability manifest with ClawHub and direct-install visibility, consent/audit/trust integration, signing or attestation expectations, and a clear relationship to before_install and before_tool_call hooks.

## Solution Assessment

Yes directionally, but not as a single automated implementation. The maintainable path is staged maintainer product/security design, likely starting with a public schema and diagnostics before consent, enforcement, trust grading, signing, or attestation behavior.
