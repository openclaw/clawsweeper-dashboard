## Summary

Keep open: current main and v2026.6.8 still provide SKILL.md/frontmatter skills, ClawHub verification, install policy hooks, and generic before_install/before_tool_call seams, but not the requested skill.yaml permission manifest, install-time consent review, skills audit/trust grades, signing, or attestation contract.

## What This Changes

The issue requests a standard skill permission manifest format for skills, including declared permissions, install review, audit/trust reporting, signing, and attestation.

## Best Possible Solution

Keep this open for explicit maintainer security/product design: choose a versioned skill permission vocabulary, define how it relates to ClawHub verification and runtime hooks, and phase implementation through parser/types, install/update UX, audit/trust output, and signing/attestation policy.
