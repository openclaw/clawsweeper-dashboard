## Summary

Current main has strengthened skill-package trust and generic install/tool-call policy hooks, but it still lacks the requested standardized capability declaration, install-time permission consent, skill-scoped authorization, audit grades, signing, and attestation. Keep this security-sensitive product proposal open, while maintainers consolidate it with the narrower manifest RFC and broader enforcement tracker rather than starting an autonomous implementation.

## What This Changes

Define a versioned skill permission manifest with install review, auditing, signing, attestation, and runtime-enforcement integration.

## Best Possible Solution

Adopt one canonical, versioned skills-first capability vocabulary, integrate it with ClawHub and direct install/update inspection, then add consent and runtime enforcement only after maintainers define compatibility, trust, revocation, and migration semantics across the existing hooks.

## Reproduction Assessment

Not applicable: this is a defense-in-depth feature proposal, and current-main source inspection directly confirms the requested manifest, consent, and enforcement contract is absent.

## Solution Assessment

Unclear as proposed: a separate skill.yaml plus grades, signing, and enforcement is too broad to adopt wholesale; the narrower visibility-first RFC is a better schema-design starting point, with runtime authorization handled as a coordinated follow-up.
