## Summary

Keep open. Current main and v2026.6.10 have SKILL.md/frontmatter skills, ClawHub verification/trust surfaces, install-policy hooks, and generic before_install/before_tool_call seams, but they still do not implement the requested versioned skill.yaml permission manifest with install consent, skills audit/trust grades, signing, and attestation.

## What This Changes

The issue requests a standardized skill.yaml permission manifest plus install-time review, audit/trust reporting, signing, and attestation for skills.

## Best Possible Solution

Keep this open for maintainer security/product design around one versioned skill permission contract coordinated with ClawHub verification, install/update consent, audit/trust reporting, signing, attestation, and runtime enforcement hooks.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 expose SKILL.md metadata, ClawHub verification, install policy, and hooks, but no standardized skill.yaml permission manifest or skills audit/consent workflow.

## Solution Assessment
