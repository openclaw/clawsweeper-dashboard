## Summary

Keep open. Current main still has `SKILL.md`/frontmatter skills, ClawHub trust-envelope verification, heuristic skill scanning, and generic install/tool-call hooks, but it does not implement the requested standardized `skill.yaml` permission manifest, install-time consent, `skills audit` trust grades, local signing verification, or attestation chain.

## What This Changes

The issue requests a standardized `skill.yaml` permission manifest for skills, plus install-time review, auditing, trust scoring, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design of a versioned skill permission manifest, with install/update consent, ClawHub/direct-install verification, audit/trust reporting, signing/attestation expectations, and a clear relationship to `before_install` and `before_tool_call` hooks.
