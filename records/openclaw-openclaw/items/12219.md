## Summary

Keep open. Current main still lacks the requested standardized skill.yaml permission manifest, install-time consent review, audit/trust reporting, signing, and attestation contract; the newer inspect-only capability-manifest RFC is related but narrower and does not supersede this broader product/security design.

## What This Changes

Request a standardized skill permission manifest with declared network, filesystem, shell, and credential permissions plus install review, audit/trust scoring, signing, attestation, and ClawHub integration.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission manifest that integrates with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing/attestation expectations, and documented runtime-enforcement boundaries.
