## Summary

Keep open: current main and v2026.6.6 have SKILL.md/frontmatter skills, ClawHub verification, install policy, and generic install/tool-call hooks, but they still do not implement the requested skill.yaml permission manifest standard, install-time consent review, skills audit/trust grades, signing, or attestation contract.

## What This Changes

The issue requests a standardized skill permission manifest (`skill.yaml`) with permission declarations, install-time review, audit/trust reporting, signing, and attestations.

## Best Possible Solution

Keep this open for explicit maintainer product/security design of one versioned skill permission manifest contract coordinated with ClawHub verification, install/update UX, audit/trust reporting, signing, attestations, and runtime enforcement hooks.

## Reproduction Assessment

Not applicable as a runtime bug: this is a feature/security design request. Source inspection does show the requested manifest, consent, audit, signing, and attestation surfaces are absent from current main and v2026.6.6.

## Solution Assessment
