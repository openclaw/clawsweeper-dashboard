## Summary

Keep open. Current main still has SKILL.md/frontmatter skills, ClawHub scan visibility, source-install scanning, and generic install/tool-call hooks, but not the requested standardized skill.yaml permission manifest with install-time consent, skills audit/trust grades, signing, attestation, or ClawHub trust badges. The linked openclaw-secure package, shell-security-ultimate ClawHub skill, globalcaos/tinkerclaw fork, and related open RFCs are useful prior art rather than a settled OpenClaw implementation.

## What This Changes

The issue requests a standardized skill permission manifest, proposed as skill.yaml, with declared permissions, install-time review, audit/trust scoring, signing, attestation, and ClawHub trust visibility.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission/capability manifest integrated with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.

## Solution Assessment
