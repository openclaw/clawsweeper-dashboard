## Summary

Keep open. Current main still has SKILL.md/frontmatter skills, ClawHub scan visibility, and generic before_install/before_tool_call hooks, but not the requested standardized skill.yaml permission manifest, install-time consent/audit flow, trust grades, signing, or attestation. The linked openclaw-secure package, shell-security-ultimate ClawHub skill, and globalcaos/tinkerclaw fork are useful prior art rather than a settled core implementation.

## What This Changes

The issue requests a standardized skill.yaml permission manifest so skills can declare required network, filesystem, shell, and credential access before install or use.

## Best Possible Solution

Keep this open for a maintainer-designed, versioned skill permission/capability manifest integrated with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.
