## Summary

Keep open. Current main and the latest release still document and install SKILL.md/frontmatter skills with ClawHub scan visibility and generic hooks, but they do not provide the requested skill.yaml permission manifest, install-time consent, skills audit/trust grades, signing, attestation, or ClawHub trust badges.

## What This Changes

Request a standardized skill.yaml permission manifest for skills, plus install/audit/ClawHub integration for user-visible permission review, trust scoring, signing, and attestation.

## Best Possible Solution

Design a maintainer-owned, versioned skill permission or capability manifest that integrates with SKILL.md skills, ClawHub install/update review, audit/trust reporting, signing or attestation expectations, and documented runtime-enforcement boundaries.

## Reproduction Assessment

Not applicable: this is a feature/security design request, not a runtime regression. Static inspection of current main and v2026.5.22 gives a high-confidence absence check for the requested manifest, consent, audit/trust, signing, and attestation surfaces.

## Solution Assessment

Unclear whether a separate skill.yaml file is the final best contract. The maintainable path is explicit product/security design coordinated with related capability-manifest, runtime-enforcement, provenance, signing, and ClawHub ecosystem threads.
