## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop trusting caller-supplied announce metadata:** `src/agents/tools/sessions-send-tool.ts:445`
  - body: In delegated mode this passes `hookTask?.runtime?.roundOneReply`, parsed from raw tool args, into `startA2AFlow`. A caller can seed A2A ping-pong or announce content without it coming from the delegated run or plugin-owned result; derive that seed only from hook output or verified reply history.
  - confidence: 0.87
- **[P2] Invoke the hook before reading reply history:** `src/agents/tools/sessions-send-tool.ts:414`
  - body: For waited sends, this path reads the baseline `chat.history` snapshot before `runSessionsSend`; if history access fails, a plugin that would handle direct or delegated delivery never runs. Move the core baseline read into fallback or delegated wait handling so hook-owned delivery is not blocked by core history access.
  - confidence: 0.8
- **[P2] Keep requester identity host-owned:** `src/agents/tools/sessions-send-tool.ts:141-150`
  - body: These fields prefer `params.task.requester.*` over the trusted requester values from the tool context. A caller can spoof requester metadata exposed to plugins; populate `task.requester` only from host-owned defaults, or remove it and rely on hook context for trusted identity.
  - confidence: 0.84
- **[P3] Export the hook task and cancel-target types:** `src/plugin-sdk/core.ts:102-106`
  - body: The public event/result types refer to `PluginHookSessionsSendTask` and `PluginHookSessionsSendCancelTarget`, but `plugin-sdk/core` only re-exports context/event/result. Plugin authors cannot name the full contract from the same SDK surface without type gymnastics.
  - confidence: 0.75
