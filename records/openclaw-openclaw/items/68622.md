## Summary

Keep this PR open. Current main still lacks a `sessions_send` plugin hook, so the contribution is not obsolete, but this branch should not merge as-is because delegated handling trusts caller-supplied announce metadata and invokes the hook only after a core history read; the open replacement PR #75369 addresses the main blockers but is not merged yet.

## What This Changes

The branch adds a typed `sessions_send` plugin hook to the hook contracts, runner, and Plugin SDK exports, wires `sessions_send` to honor direct or delegated hook results before core fallback, updates the generated SDK API baseline hash, and adds sessions tool tests for declined and delegated hook paths.

## Best Possible Solution

Land a rebased implementation, either by updating this branch or using #75369 plus the missing subtype exports, that adds the generic typed hook seam, invokes hook-owned handling before core-only history reads, seeds delegated A2A only from plugin-owned dispatch output or verified run history, and preserves current main A2A provenance, ACP-skip, and nested-lane behavior.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: a delegated hook with `timeoutSeconds: 0` and caller args containing `task.runtime.roundOneReply` reaches `startA2AFlow` in this PR, and current A2A code treats that value as the primary/latest reply. The hook-order bug is reproduced by making the pre-hook `chat.history` baseline read fail before a plugin direct/delegated handler can run.

## Solution Assessment

No. The hook direction is plausible and useful, but this branch is not the best merge shape until the trust boundary, hook ordering, current-main rebase preservation, and public subtype exports are all handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Stop trusting caller-supplied announce metadata:** `src/agents/tools/sessions-send-tool.ts:445`
  - body: In delegated mode this passes `hookTask?.runtime?.roundOneReply`, parsed from raw tool args, into `startA2AFlow`. Since tool args can include extra fields, a caller can seed A2A ping-pong or announce content without it coming from the delegated run or plugin-owned result; derive the seed from hook output or waited reply history instead.
  - confidence: 0.86
- **[P2] Invoke the hook before reading core reply history:** `src/agents/tools/sessions-send-tool.ts:414`
  - body: For waited sends, the code still takes `baselineReply` before `runSessionsSend`; if `chat.history` fails, a plugin that would handle direct or delegated delivery never runs. Move the core baseline read into the fallback or delegated wait path so hook-owned delivery is not blocked by core history access.
  - confidence: 0.78
- **[P3] Export the hook task and cancel-target types:** `src/plugin-sdk/core.ts:102-106`
  - body: The new public event/result types refer to `PluginHookSessionsSendTask` and `PluginHookSessionsSendCancelTarget`, but `plugin-sdk/core` only re-exports context/event/result. Plugin authors cannot name the task or cancel-target contract from the same SDK surface without type gymnastics.
  - confidence: 0.74
