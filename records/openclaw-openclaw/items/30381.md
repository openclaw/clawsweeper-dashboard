## Summary

Keep open. Current main and the latest release still select the explicit agent header, then reject a generic OpenAI request-body model before dispatch; the production SlurpBot Ops comments support that this is still an active compatibility bug, and the related PR is closed unmerged.

## What This Changes

The issue requests that `/v1/chat/completions` ignore the request-body `model` when a valid `x-openclaw-agent-id` selects an agent, using that agent's configured model instead.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative for agent choice, skip request-body model validation only in that case, and keep backend model overrides in `x-openclaw-model` across OpenAI-compatible handlers.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` reaches `resolveOpenAiCompatModelOverride`, which rejects the generic model before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving `x-openclaw-model` and invalid explicit selector behavior.
