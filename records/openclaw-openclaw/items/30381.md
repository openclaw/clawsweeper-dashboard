## Summary

Keep open. Current main and the latest shipped release still select the explicit agent header first, then reject a generic OpenAI request-body model before dispatch, so the reported OpenAI-compatible routing bug remains valid.

## What This Changes

The issue asks Chat Completions requests with `x-openclaw-agent-id` to ignore a generic request-body `model` and use the selected agent's configured model.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative, skip body-model agent-selector validation only in that case, and keep backend overrides in `x-openclaw-model` across Chat Completions and Responses.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` selects the header agent, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving `x-openclaw-model` and invalid explicit selector behavior.
