## Summary

Keep open. Current main still rejects generic OpenAI-compatible body model values before dispatch even when an agent header has selected the OpenClaw agent, and recent issue discussion confirms this is still affecting a production integration. The bug is narrow and actionable.

## What This Changes

The request is to ignore the chat-completions request-body model when x-openclaw-agent-id selects an agent, using the selected agent's configured model unless x-openclaw-model explicitly overrides it.

## Best Possible Solution

Gateway OpenAI-compatible HTTP should treat the body model as an agent selector only when no explicit agent header selected the agent; with an agent header it should route to that agent and use its configured model, while keeping x-openclaw-model as the only backend override and covering the chosen chat/responses behavior in tests and public docs.
