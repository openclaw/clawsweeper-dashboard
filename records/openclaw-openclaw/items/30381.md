## Summary

Keep open. Current main and the latest release still select the header agent, then reject a generic body model before dispatch, and the issue discussion includes repeated active production confirmations.

## What This Changes

The issue asks the OpenAI-compatible Chat Completions path to ignore the request body model when a valid x-openclaw-agent-id header selects an agent.

## Best Possible Solution

Make a valid explicit agent header authoritative for agent routing, treat the body model as OpenAI SDK compatibility input on that path, preserve x-openclaw-model for backend overrides, and cover both Chat Completions and Responses with focused regression tests.

## Reproduction Assessment

Yes, source-level. A request with x-openclaw-agent-id selecting beta and body model gpt-4 selects beta first, then resolveOpenAiCompatModelOverride rejects gpt-4 before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving default-agent aliases, x-openclaw-model, and invalid explicit selector behavior.
