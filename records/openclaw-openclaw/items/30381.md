## Summary

Keep open. Current main and the latest release still let a valid agent header choose the agent, but then validate the body `model` as an OpenClaw agent selector and can reject generic SDK model IDs before dispatch.

## What This Changes

The issue asks Chat Completions to treat a valid `x-openclaw-agent-id` as authoritative and ignore generic request-body `model` IDs on that path.

## Best Possible Solution

Make a valid explicit agent header authoritative for agent routing, treat the body `model` as OpenAI SDK compatibility input on that path, preserve `x-openclaw-model` for backend overrides, and cover both Chat Completions and Responses with focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows a valid `x-openclaw-agent-id` can select the agent, but the same request still validates a generic body `model` through `resolveOpenAiCompatModelOverride` and can return HTTP 400 before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid explicit selector behavior.
