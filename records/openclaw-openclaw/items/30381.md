## Summary

Keep open. Current main still selects the header agent first, then rejects a generic request-body `model` through the shared OpenAI compatibility validator before dispatching the chat completion.

## What This Changes

The issue requests that `/v1/chat/completions` ignore the request-body `model` when a valid `x-openclaw-agent-id` header selects an agent.

## Best Possible Solution

Preserve agent-first routing, skip body-model agent-selector validation only after a valid explicit agent header has selected the agent, keep `x-openclaw-model` as the backend override path, and add regression coverage for chat completions plus the shared Responses helper path.

## Reproduction Assessment

Yes. The current source path is enough to reproduce: send `/v1/chat/completions` with a valid `x-openclaw-agent-id` and a generic body `model` such as `gpt-4`; the header selects the agent, then the body model triggers the invalid-model 400 before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to make the OpenAI compatibility model-validation path aware of a valid explicit agent header while preserving existing rejection when no valid header selected the agent.
