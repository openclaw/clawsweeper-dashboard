## Summary

Keep open. Current main and v2026.5.20 still select the explicit agent header first, then reject a generic request body model before dispatch; the related open PR covers unknown-agent rejection, not this valid-header compatibility case.

## What This Changes

The issue asks the OpenAI-compatible Chat Completions endpoint to ignore the request body model when a valid `x-openclaw-agent-id` header selects the agent.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative while preserving `x-openclaw-model`, `openclaw/default`, default-agent routing, and unknown-agent validation across Chat Completions and Responses.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` selects the header agent, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid explicit selector behavior.
