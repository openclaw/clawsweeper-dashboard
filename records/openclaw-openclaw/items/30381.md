## Summary

Keep open. Current main and the latest shipped release still route the explicit agent header first but then reject a generic OpenAI request-body model before dispatch; the related open PR addresses unknown-agent rejection, not this valid-header compatibility case.

## What This Changes

The issue requests that Chat Completions requests with a valid `x-openclaw-agent-id` header ignore the request body `model` selector and use the targeted agent's configured model.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative while preserving `x-openclaw-model`, agent-target model selectors, default-agent routing, and unknown-agent rejection across Chat Completions and Responses.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` selects the header agent, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving `x-openclaw-model` and invalid explicit selector behavior.
