## Summary

Keep open. Current main and v2026.5.22 still select the header agent, then reject a generic request-body model before dispatch, so the reported OpenAI-compatible routing bug remains valid and narrow.

## What This Changes

Requests to `/v1/chat/completions` with `x-openclaw-agent-id` should ignore the request body `model` value and use the selected agent's configured model.

## Best Possible Solution

Make a resolved explicit agent header authoritative for agent selection, skip body-model selector validation only for that selected-agent case, and preserve `x-openclaw-model` as the backend model override path.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` selects the header agent, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving `x-openclaw-model` and invalid explicit selector behavior.
