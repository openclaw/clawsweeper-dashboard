## Summary

Keep open. Current `main` and release `v2026.5.20` still validate the request body `model` after resolving the explicit agent header, so a generic OpenAI SDK model can still be rejected before dispatch; the related open PR at https://github.com/openclaw/openclaw/pull/45344 covers unknown-agent rejection, not this header-precedence compatibility bug.

## What This Changes

The issue asks the OpenAI-compatible Chat Completions gateway to ignore the request body model when a valid `x-openclaw-agent-id` header selects an agent.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative while preserving `x-openclaw-model` backend overrides and unknown-agent validation across Chat Completions and Responses.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id` selecting an agent and body `model: "gpt-4"` selects the header agent first, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid explicit selector behavior.

## Repair Work Prompt

Fix the OpenAI-compatible gateway header-precedence bug for this issue. When a valid explicit `x-openclaw-agent-id` or `x-openclaw-agent` header selects an agent, do not reject or reinterpret a generic request body `model` such as `gpt-4`; use the selected agent's configured model unless `x-openclaw-model` is provided. Preserve the documented agent-target contract for `model: "openclaw"`, `model: "openclaw/default"`, `model: "openclaw/<agentId>"`, and `agent:<agentId>`, preserve `x-openclaw-model` backend override validation, and coordinate with https://github.com/openclaw/openclaw/pull/45344 so explicit unknown agents still fail once that validation lands. Add focused regression coverage for Chat Completions and Responses; do not add a config option or make the OpenAI body `model` a backend provider/model override. Update docs only if the implementation changes the documented contract.
