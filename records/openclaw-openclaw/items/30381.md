## Summary

Keep open. Current main still lets `x-openclaw-agent-id` select the agent, then rejects a generic OpenAI-compatible request-body `model` before dispatch, so the reported production compatibility problem remains valid and narrowly fixable.

## What This Changes

The issue asks Chat Completions to ignore the request-body `model` when a valid `x-openclaw-agent-id` header selects the agent, using the agent's configured model instead.

## Best Possible Solution

Make an accepted explicit agent header authoritative for agent selection, treat the body `model` as SDK compatibility input only on that path, preserve `x-openclaw-model` as the backend model override, and add focused Chat Completions plus Responses regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `x-openclaw-agent-id: beta` selects `beta`, then `model: gpt-4` is still validated by `resolveOpenAiCompatModelOverride` and rejected before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after an explicit valid agent header has selected the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid-selector behavior.
