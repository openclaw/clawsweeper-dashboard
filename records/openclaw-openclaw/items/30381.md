## Summary

Keep open. Current main and v2026.5.12 still select the header agent first, then validate the body `model` as an OpenClaw agent selector and can reject generic SDK model IDs before dispatch; the issue discussion also has repeated April 2026 confirmations that SlurpBot Ops still needs this fixed.

## What This Changes

The issue requests that OpenAI-compatible Chat Completions ignore the request body `model` when a valid `x-openclaw-agent-id` header selects an agent.

## Best Possible Solution

Make a valid explicit agent header authoritative for agent routing, treat the body `model` as OpenAI SDK compatibility input on that path, preserve `x-openclaw-model` for backend overrides, and cover both Chat Completions and Responses with focused regression tests.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id: beta` and `model: gpt-4` selects `beta` in `resolveAgentIdForRequest`, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` as an invalid OpenClaw agent selector before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid explicit selector behavior.
