## Summary

The reported behavior still exists, but current source, tests, and shipped documentation define it as an intentional agent-first contract: the body `model` must name an OpenClaw agent target, while `x-openclaw-model` controls backend-model overrides. Keep this open for a maintainer product decision on whether SDK compatibility warrants making the body model inert when an agent header is present.

## What This Changes

Allow requests with a valid `x-openclaw-agent-id` header to ignore a generic OpenAI body model and use the selected agent’s configured model.

## Best Possible Solution

If maintainers approve broader SDK compatibility, make a shared, documented precedence rule where a valid explicit agent header renders the body model routing-only and inert, while preserving strict agent aliases without a header and keeping `x-openclaw-model` as the sole validated backend-model override.

## Reproduction Assessment

Yes. Current source and shipped tests deterministically reject a generic body model even when a valid agent header independently selects the agent; this reproduces the report as intentional current behavior rather than a regression.

## Solution Assessment

Unclear. The proposed precedence rule is mechanically narrow, but it changes a deliberate shipped API contract, so maintainers must first decide whether compatibility with SDK-required placeholder models outweighs keeping one strict meaning for `model`.
