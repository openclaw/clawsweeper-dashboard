## Summary

Keep open. Current main and the latest v2026.5.19 release still select the explicit header agent, then validate the original OpenAI body `model` and reject generic SDK model IDs before dispatch; the issue also has repeated production confirmations and only an overlapping, still-open unknown-agent PR.

## What This Changes

Requests that Chat Completions ignore the OpenAI request-body `model` when `x-openclaw-agent-id` selects a valid agent, using that agent's configured model instead.

## Best Possible Solution

Make a valid explicit agent header authoritative for agent routing, treat the body `model` as OpenAI SDK compatibility input on that path, preserve `x-openclaw-model` for backend overrides, and cover both Chat Completions and Responses with focused regression tests.

## Reproduction Assessment

Yes, source-level. A request with `x-openclaw-agent-id: beta` and body `model: gpt-4` selects `beta`, then `resolveOpenAiCompatModelOverride` rejects `gpt-4` before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selected the agent, while preserving default-agent aliases, `x-openclaw-model`, and invalid explicit selector behavior.
