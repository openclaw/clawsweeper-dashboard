## Summary

Keep open. Current main and the latest release still choose the header-selected agent first, then reject a generic OpenAI request-body model before dispatch, so the reported OpenAI-compatible routing bug remains valid and narrow.

## What This Changes

The issue requests that OpenAI-compatible chat completions ignore the request body model when a valid x-openclaw-agent-id header selects an agent.

## Best Possible Solution

Make a valid explicit agent-header selection authoritative, skip body-model agent-selector validation only for that selected-agent case, and keep x-openclaw-model as the backend model override path.

## Reproduction Assessment

Yes, source-level. A request with x-openclaw-agent-id selecting an agent and body model: "gpt-4" selects the header agent, then resolveOpenAiCompatModelOverride rejects gpt-4 before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to skip body-model agent-selector validation only after a valid explicit agent header selects the agent, while preserving x-openclaw-model and invalid explicit selector behavior.
