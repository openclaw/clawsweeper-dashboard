## Summary

This PR should stay open: current main still lacks the requested inline tool-dispatch typing feedback, but the branch is conflicting, lacks real Telegram proof, and the proposed raw typing start bypasses the policy-aware typing path.

## What This Changes

The PR adds an awaited `typing.startTypingLoop()` before `tool.execute(...)` in the inline tool-dispatched skill path.

## Best Possible Solution

Add inline tool-dispatch typing through the existing policy-aware typing signaler, keep typing side effects off the tool-execution critical path, cover allowed/suppressed/failure cases, and attach redacted Telegram proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head adds a raw awaited typing start inside the tool execution `try`, while inline actions run before normal typing policy resolution. The intended visible Telegram improvement itself is not proven in a real setup.

## Solution Assessment

No. A raw `typing.startTypingLoop()` call in `handleInlineActions` is not the narrow maintainable solution because inline dispatch needs to preserve the same typing policy/signaler contract used by normal tool execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep typing failures from aborting tools:** `src/auto-reply/reply/get-reply-inline-actions.ts:246`
  - body: On the PR head, `startTypingLoop()` can reject through the typing start guard. Because this new await is inside the same `try` block as `tool.execute()`, a transient channel typing failure would return the error reply before the skill tool runs.
  - confidence: 0.88
- **[P2] Respect typing policy before starting inline typing:** `src/auto-reply/reply/get-reply-inline-actions.ts:246`
  - body: `handleInlineActions` starts typing unconditionally even though the normal run path resolves suppression and typing mode later. Tool-dispatched commands can therefore emit typing for paths configured to stay silent, such as `typingMode: "never"`, explicit suppression, heartbeat, system, or internal-webchat runs.
  - confidence: 0.91
