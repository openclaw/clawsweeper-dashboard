## Summary

Keep open: current main still lacks the requested feedback path, but this PR should not merge as-is because the direct typing start bypasses established typing suppression policy and the branch is currently dirty against main.

## What This Changes

The PR adds an awaited typing-loop start before executing a tool-dispatched skill command in the inline action path.

## Best Possible Solution

Start typing for tool-dispatched skills through the existing policy-aware typing semantics, keep typing-start failures off the reply critical path, add focused coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: the PR inserts an unconditional `typing.startTypingLoop()` in the inline tool branch, while inline actions return before the normal run typing policy is resolved.

## Solution Assessment

No. The direct controller call is not the best fix; inline tool dispatch should use or receive the existing policy-aware typing signaler and cover both allowed and suppressed starts.
