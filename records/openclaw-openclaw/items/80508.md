The issue requests durable provenance tracking for generated Linux systemd gateway units so update-mode doctor can auto-update untouched generated units while preserving operator-modified or legacy units.

## Best Possible Solution

Land a corrected provenance implementation, through the linked PR or a replacement, that stores generated-unit ownership in an approved state layer, auto-updates only unchanged generated units, writes non-mutating suggested unit/env artifacts, and proves the Linux update path.
