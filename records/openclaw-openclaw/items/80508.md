## Summary

Keep open. This is protected by the `maintainer` label, and current main still has the conservative whole-unit rewrite/stage machinery but no durable generated-unit provenance or suggested-unit workflow for safe update-time rewrites.

## What This Changes

Add durable provenance records for generated Linux systemd gateway units so update-mode doctor can distinguish untouched generated units from operator-modified or legacy units.

## Best Possible Solution

Implement a state-backed unit provenance contract in gateway install and update-mode doctor, preserve modified or legacy units with suggested output and a diff, and keep explicit force reinstall as the replacement path.

## Reproduction Assessment

Not applicable as a bug reproduction for this issue: it requests a new provenance mechanism. Source inspection does show the missing capability in the current doctor/systemd write path.

## Solution Assessment

Unclear until maintainer direction settles the storage boundary. The requested provenance design is more complete than the linked conservative deferral, but it should be integrated with the broader state-store plan rather than added as an ad hoc sidecar.
