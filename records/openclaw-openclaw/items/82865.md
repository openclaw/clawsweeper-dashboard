## Summary

Keep open: current main still has the Docker image defaulting to `--allow-unconfigured`, and source inspection shows that flag bypasses the gateway.mode startup guard while container bind defaults can resolve to a non-loopback listener. The issue is also paired with the open closing PR https://github.com/openclaw/openclaw/pull/82866, so the right cleanup path is to review that PR or choose a maintainer-owned alternative before closing this issue.

## What This Changes

The issue asks OpenClaw to prevent Docker gateway startup from bypassing configuration and auth validation through `--allow-unconfigured` when the container may bind on a non-loopback interface.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/82866 is reviewed and either merged or replaced with a maintainer-approved fix that fails closed for Docker/non-loopback startup without durable local-mode config and explicit auth/trusted-proxy.

## Reproduction Assessment

Yes from source inspection, though I did not run Docker in this read-only review. The Docker CMD, container bind default, `--allow-unconfigured` guard bypass, and bootstrap-token exception give a high-confidence current-main reproduction path for the reported startup behavior.

## Solution Assessment

Unclear whether the linked PR is the final best fix because published cloud bootstrap docs also use `--allow-unconfigured` with non-loopback bind. The best direction is fail-closed non-loopback startup without explicit durable auth while preserving or intentionally replacing supported bootstrap flows.
