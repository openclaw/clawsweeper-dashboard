## Summary

Keep open: current main still ships the Docker image CMD with `--allow-unconfigured`, and the linked open contributor PR is the right place to review the fix before resolving this issue. The exact unauthenticated-access impact still needs live Docker validation because current startup also generates an ephemeral token and enforces non-loopback auth and Control UI origin guards.

## What This Changes

The issue asks OpenClaw to stop allowing the official Docker gateway image to start in an unconfigured bootstrap mode that can interact with network-facing container defaults.

## Best Possible Solution

Make container startup fail closed unless a valid auth path or a reviewed loopback-only bootstrap path is explicit, and add focused Docker/gateway startup coverage for that contract.

## Reproduction Assessment

No high-confidence live reproduction from this review. Source inspection reproduces the Dockerfile flag and startup-guard bypass, but the claimed unauthenticated network access needs a built-container run because token generation, non-loopback auth checks, and Control UI origin checks also execute.

## Solution Assessment

Unclear pending review of the linked PR. Constraining `--allow-unconfigured` for Docker/container startup is likely the right direction, but the final fix should preserve a deliberate bootstrap path and avoid breaking documented setup flows.
