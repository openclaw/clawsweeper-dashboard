## Summary

Keep open. Current main has partial label, derived-title, cleanup, and session-maintenance pieces, but the requested core UX is still not implemented: no `/session rename` or persisted auto-title flow, keep-mode completed subagents remain visible until general retention cleanup, and the Control UI still lacks channel/kind/status filtering or channel/kind grouping.

## What This Changes

The issue requests human-readable session names or auto-titles, automatic cleanup or retention for completed subagent sessions, and grouped/filterable session selection in the Control UI.

## Best Possible Solution

Split this into maintainer-scoped work: define the session naming/display contract first, choose completed-subagent retention and cleanup semantics, then add the matching Gateway/session APIs and Control UI channel/kind/status grouping or filters.

## Reproduction Assessment

Partially yes. Static inspection on current main reproduces the missing rename command, missing persisted auto-title path, explicit-only delete cleanup for run-mode subagents, and absent channel/kind/status filters; the May 2 comment provides live current-release confirmation, but I did not run a live Gateway session.

## Solution Assessment

Not applicable as a patch review. The requested direction is valid, but the best maintainable path is a split product/API decision followed by narrower implementation PRs rather than one broad cleanup patch.
