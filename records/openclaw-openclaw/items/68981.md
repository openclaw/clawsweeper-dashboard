## Summary

Keep open. Current main has useful partial session-label and explicit cleanup support, but the central enhancement remains unsolved: there is no `/session rename` or persisted auto-title flow, default keep-mode completed subagents still keep their session records visible, and the Control UI still lacks first-class channel/kind/status filtering or channel/kind grouping.

## What This Changes

Issue requests human-readable session naming or auto-title support, automatic cleanup of completed subagent sessions, and Control UI session filtering/grouping by channel, kind, status, or label.

## Best Possible Solution

Keep this as a core session-management enhancement, but split or explicitly scope the work: define a naming contract, decide whether auto-title generation is allowed and when it persists, choose the default retention/visibility policy for completed run-mode subagents, then add the matching sessions API and Control UI channel/kind/status filtering or grouping.

## Reproduction Assessment

Partially yes. Static inspection reproduces the missing `/session rename`, missing persisted auto-title path, explicit-only cleanup behavior, and absent Control UI channel/kind/status filters; the exact `archiveAfterMinutes` live timer behavior from the comment was not rerun here.

## Solution Assessment

Not applicable as a patch review. The requested direction is valid but too broad for one automatic cleanup fix; the maintainable route is a maintainer-scoped product/API decision followed by narrower implementation PRs.
