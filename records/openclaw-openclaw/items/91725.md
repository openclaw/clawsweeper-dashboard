## Summary

The central Apple `container` integration belongs as a ClawHub plugin: current main already exposes the sandbox registration, lifecycle-manager, workdir, and plugin-owned configuration seams, and a prior implementation exercised them against the real Apple runtime. The separate package setup-message mismatch is valid but should become a focused core issue rather than keeping this combined request open.

## What This Changes

Add an Apple `container` sandbox backend for macOS and correct the sandbox-image setup guidance for packaged installs.

## Best Possible Solution

Publish and maintain the Apple `container` backend as a ClawHub plugin using `openclaw/plugin-sdk/sandbox`, and file the packaged-install setup guidance mismatch as a separate narrow core issue.

## Reproduction Assessment

## Solution Assessment

No, bundling this vendor-specific backend in core is not the best path because the public plugin SDK and OpenShell precedent already provide the narrower ownership boundary; only a concrete generic SDK gap should return to core.
