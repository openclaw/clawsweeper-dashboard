## Summary

Keep open. Current main still blocks a read-scoped paired CLI before the same-device reject logic can run, and the related open PR only widens list/approve fallback behavior rather than adding a bounded recovery path for self-reject or narrower repair requests.

## What This Changes

The issue requests a supported recovery path for a read-scoped paired CLI stuck behind an over-scoped auto-reissued device repair request.

## Best Possible Solution

Define a bounded recovery contract that lets a limited paired CLI clear or replace only its own pending repair request, then align gateway authorization, CLI behavior, docs, and regression tests.

## Reproduction Assessment

Yes. A source-backed reproduction is a paired CLI session with only `operator.read`, a pending broader repair request for the same device, and `openclaw devices reject <requestId>` failing at global scope authorization before the same-device handler check.

## Solution Assessment

Unclear. The issue identifies a real deadlock, but the best fix needs maintainer/security choice between same-device self-reject, narrower repair-scope requests, or another bootstrap recovery flow.
