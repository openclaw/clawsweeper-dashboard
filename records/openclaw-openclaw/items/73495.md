## Summary

Keep open. Current main still has the Telegram doc/runtime mismatch and lacks the proposed `openclaw doctor` warning, so this PR remains a useful implementation candidate for the still-open paired bug. The latest PR head fixes earlier account-merge and sanitization concerns, but the new scanner still has two narrow runtime-parity gaps around legacy streaming-off aliases and an empty `accounts: {}` default-account setup.

## What This Changes

This PR adds Telegram and streaming documentation notes, a changelog entry, a Telegram `replyToMode` versus tool-progress doctor warning scanner with tests, and removes stale `await`s in active-memory tests.

## Best Possible Solution

Land this PR after tightening the doctor scanner to mirror Telegram runtime stream-mode and account-selection semantics, while keeping the runtime quote-reply exclusion unchanged and using docs plus `openclaw doctor` as the user-facing guardrail for the linked bug.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Telegram config with `replyToMode` not `off`, preview streaming active, and default tool-progress preview; current main creates no answer preview stream in that path, and the PR body includes matching live trace evidence.

## Solution Assessment

Yes, with small scanner repairs. A docs note plus a doctor warning is the narrowest maintainable fix for the silent doc/runtime mismatch; a runtime fallback that emits separate progress messages while quote-reply is enabled would be a broader product behavior change.
