## Summary

Keep open: current main still has the Telegram doc/runtime/doctor gap, and this PR is a focused implementation for the linked bug. The patch is mostly sound, but the new doctor scanner has two minor parity gaps around legacy streaming-off aliases and empty account maps that should be fixed or explicitly accepted before merge.

## What This Changes

The PR adds Telegram and streaming docs notes, a changelog entry, a new `openclaw doctor` warning scanner with tests for Telegram `replyToMode` and tool-progress preview conflicts, and removes stale awaits in active-memory tests.

## Best Possible Solution

Land the PR after tightening the scanner to match Telegram's runtime streaming/account resolution, keeping the runtime exclusion unchanged and using docs plus `openclaw doctor` as the user-facing guardrail. A separate product follow-up can decide whether Telegram should later add a non-preview fallback for tool-progress visibility while quote-reply is enabled.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Telegram config with `replyToMode` not `off`, preview streaming active, and default tool-progress preview; current main creates no answer preview stream in that path, and the PR body includes matching live trace evidence.

## Solution Assessment

Yes, with small scanner fixes. Docs plus a doctor warning are the narrowest maintainable solution for the silent doc/runtime mismatch; a runtime separate-message fallback would be a larger product behavior change.
