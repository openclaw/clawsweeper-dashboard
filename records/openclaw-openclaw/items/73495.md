## Best Possible Solution

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Telegram config with `replyToMode` not `off`, preview streaming active, and default tool-progress preview; current main creates no answer preview stream in the native quote path, and the PR body includes matching live trace evidence.

## Solution Assessment

Yes, with the scanner parity fixes above. Docs plus a doctor warning are the narrowest maintainable fix for the silent doc/runtime mismatch; a runtime fallback that emits separate progress messages while quote-reply is enabled would be a larger behavior change.

  - confidence: 0.89

  - body: When `accounts` is `{}`, this branch returns without evaluating `channels.telegram`. Runtime account selection falls back to the implicit `default` account when no account IDs are configured, so a top-level `replyToMode: "first"` conflict would remain silent in that default-account path.
  - confidence: 0.86
