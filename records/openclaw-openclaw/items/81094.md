## Summary

Keep open: the PR is a focused fix for a source-proven memory-file mismatch, but it leaves the gateway doctor backfill path internally inconsistent by listing slugged files and then dropping them during date extraction.

## What This Changes

The PR widens memory-core, gateway doctor, docs, tests, and changelog surfaces to accept `memory/YYYY-MM-DD-<slug>.md` daily memory filenames alongside date-only files.

## Best Possible Solution

Land the slugged daily-file matcher fix after making gateway doctor backfill use the same slug-aware date parser and adding focused doctor regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main writes `memory/YYYY-MM-DD-<slug>.md` from the session-memory hook while current consumers only accept `YYYY-MM-DD.md`; a focused Node regex check also shows the PR's widened doctor enumerator still feeds paths that the unchanged doctor date extractor rejects.

## Solution Assessment

No as submitted. Widening the existing matchers is the right narrow fix, but the gateway doctor backfill path needs a shared slug-aware matcher for both enumeration and day extraction before this is the best implementation.
