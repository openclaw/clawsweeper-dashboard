## Summary

Keep open; this is a recent focused bug-fix PR with sufficient real behavior proof, current main still lacks `attempt_dispatch` in the cron execution-start classifier, and I found no blocking review findings.

## What This Changes

The PR adds `attempt_dispatch` to the cron execution-start classifier and adds a regression test for isolated cron runs that pass the pre-execution watchdog window.

## Best Possible Solution

Land this narrow classifier and regression test, then keep any richer runner-side `model_call_started` telemetry work separate.

## Reproduction Assessment

Yes. Current source shows an isolated cron run can reach `attempt_dispatch` while the pre-execution watchdog remains armed, and the linked reports provide live beta.4 and beta.6 logs for that phase; I did not run a live cron job in this read-only review.

## Solution Assessment

Yes. Adding the existing `attempt_dispatch` phase to the private execution-start classifier is the narrowest maintainable unblocker for this regression; fuller runner telemetry can be handled separately.
