## Summary

Keep open: current main and v2026.6.8 still rely on the merged bucket workaround while preserving the same Discord 25-option, 5-row, and 100-character custom ID constraints, so the structural picker design problem remains. The merged related PR is useful partial relief, but no canonical follow-up has superseded this design tracker.

## What This Changes

The issue requests a Discord picker design follow-up for large wildcard model/provider configs constrained by Discord select-option, action-row, and custom ID limits.

## Best Possible Solution

Keep this open as the Discord picker design tracker and choose a durable search, modal, or short-lived state-token direction only if large-catalog user pain warrants expanding the picker UX.
