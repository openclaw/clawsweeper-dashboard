## Best Possible Solution

Recover lost-context runs only from a current-run terminal success signal, preferably the private transcript/status boundary used by https://github.com/openclaw/openclaw/pull/94420, and keep the lost-context fallback for display-only or non-terminal output.

## Reproduction Assessment

Yes, source-level: current main can reach the stale active-run lost-context fallback after session reconciliation returns null, while the announce path can still read child output. I did not run a live Agent Teams reproduction in this read-only pass.

## Solution Assessment

No. The bugfix direction is valid, but this patch uses display-oriented captured output as the success predicate; the safer repair is an explicit current-run terminal success signal such as private transcript or persisted session status.
