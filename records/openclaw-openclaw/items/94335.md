## Summary

## Best Possible Solution

Constrain lost-context recovery to an explicit current-run terminal-success signal, keep no-output/non-terminal/read-failure paths on the lost-context error, and require redacted real runtime proof before merge.

No. The bugfix direction is valid, but this patch uses display-oriented captured output as the success predicate; the safer repair is an explicit current-run terminal success signal such as private-transcript or persisted session status.
