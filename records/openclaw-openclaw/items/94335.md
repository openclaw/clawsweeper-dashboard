## Summary

Keep open: this targets a real Agent Teams lifecycle bug, but the patch turns any non-empty display-captured child output into a successful subagent completion without proving a current-run terminal success signal, and the supplied proof is only synthetic Vitest coverage.

## What This Changes

The branch adds a stale subagent sweep branch that completes a lost-context run as `ok` when captured child output is non-empty, plus registry tests for output, no-output, and capture-error cases.

## Best Possible Solution

Gate lost-context recovery on a current-run terminal success signal, preserve the lost-context fallback for non-terminal or unreadable cases, and require redacted real runtime proof before merge.

## Reproduction Assessment

Yes, source-level: current main can reach the stale-run lost-context fallback after session reconciliation returns null, while the announce path can still read child output. I did not run a live Agent Teams reproduction in this read-only pass.

## Solution Assessment

No. The bugfix direction is valid, but this patch uses display-oriented captured output as the success predicate; the safer repair is an explicit current-run terminal success signal such as private-transcript or persisted session status.
