## What This Changes

The branch adds a core resolver for channel/account `maxLinesPerMessage`, forwards it with `parseMode` to `sendDurableMessageBatch`, and adds `sendMessage` unit coverage.

## Best Possible Solution

Expose or reuse a channel-owned outbound formatting resolver so core forwards effective formatting without owning Discord config semantics, then prove the CLI send/readback path.

## Reproduction Assessment

Yes from source with high confidence: current `sendMessage` forwards only `parseMode`, while Discord chunks using `formatting.maxLinesPerMessage` or its 17-line default. I did not run a live Discord send in this read-only review.

## Solution Assessment

No. Forwarding effective max-lines formatting is the right direction, but resolving Discord-style account/default/fallback config in generic core is not the best boundary; the value should come from a channel-owned resolver or maintainer-approved shared seam.
