## Summary

Keep open. The PR targets a real P1 startup bug, but the live head still has two source-reproducible blockers and only mocked/unit proof instead of real gateway startup proof.

## What This Changes

Adds a gateway startup helper to install and enable the Codex runtime plugin for configured OpenAI model refs, wires it into plugin bootstrap, and adds focused mocked tests.

## Best Possible Solution

Keep the required native harness plugin startup path, but make it policy-respecting before any install, thread the adjusted activation source into deferred load/reload, and require redacted real startup proof.

## Reproduction Assessment

Yes. Current main and the PR head make the failure source-reproducible through OpenAI/Codex model routing, plugin startup planning, activation-source allowlists, and the deferred runtime loader; I did not run a live gateway in this read-only review.

## Solution Assessment

No. Installing before gateway plugin loading is the right shape, but this branch still installs when explicit plugin policy blocks activation and still leaves the real deferred runtime load using the raw activation source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Skip installs when plugin policy blocks Codex:** `src/commands/codex-runtime-plugin-install.ts:132-145`
  - body: When `plugins.enabled` is false or `plugins.deny` contains `codex`, `ensureCodexEnabledInStartupConfig` only logs and returns the original config. If the install record is missing, the caller still enters the npm install branch, so gateway startup can download/write `@openclaw/codex` even though Codex cannot activate. Return before invoking the installer when explicit plugin policy blocks Codex.
  - confidence: 0.9
- **[P2] Thread adjusted activation source into deferred loads:** `src/gateway/server-startup-plugins.ts:101`
  - body: The adjusted activation source only affects bootstrap planning. In the real server path, plugin runtime loading is deferred and `server.impl.ts` still passes the raw startup activation source to `loadGatewayStartupPluginRuntime`, so a restrictive allowlist can remove `codex` again before the harness registers. Return the adjusted activation source from bootstrap and use it for deferred load/reload paths.
  - confidence: 0.88
