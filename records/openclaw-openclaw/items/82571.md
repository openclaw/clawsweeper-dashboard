## Summary

Keep open: the follow-up commit fixed the prior prompter and activation-source issues, but the branch still runs a package installer from gateway startup, which can break immutable/Nix-managed installs and needs maintainer policy review plus real behavior proof.

## What This Changes

The PR adds a gateway-startup Codex runtime plugin installer/enabler for configured OpenAI model refs, updates activation-source allowlist handling, and adds focused unit coverage.

## Best Possible Solution

Keep plugin install/repair in onboarding, doctor, or update repair paths unless maintainers explicitly choose startup installs; rely on the current PI fallback and doctor repair for missing Codex states.

## Reproduction Assessment

Yes, by source inspection for the PR-introduced failure path: configured OpenAI model plus absent Codex reaches ensureOnboardingPluginInstalled during gateway startup, and that installer throws under OPENCLAW_NIX_MODE=1. I did not run a live gateway in this read-only review.

## Solution Assessment

No. Installing packages from gateway startup is not the narrowest maintainable fix while current main already has PI fallback and doctor repair; maintainers should decide the startup-side-effect policy before this approach can merge.
