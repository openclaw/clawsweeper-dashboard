## Summary

Keep this PR open. The reported Codex harness registration gap is source-reproducible on current main and the PR is aimed at the right gateway/plugin boundary, but the latest diff still has an over-broad startup install trigger and the contributor proof does not yet show the full gateway startup path.

## What This Changes

The PR adds a gateway startup path that installs/enables the Codex runtime plugin for configured OpenAI/OpenAI Codex model refs, threads the adjusted activation source through plugin loading, and adds focused tests.

## Best Possible Solution

Keep Codex plugin-owned, but narrow the startup check to configured agent-harness routes that can actually select Codex, then require redacted gateway startup proof showing install, load, and harness registration before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main can require the Codex harness for configured OpenAI/Codex agent runtimes while the only automatic install path is onboarding/model selection, so an upgraded gateway can reach harness selection without the plugin registered.

## Solution Assessment

No, not yet. The startup-bootstrap direction is right, but the implementation should not use the broad configured-model collector for a startup npm install because that includes non-agent OpenAI surfaces that do not need the Codex harness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Limit startup Codex installs to agent-harness models:** `src/commands/codex-runtime-plugin-install.ts:128-129`
  - body: The startup check uses `collectConfiguredModelRefs(params.cfg)` wholesale, but that collector includes non-agent refs such as image/video/music/pdf model fields and TTS summary models. A gateway configured only for direct OpenAI image or TTS usage would now install and enable `@openclaw/codex` at startup even though no Codex harness route can run. Please filter this to model refs that can actually drive agent harness selection, or resolve the effective harness policy before starting the managed install.
  - confidence: 0.84

## Security Review
