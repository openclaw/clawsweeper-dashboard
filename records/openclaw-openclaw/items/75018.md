  - body: Gradium defines `text` as a segment and emits separate `end_text`, `step`, `flushed`, and `end_of_stream` messages, while Voice Call treats `onTranscript` as a final user turn. Calling `onTranscript` for every `text` event can make the agent respond to partial speech or duplicate turns. ([docs.gradium.ai](https://docs.gradium.ai/api-reference/endpoint/stt-websocket))
  - confidence: 0.88
- **[P1] Require explicit Gradium STT opt-in:** `extensions/gradium/realtime-transcription-provider.ts:197-198`
  - body: `isConfigured` returns true from `GRADIUM_API_KEY` alone. Because Voice Call auto-selects the first configured realtime transcription provider when `streaming.provider` is unset, existing Gradium TTS users can route live call audio to Gradium without selecting Gradium STT or adding STT provider config.
  - confidence: 0.9
- **[P2] Remove the hidden Gradium endpoint fallback:** `extensions/gradium/realtime-transcription-provider.ts:93-94`
  - body: `GRADIUM_BASE_URL` is not declared in the Gradium manifest or docs, but this fallback can change the WebSocket origin for live audio and the `x-api-key` header. Keep endpoint override in explicit provider config, or deliberately declare, document, and test the env surface.
  - confidence: 0.88
