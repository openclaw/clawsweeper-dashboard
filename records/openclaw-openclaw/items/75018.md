## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wait for Gradium end-of-turn before committing transcripts:** `extensions/gradium/realtime-transcription-provider.ts:152`
  - body: The provider calls `onTranscript` for every Gradium `text` event, but Gradium's realtime API emits `end_text`, VAD `step`, and `flushed` to identify finalized segments and turn boundaries. Voice Call treats every `onTranscript` callback as a final committed user turn, so this can make agents answer partial segments or duplicate turns.
  - confidence: 0.9
- **[P1] Require explicit Gradium STT opt-in:** `extensions/gradium/realtime-transcription-provider.ts:197-198`
  - body: `isConfigured` returns true from `GRADIUM_API_KEY` alone. Because Voice Call auto-selects the first configured realtime transcription provider when `streaming.provider` is unset, existing Gradium TTS users can route live call audio to Gradium without selecting Gradium STT or adding STT provider config.
  - confidence: 0.9
- **[P2] Remove the hidden Gradium endpoint fallback:** `extensions/gradium/realtime-transcription-provider.ts:93-94`
  - body: `GRADIUM_BASE_URL` is not declared in the Gradium manifest or STT docs, but this fallback can change the WebSocket origin for live audio and the `x-api-key` header. Keep endpoint override in explicit provider config, or deliberately declare, document, and test the env surface.
  - confidence: 0.88
