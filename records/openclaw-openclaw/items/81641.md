## Summary

Keep open. The PR fixes a real CLI list/remove mismatch and includes sufficient terminal proof, but the current diff still misclassifies explicit empty aliases as materialized built-ins, conflicting with the existing defaults contract.

## What This Changes

The PR exports the default model alias table, adds a built-in-alias-specific error for `models aliases remove`, and adds focused alias command tests.

## Best Possible Solution

Keep the focused error-message fix, but make the built-in check match `applyModelDefaults` by requiring `alias === undefined` and add empty-alias regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `list` reads resolved config while `remove` mutates source config, so a configured built-in target with no source alias is listed but currently rejected as not found.

## Solution Assessment

No, not yet. The fix direction is narrow and maintainable, but the PR should use the same `alias === undefined` materialization contract as `applyModelDefaults` instead of a falsy alias check.
