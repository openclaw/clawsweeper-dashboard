The issue reports that Feishu `message` tool `.xlsx` attachments fail with a 400 invalid-parameter error during file upload.

## Best Possible Solution

Confirm the accepted Feishu `file_type` for `.xlsx`, then add a focused Feishu media regression and the narrowest routing or retry fix that preserves existing `.xls`, document, audio, video, image, and generic file sends.
