## Summary

Keep open: current main and the latest shipped release still route `.xlsx` Feishu message-tool attachments through the file upload path as `file_type: "xls"`, and the report describes a current 400 failure for that exact surface. The remaining uncertainty is the Feishu contract: official Feishu/Lark docs and SDK types document `xls`/`stream`, not `xlsx`, so the repair should be confirmed against live Feishu behavior before changing the upload mapping.

## What This Changes

The issue reports that Feishu `message` tool attachments fail for `.xlsx` files with a 400 invalid request parameter.

## Best Possible Solution

Confirm the accepted Feishu upload value for `.xlsx`, then add a focused regression and adjust only the spreadsheet upload routing or fallback while preserving existing `.xls`, image, audio, video, and document sends.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection confirms the reported `.xlsx` path currently uploads as `file_type: "xls"`, which is the suspicious parameter path, but Feishu's public docs and SDK types still document `xls`/`stream` rather than `xlsx`.

## Solution Assessment

Unclear as written because the reporter's successful `xlsx` upload conflicts with the official docs and SDK type contract. The maintainable fix is a narrow, tested routing or fallback change after confirming the real Feishu API behavior.
