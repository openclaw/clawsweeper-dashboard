## Summary

## What This Changes

The PR buffers block-kind replies in `createReplyDispatcher` until a final reply arrives, drops buffered blocks when the final is `NO_REPLY`, and adds regression tests for that turn-level suppression path.

## Best Possible Solution

Implement turn-level suppression while preserving media-bearing final normalization and the dispatcher `sendChain`/`waitForIdle` contract for block delivery, then validate with focused CI plus redacted real Telegram or gateway proof.
