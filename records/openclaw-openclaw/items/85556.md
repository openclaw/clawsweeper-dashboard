## Summary

Keep this PR open: it targets a real Claude CLI availability bug, but the patch starts the sub-agent monitor only from spawn argv, so fresh live sessions can still miss the actual Claude `session_id` and hit the same abort path.

## What This Changes

The PR adds a Claude CLI sub-agent transcript activity monitor, wires it into the Claude live-session lifecycle, and adds focused monitor tests.

## Best Possible Solution

Track the Claude CLI session id as live-session state and start the sub-agent monitor when either resume argv or parsed JSONL `session_id` supplies it, with regression coverage and redacted live proof for fresh and resumed sessions.

## Reproduction Assessment

Yes. The PR body provides before logs for the watchdog abort, and source inspection shows current main has no sub-agent `run.progress` path while the diagnostic watchdog relies on `lastProgressAgeMs`.

## Solution Assessment

No. Polling the sub-agent transcript is a reasonable narrow fix, but the implementation should be driven by the live session's actual parsed Claude session id rather than only by spawn argv.
