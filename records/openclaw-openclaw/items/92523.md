## Label Justifications

- `P1`: The issue describes an unattended agent workflow that can stop producing heartbeat turns until manual database and gateway intervention.
- `impact:crash-loop`: The reported failure is a persistent availability deadlock where the gateway keeps accepting wake events but the affected agent does not dispatch new turns.
- `impact:session-state`: The suspected root is stale persisted task-flow/session busy state that outlives the real callback lifecycle and blocks later session work.

## Summary

Keep this issue open: current main still leaves non-terminal TaskFlows in `waiting`, the built-in maintenance path only reconciles cancelled/terminal/lost flows, and an open fix PR now owns the proposed expiry work. The reporter's clarification that `openclaw tasks maintenance --apply` cannot clear `waiting` rows matches current source, so the issue remains a credible P1 availability bug rather than cleanup-closeable noise.

## What This Changes

The issue requests expiry or cleanup for orphaned TaskFlows stuck in `waiting` so they cannot permanently hold busy state and suppress future agent heartbeats.

## Best Possible Solution

Review and land a narrow task-flow lifecycle fix that expires genuinely orphaned `waiting` flows through the core registry/maintenance owner, with regression coverage for fresh versus stale waiting flows and upgrade-safe behavior for existing state rows.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not fully: current source confirms `waiting` TaskFlows are never expired by maintenance, but this review did not reproduce the reporter's live gateway deadlock or prove the exact `flow_runs` to heartbeat busy-counter path end to end.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer review of the timeout contract: task-flow-owned expiry is the right boundary, but the timeout duration, terminal status, startup behavior, and any final-delivery coupling need to be settled before treating the open PR as the final fix.
