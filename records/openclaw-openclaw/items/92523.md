## Summary

Keep this open: current main and the latest release still leave `waiting` TaskFlows out of maintenance recovery, and the open linked fix PR has not merged yet. ## What This Changes

The issue requests recovery for orphaned managed TaskFlows stuck in `waiting` so heartbeat dispatch is not permanently blocked.

## Best Possible Solution

Define core TaskFlow abandonment/deadline semantics that transition orphaned waiting managed flows to a canonical terminal state and prove the affected heartbeat can run again.

## Reproduction Assessment

Source-reproducible but not live-reproduced: current main can keep `waiting` TaskFlows indefinitely and maintenance does not recover them. The exact live `requests-in-flight` blocker still needs a diagnostic or PR proof path.

## Solution Assessment

Unclear as submitted: expiring old waiting flows during maintenance is plausible, but the best fix must define safe abandonment semantics and show the heartbeat busy resource is released.
