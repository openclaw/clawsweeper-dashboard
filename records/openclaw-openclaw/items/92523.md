## What This Changes

Issue reports orphaned managed TaskFlows stuck in `waiting` status allegedly leaving heartbeat dispatch permanently skipped as `requests-in-flight`.

## Best Possible Solution

Reproduce the full gateway deadlock, identify the actual busy-state owner, then add explicit controller-owned wait abandonment or deadline semantics with startup/runtime repair that preserves legitimate long waits.

## Reproduction Assessment

## Solution Assessment

No; an unconditional age-based expiry is only a plausible mitigation, while the safer fix needs explicit wait-abandonment semantics and gateway-level proof tied to the actual blocked resource.
