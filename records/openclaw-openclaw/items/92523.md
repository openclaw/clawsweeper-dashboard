## Label Justifications

- `P1`: The report describes a real unattended deployment where accepted wake events and scheduled heartbeats stop producing agent turns until manual database and gateway intervention.
- `impact:crash-loop`: The affected agent becomes functionally unavailable and requires operator intervention to recover, which is a process-availability failure even without a crash.
- `impact:message-loss`: The report and follow-up describe wake and voice-input events being accepted but not consumed while heartbeat dispatch remains blocked.
- `impact:session-state`: The core problem is persisted TaskFlow wait state, runtime activity state, and pending delivery state failing to converge across restarts.

## Summary

Keep this issue open: current main and the latest release still leave managed TaskFlows in `waiting` without expiry or startup/runtime recovery, and the linked fix PR closed unmerged. The exact heartbeat busy-state owner still needs maintainer-guided reproduction because `runHeartbeatOnce` gates on runtime lanes, active runs, and recent final delivery state rather than directly querying `flow_runs`.

## What This Changes

The issue asks OpenClaw core to recover abandoned managed TaskFlows stuck in `waiting` so they cannot indefinitely suppress heartbeat work.

## Best Possible Solution

Define controller-owned wait abandonment or deadline semantics, then recover stale waits through the gateway/startup/runtime owner that releases the actual busy resource, with separate handling for final-delivery retry deadlocks if reproduced.

## Reproduction Assessment

Partial: source inspection confirms current main can retain `waiting` TaskFlows indefinitely and maintenance does not touch them. No high-confidence current-main reproduction yet proves that those rows alone create the persistent heartbeat `requests-in-flight` deadlock.

## Solution Assessment

Unclear. A simple TTL sweep is a plausible mitigation, but the best fix needs explicit wait-abandonment semantics and recovery wired to the actual heartbeat busy-state owner.
