## Summary

Keep open: current main partially fixes volatile inbound metadata, but the auto-reply embedded-run path can still convert `prependContext`/prompt additions into runtime context and append them to system-prompt space, contrary to the documented hook contract.

## What This Changes

The issue asks OpenClaw to keep dynamic `before_prompt_build` `prependContext` and per-turn conversation metadata out of the cache-sensitive system prompt for chat continuations.

## Best Possible Solution

Keep the documented contract: deliver `prependContext` and volatile current-turn metadata as user-role/current-turn context, while only stable prompt text and explicit system hook fields affect system-prompt space.

## Reproduction Assessment

Yes. Source inspection shows the auto-reply embedded-run path: `prompt` and `transcriptPrompt` diverge, `prependContext` is added to `effectivePrompt`, the runtime-context split extracts the difference, and the runner appends that runtime context to the system prompt before submission.

## Solution Assessment

No. Current main’s partial metadata split is good, but the remaining runtime-context placement should be repaired narrowly so transcript hiding is preserved without changing the cache-sensitive system prompt.
