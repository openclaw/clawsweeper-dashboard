## Summary

Keep open: current main still has a source-visible auto-reply/embedded-run path where prompt additions that differ from the stored transcript are converted into hidden runtime context and appended to system-prompt space for the model call. That matches the reported `prependContext`/volatile metadata cache-stability concern even though the Codex app-server and inbound-meta helpers have separate guards that look correct in isolation.

## What This Changes

The report asks OpenClaw to stop routing `before_prompt_build` `prependContext` and volatile chat metadata into system-prompt space on chat-continuation runs so prompt caching remains warm.

## Best Possible Solution

Keep the existing hook contract: pass `prependContext` and volatile conversation/reply metadata as current-turn user context while leaving only stable core prompt text or explicit `prependSystemContext`/`appendSystemContext` in system-prompt space, with regression coverage for auto-reply embedded runs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: auto-reply passes a hidden prompt plus distinct `transcriptPrompt`, `prependContext` is added to `effectivePrompt`, the runtime-context split extracts it, and the embedded runner appends that runtime context to the system prompt before the model call.

## Solution Assessment

No, the current path is not the best way to solve this. The safer direction is to separate transcript hiding from system-prompt placement so dynamic per-turn context stays model-visible without changing the cache-sensitive system prompt.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77431. Current main source shows auto-reply/embedded chat runs pass a prompt (`queuedBody`) that differs from `transcriptPrompt`; `resolveRuntimeContextPromptParts` extracts the difference as runtimeContext, and `attempt.ts` appends that runtimeContext to the system prompt before the model call. Preserve the documented hook contract: `before_prompt_build.prependContext` and volatile conversation/reply metadata must remain current-turn user/model context, while only explicit `prependSystemContext`/`appendSystemContext` and stable core prompt text belong in system-prompt space. Keep transcript/UI hygiene intact: do not leak hidden inbound metadata into user-visible history, and do not remove the hidden runtime-context persistence mechanism unless replacing it with an equivalent user-role/turn-context delivery. Add regression coverage showing an auto-reply embedded run with `prependContext`, distinct `transcriptPrompt`, and conversation metadata does not include that dynamic text in `systemPrompt`/system context, while Codex app-server and CLI behavior remain aligned. Add a user-facing `CHANGELOG.md` fix entry. Do not add a config flag or change the public plugin hook API.
