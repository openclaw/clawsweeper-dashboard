## Summary

Keep open: current main still has a source-visible chat-continuation path where `prependContext` and other per-turn prompt additions can be split out of the visible prompt and appended to system-prompt space before the model call. The inbound metadata helper already has cache-stability guards in isolation, but the embedded-runner runtime-context placement still matches the reported cache-break mechanism.

## What This Changes

The issue asks OpenClaw to stop routing dynamic chat-continuation prompt context into the system prompt and to preserve the documented `prependContext` versus `prependSystemContext` hook contract.

## Best Possible Solution

Keep the documented hook contract: `prependContext` and volatile conversation/reply metadata should remain model-visible as current-turn user context, while only stable prompt text or explicit system hook context should enter system-prompt space.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: auto-reply supplies separate prompt and transcript prompt values, hook `prependContext` modifies the effective prompt, the runtime-context splitter extracts the difference, and the embedded runner appends it to system-prompt space before the model call.

## Solution Assessment

No. The current implementation preserves transcript hiding, but routing dynamic per-turn context through system-prompt append is not the narrowest maintainable way to satisfy the documented hook contract or provider prefix-cache stability.
