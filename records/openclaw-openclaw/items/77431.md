## Summary

Keep open: current main still has a source-visible chat-continuation path where per-turn prompt additions and inbound user context can be split out of the visible transcript prompt and appended to system-prompt space before the model call, matching the reported prompt-cache regression.

## What This Changes

The issue asks OpenClaw to keep chat-continuation `prependContext` and volatile conversation/reply metadata out of the system prompt so prompt caching remains stable.

## Best Possible Solution

Keep the existing hook contract by submitting `prependContext` and volatile inbound/reply metadata as current-turn model-visible user context, reserving system-prompt changes for stable base prompt text and explicit system-context hook fields.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: auto-reply passes a prompt containing inbound user context plus a stripped `transcriptPrompt`, the runtime-context split extracts the difference, and the embedded runner appends that extracted context to the system prompt before submission.

## Solution Assessment

No, the current implementation is not the best way to solve the issue. The narrow maintainable fix is to separate transcript hiding from system-prompt placement so dynamic per-turn context remains model-visible without changing the provider cache prefix.
