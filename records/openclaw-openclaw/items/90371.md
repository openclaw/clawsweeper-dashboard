## Summary

The branch composes channel-supplied auto-reply `beforeDeliver` hooks with the global `reply_payload_sending` plus `message_sending` chain in both dispatcher wrappers and adds regression tests for rewrite/cancel behavior.

## Best Possible Solution

## Solution Assessment

No, not as-is. The shared dispatcher seam is the right layer for the bypass, but this branch needs exactly-once ownership so Telegram paths that already run `deliverReplies` do not invoke `message_sending` twice.
