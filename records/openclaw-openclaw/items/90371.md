## Summary

Keep this PR open for human review, but do not merge it as-is: the dispatcher-side fix addresses a real `message_sending` bypass, yet it can double-run Telegram delivery hooks and the branch is currently conflicting with `main`.

The branch changes both auto-reply dispatcher wrappers to compose channel `beforeDeliver` hooks with the global `reply_payload_sending` plus `message_sending` chain and adds dispatch regression tests for rewrite and cancel behavior.

## Best Possible Solution

## Solution Assessment

No, not as-is. The shared dispatcher seam is the right layer for the generic bypass, but this branch needs exactly-once ownership so Telegram paths that already run `deliverReplies` do not invoke `message_sending` twice.
