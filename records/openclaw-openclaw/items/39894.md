## Summary

Keep open. The outbound metadata leak is still source-reproducible on current main, but this draft PR is not mergeable as-is because it patches an older normalization spot, can alter legitimate timestamp-prefixed replies, and adds unrelated tokened CI code.

## What This Changes

The PR attempts to strip echoed inbound metadata before outbound directive parsing, add a metadata-plus-MEDIA regression test, and also changes CI, generated macOS policy output, and macOS test-runner behavior.

## Best Possible Solution

Land a focused current-main fix in the outbound payload plan that strips only copied leading inbound metadata before directive parsing, preserves MEDIA parsing and legitimate timestamp-prefixed text, adds regression coverage, and leaves unrelated CI/macOS changes out.

## Reproduction Assessment

Yes, from source inspection. A ReplyPayload whose text starts with a Conversation info fenced JSON block followed by visible text and a MEDIA directive reaches createOutboundPayloadPlanEntry(), where parseReplyDirectives() receives the raw text before any metadata stripping.

## Solution Assessment

No, this PR is not the best solution as-is. The narrow fix belongs at the current payload-plan parse point with leading-metadata-only stripping, while the unrelated CI/macOS/test-runner edits should be dropped.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - body: The workflow exports GITHUB_TOKEN and then executes scripts/ci-list-pr-files.mjs from the PR checkout.   - confidence: 0.87
- **[P2] Strip metadata at the payload-plan parse point:** `src/infra/outbound/payloads.ts:49`
  - body: The branch changes the older normalizeReplyPayloadsForDelivery() loop, but current main routes delivery and JSON normalization through createOutboundPayloadPlanEntry(), where parseReplyDirectives(payload.text ?? "") still sees raw metadata. Rebase the fix so the strip happens before that current parse point.
  - confidence: 0.9
- **[P2] Avoid stripping ordinary timestamp-prefixed replies:** `src/infra/outbound/payloads.ts:49`
  - body: stripInboundMetadata() removes leading OpenClaw timestamp prefixes even when no metadata block is present. Applying it to every outbound payload can silently change legitimate assistant replies such as logs or transcripts; use a leading metadata-only outbound helper instead.
  - confidence: 0.82
