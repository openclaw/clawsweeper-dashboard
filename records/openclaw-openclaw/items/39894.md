## Summary

Keep open: the outbound metadata leak is still source-reproducible on current main, but this PR is not mergeable as-is because it targets stale normalization code, strips too broadly, and mixes the bug fix with unrelated CI/platform changes including a workflow-token exposure.

## What This Changes

The PR strips inbound metadata before outbound directive parsing, adds a metadata-plus-MEDIA regression test, and also changes CI PR-file detection, generated macOS Swift policy punctuation, and macOS CI Vitest pool selection.

## Best Possible Solution

Land a focused current-main fix in the outbound payload plan that strips only copied leading inbound metadata before directive parsing, preserves MEDIA parsing, adds regression coverage, and leaves unrelated CI/macOS changes out.

## Reproduction Assessment

Yes, from source inspection. A ReplyPayload whose text starts with a Conversation info fenced JSON block followed by visible text and a MEDIA directive reaches createOutboundPayloadPlanEntry(), which parses payload.text before any inbound metadata strip.

## Solution Assessment

No, this PR is not the best solution as-is. The narrow fix belongs at the current payload-plan parse point with leading-metadata-only stripping, while the unrelated CI/macOS/test-runner edits should be dropped.

  - confidence: 0.87

  - body: The branch changes the older normalizeReplyPayloadsForDelivery() loop, but current main now routes delivery and JSON normalization through createOutboundPayloadPlanEntry(), where parseReplyDirectives(payload.text ?? "") still sees raw metadata. Rebase the fix so the strip happens before that current parse point.
  - confidence: 0.9
- **[P2] Preserve timestamp-prefixed outbound text:** `src/infra/outbound/payloads.ts:49`
  - body: stripInboundMetadata() removes leading OpenClaw timestamp prefixes even when no metadata block is present. Applying it to every outbound payload can silently change legitimate assistant replies such as logs or transcripts; use a leading metadata-only outbound helper instead.
  - confidence: 0.82
