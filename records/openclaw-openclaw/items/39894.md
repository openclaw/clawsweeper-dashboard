## Summary

Keep this PR open. Current main still has the outbound metadata leak on the live payload planning path, and the PR contains the right core idea, but the branch is not merge-ready because the parse path moved after the PR was written and the branch also mixes unrelated CI, macOS, and test-runner changes that need separate review or removal.

## What This Changes

The PR strips inbound metadata before outbound reply directive parsing, adds a metadata-plus-MEDIA regression test, and also changes CI secret-scan fallback behavior, generated macOS Swift policy punctuation, and macOS CI Vitest pool selection.

## Best Possible Solution

Land a focused current-main fix that strips inbound metadata inside createOutboundPayloadPlanEntry() before parseReplyDirectives(), keeps MEDIA directive parsing intact, and adds the regression in src/infra/outbound/payloads.test.ts. Split or drop the CI, macOS generated Swift, and macOS Vitest-pool changes from this PR before merge, or open a narrow replacement PR that credits this contribution.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a ReplyPayload whose text starts with a Conversation info fenced JSON block followed by visible text and a MEDIA directive; current main parses that raw text in createOutboundPayloadPlanEntry() on the live delivery route before any metadata strip.

## Solution Assessment

No, not as-is. The core idea is the right narrow fix, but the PR targets the older normalizer body and mixes unrelated security-sensitive CI/macOS/test-runner changes; the safer solution is the same strip in the current payload plan plus a focused outbound regression test.
