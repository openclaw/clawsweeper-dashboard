## Summary

Keep open: current main still lacks the run-scoped announce-state behavior, so the PR has unique useful work, but the branch is not mergeable as-is because it still has correctness blockers and no required real behavior proof.

## What This Changes

The PR threads `sourceRunId` through subagent and media announce delivery, persists pending/delivered announce IDs on run records, clears stale pending state on restore, and adds regression tests.

## Best Possible Solution

Keep the run-scoped announce-state direction, but rebase it onto current main, make grouped delivery clear every represented source run, make restore persistence single-owner, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough to reproduce the blockers: collect mode aggregates multiple announce items into one send while marking only one source delivered, and stale restore cleanup persists in both helper and caller.

## Solution Assessment

No. The direction is reasonable, but this is not the best mergeable solution until it is rebased onto current main, fixes grouped source-run delivery and persistence ownership, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Mark every collected announce source as delivered:** `src/agents/subagent-announce-delivery.ts:309-313`
  - body: `collect` mode drains all queued announce items into one aggregate send built from the last item, but this delivery path marks only that last item's `sourceRunId` as delivered. Earlier grouped runs keep `pendingAnnounceId` set and never get the resume callback after the aggregate announce succeeds; carry all grouped source run IDs through the send or clear all grouped pending markers on success.
  - confidence: 0.86
- **[P2] Let one layer persist stale announce cleanup:** `src/agents/subagent-registry.ts:490-491`
  - body: `clearStaleSubagentAnnouncePendingState()` already flushes when it clears pending announce fields, and this restore block immediately calls `persistSubagentRuns()` again on a truthy return. Make either the helper or the restore caller own the disk write.
  - confidence: 0.92
