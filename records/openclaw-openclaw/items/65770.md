## Summary

Keep open: current main still lacks the run-scoped announce markers, and the live PR head still has source-reproducible correctness blockers, no real behavior proof, and a conflicting branch state.

## What This Changes

The PR threads `sourceRunId` through subagent and media announce delivery, stores pending/delivered announce markers on subagent run records, clears stale pending announce state during restore, and adds regression tests.

## Best Possible Solution

Keep the run-scoped announce-state design, but rebase onto current main, clear every represented source run in collected queue delivery, make stale-cleanup persistence single-owner, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head and current queue path is enough: collect mode sends one aggregate item based on the last queued item, while the PR marks only that item's `sourceRunId`, and stale restore cleanup persists in both helper and caller.

## Solution Assessment

No. The direction is reasonable, but the current PR is not the best mergeable form until grouped delivery state, persistence ownership, branch conflicts, and real-run proof are addressed.
