## Summary

The PR remains useful but is not merge-ready: current main still lacks run-scoped subagent announce delivery state, and the branch still has concrete correctness blockers plus no real behavior proof.

## What This Changes

The branch threads sourceRunId through subagent and media announce delivery, stores pending/delivered announce IDs on subagent run records, clears stale pending announce state on restore, and adds focused regression tests.

## Best Possible Solution

Keep the run-scoped announce-state direction, but fix grouped queue delivery so every represented source run is marked delivered/resumed, make restore cleanup persistence single-owner, then require redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection is enough for the current PR blockers: collect mode aggregates multiple queue items while the patch marks only one source run delivered, and stale restore cleanup persists in both helper and caller.

## Solution Assessment

No. The design is reasonable, but this PR is not the best mergeable solution until grouped delivery and persistence ownership are corrected and after-fix real behavior is shown.
