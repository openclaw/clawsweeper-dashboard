## Summary

Keep open: the PR is a plausible run-scoped announce-state improvement, but the current patch still has blocking correctness issues in queued delivery cleanup and restore persistence, and the external PR lacks after-fix real behavior proof.

## What This Changes

The branch threads `sourceRunId` through subagent and media announce delivery, adds pending/delivered announce fields to subagent run records, clears stale pending state on restore, and adds focused regression tests.

## Best Possible Solution

Keep the run-scoped announce-state design, but make restore cleanup persistence single-owner, update collected queue deliveries so every represented source run is marked delivered/resumed, then add redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection is enough for the PR blockers: collect-mode queue delivery sends a grouped batch as the last item only, and the stale-restore path persists the same mutation in both helper and caller.

## Solution Assessment

No. The requested run-scoped tracking is a reasonable direction, but the current patch is not the best mergeable solution until grouped queue delivery and persistence ownership are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Mark every collected announce source as delivered:** `src/agents/subagent-announce-delivery.ts:306-310`
  - body: `collect` mode drains several queue items by sending one aggregate built from `...last`; with this change, `markSubagentAnnounceDelivered` receives only that last item's `sourceRunId`. Earlier queued runs in the same group keep `pendingAnnounceId` and are never resumed after the batch is delivered, so carry all grouped source run IDs through the collect send or clear all grouped pending markers on success.
  - confidence: 0.86
- **[P2] Let only one layer persist stale announce cleanup:** `src/agents/subagent-registry.ts:487-489`
  - body: `clearStaleSubagentAnnouncePendingState()` already flushes `subagentRuns` when it clears pending announce fields, so this restore block immediately persists the same registry again whenever stale pending state exists. Make either the helper or the restore caller own the disk write.
  - confidence: 0.92
