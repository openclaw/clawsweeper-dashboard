## Summary

Keep open. Current main still lacks the run-scoped announce tracking this PR adds, but the PR has a concrete P2 correctness issue: stale pending restore cleanup can flush the same run registry twice.

## What This Changes

The PR threads `sourceRunId` through subagent announce delivery and media completion paths, adds run-record pending/delivered announce fields plus cleanup helpers, and extends subagent registry/delivery tests.

## Best Possible Solution

Keep the run-scoped announce-state direction, but make persistence ownership single-layer, rebase onto current main, rerun the targeted subagent announce/registry tests, and update the subagents docs once restart-pending announce behavior is actually durable.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection: delivery, queue, run-record, restore, and docs surfaces do not contain the new run-scoped announce state, while the PR diff shows the intended additions.

## Solution Assessment

No, not as currently proposed. The feature boundary is narrow and appropriate, but the restore cleanup should either self-persist or return a mutation signal for the caller to persist, not both.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
