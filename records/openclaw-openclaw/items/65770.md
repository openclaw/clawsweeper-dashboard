## What This Changes

The branch adds sourceRunId to subagent/media announce delivery, records pending/delivered announce IDs on run records, clears stale pending announce state on restore, and adds focused regression tests.

## Best Possible Solution

Rebase and rework the run-scoped announce-state idea onto current main, clear every represented grouped source run on delivery, make restore persistence single-owner, keep focused regression coverage, and add redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection of the live PR head shows collect mode aggregating multiple announce items into one send that only marks the last source run delivered, and restore cleanup persisting in both the helper and caller.

## Solution Assessment

No. The direction is reasonable, but this branch needs a rebase, grouped-source delivery cleanup, single-owner restore persistence, and real behavior proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Mark every collected announce source as delivered:** `src/agents/subagent-announce-delivery.ts:309-313`
  - body: `collect` mode can drain multiple queued announce items into one aggregate send, but this delivery path marks only the aggregate item's `sourceRunId` delivered. Earlier queued runs can keep `pendingAnnounceId` set and miss the resume callback after the aggregate announce succeeds; carry all represented source runs through the send or clear all pending markers represented by the grouped delivery.
  - confidence: 0.86
- **[P2] Make stale announce cleanup persistence single-owner:** `src/agents/subagent-registry.ts:490-491`
  - body: `clearStaleSubagentAnnouncePendingState()` already flushes when it clears pending announce fields, and this restore block immediately calls `persistSubagentRuns()` again on a truthy return. Pick one layer to own the disk write so cold-start restore does not perform duplicate identical flushes.
  - confidence: 0.92
