## Summary

Keep open. This is a new Control UI composer capability, not broken existing behavior: current main has slash-command suggestions only, and the plugin UI descriptor surface does not yet expose a composer diagnostics/highlighting hook. The CausaMem and Hermes-skill comments are useful context/workarounds, but they do not make the requested input-side UI behavior available in OpenClaw today.

## What This Changes

The issue requests configurable Chinese typo and grammar diagnostics in the chat input composer with suggestions or highlighting but no automatic modification.

## Best Possible Solution

Decide the core-versus-plugin API boundary first, then implement an accessible composer diagnostics layer or a plugin-facing composer diagnostics seam that lets dictionaries and language engines live outside core.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of the composer and plugin host-hook types confirms current main lacks general typo/grammar diagnostics for chat input.

## Solution Assessment

Unclear. The requested UX is plausible, but the narrow maintainable solution depends on maintainer choices about core UI versus plugin API, config shape, language engine, and privacy/performance tradeoffs.
