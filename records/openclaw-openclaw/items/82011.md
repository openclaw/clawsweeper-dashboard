## Summary

Keep open. This is a new Control UI composer feature request, not a broken existing contract: current main only has slash-command-style composer suggestions, and the existing plugin UI descriptor surface does not expose a composer text-highlighting or grammar-check hook, so maintainers need to decide product/API scope before implementation.

## What This Changes

The issue requests configurable pre-send typo and grammar detection/highlighting in chat input, with dictionary-based suggestions and no automatic text mutation.

## Best Possible Solution

Decide the product/API boundary first, then implement either an accessible core composer suggestion layer with config/docs or a plugin-facing composer diagnostics seam that lets a ClawHub plugin own dictionaries and language engines.
