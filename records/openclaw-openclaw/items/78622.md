## Summary

Current main and the shipped v2026.5.19 release already use npm resolution metadata when formatting plugin update dry-run target versions, so the reported `unknown` target-version problem is fixed.

## What This Changes

The issue asks for `openclaw plugins update --all --dry-run` to print resolved npm plugin target versions instead of `unknown` when npm metadata resolution succeeded.

## Best Possible Solution

Keep the shipped updater fallback and close this report rather than reopening duplicate fix branches for behavior already present in releases.

## Reproduction Assessment

Yes for the historical bug: the installer dry-run returned `npmResolution.version` without top-level `version`, while the old formatter ignored that metadata. Current main no longer reproduces because the formatter now consumes `npmResolution.version`.

## Solution Assessment

Yes. The shipped implementation fixes the narrow formatter/data-contract mismatch by centralizing npm result-version extraction and using it before falling back to `unknown`.
