## Summary

Keep open; current main still has the reported source path where npm dry-run update results can carry `npmResolution.version` without a top-level `version`, while the update outcome formats only `probe.version`.

## What This Changes

Request that `openclaw plugins update --all --dry-run` show the resolved npm target version instead of `unknown` for npm-installed plugins.

## Best Possible Solution

Derive the dry-run target version from `probe.version ?? probe.npmResolution?.version`, use that same value in the outcome object and unchanged check where appropriate, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, from source: mock an npm dry-run install result with `ok: true`, `npmResolution.version`, and no top-level `version`; current main formats only `probe.version`, so the outcome is `unknown`. I did not run the test because this review is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to fall back to `npmResolution.version` in `src/plugins/update.ts`; changing the CLI logger would be too late because it only prints the already-built outcome message.
