## Summary

Keep open: current main still misses the `probe.npmResolution.version` fallback in the npm plugin update dry-run formatter, and the linked closing PR is still open.

## What This Changes

The issue asks for `openclaw plugins update --all --dry-run` to show resolved npm target versions instead of `unknown` when npm metadata includes a version.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78751 or an equivalent current-main patch that keeps the exact-spec fallback, adds the `npmResolution.version` fallback, and includes regression coverage plus proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock a successful npm dry-run probe with `npmResolution.version` and no top-level `version` for an unpinned npm spec; current main reads only `probe.version` or an exact-spec fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to derive the dry-run target from `probe.version ?? probe.npmResolution?.version ?? exact-spec fallback` in `src/plugins/update.ts`.
