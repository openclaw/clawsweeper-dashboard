## Summary

Keep open: current main still does not use `probe.npmResolution.version` in the npm dry-run update formatter, and the open closing PR has not merged yet.

## What This Changes

The issue requests that `openclaw plugins update --all --dry-run` show resolved npm target versions instead of `unknown` when npm metadata contains the target version.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78751 or an equivalent patch that preserves the exact-spec fallback, falls back to `npmResolution.version`, and adds regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock a successful npm dry-run probe with `npmResolution.version` and no top-level `version` for an unpinned npm spec; current main still formats that path without reading `npmResolution.version`.

## Solution Assessment

Yes. The narrow fix is to derive the dry-run target from `probe.version ?? probe.npmResolution?.version ?? exact-spec fallback` in `src/plugins/update.ts`, because the CLI prints the already-built outcome message.
