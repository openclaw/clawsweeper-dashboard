## What This Changes

The issue reports that `openclaw plugins update --all --dry-run` can print `unknown` instead of the resolved npm target version for npm-installed plugins.

## Best Possible Solution

Land one canonical fix that preserves the current exact-spec fallback and also falls back to `probe.npmResolution?.version`, with focused regression coverage and release notes handled during landing.

## Reproduction Assessment

Yes, by source inspection: mock a successful npm dry-run probe with `npmResolution.version` and no top-level `version` for an unpinned spec, and current main's formatter still ignores that metadata. I did not run the test because this review is read-only and tests can write artifacts.

## Solution Assessment

Yes. The narrow maintainable fix is to derive the dry-run target from `probe.version ?? probe.npmResolution?.version ?? exactSpecFallback` in `src/plugins/update.ts`; the issue should remain open until one linked PR lands.
