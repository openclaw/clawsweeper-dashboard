## Summary

Keep open: this is a plausible bug-fix PR, but it is not mergeable because it does not update the gateway agent store write path reported in the linked regression, rejects valid custom transcript paths, and lacks real behavior proof.

## What This Changes

The branch adds a sessionFile/sessionId basename compatibility gate in session path resolution plus regression tests for stale path fallback, topic paths, and auto-reply daily reset rotation.

## Best Possible Solution

Fix the gateway reset write path so generated session-id rollovers derive and persist new generated transcript filenames while preserving explicit custom/fork transcript names, then cover gateway agent and sessions.reset behavior with a changelog entry.

## Reproduction Assessment

Yes. The linked issue provides concrete store/filesystem evidence, and current main shows a source path where gateway daily rollover changes sessionId but persists a patch that omits sessionFile, preserving the old value.

## Solution Assessment

No. The helper-level basename filter is too broad and does not persist the rotated sessionFile in the gateway agent path; the narrower maintainable fix belongs in the reset write/rewrite path that knows whether a sessionId actually changed.
