## Summary

Keep open. Current main still contains the source-level denial follow-up, `exec.denied` wakeup, and missing reported Chinese stop phrases, and the open closing PR at https://github.com/openclaw/openclaw/pull/85194 should be reviewed and landed before this issue closes.

## What This Changes

The issue reports that denied exec approvals and stop messages can feed additional agent work, producing a denial retry loop, delayed command flood, and dropped user messages.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/85194; after it merges, close this issue as implemented and leave broader approval-pause or batching policy to the related open items.

## Reproduction Assessment

Yes. Current main still routes normal-session denied approval results into follow-up agent work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

Yes. The narrow best fix is to make denied exec outcomes terminal for ordinary sessions, stop `exec.denied` wakeups, and route the reported phrases through the existing abort cleanup path, not to add a denial-count heuristic.
