## Summary

Keep open. Current main and the latest shipped release still have the source-level denial-loop ingredients: ordinary-session denied exec approvals can schedule agent follow-up work, node `exec.denied` events still wake sessions, and the reported `停下来` / `暂停` stop texts are not abort triggers; the attempted fix PR was closed unmerged and does not supersede this report.

## What This Changes

The issue reports runaway agent retries after denied exec approvals, delayed denial/completion wakeups, stop commands not interrupting promptly, and queue overflow dropping user messages.

## Best Possible Solution

Land a focused denial/stop repair: denied exec approvals should not spawn new agent work, reported stop phrases should enter the existing abort path, and stop should clear pending follow-up/session work while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into follow-up agent work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

No current implementation fully solves the issue. The safest path is a narrow denied-exec and abort-trigger fix with regression coverage, leaving broader approval-pause and batching policy to the related open issues.
