## Summary

Keep open. Current main still has source-level denial-loop ingredients: normal-session denied exec approvals can call the agent again, node `exec.denied` events enqueue wakeable system events, and the reported `停下来` / `暂停` stop texts are not abort triggers; the related attempted fix was closed unmerged.

## What This Changes

The issue asks OpenClaw to stop runaway exec approval denial loops by treating repeated denials and stop messages as hard interrupts, suppressing denied-completion agent wakeups, and keeping pending work cancellable.

## Best Possible Solution

Land a focused denial/stop circuit breaker: denied exec approvals should not spawn new agent work, reported stop phrases should hit the existing abort path, and stop should clear pending follow-up/session work while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into agent follow-up work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

No current implementation fully solves the issue. The safest path is a narrow denied-exec and abort-trigger fix with regression coverage, leaving broader approval-pause and batching policy to the related open issues.
