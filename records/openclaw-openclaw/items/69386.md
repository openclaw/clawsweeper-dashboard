## Summary

Keep open. Current main still routes normal-session denied exec approval results into follow-up agent work, node `exec.denied` events still wake sessions, and the reported Chinese stop phrases are not recognized; the related attempted fix was closed unmerged and does not supersede this bug.

## What This Changes

The issue reports that denied exec approvals can create repeated agent turns that ignore stop messages, hide delayed commands, and drop queued user messages during overflow.

## Best Possible Solution

Land a focused bug fix: denied exec approvals should be recorded without spawning new agent work, `停下来` and `暂停` should enter the existing fast-abort path, and fast abort should clear pending follow-up/session work while broader approval scheduling stays with the related open issues.

## Reproduction Assessment

Yes, at source level. Denying a gateway exec approval still reaches `sendExecApprovalFollowup`, which calls the agent for ordinary sessions, while the reported stop phrases do not match the shared abort parser.

## Solution Assessment

Yes for a narrow bug-fix slice. Suppressing denied exec follow-up agent turns, adding the reported abort phrases, and using the existing queue cleanup path is more maintainable than adding a separate denial-streak scheduler here.
