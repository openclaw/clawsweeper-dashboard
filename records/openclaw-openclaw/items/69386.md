## Summary

Keep open. Current main still lets denied exec approvals create follow-up agent work for ordinary sessions, node `exec.denied` events still enqueue wakeable system events, and the reported Chinese stop phrases are still absent from the abort trigger set.

## What This Changes

The issue reports that exec denial completions and unrecognized stop messages can keep creating agent turns, delayed commands, and queue overflow after the user tries to stop.

## Best Possible Solution

Land a focused fix that circuit-breaks denied exec followups, treats the reported stop phrases as fast aborts, and clears pending followup/session work on stop while leaving approval batching and pause semantics to the related open issues.

## Reproduction Assessment

Yes. Source inspection on current main shows the denied-followup and node `exec.denied` paths still enqueue agent work, and current tests assert those behaviors; the missing `停下来` and `暂停` triggers are visible in the abort trigger set.

## Solution Assessment

No current implementation fully solves it. The safest path is a narrow denial/abort/queue circuit breaker with regression coverage, not the closed unmerged task-flow-only PR and not the broader approval-pause work tracked elsewhere.
