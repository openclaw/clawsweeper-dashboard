## Summary

Keep open. Current main and the latest shipped tag still contain the denial-loop ingredients: denied exec approvals can create ordinary-session follow-up agent work, remote node `exec.denied` events enqueue wakeups, and the reported Chinese stop phrases are not abort triggers.

## What This Changes

The issue asks OpenClaw to stop runaway agent retries after repeated denied exec approvals, recognize stop text as a priority abort, and avoid queue overflow or delayed command floods.

## Best Possible Solution

Land a focused denial/stop repair: denied exec approvals should not spawn new agent work, the reported stop phrases should enter the existing abort path, and abort should clear pending follow-up/session work while preserving denial auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into follow-up agent work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

Yes for a narrow repair, but not via the closed related PR. The safer fix should target the production exec follow-up, node-event wake, abort trigger, and queue cleanup paths rather than task-flow denial keyword scanning.
