## Summary

Keep open. Current main still lets denied exec completions create agent followup work for normal sessions, node `exec.denied` events still enqueue wakeable system events, and the exact reported Chinese stop phrases are not recognized; existing `/stop` cleanup only helps after an abort phrase is detected.

## What This Changes

The issue asks OpenClaw to stop runaway exec-denial loops by treating denials and stop commands as high-priority cancellation signals, suppressing denied completion turns, exposing/canceling delayed commands, and preventing queue overflow.

## Best Possible Solution

Land a focused production fix that circuit-breaks denied exec followups, treats the reported stop phrases as fast aborts, and clears pending followup/session queues on stop, while leaving broader approval batching and pause semantics to #69570 and #70953.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on current main: denied exec approval followups for normal sessions still call the agent, and node `exec.denied` events still enqueue system work and heartbeat wakes; tests currently assert both behaviors.

## Solution Assessment

No current implementation fully solves the issue. The closed unmerged PR #69387 targeted task-flow creation, but the remaining safer fix is a narrow denial/abort/queue circuit breaker with regression coverage in the exec followup, node-event, abort, and queue paths.

## Repair Work Prompt

Fix the denial-loop issue at https://github.com/openclaw/openclaw/issues/69386. Current main still calls the `agent` tool for denied exec approval followups in normal sessions, enqueues/wakes node `exec.denied` events, and does not recognize the reported Chinese stop phrases `停下来` or `暂停`. Implement a focused repair: denied exec completions must not create retry-capable agent turns for normal sessions, repeated denials should circuit-break rather than generate command variants, explicit stop phrases should go through fast abort, and fast abort should clear pending followups/session command lanes for the affected session. Preserve useful user-visible denial status without waking a new model turn. Do not implement broad approved-completion batching or full approval-pending pause semantics except where required for this denial fix; coordinate those with #69570 and #70953. Update regression tests and add a single-line user-facing `CHANGELOG.md` fix entry.
