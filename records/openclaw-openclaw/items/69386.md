## Summary

Keep open. Current main still has the reported denial-loop ingredients: ordinary denied exec approvals can resume an agent turn, node `exec.denied` events still enqueue wakeable system events, and the reported `停下来`/`暂停` stop phrases are not recognized as abort triggers.

## What This Changes

The issue reports that repeated exec denials and stop messages can leave the agent retrying commands, enqueueing delayed denial/completion events, and overflowing queued user messages.

## Best Possible Solution

Land a focused fix that treats repeated denials and stop requests as a hard interruption for the current session: stop denied exec agent wakeups, recognize the reported stop phrases, and clear pending follow-up/session work without redesigning approval batching.

## Reproduction Assessment

Yes, at source level. Current main still routes denied approval results into ordinary agent follow-up work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

No current implementation fully solves it. The safest fix is a narrow denied-exec/abort/queue circuit breaker with regression coverage, not the closed task-executor-only PR and not the broader approval-pause redesign tracked elsewhere.
