## Summary

Keep open. Current main still routes denied exec completions back into normal agent follow-up work and node `exec.denied` events still enqueue wakeable system events; the exact reported stop phrases are not covered by abort trigger tests or source.

## What This Changes

The issue requests that denied exec approvals and stop messages halt runaway agent retry/follow-up queues instead of spawning more agent work and dropping user messages.

## Best Possible Solution

Land a focused production fix that suppresses or circuit-breaks denied exec followups for normal sessions, stops waking sessions for denied node exec events unless explicitly needed, recognizes the reported stop phrases, and keeps queue cleanup on the existing abort path.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on current main: denied approvals and node `exec.denied` events still create wakeable agent work, and the reported `停下来` and `暂停` phrases are absent from abort recognition.

## Solution Assessment

No current implementation fully solves the issue. The safer path is a narrow denial/abort/queue circuit breaker in the exec followup, node event, abort trigger, and queue cleanup tests rather than the closed Task Flow-only PR.
