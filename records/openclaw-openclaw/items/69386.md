## Summary

Keep open. Current main and v2026.5.18 still route ordinary denied exec approvals into follow-up agent work, still enqueue and wake sessions for node `exec.denied` events, and still do not recognize the reported `停下来` / `暂停` stop phrases; the linked fix PR was closed unmerged and does not supersede this report.

## What This Changes

The issue requests that denied exec approvals and stop commands halt the runaway agent loop, preserve pending-command visibility/cancellation, and prevent queued user stop messages from being dropped.

## Best Possible Solution

Land a focused denial/stop repair: denied exec approvals should not spawn new agent work, the reported stop phrases should enter the existing abort path, and stop should clear pending follow-up/session work while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into follow-up agent work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

No current implementation fully solves the issue. The safest path is a narrow denied-exec and abort-trigger fix with regression coverage, leaving broader approval-pause and batching policy to the related open issues.
