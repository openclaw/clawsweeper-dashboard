## Summary

Keep open. Current main still has the reported denial-loop ingredients: ordinary denied exec approvals can resume an agent turn, node `exec.denied` events still enqueue wakeable system events, and the reported Chinese stop phrases are still not recognized as fast aborts.

## What This Changes

The issue reports runaway exec-approval denial loops where stop messages are ignored or delayed, denied completions generate more agent turns, hidden delayed commands continue, and inbound stop messages can be dropped behind queued work.

## Best Possible Solution

Land a focused fix that circuit-breaks denied exec follow-up loops, recognizes the reported stop phrases as fast aborts, and clears pending follow-up/session work on stop while leaving broader approval batching and pause semantics to their canonical issues.

## Reproduction Assessment

Yes, at source level. Current main still routes denied approval results into follow-up agent work for ordinary sessions, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` fast-abort triggers.

## Solution Assessment

No current implementation fully solves it. The safest path is a narrow denial/abort/queue circuit breaker with regression coverage, not the closed task-flow-only PR and not the broader approval-pause redesign tracked elsewhere.
