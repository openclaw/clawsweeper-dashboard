## Summary

Keep open. Current main still turns denied exec approvals into follow-up agent work for normal sessions, still wakes sessions on node `exec.denied` events, and still does not recognize the reported `停下来` / `暂停` stop phrases.

## What This Changes

The issue reports a runaway exec-approval denial loop where stop messages are delayed or dropped while denied async exec completions keep creating agent turns.

## Best Possible Solution

Land a focused denial/stop repair: denied exec approvals should not spawn new agent work, the reported stop phrases should enter the existing abort path, and stop should clear pending follow-up/session work while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into follow-up agent work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

Yes for a narrow bug-fix slice. Suppress denied exec follow-up agent turns, add the reported abort phrases, and use the existing queue cleanup path while leaving broader approval-pause and batching policy to related issues.
