## Summary

Keep open. ## What This Changes

The issue reports that denied exec approvals can trigger repeated agent follow-up turns while stop messages are delayed, dropped, or not recognized.

## Best Possible Solution

Land a focused denial/stop repair: denied exec approvals and node `exec.denied` events should not spawn new agent turns, the reported stop phrases should enter the existing abort path, and stop should clear pending session/follow-up work while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Current main still routes normal-session denied approval results into agent follow-up work, still wakes sessions on node `exec.denied`, and lacks the reported `停下来` and `暂停` abort triggers.

## Solution Assessment

Yes, with a narrow scope adjustment. Suppressing denied exec follow-up agent turns, adding the reported abort phrases, and using the existing queue cleanup path is more maintainable than a denial-count heuristic or a broad approval scheduler rewrite.
