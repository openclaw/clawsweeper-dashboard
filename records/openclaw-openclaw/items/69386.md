- `impact:message-loss`: The issue explicitly reports queue overflow dropping 47 user messages, including stop commands and log-path context.

## Summary

Keep open. Current main still has source-level denial-loop paths: denied gateway exec approvals can spawn an ordinary-session follow-up agent turn, node `exec.denied` events still enqueue wakeable system events, and the reported `停下来` / `暂停` stop texts are not abort triggers.

## What This Changes

The issue reports denial-triggered exec retry loops, delayed denied-command follow-ups, invisible pending work, dropped stop messages, and missing high-priority stop handling during webchat exec approvals.

## Best Possible Solution

Land a focused denial/stop circuit breaker: denied exec approvals should not spawn ordinary-session agent work, reported stop phrases should hit the existing abort path, and all abort-trigger paths should clear pending session follow-ups while preserving approval auditability.

## Reproduction Assessment

Yes, at source level. Configure exec approvals, deny a normal-session gateway exec request, and current main can still create a denied follow-up agent turn while the reported Chinese stop phrases do not enter the abort path.

## Solution Assessment

No current implementation fully solves the issue. The narrowest maintainable fix is to suppress denied ordinary-session agent wakeups, add the missing stop phrases, clear queued follow-ups on abort triggers, and leave broad batching or approval-pause policy to the related scheduler issues.
