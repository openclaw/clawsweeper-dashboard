## Summary

Keep open. Current main still lets denied exec approvals create follow-up agent work for ordinary sessions, node `exec.denied` events still enqueue wakeable system events, and the reported Chinese stop phrases are still absent from the abort trigger set.

## What This Changes

The issue asks OpenClaw to stop runaway exec denial loops, prioritize stop commands, avoid denied-command follow-up floods, and prevent queue overflow during approval denials.

## Best Possible Solution

Land a focused fix that circuit-breaks denied exec followups, treats the reported stop phrases as fast aborts, and clears pending followup/session work on stop while leaving approval batching and pause semantics to the related open issues.

## Reproduction Assessment

Yes, at source level. Current main still enqueues agent followups for denied exec approvals and node `exec.denied` events, while the reported stop phrases are not recognized by the abort trigger set.

## Solution Assessment

No current implementation fully solves it. The safest path is a narrow denial/abort/queue circuit breaker with regression coverage, not the closed task-flow-only PR and not the broader approval-pause redesign tracked elsewhere.

Implement a focused circuit breaker so denied exec approvals do not start another agent turn, node denied events do not wake the agent into retry loops, and the reported stop phrases are treated as fast aborts that clear pending followup/session work.
