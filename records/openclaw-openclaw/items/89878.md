## What This Changes

The PR changes document and web content extractor public-artifact loading plus web extractor resolution so throwing or malformed extractor factories/plugins do not disable healthy sibling extractors.

## Best Possible Solution

Keep the per-factory and per-plugin failure isolation, preserve valid descriptor-owned enumerable state in both normalizers, refresh the branch, and rerun focused plus changed-surface validation.

## Reproduction Assessment

Yes, by source inspection. Current main can abort web factory loading on a throwing factory and can abort document artifact loading when descriptor getters throw; the PR tests encode those paths, though this read-only review did not execute them.

## Solution Assessment

No, not as written. The loader/runtime boundary is the right place for failure isolation, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
