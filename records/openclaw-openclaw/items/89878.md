## Summary

Keep open; this is a focused maintainer bugfix for extractor failure isolation, but the branch is not merge-ready because the new descriptor normalizers can drop extractor-owned receiver state while keeping the original extract function.

## What This Changes

The PR changes bundled document and web content extractor artifact loading to isolate bad factory/descriptor outputs, lets web extractor resolution skip one failed plugin when another succeeds, and adds focused regression tests.

## Best Possible Solution

Keep the factory/descriptor failure isolation, but preserve extractor receiver state explicitly, for example by binding extract to the original descriptor or carrying a deliberate receiver, then keep the focused regression coverage.

## Reproduction Assessment

Yes from source inspection: current main web extractor factory loading can throw before later healthy extractors load, while the PR's introduced receiver-state regression is reproducible with a valid extractor object whose extract method reads descriptor-owned state. I did not run tests because this review is read-only.

## Solution Assessment

No as written; isolating factory and descriptor failures in the artifact loaders is the right layer, but the normalizers should not discard the descriptor state needed by the retained extract function.
