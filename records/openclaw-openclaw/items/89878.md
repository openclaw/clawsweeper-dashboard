## Summary

Keep open: the PR addresses a real extractor failure-isolation bug on current main, but the patch still has a blocking compatibility regression because both new normalizers replace valid plugin descriptors with minimal objects and drop descriptor-owned state that current shallow-copy registration preserves.

## What This Changes

The branch isolates throwing or malformed document and web-content extractor factories, lets web-content extractor runtime skip failed extractor plugins when another loads, and adds focused regression tests.

## Best Possible Solution

Keep the failure-isolation behavior, preserve the validated descriptor object or its enumerable state in both normalizers, add stateful method-style extractor regression tests, then refresh the branch and run the focused plus changed gates.

## Reproduction Assessment

Yes by source inspection: current main calls web extractor factories without isolation and validates document descriptors after the factory try/catch. I did not execute tests in this read-only review, but the PR's focused tests describe the failing paths.

## Solution Assessment

No, not as written. The failure-isolation direction is the right owner-boundary fix, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
