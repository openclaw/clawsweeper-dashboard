## Summary

Keep open. Current main still has the extractor failure-isolation bug, but this draft PR is not merge-ready because its new descriptor normalizers can drop valid extractor-owned state that current shallow-copy registration preserves.

## What This Changes

The PR changes bundled document and web content extractor loading so malformed or throwing extractor descriptors do not prevent healthy extractors from being used, and adds focused regression tests.

## Best Possible Solution

Keep the failure isolation, preserve valid extractor descriptor state and receiver semantics in both normalizers, add stateful extractor regression tests, then rebase and rerun focused plus changed-surface gates.

## Reproduction Assessment

Yes. Source inspection shows current main can abort web extractor collection on a throwing factory and can abort document descriptor validation when a getter throws; the PR tests encode those paths, though no tests were run in this read-only review.

## Solution Assessment

No, not as written. Failure isolation is the right owner-boundary direction, but replacing valid descriptors with minimal objects is not the safest compatibility-preserving implementation.
