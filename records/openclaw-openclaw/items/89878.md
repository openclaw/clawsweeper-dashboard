## Summary

Keep open. The failure-isolation bugfix is useful and source-backed, but the PR is still not merge-ready because both new descriptor normalizers replace valid extractor descriptors with minimal objects, which can break descriptors that rely on enumerable descriptor-owned state; live GitHub metadata also reports the draft branch as conflicting.

## What This Changes

The PR isolates throwing or malformed document and web extractor factories, skips failed web extractor plugin loads when another enabled extractor succeeds, and adds focused regression tests.

## Best Possible Solution

Keep the per-factory and per-plugin failure isolation, preserve valid descriptor object/enumerable state in both public-artifact normalizers, then refresh the branch and rerun focused plus changed-surface validation.

## Reproduction Assessment

Yes. Source inspection shows current main can abort web factory loading on a throwing factory and can abort document artifact loading when descriptor getters throw; the PR tests encode those paths, though this read-only review did not execute them.

## Solution Assessment

No, not as written. The loader/runtime boundary is the right place for failure isolation, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
