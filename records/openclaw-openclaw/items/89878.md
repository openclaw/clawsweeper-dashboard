## Summary

Keep open: the PR targets a real extractor-loading failure, but it still has a merge-blocking compatibility regression because the new normalizers strip descriptor-owned state before existing method-style extractor calls can run.

## What This Changes

The branch isolates throwing or malformed document/web extractor factory outputs, lets web extractor resolution skip failed extractor plugins, and adds focused regression tests.

## Best Possible Solution

Keep the failure isolation behavior, but preserve the validated descriptor object or its enumerable state for valid extractors and add regression tests for stateful method-style document and web extractors before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows web extractor factories are called without isolation and document descriptor validation can throw outside the factory try/catch; the PR's focused tests describe the failing paths.

## Solution Assessment

No, not as written. The failure-isolation direction is the right owner-boundary fix, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
