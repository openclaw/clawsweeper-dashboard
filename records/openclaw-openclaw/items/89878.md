## What This Changes

The PR isolates throwing or malformed document/web content extractor factories, lets web extractor resolution skip a failed plugin when another succeeds, and adds focused regression tests.

## Best Possible Solution

Keep the failure isolation behavior, but preserve valid descriptor objects or their enumerable state in both normalizers and add stateful method-style extractor tests before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows web factory invocation is not isolated and document descriptor validation can throw outside the factory try/catch; the PR tests encode those failing paths.

## Solution Assessment

No, not as written. Failure isolation belongs in these loader/runtime seams, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
