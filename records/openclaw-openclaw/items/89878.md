## Summary

Keep open. The failure-isolation direction is valid, but the patch is not merge-ready because the new normalizers drop descriptor-owned state that current registration preserves while callers invoke extractors as methods.

## What This Changes

The PR changes document and web extractor public-artifact loaders to isolate throwing or malformed factories, skips failed web extractor plugins when others load, and adds focused regression tests.

## Best Possible Solution

Keep the new per-factory and per-plugin failure isolation, preserve valid descriptor object/state in both normalizers, then rebase and rerun focused plus changed-surface proof.

## Reproduction Assessment

Yes. Source inspection shows the current-main web factory path can throw before later factories load, and document descriptor getters are read outside the factory try/catch; the PR tests encode those paths, though I did not execute them in this read-only review.

## Solution Assessment

No, not as written. The isolation belongs in this loader/runtime boundary, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
