## Summary

Keep open: the failure-isolation direction fixes a real plugin extractor loader bug, but the current patch is not merge-ready because both new normalizers replace valid descriptors with minimal objects and can break descriptor-owned state that current registration preserves.

## What This Changes

The PR changes document and web content extractor public-artifact loading to isolate factory and descriptor failures, skip failed web extractor plugins when another plugin succeeds, and add focused regression tests.

## Best Possible Solution

Keep the new factory/descriptor failure isolation, preserve valid descriptor receiver/state for both extractor surfaces, add method-style stateful regression tests, then rebase and rerun focused plus changed-surface gates.

## Reproduction Assessment

Yes. Source inspection shows current main can abort web factory collection on one throwing factory and can let document descriptor getter failures escape validation; the PR adds focused tests for those paths, though this read-only review did not execute them.

## Solution Assessment

No, not as written. The failure-isolation boundary is the right fix location, but the normalizers should preserve valid descriptor state instead of replacing descriptors with minimal objects.
