## Summary

Keep open. The failure-isolation direction is useful and still necessary on current main, but this draft maintainer PR still has a blocking compatibility defect: both new normalizers can drop descriptor-owned state that existing method-style extractors can observe through `this`.

## What This Changes

The branch isolates document and web content extractor factory/descriptor failures so healthy extractor entries can still load, and adds focused regression coverage for those failure paths.

## Best Possible Solution

Keep the per-factory failure isolation, but preserve valid descriptor-owned enumerable state inside both normalizers and cover method-style document and web extractors before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows web factory calls can throw before later factories load and document descriptor reads happen outside the isolated factory catch; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently written. Failure isolation belongs in this plugin public-artifact loader layer, but the implementation should preserve valid descriptor state instead of replacing descriptors with minimal objects.
