## Summary

Keep open for maintainer product review: current main has not implemented Iroh, the latest release does not contain it, and the proposal would replace or augment core Gateway transport and state-sync architecture rather than fix a narrow existing bug.

## What This Changes

The issue requests an Iroh-based peer-to-peer Gateway transport and decentralized workspace/session state synchronization design for OpenClaw companion devices.

## Best Possible Solution

Keep this as a maintainer-owned architecture/RFC discussion and split any future work into a narrow Gateway/node transport prototype before considering workspace or SQLite-backed state synchronization.

## Reproduction Assessment

Not applicable: this is an architectural feature proposal, not a bug report with a failing current-main behavior. Source inspection confirms the current implemented path is WebSocket/HTTP plus Bonjour, Tailscale, SSH, direct URL, and local SQLite/workspace state.

## Solution Assessment

Unclear: Iroh may be a plausible direction, but it is not yet proven to be the best OpenClaw solution because it would add a new core transport, native dependency packaging, relay policy, config, and data-sync model. A focused RFC/prototype should decide transport boundaries before any state-sync implementation.
