## Summary

Keep open for maintainer architecture, product, and security review: current main and the latest release still use Gateway HTTP/WebSocket plus Tailscale/SSH/LAN remote access, while this request proposes a new Iroh P2P transport and cross-device state-sync architecture.

## What This Changes

The issue requests an Iroh-based Gateway and companion-node communication layer with NAT traversal, stable cryptographic node identity, and CRDT-style workspace/session/memory sync.

## Best Possible Solution

Keep this as a maintainer-owned architecture RFC and split any future work into a transport prototype, security model, and state-sync data-model design before implementation.
