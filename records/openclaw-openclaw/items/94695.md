## Summary

Keep open for maintainer architecture, product, and security review: current main and the latest release still document and implement the Gateway as HTTP/WebSocket, while this request proposes a new Iroh peer-to-peer transport plus cross-device state sync.

## What This Changes

The issue requests an Iroh-based core networking and synchronization architecture for zero-config peer-to-peer Gateway access, mobile roaming, and multi-device state sync.

## Best Possible Solution

Keep this as a maintainer-owned architecture RFC and split any future work into a narrow transport prototype, a separate security model, and a separate state-sync data-model design before implementation.
