## Summary

Keep open for maintainer architecture, product, and security review: current main and v2026.6.9 still use Gateway HTTP/WebSocket plus Tailscale/SSH remote-access paths, while this request would add a new Iroh peer-to-peer transport and cross-device state-sync model.

## What This Changes

The issue requests embedding Iroh as a core peer-to-peer Gateway transport and CRDT-backed sync layer for companion nodes and workspace state.

## Best Possible Solution

Keep this as a maintainer-owned architecture RFC and split any future work into separate transport, security model, native packaging, config/defaults, and state-sync data-model decisions before implementation.
