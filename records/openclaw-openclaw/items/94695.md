## Summary

Keep open for maintainer architecture, product, and security review: current main still uses the Gateway WebSocket/HTTP transport with Tailscale/Bonjour/SSH discovery paths, while this request proposes a new core P2P transport plus cross-device state sync.

## What This Changes

Request to adopt Iroh for zero-config peer-to-peer Gateway communication, mobile roaming, and multi-device state synchronization.

## Best Possible Solution

Keep this as a maintainer-owned RFC and split any future work into a narrow Gateway transport prototype first, with separate security-reviewed decisions for relay policy, auth compatibility, packaging, config, and state sync.
