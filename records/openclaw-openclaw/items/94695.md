## Summary

Keep open for maintainer architecture, product, and security review: current main still exposes Gateway control and companion-node traffic through HTTP/WebSocket plus documented Tailscale/VPN or SSH remote-access paths, and no current Iroh/P2P state-sync implementation or canonical replacement item exists.

## What This Changes

The issue requests a core Iroh-based peer-to-peer networking and CRDT state-sync architecture for OpenClaw gateway and companion-device communication.

## Best Possible Solution

Keep this as a maintainer-owned RFC and split future work into a narrow Gateway transport prototype first, with separate security-reviewed decisions for relay policy, auth compatibility, packaging, config, and state sync.
