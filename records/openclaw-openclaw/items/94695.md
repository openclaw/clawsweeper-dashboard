## Summary

Keep open for maintainer architecture and security review: current main and v2026.6.8 still use HTTP/WebSocket Gateway transport with Tailscale, Bonjour, and SSH discovery, while this asks for a broad new core P2P transport and state-sync architecture.

## What This Changes

Request an Iroh-based Gateway and companion-node P2P transport plus decentralized workspace and session state synchronization.

## Best Possible Solution

Keep this as a maintainer-owned RFC, split any work into a narrow Gateway transport and pairing threat-model prototype first, and evaluate workspace or SQLite-backed state sync only after the transport/security direction is approved.
