## Summary

Keep open for maintainer architecture, product, and security review: current main and v2026.6.9 still use Gateway WebSocket/HTTP transport with Tailscale/SSH/Bonjour-style access paths, and the closest related issues cover only parts of this Iroh transport plus state-sync proposal.

## What This Changes

The issue requests an Iroh-based peer-to-peer communication layer for zero-config NAT traversal, mobile companion roaming, and cross-device workspace/session/state synchronization.

## Best Possible Solution

Keep this as a maintainer-owned RFC candidate and split any accepted direction into separate transport, identity/auth, relay/config/packaging, and state-sync designs while preserving the current WebSocket Gateway path as the default until upgrade safety is proven.

## Reproduction Assessment

Not applicable: this is an architectural feature proposal, not a bug report with a current-main failure path. Source, docs, current-main, and release inspection confirm the requested Iroh transport and state-sync behavior is absent rather than broken.

## Solution Assessment

Unclear: Iroh is a possible design direction, but the issue does not yet prove it is the best OpenClaw solution because transport ownership, security model, native packaging, config defaults, and state-sync boundaries need maintainer decisions first.
