## Summary

## What This Changes

## Best Possible Solution

Land the narrow task-override fix after the contributor adds redacted real restart/recovery proof and the small comment wording is clarified during maintainer review.

## Reproduction Assessment

Yes from source inspection: current main dispatches steered/wake/resume text but remaps the persisted run without updating `runRecord.task`, and recovery later rebuilds from that stored task. I did not run the live gateway restart flow.

## Solution Assessment

Yes. Passing an optional task override through `replaceSubagentRunAfterSteer` is the narrowest maintainable fix because existing callers can keep the old task while steer, wake, and orphan-resume paths persist the text they actually dispatched.
