## Summary

Keep open: current main and v2026.6.10 still preserve stale replacement-run task text, while this PR is a focused, well-proven fix that still needs maintainer acceptance of the new persisted-task semantics before merge.

## What This Changes

This PR adds an optional replacement-run `task` override and threads steer, descendant wake, orphan recovery, and completed-session follow-up prompt text through the agents/gateway call sites.

## Best Possible Solution

Land the focused registry-level task override after maintainers accept that replacement runs should persist the dispatched continuation prompt as the canonical restart task.

## Reproduction Assessment

Yes, at source level: current main preserves the previous `task` during run replacement while `buildResumeMessage` reconstructs restart text from that persisted task. I did not run a live failing restart in this read-only review.

## Solution Assessment

Yes as an implementation shape, pending product acceptance. Adding the override at `replaceSubagentRunAfterSteer` and passing the dispatched prompt from each replacement caller is the narrowest maintainable fix I found; a separate continuation field would be a larger state-model decision.
