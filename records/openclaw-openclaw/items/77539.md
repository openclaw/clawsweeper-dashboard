## Summary

Keep this PR open: current main still carries the stale pre-steer task through the restart recovery path, and the proposed task override is a narrow, backwards-compatible fix with no blocking review findings from this read-only sweep.

## What This Changes

The PR adds an optional task override to subagent run replacement, threads steered/wake/orphan-resume messages through the replacement paths, adds regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the narrow task-override path with the regression tests so replacement runs persist the exact message that was dispatched for restart recovery.

## Solution Assessment

Yes. Adding an optional `task` override at `replaceSubagentRunAfterSteer` is the narrowest maintainable fix because existing callers keep old behavior while steer, wake, and orphan-resume paths persist the dispatched prompt text.
