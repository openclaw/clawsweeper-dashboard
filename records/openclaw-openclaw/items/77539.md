Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve stable task text after orphan resumes:** `src/agents/subagent-orphan-recovery.ts:158`
  - body: `buildResumeMessage()` treats `task` as the original task when constructing the recovery prompt. By storing the generated `resumeMessage` back into the replacement run, a second gateway restart before that resumed run finishes will wrap the previous synthetic resume prompt as the original task, nesting recovery text and losing the stable user or steered task. Leave the orphan-resume replacement task unset, or carry a separate stable task field, and add a repeated-recovery regression test.
  - confidence: 0.9
