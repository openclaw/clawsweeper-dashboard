## Summary

Keep open: the PR targets a real source-reproducible subagent restart bug, but it is not merge-ready because the orphan recovery path can persist a synthetic resume prompt as task state and the supplied proof does not show an actual steer-then-restart recovery.

## What This Changes

The PR adds an optional task override to subagent run replacement, passes steer/wake/orphan-resume messages through it, adds registry regression coverage, and updates the changelog.

## Best Possible Solution

Persist task overrides only for user-dispatched steer and wake continuation prompts, keep orphan recovery's stored task as the actual underlying task, add repeated orphan-restart coverage, and require redacted runtime recovery proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main dispatches the steered message but replacement records retain source.task, and restart recovery rebuilds from runRecord.task. I did not run a live gateway restart in this read-only pass.

## Solution Assessment

No, not as written. The optional task override is the right narrow shape for steer and wake replacement, but passing the synthetic orphan-resume prompt as task should be removed or covered by repeated-restart proof that preserves task semantics.
