## Summary

Keep open: the PR targets a real source-reproducible subagent restart bug, but the patch now introduces a concrete orphan-recovery task-state regression and the supplied proof still does not show an actual steer-then-restart recovery using the steered task text.

## What This Changes

The PR adds an optional task override to subagent run replacement, threads steer/wake/orphan-resume messages into replacement records, adds registry regression tests, and updates the changelog.

## Best Possible Solution

Persist the task override for steer and wake paths, keep orphan recovery’s stored task as the current user/task text instead of the synthetic resume wrapper, add repeated orphan-restart coverage, and land after redacted runtime recovery proof.

## Reproduction Assessment

Yes, source-reproducible: current main dispatches the steered message but replacement records retain source.task, and restart recovery rebuilds from runRecord.task. I did not run a live gateway restart in this read-only pass.

## Solution Assessment

No, not as written. The optional task override is the right narrow shape for steer and wake replacement, but passing the synthetic orphan-resume prompt as task should be removed or covered by a repeated-restart proof that preserves task semantics.
