## What This Changes

The PR adds an optional task override to subagent run replacement so steered, wake, and orphan-resume messages can be persisted for restart recovery.

## Best Possible Solution

Land the narrow task-override fix after the contributor adds real restart/recovery proof and maintainer review confirms the rebased patch against current main.

## Reproduction Assessment

Yes from source inspection: current main dispatches the steered message but remaps the persisted run without updating `runRecord.task`, and restart recovery later rebuilds the prompt from that stored task. I did not run the live gateway restart flow in this read-only review.

## Solution Assessment

Yes. Adding an optional task override at `replaceSubagentRunAfterSteer` is the narrowest maintainable fix because old callers preserve the previous task while steer, wake, and orphan-resume paths can persist the prompt text they actually dispatch.
