## Best Possible Solution

Land the focused PR after maintainers explicitly accept that replacement-run task state should track the dispatched continuation prompt for restart recovery.

## Reproduction Assessment

Yes, source-reproducible: current main preserves the prior run task through replacement while orphan recovery rebuilds the resume prompt from that stored task. I did not run a live failing current-main restart in this read-only review.

## Solution Assessment

Yes, with a maintainer semantic decision: placing the override in replaceSubagentRunAfterSteer and passing the dispatched prompt from each replacement caller is the narrowest maintainable fix I found. A separate continuation-prompt field would preserve the old meaning of task, but current recovery already reads task and this PR keeps the repair localized.
