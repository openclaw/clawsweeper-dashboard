## What This Changes

The PR threads an optional replacement-run `task` override through subagent steer, descendant wake, orphan recovery, and completed-session follow-up paths so restart redispatch uses the latest dispatched prompt.

## Best Possible Solution

Land the focused registry-level task override after maintainers explicitly accept that replacement runs should persist the dispatched continuation prompt as the canonical restart task.

## Reproduction Assessment

## Solution Assessment

Yes as an implementation shape, pending maintainer acceptance. Adding the override at `replaceSubagentRunAfterSteer` and passing the dispatched prompt from each replacement caller is the narrowest maintainable fix I found; a separate continuation field would be a larger state-model decision.
