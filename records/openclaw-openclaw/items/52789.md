## Summary

Keep open. Current main still does not implement cron `delivery.mode: "session"`; the related external workaround proves the workflow is useful, but not that OpenClaw core has the requested deterministic cron delivery mode.

## What This Changes

Add a first-class cron delivery mode that routes completed cron output to another internal session via `sessions_send` instead of chat announce, webhook, or model-issued tool calls.

## Best Possible Solution

Keep this open for maintainer product/API review; if approved, add an additive cron session-delivery contract that reuses `sessions_send` authorization, visibility, provenance, diagnostics, and thread-target semantics across runtime, protocol, UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. Static current-main checks show cron types, planner, gateway schema, Control UI, and docs still expose only `none`, `announce`, and `webhook`, so the requested capability is absent with high confidence.

## Solution Assessment

Unclear until maintainers decide the public API and policy semantics. A framework-owned mode is plausible for reliability, but the maintainable path should reuse or mirror `sessions_send` visibility, A2A allow-list, provenance, thread restrictions, and diagnostics rather than adding a loose cron-only send branch.
