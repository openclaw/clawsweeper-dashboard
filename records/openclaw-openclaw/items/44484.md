## Summary

Keep open. Current main has useful session-scoped effective-tool visibility, but the requested strict inheritance mode, declared-vs-effective diff, and general clipping reason trace are still missing.

## What This Changes

The issue requests strict declared-vs-effective tool inheritance, per-session effective/removed tool visibility, and structured reasons when runtime layers block a tool.

## Best Possible Solution

Add an additive policy-resolution trace that records declared inputs, effective tools, removed tools, and removal layer/reason, then add strict inheritance as an explicit fail-closed mode after policy precedence is designed across runtime, protocol, UI, and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows the final effective inventory exists, while the strict inheritance key, declared/removed comparison, and general removal reason fields are absent from config, protocol, and the policy pipeline.

## Solution Assessment

No. The current `tools.effective` path is a useful partial solution, but the requested strict mode should wait for a clear cross-layer policy and trace contract rather than being patched into one runtime step.
