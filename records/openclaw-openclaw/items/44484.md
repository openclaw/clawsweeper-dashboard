## Summary

Keep open. Current main has session-scoped effective tool visibility through `tools.effective` and the Control UI, plus a narrow sandbox blocked-message helper, but it still does not provide strict declared-vs-effective inheritance or a generic per-tool clipping/removal trace across profile, provider, agent, group, sandbox, subagent, runtime, approval, and transport layers. The open #59898 PR is adjacent prompt-truthfulness work, not a closing implementation for this umbrella request.

## Best Possible Solution

Keep this open as the umbrella for tool capability determinism. The best path is a maintainer-approved design for a generic policy-resolution trace that records declared policy inputs, effective tools, removed tools, removal layer/source/reason, and user-facing remediation where possible; expose it through `tools.effective`, Control UI, and debug/status/log surfaces; then add strict inheritance as an explicit mode that either preserves parity or fails visibly when runtime/security layers cannot honor it.
