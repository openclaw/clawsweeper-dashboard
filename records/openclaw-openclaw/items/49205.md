## Summary

Keep open. Current main and the latest release still do not contain a supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the remaining fix needs maintainer decisions about the Open WebUI sync boundary before implementation.

## What This Changes

The issue asks OpenClaw to make Control UI-originated turns in an Open WebUI-backed chat appear in Open WebUI's visible chat history as normal message bubbles.

## Best Possible Solution

Define a supported Open WebUI synchronization boundary first, then add narrowly scoped future-turn mirroring with live Open WebUI proof and an explicit policy for already-missed historical turns.

## Reproduction Assessment

No live high-confidence reproduction was run in this review because I did not start Open WebUI Docker. Source inspection and the issue's detailed repro make the gap source-reproducible: `chat.send` updates OpenClaw transcript/context while no current path writes Open WebUI visible history.

## Solution Assessment

No; the direct SQLite/container write proposed in the issue is a plausible local repair but not yet the best proven maintainer solution. Maintainers should first choose the supported Open WebUI API or DB boundary, opt-in configuration, failure behavior, and historical backfill policy.
