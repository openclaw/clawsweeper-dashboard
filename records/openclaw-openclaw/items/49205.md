## Summary

Keep open. Current main still only persists/broadcasts OpenClaw transcript state for `chat.send` and has no Open WebUI visual-history mirror, while the remaining fix needs maintainer product/security judgment because it would write third-party Open WebUI storage or define a new integration seam.

## What This Changes

The issue requests that Control UI turns written into an Open WebUI-backed chat be mirrored into Open WebUI’s visible chat history, not only OpenClaw transcript/model context.

## Best Possible Solution

Define an owner-approved Open WebUI visual-sync contract, preferably through a stable API/plugin/event seam rather than direct DB mutation, then add tested mirroring for new Control UI turns and document separate manual backfill for old chats.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection matches the reported missing mirror path, and the reporter supplied detailed local runtime verification, but a real Open WebUI plus Control UI replay was not run.

## Solution Assessment

No. The proposed direct `docker exec`/SQLite helper may prove the concept, but the best maintainable fix needs maintainer approval of the integration boundary, configuration, security posture, and Open WebUI schema/version contract.
