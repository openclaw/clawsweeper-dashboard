## Summary

Keep open. Current main still updates OpenClaw transcript/model-context paths for `chat.send`, but I found no implementation that mirrors external Control UI turns into Open WebUI's own visible chat-history store; the remaining fix also needs maintainer judgment on direct third-party DB writes versus a supported plugin/runtime hook.

## What This Changes

The issue asks OpenClaw to make successful external Control UI turns in an Open WebUI-backed UUID chat appear in Open WebUI's visible chat history as well as OpenClaw transcript/model context.

## Best Possible Solution

Add a supported post-turn sync path for successful external `chat.send` calls into WebUI-backed UUID sessions, or define a narrow plugin/runtime hook if OpenClaw core should not mutate Open WebUI storage directly; cover both transcript context and Open WebUI visible history in regression or live-smoke validation.
