## Summary

Keep open. Current main and v2026.6.10 still write Control UI `chat.send` turns through OpenClaw transcript/session paths and live Gateway delivery, but I found no supported path that mirrors those turns into Open WebUI's own visible-history store; the linked open PRs cover adjacent session identity/header issues rather than this visual-history sync.

## What This Changes

The issue asks for Control UI turns in an Open WebUI-backed chat to be mirrored into Open WebUI visible chat history as normal message bubbles.

## Best Possible Solution

Define a maintainer-approved, opt-in Open WebUI visible-history sync and backfill contract, then implement it with focused gateway coverage and Docker Open WebUI proof, or explicitly document that this co-editing path is unsupported.

## Reproduction Assessment

Yes, at source level: current `chat.send` persists and broadcasts through OpenClaw session/transcript paths, and targeted source search found no current gateway path that writes Open WebUI visible-history rows for Control UI turns.
