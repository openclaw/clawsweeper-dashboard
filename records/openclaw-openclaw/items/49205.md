## Summary

Keep open. Current main still records Control UI `chat.send` turns into OpenClaw session/transcript state and broadcasts them to OpenClaw clients, but I found no supported runtime path that mirrors those turns into Open WebUI's own visible-history store; the related UUID-session work is adjacent and explicitly leaves this DB/visual sync out of scope.

## What This Changes

The issue asks for Control UI-originated turns in Open WebUI-backed chats to appear in Open WebUI visible chat history as normal message bubbles.

## Best Possible Solution

Define a supported, opt-in Open WebUI visible-history sync contract for future external Control UI turns, with explicit upgrade/backfill policy and Docker Open WebUI proof.

## Reproduction Assessment

Yes at source level, but not by a live Docker run in this read-only review. The reported local proof plus current `chat.send` and transcript code show no path that updates Open WebUI's visible-history store after a Control UI-originated turn.

## Solution Assessment

Unclear. The proposed direct DB mirror is plausible for the observed setup, but the best fix needs maintainer agreement on the supported Open WebUI state boundary, opt-in/config shape, and historical backfill policy.
