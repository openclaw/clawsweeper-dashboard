## Summary

Keep open. Current main and the latest release still lack any supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the related open UUID-session PR explicitly leaves this visual-sync behavior out of scope.

## What This Changes

The issue asks OpenClaw to make Control UI-originated turns in an Open WebUI-backed chat appear in Open WebUI's visible chat history as normal user/assistant bubbles.

## Best Possible Solution

Define a supported Open WebUI synchronization boundary first, then implement future-turn mirroring with live Open WebUI proof and a separate decision for historical backfill.

## Reproduction Assessment

No live high-confidence reproduction was run here. The reporter's detailed repro plus current source absence of an Open WebUI visible-history write path make the gap source-reproducible.

## Solution Assessment

No; the direct SQLite/container write proposed in the issue is plausible but not yet the best proven maintainer solution. Maintainers should choose the supported Open WebUI API or DB boundary, opt-in/failure behavior, and backfill policy first.
