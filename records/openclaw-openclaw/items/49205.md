## Summary

Keep open. Current main still has no Open WebUI visual-history mirror for Control UI `chat.send` turns, and the remaining fix needs maintainer product/security direction because it would define how OpenClaw writes or syncs a third-party Open WebUI history store.

## What This Changes

The issue requests that Control UI turns written into an Open WebUI-backed chat also appear in Open WebUI’s visible chat history.

## Best Possible Solution

Define an owner-approved Open WebUI visual-sync contract, preferably through a stable API/plugin/event seam, then add tested mirroring for new Control UI turns and document separate manual backfill for old chats.

## Reproduction Assessment

No, not as a high-confidence live reproduction in this review. Source inspection matches the reported missing mirror path and the reporter supplied detailed local runtime proof, but I did not run a real Open WebUI plus Control UI replay against current main.

## Solution Assessment

No, the proposed direct `docker exec`/SQLite helper is not clearly the best maintainable solution. It proves the storage gap, but the safer path is a maintainer-approved sync/API boundary with explicit configuration and schema/version handling.
