## Summary

Keep open. Current main still has the Control UI `chat.send` path persisting and broadcasting OpenClaw transcript state, but no supported reverse-sync path into Open WebUI visible history; the related Open WebUI UUID-session PR explicitly leaves DB/visual mirroring as separate follow-up work.

## What This Changes

Request that Control UI turns written into an Open WebUI-backed chat also appear as normal message bubbles in Open WebUI visible chat history.

## Best Possible Solution

Define a supported, opt-in Open WebUI visible-history sync contract for future external Control UI turns, with an explicit backfill policy and Docker Open WebUI proof.

## Reproduction Assessment

Yes at source level, but not by a live run in this read-only review. The reported steps plus current `chat.send` and transcript code show no path that updates Open WebUI's visible-history store after a Control UI-originated turn.

## Solution Assessment

Unclear. The proposed direct DB mirror is plausible, but the best fix needs maintainer agreement on the supported Open WebUI state boundary, opt-in/config shape, and historical backfill policy.
