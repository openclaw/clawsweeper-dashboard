## Summary

Keep open. Current main and v2026.6.6 still lack a supported path that mirrors Control UI-originated turns into Open WebUI's own visible chat store, and the remaining work needs maintainer direction on the external Open WebUI sync boundary before implementation.

## What This Changes

The issue requests that Control UI-originated turns in an Open WebUI-backed chat appear in Open WebUI's visible chat history as normal user and assistant messages.

## Best Possible Solution

Define a supported Open WebUI synchronization boundary first, then implement narrowly scoped future-turn mirroring with live Open WebUI proof and an explicit policy for already-missed historical turns.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. Source inspection and the detailed reporter repro make the gap source-reproducible: `chat.send` persists OpenClaw transcript/model context while no current path writes Open WebUI visible history.

## Solution Assessment

No; the direct SQLite/container write proposed in the issue is a plausible local repair, but not yet the best maintainer-approved solution. Maintainers should first choose the supported Open WebUI API or DB boundary, opt-in configuration, failure behavior, and historical backfill policy.
