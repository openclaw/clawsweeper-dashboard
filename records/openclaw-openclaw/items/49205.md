## Summary

Keep open. Current main and the latest release still do not provide a supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the related UUID-session PR is still open and explicitly leaves this visual-sync behavior out of scope.

## What This Changes

Request that Control UI-originated turns for Open WebUI-backed UUID chats update both OpenClaw transcript/model context and Open WebUI visible chat history.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract for new external turns, preferably through a supported Open WebUI boundary when available, plus a documented backfill path for already-affected chats.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection plus the reporter's local proof gives a credible path: Control UI chat.send updates OpenClaw transcript and Gateway subscribers, while no Open WebUI visible-history update path exists.

## Solution Assessment

No, the direct Docker exec/SQLite diff is useful proof but not the best maintainable solution by itself. A safe implementation needs a supported sync boundary, opt-in configuration, schema/version handling, live proof, and a backfill policy.
