## Summary

Keep open. Current main still lacks any supported path that mirrors Control UI-originated turns into Open WebUI's own visible chat store, and the reporter's local SQLite proof is useful evidence but needs maintainer product/security direction before it becomes an OpenClaw implementation.

## What This Changes

The issue asks OpenClaw to make external Control UI turns in Open WebUI-backed chats appear in Open WebUI's visible message history, not only in OpenClaw transcript/model context.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract for new external turns, plus a documented backfill path for already-affected chats.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection still matches the missing mirror path, and the reporter supplied detailed local verification, but I did not replay a real Open WebUI plus Control UI session.

## Solution Assessment

No, the direct Docker exec/SQLite proof is not the best maintainable solution by itself. A durable fix should first choose the supported Open WebUI sync boundary, opt-in configuration, schema/version contract, and backfill policy.
