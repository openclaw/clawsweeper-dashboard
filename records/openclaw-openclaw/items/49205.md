## Summary

Keep open. Current main still has no supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the existing Open WebUI/session work only covers API connectivity and UUID session identity, not Open WebUI chat-history persistence.

## What This Changes

The issue requests that Control UI-originated turns in an Open WebUI-backed chat be mirrored into Open WebUI visible chat history as normal user and assistant messages.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract for external turns, preferably through a supported Open WebUI API when available, with schema/version handling, live proof, and a documented backfill path for already affected chats.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection plus the reporter's local proof gives a credible path: Control UI chat.send updates OpenClaw transcript and Gateway subscribers, while no Open WebUI visible-history update path exists on current main.

## Solution Assessment

No, the direct Docker exec/SQLite local diff is useful proof but not the best maintainable solution by itself. A safe fix needs a supported sync boundary, opt-in configuration, schema/version handling, live proof, and a backfill policy.
