## Summary

Keep open. Current main still appears to have the reported OpenClaw transcript/Open WebUI visible-history split, and the remaining fix needs maintainer design review because the proposed local proof writes third-party Open WebUI storage directly.

## What This Changes

The issue asks OpenClaw to make Control UI turns sent into an Open WebUI-backed chat appear in Open WebUI's visible chat history, not only in OpenClaw transcript/model context.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract for new external turns, with a separate documented backfill path for already-missed history.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection still matches the missing mirror path, and the reporter supplied detailed local verification, but I did not replay a real Open WebUI plus Control UI session.

## Solution Assessment

No, the direct `docker exec`/SQLite proof is not the best maintainable solution by itself. The durable fix should first choose a supported Open WebUI sync boundary, configuration model, schema/version contract, and backfill policy.
