## Summary

Keep open. Current main and the latest release still lack any supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the related UUID/session-alias work does not cover Open WebUI chat-history persistence.

## What This Changes

The issue reports that Control UI-originated messages in an Open WebUI-backed chat reach OpenClaw transcript/model context but do not appear in Open WebUI visible chat history.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract for external turns, preferably through a supported Open WebUI API when available, with schema/version handling, live proof, and a documented backfill path for already affected chats.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection plus the reporter's local proof gives a credible path: chat.send updates OpenClaw transcript and Gateway subscribers, while no Open WebUI visible-history update path exists on current main.

## Solution Assessment

No, the direct Docker exec/SQLite local diff is useful proof but not the best maintainable solution by itself. A safe fix needs a supported sync boundary, opt-in configuration, schema/version handling, live proof, and a backfill policy.
