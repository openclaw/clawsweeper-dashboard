## Summary

Keep open. Current main still writes Control UI turns to OpenClaw transcript/live chat surfaces but has no supported reverse-sync path into Open WebUI visible history, and the fix needs a maintainer decision about the external Open WebUI state boundary.

## What This Changes

The issue asks OpenClaw to mirror Control UI-originated turns for Open WebUI-backed chats into Open WebUI visible history as well as OpenClaw transcript/model context.

## Best Possible Solution

Define a supported, opt-in Open WebUI visible-history sync contract that mirrors only external Control UI turns for future messages, states the backfill policy, and is validated with the Docker Open WebUI user path.

## Reproduction Assessment

No live end-to-end reproduction was run in this read-only sweep. Source inspection gives a high-confidence repro path: Control UI uses `chat.send`, current main persists OpenClaw transcript/live events, and there is no Open WebUI visible-history mirror path.

## Solution Assessment

Unclear. The locally proposed direct DB mirror is plausible, but the best fix needs maintainer agreement on a supported Open WebUI sync boundary, opt-in/config shape, and historical backfill policy before implementation.
