## Summary

Keep open. Current main and the latest release still lack a supported reverse-sync path from Control UI `chat.send` into Open WebUI visible history, and the remaining fix needs maintainer decisions about the external Open WebUI state boundary and backfill policy.

## What This Changes

The issue asks OpenClaw to make Control UI-originated turns in an Open WebUI-backed chat appear in Open WebUI's visible chat history, not only in OpenClaw transcript/model context.

## Best Possible Solution

Define a supported Open WebUI synchronization boundary, then implement future-turn mirroring with focused eligibility tests and live Open WebUI Docker proof; handle old missed turns through an explicit repair/backfill path if approved.

## Reproduction Assessment

No live Docker repro was run in this read-only review. Source inspection gives a high-confidence gap: `chat.send` and `chat.history` handle OpenClaw transcript/live delivery, while current main has no Open WebUI mirror helper or DB/API sync path.

## Solution Assessment

Unclear/no. The locally proposed direct SQLite/container write is a plausible future-turn repair, but the best maintainer solution needs a supported Open WebUI API or DB boundary, opt-in/config/security behavior, and historical backfill policy.
