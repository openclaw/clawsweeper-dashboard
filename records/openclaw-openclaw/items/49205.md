## Summary

Keep open. Current main and the latest release still lack a supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the remaining work needs a maintainer decision about the Open WebUI sync boundary before implementation.

## What This Changes

The issue asks OpenClaw to keep Open WebUI visible chat history in sync when Control UI writes into the same Open WebUI-backed chat.

## Best Possible Solution

Choose a supported opt-in Open WebUI sync boundary, then implement future-turn mirroring with focused gateway coverage, live Open WebUI proof, and a separate policy for historical backfill.

## Reproduction Assessment

No live high-confidence repro was run in this read-only review. Source inspection plus the reporter's detailed local verification make the gap source-reproducible: OpenClaw records chat context, but no current code writes Open WebUI visible history.

## Solution Assessment

No; the issue's direct `docker exec` SQLite mirror is a plausible local repair but not yet the best maintainable OpenClaw solution. Maintainers should first choose the supported API or DB boundary, opt-in configuration, failure behavior, and historical backfill policy.
