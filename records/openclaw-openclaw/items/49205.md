## Summary

Keep open. Current main still lacks an Open WebUI visual-history mirror for Control UI turns, and the remaining work needs maintainer product/security judgment because the proven local approach writes third-party Open WebUI storage directly.

## What This Changes

Request that Control UI messages sent into an Open WebUI-backed chat appear in Open WebUI visible history, not only in OpenClaw transcript/model context.

## Best Possible Solution

Define an owner-approved, opt-in Open WebUI visual-sync contract that mirrors new external turns through a stable API/plugin/event boundary and documents historical backfill separately.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection still matches the missing mirror path, and the reporter supplied detailed local verification, but this review did not replay a real Open WebUI plus Control UI session.

## Solution Assessment

No, the direct `docker exec`/SQLite proof is not the best maintainable solution by itself. The durable fix should first choose a supported Open WebUI sync boundary, configuration model, and schema/version contract.
