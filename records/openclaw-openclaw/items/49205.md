## Summary

Keep open. Current main and the latest release still lack a supported path that mirrors Control UI-originated turns into Open WebUI's visible chat store, and the open UUID-session PR explicitly excludes this visual-sync behavior.

## What This Changes

The issue requests that Control UI-originated messages in Open WebUI-backed chats be mirrored into Open WebUI's visible chat history as well as OpenClaw transcript/model context.

## Best Possible Solution

Define a supported Open WebUI synchronization boundary first, then add narrowly scoped future-turn mirroring with live Open WebUI proof and an explicit policy for already-missed historical turns.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. Source inspection and the issue's detailed repro make the gap source-reproducible because current `chat.send` updates OpenClaw transcript/context while no current path writes Open WebUI visible history.

## Solution Assessment

No; the direct SQLite/container write proposed in the issue is a plausible local repair but not yet the best proven maintainer solution. Maintainers should first choose the supported Open WebUI API or DB boundary, opt-in configuration, failure behavior, and historical backfill policy.
