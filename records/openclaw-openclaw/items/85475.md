## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Gate live cursor advances on full catchup success:** `extensions/imessage/src/monitor/monitor-provider.ts:958`
  - body: `querySucceeded` does not mean every chat caught up: `runIMessageCatchup` catches per-chat `messages.history` failures and continues. A newer live row can then advance the persisted cursor past rows from the failed chat, so those offline rows are skipped on restart; carry an all-chats-complete flag or keep live advancement disabled after any per-chat history failure.
  - confidence: 0.91
- **[P2] Advance coalesced buckets to the max source row:** `extensions/imessage/src/monitor/monitor-provider.ts:359-369`
  - body: `maybeAdvanceLiveCatchupCursor` reads only `message.id`, but multi-entry debounce flushes call `handleMessageNow` with `combineIMessagePayloads`, which preserves the first payload id while using the latest timestamp. With `coalesceSameSenderDms`, later rows in the same handled bucket can replay after restart; track the max source rowid/date or advance each source row after the combined turn succeeds.
  - confidence: 0.84
