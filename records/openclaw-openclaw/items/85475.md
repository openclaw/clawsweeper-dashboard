## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Serialize live cursor read-modify-write:** `extensions/imessage/src/monitor/catchup.ts:292-304`
  - body: `advanceIMessageCatchupCursor` loads, checks, and saves the cursor without the per-file queue used by other persistent stores. Live iMessage notifications are fire-and-forget and the debouncer only serializes same-key work, so two different chats can both read the old cursor and a later lower-row write can overwrite a higher cursor, leaving already handled rows replayable after restart. Wrap this read-modify-write in a per-account/file queue or equivalent atomic helper and add a regression for out-of-order concurrent advances.
