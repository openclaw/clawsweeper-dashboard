## Summary

Keep open: this is a concrete shipped Discord regression report, and current main has adjacent readiness/status hardening but not enough proof that the 2026.5.2/2026.5.3 inbound READY hang is fixed.

## What This Changes

The issue reports a Discord channel regression where the gateway WebSocket stays before READY on 2026.5.2/2026.5.3, so inbound MESSAGE_CREATE events never wake the agent while outbound REST sends still work.

## Best Possible Solution

Keep the issue open and isolate whether the packaged Discord gateway fails before HELLO, IDENTIFY, READY, or listener dispatch; then land a narrow Discord plugin fix with regression coverage and a changelog entry.

## Reproduction Assessment

No. The issue includes concrete shipped-package steps, but this review did not have Discord credentials and current-main source inspection did not expose a deterministic failing READY path.

## Solution Assessment

Unclear. The maintainable fix likely belongs in the Discord plugin gateway startup/readiness path, but the root cause should be isolated before changing intents, timers, or status semantics.
