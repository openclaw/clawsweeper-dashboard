## Summary

Keep open. Current main still has no configurable `memorySearch.ignorePaths` or equivalent exclusion contract, and the observable problem remains: built-in memory indexing walks `memory/` recursively, QMD default collections include `memory/**/*.md`, and the watcher ignores only a fixed set of infrastructure directories that does not include `dreaming`. Related dreaming hardening addresses promotion/self-ingestion, not ordinary search indexing or QMD collection scope.

## What This Changes

The issue requests a configurable `memorySearch.ignorePaths` setting so generated `memory/dreaming/**` markdown can be excluded from memory indexing and watching without hardcoding dreaming as a default exclusion.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with shared, tested matching semantics and wire it through config schema/types, resolved agent config, built-in file listing, built-in watcher filtering, QMD collection/update scope, reindex-state invalidation, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless a maintainer explicitly chooses a default exclusion policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a markdown file under `memory/dreaming/light/` or `memory/dreaming/deep/`: current `listMemoryFiles` recursively walks `memory/`, and QMD default memory collections bind `memory` with `**/*.md`, with no configurable ignore setting to prevent indexing.

## Solution Assessment

Yes, the config-based direction is the narrowest maintainable solution for this report if implemented with one shared glob/predicate model across backends. Hardcoding `dreaming` or changing only the watcher would leave either policy flexibility or search-index behavior unresolved.
