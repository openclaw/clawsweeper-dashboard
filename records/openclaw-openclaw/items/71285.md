## Summary

Keep open. Current main and the latest release still lack any memory-search exclusion config, while builtin and QMD memory indexing still include generated `memory/dreaming/**/*.md` reports; this needs maintainer product direction before adding a public config/indexing contract.

## What This Changes

The issue requests a configurable `memorySearch` exclusion path so generated dreaming reports and similar directories can be kept out of memory search indexing and watching.

## Best Possible Solution

Define one documented generated-artifact exclusion contract for memory search, then implement it consistently across schema/types, builtin scanning, QMD collections/update behavior, watcher filtering, docs, status output, tests, and reindex guidance.

## Reproduction Assessment

Yes, at source level: builtin indexing recurses through `memory/`, while QMD default collections include `memory/**/*.md`, so Markdown files under `memory/dreaming/` are eligible for the memory corpus. I did not run a live `memory_search` query in this read-only review.

## Solution Assessment

Unclear. A config-based exclusion is plausible, but the best fix needs maintainer agreement on matcher semantics, defaults, QMD/builtin parity, existing-index cleanup, and whether generated dream reports should instead be tagged or ranked separately.
