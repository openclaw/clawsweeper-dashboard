## Summary

Keep open. Current main still has no `memorySearch.ignorePaths` or equivalent exclusion contract, and both builtin and QMD memory indexing can include Markdown under `memory/dreaming/**`; the related dreaming PRs address promotion into durable memory, not ordinary search/index scope.

## What This Changes

Request adds a configurable `memorySearch.ignorePaths` exclusion surface so `memory/dreaming/**` and other directories can be omitted from memory search indexing and watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent exclusion contract with shared matching semantics across schema/types, resolved config, builtin file enumeration, memory watchers, QMD setup, reindex state, generated docs, and focused tests; keep dreaming exclusion opt-in unless maintainers choose a default.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a `memory/dreaming/light/example.md` file sits under the recursively walked `memory` tree, and QMD default collections index `memory/**/*.md` with no ignore config.

## Solution Assessment

Yes, a config-backed ignore-path contract is the narrow maintainable direction. A hardcoded dreaming-only skip or watcher-only glob would leave extra-path noise and backend divergence unresolved.
