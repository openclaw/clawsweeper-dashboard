## Summary

Keep open. Current main and the latest release still lack a `memorySearch` ignore/exclusion config, and both builtin and QMD memory indexing still include `memory/dreaming/**/*.md`; the community `memory-lancedb-dreaming` plugin is useful context but does not resolve the core memory-search contract request.

## What This Changes

Request adding configurable memory-search ignore paths so generated `memory/dreaming/` reports can be excluded from indexing, search results, and watcher coverage.

## Best Possible Solution

Define one documented generated-artifact exclusion contract for memory search, then implement it consistently across schema/types, builtin scanning, QMD collections/update behavior, watcher filtering, docs, status output, tests, and reindex guidance.

## Reproduction Assessment

Yes at source level. Current main still recursively indexes `memory/`, QMD still includes `memory/**/*.md`, and the requested exclusion config does not exist; I did not run a live memory-search query in this read-only review.

## Solution Assessment

Unclear. A config-based exclusion is plausible, but the best fix needs maintainer agreement on matcher semantics, defaults, QMD/builtin parity, existing-index cleanup, and whether dreaming reports should instead be tagged or ranked separately.
