## Summary

Keep open. Current main still has no `memorySearch.ignorePaths` or equivalent exclusion contract, and both builtin and QMD memory indexing can still include generated Markdown under `memory/dreaming/**`. The related dreaming promotion fixes address durable-memory contamination, not ordinary search-index scope.

## What This Changes

Add a configurable memory-search ignore path mechanism so generated dreaming reports and other unwanted directories can be excluded from indexing and file watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent exclusion contract, merge it across defaults and per-agent config, and apply it consistently to builtin file enumeration, memory-core watchers, QMD collection/index setup or filtering, reindex state, schema/help/generated docs, and focused tests. Keep dreaming exclusion opt-in unless maintainers explicitly choose a default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: create `memory/dreaming/light/example.md`; current builtin listing recursively walks `memory`, QMD default collections index `memory/**/*.md`, and neither path has a configured exclusion for the `dreaming` segment.

## Solution Assessment

Yes. A config-backed ignore-path contract is the narrow maintainable fix because it covers dreaming report pollution and broader dependency-folder noise without hardcoding only one OpenClaw-owned directory; the implementation should avoid the earlier #6362 mismatch where watcher and indexer glob behavior could diverge.

## Repair Work Prompt

Fix the memory-search exclusion gap described in https://github.com/openclaw/openclaw/issues/71285. Add a documented `memorySearch.ignorePaths` or equivalent config-backed exclusion contract for memory search. Merge defaults and per-agent values, include it in resolved memory-search config, and apply it consistently to builtin `listMemoryFiles`/recursive walking, memory-core watcher ignores, QMD default/custom collection indexing or prefiltering where supported, and reindex state so changing ignore patterns invalidates stale scope. Avoid the earlier #6362 pitfall where chokidar treats patterns as globs but index enumeration only extracts directory names; use one shared matcher or equivalent consistent semantics. Add focused tests for builtin indexing, QMD config/collections or filtering, watcher behavior, resolver/schema docs, reindex-state invalidation, and config docs/help. Add a user-facing changelog entry with appropriate non-bot credit. Do not change dreaming promotion heuristics or hardcode unrelated OpenClaw-owned paths as universal policy.
