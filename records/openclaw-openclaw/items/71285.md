## Summary

Keep open. Current main still lacks a configurable `memorySearch.ignorePaths` contract, and the merged dreaming hardening only covers promotion/self-ingestion paths rather than built-in file enumeration, watcher scope, or QMD default `memory/**/*.md` collection scope. The closed #75277 is a narrower QMD symptom, while #6362 was a closed unmerged prior attempt, so this issue remains the canonical repair target.

## What This Changes

The issue requests configurable `memorySearch.ignorePaths` support so users can exclude `memory/dreaming/**` and other paths from memory search indexing and watcher scope.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with consistent matching semantics across config schema/types, resolved agent config, built-in file listing, built-in watcher filtering, reindex-state hashing, QMD default collection/update scope where feasible, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: create a markdown file under `memory/dreaming/light/` and use current defaults, where `listMemoryFiles` recursively walks `memory/` and QMD default collections index `memory/**/*.md`.

## Solution Assessment

Yes. A config-based exclusion is the narrow maintainable direction because it solves dreaming report pollution and the broader extra-path dependency-folder problem without hardcoding one OpenClaw-owned directory as policy.
