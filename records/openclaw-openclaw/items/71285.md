## Summary

Keep open. Current main still lacks a memory-search exclusion contract, and both builtin and QMD memory search can still index or return `memory/dreaming/**`; the merged dreaming PRs fixed promotion contamination, not ordinary recall/search scope.

## What This Changes

Request to add a configurable `memorySearch` ignore or exclusion list so `memory/dreaming/**` and other noisy paths can be excluded from memory search indexing and watchers.

## Best Possible Solution

Add a documented exclusion contract shared by schema/types, config resolution, builtin indexing, QMD collection/search behavior, memory-core watchers, reindex state, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: add a Markdown file under `memory/dreaming/light/`; builtin indexing walks `memory/` recursively, and QMD default collections index `memory/**/*.md` without a dream-report filter.

## Solution Assessment

Unclear until maintainers choose the public config semantics and defaults. A config-backed exclusion contract is likely the maintainable direction, but a hardcoded dreaming-only skip or divergent glob handling would be weaker.
