## Summary

Keep open. Current main still lacks a `memorySearch.ignorePaths` or equivalent exclusion contract, and both builtin and QMD memory search paths can still index `memory/dreaming/**`; the related merged dreaming fixes address promotion hygiene, not ordinary recall/search scope.

## What This Changes

Request to add config-driven memory search ignore paths so users can exclude `memory/dreaming/**` and other directories from indexing, search results, and watcher scope.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent exclusion contract with shared matching semantics across schema/types, resolved config, builtin indexing, memory-core watchers, QMD collection setup/search behavior, reindex state, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: place a Markdown file under `memory/dreaming/light/`; builtin indexing walks `memory/` recursively, and QMD default collections index `memory/**/*.md` without a dream-report filter.

## Solution Assessment

Unclear until maintainers choose the public config semantics and defaults. A config-backed ignore contract is the likely maintainable direction, while a hardcoded dreaming-only skip would leave extra-path noise and backend divergence unresolved.
