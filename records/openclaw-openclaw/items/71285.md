## Summary

Keep open. Current main still lacks a `memorySearch` ignore/exclude config, and both builtin and QMD memory search can include optional `memory/dreaming/**` phase reports; the related merged fixes cover promotion hygiene, not recall/index scope.

## What This Changes

Request to add a configurable memory-search exclusion list so users can keep `memory/dreaming/**` and other directories out of builtin/QMD indexing and watchers.

## Best Possible Solution

Add a documented exclusion/filter contract with shared semantics across schema/types, resolved config, builtin file enumeration, memory watchers, QMD collections/search behavior, reindex state, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: add a Markdown report under `memory/dreaming/deep/`; builtin indexing recursively lists `memory/`, and QMD’s default collection indexes `memory/**/*.md` with no exclusion filter.

## Solution Assessment

Unclear until maintainers choose the public semantics. A config-backed exclusion contract looks maintainable, while a hardcoded dreaming-only skip would leave broader `extraPaths` noise and backend parity questions unresolved.
