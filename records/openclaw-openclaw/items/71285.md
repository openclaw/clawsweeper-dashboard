## Summary

Keep open. Current main still has no memory-search exclusion contract, and source inspection shows both builtin and QMD memory search can still index `memory/dreaming/**`; the related fixes addressed promotion contamination or stale superseded branches, leaving this as the canonical design tracker.

## What This Changes

The issue requests a configurable `memorySearch` exclusion list so `memory/dreaming/**` can be omitted from memory search indexing and watcher scope.

## Best Possible Solution

Add a documented, shared memory-search exclusion contract that flows through schema/types, config resolution, builtin indexing/watch, QMD collection/search behavior, reindex state, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Markdown file under `memory/dreaming/light/` is included by builtin `listMemoryFiles` and by QMD's `memory/**/*.md` collection; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Unclear. A config-backed exclusion is likely the maintainable direction, but maintainers still need to choose the public key name, matcher semantics, defaults, and QMD/builtin parity before a patch should land.
