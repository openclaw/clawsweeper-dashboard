## Summary

Keep open. Current main still lacks a public memory-search exclusion contract, and source inspection shows both builtin and QMD memory search can still index and return `memory/dreaming/**`; newer related work closed stale PR attempts as superseded by this canonical issue rather than solving it.

## What This Changes

The issue requests a configurable memory search exclusion list so `memory/dreaming/**` can be omitted from indexing, search results, and file watching.

## Best Possible Solution

Add a documented exclusion contract shared by schema/types, config resolution, builtin indexing, QMD collection/search/update behavior, watchers, reindex state, docs, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: add a Markdown file under `memory/dreaming/light/`; builtin indexing walks `memory/` recursively, and QMD default collections index `memory/**/*.md` without a dream-report filter.

## Solution Assessment

Unclear until maintainers choose the public config key, matcher semantics, and defaults. A shared config-backed exclusion contract looks maintainable, but a hardcoded dreaming-only skip or divergent glob handling would be weaker.
