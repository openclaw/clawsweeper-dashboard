## Summary

Keep open. Current main still has no configurable `memorySearch.ignorePaths` contract, and the implemented dreaming hardening only addresses promotion/self-ingestion paths, not the built-in memory file enumeration, built-in watcher scope, or QMD default `memory/**/*.md` collection. The closed #75277 is a narrower QMD symptom and #6362 was a closed unmerged prior attempt, so this issue remains the best canonical repair target.

## What This Changes

Request to add configurable `memorySearch.ignorePaths` support so users can exclude `memory/dreaming/**` and other paths from memory search indexing and watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with consistent glob semantics across config schema/types, resolved agent config, built-in file listing, built-in watcher filtering, reindex-state hashing, QMD default collection/update scope where feasible, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: create a markdown file under `memory/dreaming/light/` and use current defaults, where `listMemoryFiles` recursively walks `memory/` and QMD default collections index `memory/**/*.md`.

## Solution Assessment

Yes. A config-based exclusion is the narrow maintainable direction because it solves dreaming report pollution and the broader extra-path dependency-folder problem without hardcoding one OpenClaw-owned directory as policy.
