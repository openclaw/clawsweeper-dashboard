## Summary

Keep open. Current main still lacks a `memorySearch.ignorePaths` or equivalent exclusion contract, and source inspection shows both builtin and QMD memory search can still index Markdown under `memory/dreaming/**`; the related dreaming PRs address promotion hygiene, not ordinary search/index scope.

## What This Changes

Issue requests a configurable `memorySearch.ignorePaths` exclusion mechanism so generated `memory/dreaming/**` Markdown can be omitted from memory search indexing and watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent exclusion contract with shared matching semantics across config schema/types, resolved config, builtin indexing, memory-core watchers, QMD scope, reindex state, generated docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: place a Markdown file under `memory/dreaming/light/`; builtin indexing walks `memory/` recursively, and QMD default collections index `memory/**/*.md`.

## Solution Assessment

Unclear until maintainers choose the public config semantics and defaults. A config-backed ignore contract is the narrow maintainable direction; a hardcoded dreaming-only skip would leave extra-path noise and backend divergence unresolved.
