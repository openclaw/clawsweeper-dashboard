## Summary

Keep open. Current main still has no `memorySearch.ignorePaths` or equivalent exclusion contract, and the remaining bug is not covered by the merged dreaming promotion/self-ingestion fixes: built-in listing, built-in watcher scope, and QMD default collection setup still include ordinary markdown under `memory/`, including `memory/dreaming/**`.

## What This Changes

The issue requests a configurable `memorySearch.ignorePaths` mechanism so `memory/dreaming/**` and other unwanted directories can be excluded from memory search indexing and watcher scope.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with consistent matching across schema/types, resolved agent config, built-in file enumeration, built-in watcher filtering, reindex-state hashing, QMD default collection/update scope where feasible, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a product default exclusion.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with current defaults, a markdown file at `memory/dreaming/light/example.md` is under the recursive built-in `memory` walk and under the QMD default `memory/**/*.md` collection, while the watcher ignore predicate has no `dreaming` or config-pattern exclusion.

## Solution Assessment

Yes. A config-based exclusion is the narrow maintainable direction because it solves dreaming report pollution and the broader dependency-folder/extra-path noise problem without hardcoding one OpenClaw-owned directory as universal policy.
