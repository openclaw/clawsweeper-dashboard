## Summary

Keep open. Current main still lacks a `memorySearch.ignorePaths` or equivalent exclusion contract, and the existing builtin, watcher, and QMD paths still include ordinary Markdown under `memory/dreaming/**`; the merged dreaming promotion fixes address adjacent self-ingestion/promotion leaks, not search-index scope.

## What This Changes

Request to add a configurable memory search ignore-path mechanism so users can exclude `memory/dreaming/**` and other noisy directories from indexing and watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent contract and wire it consistently through schema/types, resolved agent config, builtin file enumeration, builtin watcher filtering, QMD default/custom collection setup where supported, reindex scope hashing, generated config docs, and focused regression tests. Keep dreaming exclusion opt-in unless maintainers explicitly choose a product default.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: place a Markdown file at `memory/dreaming/light/example.md`; current builtin listing recursively walks `memory`, QMD default collections index `memory/**/*.md`, and the memory watcher does not ignore a `dreaming` segment or any configured pattern.

## Solution Assessment

Yes. A config-based ignore-path contract is the narrow maintainable fix because it covers dreaming report pollution and broader dependency-folder noise without hardcoding only one OpenClaw-owned directory as universal policy.
