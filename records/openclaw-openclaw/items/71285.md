## Summary

Keep open. Current main and the latest release still lack a memory-search exclusion contract, and source inspection shows both builtin and QMD memory search can include `memory/dreaming/**`; the ClawHub plugin comment is a useful workaround, but it does not replace the missing core/plugin API decision.

## What This Changes

The issue requests a configurable memory-search exclusion path so generated dreaming artifacts under `memory/dreaming/**` can be omitted from indexing and watcher work.

## Best Possible Solution

Define one documented memory-search exclusion contract across schema/types, config resolution, builtin indexing/watch, QMD collections/update behavior, docs, and reindex cleanup, while preserving the ClawHub plugin as an optional workaround.

## Reproduction Assessment

Yes. Source inspection shows `listMemoryFiles` recursively includes Markdown under `memory/`, and QMD default collections use `memory` with `**/*.md`, so `memory/dreaming/**/*.md` remains in scope unless another filter is added.

## Solution Assessment

Unclear. A configurable exclusion is plausible, but maintainers still need to choose the public key name, matcher semantics, defaults, QMD/builtin parity, and reindex cleanup behavior before implementation.
