## Summary

Keep open. Current main still lacks a configurable `memorySearch.ignorePaths` contract, and the remaining observable problem is not solved by the merged dreaming self-ingestion hardening: builtin file enumeration, builtin watcher scope, and QMD default memory collection setup can still include markdown under `memory/dreaming/**`. The closed #75277 is a narrower QMD symptom and #6362 was a closed unmerged prior implementation attempt, so this issue remains the best canonical repair target.

## What This Changes

Request to add a `memorySearch.ignorePaths` config option that can exclude `memory/dreaming/**` and other directories from memory search indexing and watcher scope.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with consistent, tested semantics across config schema/types, resolved agent config, builtin file listing, builtin watcher filtering, reindex-state scope hashing, and QMD default collection/update behavior where feasible. Keep `memory/dreaming/**` opt-in unless maintainers explicitly decide that generated dreaming reports should be excluded by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available on current main: create a markdown file under `memory/dreaming/light/`, leave memory search defaults enabled, and the builtin `listMemoryFiles` path and QMD default `memory/**/*.md` collection both make that file eligible for ordinary memory search indexing.

## Solution Assessment

Yes. A config-based exclusion is the narrow maintainable direction because it addresses dreaming report pollution and broader dependency-folder/extra-path noise without hardcoding one OpenClaw-owned directory as policy; the fix should avoid the earlier #6362 pitfall of exposing glob wording while only honoring a narrow directory-name subset.

- QMD may need a backend-specific translation or documented limitation if its scanner cannot express the same ignore semantics as the builtin walker and watcher.
