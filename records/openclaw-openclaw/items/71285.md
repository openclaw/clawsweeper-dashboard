## Summary

Keep open. Current main still lacks a `memorySearch.ignorePaths` or equivalent exclusion contract, and the builtin/QMD memory indexing paths can still include Markdown under `memory/dreaming/**`; the related dreaming promotion fixes address durable-memory contamination, not normal search-index scope.

## What This Changes

Add a configurable memory-search ignore path mechanism so generated dreaming reports and other unwanted directories can be excluded from memory_search indexing and file watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` or equivalent exclusion contract, merge it across defaults and per-agent config, and apply it consistently to builtin file enumeration, memory watchers, QMD collection/index setup where supported, reindex state, schema/help/generated docs, and focused tests. Keep dreaming exclusion opt-in unless maintainers explicitly choose a product default.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create `memory/dreaming/light/example.md`; current builtin listing recursively walks `memory`, QMD default collections index `memory/**/*.md`, and neither path has a configured exclusion for the `dreaming` segment.

## Solution Assessment

Yes. A config-backed ignore-path contract is the narrow maintainable fix because it covers dreaming report pollution and broader dependency-folder noise without hardcoding only one OpenClaw-owned directory; the implementation should avoid the earlier #6362 mismatch where watcher and indexer glob behavior could diverge.
