## Summary

Keep open. Current main still lacks a memorySearch ignore/exclude config, and both builtin and QMD memory search can still index or return `memory/dreaming/**`; the related merged dreaming work fixed promotion contamination, not ordinary recall/search scope.

## What This Changes

Requests a `memorySearch` exclusion configuration so users can omit `memory/dreaming/**` from memory search indexing and watcher scope.

## Best Possible Solution

Define a documented exclusion contract shared across schema/types, config resolution, builtin scanning, QMD collection/search behavior, watchers, reindex state, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: add a Markdown file under `memory/dreaming/light/`; builtin indexing recursively walks `memory/`, and QMD default collections index `memory/**/*.md` without a dream-report filter.

## Solution Assessment

Unclear until maintainers choose the public contract. A config-backed exclusion is plausible, but the maintainable path needs agreed key names, matching semantics, defaults, and builtin/QMD parity before implementation.
