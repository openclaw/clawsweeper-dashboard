## Summary

## What This Changes

Request to add a config-based memorySearch exclusion so generated memory/dreaming reports can be omitted from memory search indexing and watch behavior.

## Best Possible Solution

Land the open candidate PR or a successor only after the accepted contract covers schema/types/docs, builtin scan/watch behavior, QMD parity, index identity or reindex cleanup, and focused proof; then close this issue as implemented.

Unclear. A config-based exclusion is plausible, but the best solution needs maintainer agreement on config naming, matcher semantics, backend parity, watcher behavior, and stale-index cleanup before this issue is solved.
