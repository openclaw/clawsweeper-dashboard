## Summary

Keep open. Current main still allows stored per-agent auth-profile credentials to remain runtime candidates after the matching `openclaw.json` declaration is removed, and the embedded Pi path can inject that selected credential without an undeclared-profile guard or warning.

## What This Changes

The issue asks OpenClaw to stop, warn about, or diagnose Pi/runtime use of auth-profile credentials whose `auth.profiles` declaration was removed from `openclaw.json`.

## Best Possible Solution

Adopt declaration-aware credential authority for runtime selection, with a non-destructive diagnostic or quarantine path and a visible warning/doctor check for stored profiles that are no longer declared.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to remove a provider's `auth.profiles` declaration while leaving a usable stored credential, then follow `resolveAuthProfileOrder` through Pi `profileCandidates` into `setRuntimeApiKey`.

## Solution Assessment

Unclear pending maintainer/security policy. A declaration-aware guard or prominent drift warning is safer than silent selection, while unconditional startup pruning needs compatibility review.
