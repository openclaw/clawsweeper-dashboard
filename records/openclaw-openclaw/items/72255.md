## Summary

Keep open. Current main still allows stored per-agent auth-profile credentials to be selected when the matching `openclaw.json` declaration is absent, and the Pi embedded path can inject that resolved credential without an undeclared-profile guard or warning.

## What This Changes

The issue asks OpenClaw to stop or loudly warn when Pi fallback uses per-agent `auth-profiles.json` credentials whose `auth.profiles` declarations were removed from `openclaw.json`.

## Best Possible Solution

Add declaration-aware credential authority for runtime selection, with a non-destructive diagnostic or quarantine path for stored credentials that no longer have `openclaw.json` profile declarations.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to remove the provider’s `auth.profiles` declaration, leave a usable provider credential in `auth-profiles.json`, and follow `resolveAuthProfileOrder` through `resolveApiKeyForProvider` into Pi `setRuntimeApiKey`.

## Solution Assessment

No. The current silent stored-profile fallback is not the best solution for removed declarations; a safer fix should add a declaration-aware guard or loud warning before considering destructive pruning.
