## Summary

Keep open. Current main still allows stored auth-profile credentials to remain selectable when the matching `openclaw.json` declaration is gone, and the Pi runtime can inject that resolved credential without an undeclared-profile guard or high-visibility warning.

## What This Changes

The issue asks OpenClaw to stop or loudly warn when Pi selects stored per-agent auth-profile credentials whose `openclaw.json` `auth.profiles` declaration was removed.

## Best Possible Solution

Runtime auth resolution should be declaration-aware and diagnostics should expose or quarantine undeclared stored credentials without silently using them for live calls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to remove the provider's `auth.profiles` declaration while leaving a valid stored credential, then follow `resolveAuthProfileOrder` into Pi `profileCandidates` and `setRuntimeApiKey`.

## Solution Assessment

No. The current silent stored-profile fallback is not the best solution for removed declarations; a safer fix should add a declaration-aware guard or warning before considering any destructive cleanup.
