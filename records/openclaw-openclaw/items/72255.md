## Summary

Keep open. Current main still allows stored per-agent auth-profile credentials to remain selectable when the matching `openclaw.json` declaration is gone, and the Pi embedded path can inject the selected credential without an undeclared-profile guard or warning.

## What This Changes

The issue asks OpenClaw to stop or warn when Pi/model auth resolves stored per-agent credentials whose `auth.profiles` declaration was removed from `openclaw.json`.

## Best Possible Solution

Make runtime auth selection declaration-aware with a non-destructive diagnostic or quarantine path, preserving intentional legacy drift repair only where explicitly safe.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to remove a provider profile from `cfg.auth.profiles`, leave a usable matching credential in `auth-profiles.json`, and follow `resolveAuthProfileOrder` through Pi `profileCandidates` into `setRuntimeApiKey`.

## Solution Assessment

No. The current silent stored-profile fallback is not the best solution for removed declarations; the safer direction is an owner-approved guard or warning before any destructive pruning behavior.
