## What This Changes

The issue reports that Signal outbound send and probes work on v2026.4.29, but inbound Signal DMs stop triggering agent replies while the SSE receive path repeatedly times out and `openclaw channels status` can time out.

## Best Possible Solution

Review and land or explicitly supersede PR #75427, then verify Signal inbound DMs and `openclaw channels status` on the affected signal-cli 0.14.3 setup. If the status timeout survives the SSE receive fix, track it as a separate gateway/channel-status responsiveness bug with fresh logs.

## Reproduction Assessment

Yes. The report gives a high-confidence live reproduction path on v2026.4.29 with signal-cli 0.14.3: outbound sends succeed, inbound DMs stop replying, the SSE log repeats 10s timeouts, and `channels status` times out while deep gateway status remains healthy.

## Solution Assessment

Mostly yes for the SSE receive failure: disabling the long-lived SSE header deadline is the narrowest maintainable fix for quiet signal-cli streams. It is still unclear whether this fully solves the `channels status` timeout without post-fix live verification.
