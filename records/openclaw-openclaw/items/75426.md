The issue reports unreliable Signal inbound DM handling, repeated SSE receive timeouts, and `openclaw channels status` timeouts despite healthy outbound Signal sending.

## Best Possible Solution

Keep the shipped Signal-local receive-stream fix; any remaining Signal inbound failures should be reported as a fresh current-release issue with new logs because the 10s SSE wedge is already addressed.

## Reproduction Assessment

Yes. The high-confidence source path is a quiet Signal `/api/v1/events` SSE receive stream that previously hit the generic 10s header deadline, matching the reported timeout/reconnect logs.

## Solution Assessment

Yes. The shipped fix is the narrow Signal-plugin solution: disable the short deadline only for the long-lived receive stream while preserving bounded RPC/check/send behavior and explicit positive SSE deadlines.
