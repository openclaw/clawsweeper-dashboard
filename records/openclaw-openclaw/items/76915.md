## Summary

Keep open: current main still suppresses same-target post-message-tool final replies, and the linked fix PR is closed unmerged; the remaining question is delivery policy rather than a safe cleanup close.

## What This Changes

Request that distinct assistant final text after a successful same-turn `message` tool send also be delivered to the originating messaging channel.

## Best Possible Solution

Define one explicit same-turn message-tool policy, then update code, tests, and docs together so duplicate tool echoes stay suppressed while any maintainer-approved distinct final commentary is delivered predictably.

## Reproduction Assessment

Yes, from source inspection: current main records successful message-tool sends, resolves same-route suppression, and drops all final payloads for that route. Existing tests also assert that a different final text is dropped after a same-target send.

## Solution Assessment

Unclear. Delivering every post-tool final reply is not clearly the best fix because current docs and tests encode message-tool-only/private-final behavior; maintainers should choose whether distinct final text should pass through and how duplicate suppression should be bounded.
