## Summary

Keep open: current main still has a source-reproducible path that suppresses same-target final assistant text after a successful message tool send, and the proposed behavior conflicts with current message-tool-only docs/tests enough to need maintainer policy judgment before a repair PR.

## What This Changes

The issue asks for distinct assistant final text produced after a successful message tool dispatch to also be delivered to the same messaging channel.

## Best Possible Solution

Define the same-turn policy explicitly, then update the suppression logic and tests so duplicate message-tool echoes remain suppressed while any approved distinct final commentary behavior is documented for `automatic` versus `message_tool` delivery modes.

## Reproduction Assessment

Yes from source inspection, not from a live Telegram run: current main has tests and code that drop same-target final replies after a successful message tool send, including when the final text differs from the tool text.

## Solution Assessment

Unclear. A narrow code change is feasible, but delivering every post-tool final reply is not clearly the best fix until maintainers decide how this should interact with documented message-tool-only private final replies and duplicate suppression.
