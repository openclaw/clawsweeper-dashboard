## Summary

Keep open: current main still has a source-reproducible same-target suppression path for post-message-tool final text, but that behavior is also tied to documented message-tool/private-final policy, so it needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that assistant final text after a successful same-turn message tool dispatch also be delivered to the originating messaging channel.

## Best Possible Solution

Define one explicit delivery policy: either keep same-turn final text private and require all visible context in the message tool payload, or forward only distinct nonduplicate final text while preserving duplicate/media suppression and updating tests, prompts, and docs.

## Reproduction Assessment

Yes, from source inspection rather than a live Telegram run: same-target successful message-tool sends set suppressReplies, and buildReplyPayloads returns no final payloads even when the final text differs from the tool text.

## Solution Assessment

Unclear. Delivering every post-tool final reply is not clearly the best fix because current docs and prompts intentionally make message-tool-only final text private; the safer path is an explicit maintainer decision on duplicate-vs-additional-text semantics.
