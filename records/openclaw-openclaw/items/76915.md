## Summary

Keep open: current main still has a source-reproducible path that suppresses post-message-tool final text for the same provider/target, but current docs, prompts, and tests also encode message-tool-only/private-final behavior, so changing this needs maintainer policy judgment.

## What This Changes

The issue requests that assistant final text generated after a successful same-turn messaging tool send also be delivered to the originating messaging channel.

## Best Possible Solution

Define one explicit same-turn policy: either keep all channel-visible context inside the message tool payload, or deliver only distinct nonduplicate final text while preserving duplicate suppression and updating docs/tests/prompts together.

## Reproduction Assessment

Yes, from source inspection rather than a live Telegram run. Current main has tests and code paths that suppress a distinct final reply after a successful same-provider/same-target message-tool send.

## Solution Assessment

Unclear. The requested behavior is plausible, but it conflicts with current message-tool-only/private-final docs and tests, so the best fix depends on a maintainer decision about duplicate versus additional final text semantics.
