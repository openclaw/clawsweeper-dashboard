## What This Changes

The PR reorders task upsert/delete persistence before in-memory task-registry mutation and adds a regression test for sqlite-direct read divergence when upsert persistence throws.

## Best Possible Solution

Make delete persistence a single atomic boundary that projects both task and delivery-state deletion together for snapshot fallback and avoids redundant sqlite writes, then add regression coverage for that delete path.
