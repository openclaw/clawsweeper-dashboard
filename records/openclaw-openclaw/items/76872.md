## Summary

Source inspection confirms the reported doctor/config consistency bug remains on current main, but this issue should stay open because an open PR already references it with closing syntax and is the implementation candidate.

## What This Changes

The issue requests that doctor/config validation stop warning about a channel-derived `discord` entry in `plugins.allow` when gateway auto-enable re-adds that entry for an enabled Discord channel.

## Best Possible Solution

Land or close the linked fix PR after review so validation and doctor use the same channel-aware model for known channel ids while preserving warnings for truly unknown plugin ids.

## Reproduction Assessment

Yes from source inspection: in an installed-package layout without a Discord plugin manifest, `plugins.allow: ["discord"]` is not in validation's plugin-id set and still warns, while configured-channel auto-enable can re-add the built-in channel id. I did not run the live macOS gateway restart path.

## Solution Assessment

Yes: the narrowest maintainable solution is to make stale `plugins.allow` validation channel-aware, matching the existing doctor stale-plugin helper, while keeping unknown plugin ids warned or rejected as today.
