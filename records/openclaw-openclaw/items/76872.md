## Summary

Keep open: current main still has the doctor/config consistency bug, and open PR #76879 already references this issue with a targeted fix. The issue should stay open until that PR is reviewed and either landed or closed.

## What This Changes

The issue reports that doctor warns about `discord` in `plugins.allow` while gateway startup re-adds that channel id when `channels.discord.enabled` is set.

## Best Possible Solution

Review and land or close PR #76879 so config validation and gateway auto-enable agree on recognized channel ids.

## Reproduction Assessment

Yes. Source inspection on current main shows validation warns for any `plugins.allow` id missing from the plugin manifest registry, while auto-enable can persist a configured built-in channel id such as `discord` when no plugin manifest claims it.

## Solution Assessment

Yes. The narrow maintainable fix is to make stale allowlist validation channel-aware, matching the existing doctor stale-plugin scan, without weakening validation for genuinely unknown plugin ids.
