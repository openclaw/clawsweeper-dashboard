## Summary

Keep open: current main has the intended Slack config-reload wiring, but the reported Sandpaw/container or panel-write miss has not been reproduced or ruled out, and the related native Slack token-rotation issue tracks a different OAuth-refresh feature.

## What This Changes

Slack token-string rotation should reliably restart the running Slack channel so Bolt does not keep using a stale cached token.

## Best Possible Solution

Keep the generic Slack channel restart invariant, then reproduce or rule out the Sandpaw/container and panel `config.patch` paths before choosing between watcher hardening, a control-plane write follow-up, or operator polling guidance.

## Reproduction Assessment

No. Source inspection shows why a missed restart would keep the old Bolt App token, and a contributor could not reproduce normal direct edits on current main, but the reported Sandpaw container/panel path has not been reproduced or ruled out.

## Solution Assessment

Unclear. Current main already uses the right general reload boundary, so an explicit panel-triggered restart is only a plausible mitigation until the exact missed path is identified.
