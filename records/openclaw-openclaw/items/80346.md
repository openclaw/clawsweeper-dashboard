## What This Changes

Ensure Slack channel instances are reliably restarted and rebuilt after `channels.slack.botToken` or `appToken` changes so Slack Bolt does not keep stale authorization state.

## Best Possible Solution

Keep the generic Slack channel restart invariant, then reproduce or rule out the Sandpaw/container and panel `config.patch` paths before choosing watcher hardening, explicit control-plane restart follow-up, or operator guidance.

## Solution Assessment

Unclear. Rebuilding the Slack channel on `channels.slack` changes is the right owner-boundary solution for stale Bolt state, but the best remaining fix depends on which unproven edge actually skipped the restart.
