## Summary

Keep open: current main has the intended Slack restart wiring and an existing live run could not reproduce the miss, but the reported Sandpaw/container path was not exercised and may still bypass or lose the reload trigger.

## What This Changes

Issue reports that rotating `channels.slack.botToken` can leave the running Slack Bolt app using the old cached authorization until the container is restarted.

## Best Possible Solution

Confirm the Sandpaw/container save path, then make that path deterministically enqueue the existing Slack channel restart or add a bounded watcher fallback for deployments where direct config-file events are unreliable.

## Reproduction Assessment

No. Current main source routes `channels.slack.*` changes to a Slack restart and an existing live main run did not reproduce missed restarts, but the reported Sandpaw/container environment was not exercised.

## Solution Assessment

Unclear. The existing owner boundary is correct, but the fix should target the confirmed failing delivery path rather than adding speculative Slack-specific shims.
