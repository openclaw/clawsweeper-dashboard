Keep open. Current main has the intended Slack reload wiring and a non-container current-main live check did not reproduce missed restarts, but the reported Sandpaw/container and possible panel `config.patch` path has not been reproduced or ruled out.

## What This Changes

The issue requests reliable Slack channel restart when `channels.slack.botToken` or `appToken` changes so the running Bolt App stops using a stale token.

## Best Possible Solution

Add focused live proof for Slack token-only rotation through the managed/container direct-file and panel `config.patch` paths, then fix the generic config reload notification/restart boundary unless that proof shows a Slack-specific explicit restart hook is required.

## Reproduction Assessment

No. A current-main non-container Slack run exercised token-only edits without missed restarts, but the reported Sandpaw/container, panel, and long-session path has not been reproduced or ruled out.

## Solution Assessment

Unclear. The source wiring already restarts Slack for detected `channels.slack.*` diffs, so the maintainable fix should wait for proof that isolates watcher delivery, in-process write notification, or Slack restart execution.
