Keep open. Current main has the expected generic Slack restart wiring and a contributor could not reproduce missed restarts on non-container current main, but the reported Sandpaw/container token-rotation miss is concrete and I found no merged fix or canonical issue that proves that environment-specific path is resolved.

## What This Changes

The issue reports that rotating `channels.slack.botToken` can leave the running Slack Bolt app using a cached old token until the gateway/container is restarted.

## Best Possible Solution

Add focused live or container proof for direct file edits and panel/config.patch token rotation, then fix the generic gateway config reload notification path unless that proof shows a Slack-specific panel restart is required.

## Reproduction Assessment

No, we do not have a high-confidence reproduction for the reported current-main managed/container path. A contributor tested current main outside a container and saw every Slack token edit restart as expected, but that does not cover Sandpaw, Docker overlayfs, long-session watcher state, or panel config.patch writes.

## Solution Assessment

Unclear. The existing generic reload path is the right owner boundary if the diff/notification path is faulty; a Slack-specific restart after panel save should be a fallback only if focused proof shows the generic reloader is not reachable for that write path.
