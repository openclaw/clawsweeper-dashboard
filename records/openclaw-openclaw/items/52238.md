## Summary

Keep this issue open: current main still routes the reported Feishu topic-group shape as direct because group detection depends on `chat_type`, and open PR #52342 is explicitly paired with this issue.

## What This Changes

Feishu topic-group messages that arrive with `chat_type: "p2p"` should be routed as group/thread chats instead of direct sessions.

## Best Possible Solution

Land PR #52342 or a focused replacement that centralizes Feishu chat classification, treats verified topic/open-chat evidence as group routing, adds regression tests, and leaves the distinct outbound-session issue to #58537.

## Reproduction Assessment

Yes. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `thread_id` or `root_id` reaches current main with `isGroup === false` and routes as a direct peer.

## Solution Assessment

No for current main, because the implementation still depends on normalized `chat_type` for the group/direct decision. The maintainable fix is a shared classifier used by message routing and mention-forward handling, with coverage for anomalous topic events and true DMs.
