## Summary

Keep open: current main and v2026.6.8 still classify `p2p` as non-group, while current p2p thread coverage makes the suggested `root_id`/`thread_id` or `oc_`-prefix-only fix unsafe without a verified Feishu discriminator.

## What This Changes

Feishu topic-group messages delivered as `p2p` should route to the Feishu group/topic session instead of the direct or web main session.

## Best Possible Solution

Keep this as the canonical Feishu group/direct classification bug and land a Feishu-local shared classifier that uses verified topic/OpenChat metadata while preserving p2p direct, p2p thread, card-action, reaction, and native `topic_group` behavior.

## Reproduction Assessment

Yes, at source level: a Feishu event logged as `chat_type: "p2p"` reaches current main with `isGroup === false` and follows the direct route. I did not run a live Feishu replay in this read-only review.

## Solution Assessment

No, not with the originally suggested broad `root_id`/`thread_id` or `oc_` prefix-only classifier. The fix should use a verified Feishu topic/OpenChat discriminator and regression tests for true p2p direct/thread cases.
