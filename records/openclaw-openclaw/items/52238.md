## Summary

Keep open: current main and the latest release still classify Feishu inbound group/direct routing from `chat_type` only, so the reported `p2p` OpenChat/topic shape remains source-reproducible. The linked closing PR is still open, conflicting, and under maintainer review, while https://github.com/openclaw/openclaw/issues/58537 tracks the broader outbound/session-consistency follow-up rather than replacing this issue.

## What This Changes

Feishu topic/OpenChat group messages reported as `chat_type: "p2p"` should route as group/topic messages instead of direct web-main-session traffic.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/52342 with a shared Feishu classifier based on verified topic/OpenChat metadata, then keep outbound duplicate-session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes. By source inspection, a Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`; I did not run a live Feishu account.

## Solution Assessment

No. Current main ignores the topic/OpenChat evidence, and the existing PR touches the right call sites but has prior review concerns around prefix-only classification; the safer solution is a verified shared Feishu classifier applied consistently to inbound routing and mention-forward detection.
