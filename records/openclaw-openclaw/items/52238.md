## Summary

Keep this issue open. Current main still classifies Feishu routing from `chat_type` only, so the reported `chat_type: "p2p"` topic-group event remains on the direct-message path; the supplied GitHub context also shows open PR #52342 as an unmerged implementation candidate explicitly paired with this issue.

## What This Changes

The issue asks OpenClaw to route Feishu topic-group messages as group/topic conversations instead of treating anomalous `p2p` events as direct messages.

## Best Possible Solution

Keep the tracker open and review or replace PR #52342 with a focused Feishu plugin patch that centralizes chat classification, treats verified topic/open-chat evidence as group routing, and adds regression tests for anomalous p2p topic events, true p2p DMs, mention-forward behavior, and card-action contexts.

## Reproduction Assessment

Yes. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `thread_id` or `root_id` would set `isGroup` to false on current main, skip `resolveFeishuGroupSession`, and route as a direct peer.

## Solution Assessment

No for current main, because it still depends on normalized `chat_type` for the group/direct decision. The best fix should avoid a bare prefix heuristic unless Feishu API behavior is verified, and should use topic/root evidence or chat metadata with regression tests.
