## Summary

Keep open: current main and v2026.6.6 still route Feishu events logged as `chat_type: "p2p"` through the direct path, while the prior prefix-only fix PR was closed unmerged as unsafe and the adjacent open topic-anchor PR is not a clean replacement for this canonical routing bug.

## What This Changes

The issue asks OpenClaw to route Feishu topic-group messages delivered as `p2p` back to the Feishu topic/group instead of the direct/web session.

## Best Possible Solution

Keep this issue as the canonical Feishu group/direct classification bug, and land a Feishu-owned classifier that uses verified topic/OpenChat metadata while preserving current p2p direct, card-action, and reaction safeguards.

## Reproduction Assessment

Yes. Source inspection plus reporter logs show that a Feishu event delivered as `chat_type: "p2p"` reaches current main with `isGroup === false` and therefore follows the direct route; I did not run a live Feishu replay in this read-only review.

## Solution Assessment

No, not with the previously proposed prefix-only or broad root/thread-only shape. The maintainable fix should use a verified Feishu topic/OpenChat discriminator and regression tests for real p2p `oc_` and direct-thread cases.
