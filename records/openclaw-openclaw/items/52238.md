## Summary

Keep this issue open: current main still classifies the reported Feishu topic-group shape from `chat_type` alone, and an open closing PR is already paired with the issue but has unresolved review concerns.

## What This Changes

Feishu topic-group messages reported as `chat_type: "p2p"` with topic/open-chat metadata should route as group messages instead of direct messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/52342 or a focused replacement that uses a verified Feishu topic/OpenChat signal, keeps native `topic_group` and true DM behavior intact, covers bot routing plus mention-forward, and leaves outbound session consistency to https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `thread_id` or `root_id` reaches current main with `isGroup === false` and is routed as a direct peer.

## Solution Assessment

No for current main, because it still relies only on normalized `chat_type` for group/direct routing. The maintainable fix is a shared Feishu-owned classifier that uses verified topic/open-chat evidence rather than a broad prefix-only rule.
