## Summary

Keep open: current main and v2026.6.10 still treat the reported Feishu `chat_type: "p2p"` event as non-group, and related history shows a broad `oc_` or `root_id`/`thread_id` heuristic can regress true p2p routing.

## What This Changes

The issue requests fixing Feishu topic-group events that arrive as `p2p` so they route and reply as Feishu group/topic messages instead of direct or web sessions.

## Best Possible Solution

Keep this as the canonical Feishu inbound group/direct classification bug and land a Feishu-local shared classifier with regressions for topic-group-as-p2p, true p2p DM threads, mention forwarding, and outbound mirror/session behavior.

## Reproduction Assessment

Yes at source level: a Feishu event with the reported `chat_type: "p2p"` reaches current main with `isGroup === false` and routes as direct. I did not run a live Feishu/Lark replay in this read-only review.

## Solution Assessment

No for the broad proposed `root_id`/`thread_id` or `oc_`-prefix-only heuristic, because current p2p tests and prior Feishu history show those signals are ambiguous. The best fix is a shared Feishu classifier backed by verified topic/OpenChat metadata and regression coverage for true p2p paths.
