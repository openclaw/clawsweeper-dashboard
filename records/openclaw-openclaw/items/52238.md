## Summary

Keep open: current main and v2026.5.20 still route Feishu inbound group/direct behavior from `chat_type` only, so the reported `p2p` topic/OpenChat shape remains source-reproducible. The linked closing PR at https://github.com/openclaw/openclaw/pull/52342 is still open and conflicting, while https://github.com/openclaw/openclaw/issues/58537 tracks the broader outbound/session consistency follow-up rather than replacing this issue.

## What This Changes

Feishu topic-group messages reported as `chat_type: "p2p"` should be routed as group/topic traffic instead of direct/web-main-session traffic.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/52342 with a verified shared Feishu classifier for inbound routing and mention-forward detection, while keeping outbound session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes. By source inspection, a Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`; I did not run a live Feishu account.

## Solution Assessment

Unclear for the exact proposed discriminator, but the target behavior is valid. The maintainable path is a verified shared Feishu classifier applied consistently to inbound routing and mention-forward detection, not closing this before the linked implementation lands.
