## Summary

Keep this issue open: current main and the latest release still derive Feishu group/direct routing from `chat_type` alone, so the reported `p2p` topic-group shape remains on the direct path. The paired closing PR is still open, conflicting, and has unresolved review concerns around the discriminator.

## What This Changes

The issue requests that Feishu topic/OpenChat group messages reported as `chat_type: "p2p"` route as group messages instead of direct messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/52342 or a focused replacement that uses a verified Feishu topic/OpenChat signal, preserves native `topic_group` and true DM/card-action behavior, and leaves outbound session consistency to https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` reaches current main with `isGroup === false` and routes as a direct peer.

## Solution Assessment

Unclear for the exact discriminator, but current main is not the best solution because it ignores topic/open-chat evidence. The maintainable fix is a shared Feishu classifier backed by verified event metadata rather than chat type alone or a bare `oc_` prefix rule.
