## Summary

Keep open: source inspection of current main and v2026.5.18 shows the Feishu inbound handler still classifies group/direct traffic from `chat_type` only, so the reported `p2p` topic/OpenChat shape still reaches direct routing. The linked closing pull request remains open, conflicting, and has unresolved classifier concerns.

## What This Changes

The issue asks for Feishu Topic Group/OpenChat webhook messages reported as `p2p` to be classified and routed as group messages instead of direct messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/52342 or a focused replacement that uses verified Feishu topic/OpenChat metadata in a shared inbound classifier, while keeping outbound duplicate-session consistency tracked at https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu message with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` reaches current main with `isGroup === false` and routes as a direct peer.

## Solution Assessment

No, current main is not the best solution because it ignores topic/OpenChat evidence. The maintainable fix is a shared Feishu classifier backed by verified event or chat metadata, not a bare prefix rule that could regress protected DM paths.
