## Summary

Keep open: current main and v2026.6.10 still classify reported Feishu `oc_...` events with `chat_type: "p2p"` as non-group, while existing p2p thread coverage makes the proposed `root_id`/`thread_id` or `oc_`-prefix-only fix unsafe without a verified Feishu discriminator.

## What This Changes

The issue asks OpenClaw to route Feishu Topic Group/OpenChat messages reported as `p2p` back to the Feishu group/topic instead of a direct or web session.

## Best Possible Solution

Keep this as the canonical Feishu inbound group/direct classification bug and land a Feishu-local shared classifier that uses verified topic/OpenChat metadata while preserving true p2p direct/thread behavior.

## Reproduction Assessment

Yes, at source level: a Feishu event logged as `chat_type: "p2p"` reaches current main with `isGroup === false` and follows the direct route. I did not run a live Feishu/Lark replay in this read-only review.

## Solution Assessment

No, not with the originally suggested broad `root_id`/`thread_id` or `oc_` prefix-only classifier. The safest fix needs a verified Feishu topic/OpenChat discriminator plus regressions proving true p2p direct and p2p thread cases remain direct.
