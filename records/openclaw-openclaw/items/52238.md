## Summary

Keep this issue open: current main still classifies the reported Feishu topic-group shape from `chat_type` only, and an open unmerged PR is explicitly paired with this issue.

## What This Changes

Feishu topic-group messages whose webhook reports `chat_type: "p2p"` should route as group/topic messages instead of direct messages.

## Best Possible Solution

Keep this tracker open and land PR #52342 or a focused replacement that centralizes Feishu chat classification, treats verified topic/open-chat evidence as group routing, adds regression tests, and leaves the distinct outbound-session issue to #58537.

## Reproduction Assessment

Yes. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `thread_id` or `root_id` reaches current main with `isGroup === false`, skips group-session derivation, and routes as a direct peer.

## Solution Assessment

No for current main, because the implementation still depends on normalized `chat_type` for the group/direct decision. A maintainable fix should share one classifier across message routing and mention-forward handling, with tests for anomalous topic events and true DMs.
