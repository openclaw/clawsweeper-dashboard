## What This Changes

The issue reports that Feishu topic-group/OpenChat messages with `chat_type: "p2p"` and `oc_...` chat IDs are routed as direct messages instead of group/topic conversations.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/52342 with a verified shared Feishu classifier for inbound routing and mention-forward detection, while keeping outbound session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Solution Assessment

Unclear for the exact discriminator, but the target behavior is valid. The maintainable path is a verified shared Feishu classifier applied consistently to inbound routing and mention-forward detection, not closing this before the linked implementation lands.
