## Summary

Keep open. Current main still routes failed overflow and compaction recovery through the session-reset path that writes a generated session id into the active store entry, and the linked fixing PR remains open and unmerged.

## What This Changes

The issue asks OpenClaw to preserve the active WebChat `agent:main:main` session mapping across context-overflow or compaction-failure handling instead of silently remapping it to a fresh session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70479 or an equivalent refreshed patch that preserves the active logical session mapping unless rotation is explicit and auditable.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: drive an embedded context-overflow payload or thrown compaction-failure error while reset recovery succeeds, and current main reaches the reset helper that rewrites the active session entry to a generated session id.

## Solution Assessment

Yes for keeping this open. The narrow maintainable fix is to stop failed overflow or compaction recovery from using the session-reset path, while preserving current overflow guidance and adding regression coverage.
