## Summary

Keep open. Current main still routes context-overflow and compaction-failure recovery through the session reset path that rewrites the active session mapping, and the open closing implementation PR has not merged.

## What This Changes

The issue asks OpenClaw to preserve the active WebChat `agent:main:main` session mapping through context overflow and compaction failure instead of silently rotating to a fresh session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70479 or an equivalent patch that preserves the active logical session mapping unless a reset is explicit and auditable.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: drive the embedded runner through a context-overflow payload or thrown compaction-failure error while reset recovery succeeds, and current main reaches the reset path that writes a generated session id into the active store row. I did not run a live WebChat overflow because this review is read-only.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable solution is to preserve the existing session mapping through overflow or compaction recovery failure and make any true rotation explicit, which is the direction of the open closing PR.
