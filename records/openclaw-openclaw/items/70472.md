## Summary

Keep open. Current main still routes context-overflow payloads and thrown compaction failures into the session-reset path, which rewrites the active session row to a new session id; the open closing PR #70479 is still unmerged, so this issue remains valid.

## What This Changes

Preserve the active `agent:main:main` WebChat session mapping across context-overflow and compaction-checkpoint failures unless the user explicitly resets the session.

## Best Possible Solution

Preserve the existing logical session mapping when overflow or compaction recovery fails, and only rotate to a new transcript through an explicit, durable, auditable reset path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to drive the embedded runner through a context-overflow error payload or thrown compaction-failure error while reset recovery succeeds; current main then calls `resetReplyRunSession`, whose test asserts the active store row rotates to a generated session id.

## Solution Assessment

Yes for keeping this issue open. The current main reset path is not the fix; the narrow maintainable path is to review and land #70479 or an equivalent patch that preserves the active mapping unless rotation is explicit.
