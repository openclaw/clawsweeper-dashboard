## Summary

Keep open. Current main still routes embedded context-overflow payloads and thrown compaction failures through the session reset path, and that path rewrites the active session store row to a new session id; #70479 is still the open, unmerged closing PR for this issue.

## What This Changes

The issue reports that context overflow or compaction failure can silently remap the active WebChat `agent:main:main` session to a new transcript instead of preserving the existing logical session.

## Best Possible Solution

Preserve the existing logical session mapping during overflow and failed compaction recovery, and only rotate to a new transcript through an explicit, durable, auditable reset path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to make the embedded runner return a context-overflow meta error or throw a compaction failure while `resetSessionAfterCompactionFailure` succeeds; current main then calls `resetReplyRunSession`, whose unit test asserts the active store row rotates to a generated session id.

## Solution Assessment

Yes for keeping this issue open. The current reset path is not the best fix because it changes the active session id; #70479 or an equivalent narrow fix should preserve the active mapping unless rotation is explicit.
