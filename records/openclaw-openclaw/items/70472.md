## Summary

Keep open. Current main still routes overflow and compaction-failure recovery through the session-reset helper that can remap the active session key, and the open closing implementation at https://github.com/openclaw/openclaw/pull/70479 has not merged.

## What This Changes

The issue asks OpenClaw to preserve the active WebChat `agent:main:main` session mapping when context overflow or compaction checkpoint handling fails.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70479 or an equivalent refreshed patch that preserves the active logical session mapping after overflow or compaction failure with focused regression coverage.

## Reproduction Assessment

Yes, source-level. A context-overflow payload or thrown compaction-failure error with reset recovery available still reaches `resetSessionAfterCompactionFailure`, which in turn rewrites the active session mapping to a generated session id.

## Solution Assessment

Yes for keeping this open. The narrow maintainable path is to stop failed overflow or compaction recovery from silently rotating the active session, and the existing closing PR should be reviewed or replaced rather than closing the issue early.
