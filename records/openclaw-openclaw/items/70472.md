## Summary

Keep open. Current main still sends failed overflow and compaction recovery through a reset path that rewrites the active session mapping, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports that context overflow or compaction failure can orphan the active WebChat `agent:main:main` mapping and silently bind it to a new session id.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70479 or an equivalent refreshed patch that preserves the active logical session mapping unless the user explicitly resets it, while keeping current overflow guidance and adding regression coverage.

## Reproduction Assessment

Yes, at source level. Current main reaches `resetSessionAfterCompactionFailure` from embedded context-overflow payloads and thrown compaction failures, and that helper rewrites the active session mapping to a generated session id.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix is to stop failed overflow or compaction recovery from silently rotating the active session, and the open linked PR is the current implementation path to review or refresh.
