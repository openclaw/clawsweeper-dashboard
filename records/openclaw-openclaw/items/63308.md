## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` selectors and still returns the reported mixed-selector error before resolution; the latest release has the same guard, while the relevant runtime and schema/docs PRs remain open and unmerged.

## What This Changes

The issue asks for `sessions_send` to avoid invalid mixed `sessionKey` plus `label` calls through clearer schema/tool guidance and/or by treating `sessionKey` as authoritative.

## Best Possible Solution

Land one canonical fix, likely https://github.com/openclaw/openclaw/pull/74009 or an equivalent replacement, that makes `sessionKey`/session-id input authoritative, preserves visibility and A2A safeguards, updates model-facing guidance, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `createSessionsSendTool().execute` with non-empty `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before gateway resolution.

## Solution Assessment

Yes. Executor-side normalization plus schema/tool/docs guidance is the narrow maintainable direction; docs-only guidance would not protect callers that already submit redundant selectors.
