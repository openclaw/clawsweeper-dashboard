## Summary

Keep open. Current main and the latest release still expose independent `sessions_send` selectors and still reject mixed `sessionKey` plus `label` payloads before resolution; open PRs cover parts of the fix but none has merged.

## What This Changes

Make `sessions_send` guide or normalize callers so `sessionKey` and `label` are not submitted as conflicting independent selectors.

## Best Possible Solution

Land one canonical fix that treats explicit `sessionKey` or session-id input as authoritative, keeps visibility/A2A/thread safeguards intact, updates schema and tool guidance, and adds focused regression tests for `sessionKey`-only, `label`-only, and mixed-selector calls.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `createSessionsSendTool().execute` with non-empty `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before gateway resolution.

## Solution Assessment

Yes. Executor-side normalization plus schema/tool/docs guidance is the narrow maintainable fix; docs-only guidance would not protect callers that already submit redundant selectors.
