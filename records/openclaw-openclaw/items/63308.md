## Summary

Keep open. Current `main` and the latest release still reject `sessions_send` payloads that include both `sessionKey` and `label`, while related open PRs already cover the runtime and schema/docs repair paths.

## What This Changes

`sessions_send` should guide or normalize mixed `sessionKey` and `label` calls, preferably treating `sessionKey` as the authoritative target when present.

## Best Possible Solution

Land one canonical fix that makes `sessionKey` authoritative when present, ignores redundant `label` and `agentId` on that path, updates model-facing guidance, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a call to `createSessionsSendTool().execute` with both `sessionKey` and a non-empty `label` reaches the exact guard and returns the reported error before gateway resolution.

## Solution Assessment

Yes. Runtime `sessionKey` precedence plus schema/tool/docs guidance is the narrow maintainable fix; schema text alone is weaker because it leaves injected or over-specified payloads failing at runtime.
