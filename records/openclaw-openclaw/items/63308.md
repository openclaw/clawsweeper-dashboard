## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still returns the reported mixed-selector error before resolving or sending, while related open PRs already cover the runtime and schema/docs directions.

## What This Changes

The issue requests clearer `sessions_send` selector guidance and/or executor normalization so calls with both `sessionKey` and `label` do not fail unnecessarily.

## Best Possible Solution

Land one canonical fix that makes `sessionKey` authoritative when present, ignores redundant `label` and `agentId` on that path, documents both valid addressing modes, and adds focused regression coverage.

## Reproduction Assessment

Yes. A focused call to `createSessionsSendTool().execute` with both `sessionKey` and `label` reaches the exact guard at `src/agents/tools/sessions-send-tool.ts:134` and returns the reported error before gateway resolution.

## Solution Assessment

Yes. The narrow maintainable direction is runtime `sessionKey` precedence plus schema/tool/docs guidance; schema text alone is not enough for the real-world injected-label path.
