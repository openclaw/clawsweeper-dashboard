## Summary

Keep open: the underlying gateway cleanup gap is source-visible, but this patch currently turns every owner WebSocket close into immediate cancellation, which conflicts with existing reconnect and late-join behavior.

## What This Changes

The PR imports the existing chat abort helper into the gateway WebSocket close handler, aborts matching `chatAbortControllers` entries on owner connection close, logs that path, and adds a focused unit test.

## Best Possible Solution

Define disconnect ownership semantics, then cancel abandoned runs through a grace/rebind-aware cleanup path that preserves reconnect and page-refresh late joining.

## Reproduction Assessment

Yes. Source inspection shows current main registers owner connection ids but the close handler does not consume them, and the PR body supplies before/after production-bundle log proof.

## Solution Assessment

No, not as written. The existing abort helper is the right primitive, but firing it on every socket close should be replaced with a grace/rebind path or explicit cancellation signal.
