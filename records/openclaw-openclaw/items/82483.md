## What This Changes

Adds a WebSocket close-handler loop that aborts owned chat abort controllers via the existing abort helper, plus a regression test for matching versus other connection owners.

## Best Possible Solution

Land this PR or an equivalent narrow gateway lifecycle patch after green merge validation, then close the linked bug report as fixed.

## Reproduction Assessment

Yes. Source inspection shows current main registers `ownerConnId` for chat and agent runs while the WebSocket close handler does not inspect `chatAbortControllers`, and the linked report provides a concrete five-trial wire-level reproduction.

## Solution Assessment

Yes. Calling the existing `abortChatRunById` helper only for entries owned by the closing connection is the narrowest maintainable fix because it reuses existing cancellation, cleanup, broadcast, and lifecycle semantics.
