## Summary

Keep open: this PR targets a real memory-wiki session-visibility bug, but the latest diff still drops archived transcript owner metadata and needs real behavior proof before merge.

## What This Changes

The PR threads session and sandbox context into memory-wiki wiki_search/wiki_get, filters shared-memory session hits, guards direct shared-memory reads, adds tests, and records a changelog security fix.

## Best Possible Solution

Keep the fix in memory-wiki, but reuse the owner-aware SDK transcript identity flow for both search filtering and direct reads, add archived transcript regression coverage, then merge after maintainer security review and real behavior proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because wiki_search maps shared-memory session hits directly and wiki_get reads shared-memory candidates without a guard; the remaining PR regression follows from a sessions/<agent>/<stem>.jsonl.deleted.* path with an empty live store.

## Solution Assessment

No, not as written. The plugin-local fix is the right boundary, but it should use extractTranscriptIdentityFromSessionsMemoryHit and pass archivedOwnerAgentId for archived paths before merge.
