## Summary

Keep open for maintainer review: this PR has the protected `maintainer` label, so it should not be cleanup-closed by this workflow. The patch is directionally consistent with current main because memory-wiki currently forwards a session key but does not enforce the session visibility guard already used by `memory_search`; I found no discrete blocking review finding in the supplied diff.

## What This Changes

The PR propagates trusted sandbox/session context into memory-wiki `wiki_search` and `wiki_get`, filters session-sourced shared-memory hits by session visibility, skips disallowed direct session-memory reads, and adds regression coverage.

## Best Possible Solution

Have a maintainer review and land this PR if CI stays green, keeping the policy enforcement inside the memory-wiki plugin while relying on the existing plugin SDK visibility helpers rather than adding a new core special case.

Yes. ## Solution Assessment
