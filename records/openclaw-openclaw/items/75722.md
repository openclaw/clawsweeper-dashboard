## Summary

This PR addresses a valid memory-wiki session-visibility gap, but it should stay open because the current diff drops archived transcript owner metadata instead of matching the existing memory_search visibility contract.

## What This Changes

The PR threads sandbox/session context into memory-wiki wiki_search and wiki_get, filters shared-memory session hits, guards direct shared-memory reads, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the fix in the memory-wiki plugin, but update the checker to reuse the owner-aware transcript identity contract from memory_search and rebase without dropping current main's balanced corpus merge behavior.

## Reproduction Assessment

Yes. Current main is source-reproducible: wiki_search maps shared-memory session hits directly, and wiki_get falls back to manager.readFile without a session-visibility guard.

## Solution Assessment

No, not as written. The direction is the right narrow owner-side fix, but the implementation should preserve archived transcript owner metadata via extractTranscriptIdentityFromSessionsMemoryHit and keep current main's mergeWikiSearchCorpusResults behavior after rebase.
