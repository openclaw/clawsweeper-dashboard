## What This Changes

The PR adds a gateway request-context guard that rejects reserved `x-openclaw-session-key` namespaces in the OpenAI-compatible and OpenResponses HTTP handlers and adds resolver/endpoint tests.

## Best Possible Solution

Merge the shared reserved-session guard after docs and redacted real HTTP proof, while tracking session-history and tools-invoke ingress separately unless maintainers choose an all-ingress policy.

## Reproduction Assessment

Yes, source-reproducible: current main returns explicit `x-openclaw-session-key` headers unchanged and both affected HTTP handlers use that shared request context before dispatch. I did not run a live gateway request because this review is read-only.

## Solution Assessment

Yes for the runtime code shape: a shared resolver guard with a dedicated error type is the narrow maintainable fix. The remaining blockers are docs, real behavior proof, and maintainer acceptance of the compatibility break.
