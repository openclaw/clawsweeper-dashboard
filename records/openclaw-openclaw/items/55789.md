## Summary

Keep open. The runtime guard is a narrow, appropriate hardening for the OpenAI-compatible and OpenResponses HTTP paths, but the PR still needs the documented header contract and changelog updated before merge.

## What This Changes

The branch rejects reserved `x-openclaw-session-key` overrides in shared gateway HTTP request context, maps that specific error to HTTP 400 for OpenAI-compatible and OpenResponses endpoints, and adds focused regression tests.

## Best Possible Solution

Land the narrow runtime guard after adding the HTTP API docs note and a single-line Unreleased changelog entry, while keeping #55928 and broader `/tools/invoke` policy work separate.

## Reproduction Assessment

Yes, from source inspection. Current main returns an explicit `x-openclaw-session-key` unchanged and both affected HTTP handlers use the resolved context, so an internal namespace header can reach the agent layer.

## Solution Assessment

Yes for the runtime fix, with docs and changelog follow-through still needed. Blocking in shared request-context resolution and mapping only `GatewaySessionKeyOverrideError` to 400 is the narrowest maintainable fix for these two HTTP surfaces.
