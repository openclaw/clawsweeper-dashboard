## Summary

Keep open. The runtime patch looks correct, the prior broad-error-mapping concern has been addressed, and PR-head checks have no failures, but the PR still needs the required docs and changelog follow-through before merge.

## What This Changes

The PR rejects OpenAI-compatible and OpenResponses `x-openclaw-session-key` values targeting `subagent:`, `acp:`, or `cron:` internal namespaces, maps that specific error to HTTP 400, and adds request-context plus endpoint regression tests.

## Best Possible Solution

Land the narrow runtime guard after adding the missing HTTP API docs note and a single-line Unreleased changelog entry, while keeping #55928 and any `/tools/invoke` policy work separate.

## Reproduction Assessment

Yes. Source inspection on current main shows an explicit `x-openclaw-session-key: agent:main:subagent:worker` is returned unchanged by shared request-context resolution and then used by both HTTP endpoint handlers.

## Solution Assessment

Yes for the runtime fix, with docs and changelog follow-through. Blocking in shared request-context resolution and mapping only `GatewaySessionKeyOverrideError` to 400 is the narrowest maintainable fix for these two HTTP surfaces.
