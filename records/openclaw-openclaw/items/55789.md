## What This Changes

The PR rejects reserved `x-openclaw-session-key` values for internal `subagent:`, `acp:`, and `cron:` namespaces on the OpenAI-compatible and OpenResponses HTTP endpoints.

## Best Possible Solution

Merge the shared reserved-session guard after public docs and redacted real HTTP proof, while tracking session-history and tools-invoke ingress separately unless maintainers choose one all-ingress policy.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `x-openclaw-session-key` values are returned unchanged and both affected HTTP handlers pass that context into agent dispatch.

## Solution Assessment

Yes for the runtime code shape: a shared resolver guard with a dedicated error type is narrow and maintainable. The remaining blockers are docs, real behavior proof, and maintainer acceptance of the compatibility boundary.
