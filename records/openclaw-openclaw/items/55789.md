## Summary

Keep open: this is still useful gateway security hardening, and the author addressed the earlier narrow-error and reserved-prefix test feedback, but the external PR still lacks contributor-supplied real HTTP proof and the public gateway docs still need the new reserved-namespace exception.

## What This Changes

The branch adds a shared gateway request-context guard that rejects reserved internal `x-openclaw-session-key` prefixes on the OpenAI-compatible and OpenResponses HTTP endpoints and returns `400 invalid_request_error`.

## Best Possible Solution

Land the shared request-context guard after adding the docs exception and contributor-supplied real HTTP proof, while keeping the companion session-history hardening in https://github.com/openclaw/openclaw/pull/55928 separate.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `x-openclaw-session-key` values are returned unchanged and both OpenAI-compatible HTTP handlers consume the shared context before dispatch; I did not run a live HTTP repro in this read-only review.

## Solution Assessment

Yes for the runtime approach. A shared resolver guard plus a dedicated `GatewaySessionKeyOverrideError` is the narrow maintainable fix; the remaining needs are docs and real behavior proof, not a different code design.
