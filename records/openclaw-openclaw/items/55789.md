## Summary

Keep open: the PR is a plausible narrow gateway security hardening, and current main still forwards these internal session-key overrides. It is not merge-ready because the external PR still has only test/CI-style proof and the public HTTP docs still describe unrestricted explicit session routing.

## What This Changes

The PR adds a shared gateway request-context guard that rejects reserved `x-openclaw-session-key` header overrides in the OpenAI-compatible and OpenResponses HTTP handlers, plus request-context and endpoint tests.

## Best Possible Solution

Land the shared request-context guard after adding the docs exception and contributor-supplied real HTTP proof; track broader all-HTTP session ingress policy separately if maintainers want it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `x-openclaw-session-key: agent:main:subagent:worker` to either OpenAI-compatible HTTP endpoint on current main and the shared resolver returns it unchanged before dispatch.

## Solution Assessment

Yes for the runtime approach. A shared resolver guard plus a dedicated `GatewaySessionKeyOverrideError` is the narrow maintainable fix; the PR still needs a docs note and real behavior proof before merge.

## Review Findings
