## Best Possible Solution

Merge the shared guard after documenting the reserved namespace exception and adding redacted real gateway proof, while keeping companion ingress-path guards tracked separately.

## Reproduction Assessment

Yes, source-reproducible: current main returns explicit `x-openclaw-session-key` headers unchanged and both affected HTTP handlers use that shared context before dispatch. I did not run a live gateway request because this review is read-only.

## Solution Assessment

Yes for the runtime code shape: a shared resolver guard with a dedicated error type is the narrow maintainable fix. The remaining blockers are docs, real behavior proof, and maintainer acceptance of the compatibility break.
