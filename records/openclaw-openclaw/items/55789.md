## What This Changes

The PR adds a shared gateway request-context guard that rejects reserved `x-openclaw-session-key` namespaces on OpenAI-compatible and OpenResponses HTTP requests, returning `400 invalid_request_error` with focused tests.

## Best Possible Solution

Land the shared request-context guard after documenting the reserved namespace exception and adding redacted real HTTP proof, while keeping the session-history hardening in https://github.com/openclaw/openclaw/pull/55928 distinct.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `x-openclaw-session-key` values are returned unchanged and both OpenAI-compatible HTTP handlers use that context before dispatch; I did not run a live HTTP repro in this read-only review.

## Solution Assessment

Yes for the runtime approach. A shared resolver guard plus a dedicated error type is the narrow maintainable fix; the remaining needs are docs, contributor real behavior proof, and maintainer acceptance of the intentional compatibility break.
