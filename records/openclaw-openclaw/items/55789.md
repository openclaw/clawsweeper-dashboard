## Summary

Keep open: current main still accepts reserved internal session keys from the HTTP header, and the PR’s code direction is correct, but this external PR still lacks real after-fix HTTP proof and leaves the public HTTP docs stale.

## What This Changes

The branch rejects reserved `x-openclaw-session-key` namespaces in the OpenAI-compatible and OpenResponses HTTP endpoints with a 400 response and adds focused regression coverage.

## Best Possible Solution

Land the shared request-context guard after adding a short docs note for the reserved namespace 400 behavior and redacted real HTTP proof from the contributor.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `x-openclaw-session-key: agent:main:subagent:worker` to either OpenAI-compatible HTTP endpoint on current main and the shared resolver returns it unchanged before dispatch.

## Solution Assessment

Yes for the runtime approach. A shared resolver guard plus a dedicated `GatewaySessionKeyOverrideError` is the narrow maintainable fix; the remaining gaps are docs and real behavior proof.
