## What This Changes

The PR rejects explicit `x-openclaw-session-key` header values that target `subagent:`, `acp:`, or `cron:` namespaces in the OpenAI-compatible and OpenResponses HTTP endpoints, maps the dedicated error to HTTP 400, and adds resolver/endpoint tests.

## Best Possible Solution

Merge the shared reserved-session guard after docs and redacted real HTTP proof, while keeping session-history and tools-invoke ingress policy as separately tracked maintainer decisions unless maintainers want one all-ingress change.

## Reproduction Assessment

Yes, source-reproducible: current main returns explicit `x-openclaw-session-key` headers unchanged and both affected HTTP handlers use that shared request context before dispatch. I did not run a live gateway request because this review is read-only.

## Solution Assessment

Yes for the runtime code shape: a shared resolver guard with a dedicated error type is the narrow maintainable fix. The remaining blockers are docs, real behavior proof, and maintainer acceptance of the compatibility break.
