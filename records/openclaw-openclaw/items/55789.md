## Summary

Keep open: current main still accepts explicit internal session-key overrides, and this PR is the right narrow runtime hardening, but the public HTTP docs and required changelog entry are still missing before merge.

## What This Changes

The PR adds a reserved-prefix guard to OpenAI-compatible and OpenResponses HTTP session-key resolution, maps only that guard to a 400 invalid_request_error, and adds regression coverage.

## Best Possible Solution

Land the runtime guard after adding the HTTP API docs note and a single-line Unreleased Fixes changelog entry, while keeping session-history and /tools/invoke policy work separate.

## Reproduction Assessment

Yes. Source inspection on current main shows x-openclaw-session-key is returned unchanged by shared request-context resolution and then consumed by both HTTP endpoint handlers.

## Solution Assessment

Yes for the runtime approach. Blocking the reserved prefixes in shared request-context resolution and mapping only GatewaySessionKeyOverrideError to 400 is the narrow maintainable fix; the remaining work is docs and changelog follow-through.
