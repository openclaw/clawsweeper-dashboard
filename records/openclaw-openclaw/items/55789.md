## Summary

Keep this PR open: current main still forwards explicit reserved session keys unchanged, and the PR is a narrow security hardening, but it still needs contributor-supplied real HTTP proof and a docs update before merge.

## What This Changes

The PR adds a shared gateway guard that rejects `subagent:`, `acp:`, and `cron:` `x-openclaw-session-key` overrides on OpenAI-compatible HTTP endpoints and maps that dedicated error to `400 invalid_request_error`.

## Best Possible Solution

Land the shared request-context guard after documenting the reserved namespace exception and adding redacted real HTTP proof, while keeping the separate session-history hardening in https://github.com/openclaw/openclaw/pull/55928 distinct.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `x-openclaw-session-key` values are returned unchanged and both OpenAI-compatible HTTP handlers consume the shared context before dispatch; I did not run a live HTTP repro in this read-only review.

## Solution Assessment

Yes for the runtime approach. A shared resolver guard plus a dedicated error type is the narrow maintainable fix; the remaining needs are docs, contributor real behavior proof, and maintainer acceptance of the intentional compatibility break.
