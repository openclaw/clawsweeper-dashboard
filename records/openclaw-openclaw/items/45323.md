## Summary

Current main does not provide agent mention autocomplete, so the request remains useful. However, the proposed architecture duplicates the existing agent-list contract and couples a bounded suggestion feature to a much larger rich-text editor migration; maintainer product direction is still needed, especially given the related generic autocomplete request.

## What This Changes

Add agent mention autocomplete and styled mention rendering to the Control UI chat composer and history.

## Best Possible Solution

Extend the existing chat composer combobox with an `@` trigger backed by `agents.list`, preserve plain-text serialization, and consider editable chips and history decoration only as a separately approved follow-up.

## Reproduction Assessment

Not applicable: this requests a new composer capability rather than reporting broken defined behavior; current main has slash-command suggestions but no agent mention autocomplete.

## Solution Assessment

No: the desired UX is reasonable, but a new directory RPC and immediate contenteditable migration are not the narrowest maintainable solution because current agent metadata and composer suggestion infrastructure can support a staged implementation.
