## Summary

Keep this open. Current main still has slash-command autocomplete and an `agents.list` RPC, but it does not implement `@` mention autocomplete, composer chips, known-agent validation, or chat-history mention chips.

## What This Changes

Add Slack-style `@agent` autocomplete and mention-chip rendering to the Control UI chat composer using known agent metadata.

## Best Possible Solution

Implement this as a Control UI product task that reuses `agents.list` metadata where sufficient, adds only additive Gateway metadata if needed, serializes selected mentions as plain `@agent` text, and renders chips only for known agents.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug, but the current-main absence check is high confidence: the composer is slash-menu and textarea based, and targeted searches found no `@` mention autocomplete, chip, or `agents.directory` implementation.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact proposed `contenteditable` plus new RPC design; the maintainable path should decide the composer model first and reuse `agents.list` unless extra metadata is actually required.
