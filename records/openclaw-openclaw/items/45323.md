## Summary

Keep open: current main and v2026.6.10 still only implement slash-command suggestions in a textarea composer, and the related custom-term autocomplete issue at https://github.com/openclaw/openclaw/issues/57142 overlaps on composer infrastructure but does not supersede this agent-mention request.

## What This Changes

The issue requests Slack-style `@agent-name` autocomplete, validated mention chips, and mention rendering in the Control UI chat composer.

## Best Possible Solution

Define one shared, accessible Control UI composer suggestion model and reuse or deliberately extend existing agent metadata APIs before implementing agent mention chips.

## Reproduction Assessment

Yes, as a source-level absence check. Typing `@` cannot open suggestions because the only suggestion updater is slash-only, the composer is still a textarea, and history rendering has no validated mention-chip pass.

## Solution Assessment

Unclear. The UX direction is plausible and aligned, but the proposed dedicated `agents.directory` endpoint and rich composer architecture need maintainer scoping against existing `agents.list` and `agent.identity.get` APIs.
