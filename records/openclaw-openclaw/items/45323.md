## Summary

Keep open: current main and v2026.6.8 still only expose slash-command suggestions in a textarea composer, with no @agent trigger, mention chips, or agent-validated mention rendering. The related custom-term autocomplete issue at https://github.com/openclaw/openclaw/issues/57142 overlaps on shared composer direction but does not supersede this agent-mention request.

## What This Changes

Request to add Slack-style @agent autocomplete, mention chips, agent-validated history rendering, and supporting agent directory metadata for the Control UI chat composer.

## Best Possible Solution

Choose a shared Control UI composer suggestion model and agent metadata source, then implement agent mentions with accessible UI, plain-text serialization, history rendering, Gateway/protocol tests, and no duplicate identity API unless existing endpoints are insufficient.

## Reproduction Assessment

Yes. Source inspection shows typing @ cannot open any suggestion path because updateSlashMenu only matches slash patterns, the composer is a textarea, and there is no mention-chip or validated history-rendering path.

## Solution Assessment

Unclear. The UX direction is plausible and aligned, but the proposed agents.directory endpoint and rich composer architecture need maintainer scoping; reusing agents.list or agent.identity.get may be the better API boundary.
