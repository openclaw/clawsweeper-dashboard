## Best Possible Solution

Define one accessible Control UI composer suggestion contract, reuse existing agent metadata APIs unless a new boundary is justified, then implement agent mentions with focused composer, history-rendering, Gateway/protocol, and UI tests.

## Reproduction Assessment

## Solution Assessment

Unclear. The UX direction is plausible, but the proposed new `agents.directory` RPC and rich composer shape are not proven to be the best boundary while existing agent metadata APIs already expose much of the source data.
