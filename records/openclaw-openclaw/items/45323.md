## Summary

Keep this open. Current main has slash-command completion and an `agents.list` RPC, but it still lacks the requested `@` trigger, mention-chip composer, known-agent validation/rendering, and chat-history mention chips; this is a feature/product task rather than an implemented cleanup close.

## What This Changes

Request adds Slack-style `@agent` autocomplete, selectable mention chips in the Control UI chat composer, known-agent chip rendering in chat history, and a Gateway agent-directory data source.

## Best Possible Solution

Implement a Control UI mention feature with known-agent validation, plain-text serialization, and history rendering after deciding the composer architecture and any additive Gateway metadata needs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug; as an absence check, yes: current-main composer state, input handling, history rendering, Gateway handlers, and targeted searches show no `@` mention menu, rich mention chips, or `agents.directory` method.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact `contenteditable` plus new-RPC design; the maintainable path should first decide the composer UX and reuse `agents.list` unless a small additive protocol field is needed.
