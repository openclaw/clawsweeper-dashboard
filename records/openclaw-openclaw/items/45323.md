Keep open. Current main and the latest release still only have slash-command autocomplete in a textarea composer; the requested agent @mention autocomplete/chip UX remains missing and needs maintainer product direction rather than cleanup closure.

## What This Changes

Add Slack-style agent @mention autocomplete, visual mention chips, and history rendering to the Control UI chat composer.

## Best Possible Solution

Define one accessible Control UI composer suggestion contract that reuses `agents.list` identity metadata where possible, then implement agent @mentions, chip rendering, serialization, history styling, and focused UI/Gateway tests.

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks @agent mention autocomplete, chips, and mention-aware history rendering.

## Solution Assessment

Unclear. The user-facing direction is plausible, but the proposed new `agents.directory` endpoint is not obviously the best API because `agents.list` already exposes identity name and emoji; maintainers should choose the shared composer/API shape first.
