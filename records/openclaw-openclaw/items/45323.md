## Summary

Keep this open. Current main still has slash-command autocomplete and an `agents.list` RPC, but it does not implement the requested `@` trigger, mention chip composer, known-agent validation/rendering, or chat-history mention chips.

## What This Changes

The issue requests Slack-style `@agent-name` autocomplete in the Control UI chat composer, including known-agent suggestions, mention chips, plain-text serialization, and styled history rendering.

## Best Possible Solution

Implement this as a Control UI product task that reuses existing agent summary data where possible, adds only additive Gateway metadata if needed, serializes selected mentions back to plain `@agent` text, and renders chips only for known agents.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug, but yes for the current-main absence check: the composer only updates slash state from textarea input, and targeted source searches found no `@` mention, chip, or directory implementation.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact proposed `contenteditable` plus new `agents.directory` design; the maintainable path should decide the composer UX first and reuse `agents.list` unless additional metadata is required.
