## Summary

Keep open: current main and the latest release still have a textarea composer with slash-command suggestions only, no agent @mention autocomplete, no mention-chip history rendering, and no `agents.directory` RPC. The related custom-term suggestions issue is adjacent, not a canonical replacement, and the remaining work needs maintainer UX/protocol direction.

## What This Changes

Add Slack-style @mention autocomplete, styled mention chips, plain-text serialization, and known-agent-aware rendering to the Control UI chat composer.

## Best Possible Solution

Define one accessible Control UI composer suggestion contract, decide whether `agents.list` is sufficient or a narrower directory RPC is needed, then implement @mention autocomplete, plain-text serialization, and directory-aware history rendering with focused coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection of the composer, slash menu, keyboard handling, history renderer, Gateway descriptors, latest release tag, and related issue gives high confidence that the requested feature is absent.

## Solution Assessment

Unclear. The requested UX is reasonable, but maintainers should first choose whether this belongs in a shared composer suggestion framework and whether `agents.list` is sufficient or a new `agents.directory` protocol method is warranted.
