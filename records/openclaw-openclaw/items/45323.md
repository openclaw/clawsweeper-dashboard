Keep open: current main still has a textarea-based Control UI composer with slash-command suggestions only, no @agent autocomplete or mention chips, and no `agents.directory` RPC. This is a real feature request that needs maintainer UX/protocol direction rather than conservative cleanup.

## What This Changes

Add Slack-style @agent autocomplete, mention chips, plain-text serialization, history rendering, and an agent directory data source to the Control UI chat composer.

## Best Possible Solution

Define one accessible Control UI composer suggestion contract, decide whether `agents.list` is sufficient or a new directory RPC is warranted, then implement @agent autocomplete, plain-text serialization, and directory-aware history rendering with focused UI and gateway coverage.
