## Summary

Keep open: current main still does not provide Control UI agent @mention autocomplete, rich mention chips, mention-aware history rendering, or a dedicated agents.directory RPC. The request is a new UI/protocol feature with an overlapping composer-suggestion request at https://github.com/openclaw/openclaw/issues/57142, so it needs maintainer product direction rather than conservative cleanup closure.

## What This Changes

Add Slack-style @agent autocomplete and visual mention chips to Control UI chat, backed by an agent directory source and plain-text serialization.

## Best Possible Solution

Define one Control UI composer suggestion contract, reuse agents.list identity metadata unless a narrower directory RPC is justified, then implement accessible @mention autocomplete, chip serialization, and history rendering with focused coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection of the composer, slash-menu handling, history renderer, and Gateway methods is a high-confidence way to verify the requested @mention feature is absent on current main.

## Solution Assessment

Unclear. The requested user experience is reasonable, but maintainers should first decide whether to reuse agents.list or add agents.directory, and whether this should be a standalone mention implementation or part of a broader composer suggestion framework.
