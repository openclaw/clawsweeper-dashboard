## Summary

Keep this open. Current main has adjacent slash-command autocomplete and an existing `agents.list` data surface, but the requested `@` mention suggestions, composer chips, known-agent validation, and styled mention rendering in chat history are still missing.

## What This Changes

Add Slack-style `@agent` autocomplete, validated/styled mention chips in the composer and history, and an agent directory data source for Control UI chat.

## Best Possible Solution

Implement this as a Control UI product task: use existing agent summary data if it is sufficient, add only additive Gateway protocol fields or a directory method if needed, serialize selected mentions back to plain `@agent` text, and render chips only for known agents.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for the current-main absence check: typing `@` has no mention-menu path because the composer only updates slash state and searches found no mention-chip or known-agent renderer.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact proposed contenteditable plus new `agents.directory` design; the maintainable path should decide the composer UX first and reuse `agents.list` unless the existing summary data is insufficient.
