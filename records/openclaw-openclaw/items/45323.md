## Summary

This should stay open: current main still has slash-command autocomplete and an agent list RPC, but not `@agent` autocomplete, mention chips, or mention rendering in chat history. The request is a new Control UI/protocol UX feature, not a broken existing contract or cleanup-close candidate.

## What This Changes

Request Slack-style `@agent` autocomplete, rich mention chips, chat-history mention rendering, and possibly a new agent directory RPC for the Control UI chat.

## Best Possible Solution

Define the mention UX and protocol boundary first, likely reusing `agents.list` metadata where sufficient, then add focused composer/history support with accessibility, paste, serialization, and slash-command regression coverage.

## Reproduction Assessment

## Solution Assessment

Unclear. The user-facing direction is reasonable, but the proposed new `agents.directory` RPC may be unnecessary because `agents.list` already exposes ids and identity metadata; maintainers should decide the contract before implementation.
