## Summary

Keep this issue open. Current main has adjacent slash-command autocomplete and an agents.list data surface, but it still lacks @mention suggestions, mention chips, known-agent validation in the composer, and styled mention rendering in chat history.

## What This Changes

The issue requests Slack-style @agent mention autocomplete, selectable/styled mention chips, plain-text serialization, and known-agent mention rendering in Control UI chat.

## Best Possible Solution

Treat this as a Control UI product task: implement tested @mention autocomplete against existing agent summary data if sufficient, add only additive Gateway protocol fields or an agents.directory endpoint if needed, serialize selected mentions back to plain @agent text, and render only known-agent mentions in history.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for the current-main absence check: typing @ has no mention-menu path because the composer only calls updateSlashMenu, and searches found no mention-chip or known-agent mention renderer.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact proposed contenteditable plus agents.directory design; the maintainable path should decide the composer UX first and reuse agents.list unless the existing summary data is insufficient.
