## Summary

This should stay open: current main has slash-command autocomplete and an agent list RPC, but not Slack-style `@agent` autocomplete, inline mention chips, or mention rendering in chat history. The request is a new Control UI feature with protocol/UX choices, not a broken existing contract.

## What This Changes

Add Slack-style `@agent-name` autocomplete, mention chips, plain-text serialization, and history rendering to the Control UI chat composer.

## Best Possible Solution

Define the mention UX and protocol boundary first, likely reusing `agents.list` metadata where sufficient, then add focused Control UI composer/history support with regression coverage around slash commands, paste, serialization, and accessibility.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the current main behavior is limited to slash-command autocomplete plus plain-text `chat.send`.

## Solution Assessment

Unclear. The user-facing direction is reasonable, but adding a new `agents.directory` RPC may be unnecessary because `agents.list` already exposes agent ids and identity metadata; maintainers should decide the contract before implementation.
