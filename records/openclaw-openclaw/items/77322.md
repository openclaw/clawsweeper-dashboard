## Summary

Keep open: current main still has a source-reproducible stale implicit model cache path, and the two linked closing PRs are open implementation candidates rather than merged fixes.

## What This Changes

The issue requests that `/new` and related session creation paths stop carrying stale persisted `model`/`modelProvider` runtime fields after `agents.defaults.model.primary` changes when no explicit model override exists.

## Best Possible Solution

Review and land one focused fix that clears or recomputes stale runtime `model`/`modelProvider` state on `/new` and model-less session creation while preserving explicit user overrides.

## Reproduction Assessment

Yes, from source inspection: seed a persisted session row with plain `model`/`modelProvider` and no override fields, then follow the `/new` reset path; the new entry omits those fields and the store merge preserves the old values.

## Solution Assessment

Yes, the narrow maintainable solution is to use the existing default-reset behavior to delete or recompute runtime model fields when no explicit model is supplied, with regression coverage for both the auto-reply reset and gateway create/patch paths.
