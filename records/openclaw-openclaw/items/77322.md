## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path, and open PR #77339 already tracks the fix with closing syntax.

## What This Changes

The issue asks for `/new` and no-explicit-model session creation to resolve from the live `agents.defaults.model.primary` instead of carrying stale persisted `model`/`modelProvider` fields.

## Best Possible Solution

Review and land #77339 or an equivalent narrow fix that clears stale plain runtime `model`/`modelProvider` during auto-reply reset and covers the no-explicit-model gateway create/reset boundary with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session row with plain `model`/`modelProvider` and no overrides, change the default, then run auto-reply `/new` or create without an explicit model; current main can preserve the stale runtime fields.

## Solution Assessment

Yes. The narrow maintainable solution is to treat absent explicit model selection as reset-to-default at the session boundary or explicitly delete plain runtime cache fields when writing a fresh reset row, while preserving intentional user overrides.
