## Summary

Keep open: current main still has the stale runtime model cache path described in the report, and open PR #77339 uses closing syntax for the same bug, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue reports that plain session `model`/`modelProvider` runtime cache fields survive default-model changes and `/new` even when no explicit `modelOverride` exists.

## Best Possible Solution

Review and land PR #77339 or an equivalent focused fix that clears or recomputes runtime `model`/`modelProvider` during `/new` and reset when no explicit override exists, with regression coverage for the plain cache path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session entry with plain `model`/`modelProvider` and no override fields, run the auto-reply `/new` or no-model `sessions.create` path, and the current merge/no-op logic preserves the stale runtime fields.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing reset-to-default model path or explicitly delete runtime model cache fields on reset; adding provider policy or new configuration is unnecessary.
