## Summary

Keep open: current main still has the source-reproducible stale runtime model cache path, and the linked closing fix PR is still open and unmerged.

## What This Changes

The issue asks for `/new` and reset-created channel sessions to stop carrying stale persisted `model`/`modelProvider` runtime cache fields after `agents.defaults.model.primary` changes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77339 or an equivalent narrow patch that clears runtime `model`/`modelProvider` cache fields during reset-created channel sessions while preserving explicit user overrides.

## Reproduction Assessment

Yes. Seed a persisted channel session row with `modelProvider` and `model` but no overrides, trigger `/new`, and current main's reset merge preserves those fields before model resolution prefers them over the current default.

## Solution Assessment

Yes. Clearing only the runtime cache fields at the reset/create boundary is the narrow maintainable fix because explicit selections already live in `modelOverride` and `providerOverride`.
