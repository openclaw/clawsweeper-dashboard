## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path, and open PR #77339 is the active closing implementation candidate.

## What This Changes

The issue reports that channel session rows keep a cached implicit `model` across `/new` and default-model hot reload, causing new runs to ignore the current `agents.defaults.model.primary`.

## Best Possible Solution

Review and land #77339 or an equivalent narrow fix that clears runtime `model`/`modelProvider` during reset/create while preserving explicit user overrides.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: seed a channel session row with plain `model`/`modelProvider` and no overrides, then run `/new` or `sessions.create` without an explicit model and current main can preserve the old runtime fields.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or reset runtime model cache fields at the reset/create boundary and add regression coverage; the already-open closing PR appears to target that behavior.
