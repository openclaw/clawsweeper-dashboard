## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path, and open PR #77339 already references this issue with closing syntax, so the issue should stay open until that fix or an equivalent one merges.

## What This Changes

The issue asks for `/new`/reset and default-model hot reload to stop carrying a persisted plain session `model`/`modelProvider` cache when no explicit `modelOverride` exists.

## Best Possible Solution

Land #77339 or an equivalent narrow fix that deletes or recomputes runtime `model`/`modelProvider` on reset/create when no explicit model override exists, with regression coverage for persisted plain runtime fields.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session row with plain `model`/`modelProvider` and no overrides, then run `/new` or `sessions.create` without an explicit model and current main can preserve the old runtime fields.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or recompute runtime model cache fields at the reset/create boundary while preserving explicit user overrides; #77339 is already the active implementation candidate.
