## What This Changes

The issue reports that per-channel session `model` and `modelProvider` cache fields survive `/new` after `agents.defaults.model.primary` changes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77339 or an equivalent narrow patch that clears runtime `model`/`modelProvider` cache fields during reset-created channel sessions while preserving explicit user overrides.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a persisted channel session row with `modelProvider`/`model` and no overrides, trigger `/new`, and the merge path preserves those fields before model resolution prefers them over the current default.

## Solution Assessment

Yes. Clearing the runtime cache fields at the reset/create boundary is the narrow maintainable fix because explicit selections already live in `modelOverride` and `providerOverride`.
