## Summary

Keep open: current main still has a source-reproducible auto-reply reset path that can preserve stale `model`/`modelProvider` fields, and open PR #77339 uses closing syntax for this issue but is not merged yet.

## What This Changes

The issue asks for `/new` and reset-created channel sessions to resolve from the live `agents.defaults.model.primary` when no explicit model override exists.

## Best Possible Solution

Review and land #77339, or an equivalent patch that clears/replaces `model` and `modelProvider` on auto-reply reset before the store merge, with regression coverage for a session that has no explicit override fields.

## Reproduction Assessment

Yes. Seed an auto-reply channel session entry with plain `model`/`modelProvider` and no override fields, send an authorized `/new`, and the current reset merge can preserve those omitted runtime fields from the old store row.

## Solution Assessment

Yes. Clearing or recomputing runtime model fields during auto-reply reset is the narrow fix; adding a new config option or changing provider policy is unnecessary.
