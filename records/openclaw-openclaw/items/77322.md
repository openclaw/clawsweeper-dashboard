## Summary

Keep open: current main still has a source-reproducible path where auto-reply `/new` can preserve stale plain `model`/`modelProvider` fields, and open PR #77339 already references this issue with closing syntax.

## What This Changes

The issue asks for `/new` and reset-created channel sessions without explicit model overrides to use the live `agents.defaults.model.primary` instead of a cached `session.model`.

## Best Possible Solution

Land #77339 or an equivalent narrow fix that clears stale plain runtime `model`/`modelProvider` fields during reset/create while preserving intentional user overrides.

## Reproduction Assessment

Yes. Seed a channel session row with plain `model`/`modelProvider` and no overrides, change the default model, then trigger `/new`; source inspection shows the reset entry is merged over the old row without deleting those runtime fields.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or recompute runtime model cache fields at the reset/create boundary and add regression coverage; the already-open PR appears scoped to that behavior, so the issue should stay open until merge.
