## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path in the auto-reply reset flow, and open PR #77339 already tracks the fix with closing syntax.

## What This Changes

Changing `agents.defaults.model.primary` should be honored by a `/new` channel session when no explicit model override exists.

## Best Possible Solution

Land #77339 or an equivalent narrow fix that clears stale runtime `model`/`modelProvider` at the auto-reply reset boundary while preserving explicit user overrides and adding regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a channel session row with plain `model`/`modelProvider` and no overrides, then run `/new`; current main merges the old runtime fields into the new store row.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or recompute runtime model fields during auto-reply reset/create while preserving explicit user selections; the linked open PR appears to target that boundary.
