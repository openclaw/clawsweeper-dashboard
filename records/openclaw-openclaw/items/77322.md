## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path, and open PR #77339 already tracks the fix with closing syntax.

## What This Changes

The issue requests that `/new` and default-model hot reload stop carrying stale session `model`/`modelProvider` cache when no explicit override exists.

## Best Possible Solution

Review and land #77339 or an equivalent narrow fix that clears runtime `model`/`modelProvider` during reset/create while preserving explicit user overrides.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session row with plain `model`/`modelProvider` and no overrides, then run `/new` or `sessions.create` without an explicit model and current main preserves the old runtime fields.

## Solution Assessment

Yes. The maintainable fix is to clear or reset runtime model cache fields at the reset/create boundary and add regression coverage; the already-open PR appears to target that exact behavior, so the issue should stay open until merge.
