## Summary

## What This Changes

The issue reports that `/new` preserves cached `session.model`/`modelProvider` fields and ignores later `agents.defaults.model.primary` changes when no explicit model override exists.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77339 or an equivalent narrow repair that clears stale runtime `model`/`modelProvider` cache fields during auto-reply resets while preserving explicit user overrides.

## Reproduction Assessment

Yes. Seed a persisted channel session entry with `modelProvider` and `model` but no explicit override, trigger `/new`, and current main's merge path preserves those fields before `resolveSessionModelRef` prefers them over the configured default.

## Solution Assessment

Yes. The best fix is the narrow reset-path cleanup already represented by the open PR: clear only runtime cache fields while leaving explicit `modelOverride`/`providerOverride` selection logic intact.
