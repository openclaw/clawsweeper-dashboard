## Summary

Keep open: current main still has the embedded-run helper family internally, but the injected plugin runtime surface does not expose the requested abort, wait, active-check, or queue helpers. The related runId-based request at https://github.com/openclaw/openclaw/issues/52412 and the closed gateway PR at https://github.com/openclaw/openclaw/pull/67586 are adjacent, not superseding, because they do not cover this sessionId-scoped plugin runtime API.

## What This Changes

Expose the embedded Pi run abort, wait, active-check, and queue helpers through `api.runtime.agent` for plugin code.

## Best Possible Solution

Add an additive, documented, sessionId-scoped `api.runtime.agent` surface with lazy bindings, types, SDK mock coverage, runtime shape tests, docs, and explicit queue semantics, while keeping the separate runId abort API tracked independently.

## Reproduction Assessment

Yes. Static source inspection on current main shows the helper family exists in `src/agents/pi-embedded.ts` and `runs.ts`, while the plugin runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes with an API-shape caveat. The maintainable direction is a narrow sessionId-scoped runtime contract, but maintainers should decide whether the queue member should use the current deprecated boolean helper or a stronger outcome/async form.
