## Summary

Keep open: current main still lacks the requested `api.runtime.agent` helper family, while the lower-level embedded-run helpers already exist. The related gateway/runId abort work is adjacent but does not cover this sessionId-based injected runtime surface.

## What This Changes

Expose the existing embedded Pi run abort, wait, active-check, and queue helpers on `api.runtime.agent` for plugin runtime callers.

## Best Possible Solution

Add a narrow, documented, additive sessionId-based `api.runtime.agent` surface while preserving lazy runtime loading and keeping the separate runId abort API tracked independently.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the helper family exists internally, but the injected runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes with a small API-shape caveat. The narrowest maintainable path is to expose sessionId-scoped lazy runtime bindings plus types, docs, mock, and shape coverage without expanding into the broader runId abort API or all-runs internals.
