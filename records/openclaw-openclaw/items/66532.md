## Summary

Keep open: current main still has the embedded-run abort, wait, active-state, and queue helpers internally, and also has a separate agent-harness SDK escape hatch, but the requested injected `api.runtime.agent` contract is still missing from the runtime module, builder, type, docs, mock, and shape coverage.

## What This Changes

Expose embedded Pi run abort, wait-for-end, active-state, and queue helpers through `api.runtime.agent` for plugin runtime callers.

## Best Possible Solution

Add a focused, documented, lazy `api.runtime.agent` contract for the existing embedded-run coordination helpers while keeping the broader runId/gateway abort API separate.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the API gap: the internal helpers exist under `src/agents`, but the injected plugin runtime module, builder, type, docs, mock, and shape test omit them.

## Solution Assessment

Yes. Exposing the existing helpers through the lazy runtime boundary is the narrowest maintainable fix for trusted in-process plugin callers; it should not replace or expand into the separate runId-based abort API discussion.
