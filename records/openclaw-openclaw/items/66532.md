## Summary

Keep open: current main still has the embedded-run coordination helpers internally, but the injected `api.runtime.agent` contract exposes only the run starters and session helpers. The related runId-based abort request remains open and the closed gateway abort PR explicitly left this plugin runtime surface out of scope, so this is still a valid narrow feature request rather than an implemented or duplicate item.

## What This Changes

Expose existing embedded Pi run abort, wait, active-state, and queue helpers through the injected `api.runtime.agent` plugin runtime surface.

## Best Possible Solution

Expose and document only the existing sessionId-based embedded-run coordination helpers on `api.runtime.agent`, preserving lazy runtime loading and leaving the broader runId abort API separate.

## Reproduction Assessment

Yes. Static inspection of current origin/main reproduces the API gap: the internal helpers exist under `src/agents`, while the injected plugin runtime module, builder, type, docs, mock, and shape test omit them.

## Solution Assessment

Yes. Adding a focused lazy runtime binding, type contract, docs, mock, and shape coverage is the narrowest maintainable fix for trusted in-process plugin callers; it should not expand into the separate runId-based abort API.
