## Summary

Keep open. Current main still exposes only the embedded run starters through the plugin runtime, while the internal embedded-run helpers and the gateway abort work are adjacent but do not solve this trusted in-process plugin API gap.

## What This Changes

Expose embedded Pi run cancel, wait, active-state, and queued-message helpers through `api.runtime.agent` so plugins can coordinate embedded runs without reading global state.

## Best Possible Solution

Add additive, documented `api.runtime.agent` helpers backed by the existing embedded-run registry semantics, with runtime shape, mock, API-baseline, docs, and changelog coverage.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the API gap: the internal helpers exist under `src/agents`, but the plugin runtime barrel, runtime object, type contract, docs, mock, and shape test omit them.

## Solution Assessment

Yes. Exposing the existing embedded-run helpers through the lazy plugin runtime is the narrowest maintainable solution for trusted in-process plugins; the gateway/runId abort work should remain a separate contract.
