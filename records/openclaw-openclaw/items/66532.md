## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent` helper family. The internal sessionId helpers and separate public SDK aliases exist, but bundled plugin runtime callers still cannot access these helpers through the `api.runtime.agent` object requested here.

## What This Changes

Expose the existing sessionId-based embedded Pi run abort, wait, active-check, and queue helpers through `api.runtime.agent` for bundled plugin runtime callers.

## Best Possible Solution

Add a narrow, documented, additive `api.runtime.agent` surface for the existing sessionId-based embedded-run helpers while preserving lazy runtime loading and keeping runId abort work separate.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the helper family exists internally, while the injected plugin runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes. The narrowest maintainable fix is a lazy runtime binding plus type, mock, docs, and shape coverage for the existing sessionId helpers; it should not expand into the separate runId abort API.
