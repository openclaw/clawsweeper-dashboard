## Summary

Keep open: current main still has the internal embedded-run helpers, but `api.runtime.agent` does not expose the requested abort/wait/active/queue family. The related gateway/runId abort work is adjacent and does not cover this sessionId-based injected plugin runtime surface.

## What This Changes

Request to expose sessionId-based embedded-run abort, wait, active-check, and queue helpers through `api.runtime.agent` for plugin code.

## Best Possible Solution

Add a narrow, documented, additive sessionId-scoped `api.runtime.agent` surface with lazy bindings, types, mock coverage, shape tests, docs, and boundary checks while keeping the runId abort API tracked separately.

## Reproduction Assessment

Yes. Static inspection of current main shows the helper family exists internally, while the injected runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes, with an API-shape caveat. The narrow maintainable path is to expose sessionId-scoped lazy runtime bindings and document queue semantics, without expanding into the broader runId abort API or all-runs internals.
