## Summary

Keep open: current main and the latest release still expose the embedded-run helper family internally but do not plumb the requested sessionId-scoped abort/wait/active/queue helpers through `api.runtime.agent`; the related gateway and runId abort items are adjacent rather than superseding this plugin runtime surface.

## What This Changes

Expose sessionId-scoped embedded Pi run abort, wait, active-state, and queue helpers on the injected plugin runtime agent API.

## Best Possible Solution

Add a narrow, documented, additive sessionId-scoped `api.runtime.agent` surface with lazy bindings, types, SDK mock support, shape tests, docs, and explicit queue/abort semantics.

## Reproduction Assessment

Yes. Static inspection of current main and the latest release shows the helper family exists internally while the injected runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes, with an API-shape caveat. The maintainable path is an additive sessionId-scoped runtime surface, but maintainers should decide queue outcome semantics and avoid publishing broad all-runs abort behavior.
