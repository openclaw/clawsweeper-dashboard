## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent` helper family. The CoClaw report identifies a real plugin runtime API gap: the sessionId-based helpers exist internally, but only the run starters are wired into the trusted plugin runtime, and the related public SDK aliases plus the open runId abort request do not cover this exact surface.

## What This Changes

Expose the existing sessionId-based embedded Pi run coordination helpers on the injected `api.runtime.agent` surface for bundled plugins.

## Best Possible Solution

Add a narrow, documented, additive `api.runtime.agent` surface for the existing sessionId-based embedded-run helpers while preserving lazy runtime loading and keeping runId abort work separate.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the API gap: the internal helpers exist under `src/agents`, while the injected plugin runtime module, builder, type, docs, mock, and shape test omit them.

## Solution Assessment

Yes. A focused lazy runtime binding plus type, mock, docs, and shape coverage is the narrowest maintainable path for trusted in-process plugin callers; it should not expand into the separate runId abort API.
