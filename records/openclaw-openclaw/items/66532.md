## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent` helper family. The public `agent-harness` SDK path covers part of the steer/abort need, but the requested runtime object still omits abort, wait, active-check, and queue helpers for plugins that should use injected host-side runtime APIs.

## What This Changes

Expose existing embedded-run abort, wait, active-check, and queue helpers on the injected `api.runtime.agent` surface.

## Best Possible Solution

Add additive, documented, sessionId-scoped lazy runtime methods on `api.runtime.agent`, with type/mock/shape/doc coverage and no expansion into the separate runId abort API.

## Reproduction Assessment

Yes. Static inspection on current main reproduces the API gap: the helper family exists internally, while the injected plugin runtime wrapper, builder, type, docs, mock, and shape test omit it.

## Solution Assessment

Yes. The narrow maintainable path is a lazy runtime binding plus type, mock, docs, and shape coverage for the existing sessionId helpers; keep the broader runId cancellation API and queue-acceptance semantics separate.
