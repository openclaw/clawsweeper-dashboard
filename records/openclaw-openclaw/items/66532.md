## Summary

Keep open: current `main` still lacks the requested injected `api.runtime.agent` helper family. The closed public SDK alias path from https://github.com/openclaw/openclaw/issues/49624 helps external harness-style plugins, and the gateway abort work in https://github.com/openclaw/openclaw/issues/66531 is separate, but neither puts these sessionId helpers on the plugin runtime object requested here.

## What This Changes

The issue requests exposing existing embedded Pi run cancellation, wait, active-check, and queue helpers through `api.runtime.agent` for bundled plugins.

## Best Possible Solution

Expose the existing sessionId-based embedded-run helpers as a narrow, documented, additive `api.runtime.agent` surface while preserving lazy runtime loading and keeping runId abort work separate.

## Solution Assessment

Yes. The narrowest maintainable solution is to add lazy runtime bindings plus type, mock, docs, and shape coverage for the existing sessionId helpers without expanding into the separate runId abort API.

Add lazy runtime exports/bindings for `abortEmbeddedPiRun`, `waitForEmbeddedPiRunEnd`, `isEmbeddedPiRunActive`, and `queueEmbeddedPiMessage`; update `PluginRuntimeCore['agent']`, `createRuntimeAgent()`, the runtime shape test, plugin runtime mock, SDK runtime/migration docs, and changelog if user-visible. Keep the change additive and avoid exposing process-global internals directly.
