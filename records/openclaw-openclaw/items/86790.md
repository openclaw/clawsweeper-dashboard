## Summary

Keep open: current main still emits `plugins.metadata.scan` spans on plugin metadata memo hits, the latest release has the same behavior, and the linked closing PR remains open rather than merged.

## What This Changes

The issue reports that plugin metadata memo cache hits still emit `plugins.metadata.scan` diagnostics spans, flooding hot-path diagnostics timelines even when the cached snapshot is reused.

## Best Possible Solution

Land a focused change that returns memoized plugin metadata snapshots directly on hits while preserving real scan instrumentation on misses, preferably by refreshing and proving the linked PR.

## Reproduction Assessment

Yes, source-level. Current main wraps memo hits in `measureDiagnosticsTimelineSpanSync("plugins.metadata.scan")`, and that helper emits timeline span events when diagnostics are enabled; I did not run a live gateway profile in this read-only review.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable repair: return the memoized snapshot directly on cache hits while leaving real scan spans on misses. The issue should remain open until that implementation merges or maintainers choose a replacement path.
