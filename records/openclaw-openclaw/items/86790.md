## Summary

Keep open: current main and latest release still emit `plugins.metadata.scan` spans for metadata memo hits, while the linked closing PR is open and not merged.

## What This Changes

The issue asks OpenClaw to return cached plugin metadata snapshots without emitting full `plugins.metadata.scan` diagnostics spans on cache hits.

## Best Possible Solution

Rebase and land the linked narrow fix, or an equivalent patch, so cache misses keep real scan spans while memo hits return the cached snapshot without timeline span creation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: repeated `loadPluginMetadataSnapshot` calls hit the memo, and current main still wraps those hits in `measureDiagnosticsTimelineSpanSync`. I did not run a live gateway profile in this read-only review.

## Solution Assessment

Yes, the best fix is narrow: return the memoized snapshot directly on cache hits while leaving real scan misses instrumented and covered by a timeline-count regression test.
