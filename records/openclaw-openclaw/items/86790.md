## Summary

Keep open: current main and latest release still emit `plugins.metadata.scan` spans on plugin metadata memo hits, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks cache-hit plugin metadata snapshot lookups to return memoized data without emitting full `plugins.metadata.scan` diagnostics spans.

## Best Possible Solution

Keep real scan spans on cache misses, bypass diagnostics span creation on memo hits, and cover the timeline event count with the focused regression test in the linked PR.

## Reproduction Assessment

Yes, from source inspection: current main returns memo hits through `measureDiagnosticsTimelineSpanSync`, and that helper emits timeline start/end events when diagnostics timeline is enabled. I did not run the test path because this review is read-only.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: return the memoized snapshot directly on cache hits while leaving real scans instrumented and adding focused timeline-count coverage.
