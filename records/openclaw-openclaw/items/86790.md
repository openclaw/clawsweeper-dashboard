## What This Changes

The issue reports that plugin metadata memo hits still emit `plugins.metadata.scan` diagnostics spans, producing noisy hot-path timelines.

## Best Possible Solution

## Solution Assessment

Yes. The narrow maintainable solution is to return the memoized snapshot directly on cache hits while leaving real metadata scan instrumentation on misses; the linked PR targets that exact path but remains open.
