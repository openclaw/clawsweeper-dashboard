## Summary

Keep open: current main and the latest release still emit `plugins.metadata.scan` spans on plugin metadata memo hits, and the issue is already paired with an open closing PR that should land or be replaced before the issue closes.

## What This Changes

The issue asks OpenClaw to stop emitting `plugins.metadata.scan` diagnostics spans for memoized plugin metadata cache hits while keeping scan spans for real metadata scans.

## Best Possible Solution

Refresh, review, and land or replace https://github.com/openclaw/openclaw/pull/86796 so memo hits return directly while real metadata scans keep diagnostics coverage.

## Reproduction Assessment

Yes, at source level. Current main wraps metadata memo hits in `measureDiagnosticsTimelineSpanSync("plugins.metadata.scan")`, and the diagnostics helper emits span events when timeline diagnostics are enabled; I did not run a live gateway profile in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to return the memoized snapshot directly on cache hits while leaving real scan spans on misses; the linked PR targets that exact path but remains open.
