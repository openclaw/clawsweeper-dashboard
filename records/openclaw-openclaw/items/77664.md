## Summary

Keep open: current main still emits the misleading embeddings error for documented BM25-only QMD status, and this narrow owner-local PR aligns the probe result with the docs and CLI rendering without review findings.

## What This Changes

The PR makes `probeEmbeddingAvailability()` return `{ ok: false, error: undefined }` for QMD `searchMode="search"`, updates the QMD manager test expectation, and adds a changelog fix entry.

## Best Possible Solution

Merge a narrow QMD manager fix with the regression test and changelog so BM25-only `memory status --deep` no longer displays an embeddings error.

## Reproduction Assessment

Yes, source-reproducible: current main clears QMD vector detail for `searchMode="search"`, then converts that missing detail into the default embeddings error, and the current focused test asserts the misleading string.

## Solution Assessment

Yes. Handling the BM25-only case inside the QMD manager is the narrowest maintainable fix because it keeps QMD-specific semantics with the QMD owner code and preserves real vector-mode diagnostics for `vsearch` and `query`.
