## Summary

Keep open: current main and the latest release still miss the Kiro stale-resume `Internal error` recovery path, and the open linked PR is the active fix candidate but has not merged.

## What This Changes

Recover thread-bound ACP persistent sessions after Kiro invalidates a stored resume ID, so the manager discards stale identity and retries with a fresh session instead of failing the turn.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/93547, or an equivalent structured `SESSION_RESUME_REQUIRED` recovery, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes at source level: current main and the latest release require stale-resume wording like `resource not found`, while the report and acpx contract support Kiro's `Internal error` resume-required shape. I did not run a live Kiro backend in this read-only issue review.

## Solution Assessment

Yes: the narrow maintainable solution is to preserve and match acpx's structured `SESSION_RESUME_REQUIRED` signal rather than expanding backend reason-text regexes. The open linked PR is the current candidate for that path.
