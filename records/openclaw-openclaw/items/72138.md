## Summary

Keep open. Current main still lacks Feishu normal-reply sent-hook emission, and this PR remains the focused implementation candidate, but the branch is stale against newer Feishu dispatcher behavior and still lacks real Feishu runtime proof.

## What This Changes

The PR adds Feishu normal-reply `message_sent` and internal `message:sent` hook emission, session/group metadata plumbing, streaming-card message-id access, and dispatcher tests.

## Best Possible Solution

Land a rebased Feishu-local dispatcher fix that mirrors canonical sent-hook emission while preserving current Feishu streaming, mention, voice-media, block-streaming, and missing-thread fallback behavior, with real Feishu proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Feishu normal replies are sent by `createFeishuReplyDispatcher`, while canonical plugin `message_sent` and internal `message:sent` emission lives in the shared outbound path that this dispatcher bypasses.

## Solution Assessment

No, not as-is. The dispatcher is the right fix boundary, but the branch must be rebased over current main without restoring older Feishu behavior and needs real Feishu proof before merge.
