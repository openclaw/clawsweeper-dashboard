## Summary

Keep open. Current main and the latest release still route the visible WebChat attach button through a document-wide hidden-input click path, while the focused tests exercise direct input change events rather than the reported button-to-picker route.

## What This Changes

The issue reports that the WebChat upload button in the Control UI does nothing when clicked in Chrome on Windows.

## Best Possible Solution

Keep the existing attachment pipeline, make the chat composer attach control target its own file input through a component-scoped ref or label-backed control, and add coverage for the visible button-to-input activation path.

## Reproduction Assessment

No high-confidence live Chrome/Windows reproduction has been run. Source inspection still traces the reported route to a brittle and untested visible button-to-hidden-input activation path on current main and v2026.5.12.

## Solution Assessment

Yes. A narrow Control UI repair to the attach button's activation target plus focused view-level coverage is the maintainable solution; downstream attachment serialization and Gateway media handling should stay unchanged.
