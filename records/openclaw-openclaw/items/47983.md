## Summary

Keep open. Current main supports selected-file handling and generic non-video attachment delivery, but the visible Control UI/WebChat attach button still uses a document-wide hidden-input click path and there is no regression test for the reported click-to-picker route; merged #72299 addressed downstream attachment handling, not this activation path.

## What This Changes

Make the WebChat/Control UI upload button open the file chooser and allow selected files to be sent as attachments.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media behavior, but make the chat composer attach control target its own file input through a component-scoped ref or label-backed input, add type="button", and cover the visible click-to-input activation path in chat view tests.
