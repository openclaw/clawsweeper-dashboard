## Summary

Keep open. Current main and the latest release still leave the visible WebChat attach button on a document-wide hidden-input click path, and the focused attachment tests still bypass that visible button path.

## What This Changes

The issue asks for the WebChat upload button to open the file chooser and send selected files as chat attachments.

## Best Possible Solution

Keep the existing attachment pipeline, but make the composer attach control target its own file input through a component-scoped ref or label-backed input and add regression coverage for clicking the visible button.

## Reproduction Assessment

No high-confidence live reproduction was run. The reported route is traceable in current source and the latest release: the visible button uses a global hidden-input click path, while tests only exercise direct input change events.

## Solution Assessment

Yes. A narrow Control UI fix to the button-to-input activation path plus focused view-level coverage is the maintainable solution; downstream attachment serialization and Gateway media handling should stay unchanged.
