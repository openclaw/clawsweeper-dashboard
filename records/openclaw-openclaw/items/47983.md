## Summary

Keep open. Current main still documents and renders WebChat attachment uploads, but the visible Attach file control still activates the hidden input through an unscoped `document.querySelector` path and the regression tests only dispatch `change` directly on the hidden input, so the reported visible-button failure is not safely closeable.

## What This Changes

The issue reports that the WebChat upload button in OpenClaw 2026.3.13 on Chrome/Windows does not open the file picker.

## Best Possible Solution

Keep the existing attachment serialization and gateway media behavior intact, but make the composer Attach file control target its own file input through a component-scoped ref or label-backed input, add `type="button"`, and cover the visible click-to-input activation route in chat view tests.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the reported visible activation path still exists and lacks regression coverage, but this review did not run a Chrome/Windows native file-picker smoke.

## Solution Assessment

Yes. A narrow Control UI repair to the button-to-input activation path plus a focused view-level regression test is the smallest maintainable fix; downstream attachment payload and gateway media handling should stay unchanged.
