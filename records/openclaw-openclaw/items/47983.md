## Summary

Keep open. Current main still renders the hidden WebChat/Control UI file input and downstream attachment pipeline, but the visible upload button still activates it through an unscoped `document.querySelector` path and existing tests only dispatch `change` on the hidden input directly.

## What This Changes

The issue asks for the WebChat/Control UI upload button to open the browser file picker and allow selected files to be sent as chat attachments.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media behavior intact, but make the composer Attach file control target its own input through a scoped ref or label-backed control, add `type="button"`, and add a regression test for clicking the visible control.

## Solution Assessment
