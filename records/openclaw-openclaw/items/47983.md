## Summary

Keep open. Current main still renders the WebChat attachment input and downstream file pipeline, but the visible attach control still uses an unscoped document-wide lookup to activate the hidden input and there is no regression test for the reported visible-button path.

## What This Changes

Clicking the WebChat/Control UI upload button should open the file picker and allow selected files to be sent as attachments.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media behavior intact, but make the chat composer Attach file control target its own file input through a component-scoped ref or label-backed input, add type="button", and add a regression test that clicks the visible control and verifies the hidden input activation path.

## Reproduction Assessment

Unclear. The issue gives concrete Chrome/Windows steps and source inspection shows the same visible-button activation path remains, but this read-only review did not run a live browser file-picker smoke.

## Solution Assessment

Yes. A narrow Control UI fix to the button-to-input activation path plus a view-level regression test is the smallest maintainable repair; downstream attachment serialization and Gateway media handling should stay unchanged for this bug.
