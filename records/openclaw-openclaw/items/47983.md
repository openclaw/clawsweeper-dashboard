## Summary

Keep open. Current main still supports selected-file handling and downstream non-image attachment payloads, but the visible WebChat Attach file button still activates the hidden input through a document-wide selector, and no regression test covers clicking that visible control.

## What This Changes

The issue asks for the WebChat upload button to open the browser file picker and allow selected files to be sent as chat attachments.

## Best Possible Solution

Keep the existing attachment serialization and gateway media behavior intact, but make the chat composer Attach file control target its own file input through a component-scoped ref or label-backed input, add type="button", and cover the visible button-to-input activation path in chat view tests.

## Reproduction Assessment

Unclear. The issue gives concrete Chrome/Windows steps and current source still shows the same visible-button activation path, but this read-only review did not run a live browser file-picker smoke against current main.

## Solution Assessment

Yes. A narrow Control UI fix to the button-to-input activation path plus a focused view-level regression test is the smallest maintainable repair; downstream attachment serialization and Gateway media handling should stay unchanged for this bug.
