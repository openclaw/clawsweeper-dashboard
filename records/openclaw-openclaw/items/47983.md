## Summary

Keep open. Current main still has the visible WebChat attach control routed through a document-wide hidden-input click path, and the tests only exercise direct change events on the hidden input rather than the user-visible button path reported as dead.

## What This Changes

The issue reports that the WebChat upload button in OpenClaw 2026.3.13 does not open the file picker on Chrome/Windows.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media handling, but make the composer attach control target its own file input through a component-scoped ref or label-backed input and cover that visible-button path in a regression test.

## Reproduction Assessment

Unclear. The report gives concrete Chrome/Windows steps and current source still shows the same visible-button activation path, but this read-only review did not run a live browser file-picker smoke.

## Solution Assessment

Yes. A narrow Control UI fix to the button-to-input activation route plus a view-level regression test is the smallest maintainable repair; downstream attachment handling should stay unchanged.
