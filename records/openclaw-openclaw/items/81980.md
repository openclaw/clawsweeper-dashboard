## What This Changes

The issue reports that `openclaw devices approve <deviceId>`/`device.pair.approve` fails with `unknown requestId` on released Linux npm installs while list and health WebSocket calls work.

## Best Possible Solution

Keep the request-id based approval contract, but add a focused fix or diagnostic once exact list/approve evidence shows whether current pending state is being lost, superseded, or confused with a device id.
