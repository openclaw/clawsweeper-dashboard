## Summary

Keep open. The runtime hardening is useful source work, but this PR should not merge as-is because it breaks supported attachment-only chat sends and its reply-normalizer hunk is stale against current main.

## What This Changes

The PR adds runtime guards around reply payload text, `chat.send` message sanitization, chat abort parsing, and TUI stream text trimming to avoid unsafe `trim()` calls.

## Best Possible Solution

Land a narrow current-main fix that normalizes reply payload text once, preserves attachment-only `chat.send`, adds focused regression coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes. A focused unit test can call `normalizeReplyPayload` with a truthy non-string runtime `text` plus media or channel data and prove the raw value reaches string-only silent-token helpers on current main.

## Solution Assessment

No as-is. The PR is directionally useful, but the sanitizer change rejects valid attachment-only sends and the normalizer guard needs to be reapplied to current main's `hasContent` and `normalizeOptionalString` shape.
