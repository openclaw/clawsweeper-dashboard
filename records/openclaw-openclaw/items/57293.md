## Summary

Keep open. The config-schema bug is still present on current main and this PR contains the right narrow schema move, but it should not merge as-is because it also carries stale systemd `EnvironmentFile=` changes that current main has since superseded and that would reintroduce service env precedence and secret-handling risk.

## What This Changes

The PR moves `approvalRunningNoticeMs` into the shared exec config schema and adds systemd `EnvironmentFile=` handling that filters state-dir `.env` keys out of inline service environment entries.

## Best Possible Solution

Land or replace this with a config-only fix that adds the key to `ToolExecBaseShape`, updates the current generated/config docs surfaces that still exist, adds focused `ToolsSchema` regression coverage, and leaves the current curated systemd env-file implementation intact.

## Reproduction Assessment

Yes, at source level. Current main documents and reads `tools.exec.approvalRunningNoticeMs`, but strict global `ToolExecSchema` is built from a base shape that omits the key.

## Solution Assessment

No. The schema move is the right fix, but bundling stale systemd secret-handling changes is not the narrowest maintainable solution for this config bug.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the curated systemd env file:** `src/daemon/systemd.ts:455`
  - body: The new unit points `EnvironmentFile=` at the raw state-dir `.env`. Current main writes a curated `gateway.systemd.env` and filters stale or inline-managed keys; loading raw `.env` imports unrelated assignments and bypasses that protection.
  - confidence: 0.9
- **[P2] Preserve fresh inline service overrides:** `src/daemon/systemd.ts:464-466`
  - body: Filtering inline values only because the key exists in `.env` lets stale file values win at service start, since systemd `EnvironmentFile=` entries override `Environment=` entries. Fresh gateway tokens or ports supplied by the install plan can be replaced by older file values.
  - confidence: 0.9
