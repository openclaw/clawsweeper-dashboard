## Summary

Keep this issue open: current main and the latest release still lack the 431 split classification, and the open linked fix PR at https://github.com/openclaw/openclaw/pull/92650 is the right owner for the remaining work.

## What This Changes

Large built-in memory imports using OpenAI or OpenAI-compatible embeddings can fail with 431 unless OpenClaw internally splits oversized embedding batches or emits an actionable batching error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92650 or an equivalent memory-core classifier regression, then close this issue when the fix merges.

## Reproduction Assessment

Yes, from source inspection rather than a local run. The issue gives a concrete Docker/OpenAI repro, and current main routes OpenAI embedding HTTP errors into a split classifier that does not match the reported 431 shape.

## Solution Assessment

Yes. The existing shared memory-core split classifier is the narrowest maintainable fix boundary; adding CLI-only chunking guidance or a new config knob would duplicate the existing bisection mechanism.
