## Summary

Keep open: the feature direction is plausible, but the proposed incident stream currently loses session identity for hidden external-channel runs, and the supplied terminal proof exercises the new helper directly rather than a reply-runner fallback path.

## What This Changes

Adds a typed `incident` agent event and reply-runner fallback classification for command-lane, response-generation, session-history, and generic runtime failures.

## Best Possible Solution

Add the structured incident event through a redaction-safe session identity path, cover hidden external-channel incidents in tests, and prove the real reply-runner fallback integration before merge.

## Reproduction Assessment

Yes, at source level for the review finding: external-channel runs register hidden context and current `emitAgentEvent` strips `sessionKey` from every non-lifecycle hidden event. I did not run a full gateway timeout reproduction.

## Solution Assessment

No. Emitting structured incidents is a reasonable direction, but this patch should preserve session identity for the intended observer path and prove the integrated fallback behavior, not only the helper.
