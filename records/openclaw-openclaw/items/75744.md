## Summary

Keep open. Current main still has a source-level path where a scoped model-picker clear can remove the provider's selectable models while `agents.defaults.model.primary` is preserved unconditionally, leaving the dangling default described in the report.

## What This Changes

The issue requests that `openclaw configure` prevent model allowlist changes from leaving `agents.defaults.model.primary` or fallbacks pointing at deselected provider models.

## Best Possible Solution

Keep this issue open and update the generic configure/model-picker flow so provider-scoped removals cannot leave default model references dangling; the safest behavior is to block or ignore deselection of an in-scope primary/fallback until the user chooses a replacement or clears the default deliberately.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: return an empty scoped selection from `promptModelAllowlist` for a scope containing the configured primary, then observe `applyModelFallbacksFromSelection` preserving that primary while `applyModelAllowlist` removes the scoped allowlist entry.

## Solution Assessment

No. The current implementation is not the best way to solve the issue because it preserves `primary` unconditionally; a narrower maintainable fix should validate or repair primary/fallback references before applying scoped allowlist removals.
