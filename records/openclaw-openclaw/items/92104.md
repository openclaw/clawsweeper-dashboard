## Summary

Current main now resolves bundled provider static-catalog models through the shared async resolver used by image/PDF understanding, addressing the reported divergence between model listing and execution. The implementation merged after v2026.6.6 and is not yet in a release.

## What This Changes

Fix image/PDF tools rejecting configured or catalog-visible image-capable models as unknown while the same models work for normal agent runs.

## Best Possible Solution

Keep the merged shared provider static-catalog fallback as the canonical resolution path and retire the older overlapping issue-specific PR rather than maintaining a second media-only capability repair.

## Reproduction Assessment

Yes at source level. The reported configurations map directly to the media resolver's static-catalog fallback path, and the merged fix includes live proof of the same visible-in-catalog but unknown-at-execution failure class plus focused regression coverage.

## Solution Assessment

Yes. Fixing bundled provider static-catalog resolution in the shared async resolver is narrower and more maintainable than adding a media-only fallback, and it benefits every caller using the same resolution contract.
