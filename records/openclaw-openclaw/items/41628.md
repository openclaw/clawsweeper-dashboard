## Summary

Keep open: current main already has unique-prefix approval lookup, but the reply-to approval shortcut remains unique to this PR. The patch still has approval-routing correctness issues and lacks the required real Telegram proof for a reply-context change.

## What This Changes

The PR adds a single-token `/approve <decision>` path that extracts the approval ID from the replied message body and updates exec/plugin approval prompts to advertise reply-to and prefix shortcuts.

## Best Possible Solution

Keep the reply-to shortcut, parse the full generated ID line, render plugin tips from request-scoped decisions, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR regex truncates whitespace-containing IDs, and the plugin tip hard-codes decisions that current main validates per request. I did not run tests because this is a read-only review.

## Solution Assessment

No. The feature direction is narrow and useful, but the patch should preserve full generated IDs, avoid misleading plugin approval text, and provide real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Parse the full replied approval ID line:** `src/auto-reply/reply/commands-approve.ts:40`
  - body: `APPROVAL_ID_RE` captures only `\S+`, so a replied `ID: task 1` line submits `task` even though approval IDs are generic non-empty strings and the decision-first parser already preserves spaces. Capture and trim the full generated ID line before resolving it.
  - confidence: 0.93
- **[P2] Keep plugin approval tips request-scoped:** `src/infra/plugin-approvals.ts:70-72`
  - body: The added plugin approval tip always advertises `allow-always`, but plugin approvals can restrict `allowedDecisions` and the resolver rejects unavailable decisions. Build this reply-to tip from the request's allowed decision list so unavailable actions are not shown.
  - confidence: 0.95

## Security Review
