## Summary

Keep open: current main already has ID prefix lookup, but this PR still contains useful reply-to approval work that is not on main and needs a P2 extractor fix plus real Telegram proof before merge.

## What This Changes

The PR adds reply-to `/approve <decision>` ID extraction from `ReplyToBody`, updates exec/plugin approval hint text, and adds approval command and forwarder tests.

## Best Possible Solution

Keep the existing main prefix lookup, fix the reply-to extractor to use the whole `ID:` line, add whitespace/custom-ID regression coverage, and attach redacted Telegram proof of the visible approval flow.

## Reproduction Assessment

Yes for the review finding via source inspection: in the PR head, replying to a message containing `ID: task 1` with `/approve allow-once` returns `task` from `extractApprovalIdFromReplyBody`. I did not run tests because this was a read-only review.

## Solution Assessment

No. The reply-to shortcut is a reasonable narrow direction, but the implementation should preserve the full approval ID line and prove the visible Telegram behavior before merge.
