## Summary

Keep open: current main already has the prefix-matching half, but the reply-to shortcut is still unique work and the latest PR head still has correctness issues around approval ID parsing and plugin approval hint text. The external PR also lacks redacted real Telegram proof for the changed chat flow.

## What This Changes

The PR adds single-token `/approve <decision>` parsing that extracts an approval ID from `ReplyToBody`, updates approval request hint text, and adds parser/handler/forwarder tests.

## Best Possible Solution

Keep the reply-to shortcut, parse and trim the complete generated `ID:` line, build plugin hint text from each request's allowed decisions, and attach redacted real Telegram proof before merge.

## Reproduction Assessment

Yes. The review findings are reproducible by source inspection on the PR head: the reply regex truncates whitespace-containing IDs, and the plugin approval tip hard-codes decisions that current main validates per request.

## Solution Assessment

No. The feature direction is narrow and useful, but the current patch should preserve full approval IDs, keep plugin approval copy request-scoped, and provide real Telegram proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Parse the full replied approval ID line:** `src/auto-reply/reply/commands-approve.ts:40`
  - body: `APPROVAL_ID_RE` captures only `\S+`, so a replied `ID: task 1` line submits `task` even though approval IDs are generic non-empty strings and approval resolution accepts exact IDs or unique prefixes. Capture and trim the full generated ID line before resolving it.
  - confidence: 0.93
- **[P2] Keep plugin approval tips request-scoped:** `src/infra/plugin-approvals.ts:70-72`
  - body: The new plugin approval tip always advertises `allow-always`, but plugin approvals can restrict `allowedDecisions` and the resolver rejects unavailable decisions. Build this reply-to tip from `resolvePluginApprovalRequestAllowedDecisions(request.request)` like the existing reply line.
  - confidence: 0.95

## Security Review

Status: needs_attention
