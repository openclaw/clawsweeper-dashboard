## Summary

Keep open: current main already has ID-prefix approval lookup, but this PR still adds a meaningful reply-to approval feature and the current diff truncates whitespace-containing replied IDs while lacking real Telegram proof.

## What This Changes

The PR adds reply-to `/approve <decision>` parsing from `ReplyToBody`, updates exec/plugin approval hints to advertise reply-to and ID-prefix shortcuts, and adds approval parser/handler/forwarder tests.

## Best Possible Solution

Preserve the reply-to shortcut, capture and trim the full `ID:` line, add whitespace/custom-ID regression coverage, and attach redacted real Telegram proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR captures only `\S+` from replied `ID:` lines while current main accepts generic non-empty IDs and preserves multi-token explicit IDs. I did not run tests because this review is read-only.

## Solution Assessment

No. The reply-to approval feature is a reasonable narrow improvement, but this implementation should capture the full replied ID line and include real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect
