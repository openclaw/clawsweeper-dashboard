## Summary

Keep open. The remaining PR payload is useful reply-to approval support, but the current patch can truncate valid approval IDs from replied messages and it lacks inspectable after-fix Telegram proof.

## What This Changes

The PR branch changes chat `/approve` parsing to allow `/approve <decision>` when replying to an approval request, extracts the ID from `ReplyToBody`, and updates approval hint text/tests for reply and prefix shortcuts.

## Best Possible Solution

Capture and trim the whole `ID:` line, add focused regression coverage for whitespace IDs and conflicting prefixes, then provide redacted live Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reply to a message containing `ID: task 1` with `/approve allow-once`, and the PR extractor submits only `task`.

## Solution Assessment

No. The reply-to shortcut is a reasonable direction, but the implementation should preserve the full approval ID line and prove the visible Telegram behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Capture the full replied approval ID line:** `src/auto-reply/reply/commands-approve.ts:41`
