## Summary

Keep open. The reply-to `/approve` shortcut is useful and current `main` already has prefix lookup, but this branch still has approval-ID parsing and plugin decision-text blockers plus missing real Telegram proof.

## What This Changes

The branch adds reply-to `/approve <decision>` ID extraction from `ReplyToBody`, exports/tests approval parser helpers, and updates exec/plugin approval request text to advertise reply-to and prefix shortcuts.

## Best Possible Solution

Keep the reply-to shortcut, but parse the generated approval ID safely, keep plugin tip text based on each request's allowed decisions, rebase on current main, and attach redacted Telegram proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR truncates whitespace-containing replied IDs and advertises unavailable plugin decisions while current main accepts generic IDs and validates request-scoped decisions. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is narrow and useful, but this patch should preserve full generated IDs, avoid misleading plugin approval text, and provide real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Parse the exact generated approval ID line:** `src/auto-reply/reply/commands-approve.ts:40`
  - body: `APPROVAL_ID_RE` captures only `\S+`, so a replied `ID: task 1` line submits `task` even though approval IDs are generic non-empty strings and explicit `/approve allow-once <id>` preserves spaces. Parse and trim the full generated ID line before sending it to the prefix-capable resolver.
  - confidence: 0.93
- **[P2] Keep plugin approval tips request-scoped:** `src/infra/plugin-approvals.ts:71-72`
  - body: The added plugin approval tip always advertises `allow-always`, but plugin approvals can restrict `allowedDecisions`; current main text/tests and the resolver reject unavailable decisions, so requests that allow only `allow-once|deny` would display an action that fails.
  - confidence: 0.95
