## Summary

Keep open. The reply-to approval shortcut is useful, but the current extractor can capture a spoofed or truncated ID before the canonical approval ID, so the approval path needs another branch update before merge.

## What This Changes

The branch adds reply-to `/approve <decision>` ID extraction from `ReplyToBody`, exports parser/extractor helpers for tests, updates exec/plugin approval hint text, and expands approval tests.

## Best Possible Solution

Update the extractor to parse the canonical rendered approval ID as a full trimmed line from the known approval message structure, add regression coverage for embedded `ID:` lines and whitespace IDs, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a plugin approval whose title or description contains a newline followed by `ID: plugin:other` before the canonical ID, then replying with `/approve allow-once`; a custom exec ID containing whitespace also reproduces truncation.

## Solution Assessment

No. Reply-to-message approval is the right UX direction, but first-`ID:` and `\S+` extraction is not the narrowest safe implementation for an approval path.

## Review Findings
