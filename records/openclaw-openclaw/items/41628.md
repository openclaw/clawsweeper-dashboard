## Summary

Keep open: current main already covers prefix approval lookup, but the remaining reply-to shortcut still has a concrete approval-ID truncation bug and lacks required real Telegram proof before merge.

## What This Changes

The PR adds reply-to `/approve <decision>` ID extraction from `ReplyToBody` and updates exec/plugin approval request text to advertise reply-to and prefix shortcuts.

## Best Possible Solution

Keep the current main prefix lookup, fix the reply-to extractor to capture and trim the full `ID:` line, add whitespace/custom-ID regression coverage, and attach redacted Telegram proof of the visible flow.

## Reproduction Assessment

Yes for the review finding by source inspection: in the PR head, `ID: task 1` is captured as `task` while current main accepts generic non-empty approval IDs and resolves submitted values as exact IDs or unique prefixes. I did not run tests because this review is read-only.

## Solution Assessment

No. The reply-to shortcut is a reasonable narrow feature, but this implementation should preserve the full approval ID line and include real Telegram proof before merge.
