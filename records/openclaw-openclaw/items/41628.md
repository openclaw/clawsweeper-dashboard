Full review comments:

- **[P1] Keep the current gateway scope gate:** `src/auto-reply/reply/commands-approve.ts:224-228`
  - body: This branch still calls `requireGatewayClientScopeForInternalChannel`, while current main uses `requireGatewayClientScope` so every gateway client needs `operator.approvals` or `operator.admin` for `/approve`.   - confidence: 0.88
- **[P2] Capture the full replied approval ID line:** `src/auto-reply/reply/commands-approve.ts:40`
  - body: `APPROVAL_ID_RE` captures only `\S+`, but approval IDs are generic non-empty strings. A replied message with `ID: task 1` would submit `task`, which can fail, become ambiguous, or hit a different exact pending ID once prefix matching is in play; capture and trim the whole line instead.
  - confidence: 0.86
- **[P2] Preserve request-scoped plugin approval decisions:** `src/infra/plugin-approvals.ts:72`
  - body: Current main formats plugin approval text from `resolvePluginApprovalRequestAllowedDecisions`, but this branch adds a hard-coded reply-to tip that always advertises `allow-always`. Restricted plugin approvals can then show a command the server rejects, so the tip needs to use the request's allowed decisions too.
  - confidence: 0.82
