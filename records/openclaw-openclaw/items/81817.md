## Summary

Keep open: current main still writes skill download responses to staging without advertised or streamed size enforcement, and an open linked PR is the right fix path before this issue closes.

## What This Changes

The issue requests bounding skill `download` installer responses by advertised `Content-Length` and streamed bytes before writing staging files.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81818 with a narrow installer cap that rejects oversized advertised lengths, enforces the same 256 MiB cap while streaming, and keeps focused regression/docs coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock `installDownloadSpec` with `Content-Length: 268435457`, a small body, and `extract: false`; current main has no preflight or streaming byte cap before returning success.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the adjacent marketplace download pattern by rejecting oversized advertised lengths and counting streamed bytes before the staging file can exceed the cap.
