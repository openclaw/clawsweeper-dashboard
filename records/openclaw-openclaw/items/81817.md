## Summary

Keep open: current main still has the unbounded skill download response path, and the open linked PR should be reviewed or landed before this issue closes.

## What This Changes

The issue requests bounding skill `download` installer responses by advertised and streamed byte counts before remote bodies are written to staging.

## Best Possible Solution

Review and land the linked focused PR or an equivalent bounded download implementation, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock `installDownloadSpec` with a successful response advertising `Content-Length: 268435457` and `extract: false`; current main has no preflight or streaming byte cap before success.

## Solution Assessment

Yes. The narrow maintainable solution is to follow the adjacent marketplace pattern: reject oversized advertised lengths and count streamed bytes before the staging file can exceed the shared 256 MiB artifact cap.
