## Summary

Keep open: current main still streams skill download responses into `.openclaw-download-staging` without checking advertised or streamed size, and the linked open PR at https://github.com/openclaw/openclaw/pull/81818 is the active fix path rather than a reason to close early.

## What This Changes

Add a bounded 256 MiB response-size check for skill `download` installers before and during streaming.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81818, then close this issue after the fix is on main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock a successful `installDownloadSpec` download response with `Content-Length: 268435457`, `extract: false`, and a small body; current main has no preflight or streaming cap before success.

## Solution Assessment

Yes. The narrow maintainable solution is to add the same 256 MiB advertised-length and streamed-byte cap shape already used by the adjacent marketplace download path.
