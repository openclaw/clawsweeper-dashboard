## Summary

Keep open: current main still streams skill download installer responses to the staging file without an advertised-size or streamed-byte cap, and the linked closing PR is still open and unmerged.

## What This Changes

The issue asks OpenClaw to bound skill download installer response sizes before and during staging-file writes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81818 or an equivalent replacement, then close this issue after the bounded installer behavior reaches main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock installDownloadSpec with a successful download response advertising Content-Length: 268435457 and extract: false; current main has no preflight or streaming byte cap before success.

## Solution Assessment

Yes. The narrow maintainable fix is to reject oversized advertised lengths and enforce the same 256 MiB cap while streaming, matching the existing marketplace download pattern.
