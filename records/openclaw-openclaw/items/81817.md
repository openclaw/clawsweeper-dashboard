## Summary

Keep this open: current main still streams skill download response bodies into `.openclaw-download-staging` without a `Content-Length` preflight or streamed byte cap, latest release `v2026.6.10` is still affected, and the linked fixing PR remains open but conflicting on the old `src/agents` path.

## What This Changes

Skill download installers should reject oversized advertised or streamed remote responses before writing unbounded data to staging.

## Best Possible Solution

Port or replace the linked fix on `src/skills/lifecycle/install-download.ts`, add lifecycle tests for advertised oversized responses and streamed overflow, and keep docs clear about the 256 MiB installer boundary.

## Reproduction Assessment

Yes. Source inspection of current main shows the active lifecycle downloader pipes successful response bodies into staging without advertised-size or streamed-byte enforcement, matching the reported mocked reproduction shape.

## Solution Assessment

Yes on direction, but the fix is not landed yet. The maintainable fix belongs in the current lifecycle downloader and tests, not in the stale `src/agents` path used by the open PR.
