## Summary

Keep this open: current main and v2026.6.8 still stream skill download responses into staging without an advertised-size preflight or streamed-byte cap. The linked fixing PR remains open and targets the removed `src/agents` installer path, so maintainers should port, replace, or close that PR before this issue can be resolved.

## What This Changes

Add advertised and streamed response-size limits to skill `download` installers so oversized remote artifacts fail before filling local staging storage.

## Best Possible Solution

Port the advertised `Content-Length` preflight and streamed byte cap to `src/skills/lifecycle/install-download.ts`, add focused lifecycle regression coverage, update the skills docs, and land the linked or replacement PR.
