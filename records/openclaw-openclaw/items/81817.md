## Summary

Keep this open: current main and latest v2026.6.8 still pipeline skill download response bodies into staging without a `Content-Length` preflight or streamed byte cap, and the open linked fixing PR has not merged.

## What This Changes

The issue reports that skill download installers can stream oversized remote responses into staging before installer size limits are enforced.

## Best Possible Solution

Port or replace the linked fix so the current lifecycle downloader rejects oversized advertised responses and enforces the same byte cap while streaming, with focused regression coverage and docs for the limit.

## Reproduction Assessment

Yes. Source inspection of current main shows the live lifecycle downloader pipes successful response bodies into staging without checking `Content-Length` or counting streamed bytes, matching the oversized response reproduction described in the issue.

## Solution Assessment

Yes on direction, but not yet landed. The narrow repair is to add the cap in `src/skills/lifecycle/install-download.ts` and cover both advertised-size and streaming-overflow cases in the colocated lifecycle test.
