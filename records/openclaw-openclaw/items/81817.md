## Summary

Keep open: current `main` still streams skill download responses to staging without an advertised-size or streamed-byte cap, and the linked closing PR remains open and unmerged.

## What This Changes

Request to enforce the shared 256 MiB installer artifact limit for skill `download` responses before and during streaming.

## Best Possible Solution

Land or replace the linked focused PR with a bounded `installDownloadSpec`, regression coverage, and the docs note, then close this issue once the fix is on `main`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock `installDownloadSpec` with a successful response advertising `Content-Length: 268435457`, a small body, and `extract: false`; current `main` has no cap before returning success.

## Solution Assessment

Yes. The narrow maintainable solution is the linked approach: reject oversized advertised lengths and count streamed bytes before the staging file can exceed the shared 256 MiB artifact cap.
