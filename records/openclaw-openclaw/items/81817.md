## Summary

Keep open: current main and v2026.5.12 still pipe skill download responses to staging without an advertised or streamed size cap, and the linked closing PR is open but unmerged.

## What This Changes

Request to cap skill `download` installer responses by rejecting oversized `Content-Length` values and enforcing the same limit while streaming.

## Best Possible Solution

Review and land or replace the linked PR with a narrow 256 MiB advertised and streamed byte cap, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Mock `installDownloadSpec` with `Content-Length: 268435457`, a small successful body, and `extract: false`; current main has no preflight or streaming byte cap before returning success.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the existing marketplace download pattern with a 256 MiB `Content-Length` preflight and streamed byte counter.
