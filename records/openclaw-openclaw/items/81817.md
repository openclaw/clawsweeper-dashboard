## Summary

Keep open: current main still streams skill download bodies into staging without checking `Content-Length` or counting streamed bytes, and the open draft PR that references this issue should be reviewed or replaced before the issue closes.

## What This Changes

Cap skill `download` installer responses at the accepted artifact limit before and during streaming.

## Best Possible Solution

Land or replace the linked PR with a narrow `installDownloadSpec` cap that rejects oversized `Content-Length`, enforces the same 256 MiB limit while streaming, and records focused regression/docs coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock `installDownloadSpec` with a successful `download` response advertising `Content-Length: 268435457` and `extract: false`; current main has no preflight or streaming byte cap before returning success.

## Solution Assessment

Yes. The narrow maintainable fix is to copy the adjacent marketplace pattern: reject oversized advertised lengths and count bytes while streaming before the staging file can exceed the shared 256 MiB artifact cap.
