- `impact:security`: The issue concerns untrusted skill installer metadata causing unbounded remote-response writes before safety limits apply.

## Summary

Keep open: current main and v2026.5.18 still pipe skill download responses to staging without advertised or streamed byte caps, and the open closing PR is the right fix path.

## What This Changes

Skill download installers should reject advertised or streamed remote responses over the accepted 256 MiB artifact limit before writing them to staging.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81818, then close this issue once the bounded download installer fix is on main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock `installDownloadSpec` with `Content-Length: 268435457`, a small body, and `extract: false`; current main has no size preflight or streaming cap before success.

## Solution Assessment

Yes. The narrow maintainable fix is to follow the existing marketplace pattern by rejecting oversized advertised lengths and counting streamed bytes before the staging file can exceed the shared 256 MiB cap.
