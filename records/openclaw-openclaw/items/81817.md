## Summary

Keep this open: current main still streams skill download responses into staging without a `Content-Length` preflight or streaming byte limit, and the linked fix PR is still open but conflicts with the refactored current installer path.

## What This Changes

The issue asks OpenClaw to bound remote response sizes for skill download installers before and during staging writes.

## Best Possible Solution

Port the advertised `Content-Length` preflight and streaming byte cap into the current `src/skills/lifecycle` installer path, update the colocated regression test and skills docs, then close this issue when the linked or replacement PR lands.

## Reproduction Assessment

Yes. Current `main` still pipelines the successful remote response body directly to a staging file in `src/skills/lifecycle/install-download.ts`, and the issue provides a focused oversized `Content-Length` mock case for the missing guard.

## Solution Assessment

Yes, the requested fail-closed size cap is the right repair direction, but it must be applied to the current lifecycle installer path rather than the stale `src/agents` path in the open PR.
