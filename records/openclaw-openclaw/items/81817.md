Keep this open: current main and v2026.6.10 still stream skill download response bodies into staging without a Content-Length preflight or streamed byte cap, and the linked fixing PR is still open and conflicting.

## What This Changes

The issue requests fail-closed advertised-size and streamed-byte limits for skill download installers before remote bodies are written to staging.

## Best Possible Solution

Port or replace the linked fix on the current src/skills/lifecycle downloader, add advertised-size and streaming-overflow regression coverage, and keep docs clear about the 256 MiB boundary.
