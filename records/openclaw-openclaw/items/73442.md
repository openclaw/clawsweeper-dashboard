## Summary

Keep open: this protected maintainer draft is not implemented on current main, and the current PR head still leaves a security-sensitive stat race in the new filesystem boundary helper.

## What This Changes

This PR adds root-scoped fs-safe stat, directory-list, and rename helpers with pinned path-helper support, tests, plugin SDK baseline hashes, a changelog entry, and a small doctor type cleanup.

## Best Possible Solution

Keep the PR open and revise statPathWithinRoot to stat through a pinned parent/dirfd path or revalidate the parent canonical path after lstat while preserving only the final symlink, then add a stat rebind regression test and rerun the fs-safe and changed-gate checks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to validate root/slot/file.txt, replace slot with a symlink to an outside directory before fs.lstat runs, and call statPathWithinRoot with followSymlinks:false; this read-only review did not execute it.

## Solution Assessment

No. The PR direction is useful, and the earlier static symlink-parent issue was partially addressed, but the stat helper still needs a pinned or post-stat parent-canonicalization step to be a safe boundary primitive.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pin or revalidate stat parents before lstat:** `src/infra/fs-safe.ts:367-368`
  - body: With followSymlinks:false, the helper validates the path first and then calls fs.lstat on the lexical canonical path. If an in-root parent is replaced by a symlink to an outside directory between those steps, lstat follows that parent, while realPath is still set to the in-root string, so callers can receive metadata for an out-of-root file. Use a pinned parent/dirfd stat path or revalidate the parent after lstat while preserving only the final symlink.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds a filesystem-boundary helper with a remaining parent-rebind escape in the non-follow stat path.

Concerns:

- **[medium] Stat helper can misclassify out-of-root metadata:** `src/infra/fs-safe.ts:367`
  - body: The new followSymlinks:false stat path calls fs.lstat after preflight validation and then treats the lexical canonical path as realPath, so a parent directory replaced by a symlink after validation can point the stat outside the root without failing the final root check.
  - confidence: 0.87
