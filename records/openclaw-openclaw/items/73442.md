## Summary

Keep open: this PR is draft, maintainer-labeled, member-authored, and the latest head still has a P1 filesystem-boundary issue in the new stat helper; current main does not already contain this helper surface.

## What This Changes

The branch adds root-scoped fs-safe stat, directory-list, and same-root rename helpers, tests, generated SDK baseline hashes, and a changelog entry.

## Best Possible Solution

Revise `statPathWithinRoot` to stat through a pinned parent/dirfd path or revalidate the parent after `lstat`, then add a parent-rebind regression test and rerun the focused test/type gates before maintainer/security review.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to validate an in-root `slot/file.txt`, swap `slot` to a symlink targeting an outside directory before `fs.lstat`, and call `statPathWithinRoot` with `followSymlinks:false`; this read-only review did not execute it.

## Solution Assessment

No. The helper direction is useful, but the current stat implementation is not the narrowest safe boundary primitive until the stat operation is pinned to the parent or the parent is revalidated after `lstat`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pin or revalidate stat parents before lstat:** `src/infra/fs-safe.ts:367-368`
  - body: With `followSymlinks:false`, this validates the path and then calls `fs.lstat` on `resolved.canonicalPath`. If an in-root parent is replaced by a symlink to an outside directory between those steps, `lstat` follows that parent while `realPath` remains the in-root string, so callers can receive metadata for an out-of-root file. Use a pinned parent/dirfd stat path or revalidate the parent after `lstat` while preserving only the final symlink.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds filesystem-boundary helpers, and the new stat path still has a parent-rebind escape.

Concerns:

- **[medium] Stat helper can misclassify out-of-root metadata:** `src/infra/fs-safe.ts:367`
  - body: The `followSymlinks:false` branch calls `fs.lstat` after preflight validation and then treats the lexical canonical path as `realPath`, so a parent directory replaced by a symlink after validation can point the stat outside the root without failing the final root check.
  - confidence: 0.87
