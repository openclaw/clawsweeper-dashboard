## Summary

Keep open: this is a draft, maintainer-labeled filesystem-boundary PR, and the latest head fixes the earlier no-overwrite clobber and missing stat-field issues but still has a followed-stat symlink race before merge.

## What This Changes

The PR adds root-scoped `fs-safe` stat, directory-list, and same-root rename helpers, extends the pinned Python helper, adds `fs-safe` coverage, updates generated SDK baseline hashes, and adds a changelog entry.

## Best Possible Solution

Keep the bounded-helper direction, but pin or revalidate followed stat targets at use time, make the synthetic `Stats` predicates truthful, then rerun the focused fs-safe tests/type gates before maintainer/security review.

## Reproduction Assessment

Yes, at source level. Mock `runPinnedPathHelper` for `operation: "stat"` to replace the final in-root file with a symlink to an outside file after `fs.realpath` and before the helper runs; this read-only review did not execute tests.

## Solution Assessment

No. The pinned helper approach is the right direction and the latest no-overwrite rename fix is materially better, but followed stat targets still need use-time pinning/revalidation and the returned `Stats` shape should match Node semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Pin followed stat targets at use time:** `src/infra/fs-safe.ts:474-479`
  - body: `statPathWithinRoot` verifies `realPath` before spawning the pinned helper, but the helper later calls `os.stat(..., follow_symlinks=True)` on the final basename. If that basename is swapped for a symlink to an outside file between those steps, the returned `stat` is for the outside target while the final inside check still uses the stale in-root `realPath`.
  - confidence: 0.84
- **[P3] Compute all stat predicates from mode:** `src/infra/fs-safe.ts:162-165`
  - body: `statsLikeFromPinnedPayload` advertises a `Stats` object but hard-codes `isBlockDevice`, `isCharacterDevice`, `isFIFO`, and `isSocket` to false. For those file types, callers get a `Stats` shape that disagrees with `mode`, unlike Node's real `fs.stat`/`lstat` result.
  - confidence: 0.75

## Security Review

Status: needs_attention

Summary: Needs attention: the latest head fixes the earlier rename clobber concern, but followed stat can still return outside-root metadata after a raced final symlink swap.

Concerns:

- **[medium] Followed stat can escape through a raced final symlink:** `src/infra/fs-safe.ts:478`
  - body: The JS path bounds `realPath` before spawning the pinned helper, then the helper follows the final basename with `os.stat(..., follow_symlinks=True)`. A workspace-controlled final component swapped to an outside symlink after the precheck can leak outside-file metadata through the new stat helper.
  - confidence: 0.84
