## Summary

Keep open. This is a protected maintainer-labeled draft by a MEMBER, current main does not already include the requested helpers, and the live PR head still has a blocking filesystem-boundary issue in the new stat helper.

## What This Changes

This draft PR adds root-scoped stat, directory-listing, and same-root rename helpers to `fs-safe`/the pinned path helper, with colocated `fs-safe` tests and an unreleased changelog entry.

## Best Possible Solution

Fix `statPathWithinRoot` to reuse the existing boundary canonicalization path or equivalent parent-symlink resolution while preserving only the final symlink when explicitly requested, add a regression test for an in-root symlinked parent pointing outside the root, then land only after maintainer/security review and changed-gate proof.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `root/link/file.txt` where `link` is a symlink to an outside directory, then calling `statPathWithinRoot({ rootDir: root, relativePath: "link/file.txt", followSymlinks: false })`; this read-only review did not execute it.

## Solution Assessment

No. The PR direction is useful, but the current implementation is not the safest narrow solution because the new stat helper bypasses the canonical boundary traversal used by the safer pinned helper path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Canonicalize symlink parents when statting paths:** `src/infra/fs-safe.ts:371`
  - body: With `followSymlinks: false`, `fs.lstat` avoids following only the final path component; symlinks in parent directories are still traversed. Because this branch then treats the lexical `resolved` path as `realPath`, `root/link/file.txt` where `link` points outside the root can pass the root check while returning metadata for an out-of-root target. Use the boundary resolver or canonicalize parents while preserving only the final symlink.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff has no workflow, dependency, lockfile, or release-script changes, but it introduces a filesystem-boundary helper with a symlink-parent escape in the `followSymlinks: false` stat path.

Concerns:
