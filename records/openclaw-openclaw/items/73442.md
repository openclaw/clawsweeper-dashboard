## Summary

Keep open: this draft MEMBER PR has the protected `maintainer` label, and the latest head still leaves source-level filesystem-boundary gaps in the new Windows stat, directory-listing, and rename paths.

## What This Changes

The draft PR adds root-scoped `fs-safe` stat, readdir, and rename helpers, extends the pinned Python path helper, adds fs-safe tests, refreshes plugin SDK API hashes, and adds a changelog entry.

## Best Possible Solution

Keep the pinned-helper design, but make stat, readdir, and rename equivalently bounded on Windows or fail closed there, then cover the platform fallback expectations before rerunning the fs-safe and type gates.

## Reproduction Assessment

Yes, at source level. On Windows, after pre-resolution returns, a raced in-root parent junction or symlink can redirect the subsequent `fs.stat`, `fs.readdir`, or `fs.rename` path-string fallback while the code still relies on stale resolved paths.

## Solution Assessment

No. The primary pinned-helper path and the latest fail-closed helper-spawn handling are the right direction, but the Windows branches must not remain less bounded than the helper API being added.

## Review Findings

  - body: The Windows branch stats `resolved.canonicalPath` after a pre-resolution pass. If an in-root parent is swapped to an outside junction or symlink between those steps, this can return outside metadata while the later root check still uses the stale `realPath`; keep this handle-pinned/revalidated or fail closed on Windows.
  - confidence: 0.82
- **[P2] Keep Windows directory listing pinned:** `src/infra/fs-safe.ts:550-551`
  - body: `readdirWithinRoot` falls back to `readdirWithinRootLegacy` on Windows after resolving the directory. A raced parent junction or symlink can redirect the `fs.readdir` path string outside the root, so this branch needs equivalent pinning or a fail-closed behavior.
  - confidence: 0.84
- **[P2] Avoid Windows path-string rename fallback:** `src/infra/fs-safe.ts:941-942`
  - body: The Windows rename path resolves `from` and `to`, then calls the legacy `fs.rename` helper. A raced source or destination parent can redirect that path-string rename outside the root; keep rename bounded on Windows or fail closed instead.
  - confidence: 0.83
