## Summary

Keep open for maintainer review: current main and the latest release still use the raw reachable-target count for `multiple_gateways`, while this PR applies a bounded identity-plus-port dedupe with regression coverage and detailed live SSH-tunnel proof. I found no blocking correctness or security findings.

## What This Changes

The PR updates gateway probe/status warnings to compare reachable targets by gateway self identity plus gateway port, adds regression tests, and aligns gateway docs with identity-based `multiple_gateways` semantics.

## Best Possible Solution

Land the bounded identity-based warning fix after maintainer review and normal CI, keeping the conservative warnings for unknown, distinct, or different-port reachable gateways.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.5.22` emit `multiple_gateways` whenever `reachable.length > 1`, and the probe path can include both `sshTunnel` and `configRemote` when their URLs differ. I did not rerun the live CLI scenario in this read-only review.

## Solution Assessment

Yes, this is the narrow maintainable fix: compare the gateway self identity plus logical gateway port, while preserving warnings for unknown identities, distinct identities, and same-host different-port setups.
