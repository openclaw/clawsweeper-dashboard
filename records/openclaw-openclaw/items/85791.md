## What This Changes

The PR changes gateway probe warning logic, tests, docs, and changelog text so duplicate reachable transports with the same gateway identity and logical port do not emit `multiple_gateways`.

## Best Possible Solution

Land the identity-aware gateway warning fix after maintainer review and current-head validation, then let the linked gateway issue close through the merge.

## Reproduction Assessment

## Solution Assessment

Yes; comparing gateway self identity plus logical gateway port in warning construction is narrower than skipping the configured remote probe and still preserves warnings for distinct, unknown, or different-port gateways.
