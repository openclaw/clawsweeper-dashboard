## What This Changes

This PR changes gateway probe/status warning generation to dedupe reachable targets by gateway self identity, adds a process-level self `instanceId`, updates gateway docs, and adds regression tests.

## Best Possible Solution

Land the PR after maintainer acceptance of the conservative identity policy and current checks stay green.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer policy acceptance. Dedupe at warning construction preserves probing all transports, and using existing self presence/protocol `instanceId` is narrower than skipping a direct-path check.
