## Summary

Current main still emits `multiple_gateways` solely from multiple reachable targets, while this PR changes the behavior to identity-based dedupe with conservative fallback, docs, tests, and real CLI proof. I found no blocking review findings; the protected `maintainer` label leaves the merge decision with maintainers.

## What This Changes

The PR updates gateway probe/status warning logic to suppress `multiple_gateways` for multiple transports sharing a provable gateway identity, adds self `instanceId` presence metadata, and updates gateway tests and docs.

## Best Possible Solution

Review and land this PR if maintainers agree with the identity-based warning semantics, preserving the conservative warning for unknown identities.

## Reproduction Assessment

Yes. Current main's source path warns on `reachable.length > 1`, and the linked report plus PR proof describe the SSH-tunnel-plus-remote-URL path; I did not rerun the live setup during this read-only review.

## Solution Assessment

Yes. Identity-based dedupe in `buildGatewayStatusWarnings` is the narrowest maintainable fix I saw, because it suppresses only provably same-gateway transports while keeping unknown identity cases conservative.
