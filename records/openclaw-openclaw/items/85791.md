The PR changes gateway probe warning construction so duplicate reachable transports with the same gateway self identity and logical port no longer emit `multiple_gateways`, with regression tests, docs, and a changelog entry.

## Best Possible Solution

Yes, the proposed fix is the narrow maintainable path: compare gateway self identity plus logical port at warning construction time while preserving warnings for unknown, distinct, and different-port gateways.
