## Summary

Keep open; the branch fixed the earlier legacy config migration concern, but the current PR head still has a P2 behavioral regression where active heartbeats can be steered before the drop policy runs.

## What This Changes

The PR makes non-interrupt mid-turn prompts steer active runs by default, removes steering-specific public queue modes, batches Codex app-server steering, and updates docs, config metadata, changelog, and tests.

## Best Possible Solution

Keep the default-steering simplification and legacy config migration, but guard heartbeats so active heartbeat runs still drop instead of being injected into a streaming user run.

## Reproduction Assessment

Yes for the review finding from source inspection: on PR head, an active streaming session with `opts.isHeartbeat` and a non-interrupt queue mode sets `shouldSteer` true, and `runReplyAgent` attempts steering before resolving the heartbeat drop action. I did not execute the PR branch.

## Solution Assessment

No: the main direction is maintainable and the migration is now present, but the implementation needs a heartbeat guard or ordering change so scheduled heartbeats keep the existing drop behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve heartbeat drops before steering:** `src/auto-reply/reply/get-reply-run.ts:887`

Add a heartbeat guard before steering or run the drop decision first.
