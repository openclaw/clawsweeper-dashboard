## Summary

Keep this PR open. The underlying Slack block-action/thread routing bug is real on current main, but the draft branch currently changes ordinary top-level Slack DM session routing in a way that contradicts current tests/docs and should be narrowed before merge.

## What This Changes

This PR routes Slack block actions through thread-aware Slack session handling, adds interactive message owner caching and wake behavior, adjusts threaded reply delivery/session repair behavior, and updates Slack/agent tests plus the changelog.

## Best Possible Solution

Keep the Slack-owned interaction fix, but preserve top-level DM session continuity and route button clicks with explicit interaction owner/thread metadata before final Slack maintainer review and validation.

## Reproduction Assessment

Yes. Current main source shows block-action events carrying threadTs in the payload but not into session resolution, and PR-head source shows the introduced top-level DM session-key change against an unchanged integration test.

## Solution Assessment

No. The Slack plugin is the right boundary, but the current draft is not the best final fix because it broadens the routing change to ordinary top-level DMs instead of limiting the repair to block-action ownership/thread routing.
