## Summary

Keep open. This draft targets a real Slack block-action thread-routing bug, but the current branch still regresses ordinary top-level Slack DM session continuity and overlaps a session-repair fix already present on current main.

## What This Changes

The PR routes Slack block-action interactions through thread-aware Slack session handling, records interactive message ownership for tool-sent replies, adjusts threaded reply/session repair behavior, and adds Slack/agent tests plus changelog entries.

## Best Possible Solution

Keep the Slack plugin-owned fix direction, but narrow it to block-action owner/thread routing and wake behavior while preserving stable top-level DM sessions and rebasing away already-landed session-repair changes.

## Reproduction Assessment

Yes. Source inspection on current main shows block-action `threadTs` is parsed but not passed into system-event session resolution, and the PR diff shows a separate top-level DM routing regression against existing stable-session coverage.

## Solution Assessment

No. The Slack plugin is the right boundary, but this draft broadens the fix to ordinary top-level DMs instead of limiting the repair to explicit interaction owner/thread metadata and wake routing.
