## What This Changes

The PR threads Slack `threadTs` through system-event session-key resolution and block-action enqueueing, with regression coverage for thread-suffixed Slack interaction sessions.

## Solution Assessment

Yes, with a changelog addition. The proposed path is the narrowest maintainable fix because it stays inside the Slack plugin, reuses the existing SDK thread-session helper, and aligns system-event routing with inbound Slack thread replies.
