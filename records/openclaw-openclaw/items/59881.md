## Summary

Keep this open. Current main still constructs agent plugin hook contexts with `ctx.channelId` from `messageChannel ?? messageProvider`, while the Discord queued-run path passes the provider surface as `messageChannel` and carries the actual conversation target separately, so the reported per-channel plugin isolation bug remains plausible and narrow.

## What This Changes

Ensure plugin hook contexts expose the real conversation/channel identifier in `ctx.channelId` and the provider/surface name in `ctx.messageProvider`.

## Best Possible Solution

Add a shared hook-context derivation helper that preserves `ctx.messageProvider` as the provider/surface and derives `ctx.channelId` from trusted conversation metadata or the parsed base session key, then use it consistently across agent hook contexts.

## Reproduction Assessment

Yes. A queued Discord run with `SessionKey: agent:main:discord:channel:c1`, `OriginatingChannel: discord`, and `OriginatingTo: channel:c1` reaches `runEmbeddedPiAgent` with `messageChannel: discord`, and current hook construction exposes `ctx.channelId` as `discord`.

## Solution Assessment

Yes. Fixing the shared hook-context construction is the narrowest maintainable solution; plugin-local parsing workarounds would leave the SDK contract misleading for every hook consumer.

## Repair Work Prompt

Fix the plugin hook context bug reported at https://github.com/openclaw/openclaw/issues/59881. Current main exposes `ctx.channelId` as the provider/surface for queued Discord channel runs because hook contexts use `params.messageChannel ?? params.messageProvider`, while the true conversation target is available in `messageTo`, `currentChannelId`, `OriginatingTo`, or the session key. Add one shared derivation helper for agent hook contexts that preserves `ctx.messageProvider` as the provider/surface and derives `ctx.channelId` as the real conversation/channel id, falling back to `parseRawSessionConversationRef` on the base session key when trusted run metadata is missing. Apply it consistently to `before_prompt_build`, `agent_end`, `llm_input`, `llm_output`, CLI runner hook contexts, and the harness helper if needed. Cover a Discord channel-session case such as `agent:main:discord:channel:c1` with `messageChannel/originatingChannel: discord` and `messageTo/originatingTo: channel:c1`. Do not change the Hindsight plugin, do not add core special-cases for one plugin id, and keep core extension-agnostic. Add a user-facing `CHANGELOG.md` fix entry crediting `Thanks @bradfreels`.
