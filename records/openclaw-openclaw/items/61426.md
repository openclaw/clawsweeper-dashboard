## Summary

Keep open. Current main still allows default isolated `agentTurn` cron deliveries to resolve as implicit `announce`/`last` deliveries and then queues main-session awareness after success, so the reported accumulation path remains valid; the related fix PR #61658 was closed unmerged.

## What This Changes

Cron jobs with `sessionTarget: "isolated"` should not let implicit/default delivery output accumulate in the main conversation session.

## Best Possible Solution

Gate main-session cron awareness on explicitly resolved delivery targets, keep explicit delivery awareness intact, and add regression tests for implicit `last`/default announce delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: omit delivery on an isolated `agentTurn`, let normalization create announce delivery, resolve it through `last` as an implicit target, then observe `dispatchCronDelivery()` queue awareness because the gate ignores target mode.

## Solution Assessment

Yes. The narrow explicit-target gate is the best fix because it removes only the implicit/default main-session mirroring while preserving the intentional explicit awareness path from #52136.
