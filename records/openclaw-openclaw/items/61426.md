## Summary

Keep open. Current main still appears to reproduce the reported path: default isolated `agentTurn` cron jobs become implicit `announce` deliveries, and `dispatchCronDelivery()` still queues a main-session awareness event after every successful isolated delivery without checking whether the delivery target was implicit. The linked fix PR #61658 proposed the missing explicit-target gate and regression tests, but it is closed unmerged.

## What This Changes

Cron jobs with `sessionTarget: "isolated"` should not let default implicit delivery mirror their output into the main session and grow that session's context.

## Best Possible Solution

Apply the narrow explicit-target awareness gate from #61658 or an equivalent implementation: only queue main-session cron awareness when the resolved delivery target is explicit, while preserving explicit delivery behavior and adding regression tests for implicit `last` delivery and pinned-channel implicit recipient cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an isolated `agentTurn` without explicit delivery defaults to announce, resolves through `last` as an implicit target, then successful direct delivery still calls `queueCronAwarenessSystemEvent()` for the main session.

## Solution Assessment

Yes. The explicit-only awareness gate is the narrowest maintainable solution because it stops default isolated cron output from entering the main session without removing the intentional awareness path for explicit external targets.
