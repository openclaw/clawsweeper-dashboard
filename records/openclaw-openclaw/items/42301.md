## Summary

Keep this PR open: it targets a real Feishu media-key regression that is still present on current main, but the current patch still weakens fail-closed validation for the literal `..` token and lacks after-fix real Feishu proof.

## What This Changes

The PR removes Feishu external-key validation's broad `..` substring rejection while preserving slash/backslash rejection and adds regression tests for embedded-dot-dot media keys.

## Best Possible Solution

Land a minimal Feishu plugin validator change that allows embedded `..` opaque keys while rejecting exact `.`/`..`, path separators, empty/control/oversized values, with focused tests and redacted real Feishu image-analysis proof.

## Reproduction Assessment

Yes, by source inspection: current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and inbound image/file keys flow through that helper before media download. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The owner-module fix is the right narrow direction, but this patch should also reject exact dot-segment tokens and include real after-fix Feishu evidence before merge.

Overall correctness: patch is incorrect
