## Summary

Keep open: this PR remains the canonical fix path for the linked Feishu image-analysis regression, but the latest head still accepts the literal `..` key and lacks real Feishu after-fix proof.

## What This Changes

The PR removes the broad `..` substring rejection from Feishu external-key normalization while preserving slash/backslash rejection and adds regression tests for embedded dot-dot keys.

## Best Possible Solution

Amend the Feishu validator to allow embedded `..` in opaque keys while still rejecting exact `..`, path separators, empty/control/oversized values, then back it with focused tests and real Feishu proof.

## Reproduction Assessment

Yes, by source inspection. Current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and Feishu image/file keys flow through that helper before download; I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The owner-module change is the right narrow direction, but the current patch should still reject the exact `..` token and include real after-fix Feishu evidence before merge.
