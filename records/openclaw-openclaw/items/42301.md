## Summary

Keep this PR open: it targets a valid Feishu media regression and the latest diff preserves slash/backslash rejection, but it still weakens validation by accepting the literal `..` key and lacks the required changelog entry.

## What This Changes

The PR changes Feishu external-key validation to allow embedded `..` substrings and adds validator tests for embedded-dot-dot keys plus rejected path separators.

## Best Possible Solution

Amend the Feishu plugin patch to preserve separator and control-character rejection, allow embedded `..` in otherwise opaque keys, explicitly reject the exact `..` token, cover those cases in tests, and add a single-line Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main returns `undefined` for embedded-dot-dot keys such as `img_v2_test..key`, and the PR diff would make the literal `..` key pass validation.

## Solution Assessment

No. The owner-module, minimal validation change is the right direction, but the current patch should also reject exact `..` and include the required changelog entry before merge.
