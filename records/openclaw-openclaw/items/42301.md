## Summary

Keep open: the PR addresses a valid Feishu image-analysis bug that is not fixed on current main, but the current head still needs a small validator hardening change and real Feishu behavior proof before merge.

## What This Changes

The PR removes the Feishu external-key validator’s broad `..` substring rejection while preserving slash/backslash rejection and adds regression tests for embedded-dot-dot keys.

## Best Possible Solution

Land a minimal Feishu plugin validator update that allows embedded `..` opaque keys while rejecting literal dot-segment tokens and path separators, with focused tests and redacted Feishu proof.

## Reproduction Assessment

Yes, by source inspection: current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and inbound parsing/download paths call that helper. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No. The owner-module fix is the right narrow direction, but this patch should still reject standalone `..` and include real after-fix Feishu evidence before merge.
