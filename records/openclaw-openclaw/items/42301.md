## Summary

Keep open: the PR targets a real Feishu image-analysis regression that is still present on current main, but the latest branch accepts the literal `..` external key and still lacks real after-fix Feishu proof.

## What This Changes

The PR removes Feishu external-key validation's broad `..` substring rejection while preserving slash/backslash rejection and adds regression tests for embedded-dot-dot keys.

## Best Possible Solution

Amend the Feishu validator to allow embedded `..` in opaque keys while still rejecting exact `..`, path separators, empty/control/oversized values, then land with focused tests and redacted real Feishu proof.

## Reproduction Assessment

Yes, by source inspection. Current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and Feishu image/file keys flow through that helper before download; I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The owner-module change is the right narrow direction, but the current patch should still reject the exact `..` token and include real after-fix Feishu evidence before merge.
