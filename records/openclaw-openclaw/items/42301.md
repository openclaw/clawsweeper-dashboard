## Summary

Keep open. Current main still rejects legitimate Feishu media keys containing embedded `..`, so this PR targets a real bundled Feishu regression linked to #42257, but the latest patch still weakens fail-closed validation by accepting the exact `..` token and also lacks the required changelog entry.

## What This Changes

The PR removes the blanket `..` rejection from Feishu external-key normalization while keeping slash/backslash rejection and adds tests for embedded dot-dot keys and path-separator rejection.

## Best Possible Solution

Amend this Feishu plugin PR to keep empty, length, control-character, slash, and backslash rejection; allow embedded `..` inside otherwise opaque keys; explicitly reject the exact `..` token; add regression coverage for both accepted embedded keys and rejected dot-token/path-separator cases; and include the required changelog entry.

## Reproduction Assessment

Yes. On current main, `normalizeFeishuExternalKey("img_v2_test..key")` returns `undefined`, which drops the media key before image analysis; from the PR diff, `normalizeFeishuExternalKey("..")` would now return `".."` and flow into Feishu media path parameters.

## Solution Assessment

No. The owner module and minimal removal of the broad substring check are the right direction, but the final patch should preserve fail-closed behavior for exact dot-dot tokens and add the missing user-facing changelog entry.
