## Summary

Keep open: current main still rejects legitimate Feishu external keys containing embedded `..`, and this PR targets the right owner module, but the head now accepts the literal `..` key and lacks real Feishu image-analysis proof.

## What This Changes

The PR removes the Feishu external-key validator’s broad `..` substring rejection while keeping slash/backslash rejection and adds regression tests for embedded-dot-dot keys.

## Best Possible Solution

Land a minimal Feishu plugin validator update that allows embedded `..` opaque keys while rejecting literal `..` and path separators, with focused tests and redacted real Feishu proof.

## Reproduction Assessment

Yes, by source inspection: current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and inbound parsing/download paths call that helper. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No. The owner-module fix is the right narrow direction, but this patch should still reject standalone `..` and include real after-fix Feishu evidence before merge.

## Review Findings

Overall correctness: patch is incorrect
