## Summary

Keep this PR open: current main still rejects embedded `..` Feishu media keys, so the central bug fix is useful, but the branch still needs a small validation hardening change and real Feishu runtime proof before merge.

## What This Changes

The PR removes Feishu external-key rejection for embedded `..` substrings while keeping slash/backslash rejection and adds focused validator regression tests.

## Best Possible Solution

Amend the Feishu validator to allow embedded `..` in opaque media keys while still rejecting separators, empty/control/oversized values, and exact `..`, then back it with focused tests and real Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `img_v2_test..key` is rejected by `normalized.includes("..")`, and the same helper gates Feishu image/file keys before download.

## Solution Assessment

No. The minimal owner-module fix is the right direction, but the branch should also reject the literal `..` token and provide real after-fix Feishu evidence before merge.
