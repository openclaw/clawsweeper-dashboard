## Summary

Keep this PR open: current main still rejects legitimate embedded-dot-dot Feishu media keys, and this PR remains the canonical focused fix path, but the current patch still needs a small fail-closed validator hardening change plus real Feishu behavior proof before merge.

## What This Changes

The PR removes Feishu external-key validation's broad embedded-dot-dot rejection while preserving path-separator rejection and adding regression tests for embedded `..` keys.

## Best Possible Solution

Amend the Feishu validator to allow embedded `..` in opaque keys while rejecting exact `..`, path separators, empty/control/oversized values, then land with focused tests and redacted real Feishu proof.

## Reproduction Assessment

Yes, by source inspection: current main rejects any Feishu external key containing `..`, and inbound image/file keys flow through that helper before download. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The owner-module change is the right narrow direction, but the current patch should explicitly reject the exact `..` token and include real after-fix Feishu evidence before merge.
