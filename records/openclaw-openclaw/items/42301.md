## Summary

Keep open: current main still rejects embedded-dot-dot Feishu media keys, so the PR is useful, but the PR head still accepts the literal `..` token and lacks real Feishu image-analysis proof.

## What This Changes

The PR removes Feishu external-key validation's broad `..` substring rejection while preserving slash/backslash rejection and adds regression tests for embedded-dot-dot media keys.

## Best Possible Solution

Land a minimal Feishu plugin validator change that allows embedded `..` opaque keys while rejecting literal `..`, path separators, empty/control/oversized values, with focused tests and redacted real Feishu image-analysis proof.

## Reproduction Assessment

Yes, by source inspection: current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and inbound parsing/download paths call that helper. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No. The owner-module fix is the right narrow direction, but this patch should still reject standalone `..` and include real after-fix Feishu evidence before merge.
