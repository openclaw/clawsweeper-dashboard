## Summary

Keep this PR open: it addresses a real Feishu media regression and has already incorporated the earlier slash/backslash security feedback, but the current patch still accepts the standalone `..` key and is missing the required user-facing changelog entry.

## What This Changes

The PR changes Feishu external-key validation to allow embedded `..` substrings while preserving slash/backslash rejection, and adds validator regression tests for those cases.

## Best Possible Solution

Amend the Feishu plugin validator to accept embedded `..` in opaque media keys while still rejecting path separators, control characters, empty/oversized values, and exact `.`/`..`, then cover those cases in tests and add one Unreleased Fixes changelog line.

## Reproduction Assessment

Yes. Source inspection on current main shows embedded-dot-dot keys are rejected by `normalized.includes("..")`, and the same helper is used before Feishu media keys are routed to downloads.

## Solution Assessment

No. The owner-module, minimal validation change is the right direction, but the current patch should also reject exact dot-segment tokens and include the required changelog entry before merge.
