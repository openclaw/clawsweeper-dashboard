## Summary

Keep open: current main still rejects Feishu media keys with embedded `..`, so this PR remains the canonical fix path for the linked Feishu image-analysis regression, but the branch is not merge-ready because it accepts the exact `..` token and lacks real after-fix Feishu proof.

## What This Changes

The PR relaxes Feishu external-key validation to allow embedded `..` substrings while preserving slash and backslash rejection, and adds regression tests for those key shapes.

## Best Possible Solution

Amend the Feishu validator to allow embedded `..` in opaque keys while still rejecting exact `..`, path separators, empty/control/oversized values, then land with focused tests and redacted real Feishu proof.

## Reproduction Assessment

Yes, by source inspection. Current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey`, and Feishu image/file keys flow through that helper before download; I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The owner-plugin change is the right narrow direction, but the current patch should still reject the exact `..` token and include real after-fix Feishu evidence before merge.
