## Summary

Keep open: current main still rejects embedded-dot-dot Feishu media keys, and this PR is the canonical focused fix path, but the current patch still weakens fail-closed validation for the literal `..` token and lacks real Feishu behavior proof.

## What This Changes

The PR removes the broad `..` substring rejection in the Feishu external-key validator while preserving slash/backslash rejection and adding regression tests for embedded-dot-dot keys.

## Best Possible Solution

Land a minimal Feishu validator update that allows embedded `..` opaque keys while rejecting path separators and exact dot-segment tokens, with focused tests and redacted Feishu image-analysis proof.

## Reproduction Assessment

Yes. By source inspection, current main rejects embedded-dot-dot Feishu keys in `normalizeFeishuExternalKey()`, and inbound parsing/download paths call that helper before image/resource handling.

## Solution Assessment

No. The owner-module fix is the right narrow direction, but the patch should still reject the standalone `..` token and include real after-fix Feishu evidence before merge.
