## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile token material into the CLI auth epoch, so token rotation or a transient token-shaped Claude parse can still reset same-version sessions; PR #74493 is open, unmerged, and directly targets this issue.

## What This Changes

Make Claude CLI auth-epoch hashing stable across OAuth token rotation so active conversations are not reset mid-session.

## Best Possible Solution

Review and land PR #74493, or an equivalent narrow patch, so Claude token-shaped epochs and token auth-profile epochs are identity-based, the auth-epoch version is bumped, and regression tests cover token rotation stability plus real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to stub `readClaudeCliCredentialsCached` to return a Claude CLI credential with `type: "token"`, change only `credential.token`, and compare `resolveCliAuthEpoch`; current main changes the epoch, and same-version session reuse then resets with `reason=auth-epoch`.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer, paired with an auth-epoch version bump, is the narrow maintainable fix for this reset loop; parser hardening can remain a defense-in-depth follow-up.
