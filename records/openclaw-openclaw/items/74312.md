## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, and #74493 is an open unmerged closing PR for the same focused fix.

## What This Changes

The issue requests stabilizing Claude CLI auth-epoch hashing so OAuth token refreshes and partial keychain reads do not reset live sessions.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, with identity-only token hashing, an auth-epoch version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. The source-level path is high-confidence: missing or empty `refreshToken` can produce a token credential, token credentials are hashed with `credential.token`, and same-version auth-epoch changes invalidate reusable CLI sessions; no live macOS keychain run was attempted in this read-only review.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer with a version bump is the narrowest maintainable fix for the reset loop; parser hardening can remain a follow-up unless review finds separate persisted-profile correctness issues.
