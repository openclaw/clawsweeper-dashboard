## Summary

Keep open. Current main still hashes Claude/token-shaped credentials by the rotating token value, and an open closing PR (#74493) already targets the issue, so this issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Claude CLI OAuth token rotation can transiently parse as token credentials and flip the CLI auth epoch, resetting live sessions mid-conversation.

## Best Possible Solution

Land #74493, or an equivalent narrow replacement, so Claude/token-shaped local and auth-profile epochs are identity-only, the epoch version is bumped, and regression tests plus changelog coverage ship together.

## Reproduction Assessment

Yes. The source-level trigger is reproducible on current main because missing `refreshToken` falls through to a token credential, token credentials include the rotating token in the epoch hash, and the current test explicitly expects token rotation to flip the epoch.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix: make token-shaped Claude/auth-profile epoch hashing identity-only and bump the epoch version, while leaving parser hardening as optional follow-up unless review finds a separate need.
