## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, and the open closing PR at https://github.com/openclaw/openclaw/pull/74493 is the focused implementation path.

## What This Changes

Prevent Claude CLI OAuth token rotation or partial keychain reads from flipping the CLI auth epoch and resetting active sessions mid-conversation.

## Best Possible Solution

End state should be identity-stable auth epochs for Claude token-rotation races, with regression coverage and preserved invalidation for real account or credential-boundary changes.

## Reproduction Assessment

Yes, at source level. Current main can parse a partial Claude OAuth credential as token-shaped, hash rotating token material into the auth epoch, and reset same-version reusable sessions when that hash changes; no live macOS Keychain race was run here.

## Solution Assessment

Yes for the issue direction. The maintainable path is a narrow identity-stable Claude/auth-profile epoch fix with an auth-epoch version bump and regression tests, and the linked PR is the active implementation path.
