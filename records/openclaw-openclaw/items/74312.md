## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, the parser can still produce that token shape during a partial Claude credential read, and the open linked fix PR remains the focused implementation path.

## What This Changes

The issue reports Claude CLI OAuth token rotation changing auth epochs and resetting active sessions mid-conversation.

## Best Possible Solution

Land or replace the focused auth-epoch fix so Claude token-rotation races are identity-stable while preserving invalidation for real account or credential-boundary changes.

## Reproduction Assessment

Yes, at source level. Current main can parse a partial Claude OAuth credential as token-shaped, hash rotating token material into the auth epoch, and reset same-version reusable sessions when that hash changes.

## Solution Assessment

Yes for the issue direction. The maintainable path is a narrow identity-stable Claude/auth-profile epoch fix with an auth-epoch version bump and regression coverage; the linked PR is the active implementation path.
