## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, and #74493 is still the focused closing PR path for the fix.

## What This Changes

Stabilize Claude CLI session auth epochs across OAuth token rotation so active conversations are not reset by transient token changes.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, with identity-only token hashing, an auth-epoch version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the parser can produce a token credential when `refreshToken` is missing, token encoders include `credential.token`, and same-version auth-epoch mismatches reset reusable CLI sessions; no live macOS keychain race was run.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer plus a version bump is the narrowest maintainable fix and matches the existing OAuth/Gemini precedent; parser hardening can remain a follow-up if maintainers want defense in depth.
