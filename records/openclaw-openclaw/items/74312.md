## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, and the linked closing PR remains the focused path rather than a reason to close before merge.

## What This Changes

Stabilize claude-cli auth-epoch hashing so OAuth token rotation does not reset active CLI sessions mid-conversation.

## Best Possible Solution

Land #74493, or an equivalent narrow replacement, with identity-only token hashing, an auth-epoch version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the parser can produce a token credential when `refreshToken` is missing, token encoders include `credential.token`, and same-version auth-epoch mismatches reset reusable CLI sessions; no live macOS keychain race was run.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer plus a version bump is the narrowest maintainable fix and matches the existing OAuth/Gemini precedent; parser hardening can remain a follow-up if maintainers want defense in depth.
