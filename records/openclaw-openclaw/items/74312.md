## Summary

Keep open. Current main still hashes Claude/token-shaped credentials by rotating token material, and open PR #74493 already targets the issue with the focused fix, so this issue should remain open until that PR is merged or replaced.

## What This Changes

Stop Claude CLI token rotation from changing the CLI auth epoch and resetting active sessions mid-conversation.

## Best Possible Solution

Land #74493, or an equivalent narrow replacement, so Claude/token-shaped local and auth-profile epochs are identity-only, the epoch version is bumped, and regression tests plus changelog coverage ship together.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a missing `refreshToken` falls through to a token credential, token encoders hash `credential.token`, and same-version auth-epoch mismatch invalidates the reusable CLI session.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix: make token-shaped Claude/auth-profile epoch hashing identity-only and bump the epoch version, while leaving parser hardening as optional follow-up unless review finds a separate need.
