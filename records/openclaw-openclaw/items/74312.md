## Summary

Keep open. Current main still lets rotating token material change the Claude CLI auth epoch, and an open PR with closing syntax directly targets the issue, so this should stay open until that PR lands or is replaced.

## What This Changes

Make Claude CLI and auth-profile auth epochs stable across OAuth token rotation so active CLI sessions are not reset unless the account identity changes.

## Best Possible Solution

Land PR #74493 or an equivalent narrow patch that makes Claude token-shaped credentials and token auth-profile epochs identity-based, bumps the auth-epoch version for a clean migration, and keeps regression tests for token rotation stability plus real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to have `readClaudeCliCredentialsCached` return a Claude CLI `type: "token"` credential, change only `credential.token`, and compare `resolveCliAuthEpoch`; current main changes the epoch, and same-version session reuse then resets with `auth-epoch`.

## Solution Assessment

Yes. Identity-only hashing in the auth-epoch encoder is the narrow maintainable fix for the reset loop; parser hardening can remain a separate defense-in-depth follow-up after the epoch contract is corrected.
