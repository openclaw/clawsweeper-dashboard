## Summary

Keep open. Current main still contains the reported auth-epoch reset path, and open PR #74493 already uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Claude CLI auth-epoch hashing ignore rotating token material so long-running sessions do not reset during OAuth token rotation.

## Best Possible Solution

Review and land #74493 or an equivalent narrow patch that makes Claude token-shaped epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps the auth-epoch version, and keeps regression coverage for token rotation stability and real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return a Claude CLI credential with type token, change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch, and same-version epoch mismatches reset sessions with reason auth-epoch.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrowest maintainable fix for the observed reset loop; parser hardening can remain defense-in-depth after the main token-epoch contract is fixed.
