## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile token material into the CLI auth epoch, so token rotation or a transient token-shaped parse can still reset same-version CLI sessions; the linked open PR #74493 is the right active implementation path and should be reviewed before this issue closes.

## What This Changes

Prevent Claude CLI token rotation or transient token-shaped credential parses from changing the CLI auth epoch and resetting active sessions.

## Best Possible Solution

Review and land #74493 or an equivalent narrow patch that makes Claude token-shaped epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps the auth-epoch version, and preserves regression coverage for token rotation stability and real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to have readClaudeCliCredentialsCached return a Claude CLI credential with type "token", change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch, and same-version epoch mismatches reset sessions with reason "auth-epoch".

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrow maintainable fix for the reset loop; parser hardening can remain a separate defense-in-depth follow-up after the main token-epoch contract is corrected.
