## Summary

Keep open. Current main still has the token-inclusive Claude CLI auth-epoch behavior reported here, and the linked open PR is the active implementation path rather than a reason to close the issue before merge.

## What This Changes

The issue reports that Claude CLI token rotation can flip the auth epoch and reset live sessions mid-conversation.

## Best Possible Solution

Review and land PR #74493, or an equivalent identity-only auth-epoch fix, so token rotation and partial Claude keychain reads no longer reset sessions while real identity changes still invalidate reuse.

## Reproduction Assessment

Yes. Current main has a focused code-level reproduction: a Claude token credential change changes the epoch, and the parser can fall into that token shape when `refreshToken` is missing.

## Solution Assessment

Yes. Identity-only hashing at the epoch encoder is the narrowest maintainable fix because it makes the parser race harmless without changing credential parsing or session reuse semantics.
