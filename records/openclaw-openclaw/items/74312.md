## Summary

Keep open. Current main still hashes rotating token material for Claude/token-shaped credentials and auth-profile token credentials, while an open closing PR (#74493) already carries the focused fix, so this issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Prevent Claude CLI OAuth token rotation or transient token-shaped credential parses from changing auth-epoch and resetting active CLI sessions.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, so Claude/token-shaped local and auth-profile epochs are identity-only, the epoch version is bumped, and regression plus changelog coverage ship together.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a missing or empty `refreshToken` parses as a token credential, token credentials include the rotating token in the epoch hash, and the current regression test expects token changes to flip the epoch.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix: stabilize token-shaped epoch hashing on identity fields and bump the epoch version, while leaving parser hardening as optional defense-in-depth.
