## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, and #74493 is an open closing PR for the focused fix, so this issue should remain open until that PR merges or is replaced.

## What This Changes

The issue reports Claude CLI auth-epoch resets caused by token-shaped credentials hashing rotating token material during OAuth token rotation.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, with identity-only token hashing, an auth-epoch version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. The source-level path is high-confidence: missing or empty `refreshToken` can produce a token credential, token credentials are hashed with `credential.token`, and same-version auth-epoch changes invalidate reusable CLI sessions.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer plus a version bump is the narrowest maintainable fix for the reset loop; parser hardening can remain a follow-up unless review finds separate persisted-profile correctness issues.
