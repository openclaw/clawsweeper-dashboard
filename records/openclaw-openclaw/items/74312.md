## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile token material into the CLI auth epoch, so token rotation or a transient token-shaped parse can still reset same-version CLI sessions; the linked open PR #74493 remains the active implementation path, and the reporter follow-up supports its version-bump approach.

## What This Changes

The issue reports Claude CLI auth-epoch hashes changing during OAuth token rotation, causing live CLI sessions to reset mid-conversation.

## Best Possible Solution

Review and land #74493 or an equivalent narrow patch that makes Claude token-shaped epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps CLI_AUTH_EPOCH_VERSION, and preserves tests for token rotation stability plus real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to make readClaudeCliCredentialsCached return a Claude CLI type "token" credential, change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch and same-version mismatches reset with reason "auth-epoch".

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrow maintainable fix for the reported reset loop; parser hardening can remain a defense-in-depth follow-up because stable hashing makes the transient token-shaped parse harmless.
