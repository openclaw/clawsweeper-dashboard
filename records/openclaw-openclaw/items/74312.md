## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped local credentials and token auth-profile credentials, and open PR #74493 is the active closing implementation candidate.

## What This Changes

Claude CLI session auth epochs should ignore rotating token material so OAuth token refreshes do not reset active conversations.

## Best Possible Solution

Land PR #74493, or an equivalent narrow replacement, with identity-only token credential hashing, an epoch-version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. The source-level path is reproducible on current main: a missing `refreshToken` falls through to a token credential, token hashes include the rotating token, and same-version auth-epoch differences invalidate reusable CLI sessions.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrowest maintainable fix for the reported reset loop; parser hardening can remain a defense-in-depth follow-up unless review finds a separate persisted-profile correctness bug.
