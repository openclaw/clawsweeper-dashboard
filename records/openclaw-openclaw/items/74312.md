## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and token auth-profile credentials, while the open closing PR at https://github.com/openclaw/openclaw/pull/74493 is the focused implementation path.

## What This Changes

The issue requests that Claude CLI OAuth token rotation and transient token-shaped keychain reads stop changing auth-epoch and resetting sessions mid-conversation.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74493, or an equivalent narrow replacement, with identity-only token hashing, an auth-epoch version bump, and regression coverage for token rotation and real identity changes.

## Reproduction Assessment

Yes, at source level. Current main can parse a partial Claude OAuth credential as a token credential, hash the token into the auth epoch, and reset same-version reusable sessions when that hash changes; I did not run the live macOS Keychain race.

## Solution Assessment

Yes. Identity-only hashing for token-shaped Claude/profile credentials plus an auth-epoch version bump is the narrow maintainable fix, and parser hardening can remain a separate defense-in-depth follow-up if maintainers want it.
