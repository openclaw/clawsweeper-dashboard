## Summary

Keep open. Current main still hashes rotating token material for Claude/token-shaped local credentials and token auth-profile credentials, while the parser can still fall through to a token credential when `refreshToken` is missing; an open closing PR (#74493) already targets this issue and should be reviewed or replaced before closing.

## What This Changes

Prevent Claude CLI auth epochs from changing during OAuth token rotation when credentials are transiently parsed as token-shaped, preserving live sessions across refreshes.

## Best Possible Solution

Land #74493, or an equivalent narrow replacement, with identity-only hashing for Claude/token-shaped local and auth-profile epochs, an epoch-version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. The source-level path is reproducible on current main: a missing `refreshToken` falls through to a token credential, token credential hashes include the rotating token, and same-version auth-epoch differences invalidate reusable CLI sessions.

## Solution Assessment

Yes. Identity-only token hashing at the auth-epoch layer is the narrowest maintainable fix for the reported reset loop; parser hardening can remain a follow-up unless review finds a separate persisted-profile correctness issue.
