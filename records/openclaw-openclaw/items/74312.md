## Summary

Keep open. Current main still hashes Claude/token-shaped credentials and auth-profile token credentials by rotating token material, and the provided GitHub context shows #74493 is the open closing PR for the focused fix.

## What This Changes

Prevent Claude CLI OAuth token rotation from flipping auth-epoch hashes and resetting active CLI sessions mid-conversation.

## Best Possible Solution

Land #74493, or an equivalent narrow replacement, so Claude/token-shaped local and auth-profile epochs are identity-only, the epoch version is bumped, and regression tests plus changelog coverage ship together.

## Reproduction Assessment

Yes. The source-level path is reproducible on current main: missing `refreshToken` falls through to a token credential, token credentials include rotating token material in the epoch hash, and the current test expects token rotation to flip the epoch.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it makes token-shaped Claude/auth-profile epoch hashing identity-only while preserving account-change invalidation through stable identity fields and an epoch-version bump.
