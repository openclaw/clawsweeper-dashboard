## Summary

Keep open. Current main still hashes rotating token material for Claude token-shaped credentials and auth-profile token credentials, while the open closing PR #74493 already carries the focused fix and should be reviewed or replaced before this issue closes.

## What This Changes

Prevent Claude CLI OAuth token rotation or transient token-shaped credential parses from changing auth-epoch and resetting active CLI sessions.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, so Claude token-shaped local and auth-profile epochs become identity-only with an epoch-version bump, regression tests, and changelog coverage.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a missing or empty `refreshToken` parses as a token credential, token credentials include the rotating token in the epoch hash, and same-version auth-epoch differences invalidate reusable CLI sessions.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer, with an epoch-version bump, is the narrowest maintainable fix for the reset loop; parser hardening can remain optional defense-in-depth unless a separate persisted-profile bug is confirmed.
