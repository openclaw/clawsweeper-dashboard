## Summary

Keep open. Current main still hashes Claude/token-shaped credentials and token auth profiles by token material, and the open closing PR #74493 is unmerged, so this issue should remain open until that fix or an equivalent replacement lands.

## What This Changes

Claude CLI auth epochs should stay stable across OAuth token rotation and partial keychain reads instead of resetting live sessions mid-conversation.

## Best Possible Solution

Review and land #74493, or an equivalent narrow replacement, so Claude/token-shaped local and auth-profile epochs are identity-only, the epoch version is bumped, and regression plus changelog coverage ships together.

## Reproduction Assessment

Yes. The source-level path is reproducible on current main: missing `refreshToken` can produce a token credential, token credentials include rotating token material in the epoch hash, and same-version epoch mismatches reset sessions.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: remove rotating token material from the epoch identity and bump `CLI_AUTH_EPOCH_VERSION`; parser hardening can remain a separate defense-in-depth follow-up if maintainers want it.
