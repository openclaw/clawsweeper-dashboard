## Repair Work Prompt

Fix Google Chat issue https://github.com/openclaw/openclaw/issues/65007 in a narrow plugin-local PR. Current main still has three failures: add-on lifecycle payloads are not normalized, wildcard routed spaces with no sender allowlist retain sender `allowlist` blocking, and outbound reply delivery can pass a message resource as Google Chat `thread.name`. Treat related PR https://github.com/openclaw/openclaw/pull/65058 as useful parser-only source context, but cover all three surfaces. In `extensions/googlechat`, expand `parseGoogleChatInboundPayload` to map `chat.messagePayload`, `chat.addedToSpacePayload`, and `chat.removedFromSpacePayload` to standard `MESSAGE`, `ADDED_TO_SPACE`, and `REMOVED_FROM_SPACE` events while preserving `authorizationEventObject.systemIdToken`. Align group sender policy so a matched wildcard or routed group without `users`/`groupAllowFrom` allows senders according to `resolveSenderScopedGroupPolicy` and still honors mention gating. Normalize Google Chat reply threading so only `spaces/.../threads/...` is sent as the Chat thread; when implicit or `[[reply_to_current]]` yields `spaces/.../messages/...`, fall back to the inbound thread resource where available or omit the thread rather than sending an invalid resource. Add plugin-local regression tests for each path and a single-line `CHANGELOG.md` fix entry. Do not change core contracts unless a tiny generic seam is strictly necessary.
