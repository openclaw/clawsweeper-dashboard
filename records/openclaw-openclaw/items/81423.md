## Summary

Keep open: this is a plausible focused bug fix, but the latest patch still has two blocking correctness issues and only mock/runtime-boundary proof for the changed behavior.

## What This Changes

The PR changes memory-core QMD v2 search payload construction so lexical search keeps the raw query while vec/hyde use a hyphen-normalized query, with focused QMD manager tests and changelog text.

## Best Possible Solution

Land a cleaned patch that preserves raw QMD lexical search, normalizes every word-internal hyphen only for semantic searches, restores clean changelog encoding, and includes live QMD proof or an explicit maintainer waiver.

## Reproduction Assessment

Yes at source level. Current main sends raw hyphenated queries to QMD vec/hyde, QMD v2.0.1 rejects those semantic queries, and OpenClaw then falls back to builtin search on the primary QMD error.

## Solution Assessment

No for this exact patch. Semantic-only normalization is the right narrow direction, but the regex misses adjacent short dashed tokens and the changelog/proof blockers still need resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Normalize adjacent hyphens without consuming letters:** `extensions/memory-core/src/memory/qmd-manager.ts:3155`
