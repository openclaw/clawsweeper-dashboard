## Summary

Keep open: this is the right bug area, but the current PR head still has two blocking patch issues in the normalizer and changelog encoding, and the protected maintainer label also makes it non-closeable by cleanup policy.

## What This Changes

The PR changes memory-core QMD v2 search payload construction so lex keeps the raw query while vec/hyde use a hyphen-normalized query, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a cleaned patch that normalizes every word-internal hyphen only in QMD semantic queries, preserves the raw lex query, restores unchanged UTF-8 changelog content, and keeps focused hybrid/vector-only coverage.

## Reproduction Assessment

Yes at source level. Current main sends hyphenated text to QMD vec/hyde, QMD v2.0.1 rejects semantic queries matching `/-\w/`, and OpenClaw falls back from the failed QMD manager to the builtin index.

## Solution Assessment

No for this exact head. Semantic-only normalization is the right narrow fix, but the implementation must replace only the hyphen so overlapping chains are handled, and the unrelated changelog encoding damage must be removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Normalize every word-internal hyphen:** `extensions/memory-core/src/memory/qmd-manager.ts:3155`
  - body: `normalizeQmdSemanticQuery()` consumes both neighboring word characters, so overlapping chains are skipped: `x-y-z` becomes `x y-z`. Because QMD v2.0.1 rejects semantic queries matching `/-\w/`, those vec/hyde searches can still fail and trigger the builtin fallback; replace only the hyphen and add a chained-token regression case.
  - confidence: 0.94
- **[P2] Restore the changelog encoding:** `CHANGELOG.md:1`
  - body: This head adds a UTF-8 BOM and rewrites unrelated existing non-ASCII changelog text as mojibake. That would publish corrupted release notes, so restore the file's original UTF-8 content and keep only the intended Memory/QMD entry.
  - confidence: 0.98
