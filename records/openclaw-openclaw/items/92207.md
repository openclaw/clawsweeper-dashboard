## Summary

The report is a valid, newly shipped memory-wiki regression and current `main` still throws when a plugin-supplied public artifact omits `agentIds`. This issue should remain open because three active PRs explicitly claim it; maintainers should select one focused boundary fix rather than create another implementation.

## What This Changes

Prevent all `wiki_*` tools from crashing when a public memory artifact lacks `agentIds`.

## Best Possible Solution

Land one focused PR that normalizes omitted `agentIds` once at `listActiveMemoryPublicArtifacts`, preserves the required `string[]` shape for downstream consumers, adds a malformed-provider regression test, and demonstrates a redacted affected `wiki_search` or `wiki_status` call succeeding after the patch.

## Reproduction Assessment

Yes at source level with high confidence: have a registered public-artifact provider return one artifact without `agentIds`, then call `listActiveMemoryPublicArtifacts`; current `main` reaches the failing array spread. The reporter independently observed the same failure on OpenClaw 2026.6.5 on Windows 11.

## Solution Assessment

Yes, boundary normalization is the narrowest maintainable solution because all consumers then receive the declared required-array shape. Extra wiki-local guards or defaults for unrelated required fields should be excluded unless separately reproduced.
