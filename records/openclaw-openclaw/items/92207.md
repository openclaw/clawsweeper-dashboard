## Label Justifications

- `P1`: Affected users cannot use any memory-wiki tool in the workspace, breaking an established agent workflow.
- `impact:other`: The defect disables the memory-wiki tool family, but the evidence shows a tool invocation exception rather than a process-level crash or restart loop.

## Summary

Current main still has the reported undefined-array throw, while multiple open PRs explicitly reference this issue. Keep the issue open until one focused implementation merges; the shared artifact-listing boundary with exact regression tests is the strongest current path.

## What This Changes

Treat missing `agentIds` from memory public artifacts as an empty list before cloning and sorting them.

## Best Possible Solution

Normalize absent `agentIds` once in `listActiveMemoryPublicArtifacts`, continue returning a canonical `string[]`, and retain focused regression coverage for both cloning and deterministic sorting.

## Reproduction Assessment

Yes at source level: register a memory capability whose artifact listing returns one item with `agentIds: undefined`, then call `listActiveMemoryPublicArtifacts`; current main throws at the direct spread. The reporter also observed the same failure on OpenClaw 2026.6.5 on Windows 11.

## Solution Assessment

Yes. Normalizing at the shared active-artifact boundary is narrower and more maintainable than adding defensive guards throughout memory-wiki, and the preferred candidate directly tests the malformed runtime state.
