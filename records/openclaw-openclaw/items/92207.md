## Summary

Current main and v2026.6.5 both crash if an active memory plugin returns a public artifact without `agentIds`; no open PR specifically owns this fix, so the issue remains necessary and is a focused repair candidate.

## What This Changes

Guard memory-wiki public-artifact handling so missing agent IDs do not crash wiki tools.

## Best Possible Solution

Keep `MemoryPluginPublicArtifact` canonical and strict, normalize a missing or non-array `agentIds` value to an empty array once in `listActiveMemoryPublicArtifacts`, and add focused coverage proving deterministic listing and memory-wiki bridge sync no longer throw.

## Reproduction Assessment

Yes at source level. Returning one otherwise valid public artifact with omitted `agentIds` reaches the unconditional spread in both v2026.6.5 and current main; the reporter also supplied a concrete released Windows workflow and successful local after-fix validation.

## Solution Assessment

No as proposed. The narrow maintainable solution is a source-level normalization at the shared active-plugin boundary, not edits to generated bundles or defaults for unrelated required fields.
