## Summary

Keep open. Current main still renders Feishu message-tool presentation buttons/selects as markdown fallback, and the PR targets the right bug, but the patch drops mixed message text and breaks the intended lazy channel-runtime boundary.

## What This Changes

The PR exports Feishu outbound card button/block helpers and changes the Feishu message-tool card builder to map presentation blocks through those helpers.

## Best Possible Solution

Share a lightweight Feishu presentation-card renderer between the message-tool and outbound paths, preserve text before native block elements, keep runtime imports lazy, and cover the mixed text/button/select cases in tests plus changelog.

## Reproduction Assessment

Yes. Invoke the Feishu message action with a `presentation` button block, or extend `extensions/feishu/src/channel.test.ts`; current main produces fallback markdown while `extensions/feishu/src/outbound.test.ts` shows the native action-card behavior to match.

## Solution Assessment

No. Reusing the existing renderer is the right direction, but this patch is not the best solution as-is because it drops mixed message text and imports the runtime-heavy outbound module from the channel entry.

## Review Findings

Overall correctness: patch is incorrect

  - body: When a send includes both `message`/`text` and `presentation`, `handleAction` passes the text as `fallbackText`.
