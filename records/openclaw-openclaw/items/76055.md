## Summary

Keep open. The PR targets a real Feishu message-tool gap, but the branch should not merge as-is because it drops `text/message` content when presentation blocks render and pulls outbound runtime code into the channel entry path.

## What This Changes

The PR exports Feishu outbound card-rendering helpers and changes the Feishu message-tool presentation card builder to render presentation blocks through them.

## Best Possible Solution

Share a lightweight Feishu presentation-card renderer between the message-tool and deliver-pipeline paths, preserve text fallback before block elements, keep runtime imports lazy, and cover native button plus select fallback behavior in tests.

## Reproduction Assessment

Yes. The relevant path is deterministic: invoke the Feishu message action with `presentation` button blocks, or extend `extensions/feishu/src/channel.test.ts`; current main produces markdown fallback while `extensions/feishu/src/outbound.test.ts` shows the native action-card behavior to match.

## Solution Assessment

No. Reusing the existing Feishu card-element behavior is the right direction, but the proposed implementation should preserve `text/message` content and avoid importing `outbound.js` from the channel entry module.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve message text before presentation blocks:** `extensions/feishu/src/channel.ts:155`
  - body: The new card builder only uses `params.fallbackText` when no presentation block produced an element. Feishu message-tool sends can include both `text/message` and `presentation`; the existing outbound renderer prepends that text before block elements, so this drops the actual message whenever a button, select, or text block renders. Add the fallback text as a markdown element before mapping blocks, or share the outbound card builder exactly.
  - confidence: 0.89
- **[P2] Keep outbound runtime out of the channel entry:** `extensions/feishu/src/channel.ts:28-31`
  - body: Importing `buildFeishuCardElementForBlock` from `./outbound.js` makes the Feishu channel plugin entry eagerly load `outbound.ts`, which pulls in send/client modules and Lark SDK module side effects. Current `channel.ts` deliberately loads those paths through `loadFeishuChannelRuntime`; move the pure presentation renderer to a lightweight helper used by both files instead.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/feishu/src/channel.ts:147`
  - body: This is a user-facing Feishu rendering fix, but the diff only changes channel/outbound code. Repo policy expects user-visible `fix` changes to add a single-line entry under the active `CHANGELOG.md` Fixes section.
  - confidence: 0.95
