## Summary

Keep open: current main still lacks this feature, but the PR is not ready to merge because the new public hook misses existing plugin access-control and timeout contracts, lacks docs, and has no real behavior proof.

## What This Changes

The branch adds a public `agent_error` plugin hook, invokes it from embedded Pi error lifecycle handling before broadcasting the error, adds hook runner types/tests, and updates the changelog.

## Best Possible Solution

Refresh the PR so `agent_error` is a documented, conversation-access-gated hook that honors existing hook timeout semantics and is proven with redacted live provider/plugin error output.

## Reproduction Assessment

Yes for the PR review findings by source inspection: PR head adds `agent_error` while omitting it from conversation gating and directly awaits handlers in a custom runner. The requested feature itself is absent on current main, so the PR is not obsolete.

## Solution Assessment

No. A dedicated hook is a plausible seam, but this implementation is not the best landing shape until it matches the existing access-control, timeout, documentation, and real-proof expectations for public plugin APIs.

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook names without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register it without `plugins.entries.<id>.hooks.allowConversationAccess=true`, unlike adjacent provider/conversation hooks.   - confidence: 0.93

  - body: The new runner awaits each `agent_error` handler directly, so `api.on(..., { timeoutMs })`, operator `hooks.timeouts.agent_error`, and default modifying-hook timeout handling never apply per handler. A slow first plugin can starve later handlers until the lifecycle-level race discards all replacements.
  - confidence: 0.9

  - body: The PR exposes a new public hook but leaves the hook catalog and conversation-access docs without its event shape, replacement semantics, or access requirements. Public plugin authors need that documented contract before depending on the hook.
  - confidence: 0.82
