## Summary

Keep open: current main still lacks this hook, but the PR should not merge as-is because it bypasses the existing conversation-access guard, ignores per-handler hook timeout contracts, and is stale against current lifecycle delivery-evidence logic.

## What This Changes

The PR adds a public `agent_error` plugin hook, invokes it during embedded Pi lifecycle error handling with a 2s outer timeout, adds lifecycle tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR active, but require a branch update that treats `agent_error` as a guarded conversation hook, preserves current delivery-evidence behavior on rebase, honors per-handler hook timeouts while preserving chaining, documents the public hook, and credits the contributor in the changelog.

## Reproduction Assessment

Yes. A static reproduction is to register a non-bundled plugin with `api.on("agent_error", ...)` and no `allowConversationAccess`; the PR adds the hook name but not the guarded-name list, so the registry would accept it. A second focused check is a never-settling `agent_error` handler with `timeoutMs`, which the PR runner awaits directly instead of timing out per handler.

## Solution Assessment

No. A dedicated hook is a plausible seam, but this implementation is not the best way to land it because it adds a public plugin API without extending the existing security, timeout, docs, and current-main lifecycle contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook list without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register this hook without `plugins.entries.<id>.hooks.allowConversationAccess=true`. Since the hook receives provider lifecycle error text before user broadcast, it should be guarded like adjacent conversation hooks such as `agent_end`, `llm_input`, and `llm_output`.
  - confidence: 0.91
- **[P2] Honor per-handler timeouts in agent_error:** `src/plugins/hooks.ts:690-695`
  - body: This loop awaits each `agent_error` handler directly, bypassing the existing `timeoutMs` handling used by modifying hooks. A slow handler with its own timeout can block later handlers until the outer 2s lifecycle race drops all replacements, which violates the documented per-hook timeout contract.
  - confidence: 0.88

  - body: The branch still computes deterministic side effects from only sent text/media arrays, but current main now uses `hasCommittedMessagingToolDeliveryEvidence(ctx.state)` so target-only committed deliveries also count. Please rebase and keep that helper, otherwise target-only delivered runs can be marked abandoned or replayable again.
  - confidence: 0.82
