## Summary

Keep open: the hook is a plausible public plugin seam, but the PR still has merge-blocking access-control and timeout-contract defects before it can land safely.

## What This Changes

The branch adds a public `agent_error` plugin hook, calls it from embedded Pi agent-error lifecycle handling, adds lifecycle tests, and records a changelog entry.

## Best Possible Solution

Land an updated hook that treats provider error replacement as a guarded conversation hook, preserves existing hook timeout semantics, documents the public API/access policy, and validates the plugin and lifecycle tests.

## Reproduction Assessment

Yes. Source inspection of the PR is enough: a non-bundled plugin can register `api.on("agent_error", ...)` without `allowConversationAccess` because the PR adds the public hook name but not the guarded conversation-hook list, and a slow handler shows the timeout defect because `runAgentError` awaits handlers directly.

## Solution Assessment

No. A dedicated hook is plausible, but this implementation should not merge until it follows the existing conversation-access guard, per-handler timeout helper, docs, changelog, and regression-test contracts.
