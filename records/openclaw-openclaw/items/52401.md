## Summary

Keep open: current main does not have the requested hook, but this PR is not merge-ready because it exposes provider lifecycle error text without the existing conversation-access guard and bypasses the standard per-handler hook timeout contract.

## What This Changes

Adds a public `agent_error` plugin hook that lets plugins replace embedded Pi lifecycle error messages before they are broadcast.

## Best Possible Solution

Land a documented, conversation-access-gated `agent_error` hook that uses the standard modifying-hook timeout path and includes redacted real plugin/provider proof.

## Reproduction Assessment

Yes for the PR review findings by source inspection: PR head adds `agent_error` outside `CONVERSATION_HOOK_NAMES` and awaits each handler directly. The requested hook itself is absent on current main, so the PR is not obsolete.

## Solution Assessment

No. A dedicated hook is plausible, but this branch is not the best landing shape until it matches existing access-control, timeout, documentation, and proof expectations for public plugin APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook names without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register for provider lifecycle error text without `plugins.entries.<id>.hooks.allowConversationAccess=true`, unlike adjacent raw conversation hooks. Add it to the guard and cover blocked/allowed registration.
  - confidence: 0.93
- **[P2] Honor per-handler hook timeouts:** `src/plugins/hooks.ts:690-695`
  - body: `runAgentError` directly awaits each handler, so `api.on(..., { timeoutMs })`, operator `hooks.timeouts.agent_error`, and the standard modifying-hook timeout path are skipped. A slow first plugin can consume the lifecycle-level 2s race and prevent later handlers from running.
  - confidence: 0.9
- **[P3] Document the public agent_error hook:** `src/plugins/hook-types.ts:68`
  - body: This exposes a new typed public hook but leaves `docs/plugins/hooks.md` without its event shape, replacement semantics, access requirement, and timeout behavior. Plugin authors need that contract before relying on it.
  - confidence: 0.82
