## Summary

Keep this PR open. Current main still lacks an `agent_error` hook and the requested plugin seam is concrete, but the proposed patch should not merge as-is because it adds a new error-content hook without the existing conversation-access guard and bypasses the documented per-handler timeout contract.

## What This Changes

The PR adds a public `agent_error` plugin hook, invokes it from embedded Pi agent error lifecycle handling with a 2s lifecycle timeout, adds lifecycle tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR active as a core plugin API proposal, but update the branch so `agent_error` follows the existing conversation-access policy, each handler goes through the established timeout helper while preserving chained replacements, public docs describe the processed error semantics, and the changelog entry credits the contributor before maintainer merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to register a non-bundled plugin with `api.on("agent_error", ...)` and no `allowConversationAccess`; because the PR adds the hook name but not the guarded-name list, the registry would accept it. A second focused check is a never-resolving `agent_error` handler with `timeoutMs`, which the PR runner awaits directly instead of timing out per handler.

## Solution Assessment

No. The hook is a plausible seam, but this patch is not the best way to land it because it bypasses existing plugin access and timeout contracts; the safer solution is to extend those contracts explicitly rather than adding a one-off lifecycle race around a custom runner.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook list without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register this hook without `plugins.entries.<id>.hooks.allowConversationAccess=true`. This hook receives provider lifecycle error text before `buildTextObservationFields` shaping, so it should be guarded like adjacent conversation hooks.
  - confidence: 0.91
- **[P2] Honor per-handler timeouts in agent_error:** `src/plugins/hooks.ts:690-695`
  - body: This custom loop awaits each `agent_error` handler directly, bypassing `getModifyingHookTimeoutMs` and `withHookTimeout`. A handler registered with `api.on(..., { timeoutMs })` can block later handlers until the outer lifecycle 2s race discards all replacements, contrary to the documented per-hook timeout contract.
  - confidence: 0.88
- **[P3] Add changelog contributor attribution:** `CHANGELOG.md:3135`
  - body: The added user-facing changelog bullet has no `Thanks @...` credit, but repo changelog rules require each added entry to include at least one contributor attribution. Please credit the PR contributor rather than a bot or maintainer handle.
  - confidence: 0.94
