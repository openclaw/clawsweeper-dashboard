## Best Possible Solution

Land a refreshed hook only after it follows existing plugin contracts: conversation-access gating, per-handler timeout behavior, public docs/tests, and redacted real behavior proof.

The requested feature itself is absent on current main, so the PR is not obsolete.

## Solution Assessment

No. A dedicated hook is a plausible seam, but this implementation is not the best landing shape until it matches the existing access-control, timeout, documentation, and real-proof expectations for public plugin APIs.
