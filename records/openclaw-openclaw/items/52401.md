## Summary

Keep this PR open: current main still lacks the requested hook, but the branch has concrete public plugin API blockers around conversation-access policy and hook timeout semantics before it can merge.

## What This Changes

The PR adds an `agent_error` plugin hook, invokes it from embedded Pi lifecycle error handling before broadcasting provider errors, adds hook runner types/tests, and updates the changelog.

## Best Possible Solution

Land a refreshed hook only after it follows existing plugin contracts: guard it as conversation-access data, preserve per-handler timeout behavior, document the access policy, and keep lifecycle error delivery bounded.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register `agent_error` from a non-bundled plugin without `allowConversationAccess`, and the PR-head guard list would not block it; a slow handler with `timeoutMs` is also awaited directly in the PR-head loop.

## Solution Assessment

No. A dedicated hook is a plausible seam, but this implementation is not the best landing shape until it matches the existing conversation-access, timeout, docs, and changelog contracts for public plugin APIs.
