## What This Changes

The issue reports Mistral thinking responses being stored and displayed as repeated `[object Object]` text in assistant messages and memory.

## Best Possible Solution

Keep this open until a narrow transport-level normalization fix for structured OpenAI-compatible completion deltas is merged and validated, preferably by landing or replacing https://github.com/openclaw/openclaw/pull/75339 with real Mistral behavior proof.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize structured content blocks in the shared OpenAI-compatible completions transport before appending text or thinking deltas, not to repair already-corrupted session files as the primary runtime fix.
