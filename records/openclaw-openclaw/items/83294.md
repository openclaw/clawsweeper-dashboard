## Summary

Keep this PR open, but do not merge it as written: it addresses a real plaintext-secret persistence problem, yet the diff can silently remove the only usable auth source for existing models.json-only custom providers without a migration or maintainer-approved compatibility policy.

## What This Changes

The PR adds a write-time sanitizer that removes non-marker provider apiKey values from generated models.json and adds focused planning coverage for configured and existing provider entries.

## Best Possible Solution

Remove plaintext persistence only with an explicit migration to auth profiles or SecretRefs, or a fail-closed doctor/audit path that prevents rewriting away a provider’s only credential.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main preserves existing models.json apiKey values during merge and resolves non-marker custom provider apiKey values as usable models.json auth.

## Solution Assessment

No. Stripping before write is the right security boundary, but this implementation needs a migration, preservation, or explicit block for models.json-only credentials before it is a safe fix.
