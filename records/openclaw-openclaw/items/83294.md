## Summary

Keep open: this targets a real security surface, but the patch currently strips non-marker `apiKey` values from persisted providers without preserving or migrating a runtime auth source for `models.json`-only providers, and the external PR has only unit-test proof.

## What This Changes

The branch adds a planner sanitizer that removes non-marker provider `apiKey` fields before writing `models.json` and adds unit coverage for configured and merged existing providers.

## Best Possible Solution

Implement the sanitizer with an explicit auth-source migration or fail-closed doctor/audit path so plaintext no longer persists while existing provider auth is not silently lost.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current `main` writes merged providers, including preserved plaintext `apiKey` values, directly to `models.json`. I did not run the write path because this review is read-only.

## Solution Assessment

No, not as written. Stripping before write is the right boundary, but the patch also drops existing `models.json`-only credentials without an alternate auth source or migration proof.
