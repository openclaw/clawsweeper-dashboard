## Summary

Keep open: current main and the latest release still send raw hyphenated queries into QMD semantic sub-searches and fall back to builtin on the resulting QMD error. The linked draft fix uses closing syntax and is the right implementation path, so this issue should remain open until that PR merges or is explicitly rejected.

## What This Changes

`memory_search` should preserve QMD lexical search for hyphenated queries instead of falling back to builtin when QMD semantic sub-search validation rejects them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent memory-core fix that keeps raw QMD lex queries while sending semantic-safe `vec`/`hyde` payloads with focused tests and live QMD proof or a maintainer waiver.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main sends the raw dashed query to QMD `lex`, `vec`, and `hyde`, and QMD v2.0.1 rejects dashed semantic queries before lexical results can be returned. I did not run a live QMD daemon in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix because it preserves exact lexical recall and changes only semantic QMD sub-search payloads. The issue should stay open until that PR is reviewed, proven, and merged or replaced.
