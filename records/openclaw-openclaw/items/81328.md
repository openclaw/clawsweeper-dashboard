## Summary

Keep open: current main still sends raw hyphenated queries into QMD semantic sub-searches, and any resulting QMD failure flips the whole memory manager to the builtin index instead of preserving QMD lexical recall.

## What This Changes

The issue asks OpenClaw to prevent QMD semantic validation failures on hyphenated `memory_search` queries from taking down QMD recall and falling back to builtin search.

## Best Possible Solution

Keep QMD lexical recall available for dashed identifiers and dates by avoiding semantic validation on raw hyphenated tokens in vec/hyde paths, with regression coverage for QMD query construction and fallback behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: QMD MCP query mode sends the raw query to `lex`, `vec`, and `hyde`, QMD v2.0.1 rejects hyphenated semantic queries, and OpenClaw then treats the primary QMD search as failed. A live binary run was not performed in this read-only checkout.

## Solution Assessment

Yes, the narrow maintainable direction is to preserve raw lex search while sanitizing or isolating only vec/hyde payloads, and to add targeted tests around dashed tokens and fallback behavior.
