## Summary

Keep open: current main still sends the same raw QMD query to lexical and semantic sub-searches, so the reported compatibility bug remains for affected QMD releases. There is already an open draft fix PR with closing syntax, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests preserving QMD lexical memory search for hyphenated queries instead of letting vec/hyde validation failures force fallback to the builtin index.

## Best Possible Solution

Review and land the linked memory-core fix, or replace it with an equivalent narrow change that keeps raw lexical QMD search while avoiding semantic validation on raw hyphenated tokens.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main sends raw dashed queries to QMD `vec` and `hyde`, affected QMD releases reject those semantic queries, and OpenClaw then falls back to builtin search. A live QMD daemon run was not performed in this read-only checkout.

## Solution Assessment

Yes, the best fix direction is the narrow one already proposed by the linked draft PR: preserve the raw query for QMD `lex` and normalize or isolate only semantic sub-searches, with focused regression tests.
