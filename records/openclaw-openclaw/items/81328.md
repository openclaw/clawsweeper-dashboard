## Summary

Keep open: current main and the latest shipped release still send hyphenated text unchanged into QMD semantic searches, and the active linked fix has not merged yet.

## What This Changes

Fix `memory_search` so QMD semantic validation failures for hyphenated tokens do not discard otherwise valid QMD lexical results and fall back entirely to builtin search.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent narrow patch, then close this issue after the fix is merged and validated.

## Reproduction Assessment

Yes, at source level. Current main sends hyphenated text to QMD `vec`/`hyde`, QMD v2.0.1 rejects semantic queries matching `/-\w/`, and OpenClaw falls back to builtin after the primary QMD error.

## Solution Assessment

Yes. The narrow maintainable fix is semantic-only normalization or sub-search error isolation that keeps raw lexical QMD search intact; the linked PR follows that direction but has not merged.
