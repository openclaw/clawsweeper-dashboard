## Summary

Keep open: current main still has the QMD semantic-query failure path, and an open unmerged PR already tracks the intended fix with closing syntax.

## What This Changes

Fix `memory_search` so hyphenated QMD semantic queries do not cause the whole QMD search to fall back to the builtin index.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent narrow patch that keeps raw lexical QMD search and normalizes or isolates only semantic QMD searches, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. With QMD semantic mode enabled, current main sends hyphenated tokens into `vec`/`hyde`, QMD v2.0.1 rejects them, and OpenClaw falls back to builtin after the primary QMD exception.

## Solution Assessment

Yes. Semantic-only normalization or per-subsearch isolation is the narrow maintainable fix because it preserves raw lexical matching while avoiding QMD's semantic validation bug.
