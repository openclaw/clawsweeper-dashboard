## Summary

Keep open: current main still sends raw hyphenated queries into QMD semantic sub-searches and treats the resulting primary QMD error as a fallback-to-builtin condition; the linked closing PR remains open and unmerged.

## What This Changes

The issue requests a memory_search/QMD fix so hyphenated tokens do not cause QMD semantic validation failures to abort the whole QMD search path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent semantic-only normalization/error-isolation patch that preserves raw QMD lexical search, then close this issue after merge and validation.

## Reproduction Assessment

Yes, at source level. Current main sends raw dashed tokens to QMD vec/hyde for unified query payloads, QMD v2.0.1 rejects those semantic queries, and OpenClaw falls back to builtin search when the primary QMD search throws.

## Solution Assessment

Yes in direction. Semantic-only normalization or per-subsearch error isolation is the narrow maintainable fix because it keeps raw lexical recall intact while avoiding QMD semantic validation failures.
