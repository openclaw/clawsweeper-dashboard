## Summary

Keep open: current main still sends raw hyphenated queries into QMD semantic sub-searches and falls back to the builtin index on the resulting QMD error. The linked draft pull request is the active fix path, so this issue should stay open until that PR merges or is replaced.

## What This Changes

Fix `memory_search` so QMD lexical recall is preserved for hyphenated tokens while semantic QMD sub-searches avoid qmd validation failures.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423 or an equivalent narrow patch that preserves raw QMD lexical search while normalizing only semantic `vec`/`hyde` payloads.

## Reproduction Assessment

Yes at source level: current main sends raw hyphenated tokens to QMD `vec`/`hyde`, QMD v2.0.1 rejects those semantic queries, and OpenClaw then falls back from QMD to builtin search. I did not establish a live daemon reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable solution is to keep exact lexical search on the raw query and normalize only semantic QMD sub-searches. The open draft PR follows that direction but should not close this issue until it merges or is replaced.
