## Summary

Keep open: current main still sends raw hyphenated queries into QMD semantic sub-searches and falls back to builtin on primary QMD errors, while an open draft PR already tracks the intended fix but has not merged.

## What This Changes

The issue asks `memory_search` to preserve QMD lexical recall when hyphenated tokens make QMD semantic sub-search validation fail.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent narrow patch, then close this issue after the fix is merged and validated.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main sends the raw hyphenated query to QMD `vec`/`hyde`, QMD v2.0.1 rejects those semantic queries, and OpenClaw then falls back to builtin on the primary QMD error.

## Solution Assessment

Yes. The best fix is semantic-only normalization or isolation that keeps raw lexical QMD search intact; the linked draft PR follows that direction but should stay paired with this issue until merged or replaced.
