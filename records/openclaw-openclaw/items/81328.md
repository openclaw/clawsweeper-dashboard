## Summary

Keep open: current main and the latest shipped release still contain the reported QMD raw-query path, and an open draft fix with closing syntax already owns the implementation path.

## What This Changes

Fix `memory_search` so hyphenated QMD queries keep raw lexical search instead of letting semantic validation force builtin fallback.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81423 so raw QMD lexical recall is preserved while semantic `vec`/`hyde` payloads avoid QMD hyphen validation.

## Reproduction Assessment

Yes, at source level: `memory_search` passes the raw query to QMD v2 searches, `vec`/`hyde` receive dashed tokens, QMD v2.0.1 rejects those semantic queries, and OpenClaw then falls back to builtin. I did not run a live QMD daemon in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to preserve raw `lex` search while normalizing or isolating only semantic QMD sub-searches. The existing draft PR follows that direction and should wait for live proof or an explicit maintainer proof waiver.
