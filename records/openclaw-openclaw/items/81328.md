## Summary

Keep open: current main and the latest release still send raw hyphenated queries into QMD semantic sub-searches, and an open draft fix already references this issue with closing syntax.

## What This Changes

The issue requests preserving QMD lexical recall for hyphenated tokens while preventing QMD semantic validation failures from forcing a builtin-memory fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81423, or an equivalent narrow patch, after live QMD proof or an explicit maintainer proof waiver.

## Reproduction Assessment

Yes. Source inspection plus QMD v2.0.1 source gives a high-confidence reproduction path: `query` mode sends `sqlite-vec` to `vec`/`hyde`, QMD rejects it, and OpenClaw falls back to builtin on the thrown QMD error.

## Solution Assessment

Yes. The best fix is the linked semantic-only normalization approach: keep raw `lex` queries intact and normalize only `vec`/`hyde` payloads, with focused regression coverage.
