## Summary

## What This Changes

## Best Possible Solution

Keep warning-only behavior as the default and land the bounded opt-in retry only if maintainers explicitly accept private-text replay for recovery; preserve the one-shot marker, collect isolation, and user-turn persistence suppression as required invariants.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers want an opt-in core recovery mode. A default-off, one-shot message-tool retry is narrower and safer than directly bypassing source-reply suppression.
