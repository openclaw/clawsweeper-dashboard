## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Bound stranded-reply retries:** `src/auto-reply/reply/agent-runner.ts:2349`
  - body: This enqueue uses dedupe mode `"none"` and the retry run inherits the same `message_tool_only` source-reply mode. If the retry model again emits a long final answer without calling `message(action=send)`, the detector will enqueue another retry, so one missed tool call can become an unbounded follow-up loop. Add a one-shot marker/counter and a regression test that a failed retry stops with a warning/manual recovery path.
  - confidence: 0.9
- **[P1] Keep retry prompts out of user history:** `src/auto-reply/reply/agent-runner.ts:2337-2347`
  - body: Clearing `transcriptPrompt` and the recorder is not enough to make this retry internal: the spread run still does not set `run.suppressNextUserMessagePersistence`, and the embedded runner defaults that flag to false before building the prompt submission. That means the recovery instruction and original answer can be recorded as a normal user turn in the Codex session, polluting future context. Mark the retry as non-persistent/internal and cover it with a session-history regression test.
  - confidence: 0.87
