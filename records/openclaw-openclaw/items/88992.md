## Summary

## What This Changes

The PR adds default-off `messages.strandedReplyRecovery`, enqueues one bounded message-tool retry for stranded private finals, strengthens prompt/docs/schema coverage, and adds unit plus qa-lab scenario proof.

## Best Possible Solution

Land only if maintainers accept the default-off recovery flag and privacy tradeoff; otherwise keep the current warning-only behavior and resolve the linked issue with that product decision.

## Reproduction Assessment

Yes. Current main is source-reproducible: it detects and warns on substantive `message_tool_only` finals but suppresses automatic delivery, and the linked issue provides live Telegram observations for the same failure class.

## Solution Assessment

Yes, with a maintainer decision. A default-off retry is a safer fit than auto-publishing private final text because it preserves the documented default contract, but the new config/privacy direction must be accepted before merge.
