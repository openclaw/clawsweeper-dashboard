## Summary

Keep open for maintainer review: the patch is technically coherent and current main still only warns for stranded `message_tool_only` finals, but the PR adds a default-off recovery mode that changes delivery and private-text handling when enabled.

## What This Changes

The PR adds default-off `messages.strandedReplyRecovery`, strengthens message-tool prompt guidance, enqueues one bounded `message(action=send)` retry for stranded `message_tool_only` finals when enabled, and adds config, docs, unit, queue, and qa-lab coverage.

## Best Possible Solution

Land only after maintainers explicitly accept the default-off config surface and private-text retry boundary; otherwise keep warning-only behavior and track recovery policy separately.

## Solution Assessment

Unclear until maintainer acceptance: the follow-up retry is safer than auto-publishing because it preserves the message-tool send path and default contract, but the new config and private-text replay are product/security choices.
