## What This Changes

Adds an opt-in `messages.strandedReplyRecovery` flag that enqueues one bounded `message(action=send)` follow-up retry for substantive stranded `message_tool_only` finals, with schema/docs, queue isolation, system-prompt guidance, and QA coverage.

## Best Possible Solution

Land a current-head version only if maintainers accept the default-off recovery policy, keeping the one-shot retry, persistence suppression, queue isolation, schema/docs, and QA scenario together.

## Solution Assessment

Yes for implementation shape, pending maintainer policy acceptance. The default-off one-shot retry is safer than suppression bypass and preserves current defaults, but the new config/privacy behavior is a maintainer decision.
