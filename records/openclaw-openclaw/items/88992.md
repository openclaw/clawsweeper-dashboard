## Summary

Keep open: this is a plausible fix candidate for the open message-loss issue, but the new qa-lab recovery scenario still needs a mock-openai guard and the default-off recovery policy remains compatibility/privacy-sensitive maintainer surface.

## What This Changes

The branch adds a default-off `messages.strandedReplyRecovery` config path that can enqueue one non-collected follow-up retry for stranded `message_tool_only` finals, plus docs, prompt, schema, and qa-lab coverage.

## Best Possible Solution

Land a maintainer-accepted, default-off recovery mode with deterministic mock-only QA coverage and clear docs that default `message_tool_only` privacy remains unchanged.

## Solution Assessment

No, not as submitted. The default-off follow-up retry is a plausible bounded solution, but the new deterministic QA scenario needs a mock-only guard and the recovery policy needs explicit maintainer acceptance because current docs intentionally keep such finals private by default.
