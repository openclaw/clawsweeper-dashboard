## Summary

## What This Changes

This PR changes `runReplyAgent` to enqueue a follow-up retry for stranded `message_tool_only` final replies and adds unit coverage for retry, room-event gating, and transcript-field clearing.

## Best Possible Solution

Land this only as a bounded one-shot recovery that preserves tool-only delivery, avoids misleading session persistence, and is proven in a real Telegram or equivalent channel; otherwise keep the current warn-private behavior.

## Reproduction Assessment

Yes at source level: current main detects substantive private finals in `message_tool_only`, and the PR's retry keeps the same mode, so a second missed tool call can re-enter the detector. I did not establish a live channel reproduction in this read-only review.

## Solution Assessment

No, not yet: follow-up retry is a plausible safer shape than suppression bypass, but it needs a one-shot bound, live delivery proof, and maintainer approval for the privacy-sensitive behavior change.
