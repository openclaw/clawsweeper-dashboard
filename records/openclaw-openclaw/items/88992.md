## What This Changes

Adds an opt-in `messages.strandedReplyRecovery` flag that keeps `message_tool_only` unchanged by default and, when enabled, enqueues one non-batched follow-up retry to deliver a stranded assistant final through `message(action=send)`, with docs, schema, unit coverage, and a qa-lab scenario.

## Best Possible Solution

Land the default-off recovery only if maintainers accept the new config and privacy boundary, with current-main checks refreshed; otherwise keep the current warning/private-default behavior.

## Reproduction Assessment

## Solution Assessment

Yes, as an acceptable mitigation: default-off, one-shot retry is safer than bypass-publishing private text and preserves the documented default. The final product choice is whether OpenClaw should add this opt-in config surface at all.
