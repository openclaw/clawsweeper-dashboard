## Summary

The PR addresses a real silent message-loss path, but it is not merge-ready because it changes the `message_tool_only` privacy/delivery contract from warning-only silence to a synthetic retry without live channel proof or updated public docs. The current patch is a plausible direction after the suppression-bypass rewrite, yet maintainers still need to accept the policy change and see exact-once Telegram or equivalent real behavior proof.

## What This Changes

The branch changes `runReplyAgent` to enqueue a follow-up retry when a substantive `message_tool_only` final message is detected without a successful message-tool send, skips that detection for room events, and adds mocked retry/gate tests.

## Best Possible Solution

Land only after maintainers accept the retry policy, the documented tool-only reply contract is updated, and a redacted Telegram or equivalent live run proves exact-once delivery through `message(action=send)` with room-event silence unchanged.
