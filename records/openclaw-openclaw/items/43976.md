## Summary

Keep open. Current main has adjacent `/agents`, `/focus`, session-binding, route-binding, TUI, and Control UI agent switching support, but not the requested channel-chat `/agent <name>` workflow for temporarily switching a direct conversation's configured Agent without editing config or restarting Gateway.

## What This Changes

The issue requests `/agents` and `/agent <name>` chat commands to temporarily switch the current channel conversation to another configured Agent.

## Best Possible Solution

Define a session-level configured-Agent override or conversation-binding path that reuses existing routing primitives, is gated like other control commands, documents its relationship to `/focus` and `sessions_spawn`, and adds direct-chat precedence coverage.

## Solution Assessment

Unclear. A dedicated `/agent <name>` workflow may be the right UX, but coding it safely requires a maintainer decision on routing precedence, authorization, persistence, and how it differs from existing focus/session binding behavior.
