## Summary

Keep open. Current main has adjacent TUI `/agent`, Gateway `agents.list`, route bindings, and channel `/agents`/`/focus`, but it still lacks the requested channel-chat `/agent <name>` flow for temporarily switching a direct conversation to another configured Agent without editing bindings or restarting Gateway.

## What This Changes

The issue requests channel-chat commands for session-scoped switching between configured Agents without restarting the Gateway or changing global bindings.

## Best Possible Solution

Define a session-level configured-Agent override or conversation-binding path that reuses existing routing/session primitives, documents precedence versus `/focus`, TUI `/agent`, `sessions_spawn`, and bindings, and preserves per-agent memory isolation.
