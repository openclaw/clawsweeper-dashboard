## Summary

Keep open. Current main still has static configured-agent bindings, TUI-side agent switching, and thread-bound subagent/ACP focus, but not a live chat `/agent <id>` route override for configured agents; that feature needs explicit product and security policy.

## What This Changes

Request adds chat commands to list configured agents and temporarily switch the current conversation's bound agent without restarting Gateway or mutating global bindings.

## Best Possible Solution

Add an authorized per-conversation configured-agent route override that expires or resets cleanly and preserves each target agent's workspace, auth, tools, sandbox, and memory boundaries.

## Reproduction Assessment

Not applicable as a runtime bug; this is a missing feature. Source inspection shows current chat commands only expose `/agents` for thread-bound sessions and `/focus` for subagent/ACP session targets, while inbound routing still falls through static bindings.

## Solution Assessment

Unclear until maintainers choose the command namespace and authorization policy. A per-conversation override before static binding fallback looks narrower than hot-editing global `bindings`, but it needs allowed-agent, reset/expiry, audit, and isolation rules.
