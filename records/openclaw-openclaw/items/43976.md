## Summary

Keep open. Current main has related `/agents`, `/focus`, `sessions_spawn`, and static route binding support, but it does not provide a session-scoped `/agent <name>` chat command that temporarily switches a direct conversation's active agent without changing config or restarting Gateway.

## What This Changes

Request session-scoped chat commands to list configured agents and temporarily switch the current channel conversation to another agent without restarting Gateway.

## Best Possible Solution

Design a session-level agent override or conversation-binding path that reuses existing routing and binding primitives, gates it like other control commands, documents its relationship to `/focus` and `sessions_spawn`, and adds direct-chat precedence tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows no existing `/agent <name>` switch while adjacent `/agents` and `/focus` behavior is already covered by tests.

## Solution Assessment

Unclear. A dedicated switch command may be the right user workflow, but it should be designed against the existing `/focus`, subagent, session binding, and route precedence model before coding.
