## Summary

Keep open: current main and v2026.6.8 still lack the requested `/agent <id>` configured-agent switch. Adjacent `/focus` and ACP current-conversation binding route to existing session targets, but they do not list or switch configured agents for a direct chat without product and security decisions.

## What This Changes

Request to add chat commands that list configured agents and temporarily switch one conversation's active agent without restarting Gateway.

## Best Possible Solution

Design an authorization-aware conversation-level agent routing override over canonical session or binding state, reusing existing current-conversation binding semantics where possible and keeping workspace, memory, and credential boundaries explicit.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main does not provide a direct `/agent <id>` configured-agent switch for channel conversations.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer direction is an authorization-aware conversation-routing design, not hot-editing global `bindings[]` or adding an unowned override file.
