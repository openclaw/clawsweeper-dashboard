## Summary

Keep open. Current main still has adjacent `/agents`, `/focus`, TUI agent switching, `sessions_spawn`, conversation binding, and static route binding support, but it does not implement the requested channel-chat `/agent <name>` workflow for temporarily switching a direct conversation's configured agent without editing config or restarting Gateway.

## What This Changes

Request a channel chat command to list configured agents and temporarily switch the current direct conversation to a selected agent at session scope without restarting Gateway.

## Best Possible Solution

Design a session-level configured-agent override or conversation-binding path that reuses existing routing primitives, is gated like other control commands, documents its relationship to `/focus` and `sessions_spawn`, and adds direct-chat precedence tests.

## Solution Assessment
