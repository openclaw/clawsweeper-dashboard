## Summary

Keep open. Current main and the latest shipped release still have adjacent `/agents`, `/focus`, `sessions_spawn.agentId`, TUI-only `/agent <id>`, and config route bindings, but not the requested channel-chat `/agent <name>` workflow for temporarily switching a direct conversation to another configured Agent without editing bindings or restarting Gateway.

## What This Changes

Request a chat command workflow to list configured Agents and temporarily switch a channel conversation's active Agent without editing bindings or restarting Gateway.

## Best Possible Solution

Define a session-level configured-Agent override or conversation-binding path that reuses existing routing primitives, is gated like other control commands, documents its relationship to `/focus` and `sessions_spawn`, and adds direct-chat precedence coverage.

## Solution Assessment
