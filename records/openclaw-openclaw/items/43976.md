## Summary

Keep open: current main and v2026.6.6 still provide static configured route bindings plus thread/session-focused `/agents` and `/focus`, but not a direct-chat `/agent <id>` command that temporarily switches the active configured agent. ## What This Changes

Request a channel command flow that lists configured agents and temporarily switches the current direct conversation to another agent without editing `bindings[]` or restarting Gateway.

## Best Possible Solution

Design an authorization-aware per-conversation active-agent state path that reuses routing/session-binding primitives, preserves per-agent workspace and memory isolation, and avoids hot-editing global `bindings[]`.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer direction is a generic, authorization-aware conversation/session routing design, not chat-driven mutation of global route bindings or a new JSON override store.
