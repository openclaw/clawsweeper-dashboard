## Summary

Keep open. Current main and the latest shipped release still provide adjacent `/agents`, `/focus`, TUI/Web agent selection, and config bindings, but not the requested channel-chat `/agent <name>` workflow for temporarily switching a direct conversation to another configured Agent without restarting Gateway.

## What This Changes

Request a channel-chat command flow to list configured Agents and temporarily switch the current conversation's Agent binding at session scope.

## Best Possible Solution

Define a session-level configured-Agent override or conversation-binding path that reuses existing routing primitives, is explicitly authorized, documents its relationship to `/focus` and `sessions_spawn`, and adds direct-chat precedence coverage.

## Solution Assessment

Unclear. A dedicated `/agent <name>` workflow may be the right UX, but the maintainable path needs a product decision on routing precedence, authorization, persistence, rollback, and interaction with existing focus/session binding behavior.
