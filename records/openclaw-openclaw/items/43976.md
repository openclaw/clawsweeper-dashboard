## Summary

Keep open: current main and v2026.6.6 still provide static configured routing plus thread/subagent-focused `/focus`, `/unfocus`, and `/agents`, but not a direct-chat `/agent <name>` command that temporarily switches the active configured agent. The request remains a valid feature idea, but it crosses session-state and security boundaries and needs maintainer product/security direction before implementation.

## What This Changes

Add chat commands to list configured agents and temporarily switch a direct conversation's active agent without restarting Gateway or mutating global bindings.

## Best Possible Solution

Design an authorization-aware per-conversation active-agent state path that reuses routing/session-binding primitives, preserves per-agent workspace and memory isolation, and avoids hot-editing global `bindings[]` or adding ad hoc JSON runtime state.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current channel commands do not provide `/agent <name>` active-agent switching for a direct conversation.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer direction is a generic, authorization-aware conversation/session routing design, not chat-driven mutation of global route bindings or a new JSON override store.
