## Summary

Keep open. Current main still does not provide a live channel-chat `/agent <id>` command or per-conversation configured-agent override; the existing channel slash command surface only lists thread-bound subagents/ACP bindings, so this remains a product and security design request.

## What This Changes

The issue requests slash commands to list configured agents and temporarily switch a single chat conversation to another configured agent without restarting the Gateway.

## Best Possible Solution

Add an authorized per-conversation configured-agent route override before static binding fallback, with agent discovery, reset/expiry, auditability, and preserved workspace/auth/tool/memory isolation.

## Reproduction Assessment

Not applicable as a runtime bug; this is a missing feature. The absence is high-confidence from the command registry, `/agents` handler, `/focus` target resolver, route resolver, and docs inspection on current main.

## Solution Assessment

Unclear until maintainers decide command namespace and security policy. A session-level route override is narrower than hot-editing global `bindings`, but it still needs explicit authorization and isolation rules.
