## Summary

Keep open. Current main has multi-agent route bindings, per-conversation `/focus`, and `/agents` for thread-bound session work, but it still does not provide the requested direct-chat `/agent <name>` temporary switch among configured agents without editing bindings or restarting the Gateway.

## What This Changes

The issue requests channel-chat commands to list configured agents and temporarily switch the current conversation to another agent at session scope without restarting the Gateway.

## Best Possible Solution

Define a generic per-conversation configured-agent override that reuses routing/session primitives, documents precedence, and preserves per-agent stores and memory isolation.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows no current channel-chat `/agent <name>` command contract to reproduce as broken.

## Solution Assessment

Unclear. The requested UX may be right, but maintainers need to choose whether the durable design is a route override, conversation binding, spawned/proxy session, or live binding update.
