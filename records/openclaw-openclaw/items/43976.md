## Summary

Keep open. Current main and the latest shipped tag still provide configured multi-agent bindings, `/focus`/`/unfocus`, and `/agents` for thread-bound subagents/session bindings, but they do not provide the requested channel-chat `/agent <name>` temporary configured-agent switch without changing bindings or restarting the Gateway.

## What This Changes

The issue requests a channel-chat command that temporarily switches the current conversation to another configured agent without restarting the Gateway or changing global bindings.

## Best Possible Solution

Define a generic per-conversation configured-agent override that reuses routing/session primitives, documents precedence, and preserves per-agent stores, auth boundaries, and memory isolation.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms there is no existing channel-chat `/agent <name>` command contract to reproduce as broken.

## Solution Assessment

Unclear. The requested UX is plausible, but maintainers need to choose whether the durable design is a route override, conversation binding, spawned/proxy session, or live binding update.
