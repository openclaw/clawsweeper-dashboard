## Summary

Keep open. Current main has static configured-agent routing, subagent/ACP focus, and local TUI/CLI agent selection, but it still lacks a chat-time configured-agent override for live channel messages; the remaining work needs product and security policy.

## What This Changes

The issue requests live chat commands to list configured agents and temporarily switch the current conversation's agent without restarting Gateway.

## Best Possible Solution

Add an owner-authorized per-conversation configured-agent override tier with discovery, reset, expiry/audit behavior, and strict workspace/auth/tool/memory isolation.

## Reproduction Assessment

Not applicable as a runtime bug; this is a missing feature. Source and docs checks show chat `/agents` is subagent/session-binding oriented and no `/agent <id>` configured-agent override is registered.

## Solution Assessment

Unclear until maintainers decide command namespace and authorization policy. A session-level route override is narrower than hot-editing global bindings, but it still needs explicit security rules and isolation semantics.
