## Summary

Keep open. Current main has multi-agent route bindings, thread/conversation `/focus`, and `/agents` for bound subagent/session work, but it still does not provide the requested direct-chat `/agent <name>` temporary switch across configured agents without changing bindings or restarting the Gateway.

## What This Changes

Request a channel chat command to list configured agents and temporarily switch the current conversation to another configured agent without restarting the Gateway.

## Best Possible Solution

Define a generic per-conversation configured-agent override that reuses routing/session primitives, documents precedence, and preserves per-agent stores, auth boundaries, and memory isolation.
