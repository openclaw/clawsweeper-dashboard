## Summary

Keep open. Current main still supports static configured-agent bindings plus thread-bound subagent/ACP focus, but not a `/agent <id>` configured-agent override for a live chat; adding that route switch needs explicit product and security policy.

## What This Changes

Request adding chat commands to list configured agents and temporarily switch the current conversation to another configured agent without restarting the Gateway.

## Best Possible Solution

Add an authorized per-conversation configured-agent route override before static binding fallback, with agent discovery, reset/expiry, auditability, and preserved workspace/auth/tool/memory isolation.

## Reproduction Assessment

Not applicable as a runtime bug; this is a missing feature. The absence is high-confidence from the command registry, `/agents` handler, `/focus` target resolver, route resolver, and docs inspection.

## Solution Assessment

Unclear until maintainers decide command namespace and security policy. A session-level route override is narrower and safer than hot-editing global `bindings`, but it still needs explicit authorization and isolation rules.
