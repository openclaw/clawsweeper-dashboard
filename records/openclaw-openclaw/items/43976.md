## Summary

Keep open. Current main and the latest shipped release still support configured multi-agent routing, CLI/UI/TUI agent selection, and channel `/focus` plus `/agents`, but not the requested channel-chat `/agent <name>` temporary switch for one conversation.

## What This Changes

Request a channel chat command that temporarily switches the current conversation to another configured Agent without editing bindings or restarting the Gateway.

## Best Possible Solution

Define a generic per-conversation configured-agent override that documents routing precedence and preserves per-agent session stores, memory, and auth boundaries.
