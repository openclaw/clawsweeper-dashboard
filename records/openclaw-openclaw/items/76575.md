## Summary

Keep open. Current main has a generic Telegram document download path, but the reported symptom still matches a shipped path where Telegram `getFile` can return `null` for oversized Bot API downloads and the turn then proceeds without `MediaPaths`; the issue needs PPTX size and log evidence before safe cleanup or repair.

## What This Changes

Telegram direct-chat PPTX uploads should arrive as document media attachments like PDFs instead of being delivered as text-only turns.

## Best Possible Solution

Keep this issue open until size and logs identify the failing path; then either fix Telegram document download or warning behavior, or document the Bot API oversized-file limitation while tracking document text extraction separately in #12047.

## Reproduction Assessment

Unclear for the exact reported PPTX because the issue does not include file size or Telegram logs. There is a source-reproducible current-main path for the same symptom: Telegram `getFile` `file is too big` returns `null`, and the handler continues without `MediaPaths`.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on distinguishing a true PPTX download regression from Telegram's Bot API limit or the separate Office text-extraction feature tracked by #12047.
