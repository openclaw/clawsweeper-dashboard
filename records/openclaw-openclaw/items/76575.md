## Summary

Keep open. Current main has a generic Telegram document download path and PPTX MIME support, but the report still matches a current source path where Telegram `getFile` can return no attachment for oversized Bot API files, and the issue lacks the PPTX size or Telegram logs needed to choose a narrow fix.

## What This Changes

Telegram direct chats should pass PPTX document uploads through as received attachments, matching PDFs and WhatsApp.

## Best Possible Solution

Keep this open until the reporter or maintainer confirms PPTX size and Telegram logs, then either fix a real Telegram document-download regression or add explicit handling/docs for Telegram Bot API oversized files while leaving Office text extraction to #12047.

## Reproduction Assessment

No high-confidence reproduction for the exact PPTX yet because the issue lacks file size and Telegram logs. Source inspection does show a current-main path where `getFile` returning `file is too big` drops the attachment and continues with a text-only turn.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on distinguishing a true PPTX download failure from Telegram's Bot API size limit or the separate Office text-extraction work tracked by #12047.
