## Summary

Keep open. Current main has a generic Telegram document download path, but the report still matches a current source path where Telegram `getFile` reports a Bot API size limit and OpenClaw proceeds with a text-only turn, so this needs file-size/log confirmation before cleanup can safely close it.

## What This Changes

Telegram direct-chat PPTX attachments are reported as missing while text, PDFs, and the same PPTX through WhatsApp work.

## Best Possible Solution

Keep this open until the reporter or maintainer confirms file size and logs, then either fix a real Telegram document-download regression or add explicit handling/docs for Telegram Bot API oversized files while leaving Office text extraction to the shared document pipeline.

## Reproduction Assessment

Unclear for the exact reported PPTX because the issue lacks file size and Telegram logs. There is a source-reproducible current-main path for the same symptom: Telegram `getFile` `file is too big` returns `null`, and the handler continues without `MediaPaths`.

## Solution Assessment

Unclear. A narrow fix is appropriate only after distinguishing a true PPTX download bug from the Telegram Bot API size limit or the separate Office text-extraction feature tracked by #12047.
