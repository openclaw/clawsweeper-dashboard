## Summary

Keep open. Current main handles Telegram documents generally and has an unreleased `EPERM` fsync fix for the newer all-file download failure reported in the comments, but the original PPTX report still lacks the file size and Telegram media logs needed to distinguish a real PPTX download regression from Telegram's Bot API size limit.

## What This Changes

The issue asks Telegram direct chats to deliver PPTX document attachments to the agent the same way PDF and WhatsApp attachments are delivered.

## Best Possible Solution

Keep the current-main media-store `EPERM` fix, then confirm the original PPTX size and Telegram logs before adding either explicit Bot API limit handling or a narrow Telegram document-download regression fix; keep Office text extraction tracked separately.
