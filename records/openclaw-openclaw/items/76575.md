## Summary

Keep open. Current main has a generic Telegram document download path and an EPERM media-store fix for the newer all-file failure report, but the original PPTX symptom still lacks the file size/log evidence needed to distinguish a Telegram Bot API size limit from a real PPTX-specific download regression.

Telegram direct chats should receive PPTX document attachments and pass them into the agent context like PDF attachments.

## Best Possible Solution

Keep this open until the reporter or maintainer confirms PPTX size and Telegram logs, then fix the narrow download/limit-handling path while leaving Office text extraction to the separate document-extraction work.
