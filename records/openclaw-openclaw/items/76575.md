## Summary

## What This Changes

Report asks for Telegram direct-chat PPTX uploads to arrive as attachments like PDFs instead of being reduced to text-only turns.

## Best Possible Solution

Keep the generic Telegram document download path and shared media-store `EPERM` tolerance; route any remaining request to read PPTX contents into the separate document-extraction work tracked by #12047.

## Reproduction Assessment

No high-confidence live Telegram reproduction was run in this read-only sweep. Source inspection shows current in-limit document attachments are downloaded and forwarded, and the reported `EPERM` save failure is covered by a current regression test.

## Solution Assessment

Yes. Handling `EPERM` as best-effort in the shared media store is the narrowest maintainable fix for the concrete follow-up failure, and the existing Telegram document path is already MIME-agnostic for PPTX-sized files Telegram can return via `getFile`.
