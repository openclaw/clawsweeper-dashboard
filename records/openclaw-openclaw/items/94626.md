## Summary

Keep open. Current main still has a source-provable `/status` failure path where status generation can reject before a reply payload exists, and latest release v2026.6.8 has the same core shape; no open or merged related PR was found that owns this exact LINE/status reliability work.

## What This Changes

The issue reports intermittent missing LINE responses for `/status`, attributed to status generation errors, lazy dynamic import rejection caching, and LINE reply-token timing.

## Best Possible Solution

Make `/status` return a bounded fallback reply when optional status sections fail, preserve LINE push fallback behavior, and add focused regression coverage for status-builder rejection and LINE reply-token fallback visibility.

## Reproduction Assessment

No for the full live LINE intermittent symptom; this review did not run a LINE webhook roundtrip. Yes for a narrower source-proven path: if `buildStatusText` rejects, `buildStatusReply` and `handleStatusCommand` currently propagate the rejection instead of producing a reply.

## Solution Assessment

Yes, keeping this open is the right outcome. A blanket top-level catch alone would be a useful fallback but the best repair should also handle rejected cached lazy imports or optional status sections deliberately and prove LINE fallback behavior.
