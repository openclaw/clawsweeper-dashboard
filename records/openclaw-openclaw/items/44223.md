The PR changes TUI Ctrl+C handling so Ctrl+C can abort active runs, reset stale exit-confirmation state, recover abort failures, preserve partial assistant text on abort, and add focused tests.

Maintainers should first choose the intended Ctrl+C policy; if Ctrl+C should abort active runs, rebase onto current main, preserve pendingChatRunId abort support, update shortcut docs/help/status copy, retain the partial-output behavior, and add redacted real TUI proof.
