## Summary

Keep open. Current main still has the clear/warn/exit-only Ctrl+C behavior, and this PR is a plausible fix direction, but it changes the documented TUI shortcut contract and conflicts with the still-open hint-only path in #38501/#38502; the docs and changelog gaps also remain blocking before merge.

## What This Changes

The PR changes TUI Ctrl+C handling to abort active runs ahead of draft clear/exit handling, manage pending/stale abort state, preserve partial text on aborted runs, and add focused TUI tests.

## Best Possible Solution

Maintain one explicit TUI shortcut contract: either land this abort-first behavior with matching docs, changelog, and focused validation, or keep Ctrl+C as clear/exit and resolve the confusion through the #38501/#38502 hint-only path.

## Reproduction Assessment

Yes. Source inspection on current main shows Ctrl+C uses only editor text/timing and cannot call abortActive, so with an active run and non-empty draft it takes the clear-input path instead of aborting.

## Solution Assessment

Unclear. The patch is a narrow implementation if maintainers want Ctrl+C to abort active runs, but it is not clearly the best solution until the documented shortcut contract is chosen against the open hint-only alternative.
