## Summary

Keep open. Current main still does not implement the proposed Ctrl+C active-run abort behavior, and the PR remains useful source work, but it is not merge-ready because it lacks real interactive TUI proof, conflicts with newer pending-run abort handling, and changes documented shortcut semantics that need maintainer agreement.

## What This Changes

This PR changes TUI Ctrl+C handling so active runs abort before draft clearing or exit, resets abort/exit state around aborts, preserves partial assistant text on aborted runs, and adds focused unit coverage.

## Best Possible Solution

Decide whether Ctrl+C should become an active-run abort shortcut; if yes, rebase onto current TUI state handling, preserve pending-run abort support, update shortcut/help copy, add regression coverage, and provide redacted real TUI proof.

## Reproduction Assessment

Yes, source-reproducible: current main's Ctrl+C handler only resolves clear/warn/exit while Esc and /abort call abortActive. I did not run a live interactive TUI session in this read-only review.

## Solution Assessment
