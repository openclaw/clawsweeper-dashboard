## Summary

## What This Changes

This PR changes TUI Ctrl+C handling to abort active runs before clear/exit, track pending abort RPCs, adjust abort cleanup, preserve partial text on aborted events, and add focused unit tests.

## Best Possible Solution

Decide whether Ctrl+C should become an abort shortcut; if yes, rebase over pendingChatRunId, update docs/help, and require real TUI proof, otherwise keep Ctrl+C as clear/exit and split the stale-abort and partial-output fixes.

## Solution Assessment

No as written: the stale-abort and partial-output pieces are plausible, but the Ctrl+C shortcut change needs maintainer policy acceptance, docs/help alignment, a rebase over pendingChatRunId, and real behavior proof.
