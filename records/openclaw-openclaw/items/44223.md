## Summary

Keep open: the PR contains useful TUI abort-handling work, especially partial-output preservation, but it is not merge-ready because it changes the documented Ctrl+C shortcut policy, is conflicting against current main's pending-run abort handling, and lacks real interactive TUI proof.

## What This Changes

The PR changes TUI Ctrl+C handling to abort active runs before clearing input, resets abort/exit state around aborts, preserves partial assistant text on aborted events, and adds focused TUI unit tests.

## Best Possible Solution

Preserve the current Esc and /abort abort contract unless maintainers explicitly approve Ctrl+C abort semantics; if approved, rebase over current main, update docs/help copy, keep pendingChatRunId behavior, and provide real TUI proof.

## Reproduction Assessment

## Solution Assessment

No, not as-is: the partial-output preservation is useful, but changing Ctrl+C into an abort shortcut needs explicit maintainer policy, docs/help alignment, current-main rebase, and real behavior proof.
