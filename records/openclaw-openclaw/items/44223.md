## Summary

Keep open. Current main still uses Ctrl+C only for clear/warn/exit behavior, so the PR is not obsolete, but the patch remains blocked by missing user-facing shortcut documentation and changelog coverage, plus a maintainer decision on whether Ctrl+C should become abort-first instead of the hint-only contract tracked by related TUI items.

## What This Changes

The PR changes TUI Ctrl+C handling to abort active runs before draft clear or exit handling, clear stale/pending abort state, preserve partial assistant text on aborted runs, and add focused regression tests.

## Best Possible Solution

Maintain one explicit TUI shortcut contract. If maintainers want Ctrl+C to abort active runs, land this PR or an equivalent patch with docs, changelog, and focused TUI validation; if not, keep Ctrl+C as clear/exit and resolve the confusion through the active-run hint path in #38501/#38502.

## Reproduction Assessment

Yes. Current main gives a source-level reproduction path: with an active run and non-empty draft, Ctrl+C resolves using only editor text/timing and takes the clear-input branch instead of aborting the run.

## Solution Assessment

Unclear. The PR is a narrow implementation if abort-first Ctrl+C is the intended contract, but it is not clearly the best product solution until maintainers choose between this behavior and the hint-only alternative.
