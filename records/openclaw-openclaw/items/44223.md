Overall correctness: patch is incorrect

Full review comments:

- **[P1] Do not silently change the Ctrl+C shortcut contract:** `src/tui/tui.ts:306-310`
  - body: Current docs say Esc aborts active runs and Ctrl+C clears input or exits.   - confidence: 0.9

  - body: Current main can abort `pendingChatRunId` before the first chat event arrives, but this PR's Ctrl+C decision only checks `activeChatRunId`. If Ctrl+C becomes an abort shortcut, the rebase needs to preserve the pending-run path so it does not miss or regress the newer waiting-gap fix used by Esc and `/abort`.
  - confidence: 0.87
