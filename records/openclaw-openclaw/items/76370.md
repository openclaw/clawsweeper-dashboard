## Summary

Keep open: current main still exposes and consumes model-supplied exec policy fields, and this PR is the active focused fix. The diff matches the reported boundary issue and I found no blocking review findings.

## What This Changes

The PR removes exec policy/routing/elevation fields from the model-facing exec schema, ignores those fields at dispatch, updates trusted chat-command call sites to rely on exec defaults, and adds docs/tests/changelog coverage.

## Best Possible Solution

Land this PR or an equivalent boundary split that leaves only command/runtime inputs model-visible and keeps policy, routing, elevation, and node selection sourced from config or session defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows the exec schema exposes the reported fields and the dispatch path consumes them into routing, security, approval, elevation, and node selection.

## Solution Assessment

Yes. Removing the fields from the model schema and ignoring any supplied policy fields at execution is the narrowest maintainable fix, while trusted commands keep policy through exec defaults.
