## Summary

Keep open: this is the active focused fix for a valid exec policy-boundary bug, and the diff looks directionally correct, but it should get maintainer review and exact-head validation before merge because it changes security-sensitive exec behavior.

## What This Changes

The branch removes model-visible exec policy fields, ignores supplied exec policy args at dispatch, moves trusted chat-command policy to exec defaults, and updates regression coverage, docs, and changelog.

## Best Possible Solution

Land this or an equivalent model/operator exec split after maintainer review, keeping exec policy sourced from config, session defaults, approvals, and trusted command defaults while model args remain limited to command/runtime inputs.

Yes. Source inspection on current main shows the model-visible exec schema exposes `security`/`ask`/`host`/`elevated`/`node` and dispatch consumes those fields; #75811 adds deterministic vLLM reproduction for `security: "deny"`.

## Solution Assessment

Yes. Removing those fields from the model schema and ignoring supplied policy args at dispatch is the narrowest maintainable fix, while trusted diagnostics/export/bash policy remains in exec defaults.
