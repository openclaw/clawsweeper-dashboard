## Summary

Keep open for normal PR handling: current main still exposes and consumes model-supplied exec policy fields, while this PR narrows that boundary, includes focused regression coverage, and has no actionable review findings from the diff or CI evidence.

## What This Changes

The PR removes exec policy controls from the model-facing schema, ignores those fields in exec tool-call args, moves trusted command policy through exec defaults, and updates docs, changelog, and tests.

## Best Possible Solution

Land this PR after normal maintainer review so configured exec defaults and session defaults remain the only source for policy controls, closing #75811 when merged.
