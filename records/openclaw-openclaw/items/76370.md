## Summary

Keep open: the PR is aimed at a real exec policy-boundary problem, but the current patch is not safe to merge because it ignores policy fields only at execute time while the pre-exec prepare/finalize path can still resolve hook state from raw model-supplied host/elevated args. It also lacks real behavior proof and is currently conflicting with the base branch.

## What This Changes

The PR removes exec policy fields from the model-facing schema and execute args, updates trusted chat command exec-call params, docs, changelog, and regression tests.

## Best Possible Solution

Normalize operator-only exec fields at the first model-argument boundary so hooks, finalize, and execute all use configured defaults and trusted command/session policy rather than raw model args.

## Reproduction Assessment

Yes. Current main still exposes the exec policy fields in execSchema, and the PR's source diff leaves the pre-exec prepare/finalize host resolver reading raw host/elevated args before execute ignores them.

## Solution Assessment

No. The right direction is narrow and maintainable, but this patch needs to normalize or strip operator-only fields before prepare/finalize hooks as well as execute, and maintainers still need to accept the compatibility impact of removing per-call routing/elevation.
