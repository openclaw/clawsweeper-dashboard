## Summary

Keep open: current main still exposes and consumes model-supplied exec policy controls, while the PR is a narrow, apparently correct fix path for #75811. It should not be closed as cleanup because it is not implemented on main yet, and the GitHub API currently reports the branch as unmergeable against the base.

## What This Changes

The PR removes model-facing exec policy fields and changes exec dispatch, trusted chat-command calls, tests, docs, and changelog so exec policy comes from configured defaults rather than tool-call args.

## Best Possible Solution

Land this or an equivalent model/operator exec split after rebase and exact-head validation, keeping exec policy controlled by config, session defaults, approvals, and trusted command defaults while model args stay limited to command/runtime inputs.

## Reproduction Assessment

Yes. Source inspection on current main shows the model-visible schema exposes `security`/`ask`/`host`/`elevated`/`node` and dispatch consumes those fields; #75811 also provides a deterministic vLLM case where `security: "deny"` self-denies exec.

## Solution Assessment

Yes. Removing policy controls from the model schema and ignoring any supplied policy args at dispatch is the narrowest maintainable fix, while trusted diagnostics, trajectory export, and `/bash` policy continue through exec defaults.
