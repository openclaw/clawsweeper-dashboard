## Summary

## What This Changes

The PR removes `security`, `ask`, `host`, `elevated`, and `node` from the exec model schema, stops reading those fields from exec tool-call args, and updates related docs and tests.

## Best Possible Solution

Split the provider-visible exec schema from the trusted runtime exec argument contract so models cannot see policy controls while internal defaults, hooks, command helpers, node routing, elevation, and approval hardening continue to work.

## Solution Assessment

No. Hiding policy controls from model definitions is the right direction, but this patch also removes trusted runtime args; the safer fix is a separate model schema projection while keeping internal exec controls.
