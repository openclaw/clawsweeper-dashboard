## Summary

Keep open: current main and latest release still contain the relay behavior implicated by the reports, and https://github.com/openclaw/openclaw/pull/94515 is an open closing PR rather than a merged fix. The later field comments add credible load evidence that the merged lifetime-bound fix did not resolve the CPU/memory fan-out mode.

## What This Changes

The issue reports Codex native PreToolUse hook relay fan-out spawning heavyweight `openclaw-hooks` processes that starve gateway responsiveness and can OOM small hosts.

## Best Possible Solution

Review and either land or revise https://github.com/openclaw/openclaw/pull/94515, then decide whether a follow-up is needed for global relay concurrency or a lightweight relay path.

## Reproduction Assessment

Yes for a source-level reproduction path: current main still enables Codex native relay, keeps PreToolUse active for implicit loop detection, and Codex spawns command hooks as subprocesses. I did not run the reported multi-agent live load scenario, so this is not marked fully reproduced.

## Solution Assessment

Unclear as a complete fix: omitting idle PreToolUse relays is a narrow and likely useful repair, but maintainer review should decide whether the remaining field evidence also requires OS-level concurrency gating or a lighter relay executable.
