## Summary

Keep open: current main still enables Codex native hook relay and can install synchronous PreToolUse command hooks that Codex runs as separate child processes; the later relay-lifetime fix only bounds stuck children and does not remove the per-tool-call fan-out reported here.

## What This Changes

The issue reports Codex native PreToolUse relay process fan-out causing CPU and memory pressure that stalls gateway RPC during tool-call bursts.

## Best Possible Solution

Make the Codex native relay install or invoke heavyweight command hooks only when current configuration and registered hook policy prove there is real local work, while preserving explicit trusted-policy safety behavior.

## Reproduction Assessment

Yes by source path, but not by a fresh live run in this review. Current main still enables Codex native relay, can install synchronous PreToolUse command hooks, and Codex executes selected command hooks as child processes; the comments add live OOM/load evidence on shipped builds.

## Solution Assessment

No for the blunt workaround of disabling native relay, because that drops useful hook safety behavior. The best fix needs a maintainer-chosen cheap, bounded, or resident relay path for loop-detection/no-op cases while preserving real before-tool policy enforcement.
