## Summary

Keep open: current main still has a source-level path where Codex can install synchronous native `PreToolUse` command hooks that spawn the full OpenClaw relay CLI per selected tool event, and the recent merged lifetime fix only bounds stuck children rather than removing the short-lived CPU/RSS fan-out described in the latest comments.

## What This Changes

The issue asks OpenClaw to prevent Codex native hook relay process fan-out from stalling gateway RPC or OOM-restarting small hosts during tool-call bursts.

## Best Possible Solution

Keep this open until a maintainer-approved Codex native relay change preserves hook safety while avoiding full OpenClaw CLI process fan-out for high-frequency or loop-detection-only tool events.

## Reproduction Assessment

Yes at source level, with strong user-provided live load evidence: current main still installs process-backed Codex hook commands for selected native events, and comments report repeatable short-lived relay swarms. I did not reproduce the full multi-agent OOM path locally in this read-only pass.

## Solution Assessment

Unclear as an implementation direction: disabling native hook relay is an effective workaround but not the best product fix because it drops safety and plugin behavior. The likely best fix is a slim/resident or more selective relay path that keeps PreToolUse policy semantics without full CLI fan-out.
