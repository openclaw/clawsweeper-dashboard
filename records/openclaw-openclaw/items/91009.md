## Summary

Keep open: the current main branch added a relay lifetime deadline, but it still hard-enables Codex native hook relay and can still install synchronous PreToolUse command hooks for loop-detection work, so the reported CPU-bound per-tool child-process fan-out is not solved or superseded.

## What This Changes

The issue reports Codex native PreToolUse relay subprocess fan-out causing gateway RPC stalls, high CPU, and OOM/crash-loop behavior under tool bursts.

## Best Possible Solution

Keep this open until Codex-native PreToolUse safety no longer pays full OpenClaw CLI startup per tool event, likely through an in-process or resident relay path with bounded concurrency, diagnostics, and regression proof for gateway responsiveness under tool bursts.

## Reproduction Assessment

Yes for source-level reproduction: current main still registers synchronous Codex PreToolUse command hooks when loop detection may run, and upstream Codex executes matching command hooks as child processes before tools. I did not run a fresh multi-agent live repro in this read-only review, but the issue discussion includes independent live CPU and OOM evidence.

## Solution Assessment

Unclear as a final implementation choice: the merged lifetime deadline is useful for stuck relays, but it is not the best complete solution for CPU-bound fan-out because it leaves full CLI subprocess startup on the per-tool path. Maintainers should choose the safety boundary before a fix changes PreToolUse registration, loop detection placement, or fail-open/fail-closed behavior.
