## What This Changes

The PR filters unsafe subagent announce output, adds Telegram thread-binding public placement metadata, and resolves persistent thread-bound session delivery through session bindings when hooks omit a direct delivery origin.

## Best Possible Solution

Make Telegram's runtime `conversationBindings` and `thread-binding-api.ts` report the same child placement, add coverage for both loaded-plugin and public-artifact paths, keep the unsafe-output filter, and require redacted real Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection is enough: `supportsAutomaticThreadBindingSpawn` reads loaded plugin placement first, and the loaded Telegram plugin still reports `current`, so the new `child` artifact is ignored after Telegram is loaded.

## Solution Assessment

No. The unsafe-output filtering direction is good, but the Telegram spawn-policy fix needs one consistent plugin-owned placement contract across loaded and public-artifact paths plus real Telegram proof.
