## Summary

Keep this open. Current main still has the core failure window described in the report: TUI reconnect performs an immediate history load and has active-run watchdog recovery, but it does not backfill on sequence gaps or subscribe to persisted `session.message` transcript updates for the currently watched session, while the Web UI does handle that event path.

## What This Changes

The issue requests TUI reconnect/backfill behavior so a post-gateway-restart heartbeat recovery reply visible in Web UI also appears in the watched TUI session.

## Best Possible Solution

Add TUI-owned reconnect/backfill parity for the currently watched session: recover on Gateway sequence gaps and post-reconnect idle windows by using the existing session transcript subscription or a debounced history reload, while deferring reloads during active runs and flushing once the run reaches a terminal state.

## Reproduction Assessment

Yes. A focused reproduction can simulate a Gateway reconnect or sequence gap where the heartbeat reply is persisted after TUI's immediate reconnect `loadHistory()` and no matching live `chat` event reaches TUI; current source shows no backfill path for that case.

## Solution Assessment

No, current main is only a partial solution. The active-run reconnect watchdog from #74224 is useful, but the narrowest complete fix is TUI backfill/subscription handling for sequence gaps and idle post-reconnect transcript updates.

## Repair Work Prompt

Fix the TUI missed-post-restart-heartbeat backfill bug tracked by https://github.com/openclaw/openclaw/issues/68970. Current main reloads history once on reconnect and only recovers tracked active runs; it does not recover sequence gaps or idle post-reconnect transcript writes. Implement the fix in the TUI-owned surface, preferably by subscribing/resubscribing the current session to existing `sessions.messages.subscribe` / `session.message` events or by adding a debounced delayed history reload on reconnect and `onGap`. Resubscribe/cleanup when the TUI session changes. Do not change heartbeat generation, routing policy, or unrelated Gateway semantics unless a small generic client API wrapper is required. Add focused tests for: `onGap` triggers recovery; a `session.message` or delayed post-reconnect write for the current session reloads history when idle; reload is deferred while active and replayed after terminal; other-session transcript events are ignored.
