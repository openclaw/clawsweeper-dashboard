## Summary

Keep open. Current main still has the TUI recovery gap: it does not subscribe to or handle persisted `session.message` transcript events, only does one reconnect history load, and reports sequence gaps without forcing backfill; the open live-transcript PR is related but not merged.

## What This Changes

The issue reports that after a gateway restart, the TUI misses a heartbeat recovery reply that is visible in Web UI.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73402 with TUI active-session transcript subscription/backfill, and include reconnect/gap history recovery for post-restart heartbeat replies.

## Reproduction Assessment

Yes, at source level: simulate a reconnect or event gap where the heartbeat reply is appended after the TUI's immediate reconnect `loadHistory()` and no matching live `chat` event reaches TUI. Current main has no `session.message` handler or gap reload path to recover that append.

## Solution Assessment

Yes, the narrow maintainable solution is to reuse the existing Gateway transcript/session event contracts in the TUI, with active-run deferral and gap/reconnect backfill. The merged reconnect watchdog fix is useful but does not replace transcript subscription/backfill.
