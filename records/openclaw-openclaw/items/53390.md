## Summary

Keep open: current main still routes browser text snapshots through page/root accessibility snapshots with no viewport, scroll-offset, or visible-only contract, so the reported scrolled-page mismatch remains a valid source-level browser behavior issue that needs a maintainer decision on snapshot semantics.

## What This Changes

Requests browser snapshot text output to reflect the current scrolled viewport instead of continuing to return the page/root snapshot content from the top.

## Best Possible Solution

Define a documented snapshot scope contract, then implement viewport-aware text snapshots consistently across the tool schema, action query, route plan, Playwright/CDP/Chrome MCP backends, docs, and long-page regression proof while preserving full-tree access if maintainers want both behaviors.

## Reproduction Assessment

Yes, at source level. The issue gives a concrete long-page scroll-then-snapshot sequence, and current main still carries no viewport or scroll-bounds state before reading page/root accessibility snapshots.

## Solution Assessment

Unclear until maintainers choose the snapshot contract. A backend-only tweak would be incomplete; the maintainable fix needs one documented behavior across all snapshot implementations and a compatibility path if full-tree snapshots remain supported.
