## Summary

Keep open. Current main still lacks a visible-viewport or scroll-state text snapshot contract after browser scrolling, and the related FSB discussion plus https://github.com/openclaw/openclaw/issues/41949 make this a real browser-read design issue rather than an already-fixed defect.

## What This Changes

The issue requests browser snapshots that reflect the currently visible viewport after scrolling instead of returning a full/tree read that can still emphasize top-of-page content.

## Best Possible Solution

Define the browser snapshot/read contract so agents can distinguish the visible viewport after scroll from full-page/ref inspection, then implement that consistently across Playwright, raw CDP, and Chrome MCP paths with docs and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current snapshot paths do not expose viewport or scroll metadata, but the exact after-scroll GitHub Issues output still needs live current-main proof.

## Solution Assessment

Unclear. The requested viewport-aware behavior is useful, but the safest solution needs maintainer direction on changing snapshot semantics versus adding a separate visible-viewport read or metadata while preserving full-tree refs.
