## Summary

Keep open: current main still does not carry scroll or viewport scope into browser text snapshots, so the reported user problem remains unresolved and should not be swept closed.

## What This Changes

The issue requests that `browser snapshot` reflect the currently visible viewport after scrolling instead of returning top-of-page or root page content.

## Best Possible Solution

Define the viewport-aware snapshot contract first, then implement it consistently across the tool schema, client, route plan, Playwright/CDP/Chrome MCP paths, docs, and long-page scroll regression coverage.

## Reproduction Assessment

Yes, at source level. The issue gives a concrete long-page scroll sequence, and current main’s snapshot path has no scroll or viewport state before reading page/root accessibility snapshots; I did not live-run Chromium in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. The safest path is a documented viewport-aware mode or explicit default behavior change with regression coverage, rather than a backend-only tweak in one snapshot implementation.
