## Summary

Keep open. Current main still lacks a viewport-aware browser text snapshot contract: the public schema, action query, route plan, docs, and Playwright/CDP/Chrome MCP snapshot paths expose snapshot format, refs, selector/frame, labels, URLs, and size limits, but no current-viewport or visible-only mode, and the implementations still read page/root accessibility snapshots without viewport filtering.

## What This Changes

The issue requests that browser text snapshots reflect the currently visible viewport after scrolling instead of continuing to show top-of-page content.

## Best Possible Solution

Define and document a viewport-aware browser text snapshot contract, then implement it consistently through the browser tool schema, action query, snapshot route plan, Playwright path, CDP fallback, Chrome MCP path, and CLI/docs. Add a long-page regression that scrolls first and asserts the next text snapshot reflects visible content after the scroll.
