## Summary

Keep open: current main still lacks a viewport-aware text snapshot contract, and the active snapshot paths read page/root accessibility trees rather than a visible viewport slice. A prior enhanced-snapshot PR was closed as the wrong shape, so there is no merged or canonical replacement that resolves this issue.

## What This Changes

The issue reports that browser text snapshots continue to show top-of-page content after a successful scroll action.

## Best Possible Solution

Define one documented viewport-aware snapshot contract, then implement it consistently across the tool schema, route plan, Playwright, CDP, Chrome MCP, docs, and long-page regression coverage while preserving full-tree access if maintainers want both behaviors.

## Reproduction Assessment

Yes at source level. The issue gives a concrete long-page scroll-then-snapshot sequence, and current main still has no viewport or scroll-bounds data before reading page/root accessibility snapshots.

## Solution Assessment

Unclear until maintainers choose the snapshot contract. A backend-only tweak would be incomplete; the maintainable fix needs one documented behavior across all snapshot implementations, with a compatibility path for full-tree snapshots if needed.
