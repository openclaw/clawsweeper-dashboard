## Summary

Keep open. Current main still returns browser text snapshots from the full accessibility/role tree without scroll or viewport metadata, and the issue discussion points to a real contract gap rather than an already-fixed bug; the permanent fix needs maintainer direction on viewport-only reads versus full-tree snapshots with explicit viewport state.

## What This Changes

The issue requests that browser `snapshot` reflect the current viewport after scroll actions instead of presenting top-of-page/full-tree content as if no scroll happened.

## Best Possible Solution

Define the browser read contract so snapshots expose current viewport and scroll state while preserving a deliberate full-tree or raw snapshot path for complete refs.

## Reproduction Assessment

No live high-confidence current-main repro was run. Source inspection gives a medium-confidence reproduction path because default snapshot output is full-tree style and lacks viewport/scroll metadata after an action changes scroll position.

## Solution Assessment

Unclear. The requested direction is useful, but the best fix needs maintainer direction because viewport-only output, full-tree output with viewport metadata, and separate visible-read actions have different automation tradeoffs.
