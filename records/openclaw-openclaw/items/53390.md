The issue requests that browser snapshots reflect the current scrolled viewport instead of returning the initial top-of-page content.

## Best Possible Solution

Define the browser read contract first, then implement either viewport-aware snapshots with scroll metadata or a distinct visible-viewport read while keeping full-tree/raw snapshots explicit.
