## Summary

Keep open. Current main and v2026.6.8 still lack a browser snapshot contract that distinguishes visible viewport state from full-page/full-tree state after scrolling, and the FSB comment adds useful prior art rather than making this cleanup-close material.

## What This Changes

The issue asks `browser snapshot` to reflect the currently visible viewport after scrolling instead of returning top/full-tree content that can look unchanged.

## Best Possible Solution

Define a browser read contract that distinguishes visible-viewport state from full-tree/full-page snapshots, then update Playwright, raw CDP, Chrome MCP, tests, and docs consistently.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence reproduction path because current snapshot helpers build tree snapshots without scroll/viewport state, matching the reported ambiguity after scrolling.

## Solution Assessment

Unclear. The requested behavior is useful, but the best fix depends on a maintainer contract decision: change default snapshot semantics or add a distinct visible-viewport read while preserving full-tree refs.
