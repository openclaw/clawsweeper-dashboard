## Summary

Keep open: current main still routes browser text snapshots through page/root accessibility-tree paths without scroll or viewport state, so the reported behavior is not resolved. A maintainer should choose the snapshot contract before changing default output semantics.

## What This Changes

The issue requests that browser text snapshots reflect the currently visible viewport after scrolling instead of returning top-of-page content.

## Best Possible Solution

Define a viewport-aware snapshot contract, then implement it consistently across the tool schema, client/action query, route plan, Playwright/CDP/Chrome MCP backends, docs, and long-page regression coverage while preserving full-tree access if needed.

## Reproduction Assessment

Yes, at source level. The issue gives a concrete long-page scroll sequence, and current main has no scroll or viewport state before reading page/root accessibility snapshots; I did not run live Chromium.

## Solution Assessment

Unclear until maintainers choose the contract. A backend-only tweak in one path would be incomplete; the safer solution is a documented default behavior change or explicit viewport-aware mode across all snapshot implementations.
