## Summary

Keep open: current main still lacks a viewport-aware browser text snapshot contract, and the snapshot implementations still read page/root accessibility snapshots without scroll or viewport filtering.

## What This Changes

Requests that `browser snapshot` reflect the current viewport after scrolling instead of returning initial top-of-page content.

## Best Possible Solution

Define and document a viewport-aware browser text snapshot contract, then implement it consistently through the tool schema, client, route plan, Playwright/CDP/Chrome MCP paths, CLI/docs, and long-page scroll regression coverage.

## Reproduction Assessment

Yes. The issue gives a concrete long-page scroll sequence, and current main’s snapshot source path has no viewport or scroll scope before reading page/root accessibility snapshots; I did not live-run Chromium in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. The safest direction is a documented viewport-aware mode or explicit default change with regression coverage rather than an isolated implementation tweak in one snapshot backend.
