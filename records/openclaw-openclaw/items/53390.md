## Summary

Keep open: current main still has no viewport-aware text snapshot contract, and the active snapshot paths still read page/root/full accessibility trees rather than the current scrolled viewport.

## What This Changes

The issue asks for the browser `snapshot` action to return content from the currently visible scrolled viewport instead of top-of-page content.

## Best Possible Solution

Define and document the text snapshot scope, then implement viewport-aware behavior consistently across schema, action query, route planning, Playwright, CDP, Chrome MCP, docs, and long-page regression proof while preserving full-tree access if required.

## Reproduction Assessment

Yes, at source level. The issue gives a concrete long-page scroll sequence, and current main still snapshots page/root/full accessibility trees with no viewport or scroll-state input; I did not run live Chromium.

## Solution Assessment

Unclear until maintainers choose the contract. A one-backend tweak would be incomplete; the maintainable fix needs a documented snapshot-scope decision across all browser snapshot implementations.
