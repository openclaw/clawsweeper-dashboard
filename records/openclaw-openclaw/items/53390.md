## Summary

Keep open. Current main still routes browser snapshots through full AI/ARIA/accessibility-tree paths without a defined visible-viewport or scroll-position contract, and the related FSB comment supports treating this as a browser read-contract decision rather than cleanup-close material.

## What This Changes

The issue requests browser snapshot output that reflects the currently visible viewport after page scrolling instead of returning initial top-of-page content.

## Best Possible Solution

Define the browser read contract first: either make snapshots viewport-aware with scroll metadata or add a distinct visible-viewport read while keeping full-tree/raw snapshot behavior explicit, then prove all Playwright, raw CDP, and Chrome MCP paths.

## Reproduction Assessment

No. No high-confidence live reproduction was run, but source inspection shows all normal snapshot paths still omit viewport and scroll metadata and rely on full AI/ARIA/accessibility-tree reads.

## Solution Assessment

Unclear. The requested behavior is useful, but the best fix depends on a maintainer contract choice between changing snapshot semantics and adding a separate visible-viewport read.
