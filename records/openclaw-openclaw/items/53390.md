## What This Changes

The issue asks for browser text snapshots to reflect the current viewport after scrolling instead of returning top/full-page content.

## Best Possible Solution

Define one documented viewport-aware snapshot contract across the tool schema, route plan, Playwright, CDP, Chrome MCP, docs, and long-page regression coverage while preserving full-tree access if maintainers want both behaviors.

## Reproduction Assessment

Yes, at source level: the reported scroll-then-snapshot flow maps to current code paths that read page/root accessibility snapshots without viewport, scroll offset, visible-only, or bounds state. I did not establish a live Chromium trace in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the snapshot contract. A backend-only tweak would be incomplete because the maintainable fix needs consistent behavior across Playwright, CDP, Chrome MCP, the tool schema, docs, and compatibility expectations for full-tree snapshots.
