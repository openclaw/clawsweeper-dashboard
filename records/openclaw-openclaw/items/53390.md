## Summary

Keep open. Current main still treats browser text snapshots as full accessibility or role-tree reads with refs, not a visible-viewport read after scroll, and the FSB discussion reinforces that this is a real browser-read contract decision rather than an already-fixed defect.

## What This Changes

The issue requests browser snapshot output that reflects the scrolled visible viewport, or otherwise exposes viewport and scroll state, after browser act or evaluate scroll actions.

## Best Possible Solution

Define the browser snapshot/read contract so agents can tell what is currently visible after scroll while preserving an explicit full-tree/ref path for complete page inspection.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current snapshots come from full accessibility/role-tree paths without viewport or scroll metadata, which is a medium-confidence reproduction path for the reported ambiguity.

## Solution Assessment

Unclear. The requested viewport-aware behavior is useful, but the best fix needs maintainer choice between changing snapshot output, adding viewport/scroll metadata, or adding a separate visible-viewport read.
