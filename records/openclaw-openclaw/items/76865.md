## Summary

Keep open: current main has several relevant update, plugin-sync, and LaunchAgent safeguards, but there is still no high-confidence integrated macOS dev/source-to-stable cutover proof for the combined v2026.5.2 incident.

## What This Changes

The issue asks that macOS dev/source-to-stable package cutovers keep service wrapper/env state, package root, plugin install metadata, Discord channel loading, and Lossless Claw context-engine registration consistent or fail closed.

## Best Possible Solution

Add or run a targeted macOS package-cutover validation that proves the service wrapper/env files, package root, plugin install index/load paths, Discord channel startup, and selected context engine are consistent after stable cutover, then patch the narrow failing seam if that validation fails.

## Reproduction Assessment

No. The issue contains useful incident logs, but it intentionally avoids rerunning the risky cutover, and the current update-channel-switch smoke is a Linux Docker `--no-restart` lane that does not exercise macOS LaunchAgent, Discord, or Lossless Claw readiness.

## Solution Assessment

Unclear. Current main has the right kinds of safeguards, but the best repair cannot be chosen until an integrated macOS cutover check shows whether the remaining failure is in service-env refresh, package swap, plugin sync, runtime loading, or context-engine activation.
