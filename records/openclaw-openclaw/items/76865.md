## Summary

Keep open: current main has several relevant update/plugin safeguards and an unreleased LaunchAgent recovery fix, but there is not enough end-to-end proof that the fresh v2026.5.2 cutover incident and service-wrapper filename corruption are fully resolved.

## What This Changes

The issue reports that a macOS dev/source to stable package cutover can leave the service wrapper, bundled Discord channel loading, Lossless Claw registration, plugin install metadata, and packaged CLI module state inconsistent.

## Best Possible Solution

Add a targeted package-cutover validation path that exercises dev/source to stable on macOS, verifies launchd wrapper/env state, staged package dist inventory, plugin install index sync, Discord/channel loading, Lossless Claw startup registration, and fails nonzero with recovery guidance on any drift.

## Reproduction Assessment

No. The issue preserves log symptoms from a local v2026.5.2 macOS cutover, but it intentionally avoids rerunning the path and I did not find an end-to-end current-main test that recreates the wrapper plus plugin drift state.

## Solution Assessment

Unclear. Current main has the right safeguard pieces, but the best fix cannot be declared without a targeted package-cutover regression or maintainer validation on macOS.
