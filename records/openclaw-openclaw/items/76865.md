The issue reports that a macOS dev/source-to-stable package cutover can leave service wrapper paths, bundled/plugin channel loading, plugin install metadata, and context-engine registration inconsistent.

## Best Possible Solution

Add or run a targeted macOS package-cutover validation path that proves service wrapper/plist state, package root, plugin install records/load paths, Discord readiness, and the selected context engine remain consistent or fail closed with actionable repair guidance.
