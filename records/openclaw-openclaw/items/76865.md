## Summary

Keep open: current main has relevant staged update, plugin repair, LaunchAgent recovery, and restart health safeguards, but there is no high-confidence proof that the combined macOS dev/source-to-stable cutover incident is fully fixed.

## What This Changes

A dev/source-to-stable package update should keep the service wrapper, plugin/channel load paths, install metadata, context engine registration, and packaged CLI imports consistent or fail closed with repair guidance.

## Best Possible Solution

Add or run a targeted macOS package-cutover validation path that proves wrapper/plist state, package root, plugin install records/load paths, Discord readiness, and the configured context engine remain consistent or fail closed with actionable recovery guidance.

## Reproduction Assessment

No. The report includes concrete v2026.5.2 logs, but there is no current-main repro; the available update-channel Docker lane switches dev/stable with `--no-restart` and does not exercise macOS LaunchAgent service-env wrapper state.

## Solution Assessment

Unclear. Current main has directionally correct safeguards, but the narrowest maintainable fix depends on an integrated macOS cutover run that can isolate whether wrapper generation, plugin sync, restart health, or package import state still fails.
