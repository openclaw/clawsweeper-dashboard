## Summary

Keep open: current main has several relevant safeguards for staged package updates, post-update plugin repair, and macOS LaunchAgent recovery, but I did not find high-confidence end-to-end proof that the reported v2026.5.2 macOS dev/source-to-stable cutover incident is fully fixed.

## What This Changes

The issue reports a macOS arm64 dev/source-to-stable package cutover incident where service env wrapper state, bundled Discord loading, plugin install metadata, Lossless Claw context-engine registration, and packaged dist imports became inconsistent.

## Best Possible Solution

Add or run a targeted macOS package-cutover validation path that proves service wrapper/plist state, package root, plugin install records/load paths, Discord readiness, and the selected context engine remain consistent or fail closed with actionable recovery guidance.

## Reproduction Assessment

No. The issue preserves concrete v2026.5.2 logs, but there is no current-main end-to-end repro, and the existing update-channel Docker lane uses `--no-restart` so it does not exercise macOS LaunchAgent wrapper state.
