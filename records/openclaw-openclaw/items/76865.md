## Summary

Keep open: current main has safeguards for staged package updates, plugin repair, selected context-engine startup, and macOS LaunchAgent recovery, but I did not find integrated proof that the reported macOS dev/source-to-stable cutover with Discord and Lossless Claw now stays consistent end to end.

## What This Changes

Ensure a dev/source-to-stable package cutover leaves the service wrapper, package runtime, plugin install metadata, Discord channel, and selected context engine consistent or fails closed with actionable recovery.

## Best Possible Solution

Add or run a targeted macOS package-cutover validation path that checks service env/plist state, package root, plugin install records/load paths, Discord readiness, and selected context-engine registration, then patch the first failing boundary.

## Reproduction Assessment

No. The issue has concrete v2026.5.2 incident logs, but no current-main reproducer, and the available channel-switch Docker lane uses --no-restart while disabling bundled plugins.

## Solution Assessment

Unclear. The current staged install, plugin repair, context-engine preload, and LaunchAgent recovery paths are directionally correct, but the narrow fix cannot be chosen without an integrated macOS cutover run.
