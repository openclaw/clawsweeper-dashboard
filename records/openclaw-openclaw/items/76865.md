## Summary

Keep open: current main and v2026.5.3 contain several adjacent safeguards for package updates, LaunchAgent recovery, plugin sync, and context-engine startup, but I did not find end-to-end proof that the combined macOS dev/source-to-stable cutover incident is fixed.

## What This Changes

The issue reports that a macOS dev/source-to-stable package cutover can leave the service wrapper, bundled channel/plugin loading, plugin install metadata, context-engine registration, and runtime imports inconsistent.

## Best Possible Solution

Add or run a targeted macOS managed-service cutover validation that proves wrapper/plist state, package root, plugin install records/load paths, Discord readiness, and the selected context engine remain consistent or fail closed with clear repair output.

## Reproduction Assessment

No. The report preserves concrete v2026.5.2 incident logs and source inspection shows related paths, but there is no clean current-main macOS dev/source-to-stable cutover reproduction or passing validation proof.

## Solution Assessment
