## Summary

Keep open: current main still lacks inline provenance metadata for injected prompt context, and the same-author Phase 1 PR #54830 remains open with closing syntax, so this issue should not be closed before that PR is reviewed, merged, or rejected.

## What This Changes

The issue requests source, injection-time, and volatility metadata on injected prompt context segments so agents can reason about freshness and stale context.

## Best Possible Solution

Keep this issue open while maintainers review PR #54830 as the immediate Phase 1 candidate. If accepted, land a focused metadata/tagging implementation first, then split memory timestamps, stronger prompt rules, deterministic refresh, and adjacent tag-registry/security provenance work into narrower follow-ups.
