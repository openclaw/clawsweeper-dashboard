This is a product/API RFC with active linked work, not a cleanup-close candidate.

## What This Changes

The issue requests model-visible provenance and freshness metadata for injected context segments such as bootstrap files, memory results, and eventually orchestration refresh decisions.

## Best Possible Solution

Keep this issue open while maintainers decide the Phase 1 contract in https://github.com/openclaw/openclaw/pull/54830, then split memory freshness, stronger prompt rules, and orchestration refresh into narrower follow-ups or align them with https://github.com/openclaw/openclaw/issues/67552 and https://github.com/openclaw/openclaw/issues/69541.
