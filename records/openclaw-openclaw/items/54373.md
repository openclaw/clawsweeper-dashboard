## Summary

Keep open: current main still lacks the requested inline provenance metadata for injected prompt context, and the same-author Phase 1 PR #54830 is still open and unmerged with closing syntax for this issue. This should remain a maintainer review/product decision rather than being closed or routed to a separate repair lane.

## What This Changes

The RFC requests source, injection-time, and volatility metadata for injected context segments, followed by memory freshness, stronger prompt rules, and deterministic stale-context refresh.

## Best Possible Solution

Keep this RFC open while maintainers review PR #54830 as the immediate Phase 1 candidate. If accepted, land the focused metadata/tagging implementation first, then split memory timestamps, prompt reasoning rules, deterministic refresh, and adjacent tag-registry/security provenance work into narrower follow-ups.
