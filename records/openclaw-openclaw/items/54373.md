## Summary

Keep open. Current main still renders injected bootstrap/project context as flat path/content text without inline source, injected-at, or volatility metadata, and the same-author Phase 1 PR #54830 remains the open implementation candidate with closing syntax, so the issue should not be closed before that PR is reviewed, merged, or rejected.

## What This Changes

Request to add source, injection-time, and volatility provenance metadata to injected context segments, then use that metadata for memory freshness, agent guidance, and eventual deterministic refresh behavior.

## Best Possible Solution

Keep the RFC open while maintainers review PR #54830 as the immediate Phase 1 candidate. If accepted, land the focused metadata/tagging implementation first, then split memory timestamps, stronger prompt rules, deterministic refresh, and adjacent tag-registry/security provenance work into narrower follow-ups.

Not applicable as a feature RFC. Current-main source inspection gives a high-confidence verification path: the embedded context type, bootstrap assembly, prompt renderer, docs, and tests still expose only flat project-context injection.

## Solution Assessment
