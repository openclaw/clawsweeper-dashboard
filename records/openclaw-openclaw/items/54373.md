Keep open: current main and v2026.6.6 still render injected bootstrap context as flat path/content without inline provenance, and the same-author Phase 1 implementation at https://github.com/openclaw/openclaw/pull/54830 remains open with closing syntax, conflicts, and proof blockers.

## What This Changes

The issue requests source, injection-time, and volatility metadata for injected agent context segments so agents can distinguish stale bootstrap or memory context from fresh tool-read content.

## Best Possible Solution

Keep this issue open while maintainers decide the provenance contract and review the linked Phase 1 PR; split memory freshness, stronger prompt rules, and orchestration auto-refresh into narrower follow-ups if Phase 1 lands.
