## Summary

Keep open: current main still renders injected bootstrap/project context as flat path/content text without inline source, injection-time, or volatility metadata, and the same-author Phase 1 implementation PR remains open and unmerged with closing syntax for this issue.

## What This Changes

The RFC requests source, injection-time, and volatility metadata for injected context segments, followed by memory freshness metadata, provenance-aware prompt rules, and deterministic stale-context refresh.

## Best Possible Solution

Keep this issue open while maintainers review PR #54830 as the immediate Phase 1 candidate. If accepted, land focused support for context-file metadata, bootstrap assembly, prompt rendering, and tests, then track memory freshness, stronger prompt rules, and deterministic refresh separately or align them with adjacent provenance/tag-registry RFCs.
