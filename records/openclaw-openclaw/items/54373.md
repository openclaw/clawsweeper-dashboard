## Summary

Keep open: current main and v2026.6.10 still render injected bootstrap context as flat path/content text with no inline source or volatility metadata, and the same-author Phase 1 implementation PR remains open and conflicting. The remaining decision is product-sensitive context/session-state design rather than conservative cleanup.

## What This Changes

The RFC asks OpenClaw to attach source, injected-at, and volatility metadata to injected context segments so agents can reason about freshness.

## Best Possible Solution

Keep this issue as the canonical RFC while maintainers review https://github.com/openclaw/openclaw/pull/54830; if accepted, land a focused Phase 1 and split memory freshness plus deterministic refresh into narrower follow-ups.
