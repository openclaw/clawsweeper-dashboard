## Summary

Keep open: current main and v2026.5.18 still have an inconsistent descriptor boundary. The runnable QA CLI remains gated, but the raw descriptor export leaks the `qa` entry into at least the argv root-command classifier.

## What This Changes

Make the sub-CLI descriptor surface respect the private QA CLI gate so `qa` is not visible when `OPENCLAW_ENABLE_PRIVATE_QA_CLI` is unset.

## Best Possible Solution

Keep a single gated descriptor surface for public/default consumers, make any unfiltered catalog private or explicitly named internal, and add regression coverage for disabled and enabled private QA mode.

## Reproduction Assessment

Yes, from source inspection: with `OPENCLAW_ENABLE_PRIVATE_QA_CLI` unset, `getSubCliEntries()` filters `qa` while `SUB_CLI_DESCRIPTORS` remains raw and is consumed by `argv.ts`. A fix PR should turn that into an executable regression test.

## Solution Assessment

Yes, the recommended direction is the narrow maintainable fix: remove or gate the raw export and update consumers to use the gated accessor unless they truly need a private internal catalog.
