## Summary

Keep open: current main still omits the Discord `deliveryOrigin` needed for direct initial child delivery, and the linked closing PR at https://github.com/openclaw/openclaw/pull/83172 is still open, so this issue should close only after that fix lands.

## What This Changes

Discord thread-bound delegated child sessions should route their initial child reply into the newly bound Discord thread instead of the parent channel.

## Best Possible Solution

Review and land the linked Discord hook fix, then close this issue with the merge commit and release provenance once it ships.

## Reproduction Assessment

Yes. Source inspection on current main shows the Discord hook creates the binding without returning `deliveryOrigin`, while the core spawn path only delivers the initial child run directly when a routable bound origin is present.

## Solution Assessment

Yes. Returning the Discord-owned bound thread origin through the existing spawning-hook contract is the narrow maintainable fix; the open linked PR is the right path to review rather than opening a duplicate repair branch.
