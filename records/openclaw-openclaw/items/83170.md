## Summary

Keep open: current main still matches the Discord routing gap, and the linked closing PR is open and unmerged, so this issue should track review and landing of that fix.

## What This Changes

The issue reports that Discord thread-bound `sessions_spawn(thread=true, mode="session")` binds a child session but sends the initial child reply to the parent channel because the hook omits a routable delivery origin.

## Best Possible Solution

Land a narrow Discord hook fix that returns the created thread as `deliveryOrigin` and extends the hook regression test, then close this issue after the linked PR merges.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the Discord hook creates the thread binding but returns no `deliveryOrigin`, while the generic spawn path only delivers the initial child run directly when that origin is present. I did not run a live Discord E2E in this read-only review.

## Solution Assessment

Yes, the narrow owner-boundary fix is to return the bound Discord thread origin from `subagent_spawning` and test that contract. The open linked PR appears to target that seam, so the issue should stay open until that PR is reviewed and merged.
