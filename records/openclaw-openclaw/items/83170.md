The linked closing PR at https://github.com/openclaw/openclaw/pull/83172 is open and should remain the fix path until it lands.

## What This Changes

Discord thread-bound delegated subagent sessions should route the initial child reply into the bound Discord thread after binding succeeds.

## Best Possible Solution

Land a narrow Discord hook fix that returns the bound thread as deliveryOrigin and extends the Discord hook regression test, then close this issue after merge.

## Solution Assessment

Yes, the narrow owner-boundary fix is to return the created Discord thread target from the Discord subagent_spawning hook and cover that contract in its hook test. A generic spawn refactor would be broader than the observed defect.
