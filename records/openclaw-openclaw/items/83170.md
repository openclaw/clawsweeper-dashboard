## Summary

Keep open: current main still matches the reported Discord routing gap, and the linked closing PR is open and unmerged, so this issue should track review and landing of that fix rather than be closed now.

## What This Changes

Discord thread-bound delegated child sessions are reported to bind a thread but route the initial child reply to the parent channel because the hook does not return a delivery origin.

## Best Possible Solution

Land a narrow Discord hook fix that returns the created thread as deliveryOrigin and extends the hook regression test, then close this issue after merge.

## Reproduction Assessment

Yes, from source: the Discord hook creates/binds the thread but returns no deliveryOrigin, while generic spawn only routes the initial child run to a bound thread when a routable deliveryOrigin is present. I did not run a live Discord E2E in this read-only review.

## Solution Assessment

Yes, the best fix direction is the narrow owner-boundary change in the Discord subagent_spawning hook, not a generic spawn refactor. The linked PR appears to target that seam but should be reviewed and verified before merge.
