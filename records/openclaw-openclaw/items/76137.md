## Summary

Keep open. The patch is directionally right, but it still does not match the normal agent-turn tool surface because it omits sandbox policy and provider/model-scoped policy in an authorization-sensitive path.

## What This Changes

The PR adds policy-pipeline filtering to `command-dispatch: tool` skill slash commands in `src/auto-reply/reply/get-reply-inline-actions.ts`.

## Best Possible Solution

Route inline tool-dispatched skill commands through the same canonical tool-builder or a shared effective-policy helper used by normal turns, including runtime policy session selection, sandbox context, target-session group metadata, sender identity, provider/model policy, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure a restricted profile, provider policy, sandbox policy, or leaf subagent, add a user-invocable tool-dispatch skill for a denied tool such as `sessions_spawn`, and compare the inline path with the normal effective tool list.

## Solution Assessment

No. The requested direction is correct, but this implementation duplicates only part of the pipeline; reusing `createOpenClawCodingTools(...)` or extracting the shared policy context is the safer maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Thread model-scoped policy into inline dispatch:** `src/auto-reply/reply/get-reply-inline-actions.ts:316-320`
  - body: Normal agent turns call `resolveEffectiveToolPolicy` with `modelProvider` and `modelId`, so `tools.byProvider` entries keyed to a provider or provider/model are applied. This new inline path omits the already-resolved `provider` and `model`, so provider-scoped allow/deny/profile policy can still be bypassed by a tool-dispatch skill command.
  - confidence: 0.89
- **[P1] Enforce sandbox policy for inline tool dispatch:** `src/auto-reply/reply/get-reply-inline-actions.ts:347-362`
  - body: The new pipeline stops at group policy plus subagent policy, but normal agent turns also carry `sandbox?.tools` and run a `sandbox tools.allow` step before exposing tools. In a sandbox-restricted session, this branch can still select a tool that the normal turn would remove, so the security bug remains incomplete.
  - confidence: 0.9
