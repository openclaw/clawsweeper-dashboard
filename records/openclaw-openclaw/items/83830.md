## Summary

Keep open: current main and v2026.5.18 still select light Dream Diary inputs from recent recall entries without diary-history context, then append timestamped narratives without semantic duplicate suppression. Related work overlaps, especially https://github.com/openclaw/openclaw/pull/70332 and https://github.com/openclaw/openclaw/issues/72096, but those do not fully close this first-meeting/early-memory dominance report.

## What This Changes

The issue reports that Dream Diary entries repeatedly narrate first-day or first-meeting memories because early short-term recall fragments keep dominating diary generation.

## Best Possible Solution

Refresh or replace the overlapping Dream Diary hardening work with a memory-core fix that carries recent diary/source fingerprints or sweep context into selection/prompting, de-emphasizes already-written snippets without hiding legitimate recurrence, and adds regression coverage for repeated first-meeting-style inputs.

No high-confidence live reproduction is available because the reporter did not include DREAMS.md/store artifacts. Source inspection does show the plausible path: reinforced recall entries can be selected repeatedly, the prompt lacks prior-entry context, and append preserves timestamped repeats.

## Solution Assessment

Unclear for the exact implementation because the issue proposes several fixes. The maintainable direction is a bounded memory-core change that combines recent-entry context or filtering with tests, coordinated with the existing duplicate-body PR rather than a broad scoring rewrite first.
