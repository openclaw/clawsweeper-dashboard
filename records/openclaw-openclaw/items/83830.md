## Summary

Keep open: current main and v2026.5.18 still select light-phase diary snippets by recent recall time then recall count, build the narrative prompt from snippets/themes only, and append diary entries without semantic overlap checks. The report is source-reproducible but still lacks live DREAMS.md/store evidence for the exact repeated “first day” output.

## What This Changes

The issue reports that Dream Diary entries repeatedly reuse first-day memory fragments because short-term recall scoring keeps early memories in the light-phase diary input.

## Best Possible Solution

Make memory-core’s diary pipeline carry sweep context and recent diary/source fingerprints into prompt or selection, de-emphasize already-written snippets carefully, and add regression coverage for repeated diary material.

## Reproduction Assessment

No high-confidence live reproduction was established because the issue has no DREAMS.md, short-term-recall.json, logs, or screenshots. Source inspection does show the reported path: repeated recall entries can remain in the light input set, the prompt lacks temporal/history context, and append does not prevent semantic repetition.

## Solution Assessment

Unclear for the exact implementation because the issue lists several plausible fixes. The maintainable direction is a bounded memory-core change that adds context or recent-entry filtering with tests, while preserving legitimate recurring memories.
