## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve xhigh instead of mapping it to max:** `extensions/acpx/src/runtime.ts:436`
  - body: Claude's SDK and ACP adapter treat `xhigh` and `max` as distinct supported effort values. Mapping xhigh-style aliases to `max` can reject sessions on models that advertise `xhigh` but not `max`, or request a stronger effort than the user selected; return `xhigh` for xhigh/extra-high aliases and reserve `max` for explicit max input.
  - confidence: 0.9
- **[P2] Normalize advertised effort values for Claude ACP:** `extensions/acpx/src/runtime.ts:1008`
  - body: The control plane resolves stored `thinking` to the adapter-advertised `effort` key, and explicit `/config effort ...` reaches this branch as `key === "effort"`. Because this condition skips `effort`, values like `x-high` bypass the normalizer and are forwarded unchanged to an adapter that validates option values.
  - confidence: 0.94
