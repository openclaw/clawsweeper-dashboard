## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve xhigh instead of mapping it to max:** `extensions/acpx/src/runtime.ts:436`
  - body: Claude's SDK and ACP adapter treat `xhigh` and `max` as distinct effort values, and the adapter validates against the active model's advertised values. Returning `max` for `xhigh`/`x-high` can reject models that advertise `xhigh` but not `max`, or silently request a stronger effort than the user chose; map xhigh aliases to `xhigh` and reserve `max` for explicit max input.
  - confidence: 0.9
- **[P2] Normalize advertised effort values for Claude ACP:** `extensions/acpx/src/runtime.ts:1008`
  - body: Current main resolves stored `thinking` through advertised config keys, and explicit native effort updates also call `AcpxRuntime` with `key === "effort"`. This condition excludes `effort`, so aliases like `x-high` bypass the new normalizer and are forwarded unchanged to Claude ACP's exact-value validator; include `effort` and add a regression test.
  - confidence: 0.94
