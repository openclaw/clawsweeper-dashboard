## Summary

Keep open: the PR targets a real current-main Claude ACP config-control failure, but it still has two blocking effort-normalization bugs against the control-plane alias flow and upstream Claude ACP contracts.

## What This Changes

The branch adds Claude ACP command detection and config filtering/remapping in ACPX, expands ACPX runtime coverage, and updates two existing tests.

## Best Possible Solution

Keep the Claude-specific ACPX filtering approach, but normalize every Claude effort entry point while preserving `xhigh` and `max` as separate dependency-supported values.

## Reproduction Assessment

Yes. Current main's ACPX config path forwards non-Codex config controls unchanged, and the published Claude ACP adapter rejects the reported unknown ids or invalid values, so the failure is source-reproducible without a live run.

## Solution Assessment

No, not yet. The PR fixes the right runtime boundary, but it should preserve `xhigh` instead of mapping it to `max` and include the advertised `effort` key in the Claude normalization branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve xhigh when normalizing Claude effort:** `extensions/acpx/src/runtime.ts:440`
  - body: Claude's SDK and ACP adapter treat `xhigh` and `max` as distinct effort values, and the adapter validates against the active model's advertised values. Mapping `xhigh`/`x-high` aliases to `max` can reject sessions on models that advertise `xhigh` but not `max`, or silently request a stronger effort than the user selected. Map xhigh aliases to `xhigh` and reserve `max` for explicit max input.
  - confidence: 0.9
- **[P2] Normalize Claude effort config updates too:** `extensions/acpx/src/runtime.ts:1008`
  - body: The control plane resolves stored `thinking` to the adapter-advertised `effort` key, and explicit `/config effort ...` also reaches this branch as `key === "effort"`. Because this condition skips `effort`, values like `x-high` bypass the normalizer and are forwarded unchanged to an adapter that validates option values. Include `effort` in this path and add a regression test for the advertised-key flow.
  - confidence: 0.94
