## Summary

Keep open: the PR addresses a real Claude ACP config-control bug and has sufficient live proof, but the current branch still mishandles Claude effort aliases by collapsing `xhigh` to `max` and skipping the advertised `effort` key.

## What This Changes

The PR adds Claude ACP command detection and config filtering/remapping in the ACPX runtime, expands runtime coverage, and adjusts two existing test assertions.

## Best Possible Solution

Keep the Claude ACP-specific filtering direction, but preserve `xhigh` as a distinct dependency-supported effort value and normalize both generic thinking aliases and advertised `effort` flows before merge.

## Reproduction Assessment

Yes. Current main forwards generic non-Codex ACP config controls unchanged, and the published Claude ACP adapter rejects unknown config ids and invalid option values, so the reported crash path is source-reproducible without adding new product behavior.

## Solution Assessment

No, not yet. The PR is at the right ACPX runtime boundary, but it should preserve dependency-supported effort values and include `effort` in the Claude normalization branch before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve xhigh instead of mapping it to max:** `extensions/acpx/src/runtime.ts:436`
  - body: Claude SDK exposes `xhigh` and `max` as distinct `EffortLevel` values, and Claude ACP validates against the selected model's advertised effort options. Returning `max` for `xhigh` aliases can reject models that advertise `xhigh` without `max`, or request stronger effort than selected; return `xhigh` for those aliases instead.
  - confidence: 0.9
- **[P2] Normalize advertised effort values for Claude ACP:** `extensions/acpx/src/runtime.ts:1008`
  - body: The control plane resolves stored `thinking` to the adapter-advertised `effort` key, and explicit config changes can also arrive as `key === "effort"`. This condition skips `effort`, so values like `x-high` are forwarded unchanged and Claude ACP can reject them; include `effort` in this normalized path and cover it in tests.
  - confidence: 0.94
