## Risks / Open Questions

- Because repair runs after live debounce classification, users with iMessage DM coalescing enabled can still have anchorless group events delayed or merged under the raw chat_id=0 DM-style key before recovery.
- If two anchorless group payloads from the same sender arrive inside the debounce window, the merged payload can be recovered and routed using only the first GUID, which is a message-delivery and privacy boundary risk.
- The PR intentionally changes unrecoverable malformed payload behavior to fail closed, which can drop messages that current installations would otherwise process as DMs; maintainers should explicitly accept that behavior with live proof.
- The supplied proof is simulated terminal output from a local branching script and does not show the real imsg/OpenClaw watch.subscribe path after the patch.
