## Summary

Keep open. Current main still has unguarded `process.cwd()` calls in the early CLI startup path, and PR #74994 is the open closing implementation candidate rather than a merged fix.

## What This Changes

Handle deleted current working directories gracefully during CLI startup instead of throwing `ENOENT: uv_cwd`.

## Best Possible Solution

Land PR #74994 or an equivalent shared safe-cwd startup guard with regression coverage for main-module detection, CLI/runtime dotenv loading, and required home-dir fallbacks.

## Reproduction Assessment

Yes. The reporter's create, enter, delete, then run path maps directly to current-main `process.cwd()` calls that execute before `openclaw tui` can recover.

## Solution Assessment

Yes, as a direction. A shared guarded cwd helper across the early startup consumers is the narrowest maintainable fix; closing the issue before that lands would drop the tracking bug.
