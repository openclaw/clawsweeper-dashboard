## What This Changes

The issue requests that OpenClaw CLI/TUI startup tolerate a deleted current working directory instead of crashing with `ENOENT: uv_cwd`.

## Best Possible Solution

Keep this issue open until one linked PR lands caller-specific deleted-cwd handling across CLI startup, dotenv, PATH/home fallback, and TUI runtime paths, then close with merge and release provenance.

## Reproduction Assessment

Yes, source-level. A deleted-cwd Node probe reproduced `ENOENT`/`uv_cwd`, and current main still reaches direct `process.cwd()` calls in CLI dotenv and TUI startup/runtime paths.

## Solution Assessment

Yes, the best shape is caller-specific nullable cwd handling rather than a generic HOME/tmp fallback, because PATH, dotenv, home resolution, and TUI spawn cwd have different trust and fallback semantics.
