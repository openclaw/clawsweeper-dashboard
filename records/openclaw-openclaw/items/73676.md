## Summary

Keep open. Current main still has deleted-cwd crash paths in early CLI startup, dotenv loading, and home-dir fallback code, and the linked PR #74994 is open rather than merged.

## What This Changes

Handle OpenClaw CLI startup when launched from a deleted working directory so it does not crash with an unhandled `uv_cwd` error.

## Best Possible Solution

Review and land PR #74994 or an equivalent shared safe-cwd startup fix with regression coverage for main-module detection, CLI/runtime dotenv loading, and required home-dir fallbacks.

## Reproduction Assessment

Yes. The reporter's create/cd/delete/run path maps directly to current-main `process.cwd()` calls that execute during CLI startup and are known to throw when the process cwd has been removed.

## Solution Assessment

Yes, as a direction. A shared guarded cwd helper across the early startup consumers is the narrowest maintainable fix; patching only `shouldLoadCliDotEnv` would leave `isMainModule`, dotenv loaders, and home-dir fallbacks exposed.
