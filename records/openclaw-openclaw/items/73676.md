## Summary

Keep open. Current main still has unguarded `process.cwd()` calls on the CLI/TUI startup path, and the linked closing fix at https://github.com/openclaw/openclaw/pull/74994 is still open rather than merged.

## What This Changes

The issue asks OpenClaw CLI/TUI startup to tolerate a deleted current working directory instead of crashing with `uv_cwd`.

## Best Possible Solution

Land the linked shared safe-cwd startup fix, or an equivalent guarded cwd implementation, with regression coverage across early startup cwd consumers.

## Reproduction Assessment

Yes, source-backed. The reporter's create/cd/delete/run path maps to current-main early `process.cwd()` calls, and a focused Node probe confirms deleted cwd throws `ENOENT`/`uv_cwd`.

## Solution Assessment

Yes as a direction. A shared guarded cwd helper across entry detection, dotenv loading, home-dir fallback, PATH bootstrap, and TUI startup is narrower and more maintainable than patching only one call site.
