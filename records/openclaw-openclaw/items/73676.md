## Summary

Keep open. Current main still has unguarded `process.cwd()` calls on the deleted-cwd startup path, and PR #74994 is an open unmerged closing implementation candidate.

## What This Changes

Handle deleted current working directories during CLI startup so OpenClaw does not crash with `ENOENT: uv_cwd`.

## Best Possible Solution

Review and land PR #74994, or an equivalent shared safe-cwd startup fix with focused regression coverage across the remaining raw-cwd callers.

## Reproduction Assessment

Yes. The reporter's create/cd/delete/run path maps to current-main startup code that still calls `process.cwd()` before command handling and before dotenv loading can recover.

## Solution Assessment

Yes, as a direction. A shared guarded cwd helper across startup consumers is the narrow maintainable fix; patching only `shouldLoadCliDotEnv` would leave other deleted-cwd paths open.
