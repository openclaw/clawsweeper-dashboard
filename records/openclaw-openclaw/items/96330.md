## Summary

Current main and the latest release already handle the reported WebChat/Control UI plugin approval flow: plugin approval events are enqueued, rendered in the shared approval modal, resolved through `plugin.approval.resolve`, and Skill Workshop/before_tool_call approvals now carry the turn-source routing fields needed to stay pending.

## What This Changes

The issue reports that `skill_workshop(action=apply)` in WebChat does not show plugin approval prompts and then times out.

## Best Possible Solution

Keep the shipped generic Control UI plugin approval path; remaining generic Skill Workshop timeout cases should continue in their narrower open timeout reports.

## Reproduction Assessment

No. I did not run a live WebChat flow in this read-only review, but current main and `v2026.6.10` source plus tests show the reported missing `plugin.approval.requested` handler, modal rendering, and route forwarding are present.

## Solution Assessment

Yes. The shipped generic solution is the maintainable path: WebChat consumes the shared plugin approval event and modal, while before_tool_call approvals forward turn-source metadata instead of adding a WebChat-only workaround.
