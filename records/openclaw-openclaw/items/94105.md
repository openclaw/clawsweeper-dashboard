## Summary

## Best Possible Solution

Land only after the code scopes side-effecting-tool explicit silence to the accepted `system_event` contract, preserves that event kind through queued paths, and maintainers explicitly accept the new public event kind.

## Solution Assessment

No, not yet. A distinct event kind is a plausible owner-boundary fix, but the current patch broadens existing allowed silent contexts and misses queued metadata propagation; the safer solution is to narrow those two paths before maintainer product acceptance.
