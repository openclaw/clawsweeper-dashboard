## Summary

Keep open: the reported browser upload bug is real, but this patch is incomplete because the real browser route and Playwright upload validators still enforce only `DEFAULT_UPLOAD_DIR`, and the external PR only supplies mocked Vitest proof while widening a local file-access boundary.

## What This Changes

The PR adds an inbound media directory fallback to `browser.upload` path normalization and two browser-tool unit tests for fallback acceptance and outside-root rejection.

## Best Possible Solution

Centralize one allowed browser-upload root policy for temp uploads plus managed inbound media across the tool wrapper, CLI/server route, and Playwright/Chrome MCP use-time validators, then cover it with focused route/use-time tests and redacted real WebChat upload proof.

## Reproduction Assessment

Yes. Source inspection shows managed inbound media is stored under the config media root while the current browser upload wrapper, server route, and Playwright validators all accept only `DEFAULT_UPLOAD_DIR`; I did not run a live WebChat flow in this read-only review.

## Solution Assessment

No. The PR targets the right bug, but changing only the tool wrapper leaves the real upload path validators unchanged; the maintainable fix is a shared allowed-root policy with route and use-time coverage.
