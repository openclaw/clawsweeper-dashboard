## Summary

Keep open: the PR is a plausible fix direction, but it is still incomplete because the default/ref browser upload path can accept an inbound media path at the route layer and then have the armed Playwright file chooser reject it at use time. The file-access boundary expansion also needs explicit security/product approval and real WebChat/Gateway/browser proof before merge.

## What This Changes

The PR adds inbound media fallback validation for browser upload paths in the browser tool wrapper, the file-chooser route, the Playwright direct input path, and browser-tool tests.

## Best Possible Solution

Centralize the allowed browser-upload roots so wrapper, route, direct input, Chrome MCP, and armed file-chooser paths share the same policy, then land only after maintainer approval and real WebChat-to-browser upload proof.

## Reproduction Assessment

Yes from source inspection: current main validates browser.upload paths against DEFAULT_UPLOAD_DIR while inbound media is saved under CONFIG_DIR/media/inbound. I did not run a live WebChat/Gateway/browser flow in this read-only review.

## Solution Assessment

No, not yet: the proposed fallback covers several validators but misses the armed file-chooser use-time validator. The maintainable fix is a shared upload-root policy with coverage for every browser upload path.
