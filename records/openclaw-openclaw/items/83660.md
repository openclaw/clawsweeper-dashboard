## Summary

Keep this PR open: it addresses a real source-reproducible browser upload bug, but the patch only updates the tool-side validation and leaves the node browser proxy route rejecting the same inbound media paths.

## What This Changes

The PR adds an inbound media directory fallback to `browser.upload` path validation and adds two focused browser tool tests.

## Best Possible Solution

Centralize the allowed browser-upload roots so both local and proxy upload paths accept managed inbound media through the same fs-safe validation policy, then land with proxy-route coverage and real redacted behavior proof.

## Reproduction Assessment

Yes from source: current main saves inbound media under `resolveConfigDir()/media/inbound`, while both browser upload validation layers scope uploads to `DEFAULT_UPLOAD_DIR`. I did not run a live WebChat flow in this read-only review.

## Solution Assessment

No. Allowing managed inbound media is plausible, but the maintainable fix should update both local and proxy upload validation through one shared policy and include security approval plus live behavior proof.
