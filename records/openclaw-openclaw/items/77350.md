## Summary

Keep open: current main still exposes and can send the invalid OpenRouter DeepSeek V4 `max` effort, and #77380 is an open linked fix PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports OpenRouter DeepSeek V4 Pro requests failing with HTTP 400 because OpenClaw sends an invalid `reasoning_effort` after the 2026.5.3-1 upgrade.

## Best Possible Solution

Fix the OpenRouter-owned DeepSeek V4 thinking profile and wire mapping to send only OpenRouter-accepted efforts, then close this via the linked PR after merge.

## Reproduction Assessment

Yes for the invalid `max` path: current source maps `xhigh`/`max` to `reasoning_effort: "max"`, and an existing test asserts that payload. I did not run a live OpenRouter request, and the `high` variant should be covered in the fix validation.

## Solution Assessment

Yes, the narrow maintainable fix is to correct the OpenRouter DeepSeek V4 effort profile/mapping and update the focused regression tests. A global thinking config change would be broader than the demonstrated provider regression.
