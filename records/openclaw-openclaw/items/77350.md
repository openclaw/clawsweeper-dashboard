## Summary

Keep open: current main still exposes and sends OpenRouter DeepSeek V4 `reasoning_effort: "max"`, and the active closing path is open PR #77423 rather than a merged fix.

## What This Changes

The issue reports a regression where OpenRouter-routed DeepSeek V4 Pro requests fail because OpenClaw sends an invalid reasoning effort value.

## Best Possible Solution

Review and land #77423 if it remains correct; otherwise apply the same narrow OpenRouter-owned mapping fix so OpenClaw advertises and sends only OpenRouter-accepted effort values while preserving direct DeepSeek V4 behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows OpenRouter DeepSeek V4 `xhigh` can be converted to `reasoning_effort: "max"`, and an existing test asserts that payload; I did not perform a live provider call.

## Solution Assessment

Yes. The narrow provider-owned fix is to remove `max` from OpenRouter DeepSeek V4 profiles and map stale OpenRouter `max`/`xhigh` values to `xhigh`, while keeping native DeepSeek V4 `max` behavior unchanged.
