## Summary

Keep open: current main still advertises `max` for OpenRouter DeepSeek V4 and can send `reasoning_effort: "max"` on verified OpenRouter routes, while the linked open fix PR #77423 is the active implementation path and should resolve this before the issue is closed.

## What This Changes

The issue asks OpenClaw to stop sending OpenRouter-invalid DeepSeek V4 `reasoning_effort` values after the 2026.5.3-1 regression.

## Best Possible Solution

Review and land or close PR #77423, with regression coverage ensuring OpenRouter receives only supported effort values while native DeepSeek keeps its provider-specific `max` behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: OpenRouter DeepSeek V4 with `xhigh` reaches the wrapper that writes `reasoning_effort: "max"`, and an existing test asserts that payload.

## Solution Assessment

Yes. The narrow maintainable fix is provider-owned: remove `max` from the OpenRouter DeepSeek V4 profile and map stale `max` overrides to OpenRouter-supported `xhigh`, without changing global thinking semantics.
