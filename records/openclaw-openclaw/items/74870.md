## Summary

Keep this PR open. The Homebrew skip-and-hint direction is useful and current main still lacks that behavior, but the branch returns before persisting the notification fields it uses for de-duplication, so the new hint can repeat for the same version/tag when startup hints are disabled.

## What This Changes

The PR adds Homebrew install detection, skips gateway auto-update for detected Homebrew-managed package roots with a `brew upgrade openclaw` log hint, and adds infra tests plus a changelog entry.

## Best Possible Solution

Amend the PR so the Homebrew early-return path persists the same notification fields it uses to suppress repeat hints, and add regression coverage for `update.checkOnStart: false`. Keep CLI-side Homebrew update parity separate unless maintainers broaden the scope.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is the PR branch with a Homebrew package root, auto-update enabled, and `update.checkOnStart: false`: current main proves auto-update still runs in that mode, while the PR branch reads but does not write the notification fields needed to suppress repeat Homebrew hints.

## Solution Assessment

No as proposed. The skip-and-hint approach is narrow and maintainable if maintainers accept it, but this implementation needs to persist de-duplication state before it satisfies its own once-per-version/tag contract.
