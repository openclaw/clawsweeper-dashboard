## Summary

Keep open. The linked Homebrew auto-update bug is still real on current main, but this PR still gives Homebrew users the generic `openclaw update` hint before the new `brew upgrade openclaw` skip hint, and it lacks after-fix proof from a real Homebrew-managed install.

## What This Changes

The PR adds Homebrew package-root detection, skips gateway auto-update for detected Homebrew installs with a brew-upgrade log hint, persists one-shot notification state, and adds tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware updater contract only after the runtime hints, docs, tests, and changelog consistently direct Homebrew users to the supported update path and the PR includes real Homebrew proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a Homebrew package root, `update.auto.enabled`, default `update.checkOnStart`, and an available newer version, PR head logs the generic `openclaw update` hint before reaching the Homebrew skip branch; I did not run a live Homebrew gateway.

## Solution Assessment

No. Skipping the doomed npm auto-update is plausible, but this patch still emits contradictory runtime guidance, leaves docs out of sync, and lacks real Homebrew after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip the generic updater hint for Homebrew installs:** `src/infra/update-startup.ts:423`
  - body: With default `update.checkOnStart`, the generic `Run: openclaw update` notification runs before this Homebrew branch, so Homebrew users can receive both the npm-updater hint and the new `brew upgrade openclaw` hint. Move or specialize Homebrew detection before the generic notification, and add coverage that the generic hint is not emitted for Homebrew roots.
  - confidence: 0.88
