## Summary

Keep this PR open, but it still needs the previously identified branch repair before merge. Current main does not have the Homebrew auto-update skip, and the PR is aimed at the linked open Homebrew updater bug, but the new early-return path does not persist the notification state it uses for de-duplication.

## What This Changes

The PR adds Homebrew package-root detection, skips gateway auto-update for detected Homebrew-managed installs with a `brew upgrade openclaw` log hint, adds infra tests, and adds a changelog entry.

## Best Possible Solution

Amend the PR so the Homebrew skip branch persists the same notification fields it reads for de-duplication, and add focused regression coverage for the `update.checkOnStart: false` case while keeping explicit `openclaw update` CLI parity out of scope.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: use a Homebrew package root, an available version, `update.auto.enabled: true`, and `update.checkOnStart: false`; the PR branch logs from `state.lastNotified*` but writes unchanged `nextState`, so the next eligible check logs again.

## Solution Assessment

No as proposed. The Homebrew skip-and-hint direction is narrow and maintainable for the linked bug, but the implementation must persist the notification state before it satisfies its one-shot hint contract.
