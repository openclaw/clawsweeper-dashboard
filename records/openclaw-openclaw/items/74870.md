## Summary

Keep open: the PR is a plausible fix direction for the Homebrew auto-update bug, but the latest head still leaves contradictory runtime guidance for default Homebrew installs and does not include real after-fix proof from a Homebrew-managed OpenClaw install.

## What This Changes

The PR adds Homebrew package-root detection, skips the startup auto-update child for detected roots, logs a brew upgrade hint, persists notification state, and adds tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware update contract only after runtime hints, docs, tests, and real Homebrew proof consistently direct Homebrew users to either an automatic brew path or a manual `brew upgrade openclaw` path.

## Reproduction Assessment

Yes. Source inspection of the PR branch reproduces the remaining defect: with default `update.checkOnStart`, the generic notification runs before the Homebrew skip branch, so a Homebrew root can receive contradictory hints.

## Solution Assessment

No. Skipping the doomed npm auto-update is plausible if Homebrew remains manual, but the patch should move or specialize Homebrew detection before the generic hint, align docs with that contract, and include real Homebrew proof.
