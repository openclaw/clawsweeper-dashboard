## Summary

Keep open. The PR targets a real Homebrew auto-update gap that current main still lacks, but the proposed early-return path does not persist the one-shot notification state it relies on, so the promised once-per-version/tag hint can repeat when startup hints are disabled.

## What This Changes

The PR adds Homebrew install detection, skips gateway auto-update on detected Homebrew-managed package roots with a `brew upgrade openclaw` log hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Amend the PR so the Homebrew skip path persists the same notification fields it uses for de-duplication, and add regression coverage for the `update.checkOnStart: false` case. Keep CLI-side Homebrew update parity separate unless maintainers explicitly broaden the scope.

## Reproduction Assessment

Yes. Current main has a focused test showing auto-update still runs when `checkOnStart` is false, and the PR diff shows the new Homebrew branch returns without updating the notification fields used to suppress repeat logs.

## Solution Assessment

No as proposed. The Homebrew skip-and-hint direction is narrow and maintainable, but the implementation must persist the notification state before it satisfies its own one-shot hint contract.
