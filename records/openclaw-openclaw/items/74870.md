## Summary

Keep this PR open, but do not merge it as-is. The Homebrew skip direction is narrow and useful, but the current PR head still emits the generic `openclaw update` hint before the Homebrew-specific skip path, the public updating docs do not mention the Homebrew exception, and the PR is currently dirty against `main`.

## What This Changes

The PR adds Homebrew install detection, skips gateway auto-update for detected Homebrew package roots with a `brew upgrade openclaw` hint, adds focused infra tests, and updates the changelog.

## Best Possible Solution

Land a Homebrew-aware auto-update skip only after the runtime hint path and public updating docs consistently direct Homebrew users to `brew upgrade openclaw`, with regression coverage proving the generic `openclaw update` hint is not emitted for Homebrew installs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Homebrew package root with `update.auto.enabled` and default `update.checkOnStart`: on the PR head, the generic startup hint runs before the new Homebrew detection branch.

## Solution Assessment

No as proposed. Skipping npm-based auto-update for Homebrew installs is the right narrow fix, but the branch must suppress or specialize the generic hint and document the Homebrew update path before merge.
