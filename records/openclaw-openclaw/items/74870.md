## Summary

Keep open: the Homebrew skip is the right direction, but the latest head still leaves a default Homebrew install able to see the generic `openclaw update` hint before the brew-specific skip hint, and the public updating docs still omit the Homebrew exception.

## What This Changes

The branch adds Homebrew install detection to gateway startup auto-update, skips the auto-update child on detected brew roots, logs a `brew upgrade openclaw` hint, persists skip notification state, and adds tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware skip-and-hint path only after runtime hints and public updating docs consistently direct Homebrew installs to `brew upgrade openclaw`.

## Reproduction Assessment

Yes. A Homebrew-style package root with `update.auto.enabled` and default `update.checkOnStart` reaches the generic startup notification before the new skip branch, so the PR head can emit `Run: openclaw update` before the brew hint.

## Solution Assessment

No as proposed. Skipping npm-based auto-update for Homebrew-managed installs is the narrow maintainable direction, but the patch must move or specialize detection before the generic hint and document the exception.
