## Summary

Keep open: this is a plausible fix for the Homebrew auto-update failure, but the PR still emits the generic `openclaw update` hint before the new Homebrew skip hint, leaves the public updating docs inconsistent with the new manual `brew upgrade openclaw` path, and does not include after-fix real Homebrew runtime proof.

## What This Changes

The PR adds Homebrew install detection, skips gateway auto-update for detected Homebrew package roots with a `brew upgrade openclaw` log hint, persists skip notification state, and adds tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware updater path only after runtime notifications and public docs consistently direct Homebrew users to `brew upgrade openclaw`, with regression coverage for suppressing the generic hint and real Homebrew proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Homebrew package root with `update.auto.enabled`, default `update.checkOnStart`, and an available newer version: the generic startup hint logs before the PR's Homebrew skip branch.

## Solution Assessment

No. Skipping the doomed npm-based auto-update is a plausible direction, but this patch still gives contradictory runtime guidance and does not update the documented auto-updater contract for Homebrew installs.
