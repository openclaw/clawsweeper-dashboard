## Summary

Keep this PR open. Current main still lacks the Homebrew auto-update skip, and the PR is a plausible narrow fix direction, but the proposed early-return branch still has a P2 correctness bug: it logs the Homebrew skip hint without persisting the notification state when startup hints are disabled.

## What This Changes

The PR adds Homebrew install detection, skips the npm-based gateway auto-update path for Homebrew-managed package roots with a brew upgrade log hint, adds updater/Homebrew tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but persist the Homebrew skip notification state whenever that branch logs and add regression coverage for `update.checkOnStart: false`. CLI-side Homebrew update parity can remain a separate follow-up if maintainers want it.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Homebrew-managed package root with auto-update enabled and `update.checkOnStart: false`; running eligible update checks repeatedly would log from stale `state.lastNotified*` because the PR writes `nextState` without updating those fields.

## Solution Assessment

No as proposed. The skip-and-hint design is a maintainable narrow answer for Homebrew-managed installs, but the branch must persist its notification state before it satisfies the one-shot hint contract claimed by the PR.
