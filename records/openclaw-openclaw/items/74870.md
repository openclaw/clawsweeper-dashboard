## Summary

Keep this PR open. The Homebrew skip direction and persistence follow-up are useful, but the patch still emits the generic `openclaw update` startup hint before the Homebrew-specific skip hint, and the public updating docs still describe stable/beta auto-apply without a Homebrew exception.

## What This Changes

The PR adds Homebrew install detection to gateway auto-update, skips npm-based auto-update for detected Homebrew roots with a `brew upgrade openclaw` hint, persists skip notification state, and adds infra tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware skip-and-hint path only after runtime hints and public updating docs consistently direct Homebrew users to `brew upgrade openclaw`, with tests proving the generic `openclaw update` hint is not emitted for Homebrew installs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Homebrew package root with `update.auto.enabled` and default `update.checkOnStart`: the PR head logs the generic `openclaw update` startup hint before entering the new Homebrew skip branch.

## Solution Assessment

No as proposed. Skipping the npm-based auto-update for Homebrew installs is the right narrow direction, but the fix is not complete until the first runtime hint and docs agree with the Homebrew-specific behavior.
