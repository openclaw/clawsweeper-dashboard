## Summary

Keep open: the PR fixes the core Homebrew auto-update no-op by skipping the npm-based child process, and the follow-up commit fixes repeated skip notifications, but the runtime and public docs still leave Homebrew users with conflicting update guidance.

## What This Changes

The PR adds Homebrew package-root detection, skips the gateway startup auto-update for detected Homebrew installs, logs and persists a brew-upgrade hint, and adds tests plus a changelog entry.

## Best Possible Solution

Land a Homebrew-aware startup updater path that never attempts npm auto-apply for brew installs and consistently tells users, in logs and docs, to use `brew upgrade openclaw`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Homebrew package root with `update.auto.enabled` and default `update.checkOnStart`: the PR skips the auto-update child, but only after the generic `openclaw update` startup hint path has already run.

## Solution Assessment

No as proposed. The skip-and-persist direction is narrow and maintainable, but it should special-case Homebrew guidance before the generic hint and update the public updating docs before it fully solves the linked bug.
