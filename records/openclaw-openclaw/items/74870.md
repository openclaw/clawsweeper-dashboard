## Summary

Keep open: current main still lacks a Homebrew-aware auto-update skip, and the PR is useful, but the branch still places Homebrew detection after the generic `openclaw update` startup hint and is currently dirty against main.

## What This Changes

The PR adds Homebrew install detection for gateway auto-update, skips the npm-based auto-update child for detected Homebrew roots, logs a `brew upgrade openclaw` hint, persists notification state, and adds tests plus a changelog entry.

## Best Possible Solution

Land a rebased Homebrew-aware skip path that detects or specializes Homebrew before any generic update hint, preserves one-shot notification state, documents `brew upgrade openclaw`, and keeps non-Homebrew updater behavior unchanged.

## Reproduction Assessment

Yes. On the PR head, a Homebrew package root with `update.auto.enabled` and default `update.checkOnStart` reaches the generic startup notification before the added Homebrew skip branch; the added test would still pass if both messages are logged.

## Solution Assessment

No as proposed. Skipping the npm-based child for Homebrew installs is the right narrow direction, but the implementation must avoid the conflicting generic hint and update public docs before merge.
