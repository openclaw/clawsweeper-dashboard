## Summary

Keep open; this PR is a focused Kimi-provider fix for a source-reproducible cache-marker bug, and current main still lacks the strip in the Kimi wrapper.

## What This Changes

The PR strips Anthropic `cache_control` markers from Kimi provider request payloads, adds captured-payload regression coverage, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Land the Kimi-local payload strip with regression coverage while preserving generic Anthropic, OpenRouter, Vertex, Bedrock, and custom Anthropic-compatible cache behavior outside Kimi.

## Reproduction Assessment

Yes, source-level. Current main sends Kimi through `anthropic-messages`, the shared policy can inject `cache_control` before provider payload hooks, and the current Kimi wrapper does not remove it.

## Solution Assessment

Yes. The PR fixes the failure at the Kimi provider boundary, which is narrower than changing core Anthropic cache behavior or disabling cache markers for every Anthropic-compatible provider.
