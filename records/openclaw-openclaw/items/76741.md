## Summary

Keep open: the Kimi-local fix direction is valid, but the current PR over-strips nested payload data and should preserve arbitrary tool-call input fields before merge.

## What This Changes

The PR adds a Kimi provider payload patch to remove Anthropic `cache_control` fields before sending requests, adds Kimi stream regression coverage, and updates the changelog.

## Best Possible Solution

Keep the Kimi-local payload patch, but constrain stripping to Anthropic cache-marker positions while preserving arbitrary tool-use data, with regression coverage and the changelog entry.

## Reproduction Assessment

Yes for a source-level reproduction: Kimi uses `anthropic-messages`, cache retention reaches `buildAnthropicParams`, the shared policy adds `cache_control`, and current main's Kimi wrapper does not remove it. I did not run a live Kimi request in this read-only review.

## Solution Assessment

No as written. A Kimi-local strip is the right boundary, but the PR should avoid deleting arbitrary nested tool-call input fields that happen to be named `cache_control`.

## Review Findings

Overall confidence: 0.86

- **[P2] Preserve tool-call input fields while stripping cache markers:** `extensions/kimi-coding/stream.ts:226`
