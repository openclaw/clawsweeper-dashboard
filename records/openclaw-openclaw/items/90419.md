## Summary

Keep this PR open: current main still has the active retained-write early-return path, and the branch is a plausible narrow fix, but it still lacks focused coverage and after-fix real behavior proof for the session-lock timeout scenario.

## What This Changes

The PR adds deferred retained-lock idle callbacks so embedded attempt session locks release after abort or dispose is invoked from inside an active retained write.

## Best Possible Solution

Land a deferred-release fix only after adding focused active-write dispose/abort coverage plus redacted after-fix runtime proof for the orphan-lock scenario.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. Source inspection plus the PR's pre-fix timeout log show a clear current-main path where release/dispose called inside an active retained write returns without releasing the held lock.

## Solution Assessment

Mostly yes, but not merge-ready: deferring release until the retained-use counter drains is the right narrow shape and avoids force-release overlap, but it still needs focused regression coverage and after-fix real behavior proof.
