## Summary

This PR should stay open but is not merge-ready: the linked v2026.6.5 path already projects recognized `sessions_send` handoffs as assistant messages with a guaranteed non-empty forwarded label, so direct empty-label renderer fixtures do not reproduce the reported wrong attribution, side, or styling. ## What This Changes

The PR converts empty or whitespace-only WebChat `senderLabel` values to null during group rendering so resolved user or assistant names are used, and adds two direct rendering tests.

## Best Possible Solution

Reproduce the reported multi-agent payload through the shipped gateway projection and WebChat item builder, then fix the upstream recognition or provenance boundary if it fails; if a separate production blank-label case exists, scope and prove that independently.

## Solution Assessment
