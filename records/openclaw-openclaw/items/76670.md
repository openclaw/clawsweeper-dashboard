Keep this issue open: current main contains a Mattermost URL prompt, but the interactive wizard still applies the bot-token credential before the URL input, and that partial apply goes through validation that requires both values. ## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76670. The Mattermost interactive setup wizard currently asks for a bot token before the base URL; the default credential fallback calls the setup adapter with only `botToken`, and Mattermost validation rejects it because `httpUrl` is missing, so users never reach the URL prompt. Keep the fix plugin-owned if possible: add Mattermost wizard `applySet` handlers or another narrowly scoped accumulation path so interactive setup can collect and persist both bot token and base URL in one pass. Do not relax `mattermostSetupAdapter.validateInput` for non-interactive/scripted setup; `--bot-token` plus `--http-url` should still be required unless `--use-env` is used. Add regression coverage that simulates the wizard entering token then URL without throwing and verifies `baseUrl` is written. Add a `CHANGELOG.md` fix entry because this is user-facing onboarding behavior.
