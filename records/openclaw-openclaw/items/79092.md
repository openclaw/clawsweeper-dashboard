## Summary

Keep open: this is an active protected-label PR, and the latest head still has concrete Teams and Matrix authorization regressions plus stale real-behavior proof.

## What This Changes

This PR adds an experimental shared channel ingress runtime/SDK, migrates bundled message-channel authorization paths, updates docs/API baselines, and adds Discord/Slack authorization hardening.

## Best Possible Solution

Preserve each channel's existing authorization contract inside the shared ingress adapters, then refresh exact-head terminal/Testbox proof before maintainers reconsider the SDK boundary.

## Reproduction Assessment

Yes for the Teams and Matrix review regressions by source-level comparison between current main and PR head. I did not run live channel callbacks, and the broader refactor is not a single bug reproduction target.

## Solution Assessment

No as currently proposed. Centralizing ingress matches the architecture direction, but the PR must keep Teams group allowlists isolated from DM allowlists and merge Matrix room/global command allowlists before it is the maintainable fix.

Current main passes room users and the global group allowlist as separate command authorizers, so a sender allowed only by `groupAllowFrom` remains command-authorized; this refactor blocks that valid mixed configuration.
  - confidence: 0.89
