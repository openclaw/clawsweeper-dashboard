## Summary

Keep open: this protected-label channel authorization refactor still has blocking regressions in Microsoft Teams group sender scope and Matrix room command authorization, and the PR body proof is stale for the current head.

## What This Changes

The PR adds an experimental channel ingress runtime/SDK, updates docs and SDK exports, and migrates many bundled channel ingress and command authorization paths onto it.

## Best Possible Solution

Keep the shared ingress direction only after the migrated channels preserve their existing authorization contracts, add focused regression coverage for the failing cases, and refresh current-head real behavior proof.

## Reproduction Assessment

Yes for the two concrete regressions by source comparison between current main and PR head: Teams group sender fallback and Matrix mixed room command allowlists. I did not run live callbacks, and the broader refactor is not a single runtime reproduction target.

## Solution Assessment

No as currently proposed. Centralizing ingress fits the architecture direction, but the PR must preserve Teams group allowlist isolation and Matrix roomUsers OR global groupAllowFrom command behavior before merge.
