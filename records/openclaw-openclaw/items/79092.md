## Summary

This PR should stay open: the shared ingress direction is useful, but the current head still has authorization regressions in Teams and Matrix and the protected `maintainer` label requires explicit maintainer handling.

## What This Changes

The PR adds an experimental shared channel ingress SDK/runtime API, migrates many bundled channel plugins to it, updates docs/baselines/tests, and includes Discord and Slack authorization hardening fixes.

## Best Possible Solution

Keep the shared ingress architecture direction, but preserve each channel's existing authorization contract, add focused regression coverage for Teams and Matrix, refresh exact-head proof, and then have maintainers review the experimental SDK boundary.

## Reproduction Assessment

Yes for the blocking review findings by source inspection: current main keeps Teams group sender checks from inheriting the DM allowlist and keeps Matrix room/global command allowlists separate, while the PR head changes both semantics. I did not run tests because this is a read-only cleanup review.

## Solution Assessment

No, not as currently proposed. Centralizing ingress fits the architecture direction, but the patch must preserve existing channel authorization semantics and refresh proof for the exact current head before it is mergeable.
