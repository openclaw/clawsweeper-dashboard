## Summary

Keep open: this PR targets a real payload-policy bug and includes after-fix terminal proof, but the current diff still has a blocking review issue because exec/bash failures can produce duplicate user-visible warnings when the assistant already reports the command failure.

## What This Changes

The branch moves default exec/bash tool failures into the mutating-tool warning path and updates payload tests to show concise failures while hiding raw stderr.

## Best Possible Solution

Keep the targeted payload-policy direction, but make exec/bash warnings skip when the assistant already visibly reports the command failure and cover that with regression tests.

## Reproduction Assessment

Yes. The deterministic reproduction is a buildEmbeddedRunPayloads call with assistant success text plus lastToolError for exec; current main suppresses it by source, while the PR body shows the after-fix terminal output for that path.

## Solution Assessment

No, not yet. Surfacing concise exec/bash failures is the right narrow area, but the PR should also recognize explicit exec failure acknowledgements so it does not add a second warning after the assistant already tells the user the command failed.
