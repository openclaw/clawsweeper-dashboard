## Summary

Keep open: current main partially narrows the active-turn metadata path by concatenating inbound context into the prompt, but OpenRouter still uses passthrough replay hooks that disable Anthropic turn normalization for Claude/OpenRouter routes, and no live provider proof shows the reported OpenRouter payload is fixed.

## What This Changes

The issue requests merging group-chat metadata with the actual inbound message, or otherwise normalizing adjacent user-role turns, so Anthropic/OpenRouter primary models do not reject group-chat turns.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep current behavior, then verify with focused replay-policy tests and redacted live Discord/OpenRouter proof.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. Source inspection gives a medium-confidence path: Discord topic metadata reaches current inbound context and active prompt concatenation, while OpenRouter's passthrough replay policy disables Anthropic user-turn merging if adjacent user-role replay messages reach dispatch.

## Solution Assessment

Partly. The broad request to merge metadata with the current prompt matches the active-turn direction already present on main, but the safer remaining fix is model-ref-aware OpenRouter Anthropic normalization with tests and live proof rather than moving all Discord metadata into trusted system context.
