## Summary

Keep open: current main has narrowed the fresh active-turn metadata path, but OpenRouter still uses a passthrough replay policy that disables Anthropic turn normalization for OpenRouter-hosted Claude models, and there is no linked closing PR for this issue.

## What This Changes

The issue reports Discord group-chat metadata being sent as a separate user-role turn before the real user message, causing OpenRouter Anthropic routes to reject or fall back.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep their existing behavior, then verify with focused replay-policy tests and a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a medium-confidence path: Discord topic metadata enters untrusted context, active prompt assembly now concatenates that context with the current prompt, and OpenRouter disables Anthropic turn normalization if adjacent user-role turns reach replay/provider dispatch.

## Solution Assessment

Partly. Merging metadata into the active prompt is already current-main behavior, so the safer remaining direction is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
