## Summary

Keep open: current main does not prove the reported OpenRouter Anthropic route is fixed. The active Discord prompt path now joins metadata with the submitted prompt, but OpenRouter still opts into passthrough replay hooks that disable Anthropic turn normalization if adjacent user-role turns reach replay/provider dispatch.

## What This Changes

The issue reports that Discord group chat metadata and the actual inbound message are sent as adjacent user-role messages, causing Anthropic/OpenRouter requests to fail and fall back to other models.

## Best Possible Solution

Add model-ref-aware OpenRouter Anthropic/Claude turn normalization at the provider replay boundary, preserve OpenRouter Gemini/OpenAI passthrough behavior, and verify with focused regression tests plus a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and issue evidence give a plausible OpenRouter Anthropic replay-path problem, but current active prompt assembly already concatenates Discord context with the visible user prompt before submission.

## Solution Assessment

Partly. Merging metadata into the active prompt is already how current main handles this surface, so the safer remaining fix is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
