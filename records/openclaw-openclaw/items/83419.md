## Summary

Keep open: current main appears to concatenate active Discord metadata into the current prompt, but OpenRouter still advertises a passthrough replay policy that leaves Anthropic turn normalization disabled for OpenRouter-hosted Claude routes. The issue is recent, active, and still needs maintainer policy plus live Discord/OpenRouter proof before it can be considered fixed or queued automatically.

## What This Changes

The issue requests that group-channel metadata and the actual inbound message be sent as one model-bound user turn, or that Anthropic/OpenRouter dispatch squashes adjacent user turns.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep their existing behavior, then verify with focused tests and a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: Discord topic metadata enters untrusted context and current prompt assembly now concatenates it with the prompt, while OpenRouter's passthrough replay policy would still skip Anthropic turn normalization if adjacent user turns reach replay/provider dispatch.

## Solution Assessment

Partly. Merging metadata into the active prompt is already how current main handles the obvious first-turn surface, so the safer remaining fix is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
