## Summary

Keep open: current main already joins current inbound metadata into the active prompt, but OpenRouter still opts into passthrough replay policy that disables Anthropic turn normalization for Claude/OpenRouter routes; the remaining behavior needs maintainer policy and live provider proof.

## What This Changes

The issue asks OpenClaw to prevent group-chat metadata from creating adjacent user-role messages for Anthropic/OpenRouter requests.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep existing behavior, then verify with focused replay-policy tests and a redacted live Discord/OpenRouter check.

## Solution Assessment
