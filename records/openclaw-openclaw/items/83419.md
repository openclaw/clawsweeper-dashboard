## Summary

Keep open: current main folds the active Discord metadata into the prompt body, but OpenRouter still owns a passthrough replay policy that disables Anthropic turn normalization if adjacent user turns reach provider replay. The issue has strong source signal and reporter payloads, but it still needs maintainer policy review plus live Discord/OpenRouter proof before a safe fix is chosen.

## What This Changes

The issue asks OpenClaw to prevent Discord group-chat metadata and the actual inbound message from reaching OpenRouter Anthropic models as adjacent user-role messages.

## Best Possible Solution

Add a model-ref-aware OpenRouter Anthropic/Claude replay-normalization path, preserve existing passthrough behavior for non-Anthropic OpenRouter models, and verify it with focused regression tests plus a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection plus the reporter's intercepted payload give a medium-confidence path through Discord untrusted context and OpenRouter's Anthropic-normalization-disabled replay policy.

## Solution Assessment

Partly. Merging active metadata into the prompt body already exists on current main, so the safer remaining direction is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
