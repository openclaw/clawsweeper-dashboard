## Summary

Keep open: current main appears to have narrowed the first-turn active prompt problem by concatenating inbound context with the current prompt, but the OpenRouter passthrough replay policy still disables Anthropic turn normalization for Claude/OpenRouter routes and no current-main live Discord/OpenRouter payload proves the report is fixed.

## What This Changes

The issue asks OpenClaw to avoid sending Discord group-chat metadata and actual user content as adjacent user-role messages to Anthropic models routed through OpenRouter.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get the needed turn normalization while Gemini and other passthrough routes keep current behavior, then verify with focused replay tests and a redacted live Discord/OpenRouter payload.

## Reproduction Assessment

No high-confidence live reproduction was established against current main. Source inspection gives a medium-confidence path: Discord topic metadata enters untrusted context and active prompt assembly now concatenates it with the current prompt, but OpenRouter disables Anthropic turn normalization if adjacent user turns reach replay/provider dispatch.

## Solution Assessment

Partly. Current main already uses prompt concatenation for the obvious active-turn metadata path, so the safer remaining fix is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
