## Summary

Keep open: current main does not prove the exact Discord-topic-as-separate-user-message diagnosis, but the report identifies a credible OpenRouter Anthropic compatibility gap because OpenRouter's replay policy still disables the existing Anthropic same-role merge path.

## What This Changes

The issue asks OpenClaw to prevent group-channel metadata and the actual inbound message from reaching Anthropic-backed OpenRouter models as consecutive user-role messages.

## Best Possible Solution

Add a model-aware OpenRouter replay-policy fix that enables Anthropic-compatible turn validation only for Anthropic/Claude model refs, with regression coverage that preserves existing Gemini/OpenAI OpenRouter behavior.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows a credible provider-policy path because OpenRouter Anthropic model refs use OpenAI completions while OpenRouter replay policy disables the existing Anthropic turn merge.

## Solution Assessment

Yes for the provider-adapter direction, but not for moving Discord topic metadata into trusted system context. The narrow maintainable fix is model-ref-scoped Anthropic turn validation in OpenRouter replay handling, with tests proving non-Anthropic OpenRouter behavior stays unchanged.
