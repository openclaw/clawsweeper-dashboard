## Summary

Keep open: current main still has a credible OpenRouter Anthropic compatibility gap because OpenRouter owns a replay policy that disables Anthropic turn normalization, while the reported Discord metadata path is concrete enough to need maintainer follow-up rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to prevent group-chat metadata injection from producing consecutive user-role messages in Anthropic/OpenRouter requests.

## Best Possible Solution

Add a model-ref-aware OpenRouter Anthropic/Claude replay policy or payload normalization that merges consecutive user turns only for OpenRouter Anthropic model refs, preserves existing Gemini/OpenAI passthrough behavior, and proves both with focused regression coverage plus one live OpenRouter repro.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a medium-confidence path: OpenRouter disables Anthropic turn validation while the report supplies an intercepted payload showing adjacent user turns.

## Solution Assessment

Partly. Provider-boundary normalization for OpenRouter Anthropic model refs looks like the maintainable fix, while moving all Discord topic metadata into trusted system context would conflict with the current untrusted-context design.
