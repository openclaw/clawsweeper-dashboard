## Summary

Keep open: current main only partially narrows the report. Active Discord metadata is now concatenated into the current prompt body, but OpenRouter still owns a passthrough replay policy that disables Anthropic turn normalization for OpenRouter-routed Claude models, so this still needs maintainer policy and live provider proof.

## What This Changes

The issue requests that group-chat metadata and the actual inbound message be sent as one model user turn, or otherwise normalized before Anthropic/OpenRouter dispatch.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep their existing behavior, then verify with focused tests and redacted live Discord/OpenRouter proof.

No high-confidence live reproduction was established. ## Solution Assessment
