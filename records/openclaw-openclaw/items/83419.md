## Summary

Keep open: current main appears to have folded active Discord metadata into the current prompt body, but OpenRouter still registers passthrough replay hooks that explicitly disable Anthropic turn normalization, so the remaining Claude/OpenRouter compatibility question is not proven fixed and still needs maintainer policy plus live proof.

## What This Changes

The issue requests that Discord group-chat metadata and the actual inbound message stop reaching Anthropic/OpenRouter as consecutive user-role messages.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Claude/Anthropic routes get Anthropic turn normalization while Gemini and other passthrough routes keep their current behavior, then verify with focused replay-policy tests and a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a medium-confidence path: active Discord metadata is concatenated into the current prompt, but OpenRouter still disables Anthropic turn normalization if adjacent user-role turns reach replay/provider dispatch.

## Solution Assessment

Unclear as a direct metadata-merge fix: current main already concatenates active metadata with the prompt body. The better remaining direction is a scoped OpenRouter Claude/Anthropic replay policy decision, not moving untrusted Discord topic metadata into trusted system context.
