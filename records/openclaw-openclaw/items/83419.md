## Summary

Keep open: current main partially addresses the active-turn metadata shape by concatenating current inbound context into the prompt, but OpenRouter still owns a passthrough replay policy that disables Anthropic turn normalization for Claude/OpenRouter routes, and later comments report still-active duplicate processing on other surfaces.

## What This Changes

The issue reports that injected channel/user metadata and real inbound text can create adjacent user-role turns, causing Anthropic/OpenRouter rejection, fallback, or duplicate replies.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep existing behavior, then verify the inbound duplicate-turn path with focused tests and a redacted live provider check.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: active prompt metadata is now concatenated, but OpenRouter still disables Anthropic turn normalization if adjacent user-role turns reach replay/provider dispatch.

## Solution Assessment

Partly. The current prompt concatenation is the right shape for the basic first-turn metadata surface, but the remaining fix should be a scoped OpenRouter Anthropic/Claude replay-policy decision plus live proof, not a broad same-role merge for every OpenRouter model.
