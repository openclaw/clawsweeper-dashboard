## Summary

Keep open: current main does not prove the reported OpenRouter Anthropic path is fixed. Active Discord metadata is now assembled into the prompt body, but OpenRouter still uses passthrough replay hooks that disable Anthropic turn normalization when adjacent user-role turns reach replay/provider dispatch.

## What This Changes

The issue asks OpenClaw to prevent Discord group-chat metadata from creating adjacent user-role messages before OpenRouter Anthropic requests are sent.

## Best Possible Solution

Add model-ref-aware OpenRouter Anthropic/Claude turn normalization at the provider replay boundary, preserve OpenRouter Gemini/OpenAI passthrough behavior, and verify with focused regression tests plus a redacted live Discord/OpenRouter check.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and issue evidence give a medium-confidence path: Discord topic metadata enters untrusted context and OpenRouter disables Anthropic turn normalization, but current active prompt assembly already concatenates metadata with the user body.

## Solution Assessment

Partly. Merging metadata into the active prompt is already how current main handles this surface, so the safer remaining fix is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
