## Summary

Keep open: current main partially reduces the active-turn prompt split by concatenating inbound context into the user prompt, but the OpenRouter route still opts into passthrough replay policy with Anthropic turn validation disabled and lacks live OpenRouter proof.

## What This Changes

The issue asks OpenClaw to prevent group-channel metadata and the actual Discord message from reaching Anthropic/OpenRouter as consecutive user-role messages.

## Best Possible Solution

Add model-ref-aware OpenRouter Anthropic/Claude turn normalization at the provider replay boundary while preserving existing Gemini/OpenAI passthrough behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source plus issue evidence gives a medium-confidence path: Discord metadata enters untrusted context and OpenRouter disables Anthropic turn validation, but current active prompt assembly already concatenates context with the user body.

## Solution Assessment

Partly. Merging metadata into the active prompt is already covered on current main, so the safer remaining direction is scoped OpenRouter Anthropic/Claude replay normalization plus live provider proof.
