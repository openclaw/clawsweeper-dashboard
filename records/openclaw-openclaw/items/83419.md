## Label Justifications

- `P1`: The report describes a configured group-chat Anthropic/OpenRouter workflow falling through from the primary model to fallback providers for real users.
- `impact:session-state`: The affected behavior is prompt/replay turn shape and same-role ordering in the session context sent to the provider.
- `impact:message-loss`: Provider rejection and fallback can prevent the configured model from handling incoming channel messages correctly.
- `impact:auth-provider`: The remaining uncertainty is specifically tied to provider/model routing and OpenRouter-hosted Anthropic compatibility.

## Summary

Keep open: current main appears to have narrowed the first-turn prompt split by concatenating active inbound context with the current prompt, but OpenRouter still opts into a passthrough replay policy that disables Anthropic turn normalization for Claude/OpenRouter routes, so the remaining provider-compatibility question is not proven fixed.

## What This Changes

The issue asks OpenClaw to avoid separate consecutive user-role entries by merging group-chat metadata with the real inbound message or by normalizing Anthropic/OpenRouter turns before dispatch.

## Best Possible Solution

Make OpenRouter replay policy model-ref-aware so Anthropic/Claude routes get Anthropic turn normalization while Gemini and other passthrough routes keep current behavior, then verify with focused policy tests and a redacted live Discord/OpenRouter payload.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: Discord metadata is prompt-local now, but OpenRouter still disables Anthropic turn validation if adjacent user-role turns reach replay/provider dispatch.

## Solution Assessment

Partly. Merging active metadata into the prompt is already current-main behavior, so the safer remaining direction is scoped OpenRouter Anthropic/Claude replay normalization rather than moving Discord topic metadata into trusted system context.
