Improve plugin trust/provenance warnings so first-time users with an empty `plugins.allow` get actionable plugin IDs, config guidance, and CLI discovery steps.

## Best Possible Solution

Keep the plugin trust warning, land focused loader warning/docs/test remediation with exact plugin-id guidance, and leave channel aliasing or auto-load policy changes to https://github.com/openclaw/openclaw/issues/68352.

## Solution Assessment

Yes for the scoped warning-copy/docs/test direction. It preserves the security trust signal while making the warning actionable; suppressing warnings, channel aliases, and auto-load policy changes are broader work tracked separately.
