## What This Changes

The request is to make fresh-install plugin allowlist/provenance warnings actionable by showing exact plugin IDs, config remediation, and plugin list/inspect guidance.

## Best Possible Solution

Keep the plugin trust warning, add actionable ID/config/CLI guidance in loader warnings plus docs/tests, and leave broader alias/autoload policy to https://github.com/openclaw/openclaw/issues/68352.

## Reproduction Assessment

Yes. Source inspection on current main shows that with plugins enabled, an empty `plugins.allow`, and a workspace/global non-bundled plugin discovered, the loader emits the generic empty-allowlist warning and can also emit the generic untracked-provenance warning.

## Solution Assessment

Yes for the scoped warning-copy/docs/test direction. It preserves the security trust signal while making the warning actionable; suppressing warnings, channel aliases, and auto-load policy changes are broader work tracked separately.
