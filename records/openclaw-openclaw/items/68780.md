## Summary

Keep open. Current main still emits generic plugin trust warnings for workspace/global discovery with an empty allowlist, and the maintainer discussion already scopes the remaining work as warning UX and docs rather than an obsolete report.

## What This Changes

Request to make first-run plugin allowlist and provenance warnings less alarming and more actionable when non-bundled plugins are discovered without configured trust.

## Best Possible Solution

Preserve the trust warning while improving the message and docs to include exact plugin IDs, a minimal `plugins.allow` snippet, and `openclaw plugins list --enabled --verbose` / `openclaw plugins inspect` discovery guidance.

## Reproduction Assessment

Yes. The source and regression tests provide a high-confidence reproduction path: workspace/global plugin discovery with an empty allowlist still warns, and untracked loaded non-bundled plugins still produce provenance warnings.

## Solution Assessment

Yes for the maintainer-scoped direction: a focused warning-copy, docs, and regression-test update is the narrowest maintainable fix. Suppressing the warnings or changing plugin loading policy is not the best path without separate product/security approval.
