## Summary

Keep open. Current main still has the reported sandbox skill immutability gap, and maintainer discussion already calls for a secops design rather than cleanup closure.

## What This Changes

Request to make sandbox skill directories read-only across workspaceAccess modes, preferably by mounting eligible skill source directories as read-only instead of writable copies.

## Best Possible Solution

Implement a maintainer-approved sandbox skill mount model that derives eligible skill sources from the skills loader, adds read-only overlays after broader mounts, avoids writable duplicate skill copies when source mounts are available, and updates Docker, remote backends, config hashing, fs bridges, docs, and tests together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: ro/none modes call syncSkillsToWorkspace into a writable sandbox workspace, while rw mode still exposes workspace skills through the writable /workspace mount.

## Solution Assessment

Unclear as a final patch. Read-only bind overlays are the right Docker primitive, but the maintainable fix needs secops-approved cross-backend semantics rather than a Docker-only change.
