## Summary

Keep open: current main still lacks generated read-only per-skill sandbox mounts, and recent issue discussion already routes this as sandbox/secops design rather than cleanup closure.

## What This Changes

The request is to make sandbox-visible skill directories immutable by mounting eligible skill source directories read-only across all workspaceAccess modes.

## Best Possible Solution

Implement a secops-reviewed sandbox contract that derives eligible skill source directories from the skills loader, mounts them read-only after broader workspace mounts, avoids writable duplicate copies, updates FS bridge/config-hash behavior, defines non-Docker semantics, and documents/tests all workspaceAccess modes.

## Reproduction Assessment

Yes, at source level. Current main still copies skills for ro/none sandbox workspaces and exposes workspace skills through the writable workspace mount for rw mode; I did not run a live Docker write attempt in this read-only review.

## Solution Assessment

Unclear as filed. Read-only Docker overlays are a plausible enforcement direction, but the maintainable fix needs loader-derived mounts plus config-hash, FS-bridge, SSH/OpenShell, docs, and test follow-through.
