## Summary

Keep open. Current main still leaves sandbox-visible skills writable in the reported paths, and the existing maintainer discussion routes this to secops design rather than conservative cleanup closure.

## What This Changes

Request read-only bind-mounted skill directories for sandboxed agents instead of writable workspace copies or writable workspace skill paths.

## Best Possible Solution

Define an approved sandbox skill-source mount model that derives eligible skill directories from the loader, exposes them read-only without writable duplicates, and updates Docker, fs bridges, config hashing, backend docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: ro/none modes call syncSkillsToWorkspace into a sandbox workspace, while rw mode exposes workspace skills through the writable /workspace mount.

## Solution Assessment

No current implementation solves it. Read-only Docker overlays are likely the right primitive for Docker, but the best fix needs a cross-backend contract rather than a Docker-only patch.
