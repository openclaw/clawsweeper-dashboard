## Summary

Keep open: current main and v2026.6.10 still do not satisfy the JSON-only OTLP collector request; the Docker gateway default remains `http/protobuf`, the bundled diagnostics-otel runtime/schema/docs still exclude `http/json`, and the linked one-line default-flip PR was closed unmerged after its author agreed it was the wrong scope.

## What This Changes

The issue requests a durable gateway telemetry protocol path so JSON-only OTLP/HTTP collectors do not require manual edits that are later overwritten by service-env regeneration.

## Best Possible Solution

Choose a maintainer-approved telemetry protocol contract, then implement either end-to-end `http/json` support or a documented durable override/preservation path that survives gateway service regeneration without breaking protobuf collectors.

## Reproduction Assessment

Yes for source reproduction. Current main still defaults Docker OTLP protocol to `http/protobuf`, still rejects `http/json` in diagnostics-otel runtime/config, and still has no explicit overwrite warning contract for generated service env files; I did not run a live JSON-only collector in this read-only review.

## Solution Assessment

No for the linked default-only PR, and unclear for the final product direction. The maintainable fix needs either complete `http/json` exporter support or a deliberate durable override/preservation contract rather than only flipping the Docker fallback.
