## Summary

Keep open for maintainer direction: current main and the latest release still default Docker telemetry to `http/protobuf`, the bundled diagnostics exporter/schema/docs still do not support `http/json`, and the linked default-only PR was closed unmerged after its author agreed it was the wrong scope.

## What This Changes

The issue asks OpenClaw to provide a regeneration-safe way for gateway telemetry environments to use `OTEL_EXPORTER_OTLP_PROTOCOL=http/json` with JSON-only OTLP/HTTP collectors.

## Best Possible Solution

Choose a maintainer-approved protocol contract, then either add end-to-end `http/json` exporter support with schema/docs/tests or add a durable operator override/preservation path that survives service regeneration without changing the unsupported default accidentally.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current Docker defaults still use `http/protobuf`, and current exporter/schema/docs reject or exclude `http/json`. I did not run a live Quencher collector, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, the default-only PR path is not the best fix because it would set a protocol the bundled exporter rejects. The best fix needs a deliberate override/preservation contract or complete `http/json` support across exporter, schema, docs, and validation.
