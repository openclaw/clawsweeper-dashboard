## Summary

Keep this open. Current main and the latest release still keep the Docker OTLP protocol fallback and bundled diagnostics exporter on `http/protobuf`, while the only linked default-to-JSON PR was closed unmerged after review agreed it was the wrong scope.

## What This Changes

The issue asks OpenClaw to provide a durable way for gateway service env regeneration to support JSON-only OTLP/HTTP collectors without manual env-file edits being overwritten.

## Best Possible Solution

Keep this issue open for a maintainer choice between a durable per-profile env/protocol override, preserving operator-modified generated env values, or full end-to-end `http/json` diagnostics support.

## Reproduction Assessment

Source-reproducible: current main and `v2026.6.6` still keep the OTLP protocol default/support contract on `http/protobuf`, and the linked PR discussion confirms the JSON-only collector path is not solved. I did not run a live collector reproduction in this read-only review.

## Solution Assessment

No. The rejected default-only PR is not the best fix because it would inject a protocol the bundled diagnostics exporter does not support; the durable solution needs maintainer direction on override semantics or real JSON exporter support.
