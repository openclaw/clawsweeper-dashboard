## Summary

Keep open for maintainer direction: current main and the latest release still default Docker telemetry to `http/protobuf`, the bundled exporter/schema/docs still do not support `http/json`, and the linked one-line default PR is closed unmerged after the author agreed it was the wrong scope.

## What This Changes

The issue requests a durable gateway telemetry protocol path so JSON-only OTLP/HTTP collectors do not require regenerated service-env file edits.

## Best Possible Solution

Choose a maintainer-approved telemetry protocol contract, then either add complete `http/json` support or add a supported operator override/preservation path that survives service regeneration.

## Reproduction Assessment

Yes. Source inspection shows the current Compose default is `http/protobuf` while current exporter/schema/docs do not support `http/json`; I did not run a live JSON-only collector in this read-only pass.

## Solution Assessment

No. The default-only path is not the best solution because it would change an environment default without making `http/json` a supported bundled exporter/config contract; the safer path is a deliberate override/preservation contract or end-to-end protocol support.
