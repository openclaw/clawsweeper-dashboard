## Summary

Close as implemented. Current main and the v2026.6.6 release bound aggregate tool-result replay before model submission, with regression coverage for repeated sub-cap `toolResult` output so short follow-up messages should no longer inherit an overgrown prompt history.

## What This Changes

The issue reports that repeated long-running `toolResult` messages can accumulate in persisted session history until later short messages exceed the model context window.

## Best Possible Solution

Keep the shipped aggregate prompt-history guard, and track any narrower provider-specific truncation gaps separately if new current-version evidence appears.

## Reproduction Assessment

Yes. The original issue is source-reproducible from repeated sub-cap `toolResult` entries, and current regression tests exercise that aggregate shape while current main applies the guard before model submission.

## Solution Assessment

Yes. The implemented fix bounds provider-visible prompt history while preserving canonical session state, which is narrower and safer than rewriting every just-appended persisted entry.
