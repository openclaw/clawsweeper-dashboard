## What This Changes

Reports that repeated sub-cap process toolResult messages can accumulate in persisted session history until short follow-up Telegram messages exceed the model context window.

## Best Possible Solution

Review and land the linked PR, or an equivalent narrow fix, so persisted toolResult history has both per-result and aggregate active-branch budgets with regression coverage.

## Reproduction Assessment

Yes, source-level. Append several sub-cap toolResult messages through the guarded SessionManager path and inspect the active branch aggregate; current main caps individual results but does not bound the aggregate at persistence time.

## Solution Assessment

Yes. The linked PR's direction is narrow: keep the existing per-result cap, enforce an aggregate active-branch budget after tool results are persisted, and add focused session-guard coverage.
