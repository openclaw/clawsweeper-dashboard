## Summary

Keep open: current main still has the clean no-write/non-repair branch that prints the `openclaw doctor --fix` trailer, and an open linked PR already targets this exact bug with closing syntax.

## What This Changes

The issue asks for `openclaw doctor` to print the `--fix` trailer only when there are pending config changes that `--fix` could apply.

## Best Possible Solution

Land one focused doctor-flow fix that suppresses the clean-state trailer, preserves guidance when real pending repairs exist, and includes regression coverage for both clean and pending no-write paths.

## Reproduction Assessment

Yes, at source level: current main logs the trailer whenever no config write occurs and `--fix` was not requested. I did not run the CLI because this review is read-only, but the implicated branch and missing pending-change signal are explicit in source.

## Solution Assessment

Yes, the linked PR's pending-change gate is a narrow maintainable fix for the reported clean-state false trailer. Maintainers should still choose between that approach and the closed alternate approach of removing the write-config trailer and relying entirely on finalization hints.
