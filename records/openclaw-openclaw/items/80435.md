## Summary

Keep open: current main still has the clean no-write path that logs the `openclaw doctor --fix` trailer whenever `--fix` was not used, and https://github.com/openclaw/openclaw/pull/80455 is an open closing PR rather than a merged fix.

## What This Changes

The issue asks for `openclaw doctor` to suppress the `doctor --fix` trailer on clean non-fix runs unless there are actual pending config changes.

## Best Possible Solution

Land a narrow doctor config/write-path fix that ties the generic trailer to real pending, unapplied repairs while preserving existing specific fix hints.

## Reproduction Assessment

Yes. Source inspection on current main shows a clean no-write run reaches `runWriteConfigHealth` with `shouldRepair: false`, which still logs the trailer even when no pending changes exist; no live CLI run was needed to establish the code path.

## Solution Assessment

Yes. The narrow maintainable solution is to gate or remove the generic write-config trailer using the existing pending-change contract, then cover clean, pending-declined, pending-accepted, and `--fix` paths.
