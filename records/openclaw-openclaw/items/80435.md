## Summary

Keep open: current main still has the no-write, non-`--fix` path that emits the trailer, and an open fixing PR with closing syntax already targets this issue. The issue should remain open until a fix such as https://github.com/openclaw/openclaw/pull/80455 lands.

## What This Changes

The issue asks for clean `openclaw doctor` runs without `--fix` to omit the `Run "openclaw doctor --fix" to apply changes.` trailer unless there are pending config changes.

## Best Possible Solution

Land a single focused fix that gates the trailer on actual pending config changes and covers both clean and pending paths with regression tests, then let the linked fixing PR close this issue.

## Reproduction Assessment

Yes. Source inspection on current main shows a clean no-write run reaches `runWriteConfigHealth` with `shouldRepair: false`, and that branch still logs the trailer without knowing whether pending changes exist.

## Solution Assessment

Yes. Gating the write-config trailer on the existing pending-change signal is the narrow maintainable fix; https://github.com/openclaw/openclaw/pull/80455 appears to follow that direction with regression coverage.
