## Summary

Keep open: current main still has the no-write doctor path that logs the `openclaw doctor --fix` trailer whenever `--fix` was not used, and the linked closing PR is still open rather than merged.

## What This Changes

Suppress the generic `openclaw doctor --fix` trailer on clean `openclaw doctor` runs that have no pending config changes to apply.

## Best Possible Solution

Review and land one canonical doctor-flow fix, likely the open closing PR if maintainers prefer its pending-change gate, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a clean no-write doctor run reaches `runWriteConfigHealth` with `shouldRepair: false`, which still logs the trailer even when no pending change exists.

## Solution Assessment

Yes. The narrow maintainable fix is to tie the generic trailer to real pending repairs, or remove it from `runWriteConfigHealth` if maintainers want `finalizeDoctorConfigFlow` to own all repair hints.
