## Summary

Keep open: current main still has the clean no-write doctor path that logs the generic `openclaw doctor --fix` trailer whenever `--fix` was not used, and the only closing PR is still open rather than merged.

## What This Changes

The issue asks for `openclaw doctor` to print the generic `--fix` trailer only when a pending repair actually exists.

## Best Possible Solution

Land one focused doctor-flow fix that suppresses the clean-state trailer, preserves actionable guidance for real pending repairs, and adds regression coverage for clean, pending, and repair-mode paths.

## Reproduction Assessment

Yes, at source level. A clean run can reach `runWriteConfigHealth` with `shouldWriteConfig` false and `shouldRepair` false, and current main then logs the trailer without checking for pending changes.

## Solution Assessment

Yes, the narrow maintainable fix is to gate or remove the generic write-config trailer using the existing pending-change/fix-hint contract. The open PR at https://github.com/openclaw/openclaw/pull/80455 is the current implementation candidate, while https://github.com/openclaw/openclaw/pull/80448 was a closed alternate approach.
