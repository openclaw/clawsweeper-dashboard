## Summary

Keep open: current main still has the clean no-write `doctor:write-config` path that logs the `openclaw doctor --fix` trailer whenever `--fix` was not used, and the linked closing pull request is still open rather than merged.

## What This Changes

The issue reports that `openclaw doctor --non-interactive` prints the `openclaw doctor --fix` trailer on clean runs with no pending repairs.

## Best Possible Solution

Review and land one narrow fix, preferably the pending-change-signal approach in https://github.com/openclaw/openclaw/pull/80455, then close or supersede the competing fix attempt.

## Reproduction Assessment

Yes. Source inspection on current main shows a clean no-write run can reach `runWriteConfigHealth` with `shouldRepair: false`, which still logs the trailer even when no pending change state exists for that contribution to check.

## Solution Assessment

Yes. The narrow maintainable solution is to gate the generic write-config trailer on real pending repairs while preserving existing specific fix hints and covering clean, pending, and `--fix` paths.
