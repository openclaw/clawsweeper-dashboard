## Summary

Keep open: current main still has the clean-state trailer path, so the reported CLI output bug remains actionable. The earlier https://github.com/openclaw/openclaw/issues/24566 and https://github.com/openclaw/openclaw/pull/24666 only suppressed the message when `--fix` was already running; they did not gate the plain `doctor` clean-state case on actual pending changes.

## What This Changes

Suppress the generic `doctor --fix` apply-changes trailer on clean `openclaw doctor` runs that have no pending repairs.

## Best Possible Solution

Gate or remove the generic write-config trailer so clean doctor runs end quietly, while real pending repair hints continue to come from the pending-change paths.

## Reproduction Assessment

Yes. Source inspection on current main shows a clean no-write run reaches `runWriteConfigHealth` with `shouldRepair: false`, which still logs the trailer even when no pending changes exist.

## Solution Assessment

Yes. The narrow maintainable fix is in the doctor write-config/finalization output path with regression tests for clean no-trailer behavior and pending-change guidance; broader lint or dry-run work is not needed for this bug.
