## Summary

## What This Changes

Clean `openclaw doctor` runs without `--fix` should stop printing the `doctor --fix` trailer unless pending config changes exist.

## Best Possible Solution

Land one canonical doctor-flow fix so clean no-write doctor runs end at `Doctor complete.` while pending repair guidance remains actionable and covered by regression tests.

## Reproduction Assessment

## Solution Assessment

Yes, a focused doctor-flow fix is the right shape. The open implementation choice is whether to thread pending-change state into `runWriteConfigHealth` or remove that trailer and keep fix hints owned by `finalizeDoctorConfigFlow`.
