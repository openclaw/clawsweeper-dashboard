## Summary

## What This Changes

The issue reports that `openclaw doctor --non-interactive` prints the `openclaw doctor --fix` trailer even when no pending config changes remain.

## Best Possible Solution

Land one focused doctor-flow fix that suppresses the clean-state trailer while preserving actionable repair guidance for real pending config changes.

## Reproduction Assessment

Yes. Source inspection of current `origin/main` shows a clean no-write doctor run can reach `runWriteConfigHealth` and emit the trailer solely because `--fix` was not used.

## Solution Assessment

Yes. The narrow maintainable solution is to gate or remove the write-config trailer using the existing pending-change/fix-hint contract, then cover clean, pending, accepted, and repair-mode paths.
