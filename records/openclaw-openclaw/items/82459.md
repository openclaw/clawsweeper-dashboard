## Summary

Keep open: the PR addresses a real source-reproducible reset/ACPX bug cluster, but the current branch still weakens persisted session validation by accepting entries without the required `sessionId`, and its real-behavior proof covers ACPX cleanup but not the gateway reset transcript rotation. The protected `maintainer` label also means this needs explicit maintainer handling rather than cleanup closure.

## What This Changes

The PR rotates reset transcript files when `sessions.reset` creates a new session id, expands ACPX orphan reaping to plugin-local ACP adapter packages while narrowing launch-lease injection to generated wrappers, and adds session/ACPX/cron tests plus a changelog entry.

## Best Possible Solution

Land a focused branch that preserves the reset transcript rotation and ACPX orphan cleanup, keeps current persisted-session-id validation intact, and adds redacted real reset-flow proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main passes the old `sessionFile` through reset while the resolver trusts explicit paths, and current ACPX startup reaping does not recognize plugin-local adapter package roots.

## Solution Assessment

No, not as-is. The reset/ACPX fix direction is appropriate, but the branch should keep rejecting persisted session entries without a valid `sessionId` and should prove the gateway reset behavior in a real setup.
