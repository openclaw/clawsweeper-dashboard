## Summary

Keep open: the fix direction addresses valid current-main bugs, but this PR still has session-id validation regressions and lacks real reset-flow proof; the protected `maintainer` label also requires explicit maintainer handling.

## What This Changes

This PR rotates reset transcript files to the new session id, expands ACPX startup orphan reaping for plugin-local ACP adapters, narrows launch-lease injection to generated wrappers, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land a focused branch that keeps reset transcript rotation and ACPX orphan cleanup, preserves the required session-id validation contract, and includes redacted real reset proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main passes the old `sessionFile` through reset while the resolver trusts explicit paths, and current ACPX startup reaping does not recognize plugin-local adapter roots.

## Solution Assessment

No, not as-is. The reset and ACPX fix direction is appropriate, but the branch should keep the current persisted-session-id validation contract and prove the gateway reset behavior in a real setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep rejecting entries without sessionId:** `src/config/sessions/store-entry-shape.ts:34-36`
  - body: This branch now accepts any persisted session object where `sessionId` is absent and returns it as a `SessionEntry`. `SessionEntry.sessionId` is required, and load/read callers use it for transcript, reset, cleanup, and active-session paths, so a metadata-only corrupt row can survive normalization instead of being pruned. Keep the missing-id rejection and add a regression case.
  - confidence: 0.9
- **[P2] Keep the store validator aligned with transcript ids:** `src/config/sessions/store-entry-shape.ts:11-18`
  - body: Replacing `validateSessionId` with this broader regex accepts ids that the transcript path resolver still rejects, such as ids with `:`/`@` or over 128 characters. A persisted row with that id and no valid `sessionFile` can now survive store normalization and later fail when history/reset/cleanup derives its transcript path. Keep this normalizer on the same `validateSessionId` contract unless all session-file contracts change together.
  - confidence: 0.86
