## Summary

Keep open: current main still lacks terminal status persistence for isolated cron sessions, so this PR targets a real bug, but the current head has blocking correctness issues and only test/format proof rather than after-fix runtime proof.

## What This Changes

The PR sets isolated cron session entries to running/done/failed/timeout during isolated cron execution and adds focused session-status regression tests.

## Best Possible Solution

Land a rebased isolated-cron fix that records lifecycle status inside the protected execution window, derives terminal status after delivery outcome, and logs best-effort persistence failures without changing return semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows isolated cron runs can return ok/error/timeout while the session entry is only persisted with metadata, and the linked report gives matching sessions.json versus run-log evidence.

## Solution Assessment

No. The PR targets the right module, but it persists done/failed before delivery can still fail, writes running too early in setup, and lets session-store write failures mask the original cron result.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Persist status after delivery outcome is known:** `src/cron/isolated-agent/run.ts:572-573`
  - body: The new done/failed write runs before dispatchCronDelivery. Non-best-effort direct delivery can still return an error result, so a run can return error while the session store has already been persisted as done for a non-fatal payload.
  - confidence: 0.88
- **[P2] Guard running status until setup is protected:** `src/cron/isolated-agent/run-session-state.ts:74`
  - body: Adding status = "running" in markCronSessionPreRun makes the pre-run persist record running before later setup work can throw outside the protected execution catch. That path can leave the stored row stuck running.
  - confidence: 0.82
- **[P2] Keep terminal status persistence best-effort:** `src/cron/isolated-agent/run.ts:717`
  - body: The catch and timeout paths now await persistSessionEntry before returning the structured cron result. If the session-store write fails, the function rejects and masks the original timeout or run failure.
  - confidence: 0.8
