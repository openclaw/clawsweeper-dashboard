Keep the existing WhatsApp proxy wiring and use the documented managed proxy config or durable OPENCLAW_PROXY_URL for the installed Gateway service.

## Solution Assessment

Yes. The narrow maintainable solution is the existing service proxy configuration path, not another WhatsApp code change, unless a new report proves the Gateway process has the configured proxy and still cannot connect.
