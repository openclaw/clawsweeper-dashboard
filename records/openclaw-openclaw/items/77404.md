## Summary

Keep this open. The previous proxy fix for #72547 is present in current source and changelog, but this report describes a newer gateway-runtime symptom where the account is linked while the LaunchAgent-backed listener remains running but not connected.

## What This Changes

The issue reports that WhatsApp QR login succeeds behind HTTP/HTTPS proxy env vars, but the gateway never reaches connected state or receives WhatsApp messages.

## Best Possible Solution

Add or confirm a narrow gateway-side diagnostic/fix that verifies the WhatsApp listener reaches connected after QR login and clearly reports missing gateway proxy inheritance when that is the blocker.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows a plausible current-main path to `running: true` and `connected: false` after `channels.start`, but live WhatsApp/proxy gateway logs are needed to prove the exact failure.

## Solution Assessment

Unclear. The existing #72547 proxy-agent fix is present, so the safest next step is to distinguish gateway service env inheritance from a remaining Baileys/proxy connection failure before changing runtime behavior.
