## Summary

Keep open: current main still contains the fail-closed memory index identity gates that match the reported `index scope changed` and `index metadata is missing` failure class, and the related open reports/PRs only cover parts of the broader memory-core metadata cluster.

## What This Changes

The issue asks OpenClaw to make `openclaw memory index --force` and gateway `memory_search` agree on memory index scope/provider metadata so rebuilt indexes return results.

## Best Possible Solution

Consolidate the memory-core identity/reindex cluster, then fix the bundled memory plugin so force rebuild, gateway search, and background sync use one canonical provider/scope identity while preserving true mismatch protection with live upgrade proof.

## Reproduction Assessment

No live high-confidence reproduction was run here. Source inspection plus multiple detailed reporter comments show the same current identity gates and sync/search path, so the report is source-reproducible at medium confidence.

## Solution Assessment

Unclear as proposed. The fix should be manager-level metadata/reindex consistency; changing scope validation to warn-only or tying background validation to sync knobs needs maintainer judgment because it can allow stale indexes to be searched.
