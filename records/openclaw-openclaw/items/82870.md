## Summary

Keep open. The underlying media-loss bug is source-reproducible and the lower-level agent fix is directionally right, but the current PR head still blurs pending-only versus attempt-level media semantics and its live Telegram proof predates the force-pushed head.

## What This Changes

The branch makes tool-produced media override assistant-authored media directives in embedded agent replies, preserves attempt-level tool media for final payload assembly, records media trace metadata, adds tests, and updates browser screenshot skill guidance.

## Best Possible Solution

Land an agent-level media precedence fix that keeps tool artifacts authoritative while preserving separate pending-only and attempt-level media APIs, backed by current-head Telegram/browser proof.

## Reproduction Assessment

Yes. Source inspection shows current main lets assistant-authored media outrank queued tool media in both block-reply consumption and final payload assembly; I did not run live Telegram proof in this read-only review.

## Solution Assessment

Unclear as submitted. Fixing the lower-level agent media contract is the right direction, but this patch should split or deliberately rename/update pending-only versus attempt-level media access before it is the best mergeable solution.
