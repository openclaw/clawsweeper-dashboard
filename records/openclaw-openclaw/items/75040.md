Keep open. Current main still has the shared OpenAI-compatible `extra_body` collision path, and the related open PRs address overlapping DeepSeek symptoms without settling the shared request-precedence contract.

## What This Changes

Fix the WARN where `params.extra_body.thinking` overwrites provider-generated `thinking` fields across OpenAI-compatible provider request payloads.

## Best Possible Solution

Define a shared OpenAI-compatible `extra_body` collision rule that preserves explicit override precedence, suppresses equivalent-value noise, normalizes disabled thinking aliases before provider wrappers, and keeps native Ollama `think` handling in the Ollama plugin.

## Reproduction Assessment

Yes for the core warning path: a provider wrapper can write `payload.thinking`, then the later `extra_body` wrapper warns and overwrites the same key on current main. The reported live Ollama timeout/stuck-session behavior still lacks a high-confidence source-only reproduction.

## Solution Assessment

Unclear because this issue has no patch and the related PRs solve narrower pieces. The maintainable fix should coordinate shared precedence, thinking alias normalization, and provider replay behavior instead of landing another provider-only workaround.
