## Summary

Keep open. Current main and the latest release still have the shared OpenAI-compatible collision path where a provider wrapper can write `payload.thinking` before `extra_body` logs and overwrites the same key; the DeepSeek path is source-reproducible, while the reported live Ollama timeout causality remains unproven.

## What This Changes

The issue asks OpenClaw to stop duplicate `thinking` request writes and `extra_body` collision warnings while keeping disabled-thinking behavior reliable across providers.

## Best Possible Solution

Define one shared OpenAI-compatible thinking precedence rule that normalizes disabled aliases, preserves `extra_body` final override semantics, avoids or suppresses equivalent `thinking` collisions, and leaves native Ollama `think` mapping in the Ollama plugin.

## Reproduction Assessment

Yes for the core warning path: current source shows a provider wrapper can write `payload.thinking`, then the later `extra_body` wrapper logs and overwrites the same key. The live Ollama timeout/stuck-session impact is not reproduced by source inspection alone.

## Solution Assessment

No patch is attached, and the best fix is broader than either open adjacent PR by itself. A coordinated request-precedence and disabled-thinking cleanup is safer than a provider-by-provider workaround that preserves the repeated warning path.
