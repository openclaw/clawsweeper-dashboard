## Label Justifications

- `P1`: The latest release can disable semantic memory retrieval after normal live use, but current main already contains the repair.
- `impact:session-state`: The bug leaves persistent memory chunks present while losing the index identity required to retrieve them.

## Summary

Current main prevents this failure by clearing database-scoped metadata cache state when safe reindex switches to its temporary database, then writing metadata before publication; v2026.6.5 lacks that reset, matching the reported released behavior.

## What This Changes

Prevent live semantic memory search from leaving indexed chunks without the builtin index metadata row.

## Best Possible Solution

Keep the current database-scoped metadata cache reset and regression coverage, and ship the implementation in the next release rather than adding another runtime fallback.

## Reproduction Assessment

Yes. The report provides a precise released-version sequence with before-and-after SQLite checks, and the v2026.6.5 source contains the stale database-scoped metadata cache behavior that explains it.

## Solution Assessment

Yes. Clearing the metadata serialization cache when changing database handles fixes the root invariant without adding a fallback or parallel metadata path.
