## Summary

Keep open. Current main and release v2026.4.10 already spread heartbeat runs across stable per-agent phases, but cron `every` schedules still retain their shared anchor cadence and the requested public scheduling controls remain absent; deciding whether to extend automatic spreading or add configuration requires maintainer product judgment.

## What This Changes

Add prime-derived cadence and deterministic phase controls for heartbeat and cron interval schedules to reduce synchronized request spikes.

## Best Possible Solution

Adopt one deterministic phase-spreading rule for newly created cron interval jobs while preserving persisted anchors, keep heartbeat’s shipped automatic behavior, and add a public exact-phase option only if maintainers decide that cron expressions do not adequately cover fixed wall-clock requirements; avoid prime interval transformation unless load measurements demonstrate an additional benefit.

## Reproduction Assessment

Partly yes from source: heartbeat phase spreading is directly verifiable, and cron `every` schedules with equal anchors and intervals remain aligned. The report does not include a measured current-main multi-agent load reproduction showing provider spikes or failure frequency.

## Solution Assessment

No, not as proposed. Prime intervals are not the narrowest solution to synchronized phases; deterministic phase spreading addresses that invariant more directly, while exact `:55` wall-clock scheduling is better represented by an appropriate cron expression than by jitter.

## Risks / Open Questions

- New cadence and phase fields would expand an already large public configuration surface and create additional scheduling modes that must stay consistent across schema, CLI, status output, persistence, and docs.
- Automatically shifting existing cron interval anchors could change exact timing for upgraded jobs; preserving existing anchors while changing only new jobs would create separate fresh-install and upgrade behavior.
- Prime-number interval transformation adds semantic complexity but does not prevent same-phase alignment by itself, so it may become a competing policy without materially improving on deterministic phase spreading.
