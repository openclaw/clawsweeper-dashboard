## Summary

Keep open. Current main already mitigates part of the spike problem with stable heartbeat phase scheduling and cron top-of-hour staggering, but the requested prime cadence mode and cron `every` jitter/config surface are not implemented and need maintainer product/API judgment.

## What This Changes

The issue requests prime-derived heartbeat and cron interval scheduling plus deterministic per-agent phase jitter to reduce synchronized bursts.

## Best Possible Solution

Decide a public scheduler policy first; if accepted, add backward-compatible schema, protocol, CLI, docs, and tests for any opt-in prime/`every` jitter behavior while preserving exact schedules unless maintainers intentionally change defaults.
