## Summary

Keep open. Current main partially addresses load spreading with internal heartbeat phasing and cron-expression top-of-hour staggering, but it still lacks the requested public prime cadence mode, configurable heartbeat jitter/window, cron every jitter/prime support, and status/docs exposure.

## What This Changes

Request adds backward-compatible prime cadence and deterministic phase-jitter controls for heartbeat and cron every schedules to reduce synchronized load spikes.

## Best Possible Solution

Keep this as a scheduler/API design item and land an additive public contract for accepted heartbeat and cron every jitter/prime semantics across config, protocol, CLI/status, docs, and focused scheduler tests.
