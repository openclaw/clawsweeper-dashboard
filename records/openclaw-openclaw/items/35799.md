## Summary

The central heartbeat anti-herding request has shipped, but current cron `every` schedules still lack explicit deterministic phase or stagger control, leaving a narrower product decision unresolved.

## What This Changes

Add optional prime-derived heartbeat and cron interval modes plus stable per-agent scheduling offsets to reduce synchronized load spikes.

## Best Possible Solution

Keep the shipped deterministic heartbeat phase implementation and narrow future work to one explicit stable phase/anchor contract for cron `every` schedules, preferably reusing generalized stagger or schedule-identity machinery without prime interval rewriting.

## Reproduction Assessment

Yes for the narrowed scheduling gap: current source proves heartbeat is phase-spread, while cron `every` has only fixed interval and anchor semantics and rejects stagger options. This review did not reproduce provider load spikes in a live multi-gateway setup.

## Solution Assessment

No, the combined proposal is not the best solution: heartbeat phase jitter is already shipped, and prime cadence is less predictable than a single explicit cron interval phase or anchor contract.
