## Summary

Close as duplicate/superseded by the open canonical browser-isolation request in #37487. Current main already has named browser profiles with dedicated managed profile directories and existing-session `userDataDir`, while the remaining per-agent binding, proxy/CDP isolation, and concurrency design is the same work tracked there.

## What This Changes

Request per-agent browser storage/profile isolation so multiple agents can use separate cookies, localStorage, CDP endpoints, and related browser profile settings.

## Best Possible Solution

Use #37487 as the canonical design and implementation thread for first-class agent-to-browser-profile binding, per-profile proxy/CDP policy, and concurrency guarantees, while keeping the existing named-profile workaround documented.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection verifies that `agents.list[].browser.userDataDir` is not a current config surface and that browser isolation is currently expressed through `browser.profiles` plus explicit profile routing.

## Solution Assessment

Yes. Closing this as duplicate/superseded by open #37487 is the narrowest maintainer cleanup because current main only partially covers the use case, and the remaining per-agent isolation design is already tracked in the broader canonical issue.
