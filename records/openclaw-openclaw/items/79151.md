## Summary

This PR should stay open for normal maintainer handling: it has the protected `maintainer` label, and the latest head still leaves concrete Codex requirements-policy gaps on Windows, read-only-only policies, and remote sandbox sections.

## What This Changes

This PR changes Codex app-server config, tests, docs, and changelog so implicit local stdio defaults switch from YOLO to guardian when Codex requirements disallow `danger-full-access`.

## Best Possible Solution

Keep the fix in the Codex plugin app-server boundary, but share a platform-aware, TOML-structured requirements-policy helper that chooses an actually allowed sandbox and applies only the relevant remote-host policy.

## Reproduction Assessment

Yes from source inspection: current main defaults implicit local stdio Codex app-server sessions to `danger-full-access`, while upstream Codex requirements can reject that sandbox. I did not run a failing live managed host in this read-only review.

## Solution Assessment

No: the direction is narrow, but this implementation only mirrors part of Codex's requirements logic. The safer fix is a shared policy parser/resolver that matches Codex's platform path, allowed-mode selection, and remote-host scoping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use Codex's platform requirements path:** `extensions/codex/src/app-server/config.ts:7`
  - body: This hard-codes `/etc/codex/requirements.toml`, but Codex loads managed requirements from `%ProgramData%\OpenAI\Codex\requirements.toml` on Windows. A constrained Windows stdio host will still miss the policy, keep YOLO defaults, and send the rejected `danger-full-access` sandbox.
  - confidence: 0.91
- **[P2] Select an actually allowed sandbox mode:** `extensions/codex/src/app-server/config.ts:525`
  - body: When `allowed_sandbox_modes = ["read-only"]`, this branch switches to guardian but still sends `sandbox: "workspace-write"`. Codex will reject that too, so read-only-only managed hosts continue to fail at turn start.
  - confidence: 0.94
- **[P2] Parse only applicable requirements entries:** `extensions/codex/src/app-server/config.ts:563`
  - body: This regex can match nested `[[remote_sandbox_config]]` entries that Codex applies only after hostname matching. A remote-only policy for another host can incorrectly downgrade local stdio defaults on unaffected machines.
  - confidence: 0.88
