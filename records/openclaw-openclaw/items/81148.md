## Summary

Keep this PR open, but do not merge it as-is: the TDZ fix is useful and has real behavior proof, but the optional private task-runtime fallback can silently disable Codex native subagent task mirroring when the bundled plugin is loaded from source.

## What This Changes

The PR moves Codex app-server client factory override storage into `client-factory.ts`, adds a test, makes native subagent task-runtime loading optional, and expands the Codex private SDK alias check for npm-installed `@openclaw/codex`.

## Best Possible Solution

Keep the factory-storage move, but resolve the optional private task runtime through an alias-aware path and test both helper-available and helper-missing cases.

## Reproduction Assessment

Yes. The PR body gives a concrete installed CLI/Gateway reproduction and after-fix live output, while current-main source shows the module-scope factory storage that matches the reported TDZ failure.

## Solution Assessment

No. The client-factory refactor is a good fix direction, but the native task-runtime fallback should not bypass the source/plugin-loader alias path and silently turn off Codex subagent task mirroring.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve source-loaded Codex task mirroring:** `extensions/codex/src/app-server/native-subagent-task-mirror.ts:44-60`
  - body: This `createRequire` lookup does not use the plugin loader's alias map. For a source-loaded bundled Codex plugin, the private `openclaw/plugin-sdk/codex-native-task-runtime` subpath is only supplied by that alias layer, so this throws and the catch caches `unavailableRuntime`; the app keeps running but native subagent task runs stop being mirrored. Please resolve the optional helper through an alias-aware path, or add an equivalent source fallback, before falling back to the no-op runtime.
  - confidence: 0.86
