## Summary

Keep open: the reported TDZ failure is credible and the supplied proof is sufficient, but the current patch still needs changes because its task-runtime fallback bypasses the alias-only helper used by source-loaded Codex and can stop native subagent task mirroring.

## What This Changes

The PR moves Codex app-server client factory test override storage into `client-factory.ts`, makes native subagent task-runtime loading optional, and expands Codex SDK alias detection for npm-installed plugin paths.

## Best Possible Solution

Keep the TDZ factory-storage split, preserve alias-aware access to the private task runtime for bundled/source Codex, and fall back to no-op only when the helper is genuinely absent in the npm-installed layout.

## Reproduction Assessment

Yes. The PR body supplies real after-fix CLI/Gateway output, and the review finding is source-reproducible with a native `createRequire` probe returning `MODULE_NOT_FOUND` while current SDK alias tests show the intended alias path.

## Solution Assessment

No as written. The factory split is the right narrow fix, but the default task-runtime resolution should not use only native `createRequire` when source-loaded bundled Codex depends on the plugin loader alias.

## Review Findings
