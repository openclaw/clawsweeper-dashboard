## What This Changes

The issue reports duplicate outbound OpenAI realtime voice greetings when voice-call initial greetings and OpenAI server VAD auto-response both create responses.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/86285, or a replacement with equivalent live Twilio/OpenAI proof, then close this issue from the merged implementation.

## Reproduction Assessment

Yes, at source level: current main queues an explicit ready-trigger greeting while OpenAI server VAD auto-response remains enabled by default. I did not place a fresh live Twilio/OpenAI call in this read-only review.

## Solution Assessment

Yes, the best fix direction is narrow OpenAI-specific first-turn collision handling while preserving the shared explicit greeting path for non-OpenAI providers such as Google. Broadly dropping initial greetings or disabling auto-response for the whole call would be a riskier fix.
