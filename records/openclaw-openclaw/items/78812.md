## Summary

The failure occurs after OpenClaw has delegated login to the external `@tencent-weixin/openclaw-weixin` plugin; current core only installs, loads, and calls that plugin, while the reported failing Tencent iLink QR endpoint returned a QR payload during direct live verification.

## What This Changes

The issue asks OpenClaw to make `openclaw channels login --channel openclaw-weixin` show a WeChat QR code instead of failing with `TypeError: fetch failed`.

## Best Possible Solution

Keep OpenClaw core's generic channel-plugin login contract unchanged and route persistent Weixin QR fetch failures to the external plugin or host-network diagnostics unless new evidence shows a missing OpenClaw SDK/proxy seam.

## Reproduction Assessment

No. The issue log matches the external plugin's QR-fetch failure path, but a direct POST to the same Tencent iLink QR endpoint returned HTTP 200 with `ret: 0` during this review.

## Solution Assessment

Yes. Closing this in the OpenClaw source repo is the narrowest maintainable outcome because core delegates to the plugin and the documented owner of Tencent iLink login behavior is the external Weixin package.
