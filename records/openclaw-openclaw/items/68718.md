## Summary

Keep open. Current main has the detached-runtime seam maintainers asked future durable execution work to target, but it does not implement this PR's durable queue, and the branch still has blocking state-machine defects plus a concrete install-security scanner concern.

## What This Changes

The PR adds a SQLite-backed `src/minions` durable job queue with gateway startup, CLI, protocol, docs, TaskRegistry shadow-write, subagent spawn tracking, handler stubs, and a `openclaw/plugin-sdk/minions-runtime` SDK subpath.

## Best Possible Solution

Keep Tasks and Task Flow as the canonical core ledger/orchestration model, preserve the landed `registerDetachedTaskRuntime` seam, and rework durable queue/retry/recovery mechanics as a smaller executor behind that contract. Split the install-security scanner behavior into its own narrowly reviewed PR.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the patch-level defects: the PR head shows deterministic state-machine paths for duplicate respawn and stuck parent jobs. The broader durable-executor request is a design change rather than a single current-main runtime reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. The safer direction is the existing detached-runtime contract plus a narrow durable executor behind it, not a second core `minions` substrate with separate CLI, protocol, docs, and state beside Tasks/TaskFlow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep tracked spawns non-expiring until lifecycle updates them:** `src/agents/tools/sessions-spawn-tool.ts:360-366`
  - body: The tracking row is inserted as an `active` job with a 30s lock, but the original live subagent run does not renew that lock or complete the minion row. Once `lock_until` expires, normal long-running or already-finished spawns can be requeued as stalled and launched again without a gateway crash.
  - confidence: 0.88
- **[P1] Resolve parents for ignored child failures:** `src/minions/queue.ts:842-847`
  - body: `applyChildFailPolicy` only resolves parents for `remove_dep`. For `ignore` or `continue`, a parent whose remaining children all fail or die can stay in `waiting-children` forever because no later completion calls `resolveParentIfDone`.
  - confidence: 0.9
- **[P2] Apply parent policy to stalled dead-lettered children:** `src/minions/queue.ts:563-577`
  - body: When `handleStalled()` marks a child job `dead` after max stalled count, it returns the row without applying the child's `on_child_fail` policy. A fan-out parent can remain stuck in `waiting-children` or miss `fail_parent` behavior after crash recovery exhausts retries.
  - confidence: 0.84
- **[P2] Resolve parent state after child cancellation:** `src/minions/queue.ts:384-390`
  - body: `cancelJob()` cancels the requested subtree but never resolves the cancelled child's parent. Cancelling the last unfinished child directly can leave the parent in `waiting-children` because no later child completion will trigger parent resolution.
  - confidence: 0.8
- **[P2] Tighten workspace symlink trust in the scanner:** `src/plugins/install-security-scan.runtime.ts:221-222`
  - body: The install-security scan change allows outside-install-root symlink targets when they sit under sibling `extensions`, `packages`, or `ui` workspace paths. That weakens the blocked-dependency scan for plugin installs and should be split or narrowed to exact trusted OpenClaw peer links with focused tests.
  - confidence: 0.79
