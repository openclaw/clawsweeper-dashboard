## Summary

Keep open. Current main still omits both elapsed duration and the requester-approved absolute start time from the shared status footer, while the session start timestamp is already available and related timing items cover different surfaces.

## What This Changes

Request to add a session lifetime segment, either elapsed duration or an accepted start timestamp, to the shared status footer.

## Best Possible Solution

Add one shared session-lifetime segment in `buildStatusMessage()` using `sessionEntry.sessionStartedAt` or `resolveSessionLifecycleTimestamps()`, before the existing update-age text, with `/status` and `session_status` coverage.

## Reproduction Assessment

Yes. Source inspection on current main is enough: status callers pass the session entry into the shared renderer, but the renderer only emits the session key plus update age or no-activity text.

## Solution Assessment

Yes. The narrow maintainable solution is a shared status-renderer change using the existing lifecycle timestamp source, not duplicated formatting in the slash-command or agent-tool callers.
