## What This Changes

Request to show a session start time or elapsed duration in the shared status footer.

## Best Possible Solution

Add one shared footer segment from the existing lifecycle timestamp source so chat `/status` and the `session_status` tool show consistent session start visibility without duplicating formatting in callers.

## Solution Assessment

Yes. The narrow maintainable path is to update the shared status renderer using `sessionStartedAt` or `resolveSessionLifecycleTimestamps()`, not the separate current-time or per-reply response-duration surfaces.
