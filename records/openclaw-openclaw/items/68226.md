## Summary

Keep open. Current main and v2026.5.7 still render the shared session footer with only the session key plus update age/no-activity text, while the session start timestamp is already available; related timing items cover different surfaces and do not supersede this request.

## What This Changes

Request to add elapsed session lifetime, or an accepted absolute start time, to the shared status footer's Session line.

## Best Possible Solution

Add one shared session-lifetime segment in the status renderer from `sessionStartedAt` or `resolveSessionLifecycleTimestamps()`, before the existing update-age text, with focused coverage for `/status` and `session_status`.

## Reproduction Assessment

Yes. Source inspection on current main is enough: a status request with a `sessionEntry.sessionStartedAt` still reaches `buildStatusMessage()`, whose footer only renders the session key plus update age or no-activity text.

## Solution Assessment

Yes. The narrow maintainable solution is a shared status-renderer change using the existing session lifecycle timestamp source, not duplicated formatting in the slash-command or agent-tool callers.
