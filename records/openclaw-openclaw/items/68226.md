## Summary

Keep open. Current main and the latest release still omit session duration or start time from the shared status footer, while the session start timestamp already exists and no related open PR supersedes this specific footer request.

## What This Changes

Request to add session elapsed duration or a start timestamp to the shared status footer shown by `/status` and `session_status`.

## Best Possible Solution

Add one shared session-lifetime segment in `buildStatusMessage()` using the existing session lifecycle timestamp source so `/status` and `session_status` stay consistent.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: `buildStatusMessage()` receives session context but renders only the session key plus update age/no-activity, while lifecycle code and docs show `sessionStartedAt` is available.

## Solution Assessment

Yes. The narrow maintainable solution is to update the shared status renderer using `sessionStartedAt` or `resolveSessionLifecycleTimestamps()`, not to duplicate formatting in `/status`, `session_status`, or the separate per-reply usage footer.
