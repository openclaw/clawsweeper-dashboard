## What This Changes

The issue asks for `openclaw models list` to respect `models.mode: "replace"` by listing only configured provider models instead of auth-backed built-in catalog rows.

## Best Possible Solution

Land one reviewed candidate fix that makes the default list command follow replace-mode configured-provider scope while preserving intentional full-catalog and provider-filter browse behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with replace mode and auth for unconfigured built-in providers, plain `openclaw models list` still reaches `appendAuthenticatedCatalogRows` and can append those providers.

## Solution Assessment

Yes. The narrow maintainable fix is to gate the default model-list row source by the documented replace-mode provider scope, while reviewing linked PRs for `--all`, provider-filter, and wildcard compatibility.
