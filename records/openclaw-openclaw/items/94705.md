## Summary

Keep open. Current `main` still has a CLI-specific visibility gap: the default `openclaw models list` path appends auth-available catalog rows after configured rows, while the docs and Gateway sibling behavior reserve the full catalog for explicit browse/all views.

## What This Changes

The issue reports that `models.mode: "replace"` and a configured model/provider allowlist do not constrain default `openclaw models list` output.

## Best Possible Solution

Align default CLI `openclaw models list` with the existing Gateway configured-view policy: honor `agents.defaults.models`/configured visibility by default, keep `--all` as the full-catalog diagnostic path, and add focused CLI regression coverage.

## Reproduction Assessment

No live high-confidence repro was run. Source inspection gives a credible reproduction path: default CLI listing appends auth-backed catalog rows after configured rows even when an allowlist should be the visible model set.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing configured/allowlist visibility policy for the CLI default path and preserve explicit full-catalog discovery under `--all`, rather than adding a new knob.
