## Summary

Keep open: current main and the latest release still route `openclaw models list` through catalog sources that can add authenticated or built-in provider rows in replace mode, and multiple open linked PRs are already candidate fixes.

## What This Changes

Make `openclaw models list` honor `models.mode: "replace"` so the displayed models match the configured provider/model scope.

## Best Possible Solution

Review and land one focused fix, preferably the candidate with real CLI proof and regression coverage, so replace-mode list output is configured-scope by default without changing merge-mode or intentional full-catalog browse behavior.

## Reproduction Assessment

Yes at source level, but not with a live command in this read-only review. Current main still lets the default list path append authenticated catalog rows and lets cascade plans include catalog rows without a replace-mode gate.

## Solution Assessment

Yes, a focused fix in `models list` source orchestration is the best path. It should mirror the existing picker/configured-catalog contract while preserving merge-mode and explicit full-catalog browse behavior.
