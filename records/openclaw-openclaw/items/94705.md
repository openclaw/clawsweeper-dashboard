## Summary

Keep open: current main still appends authenticated catalog rows in the default `openclaw models list` path, while the config contract says replace mode uses only configured providers. Adjacent fixes covered the model picker, Control UI, gateway configured view, and provider-discovery startup cost, but no open PR directly owns this local CLI list output gap.

## What This Changes

The issue asks for `openclaw models list` to honor `models.mode: "replace"` by showing only configured or allowlisted provider models.

## Best Possible Solution

Update the CLI model-list default source selection so replace mode lists only configured or allowlisted provider rows, while preserving `--all` as the explicit full-catalog diagnostic path.

## Reproduction Assessment

Source-reproducible: current default CLI list source selection appends authenticated catalog rows after configured rows, and the replace-mode contract says only configured providers should be used. I did not run the CLI because this read-only review avoided commands that may create runtime state.

## Solution Assessment

Yes, if implemented narrowly in the CLI list source selection. Default list output should share configured-only behavior under replace mode, while `openclaw models list --all` remains the full diagnostic catalog.
