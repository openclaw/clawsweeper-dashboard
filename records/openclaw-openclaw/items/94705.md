## Summary

Keep open: current main and the latest release still contain the unguarded default `models list` path, and the open linked PRs are candidate fixes rather than merged fixes.

## What This Changes

The issue asks for `openclaw models list` to respect `models.mode: "replace"` by hiding auth-backed built-in providers outside the configured provider/model scope.

## Best Possible Solution

Land one reviewed fix that makes the default list command follow replace-mode configured-provider scope while preserving documented explicit browse paths such as `--all` and provider filters.

## Reproduction Assessment

Yes, at source level. With replace mode plus auth for an unconfigured catalog provider, the default list path still reaches `appendAuthenticatedCatalogRows` and can append that provider.

## Solution Assessment

Yes. The best fix is a narrow default-list gate for replace-mode configured-provider scope, while keeping explicit full-catalog browse behavior intact unless maintainers deliberately change that public contract.
