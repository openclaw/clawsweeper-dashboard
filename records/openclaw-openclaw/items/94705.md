## Summary

Keep open: current main and v2026.6.9 still route default `openclaw models list` through an authenticated catalog append with no replace-mode guard, and the linked PRs are unmerged candidate fixes rather than an implemented fix.

## What This Changes

The issue requests that `openclaw models list` honor `models.mode: "replace"` by showing only configured/allowed provider models instead of auth-backed built-in catalog rows.

## Best Possible Solution

Land one reviewed linked PR that gates the default authenticated-catalog append for replace mode while preserving explicit `--all` and provider-filtered browse behavior, then close or redirect the remaining same-root PRs.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Plain `openclaw models list` skips the full/provider source-plan cascade, enters `appendConfiguredModelRowSources`, and then appends authenticated catalog rows without checking `models.mode`.

## Solution Assessment

Yes, the best fix is narrow: make the default authenticated-catalog append respect `models.mode: "replace"` while keeping explicit `--all` and provider-filtered browse paths as full-catalog diagnostics.
