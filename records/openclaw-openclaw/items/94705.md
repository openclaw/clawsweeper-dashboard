## Summary

Keep open: current main and v2026.6.8 still contain the unguarded default `models list` path, and open linked PRs are candidate fixes rather than merged fixes.

## What This Changes

The issue reports that `openclaw models list` shows built-in/authenticated providers even when `models.mode` is `"replace"` with a restricted provider list.

## Best Possible Solution

Land one reviewed fix that makes the plain list command follow the same replace-mode provider scope as the picker, while preserving explicit browse paths such as `--all` and intentional wildcard-provider behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with replace mode and auth for unconfigured built-in providers, plain `openclaw models list` still reaches `appendAuthenticatedCatalogRows` and can append those providers.

## Solution Assessment

Yes. The narrow maintainable fix is to make the default list row source respect replace-mode configured-provider scope, matching the documented contract and sibling picker behavior while preserving explicit browse behavior.
