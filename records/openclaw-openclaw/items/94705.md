## Summary

Keep open: current main still has the default `openclaw models list` path appending authenticated catalog rows without a `models.mode: "replace"` guard, v2026.6.8 contains the same path, and open closing PRs already own the fix path.

## What This Changes

The issue requests that `openclaw models list` respect `models.mode: "replace"` by showing only configured provider models instead of authenticated built-in provider catalogs.

## Best Possible Solution

Land one narrow default-list fix that gates authenticated catalog rows under replace mode while preserving explicit full-catalog browse behavior, then let the merged PR close this issue.

## Reproduction Assessment

Yes, at source level: current main appends authenticated catalog rows in the default model-list path without checking `models.mode`, and the reporter provided concrete v2026.6.8 output showing leaked providers. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes, the best fix shape is narrow: gate the default authenticated catalog append for replace-mode configs while preserving explicit `--all` and provider-filtered catalog browsing. https://github.com/openclaw/openclaw/pull/94825 is the strongest current candidate, but it is not merged yet.
