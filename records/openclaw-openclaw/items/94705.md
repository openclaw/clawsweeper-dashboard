## Summary

Keep open: current main and v2026.6.10 still append auth-backed catalog rows in the default `openclaw models list` path without a replace-mode gate, and the linked fixes are still open PR candidates rather than merged implementation.

## What This Changes

The issue asks for `openclaw models list` to respect `models.mode: "replace"` by showing only configured provider/model entries instead of authenticated built-in catalog rows.

## Best Possible Solution

Review and land one focused linked fix that gates default catalog appends in replace mode while preserving documented explicit browse behavior such as `openclaw models list --all`.

## Reproduction Assessment

Yes. Source inspection on current main shows the default `models list` path still calls `appendAuthenticatedCatalogRows` without checking `models.mode`, which matches the reported leak of authenticated built-in providers.

## Solution Assessment

Yes for the issue direction, but not by opening another fix PR. The best path is to review and land one of the existing focused candidates with coverage for default list, provider-filtered list, and `--all` behavior.
