## Summary

Keep open: current main still has a distinct `openclaw models list` gap where the default CLI list path appends authenticated catalog rows without a `models.mode: "replace"` filter. Older related work fixed Control UI/gateway configured-view and CLI picker surfaces, but not this local CLI listing command.

## What This Changes

The issue reports that `models.mode: "replace"` does not constrain `openclaw models list` to configured providers.

## Best Possible Solution

Update the CLI model-list default source selection so replace mode lists only configured/allowlisted provider rows, while preserving `--all` as the explicit full-catalog diagnostic path.

## Reproduction Assessment

Source-reproducible: the default CLI list path includes authenticated catalog rows after configured rows, and the suppression helper has no replace-mode filter. I did not run the CLI because the review is read-only and runtime commands may create state.

## Solution Assessment

Yes, if implemented narrowly in the CLI list source selection: default list output should share configured-only behavior under replace mode, while `openclaw models list --all` remains the full diagnostic catalog.
