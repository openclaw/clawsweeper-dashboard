## Summary

Keep open. The durable-delivery direction is useful, but current PR head still stores command retry text before outbound filtering and replays pending heartbeat replies without preserving the original delivery route.

## What This Changes

The PR adds durable pending-final-delivery state for main sessions/subagents, restart and heartbeat replay handling, dispatch clearing, tests, and a changelog entry.

## Best Possible Solution

Keep durable retry state, but persist only sendable payloads together with the original delivery context and replay/clear them through the same delivery contract used by the first attempt.

## Reproduction Assessment

Yes. Source inspection of PR head shows raw command payload capture before outbound filtering and heartbeat replay through `getReplyFromConfig`; current docs also show CLI delivery can target a route separate from the main session.

## Solution Assessment

No. Durable retry is the right direction, but this patch must preserve the original route and reuse sendable-payload filtering before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Replay pending replies through the original route:** `src/auto-reply/reply/get-reply.ts:343`
  - body: The heartbeat replay path returns stored `pendingFinalDeliveryText` as a normal reply without persisting or using the original channel, target, account, or thread. CLI runs can use an agent main session while delivering to Slack or Telegram, so a retry can surface the completed reply in a different route than the failed original delivery.
  - confidence: 0.87
- **[P2] Filter command retry text through sendable payloads:** `src/agents/agent-command.ts:1257-1259`
  - body: This persists every raw payload's text before `deliverAgentCommandResult` builds the outbound payload plan. That plan suppresses `isReasoning` payloads, so a failed send or restart after this write can make hidden reasoning text replay later as a plain heartbeat reply.
  - confidence: 0.88
