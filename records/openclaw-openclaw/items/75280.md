## Summary

Keep open. The latest PR head still persists and replays pending final text before matching the existing delivery-policy boundary, so hidden/tool-only/reasoning or local command output can become visible after heartbeat or restart recovery.

## What This Changes

The PR adds durable pending-final-delivery state for main and subagent sessions, heartbeat/restart replay, dispatch clearing, focused tests, and a changelog entry.

## Best Possible Solution

Keep durable retry state, but derive it from the same policy-eligible sendable payload set used by final dispatch and mark or clear command delivery only when an external send is requested and confirmed queued or routed.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a message_tool_only or send-policy-denied final reply, a reasoning-only final payload, or an agent command run with deliver false; the PR can persist that text and heartbeat replay can return it as normal visible text.

## Solution Assessment

No. Durable retry is the right direction, but this patch stores every final or local payload before proving it is visible and delivery-eligible, so the safer fix is to move capture and clearing behind the existing delivery contract.
