## Summary

Keep open. The PR addresses a real durable-delivery gap, but current head still captures retry text before existing visibility gates and clears command retry state without confirmed channel delivery, so it can replay suppressed, hidden, or local-only output.

## What This Changes

The PR adds durable pending-final-delivery state for main sessions and subagent completion delivery, heartbeat/restart replay paths, dispatch clearing, tests, and a changelog entry.

## Best Possible Solution

Keep durable retry state, but capture only policy-eligible non-reasoning outbound payloads at the delivery boundary and clear markers only after confirmed queued or routed delivery across generic, hook, and command paths.

## Reproduction Assessment

Yes. Static reproduction paths are a message_tool_only or sendPolicy-denied final reply, a reasoning final payload, or an agent command run with deliver false or best-effort delivery failure; the PR stores or clears retry state before those paths are proven visible and delivered.

## Solution Assessment

No. Durable delivery retry is the right direction, but this patch places capture, replay, and cleanup outside the existing delivery contract; the safer implementation is to derive retry state from the same sendable payload set and confirmed delivery result used by dispatch.

  - confidence: 0.88

  - confidence: 0.84
