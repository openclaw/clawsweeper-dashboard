## Summary

Keep open. The latest head still persists final text before the existing delivery eligibility gates, so private, suppressed, reasoning-only, or local command output can enter durable retry state and later be replayed or cleared incorrectly.

## What This Changes

The PR adds durable pending-final-delivery state across main-session and subagent delivery, heartbeat/restart replay paths, dispatch clearing, tests, and a changelog entry.

## Best Possible Solution

Keep durable retry state, but derive it from the same policy-eligible sendable payload set used by final dispatch and clear it only after confirmed queued or routed outbound delivery.

## Reproduction Assessment

Yes. Static reproduction paths are a message_tool_only or send-policy-suppressed final reply, a reasoning final payload, or an agent command run with deliver false or best-effort delivery failure; the PR stores those payloads before current delivery gates decide they are visible and sendable.

## Solution Assessment

No. Durable retry is the right direction, but this patch captures and clears retry state outside the existing delivery contract; the safer solution is to move capture and cleanup behind dispatch/outbound success semantics.
