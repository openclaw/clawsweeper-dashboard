## Summary

Keep open. The durable-delivery direction is useful, but the current PR still persists final text before the existing delivery-intent and hidden-payload filters run, which can replay suppressed output later through heartbeat recovery.

## What This Changes

The PR adds durable pending-final-delivery state for main/subagent sessions, replays pending main-session final text through restart/heartbeat recovery, clears markers after successful final dispatch, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the durable retry marker, but persist only dispatch-eligible final payload text when automatic source delivery is actually intended, then clear the marker after the real final-dispatch success boundary.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a `message_tool_only` or send-policy-denied turn that returns final text: the PR stores `pendingFinalDeliveryText`, dispatch suppresses final delivery and does not clear it, and a later heartbeat can replay that text.

## Solution Assessment

No. The approach is directionally maintainable, but the capture gate is too broad; it should reuse the same delivery-intent and sendable-payload filtering as final dispatch.

Full review comments:

- **[P2] Gate pending capture to deliverable automatic replies:** `src/auto-reply/reply/agent-runner.ts:1814`
  - body: The marker is written for any final payload before dispatch applies source-delivery suppression. In `message_tool_only` or send-policy-denied turns, dispatch intentionally keeps final output private and will not clear this marker, so heartbeat recovery can later deliver text that should have stayed local.
  - confidence: 0.9
- **[P2] Filter hidden payloads before persisting retry text:** `src/auto-reply/reply/agent-runner.ts:1815-1818`
  - body: `pendingText` is built from every payload text, while final dispatch skips reasoning and other non-user-visible payloads. If that marker is retried later, hidden text is returned as a plain final reply, so build the marker from the same sendable payload set dispatch would actually send.
  - confidence: 0.88
- **[P2] Only mark command payloads when delivery is requested:** `src/agents/agent-command.ts:1204`
  - body: This branch marks main-session agent command output as pending even when `opts.deliver` is false and the expected result is local/JSON output. A restart between the marker write and cleanup can leave a durable marker that recovery sends to the session's last channel.
  - confidence: 0.83
