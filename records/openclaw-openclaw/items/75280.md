## Summary

Keep open. The latest head still persists final text before the existing delivery-eligibility checks, so suppressed or hidden replies can be replayed later as ordinary visible output; the agent-command path also marks local/non-delivered output as pending.

## What This Changes

The PR adds durable pending-final-delivery state for main and subagent sessions, restart and heartbeat replay paths, dispatch clearing, focused tests, and a changelog entry.

## Best Possible Solution

Keep the durable retry state, but persist only visible delivery-eligible final payloads when automatic channel delivery is intended, and clear markers only after a proven route or queued send success.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a message_tool_only or send-policy-denied final reply, a reasoning-only final payload, or an agent command run with deliver:false; the PR can persist that text and heartbeat replay can return it as normal visible text.

## Solution Assessment

No. The durable-retry direction is sound, but the capture and clear boundaries need to match the existing delivery contract instead of storing every final payload or local command result.
