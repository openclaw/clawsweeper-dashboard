## Summary

Keep open: the durable delivery direction is valid, but the current head persists and replays final text outside the existing delivery visibility contract, creating replay paths for suppressed, reasoning, or local command output.

## What This Changes

The branch adds durable pending-final-delivery state and restart/heartbeat replay handling for main sessions and subagents, plus dispatch cleanup tests and a changelog entry.

## Best Possible Solution

Keep durable retry state, but derive it from the same policy-eligible sendable payload set used by final dispatch and clear it only after confirmed queued or routed outbound delivery.

## Reproduction Assessment

Yes. Static reproduction paths are a `message_tool_only` or sendPolicy-denied final reply, a reasoning final payload, or an agent command run with `deliver: false` or best-effort delivery failure followed by heartbeat/restart replay.

## Solution Assessment

No. Durable final-delivery retry is the right product direction, but this implementation captures and clears state outside the existing delivery contract; the safer fix belongs inside the policy-filtered dispatch and command delivery success paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate retry capture to sendable final replies:** `src/auto-reply/reply/agent-runner.ts:1815-1827`
  - body: The marker is written before dispatch applies `sourceReplyDeliveryMode` or sendPolicy suppression. A `message_tool_only` or send-policy-denied turn can persist text that normal delivery intentionally keeps private, and heartbeat replay can later return it as a visible reply.
  - confidence: 0.91
- **[P2] Filter hidden payloads before persisting retry text:** `src/auto-reply/reply/agent-runner.ts:1816-1819`
  - body: `pendingText` joins every final payload's text, but dispatch skips replies where `isReasoning` is true. If a reasoning payload is captured here, heartbeat replay returns it as plain visible text instead of preserving the hidden reasoning lane.
  - confidence: 0.89
- **[P2] Only mark command output when delivery is requested:** `src/agents/agent-command.ts:1250-1264`
  - body: This branch marks main-session command output before checking `opts.deliver`. For local, JSON, or stdout-only agent commands, a restart between the marker write and cleanup can replay local command output to the session's channel.
  - confidence: 0.88
- **[P2] Keep command markers until delivery succeeds:** `src/agents/agent-command.ts:1288-1303`
  - body: The marker is cleared whenever `deliverAgentCommandResult` returns, but that function also returns for `deliver=false` and for best-effort per-payload failures reported through `onError`. Clearing here can drop retry state without a confirmed queued or routed send.
  - confidence: 0.84
