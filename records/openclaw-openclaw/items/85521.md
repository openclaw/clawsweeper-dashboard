## What This Changes

The issue reports that `saveAuthProfileStore` overwrites the active runtime auth-profile snapshot with a disk-filtered store, dropping runtime-only external CLI OAuth credentials such as `anthropic:claude-cli` from in-process state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85558, or an equivalent narrow auth-store fix, then close this issue through the merged fixing PR.

## Reproduction Assessment

Yes for source-level reproduction: current main still filters runtime-only external OAuth profiles for persistence and then writes that filtered store into the active runtime snapshot. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs at the auth-profile snapshot boundary: keep disk persistence filtered while preserving or rehydrating runtime-only external OAuth profiles in active snapshots. The open fixing PR is the right current path to review rather than opening another repair lane.
