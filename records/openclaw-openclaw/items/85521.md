## Summary

Current main and the latest release still contain the reported auth-profile snapshot bug, and an open PR already references this issue with closing syntax, so this issue should stay open until that fix or an equivalent one lands.

## What This Changes

The issue reports that `saveAuthProfileStore` can replace the active runtime auth-profile snapshot with a disk-filtered store and drop runtime-only external CLI OAuth credentials such as `anthropic:claude-cli`.

## Best Possible Solution

Land a narrow auth-profile store fix that keeps disk persistence filtered while refreshing active runtime snapshots with a runtime-safe store that preserves runtime-only external OAuth profiles.

## Reproduction Assessment

Yes for source-level reproduction: current main still filters runtime-only external OAuth profiles for persistence and then writes that filtered store into the active runtime snapshot. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs at the auth-profile snapshot boundary: keep disk persistence filtered while preserving or rehydrating runtime-only external OAuth profiles in active snapshots. The open fixing PR is the right current path to review rather than opening another repair lane.
