## Summary

Keep this open: current main still has the source-backed auth-profile runtime snapshot bug, and two open closing PRs already target the fix. The issue should remain open until one of those PRs lands or maintainers choose a different canonical fix path.

## What This Changes

The issue reports that `saveAuthProfileStore` replaces the active runtime auth-profile snapshot with a disk-filtered store, dropping runtime-only external OAuth profiles from embedded-agent credential resolution.

## Best Possible Solution

Review and land the focused canonical fix, preferably https://github.com/openclaw/openclaw/pull/85558 or an equivalent patch that preserves disk filtering while keeping active runtime snapshots hydrated, then close this issue after merge.

## Reproduction Assessment

Yes, source-level: current main still writes the disk-filtered `localStore` into an active runtime snapshot, and snapshot-backed embedded auth reads can then resolve without the external OAuth profile. I did not run a live gateway reproduction because this review was read-only.

## Solution Assessment

Yes, the best fix belongs at the auth-profile store snapshot boundary: keep external-CLI credentials out of persisted files while updating the active runtime snapshot from a runtime-safe full store or equivalent overlay path. The open PRs are implementation candidates, so the issue should remain open until one lands.
