## Summary

Keep open: the diagnostic direction is useful, but the current patch sends a raw config-derived model string to both doctor output and debug logs, so it needs a small sanitization repair before merge.

## What This Changes

Adds a debug warning, a doctor warning hook, and resolver unit tests for string-form `agents.defaults.model` configs that cannot carry fallback lists.

## Best Possible Solution

Keep the diagnostic approach, but route displayed model refs through the existing log/doctor sanitizers and add unsafe-character regression coverage while preserving current string-primary/object-fallback semantics.

## Reproduction Assessment

Yes, at source level: current main returns `[]` for string model input, and runtime fallback candidate construction consumes that helper. I did not run a live overload scenario in this read-only review.

## Solution Assessment

No. Warning users is a reasonable narrow fix direction, but this patch should sanitize config-derived model strings before terminal or log output and cover unsafe characters.
