## Summary

This PR should stay open because it addresses a real diagnostics gap for string-form model configs, but the current patch still emits config-derived model strings directly into doctor output and debug logs. The branch has after-fix terminal proof, so the remaining work is a narrow sanitization repair rather than a reason to close.

## What This Changes

The PR adds a one-time debug diagnostic for string model configs, wires an `openclaw doctor` warning for `agents.defaults.model` strings, and adds resolver tests for fallback extraction.

## Best Possible Solution

Keep the warning-based diagnostics, but sanitize model refs in both the doctor note and debug log, add unsafe-character regression coverage, and preserve the documented string-as-primary-only config semantics.

## Reproduction Assessment

Yes. Source inspection shows string model input reaches `resolveAgentModelFallbackValues()` and resolves to an empty fallback list, while the PR body demonstrates the patched diagnostic behavior with copied terminal output.

## Solution Assessment

No. The diagnostic approach is narrow and appropriate, but the current implementation should sanitize config-derived model strings before writing them to doctor output or debug logs.
