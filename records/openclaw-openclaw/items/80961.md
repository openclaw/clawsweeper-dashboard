Keep open: the PR addresses a real source-backed fallback diagnostics gap and has sufficient real behavior proof, but the latest diff still sends config-derived model strings to doctor output and debug logs without the existing sanitizer.

## What This Changes

The PR adds a debug warning, an `openclaw doctor` warning, and resolver unit tests for string-form `agents.defaults.model` configs that cannot carry fallbacks.

## Best Possible Solution

Keep the diagnostic approach, sanitize displayed model refs through the existing log/doctor sanitizers, and add unsafe-character regression coverage without changing string-primary/object-fallback semantics.

## Reproduction Assessment

Yes. At source level, current main returns `[]` for string model input and the runtime fallback path consumes that helper; the PR body also shows after-fix live output for the helper and doctor warning.

## Solution Assessment

No. Warning users is a reasonable narrow direction, but this implementation should sanitize config-derived model strings before terminal/log output and cover unsafe characters in tests.
