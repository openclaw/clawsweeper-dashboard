## Summary

Keep open: the PR addresses a valid source-backed fallback diagnostic, but the latest diff still interpolates config-controlled model strings directly into terminal and debug-log output. That needs a narrow sanitization repair before merge.

## What This Changes

The branch adds a debug warning, a doctor diagnostic, and resolver tests for string-form `agents.defaults.model` values that cannot carry fallbacks.

## Best Possible Solution

Keep the diagnostic approach, sanitize model refs before both terminal and log output, and add unsafe-character regression coverage while preserving string-primary/object-fallback semantics.

## Reproduction Assessment

Yes, at source level: current main returns `[]` for string input in `resolveAgentModelFallbackValues()`, and runtime fallback/gateway paths consume that helper. I did not run a live overload scenario in this read-only review.

## Solution Assessment

No. Warning users is a reasonable narrow direction, but this implementation should sanitize config-derived strings before emitting terminal notes or debug logs and should cover the unsafe-character case in tests.
