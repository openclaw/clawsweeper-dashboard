## Summary

Keep this open: current main still renders the Runtime agent label from the technical slot id, and an open linked PR with closing syntax already targets the exact fix. The issue should close after that PR or an equivalent patch reaches main.

## What This Changes

The issue asks OpenClaw to render the configured agent identity name in Runtime prompt metadata instead of only the technical agent slot id.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that threads `identity.name` into runtime prompt metadata while preserving the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: current main passes only `agentId` into runtime prompt metadata and renders that value as `agent=...`, so a config with slot id `main` and identity name `Runt` cannot render `Runt` in the Runtime line. I did not reproduce the probabilistic model response drift in a live Telegram session.

## Solution Assessment

Yes. Threading the existing configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader follow-up, not required for this bug.
