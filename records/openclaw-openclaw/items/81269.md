## Summary

Keep this open: current main and the latest release still render the Runtime agent label from the technical slot id, and https://github.com/openclaw/openclaw/pull/81308 is an open closing PR for the same bug. The issue should close after that PR, or an equivalent fix, lands on main.

## What This Changes

The issue asks for the Runtime prompt line to use configured `agents.list[].identity.name` instead of the technical agent slot id, while preserving the slot id for diagnostics.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name` through the existing agent config path, displays it first, and keeps the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches current `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader follow-up, not required for this bug.
