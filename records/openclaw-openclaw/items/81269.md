## Summary

Keep this open: current main still renders the Runtime line from the technical agent slot id and does not thread the configured identity name into the runtime prompt inputs.

## What This Changes

The issue asks OpenClaw to stop presenting only the agent slot id in the Runtime system-prompt line when `agents.list[].identity.name` is configured.

## Best Possible Solution

Carry the resolved identity name into runtime prompt metadata and render it before or alongside the stable agent id, without adding a new prompt prefix/suffix config surface for this bug.

## Reproduction Assessment

Yes, as a source-level reproduction: a distinct `agentId` and `identity.name` still produce `Runtime: agent=<slot id>` because only `agentId` is available to `buildRuntimeLine`. I did not establish a live Telegram reproduction in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to thread the already-configured identity name into `runtimeInfo` and render it with the slot id preserved for diagnostics. Adding `systemPromptPrefix` or `systemPromptSuffix` would be a broader feature and should not be required for the default behavior fix.

## Repair Work Prompt

Fix the Runtime system-prompt identity label bug reported in https://github.com/openclaw/openclaw/issues/81269. When an agent has a stable slot id such as `main` and a configured `agents.list[].identity.name` such as `Runt`, the Runtime line should not present only `agent=main` as the model-facing identity anchor. Thread the resolved identity name from agent config into the runtime prompt params and render it before or alongside the stable agent id, preferably preserving the technical id for diagnostics, for example `name=Runt | agent=main`. Do not add a new `systemPromptPrefix`, `systemPromptSuffix`, or generic instructions config in this PR. Add focused regression tests for distinct id/name in `buildSystemPromptParams`, `buildRuntimeLine`, and the assembled agent system prompt. Update changelog for the user-visible prompt behavior fix; docs only if the rendered Runtime fields are documented. Validate with the focused agent prompt tests and changed checks.
