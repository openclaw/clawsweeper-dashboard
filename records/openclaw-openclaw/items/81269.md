## Summary

Keep this open: current origin/main and the latest release still render the Runtime `agent=` value from the technical slot id, while an open linked PR with closing syntax is already the active fix path.

## What This Changes

The issue requests that Runtime prompt metadata use `agents.list[].identity.name` for the visible agent label instead of only the stable agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name`, displays it first, and keeps the stable slot id for diagnostics.

## Reproduction Assessment

Yes, at source level: a config with slot id `main` and `identity.name` `Runt` still reaches current `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; prompt prefix/suffix config would be a broader follow-up, not required for this bug.
