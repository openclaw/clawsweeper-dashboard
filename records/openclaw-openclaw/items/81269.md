## Summary

Keep this open: current main still renders the Runtime line from the technical agent slot id only, and no duplicate or open PR appears to own the fix.

## What This Changes

The issue asks OpenClaw to use configured `identity.name` in runtime prompt metadata instead of anchoring the model only on the agent slot id.

## Best Possible Solution

Thread the resolved identity name into runtime prompt metadata and render it before or alongside the stable agent id, without adding prompt prefix/suffix config for this bug.

## Reproduction Assessment

Yes, as a source-level reproduction: `buildSystemPromptParams` accepts config and `agentId` but returns runtime metadata with only `agentId`, and `buildRuntimeLine` renders that as `agent=<slot id>`. I did not establish a live Telegram reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the already-configured identity name into runtime prompt metadata while preserving the stable id for diagnostics; `systemPromptPrefix` or `systemPromptSuffix` would be a broader config feature.
