## Summary

Keep this open: current main still renders the Runtime `agent=` value from the technical slot id and does not thread `identity.name` into runtime prompt metadata. The linked fix PR is open and references this issue with closing syntax, so the issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to render configured `agents.list[].identity.name` in the Runtime prompt instead of using only the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308, or an equivalent narrow fix that resolves `identity.name` into runtime metadata and preserves the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live chat session.

## Solution Assessment

Yes. Threading the already-configured identity name into `runtimeInfo` and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader feature, not required for this bug.
