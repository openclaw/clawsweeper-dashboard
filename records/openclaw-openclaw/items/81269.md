## Summary

Keep this open: current main and the latest release still render the Runtime agent label from the technical slot id, and an open linked PR already owns the fix path with closing syntax. The source-level bug is clear, and martingarramon's comment confirms the same code gap.

## What This Changes

The issue asks for the system-prompt Runtime metadata to prefer `agents.list[].identity.name` over the agent slot id when labeling the agent.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves the configured identity name through the agent config, renders it first, and preserves the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches current main with only `agentId`, so the Runtime line can only render `agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix or suffix config would be a broader follow-up feature, not required for this bug.
