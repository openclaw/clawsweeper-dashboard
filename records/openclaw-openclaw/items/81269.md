## Summary

Keep this open: current main still renders the Runtime `agent` value from the technical slot id and prompt params do not carry `identity.name`; the linked closing PR is still open and should be reviewed or landed before this issue closes.

## What This Changes

The issue requests that Runtime prompt metadata use configured `agents.list[].identity.name` instead of the technical agent slot id when present.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308, or an equivalent narrow fix that resolves the configured identity name into runtime metadata while preserving the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches Runtime prompt assembly with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the existing configured identity name into `runtimeInfo` and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader feature, not required for this bug.
