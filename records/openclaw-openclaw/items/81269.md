## Summary

Keep this open: current main still renders the Runtime line from the technical agent id, and the issue is already paired with an open fix PR that uses closing syntax.

## What This Changes

The issue asks for runtime prompt metadata to use `agents.list[].identity.name` instead of only the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that threads `identity.name` into runtime prompt metadata while preserving the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: current main only passes `agentId` into runtime prompt params and renders it as `agent=...`, so a slot id `main` with `identity.name` `Runt` cannot render `Runt` there. I did not reproduce the intermittent Telegram model drift live.

## Solution Assessment

Yes. The narrow maintainable fix is to carry the configured identity name through the existing prompt-param path and render it before or alongside the stable id; adding prompt prefix/suffix config would be broader than this bug requires.
