## Summary

Keep this open: current main still renders the Runtime agent label from the technical slot id, and an open linked PR uses closing syntax for this issue. The source-level bug is clear, but the issue should close only after that PR lands or an equivalent fix reaches main.

## What This Changes

The issue requests changing the Runtime system-prompt line to use `agents.list[].identity.name` instead of the technical agent slot id, while optionally preserving the slot id for diagnostics.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name` through the agent config, displays it first, and keeps the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches current `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader follow-up, not required for this bug.
