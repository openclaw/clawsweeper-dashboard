## Summary

Keep this open: current main still renders the Runtime agent label from the technical slot id, and the linked fix PR is open with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests changing the Runtime system-prompt line to use the configured `agents.list[].identity.name`, or to show it before the technical agent id, so agents do not identify as slot ids.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308, or an equivalent narrow fix that resolves `identity.name` into runtime metadata and preserves the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into `runtimeInfo` and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader feature, not required for this bug.
