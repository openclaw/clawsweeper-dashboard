## Summary

Keep this open: current main still renders the Runtime `agent=` value from the technical slot id only, and an open linked PR uses closing syntax and directly addresses the missing `identity.name` plumbing.

## What This Changes

The issue requests that the Runtime system-prompt line use the configured agent identity name instead of only the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308, or an equivalent narrow fix that displays the configured identity name while preserving the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches Runtime prompt assembly with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader follow-up, not required for this bug.
