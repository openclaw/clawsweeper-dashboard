## Summary

Keep this open: current main still builds the Runtime agent label from the technical slot id, and the linked fix PR is open with closing syntax for this issue, so the issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to use the configured agent identity name, rather than the technical agent slot id, in Runtime system-prompt metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name`, displays it first, and keeps the stable slot id available for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches current `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live Telegram session.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader follow-up, not required for this bug.
