## Summary

Keep this open: current `main` and the latest release still render the Runtime `agent=` value from the technical slot id, while https://github.com/openclaw/openclaw/pull/81308 is an open linked fix with closing syntax. The source-level bug is clear, but this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks for the Runtime prompt line to use configured `agents.list[].identity.name` for the visible agent label instead of only the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name`, displays it first, and keeps the stable slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: a config with slot id `main` and `identity.name` `Runt` still reaches current `buildRuntimeLine` with only `agentId`, so current main can only render `Runtime: agent=main`. I did not establish the intermittent model-level drift in a live chat session.

## Solution Assessment

Yes. Threading the configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; adding prompt prefix/suffix config would be a broader optional follow-up.
