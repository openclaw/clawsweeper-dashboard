## Summary

Keep open: current main and latest release still render Runtime metadata from the technical agent id only, while an open linked PR already targets the fix and has not merged.

## What This Changes

The issue requests that Runtime prompt metadata use `agents.list[].identity.name` as the visible agent label instead of the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 if CI and maintainer review remain clean, preserving the slot id as diagnostic metadata while displaying the configured identity name first.

## Reproduction Assessment

Yes, at source level: a config can define a distinct `identity.name`, but current prompt params do not carry it and `buildRuntimeLine` renders only `agentId`. I did not run a live Telegram conversation.

## Solution Assessment

Yes. The narrow maintainable solution is to thread the existing configured identity name into runtime prompt metadata and keep the stable slot id separately for diagnostics; adding prompt prefix or suffix config is broader than this bug requires.
