## Summary

Keep this open: current main still builds the Runtime line from the technical agent slot id, does not pass the configured identity name into runtime prompt params, and the linked closing PR is still open rather than merged.

## What This Changes

The issue requests that runtime prompt metadata display `agents.list[].identity.name` as the agent label instead of only the stable agent slot id.

## Best Possible Solution

Land the linked narrow prompt plumbing that resolves `identity.name` through existing agent config, renders it as `agent=...`, and preserves the stable slot id separately for diagnostics.

## Reproduction Assessment

Yes. As a source-level path, current main's runtime prompt params cannot carry `identity.name`, and `buildRuntimeLine` can only render `Runtime: agent=<slot id>` for a config with a distinct id and identity name.

## Solution Assessment

Yes. The narrow maintainable fix is to thread the existing configured identity name into runtime metadata and keep the slot id as a separate diagnostic field; adding prompt prefix/suffix config would be broader than needed for this bug.
