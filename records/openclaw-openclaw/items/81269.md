## Summary

Keep this open: current main still renders the Runtime agent label from the technical slot id, and the linked closing PR remains open rather than merged. The source-level bug is clear and already has an active narrow fix path.

## What This Changes

The issue requests that Runtime prompt metadata use the configured agent identity name instead of only the technical agent slot id.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name`, displays it first, and preserves the stable slot id for diagnostics.

## Reproduction Assessment

Yes, at source level: a config with agent id `main` and `identity.name` `Runt` still reaches `buildRuntimeLine` with only `agentId`, so current main cannot render the configured identity name there. I did not establish the intermittent model-level drift in a live Telegram chat.

## Solution Assessment

Yes. Threading the already-configured identity name into runtime metadata and rendering it before or alongside the stable slot id is the narrow maintainable fix; additive prompt prefix/suffix config would be a broader follow-up, not required for this bug.
