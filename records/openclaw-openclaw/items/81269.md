## Summary

Keep this open: current main still renders the Runtime agent label from the technical slot id, while the linked fix PR remains open with closing syntax for this issue. The source-level bug is clear enough to route, but the issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Runtime system-prompt line use the configured agent identity name instead of only the technical agent slot id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81308 or an equivalent narrow fix that resolves `identity.name` through agent config, renders it first, and preserves the slot id for diagnostics.

## Reproduction Assessment

Yes, as a source-level reproduction: on current main, `buildSystemPromptParams` only carries `agentId` and `buildRuntimeLine` renders `agent=<agentId>`, so a config with id `main` and identity name `Runt` cannot render `Runt` in the Runtime line. I did not reproduce the intermittent model-level drift in Telegram.

## Solution Assessment

Yes. Threading the configured identity name into runtime metadata and keeping the stable slot id as a separate diagnostic field is the narrow maintainable fix; per-agent prompt prefix/suffix config would be a broader follow-up.
