## Summary

## What This Changes

## Best Possible Solution

Keep the classifier fix, scope poison state by account or spool plus update id, add cross-spool regression coverage, and require redacted live Telegram proof or an explicit maintainer override before merge.

## Reproduction Assessment

Yes, at source level: current main routes unclassified `bot.handleUpdate` failures through `releaseTelegramSpooledUpdateClaim`, so `ERR_MODULE_NOT_FOUND` and repeated unknown failures can retry. I did not run a live Telegram gateway reproduction.

## Solution Assessment

No. The classifier belongs in the Telegram polling owner module, but the poison counter needs account or spool scope plus real transport proof before this is the safest fix.
