## Summary

Keep open; current main still wakes only the sentinel session, and the PR addresses a real restart-recovery gap, but the branch needs rework before merge because its fan-out queues events without targeted heartbeat wakes and includes non-resumable session rows.

## What This Changes

The PR adds a gateway restart helper that enumerates agent session stores and enqueues GatewayRestart system events for the triggering session plus recently updated sessions.

## Best Possible Solution

Implement restart fan-out on current main using the same event-plus-heartbeat wake path for every selected resumable session, with deterministic selection, current session-kind filters, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main only wakes the sentinel session, and the PR’s fan-out path can be checked directly in the diff; I did not run tests because this was a read-only review.

## Solution Assessment

No. The direction is right, but this PR is not the best current implementation until it is rebased/replaced to use current heartbeat wake semantics and skip non-resumable session kinds.
