## Summary

Keep open. Current main still only injects the restart sentinel wake for the sentinel session, so the PR is not obsolete, but the proposed patch is not merge-ready because its cross-session fan-out enqueues system events without requesting heartbeat wakes and it selects session kinds that current recovery paths intentionally handle separately.

## What This Changes

The PR adds `wakeRecentlyActiveSessions()` to `src/gateway/server-restart-sentinel.ts` to enumerate agent session stores after a gateway restart and enqueue `GatewayRestart` system events for the triggering session plus sessions updated within the previous 60 seconds.

## Best Possible Solution

Implement this as restart-recovery infrastructure on current main: deterministic session selection, explicit filters for pseudo/cron/ACP/subagent/non-resumable sessions, one queued `GatewayRestart` plus `requestHeartbeatNow` per selected session, and focused tests covering cross-session selection and skipped session kinds. Do not expand this PR into the broader generic `sessions_broadcast` or plugin tool-refresh work.

## Reproduction Assessment

Yes. On current main, the restart startup task reaches `enqueueRestartSentinelWake` only for the sentinel session key, and a focused search finds no helper that enumerates other recently active session stores.

## Solution Assessment

No. The PR direction is plausible, but the branch should be rebased and reworked to use the current restart-sentinel wake/heartbeat path and the same resumable-session filters used by current recovery code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Request a heartbeat for every selected session:** `src/gateway/server-restart-sentinel.ts:179`
  - body: `wakeRecentlyActiveSessions` enqueues the restart event for additional sessions, but `enqueueSystemEvent` only stores text for the next prompt. Without also calling `requestHeartbeatNow({ reason: "wake", sessionKey })`, those sessions can still stay idle until a later heartbeat or user message, which is the behavior this PR is trying to fix.
  - confidence: 0.86
- **[P2] Skip non-resumable session kinds:** `src/gateway/server-restart-sentinel.ts:166-170`
  - body: The new loop only skips cron run keys plus `global` and `unknown`, so it can inject trusted restart events into ACP or subagent sessions that current recovery paths filter or handle separately. Add the existing ACP/subagent/cron and stored metadata filters before cross-session wake delivery.
  - confidence: 0.78
