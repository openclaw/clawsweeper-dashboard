## Summary

Keep open: current main still wakes only the sentinel session, but this PR is not mergeable because its fan-out path does not request heartbeat wakes and its session selection is broader than current resumable-session contracts allow.

## What This Changes

The PR modifies `src/gateway/server-restart-sentinel.ts` to enumerate agent session stores after restart and enqueue `GatewayRestart` events for the triggering session plus sessions updated in the prior 60 seconds.

## Best Possible Solution

Implement fan-out on current main using shared resumable-session selection and the same event-plus-heartbeat wake semantics for every selected session, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level: current main calls the restart wake once for the sentinel session, and the focused test asserts a single system event plus heartbeat. I did not run a live restart or tests in this read-only review.

## Solution Assessment

No. The PR direction is right, but the implementation should be rebased or replaced so fan-out uses current heartbeat wake semantics and current session-kind filters before merge.
