## Summary

Keep open: current main still has the Discord-visible restart interruption gap. The Discord route is persisted, but the abort path returns silently and restart recovery resumes internally with `deliver: false`, so the original channel still gets no automatic recovery notice.

## What This Changes

The issue asks OpenClaw to resume or visibly notify Discord users when a gateway restart interrupts an in-flight Discord-backed turn.

## Best Possible Solution

Implement one generic, idempotent post-restart interruption notice for external channel-originated main sessions using persisted delivery context or last-route metadata, while keeping hidden automatic resume limited to safe local/internal sessions.

## Reproduction Assessment

Yes, source-level with high confidence. Discord persists the route, the Discord abort branch returns silently, and current restart recovery resumes with `deliver: false` instead of notifying the originating channel.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if implemented channel-agnostically. A Discord-only fallback would be weaker than using the existing delivery-context and route contracts, and the repair should reconcile the overlap with https://github.com/openclaw/openclaw/pull/74687.
