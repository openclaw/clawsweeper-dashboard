## Summary

Keep open: current main still has the Discord-visible restart interruption gap. Restart-aborted main sessions are resumed through a hidden `deliver: false` agent turn, and the Discord abort path still returns without posting a recovery notice to the originating channel.

## What This Changes

Add automatic safe resume or a same-channel recovery notice for Discord-backed turns interrupted by a gateway restart.

## Best Possible Solution

Implement one generic, idempotent post-restart interruption notice for external channel-originated main sessions using persisted delivery context or last-route metadata, while keeping hidden automatic resume limited to safe local/internal sessions.

## Reproduction Assessment

Yes, source-level with high confidence. Discord persists enough route metadata, the Discord abort branch returns silently, and current restart recovery resumes with `deliver: false` instead of notifying the originating channel.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if implemented channel-agnostically. A Discord-only fallback would be weaker than using the existing delivery-context and route contracts, and the repair should reconcile the overlap with the open restart-recovery pull requests.
