## Summary

Keep open. The mathematical fix is valid, but this PR targets the old `src/memory/internal.ts` helper while current `main` routes runtime memory scoring through the memory-host SDK helper, which still has the same truncated-norm behavior.

## What This Changes

The PR adds trailing-dimension norm accumulation to `src/memory/internal.ts` and adds colocated `cosineSimilarity` regression tests under `src/memory/internal.test.ts`.

## Best Possible Solution

Port the zero-padding cosine semantics to `packages/memory-host-sdk/src/host/internal.ts`, cover the SDK helper and memory-core fallback search path, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce: current main computes `[1, 1]` versus `[1]` over only the shared first dimension for both norms, which returns `1` instead of `1 / Math.sqrt(2)`.

## Solution Assessment

No. The algorithm in the PR is the right mathematical direction, but the patch is aimed at a removed helper; the maintainable fix belongs in the active memory-host SDK helper and its callers' tests.
