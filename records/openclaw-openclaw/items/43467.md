## Summary

Keep open: the reported cosine-similarity bug is valid, but the PR targets a removed memory helper while current main still runs the truncating implementation through the memory-host SDK path.

## What This Changes

The PR updates the old `src/memory/internal.ts` cosine helper and adds tests under `src/memory/internal.test.ts` so different-length vectors include trailing dimensions in norm calculations.

## Best Possible Solution

Port zero-padding cosine semantics to `packages/memory-host-sdk/src/host/internal.ts`, cover that helper and the memory-core fallback search path, and add one `CHANGELOG.md` Fixes entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `cosineSimilarity([1, 1], [1])` accumulates both norms only for the shared first element, so the helper would return `1` instead of `1 / Math.sqrt(2)`; fallback search calls that helper.

## Solution Assessment

No. The algorithm in the PR is the right zero-padding direction, but the patch changes a stale path; the maintainable fix belongs in the active memory-host SDK helper with runtime-path coverage.
