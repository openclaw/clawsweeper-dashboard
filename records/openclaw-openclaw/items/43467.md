## Summary

Keep open. The reported cosine-similarity bug is valid on current main, but this PR edits the removed `src/memory/internal.ts` path, while active fallback vector search uses the memory-host SDK helper that still truncates norms to the shared prefix.

## What This Changes

The PR adds trailing-dimension norm accumulation to `src/memory/internal.ts` and adds colocated `cosineSimilarity` regression tests under `src/memory/internal.test.ts`.

## Best Possible Solution

Keep the zero-padding cosine semantics from this PR, but implement them in `packages/memory-host-sdk/src/host/internal.ts` with focused SDK tests and a fallback vector-search regression so the active product path cannot drift.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main computes `cosineSimilarity([1, 1], [1])` using only the shared first dimension for both norms, which yields `1` instead of `1 / Math.sqrt(2)`.

## Solution Assessment

No. The algorithm proposed by the PR is the right mathematical fix, but this branch targets a removed helper; the maintainable solution is to port the same norm semantics to the active memory-host SDK helper and tests.
