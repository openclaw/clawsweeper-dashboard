## Summary

Keep open. The reported cosine-similarity bug is still present on current main, but this PR changes the removed `src/memory/internal.ts` path instead of the active memory-host SDK helper used by fallback vector search, so it should not merge as-is.

## What This Changes

The PR adds trailing-dimension norm accumulation to `src/memory/internal.ts` and adds cosine-similarity regression tests under `src/memory/internal.test.ts`.

## Best Possible Solution

Keep the zero-padding cosine semantics from this PR, but implement them in `packages/memory-host-sdk/src/host/internal.ts` with focused package tests and, if practical, a fallback vector-search regression so the active product path cannot drift.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main computes `cosineSimilarity([1, 1], [1])` using only the shared first dimension for both norms, which yields an inflated score of `1` instead of `1 / Math.sqrt(2)`.

## Solution Assessment

No. The algorithm in this PR is the right mathematical fix, but the submitted patch is aimed at a removed path; the narrow maintainable solution is to port the same norm semantics to the active memory-host SDK helper and tests.
