## Summary

Keep open: the reported cosine-similarity bug is real, but this PR edits the old `src/memory/internal.ts` helper while current main routes memory fallback vector scoring through the memory-host SDK helper, which still has the truncating norm calculation.

## What This Changes

The PR adds trailing-dimension norm accumulation and tests for `cosineSimilarity` in the old `src/memory/internal.ts` helper.

## Best Possible Solution

Port the zero-padding cosine semantics to `packages/memory-host-sdk/src/host/internal.ts`, cover the helper and fallback vector-search path, and add a single user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `cosineSimilarity([1, 1], [1])` would accumulate both norms only for the shared first element and return `1`; fallback vector search calls that helper.

## Solution Assessment

No. The algorithm in the PR is the right zero-padding direction, but the patch edits a stale helper; the narrow maintainable fix belongs in the active memory-host SDK helper with runtime-path coverage.
