Fix macOS Canvas remote-node presentation so protected Canvas hosted paths load through gateway auth or capability URLs instead of showing Unauthorized.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/91443 with a node-session capability URL resolver for hosted Canvas/A2UI targets, refresh expired capabilities, preserve arbitrary external/local targets, and redact capability-bearing diagnostics.
