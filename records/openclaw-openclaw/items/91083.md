## What This Changes

The issue reports that macOS Canvas WKWebView loads hosted Canvas paths without gateway authorization when presenting content on remote nodes.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/91443 with a node-session capability URL resolver for hosted Canvas/A2UI targets, refresh expired capabilities, preserve arbitrary external/local targets, and redact capability-bearing diagnostics.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection still shows the plausible current-main failure path: node Canvas targets are passed unchanged and protected HTTP(S) Canvas loads use a plain WebView request without capability rewriting.

## Solution Assessment

Yes, the capability URL direction is the best fit. The safer repair should use the existing node-session current/refresh capability URL path rather than bearer-header injection, query tokens, or unauthenticated Canvas routes.
