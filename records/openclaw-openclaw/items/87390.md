## What This Changes

The PR makes the Real behavior proof parser fence-aware for Markdown section slicing, proof-field extraction, and missing-proof scans, with a regression test for copied headings and proof-template lines inside fenced terminal output.

## Best Possible Solution

Keep the fence-aware parsing direction, but reject fenced missing-proof values in the actual `What was not tested` field and add focused regression coverage before merge.

## Reproduction Assessment

Yes, by source inspection: an external PR body with real evidence plus `What was not tested:` as a fenced block containing only `not tested` would pass because the field check sees fence markers and the later section scan drops fenced lines.

## Solution Assessment

No. The fence-aware parsing approach is the right direction, but the current implementation is incomplete until field-level missing-proof values are validated before fenced transcript content is ignored.
