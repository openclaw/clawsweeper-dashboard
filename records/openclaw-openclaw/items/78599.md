## Summary

Keep this PR open, but it should not merge as-is: the claimed missing exports are not reproducible from current main's locked dependency contract, and the patch weakens fs-safe root-file and durable-queue protections while leaving the shared session queue path uncovered.

## What This Changes

The branch inlines root JSON and outbound durable queue helpers, adds a committed build log, and changes daemon status plus WebSocket preauth-close logging behavior and tests.

## Best Possible Solution

Confirm any real dependency-resolution mismatch, then fix the package/lock/upstream fs-safe boundary or add a shared wrapper that preserves fs-safe contracts; split daemon/WebSocket logging into a separate PR.

## Reproduction Assessment

No. Current main pins and locks @openclaw/fs-safe to 0.2.2, and the published 0.2.2 package exports the named JSON and durable queue APIs; the only failing evidence is the contributor's local log.

## Solution Assessment

No. Inlining raw-fs replacements is not the narrowest maintainable fix for a package-boundary issue, and the branch does not cover the shared session queue path that would still import the allegedly missing store helpers.

  - body: The new helper accepts rootRealPath/rejectHardlinks-style options but then reads through a path-prefix check plus fs.readFileSync.   - confidence: 0.92

  - body: The local queue helpers replace replaceFileAtomic, safe entry-id validation, restrictive modes, bounded no-follow reads, and corrupt-entry skipping with basic fs calls. That weakens persisted delivery recovery and can make scans throw or follow unsafe queue files.
  - confidence: 0.9

If @openclaw/fs-safe/store is really missing those exports, the build would still fail on that shared session queue path.
  - confidence: 0.86
