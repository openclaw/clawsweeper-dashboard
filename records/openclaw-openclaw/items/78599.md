## Summary

Keep open and do not merge as-is: the claimed missing exports are not reproduced against current main's lockfile and the published package contract, while the branch replaces fs-safe boundaries with weaker raw-fs code and leaves the shared session queue path unresolved.

## What This Changes

The branch inlines root JSON and outbound durable queue helpers, commits a local build log, and adds daemon status plus WebSocket pre-auth close logging changes.

## Best Possible Solution

If a real dependency-resolution mismatch exists, fix the package or lock boundary, or add a shared infra wrapper that preserves fs-safe contracts, covers both outbound and session queues, and splits daemon/WebSocket logging into separate focused work.

## Reproduction Assessment

No. Current main pins @openclaw/fs-safe to 0.2.2 in the lockfile, and the published 0.2.2 package exports the JSON and durable queue names this PR says are missing; I did not establish a failing current-main build path.

## Solution Assessment

No. Inlining raw filesystem replacements is not the narrowest maintainable fix for a package boundary issue; the safer path is to preserve fs-safe’s root and queue contracts or correct dependency resolution.
