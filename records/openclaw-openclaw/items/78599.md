## Best Possible Solution

Full review comments:

- **[P1] Keep root JSON reads on the fs-safe boundary:** `src/infra/json-files.ts:47`
  - body: The new helper replaces the upstream `readRootStructuredFileSync`/`openRootFileSync` path with a string-prefix check and `fs.readFileSync`. That drops `rootRealPath`, `rejectHardlinks`, `maxBytes`, and symlink/hardlink validation from the published fs-safe contract, so a symlink inside the root can still read outside the root. Please keep this on the fs-safe root-file boundary or a local equivalent with the same checks.
  - confidence: 0.92
- **[P2] Preserve the durable queue store contract:** `src/infra/outbound/delivery-queue-storage.ts:101-108`
  - body: The local durable-queue helpers drop the upstream store's safe entry-id validation, 0700/0600 modes, atomic replace behavior, bounded no-follow regular-file reads, and corrupt-entry skipping. This weakens persisted delivery recovery and can make queue scans throw or follow unsafe queue files; use the fs-safe helper or a shared equivalent with the same contract.
  - confidence: 0.9
- **[P2] Fix the remaining session queue imports:** `src/infra/outbound/delivery-queue-storage.ts:83`
  - body: This restores the allegedly missing store helpers only for outbound delivery, but current main also imports the same helpers in `src/infra/session-delivery-queue-storage.ts`. If `@openclaw/fs-safe/store` is really missing those exports, `pnpm build` will still fail there, so the fix needs to cover the shared session queue path too.
