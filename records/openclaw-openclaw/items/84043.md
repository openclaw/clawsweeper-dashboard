## Summary

Keep open for normal PR review and merge handling: this maintainer-labeled PR addresses a real source-proven update/protocol-skew bug that is not already fixed on current main, and I found no blocking correctness or security findings.

## What This Changes

The PR lowers the gateway probe protocol floor to v3, adds probe-only previous-protocol tests, emits a split shell/service root warning during package updates, and updates docs plus changelog.

## Best Possible Solution

Merge the probe-only compatibility floor and split-root diagnostic after required maintainer checks, while keeping normal clients on the current protocol floor.

## Reproduction Assessment

Yes from source inspection: current main's restart health path uses probeGateway, while the server still requires probe clients to support protocol v4, so a v3 updater probing a v4 gateway would hit protocol mismatch. I did not run a live package-update restart in this read-only review.

## Solution Assessment

Yes. Lowering only MIN_PROBE_PROTOCOL_VERSION while leaving MIN_CLIENT_PROTOCOL_VERSION at v4, and testing both probe acceptance and non-probe rejection, is the narrow maintainable fix for one-version update skew.
