## Summary

Keep open: current main still has only partial Discord mention safeguards and does not implement the central strict active-target behavior for other configured agents, quoted/later mentions, and role/bot edge cases.

## What This Changes

The issue requests stricter Discord preflight routing so the current agent only handles messages clearly addressed to it and drops messages aimed at other agents, unrelated roles, quoted mentions, or other configured bots.

## Best Possible Solution

Keep this open for maintainer consolidation: define Discord active-target precedence in preflight/routing, keep role-trigger and slash-command semantics aligned with their narrower open issues, and land focused regression coverage for the chosen policy.

## Reproduction Assessment

Yes, at source level: current main still resolves the Discord route before active target signals and checks mentions against full base text/current bot identity. I did not run a live Discord multi-agent reproduction in this read-only pass.

## Solution Assessment

No, the large local patch described in the report is only a plausible implementation. The best fix needs a maintainer-approved Discord target-precedence policy, then a bounded preflight/routing change with focused tests.
