## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause the wrong agent to answer or the intended agent not to answer.
- `impact:message-loss`: Discord messages can be dropped, consumed by a default owner, or routed away from the intended bot or agent.

## Summary

Keep open: current main and the latest release still resolve Discord routing before cross-bot target selection, compare mention/reply metadata only to the current handler bot, and scan configured mention text without the reported opening-attention window. The remaining issue is valid, but the final precedence and role semantics need maintainer product review before a repair lane should act.

## What This Changes

The issue requests tighter Discord preflight routing so other-agent, role, quoted/later mention, and bot-authored signals do not incorrectly admit messages for the current agent.

## Best Possible Solution

Confirm the Discord active-target contract, then implement the Discord-owned target selection before default routing with a configured bot/account/agent map, bounded attention-text matching, role handling, bot-loop suppression, and focused tests for default-owner and other-agent cases.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before mention/reply target selection, compares mention/reply metadata only to the current handler botUserId, and lacks the reported attention-window guard. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed; the requested direction is plausible, but the durable fix needs an approved precedence contract for cross-bot targets, role mentions, quoted text, and bot-authored messages.
