## Summary

Keep open: current main still scopes Discord mention detection to the full message text and the running bot only, while the related Discord target-routing, role-trigger, slash-command, and suppression fixes are still open partial items rather than merged behavior.

## What This Changes

Tighten Discord preflight mention routing so only the currently addressed agent wakes, while ignoring later quoted mentions, other-agent targets, unrelated roles, and cross-agent bot messages.

## Best Possible Solution

Implement one coordinated Discord active-target policy in the plugin: derive bounded opening attention text, use native Discord mention/reference metadata and configured bot/account bindings, suppress non-target agents, and align role and slash-command behavior with the related trackers.

## Reproduction Assessment

Yes, at source level. Current main still passes full baseText into mention matching and has no configured-other-agent competing-target check; I did not run a live Discord route in this read-only review.

## Solution Assessment

Unclear for the full local patch shape. Bounded attention and competing-agent suppression look directionally right, but role labels, plain-text delegation, bot-authored messages, and slash-command targeting need maintainer policy alignment first.
