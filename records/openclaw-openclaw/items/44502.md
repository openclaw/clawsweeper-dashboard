## Summary

Keep open: current main and the latest release still resolve Discord routing before active target selection, use only the running bot account for explicit mention/reply checks, and do not implement the requested attention-window or other-configured-agent suppression. The open related issues cover important slices, but none fully replaces this report.

## What This Changes

The issue requests stricter Discord preflight/mention routing so only the actively addressed configured agent handles a guild message and quoted/later or unrelated role/agent signals are ignored.

## Best Possible Solution

Confirm the Discord active-target precedence policy, then implement it in the Discord plugin preflight/routing path with focused tests for explicit other-agent targets, reply targets, quoted/later mentions, role mentions, and configured bot senders.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before target selection and checks mentions/replies only against the running bot user ID, with full baseText used for text mention matching. I did not run a live Discord multi-agent repro in this read-only review.

## Solution Assessment

No, the large local patch described in the report is only a plausible implementation. The best fix needs maintainer-approved Discord target-precedence semantics, then a bounded plugin preflight/routing change with regression coverage.
