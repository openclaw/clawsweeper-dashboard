## Label Justifications

- `impact:message-loss`: The reported gating decisions can suppress intended messages or cause them to be consumed by the wrong Discord agent.

## Summary

Current main still scans the complete message for configured mention patterns, so later quoted or pasted text can satisfy Discord mention gating; however, the report also combines this reproducible defect with separate product-policy questions about role triggers and plain-text agent delegation.

## What This Changes

Tighten Discord inbound preflight so only the intended agent responds, excluding misleading quoted mentions, unrelated roles, cross-agent bot traffic, and messages targeting another configured agent.

## Best Possible Solution

Implement a focused Discord-plugin fix that prevents configured text patterns in clearly quoted or non-current content from satisfying mention gating while preserving native mention IDs and replies; handle role triggers and free-text cross-agent delegation through their canonical product discussions.

## Reproduction Assessment

Yes at source level for the quoted-text class: configured mention regexes run against the complete message body. The competing-agent, role, and bot-loop scenarios still need focused current-main or live Discord verification.

## Solution Assessment

No, the reported 954-line combined approach is not yet the best solution. A narrow quoted-text regression fix should use existing native identity and routing facts, while role and plain-name targeting require separate product decisions.
