## Summary

Keep open: current main still uses whole-message Discord mention matching, current-bot-only mention/reply state, and generic other-mention suppression, so the requested strict attention-window and configured-agent targeting behavior is not implemented. Related open PRs and issues are useful context, but none has landed the central fix.

## What This Changes

Tighten Discord preflight so only the actively addressed current agent passes mention gating, while dropping quoted/later mentions, unrelated roles, other configured-agent targets, and configured-agent bot loops.

## Best Possible Solution

Land one focused Discord plugin repair that narrows mention matching to an opening attention window, computes configured-agent targets from reliable Discord metadata/config, suppresses default/current-agent handling when another configured agent is targeted, and adds colocated regression tests.

## Reproduction Assessment

Yes, at source level. The issue gives concrete multi-agent Discord scenarios, and current main still passes full message text into mention detection without attention-window or competing configured-agent checks.

## Solution Assessment

Yes in direction, but it needs maintainer scoping. The maintainable fix is a Discord-owned preflight/routing change, reconciled with the open route-owner and suppression PRs and kept separate from broader role-trigger or content-routing features.
