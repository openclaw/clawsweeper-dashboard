## Summary

Keep open: current main still uses broad Discord mention matching and current-bot-only mention/reply state, while the related open Discord routing PRs and issues show the coordinated fix has not landed.

## What This Changes

The issue asks Discord preflight to tighten multi-agent mention gating by limiting active mention detection, dropping other-agent or unrelated-role targets, ignoring quoted later mentions, and preventing bot-loop routing cases.

## Best Possible Solution

Implement one Discord-owned routing repair that derives active target facts from Discord mention/reference metadata and configured agent bindings, suppresses non-target agents before default routing, and covers attention-window, role, and bot-loop cases with focused preflight tests.

## Reproduction Assessment

Yes, at source level. The reported cases map to current-main code paths that still scan full message text and only compare mention/reply metadata to the current bot ID; I did not run a live Discord reproduction.

## Solution Assessment

Unclear for the full proposed patch shape. Tightening active attention and configured-agent targeting is the right direction, but role-label and plain-text delegation semantics should be reconciled with the open Discord routing and role-mention items before implementation.
