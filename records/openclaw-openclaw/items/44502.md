Keep this open for maintainer consolidation: define Discord target precedence in the plugin, keep role-trigger and slash-command targeting in their narrower issues, and add focused preflight/routing tests before any fix lands.

## Reproduction Assessment

## Solution Assessment

No, the large attention-window patch described in the report is only one plausible implementation. The best path needs a maintainer-approved Discord target-selection policy before default routing, with separate treatment for role-trigger and slash-command semantics.
