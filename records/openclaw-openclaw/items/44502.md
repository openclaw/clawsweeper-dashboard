## Summary

Keep open: current main and v2026.6.10 still derive Discord mention state from whole message text, compare explicit/reply targets only to the running bot id, and do not resolve other configured Discord agents before routing. The issue remains valid, but the final Discord active-target and role semantics need maintainer confirmation before an automated fix should change behavior.

## What This Changes

The issue requests stricter Discord preflight routing so only the actively addressed agent passes mention gating while quoted mentions, unrelated roles, other configured agents, and bot-authored loop paths are dropped.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection before default routing in the Discord plugin with focused regression coverage for other-agent, role, quoted mention, and bot-authored cases.

## Reproduction Assessment

Yes, at source level: current main passes full Discord message text into mention regex matching, compares explicit/reply signals only to the running bot id, and lacks other-configured-agent target resolution. I did not run a live Discord replay in this read-only review.

Unclear until maintainers confirm the product contract; the attention-window direction is plausible, but the durable fix needs approved precedence for cross-bot targets, roles, quoted text, home-channel defaults, and bot-authored messages.
