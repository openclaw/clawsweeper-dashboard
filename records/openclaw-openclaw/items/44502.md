## Label Justifications

- `impact:message-loss`: Discord messages can be dropped, consumed by a default owner, or routed away from the intended bot or agent.

## Summary

Keep open: current main and v2026.6.8 still resolve Discord routing before active target selection, compare explicit mention and reply metadata only to the current handler bot, and scan configured mention text without the reported opening attention window.

## What This Changes

The issue asks Discord preflight to tighten active-target mention gating so other-agent, role, quoted or later mention, and bot-authored cases do not route to the wrong agent.

## Best Possible Solution

Confirm the Discord active-target contract, then implement Discord-owned target selection before default routing with a configured bot/account/agent map, bounded attention-text matching, role handling, bot-loop suppression, and focused tests for other-agent, default-owner, quoted/later mention, unrelated-role, and bot-authored cases.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before active target selection, compares mention/reply metadata only to the current handler botId, and lacks the reported attention-window guard.

## Solution Assessment

Unclear until maintainers confirm the policy; the requested shape is plausible, but the durable fix needs an approved precedence contract for cross-bot targets, role mentions, quoted text, and bot-authored messages.
