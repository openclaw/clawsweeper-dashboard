## What This Changes

The issue requests stricter Discord preflight routing so messages targeting other agents, unrelated roles, quoted mentions, or other agent bots do not wake the current agent incorrectly.

## Best Possible Solution

Implement one coordinated Discord active-target policy that uses Discord mention/reference metadata plus configured agent bindings to suppress non-target agents before default routing, while resolving role triggers and slash-command targeting through the related policy items.

Yes.
