## Summary

Keep open: current main still uses full-message Discord mention matching and current-bot-only mention/reply state, so the reported active-target, quoted-mention, role, and cross-agent routing cases are not resolved by shipped code or a merged PR.

## What This Changes

The issue requests stricter Discord preflight routing so messages only wake the actively addressed agent and ignore competing agents, unrelated roles, quoted mentions, and bot-loop cases.

## Best Possible Solution

Implement one Discord-owned active-target policy that uses bounded opening attention text plus native mention/reply metadata and configured bot/account bindings, while reconciling role-trigger and slash-command behavior with the narrower open trackers.

## Reproduction Assessment

Yes, at source level. Current main still passes full Discord baseText into regex mention matching and only compares native mention/reply metadata to the running botUserId; I did not run a live Discord route in this read-only review.

## Solution Assessment

Unclear for the full local patch shape. Bounded attention and competing-agent suppression look directionally right, but plain-text delegation, role labels, bot-authored messages, and slash-command targeting need one maintainer-approved Discord routing policy.
