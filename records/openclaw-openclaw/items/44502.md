## Summary

Keep this issue open. Current main still uses the full Discord message body for mention-pattern matching, has no Discord attention-window helper, does not detect competing configured Discord agents before handling, and lacks regression coverage for the quoted/later mention, plain-text delegation, role-label, and configured-agent bot-loop cases described in the report.

## Best Possible Solution

Keep this issue open and implement a focused Discord plugin fix in extensions/discord/src/monitor/message-handler.preflight.ts. The fix should derive a short opening attention text for mention matching, detect other configured Discord-bound agents explicitly or plainly addressed as competing targets, handle role attention only when it safely maps to the current agent, drop competing-agent bot-loop cases, and add colocated regression tests for each reported scenario. Update docs/channels/discord.md only if the user-visible mention semantics change.

## Repair Work Prompt

Fix the Discord mention-gating/routing regression from openclaw/openclaw#44502 in the Discord plugin. In `extensions/discord/src/monitor/message-handler.preflight.ts`, scope mention-pattern matching to a short opening attention text instead of the full message body, so quoted/later examples do not satisfy `requireMention`. Detect configured Discord-bound agents that are explicitly mentioned or plainly delegated to as competing targets and drop for the current agent when the message is addressing another configured agent. Tighten role handling so unrelated role mentions do not count for the current agent, while safe role labels/metadata that map to the current agent remain valid. Drop messages from another configured agent bot in mention-only paths to avoid bot loops. Preserve existing behavior for explicit native bot mentions, reply-to-bot activation, bound-thread bypasses, `ignoreOtherMentions`, `allowBots="mentions"`, and control-command gating. Add focused tests in `extensions/discord/src/monitor/message-handler.preflight.test.ts` for: another configured agent explicitly mentioned in a home channel, plain-text delegation like "get Codey to help", plain agent names not counting as valid mentions in mention-gated channels, quoted/later mentions ignored, unrelated role mentions dropped, role labels that map to the current agent allowed, non-agent human mentions still handled under existing policy, and another configured agent bot dropped in mention-only channels. Review overlapping open PRs #47393 and #67460 before implementation to avoid conflicting semantics, but do not broaden this PR into their unrelated routing or multi-channel changes. Update `docs/channels/discord.md` only if the user-visible mention semantics need documenting. Do not move Discord-owned behavior into core unless a small generic SDK seam is clearly required by multiple plugins. Changelog is not required unless the final behavior change is user-visible enough for release notes.
