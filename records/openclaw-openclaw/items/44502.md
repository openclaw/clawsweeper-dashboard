## Summary

Keep this issue open. Current main still scans Discord mention text from the body-derived message text and has only broad user/role mention suppression, so it does not implement the requested attention-window, competing configured-agent, role-label, or configured-agent bot-loop handling. The related open PR #47393 is useful overlap for channel ownership and explicit alternate-bot mentions, but it is not a complete fix for this report.

## What This Changes

The issue requests stricter Discord preflight routing so only messages clearly addressing the current agent in the opening attention text pass mention gates, while other-agent targets, unrelated roles, quoted mentions, and other configured agent bots are dropped.

## Best Possible Solution
