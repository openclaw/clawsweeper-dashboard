## Summary

Keep open: current main still performs Discord mention matching over the full message text, only compares mention/reply state to the running bot ID, and lacks configured-agent targeting or attention-window filtering for the reported multi-agent cases.

## What This Changes

The issue requests stricter Discord mention-gating so only the actively addressed agent handles a message and quoted, unrelated role, other-agent, or other-bot signals are dropped.

## Best Possible Solution

Implement one Discord-owned routing fix that derives a short active attention window, maps configured Discord bot/user/role signals to OpenClaw agents, suppresses non-target agents before default routing, and proves the behavior with focused preflight tests.

## Reproduction Assessment

## Solution Assessment
