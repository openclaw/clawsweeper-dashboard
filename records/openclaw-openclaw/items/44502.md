## Summary

Keep open: current main and v2026.6.10 still choose the Discord route before any cross-bot active-target decision, compare mention/reply metadata only to the running bot id, and lack the reported opening-attention-window and other-configured-agent checks. The issue remains valid, but the permanent precedence and role semantics need maintainer product review before an automated repair lane should act.

## What This Changes

The issue requests stricter Discord mention-gating so messages targeting another configured agent, unrelated roles, quoted later mentions, or other agent bots are not handled by the wrong current agent.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection before default routing in the Discord plugin with focused regression coverage for other-agent, role, quoted mention, and bot-authored cases.

## Reproduction Assessment

Yes, at source level: current preflight compares mention/reply target signals only to the running bot id, selects the route before target resolution, and lacks bounded attention text. No live Discord replay was run in this read-only review.
