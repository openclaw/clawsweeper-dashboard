## Summary

Keep open: current main and v2026.6.10 still derive Discord mention decisions from the full message text and current bot id, while routing is resolved before any cross-agent active-target selection. The report remains source-reproducible, but the durable fix needs maintainer confirmation for multi-agent target precedence, role mentions, quoted text, and bot-authored messages.

## What This Changes

Reports that Discord preflight mention gating can treat the wrong agent, role, quoted text, or bot-authored message as active attention in multi-agent setups.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection inside the Discord plugin before default routing with focused regression coverage for other-agent, role, quoted-mention, and bot-authored cases.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before cross-bot target selection, matches configured mention regexes against the full message text, and compares explicit/reply metadata only to the current handler bot id. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract. The attention-window direction is plausible, but the durable fix needs approved precedence for cross-bot targets, role mentions, quoted text, home-channel defaults, and bot-authored messages.
