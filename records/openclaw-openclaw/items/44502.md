## Summary

Keep open: current main and the latest release still use full-message Discord mention matching with only the running bot identity, and the remaining routing/role/slash-command pieces are split across open partial work rather than merged behavior.

## What This Changes

The issue requests stricter Discord preflight so only the actively addressed current agent can pass mention gating in multi-agent Discord channels.

## Best Possible Solution

Implement one coordinated Discord-owned active-target policy using bounded opening attention text plus native Discord mention/reference metadata and configured bot/account bindings, while reconciling role-trigger and slash-command semantics with their canonical trackers.

Yes, at source level. Current main still passes full Discord baseText into regex mention matching and has no configured-other-agent competing-target check; I did not run a live Discord route in this read-only review.

Unclear for the full local patch shape. Bounded attention and competing-agent suppression look directionally right, but role labels, plain-text delegation, bot-authored messages, and slash-command targeting need one maintainer-approved policy boundary.
