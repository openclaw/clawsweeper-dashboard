## Summary

Keep open: current main still leaves the Discord active-target bug unresolved. The code continues to match configured mention patterns against the full Discord message text, compares native mention/reply state only to the current bot, and has no merged fix for the other-agent, quoted-mention, role, or cross-agent bot cases.

## What This Changes

The issue requests stricter Discord preflight attention scoping so quoted/later mentions, unrelated roles, other configured agents, and bot-authored cross-agent messages do not route to the wrong agent.

## Best Possible Solution

Define one Discord-owned active-target policy that combines bounded opening attention text, native Discord mention/reply metadata, configured bot/account bindings, role-trigger semantics, and multi-agent live proof.

## Reproduction Assessment

Yes, at source level. Current main still feeds full Discord message text into mention regex matching and only checks native mention/reply state against the current bot id; I did not run a live Discord route in this read-only review.

## Solution Assessment

Unclear for the reporter's full local patch shape. Bounded attention and competing-agent suppression are directionally right, but role labels, plain-text delegation, bot-authored messages, and slash-command targeting need one maintainer-approved Discord routing policy.
