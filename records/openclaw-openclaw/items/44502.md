## Summary

Keep open: current main and v2026.6.10 still resolve Discord routing before cross-bot target selection, match mention regexes against the full message text, and only compare explicit/reply mention signals to the running bot id. The report remains valid, but the final active-target, role, quoted-text, and bot-authored-message semantics need maintainer product confirmation before a fix should change multi-agent Discord behavior.

## What This Changes

The issue requests stricter Discord preflight mention gating so messages only wake the intended current agent and do not route based on other-agent, role, quoted, or bot-authored signals.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection in the Discord plugin before default routing with focused regression coverage for other-agent, role, quoted mention, and bot-authored cases.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before cross-bot mention or reply-target selection, uses the full message text for regex mention matching, and compares explicit/reply metadata only to the current handler botUserId. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract. The attention-window direction is plausible, but the durable fix needs approved precedence for cross-bot targets, role mentions, quoted text, home-channel defaults, and bot-authored messages.
