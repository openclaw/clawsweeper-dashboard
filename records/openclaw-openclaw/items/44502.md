## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause the wrong agent to answer or the intended agent not to answer.
- `impact:message-loss`: Messages can be dropped, consumed by the wrong default owner, or routed away from the intended Discord bot/agent.
- `impact:session-state`: Wrong-agent Discord routing can attach a user turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and the latest release still only partially guard Discord mention-gated routing. The implementation resolves the Discord route before cross-bot target selection, checks explicit and reply mentions only against the running bot account, and lacks a configured other-agent/role suppression result for the full behavior reported here.

## What This Changes

The issue requests stricter Discord mention gating and active-target routing so messages intended for another configured agent, unrelated role mentions, quoted mentions, and other agent bots are dropped before the current agent handles them.

## Best Possible Solution

Confirm the Discord active-target policy, then implement target selection in the Discord plugin before default routing with a configured bot/account/agent map, bounded attention-text matching, role handling, and suppression tests for default-owner and other-agent cases.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before cross-bot target selection and compares mention/reply metadata only to the running botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed; the requested direction is plausible, but the durable fix needs a chosen precedence contract and ownership boundary for cross-bot target maps, quoted mention handling, role mentions, and bot-loop behavior.
