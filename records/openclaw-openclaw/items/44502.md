## Label Justifications

- `impact:message-loss`: Discord messages can be dropped, consumed by a default owner, or routed away from the intended bot or agent.

## Summary

Keep open: current main and the latest release still use current-bot-only Discord mention and reply checks, route before cross-bot target selection, and scan the supplied message text without the reported opening-attention window. The remaining work is real, but tightening multi-bot Discord precedence and role behavior needs maintainer product judgment before automation.

## What This Changes

The issue requests stricter Discord preflight routing so messages target only the active intended agent and ignore other-agent, unrelated-role, quoted/later, and bot-authored loop signals.

## Best Possible Solution

Confirm the Discord active-target policy, then implement target selection in the Discord plugin before default routing with a configured bot/account/agent map, bounded attention-text matching, role handling, and suppression tests for default-owner and other-agent cases.

## Reproduction Assessment

Yes, at source level: current main resolves the Discord route before cross-bot target selection and compares mention/reply metadata only to the running botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed; the requested direction is plausible, but the durable fix needs a chosen precedence contract and ownership boundary for cross-bot target maps, quoted mention handling, role mentions, and bot-loop behavior.
