## Summary

Keep open. Current main still leaves the reported diagnostics gap: signal, timeout, no-output-timeout, and failure-kind metadata exists in the supervisor/runtime path, but foreground exec results and process.poll details do not expose enough structured termination context for the macOS SIGKILL cases.

## What This Changes

The issue requests richer user-facing diagnostics for intermittent macOS exec SIGKILL failures, including signal/cause metadata, output tails, timeout context, and log/run correlation.

## Best Possible Solution

Add an additive exec/process diagnostics contract that carries existing supervisor/runtime metadata into foreground exec and process.poll results, while keeping kill policy and broad recursive-scan prevention as separate follow-ups.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is source-reproducible by tracing simulated RunExit signal/timeout outcomes through buildExecForegroundResult and process.poll details on current main.

## Solution Assessment

Yes for the first repair step: additive metadata propagation is the narrowest maintainable fix because current main already captures the relevant signal and timeout state internally. Changing supervisor SIGKILL policy or adding broad-scan prevention would be broader follow-up work.
