## Summary

Keep open. Current main still captures signal/timeout context internally, but foreground exec details and process.poll structured details do not expose enough of that metadata to explain SIGKILL, timeout, or no-output-timeout outcomes; the related open PR only addresses a node-host signal path and does not cover this broader diagnostics gap.

## What This Changes

The issue asks OpenClaw to surface actionable SIGKILL and timeout metadata for exec/process failures, especially for macOS broad state/log scans.

## Best Possible Solution

Expose the existing RunExit, ExecProcessOutcome, and FinishedSession termination metadata in backward-compatible exec and process result details, with focused regression coverage and separate tracking for signal escalation or broad-scan guardrails.

## Reproduction Assessment

No for the intermittent macOS SIGKILL trigger; the issue does not provide a deterministic current-main reproduction. Yes at source level for the diagnostics gap: current main carries exitSignal, failureKind, reason, timedOut, and noOutputTimedOut internally, then drops key fields from user-facing structured details.

## Solution Assessment

Yes for the narrow first repair. Additive metadata propagation is the maintainable fix for this issue; signal escalation, OS-cause telemetry, and broad-scan guardrails should stay as separate focused work.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72240 by propagating existing exec/process termination metadata into user-facing structured results. On current main, RunExit already carries reason, exitSignal, timedOut, and noOutputTimedOut; ExecProcessOutcome carries exitSignal, failureKind, reason, and timedOut; FinishedSession carries exitSignal and exitReason. Add backward-compatible optional fields to ExecToolDetails and foreground exec result details, and add exitSignal/exitReason or equivalent cause fields to process.poll details for both finished-session and just-exited paths. Preserve existing text output and result status semantics. Add focused tests for foreground timeout/signal/no-output-timeout details and process.poll finished/just-exited signal details. Do not implement SIGTERM escalation, OS OOM detection, memory/RSS tracking, or broad recursive-scan guardrails in this PR; those belong to separate related trackers such as https://github.com/openclaw/openclaw/issues/66399 and https://github.com/openclaw/openclaw/issues/69242. Check https://github.com/openclaw/openclaw/pull/68680 for the node-host signal sub-path, but do not treat a node-host-only change as sufficient for this issue. Add a single-line Fixes changelog entry if repo policy requires it for this user-visible diagnostics change.
