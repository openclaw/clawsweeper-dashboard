## Summary

Keep open. Current main still captures termination metadata internally but loses key fields at the foreground exec and process.poll result boundaries, so the user-facing diagnostics gap remains source-reproducible even though the intermittent macOS SIGKILL itself is not deterministic.

## What This Changes

Request clearer user-facing diagnostics when macOS exec/process commands are killed with SIGKILL, especially during broad scans over large OpenClaw state directories.

## Best Possible Solution

Propagate existing termination metadata into backward-compatible exec/process result details first, with signal escalation and broad-scan guardrails handled as separate focused follow-ups.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report does not provide a deterministic current-main reproduction. Yes source-level for the diagnostics gap: current code captures exitSignal, failureKind, timeout flags, and supervisor reason internally, then omits key fields from user-facing structured results.

## Solution Assessment

Yes. Additive metadata propagation is the narrowest maintainable first fix because current main already has most of the requested termination context; changing signal policy or adding broad-scan guardrails should stay separate.
