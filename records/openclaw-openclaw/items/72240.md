## Summary

Keep open. Current main still leaves the reported diagnostics gap: exec and supervisor internals capture signal, timeout, and failure-kind data, but foreground exec results and process polling details do not expose enough structured termination metadata for macOS SIGKILL RCA. The issue is recent, contributor-authored, has related open reports, and no merged/current-main fix fully implements the requested diagnostic contract.

## What This Changes

The issue requests structured diagnostics for intermittent macOS exec SIGKILL failures, including kill cause, timeout/cancellation metadata, output tails, and run/log correlation.

## Best Possible Solution

Add an additive exec/process diagnostics contract that carries existing supervisor/runtime metadata into foreground exec results, process.poll details, and diagnostic/log correlation: exit signal, failure kind, termination reason, timeout/no-output-timeout state, timeout budget when known, bounded output tail context, and a useful run/session/log id. Keep signal escalation policy and broad-scan guardrails as separate follow-ups unless maintainers explicitly fold them in.

## Reproduction Assessment

Unclear: the report does not provide a deterministic macOS reproduction for the intermittent SIGKILL. The diagnostic gap itself is reproducible from source by tracing simulated signal, timeout, and cancellation outcomes through buildExecForegroundResult and process.poll details on current main.

## Solution Assessment

Yes for the next repair step: additive metadata propagation is the narrowest maintainable solution for the user-visible diagnostic gap and can be covered with focused unit tests. Changing supervisor kill policy or warning on recursive scans would be broader follow-up work, not the first fix for this issue.
