## Summary

Keep open. Current main still drops actionable SIGKILL diagnostics at the foreground exec and process.poll result boundary, even though the runtime and supervisor already carry exit signal, timeout, no-output-timeout, and failure-kind context.

## What This Changes

The issue asks OpenClaw exec/process failures to surface enough SIGKILL metadata to distinguish timeout, cancellation, supervisor kill, OS kill, and captured output context.

## Best Possible Solution

Add backward-compatible termination diagnostic fields to foreground exec and process.poll results by propagating existing supervisor/runtime metadata, while keeping signal policy and broad-scan guardrails in separate focused work.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report still lacks a deterministic current-main reproduction. Yes for the diagnostics gap: current source shows termination metadata is captured internally and then omitted from foreground exec/process.poll structured results.

## Solution Assessment

Yes for the narrow first repair: additive metadata propagation is the most maintainable way to make existing failure context visible without changing kill policy. SIGTERM/SIGKILL behavior and broad-scan guardrails should remain separate follow-up issues.
