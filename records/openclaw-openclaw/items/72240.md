## Summary

Keep open. Current main still loses structured SIGKILL/timeout diagnostics at the user-facing exec and process result boundary even though the supervisor/runtime already captures the relevant termination context.

## What This Changes

Improve exec/process SIGKILL diagnostics by surfacing termination cause metadata, captured output context, and correlation data for intermittent macOS hard kills.

## Best Possible Solution

Add backward-compatible exec/process termination diagnostic fields that propagate existing supervisor/runtime metadata to foreground exec and process.poll results, while leaving kill policy and broad-scan guardrails to separate focused issues.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report still lacks a deterministic current-main reproduction. Yes for the diagnostics gap: current source shows exitSignal, failureKind, timedOut, noOutputTimedOut, and termination reason are captured internally and then omitted from foreground exec/process.poll structured results.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already has most of the requested termination context. Changing SIGTERM/SIGKILL policy or blocking broad scans should remain separate work tied to the related open reports.
