## Summary

Keep open. Current main still has the diagnostics gap: exec/supervisor internals carry signal, timeout, and failure-kind metadata, but foreground exec results and the main process polling details do not expose enough structured termination data for the reported macOS SIGKILL cases.

## What This Changes

Request richer user-facing diagnostics for intermittent macOS exec SIGKILL failures, including structured kill metadata, output tail preservation, log/run correlation, and possible guardrails for broad scans over large OpenClaw state directories.

## Best Possible Solution

Add an additive exec/process diagnostics contract that carries existing supervisor/runtime metadata into foreground exec and process polling results: exit signal, failure kind, termination reason, timeout/no-output-timeout state, timeout budget when known, bounded output tail context, and useful session/run/log correlation. Keep signal-escalation policy and broad-scan prevention as separate follow-ups unless maintainers explicitly fold them in.

## Reproduction Assessment

Unclear: there is no deterministic macOS reproduction for the intermittent SIGKILL pattern. The diagnostics gap is reproducible from source by tracing simulated signal, timeout, and cancellation outcomes through `buildExecForegroundResult` and `process.poll` details on current main.

## Solution Assessment

Yes for the first repair step: additive metadata propagation is the narrowest maintainable fix for the user-visible diagnostic gap and can be covered with focused unit tests. Changing supervisor kill policy or warning on broad scans would be a separate, broader solution path.
