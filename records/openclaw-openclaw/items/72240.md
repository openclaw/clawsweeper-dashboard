## Summary

Keep open. Current main still has a source-backed diagnostics gap: exec/runtime and supervisor code record SIGKILL-related metadata internally, but foreground exec and process.poll result details still omit key fields needed for user-facing root cause analysis.

## What This Changes

The issue asks OpenClaw to expose actionable cause metadata when macOS exec commands are killed with SIGKILL and to consider safer handling for broad scans over large state/log directories.

## Best Possible Solution

Add backward-compatible termination metadata to exec and process.poll result details with regression coverage, while leaving signal escalation and broad-scan guardrails to their focused related issues.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report does not provide a deterministic current-main reproduction. Yes source-level for the diagnostics gap: current main records exitSignal, failureKind, timeout flags, and termination reason internally, then omits key fields from foreground exec and process.poll details.

## Solution Assessment

Yes for the narrow repair. Additive metadata propagation is the smallest maintainable fix for the confirmed gap; changing signal policy, adding RSS tracking, or guarding broad scans should stay separate unless maintainers expand scope.
