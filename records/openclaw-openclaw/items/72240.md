## Summary

Keep open. Current main still records termination cause internally but does not consistently expose it through foreground exec or process.poll structured result details, so the user-facing SIGKILL diagnostics gap remains source-reproducible.

## What This Changes

The issue requests richer user-facing diagnostics when macOS exec/process commands are SIGKILLed, including cause metadata, stdout/stderr tail, log pointers, and possible broad-scan guardrails.

## Best Possible Solution

Propagate existing termination metadata into backward-compatible exec and process result details first, leaving SIGTERM escalation and broad-scan guardrails to their canonical follow-ups.

## Reproduction Assessment

No for the intermittent macOS SIGKILL trigger itself: the report does not provide a deterministic current-main reproduction. Yes at source level for the diagnostics gap: current code carries exitSignal, failureKind, timeout, and supervisor reason internally, then drops key fields from user-facing structured details.

## Solution Assessment

Yes for a narrow first repair. Additive metadata propagation is the maintainable fix for this issue; changing signal policy or adding broad-scan guardrails should be handled separately.
