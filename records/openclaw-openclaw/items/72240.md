## Summary

Keep open. Current main still captures termination metadata internally but drops key fields at the foreground exec and process.poll result boundaries, so the diagnostic gap is source-reproducible even though the intermittent macOS SIGKILL itself is not deterministic.

## What This Changes

The issue asks OpenClaw to surface actionable SIGKILL diagnostics for exec/process failures, especially on macOS sessions with large local state and active gateway load.

## Best Possible Solution

Add backward-compatible termination diagnostic fields to foreground exec and process.poll results while leaving signal escalation and broad-scan guardrails to separate focused work.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report still lacks a deterministic current-main reproduction. Yes source-level for the diagnostics gap: current main stores exitSignal, failureKind, timeout, no-output-timeout, and termination reason data internally, then omits key fields from user-facing structured results.

## Solution Assessment

Yes for the narrow repair: propagating existing supervisor/runtime metadata is the smallest maintainable first fix. Changing SIGTERM/SIGKILL policy or adding recursive-scan guardrails should remain separate because those require different behavior decisions and validation.
