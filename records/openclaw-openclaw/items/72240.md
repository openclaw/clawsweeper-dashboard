## Summary

Keep open. Current main and v2026.6.6 already distinguish supervisor timeouts/cancels from bare signal exits and preserve output tail, but the remaining user-visible SIGKILL branch still renders as a sparse signal message without duration or an external/OS-kill hint, and peak memory is not captured.

## What This Changes

The issue asks OpenClaw to make intermittent macOS exec SIGKILL failures diagnostically actionable instead of returning a bare signal message.

## Best Possible Solution

Land a bounded diagnostics fix that keeps supervisor ownership, adds duration and an external/OS-kill hint for bare signal exits, and leaves peak-memory tracking as separate follow-up work unless measured safely.

## Reproduction Assessment

Yes at source level for the missing-diagnostics part: current main still formats bare signal exits without duration or OS-kill guidance. I did not establish a deterministic live macOS SIGKILL reproduction because the original trigger is intermittent and this review is read-only.

## Solution Assessment

Yes, a narrow propagation/rendering fix is the best path. The supervisor already owns termination facts, so the repair should expose and explain those facts rather than changing kill behavior, adding config, or trying to infer peak memory without measurement.
