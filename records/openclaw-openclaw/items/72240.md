## Summary

Keep open. Current main and the latest release have partial SIGKILL handling, but foreground exec and process polling still flatten the termination data into sparse user-visible details, so the broader diagnostics gap from this issue remains actionable.

## What This Changes

The issue asks OpenClaw to expose useful diagnostics when macOS exec commands are killed with SIGKILL, including cause, timing, stdout/stderr tail, supervisor/cancel context, memory hints, and log/run pointers.

## Best Possible Solution

Surface the existing supervisor/runtime termination facts through foreground exec and process.poll details, with focused coverage for signal, overall-timeout, no-output-timeout, and manual-cancel paths, without adding a new config surface.

## Reproduction Assessment

Yes for the missing-diagnostics part by source inspection: current main can carry signal/timeout facts internally while the user-visible result details omit key fields. I did not establish a live deterministic macOS SIGKILL repro because the original trigger is intermittent and this review is read-only.

## Solution Assessment

Yes, a bounded propagation fix is the best path. The supervisor should remain the owner of termination facts, while exec/process result details expose those facts for RCA instead of adding a separate policy or config layer.
