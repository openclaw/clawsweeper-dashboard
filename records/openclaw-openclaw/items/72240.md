## Summary

Keep open. Current main still drops actionable SIGKILL diagnostics at the foreground exec and process.poll structured-result boundary, even though the runtime and supervisor already carry signal, timeout, no-output-timeout, and failure-kind context.

## What This Changes

The issue asks OpenClaw exec/process failures to surface enough SIGKILL metadata to distinguish timeout, cancellation, supervisor kill, OS kill, and captured output context.

## Best Possible Solution

Add backward-compatible termination diagnostic fields to foreground exec and process.poll results by propagating existing supervisor/runtime metadata, while leaving signal policy and broad-scan guardrails to focused follow-up work.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report lacks a deterministic current-main reproduction. Yes for the diagnostics gap: current source shows termination metadata is captured internally and then omitted from foreground exec/process.poll structured results.

## Solution Assessment

Yes for the narrow first repair: additive metadata propagation is the most maintainable way to make existing failure context visible without changing kill policy. SIGTERM/SIGKILL behavior and broad-scan guardrails should remain separate follow-up issues.
