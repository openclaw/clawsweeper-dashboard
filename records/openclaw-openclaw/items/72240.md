## Summary

Keep open. Current main still drops structured termination metadata between the supervisor/runtime layers and user-visible exec/process results, so the reported SIGKILL diagnostics gap remains valid.

## What This Changes

The issue asks OpenClaw to surface enough structured diagnostics to explain intermittent macOS exec SIGKILL failures, especially during broad scans over large state and log directories.

## Best Possible Solution

Add backward-compatible exec/process diagnostic fields that propagate existing supervisor/runtime termination metadata into foreground exec and process.poll results, leaving kill-policy changes and broad-scan guardrails to separate focused follow-ups.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is source-reproducible by tracing a simulated signal or timeout outcome through RunExit and ExecProcessOutcome into foreground exec and process.poll details.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already captures most of the requested termination context internally. Changing supervisor signal policy or blocking broad scans would be broader work tied to related reports such as #66399 and #69242.
