## Summary

Keep open. Current main still leaves the reported user-facing diagnostics gap: exec/supervisor internals capture signal, timeout, and failure-kind data, but foreground exec results and `process.poll` details do not consistently surface enough structured termination context to diagnose the macOS SIGKILL cases. Related open issues cover adjacent kill-policy and broad-search symptoms, but this issue remains the narrow diagnostics contract work.

## What This Changes

Request structured diagnostics for intermittent macOS exec SIGKILLs, including termination cause metadata, captured output tail, run/log correlation, and safer handling for broad OpenClaw state scans.

## Best Possible Solution

Add backward-compatible exec/process diagnostic fields that propagate existing supervisor/runtime metadata into foreground exec and `process.poll` results; leave signal escalation policy and broad recursive-scan guardrails as separate follow-ups.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is reproducible from current source by tracing a simulated signal or timeout outcome through `RunExit` and `ExecProcessOutcome` into the foreground exec and `process.poll` detail builders.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already captures most of the requested termination context internally. Changing kill policy or preventing broad scans would be broader work tied to the related open reports.
