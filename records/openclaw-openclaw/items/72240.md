## Summary

Keep open. Current main still has the reported diagnostics gap: exec and supervisor internals capture signal, timeout, reason, and failure-kind data, but foreground exec results and `process.poll` structured details still do not surface enough of that metadata for the macOS SIGKILL cases.

## What This Changes

The issue asks OpenClaw to explain intermittent macOS exec SIGKILL failures with structured termination metadata, output tails, and run/log correlation.

## Best Possible Solution

Add backward-compatible termination metadata to exec/process result details while leaving signal escalation policy and broad recursive-scan guardrails to their existing follow-up issues.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is source-reproducible by tracing a simulated signal or timeout outcome through `RunExit` and `ExecProcessOutcome` into foreground exec and `process.poll` details.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already captures most of the requested termination context internally. Changing kill policy or preventing broad scans would be broader work tied to related open reports.
