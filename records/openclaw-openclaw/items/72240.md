## Summary

Keep open. Current main still has the reported diagnostics gap: exec and supervisor internals know the signal, timeout, and failure kind, but foreground exec results and process poll details do not expose enough structured termination context to diagnose the macOS SIGKILL cases.

## What This Changes

The issue requests structured diagnostics for intermittent macOS exec SIGKILL failures, including termination cause, timeout context, output tail, and log/run correlation.

## Best Possible Solution

Add additive exec/process diagnostics fields that propagate existing supervisor/runtime metadata into foreground exec and process.poll results, while leaving kill policy and broad-scan guardrails as separate follow-ups.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is reproducible from current source by tracing a simulated signal or timeout outcome through buildExecForegroundResult and process.poll details.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already captures most of the needed termination data internally. Changing supervisor kill policy or adding recursive-scan prevention would be broader follow-up work.
