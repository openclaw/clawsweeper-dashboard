## Summary

Keep open. Current main and v2026.6.6 still leave the central SIGKILL diagnostics gap in user-facing exec/process results, and the related diagnostics PRs are open rather than merged, so this is not implemented or safely superseded yet.

## What This Changes

The issue requests clearer diagnostics when macOS exec commands are killed with SIGKILL, especially during broad scans over large OpenClaw state or log directories.

## Best Possible Solution

Land or adapt the structured metadata path from https://github.com/openclaw/openclaw/pull/89104, then add the macOS/external-SIGKILL visible hint and duration wording needed by this issue without changing kill behavior or adding config.

## Reproduction Assessment

Yes for the missing-diagnostics behavior by source inspection: current main can produce a signal-failure result whose visible reason is only `Command aborted by signal SIGKILL` plus any captured output, while timeout/cancel causes are distinguished internally. I did not reproduce the intermittent macOS OS-level SIGKILL trigger live.

## Solution Assessment

Yes for the direction, but not as a duplicate new PR while related diagnostics branches are open. The maintainable path is to reuse the supervisor-owned termination facts and avoid broad resource-policy changes or new config.
