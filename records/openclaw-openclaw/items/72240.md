## Summary

Keep open. Current main still has the reported diagnostics gap: supervisor/runtime code captures signal, timeout, and failure classification data, but foreground exec results and `process.poll` details still drop key structured termination metadata needed to diagnose the macOS SIGKILL cases.

## What This Changes

Request structured diagnostics for intermittent macOS exec SIGKILLs, including termination cause metadata, captured output tail, run/log correlation, and safer handling for broad OpenClaw state scans.

## Best Possible Solution

Add backward-compatible exec/process diagnostic fields that propagate existing supervisor/runtime metadata into foreground exec and `process.poll` results; keep signal escalation policy and broad recursive-scan guardrails as separate follow-ups.

## Reproduction Assessment

Unclear: there is no high-confidence deterministic macOS reproduction for the intermittent SIGKILL itself. The diagnostics gap is source-reproducible by tracing a simulated signal or timeout outcome through `RunExit` and `ExecProcessOutcome` into foreground exec and `process.poll` details.

## Solution Assessment

Yes: additive metadata propagation is the narrowest maintainable first fix because current main already captures most of the requested termination context internally. Changing kill policy or preventing broad scans would be broader work tied to related open reports.
