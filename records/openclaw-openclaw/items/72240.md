## Summary

Keep open. Current main and v2026.6.6 still surface bare signal failures as `Command aborted by signal SIGKILL`; the supervisor already distinguishes timeout/cancel/signal and carries duration, but foreground exec and process polling still omit useful termination metadata. Open PR #89104 covers much of the structured metadata path, but it is unmerged and references the related Linux issue rather than closing this macOS report.

## What This Changes

The issue asks OpenClaw to surface actionable diagnostics when macOS exec commands are killed with SIGKILL.

## Best Possible Solution

Land or adapt the structured termination-metadata work from PR #89104, then add the small signal-branch duration and external OS-kill hint for bare macOS SIGKILL; keep peak-memory capture as a separate follow-up.

## Reproduction Assessment

Yes at source level: current main carries termination facts internally but still drops or under-formats them in user-visible exec/process results; no deterministic live macOS SIGKILL repro was established.

## Solution Assessment

Yes: the best fix is bounded propagation and formatting of existing supervisor facts, not a new config surface or an attempt to solve all OS memory-pressure kills in this issue.
