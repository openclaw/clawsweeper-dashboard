## Summary

Keep open. Current main still records signal, timeout, and supervisor reason internally, but foreground exec and process.poll result details omit enough of that metadata that the user-facing SIGKILL diagnostic gap remains source-reproducible.

## What This Changes

Request clearer diagnostics when macOS exec commands are killed with SIGKILL, including cause metadata, output tail, run/log pointers, and safer handling for broad scans.

## Best Possible Solution

Propagate the existing termination metadata into backward-compatible exec and process result details first, then track signal escalation or broad-scan guardrails as separate focused work.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report does not provide a deterministic current-main reproduction. Yes source-level for the diagnostics gap: current code records exitSignal, failureKind, timeout flags, and supervisor reason, then omits key fields from foreground exec and process.poll details.

## Solution Assessment

Yes for a narrow first fix. Additive metadata propagation is the safest maintainable step because the supervisor/runtime already collect the relevant fields; signal policy and broad-scan protection should remain separate follow-ups.
