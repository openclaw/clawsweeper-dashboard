## Summary

Keep open. Current main still records termination cause internally but does not consistently expose signal/failure metadata through foreground exec or process.poll structured results, so the user-facing SIGKILL diagnostics gap remains source-reproducible.

## What This Changes

The issue asks OpenClaw to surface actionable kill metadata for intermittent macOS exec SIGKILL failures and optionally guard broad scans over large OpenClaw state/log directories.

## Best Possible Solution

Propagate existing supervisor/runtime termination metadata into backward-compatible exec and process result details, then cover signal, timeout, and no-output-timeout paths with focused regression tests.
