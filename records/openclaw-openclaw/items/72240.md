## Summary

Keep open. Current main still drops captured SIGKILL and timeout metadata before the user-facing foreground exec and process.poll structured results, so the diagnostic gap reported here remains valid even though the intermittent macOS kill itself is not deterministic.

## What This Changes

Request clearer user-facing diagnostics when macOS exec commands are killed with SIGKILL, including cause metadata, output tails, run/log pointers, and safer handling for broad scans.

## Best Possible Solution

Add backward-compatible termination diagnostic fields to exec and process.poll results using the existing RunExit, ExecProcessOutcome, and FinishedSession metadata; keep signal policy and scan guardrails as separate focused work.

## Reproduction Assessment

No for the intermittent macOS SIGKILL itself: the report still lacks a deterministic current-main reproduction. Yes for the diagnostics gap: source inspection shows exitSignal, failureKind, timedOut, noOutputTimedOut, and termination reason are captured internally and then omitted from the user-facing structured results.

## Solution Assessment

Yes for a first repair: additive metadata propagation is the narrowest maintainable fix because current main already captures most of the requested context. Changing SIGTERM/SIGKILL policy or blocking broad scans should be handled separately.
