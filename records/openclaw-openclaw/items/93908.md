## Best Possible Solution

Confirm the Windows npm-global runtime path first; if the gateway is genuinely launched from 2026.6.8 while reporting 2026.5.28, fix the update/service/plugin discovery path that leaks the stale compatibility host version and add focused regression coverage.

## Reproduction Assessment

## Solution Assessment

Unclear. Existing split-brain troubleshooting may be the right user fix if an old gateway binary is running, but if current 2026.6.8 can leak a stale compatibility host during update or startup, a narrow code fix belongs in update/service/plugin discovery rather than closing as support.
