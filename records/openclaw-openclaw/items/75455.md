## Summary

Keep open. Current main still has the reported DM `chatIdentifier` gap, but this PR is not merge-ready because it routes every non-empty DM identifier through the first-match `chat_identifier` path and can bypass the existing iMessage-over-SMS handle ranking.

## What This Changes

The PR changes BlueBubbles inbound DM chat-guid fallback to prefer a non-empty inbound `chatIdentifier` over `senderId` and updates the focused resolver test.

## Best Possible Solution

Revise the PR so full or opaque DM identifiers use identifier resolution, handle-like identifiers preserve the service-aware handle resolver, group behavior stays unchanged, and tests plus changelog cover both paths.

## Reproduction Assessment

Yes. Current main reproduces the original gap through `buildBlueBubblesInboundChatResolveTarget`, and the PR regression is reproducible by using a bare phone/email `chatIdentifier` when chat query results list SMS before iMessage.

## Solution Assessment

No, not as currently proposed. Preferring precise inbound identifiers is the right direction, but the maintainable fix must distinguish full/opaque identifiers from handle-like values so existing handle ranking remains in force.
