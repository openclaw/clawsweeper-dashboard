## Summary

This PR should stay open: it targets a real BlueBubbles DM routing gap, but the current diff introduces a service-ranking regression for handle-like chat identifiers and still needs the required changelog entry.

## What This Changes

The PR changes BlueBubbles inbound DM fallback resolution to prefer a trimmed chatIdentifier over senderId and updates the focused resolver test.

## Best Possible Solution

Update the PR so full or opaque DM identifiers use the identifier path while bare phone/email identifiers keep the handle-ranking behavior, then add regression tests and an Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main ignores DM chatIdentifier fallback, while this PR would send bare handle-like chat identifiers through the chat_identifier path that returns the first SMS/iMessage match before handle ranking.

## Solution Assessment

No, not as currently proposed. Preferring precise inbound identifiers is the right direction, but the patch needs to classify handle-like identifiers so it does not bypass the documented iMessage-over-SMS routing contract.
