## Summary

Keep open. Current main still has the reported DM `chatIdentifier` gap, but this PR's current implementation is not merge-ready because it sends every DM `chatIdentifier` down the first-match `chat_identifier` path and can bypass the existing iMessage-over-SMS handle ranking.

## What This Changes

The PR changes BlueBubbles inbound DM chat-guid fallback to prefer a non-empty inbound `chatIdentifier` over `senderId` and updates the focused resolver test.

## Best Possible Solution

Update the PR so full or opaque DM chat identifiers use the identifier path while handle-like identifiers preserve the documented iMessage-over-SMS handle ranking, then add regression tests and a changelog entry.

## Reproduction Assessment

Yes. Current main reproduces the original gap through the focused resolver helper/test, and the PR regression can be reproduced with a bare phone/email `chatIdentifier` when chat query results list SMS before iMessage.

## Solution Assessment

No, not as currently proposed. Preferring the inbound identifier is the right direction for precise identifiers, but blindly returning `chat_identifier` is not the narrowest maintainable fix because it skips the established handle service-ranking contract.
