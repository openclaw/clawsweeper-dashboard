## Summary

Keep open: the PR targets a real BlueBubbles DM routing gap, but the current patch is not merge-ready because handle-like identifiers can bypass the established iMessage-over-SMS ranking and the user-facing fix lacks a changelog entry.

## What This Changes

The PR makes BlueBubbles direct-message inbound chat resolution prefer a non-empty `chatIdentifier` over `senderId` and updates the focused resolver tests.

## Best Possible Solution

Classify or resolve DM identifiers so precise/full/opaque chat identifiers use the identifier path while bare phone/email handles preserve service-aware handle ranking, then cover both cases with tests and changelog.

## Reproduction Assessment

Yes. The original gap is visible in the focused `buildBlueBubblesInboundChatResolveTarget` helper behavior on current main, and the PR regression can be reproduced with a bare phone/email identifier when SMS appears before iMessage in chat query results.

## Solution Assessment

No. Preferring precise inbound identifiers is the right direction, but returning `chat_identifier` for every non-empty DM identifier is not the narrowest maintainable fix because it bypasses the existing handle ranking contract.
