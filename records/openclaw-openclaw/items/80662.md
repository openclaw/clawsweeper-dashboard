## Summary

Keep open and do not merge as-is: the visible Agents directory idea has some screenshot proof, but the branch mixes it with unrelated core/storage changes and introduces blocking security, durability, and type issues.

## What This Changes

This PR replaces the Agents tab dropdown with a searchable card directory and deep-linkable detail view, while also changing URL sync, i18n, daemon/WebSocket logging, durable queue storage, root JSON helpers, tests, and adding a local build log.

## Best Possible Solution

Trim or split the branch to the Agents directory UI, preserve the fs-safe-backed root-file and queue helpers, remove generated artifacts, fix the type/test issues, and then get UI/core owner review.

## Reproduction Assessment

Yes for the PR findings: the raw PR head and current source show the swapped helpers, generated artifact, and test type mismatch. I did not run runtime checks because this review was read-only.

## Solution Assessment

No. The UI directory work should not be merged together with unrelated core storage/root-file rewrites, and the safer path is to keep the fs-safe dependency contracts while narrowing the PR to the user-facing Agents change.

- **[P1] Keep root JSON reads on fs-safe:** `src/infra/json-files.ts:47`
  - body: This replaces the fs-safe root readers with a direct `fs.readFileSync` after only a normalized prefix check, while callers pass `rootRealPath` and `rejectHardlinks` for plugin roots. That drops the symlink/hardlink and bounded-read protections that keep plugin manifests/config reads inside the intended root.
  - confidence: 0.93
- **[P1] Preserve durable queue file protections:** `src/infra/outbound/delivery-queue-storage.ts:102-103`
  - body: The inlined queue helpers use direct `writeFile`, `readFile`, and `rename`, dropping the upstream queue id validation, 0700/0600 modes, atomic replace, no-follow/same-file checks, and max-byte reads. Delivery queue entries contain outbound payload and recovery state, so this is a security and reliability regression.
  - confidence: 0.92

  - body: `AgentsDirectoryState` only declares `query`, `sortDir`, and `defaultFilter`, but this test fixture adds `runtimeFilter` and `workspaceFilter`. That excess property should fail the test type check unless the state type and implementation are actually extended.
  - confidence: 0.88
