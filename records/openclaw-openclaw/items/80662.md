## Summary

Keep open, but this patch should not merge as-is: the current PR head still contains the previously reported fs-safe regressions, a type/test mismatch, a committed local build log, insufficient real behavior proof, and a conflicting branch state.

## What This Changes

The PR replaces the Agents tab dropdown with a searchable card directory and also changes URL sync, i18n outputs, daemon/gateway logging, root JSON helpers, durable queue storage, and tests.

## Best Possible Solution

Split or narrow the branch to the Agents directory UI, keep fs-safe-backed root-file and queue helpers, remove generated/build-log churn that is not needed, fix the type mismatch, and require after-fix runtime proof plus UI/core owner review.

## Reproduction Assessment

Yes for the review findings: the PR head and current-main source show the helper rewrites, excess test fields, and committed build log. I did not run runtime checks because this review was read-only.

## Solution Assessment

No. A card directory may be a viable product direction, but this branch is not the best path because it combines that UI work with unrelated core storage rewrites that weaken dependency-backed safety contracts.
