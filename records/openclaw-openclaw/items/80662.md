## Summary

Keep open: the UI direction may be useful, but this branch is not mergeable because it mixes the Agents directory feature with security-sensitive fs-safe regressions, a type/test mismatch, a committed build log, and insufficient real behavior proof.

## What This Changes

This PR replaces the Agents tab dropdown with a searchable card directory and deep-linkable detail flow, while also changing URL sync, i18n output, daemon/WebSocket logging, durable queue storage, root JSON helpers, tests, and adding a local build log.

## Best Possible Solution

Narrow or split the branch to the Agents directory UI, keep the fs-safe-backed root-file and queue helpers, remove generated/local artifacts, fix the type mismatch, add real behavior proof, and route UI/core boundary decisions to owners.

## Reproduction Assessment

Yes for the review findings: the PR head source and the @openclaw/fs-safe contract show the helper replacements, committed artifact, and type mismatch. I did not run tests because this review was read-only.

## Solution Assessment

No. The directory UI may be a reasonable product direction, but the current patch is not the best implementation because it combines that UI change with unrelated and security-sensitive core storage/root-file rewrites.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep root JSON reads on fs-safe:** `src/infra/json-files.ts:47`
  - body: This replaces the fs-safe root readers with a direct `fs.readFileSync` after only a normalized prefix check, while callers pass `rootRealPath` and `rejectHardlinks` for plugin roots. That drops the symlink/hardlink and bounded-read protections that keep plugin manifests/config reads inside the intended root.
  - confidence: 0.93
- **[P1] Preserve durable queue file protections:** `src/infra/outbound/delivery-queue-storage.ts:102-103`
  - body: The inlined queue helpers use direct `writeFile`, `readFile`, and `rename`, dropping the upstream queue id validation, 0700/0600 modes, atomic replace, no-follow/same-file checks, and max-byte reads. Delivery queue entries contain outbound payload and recovery state, so this is a security and reliability regression.
  - confidence: 0.92
- **[P2] Remove unknown directory filter fields:** `ui/src/ui/views/agents.test.ts:54-55`
  - body: `AgentsDirectoryState` only declares `query`, `sortDir`, and `defaultFilter`, but this test fixture adds `runtimeFilter` and `workspaceFilter`. That excess property should fail the test type check unless the state type and implementation are actually extended.
  - confidence: 0.88
- **[P3] Drop the committed build log:** `build_output.txt:1`
  - body: The branch adds `build_output.txt`, a failed local build transcript with machine-local paths and ANSI output. It is generated local output, not source or test data, and should be removed before merge.
  - confidence: 0.9
