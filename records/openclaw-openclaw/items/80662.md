## Summary

This PR should stay open for maintainer review: the Agents directory idea is a real UI feature, but the branch mixes it with unrelated core filesystem/storage rewrites that introduce security regressions and the submitted proof still does not demonstrate the claimed search/back-navigation behavior.

## What This Changes

The PR replaces the Agents tab selector with a searchable card directory and URL-synced detail navigation while also changing i18n data, daemon/WebSocket logging, root JSON helpers, durable outbound queue storage, and adding a local build log.

## Best Possible Solution

Split or trim the branch to the Agents directory UI, keep the fs-safe-backed root-file and durable queue helpers intact, remove generated/local artifacts that are not part of the feature, and add real behavior proof for the directory flow.

## Reproduction Assessment

Yes for the review findings: the PR head and current-main source show the helper replacements, committed build transcript, and test type mismatch. I did not run tests because this was a read-only review.

## Solution Assessment

No. A searchable Agents directory may be a reasonable direction, but this patch is not the best solution because it bundles that UI work with unrelated storage, root-file, daemon, gateway, generated i18n, and local artifact changes.
