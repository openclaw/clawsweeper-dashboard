## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Build the timestamps URL after the voice id:** `extensions/elevenlabs/tts.ts:136`
  - body: This replacement maps `/v1/text-to-speech/<voice>` to `/v1/text-to-speech/with-timestamps<voice>`, but ElevenLabs documents `/v1/text-to-speech/:voice_id/with-timestamps`. Timestamp requests will hit the wrong route.
  - confidence: 0.97
- **[P1] Preserve the Buffer contract for telephony TTS:** `extensions/elevenlabs/tts.ts:121-128`
  - body: Changing `elevenLabsTTS` to return `{ audioBuffer, wordTimestamps }` leaves `synthesizeTelephony` returning that object as `audioBuffer`, while `SpeechTelephonySynthesisResult.audioBuffer` is still a `Buffer`. Telephony callers will receive the wrong shape.
  - confidence: 0.96
- **[P2] Expose timestamps on the textToSpeech result contract:** `extensions/speech-core/src/tts.ts:1167`
  - body: `textToSpeech` now returns `wordTimestamps`, but `TtsResult` in speech-core and the SDK `TextToSpeech` result type still omit that field. Keep the runtime return and public contract aligned before exposing this property.
  - confidence: 0.9
- **[P2] Import a real provider type in the new test:** `extensions/speech-core/src/tts-proof.test.ts:2`
  - body: From `extensions/speech-core/src`, `../../src/tts/provider-types` resolves under `extensions/src`, which does not exist, and `SpeechProviderPlugin` is not exported by the current provider type file. The new test will not compile as written.
  - confidence: 0.94
- **[P2] Wire the test through synthesizeSpeech config:** `extensions/speech-core/src/tts-proof.test.ts:24-28`
  - body: `synthesizeSpeech` requires `cfg` and resolves providers from configuration; it does not accept direct `provider` or `target` parameters. This mock provider is not wired into the production path the test claims to prove.
  - confidence: 0.93
