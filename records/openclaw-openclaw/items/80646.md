## Summary

Keep open. The PR is a recent external contribution and the current head is not mergeable as-is: the TTS changes break existing output assembly and provider contracts, while the useful memory work overlaps a narrower open fix.

## What This Changes

This PR changes memory-core dreaming chunk/token dedupe and expands ElevenLabs, speech-core, and plugin SDK TTS result types to carry character timestamp metadata, adding a new speech-core proof test.

## Best Possible Solution

Split or substantially rework the branch: land the memory fix through the focused memory-core path and design TTS timestamps through the provider, SDK, core, tests, and UI/API expectations tracked by the TTS feature request.

## Reproduction Assessment

Yes. The PR-head source itself shows the blockers: textToSpeech returns before current output assembly, ElevenLabs timestamp parsing uses the wrong endpoint/field and an undefined variable, telephony returns an object as a Buffer, and speech-core reads a field missing from the provider contract.

## Solution Assessment

No. The current patch changes shared TTS return shapes without updating all contracts and mixes a memory bug fix with a broader timestamp feature; the safer path is focused memory coverage plus an end-to-end TTS contract design using the documented vendor timing API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Restore textToSpeech output assembly:** `extensions/speech-core/src/tts.ts:1124`
  - body: The added early return exits before textToSpeech writes audioPath, runs channel pre-transcode, or computes audioAsVoice, and it leaves the original block outside the function referencing `synthesis` out of scope. Keep the existing output assembly and add timestamp metadata without returning the raw in-memory synthesis object.
  - confidence: 0.98
- **[P1] Use the documented ElevenLabs timing response:** `extensions/elevenlabs/tts.ts:137-159`
  - body: The timing endpoint is `/v1/text-to-speech/:voice_id/with-timestamps` and returns `audio_base64` plus alignment; this patch builds `/v1/text-to-speech-with-timestamps/:voice_id`, checks an undefined `contentType`, and decodes `json.audio`, so it will not compile or parse the vendor response. ([elevenlabs.io](https://elevenlabs.io/docs/api-reference/text-to-speech/convert-with-timestamps?utm_source=openai))
  - confidence: 0.96
- **[P1] Keep telephony returning a Buffer:** `extensions/elevenlabs/speech-provider.ts:580-598`
  - body: Changing `elevenLabsTTS` to return an object means `synthesizeTelephony` now returns that object in the `audioBuffer` field. The telephony path must unwrap `synthesis.audioBuffer`, or `SpeechTelephonySynthesisResult` no longer receives a Buffer.
  - confidence: 0.95
- **[P1] Add timestamps to the provider contract first:** `extensions/speech-core/src/tts.ts:1323`
  - body: `synthesizeSpeech` reads `synthesis.wordTimestamps`, but `SpeechProviderPlugin.synthesize` still returns `SpeechSynthesisResult` from `src/tts/provider-types.ts`, which has no timestamp field. Add the provider-side contract and update tests before reading the field in speech-core.
  - confidence: 0.93
- **[P3] Replace the no-op proof test with a real regression:** `extensions/speech-core/src/tts-proof.test.ts:33`
  - body: The new test never installs the mock provider or calls synthesizeSpeech; it only asserts true. Use the existing speech-core test harness to assert timestamp propagation so the regression can fail when the wiring breaks.
  - confidence: 0.9
