## Summary

Keep open: this external PR still has source-level correctness blockers in the TTS path and lacks after-fix real behavior proof; the memory portion overlaps a narrower open fix while the timestamp feature needs contract/product review.

## What This Changes

The PR modifies memory-core dreaming ingestion/dedupe and expands ElevenLabs, speech-core, and SDK TTS result types to carry timestamp metadata.

## Best Possible Solution

Split or substantially rework the branch: keep the memory fix aligned with https://github.com/openclaw/openclaw/pull/80620 or a focused replacement, and design TTS timestamps end-to-end against https://github.com/openclaw/openclaw/issues/80630.

## Reproduction Assessment

Yes. PR-head source gives a high-confidence reproduction of the blockers: textToSpeech is malformed, ElevenLabs timestamp parsing uses the wrong contract, telephony receives an object instead of a Buffer, and speech-core reads a field missing from the provider contract.

## Solution Assessment

No. This is not the best way to solve it because it changes shared return shapes without updating the full provider contract/caller set and mixes a memory bug fix with broader TTS timestamp feature work.

  - confidence: 0.95

  - body: synthesizeSpeech reads synthesis.wordTimestamps, but SpeechProviderPlugin.synthesize still returns SpeechSynthesisResult from src/tts/provider-types.ts without that field. Add the provider-side contract and tests before consuming the metadata in speech-core.
  - confidence: 0.93
