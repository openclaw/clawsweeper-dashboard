Full review comments:

- **[P1] Build the timing endpoint after the voice id:** `extensions/elevenlabs/tts.ts:132`
  - body: This replacement turns `/v1/text-to-speech/{voiceId}` into `/v1/text-to-speech/with-timestamps{voiceId}`. ElevenLabs documents the timing endpoint as `/v1/text-to-speech/voice_id/with-timestamps`, so normal synthesis will call an invalid endpoint before any timestamp parsing can work.
  - confidence: 0.96
- **[P1] Keep telephony synthesis returning a Buffer:** `extensions/elevenlabs/tts.ts:118`
  - body: Changing `elevenLabsTTS` from `Promise<Buffer>` to an object also affects `synthesizeTelephony`, which still assigns the helper result directly to `audioBuffer`. Successful telephony calls would return `{ audioBuffer, wordTimestamps }` where callers expect a `Buffer`.
  - confidence: 0.94
- **[P1] Port timestamp propagation to the current speech runtime:** `extensions/speech-core/src/tts.ts:125-130`
  - body: Current main routes TTS through `packages/speech-core/src/tts.ts` via the plugin SDK facade, while `extensions/speech-core/src/tts.ts` no longer exists. The timestamp propagation and test added under the old extension path will not cover the current runtime and need to be rebased/ported.
  - confidence: 0.91
