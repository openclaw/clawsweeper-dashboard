## Summary

Keep open: the goals are useful, but the current PR head is source-breaking in the TTS path, still lacks contributor-supplied real behavior proof, and mixes a memory bug fix with a broader TTS timestamp/API feature that needs rework.

## What This Changes

The PR changes memory-core dreaming ingestion/deduplication and expands ElevenLabs, speech-core, and plugin SDK TTS result types to carry word timestamp metadata.

## Best Possible Solution

Split or substantially rework the branch: keep the memory bug fix aligned with https://github.com/openclaw/openclaw/pull/80620 or a focused replacement, and implement TTS timestamps through the documented ElevenLabs endpoint with provider, SDK, core, tests, and UI/API expectations aligned with https://github.com/openclaw/openclaw/issues/80630.

## Reproduction Assessment

Yes. The PR-head source itself reproduces the blockers: textToSpeech is malformed, ElevenLabs timestamp parsing references an undefined variable and wrong response field, telephony returns the new object shape as a Buffer, and speech-core reads a field missing from the provider contract. I did not run tests because this review is read-only.

## Solution Assessment

No. The current patch is not the best way to solve it because it changes shared TTS return shapes without updating all callers/contracts and mixes a memory bug fix with a broader timestamp feature; the safer path is focused memory coverage plus an end-to-end TTS contract change using the vendor timing API.

  - body: The added early return exits before textToSpeech writes the audioPath, runs channel pre-transcode, or computes audioAsVoice, and it leaves the original block outside the function referencing synthesis out of scope. Keep the existing output assembly and add timestamp metadata without returning the raw in-memory synthesis object.
  - confidence: 0.98

  - confidence: 0.96

  - confidence: 0.95

  - body: synthesizeSpeech reads synthesis.wordTimestamps, but SpeechProviderPlugin.synthesize still returns SpeechSynthesisResult from src/tts/provider-types.ts, which has no timestamp field. Add the provider-side contract and update tests before reading the field in speech-core.
