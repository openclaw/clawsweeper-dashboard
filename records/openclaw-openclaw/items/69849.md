## Summary

Keep open. Current main still consumes ACP SDK schemas that leave the reported optional/nullish array fields without default empty arrays, and the bundled ACPX Claude path still includes transitive ACP SDK 0.20.0 while the latest published ACP SDK remains 0.21.0 with the same schema shape. The report is valid, but the next step is a maintainer dependency-strategy choice rather than a safe close.

## What This Changes

The issue asks OpenClaw to prevent Claude ACP registration failures by ensuring optional array fields in ACP SDK Zod schemas default to empty arrays.

## Best Possible Solution

Resolve this through an explicit dependency strategy: prefer bumping to upstream ACP SDK, Claude adapter, and ACPX releases that add default empty arrays to the generated schemas; if upstream remains unavailable, approve a tightly scoped temporary patch with package, lockfile, patch metadata, and focused ACP schema coverage aligned.

## Reproduction Assessment

Yes for the schema-contract reproduction: current main pins the affected ACP packages and the latest published SDK tarball still shows the optional/nullish arrays without .default([]). A live Claude registration repro was not run in this read-only pass.

## Solution Assessment

Unclear because no patch is proposed here. Adding defaults to the generated schema is likely the correct functional fix, but the maintainable OpenClaw path should be either an upstream fixed dependency bump or a maintainer-approved temporary patch.
