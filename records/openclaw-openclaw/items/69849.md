## Summary

Keep open. Current main still depends on the latest published ACP SDK schema shape where the reported optional/nullish array fields lack default empty arrays, and OpenClaw has no SDK patch in place; the cross-reference to anomalyco/opencode#23761 points to the same dependency-level compatibility problem.

## What This Changes

The issue asks OpenClaw ACP/ACPX to avoid Claude schema validation failures caused by optional ACP SDK array parameters that do not default to empty arrays.

## Best Possible Solution

Ship an upstream ACP SDK fix, or approve a tightly scoped temporary dependency patch that defaults the affected generated array schemas to [] and covers Claude ACP registration/schema behavior.

## Reproduction Assessment

Yes for the schema-contract path: current main pins the affected ACP packages, and the latest published SDK tarball still exposes the optional/nullish arrays without .default([]). A live Claude registration repro was not run because this read-only pass had no relevant credentials available.

## Solution Assessment

Unclear because no patch is proposed here. Adding defaults to the generated ACP schemas is the likely functional fix, but the maintainable OpenClaw path should be an upstream package bump if available or a maintainer-approved temporary patch.
