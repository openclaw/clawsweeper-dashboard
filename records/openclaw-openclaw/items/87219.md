## Summary

Keep open. I found no blocking review findings.

## What This Changes

This PR lets non-sandbox agent read tools resolve `media://inbound/...` to validated media-store paths for custom workspaces and adds workspace path tests.

## Best Possible Solution

Land the narrow media-reference fix after normal CI, keeping the workspace-only exception limited to validated inbound media-store references.

## Reproduction Assessment

Yes from source inspection, not from executing current main. Current main guards read paths before any inbound-media normalization and then passes the original path to upstream read execution, so `media://inbound/...` can be treated as a workspace-relative path.

## Solution Assessment

Yes. Reusing the existing `resolveInboundMediaReference` contract before upstream read execution is narrower than adding workspace-specific path hacks or broadening general filesystem roots.
