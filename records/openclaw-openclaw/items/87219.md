## Summary

Keep open: the PR is a focused fix for the custom-workspace inbound media bug, but the current patch adds generic read-wrapper media resolution that can break the existing sandbox media-ref path when the host media store also contains the inbound file.

## What This Changes

The PR resolves `media://inbound/...` references to managed inbound media files for non-sandbox host read tools and adds custom-workspace regression coverage.

## Best Possible Solution

Keep the managed inbound-media exception scoped to the non-sandbox host workspace guard, remove generic pre-resolution from `createOpenClawReadTool`, and add a sandbox regression where both the host media store and staged sandbox file exist.

## Reproduction Assessment

Yes from source inspection: current main routes sandbox media refs through `resolveMediaReferenceSandboxPath`, while the PR’s generic pre-resolution can intercept the same ref when the host media store has that id. I did not run tests because this review is read-only.

## Solution Assessment

No. The host guard exception is the right narrow direction, but resolving managed media in the generic read wrapper is the wrong layer because it runs before sandbox-specific path handling.
