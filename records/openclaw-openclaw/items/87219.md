## Summary

Keep this PR open: the non-sandbox custom-workspace fix is useful, but the current patch still introduces a blocking sandbox read regression by resolving managed media URIs in the generic read wrapper before sandbox-specific staging can run.

## What This Changes

The PR changes the agent read wrapper and workspace root guard so `media://inbound/...` references can resolve to managed inbound media files, with custom-workspace regression tests.

## Best Possible Solution

Resolve managed inbound refs for non-sandbox custom workspaces without changing sandbox read URI semantics, then add sandbox regression coverage and current-head real behavior proof.

## Reproduction Assessment

Yes for the PR blocker via source inspection: sandboxed reads call `createOpenClawReadTool`, and the PR resolves media URIs before sandbox operations can rewrite them. I did not run tests because this review is read-only.

## Solution Assessment

No. The non-sandbox custom-workspace behavior should be fixed at the host read guard or a host-only option, not by globally rewriting read args inside the generic wrapper used by sandboxed reads.
