## Summary

Keep open: the PR targets a real sessions_send metadata bug, but the proposed statuses are not merge-ready because the wait path reports delivery before the asynchronous A2A announce flow has completed or reported success. ## What This Changes

The PR changes `sessions_send` responses from the existing pending/skipped delivery metadata to sent/delivered/accepted/skipped statuses and updates focused sessions_send assertions.

## Best Possible Solution

Revise the async A2A response contract so normal async paths either omit delivery metadata or report only an observed completion state, while preserving the documented skipped case and adding real sessions_send proof.

## Reproduction Assessment

Yes, by source inspection: current main constructs non-skipped sessions_send delivery metadata as pending before launching the async A2A flow, then returns that object from normal response paths. I did not run a live A2A scenario in this read-only review.

## Solution Assessment

No: this PR is a plausible narrow patch, but it reports `delivered` before the asynchronous announce delivery can complete or fail. The safer fix is to make the response reflect only observable state or omit async delivery metadata.
