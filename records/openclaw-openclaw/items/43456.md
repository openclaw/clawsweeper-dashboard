## Summary

Keep open. Current main still preserves Mattermost thread IDs, but the Mattermost oncall gate only sees explicit mention facts, so unmentioned replies to bot-authored posts still drop as missing mentions.

## What This Changes

The issue requests Mattermost oncall wake-up behavior when a user replies to a bot-authored post, matching Telegram-style implicit reply-to-bot activation.

## Best Possible Solution

Implement Mattermost-local reply-to-bot detection, verify the referenced root or parent post was authored by the bot, feed reply_to_bot into the implicit mention path, and cover it with plugin tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unmentioned non-direct Mattermost post with root_id reaches the oncall gate with wasMentioned false, and the gate has no implicit reply-to-bot input before returning missing-mention.

## Solution Assessment

Yes. The narrow maintainable path is a Mattermost plugin-local referenced-post author check wired into the existing implicit mention contract; waking on every thread reply or changing core defaults would be broader than needed.
