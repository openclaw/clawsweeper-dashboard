## Summary

Keep open. Current main still has Mattermost thread routing, but `oncall` gating only wakes on explicit mention, authorized control-command bypass, or `onchar`, with no reply-to-bot implicit mention path.

## What This Changes

Request Mattermost to wake an `oncall` agent when a user replies to a bot-authored Mattermost post, matching Telegram-style reply behavior without a new config option.

## Best Possible Solution

Add Mattermost plugin-local referenced-post author detection, feed `reply_to_bot` into the existing mention decision path, and document Mattermost as a supported implicit-reply channel.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: an unmentioned Mattermost post with `root_id` reaches `evaluateMattermostMentionGate` with `wasMentioned: false` and is dropped as `missing-mention`.

## Solution Assessment

Yes. The narrowest maintainable solution is Mattermost plugin-local referenced-post author detection wired into the existing `reply_to_bot` implicit mention contract; waking on any thread reply or changing core defaults would be broader than needed.
