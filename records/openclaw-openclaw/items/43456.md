## Summary

Keep open. Current main still routes Mattermost thread replies through the normal `oncall` mention gate, but it does not resolve whether the referenced root post was authored by the bot, so reply-to-bot messages remain invisible unless they contain an explicit mention or use a broader chat mode.

## What This Changes

The issue requests that Mattermost `oncall` agents wake when a user replies to a bot-authored Mattermost post, matching Telegram reply behavior without adding a new config option.

## Best Possible Solution

Confirm the intended default wake policy, then add Mattermost plugin-local referenced-post author detection and map bot-authored root replies into the existing implicit mention path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unmentioned Mattermost post with `root_id` reaches the `oncall` gate with `wasMentioned: false`, and no referenced-post author check exists before `missing-mention` can be returned.

## Solution Assessment

Yes, assuming maintainers accept the default wake policy. The narrowest maintainable implementation is Mattermost plugin-local referenced-post detection wired into the existing implicit `reply_to_bot` mention contract.
