## Summary

Keep open. Current main still has Mattermost thread routing, but it does not convert replies to bot-authored posts into implicit mentions, so unmentioned `oncall` thread replies are still dropped.

## What This Changes

Request Mattermost `oncall` mode to wake when a user replies to a bot-authored post in threads or inline replies without adding a new config option.

## Best Possible Solution

Add Mattermost plugin-local referenced-post author detection, feed `reply_to_bot` into the existing mention decision path, and cover it with focused monitor/gating/client tests plus docs and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unmentioned Mattermost post with `root_id` reaches the oncall gate with `wasMentioned: false`, and no implicit reply-to-bot signal exists before `missing-mention` is returned.

## Solution Assessment

Yes. The narrowest maintainable solution is Mattermost plugin-local referenced-post author detection wired into the existing implicit mention contract; changing core defaults or waking on any thread reply would be broader than needed.
