## Summary

Keep open. Current main still preserves Mattermost thread context but does not turn replies to bot-authored posts into implicit mentions, so default oncall mode still drops unmentioned threaded replies.

## What This Changes

Mattermost should wake an oncall agent when a user replies to a post authored by that agent, matching Telegram-style reply-to-bot behavior.

## Best Possible Solution

Add Mattermost plugin-local reply-to-bot detection, verify the referenced post was authored by the bot, pass `reply_to_bot` through the shared implicit mention decision path, and update Mattermost tests, docs, and changelog.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: an unmentioned Mattermost post with `root_id` reaches the oncall gate with `wasMentioned: false`, and no reply-to-bot fact is available before the gate returns `missing-mention`. I did not run live Mattermost because this review is read-only and `pnpm` is unavailable in the environment.

## Solution Assessment

Yes. The narrow maintainable fix is plugin-local Mattermost wiring into the existing shared implicit mention contract; changing core defaults or treating any thread id as a mention would be broader and less safe.
