## Summary

Keep open. Current main still has the Mattermost wake gap: threaded Mattermost posts are routed into thread-scoped sessions, but the oncall wake gate only sees explicit mention, authorized command bypass, or onchar trigger state, so an unmentioned reply to a bot-authored post is still dropped.

## What This Changes

Mattermost should wake default oncall agents when a user replies to the bot's own message, including threaded replies, without requiring a repeated @mention.

## Best Possible Solution

Add a narrow Mattermost plugin implementation that resolves reply metadata, verifies the referenced post was authored by the bot, passes `reply_to_bot` through the shared mention decision path or an equivalent Mattermost gate extension, and documents Mattermost as a supported implicit-reply channel once regression tests pass.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in default oncall channel mode, an unmentioned Mattermost post with a `root_id` reaches the current handler with `wasMentioned: false`, and the gate returns `missing-mention` because no reply-to-bot fact is available.

## Solution Assessment

Yes. The maintainable solution is a plugin-local Mattermost wake fix using the existing shared implicit mention contract, with an API check of the referenced post author; changing core defaults or accepting raw thread ids as mentions would be broader and less safe.
