## Summary

Keep open. Current main still preserves Mattermost thread ids but does not turn replies to bot-authored posts into implicit mentions, so default `oncall` mode still drops unmentioned threaded replies.

## What This Changes

Request Mattermost default `oncall` behavior to wake when a user replies to the bot's own post, matching Telegram-style reply wakeups without a new config option.

## Best Possible Solution

Add Mattermost plugin-local reply-to-bot detection, verify the referenced post was authored by the bot, feed `reply_to_bot` into the existing implicit mention decision path, and cover it with Mattermost gating/monitor tests plus docs and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unmentioned Mattermost post with `root_id` reaches the oncall gate with `wasMentioned: false`, and no reply-to-bot signal exists before `missing-mention` is returned.

## Solution Assessment

Yes. The narrowest maintainable solution is Mattermost plugin-local referenced-post author detection wired into the existing implicit mention contract; changing core defaults or waking on any thread reply would be broader than needed.
