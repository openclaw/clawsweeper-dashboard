## Summary

Keep open. The force-push fixed the earlier mocked-export defect and the core fix shape is plausible, but this PR is not merge-ready because the real-runtime proof is still insufficient and the upgrade behavior for existing stored CLI prompt hashes can cause a one-time context reset.

## What This Changes

The branch adds a channel-neutral direct-chat prompt builder, uses it only for CLI reuse-hash static prompt input, updates the media-only mock, and adds direct-chat regression coverage.

## Best Possible Solution

Keep live provider labels in the actual prompt, make only the CLI reuse identity channel-neutral, and handle or explicitly approve the stored-hash transition before proving the real cross-channel claude-cli continuity path.

## Reproduction Assessment

Yes. Current main source shows provider-specific direct-chat context flows into the CLI static prompt hash while `session.dmScope:"main"` can route multiple direct channels to one session key, so channel switching can deterministically change `extraSystemPromptHash`.

## Solution Assessment

Mostly, but not yet complete. Normalizing only the reuse-hash input is the narrow fix shape, but the PR still needs upgrade-safe handling or approval for existing stored hashes plus real runtime proof.
