## Summary

Current main still matches the reported cron delivery bug: best-effort isolated cron delivery reaches the dispatch path, but active descendant subagents still trigger the full-timeout wait and active-descendant delivery suppression without checking `deliveryBestEffort`; the latest release tag has the same behavior and the linked fix PR is closed unmerged.

## What This Changes

An isolated cron job with `delivery.bestEffort: true` should deliver the parent run output promptly instead of waiting for fire-and-forget descendant subagents to drain.

## Best Possible Solution

Make the active-descendant wait and suppression paths respect the existing `deliveryBestEffort` flag while preserving non-best-effort orchestration semantics and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: `deliveryBestEffort: true` is passed into dispatch, but active descendants still take the unguarded wait and suppression branches; I did not run a live cron job in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to gate the active-descendant wait and suppression on `!params.deliveryBestEffort`, leaving non-best-effort descendant follow-up behavior intact.
