## Summary

Current main still matches the reported bug: the existing best-effort flag reaches isolated cron delivery, but active descendant subagents still trigger the full-timeout wait and the active-descendant delivery suppression without checking that flag. The related fix PR was closed unmerged, so the issue should stay open and is a good narrow repair candidate.

## What This Changes

The issue reports that best-effort isolated cron delivery waits for active fire-and-forget subagents until timeout instead of delivering the parent result immediately.

## Best Possible Solution

Make the active-descendant wait and suppression paths respect the existing best-effort flag, while preserving non-best-effort descendant follow-up semantics and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: `deliveryBestEffort: true` reaches dispatch, then active descendants still take the unguarded wait and suppression branches; I did not run a live cron job in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to gate the active-descendant wait and suppression on non-best-effort delivery, with tests proving best-effort delivers immediately while non-best-effort still waits for descendant summaries.
