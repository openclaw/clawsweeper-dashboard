## Summary

Keep open: current main still has the unguarded best-effort cron descendant wait and active-descendant delivery suppression, and the only directly linked fix PR was closed unmerged. The bug is source-reproducible and narrow enough for a focused repair PR with regression coverage.

## What This Changes

The issue requests that isolated cron jobs with `delivery.bestEffort: true` deliver the parent run result immediately instead of waiting for active fire-and-forget subagents until the cron timeout.

## Best Possible Solution

Land a small cron delivery fix that makes best-effort isolated delivery skip descendant waiting and active-descendant suppression while preserving non-best-effort orchestration behavior and adding focused regression tests.

## Reproduction Assessment

Yes, at source level. On current main, an isolated agentTurn cron with `delivery.bestEffort: true`, parent synthesized text, and active descendant runs enters `waitForDescendantSubagentSummary` with the full cron timeout, then can return through the active-descendant suppression path instead of delivering immediately.

## Solution Assessment

Yes. The narrowest maintainable fix is to consult `deliveryBestEffort` at the descendant wait and active-descendant suppression gates in `dispatchCronDelivery`, with tests preserving the existing non-best-effort behavior and defining stale-interim handling.
