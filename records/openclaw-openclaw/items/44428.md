## Summary

Current main still matches the reported bug: `deliveryBestEffort` reaches isolated cron delivery, but active descendant subagents still trigger the full-timeout wait and active-descendant delivery suppression without checking that flag. The linked fix PR was closed unmerged, and exact GitHub searches did not find an open replacement, so this should stay open as a narrow repair candidate.

## What This Changes

The issue requests that isolated cron jobs with `delivery.bestEffort: true` deliver the parent result immediately instead of waiting for fire-and-forget subagents.

## Best Possible Solution

Gate the descendant wait and active-subagent suppression on non-best-effort delivery, preserve required-delivery follow-up behavior, and add focused regression coverage for best-effort with active descendants.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: `deliveryBestEffort: true` reaches dispatch, then active descendants still take unguarded wait and suppression branches using the full timeout budget.

## Solution Assessment

Yes. The existing config flag already expresses the intended policy, so the narrow maintainable fix is to use it in the active-descendant branches rather than adding a new mode or changing subagent execution.
