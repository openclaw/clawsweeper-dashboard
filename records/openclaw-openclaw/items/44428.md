## Summary

Current main still matches the reported bug. `delivery.bestEffort` is resolved and passed into isolated cron delivery, but text delivery still waits for active descendant subagents with the full cron timeout and still suppresses parent delivery while descendants are active without checking that flag. The latest release tag in this checkout contains the same active-descendant wait/suppression behavior, and the related fix PR #46160 is closed unmerged.

## Best Possible Solution

Keep this open and land a narrow cron delivery fix: when `delivery.bestEffort` is true, isolated cron text delivery should not wait for active fire-and-forget descendants and should not suppress the parent delivery solely because descendants are active. Preserve existing non-best-effort descendant synthesis/suppression behavior, preserve completed-descendant fallback when descendants already finished, and add regression coverage for best-effort plus active descendants.
