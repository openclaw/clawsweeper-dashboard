## Summary

Current main still matches the reported bug: `deliveryBestEffort` reaches isolated cron delivery, but active descendants still trigger the full-timeout wait and active-descendant delivery suppression without checking that flag. The linked repair PR is closed unmerged and a live PR search did not find an open replacement, so this should stay open as a narrow repair candidate.

## What This Changes

An isolated cron job with `delivery.bestEffort: true` should deliver its own output without waiting for active fire-and-forget subagents.

## Best Possible Solution

Gate the descendant wait and active-subagent suppression on non-best-effort delivery, preserve required-delivery follow-up behavior, and add focused regression coverage for best-effort with active descendants.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: `deliveryBestEffort: true` reaches dispatch, then active descendants still take unguarded wait and suppression branches using the full timeout budget.

## Solution Assessment

Yes. The existing config flag already expresses the intended policy, so the narrow maintainable fix is to use it in the active-descendant branches instead of adding a new mode or changing subagent execution.
