## Summary

This PR is still necessary: current main still uses an unbounded `response.json()` on the embedded OpenRouter capability catalog, while the branch applies the shared byte-capped reader with focused streamed-overflow and SQLite cache coverage. I found no line-level correctness finding, but the new 16 MiB fail-closed limit remains a compatibility tradeoff for maintainers to accept before merge.

## What This Changes

The branch replaces the embedded OpenRouter model-capability catalog `response.json()` read with `readResponseWithLimit` and adds streamed overflow plus chunked SQLite cache regression tests.

## Best Possible Solution

Land the focused runtime bounded-read fix once maintainers accept 16 MiB as the OpenRouter capability catalog ceiling; keep the adjacent model-scan and plugin-SDK catalog hardening in their separate PRs.

## Solution Assessment
