## Summary

Keep open. Current main still encodes the older Anthropic long-context contract: OAuth/setup-token requests strip the 1M beta, 4.6 context resolution remains `context1m`-gated except for newer Opus 4.7 handling, `models list` rows still report raw catalog/config context metadata, and docs still describe 1M as beta-gated/OAuth-rejected despite Anthropic's March 13, 2026 GA announcement for Opus 4.6 and Sonnet 4.6.

## What This Changes

The issue requests support for Anthropic Claude Opus 4.6 1M context with OAuth/Claude Max auth, including correct runtime context and `models list` reporting instead of a 200k/195k fallback.

## Best Possible Solution

Update the Anthropic plugin-owned long-context contract in one coordinated change: treat eligible Opus/Sonnet 4.6 as GA 1M where Anthropic now allows it, align OAuth/setup-token header behavior with the no-beta-required contract, preserve correct older/ineligible fallbacks, and keep runtime budgeting, models/status display, docs, and tests consistent.

## Reproduction Assessment

Yes for a source-level reproduction: current main has code and tests that strip the 1M beta for OAuth credentials and keep Opus 4.6 at 200k unless `context1m` is set. I did not run a live Anthropic request in this read-only review.

## Solution Assessment

Unclear. The requested direction matches Anthropic's published 4.6 GA contract, but the safest repository fix needs maintainer/provider-contract review so request headers, OAuth credentials, runtime context, display, docs, and overlapping OAuth-header work do not drift.
