## Summary

Keep open. Current main fixed part of the 4.6 context denominator path, but the Anthropic OAuth/setup-token request path still strips the `context-1m-2025-08-07` beta and docs/tests still encode that fallback, so the reported OAuth long-context problem is not fully resolved.

## What This Changes

The issue asks OpenClaw to report and use the 1M context window for Anthropic Opus 4.6 with OAuth/Claude Max credentials instead of stripping the long-context beta and falling back to about 200k.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then align the Anthropic plugin wrapper, context resolver, model/status display, docs, and tests so eligible OAuth users get a real 1M path and ineligible credentials get an explicit fallback.

## Reproduction Assessment

Yes for the OpenClaw side: source inspection shows an `sk-ant-oat` request with `context1m: true` still has `context-1m-2025-08-07` filtered out, and tests assert that behavior. I did not run a live Anthropic OAuth request, so provider eligibility still needs contract confirmation.

## Solution Assessment

Unclear. The requested direction appears plausible, but the maintainable fix needs provider-contract review so request headers, context budgeting, docs, and auth-specific fallback behavior do not drift.
