## Summary

Keep open. Current main has fixed part of the context-denominator behavior for Anthropic Opus/Sonnet 4 models, but the Anthropic OAuth/setup-token request path still strips `context-1m-2025-08-07`, and current docs/tests still encode that OAuth fallback as intentional.

## What This Changes

The issue asks OpenClaw to support Claude Opus 4.6 1M context for OAuth/Claude Max users instead of reporting or enforcing a 200k/195k path.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then align the Anthropic wrapper, context budgeting, models/status display, docs, and tests so eligible OAuth users get real 1M requests and ineligible credentials get an explicit fallback.

## Reproduction Assessment

Yes for the OpenClaw-side path: source inspection shows `context1m: true` plus an `sk-ant-oat` credential still filters out `context-1m-2025-08-07`, and a test asserts that behavior. A live Anthropic OAuth request was not run, so provider eligibility remains the external contract gap.

## Solution Assessment

Unclear. The maintainable fix is not a resolver-only change; it needs an auth-aware provider-contract decision that keeps request headers, context planning, display, docs, and tests in agreement.
