## Summary

Keep open. Current main still encodes the older Anthropic 1M-context contract for Opus 4.6: 1M context is `context1m`-gated, OAuth-style tokens strip the long-context beta, model-list rows use raw catalog context metadata, and docs still describe 4.6 long context as beta/OAuth-rejected even though Anthropic release notes now say Opus 4.6 and Sonnet 4.6 work over 200k automatically with no beta header. 

## What This Changes

The issue requests that Anthropic Opus 4.6 OAuth/Claude Max users get the GA 1M context window instead of a 200k display/runtime fallback.

## Best Possible Solution

Update the Anthropic plugin-owned long-context contract in one coordinated change: treat eligible Opus/Sonnet 4.6 as GA 1M where Anthropic now allows it, remove or bypass obsolete beta/OAuth fallback behavior for that path, preserve correct older/ineligible fallbacks, and align models/status display, runtime budgeting, docs, and tests.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction on current main: `context1m` remains required for 4.6 to resolve to 1M, OAuth requests still strip the long-context beta, and tests/docs enforce that old behavior. I did not run a live Anthropic OAuth request in this read-only review.

## Solution Assessment

Unclear. The requested direction matches Anthropic's current public 4.6 GA contract, but the safest repository fix needs maintainer/provider-contract review so resolver behavior, OAuth/setup-token headers, runtime context, display, docs, and related OAuth-header work stay in sync.
