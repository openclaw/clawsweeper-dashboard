## Summary

Keep open. Current main fixed the context-window denominator for explicit Anthropic Opus/Sonnet 4 refs, but it still removes the Anthropic 1M beta from OAuth/setup-token requests and the docs/tests still encode that fallback, so the remaining work needs provider-contract review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to allow Anthropic Claude Opus 4.6 1M context for OAuth/Claude Max users instead of showing or enforcing the older 200k-class limit.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then align the Anthropic wrapper, context budgeting, models/status display, docs, and tests so eligible OAuth users get a real 1M request path while ineligible credentials get a clear fallback.

## Reproduction Assessment

Yes for the OpenClaw-side behavior: source inspection shows `context1m: true` plus an `sk-ant-oat` credential still filters out `context-1m-2025-08-07`, and the focused test asserts that fallback. A live Anthropic OAuth request was not run, so provider eligibility remains the external contract gap.

## Solution Assessment

Unclear. A resolver-only or display-only change is not enough; the maintainable fix needs an auth-aware provider-contract decision that keeps request headers, context planning, docs, and tests consistent.
