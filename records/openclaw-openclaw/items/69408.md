## Summary

Keep open. Current main and v2026.5.18 still emit same-run generic loop warnings at warning level, and the cron-to-agent hook path still lacks the isolated-cron schedule/lifetime context needed for a conservative info-level demotion.

## What This Changes

The issue requests demoting loop-detection warnings to info for bounded isolated cron sessions while preserving warning/critical behavior elsewhere.

## Best Possible Solution

Thread typed isolated-cron context and schedule/run boundedness into the tool-loop diagnostic path, then demote only non-critical bounded isolated-cron warnings to info.

## Reproduction Assessment

Yes, source-backed: repeated same-tool/same-args calls in one run still reach the generic_repeat warning branch, then the hook logs and emits warning-level diagnostics. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded interval proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
