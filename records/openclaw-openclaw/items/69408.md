## Summary

Keep open. Current main still treats repeated generic tool calls inside the same run as warning-level loop diagnostics, and the isolated cron path still does not pass session target, schedule interval, or bounded-lifetime metadata into that decision.

## What This Changes

Request demoting loop-detection warnings to info for bounded isolated cron sessions whose run lifetime is no longer than the schedule interval.

## Best Possible Solution

Thread typed isolated-cron run and schedule metadata into the loop-warning path, demote only non-critical bounded isolated-cron diagnostics to info, and preserve critical blocking unchanged.

## Reproduction Assessment

Yes, source-backed. A repeated `exec` call within one `runId` reaches `generic_repeat` and is emitted as warning-level log and diagnostic output; this review did not run a live cron job.

## Solution Assessment

Yes for a conservative typed-metadata demotion; no for the hardcoded five-minute proxy in the issue body. The narrow maintainable path is to prove isolated target plus bounded schedule/run lifetime before lowering severity.
