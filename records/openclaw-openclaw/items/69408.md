## Summary

Keep open. Current main still emits the reported loop-detection path at warning level and the hook/diagnostic contracts do not carry isolated-cron bounded-lifetime metadata needed for a conservative demotion.

## What This Changes

The issue requests lowering non-critical loop-detection warnings to info for bounded isolated cron sessions so legitimate retry-heavy jobs stop filling the gateway error log.

## Best Possible Solution

Thread typed isolated-cron target and schedule/run timing metadata into the loop-warning path, demote only non-critical bounded isolated-cron diagnostics to info, and keep critical plus non-isolated warnings unchanged.

## Reproduction Assessment

Yes, source-backed. Repeated same-tool/same-args calls within one run still reach generic_repeat warning-level detection and warning-level hook diagnostics; this review did not execute a live cron job.

## Solution Assessment

Yes for a conservative typed-metadata demotion; no for the proposed hardcoded five-minute proxy. The maintainable path should prove the run is isolated and bounded by real schedule/run metadata before lowering severity.
