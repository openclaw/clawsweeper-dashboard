## Summary

Keep open. Current main and v2026.5.18 still emit same-run generic loop warnings at warning level, and the hook/diagnostic path does not carry isolated-cron boundedness metadata needed for a conservative info-level demotion.

## What This Changes

The issue requests lowering loop-detection logging from warning to info for bounded isolated cron sessions whose run lifetime stays within the cron interval.

## Best Possible Solution

Thread typed isolated-cron context and schedule/run boundedness into tool-loop diagnostics, demote only bounded isolated-cron warning events to info, and keep critical or non-isolated behavior unchanged.

## Reproduction Assessment

Yes, source-backed: repeated same-tool/same-args calls in one run still reach generic_repeat warning-level detection, then the hook logs warning-level diagnostics. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded five-minute proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
