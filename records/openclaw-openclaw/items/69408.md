## Summary

Keep open. Current main and the latest stable release still emit same-run loop-detection warnings at warning level, and the cron/agent hook path still lacks isolated-run schedule and lifetime metadata needed for a conservative info-level demotion.

## What This Changes

The issue asks to demote false-positive loop-detection warnings from bounded isolated cron sessions from warn to info while preserving the audit trail.

## Best Possible Solution

Thread typed isolated-cron context and schedule/run boundedness into the tool-loop diagnostic path, demote only non-critical bounded isolated-cron warnings to info, and preserve current warning/critical behavior everywhere else.

## Reproduction Assessment

Yes, source-backed: repeated same-tool/same-args calls in one run still reach generic_repeat warning-level detection, then the hook logs and emits warning-level diagnostics. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded interval proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
