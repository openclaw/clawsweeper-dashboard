## Summary

Keep open. Current main still emits the reported generic repeated-tool loop path as warning-level logs/diagnostics, and isolated cron execution still does not carry session target, schedule interval, or run-lifetime context into the loop-warning decision.

## What This Changes

The issue asks OpenClaw to demote non-critical loop warnings to info for bounded isolated cron sessions whose run lifetime is no longer than the cron interval.

## Best Possible Solution

Thread typed isolated-cron schedule/run metadata into the tool-loop path, demote only non-critical bounded isolated-cron diagnostics to info, and preserve critical blocking behavior unchanged.

## Reproduction Assessment

Yes, source-backed. An isolated cron run that makes repeated `exec` calls in one run reaches the generic-repeat warning path, while the current hook has only `runId` context and logs the result as warn-level.

## Solution Assessment

Yes, if implemented with typed cron metadata and a conservative schedule bound. The hardcoded five-minute proxy from the issue body is not the best direction because it could demote real warnings without proving the run is bounded by its schedule.
