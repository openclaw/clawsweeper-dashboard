## Summary

Keep open. Current main still emits repeated generic tool-call loop detections as warning-level logs/diagnostics, and isolated cron execution still does not carry session target, schedule interval, or elapsed run-lifetime context into the loop-warning decision.

## What This Changes

Downgrade non-critical loop-detection warnings to info for isolated cron sessions that are proven bounded by their schedule interval.

## Best Possible Solution

Carry typed isolated-cron schedule and run-lifetime context into the loop-warning path, demote only non-critical bounded isolated-cron diagnostics to info, and preserve critical loop blocking unchanged.

## Reproduction Assessment

Yes. The issue gives a concrete isolated-cron repeated-exec path, and current source shows repeated same-run generic calls still produce warning-level loop results while cron execution does not provide bounded schedule context to the hook.

## Solution Assessment

Yes. The maintainable fix is to pass typed schedule/run metadata into the loop-warning decision and add an info-level diagnostic path; a hardcoded interval proxy would be weaker and risk demoting real warnings.
