## Summary

Keep open: current main and the latest release still emit same-run generic tool-loop diagnostics at warning level, and the cron-to-tool-hook path still lacks typed bounded-isolated-run metadata for a safe info-level demotion. The related run-scoped cron session-key fix is useful context, but it only fixed cross-run accumulation, not repeated exec calls inside one bounded isolated run.

## What This Changes

Demote loop-detection warnings from warn to info for bounded isolated cron sessions whose run lifetime is no longer than the cron interval.

## Best Possible Solution

Thread typed bounded isolated-cron context into tool-loop diagnostics, allow non-critical tool-loop events to be logged at info, and demote only bounded isolated-cron warnings while preserving critical and non-isolated behavior.

## Reproduction Assessment

Yes. Source inspection and existing tests show repeated same-tool/same-args calls within one run still reach `generic_repeat` warning-level detection, then the hook logs warning-level diagnostics; no live cron run was executed.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded five-minute proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
