## Summary

Keep open. Current main still emits repeated generic tool-call loop diagnostics at warning level without isolated-cron schedule or bounded-lifetime context, and no live related issue or PR appears to own this exact log-severity refinement.

## What This Changes

The issue requests demoting loop-detection warnings to info for bounded isolated cron sessions while preserving stronger warnings outside that context.

## Best Possible Solution

Thread typed isolated-cron run and schedule metadata into the tool-loop warning path, demote only non-critical bounded isolated-cron diagnostics to info, and leave critical blocking plus non-isolated warnings unchanged.

## Reproduction Assessment

Yes, source-backed. A repeated same-tool/same-args call within one run reaches `generic_repeat`, returns warning-level, and the hook emits warning-level logs/diagnostics; this review did not run a live cron job.

## Solution Assessment

Yes for a conservative typed-metadata demotion; no for the issue body's hardcoded five-minute proxy. The maintainable fix should use actual cron/session metadata and prove bounded lifetime before lowering severity.
