## Summary

Keep open. Current main still emits the reported generic tool-loop path at warning level and the hook/diagnostic contracts still lack isolated-cron bounded-lifetime metadata, so this is not implemented or superseded.

## What This Changes

The issue requests demoting non-critical loop-detection warnings to info for bounded isolated cron sessions.

## Best Possible Solution

Thread typed isolated-cron target and schedule/run timing metadata into the loop-warning path, demote only bounded non-critical isolated-cron diagnostics to info, and keep critical plus non-isolated warnings unchanged.

## Reproduction Assessment

Yes, source-backed. Repeated same-tool/same-args calls still reach a generic_repeat warning result and the hook emits that result at warning level; I did not run the full cron log-volume measurement.

## Solution Assessment

Yes for a conservative typed-metadata demotion; no for the proposed hardcoded five-minute proxy. The maintainable path should prove the run is isolated and bounded by real schedule/run metadata before lowering severity.
