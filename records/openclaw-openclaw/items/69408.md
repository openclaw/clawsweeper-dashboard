## Summary

Keep open. Current main still emits the reported non-critical generic repeated-tool path as warning-level subsystem and diagnostic logs, and isolated cron execution still does not pass bounded session-target or schedule-interval context into that decision.

## What This Changes

The issue asks to demote non-critical loop warnings to info for isolated cron sessions whose run lifetime is bounded by the cron interval.

## Best Possible Solution

Thread typed bounded isolated-cron metadata into the tool-loop path, demote only non-critical bounded cron diagnostics to info, and preserve critical blocking and unsafe warning behavior unchanged.

## Reproduction Assessment

Yes, source-backed. An isolated cron run that makes repeated identical `exec` calls in one run reaches the generic warning branch, and the current hook has no bounded isolated-cron context to choose info-level logging.

## Solution Assessment

Yes, if implemented with typed cron metadata and a conservative schedule-bound check. The hardcoded five-minute proxy suggested in the issue body is not the best approach because it could demote real warnings without proving the run is bounded by its schedule.
