## Summary

Keep open. Current main still emits repeated tool-call detections as warning-level logs/diagnostics, and isolated cron execution does not pass target, schedule interval, or run-lifetime context into the loop-warning decision.

## What This Changes

The issue asks OpenClaw to downgrade non-critical loop-detection logs from warn to info for bounded isolated cron sessions whose run lifetime stays within their schedule interval.

## Best Possible Solution

Add typed isolated-cron run context to the before-tool-call loop path and demote only non-critical loop diagnostics when the run is proven bounded by its schedule interval.

## Reproduction Assessment

Yes. The issue gives a concrete isolated-cron repeated-exec path, and current source shows repeated same-run tool calls still produce warning-level loop results while cron execution lacks bounded schedule context in the hook.

## Solution Assessment

Yes for the requested direction. The maintainable fix should carry typed schedule/run metadata into the loop-warning path and add an info-level diagnostic path, not rely on a hardcoded five-minute proxy.
