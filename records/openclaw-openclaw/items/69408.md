## Summary

Keep open: current main and v2026.5.18 still emit same-run generic tool-loop diagnostics at warning level, and the cron-to-tool-hook path still lacks typed bounded-isolated-run metadata for a conservative info-level demotion.

## What This Changes

Request to demote bounded isolated-cron tool-loop warnings from warn to info so legitimate repeated exec retries do not fill `gateway.err.log`.

## Best Possible Solution

Thread typed isolated-cron context and schedule/run boundedness into tool-loop diagnostics, demote only bounded isolated-cron warning events to info, and keep critical or non-isolated behavior unchanged.

## Reproduction Assessment

Yes. Source inspection and existing tests show repeated same-tool/same-args calls within one run still reach `generic_repeat` warning-level detection, then the hook logs warning-level diagnostics; no live cron run was executed in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded five-minute proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
