## Summary

Keep open. Current main still emits generic tool-loop diagnostics at warning level without isolated-cron bounded-lifetime metadata, and live searches did not find a canonical merged/open fix for this exact severity refinement.

## What This Changes

The issue requests demoting bounded isolated-cron loop-detection warnings from warning to info while preserving audit visibility.

## Best Possible Solution

Thread typed isolated-cron target and schedule/run timing metadata into the tool-loop warning path, demote only non-critical bounded isolated-cron diagnostics to info, and leave critical blocking plus non-isolated warnings unchanged.

## Reproduction Assessment

Yes, source-backed. Repeated same-tool/same-args calls in one run reach `generic_repeat`, return warning-level, and the hook emits warning-level logs/diagnostics; this review did not execute a live cron job.

## Solution Assessment

Yes for a conservative typed-metadata demotion; no for the hardcoded five-minute proxy. The maintainable fix should use actual cron/session/run metadata and prove bounded lifetime before lowering severity.
