## Summary

Keep open. Current main and the latest shipped release still emit generic same-run loop warnings at warning level, and the tool hook does not carry enough isolated-cron boundedness metadata to safely demote only the intended cron cases.

## What This Changes

Request to downgrade loop-detection warning logs to info for bounded isolated cron sessions whose run lifetime is shorter than the cron interval.

## Best Possible Solution

Thread typed isolated-cron boundedness into the tool-loop hook and diagnostic path, then demote only non-critical bounded isolated-cron warning events to info while leaving critical and non-isolated behavior unchanged.

## Reproduction Assessment

Yes, source-backed: repeated same-tool/same-args calls in one run still reach generic_repeat warning-level detection, and the before-tool-call hook logs warning-level diagnostics. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded five-minute proxy. The maintainable fix should prove the run is isolated and bounded by actual schedule/run timing before lowering severity.
