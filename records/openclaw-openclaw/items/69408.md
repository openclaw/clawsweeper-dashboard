## Summary

Keep open. Current main and the latest stable tag still emit same-run generic loop detections at warning level, while the cron-to-agent hook path does not carry the isolated-run schedule/lifetime metadata needed for a conservative info-level demotion.

## What This Changes

The issue requests lowering warning-level tool-loop diagnostics to info for bounded isolated cron runs that complete within their cron interval.

## Best Possible Solution

Thread typed isolated-cron schedule and run-boundedness context into the tool-loop diagnostic path, demote only non-critical bounded isolated-cron warnings to info, and preserve current critical/non-isolated behavior.

## Reproduction Assessment

Yes, source-backed. Repeated same-run same-tool/same-args calls still hit the generic_repeat warning branch and warning-level hook diagnostics; I did not run a live cron job in this read-only review.

## Solution Assessment

Yes for a typed metadata-based demotion; no for the proposed hardcoded interval proxy. The maintainable fix should prove the run is isolated and bounded by real schedule/run timing before lowering severity.
