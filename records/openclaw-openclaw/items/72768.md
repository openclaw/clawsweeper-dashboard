## Summary

Keep open. Current main still lacks a `config.changed` gateway event, so the feature is not obsolete, but the PR has blocking correctness and security issues: it compares two post-write snapshots instead of previous-to-next config state, can emit nested sensitive values, and does not wire the new payload schema into the public protocol registry.

## What This Changes

The PR adds a `config.changed` Gateway event with a structured config diff, admin-scoped broadcast wiring, listener cleanup, feature-list registration, and a new event payload schema for config hot-reload notifications.

## Best Possible Solution

Revise the PR so `config.changed` is built from a true previous-to-next config comparison, deep-redacts every emitted value before broadcast, and wires the event payload through the protocol schema registry, docs, and focused tests while preserving the ADMIN scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a config write notification contains only post-write `sourceConfig` and `runtimeConfig`, so the PR cannot produce the advertised previous-to-next diff; a nonsensitive parent object containing nested `apiKey` or `token` fields can also reproduce the redaction bypass when emitted as a whole value.

## Solution Assessment

No. The listener and admin broadcast seam are reasonable, but this implementation needs previous-state data, value-wide deep redaction, and protocol export wiring before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Compare against the previous config snapshot:** `src/gateway/server-runtime-subscriptions.ts:101`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both post-write snapshots. For a normal edit, this can return no diff or source-vs-runtime noise instead of the user's before/after config change. Capture/pass the previous compare snapshot before broadcasting.
  - confidence: 0.93
- **[P1] Deep-redact emitted diff values:** `src/gateway/config-diff.ts:76`
  - body: The helper only redacts by the immediate key before recursion, then emits whole `before`/`after` values for mixed-type and depth-cap changes. A nonsensitive parent containing nested credentials can be broadcast unredacted to admin clients.
  - confidence: 0.9
- **[P2] Export the new protocol schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: The PR adds a public event payload schema but does not export it from the schema barrel or register it in `ProtocolSchemas`, leaving protocol consumers and generated clients unable to discover the new contract.
  - confidence: 0.86
