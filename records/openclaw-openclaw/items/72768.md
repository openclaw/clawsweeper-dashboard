## What This Changes

The PR adds an ADMIN-scoped Gateway `config.changed` broadcast with structured config diffs, event discovery, protocol schema, scope gating, and shutdown cleanup wiring.

## Best Possible Solution

Emit from the reload comparison path or carry a real previous snapshot, recursively redact all emitted values, register/export the schema, preserve current `diffConfigPaths`, and add focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes for the review findings via source inspection: the PR diff compares post-write source/runtime projections, and its diff helper emits whole values after only immediate-key redaction. I did not run a live gateway repro.

## Solution Assessment

No. The event direction is plausible, but this patch hooks the wrong state boundary for before/after semantics, leaves shallow redaction in a security-sensitive payload, and does not register the protocol schema.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Compare against the previous config snapshot:** `src/gateway/server-runtime-subscriptions.ts:104`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both post-write projections, and external file-watcher hot reloads do not flow through this listener. This can suppress ordinary edits or report source-vs-runtime noise instead of the advertised before/after config change.
  - confidence: 0.93
- **[P1] Deep-redact values before broadcasting the diff:** `src/gateway/config-diff.ts:52`
  - body: `redactIfSensitive` only checks the immediate key before emitting a whole value. If a nonsensitive parent such as `providers` is added, removed, changes type, or hits the depth cap, nested credentials can be broadcast to ADMIN clients unredacted.
  - confidence: 0.91
- **[P2] Register the new protocol schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: Adding `schema/config-diff.ts` alone leaves the event payload schema undiscoverable because the schema barrel and `ProtocolSchemas` map are not updated. Protocol/codegen consumers cannot validate or reuse the new event contract.
  - confidence: 0.86
