## Summary

Keep open. Current main still lacks the requested `config.changed` event, but this PR is not ready to merge because it compares post-write source/runtime projections instead of previous-to-next reload state and can broadcast unredacted nested secrets.

## What This Changes

The PR adds a Gateway `config.changed` event, a config diff helper/schema, ADMIN-scoped broadcast wiring from runtime config write listeners, and close-handler cleanup for the new subscription.

## Best Possible Solution

Emit `config.changed` from the reload comparison path, or pass a real previous snapshot into a notification, then deep-redact arbitrary emitted values, register/export the protocol schema, and add focused tests plus docs/changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: ordinary writes can produce no event because source and runtime are both post-write, while adding/removing or type-changing a nonsensitive object containing nested credentials can put the secret into the broadcast payload.

## Solution Assessment

No. The ADMIN-scoped event is a plausible feature direction, but the runtime write-listener hook is too low-level for before/after hot-reload semantics; the existing reload path is the narrower maintainable place to emit it.

## Review Findings

Full review comments:

- **[P1] Compare against the previous reload snapshot:** `src/gateway/server-runtime-subscriptions.ts:104`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both post-write projections, and file-watcher reloads do not flow through this listener. This can suppress normal edits or report source-vs-runtime noise instead of the advertised before/after config change; emit from the reload path's previous-to-next comparison state or pass an explicit previous snapshot.
  - confidence: 0.93
- **[P1] Deep-redact emitted diff values:** `src/gateway/config-diff.ts:76`
  - body: The helper redacts only by the immediate key before emitting whole `before`/`after` values for mixed-type and depth-capped changes. A nonsensitive parent object containing nested credentials can still be broadcast unredacted to admin clients, so every emitted value needs deep redaction.
  - confidence: 0.91
- **[P2] Register the protocol schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: This adds a public event payload schema, but the new schema is not exported from the schema barrel or registered in `ProtocolSchemas`. Protocol/codegen consumers that rely on the exported schema map cannot discover or validate the new contract.
  - confidence: 0.86
