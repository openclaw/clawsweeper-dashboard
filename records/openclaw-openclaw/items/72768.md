## Summary

Keep open: current main still lacks the requested Gateway config-change event, but this PR has correctness and security blockers plus missing real behavior proof, so it is not a cleanup-close candidate.

## What This Changes

The PR adds a Gateway `config.changed` event, config diff payload/schema, ADMIN-scoped broadcast on runtime config write notifications, and close-handler cleanup for the new subscription.

## Best Possible Solution

Emit the event from the reload comparison path or pass a real previous snapshot into the notification, deep-redact all emitted values, register/export the schema, and add focused tests plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ordinary runtime writes compare post-write source/runtime projections, while nested credentials under nonsensitive parent keys can reach emitted `before`/`after` values.

## Solution Assessment

No. The event direction is plausible, but the current write-listener hook is not the narrowest maintainable source for previous-to-next hot-reload semantics, and the payload redaction/schema registration need revision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Compare against a real previous config snapshot:** `src/gateway/server-runtime-subscriptions.ts:104`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both post-write projections, and file-watcher reloads do not flow through this listener. This can suppress normal edits or report source-vs-runtime noise instead of the advertised before/after config change; emit from the reload comparison path or pass an explicit previous snapshot.
  - confidence: 0.93
- **[P1] Deep-redact every emitted diff value:** `src/gateway/config-diff.ts:76`
  - body: The helper redacts only by the immediate key before emitting whole `before`/`after` values for mixed-type and depth-capped changes. A nonsensitive parent object containing nested credentials can still be broadcast unredacted to ADMIN-scoped clients, so emitted values need recursive redaction.
  - confidence: 0.91
- **[P2] Register the config-diff protocol schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: Adding `src/gateway/protocol/schema/config-diff.ts` alone leaves the schema undiscoverable because the schema barrel and `ProtocolSchemas` map are not updated. Protocol/codegen consumers that rely on the exported schema map cannot validate the new event contract.
  - confidence: 0.86
