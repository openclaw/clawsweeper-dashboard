## Summary

Keep open. Current main does not implement the requested `config.changed` Gateway event, and the active PR still has blocking correctness and security issues in the proposed public config-diff payload path.

## What This Changes

The PR adds a Gateway `config.changed` event, config diff helper and schema file, ADMIN-scoped broadcast wiring from runtime config write listeners, event-list registration, and server-close cleanup for the new listener.

## Best Possible Solution

Emit `config.changed` from the reload comparison path, or extend the write notification with an explicit previous snapshot, then deep-redact every emitted value and register the protocol schema for generated consumers. Add focused tests for in-process writes and watcher-driven reloads, and include a user-facing changelog entry if the event remains public.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a normal config write notification contains only post-write source/runtime projections, while a nested secret under a non-sensitive parent can reach a whole-object `before` or `after` entry through the mixed-type or depth-cap branches.

## Solution Assessment

No. The ADMIN-scoped event direction is plausible, but this implementation hooks too low in the write-notification path and does not fully sanitize arbitrary diff values; the safer solution is to emit from the reload state transition that already owns previous-to-next comparison.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Compare against the previous config snapshot:** `src/gateway/server-runtime-subscriptions.ts:104`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both post-write projections, so this can suppress ordinary config edits or report source-vs-runtime noise instead of the advertised before/after config change. Emit from the reload path's previous/next compare state, or pass an explicit previous snapshot through the notification.
  - confidence: 0.93
- **[P1] Deep-redact emitted diff values:** `src/gateway/config-diff.ts:76`
  - body: The helper redacts only by immediate key before recursion, then emits whole `before`/`after` values for mixed-type and depth-cap changes. A non-sensitive parent object containing nested credentials can still be broadcast to admin clients without redaction.
  - confidence: 0.9
- **[P2] Register the new protocol schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: Adding this schema file alone leaves the new event payload out of the schema barrel and `ProtocolSchemas`, so protocol schema export/codegen consumers cannot discover the `config.changed` contract.
  - confidence: 0.86

## Security Review
