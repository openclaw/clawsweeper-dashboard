## Summary

Keep open. Current main does not implement `config.changed`, and this PR still has blocking correctness and security issues: it compares post-write source/runtime snapshots instead of previous/next config, can emit unredacted nested secrets, and leaves the new payload schema outside the protocol registry/codegen surface.

## What This Changes

The PR adds a `config.changed` Gateway broadcast, config diff helper and payload schema, ADMIN-scoped event gating, feature-list registration, and subscription cleanup for runtime config write notifications.

## Best Possible Solution

Revise the PR so `config.changed` is built from a true previous-to-next config comparison, deep-redacts every emitted value before broadcast, and wires the event payload through the protocol schema registry, docs, generated client surface, and focused tests while preserving ADMIN scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a config write notification contains only committed post-write `sourceConfig` and `runtimeConfig`, so a normal model-name edit can produce no event, while a nonsensitive parent object changing type with nested `apiKey` or `token` fields can reproduce the redaction leak.

## Solution Assessment

No. The listener and ADMIN-scoped broadcast seam are reasonable, but this implementation needs previous-state data, value-wide deep redaction, and protocol export/codegen wiring before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Compare against the previous config snapshot:** `src/gateway/server-runtime-subscriptions.ts:101`
  - body: `evt.sourceConfig` and `evt.runtimeConfig` are both committed post-write projections, so this emits source-vs-runtime noise or no event instead of the user's before/after config change. Capture or pass the previous compare snapshot before broadcasting.
  - confidence: 0.93
- **[P1] Deep-redact emitted diff values:** `src/gateway/config-diff.ts:76`
  - body: The helper redacts only by immediate key before recursion, then emits whole `before`/`after` values for mixed-type and depth-cap changes. A nonsensitive parent containing nested credentials can be broadcast unredacted to admin clients.
  - confidence: 0.9
- **[P2] Register the new payload schema:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: The PR adds a public event payload schema, but the schema is not exported from the protocol schema barrel or registered in `ProtocolSchemas`, so generated protocol artifacts and consumers cannot discover the new contract.
  - confidence: 0.86
