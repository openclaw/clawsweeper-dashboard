## Summary

Keep open. Current main has adjacent guardrails for spawn depth, active-child limits, optional tool-loop detection, and bounded orphan recovery, but it still lacks a native subagent failure circuit breaker with persisted failure counts, tripped/cooldown state, force/reset controls, notifications, or CLI/UI-visible breaker state; the comments add concrete production cost and reliability evidence.

## What This Changes

Request to add a configurable native circuit breaker for repeated failing subagent spawns, including cooldown, override, notification, and status visibility.

## Best Possible Solution

Add a core-owned subagent breaker around sessions_spawn and completion bookkeeping with persisted scoped state, failure classification, cooldown/half-open handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with generic loop detection and provider cooldowns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks the requested native per-subagent failure counter, cooldown, and breaker state.

## Solution Assessment

Unclear as a final design. A native core breaker is the right boundary for the requested reliability guarantee, but the exact config/defaults and failure taxonomy need maintainer product review before implementation.
