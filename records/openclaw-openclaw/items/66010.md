## Summary

Keep open. Current main has adjacent depth, active-child, tool-loop, and orphan-recovery guardrails, but it still does not implement the requested native sub-agent failure circuit breaker with persisted failure counts, tripped/cooldown state, force/reset controls, notifications, or CLI/UI-visible breaker state; the discussion adds concrete production cost and multi-agent reliability evidence.

## What This Changes

The issue requests a configurable native circuit breaker for repeated sub-agent spawn failures.

## Best Possible Solution

Add a core-owned subagent breaker around sessions_spawn and completion bookkeeping with persisted scoped health state, failure classification, cooldown/half-open handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with loop detection and provider cooldowns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection against current origin/main gives high-confidence proof that the requested native per-subagent failure counter, cooldown, and breaker state are absent.

## Solution Assessment

Unclear as a final design. A native breaker is the right boundary for the requested reliability guarantee, but the exact failure taxonomy, scope, defaults, cooldown behavior, and operator controls need maintainer product review before implementation.
