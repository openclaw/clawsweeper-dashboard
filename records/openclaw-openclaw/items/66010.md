## Summary

Keep open. Current main has adjacent protections for tool-call loops, idle-timeout cost runaway, spawn depth, active-child limits, run timeouts, and bounded orphan recovery, but it still does not implement the requested native subagent failure circuit breaker with persisted scoped failure counts, cooldown or half-open state, force/reset controls, notifications, and CLI/UI-visible breaker status.

## What This Changes

Request adds a native configurable circuit breaker for repeated subagent spawn failures with failure counters, cooldown behavior, override controls, notifications, and status visibility.

## Best Possible Solution

Design a core-owned subagent breaker around sessions_spawn and completion bookkeeping with persisted scoped state, failure classification, half-open cooldown handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with existing loop/provider/cost guardrails.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high-confidence proof that current main lacks native per-subagent failure counting, cooldown or half-open state, and force/reset/status controls.

## Solution Assessment

Unclear as a final design. A native core breaker is plausible for the requested reliability guarantee, but failure taxonomy, default thresholds, scope, cooldown/half-open behavior, operator controls, and overlap with provider/cost guardrails need maintainer product review first.
