## Summary

Keep open. Current main still lacks a native subagent cascade breaker: the spawn path has depth, concurrency, allowlist, sandbox, and timeout guardrails plus per-run outcome tracking, but no persisted failure counter, tripped/cooldown state, force/reset control, or UI/CLI-visible breaker state; the discussion adds concrete production reliability and cost evidence.

## What This Changes

Add a configurable native circuit breaker for repeated failing subagent spawns, with persisted failure counts/cooldowns, override/reset behavior, notifications, and status visibility.

## Best Possible Solution

Add a core-owned subagent breaker around sessions_spawn and the subagent registry with persisted scoped state, structured breaker errors, force/reset controls, config/docs/status/UI surfaces, and maintainer-approved failure classification semantics.

## Reproduction Assessment

Yes. A static reproduction path is high confidence: repeatedly invoke sessions_spawn or spawnSubagentDirect for a target that completes with error/timeout, and current main can keep allocating new child sessions because no aggregate failure lookup exists before spawn.

## Solution Assessment

Unclear as a final design, but the native core direction is appropriate. The exact defaults, failure taxonomy, breaker scope, half-open behavior, and operator controls need maintainer review before implementation.
