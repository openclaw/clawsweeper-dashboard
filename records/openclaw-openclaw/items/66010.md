## Summary

Keep open. Current main and v2026.6.10 have related retry and subagent guardrails, but they still do not provide the requested persisted subagent spawn circuit breaker with failure classification, cooldown or half-open state, override, notification, and CLI/UI state.

## What This Changes

Add a native configurable circuit breaker for subagent spawns that tracks repeated failures, blocks or cools down retries, supports explicit override and notification, and exposes breaker state.

## Best Possible Solution

Define one owner-approved subagent health model that persists per-agent/task failure receipts, classifies failures, gates sessions_spawn with structured breaker state, and surfaces recovery controls without duplicating provider or session-level breakers.

## Reproduction Assessment

Yes for source-level missing capability, but no live retry storm was reproduced in this read-only pass. Current main delegates sessions_spawn directly and records subagent outcomes without persisted breaker receipts or cooldown state.

## Solution Assessment

Unclear as a complete implementation plan. The direction fits core reliability, but maintainers still need to choose failure taxonomy, scope, persistence shape, defaults, half-open behavior, notification channel, and override semantics.
