## Summary

Keep open: current main has subagent depth, active-child, timeout, and adjacent loop guards, but it still lacks the requested persisted subagent cascade circuit breaker with failure classification, cooldown or half-open state, override, notification, and status surfacing.

## What This Changes

Request a native subagent spawn circuit breaker that tracks consecutive failures by agent or task signature, blocks further spawns after a threshold, cools down, supports override/reset, and surfaces state to operators.

## Best Possible Solution

Design one owner-approved subagent health path that persists per-agent/task failure receipts, classifies failures, gates sessions_spawn with structured breaker state, and surfaces recovery controls without duplicating existing provider/session/tool guards.
