## Summary

Keep open: current main has tool-loop and orphan-recovery guardrails that reduce some runaway patterns, but it still lacks the requested native sub-agent spawn breaker with persisted per-agent/task failure counts, cooldown, override, notification, and status surfaces.

## What This Changes

Issue #66010 requests a native, configurable circuit breaker for repeated sub-agent spawn failures in multi-agent deployments.

## Best Possible Solution

Define a bounded native sub-agent breaker contract at the spawn/registry boundary, backed by canonical SQLite state, with decided failure classes, scope, cooldown/half-open behavior, manual reset/override, notifications, and docs/status surfaces.

## Reproduction Assessment

Yes, source-reproducible: current main exposes spawn limits and task terminal state but no per-subagent/task failure counter or breaker lookup in the spawn path; I did not run a live costly cascade.

## Solution Assessment

Unclear: a native breaker is plausible, but maintainers still need to decide scope, failure classification, persistence, half-open/cooldown semantics, provider-breaker separation, and status UX.
