## Summary

Keep open. Current main has adjacent guardrails for spawn depth, active child limits, optional tool-loop detection, and bounded orphan recovery, but it still lacks the requested native persisted subagent failure circuit breaker with cooldown, reset/override controls, notifications, and CLI/UI-visible state; the comments add concrete production reliability and cost evidence.

## What This Changes

Add a native configurable subagent spawn circuit breaker with failure counters, cooldown/reset/override behavior, notifications, and status surfaces.

## Best Possible Solution

Design a core subagent breaker around sessions_spawn and completion bookkeeping with persisted scoped state, failure classification, cooldown/half-open handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with existing loop detection and provider cooldowns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks a native per-subagent failure counter, cooldown, tripped state, and reset/override surface.

## Solution Assessment

Unclear as a final design. A native core breaker is the right boundary for the requested reliability guarantee, but failure taxonomy, scope, defaults, cooldown/half-open behavior, and operator controls need maintainer product/API review first.
