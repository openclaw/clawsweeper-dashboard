## Summary

Keep open. Current main and the latest release have useful adjacent guardrails for tool loops, unknown tools, subagent run timeouts, active-child limits, and orphan recovery, but they do not implement the requested native sub-agent failure breaker with persistent consecutive-failure tracking, cooldown/override state, notification, or Control UI/CLI status.

## What This Changes

The issue requests a native sub-agent circuit breaker that tracks repeated failures, applies cooldown and override behavior, and surfaces breaker state to operators.

## Best Possible Solution

Define one durable subagent reliability contract, or explicitly fold this into https://github.com/openclaw/openclaw/issues/62615, with failure classification, scope, cooldown/probe behavior, operator override, and status surfacing decided before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction path; this is a feature request. Source inspection gives high-confidence proof that current main and v2026.6.6 lack the requested subagent circuitBreaker config, spawn-time failure gate, and durable breaker state.

## Solution Assessment

Unclear until maintainers choose the product boundary. A sub-agent breaker is directionally plausible, but the maintainable path should integrate with existing subagent run records, task registry, tool-loop detection, provider cooldowns, and the broader session-health breaker instead of adding an isolated guard.
