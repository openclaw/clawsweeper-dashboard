## Summary

Keep open: current main has sub-agent depth, active-child, timeout, task-status, and other loop guardrails, but it still lacks the requested native sub-agent spawn breaker with persisted per-agent/task failure counts, cooldown or half-open state, override, notification, and status surfaces.

## What This Changes

Request adds a native sub-agent spawn circuit breaker with per-agent/task failure tracking, configurable thresholds and cooldowns, override, notifications, and UI/CLI status visibility.

## Best Possible Solution

Design a core-owned breaker at the sessions_spawn and subagent registry boundary, backed by canonical durable state and explicit config/status contracts, with tests for default compatibility, failure classification, cooldown reset, override, and UI/CLI visibility.

## Reproduction Assessment

Yes for the source-level gap: current main's spawn, config, registry, and status paths lack any per-subagent/task breaker counter or trip check. I did not run a live costly cascade because this is a feature request and the missing behavior is visible in source.

## Solution Assessment

Unclear: a native breaker is plausible and aligned with reliability goals, but the requested design crosses config, durable state, UI/CLI, and provider-failure policy. Maintainers should settle the contract before implementation.
