## Summary

Keep open. Current main has adjacent loop, timeout, depth, active-child, and orphan-recovery guardrails, but it still lacks the requested native subagent failure breaker with scoped failure counts, cooldown or half-open state, force/reset controls, notification, and CLI/UI-visible status.

## What This Changes

Request a configurable native circuit breaker for repeated subagent spawn or run failures in multi-agent deployments.

## Best Possible Solution

Design a core-owned breaker around sessions_spawn and completion bookkeeping with persisted scoped state, failure classification, cooldown/half-open handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with existing loop and provider guardrails.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high-confidence proof that current main lacks native per-subagent failure counting, cooldown or half-open state, force/reset controls, and status surfaces.

## Solution Assessment

Unclear as a final design. A native core breaker is plausible for the requested reliability guarantee, but failure taxonomy, scope, defaults, operator controls, and overlap with provider/cost guardrails need maintainer product review first.
