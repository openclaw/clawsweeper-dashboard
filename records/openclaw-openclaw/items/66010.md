## Summary

Keep open. Current main has adjacent guardrails for tool-loop detection, spawn depth, active-child limits, run timeouts, and bounded orphan recovery, but it still lacks the requested native subagent failure circuit breaker with persisted failure counts, cooldown/half-open state, force/reset controls, notifications, and CLI/UI-visible breaker status; the comments add concrete production reliability and cost evidence.

## What This Changes

Request to add a configurable native circuit breaker for repeated subagent spawn/run failures, including cooldown, override, notification, and status surfaces.

## Best Possible Solution

Add a core-owned subagent breaker around sessions_spawn and completion bookkeeping with persisted scoped state, failure classification, cooldown/half-open handling, explicit force/reset controls, notifications, docs, and status surfaces coordinated with loop detection and provider cooldowns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks the requested native per-subagent failure counter, cooldown, and breaker state.

## Solution Assessment

Unclear as a final design. A native core breaker is a plausible boundary for the requested reliability guarantee, but failure taxonomy, default thresholds, scope, half-open behavior, and operator controls need maintainer product review first.
