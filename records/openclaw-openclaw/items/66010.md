## Summary

Keep open. Current main still lacks a native subagent cascade circuit breaker: the spawn path has per-call guardrails and per-run tracking, but no persisted failure counter, tripped/cooldown state, force/reset controls, or UI/CLI-visible breaker state; the discussion adds concrete production reliability and cost evidence.

## What This Changes

The issue requests a configurable native circuit breaker for repeated subagent spawn failures, with cooldowns, override/reset behavior, notifications, and status visibility.

## Best Possible Solution

Add a core-owned subagent breaker around sessions_spawn and the subagent registry with persisted scoped state, structured breaker errors, explicit force/reset controls, config/docs/status surfaces, and maintainer-approved failure classification semantics.

## Reproduction Assessment

Yes. A static reproduction path is high confidence: repeated failing calls through spawnSubagentDirect can create new child runs because current main has no aggregate failure state or pre-spawn breaker check.

## Solution Assessment

Unclear as a final design, but the core direction is right. A native breaker is the maintainable boundary, while the exact config/defaults and failure taxonomy need maintainer review before implementation.
