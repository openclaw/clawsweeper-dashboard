## Summary

Keep open for maintainer review: the patch conflicts with current main’s documented runtime-policy direction by reintroducing session-level runtime pins, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR persists registered harness runtime overrides from `/model --runtime`, clears stale `agentHarnessId` values, and makes `agentRuntimeOverride` win over embedded harness pins in agent command execution.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy unless maintainers explicitly approve session runtime overrides with matching docs, migrations, stale-state handling, and proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests show session runtime overrides are intentionally ignored, and the PR does not provide a real failing current-main run.

## Solution Assessment

No. The proposed path resurrects session-level runtime pins, while current main’s maintainable direction is provider/model runtime policy; a safer fix would extend that policy path or get explicit maintainer approval for a session override design.
