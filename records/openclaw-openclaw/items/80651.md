## Summary

Keep open for maintainer review: this draft PR changes runtime-selection policy in the opposite direction from current docs and tests by restoring session-level runtime pins, and it still lacks real behavior proof.

## What This Changes

The PR lets `/model ... --runtime <registered harness>` persist session runtime overrides, clears stale harness pins on runtime changes, and passes non-CLI session overrides into embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a session override design with docs, doctor or stale-state handling, and real proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests intentionally ignore session-level runtime pins. The PR body provides targeted test output, not a failing current-main real session.

## Solution Assessment

No. The proposed path makes session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless maintainers choose a documented session override model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Keep session runtime pins out of harness selection:** `src/agents/command/attempt-execution.ts:526`
  - body: The added branch returns a persisted `agentRuntimeOverride` as `agentHarnessId` for non-CLI runs. Please keep this path on provider/model policy, or add an owner-approved design with docs, stale-state handling, and real proof.
