## Summary

Keep open: the PR is draft, lacks real behavior proof, and conflicts with current main's documented provider/model-scoped runtime policy by reintroducing session runtime pins into harness selection.

## What This Changes

This PR lets `/model ... --runtime <registered harness>` persist a custom harness runtime, clears stale session harness pins, and makes agent execution prefer `agentRuntimeOverride` over `agentHarnessId`.

## Best Possible Solution

Keep runtime selection provider/model-scoped; if session-level runtime overrides are desired, define and document that product contract with migration/doctor handling and real custom-harness proof.

## Reproduction Assessment

Not applicable as a strict current-main bug reproduction: current main intentionally ignores session runtime pins and routes through provider/model policy. The PR body provides targeted Vitest output, not a failing current-main real session.

## Solution Assessment

No. The proposed path stores and uses session-level runtime pins despite current docs and tests; provider/model runtime policy, or an explicitly approved session-override design, is the safer maintainable direction.
