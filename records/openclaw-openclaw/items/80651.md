## Summary

Keep this PR open, but the current patch is not merge-ready: it reintroduces session-level runtime pins that current main intentionally ignores, and the contributor has not provided real behavior proof.

## What This Changes

The PR changes model directive persistence and agent command execution so registered harness runtime overrides can be stored and can take precedence over stale embedded harness pins.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy unless maintainers approve a session-level runtime override design with docs, stale-state handling, migration or doctor coverage, and real proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests define the opposite behavior by ignoring session runtime pins. The PR body provides targeted Vitest output, not a failing current-main session.

## Solution Assessment

No. The proposed implementation persists and uses session-level runtime pins; the safer maintainable path is provider/model runtime policy, or an explicitly approved session override design.
