## Summary

Keep open: current main deliberately routes runtime selection through provider/model policy, while this draft PR makes persisted session runtime state authoritative again and has no real behavior proof.

## What This Changes

The PR persists registered `/model --runtime` harness ids, clears stale `agentHarnessId` values, and lets session `agentRuntimeOverride` win over an existing session harness pin.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a documented session override design with doctor/stale-state handling and real setup proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests intentionally ignore session-level runtime pins. The PR body provides targeted tests, not a failing real current-main session.

## Solution Assessment

No. As submitted, the PR makes persisted session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless maintainers choose a new documented session override model.
