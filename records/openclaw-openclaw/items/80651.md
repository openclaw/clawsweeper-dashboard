## Summary

Keep open for human review, but the current diff should not merge: it reintroduces session-level runtime pins that current main docs and tests intentionally ignore, and the PR still lacks real behavior proof.

## What This Changes

The PR changes `/model --runtime` persistence and agent execution so registered plugin harness ids can be saved as session runtime overrides and override stale session harness pins.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a documented session override design with stale-state handling, doctor behavior, and real proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main intentionally ignores session-level runtime pins and has docs/tests for that contract. The PR body provides targeted test output, not a failing current-main real session.

## Solution Assessment

No. The proposed path makes persisted session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless maintainers choose a documented session override model.
