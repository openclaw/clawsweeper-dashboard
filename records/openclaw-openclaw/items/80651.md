## Summary

Keep this PR open for maintainer review rather than cleanup closure: it is a recent draft external PR that conflicts with current main’s provider/model runtime-policy contract and still lacks real behavior proof.

## What This Changes

The PR lets `/model ... --runtime <registered harness>` persist a session runtime override, clears stale `agentHarnessId`, and passes session `agentRuntimeOverride` into embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a documented session-override design with stale-state repair, doctor guidance, and real runtime proof.

## Reproduction Assessment

Not applicable as a strict current-main bug reproduction: current main source, docs, and tests intentionally ignore session runtime pins. The PR body provides targeted test output, not a failing real OpenClaw session or custom harness run.

## Solution Assessment

No. The proposed path makes session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless maintainers choose a documented session override model.
