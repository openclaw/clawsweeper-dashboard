## Summary

Keep this PR open for maintainer review, but it should not merge as-is: the patch reintroduces session-level runtime pins that current main deliberately ignores in favor of provider/model runtime policy, and the external PR still lacks real behavior proof.

## What This Changes

The PR changes model directive persistence and agent command harness selection so a registered session runtime override can replace a stale pinned harness id.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy unless maintainers approve a documented session override design with stale-state repair, doctor coverage, regression tests, and real custom-harness proof.

## Solution Assessment

No. The proposed path persists and reuses session runtime pins, while the current maintainable contract is provider/model runtime policy unless maintainers explicitly approve a new session override design.
