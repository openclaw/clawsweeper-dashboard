## Summary

Keep open for maintainer policy review, but the current patch is not correct: current main intentionally ignores session runtime pins in favor of provider/model runtime policy, while this PR reintroduces session-level harness precedence without real behavior proof.

## What This Changes

The PR persists registered `/model --runtime` harness ids in session state, clears stale `agentHarnessId` pins, and lets session `agentRuntimeOverride` override embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers approve a documented session-override design with stale-state repair, docs, tests, and real runtime proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main docs and tests intentionally ignore session `agentHarnessId`/`agentRuntimeOverride` and use provider/model runtime policy. The PR body supplies targeted Vitest output, not a failing real current-main session.

## Solution Assessment

No. Returning session `agentRuntimeOverride` as `agentHarnessId` reintroduces stale session-pin authority; provider/model policy, or an owner-approved session override design with migrations and docs, is the safer maintainable path.
