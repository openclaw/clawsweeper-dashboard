## Summary

Keep open for maintainer review, but this branch should not merge as-is: current main deliberately ignores session runtime pins in favor of provider/model runtime policy, and the PR has only targeted test output rather than real custom-harness session proof.

## What This Changes

The PR makes `/model ... --runtime <registered harness>` persist a session runtime override, clear stale `agentHarnessId` pins on runtime changes, and prefer that override in non-CLI agent execution.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a documented session override design with stale-state repair, diagnostics, and real custom-harness proof.

## Reproduction Assessment

No. Current main source, docs, and tests intentionally ignore session-level runtime pins, and the PR body provides targeted test output rather than a failing current-main real session.

## Solution Assessment

No. The proposed path makes session runtime state authoritative again; the safer maintainable direction is provider/model runtime policy unless maintainers choose a documented session override model.
