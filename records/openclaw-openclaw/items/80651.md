## Summary

Keep open for maintainer runtime-policy review: the PR is a recent draft with a blocking correctness issue because it changes behavior that current main documents and tests as intentionally ignored, and it lacks real behavior proof.

## What This Changes

The PR persists registered harness runtime overrides from `/model --runtime`, clears stale harness pins, and lets a stored runtime override supersede a stored `agentHarnessId` during embedded agent execution.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy unless maintainers explicitly approve a session-level runtime override design with docs, stale-state cleanup, migration or doctor coverage, and real session proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests define the opposite behavior by ignoring session runtime pins. The PR body provides targeted test output, not a failing current-main real-session reproduction.

## Solution Assessment

No. The proposed implementation persists and uses session-level runtime pins, while the safer maintainable path is provider/model runtime policy or an explicitly approved session override design.
