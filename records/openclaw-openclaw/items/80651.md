## Summary

Keep open: current main intentionally routes runtimes through provider/model policy and ignores session-level runtime pins, while this PR reintroduces session-persisted runtime selection and lacks real behavior proof.

## What This Changes

The PR lets registered harness runtime overrides persist from `/model --runtime`, clears stale `agentHarnessId` pins, and passes session `agentRuntimeOverride` into embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers approve and document a session override design with stale-state handling, doctor behavior, and real proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests intentionally ignore session `agentHarnessId` and `agentRuntimeOverride` for routing. The PR body provides targeted test output, not a failing current-main real session.

## Solution Assessment

No. The branch makes persisted session runtime state authoritative again; provider/model runtime policy is the narrower maintainable path unless maintainers explicitly choose a documented session override model.
