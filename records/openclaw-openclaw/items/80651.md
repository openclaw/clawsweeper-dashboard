## Summary

Keep open for maintainer handling: the draft PR conflicts with current runtime-policy docs and tests that intentionally ignore session-level runtime pins, and it still lacks contributor real behavior proof from a real custom harness run.

## What This Changes

The PR persists registered harness runtimes from `/model --runtime`, clears stale `agentHarnessId`, and lets non-CLI `agentRuntimeOverride` override embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers approve a documented session override design with doctor or stale-state handling and real custom-harness proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main intentionally ignores session-level runtime pins by source, docs, and tests. The PR body provides targeted test output, not a failing real current-main session.

## Solution Assessment

No. The proposed path makes persisted session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless maintainers choose a documented session override model.
