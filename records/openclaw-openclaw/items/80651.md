## Summary

Keep this PR open but do not merge it as-is: the branch reverses current provider/model-scoped runtime policy, the PR is draft/conflicting, and it still lacks real behavior proof from a real session or custom harness run.

## What This Changes

The branch persists registered plugin harness runtime overrides from `/model --runtime`, clears stale `agentHarnessId`, and lets session `agentRuntimeOverride` drive embedded harness selection.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy; if session-level runtime override is desired, design it explicitly with docs, doctor or migration handling for stale state, and real setup proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main defines the opposite behavior by ignoring session runtime pins. The PR body provides targeted Vitest output, not a failing current-main session or real custom-harness run.

## Solution Assessment

No. The patch implements session-level runtime pins, while current docs and tests route through provider/model policy; the safer path is an explicitly approved runtime-policy design if this capability is desired.
