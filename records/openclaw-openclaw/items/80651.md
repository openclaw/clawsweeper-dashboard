## Summary

Keep open, but the PR should not merge as-is: current main deliberately ignores session-level runtime pins in favor of provider/model runtime policy, and this branch conflicts with that contract while still lacking real behavior proof.

## What This Changes

The branch makes `/model --runtime <registered harness>` persist `agentRuntimeOverride`, clears stale `agentHarnessId`, and lets that session runtime override drive embedded harness selection.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly approve a documented session-override design with stale-state handling and real proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a strict bug reproduction: current main defines session runtime pins as ignored legacy state, and the PR body only provides targeted test output for the proposed behavior.

## Solution Assessment

Is this the best way to solve the issue? No. The proposed path makes session runtime state authoritative again; the safer maintainable path is provider/model runtime policy unless owners choose a documented session override model.
