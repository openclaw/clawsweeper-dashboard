## Summary

Keep open: the PR changes runtime-selection policy in a way that conflicts with current main’s documented provider/model policy, has a blocking correctness finding, and still lacks real behavior proof from a real custom-harness session.

## What This Changes

The PR persists registered plugin runtime overrides from `/model --runtime`, clears stale harness pins, and lets session `agentRuntimeOverride` take precedence over stored `agentHarnessId` in agent command execution.

## Best Possible Solution

Keep runtime selection on provider/model-scoped policy unless maintainers approve a session-level runtime override design with matching docs, stale-state handling, migration or doctor coverage, and real behavior proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current main source, docs, and tests define session runtime pins as ignored. The PR body provides targeted test output for a proposed behavior change, not a failing current-main behavior under the documented contract.

## Solution Assessment

No. The proposed implementation persists and uses session-level runtime pins, while the safer maintainable path is provider/model runtime policy unless maintainers explicitly approve a new session override contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep session runtime pins out of harness selection:** `src/agents/command/attempt-execution.ts:521-527`
