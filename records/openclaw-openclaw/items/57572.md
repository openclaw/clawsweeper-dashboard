## Summary

Keep this draft PR open: the restart-mode capability is not on current main, but the proposed implementation still has compile/schema/runtime blockers, lacks real behavior proof, and needs maintainer agreement on the gateway recovery policy.

## What This Changes

Adds global and per-channel health-monitor restart-mode config and threads it through gateway startup/reload so stale-channel recovery can skip the restart-attempt reset.

## Best Possible Solution

If maintainers accept the capability, implement it with strict schema/help/docs/generated metadata, focused tests, and a real manager or provider seam that preserves restart state correctly or performs channel-owned reconnect.

## Reproduction Assessment

Partially yes. Source inspection shows current main lacks the restart-mode feature and confirms the PR blockers, but the live Anthropic prompt-cache cost spike was not independently reproduced.

## Solution Assessment

No. The PR is not the best fix yet because the config is schema-rejected, the helper does not compile as written, and the graceful path still goes through a manager start path that clears restart attempts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the injected runtime config in the helper:** `src/gateway/server-channels.ts:238`
  - body: The new helper calls loadConfig(), but server-channels.ts does not import that symbol and already has getRuntimeConfig() injected. This fails typecheck and would also bypass the config source used by reloads and tests.
  - confidence: 0.95
- **[P2] Add strict schema support for restart-mode config:** `src/config/types.gateway.ts:464`
  - body: Adding channelHealthRestartMode and healthMonitor.restartMode only to TypeScript types leaves the strict zod schemas rejecting those keys before the health monitor can read them. Add the gateway enum, shared channel health-monitor schema field, and config validation coverage.
  - confidence: 0.92
- **[P2] Preserve attempts through the manager start path:** `src/gateway/channel-health-monitor.ts:186-192`
  - body: The graceful branch skips resetRestartAttempts(), but channelManager.startChannel() still calls startChannelInternal() without preserveRestartAttempts, which deletes the counter. Add a manager seam that passes the preserve option or implement a real channel-owned reconnect hook.
  - confidence: 0.9
- **[P2] Update ChannelManager test doubles for the new method:** `src/gateway/server-channels.ts:151-159`
  - body: Making getChannelHealthMonitorConfig a required ChannelManager member means existing typed test doubles that construct ChannelManager directly no longer satisfy the interface. Update the affected mocks or make the new seam optional with a safe default.
  - confidence: 0.86
