## Summary

Keep open. Current main still lacks the requested restart-mode config or lightweight reconnect path, and the PR is not ready to merge because the proposed patch has concrete build/config/runtime blockers.

## What This Changes

The PR adds a gateway and per-channel health-monitor restart-mode setting and threads it into health-monitor startup/reload so stale-channel recovery can skip resetting restart attempts.

## Best Possible Solution

Keep the PR open for maintainer review and rework. If this capability is accepted, it should add schema/docs/generated metadata/tests plus a real manager/provider seam that preserves restart state or performs a channel-owned reconnect; otherwise maintainers should explicitly close or reframe it as superseded by #69833 and the existing per-channel health-monitor opt-out.

## Reproduction Assessment

Partially yes. The missing current-main feature and the PR blockers are reproducible by source inspection and focused config/type tests; the live Anthropic prompt-cache cost spike is not independently reproduced in the PR data.

## Solution Assessment

No. The proposed approach is not yet the best fix because the config keys are not accepted by strict schemas, the new helper does not compile as written, and the graceful path still calls the public startChannel path that resets restart attempts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the injected runtime config in the new helper:** `src/gateway/server-channels.ts:238`
  - body: The new helper calls `loadConfig()`, but `server-channels.ts` does not import that symbol, so this patch fails typecheck/build. Use the existing `getRuntimeConfig()` closure here; it also keeps reloads and tests on the same injected config source.
  - confidence: 0.95
- **[P2] Wire restart mode into strict config validation:** `src/config/types.gateway.ts:464`
  - body: Adding `channelHealthRestartMode` and `healthMonitor.restartMode` only to TypeScript types leaves the strict zod schemas rejecting those keys before the health monitor can read them. The gateway schema and shared channel health-monitor schema need matching enum fields and tests.
  - confidence: 0.92
- **[P2] Preserve attempts when graceful mode starts the channel:** `src/gateway/channel-health-monitor.ts:186-192`
  - body: Skipping `resetRestartAttempts()` here is not enough because `channelManager.startChannel()` still calls `startChannelInternal()` without `preserveRestartAttempts`, which deletes the counter. The graceful path needs a manager seam that actually passes the preserve option or a real reconnect hook.
  - confidence: 0.9
- **[P2] Update ChannelManager test doubles for the new method:** `src/gateway/server-channels.ts:157-162`
  - body: Making `getChannelHealthMonitorConfig` a required `ChannelManager` member means existing typed test doubles that construct `ChannelManager` directly no longer satisfy the interface. Update the affected mocks or make the new seam optional with a safe default.
  - confidence: 0.86
