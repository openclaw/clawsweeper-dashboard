## Summary

This PR should stay open for maintainer/product review rather than cleanup: current main still lacks a restart-mode capability, but the draft branch has correctness blockers and no after-fix real behavior proof.

## What This Changes

Adds global and per-channel health-monitor restart-mode config and threads it through gateway startup/reload so stale-channel recovery can skip the restart-attempt reset.

## Best Possible Solution

If maintainers accept the capability, implement it with strict schema/help/docs/generated metadata, focused tests, and a real manager or provider seam that preserves restart state correctly or performs channel-owned reconnect.

## Reproduction Assessment

Partially yes. Source inspection shows current main lacks restart-mode support and follows the stop/reset/start path, but the live Anthropic prompt-cache cost spike was not independently reproduced.

## Solution Assessment

No. The proposed implementation is not the best fix yet because the config keys are schema-rejected, the helper does not compile as written, and the graceful path still goes through a manager start path that clears restart attempts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the injected runtime config in the helper:** `src/gateway/server-channels.ts:238`
  - body: The new helper calls loadConfig(), but server-channels.ts does not import that symbol and createChannelManager already receives getRuntimeConfig. This fails typecheck and would bypass the runtime/reload config source used by the gateway.
  - confidence: 0.95
- **[P2] Add strict schema support for restart-mode config:** `src/config/types.gateway.ts:464`
  - body: Adding channelHealthRestartMode and healthMonitor.restartMode only to TypeScript types leaves the strict zod schemas rejecting those keys before the health monitor can read them. Add the gateway enum, channel health schema field, and validation coverage.
  - confidence: 0.92
- **[P2] Preserve attempts through the manager start path:** `src/gateway/channel-health-monitor.ts:186-193`
  - body: The graceful branch skips resetRestartAttempts(), but then calls channelManager.startChannel(), whose manager path clears restartAttempts unless preserveRestartAttempts is passed. Add a manager seam or implement a channel-owned reconnect hook.
  - confidence: 0.9
- **[P2] Update ChannelManager test doubles for the new method:** `src/gateway/server-channels.ts:151-159`
  - body: Making getChannelHealthMonitorConfig a required ChannelManager member means existing typed test doubles that construct ChannelManager directly no longer satisfy the interface. Update the affected mocks or make the seam optional with a safe default.
  - confidence: 0.86
