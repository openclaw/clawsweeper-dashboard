## Summary

This PR should stay open for maintainer review: current main still lacks a restart-mode feature, and the branch is a draft external feature/config change with blocking correctness gaps plus missing real behavior proof.

## What This Changes

The branch adds global and per-channel channel-health restart-mode config and threads it through gateway startup/reload so stale-channel recovery can skip a restart-attempt reset.

## Best Possible Solution

If maintainers accept the capability, implement it through strict config/schema/docs support plus a real manager or channel reconnect seam that preserves the intended runtime state, with focused regression tests and redacted live proof of stale-socket recovery behavior.

## Reproduction Assessment

Partially yes. Source inspection gives a clear current-main path for the existing stop/reset/start health-monitor behavior, but the live Anthropic prompt-cache cost spike and after-fix improvement are not reproduced with high confidence.

## Solution Assessment

No. The proposed branch is not the best mergeable solution yet because the new config keys are not accepted by strict schemas, and the graceful path still routes through a manager start path that clears restart attempts.
