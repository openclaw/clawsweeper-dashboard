## Summary

Keep open. Current main still lacks restart-mode support and the linked feature request is routed to this PR, but this draft external feature/config PR is not ready to merge because it lacks real behavior proof and has concrete correctness blockers in config validation, restart-state preservation, and typed test doubles.

## What This Changes

The branch adds global and per-channel channel-health restart-mode config and threads it through gateway startup/reload so stale-channel recovery can skip a restart-attempt reset.

## Best Possible Solution

If maintainers accept the capability, implement a documented strict config/schema path plus a manager or provider reconnect seam that actually preserves restart state or performs channel-owned reconnect, with focused tests and redacted real behavior proof.

## Reproduction Assessment

Partially yes. Source inspection shows current main lacks restart-mode support and follows the stop/reset/start path, but the live Anthropic prompt-cache cost spike was not independently reproduced.

## Solution Assessment

No. The proposed diff is not the best fix yet because the new config keys are not accepted by strict schemas, graceful mode still goes through a start path that clears attempts, and required ChannelManager test doubles are not updated.
