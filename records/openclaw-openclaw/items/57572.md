## Summary

Keep open: current main still lacks this restart-mode surface, but the draft external feature/config PR is not mergeable as written and still lacks after-fix real behavior proof.

## What This Changes

The PR adds global and per-channel channel-health restart-mode config and threads it through gateway startup/reload so stale-channel recovery can skip the restart-attempt reset.

## Best Possible Solution

If maintainers accept the capability, implement a documented schema-backed restart/reconnect contract that preserves intended runtime state, add focused coverage, and require redacted live stale-socket proof before merge.

## Reproduction Assessment

Partially yes. Source inspection shows current main's health monitor uses stop/reset/start for unhealthy channels, but the reported Anthropic prompt-cache cost spike and after-fix improvement have not been reproduced in a live setup.

## Solution Assessment

No. The branch is not the best mergeable solution yet because the new config keys are absent from strict schemas, and the graceful path still calls a public startChannel path that clears restart attempts.
