- **[P1] Pass the recovery stop flag:** `src/gateway/channel-health-monitor.ts:185`
  - body: This branch stops a running channel with `stopChannel(channelId, accountId)`. On current main, `stopChannel` defaults to a manual stop, records the account in `manuallyStopped`, and the following `startChannel` returns without restarting it, so stale-socket recovery can leave delivery down after rebase. Keep the health-monitor recovery call non-manual and cover it with a focused test.
  - confidence: 0.94
- **[P2] Add restart-mode schema support:** `src/config/types.gateway.ts:464`
  - body: Adding `channelHealthRestartMode` and per-channel `healthMonitor.restartMode` only to TypeScript types leaves the strict config schemas unaware of the new keys. User config with the documented option will be rejected before the monitor can read it; add gateway/channel Zod schema, metadata/help/docs, and validation coverage.
  - confidence: 0.93
- **[P2] Preserve attempts through the start path:** `src/gateway/channel-health-monitor.ts:193`
  - body: The graceful branch skips `resetRestartAttempts()`, but then calls the public `startChannel()` wrapper. Current main's public start path does not pass `preserveRestartAttempts`, so it still clears attempt state; expose a manager option or real reconnect seam and test the graceful mode.
  - confidence: 0.91
- **[P2] Wire the current runtime-service startup path:** `src/gateway/server.impl.ts:1144-1146`
  - body: Current main starts the health monitor through `startGatewayRuntimeServices()` and `startGatewayChannelHealthMonitor()`, while this branch only threads the option through older `server.impl.ts` startup wiring. Rebase and pass the option through the active runtime-service helper so the global config is actually applied.
  - confidence: 0.9
- **[P2] Restart the monitor when the mode changes:** `src/gateway/server-reload-handlers.ts:105-107`
  - body: The reload handler only forwards `defaultRestartMode` if the health monitor is already being recreated, but the reload plan has no rule for `gateway.channelHealthRestartMode`. Changing this key would not take effect until a full restart; add the reload-plan rule and coverage.
  - confidence: 0.88
