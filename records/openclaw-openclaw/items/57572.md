## Summary

Keep open: current main does not implement this restart-mode option, but the draft branch is not correct enough to merge because it has compile, config-schema, and runtime-state blockers. The related #69833 reduced false stale-socket restarts, but it does not supersede this configurable restart-mode proposal.

## What This Changes

The branch adds global and per-channel health-monitor restart-mode config and threads it into gateway startup/reload so stale-channel recovery can skip resetting restart attempts.

## Best Possible Solution

If maintainers accept the capability, implement it with strict schema/docs/generated metadata, focused tests, changelog coverage, and a real manager/provider seam that either preserves restart state correctly or performs channel-owned reconnect.

## Reproduction Assessment

Partially yes. The missing current-main feature and the PR blockers are reproducible by source inspection; the live Anthropic prompt-cache cost spike is not independently reproduced here.

## Solution Assessment

No. The proposed implementation is not the best fix yet because the new config is schema-rejected, the new helper does not compile as written, and the graceful path still resets restart attempts through startChannel.

  - confidence: 0.92
