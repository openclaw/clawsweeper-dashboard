## What This Changes

The PR updates `.github/ISSUE_TEMPLATE/bug_report.yml` to make the existing rendered `logs` textarea logs-only and add a separate unrendered `screenshots_evidence` textarea.

## Best Possible Solution

Land this focused issue-template fix, or switch the new screenshots/evidence control to `type: upload` if maintainers want an explicit file picker, while keeping shell rendering limited to logs.

## Reproduction Assessment

Yes. Current main reproduces the configuration problem directly: the bug template asks for screenshots and recordings in a textarea that also has `render: shell`, and GitHub documents that `render` prevents file-attachment expansion for textareas. ## Solution Assessment

Yes. Splitting logs from screenshots/evidence is the narrowest maintainable correction because it preserves shell highlighting for logs while moving attachments to a field without `render`; `type: upload` is a maintainer preference, not a required correctness change.
