## Summary

## What This Changes

The PR extracts `deriveSessionTitle` into a dedicated Gateway module/test file and changes title precedence so explicit session labels beat transcript-derived text while origin labels remain a later fallback.

## Best Possible Solution

Preserve the new explicit-label precedence, keep origin labels behind transcript-derived titles, normalize every persisted title candidate with the shared string normalizer, and resolve the main-branch conflict before merging.

## Reproduction Assessment

Yes. Source inspection shows current main persists session labels but `deriveSessionTitle` ignores `entry.label`, and the patch defect is also source-reproducible because JSON-loaded title fields are not fully type-normalized before the new `.trim()` calls.

## Solution Assessment

No, not as currently written. The precedence change is the right narrow direction, but the implementation must keep current-main normalization semantics and resolve the merge conflict before it is the best fix.
