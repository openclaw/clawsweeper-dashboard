## Summary

Keep open. This PR addresses a real Gateway/TUI title precedence bug and has sufficient terminal proof, but the current diff still introduces a regression by calling `.trim()` directly on persisted session-store title fields that current main safely normalizes.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, and adds tests plus precedence for explicit labels before transcript text and origin labels after transcript text.

## Best Possible Solution

Merge the narrow extraction and explicit-label precedence fix after restoring `normalizeOptionalString` handling for persisted title fields and adding regression coverage; keep the broader inbound-metadata and external-displayName work with https://github.com/openclaw/openclaw/issues/39722 and https://github.com/openclaw/openclaw/pull/39790.

## Reproduction Assessment

Yes, by source inspection: current main passes transcript `firstUserMessage` into `deriveSessionTitle`, the TUI displays `derivedTitle` first, and the helper ignores explicit `entry.label`, so a labeled heartbeat-style transcript can surface transcript text.

## Solution Assessment

No, not as submitted. The precedence order is the right narrow fix, but the extracted helper should preserve current-main unknown-field normalization before merge.
