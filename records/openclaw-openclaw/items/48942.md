## Summary

Keep open: current main and the latest release still omit explicit session labels from derived titles, and the PR is a useful focused fix, but it still drops current normalization semantics and the branch is dirty against main.

## What This Changes

The PR extracts Gateway `deriveSessionTitle` into a dedicated module/test file and adds precedence for explicit session labels ahead of transcript text, with origin labels as a later fallback.

## Best Possible Solution

Preserve explicit-label precedence, keep origin labels behind transcript-derived titles, normalize every persisted title candidate with the existing string normalizer, and merge only after the branch is clean with focused tests.

## Reproduction Assessment

Yes. Source inspection shows current main persists session labels but `deriveSessionTitle` ignores `entry.label`; the patch defect is also source-reproducible because JSON-loaded non-string title fields can reach the new `.trim()` calls.

## Solution Assessment

No, not as currently written. The precedence change is the right narrow direction, but the implementation needs to keep current-main normalization semantics and resolve the dirty branch before it is the best fix.
