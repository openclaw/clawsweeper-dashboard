## Summary

Keep open: current main still lacks explicit session-label precedence, so the PR is useful, but the latest patch would make malformed persisted title fields throw during session listing and omits required release-note coverage.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, moves the related tests, and changes title precedence so explicit session labels outrank transcript-derived titles while origin labels remain a fallback.

## Best Possible Solution

Merge a corrected version that keeps the extraction, compatibility re-export, explicit label precedence, and origin-label fallback ordering while defensively normalizing persisted title candidates and adding a changelog entry.

## Reproduction Assessment

Yes. Current main can be checked by calling `deriveSessionTitle({ sessionId, updatedAt, label: "My App" }, "heartbeat")`, which still returns the transcript fallback; the PR regression is source-reproducible with malformed store data such as `displayName: 42`.

## Solution Assessment

No as submitted. The extraction and explicit-label precedence are the right narrow direction, but the helper should preserve `normalizeOptionalString` behavior for persisted fields and document the user-facing fix before merge.
