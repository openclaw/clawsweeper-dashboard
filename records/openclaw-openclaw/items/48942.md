## Summary

Keep open: this PR addresses a valid session-title precedence bug and supplies direct behavior proof, but the current head still adds unsafe `.trim()` calls on shallowly validated persisted fields and omits the required changelog entry for a visible session title fix.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, moves focused tests, and changes precedence so explicit labels beat transcript-derived text while origin labels remain a fallback.

## Best Possible Solution

Merge a corrected version that keeps the extraction and compatibility re-export, uses `normalizeOptionalString` for persisted title candidates, keeps `origin.label` below transcript text, and records the user-facing fix in `CHANGELOG.md`.
