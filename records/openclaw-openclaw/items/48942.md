## Summary

Keep open: the PR still has a concrete P2 regression from dropping current-main string normalization while moving the gateway title helper. The label-precedence fix is useful, but the branch should normalize persisted title metadata before merge.

## What This Changes

The PR extracts gateway session-title derivation into a dedicated module and changes title precedence so explicit session labels survive transcript-derived heartbeat titles.

## Best Possible Solution

Keep the extraction and explicit-label precedence, but restore normalizeOptionalString handling for all persisted title metadata and add focused regression coverage for malformed stored title fields.

## Reproduction Assessment

Yes, by source inspection. Current main passes transcript firstUserMessage into deriveSessionTitle, and the TUI displays derivedTitle before displayName, so a labeled entry with heartbeat transcript text can still derive the heartbeat text.

## Solution Assessment

No, not as written. The precedence order is the right narrow fix, but the new helper should preserve current-main normalization for JSON-loaded session metadata before merge.
