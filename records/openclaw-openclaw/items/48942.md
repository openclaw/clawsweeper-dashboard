## Best Possible Solution

Keep the extraction and explicit-label precedence, but preserve current-main string normalization for all persisted title candidates before merging.

## Reproduction Assessment

Yes for the patch defect from source: current main normalizes persisted title fields as unknown values, while the PR directly calls `.trim()` on those persisted fields. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently written; the precedence order is a good scoped solution, but it should reuse the existing normalization contract before trimming persisted fields.
