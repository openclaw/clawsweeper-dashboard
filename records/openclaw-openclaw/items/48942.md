## Summary

Keep open: this is a plausible focused gateway bug fix, but the latest PR head still drops current-main string normalization for persisted title fields, which can turn malformed session metadata into a sessions.list throw.

## What This Changes

The PR extracts `deriveSessionTitle` into a dedicated gateway module, adds focused tests, and changes title precedence so explicit session labels beat transcript-derived titles while origin labels remain a later fallback.

## Best Possible Solution

Keep the explicit-label precedence fix and extraction, restore `normalizeOptionalString` handling for persisted title metadata, and leave broader inbound metadata/displayName title work to the existing related issue and PR.

## Reproduction Assessment

Yes, by source inspection. Current main passes transcript `firstUserMessage` into `deriveSessionTitle`, and that helper checks transcript text before any stored explicit session label, so a labeled session with heartbeat text can derive `heartbeat`.

## Solution Assessment

No, not as written. The extraction and precedence order are the right boundary, but the helper should preserve current-main string normalization for JSON-loaded session fields before merge.
