## What This Changes

The PR extracts Gateway `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils.ts`, adds focused tests, and changes title precedence so explicit labels win before transcript text while origin labels remain a fallback.

## Best Possible Solution

Land a scoped title-precedence extraction after preserving current-main normalization for all persisted title candidates and keeping the broader inbound-metadata/displayName behavior in the existing linked title work.

## Reproduction Assessment

Yes. By source inspection, current main ignores `entry.label` and derives from `firstUserMessage` after `displayName` and `subject`, so a labeled entry with heartbeat-style transcript text can still display the transcript title instead of the explicit label.

## Solution Assessment

No, not as-is. The precedence direction is maintainable, but the implementation must normalize all persisted title candidates before trimming and should leave the broader inbound-metadata/displayName fix to the linked title work or an equivalent replacement.
