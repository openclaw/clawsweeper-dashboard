Keep open: the PR is a useful Gateway session-title fix, but it is not merge-ready because the refreshed helper still replaces current-main safe string normalization with direct `.trim()` calls on persisted metadata, and GitHub reports the branch as conflicting.

## What This Changes

This PR extracts Gateway `deriveSessionTitle` into `src/gateway/session-title.ts`, moves its tests into a dedicated file, re-exports the helper from `session-utils.ts`, and changes title precedence to prefer explicit labels before transcript text while using origin labels as a later fallback.

## Best Possible Solution

Land this as a narrow label-precedence extraction only after preserving current-main `normalizeOptionalString` semantics, adding malformed-metadata regression coverage, and rebasing; leave the broader inbound-metadata/displayName title fix to https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement.

## Reproduction Assessment

Yes. By source inspection, current main checks `displayName` and `subject` before transcript text but ignores `entry.label`, so a labeled session can still display a heartbeat-style first message instead of the explicit label.

## Solution Assessment

No, not as-is. The title precedence direction is maintainable, but the implementation needs to preserve `normalizeOptionalString` behavior for persisted candidates before it is safe to merge.
