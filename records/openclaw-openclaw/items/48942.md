## Summary

Keep open: the PR addresses a valid source-reproducible gateway title precedence bug, but the patch should retain the existing string normalization guard before merge because session entries are loaded from JSON.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, changes title precedence to prefer explicit session labels before transcript text while keeping origin labels behind transcript text, and moves focused tests into a dedicated module.

## Best Possible Solution

Keep the narrow precedence fix and extraction, but normalize persisted title fields with the existing string helper before merging.

## Reproduction Assessment

Yes, by source inspection. Current main returns transcript-derived `firstUserMessage` before consulting `entry.label`, so a labeled session with heartbeat transcript text derives the title from `heartbeat`.

## Solution Assessment

No, not as written. The extraction and precedence order are the right boundary, but the helper should preserve `normalizeOptionalString` behavior for JSON-loaded session fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve string normalization before trimming:** `src/gateway/session-title.ts:31`
  - body: `loadSessionStore` parses `sessions.json` and only checks that the parsed value is an object before treating entries as `SessionEntry`, so current main uses `normalizeOptionalString(entry.displayName)` and `normalizeOptionalString(entry.subject)` to ignore malformed persisted values. The extracted helper now calls `.trim()` directly on `displayName` and the new label fields, so a non-string persisted value can throw while listing sessions instead of falling through. Please keep the existing normalization pattern for all title metadata fields.
