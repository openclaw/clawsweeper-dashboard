## Summary

Keep open: this PR addresses a real source-reproducible Gateway session-title precedence gap and has sufficient terminal proof, but the current patch still regresses persisted metadata handling by trimming store-loaded title fields directly instead of preserving current-main normalization.

## What This Changes

The PR extracts `deriveSessionTitle` into a dedicated Gateway module, re-exports it from `session-utils`, moves/consolidates tests, and adds explicit `entry.label` plus fallback `origin.label` precedence around transcript-derived titles.

## Best Possible Solution

Merge the narrow extraction and explicit-label precedence fix after restoring `normalizeOptionalString` handling for all persisted title candidates and adding regression coverage; keep inbound metadata stripping and external displayName routing with https://github.com/openclaw/openclaw/issues/39722 and https://github.com/openclaw/openclaw/pull/39790.

## Reproduction Assessment

Yes, by source inspection: current main feeds transcript `firstUserMessage` into `deriveSessionTitle`, the TUI renders `derivedTitle` before `displayName`, and the helper ignores explicit `entry.label`. The patch regression is also source-reproducible because store-loaded title metadata is not fully type-normalized before the new `.trim()` calls.

## Solution Assessment

No, not as submitted. The extraction and precedence order are the right narrow direction, but the helper should preserve current-main normalization for persisted title fields before merge.
