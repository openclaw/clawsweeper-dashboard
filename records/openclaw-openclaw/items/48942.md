## What This Changes

The PR extracts Gateway `deriveSessionTitle` into a dedicated module, adds focused tests, preserves explicit session labels ahead of transcript-derived titles, keeps origin labels as later fallbacks, and re-exports the helper from `session-utils.ts`.

## Best Possible Solution

Land this as a scoped title-precedence extraction only after preserving current-main normalization for persisted title candidates, adding malformed-metadata regression coverage, and rebasing against main.

## Reproduction Assessment

Yes. By source inspection, current main ignores `entry.label` and derives from `firstUserMessage` after `displayName` and `subject`, so a labeled entry with heartbeat-style transcript text can still display the transcript title instead of the explicit label.

## Solution Assessment

No, not as-is. The precedence direction is maintainable, but the implementation needs to normalize all persisted title candidates before trimming and should leave the broader inbound-metadata/displayName fix to the linked title PR or an equivalent replacement.
