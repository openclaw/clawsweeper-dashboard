## Summary

Keep open: current main has not adopted explicit session-label precedence in derived titles, but this PR still needs a narrow repair before merge because the extracted helper calls `.trim()` directly on persisted title fields.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, keeps a compatibility re-export from `session-utils`, and adds tests for explicit label and origin-label title precedence.

## Best Possible Solution

Keep the extraction and precedence tests, but guard every store-backed title candidate with string normalization before merge and leave broader metadata/displayName title work on the canonical related issue.

## Reproduction Assessment

Yes at source level: current main ignores `entry.label`, so a labeled entry plus heartbeat-style transcript text still produces the transcript-derived title. The PR defect is also source-reproducible because persisted JSON fields reach this helper without string coercion.

## Solution Assessment

Mostly yes: `deriveSessionTitle` is the right narrow boundary for explicit-label precedence, and the PR keeps origin labels behind transcript text. It is not the best mergeable fix until store-backed title fields use guarded string normalization.
