## Summary

Keep open: the branch addresses a real current-main title precedence gap, but it still drops the existing normalization guard for stored session metadata, so it should not merge until that narrow regression and the current branch conflict are fixed.

## What This Changes

The PR extracts `deriveSessionTitle` into a dedicated gateway module with focused tests and changes title precedence so explicit session labels win before transcript text while origin labels stay behind transcript summaries.

## Best Possible Solution

Keep the explicit-label precedence fix and extraction, restore `normalizeOptionalString` handling for all persisted title metadata fields, and leave the broader inbound metadata/displayName title work to the existing related issue and PR.

## Reproduction Assessment

Yes, by source inspection. Current main reads transcript `firstUserMessage` before any stored session label, and sessions.list passes that transcript field into `deriveSessionTitle`, so a labeled session with heartbeat text derives `heartbeat`.

## Solution Assessment

No, not as written. The extraction and precedence order are the right boundary, but the helper should preserve current-main string normalization for JSON-loaded session fields before merge.
