## Summary

Keep open: the PR is a plausible narrow fix for labeled session titles, but the current head regresses defensive handling of persisted session-store fields by calling string methods directly on shallowly validated JSON values.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it, moves focused tests, and changes title precedence so explicit labels beat transcript text while origin labels remain a fallback.

## Best Possible Solution

Merge a corrected version that keeps explicit `entry.label` ahead of transcript text, keeps `origin.label` behind transcript text, preserves the compatibility re-export, and normalizes persisted title candidates defensively.

## Reproduction Assessment

Yes. Source inspection on current main shows an entry with `label: "My App"` and first user message `heartbeat` still derives `heartbeat` because `entry.label` is not checked before transcript text.

## Solution Assessment

No as submitted. The precedence and extraction direction are narrow and maintainable, but the helper should keep `normalizeOptionalString` for persisted title candidates before merge.
