## Summary

Keep open: this PR addresses a real Gateway session-title precedence bug and has sufficient helper-level proof, but the current diff still drops current-main string normalization for persisted title metadata and the PR head is merge-conflicting with current main.

## What This Changes

The PR extracts `deriveSessionTitle` into a Gateway module, moves focused tests, re-exports the helper from `session-utils`, and changes title precedence to preserve explicit labels before transcript fallbacks while keeping origin labels as a later fallback.

## Best Possible Solution

Merge the narrow extraction and explicit-label precedence fix after restoring `normalizeOptionalString` handling for all persisted title candidates and adding malformed-metadata regression coverage; keep broader inbound metadata stripping and external displayName routing with https://github.com/openclaw/openclaw/issues/39722 and https://github.com/openclaw/openclaw/pull/39790.

## Reproduction Assessment

Yes. By source inspection, current main stores explicit labels on `SessionEntry`, passes transcript `firstUserMessage` into `deriveSessionTitle`, and renders `derivedTitle` before `displayName`, while the helper ignores `entry.label` before transcript-derived fallback.

## Solution Assessment

No, not as submitted. The extraction and precedence order are the right narrow direction, but the helper should normalize persisted `displayName`, `subject`, `label`, and `origin.label` values before trimming or returning them.
