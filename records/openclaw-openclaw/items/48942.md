## Summary

Keep open: this PR has useful, unique gateway title behavior, but the current head still introduces a blocking normalization regression for persisted session metadata.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, changes title precedence so explicit session labels beat transcript text, keeps origin labels behind transcript text, and moves focused tests into a dedicated module.

## Best Possible Solution

Keep the extraction and explicit-label precedence, but normalize every persisted title field before checking or returning it and add focused regression coverage for malformed stored title metadata.

## Reproduction Assessment

Yes, by source inspection. Current main passes transcript text into `deriveSessionTitle`, ignores explicit `entry.label`, and the TUI displays `session.derivedTitle` before `session.displayName`; the PR regression is also source-reproducible from the direct `.trim()` calls on persisted fields.

## Solution Assessment

No, not as written. The precedence order is the right narrow fix, but the extracted helper should preserve current-main `normalizeOptionalString` handling for JSON-loaded title metadata before merge.
