## Summary

Keep open: current main still allows transcript-derived text such as `heartbeat` to outrank an explicit session label, and this PR is a focused fix with adequate terminal proof and no blocking diff findings. The broader inbound-metadata title issue remains separate in https://github.com/openclaw/openclaw/issues/39722 and https://github.com/openclaw/openclaw/pull/39790.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, makes explicit session labels outrank transcript-derived titles, keeps origin labels behind real transcript text, and moves focused title tests into a dedicated module.

## Best Possible Solution

Land this narrow precedence fix after required-check handling, and keep the inbound metadata stripping and outer displayName work in https://github.com/openclaw/openclaw/issues/39722 and https://github.com/openclaw/openclaw/pull/39790.

## Reproduction Assessment

Yes, by source inspection. Current main returns `firstUserMessage` before consulting `entry.label`, so a labeled session with heartbeat transcript text will derive the title from `heartbeat`.

## Solution Assessment

Yes. The PR keeps the fix in the pure title helper, preserves the existing `session-utils` export, adds focused regression coverage, and keeps auto-populated origin labels behind real transcript text.
