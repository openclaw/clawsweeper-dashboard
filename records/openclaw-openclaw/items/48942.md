## Summary

Keep open: the branch addresses a real title-precedence bug, but it still introduces a regression by trimming JSON-loaded session title metadata without the current-main normalization guard.

## What This Changes

The PR extracts `deriveSessionTitle` into a dedicated gateway helper, re-exports it from `session-utils`, and adds focused tests for explicit label and origin label title precedence.

## Best Possible Solution

Keep the extraction and explicit-label precedence, restore `normalizeOptionalString` handling for persisted title metadata, and leave the broader metadata-stripping/displayName work to the related TUI title thread.

## Reproduction Assessment

Yes, by source inspection. Current main passes transcript `firstUserMessage` into `deriveSessionTitle`, the TUI displays `derivedTitle` first, and the helper ignores `entry.label`, so a labeled heartbeat transcript can surface transcript text.

## Solution Assessment

No, not yet. The precedence order is the right narrow fix, but the new helper should normalize unknown persisted title fields before trimming and must keep transcript text ahead of auto-populated `origin.label`.
