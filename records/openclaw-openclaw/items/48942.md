## Summary

Keep open: the PR contains a useful gateway session-title fix, but the branch still regresses the current guarded normalization pattern by trimming persisted title metadata directly before session listing can fall back.

## What This Changes

The branch extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, and adds tests while changing title precedence to prefer explicit session labels and use origin labels as a fallback.

## Best Possible Solution

Keep the extraction and explicit-label precedence, but normalize every store-backed title candidate before trimming and leave broader metadata/title-quality work to the canonical open issues.

## Reproduction Assessment

Yes, at source level. Current main ignores `entry.label`, and the branch defect is source-reproducible because persisted JSON entries reach `deriveSessionTitle` without coercing all title metadata fields to strings.

## Solution Assessment

Mostly yes. `deriveSessionTitle` is the right narrow boundary for explicit-label precedence, but the mergeable version should preserve guarded normalization for all store-backed title fields.
