## Summary

Keep open: the PR is useful and targets a real current-main session-title bug, but the latest diff still introduces a defensive-normalization regression and is missing the required changelog entry.

## What This Changes

The branch extracts `deriveSessionTitle` into a dedicated gateway module, re-exports it from `session-utils`, moves the focused tests, and changes title precedence so explicit labels outrank transcript-derived fallbacks while origin labels remain a later fallback.

## Best Possible Solution

Merge a corrected version that keeps the extraction and compatibility re-export, preserves `normalizeOptionalString` for persisted title candidates, keeps explicit `entry.label` ahead of transcript text and `origin.label` behind it, and adds an Unreleased Fixes changelog entry.

## Solution Assessment
