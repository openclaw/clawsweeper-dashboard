## Summary

Keep open: the PR is useful and has sufficient helper-level proof, but the current head still introduces a blocking robustness regression by trimming persisted title fields without the current-main string normalization guard.

## What This Changes

The branch extracts Gateway `deriveSessionTitle` into `src/gateway/session-title.ts`, moves focused tests there, re-exports the helper from `session-utils`, and adds explicit session label plus origin label precedence.

## Best Possible Solution

Merge the extraction and explicit-label precedence after restoring `normalizeOptionalString` handling for all persisted title candidates; leave the broader inbound metadata and outer displayName fixes to the linked session-title issue and PR.

## Reproduction Assessment

Yes, by source inspection: current main calls `deriveSessionTitle(entry, fields.firstUserMessage)` for TUI-visible derived titles and ignores explicit `entry.label`. The PR-head regression is also source-reproducible because the extracted helper directly trims persisted title metadata that store loading does not fully normalize.

## Solution Assessment

No, not as submitted. The extraction and precedence order are the right narrow direction, but the helper needs to preserve current-main normalization before it is safe to merge.
