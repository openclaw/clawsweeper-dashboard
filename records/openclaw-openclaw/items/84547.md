## Best Possible Solution

Keep the memory reduction, but implement derived alias metadata through WeakMap side caches or guarded no-mutation fallbacks aligned with current main, then rebase and validate the focused sdk-alias and script lint checks.

## Solution Assessment

No. The performance goal is reasonable, but current main's WeakMap side-cache pattern is the safer implementation because it preserves external object ownership and extensibility assumptions.
