## What This Changes

The PR changes plugin loader alias caching to reuse content-identical alias maps, stores derived cache metadata on alias map objects, updates memoization expectations, and adds a heap-snapshot proof script.

## Best Possible Solution

Preserve the content-dedup memory win only if it is rebuilt on top of current main's side-cache approach, keeps caller-owned alias maps immutable-safe, and fixes the proof-script lint failure.

## Reproduction Assessment

Yes. Source inspection plus a focused Node check show that Object.defineProperty throws when adding the PR's symbol metadata to a frozen alias map, and the exported/internal loader path accepts caller-provided alias maps.

## Solution Assessment

No. The performance direction is plausible, but the best fix should layer content dedupe onto current main's WeakMap side-cache pattern rather than mutating caller-owned alias maps.
