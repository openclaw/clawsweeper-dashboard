## Summary

Keep open. Current main still reuses the cached bundled MCP client/transport on tool calls with no stale SSE session detection or reconnect, and the adjacent open PR only targets streamable-http recovery.

## What This Changes

The issue requests automatic recovery for SSE-backed MCP tool sessions after a downstream server restart instead of requiring an OpenClaw gateway restart.

## Best Possible Solution

Add bounded SSE stale-session recovery in the session MCP runtime: detect known lost-session responses, recreate only the affected server client/transport, refresh relevant catalog/tool metadata, and retry the original call once while leaving stdio behavior unchanged.

## Reproduction Assessment

Yes. The issue provides a concrete successful-call, downstream SSE server restart, second-call path, and current source confirms the same cached session is reused without reconnect or retry; a regression test can model this with an SSE probe server that invalidates the session before the second call.

## Solution Assessment

Yes. The narrow maintainable fix is runtime-local SSE stale-session recovery with one retry and metadata refresh, ideally sharing helper code with streamable-http recovery if that related PR lands first.
