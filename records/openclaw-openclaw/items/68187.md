## Summary

Keep open: current main and v2026.6.8 still reuse the cached MCP client in `callTool()` without stale-session reconnect, while the two related reconnect PRs remain open with proof and retry-safety blockers.

## What This Changes

The issue asks OpenClaw to recover or recreate SSE-backed MCP client sessions after a downstream server restart instead of leaving tool calls stuck on stale session state.

## Best Possible Solution

Land one proof-backed MCP runtime fix that replaces stale per-server sessions safely and only retries when delivery semantics are acceptable, while leaving catalog reload work to https://github.com/openclaw/openclaw/issues/91556 and startup retry-storm work to https://github.com/openclaw/openclaw/issues/68527.
