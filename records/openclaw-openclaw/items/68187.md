## Summary

Keep open. Current main still keeps and reuses the cached bundled MCP client/transport for SSE-backed tool calls without stale-session detection, reconnect, or retry, and the adjacent open PR only addresses streamable-http recovery.

## What This Changes

OpenClaw should recover SSE-backed bundled MCP tool sessions after downstream server restarts instead of requiring a gateway restart.

## Best Possible Solution

Add bounded SSE stale-session recovery in the session MCP runtime: classify known lost-session POST errors, rebuild only the affected SSE client/transport, refresh tool metadata as needed, and retry the original call once while leaving stdio behavior unchanged.

## Reproduction Assessment

Yes. The issue gives a concrete successful-call, downstream SSE server restart, second-call path, and current source confirms the same cached session is reused without reconnect or retry; I did not run a live SSE server, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes. The narrow maintainable fix is runtime-local SSE stale-session recovery with one retry and metadata refresh, preferably sharing helper code with the streamable-http recovery path if that lands first.
