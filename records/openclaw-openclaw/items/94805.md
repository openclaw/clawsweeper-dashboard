## What This Changes

This PR adds a core `openclaw snapshot` CLI with local SQLite snapshot create, verify, restore, and list flows, plus snapshot provider code, docs, registration, and tests.

## Best Possible Solution

Land the snapshot command only after the target contract is explicit: either remove `memory-search`, implement a genuinely memory-only artifact, or rename/document it as a full agent-database snapshot with fresh proof and maintainer acceptance.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the review issue: the PR head maps `--target memory-search` to the full agent SQLite path, while current main stores auth-profile and memory tables in that same database.

## Solution Assessment

No. The core snapshot command is a plausible direction, but labeling a full agent DB export as `agent-memory-search` is not the best solution unless maintainers explicitly choose that broader contract and update docs/proof accordingly.
