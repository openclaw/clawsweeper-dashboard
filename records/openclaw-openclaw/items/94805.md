## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve the global SQLite symlink path invariant:** `src/commands/snapshot.ts:166`
  - body: `--target global` passes the canonical `state/openclaw.sqlite` path straight to the snapshot provider. Current backup code realpaths that same canonical global DB first because SQLite otherwise looks for WAL/SHM sidecars beside the symlink, so symlinked state roots can snapshot stale or incomplete data here while backup preserves it. Resolve the named global target the same way and add the symlinked WAL regression for `snapshot create`.
  - confidence: 0.91
- **[P2] Record userVersion from the generated artifact:** `packages/memory-host-sdk/src/host/sqlite-snapshot.ts:23-27`
  - body: `createVacuumedSqliteSnapshot` reads `PRAGMA user_version` before `VACUUM INTO`, but the output database is the artifact contract. If a live migration commits between that read and the VACUUM statement, the manifest can describe a different version than the restored database. Read the value from the generated snapshot file after VACUUM or otherwise tie the metadata to the same snapshot.
  - confidence: 0.78
