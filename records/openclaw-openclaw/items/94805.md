## Summary

Keep this draft open: the core snapshot command is useful and the earlier retired-config blocker was addressed, but the current `memory-search` target now labels a full per-agent database snapshot as memory-only and can include auth/session state in an artifact operators may sync under the wrong assumptions.

## What This Changes

The PR adds a core `openclaw snapshot create|verify|restore|list` CLI, local SQLite snapshot artifacts, named OpenClaw database targets, shared snapshot helpers, docs, and tests.

## Best Possible Solution

Ship the snapshot command only after target scopes are unambiguous: either remove/rename `memory-search` as a full agent-database snapshot, or implement a clearly documented memory-only export that excludes auth/session state and has fresh packaged proof.
