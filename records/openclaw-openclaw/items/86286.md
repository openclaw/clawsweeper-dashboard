## Summary

Keep open: the patch is a useful narrow start, but it sanitizes only the final assistant message and leaves streamed text events able to deliver the raw Kimi reasoning prefix; after-fix real behavior proof is also insufficient.

## What This Changes

The PR adds an Ollama Kimi Cloud inline-reasoning sanitizer in `extensions/ollama/src/stream.ts` and direct `buildAssistantMessage` regression tests in `extensions/ollama/src/stream-runtime.test.ts`.

## Best Possible Solution

Apply Kimi Cloud inline-reasoning stripping before any stream text event or partial message is published, cover the streamed path with a mocked NDJSON regression test, then attach redacted real Kimi Cloud output when access is available.

## Reproduction Assessment

Yes, source-reproducible: current and PR-head stream code appends every Ollama `message.content` chunk to visible `text_delta` events, and the linked issue includes Kimi Cloud config plus screenshot evidence. I did not run a live Kimi Cloud subscription repro here.

## Solution Assessment

No. Sanitizing only `buildAssistantMessage` is not the best fix because OpenClaw publishes streaming text events before that final `done` message; the fix should sanitize or buffer in the streaming emission path.
