## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match the reported delimiter with trailing whitespace:** `extensions/ollama/src/stream.ts:367`
  - body: The linked issue's example has the U+FE0F delimiter followed by a space before the visible answer, but this regex only matches when U+FE0F is immediately followed by a non-space. That means the reported final-message shape would keep the raw reasoning text visible; add coverage for `... \uFE0F Answer` and consume the optional delimiter whitespace.
  - confidence: 0.94
- **[P1] Hold Kimi prefix chunks until they are safe:** `extensions/ollama/src/stream.ts:1287-1291`
  - body: When a streamed Kimi response sends the reasoning prefix before the delimiter, `stripKimiInlineReasoningFromVisibleText` returns the raw accumulated prefix and this path publishes it as `text_delta`/partial content. The new stream test starts with a chunk that already contains the delimiter, so it misses the leak; buffer or suppress suspect Kimi prefix text until the boundary/final classification is known.
  - confidence: 0.92
- **[P2] Strip short answers after a valid delimiter:** `extensions/ollama/src/stream.ts:396-397`
  - body: This guard keeps the entire raw response when the answer after a valid Kimi delimiter is shorter than eight characters. A response like long private reasoning plus `\uFE0F OK.` would still expose the reasoning in the final message; once Kimi scope, long prefix, and delimiter are matched, only an empty answer should keep the raw text or route to a safer fallback.
  - confidence: 0.84
