## Summary

Keep open. Current main and v2026.5.22 include useful Sharp packaging and bounded missing-optimizer fallbacks, but the Android/Termux update failure remains: Sharp 0.34.5 still has no android-arm64 package, and OpenClaw does not yet have a Termux-aware native image dependency repair policy.

## What This Changes

The issue asks OpenClaw to handle Android/Termux Sharp native-module failures after updates, either by repairing native builds or falling back safely when image optimization is unavailable.

## Best Possible Solution

Decide the Android/Termux gateway support boundary; if supported, add a documented doctor/update native image dependency repair path or equivalent backend strategy with live Termux proof while preserving the current bounded fallbacks.

## Reproduction Assessment

No live Android/Termux current-main reproduction was run. Source-level reproduction is credible because Sharp 0.34.5 lacks an android-arm64 package, current resize/HEIC paths still require a usable backend, and update can omit failed optional native dependencies.

## Solution Assessment

Unclear. The shipped optional dependency and bounded raw-image fallback are useful mitigations, but the durable fix depends on whether maintainers want OpenClaw update or doctor to own Termux native-module repair.
