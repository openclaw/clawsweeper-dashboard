## Summary

Keep open. Current main and v2026.5.22 have partial mitigations, but the Android/Termux sharp failure remains reachable because sharp 0.34.5 has no android-arm64 prebuilt and OpenClaw still needs an image backend for HEIC, over-cap, or resize-required media.

## What This Changes

The issue requests durable Android/Termux handling for sharp native-module failures after OpenClaw updates, including rebuild/repair support or a safe fallback path.

## Best Possible Solution

Maintain the current safe fallback/backend chain and have maintainers decide the Android/Termux Gateway support boundary; if supported, add a documented update/doctor repair or Termux-native backend path.

## Reproduction Assessment

Yes, source-reproducible. Current main still uses sharp 0.34.5 and sharp's own v0.34.5 platform list has no Android prebuilt, while OpenClaw still reaches image backends for resize, HEIC conversion, and over-cap images.

## Solution Assessment

Unclear. The current fallback and system-backend chain are good partial mitigations, but the durable rebuild/update behavior depends on whether maintainers want to support Android/Termux as a Gateway host.
