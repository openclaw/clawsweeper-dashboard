## Summary

Keep open. Current main and v2026.5.22 shipped useful Sharp packaging and media fallback mitigations, but the Android/Termux update case is not fully solved: Sharp still has no android-arm64 prebuilt/libvips optional package, resize/HEIC paths can still require a working image backend, and OpenClaw does not have a Termux-aware update or doctor repair path.

## What This Changes

The issue asks OpenClaw to preserve image-tool behavior on Android/Termux after updates by handling Sharp's missing android-arm64 prebuilt through native-module repair or safe fallback behavior.

## Best Possible Solution

Define the Android/Termux Gateway support boundary; if supported, add a documented Termux-aware doctor/update native dependency repair or backend discovery path while preserving the current safe media fallbacks.

## Reproduction Assessment

No live current-main Termux reproduction was run. Source-level reproduction is clear enough to keep this open: Sharp 0.34.5 lacks android-arm64 packages and current resize/HEIC paths still need a working image backend or external tool.

## Solution Assessment

Unclear. The shipped root optional dependency and bounded raw-image fallback are good mitigations, but the durable fix depends on whether maintainers support Android/Termux as a Gateway host and where native-module repair belongs.
