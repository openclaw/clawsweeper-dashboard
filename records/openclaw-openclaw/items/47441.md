## Summary

Keep open. Current main and v2026.5.22 include important Sharp packaging and generic media fallback mitigations, but the Android/Termux update failure is still not fully owned: upstream Sharp 0.34.5 has no android-arm64 package, resize/HEIC paths still need a working backend, and OpenClaw has no Termux-aware native dependency repair policy.

## What This Changes

The issue requests durable handling for Sharp native-module failures on Android/Termux after OpenClaw updates, including possible update/doctor repair or safe image fallback behavior.

## Best Possible Solution

Decide the Android/Termux Gateway support boundary; if supported, add a documented Termux-aware update or doctor repair path for native image dependencies while preserving current safe media fallbacks and clear failure messages.

## Reproduction Assessment

No live Android/Termux current-main reproduction was run. Source-level reproduction is credible: Sharp 0.34.5 has no android-arm64 package, current resize/HEIC paths still require an available backend, and update fallback can omit optional native dependencies after build failures.

## Solution Assessment

Unclear. The shipped root optional dependency and bounded raw-image fallback are good mitigations, but the durable fix depends on a maintainer decision about Android/Termux Gateway support and native-module repair ownership.
