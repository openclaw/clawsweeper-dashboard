## Summary

Keep open. Current main still depends on sharp for resize/HEIC/image-op paths on non-sips platforms, sharp 0.34.5 still has no Android prebuilt target, and OpenClaw has only partial safe-original fallbacks rather than a durable Android/Termux update or native-module policy.

## What This Changes

The issue asks OpenClaw to handle Android/Termux sharp native-module failures after updates through native rebuild support, a repair command, or graceful image fallback.

## Best Possible Solution

Maintainers should decide the Android/Termux Gateway support policy, keep the safe-image fallbacks, handle root package metadata in https://github.com/openclaw/openclaw/issues/77760, and either add tested Termux native-module repair or document the unsupported path clearly.

## Reproduction Assessment

Yes, by source and dependency-contract inspection: current non-sips resize/HEIC paths still load sharp, and sharp 0.34.5 has no android-arm64 prebuilt target. I did not run the reporter's Android/Termux setup live.

## Solution Assessment

Unclear. A bounded fallback is already partly implemented, but fully resolving this issue requires a maintainer choice between Termux native rebuild support, clearer unsupported-platform behavior, and broader raw-image fallback semantics.
