## Summary

Keep open. Current main still has reachable image paths that require the sharp-backed media-understanding runtime on non-macOS-sips platforms, and the updater only retries npm installs with omitted optional dependencies rather than repairing the Android/Termux toolchain.

## What This Changes

The issue requests Android/Termux handling for sharp native-module failures after update, via Termux-aware rebuild tooling, a native-module fix command, or graceful image fallback when sharp is unavailable.

## Best Possible Solution

Add a bounded backend-unavailable fallback in shared image ingestion/optimization that preserves already-safe image inputs and fails clearly only when transformation is required; keep Termux native rebuild automation separate.

## Reproduction Assessment

Yes for the current-main application path: force the bundled sharp import/load to fail, then process a non-GIF optimized image, HEIC input, or managed image requiring resize. The exact Android/Termux device setup remains reporter-provided rather than locally reproduced.

## Solution Assessment

Partly: graceful fallback is the narrowest maintainable first fix for the user-visible failure. Automatic Termux rebuild/update repair is a broader native packaging decision and should not be bundled into that first fix.
