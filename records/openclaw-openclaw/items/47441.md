## Summary

Keep open. Current main and v2026.5.22 contain useful Sharp packaging and media fallback mitigations, but the Android/Termux failure is still not fully owned because Sharp 0.34.5 has no android-arm64 package and OpenClaw has no Termux-aware update or doctor repair policy for native image dependencies.

## What This Changes

The issue requests a durable fix for recurring Android/Termux Sharp native-module failures after OpenClaw updates, affecting image-tool and Telegram gateway workflows.

## Best Possible Solution

Decide the Android/Termux Gateway support boundary; if supported, add a documented Termux-aware native image dependency repair path while preserving current bounded media fallbacks and clear backend-unavailable errors.

## Reproduction Assessment

No live Android/Termux current-main reproduction was run. Source-level reproduction is credible because Sharp 0.34.5 lacks an android-arm64 package, current resize/HEIC paths still require a usable backend, and update can retry without optional native dependencies.

## Solution Assessment

Unclear. The shipped root optional dependency and bounded raw-image fallback are good mitigations, but the durable fix depends on whether maintainers want OpenClaw update or doctor to own Termux native-module repair.
