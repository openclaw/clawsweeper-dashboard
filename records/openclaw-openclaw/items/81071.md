Keep this PR open: it is a recent external security-hardening PR with the protected `maintainer` label, no after-fix real behavior proof, and a small protocol-doc mismatch to resolve before merge.

## What This Changes

The PR adds run IDs to gateway-issued node `system.run` invokes and rejects node exec lifecycle events unless they match a pending invoke for the same node connection, run id, and session.

## Best Possible Solution

Keep the narrow NodeRegistry-backed provenance check, add the bridge-protocol contract update, and require redacted terminal/live gateway-node proof before maintainer merge.

## Reproduction Assessment

Yes, from source inspection: current main accepts parsed node exec lifecycle payloads and queues command-completion context without a pending-invoke check. I did not run a live gateway/node replay in this read-only review.

## Solution Assessment

Mostly yes: binding accepted events to the issuing connection, run id, and session in NodeRegistry is the narrow maintainable seam. The PR still needs protocol docs aligned and real behavior proof before merge.
