## Summary

Keep open: this is an active external security-hardening PR with a protected `maintainer` label. The implementation direction looks mostly correct, but it still lacks external real behavior proof and leaves the public node exec-event protocol docs out of sync.

## What This Changes

The PR adds gateway/node run-id provenance checks for `node.event` exec lifecycle handling, updates macOS node run-id propagation, and adds focused TypeScript and Swift tests.

## Best Possible Solution

Keep the narrow NodeRegistry-backed provenance gate, document the public gateway/node exec-event contract, and require redacted live or terminal gateway-node proof before maintainer merge.

## Reproduction Assessment

Yes. Source inspection of current main shows standalone exec lifecycle payloads can enqueue command-completion context without a pending-invoke provenance check; I did not run a live gateway/node replay in this read-only review.

## Solution Assessment

Mostly yes. Binding accepted lifecycle events to the issuing node connection, run id, and session in NodeRegistry is the narrow maintainable fix, but docs and real behavior proof still need completion.
