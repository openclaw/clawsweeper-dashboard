## Summary

Keep open: current main still has the security-sensitive acceptance path and this PR is a plausible narrow hardening, but it has protected maintainer handling, lacks required real behavior proof, and still needs protocol docs for the changed node exec-event contract.

## What This Changes

The branch adds generated run IDs and NodeRegistry-backed provenance checks for node `system.run` exec lifecycle events, updates macOS runId propagation, and adds focused TypeScript and Swift tests.

## Best Possible Solution

Keep the narrow NodeRegistry-backed provenance gate, document the gateway node exec-event contract, and require redacted live or terminal gateway-node proof before maintainer merge.

## Reproduction Assessment

Yes. Source inspection of current main shows standalone `node.event` exec lifecycle payloads can enqueue system-event and heartbeat context without a pending-invoke provenance check; I did not run a live WebSocket replay.

## Solution Assessment

Mostly yes. Binding accepted lifecycle events to the issuing node connection, run id, and session in `NodeRegistry` is the narrow maintainable fix, but the docs and real behavior proof still need completion.
