## Summary

Keep open: this is a focused security hardening fix for a source-reproducible gateway/node exec-event gap, but the external PR still lacks real behavior proof and the protected `maintainer` label requires explicit maintainer handling.

## What This Changes

The PR adds generated run IDs to gateway-issued node `system.run` invokes, records pending run provenance in `NodeRegistry`, rejects unmatched node exec lifecycle events, and adds focused unit coverage.

## Best Possible Solution

Keep the narrow `NodeRegistry`-backed provenance gate, document the bridge protocol contract, and require redacted live or terminal gateway-node proof before maintainer merge.

## Reproduction Assessment

Yes. Source inspection of current main shows `node.event` exec lifecycle payloads can enqueue command-completion context without a pending-invoke provenance check; I did not run a live gateway/node replay in this read-only review.

## Solution Assessment

Mostly yes. Binding accepted lifecycle events to the issuing node connection, run id, and session in `NodeRegistry` is the narrow maintainable fix, but the docs and real behavior proof still need to be completed before merge.
