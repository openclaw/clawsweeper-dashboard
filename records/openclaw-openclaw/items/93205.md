## Summary

Keep open: current main and the latest release still drop Mattermost posts after `normalizeMention` leaves an empty body, even when the raw post contained the bot mention. The issue is source-reproducible, split from the broader Mattermost umbrella, and has a narrow plugin-owned fix path.

## What This Changes

The issue asks Mattermost to wake the agent when a posted message contains only a bot @mention and no remaining normalized body text.

## Best Possible Solution

Add a Mattermost-owned regression fix that treats an intentional bare bot mention as a wake/dispatch event while still dropping truly empty unmentioned posts.

## Reproduction Assessment

Yes. Source inspection shows the Mattermost handler detects the raw mention, strips it with `normalizeMention`, then returns before dispatch when the normalized body is empty.

## Solution Assessment

Yes. The maintainable fix belongs in the Mattermost inbound normalization/empty-body decision, with tests pinning mention-only wake behavior rather than a new config option or a core dispatch change.
