## Summary

Keep open: current main and v2026.6.6 still drop Mattermost bare bot mentions after mention normalization, while open closing PRs now own the fix path. This issue should stay open until one canonical PR lands.

## What This Changes

Mattermost should treat a bare bot @mention with no body text as an agent wake event instead of dropping it after `normalizeMention` clears the body.

## Best Possible Solution

Land one focused Mattermost plugin fix from the open closing PRs that keeps intentional bare bot mentions while preserving drops for truly empty unmentioned posts, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows a raw bot mention can pass mention detection, `normalizeMention` can strip it to an empty body, and the current `if (!bodyText)` guard returns before dispatch; no live workspace repro was run.

## Solution Assessment

Yes. The maintainable fix belongs in the Mattermost inbound empty-body decision with regression coverage, not in core dispatch or a new config option; the specific PR choice should be handled in PR review.
