## Summary

Keep open. The reported response-prefix bug is still source-reproducible on current main, but this PR branch is stale against the current cron dispatch path and still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR imports response-prefix config resolution into cron delivery dispatch, prepends configured prefixes to cron outbound text payloads, and adds a response-prefix regression test file.

## Best Possible Solution

Port response-prefix application onto the current isolated cron delivery path after silent/TTS payload normalization and before durable send, keep awareness/run-record text unprefixed, update current regression coverage, and add redacted real channel proof.

## Reproduction Assessment

Yes, at source level. Current main sends normalized cron direct-delivery payloads without resolving `messages.responsePrefix`, while normal routed replies do resolve and normalize that prefix.

## Solution Assessment

No, not as-is. The intended fix is narrow, but this branch must be rebased onto the current dispatch params and apply the prefix to the current `payloadsForDelivery` path without bypassing silent or TTS handling.
