## Summary

Keep open for normal maintainer review. The patch appears correct and aligns the SDK with the existing Gateway `tools.effective` `sessionKey` contract, but it intentionally tightens a public SDK method signature and hosted validation was still pending.

## What This Changes

The PR requires `oc.tools.effective()` callers to pass `ToolsEffectiveParams`, adds SDK-side nonblank `sessionKey` validation before RPC, exports the params type, and adds regression coverage.

## Best Possible Solution

Land the SDK boundary validation after hosted checks if maintainers accept the intentional public SDK signature tightening; otherwise keep the runtime guard and add a temporary compatibility overload.

## Reproduction Assessment

Yes. Current main is source-reproducible: `oc.tools.effective()` accepts missing params in the SDK and forwards them to RPC, while the Gateway schema and handler reject missing `sessionKey`.

## Solution Assessment

Yes. Validating `sessionKey` in `ToolsNamespace.effective` is the narrow SDK-layer fix because it mirrors the existing Gateway contract without loosening server validation or touching unrelated callers.
