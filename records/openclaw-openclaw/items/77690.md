## Summary

Keep open: this PR targets a valid WebChat selector bug with a narrow UI-state fix, but it is still draft and lacks required after-fix real behavior proof; there is also a low-priority plain-session default-agent edge to address before landing.

## What This Changes

The PR changes the WebChat model selector state to label the default option from the active agent model, adds focused selector-state regression coverage, and adds a changelog entry.

## Best Possible Solution

Resolve the active agent from the parsed agent session key or configured default agent, keep the active-row fallback for unloaded agent lists, retain focused regression coverage, and add redacted after-fix WebChat or terminal proof before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main builds the `Default (...)` label from global session defaults while WebChat renders that label directly and gateway rows expose the active agent/session model identity.

## Solution Assessment

Mostly yes. The PR is aimed at the narrow selector-state boundary, but the new agent lookup should account for `agentsList.defaultId` for plain session keys and the contributor still needs accepted after-fix real behavior proof.
