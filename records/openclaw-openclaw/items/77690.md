The PR changes WebChat model-selector state so the `Default (...)` label can prefer the active agent's configured model, adds focused selector-state coverage, and edits `CHANGELOG.md`.

## Best Possible Solution

Land a narrow WebChat selector fix that derives the default label from the active agent's configured model, mirrors the existing default-agent alias contract, includes real WebChat proof, and leaves `CHANGELOG.md` to release generation.

## Reproduction Assessment

## Solution Assessment

No, not yet: using `agentsList` is the right seam, but the PR should reuse the configured default-agent fallback for plain session keys, remove the release-owned changelog edit, and add real WebChat proof.
