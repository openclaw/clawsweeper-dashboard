## Summary

Keep open: the PR is the active narrow fix for the WebChat default-model label bug, but it is still draft, lacks accepted after-fix real behavior proof, and has a small plain-session default-agent edge to resolve or explicitly accept.

## What This Changes

The PR changes WebChat selector state to derive the Default label from the active agent/session model before global defaults, with focused regression coverage and a changelog entry.

## Best Possible Solution

Resolve the selector default label from the parsed agent id or configured default agent, fall back through the active session row and global defaults, keep focused regression coverage, and require redacted after-fix proof before landing.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main builds defaultLabel from global session defaults while WebChat renders that label directly and gateway rows expose agent-scoped model identity.

## Solution Assessment

Mostly yes. The PR targets the narrow selector-state boundary, but the active-agent lookup should account for agentsList.defaultId for plain session keys and the contributor still needs accepted after-fix real behavior proof.
