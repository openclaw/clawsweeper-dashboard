## Summary

Keep open: this is a plausible narrow WebChat bug fix, but the PR is still draft, lacks required after-fix real behavior proof, and leaves a small default-agent edge case in the new lookup.

## What This Changes

The PR changes WebChat selector state to derive the Default label from the active agent model, adds selector regression coverage, and adds a changelog line.

## Best Possible Solution

Resolve the active agent from the parsed agent session key or configured default agent, keep the active-row fallback for unloaded agent lists, retain focused regression coverage, and add redacted after-fix proof before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main builds the Default label from global session defaults while WebChat renders that label directly and gateway rows expose the active agent/session model identity.

## Solution Assessment

Mostly yes. The PR targets the narrow selector-state boundary, but the new agent lookup should account for `agentsList.defaultId` for plain session keys and the contributor still needs accepted after-fix real behavior proof.
