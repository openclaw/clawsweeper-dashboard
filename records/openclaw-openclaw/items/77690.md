## Summary

Keep open: this is a focused bug-fix PR for a still-valid WebChat label mismatch, but it is still draft, lacks accepted after-fix real behavior proof, and retains a small default-agent edge in the proposed helper.

## What This Changes

The PR updates WebChat model selector state, focused selector tests, and the changelog so the `Default (...)` label prefers the active agent model before global defaults.

## Best Possible Solution

Resolve the default label from the parsed agent session key or configured default agent, keep the active-row fallback for unloaded agent lists, retain focused regression coverage, and add redacted after-fix proof before landing.

## Reproduction Assessment

Yes, as a source-level reproduction: current main builds the WebChat `Default (...)` label from global session defaults while WebChat renders that label directly and gateway rows expose agent-scoped model identity. I did not run a live WebChat session in this read-only review.

## Solution Assessment

Mostly yes. The selector-state helper is the narrow maintainable boundary, but the PR should resolve plain session keys through `agentsList.defaultId` and still needs accepted after-fix real behavior proof.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P3] Use the configured default agent for plain session keys:** `ui/src/ui/chat-model-select-state.ts:38-39`
  - body: `resolveAgentIdFromSessionKey(state.sessionKey)` maps a plain key such as `main` to `main`, while nearby WebChat paths treat plain keys as `agentsList.defaultId`.   - confidence: 0.75
