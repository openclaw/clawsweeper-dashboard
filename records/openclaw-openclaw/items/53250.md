## Summary

Keep open. Current `main` and the latest release still route unavailable exec approvals through generic recovery copy that does not name the dashboard bootstrap command or approval-inspection command, while the CLI/docs surfaces needed for the requested guidance already exist.

## What This Changes

Request clearer exec approval timeout/unavailable messaging with Control UI access, channel setup, and pending/effective approval inspection hints.

## Best Possible Solution

Update the shared exec approval unavailable reply so no-route, disabled, and unsupported paths give token-safe recovery steps: open the Control UI via `openclaw dashboard --no-open`, inspect effective policy with `openclaw approvals get` or its alias, and enable the appropriate channel `execApprovals.enabled` or approver config only when that channel supports native approvals.

## Reproduction Assessment

Yes. A source-level reproduction is to call `buildExecApprovalUnavailableReplyPayload()` for `no-approval-route` or `initiating-platform-disabled`; the current helper returns generic Web UI/terminal UI text and the existing tests cover those branches without the requested recovery hints.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared helper and focused test update, with token-safe wording and existing CLI/docs targets; the closed PR is useful context but needs a fresh current-main implementation.
