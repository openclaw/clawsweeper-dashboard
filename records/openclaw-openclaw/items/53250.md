## Summary

Keep open. Current main still uses generic exec-approval unavailable copy and does not include the concrete Control UI bootstrap or approval-inspection commands requested, while the needed CLI/docs surfaces already exist and the directly linked fix PR was closed unmerged.

## What This Changes

Improve exec approval timeout/unavailable messages with token-safe Control UI access, native approval setup, and approval policy inspection guidance.

## Best Possible Solution

Add concise recovery guidance to the shared approval-unavailable reply path so no-route, disabled-platform, and unsupported-platform cases tell operators how to open the Control UI, inspect the effective approval policy, and enable native channel approvals without hard-coding secrets or duplicating channel-specific setup rules.

## Reproduction Assessment

Yes. A source-level reproduction is to call `buildExecApprovalUnavailableReplyPayload()` with `reason: "no-approval-route"` or `reason: "initiating-platform-disabled"`; current main returns generic Web UI/terminal UI/native-client text and lacks the requested commands.

## Solution Assessment

Yes. The narrowest maintainable repair is a shared helper/test update, preserving existing channel-specific setup providers and adding token-safe command hints rather than adding a new CLI or changing approval policy behavior.
