## Summary

Keep open: the PR targets a real continuation-skip gap, but the current fallback is too broad because it treats heartbeat-only assistant replies as completed bootstrap evidence and still needs the required changelog entry.

## What This Changes

The branch makes `hasCompletedBootstrapTurn` count assistant messages without a full-bootstrap marker and adds regression coverage for assistant-only continuation and post-compaction assistant turns.

## Best Possible Solution

Amend the detector so assistant-message compatibility only accepts ordinary non-heartbeat continuation evidence, preserves post-compaction reinjection, and ships with focused regression coverage plus a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a shared-session tail with a heartbeat prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, then a normal `contextInjection: "continuation-skip"` turn; this branch would classify that heartbeat assistant row as completed bootstrap evidence.

## Solution Assessment

No for the current patch. The fallback belongs near `hasCompletedBootstrapTurn`, but it should skip recognized heartbeat-only pairs or otherwise require non-heartbeat assistant/marker evidence.

## Review Findings
