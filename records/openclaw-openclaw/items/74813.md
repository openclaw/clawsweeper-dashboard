## Summary

Keep open. The continuation-skip gap is still relevant, but this draft PR is too broad because it treats any assistant transcript row as completion evidence, including heartbeat-only acknowledgements that do not prove a full bootstrap happened, and it still lacks real behavior proof.

## What This Changes

The branch changes `hasCompletedBootstrapTurn()` to treat flushed assistant messages as completion evidence without a full-bootstrap marker and updates regression tests for assistant-only and post-compaction transcripts.

## Best Possible Solution

Revise the helper to accept only ordinary post-bootstrap continuation evidence while preserving heartbeat/lightweight and post-compaction reinjection, then add redacted runtime proof for the continuation-skip path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a transcript containing only a heartbeat poll and `HEARTBEAT_OK` assistant reply, with no full-bootstrap marker, would make the PR fallback report completion for the next normal continuation-skip run.

## Solution Assessment

No. The helper is the right place to fix the missing-marker subcase, but the proposed any-assistant fallback should ignore recognized heartbeat-only pairs or require ordinary non-heartbeat continuation evidence.
