## Summary

Keep open: the PR targets a real continuation-skip bug, but the proposed fallback is too broad and can let heartbeat-only assistant replies satisfy bootstrap completion. It also lacks after-fix real behavior proof for the external contributor merge gate.

## What This Changes

The PR changes bootstrap continuation detection so assistant transcript rows can satisfy completion without a full-bootstrap marker, with focused tests for assistant-only and post-compaction turns.

## Best Possible Solution

Keep the PR open for a narrowed patch that treats ordinary post-bootstrap assistant turns as continuation evidence while preserving heartbeat/lightweight and post-compaction reinjection, then add redacted real behavior proof.

## Reproduction Assessment

Yes. A source-level reproduction is a shared-session transcript containing only a light heartbeat prompt and HEARTBEAT_OK assistant reply, no full-bootstrap marker, followed by a normal continuation-skip turn; the PR fallback would return true for that heartbeat reply.

## Solution Assessment

No for the PR as written. The helper is the right place to fix this, but the fallback should ignore recognized heartbeat-only pairs or require ordinary non-heartbeat continuation evidence before skipping bootstrap.
