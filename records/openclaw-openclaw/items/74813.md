## Summary

Keep open. The PR targets a real continuation-skip gap that still exists on current main, but the proposed assistant-message fallback is too broad and can skip full bootstrap after heartbeat-only shared-session turns.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` so any flushed assistant message can count as continuation evidence without a full-bootstrap marker, and updates regression tests for assistant-only and post-compaction transcript tails.

## Best Possible Solution

Amend the PR so assistant-message compatibility only accepts ordinary completed assistant turns, ignores recognized heartbeat-only user/assistant pairs, preserves the post-compaction guard, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a session tail with a heartbeat user prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, `contextInjection: "continuation-skip"`, and then a normal non-heartbeat run; the PR fallback would return true and skip workspace bootstrap injection.

## Solution Assessment

No for the PR as written. The fallback belongs in `hasCompletedBootstrapTurn`, but counting every assistant row is broader than the heartbeat/lightweight-bootstrap contract; the safer fix is to continue scanning past heartbeat-only pairs or require a non-heartbeat assistant/marker.
