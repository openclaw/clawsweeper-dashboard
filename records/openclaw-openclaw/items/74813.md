## Summary

Keep open. The PR targets a real continuation-skip gap on current main, but the proposed fallback is too broad because it lets heartbeat-only assistant replies satisfy bootstrap completion, and the user-visible fix still needs a changelog entry.

## What This Changes

The PR changes bootstrap continuation detection to treat any flushed assistant message as completed continuation evidence when no full-bootstrap marker exists, and updates helper tests around assistant-only and post-compaction transcripts.

## Best Possible Solution

Constrain assistant-message compatibility so heartbeat-only pairs cannot satisfy bootstrap completion, preserve post-compaction reinjection, and add focused regression coverage plus a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a shared-session transcript containing only a light heartbeat prompt and `HEARTBEAT_OK` assistant reply, no full-bootstrap marker, followed by a normal continuation-skip turn; the PR fallback would treat that heartbeat reply as completed bootstrap evidence.

## Solution Assessment

No for the PR as written. The fallback belongs near `hasCompletedBootstrapTurn`, but it should ignore recognized heartbeat-only pairs or require ordinary non-heartbeat continuation evidence before skipping bootstrap.
