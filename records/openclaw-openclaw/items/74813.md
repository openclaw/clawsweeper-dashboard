## Summary

Keep open. Current main still uses marker-only continuation detection, and this PR addresses a real gap, but the proposed fallback is too broad because it would let heartbeat-only assistant replies count as completed bootstrap evidence for later normal turns.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` to treat assistant messages without the full-bootstrap marker as completed continuation evidence and updates regression tests for assistant-only and post-compaction transcript tails.

## Best Possible Solution

Keep the assistant-message compatibility direction, but constrain completion detection so recognized heartbeat-only user/assistant pairs are ignored or scanned past, then cover that transcript shape with a focused regression test before merging.

## Reproduction Assessment

Yes. A source-level reproduction is a session JSONL tail containing a heartbeat user prompt followed by an assistant `HEARTBEAT_OK`, no full-bootstrap marker, `contextInjection: "continuation-skip"`, and then a normal follow-up run; the PR fallback would treat that as completed bootstrap evidence.

## Solution Assessment

No. The target area is right, but treating every assistant row as safe continuation evidence is broader than the heartbeat/lightweight bootstrap contract; the safer fix is to require non-heartbeat assistant evidence or continue scanning past heartbeat-only pairs.
