## Summary

Keep open. Current main still has marker-only continuation detection, so the PR targets a real `continuation-skip` gap, but the patch is too broad because it treats any assistant transcript row, including legacy heartbeat acknowledgements, as safe bootstrap-completion evidence and it lacks the required changelog entry.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` to treat assistant message records without full-bootstrap markers as completed continuation evidence and updates focused bootstrap tests.

## Best Possible Solution

Keep the helper change, but constrain assistant-message compatibility to ordinary non-heartbeat continuation evidence, preserve post-compaction reinjection, add heartbeat-only regression coverage, and add a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a shared-session transcript with only a lightweight heartbeat prompt plus `HEARTBEAT_OK` assistant reply and no full-bootstrap marker would satisfy the PR fallback before a later normal `continuation-skip` turn.

## Solution Assessment

No. The branch changes the right helper, but accepting any assistant message is not the narrowest maintainable fix; it should ignore recognized heartbeat-only pairs or require non-heartbeat continuation evidence before skipping bootstrap.
