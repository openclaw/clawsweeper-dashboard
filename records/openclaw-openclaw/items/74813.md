## Summary

Keep this PR open. Current main still has the marker-only continuation detector, so the proposed fix is not obsolete, but the PR as written would count heartbeat-only assistant acknowledgments as safe continuation evidence and it misses the required changelog entry.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` so a flushed assistant message can satisfy `continuation-skip` without a full-bootstrap marker, and updates bootstrap-files regression tests for assistant-only and post-compaction transcript tails.

## Best Possible Solution

Amend the detector so assistant-message compatibility accepts only ordinary completed non-heartbeat turns, skips or scans past recognized heartbeat-only pairs, preserves the post-compaction reinjection guard, and adds focused regression coverage plus a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a session tail with a heartbeat user prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, `contextInjection: "continuation-skip"`, followed by a normal turn; the PR fallback would return true and skip workspace bootstrap injection.

## Solution Assessment

No for the current diff. The fallback belongs near `hasCompletedBootstrapTurn`, but counting every assistant row is broader than the heartbeat/lightweight-bootstrap contract; the safer solution is to require a non-heartbeat assistant turn or marker while continuing to honor compaction.
