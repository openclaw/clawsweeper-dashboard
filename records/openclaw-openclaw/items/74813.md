## Summary

Keep open. The PR addresses a real continuation-skip gap on current main, but the proposed fallback is broader than the heartbeat/lightweight-bootstrap contract and can skip full bootstrap after heartbeat-only transcript tails; the existing ClawSweeper review comment’s heartbeat concern is supported by current source and tests.

## What This Changes

The PR changes continuation-skip transcript detection so assistant messages without a full-bootstrap marker can count as completed continuation evidence, with focused regression tests.

## Best Possible Solution

Amend the PR so assistant-message compatibility only accepts ordinary completed non-heartbeat continuation evidence, preserves post-compaction reinjection, and adds focused regression coverage plus a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a shared session tail with a heartbeat prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, then a normal `contextInjection: "continuation-skip"` turn; the PR fallback would classify that heartbeat assistant row as completed bootstrap evidence.

## Solution Assessment

No for the PR as written. The fallback belongs near `hasCompletedBootstrapTurn`, but counting every assistant row is too broad; the safer implementation should skip recognized heartbeat-only pairs or require non-heartbeat assistant/marker evidence.
