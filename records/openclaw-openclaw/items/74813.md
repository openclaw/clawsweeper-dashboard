## Summary

Keep open. The PR targets a real continuation-skip gap, but the current diff would also treat heartbeat-only assistant replies as completed bootstrap evidence, which can skip workspace bootstrap before any full non-heartbeat turn has completed; it also needs the required changelog entry.

## What This Changes

The PR changes bootstrap continuation detection so assistant JSONL messages without the full-bootstrap marker can count as completed turns, and updates bootstrap-file regression tests.

## Best Possible Solution

Constrain assistant-message compatibility so heartbeat-only pairs cannot satisfy bootstrap completion, preserve post-compaction reinjection, and add focused regression coverage plus a single-line changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main still rejects assistant-only no-marker transcripts, and the PR fallback would accept a shared-session transcript whose only completed assistant row is a light heartbeat HEARTBEAT_OK reply.

## Solution Assessment

No for the PR as written. The fallback belongs near hasCompletedBootstrapTurn, but it should ignore recognized heartbeat-only pairs or require ordinary non-heartbeat continuation evidence before skipping bootstrap.
