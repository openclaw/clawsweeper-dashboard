## What This Changes

The PR extends Windows QMD `.cmd` shim parsing for quoted absolute JS/EXE entrypoints, changes QMD fallback resolution to `@tobilu/qmd`, and adds focused unit coverage.

## Best Possible Solution

Land the no-shell absolute-entrypoint resolver and scoped QMD fallback after redacted Windows proof is attached, while keeping the shared and memory-host resolver behavior aligned.

## Reproduction Assessment

Yes, from source inspection: current main skips quoted absolute `.cmd` entrypoints because they lack a `%~dp0` token, then fails closed without shell execution. I did not run a live Windows reproduction in this read-only review.

## Solution Assessment

Mostly yes: extending the existing no-shell resolver and correcting the QMD package fallback is the narrow fix. The main improvement would be avoiding long-term drift between the shared resolver and the memory-host SDK copy if the package boundary permits it.
