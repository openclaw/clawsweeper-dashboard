## Summary

Keep open: the PR targets a real Windows QMD wrapper gap, but the current branch still has a blocking package-entrypoint regression, lacks the required changelog entry, and still has no attached real Windows behavior proof.

## What This Changes

The PR adds quoted absolute Windows `.cmd` shim parsing for QMD, switches QMD fallback lookup to `@tobilu/qmd`, duplicates the resolver into the memory-host SDK, and updates focused tests.

## Best Possible Solution

Land a rebased Windows-only fix that parses quoted absolute `.cmd` entrypoints, safely resolves or rejects the real `@tobilu/qmd` wrapper shape, keeps shared resolver behavior aligned, adds upstream-shaped tests and changelog, and includes redacted Windows proof.

## Reproduction Assessment

Yes, by source inspection. Current main only unwraps `%~dp0`-relative quoted `.cmd` shim targets, so a quoted absolute JS entrypoint falls through to unresolved-wrapper handling without shell execution.

## Solution Assessment

No, not as currently proposed. The absolute shim parser is the right narrow direction, but the scoped fallback must handle the real `@tobilu/qmd` `bin/qmd` wrapper safely, and the PR still needs changelog plus real Windows proof.
