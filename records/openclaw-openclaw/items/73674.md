## Summary

Keep open: the QMD Windows resolver fix is source-supported, but the current branch is not merge-ready because it lacks inspectable real Windows proof and now adds provider-returned image URL downloads that bypass the existing SSRF-guarded image transport.

## What This Changes

The branch adds Windows `.cmd` shim absolute-entrypoint resolution and scoped QMD fallback, updates QMD tests, replaces the memory-host SDK resolver re-export with a local copy, and adds OpenAI-compatible image URL response fetching.

## Best Possible Solution

Land the narrow QMD resolver and scoped package-name fix, keep the memory-host SDK on the shared resolver seam, and remove or SSRF-guard the unrelated image URL support before merge.

## Reproduction Assessment

Yes, source-level: current main skips quoted absolute `.cmd` entrypoints without `%~dp0` and then falls back to package `qmd` while the supported package is `@tobilu/qmd`. I did not establish a live Windows reproduction in this read-only review.

## Solution Assessment

No, not as-is. The QMD resolver direction is narrow, but the branch should drop or guard the unrelated image URL downloader and preserve the shared memory-host resolver before it can be considered the best fix.
