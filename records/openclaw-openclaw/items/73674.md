## What This Changes

The branch adds quoted absolute `.cmd` entrypoint parsing for QMD on Windows, switches QMD fallback lookup to `@tobilu/qmd`, and updates focused resolver and memory tests.

## Best Possible Solution

Apply the parser and scoped package fallback through the central plugin SDK resolver, keep the host SDK file bridged through `openclaw-runtime-io`, and merge only after redacted Windows runtime proof is added.

## Reproduction Assessment

Yes, by source inspection: current main only handles `%~dp0`-relative quoted shim targets and still passes package fallback name `qmd`, so the reported absolute `@tobilu/qmd` wrapper path falls through to the fail-closed error. I did not run a live Windows shell in this read-only review.

## Solution Assessment

No, not as currently diffed: the central parser and package-name changes are the right narrow fix, but replacing the host SDK bridge with a resolver copy is not the best maintainable path.
