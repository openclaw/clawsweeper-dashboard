## Summary

## What This Changes

The PR updates Windows QMD `.cmd` shim resolution in the plugin SDK and memory-host SDK, switches QMD package fallback lookup to `@tobilu/qmd`, and adds focused tests.

## Best Possible Solution

Land the Windows-only absolute-entrypoint parser after the QMD fallback unwraps or rejects the real upstream wrapper safely, covers both resolver call paths with upstream-shaped tests, and records the fix in the changelog.

## Reproduction Assessment

Yes, by source inspection and upstream package inspection. Current main ignores quoted absolute `.cmd` entrypoints, while the PR's scoped fallback maps the real `@tobilu/qmd` package to `bin/qmd`, a shell wrapper that the resolver would run with Node.

## Solution Assessment

No, not as currently proposed. The absolute `.cmd` parsing is the right narrow direction, but the scoped fallback must resolve the real JavaScript entrypoint or fail closed, and the user-facing fix needs a changelog entry.
