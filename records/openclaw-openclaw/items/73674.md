## Summary

Keep open. The Windows absolute `.cmd` shim parsing is useful, but the PR still resolves the real `@tobilu/qmd` package fallback to a POSIX `bin/qmd` wrapper as if it were a Node entrypoint, and it is missing the required changelog entry.

## What This Changes

The PR adds Windows absolute `.cmd` shim entrypoint parsing for QMD, changes QMD fallback lookup to `@tobilu/qmd`, expands the memory-host resolver path, and updates focused tests.

## Best Possible Solution

Land a Windows-only resolver update that accepts safe absolute JS/EXE shim targets, safely unwraps or rejects the real `@tobilu/qmd` `bin/qmd` wrapper, keeps plugin SDK and memory-host behavior aligned, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, by source inspection. Current main fails the quoted absolute `.cmd` shim path, and the PR’s scoped fallback issue is reproducible from the published `@tobilu/qmd@2.1.0` package metadata and tarball shape.

## Solution Assessment

No, not as currently proposed. The absolute shim parser is the right narrow direction, but the scoped package fallback must handle the real upstream wrapper before merge.
