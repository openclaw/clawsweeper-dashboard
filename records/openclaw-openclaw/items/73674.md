## Summary

Keep open: the Windows absolute `.cmd` parsing direction is valid, but the branch still changes QMD fallback to `@tobilu/qmd` without handling the real upstream `bin/qmd` shell wrapper, and it is missing both the required changelog entry and attached real behavior proof.

## What This Changes

The PR adds quoted absolute Windows `.cmd` shim parsing for QMD, changes QMD fallback package lookup to `@tobilu/qmd`, and updates focused memory/runtime tests.

## Best Possible Solution

Land a rebased Windows-only fix that parses quoted absolute `.cmd` entrypoints, keeps the memory-host resolver aligned with the shared helper, safely resolves or rejects the real `@tobilu/qmd` wrapper shape, adds upstream-shaped tests and a changelog entry, and includes redacted Windows proof.

## Reproduction Assessment

Yes, by source inspection. Current main only unwraps `%~dp0`-relative quoted shim targets, and the upstream `@tobilu/qmd@2.1.0` package shape makes the proposed scoped fallback path reproducibly unsafe without a live Windows run.

## Solution Assessment

No, not as currently proposed. The absolute shim parser is the right narrow direction, but the package fallback must handle `@tobilu/qmd/bin/qmd` correctly and the PR still needs changelog plus real runtime proof.
