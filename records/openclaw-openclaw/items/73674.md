## Summary

Keep open: the Windows QMD shim bug is real on current main, but this PR is not merge-ready because its scoped package fallback can resolve the real `@tobilu/qmd` POSIX launcher and run it through Node, and it lacks the required changelog entry.

## What This Changes

The PR adds quoted absolute Windows `.cmd` shim entrypoint parsing, switches QMD fallback package lookup to `@tobilu/qmd`, expands the memory-host resolver, and updates focused tests.

## Best Possible Solution

Land the absolute-entrypoint parser after QMD fallback safely recognizes a real JS/EXE entrypoint or rejects unsupported package wrappers, keeps SDK and memory-host behavior aligned, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows current main skips absolute quoted `.cmd` shim targets without `%~dp0`, and upstream package inspection shows the proposed scoped fallback can return a POSIX shell wrapper that the resolver treats as JavaScript.

## Solution Assessment

No, not as currently proposed. The absolute shim parser is the right narrow direction, but the scoped fallback and fixtures must match the real `@tobilu/qmd` package shape before this can be merged safely.
