## Summary

Keep open: the QMD Windows shim fix is plausible, but current head added unrelated OpenAI-compatible image URL fetching that bypasses OpenClaw’s guarded provider HTTP path, and the external PR still lacks inspectable real Windows behavior proof.

## What This Changes

The branch fixes Windows QMD `.cmd` shim resolution and scoped QMD fallback, adds tests, copies Windows spawn resolution into memory-host SDK, and also adds OpenAI-compatible image URL response parsing/fetching.

## Best Possible Solution

Keep the narrow QMD no-shell absolute shim and scoped fallback, remove or split the image URL feature unless it uses existing guarded provider HTTP plumbing, then land after redacted Windows proof is added.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main skips quoted absolute `.cmd` entrypoints without `%~dp0` and uses `packageName: "qmd"` while the supported package is `@tobilu/qmd`. I did not run a live Windows reproduction in this read-only review.

## Solution Assessment

No as-is: the QMD resolver change is the narrow maintainable direction, but the unrelated image URL support needs to be split or routed through the existing guarded provider HTTP path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83
