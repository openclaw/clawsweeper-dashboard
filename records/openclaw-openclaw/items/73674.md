## Summary

Keep this PR open: the QMD Windows shim fix is source-backed and points at a real current-main gap, but the latest head still lacks inspectable after-fix Windows proof and includes an unrelated provider-controlled image URL fetch that bypasses existing guarded HTTP policy.

## What This Changes

The PR updates QMD Windows `.cmd` shim resolution and scoped package fallback handling, adds related tests, and adds async OpenAI-compatible image URL response fetching.

## Best Possible Solution

Land the narrow QMD shim and scoped fallback fix only after redacted Windows runtime proof is added, and remove, split, or guard the image URL download path through provider HTTP policy before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main skips quoted absolute `.cmd` shim entrypoints without `%~dp0`, and QMD fallback still points at `qmd` while the documented package is `@tobilu/qmd`. I did not establish live Windows reproduction in this read-only Linux review.

## Solution Assessment

No as-is. The QMD resolver and scoped fallback changes are the maintainable direction, but the unrelated image URL fetch must be removed, split, or routed through guarded provider HTTP before this PR is mergeable.
