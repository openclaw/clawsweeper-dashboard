## Summary

Keep open: the Windows QMD shim bug is real and the absolute `.cmd` parser is useful, but this patch still resolves the real `@tobilu/qmd` `bin/qmd` shell wrapper as a Node entrypoint, lacks the required changelog entry, and has no inspectable real Windows proof.

## What This Changes

The PR parses quoted absolute Windows `.cmd` shim entrypoints for QMD, changes QMD fallback lookup to `@tobilu/qmd`, duplicates the Windows spawn resolver into the memory-host SDK, and adds focused tests.

## Best Possible Solution

Land a rebased Windows-only fix that parses quoted absolute `.cmd` entrypoints, safely maps or rejects `@tobilu/qmd`'s real `bin/qmd` wrapper shape in both call paths, adds upstream-shaped tests and changelog, and includes redacted Windows proof.

## Reproduction Assessment

Yes, by source inspection. Current main only unwraps quoted `%~dp0`-relative `.cmd` shim targets, so a `qmd.cmd` containing `node "E:\...\qmd.js" %*` reaches unresolved-wrapper handling without shell fallback.

## Solution Assessment

No, not as currently proposed. The absolute shim parser is the right narrow direction, but the scoped fallback must handle the real `@tobilu/qmd` `bin/qmd` wrapper safely, and the PR still needs changelog plus real Windows proof.
