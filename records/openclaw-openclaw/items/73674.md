## Summary

Keep open: current main still has the Windows QMD absolute `.cmd` shim gap, but this PR is not ready because the scoped package fallback can resolve `@tobilu/qmd` to its POSIX shell wrapper and the user-facing changelog entry is still missing.

## What This Changes

The PR adds quoted absolute Windows `.cmd` entrypoint parsing for QMD, switches QMD fallback lookup to `@tobilu/qmd`, expands the memory-host resolver path, and updates focused tests.

## Best Possible Solution

Land the Windows-only absolute-entrypoint parser after the QMD fallback safely unwraps or rejects the real `@tobilu/qmd/bin/qmd` wrapper, adds upstream-shaped tests in both call paths, keeps resolver behavior aligned, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, by source inspection. Current main only accepts `%~dp0`-relative quoted shim targets, and upstream `@tobilu/qmd@2.1.0` metadata/tarball shape makes the PR's scoped-fallback regression clear without needing a live Windows run.

## Solution Assessment

No, not as currently proposed. The absolute `.cmd` shim parsing is the right narrow direction, but the scoped fallback must match the real package shape instead of handing a POSIX shell wrapper to Node, and the required changelog entry is missing.
