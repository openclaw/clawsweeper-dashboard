## Summary

Keep open: current main still has the Windows QMD shim gap and this PR is aimed at the right resolver path, but the latest patch remains blocked by the real `@tobilu/qmd` package shape and a missing changelog entry.

## What This Changes

The PR adds quoted absolute Windows `.cmd` entrypoint parsing, switches QMD fallback lookup to `@tobilu/qmd`, expands memory-host resolver code, and updates focused tests.

## Best Possible Solution

Land the absolute-entrypoint parser after the scoped fallback safely unwraps or rejects `@tobilu/qmd/bin/qmd`, keeps SDK and host behavior aligned, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows absolute quoted QMD `.cmd` shims without `%~dp0` still reach unresolved-wrapper handling, and the PR regression is source-reproducible from the `@tobilu/qmd@2.1.0` package metadata and tarball.

## Solution Assessment

No, not as currently proposed. The quoted absolute shim parsing is the right narrow direction, but the scoped package fallback must match the real upstream package shape instead of treating its shell wrapper as JavaScript.
