## Best Possible Solution

Land this after the contributor adds redacted, inspectable Windows QMD runtime proof while keeping the fix limited to the Windows spawn resolver and QMD package-name fallback.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: extend the existing Windows wrapper resolver without enabling shell execution and correct the QMD package fallback name. The remaining blocker is proof, not a different code shape.
