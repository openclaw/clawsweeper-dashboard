## Best Possible Solution

## Solution Assessment

Mostly yes: extending the existing no-shell Windows resolver and correcting the scoped QMD fallback is the narrow maintainable fix. Keeping the memory-host SDK aligned through the shared resolver would be cleaner, but the copied resolver is not a blocking correctness issue here.
