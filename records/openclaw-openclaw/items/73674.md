## Summary

Keep open. Current main still fails the quoted-absolute QMD `.cmd` wrapper shape described in the PR, and the branch addresses that gap without an obvious blocking correctness or supply-chain regression; it still needs normal maintainer review, a rebase/current-main validation pass, and targeted Windows/QMD test proof before merge.

## What This Changes

The PR teaches the Windows spawn resolver to unwrap quoted absolute JS/EXE entrypoints from `.cmd` shims, changes QMD fallback package lookup to `@tobilu/qmd`, adds focused resolver coverage, and updates the memory-core QMD test fixture.

## Best Possible Solution

Land a narrow current-main fix that accepts existing quoted absolute JS/EXE targets from Windows QMD `.cmd` wrappers without shell execution, uses the scoped `@tobilu/qmd` package for fallback metadata lookup, and keeps the shared SDK and memory-host resolver behavior aligned with focused tests.

## Reproduction Assessment

Yes. A static current-main reproduction is to use a wrapper containing `node "E:\\...\\@tobilu\\qmd\\...\\qmd.js" %*`; the current parser has no `%~dp0` match for that token and reaches the unresolved-wrapper error path.

## Solution Assessment

Yes, with a merge-shape caveat. Parsing an existing absolute entrypoint and retaining fail-closed behavior is the narrow maintainable fix; the safer implementation should either keep one shared resolver source or document why the memory-host package needs a generated/local copy.
