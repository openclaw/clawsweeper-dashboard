## Summary

Keep this PR open: the QMD Windows shim fix is useful and source-reproducible on current main, but the current branch still needs inspectable real Windows behavior proof and contains an unrelated security-sensitive image URL fetch path that should not merge as-is.

## What This Changes

The PR updates QMD Windows `.cmd` shim and scoped package fallback resolution, and also adds async OpenAI-compatible image URL response fetching.

## Best Possible Solution

Keep the narrow QMD no-shell absolute shim and scoped fallback fix, split or guard the image URL support through provider HTTP policy, then land only after redacted Windows runtime proof is added.

## Reproduction Assessment

Yes. Source inspection on current main shows the quoted absolute `.cmd` shim path is skipped without `%~dp0`, and the QMD package fallback still targets `qmd` while docs and npm use `@tobilu/qmd`; I did not run live Windows validation in this read-only review.

## Solution Assessment

No as-is. The QMD resolver/fallback change is the narrow maintainable direction, but the unrelated image URL path must be removed, split, or routed through guarded provider HTTP before merge.
