Keep open: current main and v2026.6.10 still show the Slack presentation downgrade at source level, and two open linked PRs already claim the narrow fix. This should close only after one candidate lands or maintainers choose a replacement path.

## What This Changes

The issue reports that Slack `openclaw message send --presentation` downgrades portable presentation payloads to plain text and returns a text receipt instead of a card receipt.

## Best Possible Solution

Land one reviewed linked PR, or replace them with the same narrow fix: bridge Slack facade `presentationCapabilities` and `renderPresentation`, then make presentation receipts outrank fallback text while preserving text-only sends.

## Reproduction Assessment

Yes, at source level. Core falls back to text when the Slack facade lacks `renderPresentation`, and the receipt helper checks text before presentation; I did not run the live Slack CLI repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge the existing Slack renderer through the facade and correct receipt-kind precedence, which is what the linked candidate PRs target.
