## What This Changes

The issue reports that Slack `presentation` payloads are downgraded to plain text instead of rendering native Block Kit cards.

## Best Possible Solution

Land one canonical Slack outbound facade fix that forwards the existing presentation capabilities/renderer and correctly classifies rendered Slack card receipts, then close this issue after the merged fix is proven.

## Reproduction Assessment

Yes, at source level. The documented `--presentation` path reaches core rendering, current Slack's registered facade lacks the presentation renderer/capabilities that core expects, and receipt inference checks text before presentation for text-plus-presentation payloads.

## Solution Assessment

Yes. The maintainable fix is narrow: bridge the existing Slack presentation capabilities and renderer through the active Slack facade, then classify rendered native card data without adding a new API or config surface.
