## Summary

Keep open: current main still has the reported Slack presentation downgrade. The documented presentation path reaches core delivery, but the active Slack outbound facade lacks the presentation renderer/capabilities needed before core falls back to text.

## What This Changes

The issue reports that documented Slack `presentation` message payloads are delivered as plain text instead of Slack Block Kit cards.

## Best Possible Solution

Expose Slack's existing presentation capabilities and renderer through the active Slack channel outbound facade, preserving the existing plugin boundary and adding focused regression coverage for the CLI/core send path.

## Reproduction Assessment

Yes at the source level. The issue provides release evidence, and current main still has a path where `presentation` reaches core delivery but Slack's active handler has no renderer, so core sends fallback text.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge the existing Slack renderer/capabilities onto `slackChannelOutbound`; adding config, changing fallback semantics, or broadening the Slack-native Block Kit API would be unnecessary.
