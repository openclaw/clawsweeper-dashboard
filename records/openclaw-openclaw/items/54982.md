## Summary

Keep this PR open: current main still lacks the requested TUI preservation mode, and the closed feature request at https://github.com/openclaw/openclaw/issues/11669 points remaining work here. The branch still has source-proven preserve-mode edge-case bugs and is dirty against current main, so it needs repair before merge.

## What This Changes

The PR adds an optional `--preserve-intermediate` TUI flag, threads it through TUI options, changes ChatLog to append streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land an updated optional TUI preservation flag that tracks all preserved chunks per run, removes the whole run on discarded finals, dedupes unchanged final snapshots, documents the log-budget tradeoff, and rebases onto current TUI state.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: preserve mode appends multiple chunks while tracking only the newest one, and finalization appends another snapshot unconditionally. Current-main bug reproduction is not applicable because this is new feature work and main still has the existing in-place behavior.

## Solution Assessment

No. The opt-in CLI/TUI boundary is reasonable, but the implementation should track preserved chunks by run, remove them all on discarded finals, skip unchanged final snapshots, document capacity behavior, and rebase onto current main.

  - confidence: 0.8
