## Summary

Keep this PR open: current main still lacks the requested TUI preservation mode, and the closed feature issue explicitly points here as the canonical implementation candidate. The PR is not ready to merge because preserve mode leaves phantom partial replies after suppressed finals and can duplicate the final snapshot.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through `TuiOptions` into `ChatLog`, appends assistant stream snapshots in preserve mode, and adds focused CLI/docs tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving suppressed-output behavior, avoiding duplicate final entries, and documenting the log-budget trade-off in both TUI references.

## Reproduction Assessment

Yes for the PR-head defects: source inspection gives focused paths by enabling preserve mode, appending multiple deltas for one run, then either dropping the assistant on a suppressed final or finalizing with unchanged text. The central after-fix behavior is also shown by the PR body screenshots.

## Solution Assessment

No. The `ChatLog` option boundary is reasonable, but preserve mode should track all preserved components for a run, drop the full run when output is suppressed, skip unchanged final snapshots, and update both option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop the full preserved run when final output is suppressed:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, each delta appends a component but only the newest component is tracked. Existing missing-final and suppressed `(no output)` paths call `dropAssistant`, so this removes only the latest chunk and leaves earlier partial assistant text visible as a phantom reply.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.83
- **[P3] Document the flag in option references:** `docs/cli/tui.md:28-29`
  - body: The PR adds the new flag only to an example. The CLI options table and TUI guide option list still omit it and the per-delta log-budget trade-off, so users scanning references would not find the supported behavior.
  - confidence: 0.78
