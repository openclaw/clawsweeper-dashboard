## Summary

Keep this PR open but require changes before merge: current main still lacks the requested TUI preservation option, and this branch is the canonical implementation path, but the proposed ChatLog preserve mode can leave phantom partial replies and duplicate final snapshots.

## What This Changes

Adds a default-off `--preserve-intermediate` TUI CLI flag, threads it through TuiOptions into ChatLog, preserves streamed assistant snapshots as separate entries, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while keeping no-output suppression, avoiding duplicate finals, documenting the log-budget trade-off, and covering the behavior with focused tests.

## Reproduction Assessment

Yes for the PR-head defects: source inspection gives focused paths by enabling preserve mode, appending multiple deltas for one run, then either dropping the assistant on a suppressed final or finalizing with unchanged text. I did not execute tests because this review is read-only.

## Solution Assessment

No. The CLI/TuiOptions/ChatLog boundary is reasonable, but the implementation should track all preserved components for a run, drop the full preserved run on suppression, skip unchanged final snapshots, and update the option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop all preserved chunks when output is suppressed:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, each delta appends a component but `streamingRuns` tracks only the latest one. Existing final handlers call `dropAssistant` for missing finals and suppressed `(no output)`, so earlier chunks stay visible even though the run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.83
- **[P3] Document the option wherever users look it up:** `docs/cli/tui.md:28-29`
  - body: The PR adds the new flag only to an example. The CLI options table and TUI guide still omit the supported flag and the per-delta log-budget trade-off, so users scanning references would not find the behavior.
  - confidence: 0.78
