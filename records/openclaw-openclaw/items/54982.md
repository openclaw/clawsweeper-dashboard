## Summary

Keep open: current main still lacks the requested TUI preservation flag, and this PR is a plausible opt-in implementation, but the PR head has preserve-mode state bugs and is currently merge-conflicting against main.

## What This Changes

The PR adds a `--preserve-intermediate` TUI flag, wires it through CLI options into `ChatLog`, appends streamed assistant updates as separate entries, and adds docs plus tests.

## Best Possible Solution

Keep the opt-in flag direction if maintainers want the feature, but rebase over current TUI, track all preserved components per run, suppress unchanged duplicate finals, and document the bounded log budget.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: preserve mode appends multiple chunks while tracking only the latest component, and current final-event handling relies on `dropAssistant` to discard no-output runs. Current-main bug reproduction is not applicable because this is new feature work and main still has the documented in-place streaming behavior.

## Solution Assessment

No. The opt-in flag boundary is reasonable, but this branch should track every preserved component for a run, skip unchanged final snapshots, document the fixed log budget, and rebase cleanly before maintainer feature review.
