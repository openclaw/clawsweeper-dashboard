## What This Changes

The PR adds an opt-in `openclaw tui --preserve-intermediate` flag that appends streamed assistant TUI updates as separate chat-log entries, plus CLI wiring, tests, and docs.

## Best Possible Solution

Land the opt-in TUI preservation mode only after it preserves existing silent/no-output cleanup semantics, avoids duplicate terminal snapshots, and is rebased onto the current ChatLog/event-handler state with focused tests.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: current TUI final handling calls `dropAssistant` for silent/no-output finals, while the PR's preserve mode only tracks and removes the latest component. The attached screenshots demonstrate the requested visible behavior but not the cleanup edge cases.

## Solution Assessment

No, not as written. The opt-in flag is plausible, but the implementation needs to preserve existing no-output cleanup semantics, avoid duplicate final snapshots, and rebase onto current main before it is the best maintainable solution.
