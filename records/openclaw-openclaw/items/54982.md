## Summary

Keep open: current `main` still lacks the requested TUI preservation option, and this PR is useful but not yet correct because preserve mode only tracks the newest chunk for cleanup, duplicates unchanged finals, and under-documents the new log-budget behavior.

## What This Changes

Adds a default-off `--preserve-intermediate` TUI flag, TuiOptions/ChatLog plumbing, CLI docs, and focused tests to show intermediate assistant stream snapshots as separate entries.

## Best Possible Solution

Land a rebased default-off TUI option that preserves intermediate snapshots, drops all chunks for discarded runs, avoids duplicate unchanged finals, and documents the fixed log-budget tradeoff in both CLI and web TUI references.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either a missing/suppressed final or a final payload equal to the latest delta. Current `main` lacks the feature, so this is PR-behavior reproduction rather than a current-main bug reproduction.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks per run, remove them together when output is discarded, skip unchanged final snapshots, and update the docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Track all preserved chunks per run:** `src/tui/components/chat-log.ts:83-84`
  - body: In preserve mode, each delta appends a component but `streamingRuns.set` overwrites the run with only the newest one. Existing final paths call `dropAssistant` when output is absent or suppressed, so earlier chunks for that run remain visible as phantom assistant output.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:116-118`
  - body: `finalizeAssistant` always appends a new preserved component before clearing the run. When the final payload repeats the latest streamed snapshot, the log shows a duplicate trailing entry and burns the fixed component budget sooner.
  - confidence: 0.84
- **[P3] Document the preserve-mode tradeoff:** `docs/cli/tui.md:28-29`
  - body: The PR only adds the flag to an example while the web guide still says streaming responses update in place and neither reference option list describes the flag or log-budget tradeoff. Users scanning the reference would miss the new mode.
  - confidence: 0.8
