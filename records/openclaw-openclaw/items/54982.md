## Summary

Keep open: current main still lacks the requested TUI preservation mode and this PR remains the canonical implementation path for the closed feature request at https://github.com/openclaw/openclaw/issues/11669, but the patch needs focused repair before merge because preserve mode breaks existing no-output suppression and can duplicate final snapshots.

## What This Changes

Adds an opt-in `--preserve-intermediate` TUI flag, threads it through CLI/TUI options into `ChatLog`, updates CLI docs, and adds focused tests for preserving streamed assistant snapshots.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving no-output suppression, avoiding duplicate finals, and documenting the option in the public references.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows `ChatLog(40, { preserveIntermediate: true })` with two `updateAssistant` calls followed by `dropAssistant` leaves earlier chunks behind, and `finalizeAssistant` with unchanged final text appends a duplicate. For the requested feature itself, current main source and docs show the flag is not implemented yet.

## Solution Assessment

No. The TUI/`ChatLog` boundary is the right place for the option, but the implementation should track all preserved components for a run, drop the whole run when final output is suppressed, and skip unchanged final snapshots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop all preserved chunks for suppressed finals:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, each delta appends a component but only the newest component is tracked. Existing no-message and suppressed `(no output)` final paths call `dropAssistant`, so this removes only the latest chunk and leaves earlier partial assistant text visible as a phantom reply.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the final payload, producing duplicate trailing assistant entries and consuming the fixed log budget sooner.
  - confidence: 0.8
- **[P3] Document the flag in the options references:** `docs/cli/tui.md:28-29`
  - body: The PR adds an example for the new flag, but after rebasing onto current main the CLI and TUI options references also need the new option. Otherwise the public option lists will disagree with the command surface.
  - confidence: 0.72
