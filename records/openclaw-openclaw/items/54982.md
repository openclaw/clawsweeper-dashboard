## What This Changes

The PR adds an opt-in `openclaw tui --preserve-intermediate` mode that appends streaming assistant updates as separate TUI log entries, with CLI wiring, tests, and docs.

## Best Possible Solution

Land the opt-in TUI preservation mode only after it preserves existing silent/no-output semantics, avoids duplicate terminal snapshots, and is rebased onto the current ChatLog/event-handler state with focused tests.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: current TUI final handling calls `dropAssistant` for silent/no-output finals, while the PR's preserve mode only tracks and removes the latest component. The original request is a feature, and the attached screenshots demonstrate the visible requested behavior.

## Solution Assessment

No, not as written. The opt-in flag is plausible, but the implementation needs to preserve existing no-output cleanup semantics and avoid duplicate final snapshots before it is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Drop all preserved chunks for suppressed finals:** `src/tui/components/chat-log.ts:141-143`
  - body: `dropAssistant()` is the path used when a final event has no message or resolves to a suppressed `(no output)` placeholder. In preserve mode this keeps every earlier delta in the log and removes only the latest tracked component, so users can see phantom assistant text for a run the TUI intentionally treated as having no output. Track the preserved components per run and remove them all on drop.
  - confidence: 0.91
- **[P2] Avoid duplicating the unchanged final snapshot:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant()` always appends another component in preserve mode. For normal streams the final payload often repeats the latest displayed text, so this creates a duplicate trailing assistant entry and consumes the fixed log budget without preserving a new state. Reuse or skip the latest preserved component when the final text is unchanged.
  - confidence: 0.82
