## Summary

This PR should stay open but needs changes before merge: the opt-in TUI preservation path is useful, but the current branch can leave discarded partial output visible and can duplicate the final assistant snapshot. The attached screenshots are enough happy-path real behavior proof, and the security pass did not find a concrete supply-chain concern.

## What This Changes

Adds an opt-in `openclaw tui --preserve-intermediate` flag, threads it through TUI options into `ChatLog`, and adds docs plus focused CLI/ChatLog tests.

## Best Possible Solution

Land a default-off TUI flag once preserve mode removes all chunks for discarded runs, avoids unchanged final duplicates, and documents the fixed chat-log budget.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either an absent/suppressed final or a final payload equal to the latest streamed text.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in CLI/TUI boundary is reasonable, but the branch should track all preserved chunks for a run, remove the whole run on discarded finals, skip unchanged final snapshots, and document the log-budget tradeoff.
