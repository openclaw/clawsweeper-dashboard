## Summary

Keep this PR open: the requested TUI feature is still absent from current main, and the PR is useful, but its current implementation has a concrete preserve-mode suppression bug plus smaller polish gaps before it should merge.

## What This Changes

The PR adds an opt-in `--preserve-intermediate` TUI flag, threads it through CLI/TUI options into `ChatLog`, appends streamed assistant updates as separate log entries, and adds focused CLI docs and tests.

## Best Possible Solution

Land a rebased, default-off TUI option that records intermediate chunks without breaking no-output suppression or duplicating final snapshots, with focused tests plus user-facing docs and changelog coverage.

## Reproduction Assessment

Yes. A focused PR-head source repro can construct `ChatLog(40, { preserveIntermediate: true })`, call two `updateAssistant` calls for one run, then exercise `dropAssistant` or `finalizeAssistant` and observe either retained phantom partials or a duplicate final snapshot.

## Solution Assessment

No. The TUI/`ChatLog` boundary is the right place for the option, but the implementation should track all preserved components for a run, remove the whole run when output is suppressed, skip unchanged final snapshots, and preserve current default rendering semantics.
