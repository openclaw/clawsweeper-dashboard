## Summary

Keep open: current main still lacks the requested TUI preservation option and the closed feature request points here, but the PR-head implementation leaves discarded preserved chunks visible and duplicates unchanged finals, so it needs small ChatLog/docs repairs before maintainer feature review.

## What This Changes

The branch adds an opt-in `--preserve-intermediate` TUI flag, threads it into `ChatLog`, appends streamed assistant updates as separate entries, and adds CLI/docs/tests coverage.

## Best Possible Solution

Keep the opt-in CLI/TUI direction, but repair preserve-mode run ownership so discarded runs remove all preserved chunks, skip unchanged final snapshots, and document the bounded log budget before maintainer feature review.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: preserve mode appends multiple chunks while tracking only the latest component, and current final-event handling relies on `dropAssistant` to discard no-output runs. Current-main bug reproduction is not applicable because this is new feature work and main still has the documented in-place streaming behavior.

## Solution Assessment

No. The opt-in flag boundary is reasonable, but this branch should track every preserved component for a run, skip unchanged final snapshots, and document the fixed log budget before merge.
