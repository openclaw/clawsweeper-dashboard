## Summary

Keep open: current main still lacks this feature, the linked feature issue was closed in favor of this open PR, and the PR has source-reproducible preserve-mode defects that need repair before merge.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI CLI flag, threads it into `ChatLog`, preserves streaming assistant snapshots as separate entries, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the log-budget trade-off, and keeping growth bounded.

## Reproduction Assessment

Yes for the PR-head defects: source inspection gives focused paths by enabling preserve mode, appending multiple deltas for one run, then either dropping the assistant on a suppressed final or finalizing with unchanged text. I did not execute tests because this review is read-only.

## Solution Assessment

No. The `ChatLog` and TUI option boundary is reasonable, but the implementation should track preserved components for a run, drop the full preserved run when output is suppressed, skip unchanged final snapshots, and update both option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop the whole preserved run on suppressed output:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, every delta appends a component but only the newest component is tracked. Existing missing-final and suppressed `(no output)` paths call `dropAssistant`, so this removes only the latest chunk and leaves earlier partial assistant text visible as a phantom reply.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.83
- **[P3] Document the flag in the option references:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only to an example. The CLI options table and TUI guide option list still omit it and the per-delta log-budget trade-off, so users scanning references would not find the supported behavior.
  - confidence: 0.78
