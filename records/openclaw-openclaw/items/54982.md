## Summary

Keep this PR open: current main still lacks the requested TUI preserve-intermediate mode, and the closed feature request points here as the canonical implementation path. The branch needs changes before merge because preserve mode leaves phantom chunks for dropped runs, can duplicate unchanged final output, and only documents the flag in an example.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through TuiOptions/runTui/ChatLog, appends streamed assistant snapshots, and adds focused CLI docs and tests.

## Best Possible Solution

Land a rebased implementation that preserves intermediate snapshots while tracking all chunks per run, removes suppressed runs cleanly, avoids duplicate final snapshots, and documents the option plus log-budget trade-off in both CLI and web TUI references.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, append multiple deltas for one run, then either drop a suppressed final or finalize with text equal to the latest delta. Current main does not reproduce the new mode because the flag is not present there.

## Solution Assessment

No, not as currently implemented. The CLI/TUI option boundary is reasonable, but the branch should track preserved chunks per run, drop all chunks when output is suppressed, skip unchanged final snapshots, and update the reference docs before merge.
