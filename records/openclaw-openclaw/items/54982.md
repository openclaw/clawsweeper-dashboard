## Summary

Keep open: current main still lacks the requested `--preserve-intermediate` TUI behavior, and this PR is the canonical implementation path for the closed feature request, but the PR-head implementation still has narrow preserve-mode state and documentation defects that should be fixed before merge.

## What This Changes

Adds a default-off `--preserve-intermediate` TUI flag, threads it into `ChatLog`, preserves stream snapshots as separate log entries, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate snapshots while honoring no-output suppression, avoiding duplicate terminal snapshots, documenting the log-budget tradeoff, and covering those paths with focused tests.

## Reproduction Assessment

Yes for the PR-head defects: source inspection gives a focused path by enabling preserve mode, appending multiple deltas for one run, then either dropping the assistant on a missing/suppressed final or finalizing with unchanged text. I did not execute tests because this review is read-only.

## Solution Assessment

No. The CLI/TuiOptions/ChatLog boundary is reasonable, but the implementation should track all preserved components for a run, drop the full preserved run on suppression, skip unchanged final snapshots, and update both option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - body: In preserve mode, each delta appends a component but `streamingRuns` tracks only the newest one.   - body: The PR adds the new flag only to an example. The CLI options table and web TUI options list still omit the supported flag and the per-delta log-budget tradeoff, so users scanning references would not find the behavior.
  - confidence: 0.78
