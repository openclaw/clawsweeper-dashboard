## What This Changes

The PR adds an opt-in `openclaw tui --preserve-intermediate` CLI flag, threads it through TUI options, changes `ChatLog` to append streamed assistant snapshots, and adds docs plus focused tests.

## Best Possible Solution

Keep the opt-in TUI flag idea, but implement preserve mode as a per-run lifecycle that removes all preserved chunks for discarded runs and suppresses unchanged final duplicates while preserving current default TUI behavior.

## Reproduction Assessment

Yes. Source inspection on the PR head shows preserve mode appends multiple components while tracking only the latest run component, and the unchanged event handler calls `dropAssistant` for silent/no-output finals.

## Solution Assessment

No, not yet. The flag is a plausible narrow solution, but the implementation needs per-run cleanup, unchanged-final de-duplication, and a conflict-aware rebase before it is the maintainable fix.
