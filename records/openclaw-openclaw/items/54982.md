## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Remove all preserved chunks on dropped assistant runs:** `src/tui/components/chat-log.ts:141-143`
  - body: With `preserveIntermediate` enabled, every display update replaces `streamingRuns[runId]` with the newest component, but current main calls `dropAssistant` when a final has no message or when an external final resolves to suppressed `(no output)`. This path removes only the newest component and leaves earlier preserved chunks visible as phantom assistant output for a run that was intentionally discarded; track all preserved components for the run and remove the whole set on drop.
  - confidence: 0.9
- **[P2] Document the scrollback budget cost of preserved updates:** `src/cli/tui-cli.ts:21-24`
  - body: The new CLI help describes preserving stream updates in history, but the implementation appends a separate component for each changed display update under the existing fixed chat-log cap. Long streams can therefore evict older conversation entries much faster, so the flag docs/help should state that preserved updates consume scrollback or the implementation should batch/bound snapshots.
  - confidence: 0.78
