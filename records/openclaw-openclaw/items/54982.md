## Summary

Keep open: current main still lacks the requested TUI preservation feature, and this PR is the canonical implementation path for the closed feature request, but the PR head still has narrow preserve-mode correctness and docs gaps before merge.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it into `runTui`, makes `ChatLog` append streamed assistant snapshots, and adds focused CLI docs/tests.

## Best Possible Solution

Land a rebased default-off flag implementation that preserves intermediate snapshots without breaking no-output suppression, avoids duplicate final snapshots, documents the log-budget trade-off, and keeps current pending-user ChatLog behavior.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: enable preserve mode, append multiple deltas, then either drop a missing/suppressed final or finalize with text equal to the latest delta. I did not execute tests because this review is read-only.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the implementation should track all preserved chunks for a run, drop them together on suppression, skip unchanged final snapshots, and update the reference docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop every preserved chunk for suppressed runs:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, every delta appends a component but `streamingRuns` points only at the latest one. Existing final handlers call `dropAssistant` for missing finals and suppressed `(no output)`, so earlier chunks stay visible even though the run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-118`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document the flag in reference sections:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only to an example. The CLI options table and web TUI guide still omit the supported option and the per-delta log-budget trade-off, so users scanning references would not find the behavior.
  - confidence: 0.8
