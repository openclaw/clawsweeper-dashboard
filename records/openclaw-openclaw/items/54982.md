## Summary

Keep open: current main still lacks the requested TUI preservation flag, and the PR is useful, but PR head has source-reproducible preserve-mode state bugs that should be fixed before merge.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI/TUI options into `ChatLog`, updates CLI docs, and adds focused tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the option in CLI and web references, and keeping log growth bounded.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows preserve mode appends multiple assistant components for one run while `dropAssistant` can remove only the latest, and `finalizeAssistant` appends even when the final text matches the latest streamed text. I did not run tests because this was a read-only review.

## Solution Assessment

No. The `ChatLog`/TUI boundary is the right place for this option, but the implementation should track all preserved components for a run, drop the whole run when final output is suppressed, skip unchanged final snapshots, and update both option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Drop the full preserved run when final output is suppressed:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, each delta appends a component but only the newest component is tracked. Existing no-message and suppressed `(no output)` final paths call `dropAssistant`, so this removes only the latest chunk and leaves earlier partial assistant text visible as a phantom reply.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the final payload, producing duplicate trailing assistant entries and consuming the fixed log budget sooner.
  - confidence: 0.82
- **[P3] Document the flag in option references:** `docs/cli/tui.md:28-29`
  - body: The PR adds the new flag only to an example. The CLI options table and the TUI guide option list are the public references for supported flags, so users scanning those sections would not find it or its log-budget trade-off.
  - confidence: 0.75
