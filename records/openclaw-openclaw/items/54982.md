## Summary

Keep open: current main still lacks this opt-in TUI behavior, and the PR is a plausible implementation path, but PR head has preserve-mode state bugs that should block merge until repaired.

## What This Changes

The branch adds a `--preserve-intermediate` TUI option, threads it through CLI/TuiOptions/runTui/ChatLog, appends streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off implementation that preserves intermediate snapshots without regressing no-output suppression, duplicate prevention, bounded log behavior, or CLI/web references.

## Reproduction Assessment

Yes, by source inspection: enable preserve mode, append multiple deltas for one run, then either drop a suppressed final or finalize with text equal to the latest delta. I did not run tests because this review is read-only.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks for a run, drop them together when output is suppressed, skip unchanged final snapshots, and update both CLI and web references before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop every preserved chunk for suppressed runs:** `src/tui/components/chat-log.ts:81-84`
  - body: In preserve mode, each delta appends a new component but `streamingRuns` is overwritten with only the latest one. Existing final handlers call `dropAssistant` for missing finals and suppressed external `(no output)`, so earlier chunks remain visible as phantom assistant output even though the run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-118`
  - body: `finalizeAssistant` always appends a fresh preserved component before deleting the run. Normal streams can repeat the terminal payload, producing a duplicate trailing entry and consuming the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document the flag in reference sections:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only to an example. Current main has CLI and web TUI option/reference sections, and the web page still says streaming responses update in place until finalized, so users scanning references would miss the new mode and its per-delta log-budget trade-off.
  - confidence: 0.8
