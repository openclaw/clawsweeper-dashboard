## Summary

Keep this PR open because it is the canonical implementation path for a real TUI feature that current main still lacks, but the current branch should not merge as-is due to source-reproducible preserve-mode defects and a stale conflicting head.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI/TuiOptions/runTui/ChatLog, appends streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased default-off implementation that tracks all preserved chunks per run, drops them together when output is suppressed, skips unchanged final snapshots, and updates both CLI and web docs with the log-budget trade-off.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, append multiple deltas for one run, then either drop a suppressed final or finalize with text equal to the latest delta. I did not run tests because this review is read-only.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks for a run, drop them together when output is suppressed, skip unchanged final snapshots, update both CLI and web references, and rebase onto current main before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop all preserved chunks for suppressed runs:** `src/tui/components/chat-log.ts:81-84`
  - body: In preserve mode, each delta appends a new component but `streamingRuns` is overwritten with only the latest one.   - confidence: 0.91

  - body: `finalizeAssistant` always appends a fresh preserved component before deleting the run. Normal streams can repeat the terminal payload, producing a duplicate trailing entry and consuming the fixed chat-log budget sooner.
  - confidence: 0.84

  - body: The PR adds the flag only to an example. Current main has CLI and web TUI option/reference sections, and the web page still says streaming responses update in place until finalized, so users scanning references would miss the new mode and its per-delta log-budget trade-off.
  - confidence: 0.8
