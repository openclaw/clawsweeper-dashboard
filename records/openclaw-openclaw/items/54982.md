## Summary

Keep open: current main still lacks a TUI preservation option, and this PR is the canonical implementation path for the closed feature request, but the branch still has source-proven preserve-mode state bugs that should block merge.

## What This Changes

The PR adds a `--preserve-intermediate` TUI flag, threads it through TUI options, changes ChatLog preserve-mode rendering to append streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land the optional TUI flag after repairing preserve-mode run tracking/finalization, adding focused regression tests, and documenting the fixed log budget.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: preserve mode appends multiple chunks, tracks only the latest chunk for dropping, and appends the final text again. Current-main bug reproduction is not applicable because this is new feature work and main still has the existing in-place behavior.

## Solution Assessment

No. The opt-in CLI/TUI boundary is reasonable, but the branch should track all preserved chunks for a run, remove the whole run on discarded finals, skip unchanged final snapshots, and document the log-budget tradeoff.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Drop every preserved chunk for discarded runs:** `src/tui/components/chat-log.ts:83-84`
  - body: In preserve mode, every delta appends a component, but this map assignment keeps only the newest one. When a final event has no message or an external `(no output)` placeholder is suppressed, `dropAssistant` removes only that latest component and leaves earlier partial assistant text visible for a run that was explicitly discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots in preserve mode:** `src/tui/components/chat-log.ts:116`
  - body: `finalizeAssistant` always appends another preserved component before clearing the run. When the final text repeats the latest streamed snapshot, the TUI shows a duplicate trailing assistant entry and consumes the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document the preserve-mode log budget:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only as an example. The CLI options table and TUI guide should explain that each preserved stream update creates a log entry and can displace older history in long responses.
  - confidence: 0.8
