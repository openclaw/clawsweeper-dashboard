## Summary

Keep open: current main still lacks the requested TUI preservation mode, and the linked feature request was closed as superseded by this PR. The branch is useful but still has preserve-mode state defects that can leave discarded partial output visible, duplicate unchanged final snapshots, and under-document the new mode.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI/TuiOptions/runTui/ChatLog, appends streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased default-off implementation that preserves intermediate snapshots while preserving no-output drop semantics, avoiding duplicate finals, and documenting the log-budget tradeoff in CLI/web references.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either a no-message/suppressed `(no output)` final or a final payload equal to the latest delta. Current main lacks the feature, so this is a PR-behavior reproduction rather than a current-main bug repro.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks per run, remove them together when a run is discarded, skip unchanged final snapshots, and update the CLI and web references before merge.
