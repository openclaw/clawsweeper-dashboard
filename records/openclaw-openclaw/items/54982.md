## Summary

Keep open: current main still lacks the requested TUI preservation mode and the linked feature issue was intentionally routed here, but the branch would break existing no-output/drop semantics and needs narrow ChatLog/docs repair before merge.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI options into `runTui`/`ChatLog`, appends streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased default-off implementation that stores all preserved components per run, drops them when final output is intentionally suppressed, skips duplicate final snapshots, and documents the log-budget tradeoff.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either a missing/suppressed final or a final payload equal to the latest delta. Current main lacks the feature, so this is PR-behavior reproduction rather than a current-main bug repro.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks per run, remove them together when a run is discarded, skip unchanged final snapshots, and update the CLI and web references before merge.
