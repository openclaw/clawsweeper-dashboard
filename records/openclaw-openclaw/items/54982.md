## Summary

Keep open: current main still lacks the requested TUI preservation option and this PR is the canonical implementation path, but the PR-head implementation breaks existing no-output suppression and can duplicate final snapshots, so it needs a narrow repair before merge.

## What This Changes

Adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI/TUI options into `ChatLog`, appends streamed assistant snapshots as separate entries, updates CLI docs, and adds focused tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the option in CLI and web references, and keeping log growth bounded.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows preserve mode appends multiple assistant components for one run while `dropAssistant` can remove only the latest, and `finalizeAssistant` appends even when the final text matches the latest streamed text. I did not run tests because this was a read-only review.

## Solution Assessment

No. The TUI/`ChatLog` boundary is the right place for the option, but the implementation should track all preserved components for a run, drop the whole run when final output is suppressed, skip unchanged final snapshots, and update both option references.

  - body: The PR adds the new flag only to an example. The CLI options table and the TUI guide option list still omit it, so users scanning the public references would not find the supported flag.
