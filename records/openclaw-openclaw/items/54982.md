## Summary

Keep open: current main still lacks the requested TUI preservation mode and the related feature issue was closed in favor of this PR, but the PR head still has preserve-mode state bugs that can leave phantom partial replies and duplicate final entries before it is mergeable.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI CLI option, threads it through TUI options into `ChatLog`, documents an example, and adds focused tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate assistant snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the option in both references, and keeping log growth bounded.

## Reproduction Assessment

Yes for the PR-head defects: source inspection gives a focused path by enabling preserve mode, appending multiple deltas for one run, then either dropping the assistant on a suppressed final or finalizing with unchanged text. I did not execute tests because the review is read-only.

## Solution Assessment

No. The `ChatLog` and TUI option boundary is reasonable, but the implementation should track preserved components for a run, drop the full preserved run when output is suppressed, skip unchanged final snapshots, and update both option references.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Drop the full preserved run when output is suppressed:** `src/tui/components/chat-log.ts:141-143`
  - body: In preserve mode, each delta appends a component but only the newest component is tracked. Existing missing-final and suppressed `(no output)` paths call `dropAssistant`, so this removes only the latest chunk and leaves earlier partial assistant text visible as a phantom reply.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-119`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.82
- **[P3] Document the flag in option references:** `docs/cli/tui.md:28-29`
  - body: The PR adds the new flag only to an example. The CLI options table and TUI guide option list are the public references for supported flags, so users scanning those sections would not find it or its log-budget trade-off.
  - confidence: 0.75
