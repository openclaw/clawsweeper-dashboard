## Summary

Keep open. Current main still lacks the requested TUI preservation flag, and this PR is the canonical implementation path for the closed feature request, but the patch still has preserve-mode state defects around suppressed finals and duplicate final snapshots plus an incomplete docs update.

## What This Changes

The branch adds a default-off `--preserve-intermediate` TUI flag, wires it through TUI options into `ChatLog`, preserves streamed assistant snapshots as separate entries, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the log-budget trade-off, and covering those paths with focused tests.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: enable preserve mode, append multiple deltas, then either drop a missing/suppressed final or finalize with unchanged text. I did not execute tests because this review is read-only.

## Solution Assessment

No. The CLI/TuiOptions/ChatLog boundary is reasonable, but the implementation should track and drop the full preserved run on suppression, skip unchanged final snapshots, and document the option in the reference surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop every preserved chunk for suppressed runs:** `src/tui/components/chat-log.ts:137-140`
  - body: In preserve mode, each delta appends a component but `streamingRuns` tracks only the latest one. Existing final handlers call `dropAssistant` for missing finals and suppressed `(no output)`, so earlier chunks stay visible even though the run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:112-118`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.83
- **[P3] Document the new flag in references:** `docs/cli/tui.md:25-29`
  - body: The PR adds the flag only to an example. The CLI options table and web TUI guide still omit the supported option and the per-delta log-budget trade-off, so users scanning references would not find the behavior.
  - confidence: 0.78
