## Summary

Keep open: current main still lacks the requested TUI preservation option and the linked feature issue was closed in favor of this PR, but the PR head still has preserve-mode state bugs around discarded finals and duplicate final snapshots.

## What This Changes

This PR adds a default-off `openclaw tui --preserve-intermediate` option, passes it into `ChatLog`, appends streamed assistant updates as separate entries in preserve mode, and adds docs plus focused tests.

## Best Possible Solution

Land a default-off TUI flag whose `ChatLog` tracks preserved stream components per run, removes the whole preserved run on discarded finals, skips unchanged final duplicates, and documents the fixed log-budget tradeoff.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either an absent/suppressed final or a final payload equal to the latest streamed text. Current-main bug reproduction is not applicable because this is a new feature and current main only has the existing in-place behavior.

## Solution Assessment

No. The opt-in CLI/TUI boundary is reasonable, but the branch should track all preserved chunks for a run, remove the whole run on discarded finals, skip unchanged final snapshots, and update the current docs table before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track every preserved chunk for a run:** `src/tui/components/chat-log.ts:83-84`
  - body: In preserve mode, every delta appends a component, but this map assignment keeps only the newest one. When a final event has no message or an external `(no output)` placeholder is suppressed, `dropAssistant` removes only that latest component and leaves earlier partial assistant text visible for a run that was explicitly discarded.
  - confidence: 0.91
- **[P3] Skip unchanged preserved final snapshots:** `src/tui/components/chat-log.ts:115-118`
  - body: `finalizeAssistant` always appends another preserved component before clearing the run. When the final text repeats the latest streamed snapshot, the TUI shows a duplicate trailing assistant entry and consumes the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document the preserve-mode log budget:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only as an example. The current CLI options table and TUI guide should explain that each preserved stream update creates a log entry and can displace older history in long responses.
  - confidence: 0.8
