This PR adds a `--preserve-intermediate` TUI flag, threads it into `ChatLog`, appends streamed assistant updates as separate entries, and adds focused CLI/TUI tests plus docs.

## Best Possible Solution

Rebase over current main, keep the opt-in flag only if maintainers want this TUI mode, track all preserved components for a run, clear them on discarded/no-output finals, skip duplicate final snapshots, and document the bounded log-budget tradeoff.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: preserve mode appends multiple chunks while tracking only the latest component, and current final-event handling relies on `dropAssistant` to discard no-output runs. Current-main bug reproduction is not applicable because this is new feature work.

## Solution Assessment

No. The opt-in flag boundary is plausible, but the branch should track all preserved components for a run, skip unchanged final snapshots, document the fixed log budget, and rebase cleanly before maintainer review.
