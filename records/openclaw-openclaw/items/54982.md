## Summary

Keep open: current main does not implement this feature, and the PR still has a P2 preserve-mode state bug plus smaller docs/changelog gaps before it is mergeable.

## What This Changes

The PR adds an opt-in `--preserve-intermediate` TUI CLI flag, threads it into `ChatLog`, appends streamed assistant updates as separate entries, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI option that preserves intermediate chunks without breaking no-output suppression or duplicating final snapshots, with focused tests plus docs and changelog coverage.

## Reproduction Assessment

Yes for the PR-head defects. Source inspection shows `ChatLog(40, { preserveIntermediate: true })` followed by multiple `updateAssistant` calls and then `dropAssistant` or an unchanged `finalizeAssistant` will retain phantom partials or append a duplicate final snapshot.

## Solution Assessment

No. The TUI/`ChatLog` boundary is the right place for the option, but the implementation should track all preserved components for a run, remove the whole run when output is suppressed, skip unchanged final snapshots, and preserve current default semantics.
