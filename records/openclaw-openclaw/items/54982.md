## Summary

Keep this PR open: current main still lacks the requested preserve-intermediate TUI mode, but the PR head has concrete ChatLog/docs defects and is currently conflicting with main.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through `runTui` and `ChatLog`, appends streamed assistant snapshots, and adds focused CLI docs/tests.

## Best Possible Solution

Land a rebased default-off implementation that tracks preserved chunks per run, drops them together when output is discarded, skips unchanged final snapshots, and documents the log-budget tradeoff in CLI and web references.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either a missing/suppressed final or a final payload equal to the latest delta. Current main lacks the requested feature, so this is PR-behavior reproduction rather than a current-main bug reproduction.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks per run, remove them together when a run is discarded, skip unchanged final snapshots, update the docs, and rebase against current main.
