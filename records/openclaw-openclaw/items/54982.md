## Summary

Keep this PR open: current main still lacks the requested TUI preservation option, and this branch is the canonical implementation path, but the PR-head patch still has source-reproducible ChatLog preserve-mode defects that should block merge.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it through CLI options into `runTui`, appends streamed assistant snapshots in `ChatLog`, and adds focused docs/tests.

## Best Possible Solution

Land a rebased default-off flag implementation that preserves intermediate snapshots without breaking suppressed-output handling, avoids duplicate final snapshots, and documents the option in the CLI and web TUI references with the log-budget trade-off.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: enable preserve mode, append multiple deltas, then finalize through a missing/suppressed output path and earlier chunks remain because only the latest component is tracked. A same-text final path is also source-reproducible because preserve-mode finalization always appends a new component.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks for a run, drop them together when output is suppressed, skip unchanged final snapshots, and update the reference docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop every preserved chunk for suppressed runs:** `src/tui/components/chat-log.ts:81-84`
  - body: Preserve mode overwrites the run's tracked component on every delta, so `dropAssistant` only knows about the newest chunk. Existing final handlers call `dropAssistant` for missing finals and suppressed `(no output)`, leaving earlier chunks visible even though that run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:115-118`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal final payloads can repeat the terminal display text, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document the flag in reference sections:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only to an example. Current main has a CLI options table and web TUI options section that should include the supported option and the per-delta log-budget trade-off so users scanning references can find it.
  - confidence: 0.8
