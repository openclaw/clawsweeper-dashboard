## Summary

Keep open: current main still lacks the requested TUI preserve-intermediate mode, and the closed feature request points here as the remaining implementation path. The branch is not merge-ready because preserve mode only tracks the latest chunk for a run, can leave phantom intermediate output after no-output finals, duplicates unchanged final snapshots, and is currently conflict-marked against main.

## What This Changes

The PR adds a default-off `--preserve-intermediate` TUI flag, threads it into `runTui` and `ChatLog`, appends streamed assistant snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased default-off implementation on current main that tracks all preserved chunks per run, drops them together when output is discarded, skips unchanged final snapshots, and documents the log-budget tradeoff in CLI and web references.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either a missing/suppressed final or a final payload equal to the latest delta. Current main lacks the requested feature, so this is PR-behavior reproduction rather than a current-main bug reproduction.

## Solution Assessment

No. The CLI/TUI option boundary is reasonable, but the branch should track all preserved chunks per run, remove them together when a run is discarded, skip unchanged final snapshots, update the docs, and rebase against current main.
