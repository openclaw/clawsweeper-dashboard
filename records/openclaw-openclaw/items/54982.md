## Summary

Keep open: current main still lacks the requested TUI preservation option, and this PR is the canonical implementation path, but the branch has narrow preserve-mode correctness and docs gaps that should be fixed before merge.

## What This Changes

This PR adds a default-off `--preserve-intermediate` TUI flag, threads it through TUI options into `ChatLog`, appends preserved streaming snapshots, and adds focused docs/tests.

## Best Possible Solution

Land a rebased, default-off TUI flag that preserves useful intermediate snapshots while preserving no-output suppression, avoiding duplicate finals, documenting the log-budget trade-off, and covering those paths with focused tests.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: enable preserve mode, append multiple deltas, then finalize with either no message/suppressed `(no output)` or a final text equal to the latest snapshot. Existing current-main handlers already exercise the relevant `dropAssistant` paths.

## Solution Assessment

No. The CLI/TuiOptions/ChatLog boundary is reasonable, but the implementation should track all preserved components for a run when dropping suppressed output, skip unchanged final snapshots, and update the reference docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Drop every preserved chunk for suppressed runs:** `src/tui/components/chat-log.ts:137-140`
  - body: In preserve mode, each delta appends a component but `streamingRuns` tracks only the latest one. Existing final handlers call `dropAssistant` for missing finals and suppressed `(no output)`, so earlier chunks stay visible even though the run was intentionally discarded.
  - confidence: 0.91
- **[P3] Skip unchanged final snapshots:** `src/tui/components/chat-log.ts:112-118`
  - body: `finalizeAssistant` appends a fresh component in preserve mode even when the final text matches the latest streamed text. Normal streams can repeat the terminal payload, producing duplicate trailing entries and consuming the fixed chat-log budget sooner.
  - confidence: 0.83
- **[P3] Document the flag in reference tables:** `docs/cli/tui.md:25-26`
  - body: The PR adds the new flag only to an example. The CLI options table and web TUI guide still omit the supported option and the per-delta log-budget trade-off, so users scanning references would not find the behavior.
  - confidence: 0.78
