## Summary

Keep this PR open: current main still lacks the requested opt-in TUI preservation mode, and the linked feature issue was closed in favor of this PR, but the patch should not merge until preserved assistant runs clean up suppressed finals and avoid duplicate terminal chunks.

## What This Changes

The PR adds an opt-in `openclaw tui --preserve-intermediate` flag, threads it through TUI options, changes `ChatLog` to append streamed assistant snapshots, and adds docs plus focused tests.

## Best Possible Solution

Keep the opt-in TUI flag idea, but implement preserve mode as a per-run lifecycle that can remove all preserved chunks for discarded runs and suppress unchanged final duplicates while preserving current default TUI behavior.

## Reproduction Assessment

Yes, by source inspection: in preserve mode, multiple updates for one run leave only the latest component tracked, while final no-message and `(no output)` paths call `dropAssistant`. I did not run tests because this was a read-only review, but the failing paths are direct from the PR head and current event-handler contract.

## Solution Assessment

No, not yet: the flag is a plausible narrow solution, but the current implementation treats each delta as an independent final log entry without enough per-run cleanup. The safer path is to keep the opt-in flag while repairing run tracking, final de-duplication, and rebase drift.
