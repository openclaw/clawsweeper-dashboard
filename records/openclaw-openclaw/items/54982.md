## Summary

Keep open: current main still lacks the requested TUI preservation option, and the closed feature request points to this PR as the canonical implementation path. The PR is useful, but the current branch still has localized preserve-mode correctness blockers and a docs gap before merge.

## What This Changes

The PR adds a default-off `openclaw tui --preserve-intermediate` flag, threads it into `ChatLog`, and adds docs plus focused CLI/ChatLog tests.

## Best Possible Solution

Land a rebased default-off TUI flag that preserves intermediate chunks while preserving discarded-output suppression, avoiding duplicate final snapshots, and documenting the fixed log-budget tradeoff.

## Reproduction Assessment

Yes, for the PR-head defects by source inspection: enable preserve mode, stream multiple deltas, then receive either an absent/suppressed final or a final payload equal to the latest streamed text. Current-main bug reproduction is not applicable because this is a new feature and current main only has the existing in-place behavior.

## Solution Assessment

No. The opt-in CLI/TUI boundary is reasonable, but the branch should track all preserved chunks for a run, drop the whole preserved run on discarded finals, skip unchanged final snapshots, document the tradeoff, and rebase before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track every preserved chunk for a run:** `src/tui/components/chat-log.ts:83-84`
  - body: In preserve mode, every delta appends a component, but this map assignment keeps only the newest one. When a final event has no message or an external `(no output)` placeholder is suppressed, `dropAssistant` removes only that latest component and leaves earlier partial assistant text visible for a run that was explicitly discarded.
  - confidence: 0.91
- **[P3] Avoid duplicate final snapshots:** `src/tui/components/chat-log.ts:116`
  - body: `finalizeAssistant` always appends another preserved component before clearing the run. When the final text repeats the latest streamed snapshot, the TUI shows a duplicate trailing assistant entry and consumes the fixed chat-log budget sooner.
  - confidence: 0.84
- **[P3] Document preserve-mode log budget:** `docs/cli/tui.md:28-29`
  - body: The PR adds the flag only as an example. The CLI options table and TUI guide should explain that each preserved stream update creates a log entry and can displace older history in long responses.
  - confidence: 0.8
