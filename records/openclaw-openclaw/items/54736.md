## Summary

Keep open. Current main and v2026.5.18 still classify only one exact silent token, so doubled token-only payloads can normalize down to a visible `NO_REPLY`; https://github.com/openclaw/openclaw/pull/73153 is an open broader candidate but it has not merged or closed this issue.

## What This Changes

The issue requests that OpenClaw suppress doubled token-only silent replies such as `NO_REPLY\n\nNO_REPLY` instead of delivering leftover control text to channel plugins.

## Best Possible Solution

Update the shared silent-token predicate or normalizer so token-only repeated silent markers are suppressed, while preserving substantive trailing-token behavior and coordinating with the active reasoning-prefixed PRs.

## Reproduction Assessment

Yes, at source level. Current main still uses the exact-token regex and a read-only regex check shows `NO_REPLY\n\nNO_REPLY` misses exact suppression and strips to a visible `NO_REPLY`.

## Solution Assessment

Yes for the narrow doubled-token case. The maintainable fix belongs in the shared token classifier or normalization contract, with tests that preserve the existing substantive-reply protections.
