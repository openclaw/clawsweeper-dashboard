- **[P1] Gate proof logging behind operator-owned opt-in:** `extensions/telegram/src/runtime-proof.ts:88-94`
  - body: `resolveTelegramRuntimeProofRunId` accepts run IDs from inbound text, and dispatch passes raw Telegram message fields into that resolver.   - body: `hashTelegramRuntimeProofId` emits a deterministic unsalted SHA-256 prefix for session and message identifiers. Telegram IDs and session keys remain linkable across proof runs and may be guessable for low-entropy values, so use a run-scoped salt, HMAC, or opaque per-run nonce.
  - confidence: 0.88

  - body: The branch appends a new Telegram-specific JSONL proof file even though current diagnostics already define a share-safe redaction and export contract. Extend that diagnostics path or prove why it is insufficient before adding a second retention and access surface.
