## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Use non-linkable proof identifiers:** `extensions/telegram/src/runtime-proof.ts:113`
  - body: These proof records are meant to be safe to share, but a deterministic unsalted 12-hex SHA over Telegram/session/message identifiers remains stable across runs and can be brute-forced for low-entropy numeric IDs. Use a run-scoped salt/HMAC or opaque per-run nonce so proof can correlate events without exposing stable identifiers.
  - confidence: 0.84
- **[P2] Use a reviewed OpenClaw proof opt-in:** `extensions/telegram/src/runtime-proof.ts:84-94`
  - body: This makes STOMME_E2E_RUN_ID, inbound run_id message text, and e2e config slugs production proof triggers. Keep harness-specific triggers outside the runtime or replace them with a documented OpenClaw-owned opt-in so real users cannot accidentally enable proof artifact logging.
  - confidence: 0.86
