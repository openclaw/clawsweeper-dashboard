## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Use non-linkable proof identifiers:** `extensions/telegram/src/runtime-proof.ts:113`
  - body: These proof records are meant to be safe to share, but a deterministic unsalted 12-hex SHA over Telegram/session/message identifiers remains stable across runs and can be brute-forced for low-entropy numeric IDs. Use a run-scoped salt/HMAC or opaque per-run nonce so proof can correlate events without exposing stable identifiers.
  - confidence: 0.84
- **[P2] Remove the STOMME-only runtime contract:** `extensions/telegram/src/runtime-proof.ts:84`
  - body: This makes STOMME_E2E_RUN_ID a production Telegram runtime trigger. OpenClaw env surfaces need an explicit product-owned contract; a project-specific harness name should stay in the harness or become a reviewed OpenClaw-owned opt-in.
  - confidence: 0.9
