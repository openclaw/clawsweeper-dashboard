## Summary

Keep open: current main still performs the central `openai-codex/*` to `openai/*` doctor rewrite without an auth-credential precondition, and the linked fix PR is still open/draft rather than merged or shipped.

## What This Changes

The issue asks `openclaw doctor --fix` to preserve or warn on active `openai-codex/*` routes on Codex-OAuth-only hosts instead of automatically rewriting them to `openai/*` without proving the target auth and billing route.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79494 or an equivalent auth-credential-gated doctor repair that skips or warns on Codex-OAuth-only hosts, keeps the runtime-pin cleanup, and adds regression coverage plus docs before closing this as fixed.

## Reproduction Assessment

Yes for the remaining current-main behavior from source: `maybeRepairCodexRoutes` still rewrites `openai-codex/*` to `openai/*` under `--fix`. The older release-reported `agentRuntime.id="codex"` mutation is no longer reproduced by current tests, which assert the runtime pin stays absent.

## Solution Assessment

No, current main is not the best end state for the reported OAuth-only fleet because it still mutates the working route without credential proof. The linked draft PR's credential-store precondition is the narrower maintainable direction, pending maintainer acceptance of the route policy.
