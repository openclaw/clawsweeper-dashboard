## Summary

Keep open: current main still intentionally rewrites `openai-codex/*` model refs to `openai/*` during `doctor --fix`. The release-reported whole-agent `agentRuntime.id="codex"` mutation is no longer present on current main, but the remaining auth/billing safety question is unresolved and is corroborated by related live reports.

## What This Changes

The issue asks `openclaw doctor --fix` to preserve or warn on `openai-codex/*` routes unless it can prove the rewritten `openai/*` route will use the intended Codex OAuth profile.

## Best Possible Solution

Keep the issue open and align doctor, docs, and runtime policy so route repair only mutates Codex OAuth routes when the resulting auth/profile selection is proven safe; otherwise doctor should warn and leave the route unchanged.

## Reproduction Assessment

Yes for the remaining current-main behavior: source and tests show `doctor --fix` still rewrites `openai-codex/*` to `openai/*`. The exact release-reported runtime pin is no longer reproduced on current main because tests now assert no whole-agent `agentRuntime.id` is written.

## Solution Assessment

Unclear as an automated fix. A narrow doctor precondition or warning path could be maintainable, but maintainers first need to confirm the intended boundary between canonical `openai/*` routing and fail-closed Codex OAuth billing/auth safety.
