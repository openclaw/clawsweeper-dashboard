## Summary

Keep open: this PR targets a real Matrix reasoning-message loss bug, but the patch is too narrow because current generic dispatch still suppresses explicit reasoning before Matrix delivery sees it, and it also turns unflagged reasoning-looking fallback text into visible Matrix notices.

## What This Changes

The PR changes Matrix reply delivery and Matrix send options so reasoning-like Matrix replies are sent as `m.notice`, with Matrix reply tests updated for notice delivery.

## Best Possible Solution

Use a narrow Matrix dispatch opt-in that forwards only explicit `isReasoning` payloads as `m.notice`, preserves raw reasoning-looking suppression and `/reasoning off`, covers final/block/draft paths, and includes real runtime proof.

## Reproduction Assessment

Yes. Source inspection shows `/reasoning on` can produce `isReasoning` replies, while current generic dispatch and Matrix delivery both suppress them before send; I did not run a live Matrix homeserver reproduction.

## Solution Assessment

No. The PR changes only the Matrix delivery helper after generic dispatch and broadens the fallback layer; the safer fix is an explicit Matrix opt-in through dispatch while keeping unflagged reasoning-looking text hidden.
