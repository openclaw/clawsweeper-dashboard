## Summary

Keep open: the bounded-read hardening is useful, but this branch is not merge-ready because it bypasses the existing shared provider JSON helper, lowers some provider JSON success paths to a new 1 MiB cap, wraps some overflow failures as malformed JSON, and still lacks representative changed-module proof.

The PR replaces direct JSON response parsing with file-local 1 MiB `readResponseWithLimit` wrappers across Feishu, Browser, MS Teams, Azure Speech, Bedrock Mantle, Google Chat, HuggingFace, and Perplexity, and adds helper-level proof tests/scripts.

## Best Possible Solution

Use `readProviderJsonResponse` or one maintainer-approved shared JSON cap helper, preserve the existing provider cap unless maintainers explicitly choose otherwise, remove duplicated local parsers, and add representative module-path proof.
