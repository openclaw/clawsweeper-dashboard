## Summary

Keep this PR open, but it is not merge-ready: the hardening goal is valid, while the branch still duplicates the existing shared provider JSON reader, changes some established 16 MiB success-response caps to 1 MiB, and hides overflow errors behind malformed/empty-response handling.

## What This Changes

## Best Possible Solution

Use `readProviderJsonResponse` or one maintainer-approved shared JSON cap helper, preserve the existing provider cap unless maintainers explicitly choose another contract, remove duplicated local parsers, and add representative module-path proof.
