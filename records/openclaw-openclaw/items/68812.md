## Summary

Keep open. Current main and the latest stable release still hardcode memory embedding retry attempts, base delay, and max delay; only inline/non-batch indexing concurrency is configurable today, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests openclaw.json knobs for memory embedding concurrency and retry/backoff settings so rate-limited embedding providers can be tuned without patching bundled code.

## Best Possible Solution

Review, fix, and land or replace https://github.com/openclaw/openclaw/pull/68822 with validated agent-scoped `memorySearch` retry/backoff settings that preserve the existing `remote.nonBatchConcurrency` path and update schema, types, help, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows retry attempts, base delay, and max delay are still hardcoded, while the strict memory search config currently exposes only `remote.nonBatchConcurrency` for this area.

## Solution Assessment

Unclear as proposed. Configurable retry/backoff is reasonable, but the maintainable solution should extend the existing resolved `agents.*.memorySearch` contract instead of adding a competing or untyped config location.
