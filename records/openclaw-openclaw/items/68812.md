## Summary

Keep open. Current main only exposes non-batch indexing concurrency; the retry max-attempt, base-delay, and max-delay values remain hardcoded, and #68822 is still an open, unmerged closing PR for this issue.

## What This Changes

Expose configurable memory embedding indexing concurrency and retry backoff knobs in `openclaw.json` for rate-limited embedding providers.

## Best Possible Solution

Extend the existing `agents.defaults.memorySearch` contract with validated retry tuning fields, wire them into memory-core retry behavior, and land the linked PR or a successor with schema, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the missing configuration path: retry values are constants in memory-core and the runtime schema/docs expose only concurrency and batch/timeout options. A live Gemini 429 is not needed to verify the requested knobs are unavailable.

## Solution Assessment

Unclear as originally proposed. Configurable retry behavior is the right direction, but the maintainable shape should align with the current `agents.defaults.memorySearch` config surface before merge.
