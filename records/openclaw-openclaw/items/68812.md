## Summary

Keep open. Current main covers the concurrency part through the existing `agents.defaults.memorySearch.remote.nonBatchConcurrency` contract, but retry/backoff tuning is still hardcoded and the issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/68822.

## What This Changes

Request to make memory embedding indexing concurrency and retry/backoff parameters configurable in `openclaw.json` for rate-limited embedding providers.

## Best Possible Solution

Extend the existing agent-scoped `agents.defaults.memorySearch` contract with typed, validated retry/backoff settings, keep the shipped non-batch concurrency path, update docs/tests, and resolve the linked implementation PR.

## Reproduction Assessment

Yes. Source inspection on current main proves the remaining gap: retry/backoff values are still hardcoded and no typed config/schema/docs path exposes them, while concurrency is already covered through `remote.nonBatchConcurrency`.

## Solution Assessment

Unclear as proposed. Configurable retry/backoff is a reasonable direction, but the maintainable solution should align with the existing `agents.defaults.memorySearch` contract instead of adding a competing config path.
