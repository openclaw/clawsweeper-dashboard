Keep open. Current main only covers the concurrency part through the existing memorySearch contract; retry/backoff tuning is still unavailable, and the open closing PR should be reviewed or replaced before this issue closes.

The issue requests configurable memory embedding concurrency and retry/backoff settings for rate-limited embedding providers.

## Best Possible Solution

Add typed, validated retry/backoff settings to the existing agent-scoped memorySearch contract, keep the shipped non-batch concurrency path, update docs/tests, and land or replace the open closing PR.

## Solution Assessment
