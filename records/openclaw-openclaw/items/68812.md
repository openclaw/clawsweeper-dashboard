## Summary

Keep open. Current main exposes non-batch embedding concurrency, but retry attempts/base delay/max delay are still hardcoded, and #68822 is an open, unmerged closing PR for the remaining work.

## What This Changes

Request to make memory embedding indexing concurrency and retry backoff parameters configurable instead of hardcoded for rate-limited embedding providers.

## Best Possible Solution

Extend the existing `agents.defaults.memorySearch` contract with validated retry tuning fields, wire them into memory-core retry behavior, and land #68822 or a successor with schema, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the missing configuration path: concurrency is configurable, but retry max attempts, base delay, and max delay remain constants with no schema or docs fields. A live Gemini 429 is not needed to verify the requested knobs are unavailable.

## Solution Assessment

Unclear as originally proposed. Configurable retry behavior is the right direction, but the maintainable fix should align with the current `agents.defaults.memorySearch` config surface and be resolved through #68822 or a successor PR.
