Keep open. Current main has configurable non-batch embedding concurrency, but retry max attempts, base delay, and max delay remain hardcoded, and #68822 is still an open unmerged PR that explicitly targets this issue.

Request to make memory embedding indexing concurrency and retry backoff parameters configurable for rate-limited embedding providers.

## Best Possible Solution

Extend the existing agents.defaults.memorySearch contract with validated retry tuning fields, wire them into memory-core retry behavior, and land #68822 or a successor with schema, docs, changelog, and focused tests.
