## Summary

Keep open. Current main and the latest release cover the concurrency part through `agents.defaults.memorySearch.remote.nonBatchConcurrency`, but embedding retry attempts/base delay/max delay are still hardcoded; the open closing PR at https://github.com/openclaw/openclaw/pull/68822 should be reviewed or replaced before this issue closes.

## What This Changes

Request to add configurable memory embedding concurrency and retry/backoff parameters for rate-limited embedding providers.

## Best Possible Solution

Extend the existing agent-scoped `memorySearch` schema, resolver, docs, and tests with typed retry/backoff settings while preserving `remote.nonBatchConcurrency`, then land or replace https://github.com/openclaw/openclaw/pull/68822.

## Reproduction Assessment

Yes, as a source-reproducible missing config surface. Current main still passes hardcoded retry constants into the memory embedding retry loop, while schema/resolved config/docs expose only `remote.nonBatchConcurrency` for this area.

## Solution Assessment

Unclear. Configurable retry/backoff is a reasonable direction, but the maintainable fix should align with the existing agent-scoped `memorySearch` contract rather than adding a competing config path.
