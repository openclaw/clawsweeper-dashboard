## Summary

Keep open. Current main already exposes non-batch memory embedding concurrency through `agents.defaults.memorySearch.remote.nonBatchConcurrency`, but retry max-attempt/base-delay/max-delay values remain hardcoded, and the direct closing PR at https://github.com/openclaw/openclaw/pull/68822 is still open and unmerged.

## What This Changes

Request to expose memory embedding indexing concurrency and retry backoff parameters as validated `openclaw.json` settings for rate-limited embedding providers.

## Best Possible Solution

Add typed, validated retry/backoff settings to the existing agent-scoped `memorySearch` contract, keep the shipped `remote.nonBatchConcurrency` path for concurrency, update docs/tests, and land or replace the open closing PR.

## Reproduction Assessment

Yes. Source inspection against current main verifies that retry tuning remains unavailable in schema/resolved config and hardcoded in memory-core; a live Gemini 429 is not needed to prove the missing knobs.

## Solution Assessment

Unclear. Configurable retry/backoff is a reasonable direction, but the maintainable solution should align with the existing `agents.defaults.memorySearch` contract rather than adding an unvalidated or competing config path.
