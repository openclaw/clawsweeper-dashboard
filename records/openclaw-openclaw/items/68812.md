## Summary

Keep open. Current main has configurable non-batch embedding concurrency, but the embedding retry attempt/base-delay/max-delay values remain hardcoded, and #68822 is still an open, unmerged closing PR for this issue.

## What This Changes

The issue requests configurable memory embedding concurrency and retry backoff settings so rate-limited providers such as Gemini can avoid repeated 429 failures without patching bundled code.

## Best Possible Solution

Extend the existing `agents.defaults.memorySearch` contract with validated retry tuning fields, wire them into memory-core retry behavior, and land #68822 or a successor with schema, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the missing retry configuration path: retry values are constants in memory-core and the runtime schema/docs expose only concurrency and batch/timeout options. A live Gemini 429 is not needed to verify the requested knobs are unavailable.

## Solution Assessment

Unclear as originally proposed. Configurable retry behavior is the right direction, but the maintainable shape should align with the current `agents.defaults.memorySearch` config surface before merge.
