## Summary

Keep open. Current main and v2026.5.12 cover the concurrency portion through `agents.defaults.memorySearch.remote.nonBatchConcurrency`, but embedding retry attempts, base delay, and max delay are still hardcoded; the linked closing PR at https://github.com/openclaw/openclaw/pull/68822 should be reviewed, repaired, or replaced before this issue closes.

## What This Changes

The issue requests configurable memory embedding concurrency and retry/backoff settings in `openclaw.json` for rate-limited embedding providers.

## Best Possible Solution

Extend the existing agent-scoped `memorySearch` schema, resolver, docs, and tests with validated retry/backoff settings while preserving the current `remote.nonBatchConcurrency` contract, then land or replace https://github.com/openclaw/openclaw/pull/68822.

## Reproduction Assessment

Yes, as a source-reproducible missing config surface. Current main and v2026.5.12 still pass hardcoded retry constants into the memory embedding retry loop while exposing only the existing non-batch concurrency knob.

## Solution Assessment

Unclear as proposed. Configurable retry/backoff is reasonable, but the maintainable path should extend the existing agent-scoped `memorySearch` contract rather than adding a competing or untyped config location.
