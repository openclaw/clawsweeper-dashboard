## Summary

Keep open. Current main exposes non-batch indexing concurrency, but embedding retry attempts, base delay, and max delay are still hardcoded, and #68822 is still an open unmerged closing PR for this issue.

## What This Changes

The issue requests configurable memory embedding indexing concurrency and retry backoff parameters for rate-limited embedding providers.

## Best Possible Solution

Extend the existing `agents.defaults.memorySearch` contract with validated retry tuning fields, then land #68822 or a successor with schema, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the missing configuration path: retry values are constants in memory-core and the runtime schema/docs expose only concurrency and batch/timeout options.

## Solution Assessment

Unclear as originally proposed. Configurable retry behavior is the right direction, but the maintainable shape should align with the current `agents.defaults.memorySearch` config surface before merge.
