## What This Changes

Investigate and reduce high CPU/load on small Linux hosts during native Codex runtime runs, especially hook relay process fanout and concurrent gateway session/history work.

## Best Possible Solution

Profile the low-core native Codex path and add measured default admission/backpressure or a cheaper relay execution path while preserving existing native hook semantics and session/history bounds.

## Reproduction Assessment

Unclear whether we have a high-confidence full reproduction on current main. The issue provides concrete live evidence and source inspection confirms the named hook relay and gateway session/history paths, but I did not run a current-main 2-vCPU Codex stress scenario.

## Solution Assessment

Unclear whether the current mitigations are enough. The direct relay bridge and lightweight session/history work are good partial fixes, but a maintainable final fix needs profiling before choosing relay backpressure, batching, lower-cost execution, or broader gateway workload admission.
