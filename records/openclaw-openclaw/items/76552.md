## Summary

Keep open. The report is recent and strengthened by 2026.5.3 A/B controls; current main still installs synchronous Codex command hooks that shell through `openclaw hooks relay` for `post_tool_use`, and the session/history mitigations are bounded but do not prove this combined low-core saturation path is fixed.

## What This Changes

The issue requests reducing CPU fan-out and gateway event-loop pressure when native Codex runtime tasks trigger hook relay plus session/history/status work on low-core hosts.

## Best Possible Solution

Profile the low-core native Codex path, then add measured default backpressure or cheaper relay execution for Codex native hook events while preserving plugin hook and permission semantics.

## Reproduction Assessment

No high-confidence current-main reproduction was run. The issue provides detailed low-core Linux samples and A/B controls, and source inspection confirms the named per-event relay path, but I did not establish a deterministic current-main Codex stress reproduction.

## Solution Assessment

Unclear. Default backpressure or a cheaper in-process/pooled relay likely belongs in the Codex native hook path, but maintainers need profile data before choosing throttling, batching, disabling `PostToolUse`, or broader gateway admission control.
