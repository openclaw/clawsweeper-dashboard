Full review comments:

- **[P1] Enforce Slack limits before reading the body:** `src/slack/monitor/provider.ts:236`
  - body: The new handler buffers the network request with readRequestBody(req) before installing the 1 MB / 30 s guard, so a large or slow unsigned request can consume memory or hold the socket before limits apply. Use the existing limited reader for the initial preread or avoid prereading.
  - confidence: 0.94
- **[P2] Replay Slack requests through a real readable wrapper:** `src/slack/monitor/provider.ts:258-268`
  - body: mockReq is created from the already-consumed original request, so readable state and getters still come from that drained stream. If Bolt or raw-body checks stream state before reading, it can skip the replayed body; build the request around the PassThrough and copy only HTTP fields.
  - confidence: 0.82
- **[P2] Remove the committed gateway log file:** `openclaw-2026-03-07.log:1`
  - body: The PR adds a root runtime log file with local diagnostic output and path metadata. That artifact is unrelated to the code fix and should not be shipped in the repository.
  - confidence: 0.91

  - body: The code changes the give-up snapshot from the internal attempt value to MAX_RESTART_ATTEMPTS, but the provided file list does not update the current regression test that asserts reconnectAttempts is 11. Keep the test expectation aligned with the intended public behavior before merge.
