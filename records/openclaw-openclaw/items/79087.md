## Summary

Close as duplicate: current main still has the updater root-selection gap, but the same remaining npm-prefix staging problem is already tracked by https://github.com/openclaw/openclaw/issues/78775 and its open implementation PR https://github.com/openclaw/openclaw/pull/78778.

## What This Changes

Make `openclaw update` choose the invoked OpenClaw package's npm install root over a stale command-reported npm global root, and expose root mismatches in JSON/dry-run output.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/78775 as the canonical tracker and land a rebased updater fix that preserves pnpm custom-root behavior while adding npm package-root fallback coverage and any desired mismatch diagnostics.

## Reproduction Assessment

Yes, at source level: set `pkgRoot` to a user npm prefix while the npm command reports `/usr/lib/node_modules`, and `resolveGlobalInstallTarget` still selects the reported npm root for npm installs. I did not run a live mutating updater because this review is read-only.
