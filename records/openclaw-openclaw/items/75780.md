## Summary

Keep open. Current main still has the reported Gateway no-auth backend handshake gap, and open PR #75781 references this issue with closing syntax, so this issue should remain open until that PR or a corrected replacement lands.

## What This Changes

Reported that `gateway.auth.mode: "none"` causes local backend `gateway-client` connections used by subagent spawning and gateway tools to fail with `1008: device identity required`.

## Best Possible Solution

Review and land #75781 or a corrected replacement that treats no-auth as satisfied only for trusted local backend gateway-client coordination while preserving the existing remote, browser-origin, proxy, and device-boundary checks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: no-auth returns method `none`, `sharedAuthOk` remains false, the backend self-pairing skip excludes `none`, and the missing-device path closes with `1008: device identity required`.

## Solution Assessment

Unclear for the proposed one-line helper fix. The narrow maintainable fix should cover both the auth-context shared-auth state and the backend skip helper, with tests that prove local backend no-auth works while non-local/browser/proxy paths still fail closed.
