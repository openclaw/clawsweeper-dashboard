## Summary

Keep open. Current main still has the reported Gateway no-auth backend handshake gap, and open PR #75781 references this issue with closing syntax, so this should close only after that PR or a corrected replacement lands.

## What This Changes

The issue reports that `gateway.auth.mode: "none"` causes local backend RPC and subagent WebSocket connections to fail with `1008: device identity required`.

## Best Possible Solution

Land #75781 or a corrected replacement that treats no-auth as satisfied only for local backend Gateway-client coordination, with tests covering auth state, helper behavior, and preserved remote/browser/proxy rejection.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: no-auth returns method `none`, the shared-auth and backend-skip gates do not accept it, and a device-less backend connection can fall through to the `device identity required` close path.

## Solution Assessment

No for the issue’s helper-only proposed fix. The maintainable fix should cover both the auth-state classification and backend self-pairing helper, then prove the no-auth bypass remains local/backend-only.
