## Summary

Keep open. Current main still has the reported Gateway no-auth backend handshake gap, and open PR #75781 references this issue with closing syntax, so the issue should remain open until that PR or a corrected replacement lands.

## What This Changes

Allow local internal backend Gateway RPC connections, including subagent spawning and gateway tools, to work under `gateway.auth.mode: "none"` without requiring device identity.

## Best Possible Solution

Review and land #75781 or a corrected replacement that permits only trusted local `gateway-client` backend coordination under no-auth mode, with tests preserving remote/browser/proxy rejection.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: no-auth returns method `none`, current shared-auth/backend skip checks do not accept it, and a device-less backend connect reaches the `1008: device identity required` branch.

## Solution Assessment

Unclear. The proposed direction is narrow, but the final fix should cover both `sharedAuthOk` and backend helper gating while keeping the bypass local-only.
