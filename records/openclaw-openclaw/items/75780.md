## Summary

Keep open. Current main still has the no-auth Gateway backend handshake gap, and open PR #75781 is already paired to this issue as the implementation candidate.

## What This Changes

The issue reports that `gateway.auth.mode: "none"` causes local backend RPC and subagent connections to fail device identity checks with `1008: device identity required`.

## Best Possible Solution

Review and land #75781 or a corrected replacement that makes no-auth local backend `gateway-client` connections satisfy the narrow backend pairing bypass while adding regression coverage for auth-context and full handshake behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: no-auth returns method `none`, `sharedAuthOk` remains false, the backend self-pairing helper excludes `none`, and the missing-device branch closes with `1008: device identity required`.

## Solution Assessment

Unclear for the proposed one-line helper change because current main also leaves `sharedAuthOk` false for no-auth. The maintainable fix should cover both auth-context state and the backend handshake path, with local-only regression tests.
