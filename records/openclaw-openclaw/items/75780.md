Keep open. Current main still rejects the reported no-auth backend self-pairing path, and open PR #75781 is the paired implementation candidate that should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests that local internal Gateway backend RPC and subagent-spawn connections work under `gateway.auth.mode: "none"` without requiring device identity.

## Best Possible Solution

Land #75781 only if review confirms the full no-auth backend handshake path is covered, or replace it with a narrow Gateway auth fix that updates the helper/auth-context behavior and tests together.

Yes. A high-confidence source-level reproduction exists: no-auth returns method `none`, current shared-auth and backend self-pairing checks do not accept it, and device-less backend connections reach the `1008: device identity required` branch.
