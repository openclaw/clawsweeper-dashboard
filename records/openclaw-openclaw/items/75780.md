## Summary

Keep open. Current main still has the reported Gateway no-auth backend self-pairing gap, and open PR #75781 references this issue with closing syntax, so the issue should stay open until that PR or a corrected replacement lands.

## What This Changes

The issue reports that `gateway.auth.mode: "none"` causes local backend RPC and subagent WebSocket connections to fail with `1008: device identity required`.

## Best Possible Solution

Land a narrow Gateway auth fix through #75781 or a corrected replacement that allows only trusted local backend `gateway-client` coordination under `auth.mode: "none"` with helper, auth-context, and handshake coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: no-auth returns method `none`, current shared-auth and backend self-pairing checks do not accept it, and missing device identity reaches the `1008: device identity required` branch.

## Solution Assessment

Unclear for the helper-only proposed direction. The safer maintainable fix should exercise the full no-auth backend handshake path and avoid widening the pairing bypass beyond local backend coordination.
