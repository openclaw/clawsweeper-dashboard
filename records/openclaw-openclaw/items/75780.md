## Summary

Keep open. Current main still matches the reported failure shape, and the issue is explicitly paired with open PR #75781, so the issue should remain open until that fix or an equivalent maintainer-approved patch merges.

## What This Changes

The issue asks for `gateway.auth.mode: "none"` to allow trusted local backend GatewayClient RPCs used by subagent spawning, gateway tool calls, and CLI control-plane calls without requiring device identity.

## Best Possible Solution

Review and land #75781 or an equivalent narrow patch that treats `authMethod: "none"` as satisfied only for local backend `gateway-client` self-pairing, with regression tests that preserve rejection for remote, browser-origin, CLI, and node-role device-less callers.

## Reproduction Assessment

Yes. A source-level reproduction is clear: under `auth.mode: "none"`, auth returns `method: "none"`, the backend self-pairing helper does not skip pairing, and the missing-device branch closes the WebSocket with `1008: device identity required`.

## Solution Assessment

Yes for the issue workflow: keeping this open behind the linked fix PR is the right path. The code fix should stay scoped to backend loopback self-pairing rather than broadening generic no-auth pairing behavior.
