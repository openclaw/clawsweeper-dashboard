## Summary

Keep open. The requested gateway restart fix still addresses a source-visible race on current main, but this PR is not merge-ready because its shutdown abort path is stale against the current ChatAbortOps contract and after-fix real behavior proof is still absent.

## What This Changes

The PR adds a restart-only gateway close drain that forwards remaining drain budget into `server.close()`, waits on pending reply dispatchers and active chat runs, aborts timed-out chat runs, and adds gateway plus qa-lab coverage.

## Best Possible Solution

Land a rebased gateway-scoped fix that passes remaining restart drain budget into close, drains or aborts pending replies and active chat runs through the current abort cleanup contract, and keeps unrelated qa-lab mock changes out of this PR unless they are required.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Current docs promise safe restart drains reply delivery, current main counts pending replies before restart, and current close lacks a pending-reply drain before chat state and sockets are torn down.

## Solution Assessment

Yes for the direction, but no for the submitted branch as-is. A bounded close-time drain is the narrow maintainable fix, but the branch must be rebased onto the current shutdown/session-drain code and pass the full ChatAbortOps state into shutdown abort cleanup.
