## Summary

Keep open. The reported restart reply-loss gap is credible on current main, but this external PR is not mergeable as-is because it lacks real gateway restart proof, is conflicting with current shutdown code, and its new shutdown abort path misses the current chat delta cleanup contract.

## What This Changes

The PR adds a restart-only gateway close drain for pending reply dispatchers and active chat runs, passes remaining restart drain budget from the CLI run loop, and updates gateway plus qa-lab mock tests.

## Best Possible Solution

Land a rebased gateway-scoped fix that carries the remaining restart drain budget into close, drains or aborts pending replies and active chat runs through the current ChatAbortOps contract, and preserves current session-end drain behavior.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Current docs promise safe restart drains reply delivery, current main counts pending replies before restart, but `server.close` receives no drain budget and the close path has no pending-reply drain before chat state and sockets are torn down.

## Solution Assessment

Yes for the direction, but no for the submitted branch as-is. A bounded close-time drain is the narrow maintainable fix, but the branch must be rebased onto current shutdown/session-drain code and wire the full ChatAbortOps contract.
