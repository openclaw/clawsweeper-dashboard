## Summary

Keep open: current main still lacks a close-time pending-reply drain, but this PR is not merge-ready because the submitted shutdown abort path does not satisfy the current chat-abort contract and the PR still lacks real behavior proof.

## What This Changes

The PR adds a restart-only gateway close drain budget for pending replies and active chat runs, wires the remaining budget from the run loop into `server.close`, and adds gateway plus qa-lab mock coverage.

## Best Possible Solution

Land a rebased gateway-scoped fix that carries the remaining restart drain budget into close, drains or aborts pending replies and active chat runs through the current shutdown/chat-abort contracts, removes unrelated qa-lab changes, and includes redacted live restart proof.

## Reproduction Assessment

Yes from source inspection, not live execution. Current main counts pending replies in pre-restart deferral, but `server.close` still receives only `reason`/`restartExpectedMs` and `server-close.ts` has no close-time `drainTimeoutMs` or `getTotalPendingReplies` phase before chat state is cleared.

## Solution Assessment

Yes for the direction, but not for the submitted branch as-is. A bounded close-time drain is the narrow maintainable fix, but it must be rebased into the current `ChatAbortOps` and shutdown contracts before merge.
