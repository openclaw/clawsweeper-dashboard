## Summary

Keep open: the patch targets the right trajectory flush path, but changing the shared trajectory size constant leaves the public trajectory docs and required changelog out of sync.

## What This Changes

The PR adds a `setImmediate` yield to queued file writes, lowers the trajectory runtime file cap from 50 MiB to 10 MiB, and raises the agent cleanup step timeout from 10s to 30s.

## Best Possible Solution

Keep the PR open, align the trajectory docs and changelog with the intended cap behavior, and confirm whether the export limit should follow the runtime write cap or remain a separate 50 MiB export guard.

## Reproduction Assessment

No high-confidence live reproduction was established. The current-main source path is clear: trajectory recorders enqueue JSONL writes, cleanup waits on `flush()`, and the cleanup timeout only logs while the cleanup promise may continue.

## Solution Assessment

Unclear as written. Yielding between queued writes is a plausible narrow mitigation, but lowering a shared runtime/export limit requires the docs and release notes to move with it, or the export limit should be split from the runtime write cap.
