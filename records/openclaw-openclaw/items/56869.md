## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use NO_PROXY values the process proxy actually honors:** `docs/install/docker.md:207`
  - body: The Docker example presents CIDR ranges as generic `NO_PROXY` bypasses. Current OpenClaw adds CIDR checks in its helper path, but gateway startup also installs Undici's `EnvHttpProxyAgent`, whose `NO_PROXY` parser only handles host, subdomain, and optional port matching. Users can copy this and still send private-IP traffic through the proxy. Replace the CIDR entries with exact host/domain examples or explicitly scope CIDR support to the guarded OpenClaw helper paths.
  - confidence: 0.87
- **[P3] Remove the duplicate host-gateway instructions:** `docs/install/docker.md:212-217`
  - body: Current main already includes the real `extra_hosts` mapping in `docker-compose.yml`, and the Docker page already says the bundled compose maps `host.docker.internal` for Linux. Adding a second commented stanza and telling Linux users to add it can lead to stale copy-paste config or duplicate YAML keys. Reword this as already present in the bundled compose, with only custom compose files needing the mapping.
  - confidence: 0.91
- **[P2] Qualify the all-outbound proxy claim:** `docs/install/docker.md:219`
  - body: Raw `HTTP_PROXY` and `HTTPS_PROXY` env vars do not guarantee every gateway outbound path is routed through the local proxy; current network proxy docs call out raw socket, native addon, child-process, and owner-transport limits. This sentence should say supported HTTP/WebSocket/provider/channel paths, or point operators to managed proxy configuration for central egress control.
  - confidence: 0.89
