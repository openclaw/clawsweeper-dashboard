Full review comments:

- **[P2] Use NO_PROXY entries the env proxy actually honors:** `docker-compose.yml:32`
  - body: The new sample includes CIDR ranges, but Telegram and other env-proxy paths instantiate Undici's `EnvHttpProxyAgent`, whose upstream `NO_PROXY` matcher does not honor CIDR entries directly. Users copying this will expect private-IP ranges to bypass the proxy when some gateway paths will still proxy them.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker guide repeats the CIDR `NO_PROXY` values. Since the runtime paths named by the PR include EnvHttpProxyAgent-backed transports, the published guide should avoid CIDR examples or clearly scope them to paths that OpenClaw prefilters itself.
  - confidence: 0.9
- **[P2] Remove the duplicate extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already includes an active `extra_hosts` mapping for `host.docker.internal` in the gateway service. Adding a second commented service-level block invites users to uncomment a duplicate key instead of relying on the existing mapping.
  - confidence: 0.86
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: `HTTP_PROXY` and `HTTPS_PROXY` cover env-aware HTTP(S) clients, not every possible gateway egress path; the repo's managed proxy docs are the broader routing contract. This sentence should be narrowed so Docker users do not treat these env vars as full process-level proxy coverage.
  - confidence: 0.84
