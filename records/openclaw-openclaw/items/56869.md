## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope CIDR NO_PROXY to paths that honor it:** `docker-compose.yml:32`
  - body: The Compose sample uses CIDR `NO_PROXY` ranges, but those env vars are also consumed by Undici EnvHttpProxyAgent paths whose matcher only handles host/suffix/port entries. OpenClaw's CIDR extension is helper-scoped, so this example needs exact supported entries or a clear limitation before users rely on it as a gateway-wide private-network bypass.
  - confidence: 0.86
- **[P2] Document the NO_PROXY CIDR limitation:** `docs/install/docker.md:195`
  - body: The docs repeat the CIDR `NO_PROXY` sample without explaining that not every env-proxy path honors CIDR entries. Users following this can still see private or intranet IP requests go through the proxy on EnvHttpProxyAgent-backed paths.
  - confidence: 0.86
- **[P2] Remove the duplicate extra_hosts snippet:** `docker-compose.yml:33-34`
  - body: Current main already includes an active `extra_hosts` mapping for `host.docker.internal` in the gateway service. Adding a second commented service-level block tells users to add something that is already present and can create duplicate-key confusion.
  - confidence: 0.9
- **[P2] Narrow the all-outbound guarantee:** `docs/install/docker.md:207`
  - body: The new docs say these env vars ensure all outbound requests from the gateway container use the proxy, but current OpenClaw docs limit proxy coverage to supported HTTP/WebSocket paths and call out raw socket and child-process gaps. Reword this to the supported Docker gateway HTTP(S) paths or point to the managed proxy guide.
  - confidence: 0.82
