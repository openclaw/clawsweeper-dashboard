Full review comments:

- **[P2] Avoid CIDR NO_PROXY in raw env examples:** `docker-compose.yml:32`
  - body: The sample includes CIDR ranges, but the raw env-proxy paths instantiate Undici EnvHttpProxyAgent, whose NO_PROXY matcher does not implement CIDR matching. Users can expect private IP ranges to bypass the proxy while those requests still go through it; use supported host/domain entries or document the limitation explicitly.
  - confidence: 0.88
- **[P2] Remove the stale extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already defines `extra_hosts` for `host.docker.internal` on the gateway service. Adding another commented service-level block tells users to add a duplicate key if they uncomment it, so this should be deleted or changed to custom-compose-only guidance.
  - confidence: 0.92
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:206`
  - body: Raw HTTP_PROXY/HTTPS_PROXY examples do not guarantee every outbound request from the container is routed through the proxy; current docs describe proxy routing as process/client-specific with raw socket limits. Please narrow this sentence or point users at the managed proxy docs for central egress control.
  - confidence: 0.86
