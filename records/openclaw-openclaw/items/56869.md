## Summary

Keep open. The Docker proxy guidance is useful in principle, but the patch still overstates proxy coverage, includes misleading bypass examples, duplicates an existing Compose host mapping, and lacks convincing after-fix Docker proxy proof.

## What This Changes

The PR adds commented HTTP_PROXY/HTTPS_PROXY/NO_PROXY and extra_hosts examples to docker-compose.yml plus a Docker install docs section for local outbound proxy setup.

## Best Possible Solution

Keep the Docker docs addition only after it is scoped to supported HTTP(S)/Telegram-style transports, links to the managed Network proxy docs, removes redundant extra_hosts guidance, and includes redacted proof from a real Docker proxy run.

## Reproduction Assessment

Not applicable. This is a docs/Compose-example PR rather than a bug report, and the source review establishes the relevant current proxy behavior without a failing reproduction path.

## Solution Assessment

No. The maintainable direction is to align Docker guidance with the existing managed proxy contract and avoid raw env-var claims that overpromise routing or bypass behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Remove CIDR ranges from the raw NO_PROXY sample:** `docker-compose.yml:29`
  - body: The sample uses CIDR entries, but the gateway env-proxy path installs Undici's EnvHttpProxyAgent, whose NO_PROXY parser only handles exact host/subdomain entries plus optional ports. Users copying this will expect private-IP destinations to bypass the proxy, but those CIDR entries will not be honored by the underlying dispatcher.
  - confidence: 0.85
- **[P2] Scope the outbound proxy claim to supported transports:** `docs/install/docker.md:205`
  - body: The new docs say these env vars route all outbound gateway requests through the proxy. Current proxy docs and runtime only guarantee supported HTTP(S)/Undici/WebSocket-style routing, while raw net/tls/http2, native addons, and some child processes can bypass Node-level proxy hooks. Please narrow this wording or point users to the managed proxy docs for central egress control.
  - confidence: 0.88
- **[P2] Drop the duplicate extra_hosts example:** `docker-compose.yml:30-31`
  - body: Current docker-compose.yml already has an active host.docker.internal:host-gateway mapping for openclaw-gateway. Adding a second commented extra_hosts block tells users to uncomment a duplicate key in the same service, which can make the Compose file ambiguous or invalid depending on parser behavior.
  - confidence: 0.82
