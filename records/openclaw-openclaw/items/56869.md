## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove CIDR entries from raw NO_PROXY examples:** `docs/install/docker.md:195`
  - body: The sample adds CIDR ranges to `NO_PROXY`, but raw Undici `EnvHttpProxyAgent` does not parse CIDR entries; it only does host/suffix matching with optional ports. OpenClaw's CIDR matcher runs before selected guarded/node-proxy paths, but the global env dispatcher and Telegram env-proxy dispatcher still delegate bypass matching to Undici, so private-IP targets can still be proxied. Please use supported host/suffix examples or document the limitation in both examples.
  - confidence: 0.88
- **[P2] Do not promise every outbound path uses the proxy:** `docs/install/docker.md:205`
  - body: This statement is broader than the runtime contract. Standard env proxy variables affect env-aware HTTP(S)/Undici/agent paths, but current OpenClaw docs and code still have paths such as managed browser traffic, raw sockets, and HTTP/2 helpers that are not automatically covered by these Docker env vars. Qualify the wording so operators do not treat this as a complete egress boundary.
  - confidence: 0.82
- **[P3] Drop the stale extra_hosts example:** `docker-compose.yml:33-34`
  - body: Current main already includes `extra_hosts: - "host.docker.internal:host-gateway"` for `openclaw-gateway`. Adding a second commented block above `volumes` tells users of the bundled compose file to uncomment a duplicate service key rather than changing proxy env; remove it or scope the docs to custom compose files only.
  - confidence: 0.8
