## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use NO_PROXY entries the gateway dispatcher honors:** `docker-compose.yml:32`
  - body: The sample uses CIDR entries such as `10.0.0.0/8` and `172.16.0.0/12`, but the standard gateway env path installs Undici's `EnvHttpProxyAgent`, whose `NO_PROXY` parser checks host/domain entries and optional ports, not CIDR ranges. Users can follow this sample and still send private-IP traffic through the proxy.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker docs repeat the same CIDR `NO_PROXY` value for gateway-container proxy setup. Because ordinary global fetch paths use Undici's env proxy dispatcher, the docs should use exact host/domain entries or explicitly limit CIDR guidance to OpenClaw-specific guarded proxy paths.
  - confidence: 0.9
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: This sentence promises that all outbound requests from the gateway container are routed through the proxy, but current Network proxy docs scope coverage to process-local HTTP/WebSocket paths and warn that raw sockets, native addons, and non-OpenClaw child processes can bypass Node-level routing.
  - confidence: 0.86
- **[P3] Drop the stale extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already defines `host.docker.internal:host-gateway` for `openclaw-gateway`. Adding a second commented `extra_hosts` block under the same service can lead users to uncomment it and create a duplicate YAML key.
  - confidence: 0.88
