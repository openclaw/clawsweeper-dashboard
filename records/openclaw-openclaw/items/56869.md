## Summary

Keep open: the Docker docs and Compose proxy examples are a useful focused change, but this PR is not merge-ready because the `NO_PROXY` samples teach CIDR bypasses that the gateway-global Undici dispatcher will not honor, the `extra_hosts` sample duplicates current Compose state, and the attached screenshot does not prove a proxied Docker request after the change.

## What This Changes

The PR adds commented `HTTP_PROXY`, `HTTPS_PROXY`, and `NO_PROXY` examples to `docker-compose.yml` plus a Docker install docs proxy configuration section.

## Best Possible Solution

Land a Docker docs and Compose example that uses dispatcher-supported proxy bypass values, avoids repeating the already-active host-gateway mapping, and includes redacted proof that a Docker gateway request traversed the configured proxy.

## Reproduction Assessment

Yes. The review issue is source-reproducible by comparing the PR's CIDR `NO_PROXY` samples with gateway startup's `EnvHttpProxyAgent` bootstrap and Undici v8.2.0's matcher source; no live Docker repro was run in this read-only review.

## Solution Assessment

No. Docs and Compose comments are the right surface, but this patch should align its examples with current gateway dispatcher semantics and current Compose state before merge.

## Review Findings

Full review comments:

- **[P2] Use dispatcher-supported NO_PROXY examples:** `docker-compose.yml:32`
  - body: The added sample uses CIDR entries such as `10.0.0.0/8`, but gateway startup installs Undici's `EnvHttpProxyAgent` for global gateway egress, and Undici v8.2.0 only bypasses exact hosts, subdomain suffixes, and optional ports.   - confidence: 0.9
