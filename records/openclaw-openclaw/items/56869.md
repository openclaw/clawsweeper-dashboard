## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove CIDR ranges from the NO_PROXY example:** `docs/install/docker.md:195`
  - body: This example is for standard HTTP_PROXY/HTTPS_PROXY in the gateway process, but Undici's EnvHttpProxyAgent only treats NO_PROXY as exact host/domain-suffix entries with optional ports. CIDRs such as `10.0.0.0/8` will still be proxied for that path, so users copying the sample can misroute private or intranet traffic.
  - confidence: 0.88
- **[P2] Narrow the all-outbound proxy guarantee:** `docs/install/docker.md:207`
  - body: The docs say all outbound requests from the gateway container are routed through the local proxy, but current OpenClaw proxy docs describe only supported process-local HTTP/WebSocket routing and call out raw sockets, child processes, and custom transport exceptions. Reword this as scoped proxy guidance or link to the managed proxy page so operators do not treat it as a complete egress boundary.
  - confidence: 0.9
- **[P3] Scope the extra_hosts instructions to custom Compose files:** `docs/install/docker.md:200-205`
  - body: Current main already includes `extra_hosts: "host.docker.internal:host-gateway"` in the bundled Compose service. Keeping this as a required Linux step makes the Docker docs and Compose comments stale for the default setup; either remove it or say it only applies to custom Compose files.
  - confidence: 0.82
