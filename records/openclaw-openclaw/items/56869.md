## Summary

Keep open: the Docker proxy docs example is useful, but the patch still needs changes because the CIDR `NO_PROXY` samples do not match the gateway-wide Undici dispatcher path and current main already has host-gateway mapping; the supplied screenshot also does not prove the proxied Docker path.

## What This Changes

The PR adds commented `HTTP_PROXY`, `HTTPS_PROXY`, and `NO_PROXY` examples to `docker-compose.yml` plus a Docker install docs section for outbound proxy setup.

## Best Possible Solution

Land a docs/comment update that uses `NO_PROXY` examples supported by the gateway dispatcher, removes duplicated host-gateway guidance, and links users to the managed network proxy docs where appropriate.

## Reproduction Assessment

Yes. The review issue is source-reproducible by comparing the PR's CIDR `NO_PROXY` samples with current gateway env-proxy bootstrapping and upstream Undici v8.2.0 `EnvHttpProxyAgent` source.

## Solution Assessment

No. A docs-only Docker example is the right surface, but this patch should align the samples with the actual dispatcher semantics and current Compose file before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use NO_PROXY entries the gateway dispatcher honors:** `docker-compose.yml:32`
  - body: The sample includes CIDR entries such as `10.0.0.0/8`, but gateway startup installs Undici's `EnvHttpProxyAgent` for standard proxy env, and upstream Undici only checks exact host/subdomain plus optional port. These tokens will not bypass proxying for ordinary gateway egress, so the sample should use supported entries or the dispatcher path should gain CIDR support first.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The docs repeat the same CIDR-based `NO_PROXY` value. Users following this for mixed local or intranet Docker setups can still have private-IP destinations sent through the proxy because the gateway-wide Undici matcher does not parse CIDR ranges.
  - confidence: 0.9
- **[P3] Drop the duplicate extra_hosts guidance:** `docker-compose.yml:33-34`
  - body: Current main already defines `host.docker.internal:host-gateway` for `openclaw-gateway`. Keeping a second commented block under the same service is stale after rebase and can lead users to duplicate the service key when uncommenting the sample.
  - confidence: 0.84
