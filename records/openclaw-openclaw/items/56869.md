## Summary

Keep open: the Docker proxy guidance is useful, but the PR still includes NO_PROXY examples that some gateway-wide EnvHttpProxyAgent paths will not honor, duplicates an extra_hosts mapping already active on main, and lacks sufficient real behavior proof.

## What This Changes

The PR adds commented HTTP_PROXY, HTTPS_PROXY, and NO_PROXY examples to docker-compose.yml plus a Docker install docs section for outbound proxy setup.

## Best Possible Solution

Land a corrected Docker docs/comment update that uses supported NO_PROXY examples, avoids duplicate host-gateway guidance, and links to Network proxy docs for managed egress control.

## Reproduction Assessment

Yes. Source inspection reproduces the review findings by comparing the PR samples with current gateway EnvHttpProxyAgent bootstrap and Undici v8.2.0 NO_PROXY matching; no live Docker proxy run was performed.

## Solution Assessment

No. A docs-only Docker example is the right surface, but this patch should align its examples with gateway-wide dispatcher semantics and current Compose state before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use NO_PROXY entries all gateway paths honor:** `docker-compose.yml:32`
  - body: The sample includes CIDR entries such as 10.0.0.0/8 and 172.16.0.0/12 as if they bypass all gateway env-proxy traffic. Gateway startup installs Undici's EnvHttpProxyAgent for standard proxy env, and that matcher only honors exact hosts, subdomain suffixes, and optional ports; OpenClaw's CIDR support is only a pre-dispatcher trusted-env-proxy gate. Users can still see private-IP traffic proxied in ordinary global-dispatcher paths.
  - confidence: 0.88
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker docs repeat the same CIDR NO_PROXY value. Because the published example is described as gateway-container outbound proxy setup, it should not teach bypass tokens that Undici's gateway-wide EnvHttpProxyAgent ignores for normal global fetch paths; use exact host/domain entries or document the narrower scope.
  - confidence: 0.88
- **[P3] Drop the stale extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already defines host.docker.internal:host-gateway for the openclaw-gateway service. Keeping a second commented extra_hosts block under the same service can lead users to duplicate the key when uncommenting the sample.
  - confidence: 0.86
