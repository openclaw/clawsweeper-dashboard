## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use NO_PROXY entries the env proxy actually honors:** `docker-compose.yml:32`
  - body: The sample includes CIDR ranges, but the gateway's global env-proxy path installs Undici EnvHttpProxyAgent, whose NO_PROXY parser does not implement CIDR matching. Users copying this can expect private IP ranges to bypass the proxy while EnvHttpProxyAgent-backed requests still proxy those destinations.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker guide repeats the CIDR NO_PROXY values while describing general gateway-container proxy setup. Because current proxy coverage has mixed paths, the published sample can mislead users running mixed public and private-network traffic.
  - confidence: 0.88
- **[P2] Remove the duplicate extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already includes an active extra_hosts mapping for host.docker.internal in the bundled Compose service. Adding another commented service-level block invites users to uncomment a duplicate key instead of relying on the existing mapping.
  - confidence: 0.88
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: HTTP_PROXY and HTTPS_PROXY cover env-aware HTTP clients and selected OpenClaw proxy paths, not every possible gateway egress path. The docs should match the existing network proxy docs and avoid promising that all outbound requests route through these env vars.
  - confidence: 0.84
