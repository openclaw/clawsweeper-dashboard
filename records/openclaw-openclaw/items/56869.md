## Summary

Keep this PR open: the Docker proxy guidance is useful in scope, but the current patch still documents `NO_PROXY` CIDR entries that the gateway-wide Undici env proxy dispatcher will not honor, duplicates `extra_hosts` guidance already active on main, and the attached proof does not show the Docker proxy path being exercised.

## What This Changes

The PR adds commented Docker Compose proxy environment examples and a Docker install docs section for outbound proxy setup in container deployments.

## Best Possible Solution

Land a Docker docs/comment update that uses `NO_PROXY` entries the gateway dispatcher supports, removes duplicate host-gateway guidance, and points users to the managed network proxy docs when they need policy-grade egress control.

## Reproduction Assessment

Yes. The review issue is source-reproducible by comparing the PR's CIDR `NO_PROXY` samples with gateway startup's `EnvHttpProxyAgent` bootstrap and Undici v8.2.0's matcher source.

## Solution Assessment

No. A docs-only Docker example is the right surface, but this patch should align its examples with the actual dispatcher semantics and current Compose file before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use NO_PROXY entries the gateway dispatcher honors:** `docker-compose.yml:32`
  - body: The sample includes CIDR entries such as `10.0.0.0/8`, but gateway startup installs Undici's `EnvHttpProxyAgent` for standard proxy env, and that matcher only handles exact host/subdomain plus optional port. Users following this sample can still have private-IP destinations sent through the proxy, so the example should use supported entries or the dispatcher should gain CIDR support first.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The docs repeat the same CIDR-based `NO_PROXY` value. For mixed local or intranet Docker setups, those CIDR tokens will not bypass the gateway-wide Undici env proxy dispatcher, so the published sample would teach a bypass that does not work for ordinary gateway egress.
  - confidence: 0.9
- **[P3] Drop the duplicate extra_hosts guidance:** `docker-compose.yml:33-34`
  - body: Current main already defines `host.docker.internal:host-gateway` for the gateway service. Keeping a second commented block under the same service is stale after rebase and can lead users to duplicate the service key when uncommenting the sample.
  - confidence: 0.84
