## Summary

Keep open: the Docker proxy docs/example change is useful, but the current PR still has proxy-contract bugs and the attached proof does not show the edited Docker proxy environment or a redacted proxied request after the patch.

## What This Changes

The PR adds commented `HTTP_PROXY`/`HTTPS_PROXY`/`NO_PROXY` examples to `docker-compose.yml` and a Docker install docs section for host proxy egress.

## Best Possible Solution

Land a corrected Docker docs/comment update that uses `NO_PROXY` entries honored by the gateway proxy path, removes redundant host-gateway guidance, scopes the outbound-routing claim, and links to the Network proxy docs for managed egress control.

## Reproduction Assessment

Yes. Source inspection of the PR head, current gateway proxy bootstrap, OpenClaw proxy matcher, and upstream Undici source reproduces the review findings without needing a live Docker run.

## Solution Assessment

No. Docker docs and commented examples are the right surface, but this patch should align its examples with supported proxy semantics and current Compose state before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use NO_PROXY entries the gateway dispatcher honors:** `docker-compose.yml:32`
  - body: The sample includes CIDR entries such as `10.0.0.0/8` and `172.16.0.0/12`, but gateway startup installs Undici's `EnvHttpProxyAgent` for standard proxy env and that upstream matcher only handles exact hosts, subdomain suffixes, and optional ports. Users can still see private-IP traffic proxied despite following this sample.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker docs repeat the same CIDR `NO_PROXY` value for gateway-container proxy setup. Because ordinary global fetch paths use Undici's env proxy dispatcher, the docs should use exact host/domain entries or clearly document any narrower OpenClaw-specific matcher scope.
  - confidence: 0.9
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: This sentence promises that all outbound requests from the gateway container are routed through the proxy, but OpenClaw's existing proxy docs scope this to supported process-local HTTP/WebSocket paths and warn that raw sockets, native addons, and non-OpenClaw child processes can bypass Node-level routing. The Docker section should avoid overpromising complete container egress control.
  - confidence: 0.86
- **[P3] Drop the stale extra_hosts block:** `docker-compose.yml:33-34`
  - body: Current main already defines `host.docker.internal:host-gateway` for `openclaw-gateway`. Adding a second commented `extra_hosts` block under the same service can lead users to uncomment it and create a duplicate YAML key.
  - confidence: 0.88
