Overall confidence: 0.86

Full review comments:

- **[P2] Clarify CIDR NO_PROXY support in the Compose sample:** `docker-compose.yml:32`
  - body: The `NO_PROXY` sample uses CIDR ranges as a general Docker gateway example, but EnvHttpProxyAgent itself only matches exact hosts, suffixes, and optional ports. OpenClaw's CIDR extension is scoped to its own trusted-env proxy decision, so this sample can make users expect private-IP bypasses that some gateway HTTP(S) paths will not provide.
  - confidence: 0.86
- **[P2] Clarify CIDR NO_PROXY support in the docs sample:** `docs/install/docker.md:195`
  - body: The docs repeat the same CIDR `NO_PROXY` value without explaining that it is not the general Undici `EnvHttpProxyAgent` contract. Users relying on this for private-network bypasses can still have those requests sent through the proxy.
  - confidence: 0.84
- **[P3] Remove duplicate Linux extra_hosts guidance:** `docs/install/docker.md:200-205`
  - body: Current `docker-compose.yml` already includes the Linux host-gateway mapping for `host.docker.internal`. Telling readers to add the block again can create duplicate `extra_hosts` keys if copied literally.
  - confidence: 0.87
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: This sentence says all outbound requests from the gateway container route through the local proxy, but `NO_PROXY` intentionally bypasses some targets and not every dependency shares the same env-proxy behavior. Narrow it to HTTP(S) egress that honors `HTTP_PROXY`/`HTTPS_PROXY`.
  - confidence: 0.78
