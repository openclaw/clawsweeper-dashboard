- **[P2] Use NO_PROXY entries the env proxy actually honors:** `docker-compose.yml:32`
  - body: The new Compose sample includes CIDR ranges, but the gateway's global env-proxy dispatcher uses Undici's `EnvHttpProxyAgent`, whose `NO_PROXY` parser does not honor CIDR entries.   - confidence: 0.9

  - confidence: 0.9

  - body: Current main already includes an active `extra_hosts` mapping for `host.docker.internal` in the bundled Compose service. Adding another commented service-level block invites users to uncomment a duplicate key instead of relying on the existing mapping.
  - confidence: 0.88

  - body: `HTTP_PROXY` and `HTTPS_PROXY` cover env-aware HTTP clients, not every possible gateway egress path. The new sentence should match the existing network-proxy docs and avoid promising that all outbound requests route through these env vars.
  - confidence: 0.84
