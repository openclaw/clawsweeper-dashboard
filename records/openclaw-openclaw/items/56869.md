## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use NO_PROXY entries the env dispatcher honors:** `docker-compose.yml:32`
  - body: The sample includes CIDR ranges, but ordinary gateway env-proxy routing installs Undici's `EnvHttpProxyAgent`, whose `NO_PROXY` parser checks host/domain entries and optional ports. Users can follow this sample and still proxy private-IP destinations unless a specific OpenClaw helper pre-checks the target.
  - confidence: 0.9
- **[P2] Use supported NO_PROXY entries in the docs sample:** `docs/install/docker.md:195`
  - body: The Docker docs repeat the same CIDR `NO_PROXY` value for gateway-container proxy setup. The docs should either use exact host/domain entries for the standard env dispatcher path or explicitly limit CIDR guidance to the OpenClaw helper paths that implement that extension.
  - confidence: 0.9
- **[P2] Scope the all-outbound proxy claim:** `docs/install/docker.md:207`
  - body: This sentence promises that all outbound requests from the gateway container are routed through the proxy, but current proxy docs scope coverage to process-local HTTP/WebSocket paths and note that custom transports may need explicit proxy wiring. The Docker docs should avoid promising OS-level or every-client egress routing.
  - confidence: 0.86
- **[P3] Remove the duplicate extra_hosts guidance:** `docker-compose.yml:33-34`
  - body: Current main already defines `host.docker.internal:host-gateway` for `openclaw-gateway`. Adding a second commented `extra_hosts` block under the same service can lead users to uncomment it and create a duplicate YAML key.
  - confidence: 0.88
