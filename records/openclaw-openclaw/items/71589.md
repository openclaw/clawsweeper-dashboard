Keep open: the PR addresses a real user-visible agent reply leak, but the current patch still suppresses only after delayed phase/tool-use metadata is known, so an earlier phaseless text_delta can already emit partial or block output. It also needs the required changelog entry before merge.

## What This Changes

The PR expands embedded assistant-output suppression for tool-use continuations, clears related buffers, preserves explicit final_answer streaming, adds regression coverage, and includes adjacent qa-lab mock routing, status test, and lockfile maintenance changes.

## Best Possible Solution

Land a narrow embedded-agent suppression fix that quarantines phase-pending assistant text until it is known to be user-visible, suppresses commentary/tool-use output before any partial, block, or final reply can escape, preserves explicit final_answer output, and covers message_end, text_end, blockReplyChunking, and partial-reply regressions with a changelog entry.

## Reproduction Assessment

Yes. Emit an assistant text_delta with no phase/tool metadata while partial replies or text_end block chunking are enabled, then classify the same assistant message as commentary/toolUse at text_end or message_end; the current PR can still emit before that later classification.

## Solution Assessment

No, not yet. Expanding the suppression predicate and clearing buffers is directionally right, but clearing after classification is too late for already-emitted partial/block output; the safer fix is to hold phase-pending chunks until classification is known.
