## Summary

Keep open: the underlying tool-use text leak is valid, but this PR is not merge-ready because phase-pending assistant text can still be emitted before the new suppression sees enough metadata, and contributor real behavior proof is still missing.

## What This Changes

This PR expands embedded agent assistant-output suppression for unphased tool-use/commentary turns, clears pending visible buffers, preserves `final_answer` streaming, and updates qa-lab/status/lockfile tests.

## Best Possible Solution

Hold phase-pending assistant text until visibility classification is known, suppress commentary/tool-use output before any partial, block, or final delivery, and preserve explicit `final_answer` streams.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit an assistant `text_delta` without phase/tool metadata while partial replies or `text_end` block delivery are enabled, then classify the same assistant message as commentary/tool-use at `text_end` or `message_end`.

## Solution Assessment

No. Expanding the suppression predicate and clearing buffers after classification helps, but it cannot retract already emitted partial or block output; the safer fix is to quarantine phase-pending chunks before visibility callbacks.
