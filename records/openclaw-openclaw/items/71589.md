## Summary

Keep open: this PR targets a real channel-visible agent bug, but the latest branch can still emit phase-pending assistant text before later suppression and it lacks required after-fix real behavior proof from a real setup.

## What This Changes

The branch extends embedded agent output suppression for unphased tool-use continuations, clears suppressed buffers, preserves explicit final_answer streaming, and adds adjacent qa-lab/status/lockfile changes.

## Best Possible Solution

Hold phase-pending assistant text until final/user-visible classification is known, then suppress commentary/tool-use output before any partial, block, or final delivery while preserving explicit final_answer streams.

## Reproduction Assessment

Yes, source-reproducible. Emit an assistant text_delta with no phase/tool metadata while partial replies or text_end block chunking are enabled, then classify the same assistant message as commentary/toolUse at text_end or message_end; the first chunk can already have emitted.

## Solution Assessment

No, not yet. Expanding the suppression predicate and clearing buffers helps once classification is known, but it cannot retract already emitted partial or block output; quarantine before visibility decisions is the narrower maintainable fix.
