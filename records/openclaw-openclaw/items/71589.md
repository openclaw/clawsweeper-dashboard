## Summary

Keep open: current main still only suppresses commentary-phased assistant output, while this PR attempts a useful unphased tool-use fix that still has a delivery-order bug and lacks required real behavior proof.

## What This Changes

The PR expands embedded agent assistant-output suppression for tool-use/commentary turns, preserves `final_answer` partial streaming, and adds related agent, qa-lab, status, and lockfile test/maintenance updates.

## Best Possible Solution

Implement one agent delivery fix that quarantines unclassified assistant text until phase/tool-use visibility is known, then land it with focused regression coverage and redacted real channel proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an assistant `text_delta` without phase/tool metadata can emit through partial or block delivery, then later `text_end`/`message_end` classifies the same turn as commentary or tool-use.

## Solution Assessment

No. Expanding the predicate and clearing buffers after classification helps, but the safer solution is to hold phase-pending chunks before any user-visible callback can fire.
