## Summary

Keep open. The underlying user-visible tool-use text leak is real and current main does not implement this PR, but the patch still allows phase-pending assistant text to escape as a partial or block-streamed reply before later commentary/tool-use classification triggers suppression.

## What This Changes

The PR extends embedded agent assistant-output suppression for unphased tool-use continuations, preserves explicit final_answer streaming, adds regression coverage, and includes adjacent qa-lab/status/lockfile maintenance hunks.

## Best Possible Solution

Land a narrow embedded-agent suppression fix that quarantines phase-pending assistant text until it is known to be user-visible, suppresses commentary/tool-use output before any partial, block, or final reply can escape, preserves explicit final_answer output, and adds message_end, text_end, blockReplyChunking, and partial-reply regressions plus the required changelog entry.

## Reproduction Assessment

Yes. Emit an assistant text_delta with no phase/tool metadata while partial replies or text_end block chunking are enabled, then classify the same assistant message as commentary/toolUse at text_end or message_end; the current code path appends and can emit before that later classification.

## Solution Assessment

No, not yet. Expanding the suppression predicate and clearing buffers is the right direction, but clearing after classification is too late for already-emitted partial/block output; the safer fix is to hold phase-pending chunks until classification is known.
