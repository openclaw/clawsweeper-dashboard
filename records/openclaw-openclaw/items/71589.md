## Summary

Keep this PR open for author or maintainer follow-up: it targets a real user-visible leak path, but the current patch still allows phase-pending assistant text to be emitted before later suppression metadata arrives, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR expands embedded agent output suppression for unphased tool-use/commentary assistant turns, clears pending visible buffers, preserves `final_answer` streaming, and adjusts adjacent qa-lab/status/lockfile tests.

## Best Possible Solution

Hold phase-pending assistant text until user-visible classification is known, suppress commentary/tool-use output before any partial, block, or final delivery, and preserve explicit `final_answer` streams.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit an assistant `text_delta` without phase/tool metadata while partial replies or `text_end` block delivery are enabled, then classify the same assistant message as commentary/tool use at `text_end` or `message_end`.

## Solution Assessment

No. Expanding the suppression predicate and clearing buffers after classification helps, but it cannot retract already emitted partial or block output; the safer fix is to quarantine phase-pending chunks before visibility callbacks.
