## Summary

Keep open. The reported tool-use/intermediate text leak is still source-reproducible, but this PR is not mergeable as-is because it can still emit an unclassified chunk before later suppression runs and it lacks after-fix real behavior proof.

## What This Changes

The PR extends embedded agent output suppression for unphased tool-use continuations, clears suppressed buffers, preserves `final_answer` streaming, and adds adjacent qa-lab/status/lockfile test maintenance.

## Best Possible Solution

Hold phase-pending assistant text until user-visible classification is known, then suppress commentary/tool-use output before partial, block, or final delivery while preserving explicit `final_answer` streams.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit an assistant `text_delta` with no phase/tool metadata while partial replies or `text_end` block delivery are enabled, then classify the same assistant message as commentary/tool use at `text_end` or `message_end`.

## Solution Assessment

No, not yet. Expanding the suppression predicate and clearing buffers after classification helps, but it cannot retract already emitted partial or block output; the safer fix is to quarantine phase-pending chunks before visibility callbacks.
