## Summary

Keep open: current main still has the security-facing diagnostics gap the reporter describes. CLI backends expose native tools outside OpenClaw's tool catalog, while doctor/docs do not clearly warn that the sandbox, fs workspace, allow/deny, and sessions_send settings only govern OpenClaw PI tools in that runtime shape.

## What This Changes

Request doctor warnings, docs callouts, and clearer allowlist guidance when CLI runtimes make OpenClaw sandbox/tool/session settings ineffective for native CLI tools.

## Best Possible Solution

Add an explicit, security-reviewed doctor/audit warning plus docs callouts for CLI-native runtimes and non-default sandbox/tool/session settings, while preserving the boundary that OpenClaw cannot intercept native CLI tools.

## Reproduction Assessment

Yes from source inspection for the diagnostics gap, but no live host reproduction was run. Current main marks CLI backends as native-tool capable and the doctor path does not warn about that combination.

## Solution Assessment

Yes, the requested warning and documentation path is the narrow maintainable direction. Making CLI runtimes actually enforce these settings would be a much larger runtime interception feature.
