## Summary

Keep open: current main still does not add the requested doctor/docs warning for supported CLI backends. The code now has stronger CLI-runtime contracts and `codex-cli` is no longer bundled, but the central diagnostics gap remains for CLI backends that can expose native tools outside OpenClaw's tool catalog.

## What This Changes

Request doctor warnings, docs callouts, and clearer allowlist messaging when CLI runtimes make sandbox, filesystem, or session-tool config ineffective.

## Best Possible Solution

Add a security-reviewed doctor warning plus public docs callouts for supported CLI backends where OpenClaw sandbox/tool/session controls do not govern native CLI surfaces.

## Reproduction Assessment

Yes, source-reproducible for the diagnostics gap: current main has CLI-native-tool contracts and generic doctor/security warnings, but no doctor/docs check for the requested no-op configuration combination. No live CLI host reproduction was run.

## Solution Assessment

Yes, the warning/docs direction is the narrow maintainable path. Enforcing OpenClaw sandbox or session-tool policy inside third-party CLI-native tools would be a larger runtime-specific interception feature.
