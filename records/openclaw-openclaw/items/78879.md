## Summary

Keep open: current main partially mitigates runtime `toolsAllow` for native-tool CLI backends, but it still lacks the requested proactive doctor/docs warning for explicit sandbox, filesystem, tool, and session policy expectations under CLI runtimes.

## What This Changes

Request a doctor warning, docs callouts, and clearer messaging when CLI runtimes make OpenClaw sandbox, filesystem, allow/deny, or `sessions_send` policy ineffective or unenforceable.

## Best Possible Solution

Add a narrow doctor/doctor --lint diagnostic and docs callouts for selected CLI runtimes with explicit sandbox/fs/tool/session expectations, while keeping the existing runtime fail-closed behavior for unenforceable `toolsAllow`.

## Reproduction Assessment

Yes. Source inspection shows CLI backends fail closed for `toolsAllow` at runtime, but current doctor/docs still lack the requested proactive warning for broader sandbox, filesystem, and session policy mismatch.

## Solution Assessment

Likely yes: warning plus docs is the right level because hard-failing CLI runtimes would be too disruptive, but maintainers should first define the exact supported warning scope.
