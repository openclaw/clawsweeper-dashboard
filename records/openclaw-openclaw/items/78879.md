## Summary

Keep open: current main has stronger CLI-native-tool contracts and generic security warnings, but it still does not provide the requested doctor/docs warning for CLI backends where OpenClaw sandbox/tool/session config does not govern native CLI tools.

## What This Changes

The issue requests doctor diagnostics and documentation callouts for OpenClaw sandbox/tool/session config that does not govern CLI-runtime native tools.

## Best Possible Solution

Add a security-reviewed doctor warning plus public docs callouts that explain which OpenClaw controls apply only to OpenClaw/PI tools and point the unknown-allowlist warning to that runtime-boundary documentation.

Yes, source-reproducible for the diagnostics gap: current main has CLI native-tool contracts and generic doctor/security warnings, but no doctor/docs check for the requested no-op configuration combination. No live CLI host reproduction was run.

## Solution Assessment

Yes, the warning/docs direction is the narrow maintainable approach. Enforcing OpenClaw sandbox or session-tool policy inside third-party CLI-native tools would be a larger runtime-specific interception feature.
