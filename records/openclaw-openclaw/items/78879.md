## Summary

Keep open: current main still has the reported diagnostics gap for CLI-native runtimes, and the linked doctor migration PR does not implement or supersede this warning/docs work.

## What This Changes

Request doctor warnings, docs callouts, and clearer allowlist log messaging when CLI runtimes make OpenClaw sandbox/tool/session settings ineffective.

## Best Possible Solution

Add a security-reviewed doctor/audit warning and public docs callouts for CLI-native runtimes while keeping the actual native-tool interception/enforcement question out of scope.

## Reproduction Assessment

Yes, for the diagnostics gap: current main shows CLI backends can expose native tools while doctor and docs do not warn about the inert OpenClaw PI-tool settings. I did not run a live host CLI reproduction of cross-workspace file access.

## Solution Assessment

Yes, the requested warning/docs/log-link path is the narrow maintainable direction. Making CLI native tools obey OpenClaw sandbox/fs/session policy would be a much larger runtime interception feature.
