## Summary

Keep open: current main still has the security-facing diagnostics/docs gap, and the linked SQLite refactor PR is broad, open, protected, and not a closing implementation for this request.

## What This Changes

The issue requests doctor warnings and docs callouts when CLI runtimes make sandbox, filesystem, tool policy, and sessions_send settings ineffective for the CLI binary's native tools.

## Best Possible Solution

Add a security-reviewed doctor/audit warning plus docs callouts that explain CLI-native runtimes are not PI-tool-equivalent, without implying OpenClaw can intercept or sandbox unsupported native CLI tools.

## Reproduction Assessment

Yes at source level: current main shows CLI backends with always-on native tools while doctor lacks a CLI-runtime mismatch check and the targeted docs lack the caveat. I did not run the reporter's live two-workspace claude-cli scenario.

## Solution Assessment

Yes, the warning/docs path is the narrow maintainable direction. Making CLI runtimes enforce these settings would require a larger native-tool interception feature and should not be implied by a diagnostics fix.
