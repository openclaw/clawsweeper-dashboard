## Summary

Keep open: current main still lacks the requested security-facing doctor/docs warning for CLI-native runtimes. Source and docs show the runtime boundary exists, but doctor does not flag configured sandbox/tool/session settings that only govern OpenClaw PI tools while CLI-native tools remain owned by the CLI process.

## What This Changes

Request doctor warnings plus docs/log callouts when CLI runtimes make sandbox, fs workspace, tool allow/deny, and sessions_send settings ineffective for native CLI tools.

## Best Possible Solution

Add a concise security-reviewed doctor/audit warning and docs callouts for CLI-native runtimes with non-default sandbox, fs workspace, allow/deny, or session-tool settings, while preserving the runtime boundary rather than trying to intercept native CLI tools.

## Reproduction Assessment

Yes for the diagnostics gap: current main clearly declares CLI-native tools and the doctor path lacks a matching warning. The reporter's full cross-workspace live bypass was not rerun in this read-only review.

## Solution Assessment

Yes, the requested warning/docs path is the narrow maintainable direction. Enforcing these settings inside Claude/Codex/Gemini native tools would be a larger runtime interception feature and should not be assumed here.
