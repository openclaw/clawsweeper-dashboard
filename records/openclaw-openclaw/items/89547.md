## Summary

This PR remains necessary because current main has no equivalent Telegram group-history control. The implementation consistently applies one per-account mode to cached, pending, and recovered-topic context, focused tests cover those paths, and exact-head Mantis proof now shows the visible before/after behavior; no actionable correctness or security defect was found. Merge still requires an explicit maintainer decision because changing the unset default to `mention-only` intentionally removes ambient recent context from existing Telegram group installations.

## What This Changes

Adds per-account Telegram group-history modes (`mention-only`, `recent`, and `none`), defaults unset configurations to mention-only, and applies the selected mode to cached, pending, and recovered forum-topic context.

## Best Possible Solution

Keep the unified per-account mode and consistent context gating, then merge only after maintainers explicitly accept `mention-only` as the upgrade default; if backward compatibility is preferred, retain the new modes but change only the fallback to `recent`.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this PR intentionally adds a context policy and changes a default, while exact-head native Telegram proof demonstrates the intended before/after behavior.
