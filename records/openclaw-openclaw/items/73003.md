## Summary

Keep open: current main has the merged z.ai-specific mitigation from #76056, but the guard still is not aware of explicit `safeguard` mode, and open PR #73839 targets this issue with closing syntax.

## What This Changes

The issue reports that `compaction.mode = "safeguard"` can leave Pi SDK auto-compaction enabled, causing repeated `already_compacted_recently` failures that block OpenClaw safeguard compaction.

## Best Possible Solution

Review and land #73839, or an equivalent patch that threads the effective compaction mode into the existing guard across embedded, CLI, reload, and compaction-session paths while preserving the #76056 z.ai mitigation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use `compaction.mode = "safeguard"` with the legacy or another non-owning context engine and a non-z.ai provider, and current main activates safeguard while leaving Pi auto-compaction enabled.

## Solution Assessment

Yes. Extending the existing Pi auto-compaction guard to account for the effective safeguard mode is the narrow maintainable fix; retrying around `already_compacted_recently` would leave two compaction owners competing.
