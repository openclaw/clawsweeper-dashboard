## Summary

Current main still has the reported safeguard/Pi auto-compaction ownership mismatch, and PR #73839 is already the open linked implementation candidate with closing syntax. This issue should stay open until that PR or an equivalent patch lands.

## What This Changes

The issue requests that OpenClaw disable or coordinate Pi SDK auto-compaction when safeguard compaction mode is active so OpenClaw-owned preflight compaction is not blocked by Pi's recent-compaction state.

## Best Possible Solution

Land PR #73839 or an equivalent narrow patch that centralizes the OpenClaw-owns-compaction decision, covers explicit safeguard mode and provider-forced safeguard mode, applies it consistently in embedded and CLI paths, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure agents.defaults.compaction.mode as safeguard without a context engine that declares ownsCompaction, then current main installs the safeguard extension while leaving the Pi auto-compaction guard false.

## Solution Assessment

Yes. Threading a resolved safeguard/OpenClaw-owned compaction signal into the existing Pi auto-compaction guard is the narrow maintainable fix; channel-specific workarounds or retry logic would leave the competing compaction ownership unresolved.
