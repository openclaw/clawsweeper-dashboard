## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled when safeguard mode is active without an owning context engine, and open PR #73839 is the active unmerged implementation candidate linked with closing syntax for this report.

## What This Changes

The issue requests that OpenClaw disable or coordinate Pi SDK auto-compaction when `agents.defaults.compaction.mode` effectively resolves to `safeguard`.

## Best Possible Solution

Land #73839 or an equivalent narrow patch so the Pi auto-compaction guard treats effective safeguard mode as OpenClaw-owned across embedded and CLI preflight paths, including provider-forced safeguard mode, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure `agents.defaults.compaction.mode = "safeguard"` without an owning context engine, then current main enables the safeguard extension while the Pi auto-compaction guard still returns false.

## Solution Assessment

Yes. Centralizing the policy in `pi-settings` and threading effective compaction mode through the embedded and CLI call sites is the narrow maintainable fix; channel-specific recovery logic would be broader.
