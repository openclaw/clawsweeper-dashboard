## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled for explicit safeguard mode, and open PR #73839 is already paired to this issue with a narrow unmerged fix.

## What This Changes

Coordinate Pi SDK auto-compaction with OpenClaw safeguard compaction so the two compaction lifecycles do not block each other in long sessions.

## Best Possible Solution

Review and land #73839 or an equivalent patch that feeds the effective safeguard compaction mode into the existing Pi auto-compaction guard for embedded and CLI lifecycles, with focused guard tests.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the default legacy context engine: current main installs the safeguard extension, but the Pi auto-compaction guard sees no `ownsCompaction` flag and does not disable Pi auto-compaction.

## Solution Assessment

Yes. Extending the existing guard to account for effective safeguard mode is the narrow maintainable fix; channel-specific workarounds or retry loops would leave two compaction owners competing.
