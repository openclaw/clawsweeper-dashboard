## Summary

Keep open: current main still leaves Pi auto-compaction enabled for explicit safeguard mode unless a context engine also declares `ownsCompaction`, and open PR #73839 is already paired to this issue with closing syntax but has not merged.

## What This Changes

The issue reports that `compaction.mode = "safeguard"` can be blocked by Pi SDK auto-compaction because OpenClaw only disables Pi compaction for context engines that own compaction.

## Best Possible Solution

Land #73839 or an equivalent narrow change that threads the effective compaction mode into the Pi auto-compaction guard for both embedded and CLI compaction lifecycles, with regression tests around safeguard/default/off behavior.

## Reproduction Assessment

Yes. Configure explicit safeguard mode with the default legacy context engine: current main installs the safeguard runtime from config, but the Pi guard still returns false because legacy `ContextEngineInfo` lacks `ownsCompaction`.

## Solution Assessment

Yes. Disabling Pi auto-compaction when OpenClaw's effective compaction mode is safeguard is the narrow maintainable fix; channel-specific retries or log handling would leave two compaction owners competing.
