## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled when safeguard mode is configured without an owning context engine, and open PR #73839 is the active unmerged implementation candidate linked with closing syntax for this report.

## What This Changes

The issue requests coordinating or disabling Pi SDK auto-compaction when OpenClaw safeguard compaction owns the compaction lifecycle, preventing repeated recent-compaction failures and unchecked session growth.

## Best Possible Solution

Land #73839 or an equivalent narrow patch so the Pi auto-compaction guard treats the effective safeguard compaction mode as OpenClaw-owned across embedded and CLI paths, with focused tests for default, safeguard, and context-engine-owned behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure `agents.defaults.compaction.mode = "safeguard"` without an owning context engine, then current main enables the safeguard extension while `shouldDisablePiAutoCompaction` still returns false because it only checks `ownsCompaction`.

## Solution Assessment

Yes. Centralizing the policy in `pi-settings` and threading the effective compaction mode through the embedded and CLI call sites is the narrow maintainable fix; channel-specific recovery logic would be broader and more fragile.
