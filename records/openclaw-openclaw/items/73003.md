## Summary

Keep open: current main still lacks an explicit safeguard-mode input to the Pi auto-compaction guard for the legacy/non-owning context-engine path, and open PR #73839 references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The report asks OpenClaw to disable or coordinate Pi SDK auto-compaction when `agents.defaults.compaction.mode` is `safeguard` so safeguard preflight compaction is not blocked by repeated `already_compacted_recently` failures.

## Best Possible Solution

Land #73839 or an equivalent mode-aware guard across embedded, CLI, and post-reload paths, with regression coverage for safeguard mode and the legacy context engine.

## Reproduction Assessment

Yes. A source-level reproduction is `agents.defaults.compaction.mode = "safeguard"` with the legacy context engine and a non-silent-overflow provider: current main enables the safeguard extension but the Pi guard sees neither `ownsCompaction` nor a mode flag, so Pi auto-compaction can remain enabled.

## Solution Assessment

Yes. Extending the existing guard to use the effective compaction mode is the narrow maintainable fix; retrying after `already_compacted_recently` would leave two compaction owners competing.
