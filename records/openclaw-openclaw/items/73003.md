## Summary

Keep open: current main still enables the safeguard extension without disabling Pi SDK auto-compaction for safeguard mode, and open PR #73839 is the paired unmerged implementation candidate that uses closing syntax for this report.

## What This Changes

The issue asks OpenClaw to prevent Pi SDK auto-compaction from blocking OpenClaw safeguard compaction during long-running sessions configured with `compaction.mode = "safeguard"`.

## Best Possible Solution

Review and land #73839, or an equivalent narrow patch, so the Pi auto-compaction guard treats configured safeguard mode as OpenClaw-owned compaction in both embedded and CLI compaction paths while preserving default/off and context-engine-owned behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure `agents.defaults.compaction.mode = "safeguard"` without a context engine that owns compaction; current main enables the safeguard extension while the Pi auto-compaction guard remains false, leaving the SDK's default auto-compaction active.

## Solution Assessment

Yes. The narrow maintainable fix is to centralize the policy in `pi-settings` and thread the configured compaction mode through embedded and CLI call sites; the paired PR takes that direction, but it is not merged on current main.
