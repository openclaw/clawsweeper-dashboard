## Summary

Keep open: current main still does not disable Pi auto-compaction based on explicit safeguard mode, and open PR #73839 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

The issue asks OpenClaw to coordinate or disable Pi SDK auto-compaction when safeguard compaction owns the lifecycle, preventing repeated `already_compacted_recently` failures.

## Best Possible Solution

Land #73839 or an equivalent mode-aware guard that disables Pi auto-compaction whenever the effective compaction mode is safeguard, including embedded, CLI, and post-reload paths, with regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is `agents.defaults.compaction.mode = "safeguard"` with the legacy context engine: current main loads the safeguard extension, but the Pi auto-compaction guard sees neither `ownsCompaction` nor a mode flag and can leave Pi auto-compaction enabled.

## Solution Assessment

Yes. Extending the existing guard to use the effective compaction mode is the narrow maintainable fix; retrying after `already_compacted_recently` or channel-specific workarounds would leave two compaction owners competing.
