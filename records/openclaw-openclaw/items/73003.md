## Summary

Keep open. Current main still disables Pi SDK auto-compaction only for context engines with `ownsCompaction`, while safeguard mode is enabled separately and can run without an owning context engine. The issue also has open PR #73839 linked with closing syntax, so the issue should stay open until that implementation or an equivalent fix lands.

## What This Changes

Ensure safeguard compaction coordinates with or disables Pi SDK auto-compaction so OpenClaw's preflight/safeguard compaction is not repeatedly blocked by recent Pi compaction entries.

## Best Possible Solution

Land #73839 or an equivalent narrow patch that centralizes the ownership policy in `pi-settings`, threads the effective safeguard compaction mode through embedded and CLI call sites, and adds focused regression coverage for default, safeguard, and context-engine-owned behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure `agents.defaults.compaction.mode = "safeguard"` without an owning context engine, then current main enables the safeguard extension while `shouldDisablePiAutoCompaction` still returns false because it only checks `ownsCompaction`.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: make the shared Pi auto-compaction guard aware of the effective safeguard mode instead of adding channel-specific recovery or special-casing Telegram sessions.
