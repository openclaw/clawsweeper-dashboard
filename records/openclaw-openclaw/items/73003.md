## Summary

Keep open: current main has a shipped z.ai/silent-overflow mitigation, but it still does not disable Pi auto-compaction merely because the effective compaction mode is `safeguard`, and open PR #73839 still targets this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to disable Pi SDK auto-compaction when safeguard mode owns the preflight compaction lifecycle.

## Best Possible Solution

Land #73839 or an equivalent patch that threads the effective safeguard compaction mode into the existing Pi auto-compaction guard across embedded, CLI, and reload paths, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.compaction.mode = "safeguard"` with the legacy or another non-owning context engine and a non-z.ai-detected provider, and current main still leaves Pi auto-compaction enabled.

## Solution Assessment

Yes. Extending the existing guard to consider the effective safeguard mode is the narrow maintainable fix; retrying around `already_compacted_recently` would leave two independent compaction owners competing.
