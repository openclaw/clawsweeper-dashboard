## Summary

Keep open: current main still only disables Pi auto-compaction when a context engine declares `ownsCompaction`, while safeguard mode is enabled through a separate config path. The issue is also paired with open PR #73839 using closing syntax, so it should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue requests disabling Pi SDK auto-compaction when OpenClaw safeguard compaction owns the session compaction lifecycle.

## Best Possible Solution

Centralize compaction ownership in `pi-settings` so explicit safeguard mode and provider-forced safeguard both disable Pi auto-compaction in embedded and CLI paths, with focused tests for default, safeguard, provider, and context-engine-owned cases.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the legacy context engine; current main installs the safeguard extension but the Pi auto-compaction guard still returns false because `ownsCompaction` is unset.

## Solution Assessment

Yes. Threading the effective compaction mode into the existing Pi auto-compaction guard is the narrow maintainable fix; retry loops or channel-specific workarounds would leave two compaction owners competing.
