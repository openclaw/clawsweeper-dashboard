## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled for explicit safeguard mode, and open PR #73839 is already paired to this issue with a narrow fix that has not merged.

## What This Changes

The issue reports that safeguard-mode compaction can be blocked by Pi SDK auto-compaction because OpenClaw's Pi guard only honors context-engine ownership.

## Best Possible Solution

Review and land #73839 or an equivalent patch that passes the effective safeguard compaction mode into the Pi auto-compaction guard for both embedded and CLI lifecycles, including after resource-loader reloads.

## Reproduction Assessment

Yes. Configure agents.defaults.compaction.mode = "safeguard" with the default legacy context engine: current main installs the safeguard extension, but the Pi auto-compaction guard sees no ownsCompaction flag and does not call setCompactionEnabled(false).

## Solution Assessment

Yes. Extending the existing guard to use the effective compaction mode is the narrow maintainable fix; channel-specific workarounds or retry loops would leave two compaction owners competing.
