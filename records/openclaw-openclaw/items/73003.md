## Summary

Keep open: current main still leaves Pi auto-compaction enabled for safeguard-mode runs unless a context engine declares `ownsCompaction`, and the issue is already paired with open PR #73839 using closing syntax. The issue should remain open until that PR or an equivalent provider-aware guard fix lands.

## What This Changes

The issue requests disabling or coordinating Pi SDK auto-compaction when OpenClaw safeguard compaction owns the compaction lifecycle, preventing repeated `already_compacted_recently` failures and unchecked session growth.

## Best Possible Solution

Land a narrow fix through #73839 or an equivalent PR that centralizes the OpenClaw-owned compaction predicate, preserves context-engine `ownsCompaction`, treats explicit and provider-backed safeguard mode as OpenClaw-owned, applies it in embedded and CLI paths, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the default legacy context engine: current main installs the safeguard extension, but the Pi guard remains false because legacy context info has no `ownsCompaction` flag.

## Solution Assessment

Yes. Extending the existing Pi auto-compaction guard is the narrow maintainable solution; it should derive effective safeguard ownership, including provider-forced safeguard mode, rather than adding channel-specific workarounds or retry loops.
