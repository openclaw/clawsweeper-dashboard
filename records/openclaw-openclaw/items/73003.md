## Summary

Keep open: current main still leaves Pi auto-compaction enabled for explicit safeguard mode, and open PR #73839 is the active paired fix that has not merged.

## What This Changes

The issue asks OpenClaw to prevent safeguard compaction from being permanently blocked by Pi SDK auto-compaction after recent compaction boundaries.

## Best Possible Solution

Review and land #73839 or an equivalent patch that passes the effective safeguard compaction mode into the Pi auto-compaction guard for embedded and CLI lifecycles, including reload-safe coverage.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the default legacy context engine: current main installs the safeguard extension, but the Pi auto-compaction guard receives no `ownsCompaction` flag and returns `disabled: false`.

## Solution Assessment

Yes. Extending the existing guard to disable Pi auto-compaction when OpenClaw's effective compaction mode is safeguard is the narrow maintainable fix; retry loops or channel-specific workarounds would leave two compaction owners competing.
