## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled for explicit safeguard mode, and open PR #73839 already references this issue with closing syntax but has not merged.

## What This Changes

The issue reports that OpenClaw safeguard compaction can be permanently blocked by Pi SDK auto-compaction because the guard only disables Pi compaction for engines that declare `ownsCompaction`.

## Best Possible Solution

Review and land #73839 or an equivalent patch that passes the effective safeguard compaction mode into the Pi auto-compaction guard for embedded and CLI lifecycles, including reload paths.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the default legacy context engine; current main installs the safeguard extension but the Pi guard sees no `ownsCompaction` flag and does not call `setCompactionEnabled(false)`.

## Solution Assessment

Yes. Extending the existing guard to use the effective compaction mode is the narrow maintainable fix; channel-specific workarounds or retry loops would leave two compaction owners competing.
