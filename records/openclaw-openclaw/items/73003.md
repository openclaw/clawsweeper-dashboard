## Summary

Keep open: current main still only disables Pi auto-compaction when a context engine declares ownership, while safeguard mode is enabled through a separate compaction-mode path. Open PR #73839 is already paired to this issue with closing syntax and remains unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Disable Pi SDK auto-compaction whenever OpenClaw's safeguard compaction lifecycle is active, including explicit safeguard mode and provider-forced safeguard mode.

## Best Possible Solution

Unify compaction ownership in `pi-settings` so effective safeguard mode disables Pi auto-compaction in both embedded and CLI lifecycles, with focused regression tests, then land #73839 or an equivalent patch.

## Reproduction Assessment

Yes. Configure `agents.defaults.compaction.mode = "safeguard"` with the default legacy context engine: current main installs the safeguard extension, but the Pi auto-compaction guard still returns false because `ownsCompaction` is unset.

## Solution Assessment

Yes. Threading the effective compaction mode into the existing Pi auto-compaction guard is the narrow maintainable fix; retry loops or channel-specific workarounds would leave two compaction owners competing.
