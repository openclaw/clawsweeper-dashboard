## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep hidden skills out of route candidates:** `src/agents/tools/skill-route-tool.ts:61`
  - body: Avoid falling back to `snapshot.skills` here. Current `buildWorkspaceSkillSnapshot` intentionally keeps `snapshot.skills` as the broader eligible list while `resolvedSkills` is the prompt-visible list; when every skill is hidden from model invocation, this fallback still registers `local_skill_route` and returns hidden skill names to the model. Use `resolvedSkills` only, and omit the tool when that list is empty.
  - confidence: 0.92
- **[P2] Compute ambiguity before limiting results:** `src/agents/tools/skill-route-tool.ts:153-156`
  - body: Determine `matched` vs `ambiguous` from the full sorted strong-match set before applying the requested output limit. With the current order, `limit: 1` removes the runner-up before the status check, so a near-tied ambiguous query can incorrectly return `matched`.
  - confidence: 0.9
