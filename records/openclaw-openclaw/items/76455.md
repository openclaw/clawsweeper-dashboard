## Summary

Keep open. The trust-bypass concern is real, but the patch is not merge-ready because it tightens the bypass without updating the existing official npm install metadata, docs, and tests that still rely on bare `@openclaw/*` specs with no integrity input.

## What This Changes

The PR requires trusted official `@openclaw/*` npm installs to use an exact-version selector plus non-empty `expectedIntegrity` before bypassing the install security scan, and adds focused install tests for that policy.

## Best Possible Solution

Land the exact-version-plus-integrity trust policy together with official pinned metadata or a documented ClawHub-first replacement path, updated npm-spec coverage, and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible. Current main trusts official npm plugin installs by package/plugin identity only, and the existing npm-spec tests exercise bare official packages with scanner-critical launch code that succeeds only because of that bypass.

## Solution Assessment

No, not as submitted. The helper-level security direction is sound, but the PR needs to update the supported install surfaces that still use bare specs and cannot pass `expectedIntegrity`, otherwise documented official installs and existing coverage will fail.
