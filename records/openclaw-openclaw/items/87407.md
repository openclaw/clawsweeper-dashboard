## Label Justifications

- `impact:auth-provider`: The affected behavior is provider transport failure handling, model choice, and fallback routing across Anthropic primary and OpenAI fallback chains.

## Summary

Keep open: current main still routes Anthropic Messages through guarded model fetch, and a first pre-response `fetch failed`/`UND_ERR_SOCKET` is logged, lease-released, and rethrown into failover classification instead of getting a same-provider retry. The linked open fix PR is the active candidate, so this issue should stay open until a fix merges.

## What This Changes

The issue requests a bounded same-provider retry or transport fix so transient Anthropic socket failures do not silently fall through to a different provider fallback mid-turn.

## Best Possible Solution

Land a reviewed guarded-transport fix that retries the same provider once for clearly transient pre-response socket failures before model fallback, with regression coverage and affected-environment proof.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows the first transport failure, and classifies bare fetch/network failures as timeout fallback attempts. I did not live-reproduce the multi-hour stale-socket trigger in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a bounded same-provider retry in guarded model transport before model fallback runs. The open linked PR should be reviewed and proven rather than closing this issue before merge.
