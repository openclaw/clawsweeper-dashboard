## Summary

Keep open: the report matches current source behavior where Anthropic transport fetch failures are logged and thrown, then model fallback can silently continue to a different provider instead of retrying the primary provider once.

## What This Changes

The issue asks OpenClaw to prevent transient Anthropic `UND_ERR_SOCKET` fetch failures from silently switching a turn to a cross-provider fallback model.

## Best Possible Solution

Add a narrow retry-once for replayable pre-response Undici socket/fetch failures in the Anthropic/shared model transport path before the configured model fallback ladder runs.

## Reproduction Assessment

Yes for the fallback behavior: current source and tests show transport `fetch failed` errors are classified as timeout and can advance to another provider. I did not live-reproduce the multi-hour Anthropic stale-socket trigger.

## Solution Assessment

Yes, a bounded retry-once before model fallback is the narrowest maintainable direction; disabling keep-alive globally would be broader and riskier than retrying a replayable pre-response socket failure.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/87407. Current Anthropic Messages transport calls `buildGuardedModelFetch`; a `TypeError("fetch failed")` with cause code `UND_ERR_SOCKET` is logged and rethrown, then model fallback classifies it as `timeout` and may switch to another provider. Add a narrow retry-once before model fallback for replayable pre-response transient Undici/socket fetch failures, preferably in `src/agents/provider-transport-fetch.ts` or a small local helper used by the Anthropic transport. Retry only when the request body can be safely replayed and the request was not aborted; do not retry consumed streams, `AbortError`, or non-transient provider errors. Preserve SSRF, dispatcher, proxy, timeout refresh, local service lease cleanup, and existing fallback policy. Do not add config/default surfaces or changelog edits. Add regression tests covering `UND_ERR_SOCKET`/`fetch failed` retry success and no retry for abort or non-replayable bodies.
