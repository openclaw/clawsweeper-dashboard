## Summary

Keep open: current main and the latest release still route Anthropic Messages through guarded fetch and rethrow the first pre-response transport failure into model fallback without a same-provider retry. The issue has an open closing PR, so it should stay open until that PR lands or maintainers choose a different fix path.

## What This Changes

The issue reports Anthropic `UND_ERR_SOCKET`/`fetch failed` transport errors causing immediate cross-provider model fallback instead of a bounded retry on the primary provider.

## Best Possible Solution

Land a reviewed guarded-transport fix that retries the same provider once for clearly transient pre-response socket failures before model fallback, with regression coverage and affected-environment proof.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows the first pre-response transport failure, and failover classifies socket/network failures as timeout. I did not live-reproduce the multi-hour stale-socket trigger in this read-only review.

## Solution Assessment

Yes in direction: the narrow maintainable fix is a bounded same-provider retry in guarded model transport before model fallback runs. The open linked PR should be reviewed and proven rather than closing this issue before merge.
