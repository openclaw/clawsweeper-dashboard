## Label Justifications

- `P1`: The issue reports an urgent provider-routing regression where Anthropic primary requests can silently fall through to a different provider during real agent turns.
- `impact:auth-provider`: The affected behavior is provider routing and model fallback between Anthropic and OpenAI/Codex based on transport/auth-provider execution failures.

## Summary

Keep open: current main still lets Anthropic text-generation transport failures escape the guarded fetch path directly into model fallback, so a transient `fetch failed`/`UND_ERR_SOCKET` can still advance to another provider before a same-key retry. PR #87480 is related but remains open and unmerged, so the issue is still the canonical user-impact tracker.

## What This Changes

The issue requests a bounded same-provider retry or equivalent Anthropic transport hardening so transient socket failures do not silently switch an Anthropic primary model to an OpenAI/Codex fallback mid-turn.

## Best Possible Solution

Land or replace #87480 with a guarded-transport same-candidate retry for transient Anthropic text-create fetch/socket failures, plus focused fallback tests and live or reporter proof.

## Reproduction Assessment

Yes from source inspection: the Anthropic text path still rethrows transient guarded-fetch failures into fallback without a same-key retry, although this read-only pass did not run a multi-hour live reproduction.

## Solution Assessment

Yes: a bounded retry on the same Anthropic candidate for transient pre-response fetch/socket errors is the narrowest fix; changing only SDK dispatcher settings is not enough unless it proves coverage of the guarded text path and cold-start failures.
