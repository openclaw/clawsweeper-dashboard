## Label Justifications

- `P1`: The issue reports a silent first-run onboarding skip that affects real managed workspaces and already has two independent confirmations.
- `impact:data-loss`: OpenClaw can delete a user-provided `BOOTSTRAP.md` before the intended onboarding flow consumes it.
- `impact:session-state`: Workspace setup state can be marked complete even though onboarding markers and first-run work are still absent.

## Summary

Keep this open: current `main` and the latest release still use profile-file diffs as bootstrap completion evidence, and two open PRs already reference this issue with closing syntax. The right cleanup path is to review and land one canonical fix PR, not close the issue early.

## What This Changes

Fresh preseeded workspaces with custom `SOUL.md`, `IDENTITY.md`, or `USER.md` can have `BOOTSTRAP.md` deleted and `setupCompletedAt` written before first-run onboarding completes.

## Best Possible Solution

Keep this issue open until maintainers land or reject a canonical fix, with https://github.com/openclaw/openclaw/pull/91955 currently the strongest linked candidate because it includes focused before/after proof.

## Reproduction Assessment

Yes. Current-main source shows a high-confidence reproduction path: preseed `BOOTSTRAP.md` plus custom profile files, then `ensureAgentWorkspace` can mark `bootstrapSeededAt`, treat profile diffs as completion evidence, write `setupCompletedAt`, and remove `BOOTSTRAP.md`.

## Solution Assessment

Yes. The narrow maintainable fix is in the workspace bootstrap reconciler: distinguish first-lifecycle preseeded profile defaults from real completion evidence, without adding a new config mode for this bug.
