## What This Changes

The issue reports that overflow auto-compaction can leave the gateway process alive but unresponsive, causing supervised restarts to collide with the existing lock and port.

## Best Possible Solution

Reproduce with a sanitized diagnostics bundle from the affected system, then fix the narrow failing boundary: either bound/isolate post-compaction plugin side effects, harden gateway liveness during compaction retry, or repair documented/systemd unit drift if that is the remaining loop trigger.

## Reproduction Assessment

No high-confidence end-to-end reproduction exists from this review. Source inspection does confirm the reported lock/restart path is plausible when a supervised restart races an alive but unhealthy gateway, and the issue logs provide a concrete production trace around auto-compaction.

## Solution Assessment

Unclear. The suggested lock heartbeat or forced-exit ideas may be part of a fix, but the safer first step is to isolate whether the hang is core compaction retry, post-compaction memory/plugin side effects, or an out-of-date systemd unit before changing stale-lock ownership rules.
