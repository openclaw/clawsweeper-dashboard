## Summary

Keep open: current `main` and the latest release still reject `HEARTBEAT_OK` acknowledgements when the assistant content includes reasoning/thinking non-text blocks, and the related fixes are open PRs rather than merged implementation.

## What This Changes

The issue requests fixing heartbeat transcript pruning so short `HEARTBEAT_OK` acknowledgements with reasoning/thinking blocks are recognized and removed from future model request history.

## Best Possible Solution

Land one focused heartbeat-filter fix with regression coverage for reasoning/thinking/redacted thinking blocks while preserving the tool-call and meaningful non-text safety boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an assistant content array with a `thinking` block plus a `text` block containing `HEARTBEAT_OK` sets `hasNonTextContent` and makes `isHeartbeatOkResponse` return false before the ack check.

## Solution Assessment

Yes. The best fix is a narrow classifier change in `src/auto-reply/heartbeat-filter.ts` plus focused regression tests, not a new config surface or broader heartbeat policy change.
