## Summary

Keep open: current main and the latest release still reject short HEARTBEAT_OK acknowledgements when the assistant content includes reasoning/thinking blocks, and the linked fixes are open PRs rather than merged implementation.

## What This Changes

The issue asks OpenClaw to prune heartbeat prompt/ack transcript spans even when the HEARTBEAT_OK assistant message also contains model reasoning or thinking blocks.

## Best Possible Solution

Land one focused heartbeat-filter fix with regression coverage for reasoning, thinking, and redacted thinking blocks while preserving tool-call and non-text safety boundaries, then close this issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an assistant content array with a reasoning/thinking block plus a text HEARTBEAT_OK block sets hasNonTextContent and makes isHeartbeatOkResponse return false; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The maintainable fix belongs in the heartbeat acknowledgement classifier plus focused regression tests, not in a new config surface or broader heartbeat policy change.
