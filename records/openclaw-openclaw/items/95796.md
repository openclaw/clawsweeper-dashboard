## Summary

Keep open: current main still rejects HEARTBEAT_OK acknowledgements when the assistant content includes reasoning/thinking blocks, and the related fixes are open PRs rather than merged implementation.

## What This Changes

The issue reports that heartbeat transcript pruning fails to remove short HEARTBEAT_OK turns when the assistant message also contains reasoning or thinking content blocks.

## Best Possible Solution

Land one focused heartbeat-filter fix with regression coverage for reasoning/thinking/redacted_thinking blocks while preserving tool-call and non-text safety boundaries, then close this issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an assistant content array with a reasoning/thinking block plus a text HEARTBEAT_OK block sets hasNonTextContent and makes isHeartbeatOkResponse return false; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The best fix is a narrow classifier change in src/auto-reply/heartbeat-filter.ts plus focused regression tests, not a new config surface or broader heartbeat policy change.
