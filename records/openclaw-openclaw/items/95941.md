## Summary

Keep open: current main and v2026.6.9 still build command-cron delivery from stdout/stderr with no `deliveryStreams` field, and the exact companion implementation at https://github.com/openclaw/openclaw/pull/95955 was closed unmerged.

## What This Changes

The issue requests an additive `deliveryStreams` command-payload option so cron announce/webhook delivery can include only selected streams while diagnostics and watchdog monitoring still see both stdout and stderr.

## Best Possible Solution

Add one canonical command-payload stream filter only if maintainers accept the config surface; keep the default as both streams, preserve diagnostics/watchdog behavior, and align schema, runtime, CLI, docs, and focused tests.

## Reproduction Assessment

No failing bug repro is applicable because this is a feature request. Source inspection does confirm the current behavior: command summaries combine stdout and stderr, and the schema rejects unknown command-payload fields.

## Solution Assessment

Unclear pending maintainer decision: stream filtering belongs near the command summary/delivery boundary, but the exact field shape and failure-alert semantics need owner approval because this adds public config surface.
