## Summary

Keep open: current main and v2026.6.10 still lack a command-cron `deliveryStreams` field and still combine stdout/stderr for delivery, while the exact companion implementation PR is closed unmerged.

## What This Changes

Request to add a `deliveryStreams` command-cron payload option so announce/webhook delivery can include only stdout while stderr remains available for watchdog diagnostics and run history.

## Best Possible Solution

Approve one canonical stream-filter contract only if maintainers want this core config surface, keeping the default as both streams while preserving watchdog and run-history diagnostics.

## Reproduction Assessment

No failing bug repro is applicable because this is a feature request. Source inspection confirms current command summaries combine stdout and stderr and current schema does not accept a delivery-stream field.

## Solution Assessment

Unclear until maintainers choose the config contract. Stream filtering likely belongs near the command summary/delivery boundary, but failure-alert and truncation semantics need owner approval.
