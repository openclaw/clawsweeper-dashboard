## Summary

Keep open: current main and v2026.6.10 still lack a command-payload stream filter, and the exact companion implementation at https://github.com/openclaw/openclaw/pull/95955 is closed unmerged.

## What This Changes

Request adds a `deliveryStreams` command-cron payload option so announce/webhook delivery can include only selected stdout/stderr streams while watchdog and run-history diagnostics still observe both.

## Best Possible Solution

Add one canonical command-payload stream filter only after maintainers approve the contract; keep the default as both streams and update schema, runtime, CLI, docs, and focused tests together.

## Reproduction Assessment

Yes, source-reproducible: current code combines stdout and stderr into one command summary, the schema has no `deliveryStreams` field, and announce/webhook paths consume that summary. I did not run a live cron job in this read-only review.

## Solution Assessment

Unclear pending maintainer decision: `deliveryStreams` is a plausible narrow shape, but the best final contract depends on choices around webhook delivery, failure alerts, and truncation/diagnostics semantics.
