## Summary

Keep open: the PR targets a real OpenAI Codex OAuth stale-expiry bug, but the current diff is not landable as-is because the 30-minute cap repeats the JWT short-circuit instead of forcing an authoritative recheck, the branch needs to preserve current-main identity metadata, and real behavior proof is still missing.

## What This Changes

This PR adds an OpenAI Codex OAuth access-token JWT `exp` short-circuit and provider tests for stale stored `expires` values.

## Best Possible Solution

Land a rebased provider-local fix that avoids unnecessary Codex refreshes for valid cached JWTs while making the capped recheck reach an authoritative refresh or controlled re-import and preserving identity metadata.

## Reproduction Assessment

Yes, at source level. Seed an OpenAI Codex OAuth credential with stale stored `expires` and a long-lived JWT access token: current main enters the refresh hook, while this PR short-circuits and then repeats that same short-circuit after the synthetic cap.

## Solution Assessment

No. The provider-local JWT check is the right boundary, but this implementation stores the cap only in `expires`, so post-cap re-entry does not enforce the claimed revocation or external-rotation check.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force the cap to reach authoritative refresh:** `extensions/openai/openai-codex-provider.ts:327-330`
  - body: The 30-minute cap only updates `expires`. Once that synthetic expiry passes, the OAuth manager calls this hook again with the same long-lived JWT and this condition returns before `refreshOpenAICodexToken`, so revoked or externally rotated credentials can remain trusted until JWT expiry. Store state or change the control flow so the post-cap path reaches real refresh or a controlled re-import, and add a time-advanced regression test.
  - confidence: 0.9
- **[P3] Preserve Codex identity metadata when skipping refresh:** `extensions/openai/openai-codex-provider.ts:330`
  - body: Current main's refresh path derives and persists Codex `accountId` and `chatgptPlanType` from the access token. Returning only `{ ...cred, expires }` skips that enrichment for older profiles that predate the metadata, leaving account-scoped usage/auth handoff without fields a normal refresh would add.
  - confidence: 0.78
