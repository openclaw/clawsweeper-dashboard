## Summary

Keep open: this PR targets a real OpenAI Codex OAuth refresh bug, but the proposed 30-minute JWT recheck cap does not actually force an authoritative refresh or external re-import after the cap, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds an OpenAI Codex OAuth access-token JWT `exp` short-circuit and provider tests so stale stored `expires` values can be corrected without immediately refreshing.

## Best Possible Solution

Land a rebased provider-local fix that skips unnecessary Codex refreshes for valid cached JWTs while making any capped recheck reach real refresh or controlled external re-import and preserving identity metadata.

## Reproduction Assessment

Yes, at source level. Seed a stored OpenAI Codex OAuth credential with stale `expires` and a long-lived JWT access token: current main proceeds to provider refresh, while this PR short-circuits and repeats the same short-circuit after the synthetic cap.

## Solution Assessment

No. The provider-local JWT check is the right boundary, but this implementation stores the cap only in `expires`, so post-cap re-entry repeats the early return instead of enforcing the claimed revocation or external-rotation check.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force the cap to reach authoritative refresh:** `extensions/openai/openai-codex-provider.ts:327-330`
  - body: The 30-minute cap only updates `expires`. Once that cap passes, the OAuth manager calls this hook again with the same long-lived JWT, and this condition returns before `refreshOpenAICodexToken` again; current Codex external CLI sync is bootstrap-only once local inline tokens exist, so `codex login` rotations are not reliably picked up here either. Store state or change control flow so the post-cap path reaches real refresh or a controlled re-import, and add a time-advanced regression test.
  - confidence: 0.9
- **[P3] Preserve Codex identity metadata when skipping refresh:** `extensions/openai/openai-codex-provider.ts:330`
  - body: Current main's refresh path derives and persists Codex `accountId` and `chatgptPlanType` from the access token. Returning only `{ ...cred, expires }` skips that enrichment for older profiles that predate the metadata, leaving account-scoped usage/auth handoff without the fields that a normal refresh would add.
  - confidence: 0.78
