## Summary

Keep this PR open, but do not merge as-is: the provider-local JWT short-circuit addresses the reported refresh storm, but the proposed 30-minute recheck cap does not actually force an authoritative refresh after the cap expires.

## What This Changes

The PR adds an OpenAI Codex OAuth JWT `exp` short-circuit in the provider refresh hook plus targeted provider tests for JWT, opaque-token, malformed-token, and preservation cases.

## Best Possible Solution

Land a rebased provider-local fix that skips unnecessary OpenAI Codex refreshes for valid cached JWT access tokens while making any capped recheck reach an authoritative refresh or explicit external re-import path, preserving account metadata, and adding the required changelog entry.

## Reproduction Assessment

Yes. A deterministic unit path can seed an OpenAI Codex OAuth credential with stale stored `expires` and a long-lived JWT access token, then advance past the synthetic cap; the PR short-circuits again instead of reaching HTTP refresh.

## Solution Assessment

No. The provider-local JWT check is the right boundary, but the current cap is only a stored timestamp and is not an effective revocation or rotation bound without additional state/control flow.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Force the cap to reach authoritative refresh:** `extensions/openai/openai-codex-provider.ts:327-330`
  - body: The 30-minute cap only updates `expires`. Once that cap passes, the OAuth manager calls this hook again with the same long-lived JWT, and this condition returns before `refreshOpenAICodexToken` again; OpenAI Codex external CLI credentials are also bootstrap-only during runtime refresh, so `codex login` rotations are not picked up here. Add state/control flow plus a time-advanced test so the post-cap path reaches real refresh or an explicit re-import path.
  - confidence: 0.92

  - body: The current refresh path resolves Codex identity from the access token and persists fields such as `accountId` and `chatgptPlanType`. This early return only updates `expires`, so older profiles that lack metadata will not be enriched on the path that replaces refresh.
  - confidence: 0.8

  - body: This is a user-facing OpenAI Codex OAuth reliability fix, but the PR only changes provider code and tests. Repo policy requires a `CHANGELOG.md` entry for user-facing fixes, so release notes would omit this behavior change if merged as-is.
  - confidence: 0.88
