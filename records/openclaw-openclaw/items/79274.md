## Summary

Keep open. Current main still has the reported deep code-safety scan shape, and the related concurrency PR was closed without merging, so the central throughput and progress problem remains unresolved.

## What This Changes

Improve `openclaw security audit --deep` so deep plugin and skill code-safety scans do not cause long silent stalls, with bounded throughput, progress/timing, caching, and collector controls.

## Best Possible Solution

Add bounded deterministic concurrency plus visible deep-audit phase progress and timing first, then scope persistent cache, per-collector limits, and `--only` as follow-ups after maintainer UX/config decisions.

## Reproduction Assessment

Yes for a source-level reproduction: current main still awaits plugin roots, skill roots, and files within each root serially, and normal CLI output is built only after `runSecurityAudit` returns. I did not live-reproduce the exact 548s timing from the reporter's large macOS workspace.

## Solution Assessment

Unclear as one broad patch: bounded parallel scanning plus progress/timing is likely the narrow first fix, while persistent caching, configurable collector limits, and `--deep --only` need maintainer scope decisions.
