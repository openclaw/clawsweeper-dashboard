## Summary

Keep open. Current main still has the reported serial deep code-safety scan shape and no normal CLI output until the audit returns; the related bounded-concurrency PR was closed without merging, so it does not resolve this issue.

## What This Changes

The issue requests performance and observability improvements for `openclaw security audit --deep` when scanning many plugin and skill files.

## Best Possible Solution

Land a scoped bounded-concurrency scan plan with visible deep-audit phase progress/timing, then split persistent cache, per-collector limits, and collector selection into explicit follow-ups.

## Reproduction Assessment

Yes for a source-level reproduction: current main still awaits plugin roots, skill roots, and files within each root serially, and the CLI does not emit normal output until `runSecurityAudit` returns. I did not reproduce the reporter's exact many-file timing locally.

## Solution Assessment

Unclear as one patch. Bounded parallel scanning plus progress/timing looks like the narrow first repair, while persistent cache, per-collector limits, and `--deep --only` need maintainer UX and config scope decisions.
