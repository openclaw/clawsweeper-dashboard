## Summary

Keep open. Current main and v2026.6.8 still run deep plugin and skill code-safety scans serially at the collector and per-file levels, and the linked parallel-scan PR closed unmerged, so the central silent-stall performance problem remains.

## What This Changes

The issue requests faster and more observable `openclaw security audit --deep` code-safety scans for large plugin and skill trees.

## Best Possible Solution

Keep this issue open, capture a bounded real repro, then implement bounded scan concurrency plus stage progress/timing for deep code-safety work while deferring broader config or `--only` surface unless maintainers approve it.

## Reproduction Assessment

No, not as a live current-main reproduction. Source inspection does show the reported serial deep code-safety path through plugin collectors, skill collectors, and per-file scanner awaits.

## Solution Assessment

Unclear. Bounded concurrency plus progress/timing is the plausible narrow fix, but persistent cache, per-collector config, and `--only` need maintainer direction and live repro data before choosing the final design.
