## Summary

Keep open. Current main still serializes the deep plugin/skill code-safety scan path and normal CLI output still waits until the audit returns; the only linked implementation PR was closed unmerged, so the central performance and silent-window report remains unresolved.

## What This Changes

Request to reduce deep security audit stalls by adding bounded scanner concurrency, progress/timing visibility, persisted caching, collector limits, and optional collector selection.

## Best Possible Solution

Ship a narrow first change that preserves deterministic findings while adding bounded scanner concurrency and phase progress/timing, then split persisted cache, per-collector knobs, and `--only` into follow-ups if maintainers want them.

## Reproduction Assessment

Yes for a source-level reproduction: current main still awaits plugin roots, skill roots, and files within each root serially, and CLI output is produced only after `runSecurityAudit` returns. I did not live-reproduce the exact 548s timing from the reporter's workspace.

## Solution Assessment

Unclear as one patch. Bounded scanner concurrency plus progress/timing is the likely first maintainable fix, while persisted cache, per-collector limits, and collector-selection flags need maintainer UX/API scoping.
