## Summary

Keep open. Current main still has serial deep code-safety scanning and delayed normal CLI output; the related closed audit-hang issues addressed different plain-audit/plugin-runtime/debris paths, while this remaining throughput/progress request needs maintainer scoping.

## What This Changes

The issue reports silent, slow `openclaw security audit --deep` runs caused by serial deep code-safety scans across plugin and skill files.

## Best Possible Solution

Add a bounded, deterministic deep-audit scan plan with visible phase progress/timing first, then split persistent cache, per-collector limits, and `--only` into scoped follow-ups or an approved CLI design.

## Reproduction Assessment

Yes for a source-level reproduction: current main still awaits plugin roots, skill roots, and files within each root serially, and the CLI does not emit normal output until `runSecurityAudit` returns. I did not live-reproduce the exact 548s timing from the reporter's many-file macOS environment.

## Solution Assessment

Unclear as a single bundled fix: bounded parallel scanning plus progress/timing is likely the narrow first solution, but persistent cache, per-collector limits, and `--deep --only` add CLI/config policy that should be scoped by a maintainer before implementation.
