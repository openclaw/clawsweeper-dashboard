## Summary

Keep open. Current main still lacks the Action Button/AppIntent and Liquid Glass pieces, and the PR still has source-reproducible Live Activity lifecycle bugs that should be fixed before maintainer product review.

## What This Changes

The PR adds iOS 26-gated Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and expanded Live Activity/Dynamic Island status for the iOS app.

## Best Possible Solution

Keep the feature on the PR path if maintainers want it, but make Live Activity task cleanup owner-scoped, clear failed camera work immediately, and refresh activity identity when agent metadata changes.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: camera success HUD cleanup can reset a newer screen-recording working state, camera failures delay cleanup of the prior task, and agent metadata refresh can leave immutable activity attributes stale.

## Solution Assessment

No, not as currently patched. The feature belongs in the iOS app/widget surfaces, but the Live Activity working-state and identity updates need narrower ownership before this is a maintainable merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope HUD cleanup to the active task:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide path always calls `handleWorking(task: nil)` after its sleep. If screen recording or another long-running operation starts before that timer fires, the older camera timer can reset the active Live Activity status back to the connection state while the newer task is still running. Clear only the task this timer owns, or cancel/namespace the timer when a newer task starts.
  - confidence: 0.86
- **[P2] Refresh activity identity after agent metadata loads:** `apps/ios/Sources/Model/NodeAppModel.swift:2212`
  - body: `startActivity` can create the ActivityKit attributes before `refreshAgentsFromGateway()` resolves the default agent id/name, but the metadata refresh path does not call `refreshIdentity`. Because the widget renders `context.attributes.agentName`, the Dynamic Island or lock screen can stay on `Main` or a raw id until reconnect or manual agent switching. Refresh the active activity when the resolved name/session changes after agent metadata updates.
  - confidence: 0.78
- **[P3] Clear failed camera tasks immediately:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: Camera snap/clip publish a working task before awaiting camera work. If the operation throws, the error HUD path skips an immediate Live Activity update, so the Dynamic Island or lock screen can keep showing the failed camera task until the error HUD timer fires. Clear the task immediately on error without publishing the raw error text.
  - confidence: 0.78
