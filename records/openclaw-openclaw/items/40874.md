## Summary

Keep open: this is a real iOS feature PR that is not implemented on current main, but the branch should not merge as-is because two Live Activity state paths remain incorrect and the broad iOS 26 UX direction needs owner review.

## What This Changes

The branch adds iOS 26-gated Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and expanded Live Activity/Dynamic Island status UI in the iOS app.

## Best Possible Solution

Land the iOS platform feature only after it is rebased, owner-approved, and the Live Activity task and identity state transitions are made deterministic.

## Reproduction Assessment

Yes for the review findings by source inspection: a stale camera HUD auto-hide can clear a later screen-recording working state, and gateway metadata refresh can change the effective agent identity without recreating the ActivityKit attributes. I did not run the iOS app.

## Solution Assessment

No, not as submitted. The feature direction is plausible, but the branch should scope working-state clears to the active task and refresh ActivityKit identity after gateway metadata updates before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve newer working tasks when HUD timers expire:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide task always calls `handleWorking(task: nil)` after its sleep. If screen recording or another task starts before that old HUD timer fires, the timer clears the Live Activity back to the connection state while the newer task is still active.
  - confidence: 0.88
- **[P2] Refresh Live Activity identity after gateway metadata loads:** `apps/ios/Sources/Model/NodeAppModel.swift:689-694`
  - body: `refreshIdentity` is only called from `setSelectedAgentId`, but `refreshAgentsFromGateway()` can change the default agent, resolved name, and main session key after the Live Activity has started. Because ActivityKit attributes are immutable, the lock-screen/Dynamic Island label can stay on a fallback name or raw id until a manual agent switch or reconnect.
  - confidence: 0.84
