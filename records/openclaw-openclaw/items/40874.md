## Summary

Keep open: current main does not contain the iOS App Intent, Liquid Glass, or expanded Live Activity work, and the PR head still has a source-level Live Activity lifecycle blocker. The contributor’s screenshots are useful real-device proof and the security pass is clear, but the patch is not merge-ready.

## What This Changes

The PR adds iOS 26-gated Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and expanded Live Activity/Dynamic Island state for the iOS app.

## Best Possible Solution

Keep the feature on the PR path, but scope Live Activity working-state ownership before merge so only the operation that set a task can clear it, with camera errors clearing state without leaking raw error text.

## Reproduction Assessment

Yes. Source-level inspection of the PR head shows a camera HUD auto-hide timer can call handleWorking(task: nil) after screen recording has set its own long-running task, which can be reproduced with a focused simulator or manual sequence.

## Solution Assessment

No, not as currently patched. The feature belongs in the iOS app/widget surfaces, but the Live Activity cleanup needs owner or token scoping instead of a shared nil reset.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope HUD timer cleanup to its own task:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide path always calls `handleWorking(task: nil)` after its sleep. If screen recording or another long-running operation starts before that timer fires, the older camera timer can reset the active Live Activity status back to the connection state while the newer task is still running. Clear only the task this timer owns, or cancel/namespace the timer when a newer task starts.
  - confidence: 0.86
- **[P3] Clear live-activity task on camera failure:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: Camera snap/clip publish a working task before awaiting camera work. If the operation throws, the error HUD path skips an immediate Live Activity update, so the Dynamic Island or lock screen can keep showing the failed camera task until the error HUD timer fires. Clear the task immediately on error without publishing the raw error text.
  - confidence: 0.78
