## Summary

Keep open: current main does not implement the Action Button, Liquid Glass, and richer Dynamic Island feature set, but the PR should not merge as-is because camera capture failures can leave the Live Activity stuck in a working state.

## What This Changes

The PR adds iOS 26-gated Liquid Glass styling, an OpenTalkMode App Intent/Action Button shortcut that opens Talk Mode, and richer Dynamic Island/lock-screen Live Activity state for the iOS app.

## Best Possible Solution

Keep the feature branch open, clear the Live Activity working state on sanitized camera error paths, then review the iOS feature normally with device/build proof.

## Reproduction Assessment

Yes, by source inspection: camera snap/clip sets a Live Activity working task before awaiting capture, and the changed error HUD branch skips all Live Activity updates, leaving the previous task active. I did not run an iOS device repro in this read-only pass.

## Solution Assessment

No, not as-is. The privacy direction is right, but the error path should clear the prior working state without publishing raw error text to the Dynamic Island or lock screen.
