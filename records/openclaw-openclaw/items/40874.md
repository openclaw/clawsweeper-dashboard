## Summary

Keep open: current main does not contain the iOS 26 App Intent, Liquid Glass, or expanded Live Activity work, and PR head still has source-level Live Activity working-state cleanup bugs that should block merge.

## What This Changes

This PR adds iOS 26-gated Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and Dynamic Island/lock-screen Live Activity working-state UI to the iOS app.

## Best Possible Solution

Merge the iOS feature only after Live Activity working tasks are operation-scoped so camera failures clear safely and stale HUD timers cannot clear newer tasks.

## Reproduction Assessment

Yes, source-level. On PR head, camera snap/clip and screen-recording flows share global Live Activity working-state cleanup, so the stale-status and stale-timer paths are visible from the control flow without a device run.

## Solution Assessment

No, not as currently patched. The feature belongs in the iOS app/widget surfaces, but the working-task lifecycle needs task ownership or token scoping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear live-activity camera state on failures:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: Camera snap/clip publish `Taking photo...` or `Recording...` before awaiting the camera operation. If that operation throws, the error HUD skips the Live Activity update, so the Dynamic Island keeps showing the previous camera task until the generic error timer clears it; clear the working state immediately on error without publishing raw error text.
  - confidence: 0.84
- **[P2] Scope HUD timer cleanup to its task:** `apps/ios/Sources/Model/NodeAppModel.swift:1752-1753`
  - body: The camera HUD auto-hide timer always calls `handleWorking(task: nil)`. If screen recording or another working task starts before that timer fires, the older camera timer resets the shared Live Activity to connection state while the newer task is active; clear only the task this timer owns or namespace/cancel timers when a newer task begins.
  - confidence: 0.88
