## Summary

Keep open: the branch is not obsolete on current main, but PR-head source inspection still shows two Live Activity lifecycle defects that can make the Dynamic Island/lock-screen state stale or prematurely cleared.

## What This Changes

The PR adds iOS 26-gated Liquid Glass styling, an Open Talk Mode App Intent/App Shortcuts entry point, and expanded Live Activity/Dynamic Island connection and working-state UI in the iOS app.

## Best Possible Solution

Land an iOS-owned version after scoping Live Activity working-state cleanup to the owning task, clearing failed camera work immediately, and preserving the current AppIntent and Liquid Glass availability boundaries.

## Reproduction Assessment

Yes for the PR blockers: PR-head source inspection gives a high-confidence sequence where an older camera HUD timer can clear a newer screen-recording working state, and camera failures leave stale task text until later cleanup. I did not run a live iOS device or simulator reproduction in this read-only review.

## Solution Assessment

No, not as currently patched. The feature direction belongs in the iOS app/widget surfaces, but the Live Activity task lifecycle needs ownership-aware cleanup and immediate failure restoration before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope HUD cleanup to its own task:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide task always calls `handleWorking(task: nil)` after its delay. If a camera success HUD is visible and screen recording starts before that delay fires, the old timer restores connection state while recording is still active; track an owner/task token or clear only when the current task still matches this HUD.
  - confidence: 0.86
- **[P3] Clear Live Activity on camera failure:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: `snap` and `clip` set a camera working state before awaiting camera work, but the catch path shows an error HUD with `kind == .error`, which skips `handleWorking`. A failed camera operation leaves the previous task on the Dynamic Island/lock screen until the error HUD timer fires; clear the task immediately while still keeping raw error text off Live Activity.
  - confidence: 0.78
