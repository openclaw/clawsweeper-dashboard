## What This Changes

## Reproduction Assessment

Yes for the review findings by source inspection: a stale camera HUD timer can clear a newer screen-recording working state, and gateway metadata refresh can change ActivityKit identity without recreating immutable attributes. I did not run the iOS app.

## Solution Assessment

No, not as submitted. The Action Button path is now wired through RootCanvas, but the Live Activity task-clear and identity-refresh paths need deterministic fixes before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve newer working tasks when HUD timers expire:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The auto-hide HUD task always calls `handleWorking(task: nil)` after its sleep, even if screen recording has since set a newer long-lived working task. Because `handleScreenRecordInvoke` does not cancel this camera HUD timer, a prior camera HUD can revert Dynamic Island or lock-screen status while recording is still running.
  - confidence: 0.88
- **[P2] Refresh identity after gateway metadata changes:** `apps/ios/Sources/Model/NodeAppModel.swift:689-694`
  - body: This refresh path only runs when the selected agent changes, but `refreshAgentsFromGateway()` can change the default agent, resolved agent name, and main session key after the activity starts. Since ActivityKit attributes are immutable, the Live Activity label can stay on the fallback identity until a manual switch or reconnect.
  - confidence: 0.84
