## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve newer Live Activity work when HUD timers expire:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide task always calls `handleWorking(task: nil)` after its sleep. If a camera success HUD is still pending and screen recording or another long-lived task starts before that timer fires, the old timer clears the newer Dynamic Island working state even though work is still in progress. Track the task being cleared or only reset when the active task still matches the HUD that scheduled the timer.
  - confidence: 0.88
- **[P2] Refresh Live Activity identity after gateway metadata loads:** `apps/ios/Sources/Model/NodeAppModel.swift:689-694`
  - body: This refresh path only runs from `setSelectedAgentId`, but the reconnect flow starts the ActivityKit activity before `refreshAgentsFromGateway()` can resolve the default agent name/session key. Because ActivityKit attributes are immutable, the Dynamic Island can keep showing `Main` or a raw stale label after metadata arrives unless the user manually changes agents. Capture the previous effective identity in the gateway metadata refresh path and call `refreshIdentity` when it changes.
  - confidence: 0.84
