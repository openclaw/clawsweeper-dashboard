## Summary

Keep open. Current main does not implement this PR's central iOS App Intent/Liquid Glass/Dynamic Island work, and PR head still has source-reproducible Live Activity lifecycle bugs around shared working-state cleanup.

## What This Changes

The PR adds iOS 26-gated Liquid Glass UI, an Open Talk Mode App Intent/shortcut, and expanded Live Activity/Dynamic Island state for the iOS app.

## Best Possible Solution

Keep the feature on the PR path, but add ownership or token scoping so each Live Activity working task can only be cleared by the operation that set it.

## Reproduction Assessment

Yes. Source-level inspection of PR head shows a camera HUD auto-hide timer can call handleWorking(task: nil) after screen recording has set a long-running task, and camera failures leave the prior working task visible until the error HUD timer fires.

## Solution Assessment

No, not as currently patched. The feature belongs in the iOS app/widget surfaces, but Live Activity cleanup should be owner-scoped or task-matched instead of using a shared nil reset.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope HUD timer cleanup to its own task:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide path always calls `handleWorking(task: nil)` after its sleep. If screen recording or another long-running operation starts before that timer fires, the older camera timer can reset the active Live Activity status back to the connection state while the newer task is still running. Clear only the task this timer owns, or cancel/namespace the timer when a newer task starts.
  - confidence: 0.86
- **[P3] Clear live-activity task on camera failure:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: Camera snap/clip publish a working task before awaiting camera work. If the operation throws, the error HUD path skips an immediate Live Activity update, so the Dynamic Island or lock screen can keep showing the failed camera task until the error HUD timer fires. Clear the task immediately on error without publishing the raw error text.
  - confidence: 0.78
