## Summary

This PR should stay open: current main does not implement the Action Button/Liquid Glass/Dynamic Island feature set, and the PR branch still has two source-proven Live Activity blockers before merge.

## What This Changes

The branch adds iOS 26-gated Liquid Glass styling, an App Intent/Action Button entry into Talk Mode, and ActivityKit Dynamic Island/lock-screen connection status for the iOS app.

## Best Possible Solution

Keep the iOS 26 feature behind availability and compiler guards, but add deterministic Live Activity task ownership and refresh identity whenever the effective agent/session identity changes before merge.

## Reproduction Assessment

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve newer working tasks when HUD timers expire:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The auto-hide camera HUD task always calls `handleWorking(task: nil)` after its sleep. If screen recording starts before a prior camera HUD timer expires, the old timer can revert Dynamic Island or lock-screen status while recording is still running.
  - confidence: 0.88
- **[P2] Refresh identity after gateway metadata changes:** `apps/ios/Sources/Model/NodeAppModel.swift:689-694`
  - body: `refreshAgentsFromGateway()` can change the default agent, resolved agent name, and main session key after the activity starts, but identity refresh only runs from `setSelectedAgentId`. Since the widget displays static activity attributes, the label can stay on a fallback identity until manual switch or reconnect.
  - confidence: 0.84
