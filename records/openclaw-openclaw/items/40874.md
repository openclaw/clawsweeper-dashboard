## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard Live Activity cleanup by task ownership:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide task always calls `handleWorking(task: nil)` after its sleep. If screen recording or another Live Activity task starts before that timer fires, the older HUD timer clears the newer Dynamic Island/lock-screen working state while the newer task is still running; clear only when the current task still matches the HUD task, or move token ownership into `LiveActivityManager`.
  - confidence: 0.86
- **[P2] Refresh Live Activity labels after metadata updates:** `apps/ios/ActivityWidget/OpenClawLiveActivity.swift:56`
  - body: The widget now displays immutable `context.attributes.agentName`, but the activity can be started before gateway agent metadata/default-agent data is refreshed. Since only explicit `setSelectedAgentId` calls refresh identity, normal connection metadata updates can leave the lock-screen/Dynamic Island label stuck on `Main` or a raw id until reconnect; refresh the activity when `refreshAgentsFromGateway` changes the effective agent name or session key.
  - confidence: 0.8
