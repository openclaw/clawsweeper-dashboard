- **[P1] Consume the talk intent from RootTabs:** `apps/ios/Sources/RootCanvas.swift:200-207`
  - body: `OpenTalkModeIntent.perform()` writes the pending talk-mode flag, but this branch consumes it from `RootCanvas`. Current main mounts `RootTabs()` and the live PR is conflicting, so the Action Button/Siri flow needs a rebase that handles cold and warm launch in the active root shell.
  - confidence: 0.92
- **[P2] Only clear the task this timer started:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: This auto-hide timer always calls `handleWorking(task: nil)` after it sleeps. A camera success HUD can therefore clear a newer long-running working state, such as screen recording, because screen recording does not replace or cancel `cameraHUDDismissTask`; make the clear conditional on the displayed task still matching the one this timer created.
  - confidence: 0.86
- **[P2] Refresh activity identity after metadata changes:** `apps/ios/Sources/Model/NodeAppModel.swift:689-695`
  - body: The PR refreshes ActivityKit identity only when `setSelectedAgentId` runs, but gateway metadata refresh can also change the effective agent name or session key after the activity has already been requested. Because ActivityKit attributes are immutable, the Dynamic Island can keep showing fallback or old identity until reconnect unless metadata refresh also recreates the activity when the effective identity changes.
  - confidence: 0.82
