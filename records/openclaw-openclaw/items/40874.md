## Summary

Keep this PR open: current main does not contain the Action Button/Liquid Glass/working-task Live Activity work, and the PR head still has source-reproducible Live Activity cleanup bugs that can make Dynamic Island or lock-screen status stale or incorrect during overlapping camera and screen-recording flows.

## What This Changes

The PR adds iOS 26 Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and Live Activity/Dynamic Island task/status updates for the iOS app.

## Best Possible Solution

Keep the feature on the PR path, but make Live Activity working-state cleanup task-scoped or owner-matched and clear failed camera tasks immediately without publishing raw error text.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence sequence: start a camera success HUD, begin screen recording before the HUD auto-hide fires, and the older camera timer can call handleWorking(task: nil) while recording is still active; camera snap/clip failures also skip immediate Live Activity cleanup.

## Solution Assessment

No, not as currently patched. The feature belongs in the iOS app/widget surfaces, but the global nil reset is too broad for overlapping tasks; cleanup should be scoped to the operation that set the working state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope HUD timer cleanup to its own task:** `apps/ios/Sources/Model/NodeAppModel.swift:1753`
  - body: The camera HUD auto-hide path always calls `handleWorking(task: nil)` after its sleep. If screen recording or another long-running operation starts before that timer fires, the older camera timer can reset the active Live Activity status back to the connection state while the newer task is still running. Clear only the task this timer owns, or cancel/namespace the timer when a newer task starts.
  - confidence: 0.86
- **[P3] Clear live-activity task on camera failure:** `apps/ios/Sources/Model/NodeAppModel.swift:1736-1738`
  - body: Camera snap/clip publish a working task before awaiting camera work. If the operation throws, the error HUD path skips an immediate Live Activity update, so the Dynamic Island or lock screen can keep showing the failed camera task until the error HUD timer fires. Clear the task immediately on error without publishing the raw error text.
  - confidence: 0.78
