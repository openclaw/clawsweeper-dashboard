## Summary

Keep open: current main does not contain the App Intent/Liquid Glass additions, and the PR still has one source-level P2 regression in the camera-error Live Activity cleanup path. The contributor's device screenshots are enough real behavior proof for the visible feature path, so the remaining blocker is a focused code fix.

## What This Changes

This PR adds iOS Liquid Glass styling, an Open Talk Mode App Intent/shortcut, and expanded Dynamic Island/Live Activity connection and task status updates.

## Best Possible Solution

Keep the iOS feature direction, but restore Live Activity state on camera failures without exposing raw error text on the lock screen or Dynamic Island, then let maintainers make the final product/UX call.

## Reproduction Assessment

Yes, source-level: camera snap/clip set a working state before awaiting the camera operation, and the shared catch path calls showCameraHUD with kind .error. The PR's new kind guard skips both raw-error publication and cleanup, so a failing camera operation can leave the Live Activity stuck in the previous working state.

## Solution Assessment

No, not as currently patched: the privacy boundary should avoid publishing raw error strings while still calling handleWorking(task: nil) or an equivalent sanitized cleanup path. The rest of the feature is reasonably scoped to the iOS app and widget surfaces.
