## Summary

Keep open. Current main still builds the macOS gateway service PATH from user/version-manager directories and merges preserved PATH entries during install planning, and PR #75246 is an open, unmerged closing-syntax fix candidate, so this issue should stay open until that PR lands or maintainers choose a different policy.

## What This Changes

The issue asks for macOS gateway LaunchAgent install and doctor flows to write a deterministic canonical PATH instead of persisting interactive-shell Node version-manager and package-manager directories.

## Best Possible Solution

Review and land an approved variant of PR #75246, or close that PR with a maintainer-approved alternative that updates the macOS service PATH builder, install-plan PATH merge behavior, audit expectations, docs, and focused regression tests together.

## Reproduction Assessment

Yes for the source-level bug: current main’s Darwin service PATH helper and install-plan merge path still preserve version-manager/user PATH entries, and tests assert that behavior. The broader subprocess timeout and version-mismatch symptoms are not independently reproduced here.
