## Summary

Keep open. Current main still builds the macOS service PATH from user/version-manager directories and merges preserved plist PATH entries during install planning, while open PR #75246 uses closing syntax for this issue and has not landed.

## What This Changes

The issue asks for macOS gateway LaunchAgent install and repair flows to write a deterministic canonical PATH instead of carrying interactive-shell version-manager and package-manager directories into the plist.

## Best Possible Solution

Land a reviewed Darwin-specific fix that generates a canonical LaunchAgent PATH, avoids preserving old plist PATH entries on macOS rewrites, and updates the service-env, install-plan, audit, docs, changelog, and focused regression tests together.

## Reproduction Assessment

Yes. Source inspection and existing tests show a high-confidence reproduction path: Darwin service PATH generation includes version-manager directories, and install planning can merge existing plist PATH entries; live macOS launchd timeout behavior was not reproduced in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix for deterministic launchd behavior. The linked PR’s approach should be reviewed and landed or replaced before this issue is closed.
