## Summary

Keep open. Current main still has the reported service-environment behavior, and open PR #75246 specifically targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that macOS gateway LaunchAgent installs write a canonical minimal PATH instead of carrying interactive-shell Node version-manager and package-manager directories into the managed gateway service.

## Best Possible Solution

Review and land PR #75246, or an equivalent macOS-only fix, that generates a canonical LaunchAgent PATH, skips old plist/env-file PATH preservation on darwin, updates focused service-env/install-plan tests, and leaves Linux version-manager PATH policy to its own issue.

## Reproduction Assessment

Yes for the source-level issue: current main still builds darwin service PATHs from user/version-manager directories and still merges existing PATH entries during install planning. The live timeout/version-mismatch symptoms were not independently reproduced in this read-only checkout.

## Solution Assessment

Yes for this bug: a macOS-only canonical PATH plus darwin-specific skip of existing PATH merge is the narrow maintainable fix. Volta/version-manager opt-in behavior should remain a separate product decision, as tracked by #52184.
