## Summary

Keep open. Current main still generates macOS gateway service PATH values from user/version-manager directories and still merges preserved PATH entries during install planning, while an open, unmerged PR specifically targets this issue with closing syntax.

## What This Changes

The issue requests that macOS Gateway LaunchAgent install and repair flows write a deterministic minimal PATH instead of preserving interactive-shell or Node-version-manager PATH entries.

## Best Possible Solution

Review and land an approved macOS-only variant of PR #75246, or replace it with a maintainer-approved alternative that updates the service PATH builder, install-plan PATH merge behavior, audit/docs expectations, focused regression tests, and changelog together while keeping Linux policy separate.

Yes for the source-level issue: current main’s darwin service PATH helper includes version-manager/user directories, and install planning still merges existing PATH entries. The broader spawn timeout and gateway version-mismatch symptoms were not independently reproduced in this read-only review.

## Solution Assessment
