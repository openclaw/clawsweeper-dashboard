## Summary

Keep open. Current main still builds macOS service PATHs from user/version-manager directories and still merges existing plist PATH entries, and open PR #75246 targets this issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

Make macOS gateway LaunchAgent installation write a deterministic minimal PATH instead of capturing version-manager or package-manager entries from the user's shell or existing plist.

## Best Possible Solution

Review and land PR #75246, or an equivalent macOS-only fix, that canonicalizes LaunchAgent PATH generation, skips old plist PATH preservation on macOS, and updates focused regression tests/docs/changelog while leaving Linux/version-manager policy to separate work.

## Reproduction Assessment

Yes for the source-level bug: current main still generates darwin service PATHs with user/version-manager directories and still merges existing plist PATH entries. No live macOS launchd install was run here, so the runtime timeout symptoms remain based on the reporter's evidence.

## Solution Assessment

Yes. A macOS-only canonical LaunchAgent PATH plus regression coverage is the narrowest maintainable fix for this issue; broader version-manager preference behavior should stay separate.
