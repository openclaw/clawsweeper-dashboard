## Summary

Keep open. Current main still builds the macOS gateway service PATH from user and version-manager directories and still merges preserved PATH entries during install planning, while open PR #75246 specifically targets this behavior with closing syntax.

## What This Changes

Make macOS gateway LaunchAgent installs write a deterministic canonical PATH instead of preserving or deriving interactive version-manager PATH entries.

## Best Possible Solution

Review and land an approved macOS-only canonical PATH fix in PR #75246, or replace it with an equivalent maintainer patch that updates the service PATH builder, install-plan PATH merge behavior, audit/docs expectations, focused regression tests, and changelog while keeping Linux service PATH policy separate.

## Reproduction Assessment

Yes for the source-level issue: current main’s darwin service PATH helper includes user/version-manager paths and install planning still merges existing PATH entries. The broader timeout and version-mismatch symptoms were not independently reproduced in this read-only review.

## Solution Assessment

Unclear until maintainers finish review of PR #75246. A canonical macOS launchd PATH is a narrow maintainable direction for this report, but it should be evaluated alongside related version-manager compatibility reports such as #52184 and Linux PATH policy report #75220.
