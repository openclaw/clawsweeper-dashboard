## Summary

Keep open. Current main still generates macOS gateway service PATH entries from user and version-manager directories and still merges preserved existing PATH entries into reinstall plans; an open, unmerged closing-syntax PR already targets this issue, so the issue should remain open until that PR lands or maintainers choose a different policy.

## What This Changes

The issue requests that macOS gateway LaunchAgent installs write a canonical minimal PATH instead of preserving interactive-shell or Node version-manager directories.

## Best Possible Solution
