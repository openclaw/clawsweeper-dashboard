## Summary

Keep open: this is a draft maintainer PR, and the current head still has two source-reproducible CLI regressions in the help fast-path guards.

## What This Changes

The PR fast-paths bare `openclaw channels` parent help, lazy-loads provider-specific `channels add` options, threads argv through subcli registration, adds tests, and updates the changelog.

## Best Possible Solution

Keep the fast bare `channels` parent-help design, but normalize argv in the preAction parent-help check and register provider add options for both `channels add` and positional `channels help add`.

## Reproduction Assessment

Yes, by source inspection. Current `main` eagerly registers channel add metadata and only skips preAction startup for help/version, while the PR head's remaining guard gaps are visible for `channels help add` and duplicate Windows `node.exe` argv.

## Solution Assessment

No. The direction is narrow and maintainable, but this PR head still needs the lazy add-options guard and parent-help preAction argv normalization tightened before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve add flags for positional help:** `src/cli/channels-cli.ts:51`
  - body: OpenClaw enables `help [command]`, and Commander treats `channels help add` like help for `add`. This guard only accepts `channels add`, so that help form registers `add` without provider-specific setup options.
  - confidence: 0.9

  - confidence: 0.87
