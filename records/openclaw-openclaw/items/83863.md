## Label Justifications

- `impact:message-loss`: The reported failure hides a missing completion message or final deliverable behind successful task state.

## Summary

Keep open: current main has partial ACP `startTurn` failure handling, but legacy ACP done events and subagent `ok` finalization can still leave progress-only or undelivered completion work represented as successful durable task state.

## What This Changes

The issue requests that ACP/Codex child tasks with only progress output or failed required completion delivery stop being recorded as successful durable task outcomes.

## Best Possible Solution

Carry an explicit final-deliverable and required-completion-delivery contract through ACP and subagent task finalization, mapping missing final output or exhausted required delivery to blocked/failed terminal state while preserving valid text-only finals.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: legacy ACP text_delta plus done still reaches the succeeded task path, and subagent ok completion finalizes before failed required delivery only updates deliveryStatus. I did not execute a live ACP replay in this read-only review.

## Solution Assessment

Yes, the requested direction is the maintainable fix if it uses structured completion and delivery state rather than last-text guessing. The existing startTurn failure handling should be preserved while extending coverage to legacy ACP and subagent finalization.
