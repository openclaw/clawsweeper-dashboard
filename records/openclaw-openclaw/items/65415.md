## Summary

Keep open: the PR is still a proposed new public config knob, but current main has moved the affected timeout and announce-delivery surfaces enough that this branch needs maintainer product approval, a rebase, and contributor real behavior proof before it can be considered mergeable.

## What This Changes

The PR adds an optional `agents.defaults.subagents.gatewayTimeoutMs` config/schema key, applies it as a timeout floor in `callSubagentGateway`, and adds focused subagent spawn timeout tests.

## Best Possible Solution

If maintainers want this knob, rework it as a documented raise-only floor over current spawn/control timeout helpers and direct control paths; otherwise keep the merged in-process announce fix and existing timeout contracts without adding this config surface.

## Reproduction Assessment

Yes for the review findings by source inspection: current main has 60s/run-timeout-aware spawn budgets and direct 10s control calls that this wrapper-only patch does not align with. The original 40+ lane high-concurrency symptom was not live-reproduced in this read-only review.

## Solution Assessment

No. The proposed wrapper-level config floor is not the best fix unless maintainers first approve the public config surface and it is layered over current helpers, direct control calls, docs, and real behavior proof.
