## Summary

Keep open. Current main still has a source-level Telegram typing gap: `thinking` mode is documented to start on the first reasoning delta, but `signalReasoningDelta` returns before any renderable text exists, so long pre-first-token reasoning can leave the user without a visible keepalive.

## What This Changes

Restore continuous Telegram typing keepalives while an agent is actively reasoning, using tools, or otherwise working before the final reply.

## Best Possible Solution

Make reasoning deltas start and refresh the existing typing loop before any text delta, keep the current tool-start behavior, add focused regression coverage, and live-prove the Telegram DM indicator path after the patch.

## Reproduction Assessment

Yes, source-level: create a `thinking` mode typing signaler and call `signalReasoningDelta` before any text delta; current main returns without starting the typing loop despite the documented contract. No live Telegram DM reproduction was run in this read-only review.

## Solution Assessment

Yes for the narrow repair direction: remove the pre-text `hasRenderableText` gate from `signalReasoningDelta` and update the regression test. A new heartbeat mode or fallback indicator would be broader and should wait unless live proof shows another remaining gap.
