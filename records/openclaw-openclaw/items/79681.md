Keep open. Current main and the latest release still have the source-level `thinking` mode gap: reasoning deltas are documented to start typing, but `signalReasoningDelta` returns until a text delta has already made the reply renderable; the tool-start portion is already implemented, so the remaining fix should stay narrow.

## What This Changes

The issue requests durable Telegram typing keepalives during long active agent work, including reasoning, tool calls, and waits before the final reply.

## Best Possible Solution
