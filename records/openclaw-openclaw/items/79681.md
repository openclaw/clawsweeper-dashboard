## Summary

Keep open. Current main still has a source-level Telegram typing gap: `thinking` mode is documented to start on the first reasoning delta, but `signalReasoningDelta` still returns until renderable text has already arrived.

## What This Changes

The issue requests restoring continuous Telegram typing keepalives during active long-running agent work, especially reasoning- and tool-heavy turns.

## Best Possible Solution

Make reasoning deltas start and refresh the existing typing loop before any text delta, keep current tool-start behavior, add focused regression coverage, and live-prove the Telegram DM indicator path after the patch.

## Reproduction Assessment

Yes, at source level. Create a `thinking` mode typing signaler and call `signalReasoningDelta` before any text delta; current main returns before starting the typing loop despite the documented contract.

## Solution Assessment

Yes for the narrow repair direction. Remove or bypass the pre-text `hasRenderableText` gate for `signalReasoningDelta`, while leaving broader fallback indicators or new heartbeat modes out of this bug fix unless live proof shows another gap.
