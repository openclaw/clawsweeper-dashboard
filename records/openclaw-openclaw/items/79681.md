## Summary

Keep open: current main improves some Telegram typing paths, but `typingMode: "thinking"` still ignores reasoning deltas until visible assistant text exists, which contradicts the documented first-reasoning-delta behavior and matches the independent repro comment.

## What This Changes

The issue asks OpenClaw to keep Telegram typing indicators visible during long active agent work, especially before the first assistant text arrives.

## Best Possible Solution

Keep the issue open and fix `thinking` mode so reasoning deltas start or refresh the typing loop before visible assistant text, with tests updated to preserve message-mode suppression.

## Reproduction Assessment

Yes for source reproduction: current main still returns from `signalReasoningDelta()` before starting typing when no visible assistant text has arrived, while docs say `thinking` starts on the first reasoning delta.

## Solution Assessment

Yes: the best narrow fix is to remove the visible-text gate from reasoning-triggered typing while keeping `message` mode gated on renderable text and preserving silent/heartbeat suppression.
