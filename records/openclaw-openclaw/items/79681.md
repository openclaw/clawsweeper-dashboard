## Summary

Keep open. Current main still has a source-confirmed gap in the documented typing lifecycle: `thinking` mode says typing starts on the first reasoning delta, but the signaler ignores reasoning until renderable text has already appeared; the latest issue comment adds a real Telegram repro for that path.

## What This Changes

Telegram typing keepalives should remain visible during long active agent runs, including reasoning and tool-heavy turns before the final reply.

## Best Possible Solution

Restore the documented shared typing contract so reasoning deltas can start and refresh the typing loop before text appears, keep the existing tool-start refresh behavior, and close only after focused tests plus real Telegram proof.

## Reproduction Assessment

Yes, at source level. The docs promise `thinking` starts on the first reasoning delta, but current main returns from `signalReasoningDelta` until renderable text exists, and the latest comment reports the same failure in a real Telegram run.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared typing signaler and its tests, not a Telegram-only presence feature or new config path.
