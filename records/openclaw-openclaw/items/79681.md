## Summary

Keep open. Current main still has a source-level gap for the documented `thinking` typing mode, and the newest issue comment adds an independent real-world repro plus a plausible root cause that has not been fixed or superseded.

## What This Changes

The issue requests restoring continuous Telegram typing keepalives during active agent work.

## Best Possible Solution

Patch the shared typing signaler so reasoning deltas can start and refresh typing before renderable text, while preserving tool start/update TTL refreshes and validating with focused unit coverage plus live Telegram proof when available.

## Reproduction Assessment

Yes, source-level for the documented `thinking` mode path: reasoning stream events call `signalReasoningDelta()`, but current main returns before renderable text exists. A live current-main Telegram repro for the original `instant` setup was not run.

## Solution Assessment

Yes for the narrow source-level fix: drop the renderable-text dependency from reasoning-triggered typing and keep tool update refreshes separate from progress rendering. A durable processing-message fallback would be a separate product feature, not required to fix this regression.
