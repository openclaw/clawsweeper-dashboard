## Summary

Keep open. Current main still has a source-level Telegram typing gap: `thinking` mode is documented to start on the first reasoning delta, but `signalReasoningDelta` returns before any renderable text exists, so long pre-first-token reasoning can leave the user without a visible keepalive.

## What This Changes

The issue requests restoring durable Telegram typing keepalives while active Telegram agent work is ongoing.

## Best Possible Solution

Make the shared typing signaler treat reasoning deltas as active work in `thinking` mode before any text delta, keep the current tool-start behavior, and prove the visible Telegram DM path after the code fix.

## Reproduction Assessment

Yes, source-level: create a `thinking` mode typing signaler and call `signalReasoningDelta` before any text delta; current main returns without starting the typing loop despite the documented contract. A live Telegram DM reproduction was not run in this read-only review.

## Solution Assessment

Yes for the narrow repair direction: remove the pre-text `hasRenderableText` gate from `signalReasoningDelta` and update regression coverage. A new heartbeat mode, fallback indicator, or config knob would be broader product work unless live proof shows the narrow fix is insufficient.
