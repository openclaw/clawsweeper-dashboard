## Summary

Keep open: current main and the latest release still pass only caller-supplied `preferredKind` into id-like plugin target resolution, while this PR supplies the already-inferred kind and adds focused outbound/cron coverage. The patch looks correct, but it changes shared plugin resolver semantics and message-target routing, so it should stay open for maintainer acceptance rather than be closed as cleanup.

## What This Changes

This PR changes outbound id-like target resolution to pass the inferred target kind into plugin target resolvers and updates outbound/cron tests for `preferredKind` propagation.

## Best Possible Solution

Merge this as the shared resolver fix once maintainers accept the inferred-kind `targetResolver` contract and current checks remain green.

## Solution Assessment

Yes, pending maintainer acceptance of the shared contract. Passing the already-computed `kind` at the shared resolver seam is narrower than adding Discord-specific special cases, and cron reaches the same resolver path through shared delivery resolution.
