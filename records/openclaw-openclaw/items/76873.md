## Summary

Keep open: this is a plausible source-backed resolver bug, but the PR still has a catalog-plugin lookup regression and lacks inspectable after-fix real behavior proof, so it should not merge as-is.

## What This Changes

The branch extracts target-kind inference into target-normalization, forwards the inferred preferredKind through outbound and cron id-like resolution, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the inferred preferredKind propagation, change the shared helper to use the same loaded-or-catalog target-read lookup as neighboring helpers, add an unloaded catalog-plugin regression test, and require redacted real Discord proof before merge.

## Reproduction Assessment

Yes, at source level: current main computes or can infer target kind but does not carry it into the outbound and cron id-like resolver seams. I did not run live Discord delivery during this read-only review.

## Solution Assessment

No, not as written. Forwarding inferred preferredKind is the right narrow direction, but the new shared helper must preserve catalog-plugin inference before this is merge-ready.
