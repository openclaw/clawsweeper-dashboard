## Summary

Keep this PR open: current main still drops inferred user/direct target intent before id-like resolution, and the PR is a narrow fix using the existing resolver contract with focused tests and a changelog entry.

## What This Changes

The PR extracts shared target-kind inference and passes the inferred kind into outbound and cron id-like target resolution for Discord-style direct-message targets.

## Best Possible Solution

Land the narrow resolver fix after exact-head targeted resolver/cron tests and the changed gate pass.

## Reproduction Assessment

Yes. Source inspection on current main shows inferred target kind is computed but not forwarded in outbound id-like resolution, and cron id-like resolution has no preferred kind at all.

## Solution Assessment

Yes. Reusing shared inference and the existing `preferredKind` resolver contract is the narrowest maintainable fix; no new config, API surface, or Discord runtime send behavior is needed.
