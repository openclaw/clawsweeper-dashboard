## Summary

Keep open: current main and v2026.6.9 still pass only caller-supplied `preferredKind` into id-like plugin target resolution, while this PR source-proves and fixes the inferred DM/user kind handoff. I found no blocking correctness finding, but the change should stay in maintainer review because it changes a shared plugin resolver argument that can affect DM-vs-channel delivery.

## What This Changes

This PR changes shared outbound target resolution so inferred target kind is passed into id-like plugin resolution, with outbound resolver and cron delivery tests updated for the new `preferredKind` handoff.

## Best Possible Solution

Merge this once maintainers accept inferred-kind propagation as the shared id-like `targetResolver` contract; otherwise narrow the fix to a Discord-owned resolver path.

## Reproduction Assessment

Yes for source-level reproduction: current main computes `kind` from target inference but does not pass it into id-like plugin resolution. I did not run live Discord or local tests in this read-only review, but the PR includes redacted live Discord after-fix proof.

## Solution Assessment

Yes, pending maintainer acceptance of the shared contract. Passing the already-computed kind at the shared resolver seam is narrower than adding Discord-specific special cases, and the remaining concern is upgrade compatibility for shared plugin resolvers.
