## Summary

Keep open: the bug is valid and the PR is narrow, but the current diff collapses channel target intent to group before id-like plugin resolution, which regresses the existing preferredKind contract.

## What This Changes

The PR extracts shared target-kind inference, passes inferred kind into outbound and cron id-like target resolution, adds focused resolver/cron tests, and adds a Discord changelog entry.

## Best Possible Solution

Keep separate resolver intent and directory lookup kind: pass user/group/channel intent to id-like plugin resolution, coerce channel to group only where directory lookup or legacy result semantics require it, and keep the new DM/user regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the outbound path computes inferred kind but forwards only params.preferredKind, while cron delivery calls maybeResolveIdLikeTarget without any preferredKind.

## Solution Assessment

No. The proposed direction is narrow and maintainable, but the current diff maps channel intent to group before id-like resolver calls; the safer fix is to keep resolver preferredKind separate from directory kind.
