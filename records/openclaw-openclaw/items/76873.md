## Summary

Keep this PR open: it fixes a real current-main target-kind propagation gap, and the author added useful live Discord verification, but the diff introduces a channel-kind regression that should be repaired before merge.

## What This Changes

The PR extracts target-kind inference into target normalization, passes inferred kinds into outbound and cron id-like resolution, adds focused tests, and records a Discord changelog fix.

## Best Possible Solution

Keep the inferred channel kind on resolved targets, while coercing only the directory lookup kind if that lookup still needs group-list APIs.

## Reproduction Assessment

Yes. Source inspection on current main shows the outbound id-like path drops the computed kind and the cron path sends no preferredKind at all; the proposed tests cover those seams, though I did not run them in this read-only review.

## Solution Assessment

No. The intended fix direction is narrow, but this implementation collapses inferred channel targets to group before returning the resolved target kind; preserving channel and only adapting directory lookup is safer.
