## Summary

Keep open: the reported bug path is real and the fix direction is narrow, but the current branch still has a blocking catalog-plugin fallback regression and lacks inspectable after-fix real behavior proof.

## What This Changes

The branch moves target-kind inference into target normalization and passes the inferred preferredKind through outbound and cron id-like target resolution, with focused tests and a changelog entry.

## Best Possible Solution

Keep the preferredKind propagation, reuse the existing loaded-or-catalog target-read helper for inferred kind, add unloaded catalog-plugin regression coverage, and require redacted Discord proof before merge.

## Reproduction Assessment

Yes, at source level: current main can infer a user/direct target kind, but outbound and cron id-like resolution do not carry that inferred kind into maybeResolveIdLikeTarget. I did not run live Discord delivery during this read-only review.

## Solution Assessment

No, not as written. Forwarding the inferred preferredKind is the right narrow fix, but the new helper must preserve catalog-plugin inference before this is merge-ready.
