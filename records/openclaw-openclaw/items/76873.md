## Summary

Keep open: the reported Discord target-resolution bug is source-reproducible on current main, but the latest PR still has a correctness regression in the new inference helper and its real Discord proof is not inspectable enough for merge.

## What This Changes

The PR adds shared target-kind inference, forwards inferred kind into outbound and cron id-like target resolution, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the preferredKind propagation, make inferTargetResolveKind reuse the existing loaded-or-catalog target-read helper, add unloaded catalog-plugin regression coverage, and require inspectable redacted Discord runtime proof before merge.

## Reproduction Assessment

Yes, at source level: current main infers direct/user intent but does not pass that inferred kind into outbound and cron id-like resolution. I did not run live Discord delivery or tests during this read-only review.

## Solution Assessment

No, not as written. Forwarding preferredKind is the narrow maintainable fix, but the new helper must preserve bundled/catalog plugin inference before the patch is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:
