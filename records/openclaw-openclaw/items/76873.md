## Summary

Keep open for maintainer review: the patch addresses a source-proven shared id-like target-resolution gap, but it intentionally changes what plugin `targetResolver` implementations receive, so it needs owner acceptance rather than cleanup closure.

## What This Changes

The PR changes shared outbound id-like target resolution to pass the inferred target kind into plugin resolution and updates outbound/cron tests for `preferredKind` propagation.

## Best Possible Solution

Merge this once maintainers accept inferred-kind propagation as the shared id-like `targetResolver` contract; otherwise choose a narrower Discord-owned resolver path.

## Reproduction Assessment

Yes, from source inspection. Current main computes an inferred kind but passes only `params.preferredKind` into id-like plugin resolution, while Discord's resolver branches on `preferredKind`; I did not run tests because this review is read-only.

## Solution Assessment

Yes, pending maintainer acceptance of the shared contract. Passing the already-computed kind at the shared resolver seam is narrower than adding Discord-specific special cases, and cron reaches the same resolver path through shared delivery resolution.
