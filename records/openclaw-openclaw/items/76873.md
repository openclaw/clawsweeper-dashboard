## Summary

Keep open: current main and v2026.6.8 still pass only the caller-supplied preferred kind into id-like resolution, while this PR makes the already-inferred user/direct intent reach Discord's resolver. The patch looks narrow and correct, but the shared target-resolution behavior still deserves maintainer review before landing.

## What This Changes

The branch changes the shared outbound target resolver to pass the computed inferred kind into id-like plugin resolution and updates outbound/cron target-resolution tests.

## Best Possible Solution

Land the focused preferredKind handoff after maintainers accept the shared resolver semantics and exact current-main validation passes.

## Reproduction Assessment

Yes. Source inspection on current main shows `detectTargetKind` can infer `user`, but the id-like resolver still receives undefined unless the caller supplied `preferredKind`; Discord then chooses channel defaults unless it sees `preferredKind: "user"`.

## Solution Assessment

Yes. Passing the already-computed `kind` into the existing id-like resolver is the narrow owner-boundary fix; a Discord-specific workaround would duplicate routing policy in the plugin.
