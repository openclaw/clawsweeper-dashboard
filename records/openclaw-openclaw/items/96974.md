## Summary

Keep open. Current main still defaults Gemini-backed web_search to `gemini-2.5-flash`, but Google’s current public docs still list Gemini 2.5 Flash as Google Search-supported, so the next step is a maintainer/live key-backed check before changing the shipped default.

## What This Changes

The issue reports that Gemini-backed web_search defaults to a retired model id, causing out-of-the-box 404s unless users override the Google plugin webSearch model.

## Best Possible Solution

First verify the default with a real Gemini key; if it 404s, make a plugin-local default-model update to a Google-supported stable search-capable model while preserving explicit `webSearch.model` overrides and focused tests.

## Reproduction Assessment

Unclear for a full live reproduction: current main definitely routes the unconfigured Gemini search path to `models/gemini-2.5-flash:generateContent`, and the reporter supplied a key-backed 404. This review could not reproduce it without a real key, and Google’s current public docs still list Gemini 2.5 Flash as search-grounding supported.

## Solution Assessment

Unclear as proposed. A plugin-local default update is likely the right fix if the 404 is confirmed, but choosing a rolling alias versus a stable model needs maintainer judgment because Google documents `latest` aliases as hot-swapped.
