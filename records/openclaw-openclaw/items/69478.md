## Summary

Keep open: the merged system-event queue fix is on current main, but the reported duplicate approval cascade still has a current source path because exec approval creation, gateway registration, and approval forwarding remain keyed by fresh approval IDs rather than a stable exec intent.

## What This Changes

The issue requests deduplication or cancellation of repeated exec approval prompts for the same failover retry intent, keyed by runId/contextKey or an equivalent stable approval identity.

## Best Possible Solution

Keep this as the canonical tracker for a collision-safe exec approval intent key or stale-approval cancellation path in approval registration/forwarding, while preserving the shipped queue-side dedupe.

## Reproduction Assessment

Yes, at source level. Creating two exec approval requests for the same command/session intent with different generated IDs still reaches ID-keyed registration and forwarding paths, so the second request is treated as a separate pending approval.

## Solution Assessment

No, the current merged system-events fix is only a partial solution. The remaining maintainable fix belongs at approval registration/forwarding or failover cleanup with a stable, collision-safe approval intent identity.
