## What This Changes

The issue asks OpenClaw to coalesce or cancel repeated exec approval system events when heartbeat failover retries the same command under fresh run or approval IDs.

## Best Possible Solution

Land or replace PR #73040 with a focused fix that carries stable exec intent dedupe or cancels stale pending approval events, with regression coverage for interleaved retries under fresh IDs.

Yes. A source-level reproduction is to enqueue an approval-facing event, interleave another event, then enqueue the same command intent again under a fresh ID; current main queues the second copy because it only compares `lastText`.

## Solution Assessment

No for current main. The active PR is the right implementation candidate to review, but the merged fix should prove the actual failover path where both run IDs and approval IDs can change.
