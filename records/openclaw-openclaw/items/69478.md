## Summary

Keep open: the merged queue-side system-event dedupe is present on current main, but the reported fresh-approval-ID cascade still has a source-level path through approval creation, gateway registration, and forwarding keyed by generated approval IDs rather than a stable exec intent.

## What This Changes

The issue requests deduplication or cancellation of repeated exec approval prompts for the same heartbeat/failover exec intent when retries generate fresh approval IDs.

## Best Possible Solution

Define and test a collision-safe exec approval intent identity, or cancel stale approval requests during failover, in the approval registration/forwarding path while preserving the shipped queue-side dedupe.

## Reproduction Assessment

Yes, at source level. Current main still generates fresh approval IDs/context keys and keys manager/forwarder state by those IDs, so an identical retried exec request can become a separate pending approval.

## Solution Assessment

No. The merged system-events fix is useful but partial; the remaining fix belongs in approval registration, forwarding, or failover cleanup with a stable, collision-safe exec intent identity.
