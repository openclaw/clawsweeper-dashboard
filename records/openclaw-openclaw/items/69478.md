The issue reports duplicate Telegram exec approval prompts for the same command intent when heartbeat/failover retries re-enqueue approvals under new IDs.

Keep this issue as the canonical tracker for a stable exec approval intent identity or failover-cancellation path across retries, with security review and live Telegram proof before close.
