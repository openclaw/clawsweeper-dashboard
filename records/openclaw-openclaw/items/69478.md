## Summary

Keep open: current main still lacks stable-intent dedupe for approval-facing system events, and the open closing PR #73040 has not merged yet.

## What This Changes

The issue asks OpenClaw to coalesce or cancel repeated exec approval/system events so heartbeat failover retries do not create duplicate approval prompts under fresh IDs.

## Best Possible Solution

Keep the issue open until a merged fix carries a stable command/session-level dedupe key through exec approval/system-event enqueueing or cancels stale pending approval events before retrying, with regression tests for interleaved retries and fresh IDs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to enqueue an approval-facing event, interleave another event, then enqueue the same command intent again under a fresh run or approval ID; the current queue only checks the immediately previous text, and the report includes logs/screenshots of the live cascade.

## Solution Assessment

No for current main. The linked PR is the right candidate to review, but the maintainable fix should be validated against the actual failover path where both run IDs and approval IDs can change.
