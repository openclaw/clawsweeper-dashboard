## Summary

Keep open: current main has the queue-side system-event dedupe and later post-approval follow-up safeguards, but the initial exec approval prompt path still creates and forwards fresh approval IDs without a stable command/session intent key.

## What This Changes

The issue requests deduplication or cancellation for repeated exec approval prompts when failover retries requeue the same command intent under new approval IDs.

## Best Possible Solution

Keep this open until maintainers define and implement a stable exec approval intent identity or cancellation rule before prompt delivery, with regression coverage and live transport proof for duplicate-prompt suppression.
