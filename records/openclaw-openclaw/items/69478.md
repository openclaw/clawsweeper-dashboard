## Summary

Keep open: current main has the narrowed queue-wide keyed system-event dedupe, but the approval registration/forwarder path still keys pending approvals by fresh approval IDs and maintainer review explicitly said the merged queue fix should not close this issue by itself.

## What This Changes

The issue requests deduplication or cancellation of repeated exec approval prompts so heartbeat failover retries do not send duplicate approval requests under new IDs.

## Best Possible Solution

Keep this issue open as the canonical tracker for a security-preserving stable exec approval intent key or stale-approval cancellation path, while retaining the merged queue-side dedupe as a partial fix.

## Reproduction Assessment

Yes, at source level. Creating two approval requests with the same command/session intent but different IDs still reaches ID-keyed registration and forwarder maps, so the second request is treated as a distinct pending approval.

## Solution Assessment

No for closing this issue. The merged queue-side dedupe is a good partial fix, but the best remaining solution needs a stable approval-intent identity or cancellation path in the approval registration/forwarding flow rather than only `enqueueSystemEvent`.
