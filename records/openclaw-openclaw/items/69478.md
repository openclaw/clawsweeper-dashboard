## Summary

Keep open: current main contains the narrowed system-event queue dedupe, but the reported approval cascade still has a source-level path because approval creation, gateway registration, approval replay/runtime handling, and forwarding are keyed by fresh approval/request IDs rather than a stable exec intent.

## What This Changes

The issue reports duplicate exec approval prompts when heartbeat failover re-enqueues the same command under fresh approval/run IDs.

## Best Possible Solution

Define and test a collision-safe exec approval intent identity or stale-approval cancellation path in approval registration/forwarding while keeping the shipped queue-side dedupe intact.

## Reproduction Assessment

Yes, at source level. Two identical exec approval requests with different generated IDs still pass through ID-keyed registration and forwarding paths as separate pending approvals.

## Solution Assessment

No, the merged system-events fix is only a partial solution for the reported cascade. The remaining fix belongs in approval registration/forwarding or failover cleanup with a stable, collision-safe approval intent identity.
