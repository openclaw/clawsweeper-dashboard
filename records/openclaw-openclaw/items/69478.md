## Summary

## What This Changes

The issue reports duplicate Telegram exec approval prompts when retry/failover re-enqueues the same exec approval intent under fresh IDs.

## Best Possible Solution

Keep this issue open as the canonical tracker for a stable exec approval intent identity or cancellation path across retry/failover, with maintainer review for the security and session-state semantics.
