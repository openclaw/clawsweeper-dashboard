## Summary

Keep open: this is useful security hardening, but the current patch is not merge-ready because it leaves documented adjacent config snapshots and JSON5-authored backups outside reliable audit coverage.

## What This Changes

The PR adds adjacent config backup discovery, scans those files in `openclaw secrets audit`, and adds a regression test for a stale plaintext provider key in a backup.

## Best Possible Solution

Land this as broader secrets-audit hardening after it covers all managed adjacent config snapshots with config-compatible parsing, reports unreadable snapshots clearly, and includes redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create an adjacent `openclaw.json.pre-update` or JSON5-formatted `openclaw.json.bak` with a plaintext provider key and the proposed scan path will not report it.

## Solution Assessment

No. The PR is in the right audit surface, but the maintainable fix should reuse config-compatible parsing semantics and include the managed adjacent snapshot names OpenClaw already writes and documents.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Include managed config snapshots in backup audit:** `src/secrets/storage-scan.ts:34-38`
  - body: The matcher only returns `.bak`/`.bak.*`/`.*.bak`, but OpenClaw also writes adjacent `.pre-update`, `.rejected.*`, and `.clobbered.*` config snapshots that docs tell users to inspect. A stale plaintext key in one of those files would still let `secrets audit --check` report clean, so the new audit coverage misses part of the same managed backup surface.
  - confidence: 0.9
- **[P2] Parse backup configs with config-compatible semantics:** `src/secrets/audit.ts:269-272`
  - body: This path uses `readJsonObjectIfExists`, which calls `JSON.parse` and then silently continues when parsing fails. Active configs may be JSON5 and `.bak` is a byte copy of the config, so a commented or trailing-comma backup with plaintext secrets is scanned in name only and produces no finding.
  - confidence: 0.88
