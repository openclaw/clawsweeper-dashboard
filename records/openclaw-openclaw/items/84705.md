## Best Possible Solution

Land with the documented strict audit behavior if maintainers accept the upgrade impact; otherwise soften backup-only findings behind non-check-blocking or opt-in behavior before merge.

## Solution Assessment

Yes for the implementation boundary: the audit command is the right place to surface stale plaintext config artifacts. The remaining question is whether strict `--check` failure is the desired rollout policy or should be softened before merge.
