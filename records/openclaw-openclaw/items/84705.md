## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent `openclaw.json` backup and managed snapshot files for plaintext secrets.

## Best Possible Solution

Land the focused audit expansion if maintainers accept the stricter `--check` compatibility impact, while keeping the broader API-key protection roadmap tracked in https://github.com/openclaw/openclaw/issues/11829.

## Reproduction Assessment

Yes. Source inspection shows current main has no adjacent config backup discovery in `runSecretsAudit`, so a plaintext key in `openclaw.json.pre-update` or `openclaw.json.bak` would be missed before this branch.

## Solution Assessment

Yes, with maintainer signoff on compatibility. The branch reuses the existing secret target registry and config JSON5 parser, covers documented snapshot names, and keeps the change inside the existing audit command rather than adding a parallel scanner.
