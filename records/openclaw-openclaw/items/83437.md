## Summary

Keep open. Current main still lets memory-lancedb auto-recall prepend recalled memory as prompt text, and the direct LanceDB store/search path has no agent ownership field; related open work covers parts of the problem but does not resolve the data-only instruction boundary.

## What This Changes

The issue reports that memory-lancedb relevant-memory recall can influence agent behavior as if historical memory were task instructions, including possible cross-agent memory leakage.

## Best Possible Solution

Define a maintainer-approved memory boundary: scope direct memory-lancedb recall by agent/session or route it through the scoped active-memory path, and make recalled memory data-only at the prompt boundary with compatibility coverage and live two-agent proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the reported mechanism: current memory-lancedb auto-recall searches an unscoped store, returns recalled memory as prompt text, and the runner prepends it to the turn prompt.

## Solution Assessment

Unclear. Warning-only prompt delimiting is not the best durable fix; maintainers should choose a scoped recall/data-only boundary, with https://github.com/openclaw/openclaw/pull/40578 and https://github.com/openclaw/openclaw/issues/69541 treated as related but incomplete work.
