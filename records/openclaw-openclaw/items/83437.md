## Summary

Keep open: current main has the merged manual `memory_recall` guard, but the automatic `before_prompt_build` path still searches the shared memory database and injects recalled text into prompt context without requiring agent/session scope. The closed unmerged scoped-recall PR is useful evidence, not a landed fix.

## What This Changes

The issue reports memory-lancedb relevant memories being treated as task instructions and leaking across agent/session scope.

## Best Possible Solution

Land a maintainer-approved memory-lancedb fix that keeps recalled memories data-only and scopes automatic recall/capture to the current agent/session, with regression coverage and explicit upgrade/security proof.

## Reproduction Assessment

Yes. Source inspection shows the automatic `before_prompt_build` hook can run without agent/session scope, search the shared `db`, and return recalled memories as `prependContext`; no live LanceDB/OpenAI run was performed in this read-only review.

## Solution Assessment

Unclear as a final implementation choice. A scoped, data-only boundary is the right end state, but the automatic-store migration/fail-closed behavior needs maintainer and security approval before a new repair should land.
