## Summary

Current main partially hardens recalled memory text, but the central agent/session scope leakage remains: memory-lancedb auto-recall and auto-capture still use the shared LanceDB store and inject recalled data through prompt context. ## What This Changes

The issue asks for memory-lancedb relevant memory to stay data-only and agent/session-scoped so historical memories cannot be treated as fresh instructions or leak into unrelated agents.

## Best Possible Solution

Keep this issue open for a maintainer-approved memory-lancedb fix that makes automatic recall/capture agent/session-scoped, data-only, and manageable through documented memory inspection/removal or explicit migration semantics, with real setup proof before closing.
