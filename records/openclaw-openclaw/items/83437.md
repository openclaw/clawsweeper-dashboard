## Summary

Keep this open: current main includes the merged manual `memory_recall` hardening, but direct memory-lancedb auto-recall still searches the shared LanceDB path and returns recalled memory as `prependContext`, which the agent runners prepend to the model prompt. The remaining issue is the broader agent/session scoping and data-vs-instruction boundary, not the manual tool-output formatting fixed by the related merged PR.

## What This Changes

The issue reports that memory-lancedb recalled memories can be injected into agent context as instruction-like text, causing historical or other-agent memory to influence task scope.

## Best Possible Solution

Define a maintainer-approved memory boundary: make direct memory-lancedb auto-recall agent/session-scoped and data-only, or route automatic recall through the scoped Active Memory path with explicit compatibility coverage.

## Reproduction Assessment

Source-reproducible: current main still shows automatic memory recall returning prompt context and agent runners prepending hook context to the model prompt. A live model reproduction of command execution has not been run in this read-only review.

## Solution Assessment

No. The merged manual-recall hardening is useful but too narrow for this issue; the best fix needs an agent/session scoping and prompt-boundary decision for automatic recall.
