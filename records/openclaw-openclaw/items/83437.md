## Summary

This should stay open: current main still auto-recalls LanceDB memories into the prompt as text and the direct memory-lancedb store/search path remains unscoped. The cross-agent portion overlaps with the open agent-scoping PR, but the instruction/data boundary still needs maintainer security and product review.

## What This Changes

The issue reports that memory-lancedb relevant memory can be treated as current task instruction text and can leak across agent scopes.

## Best Possible Solution

Define a maintainer-approved memory boundary that makes recalled memory agent/session scoped and data-only, then coordinate that with the existing agent-scoping PR and any broader injected-context registry work.

## Reproduction Assessment

No high-confidence live model reproduction was run. Source inspection does show the mechanism: direct memory-lancedb auto-recall prepends recalled memory into the prompt, and the direct storage/search shape has no agent ownership field.

## Solution Assessment

Unclear. Warning text and escaping are useful mitigations, but the maintainable fix needs a product/security decision on scoped recall, default compatibility, and whether prompt-injected memory should move to a stronger data-only or registry-backed contract.
