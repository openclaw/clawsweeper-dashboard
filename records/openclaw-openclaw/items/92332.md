## Summary

Current main already provides a generic, documented `registerWebSearchProvider` plugin seam, so SCOUTS-AI does not require a missing core API. OpenClaw’s vision explicitly keeps optional service integrations externally maintained and discoverable through ClawHub; bundling a default third-party endpoint would also make core own its privacy, reliability, and service-policy lifecycle.

## What This Changes

Requests bundling SCOUTS-AI as a no-key public JSON fallback in the web search provider chain.

## Best Possible Solution

Implement SCOUTS-AI as an externally maintained code plugin using `api.registerWebSearchProvider(...)`, publish it through ClawHub, and clearly disclose endpoint ownership, query privacy, availability expectations, and configuration there.
