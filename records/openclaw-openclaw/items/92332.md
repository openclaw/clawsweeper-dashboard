## Summary

Current main already provides the plugin contract needed to implement SCOUTS-AI without a core change, while OpenClaw’s vision explicitly routes optional third-party capabilities to independently maintained plugins and ClawHub. Bundling a public no-key service would additionally change default privacy and external-service dependency for fresh installs, so the core proposal should close while the external plugin idea remains useful.

## What This Changes

Add SCOUTS-AI as a bundled, no-key fallback provider for `web_search`.

## Best Possible Solution

Implement and maintain SCOUTS-AI as an external web-search provider plugin, publish it through ClawHub, and document its third-party query/privacy behavior in that plugin.

## Reproduction Assessment

Not applicable; this issue requests a new provider and default fallback behavior rather than reporting a reproducible defect.

## Solution Assessment

No; bundling is not the best solution because the existing plugin contract supports this provider externally without making core depend on or send queries to a new public service by default.
