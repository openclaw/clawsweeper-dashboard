## Summary

The requested integration is technically viable, including against the live endpoint, but current main already exposes the generic plugin contract needed to implement it externally. Bundling it would add optional third-party service ownership and automatically transmit search queries to a new public endpoint, contrary to the repository’s lean-core and explicit-risk direction.

## What This Changes

Add SCOUTS-AI as a bundled credential-free JSON web-search provider and automatic last-resort fallback.

## Best Possible Solution

Publish and maintain SCOUTS-AI as an external code plugin through ClawHub, using the existing web-search provider contract with explicit user enablement and clear privacy documentation.

## Reproduction Assessment

Not applicable; the endpoint can be smoke-tested, but this issue proposes a new provider and automatic routing policy rather than reporting broken existing behavior.

## Solution Assessment

No for core bundling; the existing external plugin seam already supports the capability, while automatic third-party query routing requires trust, privacy, ownership, and default-policy decisions that an opt-in ClawHub plugin avoids.
