## Summary

Current main already exposes the complete public plugin seam needed for a credential-free web-search provider, while OpenClaw’s vision routes optional third-party integrations to externally maintained plugins and ClawHub. The live endpoint works, but its published terms describe a best-effort service without uptime or support commitments, reinforcing that it should be explicitly installed rather than bundled into automatic fallback routing.

## What This Changes

Bundle SCOUTS-AI as a last-resort no-key JSON provider in the default web-search auto-detection chain.

## Best Possible Solution

Maintain SCOUTS-AI in its own repository as an opt-in web-search code plugin, publish it through ClawHub, and document query forwarding, service limits, timeout behavior, and endpoint configuration; file a narrow core issue only if implementation exposes a concrete missing generic SDK capability.
