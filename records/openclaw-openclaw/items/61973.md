- **[P1] Reuse the shared update channel contract:** `src/cli/update-cli/review.ts:216-221`
  - body: This reimplements channel selection from only config/defaults. Current main also treats installed beta versions plus git tags and branches as channel sources, so beta or git users can review the wrong target and release notes.
  - confidence: 0.88
- **[P2] Limit downgrade review to package updates:** `src/cli/update-cli/review.ts:260-261`
  - body: `isDowngrade` is computed before checking install kind. Git checkouts can be ahead of npm tags, while current `openclaw update` only applies downgrade handling in the package path, so this can report the wrong action for git users.
  - confidence: 0.88

  - body: Collapsing `behind: 0` to `null` erases the fetched-current git state. If the npm lookup fails after git proved the checkout is current, the command reports unknown risk instead of preserving that git signal.
  - confidence: 0.8
