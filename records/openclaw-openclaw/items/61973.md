## Summary

Keep open. Current main still lacks the requested command, so the PR remains useful, but the patch is not ready because its downgrade assessment diverges from the shipped update path and the required user-facing changelog entry is missing.

## What This Changes

The branch adds a read-only `openclaw update review` subcommand with JSON output, npm/GitHub release-note risk heuristics, focused tests, CLI registration, and update documentation.

## Best Possible Solution

Keep this PR as the implementation candidate, repair the risk review logic so it mirrors `openclaw update` install-mode and downgrade semantics, preserve explicit git freshness states, add the changelog entry, then rebase and run the normal changed-surface gates.

## Reproduction Assessment

Yes. A unit mock with `installKind: git`, local `VERSION` greater than the npm target, and no package-mode switch exercises the PR's unconditional downgrade branch; current main only computes downgrade risk in the package path.

## Solution Assessment

No, not as currently patched. The command direction is plausible, but the implementation should reuse or closely mirror the existing update target/downgrade helper behavior instead of independently classifying git installs from npm semver alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope downgrade reporting to package updates:** `src/cli/update-cli/review.ts:261`
  - body: `isDowngrade` is computed from the npm target for every install kind, so git checkouts whose local `VERSION` is ahead of the registry tag are reported as downgrades even though `openclaw update` only applies downgrade handling in the package update path. Mirror the package-path predicate instead of evaluating this before git-specific guidance.
  - confidence: 0.88
- **[P3] Preserve zero-behind git status:** `src/cli/update-cli/review.ts:249-254`
  - body: `gitBehind` collapses a fetched `behind: 0` result to `null`, and `checkUnavailable` later treats that as no git signal. If the npm lookup fails but git fetch proved the checkout is current, the command reports an unknown review-required state instead of up to date.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/cli/update-cli.ts:160-161`
  - body: This adds a user-facing CLI command and documentation, but the branch does not update `CHANGELOG.md`. Add a single-line entry under the active `### Changes` section with appropriate contributor credit before merge.
  - confidence: 0.96
