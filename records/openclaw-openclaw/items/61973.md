## Summary

Keep this PR open, but it is not merge-ready: current main still lacks the proposed command, while the branch has correctness issues in update-channel/downgrade/git-freshness handling and still lacks real CLI behavior proof.

## What This Changes

The PR adds a read-only `openclaw update review` CLI subcommand, JSON result type, focused tests, registration under `openclaw update`, and update CLI docs.

## Best Possible Solution

Add the command by sharing or closely mirroring current update channel, target, downgrade, and git freshness helpers, then align the risk model with the release-readiness direction and require redacted real CLI output before merge.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the PR head and current-main update helpers shows the beta-channel, git-downgrade, and zero-behind cases. The new feature itself still lacks real CLI proof from `openclaw update review` and `openclaw update review --json`.

## Solution Assessment

No, not as currently patched. The useful direction is a maintainable shared update-review surface, but this branch reimplements channel and availability decisions that current main already centralizes elsewhere.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use shared channel resolution for beta installs:** `src/cli/update-cli/review.ts:219-221`
  - body: A package install running a beta `VERSION` with no config still falls back to the stable default because `effectiveChannel` only uses config/defaults. Current main resolves that case to beta, so beta users can see the wrong target and release notes.
  - confidence: 0.82
- **[P2] Gate downgrade reporting to package installs:** `src/cli/update-cli/review.ts:260-261`
  - body: `isDowngrade` is computed from the npm target for every install kind. Git checkouts whose local `VERSION` is ahead of registry tags can be reported as downgrades even though `openclaw update` only applies downgrade handling in the package update path.
  - confidence: 0.88
- **[P3] Preserve zero-behind git freshness:** `src/cli/update-cli/review.ts:249-254`
  - body: The PR stores `gitBehind` only when `behind > 0`, losing the distinction between a fetched current checkout and unknown git status. If the npm lookup fails after git proved `behind: 0`, the command reports an unknown review state instead of preserving the current git result.
  - confidence: 0.78
