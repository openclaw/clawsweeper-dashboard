## Summary

Keep open: the PR targets a real config-validation bug and supplies real behavior proof, but the patch is not merge-ready because active slash-form model aliases can bypass the suppressed-model check.

## What This Changes

The PR changes config model-reference validation to ignore inactive `agents.defaults.models` catalog entries while still validating active model selections and aliases.

## Best Possible Solution

Keep the inactive-catalog tolerance, but resolve active aliases with the existing model-selection semantics before suppression checks and add slash-form alias regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current `main` collects `agents.defaults.models` keys and validates them against manifest suppressions, so a stale inactive `openai-codex/gpt-5.3-codex-spark` catalog entry can fail config validation without executing the CLI.

## Solution Assessment

No. The proposed direction is narrow, but the implementation parses provider/model refs before alias lookup, which misses OpenClaw's existing slash-form alias behavior; reuse or match the shared alias resolver semantics before suppression checks.

## Review Findings

Overall correctness: patch is incorrect
