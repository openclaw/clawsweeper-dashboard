## Summary

Keep open: current main has the source-reproducible config validation bug, and the PR is directionally right, but latest head still has a blocking slash-form alias gap plus the required changelog entry is missing.

## What This Changes

The PR skips inactive `agents.defaults.models` catalog entries during suppressed-model validation, resolves active default-model aliases/profiles for suppression checks, and adds focused config validation tests.

## Best Possible Solution

Resolve active model refs through the existing shared model-selection alias semantics before suppression checks, keep inactive catalog entries ignored, cover slash-form/profiled aliases, and add a single-line changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main collects inactive `agents.defaults.models` keys and validates them against unconditional manifest suppressions, so a stale `openai-codex/gpt-5.3-codex-spark` catalog entry can fail config validation without being selected.

## Solution Assessment

No. The inactive-catalog tolerance is the right direction, but the latest diff still parses `provider/model` before alias lookup, which does not match OpenClaw's established slash-form alias semantics.
