## Summary

Keep open: current main still routes approved exec completions back through an agent followup prompt, and the adjacent UI filters and node-event replay dedupe do not prove the Control UI approval-completion loop is consumed exactly once.

## What This Changes

The issue reports that approving an exec command in Control UI can recursively reprocess the exec completion event until browser refresh.

## Best Possible Solution

Add server-side one-shot consumption or durable dedupe for approved exec completion followups, then cover the Control UI approval path with a regression proving one completion yields one followup and no repeated identical agent turns.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter provided concrete steps and current source shows the approval completion re-entry path, but this sweep did not execute an actual Control UI approval flow against current main.

## Solution Assessment

Yes, the narrow maintainable direction is server-side consumption or dedupe before the completion can become another agent input. Display filtering and duplicate bubble collapse should remain defense in depth, not the primary fix.
