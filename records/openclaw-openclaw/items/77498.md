## Summary

Keep open: current main has filtering for visible heartbeat/internal text, but I did not find a current server-side one-shot guard that would prove approved exec completion followups cannot be reprocessed recursively in the Control UI path.

## What This Changes

The issue reports that approving a Control UI exec command repeatedly reprocesses the `Exec finished` completion event until the browser is refreshed.

## Best Possible Solution

Add server-side one-shot consumption or durable dedupe for approved exec completion followups, then cover the Control UI approval path with a regression that proves one completion yields one followup and no repeated identical agent turns.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not yet from this sweep: the reporter provided concrete steps and source inspection shows the approval-completion re-entry path, but I did not run a live Control UI approval flow against current main.

## Solution Assessment

Is this the best way to solve the issue? A display-only filter is not enough; the maintainable fix should consume or dedupe approved exec completions before they can become another agent input, while keeping UI filtering as defense in depth.
