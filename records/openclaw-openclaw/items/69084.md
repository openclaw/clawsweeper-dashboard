## Summary

Keep this PR open for revision. The reported WebChat heartbeat-noise symptom is valid, but the proposed one-line gate collapses the separate `showOk` and `showAlerts` contracts, regressing default ACK suppression and the OK-only configuration path.

## What This Changes

The PR changes gateway WebChat heartbeat suppression from `showOk` to `showAlerts` and updates adjacent gateway agent-event tests to expect that behavior.

## Best Possible Solution

Keep ACK and alert decisions separate in the gateway chat projection: classify heartbeat output after token normalization, apply `showOk` to OK-only ACK/noise, apply `showAlerts` to non-OK alert content, and keep the broader system-event audience work in the linked Track B issues rather than solving it with this boolean swap.

## Reproduction Assessment

Yes. Applying the PR's gate to the existing gateway test setup with default resolved visibility (`showOk=false`, `showAlerts=true`) makes ACK-like heartbeat text bypass suppression; the original `showAlerts=false` symptom is also reproducible conceptually with a non-OK heartbeat response because current main's WebChat filter only checks `showOk`.

## Solution Assessment

No. Replacing `showOk` with `showAlerts` is not the best fix because it loses the documented OK-ack contract; the safer path is a small classifier that preserves both flags and covers default, alerts-off, and OK-only configurations in tests.
