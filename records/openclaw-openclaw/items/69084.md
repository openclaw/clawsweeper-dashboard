## Summary

Keep open for human PR handling: the branch is source-proven incorrect because it collapses the documented `showOk` and `showAlerts` contracts, and the author has already agreed the patch should not merge as-is.

## What This Changes

The PR changes Gateway WebChat heartbeat chat suppression from `showOk` to `showAlerts` and adjusts nearby heartbeat visibility tests around that behavior.

## Best Possible Solution

Replace this with output-aware WebChat heartbeat projection: apply `showOk` only to OK-only acknowledgments, apply `showAlerts` to non-OK alert content, and add focused tests for default and mixed flag configurations.

## Reproduction Assessment

Yes, source-reproducible. With the PR diff, default `showOk=false` and `showAlerts=true` stops suppressing ACK-like output, while `showOk=true` and `showAlerts=false` suppresses OK-only acknowledgments users explicitly enabled.

## Solution Assessment

No. A single `showAlerts` gate is not the narrow maintainable fix because it collapses two documented visibility controls; the fix needs to classify heartbeat output and apply the matching flag.
