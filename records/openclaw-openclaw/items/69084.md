## Summary

Keep open for changes: the branch is source-reproducibly incorrect because it collapses the separate `showOk` and `showAlerts` contracts, and the author later agreed this patch should not merge as-is. It also still lacks inspectable real behavior proof.

## What This Changes

The PR changes Gateway webchat heartbeat chat suppression from `showOk` to `showAlerts` and updates nearby gateway heartbeat tests.

## Best Possible Solution

Implement output-aware webchat heartbeat projection: apply `showOk` to OK-only ACKs and `showAlerts` to non-OK alert content, with focused tests for default and mixed flag configurations.

## Reproduction Assessment

Yes, source-reproducible. With the PR diff, the default `showOk=false` and `showAlerts=true` path returns false from the suppression gate and can expose ACK-like output; the `showOk=true` and `showAlerts=false` path suppresses OK-only acknowledgments that were explicitly enabled.

## Solution Assessment

No. A single `showAlerts` gate is not the best fix because it collapses two documented visibility controls; the narrow maintainable fix needs to classify the heartbeat output and apply the matching flag.
