## Summary

Keep this PR open for changes: the current diff would collapse the separate `showOk` and `showAlerts` contracts, regressing default WebChat heartbeat suppression and configured OK-only visibility.

## What This Changes

This PR changes Gateway WebChat heartbeat chat suppression from `showOk` to `showAlerts` and updates adjacent heartbeat visibility tests to match that behavior.

## Best Possible Solution

Implement output-aware WebChat heartbeat projection that applies `showOk` to OK-only acknowledgments and `showAlerts` to alert content, with focused default, alerts-off, OKs-on, and mixed-flag tests.

## Reproduction Assessment

Yes from source inspection. With current defaults `showOk=false` and `showAlerts=true`, changing the gate to `!showAlerts` makes WebChat stop suppressing heartbeat OK output that current main and tests suppress.

## Solution Assessment

No. A single `showAlerts` gate is not the narrow maintainable fix because current code, docs, tests, and prior discussion keep OK acknowledgments and alert content as separate visibility decisions.
