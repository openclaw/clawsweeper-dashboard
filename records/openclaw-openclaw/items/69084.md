## Summary

Keep this PR open for human handling: the proposed one-flag swap is known incorrect, and the author later agreed it should not merge as-is.

## What This Changes

The PR changes Gateway webchat heartbeat chat suppression from `showOk` to `showAlerts` and rewrites nearby heartbeat visibility tests around that behavior.

## Best Possible Solution

Implement output-aware webchat heartbeat projection: apply `showOk` to OK-only acknowledgments and `showAlerts` to non-OK alert content, with focused tests for default and mixed flag configurations.

## Reproduction Assessment

Yes, source-reproducible. With the PR diff, the default `showOk=false` and `showAlerts=true` path stops suppressing ACK-like output, while `showOk=true` and `showAlerts=false` suppresses OK-only acknowledgments that the config explicitly enabled.

## Solution Assessment

No. A single `showAlerts` gate collapses two documented visibility controls; the maintainable fix needs to classify heartbeat output and apply the matching flag.

## Review Findings
