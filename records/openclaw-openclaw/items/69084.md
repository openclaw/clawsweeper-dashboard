## Summary

Keep this PR open for maintainer/contributor follow-up: the branch targets a real WebChat heartbeat visibility gap, but the current patch is not mergeable because it collapses the documented `showOk` and `showAlerts` contracts and the author has agreed it should not land as-is.

## What This Changes

This PR changes Gateway WebChat heartbeat chat suppression from `showOk` to `showAlerts` and updates adjacent heartbeat visibility tests.

## Best Possible Solution

Implement output-aware WebChat heartbeat projection that applies `showOk` to OK-only acknowledgments and `showAlerts` to alert content, with focused default, alerts-off, and mixed-flag tests.

## Reproduction Assessment

Yes. The PR diff makes the default `showOk=false` / `showAlerts=true` path stop suppressing `HEARTBEAT_OK`, and makes `showOk=true` / `showAlerts=false` suppress OK acknowledgments that users explicitly enabled; I verified this from the current source and diff without running tests in the read-only checkout.

## Solution Assessment

No. A single `showAlerts` gate is not the narrow maintainable fix because the current code, docs, and tests keep OK acknowledgments and alert content as separate visibility decisions.
