## Best Possible Solution

## Reproduction Assessment

Yes, at source level: with defaults `showOk=false` and `showAlerts=true`, the PR's `!showAlerts` gate no longer suppresses `HEARTBEAT_OK` output, while current main's `!showOk` gate does. I did not run tests because this review is read-only.

## Solution Assessment

No. A single `showAlerts` gate is not the narrow maintainable solution because current source, docs, and tests keep OK acknowledgments and alert content as separate visibility decisions.
