## Summary

Keep this PR open but do not merge it as-is. The patch addresses a real WebChat heartbeat-noise symptom, but it collapses the separate `showOk` and `showAlerts` contracts and would regress default ACK suppression plus OK-only visibility configurations.

## What This Changes

The PR changes Gateway WebChat heartbeat chat suppression to use `visibility.showAlerts` instead of `visibility.showOk` and updates adjacent gateway agent-event tests to expect that behavior.

## Best Possible Solution

Keep the two visibility contracts separate in the Gateway WebChat projection: classify heartbeat output after normalization, apply `showOk` to OK-only acknowledgments, apply `showAlerts` to non-OK alert content, and add focused tests for default behavior, alerts-off behavior, and mixed OK-only configurations.

## Reproduction Assessment

Yes. The current gateway test harness already models the default `showOk=false, showAlerts=true` case and proves ACK suppression; applying the PR's gate would make that case broadcast instead. The reported `showAlerts=false` WebChat leak is plausible from the same code path but still needs an explicit regression test.

## Solution Assessment

No. The one-line `showOk` to `showAlerts` swap is not the best fix because it trades one visibility bug for another; the safer solution is a small classifier that preserves both documented flags.
