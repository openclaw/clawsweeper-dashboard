## Summary

Keep open: the reported WebChat heartbeat visibility bug is plausible, but this PR collapses `showOk` and `showAlerts` into one gate and would regress default `HEARTBEAT_OK` suppression; the author has also agreed the patch should not merge as-is.

## What This Changes

The PR changes Gateway WebChat heartbeat chat suppression from `showOk` to `showAlerts` and updates nearby heartbeat visibility tests around that behavior.

## Best Possible Solution

Replace this with output-aware WebChat heartbeat projection: apply `showOk` only to OK-only acknowledgments, apply `showAlerts` to non-OK alert content, and add focused default plus mixed-flag tests.

## Reproduction Assessment

Yes, source-reproducible. With the PR diff, the default `showOk=false` and `showAlerts=true` path stops suppressing ACK-like output, while `showOk=true` and `showAlerts=false` suppresses OK-only acknowledgments that users explicitly enabled.

## Solution Assessment

No. A single `showAlerts` gate is not the narrow maintainable fix because it collapses two documented visibility controls that current code, tests, and docs keep separate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

The WebChat projection needs to classify heartbeat output and apply `showOk` to OK-only replies and `showAlerts` to alert content.
