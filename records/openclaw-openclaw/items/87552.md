## Summary

This PR should stay open for maintainer review: the patch looks mechanically sound and the supplied terminal proof is sufficient, but it changes upgrade-visible ambiguous delivery recovery behavior and intentionally trades duplicate-send avoidance for pending unresolved delivery state.

## What This Changes

The PR changes outbound recovery so ambiguous `send_attempt_started` and `unknown_after_send` rows without available reconciliation remain pending without retry-budget burn, adds focused recovery tests, and updates Discord troubleshooting docs.

## Best Possible Solution

Merge only if maintainers accept pending no-budget deferral as the interim recovery policy while the durable delivery-ledger and UI/API visibility work remains tracked separately.

## Solution Assessment
