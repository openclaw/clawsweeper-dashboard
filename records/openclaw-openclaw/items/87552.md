## Summary

Keep open for maintainer acceptance: the patch is a focused and source-backed mitigation for ambiguous outbound delivery rows, but it intentionally changes recovery policy for existing pending rows and still leaves the broader delivery ledger/reconciliation design to the canonical issue.

## What This Changes

The PR keeps ambiguous outbound delivery entries pending without retry-budget burn when no unknown-send reconciliation adapter is available, counts completed null/unresolved reconciliation attempts against retry budget, updates focused recovery tests, and adjusts Discord troubleshooting docs.

## Best Possible Solution

Land this only if maintainers accept pending/no-budget deferral as the interim recovery policy while the durable delivery-ledger and UI/API visibility work remains tracked separately.
