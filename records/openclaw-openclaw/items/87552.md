## What This Changes

The PR keeps ambiguous outbound delivery rows pending without retry-budget consumption when no reconciliation-capable adapter is available, counts completed/null reconciliation attempts against retry budget, and updates focused recovery tests plus Discord troubleshooting docs.

## Best Possible Solution

Merge only if maintainers accept pending no-reconciler ambiguous rows as the interim recovery policy while keeping the full delivery-ledger design tracked in https://github.com/openclaw/openclaw/issues/87544.
