## Summary

Keep open for maintainer review: the patch appears technically coherent and has focused queue/SQLite proof, but it changes the recovery policy for ambiguous outbound sends by leaving entries pending instead of failing or burning retry budget when reconciliation is unavailable.

## What This Changes

The PR changes outbound delivery recovery so `send_attempt_started` and `unknown_after_send` entries are deferred as pending when unknown-send reconciliation is unavailable, updates retry handling for adapter-null reconciliation, adds recovery tests, and documents the pending Discord recovery state.

## Best Possible Solution

Merge only after maintainers explicitly accept pending deferral as the interim ambiguous-send policy or request live transport proof before adopting that policy.
