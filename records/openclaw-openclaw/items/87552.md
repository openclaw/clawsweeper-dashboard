## Summary

This PR is still useful and not obsolete, but it should stay open for maintainer acceptance of the ambiguous-delivery recovery policy rather than automatic cleanup or repair. The patch avoids blind replay and preserves no-reconciler rows for future reconciliation, while the broader delivery-ledger and operator-visibility problem remains tracked separately.

## What This Changes

The PR changes outbound delivery recovery so ambiguous post-send queue entries stay pending without consuming retry budget when no adapter reconciliation is available, while counting completed unresolved or null adapter reconciliation attempts.

## Best Possible Solution

Land this only if maintainers accept it as an interim pending-row recovery policy, while keeping the durable delivery-ledger and UI/API visibility work under https://github.com/openclaw/openclaw/issues/87544.
