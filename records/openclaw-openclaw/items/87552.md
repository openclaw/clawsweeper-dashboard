## Summary

Keep open for maintainer review: the latest head addresses the earlier retry-budget and adapter-null review concerns, but it intentionally changes ambiguous outbound delivery recovery policy and should be accepted by a maintainer before merge.

## What This Changes

The PR keeps `send_attempt_started` and `unknown_after_send` outbound queue entries pending without retry-budget consumption when no adapter reconciliation is available, while counting completed null/unresolved reconciliation attempts against retry budget and updating tests/docs.

## Best Possible Solution

Merge the narrow deferral only after maintainers accept pending ambiguous rows as the interim recovery state, while keeping the full durable delivery-ledger work in https://github.com/openclaw/openclaw/issues/87544.

## Reproduction Assessment

Yes, at source level: current main moves no-adapter ambiguous rows to failed, while PR head keeps them pending without replay or retry-budget consumption. I did not run a live transport restart reproduction in this read-only review.

## Solution Assessment

Yes for the narrow mitigation, subject to maintainer policy acceptance. It preserves the no-blind-replay invariant and now separates no-adapter deferral from retryable reconciliation failures; the full durable ledger remains separate work.
