## Label Justifications

- `P2`: This is a bounded reliability fix for outbound message recovery with limited blast radius, not an emergency runtime outage.
- `merge-risk: 🚨 compatibility`: The PR changes upgrade-visible recovery behavior for existing pending ambiguous queue rows by leaving them pending instead of failing or consuming retries.
- `merge-risk: 🚨 message-delivery`: The change deliberately trades blind replay and duplicates for unresolved pending delivery state, affecting how missing or ambiguous channel sends recover.

## Summary

Keep open for maintainer review: the patch is technically coherent and has sufficient queue-level proof, but it deliberately changes upgrade-visible outbound recovery behavior and needs explicit acceptance of that delivery tradeoff.

## What This Changes

The branch keeps `send_attempt_started` and `unknown_after_send` outbound rows pending without retry-count burn when unknown-send reconciliation is unavailable, adds recovery tests, and updates Discord troubleshooting guidance.

## Best Possible Solution

Land this only if maintainers accept pending/no-budget deferral as the interim recovery policy, while keeping the durable delivery ledger and reconciliation design canonical under https://github.com/openclaw/openclaw/issues/87544.

## Reproduction Assessment

Yes, at source level. Current main moves unreconciled ambiguous outbound entries out of pending or burns retry budget, while the PR proof demonstrates the after-fix queue/SQLite behavior.

## Solution Assessment

Yes as a narrow mitigation, but not as the full delivery-consistency solution. The durable end state still needs a transcript-to-client delivery ledger, adapter reconciliation semantics, and UI/API visibility.
