## Summary

Keep open for maintainer review. The patch appears technically sound and the real behavior proof is sufficient for the local SQLite recovery path, but it deliberately changes ambiguous outbound delivery recovery from fail-closed to pending/no-budget deferral when no reconciler is available.

## What This Changes

This PR keeps ambiguous outbound delivery rows pending without consuming retry budget when no reconciliation-capable adapter is available, adds focused recovery/reconnect coverage, and updates Discord troubleshooting docs.

## Best Possible Solution

Merge only if maintainers accept pending no-reconciler ambiguous rows as the interim recovery policy while the broader delivery-ledger design remains tracked in https://github.com/openclaw/openclaw/issues/87544.

## Solution Assessment

Yes for the narrow mitigation, subject to maintainer policy acceptance. It preserves the no-blind-replay invariant and separates no-adapter deferral from retryable reconciliation failures; the full durable ledger remains separate work in https://github.com/openclaw/openclaw/issues/87544.
