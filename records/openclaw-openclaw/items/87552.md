## Summary

Keep open for maintainer review. The patch looks like a correct narrow mitigation, but it deliberately changes ambiguous outbound recovery from terminal/retry-budget behavior to pending no-budget deferral, so the remaining decision is whether maintainers accept that interim delivery policy.

## What This Changes

The PR changes outbound queue recovery so ambiguous `send_attempt_started` and `unknown_after_send` rows without available reconciliation stay pending without retry-budget burn, counts adapter-null reconciliation as retryable unresolved, and updates focused tests plus Discord troubleshooting docs.

## Best Possible Solution

Merge only if maintainers accept pending no-budget deferral as the interim recovery policy while the durable delivery-ledger and UI/API visibility work remains tracked separately.

## Reproduction Assessment

## Solution Assessment

This is an acceptable narrow mitigation, not the full final solution. The final shape still needs a durable delivery receipt and adapter reconciliation design so ambiguous pending rows are observable and resolvable.
