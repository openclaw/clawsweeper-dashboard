## Summary

## What This Changes

The PR keeps ambiguous outbound delivery queue entries pending without retry-budget consumption when no reconciliation-capable adapter is available, while preserving retry-budget consumption for completed unresolved reconciliation attempts and updating focused tests/docs.

## Best Possible Solution

## Solution Assessment

Yes for the narrow mitigation, subject to maintainer policy acceptance. It preserves the no-blind-replay invariant and separates no-adapter deferral from retryable reconciliation failures; the full durable ledger remains separate work.
