## Summary

Keep open for maintainer review: this PR is a meaningful, recent outbound delivery recovery change that is not already implemented on current main, but it deliberately changes the recovery policy for ambiguous sends and needs maintainer acceptance before merge.

## What This Changes

The PR keeps ambiguous outbound delivery rows pending without retry-budget burn when no unknown-send reconciler is available, counts completed null/unresolved reconciliation attempts against retries, updates focused recovery tests, and adds Discord troubleshooting text.

## Best Possible Solution

Land this only if maintainers accept it as an interim pending-row recovery policy, while keeping the durable delivery-ledger and UI/API visibility work under https://github.com/openclaw/openclaw/issues/87544.

## Reproduction Assessment

Yes, at source level: current main moves or budgets ambiguous no-reconciler rows toward failure, while the PR head tests and terminal proof show the after-fix queue/SQLite row remains pending with retry count unchanged. I did not run a live transport restart in this read-only review.

## Solution Assessment

The patch is an acceptable narrow mitigation, but not the final product solution. The durable end state should still add delivery receipts, reconciliation semantics, and operator/session visibility rather than relying on pending ambiguous rows alone.
