## Summary

This PR should stay open for maintainer review: the code path is source-reproducible and I found no discrete patch defect, but it intentionally changes operator-visible outbound recovery policy and the broader delivery-ledger work remains tracked separately.

## What This Changes

The PR changes outbound recovery so ambiguous `send_attempt_started`/`unknown_after_send` entries without adapter reconciliation stay pending without retry-budget consumption, treats adapter `null` reconciliation results as retry-consuming unresolved attempts, updates coverage, and revises Discord troubleshooting docs.

## Best Possible Solution

Land this only if maintainers accept pending no-budget deferral as the interim recovery policy; otherwise revise toward an explicit failed or operator-surfaced state while keeping broader ledger design on the canonical issue.
