## Summary

Keep open for maintainer review. The current diff addresses the earlier retry-budget/null-reconciler review comments and has credible local real-queue proof, but it intentionally changes the upgrade/operator policy for ambiguous outbound rows that lack reconciliation support.

## What This Changes

The PR changes outbound delivery recovery so ambiguous post-send entries without an available reconciler stay pending without consuming retry budget, adds focused recovery coverage, and updates Discord troubleshooting docs.

## Best Possible Solution

Land this only if maintainers accept pending no-budget deferral as the interim recovery policy; otherwise revise toward an explicit failed/operator-surfaced state and keep the broader ledger design on the canonical issue.
