## Summary

Keep open for maintainer review: the patch appears to address the focused retry-budget bug, but it intentionally changes operator-visible recovery behavior for ambiguous no-reconciler delivery rows and should not be auto-closed or auto-repaired.

## What This Changes

The PR updates outbound delivery recovery so ambiguous post-send entries stay pending without retry-budget consumption when no reconciliation-capable adapter is available, with focused recovery tests and Discord troubleshooting docs.

## Best Possible Solution

Land this only if maintainers accept pending no-budget deferral as the interim recovery policy; otherwise choose an explicit failed or operator-surfaced state before merge and leave the broader ledger to https://github.com/openclaw/openclaw/issues/87544.
