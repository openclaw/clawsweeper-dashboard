## Summary

Keep open for maintainer review: the patch has sufficient proof and no concrete line-level correctness finding, but it intentionally changes outbound recovery policy for ambiguous sends and should not be auto-closed or auto-merged without maintainer acceptance of that tradeoff.

## What This Changes

The PR keeps ambiguous outbound delivery recovery entries pending without consuming retry budget when no unknown-send reconciler is available, while completed unresolved reconciliations still consume retry budget.

## Best Possible Solution

Land only if maintainers accept pending/no-budget deferral as the interim recovery policy, while keeping the broader delivery-ledger and adapter reconciliation work canonical under the linked issue.
