## Summary

## What This Changes

The PR changes outbound delivery recovery so ambiguous `send_attempt_started` and `unknown_after_send` entries without an available reconciler stay pending without consuming retry budget, treats adapter `null` reconciliation as unresolved, and updates focused recovery tests plus Discord troubleshooting docs.

## Best Possible Solution

Land only if maintainers accept pending no-budget deferral as the interim recovery policy, while keeping the broader delivery-ledger and adapter reconciliation design canonical under the linked issue.
