## Summary

Keep open. The report is current, concrete, and current main still has a source-visible Discord missing-reference path where a normal text reply with `message_reference` but no hydrated `referenced_message` can reach the agent without `ReplyToBody` or an unavailable-reference marker; the adjacent open PR only partially covers reconnect/hydration and explicitly excludes process-restart recovery and the fail-closed policy requested here.

## What This Changes

The issue asks Discord reply/reference turns after restart to hydrate or fail closed instead of answering from stale unrelated session context.

## Best Possible Solution

Fix the Discord inbound path so referenced-message turns either hydrate and inject the referenced content or carry an explicit unavailable-reference marker that prevents stale history from satisfying the direct reply question, while coordinating with the open reconnect-backfill PR.

## Reproduction Assessment

No for the full live cross-session leak; yes at source level for the missing-reference path. Current main can process a Discord text reply with `message_reference` but no hydrated `referencedMessage` without injecting `ReplyToBody` or an unavailable-reference marker.

Unclear as a complete fix. Hydrating missing references is a narrow bug fix and is partly covered by the open adjacent PR, but the fail-closed behavior and restart/session-provenance guard need maintainer coordination before a separate repair should run.
