## Summary

Keep open. Current main still has a source-visible Discord reply path where a normal text reply with `message_reference` but no hydrated `referenced_message` can reach the agent without referenced content or an explicit unavailable-reference marker, and the adjacent fix is still unmerged and partial.

## What This Changes

Report that Discord reply/reference turns can lose the referenced message after restart and answer using stale unrelated session context.

## Best Possible Solution

Fix Discord inbound handling so reply/reference turns either hydrate the referenced content or inject an explicit unavailable-reference context that prevents stale history from answering, and coordinate restart/reconnect recovery with https://github.com/openclaw/openclaw/pull/81226.

## Reproduction Assessment

No for the full live Discord cross-session leak. Yes from source for the core missing-reference path: a normal text reply with `message_reference` but no `referenced_message` does not force hydration or carry an unavailable-reference marker.

## Solution Assessment

Unclear as a complete fix. Hydrating missing references is a narrow bug fix, but the process-restart/session-provenance fail-closed behavior needs maintainer policy and should be coordinated with https://github.com/openclaw/openclaw/pull/81226.
