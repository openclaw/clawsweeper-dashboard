## Summary

Keep open: current main still has a credible Discord reply-reference gap where a message can carry `message_reference` but dispatch without `ReplyToBody`, letting a short follow-up be answered from unrelated session history. The related open issues are adjacent context/session problems, not a canonical fix for this Discord referenced-message failure mode.

## What This Changes

The issue asks OpenClaw to hydrate or fail closed on Discord reply/reference turns so missing referenced-message content cannot be replaced by stale prior task context.

## Best Possible Solution

Resolve Discord referenced messages before agent dispatch, and emit an explicit missing-reference fallback when content cannot be loaded instead of running the model on stale context.

## Reproduction Assessment

Yes, at source level: a Discord message with `message_reference` but no materialized `referenced_message` can reach dispatch without `ReplyToBody`. I did not live-reproduce the exact Ada restart transcript leak.

## Solution Assessment

Yes, the requested guardrail is the narrow maintainable direction: hydrate the reference or fail closed before model execution, while preserving existing context-visibility and media safety checks.

## Repair Work Prompt

Fix the Discord inbound reply/reference bug reported at https://github.com/openclaw/openclaw/issues/82608. Current main can receive a Discord message with `message_reference` while `referenced_message` is absent; `hydrateDiscordMessageIfNeeded` can skip hydration for a plain text reply, `resolveReplyContext` returns null, and the turn still dispatches without `ReplyToBody`. Implement a narrow Discord monitor/preflight/hydration fix: when an inbound Discord reply/reference is present, resolve and inject the referenced message before agent dispatch, or fail closed with a clear visible fallback asking for the missing referenced message. Preserve existing context visibility/allowlist behavior, SSRF/media byte bounds, reply delivery/threading behavior, and do not add a config knob. Add focused regression coverage for `message_reference` with missing/null referenced content and for the successful hydrated-reference path.
