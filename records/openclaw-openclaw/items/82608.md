## What This Changes

The issue asks OpenClaw to prevent Discord reply/reference turns from answering with stale prior task context when referenced-message context is missing or stripped after restart/session mismatch.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82801 if maintainer review agrees it fixes the active-turn context loss, then keep any remaining Discord reference hydration or fail-closed behavior tracked through https://github.com/openclaw/openclaw/pull/81226 or a narrower follow-up.

## Reproduction Assessment

Yes for the source-visible current-main failure path: active inbound reply metadata can be stripped at the embedded runner LLM boundary, and Discord missing-reference context can also be dropped before `ReplyToBody` exists. I did not reproduce the full live cross-session Discord leak in this read-only review.

## Solution Assessment

Mostly yes: the open maintainer PR is the narrowest fix for the LLM-boundary loss while preserving historical metadata stripping. The broader missing-reference hydration and fail-closed policy should be coordinated with the adjacent Discord reconnect/hydration PR rather than duplicated by a separate repair lane now.
