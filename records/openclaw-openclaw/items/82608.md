## Summary

Keep open. Current main still has a source-visible Discord reply path where a message with `message_reference` but no hydrated `referencedMessage` can enter the agent without `ReplyToBody` or an explicit missing-reference marker, while channel/session history remains available.

## What This Changes

The issue asks Discord reply/reference turns to hydrate or mark missing referenced messages and avoid answering from unrelated session context after restart.

## Best Possible Solution

Fix Discord inbound turns so reply references either hydrate and inject the referenced content or carry an explicit unavailable-reference marker that prevents stale session context from satisfying the direct reply question.

## Reproduction Assessment

No for the full live cross-session leak; this read-only review did not reproduce the gateway restart scenario. Yes for the core source path: current main can process a normal Discord text reply with `message_reference` but no hydrated `referencedMessage` without adding `ReplyToBody` or an unavailable-reference marker.

## Solution Assessment

Unclear as a complete fix. Hydrating missing references is necessary and already partly covered by an open adjacent PR, but the fail-closed and session-provenance behavior needs maintainer direction before a separate automated repair should run.
