## Summary

Keep open. Current main still has a source-visible Discord reply/reference gap: a normal text reply can skip hydration when `referenced_message` is absent, no unavailable-reference marker is injected, and normal session/history context remains available to the agent; the adjacent open PR only covers part of this surface.

## What This Changes

The issue reports a Discord reply/reference turn after restart answering with unrelated Telegram task context when the referenced Discord message was unavailable.

## Best Possible Solution

Fix Discord inbound replies so referenced-message turns either hydrate and inject the current reference or carry an explicit unavailable-reference marker that prevents stale session context from answering the direct reply question.

## Reproduction Assessment

No for the full live cross-session leak after restart. Yes at source level for the missing-reference path: current main can process a normal Discord reply with `message_reference` but no `referenced_message` without hydrating or marking the quote unavailable.

## Solution Assessment

Unclear as a complete fix. Hydrating missing references is narrow and already covered by an open adjacent PR, but the fail-closed/session-provenance behavior for unavailable references needs maintainer direction.
