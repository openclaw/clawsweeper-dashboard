## Summary

Keep open. Current main still has a source-visible Discord reply/reference gap: a normal text reply with `message_reference` but no hydrated `referencedMessage` can reach the agent without `ReplyToBody` or an explicit unavailable-reference marker, while session and channel history remain available.

## What This Changes

The issue requests safer Discord reply/reference handling so missing referenced-message context cannot be answered from stale unrelated session history.

## Best Possible Solution

Fix Discord inbound handling so referenced-message turns either hydrate and inject the referenced content or carry an explicit unavailable-reference marker that prevents stale session context from satisfying a direct reply question.

## Reproduction Assessment

No for the full live cross-session leak; this review did not run a Discord Gateway restart reproduction. Yes from source for the core missing-reference path: current main can process a normal Discord reply with `message_reference` but no `referencedMessage` without hydrating or marking the missing quote.

## Solution Assessment

Unclear as a complete fix path. Hydrating missing Discord reply references is narrow and partly covered by https://github.com/openclaw/openclaw/pull/81226, but the process-restart/session-provenance and fail-closed policy need maintainer direction before a separate automated repair should run.
