## Summary

Keep open. Current main still omits `VESSEL.md` from the bootstrap filename contract, default loader, extra-file allowlist, Gateway file API, and docs; the only exact implementation PR was closed unmerged, so this still needs maintainer product/API direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to treat `VESSEL.md` as a first-class auto-injected workspace bootstrap file for per-host agent identity.

## Best Possible Solution

Make an explicit maintainer decision on `VESSEL.md`; if accepted, implement it consistently across bootstrap constants/types, loader, extra-file validation, Gateway file APIs, sandbox seeding, docs, and focused tests without creating a default template unless that is separately approved.

## Reproduction Assessment

Not applicable as a strict bug, but the capability gap is source-reproducible: current main has no `VESSEL.md` constant, loader entry, allowlist entry, Gateway exposure, or docs entry. The concrete Slack/Ada identity failure was not independently reproduced.

## Solution Assessment

Unclear pending maintainer product approval. A direct `VESSEL.md` addition is technically narrow if accepted as a core convention, but the broader configured extra-file contract in https://github.com/openclaw/openclaw/issues/39120 may be the more maintainable policy direction.
