## Summary

Keep open. Current `main` does not implement `VESSEL.md`, exact-title/search evidence did not surface an existing canonical duplicate, and accepting another well-known bootstrap file is a maintainer product/convention decision rather than a conservative cleanup close.

## What This Changes

The issue asks OpenClaw to include `VESSEL.md` in the auto-injected workspace bootstrap files alongside files such as `IDENTITY.md` and `MEMORY.md`.

## Best Possible Solution

Make an explicit maintainer decision on whether per-host identity deserves a first-class `VESSEL.md` bootstrap convention, then implement it consistently across loader, allowlists, gateway file APIs, docs, and tests if accepted.

## Reproduction Assessment

Yes for the capability gap: source inspection on current `main` shows `VESSEL.md` is absent from the default injected file list and recognized-name allowlists. The concrete Slack/Ada identity failure itself was not independently reproduced and is not a broken documented OpenClaw contract today.

## Solution Assessment

Unclear pending maintainer product approval. The suggested implementation is narrow if `VESSEL.md` is accepted as a core convention, but a broader configurable bootstrap-file policy may be the more maintainable direction.
