Full review comments:

- **[P1] Preserve the A2A follow-up path:** `src/agents/tools/sessions-send-tool.ts:283`
  - body: This changes every sessions_send call to report delivery as skipped/none and removes the reply-back/announce flow. Current docs and tests still define peer sessions_send as an A2A conversation path, so dispatch-only behavior needs an explicit mode rather than replacing all sends.
  - confidence: 0.9
- **[P1] Preserve stored external routes for sessions_send:** `src/auto-reply/reply/session.ts:561-563`
  - body: Passing allowInterSessionExternalRoute=false into initSessionState makes sessions_send turns persist webchat/session routing over an established Telegram/Discord route. Suppress only the current outward delivery while keeping the durable external route intact for later announcements and follow-ups.
  - confidence: 0.88
- **[P2] Tighten monologue stripping before dropping blocks:** `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:419-425`
  - body: The marker-count branch keeps only the final block when broad phrases like "the prompt" and "I should" appear earlier, which can occur in legitimate multi-paragraph answers. Gate this on a stronger structural leak signal before deleting user-facing text.
  - confidence: 0.82
- **[P2] Remove the hardcoded personal-name bypass:** `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:383`
  - body: The production exclusion regex contains the literal name Bernhard, so any block mentioning that name bypasses the internal-monologue filter even when it otherwise matches the leak pattern. This looks like a debugging artifact and makes sanitizer behavior depend on one personal name.
  - confidence: 0.9
