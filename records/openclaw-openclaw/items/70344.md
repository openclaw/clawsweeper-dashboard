## Summary

Keep open. The PR targets real reply-boundary bugs that are still not fully solved on current main, but the proposed patch is not mergeable as-is because it introduces durable route-state and sanitizer regressions that need maintainer rework.

## What This Changes

The PR changes embedded assistant-output suppression, user-facing text sanitization, sessions_send A2A handling, and session route resolution/tests so tool-use preludes and sessions_send replies stay out of external chats.

## Best Possible Solution

Split the work into narrower decisions: land or adapt a structured tool-use suppression fix such as #71589, make an explicit maintainer call on sessions_send A2A announce semantics, and if current-turn outward delivery must be suppressed, do it without overwriting stored external route state. Any prelude sanitizer should be backed by stronger structural leak signals and negative tests for valid replies.

## Reproduction Assessment

Yes. The current-main code and test harnesses give a high-confidence reproduction path: emit an unphased assistant message with stopReason="toolUse" and tool-call content in the embedded subscription harness, and exercise sessions_send/session-delivery route resolution with an existing Telegram or Discord lastRoute.

## Solution Assessment

No. The PR addresses real symptoms, but this is not the best fix because it bundles separate policy decisions and introduces route persistence plus sanitizer-regression risks; the safer alternative is a narrow suppression patch plus an explicit sessions_send delivery policy change that preserves durable route state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve stored external routes for sessions_send:** `src/auto-reply/reply/session.ts:558-561`
  - body: For sessions_send inter-session turns this makes route resolution fall through to webchat/session values, and initSessionState then persists those values back to the target session. A Telegram/Discord-backed session can lose its stored route, breaking later announce, cron, or system follow-ups. Suppress only the current turn's outward delivery while keeping the persisted external route intact.
  - confidence: 0.88
- **[P2] Gate monologue stripping on stronger leak signals:** `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:418-424`
  - body: This marker-count branch keeps only the final block when earlier paragraphs contain broad phrases like "the prompt" and "I should". Those phrases can appear in legitimate multi-paragraph answers, so the sanitizer can silently delete valid user-facing content. Require stronger structural evidence of a leaked internal prelude before replacing the whole reply.
  - confidence: 0.82
- **[P2] Remove the hardcoded personal-name bypass:** `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:383`
  - body: The production exclusion regex includes the literal name `Bernhard`, so any block containing that name skips the internal-monologue filter even if it otherwise matches the leak pattern. This looks like a test/debug artifact and makes sanitizer behavior depend on one personal name.
  - confidence: 0.9
