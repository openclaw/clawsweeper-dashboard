## Summary

Keep open for maintainer review: the patch appears focused and correct, and the supplied proof exercises the recovery decision path, but it intentionally changes session/message recovery by aborting stale active-run handles when queued work is waiting.

## What This Changes

The PR adds a stale-progress liveness gate to diagnostic stuck-session recovery, threads the resolved stuck-session abort threshold into recovery requests, and adds regression tests for leaked active-run handles with queued work.

## Best Possible Solution

Land the liveness-gated recovery after owner review if the existing abort threshold is the accepted boundary; otherwise adjust the guard before merge.
