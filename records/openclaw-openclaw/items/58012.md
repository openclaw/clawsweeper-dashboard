## Summary

Keep open. Current main still routes OpenRouter through the passthrough-Gemini replay hook, and that hook does not select strict9 tool-call-id sanitization for Mistral-like model IDs; direct Mistral and the transcript hygiene docs still define strict9 behavior.

## What This Changes

The issue requests restoring strict9 tool-call-id sanitization for Mistral-family models routed through proxy providers such as OpenRouter.

## Best Possible Solution

Add provider-owned replay policy composition for proxy transports so Mistral-family wrapped model IDs select strict9 tool-call-id sanitization while preserving existing OpenRouter Gemini thought-signature handling.

## Reproduction Assessment

Yes at source level. Resolving transcript policy for OpenRouter with a wrapped Mistral model goes through the OpenRouter passthrough-Gemini hook, which currently returns no strict9 tool-call-id policy while direct Mistral returns strict9.

## Solution Assessment

Yes. The maintainable fix is a narrow provider/plugin replay-policy composition update; restoring the removed provider-capabilities fallback would fight the current plugin-owned replay architecture.
