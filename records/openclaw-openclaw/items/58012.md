## Summary

Keep open. Current main still routes OpenRouter through the passthrough-Gemini replay hook, and that hook does not select strict9 tool-call-id sanitization for wrapped Mistral model IDs even though direct Mistral does and the docs describe strict9 for Mistral model-id detection.

## What This Changes

The issue reports that Mistral-family models routed through proxy providers such as OpenRouter no longer receive strict9 tool-call-id sanitization after provider replay/capability refactors.

## Best Possible Solution

Add provider-owned replay policy composition for proxy transports so Mistral-family wrapped model IDs select strict9 tool-call-id sanitization while preserving existing OpenRouter Gemini thought-signature cleanup and replay validation behavior.

## Reproduction Assessment

Yes, at source level. Resolving transcript policy for OpenRouter with a wrapped Mistral model goes through the OpenRouter passthrough-Gemini hook, which currently returns no strict9 tool-call-id policy, while direct Mistral returns strict9.

## Solution Assessment

Yes, the maintainable fix is a narrow provider/plugin replay-policy composition update. Restoring the old deleted provider-capabilities fallback would fight the current plugin-owned replay architecture.
