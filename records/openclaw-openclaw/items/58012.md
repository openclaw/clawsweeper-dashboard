## Summary

Keep open. Current main and the latest release still leave OpenRouter on the passthrough-Gemini replay hook without strict9 tool-call-id policy for Mistral-like model ids, while direct Mistral still sets strict9; no open canonical PR currently owns the fix.

## What This Changes

The issue asks OpenClaw to restore strict9 tool-call-id sanitization for Mistral-family models routed through proxy providers such as OpenRouter.

## Best Possible Solution

Add provider-owned replay-policy composition for passthrough proxy providers so Mistral-family model ids enable strict9 while preserving the current plugin-owned replay architecture.

## Reproduction Assessment

Yes, at source level. OpenRouter resolves to the passthrough-Gemini replay hook, that hook returns no strict9 tool-call-id policy for Mistral-like ids, and replay history only rewrites ids when those policy fields are present.

## Solution Assessment

Yes, but not by restoring the deleted core provider-capabilities fallback. The maintainable fix is to compose Mistral-family strict9 detection into the current provider replay hook or shared passthrough family.
