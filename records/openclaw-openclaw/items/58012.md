## Summary

Keep open. Current main and the latest release still route OpenRouter through the passthrough-Gemini replay hook, which does not enable strict9 tool-call-id sanitization for Mistral-like model ids; the prior cross-provider strict9 fix was left behind when replay policy moved into plugin-owned hooks.

## What This Changes

The issue asks OpenClaw to restore strict9 tool-call-id sanitization for Mistral-family models routed through proxy providers such as OpenRouter.

## Best Possible Solution

Add provider-owned replay-policy composition for passthrough proxy providers so Mistral-family model ids enable strict9 while preserving the current plugin-owned replay architecture.

## Reproduction Assessment

Yes, at source level. OpenRouter resolves to the passthrough-Gemini replay hook, that hook returns no strict9 tool-call-id policy for Mistral-like ids, and replay history only rewrites ids when those policy fields are present.

## Solution Assessment

Yes, but not by restoring the deleted core provider-capabilities fallback. The narrow maintainable fix is to compose strict9 model-id detection into the current provider replay hook or shared passthrough family.
