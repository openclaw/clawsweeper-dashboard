## Summary

Keep open. Current main and the latest release still route OpenRouter through the passthrough-Gemini replay hook, which returns no strict9 tool-call-id policy for Mistral-like model ids, while direct Mistral still returns strict9; the earlier provider-capabilities fallback fix is no longer present after the plugin replay refactor.

## What This Changes

Apply strict9 tool-call-id sanitization when Mistral-family models are used through proxy providers such as OpenRouter.

## Best Possible Solution

Add provider-owned replay-policy composition for passthrough proxy providers so Mistral-family model ids enable strict9 while preserving existing Gemini thought-signature cleanup and the plugin-owned replay architecture.

## Reproduction Assessment

Yes, at source level. OpenRouter resolves to the passthrough-Gemini replay hook, that hook returns no strict9 tool-call-id policy for Mistral-like ids, and replay history only rewrites ids when those policy fields are present.

## Solution Assessment

Yes. The narrow maintainable fix is to compose strict9 model-id detection into the current provider replay hook/shared passthrough family; restoring the deleted core provider-capabilities fallback would fight the current architecture.
