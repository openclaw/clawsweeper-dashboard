## Summary

Keep open. Current main still lacks strict9 tool-call-id policy for the OpenRouter proxy replay path: OpenRouter uses the shared passthrough-Gemini replay hook, that hook only handles Gemini thought-signature cleanup, and replay history only rewrites tool-call ids when the resolved transcript policy includes both sanitizeToolCallIds and toolCallIdMode.

## What This Changes

The issue reports that Mistral-family models routed through proxy providers such as OpenRouter no longer get strict9 tool-call-id sanitization after provider replay/capability refactors.

## Best Possible Solution

Add provider-owned replay policy composition for proxy transports so Mistral-family wrapped model ids select strict9 tool-call-id sanitization while preserving existing passthrough Gemini thought-signature handling.

## Reproduction Assessment

Yes at source level. Resolving transcript policy for OpenRouter with a wrapped Mistral model uses the passthrough-Gemini replay hook, and that hook currently returns no strict9 tool-call-id policy while direct Mistral returns strict9.

## Solution Assessment

Yes. The narrow maintainable fix is provider-owned replay-policy composition in the current plugin/replay helper surface; restoring the deleted provider-capabilities fallback would fight the current architecture.
