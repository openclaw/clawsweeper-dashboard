## Summary

Keep open: the patch is a plausible focused fix for the linked read-only `/agent` mount bug and I did not find a discrete code defect, but it still needs real behavior proof beyond tests because it changes a sandbox read boundary.

## What This Changes

The PR refactors the read workspace-root guard to accept explicit additional container mounts, wires the read-only `/agent` mount for sandboxed read tools, adds regression tests and a test-helper option, and adds a changelog entry.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the read guard only remaps `containerWorkdir`, the current mounted-paths test expects `/agent` reads to fail under workspace-only mode, and docs define `/agent` as the read-only workspace mount.

## Solution Assessment

Yes, the proposed shape is the narrow maintainable fix: pass an explicit additional mount into the read guard only for `workspaceAccess: "ro"`, while leaving write/edit/apply_patch and `/tmp` policy unchanged. It still needs real behavior proof before merge.
