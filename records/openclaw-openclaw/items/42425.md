## Summary

## What This Changes

This PR adds a startup hook loader intended to load workspace-local internal hooks for every configured agent workspace and scope those handlers by workspace.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates agent workspaces while reusing current hook configuration, precedence, cleanup, plugin discovery, and path-hardening contracts.

## Reproduction Assessment

Yes from source inspection, though I did not run a live Gateway repro. Current main documents per-agent workspace hooks, but the live startup path calls loadInternalHooks once with params.defaultWorkspaceDir and discovery derives <workspace>/hooks from that single argument.

## Solution Assessment

No, not as-is. Multi-workspace loading is the right direction, but this branch must be rebased or replaced on the current post-attach startup path and preserve the existing hook opt-in, cleanup, source-precedence, plugin discovery, and path-hardening contracts.
