  - body: Current main no longer has this startup module; Gateway startup now goes through src/gateway/server-startup-post-attach.ts, which still loads hooks only from params.defaultWorkspaceDir. Wiring the new loader here would not fix the live path after rebase.
  - confidence: 0.95
- **[P2] Preserve configured hook opt-ins:** `src/hooks/loader.ts:54-55`
  - body: The new startup loader returns unless hooks.internal.enabled is truthy, but current main treats enabled entries, installs, extraDirs, and legacy handlers as valid opt-ins through hasConfiguredInternalHooks. Those configurations would stop loading hooks at Gateway startup.
  - confidence: 0.93
- **[P2] Track loader-owned registrations on reload:** `src/hooks/loader.ts:185`
  - body: registerHookEntries calls registerInternalHook without adding the registered wrapper to the loader-owned registration list. Current cleanup depends on that list, so repeated loads can leave duplicate workspace handlers registered.
  - confidence: 0.9
- **[P2] Keep current hook source precedence:** `src/hooks/loader.ts:442-443`
  - body: Suppressing shared handlers whenever a workspace-local hook has the same name lets workspace code override managed, plugin, or bundled hooks. Current policy only allows workspace hooks to override workspace hooks, so this can disable trusted shared behavior.
  - confidence: 0.89
- **[P2] Keep extra hook dirs at managed precedence:** `src/hooks/loader.ts:312-316`
  - body: hooks.internal.load.extraDirs are currently discovered as openclaw-managed sources and share managed enablement and collision policy. Loading them as openclaw-workspace changes default enablement and precedence for configured extra directories.
  - confidence: 0.88
- **[P2] Include plugin hook dirs in shared discovery:** `src/hooks/loader.ts:329`
  - body: The shared startup discovery rebuilds the source list from extra, bundled, and managed directories but omits plugin hook directories. Plugin-provided internal hooks would stop loading during Gateway startup.
  - confidence: 0.88
- **[P2] Preserve fail-closed hook path checks:** `src/hooks/loader.ts:489`
  - body: safeRealpathOrResolve falls back to path.resolve before opening and importing hook handlers, whereas current main skips unreadable hook roots after realpath failure and opens under a checked root. Keep the fail-closed path before loading trusted hook code.
  - confidence: 0.84
