## Summary

## What This Changes

This PR adds a startup-specific multi-workspace internal hook loader, scopes workspace-local handlers, adds loader tests, and wires Gateway startup to enumerate configured agent workspaces.

## Best Possible Solution

Land a rebased or replacement fix in `src/gateway/server-startup-post-attach.ts` that loads per-agent workspace hooks while preserving current opt-in, cleanup, discovery, precedence, and path-boundary contracts.

## Reproduction Assessment

Yes. Current main documents per-agent workspace hooks, but the live Gateway startup path calls `loadInternalHooks` once with `params.defaultWorkspaceDir`, and discovery derives `<workspace>/hooks` from that single argument.

## Solution Assessment

No, not as-is. Multi-workspace loading is the right direction, but this branch must be rebased or replaced on the current post-attach startup path and reuse the existing hook opt-in, source-precedence, cleanup, and path-hardening contracts.
