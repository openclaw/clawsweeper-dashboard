## Summary

Keep open: the documented non-default workspace hook bug is still present on current main, but this PR is not safe to merge as-is because it targets a removed startup module, diverges from current hook-loader contracts, and lacks real Gateway behavior proof.

## What This Changes

The PR adds a multi-workspace internal hook startup loader, wires Gateway startup to load every configured agent workspace's hooks, and expands hook loader regression tests.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates configured agent workspaces while preserving hook opt-in, plugin/managed precedence, reload cleanup, fail-closed path checks, and startup event semantics.

## Reproduction Assessment

Yes, by source inspection. Current main documents per-agent workspace hooks, but Gateway startup still calls loadInternalHooks only with params.defaultWorkspaceDir, so non-default workspace hook directories are not discovered.

## Solution Assessment

No, not as-is. Multi-workspace startup loading is the right direction, but this branch must be retargeted to the current post-attach startup path and preserve newer hook loader contracts before merge.
