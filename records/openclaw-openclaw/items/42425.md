## Summary

Keep open: the bug is still source-reproducible on current main and this PR is the canonical fix thread, but the branch is stale against the live startup module and regresses current hook opt-in, reload, precedence, plugin-source, and fail-closed path contracts.

## What This Changes

The PR adds a startup hook loader that scans all configured agent workspaces, scopes workspace-local handlers, wires Gateway startup to call it, and expands hook loader tests.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates agent workspaces while preserving current hook discovery, opt-in, precedence, reload cleanup, and path-hardening contracts.

## Reproduction Assessment

Yes, by source inspection. The docs define per-agent workspace hooks, while current main still calls loadInternalHooks with only params.defaultWorkspaceDir in the live Gateway startup path.

## Solution Assessment

No, not as-is. Multi-workspace loading is the right direction, but this branch targets an obsolete startup file and must be retargeted to server-startup-post-attach.ts while preserving newer hook-loader contracts.
