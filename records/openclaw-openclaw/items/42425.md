## Summary

Keep open. The underlying bug is still valid on current main, but this stale external PR is not mergeable as-is because it targets a removed Gateway startup surface, regresses current hook-loading contracts, and still lacks after-fix real behavior proof.

## What This Changes

The branch adds a multi-workspace internal hook startup loader, wires it into Gateway sidecar startup, and expands hook loader tests for non-default agent workspace hooks.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates configured agent workspaces while preserving current hook opt-in, plugin/managed precedence, source classification, startup event semantics, reload cleanup, and path-hardening contracts.

## Reproduction Assessment

Yes, by source inspection. Current main documents per-agent workspace hooks, but Gateway startup still calls loadInternalHooks only with params.defaultWorkspaceDir, so non-default workspace hook directories are not discovered.

## Solution Assessment

No, not as-is. Multi-workspace startup loading is the right direction, but this branch needs a rebase onto the live post-attach startup path and must preserve newer hook loader contracts before it is safe to merge.
