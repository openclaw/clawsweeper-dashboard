## Summary

Keep open: the underlying bug is still source-reproducible on current main, but this PR is stale/conflicting, lacks required real Gateway proof, and regresses current hook loading, source precedence, and path-hardening contracts.

## What This Changes

The PR adds a startup-specific internal hook loader intended to register shared hooks once and workspace-local hooks once per configured agent workspace.

## Best Possible Solution

Land a rebased or replacement change in `src/gateway/server-startup-post-attach.ts` that loads per-agent workspace hooks while preserving configured opt-ins, loader-owned cleanup, plugin/extra/shared discovery, source precedence, and fail-closed path checks, then add real Gateway proof.

## Reproduction Assessment

Yes, source-reproducible: docs define per-agent workspace hooks, while current main still calls `loadInternalHooks` once with `params.defaultWorkspaceDir`. I did not run a live Gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. Multi-workspace loading is the right direction, but this branch targets an obsolete startup file and bypasses current hook opt-in, cleanup, precedence, plugin discovery, and path-hardening contracts.

  - confidence: 0.88

  - confidence: 0.88
