## Summary

Keep open: current main still has the documented non-default workspace hook loading bug, but this stale external PR is not mergeable as-is because it targets an obsolete startup path, regresses current hook loader contracts, and lacks real Gateway behavior proof.

## What This Changes

The branch adds a startup-specific internal-hook loader, wires Gateway startup to enumerate configured agent workspace directories, and expands hook loader regression tests.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates configured agent workspaces while preserving current hook opt-ins, plugin/managed precedence, reload cleanup, fail-closed path checks, and startup event semantics.

## Reproduction Assessment

Yes, by source inspection. Current docs define per-agent workspace hooks, while current Gateway startup still passes only params.defaultWorkspaceDir into loadInternalHooks.

## Solution Assessment

No, not as-is. Multi-workspace startup loading is the right direction, but this branch must be rebased onto the live post-attach startup path and preserve newer hook loader policy, reload, plugin, and security contracts.

  - confidence: 0.89
- **[P2] Keep extra hook dirs at managed precedence:** `src/hooks/loader.ts:312-316`
  - body: Current main loads hooks.internal.load.extraDirs as openclaw-managed, sharing managed-hook enablement and collision behavior. Marking those entries as openclaw-workspace changes default enablement and precedence for configured extra hook directories.
  - confidence: 0.88

  - body: Current discovery includes plugin hook directories via resolvePluginHookDirs. The new shared startup discovery rebuilds the shared list from extra, bundled, and managed dirs only, so plugin-bundled internal hooks would stop loading at Gateway startup.
  - confidence: 0.88
- **[P2] Preserve fail-closed hook path checks:** `src/hooks/loader.ts:203`
  - body: Current main requires readable real paths and open-root checks before importing hook handlers. This branch falls back from realpath failures before opening hook code, weakening hardening around trusted local hook execution.
