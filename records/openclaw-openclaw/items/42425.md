## Summary

Keep this PR open: it targets a valid documented bug, but the branch is now dirty against current main and the proposed loader would regress current hook configuration, plugin-hook discovery, and extra-dir precedence contracts.

## What This Changes

The PR adds a startup-specific hook loader that loads workspace-local hooks for every configured agent workspace, scopes those handlers by workspace, calls it from Gateway startup, and adds regression tests.

## Best Possible Solution

Land a rebased or replacement implementation that loads workspace-local hooks for every configured agent workspace while preserving current hook gating, plugin-hook discovery, managed extra-dir precedence, loader cleanup, and post-attach startup structure.

## Reproduction Assessment

Yes, from source inspection. Current main documents per-agent workspace hooks but Gateway startup still calls loadInternalHooks with only the default workspace directory, so non-default workspace hook directories are not discovered at startup.

## Solution Assessment

No, not as-is. Multi-workspace startup loading is the right narrow direction, but this PR must be adapted to current main and preserve the current hook configuration, plugin discovery, extra-dir, and startup-post-attach contracts.
