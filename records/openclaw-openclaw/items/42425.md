## What This Changes

The PR adds a startup-specific multi-workspace internal hook loader, tests, and Gateway startup wiring to load workspace hooks for every configured agent workspace.

## Best Possible Solution

Land a rebased or replacement fix in `src/gateway/server-startup-post-attach.ts` that loads per-agent workspace hooks while preserving configured opt-ins, cleanup, plugin/extra/shared discovery, source precedence, and fail-closed path checks.

## Reproduction Assessment

Yes, source-reproducible. Current main documents per-agent workspace hooks but the live Gateway startup path calls `loadInternalHooks` once with `params.defaultWorkspaceDir`, and discovery derives `<workspace>/hooks` from that single argument.

## Solution Assessment

No, not as-is. Multi-workspace hook loading is the right direction, but this branch must be rebased onto the current post-attach startup path and preserve the existing hook opt-in, cleanup, source-precedence, plugin, and path-hardening contracts.
