## Summary

Keep this PR open: it targets a real current-main bug, but the branch is conflicting and its implementation no longer fits the current Gateway startup and hook-loading contracts.

## What This Changes

The PR adds a startup-specific multi-workspace internal hook loader and wires Gateway startup to pass every configured agent workspace.

## Best Possible Solution

Land a rebased or replacement fix in `src/gateway/server-startup-post-attach.ts` that loads per-agent workspace hooks while preserving hook opt-ins, cleanup, plugin/extra/shared discovery, source precedence, and fail-closed path checks.

## Reproduction Assessment

Yes, source-reproducible: docs define per-agent workspace hooks, and current main still calls `loadInternalHooks` once with only `params.defaultWorkspaceDir` in the live Gateway startup path. I did not run a live Gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. Multi-workspace hook loading is the right direction, but this branch must be rebased onto `src/gateway/server-startup-post-attach.ts` and preserve current hook security, cleanup, discovery, and precedence contracts.

  - body: Current main no longer uses this startup module for sidecar hook loading; Gateway sidecars load hooks from `src/gateway/server-startup-post-attach.ts`. Wiring the new loader here will not fix the live path after rebase.

  - confidence: 0.93

  - confidence: 0.9
- **[P2] Keep current hook source precedence:** `src/hooks/loader.ts:442-443`
  - body: Suppressing shared handlers whenever a workspace-local hook has the same name lets workspace code override managed, plugin, or bundled hooks. Current policy only allows workspace hooks to override workspace hooks.
  - confidence: 0.89
- **[P2] Keep extra hook dirs at managed precedence:** `src/hooks/loader.ts:312-316`
  - body: `hooks.internal.load.extraDirs` are currently discovered as managed sources and share managed enablement and collision policy. Loading them as workspace sources changes default enablement and precedence for configured extra directories.
  - confidence: 0.88
- **[P2] Include plugin hook dirs in shared discovery:** `src/hooks/loader.ts:329`
  - body: The shared startup discovery rebuilds the source list from extra, bundled, and managed directories but omits plugin hook directories. Plugin-provided internal hooks would stop loading during Gateway startup.
  - confidence: 0.88
- **[P2] Preserve fail-closed hook path checks:** `src/hooks/loader.ts:203-206`
  - body: `safeRealpathOrResolve` falls back to `path.resolve` before opening and importing hook handlers, whereas current main skips unreadable hook roots after realpath failure and imports only under a checked root.
