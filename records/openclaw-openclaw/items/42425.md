## Summary

## What This Changes

This PR adds a multi-workspace internal hook loader, wires Gateway startup to call it, and expands hook loader regression tests.

## Best Possible Solution

Land a rebased or replacement fix in src/gateway/server-startup-post-attach.ts that enumerates agent workspaces while preserving current hook opt-in, precedence, reload cleanup, plugin hook discovery, and path-hardening contracts.

## Reproduction Assessment

Yes, by source inspection. The docs define per-agent workspace hooks, while current main still calls loadInternalHooks with only params.defaultWorkspaceDir in the live Gateway startup path.

## Solution Assessment

No, not as-is. Multi-workspace loading is the right direction, but this branch must be rebased or replaced around the current post-attach startup path and newer hook-loader contracts.

  - confidence: 0.9
- **[P2] Keep current hook source precedence:** `src/hooks/loader.ts:442-443`
  - body: Current main prevents workspace hook code from overriding bundled, plugin, or managed hook code with the same name. Suppressing shared handlers whenever a same-name workspace hook exists can disable managed or plugin behavior for that workspace.
  - confidence: 0.89
- **[P2] Keep extra hook dirs at managed precedence:** `src/hooks/loader.ts:315`
  - body: Current main loads hooks.internal.load.extraDirs as openclaw-managed sources, sharing managed-hook enablement and collision behavior. Marking those entries as workspace sources changes precedence and trust/cache policy for configured extra directories.
  - confidence: 0.88
- **[P2] Include plugin hook dirs in shared loading:** `src/hooks/loader.ts:329`
  - body: Current discovery includes plugin-provided hook directories via resolvePluginHookDirs. The new shared startup discovery rebuilds shared sources from extra, bundled, and managed directories only, so plugin-bundled internal hooks would stop loading at Gateway startup.
  - confidence: 0.88
- **[P2] Preserve fail-closed hook path checks:** `src/hooks/loader.ts:203`
  - body: Current main requires readable real paths before opening and importing trusted hook handlers. This branch falls back from realpath failures to path.resolve, weakening hardening around trusted local hook execution.
  - confidence: 0.84
