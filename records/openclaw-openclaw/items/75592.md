## Summary

Keep open. The PR is in-scope for the Voice Call plugin and current main does not already implement the feature, but the proposed patch is not merge-ready because it treats caller ID as verified identity for private context injection, splits transcript logs away from the configured store, and lacks required docs/changelog coverage.

## What This Changes

The PR adds Voice Call realtime caller-context config, caller-specific greetings/profile injection, and optional realtime transcript JSONL logging.

## Best Possible Solution

Keep the feature in the Voice Call plugin, but revise it so private caller context is gated by stronger identity proof or an explicit low-assurance confirmation flow, transcript logging honors the configured store or requires an explicit path, and docs/changelog explain setup, privacy, storage, and retention.

## Reproduction Assessment

Yes. A static reproduction is clear: configure realtime.callerContext for a matching Twilio From and the PR injects profile/voice-card context into provider instructions; enable transcriptLog without a path and the patch resolves it under the hardcoded home store.

## Solution Assessment

No. The plugin-owned location is appropriate, but this is not the best maintainable solution until the caller identity trust model, transcript storage default, and user-facing privacy documentation are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Avoid treating caller ID as verified identity:** `extensions/voice-call/src/webhook/realtime-handler.ts:116-124`
  - body: The new prompt and private profile/voice-card injection are selected from callerMeta.from, but current inbound acceptance is provider-supplied From matched against allowFrom and documented as low-assurance caller-ID filtering. A spoofed caller ID could trigger personalized greetings or private context before stronger verification, so gate this behind stronger identity proof or an explicit low-assurance confirmation flow.
  - confidence: 0.88
- **[P2] Honor the configured voice-call store for transcript logs:** `extensions/voice-call/src/webhook/realtime-handler.ts:154-155`
  - body: When transcript logging is enabled without realtime.transcriptLog.path, this helper hardcodes ~/.openclaw/voice-calls/realtime-transcripts.jsonl even though the schema says the default is the voice-call store directory and CallManager already honors config.store. Installations with a custom store will silently split realtime aftercare logs into a different location.
  - confidence: 0.82
- **[P3] Document the realtime caller-context config:** `extensions/voice-call/src/config.ts:324-327`
  - body: This adds user-facing realtime.callerContext and transcriptLog keys that read local files and persist transcript fragments, but the PR does not update the Voice Call docs or CHANGELOG.md. Users need setup syntax plus privacy, storage, and retention guidance, and repo policy expects a changelog entry for this feature.
  - confidence: 0.9
