## Summary

Keep open. Current main still does not implement realtime caller-context or transcript-log support, and this PR remains blocked by caller-identity trust, transcript storage, documentation, and privacy issues in the live Voice Call path.

## What This Changes

The PR adds realtime Voice Call caller-context and transcript-log config, injects caller-specific greetings and profile files into realtime instructions, appends optional JSONL transcript fragments, and updates voice-call tests.

## Best Possible Solution

Revise the Voice Call plugin so caller-specific private context is gated by stronger identity proof or an explicit low-assurance opt-in with confirmation safeguards, make transcript logging use the configured store or require an explicit path, and document setup, privacy, and retention with a changelog entry.

## Reproduction Assessment

Yes. A static reproduction is clear: configure realtime.callerContext for a matching provider From value and the PR injects profile/voice-card text into provider instructions; configure config.store and omit realtime.transcriptLog.path and the PR uses a separate default path.

## Solution Assessment

No. The plugin-owned location is appropriate, but this PR is not the best safe implementation until the caller-ID assurance model, transcript storage default, and user-facing privacy docs are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Avoid treating caller ID as verified identity:** `extensions/voice-call/src/webhook/realtime-handler.ts:116-124`
  - body: The new prompt and private profile/voice-card injection are selected only from callerMeta.from, while current inbound acceptance is provider-supplied From matched against allowFrom and documented as low-assurance caller-ID filtering. A spoofed caller ID could trigger personalized greetings or private context before stronger verification, so gate this behind stronger identity proof or make it an explicit low-assurance flow that confirms identity before injecting private files.
  - confidence: 0.88
- **[P2] Honor the configured voice-call store for transcript logs:** `extensions/voice-call/src/webhook/realtime-handler.ts:154-155`
  - body: When transcript logging is enabled without realtime.transcriptLog.path, this helper uses a fixed home-directory default even though the schema says the default is the voice-call store directory and CallManager already honors config.store. Installations with a custom store will silently split realtime aftercare logs into a different location.
  - confidence: 0.82
- **[P3] Document realtime caller-context and transcript logging:** `extensions/voice-call/src/config.ts:324-327`
  - body: This adds user-facing realtime.callerContext and transcriptLog keys that read local files and persist transcript fragments, but the PR does not update the Voice Call docs or CHANGELOG.md. Users need setup syntax plus privacy, storage, and retention guidance, and repo policy expects a changelog entry for this feature.
  - confidence: 0.9
