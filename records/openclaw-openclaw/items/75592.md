## Summary

Keep open: the PR adds a plausible Voice Call plugin feature, but it is not already on current main and the current branch has blocking caller-identity, transcript-storage, docs/privacy, and real-behavior-proof gaps.

## What This Changes

The PR adds Voice Call realtime caller context keyed by caller ID, caller-specific greetings/profile and voice-card injection, and optional realtime transcript JSONL logging.

## Best Possible Solution

Keep the feature in the Voice Call plugin, but require a verified or explicitly low-assurance caller-context design, configured-store logging, user-facing privacy/storage docs, and redacted real-call proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows that configuring realtime.callerContext for a matching From number injects caller profile/voice-card text into provider instructions, and enabling transcriptLog without path uses a hardcoded home-store fallback.

## Solution Assessment

No. Plugin-owned placement is right, but this design is not ready until identity assurance, transcript storage, privacy documentation, and real behavior proof are settled.
