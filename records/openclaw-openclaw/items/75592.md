## Summary

Keep open: current main does not contain this Voice Call caller-context feature, but the PR is not merge-ready because it keys private context injection to low-assurance caller ID and adds transcript logging without a settled privacy/storage contract. The prior ClawSweeper review already identified the same unresolved blockers, and no follow-up patch is present on the PR head.

## What This Changes

The PR adds Voice Call realtime caller-context config keyed by caller ID, injects bounded profile/voice-card instructions and greetings, appends optional transcript JSONL logging, and updates voice-call tests.

## Best Possible Solution

Revise Voice Call to require stronger verified identity or an explicit low-assurance opt-in before private caller-file injection, align transcript logs with the configured store or explicit paths, and add privacy/retention docs plus changelog coverage.

## Reproduction Assessment

Yes. A static reproduction is enough: configure realtime.callerContext for a matching Twilio From value and the PR injects profile/voice-card context into provider instructions; configure config.store and omit realtime.transcriptLog.path and the PR still resolves logs under the default home path.

## Solution Assessment

No. The plugin-owned placement is appropriate, but the current implementation is not the safest maintainable solution until caller-ID assurance, transcript storage defaults, and user-facing privacy documentation are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Avoid treating caller ID as verified identity:** `extensions/voice-call/src/webhook/realtime-handler.ts:116-124`
  - body: The new prompt and private profile/voice-card injection are selected only from callerMeta.from, while inbound acceptance is provider-supplied From matched against allowFrom and documented as low-assurance caller-ID filtering. A spoofed caller ID could trigger personalized greetings or private context before stronger verification, so gate this behind stronger identity proof or make it an explicit low-assurance flow that confirms identity before injecting private files.
  - confidence: 0.88
- **[P2] Honor the configured voice-call store for transcript logs:** `extensions/voice-call/src/webhook/realtime-handler.ts:154-155`
  - body: When transcript logging is enabled without realtime.transcriptLog.path, this helper hardcodes ~/.openclaw/voice-calls/realtime-transcripts.jsonl even though the schema says the default is the voice-call store directory and CallManager already honors config.store. Installations with a custom store will silently split realtime aftercare logs into a different location.
  - confidence: 0.82
- **[P3] Document the realtime caller-context config:** `extensions/voice-call/src/config.ts:324-327`
  - body: This adds user-facing realtime.callerContext and transcriptLog keys that read local files and persist transcript fragments, but the PR does not update the Voice Call docs or CHANGELOG.md. Users need setup syntax plus privacy, storage, and retention guidance, and repo policy expects a changelog entry for this feature.
  - confidence: 0.9
