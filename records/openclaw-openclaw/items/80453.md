## Summary

This PR is a plausible fix for the managed-service `GH_TOKEN` path, but it should stay open because the latest head still broadens a security boundary beyond the proven credential case.

## What This Changes

The PR changes durable service env parsing, trusted gateway exec env inheritance, docs, and regression tests so service-managed credential keys like `GH_TOKEN` can reach trusted exec children.

## Best Possible Solution

Keep the service-managed `GH_TOKEN` path, but constrain trusted inheritance to an explicit credential-key policy and add regression coverage proving non-secret override-only keys stay blocked.

## Reproduction Assessment

Yes. Source inspection of current main shows `GH_TOKEN` is dropped from durable `.env` parsing and stripped from inherited host exec env, while the PR's real macOS proof shows the patched trusted exec path authenticating `gh` via `GH_TOKEN`.

## Solution Assessment

No. Reusing service-managed metadata for trusted exec is a reasonable shape, but the proposed allowlist is broader than the proven credential use case and should be narrowed before merge.

## Review Findings
