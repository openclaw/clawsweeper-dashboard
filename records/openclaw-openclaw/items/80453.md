## Summary

Keep open: the PR addresses a real current-main gap and its real behavior proof is sufficient, but the patch should not merge while service-managed metadata can broaden trusted inherited env beyond the proven credential use case.

## What This Changes

The PR lets durable service-managed env keys such as `GH_TOKEN` survive gateway install and reach trusted gateway exec children, with docs and regression coverage.

## Best Possible Solution

Keep the service-managed `GH_TOKEN` path, but constrain trusted inheritance to an explicit credential-key policy and add coverage proving noncredential override-only keys stay blocked.

## Reproduction Assessment

Yes. Current main source shows `GH_TOKEN` is filtered from durable `.env` parsing and stripped from inherited exec env; the PR screenshot shows the patched managed gateway path authenticating `gh` through `GH_TOKEN`.

## Solution Assessment

No. Reusing service-managed metadata for trusted gateway exec is a reasonable shape, but this implementation is broader than the credential use case and should be narrowed before merge.

## Review Findings

Overall correctness: patch is incorrect
