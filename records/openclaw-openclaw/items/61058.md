## What This Changes

The issue requests that `openclaw secrets audit --check` treat exact `$VAR` and `${VAR}` env references as safe references instead of `PLAINTEXT_FOUND`.

## Best Possible Solution

Land one canonical, source-aware secrets-audit fix that distinguishes authored env references from real plaintext across config, auth-profile, and `.env` audit paths, with docs and focused regression coverage.
