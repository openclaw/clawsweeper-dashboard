## Summary

Keep open. Current main still routes the expected startup-sidecars code-1013 pre-connect close through WARN, and open PR #77189 is already paired with this issue as the focused fix.

## What This Changes

The issue asks to downgrade expected gateway startup WebSocket code-1013 close-before-connect logging from WARN to debug/info or a grace-based warning policy.

## Best Possible Solution

Land a narrow gateway logging fix that downgrades only the expected startup-sidecars 1013 pre-connect close while preserving WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level: force isStartupPending() true, send a connect frame, and the handler closes with code 1013/reason "gateway starting" before a client is established; current close logging then falls through to WARN. I did not run a live gateway or tests in this read-only sweep.

## Solution Assessment

Yes. The narrow maintainable fix is to branch the pre-connect close logger on the existing startup close signal and add regression coverage; #77189 appears to target that path, so the issue should track that PR until it is resolved.
