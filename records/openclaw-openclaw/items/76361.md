## Summary

Keep open. Current main still logs the expected startup-sidecars 1013 pre-connect close through WARN, and open PR #77189 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

The issue asks to downgrade expected gateway-startup WebSocket close-before-connect events with code 1013 from WARN to debug/info while preserving warnings for unexpected failures.

## Best Possible Solution

Treat only the gateway-owned `startup-sidecars-pending` 1013 close as expected startup churn at debug level, with a regression assertion, while preserving WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level: force `isStartupPending()` true, send a connect frame, and the handler marks `startup-sidecars-pending`, replies retryable unavailable, then closes 1013 with `gateway starting`; the close handler currently routes that non-client close to WARN.

## Solution Assessment

Yes. The narrow maintainable fix is to branch the pre-connect close logger on the existing startup close cause and add a regression assertion without changing retry timing, close code, readiness, or client reconnect behavior.
