## Summary

Keep open. Current main still routes the expected startup-sidecars 1013 pre-connect close through WARN, and #77189 is an open implementation PR with closing syntax that should be reviewed, merged, or closed before this issue is resolved.

## What This Changes

Request to downgrade the expected startup-sidecars WebSocket 1013 `gateway starting` close-before-connect log from WARN to debug or info.

## Best Possible Solution

Review and land or close #77189; the final change should downgrade only the known startup 1013 close while preserving WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level: force `isStartupPending()` true, send a connect frame, and the handler will close with 1013 before `client` is established, after which the close logger falls through to WARN.

## Solution Assessment

Yes. The narrow maintainable fix is to branch the pre-connect close logger on the existing startup close cause/code/reason and add a regression assertion without changing retry timing or reconnect behavior.
