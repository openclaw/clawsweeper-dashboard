## Summary

Keep open. Current main still routes the documented startup-sidecars 1013 pre-connect close through WARN, and the linked open PR should be reviewed or landed before this issue closes.

## What This Changes

Downgrade the expected gateway startup 1013 close-before-connect log from WARN to debug/info, or warn only after repeated failures beyond a grace period.

## Best Possible Solution

Land #77189 or an equivalent narrow patch that downgrades only the expected startup-sidecars 1013 pre-connect close while preserving WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level: force `isStartupPending()` true, send a connect frame, and current main sends retryable `UNAVAILABLE`, closes 1013 `gateway starting`, then logs the no-client close through WARN. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to key the pre-connect close logger off the existing startup-sidecars 1013 close cause/code/reason, add regression coverage, and leave retry timing and reconnect behavior unchanged.
