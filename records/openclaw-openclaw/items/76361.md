## Summary

Keep open. Current main still routes the expected startup 1013 close-before-connect through the warning path, and PR #77189 is an open implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue asks OpenClaw to downgrade the gateway startup-sidecars WebSocket 1013 close-before-connect log from WARN to a lower severity because it is expected retry behavior during gateway startup.

## Best Possible Solution

Review and land or close PR #77189; the desired end state is a narrow startup-close severity downgrade with regression coverage while preserving WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level: make `isStartupPending()` true, send a connect frame, and the handler closes 1013 with reason `gateway starting`; current main then routes the pre-connect close through the WARN logger. I did not run a live gateway because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to branch the pre-connect close logger on the existing startup 1013 cause, add a regression assertion, and leave retry timing, close code, readiness, and client reconnect behavior unchanged.
