## Summary

Keep open. Current main still sends the expected retryable startup-unavailable response and 1013 close, but the pre-connect close logger routes that close through `logWsControl.warn` unless it matches the unrelated SwiftPM helper exception.

## What This Changes

Downgrade or otherwise suppress WARN-level logging for expected gateway WebSocket 1013 close-before-connect events during startup.

## Best Possible Solution

Handle `startup-sidecars-pending` 1013 pre-connect closes as expected startup behavior in the server logger while preserving WARN for unexpected handshake, auth, protocol, timeout, payload, and aborted-connect failures.

## Reproduction Assessment

Yes, at source level. Force `isStartupPending()` true, send a valid connect frame, and the handler marks `startup-sidecars-pending`, sends retryable `UNAVAILABLE`, closes 1013, then the pre-connect close handler logs the close through WARN.

## Solution Assessment

Yes. The narrow maintainable solution is to key the pre-connect close logger off the existing startup close cause and add a regression assertion, without changing retry timing, close code, readiness, or client reconnect behavior.
