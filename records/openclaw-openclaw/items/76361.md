## Summary

Keep open. Current main still routes the startup-sidecars 1013 close-before-connect path through WARN, and https://github.com/openclaw/openclaw/pull/77189 is an open closing PR for this same issue; #60536 covers a separate code-1000 health-probe warning path.

## What This Changes

Downgrade expected gateway startup WebSocket close-before-connect 1013/"gateway starting" logs from WARN to debug or info while preserving warnings for unexpected closes.

## Best Possible Solution

Review and land #77189 or an equivalent narrow change that downgrades only the expected startup 1013 close while keeping WARN for unexpected pre-connect failures.

## Reproduction Assessment

Yes, at source level. Force isStartupPending() true, send a connect frame, and the handler closes 1013 with "gateway starting" before any client is registered; the close handler then falls through to WARN on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on the existing startup close cause/code/reason and add a regression assertion, without changing retry timing, close code, readiness, or client reconnect behavior.
