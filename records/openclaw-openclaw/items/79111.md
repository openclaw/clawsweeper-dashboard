Keep open: current `main` still does not explain or rule out the reported unaudited in-place Telegram `allowFrom` widening. ## What This Changes

The issue reports that Telegram `channels.telegram.accounts.<X>.allowFrom` can be silently widened in `openclaw.json` without a corresponding `config.write` audit record.

## Best Possible Solution

Instrument or live-reproduce Telegram bot-to-bot inbound handling and config writes, then fix the identified writer so pairing approvals stay in the pairing store and any intentional config mutation goes through the audited config writer.
