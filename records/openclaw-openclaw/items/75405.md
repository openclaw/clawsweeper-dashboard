## Summary

Keep open. Current main still omits runtime binding lookup for top-level Telegram groups, and open PR #75558 uses closing syntax for this regression, so this issue should wait for that or an equivalent fix to merge.

## What This Changes

Telegram native slash commands in bound top-level group chats should route to the group’s bound non-main agent instead of being silently dropped.

## Best Possible Solution

Land PR #75558, or an equivalent narrow patch, so top-level Telegram group native commands consult runtime conversation bindings while preserving current DM and forum-topic behavior.

## Reproduction Assessment

Yes. The issue provides concrete steps and downgrade evidence, and current-main source inspection shows the top-level group branch still has no runtime binding lookup path.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable solution: update the shared Telegram conversation route helper and cover bound non-topic supergroup native commands.
