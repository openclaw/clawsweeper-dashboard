## Summary

Close as implemented. Current main and the shipped v2026.4.29 release contain the native-command group visibility and Telegram target-session fixes that address the reported 2026.4.27 behavior: routed Telegram native commands now resolve the current Telegram conversation route, preserve the routed agent target session, and keep native command replies visible in group chats instead of being suppressed silently.

## What This Changes

The issue reports that Telegram native slash commands in groups bound to non-main agents were silently dropped after upgrading to v2026.4.27, while DMs still worked.

## Best Possible Solution

Keep the v2026.4.29 Telegram native-command routing and group visibility behavior as the shipped fix; users still on v2026.4.27 should upgrade rather than downgrade to restore group slash-command replies for non-main agent sessions.

## Reproduction Assessment

Yes. The issue has concrete reproduction steps and current-main coverage exercises the same command path: native Telegram group commands are authorized, routed to target sessions, and replies remain visible instead of being suppressed.

## Solution Assessment

Yes. The current implementation fixes the narrow failure point by preserving native-command target routing and making native command replies explicitly visible in group chats, without weakening group allowlists or making Telegram-specific policy changes in unrelated channels.
