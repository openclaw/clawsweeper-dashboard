## Summary

Keep open. Current main still only consults Telegram runtime conversation bindings for DMs and forum topics, while native slash commands use that shared route helper; top-level groups without a topic still have no runtime binding lookup. Open PR #75558 already targets this regression with closing syntax, so this issue should remain open until that PR or an equivalent fix lands. https://github.com/openclaw/openclaw/issues/75405

## What This Changes

Telegram native slash commands in groups bound to non-main agents should route through the group's active conversation binding instead of being silently dropped or falling back to the wrong session.

## Best Possible Solution

Review and land PR #75558, or an equivalent narrow patch, so Telegram native commands consult runtime conversation bindings for top-level group conversation ids while preserving current DM and forum-topic behavior and adding focused non-topic supergroup regression coverage.

## Reproduction Assessment

Yes. The issue provides concrete config, Telegram delivery evidence, and downgrade proof; current-main source inspection shows the missing top-level group runtime binding lookup that explains the reported silent drop path.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix in the available evidence: update the shared Telegram conversation route helper and cover bound non-topic supergroup native commands without changing unrelated command auth or channel policy behavior.
