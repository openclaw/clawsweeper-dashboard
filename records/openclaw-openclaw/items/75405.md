## Summary

Keep open. Current main still skips runtime conversation-binding lookup for top-level Telegram groups in the native command route, and the linked closing PR is still open and unmerged, so this should remain open until that patch or an equivalent fix lands.

## What This Changes

The issue reports a regression where Telegram native slash commands in bound top-level groups are received but silently dropped instead of routing to the non-main bound agent.

## Best Possible Solution

Land PR #75558 or an equivalent narrow fix so Telegram native commands consult runtime conversation bindings for top-level groups while preserving existing DM, configured binding, and forum-topic behavior; the fix should include the regression test and a user-facing changelog entry.

## Reproduction Assessment

Yes. The report gives concrete Telegram delivery, session metadata, and downgrade evidence, and current main source inspection identifies the exact route branch that skips the bound top-level group lookup.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix I found: change the shared Telegram conversation route helper to use the chat id for top-level group runtime bindings and cover that behavior with a focused native command regression test.
