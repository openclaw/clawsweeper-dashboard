## Summary

Keep open. Current main still skips runtime conversation binding lookup for Telegram top-level groups in the native command route, and the supplied GitHub context shows open PR #75558 explicitly closing this issue, so the issue should remain open until that PR merges or is replaced.

## What This Changes

The report asks to restore Telegram native slash commands in top-level groups bound to non-main agents after a 2026.4.27 regression.

## Best Possible Solution

Telegram conversation routing should consult runtime bindings for every supported Telegram conversation id: topics keep chat:topic:id, DMs keep the direct conversation id, and top-level groups use the chat id, with focused regression coverage for native /status in a bound non-topic group.

## Reproduction Assessment

Yes. The issue has a concrete Telegram reproduction and downgrade proof, and current-main source inspection shows the top-level group runtime-binding lookup is skipped; I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is the route-helper change and regression test described in #75558, preserving existing DM and topic behavior while adding the missing top-level group binding lookup.
