## Summary

Keep open. Current main still appears to skip runtime conversation bindings for top-level Telegram groups in the native command route, and the open linked PR #75558 uses closing syntax for this issue, so the issue should remain open until that fix merges or is replaced.

## What This Changes

Restore Telegram native slash commands in bound non-main top-level group conversations so `/status`, `/new`, and `/model` dispatch to the bound agent instead of being silently dropped.

## Best Possible Solution

Review and land #75558, or an equivalent narrow patch, so Telegram native commands consult runtime conversation bindings for top-level groups while preserving existing DM and forum-topic behavior; include the expected user-facing changelog entry when the regression fix lands.

## Reproduction Assessment

Yes. The report has concrete steps and downgrade evidence, and current main source inspection confirms the route helper skips runtime binding lookup for non-topic groups; I did not run live Telegram tests because this was a read-only review.

## Solution Assessment

Yes. The linked PR is the narrowest maintainable direction I found: it changes the shared Telegram conversation route helper and adds focused regression coverage for the missing top-level group case instead of changing command auth or provider behavior.
