## What This Changes

The PR replaces floor-based video base64 cap expansion with the existing quantum-rounded helper and adds focused boundary tests for the video sizing helper.

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection. Current main reads video attachments under the raw maxBytes cap, then compares ceil-rounded base64 payload size to a floor-rounded configured cap, so values like 1 or 2 bytes can reject payloads that passed the raw-byte cap.

## Solution Assessment

Yes. Reusing estimateBase64Size in the resolver makes the configured cap and the payload estimate use the same base64 quantum rounding while preserving DEFAULT_VIDEO_MAX_BASE64_BYTES.
