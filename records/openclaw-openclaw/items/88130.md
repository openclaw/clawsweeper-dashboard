The PR should stay open for a focused repair; the maintainer label also makes this human-owned rather than cleanup-closeable.

## What This Changes

This PR threads Codex harness/runtime information into embedded compaction target resolution, updates compaction runtime-context tests and hook tests, and syncs the root shrinkwrap by removing `tokenjuice`.

## Best Possible Solution

Keep the PR open, repair the two remaining Codex runtime routing gaps, rebase onto current main, and prove both direct and queued compaction paths preserve `openai-codex` runtime auth for default OpenAI/Codex OAuth installs without breaking direct OpenAI API-key and custom-base-url paths.
