## Summary

This is a useful Perplexity integration request, but it is an optional service-specific memory embedding provider that fits the existing plugin contract and ClawHub distribution path rather than OpenClaw core cleanup.

## What This Changes

Request native `memorySearch.provider = "perplexity"` support that calls Perplexity's Embeddings API, decodes base64 int8 vectors, and normalizes them for semantic memory search.

## Best Possible Solution

Publish or extend a Perplexity memory embedding plugin through ClawHub using `registerMemoryEmbeddingProvider`; revisit core only if that implementation exposes a missing plugin API.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior. Source/docs show Perplexity is not currently a memory embedding provider and the existing Perplexity plugin only exposes web search.

## Solution Assessment

No; bundling first-class core support is not the narrowest maintainable path under the current plugin/ClawHub policy. A Perplexity embedding plugin can own the provider-specific transport, base64 decoding, and normalization behind the existing memory embedding contract.
