## Summary

This is a useful provider request, but current OpenClaw already exposes a plugin-owned memory embedding provider contract, and the project scope points optional provider integrations toward plugins and ClawHub rather than core bundling by default.

## What This Changes

Add first-class Perplexity embeddings support for `agents.defaults.memorySearch.provider = "perplexity"`, including API-key resolution, base64 int8 decoding, normalization, and model dimensions.

## Best Possible Solution

Publish a Perplexity memory-embedding provider plugin through ClawHub using the existing `memoryEmbeddingProviders` SDK seam; consider bundled promotion only with explicit maintainer product and security ownership.

## Reproduction Assessment

Not applicable: this is a feature request for a new provider integration, not a report of broken existing behavior. Source inspection confirms current memory search supports plugin-provided embedding adapters but does not have a Perplexity embedding adapter today.

## Solution Assessment

Yes: the narrow maintainable path is a provider plugin, because Perplexity decoding, auth, model defaults, and dimensions are provider-owned behavior and the SDK already has a memory embedding provider seam.
