## Summary

Keep open: the PR addresses a real Voice Call gap and includes live proof, but the patch introduces a source-confirmed transcript duplication path for Telnyx provider speech.

## What This Changes

The PR adds an optional private `objective` to Voice Call CLI, RPC, and tool initiation paths, stores it on outbound call metadata, injects it into realtime instructions, returns completed status from history, and records bot speaking events in transcripts.

## Best Possible Solution

Keep the private objective/history improvements, but scope or dedupe bot-speech transcript recording so realtime bridge output is captured without duplicating provider TTS events.

## Reproduction Assessment

Yes, from source inspection: current main lacks a per-call objective field, active-only status misses completed history, and `call.speaking` currently does not persist realtime bot speech. The PR's new duplicate-transcript bug is also source-reproducible through Telnyx `call.speak.started` plus `speak()`'s existing transcript append.

## Solution Assessment

No, not as-is. Splitting a private objective from the spoken opener fits the current realtime instruction seam, but the transcript fix should be scoped or deduped before merge.
