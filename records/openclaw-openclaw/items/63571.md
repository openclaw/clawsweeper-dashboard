## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load fallback provider owners before candidate lookup:** `src/web-search/runtime.ts:353`
  - body: Current main scopes explicit web-search provider loading to the selected provider owner's plugin. This lookup searches only the already-loaded `candidates`, so a fallback owned by another plugin can be absent and the configured chain will never try it after rebase.
  - confidence: 0.89
- **[P2] Preserve resolved config and abort signal in fallback execution:** `src/web-search/runtime.ts:375-387`
  - body: Current main resolves SecretRefs through the runtime config snapshot and passes `params.signal` to provider execution. This path uses `params.config` and calls `execute` without the signal, so fallback runs can see unresolved SecretRefs and ignore cancellation.
  - confidence: 0.9
- **[P2] Validate fallback provider ids against manifest metadata:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: The new field accepts arbitrary strings and the plugin-aware validator only checks `tools.web.search.provider`. Typoed or unavailable fallback IDs will pass validation and then be skipped or misreported instead of using the existing provider-id contract.
  - confidence: 0.87
