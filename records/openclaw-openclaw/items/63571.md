## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load fallback provider owners before candidate lookup:** `src/web-search/runtime.ts:353`
  - body: Current main scopes explicit web-search provider loading to the configured provider's owner plugin. This patch resolves fallback IDs only inside already-loaded `candidates`, so after rebase a fallback owned by another plugin can be absent and the configured chain will never try it.
  - confidence: 0.89
- **[P2] Keep resolved config and abort signal in fallback path:** `src/web-search/runtime.ts:375-387`
  - body: Current main selects the applicable runtime config before tool creation and forwards `params.signal` to execution. The new path uses `params.config` and calls `execute` without the signal, so fallback runs can miss resolved SecretRefs and ignore cancellation.
  - confidence: 0.9
- **[P2] Validate fallback provider ids against plugin metadata:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: The new `fallbacks` field accepts arbitrary strings, unlike the current provider validation path. Typos or unavailable plugin providers will be accepted and skipped or misreported instead of failing or warning through the existing provider-id contract.
  - confidence: 0.87
