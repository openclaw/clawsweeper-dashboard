## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load fallback provider owners before resolving candidates:** `src/web-search/runtime.ts:353`
  - body: Current main scopes explicit web_search provider loading to the selected provider owner before candidates are resolved. This PR looks up fallback ids only inside the already-loaded `candidates`, so a fallback owned by another plugin can be absent and never tried after rebasing onto current main.
  - confidence: 0.89
- **[P2] Preserve resolved config and abort signals:** `src/web-search/runtime.ts:375-387`
  - body: Current main resolves the applicable runtime config snapshot and forwards `params.signal` to provider execution. The new fallback path passes `params.config` to `createTool` and calls `execute(params.args)` without the signal, which can bypass resolved SecretRefs and ignore cancellation.
  - confidence: 0.9
- **[P2] Validate fallback provider ids with plugin metadata:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: Current main validates `tools.web.search.provider` against manifest-owned provider ids and stale-plugin evidence. Adding `fallbacks` as a plain string array accepts typoed or unavailable plugin providers, which makes fallback chains silently skip or misdiagnose configured providers.
  - confidence: 0.87
