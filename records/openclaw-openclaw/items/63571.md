## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Load fallback provider owners before resolving candidates:** `src/web-search/runtime.ts:353-356`
  - body: The new chain resolves fallback IDs only by searching `candidates`. On current main, provider loading can be scoped to the explicit primary provider owner, so a cross-plugin chain such as Brave with Perplexity fallback can silently omit the fallback unless fallback owner plugins are loaded too.
  - confidence: 0.86
- **[P2] Validate fallback ids instead of silently dropping them:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: `fallbacks` accepts arbitrary strings while the runtime drops entries that do not resolve. A typo can make failover look configured when no fallback will run, so this needs registry-aware validation or a clear config error path like `tools.web.search.provider`.
  - confidence: 0.85
- **[P2] Narrow retries to recoverable failures:** `src/web-search/runtime.ts:368-369`
  - body: `allowFallback` makes explicit fallback mode continue after provider failures, but the canonical request excludes auth, permission, and deterministic parameter errors. Without a recoverable-error taxonomy, the same query can be sent to extra providers after failures that should stay fail-fast.
  - confidence: 0.78
