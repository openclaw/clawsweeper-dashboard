  - body: As written in the PR head, the retry loop calls `createTool` without `agentDir` and executes without `{ signal }`.   - confidence: 0.9
- **[P2] Classify failures before retrying other providers:** `src/web-search/runtime.ts:405-410`
  - body: This catch path retries every thrown provider failure in explicit fallback mode. That can send the same query to a different provider for auth, permission, or deterministic parameter errors, while the canonical request limits fallback to recoverable quota, timeout, network, or 5xx failures.
  - confidence: 0.87
- **[P2] Reject unresolved fallback provider ids:** `src/web-search/runtime.ts:352-357`
  - body: Fallback ids that are not present in `candidates` are silently dropped by this map/filter chain. A typo or unloaded provider can make failover look configured while no fallback will run, so the runtime should surface a clear config or diagnostic error instead of omitting it.
  - confidence: 0.84
