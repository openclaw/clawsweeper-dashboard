  - body: The fallback retry loop creates tools with raw `params.config`, omits `agentDir`, and executes without `{ signal }`.   - confidence: 0.93

  - body: Adding `fallbacks` as a plain string array lets typoed provider IDs pass config validation and then get silently skipped by runtime matching. Please validate the list against the web-search provider registry contract or report a clear config issue so users do not think failover is active when it is not.
  - confidence: 0.87
- **[P2] Classify retryable errors before falling back:** `src/web-search/runtime.ts:405-410`
  - body: In explicit fallback mode, the new loop treats every thrown provider error as a reason to try the next provider. That can forward the same query after authentication, permission, or deterministic parameter failures instead of surfacing the actionable error, so the PR needs a recoverable-error contract before cross-provider retries are enabled.
  - confidence: 0.82
