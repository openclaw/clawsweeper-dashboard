## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load declared fallback provider scopes before chaining:** `src/web-search/runtime.ts:353`
  - body: On current main, explicit web-search provider loading is scoped to the owner plugin of the configured provider. This branch resolves fallback ids only from the already-loaded `candidates`, so a chain like `provider: "brave", fallbacks: ["duckduckgo"]` can silently drop DuckDuckGo instead of trying it.
  - confidence: 0.88
- **[P2] Preserve resolved config and abort signals:** `src/web-search/runtime.ts:376-387`
  - body: Current main resolves the runtime config before provider creation and forwards `{ signal: params.signal }` into execution. This branch uses `params.config` and calls `execute(params.args)`, so SecretRef-hydrated config can be missed and cancellation no longer reaches fallback providers.
  - confidence: 0.9
- **[P2] Validate fallback provider ids with manifest metadata:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: Current main validates `tools.web.search.provider` against manifest-owned web-search provider ids. Adding `fallbacks` as a plain string array without extending that validation creates a second provider-routing field where typos or stale plugin ids are accepted silently.
  - confidence: 0.82
