Overall correctness: patch is incorrect

Full review comments:

- **[P2] Classify fallback failures before retrying:** `src/web-search/runtime.ts:405-410`
  - body: In explicit fallback mode, this catch path retries every thrown provider failure. That can retry auth, permission, or deterministic parameter errors on a different provider and send the same query across another trust/cost boundary, while https://github.com/openclaw/openclaw/issues/80843 limits fallback to recoverable quota, timeout, network, or 5xx failures.
  - confidence: 0.87
- **[P2] Reject unresolved fallback provider ids:** `src/web-search/runtime.ts:352-357`
  - body: Fallback ids that are not present in `candidates` are silently dropped by this map/filter chain. A typo or unloaded provider can make failover look configured while no fallback will run, so the runtime should surface a clear config or diagnostic error instead of omitting it.
  - confidence: 0.85
- **[P2] Load fallback provider owners before matching:** `src/web-search/runtime.ts:314-318`
  - body: The candidate set is resolved before the fallback ids are considered, and explicit provider loading is scoped to the primary provider owner. Plugin-owned fallback providers that are not already in `candidates` will never be loaded, so the configured fallback chain can silently skip valid installed providers.
  - confidence: 0.82
