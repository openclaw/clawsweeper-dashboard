## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Load fallback provider owners before matching:** `src/web-search/runtime.ts:353`
  - body: `effectiveCandidates` only looks for fallback IDs inside `candidates`, but current provider resolution can scope `candidates` to the configured primary provider's owner plugin. A config such as primary Brave with fallback Tavily can silently skip Tavily because its plugin was never loaded, so the fallback chain does not honor the declared list.
  - confidence: 0.88
- **[P1] Preserve resolved config and abort signals:** `src/web-search/runtime.ts:376-387`
  - body: This loop passes `params.config` into `createTool` and calls `execute(params.args)` without the current-main resolved runtime config or `{ signal: params.signal }`. That can make SecretRef-resolved credentials unavailable to providers and leave cancelled searches running after callers abort.
  - confidence: 0.86
- **[P2] Validate fallback ids against provider manifests:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: The new public `fallbacks` field accepts any string array, but current main already validates the primary provider against manifest-backed web-search provider IDs. Typos in `fallbacks` are then skipped at runtime instead of producing a clear config error, creating a second provider source of truth.
  - confidence: 0.82
