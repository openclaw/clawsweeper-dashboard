## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Classify fallback failures before retrying:** `src/web-search/runtime.ts:405-410`
  - body: In explicit fallback mode, this catch path retries every thrown provider failure. That can retry auth, permission, or deterministic parameter errors on a different provider and send the same query across another trust/cost boundary, while the canonical request limits fallback to recoverable quota, timeout, network, or 5xx failures.
  - confidence: 0.86
- **[P2] Validate fallback provider ids:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: `fallbacks` accepts arbitrary strings while the runtime drops entries that do not resolve. A typo can make failover look configured when no fallback will run, so this needs registry-aware validation or an explicit config error path instead of silent omission.
  - confidence: 0.86
- **[P2] Load fallback provider owners before resolving candidates:** `src/web-search/runtime.ts:352-353`
  - body: The new chain only looks up fallback ids inside the already-loaded candidate list. On current main, explicit provider loading can be scoped to the selected provider owner, so a cross-plugin config such as Brave with Perplexity fallback can omit the fallback unless fallback owner plugins are loaded too.
  - confidence: 0.78
