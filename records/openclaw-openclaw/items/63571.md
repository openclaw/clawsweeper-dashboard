Full review comments:

- **[P2] Load fallback provider plugins before resolving the chain:** `src/web-search/runtime.ts:353`
  - body: The new fallback chain is built from `candidates`, but current runtime loading scopes candidates to the configured primary provider's owner plugin for explicit selections. A real cross-plugin config such as `provider: "brave"` with `fallbacks: ["perplexity"]` can load only Brave, so the fallback never resolves; expand the load scope to include fallback provider owners and test a cross-plugin chain.
  - confidence: 0.87
- **[P2] Validate fallback provider IDs before accepting config:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: `fallbacks` is accepted as arbitrary strings while the runtime silently drops entries that do not resolve. A typo can make failover appear configured when no fallback will run, so fallback IDs need the same registry-aware validation or explicit config error path as `tools.web.search.provider`.
  - confidence: 0.86
