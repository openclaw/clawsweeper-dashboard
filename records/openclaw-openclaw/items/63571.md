## Review Findings

Full review comments:

- **[P1] Preserve runtime context when retrying providers:** `src/web-search/runtime.ts:375-387`
  - body: The retry loop creates tools with raw `params.config`, no `agentDir`, and executes without `{ signal }`. Current main resolves the applicable runtime config, forwards active-agent auth context/runtime metadata, and passes the abort signal, so this can regress SecretRef config, auth profiles, and cancellation during fallback retries.
  - confidence: 0.93
- **[P2] Validate fallback provider ids before accepting config:** `src/config/zod-schema.agent-runtime.ts:303`
  - body: Adding `fallbacks` as a plain string array lets typoed provider IDs pass config validation and then get silently skipped by runtime matching. Please validate the list against the same provider registry contract used for `tools.web.search.provider`, or report a clear config issue so users do not think failover is active when it is not.
  - confidence: 0.87
- **[P2] Classify retryable errors before falling back:** `src/web-search/runtime.ts:405-410`
  - body: In explicit fallback mode, the new loop treats every thrown provider error as a reason to try the next provider. That can forward the same query after authentication, permission, or deterministic parameter failures instead of surfacing the actionable error, so the PR needs a recoverable-error contract before cross-provider retries are enabled.
  - confidence: 0.82
