## Summary

Keep open. Current main still exposes a single Control UI assistant-message action labeled “Copy as markdown” that writes the Markdown string unchanged, and the related native-app copy-button PR does not add the requested Markdown/plain-text choice.

## What This Changes

The issue requests a user-facing option to copy assistant message content as plain text instead of only Markdown.

## Best Possible Solution

Add a focused Control UI copy-format affordance around the existing assistant copy action, define the Markdown-to-plain-text conversion contract, and cover both clipboard outputs with targeted UI tests.

## Reproduction Assessment

Yes. Source inspection of current main shows the Control UI assistant action is still only `renderCopyAsMarkdownButton(markdown!)`, with no plain-text label, selector, preference, or helper wiring in that path.

## Solution Assessment

Yes, keeping this open is the right outcome. The existing implementation is not enough, and the narrow maintainable path is a Control UI feature decision plus focused tests rather than treating the related native copy-button PR as a fix.
