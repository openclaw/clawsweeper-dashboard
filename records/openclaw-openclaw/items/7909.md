## Summary

Keep open. Current main still only exposes the assistant-message copy action as Markdown, and the Control UI path has no selector, preference, or tested plain-text clipboard behavior for chat bubbles.

## What This Changes

The issue requests a Control UI option to copy assistant message content as plain text in addition to the current Markdown clipboard behavior.

## Best Possible Solution

Add a compact Control UI copy-format affordance for assistant messages, keep Markdown as the current/default behavior, implement a deterministic plain-text conversion with clear list/code/link/table behavior, and cover both clipboard outputs with focused UI tests.

## Reproduction Assessment

Yes. In current main, any assistant message with Markdown renders a single `.chat-copy-btn` labeled `Copy as markdown`, and clicking it writes the original markdown string to the clipboard with no alternate plain-text action.

## Solution Assessment

Yes. A narrow Control UI change around the existing chat copy helper and grouped-render action surface is the best fit; this does not need a core architecture change or a plugin boundary decision.
