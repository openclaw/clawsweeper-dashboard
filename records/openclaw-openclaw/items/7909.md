## Summary

Keep open. Current main still exposes only the Control UI assistant-message Markdown copy action; there is no Markdown/plain-text selector, preference, or tested plain-text clipboard path, and the related Android copy items cover a different surface.

## What This Changes

The issue requests a Control UI option to copy assistant message content as plain text instead of only Markdown.

## Best Possible Solution

Add a compact assistant-message copy-format affordance in Control UI, keep Markdown as the default behavior, define deterministic plain-text conversion semantics, and cover both clipboard outputs with focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. The missing capability is source-verifiable on current main: assistant messages render one Markdown copy action and no plain-text option.

## Solution Assessment

Yes. A narrow Control UI change around the existing chat copy helper and grouped-render action surface is the best fit; a global preference should wait unless maintainers explicitly want persisted copy-format behavior.
