## Summary

Keep open: current main and v2026.6.8 still only expose the Control UI assistant-message copy action as Markdown, and related native/Android copy items do not implement the requested Markdown/plain-text format choice.

## What This Changes

The issue requests a Control UI option to copy assistant message content as plain text instead of only Markdown.

## Best Possible Solution

Keep Markdown copy as the default and add a compact secondary plain-text action with a narrowly defined, tested conversion contract; avoid a persisted preference unless product review chooses it.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests an additional clipboard format rather than reporting broken existing behavior. Source inspection does give high confidence that current main has only the Markdown copy action and no plain-text selector.

## Solution Assessment

Unclear. The existing copy helper and grouped-render caller are the likely implementation boundary, but maintainers still need to approve the interaction model and define plain-text conversion semantics.
