## Summary

Keep open. Current main still exposes only the Markdown assistant-message copy action in Control UI, with no Markdown/plain-text selector, preference, or tested plain-text clipboard behavior.

## What This Changes

The issue asks for a Control UI option to copy assistant message content as plain text in addition to the current Markdown copy behavior.

## Best Possible Solution

Add a compact Control UI copy-format affordance for assistant messages, keep Markdown as the default/current behavior, define deterministic plain-text conversion semantics, and cover both clipboard outputs with focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. The missing behavior is still source-verifiable on current main: assistant messages render one Markdown copy action and no plain-text option.

## Solution Assessment

Yes. A narrow Control UI change around the existing chat copy helper and grouped-render action surface is the best fit; this does not need a core architecture change or plugin boundary decision.
