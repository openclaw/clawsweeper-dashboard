## Summary

Keep open, but do not route to an automatic fix yet. Current main still has only generic remote-CDP handling, but the live evidence is mixed because Browserbase's current docs and a safe endpoint probe still show the static connect endpoint exists, while Browserbase also documents explicit session creation.

## What This Changes

The issue asks OpenClaw to handle Browserbase session creation and release instead of relying on the documented static Browserbase `cdpUrl`.

## Best Possible Solution

First confirm the Browserbase contract for OpenClaw's use case; if static connect is no longer reliable, add Browserbase-owned session lifecycle support in the browser plugin and update docs/tests without changing generic remote-CDP behavior.
