## Summary

Keep open: current main still has only the generic CDP URL attach path and still documents the static Browserbase URL, while current Browserbase docs emphasize creating a session and connecting to `connectUrl`. The reported 404 is not live-reproduced and a related report says the static URL may still work, so this needs browser/product owner judgment rather than cleanup close.

## What This Changes

Request that OpenClaw replace or augment the documented static Browserbase `cdpUrl` flow with Browserbase Sessions API creation, `connectUrl` attachment, and session cleanup.

## Best Possible Solution

Define the Browserbase browser-profile contract, then either implement explicit session creation/cleanup with redaction-safe credentials or change OpenClaw docs to require users to provide a pre-created `connectUrl`.

## Reproduction Assessment

No high-confidence live reproduction of the 404 exists because the review did not have a Browserbase key. Source inspection and Browserbase docs do show OpenClaw lacks the Sessions API flow, but a related user report says the static URL can still work with the documented format.

## Solution Assessment

Unclear. The proposed `browserbase.com` URL detection is brittle; an explicit provider/profile session contract or docs-only `connectUrl` requirement would be more maintainable.
