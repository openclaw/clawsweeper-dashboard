## Summary

Keep open: current main and the latest release still document and preserve the static Browserbase `cdpUrl` flow, while current Browserbase primary API/Playwright docs use Sessions API creation followed by `connectUrl`. The exact 404 is not live-reproduced and Browserbase docs/examples are mixed, so this needs browser/product owner judgment rather than cleanup close.

## What This Changes

The issue asks OpenClaw to replace or supplement the documented static Browserbase `cdpUrl` flow with Browserbase Sessions API creation, `connectUrl` use, and session cleanup.

## Best Possible Solution

Define the Browserbase browser-profile contract, then either document pre-created `connectUrl` usage or implement explicit session creation/cleanup with redaction-safe credentials and tests.

## Reproduction Assessment

No high-confidence live reproduction of the 404 exists because this review did not use a Browserbase key. Source inspection does show OpenClaw lacks the Sessions API flow and still uses the configured static CDP URL directly.

## Solution Assessment

Unclear. The proposed `browserbase.com` URL detection is brittle; an explicit provider/profile session contract or a docs-only `connectUrl` requirement would be more maintainable.
