## Summary

Keep open. Current main still documents the obsolete static Browserbase CDP URL and still routes it through generic CDP discovery/direct attach, while Browserbase's current API requires creating a session first and connecting with the returned `connectUrl`.

## What This Changes

The issue asks OpenClaw to restore the documented Browserbase browser profile by creating Browserbase sessions through the Sessions API before CDP attach.

## Best Possible Solution

Add provider-owned Browserbase session handling inside the browser plugin: create a Browserbase session before CDP attach, connect to the returned `connectUrl`, clean up only sessions OpenClaw created, and update docs/tests/changelog while preserving generic remote CDP behavior.

## Reproduction Assessment

Yes, at source level. Current main still resolves the documented Browserbase profile to the static connect URL, that URL returns HTTP 404 in a safe probe, and the runtime has no Sessions API path before `connectOverCDP()`; I did not run a live Browserbase-key OpenClaw command.

## Solution Assessment

Yes. The narrow maintainable fix is explicit Browserbase session support in the browser plugin, not a substring-only fallback in shared CDP helpers or a core migration of generic remote-CDP behavior.

Fix https://github.com/openclaw/openclaw/issues/43455. Current docs advertise Browserbase with `cdpUrl: "wss://connect.browserbase.com?apiKey=<BROWSERBASE_API_KEY>"`, but Browserbase now requires creating a session via `POST https://api.browserbase.com/v1/sessions` with `X-BB-API-Key` and connecting to the returned `connectUrl`. Implement this inside the bundled browser plugin, not core: add typed Browserbase session resolution before CDP attach, parse the API key from the existing documented URL or an existing env-style secret path if already supported, use `projectId` only if needed because Browserbase documents it as optional, and clean up only sessions OpenClaw created. Preserve generic Browserless/bare WebSocket CDP behavior and SSRF checks. Add focused unit tests for session creation, connectUrl use, error/redaction handling, cache/release cleanup, and no-regression for generic bare `ws`/`wss` URLs. Update `docs/tools/browser.md` and `CHANGELOG.md`; do not add unrelated config knobs unless the existing config contract cannot express the Browserbase API data.
