## Summary

Keep open. Current main still documents the obsolete static Browserbase CDP URL and still routes that URL through generic CDP discovery/direct WebSocket attach, with no Browserbase Sessions API lifecycle before connecting.

## What This Changes

The issue requests Browserbase session lifecycle support because the documented static Browserbase CDP URL now returns 404.

## Best Possible Solution

Add Browserbase-owned session handling in the browser plugin: create a session before CDP attach, connect to the returned `connectUrl`, release only sessions OpenClaw created, and update docs/tests/changelog while preserving generic remote CDP behavior.

## Reproduction Assessment

Yes, at source level. Current main still resolves the documented Browserbase profile to the static connect URL, the static endpoint returns 404 in a safe probe, and the runtime has no Sessions API path before `connectOverCDP()`.

## Solution Assessment

Yes. The narrow maintainable fix is explicit Browserbase session lifecycle support in the browser plugin, not a generic core fallback or substring-only hack in shared CDP helpers.
