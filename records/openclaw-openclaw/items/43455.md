## Summary

Keep open: current main and the latest release still document and implement Browserbase as a static `cdpUrl`, while Browserbase's current Sessions API exposes `POST /v1/sessions` and returns a `connectUrl`; the related Browserbase driver PR is open and conflicting, not a merged fix.

## What This Changes

The issue requests Browserbase-aware browser profile connection handling because the documented static Browserbase CDP URL now fails and Browserbase requires session API connect URLs.

## Best Possible Solution

Land a maintainer-approved Browserbase session lifecycle path in the browser plugin or a generic remote-CDP session seam, with explicit credential handling, updated docs/schema/tests, and redacted end-to-end Browserbase proof.
