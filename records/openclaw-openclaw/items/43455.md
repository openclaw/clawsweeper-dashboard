## Summary

Keep open. Current main still documents Browserbase as a static bare WebSocket CDP profile and still routes that URL through generic CDP attach, while Browserbase’s current docs describe creating a session through the Sessions API and using the returned `connectUrl`. ([docs.browserbase.com](https://docs.browserbase.com/reference/api/create-a-session?utm_source=openai))

## What This Changes

The issue asks OpenClaw to manage Browserbase session creation and connection URLs instead of relying on the documented static Browserbase CDP WebSocket URL.

## Best Possible Solution

Add Browserbase-owned session handling in the browser plugin: create a Browserbase session before attach, connect to the returned `connectUrl`, clean up only OpenClaw-created sessions according to Browserbase lifecycle rules, and update docs/tests/changelog while preserving generic remote CDP behavior.
