## Summary

Keep open: current main still documents a static Browserbase `cdpUrl`, while Browserbase's current docs describe creating a session and connecting to the returned `connectUrl`; OpenClaw has only the generic CDP URL path and no Browserbase session lifecycle contract.

## What This Changes

Request automatic Browserbase session lifecycle support because the documented static Browserbase `cdpUrl` reportedly fails and now needs a Sessions API `connectUrl`.

## Best Possible Solution

Define the browser-profile contract for Browserbase Sessions API, then either implement explicit session creation/cleanup with redaction-safe credentials or update the docs to require users to supply a pre-created `connectUrl`.

## Reproduction Assessment

No high-confidence live repro of the reported 404 exists without a valid Browserbase key. Source and upstream-doc inspection do show current main lacks the Sessions API flow that Browserbase now documents for managed browser sessions.

## Solution Assessment

Unclear. The requested `browserbase.com` URL detection is brittle; a maintainable fix needs an explicit profile/provider session contract with credential redaction and release semantics.
