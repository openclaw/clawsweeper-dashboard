## What This Changes

The issue reports that the documented static Browserbase CDP URL no longer works and asks OpenClaw to create and manage Browserbase sessions through the Browserbase Sessions API.

## Best Possible Solution

Keep the issue open until maintainers either land a reviewed Browserbase session-driver or plugin/API path, or replace the stale Browserbase static-CDP docs with a supported alternative.

## Reproduction Assessment

Yes from source, but not via a live Browserbase account in this review. Current docs still prescribe the static Browserbase `cdpUrl`, and current runtime resolves it as a plain remote CDP URL without Browserbase session API creation.

## Solution Assessment

Unclear. A Browserbase-specific driver is a plausible fix, but the best solution must settle whether this belongs in bundled browser code, a plugin/backend seam, or docs, and must cover credential/redaction/session lifecycle safety.
