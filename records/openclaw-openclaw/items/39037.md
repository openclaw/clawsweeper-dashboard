Keep open: current main and the latest shipped release still do not extract or expose WhatsApp CTWA ad referral metadata, and the related forwarding-metadata work does not cover ad click attribution fields. ## What This Changes

Requests that the WhatsApp plugin extract Click-to-WhatsApp ad referral fields, especially `ctwaClid`, from inbound Baileys context and expose them through session or prompt metadata.

## Best Possible Solution

Add a WhatsApp-owned, typed, privacy-aware CTWA referral metadata contract that extracts only approved Baileys ad fields and exposes them through the maintainer-approved prompt, hook, or session surface with focused tests.

## Solution Assessment

Unclear as written. The direction is valid, but the best implementation is a narrow typed metadata contract with maintainer-approved privacy and exposure semantics instead of copying raw referral objects broadly.
