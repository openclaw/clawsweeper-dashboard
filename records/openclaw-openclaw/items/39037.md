## Summary

Keep open. Current main still has the same WhatsApp CTWA metadata gap: raw Baileys `contextInfo` is available, but no CTWA-specific reader or forwarding path puts `externalAdReply.ctwaClid` into the existing untrusted structured-context carrier for agents.

## What This Changes

The issue requests exposing Click-to-WhatsApp `ctwaClid` referral data from inbound WhatsApp context into session or prompt metadata.

## Best Possible Solution

Add a WhatsApp-owned CTWA referral extractor and append sanitized fields to `untrustedStructuredContext`, while deferring durable session storage to a separate session metadata design if maintainers want it later.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, and current main has no reader or carrier assignment for that CTWA payload.

## Solution Assessment

Yes. The narrow maintainable solution is additive extraction inside the WhatsApp plugin into `untrustedStructuredContext`, not a new core session metadata schema or plugin API in the first pass.
