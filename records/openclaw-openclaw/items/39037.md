## Summary

Keep open: current main and the latest shipped tag still do not extract or expose WhatsApp CTWA click/ad referral metadata. The related WhatsApp metadata threads cover GIF media metadata and forwarding signals, but not the ad-attribution `ctwaClid` use case, and the approved prompt, hook, or durable session metadata surface still needs maintainer product/privacy judgment.

## What This Changes

The issue requests exposing Click-to-WhatsApp referral click metadata from inbound WhatsApp messages to session or agent-accessible metadata.

## Best Possible Solution

Add a WhatsApp-owned, typed, privacy-aware CTWA referral metadata contract that extracts only approved Baileys ad fields and exposes them through the maintainer-approved prompt, hook, or session surface with focused tests.

## Reproduction Assessment

Yes, source-level. A synthetic inbound Baileys message carrying `contextInfo.externalAdReply.ctwaClid` would reach raw `contextInfo`, but current main has no normalizer or metadata carrier that copies those fields into queued messages, prompt context, hooks, or session metadata.

## Solution Assessment

Unclear as proposed. The direction is valid, but the maintainable fix is a narrow typed metadata contract with maintainer-approved privacy and exposure semantics rather than broadly copying raw ad referral objects.
