## Summary

Keep open. Current main still extracts raw WhatsApp Baileys `contextInfo` but has no CTWA-specific reader or forwarding path into the existing untrusted structured-context carrier, and the related forwarding metadata issue is adjacent rather than a duplicate.

## What This Changes

Expose Click-to-WhatsApp `ctwaClid` and related ad referral fields from inbound WhatsApp messages to agent-accessible metadata.

## Best Possible Solution

Add a WhatsApp-owned CTWA referral extractor and append sanitized fields to the existing untrusted structured-context carrier, while deferring durable session storage until a broader session metadata schema is designed.

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, and current main has no reader or carrier assignment for that CTWA payload.

## Solution Assessment

Yes. The narrow maintainable solution is additive extraction inside the WhatsApp plugin into `untrustedStructuredContext`, not a new core session metadata schema or plugin hook API in the first pass.
