## Summary

Keep open. Current main still has no WhatsApp CTWA extraction or propagation, while the pinned Baileys contract exposes `contextInfo.externalAdReply.ctwaClid` and the existing untrusted structured context path can carry the data to prompts.

## What This Changes

Expose Click-to-WhatsApp ad referral metadata, especially `ctwaClid`, from inbound WhatsApp messages to agent/session metadata.

## Best Possible Solution

Add a WhatsApp-owned normalizer for a bounded CTWA payload from `contextInfo.externalAdReply` into the existing untrusted structured metadata path, with focused tests and no durable persistence in the first pass.

Yes. A high-confidence source-level reproduction is to trace a Baileys-shaped inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid`; current main can read `contextInfo` but never carries that payload beyond extraction.

## Solution Assessment

Yes. The narrow maintainable solution is inside the WhatsApp plugin using `UntrustedStructuredContext`; adding a new core session schema or durable storage first would be broader than the attribution gap.
