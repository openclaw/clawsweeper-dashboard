## Summary

Keep open. Current main still drops WhatsApp Click-to-WhatsApp referral metadata: Baileys exposes `contextInfo.externalAdReply.ctwaClid`, OpenClaw can reach raw `contextInfo`, but the WhatsApp inbound normalization only forwards contact-card structured context and has no CTWA extractor/carrier.

## What This Changes

The issue requests extracting Click-to-WhatsApp ad referral fields, especially `ctwaClid`, from WhatsApp inbound context and exposing them to agents/session metadata.

## Best Possible Solution

Add a WhatsApp-owned CTWA referral extractor and append sanitized fields to `untrustedStructuredContext`, deferring durable session storage unless maintainers explicitly choose a broader session metadata design.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, and current main has no reader or carrier assignment for that CTWA payload.

## Solution Assessment

Yes, as a narrow feature direction. The maintainable fix is additive extraction inside the WhatsApp plugin into the existing untrusted structured-context carrier, not a new core session metadata schema in the first pass.
