## Summary

Current main and v2026.6.6 still omit CTWA referral data from the WhatsApp inbound and plugin-hook contracts, and no canonical implementation owns the work. The use case is valid, but maintainers must choose a typed access and retention boundary for advertising identifiers before implementation.

## What This Changes

Expose Click-to-WhatsApp referral identifiers from inbound WhatsApp messages to agent or session consumers.

## Best Possible Solution

Capture a validated minimal CTWA referral record in the WhatsApp adapter and expose it as typed data to authorized inbound hooks; keep prompt interpolation and session persistence off by default unless maintainers explicitly approve their access and retention policy.

## Reproduction Assessment

Yes at source level: current WhatsApp inbound and canonical hook types have no CTWA/referral field, so the value cannot reach plugins or session consumers through those paths. A live Click-to-WhatsApp message was not exercised.

## Solution Assessment

No, the broad proposal to place all referral fields in the preamble and optionally persist them is not the safest fix. A minimal typed, transient hook field is the narrower maintainable solution, with persistence considered separately.
