## Summary

Keep open. Current main has generic structured metadata plumbing, but it still never normalizes Baileys `contextInfo.externalAdReply.ctwaClid` or CTWA source fields into WhatsApp inbound metadata.

## What This Changes

The issue requests extraction of WhatsApp Click-to-WhatsApp ad referral metadata, especially `ctwa_clid`, from inbound message context so agents can use it for attribution workflows.

## Best Possible Solution

Add a WhatsApp-owned CTWA normalizer that forwards a minimal `externalAdReply` attribution payload through the existing untrusted structured metadata path, without adding durable session persistence in the first pass.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence synthetic reproduction: provide a Baileys message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid`; `extractContextInfo` can reach it, but the current enrichment and queued-message path never forwards it.

## Solution Assessment

Yes. The narrow maintainable fix is inside the WhatsApp plugin using `untrustedStructuredContext`; adding a core session schema, config knob, or persistence layer first would be broader than the reported attribution gap.
