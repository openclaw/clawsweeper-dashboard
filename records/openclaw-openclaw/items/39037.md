## Best Possible Solution

Add a WhatsApp-owned typed CTWA extractor for approved externalAdReply fields and expose it through the maintainer-approved untrusted context, hook, or session surface, with persistent storage only if explicitly approved.

Unclear as proposed. The direction is valid, but broadly copying referral data into session metadata or the preamble is not the narrowest maintainable fix until maintainers choose a typed, privacy-aware exposure surface.
