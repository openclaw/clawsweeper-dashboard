## Summary

Current main and the latest release still do not expose CTWA referral data from WhatsApp inbound messages into the session or plugin hook contracts. The request remains valid, but it needs maintainer product/security direction before implementation because `ctwaClid` and referral strings are advertising identifiers that may reach prompts, plugins, transcripts, or persistent session state.

## What This Changes

Expose Click-to-WhatsApp referral data, especially `ctwa_clid`/`ctwaClid`, from WhatsApp inbound messages into session or plugin-accessible metadata.

## Best Possible Solution

Expose a validated minimal CTWA referral record through a typed, authorized WhatsApp/plugin hook path and keep prompt injection or durable session persistence off by default unless maintainers explicitly approve it.

## Reproduction Assessment

Yes at source level: current WhatsApp code extracts `contextInfo` but only carries selected quote/mention/location/context fields onward, and the hook mappers have no CTWA/referral field. A live Click-to-WhatsApp webhook was not exercised.

## Solution Assessment

No for the broad preamble/session-persistence proposal; the narrower maintainable fix is typed, minimal plugin-visible CTWA referral data with explicit security and retention policy.
