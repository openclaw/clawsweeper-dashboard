## Summary

Keep open. Current main and v2026.6.5 still drop CTWA/referral metadata after extracting WhatsApp `contextInfo`, and related contextInfo-hook work is still open rather than a canonical fix.

## What This Changes

The issue asks OpenClaw’s WhatsApp channel to expose Click-to-WhatsApp referral data, especially `ctwa_clid`, to session metadata or agent-visible context.

## Best Possible Solution

Adopt a narrow WhatsApp-owned metadata exposure contract, likely sanitized untrusted context plus explicit hook metadata if approved; avoid broad durable session persistence unless maintainers decide attribution IDs belong there.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main extracts WhatsApp `contextInfo` but has no path that maps CTWA/referral fields into WhatsApp inbound context, plugin hook metadata, or session metadata. I did not run a live CTWA webhook because no real ad-click sample was available.

## Solution Assessment

Unclear. The narrow technical path is plausible, but maintainers should first choose whether CTWA referral data should be prompt-visible, plugin-hook-visible, persisted in session metadata, or plugin-owned only.
