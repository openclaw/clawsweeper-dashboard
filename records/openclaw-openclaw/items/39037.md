## Summary

Keep open. Current main and v2026.5.12 still read WhatsApp Baileys `contextInfo` for quotes/mentions but do not normalize or expose CTWA referral fields, and the related forwarding/GIF metadata items do not cover `ctwaClid`.

## What This Changes

Expose WhatsApp CTWA referral metadata from inbound Baileys context so agents can use `ctwa_clid` for Meta Conversions API attribution.

## Best Possible Solution

Add a bounded WhatsApp-owned CTWA referral extractor and append sanitized fields to the existing untrusted structured-context carrier; defer durable session storage until a broader session schema is designed.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, but current main has no reader that normalizes or forwards that CTWA payload.

## Solution Assessment

Yes. The narrow maintainable path is additive extraction inside the WhatsApp plugin into `untrustedStructuredContext`, not a new core session metadata schema or plugin hook API in the first pass.
