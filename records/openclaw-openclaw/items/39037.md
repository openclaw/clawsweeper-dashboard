## Summary

Keep open. Current main can read WhatsApp Baileys `contextInfo` and has a generic untrusted structured metadata path for agent prompts, but it still never normalizes `contextInfo.externalAdReply.ctwaClid` or the CTWA source fields, so the attribution use case remains unimplemented.

## What This Changes

The issue requests exposing WhatsApp Click-to-WhatsApp referral metadata, especially `ctwa_clid`, to agents through session metadata or the prompt preamble.

## Best Possible Solution

Add a WhatsApp-owned normalizer for a minimal CTWA payload from `contextInfo.externalAdReply` into the existing untrusted structured metadata path, with focused regression tests and no durable session persistence in the first pass.

## Reproduction Assessment

Yes. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` can be traced through `extractContextInfo`, but current main only forwards contact structured context into the agent metadata path.

## Solution Assessment

Yes. The narrow maintainable solution is inside the WhatsApp plugin using `UntrustedStructuredContext`; adding core session schema, config, or persistence first would be broader than the reported attribution gap.
