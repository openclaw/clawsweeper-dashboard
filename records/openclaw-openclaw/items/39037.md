## Summary

Keep open: current main still reads raw WhatsApp `contextInfo` but does not extract or preserve CTWA ad-click metadata, and the remaining decision is the privacy-aware exposure surface rather than whether the source data exists.

## What This Changes

The issue requests carrying Click-to-WhatsApp `ctwaClid` and related ad-referral fields from inbound WhatsApp messages into agent, hook, or session metadata.

## Best Possible Solution

Add an allowlisted WhatsApp CTWA metadata carrier through an approved untrusted context, hook, or session surface, with durable storage only if maintainers explicitly choose it.

## Reproduction Assessment

Yes, source-level. A Baileys inbound message with `contextInfo.externalAdReply.ctwaClid` can reach `extractContextInfo`, but current main has no carrier from there into queued messages, prompt metadata, hook metadata, or session metadata.

## Solution Assessment

Unclear as proposed. The direction is valid, but copying referral data broadly into session metadata or the preamble is not the narrowest maintainable fix until maintainers choose a typed, privacy-aware exposure surface.
