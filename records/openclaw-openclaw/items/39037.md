## Summary

Keep open. Current main now has a WhatsApp external-ad-reply context path for title/source/body, but it still does not extract or expose `ctwaClid` or CTWA referral data into session or plugin hook metadata, and the latest release lacks it too. The remaining blocker is maintainer product/security direction for advertising click identifiers, not source ambiguity.

## What This Changes

Request to extract Click-to-WhatsApp ad referral data, especially `ctwaClid`, from inbound WhatsApp messages and make it available to agents, plugins, or session metadata.

## Best Possible Solution

Define a minimal typed CTWA referral contract in the WhatsApp/plugin hook path, with explicit prompt-visibility and retention rules before any session persistence.

## Reproduction Assessment

Yes, source-reproducible: current main reads WhatsApp `contextInfo` and some `externalAdReply` fields, but it never forwards `ctwaClid` or a CTWA referral record into session or hook metadata. No live Click-to-WhatsApp ad webhook was exercised in this read-only review.

## Solution Assessment

No, the broad preamble/session-persistence proposal is not the best default until maintainers choose the privacy boundary. A narrow typed plugin-visible CTWA referral record with security-reviewed prompt and retention rules is the safer end state.
