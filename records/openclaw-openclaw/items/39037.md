## Summary

Keep open: current main and v2026.5.20 still extract raw WhatsApp `contextInfo` but do not carry CTWA ad-click fields into normalized inbound messages, prompt context, plugin hook metadata, or session metadata. The dependency exposes `externalAdReply.ctwaClid`, but exposing and possibly storing ad-attribution identifiers needs an explicit privacy-aware product boundary.

## What This Changes

Expose WhatsApp Click-to-WhatsApp ad referral metadata, especially `ctwaClid`, from inbound messages to agent/session-accessible metadata.

## Best Possible Solution

Keep this open and add an allowlisted WhatsApp CTWA metadata path through the approved untrusted context, hook, or session surface, with durable storage only if maintainers explicitly choose it.

## Reproduction Assessment

Yes, source-level. A Baileys inbound message with `contextInfo.externalAdReply.ctwaClid` would reach the existing `extractContextInfo` path, but current main has no carrier from there into queued messages, prompt metadata, hook metadata, or session metadata.

## Solution Assessment

Unclear as proposed. The direction is valid, but broadly copying referral data into session metadata or the preamble is not the narrowest maintainable fix until maintainers choose a typed, privacy-aware exposure surface.
