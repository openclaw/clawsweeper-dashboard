## What This Changes

Expose Click-to-WhatsApp ad referral fields from WhatsApp/Baileys inbound context as agent- or session-accessible metadata.

## Best Possible Solution

Add a WhatsApp-owned typed CTWA extractor for an allowlisted externalAdReply payload and expose it through the maintainer-approved untrusted context, hook, or session surface, with persistent storage only if explicitly approved.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with contextInfo.externalAdReply.ctwaClid would reach extractContextInfo, but current main has no carrier into queued messages, prompt metadata, hook metadata, or session metadata.

## Solution Assessment

Unclear as proposed. The direction is valid, but broadly copying referral data into session metadata or the preamble is not the narrowest maintainable fix until maintainers choose a typed, privacy-aware exposure surface.
