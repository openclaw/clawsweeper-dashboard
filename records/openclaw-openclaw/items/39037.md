## Summary

Keep open. Current main still has no WhatsApp CTWA or `ctwaClid` extraction, while the pinned Baileys contract exposes `contextInfo.externalAdReply.ctwaClid` and the existing untrusted structured metadata path can carry this to prompts without a new core session schema.

## What This Changes

Expose Click-to-WhatsApp referral metadata, especially `ctwaClid`, from inbound WhatsApp Baileys message context into session or prompt metadata.

## Best Possible Solution

Add a bounded WhatsApp-plugin normalizer for CTWA referral fields from `contextInfo.externalAdReply` into the existing untrusted structured metadata path, with no durable persistence in the first pass.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to trace a Baileys-shaped inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid`; current main can reach `contextInfo` but never normalizes or forwards that payload.

## Solution Assessment

Yes for the narrow extraction and untrusted metadata path. Promoting provider-controlled ad fields into durable session state or trusted/plain prompt text would be broader than needed for the attribution gap.
