## Summary

Keep open. Current main still drops WhatsApp Click-to-WhatsApp referral metadata: Baileys exposes the CTWA click-id fields under `contextInfo.externalAdReply`, OpenClaw extracts raw `contextInfo`, but the WhatsApp inbound normalization and prompt-context carrier never read or forward that payload.

## What This Changes

Expose WhatsApp Click-to-WhatsApp referral metadata, especially `ctwa_clid`, from inbound messages to agent/session context.

## Best Possible Solution

Add a WhatsApp-owned CTWA referral extractor and append sanitized fields to `untrustedStructuredContext`, deferring durable session storage unless maintainers choose a broader session metadata design.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, and current main has no reader or carrier assignment for that CTWA payload.

## Solution Assessment

Yes as a narrow feature direction. The maintainable fix is additive extraction inside the WhatsApp plugin into the existing untrusted structured-context carrier, not a new core session metadata schema in the first pass.
