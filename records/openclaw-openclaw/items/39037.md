## Summary

Keep open. Current main still extracts raw WhatsApp `contextInfo` but has no CTWA/referral reader or assignment into the WhatsApp structured metadata path; the related media and forwarding-metadata items do not cover this click-id attribution request.

## What This Changes

Expose WhatsApp Click-to-WhatsApp click-id/referral fields from inbound messages in agent/session metadata for attribution workflows.

## Best Possible Solution

Add a WhatsApp-owned CTWA extractor for `contextInfo.externalAdReply`, append sanitized bounded fields to `untrustedStructuredContext`, and defer durable session storage unless maintainers define a broader metadata contract.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches `extractContextInfo`, and current main has no CTWA reader or structured-context assignment for that payload.

## Solution Assessment

Yes for the narrow feature direction, but the final metadata surface still needs maintainer agreement. The maintainable path is additive WhatsApp plugin extraction into the existing untrusted structured-context carrier, not a core session schema change in the first pass.
