## Summary

Keep open. Current main still reaches WhatsApp Baileys `contextInfo`, but the bundled WhatsApp plugin only propagates existing contact/location/reply metadata and does not normalize CTWA `externalAdReply`/referral fields into the agent context; related #69374 is a neighboring forwarding-metadata request, not a replacement.

## What This Changes

Expose Click-to-WhatsApp referral data such as `ctwaClid`, source URL/type/id, and headline/title from inbound WhatsApp messages to session metadata or the agent preamble.

## Best Possible Solution

Add a WhatsApp-owned normalizer that copies a bounded CTWA referral payload from `contextInfo.externalAdReply` into existing untrusted structured context for the agent preamble, with regression tests, and defer durable session storage.

## Reproduction Assessment

Yes. Source inspection shows a synthetic Baileys inbound message with `contextInfo.externalAdReply.ctwaClid` would reach `extractContextInfo`, but no current code copies those fields into `untrustedStructuredContext` or prompt metadata.

## Solution Assessment

Yes. The narrow maintainable direction is to keep this inside the WhatsApp plugin and reuse the existing untrusted structured metadata path instead of adding core session schema, raw `contextInfo` exposure, or durable persistence first.
