## Summary

Keep open. Current main still extracts WhatsApp `contextInfo` for existing quote, mention, contact, media, and reply flows, but no WhatsApp path reads `externalAdReply.ctwaClid` or forwards CTWA/referral ad metadata into agent-accessible context; the related externalAdReply and forwarding issues are adjacent rather than replacements for this request.

## What This Changes

Expose Click-to-WhatsApp ad referral fields, especially `ctwaClid`, from WhatsApp inbound messages in session or agent-accessible metadata.

## Best Possible Solution

Normalize bounded CTWA ad fields from WhatsApp `contextInfo.externalAdReply` into the plugin-owned untrusted structured context path, with focused extraction and dispatch tests, leaving durable session storage for a separate decision if needed.

## Reproduction Assessment

Yes, source-level. A Baileys inbound message with `extendedTextMessage.contextInfo.externalAdReply.ctwaClid` reaches the current `contextInfo` extractor, but current main never normalizes or forwards that CTWA payload into agent-accessible metadata.

## Solution Assessment

Yes. The narrowest maintainable direction is WhatsApp-plugin extraction into existing `untrustedStructuredContext`; adding a new core session schema or persistent session attribute would be broader than the first pass needs.
