## Summary

Keep open. Current main still extracts raw WhatsApp contextInfo but has no CTWA/referral reader and no assignment into WhatsApp structured prompt or session metadata, while the related forwarding-metadata work covers a different contextInfo signal.

## What This Changes

Request to expose Click-to-WhatsApp ctwa_clid/referral fields from WhatsApp inbound messages in session or prompt metadata for Meta attribution workflows.

## Best Possible Solution

Add a WhatsApp-owned CTWA extractor for bounded externalAdReply fields, pass them through untrusted structured context, and defer durable session storage unless maintainers define that broader contract.

## Reproduction Assessment

Yes, source-level. A synthetic Baileys inbound message with contextInfo.externalAdReply.ctwaClid reaches extractContextInfo, and current main has no reader or structured-context assignment for that payload.

## Solution Assessment

Yes for the narrow feature direction, but the final exposure policy still needs maintainer agreement. The maintainable path is additive WhatsApp plugin extraction into untrusted structured context rather than a core session schema change in the first pass.
