## Summary

Keep this open: current main has a fix for the related `openclaw-lark` channel-owner case, but the missing-plugin repair path can still reinstall from a persisted npm record before checking whether another discovered local/global plugin already has the same manifest ID.

## What This Changes

The issue reports that `openclaw doctor --fix` can install Feishu/Lark plugin packages that conflict with an existing local plugin exposing the same plugin ID.

## Best Possible Solution

Teach doctor missing-plugin repair to treat an already-discovered, policy-eligible plugin with the same manifest ID as satisfying that plugin before npm/ClawHub reinstall, and report a clear warning instead of creating a duplicate-id conflict.

## Reproduction Assessment

No live high-confidence reproduction was run. The issue log is concrete, and current source shows a plausible current path where missing persisted install repair can run before duplicate-id de-duplication, so this is source-reproducible but not live-reproduced.

## Solution Assessment

No, not as written if it means always preferring any local extension. The safer fix is a doctor repair guard that de-dupes by manifest plugin ID while respecting existing trust, allowlist, and channel-owner policy.
