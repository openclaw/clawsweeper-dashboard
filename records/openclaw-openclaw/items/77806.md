## Summary

Keep open: current `main` still has a source-reproducible doctor repair path that can reinstall a missing managed plugin record or explicitly configured Feishu catalog plugin even when another local/config plugin already owns the same Feishu/Lark surface, which can produce the duplicate-ID warning reported here.

## What This Changes

The issue asks `openclaw doctor --fix` to avoid auto-installing Feishu/Lark plugin packages when an existing local plugin already satisfies the same plugin ID or channel ownership.

## Best Possible Solution

Make doctor repair treat an already discovered, trusted local/config plugin with the same plugin ID or effective Feishu channel ownership as satisfying the missing install, drop stale managed install records where appropriate, and keep installing only when the existing owner is blocked or not trusted.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current tests model `openclaw-lark` owning `feishu`, and the repair code still has paths that install or reinstall configured Feishu/Lark candidates despite known local ownership. I did not run a live Windows doctor pass because this review is read-only.

## Solution Assessment

Yes, a narrow source fix is appropriate: adjust `missing-configured-plugin-install.ts` to skip or retire duplicate managed installs when manifest metadata already provides the owner, with regression coverage for the reported Feishu/Lark shape. Avoid adding prompts, config flags, or hard-coded service aliases unless the narrow owner check cannot express the case.
