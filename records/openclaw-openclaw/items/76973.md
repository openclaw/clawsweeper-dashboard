## Summary

Keep this PR open: the WhatsApp group-visible-reply direction is valid, but the current patch would override direct-chat harness defaults and can make Codex-mode direct WhatsApp replies automatically visible.

## What This Changes

The PR updates WhatsApp inbound reply dispatch to resolve and forward source reply delivery mode, disables block streaming for tool-only source replies, and adds WhatsApp/shared dispatch regression tests.

## Best Possible Solution

Preserve WhatsApp group/channel tool-only behavior while leaving direct turns unset unless user config or native-command handling explicitly selects a source reply mode.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection shows the patch passes an explicit `automatic` mode for direct WhatsApp turns, while current dispatch applies Codex direct `message_tool` defaults only when the option is undefined. No live tests were run under the read-only sweep constraint.

## Solution Assessment

No. The narrow maintainable fix is to avoid overriding direct-chat harness defaults, for example by only precomputing group/channel or explicit-config modes in the WhatsApp wrapper and letting shared dispatch resolve direct defaults.
