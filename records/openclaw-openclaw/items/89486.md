## Summary

Keep open: this PR targets the right context-engine heartbeat bug, but the patch only adds optional helper parameters and does not update the actual heartbeat caller paths, so heartbeat runs would still reach `contextEngine.afterTurn()` without `isHeartbeat`. The external PR also lacks after-fix real behavior proof.

## What This Changes

Adds optional `isHeartbeat` parameters to the harness finalization and context-engine loop-hook helpers and conditionally forwards them into `contextEngine.afterTurn()`.

## Best Possible Solution

Derive `isHeartbeat` from `trigger === "heartbeat"` at the embedded/Codex attempt caller paths, pass it through both `afterTurn` paths, and add focused regression coverage plus redacted real heartbeat-run proof.

## Reproduction Assessment

Yes by source inspection: current main's SDK contract declares `afterTurn(params).isHeartbeat`, while both current `afterTurn` call objects omit it and heartbeat runs are represented by `trigger: "heartbeat"`. I did not execute the live heartbeat command in this read-only review.

## Solution Assessment

No. Helper-level forwarding is a plausible part of the fix, but without caller wiring from the existing heartbeat trigger it is not complete or merge-ready.
