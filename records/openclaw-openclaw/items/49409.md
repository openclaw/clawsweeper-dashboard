## Summary

Keep open: current main already made session-memory slugging opt-in/non-blocking and made startup context tolerate slugged reset artifacts, but this PR still carries the distinct canonical daily-writer change and is not mergeable as-is because the branch has atomicity, local-time, docs, and proof blockers.

## What This Changes

The PR changes bundled session-memory to append reset/new captures into `memory/YYYY-MM-DD.md`, adds local/opt-in LLM slug timeout plumbing, and changes the safe append helper to return open metadata.

## Best Possible Solution

Refresh or replace the branch only after maintainers confirm the canonical daily-writer contract, then implement it with local-time naming, atomic append semantics, retained workspace-routing coverage, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection shows current main still writes slugged per-capture files, while the PR diff shows a two-open daily-header path that can duplicate the header under concurrent first writes; I did not run tests because this was a read-only review.

## Solution Assessment

No. The direction may still be useful, but this branch is not the best mergeable fix until it is refreshed against current fs-safe APIs, preserves local-time and workspace-routing behavior, documents the new contract, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Make the first daily header append atomic:** `src/hooks/bundled/session-memory/handler.ts:382-393`
  - body: The branch probes the daily file with an empty append, closes it, then performs the real append based on that stale result. Two concurrent reset hooks can both see an empty/new file and both write `# YYYY-MM-DD`, so the canonical file can be malformed.
  - confidence: 0.94
- **[P2] Preserve local-time daily filenames:** `src/hooks/bundled/session-memory/handler.ts:250`
  - body: The branch derives the canonical daily filename from the UTC/ISO date instead of the current local timestamp contract. Users west of UTC can write tomorrow's memory file during local evening resets, so the daily filename should reuse the existing local date formatter.
  - confidence: 0.9
- **[P3] Document the new session-memory file contract:** `src/hooks/bundled/session-memory/handler.ts:346`
  - body: Changing output from per-capture `YYYY-MM-DD-HHMM.md` or `YYYY-MM-DD-slug.md` files to one canonical `YYYY-MM-DD.md` file is user-visible, but the PR does not update the bundled hook docs or public hook docs that still describe slugged output.
  - confidence: 0.86
