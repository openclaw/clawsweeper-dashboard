## Summary

Keep this PR open. Current main still performs the behavior the PR is trying to change: the bundled session-memory hook defaults to LLM slug generation and writes per-session `YYYY-MM-DD-slug.md` files, while startup context can read canonical daily files but the writer does not create them. The branch is useful but not merge-ready because its daily-header append path has a first-write race and it is stale against the current host-local timezone behavior.

## What This Changes

The PR changes the bundled session-memory hook to write `/new` and `/reset` dumps into a canonical daily `memory/YYYY-MM-DD.md` file, make LLM slugging opt-in with a shorter configurable timeout, and expose append metadata from `appendFileWithinRoot` for header decisions.

## Best Possible Solution

Refresh the branch against current main and keep the useful direction: canonical daily appends for the bundled session-memory hook, nonblocking or opt-in LLM slugging, local-time filename/header behavior, preserved workspaceDir/display-session-key routing, updated public and bundled hook docs, and a concurrency-safe first daily header path with focused regression coverage.

## Reproduction Assessment

Yes. The current-main gap is source-reproducible from the handler: it awaits LLM slug generation and writes `${dateStr}-${slug}.md`; the PR race is code-reproducible by two first daily appenders both completing the probe before either content append lands.

## Solution Assessment

No. The proposed direction is maintainable, but this branch is not the best current solution until it removes the two-open header race, preserves current local-time behavior, keeps workspace-routing coverage, and updates the docs that still describe slugged LLM files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Make the first daily header append atomic:** `src/hooks/bundled/session-memory/handler.ts:382-393`
  - body: The empty `appendFileWithinRoot` probe and the real content append happen in separate opens. If two `/new` or `/reset` hooks create the day's file concurrently, both can observe `createdForWrite || size === 0` and both append `# YYYY-MM-DD`, corrupting the canonical daily file with duplicate top-level headers. Move the header decision into the same append operation or otherwise serialize the first-write header path.
  - confidence: 0.94
- **[P2] Preserve local-time memory filenames:** `src/hooks/bundled/session-memory/handler.ts:250`
  - body: This branch computes the daily filename from `now.toISOString()`, which uses UTC and would undo current main's fix for host-local session-memory filenames and headers. Users west of UTC can again get tomorrow's memory file during local evening resets, so the canonical daily filename should be derived with the current local-time formatter instead.
  - confidence: 0.9
