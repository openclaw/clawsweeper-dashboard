## Summary

Keep this PR open: the Control UI rename command is a useful feature, but the patch currently advertises it through the shared native command registry without adding a matching auto-reply/native handler, and the required external real behavior proof is still insufficient.

## What This Changes

Adds `/rename <title>` with `/title` alias to Control UI slash commands, executing `sessions.patch` with a label plus focused tests and slash-command docs.

## Best Possible Solution

Choose one explicit command boundary: keep rename/title Control UI-local, or implement the full authorized native/text command path with channel coverage before exposing it through the shared registry.

## Reproduction Assessment

Yes, the blocking issue is source-reproducible: shared commands with `nativeName` are exported to native command inventories, and current main has no rename handler in the auto-reply command handler list. I did not run a live UI/channel scenario in this read-only review.

## Solution Assessment

No, not as written; the local executor path is narrow, but registering the command in the shared native registry is not safe unless the cross-channel handler and compatibility tests are added. The safer path is UI-local metadata unless maintainers intentionally want a global command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not register rename as native without a handler:** `src/auto-reply/commands-registry.shared.ts:731-733`

Keep this as a Control UI-only command, or add the cross-channel handler and tests before registering it here.
