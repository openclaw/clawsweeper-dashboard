## Summary

Keep open: the Control UI rename command is useful, but the branch registers `/rename` and `/title` as shared native commands without adding a non-UI command handler, so native command menus can expose a command that only Control UI executes. The supplied proof is helpful, but it does not show this source branch executing the command and updating the visible session title.

## What This Changes

Adds `/rename <title>` and `/title` session-label commands to shared slash-command metadata, Control UI parsing/execution, focused tests, and slash-command docs.

## Best Possible Solution

Keep the session rename command, but align the command boundary: either make it strictly Control UI-local, or add a core/native handler over `sessions.patch` with tests and proof for those provider surfaces.

## Reproduction Assessment

Not applicable as a feature PR rather than a reported bug. Source inspection does show current main supports session labels through `sessions.patch` but has no Control UI `/rename` command.

## Solution Assessment

No. The UI executor shape is reasonable, but registering the command as shared/native metadata is not the best UI-only solution unless the PR also implements the corresponding core/native handler.
