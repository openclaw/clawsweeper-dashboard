## Summary

Keep open: current main still lacks the backend subscription-teardown guarantee this report needs, so an embedded timeout can abort and unsubscribe before `agent_end` reaches the Control UI chat lifecycle path.

## What This Changes

Ensure Control UI chat leaves the `Stop` state after an embedded run timeout or abort by always delivering a terminal lifecycle/chat event.

## Best Possible Solution

Make `subscribeEmbeddedPiSession` enforce exactly one terminal lifecycle event before listener teardown on timeout/abort, with idempotency coverage against the normal `agent_end` path and Control UI chat finalization coverage.

## Reproduction Assessment

Yes, for a source-level reproduction: force an embedded timeout/abort so runner cleanup calls `unsubscribe()` before the underlying pi-agent delivers `agent_end`. No live Windows run was executed in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is a backend subscription teardown fallback that emits exactly one terminal lifecycle event. A UI `agent.wait` watchdog can help related stuck states but should be complementary, not the lifecycle contract fix.
