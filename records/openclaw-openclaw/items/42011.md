## Summary

Keep open: current main still has the source-reproducible timeout/teardown path where an embedded run can abort and unsubscribe before any terminal lifecycle event reaches Control UI. The direct fix proposal in #73105 is closed unmerged, and the open UI watchdog work in #42258 is related but does not replace the backend lifecycle contract for this issue.

## What This Changes

The issue requests that embedded run timeout/abort paths always deliver a terminal lifecycle/chat state so Control UI leaves the Stop state.

## Best Possible Solution

Add a backend subscription teardown fallback that emits exactly one terminal lifecycle event on timeout/abort teardown, idempotent with normal `agent_end`, with regression coverage for unsubscribe-before-agent_end.

## Reproduction Assessment

Yes for source-level reproduction: force an embedded timeout/abort after startup so cleanup unsubscribes before `agent_end`; current main shows that teardown path and lacks fallback terminal emission. No live Windows run was executed in this read-only pass.

## Solution Assessment

Yes: the narrowest maintainable fix is at the embedded subscription lifecycle seam, enforcing exactly one terminal event before teardown. A UI `agent.wait` watchdog can help related stuck states but should not replace the backend lifecycle contract.
