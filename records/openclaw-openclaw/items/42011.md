## Summary

Keep open: current main still allows the embedded timeout path to abort asynchronously and unsubscribe before `agent_end`, with no backend guarantee that a terminal lifecycle/chat event reaches Control UI.

## What This Changes

Ensure Control UI chat finalizes and leaves the Stop state after an embedded run timeout or abort, even if pi-agent never delivers `agent_end`.

## Best Possible Solution

Add an exactly-once backend terminal lifecycle fallback at the embedded subscription teardown seam, with regression coverage for timeout/abort unsubscribe races; a UI `agent.wait` watchdog can remain complementary.

## Reproduction Assessment

Yes, at source level: force an embedded timeout or abort so `abortRun(true)` starts `activeSession.abort()` asynchronously and runner cleanup calls `unsubscribe()` before pi-agent delivers `agent_end`. No live Windows run was executed in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to make `subscribeEmbeddedPiSession` enforce exactly one terminal lifecycle event before listener teardown. The open UI watchdog PR may mitigate stale clients, but it should not replace the backend lifecycle contract.

## Repair Work Prompt

Implement a narrow backend fix for https://github.com/openclaw/openclaw/issues/42011. Current main still lets an embedded timeout/abort call `activeSession.abort()` asynchronously and then `unsubscribe()` before `agent_end`, so Control UI may never receive `final`/`error`/`aborted` and can stay on Stop. Fix the contract at `subscribeEmbeddedPiSession`: before listener teardown, synthesize exactly one terminal lifecycle event when no normal `agent_end` terminal event was emitted. Make it idempotent with the normal `handleAgentEnd` path, preserve `setTerminalLifecycleMeta` fields (`replayInvalid`, `livenessState`, `stopReason`, `yielded`), and keep `onBeforeLifecycleTerminal` ordering. Use PR #73105 as source context only; implement fresh on current main. Do not solve this solely with a UI timer/watchdog, do not change timeout defaults, and do not add config. Add focused regression tests for unsubscribe-without-agent_end and duplicate-suppression with normal agent_end. Add a single-line user-facing changelog entry under the active Fixes section, with no bot attribution. Validate with targeted agent lifecycle tests, gateway chat lifecycle tests if chat-final behavior is touched, formatter check on touched files, then `pnpm check:changed` in the normal maintainer/Testbox lane.
