## Summary

Keep open: current main still has the backend lifecycle race described here. The timeout path aborts asynchronously, cleanup can unsubscribe before `agent_end`, and the subscription teardown path still does not synthesize a terminal lifecycle event for Control UI.

## What This Changes

This issue asks OpenClaw to guarantee that embedded-run timeout or abort paths send a terminal Control UI chat state so the chat leaves `Stop`.

## Best Possible Solution

Add an exactly-once terminal lifecycle fallback at `subscribeEmbeddedPiSession` teardown before listener removal, with regression coverage for timeout and abort unsubscribe races; keep UI watchdog recovery as a secondary mitigation only.

## Reproduction Assessment

Yes, at source level: force an embedded timeout so `abortRun(true)` calls `void activeSession.abort()` and the runner reaches `unsubscribe()` before `agent_end`. Current unsubscribe removes the listener without emitting a terminal lifecycle event; no live Windows timeout was run in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is backend lifecycle enforcement at the embedded subscription teardown seam. A UI `agent.wait` watchdog can recover stale clients, but it should not replace the backend terminal-event guarantee.
