## Summary

Close as implemented: current main ships the embedded lifecycle terminal backstop that emits a terminal lifecycle event when an embedded timeout/error turn returns without `agent_end`, and that fix is included in `v2026.4.29` and later releases.

## What This Changes

Ensure Control UI chat leaves the `Stop`/running state after an embedded run timeout or abort by guaranteeing a terminal lifecycle/chat finalization path.

## Best Possible Solution

Keep the shipped runner-level lifecycle backstop; handle distinct remaining stuck-chat cases in their narrower tracking items instead of reopening this timeout lifecycle issue.

## Reproduction Assessment

Yes. The original bug has a high-confidence source-level reproduction: an embedded timeout/error turn can return without `agent_end`; current regression coverage now simulates that missing-terminal path and verifies the synthesized terminal event.

## Solution Assessment

Yes. The shipped runner-level backstop is the narrow maintainable solution because it emits one terminal lifecycle event from the owner path and reuses the existing gateway chat finalization contract.
