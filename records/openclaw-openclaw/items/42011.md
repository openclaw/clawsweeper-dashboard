## Summary

Current main still has the backend race described by the issue: the embedded runner fire-and-forgets `activeSession.abort()` on timeout and then unsubscribes synchronously, while terminal lifecycle emission still depends on receiving `agent_end`. The closed, unmerged PR #73105 proposed the right backend seam; the open UI watchdog PR #42258 is related but tracks #42213 and does not prove this issue is fixed.

## What This Changes

The issue reports Control UI chat staying in a running Stop state after an embedded run times out without a terminal lifecycle event.

## Best Possible Solution

Add a backend contract guard at the embedded subscription seam so teardown emits exactly one terminal lifecycle event when `agent_end` was missed, with focused regression coverage for timeout/abort unsubscribe ordering.

## Reproduction Assessment

Yes from source, medium confidence: current main still has a timeout path that fire-and-forgets abort and then unsubscribes, while terminal lifecycle emission depends on an `agent_end` event reaching the subscription. I did not live-reproduce the Windows UI hang.

## Solution Assessment

No current-main fix is in place. The narrowest maintainable fix is the backend fallback terminal lifecycle at `subscribeEmbeddedPiSession.unsubscribe()`, not only a UI-side watchdog, because it protects every lifecycle consumer from the missed `agent_end` race.

## Repair Work Prompt

Fix the backend lifecycle race reported in https://github.com/openclaw/openclaw/issues/42011. On embedded run timeout/abort, `runEmbeddedAttempt` currently calls `void activeSession.abort()` and later `unsubscribe()`, while terminal lifecycle emission still depends on receiving `agent_end`; add a narrow fallback in the embedded subscription layer so teardown emits exactly one terminal lifecycle `end` or `error` when `agent_end` was missed. Keep the fix backend-owned in `src/agents/pi-embedded-subscribe*` unless a tiny generic helper is needed; do not add UI watchdog behavior, config flags, provider policy, or pi-agent-core patches. Preserve idempotency when normal `agent_end` arrives, respect existing `setTerminalLifecycleMeta` data, and ensure Gateway/Control UI receives a final chat state so `chatRunId` clears. Add focused regression coverage for unsubscribe-before-agent_end and idempotency. Add a changelog fix entry because this is user-visible.
