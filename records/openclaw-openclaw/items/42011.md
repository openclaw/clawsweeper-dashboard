## Summary

Keep open. Current main still has the embedded timeout lifecycle gap described by the reporter, and PR #73105 is an open closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that Control UI/Gateway Dashboard chat always leave the running Stop state after an embedded run timeout or abort by receiving a terminal lifecycle/chat event.

## Best Possible Solution

Review and land PR #73105, or an equivalent backend fix, so embedded pi-session teardown guarantees exactly one terminal lifecycle event when agent_end is missed. PR #42258 can still be considered as a UI recovery layer, but the backend lifecycle contract should be the primary fix.

## Reproduction Assessment

Yes. At the source/seam level, start an embedded run, trigger timeout or abort after onAgentRunStart, then let unsubscribe remove the pi-session listener before agent_end reaches the subscriber; current main has no fallback terminal lifecycle emission for that path.

## Solution Assessment

Yes. The backend subscription-teardown fix proposed by PR #73105 is the narrowest maintainable direction because every affected chat surface depends on the terminal lifecycle contract; a UI watchdog mitigates symptoms but does not repair the missing event.
