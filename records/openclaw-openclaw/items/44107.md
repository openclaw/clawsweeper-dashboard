## Repair Work Prompt

Fix the remaining browser-extension blank-page half of openclaw/openclaw#44107. Add a small pre-module Control UI bootstrap fallback in `ui/index.html` that detects when `<openclaw-app>` never upgrades/mounts within a short timeout and replaces or augments the blank page with a plain HTML diagnostic pointing users to disable browser extensions or use an Incognito/private window. Add user-facing troubleshooting docs, preferably in `docs/web/control-ui.md` and/or `docs/help/troubleshooting.md`, covering Chrome extensions with `<all_urls>`/`document_start` content scripts interfering with `http://127.0.0.1:18789/`. Add a single-line `CHANGELOG.md` fix entry because this is user-facing. Do not rework `chat.history`, markdown rendering, pagination, virtualization, or the shipped large-history safeguards unless a current-main repro proves they still freeze. Preserve existing theme bootstrap behavior and avoid leaking gateway tokens in any fallback text.
