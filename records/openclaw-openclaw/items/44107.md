## Summary

Keep open. Current main appears to have shipped the large-history and markdown freeze fixes, including bounded `chat.history` projection and the `markdown-it` ReDoS fix, but it does not implement the reported Chrome extension blank-page fallback or matching troubleshooting docs. Because this issue explicitly reports two independent failures, it is not fully implemented yet.

## What This Changes

Request to address two Control UI dashboard failures: a Chrome extension blank page before app mount, and a browser freeze while rendering a large chat history with thinking and tool content.

## Best Possible Solution

Keep the shipped `chat.history` and markdown hardening, and add a narrow Control UI bootstrap fallback plus user-facing troubleshooting docs for extension-caused blank pages. Avoid a broader chat pagination or virtualization rewrite unless a current-main large-history repro still freezes after the shipped guards.

## Reproduction Assessment

Yes, the issue gives concrete reproduction paths for both failures: enabling a `<all_urls>` document-start Chrome extension for the blank page, and loading a large thinking/tool session for the freeze. I did not live-run the browser-extension scenario in this read-only review, but current source and docs are enough to verify the remaining missing fallback.

## Solution Assessment

No, closing as implemented is not the best resolution because only the freeze half is implemented on current main. The best remaining fix is a small pre-module mount sentinel and docs entry for browser extension conflicts, while leaving the existing markdown/history fixes in place.
