## Summary

Keep open. Current main appears to cover the chat-history/browser-freeze side through bounded history payloads, markdown hardening, and live-tool-result reload fixes, but the separate Chrome extension blank-page case still lacks a startup fallback or troubleshooting docs.

## What This Changes

The issue requests fixes for two Control UI regressions: a blank dashboard when early Chrome extension content scripts interfere with module startup, and a browser freeze when large or thinking-heavy chat history renders.

## Best Possible Solution

Keep the shipped freeze protections, then add a minimal Control UI startup failure fallback and matching user-facing troubleshooting docs for browser extension conflicts before considering this issue resolved.
