## Summary

Keep open: current main and the latest release still reject configured extra bootstrap files whose basename is not in the recognized bootstrap set, and https://github.com/openclaw/openclaw/pull/74735 is the active closing implementation but remains open and conflicting.

## What This Changes

Request support for configured arbitrary workspace files to be injected into bootstrap/session context instead of limiting `bootstrap-extra-files` to standard bootstrap basenames.

## Best Possible Solution

Keep this as the canonical issue and resolve it by reviewing, rebasing, and landing https://github.com/openclaw/openclaw/pull/74735 or a narrower maintainer-approved implementation that preserves workspace guards, size budgets, session filtering, and docs.
