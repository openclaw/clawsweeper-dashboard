## Summary

Keep this open because current main still rejects explicitly configured custom `bootstrap-extra-files` basenames, and an open closing PR already covers the requested implementation space.

## What This Changes

Request support for explicitly configured arbitrary workspace files, such as `hooks/REGISTRY.yaml`, to be injected by the `bootstrap-extra-files` hook at session bootstrap time.

## Best Possible Solution

Accept one implementation that relaxes the basename allowlist for explicitly configured extra files while preserving workspace containment, ordering, session filtering, and bootstrap budget guards.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure `bootstrap-extra-files.paths` with a custom basename and the loader skips it as `invalid-bootstrap-filename` before guarded reading.

## Solution Assessment

Yes for the narrow requested behavior. The maintainable fix is to use the existing explicit `paths`/`patterns`/`files` hook contract and existing guards rather than adding a broader new agent-defaults schema first.
