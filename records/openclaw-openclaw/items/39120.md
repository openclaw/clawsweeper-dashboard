## Summary

Keep open: current main and the latest release still reject explicitly configured extra bootstrap files whose basename is outside the recognized bootstrap set, and the open closing PR is still the active unmerged implementation path.

## What This Changes

The issue requests a supported way to inject user-configured workspace files with arbitrary basenames into bootstrap/session context under the existing workspace guards.

## Best Possible Solution

Keep this as the canonical issue and resolve it by reviewing, rebasing, and landing https://github.com/openclaw/openclaw/pull/74735 or a narrower maintainer-approved implementation that preserves workspace guards, size budgets, session filtering, and docs.
