## Summary

Keep open: current main and the latest release still reject explicitly configured extra bootstrap files whose basename is not in the recognized bootstrap set, and the open closing PR at https://github.com/openclaw/openclaw/pull/74735 remains the active implementation path pending maintainer product/merge review.

## What This Changes

The issue requests deterministic bootstrap injection for arbitrary workspace files instead of limiting `bootstrap-extra-files` to recognized bootstrap basenames.

## Best Possible Solution

Keep this as the canonical issue and resolve it by reviewing, rebasing, and landing https://github.com/openclaw/openclaw/pull/74735 or a narrower maintainer-approved implementation that preserves workspace guards, budgets, and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows configured extra files with unrecognized basenames are rejected through `VALID_BOOTSTRAP_NAMES`; I did not run a write-producing local repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API. The linked PR is a plausible narrow path, but current main does not solve the issue and the implementation needs maintainer review after rebase.
