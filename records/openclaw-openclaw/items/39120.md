Keep open. Current main and latest release still reject custom bootstrap-extra-files basenames, and the linked implementation PR is open and conflicted, so this should wait for maintainer contract review rather than close as implemented.

## What This Changes

Allow bootstrap context injection to include explicitly configured arbitrary workspace-relative files while preserving workspace boundary and size guards.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/74735 or a coordinated replacement into one documented guarded bootstrap-extra-files contract, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows a configured custom file such as hooks/REGISTRY.yaml is rejected by basename before the guarded workspace read path; I did not run a mutating runtime repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. Removing or scoping the basename gate while preserving guarded workspace reads is the narrow technical path, but the open PRs differ on global arbitrary files versus exact-session semantics.
