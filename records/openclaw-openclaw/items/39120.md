## Summary

Keep open. Current main still rejects arbitrary extra bootstrap file candidates before the guarded read path, the docs and focused tests still preserve the recognized-basename limitation, and the related implementation PR #73917 is open and unmerged.

## What This Changes

The issue asks for arbitrary workspace-relative files to be injectable during bootstrap instead of limiting the bundled bootstrap-extra-files hook to recognized bootstrap basenames.

## Best Possible Solution

Keep this issue open as the canonical tracker, review and land #73917 or make an explicit product decision against the change, then close narrower duplicates once the final arbitrary extra-bootstrap-file behavior ships.
