## What This Changes

Request support for user-configured arbitrary workspace files in bootstrap context injection instead of limiting `bootstrap-extra-files` to recognized basenames.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/74735 into one documented guarded bootstrap-extra-files contract, then close this issue after merge.

## Solution Assessment

Unclear until maintainers choose the public contract. Removing or scoping the basename gate while preserving guarded workspace reads is the narrow technical path, but the open implementation path adds exact-session semantics that still need review.
