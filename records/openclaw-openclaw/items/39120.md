## Summary

Keep open: current main and the latest release still reject explicitly configured extra bootstrap files whose basename is not in the recognized bootstrap set, and the open closing PR at https://github.com/openclaw/openclaw/pull/74735 is the active but not-yet-mergeable implementation path.

## What This Changes

The issue requests expanding bootstrap-extra-files/bootstrap context loading so explicitly configured arbitrary workspace files can be injected at startup instead of only recognized bootstrap basenames.

## Best Possible Solution

Keep this as the canonical issue and resolve it by reviewing, rebasing, and landing https://github.com/openclaw/openclaw/pull/74735 or a narrower maintainer-approved implementation that preserves workspace guards, budgets, and docs.
