## Summary

Keep open: current main and the latest release still reject configured extra bootstrap files whose basename is outside the recognized bootstrap set, and the open closing PR at https://github.com/openclaw/openclaw/pull/74735 remains the active but conflicting implementation path.

## What This Changes

The issue requests support for injecting arbitrary workspace files into bootstrap/session context instead of limiting configured `bootstrap-extra-files` entries to recognized bootstrap basenames.

## Best Possible Solution

Keep this as the canonical issue and resolve it by landing https://github.com/openclaw/openclaw/pull/74735 or a narrower maintainer-approved implementation that preserves workspace guards, budgets, session filtering, and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows unrecognized configured extra-file basenames are rejected through `VALID_BOOTSTRAP_NAMES`; no write-producing local repro was run because this review is read-only.
