## Summary

Keep open: current main and the latest release still reject explicitly configured custom bootstrap basenames, and https://github.com/openclaw/openclaw/pull/74735 is an open closing implementation candidate rather than a merged fix.

## What This Changes

Request to let `bootstrap-extra-files` inject explicitly configured arbitrary workspace-relative files instead of limiting extras to the built-in bootstrap basenames.

## Best Possible Solution

Resolve this through review of https://github.com/openclaw/openclaw/pull/74735 or a narrower replacement that preserves workspace containment, size budgets, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: configure `bootstrap-extra-files.paths` with a workspace file whose basename is not in `VALID_BOOTSTRAP_NAMES`, and the loader records `invalid-bootstrap-filename` instead of injecting it.

## Solution Assessment

Unclear. The existing bundled hook is the right likely seam, but maintainers still need to decide whether the best product shape is narrow arbitrary configured paths or the broader session-scoped config proposed in the linked PR.
