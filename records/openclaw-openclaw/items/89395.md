## Summary

Keep open: this is a focused diagnostic change, but it is not merge-ready as the full fix because it turns two Feishu media fallback paths into hard errors without making SecretRef-backed media upload succeed.

## What This Changes

The PR imports the Feishu SecretRef-unavailable guard, rethrows that error from two outbound media fallback catches, and adds mocked Vitest coverage for both paths.

## Best Possible Solution

Carry resolved runtime Feishu credentials into media upload, preserve fallback for ordinary upload/network failures, and rethrow only genuinely unresolved configuration errors; if this PR stays diagnostic-only, do not close the linked media-delivery issue.

## Reproduction Assessment

Yes, from source for the diagnostic failure path: unresolved non-env Feishu `appSecret` SecretRefs throw in `resolveFeishuRuntimeAccount`, and current main catches that in the media fallback paths. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

No, not as a full fix for the linked issue. The PR is an acceptable diagnostic mitigation, but the best fix for the user-reported regression is to ensure Feishu MEDIA upload receives resolved runtime credentials and only falls back for real upload/network failures.
