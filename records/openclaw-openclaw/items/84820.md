## Best Possible Solution

Build a macOS arm64 Node 25/26 heartbeat, cron, and active-memory session-lock stress reproduction with forced GC/open-handle observation, then fix the exact session lifecycle or `@openclaw/fs-safe` sidecar-lock path and validate Node 22 plus Node 24/25/26.

## Solution Assessment

Unclear. Explicit FileHandle close discipline is the right direction, but current source and the pinned fs-safe sidecar-lock already close on normal release, so the best fix should first isolate the missed lifecycle path rather than sweeping broadly.
