## Best Possible Solution

Add a focused current-main stress reproduction for Node 24+/25 session-lock churn, then fix the exact session-lock or fs-safe sidecar-lock path so every opened lock FileHandle is explicitly closed and covered by regression proof.

## Reproduction Assessment

## Solution Assessment

Unclear. The suggested try/finally direction is right in principle, but current main already centralizes lock release paths, so the best fix should first isolate whether the leak is in session lifecycle code or the `@openclaw/fs-safe` sidecar lock manager.
