## Summary

Keep open: the underlying duplicate-block indentation bug is still source-proven in current main, but this PR does not fix the active sanitizer path and lacks required after-fix real behavior proof.

## What This Changes

This PR adds an `openclaw-*.log` ignore pattern and changes the old `errors.ts` duplicate-block helper to append the original block instead of `block.trim()`.

## Best Possible Solution

Move the retained-block whitespace preservation into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` and add a duplicated indented-block regression test while leaving broader duplicate heuristics to https://github.com/openclaw/openclaw/issues/14629.

## Reproduction Assessment

Yes, from source inspection: `normalizeReplyPayload()` reaches the active sanitizer, and duplicated indented blocks are retained with `block.trim()` when a duplicate is collapsed. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No, not as this branch is written. Preserving the original retained block is the right narrow fix, but it must be applied to the active sanitizer module with regression coverage rather than the stale `errors.ts` helper.
