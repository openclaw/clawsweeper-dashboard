## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84899. Current main accepts `openclaw tasks audit --json --limit 1abc` as limit `1` because the registered `tasks audit` command uses `parsePositiveIntOrUndefined(opts.limit)` and the lean JSON route uses `getPositiveIntFlagValue`, both backed by `Number.parseInt`. Add strict positive-integer validation for the `tasks audit --limit` option in both entry paths so partial strings, fractional values, zero, and negative values are rejected before `tasksAuditCommand` or `tasksAuditJsonCommand` runs. Keep the fix scoped to tasks-audit unless you intentionally update all shared helper callers and their compatibility tests. Add focused regressions in the CLI registration and routed-command tests, and cover the JSON command path; do not change task audit finding semantics or task registry storage. No changelog is required unless maintainers want user-facing CLI validation tightening called out.
