Fix https://github.com/openclaw/openclaw/issues/84899 by making `openclaw tasks audit --limit` require a strict positive integer token in both the registered Commander command and the lean `tasks audit --json` route. Reject supplied invalid values such as `1abc`, `1.5`, `0`, and `-1` before running the audit with a clear CLI error; preserve omitted `--limit` behavior. Prefer a scoped strict parser or explicit strict mode so unrelated commands that currently share `parsePositiveIntOrUndefined` are not changed accidentally. Add focused regression coverage for both entry paths, update helper coverage if a strict helper is introduced, and include a `CHANGELOG.md` fix entry because this is user-visible CLI behavior. Coordinate with the adjacent command-validation PR https://github.com/openclaw/openclaw/pull/83391 if maintainers choose to consolidate, but do not depend on that branch.
