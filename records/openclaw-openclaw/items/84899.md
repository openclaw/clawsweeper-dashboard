## Summary

## What This Changes

The issue asks for `openclaw tasks audit --limit` to reject partially numeric, fractional, zero, and negative values instead of silently parsing numeric prefixes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84901 or an equivalent strict parser fix, then close this issue after the merge.

## Reproduction Assessment

Yes. I did not run the CLI in this read-only review, but current source still sends both `tasks audit` entry paths through `Number.parseInt`-based positive integer parsing, matching the reporter's command output.

## Solution Assessment

Yes. The narrow maintainable fix is to use strict positive-integer parsing only for the affected `tasks audit --limit` Commander and lean JSON paths, with focused regression tests; the linked PR follows that shape.
