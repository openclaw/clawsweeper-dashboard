## Summary

Keep open: current main still suppresses unmarked final assistant text for explicit message-tool-only group/channel delivery, and current docs/tests describe that as the configured shared-room contract. The report is actionable, but changing it to auto-fallback or visible warning is a visibility-policy decision for maintainers.

## What This Changes

The issue requests a fallback or visible warning when a group/channel configured for message-tool-only visible replies emits only a final assistant response and sends nothing through the message tool.

## Best Possible Solution

If maintainers accept the policy change, add one shared, documented fallback or warning policy for normal group/channel user requests with no successful message-tool delivery, excluding room events, send-policy denies, and already-delivered side effects, then cover normal dispatch, follow-up dispatch, ACP dispatch, docs, and duplicate-delivery tests.

## Solution Assessment

Unclear: the proposed fallback would address the silent-room symptom, but it changes the current explicit tool-only privacy contract. The maintainable path is a maintainer-approved shared policy helper or warning flow rather than an ad hoc bypass in one dispatch branch.
