## What This Changes

The PR adds creation of `skills` and `.agents/skills` directories in `ensureAgentWorkspace()` so read-only sandbox skill mount sources can exist before container launch.

## Best Possible Solution

Create the host `skills` and `.agents/skills` directories in the writable sandbox layout before any backend, Docker, or browser mount resolution, then prove that path with focused regression coverage and redacted real sandbox output.

## Reproduction Assessment

Yes from source inspection: the rw sandbox setup path creates only the workspace root before mount resolution, and the resolver filters missing skill directories out of the read-only overlay list. I did not run Docker in this read-only review.

## Solution Assessment

No. The proposed helper is not the central owner for every writable sandbox launch, so the safer fix belongs in the rw sandbox setup path before mount resolution.
