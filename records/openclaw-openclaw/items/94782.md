## Summary

Keep open: this PR targets a real sandbox skill-mount bug, but the fix is still in the workspace bootstrap helper instead of the writable sandbox layout path that resolves mounts before container/browser creation, and contributor proof is source/tests only.

## What This Changes

The PR creates `skills` and `.agents/skills` directories in `ensureAgentWorkspace` before bootstrap files are written so read-only sandbox skill mounts have host sources.

## Best Possible Solution

Create the host skill directories in the shared writable sandbox setup path before Docker/browser/FS mount resolution, keep generated empty directories out of workspace wipe/content-evidence guards, and prove the read-only overlay in a real sandbox path.

## Reproduction Assessment

## Solution Assessment

No. Adding the directories in `ensureAgentWorkspace` is only a partial bootstrap-layer fix; the safer fix is in the sandbox layout path that every writable sandbox uses before mount resolution.
