## Summary

This PR is still necessary as a candidate fix for the open sandbox skill-directory bug, but the current patch is not merge-ready because it creates the directories in `ensureAgentWorkspace()` after the no-bootstrap return and misses direct `workspaceAccess: "rw"` sandbox paths that resolve mounts after only creating the workspace root.

## What This Changes

The PR adds two `fs.mkdir(..., { recursive: true })` calls to create `skills` and `.agents/skills` during agent workspace bootstrap.

## Best Possible Solution

Create the host skill directories in the writable sandbox layout before backend, Docker, browser, and bridge mount resolution, with focused regression coverage and redacted real first-launch sandbox proof.

## Reproduction Assessment

Yes, at source level: current main filters missing skill directories out of read-only mount resolution, and the writable sandbox layout can reach that resolver after only creating the workspace root. I did not run a live Docker first-launch reproduction in this read-only review.

## Solution Assessment

No. The proposed location misses no-bootstrap and direct writable-sandbox paths; the narrow maintainable fix belongs in the writable sandbox layout before mount resolution.
