## What This Changes

## Best Possible Solution

Create the host skill directories in the writable sandbox setup path before Docker/browser mount resolution, then cover that path with caller-level regression tests and real sandbox proof when feasible.

## Reproduction Assessment

Yes, from source inspection: rw sandbox setup creates only the workspace root, and the mount resolver filters missing skill directories before Docker/browser creation. I did not run a live Docker reproduction in this read-only review.

## Solution Assessment

No. The proposed location is too narrow because the rw sandbox layout path can reach mount resolution without running the new `ensureAgentWorkspace()` directory creation.
