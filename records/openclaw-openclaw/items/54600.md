## Summary

Keep open. Current main still forwards isolated cron `agentTurn` messages unchanged, so the reported bug remains valid and this PR is plausible source work, but the proposed patch should not merge as-is because its fallback logs serialized cron prompt content.

## What This Changes

This PR modifies `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from `text`, `message`, or `content`, then JSON-stringify and debug-log unrecognized shapes before calling the isolated cron runner.

## Best Possible Solution

Land a tested cron fix that preserves the public `message: string` contract: normalize or repair malformed stored payloads where appropriate, keep a narrow defensive extraction at the isolated execution boundary, and avoid logging prompt content.

## Reproduction Assessment

Yes. A runtime `agentTurn` cron job with an object-valued `payload.message` reproduces the current-main failure path because the timer forwards it unchanged and the isolated runner interpolates it into the prompt; the linked issue also provides a manual cron add/run scenario, though the object-shape source still needs confirmation.

## Solution Assessment

No, not as written. Defensive extraction is a reasonable part of the fix, but the fallback should log only anomaly shape metadata or keys, not serialized prompt content, and should be covered by a focused regression test.
