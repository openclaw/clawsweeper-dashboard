## Summary

Keep this PR open: current main still has the isolated cron message forwarding bug, and the PR is useful source work, but it should not merge as written because its fallback logs serialized prompt content and the user-facing fix is missing a changelog entry.

## What This Changes

This PR updates `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from common string fields, stringify unrecognized values, and pass the resulting string to the isolated runner.

## Best Possible Solution

Keep the public cron payload schema string-valued and add a focused, tested defensive extractor at the isolated execution boundary that never writes prompt content to logs.

## Reproduction Assessment

Yes by source inspection. An isolated `agentTurn` cron job with an object-valued `payload.message` reaches `runIsolatedAgentJob` unchanged on current main and is interpolated into the prompt string, which reproduces the `[object Object]` failure mode for plain objects.

## Solution Assessment

No, not as written. Defensive extraction is the right narrow direction, but the fallback should avoid logging prompt content, the branch should be rebased onto the current detached execution boundary, and the user-facing fix needs regression coverage and a changelog entry.
