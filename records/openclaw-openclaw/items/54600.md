## Summary

Keep open. Current `main` still has the reported cron message-forwarding failure path, but this PR should not merge as-is because its fallback debug log can expose serialized cron prompt content and the branch needs to be rebased onto the current detached execution boundary with focused regression coverage.

## What This Changes

This PR modifies `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from `text`, `message`, or `content`, then JSON-stringify and debug-log unrecognized shapes before calling the isolated cron runner.

## Best Possible Solution

Land a tested cron fix that preserves the public `message: string` contract, normalizes or defensively unwraps malformed stored payloads at the current execution boundary, and logs only shape metadata or keys for anomalies.

## Reproduction Assessment

Yes. A runtime `agentTurn` cron job with an object-valued `payload.message` reproduces on current main because the timer forwards the value unchanged and the isolated runner interpolates it into a prompt template.

## Solution Assessment

No, not as written. Defensive extraction is a reasonable part of the fix, but the fallback should avoid logging prompt content and the patch should be rebased onto the current detached execution path with regression coverage.
