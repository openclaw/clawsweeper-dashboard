## Summary

Keep open: current main partially mitigates the empty expected-model symptom, but `createWithAdapter()` still returns an embedding provider with an empty `provider.model` unchanged, and the focused source-layer backfill remains open for maintainer review.

## What This Changes

The issue reports that memory search identity checks and metadata writes can use an empty embedding provider model, leaving vector search permanently dirty after gateway/runtime paths run.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/91660 for the source-layer provider.model backfill, then track any still-reproducible agent-session metadata deletion separately.

## Reproduction Assessment

Yes. A source-level reproduction remains on current main: register a memory embedding adapter with a `defaultModel` whose `create()` returns `provider.model = ""`, and `createWithAdapter()` returns that provider unchanged.

## Solution Assessment

Yes for the narrow bug: backfilling `provider.model` in `createWithAdapter()` is the clean owner-boundary fix compared with adding downstream fallbacks at every consumer. It is not necessarily sufficient for the broader session-path metadata deletion reports.
