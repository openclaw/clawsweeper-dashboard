## Summary

Keep open: current main has the merged guard that prevents an empty expected-model string, but the source-layer provider creation path still returns `result.provider` unchanged and the focused backfill PR remains open for maintainer review.

## What This Changes

The issue reports that `memory_search` can leave the memory index permanently dirty when the gateway/session path uses an empty embedding provider model for index identity and metadata.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/91660 for the provider.model backfill, then keep broader session-path metadata deletion work tracked separately if it still reproduces.

## Reproduction Assessment

Yes for a source-level reproducer: register an embedding adapter with a `defaultModel` whose `create()` returns `provider.model = ""`, and current main returns that provider unchanged from `createWithAdapter()`. A live gateway reproduction was not run in this read-only review.

## Solution Assessment

Yes for the narrow bug: backfilling `provider.model` in `createWithAdapter()` is the clean owner-boundary fix compared with downstream fallbacks at every consumer. It may not be sufficient for the broader agent-session metadata deletion reports.
