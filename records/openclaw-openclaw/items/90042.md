## Summary

Keep open: current main partially mitigates the empty expected-model symptom, but `createWithAdapter()` still returns an adapter-provided empty `provider.model` unchanged, and the focused source-layer backfill remains in an open PR.

## What This Changes

The issue reports that `memory_search` can stay dirty or corrupt memory index identity when embedding provider/model resolution leaves `provider.model` empty during gateway or agent-session paths.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/91660 for the source-layer provider.model backfill, then keep broader memory-session metadata deletion reports separate.

## Reproduction Assessment

Yes. A source-level reproduction remains on current main: register a memory embedding adapter with a `defaultModel` whose `create()` returns `provider.model = ""`, and `createWithAdapter()` returns that provider unchanged.

## Solution Assessment

Yes for the narrow bug. Backfilling `provider.model` in `createWithAdapter()` is the clean owner-boundary fix compared with adding downstream fallbacks at every consumer, but it is not necessarily sufficient for the broader session-path metadata deletion reports.
