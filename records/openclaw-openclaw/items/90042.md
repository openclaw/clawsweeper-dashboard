## Summary

Keep open: current main has the merged empty-expected-model guard, but it still returns the embedding provider object from `createWithAdapter()` without backfilling an empty `provider.model`, and the focused source-layer fix remains open for maintainer review.

## What This Changes

The issue requests a fix for `memory_search` indexes becoming dirty when an empty embedding `provider.model` mismatches or overwrites persisted memory index identity.

## Best Possible Solution

Review and either land or reject the focused provider.model backfill in https://github.com/openclaw/openclaw/pull/91660, then track any still-reproducible agent-session metadata deletion separately.

## Reproduction Assessment

Yes for the remaining source gap: register a memory embedding adapter with a `defaultModel` whose `create()` returns `provider.model = ""`, and current main returns that provider unchanged from `createWithAdapter()`. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes for the narrow bug: backfilling `provider.model` in `createWithAdapter()` is the clean owner-boundary fix compared with adding downstream fallbacks at every consumer. It is not necessarily sufficient for the broader session-path metadata deletion reports.
