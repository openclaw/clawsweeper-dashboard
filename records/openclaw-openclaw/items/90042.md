## Summary

Keep open: current main has the merged empty-expected-model guard, but `createWithAdapter()` still returns the adapter's provider object without backfilling an empty `provider.model`, and the focused source-layer fix remains open for maintainer review.

## What This Changes

The issue asks memory-core to stop an empty embedding `provider.model` from making gateway memory indexes permanently dirty and corrupting index identity/meta state.

## Best Possible Solution

Review and land a narrow source-layer fix that normalizes the created embedding provider identity before memory manager code writes or queries index metadata.

## Reproduction Assessment

Yes at source level, though not with a live local gateway run here. Current main preserves an adapter-returned empty `provider.model` while downstream meta writes and search filters use that field, matching the reporter and commenter reproductions.

## Solution Assessment

Yes: fixing provider creation to backfill the resolved model is the narrowest maintainable repair. Adding fallbacks at every downstream `this.provider.model` use would leave multiple identity sources and more drift risk.
