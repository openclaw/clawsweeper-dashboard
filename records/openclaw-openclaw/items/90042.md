## Summary

Keep open: current main has a merged consumer-side empty-model guard, but the provider creation path still returns `result.provider` unchanged, and an open focused PR is already queued for maintainer review to backfill empty `provider.model`.

## What This Changes

The issue reports `memory_search` staying dirty because an embedding provider can expose an empty `provider.model`, corrupting memory index identity and vector search state.

## Best Possible Solution

Land or reject the focused provider-creation backfill in https://github.com/openclaw/openclaw/pull/91660, then retest the session-path metadata deletion and split any remaining failure to a narrower follow-up.

## Reproduction Assessment

No live high-confidence current-main gateway reproduction was run. Source inspection still shows the remaining backfill gap: current main resolves a model for adapter creation, returns `result.provider` unchanged, and downstream memory identity/cache/search/write paths consume `provider.model`.

## Solution Assessment

Yes for the narrow provider.model bug: backfilling in `createWithAdapter()` is the cleanest owner-boundary fix. It is not necessarily sufficient for the later session-path metadata deletion report, which should be retested after the backfill decision.
