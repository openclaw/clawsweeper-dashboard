## Summary

Keep open: current main has the merged empty-expected-model guard, but the source-layer provider creation path still returns an adapter-supplied empty `provider.model`, and a focused open PR already targets that remaining fix.

## What This Changes

The issue reports that gateway memory search can keep a vector index dirty because embedding provider model identity is empty or unavailable during runtime identity checks and metadata writes.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/91660 for the source-layer provider.model backfill, then track any remaining agent-session metadata deletion under narrower memory identity reports.

## Reproduction Assessment

Yes. A source-level reproduction remains on current main: a memory embedding adapter with a default model can return `provider.model = ""`, and `createWithAdapter()` returns that provider unchanged; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes for the narrow bug: backfilling `provider.model` at provider creation is the clean owner-boundary fix compared with adding fallbacks at each downstream identity, write, and search consumer. The broader session-path metadata deletion reports may need separate follow-up after this source fix is decided.
