## Summary

Keep this open: current main still has a plausible source path where a runtime-only continuation in a WhatsApp group can finish with session-only final text unless the resumed turn sends through the message tool, and the broader compaction tracker does not clearly replace this narrower delivery-contract issue.

## What This Changes

The issue asks OpenClaw to deliver or visibly fail final answers produced by internal continuation for a WhatsApp-bound session.

## Best Possible Solution

Define one channel-bound continuation delivery contract: preserve or derive the source delivery context, require message-tool delivery evidence when group policy is message-tool-only, and visibly fail or warn when a final answer cannot be delivered.

## Reproduction Assessment

No; there is not a high-confidence live reproduction from this review. Source inspection on current main does show a plausible path where a runtime-only WhatsApp group continuation can finish session-only unless the resumed turn sends through the message tool.

## Solution Assessment

Unclear; the desired end state is correct, but the maintainable fix needs a policy decision across internal continuation, active requester steering, and group message-tool-only delivery. A shared delivery-evidence guard is safer than a WhatsApp-only auto-send special case.
