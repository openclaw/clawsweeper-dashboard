## Summary

Keep open: the linked bug is source-reproducible, but this PR's fix changes Silent into the same isolated/no-delivery agent-turn shape as Independent session instead of preserving the documented Silent-vs-Independent distinction.

## What This Changes

This PR changes the Control UI cron quick-create Silent delivery preset from main-session/no-delivery to isolated-session/no-delivery and adds a focused regression test for that mapping.

## Best Possible Solution

Preserve Silent as a backend-supported no-notification preset without collapsing it into Independent session, likely by emitting main/systemEvent/none unless maintainers deliberately redefine the preset copy and behavior.

## Reproduction Assessment

Yes, source-level. Current main maps Silent quick-create to a main-session agentTurn job, and the backend rejects main-session cron jobs unless the payload is systemEvent; I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

No. The patch makes the job backend-valid by changing Silent into isolated/no-delivery, but the narrower maintainable fix is to keep Silent distinct as a supported main-session systemEvent job unless maintainers deliberately redefine the preset.
