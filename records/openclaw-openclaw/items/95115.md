## Summary

Keep open, but not merge-ready: the linked bug is source-reproducible, while this patch changes Silent into the same isolated agent-turn/no-delivery shape as Independent session and does not prove the real Control UI/Gateway creation flow.

## What This Changes

The PR changes the Control UI cron quick-create Silent delivery preset from `sessionTarget: "main"` to `"isolated"` and adds a node test asserting the silent preset maps to isolated/no-delivery/now.

## Best Possible Solution

Fix quick-create so Silent creates a backend-supported job while preserving its distinct main-session/system-event semantics, or explicitly update UI/docs if maintainers decide Silent should mean isolated/no-delivery.

## Reproduction Assessment

Yes, source-level: current main maps Silent quick-create to a main-session `agentTurn` job, `addCronJob` forwards that shape, and the cron service rejects main-session cron jobs unless the payload is `systemEvent`. I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

No. The patch makes the request backend-valid by changing Silent into isolated/no-delivery, but the narrower maintainable fix is to preserve Silent's distinct supported semantics or get maintainer direction to redefine it.
