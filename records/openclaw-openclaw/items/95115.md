## Summary

Keep open: current main and the latest release still contain the invalid Silent quick-create mapping, and this PR is a focused candidate fix with regression coverage and no blocking code finding from this review.

## What This Changes

The PR changes the Control UI cron quick-create Silent preset from `main` to `isolated` session targeting and adds a regression assertion for the Silent preset mapping.

## Best Possible Solution

Land one focused mapping/test fix after CI is resolved, then close the linked bug issue; a later UX copy cleanup can separate Silent and Independent wording if maintainers want those choices to feel less overlapping.

## Reproduction Assessment

Yes. Source inspection shows current main maps Silent quick-create to a `main` `agentTurn` job, `addCronJob` sends that shape to `cron.add`, and the backend rejects main-session cron jobs unless the payload is `systemEvent`.

## Solution Assessment

Yes. The quick-create wizard describes an agent prompt flow, so changing Silent to isolated/no-delivery keeps the `agentTurn` payload backend-supported and is narrower than weakening backend validation or converting the prompt into a main-session system event.
