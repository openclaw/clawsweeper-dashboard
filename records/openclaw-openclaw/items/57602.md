This PR should stay open for maintainer/contributor follow-up, not close cleanup: it contains plausible narrow observability work, but the current diff still has two concrete blockers and the linked broad issue has already been handled on current main by a different path.

## What This Changes

The PR raises some agent tool handler failures from debug to warn, adds skipped block-reply timeout payload context, and changes agent event visibility for error/tool streams.

## Best Possible Solution

Keep only the narrow diagnostics that remain valuable: make timeout payload context effective through a supported logging path, decide whether tool handler failures should be warn-level in the current scheduler, and leave hidden tool/error streams behind the existing visibility gate unless a maintainer approves a narrower terminal-event exception.
