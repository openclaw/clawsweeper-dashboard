## Summary

Keep open. The diagnostics goal is useful, but this PR still has two concrete blockers: one logging change is ineffective with the current helper contract, and one visibility change widens a hidden-run session routing boundary.

## What This Changes

The PR promotes embedded Pi tool handler failures from debug to warn with metadata, adds skipped block-reply timeout context, and makes error/tool agent events retain session visibility metadata.

## Best Possible Solution

Land only the narrow diagnostics: make skipped timeout context effective without leaking full payloads, route embedded handler failures through the current scheduler logging path, and keep hidden tool/error streams behind the existing visibility gate unless a specific terminal lifecycle exception is justified.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: the PR passes metadata to a one-argument `logVerbose`, and changing the hidden-run condition to all tool events would contradict current tests that redact non-lifecycle `sessionKey`.

## Solution Assessment

No. The PR moves toward useful observability, but the safest maintainable fix is narrower: serialize or intentionally extend timeout logging, and preserve the current hidden-run boundary for non-lifecycle tool events.

## Review Findings

Overall correctness: patch is incorrect
