## What This Changes

The issue reports a regression in OpenClaw 2026.5.7 where the Discord bot connects and appears online but never reaches gateway READY, so guild channel messages are not received.

## Best Possible Solution

Land a focused Discord plugin fix for the IDENTIFY wait/reconnect race, add null-socket regression coverage, and confirm with redacted runtime proof that READY fires and guild `MESSAGE_CREATE` events reach the handler on a real Discord setup.
