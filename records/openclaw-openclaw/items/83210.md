## Summary

Keep open: current main still builds the CLI runner system prompt with an empty prompt-visible tool list even when bundle MCP is enabled, while the loopback MCP server exposes policy-filtered gateway tools. That makes plugin prompt sections gated on available tools, including memory-core Memory Recall, source-proven missing for claude-cli runs.

## What This Changes

The issue asks OpenClaw to pass the CLI loopback tool surface into buildCliAgentSystemPrompt so plugin prompt sections such as Memory Recall reflect tools the CLI can actually call.

## Best Possible Solution

Compute a prompt-visible CLI tool surface from the existing loopback scoped resolver, pass it into the CLI system prompt and prompt report, and invalidate/reseed CLI sessions when that prompt tool surface changes.

## Reproduction Assessment

Yes: source inspection on current main follows tools: [] through buildCliAgentSystemPrompt into an empty availableTools set, while memory-core intentionally returns no Memory Recall section for that set. I did not execute a live claude-cli session.

## Solution Assessment

Yes: aligning CLI prompt tools with the same loopback resolver that serves tools/list is the narrow maintainable fix. The fix should avoid hardcoded memory tool names and include session-reuse coverage for changed prompt/tool surfaces.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/83210. In src/agents/cli-runner/prepare.ts, compute prompt-visible tools for bundle-MCP CLI runs from the same gateway-scoped loopback tool surface used by McpLoopbackToolCache/resolveGatewayScopedTools, respecting sessionKey, message provider/channel, accountId, inbound event kind, senderIsOwner, surface: "loopback", and native tool exclusions. Pass those tools to buildCliAgentSystemPrompt and buildSystemPromptReport instead of [] so memory-core and other plugin prompt sections see the tools the CLI can call. Do not hardcode memory_search/memory_get and do not advertise native tools the CLI cannot invoke. Because claude-cli uses systemPromptWhen: "first", include the prompt tool surface or full generated prompt in CLI session reuse invalidation so existing sessions refresh when this guidance changes. Add focused regression coverage for bundle-MCP CLI runs including Memory Recall when memory_search/memory_get are exposed, and for disabled-tools/override paths that must not advertise loopback tools.
