## Summary

Keep open: current main still builds the CLI backend system prompt with `tools: []`, while the bundled MCP loopback resolves a real scoped tool surface, so plugin prompt sections such as Memory Recall can still be omitted for claude-cli runs.

## What This Changes

The issue reports that CLI-backed agent prompt construction passes an empty tool list, suppressing plugin prompt sections such as memory-core Memory Recall even though loopback MCP exposes those tools.

## Best Possible Solution

Compute the CLI prompt-visible tool list from the same loopback-scoped resolver used by MCP tools/list, pass it into `buildCliAgentSystemPrompt`, and invalidate or reseed CLI sessions when that prompt tool surface changes.

## Reproduction Assessment

Yes. Source inspection on current main follows `tools: []` through `toolNames` into an empty `availableTools` set, while memory-core returns no Memory Recall section for that set and the loopback MCP path separately resolves real tools.

## Solution Assessment

Yes. Aligning CLI prompt tools with the loopback resolver is the narrow maintainable fix; avoid hardcoding memory tool names and include coverage for prompt/tool-surface changes across session reuse.
