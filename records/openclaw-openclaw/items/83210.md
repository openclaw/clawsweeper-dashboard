## Summary

Keep open: current main still builds the CLI system prompt with an empty tool list while the bundled MCP loopback exposes policy-filtered gateway tools, so plugin prompt sections such as Memory Recall can be omitted for claude-cli runs.

## What This Changes

Fix claude-cli prompt preparation so plugin-owned prompt sections receive the same policy-filtered MCP tool list exposed through the loopback, especially memory_search and memory_get.

## Best Possible Solution

Compute a prompt-visible CLI tool surface from the loopback resolver at prepare time, pass it into buildCliAgentSystemPrompt, and reset or reseed CLI sessions when that prompt tool surface changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: source inspection on current main follows tools: [] through toolNames to an empty availableTools set, while memory-core returns no Memory Recall section for that set; I did not execute a live claude-cli turn.

## Solution Assessment

Is this the best way to solve the issue? Yes, aligning CLI prompt tools with the loopback resolver is the narrow maintainable fix; avoid hardcoding memory tool names and include session-reuse coverage for changed prompt/tool surfaces.
