## Summary

Keep open: current main and the latest shipped tag still build the CLI system prompt with an empty tool list while the bundled MCP loopback exposes scoped tools, so plugin prompt sections such as Memory Recall can be omitted for claude-cli runs.

## What This Changes

The issue requests aligning the claude-cli system prompt tool list with the loopback MCP tool surface so memory-core prompt guidance is injected when memory tools are available.

## Best Possible Solution

Resolve the loopback-scoped tool surface during CLI prepare, pass it into buildCliAgentSystemPrompt and the system-prompt report, and include that prompt tool surface in CLI session reuse invalidation.

## Reproduction Assessment

Yes, from source inspection: tools: [] flows through buildCliAgentSystemPrompt into an empty availableTools set, and memory-core returns no Memory Recall section for that set. I did not execute a live claude-cli turn.

## Solution Assessment

Yes, aligning the CLI prompt tool list with the loopback resolver is the narrow maintainable fix. Avoid hardcoding memory tool names and include coverage for prompt/tool-surface changes across session reuse.
