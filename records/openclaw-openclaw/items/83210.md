## Summary

Keep open. Current main and v2026.5.12 still build the CLI-backed agent system prompt with an empty tool list while the MCP loopback exposes gateway-scoped tools, so plugin-owned prompt sections that depend on `availableTools`, including Memory Recall, can be suppressed for CLI agents.

## What This Changes

The issue requests that CLI-backed agent system prompts receive the same gateway-scoped tool names exposed through the MCP loopback so plugin prompt sections such as Memory Recall are injected when their tools are available.

## Best Possible Solution

Unify CLI system-prompt tool awareness with the MCP loopback’s scoped tool resolution, then add focused regression coverage proving Memory Recall appears only when the loopback memory tools are actually available.

## Reproduction Assessment

Yes from source: `prepareCliRunContext` passes `tools: []` into the CLI prompt builder, and memory-core emits no Memory Recall section when `availableTools` lacks `memory_search` or `memory_get`. I did not execute a live `claude-cli` turn in this read-only pass.

## Solution Assessment

Yes. Resolving and reusing the same loopback-scoped tool surface used by MCP `tools/list` before building the CLI prompt is the narrow maintainable fix, provided it preserves native-tool exclusion and owner-only policy.
