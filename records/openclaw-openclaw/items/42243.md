## Summary

Keep open: current main still turns fatal cron/provider failures into `isError` delivery payloads and passes them through the normal announce delivery path; the related contributor PR is closed unmerged and did not land a fix.

## What This Changes

The issue asks OpenClaw to stop cron `delivery.mode: "announce"` from posting provider or run failure text to Discord/Telegram when the run fails.

## Best Possible Solution

Suppress normal cron completion announce for fatal/error isolated runs using the existing structured fatal state, while keeping run history, diagnostics, and separate failure notifications intact.

## Reproduction Assessment

Yes, at source level. Current main builds fatal `isError` cron delivery payloads, passes them to `dispatchCronDelivery`, and outbound delivery still sends text-only error payloads normally.

## Solution Assessment

Yes. The narrow maintainable fix is to suppress normal completion announce when the existing fatal/error state is set; the closed raw-text guard PR was weaker because it missed structured `deliveryPayloads`.
