## Summary

Keep open. Current main still synthesizes fatal cron/provider failures into `isError` delivery payloads and passes them through normal announce delivery; the related contributor PR is closed unmerged, and the related sandbox issue fixed a different auto-reply path.

## What This Changes

The issue asks OpenClaw to stop cron `delivery.mode: "announce"` from posting raw provider error JSON or fatal run errors to Discord, Telegram, or other user-facing channels.

## Best Possible Solution

Use the structured fatal cron state to skip normal completion announce delivery for failed isolated runs, while preserving run history, diagnostics, and separate failure notifications.

## Reproduction Assessment

Yes, at source level. Current main synthesizes fatal cron outcomes into `isError` delivery payloads, passes them into `dispatchCronDelivery`, and outbound delivery sends text-only error payloads through normal channel delivery.

## Solution Assessment

Yes. The narrow maintainable fix is to suppress normal completion announce when the existing structured fatal/error state is set; a raw-text regex guard is weaker and the closed contributor PR already showed that it missed payload paths.
