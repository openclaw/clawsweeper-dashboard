## Label Justifications

- `P1`: The issue affects a real scheduled channel delivery workflow and can expose raw provider error text to external recipients.
- `impact:security`: Raw provider and operational error JSON can disclose internal details in Discord, Telegram, or other configured channels.

## Summary

Keep open: current main and the latest release still route fatal cron error payloads into normal announce delivery, and the linked fix PR is open but not merged.

## What This Changes

The issue asks OpenClaw to stop cron announce delivery from posting raw provider error JSON to external channels when a run fails.

## Best Possible Solution

Review and land the linked focused fix after real cron/channel proof shows failed runs are recorded without posting raw provider JSON to the configured announce channel.

## Reproduction Assessment

Yes, at source level. Current main turns fatal `isError` cron payloads into `deliveryPayloads`, passes them to `dispatchCronDelivery`, and the outbound contract sends text-only error payloads normally.

## Solution Assessment

Yes. The best fix is to gate fatal structured cron errors before normal announce completion delivery while keeping run diagnostics and separate failure notifications intact.
