## Summary

Keep open: current main and the latest release still allow fatal structured cron error payloads to proceed into normal announce delivery, and the linked closing PR is open but not merged.

## What This Changes

The issue requests that failed cron runs using announce delivery record errors internally without posting raw provider error JSON into configured channels.

## Best Possible Solution

Review and land the focused linked fix after confirming failed cron runs remain recorded while raw provider JSON is not posted through announce delivery.

## Reproduction Assessment

Yes, at source level. Current main still turns fatal structured cron `isError` payloads into deliverable `deliveryPayloads` and calls the normal announce dispatch path.

## Solution Assessment

Yes. The narrow maintainable fix is to suppress fatal structured cron completion payloads before normal announce delivery while preserving internal run diagnostics and separate controlled failure notifications.
