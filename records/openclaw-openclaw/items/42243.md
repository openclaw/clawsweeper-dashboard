## Summary

Keep open. Current main and the latest release still turn fatal cron run failures into `isError` delivery payloads and pass them through normal announce delivery, while the directly linked fix PR is closed unmerged.

## What This Changes

Failed cron runs with `delivery.mode: "announce" should not deliver raw provider or run error text to configured Discord or Telegram channels.

## Best Possible Solution

Add a structured fatal-run skip before normal completion announce dispatch, keeping run history, diagnostics, and failure notifications intact.

## Reproduction Assessment

Yes, at source level. A provider or run-level failure becomes an `isError` delivery payload, current main passes that payload into `dispatchCronDelivery`, and dispatch sends normal announce payloads without a fatal-status guard.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing structured fatal/error state to skip normal completion announce delivery for failed runs; a raw-text regex guard like the closed contributor PR is less robust.
