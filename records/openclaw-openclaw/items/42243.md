## Summary

Keep open. Current main still converts fatal isolated cron failures into `isError` delivery payloads and passes them through normal announce delivery, while the separate failure-notification path does not suppress the completion announce payload.

## What This Changes

The issue reports that failed cron runs with announce delivery can post raw provider error JSON to configured Discord or Telegram channels.

## Best Possible Solution

Add a structured fatal-run skip before normal completion announce dispatch, while preserving run history, diagnostics, and separate failure-notification delivery.

## Reproduction Assessment

Yes, at source level. Fatal cron outcomes become `isError` delivery payloads, current main passes them into `dispatchCronDelivery`, and dispatch sends normal announce payloads without a fatal-status guard.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing structured fatal/error state to skip normal completion announce delivery for failed runs; a raw-text regex guard like the closed contributor PR is less robust.
