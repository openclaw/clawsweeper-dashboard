## Summary

Keep open. Current main still creates fatal cron error delivery payloads and passes them through normal announce dispatch; related fixes sanitize some provider errors and add separate failure notifications, but they do not suppress failed-run completion announces.

## What This Changes

Suppress normal `delivery.mode: "announce"` channel delivery for failed cron runs so provider/run error payloads stay in diagnostics instead of external channels.

## Best Possible Solution

Use the structured fatal-run state to skip normal completion announce delivery for failed isolated cron runs, while preserving run history, diagnostics, and separate failure notifications.

## Reproduction Assessment

Yes, at source level. Fatal cron outcomes become `isError` delivery payloads, current main passes them into `dispatchCronDelivery`, and dispatch/outbound has no fatal-run guard before sending.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing structured fatal/error state to suppress normal completion announce delivery for failed runs; a raw-text regex guard is less robust and was already shown to miss payload paths.
