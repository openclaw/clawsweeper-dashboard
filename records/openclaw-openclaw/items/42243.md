## Summary

Keep open: current main and v2026.5.18 still allow fatal cron error payloads into the normal announce delivery path, and the open closing PR is the right implementation candidate to review before closing this issue.

## What This Changes

The issue asks OpenClaw to stop posting raw provider or cron error JSON to configured announce channels when a scheduled isolated cron run fails.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83724 or an equivalent narrow fix that suppresses normal completion announce for fatal structured cron errors while preserving run history and failure notifications.

## Reproduction Assessment

Yes, at source level. Current main turns fatal cron/provider failures into `isError` delivery payloads, passes them into `dispatchCronDelivery`, and outbound delivery sends text-only error payloads normally.

## Solution Assessment

Yes. The maintainable fix is to suppress normal completion announce using the existing fatal structured error state before dispatch, while leaving run diagnostics and failure notifications intact.
