## Summary

Keep this open. Current main still turns fatal cron outcomes into normal announce delivery payloads, so the raw/error text delivery route remains present despite related failure-notification and provider-error sanitization work.

## What This Changes

The issue requests that failed cron runs with announce delivery stop posting raw provider error JSON to Discord or Telegram task channels.

## Best Possible Solution

Use the structured fatal/error outcome to skip normal announce completion delivery for failed cron runs, while preserving run history and the separate failure notification path.

## Reproduction Assessment

Yes, source-reproducible: fatal cron output is converted to an isError delivery payload and current main dispatches it through announce delivery. I did not perform a live channel delivery because this pass is read-only.

## Solution Assessment

Yes, the narrow fix is to gate normal announce completion delivery on the existing structured fatal/error state rather than parsing raw provider text. Failure notifications should stay on their documented path.
