## Summary

Keep open. Current main supports skill source precedence, allowlists, env/API-key overrides, and an untyped per-skill config bag, but it does not standardize or auto-load user preferences.md or conventions.md alongside a skill; this needs maintainer product/API review rather than an automated cleanup close.

## What This Changes

Request official user-scoped per-skill preferences or conventions, preferably a markdown file loaded only when the matching skill is used.

## Best Possible Solution

Define a maintainer-approved skill preferences contract with file name, precedence, load timing, watcher/snapshot invalidation, docs, and security boundaries, then implement it once across loader, prompt, runtime refresh, and tests.

## Reproduction Assessment

Yes for a source-level feature gap, not a bug reproduction: current main loads SKILL.md skill entries and config/env overrides, but no checked path auto-loads preferences.md or conventions.md as first-class skill context.

## Solution Assessment

Unclear until maintainers choose the contract. A markdown convention may be the right user-facing shape, but a loader-only patch would be too narrow without precedence, refresh, and security semantics.
