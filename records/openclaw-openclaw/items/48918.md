## Summary

Keep open. Current main and latest release still expose a SKILL.md-only skill contract for loading, watching, prompts, config, and docs, while the related priority, truncation, and Feishu subagent issues are adjacent rather than superseding this skill-scoped preference-file request.

## What This Changes

Request standard support for per-skill user preference or convention files that load only when the relevant skill is selected.

## Best Possible Solution

Define a first-class plain-markdown companion-file contract under existing skill roots, load it at lower precedence than SKILL.md only for the selected skill, refresh snapshots on companion changes, and document precedence, safety, and token-budget semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main and v2026.5.20 give high-confidence evidence that no supported companion preferences or conventions file is loaded, watched, or documented today.

## Solution Assessment

Unclear until maintainers choose the public contract. Option A has the strongest discussion support, but filename, precedence, watcher refresh, prompt-cache behavior, path safety, and safety semantics need product/API review before implementation.
