## Summary

Keep open. Current main and the latest release still have a SKILL.md-only skill loading, prompt, watcher, and docs contract; the adjacent priority and per-user subagent requests do not supersede this skill-scoped preference-file feature.

## What This Changes

The issue asks for a supported per-skill user preferences or conventions file that loads only when that skill is used, without replacing SKILL.md or putting notes in global context.

## Best Possible Solution

Define a first-class plain-markdown companion-file contract under existing skill roots, load it at lower precedence than SKILL.md only for the selected skill, refresh snapshots on companion changes, and document precedence, safety, and token-budget semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main and v2026.5.20 give high-confidence evidence that no supported companion preferences or conventions file is loaded, watched, prompted, or documented today.

## Solution Assessment

Unclear until maintainers choose the public contract. Option A has the strongest discussion support, but filename, precedence, watcher refresh, prompt-cache behavior, path safety, and safety semantics need product/API review before implementation.
