## Summary

Keep open. Current main still implements a SKILL.md-only skill-loading, prompt, and refresh contract, while the issue discussion describes a distinct skill-scoped user context file that is not supported yet. The related skill-priority request is adjacent selection policy work, not a replacement for this companion-file behavior.

## What This Changes

The issue requests a supported per-skill user preferences or conventions file that is loaded only when that skill is selected.

## Best Possible Solution

Define a small core skills contract for a markdown companion file under existing skill roots, load it as lower-precedence local context only for the selected skill, and update discovery, prompt rendering, watcher refresh, sandbox/workspace skill sync expectations, docs, and focused tests. Keep priority-based skill selection as separate work under #50199.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence check is source and docs inspection on current main: companion preference files have no loader, prompt, or watcher contract today.

## Solution Assessment

Unclear until maintainers choose the public contract. The strongest direction from the discussion is Option A, a plain markdown companion file, but implementation should explicitly preserve SKILL.md authority and existing skill path safety rules.
