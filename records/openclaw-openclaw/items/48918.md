## Summary

Keep open. Current main and v2026.6.8 still expose a `SKILL.md`-only skill contract with structured per-skill runtime overrides, but no supported `preferences.md` or `conventions.md` sidecar loaded with a triggered skill; this remains a maintainer product/API decision rather than cleanup-close material.

## What This Changes

Request to add user-level, skill-scoped preference or convention files loaded only when a skill is triggered.

## Best Possible Solution

Define a maintainer-approved skill-preferences contract with filename, precedence, load timing, snapshot/version invalidation, security boundaries, and docs before implementing it across loader, prompt, tests, and public docs.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main still discovers and prompts `SKILL.md` only, and a targeted source/docs search found no supported `preferences.md` or `conventions.md` skill context contract.

## Solution Assessment

Unclear until maintainers choose the contract. A markdown sidecar is plausible, but a loader-only patch would be too narrow without precedence, refresh, security, and docs semantics.
