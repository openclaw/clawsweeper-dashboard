## Summary

Keep open. Current main still has a SKILL.md-only skill loading, prompt, watcher, and docs/config contract; the related priority request at https://github.com/openclaw/openclaw/issues/50199 is adjacent but does not provide skill-scoped user preference context.

## What This Changes

Request to support user-level per-skill preferences or conventions that load only when the matching skill is used.

## Best Possible Solution

Define a small core skills contract for a markdown companion file under existing skill roots, load it only with the selected skill at lower precedence than SKILL.md, and update discovery, prompt rendering, watcher refresh, runtime materialization, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main give high-confidence proof that no supported companion preferences/conventions file is loaded or watched today.

## Solution Assessment

Unclear until maintainers choose the public contract. Option A, a plain markdown companion file, is the best-supported direction in the discussion, but the filename, precedence, refresh, and safety semantics should be settled before implementation.
