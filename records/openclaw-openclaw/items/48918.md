## Summary

Keep open. Current main still has a SKILL.md-only skill loading, prompt, watcher, and documented config contract; the related priority issue is adjacent, not a canonical replacement for skill-scoped preference context.

## What This Changes

The issue requests a supported per-skill user preferences or conventions file that is loaded only when the relevant skill is triggered.

## Best Possible Solution

Define a first-class markdown companion-file contract under existing skill roots, load it at lower precedence than SKILL.md only with the selected skill, refresh snapshots on changes, and document safety/token-budget semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main give high-confidence evidence that no supported companion preferences or conventions file is loaded, watched, or documented today.

## Solution Assessment

Unclear until maintainers choose the public contract. Option A has the strongest discussion support, but filename, precedence, watcher refresh, and safety semantics should be decided before implementation.
