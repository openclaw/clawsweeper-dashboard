## Summary

Keep open. Current main still exposes a SKILL.md-only skill loading, prompt, watcher, and config/docs contract; the related priority issue is adjacent but does not provide skill-scoped user preference context.

## What This Changes

The issue requests official support for user skill-scoped preference or convention files that load only when the corresponding skill is used.

## Best Possible Solution

Define a first-class markdown companion-file contract under existing skill roots, load it at lower precedence than SKILL.md only with the selected skill, refresh snapshots on changes, and document the safety and token-budget semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main give high-confidence proof that no supported companion preferences or conventions file is loaded or watched today.

## Solution Assessment

Unclear until maintainers choose the public contract. Option A has the strongest support in the discussion, but filename, precedence, watcher refresh, and safety semantics should be decided before implementation.
