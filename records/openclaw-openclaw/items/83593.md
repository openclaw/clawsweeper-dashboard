## Summary

Keep open. Current main still has a source-level path where same-agent `sessions_spawn` can carry the caller workspace into the child run, and `agent-command` then calls `ensureAgentWorkspace` on that path; `ensureAgentWorkspace` writes the reported default markdown files before it reads `setupCompletedAt`.

## What This Changes

The issue asks OpenClaw to stop recreating default bootstrap markdown files at the repository workspace root when a subagent is spawned for an agent whose actual workspace files live under a subdirectory.

## Best Possible Solution

Make same-agent subagent runs resolve and preserve the configured target agent workspace unless an explicit sandbox or caller-approved workspace override is present, then add a regression that the repository root stays clean when the active agent workspace is `main/`.

## Reproduction Assessment

Yes, at source level rather than live runtime: a same-agent subagent can inherit `ctx.workspaceDir`, and the child run will seed bootstrap templates at that inherited path before honoring `setupCompletedAt`. I did not replay the Telegram/macOS flow.

## Solution Assessment

Yes, the maintainable fix is to correct the subagent workspace handoff and add a focused regression; using `skipOptionalBootstrapFiles` is only a partial workaround and does not cover `TOOLS.md` or wrong-directory routing.
