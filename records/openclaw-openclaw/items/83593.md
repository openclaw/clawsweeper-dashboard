## What This Changes

The issue asks OpenClaw to stop subagent or session startup from recreating default bootstrap Markdown files in a repository root when the configured agent workspace lives elsewhere.

## Best Possible Solution

Review and land one narrow fix that resolves the spawned or nested workspace directory correctly before workspace seeding, preserves the existing setup/bootstrap contracts, and adds focused regression coverage for a clean repository root.

## Reproduction Assessment

Yes, at source level: same-agent subagent metadata can carry the caller workspace into the child run, and agent-command then calls ensureAgentWorkspace on that inherited path before completion state can stop file writes. ## Solution Assessment

Yes for the issue direction, but not yet for the linked PR implementations. The maintainable solution is to correct workspace resolution or add a narrowly scoped runtime no-seed path while preserving setup/onboarding repair behavior.
