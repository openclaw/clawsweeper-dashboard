## Summary

Keep open: current main still has the source-level path that can seed bootstrap markdown files into the selected workspace before honoring completion state, and the two linked fix PRs are still open rather than merged.

## What This Changes

Prevent subagent or session workspace setup from recreating OpenClaw bootstrap/persona markdown files in a repository root when setup is already complete or the active agent workspace owns those files.

## Best Possible Solution

Review and land a narrow fix that keeps subagent/session setup pointed at the correct agent workspace, avoids seeding repository-root bootstrap files for already configured workspaces, and adds regression coverage for the reported root-plus-main layout and nested worktree trigger.

## Reproduction Assessment

Yes, at source level: same-agent subagent spawning can carry ctx.workspaceDir into spawnedWorkspaceDir, and the child command then calls ensureAgentWorkspace on that path before completion state prevents template writes. I did not replay the live macOS Telegram flow.

## Solution Assessment

Yes, the best fix is in core agent workspace routing/seeding with focused regressions; skipOptionalBootstrapFiles is only a partial workaround because it does not cover required files or wrong-directory selection.
