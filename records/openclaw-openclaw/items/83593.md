## What This Changes

The issue reports that OpenClaw recreates root-level agent bootstrap markdown files in repository/worktree roots during subagent spawn or nested session start.

## Best Possible Solution

Fix the runtime workspace resolution/seeding boundary so subagent and nested session-start paths do not seed repository roots, preserve existing bootstrap opt-out contracts, and land or replace one focused closing PR with regression coverage.

## Reproduction Assessment

Yes, at source level: current main can route spawned metadata into `agent-command`, and `ensureAgentWorkspace` writes the reported files before honoring `setupCompletedAt`. I did not replay the live Telegram/macOS flow.

## Solution Assessment

Yes, the best fix is a narrow workspace handoff/seeding repair with regression coverage; broad changes to shared workspace setup policy should be avoided unless maintainers explicitly choose that contract.
