## Summary

Keep open. Current main still has a source-level path where same-agent subagent spawning can preserve the caller workspace into the child run, and the child agent command then calls workspace seeding on that path before setup completion state is consulted; two open PRs already reference this issue as fix candidates.

## What This Changes

Prevent subagent spawning from recreating default bootstrap markdown files at the repository/workspace root when the active agent workspace files live under a per-agent directory such as `main/`.

## Best Possible Solution

Land one narrow fix that preserves the effective per-agent workspace for same-agent spawned runs or prevents optional template seeding in established workspaces, with a regression proving the repository root stays clean when `main/` holds the bootstrap files.

## Reproduction Assessment

Yes, at source level: same-agent `sessions_spawn` can inherit `ctx.workspaceDir`, the child run prefers stored spawned workspace metadata, and `ensureAgentWorkspace` seeds files there before reading `setupCompletedAt`. I did not replay the macOS Telegram flow.

## Solution Assessment

Yes, the maintainable direction is a narrow workspace-handoff or established-workspace seeding fix plus focused regression coverage; the existing skip-optional config is only a workaround and does not cover `TOOLS.md` or wrong-directory routing.
