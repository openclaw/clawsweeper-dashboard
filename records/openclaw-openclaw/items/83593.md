## Summary

Keep open. Current main still has a source-level path where same-agent subagent spawns can persist the caller workspace and then seed bootstrap markdown files into that inherited directory before honoring completed setup state.

## What This Changes

The issue requests that subagent spawn stop recreating default bootstrap/persona markdown files at the repository workspace root when the active agent workspace lives under `main/`.

## Best Possible Solution

Fix same-agent subagent workspace handoff so child runs resolve the configured agent workspace unless an explicit `cwd` or sandbox workspace is intended, then add a regression that the repository root remains clean when `main/` contains the bootstrap files.

## Reproduction Assessment

Yes, at source level: same-agent spawn can inherit `ctx.workspaceDir`, persist it as `spawnedWorkspaceDir`, and the child run seeds bootstrap files at that path before checking completed setup state. I did not run the Telegram/macOS scenario live.

## Solution Assessment

Yes, the best fix is a narrow workspace handoff correction plus a focused regression; relying on `skipOptionalBootstrapFiles` would be incomplete because required files still write and the wrong-directory routing remains.
