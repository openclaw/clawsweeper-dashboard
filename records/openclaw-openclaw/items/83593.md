## Summary

Keep open. Current main and the latest release still have a source-level path where spawned or nested workspace runs can seed bootstrap markdown files into the wrong directory, and two open linked fix PRs already own proposed repairs, so this issue should close only after a fix lands.

## What This Changes

The issue asks OpenClaw to stop subagent/session workspace bootstrap from creating AGENTS/SOUL/TOOLS/IDENTITY/USER/HEARTBEAT files in the repository root when the real agent workspace or completed workspace state lives elsewhere.

## Best Possible Solution

Land one narrow workspace-boundary fix that prevents spawned or nested project sessions from treating the repository root as the agent bootstrap root, while preserving intentional required-file repair in the actual agent workspace.

## Reproduction Assessment

Yes, at source level: current main shows the spawned workspace override can reach ensureAgentWorkspace, which writes the reported bootstrap files before consulting workspace completion state. I did not replay the Telegram/macOS runtime flow.

## Solution Assessment

Yes, the best fix is to correct workspace resolution and parent/completed-workspace handling, then add regression coverage. Relying only on skipOptionalBootstrapFiles is not sufficient because AGENTS.md/TOOLS.md and wrong-directory state creation remain.
