## What This Changes

Spawning a subagent should stop recreating bootstrap markdown templates at the workspace repository root when the active agent workspace lives under `main/` and setup is complete.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83608 with a narrow fix that corrects subagent workspace handoff or runtime-only bootstrap seeding while preserving explicit setup behavior and adding a regression for a clean repository root.

Yes, at source level: same-agent `sessions_spawn` can inherit `ctx.workspaceDir`, and the child run then calls `ensureAgentWorkspace` on that inherited path before setup state prevents template writes. I did not replay the full Telegram/macOS flow.

## Solution Assessment
