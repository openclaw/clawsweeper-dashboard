## Summary

Keep open. Current main still has a source-level path where same-agent subagent spawning can preserve the caller workspace as spawned metadata, and the child run then calls `ensureAgentWorkspace` on that path, which still writes the reported bootstrap files before reading setup completion state. Two open PRs already reference this issue with closing syntax, so cleanup should wait for a reviewed fix to land rather than closing early.

## What This Changes

The issue reports that spawning a subagent recreates bootstrap markdown files at the workspace repository root instead of respecting the per-agent workspace layout.

## Best Possible Solution

Review and land a narrow fix that corrects subagent workspace handoff or runtime seeding scope while preserving existing bootstrap-file setup contracts, then close this issue after the merged PR proves the workspace root stays clean.

## Reproduction Assessment

Yes, at source level: a same-agent `sessions_spawn` can carry `ctx.workspaceDir` into spawned metadata, and the child run then seeds bootstrap files in that selected directory. I did not replay the live Telegram/macOS path.

## Solution Assessment

Unclear whether either linked PR is best as-is because prior reviews flagged blockers. The maintainable solution is a narrow subagent workspace handoff/runtime seeding fix with regression coverage, not a broad shared setup policy change.
