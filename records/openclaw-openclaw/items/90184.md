- **[P2] Guard rename refreshes against newer searches:** `ui/src/ui/chat/session-controls.ts:341`
  - body: After a save succeeds, `refreshChatSessionPickerAfterRename` starts an untracked `sessions.list` request and later assigns `state.chatSessionPickerResult` unconditionally. Please tie this refresh to the existing request-generation guard or re-check the captured query before assigning.
