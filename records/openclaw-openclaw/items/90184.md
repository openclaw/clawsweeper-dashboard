Full review comments:

- **[P2] Guard the post-rename refresh against stale picker state:** `ui/src/ui/chat/session-controls.ts:338`
  - body: `refreshChatSessionPickerAfterRename` captures the current picker query/result before awaiting `sessions.list`, but applies `state.chatSessionPickerResult = page` afterward without checking that the picker is still on the same query or that no newer search request has completed. Because `saveChatSessionPickerRename` clears `chatSessionPickerSavingKey` before launching this fire-and-forget refresh, a user can type a new search while the refresh is in flight and then have the older rename refresh overwrite the newer search results. Reuse the existing request-generation guard or re-check the captured query/open state before applying the response.
  - confidence: 0.86
