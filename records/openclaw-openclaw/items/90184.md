Full review comments:

- **[P2] Guard rename refreshes against newer searches:** `ui/src/ui/chat/session-controls.ts:338`
  - body: After a save succeeds, `refreshChatSessionPickerAfterRename` starts an untracked `sessions.list` request and later assigns `state.chatSessionPickerResult` unconditionally. Since the saving key is cleared before those refreshes settle, a user can start or finish a new picker search in the meantime; if this older refresh returns last, the picker shows stale rows while `chatSessionPickerAppliedQuery` reflects the newer search. Please tie this refresh to the existing request-generation guard or re-check the captured query before assigning.
