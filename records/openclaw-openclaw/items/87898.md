## What This Changes

The PR adds a `src/secrets/audit.ts` heuristic that suppresses `PLAINTEXT_FOUND` when an audited config value is an uppercase env-var-like string ending in `_KEY`, `_TOKEN`, `_SECRET`, or `_PASSWORD`.

## Best Possible Solution

Keep the audit's known-marker and SecretRef contract, fix the generated `models.json` false positive by recognizing values tied to configured env SecretRefs or known markers, and add regression tests for `openclaw.json`, generated `models.json`, and arbitrary all-caps plaintext.

## Reproduction Assessment

Yes from source inspection: current `collectModelsJsonSecrets` warns on non-known marker `models.json` apiKey strings, while the proposed patch only changes `collectConfigSecrets`. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The safer fix is to preserve the existing SecretRef/known-marker contract, repair the generated `models.json` scanner, and avoid a broad all-caps suffix heuristic that can hide real plaintext secrets.
