## Summary

Keep this PR open but do not merge it as-is: it addresses a real secrets-audit annoyance, yet the current diff broadens plaintext suppression in a security-sensitive path and still misses the linked generated models.json case.

## What This Changes

The branch changes `src/secrets/audit.ts` so plaintext findings are skipped when a config secret value is an uppercase string ending in `_KEY`, `_TOKEN`, `_SECRET`, or `_PASSWORD`.

## Best Possible Solution

Define the supported non-secret marker contract first, then implement the narrow audit behavior in the owning scanner with tests that preserve warnings for arbitrary all-caps literal secrets.

## Reproduction Assessment

Yes, from source and test evidence rather than a live command: `collectModelsJsonSecrets` still flags unknown env-like models.json apiKey strings, and existing marker tests intentionally reject arbitrary all-caps markers. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The current PR is too broad for openclaw.json security targets and too narrow for the linked models.json report; the better fix is a maintainer-approved marker contract with focused scanner coverage.
