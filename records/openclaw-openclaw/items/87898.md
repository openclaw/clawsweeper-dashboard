## Summary

Keep this PR open, but the current patch is not safe to merge: it broadens a security audit exception from recognized SecretRef/non-secret markers to arbitrary all-caps suffixes, and it does not actually fix the linked generated models.json path.

## What This Changes

The branch adds a generic env-var-name heuristic that skips PLAINTEXT_FOUND for openclaw.json secret target values such as OPENAI_API_KEY or OPENCLAW_DEEPSEEK_API_KEY.

## Best Possible Solution

Keep the plaintext audit strict by default and handle false positives through an explicit SecretRef or recognized-marker contract that is applied consistently to openclaw.json and generated models.json, with committed tests for both marker and literal-secret cases.

## Reproduction Assessment

Yes. Source inspection shows current main flags non-empty openclaw.json secret strings and generated models.json apiKey values unless they are valid SecretRefs or recognized non-secret markers; I did not run the CLI because this was a read-only review.

## Solution Assessment

No. The PR is a plausible UX shortcut, but the safer fix is an explicit marker or SecretRef contract that does not make arbitrary all-caps secret-looking values invisible to the plaintext audit.
