## Summary

Keep open: the PR is not merge-ready because the suffix heuristic weakens secrets-audit coverage and does not update the separately scanned models.json path from the linked report. Current source and docs already define structured SecretRefs and known non-secret markers as the supported non-secret contract.

## What This Changes

The PR adds a suffix-based helper that skips PLAINTEXT_FOUND for all-caps _KEY/_TOKEN/_SECRET/_PASSWORD string values discovered in openclaw.json secret targets.

## Best Possible Solution

Keep the existing structured SecretRef and known-marker contract unless maintainers explicitly approve a narrower marker policy, then cover both openclaw.json and models.json with regression tests that preserve all-caps literal secret detection.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source: current main flags string secret targets in collectConfigSecrets and only skips known models.json markers in collectModelsJsonSecrets; I did not run the CLI because this review is read-only.

## Solution Assessment

Is this the best way to solve the issue? No; a broad suffix heuristic is a security-audit policy change, while the safer path is a narrow supported marker or structured SecretRef contract with coverage for both source config and generated models.json.
