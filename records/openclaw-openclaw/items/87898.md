Keep open: the updated terminal proof is enough for the real-behavior gate, but the patch is not merge-ready because the new suffix heuristic can suppress real plaintext secret findings and does not cover the models.json path described in the linked report/proof.

## What This Changes

The PR adds a suffix-based helper in src/secrets/audit.ts that skips PLAINTEXT_FOUND for openclaw.json secret target values that look like all-caps environment variable names.

## Best Possible Solution

Keep arbitrary string secret values auditable, and solve the false positive through documented SecretRef forms or a narrowly scoped known-marker exception with negative tests that literal all-caps secrets still warn.

## Reproduction Assessment

Yes for the review blocker: source inspection shows the new check skips any config secret target whose string value matches the suffix regex after plaintext detection. I did not run tests because this review is read-only.

## Solution Assessment

No. The broad suffix heuristic is not the best fix; the safer path is to use existing SecretRef/$VAR semantics or a narrowly scoped known-marker exception with negative coverage for literal secrets.
