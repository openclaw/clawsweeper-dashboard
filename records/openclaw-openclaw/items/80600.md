## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Narrow the module-access check to real import syntax:** `src/agents/code-mode.ts:302`
  - body: The regex treats identifiers that start with `import` as forbidden module access because the `\w` branch accepts the next character. A normal snippet like `const important = 1` is rejected before it reaches QuickJS, so common user code fails in code mode.
  - confidence: 0.93
- **[P1] Guard waits against concurrent resume races:** `src/agents/code-mode.ts:673-686`
  - body: `runWait` leaves the run in `activeRuns` while awaiting pending bridge requests, so two parallel `wait` calls for the same `runId` can both resume the same snapshot. That can run the post-await code twice and duplicate side-effecting nested tool calls.
  - confidence: 0.9
- **[P2] Fail unresolved promises instead of empty waits:** `src/agents/code-mode.worker.ts:363-366`
  - body: This condition snapshots any still-pending result promise even when no host bridge request was queued. Code such as `await new Promise(() => {})` returns `waiting` with no pending tool calls, and `wait` can only restore the same stuck state until expiry.
  - confidence: 0.9

## Security Review

Status: needs_attention

  - body: Two `wait` calls can resume one suspended snapshot because the run is removed only after pending bridge work settles; repeated resume can duplicate nested tool calls after the awaited point.
