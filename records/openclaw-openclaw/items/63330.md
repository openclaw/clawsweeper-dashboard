Overall confidence: 0.97

Full review comments:

- **[P1] Mark the gateway restart tool owner-only:** `extensions/gateway-restart/index.ts:89-94`
  - body: This privileged tool is registered without `ownerOnly: true`. Unlike the built-in gateway tool, `gateway_restart` is not covered by the core-name owner gate, so a non-owner sender could invoke a gateway-wide restart and any configured pre-command when this plugin is enabled.
  - confidence: 0.98
- **[P1] Use the canonical restart policy and scheduler:** `extensions/gateway-restart/index.ts:164`
  - body: Do not call `process.exit(0)` directly here. This bypasses `commands.restart`, restart coalescing, reply-drain ordering, sentinel lifecycle hooks, and the shared SIGUSR1 restart scheduler already used by the gateway tool, creating conflicting restart behavior and potentially terminating the process while replies are still dispatching.
  - confidence: 0.98
- **[P1] Preserve the session-specific workspace:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: The builder always resolves the agent's default workspace instead of preferring the persisted session workspace. Spawned and other workspace-scoped sessions can therefore resume against the wrong repository or files and perform side effects outside the original turn's workspace.
  - confidence: 0.95
- **[P2] Keep the restart continuation until it is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: A successful return only confirms an in-memory enqueue, but this immediately removes the sole restart marker. If the gateway exits again before the queue drains, the promised continuation is permanently lost; retain durable ownership until processing completes or use the canonical restart-continuation lifecycle.
  - confidence: 0.94
