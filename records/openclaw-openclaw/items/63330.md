## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Remove the duplicate gateway restart authority:** `extensions/gateway-restart/index.ts:89-95`
  - body: Current main already routes restart through the gateway tool with `commands.restart`, a trusted runtime session key, the restart sentinel, and `scheduleGatewaySigusr1Restart`. This new plugin adds a separate `gateway_restart` tool with its own marker file, `process.exit`, command allowlist, and model-supplied `sessionKey`, which creates a competing security and recovery path; reuse the existing restart owner path or wait for a maintainer-approved SDK seam.
  - confidence: 0.9
- **[P1] Preserve full cold-session run context:** `src/auto-reply/reply/queue/build-followup-run.ts:57-119`
  - body: The builder reconstructs only a subset of `FollowupRun` and hardcodes owner/reply defaults, while the runner consumes group, sender, account, skills, elevated execution, workspace/cwd, reply, and delivery fields. Cold resumed turns can run under the wrong policy, workspace, tool defaults, or reply behavior unless this path preserves the same context normal follow-up runs carry.
  - confidence: 0.87
- **[P2] Keep the marker until callback processing is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` only places work into the process-local follow-up queue; deleting the marker immediately after enqueue means a second restart or crash before the queue drains permanently loses the callback. Use the existing durable restart continuation path or acknowledge processing only after the callback is durably delivered or requeued.
  - confidence: 0.86
- **[P2] Enable dedupe for plugin-enqueued followups:** `src/plugins/runtime/runtime-followup.ts:30-36`
  - body: The API returns `false` for not found or deduped, but this call passes dedupe mode `none`, so duplicate plugin retries or repeated marker processing can enqueue duplicate follow-up turns for the same session and prompt. Use an existing message or prompt dedupe mode, or define an explicit idempotency key contract for plugin followups.
  - confidence: 0.84
- **[P2] Declare the TypeBox runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: `index.ts` imports `@sinclair/typebox`, but the new plugin package declares no dependency, unlike existing bundled plugins that declare their TypeBox dependency. Standalone plugin/package resolution can fail or rely on root leakage; either use the supported `typebox` package and declare it here or add the correct plugin-local dependency.
  - confidence: 0.82
