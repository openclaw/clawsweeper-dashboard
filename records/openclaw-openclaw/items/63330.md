## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Require trusted-owner gating for gateway_restart:** `extensions/gateway-restart/index.ts:89-95`
  - body: This registers `gateway_restart` as a normal plugin tool even though it can run configured pre-commands and exits the whole gateway. Current gateway owner-only filtering only denies the core tool names `cron`, `gateway`, and `nodes`, so a non-owner session that can use this plugin tool can trigger a global restart; gate it through an owner/trusted-operator policy or the existing core restart authority before exposing it.
  - confidence: 0.88
- **[P1] Honor commands.restart before exiting the gateway:** `extensions/gateway-restart/index.ts:160-164`
  - body: The core `gateway` tool refuses restart when `commands.restart=false`, but this plugin writes the marker and schedules `process.exit(0)` without checking that operator setting. Enabling the plugin would silently reopen gateway restarts for installations that deliberately disabled the public restart command.
  - confidence: 0.86
- **[P1] Rebuild followups with the full stored session context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder falls back to the default agent workspace and only copies a subset of session metadata. Current sessions persist spawned workspace/cwd and group/account context, and the followup runner passes those fields into `runEmbeddedAgent`; omitting them can resume a spawned or group session in the wrong workspace, account, or policy scope.
  - confidence: 0.84
- **[P2] Keep the restart marker until continuation is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` only puts the item in the in-memory followup queue, but the watcher deletes the restart marker immediately after that boolean returns true. If the gateway crashes or is restarted again before the queued turn drains, the marker is gone and the promised post-restart continuation is lost; keep the marker until durable delivery/processing or depend on a durable queue handoff.
  - confidence: 0.82
- **[P2] Keep Multipass concurrency forwarding intact:** `extensions/qa-lab/src/multipass.runtime.ts:365-368`
  - body: This branch removes the `concurrency` parameter and `--concurrency` forwarding from the Multipass plan. Current CLI code forwards `--runner multipass --concurrency`, so merging this drift would make that option stop affecting Multipass QA runs.
  - confidence: 0.8
- **[P2] Declare the plugin's TypeBox runtime dependency:** `extensions/gateway-restart/package.json:15`
  - body: `index.ts` imports the runtime `Type` value from `@sinclair/typebox`, but the new plugin package declares no dependencies and the lockfile entry for `extensions/gateway-restart` is empty. Bundled plugins are reviewed against the same boundary third-party plugins see, so the owner package needs to carry its runtime dependency.
  - confidence: 0.78
