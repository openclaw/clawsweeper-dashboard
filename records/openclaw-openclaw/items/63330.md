## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Derive callbacks from trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up turn. The current gateway restart path prefers the runtime `agentSessionKey` so model-supplied params cannot queue work into another session.
  - confidence: 0.91
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This plugin writes its own JSON marker and exits the process, bypassing the existing `commands.restart` guard, SQLite restart sentinel, restart scheduler/coalescing hooks, and durable continuation queue. That creates a second restart authority with different policy and recovery behavior.
  - confidence: 0.9
- **[P1] Preserve complete follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder reconstructs only a subset of `FollowupRun` and hardcodes owner trust/default workspace values, leaving out fields used for group policy, sender/account context, cwd/elevation, provenance, and reply delivery.
  - confidence: 0.88
- **[P2] Keep the retry marker until delivery is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: The watcher removes the marker as soon as `enqueueFollowupTurn` returns true, but that only proves an in-memory follow-up queue accepted the item. A crash before the queued turn drains can still lose the restart continuation with no retry marker left.
  - confidence: 0.82
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/index.ts:4-5`
  - body: The plugin imports `@sinclair/typebox`, but its package metadata declares no dependencies and the repo's bundled plugins use the `typebox` package. An isolated plugin install can fail to resolve the runtime schema import.
  - confidence: 0.86
