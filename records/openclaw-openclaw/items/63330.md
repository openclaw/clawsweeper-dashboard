## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route callbacks through a trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up. Current gateway restart already prefers the runtime session key for scheduler ownership and sentinel routing, so this parallel path can target the wrong session instead of the active caller.
  - confidence: 0.89
- **[P1] Honor the canonical restart policy before exiting:** `extensions/gateway-restart/index.ts:160-164`
  - body: This path writes its own marker and calls `process.exit(0)` without the current gateway restart `commands.restart` check, scheduler, coalescing, or sentinel cleanup hooks. An enabled plugin can therefore bypass an operator's global restart-disable policy and current restart coordination.
  - confidence: 0.88
- **[P1] Preserve full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: `buildFollowupRunForSession` reconstructs only a subset of `FollowupRun.run` and hardcodes owner/block behavior. The current queue and runner use account, group, sender, channel context, workspace/cwd, elevation, provenance, and delivery fields for policy and routing, so cold plugin follow-ups can resume under the wrong context.
  - confidence: 0.87
- **[P2] Keep restart state durable until the callback runs:** `extensions/gateway-restart/index.ts:221-223`
  - body: Deleting the marker immediately after `enqueueFollowupTurn()` only proves an in-memory enqueue, not that the continuation executed or was delivered. A crash before drain loses the callback, and the JSON sidecar also bypasses the current SQLite restart sentinel durability model.
  - confidence: 0.84
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The new plugin imports `@sinclair/typebox`, but its package manifest has no `dependencies` entry. Standalone plugin installs outside the monorepo hoist can fail to load before registration with `ERR_MODULE_NOT_FOUND`.
  - confidence: 0.78
- **[P2] Restore the global timer spy after tests:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level spy replaces `global.setTimeout` for the whole worker and is never restored. In Vitest lanes that run with `isolate: false`, later test files in the same worker can inherit the mocked timer implementation and become order-dependent.
  - confidence: 0.82
