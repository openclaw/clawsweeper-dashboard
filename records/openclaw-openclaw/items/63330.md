## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor restart policy before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: This tool schedules a gateway-wide `process.exit(0)` without checking `commands.restart`. The existing core gateway restart tool rejects restarts when `commands.restart=false`, so enabling this plugin creates a second restart path that bypasses an operator's documented disable switch.
  - confidence: 0.95
- **[P1] Preserve group policy context for cold followups:** `src/auto-reply/reply/queue/build-followup-run.ts:107-114`
  - body: The rebuilt `run` omits `groupId`, `groupChannel`, and `groupSpace` from the persisted session entry. Those fields are an authorization signal for group-scoped tool policy, so subagent or group followups can resume with a broader or different tool set than the original session.
  - confidence: 0.9
- **[P2] Reuse the session workspace and cwd:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: Cold followups always use the agent default workspace and never restore `spawnedWorkspaceDir` or `spawnedCwd`. Spawned sessions persist those fields specifically so later turns run in the same task workspace, so this can execute post-restart commands against the wrong repository or directory.
  - confidence: 0.92
- **[P2] Keep restart markers until the callback is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` only confirms an in-process queue insert, but the marker is deleted immediately afterward. If the gateway crashes or restarts again before the queue drains, the in-memory followup is gone and the marker has already been removed, silently losing the promised callback.
  - confidence: 0.93
- **[P2] Declare the TypeBox runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The new plugin imports `@sinclair/typebox`, but its package manifest declares no dependencies. Standalone plugin installs can therefore load this package outside the monorepo hoist and fail with a missing module before the plugin registers.
  - confidence: 0.88
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:355-368`
  - body: The PR's multipass plan no longer accepts or appends `--concurrency`, while the CLI still parses and passes that option to `runQaMultipass`. `--runner multipass --concurrency <n>` would be silently ignored, changing QA run time and hiding concurrency-related failures.
  - confidence: 0.86
- **[P2] Restore the global timer spy:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level `setTimeout` spy is never restored, and this repo's unit-fast lane can run with `isolate=false`. Later tests in the same worker can inherit the mocked timer implementation and become order-dependent or hang.
  - confidence: 0.82
