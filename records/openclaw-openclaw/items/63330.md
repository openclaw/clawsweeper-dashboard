## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route restart continuation through the trusted gateway path:** `extensions/gateway-restart/index.ts:36-39`
  - body: This tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up, while the current gateway restart path derives session context from runtime state, checks `commands.restart`, and writes the canonical sentinel. A parallel plugin restart authority can target the wrong session and bypass the existing restart boundary, so this should route through the trusted gateway restart/update path or wait for an explicit maintainer API decision.
  - confidence: 0.89
- **[P1] Preserve full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: `buildFollowupRunForSession` rebuilds only a subset of `FollowupRun.run` and hardcodes owner/workspace/block behavior. The follow-up queue and runner use workspace, cwd, group/account, sender, skills, elevation, provenance, and delivery fields for policy and routing, so cold plugin follow-ups can execute under the wrong context or with the wrong authorization.
  - confidence: 0.87
- **[P2] Use the canonical restart sentinel instead of a marker file:** `extensions/gateway-restart/index.ts:160-164`
  - body: The plugin writes `restart-pending.json` and deletes it after an in-memory enqueue, but current restart continuation state is SQLite-backed and drained through the restart/session-delivery queue. A crash or restart after enqueue but before the callback is processed can lose the continuation, and the extra JSON sidecar creates a second cleanup and retry model.
  - confidence: 0.84
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-14`
  - body: `extensions/gateway-restart/index.ts` imports `@sinclair/typebox`, but this new plugin package declares no dependencies. Because the manifest advertises an installable npm package, standalone plugin installs can fail to load unless the runtime dependency is declared in the plugin package.
  - confidence: 0.8
