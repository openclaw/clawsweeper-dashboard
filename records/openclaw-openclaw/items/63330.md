## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the existing restart-continuation path:** `extensions/gateway-restart/index.ts:89-94`
  - body: Current main already supports post-restart work through the built-in `gateway` tool's `continuationMessage` and restart sentinel queue. Adding a second `gateway_restart` marker/`process.exit` path here bypasses the existing `commands.restart` gate and restart-sentinel semantics, so the two restart contracts can drift.
  - confidence: 0.9
- **[P1] Do not trust the model with the target session key:** `extensions/gateway-restart/index.ts:128-129`
  - body: The tool asks the model for `sessionKey`, persists that value, and the watcher later enqueues an owner-trusted followup to it. The current gateway restart path explicitly uses the runtime session over model-supplied params for this reason, so this can route trusted work into the wrong session.
  - confidence: 0.92
- **[P1] Preserve the full session execution context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: Cold followups are rebuilt with only a subset of the session state, but `FollowupRun` execution consumes account, group, elevated, cwd, workspace, and reply-break fields. Leaving those unset means owner-trusted plugin followups can run with the wrong workspace or tool-policy context for spawned/group sessions.
  - confidence: 0.86
- **[P2] Declare the runtime TypeBox dependency:** `extensions/gateway-restart/index.ts:4-5`
  - body: `index.ts` imports `@sinclair/typebox`, but `@openclaw/gateway-restart-plugin` declares no dependency for it. Extension runtime dependencies must belong to the owning plugin package, otherwise installed/plugin package resolution relies on root or transitive deps.
  - confidence: 0.83
