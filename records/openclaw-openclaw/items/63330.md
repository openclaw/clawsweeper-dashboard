Overall correctness: patch is incorrect

Full review comments:

- **[P1] Route callbacks through trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool takes `sessionKey` from model/tool-call input and later uses it for an owner-trusted follow-up turn. The existing gateway restart path intentionally prefers the runtime `agentSessionKey` so model-supplied params cannot queue work into another session.
  - confidence: 0.91
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:155-160`
  - body: This plugin writes its own marker and exits the process, bypassing the existing `commands.restart` guard, SQLite restart sentinel, SIGUSR1 scheduling hooks, and durable restart continuation queue. That creates a second restart authority with different policy and recovery behavior.
  - confidence: 0.9
- **[P1] Preserve complete follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder only reconstructs a subset of `FollowupRun` and hardcodes owner trust/default workspace values, leaving out fields used for group policy, sender/media routing, cwd, elevation defaults, skills, provenance, and reply-delivery behavior.
  - confidence: 0.88
- **[P2] Keep Multipass concurrency forwarding:** `extensions/qa-lab/src/multipass.runtime.ts:365`
  - body: `runQaMultipass` still receives parsed concurrency from the CLI, but this PR removes forwarding to the guest `pnpm openclaw qa suite` command. `--runner multipass --concurrency N` would silently fall back to the guest default.
  - confidence: 0.86
- **[P2] Declare the runtime schema dependency:** `extensions/gateway-restart/index.ts:4-5`
  - body: The new plugin imports `@sinclair/typebox`, but its package manifest declares no dependency and extension policy requires runtime imports to belong to the owning plugin package. Standalone plugin installs can fail with module resolution errors.
  - confidence: 0.86
