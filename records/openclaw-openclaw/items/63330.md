## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the duplicate gateway restart path:** `extensions/gateway-restart/index.ts:89-95`
  - body: Current main already has a core gateway restart tool that checks commands.restart, uses the trusted runtime session key, writes the canonical restart sentinel, and supports continuation messages. This new tool adds a second model-callable restart authority with a model-supplied sessionKey, its own marker, and direct process exit, so the two paths can diverge in policy and recovery semantics.
  - confidence: 0.92
- **[P1] Preserve the session workspace and run context:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: The cold-run builder resolves the default agent workspace and only reconstructs a narrow subset of FollowupRun. Normal reply runs carry workspace/cwd, account, group, sender, routing, skills, elevated, and delivery fields; omitting them can resume spawned or group sessions in the wrong context with the wrong tool policy or reply behavior.
  - confidence: 0.9
- **[P2] Keep the marker until callback processing is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: enqueueFollowupTurn only proves the continuation entered an in-memory queue. Removing the marker immediately after that means a crash or second restart before the queue drains permanently drops the resume callback.
  - confidence: 0.87
- **[P2] Enable dedupe for runtime followup enqueue:** `src/plugins/runtime/runtime-followup.ts:30-36`
  - body: The public API comment says enqueueFollowupTurn returns false when a turn is deduped, but this call passes dedupe mode none. Retried plugin events or repeated startup processing can enqueue duplicate followup turns instead of being coalesced.
  - confidence: 0.84
- **[P2] Declare the plugin TypeBox dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The gateway-restart runtime imports @sinclair/typebox, but the plugin package declares no dependency or optionalDependency. A packaged or externally installed plugin can fail to load when the dependency is not hoisted from the repo checkout.
  - confidence: 0.82
- **[P2] Forward multipass concurrency into the guest command:** `extensions/qa-lab/src/multipass.runtime.ts:329-341`
  - body: This branch removes concurrency from the multipass plan and no longer forwards --concurrency to the guest qa suite, while the current QA CLI and multipass runner still expose that option. Requested parallel QA runs would silently execute with the default concurrency.
  - confidence: 0.86
- **[P2] Restore the global timer spy after the test file:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: The test installs a top-level spy on global setTimeout and never restores it. In shared Vitest workers, later tests can inherit the mocked timer behavior and hang or skip real timer scheduling unexpectedly.
  - confidence: 0.78
