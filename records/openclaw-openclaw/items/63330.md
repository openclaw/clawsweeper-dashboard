  - body: The new plugin imports `@sinclair/typebox`, but this repo and the plugin package do not declare that package; existing plugins use `typebox`. As written, the plugin can fail to compile or load outside a hoisted/transitive accident, so switch the import to the repo package or add the plugin-owned runtime dependency and lockfile entry.
  - confidence: 0.93

  - body: This path schedules a gateway process exit without checking `commands.restart`. The existing core gateway restart tool fails closed when that flag is false, so enabling this plugin would reintroduce restarts for operators who explicitly disabled them.
  - confidence: 0.94
- **[P1] Reuse the persisted session context:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: The cold follow-up builder always uses the agent default workspace and does not populate the account, group, sender, `bashElevated`, or configured block-streaming fields that current follow-up execution forwards into the embedded agent. Post-restart or deferred turns can therefore run in the wrong workspace or with different tool/policy context than the original session.
  - confidence: 0.92
- **[P2] Normalize the persisted session file:** `src/auto-reply/reply/queue/build-followup-run.ts:83-85`
  - body: Taking `sessionEntry.sessionFile?.trim()` directly bypasses `resolveSessionFilePath(...)`, which is the current containment and stale/corrupt fallback boundary for stored transcript paths. Cold follow-ups should always resolve through that helper before using the transcript path.
  - confidence: 0.88
- **[P2] Restore multipass concurrency forwarding:** `extensions/qa-lab/src/multipass.runtime.ts:336`
  - body: This branch removes the `concurrency` parameter and no longer appends `--concurrency` to the guest `qa suite` command, while the CLI still parses and passes that option. Users requesting a specific multipass concurrency would silently get the default worker count.
  - confidence: 0.9
- **[P2] Keep the callback marker until durable processing:** `extensions/gateway-restart/index.ts:223`
  - body: `enqueueFollowupTurn()` only proves an in-memory enqueue, but the watcher deletes the restart marker immediately afterward. If the gateway crashes or restarts again before the queue drains and delivers, the promised resume callback is permanently lost.
  - confidence: 0.86
