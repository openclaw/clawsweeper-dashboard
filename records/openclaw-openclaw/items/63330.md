## Summary

Keep this PR open. Current main does not yet expose the requested plugin follow-up turn runtime or the proposed gateway-restart plugin, but the current branch still has blocking correctness and security issues around restart authorization, restart-disable policy, cold-session context reconstruction, durable continuation handoff, and an unrelated qa-lab concurrency regression.

## What This Changes

The branch adds a plugin runtime follow-up turn API, a cold-session follow-up run builder, an opt-in gateway-restart plugin, related tests/mocks, lockfile wiring, and small qa-lab/test adjustments.

## Best Possible Solution

Keep the PR open for maintainer API/product/security review. If the follow-up API is accepted, rebase it onto current main, split the generic API from the restart plugin where practical, use the durable session-delivery/restart-continuation path for post-restart work, preserve the same workspace/account/group/elevation context as normal turns, add docs and changelog coverage, and make the restart plugin owner-only with the existing restart-disable guardrails.

## Reproduction Assessment

Yes. The blocking issues reproduce by source inspection of the PR diff against current main: the new tool lacks owner-only gating and restart-disable checks, the cold run builder omits policy-sensitive FollowupRun fields, and the qa-lab diff removes a still-used concurrency path.

## Solution Assessment

No. A generic plugin follow-up helper may be the right end state, but this implementation reconstructs core reply state independently and bundles a security-sensitive restart plugin instead of building on current main's durable restart continuation and session-delivery mechanisms.

Full review comments:

- **[P2] Use the supported TypeBox package:** `extensions/gateway-restart/index.ts:4-5`
  - body: Current bundled plugins and package metadata use `typebox`, not `@sinclair/typebox`. This new import can fail after install or reintroduce an unsupported dependency surface; use the repo-supported package and declare any plugin-local dependency if needed.
  - confidence: 0.87
- **[P1] Mark gateway_restart owner-only:** `extensions/gateway-restart/index.ts:91-98`
  - body: `gateway_restart` is not registered with `ownerOnly: true`, and current tool policy only auto-gates known core names such as `gateway`. If this plugin is enabled, non-owner senders who can access plugin tools can restart the gateway and run allowlisted pre-commands.
  - confidence: 0.95
- **[P2] Reject blank session keys before restarting:** `extensions/gateway-restart/index.ts:127-129`
  - body: The schema accepts any string and the tool writes `parsed.sessionKey` into the marker before exiting. A blank or whitespace session key can restart the whole gateway with no usable callback target, so validate it before running commands or scheduling exit.
  - confidence: 0.84
- **[P2] Honor commands.restart before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: This path writes the marker and calls `process.exit(0)` without checking the global `commands.restart` setting. The built-in gateway restart tool rejects when that flag is false, so this plugin bypasses an operator's explicit restart-disable policy.
  - confidence: 0.92
- **[P1] Reuse the persisted session workspace:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: `buildFollowupRunForSession` always uses the agent default workspace. Spawned sessions persist `spawnedWorkspaceDir`, so a cold follow-up after restart can resume in the wrong repository and run commands against the wrong files.
  - confidence: 0.9
- **[P2] Preserve run policy context for cold followups:** `src/auto-reply/reply/queue/build-followup-run.ts:107-139`
  - body: The reconstructed run omits account, group, and `bashElevated` context even though follow-up execution and tool policy consume those fields. Cold plugin follow-ups can use the wrong account, miss group-scoped policy, or lose elevated execution defaults compared with a normal turn.
  - confidence: 0.88
- **[P2] Keep restart continuations durable until processed:** `extensions/gateway-restart/index.ts:221-223`
  - body: The watcher removes the restart marker as soon as `enqueueFollowupTurn()` returns true, but that only proves an in-memory queue accepted the item. A second crash before the drain finishes loses the callback permanently, unlike current main's durable restart-continuation queue.
  - confidence: 0.87
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:365`
  - body: The PR removes `concurrency` from the multipass plan and stops appending `--concurrency`, while the qa-lab CLI still parses and passes that option. `--runner multipass --concurrency <n>` would be silently ignored.
  - confidence: 0.83
- **[P3] Restore the global timer mock:** `extensions/gateway-restart/index.test.ts:22-24`
  - body: The test file replaces `global.setTimeout` at module scope and only clears mocks in `beforeEach`. With non-isolated Vitest workers, later tests in the same process can inherit the mocked timer implementation.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff introduces gateway restart and shell-command execution paths that do not yet match current owner-only, restart-disable, and durable continuation guardrails.

Concerns:

- **[high] Restart tool is callable by non-owner senders:** `extensions/gateway-restart/index.ts:91`
  - body: `gateway_restart` lacks `ownerOnly: true`, and generic tool policy will not infer owner-only status for this plugin-specific name. If enabled, this exposes process restart and allowlisted shell command execution to non-owner users who can access plugin tools.
  - confidence: 0.95
- **[medium] Restart-disable policy is bypassed:** `extensions/gateway-restart/index.ts:164`
  - body: The plugin restart path does not check `commands.restart=false`, unlike the built-in gateway tool. Operators who disabled restart actions could still be restarted through this new plugin.
  - confidence: 0.92
- **[medium] Cold followups can lose policy context:** `src/auto-reply/reply/queue/build-followup-run.ts:107`
  - body: The follow-up builder omits group, account, and elevation fields from the reconstructed run. For cold sessions, that can change account-scoped behavior or skip group-scoped tool restrictions compared with the original session.
  - confidence: 0.86
