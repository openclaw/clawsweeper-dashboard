## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Derive callbacks from trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up turn. Current `gateway` restart uses the runtime `agentSessionKey` first so model-supplied params cannot queue work into another session; this path should derive the callback target from trusted tool/session context instead.
  - confidence: 0.91
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This plugin writes its own JSON marker and exits the process, bypassing `commands.restart`, the SIGUSR1 scheduler/coalescing hooks, and the SQLite restart sentinel/session-delivery recovery path. That creates a second restart authority with different policy and durability behavior.
  - confidence: 0.9
- **[P1] Preserve complete follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder reconstructs only a subset of `FollowupRun` and defaults owner/workspace execution from agent config, leaving out fields the runner uses for group policy, sender/account context, cwd/elevation, provenance, and reply delivery. Cold follow-ups can therefore run in the wrong context or deliver incorrectly.
  - confidence: 0.87
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: `index.ts` imports `@sinclair/typebox`, but the plugin package declares no dependency. External/plugin-local installs rely on package metadata, so this can fail with `ERR_MODULE_NOT_FOUND` once the plugin is installed outside the root workspace dependency graph.
  - confidence: 0.86
- **[P2] Keep the marker until delivery is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` only proves an in-memory queue accepted the item, but the marker is deleted immediately afterward. If the process crashes or restarts before the follow-up drains and delivers, the continuation is lost; the canonical restart path keeps durable session-delivery state before clearing the sentinel.
  - confidence: 0.84
