## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Route restart continuation through the trusted gateway path:** `extensions/gateway-restart/index.ts:36-39`
  - body: This tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up, while the current gateway restart path checks `commands.restart`, derives the active session identity when available, and writes the canonical restart sentinel. A parallel plugin restart authority can target the wrong session or bypass the existing restart boundary, so this should route through the trusted gateway restart/update path or wait for an explicit maintainer API decision.
  - confidence: 0.89
- **[P1] Preserve full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: `buildFollowupRunForSession` rebuilds only a subset of `FollowupRun.run` and hardcodes owner/workspace/block behavior. The follow-up queue and runner use workspace, cwd, group/account, sender, skills, elevation, provenance, and delivery fields for policy and routing, so cold plugin follow-ups can execute under the wrong context or with the wrong authorization.
  - confidence: 0.87
- **[P2] Use the canonical restart sentinel instead of a marker file:** `extensions/gateway-restart/index.ts:160-164`
  - body: The plugin writes a separate JSON marker and removes it immediately after in-memory enqueue succeeds, while current restart continuation state is SQLite-backed and drained through the session-delivery queue. A crash after enqueue or concurrent restart can lose or overwrite the callback, and the parallel sidecar violates the repo's runtime-state ownership model.
  - confidence: 0.84
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The new plugin imports `@sinclair/typebox`, but its package has no `dependencies` entry. Standalone plugin installs outside the monorepo hoist can fail with `ERR_MODULE_NOT_FOUND`, so the runtime dependency needs to be declared or the plugin should use an already-provided SDK schema surface.
  - confidence: 0.78
- **[P2] Restore the setTimeout spy after each test:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level spy replaces `global.setTimeout` for the whole worker and `beforeEach` only clears call history. OpenClaw test lanes run some Vitest projects with `isolate: false`, so later files in the same worker can inherit the mocked timer implementation and become order-dependent.
  - confidence: 0.82
