## Summary

Keep open. Current main does not implement the proposed plugin follow-up API, and the PR still has blocking correctness/security issues plus missing inspectable real-behavior proof.

## What This Changes

The PR adds a plugin runtime follow-up-turn API, a cold-session follow-up-run builder, an opt-in gateway-restart plugin, and related tests/mocks.

## Best Possible Solution

Split the generic follow-up API from the restart plugin, reuse the durable restart-continuation path, preserve full session policy/workspace context, and gate any restart tool with owner-only plus commands.restart.

## Reproduction Assessment

Yes. The blocking issues are source-reproducible from the PR head: gateway_restart lacks current restart guardrails, cold follow-up runs omit session policy/workspace context, and the qa-lab diff removes supported concurrency forwarding.

## Solution Assessment

No. A plugin follow-up helper may be useful, but this branch reconstructs reply state independently and adds a separate restart marker instead of building on current main's durable restart/session-delivery path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark gateway_restart owner-only:** `extensions/gateway-restart/index.ts:89-95`
  - body: This tool can restart the whole gateway and run allowlisted shell commands, but registration omits `ownerOnly`. Current policy only infers owner-only for known core names like `gateway`, so enabled plugin users who are not owners can invoke this restart path.
  - confidence: 0.95
- **[P2] Honor restart-disable policy before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: The plugin writes its marker and schedules `process.exit(0)` without checking `commands.restart`. The built-in gateway restart path rejects when that flag is false, so this bypasses an operator's explicit restart-disable policy.
  - confidence: 0.92
- **[P1] Reuse the session workspace for follow-up runs:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: `buildFollowupRunForSession` always resolves the agent default workspace instead of preferring `sessionEntry.spawnedWorkspaceDir`. Cold follow-ups for spawned/subagent sessions can resume in the wrong repository and run commands against the wrong files.
  - confidence: 0.9
- **[P2] Carry session policy context into cold runs:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The reconstructed run omits `agentAccountId`, `groupId`/`groupChannel`/`groupSpace`, and `bashElevated`, even though the follow-up runner forwards those fields into `runEmbeddedPiAgent`. Cold plugin follow-ups can use the wrong account, miss group-scoped restrictions, or lose elevated execution defaults.
  - confidence: 0.88
- **[P2] Keep restart markers until the callback is processed:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` returning true only confirms an in-memory queue accepted the item. Deleting the marker immediately means a second crash before drain/delivery can permanently lose the post-restart callback.
  - confidence: 0.87
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:329-341`
  - body: The PR removes `concurrency` from the Multipass plan and no longer appends `--concurrency`, while the qa-lab CLI still accepts and passes that option. Multipass QA runs with a requested concurrency would silently fall back to the default worker count.
  - confidence: 0.83
- **[P2] Use the supported TypeBox package:** `extensions/gateway-restart/index.ts:4-5`
  - body: The new plugin imports `@sinclair/typebox`, but this repo is pinned to `typebox` and the plugin package declares no `@sinclair/typebox` dependency. A plugin-local install can fail to load before registration.
  - confidence: 0.87
- **[P2] Reject blank session keys before restarting:** `extensions/gateway-restart/index.ts:37-39`
  - body: The schema accepts any string and the tool writes `parsed.sessionKey` into the marker before exiting. A blank or whitespace key can restart the gateway with no usable callback target, so validate it before pre-commands or process exit.
  - confidence: 0.84
- **[P3] Restore the global timer mock:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: The test file replaces global `setTimeout` at module scope and only clears mocks in `beforeEach`. With non-isolated Vitest workers, later tests in the same process can inherit the mocked timer implementation.
  - confidence: 0.8
- **[P2] Truncate command output in restart responses:** `extensions/gateway-restart/index.ts:166-180`
  - body: The tool appends full stdout from each pre-command directly into the tool response. Verbose commands such as the default install command can create oversized tool payloads right before restart.
  - confidence: 0.78
