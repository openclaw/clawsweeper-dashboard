## Summary

Keep open. Current main does not already provide the proposed plugin follow-up-turn API, and this PR is still an active feature/API proposal with source-reproducible correctness and security blockers plus missing inspectable real behavior proof.

## What This Changes

The PR adds a plugin runtime follow-up-turn API, a cold-session FollowupRun builder, an opt-in gateway-restart plugin, and related tests/qa-lab adjustments.

## Best Possible Solution

Split the generic follow-up API from the restart plugin, define the SDK contract, preserve full session/workspace/policy context, reuse existing restart-sentinel and durable delivery mechanisms, and move optional restart plugin promotion through ClawHub unless maintainers explicitly want it bundled.

## Reproduction Assessment

Yes. The merge blockers are source-reproducible from the PR head and current main: the restart tool lacks owner-only and restart-disable guards, cold follow-up runs omit persisted session context, the marker is deleted after in-memory enqueue, and qa-lab multipass concurrency is dropped.

## Solution Assessment

No. A plugin follow-up helper may be useful, but this branch reconstructs reply state independently and adds a separate restart marker instead of preserving the existing session policy/delivery context and using the current restart/durable-delivery paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mark gateway_restart owner-only:** `extensions/gateway-restart/index.ts:89-95`
  - body: This registers a gateway-wide restart and allowlisted shell-command tool without `ownerOnly`. Current owner-only policy only auto-matches core names like `gateway`, `cron`, and `nodes`, so enabling this plugin can expose restart and pre-command execution to non-owner senders.
  - confidence: 0.95
- **[P2] Honor restart-disable policy before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: The plugin schedules `process.exit(0)` without checking `commands.restart`. The built-in gateway tool rejects restarts when that flag is false, so this bypasses an operator's explicit restart-disable policy.
  - confidence: 0.92
- **[P1] Reuse the session workspace for follow-up runs:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: The builder always resolves the agent default workspace and ignores `sessionEntry.spawnedWorkspaceDir`. Cold follow-ups for spawned sessions can resume in the wrong repository and run commands against the wrong files.
  - confidence: 0.91
- **[P2] Carry session policy context into cold runs:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The reconstructed run sets `senderIsOwner: true` but omits account, group, sender, owner-list, and elevated execution context that the followup runner forwards into the embedded agent. Cold plugin follow-ups can run with wrong routing or tool-authorization context.
  - confidence: 0.88
- **[P2] Keep restart markers until callback delivery is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` only confirms an in-memory queue accepted the item. Deleting the marker immediately means a second crash or restart before drain/delivery can permanently lose the post-restart callback.
  - confidence: 0.87
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:330-341`
  - body: The PR head omits `concurrency` from the Multipass plan and no longer appends `--concurrency`, while current main still accepts and forwards that option. Requested multipass concurrency would silently fall back to the default.
  - confidence: 0.84
- **[P2] Use the supported TypeBox package:** `extensions/gateway-restart/index.ts:4-5`
  - body: The new plugin imports `@sinclair/typebox`, but its package declares no dependency and the repo's extension pattern uses `typebox`. Plugin-local installs can fail before registration with a missing module.
  - confidence: 0.86
- **[P2] Reject blank session keys before restarting:** `extensions/gateway-restart/index.ts:37-39`
  - body: The schema accepts any string and the tool writes `parsed.sessionKey` before exiting. A blank or whitespace key can restart the gateway with no usable callback target.
  - confidence: 0.84
- **[P2] Truncate pre-command output before returning it:** `extensions/gateway-restart/index.ts:166-180`
  - body: The tool appends full stdout from every pre-restart command directly into the tool response. Verbose commands such as the default install command can create oversized tool payloads right before restart.
  - confidence: 0.78
- **[P2] Normalize stored session files before use:** `src/auto-reply/reply/queue/build-followup-run.ts:83-85`
  - body: The builder trusts `sessionEntry.sessionFile` directly before falling back to `resolveSessionFilePath`. Stale or malformed stored paths can make cold follow-up runs use an unintended transcript location instead of the canonical normalized path.
  - confidence: 0.76
- **[P3] Respect the session block streaming break:** `src/auto-reply/reply/queue/build-followup-run.ts:119`
  - body: The builder hardcodes `blockReplyBreak` to `text_end`, so follow-up callbacks ignore deployments that configure message-end coalescing for normal runs and may chunk replies differently from the active session.
  - confidence: 0.7
- **[P3] Restore the global timer mock after tests:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: The test file installs a top-level `setTimeout` spy and only clears mocks between tests. With non-isolated Vitest workers, later tests in the same process can inherit the mocked timer implementation.
  - confidence: 0.8
