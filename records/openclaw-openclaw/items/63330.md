## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route restart continuation through the trusted gateway path:** `extensions/gateway-restart/index.ts:36-39`
  - body: This tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up, while the current gateway restart path derives session context from runtime state, checks `commands.restart`, and writes the canonical sentinel. A parallel plugin restart authority can target the wrong session and bypass the existing restart boundary, so this should route through the trusted gateway restart/update path or wait for an explicit maintainer API decision.
  - confidence: 0.89
- **[P1] Preserve full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold builder only reconstructs a small subset of `FollowupRun` and hardcodes owner trust, while normal queued runs carry account, group, sender, cwd, skills, elevation, reply, and policy context. Cold plugin follow-ups can therefore resume in the wrong workspace or with the wrong tool/session policy, so the builder needs to preserve the same trusted session context as the existing run path before exposing this API.
  - confidence: 0.9
- **[P2] Use the canonical restart sentinel instead of a marker file:** `extensions/gateway-restart/index.ts:160-164`
  - body: The plugin writes its own JSON marker and schedules `process.exit`, then deletes that marker once `enqueueFollowupTurn()` returns, which only proves in-memory enqueue. Current restart continuation state is SQLite-backed and includes durable delivery/idempotency handling, so another crash before the drain executes can lose the callback and leaves operators with a second restart state format to support.
  - confidence: 0.86
- **[P2] Declare the TypeBox runtime dependency:** `extensions/gateway-restart/package.json:7-14`
  - body: `extensions/gateway-restart/index.ts` imports `@sinclair/typebox` at runtime, but this package manifest declares no runtime dependencies. The existing extension dependency contract requires package imports to be declared, and a packaged install outside the repo hoist can fail with module-not-found unless `@sinclair/typebox` is listed here.
  - confidence: 0.92
- **[P2] Honor follow-up dedupe for plugin enqueues:** `src/plugins/runtime/runtime-followup.ts:30-37`
  - body: This call passes dedupe mode `"none"`, even though the new API documentation says `false` is returned when a turn is deduped. Repeated plugin retries or duplicate service starts can enqueue duplicate follow-up turns for the same session, so the API needs an idempotency/dedupe contract before becoming public.
  - confidence: 0.82
