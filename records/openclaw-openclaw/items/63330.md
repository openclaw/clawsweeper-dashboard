## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Derive the callback session from trusted context:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool schema lets the model provide `sessionKey`, and that value is written into the restart marker for the later owner-trusted follow-up. A tool call should not be able to choose an arbitrary callback session; derive this from trusted tool/runtime context like the existing gateway restart path does.
  - confidence: 0.91
- **[P1] Route restarts through the canonical gateway path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This path writes its own marker file and calls `process.exit(0)`, bypassing `commands.restart`, `scheduleGatewaySigusr1Restart`, restart deferral, emit hooks, and the SQLite restart sentinel. Reuse the canonical gateway restart machinery instead of adding a second restart authority.
  - confidence: 0.93
- **[P1] Preserve the full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The rebuilt cold `FollowupRun` omits context that current queues and runners use for execution and delivery, including account/group identity, spawned cwd/workspace, sender/provenance, elevation defaults, and configured block break behavior. Cold follow-ups can resume with different policy or routing than the original session.
  - confidence: 0.88
- **[P2] Declare the TypeBox runtime dependency:** `extensions/gateway-restart/index.ts:4-5`
  - body: The new plugin imports `@sinclair/typebox`, but its package manifest has no runtime dependencies. Installed plugin packages outside the monorepo hoist can fail to load before registration, so the dependency needs to be declared in the plugin package.
  - confidence: 0.86
- **[P2] Keep restart state until continuation is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: Deleting the marker immediately after `enqueueFollowupTurn()` only proves an in-memory queue item was accepted, not that the callback turn ran or delivered. A crash or second restart before drain can permanently lose the promised continuation.
  - confidence: 0.84
