## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the duplicate gateway restart path:** `extensions/gateway-restart/index.ts:89-95`
  - body: Current main already has a core `gateway` tool restart path that checks `commands.restart`, uses the runtime session key, writes the canonical restart sentinel, and supports continuation turns. This PR registers a separate `gateway_restart` tool that accepts a model-supplied `sessionKey`, writes its own marker, and exits the process directly, creating a second restart authority with different security and recovery semantics.
  - confidence: 0.91
- **[P1] Preserve the session workspace and run context:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: `buildFollowupRunForSession` rebuilds a run from stored session state but falls back to the agent default workspace and omits normal-run context such as `agentAccountId`, group fields, `cwd`, `bashElevated`, and the resolved block streaming break. Cold post-restart or deferred turns can therefore run in the wrong workspace or with different account/group/tool policy than the session that queued them.
  - confidence: 0.9
- **[P2] Keep the restart marker until callback processing is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: Deleting the marker immediately after `enqueueFollowupTurn()` only proves an in-memory queue accepted the item. If the gateway crashes or restarts again before the queue drains, the callback is lost because the marker was removed and the followup queue is not durable in this PR.
  - confidence: 0.86
- **[P2] Enable dedupe for runtime followup enqueue:** `src/plugins/runtime/runtime-followup.ts:30-37`
  - body: The public type comment says `enqueueFollowupTurn` returns false when an item is deduped, but the implementation passes the dedupe mode as `"none"`. Duplicate plugin retries for the same prompt/session will all enqueue and execute instead of returning false for duplicates.
  - confidence: 0.84
- **[P2] Declare the plugin TypeBox dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: `index.ts` imports `@sinclair/typebox`, but the new plugin package has no `dependencies` entry. Installed or packaged plugin execution outside the monorepo hoist can fail before registration with a missing module error.
  - confidence: 0.88
- **[P2] Forward multipass concurrency into the guest command:** `extensions/qa-lab/src/multipass.runtime.ts:329-341`
  - body: The PR removes `concurrency` from the Multipass plan and no longer appends `--concurrency`, while the CLI still parses and passes that option into `runQaMultipass`. `--runner multipass --concurrency <n>` would be silently ignored.
  - confidence: 0.89
- **[P2] Restore the global timer spy after the test file:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level `setTimeout` spy is never restored, and `beforeEach` only clears call history. In isolate-false Vitest lanes, later files in the same worker can inherit the mocked timer and become order-dependent.
  - confidence: 0.84
