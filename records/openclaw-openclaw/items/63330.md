## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Derive callbacks from trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool takes `sessionKey` from model/tool-call input and later uses it for an owner-trusted follow-up turn. The current gateway restart path intentionally prefers the runtime `agentSessionKey` so model-supplied params cannot queue work into another session.
  - confidence: 0.91
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This plugin writes its own marker and exits the process, bypassing the existing `commands.restart` guard, SQLite restart sentinel, SIGUSR1 scheduling hooks, and durable restart continuation queue. That creates a second restart authority with different policy and recovery behavior.
  - confidence: 0.9
- **[P1] Preserve complete follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder only reconstructs a subset of `FollowupRun` and hardcodes owner trust/default workspace values, leaving out fields used for group policy, sender/account context, cwd/elevation, provenance, and reply delivery.
  - confidence: 0.88
- **[P2] Keep callback state until the follow-up turn is durable:** `extensions/gateway-restart/index.ts:215-223`
  - body: `enqueueFollowupTurn()` returning true only proves an in-memory enqueue; deleting the marker immediately means a crash or second restart before queue drain permanently loses the callback.
  - confidence: 0.86
- **[P2] Declare the runtime schema dependency:** `extensions/gateway-restart/index.ts:4-5`
  - body: The plugin imports `@sinclair/typebox`, but its package declares no dependency and repo plugin convention uses `typebox`; external/plugin-local installs can fail before registration with a missing module.
  - confidence: 0.84
- **[P2] Restore the mocked timer after gateway-restart tests:** `extensions/gateway-restart/index.test.ts:24-26`
  - body: This top-level `setTimeout` spy is never restored, so later tests in the same Vitest worker can inherit mocked timer behavior and become order-dependent.
  - confidence: 0.82
- **[P2] Forward multipass concurrency to the QA suite:** `extensions/qa-lab/src/multipass.runtime.ts:368`
  - body: The PR branch's Multipass plan no longer appends `--concurrency`, so `--runner multipass --concurrency <n>` is silently ignored even though the QA suite still accepts the option.
  - confidence: 0.8
- **[P2] Honor the documented follow-up dedupe contract:** `src/plugins/runtime/runtime-followup.ts:30-36`
  - body: The new API says enqueue can return false when a turn is deduped, but the implementation disables dedupe with mode `"none"`, so plugin retries can enqueue duplicate owner-trusted turns.
  - confidence: 0.78
