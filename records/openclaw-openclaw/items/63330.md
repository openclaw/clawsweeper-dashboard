## Summary

Keep open: this is a useful feature/API proposal, but the current branch still has source-reproducible correctness and security blockers plus missing inspectable real behavior proof.

## What This Changes

The branch adds a plugin runtime follow-up-turn API, a cold-session follow-up-run builder, an opt-in gateway-restart plugin, related tests, and small qa-lab/test-helper updates.

## Best Possible Solution

Split the generic follow-up API from the restart plugin, reuse existing durable restart-continuation plumbing, preserve full session policy/workspace context, and gate any restart tool with owner-only plus commands.restart.

## Reproduction Assessment

Yes. The blockers are source-reproducible from the PR head: gateway_restart lacks owner-only and restart-disable guards, cold follow-up runs omit current session context, and qa-lab multipass concurrency forwarding is removed.

## Solution Assessment

No. A plugin follow-up helper may be valuable, but this branch reconstructs reply state independently and adds a separate restart marker instead of building on the current durable restart/session-delivery path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark gateway_restart owner-only:** `extensions/gateway-restart/index.ts:89-95`
  - body: This tool can restart the whole gateway and run allowlisted shell commands, but registration omits ownerOnly. Current policy only infers owner-only for known core names like gateway, so enabled plugin users who are not owners can invoke this restart path.
  - confidence: 0.95
- **[P2] Honor restart-disable policy before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: The plugin writes its marker and schedules process.exit without checking commands.restart. The built-in gateway restart path rejects when that flag is false, so this bypasses an operator's explicit restart-disable policy.
  - confidence: 0.92
- **[P1] Reuse the session workspace for follow-up runs:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: buildFollowupRunForSession always resolves the agent default workspace instead of preferring sessionEntry.spawnedWorkspaceDir. Cold follow-ups for spawned sessions can resume in the wrong repository and run commands against the wrong files.
  - confidence: 0.9
- **[P2] Carry session policy context into cold runs:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The reconstructed run omits account, group, and elevated execution defaults even though the follow-up runner forwards those fields into runEmbeddedPiAgent. Cold plugin follow-ups can use the wrong account, miss group-scoped restrictions, or lose elevated defaults.
  - confidence: 0.88
- **[P2] Keep restart markers until callback delivery is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: enqueueFollowupTurn returning true only confirms an in-memory queue accepted the item. Deleting the marker immediately means a second crash before drain or delivery can permanently lose the post-restart callback.
  - confidence: 0.87
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:329-341`
  - body: The PR removes concurrency from the Multipass plan and no longer appends --concurrency, while the qa-lab CLI still accepts and passes that option. Requested multipass concurrency would silently fall back to the default worker count.
  - confidence: 0.83
- **[P2] Use the supported TypeBox package:** `extensions/gateway-restart/index.ts:4-5`
  - body: The new plugin imports @sinclair/typebox, but this repo is pinned to typebox and the plugin package declares no @sinclair/typebox dependency. Plugin-local installs can fail to load before registration.
  - confidence: 0.87
- **[P2] Reject blank session keys before restarting:** `extensions/gateway-restart/index.ts:37-39`
  - body: The schema accepts any string and the tool writes parsed.sessionKey into the marker before exiting. A blank or whitespace key can restart the gateway with no usable callback target.
  - confidence: 0.84
- **[P3] Restore the global timer mock:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: The test file replaces global setTimeout at module scope and only clears mocks in beforeEach. With non-isolated Vitest workers, later tests in the same process can inherit the mocked timer implementation.
  - confidence: 0.8
- **[P2] Truncate pre-command output before returning it:** `extensions/gateway-restart/index.ts:166-180`
  - body: The tool appends full stdout from every pre-restart command directly into the tool response. Verbose commands such as the default install command can create oversized tool payloads right before restart.
  - confidence: 0.78
- **[P2] Normalize stored session file paths:** `src/auto-reply/reply/queue/build-followup-run.ts:83-85`
  - body: The builder trusts sessionEntry.sessionFile directly before falling back to resolveSessionFilePath. If a stored entry contains an outdated or malformed path, cold follow-up runs can use an unintended transcript location instead of the canonical normalized path.
  - confidence: 0.76
- **[P2] Respect the session block streaming break:** `src/auto-reply/reply/queue/build-followup-run.ts:119`
  - body: The builder hardcodes blockReplyBreak to text_end, so plugin-enqueued follow-ups ignore deployments or sessions configured for message_end and can stream differently than normal user-triggered turns.
  - confidence: 0.78
