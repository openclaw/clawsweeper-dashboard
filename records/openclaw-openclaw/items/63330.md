## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate gateway_restart through restart authorization:** `extensions/gateway-restart/index.ts:89-95`
  - body: This registers `gateway_restart` as an ordinary plugin tool, then lets it run configured shell commands and exit the gateway without checking sender ownership or `commands.restart`. Current core restart goes through the `gateway` tool and rejects restart when `commands.restart=false`, so enabling this plugin creates a second, weaker restart surface for non-owner or policy-disabled callers.
  - confidence: 0.91
- **[P1] Preserve full session context for cold followups:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The rebuilt `FollowupRun` only carries a narrow subset of the normal queued-turn metadata and forces values like `senderIsOwner: true`, default workspace, and `blockReplyBreak: "text_end"`. Cold followups can therefore resume in the wrong workspace/account/group/tool-policy/elevation context compared with the original session; build this from the same session/run metadata used by normal queued followups instead of inventing a partial shape.
  - confidence: 0.88
- **[P2] Keep the restart marker until the callback is processed:** `extensions/gateway-restart/index.ts:221-223`
  - body: `enqueueFollowupTurn()` returns after putting the callback into the in-memory followup queue, but the marker is deleted immediately afterward. If the gateway crashes or restarts again before the queued turn drains, the only durable resume record is already gone, so the post-restart callback is silently lost.
  - confidence: 0.87
- **[P2] Forward multipass concurrency to the guest suite:** `extensions/qa-lab/src/multipass.runtime.ts:365-368`
  - body: The patch removes the `concurrency` parameter from the multipass plan and stops appending `--concurrency`, while the CLI still accepts and passes that option to `runQaMultipass`. `openclaw qa suite --runner multipass --concurrency N` will silently run with the default worker count instead of the requested value.
  - confidence: 0.9
- **[P2] Restore the global timer spy after tests:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level `setTimeout` spy replaces the global implementation for the whole Vitest worker and `beforeEach` only clears call history. With the repo's non-isolated test mode, later files in the same worker can inherit the mocked timer implementation and become order-dependent.
  - confidence: 0.84
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: `extensions/gateway-restart/index.ts` imports `@sinclair/typebox`, but the new plugin package declares no dependencies. Installing or loading the plugin package outside the monorepo hoist can fail at runtime with a missing module before the plugin registers.
  - confidence: 0.79
