## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use trusted session context for restart callbacks:** `extensions/gateway-restart/index.ts:129`
  - body: `sessionKey` comes from model-supplied tool params and is later used to enqueue an owner-trusted post-restart follow-up. Current gateway restart intentionally prefers the runtime session key over model params and has a regression test for this; derive the key from trusted tool/runtime context or reuse the existing gateway restart path.
  - confidence: 0.94
- **[P1] Honor restart-disabled policy before exiting:** `extensions/gateway-restart/index.ts:164`
  - body: The plugin writes a marker and exits the gateway without checking `commands.restart=false`. Current `/restart`, SIGUSR1, and gateway tool restart paths treat that config as the operator's restart kill switch, so this plugin would bypass an existing security/ops policy when enabled.
  - confidence: 0.91
- **[P1] Rebuild followup runs with full session policy context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder reconstructs only a subset of `FollowupRun.run` and omits fields that current drain keys use for authorization, routing, and exec context such as `agentAccountId`, group fields, `runtimePolicySessionKey`, sender/channel context, and `bashElevated`. Cold follow-up turns can therefore resume with different policy or delivery context than the original session.
  - confidence: 0.88
- **[P2] Keep the marker until the followup turn actually runs:** `extensions/gateway-restart/index.ts:221-223`
  - body: Deleting the marker immediately after `enqueueFollowupTurn()` only proves the item was accepted into an in-memory queue. If the gateway crashes or restarts again before the drain runs, the callback is lost permanently, which breaks the restart-resume guarantee this plugin is meant to provide.
  - confidence: 0.86
- **[P2] Restore the global timer mock after each plugin test:** `extensions/gateway-restart/index.test.ts:24-26`
  - body: This top-level `setTimeout` spy is never restored; `beforeEach` only clears call history. In the repository's non-isolated Vitest lanes, that can leak a mocked real timer into later tests and hang or mask unrelated behavior.
  - confidence: 0.9
