Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route restart continuation through the trusted gateway path:** `extensions/gateway-restart/index.ts:36-39`
  - body: This tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up, while the current gateway restart path derives session context from runtime state, checks `commands.restart`, and writes the canonical sentinel. A parallel plugin restart authority can target the wrong session and bypass the existing restart boundary, so route this through the trusted gateway restart/update path or wait for an explicit maintainer API decision.
  - confidence: 0.9
- **[P1] Preserve full trusted context for cold follow-ups:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The reconstructed run only fills a subset of the stored session context and hardcodes `blockReplyBreak`. Current follow-up runs and tool policy consume cwd, spawned workspace, group/account/sender metadata, skillsSnapshot, bashElevated, reply modes, and related state, so cold plugin follow-ups can run in the wrong workspace or with the wrong effective tool policy unless the builder restores the full trusted context or the API is narrowed.
  - confidence: 0.88
- **[P2] Use the canonical restart sentinel instead of a marker file:** `extensions/gateway-restart/index.ts:221-223`
  - body: The plugin writes a separate JSON marker and later deletes it once `enqueueFollowupTurn()` returns true, but that only confirms an in-memory queue enqueue, not durable execution or delivery. A second crash before drain loses the callback; reuse the existing SQLite-backed restart sentinel/session-delivery queue or add equivalent durable acknowledgement semantics.
  - confidence: 0.85
- **[P2] Keep QA Multipass options when rebasing:** `extensions/qa-lab/src/multipass.runtime.ts:355-368`
  - body: The PR head reverts the Multipass runner command construction to an older shape and no longer forwards current options such as `--transport`, `--thinking`, `--allow-failures`, `--concurrency`, runtime pair, channel driver, or enabled plugins. Those controls are parsed and passed by current main, so the branch needs a rebase/repair instead of dropping them from QA runs.
  - confidence: 0.86
- **[P2] Restore the global timer mock after tests:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: This top-level spy replaces `global.setTimeout` for the whole worker and `beforeEach` only clears mock calls. The repo runs Vitest with `--isolate=false` in many lanes, so later test files in the same worker can inherit the fake zero-timeout behavior; restore the spy in cleanup or scope it to the tests that need it.
  - confidence: 0.82
