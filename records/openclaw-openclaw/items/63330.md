Full review comments:

- **[P1] Remove the duplicate gateway restart authority:** `extensions/gateway-restart/index.ts:89-95`
  - body: Current main already has a core gateway restart tool that checks commands.restart, uses the trusted runtime session key, writes the canonical restart sentinel, and supports continuation messages. This new tool adds a second model-callable restart authority with a model-supplied sessionKey, its own marker, and direct process exit, so policy and recovery can diverge.
  - confidence: 0.92
- **[P1] Preserve full cold-session run context:** `src/auto-reply/reply/queue/build-followup-run.ts:57`
  - body: The builder resolves the default agent workspace and only reconstructs a narrow subset of FollowupRun. Normal reply runs carry workspace/cwd, account, group, sender, skills, elevated, and delivery policy fields; omitting them can resume spawned or group sessions in the wrong execution and policy context.
  - confidence: 0.9
- **[P2] Keep the marker until callback processing is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: enqueueFollowupTurn returning true only confirms in-memory enqueue. Deleting the marker immediately means a second crash or restart before the queue drains can permanently lose the post-restart callback.
  - confidence: 0.87
- **[P2] Enable dedupe for plugin-enqueued followups:** `src/plugins/runtime/runtime-followup.ts:30-37`
  - body: The API contract returns false when a duplicate is not enqueued, but this call passes dedupeMode none, so duplicate plugin retries can enqueue duplicate agent turns instead of being coalesced.
  - confidence: 0.84
- **[P2] Declare the gateway-restart runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: extensions/gateway-restart/index.ts imports @sinclair/typebox, but the plugin package declares no dependencies. Installed plugin packages can fail at runtime unless the dependency is owned by this package.
  - confidence: 0.82
