## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route callbacks through a trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool asks the model for `sessionKey` and later uses that value for an owner-trusted follow-up. Current gateway restart already prefers the runtime session key for scheduler ownership and sentinel routing, so this parallel path can target the wrong session instead of the active caller.
  - confidence: 0.89
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This path writes its own marker and calls `process.exit(0)` without the current gateway restart `commands.restart` check, scheduler, coalescing, or sentinel cleanup hooks. An enabled plugin can bypass the operator's global restart policy and current restart coordination.
  - confidence: 0.88
- **[P1] Preserve the full follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: `buildFollowupRunForSession` reconstructs only a subset of `FollowupRun.run` while setting owner trust. Current queue and runner code carry account, group, sender, channel, cwd, elevation, provenance, and delivery fields, so cold follow-ups can resume under the wrong context.
  - confidence: 0.87
- **[P2] Keep restart state durable until continuation runs:** `extensions/gateway-restart/index.ts:221-223`
  - body: The watcher deletes the marker immediately after `enqueueFollowupTurn()` returns true, but that only proves in-memory enqueue. A crash before the follow-up drains loses the callback and bypasses the durable SQLite session-delivery/restart state used elsewhere.
  - confidence: 0.84
- **[P2] Declare the TypeBox runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The plugin imports `@sinclair/typebox`, but its package metadata declares no dependencies. If this package is installed as the manifest advertises, standalone resolution can fail outside the monorepo workspace.
  - confidence: 0.81
- **[P3] Restore the global timer spy:** `extensions/gateway-restart/index.test.ts:23-26`
  - body: The test installs a top-level `setTimeout` spy and never restores it. With the repo's `--isolate=false` testing style, that can leak mocked timer behavior into later tests in the same worker.
  - confidence: 0.76
