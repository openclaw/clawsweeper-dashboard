- **[P1] Use the existing restart-continuation path:** `extensions/gateway-restart/index.ts:89-95`
  - body: Current main already gives the gateway restart tool a `continuationMessage` path backed by the restart sentinel, command authorization, session ownership guards, and delivery-context handling.   - confidence: 0.9
- **[P1] Do not trust the model with the target session key:** `extensions/gateway-restart/index.ts:128-129`
  - body: The tool accepts `sessionKey` from model parameters and persists it into the restart marker, then the watcher enqueues the followup as `senderIsOwner: true`. A model-supplied key can route privileged continuation work into the wrong session; the existing gateway restart path uses the trusted runtime session key instead.
  - confidence: 0.88
- **[P1] Preserve the full session execution context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The reconstructed `FollowupRun` omits persisted account, group, cwd/workspace, skills, auth/elevation, reply-break, and provenance fields that the current followup runner consumes. Cold followups can therefore resume with the wrong workspace, policy, model routing, or delivery context.
  - confidence: 0.9
- **[P2] Forward multipass concurrency:** `extensions/qa-lab/src/multipass.runtime.ts:365-368`
  - body: Current qa-lab CLI support parses `--concurrency` and main forwards it to multipass, but this branch removes the parameter from the command construction. Users running `--runner multipass --concurrency N` would silently lose their requested parallelism.
  - confidence: 0.86
- **[P2] Declare the runtime TypeBox dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: `extensions/gateway-restart/index.ts` imports `@sinclair/typebox`, but the plugin package manifest declares no dependency and current root/plugin manifests do not provide it. Installed plugin-package execution can fail with a missing module error.
