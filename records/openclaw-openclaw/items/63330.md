Overall correctness: patch is incorrect

Full review comments:

- **[P1] Derive callbacks from trusted session identity:** `extensions/gateway-restart/index.ts:36-39`
  - body: The tool asks the model for sessionKey and later uses that value for an owner-trusted follow-up turn. The current gateway restart path prefers the runtime agentSessionKey so model-supplied params cannot queue work into another session.
  - confidence: 0.91
- **[P1] Use the canonical gateway restart path:** `extensions/gateway-restart/index.ts:160-164`
  - body: This plugin writes its own JSON marker and exits the process, bypassing the existing commands.restart guard, SQLite restart sentinel, restart scheduler/coalescing hooks, and continuation routing. That creates a second restart authority with different policy and recovery behavior.
  - confidence: 0.9
- **[P1] Preserve complete follow-up run context:** `src/auto-reply/reply/queue/build-followup-run.ts:107-119`
  - body: The cold-session builder reconstructs only a subset of FollowupRun and hardcodes owner trust/default workspace values, leaving out fields used for group policy, sender/account context, cwd/elevation, provenance, and reply delivery.
  - confidence: 0.88
- **[P2] Keep the marker until continuation delivery is durable:** `extensions/gateway-restart/index.ts:221-223`
  - body: enqueueFollowupTurn only confirms the item was put in an in-memory queue, but the marker is removed immediately after that. A crash before the queued turn drains would still lose the post-restart continuation.
  - confidence: 0.84
- **[P2] Declare the plugin runtime dependency:** `extensions/gateway-restart/package.json:1-15`
  - body: The plugin imports @sinclair/typebox at runtime, but its package manifest declares no dependency for it. An installed plugin package can fail to load with ERR_MODULE_NOT_FOUND instead of registering the restart tool.
  - confidence: 0.82
