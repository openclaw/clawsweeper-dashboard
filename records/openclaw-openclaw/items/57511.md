## Summary

Keep open. Current main does not implement Microsoft Teams live voice, so the PR is not redundant; however, the draft branch is still too broad and has concrete correctness and security blockers before it can merge.

## What This Changes

This draft PR adds Microsoft Teams live voice documentation, manifest/config/schema types, a TypeScript voice manager and gRPC bridge, and a Windows .NET media-worker scaffold.

## Best Possible Solution

Land this as a sequence of owner-reviewed PRs: first document capability tiers and a minimal disabled-by-default config/negotiation contract, then separately add an authenticated worker bridge, pinned .NET worker build, compliance-gated audio handling, and live Teams/RSC validation proof.

## Reproduction Assessment

Yes for the review blockers: static inspection of the supplied diff reproduces missing dependency declarations, insecure bridge setup, direct file-byte playback into a PCM API, receive-only playback configuration, and floating NuGet refs. No live Teams E2E reproduction is available from the supplied context, and the PR test plan leaves that proof unchecked.

## Solution Assessment

No. The all-in-one draft is not the narrowest maintainable path because it combines public SDK/config changes, external Windows infrastructure, compliance policy, realtime media behavior, and documentation before the dependency, security, and Microsoft permission contracts are proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authenticate the worker bridge before sending secrets:** `extensions/msteams/src/voice/worker-bridge.ts:113`
  - body: `connect()` creates an insecure gRPC channel while `JoinMeetingRequest` carries the app secret and the worker is documented as a network service. Require TLS/mTLS or a scoped auth token before exposing call/audio control over this bridge.
  - confidence: 0.92
- **[P2] Declare the gRPC client dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:27-35`
  - body: The bridge imports `@grpc/grpc-js` and `@grpc/proto-loader`, but the PR does not add them to the Teams plugin package or lockfile. Typecheck/package builds will fail and voice negotiation will throw when enabled.
  - confidence: 0.95
- **[P2] Convert TTS output before streaming it as PCM:** `extensions/msteams/src/voice/manager.ts:461-465`
  - body: `textToSpeech()` returns an audio file in the provider output format, but the worker API expects raw 16 kHz mono PCM. Reading the file bytes directly will send compressed/container data as PCM unless this requests telephony PCM output or transcodes first.
  - confidence: 0.88
- **[P2] Open a send-capable audio socket for playback:** `extensions/msteams/media-worker/CallHandler.cs:95-99`
  - body: The worker creates the call audio socket as receive-only but later uses that socket for playback. A receive-only media stream cannot support the advertised listen-and-speak live voice path.
  - confidence: 0.82
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:10-14`
  - body: The new .NET project restores wildcard package ranges such as `1.2.*`, `2.*`, and `3.*`. Pin exact versions so Graph/gRPC/protobuf build and runtime behavior cannot change without a source diff.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete bridge authentication issue and nondeterministic .NET dependency resolution.

Concerns:

- **[high] Unauthenticated insecure worker bridge:** `extensions/msteams/src/voice/worker-bridge.ts:113`
  - body: The TypeScript client uses insecure gRPC while the worker bridge carries app secrets plus call/audio control methods. If the worker port is reachable, this can expose credentials, meeting audio, and call control.
  - confidence: 0.92
- **[medium] Floating NuGet package ranges:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:10`
  - body: The media worker uses wildcard versions for Microsoft Graph Communications, gRPC, and protobuf packages, weakening supply-chain reproducibility and allowing restore-time behavior changes without a source diff.
  - confidence: 0.9
