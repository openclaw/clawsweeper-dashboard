## Summary

Keep open: this recent draft PR is not implemented on current main, has maintainer-requested decomposition, and still has concrete security and functional blockers before it can merge.

## What This Changes

The PR adds Microsoft Teams live voice docs/config/schema, a TypeScript voice manager and gRPC worker bridge, a Teams manifest template, and a Windows .NET media-worker scaffold.

## Best Possible Solution

Land this as a narrower owner-reviewed sequence: capability-tier docs/config first, then a secure deterministic worker protocol, then compliance-gated live audio with real Teams/RSC proof.

## Reproduction Assessment

Yes for the review blockers: static inspection of the PR head reproduces the wrong proto path, insecure bridge, SecretRef drop, undeclared deps, raw TTS-to-PCM path, subscriber leaks, receive-only socket, and floating NuGet ranges. No live Teams end-to-end reproduction is available from the supplied context.

## Solution Assessment

No. The current all-in-one PR is not the best way to solve the feature because it combines public contract, external Windows worker infrastructure, security-sensitive media control, compliance policy, and realtime audio before the dependency and permission contracts are proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Resolve the worker proto from the Teams package path:** `extensions/msteams/src/voice/worker-bridge.ts:22`
  - body: `../../../media-worker` is resolved from `extensions/msteams/src/voice`, which points at `extensions/media-worker` instead of the PR's `extensions/msteams/media-worker`. The first `connect()` call will fail before loading the gRPC service definition.
  - confidence: 0.94
- **[P1] Authenticate the worker bridge before sending secrets:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The client connects with `createInsecure()` while `joinMeeting` sends the Teams app secret, and the worker listens on all interfaces. Remote worker deployments need TLS/mTLS or scoped bridge authentication before credentials and call control cross this channel.
  - confidence: 0.94
- **[P2] Resolve SecretRef app passwords before joining calls:** `extensions/msteams/src/voice/manager.ts:147-150`
  - body: Normal Teams configs can store `appPassword` as a SecretRef, but this code sends an empty secret for anything except a raw string. Use the existing Teams credential/secret resolution path and fail early when the secret cannot be resolved.
  - confidence: 0.9
- **[P2] Convert TTS output before sending PCM playback:** `extensions/msteams/src/voice/manager.ts:486-489`
  - body: `textToSpeech()` returns a provider output file such as MP3/WAV/Opus, but the worker playback path documents raw 16 kHz mono PCM. Decode/resample through the telephony TTS contract or carry format metadata and decode in the worker.
  - confidence: 0.89
- **[P2] Remove stream subscribers when clients disconnect:** `extensions/msteams/media-worker/BridgeService.cs:84`
  - body: `SubscribeUnmixedAudio` adds a delegate to `session.AudioSubscribers` but the `finally` block only completes the local channel. Since the collection has no removal path, disconnected clients leave dead subscribers attached for the call lifetime; the same pattern exists for event subscribers.
  - confidence: 0.86
- **[P2] Declare the gRPC runtime dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:28-38`
  - body: The new bridge imports `@grpc/grpc-js` and `@grpc/proto-loader`, but the Teams package does not declare them. Enabling voice will fail at runtime or during package/type checks before capability negotiation can complete.
  - confidence: 0.95
- **[P2] Open a send-capable audio socket for playback:** `extensions/msteams/media-worker/CallHandler.cs:123-126`
  - body: The worker creates the audio socket as `StreamDirection.Recvonly`, then later uses the same socket for playback injection. A receive-only stream cannot support the advertised listen-and-speak live voice path.
  - confidence: 0.83
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11-15`
  - body: Wildcard NuGet ranges such as `1.2.*`, `2.*`, and `3.*` make the Windows worker build nondeterministic and let Graph/gRPC runtime behavior change without a source diff. Use exact versions and update them deliberately.
  - confidence: 0.9
- **[P2] Map speaker ids to real participants before forwarding audio:** `extensions/msteams/media-worker/CallHandler.cs:495-498`
  - body: `ResolveSpeaker()` iterates participants but never maps the media speaker id to a participant; it always returns synthetic IDs like `Speaker-<id>`. That breaks the per-speaker attribution promised by the live voice path.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces an unauthenticated plaintext media-worker bridge carrying app secrets and nondeterministic .NET dependency resolution.

Concerns:

- **[high] Unauthenticated plaintext worker bridge:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The TypeScript bridge uses insecure gRPC while `JoinMeetingRequest` carries the app secret and call/audio control methods. Because the worker listens on all interfaces and docs describe remote workers, this can expose credentials, meeting audio, and call control if the port is reachable.
  - confidence: 0.94
- **[medium] Floating NuGet package ranges:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11`
  - body: The media worker uses wildcard versions for Microsoft Graph Communications, gRPC, protobuf, and Grpc.Tools, weakening supply-chain reproducibility and allowing build/runtime behavior to change without a source diff.
  - confidence: 0.9
