Full review comments:

- **[P1] Authenticate the worker bridge before sending Teams secrets:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The bridge creates an insecure gRPC channel while `JoinMeeting` sends tenant ID, app ID, and app secret, and the worker listens on all interfaces. Remote worker support needs TLS/mTLS or scoped bridge authentication before credentials and call control cross this channel.
  - confidence: 0.94
- **[P1] Keep voice types out of the public SDK until reviewed:** `src/plugin-sdk/msteams.ts:74-75`
  - body: Exporting `MSTeamsVoiceConfig` and `MSTeamsVoicePermissionMode` creates a public plugin SDK contract before the worker protocol, permission model, and transport security are proven. Keep the first pass plugin-private or land the SDK contract separately with explicit review.
  - confidence: 0.88
- **[P2] Resolve the worker proto from the Teams package path:** `extensions/msteams/src/voice/worker-bridge.ts:22`
  - body: `../../../media-worker` is resolved from `extensions/msteams/src/voice`, which points at `extensions/media-worker` instead of `extensions/msteams/media-worker`. The first connect call will fail before loading the gRPC service definition.
  - confidence: 0.94
- **[P2] Resolve SecretInput app passwords before joining calls:** `extensions/msteams/src/voice/manager.ts:147-150`
  - body: Normal Teams configs can store `appPassword` as `SecretInput`, but this code sends an empty secret for anything except a raw string. Reuse the existing Teams credential resolution path and fail early when the secret cannot be resolved.
  - confidence: 0.9
- **[P2] Declare the gRPC runtime dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:35-38`
  - body: The new bridge imports `@grpc/grpc-js` and `@grpc/proto-loader`, but the Teams package does not declare them. Enabling voice can fail at runtime or during package checks before capability negotiation completes.
  - confidence: 0.95
- **[P2] Convert TTS output to raw telephony PCM before playback:** `extensions/msteams/src/voice/manager.ts:484-489`
  - body: `textToSpeech()` writes provider output files with provider-specific formats, while the worker playback path expects raw 16 kHz mono PCM. Decode and resample before sending frames, or carry format metadata and decode in the worker.
  - confidence: 0.89
- **[P2] Remove stream subscribers when clients disconnect:** `extensions/msteams/media-worker/BridgeService.cs:84`
  - body: `SubscribeUnmixedAudio` and `SubscribeEvents` add callbacks to subscriber collections backed by `ConcurrentBag`, but their finally blocks only complete local channels. Disconnected clients leave dead subscribers attached for the call lifetime.
  - confidence: 0.86
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11-15`
  - body: Wildcard NuGet ranges make the Windows worker build nondeterministic and let Graph, gRPC, and protobuf runtime behavior change without a source diff. Use exact versions and update them deliberately.
  - confidence: 0.9
- **[P2] Map media speaker IDs to Teams participants:** `extensions/msteams/media-worker/CallHandler.cs:482-499`
  - body: `ResolveSpeaker` iterates participants but never maps the media speaker ID to a participant; it always returns synthetic IDs like `Speaker-<id>`. That breaks the per-speaker attribution promised by the live voice path.
  - confidence: 0.82
