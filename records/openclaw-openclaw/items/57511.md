## Summary

Keep open. Current main does not implement Teams voice, and this draft PR still carries reviewed functional/security blockers plus missing real Teams worker proof, so it needs decomposition and contributor/maintainer review rather than cleanup closure.

## What This Changes

The branch adds Microsoft Teams live voice support with a TypeScript voice manager/gRPC bridge, Windows .NET media-worker scaffold, Teams manifest, docs, and config/SDK exports.

## Best Possible Solution

Land this as staged, owner-reviewed work: plugin-private capability/config status first, a secure deterministic worker protocol next, and realtime audio/STT/TTS only after live-worker, permission, and compliance proof exist.

## Reproduction Assessment

Yes for the PR blockers: PR-head source inspection shows the wrong proto path, insecure bridge, SecretInput drop, raw TTS handoff, subscriber leaks, fabricated speaker identity, and floating NuGet ranges. No real Teams worker reproduction/proof is available from the PR body or comments.

## Solution Assessment

No. The all-in-one branch is not the best way to land this feature because it bundles public contracts, external Windows worker infrastructure, security-sensitive media control, compliance policy, and realtime audio before the dependency and permission contracts are proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Authenticate the worker bridge before sending Teams secrets:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The bridge connects with `createInsecure()` while `JoinMeeting` carries tenantId, appId, and appSecret, and the worker listens on all interfaces. Remote worker deployments need TLS/mTLS or scoped bridge authentication before credentials and call control cross this channel.
  - confidence: 0.94
- **[P2] Resolve the worker proto from the Teams package path:** `extensions/msteams/src/voice/worker-bridge.ts:22`
  - body: `../../../media-worker` is resolved from `extensions/msteams/src/voice`, which points at `extensions/media-worker` instead of `extensions/msteams/media-worker`. The first `connect()` call will fail before loading the gRPC service definition.
  - confidence: 0.94
- **[P2] Resolve SecretInput app passwords before joining calls:** `extensions/msteams/src/voice/manager.ts:147-150`
  - body: Normal Teams configs can store `appPassword` as a `SecretInput`, but this code sends an empty secret for anything except a raw string. Use the existing Teams credential/secret resolution path and fail early when the secret cannot be resolved.
  - confidence: 0.9
- **[P2] Declare the gRPC runtime dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:35-38`
  - body: The new bridge imports `@grpc/grpc-js` and `@grpc/proto-loader`, but the Teams package does not declare them. Enabling voice can fail at runtime or during package/type checks before capability negotiation completes.
  - confidence: 0.95
- **[P2] Convert TTS output before streaming PCM playback:** `extensions/msteams/src/voice/manager.ts:489`
  - body: `textToSpeech()` returns provider output files such as MP3, WAV, Opus, or provider-specific formats, but the worker playback path expects raw 16 kHz mono PCM. Decode/resample through a telephony TTS contract or carry format metadata and decode in the worker.
  - confidence: 0.89
- **[P2] Remove stream subscribers on disconnect:** `extensions/msteams/media-worker/BridgeService.cs:84`
  - body: `SubscribeUnmixedAudio` and `SubscribeEvents` add callbacks to session subscriber collections backed by `ConcurrentBag`, but their finally blocks only complete local channels. Disconnected clients leave dead subscribers attached for the call lifetime.
  - confidence: 0.86
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11-15`
  - body: Wildcard NuGet ranges make the Windows worker build nondeterministic and let Graph/gRPC/protobuf runtime behavior change without a source diff. Use exact versions and update them deliberately.
  - confidence: 0.9
- **[P2] Map media speaker IDs to Teams participants:** `extensions/msteams/media-worker/CallHandler.cs:495-499`
  - body: `ResolveSpeaker()` iterates participants but never maps the media speaker ID to a participant; it always returns synthetic IDs like `Speaker-<id>`. That breaks the per-speaker attribution promised by the live voice path.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces an unauthenticated plaintext media-worker bridge carrying Teams credentials and nondeterministic .NET dependency resolution.

Concerns:

- **[high] Unauthenticated plaintext worker bridge:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The TypeScript bridge uses insecure gRPC while `JoinMeetingRequest` carries app secrets and call/audio control methods; the worker binds on all interfaces, so a reachable worker port can expose credentials and call control.
  - confidence: 0.94
- **[medium] Floating NuGet package ranges:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11`
  - body: The media worker uses wildcard versions for Microsoft Graph Communications, gRPC, protobuf, and Grpc.Tools, weakening supply-chain reproducibility and allowing build/runtime behavior to change without a source diff.
  - confidence: 0.9
