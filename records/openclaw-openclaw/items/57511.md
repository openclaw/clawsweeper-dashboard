## Summary

Keep open: current main does not implement Teams live voice, and this draft PR is still blocked by prior changes-requested review, missing real behavior proof, and concrete security/functional defects in the proposed worker bridge.

## What This Changes

The PR adds Microsoft Teams live voice tiers, plugin voice config, a TypeScript voice manager/gRPC bridge, a Windows .NET media worker scaffold, a Teams manifest template, docs, and an SDK export.

## Best Possible Solution

Land this as staged, owner-reviewed work: plugin-private capability/config status first, a secure deterministic worker protocol with build proof next, then realtime audio/STT/TTS after live Teams permission and compliance proof exist.

## Reproduction Assessment

Yes for the PR blockers: PR-head source inspection shows the wrong proto path, insecure bridge, SecretInput drop, missing runtime dependencies, raw TTS handoff, subscriber leaks, synthetic speaker identity, and floating NuGet ranges. No real Teams worker reproduction or after-fix proof is available from the PR body or comments.

## Solution Assessment

No. The all-in-one branch bundles public contracts, external Windows worker infrastructure, security-sensitive media control, compliance policy, and realtime audio before the dependency and permission contracts are proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Authenticate the worker bridge before sending Teams secrets:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The bridge creates an insecure gRPC channel while JoinMeeting sends tenantId, appId, and appSecret, and the worker listens on all interfaces. Remote worker support needs TLS/mTLS or scoped bridge authentication before credentials and call control cross this channel.
  - confidence: 0.94
- **[P2] Resolve the worker proto from the Teams package path:** `extensions/msteams/src/voice/worker-bridge.ts:22`
  - body: ../../../media-worker is resolved from extensions/msteams/src/voice, which points at extensions/media-worker instead of extensions/msteams/media-worker. The first connect() call will fail before loading the gRPC service definition.
  - confidence: 0.94
- **[P2] Resolve SecretInput app passwords before joining calls:** `extensions/msteams/src/voice/manager.ts:147-150`
  - body: Normal Teams configs can store appPassword as SecretInput, but this code sends an empty secret for anything except a raw string. Reuse the existing Teams credential resolution path and fail early when the secret cannot be resolved.
  - confidence: 0.9
- **[P2] Declare the gRPC runtime dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:35-38`
  - body: The new bridge imports @grpc/grpc-js and @grpc/proto-loader, but the Teams package does not declare them. Enabling voice can fail at runtime or during package/type checks before capability negotiation completes.
  - confidence: 0.95
- **[P2] Convert TTS output before streaming PCM playback:** `extensions/msteams/src/voice/manager.ts:489`
  - body: textToSpeech() returns provider output files such as MP3, WAV, Opus, or provider-specific formats, but the worker playback path expects raw 16 kHz mono PCM. Decode/resample through an explicit telephony contract or carry format metadata and decode in the worker.
  - confidence: 0.89
- **[P2] Remove stream subscribers on disconnect:** `extensions/msteams/media-worker/BridgeService.cs:84`
  - body: SubscribeUnmixedAudio and SubscribeEvents add callbacks to session subscriber collections backed by ConcurrentBag, but their finally blocks only complete local channels. Disconnected clients leave dead subscribers attached for the call lifetime.
  - confidence: 0.86
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11-15`
  - body: Wildcard NuGet ranges make the Windows worker build nondeterministic and let Graph/gRPC/protobuf runtime behavior change without a source diff. Use exact versions and update them deliberately.
  - confidence: 0.9
- **[P2] Map media speaker IDs to Teams participants:** `extensions/msteams/media-worker/CallHandler.cs:495-499`
  - body: ResolveSpeaker() iterates participants but never maps the media speaker ID to a participant; it always returns synthetic IDs like Speaker-<id>. That breaks the per-speaker attribution promised by the live voice path.
  - confidence: 0.82
