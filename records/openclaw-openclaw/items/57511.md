## Summary

Keep open: current main does not implement Teams live voice/media-worker support, and this draft PR is still blocked by maintainer-requested decomposition, source-reproducible functional/security defects, and missing live worker proof.

## What This Changes

The PR adds Microsoft Teams voice capability docs/config, a TypeScript voice manager and gRPC worker bridge, a Teams manifest template, and a Windows .NET media-worker scaffold.

## Best Possible Solution

Land this as an owner-reviewed sequence: capability-tier docs/config first, then a secure deterministic worker protocol, then compliance-gated audio/STT/TTS with real Teams/RSC proof.

## Reproduction Assessment

Yes for the review blockers: static inspection of PR head reproduces the wrong proto path, plaintext unauthenticated bridge, SecretRef drop, undeclared dependencies, raw TTS bytes, subscriber leaks, and floating NuGet ranges. No live Teams worker reproduction/proof is available from the supplied PR context.

## Solution Assessment

No. The all-in-one PR is not the best way to solve the feature because it bundles public contracts, external Windows worker infrastructure, security-sensitive media control, compliance policy, and realtime audio before the dependency and permission contracts are proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Authenticate the worker bridge before sending Teams secrets:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The client uses `createInsecure()` while `JoinMeeting` sends the Teams app secret, and the worker binds gRPC with `ListenAnyIP`. Remote worker deployments need TLS/mTLS or scoped bridge authentication before credentials and call control cross this channel.
  - confidence: 0.94
- **[P2] Resolve the worker proto from the Teams package path:** `extensions/msteams/src/voice/worker-bridge.ts:22`
  - body: `../../../media-worker` is resolved from `extensions/msteams/src/voice`, which points at `extensions/media-worker` instead of `extensions/msteams/media-worker`. The first `connect()` call will fail before loading the gRPC service definition.
  - confidence: 0.94
- **[P2] Resolve SecretRef app passwords before joining calls:** `extensions/msteams/src/voice/manager.ts:147-150`
  - body: Normal Teams configs can store `appPassword` as a SecretRef, but this code sends an empty secret for anything except a raw string. Use the existing Teams credential/secret resolution path and fail early when the secret cannot be resolved.
  - confidence: 0.9
- **[P2] Declare the gRPC runtime dependencies:** `extensions/msteams/src/voice/worker-bridge.ts:35-38`
  - body: The new bridge imports `@grpc/grpc-js` and `@grpc/proto-loader`, but the Teams package does not declare them. Enabling voice will fail at runtime or during package/type checks before capability negotiation can complete.
  - confidence: 0.95
- **[P2] Convert TTS output before streaming PCM playback:** `extensions/msteams/src/voice/manager.ts:486-489`
  - body: `textToSpeech()` returns a provider output file such as MP3/WAV/Opus, but the worker playback path documents raw 16 kHz mono PCM. Decode/resample through the telephony TTS contract or carry format metadata and decode in the worker.
  - confidence: 0.89
- **[P2] Remove stream subscribers on disconnect:** `extensions/msteams/media-worker/BridgeService.cs:84`
  - body: `SubscribeUnmixedAudio` adds a delegate to `session.AudioSubscribers` but the `finally` block only completes the local channel. Disconnected clients leave dead subscribers attached for the call lifetime; the same pattern should be checked for event subscribers.
  - confidence: 0.86
- **[P2] Pin the media worker package versions:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11-15`
  - body: Wildcard NuGet ranges such as `1.2.*`, `2.*`, and `3.*` make the Windows worker build nondeterministic and let Graph/gRPC runtime behavior change without a source diff. Use exact versions and update them deliberately.
  - confidence: 0.9
- **[P2] Map media speaker IDs to Teams participants:** `extensions/msteams/media-worker/CallHandler.cs:482-499`
  - body: `ResolveSpeaker()` iterates participants but never maps the media speaker ID to a participant; it always returns synthetic IDs like `Speaker-<id>`. That breaks the per-speaker attribution promised by the live voice path.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces an unauthenticated plaintext media-worker bridge carrying app secrets and nondeterministic .NET dependency resolution.

Concerns:

- **[high] Unauthenticated plaintext worker bridge:** `extensions/msteams/src/voice/worker-bridge.ts:121`
  - body: The TypeScript bridge uses insecure gRPC while `JoinMeetingRequest` carries the app secret and call/audio control methods; the worker binds on all interfaces, so a reachable worker port can expose credentials and call control.
  - confidence: 0.94
- **[medium] Floating NuGet package ranges:** `extensions/msteams/media-worker/TeamsMediaWorker.csproj:11`
  - body: The media worker uses wildcard versions for Microsoft Graph Communications, gRPC, protobuf, and Grpc.Tools, weakening supply-chain reproducibility and allowing build/runtime behavior to change without a source diff.
  - confidence: 0.9
