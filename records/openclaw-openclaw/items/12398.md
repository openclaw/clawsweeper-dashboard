## Summary

Keep open. Current main still lacks a cron postcondition contract or scheduler-owned verifier, and the remaining work is a feature/security/product design rather than conservative cleanup.

## What This Changes

Requested an optional cron `postconditions` field that runs deterministic post-run checks for isolated agent-turn jobs and feeds failures into retry or escalation.

## Best Possible Solution

Add a reviewed additive cron post-run verification contract, or first add a fail-capable plugin hook that preserves scheduler-owned retry and escalation semantics.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives high-confidence proof that the requested postcondition surface is absent from current main.

## Solution Assessment

Unclear. Deterministic post-run verification addresses the reported outcome gap, but the proposed shell-command shape needs maintainer decisions on core-vs-plugin ownership and host command execution boundaries.
