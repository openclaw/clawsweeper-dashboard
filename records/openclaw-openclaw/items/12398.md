## Summary

Keep open. Current main still lacks a cron postcondition contract or scheduler-owned verifier, and the existing discussion already kept this issue open for maintainer design follow-up rather than cleanup closure.

## What This Changes

Add an optional cron `postconditions` verifier that runs deterministic checks after isolated agent jobs and uses the result for retry or escalation.

## Best Possible Solution

Add a reviewed additive cron post-run verification contract, or first add a fail-capable cron post-run plugin hook with scheduler-owned retry and escalation semantics.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives high-confidence absence of the requested postconditions surface across cron types, protocol, runtime execution, docs, and plugin hooks.

## Solution Assessment

Unclear. Deterministic post-run verification addresses the reported outcome gap, but the proposed shell-command shape needs maintainer decisions on core-vs-plugin ownership and host command execution boundaries.
