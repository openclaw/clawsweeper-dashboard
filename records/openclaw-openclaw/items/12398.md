## Summary

Keep open. Current main still lacks a cron postcondition contract or scheduler-owned verifier, and the live issue labels plus prior comments point to product and security review rather than conservative cleanup.

## What This Changes

The issue requests optional cron postconditions that run deterministic checks after isolated agent jobs and drive retry or escalation when an agent reports success without completing the intended outcome.

## Best Possible Solution

Add a reviewed additive cron post-run verification contract, or first add a fail-capable plugin hook that preserves scheduler-owned retry and escalation semantics.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives high-confidence proof that the requested cron postcondition surface is absent from current main.

## Solution Assessment

Unclear. Deterministic post-run verification addresses the reported outcome gap, but the proposed shell-command shape needs maintainer decisions on core-vs-plugin ownership and host command execution boundaries.
