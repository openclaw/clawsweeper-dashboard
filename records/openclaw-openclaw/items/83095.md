## What This Changes

The PR makes Gateway scoped URL normalization overwrite conflicting `oc_cap` query tokens with the path capability and updates injected Canvas live reload to prefer a leading path-scoped token before query fallback.

## Best Possible Solution

Land the narrow Gateway plus Canvas injection fix after current-head live Canvas diagnostics prove both the HTTP rewrite and live-reload WebSocket use the path-scoped token.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main and `v2026.6.11` preserve stale query `oc_cap` in the rewritten URL while injected Canvas live reload reads only `location.search`. A scoped Canvas URL with a conflicting stale query token exercises the mismatch.

## Solution Assessment

Yes for the code shape, but not yet merge-ready. Updating both the Gateway normalizer and injected Canvas WebSocket token selection is the narrow maintainable fix; changing only one side would still let HTTP and live reload disagree.
