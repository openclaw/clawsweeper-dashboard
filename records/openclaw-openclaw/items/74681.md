## Summary

Keep open. Current main still defers isolated cron `agentTurn` timeout arming until the execution-start callback, and the requested dispatch-time hard wall deadline would change the documented and tested execution-budget contract from #41783.

## What This Changes

Request hard wall-time enforcement of isolated cron `agentTurn` `timeoutSeconds` from job start rather than from the execution-start callback.

## Best Possible Solution

Define separate total wall-clock deadline and execution-budget semantics for isolated cron `agentTurn` jobs, then document and test both while preserving queued-run protection.

## Reproduction Assessment

Yes. A source-level reproduction is to delay `onExecutionStarted` past `timeoutSeconds` in an isolated `agentTurn`; current regression coverage explicitly asserts the abort signal remains un-aborted during that wait.

## Solution Assessment

No for directly moving the existing `timeoutSeconds` timer to dispatch. The safer maintainable direction is additive or split timeout semantics so a hard wall deadline can coexist with the current execution-start budget.
