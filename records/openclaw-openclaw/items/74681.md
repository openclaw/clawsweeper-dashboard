## Summary

Keep open. Current main still intentionally does not implement a dispatch-time hard wall for isolated cron agentTurn timeoutSeconds; changing that behavior would alter the documented and regression-tested execution-timeout contract.

## What This Changes

The issue asks for isolated cron agentTurn timeoutSeconds to be enforced from job dispatch as a hard wall-time deadline independent of runner callbacks.

## Best Possible Solution

Add a documented separate hard wall ceiling or split setup, queue, execution, and absolute-deadline budgets for isolated cron runs while preserving the existing execution-timeout semantics.

## Reproduction Assessment

Yes, at source level. Current main still delays isolated cron agentTurn timeoutSeconds until the runner callback path and has tests that assert time spent waiting before that callback does not consume the configured execution timeout.

## Solution Assessment

No for reinterpreting the existing timeoutSeconds field as a dispatch-time wall clock. The safer maintainable path is an additive hard ceiling or split timeout contract so operators can request a total deadline without undoing the current queue-wait fix.
