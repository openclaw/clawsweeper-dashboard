## Summary

Keep open. Current main still defers isolated cron `agentTurn` timeout enforcement until execution-start callbacks, and the existing docs/tests preserve the #41783 execution-budget behavior that this report would change.

## What This Changes

The issue asks OpenClaw to enforce isolated cron `agentTurn` `timeoutSeconds` as a hard wall-time deadline from job start instead of from a later execution-start callback.

## Best Possible Solution

Define the cron timeout contract explicitly, then add separate or additive deadline handling so isolated `agentTurn` jobs can have a reliable total wall-time cap without losing the existing execution-budget protection.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an isolated `agentTurn` fake-timer run that delays `onExecutionStarted` past `timeoutSeconds`; current regression coverage already demonstrates the timeout is not armed until that callback.

## Solution Assessment

No for a direct hard wall-time replacement. The safer solution is a split or additive total-deadline contract that preserves the shipped #41783 execution-budget behavior while addressing callback/preparation overshoot.
