## Summary

Keep open. Current main still arms isolated cron `agentTurn` timeouts from the execution-start callback, and the requested dispatch-time hard wall deadline conflicts with the documented and tested #41783 execution-budget semantics, so this needs maintainer product-contract judgment rather than conservative cleanup.

## What This Changes

Request that isolated cron `agentTurn` `timeoutSeconds` be enforced as a hard wall-time deadline from job start instead of being armed by the execution-start callback.

## Best Possible Solution

Define separate total wall-time and execution-budget semantics for isolated cron `agentTurn` jobs, then document and test both while preserving the #41783 queue-wait protection.

## Reproduction Assessment

Yes, source-level. Delay `onExecutionStarted` past `timeoutSeconds` in an isolated `agentTurn`; current regression coverage asserts the abort signal remains un-aborted during that wait and only aborts after the callback starts the timer.

## Solution Assessment

No for directly moving the existing timer to dispatch. The safer maintainable direction is an additive or split timeout contract so a hard wall deadline can coexist with the current execution-start budget.
