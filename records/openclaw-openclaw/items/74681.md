## Summary

Keep open. Current main still arms the cron timeout for non-main isolated agentTurn jobs from the execution-start callback, and the requested dispatch-time hard wall conflicts with the documented #41783 execution-budget contract, so this needs maintainer product/contract review rather than cleanup closure.

## What This Changes

The issue requests making isolated cron agentTurn timeoutSeconds a hard dispatch-time wall-clock deadline instead of a timer armed when execution starts.

## Best Possible Solution

Define explicit total wall-clock and execution-budget semantics for isolated cron agentTurn jobs, document the distinction, and add regression coverage for delayed execution start, cold bootstrap, and timed-out cleanup.

## Reproduction Assessment

Yes, at source level. Delay onExecutionStarted past timeoutSeconds in an isolated agentTurn; current main keeps the abort signal un-aborted until execution-start, and the existing regression test asserts that behavior.

## Solution Assessment

No, not as a direct move of the existing timer to dispatch. The safer solution is an additive or split timeout contract so a hard wall deadline can coexist with the current #41783 execution-budget protection.
