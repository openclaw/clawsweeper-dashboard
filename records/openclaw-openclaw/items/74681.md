## Summary

Keep open. Current main still defers isolated cron `agentTurn` timeout arming until the execution-start callback, and changing `timeoutSeconds` into a dispatch-time hard wall deadline would alter the documented and tested #41783 execution-budget contract.

## What This Changes

Request a hard wall-time timeout for isolated cron `agentTurn` jobs so `timeoutSeconds` kills the session from job start rather than after `onExecutionStarted`.

## Best Possible Solution

Define separate total wall-time and execution-budget semantics for isolated cron `agentTurn` jobs, then document and test both while preserving the #41783 queue-wait protection.

## Reproduction Assessment

Yes. A source-level reproduction is to delay `onExecutionStarted` past `timeoutSeconds` in an isolated `agentTurn`; current regression coverage already asserts the abort signal remains un-aborted during that wait.

## Solution Assessment

No for directly moving the existing timer to job dispatch. The safer maintainable direction is an additive or split timeout contract so a hard wall deadline can coexist with the current execution-start budget.
