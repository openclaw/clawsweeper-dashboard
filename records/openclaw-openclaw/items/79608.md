## Summary

Keep open: current main covers DeepSeek V4 replay shaping for direct or provider-owned routes, but a custom unowned OpenAI-compatible proxy still misses the stream wrapper that backfills `reasoning_content`.

## What This Changes

Request to apply DeepSeek V4 `reasoning_content` replay payload shaping to custom OpenAI-compatible proxy providers.

## Best Possible Solution

Add a narrow unowned OpenAI-compatible DeepSeek V4 stream-wrapper fallback keyed to exact DeepSeek V4 model ids and `openai-completions`, with regression tests and no new config option.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: an unowned `models.providers` OpenAI-compatible DeepSeek V4 model has no runtime plugin, so the wrapper that adds `reasoning_content` is never applied. I did not live-test an external proxy.

## Solution Assessment

Is this the best way to solve the issue? Yes in direction: use the existing shared DeepSeek V4 wrapper at the provider stream hook/fallback seam, not by importing `extensions/deepseek` internals into core or adding a user config knob.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79608. On current main, a custom `models.providers.<id>` provider with `api: "openai-completions"` and model id `deepseek-v4-flash` or `deepseek-v4-pro` has no owning runtime plugin, so `wrapProviderStreamFn` returns undefined and `createDeepSeekV4OpenAICompatibleThinkingWrapper` is not applied. Add a narrow fallback for unowned custom OpenAI-compatible DeepSeek V4 models that uses the existing shared wrapper to backfill `reasoning_content` and DeepSeek thinking controls. Do not import from `extensions/deepseek/**`, do not change provider-owned behavior for `deepseek`, `openrouter`, `opencode-go`, or `venice`, and do not add a config option. Add regression coverage for thinking enabled/off on an unowned custom proxy and negative coverage for non-DeepSeek models. Add a single-line `CHANGELOG.md` Fix entry if the change is user-facing.
