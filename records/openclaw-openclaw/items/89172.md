## What This Changes

The PR probes outgoing Feishu audio with `ffprobe`, sends the resulting millisecond duration in the Feishu file upload request, and adds tests for duration propagation and probe-failure fallback.

## Best Possible Solution

Land this Feishu-local duration propagation fix, or an equivalent scoped implementation, and let the linked issue close after merge.

## Reproduction Assessment

Yes at source level. Current main routes Ogg/Opus Feishu sends as audio but uploads them without the Feishu `duration` field that the official API says controls visible duration display.

## Solution Assessment

Yes. Probing the prepared bytes after any Feishu voice transcode and attaching `duration` to `im.file.create` is the narrow owner-boundary fix; adding duration to message content would not match the current Feishu audio content shape.
