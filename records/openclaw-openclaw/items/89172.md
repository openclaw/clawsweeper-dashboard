## What This Changes

The PR probes prepared Feishu audio bytes with `runFfprobe`, forwards the millisecond duration to `im.file.create`, and adds Feishu media tests for duration propagation and probe-failure fallback.

## Best Possible Solution

Land this focused Feishu-local upload-duration fix after ordinary maintainer review, then let the linked canonical issue close after merge.

## Reproduction Assessment

## Solution Assessment

Yes. This is the best current fix shape because it measures the final prepared audio bytes in the Feishu plugin, uses the existing bounded ffprobe helper, leaves message content unchanged, and gracefully preserves the old send behavior if probing fails.
