## What This Changes

The PR probes prepared Feishu Ogg/Opus audio with ffprobe, passes the resulting millisecond duration to the Feishu file upload request for audio messages, and adds focused media tests for success and probe-failure fallback.

## Best Possible Solution

Land the Feishu-local upload-duration fix after normal CI and maintainer review, then let the linked bug close through the PR relationship when it merges.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.5.28 upload Feishu audio without duration while the official Feishu upload contract says omitted duration prevents displaying the concrete audio length; I did not run a live before-fix send.

## Solution Assessment

Yes. The PR uses the existing Feishu upload duration field after final voice media preparation, preserves the message content shape, and falls back to existing send behavior if probing fails.
