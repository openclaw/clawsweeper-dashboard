## Summary

Keep this PR open: current main and the latest release still omit Feishu upload duration for voice/audio sends, while this focused PR adds the missing duration propagation with regression coverage and sufficient real Feishu screenshot proof.

## What This Changes

The PR probes prepared Feishu audio bytes with `runFfprobe`, forwards the resulting millisecond duration to `im.file.create`, and adds Feishu media tests for duration propagation and probe-failure fallback.

## Best Possible Solution

Land this focused Feishu-local upload-duration fix after maintainer and branch-protection review, then let the canonical linked issue close after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the Feishu message-tool media path reaches `sendMediaFeishu`, routes Ogg/Opus as audio, and uploads without `duration`; the linked issue supplies the user-facing Feishu voice-bubble symptom.

## Solution Assessment

Yes. This is the best current fix shape because it measures the final prepared audio bytes in the Feishu plugin, uses the existing bounded ffprobe helper, leaves message content unchanged, and preserves send behavior when probing fails.
