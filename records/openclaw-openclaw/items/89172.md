## Summary

Keep this PR open: current main and the latest release still omit Feishu upload duration for voice/audio sends, while this branch is a focused candidate fix with source-level justification, regression tests, and sufficient live Feishu screenshot proof.

## What This Changes

The branch imports `runFfprobe`, probes prepared Feishu audio bytes, forwards the millisecond duration to `im.file.create`, and tests duration propagation plus probe-failure fallback.

## Best Possible Solution

Land this Feishu-local upload-duration fix, or an equivalent scoped implementation, then let the linked issue close after merge.

## Reproduction Assessment

Yes at source level. The Feishu message-tool media path reaches `sendMediaFeishu`, current main routes Ogg/Opus as `msg_type: "audio"`, and the upload call still omits the documented `duration` field.

## Solution Assessment

Yes. Probing the prepared bytes after any Feishu voice transcode and attaching `duration` to `im.file.create` is the narrow owner-boundary fix; adding duration to message content would not match the current Feishu `audio` content shape, which only sends `{file_key}`.
