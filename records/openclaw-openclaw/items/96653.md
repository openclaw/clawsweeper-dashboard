## Summary

Keep open: current main still routes every queued reply through command-secret gateway resolution for agent-runtime targets, those targets include `models.providers.*.apiKey`, and the only preflight skip is limited to gateway credential exec refs. The gateway handler still returns a generic `UNAVAILABLE` payload, and no related PR appears to own this exact reply-path log-noise fix.

## What This Changes

The issue asks to stop per-reply `secrets.resolve` `UNAVAILABLE` log noise for exec-backed model-provider SecretRefs and improve safe diagnostics for the underlying failure.

## Best Possible Solution

Land a focused core fix that avoids the unnecessary reply-path gateway RPC for exec-backed agent-runtime/model-provider SecretRefs, preserves local fallback, and keeps any new diagnostics redacted.

## Solution Assessment
