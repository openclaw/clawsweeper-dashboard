- **[P1] Use the current targeted heartbeat contract:** `src/cron/service/timer.ts:356`
  - body: The new branch calls `state.deps.requestHeartbeatNow`, but current `CronServiceDeps` exposes `requestHeartbeat(opts)` and Gateway resolves the target agent/session through that request.   - confidence: 0.95

  - body: The help text advertises `agent-turn`, but the action handler still rejects every value except `announce` and `webhook`. The documented `openclaw cron edit ... --failure-alert-mode agent-turn` path therefore fails before sending `cron.update`.
  - confidence: 0.96
- **[P1] Update the Gateway cron protocol schemas:** `src/config/zod-schema.ts:556`
  - body: The PR widens config validation, but the Gateway `CronFailureAlertSchema` and `CronFailureDestinationSchema` still validate mode as only `announce | webhook`. Gateway-backed `cron.add` and `cron.update` requests carrying the new value will be rejected at protocol validation.
  - confidence: 0.94
- **[P1] Preserve agent-turn when applying failure-alert patches:** `src/cron/types.ts:76`
  - body: The type change says `agent-turn` is valid, but `mergeCronFailureAlert` still rewrites any mode other than `announce` or `webhook` to `undefined`. Editing an existing job to the new mode would not persist the requested value.
  - confidence: 0.93
- **[P2] Do not treat agent-turn failureDestination as unset:** `src/cron/delivery.ts:205-209`
  - body: Returning `null` when `resolvedMode` is `agent-turn` makes Gateway treat it as no explicit failure destination and then fall back to primary announce delivery. That contradicts the configured mode and can still send an outbound alert instead of using agent-turn semantics.
  - confidence: 0.89
- **[P2] Downgrade failure text before injecting it as a system event:** `src/cron/service/timer.ts:355`
  - body: This enqueues cron failure text without `forceSenderIsOwnerFalse`, so Gateway will treat the event as trusted by default. Failure text can include tool/provider/runtime output, so the agent-turn path should preserve the untrusted sender boundary like hook and exec event relays do.
  - confidence: 0.82
