## Summary

Keep open. The PR is useful source work for the linked cron failure-alert feature, but it is not mergeable against current main because several public entry points still reject or drop `agent-turn`, and the timer branch uses an obsolete heartbeat dependency.

## What This Changes

The PR widens cron failure-alert and failure-destination mode types/config schemas to include `agent-turn`, adds a timer failure-alert branch that enqueues a system event and wakes the agent, and updates CLI help plus a gateway guard.

## Best Possible Solution

Land a rebased, failure-alert-only implementation that supports `failureAlert.mode: "agent-turn"` across config, CLI, Gateway protocol, cron update normalization, agent tool schemas, targeted heartbeat wake, docs/changelog, and focused tests; leave `failureDestination` announce/webhook-only unless separately designed.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the advertised CLI value is rejected by the parser, Gateway and agent tool schemas reject it, cron update normalization drops it, and the timer wake call does not match current main's cron service dependency contract.

## Solution Assessment

No. The direction is reasonable, but this patch is not the best current fix because it widens public contracts before all entry points and dispatch paths agree; a narrow failure-alert implementation with targeted wake semantics is safer.
