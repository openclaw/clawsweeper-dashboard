## Summary

Keep open. Current main still does not implement `failureAlert.mode: "agent-turn"`, and this PR is useful but not merge-ready: the advertised value is not accepted consistently by the CLI, Gateway protocol, or cron update path, the failure-destination widening is inconsistent with runtime dispatch, and the new heartbeat wake is not targeted.

## What This Changes

The PR widens cron failure-alert and failure-destination mode types/config schemas to include `agent-turn`, adds a timer failure-alert branch that enqueues a system event and wakes heartbeat, and updates CLI help plus a gateway guard.

## Best Possible Solution

Land a rebased implementation that adds agent-triggered failure alerts through the current cron service, CLI, Gateway protocol, config schema, and agent-tool schema, with focused tests and a changelog entry; keep `delivery.failureDestination` announce/webhook-only unless a separate dispatch design is approved.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the PR's advertised CLI value remains rejected by the parser, Gateway schemas, and cron update normalizer, and the new wake call lacks the target fields used by the existing cron heartbeat paths.

## Solution Assessment

No. The direction is useful, but the patch is not the best current fix because it widens multiple public contracts before all entry points, normalizers, and dispatch paths agree; a narrower current-main implementation of `failureAlert.mode` is safer.

  - confidence: 0.94
