  - confidence: 0.94

  - body: This branch calls `requestHeartbeatNow`, but current main's cron service dependency exposes `requestHeartbeat({ source, intent, reason, agentId, sessionKey })`. Rebase this onto the current API and target the failing job's agent/session so one cron failure does not wake unrelated agents.

- **[P1] Do not treat agent-turn failureDestination as unset:** `src/cron/delivery.ts:209-210`
  - body: Returning `null` for `delivery.failureDestination.mode: "agent-turn"` makes the Gateway treat the configured failure destination as absent instead of executing agent-turn semantics. That can fall back to the primary outbound announce path or suppress the configured behavior.
  - confidence: 0.9

  - confidence: 0.86
