  - body: The PR widens config/runtime types to include `agent-turn`, but gateway-backed `cron.add` and `cron.update` still validate failure-alert and failure-destination modes as only `announce | webhook`. Update the gateway protocol schemas and tests before exposing this as a supported mode.
  - confidence: 0.93
- **[P1] Do not drop agent-turn failure destinations:** `src/cron/delivery.ts:208`
  - body: Returning `null` for `resolvedMode === "agent-turn"` makes the finished-run dispatcher treat the explicit failure destination as absent. Current dispatch code falls back to the primary announce target when no explicit failure destination resolves, so a job configured for `agent-turn` can still send an outbound alert instead of triggering the agent.
  - confidence: 0.9
- **[P1] Target the heartbeat wake to the failed job:** `src/cron/service/timer.ts:356`
  - body: This enqueues the system event for `params.job.agentId` but requests the heartbeat without `agentId` or `sessionKey`. Untargeted heartbeat wakes are process-wide, so one cron failure can wake unrelated configured agents instead of only the failed job's session.
  - confidence: 0.88
