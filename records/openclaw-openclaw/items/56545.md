## Summary

Keep open: current main still has only announce/webhook failure-alert modes, and this PR is useful source work but has high-confidence blockers against current cron entry points.

## What This Changes

Adds an opt-in cron failure-alert mode intended to enqueue a system event and wake the affected agent instead of sending an outbound alert.

## Best Possible Solution

Land a rebased, failure-alert-only implementation that supports `failureAlert.mode: "agent-turn"` end to end across config, CLI, Gateway protocol, cron normalization, targeted heartbeat wake, agent tool schema, tests, docs, and changelog.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main rejects or drops the advertised value in the CLI parser, Gateway schemas, cron update normalization, and agent tool schema, and the PR's timer wake call no longer matches the current cron dependency contract.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best current fix because it widens public contracts before all entry points and dispatch paths agree; a narrower failure-alert-only implementation is safer.

  - confidence: 0.94
