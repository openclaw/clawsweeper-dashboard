Full review comments:

- **[P1] Keep per-agent compaction models exact:** `src/agents/pi-embedded-runner/compact.ts:424`
  - body: This passes the agent id into the compaction target resolver, but the guard above still only treats `agents.defaults.compaction.model` as explicit. With only `agents.list[].compaction.model` set and session fallbacks configured, this path enters `runWithModelFallback`, so a supposedly exact per-agent compaction model can retry on the session fallback provider instead of staying on the configured provider/model.
  - confidence: 0.9
- **[P1] Preserve tool-model timeout config:** `src/config/zod-schema.agent-defaults.ts:51-56`
  - body: These defaults currently use `AgentToolModelSchema`, which accepts `{ primary, fallbacks, timeoutMs }`. Switching them to `AgentModelSchema` makes existing configs such as `agents.defaults.musicGenerationModel.timeoutMs` fail strict validation even though the music generation docs and runtime still support that setting.
  - confidence: 0.93
- **[P1] Reuse the existing byte-size validator:** `src/config/zod-schema.agent-runtime.ts:924`
  - body: The previous schema delegated byte-size strings to `isValidNonNegativeByteSizeString`, which accepts the same decimal and short-unit forms handled by `parseByteSize` such as `1.5mb` or `10m`. This new regex only accepts integer values with long units, so existing valid `maxActiveTranscriptBytes` or `memoryFlush.forceFlushTranscriptBytes` configs can be rejected on upgrade.
  - confidence: 0.86
