## Summary

Keep open. Current main has per-account channel start/stop hooks, opt-in channel lifecycle helpers, and plugin host cleanup callbacks, but not the requested gateway-owned account-scoped resource manager for channel plugins; the remaining work is a public SDK/channel lifecycle design decision.

## What This Changes

Introduce a centralized account-scoped PluginResourceManager-style interface so channel plugins can start, stop, and fully clean up async resources consistently through the gateway.

## Best Possible Solution

Make an explicit maintainer API decision: extend the plugin SDK with a backward-compatible account-scoped channel resource lifecycle seam and prove it by migrating Feishu plus another high-risk channel, or consolidate this channel-specific request under https://github.com/openclaw/openclaw/issues/49804 after the Feishu fix is handled.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection does confirm current main lacks the requested account-scoped resource manager and still has plugin-specific cleanup paths that motivated the proposal.

## Solution Assessment

Unclear. A centralized manager may be the right end state, but the maintainable path needs a maintainer-owned SDK/channel lifecycle decision before changing third-party plugin contracts.
