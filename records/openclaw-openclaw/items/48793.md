## Summary

Keep open. Current main still has per-account channel lifecycle hooks plus opt-in helper utilities, but no centralized account-scoped PluginResourceManager-style contract; the motivating Feishu close-before-delete cleanup shape is also still present while the narrower Feishu fix PR and the broader lifecycle-design tracker remain open.

## What This Changes

The issue requests a centralized account-scoped plugin resource manager for async channel plugin startup, stop, and full teardown.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async lifecycle seam to the plugin SDK and prove it by migrating Feishu plus another high-risk channel with contract tests, or consolidate this channel-specific request into #49804 after the Feishu bug is handled separately.

## Reproduction Assessment

Not applicable as a feature request, but code inspection gives a high-confidence verification path: the requested manager API is absent, while Feishu still has close-before-delete cleanup code that motivates the request. I did not run live leak reproduction because this review is read-only.

## Solution Assessment

Unclear whether the exact PluginResourceManager shape is the best fix. A narrower, documented SDK lifecycle seam with migration tests would be safer than an ad hoc broad gateway-owned resource manager.
