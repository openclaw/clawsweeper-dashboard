## Summary

Keep open. Current main still exposes per-account channel start/stop hooks and opt-in lifecycle helpers, not a gateway-owned async PluginResourceManager contract, and the motivating Feishu HTTP server close-before-delete cleanup path remains visible. This is a public plugin SDK/channel lifecycle design change, so it needs maintainer API review rather than conservative cleanup.

## What This Changes

Request to add a centralized PluginResourceManager-style lifecycle interface so channel plugins can register async start, stop, and full cleanup behavior consistently across accounts and gateway restarts.

## Best Possible Solution

Maintainers should decide the lifecycle contract first: either add a backward-compatible, account-scoped async resource lifecycle seam in the plugin SDK/gateway and prove it by migrating Feishu plus at least one other high-risk channel with contract tests, or explicitly consolidate the generic resource-manager design into #49804 while keeping the Feishu leak tracked through #48183/#48588.

## Reproduction Assessment

Not applicable as a direct reproduction because this is a feature/design request. The source check gives high confidence that the requested API is absent, and the motivating Feishu close-before-delete path remains visible on current main.

## Solution Assessment

Unclear until maintainer API review. A centralized PluginResourceManager may be the right direction, but the safer next step is to decide whether this belongs as a channel-account SDK seam here or as part of the broader #49804 resource lifecycle design.
