## Summary

Close as superseded by the broader open lifecycle tracker. Current main still does not implement a centralized PluginResourceManager-style channel resource registry, but the same remaining design work is already covered by #49804, while the concrete Feishu cleanup race stays tracked by #48183 and open PR #48588.

## What This Changes

Request to add a centralized PluginResourceManager start/stop/cleanup interface so channel plugins release async resources consistently during gateway restarts.

## Best Possible Solution

Use #49804 as the single design tracker for a standard async resource lifecycle API, and keep #48183/#48588 focused on the immediate Feishu HTTP-server cleanup bug. Any eventual core change should be an additive, documented plugin SDK/channel lifecycle seam with contract tests before channel migrations.

## Reproduction Assessment

Not applicable as a central API feature request. The motivating bug has source-level evidence in Feishu cleanup paths, but the generic PluginResourceManager request is reproduced by current-main contract inspection rather than a runtime test.

## Solution Assessment

Yes for cleanup triage: closing this narrower channel-plugin proposal in favor of #49804 avoids splitting the same public SDK lifecycle design across two open issues. The eventual implementation should still be decided in the canonical tracker, not attempted as a narrow automated fix from this issue.
