## Summary

Keep open. Current main and the latest shipped release still lack a gateway-owned PluginResourceManager or equivalent central async resource registry for channel plugins, and the broader lifecycle tracker overlaps but does not clearly supersede this channel-plugin SDK/API request.

## What This Changes

The issue requests a centralized PluginResourceManager-style async lifecycle interface for channel plugins so gateway shutdown and restart can await plugin resource cleanup consistently.

## Best Possible Solution

Make an explicit maintainer API decision: either consolidate this into https://github.com/openclaw/openclaw/issues/49804 as the canonical lifecycle design tracker, or define a backward-compatible channel SDK lifecycle seam and prove it by migrating Feishu plus another high-risk channel with contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a failing behavior with an already-defined contract. Source inspection gives high confidence that current main lacks the requested manager and still uses per-account hooks plus opt-in lifecycle helpers.

## Solution Assessment

Unclear. A centralized manager may be the right end state, but the maintainable path needs maintainer SDK/channel lifecycle direction before changing public plugin contracts or consolidating into the broader lifecycle tracker.
