## Summary

Keep open. Current main still has per-account channel start/stop hooks and opt-in lifecycle helpers, but not the requested gateway-owned account-scoped resource manager; the remaining work is a public plugin SDK/channel lifecycle design decision, with the immediate Feishu leak still tracked separately.

## What This Changes

The issue requests a centralized PluginResourceManager-style lifecycle interface so channel plugins can consistently await async startup and cleanup resources.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async lifecycle seam to the plugin SDK and prove it by migrating Feishu plus another high-risk channel, or formally consolidate this channel-specific request into https://github.com/openclaw/openclaw/issues/49804 while keeping the Feishu fix separate.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection does confirm current main lacks the requested account-scoped resource manager and still has plugin-specific cleanup paths that motivated the proposal.

## Solution Assessment

Unclear. A centralized manager may be the right end state, but the maintainable path needs a maintainer-owned SDK/channel lifecycle decision before changing third-party plugin contracts.
