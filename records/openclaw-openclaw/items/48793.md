## Summary

Keep open. Current main still has per-account channel lifecycle hooks and opt-in SDK helpers, not a gateway-owned PluginResourceManager, and the broader ResourceManager tracker overlaps but has not clearly superseded this channel-plugin-specific SDK/API request.

## What This Changes

The issue requests a centralized async resource lifecycle contract for channel plugins so gateway restarts and cleanup consistently await plugin-owned resources.

## Best Possible Solution

Make a maintainer-owned API decision: either fold this channel-specific request into https://github.com/openclaw/openclaw/issues/49804, or define a backward-compatible channel SDK lifecycle seam and prove it by migrating Feishu plus another high-risk channel with contract tests.
