## Summary

Keep open. Current main and v2026.5.22 still use hook-based channel lifecycle plus opt-in helper functions, not a gateway-owned PluginResourceManager; the related broader tracker is open but has not clearly superseded this channel-specific SDK/API decision.

## What This Changes

The issue requests a centralized PluginResourceManager interface so channel plugins use consistent async start, stop, and cleanup semantics.

## Best Possible Solution

Make a maintainer-owned API decision: either explicitly consolidate this channel-plugin lifecycle request into https://github.com/openclaw/openclaw/issues/49804, or design a backward-compatible account-scoped SDK lifecycle seam and prove it on Feishu plus another high-risk channel.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection confirms current main and v2026.5.22 lack the centralized manager, while the immediate Feishu cleanup bug has its own source-reproducible issue and open PR.

## Solution Assessment

Unclear. A centralized manager may be the right end state, but the maintainable path needs maintainer direction on the public SDK/gateway lifecycle boundary and its relationship to the broader ResourceManager tracker.
