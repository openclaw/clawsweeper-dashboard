## Summary

Keep open: current main still has the released-prompt fingerprint fence and throws when the session JSONL changes before the attempt reacquires the write lock. The issue is already paired with open pull request https://github.com/openclaw/openclaw/pull/84250, which uses closing syntax and targets the remaining same-process owned-write gap, so this issue should stay open until that PR is reviewed and either lands or is rejected.

## What This Changes

Embedded agent runs can fail with `EmbeddedAttemptSessionTakeoverError` when a session file changes while the prompt lock is released during model I/O.

## Best Possible Solution

Review and land a narrow ownership-aware session-fence fix such as https://github.com/openclaw/openclaw/pull/84250, then close this issue after the merged commit or release proves same-process owned writes no longer trigger takeover while external edits still fail closed.

## Reproduction Assessment

Yes for source-level reproduction: current tests release the prompt lock, append to the session JSONL, and assert `EmbeddedAttemptSessionTakeoverError` on the next write-lock reacquire. I did not run a live channel repro in this read-only pass.

## Solution Assessment

Yes, keeping this open behind the linked fix PR is the right path. Current main has a partial delivery-mirror mitigation, but the open fix is aimed at the broader same-process owned-write gap while preserving external takeover rejection.
