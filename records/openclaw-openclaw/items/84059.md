## What This Changes

The issue reports an upgrade regression where embedded agent turns fail with `EmbeddedAttemptSessionTakeoverError` after same-session transcript/session writes occur while the prompt lock is released.

## Best Possible Solution

Land a narrow ownership-aware session-fence fix that preserves external takeover detection while tolerating legitimate OpenClaw-owned same-process transcript writes, then close this issue from the merged PR.

## Solution Assessment

Not yet, because the fix has not merged and multiple open PRs propose different trust boundaries. The maintainable solution should keep the takeover fence fail-closed for external edits while accepting only verified OpenClaw-owned same-process writes.
