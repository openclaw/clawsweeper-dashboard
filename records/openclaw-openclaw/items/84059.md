The issue asks OpenClaw to stop false-positive `EmbeddedAttemptSessionTakeoverError` failures when legitimate same-process session writes occur while the embedded prompt lock is released.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84250 or a narrow replacement that accepts only OpenClaw-owned same-process transcript writes while preserving external takeover rejection tests.
