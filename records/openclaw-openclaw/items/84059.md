## Summary

Keep open: current main still contains the strict embedded session-file fence for the general same-process write case, and the active fix path is the open closing PR at https://github.com/openclaw/openclaw/pull/84250. ## What This Changes

The issue reports a regression where embedded runs fail when the session JSONL changes during the released prompt-lock window, causing channel replies to fail or drop.

## Best Possible Solution

Review and land the ownership-aware session-fence fix in https://github.com/openclaw/openclaw/pull/84250, preserve external takeover detection, then close this issue with the merged commit and release provenance.

## Reproduction Assessment

Yes, at source level: release one embedded attempt for prompt I/O, let another same-process lock-owning writer append to the same session file, then have the first controller reacquire the write lock. Current main's controller-local fingerprint comparison will reject that changed file as a takeover.

## Solution Assessment

Yes, the active ownership-aware fence direction is the right maintainable fix because it preserves the takeover guard while distinguishing narrow OpenClaw-owned transcript writes from unowned external edits.
