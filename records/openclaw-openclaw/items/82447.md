## Summary

Keep open: current main still globally backfills missing array `items` as `items: {}`, the Xiaomi plugin does not yet own a tool-schema normalization hook, and the latest issue discussion says the Token Plan endpoint still needs targeted validation rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop Xiaomi MiMo Token Plan tool-heavy `openai-completions` runs from failing with `400 Param Incorrect` after the 2026.5.12 array `items` schema backfill.

## Best Possible Solution

Add a Xiaomi/`xiaomi-native` scoped tool-schema compatibility path that removes or rewrites only provider-rejected synthetic schema additions for Xiaomi Token Plan/custom routes, with focused schema tests and redacted live Token Plan proof.

## Reproduction Assessment

No high-confidence live reproduction was performed because the Token Plan endpoint needs credentials/subscription access. Source inspection does reproduce the outbound `items: {}` backfill on current main, and multiple issue comments report the same provider/model/error cluster.

## Solution Assessment

No, closing or globally reverting the schema normalizer is not the best path. The safer fix is targeted Xiaomi/`xiaomi-native` compatibility that preserves typed `items` and avoids re-breaking providers that require explicit array item schemas.
