## Summary

Keep open. Current main still has adjacent mitigations, but it does not implement a main-session policy for provider `stopReason: "length"` that preserves truncation metadata, blocks unsafe tool execution, or gives bounded model-visible recovery feedback.

## What This Changes

Request main interactive agent-session detection and recovery for output truncation, including tool-call validation, model-facing recovery prompts, and optional retry/config policy.

## Best Possible Solution

Add a documented main-session truncation policy that preserves provider length metadata, blocks unsafe truncated tool calls before execution, gives bounded recovery feedback, and leaves provider-specific large-argument repair on narrower trackers.

## Reproduction Assessment

Yes, at source level: current main normalizes provider length stops but the main-session incomplete-turn and tool-result guards do not treat `length` as an incomplete tool-call condition. I did not run a live provider replay in this read-only review.

## Solution Assessment

Unclear as a final design. The detection and fail-closed direction is sound, but the requested config modes, retry budgets, and circuit breaker need maintainer runtime-policy judgment before implementation.
