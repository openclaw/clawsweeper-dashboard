## Summary

Keep this open. Current main has partial mitigations for adjacent failure modes, but it still does not implement a main-session `stopReason: "length"` detection/recovery policy; a later ClawSweeper review of a broader duplicate also treated this issue as the canonical tracker for the remaining main-session truncation work.

## What This Changes

The issue requests main interactive agent sessions to detect provider output truncation, avoid executing suspect truncated tool calls, and give the model bounded recovery feedback.

## Best Possible Solution

Add a main-session runtime policy that preserves provider length metadata, blocks or errors unsafe truncated tool calls before execution, gives bounded model-visible recovery feedback, and keeps transport-specific large-argument bugs on their narrower trackers.

## Reproduction Assessment

Yes, at source level: current main maps provider length stops but has no general main-session branch that treats `length` as incomplete before tool execution. I did not run a live Bedrock/OpenAI-compatible replay in this read-only review.

## Solution Assessment

Unclear as a final design. The detection and recovery direction is sound, but the proposed config modes, retry budgets, and circuit breaker need maintainer policy before a safe implementation PR.
