## Summary

Keep open. Current main has adjacent mitigations for malformed tool-call arguments and empty terminal turns, but it still does not implement the requested main-session `stopReason: "length"` detection/recovery policy; related broader work was closed back to this issue as the canonical main-session tracker.

## What This Changes

Request main interactive-agent detection and recovery when provider output stops with `stopReason: "length"`, especially before executing truncated tool calls.

## Best Possible Solution

Define a main-session policy for provider `length` stops, then add focused regression coverage that prevents unsafe truncated tool-call execution while coordinating separate subagent and Bedrock-specific work in the linked issues.

## Reproduction Assessment

Yes, source-level. At the target main SHA, a main-session assistant turn with `stopReason:"length"` and visible text is not treated as incomplete, and `length` is not excluded from assistant tool-call extraction; I did not run a live provider reproduction in this read-only review.

## Solution Assessment

No, not as written for automation. The broad config, retry-budget, and circuit-breaker proposal needs maintainer policy; the narrow first implementation should preserve provider stop reason and block or recover unsafe `length`-terminated tool calls with regression tests.
