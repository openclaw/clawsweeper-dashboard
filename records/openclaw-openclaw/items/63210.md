## Summary

Keep open. Current main has adjacent incomplete-turn and tool-argument repair mitigations, but it still lacks a main-session policy that treats provider `stopReason: "length"` as unsafe before terminal handling or tool execution.

## What This Changes

Request detection and recovery for main-session output truncation, including preserving provider length metadata, validating suspect tool calls, and adding bounded model-visible retry behavior.

## Best Possible Solution

Add a main-session runtime policy that preserves provider length metadata, blocks or errors unsafe truncated tool calls before execution, gives bounded model-visible recovery feedback, and coordinates defaults with https://github.com/openclaw/openclaw/issues/63188 and https://github.com/openclaw/openclaw/issues/70033.

## Reproduction Assessment

Yes, at source level: current main maps provider length stops but does not route `length` through the main-session incomplete-turn or no-execute recovery paths. I did not run a live provider replay in this read-only review.

## Solution Assessment

Unclear as a final design. The detection and no-execute direction is sound, but the requested config modes, retry budget, and circuit breaker need maintainer runtime-policy judgment before a safe implementation.
