Full review comments:

- **[P2] Account for reconciler-triggered delivery attempts:** `extensions/codex/src/app-server/native-subagent-monitor.ts:641`
  - body: This cap only advances inside `scheduleCompletionDeliveryRetry`, but stdio Codex monitors also reconcile pending codex-native task rows through `reconcileKnownTaskRowsForParent()`. That path calls `processCompletion()` and then `deliverPendingCompletion()` even while `completionDeliveryTimer` is armed, so a permanently non-durable completion can re-enter the parent between scheduled retries without consuming the retry schedule. Skip reconciliation while a completion-delivery retry is armed, or make every delivery attempt share the same bounded state.
  - confidence: 0.82
