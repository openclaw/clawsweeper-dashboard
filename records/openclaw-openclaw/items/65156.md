## Summary

Keep open. Current main still treats a successful sqlite-vec native load as availability without proving `vec_version()` or `vec0` registration, so the reported load-success/no-functions failure mode is still a valid memory vector bug.

## What This Changes

The issue reports that sqlite-vec can load successfully while registering no vec functions, leaving memory vector indexing degraded to FTS-only after OpenClaw 2026.4.11.

## Best Possible Solution

Add a post-load sqlite-vec health probe in the memory-host loader, have Memory Core treat failed `vec_version` or temporary `vec0` creation as unavailable with a clear `loadError`, and preserve the existing degraded fallback.

## Reproduction Assessment

Yes. This is source-reproducible on current main by making the native sqlite-vec load call succeed while `vec_version()` or a temporary `vec0` table probe fails; the issue also has independent Linux, Windows, and macOS confirmations.

## Solution Assessment

No for the suggested dependency pin or rebuild as the first repository fix. The narrow maintainable repair is a loader-level sqlite-vec usability probe with regression coverage and clear degraded-state reporting.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/65156. Current main treats `loadExtension`/`sqliteVec.load` success as sqlite-vec availability, but reporters reproduced a silent ABI/no-functions failure where `vec_version()` and `vec0` are unavailable after load. Implement a narrow post-load health probe in `packages/memory-host-sdk/src/host/sqlite-vec.ts`: after either explicit `extensionPath` load or bundled `sqliteVec.load`, verify sqlite-vec usability with `vec_version()` and a temporary `vec0` virtual table or equivalent minimal probe. If the probe fails, return `ok: false` with an actionable error so Memory Core keeps the existing degraded fallback and does not cache `vector.available = true`. Add focused regression tests for load-success/probe-failure and load-success/probe-success. Update Memory Core tests only as needed to preserve warning/status behavior. Do not pin or rebuild sqlite-vec, do not add a new config option, and do not fold in the Windows DLL suffix PR unless the health-probe fix needs a small compatibility adjustment. Add a single-line user-facing CHANGELOG.md fix entry because this affects memory vector indexing.
