## Summary

Keep open. Current main still treats a successful sqlite-vec load call as availability, so the reported load-success/no-functions failure remains unguarded despite newer status and packaging fixes.

## What This Changes

The issue asks OpenClaw to restore sqlite-vec-backed memory vector indexing when sqlite-vec loads but fails to register vector functions because of a SQLite ABI mismatch.

## Best Possible Solution

Memory loading should only mark sqlite-vec available after a post-load `vec_version()` and/or temporary `vec0` probe succeeds, then preserve a clear `loadError` and continue through the existing degraded fallback path when that probe fails.

## Reproduction Assessment

Yes. The issue has concrete Linux steps plus multiple platform confirmations through 2026.5.3-1, and current main is source-reproducible by making the load call succeed while `vec_version` or temporary `vec0` creation fails.

## Solution Assessment

No for a dependency pin or binary rebuild as the first repository fix. The narrow maintainable fix is a loader-level sqlite-vec health probe with regression coverage and existing fallback behavior preserved.

## Repair Work Prompt

Fix the Memory Core sqlite-vec load-success/no-functions bug tracked by https://github.com/openclaw/openclaw/issues/65156. In `packages/memory-host-sdk/src/host/sqlite-vec.ts`, make `loadSqliteVecExtension` verify sqlite-vec usability after `db.loadExtension` or `sqliteVec.load` succeeds, preferably by probing `vec_version()` and/or creating/dropping a temporary `vec0` virtual table. Return `{ ok: false, error }` when the probe fails so `extensions/memory-core/src/memory/manager-sync-ops.ts` preserves `vector.loadError` and falls back cleanly instead of caching a false available state. Add focused regression coverage where the load call succeeds but the function/module probe fails. Preserve existing optional sqlite-vec behavior and user `extensionPath` support. Do not add a config option, pin/rebuild sqlite binaries, or broaden dependency policy without maintainer approval. Add a user-facing `CHANGELOG.md` fix entry.
