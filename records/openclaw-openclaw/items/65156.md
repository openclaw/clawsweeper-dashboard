## Summary

Keep open. Current main still accepts a successful sqlite-vec native load as proof that vec0 is usable, while this report is specifically the load-success/no-functions failure mode on Linux.

## What This Changes

Fix memory vector search when sqlite-vec loads but does not register vec0 functions or modules after a SQLite ABI mismatch.

## Best Possible Solution

Memory Core should only mark sqlite-vec available after a post-load probe proves vec_version and the vec0 module work, then preserve a clear loadError and continue through the existing degraded fallback path when the probe fails.

## Reproduction Assessment

Yes. The issue has concrete Linux steps plus two independent confirmations, and current main can be unit-reproduced by making the sqlite-vec load call succeed while a vec_version or temporary vec0 table probe fails.

## Solution Assessment

No for the suggested dependency pin or rebuild as the first repository fix; those are broader dependency-management choices. The narrower maintainable fix is a loader-level sqlite-vec health probe with regression coverage and clear degraded-state reporting.

## Repair Work Prompt

Fix this issue by adding a narrow sqlite-vec post-load usability probe. The loader must not return ok=true unless the loaded extension actually registered the expected sqlite-vec surface, preferably by checking `SELECT vec_version()` and/or creating and dropping a temporary `vec0` virtual table without leaving persistent artifacts. Wire failures into the existing `{ ok:false, error }` / `vector.loadError` path so Memory Core does not cache `vector.available=true` for unusable native loads, and preserve existing degraded fallback behavior. Do not pin or upgrade sqlite-vec, SQLite, Node, or better-sqlite3 without maintainer approval, and do not fold in the Windows .dll retry from #69059 unless it is strictly needed for this probe. Add regression coverage for load succeeds but the health probe fails, plus any Memory Core warning/status coverage needed. Add a single-line CHANGELOG.md fix entry because this affects user-facing memory vector search.
