## Summary

Keep open. Current main and the latest release still mark sqlite-vec available immediately after a successful load call, while this report is specifically about the silent load-success/no-functions failure mode confirmed across multiple platforms.

## What This Changes

The issue asks OpenClaw to restore memory vector search when sqlite-vec loads but does not register usable vector functions after an SQLite ABI mismatch.

## Best Possible Solution

Add a post-load sqlite-vec usability probe before marking the vector store available, preserve a clear `loadError` when the probe fails, and keep the degraded FTS fallback working.

## Reproduction Assessment

Yes. There is a high-confidence source-level path: make `loadExtension` or `sqliteVec.load` succeed, then make `vec_version()` or temporary `vec0` creation fail; current main still has no post-load probe, and issue comments confirm the symptom on Linux, Windows, and macOS.

## Solution Assessment

No for the suggested dependency pin or rebuild as the first repository fix, because that is broader dependency policy. The narrow maintainable fix is a loader-level sqlite-vec health probe with regression coverage and clear degraded-state reporting.
