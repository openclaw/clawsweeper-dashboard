## Summary

Keep open. ## What This Changes

The issue requests restoring builtin memory vector search when sqlite-vec appears to load but registers no usable SQLite functions or `vec0` module.

## Best Possible Solution

Add a sqlite-vec usability probe before marking the vector store available, report probe failures clearly, preserve explicit `extensionPath` and platform-variant fallback behavior, and keep degraded FTS/in-process fallback working.

## Reproduction Assessment

Yes by source inspection, though not by a live affected-host run: make `db.loadExtension(...)` or `sqliteVec.load(db)` succeed while `vec_version()` or `CREATE VIRTUAL TABLE ... USING vec0` fails, and current main still records the vector store as available before any usability proof.

## Solution Assessment

No, the issue's suggested dependency pin or source rebuild is broader than needed. The narrow maintainable fix is a post-load sqlite-vec health probe with regression coverage and accurate degraded-state reporting.
