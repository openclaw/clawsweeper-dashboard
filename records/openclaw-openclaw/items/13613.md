## Summary

Keep open. Current main and the latest release still implement cron as a Gateway-local, file-backed scheduler with process-local locking/status/CLI surfaces, while the related horizontal-scaling proposal remains open rather than a merged replacement for this cron-specific HA request.

## What This Changes

The issue requests distributed coordination for cron jobs so multiple Gateway instances do not execute the same scheduled work in HA deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it with versioned config or plugin API boundaries, health/status visibility, CLI inspection, docs, and multi-process regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source, docs, release-tag inspection, and the existing discussion give high confidence that OpenClaw currently lacks distributed cron coordination.

## Solution Assessment

Unclear until maintainers choose the architecture. The maintainable path is likely a generic distributed scheduler ownership contract, possibly plugin-capable, rather than committing directly to one hardcoded Redis-only shape from the issue body.
