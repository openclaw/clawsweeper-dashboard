## Summary

Keep open. Current main still has Gateway-local cron scheduling with local state, process-local locking, and local status/CLI surfaces; no distributed lease, leader election, shared backend, or HA cron ownership model has landed, and the prior discussion also kept this open rather than treating the broader horizontal-scaling proposal as a replacement.

## What This Changes

Request adding distributed coordination or leader election so cron jobs run once across multiple OpenClaw Gateways in HA deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it behind a versioned config or plugin/API boundary with health/status visibility, CLI inspection, docs, and cross-process tests.

## Reproduction Assessment

Yes for the observable HA gap. Source inspection shows independent Gateway processes have no shared lease or leader check, while the only duplicate-run guard is process-local and the existing duplicate test stays inside one Node process.

## Solution Assessment

Unclear. Redis, etcd, DynamoDB locks, or leader election are plausible, but maintainers first need to choose the supported ownership model, backend/API surface, failure semantics, health reporting, and relationship to #25883.
