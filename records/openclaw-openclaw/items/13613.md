## Summary

Keep open. Current main still implements cron as a Gateway-local scheduler with local config/state and process-local locking; the HA coordination request remains valid feature work and still needs maintainer architecture/product direction.

## What This Changes

Add distributed ownership or locking so multiple Gateway instances do not execute the same cron jobs concurrently in HA deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it with versioned config or plugin API boundaries, health/status visibility, CLI inspection, docs, and multi-process regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source and docs inspection give high confidence that current main lacks distributed cron coordination today.

## Solution Assessment

Unclear until maintainers choose the architecture. A generic distributed scheduler ownership or lease contract is likely more maintainable than committing directly to one hardcoded Redis-only shape from the issue body.
