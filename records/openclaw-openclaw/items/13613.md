## Summary

Keep open. Current main still implements cron as Gateway-local scheduling with local files, process-local locking, and local status/CLI surfaces; no distributed lease, leader election, shared coordination backend, or HA cron ownership model has landed.

## What This Changes

The issue requests distributed coordination for cron jobs so HA or multi-gateway deployments do not execute the same scheduled job independently.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it behind a versioned config or plugin/API boundary with health/status visibility, CLI inspection, docs, and cross-process tests.

## Reproduction Assessment

Yes for the observable HA gap. Source inspection shows independent Gateway processes have no shared lease or leader check, while the only duplicate-run guard is process-local and the current duplicate test stays inside one Node process.

## Solution Assessment

Unclear. Distributed locks or leader election are plausible, but maintainers first need to choose the supported ownership model, backend/API surface, failure semantics, health reporting, and relationship to #25883.
