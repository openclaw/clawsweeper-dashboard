## Summary

Keep open. Current main still has a Gateway-local cron scheduler with local files, process-local locking, and local status/CLI surfaces; no distributed lease, leader election, shared coordination backend, or HA cron ownership model has landed.

## What This Changes

The issue requests distributed coordination for cron jobs across multiple OpenClaw Gateway instances in high-availability deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it behind a versioned config or plugin/API boundary with health/status visibility, CLI inspection, docs, and cross-process tests.

## Reproduction Assessment

Yes for the observable HA gap. Source inspection shows independent Gateway processes have no shared lease or leader check, while the only duplicate-run guard is process-local and the current duplicate test stays inside one Node process.

## Solution Assessment

Unclear. Distributed locks or leader election are plausible, but maintainers first need to choose the supported ownership model, backend/API surface, failure behavior, health reporting, CLI surface, and relationship to #25883.
