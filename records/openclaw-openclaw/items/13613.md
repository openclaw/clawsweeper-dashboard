- `P2`: This is a normal-priority production HA feature gap with meaningful but bounded scheduler and delivery impact.
- `impact:message-loss`: Uncoordinated cron execution across multiple Gateways can duplicate channel deliveries and scheduled notifications.

## Summary

Keep open: current main and the latest release still have Gateway-local cron execution with process-local serialization/active-run guards and no distributed lease, leader, coordination backend, HA health/status, or cron leader CLI surface.

## What This Changes

Request distributed coordination or locking so scheduled cron jobs do not run redundantly across multiple OpenClaw Gateways in HA deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement shared claims/leases, health and CLI visibility, docs, and multi-process Gateway tests while coordinating with https://github.com/openclaw/openclaw/issues/25883.

## Reproduction Assessment

No bug-style runtime repro is applicable; this is a feature gap. Source and docs show cron still lacks distributed ownership, leader election, or a coordination backend.

## Solution Assessment

Unclear until maintainers choose the architecture. Distributed locks or leader election are plausible, but the durable fix needs a core or plugin API boundary, failure semantics, and upgrade-safe config design.
