## Summary

Keep open: current main and the latest release still lack cross-Gateway cron ownership, and the live discussion/labels show this remains a maintainer product-decision item rather than closeable cleanup.

## What This Changes

The issue requests distributed coordination for cron execution across multiple Gateway instances so HA deployments do not run the same scheduled jobs more than once.

## Best Possible Solution

Define a supported cron ownership contract first, then add an atomic fenced claim or leader lease at the scheduler boundary with health/status visibility, operator inspection, upgrade-safe configuration, and separate-process tests.

## Reproduction Assessment

Yes, from source with high confidence: independent Gateway processes can load and reserve due jobs without any shared lease, and the current process-local lock cannot coordinate across processes. No live multi-host reproduction was run in this read-only review.

## Solution Assessment

No, not as directly proposed yet: Redis, etcd, and DynamoDB are plausible backends, but maintainers first need to choose the supported topology and whether coordination belongs in core SQLite, a plugin API, or the broader horizontal-scaling design.
