## Summary

Keep open: current main and v2026.6.5 still lack distributed cron ownership, leader election, a coordination backend, HA status, or a cron leader CLI surface; the open horizontal-scaling proposal is related but broader and does not supersede this cron-specific feature gap.

## What This Changes

The issue requests distributed coordination for cron jobs across multiple OpenClaw Gateway instances in HA deployments.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement shared claims or leases, health and CLI visibility, docs, and multi-process Gateway tests while coordinating with https://github.com/openclaw/openclaw/issues/25883.

## Reproduction Assessment

No bug-style runtime reproduction is applicable because this is a feature gap. Source and docs give a high-confidence source-level reproduction: cron still lacks distributed ownership, leader election, or a coordination backend.

## Solution Assessment

Unclear until maintainers choose the architecture. Distributed locks or leader election are plausible, but the maintainable fix needs a core or plugin API boundary, failure semantics, and upgrade-safe config design.
