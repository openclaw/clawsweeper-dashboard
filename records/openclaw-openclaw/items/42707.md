## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize Telegram route prefixes before binding:** `extensions/telegram/src/subagent-hooks.ts:51-59`
  - body: `resolveTelegramConversationId` strips only `channel:` and `telegram:`, so valid targets like `telegram:group:-100123:topic:55` or `tg:group:-100123` can be stored as `group:-100123...` and later fail to route to the real Telegram chat. Reuse the existing Telegram target parser/normalizer before binding.
  - confidence: 0.88
- **[P2] Move hooks behind the current bundled-channel boundary:** `extensions/telegram/src/subagent-hooks.ts:4-8`
  - body: The new hook module imports core internals through `../../../src/...` and the entrypoint uses the old direct `register()` shape. Current bundled channels register subagent hooks from `registerFull` through a lazy `*-hooks-api.ts` boundary, so this needs a current-main rewrite before merge.
  - confidence: 0.86
- **[P3] Initialize the lazy global service before asserting it:** `src/infra/outbound/session-binding-service.test.ts:217-221`
  - body: The added test reads `globalThis.__openclawSessionBindingService` before calling `getSessionBindingService()`, but the PR initializes that global lazily in the getter. If this test runs first or alone, `globalService` is undefined and the assertion fails.
  - confidence: 0.86
