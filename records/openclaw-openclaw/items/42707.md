## Summary

This PR targets a real Telegram thread-bound subagent-session gap that is still visible on current main, but the submitted branch should not merge as-is because it is stale against the current bundled-channel entrypoint, has Telegram target normalization bugs, and still lacks contributor-supplied real Telegram proof.

## What This Changes

The branch adds Telegram subagent lifecycle hooks, session-resume delivery-context hydration, shared session-binding globals, and tests for topic-bound `sessions_spawn(..., thread=true, mode="session")`.

## Best Possible Solution

Rework the fix on current main as a lazy Telegram `subagent-hooks-api.ts` registered from `registerFull`, use SDK conversation-runtime and Telegram target parsing surfaces, add focused regression tests, and provide redacted live Telegram proof.

## Reproduction Assessment

Yes at source level: current main still gates thread-bound subagent session spawns on `subagent_spawning`, while the Telegram bundled entry does not register that hook. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this exact patch is not the best fix because it bypasses the current bundled-channel hook boundary, deep-imports core internals from extension production code, and mishandles supported Telegram target prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize Telegram route prefixes before binding:** `extensions/telegram/src/subagent-hooks.ts:55-59`
  - body: This resolver strips only `channel:` and `telegram:` before storing the conversation ID. Supported targets such as `telegram:group:-100123:topic:55` and `tg:group:-100123` can be stored as `group:-100123...`, which later cannot resolve to the real Telegram chat; reuse the current Telegram target parser/normalizer before binding.
  - confidence: 0.88
- **[P2] Use the current bundled-channel hook boundary:** `extensions/telegram/src/subagent-hooks.ts:7-9`
  - body: The new hook module imports core internals through `../../../src/...` and is wired into the old direct `register()` shape. Current bundled channels register subagent hooks from `registerFull` through a lazy `*-hooks-api.ts` barrel and SDK exports, so this needs a current-main rewrite before it can work cleanly.
  - confidence: 0.83
- **[P3] Initialize the lazy global service before asserting it:** `src/infra/outbound/session-binding-service.test.ts:219-223`
  - body: The added test reads `globalThis.__openclawSessionBindingService` before calling `getSessionBindingService()`, but the PR initializes that global lazily in the getter. If this test runs first or in isolation, `globalService` is `undefined` and the assertion fails.
  - confidence: 0.86
