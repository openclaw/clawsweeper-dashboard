## Summary

Keep open. Current main still lacks Telegram subagent lifecycle hook registration, so the reported Telegram topic-bound `sessions_spawn` behavior remains missing. The source PR is useful context, but it is stale against the current bundled channel entry layout and has concrete functional defects that should block merging as-is.

## What This Changes

The PR branch adds Telegram subagent lifecycle hook registration, Telegram hook handlers, resumed-session routing hydration, session-binding singleton changes, and tests for topic-bound Telegram subagent session spawning.

## Best Possible Solution

Land a refreshed Telegram-local replacement that mirrors Discord's registerFull hook bridge, uses `openclaw/plugin-sdk/conversation-runtime` and Telegram-local target parsing, hydrates resumed agent runs from persisted session delivery context, and updates the native subagent docs/tests once Telegram support is enabled.

## Reproduction Assessment

Yes. A static reproduction path is a Telegram requester calling `sessions_spawn` with `thread: true` and `mode: "session"`: current main has no Telegram `subagent_spawning` registration, so the core spawn path returns the hook-unavailable error before any Telegram binding can be prepared.

## Solution Assessment

No. The intended direction is right, but this patch is not the best current fix because it uses stale entrypoint/API assumptions, imports across the extension boundary, and mishandles supported Telegram target prefixes; a boundary-compliant replacement should reuse the current bundled entry contract and Telegram parser.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the current plugin SDK hook seam:** `extensions/telegram/src/subagent-hooks.ts:6-8`
  - body: The new Telegram production hook module imports core internals through `../../../src/...`, including a stale `src/telegram` path, instead of using plugin SDK exports and Telegram-local barrels. Extensions are expected to cross into core through `openclaw/plugin-sdk/*`, and the current bundled entry registers full hooks through `registerFull`, so this branch will fail the current boundary/build shape until it is rewired.
  - confidence: 0.9
- **[P2] Normalize Telegram targets before binding:** `extensions/telegram/src/subagent-hooks.ts:51-56`
  - body: `resolveTelegramConversationId` only strips `channel:` and `telegram:`, so valid targets such as `telegram:group:-100123:topic:55` or `tg:-100123:topic:55` become malformed conversation IDs. Those malformed IDs later produce delivery targets that the Telegram send path cannot resolve; reuse the existing Telegram target parser instead.
  - confidence: 0.88
- **[P3] Initialize the global service before asserting it:** `src/infra/outbound/session-binding-service.test.ts:216-224`
  - body: The added test reads `globalThis.__openclawSessionBindingService` before calling `getSessionBindingService()`, but the proposed global service is initialized lazily. When this test runs in isolation, the read returns `undefined` and the final identity assertion fails.
  - confidence: 0.86
