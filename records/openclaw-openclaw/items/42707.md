## Summary

Keep open: current main still lacks Telegram subagent lifecycle hook registration, but this PR is not merge-ready because it predates the current bundled-plugin seam layout and has concrete target-parsing and test-isolation defects. The related open feature request #42986 is the canonical place to settle the remaining Telegram thread-binding behavior.

## What This Changes

The PR adds Telegram subagent lifecycle hook registration, session-resume delivery-context hydration, shared global session-binding state, a Telegram thread-binding prewarm helper, and focused tests.

## Best Possible Solution

Implement Telegram subagent lifecycle support on current main through `registerFull`, plugin-owned/public APIs, existing Telegram target parsing, focused regression tests, docs or changelog as needed, and a clear placement policy.

## Reproduction Assessment

Yes, at source level: current main rejects `thread:true` subagent spawns when no `subagent_spawning` hook is registered, and the Telegram entry does not register one. This review did not run live Telegram or tests because it was constrained to read-only inspection.

## Solution Assessment

No. The direction is useful, but this PR is not the best current fix because it imports core internals from a plugin, misses existing Telegram target normalization, and needs a current-main rewrite around the modern channel entry seam.

  - body: This new Telegram production file imports core internals via `../../../src/...`. Bundled plugins now need plugin-sdk or plugin-local seams, and current main keeps Telegram thread-binding code under the plugin, so this will not fit or build cleanly after rebase.

- **[P2] Normalize Telegram targets before binding:** `extensions/telegram/src/subagent-hooks.ts:60-64`
  - body: The resolver only strips `channel:` and `telegram:`, so supported forms like `telegram:group:-100123:topic:55` or `tg:group:-100123` become malformed conversation IDs. Reuse the existing Telegram target parser before binding and delivery routing.
  - confidence: 0.88
- **[P3] Initialize the lazy service before reading global state:** `src/infra/outbound/session-binding-service.test.ts:202-224`
  - body: The test reads `__openclawSessionBindingService` before `getSessionBindingService()` lazily creates it, so isolated or reordered runs can see `undefined`. Capture the service first, then compare the global value to that instance.
  - confidence: 0.79
