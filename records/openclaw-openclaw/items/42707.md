## Summary

The PR should stay open because the Telegram thread-bound subagent hook gap still exists on current main and this branch is a focused implementation candidate, but it is not merge-ready without rebase, target-normalization fixes, and real Telegram proof.

## What This Changes

The PR registers Telegram subagent lifecycle hooks, restores Telegram routing context on session resume, shares session-binding state across module instances, and adds tests for topic-bound subagent session spawning.

## Best Possible Solution

Rebase the Telegram hook work onto the current bundled-channel/plugin-boundary architecture, reuse existing Telegram target parsing and conversation-binding seams, cover DM and forum-topic routes, and add redacted live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible. Current main’s subagent spawn path gates `thread=true` session spawns on `subagent_spawning`, while the Telegram entry does not register that hook even though Telegram session-binding infrastructure exists.

## Solution Assessment

No, not as submitted. The direction is right, but the patch uses stale core-internal imports and ad hoc Telegram target parsing; the safer fix should use current plugin-boundary seams and the established Telegram target helpers.
