## Summary

Keep open: current main still lacks Telegram native subagent lifecycle hook registration, and this PR remains a relevant implementation attempt, but it is not merge-ready because it violates extension boundaries, misses Telegram target normalization, and lacks real Telegram proof.

## What This Changes

The PR adds Telegram subagent lifecycle hook registration, session-resume delivery-context hydration, shared session-binding state, a Telegram thread-binding prewarm helper, and focused tests for topic-bound subagent session spawns.

## Best Possible Solution

Refresh this work on current main by adding Telegram plugin-boundary lifecycle hooks through `registerFull`, reusing public SDK/local Telegram APIs and existing target parsing, then verify with focused tests and redacted real Telegram proof while keeping https://github.com/openclaw/openclaw/issues/42986 as the tracker.

## Reproduction Assessment

Yes, at source level: current main rejects `thread:true` native subagent spawns unless a channel registers `subagent_spawning`, and the Telegram entry still does not register that hook. I did not run live Telegram or tests because this cleanup pass is read-only.

## Solution Assessment

No. The direction is useful, but this PR is not the best current implementation because it reaches from extension code into core internals, bypasses existing Telegram target parsing, and needs a current-main rewrite around the modern channel entry seam.
