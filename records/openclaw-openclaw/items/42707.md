## Summary

Keep open: current main still lacks Telegram subagent hook registration, so the reported failure path is real, but this external PR is not merge-ready because it lacks real Telegram proof and the diff is stale/incorrect against current main.

## What This Changes

The branch registers Telegram subagent lifecycle hooks, hydrates resumed session delivery context, shares session-binding state through globals, adds a Telegram binding prewarm helper, and adds regression tests.

## Best Possible Solution

Rework the fix on current main as a lazy Telegram subagent-hooks-api.ts registered from registerFull, reuse SDK conversation-runtime and Telegram target parsing surfaces, add focused regression tests, and provide redacted live Telegram proof.

## Reproduction Assessment

Yes at source level: current main gates thread-bound subagent session spawns on subagent_spawning, while the Telegram bundled entry does not register that hook. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this exact patch is not the best fix because it bypasses the current bundled-channel hook boundary, mishandles supported Telegram target prefixes, and lacks required real Telegram proof.
