## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize Telegram targets before binding:** `extensions/telegram/src/subagent-hooks.ts:51-55`
  - body: `resolveTelegramConversationId` only strips `channel:` and `telegram:`. Supported targets such as `tg:group:-100123:topic:55` or `telegram:group:-100123:topic:55` become malformed conversation IDs like `group:-100123:topic:55`, and completion routing then builds `channel:group:-100123`, which the Telegram send path will not resolve to the original chat. Reuse the existing Telegram target parser/normalizer before binding.
  - confidence: 0.9
- **[P2] Move hook wiring behind the channel entry seam:** `extensions/telegram/src/subagent-hooks.ts:3-7`
  - body: This new production hook module deep-imports core internals and is statically imported from the Telegram entry. Current bundled channels register subagent hooks only from the full-registration path through a narrow lazy `subagent-hooks-api` barrel, and extension production code should use SDK surfaces instead of `../../../src/**` imports. Please wire Telegram the same way and expose any missing helpers through the SDK.
  - confidence: 0.82
- **[P3] Initialize the global service before reading it:** `src/infra/outbound/session-binding-service.test.ts:217-222`
  - body: The added test reads `globalThis.__openclawSessionBindingService` before forcing `getSessionBindingService()` to initialize it. If this test runs before any earlier service access, `globalService` is `undefined` and the final assertion fails; store `const service = getSessionBindingService()` before the `Reflect.get` call.
  - confidence: 0.86
