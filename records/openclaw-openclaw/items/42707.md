## Summary

Keep open: current main still lacks Telegram subagent lifecycle hook registration, so the PR addresses real remaining work, but it should not merge as-is because it crosses plugin boundaries, mishandles Telegram target formats, has a test ordering bug, and lacks real Telegram proof.

## What This Changes

The PR adds Telegram subagent lifecycle hooks, session-resume delivery-context hydration, shared session-binding state, and tests for topic-bound subagent sessions.

## Best Possible Solution

Refresh this on current main with a `registerFull` hook barrel, SDK/local seams, existing Telegram target parsing, focused regression tests, and redacted real Telegram proof while keeping https://github.com/openclaw/openclaw/issues/42986 as the tracker.

## Reproduction Assessment

Yes, at source level: current main rejects `thread:true` native subagent spawns unless a channel registers `subagent_spawning`, and the Telegram entry still does not register that hook. I did not run live Telegram or tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but this implementation is not the best current fix because it deep-imports core internals from a plugin, bypasses existing Telegram target parsing, and targets stale pre-refactor paths.
