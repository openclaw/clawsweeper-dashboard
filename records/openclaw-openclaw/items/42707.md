## Summary

Keep open: current main still has the Telegram subagent hook gap, but this PR cannot merge as-is because it targets older bundled-channel boundaries, mishandles supported Telegram target forms, has a small isolation bug in a new test, and lacks required real Telegram proof.

## What This Changes

The PR adds Telegram subagent lifecycle hook wiring, session-resume delivery-context hydration, shared session-binding state, a Telegram binding prewarm helper, and related tests.

## Best Possible Solution

Rework the fix on current main as a lazy Telegram subagent-hooks-api.ts registered from registerFull, reuse SDK conversation-runtime and Telegram target parsing APIs, add focused regression tests, and collect redacted live Telegram proof.

## Reproduction Assessment

Yes at source level: current main gates thread-bound subagent session spawns on subagent_spawning, while the Telegram bundled entry does not register that hook. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this exact patch is not the best solution because it bypasses the current bundled-channel hook boundary, mishandles supported Telegram target prefixes, and lacks required real Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize Telegram targets before binding:** `extensions/telegram/src/subagent-hooks.ts:53-60`
  - body: `resolveTelegramConversationId` strips only `channel:` and `telegram:`, so supported forms such as `telegram:group:-100123:topic:55` and `tg:group:-100123` can be stored as malformed conversation IDs. Reuse the current Telegram target parser/normalizer before creating the binding.
  - confidence: 0.88
- **[P2] Use the current bundled-channel hook boundary:** `extensions/telegram/src/subagent-hooks.ts:3-8`
  - body: The new hook module imports core internals through `../../../src/...` and the entrypoint uses the old direct `register()` shape. Current bundled channels register lifecycle hooks from `registerFull` through a lazy hook API barrel and SDK exports, so this needs a current-main rewrite before merge.
  - confidence: 0.86
- **[P3] Initialize the lazy global service before asserting it:** `src/infra/outbound/session-binding-service.test.ts:219-223`
  - body: The added test reads `globalThis.__openclawSessionBindingService` before calling `getSessionBindingService()`, but the PR initializes that global lazily in the getter. If this test runs first or in isolation, `globalService` is undefined and the assertion fails.
  - confidence: 0.86
