## Summary

Keep open: current main still lacks Telegram subagent lifecycle hook registration, so the user-visible support is not implemented; this PR is useful source work but needs current-main rework before merge.

## What This Changes

The PR adds Telegram subagent lifecycle hook wiring, session resume routing hydration, shared session-binding state, a Telegram thread-binding prewarm helper, and focused tests.

## Best Possible Solution

Rewrite the Telegram subagent hook support on current main using registerFull, plugin-sdk/local Telegram APIs, existing target parsing, focused regression tests, and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main has no Telegram subagent_spawning registration, and src/agents/subagent-spawn.ts rejects thread-bound spawns when that hook is absent.

## Solution Assessment

No. The direction is useful, but the patch predates current plugin entry/boundary design and has concrete target-normalization and test-isolation issues that should be fixed in a current-main rewrite.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep Telegram hooks behind plugin seams:** `extensions/telegram/src/subagent-hooks.ts:3-9`
  - body: This new Telegram production file imports core internals with ../../../src paths. Bundled plugins now need plugin-sdk or plugin-local seams, and current main has moved Telegram thread-binding code under the plugin, so this will not fit or build cleanly after rebase.
  - confidence: 0.86
- **[P2] Normalize Telegram targets before deriving conversation ids:** `extensions/telegram/src/subagent-hooks.ts:60-64`
  - body: The resolver only strips channel: and telegram:, so supported forms like telegram:group:-100123:topic:55 or tg:group:-100123 become malformed conversation ids. Reuse the existing Telegram target parser before binding and delivery routing.
  - confidence: 0.88
- **[P3] Initialize the lazy service before reading global state:** `src/infra/outbound/session-binding-service.test.ts:202-224`
  - body: The test reads __openclawSessionBindingService before getSessionBindingService() lazily creates it, so isolated or reordered runs can see undefined. Capture the service first, then compare the global value to that instance.
  - confidence: 0.79
