## Summary

Keep open: the fix direction is valid, but the patch currently auto-approves every supported app-server approval method in yolo mode, including permission approvals that current code can grant as session-scoped network/filesystem permissions. The external PR also has test-only validation and still needs real app-server behavior proof before merge.

## What This Changes

The PR adds a shared Codex app-server yolo auto-approval helper, wires it through main and side-thread approval handling, and adds regression tests around command/file approvals and policy boundaries.

## Best Possible Solution

Land a scoped yolo auto-approval that preserves OpenClaw policy denial, auto-accepts only the intended command/file approval families, and leaves permission approvals on the existing approval route unless maintainers explicitly choose a broader policy.

Yes, source-level: current main resolves default app-server yolo to approvalPolicy "never" and sandbox "danger-full-access", but the current approval bridge still routes supported approval requests through plugin.approval.request. I did not run a live app-server reproduction in this read-only review.

## Solution Assessment

No, not yet: the central approach is maintainable, but the implementation must avoid method-agnostic auto-approval and needs after-fix real behavior proof.
