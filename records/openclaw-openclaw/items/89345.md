This PR changes `backup verify` to resolve legacy tar hardlink linkpaths by walking the link entry's ancestor paths and adds regression coverage for valid, missing, self-link, link-to-link, tail-collision, and non-file target cases.

## Best Possible Solution

Land the narrowed verifier compatibility fix after backup/security owner sign-off, while keeping https://github.com/openclaw/openclaw/issues/89257 open for the separate create-time failure.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the best narrow code path: the verifier owns legacy archive acceptance, and current backup creation already avoids emitting new hardlink entries. The remaining question is maintainer approval of the expanded security boundary, not a mechanical patch defect.
