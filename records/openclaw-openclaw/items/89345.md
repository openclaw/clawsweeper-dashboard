## What This Changes

The PR changes `backup verify` to resolve legacy cwd-relative tar hardlink targets by walking from the link entry's ancestor directories and requiring the resolved target to be a regular `File` entry, with regression tests.

## Best Possible Solution

Land the verifier-only fix after backup/security owner approval, and keep the create-time exit-13/temp archive failure tracked in the canonical issue.

## Reproduction Assessment

Yes for the verifier half: current main source rejects the cwd-relative node-tar linkpath shape, node-tar source confirms that shape, and the PR plus linked reporter comments provide before/after CLI evidence. I did not run the real archive locally in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow verifier fix: resolve legacy targets from the link entry's ancestor chain and require a regular File entry. Changing backup creation would not repair older archives, and a loose suffix scan would weaken integrity checks.
