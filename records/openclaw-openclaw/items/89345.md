## What This Changes

This PR changes backup verification to resolve legacy cwd-relative tar hardlink targets by walking the link entry's ancestor subtree and adds verifier regression coverage.

## Best Possible Solution

Land the narrowed verifier compatibility fix after maintainer approval of the hardlink boundary, while keeping https://github.com/openclaw/openclaw/issues/89257 open for the separate create-time truncated archive failure.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 still check only the root-prepended target for cwd-relative node-tar linkpaths, and the PR plus linked issue provide before/after CLI evidence for that archive shape.

## Solution Assessment

Yes, pending maintainer boundary approval. The verifier owns compatibility for older archives, while backup creation already prevents newly created OpenClaw archives from emitting hardlink entries.
