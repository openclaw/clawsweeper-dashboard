The PR changes `backup verify` to resolve legacy cwd-relative tar hardlink targets by walking the link entry's ancestor paths and requiring the resolved target to be a regular file entry, with regression coverage.

## Solution Assessment

Yes, this appears to be the best narrow fix: the verifier owns compatibility for older archives, and the PR resolves targets from the link entry's subtree while requiring a regular file entry. The remaining question is maintainer approval of the expanded verifier boundary.
