## Summary

Keep open: the follow-up fixes the intended compatibility direction and improves proof, but the current head still validates hardlink targets against untyped archive entry paths, so self-referential or link-to-link legacy hardlink targets can pass without a real payload file.

## What This Changes

The PR updates backup verification to resolve legacy node-tar hardlink targets by walking the link entry’s ancestor directories and adds regression coverage for real deduplicated hardlink archives and tail-collision rejection.

## Best Possible Solution

Land the legacy hardlink resolver only after it validates resolved targets against typed, non-Link archive entries and adds self-link/link-to-link regression tests; keep the separate create failure tracked independently.

## Solution Assessment

No. The ancestor-walk approach is the right compatibility layer, but the implementation must resolve against typed, non-Link entries before it is the best fix.

Full review comments:

- **[P1] Require hardlink targets to resolve to payload files:** `src/commands/backup-verify.ts:314`
  - body: `archiveHasHardlinkTarget` checks resolved targets against `entries`, but that set contains every archive path, including the Link entry itself and other Link entries.
