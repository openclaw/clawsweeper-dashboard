## Summary

Keep open: current main and the latest release still lack the cwd-relative hardlink verifier fix, and this PR appears correct with sufficient real CLI proof, but it needs maintainer approval because it intentionally changes an archive verification security boundary.

## What This Changes

This PR updates `backup verify` to resolve legacy node-tar cwd-relative hardlink linkpaths against the link entry's ancestor subtree and adds regression coverage for valid and corrupt hardlink shapes.

## Best Possible Solution

Land the File-only ancestor-walk verifier compatibility fix after backup/security owner sign-off, while keeping the create-time truncated archive failure tracked in https://github.com/openclaw/openclaw/issues/89257.

## Reproduction Assessment

Yes for the verifier half: current main only handles exact/root-prefixed targets, node-tar 7.5.16 writes cwd-relative hardlink `linkpath` values, and the PR supplies before/after real CLI output plus a real `tar.c` regression fixture. I did not run the CLI locally because this review is read-only.

## Solution Assessment

Yes, this appears to be the narrowest maintainable verifier fix: resolve the legacy target from the link entry's ancestor chain and require a regular File entry. Changing backup creation would not repair older archives, and a loose suffix scan would weaken integrity checks.
