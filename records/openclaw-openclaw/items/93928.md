## Summary

Keep open: current main and the latest release still have first-page-only Feishu drive `list` and `info` behavior, and two open PRs already target the exact defect. This should track review and landing of one canonical fix rather than spawn another repair lane.

## What This Changes

The issue asks for Feishu drive `list` and `info` to follow Lark drive pagination so files beyond the first API page are reachable.

## Best Possible Solution

Land one plugin-local Feishu drive pagination fix that wires cursor inputs for `list`, drains `has_more`/`next_page_token` for `info`, preserves existing first-page behavior, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `feishu_drive` `info` for a token absent from the first `drive.file.list` response while the API reports `has_more`, or try to pass a cursor to `list` and find the schema has no cursor input.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/drive.ts` and `extensions/feishu/src/drive-schema.ts`, following the existing Feishu cursor-returning sibling patterns and adding drive regression tests.
