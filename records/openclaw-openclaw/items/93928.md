## Summary

Keep open: current main and v2026.6.8 still have first-page-only Feishu drive `list`/`info` behavior, and https://github.com/openclaw/openclaw/pull/94000 is an open closing PR for the same bug.

## What This Changes

Feishu drive `list` and `info` should follow Lark drive pagination instead of only returning or searching the first API page.

## Best Possible Solution

Review and land one plugin-local Feishu drive pagination fix, preferably the open candidate PR or a narrowed replacement, then let the issue close after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible. Current main calls `drive.file.list` only once for the affected `list` and `info` paths, and the schema cannot accept a list cursor; I did not run a live Feishu tenant test.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin by wiring the Lark SDK cursor contract for `list` and paginating `info` until the file is found or pages are exhausted; because an open closing PR exists, the best path is to review and land that candidate rather than queue another fix.
