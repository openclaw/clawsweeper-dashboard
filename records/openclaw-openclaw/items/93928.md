## Summary

Keep open: current main and v2026.6.8 still have first-page-only Feishu drive `list`/`info` behavior, and two open closing PRs already target the bug.

## What This Changes

The issue reports that Feishu drive `list` and `info` ignore Lark pagination cursors, making files beyond the first API page unreachable or falsely reported as missing.

## Best Possible Solution

Land one plugin-local Feishu drive pagination fix with explicit list cursor support, page-draining `info`, focused regression tests, and updated bundled drive skill guidance if the tool input shape changes.

## Reproduction Assessment

Yes. Current main calls `drive.file.list` only once for the affected `list` and `info` paths, and the `list` schema has no cursor input, so the reported first-page-only behavior is source-reproducible.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin by wiring the Lark SDK cursor contract for `list` and paginating `info` until the file is found or pages are exhausted; because open linked PRs already target this issue, the best path is to review and land one candidate rather than queue another fix.
