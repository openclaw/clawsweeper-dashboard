## Summary

Keep open: current main and v2026.6.8 still have first-page-only Feishu drive list/info behavior, and two open closing PRs already target the bug.

## What This Changes

The issue asks Feishu drive list/info to honor Lark pagination cursors so files beyond the first API page remain reachable.

## Best Possible Solution

Land one plugin-local Feishu drive pagination fix with explicit cursor support, page-draining info, focused regression tests, and updated drive skill guidance if the tool input shape changes.

## Reproduction Assessment

Yes, source-reproducible: current main calls drive.file.list only once for the affected list/info paths, and the schema cannot accept a list cursor. I did not run a live Feishu tenant test.

## Solution Assessment

Yes. The maintainable repair belongs in the Feishu plugin by wiring the Lark SDK cursor contract for list and paginating info until the file is found or pages are exhausted; because open linked PRs already target this issue, the best path is to review and land one candidate.
