## Summary

Keep open: current main and the latest release still have first-page-only Feishu drive `list`/`info` behavior, and https://github.com/openclaw/openclaw/pull/94000 is the active open closing PR for the bug.

## What This Changes

The issue reports that Feishu drive `list` cannot request later pages and `info` falsely reports `File not found` for files beyond the first API page.

## Best Possible Solution

Land one Feishu-plugin-local pagination fix that exposes list cursor support, drains pages for `info`, adds focused regression coverage, and uses the open linked PR as the review lane.

## Reproduction Assessment

Yes, source-reproducible: current main calls `drive.file.list` only once for the affected `list` and `info` paths, while the schema cannot accept a list cursor. I did not run a live Feishu tenant test.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin by wiring the Lark SDK cursor contract for `list` and paginating `info`; because an open linked PR already targets the bug, the best next step is to review and land or close that PR.
