## Summary

Keep open: current main and the latest release still have first-page-only Feishu drive list/info behavior, and https://github.com/openclaw/openclaw/pull/94000 plus https://github.com/openclaw/openclaw/pull/94042 remain open candidate fixes with closing links.

## What This Changes

The issue reports that Feishu drive list/info ignore the Lark pagination cursor, making files beyond the first drive.file.list page unreachable or falsely reported as missing.

## Best Possible Solution

Land one plugin-local Feishu drive pagination fix that wires cursor inputs for list, drains Lark cursor pages for info, adds focused regression coverage, and updates bundled drive guidance if the tool input shape changes.

## Reproduction Assessment

Yes, source-reproducible: current main passes only folder_token to drive.file.list for the affected paths and getFileInfo searches only the first page. I did not run a live Feishu tenant test in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs inside the Feishu plugin: use the Lark SDK cursor contract for list/info, keep sibling pagination behavior consistent, and land one linked PR rather than spawning another repair lane.
