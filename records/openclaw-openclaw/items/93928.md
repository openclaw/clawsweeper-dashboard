## Summary

Keep open: current main and v2026.6.8 still have first-page-only Feishu drive `list`/`info` behavior, while two open closing PRs already target this bug.

## What This Changes

The issue requests Feishu drive `list` cursor support and paginated `info` lookup so files beyond the first API page are reachable.

## Best Possible Solution

Land one plugin-local Feishu drive pagination fix with explicit cursor support, page-draining `info`, focused regression tests, and updated bundled drive skill guidance if the tool input shape changes.

## Reproduction Assessment

Yes, source-reproducible: current main calls `drive.file.list` only once for the affected `list` and `info` paths, and the schema cannot accept a list cursor. I did not run a live Feishu tenant test.

## Solution Assessment

Yes, the maintainable fix belongs in the Feishu plugin by wiring the Lark SDK cursor contract for `list` and paginating `info` until the file is found or pages are exhausted. Because open linked PRs already target the issue, the best path is review and land one candidate rather than queue another fix.
