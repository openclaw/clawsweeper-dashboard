## Summary

## What This Changes

The PR adds target-derived `toolContext` to forwarded exec-approval outbound delivery and focused regression coverage.

## Best Possible Solution

Land a rebased patch that carries `ChannelThreadingToolContext` from resolved exec-approval targets through delivery params, active adapter context, durable queue storage/recovery, and focused tests, with redacted live multi-channel proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves turn-source exec-approval targets, then sends forwarded notices without `toolContext`, while message-action routing uses that context to avoid ambiguous channel/target inference. I did not run a live multi-channel setup in this read-only review.

## Solution Assessment

No. Passing target-derived context is the right direction, but this PR is not the best mergeable fix until it updates the active type surface, forwards and persists the context, fixes test imports, rebases cleanly, and adds real behavior proof.
