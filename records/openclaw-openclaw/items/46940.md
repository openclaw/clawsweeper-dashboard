## Summary

Current main still drops target-derived context in forwarded exec-approval delivery, so this PR is not obsolete. The branch is still useful, but it is not mergeable as-is because the patch is stale against the active outbound type layout, still misses propagation through handler and queue replay paths, and lacks contributor-owned real behavior proof.

## What This Changes

The PR adds `toolContext` to forwarded exec-approval outbound delivery and tests that forwarded request/resolved notices carry target-derived channel context.

## Best Possible Solution

Land a rebased patch that carries `ChannelThreadingToolContext` from resolved approval targets through the active outbound context type, handler params, durable queue storage/replay, and focused tests, with redacted live multi-channel proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the approval target and sends forwarded exec-approval notices without `toolContext`, while downstream message-action routing uses that context as the channel fallback. I did not run a live multi-channel setup in this read-only review.

## Solution Assessment

No. Passing target-derived context is the right direction, but this PR is not the best complete fix until it is rebased and carries the context through the active type, channel handler, durable queue replay, and proof path.
