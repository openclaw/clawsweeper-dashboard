  - body: Current main now defines `ChannelOutboundContext` in `src/channels/plugins/outbound.types.ts`, while `types.adapters.ts` only re-exports it. After rebase, adding `toolContext` here will not type adapter contexts, so update the active outbound type instead.

  - confidence: 0.97
- **[P1] Pass toolContext into the channel handler:** `src/infra/outbound/deliver.ts:190`
  - body: Adding `toolContext` to the base context builder is not enough because the core delivery path still constructs `createChannelHandler` without `toolContext: params.toolContext`. Adapter send contexts will keep receiving `undefined`, so forwarded approvals can still lose routing context.
  - confidence: 0.95
- **[P2] Persist toolContext through delivery queue replay:** `src/infra/outbound/deliver.ts:210`
  - body: `toolContext` becomes part of delivery params, but the write-ahead queue payload, enqueue write, and recovery replay still omit it. Crash, retry, or backoff replay can resend forwarded exec-approval notices without the target-derived channel identity.
  - confidence: 0.9
