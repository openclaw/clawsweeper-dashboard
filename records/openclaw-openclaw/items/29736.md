## Summary

Keep open: current main still expands exec approvals JSON/socket defaults from `~/.openclaw` while the focused implementation PR remains open, and the issue discussion includes multiple relocated/profile deployments affected by the same path split.

## What This Changes

Exec approvals should read and write the approvals file and socket under the active OpenClaw state root instead of `~/.openclaw` when state-dir/profile isolation is configured.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002, or a maintainer-approved replacement, so approvals JSON, default socket path, effective-policy source text, docs, and focused tests all follow the active state-dir/profile contract while preserving the unset default fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction on current main is to set `OPENCLAW_STATE_DIR` conceptually and inspect the resolvers: `resolveStateDir()` honors the override, while `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand `~/.openclaw/exec-approvals.*`.

## Solution Assessment

Yes for the issue boundary, but not yet implemented on main. The narrow maintainable fix belongs in the shared exec approvals resolver plus effective-policy/docs/tests, with maintainer/security review of migration and socket-root safety before landing.
