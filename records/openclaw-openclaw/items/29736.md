## Summary

Keep open: current main still resolves exec approvals JSON, socket, effective-policy text, CLI access, and gateway read/write paths through home-backed defaults, while the focused implementation PR remains open and conflicting.

## What This Changes

The issue reports that exec approvals JSON and socket defaults are written under `~/.openclaw` instead of the active OpenClaw state directory when `OPENCLAW_STATE_DIR` or profile-scoped state is configured.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved replacement that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving the unset fallback and updating focused tests/docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: set `OPENCLAW_STATE_DIR` conceptually and inspect that `resolveStateDir()` honors it while `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand hard-coded `~/.openclaw/exec-approvals.*` defaults.

## Solution Assessment

Yes for the bug boundary, but not yet landed. The narrow maintainable fix belongs in the shared exec approvals resolver plus effective-policy display, CLI/gateway/runtime regression tests, and public approvals docs, with maintainer/security acceptance of migration and socket-root semantics.
