The issue requests that exec approval storage and socket paths honor the active OpenClaw state root/profile instead of silently using `~/.openclaw`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while documenting migration and socket-root behavior.

## Solution Assessment

Yes for the bug boundary, but not as an unattended repair. The maintainable fix is the open state-dir-aware resolver PR or an equivalent change with explicit migration and socket-root safety semantics.
