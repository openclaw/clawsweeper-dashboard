## Summary

Keep this issue open: current main still routes exec approvals file/socket storage through home-backed defaults instead of the active state-dir contract, and the focused implementation candidate is still open and unmerged.

## What This Changes

The issue reports that exec approvals storage ignores the active OpenClaw state root/profile and falls back to home-backed `~/.openclaw` paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving the unset fallback and updating focused tests/docs.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approvals file/socket resolvers still expand hard-coded home-backed defaults and the CLI/gateway call paths use those helpers.

## Solution Assessment

Yes for the bug boundary. A shared state-dir-aware exec approvals resolver plus effective-policy/docs/test updates is the narrow maintainable fix, but the open PR still needs maintainer/security review before landing.
