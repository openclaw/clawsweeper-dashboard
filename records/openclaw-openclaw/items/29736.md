## Summary

Keep this issue open: current main still resolves exec approvals storage through home-backed defaults instead of the active state-dir contract, and the focused implementation candidate remains open and unmerged.

## What This Changes

Make exec approvals JSON and socket storage follow the active OpenClaw state root/profile instead of always using the user-home OpenClaw directory.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving the unset fallback and updating focused tests/docs.

## Reproduction Assessment

Yes. Source inspection shows `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approvals file/socket resolvers still expand home-backed defaults and CLI/gateway paths call those helpers; issue comments add container and profile confirmations.

## Solution Assessment

Yes for the bug boundary. A shared state-dir-aware exec approvals resolver plus effective-policy, docs, and focused regression coverage is the narrow maintainable fix; the open PR is the current candidate.
