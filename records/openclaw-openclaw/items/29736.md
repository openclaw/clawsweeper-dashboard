## Summary

Current main remains affected, and an open fix PR already owns implementation; the protected maintainer and security-review labels also require explicit human handling rather than cleanup closure.

## What This Changes

Make exec approval file and socket resolution follow the configured active state directory instead of the default home-relative directory.

## Best Possible Solution

Use the canonical state directory for both approval artifacts, add focused state-root and profile-isolation coverage, and define a security-safe doctor migration or explicit non-migration policy for existing home-relative approval files.

## Reproduction Assessment

Yes at source level: current main still expands the hardcoded home-relative paths, directly matching multiple independent profile, service, and container reports. A live current-main run was not performed because this review is read-only.

## Solution Assessment

The canonical resolver change is the correct implementation direction, but maintainers must decide whether and how existing approval policy is migrated because silent fallback or policy loss would be security-sensitive.
