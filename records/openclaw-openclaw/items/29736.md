## Summary

Current main still hardcodes exec approval storage under the user home, so the reported state-root and profile isolation bug remains. Keep this issue open while maintainers resolve the security-sensitive migration and socket-trust questions on the conflicting canonical repair PR.

## What This Changes

Make exec approvals file and socket storage follow the active OpenClaw state directory and profile instead of always using `~/.openclaw`.

## Best Possible Solution

Resolve approval JSON, socket, CLI reporting, and profile behavior through one canonical state-root contract, with a doctor-owned migration for existing policy and explicit secure-directory checks, then prove both fresh-install and upgrade behavior.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction: set `OPENCLAW_STATE_DIR` or a profile-specific root and both approvals resolvers still expand fixed home-relative paths; several independent deployment reports corroborate the result.

## Solution Assessment

No, a path-only replacement is not the complete best fix. The correct solution must also preserve or deliberately migrate existing restrictive policy and enforce a trustworthy socket-directory boundary.
