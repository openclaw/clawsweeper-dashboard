## Summary

Keep open. Current main still resolves exec approval file/socket defaults through home-backed `~/.openclaw` paths instead of the active state-dir/profile contract, the latest release still contains that resolver, and the focused fix PR #74002 remains open and dirty rather than merged.

## What This Changes

Exec approvals should store and read their JSON file, socket path, and effective-policy host path from the active OpenClaw state root/profile instead of silently falling back to `~/.openclaw`.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy host paths from the active state-dir/profile contract, preserves legacy fallback when no state override is set, and updates focused resolver/CLI/gateway tests plus public approvals docs.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to set `OPENCLAW_STATE_DIR` to a non-default root and observe that current `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand `~/.openclaw/exec-approvals.*`; the issue comments also include concrete CLI, profile, and container repros.

## Solution Assessment

Yes. The narrowest maintainable fix is the existing #74002 direction: repair the resolver/effective-policy layer to use the state-dir contract, with focused tests and docs. A duplicate automated repair should not be queued while that PR is open.
