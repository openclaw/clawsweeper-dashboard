## Summary

Keep open. Current main and the latest release still resolve exec approvals file and socket defaults through home-backed paths instead of the active state-dir/profile contract, and #74002 is the focused open implementation candidate rather than a merged fix.

## What This Changes

The issue asks for exec approvals file and socket storage to follow the active OpenClaw state root/profile instead of defaulting to home-backed `.openclaw` paths.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy host paths from the active state-dir/profile contract, preserves the legacy unset fallback, and updates focused tests and public approvals docs.

Yes. A high-confidence source-level reproduction is to set `OPENCLAW_STATE_DIR` to a non-default root and observe that the current resolver functions still expand the home-backed approvals defaults; the issue comments also include CLI, profile, and container repros.

## Solution Assessment
