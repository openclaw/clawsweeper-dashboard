- `impact:security`: The affected state controls host command-execution approval policy, allowlists, and approval socket routing.

## Summary

Keep open: current main and the latest release still resolve exec approvals through home-backed defaults instead of the active state-dir contract, and the canonical implementation candidate at https://github.com/openclaw/openclaw/pull/74002 is still open and unmerged.

## What This Changes

The issue requests that exec approvals storage follow the active OpenClaw state directory/profile root instead of the home-directory default.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy display paths from the active state-dir/profile contract while preserving the unset legacy fallback and adding focused migration, docs, and regression coverage.

## Reproduction Assessment

Yes. A source-level repro is clear: with OPENCLAW_STATE_DIR set to a non-default root, current main's file/socket resolvers still return home-backed defaults, and CLI/Gateway helpers call those resolvers; multiple reporters confirmed the split-root behavior in real deployments.

## Solution Assessment

Yes for the bug boundary, but not as another unattended fix. The best path is the existing state-dir-aware PR or an equivalent maintainer-reviewed change with explicit legacy-store migration and socket-root safety semantics.
