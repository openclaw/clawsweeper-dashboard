## Summary

Keep this open: current main still routes exec approvals JSON, socket, effective-policy display, CLI access, and gateway read/write paths through home-backed defaults while the canonical implementation PR remains open and unmerged.

## What This Changes

Exec approvals should follow the active OpenClaw state root or profile state directory instead of always using `~/.openclaw/exec-approvals.*`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving unset fallback behavior and updating focused tests/docs.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approvals file/socket resolvers still expand hard-coded home-backed defaults that the CLI and gateway call paths use.

## Solution Assessment

Yes for the bug boundary. A shared state-dir-aware exec approvals resolver plus effective-policy/docs/test updates is the narrow maintainable fix, but the existing PR still needs maintainer/security review before merge.
