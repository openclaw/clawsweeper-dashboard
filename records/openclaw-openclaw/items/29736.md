## Summary

Close as implemented on current main: merged PR https://github.com/openclaw/openclaw/pull/92056 moves exec-approvals file/socket resolution under the active state dir, adds guarded legacy migration and fail-closed behavior, and updates tests/docs. The fix is main-only; latest stable v2026.6.5 still has the old home-relative defaults.

## What This Changes

The issue reports that exec approvals ignore the active OpenClaw state root and persist under the default home directory instead of `OPENCLAW_STATE_DIR` or profile state.

## Best Possible Solution

Keep the merged current-main implementation and close this older report; release notes should mention the fix when it ships after v2026.6.5.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the historical bug from source and release inspection: v2026.6.5 still hardcodes `~/.openclaw`, while current main no longer does. I did not run a live filesystem repro because this was a read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes. The landed fix keeps runtime on the canonical state-dir path while using doctor/startup migration and fail-closed handling for legacy home approvals, which matches the repo policy for state migration boundaries.
