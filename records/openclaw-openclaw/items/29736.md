## Summary

Keep this issue open: current main and the latest shipped tag still resolve exec approvals JSON, socket, and effective-policy source text through home-backed defaults, while the canonical implementation PR remains open and unmerged. The issue discussion has multiple independent confirmations, including container and profile-isolation cases where this can bypass intended host exec policy or make exec unusable.

## What This Changes

Exec approvals should read and write the approvals file and socket under the active OpenClaw state root/profile instead of always using `~/.openclaw`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or an equivalent maintainer-approved fix that derives approvals JSON, default socket, and effective-policy display paths from the active state-dir/profile contract while preserving the unset fallback and adding focused regression coverage plus docs updates.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Set `OPENCLAW_STATE_DIR` to a non-default root: `resolveStateDir()` honors it, but the exec approvals resolvers still return home-backed paths and the CLI/gateway paths call those helpers.

## Solution Assessment

Yes for the bug boundary, but the existing fix path needs maintainer/security acceptance before merge. The narrow maintainable solution is a shared state-dir-aware exec approvals resolver with effective-policy, docs, and regression-test updates, consolidated through the open canonical PR rather than another branch.
