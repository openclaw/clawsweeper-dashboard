## Summary

Keep this open: current main and the latest release still resolve exec approvals file/socket and effective-policy display through home-backed defaults, while the focused implementation PR remains open and unmerged.

## What This Changes

Exec approvals should follow the active OpenClaw state root instead of silently reading or writing under the default home-backed approvals path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving the unset fallback and adding focused coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approvals file/socket resolvers still expand hard-coded home-backed defaults used by CLI and gateway paths.

## Solution Assessment

Yes for the bug boundary. A shared state-dir-aware exec approvals resolver with effective-policy, docs, and regression-test updates is the narrow maintainable fix; the active PR still needs maintainer/security review before merge.
