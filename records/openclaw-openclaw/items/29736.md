## Summary

Keep open: current main and the latest release still resolve exec approvals through home-backed defaults instead of the active state-dir contract, and the focused fix candidate at https://github.com/openclaw/openclaw/pull/74002 is still open and unmerged.

## What This Changes

The issue reports that exec approvals file and socket state ignore the configured active OpenClaw state root/profile and instead use the home-backed approval store.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, socket, and effective-policy source paths from the active state-dir/profile contract with focused migration, fallback, docs, and tests.

## Reproduction Assessment

Yes. Source inspection shows the approvals file/socket resolvers ignore `OPENCLAW_STATE_DIR` while `resolveStateDir()` honors it, and multiple reporter comments describe the same split-root behavior in real deployments.

## Solution Assessment

Yes for the bug boundary, but not as a new unattended repair. The best path is the existing state-dir-aware resolver PR or an equivalent maintainer-reviewed change with explicit legacy-store migration and socket-root safety semantics.
