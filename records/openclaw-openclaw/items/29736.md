Exec approvals should resolve file, socket, and effective-policy paths from the active OpenClaw state root instead of always using `~/.openclaw`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or an equivalent maintainer-approved fix that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while documenting migration and socket-root behavior.

## Solution Assessment
