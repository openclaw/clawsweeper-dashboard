## What This Changes

The issue reports that exec approvals file/socket storage ignores the active OpenClaw state root and falls back to home-backed OpenClaw state paths.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host-source path from the active state-dir/profile contract while preserving the unset default behavior.

Yes. A high-confidence source-level reproduction is to set `OPENCLAW_STATE_DIR` to a non-default root and inspect the current resolver path: current main still expands the home-backed exec approval defaults; issue comments add CLI, profile, and container repros.
