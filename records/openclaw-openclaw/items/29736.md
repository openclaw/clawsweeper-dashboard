## Summary

Keep open. Current main still routes exec approval file and socket defaults through `~/.openclaw`, and the focused fix PR #74002 is still open and unmerged.

## What This Changes

The issue asks for exec approval storage and reporting to follow the active state directory/profile instead of the home-backed `~/.openclaw` defaults.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, socket path, and effective-policy host path from the active state-dir/profile contract while preserving default install behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path is to set `OPENCLAW_STATE_DIR` to a non-default root and observe that the current resolver still expands `~/.openclaw/exec-approvals.*`; the issue comments add CLI, profile, and container repros.

## Solution Assessment

Yes. The narrow maintainable solution is to repair the shared exec-approvals resolver/effective-policy layer and cover the CLI/gateway-facing behavior with focused tests and docs.
