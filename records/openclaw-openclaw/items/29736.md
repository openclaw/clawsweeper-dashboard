## Summary

Keep open: current main still resolves exec approvals JSON, socket defaults, CLI/gateway reads and writes, and effective-policy host-source text through home-backed defaults while the focused implementation PR remains open, unmerged, and currently conflicting.

## What This Changes

The issue reports that exec approvals storage ignores the active OpenClaw state root and writes to the home-backed approvals file instead.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved replacement that derives approvals JSON, default socket path, and effective-policy host-source text from the active state-dir/profile contract while preserving the unset default fallback and adding focused docs/tests.

Yes. A high-confidence source reproduction is to set `OPENCLAW_STATE_DIR` conceptually and inspect current main: the exec approval resolvers still expand fixed `~/.openclaw/exec-approvals.*` defaults while `resolveStateDir()` honors the override.

## Solution Assessment

Yes for the issue boundary, but unclear as merge-ready for the open PR. The narrow maintainable fix belongs in the shared exec approvals resolver plus effective-policy/docs/tests, with maintainer/security review of migration and socket-root safety before landing.
