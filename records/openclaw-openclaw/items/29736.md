## Summary

Keep this issue open: current main still resolves exec approvals file/socket storage through home-backed defaults instead of the active state-dir contract, and the canonical implementation candidate at https://github.com/openclaw/openclaw/pull/74002 remains open and unmerged.

## What This Changes

Exec approvals should store and read their JSON and socket state under the active OpenClaw state root instead of a separate home-backed `.openclaw` location.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved replacement that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while preserving the unset fallback and updating focused tests/docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR` conceptually set, `resolveStateDir()` honors it while the exec approvals path/socket resolvers still use home-backed defaults.

## Solution Assessment

Yes for the bug boundary, but it is not landed. The narrow maintainable fix belongs in the shared exec approvals resolver plus effective-policy display, CLI/gateway/runtime regression tests, docs, and maintainer-approved security semantics.
