## Summary

Keep this issue open: current main still resolves exec approvals storage and effective-policy source text through home-backed defaults, while the focused implementation candidate remains open and needs maintainer/security review.

## What This Changes

Exec approvals should follow the active OpenClaw state root/profile instead of reading or writing the default home-backed approvals file.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or an equivalent maintainer-approved fix that derives approvals JSON, socket, and effective-policy host paths from the active state-dir/profile contract while preserving intentional fallback and migration semantics.

## Reproduction Assessment

Yes. Source inspection is high-confidence: `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, but current exec approvals resolvers still expand hardcoded `~/.openclaw/exec-approvals.*` and the CLI/gateway callers use those helpers.

## Solution Assessment

Yes for the bug boundary, but not as a new automated fix PR. The narrow maintainable path is the existing state-dir-aware resolver PR or a maintainer-approved equivalent, with explicit acceptance of migration and socket-root behavior.
