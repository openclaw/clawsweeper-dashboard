## Summary

Keep open. Current main and the latest release still resolve exec approvals file/socket defaults through `~/.openclaw`, while the focused fix PR #74002 remains open and unmerged.

## What This Changes

Exec approvals should store their JSON file and socket under the active state-dir/profile root instead of the home-backed default.

## Best Possible Solution

Derive the approvals JSON path, default socket path, and effective-policy host-source display from the active state-dir/profile contract while preserving legacy defaults and adding focused regression tests/docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: set `OPENCLAW_STATE_DIR` to a non-default root and the current resolver still expands `~/.openclaw/exec-approvals.*`; the issue discussion adds CLI, profile, and container repros.

## Solution Assessment

Yes. The best path is #74002 or an equivalent narrow resolver/effective-policy fix, reviewed for the existing security comments, rather than another duplicate repair branch.
