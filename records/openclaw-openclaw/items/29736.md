## Summary

Keep open. Current `main` and the latest release still resolve exec approvals through the home-backed defaults, while the focused implementation candidate remains open and has unresolved upgrade/security questions.

## What This Changes

The issue reports that exec approvals JSON/socket state ignores the active OpenClaw state root and falls back to `~/.openclaw`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-chosen equivalent that derives approvals JSON, socket defaults, and effective-policy reporting from the active state-dir/profile contract while preserving or explicitly migrating existing approval policy.

## Reproduction Assessment

Yes, by source inspection: current `main` still resolves the approvals file and socket from `~/.openclaw` while `resolveStateDir()` separately honors `OPENCLAW_STATE_DIR`. I did not run a live container reproduction in this read-only review.

## Solution Assessment

No current-main solution has landed. The right direction is a state-dir-aware resolver, but the final fix needs maintainer-approved upgrade/security handling for existing stricter approval files.
