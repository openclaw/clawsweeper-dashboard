## Summary

Keep this issue open: current main and v2026.5.19 still resolve exec approvals file/socket and effective-policy host path through home-backed defaults, while the focused implementation candidate at https://github.com/openclaw/openclaw/pull/74002 remains open for maintainer/security review.

## What This Changes

Exec approvals should resolve policy file and socket state under the active OPENCLAW_STATE_DIR/profile state root instead of always using ~/.openclaw.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or an equivalent maintainer-approved fix that derives approvals JSON, socket, and effective-policy host-source paths from the active state-dir/profile contract, with explicit migration and socket-root safety semantics.
