## Summary

Keep open: current main and the latest shipped tag still resolve exec approvals through home-backed defaults while `OPENCLAW_STATE_DIR` is the active state-root contract, and the focused implementation candidate remains open at https://github.com/openclaw/openclaw/pull/74002.

## What This Changes

The issue asks for exec approvals file/socket storage and effective-policy reporting to follow the active OpenClaw state root instead of silently using `~/.openclaw`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or a maintainer-approved equivalent that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract while documenting migration and fallback behavior.

## Reproduction Assessment

Yes. Source inspection is high-confidence: `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approvals file/socket resolvers and effective-policy defaults still expand home-backed paths used by CLI, gateway, and runtime helpers.

## Solution Assessment

Yes for the bug boundary, but not as a new unattended repair. The maintainable fix is the open state-dir-aware resolver PR or an equivalent change with explicit migration and socket-root safety semantics.
