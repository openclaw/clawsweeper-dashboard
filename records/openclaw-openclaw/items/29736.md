## Summary

Keep open: current main and the latest release still resolve exec approvals through home-backed defaults, and the focused implementation candidate remains open at https://github.com/openclaw/openclaw/pull/74002.

## What This Changes

Exec approvals should store and read their file/socket under the active OpenClaw state root instead of always using the default home directory.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74002 or an equivalent maintainer-approved fix that derives approvals JSON, default socket, and effective-policy host-source paths from the active state-dir/profile contract with explicit migration and socket-root safety semantics.

Yes. Source inspection gives a high-confidence reproduction path: `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approval file/socket resolvers still expand the home-backed defaults used by CLI and gateway helpers.

## Solution Assessment

Yes for the bug boundary, but not as an unattended automation task. The maintainable fix is the open state-dir-aware resolver PR or an equivalent change with explicit migration and socket-root safety semantics.
