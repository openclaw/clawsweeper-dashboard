## Summary

Keep open. Current main still resolves exec approvals JSON, socket, CLI, gateway, and effective-policy display paths through home-backed defaults, while the focused fix PR #74002 remains open and unmerged.

## What This Changes

Requests exec approvals storage to follow the active `OPENCLAW_STATE_DIR` or profile state root instead of writing to default `~/.openclaw` paths.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host path from the active state-dir/profile contract while preserving legacy default behavior and updating focused tests/docs.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to set `OPENCLAW_STATE_DIR` to a non-default root and observe that current `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand `~/.openclaw/exec-approvals.*`; the comments add CLI, profile, and container repros.

## Solution Assessment
