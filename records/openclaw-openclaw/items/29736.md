## Summary

Keep open: current main still resolves exec approvals storage and socket defaults through `~/.openclaw`, while the focused implementation PR at https://github.com/openclaw/openclaw/pull/74002 remains open and unmerged.

## What This Changes

Exec approvals should follow the active OpenClaw state root for the approvals file, socket, and policy display instead of silently using the default home-backed path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74002 or an equivalent that derives approvals JSON, default socket, and effective-policy host path from the active state-dir/profile contract while preserving the unset default fallback and updating focused tests/docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to inspect current main with `OPENCLAW_STATE_DIR` set: the exec approvals resolver still expands fixed `~/.openclaw` defaults while the general state-dir resolver honors the override.

## Solution Assessment

Yes for the issue boundary. The narrow maintainable fix is in the shared exec approvals resolver plus effective-policy/docs/tests, with maintainer review of the security-sensitive migration and socket-root semantics before merge.
