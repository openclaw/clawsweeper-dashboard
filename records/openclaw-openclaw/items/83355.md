## Summary

The reported Docker-era Discord hook module-resolution failure is already fixed on current main and in shipped releases: bundled extension chunks now get a local `openclaw/plugin-sdk/*` alias under `dist/extensions/node_modules/openclaw`, while the separate direct-announce/orphan-prune behavior remains tracked elsewhere.

## What This Changes

The issue reports that a Discord-bound `subagent_ended` hook in an older Docker image failed to import `openclaw` when a spawned subagent completed.

## Best Possible Solution

Keep the shipped runtime alias wrapper path for bundled/plugin SDK imports, and leave any remaining Discord direct-announce content-loss investigation to the linked canonical issue.

## Reproduction Assessment

No: the historical v2026.4.15 source lacks the current runtime alias, but current main and v2026.5.18 include the alias wrapper path that resolves bundled `openclaw/plugin-sdk/*` imports.

## Solution Assessment

Yes: keeping plugin code on SDK subpaths and making Docker/package runtime staging provide a local `openclaw` package alias is the maintainable boundary-preserving fix for this module-resolution failure.
