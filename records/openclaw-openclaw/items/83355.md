## Summary

The Docker-era Discord hook import failure is fixed by shipped packaged plugin SDK alias wrappers; current main and v2026.5.18 stage `openclaw/plugin-sdk/*` under `dist/extensions`, so the Discord lazy `subagent_ended` hook can load. The newer direct-announce/orphan-prune output-loss report remains separately tracked.

## What This Changes

The issue asks for Discord `subagent_ended` hook imports in packaged Docker builds to resolve without `ERR_MODULE_NOT_FOUND` when `sessions_spawn` children finish.

## Best Possible Solution

Keep the shipped packaged plugin SDK alias/wrapper path, and handle any remaining native completion-delivery loss in the linked open issue.

## Reproduction Assessment

No high-confidence current-main failure reproduction is needed here: the report targets `ghcr.io/openclaw/openclaw:2026.4.15`, and current main plus v2026.5.18 include the packaged alias that prevents the source-level module-resolution failure.

## Solution Assessment

Yes for this signature: staging `openclaw/plugin-sdk/*` beside packaged extension chunks is the maintainable owner-boundary fix. The separate direct-announce/orphan-prune behavior should continue in the linked issue rather than reopening this older Docker import failure.
