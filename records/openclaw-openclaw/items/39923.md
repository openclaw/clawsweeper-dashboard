## Summary

Keep open: current main still uses numbered config backup rotation, the tests still assert that behavior, and open PRs #39982 and #40392 are already targeting the requested timestamp-suffix change.

## What This Changes

Replace numeric config backup rotation with bounded UTC datetime-suffixed backup filenames.

## Best Possible Solution

Review and choose one timestamp-suffix implementation, then land it with explicit legacy migration behavior, collision coverage, bounded pruning, permission hardening, and updated backup rotation tests.

## Solution Assessment

Yes if maintainers accept the product change: keeping the work contained to `src/config/backup-rotation.ts` and its focused tests is the narrow maintainable path. Current main has not implemented it, and the paired open PRs need review before this issue should close.
