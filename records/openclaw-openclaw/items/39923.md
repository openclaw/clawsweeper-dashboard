## Summary

Keep open: current main and the latest release still use numbered config backup rotation, and two open OpenClaw PRs already reference this issue with closing syntax, so the issue should wait for a canonical implementation decision rather than be closed by cleanup.

## What This Changes

Request switching config backup files from numeric `.bak.N` rotation to datetime-suffixed backup names while preserving bounded retention, permission hardening, collision handling, and legacy migration.

## Best Possible Solution

Review and land one canonical timestamp-suffix implementation, or consolidate it with https://github.com/openclaw/openclaw/issues/20460, with explicit legacy migration, collision-safe naming, bounded pruning, permission hardening, and focused tests.

## Solution Assessment

Yes if maintainers accept the naming change: `src/config/backup-rotation.ts` plus the focused backup rotation tests are the narrow implementation boundary. The mergeable solution still needs a chosen legacy-backup migration and collision policy across the open PRs.
