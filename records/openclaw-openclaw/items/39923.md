## Summary

Keep open: current main still uses numbered config backup rotation, focused tests still assert numbered `.bak.N` slots, and open PRs #39982 and #40392 already target this issue with closing syntax.

## What This Changes

Request to replace config backup numeric rotation files with timestamp-suffixed backup files that are easier to identify during recovery.

## Best Possible Solution

Review and land one chosen timestamp-suffix implementation, or a replacement, with explicit legacy migration behavior, collision coverage, bounded pruning, permission hardening, and updated backup rotation tests.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows config writes call `maintainConfigBackups`, and that helper still creates `.bak` plus numbered `.bak.N` files. No failing runtime bug is claimed; this is a feature request.

## Solution Assessment

Yes if maintainers accept the product change: keeping the implementation in `src/config/backup-rotation.ts` with focused tests is the narrow maintainable path. The issue should remain open until one linked PR or equivalent implementation merges.
