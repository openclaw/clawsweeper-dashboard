## Summary

Keep this issue open. Current main and the latest release still use the numbered config backup ring, tests still assert that behavior, and open PRs #39982 and #40392 already target this request with closing syntax, so this should remain open until maintainers choose and land one implementation or explicitly keep numeric rotation.

## What This Changes

The issue requests replacing numeric config backup rotation files with timestamp-suffixed backups while preserving bounded cleanup, permission hardening, collision handling, and migration safety.

## Best Possible Solution

Keep this issue open until one timestamp-suffix backup implementation is reviewed and landed, or maintainers explicitly decide to retain numeric rotation. The landed change should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts` with coverage for bounded timestamp pruning, collision handling, permission discovery, and legacy numeric backup migration.

## Reproduction Assessment

Yes for the current behavior: source inspection shows active config write paths call `maintainConfigBackups`, and that helper still creates `.bak` plus numbered `.bak.N` files. This is a feature request rather than a runtime failure, so static source, tests, release-tag, and related-PR verification are sufficient for triage.

## Solution Assessment

Yes, a bounded timestamp suffix scheme in the config backup module is the narrow maintainable direction if maintainers accept the product change. The best next step is to review and land one of the open paired PRs rather than start a separate ClawSweeper repair branch.
