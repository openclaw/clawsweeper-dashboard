## Summary

Keep open: current main and v2026.5.12 still use numbered config backup rotation, and open PRs https://github.com/openclaw/openclaw/pull/39982 and https://github.com/openclaw/openclaw/pull/40392 reference this issue with closing syntax. ## What This Changes

Request switching config backup files from numbered `.bak.N` rotation to UTC datetime-suffixed backup names.

## Best Possible Solution

Review and land one canonical timestamp-suffix implementation, or fold the naming decision into https://github.com/openclaw/openclaw/issues/20460, while preserving bounded retention, legacy migration, collision handling, permission hardening, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still creates `.bak` plus numbered `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

Yes, if maintainers accept the naming change, `src/config/backup-rotation.ts` plus focused backup-rotation tests are the narrow implementation boundary. The mergeable path still needs a chosen legacy-backup migration and collision policy across the open PRs.
