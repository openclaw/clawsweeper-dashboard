## Summary

Keep open: current main still uses numbered config backup rotation, and open PRs #39982 and #40392 already target the timestamp-suffix change with closing syntax, so this should not close before one implementation lands or is rejected.

## What This Changes

Request to replace config backup filenames from numeric `.bak.N` slots with UTC datetime suffixes while preserving bounded retention, chmod hardening, collision handling, and migration behavior.

## Best Possible Solution

Review and land one timestamp-suffix implementation, or a narrow replacement, with explicit legacy migration behavior, collision coverage, bounded pruning, permission hardening, and updated focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not a failing existing contract. Source inspection on current main confirms the current observable behavior is still numbered rotation through `maintainConfigBackups`.

## Solution Assessment

Yes if maintainers accept the product change: the narrow maintainable path is to keep the work contained to `src/config/backup-rotation.ts` and its focused tests. The remaining decision is the migration policy for existing numbered backups.
