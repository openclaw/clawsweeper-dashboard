## Risks / Open Questions

- The linked implementation can delete existing numbered recovery files on the first upgraded write unless maintainers explicitly choose and test preservation, conversion, or intentional cleanup.
- A same-timestamp filename collision can overwrite a retained backup on filesystems where rename replaces the destination, silently reducing recovery history.
- The narrow filename change should be coordinated with the broader backup-directory proposal so OpenClaw does not establish two successive incompatible backup layouts.
