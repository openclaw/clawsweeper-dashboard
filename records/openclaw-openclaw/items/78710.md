## Summary

Keep open for maintainer product judgment: the reported behavior is source-reproducible on current main, but it matches the current documented heartbeat contract and regression tests rather than an already-defined bug contract.

## What This Changes

The issue requests that agents without an explicit heartbeat inherit `agents.defaults.heartbeat` even when another agent has an explicit heartbeat override.

## Best Possible Solution

Maintainers should decide whether to preserve the documented allowlist behavior with clearer config guidance, or redesign heartbeat inheritance and update code, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection shows the reported configuration disables `main` because another agent has the only explicit heartbeat block; the current tests assert that behavior.

## Solution Assessment

No as an automated bug fix. The requested inheritance model conflicts with current docs and tests, so the maintainable path needs an explicit maintainer decision before code changes.
