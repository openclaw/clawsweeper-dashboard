## What This Changes

MiniMax Portal OAuth should refresh expired access tokens with stored refresh tokens instead of requiring manual re-authentication.

## Best Possible Solution

Review and land the existing MiniMax plugin PR (#77523), then close this issue after merge while keeping MiniMax-specific refresh logic inside the provider plugin.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an expired `minimax-portal` OAuth profile reaches core refresh, the current MiniMax provider has no `refreshOAuth` hook, and the pi-ai 0.71.1 fallback has no MiniMax OAuth provider.

## Solution Assessment

Yes. The narrow maintainable fix is a MiniMax plugin-owned `refreshOAuth` implementation through the existing provider seam; changing core auth for MiniMax-specific endpoint behavior would cross the plugin boundary unnecessarily.
