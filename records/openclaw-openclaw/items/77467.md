## Summary

Keep open: current main and the latest release still lack a MiniMax Portal `refreshOAuth` implementation, so expired `minimax-portal` OAuth credentials can still fall through to shared OAuth refresh code that does not include MiniMax. ## What This Changes

The issue asks for MiniMax Portal OAuth credentials to refresh automatically instead of expiring into missing-credential errors.

## Best Possible Solution

Keep this issue open until the MiniMax plugin owns a `refreshOAuth` implementation with CN/global routing, focused regression coverage, and redacted live or equivalent refresh-path proof.

## Reproduction Assessment

Yes, at source level: expired `minimax-portal` OAuth credentials enter the OAuth refresh path, but current main has no MiniMax `refreshOAuth` hook and the shared OAuth registry does not include MiniMax. I did not perform a live expiry refresh because valid MiniMax OAuth credentials are required.

## Solution Assessment

Yes, the best fix belongs in the provider-owned MiniMax `refreshOAuth` hook through the existing plugin seam, not in a core fallback. The implementation still needs owner/live validation for CN and global refresh behavior before it can safely land.
