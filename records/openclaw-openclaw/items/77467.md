## Summary

Keep open: current main and v2026.6.10 still lack a MiniMax Portal `refreshOAuth` implementation, so expired `minimax-portal` OAuth credentials can still fall through to shared refresh code that does not include MiniMax. ## What This Changes

The issue requests MiniMax Portal OAuth auto-refresh by adding a provider-owned refresh hook for expired `minimax-portal` credentials.

## Best Possible Solution

Implement a provider-owned MiniMax Portal `refreshOAuth` hook with safe CN/global routing for existing profiles, focused regression coverage, and redacted live or owner-validated refresh-path proof.

## Reproduction Assessment

Yes, at source level: expired `minimax-portal` OAuth credentials enter the shared OAuth refresh path, but current main has no MiniMax `refreshOAuth` hook and the shared OAuth registry does not include MiniMax. ## Solution Assessment

Yes. The best fix belongs in the provider-owned MiniMax `refreshOAuth` hook through the existing plugin seam, not in a core fallback; the implementation still needs owner or live validation for CN and global refresh behavior before it can safely land.
