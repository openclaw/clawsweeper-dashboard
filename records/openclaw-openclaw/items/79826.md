## Summary

Keep open: this active PR targets an open doctor diagnostic gap with source-backed behavior and sufficient terminal proof, but it still needs maintainer scope judgment on warning for documented object-form strict models plus one small predicate alignment fix.

## What This Changes

Adds a doctor warning for per-agent model configs that omit `fallbacks` while default model fallbacks are configured, wires it into the doctor config flow, and adds tests plus a changelog entry.

## Best Possible Solution

Land a warning-only doctor diagnostic after deciding string-only versus symmetric object-form scope, keeping runtime fallback semantics unchanged and matching the warning predicate to the resolver.

## Reproduction Assessment

Yes. Source inspection shows the reported shapes resolve to an empty fallback override that wins over default fallbacks; the PR body also supplies after-fix user-facing doctor output for the new diagnostic.

## Solution Assessment

Unclear pending maintainer scope. A warning-only doctor diagnostic is the narrow maintainable direction, but the object-form coverage should be an explicit maintainer choice and the predicate should mirror runtime primary normalization.
