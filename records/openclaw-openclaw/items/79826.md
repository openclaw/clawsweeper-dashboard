## Summary

Keep open: the doctor diagnostic is a plausible narrow fix, but the current head does not build because the malformed `agents.list` follow-up passes `unknown` `agentRuntime` values into `resolveRuntime` at two new call sites. Once the type blocker is fixed, maintainers still need to accept the broader object-form warning scope.

## What This Changes

The PR adds a doctor warning and tests for per-agent model configs that omit `fallbacks` while default model fallbacks exist, plus defensive malformed `agents.list` handling across agent and doctor scanners.

## Best Possible Solution

Land a doctor-only warning after fixing the type errors, while keeping runtime fallback semantics unchanged and making the object-form diagnostic scope an explicit maintainer choice.

## Reproduction Assessment

Yes. Source inspection shows the reported fallback override behavior on current main, and CI reproduces the PR blocker with TS2322 in `codex-route-warnings.ts` at the new `resolveRuntime` calls.

## Solution Assessment

No, not as the PR currently stands. The warning-only approach is maintainable, but the branch must first narrow or cast `agentRuntime` before passing it to `resolveRuntime`; maintainers can then decide the object-form warning scope.
