## Summary

Keep open: current main still lacks this doctor diagnostic, and the PR addresses a source-reproducible fallback footgun with sufficient terminal proof; the remaining questions are maintainer scope judgment for object-form warnings and a small predicate cleanup.

## What This Changes

This PR adds a doctor diagnostic, tests, config-flow wiring, and a changelog entry for per-agent model entries that omit `fallbacks` while default model fallbacks are configured.

## Best Possible Solution

Land a warning-only doctor diagnostic once maintainers accept the object-form scope and the warning predicate matches the runtime fallback override rules.

## Reproduction Assessment

Yes. Source inspection on current main shows the string and non-empty object-without-`fallbacks` shapes resolve to an empty fallback override, and the doctor flow currently has no matching warning path.

## Solution Assessment

Unclear as written. A doctor-only warning is the narrow maintainable direction, but expanding it to documented object form needs maintainer scope acceptance and the predicate should mirror `resolvePrimaryStringValue`.
