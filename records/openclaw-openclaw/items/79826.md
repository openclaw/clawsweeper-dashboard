## Summary

Keep open: the warning-only doctor direction is plausible for the linked issue, but this draft broadens the warning to documented object-form agent primaries and still needs end-to-end doctor proof plus small diagnostic hardening before merge.

## What This Changes

The PR adds doctor config analysis helpers and tests for per-agent model configs that omit `fallbacks`, wires the warning into doctor config loading, and adds a changelog entry.

## Best Possible Solution

Keep the warning-only doctor diagnostic aligned with runtime and docs: warn on truly implicit clobbers, sanitize config values before terminal output, and let maintainers decide whether object-form strict configs should warn or remain only documented.

## Reproduction Assessment

Yes, by source inspection: current main accepts the reported shapes, `resolveAgentModelFallbacksOverride` returns `[]` for non-empty per-agent string/object primaries without `fallbacks`, and gateway model resolution uses that empty override instead of default fallbacks. I did not execute a live doctor command because this was a read-only review.

## Solution Assessment

Unclear as drafted. The warning-only direction is maintainable for string-form configs, but expanding it to object-form `{ primary }` configs needs maintainer confirmation because current docs already define that shape as strict.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.78

Full review comments:

- **[P3] Match the runtime primary check before warning:** `src/commands/doctor-config-analysis.ts:143`
  - body: The runtime only disables default fallbacks for object-form models when `resolvePrimaryStringValue(raw)` is non-empty. This collector warns for any object with a `primary` key and no `fallbacks`, so accepted configs such as `{ primary: "" }` or whitespace-only primary get a `clobbers defaults` warning even though defaults continue to apply. Reuse the same primary normalization before classifying the shape.
  - confidence: 0.86
- **[P3] Sanitize model values before printing doctor warnings:** `src/commands/doctor-config-analysis.ts:165-166`
  - body: The warning interpolates agent ids, model strings, and fallback refs from local config directly into terminal output. Other doctor warning paths sanitize warning text first; without that, control characters or newlines in an accepted model string can forge extra warning lines or terminal output. Sanitize these values or the note before calling `note()`.
  - confidence: 0.78
