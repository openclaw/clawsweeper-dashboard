## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79856 on current main. `openclaw config set` and `config patch` must not report success while dropping explicitly written values that equal schema/runtime defaults, such as `channels.telegram.dmPolicy=pairing` and `channels.telegram.groupPolicy=allowlist`. Keep the fix in the generic config mutation/write boundary: likely thread explicit operation paths from `src/cli/config-cli.ts` through `replaceConfigFile`/`writeConfigFile` into `resolvePersistCandidateForWrite`, then force-include those paths in the persisted candidate when they were explicitly written and are not unset/include-owned paths. Do not solve this by changing Telegram defaults only, and do not globally change `createMergePatch` omitted-key deletion semantics unless deliberately coordinating with https://github.com/openclaw/openclaw/pull/79549. Add focused regression coverage showing default-valued explicit writes survive on disk and that runtime-only defaults still do not leak wholesale. Add a user-facing changelog entry if behavior changes. Validate with the targeted config CLI and config IO tests, then run the changed-lane gate if it stays narrow.
