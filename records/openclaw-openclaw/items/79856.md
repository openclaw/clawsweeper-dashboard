Current main can report success for `openclaw config set` / `config patch` while dropping explicit writes whose value equals a runtime schema default, because `resolvePersistCandidateForWrite` diffs `runtimeConfig` against `nextConfig`. Implement a generic shared fix, likely by adding explicit/forced written paths to `ConfigWriteOptions` and `resolvePersistCandidateForWrite`, passing `operation.setPath` from `runConfigOperations` through `replaceConfigFile`/`writeConfigFile`, and force-including those values in the persisted candidate even when equal to runtime defaults. Coordinate with adjacent PRs https://github.com/openclaw/openclaw/pull/79549 and https://github.com/openclaw/openclaw/pull/77558 because they touch nearby config persistence behavior.
