## Summary

Keep open. Current main still treats `mcp.servers` as a shared registry without a typed or documented per-agent scope, so `tools.deny` can hide tools but cannot prevent all matching MCP servers from being loaded, connected, or injected into backend config.

## What This Changes

Request adds an optional per-agent scope for `mcp.servers.*` so only matching agents spawn or receive each MCP server.

## Best Possible Solution

Add a backward-compatible MCP server scope resolved against the effective agent before server merge/materialization, remote connection, stdio spawn, project-settings injection, or CLI backend config injection; omitted scope should preserve today's all-agents behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple agents with any `mcp.servers` entry and current main builds one shared server map that embedded Pi and CLI backend paths consume before per-agent tool policy can filter tools.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The narrow maintainable direction is pre-spawn/pre-connect filtering by effective agent; `tools.deny` or post-catalog filtering is not enough.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested capability is security-sensitive because MCP scope would control process startup and remote MCP connections.

Concerns:

- **[medium] Define MCP scope before server startup:** `src/agents/pi-bundle-mcp-runtime.ts:228`
  - body: Any implementation must apply effective-agent scope before transport resolution, stdio process spawn, HTTP/SSE connection, tool cataloging, fingerprinting, and runtime caching; hiding tools later would still start or connect to disallowed MCP servers.
  - confidence: 0.92
- **[medium] Filter CLI backend MCP config before injection:** `src/agents/cli-runner/bundle-mcp.ts:175`
  - body: CLI runtime adapters currently receive the merged MCP map and inject it into downstream backend config. Per-agent scoping must happen before this config is written or passed to Claude, Codex, Gemini, or similar backends.
  - confidence: 0.86
