## Summary

Keep open. Current `main` and v2026.6.8 still lack generic per-agent `mcp.servers` scoping; the only implemented agent scope is the Codex app-server-only `codex.agents` projection, and duplicate reports were closed back to this issue as the canonical design thread.

## What This Changes

The issue requests optional per-agent scoping for MCP server entries so only matching agents spawn or receive those MCP servers.

## Best Possible Solution

Keep this as the canonical design thread for one additive pre-policy MCP server scoping contract covering schema, docs, runtime materialization, CLI/Codex adapters, subagent semantics, and doctor validation.

## Reproduction Assessment

Yes, from source: current generic MCP loading has no agent-aware filter, and bundle MCP runtimes are materialized before final tool policy filtering. I did not run a live multi-agent gateway process-count reproduction in this read-only review.

## Solution Assessment

Unclear on the exact implementation shape. The proposed top-level `agents` field is plausible, but it changes config, security, runtime materialization, CLI adapters, subagent semantics, and doctor validation, so maintainers should set the contract before implementation.
