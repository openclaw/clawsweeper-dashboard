## Summary

Keep open: current main and v2026.6.6 still only support Codex app-server MCP scoping via `mcp.servers.*.codex.agents`; the requested generic non-Codex server-scoping behavior remains unmerged and is being explored in https://github.com/openclaw/openclaw/pull/92221.

## What This Changes

Request an optional per-MCP-server agent allowlist so only matching agents receive or spawn that server while unscoped servers remain global.

## Best Possible Solution

Land one reviewed additive core contract that filters MCP servers before transport spawn or CLI config injection across embedded, CLI, Codex, and subagent paths, with aligned schema, docs, doctor diagnostics, and security tests.
