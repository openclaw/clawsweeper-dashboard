## Summary

Keep open. Current main still lacks a first-class, documented, and enforced `mcp.servers.*.agents` or equivalent cross-runtime server scope, and the remaining work is a security-sensitive public config contract rather than conservative cleanup.

## What This Changes

Request adds an optional per-server agent allowlist to `mcp.servers` so only matching agents spawn or receive those MCP servers.

## Best Possible Solution

Add a backward-compatible, schema-backed MCP server scope resolved against the effective agent before server spawn/connect, cataloging, caching, project settings snapshots, and CLI/Codex backend config injection.

## Reproduction Assessment

Yes for the feature gap: current main has no typed, documented, or enforced `mcp.servers.*.agents` contract, and the central merge/runtime paths consume a shared MCP server map without effective-agent filtering. I did not live-run the reporter's 12-agent memory measurement.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The requested direction is plausible, but the maintainable boundary is pre-spawn and pre-injection filtering rather than hiding tools after MCP servers are already started or projected.
