## Summary

Keep open. Current main and v2026.6.10 still only support Codex app-server-only `codex.agents`; the generic pre-policy `mcp.servers.*.agents` behavior remains unmerged and is being attempted in draft https://github.com/openclaw/openclaw/pull/92221.

## What This Changes

The issue requests an optional per-agent allowlist on `mcp.servers.*` to avoid spawning or exposing MCP servers to agents that do not need them.

## Best Possible Solution

Keep this as the canonical design thread, then land a maintainer-approved additive `mcp.servers.*.agents` contract that filters before MCP startup, CLI/Codex projection, inventory summaries, and final tool policy.

## Reproduction Assessment

Yes. Source inspection shows generic MCP loading has no active agent id or scope filter, and the existing documented agent scope is Codex app-server-only.

## Solution Assessment

Unclear, not yet. A top-level allowlist is a plausible owner-boundary shape, but the security-sensitive config semantics and the open draft PR's remaining defects need maintainer review before this is the best final solution.
