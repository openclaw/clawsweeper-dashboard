## Summary

Keep open. Current main and v2026.6.10 still only have Codex app-server `codex.agents` scoping; the requested generic `mcp.servers.*.agents` pre-policy MCP server scope remains unmerged and is being handled by the open draft candidate at https://github.com/openclaw/openclaw/pull/92221.

## What This Changes

The issue requests an optional per-agent allowlist on `mcp.servers.*` so only matching agents receive or spawn selected MCP servers.

## Best Possible Solution

Land a maintainer-approved version of https://github.com/openclaw/openclaw/pull/92221 or a successor that applies one agent-aware MCP server scope before CLI and embedded runtime materialization, aligns schema/docs/doctor behavior, and keeps the existing Codex-only projection as a compatible subset.

## Reproduction Assessment

Yes for the current behavior: source inspection shows generic MCP config merging and CLI/embedded MCP materialization have no active agent-id filter, while only the Codex app-server projection honors `codex.agents`. This is a feature/config gap rather than a broken established contract.

## Solution Assessment

Unclear until maintainers settle the security and config contract. The generic allowlist shape is plausible and already has a candidate PR, but the best final solution must explicitly decide glob/subagent semantics, fail-closed behavior, validation, docs, and upgrade expectations.
