## Summary

Keep open. Current main still has one shared MCP server registry that is merged before agent-specific tool policy is applied, so the reported RAM impact and least-privilege concern remain valid and need maintainer design approval.

## What This Changes

Add per-agent scoping to `mcp.servers.*` so OpenClaw only starts or injects MCP servers for matching agents.

## Best Possible Solution

Add a backward-compatible MCP server scope resolved against the effective agent before any configured or bundled MCP server is spawned, connected, injected into CLI backend config, cataloged, fingerprinted, or cached, with aligned types, schema, docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but the current gap is high-confidence from source inspection. A multi-agent config with `mcp.servers` still feeds the same merged MCP server map into each session MCP runtime before per-agent tool policy is applied.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The narrow maintainable direction is pre-spawn/pre-connect filtering of the effective MCP server map; `tools.deny` or post-catalog filtering does not solve process/RAM use or startup least privilege.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested feature is security-sensitive because MCP server scoping would become a pre-spawn and remote-connect access boundary.

Concerns:

- **[medium] Define MCP scoping before server startup:** `src/agents/pi-bundle-mcp-runtime.ts:228`
  - body: Any implementation must filter the effective MCP map before `resolveMcpTransport`, stdio process spawn, HTTP/SSE connection, cataloging, fingerprinting, and runtime caching; otherwise disallowed agents could still start or connect to sensitive MCP servers while only hiding tools later.
  - confidence: 0.92
- **[medium] Filter CLI backend MCP config before injection:** `src/agents/cli-runner/bundle-mcp.ts:175`
  - body: CLI runtime adapters currently receive the merged MCP map and inject it into downstream backend config. Per-agent scoping must happen before this config is written or passed to Claude, Codex, Gemini, or similar backends.
  - confidence: 0.86
