## Summary

Keep open. Current main still lacks a first-class, documented, and enforced per-agent scope for `mcp.servers.*`, and the remaining work is a new public config/security contract rather than conservative cleanup.

## What This Changes

Add optional per-agent scoping for MCP server definitions so multi-agent gateways only spawn or expose selected MCP servers to matching agents.

## Best Possible Solution

Add a backward-compatible, schema-backed MCP server scope resolved against the effective agent before configured or bundled MCP servers are spawned, connected, cataloged, cached, written to project settings, or injected into CLI/Codex backend config.

## Reproduction Assessment

Yes for the current feature gap, not as a strict bug: current main has no typed, documented, or enforced `mcp.servers.*.agents` contract and the central merge/runtime paths consume a shared MCP server map without effective-agent filtering. I did not run a live multi-agent memory measurement.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The maintainable direction is pre-spawn and pre-injection filtering; `tools.deny` or post-catalog filtering alone would not satisfy the reported resource and guardrail goals.
