## Summary

Keep open. Current main still has a shared `mcp.servers` registry with no typed or documented per-agent scope, and both embedded Pi and CLI backend paths consume the merged MCP map before agent-specific tool filtering can avoid startup, connection, or config injection.

## What This Changes

Request adds optional per-agent scoping for `mcp.servers` entries so multi-agent gateways only spawn or connect MCP servers for matching agents.

## Best Possible Solution

Add a backward-compatible MCP server scope resolved against the effective agent before configured or bundled MCP servers are spawned, connected, cataloged, cached, written to project settings, or injected into CLI backend config; omitted scope should preserve today’s all-agents default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure multiple agents with any `mcp.servers` entry and current main builds one shared server map that embedded Pi and CLI paths consume before effective-agent filtering.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The narrow maintainable direction is pre-spawn and pre-connect filtering; `tools.deny` or other post-catalog filtering is not sufficient.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested capability is security-sensitive because MCP scope would become a pre-spawn and remote-connect boundary.

Concerns:

- **[medium] Define MCP scope before server startup:** `src/agents/pi-bundle-mcp-runtime.ts:228`
  - body: Any implementation must apply effective-agent scope before transport resolution, stdio process spawn, HTTP/SSE connection, tool cataloging, fingerprinting, and runtime caching; hiding tools later would still start or connect to disallowed MCP servers.
  - confidence: 0.92
- **[medium] Filter CLI backend MCP config before injection:** `src/agents/cli-runner/bundle-mcp.ts:175`
  - body: CLI runtime adapters currently receive the merged MCP map and inject it into downstream backend config. Per-agent scoping must happen before this config is written or passed to Claude, Codex, Gemini, or similar backends.
  - confidence: 0.86
