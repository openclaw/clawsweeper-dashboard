## Summary

Keep open. Current main and the latest release only have Codex app-server-specific MCP scoping via `mcp.servers.<name>.codex.agents`; they do not provide the requested cross-runtime `mcp.servers.<name>.agents` server scope or a documented equivalent for embedded/CLI/runtime adapters.

## What This Changes

The issue requests an optional per-agent scope for configured MCP servers so multi-agent gateways do not spawn or expose every MCP server for every agent.

## Best Possible Solution

Define one canonical cross-runtime MCP server scoping contract, document its security semantics, and add focused schema, doctor, embedded/runtime-adapter, and upgrade coverage rather than extending the current Codex-only metadata by accident.

## Reproduction Assessment

Not applicable as a bug reproduction; source inspection confirms current main has Codex-only `codex.agents` scoping and no generic server-level agent scope for all MCP runtime paths.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed `agents` field is plausible, but the durable solution needs explicit runtime coverage and security semantics instead of another one-runtime projection knob.
