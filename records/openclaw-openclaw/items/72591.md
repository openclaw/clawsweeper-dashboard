## Summary

Keep open. Current main still lacks a typed, documented, and enforced per-agent scope for `mcp.servers.*`; the closed duplicate points back here as the canonical thread, and the prior unmerged PR was explicitly closed because it did not wire the allowlist through the real MCP runtime path.

## What This Changes

Add an optional per-agent allowlist to `mcp.servers.*` so configured MCP servers only start or expose tools for matching agents while preserving the current all-agents default when omitted.

## Best Possible Solution

Design and implement a backward-compatible, schema-backed MCP server scope resolved against the effective agent before server startup, cataloging, caching, project settings snapshots, and CLI backend config injection, with docs, doctor validation, and embedded plus CLI regression coverage.

## Reproduction Assessment

Yes for the current limitation, not as a strict bug: source inspection on current main shows no typed or enforced `mcp.servers.*.agents` contract, and the runtime/CLI paths consume a shared MCP server map without effective-agent filtering. I did not run a live multi-agent memory measurement.

## Solution Assessment

Unclear until maintainers approve the public config and security semantics. The maintainable direction is pre-spawn and pre-injection filtering; post-catalog tool hiding does not solve the reported process fan-out or guardrail goals.
