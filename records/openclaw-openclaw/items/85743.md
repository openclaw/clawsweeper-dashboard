## Summary

Keep open: current main and the latest shipped tag still have the unbounded heartbeat pending-final-delivery replay path, and an open linked PR now specifically owns the retry/TTL fix.

## What This Changes

The issue reports that heartbeat replay of stale `pendingFinalDeliveryText` can continue indefinitely because orphaned pending-final-delivery session state has no retry cap or TTL.

## Best Possible Solution

Review and land the linked bounded-replay fix, or replace it with an equivalent small patch that clears stale heartbeat pending-final-delivery state after a retry/age bound while preserving normal restart recovery below the limit.

Yes.
