## Label Justifications

- `P1`: The issue describes stale final text repeatedly delivered from persisted session state in real channel workflows.
- `impact:session-state`: The root condition is orphaned `pendingFinalDelivery*` session metadata that advances attempts without reaching a terminal state.
- `impact:message-loss`: The same cached outbound text can be replayed repeatedly and can block fresh heartbeat behavior behind stale delivery state.

## Summary

Keep open: current main and the latest release still have the unbounded heartbeat `pendingFinalDelivery` replay path, and the open linked fix PR should remain the owner until it merges or is replaced.

## What This Changes

Bound heartbeat `pendingFinalDelivery` replay so orphaned sessions stop re-sending stale cached final text after an attempt or age limit.

## Best Possible Solution

Review and land or replace the linked bounded heartbeat replay cleanup, preserving legitimate under-bound recovery while adding regression coverage for retry-limit and TTL expiry.
