## Summary

Keep open: current main and v2026.5.20 still have the unbounded heartbeat pending-final replay path, and https://github.com/openclaw/openclaw/pull/85770 is an open closing PR for the same cap/TTL fix. This issue should stay open until that PR, or an equivalent fix, lands.

## What This Changes

The issue requests a max-attempt and/or TTL fail-safe for orphaned heartbeat `pendingFinalDelivery` replay state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85770, or an equivalent bounded cleanup in `get-reply.ts`, with regression coverage for both retry-limit and TTL cleanup.
