## Summary

Keep open: current `main` and `v2026.6.9` suppress direct heartbeat replay for non-ack pending final text, but they still preserve orphaned `pendingFinalDelivery` state without the requested attempt cap or TTL cleanup. The open closing PR remains the direct implementation candidate, so this issue should stay open until that PR is repaired and merged or explicitly replaced.

## What This Changes

The issue requests a bounded fail-safe for heartbeat replay of orphaned `pendingFinalDelivery` state so stale final-delivery text cannot be resent indefinitely.

## Best Possible Solution

Land a current-main-compatible retry/TTL cutoff that clears the full pending-final field set while preserving heartbeat no-replay behavior for cached user-facing final text.
