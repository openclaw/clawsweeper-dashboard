- `impact:message-loss`: The affected path can duplicate stale outbound text on heartbeat ticks and suppress fresh heartbeat behavior behind cached replay state.

## Summary

Keep open: the unbounded heartbeat replay path is still present on current main, and the linked open PR uses closing syntax for this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests a retry-count and/or age fail-safe so orphaned `pendingFinalDelivery` heartbeat replay state cannot resend stale cached final text forever.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85770 or an equivalent narrow fix that clears stale heartbeat pending final delivery after a conservative attempt or age bound, then close this issue as implemented.
