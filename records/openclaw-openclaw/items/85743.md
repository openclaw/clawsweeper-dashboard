## Summary

Keep open: current `main` and the latest release suppress direct heartbeat replay of non-ack pending final text, but they still do not implement the requested attempt cap or TTL cleanup for orphaned `pendingFinalDelivery` state. The open closing PR remains the active candidate, so this issue should stay open until that PR is repaired and merged or replaced.

## What This Changes

The issue requests a fail-safe that clears orphaned heartbeat `pendingFinalDelivery` state after a bounded number of attempts or age.

## Best Possible Solution

Land a current-main-compatible retry/TTL cutoff that preserves heartbeat no-replay behavior and clears every current pending-final field, including `pendingFinalDeliveryIntentId`.

## Reproduction Assessment

Yes for the remaining cleanup gap: current source and tests show non-ack heartbeat pending-final state is preserved without an attempt or age bound. The original direct-replay symptom is partly mitigated on current `main` by the no-replay heartbeat behavior.

## Solution Assessment

No final fix is on `main` yet. The best path is the linked cutoff approach, but it needs to preserve current no-replay behavior and make the threshold/abandonment policy explicit for maintainers.
