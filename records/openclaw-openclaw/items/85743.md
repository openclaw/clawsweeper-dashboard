## Summary

Keep open: current `main` and latest release prevent direct heartbeat replay of non-ack pending final text, but they still preserve orphaned `pendingFinalDelivery` state indefinitely with no TTL or attempt cap. The open closing PR is still the active candidate, so this issue should stay open until that PR is repaired and merged or replaced.

## What This Changes

The issue requests a bounded TTL or attempt cap for orphaned heartbeat `pendingFinalDelivery` replay state so stale final text cannot be resent indefinitely.

## Best Possible Solution

Land or replace the open closing PR with a current-main-compatible retry/TTL cutoff that preserves heartbeat no-direct-replay behavior and clears every current pending-final field, including `pendingFinalDeliveryIntentId`.

## Reproduction Assessment

Yes. Current source and tests show non-ack heartbeat pending-final state remains set with no age or attempt bound; I did not run a live heartbeat loop in this read-only review.

## Solution Assessment

Yes, the bounded cleanup direction is the right core fix, but the exact threshold and abandonment policy need maintainer confirmation and the implementation must preserve the current no-direct-replay invariant.
