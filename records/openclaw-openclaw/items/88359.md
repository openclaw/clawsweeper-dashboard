## Summary

Keep this open; the branch targets a source-reproducible provider gate, but the current patch turns `apply_patch` into a default mutating tool for every provider without a provider-capability or upgrade decision and without sufficient real behavior proof.

## What This Changes

The PR removes the OpenAI-provider predicate from `apply_patch` tool construction and updates agent tool tests to expect `apply_patch` for providerless and non-OpenAI tool sets.

## Best Possible Solution

Make non-OpenAI `apply_patch` availability an explicit provider/model capability or operator opt-in, then align docs/schema/UI and prove a real non-OpenAI session.

## Reproduction Assessment

Yes. Source inspection of current main shows `apply_patch` construction is gated by `isOpenAIProvider`, so non-OpenAI providers do not receive the tool; no live non-OpenAI runtime reproduction was established here.

## Solution Assessment

No. Removing the gate unconditionally is not the narrowest maintainable fix because it broadens a mutating tool default beyond the documented contract; a provider/model capability or explicit opt-in is safer.
