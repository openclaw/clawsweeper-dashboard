## Summary

Keep open. Current main acknowledges bare `/reset` and `/new`, but it still emits reset/session lifecycle hooks with empty `messages` or `messageCount: 0`, so the reported model-backed lifecycle failure remains plausible and is not implemented away.

## What This Changes

The issue requests skipping reset/session-end model-backed lifecycle work when `/reset` or `/new` runs on an already fresh session with no message history.

## Best Possible Solution

Define an explicit already-fresh/no-history lifecycle contract so model-backed reset/session hooks skip LLM work while non-model observers can still see useful reset events.

## Reproduction Assessment

Yes. The issue provides concrete Telegram steps and provider logs, and current main still exposes empty `before_reset` payloads plus zero-message `session_end` metadata; I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. Skipping model-backed work for already-fresh resets is the right outcome, but blanket hook suppression may break observer plugins, so a targeted no-history signal or model-backed hook guard is the safer maintainable direction.
