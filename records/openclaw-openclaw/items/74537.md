## Summary

Keep open. Current main fixes the older bare `/reset` and `/new` command fallthrough, but it still emits empty-history reset/session lifecycle hook payloads that can plausibly trigger the reported lossless-claw model-backed wrap-up and user-visible Anthropic 400.

## What This Changes

Already-fresh `/reset` and `/new` should not trigger model-backed session-end or reset wrap-up work with an empty transcript or surface the provider error to the user.

## Best Possible Solution

Define and test the empty-transcript lifecycle contract so already-fresh resets do not run model-backed wrap-up work, while preserving lifecycle notifications or exposing an explicit empty/no-op signal that plugins can use safely.

## Reproduction Assessment

Yes for triage: the report includes concrete Telegram steps and provider/cooldown logs, and current source still emits empty reset/session lifecycle payloads. I did not run a live Telegram plus lossless-claw reproduction during this read-only review.

## Solution Assessment

Unclear. Skipping the failing model work is the right outcome, but blanket suppression of lifecycle hooks may be too broad; a safer fix is an explicit empty-transcript guard or accurate lifecycle signal consumed by model-backed plugins.
