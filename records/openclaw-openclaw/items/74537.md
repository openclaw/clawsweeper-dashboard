## Summary

Keep open. Current main still emits empty reset/session lifecycle payloads on the relevant paths, so the reported lossless-claw/provider 400 remains plausible and is not implemented away.

## What This Changes

The issue asks OpenClaw to skip or otherwise no-op model-backed reset/session-end lifecycle work when `/reset` or `/new` runs against an already-fresh session with no prior messages.

## Best Possible Solution

Define and test the empty-transcript lifecycle contract: preserve useful reset/session notifications, make the no-op state explicit, and prevent model-backed wrap-up work from running when there are no prior conversation messages.

## Reproduction Assessment

Yes for triage. The issue includes concrete steps and provider logs, and current source still emits empty `before_reset` plus zero-message `session_end` payloads; no live Telegram/lossless-claw reproduction was run because this review is read-only.

## Solution Assessment

Unclear. Skipping empty-history model work is the right outcome, but blanket suppression of lifecycle hooks may be too broad; a safer fix is a contract-level no-op signal or a narrow guard around model-backed lifecycle work.
