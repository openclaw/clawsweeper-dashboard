## Summary

Keep open. Current main still exposes empty reset/session lifecycle payloads after bare `/reset` or `/new`, so the reported lossless-claw/provider error path remains plausible and is not safe to close as implemented.

## What This Changes

The issue asks OpenClaw to skip model-backed reset/session-end lifecycle work when `/reset` or `/new` is run on an already-fresh session with no prior messages.

## Best Possible Solution

Define an explicit empty-history reset/session lifecycle contract that preserves observability while giving model-backed plugins a reliable no-op or `hasConversationHistory` signal to skip LLM work.

## Reproduction Assessment

Yes for triage. The issue includes concrete steps and provider logs, and current main still has source paths that emit empty `before_reset` payloads and zero-message `session_end` events; I did not run a live Telegram reproduction because this review is read-only.

## Solution Assessment

Unclear. Skipping model-backed work on empty-history resets is the right outcome, but a blanket hook suppression is riskier than a narrow contract-level no-op/history signal or a targeted guard for model-backed lifecycle work.
