## Summary

Keep open. Current main still permits empty `before_reset` and zero-message `session_end` lifecycle payloads after bare `/reset` or `/new`, so the reported provider failure path remains plausible and is not safe to close as implemented.

## What This Changes

The issue asks OpenClaw to avoid model-backed lifecycle work when `/reset` or `/new` is invoked on an already-fresh session with no conversation history.

## Best Possible Solution

Define an explicit empty-history reset/session lifecycle contract that preserves observability while giving model-backed plugins or harnesses a reliable no-op signal before they make an LLM call.

## Reproduction Assessment

Yes for triage. The issue includes concrete steps and provider logs, and current main still has code paths that emit empty reset/session lifecycle payloads.

## Solution Assessment

Unclear. Skipping model-backed work on empty-history resets is the right outcome, but a blanket hook suppression is riskier than a narrow no-op/history signal or targeted guard for model-backed lifecycle work.
