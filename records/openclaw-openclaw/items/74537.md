## Summary

Keep open. Current main still emits reset/session lifecycle events with empty reset payloads or zero-message session_end metadata, so the reported provider failure path remains plausible and is not implemented away.

## What This Changes

The issue asks OpenClaw to avoid user-visible LLM errors when `/reset` or `/new` is invoked on an already-fresh session with no conversation history.

## Best Possible Solution

Define an explicit no-history reset/session lifecycle contract so model-backed plugins skip LLM work while non-model observers keep useful reset visibility.

## Reproduction Assessment

Yes. The issue includes concrete Telegram steps and provider logs, and current main still exposes the empty before_reset and zero-message session_end paths; I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. Skipping model-backed work on already-fresh resets is the right outcome, but a blanket hook suppression is riskier than a narrow no-op/history signal or targeted guard for model-backed lifecycle work.
