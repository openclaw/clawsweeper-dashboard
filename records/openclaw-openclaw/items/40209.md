## Summary

Keep open. Current main still preserves system-event session-key casing while heartbeat session resolution uses lowercase canonical agent keys, so the PR addresses a real mismatch. The patch is narrow and functionally sound, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR lowercases system-event queue session keys in `requireSessionKey()` and adds regression tests for mixed-case channel-session lookup and cross-case deduplication.

## Best Possible Solution

Land this PR or an equivalent patch after adding the required `CHANGELOG.md` entry under the active Fixes section, preserving the mixed-case lookup and deduplication tests. Any separate global-scope targeted-wake behavior should be handled in a dedicated heartbeat-runner issue or PR.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to enqueue under `agent:main:slack:channel:C0XX` and peek or drain under `agent:main:slack:channel:c0xx`; current main stores and reads different map keys, and the PR adds tests for the expected cross-case behavior.

## Solution Assessment

Yes, with the changelog gap fixed. Normalizing at `requireSessionKey()` is the narrow shared point for enqueue, peek, drain, consume, and has-events; the only safer alternative would be agent-key-only normalization if maintainers want opaque non-agent queue keys to remain case-sensitive.
