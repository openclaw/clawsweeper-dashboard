## Summary

Keep open. Current main still preserves system-event session-key casing, while heartbeat resolution canonicalizes agent session keys to lowercase before peeking the queue, so the PR addresses a real mismatch. The functional patch looks narrowly correct, but it still needs the required changelog entry before it is ready to land.

## What This Changes

The PR lowercases `requireSessionKey()` in the in-memory system-events queue and adds two regression tests for mixed-case channel session lookup and cross-case deduplication.

## Best Possible Solution

Add the required changelog entry, then land this PR or an equivalent patch that normalizes system-event queue keys to the same canonical form used by heartbeat session resolution while keeping the mixed-case lookup and cross-case dedupe regression tests. If maintainers want non-agent custom system-event queues to remain case-sensitive, scope normalization to parseable agent session keys and document that contract.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to enqueue under `agent:main:slack:channel:C0XX` and peek or drain under `agent:main:slack:channel:c0xx`; current main addresses different queue keys, while the PR's tests encode the expected cross-case lookup.

## Solution Assessment

Yes, with the changelog gap fixed. Normalizing in `requireSessionKey()` is the narrow shared point for enqueue, peek, drain, consume, and has-events; the safer alternative is agent-key-only normalization if maintainers need case-sensitive opaque queues.
