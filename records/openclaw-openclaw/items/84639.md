## What This Changes

The issue requests an official Gateway API to inject short speech into an existing realtime relay session after async channel completions.

## Best Possible Solution

Design an additive, bounded, redacted Gateway session-speech contract if maintainers want async voice-channel completions to reuse the active realtime voice.

## Solution Assessment

Unclear pending maintainer API decision: the requested shape is plausible because the bridge can already inject text, but the permanent solution needs a typed Gateway protocol, authorization, bounded text handling, relay semantics, docs, and metadata-only observability.
