## Summary

Keep open: the PR still has a blocking correctness and proof gap because the drift detector compares disk config to a source snapshot that can already match disk before restart-required model/provider/auth runtime state is adopted.

## What This Changes

The branch adds a `runtimeConfig` health summary for live-vs-disk config drift, gates fingerprints to sensitive snapshots, bypasses health cache for admin callers, and adds health/reload regression tests.

## Best Possible Solution

Compare disk against a gateway-owned adopted-runtime source snapshot that advances only when affected model/provider/auth runtime state is actually adopted, then renew exact-head proof; otherwise narrow this PR to the independently proven cache-bypass and unknown-disk diagnostics.

## Reproduction Assessment

## Solution Assessment

No. Gateway health is the right surface, but comparing disk against `getRuntimeConfigSourceSnapshot()` is not the best fix until that snapshot is proven to represent config actually adopted by the gateway runtime.
