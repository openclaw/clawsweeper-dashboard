## Summary

The focused health diagnostic belongs on the existing admin-scoped health surface, and the latest patch resolves the earlier cache, invalid-disk, auth-path, and fingerprint-gating concerns. One text-output defect remains: an unreadable or missing disk config produces `state: "unknown"` internally but disappears from ordinary `openclaw health` text, and the live evidence predates the exact head.

## What This Changes

The PR adds gateway runtime-versus-disk config drift diagnostics to health output, limits fingerprints to sensitive admin snapshots, bypasses cached health for admin callers, and adds focused regression coverage.

## Best Possible Solution

Render both `drift` and `unknown` runtime-config states in ordinary text health output, retain fingerprint fields exclusively for `operator.admin` snapshots, and provide redacted exact-head terminal proof for drift, invalid-disk handling, cache freshness, and non-sensitive omission.

## Reproduction Assessment

Yes from source inspection: the gateway can retain a pinned runtime source snapshot while the disk source changes, and a fresh-enough shared health cache can hide that new state. This read-only review did not execute a live gateway reproduction.

## Solution Assessment

Mostly yes: the admin health surface and canonical source snapshots are the narrowest maintainable solution, but the text formatter must expose the new unknown state rather than silently discarding it.
