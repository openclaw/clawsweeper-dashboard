## Summary

Keep this PR open: the useful health diagnostic is still wired to the runtime source snapshot, which the gateway lifecycle can refresh from disk even when restart-required runtime behavior remains stale, and the PR's own sandbox proof still shows `state: "ok"` after the token rotation it is meant to detect.

## What This Changes

The branch adds `runtimeConfig` drift and unknown-disk diagnostics to gateway health snapshots, gates fingerprints by sensitive scope, bypasses cached health for admin callers, and adds regression tests.

## Best Possible Solution

Compare disk source against a gateway-owned snapshot updated only when model/provider/auth config is actually adopted by the running process, keep fingerprint and detailed disk-error output admin-only, and coordinate the snapshot boundary with the companion restart-policy PR.

## Reproduction Assessment

No. The source path is clear, but the submitted real sandbox after rotating `gateway.auth.token` returned `state: "ok"`; drift is currently proven only by injected unit tests that bypass the real file-watch lifecycle.

## Solution Assessment

No. Gateway health is the right diagnostic surface, but the live comparison source must be a gateway-adopted runtime snapshot rather than the promoted source snapshot refreshed by the same disk event.
