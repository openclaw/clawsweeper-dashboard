## Summary

Keep this PR open: latest head improves Control UI config i18n, but it still leaves visible English literals and one module-scope translation path that can stay on the fallback locale after a locale switch.

## What This Changes

The PR routes Control UI config-page copy through i18n, adds config-page translation keys across shipped locales including zh-CN, updates quick/form rendering tests, and regenerates i18n metadata.

## Best Possible Solution

Land a focused i18n fix that resolves config-page copy at render time, removes the remaining visible literals, keeps generated locale outputs in sync, and includes current-head zh-CN /config proof.

## Reproduction Assessment

Yes, at source level. Set the Control UI locale to zh-CN and open /config; current main contains literal English config quick labels, and latest PR head still leaves Advanced, per file/total, Saving, and module-scope t() paths.

## Solution Assessment

No. Moving config copy into i18n is the right direction, but this patch is incomplete because some labels remain literal and some translations are resolved before locale initialization or switching can be trusted.
