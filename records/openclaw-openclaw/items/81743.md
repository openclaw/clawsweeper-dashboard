The PR adds Simplified Chinese config quick-settings translations, changes config quick-settings theme and roundness labels to use i18n keys, and refreshes Control UI i18n generated metadata.

## Best Possible Solution

Merge after the PR resolves visible config labels at render time, binds the raw placeholder correctly, wires the added quick-card strings through the UI, refreshes generated i18n output with the established pipeline, and passes focused UI/i18n checks.

## Reproduction Assessment

Yes. Set the Control UI locale to zh-CN and open `/config`; source inspection shows current-main and PR-head quick-settings translation gaps, and the supplied screenshot confirms the mixed-language path.

## Solution Assessment

No. The approach is directionally right, but this patch is not yet the best maintainable fix because it leaves render-time translation gaps and a literal raw placeholder; the safer fix is to resolve all visible labels through the existing i18n render path and regenerate locale output.
