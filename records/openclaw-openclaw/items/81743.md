## Summary

Keep this PR open: the direction is useful, and the screenshot is real behavior proof, but the patch still leaves visible mixed English in the zh-CN config page and introduces at least one literal i18n binding bug.

## What This Changes

The PR localizes many Control UI config strings, adds zh-CN quick-settings labels, and changes config views to call the i18n helper for config-page text.

## Best Possible Solution

Keep the PR open and merge after the config page uses render-time translations for the visible quick settings text, fixes the malformed Lit bindings, regenerates locale output through the established pipeline, and adds focused zh-CN coverage.

## Reproduction Assessment

Yes. Set the Control UI locale to zh-CN and open `/config`; current main has hard-coded quick-settings labels, and the PR screenshot shows both the partially fixed path and remaining mixed-language strings.

## Solution Assessment

No. The approach is appropriate, but this patch needs a few narrow fixes before it is the maintainable solution: evaluate translation bindings correctly, avoid module-level translated labels, and wire the added quick-card keys into the rendered UI.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bind the raw placeholder through Lit:** `ui/src/ui/views/config.ts:1862`
  - body: This changes the raw-config textarea to `placeholder=t("config.raw.label")`, which Lit renders as the literal string instead of calling `t()`. In raw mode users will see the untranslated helper expression; bind it as `placeholder=${t(...)}` or use the existing raw placeholder key.
  - confidence: 0.94
- **[P2] Keep the local user label out of module scope:** `ui/src/ui/views/config-quick.ts:155`
  - body: `LOCAL_USER_LABEL` is computed with `t()` when the module loads, but non-English locale loading and later locale changes happen after that through the i18n manager. The supplied zh-CN screenshot still shows `You`; compute this label during render or pass it into the avatar preview helpers so it follows the active locale.
  - confidence: 0.9
- **[P2] Wire the added quick-card translations into the UI:** `ui/src/i18n/locales/zh-CN.ts:291-309`
  - body: The branch adds zh-CN keys for quick cards, but the proof still shows visible English headers and controls such as `Channels`, `Security`, `Appearance`, `Advanced`, `Import`, and `Text size`. The new keys need to be used by `config-quick.ts` before this fixes the reported mixed-language config page.
  - confidence: 0.86
