## Summary

Keep open: the patch targets a real OpenAI OAuth availability bug, but it is not mergeable because the new caps run after `postTokenForm` has already buffered the full token response.

## What This Changes

The PR changes the OpenAI ChatGPT/Codex OAuth token exchange and refresh paths to use shared bounded response readers for token endpoint error text and success JSON parsing.

## Best Possible Solution

Apply the 16 MiB cap at the first `postTokenForm` body read, keep exchange/refresh behavior stable, and add focused oversized token-response regression tests.

## Reproduction Assessment

Source-level yes, live replay no. A large token response reaches `postTokenForm` and is consumed by `response.arrayBuffer()` before any new bounded reader runs.

## Solution Assessment

No. The cap belongs at the first body read in `postTokenForm`; downstream bounded parsing of a rewrapped response is too late to prevent OOM.
