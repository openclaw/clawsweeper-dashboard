## Summary

Keep open: the PR is a useful OpenAI OAuth OOM hardening attempt, but the new bounded readers run after `postTokenForm` has already fully buffered the token response, so the central availability bug remains.

## What This Changes

The PR changes the OpenAI ChatGPT/Codex OAuth exchange and refresh paths to parse token endpoint error text and JSON through shared bounded response helpers.

## Best Possible Solution

Apply the 16 MiB cap at the first `postTokenForm` body read, preserve exchange/refresh behavior, and add focused oversized token-response regression coverage.

## Reproduction Assessment

Yes at source level, not as a live replay. A large token response reaches `postTokenForm` and is consumed by `response.arrayBuffer()` before the PR's new bounded readers can run.

## Solution Assessment

No. The cap belongs at the first body read in `postTokenForm`; downstream bounded parsing of a rewrapped response is too late to prevent the OOM class.
