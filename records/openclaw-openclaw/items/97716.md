## Summary

Keep open: the PR targets valid OpenAI OAuth OOM hardening, but the bounded readers run after the real token response has already been fully buffered, so the patch is not a safe fix yet.

## What This Changes

The PR replaces OpenAI ChatGPT/Codex OAuth token exchange and refresh text/json reads with shared bounded response helpers.

## Best Possible Solution

Apply the 16 MiB cap at the first body read inside `postTokenForm`, keep exchange and refresh semantics stable, and add oversized token-response tests plus redacted changed-path proof.

## Reproduction Assessment

Yes at source level, but not as a live replay. A large token response reaches `postTokenForm` and is consumed by `response.arrayBuffer()` before any new bounded reader in this PR runs.

## Solution Assessment

No. The cap belongs at the first body read in `postTokenForm`; downstream bounded parsing of a rewrapped response is too late to prevent OOM.
