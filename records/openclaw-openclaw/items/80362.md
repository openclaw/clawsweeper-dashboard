## Summary

Keep open: current main and v2026.6.8 still reject the bare grammY `Network request for 'sendMessage' failed!` envelope in send context, and the open linked PR is the active fix candidate rather than a reason to close this issue before merge.

## Best Possible Solution

Resolve the retry-safety decision, then land or replace the linked PR with a narrow Telegram classifier fix that uses dependency-backed evidence and proves the real transport retry path.
