Keep open: current main and shipped v2026.5.12 still keep the bare grammY send-context envelope non-recoverable, and an open closing PR is already paired with this issue but has unresolved retry-safety and proof blockers.

## What This Changes

Request to classify bare grammY `Network request for 'X' failed!` Telegram send-context errors as recoverable so transient pre-connect failures are retried instead of dropping outbound replies.
