## Summary

Keep this PR open. Current main still preserves the old explicit `sessionFile` during gateway `sessions.reset`, while the PR branch applies the existing compaction filename rewrite and includes the later fallback for custom filenames that review feedback requested.

## What This Changes

The PR exports the compaction session-file rewrite helpers, uses them in `performGatewaySessionReset`, and adds a gateway reset regression test for generated UUID transcript filenames.

## Best Possible Solution

Land this PR or a rebased equivalent that rewrites generated UUID, topic-suffix, and fork-pattern transcript filenames to the new reset `sessionId`, while preserving custom filenames. Put the regression test in the current gateway reset test layout and keep the daily-reset `resolveAndPersistSessionFile` fix as separate behavior.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: seed a resettable session entry whose `sessionFile` is named after the old session id, call `sessions.reset`, and current main's resolver will preserve that old explicit path for the new entry.

## Solution Assessment

Yes. Reusing the existing compaction rewrite logic is a narrow maintainable fix, and the PR author's second commit addresses the custom-name regression; the only design question is the helper's module boundary.
