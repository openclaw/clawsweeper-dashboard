## What This Changes

The PR makes ordinary unmanaged gateway restarts use the existing in-process fallback instead of detached-spawning, while preserving update respawn and updating focused tests plus low-power host docs/doctor wording.

## Best Possible Solution

Land the narrow ordinary-restart fallback once required validation is clean or unrelated failures are resolved, keeping update.run on the fresh-process path and the docs/doctor guidance aligned.

## Reproduction Assessment

Yes. At source level, current main with OPENCLAW_NO_RESPAWN unset, no trusted supervisor hints, and non-container Unix reaches detached spawn and then the run loop exits the supervisor-tracked PID.

## Solution Assessment

Yes. Returning disabled for ordinary unmanaged restarts reuses the existing in-process restart path and is narrower than broad generic supervisor detection, while leaving update.run fresh-process replacement intact.
