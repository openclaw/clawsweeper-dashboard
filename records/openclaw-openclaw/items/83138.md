## Summary

Keep open for normal maintainer review: current main and the latest release still detach-spawn ordinary unmanaged gateway restarts, while this PR changes the right helper to use the existing in-process fallback and I found no blocking patch defects.

## What This Changes

The PR changes ordinary unmanaged gateway restarts to use the in-process fallback, preserves update respawn behavior, and updates focused tests, doctor/docs wording, and the changelog.

## Best Possible Solution

Land the focused PR after maintainer and CI validation so ordinary unmanaged restarts use the in-process fallback while update restarts keep fresh-process replacement.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 detach-spawn unmanaged ordinary restarts, and the run loop exits on mode "spawned". A PID-tracking custom supervisor can then lose the replacement process and collide on the gateway port.

## Solution Assessment

Yes: returning disabled for ordinary unmanaged restarts is the narrowest maintainable fix because the run loop already has an in-process fallback, and the PR keeps update.run on respawnGatewayProcessForUpdate.
