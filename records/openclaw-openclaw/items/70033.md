## Summary

Keep open. Current main still leaves Amazon Bedrock Converse outside the malformed tool-call argument repair path, while the reported write/exec tools still require structured non-empty arguments, so the bug remains actionable rather than stale or superseded.

## What This Changes

The issue asks OpenClaw to recover or fail clearly when large Bedrock write/exec tool calls arrive with empty `{}` arguments.

## Best Possible Solution

Add focused Bedrock Converse coverage and either enable the existing malformed-argument repair for that stream shape or return a clear fail-closed recovery signal before write/exec validation repeats.

## Reproduction Assessment

Yes, at source level: simulate provider `amazon-bedrock` with API `bedrock-converse-stream` and a required-argument write or exec tool call whose streamed arguments remain `{}` or malformed. Current main does not install the malformed-argument repair wrapper for that path, so the call reaches normal required-parameter validation.

## Solution Assessment

Yes. A narrow Bedrock Converse repair or fail-closed retryable error is the maintainable fix; changing write/exec schemas, adding staged file arguments, or introducing broad retry configuration belongs to the linked feature trackers.
