## Summary

Keep open. Current main still leaves the reported Amazon Bedrock Converse stream path outside the malformed tool-call argument repair gate, while `write` and `exec` still require non-empty structured arguments.

## What This Changes

The issue requests a fix for Amazon Bedrock large `write`/`exec` tool calls that arrive with empty `{}` arguments instead of required `path`/`content` or `command` fields.

## Best Possible Solution

Add focused Bedrock Converse coverage and either enable the existing malformed-argument repair for that stream shape or add a Bedrock-specific fail-closed recovery, without changing `write`/`exec` schemas.

## Reproduction Assessment

Yes, at source level: simulate `amazon-bedrock` with `bedrock-converse-stream` and a required-argument tool call whose streamed arguments remain `{}` or malformed; current main does not install the existing repair wrapper for that path. I did not replay the live AWS size threshold.

## Solution Assessment

Yes. A narrow Bedrock stream repair or fail-closed retryable error is the maintainable fix; schema changes, global chunking, or a new truncation config would belong to the broader linked feature trackers.
