- `impact:session-state`: The reported failure is dropped or malformed tool-call argument state in the live agent turn before tool dispatch.

## Summary

Keep open. Current main still does not route Amazon Bedrock Converse streams through the malformed tool-call argument repair path, and the reported `write`/`exec` tools still fail on empty required arguments, so the Bedrock-specific bug remains actionable.

## What This Changes

The issue asks OpenClaw to handle large Bedrock `write`/`exec` tool-call arguments that arrive as empty `{}` without entering repeated validation-failure loops.

## Best Possible Solution

Add focused Bedrock Converse coverage and either route that stream shape through the existing malformed-argument repair or return a clear retryable truncation signal before `write`/`exec` validation repeats.

## Reproduction Assessment

Yes, at source level: use provider `amazon-bedrock` with API `bedrock-converse-stream` and a streamed `write` or `exec` tool call whose arguments remain `{}` or malformed. Current main does not install the malformed-argument repair wrapper for that path, so required-parameter validation is reached.

## Solution Assessment

Yes for the narrow bug: a Bedrock Converse repair or fail-closed retryable error at the stream/argument assembly boundary is the maintainable fix. Changing tool schemas or adding new large-argument transport APIs belongs to the related feature issues, not this bug fix.
