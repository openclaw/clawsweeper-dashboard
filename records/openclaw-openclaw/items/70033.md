## Summary

Keep open. Current main still excludes the reported Amazon Bedrock Converse stream path from malformed/empty tool-call argument repair, and no Bedrock-specific wrapper currently replaces that behavior, so large `write`/`exec` calls can still reach required-parameter validation as `{}`.

## What This Changes

The issue asks OpenClaw to retry or clearly surface large Bedrock `write`/`exec` tool calls that arrive with empty `{}` arguments instead of repeatedly validating the empty payload.

## Best Possible Solution

Extend the existing malformed/empty required-argument repair or fail-closed recovery to the Bedrock Converse stream path with focused tests, without changing `write`/`exec` schemas or adding broad chunking in this bug fix.

## Solution Assessment
