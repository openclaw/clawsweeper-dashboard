## Summary

Keep open: current main still builds Feishu agent-facing text with a standalone `[message_id: ...]` line plus the sender open ID/name, and the outbound reply sanitizer used before delivery does not strip that copied scaffold if the model echoes it.

## What This Changes

The issue asks OpenClaw to hide or optionally suppress Feishu internal metadata labels such as `message_id` and sender IDs from quoted/replied chat messages.

## Best Possible Solution

Preserve Feishu IDs for agent/tool context, but strip standalone `[message_id: ...]` and adjacent Feishu sender-label scaffolding at the user-facing reply boundary with focused regression tests.

## Reproduction Assessment

Yes, at source level. `buildFeishuAgentBody` still produces `[message_id: ...]\nou_...: ...`, and the outbound reply normalization path does not strip that Feishu scaffold before delivery if the assistant echoes it.

## Solution Assessment

Yes. The maintainable fix is a targeted user-facing sanitizer or Feishu outbound transform with regression coverage, not a new default-visible config option for internal metadata.
