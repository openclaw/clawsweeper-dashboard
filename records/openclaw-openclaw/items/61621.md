## Summary

Keep this open: current main still exposes the legacy Feishu `[message_id: ...]` plus sender-label shape to the agent prompt, and the user-facing reply sanitizer does not remove that shape if the model echoes it back.

## What This Changes

The issue asks OpenClaw to keep Feishu internal message metadata such as `message_id` and sender IDs out of user-visible quoted or replied messages.

## Best Possible Solution

Keep Feishu IDs available through structured inbound context and reply target fields, then strip copied legacy `[message_id: ...]` lines plus adjacent Feishu sender-label scaffolding from user-visible Feishu replies with targeted regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a Feishu turn whose assistant reply echoes `[message_id: om_x...]\nou_x...: ...` passes through the current user-facing sanitizer without removing that legacy metadata shape.

## Solution Assessment

No current fix is present on main. The narrowest maintainable solution is a Feishu/user-facing sanitization guard, not a default-visible `showMetadata` option for internal agent scaffolding.
