## Summary

Keep open: current main and the latest release still build Feishu agent text with a standalone `[message_id: ...]` line plus a sender label, and the outbound user-facing sanitizer does not strip that Feishu scaffold if the assistant echoes it.

## What This Changes

The issue asks OpenClaw to hide Feishu quote/reply metadata such as `[message_id: ...]` and `ou_*` sender IDs from user-visible replies.

## Best Possible Solution

Keep Feishu IDs in agent context, but strip copied Feishu metadata scaffold before outbound user-visible delivery while preserving ordinary inline user-authored text.

## Reproduction Assessment

Yes, at source level. Current Feishu code can produce `[message_id: om_x...]\nou_x...: ...`, and outbound reply normalization does not strip that scaffold before delivery if the assistant echoes it.

## Solution Assessment

Yes. The narrow maintainable fix is a user-facing sanitizer or Feishu outbound transform, not a new default-visible config option for internal scaffolding.
