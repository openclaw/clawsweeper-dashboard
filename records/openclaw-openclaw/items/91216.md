## Summary

Keep open: current main and the latest release still use a memory-core SQLite open/swap path that can expose an absent `main.sqlite` window and then treat missing metadata as a paused index. The report is source-reproducible and belongs in OpenClaw because it affects shipped memory-search stability.

## What This Changes

The issue reports that the gateway can create and keep using an empty memory SQLite database if `main.sqlite` is absent during index publication, leaving `memory_search` paused until restart.

## Best Possible Solution

Keep the issue open for a focused memory-core fix that removes the absent-file publish window and makes an auto-created empty handle recover without re-enabling stale-row searches.

## Reproduction Assessment

Yes from source, but not with a live gateway run in this read-only review. Current main opens absent memory DB paths with `DatabaseSync`, swaps by moving the live file away before moving the temp file in, and treats missing metadata as a paused index.

## Solution Assessment

Yes, a focused fix in memory-core is the right direction. The safest shape is to remove the absent-file publish window and make an empty auto-created handle recoverable without re-enabling stale rows from intentionally invalid metadata.
