## Summary

Keep open: the guard is a useful direction, but the current wiring validates before existing stream repair/decoding wrappers and can abort calls that the runtime would otherwise fix; the abort error also exposes raw tool arguments.

## What This Changes

The branch adds an embedded-runner stream guard and focused tests to abort after three consecutive identical tool schema validation failures.

## Best Possible Solution

Keep the schema-loop guard, but validate the post-normalized stream, redact diagnostics, and add regression coverage plus changelog before merge.

## Reproduction Assessment

Unclear for the original external ops report because no live model reproduction was available in this review. The main review blocker is source-reproducible: the new guard is installed before the wrappers that currently repair or decode tool calls.

## Solution Assessment

No. The narrow maintainable fix is to run schema-loop validation after the same stream normalization/repair path that tool execution sees, and to keep raw arguments out of surfaced abort errors.
