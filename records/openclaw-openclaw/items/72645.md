## Security Review

Status: needs_attention

Summary: The diff introduces OTLP privacy and cardinality concerns by exporting raw session identifiers and per-message IDs through external telemetry paths.

Concerns:

- **[medium] Raw session identifiers exported to OTLP:** `extensions/diagnostics-otel/src/service.ts:1166`
  - body: The PR adds openclaw.session_key and openclaw.session_id to exported span and metric attributes even though the current public OTel contract says spans never include session keys. External collectors may retain these routing identifiers outside the OpenClaw trust boundary.
  - confidence: 0.92
- **[medium] Unbounded IDs added as metric labels:** `extensions/diagnostics-otel/src/service.ts:1407`
  - body: The new message-processing metric attrs include run, message, chat, session key, and session ID values. These can create one time series per message/session and can overload or greatly increase retention cost in Prometheus-compatible collectors.
  - confidence: 0.9
