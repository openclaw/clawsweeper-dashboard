## Security Review

Status: needs_attention

Summary: The diff introduces concrete OTLP privacy and supply-cost concerns by exporting raw session/message identifiers and using them as metric labels.

Concerns:

- **[medium] Raw session identifiers exported to OTLP:** `extensions/diagnostics-otel/src/service.ts:1168`
  - body: The PR adds `openclaw.session_key` and `openclaw.session_id` attributes under names the current dropped-key set does not remove, despite the public OTel contract saying spans never include session keys. External collectors may retain these routing identifiers outside the OpenClaw trust boundary.
  - confidence: 0.94

  - body: The new message-processing metric attrs include run, message, chat, session key, and session id values. These can create one time series per message/session and can overload or greatly increase retention cost in Prometheus-compatible collectors.
