## Summary

Keep open: current main does not already implement the requested per-message and skill lifecycle telemetry, and the latest PR head still has blocking privacy, metric-cardinality, type-safety, docs, and changelog issues before it is mergeable.

## What This Changes

The PR adds message and skill lifecycle diagnostic events, emits them from the embedded runner, maps them to diagnostics-otel spans/metrics, adds a skill observability doc, and adjusts qa-lab dependency/test metadata.

## Best Possible Solution

Define a bounded, redacted message/skill telemetry contract in the existing OTel service and docs, add focused producer/exporter tests, include the required changelog entry, and keep raw session/message identifiers out of external metrics and spans unless maintainers explicitly approve a new contract.
