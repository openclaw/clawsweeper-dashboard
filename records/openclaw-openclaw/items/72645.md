The PR adds `message.processing.*` and `skill.execution.*` diagnostics events, emits them from the embedded runner, exports related OTel spans/metrics, updates diagnostic stability logging, adds a skill observability guide, and includes qa-lab/package/test adjustments.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw session IDs out of OTLP attributes:** `extensions/diagnostics-otel/src/service.ts:1166-1170`
  - body: This adds `openclaw.session_key` and `openclaw.session_id` to the shared run attribute helper under new snake_case names that the current dropped-key set does not remove. The public OpenTelemetry docs say exported spans never include session keys, so these identifiers should be omitted, hashed, or covered by an explicit reviewed privacy contract before they leave the process.
  - confidence: 0.92
- **[P1] Keep message metrics bounded:** `extensions/diagnostics-otel/src/service.ts:1407-1412`
  - body: `messageProcessingBaseAttrs` is reused for the new counter and duration histogram, so every run, message, chat, session key, and session ID value becomes an OTLP metric attribute. That creates unbounded Prometheus-style time series and also exports session routing values; keep those IDs on spans/log correlation only or replace them with bounded labels.
  - confidence: 0.9
- **[P1] Fix the skill attrs type before merging:** `extensions/diagnostics-otel/src/service.ts:2226-2228`
  - body: `skillExecutionBaseAttrs` is typed as returning possible boolean values, but this completed handler narrows the spread into `Record<string, string | number>`. TypeScript can reject this assignment, and the same pattern appears in the error handler, so narrow the helper return type or widen both attrs objects.
  - confidence: 0.87
- **[P2] Wire the new OTel guide into docs navigation:** `docs/diagnostics/otel-skill-observability.md:1`
  - body: The new diagnostics page starts directly with an H1 and is not added to the Mintlify sidebar, so it lacks the normal front matter/read_when metadata and user-facing navigation. Add docs front matter and `docs/docs.json` navigation, or fold this content into the existing OpenTelemetry export page.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff exports raw session identifiers and unbounded per-message identifiers through external OTLP telemetry paths.

Concerns:

- **[medium] Raw session identifiers exported to OTLP:** `extensions/diagnostics-otel/src/service.ts:1166`
  - body: The PR adds `openclaw.session_key` and `openclaw.session_id` to exported span/metric attributes even though the current public OTel contract says spans never include session keys. External collectors may retain these routing identifiers outside the OpenClaw trust boundary.
  - confidence: 0.92
- **[medium] Unbounded IDs added as metric labels:** `extensions/diagnostics-otel/src/service.ts:1407`
  - body: The new message-processing metric attrs include run, message, chat, session key, and session ID values. These can create one time series per message/session and can overload or greatly increase retention cost in Prometheus-compatible collectors.
  - confidence: 0.9
