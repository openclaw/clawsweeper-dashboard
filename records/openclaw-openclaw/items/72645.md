## Summary

Keep open. The latest PR still sends raw run/session/message/chat identifiers into default OTel spans and metric attributes contrary to the current privacy/cardinality contract, and the supplied proof does not show the new telemetry flowing through a live OTLP/Jaeger/Prometheus setup.

## What This Changes

The PR adds `message.processing.*` and `skill.execution.*` diagnostics events/exporter spans and metrics, docs, runner emissions, stability logging, and incidental macOS/test/qa-lab dependency changes.

## Best Possible Solution

Land a narrower telemetry change that keeps metric labels bounded, keeps raw identifiers out of default OTel export unless explicitly approved, adds focused exporter tests/docs, and includes redacted live collector proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows raw identifiers flowing into default span attributes and metric labels. The feature itself still lacks live collector proof for the exported spans and metrics.

## Solution Assessment

No. First-class message and skill telemetry may be useful, but this implementation should separate trace correlation from metric labels and stay within the existing OTel privacy contract unless maintainers approve a new opt-in policy.

  - body: The current OTel privacy contract and tests strip diagnostic identifiers, but these new snake_case attrs are not in `DROPPED_OTEL_ATTRIBUTE_KEYS`.   - confidence: 0.91

  - body: Adding run and session IDs to `openclaw.message.processed` and `openclaw.message.duration_ms` turns an existing bounded metric into per-run/session series and violates the documented label policy. Leave these metrics at channel/outcome level and keep correlation out of metric attrs.
  - confidence: 0.93

  - body: `messageProcessingBaseAttrs` includes run, message, chat, session key, and session id, then the same attrs are used for counters and histograms. That creates per-message/chat/session metric series and exports identifiers to metric backends; build separate low-cardinality metric attrs.
  - confidence: 0.94
- **[P1] Split run and session IDs from skill metrics:** `extensions/diagnostics-otel/src/service.ts:2321-2323`
  - body: `skillExecutionBaseAttrs` includes run and session identifiers before the attrs are used for skill counters and histograms. Skill metrics should stay bounded labels such as phase, outcome, and scope while any approved correlation data stays off metric labels.
  - confidence: 0.92
