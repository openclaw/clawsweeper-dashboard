## Summary

Keep open. The telemetry goal is plausible, but the current PR still conflicts with the existing OTel privacy and metric-cardinality contract, lacks live exporter proof, and carries unrelated macOS/dependency/test churn with failing checks.

## What This Changes

The PR adds message-processing and skill-execution diagnostics events, OTel spans and metrics, a new skill observability doc, and several unrelated test, dependency, gateway, pairing, and macOS formatting changes.

## Best Possible Solution

Land a narrower telemetry change that preserves redaction, keeps metric labels bounded, removes unrelated churn, adds focused exporter tests/docs, and includes redacted live OTLP proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows raw identifiers flowing into span attrs and metric labels, and live CI shows formatting/check failures. This is not a current-main bug reproduction; it is a proposed feature with source-reproducible merge blockers.

## Solution Assessment

No. First-class message and skill telemetry may be the right direction, but this implementation should separate correlation from bounded metric labels, stay within the current OTel privacy contract, and prove the real exporter path before merge.
