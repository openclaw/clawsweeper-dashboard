## Summary

Keep open. Current main does not already implement the requested message/skill lifecycle telemetry, and the PR still has merge-blocking type, privacy, and metric-cardinality problems in the proposed OTLP exporter contract.

## What This Changes

The PR adds `message.processing.*` and `skill.execution.*` diagnostics events, exports them as OpenTelemetry spans and metrics, adds a skill observability guide, and adjusts qa-lab package/test surfaces.

## Best Possible Solution

Keep the PR open while maintainers settle the stable event names, phase model, and OTLP privacy/cardinality policy, then land a narrowed implementation with contract tests, runner emission tests, exporter tests, stability-recorder coverage, and docs integrated into the existing OpenTelemetry page or sidebar.

## Reproduction Assessment

Yes for the review findings: static inspection of PR head `6582f290e888469701069c2203bb7b899c913423` shows the type mismatch and the exported identifier/cardinality issues at changed lines. Current-main inspection also confirms the requested lifecycle telemetry is not already implemented.

## Solution Assessment

No. The proposed direction may be useful, but this diff is not the best mergeable form until session identifiers are removed or reviewed, high-cardinality IDs are kept out of metrics, the skill attrs type is fixed, and docs/tests align with the final public contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw session IDs out of exported spans:** `extensions/diagnostics-otel/src/service.ts:1166-1170`
  - body: This adds raw `sessionKey` and `sessionId` values to the shared run attribute helper, using new snake_case keys that the current redaction denylist does not drop. The public OTel docs say exported spans never include session keys, so these identifiers need to be removed, hashed, or handled by an explicit reviewed privacy contract before they leave the process.
  - confidence: 0.92
- **[P1] Remove unbounded IDs from message metric labels:** `extensions/diagnostics-otel/src/service.ts:1407-1412`
  - body: `messageProcessingBaseAttrs` is reused for the new counter and duration histogram, so every run, message, chat, and session value becomes an OTLP metric attribute. That creates unbounded time-series cardinality in Prometheus-style backends and exports raw session routing values; keep these IDs on spans/log correlation only or replace them with bounded metric labels.
  - confidence: 0.9
- **[P1] Fix the skill attrs type before typechecking:** `extensions/diagnostics-otel/src/service.ts:2226-2228`
  - body: `skillExecutionBaseAttrs` is typed as possibly returning boolean values, but this `attrs` object narrows the spread to `Record<string, string | number>`. TypeScript can reject the assignment; the same pattern appears in `recordSkillExecutionError`, so the branch should narrow the helper return type or widen both attrs objects before merge.
  - confidence: 0.87
- **[P2] Wire the new OTel guide into docs metadata:** `docs/diagnostics/otel-skill-observability.md:1`
  - body: The new diagnostics page starts directly with an H1 and is not added to the Mintlify sidebar, so the user-facing guide lacks the normal front matter/read_when metadata and navigation entry. Add docs front matter and place it in `docs/docs.json`, or fold the content into the existing OpenTelemetry export page.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR exports raw session identifiers and unbounded per-message identifiers through external OTLP telemetry paths.

Concerns:

- **[medium] Raw session identifiers exported to OTLP:** `extensions/diagnostics-otel/src/service.ts:1166`
  - body: The PR adds `openclaw.session_key` and `openclaw.session_id` to exported span/metric attributes even though the current public OTel contract says spans never include session keys. External collectors may retain those routing identifiers outside the OpenClaw trust boundary.
  - confidence: 0.92
- **[medium] Unbounded IDs added as metric labels:** `extensions/diagnostics-otel/src/service.ts:1407`
  - body: The new message-processing metric attrs include run, message, chat, session key, and session ID values. These can create one time series per message/session and can overload or greatly increase retention cost in Prometheus-compatible collectors.
  - confidence: 0.9
