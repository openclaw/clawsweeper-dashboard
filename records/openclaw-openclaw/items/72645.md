Full review comments:

- **[P1] Keep raw session IDs out of OTLP attributes:** `extensions/diagnostics-otel/src/service.ts:1167-1170`
  - body: This adds `openclaw.session_key` and `openclaw.session_id`, but the existing drop list only removes the camelCase session keys. The current public OTel docs say spans never include session keys, so these identifiers need to be omitted, hashed, or explicitly approved before export.
  - confidence: 0.92
- **[P1] Keep message metrics bounded:** `extensions/diagnostics-otel/src/service.ts:1409-1412`
  - body: `messageProcessingBaseAttrs` includes run, message, chat, session key, and session id values, then the PR reuses those attrs for the new counter and duration histogram. That creates unbounded series and exports routing identifiers through metrics; keep these labels bounded before recording metrics.
  - confidence: 0.9
- **[P1] Fix the skill attrs type before merging:** `extensions/diagnostics-otel/src/service.ts:2226-2227`
  - body: `skillExecutionBaseAttrs` returns `Record<string, string | number | boolean>`, but the completed and error handlers narrow the spread into `Record<string, string | number>`. TypeScript can reject this assignment; narrow the helper return type or widen both `attrs` declarations.
  - confidence: 0.87
- **[P2] Wire the new OTel guide into docs navigation:** `docs/diagnostics/otel-skill-observability.md:1`
  - body: The new diagnostics page has no front matter/read_when metadata and is not added to the Mintlify navigation, so it will be hard to discover and does not follow the current docs contract. Add the metadata and navigation entry, or fold the content into the existing OpenTelemetry export page.
  - confidence: 0.78
