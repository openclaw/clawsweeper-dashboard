## Summary

Keep open. The observability direction is useful, but current PR head still puts per-run/session/message/chat identifiers on OTel metric labels, which conflicts with the existing low-cardinality metrics/privacy contract, and the supplied proof does not show the new spans or metrics in a live collector.

## What This Changes

The branch adds message-processing and skill-execution diagnostic events, diagnostics-otel spans and metrics, a skill observability doc, and supporting test/script/package stabilizations.

## Best Possible Solution

Land a narrower telemetry patch that keeps correlation IDs on spans/logs, keeps metrics bounded, derives skill scope from the effective snapshot, updates the public OTel catalog, and includes redacted live collector proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows identifier-heavy attrs flowing directly into OTel metric add/record calls and skillScope using the raw snapshot instead of the effective snapshot. For the feature itself, this is not a current-main bug reproduction path; the missing proof is a live collector run.

## Solution Assessment

No. First-class diagnostics may be the right direction, but this implementation should separate trace correlation attrs from metric labels, fix the sandbox skill-scope label, and prove the exported behavior in a real collector before merge.
