## Summary

Keep open. The PR adds useful diagnostics direction, but PR head still exports run/session/message/chat identifiers through metric attributes, conflicts with the current telemetry privacy/metric contract, lacks live OTLP collector proof, and overlaps an open broader skill lifecycle API request at https://github.com/openclaw/openclaw/issues/46780.

## What This Changes

The PR adds message-processing and skill-execution diagnostic events, emits them from the embedded runner, maps them to diagnostics-otel plugin spans/metrics, adds a skill observability doc, and includes several test/script/package adjustments.

## Best Possible Solution

Land a narrower diagnostics change that preserves low-cardinality metric/privacy guarantees, documents the approved trace correlation contract, removes unrelated churn, and includes redacted live OTLP collector proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows ID-heavy attrs flowing directly into OTLP metric add/record calls and the skillScope label using the raw snapshot input. The feature itself needs live collector proof rather than bug-style reproduction.

## Solution Assessment

No. First-class message and skill telemetry may be the right direction, but this implementation should split trace-only correlation attrs from low-cardinality metric attrs, align the privacy contract, fix the skill-scope label, and prove the exporter path in a real collector before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep session IDs out of message metrics:** `extensions/diagnostics-otel/src/service.ts:1410-1412`
  - body: This extends the existing `openclaw.message.processed` counter and duration histogram attrs with run/session identifiers. `message.processed` events already carry `sessionKey` and `sessionId`, so enabling the plugin would create per-session metric series and export identifiers that current OTEL/Prometheus docs keep out of metrics. Keep these fields off metric attrs.
  - confidence: 0.91
- **[P1] Split message-processing IDs from metric attrs:** `extensions/diagnostics-otel/src/service.ts:1452`
  - body: `messageProcessingBaseAttrs` includes run, message, chat, and session identifiers, then the same object is used for lifecycle counters and histograms. That makes each message/session a metric label set and exposes correlation IDs to metric backends; use separate low-cardinality metric attrs and reserve these IDs for an approved trace contract.
  - confidence: 0.92
- **[P1] Split skill correlation IDs from metric attrs:** `extensions/diagnostics-otel/src/service.ts:2336`
  - body: The skill lifecycle attrs include `openclaw.run_id`, `openclaw.session_key`, and `openclaw.session_id`, then record them on skill counters and histograms. Skill metrics should stay bounded, such as phase/outcome/scope, while run/session correlation belongs outside metric labels.
  - confidence: 0.89
- **[P3] Use the effective skills snapshot for scope labels:** `src/agents/pi-embedded-runner/run/attempt.ts:968-970`
  - body: `skillsSnapshotForRun` is cleared when a sandboxed run cannot use the snapshot, but `skillScope` still checks `params.skillsSnapshot`. Those runs will be labeled `snapshot` even though skill resolution uses the workspace path; base the label on `skillsSnapshotForRun` so telemetry matches runtime behavior.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff adds sensitive, high-cardinality identifiers to OTLP metric and span attributes.

Concerns:

- **[medium] Unbounded identifiers in telemetry metrics:** `extensions/diagnostics-otel/src/service.ts:1452`
  - body: The patch passes run_id, session_key, session_id, messageId, and chatId through attrs used by metric add/record calls. External or multi-tenant collectors could retain these identifiers, and the per-message/session cardinality can create unbounded time series.
  - confidence: 0.9
- **[medium] New snake_case IDs bypass the existing drop list:** `extensions/diagnostics-otel/src/service.ts:1436`
  - body: The current OTEL attribute drop list covers camelCase identifier keys, while the patch exports snake_case keys such as `openclaw.session_key` and `openclaw.run_id` through span attrs. That conflicts with the current docs promise that session keys are not exported by default.
  - confidence: 0.84
