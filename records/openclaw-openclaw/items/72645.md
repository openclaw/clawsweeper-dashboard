Full review comments:

- **[P1] Remove raw session IDs from OTel span attrs:** `extensions/diagnostics-otel/src/service.ts:1207-1211`
  - body: `addRunAttrs` now writes `openclaw.session_key` and `openclaw.session_id`, but the drop list only removes the existing camelCase keys before spans are exported. The OTel docs promise spans never include session keys, so keep these out of exported spans or use an approved hash/redaction design.
  - confidence: 0.94
- **[P1] Make skill attrs accept the base attr type:** `extensions/diagnostics-otel/src/service.ts:2354-2355`
  - body: `skillExecutionBaseAttrs` returns `Record<string, string | number | boolean>`, but completed/error attrs are annotated as `Record<string, string | number>`. Under strict TypeScript, spreading the wider record into the narrower annotation is not assignable; narrow the base return type or widen both attrs declarations.
  - confidence: 0.86
- **[P2] Keep run and session IDs off message metrics:** `extensions/diagnostics-otel/src/service.ts:1410-1412`
  - body: The existing `message.processed` counter and histogram now receive run/session identifiers as labels. That creates per-run/session series and leaks routing identifiers where current main only records bounded channel/outcome labels.
  - confidence: 0.92
- **[P2] Keep per-message IDs off lifecycle metric labels:** `extensions/diagnostics-otel/src/service.ts:1436-1441`
  - body: `messageProcessingBaseAttrs` includes run, message, chat, and session identifiers, then those attrs are recorded on the new counter and histogram. Use bounded labels for metrics and keep any approved correlation identifiers off metric label sets.
  - confidence: 0.93
- **[P2] Keep run and session IDs out of skill metrics:** `extensions/diagnostics-otel/src/service.ts:2321-2323`
  - body: `skillExecutionBaseAttrs` carries `openclaw.run_id` plus session identifiers and is recorded on skill counters/histograms. This makes skill metrics high-cardinality per run/session; use bounded labels such as phase, scope, and outcome.
  - confidence: 0.9
- **[P3] Publish the guide through canonical OTel docs:** `docs/diagnostics/otel-skill-observability.md:1`
  - body: The new diagnostics page has no front matter and is not wired into docs navigation, while the published diagnostics navigation points to `gateway/opentelemetry`. Fold the content into the canonical page or add proper metadata/navigation.
  - confidence: 0.78

  - confidence: 0.9
