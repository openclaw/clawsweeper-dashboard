## Summary

Keep open. Current main does not implement the requested per-message and skill lifecycle telemetry, but this PR is not mergeable as-is because the diff exports raw session identifiers and unbounded IDs through OTLP, still has a skill-attrs type mismatch, lacks docs navigation/changelog follow-through, and is currently dirty against the base branch.

## What This Changes

The PR adds `message.processing.*` and `skill.execution.*` diagnostics events, emits them from embedded runs and skill setup, exports related diagnostics-otel spans/metrics, adds a skill observability doc, and adjusts qa-lab/package/test files.

## Best Possible Solution

Revise the telemetry contract so OTLP exports keep session identifiers out of spans, use only bounded metric labels, fix the typing/docs/changelog gaps, and rebase the branch before normal maintainer review.
