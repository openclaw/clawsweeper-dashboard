## Summary

Keep open: current main still batches memory embeddings per file, and this PR is a plausible bounded implementation, but merging it still needs maintainer acceptance of the new opt-in memory embedding runtime contract and large-backfill behavior.

## What This Changes

This PR adds an opt-in source-wide batching path for memory embedding runtimes, enables it for OpenAI, Gemini, and Voyage, and adds regression coverage for cross-file and compatibility batching.

## Best Possible Solution

Land a bounded source-wide batching implementation after maintainers accept the opt-in runtime contract and decide whether the supplied local proof is enough for large remote-provider backfills.

## Reproduction Assessment

Yes. Current main is source-reproducible because dirty memory and session sync still call indexFile per file, and embedChunksWithBatch only submits the chunks passed for that one entry.

## Solution Assessment

Mostly yes. Moving batching to the manager/source boundary with explicit runtime opt-in is maintainable, but the plugin contract and remote-backfill proof threshold still need maintainer acceptance.
