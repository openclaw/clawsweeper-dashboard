## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Apply reset mode without cache-eligibility gating:** `src/agents/pi-embedded-runner/cache-ttl.ts:170-172`
  - body: `resolveCacheTtlMs` returns `null` before looking at the configured mode whenever the provider is not prompt-cache eligible, and the sweep then skips maintenance entirely. That makes `timeBasedContextCompact: "reset"` silently do nothing for non-cache providers even though reset only needs idle timestamps.
  - confidence: 0.95
- **[P1] Use the newest activity timestamp for expiry:** `src/gateway/session-cache-maintenance.ts:53-57`
  - body: `resolveCacheTouchAt` always prefers `lastCacheTouchAt` over `lastAssistantMessageAt`. If maintenance is enabled after later assistant activity, the sweep can use the stale cache-touch value and immediately reset or compact a recently active session.
  - confidence: 0.9
- **[P2] Deduplicate aliases before maintenance actions:** `src/gateway/session-cache-maintenance.ts:311`
  - body: The sweep acts on every raw key in one session-store snapshot. When both a legacy alias and canonical key exist for the same session, one pass can perform duplicate reset/compaction work or act on a stale alias after the canonical entry was already mutated.
  - confidence: 0.86
- **[P2] Fix the Moonshot eligibility mock:** `src/agents/pi-embedded-runner/cache-ttl.test.ts:10-11`
  - body: The mock still returns `true` for `moonshot`, but the updated expectations assert that Moonshot direct eligibility is `false`. These tests will fail as written because `isCacheTtlEligibleProvider` returns the mocked plugin value.
  - confidence: 0.98
- **[P2] Record user activity only for model turns:** `src/auto-reply/reply/session.ts:476`
  - body: `initSessionState` updates `lastUserMessageAt` for every non-heartbeat inbound turn, including command-only paths that can finish without an assistant model completion. That leaves `lastUserMessageAt > lastAssistantMessageAt`, so `isAwaitingUserReply()` suppresses maintenance until a later model turn happens.
  - confidence: 0.78
