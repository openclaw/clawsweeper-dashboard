Full review comments:

- **[P1] Apply reset mode without cache eligibility gating:** `src/agents/pi-embedded-runner/cache-ttl.ts:170-172`
  - body: `resolveCacheTtlMs` returns `null` before reading the maintenance mode when the provider is not prompt-cache eligible, and the sweep skips all maintenance when `cacheTtlMs` is null. That makes configured `timeBasedContextCompact: "reset"` silently do nothing for non-cache providers even though reset only needs idle timestamps.
  - confidence: 0.93
- **[P1] Use the newest activity timestamp for expiry:** `src/gateway/session-cache-maintenance.ts:53-57`
  - body: `resolveCacheTouchAt` always prefers `lastCacheTouchAt` over `lastAssistantMessageAt`. If maintenance was disabled for a while and later enabled, an old cache-touch timestamp can survive newer assistant turns, so the next sweep can reset or compact a recently active session immediately.
  - confidence: 0.9
- **[P2] Deduplicate aliases before maintenance actions:** `src/gateway/session-cache-maintenance.ts:310-311`
  - body: The sweep iterates raw store entries and acts on each key from a stale snapshot. If the store still has legacy and canonical aliases for the same session, one alias can reset or compact the canonical entry and the later alias can still trigger another action in the same pass.
  - confidence: 0.86
- **[P1] Return the intended Moonshot value from the mock:** `src/agents/pi-embedded-runner/cache-ttl.test.ts:10-11`
  - body: The mock still returns `true` for Moonshot while the changed tests expect `isCacheTtlEligibleProvider("moonshot", ...)` to be false. The focused test suite listed in the PR body will fail until the mock and expectations agree.
  - confidence: 0.95
- **[P2] Avoid command-only turns blocking expiry:** `src/auto-reply/reply/session.ts:471-477`
  - body: `initSessionState` stamps `lastUserMessageAt` for every non-heartbeat inbound turn before knowing whether an assistant model run will update `lastAssistantMessageAt`. Immediate command-only replies can leave `lastUserMessageAt > lastAssistantMessageAt`, making `isAwaitingUserReply()` false and suppressing reset/compact maintenance until a later model run.
  - confidence: 0.78
