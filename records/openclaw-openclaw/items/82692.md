## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve prototype-safe OpenRouter param merging:** `extensions/openrouter/provider-routing.ts:75-80`
  - body: `ctx.extraParams` can contain user/config-supplied JSON keys, and `Object.assign` invokes the `__proto__` setter on the target object. A payload with an own `__proto__` key changes the returned patch object's prototype; keep the current spread-based shape or sanitize the full patch input before assigning.
  - confidence: 0.93
- **[P1] Keep tryReadSecretFileSync missing-file semantics:** `src/infra/secret-file.ts:17-26`
  - body: The upstream `tryReadSecretFileSync` returns `undefined` for not-found paths while still throwing for unsafe files. This wrapper now calls `readSecretFileSync` for every nonblank path, so missing Telegram/IRC/etc. token files throw instead of allowing the existing fallback/logging paths to run.
  - confidence: 0.9
- **[P1] Keep the fs-safe Python helper opt-in:** `src/infra/fs-safe-defaults.ts:12-14`
  - body: Current main and the security docs configure the fs-safe Python helper as `off` unless an operator opts in. This changes existing installs to `auto` at import time, which can start a persistent Python helper during upgrade without explicit operator choice or upgrade proof.
  - confidence: 0.89
- **[P1] Do not bypass plugin-owned delivery target normalization:** `src/infra/outbound/source-delivery-plan.ts:104-112`
  - body: `sourceDeliveryTargetsMatch` currently compares through `normalizeTargetForProvider`, which delegates to channel-owned normalizers such as Telegram and Discord. Replacing that with local prefix stripping plus Feishu/Lark special cases can fail valid provider forms and cause duplicate fallback delivery or wrong delivery-target errors.
  - confidence: 0.86
