## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve tryReadSecretFileSync not-found semantics:** `src/infra/secret-file.ts:26`
  - body: This wrapper calls `readSecretFileSync` for every nonblank path, so configured-but-missing token/password files now throw instead of returning `undefined` through the upstream `tryReadSecretFileSync` contract. Current callers such as Telegram and Zalo rely on missing files resolving to controlled `none`/empty status while still throwing validation errors like symlinks; re-export the upstream helper or preserve its not-found behavior.
  - confidence: 0.93
- **[P1] Avoid Object.assign on raw OpenRouter params:** `extensions/openrouter/provider-routing.ts:75-80`
  - body: `ctx.extraParams` is caller-controlled transport data, and `Object.assign` invokes the `__proto__` setter on the target for an own `__proto__` key. Current main uses object spread, which keeps that value as an own data property instead of mutating the returned patch object's prototype.
  - confidence: 0.94
- **[P1] Keep source-delivery matching on provider normalizers:** `src/infra/outbound/source-delivery-plan.ts:112`
  - body: The local prefix stripper drops provider-owned normalization for valid target forms. For example, Discord's normalizer maps a bare numeric channel ID to `channel:<id>`; without that normalizer, a message-tool send to the same destination can be treated as unverified and trigger fallback delivery or delivery-target errors.
  - confidence: 0.88
- **[P1] Do not change the fs-safe Python default here:** `src/infra/fs-safe-defaults.ts:12-14`
  - body: This flips OpenClaw's documented default from Python helper `off` to `auto`, which can start an optional helper in existing installs unless operators opted out. That is an upgrade-sensitive runtime default change and needs a focused PR with explicit maintainer approval, docs alignment, and fresh-install plus upgrade proof.
  - confidence: 0.9
