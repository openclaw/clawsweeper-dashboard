## Summary

Keep open: current main still leaves the same-model OpenAI Responses replay path able to preserve malformed or overlong stored tool ids before splitting them into `function_call` and `function_call_output` payload fields, and PR #74850 is the open linked fix candidate.

## What This Changes

Normalize OpenAI Responses replay tool identifiers so malformed or overlong stored `call_id|item_id` pairs become valid Responses `function_call` and `function_call_output` ids.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes OpenAI Responses replay ids at the replay boundary while preserving canonical reasoning replay pairs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source inspection shows the same-model Responses path still skips normalization before raw id splitting.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because the existing generic sanitizer is intentionally disabled for Responses replay to preserve provider-owned reasoning ids.
