## Summary

Keep open: current main still has the reported OpenAI Responses replay bug, and PR #74850 is already the linked focused fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses replay tool-call identifiers before sending stored tool calls and tool results back to Responses providers.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes malformed or overlong Responses replay ids at the replay boundary while preserving canonical reasoning pairs and adding regression coverage.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source still shows the same path: same-model replay skips normalization before raw split ids are emitted into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix because generic OpenAI tool-id sanitization is disabled for Responses and canonical reasoning replay pairs need to remain intact.
