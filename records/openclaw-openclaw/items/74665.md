## Summary

Keep open. Current main still has a same-model OpenAI Responses replay path that can carry malformed or overlong `call_id|item_id` pairs into the next provider payload, and open PR #74850 is already the linked implementation candidate with closing syntax.

## What This Changes

Normalize OpenAI Responses replay tool-call and function-call item identifiers before sending follow-up payloads after tool use.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item ids at the Responses replay boundary while keeping canonical same-model reasoning pairs intact and adding regression coverage for malformed and overlong ids.

## Reproduction Assessment

Yes. The report includes a no-credential payload-capture reproduction, and current source inspection shows same-model replay skips the transport normalizer before raw `call_id|item_id` splitting into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific boundary normalizer is the narrowest maintainable direction because the current OpenAI replay policy deliberately avoids generic tool-call id sanitization to preserve native reasoning replay.
