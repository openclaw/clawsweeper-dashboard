## Summary

Keep open: current main still has the same OpenAI Responses replay path that can pass raw stored `call_id|item_id` values into the next provider payload, and open PR #74850 is already the linked fix candidate.

## What This Changes

The issue reports malformed or overlong OpenAI Responses tool identifiers being replayed into follow-up payloads after tool use.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes invalid OpenAI Responses replay ids at the replay boundary while preserving canonical reasoning replay pairs and adding regression coverage.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source inspection confirms the same-model replay path still skips the transport normalizer before splitting stored ids into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because current OpenAI replay policy intentionally avoids generic tool-call id sanitization for native reasoning replay.
