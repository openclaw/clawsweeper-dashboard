## Summary

Keep open. Current main still has Responses replay paths that can preserve raw overlong or malformed `call_id|item_id` pairs into provider payload construction, and open PR #74850 is already the linked implementation candidate for this issue.

## What This Changes

Normalize OpenAI Responses replay tool-call identifiers so replayed `function_call` and `function_call_output` payloads do not reuse overlong or malformed stored `call_id|item_id` pairs.

## Best Possible Solution

Land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item ids at the Responses replay conversion boundary while preserving valid canonical same-model reasoning replay pairs and adding regression coverage for overlong and malformed ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current source inspection shows same-model Responses replay can still preserve raw ids before direct payload splitting.

## Solution Assessment

Yes. A Responses-specific boundary normalizer is the narrowest maintainable solution; broad generic OpenAI tool-id sanitization would risk breaking valid native reasoning replay pairs.
