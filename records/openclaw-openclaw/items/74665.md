## What This Changes

Normalize malformed or overlong OpenAI Responses replay tool-call ids before they are sent back to Responses providers after tool use.

## Best Possible Solution

Land PR #74850, or an equivalent focused patch, so Responses replay normalizes malformed/overlong call ids and item ids while preserving canonical reasoning pairs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source shows same-model replay bypassing the normalizer before raw split ids are emitted.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because generic strict OpenAI tool-id sanitization is intentionally disabled for Responses and canonical reasoning pairs must remain intact.
