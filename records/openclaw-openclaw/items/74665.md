## Summary

Keep open. Current main still has a same-model OpenAI Responses replay path that can carry raw overlong or malformed `call_id|item_id` pairs into Responses payload fields, and PR #74850 is already the open implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses replay tool call IDs and item IDs before re-emitting prior tool calls and tool results.

## Best Possible Solution

Land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item IDs at the Responses replay conversion boundary while preserving valid canonical same-model reasoning replay pairs and adding regression coverage for overlong and malformed IDs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current source inspection shows same-model replay still preserves raw IDs before direct split into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific boundary normalizer is the narrowest maintainable fix; enabling broad generic OpenAI tool-ID sanitization would risk breaking valid native reasoning replay pairs.
