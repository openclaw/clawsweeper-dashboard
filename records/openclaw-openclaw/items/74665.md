## Summary

Keep open. Current main at e165b7595832850866f344031dd54b8c670bddd3 still has a same-model OpenAI Responses replay path that can split and re-emit raw overlong or malformed `call_id|item_id` values, and linked PR #74850 is already the open implementation candidate with closing syntax.

## What This Changes

Normalize OpenAI Responses replay tool identifiers so malformed or overlong stored `call_id|item_id` pairs are not sent back to Responses providers after tool use.

## Best Possible Solution

Review and land #74850, or an equivalent focused patch, that normalizes invalid OpenAI Responses `call_id` and `fc_*` item ids at the Responses replay boundary while preserving canonical reasoning replay pairs and adding regression coverage for malformed and overlong ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current main source inspection shows the same-model replay path skips the transport normalizer before raw id splitting into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because current OpenAI replay policy deliberately avoids generic tool-call id sanitization to preserve native reasoning replay.
