## What This Changes

The issue requests normalization of OpenAI Responses replay `call_id` and `function_call.id` values so tool-using conversations are not rejected after replay.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, with regression coverage for malformed and overlong Responses replay ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source still shows same-model replay skipping normalization before raw split ids are emitted into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix because generic OpenAI tool-id sanitization is disabled for Responses and valid reasoning pairs need to remain intact.
