## Summary

Keep open. Current main still leaves same-model OpenAI Responses replay exposed to raw overlong or malformed `call_id|item_id` pairs, and PR #74850 is already the open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests normalizing OpenAI Responses replay tool identifiers so `function_call.call_id`, `function_call.id`, and matching outputs stay provider-valid after tool use.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item ids at the Responses replay boundary while preserving valid same-model reasoning replay pairs and adding regression coverage for malformed and overlong ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and source inspection of current main plus `@mariozechner/pi-ai@0.71.1` confirms the same-model replay path can still pass raw ids into the Responses payload.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix; broad generic OpenAI tool-id sanitization would conflict with the current policy that preserves native Responses reasoning replay.
