## Summary

Keep open: current main still leaves same-model OpenAI Responses replay IDs unnormalized before emitting `function_call` and `function_call_output` payload fields, and PR #74850 is already the open linked fix candidate.

## What This Changes

Normalize overlong or malformed OpenAI Responses replay tool `call_id` and `function_call.id` values before sending follow-up payloads after tool use.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes malformed Responses replay ids while preserving valid `call_*|fc_*` reasoning replay pairs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current main source confirms the same-model replay path bypasses normalization before splitting stored ids into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix because current tests intentionally preserve canonical reasoning replay pairs that broad tool-id sanitization could break.
