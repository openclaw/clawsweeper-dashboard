## Summary

Keep open. Current main still leaves the same-model OpenAI Responses replay path able to carry malformed or overlong stored tool ids into provider payload fields, and #74850 is already the open implementation candidate with closing syntax.

## What This Changes

The issue requests normalization of OpenAI Responses replay tool call and item identifiers so overlong or malformed stored `call_id|item_id` pairs are not resent to Responses providers after tool use.

## Best Possible Solution

Review and land #74850, or an equivalent focused Responses-boundary patch, that normalizes malformed or overlong `call_id` and `fc_*` item ids before replay payload construction while preserving canonical reasoning pairs and adding regression coverage for the reported payload shape.

## Reproduction Assessment

Yes. The report includes a no-credential payload-capture reproduction, and current source inspection shows same-model replay can bypass `normalizeToolCallId` before raw `call_id|item_id` splitting into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because current policy deliberately avoids generic Responses tool-id sanitization to preserve native reasoning replay.
