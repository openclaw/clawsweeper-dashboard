## Summary

Keep open: current main still has a same-model OpenAI Responses replay path that can pass raw stored tool identifiers into the next Responses payload, and linked PR #74850 is already the implementation candidate.

## What This Changes

Normalize OpenAI Responses replay tool identifiers so overlong or malformed `call_id|item_id` pairs are provider-safe after tool use.

## Best Possible Solution

Keep the issue open until PR #74850 or an equivalent focused patch lands with Responses replay-id normalization and coverage for malformed, overlong, and canonical reasoning-pair ids.

## Reproduction Assessment

Yes. The report includes a no-credential payload-capture repro, and current source inspection confirms same-model replay can bypass id normalization before raw ids are split into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because current OpenAI replay policy intentionally avoids generic tool-call id sanitization to preserve native reasoning replay.
