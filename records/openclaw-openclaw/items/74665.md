## Summary

Keep open: current main still preserves same-model OpenAI Responses tool ids before raw-splitting them into `function_call` and `function_call_output` payload fields, and PR #74850 is already the open linked fix candidate.

## What This Changes

Normalize OpenAI Responses replay tool identifiers so replayed tool-call and tool-result payloads do not resend malformed or overlong stored call_id/item IDs after tool use.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes malformed OpenAI Responses replay ids at the replay boundary with regression coverage.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current main still preserves same-model stored ids before raw-splitting them into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix because generic OpenAI tool-id sanitization would conflict with intentional preservation of canonical reasoning replay pairs.
