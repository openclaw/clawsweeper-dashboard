Keep open: current main still has the reported OpenAI Responses replay bug, and PR #74850 is an open linked fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue requests normalization of OpenAI Responses replay tool identifiers so overlong or malformed stored tool-call ids are not sent back as invalid Responses `call_id` or `function_call.id` values.

## Best Possible Solution

Land PR #74850 or an equivalent focused patch that normalizes malformed Responses replay ids at the replay boundary while preserving valid canonical reasoning pairs and adding regression coverage.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source inspection shows the same path still emits raw split ids into Responses payload fields on same-model replay.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable solution because generic OpenAI tool-id sanitization is disabled for Responses and valid reasoning replay pairs need to remain intact.
