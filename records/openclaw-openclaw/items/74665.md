## Summary

Keep open. Current main still has a same-model OpenAI Responses replay path that can preserve raw overlong or malformed `call_id|item_id` pairs into provider payload construction, and open PR #74850 is already the linked implementation candidate with closing syntax.

## What This Changes

Normalize OpenAI Responses tool-call and function-call item IDs during replay so stored `call_id|item_id` pairs cannot produce invalid `function_call` or `function_call_output` payload fields.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item IDs at the Responses replay boundary while preserving valid same-model reasoning replay pairs and adding regression coverage for malformed and overlong IDs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current source plus `@mariozechner/pi-ai@0.71.1` inspection confirms the same-model replay path can still pass raw IDs into the Responses payload.

## Solution Assessment

Yes. A Responses-specific boundary normalizer is the narrowest maintainable fix; broad generic OpenAI tool-id sanitization would conflict with the current native reasoning replay preservation policy.
