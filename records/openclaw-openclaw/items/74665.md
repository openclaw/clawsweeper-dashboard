## Summary

Keep open: current main still has a same-model OpenAI Responses replay path that can carry raw overlong or malformed `call_id|item_id` pairs into Responses payload fields, and PR #74850 is already the open implementation candidate with closing syntax.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses tool-call and function-call item identifiers before replay after tool use so follow-up provider requests pass Responses API validation.

## Best Possible Solution

Keep the OpenAI reasoning replay semantics intact, but add a targeted Responses replay-boundary normalizer for malformed or overlong call ids and non-`fc_*` item ids with regression coverage; land PR #74850 if review confirms it does that cleanly.

## Reproduction Assessment

Yes. The report includes a no-credential payload-capture reproduction, and source inspection on current main shows same-model OpenAI Responses messages skip normalization before raw ids are split into outbound Responses fields.

## Solution Assessment

Yes. A Responses-specific boundary normalizer is the narrowest maintainable direction because current OpenAI replay policy deliberately avoids broad generic tool-call id sanitization to preserve native reasoning replay.
