## Summary

Keep open: current main still replays OpenAI Responses tool identifiers by splitting stored `call_id|item_id` values directly into provider payload fields, and open PR #74850 is already the linked fix candidate with closing syntax.

## What This Changes

Normalize OpenAI Responses replay tool-call and function-call item identifiers so follow-up requests do not send overlong or malformed provider IDs.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes invalid Responses replay IDs at the replay boundary while preserving canonical reasoning replay pairs and adding regression coverage.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture repro, and current source inspection confirms the same replay path still emits split stored IDs without Responses-specific normalization.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable direction because OpenAI replay policy intentionally preserves native reasoning tool ID pairs instead of enabling generic strict sanitization.
