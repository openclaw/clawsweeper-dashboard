## Summary

Keep open: current main still has a same-model OpenAI Responses replay path that can pass raw `call_id|item_id` pairs into the next payload, and open PR #74850 is already the linked implementation candidate.

## What This Changes

Normalize malformed or overlong OpenAI Responses replay tool identifiers so follow-up payloads use valid `call_id` and `fc_*` item IDs.

## Best Possible Solution

Review and land PR #74850, or an equivalent focused patch, that normalizes invalid OpenAI Responses replay ids at the replay boundary while preserving canonical reasoning replay pairs and adding regression coverage.

## Reproduction Assessment

Yes. The report includes a no-credential payload-capture repro, and current source inspection confirms same-model replay skips the transport normalizer before splitting stored ids into Responses payload fields.

## Solution Assessment

Yes. A Responses-specific replay-boundary normalizer is the narrowest maintainable fix because current OpenAI replay policy intentionally avoids generic tool-call id sanitization to preserve native reasoning replay.
