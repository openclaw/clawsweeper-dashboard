## Summary

Keep open for maintainer/product review. Current main intentionally treats generic format or invalid-request failures as terminal unless a known repair path opts into retry, so the requested blanket Anthropic fallback is not implemented and needs a narrower safety decision.

## What This Changes

Request fallback or preflight validation when Anthropic returns an invalid_request_error for a tool-call payload instead of ending the session.

## Best Possible Solution

Add a narrow, maintainer-approved validation or repair path for proven retry-safe Anthropic tool-payload invalid requests, with live repro and regression coverage, instead of making all format errors fallback-eligible.

## Reproduction Assessment

No live reproduction was established. Source inspection does show the reported class maps to the terminal format-error path and will not fall back unless allowFormatRetry is explicitly enabled.

## Solution Assessment

No. Treating every Anthropic invalid_request_error as fallback-eligible conflicts with current failover policy; a narrow validator or repair opt-in for proven retry-safe payload errors is safer.
