## Summary

Keep this issue open: current main and the latest release still use the 60 second aggregate compaction retry wait, and there are open linked closing PRs that should be reviewed instead of closing the report early.

## What This Changes

The issue reports that long in-turn compaction retries can be abandoned after a hardcoded 60 second aggregate wait, discarding valid late provider results and leaving sessions stuck in repeated overflow compaction attempts.

## Best Possible Solution

Review and land one canonical fix, with https://github.com/openclaw/openclaw/pull/94421 currently matching the source-level active-retry gap while preserving the idle deadlock guard.

## Reproduction Assessment

Yes, at source level: current main still races a fixed 60 second aggregate wait against long retry work, and the reporter supplied production trajectories where valid model results arrived after that window. I did not run a paid 200K-token provider reproduction in this read-only review.

## Solution Assessment

Yes for keeping the issue open: it is not fixed on current main or the latest release, and open closing PRs are the right implementation path. The best fix appears to preserve the idle fallback while treating the retry model request as active work rather than only scaling a constant.
