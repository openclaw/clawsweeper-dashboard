## Summary

The reported 60s aggregate-wait abandon behavior is already addressed: current main and the v2026.6.8 release keep extending the aggregate wait while compaction is actively in flight, and regression coverage verifies a 170s compaction completes without timing out.

## What This Changes

The issue asks for in-turn compaction retry waiting to consume or cancel long-running compaction results instead of abandoning valid provider output after a hardcoded 60s aggregate wait.

## Best Possible Solution

Keep the shipped in-flight-aware aggregate wait and leave the separate inner timeout/product work tracked in https://github.com/openclaw/openclaw/issues/92043.

## Reproduction Assessment

No high-confidence current-main reproduction remains: the reported old-version path is source-plausible, but current main and v2026.6.8 keep waiting while compaction is in flight and have focused coverage for a 170s in-flight compaction.

## Solution Assessment

Yes, the shipped fix is a better fit than adding a new config surface for this exact bug because it preserves the 60s fallback only for idle/lost retry resolution while consuming active compaction results that finish later.
