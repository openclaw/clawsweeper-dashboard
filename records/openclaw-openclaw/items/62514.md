## Summary

Keep this open. Current main and the latest shipped release still exclude HTTP(S) prompt image refs from native prompt-image detection, and the closing implementation PR is still open, unmerged, and conflicting.

## What This Changes

The issue reports that mixed WebUI image inputs can cause weak multimodal models to ignore an image or call the `image` tool with hallucinated image references.

## Best Possible Solution

Review, rework, and land or replace https://github.com/openclaw/openclaw/pull/63710 so uploaded images and explicit HTTP(S) image refs are merged into native vision input while preserving SSRF, sandbox, and media-fetch contracts.

## Reproduction Assessment

Yes, source-level. Current main still filters HTTP(S) prompt image refs out of native image detection, and the focused test still expects a local path plus HTTPS image URL prompt to detect only the local path.

## Solution Assessment

Yes for keeping the issue open and tying it to the native image-ingestion fix path. The linked PR targets the right surface, but the maintainable fix must preserve existing SSRF, sandbox, and media-fetch contracts before merge.
