- `impact:message-loss`: The issue is about image references being omitted from the native model payload or diverted through hallucinated tool calls, effectively losing part of the user's message.

## Summary

Keep this open. Current main still excludes HTTP(S) prompt image refs from native prompt-image detection, so mixed WebUI image turns can still depend on model tool-calling, and the linked closing PR is open, unmerged, and conflicting.

## What This Changes

The issue reports WebUI mixed image input where uploaded images plus prompt-referenced images can make weaker multimodal models call the image tool with hallucinated paths/URLs or ignore part of the image context.

## Best Possible Solution

Review, rework, and land or replace https://github.com/openclaw/openclaw/pull/63710 so uploaded and explicit HTTP(S) image refs are merged into native vision input while preserving SSRF, sandbox, and media-fetch contracts.

## Reproduction Assessment

Yes, source-level. Current main still filters HTTP(S) prompt image refs out of native image detection, and the focused tests still assert that a local path plus HTTPS image URL prompt detects only the local path.

## Solution Assessment

Unclear for the linked PR as written. It targets the right ingestion surface, but the maintainable fix must preserve the existing media SSRF and sandbox contracts and include real WebUI or runtime proof.
