## Summary

Keep this open. Current main still leaves the deterministic mixed inline-image plus HTTP(S) image URL path incomplete for native vision, and the closing implementation PR is still open and unmerged.

## What This Changes

The issue asks for WebUI multimodal image turns to avoid ignored image context and hallucinated fallback image-tool calls with weaker vision-capable models.

## Best Possible Solution

Review, rework, and land or supersede https://github.com/openclaw/openclaw/pull/63710 for safe mixed-source native image preloading, then split any remaining local-path or tool-exposure behavior into narrower follow-ups.

## Reproduction Assessment

Yes, source-level. Current main still filters HTTP(S) prompt image refs out of native image detection, and tests assert that a prompt with a local image path plus an HTTPS image URL detects only the local path.
