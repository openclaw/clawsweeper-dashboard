## Summary

Keep open: current main and v2026.6.6 still ignore HTTP(S) prompt image refs in the native prompt-image detector, while the linked closing PR at https://github.com/openclaw/openclaw/pull/63710 remains open and unmerged with security/availability concerns to resolve.

## What This Changes

This issue asks OpenClaw to handle Web UI mixed image input deterministically so weaker multimodal models do not ignore images or call the fallback image tool with hallucinated paths or URLs.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/63710, or replace it with a narrow fix that unifies supported image sources into the first native vision request while preserving remote-media security and availability boundaries.

## Reproduction Assessment
