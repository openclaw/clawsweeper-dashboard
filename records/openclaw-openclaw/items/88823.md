## What This Changes

The PR renames macOS update and launchd restart handoff shell-template variables from `status` to `exit_status` and adds assertions that generated macOS scripts avoid `status=` assignments while preserving `status=...` log wording.

## Best Possible Solution

Merge a narrowly verified template rename once real macOS launchd or zsh handoff proof confirms Update Now no longer trips the readonly `status` variable and restart exit-code logging still works.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Current source contains macOS shell templates with `status=` assignments, but the inspected runtime paths invoke `/bin/sh` or Node and local `zsh` is unavailable.

## Solution Assessment

Yes for the current source templates: renaming the internal shell variable at the generation point is the narrowest maintainable mitigation. The missing piece is real macOS or zsh proof that this is the same path reported by the linked issue.
