The observable bug is that memory-core Dream Diary narrative subagent text can be generated by a CLI-backed run but never appended to `DREAMS.md` because `generateAndAppendDreamNarrative` treats `waitForRun(status: ok)` as if the assistant transcript is already flushed, then reads `getSessionMessages` too early. Also fix the CLI compaction path so `runCliTurnCompactionLifecycle` initializes built-in context engines before resolving the legacy engine. Stay within `extensions/memory-core/src/dreaming-narrative.ts`, `src/agents/command/cli-compaction.ts`, and focused tests unless source inspection proves a smaller shared seam is needed. Prefer extracting the diary text from a completed run payload or waiting on an explicit transcript-flushed signal over adding arbitrary sleeps or broad caching. Add regression coverage for delayed transcript visibility and for CLI compaction calling context-engine initialization before resolution. Add a single-line `CHANGELOG.md` fix entry because this is user-facing. Do not add new config, product policy, or broad subagent API changes unless maintainers approve.
