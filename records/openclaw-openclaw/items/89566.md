## Summary

Keep open; the patch is not mergeable because it would suppress final Telegram error-warning replies that core intentionally emits after a successful-looking assistant answer, and the PR still lacks required real Telegram proof.

## What This Changes

The PR changes Telegram dispatch to suppress text-only post-final tool/error payloads and adds a regression test for late failed tool/final payloads after a final reply.

## Best Possible Solution

Keep post-final tool-result noise suppression, but preserve terminal final `isError` warning replies generated by core payload policy.

## Reproduction Assessment

Yes, from source: core final payload delivery loops over all replies, while core payload construction deliberately emits an assistant success reply followed by an `isError` mutating-tool warning. The PR-head Telegram guard drops that second final once the first answer starts.

## Solution Assessment

No; the Telegram-layer `payload.isError` broadening is too broad because it overrides core's terminal warning policy after an answer has started.
