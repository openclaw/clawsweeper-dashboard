## Summary

Keep open: current main still supports the reported LM Studio auth-template bug by source inspection, while the `/api/v1/chat` fallback request remains a separate provider/API scope question with no live reproduction in the discussion.

## What This Changes

The issue asks OpenClaw to expand LM Studio API-key environment templates and to support or fall back to LM Studio endpoint variants when `/v1/chat/completions` fails.

## Best Possible Solution

Resolve LM Studio configured API keys through the shared SecretInput path with focused regression coverage and real LM Studio proof, then decide native `/api/v1/chat` inference or fallback behavior as a separate provider design change.

## Reproduction Assessment

Yes for the auth half by source inspection: arbitrary LM Studio env-template `apiKey` strings can bypass SecretInput resolution unless they are registered env markers. No for the endpoint half because no live LM Studio server failure was verified.

## Solution Assessment

No as one combined fix: auth expansion is a narrow provider bug fix, while native endpoint fallback changes LM Studio inference behavior and should be scoped separately.
