## What This Changes

The issue asks the LM Studio provider to expand env-backed `apiKey` values and avoid failing chat calls by using native or fallback LM Studio endpoints.

## Best Possible Solution

Land or replace the focused SecretInput auth fix with redacted LM Studio runtime proof, then handle native `/api/v1/chat` fallback only as separate provider/plugin scope or a narrower SDK/API-gap issue.
