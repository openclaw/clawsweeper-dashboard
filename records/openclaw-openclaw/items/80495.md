## Summary

Keep open: current main still has a source-proven LM Studio auth-resolution gap for arbitrary env-template API keys, while the endpoint fallback/native API request is separate provider scope and is not solved by current main.

## What This Changes

The issue asks OpenClaw to expand LM Studio API-key environment templates and handle LM Studio endpoint incompatibility or fallback behavior.

## Best Possible Solution

Land the narrow env-template auth fix after real LM Studio proof, and handle native `/api/v1/chat` fallback as a separate provider/API scope decision.

## Reproduction Assessment

Partly yes. Source inspection gives a high-confidence reproduction path for `${LMSTUDIO_API_KEY}` being returned literally as the LM Studio API key, but the endpoint-mismatch half was not reproduced against a live LM Studio server.

## Solution Assessment

No as a single bundled fix. The auth portion should be solved through the existing SecretInput resolver, while native endpoint fallback is a separate provider behavior decision and should not be folded into an automatic bug-fix pass.
