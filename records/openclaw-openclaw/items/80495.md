## Summary

Keep open: source inspection supports a real LM Studio auth-resolution gap for non-default env-template API keys, but the native `/api/v1/chat` fallback request is a separate provider/product scope item rather than a narrow bug fix.

## What This Changes

The issue reports LM Studio failures from API-key environment-variable handling and requests native or fallback endpoint behavior for LM Studio inference.

## Best Possible Solution

Fix LM Studio `apiKey` resolution to honor the standard SecretInput/env-template contract for arbitrary `${ENV}` refs, and keep native `/api/v1/chat` inference as a separate ClawHub or explicit product/API discussion.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence path for the auth half: a non-default env-template API key is treated as a literal unless it matches the registered `LM_API_TOKEN` marker; the endpoint half was not reproduced against a live LM Studio server.

## Solution Assessment

Unclear as a single issue. The auth-resolution fix should be narrow, but endpoint fallback/native inference is not the same bug and should not be bundled without a maintainer scope decision.
