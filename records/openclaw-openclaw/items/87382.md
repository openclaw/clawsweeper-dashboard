## Summary

Current main now handles the reported already-paired node prompt loop at the gateway reconnect boundary: unchanged paired-node reconnects do not create a fresh approval request, and stale same-node pending requests are cleaned up after a successful hello while real approval-surface expansions still require reapproval.

## What This Changes

The issue reports repeated macOS `Allow node to connect?` prompts for an already-paired node after upgrade/restart.

## Best Possible Solution

Keep the current main gateway-side reapproval cleanup and diagnostics behavior, and ship it in the next release.

No current-main failure path remains: the historic 2026.5.26 and 2026.5.28-beta reports reproduced the bug, and current source plus regression coverage show unchanged paired-node reconnects now clear stale pending prompts instead of creating another approval.

## Solution Assessment

Yes. Fixing this at the gateway paired-node reconnect and cleanup boundary is the best shape because it suppresses stale same-surface prompts without auto-approving expanded commands, capabilities, or permissions.
