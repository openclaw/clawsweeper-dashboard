## Summary

Keep open: the Control UI preview feature is still absent from current main, but this PR is not merge-ready because it mixes in an unrelated dependency override and renders transcript preview text without a display bound while an overlapping implementation remains open in #48396.

## What This Changes

This PR adds last-message preview data to Control UI session rows, requests `includeLastMessage` from `sessions.list`, renders a timestamped preview under each session key, and adds a pnpm `punycode` override.

## Best Possible Solution

Land one narrowed implementation for #47120 that uses existing `sessions.list` preview support, leaves dependency resolution unchanged, normalizes and truncates the visible preview, adds focused UI/controller coverage and a changelog entry, then supersedes the duplicate PR path.

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature PR; source inspection gives a high-confidence verification path because current main lacks the UI request/type/render path while backend preview support already exists.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is. The maintainable solution is to keep the UI feature but remove the dependency override, render a normalized bounded preview, add tests/changelog, and reconcile the overlapping #48396 path.
