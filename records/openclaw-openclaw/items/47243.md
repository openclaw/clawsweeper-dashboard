## Summary

Keep open. Current main still lacks the requested Control UI session timestamp/preview display, and this PR is not ready to merge because it mixes in an unrelated dependency override, enables per-session transcript reads without a bound, and renders previews without clamping; it also overlaps the still-open #48396 implementation path for #47120.

## What This Changes

The PR adds a `lastMessagePreview` UI row field, requests `includeLastMessage` from `sessions.list`, renders `[MM/DD HH:mm]` preview text under each session key, and adds a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed implementation for #47120: keep dependency resolution unchanged, request previews only through a bounded or capped list path, normalize and truncate the preview with tooltip access to the full text if needed, add focused controller/view coverage plus a changelog entry, then close or supersede the duplicate PR path.

## Reproduction Assessment

Not applicable as a feature PR, but the review issues have a high-confidence source verification path. Current main lacks the UI request/type/render path, while the PR diff adds the preview path plus the unbounded fetch and dependency override concerns.

## Solution Assessment

No. The safer solution is a smaller Control UI change that uses the existing backend preview option only when the list is bounded, clamps displayed text, adds focused tests and changelog coverage, and avoids unrelated package-resolution changes.
