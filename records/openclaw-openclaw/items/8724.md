## Summary

Keep open. Current main still lacks the requested typed global/per-model non-resetting generation timeout; the existing controls are a run-level timeout, provider HTTP request timeout metadata, and a reset-on-chunk idle watchdog.

## What This Changes

Request adding global and per-model generation timeout config so long-running streaming model loops can be aborted and routed through fallback handling.

## Best Possible Solution

Add a documented backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget per fallback candidate, and enforce it with a non-resetting timer during active generation.

## Reproduction Assessment

Yes. The gap is reproducible by source/schema inspection: the proposed per-model key is outside the strict model catalog schema, and runtime enforcement still uses params.timeoutMs plus a reset-on-chunk idle watchdog.

## Solution Assessment

Yes in direction. A narrow core config/schema/docs change plus embedded-runner enforcement is the maintainable path; first-token timeout and provider-level skip behavior should remain separate work.
