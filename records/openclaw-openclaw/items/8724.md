## Summary

Keep open: current main has provider-level request/idle timeout controls, but still lacks the requested typed global/per-model non-resetting generation timeout and per-candidate enforcement. The recent discussion also treats this as an open product/API decision rather than a duplicate or stale support thread.

## What This Changes

Requested support for global and per-model generation timeout configuration so model attempts can be aborted after a configured total generation budget.

## Best Possible Solution

Define one backwards-compatible generation-timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget for each selected fallback candidate, and update schema, docs, logging/fallback classification, and focused runner/fallback tests together.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the active watchdog is idle/request based rather than a total generation budget. I did not replay the original live Gemini loop in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation.
