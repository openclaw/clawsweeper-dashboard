## Summary

Keep open. Current main still lacks the requested typed global/per-model non-resetting generation timeout; existing controls are a run-level timeout, provider request timeout plumbing, and a reset-on-chunk idle watchdog.

## What This Changes

Add global and per-model generation timeout configuration so long-running model generation can be aborted and handed to fallback handling.

## Best Possible Solution

Add a documented backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget per fallback candidate, and enforce it with a non-resetting timer during active generation.

## Reproduction Assessment

Yes. The gap is reproducible by source/schema inspection: the proposed per-model key is outside the strict schema, and runtime enforcement still uses params.timeoutMs plus a reset-on-chunk idle watchdog.

## Solution Assessment

Yes in direction. A narrow core config/schema/docs change plus embedded-runner enforcement is the maintainable path; first-token/provider-skip behavior should remain separate work.
