## Summary

Keep this issue open: current main has provider-request, stream-idle, and whole-run limits, but still cannot resolve a typed total-generation budget for each selected model; the public config name, deadline precedence, and fallback semantics remain maintainer product decisions.

## What This Changes

Requests configurable global and per-model total generation timeouts that abort runaway model generation and optionally trigger fallback.

## Best Possible Solution

Add one canonical agent-default generation deadline with a per-model override, resolve the effective value for each selected fallback candidate, and feed it into the existing non-resetting run-abort and fallback-classification path while keeping provider-request and activity-idle deadlines separate.

## Reproduction Assessment

Yes at source level: current main has a non-resetting run timer, but the strict model catalog cannot configure it per model, while request and idle deadlines use different scopes. This read-only review did not invoke a live model long enough to reproduce the reported 30-minute loop.

## Solution Assessment

Yes in direction, but not yet in exact API shape: resolving a per-model budget into the existing run timer is narrower and more maintainable than adding another timer stack or exposing the request-layer `requestTimeoutMs` as the generation contract.
