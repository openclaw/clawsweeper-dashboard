## Summary

Keep open: current main and v2026.5.20 still lack the requested typed global/per-model non-resetting generation timeout, while provider request timeouts and the reset-on-chunk idle watchdog are adjacent but not equivalent.

## What This Changes

The issue requests global and per-model generation timeout configuration so runaway or long-thinking model generations can be aborted and routed consistently.

## Best Possible Solution

Define one backwards-compatible generation-timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget per selected fallback candidate, and update schema, docs, logging/fallback classification, and focused tests together.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the active watchdog is idle/request based rather than a total generation budget. I did not replay the original live Gemini loop.

## Solution Assessment

Unclear. A dedicated generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation.
