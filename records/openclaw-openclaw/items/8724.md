## Summary

Keep open. Current main still lacks the requested typed global/per-model non-resetting generation timeout; existing behavior is limited to run-level timeout, provider request timeout metadata, and a reset-on-chunk idle watchdog.

## What This Changes

The issue requests a configurable generation timeout at the agent default and per-model catalog levels for long or looping LLM generation attempts.

## Best Possible Solution

Add a documented backwards-compatible generationTimeout contract at agent defaults and model catalog entries, resolve the effective budget per fallback candidate, and enforce it with a non-resetting timer during active generation.

## Reproduction Assessment

Yes. Source inspection against current main shows the proposed per-model key is outside the strict model catalog schema, and runtime enforcement still uses params.timeoutMs plus a reset-on-chunk idle watchdog.

## Solution Assessment

Yes in direction. A narrow config/schema/docs change plus embedded-runner enforcement is the maintainable path; first-token timeout and provider-level skip behavior should remain separate work.

## Repair Work Prompt

Implement the generation-timeout feature from this issue as a narrow OpenClaw core change. Add a backwards-compatible `generationTimeout` contract at `agents.defaults` and per-model `agents.defaults.models[provider/model]` entries, accepting duration strings such as `300s` unless existing config helpers strongly prefer a safer local convention. Resolve the effective timeout per selected fallback candidate, enforce it with a non-resetting total-generation timer during active model generation, classify/log timeout aborts so existing fallback handling can respond, and update generated schema/help/docs plus focused tests. Keep first-token timeout, provider-level skip/circuit breaking, and provider HTTP request timeout semantics out of scope. Add a user-facing changelog entry. Stop and ask for maintainer review if the implementation would require a broader runtime architecture change or a different public field name.
