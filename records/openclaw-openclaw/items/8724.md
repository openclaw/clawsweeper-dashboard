## Summary

Keep open: current main still has adjacent run-level, provider request, and stream-idle timeout controls, but it does not expose or enforce the requested typed global/per-model non-resetting generation timeout budget.

## What This Changes

The issue requests configurable global and per-model generation timeouts so runaway model generation can be aborted, logged, and optionally routed to fallback models.

## Best Possible Solution

Add a backwards-compatible generation-timeout contract at the agent default and per-model catalog levels, resolve it per fallback candidate, enforce it as a non-resetting active-generation budget, and keep first-token/provider-skip behavior separate.

## Reproduction Assessment

Yes, by source inspection: the requested per-model key is absent from the strict `agents.defaults.models` type/schema/docs, while current runtime enforcement is run-level timeout plus provider request timeout and reset-on-chunk idle timeout. A live provider loop is not needed to verify the missing config surface.

## Solution Assessment

Yes. The narrow maintainable path is a core config/schema/docs change plus embedded-runner enforcement for a selected model's total generation budget; reusing only provider request timeout or the existing idle watchdog would not satisfy the requested non-resetting behavior.
