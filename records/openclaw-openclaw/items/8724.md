## Summary

Keep open: current main still has provider request and idle timeout controls, but not the requested typed global/per-model non-resetting generation budget. Model-scoped timeout keys remain outside the strict agent defaults schema, and the public API semantics still need maintainer judgment.

## What This Changes

Request a global and per-model generation timeout config that can abort or fail over runaway LLM generation per selected model.

## Best Possible Solution

Define one typed, documented timeout contract for default and per-model generation budgets, resolve it per selected fallback candidate, and enforce it as a non-resetting budget separate from provider request, idle, first-token, and whole-run timeouts.

## Reproduction Assessment

Yes for the source-level config-contract gap: current typed and Zod schemas still reject model-scoped generationTimeout/requestTimeoutMs-style keys under agents.defaults.models. No live Gemini infinite-thinking reproduction was attempted in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, units, precedence, disable behavior, and fallback semantics. The maintainable direction is a typed per-model/global generation budget while keeping first-token, idle-timeout, provider-skip, and provider request timeout behavior separate unless explicitly approved together.
