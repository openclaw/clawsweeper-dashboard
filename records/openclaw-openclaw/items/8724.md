## Summary

Current main has adjacent provider-request, stream-idle, and whole-run timeout controls, but still lacks the requested typed per-model non-resetting generation budget; maintainers must define its exact semantics and precedence before implementation.

## What This Changes

Requests configurable global and per-model total-generation timeouts that abort runaway model generations and participate in fallback handling.

## Best Possible Solution

Define a typed per-model attempt deadline with explicit precedence beneath the global run ceiling, non-resetting enforcement during active generation, and consistent timeout classification for fallback, while keeping first-token and provider-skip policy separate.

## Reproduction Assessment

Yes at the source-contract level: current main lacks the proposed per-model timeout and its idle watchdog resets on stream progress, although this read-only review did not invoke a live runaway model.

## Solution Assessment

Unclear. A per-model control appears warranted, but maintainers must first decide whether it limits provider request time, active generation time, or the complete model attempt and how it composes with the global run ceiling.
