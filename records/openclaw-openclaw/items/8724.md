## Summary

Keep open: current main has adjacent provider request-timeout and idle-watchdog plumbing, but it still lacks the requested typed global/per-model generation timeout contract for active runaway streams. Related open work covers idle and first-token behavior, but no merged or canonical replacement fully owns the per-model generation budget request.

## What This Changes

Request adds a global and per-model generation timeout config to abort runaway model generation and optionally trigger fallback and timeout logging.

## Best Possible Solution

Define one backwards-compatible timeout contract, add it to the agent default and per-model catalog schema/docs, resolve the effective budget for each selected fallback candidate, and enforce a non-resetting generation timer separate from first-token and idle watchdog behavior.

## Reproduction Assessment

Yes for the missing config surface: current source, strict schema, docs, and related discussion show that the requested global/per-model timeout keys are absent. A live Gemini runaway loop was not replayed for this read-only review.

## Solution Assessment

Unclear. A per-model generation timeout is a plausible solution, but maintainers still need to choose the public contract and precedence against existing provider, idle, first-token, and run timeout knobs before implementation.
