## Summary

Keep this open. Current main and the latest release still have adjacent provider-request, model-idle, and whole-run timeout controls, but they do not expose the requested typed global/per-model non-resetting generation budget under `agents.defaults.models`; the remaining work needs maintainer product direction before a fix PR is safe.

## What This Changes

The issue requests global and per-model generation timeout configuration so long-running model output can be aborted or routed differently without relying only on provider-level or idle timeouts.

## Best Possible Solution

Define one typed agent default/per-model generation deadline with explicit precedence under the whole-run timeout, non-resetting enforcement per selected candidate, timeout classification/logging for fallback, and docs/schema coverage while keeping first-token/provider-skip policy separate.

## Reproduction Assessment

Yes, at the source-contract level: current main lacks the proposed per-model timeout field and the stream idle watchdog resets on stream progress. I did not run a live runaway-model reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the timeout name, scope, precedence, defaults, and fallback semantics. A per-model control appears warranted, but it should land as one coherent config/runtime contract rather than being conflated with provider request timeouts or first-token failover.
