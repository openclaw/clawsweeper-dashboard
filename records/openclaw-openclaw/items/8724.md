## Summary

Current main still lacks the requested per-model generation-timeout contract, and the active maintainer review scopes this as a bounded but product-sensitive follow-up rather than cleanup closure.

## What This Changes

The issue asks for configurable global and per-model generation timeout budgets so looping or long-running model generations can be aborted and optionally routed through fallback behavior.

## Best Possible Solution

Add an explicit, documented global and per-model generation timeout contract, resolve it per selected fallback candidate, enforce it as a non-resetting total generation budget, and leave first-token/provider-skip behavior to narrower follow-ups.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the missing config surface: adding the proposed per-model timeout key under agents.defaults.models would be rejected by the strict schema, and current runner code still enforces params.timeoutMs rather than a resolved per-model generation budget.

## Solution Assessment

Yes in direction, but the final key and semantics need maintainer choice: a non-resetting generation budget is the right core contract for this issue, while first-token fail-fast and provider-skip policy should stay separate.
