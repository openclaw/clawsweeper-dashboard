## Summary

Keep this issue open: current main and the latest release still have provider-request, stream-idle, and whole-run timeout controls, but not the requested typed per-model non-resetting generation-timeout config under agent model entries.

## What This Changes

Request configurable global and per-model generation timeouts so different models can have different total-generation budgets and fallback behavior.

## Best Possible Solution

Define one typed per-model active-generation deadline with explicit precedence under the whole-run ceiling, non-resetting enforcement during active generation, timeout classification for fallback, and aligned docs/schema coverage while keeping first-token/provider-skip behavior separate.

## Reproduction Assessment

Yes at the source-contract level: current main lacks the proposed per-model timeout field and the stream idle watchdog resets on progress. I did not run a live runaway-model reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the timeout name, precedence, default behavior, and fallback semantics. A per-model control appears warranted, but it should be one coherent config/runtime contract rather than a shortcut through provider request timeout metadata.
