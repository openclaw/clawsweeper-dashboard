## Summary

Keep open: current main still has adjacent provider request-timeout and reset-on-chunk idle-watchdog behavior, but it does not provide the requested global/per-model non-resetting generation timeout contract. Related open work covers first-token, idle, local-provider, and Gemini-thinking failures, but none fully supersedes this per-model generation-budget request.

## What This Changes

The issue requests a global and per-model generation timeout so active model streams can be aborted after a bounded total generation budget.

## Best Possible Solution

Define one supported timeout contract, add it to the agent default and per-model catalog schema/docs, resolve the effective budget per selected fallback candidate, and enforce a non-resetting generation timer without changing first-token or idle semantics by accident.

## Reproduction Assessment

Yes for the missing config surface: current source and strict schema show that the requested global/per-model timeout keys are absent. The live Gemini runaway loop itself was not replayed during this read-only review.

## Solution Assessment

Unclear. A per-model generation timeout is plausible, but maintainers still need to decide the public contract and precedence against existing provider request timeout, idle watchdog, first-token, and run-timeout controls.
