## Summary

Keep open: current main still lacks a typed global/per-model non-resetting generation timeout, and the provider timeout plus idle watchdog work is adjacent rather than the requested per-model agent config contract.

## What This Changes

The issue requests global and per-model generation timeout configuration for model calls that stream indefinitely or hang too long.

## Best Possible Solution

Define a backwards-compatible timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget for each fallback candidate, enforce it during generation, and update schema, docs, and focused tests together.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the existing watchdog is idle-based rather than a total generation budget. I did not replay the original live Gemini loop in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation.
