## Summary

Keep open: current main still has only adjacent run/provider/idle timeout paths, not the requested typed global/per-model non-resetting generation timeout config.

## What This Changes

The issue asks for configurable global and per-model generation timeouts to cap long or looping model generations independently by model.

## Best Possible Solution

Add a backwards-compatible generation timeout contract at the agent default and per-model catalog levels, resolve it per selected fallback candidate, enforce it as a non-resetting active-generation budget, and keep first-token/provider-skip behavior separate.

## Reproduction Assessment

Yes by source inspection: the proposed per-model timeout key is absent from the strict agents.defaults.models type/schema/docs, while the runtime only exposes provider-scoped requestTimeoutMs and a reset-on-chunk idle watchdog. A live provider loop is not needed to verify the missing config surface.

## Solution Assessment

Yes. A narrow core config/schema/docs plus runner enforcement change is the maintainable path; merely adding requestTimeoutMs to the schema would expose the existing idle watchdog but would not fully satisfy the non-resetting generation budget requested here.
