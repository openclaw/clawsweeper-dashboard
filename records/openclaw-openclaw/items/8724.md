## Summary

Keep open: current main still lacks the requested global/per-model generation timeout config, and adjacent provider/idle timeout work does not cover a non-resetting total generation budget for active-but-runaway streams.

## What This Changes

Add configurable global and per-model generation timeouts so runaway model generations can be aborted, logged, and possibly routed to fallback models.

## Best Possible Solution

Define one backwards-compatible timeout contract, add it to agent defaults and per-model catalog schema/docs, resolve the effective budget per fallback candidate, and enforce a non-resetting generation timer while keeping first-token, idle-timeout, and semantic repetition handling separate.

## Reproduction Assessment

Yes, by source inspection: the requested per-model/global timeout keys are absent from the strict agent model config schema and docs, while the active stream watchdog still resets around chunks. A live provider loop is not required to verify the missing config surface.

## Solution Assessment

Unclear. A per-model/global generation timeout is a reasonable direction, but maintainers should choose the public contract and its interaction with provider request timeout, first-token timeout, and idle watchdog behavior before implementation.
