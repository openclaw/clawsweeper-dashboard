## Summary

Keep open. Current main still has adjacent run-level, provider request, and reset-on-chunk idle timeout controls, but it does not expose or enforce the requested typed global/per-model non-resetting generation timeout; the supplied issue discussion already reached the same current-main conclusion and shows related first-token work is a separate concern.

## What This Changes

Request a global and per-model generation timeout that aborts runaway active model generation, reports/logs the timeout, and can feed fallback behavior.

## Best Possible Solution

Add a documented, backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget for each fallback candidate, enforce it with a non-resetting timer during active generation, and leave first-token/provider-skip behavior to separate work.

## Solution Assessment
