## Summary

Keep open: current main still has provider request timeouts and a reset-on-chunk idle watchdog, but not the requested typed global/per-model non-resetting generation timeout. The strict `agents.defaults.models` schema rejects the proposed key, and embedded attempts still use the run-level timeout.

## What This Changes

Add global and per-model generation timeout configuration that aborts runaway LLM generation and can participate in fallback handling.

## Best Possible Solution

Add a backwards-compatible `generationTimeout` contract at the agent default and per-model catalog levels, resolve it for each selected fallback candidate, enforce it with a non-resetting timer during active generation, and keep first-token/provider-skip behavior separate.

## Reproduction Assessment

Yes by source inspection: the proposed per-model key is absent from types, docs, and the strict generated schema, while attempts still use the run-level `params.timeoutMs`. A live provider loop is not needed to verify that current main lacks this configuration surface.

## Solution Assessment

Yes. A narrow core config and runner-enforcement change is the maintainable solution; provider request timeout and reset-on-chunk idle timeout address different failure modes and should not be overloaded.
