## Summary

Keep open: current main and v2026.5.22 still lack the requested typed per-model/global non-resetting generation timeout under `agents.defaults.models`. Adjacent provider timeout and idle-watchdog controls exist, but they are provider-scoped or reset on stream chunks, so they do not satisfy the requested per-model generation budget.

## What This Changes

Request to add global and per-model agent generation timeout configuration so runaway model generations can abort, log, and optionally fall through to fallback models.

## Best Possible Solution

Define one backwards-compatible per-model/global generation-timeout contract, resolve the effective budget per selected fallback candidate, enforce it with a non-resetting total timer during generation, and log/classify timeout aborts for fallback without folding first-token or idle-timeout policy into this issue.

## Reproduction Assessment

Yes for the missing configuration surface: current source and v2026.5.22 both show no accepted `agents.defaults.models.*` generation-timeout field, and the active idle watchdog resets on chunks. The exact Gemini runaway-loop scenario was not live-replayed in this read-only review.

## Solution Assessment

Unclear: the requested direction is plausible, but the best field name and precedence need maintainer decision before implementation. Current provider-level timeout is not the requested per-model non-resetting generation budget.
