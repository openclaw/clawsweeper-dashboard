Keep open: current main and v2026.6.10 still lack the requested typed global/per-model non-resetting generation timeout, while the existing provider request timeout and idle watchdog only cover adjacent request or stream-silence behavior.

## What This Changes

The issue requests a global and per-model generation timeout config that aborts runaway model generation, can participate in fallback, and logs timeout events.

## Best Possible Solution

Keep this as the canonical maintainer-reviewed timeout contract, then implement a typed generation-budget schema with explicit precedence, docs, tests, non-resetting enforcement per selected model candidate, and clear fallback/logging behavior.

## Reproduction Assessment

Yes for the missing config/runtime contract: current main still rejects the proposed per-model timeout keys and only wires provider request timeout into a reset-on-progress idle watchdog. I did not run a live hung-model reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. A typed per-model generation budget is plausible, but the best fix must define precedence against provider request timeout, run timeout, first-token behavior, and the idle watchdog.
