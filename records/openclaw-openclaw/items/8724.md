## Summary

Keep open: current main added adjacent provider-level request/idle timeout behavior, but it still lacks the requested typed global/per-model generation timeout contract and non-resetting per-candidate enforcement.

## What This Changes

The issue requests configurable global and per-model generation/request timeouts so operators can bound slow or looping model generations differently per model.

## Best Possible Solution

Define a backwards-compatible timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget for each fallback candidate, enforce it during generation, and update schema, docs, and focused tests together while preserving provider-level timeout behavior.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the existing watchdog is idle-based rather than a total generation budget. I did not replay the original live Gemini loop in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation.
