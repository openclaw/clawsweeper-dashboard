## Summary

Keep this issue open: current main has adjacent run-level, provider request, and reset-on-activity idle timeouts, but still lacks a typed global/per-model non-resetting generation budget; the remaining decision is which timeout contract and precedence OpenClaw should expose.

## What This Changes

Add configurable global and per-model generation timeouts that abort runaway generations, report the timeout, and support model fallback.

## Best Possible Solution

Define distinct run, total-generation, idle, first-token, and provider-request budgets with documented precedence, then add the approved global/per-model total-generation field to the strict schema and resolve it separately for every fallback candidate.

## Reproduction Assessment

Yes at source level: current main conclusively rejects the requested per-model field and exposes only adjacent timeout mechanisms, although this review did not run a live Gemini repetition loop.

## Solution Assessment

Unclear: a model-specific total-generation budget is a reasonable direction, but exposing existing `requestTimeoutMs` alone would retain idle semantics, while introducing `generationTimeout` requires a maintainer decision on naming, defaults, precedence, and fallback behavior.
