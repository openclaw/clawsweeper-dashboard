## Summary

Keep open: current main and the latest release still lack the requested typed global/per-model non-resetting generation-timeout config. Existing provider request and idle timeout paths are adjacent, but they do not give operators a per-model total generation budget for active runaway streams.

## What This Changes

The issue requests a global and per-model generation timeout so long-running model generations can be aborted, logged, and optionally routed through fallback.

## Best Possible Solution

Define a backward-compatible global and per-model generation-timeout contract, resolve it per selected fallback candidate, enforce it as a non-resetting active-generation timer, and document how it differs from provider request, idle, first-token, and run timeouts.

## Reproduction Assessment

Yes for the missing configuration/enforcement surface: current source shows no timeout field under strict agents.defaults.models, and the active stream watchdog resets on chunks. I did not replay a live Gemini runaway-generation loop.

## Solution Assessment

Unclear until maintainers choose the public timeout contract. A typed per-model budget is plausible, but the field name, units, precedence, and fallback/error semantics need product review before a fix PR.
