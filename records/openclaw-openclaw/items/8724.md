## Summary

Keep this issue open. Current main still has adjacent agent run, provider request, and reset-on-chunk idle timeout controls, but it does not expose or enforce the requested typed global/per-model non-resetting generation timeout for active model output loops.

## What This Changes

Request adds global and per-model generation timeout configuration so runaway streaming or planning loops abort and can fail over instead of consuming tokens indefinitely.

## Best Possible Solution

Add a documented, backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget for each fallback candidate, enforce it with a non-resetting timer during active model generation, classify/log timeout aborts for fallback handling, and leave first-token and provider-skip behavior to separate work.

## Reproduction Assessment

Yes for the missing feature: the proposed per-model key is outside the strict agents.defaults.models schema, and current runtime enforcement uses params.timeoutMs plus a reset-on-chunk idle watchdog. A live Gemini infinite-planning loop is not needed to prove the config/runtime gap.

## Solution Assessment

Yes in direction. A narrow core config/schema plus embedded-runner enforcement change is the maintainable fix; provider HTTP timeouts and first-token timeout changes solve related but different failure modes.
