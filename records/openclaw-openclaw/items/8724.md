## Summary

Keep this issue open: current main has separate provider-request, stream-idle, and whole-run limits, but still lacks the requested typed per-model non-resetting generation budget; the remaining config name, precedence, and fallback semantics require maintainer product judgment.

## What This Changes

Add a global default and per-model override for a non-resetting total generation timeout that aborts the model attempt and participates in fallback.

## Best Possible Solution

Define one typed per-candidate total-generation deadline below the whole-run ceiling, resolve global and model-specific values before dispatch, and align schema, docs, logging, abort classification, and fallback tests without overloading provider `requestTimeoutMs`.

## Reproduction Assessment

Yes, at source level: current main has provider HTTP, stream-idle, and whole-run limits, but no strict per-model non-resetting generation budget; the idle-timeout test confirms active chunks reset that watchdog.

## Solution Assessment

No, the proposed capability is valid but the exact field is not yet the best settled design; maintainers should approve a distinct canonical per-candidate deadline and its precedence rather than reuse the existing provider-oriented `requestTimeoutMs`.
