## Summary

Keep open. Current main still has adjacent run-level and provider request timeout handling, but not the requested typed global/per-model non-resetting generation-timeout budget for streaming model loops.

## What This Changes

Add configurable global and per-model generation timeouts so runaway model generations can be aborted and routed through fallback handling.

## Best Possible Solution

Add a documented backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget per fallback candidate, and enforce it with a non-resetting timer during active generation.

## Reproduction Assessment

Yes. The gap is reproducible by source/schema inspection: the proposed per-model key is rejected by strict config validation, and runtime enforcement still uses params.timeoutMs plus a reset-on-chunk idle watchdog.

## Solution Assessment

Yes in direction. A narrow core config/schema/docs change plus embedded-runner enforcement is the maintainable path; provider HTTP timeouts and first-token fail-fast behavior address adjacent but different failure modes.
