## Summary

Keep open. Current main still has adjacent provider request/idle timeout controls, but not the requested typed global/per-model non-resetting generation timeout under `agents.defaults.models`; related cleanup closed duplicates toward this issue rather than replacing it.

## What This Changes

The issue requests global and per-model generation timeout configuration so slow or looping model generations can be aborted and fallback/logging can run.

## Best Possible Solution

Maintainers should choose the canonical timeout name, scope, and fallback semantics, then add schema/docs/tests plus a non-resetting generation budget without replacing provider request or idle timeout behavior.

## Reproduction Assessment

Yes, from source inspection: the strict agent model schema rejects the proposed per-model timeout keys, and the current stream watchdog resets after each chunk rather than enforcing a total generation budget. I did not run a live hung-provider reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. A per-model generation budget is plausible, but the best fix must avoid creating a competing source of truth with provider timeout and first-token/idle-timeout behavior.
