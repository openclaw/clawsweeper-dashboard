## Summary

Keep open: current main still lacks a typed global/per-model non-resetting generation timeout, and related provider/request/idle-timeout work is adjacent rather than a full substitute.

## What This Changes

Request a global and per-model generation timeout config that aborts runaway model generation and can integrate with logging and fallback.

## Best Possible Solution

Define one backwards-compatible timeout hierarchy, add a typed global/per-agent-model generation budget, resolve it per selected fallback candidate, enforce it with a non-resetting timer, and keep first-token/provider-idle bugs in narrower issues.

## Reproduction Assessment

Yes for the missing configuration/enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the existing watchdog is reset-on-chunk rather than a total generation budget. The original live Gemini loop was not replayed in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, disable behavior, and precedence before implementation; a schema-only requestTimeoutMs addition would not cover the full non-resetting budget request.
