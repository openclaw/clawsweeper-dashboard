## Summary

Keep open: current main still has adjacent run, provider request, and idle-watchdog timeout behavior, but it does not expose or enforce the requested typed global/per-model non-resetting generation timeout contract.

## What This Changes

The issue requests a global and per-model generation timeout setting so selected model attempts can be aborted after a configured budget and routed through timeout/fallback handling.

## Best Possible Solution

Define one backwards-compatible timeout hierarchy, add a typed global/per-model generation budget at the agent model-catalog surface, resolve it per selected fallback candidate, enforce it with a non-resetting timer, and keep idle-timeout and first-token behavior in their narrower tracks.

## Reproduction Assessment

Yes for the missing configuration/enforcement surface: current source shows the proposed per-model timeout key is outside strict schema and the runner has no selected-candidate non-resetting generation budget. The original live Gemini looping scenario was not replayed.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the public contract and precedence before implementation so it does not conflict with existing run, provider, idle, first-token, and fallback timeout semantics.
