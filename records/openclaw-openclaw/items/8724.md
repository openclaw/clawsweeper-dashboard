## Summary

Keep open: current main still lacks the requested typed global/per-model generation timeout contract. Existing provider request timeouts, run-level timeout, and the reset-on-chunk idle watchdog are adjacent controls, but they do not provide a public per-model non-resetting generation budget for active runaway streams.

## What This Changes

Request to add global and per-model generation timeout configuration that aborts stuck model generation and can participate in fallback handling.

## Best Possible Solution

Define one backwards-compatible timeout contract, add it to the agent default and per-model schema/docs, resolve the effective budget per selected fallback candidate, and enforce a non-resetting generation timer separate from first-token and idle watchdog behavior.

## Reproduction Assessment

Yes for the source-level configuration gap: current types, strict schemas, docs, and runner code show the requested global/per-model generation timeout is absent. A live Gemini runaway-generation loop was not replayed in this read-only review.

## Solution Assessment

Unclear. A per-model generation timeout is a plausible fix, but maintainers still need to choose the public contract, units, and precedence against existing provider request, idle, first-token, and run timeout controls.
