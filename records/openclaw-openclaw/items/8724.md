## Summary

Keep open: current main still lacks the requested typed global/per-model generation timeout contract, and the public config name, units, precedence, and fallback semantics still need maintainer product/API judgment.

## What This Changes

The issue requests global and per-model generation timeout configuration so long-running model generation can be aborted and optionally fail over.

## Best Possible Solution

Add a typed, documented timeout contract at agent default and per-model catalog levels, resolve it per selected fallback candidate, and enforce it as a non-resetting generation budget distinct from provider request, idle, first-token, and whole-run timeouts.

## Reproduction Assessment

Yes for the config-contract gap: current main's typed and Zod schemas still reject model-scoped timeout keys under agents.defaults.models. No live Gemini infinite-thinking reproduction was attempted in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, units, precedence, and fallback semantics. The maintainable direction is a typed per-model/global budget while keeping first-token and provider-skip behavior as separate work unless explicitly approved together.
