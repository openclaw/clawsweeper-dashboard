## Label Justifications

- `P2`: The missing timeout granularity affects model reliability, latency, fallback effectiveness, and token cost with limited blast radius.
- `impact:auth-provider`: The requested control changes model-specific provider invocation and fallback routing behavior.

## Summary

Current main still lacks any typed global/per-model candidate-generation timeout, and no open viable PR owns that remaining work; existing controls are a global embedded-attempt deadline, a reset-on-progress idle watchdog, and provider-level transport request timeouts, so this issue remains necessary.

## What This Changes

Add a global default and per-model total-generation timeout that aborts and logs over-budget model attempts and can feed fallback handling.

## Best Possible Solution

Define one backward-compatible candidate-scoped timeout with explicit start/stop points and fallback classification, while keeping provider HTTP request and idle watchdog controls separate.

## Reproduction Assessment

Yes at source level: current main supports a global attempt deadline and provider-level transport timeout, but its strict per-model agent schema cannot express the requested override. The original Gemini repetition incident was not live-reproduced during this read-only review.

## Solution Assessment

Unclear as currently proposed: per-model budgeting is the right product direction, but maintainers must decide whether it governs model generation only or the entire embedded attempt before choosing the field name and enforcement point.
