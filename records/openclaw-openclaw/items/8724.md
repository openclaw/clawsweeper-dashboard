## Summary

Keep open: current main still lacks the requested global/per-model generation timeout config, and the remaining work needs a maintainer decision on the public timeout contract rather than cleanup closure.

## What This Changes

Request a global and per-model generation timeout config that aborts runaway model generations, logs the timeout, and can participate in fallback routing.

## Best Possible Solution

Define one backwards-compatible timeout contract, add it to agent defaults and per-model catalog schema/docs, resolve the effective budget per fallback candidate, and enforce a non-resetting generation timer while keeping first-token and semantic repetition behavior separate.

## Reproduction Assessment

Yes, by source inspection: the requested per-model/global timeout keys are absent from the strict agent model config schema and docs, while the active stream watchdog resets on chunks. A live provider loop is not required to verify the missing config surface.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers should choose the public config name, units, and relationship to provider request timeout, first-token timeout, and idle watchdog behavior before implementation.
