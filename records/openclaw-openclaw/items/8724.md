## Summary

Keep this issue open. Current main still has adjacent provider-request, stream-idle, and whole-run timeout controls, but it does not expose the requested typed per-model non-resetting generation budget under agent model config; the May comments correctly identify that `requestTimeoutMs` is wired only through provider/model metadata paths, not through the strict `agents.defaults.models` config surface.

## What This Changes

Request adding global and per-model generation timeout configuration so runaway model generations can be aborted and, where policy allows, routed to fallback models.

## Best Possible Solution

Define a typed per-model attempt-generation deadline with explicit precedence under the whole-run ceiling, non-resetting enforcement during active generation, consistent timeout classification for fallback, and docs/schema coverage that keeps first-token/provider-skip policy separate.

## Reproduction Assessment

Yes at the source-contract level: current main lacks the proposed per-model timeout field and the stream idle watchdog resets on stream progress. I did not run a live runaway-model reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the timeout semantics and precedence. A per-model control appears warranted, but it should be added as one coherent config/runtime contract rather than conflated with provider request timeouts or first-token failover.
