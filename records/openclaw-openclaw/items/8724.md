## Summary

Keep open: current `main` still lacks the requested per-model agent timeout config, and the live issue is already labeled as requiring maintainer/product review rather than cleanup closure.

## What This Changes

The issue requests global and per-model generation timeout configuration so slow or looping models can be aborted and routed through fallback/debug paths without relying only on provider or whole-run timeouts.

## Best Possible Solution

Add a backwards-compatible typed timeout contract at the agent default and per-model catalog levels, resolve the effective budget per fallback candidate, enforce it as a non-resetting generation budget, and document how it differs from provider request, idle, first-token, and whole-run timeouts.

## Reproduction Assessment

Yes for the config-contract gap: current source shows the strict per-model agent schema rejects the requested timeout keys, while runtime timeout handling only has adjacent provider/run/idle controls. No live Gemini runaway-loop reproduction was attempted in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, units, precedence, and fallback semantics. The maintainable direction is a typed per-model/global budget with first-token and provider-skip behavior kept as separate concerns.
