## Summary

Keep open: current main still lacks the requested typed global/per-model non-resetting generation timeout, and related idle/request timeout issues are adjacent rather than superseding this canonical feature request.

## What This Changes

The issue requests global and per-model generation timeout configuration for model runs, with abort, logging, and possible fallback after the configured budget is exceeded.

## Best Possible Solution

Define one backwards-compatible timeout hierarchy, add a typed global/per-agent-model generation budget, resolve it per selected fallback candidate, enforce it with a non-resetting timer, and keep first-token/provider-idle bugs in their narrower issues.

## Reproduction Assessment

Yes for the missing configuration/enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the existing watchdog is reset-on-chunk rather than a total generation budget. The original live Gemini loop was not replayed in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, disable behavior, and precedence before implementation; a schema-only requestTimeoutMs addition would not cover the full non-resetting budget request.
