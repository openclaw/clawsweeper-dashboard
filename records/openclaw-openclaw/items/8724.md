## Summary

Keep open: current main still lacks the requested global/per-model generation timeout config, and adjacent provider request, run-level, and reset-on-chunk idle timeout paths do not cover the non-resetting total generation budget requested here.

## What This Changes

The issue asks for a configurable global and per-model generation timeout to abort runaway model generations and optionally trigger fallback.

## Best Possible Solution

Settle the public timeout contract, then add a backwards-compatible agent default and per-model budget that is resolved per selected fallback candidate, enforced as a non-resetting active-generation limit, logged/classified for fallback, and documented with schema/test coverage.

## Reproduction Assessment

Yes, by source inspection: the requested per-model/global timeout keys are absent from the strict agent model config schema/docs, and the active stream watchdog resets on each chunk rather than enforcing a total generation budget. A live provider loop is not needed to verify the missing config surface.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers should first choose the public config name and ownership boundary so it does not conflict with the existing provider timeout and doctor migration contract.
