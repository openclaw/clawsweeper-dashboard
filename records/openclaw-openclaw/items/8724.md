## Summary

Keep open: current main still lacks the requested global/per-model generation timeout config, and the remaining work needs a maintainer decision on the public timeout contract rather than cleanup closure.

## What This Changes

Request adding global and per-model generation timeout settings that abort runaway model generation and can feed fallback handling.

## Best Possible Solution

Define one backwards-compatible public timeout contract, then wire it through agent defaults and per-model catalog entries into the selected fallback candidate's non-resetting generation budget with schema, docs, logging, and tests.

## Reproduction Assessment

Yes, by source inspection: the requested per-model/global timeout keys are absent from the strict agent model config schema and docs, while the active stream watchdog resets on each chunk. A live provider loop is not required to verify the missing config surface.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers should choose the public config name, units, and relationship to provider request timeout before implementation.
