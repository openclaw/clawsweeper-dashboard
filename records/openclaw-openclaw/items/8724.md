## Summary

Keep open: current main and the latest release still lack a typed global/per-model generation-timeout contract, and the remaining public config semantics need maintainer/product judgment before a new automated fix PR.

## What This Changes

The issue requests a global or per-model generation timeout that can abort long-running model generation, log the timeout, and optionally trigger fallback.

## Best Possible Solution

Define the supported timeout hierarchy, then add a documented global/per-model generation budget resolved per fallback candidate and enforced as a non-resetting total generation cap alongside existing idle/provider/run timeouts.

## Reproduction Assessment

Yes for the missing configuration surface: current source and the latest release schema reject the proposed per-model timeout key, and current runner code still uses params.timeoutMs plus a reset-on-chunk idle watchdog. The exact live Gemini loop was not replayed.

## Solution Assessment

Unclear. A per-model non-resetting generation budget is a plausible direction, but maintainers still need to choose the public key, units, disable behavior, and precedence with existing timeout paths.
