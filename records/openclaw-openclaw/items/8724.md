## Summary

Keep open: current main still does not expose the requested global/per-model generation timeout contract, and the remaining naming, units, precedence, and fallback behavior need maintainer product/API judgment before implementation.

## What This Changes

Add configurable global and per-model generation timeouts so slow or looping model generations can be bounded differently per model.

## Best Possible Solution

Add a typed, documented timeout contract at the agent default and per-model catalog levels, resolve it per selected fallback candidate, and enforce it as a non-resetting generation budget distinct from provider request, idle, first-token, and whole-run timeouts.

## Reproduction Assessment

Yes for the config-contract gap: current main's typed and Zod schemas reject model-scoped timeout keys under agents.defaults.models, while runtime only consumes adjacent provider-derived requestTimeoutMs and idle timers. No live Gemini infinite-thinking reproduction was attempted in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, units, precedence, and fallback semantics. The maintainable direction is a typed per-model/global budget with first-token and provider-skip behavior kept as separate work.
