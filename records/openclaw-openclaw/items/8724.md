## Summary

Keep open: current main still lacks the requested global/per-model generation timeout contract, and the existing provider/request/idle timeout paths do not provide a typed non-resetting per-model generation budget for active runaway streams.

## What This Changes

The issue requests configurable global and per-model generation timeouts to abort runaway model generations and optionally trigger fallback.

## Best Possible Solution

Define one backwards-compatible timeout contract, add it to agent defaults and per-model catalog schema/docs, resolve the effective budget per selected fallback candidate, and enforce a non-resetting generation timer separate from first-token and idle watchdog behavior.

## Reproduction Assessment

Yes, by source inspection: the requested per-model/global timeout keys are absent from the strict agent model config schema and docs, while the active stream watchdog still resets around chunks. A live provider loop is not required to verify the missing config surface.

## Solution Assessment

Unclear. A generation-timeout config is a reasonable direction, but maintainers should choose the public contract and precedence rules before implementation.
