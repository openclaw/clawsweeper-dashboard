## Summary

Keep open: current main has adjacent provider-level request timeout and idle-watchdog controls, but it still lacks the requested typed global/per-model non-resetting generation timeout contract and per-candidate enforcement.

## What This Changes

The issue requests configurable global and per-model generation timeouts so runaway or long-running model generations can be aborted and optionally routed through fallback behavior.

## Best Possible Solution

Define a backwards-compatible generation-timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget for each fallback candidate, enforce it during generation, and update schema, docs, and focused runner/fallback tests together while preserving provider-level timeout behavior.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the active watchdog is idle/request based rather than a total generation budget. I did not replay the original live Gemini loop in this read-only review.

## Solution Assessment

Unclear. A global/per-model generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation.
