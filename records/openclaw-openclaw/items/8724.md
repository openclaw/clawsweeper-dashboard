## Summary

Keep open: current main still lacks the requested typed global/per-model generation-timeout config and enforcement. Provider-level request/idle timeout support now covers adjacent slow-provider cases, but it is not a strict `agents.defaults.models` per-model non-resetting generation budget for active runaway streams.

## What This Changes

The issue requests global and per-model generation timeout configuration so operators can cap runaway model generations and tune budgets by model.

## Best Possible Solution

Define a backward-compatible global and per-model generation-timeout contract, resolve the effective budget per selected fallback candidate, enforce it as a non-resetting active-generation timer, and document how it differs from provider request, idle, first-token, and run timeouts.

## Reproduction Assessment

Yes for the missing configuration/enforcement surface: current source shows no timeout field under strict `agents.defaults.models`, and the active stream watchdog resets on chunks. I did not replay a live Gemini runaway-generation loop.

## Solution Assessment

Unclear until maintainers choose the public timeout contract. A typed per-model budget is plausible, but the field name, units, precedence, and fallback/error semantics need product review before a fix PR.
