## What This Changes

The issue requests a configurable global and per-model generation timeout that aborts runaway active generation, logs the timeout, and may trigger model fallback.

## Best Possible Solution

Keep this as the canonical maintainer-reviewed timeout contract, then implement a typed generation-budget schema with explicit precedence, docs, tests, and non-resetting enforcement per selected model candidate.

## Reproduction Assessment

Yes for the missing config/runtime contract: current main still rejects the proposed per-model timeout keys and only wires provider request timeout into a reset-on-progress idle watchdog. I did not run a live hung-model reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. A typed per-model generation budget is plausible, but the best fix must define precedence against provider request timeout, run timeout, and first-token/idle behavior.
