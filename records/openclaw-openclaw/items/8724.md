## Summary

Keep open: current main still lacks the requested typed global/per-model generation timeout contract, and the public config name, units, precedence, and fallback semantics still need maintainer product/API judgment.

## What This Changes

Request a global and per-model generation timeout that can abort runaway model generation and optionally participate in fallback routing.

## Best Possible Solution

Define one typed, documented timeout contract for default and per-model generation budgets, resolve it per selected fallback candidate, and enforce it as a non-resetting budget distinct from provider request, idle, first-token, and whole-run timeouts.

## Reproduction Assessment

Yes for the config-contract gap: current main's typed and Zod schemas still reject model-scoped generationTimeout/requestTimeoutMs-style keys under agents.defaults.models. No live Gemini infinite-thinking reproduction was attempted in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, units, precedence, and fallback semantics. The maintainable direction is a typed per-model/global budget while keeping first-token, idle-timeout, provider-skip, and provider request timeout behavior separate unless explicitly approved together.
