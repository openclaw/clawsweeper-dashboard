## Summary

Keep open: current main and v2026.5.22 still lack the requested typed global/per-model non-resetting generation timeout; the provider request timeout and reset-on-chunk idle watchdog are adjacent but do not solve the central request.

## What This Changes

The issue requests global and per-model generation timeout configuration so selected models can have different total generation budgets and timeout/fallback behavior.

## Best Possible Solution

Define one backwards-compatible generation-timeout hierarchy for agent defaults and per-model entries, resolve the effective non-resetting budget per selected fallback candidate, and update schema, docs, logging/fallback classification, and focused tests together.

## Reproduction Assessment

Yes for the missing configuration and enforcement surface: current source shows the proposed per-model timeout keys are outside strict schemas, and the active watchdog is idle/request based rather than a total generation budget. I did not replay the original live Gemini loop.

## Solution Assessment

Unclear. A dedicated generation budget is plausible, but maintainers should choose the supported field name, units, precedence, disable behavior, and fallback classification before implementation; a single requestTimeoutMs schema addition looks too narrow for the original non-resetting generation-timeout request.
