## Label Justifications

- `P2`: The missing per-model budget affects routing reliability and runaway token cost for a bounded subset of multi-model configurations.
- `impact:auth-provider`: Timeout expiry would directly change model candidate execution and fallback routing.

## Summary

Current main has a global non-resetting run deadline and adjacent provider/idle timeout controls, but it still lacks the requested typed per-model total-generation budget. The remaining work needs maintainer agreement on semantics and precedence before implementation because `generationTimeout`, provider request timeout, stream idle timeout, and the global run deadline represent different failure modes.

## What This Changes

Add a configurable global and per-model total-generation timeout that aborts runaway model execution and can trigger fallback.

## Best Possible Solution

Define one typed per-model total-attempt budget with documented precedence against the global run deadline, preserve current behavior when unset, classify expiry for fallback, and keep first-token and idle timeout controls as separate contracts.

## Reproduction Assessment

Yes at source level. The strict model-entry schema omits a timeout field, the active-stream watchdog resets on each chunk, and the non-resetting deadline is currently global rather than candidate-specific; no live runaway model was invoked during this read-only review.

## Solution Assessment

Unclear as proposed. A per-model total budget is a plausible narrow solution, but maintainers must first choose its canonical name, scope, precedence, and fallback behavior so it does not duplicate existing provider request or idle timeout controls.
