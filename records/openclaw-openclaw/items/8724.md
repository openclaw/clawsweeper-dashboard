## Summary

Keep open: current main still has adjacent provider request and stream-idle timeout controls, but not the requested typed global/per-model non-resetting generation timeout config.

## What This Changes

The issue requests configurable global and per-model generation timeouts so long-running or looping model generations can be aborted and optionally fail over.

## Best Possible Solution

Add a backwards-compatible generation timeout contract at the agent default and per-model catalog levels, resolve it per fallback candidate, and enforce it as a non-resetting active-generation budget.

## Reproduction Assessment

Yes by source inspection: the proposed per-model timeout key is absent from the strict `agents.defaults.models` type/schema/docs, while runtime enforcement is currently provider request timeout plus reset-on-chunk idle timeout. A live provider loop is not needed to verify the missing config surface.

## Solution Assessment

Yes. A narrow core config/schema/docs plus embedded-runner enforcement change is the maintainable path; exposing only the existing provider request timeout or idle watchdog would not satisfy the requested non-resetting generation budget.
