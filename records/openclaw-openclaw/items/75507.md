## Summary

Keep open: the intended launchd fix is narrowly scoped, but the current PR head is not correct because it removed the platform argument from the helper while the new darwin branch still reads `params.platform`. That should fail TypeScript and would leave the macOS path inactive at runtime.

## What This Changes

This PR changes gateway install planning, launchd tests, and the changelog so macOS launchd keeps durable managed service env values in the owner-only env file while preserving non-darwin metadata-only behavior.

## Best Possible Solution

Restore the platform plumbing into `buildGatewayInstallEnvironment`, keep the darwin-only value retention and non-darwin metadata-only behavior, then land after targeted daemon/launchd/systemd tests and exact-head CI.

## Reproduction Assessment

Yes. The PR patch itself is a high-confidence reproduction: the final head reads `params.platform` after removing it from the helper params and call, so TypeScript should reject it and runtime would never enter the darwin branch.

## Solution Assessment

No for the current head. The intended solution direction is right, but the safe implementation is to re-add the platform parameter/call or pass an explicit darwin flag so the new launchd-only branch is reachable and type-safe.
