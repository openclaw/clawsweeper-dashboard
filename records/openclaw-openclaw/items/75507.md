## Summary

Keep open: current main still drops durable managed environment values before the launchd env-file writer sees them, while this PR is a focused darwin-scoped correction with tests and no concrete blocking review finding.

## What This Changes

The PR updates macOS gateway install planning so launchd retains durable managed environment values for the owner-only LaunchAgent env file, adds focused regression coverage, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent darwin-scoped handoff fix after CI and macOS launchd validation, keeping managed values in the existing owner-only LaunchAgent env file while preserving systemd's metadata-only unit behavior.

## Reproduction Assessment

Yes. The linked issue gives concrete macOS launchd steps, and current-main source inspection shows durable .env/config values are converted to OPENCLAW_SERVICE_MANAGED_ENV_KEYS and deleted before the launchd env-file writer serializes them.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: keep durable managed values only for the darwin launchd env-file handoff, avoid putting secrets in plist EnvironmentVariables, and leave non-darwin service metadata behavior intact.
