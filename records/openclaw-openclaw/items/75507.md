## What This Changes

The PR adds a darwin-specific gateway install handoff that keeps durable managed environment values for the owner-only LaunchAgent env file, updates focused tests, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent darwin-scoped handoff fix after pinning the remaining non-darwin test expectations and passing CI, keeping managed values in the existing owner-only LaunchAgent env file while preserving systemd's metadata-only unit behavior.

## Reproduction Assessment

Yes. The product bug is reproducible from the linked macOS launchd steps and current-main code path; the review finding is reproducible by running the focused install-plan tests on macOS after the PR because unpinned tests inherit `process.platform === "darwin"`.

## Solution Assessment

Mostly yes. The darwin-only retention approach is the narrow maintainable fix, but the test suite should explicitly pin remaining linux expectations so the same focused tests pass on both macOS and Linux.
