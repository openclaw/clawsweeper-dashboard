## Summary

Keep open: the underlying bug is still valid and this PR is a plausible implementation path, but the current patch is draft/conflicting, lacks real behavior proof, and removes existing Node/macOS approvals hardening.

## What This Changes

The PR allows trusted first-level `.openclaw` symlinks for exec approvals across Node and macOS paths, adds hardening tests, and documents the supported layout.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `.openclaw` symlink while preserving current Node directory permissions, atomic write safeguards, final-file protections, deeper-symlink rejection, and macOS file locking.

## Reproduction Assessment

Yes. Source inspection shows `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and current main's approvals-directory guard rejects a symlinked `.openclaw` component before writing.

## Solution Assessment

No, not as submitted. The first-level symlink exception is the right direction, but this patch must preserve existing Node chmod hardening and macOS file-lock serialization, and maintainers should consolidate it with the overlapping open PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the macOS approvals file lock:** `apps/macos/Sources/OpenClaw/ExecApprovals.swift:345-351`
  - body: Current main serializes approvals snapshots, loads, saves, and ensure-file rewrites with `withFileLock`, but this rewrite validates and writes the security-sensitive approvals file without that lock. Keep validation and file I/O under the same lock so concurrent policy reads, socket-token updates, and writes cannot race or lose updates.
  - confidence: 0.88
- **[P2] Keep hardening the approvals directory to 0700:** `src/infra/exec-approvals.ts:285-287`
  - body: Current main chmods the exec approvals directory to owner-only permissions and has regression coverage for that behavior. This `ensureDir()` rewrite no longer chmods an existing or newly-created `.openclaw` target, so normal umasks can leave security-sensitive approvals state in a traversable 0755 directory.
  - confidence: 0.84
