## Summary

Keep open: the underlying symlinked `~/.openclaw` bug is still present on current main by source inspection, but this draft external PR removes macOS approvals-store locking, lacks after-fix real behavior proof, and overlaps another open implementation.

## What This Changes

The branch broadens Node and macOS exec-approvals path validation to allow trusted first-level `.openclaw` and state-dir symlinks, adds hardening tests, and documents the supported CLI approvals shape.

## Best Possible Solution

Land one security-reviewed implementation that accepts only validated first-level `.openclaw` symlinks while preserving macOS file locking, final-file protections, and deeper-symlink rejection.

## Reproduction Assessment

Yes, by source inspection. With `.openclaw` as a symlink under the resolved home directory, `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and current main rejects the symlinked approvals directory before writing.

## Solution Assessment

No. The trusted symlink direction is plausible, but this branch is not ready until macOS locking is preserved, contributor real behavior proof is added, and maintainers choose the scope relative to https://github.com/openclaw/openclaw/pull/72650.
