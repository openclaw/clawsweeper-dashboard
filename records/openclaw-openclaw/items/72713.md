## Summary

Keep open: the linked bug is still valid on current main, but this draft/conflicting PR is not mergeable as-is because it relaxes security-sensitive exec-approvals path handling while dropping existing macOS serialization and Node directory-permission hardening.

## What This Changes

The PR allows trusted first-level `.openclaw` symlinks for exec approvals across Node and macOS paths, adds hardening tests, and documents the supported layout.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `.openclaw` symlink while preserving current Node directory permissions, atomic write safeguards, final-file protections, deeper symlink rejection, and macOS file locking.

## Reproduction Assessment

Yes, from source inspection. `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and current `ensureDir()` still rejects a symlinked `.openclaw` directory before writing.

## Solution Assessment

No, not as submitted. The first-level symlink exception is the right direction, but the implementation must keep existing Node and macOS hardening and be consolidated with the overlapping open PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the macOS approvals file lock:** `apps/macos/Sources/OpenClaw/ExecApprovals.swift:345-351`
  - body: Current main serializes approvals snapshots, loads, saves, and ensure-file rewrites with `withFileLock`, but this rewrite validates and writes the security-sensitive approvals file without that lock. Keep validation and file I/O under the same lock so concurrent policy reads, socket-token updates, and writes cannot race or lose updates.
  - confidence: 0.88
- **[P2] Keep hardening the approvals directory to 0700:** `src/infra/exec-approvals.ts:285-287`
  - body: Current main chmods the exec approvals directory to owner-only permissions and has a regression test for that behavior. This `ensureDir()` rewrite no longer chmods an existing or newly-created `.openclaw` target, so normal umasks can leave security-sensitive approvals state in a traversable 0755 directory; restore the 0700 normalization after validating the resolved directory.
  - confidence: 0.84
