## Summary

Keep open. The PR addresses a valid, narrow Moonshot/Kimi thinking-mode replay bug: current main does not synthesize `reasoning_content` on assistant tool-call messages, while the PR adds provider-scoped payload shaping and focused tests. The patch looks functionally aligned with the upstream Kimi contract and OpenClaw's existing DeepSeek compatibility pattern, but it is missing the required changelog entry for a user-facing provider fix.

## What This Changes

The PR adds Moonshot request-payload shaping that injects an empty `reasoning_content` field on replayed assistant messages with `tool_calls` when thinking is enabled, plus focused Moonshot extra-params regression tests.

## Best Possible Solution

Land the provider-scoped Moonshot payload repair with its focused regression tests and a single active-version changelog entry, keeping the fix at the final outgoing request boundary and leaving generic replay policy unchanged.

Yes. ## Solution Assessment

Yes, with the changelog gap fixed. The provider wrapper is the narrowest maintainable place for this compatibility repair because Moonshot owns the `thinking` payload semantics, and the approach mirrors the existing DeepSeek V4 backfill pattern.
