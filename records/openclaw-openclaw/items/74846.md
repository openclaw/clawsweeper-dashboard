Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` backfill, and this PR is a narrow provider-scoped fix with focused tests, but it is not policy-complete because the user-facing provider fix has no active-version changelog entry.

## What This Changes

The PR injects an empty `reasoning_content` field into Moonshot assistant tool-call replay messages when thinking is enabled and adds Moonshot extra-params regression tests for enabled, disabled, and preexisting reasoning-content cases.

## Best Possible Solution

Land the provider-scoped Moonshot payload repair with its focused regression tests and a single active-version changelog entry, keeping the compatibility shim at the outgoing request boundary rather than changing generic transcript replay policy.

## Solution Assessment

Yes, after the changelog gap is fixed. The PR's request-boundary backfill is the narrowest maintainable fix and mirrors the existing DeepSeek V4 compatibility pattern without altering unrelated providers.
