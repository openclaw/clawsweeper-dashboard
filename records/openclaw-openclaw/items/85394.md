## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve due-by-default cron_run:** `src/agents/tools/cron-tool.ts:812-813`
  - body: Current main defaults omitted cron run mode to `due`, so a manual run only executes when the job is currently due. This per-action path defaults missing `runMode` to `force`, which can unexpectedly run scheduled jobs that existing `cron` callers would skip.
  - confidence: 0.94
- **[P1] Forward wake targets through cron_wake:** `src/agents/tools/cron-tool.ts:824-830`
  - body: The current `wake` action resolves and forwards the caller's `sessionKey`/`agentId`, including explicit-target validation, so wake events return to the originating conversation. The new `cron_wake` path sends only `{ mode, text }`, regressing non-main and multi-agent wake routing.
  - confidence: 0.93
- **[P1] Carry creator tool caps into cron_*:** `src/agents/openclaw-tools.ts:378-389`
  - body: Current main passes `cronCreatorToolAllowlist` into `createCronTool` and caps scheduled agent-turn tools on add/update. This call to `createCronTools` omits the allowlist, so per-action cron add/update can create or edit scheduled turns with tools outside the creator's restricted surface.
  - confidence: 0.92
- **[P1] Keep add/update input guardrails:** `src/agents/tools/cron-tool.ts:672-675`
  - body: Current main canonicalizes cron add/update inputs, rejects command payloads, rejects blank delivery fields, and caps edited scheduled-turn tools before gateway calls. The per-action add/update paths pass raw `job`/`patch` through the older normalizers, bypassing those current guardrails.
  - confidence: 0.9
- **[P1] Classify cron_* mutation and replay safety:** `src/agents/tools/cron-tool.ts:1170-1180`
  - body: Registering `cron_add`, `cron_update`, `cron_remove`, and `cron_wake` without updating mutation/replay classification makes the new names fall through the current exact-`cron` action logic. Mutating cron calls can be treated as non-mutating, while read-only per-action calls are not in the audited replay-safe set.
  - confidence: 0.89
