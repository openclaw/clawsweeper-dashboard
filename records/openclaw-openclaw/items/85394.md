Overall confidence: 0.88

Full review comments:

- **[P1] Classify cron_* tools as mutating before exposing them:** `src/agents/tools/cron-tool.ts:1174-1179`
  - body: This registers mutating scheduler tools such as `cron_add`, `cron_update`, `cron_remove`, `cron_run`, and `cron_wake`, but the current mutation classifier only knows the exact `cron` tool name. Since per-action calls have no `action` field, `buildToolMutationState` will treat these scheduler mutations as non-mutating, so retry/replay handling can miss cron side effects. Add cron family handling and regression coverage in the mutation/replay helpers before exposing these names.
  - confidence: 0.89
- **[P1] Carry wake session routing into cron_wake:** `src/agents/tools/cron-tool.ts:407-412`
  - body: `CronWakeSchema` only exposes `text` and `mode`, and the new wake handler forwards only `{ mode, text }`. Current main's legacy wake action resolves explicit or inferred `sessionKey`/`agentId` so wakes return to the originating conversation lane; the per-action tool needs to expose and pass the same target fields, or non-main and multi-agent wakes can land on the wrong lane.
  - confidence: 0.87
