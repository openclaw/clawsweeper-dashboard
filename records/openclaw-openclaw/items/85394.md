- **[P1] Add the cron_* tools to the core tool catalog:** `src/agents/openclaw-tools.ts:378`
  - body: `createOpenClawTools()` now exposes `cron_update` and the other per-action tools, but tool-profile filtering is driven by `src/agents/tool-catalog.ts`, where the coding profile and `group:automation` still only know about `cron`.   - confidence: 0.91
- **[P1] Mark cron_* calls as mutating side effects:** `src/agents/tools/cron-tool.ts:1161-1169`
  - body: The new `cron_add`, `cron_update`, `cron_remove`, `cron_run`, and `cron_wake` names bypass the existing cron side-effect classifiers: `tool-mutation.ts` only recognizes `cron`, and `handleToolExecutionEnd` only increments `successfulCronAdds` for `toolName === "cron"` with `action: "add"`. That means committed cron changes through the new tools can be treated as non-mutating/replay-safe and failed mutations will not be preserved like the legacy path. Please update the mutation and cron-add tracking tests for the new names before exposing them.
  - confidence: 0.89
