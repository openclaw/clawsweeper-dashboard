Full review comments:

- **[P1] Preserve due-by-default cron_run:** `src/agents/tools/cron-tool.ts:812-813`
  - body: Current main defaults omitted cron run mode to `due`, so a manual run skips jobs that are not currently due. This per-action path defaults missing `runMode` to `force`, which can unexpectedly execute scheduled jobs that existing `cron` callers would skip.
  - confidence: 0.94
- **[P1] Forward wake targets through cron_wake:** `src/agents/tools/cron-tool.ts:828-830`
  - body: Current main resolves and forwards `sessionKey` and `agentId` for wake calls, including explicit-target validation. The new `cron_wake` path sends only `{ mode, text }`, so wake events can route to the wrong session or agent lane.
  - confidence: 0.93
- **[P1] Carry creator tool caps into cron_*:** `src/agents/openclaw-tools.ts:378-389`
  - body: Current main passes `cronCreatorToolAllowlist` into the cron tool and caps scheduled agent-turn tools on add/update. This call to `createCronTools` omits the allowlist, so per-action cron add/update can create or edit scheduled turns with tools outside the creator's restricted surface.
  - confidence: 0.92
- **[P1] Keep add/update input guardrails:** `src/agents/tools/cron-tool.ts:672-675`
  - body: Current main canonicalizes cron add/update inputs, rejects command payloads, rejects blank delivery fields, and caps edited scheduled-turn tools before gateway calls. These per-action add/update paths pass raw `job` and `patch` through older normalizers, bypassing those current guardrails.
  - confidence: 0.9
- **[P1] Classify cron_* mutation and replay safety:** `src/agents/tools/cron-tool.ts:1168-1180`
  - body: Registering `cron_add`, `cron_update`, `cron_remove`, and `cron_wake` without updating current-main mutation/replay classification leaves the new names outside the audited exact-`cron` action logic. Mutating cron calls can be treated as non-mutating, while read-only per-action calls are not recognized as replay-safe.
  - confidence: 0.87
