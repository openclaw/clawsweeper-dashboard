## Summary

Keep open: the bug target is valid, but the patch short-circuits MiniMax through a core direct fetch before the existing provider transport preparation, which bypasses configured transport policy and also hard-codes usage cost to zero.

## What This Changes

The PR adds a MiniMax-specific direct Anthropic Messages response parser in the simple-completion runtime and a regression test for thinking-before-text content.

## Best Possible Solution

Handle thinking-first MiniMax responses through the existing Anthropic/simple transport or provider hook path so text extraction, auth, proxy/TLS/SSRF guards, and usage accounting stay shared.

## Reproduction Assessment

Partially: the linked issue gives a concrete `openclaw infer model run` command and MiniMax response shape, and the PR proof replays that shape after the patch. I did not establish a live current-main failure because MiniMax credentials are unavailable in this review.

## Solution Assessment

No. The current fix parses the right response shape, but the safer maintainable solution is to extend the existing Anthropic/simple transport or MiniMax provider hook instead of adding a provider-specific direct fetch in core.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve provider transport for MiniMax requests:** `src/agents/simple-completion-runtime.ts:274-281`
  - body: This direct path returns before `prepareModelForSimpleCompletion` can swap in OpenClaw's transport-aware Anthropic path. For MiniMax configs with `request.proxy`, `request.tls`, or a local service, the new raw `fetch` ignores the guarded fetch/proxy/TLS path and sends or fails against the direct endpoint; move the parser into the existing transport/provider seam instead of short-circuiting it.
  - confidence: 0.89
- **[P3] Report MiniMax usage cost from the model rates:** `src/agents/simple-completion-runtime.ts:329`
  - body: The direct parser hard-codes `usage.cost` to zero. Plugin LLM completions treat an explicit raw cost as authoritative before estimating from config, so paid MiniMax API-key calls with token usage will be logged/reported as zero cost. Calculate from the model rates or omit the explicit cost so the existing estimator can run.
  - confidence: 0.83
