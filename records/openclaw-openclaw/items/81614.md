## Summary

Keep open: the bug and proposed direction are plausible, and the PR includes useful local runtime proof, but the implementation short-circuits OpenClaw’s prepared transport path for MiniMax Anthropic Messages models.

## What This Changes

The PR adds a MiniMax-specific Anthropic Messages non-streaming parser in `src/agents/simple-completion-runtime.ts` plus a regression test for thinking-before-text responses.

## Best Possible Solution

Put the text-block extraction fix in the shared simple-completion or Anthropic transport boundary so MiniMax thinking-first responses work without bypassing provider transport policy.

## Reproduction Assessment

Yes for the proposed-patch regression: source inspection shows MiniMax Anthropic models enter the new direct branch before transport preparation. The original MiniMax service failure was not live-reproduced here, but the PR supplies a local replay of the reported response shape.

## Solution Assessment

No, not as written. The parser behavior is useful, but the maintainable fix should preserve `prepareModelForSimpleCompletion` and the OpenClaw Anthropic transport rather than adding an early provider-specific raw fetch.
