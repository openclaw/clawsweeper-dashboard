## Summary

Keep open. The SQLite-backed plugin state migration is directionally useful, but the branch currently breaks documented `voicecall --file` behavior for custom files named `calls.jsonl`; because persisted state and fallback behavior are compatibility-sensitive, that should be fixed before merge.

## What This Changes

The PR moves MS Teams pending upload and SSO token stores plus voice-call call history from JSON/JSONL sidecar files into SQLite-backed plugin state with migration and regression coverage.

## Best Possible Solution

Keep the SQLite-backed default history path, but preserve literal JSONL handling for explicit `--file` inputs and cover custom `calls.jsonl` paths in CLI tests before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: pass `openclaw voicecall tail` or `latency` an explicit existing custom path ending in `calls.jsonl`; the PR condition routes it to SQLite instead of reading that file. Current docs and current main both support treating `--file` as the literal JSONL file path.

## Solution Assessment

No, not yet. The migration direction is maintainable, but the CLI needs to distinguish the default SQLite-backed history path from an explicit JSONL file path before this is the best fix.
