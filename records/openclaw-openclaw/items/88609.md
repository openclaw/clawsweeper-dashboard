## Best Possible Solution

Keep SQLite-backed storage for the default plugin state path, preserve literal JSONL reads for explicit custom `--file` inputs, add focused CLI regression tests, and require redacted real-flow proof or explicit maintainer acceptance before merge.

## Solution Assessment

No, not as-is. The migration direction is maintainable, but the CLI must distinguish the resolved default history path from explicit user-provided JSONL files before this is the safest implementation.
