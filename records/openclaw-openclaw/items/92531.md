## Review Findings

Overall confidence: 0.92

Full review comments:

- **[P1] Export a loadable plugin entry:** `extensions/shadow-logger/index.ts:15-23`
  - body: OpenClaw loads native plugins by resolving a default/function export with `register` or `activate`; this file only exports a `ShadowLoggerPlugin` class with `onStart`, so the plugin would not register any hooks when loaded. Use `definePluginEntry({ register(api) { ... } })` for a non-channel plugin.
  - confidence: 0.92
- **[P2] Register hooks through the plugin API:** `extensions/shadow-logger/index.ts:35-42`
  - body: Calling `registerInternalHook` directly bypasses the plugin registry's hook metadata, plugin config injection, and rollback/unregister path. A plugin should call `api.registerHook(...)` from its `register(api)` entry so reloads and diagnostics stay owned by the plugin registry.
  - confidence: 0.86
- **[P2] Remove compile-blocking unused symbols:** `extensions/shadow-logger/index.ts:1-3`
  - body: `tsconfig.extensions.json` enables `noUnusedLocals` and `noUnusedParameters`, but this file imports `createSubsystemLogger` and declares an unused `runtime` parameter. That will fail extension type checking before runtime behavior is reached.
  - confidence: 0.88
