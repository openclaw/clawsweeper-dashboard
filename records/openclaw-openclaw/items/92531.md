## Summary

This optional Supabase-specific memory exporter belongs outside OpenClaw core because the existing plugin API already exposes typed message hooks; the submitted bundled implementation also bypasses that API, is not loadable through the standard plugin entry contract, and creates an unbounded sensitive-data export path.

## What This Changes

Adds a bundled shadow-logger plugin that captures inbound and outbound message events and writes dialogue content and metadata to Supabase.

## Best Possible Solution

Publish this capability through ClawHub as an external plugin using `definePluginEntry` and typed `api.on` hooks, with explicit opt-in, least-privilege credentials, a redacted field allowlist, retention documentation, and real Supabase proof.

## Reproduction Assessment

Yes for the patch defect: source inspection shows that the module lacks the default plugin definition consumed by the loader, leaving its `onStart` method without a registration path. The requested memory exporter itself is a new feature, not a current-main bug.

## Solution Assessment

No; the supported public message hooks already provide the necessary extension seam, so bundling a Supabase-specific exporter that imports core internals is not the narrowest maintainable solution.
