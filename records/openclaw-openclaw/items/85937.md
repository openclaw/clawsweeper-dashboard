## Best Possible Solution

Keep this issue open while maintainers isolate the duplicate successful-assistant producer and add a narrow idempotency guard that preserves legitimate repeated replies across separate turns.

## Solution Assessment

Unclear until the second writer is isolated; a storage fingerprint guard is a plausible backstop, but the safer fix should target the finalization, fallback, or hook path that emits the duplicate while keeping intentional repeated replies valid.
