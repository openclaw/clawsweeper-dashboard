## What This Changes

The issue reports Gemini native streaming persisting duplicate `assistant.text` rows in session JSONL while Telegram outbound dedupe hides the duplicate delivery.

## Best Possible Solution

Keep this issue open until maintainers isolate the duplicate producer and land a narrow same-turn assistant persistence or finalization guard, with outbound logging that distinguishes sent from suppressed delivery.

## Reproduction Assessment

No live high-confidence Gemini/Telegram reproduction was run. Source inspection does show that if two ordinary assistant text final writes arrive back-to-back, current main will append both with fresh ids and parent linkage rather than deduping them.

## Solution Assessment

Unclear. A storage fingerprint guard is a plausible backstop, but the best fix should first determine whether Gemini finalization, fallback, hook handling, or raw/cleaned assistant persistence is producing the second row.
