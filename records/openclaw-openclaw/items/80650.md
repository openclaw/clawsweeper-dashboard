## What This Changes

The issue reports that full `openclaw backup create` fails or hangs on Windows with long workspace paths while `--only-config` succeeds.

## Best Possible Solution

Reproduce on Windows current main with a long-path workspace, then fix the existing backup archiver or diagnostics without changing default backup scope unless owners explicitly choose that policy.

## Solution Assessment

Unclear. Replacing the backend with 7-Zip or adding broad workspace exclusions may be too wide until the node-tar Windows failure is reproduced; a narrower path-handling or diagnostic fix may be better.
