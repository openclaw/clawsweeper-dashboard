## Summary

Keep open. The report is recent and targets the documented backup flow; current main still archives workspace directories by default and has no Windows long-path proof or merged fix, while the reported BSD tar root cause does not match the current npm `tar` implementation.

## What This Changes

The issue reports that `openclaw backup create` fails or hangs on Windows when a full backup includes workspace paths longer than 260 characters, while `--only-config` succeeds.

## Best Possible Solution

Reproduce on Windows current main with a configured long-path workspace, then fix the existing archiver/path handling or diagnostics; route optional exclude-policy work through the existing native exclude discussion rather than changing backup scope incidentally.

## Reproduction Assessment

No. Source inspection confirms the full-backup versus config-only path split, but there is not yet a high-confidence Windows current-main reproduction with a path longer than 260 characters, and the stated BSD tar cause does not match current source.

## Solution Assessment

Unclear. Replacing the backend with 7-Zip or adding broad workspace exclusions may be too wide until the node-tar Windows failure is reproduced; exclude-pattern policy is related to https://github.com/openclaw/openclaw/issues/40786 and https://github.com/openclaw/openclaw/pull/44288.
