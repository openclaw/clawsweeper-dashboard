## Summary

Keep open. Current main and the latest shipped tag still run full backups through the node-tar library over workspace directories by default, while `--only-config` avoids that path; the report is recent, has no merged Windows long-path fix, and still needs a Windows current-main reproduction because the BSD tar explanation does not match the implementation.

## What This Changes

The issue asks for `openclaw backup create` to succeed on Windows when workspace files include paths over 260 characters instead of hanging or exiting 255 while `--only-config` succeeds.

## Best Possible Solution

Reproduce on Windows current main with a long-path configured workspace, then fix the existing backup archiver or diagnostics; only add broad exclusions or a backend swap if owners choose that backup policy.

## Reproduction Assessment

No. The report has concrete Windows commands and source confirms the full-backup versus config-only path split, but there is not yet a high-confidence current-main Windows reproduction of the exit 255 failure with a path over 260 characters.

## Solution Assessment

Unclear. Replacing the backend with 7-Zip or excluding workspace directories may help this setup, but the narrower maintainable path is to reproduce the node-tar Windows failure first and then target that verified cause.
