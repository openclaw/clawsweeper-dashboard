## Summary

Keep open. This is a recent external bug report against a documented backup flow; current main still archives state/workspace trees by default through node-tar, and there is no current-main Windows long-path reproduction or fix provenance that would justify cleanup closure.

## What This Changes

The issue reports that full `openclaw backup create` fails on Windows with long workspace paths while `--only-config` succeeds.

## Best Possible Solution

Reproduce on Windows current main with a workspace file path longer than 260 characters, then fix the existing backup archiver/path handling or diagnostics while keeping broader exclude-pattern design in the existing backup-exclude issue/PR.

## Reproduction Assessment

No high-confidence current-main failure reproduction is available. Source inspection confirms the full-backup versus `--only-config` path split, but the reported Windows long-path hang/exit was not reproduced and the stated BSD tar cause does not match current node-tar usage.

## Solution Assessment

Unclear. Replacing the backend with 7-Zip or adding broad exclusions is premature until the current node-tar Windows failure is reproduced; a narrower path-handling or diagnostic fix may be safer.
