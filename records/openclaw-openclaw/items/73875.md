## Summary

Keep open. Current main still forwards generic ACP runtime config keys when the backend advertises `session/set_config_option` without a strict key list, and the issue is paired with open, unmerged PR #74048 using closing syntax.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to send only adapter-supported config options so Claude ACP sessions do not fail on unsupported generic keys.

## Best Possible Solution

Keep this issue open until PR #74048 or a replacement lands a capability-aware ACP runtime-control path that sends only advertised adapter config ids and keeps generic timeout handling out of strict backend config calls.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and the current source/dependency path statically reproduces the failing precondition: OpenClaw builds generic config keys, ACPX advertises the control without keys, and the Claude adapter rejects unknown ids.

## Solution Assessment

Yes. Filtering by advertised adapter support is the narrow maintainable direction because it avoids a Claude-specific core special-case; PR #74048 is the active implementation candidate but still needs review and merge.
