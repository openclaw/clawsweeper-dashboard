## Summary

Keep open. Current main still forwards generic ACP runtime config keys when the backend advertises set_config_option without strict configOptionKeys, and open PR #74048 is the unmerged closing implementation candidate for this issue.

## What This Changes

OpenClaw-managed ACP sessions should avoid sending generic runtime config keys such as timeout, thinking, and approval_policy to Claude ACP unless the adapter advertises support.

## Best Possible Solution

Land PR #74048 or a replacement that makes ACP runtime controls capability-aware, only sends advertised adapter config IDs, and keeps backend-specific mappings out of generic core assumptions.

## Reproduction Assessment

Yes. Source inspection on current main shows the failing precondition: generic runtime keys are built and sent when set_config_option exists without strict advertised keys, while the pinned Claude ACP adapter rejects unknown config IDs.

## Solution Assessment

Yes. Capability-aware filtering or skipping is the narrow maintainable fix because it avoids a Claude-specific core special case while respecting backend-advertised ACP contracts.
