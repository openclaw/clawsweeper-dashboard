## Summary

Keep open. Current main still forwards generic ACP config keys for backends that do not advertise a strict key list, and this issue is paired with open, unmerged PR #74048 using closing syntax.

## What This Changes

OpenClaw-managed Claude ACP sessions should avoid sending unsupported generic runtime config options such as `timeout`, `thinking`, and `approval_policy` to strict ACP adapters.

## Best Possible Solution

Keep the issue open until PR #74048 or a replacement lands a capability-aware ACP runtime-control path that sends only advertised adapter config IDs and keeps generic turn timeouts out of strict adapter config unless supported.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and current main has the static precondition: generic config keys are still produced and forwarded when `configOptionKeys` is absent or empty, while the pinned Claude adapter rejects unknown IDs.

## Solution Assessment

Yes. Filtering by advertised adapter support is the narrow maintainable direction because it avoids a Claude-specific core special case; the linked PR still needs normal review, tests, and landing before this issue can close.
