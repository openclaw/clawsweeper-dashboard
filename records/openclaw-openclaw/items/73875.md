## Summary

Keep open. Current main still builds and forwards generic ACP runtime config keys to ACP backends that do not advertise a strict key list, and open PR #74048 is the unmerged closing implementation candidate for this issue.

## What This Changes

Make OpenClaw-managed ACP sessions avoid sending unsupported generic runtime config options to Claude ACP adapters unless the adapter advertises support.

## Best Possible Solution

Review and land PR #74048, or an equivalent capability-aware fix that only applies adapter-advertised ACP config options and keeps backend-specific mappings out of generic core assumptions.

## Reproduction Assessment

Yes, from source inspection. Current main still builds timeout/thinking/approval_policy config pairs and forwards them when a backend exposes set_config_option without a strict advertised key list, matching the reported Claude ACP failure precondition.

## Solution Assessment

Yes. Capability-aware filtering or skipping is the narrow maintainable fix because it respects adapter-advertised ACP contracts without adding a Claude-specific core special case.
