## Summary

Keep open. Current main still builds generic ACP runtime config keys and forwards them when the backend advertises `session/set_config_option` without a strict `configOptionKeys` list, while open PR #74048 is the unmerged closing implementation candidate.

## What This Changes

OpenClaw-managed Claude ACP sessions should avoid sending unsupported generic runtime config keys such as `thinking`, `approval_policy`, and `timeout` to strict ACP adapters.

## Best Possible Solution

Land or replace PR #74048 with a capability-aware ACP runtime-control path that sends only advertised adapter config IDs and keeps generic OpenClaw runtime keys out of strict backend config calls.

## Reproduction Assessment

Yes. The report gives concrete live steps, and current main statically reproduces the failing precondition: generic keys are built, ACPX advertises the control without strict keys, and the pinned Claude adapter rejects unknown IDs.

## Solution Assessment

Yes as a direction. Filtering by advertised adapter support, or skipping config-option calls when support is unknown, is narrower and more maintainable than a Claude-specific core special case.
