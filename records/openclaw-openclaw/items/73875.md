## Summary

Keep open. Current main still generates generic ACP runtime config keys and forwards them to any backend with `session/set_config_option` when no strict advertised key list is present, which matches the Claude ACP failure mode in the report. The issue is also paired with open, unmerged PR #74048 using closing syntax, so the safe maintainer path is to review, land, or replace that PR rather than close this issue now.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to avoid sending unsupported generic runtime config options to Claude ACP adapters and only send advertised or otherwise supported ACP config options.

## Best Possible Solution

Keep this issue open until PR #74048 or an equivalent replacement lands. The end state should make ACP runtime controls capability-aware, send only adapter-supported config keys, and keep generic run timeouts on OpenClaw's own timeout path unless a backend explicitly advertises a compatible config option.

## Reproduction Assessment

Yes. The issue provides concrete live reproduction steps, and current main has the static precondition: OpenClaw builds `thinking`, `approval_policy`, and `timeout` config options and forwards them when no strict advertised key list is available.

## Solution Assessment

Yes. Filtering by adapter-advertised support is the narrow maintainable direction because it avoids a Claude-specific core special case; the linked PR or replacement still needs normal review and validation before this can close.
