## Summary

Keep open. Current main still forwards generic ACP runtime config keys when a backend supports `session/set_config_option` but does not advertise a strict key list, and open PR #74048 is already the unmerged closing implementation candidate.

## What This Changes

Prevent OpenClaw-managed ACP sessions from sending generic runtime config options such as `timeout`, `thinking`, and `approval_policy` to Claude ACP unless the adapter advertises support.

## Best Possible Solution

Keep this issue open until PR #74048 or a replacement lands a capability-aware ACP runtime-control path that sends only adapter-advertised config option ids and maps generic OpenClaw concepts without a Claude-specific core special case.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and current source statically reproduces the failing precondition: generic runtime options are built, missing key lists remain permissive, and Claude ACP rejects unknown config ids.

## Solution Assessment

Yes. Filtering by advertised adapter support is the narrow maintainable direction; PR #74048 is the active implementation candidate but is still open and unmerged.
