## Summary

Keep open. Current main still builds generic ACP runtime config keys and forwards them whenever `session/set_config_option` is available and no strict `configOptionKeys` list is advertised; PR #74048 is the open, unmerged implementation candidate with closing syntax.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to stop sending unsupported generic runtime config options to Claude ACP adapters and apply only target-supported ACP config options.

## Best Possible Solution

Keep this open until PR #74048 or a replacement lands; the end state should be a capability-aware ACP runtime-control contract that sends only explicitly supported config keys and keeps generic run timeouts on OpenClaw's own timeout path unless a backend advertises a compatible option.

## Reproduction Assessment

Yes. The issue provides a concrete live reproduction path, and current main has the static precondition: generic ACP runtime keys are built and forwarded when `session/set_config_option` exists and no strict key list is advertised.

## Solution Assessment

Yes. Filtering by adapter-advertised support is the narrow maintainable direction because it avoids a Claude-specific core special-case; the linked PR still needs normal review and validation before the issue can close.
