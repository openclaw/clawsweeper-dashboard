## Summary

Keep open. Current main still constructs and forwards generic ACP runtime config keys when no strict `configOptionKeys` list is advertised, and open PR #74048 is the unmerged closing implementation candidate for this exact bug.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to send only adapter-supported config options so Claude ACP does not fail on unsupported generic runtime keys.

## Best Possible Solution

Land PR #74048 or an equivalent capability-aware ACP runtime-control change that skips unadvertised or unsupported config IDs while preserving adapter-specific mappings.

## Reproduction Assessment

Yes. The report gives live steps, and current-main inspection reproduces the failing precondition: OpenClaw builds `thinking`, `approval_policy`, and `timeout` config pairs and sends them when `configOptionKeys` is absent.

## Solution Assessment

Yes. Filtering by advertised ACP config support, or skipping config updates when support is unknown, is the narrow maintainable fix; a Claude-specific core special case would be weaker.
