## Summary

Keep open. Current main still generates generic ACP runtime config keys and forwards them when a backend exposes set_config_option without a strict advertised key list; PR #74048 is the open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to avoid sending unsupported generic runtime config options such as timeout, thinking, and approval_policy to Claude ACP unless the adapter advertises compatible support.

## Best Possible Solution

Land or replace the linked PR with a capability-aware ACP runtime-control change that sends only explicitly supported config keys, maps OpenClaw concepts through advertised adapter-specific keys, and keeps generic run timeouts on OpenClaw's own timeout path unless a backend advertises a compatible option.

## Reproduction Assessment

Yes. The issue gives a concrete live path, and current main has the static precondition: generic runtime keys are generated and forwarded whenever set_config_option exists and no strict configOptionKeys list is advertised.

## Solution Assessment

Yes. Filtering by adapter-advertised support is the narrow maintainable direction because it avoids a Claude-specific core special case while preserving explicit backend capabilities.
