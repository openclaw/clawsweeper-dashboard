## Summary

Keep open. Current main still builds generic ACP runtime config keys and forwards them when a runtime advertises `session/set_config_option` without a strict key list; the pinned ACPX/Claude adapter path matches the failure described, and open PR #74048 is the unmerged closing implementation candidate.

## What This Changes

OpenClaw-managed Claude ACP sessions should avoid sending generic runtime config options that the target ACP adapter does not advertise or support.

## Best Possible Solution

Review, revise, and land #74048 or a replacement that sends only supported adapter config IDs without redefining permissive backend semantics, with focused regression tests and the required changelog entry.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and current source plus pinned dependency source statically reproduces the failing precondition: OpenClaw builds generic keys, ACPX advertises config-option control without keys, and the Claude adapter rejects unknown config IDs.

## Solution Assessment

Yes on the direction, but unclear on the current open PR's exact implementation. A capability-aware fix is the narrow maintainable path; it should avoid Claude-specific core special cases while preserving the documented runtime capability contract.
