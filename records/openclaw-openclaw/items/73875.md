## Summary

Keep open. Current `main` still has the reported ACP runtime-control behavior, and open PR #74048 is already the unmerged implementation candidate using closing syntax, so this issue should stay open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to send only adapter-supported ACP config options so Claude ACP does not fail on generic keys such as `timeout`, `thinking`, or `approval_policy`.

## Best Possible Solution

Keep the issue open until PR #74048 or a replacement lands with capability-aware ACP config filtering, adapter-specific mapping where a real contract exists, and regression coverage for missing or strict config key lists.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and current `main` statically reproduces the failure precondition: ACPX advertises `session/set_config_option` without `configOptionKeys`, OpenClaw forwards generic keys, and the pinned Claude adapter rejects unknown config ids.

## Solution Assessment

Yes. Filtering by advertised adapter support is the narrow maintainable direction because it avoids a Claude-specific core special-case; the linked PR still needs normal review and validation before this issue can close.
