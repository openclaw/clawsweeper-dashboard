## Summary

Keep open: current main still computes the untrusted inbound user context but omits it from the bare reset/bootstrap prompt branch, so sender identity is unavailable to the model on that first `/new` or `/reset` turn.

## What This Changes

Bare `/new` and `/reset` should include existing Conversation info and Sender untrusted metadata in the user prompt.

## Best Possible Solution

Make reset/bootstrap prompt composition match normal turns by adding the existing user-role metadata prefix, with regression tests that keep sender fields out of the trusted system prompt and preserve transcript markers.

## Reproduction Assessment

Yes, from source inspection: a model-invoking bare `/new` or `/reset` reaches the `isBareSessionReset` branch, where `followupRun.prompt` is built without `inboundUserContext`. I did not run the test path because this review is read-only.

## Solution Assessment

Yes. Adding `inboundUserContext` to the bare reset prompt array is the narrowest maintainable fix because sender fields already belong in untrusted user-role context and the system prompt intentionally excludes them.
