## Summary

Keep open: current main still computes inbound user-context metadata but drops it from the bare reset/bootstrap prompt branch, while the trusted system prompt intentionally excludes sender fields.

## What This Changes

Bare `/new` and `/reset` session bootstrap prompts should include sender identity metadata in the user-role prompt.

## Best Possible Solution

Prepend `inboundUserContext` to model-invoking bare reset/bootstrap prompts, add regression coverage for bare `/new` and `/reset`, and keep sender fields out of the trusted system prompt.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a focused `runPreparedReply` case with a bare `/new` or `/reset` bootstrap and sender fields will call `runReplyAgent` with `followupRun.prompt` missing the mocked inbound `Conversation info` and `Sender` blocks.

## Solution Assessment

Yes. Adding `inboundUserContext` to the existing bare reset prompt array is the narrowest maintainable fix, with tests guarding both reset commands and the existing transcript-marker behavior.
