## Summary

Keep open: current main still computes inbound user context but omits it from the bare reset/bootstrap prompt branch, so sender metadata reaches normal prompts but not model-invoking bare `/new` or `/reset` startup turns.

## What This Changes

The issue requests that sender identity untrusted user-context blocks be included in bare `/new` and `/reset` model prompts.

## Best Possible Solution

Prepend `inboundUserContext` to the bare reset prompt body in `runPreparedReply`, add regression coverage for bare `/new` and `/reset` prompt contents, and preserve the trusted system prompt split.

## Reproduction Assessment

Yes, from source: a `runPreparedReply` case with `resetTriggered`, `isNewSession`, bare `/new` or `/reset`, and sender fields will build `inboundUserContext` but pass a bare-reset prompt array that omits it into `followupRun.prompt`. I did not run a live or unit reproduction because this review was read-only.

## Solution Assessment

Yes. Adding `inboundUserContext` to the existing bare-reset branch is the narrowest maintainable fix, while keeping sender fields out of `buildInboundMetaSystemPrompt` and preserving ordinary command acknowledgement behavior.
