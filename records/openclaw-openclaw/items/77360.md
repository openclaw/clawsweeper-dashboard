## Summary

Keep open: current main still builds inbound user-context metadata but omits it from the bare reset/bootstrap model prompt branch, and existing coverage does not assert sender blocks for that path.

## What This Changes

Bare `/new` and `/reset` bootstrap turns should include the same untrusted conversation and sender metadata in the model user prompt as normal inbound messages.

## Best Possible Solution

Prepend user-role inbound context to bare reset/bootstrap prompts, add regression coverage for both reset commands, and keep sender fields out of the trusted system prompt.

## Reproduction Assessment

Yes, from source: a focused `runPreparedReply` case with `resetTriggered`, a bare `/new` or `/reset`, and sender fields would reach `runReplyAgent` without `Conversation info` or `Sender` in `followupRun.prompt`. I did not run it because this review was read-only.

## Solution Assessment

Yes. Adding `inboundUserContext` to the existing bare reset prompt array is the narrowest maintainable fix, provided tests preserve the hidden transcript marker and existing bare command acknowledgement behavior.
