## Summary

Keep open. Current main still has the reported `.openclaw` symlink rejection, and this PR is useful narrow work toward #72572, but the patch is not merge-ready because it still swallows trusted-symlink resolution `ENOENT` and lacks the required user-facing changelog entry; maintainers also need to choose between this narrow path and the broader draft #72713.

## What This Changes

The PR changes exec approvals path validation to allow a vetted immediate `.openclaw` symlink under the trusted home directory and adds regression tests for accepted and unsafe symlink layouts.

## Best Possible Solution

Land one maintained fix for #72572 that allows a trusted first-level POSIX `.openclaw` symlink after ownership and permission validation, keeps deeper traversal and symlinked destination files rejected, documents the user-facing behavior in the changelog, and either closes or explicitly supersedes the competing draft path.

## Reproduction Assessment

Yes. The source-level path is high confidence: with `~/.openclaw` as a symlink under a real home, `openclaw exec-policy preset yolo` writes through `saveExecApprovals()`, whose current-main directory validation rejects the `.openclaw` component before the approvals file can be written.

## Solution Assessment

No, not as written. The one-hop trusted-symlink direction is narrow and maintainable for the reported GNU Stow/chezmoi layout, but symlink resolution errors must not be swallowed by the missing-component catch and the user-facing fix needs a changelog entry; maintainers should also decide whether #72713 is the preferred broader implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Rethrow trusted symlink resolution errors:** `src/infra/exec-approvals.ts:282`
  - body: `fs.realpathSync(current)` now runs inside the same `try` block that intentionally swallows `ENOENT` from `lstatSync` for missing path components. If `.openclaw` is a dangling symlink, trusted-target validation is skipped and the command fails later with a lower-level mkdir/stat error. Scope the `ENOENT` swallow to `lstatSync`, or rethrow once the code has identified a symlink.
