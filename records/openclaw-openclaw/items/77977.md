## Summary

Keep open: current main still accepts gateway connect-frame auth fields as unbounded schema strings, and the linked closing PR remains open rather than merged.

## What This Changes

Request to bound gateway connect-frame `auth.*` fields at the protocol schema so oversized auth payloads are rejected before handshake/auth processing.

## Best Possible Solution

Review and land a rebased https://github.com/openclaw/openclaw/pull/77538 that bounds all current `ConnectParamsSchema.auth` fields and documents the public wire limits.

## Reproduction Assessment

Yes. Source inspection shows oversized sub-64 KiB `auth.*` strings pass `ConnectParamsSchema`, then flow through the pre-auth connect validator into shared-secret or bootstrap/device-token auth paths.

## Solution Assessment

Yes in direction, but not landed. Schema-level `maxLength` rejection is the right boundary; the open PR still needs current-main coverage for `approvalRuntimeToken` and approval of the public cap values.
