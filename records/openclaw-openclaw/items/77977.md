## Summary

## What This Changes

The issue requests protocol-schema bounds for gateway connect-frame `auth.*` fields so oversized or malformed values are rejected before entering auth handling.

## Best Possible Solution

Review and land the linked PR after gateway/security owners accept or adjust the auth field limits and document the protocol contract; if maintainers reject the cap direction, close both the issue and candidate PR with that decision.

## Solution Assessment

Yes, with a maintainer decision still open. Bounding `ConnectParamsSchema` is the narrowest boundary-layer fix, but gateway/security owners need to approve the supported maximum sizes before merge.
