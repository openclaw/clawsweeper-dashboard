## Summary

Keep open, but this PR is not merge-ready: current main still needs a secret-leak mitigation, while this branch strips too broadly, conflicts with the intended skill-env delivery path, is dirty against current main, and carries unrelated runtime changes.

## What This Changes

The branch adds active skill-env key stripping to generic host exec child environments, sanitizer coverage, an exec-path test, and unrelated channel-add and cron-store changes.

## Best Possible Solution

Land a maintainer-approved scoped credential boundary or replacement that strips skill SecretRef/apiKey values from unrelated generic exec children while preserving intended skill-owned env delivery, after rebasing and removing unrelated changes.

## Reproduction Assessment

Yes, source-backed: current main resolves active skill apiKey/env values into process.env and generic host exec builds child envs from that inherited env without active-skill stripping. The over-broad PR behavior is also source-reproducible because getActiveSkillEnvKeys tracks every injected skill env key.

## Solution Assessment

No. The PR is a plausible mitigation for the leak, but the strip-all boundary is too broad until maintainers decide how intended skill CLI credentials should reach their own commands without leaking to unrelated generic exec children.
