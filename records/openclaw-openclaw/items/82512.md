## Summary

Keep open: the intended generic exec secret-isolation fix is source-backed, but the current branch is not mergeable because it also loosens persisted cron payload validation outside the PR scope.

## What This Changes

The branch adds active-skill env-key stripping to generic host exec env sanitization and tests it, while also loosening persisted cron payload validation and adjusting cron/pairing tests.

## Best Possible Solution

Keep the narrow exec sanitizer change and its focused tests, remove the unrelated cron/pairing churn, then land only after maintainer/security review reconciles the generic-exec isolation policy with the skill CLI env-inheritance request.

## Reproduction Assessment

Yes, source-backed. Current main injects active skill env keys into process.env, generic host exec builds child envs from that process env, and the sanitizer has no active-skill strip input; the cron regression is also visible directly in the PR diff.

## Solution Assessment

Mostly yes for the intended security fix: reusing the active skill env registry at the host exec sanitizer boundary matches the existing ACP precedent. As submitted, this is not the best mergeable patch because unrelated cron validation and pairing test changes should be removed first.
