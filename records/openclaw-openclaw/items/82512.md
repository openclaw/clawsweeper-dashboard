The PR strips active skill-injected env keys from inherited host exec environments, adds sanitizer and exec coverage, and also changes cron persisted-shape validation plus plugin test-env helper imports.

## Best Possible Solution

Land a narrow exec sanitizer fix that strips active skill env keys from inherited host exec env while preserving current persisted cron validation.

## Reproduction Assessment

Yes, source-backed: current main applies skill apiKey values to process.env and generic host exec sanitizes that inherited env without the active-skill strip set. The PR finding is also source-backed because the diff removes the non-empty cron message guard used before isolated cron execution.

## Solution Assessment

Mostly yes for the intended exec fix, because reusing the active skill env registry at the host exec sanitizer boundary matches the ACP spawn precedent. No as submitted, because the unrelated cron validation change can admit malformed blank agent-turn payloads.
