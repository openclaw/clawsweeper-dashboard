## Summary

Keep open: this PR targets a real skill-secret leak, but the current patch is not merge-ready because it strips too broad a key set, is stale/conflicting against current `main`, and bundles unrelated cron/channel behavior into a security-sensitive fix.

## What This Changes

The PR strips active skill-injected environment keys from generic host `exec` child environments, adds sanitizer and exec-path tests, and also changes channel-add and cron-store behavior.

## Best Possible Solution

Land a narrow, maintainer-approved credential boundary that strips skill SecretRef/apiKey values from unrelated generic exec children while preserving intended skill-owned environment delivery.

## Reproduction Assessment

Yes, source-backed: current main resolves active skill credentials into `process.env`, and generic host exec derives child envs from `process.env` without an active-skill strip. The related availability issue is also source-backed because the active-key registry includes intended configured skill env values.

## Solution Assessment

No. The PR is a plausible mitigation for the leak, but the best fix needs a narrower or explicitly approved boundary that does not also erase intended skill-owned CLI credentials.
