## Summary

Keep open: current main still collapses the aggregate status default to one model, and this PR targets the right bug but is stale against the current status overview renderer and missing the required changelog entry.

## What This Changes

The PR computes status session defaults across configured agents, adds optional variation metadata to CLI/TUI status types, renders `varies by agent`, and adds status-summary regression tests.

## Best Possible Solution

Rebase or replace the branch on current main, compute aggregate session defaults by iterating configured agents through the lazy status runtime boundary, propagate the variation marker through JSON/CLI/TUI output, and add focused tests plus a single-line changelog entry.

## Reproduction Assessment

Yes. A config with multiple `agents.list[]` entries whose effective primary models differ exercises the current aggregate `getStatusSummary()` path, while the existing per-session path already has agent-aware resolution.

## Solution Assessment

No, not as currently proposed. The aggregation direction is right, but the branch updates an older inline CLI renderer and needs to target the current overview helper while preserving the lazy runtime boundary.
