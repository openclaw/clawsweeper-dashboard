## Summary

Keep open: current main still has the misleading status default-model summary, but this PR needs a rebase to current status runtime/overview boundaries and still lacks external after-fix real behavior proof.

## What This Changes

The PR adds a per-agent variation marker to status session defaults, renders "varies by agent" in CLI/TUI output, and adds regression tests for divergent and uniform agent default models.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through status.summary.runtime.ts, updates shared CLI/TUI/JSON summary types and renderers, and includes focused regressions plus redacted real status output.

## Reproduction Assessment

Yes. Source inspection shows getStatusSummary currently resolves one default without agentId while the status runtime and docs support agents.list[].model overrides; I did not run a live CLI/TUI reproduction in this read-only pass.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch needs to move aggregation behind the current status runtime surface and update the shared Sessions overview renderer before merge.
