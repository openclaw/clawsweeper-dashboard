## Summary

Keep open: current main still has a source-visible path where the status overview/TUI can present one global session default even though agents can resolve different default models, but this external PR needs real behavior proof and a rebase/refit after later status refactors before merge.

## What This Changes

This PR adds a per-agent default-model variance marker to status summaries, carries it through CLI/TUI rendering, and adds regression coverage for divergent and uniform agent defaults.

## Best Possible Solution

Rework the patch onto the current status summary and overview helper structure so CLI JSON, CLI overview, and TUI distinguish uniform versus divergent per-agent defaults without triggering provider or plugin discovery.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure two agents with different effective default models, then inspect `openclaw status`, `status --json`, or the TUI summary. I did not run it because this review is read-only, but current main still computes and displays a single global `sessions.defaults.model`.

## Solution Assessment

Yes in direction, but not merge-ready as-is. Per-agent effective default resolution is the right narrow fix, but it needs to be rebased into the current `status.summary.runtime.ts` and `status-overview-values.ts` structure and backed by real CLI/TUI proof.
