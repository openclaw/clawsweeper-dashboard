## Summary

## What This Changes

The branch aggregates per-agent effective default models for status summaries, adds a `variesByAgent` field, renders the variation in CLI/TUI output, and adds regression tests.

## Best Possible Solution

Land a rebased implementation that aggregates effective defaults through `status.summary.runtime.ts`, updates JSON/CLI/TUI/shared overview handling, and includes focused tests plus redacted real status proof.

## Reproduction Assessment

Yes, via source inspection: current main resolves the aggregate default once without `agentId`, while the runtime resolver and docs support per-agent model overrides. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

No, not as currently proposed. The direction is right, but the diff targets an older inline renderer and bypasses the current lazy status runtime boundary; the safer fix is a rebased runtime-surface aggregation plus shared renderer update.
