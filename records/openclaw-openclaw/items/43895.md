## What This Changes

## Best Possible Solution

Land a rebased fix that computes per-agent effective defaults inside the status runtime surface, propagates the variation marker through JSON/plain/TUI shared renderers, preserves read-only startup, and includes focused tests plus redacted real output.

## Reproduction Assessment

## Solution Assessment

No. The intended behavior is right, but the branch should be rebased through status.summary.runtime.ts and status-overview-values.ts instead of editing the old inline renderer and importing model helpers directly.
