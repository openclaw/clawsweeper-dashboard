## Summary

Keep open: the aggregate status default-model bug is still source-reproducible on current main, but this PR branch is stale against the current status runtime/overview split and lacks required real behavior proof before merge.

## What This Changes

The PR adds a status-summary marker for multi-agent default-model variation, renders it in CLI/TUI status output, and adds regression tests for divergent and uniform agent defaults.

## Best Possible Solution

Land a rebased fix that computes per-agent effective default variation behind status.summary.runtime.ts, carries a backward-compatible marker through JSON/TUI types, updates the shared status overview renderer, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, via source inspection: current main resolves sessions.defaults.model once without agent context while the docs and runtime helper support per-agent model overrides. I did not run a live CLI/TUI reproduction because this review is read-only.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch targets an older inline renderer and bypasses the current lazy status runtime boundary; a rebased runtime-surface aggregation plus shared renderer update is the safer fix.
