## Summary

Keep open: the reported status display bug still exists on current main, but this PR is not merge-ready because it needs a rebase for the current shared status renderer/runtime boundary and still lacks external real behavior proof.

## What This Changes

The PR adds a `variesByAgent` marker to status defaults, renders `varies by agent` in CLI/TUI status summaries, and adds regressions for divergent and uniform per-agent model defaults.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through `status.summary.runtime.ts`, updates shared JSON/CLI/TUI status types and renderers, and includes focused regressions plus redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json`, plain `openclaw status`, or the TUI status summary. Current main resolves one default without agent context while the runtime resolver supports an agent id.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the patch must be rebased onto the current shared overview/runtime surface and keep default aggregation behind `status.summary.runtime.ts`.
