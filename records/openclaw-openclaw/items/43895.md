## What This Changes

The PR adds a status-summary marker for divergent per-agent default models, updates CLI/TUI display types, and adds regression tests.

## Best Possible Solution

Land a rebased fix that computes per-agent effective status defaults through `status.summary.runtime.ts`, propagates a variation marker through JSON/CLI/TUI/shared overview contracts, and includes focused tests plus redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json`, plain `openclaw status`, or the TUI summary. Current main resolves one aggregate default without passing an agent id.

## Solution Assessment

No. The direction is right, but the current branch targets an older inline renderer and imports model/context helpers outside the status runtime surface; the safer fix is a rebase through `status.summary.runtime.ts` and the shared overview renderer.
