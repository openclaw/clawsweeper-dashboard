## Summary

Keep open: current main still cannot express divergent per-agent default models in the status summary, and no canonical merged replacement covers that status-display gap. The branch is still useful, but the live diff is stale against the newer status runtime/overview split and lacks required real behavior proof.

## What This Changes

The PR adds a per-agent variation marker for status session defaults, renders `varies by agent` in CLI/TUI status output, and adds regression tests for divergent and uniform agent default models.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through `status.summary.runtime.ts`, updates shared CLI/TUI/JSON summary types and renderers, and includes focused regressions plus redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json` or the TUI status summary. Current main resolves one default without agent context, while the runtime resolver supports an agent id.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the patch must be rebased onto the current shared overview/runtime surface and should keep aggregation behind `status.summary.runtime.ts`.
