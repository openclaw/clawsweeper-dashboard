## Summary

Keep this PR open. https://github.com/openclaw/openclaw/pull/43895 targets a real `openclaw status` / `status --json` / TUI display bug, but the branch is not merge-ready because current `main` now renders the CLI Sessions row through the overview helper that this patch does not update, and the user-facing fix also needs a changelog entry.

## What This Changes

The PR derives the status session-default model summary from per-agent effective defaults, adds an optional variation marker to CLI/TUI status types, renders `varies by agent`, and adds status-summary regressions.

## Best Possible Solution

Rebase the branch onto current main, compute `sessions.defaults` from each configured agent's effective default through the lazy `status.summary.runtime` boundary, expose a backward-compatible variation marker in CLI JSON and TUI types, update `status-overview-values.ts` plus the TUI formatter, and cover divergent plus uniform defaults in focused tests with a changelog entry crediting the contributor.

## Reproduction Assessment

Yes. A config with multiple `agents.list[]` entries whose model primaries resolve to different models exercises the current aggregate resolver, while the existing per-session path already has agent-aware resolution.

## Solution Assessment

No, not as currently proposed. The aggregation direction is right, but the branch updates an older inline CLI renderer and needs to be rebased onto the current overview helper while preserving the lazy runtime boundary.
