Keep open: the status bug is valid, but the branch is not merge-ready because current main has since moved plain-status rendering and status model resolution behind shared/lazy runtime helpers, and the PR still lacks after-fix real behavior proof.

## What This Changes

The branch adds a `variesByAgent` marker to status session defaults, renders `varies by agent` in CLI/TUI status output, and adds regression tests for divergent and uniform per-agent model defaults.

## Best Possible Solution

Land a rebased fix that computes per-agent effective defaults through `status.summary.runtime.ts`, propagates the variation marker through JSON/plain/TUI shared renderers, preserves read-only status startup, and includes focused tests plus redacted real output.

## Reproduction Assessment

Yes by source inspection. Configure multiple agents with different `agents.list[].model` values and inspect `openclaw status --json`, plain status, or the TUI summary; current main computes the summary default once without an agent id.

## Solution Assessment

No. The intended behavior is right, but the branch should be rebased through `status.summary.runtime.ts` and `status-overview-values.ts` instead of editing the old inline renderer and importing model helpers directly.
