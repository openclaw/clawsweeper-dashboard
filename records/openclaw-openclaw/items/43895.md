## Summary

Keep open: current main still has the status-display bug, but this PR cannot merge as-is because it is stale against the current shared status renderer/runtime boundary and lacks required real behavior proof.

## What This Changes

The PR adds a `variesByAgent` status-default marker, renders `varies by agent` for divergent agent defaults, and adds regression coverage for uniform and divergent agent models.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through `status.summary.runtime.ts`, updates JSON/CLI/TUI/shared overview types and renderers, adds focused regressions, and includes redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json`, plain `openclaw status`, or the TUI status summary. Current main resolves one aggregate default without agent context while the status runtime resolver already supports an agent id.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch must be rebased onto the shared overview renderer and keep default aggregation behind the status runtime surface.
