## Summary

Keep open. The status display bug remains source-reproducible on current main and the PR direction is useful, but the branch is conflicting and the proposed patch misses the current shared status overview/runtime boundaries; contributor real behavior proof is also still missing.

## What This Changes

The PR adds a `variesByAgent` status-default marker, renders `varies by agent` in CLI/TUI status summaries, and adds regression tests for divergent and uniform agent defaults.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through `status.summary.runtime.ts`, updates shared CLI/TUI/JSON status types and renderers, and includes focused regressions plus redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json` or the TUI status summary. Current main resolves one default without agent context while the runtime resolver supports an agent id.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the patch must be rebased onto the current shared overview/runtime surface and keep default aggregation behind `status.summary.runtime.ts`.
