## Summary

Keep open: the reported display bug is still source-reproducible on current main, but this PR is stale against the current status overview/runtime structure, misses the shared human-status renderer, and still lacks required after-fix real behavior proof.

## What This Changes

The PR aggregates per-agent effective default models in status summaries, adds an optional `variesByAgent` flag for CLI/TUI rendering, and adds regression tests for divergent and uniform defaults.

## Best Possible Solution

Land a rebased fix that aggregates per-agent effective defaults through the status summary runtime surface, carries a backward-compatible variation marker through JSON/TUI types, updates the shared overview renderer, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, via source inspection: current main resolves `sessions.defaults.model` once without an agent context while docs and runtime helpers support per-agent model overrides. I did not run a live CLI reproduction because this review is read-only.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch targets an older inline renderer and bypasses the current lazy status runtime boundary; the safer solution is a rebased runtime-surface aggregation plus shared renderer update.
