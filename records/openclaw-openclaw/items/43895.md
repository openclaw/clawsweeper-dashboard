## What This Changes

The PR adds a `variesByAgent` marker to status session defaults so `openclaw status`, `status --json`, and the TUI can avoid showing one global default model when configured agents resolve to different defaults.

## Best Possible Solution

Land a rebased fix that computes per-agent effective defaults inside the status runtime surface, propagates the variation marker through JSON/plain/TUI shared renderers, keeps status startup read-only, and includes focused tests plus redacted real output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple agents with different `agents.list[].model` values and inspect `openclaw status --json`, plain status, or the TUI summary; current main computes `sessions.defaults` once without an agent id.

## Solution Assessment

No. The intended behavior is right, but the branch should be rebased through `status.summary.runtime.ts` and `status-overview-values.ts` instead of editing the old inline renderer and importing model helpers directly.
