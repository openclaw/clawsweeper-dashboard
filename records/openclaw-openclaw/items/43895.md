## Summary

Keep open: the underlying status display bug is still present on current main, but this PR is not ready because it is stale against the current status runtime/overview structure, has two concrete rebase issues, and lacks required real behavior proof.

## What This Changes

The PR adds a per-agent variation marker to status summary defaults, renders "varies by agent" in CLI/TUI status, and adds regressions for divergent and uniform agent default models.

## Best Possible Solution

Land a rebased fix that computes per-agent effective default variation behind status.summary.runtime.ts, carries a backward-compatible marker through JSON/TUI types, updates the shared status overview renderer, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves sessions.defaults.model once without agent context, while the runtime helper and docs support per-agent model overrides; I did not run a live CLI/TUI reproduction in this read-only review.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch targets an older inline renderer and bypasses the current lazy status runtime boundary, so a rebased runtime-surface aggregation plus shared renderer update is safer.
