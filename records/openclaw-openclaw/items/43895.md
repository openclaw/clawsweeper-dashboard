## Summary

Keep open: the reported status bug still exists on current main, but this PR is not mergeable as-is because it is stale/conflicting, misses the current shared status overview/runtime boundary, and still lacks external real behavior proof.

## What This Changes

The PR adds a per-agent variation marker for status session defaults, renders `varies by agent` in CLI/TUI status output, and adds regression tests for divergent and uniform effective agent default models.

## Best Possible Solution

Land a rebased fix that aggregates effective per-agent defaults through `status.summary.runtime.ts`, updates shared CLI/TUI/JSON summary types and renderers, and includes focused regressions plus redacted real status output.

## Reproduction Assessment

Yes, by source inspection: configure multiple agents with different `agents.list[].model` defaults, then inspect `openclaw status --json` or the TUI status summary. Current main resolves one default without agent context while the runtime resolver supports an agent id.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the patch must be rebased onto the current shared overview/runtime surface and keep default aggregation behind `status.summary.runtime.ts`.
