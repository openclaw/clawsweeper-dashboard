## Summary

Keep open. The PR is aimed at a real status display bug that current main still has, but the branch is stale against the current status refactor, has two merge-blocking implementation gaps, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `variesByAgent` session-default state for status summaries, renders `varies by agent` in CLI/TUI status output, and adds tests for divergent and uniform per-agent default models.

## Best Possible Solution

Implement the per-agent effective default aggregation on the current status summary runtime, carry the `varies by agent` state through JSON/CLI/TUI overview helpers, and add focused tests plus redacted real status output proof.

## Reproduction Assessment

Yes. Source inspection shows current main computes `sessions.defaults` from one global default model while the current CLI/TUI summary renderers display that value; a multi-agent config with divergent `agents.list[*].model` values is the high-confidence reproduction path.

## Solution Assessment

No. The intended fix direction is right, but this branch is not the best merge shape until it is rebased onto the current status runtime/overview helper boundaries and backed by real behavior proof.
