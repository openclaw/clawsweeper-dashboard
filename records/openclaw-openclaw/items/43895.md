## Summary

Keep open: the underlying status display bug is valid and still present on current main, but this branch is stale against the current status renderer/runtime split and needs real behavior proof before merge.

## What This Changes

The PR computes status session defaults across configured agents, adds variation metadata, renders `varies by agent` in CLI/TUI status output, and adds regression coverage.

## Best Possible Solution

Land a rebased repair that aggregates effective defaults per configured agent through the current lazy status runtime surface, propagates the variation state through JSON/CLI/TUI including the shared overview helper, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection against current main shows the aggregate default path resolves one model without `agentId`, while docs and the runtime resolver support per-agent model overrides.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch updates an older inline renderer and static summary imports instead of the current shared overview helper and lazy runtime boundary.
