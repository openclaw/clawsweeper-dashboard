## Summary

Keep this PR open. It targets a real `openclaw status`, `status --json`, and TUI display bug, but the branch is not merge-ready because current `main` now renders the CLI Sessions row through an overview helper the patch does not update, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR derives status session-default model display from per-agent effective defaults, adds an optional variation marker for CLI/TUI status types, renders `varies by agent`, and adds status-summary regression tests.

## Best Possible Solution

Rebase or replace the branch on current main, compute `sessions.defaults` across configured agents through the lazy status runtime boundary, update CLI overview and TUI rendering, and add focused tests plus a single-line changelog crediting the contributor.

## Reproduction Assessment

Yes. A config with multiple `agents.list[]` entries whose effective primary models differ exercises the current aggregate resolver, while the existing per-session path already has agent-aware resolution.

## Solution Assessment

No, not as currently proposed. The aggregation direction is right, but the branch updates an older inline CLI renderer and needs to target the current overview helper while preserving the lazy runtime boundary.
