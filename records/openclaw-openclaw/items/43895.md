## Summary

Keep open: current main still has the misleading status-default behavior, but this PR cannot merge as-is because it is stale against the current status overview/runtime layout and still lacks after-fix real behavior proof.

## What This Changes

The PR aggregates per-agent effective default models in status summaries, adds an optional variation marker, updates CLI/TUI rendering, and adds regression tests for divergent and uniform defaults.

## Best Possible Solution

Land a rebased fix that computes per-agent effective default variation behind `status.summary.runtime.ts`, carries a backward-compatible marker through JSON/TUI types, updates the shared overview renderer, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, via source inspection: current main resolves `sessions.defaults.model` once without an agent context while the runtime helper and docs support per-agent model overrides. I did not run a live CLI reproduction because this review is read-only.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch targets an older inline renderer and bypasses the current lazy status runtime boundary; a rebased runtime-surface aggregation plus shared renderer update is the safer fix.
