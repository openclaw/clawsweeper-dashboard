## Summary

Keep this PR open. Current main still has the reported status-summary bug, but the branch is stale against current main and misses the current overview renderer, so it is not mergeable as-is.

## What This Changes

The PR derives the status session-default model summary from each configured agent's effective default, adds an optional variation marker through CLI/TUI summary types, renders "varies by agent", and adds status-summary regression tests.

## Best Possible Solution

Rebase the branch onto current main, compute sessions.defaults from each configured agent's effective default through the lazy status.summary.runtime boundary, expose a backward-compatible variation marker in CLI JSON and TUI types, update status-overview-values.ts and the TUI summary renderer, and cover divergent plus uniform defaults in summary and overview tests.

## Reproduction Assessment

Yes. A config with multiple agents whose agents.list[].model primaries resolve to different models exercises the current no-agentId aggregate resolver, while the per-session path already has agent-aware resolution.

## Solution Assessment

No, not as currently proposed. The aggregation direction is right, but the branch updates an older inline status renderer and must be rebased to update the current overview helper while preserving the lazy runtime boundary.
