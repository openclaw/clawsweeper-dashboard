## Label Justifications

- `P2`: This is a normal-priority plugin SDK capability change with limited surface area, but it needs careful boundary review before merge.
- `merge-risk: 🚨 security-boundary`: The PR broadens an existing plugin trust flag from raw conversation access into active-session message and attachment delivery.
- `merge-risk: 🚨 message-delivery`: Merging this PR changes which external plugins can deliver messages or presentations into an active channel/thread.
- `merge-risk: 🚨 compatibility`: Existing operator configs that already grant `allowConversationAccess` could gain a new send effect after a plugin manifest update.

## Summary

Keep open: the seam is plausible, but the patch is not merge-ready because it reuses `hooks.allowConversationAccess` as active-session send authority and the supplied proof does not exercise successful real session presentation delivery.

## What This Changes

The PR adds a `contracts.sessionAttachments` manifest contract and lets trusted external plugins send active-session attachments, text, or `MessagePresentation` payloads through the session attachment hook.

## Best Possible Solution

Keep the manifest-declared capability, but require an explicit operator send grant or an official-install-only first phase, with docs, tests, and real delivery proof for the chosen boundary.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR passes `hookPolicy.allowConversationAccess` into `sendPluginSessionAttachment`, where it authorizes active-session sends. No current-main bug reproduction applies because this is an additive feature PR.

## Solution Assessment

No, not as written. The narrow maintainable path is to avoid reusing a read-oriented hook grant for sends, or to make that broader trust decision explicit with docs, upgrade tests, and real transport proof.

Full review comments:

- **[P1] Require a separate grant for active-session sends:** `src/plugins/host-hook-attachments.ts:214`
  - body: `allowConversationAccess` is currently documented as letting trusted non-bundled plugins read raw conversation content from typed hooks, but this condition also turns it into permission to send active-session text, attachments, and presentations.
