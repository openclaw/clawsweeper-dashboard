## What This Changes

The issue asks to address three async interleaving races in command queue reset, global hook runner initialization, and plugin binding approval persistence, with the main unmitigated bug being disk/cache divergence for allow-always approvals.

## Best Possible Solution

Land a narrow fix in src/plugins/conversation-binding.ts that serializes persistent approval saves so the final on-disk plugin-binding-approvals.json reflects the latest shared cache after overlapping allow-always approvals. Add a focused regression test in src/plugins/conversation-binding.test.ts; leave the command-queue and hook-runner behaviors unchanged unless maintainers decide to split those lower-severity findings into separate follow-ups.
