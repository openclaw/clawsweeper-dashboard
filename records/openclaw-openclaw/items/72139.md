## Summary

Keep open. Current main still source-encodes the Control UI `AUTH_TOKEN_MISMATCH` reconnect loop when no cached device-token retry is available; `chat.history` and Slack timeout mitigations exist, but the terminal auth behavior requested here remains unresolved.

## What This Changes

The issue asks Control UI to stop stale-token WebSocket retry loops, keep `chat.history` from starving the event loop, and prevent UI auth failures from degrading Slack Socket Mode.

## Best Possible Solution

Make `AUTH_TOKEN_MISMATCH` terminal unless a trusted cached-device-token retry is actually queued, preserve the one-shot cached-token recovery path, and leave WebUI history-timeout UX to the related PR if it lands.

## Reproduction Assessment

Yes for the remaining auth-loop portion from source and existing tests: current main expects a reconnect after `AUTH_TOKEN_MISMATCH` with no stored device token. The live Slack Socket Mode timing was not reproduced here.

## Solution Assessment

Yes. The narrowest maintainable fix is reconnect gating in the browser and Node Gateway clients; `chat.history` timeout handling and Slack ping tuning are useful mitigations, not the root stale-token fix.
