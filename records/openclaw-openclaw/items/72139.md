## Summary

Keep open. Current main still preserves the stale-token reconnect loop when no cached device-token retry is available, and the `chat.history` work is only partially mitigated: append-only tail reads are bounded, but branch/tree transcript paths can still take synchronous full-history paths. Slack Socket Mode timeout tuning and the related WebUI timeout PR help symptoms, but they do not close the root availability bug.

## What This Changes

The issue asks OpenClaw to stop stale Control UI auth retry loops and make `chat.history` non-blocking enough that Control UI history loads cannot starve Slack Socket Mode pongs.

## Best Possible Solution

Fix the root behavior in core: make `AUTH_TOKEN_MISMATCH` terminal when no trusted cached-device-token retry is possible or after that retry is exhausted, and make all active-branch `chat.history` paths bounded, streaming, or yielding enough that large/tree transcripts cannot monopolize the Gateway event loop. Keep Slack timeout tuning and #44897-style frontend timeouts as defensive UX mitigations, not the closure condition.

## Reproduction Assessment

Yes for the auth-loop portion: current main has a browser unit test that clears localStorage, receives `AUTH_TOKEN_MISMATCH`, and expects reconnect. The full Slack disconnect timing is not independently reproduced here, but the reporter and commenter provide concrete Socket Mode logs and the current `chat.history` implementation still has plausible synchronous starvation paths.

## Solution Assessment

Yes, the issue's requested root direction is the maintainable fix: stop stale auth reconnect churn and bound/yield Gateway history work. A frontend-only `chat.history` timeout or larger Slack pong timeout is useful mitigation but is not sufficient to solve the shared Gateway event-loop availability problem.

Fix openclaw/openclaw#72139. Current main still reconnects indefinitely on `AUTH_TOKEN_MISMATCH` when no cached device-token retry can actually be attempted, and `chat.history` still has synchronous full transcript paths for tree/branch transcripts. Scope the PR to core Gateway/UI behavior: make browser `GatewayBrowserClient` and Node `GatewayClient` treat token mismatch as terminal when there is no trusted stored-device-token retry available, and after the one retry is exhausted; update tests that currently expect reconnect without localStorage. Keep protocol/docs aligned, add a user-facing `CHANGELOG.md` fix entry with reporter credit, and avoid changing Slack transport semantics except where a test/doc reference needs to clarify that timeouts are mitigation only.
