## Summary

Keep this PR open. It is addressing a real gateway data-loss gap that still exists on current main, but the branch is stale relative to the shipped CLI fix and its exact-path guard would miss parent/child workspace overlaps that current docs and tests already protect.

## What This Changes

The PR adds an exact realpath-normalized workspace lookup, uses it in CLI identity/delete and gateway agent deletion, and adds agent-scope plus gateway mutation tests.

## Best Possible Solution

Move the overlap-aware, realpath-normalized workspace-retention logic into a shared helper used by both CLI and gateway deletion, keep the existing CLI JSON/docs contract intact, and add gateway regression coverage for same, parent, child, symlink, and unique workspace cases.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction: configure two agents with overlapping workspaces and call gateway agents.delete with default deleteFiles; the handler moves workspaceDir to Trash without consulting other agent workspaces.

## Solution Assessment

No. The exact-match helper is narrower than the shipped CLI behavior and docs; the maintainable fix is a shared overlap-aware guard that preserves parent, child, exact, and symlink-equivalent workspaces consistently across CLI and gateway paths.
