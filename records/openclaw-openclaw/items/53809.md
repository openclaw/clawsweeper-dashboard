## Summary

Keep open. Current main already fixes the CLI shared-workspace deletion path more broadly, but the gateway `agents.delete` path still moves the workspace unconditionally, and this PR is not merge-ready because its exact-only guard misses parent/child workspace overlaps that the shipped CLI behavior protects.

## What This Changes

The branch adds exact realpath-normalized workspace matching, uses it in CLI and gateway agent deletion to decide whether to trash a workspace, reuses it for identity workspace lookup, and adds exact-match tests.

## Best Possible Solution

Move the overlap-aware, realpath-normalized workspace-retention logic into a shared helper used by both CLI and gateway deletion, preserve the existing CLI JSON/docs contract, and add gateway regression coverage for same, parent, child, symlink, and unique workspace cases.

## Reproduction Assessment

Yes. Configure another agent with the same, parent, child, or symlink-equivalent workspace and call gateway `agents.delete` with default `deleteFiles`; current main attempts to trash `workspaceDir` without checking the remaining agent workspaces.

## Solution Assessment

No. Exact path equality is narrower than the shipped CLI contract; the maintainable fix is a shared overlap-aware guard applied consistently to CLI and gateway deletion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve overlapping workspaces:** `src/agents/agent-scope.ts:333`
  - body: The new helper only treats exact realpath equality as shared. Current CLI behavior and docs retain a workspace when another agent uses the same path, a parent path, a child path, or a symlink alias, so using this helper as the delete guard can still trash workspace state another agent depends on and would regress the CLI guard on main.
  - confidence: 0.9
- **[P3] Use a real post-prune mock default:** `src/gateway/server-methods/agents-mutate.test.ts:438-440`
  - body: The shared-workspace resolver is called on `result.config` after pruning, so returning the deleted agent from the default mock models an impossible state. The baseline delete test can pass because production filters that id out, rather than proving the unique-workspace case directly.
  - confidence: 0.86
