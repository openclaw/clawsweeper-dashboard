## Summary

Keep open: the useful problem is real, but this patch removes all running background tasks from channel reload deferral even though some of those tasks can still deliver requester-origin updates through channel send paths.

## What This Changes

The PR changes gateway channel reload deferral so background task counts no longer delay channel reloads, while restart and shutdown paths still count them.

## Best Possible Solution

Define an explicit non-delivering task reload policy, or filter only known safe maintenance tasks, while keeping delivery-capable tasks in the deferral boundary with focused regression coverage and real patched-gateway proof.

## Reproduction Assessment

Yes, source-level: current main counts active background tasks in channel reload deferral, and the PR body includes before-fix logs showing a channel reload held by a context-engine maintenance task. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. Excluding all active background tasks is too broad because some running tasks can still deliver through requester-origin channel paths; the safer fix is an explicit safe-to-ignore task boundary or a narrow filter for silent maintenance work.
