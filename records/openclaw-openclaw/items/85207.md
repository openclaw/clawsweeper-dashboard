## Summary

Keep open: the shipped/current drift warning now catches stale active official ACPX installs, but current main still lacks the requested bounded retry for transient Codex ACP prompt failures before prompt-side effects.

## What This Changes

The issue requests ACPX/Codex ACP hardening so stale active plugins are detected and transient pre-side-effect prompt failures get bounded retry plus sanitized diagnostics instead of bare ACP_TURN_FAILED.

## Best Possible Solution

Keep the drift warning, then add a maintainer-approved ACPX retry contract that retries only transient pre-side-effect Codex ACP prompt failures and preserves hard failure once output, tool calls, or session state can have changed.

## Reproduction Assessment

Yes at source level: current main can be inspected so a generic Codex ACP Internal error is enriched and returned without a bounded retry, while the manager retry helper only handles narrower acpx-exit or missing-resume cases. I did not run a live stale-plugin deployment reproduction.

## Solution Assessment

No, the current drift detection is only a partial solution. The narrow maintainable fix still needs a bounded pre-side-effect retry contract in the ACPX-owned runtime path, with tests for retry admission and no retry after side effects.
