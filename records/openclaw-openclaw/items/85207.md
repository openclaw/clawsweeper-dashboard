## Summary

Keep open: current main and v2026.6.10 have useful plugin-version drift diagnostics and ACPX stderr-tail enrichment, and the latest discussion confirms drift detection catches stale ACPX installs, but the requested bounded pre-side-effect ACPX prompt retry and final stale-runtime startup/update policy are still unresolved.

## What This Changes

The issue asks OpenClaw to prevent stale active ACPX/Codex runtime plugins from surfacing bare ACP_TURN_FAILED errors by detecting version drift, keeping runtime hardening active, adding bounded pre-side-effect retry, and enriching diagnostics.

## Best Possible Solution

Keep the shipped drift detection and diagnostic enrichment, then add or explicitly reject a maintainer-approved ACPX pre-side-effect retry policy with regression tests for generic Internal error/client-closed failures and no-retry-after-output behavior.
