## Summary

Keep open. Current main still lacks a certificate-expiry recovery path that can escape process-local TLS state, and the READY identity-log request is only partly covered by related Discord READY threads.

## What This Changes

The issue requests hard recovery from Discord gateway certificate-expiry zombie states and restoration of an operator-visible Discord READY identity log.

## Best Possible Solution

Add Discord-owned certificate-expiry escalation to a maintainer-approved process respawn or gateway-supervisor path, and coordinate lifecycle-backed READY/RESUMED identity logging with the active Discord READY regression work.

## Reproduction Assessment

No, not for the full live CA-rotation outage. Source inspection does show that `certificate has expired` is not a special recovery class and that current reconnect paths remain in-process.

## Solution Assessment

Unclear until maintainers choose the recovery policy. In-process reconnects and channel-local restarts help related hangs, but they are not the narrowest complete fix for a stale process-local TLS trust state.
