## Summary

Keep open. Current main and v2026.6.6 only partially cover the request through `agents.defaults.sandbox.mode` and documented multi-gateway profile isolation; `routing.rules`, `gateway.nodes.denyPaths`, and a first-class `instances.*` config contract still require maintainer product and security decisions.

## What This Changes

Request adds four config surfaces: sandbox mode, tag-based provider routing, multi-instance declarations, and gateway node path denials.

## Best Possible Solution

Split this into maintainer-approved contracts: keep sandbox activation on `agents.defaults.sandbox`, route filesystem ACL design through https://github.com/openclaw/openclaw/issues/7722, route multi-instance lifecycle through https://github.com/openclaw/openclaw/issues/39659, and design provider tag-routing only after defining precedence against existing agent/model/provider config.

## Reproduction Assessment

Not applicable as a feature/config design request. Source inspection confirms the requested fields are only partially present today: sandbox mode exists under `agents.defaults.sandbox`, while `routing.rules`, `instances`, and path deny rules remain absent or separately tracked.

## Solution Assessment
