## Summary

Keep open. Current main and v2026.6.6 only partially cover the request: sandbox mode exists under `agents.defaults.sandbox`, and multi-gateway isolation is documented through separate profiles/state dirs, but `routing.rules`, `instances.*`, and `gateway.nodes.denyPaths` remain missing design surfaces that need maintainer product and security decisions.

## What This Changes

The issue requests first-class config schema support for global sandbox mode, tag-based provider routing, multi-instance declarations, and gateway-enforced path deny rules.

## Best Possible Solution

Split the work into maintainer-approved contracts: keep sandbox activation on `agents.defaults.sandbox`, route filesystem ACL design through https://github.com/openclaw/openclaw/issues/7722, route Docker/multi-instance lifecycle through https://github.com/openclaw/openclaw/issues/39659, and define provider tag-routing precedence before adding `routing.rules`.
