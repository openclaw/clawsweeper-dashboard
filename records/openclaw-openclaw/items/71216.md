## Summary

Keep open. Current main still lacks the requested config contracts: sandboxing is agent-scoped, routing is channel/session binding plus model config rather than tag-based provider rules, multi-gateway isolation is documented through separate profiles/config/state, and gateway node policy only exposes command allow/deny shaping.

## What This Changes

The issue requests new OpenClaw config schema and enforcement surfaces for `sandbox.mode`, `routing.rules`, `instances.*`, and `gateway.nodes.denyPaths`.

## Best Possible Solution

Split this into maintainer-approved contracts for sandbox precedence, trusted routing tags, multi-instance isolation, and filesystem path-deny enforcement, then implement accepted pieces with schema, runtime enforcement, doctor checks, docs, and tests.

## Reproduction Assessment

Not applicable as a feature/config-schema request. Source and docs inspection provide the high-confidence verification path, and they show the requested contracts are absent or only covered by adjacent mechanisms.

## Solution Assessment

No, not as one schema-only change. The safer solution is separate maintainer design for each security and product boundary before adding config keys that imply gateway enforcement.
