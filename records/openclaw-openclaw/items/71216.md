## Summary

Keep open. Current main still only partially overlaps the request: sandboxing is agent-scoped, routing is binding/session based rather than tag/provider-rule based, multi-gateway operation is documented as separate config/state profiles, and path denial is split across narrower tool or node policies rather than one gateway-wide `gateway.nodes.denyPaths` contract.

## What This Changes

The issue requests new config schema surfaces for global sandbox mode, tag-based provider routing, multi-instance declarations, and gateway-wide node path denial.

## Best Possible Solution

Split the request into maintainer-approved contracts for sandbox precedence, trusted routing tags, multi-instance lifecycle/isolation, and filesystem/path-deny enforcement, with schema, runtime, docs, doctor/security-audit checks, and focused tests for each accepted surface.
