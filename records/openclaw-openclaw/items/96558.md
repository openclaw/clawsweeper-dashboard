## Summary

Keep open: current main does not yet harden encoded root-mounted `/api` and `/plugins` variants, so the useful change is not obsolete, but this PR is not merge-ready because encoded probe variants are only kept out of the SPA fallback and are not actually routed by the gateway probe stage.

## What This Changes

The branch canonicalizes encoded root-mounted Control UI route variants for gateway probe, `/api`, and `/plugins` paths and adds route-classifier regression cases.

## Best Possible Solution

Align the classifier and server probe matching so encoded probe variants either return real probe responses or the PR narrows that claim, then add integration coverage and redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection: current main serves encoded root-mounted `/api` and `/plugins` variants through the SPA fallback, and encoded probe variants are not exact-matched by the gateway probe map. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. Reusing canonicalization in the Control UI classifier is a good fit for `/api` and `/plugins`, but probe variants need server-level probe matching or the PR should remove the claim that they route to probe handling.
