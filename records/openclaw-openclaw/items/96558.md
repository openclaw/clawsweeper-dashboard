## Best Possible Solution

Canonicalize probe matching at the gateway probe stage or narrow the PR’s probe claim, then keep the `/api` and `/plugins` classifier hardening with focused classifier and server-level route proof.

## Reproduction Assessment

Yes. Source inspection shows current main serves encoded root-mounted `/api` and `/plugins` variants through the SPA fallback, and encoded probe variants are not exact-matched by the gateway probe map; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. Reusing canonicalization in the Control UI classifier is a good fit for `/api` and `/plugins`, but probe variants need server-level probe matching or the PR should remove the claim that they route to probe handling.
