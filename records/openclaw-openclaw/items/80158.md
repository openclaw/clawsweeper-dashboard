## Summary

Keep open: current main still has the reported source-of-truth mismatch, and the related SQLite/session-seam issues are open design tracks rather than merged fixes for memory-core, dreaming, markdown export, or the session-logs skill.

## What This Changes

The issue asks OpenClaw to stop treating tail-only rotated session JSONL files as complete raw history for memory-core, dreaming, markdown export, and session-log inspection.

## Best Possible Solution

Add a typed session-history provider/read seam that reports completeness, then route memory-core indexing, dreaming ingestion, markdown export, and session-logs guidance through it when a context engine owns durable history.

## Reproduction Assessment

Yes from source for the OpenClaw side: current main still enumerates and flattens only session JSONL files that would be tail-only after rotation. I did not live-run lossless-claw, so this is source-reproducible rather than fully reproduced in a real setup.

## Solution Assessment

Yes in direction, but it needs maintainer contract design: a canonical provider/completeness marker is safer than adding more JSONL scraping branches in each consumer.
