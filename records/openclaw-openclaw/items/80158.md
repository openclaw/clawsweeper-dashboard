## Summary

Keep open: current main still routes memory-core indexing, dreaming ingestion, QMD session export, and the bundled session-logs skill through session JSONL readers that cannot distinguish a complete transcript from a rotated tail checkpoint. The related SQLite/context-engine seam issues are open design tracks, not merged fixes, so this remains a valid source-reproducible bug that needs maintainer contract design before implementation.

## What This Changes

Report asks OpenClaw to stop treating session JSONL files as complete history when context-engine rotation leaves only a tail checkpoint.

## Best Possible Solution

Add a versioned session-history provider/completeness seam, then route memory-core indexing, dreaming ingestion, QMD export, and session-logs guidance through it or clearly mark JSONL-derived results incomplete.

## Reproduction Assessment

Yes, from source for the OpenClaw side: the affected consumers still enumerate and flatten only the JSONL file that would be tail-only after rotation. I did not perform a live run with a rotating context engine, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes in direction: a canonical provider/completeness contract is the maintainable fix. Scattering more JSONL readers or tail heuristics through each consumer would keep the source-of-truth mismatch in place.
