## Summary

Keep open: current main still routes memory/session consumers through direct JSONL enumeration and flattening, while the context-engine contract has no durable complete-history provider or completeness marker. The related SQLite/session-seam issues are useful open design tracks, but they are not merged fixes for this concrete memory-core, dreaming, QMD export, and session-logs behavior.

## What This Changes

The issue asks OpenClaw to stop treating tail-only session JSONL checkpoints as complete history after context-engine transcript rotation.

## Best Possible Solution

Add a typed session-history provider/completeness contract, then route memory-core session indexing, dreaming ingestion, QMD session export, and session-logs guidance through that contract when a context engine owns durable history.

## Reproduction Assessment

Yes from source for the OpenClaw side: current main still enumerates and flattens only the session JSONL files that would become tail-only after rotation. I did not live-run the external lossless-claw rotation path, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes in direction, but not as a narrow code-only repair yet. A canonical provider/completeness seam is safer than adding more JSONL scraping branches to each consumer.
