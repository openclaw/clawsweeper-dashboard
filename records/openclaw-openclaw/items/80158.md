## Summary

Keep open: current main still has memory-core, dreaming, QMD export, and the bundled session-logs skill treating session JSONL as the readable history source, while context-engine rotation can make that file a tail checkpoint. This is valid by source inspection, but the durable-history fix crosses public context-engine/session-store boundaries and should be coordinated with the open companion-seam stack.

## What This Changes

The issue asks OpenClaw session-history consumers and bundled guidance to stop treating context-engine-rotated tail JSONL files as complete raw history.

## Best Possible Solution

Add a canonical durable transcript read surface, make memory-core/dreaming/QMD/session-log consumers use it or label JSONL-derived results incomplete, and update bundled guidance to describe JSONL as the active transcript checkpoint when a context engine owns rotation.

## Reproduction Assessment

Yes by source inspection, with high confidence: current main still lists and parses session JSONL files directly for the affected consumers, and the referenced context engine rewrites that file to a preserved tail. I did not run the full live rotation scenario.

## Solution Assessment

No narrow one-file fix is sufficient. The maintainable path is a generic durable transcript provider or SQLite replay seam plus consumer/docs updates, rather than plugin-specific recovery logic in each memory path.
