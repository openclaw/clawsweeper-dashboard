## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate provider receipts before returning them:** `extensions/channel-broker/src/runtime.ts:46-50`
  - body: `parseBrokerReceipt` accepts any object from `/v1/outbound` and casts it to `BrokerReceiptV1`. A provider can return a malformed `sent` receipt with missing or wrongly typed `messageIds`, `requestId`, or `status`, and the durable send path will either commit bad metadata or fail with an incidental runtime error; validate this external JSON boundary with a schema/type guard and cover invalid sent/non-sent receipts.
  - confidence: 0.88
- **[P2] Accept the SDK capability shape in broker config:** `extensions/channel-broker/src/config-schema.ts:5-23`
  - body: The config schema only permits flat keys like `draftPreview` and `progressPreview`, but the SDK and conformance checks expect nested `delivery`, `live`, `receive`, and `native` groups. Configured provider capabilities will be rejected or normalized into fields `brokerPlatformSupports` never reads; align the schema and add a config validation test for the public capability shape.
  - confidence: 0.84
