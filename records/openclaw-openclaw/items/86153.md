Full review comments:

- **[P2] Export the runtime setter the entry declares:** `extensions/channel-broker/index.ts:29-30`
  - body: `index.ts` tells the bundled channel loader to import `setChannelBrokerRuntime` from `./runtime-api.js`, but that file does not export the setter. Current `defineBundledChannelEntry` loads the declared export and calls it during full registration, so enabling/loading this bundled channel can fail with a missing export instead of wiring the runtime. Add the setter export or point the descriptor at a narrow runtime-setter file.
  - confidence: 0.98
- **[P2] Apply pre-auth limits before broker signature verification:** `extensions/channel-broker/src/http-routes.ts:115`
  - body: This `auth: "plugin"` route reaches the handler without Gateway auth, and the broker HMAC is checked only after `readWebhookBodyOrReject` reads the body with the `post-auth` defaults. That lets an unauthenticated caller hold/read up to the larger post-auth budget before signature rejection. Use the shared pre-auth profile or explicit small maxBytes/timeout before parsing and verifying the signature.
  - confidence: 0.95
