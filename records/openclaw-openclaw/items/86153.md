## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Validate broker receipt JSON before returning it:** `extensions/channel-broker/src/runtime.ts:45-49`
  - body: The HTTP transport casts any provider object to `BrokerReceiptV1`. A 200 response with `status: "sent"` but missing or non-array `messageIds`, or mismatched request/provider/platform fields, can reach durable-send handling and either throw an unstructured runtime error or commit bad receipt metadata. Validate the receipt shape and expected IDs before returning it.
  - confidence: 0.86
- **[P2] Align capability config with the SDK shape:** `extensions/channel-broker/src/config-schema.ts:6-20`
  - body: The public SDK models platform capabilities as nested `delivery`, `live`, `receive`, and `native` objects, but this strict Zod schema only accepts flat fields such as `inbound`, `outbound`, and `progressPreview`. A provider config that follows the SDK type will be rejected before `normalizeCapabilities` can use it, so the config schema and focused tests need to match the public contract.
  - confidence: 0.78
