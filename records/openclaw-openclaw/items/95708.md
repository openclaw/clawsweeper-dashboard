## Summary

Keep this PR open: the WebChat fix is directionally right and has real visual proof, but it is not merge-ready because the new item-only preamble segments can be mis-associated with tool segments during stream reconciliation, and the branch commits proof media that repository policy keeps out of product branches.

## What This Changes

The PR changes WebChat stream handling to consume `stream:"item"` preamble progress events as keyed stream segments and materialize visible pre-final commentary before final assistant messages.

## Best Possible Solution

Update stream reconciliation so non-tool preamble segments are not matched or pruned by tool-index fallback, add a regression covering preamble-before-tool persistence, and keep proof media outside the branch as PR or artifact evidence.

## Reproduction Assessment
