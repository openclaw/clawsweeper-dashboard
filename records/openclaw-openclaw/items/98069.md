## Summary

Keep open: current main still lacks pre-apply runtime validation for Skill Workshop proposals, and the remaining work needs maintainer product/security direction because it would execute or simulate unapproved skill behavior.

## What This Changes

Request adds an optional Skill Workshop validation phase that loads pending proposals, verifies tool/schema discovery, invokes representative behavior, and reports results before apply.

## Best Possible Solution

Define a bounded Skill Workshop validation contract first, then wire one shared service through CLI, Gateway, and the agent tool with load/schema checks and isolated invocation only where side effects and expected outputs are explicit.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior bug. Source and release inspection confirm there is currently no workshop test command, no agent-tool test action, and no Gateway proposal runtime-test API.

## Solution Assessment

Unclear: pre-apply validation is directionally useful, but runtime invocation is not yet the best final shape until maintainers define sandboxing, side-effect handling, and the test-case contract. A safer MVP may start with load/discovery/schema validation and defer arbitrary side-effect execution.
