## Summary

Keep open: current main and v2026.6.10 still have Skill Workshop proposal creation, inspection, scanning, and apply/reject/quarantine flows, but no pre-apply runtime validation action that loads and invokes the proposed skill. The related open contract proposal overlaps on validation/governance but does not replace this runtime-testing request.

## What This Changes

The issue requests automated runtime validation for Skill Workshop proposals before they are applied to the workspace.

## Best Possible Solution

Define one narrow proposal runtime-validation contract first, then implement it through the existing Skill Workshop service, skill loading/status, and tool-dispatch surfaces without creating a second apply policy.

## Reproduction Assessment

Yes for the feature gap by source inspection: current CLI, agent tool, Gateway schema, docs, and tests lack a pre-apply runtime validation action. This is not a failing bug reproduction against an existing contract.

## Solution Assessment

Unclear: runtime validation is plausible and useful, but representative inputs, sandboxing, side-effect handling, and surface ownership need maintainer product/security direction before a best implementation can be chosen.
