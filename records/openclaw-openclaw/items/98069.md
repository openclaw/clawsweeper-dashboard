## Summary

Keep open: current main and v2026.6.10 have proposal storage, static scanning, apply-time write checks, rollback, CLI, Gateway, and agent-tool surfaces, but they do not provide the requested pre-apply runtime validation command or action. The remaining work is useful but needs maintainer product/security direction before implementation because it would execute or simulate proposed skill behavior before the proposal is applied.

## What This Changes

Add a Skill Workshop validation phase or command that loads pending proposals, validates schemas, and invokes proposed skill behavior with representative inputs before apply.

## Best Possible Solution

Define a bounded Skill Workshop validation contract first, then wire one shared service through CLI, Gateway, and the agent tool with load/schema checks and explicitly isolated invocation tests only where the side-effect boundary is clear.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior bug. Source and release inspection confirm there is currently no `openclaw skills workshop test` command, no agent-tool test action, and no Gateway proposal runtime-test API.

## Solution Assessment

Unclear: pre-apply validation is directionally useful, but the proposed runtime invocation phase is not yet the best final shape until maintainers define sandboxing, side-effect handling, and the test-case contract. A safer MVP may start with load/discovery/schema validation and defer arbitrary side-effect execution.
