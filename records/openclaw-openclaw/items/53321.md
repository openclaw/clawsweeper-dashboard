## Summary

Keep open. Current main has compaction, memory flush, successor transcript rotation, and manual/daily/idle reset paths, but it still does not provide the requested configurable threshold-based session split with durable continuity handoff and automatic channel binding rollover. Related duplicates already point back to this issue as the canonical tracker.

## What This Changes

Request configurable automatic session splitting/rollover when sessions approach model context limits, with memory flush, continuity context, and routing to a new backing session.

## Best Possible Solution

Keep this as the canonical core session-lifecycle design item, then implement a config-backed context-pressure rollover with durable handoff, binding-preserving session-id rotation, docs, schema, and focused regression coverage after maintainer product review.
