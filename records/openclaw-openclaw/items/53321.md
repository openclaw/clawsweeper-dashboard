## Summary

Keep open: current main and v2026.6.6 still provide compaction, successor transcript rotation, daily/idle/manual resets, and context reporting, but not the requested configurable context-threshold auto session split with durable handoff. The closed related reports consistently point back to this issue as the canonical product/API thread.

## What This Changes

Request configurable token-threshold session rollover that flushes/compacts memory, starts a fresh backing session, carries continuity state, and routes later messages there automatically.

## Best Possible Solution

Design one core session-lifecycle policy for context-pressure rollover with canonical config, durable handoff semantics, delivery identity preservation, idempotent rotation, docs, and upgrade tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new capability request. Source and docs inspection give high confidence that current main lacks the requested automatic context-pressure session split.

## Solution Assessment

Unclear as an implementation until maintainers choose the public config, default behavior, and continuity contract. The likely layer is core session lifecycle, but a code fix should not start before the rollover, handoff, and overlap with compaction/TTL are decided.
