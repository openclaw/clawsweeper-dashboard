## Summary

Keep open: current main and v2026.6.6 still handle context pressure with compaction plus daily/idle/manual session reset, not configurable token-threshold session splitting with durable handoff. This issue is the canonical open product/API design thread for the already-closed duplicate reports.

## What This Changes

The issue requests automatic session rollover near the model context limit, including pre-split memory flush, continuity context injection, and routing future messages into a fresh backing session.

## Best Possible Solution

Design one core session-lifecycle policy for context-pressure rollover, with canonical config, durable handoff semantics, delivery-context preservation, idempotent rotation, docs, and upgrade tests before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new capability request. Source and docs inspection give high confidence that current main lacks the requested automatic context-pressure session split.

## Solution Assessment

Unclear as an implementation until maintainers choose the public config and continuity contract. The right general layer appears to be core session lifecycle, but the exact rollover, handoff, and overlap with TTL/compaction need product review.
