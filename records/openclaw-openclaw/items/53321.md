## Summary

Keep open. Current main has compaction, optional memory flush, successor transcript rotation, silent reset continuity, and queued-run retargeting, but it still does not implement the requested configurable auto session split lifecycle with threshold policy, channel binding rollover, active-run boundary handling, and explicit continuity payload injection. Same-author #53330 is already closed as the duplicate, so this remains the canonical tracker.

## What This Changes

Request automatic splitting of long-running sessions near a configurable context-window threshold, preserving/bridging state into a new session and transparently routing subsequent channel messages there.

## Best Possible Solution
