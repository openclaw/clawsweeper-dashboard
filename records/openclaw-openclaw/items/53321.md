## Summary

Keep open. Current main has several mitigations around compaction, memory flush, successor transcript rotation, silent reset continuity, and queued-run retargeting, but it still does not provide the requested configurable auto session split lifecycle with threshold policy, channel binding rollover, active-run boundary handling, and explicit continuity payload injection.

## What This Changes

Add a configurable core session lifecycle that automatically starts a successor session near the context limit and carries enough state forward for user-invisible continuity.

## Best Possible Solution

Keep this as a core session-lifecycle design item: extend the existing compaction, memory flush, successor transcript, session store, channel binding, and queue-retargeting seams with documented threshold config, rollover semantics, continuity payload rules, and regression coverage.
