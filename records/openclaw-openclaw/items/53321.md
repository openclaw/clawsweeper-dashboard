## Summary

Keep open. Current main has substantial compaction, memory-flush, successor-transcript, and reset-rollover mitigations, but it still lacks the requested configurable auto session split lifecycle that would proactively roll an active agent+channel binding into a new session with an explicit continuity payload.

## What This Changes

Request automatic session rollover near the model context limit, with configurable thresholds, memory flush, continuity context injection, and routing of later messages to the new session.

## Best Possible Solution

Keep this as a core session-lifecycle design item that extends existing compaction, memory flush, successor transcript, session store, channel binding, and queue-retargeting seams with documented threshold policy and focused regression coverage.

Not applicable: this is a feature request, not a broken existing contract. The current-main negative check is high confidence because schema, runtime, docs, tests, and identifier search show compaction/rotation mitigations but not the requested auto-split lifecycle.

## Solution Assessment
