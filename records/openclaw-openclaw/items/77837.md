## Summary

Keep open: this is a current, detailed regression report against a documented external WeChat channel, and current `main` does not provide enough evidence to close it as plugin-only or already fixed.

## What This Changes

The issue reports that `@tencent-weixin/openclaw-weixin` getUpdates fails with `TypeError: fetch failed` on Windows after upgrading from OpenClaw 2026.5.3-1 to 2026.5.4.

## Best Possible Solution

Reproduce the failure on Windows with OpenClaw 2026.5.4/current `main` and the Weixin plugin, then fix the generic Gateway/plugin SDK fetch boundary if the host changed fetch behavior, or coordinate a plugin-side transport fix if the Tencent iLink runtime owns it.

## Reproduction Assessment

No, not yet. The issue gives a concrete Windows reproduction path and logs, and the Tencent endpoint is reachable from a standalone local fetch here, but this review did not run the Weixin plugin monitor on Windows against current `main`.

## Solution Assessment

Unclear. A generic core fix around Gateway network bootstrap or plugin SDK fetch handling would be preferable if OpenClaw 2026.5.4 changed the host fetch environment, but live proof is needed before choosing that over a plugin-owned iLink transport fix.
