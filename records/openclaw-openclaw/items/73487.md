## Summary

Keep open. Current main still suppresses Telegram tool-progress preview when a native quote reply blocks the answer draft stream, and neither the Telegram/streaming docs nor doctor warnings currently explain that caveat; open closing PRs #73495 and #73508 should be reviewed instead of closing early.

## What This Changes

The issue asks OpenClaw to document or warn that Telegram native quote replies can suppress `streaming.preview.toolProgress` previews when `replyToMode` is enabled.

## Best Possible Solution

Land one canonical Telegram-scoped clarification, ideally docs plus a matching doctor warning; treat a persistent separate-message fallback as separate behavior work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable Telegram preview streaming and tool progress, set `replyToMode` to `first` or `all`, then use a selected/native quote so `ReplyToQuoteText` is present; current code skips the answer stream and drops preview tool-progress events.

## Solution Assessment

Yes for the near-term direction. Documentation plus a doctor warning is the narrowest maintainable fix for the silent mismatch, while a separate-message runtime fallback would be broader behavior work.
