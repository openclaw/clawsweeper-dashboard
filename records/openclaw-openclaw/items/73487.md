## Summary

Keep open. Current main narrows the original blanket failure for plain Telegram reply-to turns, but selected/native quote replies still prevent the answer draft stream that tool-progress preview depends on; current docs and doctor output still do not surface that caveat, and two open closing PRs already target this issue.

## What This Changes

Report that Telegram tool-progress preview is silently unavailable when non-off `replyToMode` uses native quote replies, despite docs saying tool-progress preview is default-on.

## Best Possible Solution

Land the preferred Telegram-scoped clarification: document the native quote limitation and, if maintainers want config-time visibility, add an `openclaw doctor` warning that matches the current narrowed runtime behavior.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, use non-off `replyToMode`, and reply with selected/native quote text; current source and tests show that path skips the answer draft stream while plain reply-to turns are now covered by the adjacent fix.

## Solution Assessment

Yes for the near-term direction. Documenting the caveat and optionally warning in doctor is narrower than adding a persistent-message fallback, which should remain a separate product/runtime decision.
