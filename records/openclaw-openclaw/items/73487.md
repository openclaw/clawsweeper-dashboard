## Summary

Keep open. Current main still has the Telegram native-quote path that prevents creating the answer preview stream, while the Telegram docs and doctor preview warnings still do not surface the reply-to/tool-progress conflict; the issue is also paired with open closing PRs #73495 and #73508, so it should remain open until maintainers choose and land one.

## What This Changes

The issue reports that Telegram tool-progress preview is documented as enabled by default but is silently suppressed when `channels.telegram.replyToMode` enables native quote replies.

## Best Possible Solution

Land the preferred Telegram-scoped clarification so users can discover that native quote replies and preview tool-progress are mutually exclusive today, ideally with both public docs and an `openclaw doctor` warning; treat a persistent-message fallback as a separate runtime/product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger a tool-using turn on a quotable inbound message; current source shows the answer stream is not created, and the issue discussion includes two released-build confirmations.

## Solution Assessment

Yes for the near-term direction. Because the runtime exclusion appears intentional for Telegram native quote replies, documenting it and adding a doctor warning is the narrowest maintainable fix for the silent mismatch; changing delivery to a separate-message fallback should be a follow-up product choice.
