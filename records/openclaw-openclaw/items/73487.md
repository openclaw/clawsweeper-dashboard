## Summary

Keep open. Current main still has the Telegram selected/native quote path that suppresses the answer draft stream used by tool-progress previews, while docs and doctor output still do not surface that caveat; open closing PRs #73495 and #73508 should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to document or warn that Telegram tool-progress preview is disabled when native quote reply mode prevents answer-lane preview streaming.

## Best Possible Solution

Land one canonical Telegram-scoped clarification, preferably docs plus a matching doctor warning; keep a persistent separate-message fallback as separate behavior work.

## Reproduction Assessment

Yes. The source-level path is high confidence: enable Telegram preview tool progress, set `replyToMode` away from `off`, and use a selected/native quote so the answer draft stream is not created and tool-progress preview no-ops.

## Solution Assessment

Yes for the near-term direction. Documentation plus a doctor warning is the narrowest maintainable fix for the silent doc/runtime mismatch; a persistent fallback message path would be broader behavior work.
