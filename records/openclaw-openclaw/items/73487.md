## Summary

Keep open. Current main narrows the original blanket report to selected/native Telegram quote replies, but that path still prevents the answer draft stream that tool-progress preview requires; the docs and doctor output still do not surface the caveat, and open closing PRs #73495 and #73508 should be reviewed rather than closing the issue first.

## What This Changes

Telegram tool-progress preview should be documented or warned when native quote reply mode prevents the answer draft stream from rendering progress updates.

## Best Possible Solution

Land the preferred Telegram-scoped clarification that documents the selected/native quote limitation and, if maintainers want config-time visibility, adds a matching `openclaw doctor` warning; keep persistent-message fallback as a separate product/runtime decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, use non-off `replyToMode`, and send a selected/native quote reply; current source and tests show that path skips the answer draft stream, so tool-progress updates are dropped.

## Solution Assessment

Yes for the near-term direction. A docs caveat plus optional doctor warning is narrower and lower risk than adding a persistent-message fallback, which would change Telegram runtime behavior and should be a separate decision.
