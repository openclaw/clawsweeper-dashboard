## Summary

Keep open. Current main narrowed the original blanket `replyToMode != "off"` failure for plain reply-to turns, but native/selected Telegram quote excerpts still prevent the answer preview stream that tool-progress depends on, and the docs/doctor surfaces still do not explain that caveat; open closing PRs #73495 and #73508 are already attached.

## What This Changes

The issue asks OpenClaw to surface that Telegram preview tool-progress is suppressed when native quote replies prevent creating the answer preview stream.

## Best Possible Solution

Land the preferred Telegram-scoped clarification: document the native quote limitation and, if maintainers want config-time visibility, add an `openclaw doctor` warning that matches the current narrowed runtime behavior.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, use non-off `replyToMode`, and reproduce with a real selected/native quote excerpt; current source and tests show that path skips the answer draft stream, while plain reply-to turns are now covered by the adjacent #73505 fix.

## Solution Assessment

Yes for the near-term direction. Documenting the caveat and optionally warning in doctor is narrower than changing Telegram delivery semantics; a persistent-message fallback should remain a separate product/runtime decision.
