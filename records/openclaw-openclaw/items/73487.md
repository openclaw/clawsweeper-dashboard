## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents an answer preview stream from being created, while the Telegram docs, streaming docs, and doctor warnings still do not surface the `replyToMode` exclusion; linked candidate PRs #73495 and #73508 remain open and should be reviewed instead of closing this issue now.

## What This Changes

The issue asks OpenClaw to surface or resolve the Telegram doc/runtime mismatch where tool-progress preview does not render when `channels.telegram.replyToMode` enables native quote replies.

## Best Possible Solution

Land one canonical Telegram-scoped clarification, preferably documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs; keep persistent-message fallback behavior as a separate product decision if maintainers want runtime delivery changes later.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the discussion confirms this on released builds.

## Solution Assessment

Yes for the near-term fix. A docs plus doctor-warning change is the narrowest maintainable answer to the silent doc/runtime mismatch; changing runtime fallback delivery would be larger and should not block acknowledging the current limitation.
