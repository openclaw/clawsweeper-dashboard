## Summary

Keep open. Current main narrowed the original failure so plain Telegram reply-to turns can still stream previews, but selected/native quote replies still suppress the answer preview stream that tool-progress uses, and the docs/doctor surfaces still do not explain that caveat; open closing PRs #73495 and #73508 are already attached.

## What This Changes

Report that Telegram tool-progress preview can disappear without docs or doctor guidance when non-off Telegram reply threading uses a native quote excerpt.

## Best Possible Solution

Land the preferred Telegram-scoped clarification: document the native quote limitation and add a matching `openclaw doctor` warning if maintainers want config-time visibility.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled and use non-off `replyToMode` on a selected/native quote reply; current source and tests show that path skips the answer draft stream that tool-progress requires.

## Solution Assessment

Yes for the near-term direction. Documenting the caveat and optionally warning in doctor is narrower and safer than changing Telegram delivery semantics; a persistent-message fallback should be handled separately.
