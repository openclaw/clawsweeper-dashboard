## Summary

Keep open. Current main still drops Telegram tool-progress preview for selected/native quote replies because no answer draft stream is created, while docs and doctor output still do not surface that caveat; #73495 and #73508 already close this issue and should be reviewed instead of closing early.

## What This Changes

Telegram tool-progress preview is reported as silently missing when non-off `channels.telegram.replyToMode` uses native quote replies, despite docs presenting tool-progress preview as default-on.

## Best Possible Solution

Land one canonical Telegram-scoped clarification, preferably docs plus a matching doctor warning; keep a persistent separate-message fallback as separate behavior work.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, set non-off `replyToMode`, and use a selected/native quote so `ReplyToQuoteText` is present; current source and tests show that path skips the answer draft stream that renders tool-progress previews.

## Solution Assessment

Yes for the near-term direction. Documentation plus an optional doctor warning is the narrowest maintainable fix for the silent mismatch; a separate-message runtime fallback would be broader behavior work.
