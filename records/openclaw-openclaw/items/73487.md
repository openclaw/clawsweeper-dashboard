## Summary

Keep open. Current main still has a Telegram selected/native quote path where reply preview streaming is not created, so tool-progress preview cannot render, and the docs/doctor surfaces still do not disclose that caveat; the two open closing PRs should be reviewed instead of closing this issue early.

## What This Changes

The issue requests that Telegram tool-progress preview’s incompatibility with native quote reply mode be documented, warned about, or given a fallback path.

## Best Possible Solution

Land one canonical Telegram-scoped clarification that documents the selected/native quote limitation and, if maintainers want config-time visibility, adds a matching doctor warning; keep persistent-message fallback behavior as a separate product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, set non-off `replyToMode`, and use a selected/native quote so `replyQuoteText` is present; current source and tests show that path skips the answer draft stream that renders tool-progress previews.

## Solution Assessment

Yes for the near-term direction. Documentation plus an optional doctor warning is the narrowest maintainable fix for the silent mismatch; a separate-message runtime fallback would be broader behavior work.
