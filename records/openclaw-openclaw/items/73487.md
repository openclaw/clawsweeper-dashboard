## Summary

Keep open. Current main still has the Telegram native-quote path that prevents creating the answer preview stream, the public docs and doctor warning path still do not surface the reply-to/tool-progress conflict, and the issue is already paired with open closing PRs that should be reviewed instead of closing the report early.

## What This Changes

The issue requests a Telegram-scoped fix for the silent mismatch where native quote replies disable tool-progress preview despite docs describing it as enabled by default, with suggested docs, doctor-warning, or runtime fallback options.

## Best Possible Solution

The best end state is a Telegram-scoped clarification that makes the current mutual exclusion discoverable, preferably in both public docs and `openclaw doctor`; a separate persistent-message fallback can remain a later runtime/product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger a tool-using turn on a quotable inbound message; current source shows the answer stream is not created, and the issue discussion includes two released-build confirmations.

## Solution Assessment

Yes for the near-term direction. Because the native quote exclusion appears intentional in the runtime, documenting it and adding a doctor warning is the narrowest maintainable fix for the silent mismatch; a fallback delivery path would be a broader behavior change.
