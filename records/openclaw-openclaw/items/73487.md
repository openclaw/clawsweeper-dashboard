## Summary

Keep open. Current main still disables Telegram tool-progress preview whenever native quote replies prevent an answer draft stream, and neither docs nor doctor warnings surface that trade-off; open closing PRs #73495 and #73508 are the implementation candidates to review before cleanup.

## What This Changes

The issue requests documenting or warning when Telegram `replyToMode` quote replies silently prevent `streaming.preview.toolProgress`, with an optional runtime fallback as a larger alternative.

## Best Possible Solution

Land a Telegram-scoped fix that documents the quote-reply/tool-progress trade-off and, preferably, adds a doctor warning for affected configs; leave persistent-message runtime fallback as a separate product decision if maintainers want tool visibility with native quote replies later.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the discussion confirms this on released builds.

## Solution Assessment

Yes for the near-term direction, but not by closing before merge. A docs plus doctor-warning change is the narrow maintainable fix for the silent doc/runtime mismatch, while runtime fallback delivery would be broader behavior work.
