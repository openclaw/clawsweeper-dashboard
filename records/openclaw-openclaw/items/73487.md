## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents an answer preview stream, while the Telegram docs, streaming docs, and doctor preview warnings still do not explain the `replyToMode` exclusion; the two linked closing PRs remain open and should be reviewed rather than closing this issue early.

## What This Changes

The issue requests that Telegram stop silently hiding tool-progress preview updates when native quote-reply mode prevents the answer preview stream, either through docs, doctor warnings, or a runtime fallback.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs; treat persistent-message fallback as a separate follow-up if maintainers want runtime behavior changes.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the discussion confirms the behavior on released builds.

## Solution Assessment

Yes for the near-term path. A docs plus doctor-warning change is the narrowest maintainable fix for the silent doc/runtime mismatch; a runtime fallback would be larger and should not block surfacing the current limitation.
