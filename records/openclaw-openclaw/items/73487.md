## Summary

Keep open. Current main has narrowed the original blanket failure to selected/native Telegram quote replies, but that remaining path still suppresses the answer draft stream that tool-progress preview needs, while docs and doctor output still do not surface the caveat; open closing PRs #73495 and #73508 should be reviewed instead of closing this issue early.

## What This Changes

The issue asks OpenClaw to surface Telegram's reply-to/native-quote limitation for tool-progress preview through docs, doctor warnings, or a broader runtime fallback.

## Best Possible Solution

Land one canonical Telegram-scoped clarification that documents the selected/native quote limitation and, if maintainers want config-time visibility, adds a matching doctor warning; keep any persistent-message fallback as a separate product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, set non-off `replyToMode`, and use a selected/native quote so `replyQuoteText` is present; current source and tests show that path skips the answer draft stream that renders tool-progress previews.

## Solution Assessment

Yes for the near-term direction. Documentation plus an optional doctor warning is the narrowest maintainable fix for the silent mismatch; a separate-message runtime fallback would be broader behavior work.
