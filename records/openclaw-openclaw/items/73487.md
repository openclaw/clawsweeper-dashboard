## Summary

Keep open. Current main fixed the broader plain reply-to streaming regression, but native Telegram quote excerpts still suppress the answer preview stream that tool-progress depends on, and the Telegram docs/doctor surfaces still do not explain that caveat; open closing PRs #73495 and #73508 are already attached.

## What This Changes

The issue reports that Telegram tool-progress preview is silently unavailable when quote-reply settings prevent the answer draft stream, and asks for documentation, doctor warnings, or a runtime fallback.

## Best Possible Solution

Land the preferred Telegram-scoped clarification: document the native quote limitation and, if maintainers want config-time visibility, add an `openclaw doctor` warning that matches the current narrowed runtime behavior.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with tool-progress enabled, set non-off `replyToMode`, and use a real selected/native quote excerpt; current source and tests show that path skips the answer draft stream while plain reply-to streaming is now covered by #73505.

## Solution Assessment

Yes for the near-term direction. Documenting the caveat and optionally warning in doctor is narrower than changing Telegram delivery semantics; a persistent-message fallback should remain a separate runtime/product decision.
