## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents answer-lane preview streaming, and the documentation plus doctor warning surfaces still do not explain the `replyToMode` exclusion; open closing PRs #73495 and #73508 are already attached, so this should remain open until maintainers choose and land a fix.

## What This Changes

The issue requests that Telegram tool-progress preview behavior be documented or warned about when `channels.telegram.replyToMode` prevents the preview stream from being created.

## Best Possible Solution

Make the Telegram behavior explicit: document the quote-reply/tool-progress trade-off and preferably add an `openclaw doctor` warning for configs where `replyToMode` will prevent the preview stream; treat separate persistent-message fallback as later product work.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a turn with a quotable inbound message; current source shows no answer-lane stream is created, and the issue discussion includes release-build confirmations.

## Solution Assessment

Yes. Keeping this issue open while reviewing the linked PRs is the narrowest maintainable path; docs plus a doctor warning directly address the silent doc/runtime mismatch without changing Telegram delivery semantics.
