## Summary

Keep open. Current main still has the Telegram native-quote gate that prevents the answer preview stream from being created, while the Telegram and streaming docs still describe tool-progress previews as default-on without the reply-to caveat and doctor preview warnings do not surface this config conflict. The issue is already paired with open closing PRs #73495 and #73508, so it should remain open until maintainers choose and land a fix.

## What This Changes

The issue requests fixing the Telegram doc/runtime mismatch where `replyToMode` values other than `off` can silently disable tool-progress preview updates.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally including both user-facing docs and an `openclaw doctor` warning for configs where native quote replies can block preview tool-progress. Treat a persistent-message fallback as a separate runtime/product decision if maintainers want tool visibility even with quote replies enabled.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the issue discussion includes released-build confirmation from the reporter and Theo-jobs.

## Solution Assessment

Yes for the near-term direction. A docs plus doctor-warning fix is the narrowest maintainable way to resolve the silent doc/runtime mismatch; a runtime fallback to separate persistent messages would be larger behavior work and should be considered separately.
