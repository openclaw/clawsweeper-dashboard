## Summary

Keep open. Current main still has the Telegram native-quote gate that prevents creating the answer preview stream, while the docs and doctor warnings still do not surface the reply-to/tool-progress conflict. The issue is also paired with open closing PRs #73495 and #73508, so it should remain open until maintainers choose and land one of them.

## What This Changes

The issue requests that Telegram document, warn about, or otherwise handle the case where native reply-to mode silently prevents preview tool-progress updates from rendering.

## Best Possible Solution

Land the preferred Telegram-scoped clarification so users can discover that native quote replies and preview tool-progress are mutually exclusive today, ideally including both public docs and an `openclaw doctor` warning. Treat a persistent-message fallback as a separate runtime/product decision if maintainers want tool visibility while keeping native quote replies enabled.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger a tool-using turn on a quotable inbound message; current source shows the answer stream is not created, and the issue discussion includes two released-build confirmations.

## Solution Assessment

Yes for the near-term fix direction. The runtime exclusion appears intentional for Telegram native quote replies, so documenting it and adding a doctor warning is the narrowest maintainable fix for the silent doc/runtime mismatch; changing delivery to a separate-message fallback should be a follow-up product choice.
