## Summary

Keep open: current main still lacks the requested session-level model-error circuit breaker, so the PR is not obsolete, but the proposed patch is not ready to merge because it misses an advertised integration path and has correctness bugs in cron failure accounting and isolated heartbeat session ownership.

## What This Changes

The PR adds a configurable agent/session circuit-breaker module, schema/type/session-state fields, design doc and tests, plus pause/reset/alert handling in heartbeat and isolated cron paths.

## Best Possible Solution

Keep a PR or replacement open until OpenClaw has a core-owned session breaker with documented config, generated schema/docs alignment, persisted per-session state, structured model-failure signals across interactive replies, heartbeat, cron, and auto-resume paths, and correct ownership semantics for isolated versus base sessions.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing feature by showing no session-level circuitBreaker config/state/enforcement, and the PR-specific problems are statically reproducible from the proposed diff plus current cron payload and heartbeat isolated-session behavior.

## Solution Assessment

No, not as currently proposed. The maintainable direction is to introduce a structured error signal at the reply/run boundary and wire one shared breaker policy through all session-owning run paths before enabling reset/pause/alert actions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Count cron error payloads before clearing breaker state:** `src/cron/isolated-agent/run.ts:787-794`
  - body: This clears the breaker whenever `finalRunResult.meta?.error` is absent, but cron can still return fatal timeout or error payloads that are converted into job errors later. Those failures would reset the accumulated counter, so repeated timeouts never trip the breaker.
  - confidence: 0.86
- **[P2] Wire the interactive reply path or narrow the design:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:81-87`
  - body: The design lists `getReplyFromConfig()` as a required integration point, but the patch only wires heartbeat and isolated cron. Normal inbound replies can hit the same repeated model-error loop and will bypass pause/reset/alert entirely.
  - confidence: 0.82
- **[P2] Avoid resetting the main session for isolated heartbeat failures:** `src/infra/heartbeat-runner.ts:746-754`
  - body: When `heartbeat.isolatedSession` is enabled, the model call runs against the synthetic heartbeat session, but this block records and executes actions against the preflight base `entry` and `sessionKey`. A breaker `reset` can rotate the user's real conversation because only an isolated heartbeat run failed.
  - confidence: 0.76
- **[P2] Use structured error state instead of warning-prefix matching:** `src/infra/heartbeat-runner.ts:742-744`
  - body: Heartbeat error detection depends on `replyPayload.text` starting with the warning glyph. Current reply normalization can prepend a configured response prefix, and `ReplyPayload` already carries `isError`, so prefixed model-error replies can miss the breaker while ordinary warning-style text can be counted incorrectly.
  - confidence: 0.72
