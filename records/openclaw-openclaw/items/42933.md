Full review comments:

- **[P1] Track breaker state on the heartbeat run session:** `src/infra/heartbeat-runner.ts:748-753`
  - body: When `heartbeat.isolatedSession` is enabled, the model run uses `${sessionKey}:heartbeat`, but this new block records and executes breaker actions against the main `entry` and `sessionKey`. A failed ephemeral heartbeat can therefore pause or reset the user's real conversation, while the actual isolated run session is recreated each time.
  - confidence: 0.91
- **[P2] Avoid leading-glyph error detection:** `src/infra/heartbeat-runner.ts:745-746`
  - body: This only counts non-throwing model failures when the final reply text starts with `⚠️`. Replies can already have a configured response prefix prepended before this point, so prefixed failure payloads are missed; normal content beginning with that glyph can also be counted. Use structured failure metadata or an error flag before prefixing instead.
  - confidence: 0.84
