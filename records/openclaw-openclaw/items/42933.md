## Summary

Keep open. Current main still lacks the requested configurable session-level breaker, and this branch is not mergeable as-is because it misses a normal reply integration path, stores isolated heartbeat breaker state on the wrong session, mishandles timeout payloads, and lacks real behavior proof.

## What This Changes

This PR adds an `agents.*.circuitBreaker` config/schema/session-state module with pause/reset/alert actions, heartbeat and isolated-cron wiring, unit tests, and a design note.

## Best Possible Solution

Define a durable gateway/session-health contract, then either rework this branch around correct reply/heartbeat/cron state ownership with real proof or replace it with a narrower implementation tracked by https://github.com/openclaw/openclaw/issues/62615.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR blockers by source inspection: the normal reply path is omitted, isolated heartbeat state is recorded on `entry`/`sessionKey`, and cron clears breaker state for timeout payloads without `meta.error`; no live fleet reproduction was run.

## Solution Assessment

Is this the best way to solve the issue? Not yet; a session-level breaker is directionally reasonable, but this implementation needs structured error metadata across reply paths, correct isolated-session ownership, public config/docs alignment, and maintainer scope decisions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wire the breaker into normal reply runs:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:83-85`
  - body: The design lists `getReplyFromConfig()` as a required integration point, but the patch never changes that path. Normal interactive and auto-reply model failures still go straight to `runPreparedReply()` without a breaker check or recorder, so the advertised session-level breaker only protects heartbeat and cron runs.
  - confidence: 0.9
- **[P1] Record isolated heartbeat state on the run session:** `src/infra/heartbeat-runner.ts:747-753`
  - body: When `heartbeat.isolatedSession` is enabled, the model turn runs under `runSessionKey`, but this code records errors and executes reset/pause against the base `entry` and `sessionKey`. A breaker reset can rotate the user's real conversation for an ephemeral heartbeat failure, and pause state is stored on the wrong session.
  - confidence: 0.92
- **[P1] Count timeout payloads before clearing cron breaker state:** `src/cron/isolated-agent/run.ts:779-794`
  - body: This branch treats only `meta.error` as a non-throwing model failure and clears breaker state otherwise. The embedded runner can return a fatal timeout as an `isError` payload without `meta.error`, after which cron still returns an error, so repeated timeout failures never reach the breaker threshold.
  - confidence: 0.86
- **[P2] Replace emoji-prefix model error detection:** `src/infra/heartbeat-runner.ts:745-746`
  - body: Classifying heartbeat model failures with `replyText.startsWith("⚠️")` is brittle because reply normalization can prepend a configured response prefix, producing prefixed fallback errors that miss breaker increments while warning-style content can be counted as a model failure.
  - confidence: 0.82
