## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Record isolated heartbeat breaker state on the run session:** `src/infra/heartbeat-runner.ts:748-753`
  - body: When `heartbeat.isolatedSession` is enabled, the model call runs on `runSessionKey = ${sessionKey}:heartbeat`, but this block records errors and executes `reset`/`pause` against the main `entry` and `sessionKey`. A failing ephemeral heartbeat can therefore rotate the user's real conversation or pause future heartbeats based on the wrong session.
  - confidence: 0.9
- **[P2] Do not clear cron breaker state for timeout error payloads:** `src/cron/isolated-agent/run.ts:792-794`
  - body: This branch clears the breaker whenever `meta.error` is absent, but `runEmbeddedPiAgent` can return fatal prompt timeouts as `payloads[].isError` without `meta.error`; cron later reports those as errors. Repeated timeout payloads will keep clearing the counter and never trip the breaker.
  - confidence: 0.88
- **[P2] Stop keying heartbeat errors on the rendered warning prefix:** `src/infra/heartbeat-runner.ts:745-747`
  - body: The breaker only counts replies whose rendered text starts with the warning prefix. Response prefixes can be prepended before this check, and ordinary content can also start with that glyph, so deployments can either miss real model fallback errors or count normal replies as breaker failures.
  - confidence: 0.82
- **[P2] Clear heartbeat dedupe state during breaker resets:** `src/agents/circuit-breaker/state.ts:127-142`
  - body: `resetSession` is documented as `/new`-equivalent, but it leaves `lastHeartbeatText` and `lastHeartbeatSentAt` intact. After a breaker reset, the first successful heartbeat can still be suppressed by the old duplicate-delivery key.
  - confidence: 0.84
