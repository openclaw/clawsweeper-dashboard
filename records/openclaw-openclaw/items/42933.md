## Summary

Keep open: current main still does not implement the proposed configurable session-level circuit breaker, but this broad PR has blocking correctness issues, missing real behavior proof, and overlaps the open canonical feature thread for gateway/session unhealthy-state design.

## What This Changes

The PR adds an `agents.*.circuitBreaker` config surface with persisted `cb*` session state, pause/reset/alert actions, heartbeat and isolated-cron integration, tests, and a design note.

## Best Possible Solution

Define the durable session-health contract in the gateway/session lifecycle, then either rework this PR around that contract with source fixes and real proof or replace it with a narrower implementation tied to https://github.com/openclaw/openclaw/issues/62615.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR blockers by source inspection: the branch omits the normal reply integration, records isolated heartbeat failures on the base session, and clears cron breaker state for timeout payloads without `meta.error`; no live fleet reproduction was run.

## Solution Assessment

Is this the best way to solve the issue? Not yet; a session-level breaker is directionally reasonable, but this implementation needs structured error metadata across reply paths, correct isolated-session ownership, public config/docs alignment, and maintainer scope decisions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wire the breaker into normal reply runs:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:83-87`
  - body: The design lists `getReplyFromConfig()` as a required integration point, but the patch never changes that path. Normal interactive and auto-reply model failures still go straight to `runPreparedReply()` without a breaker check or recorder, so the advertised session-level breaker only protects heartbeat and cron runs.
  - confidence: 0.9
- **[P1] Record isolated heartbeat state on the run session:** `src/infra/heartbeat-runner.ts:747-753`
  - body: When `heartbeat.isolatedSession` is enabled, the model turn runs under `runSessionKey`, but this code records errors and executes reset/pause against the base `entry` and `sessionKey`. A breaker reset can rotate the user's real conversation for an ephemeral heartbeat failure, and pause state is stored on the wrong session.
  - confidence: 0.92
- **[P1] Count timeout payloads before clearing cron breaker state:** `src/cron/isolated-agent/run.ts:779-794`
  - body: This branch treats only `meta.error` as a non-throwing model failure and clears breaker state otherwise. `runEmbeddedPiAgent` can return a fatal timeout as an `isError` payload without `meta.error`, after which cron still returns an error, so repeated timeout failures never reach the breaker threshold.
  - confidence: 0.86
- **[P2] Avoid emoji-prefix model error detection:** `src/infra/heartbeat-runner.ts:745-746`
  - body: Classifying heartbeat model failures with `replyText.startsWith("⚠️")` is brittle because reply normalization can prepend a configured `responsePrefix`, producing `<prefix> ⚠️ ...` for real fallback errors. Those deployments will miss breaker increments, while legitimate warning-style content can be counted as a model failure.
  - confidence: 0.82
