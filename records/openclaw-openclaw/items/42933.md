## Summary

Keep open. Current main still lacks a session-level model-error circuit breaker, and this PR is an open broad core resilience change with concrete correctness issues in the proposed implementation, including missed cron error payloads and incorrect heartbeat session ownership.

## What This Changes

The PR adds a new agent/session circuit-breaker module, config/schema/session fields, unit tests, and pause/reset/alert handling in heartbeat and isolated cron paths.

## Best Possible Solution

Keep this PR or a narrower replacement open until OpenClaw has a core-owned session-level breaker with documented config, persisted per-session state, structured model-failure metadata across heartbeat, cron, interactive reply, and auto-resume paths, plus correct ownership for isolated run sessions.

## Reproduction Assessment

Yes. Static source inspection reproduces the missing feature on current main, and the PR-specific bugs are reproducible from the remote PR head plus current tests showing fatal timeout payloads can have isError without meta.error.

## Solution Assessment

No. The proposed direction is plausible, but this patch is not the best implementation yet because it relies on incomplete integration and partial error signals, and it can mutate the wrong session for isolated heartbeat runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Record payload errors before clearing breaker state:** `src/cron/isolated-agent/run.ts:792-794`
  - body: This clears the breaker whenever `meta.error` is absent, but embedded cron runs can still surface fatal timeout failures as `payloads[].isError` with no `meta.error`. Those runs are treated as cron errors later, so this branch can erase the accumulated counter and repeated timeouts never trip the breaker.
  - confidence: 0.86
- **[P2] Wire the interactive reply path or narrow the design:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:83-85`
  - body: The design lists `getReplyFromConfig()` as a required integration point, but the patch only wires heartbeat and isolated cron. Normal inbound replies can hit the same repeated model-error loop and will bypass pause/reset/alert entirely, so the feature is not actually session-level as documented.
  - confidence: 0.82
- **[P2] Target isolated heartbeat breaker actions to the run session:** `src/infra/heartbeat-runner.ts:747-754`
  - body: When `heartbeat.isolatedSession` is enabled, the model call uses a fresh `:heartbeat` session, but this block records and executes breaker actions against the base `entry` and `sessionKey`. A `reset` action can rotate the user's real conversation because only an ephemeral heartbeat run failed.
  - confidence: 0.74
