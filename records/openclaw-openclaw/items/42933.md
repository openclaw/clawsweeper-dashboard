## Summary

Keep open: current main has narrower timeout/failover safeguards, but it does not implement this PR's configurable session-level pause/reset/alert breaker, and the proposed patch still has correctness blockers plus missing real-behavior proof.

## What This Changes

The PR adds a configurable agent/session circuit-breaker config, persisted session state, docs/tests, and heartbeat/cron pause/reset/alert handling after consecutive model errors.

## Best Possible Solution

Define one durable session-health contract across fallback, heartbeat, cron, interactive replies, and operator recovery surfaces, then either revise this PR or replace it with a narrower implementation aligned with https://github.com/openclaw/openclaw/issues/62615.

## Reproduction Assessment

Yes for review purposes. Source inspection of the PR head shows the review findings, and source inspection of current main shows the requested configurable session-level breaker is still absent.

## Solution Assessment

No. The direction is useful, but this patch is not the best maintainable solution until it uses structured failure signals, covers interactive reply paths, and applies breaker state to the correct session owner.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Count cron fatal payloads before clearing breaker state:** `src/cron/isolated-agent/run.ts:792-794`
  - body: This clears the breaker whenever meta.error is absent, but the same path later turns fatal payloads[].isError into status: "error". Repeated payload-only timeout or fatal agent failures will reset the counter each time and never trip the breaker.
  - confidence: 0.86
- **[P2] Record isolated heartbeat failures on the isolated session:** `src/infra/heartbeat-runner.ts:750-753`
  - body: When heartbeat.isolatedSession is enabled, the run uses runSessionKey, but this block mutates entry and executes actions against sessionKey. A reset or pause can therefore affect the user's base conversation even though only the synthetic heartbeat session failed.
  - confidence: 0.78
- **[P2] Wire the interactive reply path or narrow the design:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:81-85`
  - body: The design lists getReplyFromConfig as a required integration point, but this patch only wires heartbeat and isolated cron. Normal inbound replies can hit the same repeated model-error loop and bypass pause/reset/alert entirely.
  - confidence: 0.82
