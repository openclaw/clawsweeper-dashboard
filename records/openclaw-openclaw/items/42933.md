## What This Changes

The PR adds circuit-breaker config, session-state fields, tests, and heartbeat/cron integration for pause, reset, and alert actions after consecutive model errors.

## Best Possible Solution

Rework the feature around the actual run session and structured model-error metadata, document the config contract, and prove it with focused regression tests plus redacted real fleet or alert-delivery evidence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Track isolated heartbeat failures on the run session:** `src/infra/heartbeat-runner.ts:747-753`
  - body: When `heartbeat.isolatedSession` is enabled, the model turn runs on `runSessionKey`, but this code records and persists breaker state on the base `entry`/`sessionKey`. A failed ephemeral heartbeat can therefore pause or reset the user's real session while not tracking the failing isolated session that actually ran.
  - confidence: 0.9
- **[P2] Avoid warning-prefix heartbeat error detection:** `src/infra/heartbeat-runner.ts:745-746`
  - body: This classifies model failures by checking whether the final text starts with `⚠️`, but reply normalization can prepend `responsePrefix` before the heartbeat runner receives the payload. Deployments with a response prefix will miss real fallback errors, so repeated failures will never trip the breaker.
  - confidence: 0.86
- **[P1] Keep cron breaker state for fatal error payloads:** `src/cron/isolated-agent/run.ts:792-794`
  - body: This clears breaker state whenever `meta.error` is absent, but embedded runs can return a fatal timeout as an `isError` payload without `meta.error`; later cron finalization turns that into a job error. Repeated timeouts then clear the counter every run instead of reaching the threshold.
  - confidence: 0.88
- **[P2] Clear heartbeat dedupe fields on reset:** `src/agents/circuit-breaker/state.ts:126-142`
  - body: `resetSession` is documented as the `/new` equivalent, but it leaves `lastHeartbeatText` and `lastHeartbeatSentAt` intact. After a breaker reset, duplicate suppression can drop the first successful heartbeat if it repeats the previous summary.
  - confidence: 0.78
