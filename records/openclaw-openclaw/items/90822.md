## Label Justifications

- `P2`: The original report described a normal-priority Discord guild response-delivery regression with limited channel blast radius.

## Summary

Close as implemented on main and shipped: the reported 2026.5.6 Discord guild silence matches the old tool-only group/channel default, and current source plus shipped releases now default normal group/channel replies back to automatic delivery unless the user explicitly opts into message-tool-only mode.

## What This Changes

The issue reports Discord guild/channel messages being processed and answered by the model, but the generated response not being delivered back to Discord.

## Best Possible Solution

Keep the shipped automatic group/channel default; if a user still sees this on v2026.5.18 or newer, collect active config plus verbose suppression/delivery logs and open a fresh narrowed regression report.

## Reproduction Assessment

Yes for the historical v2026.5.6 path: shipped source resolved unset group/channel visible replies to message-tool-only, matching didSendViaMessagingTool=false when a model answered privately. Current main and shipped v2026.5.18+ change that default to automatic.

## Solution Assessment

Yes. The shipped fix is the narrowest maintainable solution because it restores automatic visible delivery for normal group/channel turns while preserving explicit message-tool-only behavior for users who opt into it.
