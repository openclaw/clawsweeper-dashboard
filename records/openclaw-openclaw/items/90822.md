## Summary

Close as implemented: the reported v2026.5.6 Discord guild/channel silent-final symptom is covered by the shipped group/channel visible-reply default fix, and the reporter’s attempted v2026.6.1 update was not confirmed to have restarted the gateway onto the fixed version.

## What This Changes

Discord guild/channel mentions should deliver generated assistant responses back to the channel instead of leaving final text private with `didSendViaMessagingTool: false`.

## Best Possible Solution

Keep the shipped v2026.6.1+ behavior: normal group/channel turns deliver final replies automatically, while explicit `message_tool` and room-event modes remain opt-in quiet-room paths.

## Reproduction Assessment

No against current main for the default/unset group-channel configuration. Source and tests show Discord guild turns now resolve to automatic final delivery unless the operator explicitly opts into message-tool-only visible replies.

## Solution Assessment

Yes. The shipped shared group/channel default fix is the narrow maintainable solution; a new Discord-specific delivery shim is not needed unless a verified v2026.6.1+ automatic-mode repro appears.
