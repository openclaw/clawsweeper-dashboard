Close as implemented on main. The reported 2026.5.6 Discord guild-channel silence matches the old tool-only visible-reply default; the current code now defaults normal group/channel replies back to automatic delivery, Discord guild turns flow through that resolver, and the fix shipped in v2026.5.18.

## What This Changes

Report that Discord guild-channel messages are received and answered by the model, but the generated response is not delivered back to Discord and the session shows didSendViaMessagingTool: false.

## Best Possible Solution

Keep the automatic group/channel visible-reply default, and use explicit message_tool only for intentional lurk or ambient rooms; any remaining report should include active config and verbose sourceReplyDeliveryMode logs after upgrade/restart.

## Reproduction Assessment

No live current-main reproduction was established. The historical failure mode is source-reproducible from the old group/channel tool-only default, and current main should not reproduce it for normal guild-channel requests because the default now resolves to automatic delivery.

## Solution Assessment

Yes. The best fix was the merged default change that restores automatic final delivery for normal group/channel requests while keeping message_tool as an explicit opt-in for ambient or lurk-mode rooms.
