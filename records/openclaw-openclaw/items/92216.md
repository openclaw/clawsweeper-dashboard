## Summary

Keep this PR open for a focused repair. The reported hidden-session commentary gap is real and the supplied proof is strong, but the current classifier also treats every unphased assistant delta as commentary even though existing producers use unphased deltas for ordinary assistant output.

## What This Changes

The PR mirrors eligible hidden assistant events to exact session-message subscribers, adds focused gateway regression coverage, and documents a provider-to-channel agent-event contract.

## Best Possible Solution

Define assistant phases at the producer or normalization boundary so commentary deltas are explicitly marked and final-answer deltas are explicitly excluded, then make the hidden gateway mirror consume that closed contract and align the tests and documentation.

## Reproduction Assessment

Yes. Current source shows the omitted hidden assistant path, and the inspected real ClickClack evidence demonstrates the missing preamble before and restored commentary/tool preamble after.

## Solution Assessment

No, not as written. The gateway subscriber boundary is appropriate, but a bare delta is not a safe commentary classification under current producer contracts; explicit phase normalization is the narrower maintainable solution.
