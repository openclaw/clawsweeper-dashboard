## Summary

Keep open: current main and the latest release still read the hosted docs search response with unbounded `response.json()`, while this PR applies the existing bounded-read helper at that external response boundary and adds the requested overflow regression coverage. I found no blocking correctness or security findings.

## What This Changes

The PR replaces the docs search command's raw JSON response read with an 8 MiB `readResponseWithLimit` read and adds a regression test for oversized docs search responses.

## Best Possible Solution

Land this focused bounded-read change after required CI finishes, preserving the existing CLI output and keeping the cap local to the hosted docs search response.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` call `response.json()` after a successful docs search response, and the PR's stream test provides a high-confidence oversized-response path without needing a hostile live endpoint.

## Solution Assessment

Yes. Reusing `readResponseWithLimit` at the hosted docs response boundary is the narrowest maintainable fix and matches existing provider JSON and self-hosted discovery reader patterns.
