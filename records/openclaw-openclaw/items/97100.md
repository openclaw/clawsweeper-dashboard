## Summary

Keep this PR open. The reset-aware Slack thread reseeding direction is useful and real Slack proof is sufficient, but the current patch still breaks exported/direct Slack monitor callers that omit `channelRuntime` by treating every existing thread reply as an initial thread turn.

## What This Changes

The PR adds a reset-freshness helper for session entries, wires it through the host channel runtime, and uses it in Slack prepare so stale thread sessions can reseed bounded thread history after reset.

## Best Possible Solution

Preserve the no-`channelRuntime` existing-thread fallback, keep the reset-aware gateway behavior, and land the Slack/session fix only after focused compatibility coverage passes.
