Full review comments:

- **[P1] Stop dry-run from running non-interactive migrations:** `src/flows/doctor-health.ts:28`
  - body: Preview mode clears `repair` and `yes`, but it keeps `nonInteractive` set. `runLegacyStateHealth` treats `nonInteractive` as approval and calls the legacy state migrator, so `openclaw doctor --dry-run --non-interactive` can still mutate session/agent/WhatsApp state. Dry-run needs to block or explicitly preview that path before exposing the flag.
  - confidence: 0.91
- **[P1] Report legacy config repair gaps in preview JSON:** `src/flows/doctor-health.ts:88`
  - body: `--dry-run --json` runs the legacy config flow with repair disabled and suppresses its human notes, but nothing adds pending config repairs or unsupported legacy paths to `previewReport`. A config that `doctor --fix` would rewrite can therefore produce a clean-looking JSON preview with no skipped entry or diff. Either collect these repairs or make the structured-only limitation explicit in the report.
  - confidence: 0.86
- **[P2] Redact preview diffs before output:** `src/flows/doctor-health.ts:136`
  - body: The preview report is written directly as JSON and the report can contain free-form `before`, `after`, or `unifiedDiff` strings from repair checks. Before `--diff --json` becomes a public Doctor path, the output boundary needs redaction or a narrower diff contract so config or SecretRef-adjacent repairs cannot leak sensitive values.
  - confidence: 0.74
