- **[P1] Keep dry-run out of write paths:** `src/flows/doctor-health.ts:28`
  - body: In preview mode this rewrites the options to `repair: false`, `yes: false`, and `generateGatewayToken: false`, but then runs the normal config flow and Doctor contribution list. When config repairs are pending, `finalizeDoctorConfigFlow` can still prompt and return `shouldWriteConfig: true`, and `runWriteConfigHealth` will write because it has no dry-run guard. `openclaw doctor --dry-run` and TTY `--dry-run --json` must not be able to apply changes after a prompt; pass an explicit no-write dry-run mode through finalization and the write contribution.
  - confidence: 0.93
- **[P2] Report skipped legacy preview surfaces:** `src/flows/doctor-health.ts:78-80`
  - body: The PR adds a `skipped` array and recorder for preview reports, but the recorder has no call sites, and the one legacy repair path explicitly skipped in preview is silently omitted. Operators reading `--dry-run --json` will not know that UI protocol repair effects, and any other legacy-only repair surfaces, were not previewed. Either run the structured effect path for those contributions or record skipped entries.
  - confidence: 0.82
