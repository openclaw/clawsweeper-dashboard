## Summary

Keep open: this draft is still a useful candidate for the Doctor dry-run/diff feature, but current head can still let a dry-run use prompt-capable Doctor paths that may write config or state, and the new public CLI contract is undocumented and not proven by a successful real CLI run.

## What This Changes

This PR adds `openclaw doctor --dry-run` and `--diff`, threads structured repair preview reports into Doctor health contributions, and adds focused CLI/contribution tests.

## Best Possible Solution

Land one command-wide Doctor preview contract that cannot prompt or write, documents `--dry-run`/`--diff`/JSON/exit behavior, and includes redacted real CLI no-mutation proof before closing the canonical issue.

## Reproduction Assessment

Yes for source-level review. In a TTY dry-run, the PR still creates a prompt-capable Doctor prompter and passes it into config/contribution paths whose accepted prompts can set `shouldWriteConfig` or mutate `ctx.cfg`, after which the write-config contribution has no dry-run guard.

## Solution Assessment

No. The direction is useful, but the current patch is not the safest fix until dry-run becomes a command-wide no-prompt/no-write mode with docs and real CLI proof.
