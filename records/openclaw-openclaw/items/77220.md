## Summary

Keep open: the patch still changes a valid allowlist diagnostic path for names that are both plugin ids and agent tools, so it needs a narrow fix before merge.

## What This Changes

The PR adds manifest `contracts.tools` lookup to missing CLI command diagnostics, wires it into `runCli`, adds unit coverage, and adds a changelog entry.

## Best Possible Solution

Keep the manifest-backed agent-tool diagnostic, but make it subordinate to known plugin ids or CLI roots and cover the `browser` collision with a regression test.

## Reproduction Assessment

Yes, from source inspection. Current main reaches the misleading `plugins.allow` fallback for unknown tool-like roots, and the PR branch has a clear collision path where `browser` is both a plugin command root and a declared agent tool.

## Solution Assessment

No as currently implemented. Manifest `contracts.tools` is the right seam, but tool-owner diagnostics must skip known plugin ids or CLI roots to preserve existing allowlist guidance.
