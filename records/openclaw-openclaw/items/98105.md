## Summary

Keep open, but not merge-ready: the token rollup direction matches the linked auth-provider bug, while the new explicit-order fallback can bypass documented hard `auth.order` filtering for existing but incompatible ordered profile ids.

## What This Changes

The PR updates auth-health rollup logic so token-type claude-cli profiles can contribute to effective provider health when stale OAuth ordering or OAuth-only aggregation would otherwise report missing auth.

## Best Possible Solution

Keep the token-status repair, but narrow the explicit-order fallback to stale missing ordered profile ids so invalid existing ordered profiles still report missing.

## Solution Assessment
