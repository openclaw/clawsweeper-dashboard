## Repair Work Prompt

Fix the Control UI cron quick-create Silent delivery preset. Current main has draftToCronFormPatch set every quick-create payload to agentTurn, then Silent sets sessionTarget main and deliveryMode none; addCronJob forwards that to cron.add, where src/cron/service/jobs.ts rejects main non-systemEvent jobs. Change the quick-create mapping so Silent creates a backend-supported silent cron job, likely payloadKind systemEvent with sessionTarget main and deliveryMode none, while preserving Independent as isolated agentTurn with no delivery. Add or update focused regression tests for draftToCronFormPatch and the cron.add payload path. Do not relax assertSupportedJobSpec or add config/compat surfaces. Related silent-cron runtime issues are adjacent and should not be fixed in this PR.
