## What This Changes

The issue reports that choosing Silent in the Cron quick-create dialog makes Create do nothing instead of creating the scheduled job and closing the dialog.

## Best Possible Solution

Land one narrow quick-create fix with regression coverage for the Silent preset's backend-supported job shape, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection shows Silent quick-create creates a `main` `agentTurn` cron job, `addCronJob` sends it to `cron.add`, and the backend rejects main-session cron jobs unless the payload is `systemEvent`.

## Solution Assessment

Yes for keeping this issue open. The narrow repair belongs in the quick-create mapping and its regression test, while the exact accepted preset semantics should be settled during review of the linked PRs.
