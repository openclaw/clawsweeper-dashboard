## What This Changes

The issue reports that the Control UI cron quick-create dialog fails to create a job when the Silent delivery preset is selected.

## Best Possible Solution

Make the Silent quick-create preset emit a backend-supported silent cron job shape and add a focused regression test for the preset-to-cron.add payload.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Silent quick-create creates a main-session agentTurn job, and the backend rejects main-session cron jobs unless the payload is systemEvent.

## Solution Assessment

Yes, the best repair is at the quick-create preset mapping or its validation, not by relaxing the backend main-session cron contract that keeps agentTurn jobs isolated by default.

The current preset emits payloadKind="agentTurn", sessionTarget="main", and deliveryMode="none", but src/cron/service/jobs.ts rejects main-session cron jobs unless payload.kind is systemEvent. Add focused coverage so the Silent preset produces a supported cron.add shape and the existing Notify preset behavior stays unchanged. No CHANGELOG.md edit.
