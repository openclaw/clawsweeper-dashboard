## What This Changes

The issue reports that selecting Silent in the Control UI cron quick-create dialog leaves Create inert instead of creating the cron job and closing the dialog.

## Best Possible Solution

Land one narrow Silent quick-create mapping fix, preferably the open linked PR after proof and checks, so the UI emits a backend-supported main-session system event.

## Reproduction Assessment

Yes, source-level. Current main maps Silent quick-create to a main-session `agentTurn` job, and the backend rejects main-session cron jobs unless the payload is `systemEvent`; I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

Yes. Fixing the Silent quick-create preset to emit `systemEvent` is narrower than relaxing the backend main-session cron contract or changing `addCronJob` globally.
