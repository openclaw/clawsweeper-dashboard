## Summary

Keep open: current main still has a source-reproducible Control UI quick-create bug where the Silent preset builds a main-session agent-turn cron job, which the cron service rejects, leaving the modal open after Create.

## What This Changes

The issue reports that selecting Silent in the browser Cron quick-create dialog leaves Create inert instead of creating the job and closing the dialog.

## Best Possible Solution

Change the quick-create Silent preset to generate a supported no-delivery cron shape, preserving Notify as isolated announce and Independent session as isolated no-delivery.

## Reproduction Assessment

Yes by source inspection: the reported quick-create Silent path builds a main-session agentTurn form, and the cron service rejects main jobs unless the payload is a system event. I did not run a browser E2E in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to make the quick-create Silent preset emit a supported no-delivery cron shape and cover that mapping in focused quick-create/controller tests.
