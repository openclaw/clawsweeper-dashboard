## Summary

Keep open: current main still sends fixed workspace bootstrap files through the workspace-root guarded reader, so an outside-workspace symlinked SOUL.md is rejected and rendered as missing; the active implementation path is the open scoped PR at https://github.com/openclaw/openclaw/pull/73401.

## What This Changes

Workspace bootstrap file injection should load symlinked standard files such as SOUL.md instead of marking them missing.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/73401 with a narrowly scoped fixed-bootstrap symlink reader that preserves the existing boundary protections everywhere else.

## Reproduction Assessment

Yes, by source inspection. A sibling-workspace SOUL.md symlink resolves outside the workspace root, openRootFile rejects the boundary escape, and loadWorkspaceBootstrapFiles converts that failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the best direction is a narrow exception in the fixed workspace bootstrap reader rather than relaxing @openclaw/fs-safe or boundary-path globally. The exact allowed external target scope still needs maintainer security review.
