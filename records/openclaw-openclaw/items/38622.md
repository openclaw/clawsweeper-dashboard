## Summary

Keep open. Current main still rejects symlinked fixed workspace bootstrap files whose real targets escape the workspace boundary, matching the reported `[MISSING]` behavior; related implementation PRs are still unmerged, with #73401 open as the maintainer-labeled canonical path.

## What This Changes

The issue asks OpenClaw to load workspace bootstrap files such as `SOUL.md` through symlinks to shared files instead of injecting them as missing.

## Best Possible Solution

Land one canonical, maintainer-reviewed fix that follows explicit fixed workspace bootstrap symlinks to readable regular files, keeps existing safety guards intact, leaves extra/glob bootstrap boundaries unchanged, and adds focused regression coverage plus a changelog credit for the user-visible behavior change.

## Reproduction Assessment

Yes. The source path gives a high-confidence reproduction: a workspace `SOUL.md` symlink to a sibling/shared file is rejected by `resolveBoundaryPath`, returned as a validation failure by `openBoundaryFile`, converted into a missing bootstrap record, and rendered as `[MISSING] Expected at: ...`.

## Solution Assessment

Yes, the best fix direction is the scoped workspace-loader approach in #73401 or an equivalent successor, not a broad relaxation of generic boundary-file reads. Keep this issue open until that implementation lands and validates the fixed bootstrap-file behavior.
