## Label Justifications

- `P1`: The issue can silently strip identity, persona, and memory bootstrap context from every affected session in real user setups.
- `impact:session-state`: The failure changes the session's injected Project Context and agent identity state.
- `impact:security`: Any fix changes a local-file boundary for data entering model prompt context.

## Summary

Keep open: current main and latest release still route fixed workspace bootstrap files through a workspace-root `openRootFile` read, so external symlink targets are converted into missing Project Context entries. Related open PRs cover possible fixes, but the remaining decision is the maintainer/security policy for external local files entering prompt context.

## What This Changes

The issue asks OpenClaw to load symlinked workspace bootstrap files such as `SOUL.md` instead of injecting them as missing.

## Best Possible Solution

Keep the shared fs-safe/root-boundary helpers strict by default, then land or explicitly reject a scoped fixed-bootstrap symlink policy with clear blocked-symlink diagnostics and preserved hardlink, dangling-link, non-regular, oversized, private-state, and extra-bootstrap protections.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an external symlinked fixed bootstrap file fails the workspace-root `openRootFile` contract, is stored as missing, and renders as `[MISSING] Expected at: ...`; the discussion also includes an independent Linux reproduction.

## Solution Assessment

Unclear until maintainers settle the trust model. The scoped fixed-bootstrap approach in https://github.com/openclaw/openclaw/pull/73401 is narrower than the broad allowlist or shared-boundary alternatives, but any accepted path needs explicit security boundaries and diagnostics.
