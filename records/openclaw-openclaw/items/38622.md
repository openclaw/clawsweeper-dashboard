## Summary

Keep open: current main still routes workspace bootstrap files through the workspace-root guarded reader, so an outside-workspace symlinked `SOUL.md` is still converted into a missing bootstrap entry; the active implementation path is the open maintainer PR at https://github.com/openclaw/openclaw/pull/73401.

## What This Changes

Allow fixed workspace bootstrap files such as `SOUL.md` to load through symlinks instead of being injected as missing.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping the fix scoped to fixed bootstrap files and preserving the shared boundary-reader protections elsewhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a sibling-workspace symlinked `SOUL.md` resolves outside the workspace root, the guarded open rejects it under the root boundary, and `loadWorkspaceBootstrapFiles` converts the failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the narrow maintainable direction is the scoped fixed-bootstrap-file exception represented by the open maintainer PR, not relaxing `@openclaw/fs-safe` or the shared boundary primitive globally.
