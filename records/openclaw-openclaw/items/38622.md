## Summary

Keep open: current main and v2026.5.12 still route fixed workspace bootstrap files through a workspace-root boundary reader, so external symlink targets become missing Project Context entries. Open related PRs cover alternative fix paths, but the remaining decision is the security/product policy for which external local files may enter prompt context.

## What This Changes

Workspace bootstrap file injection should follow valid symlinked workspace files instead of rendering them as missing.

## Best Possible Solution

Keep this canonical report open until maintainers land or explicitly reject a scoped bootstrap-symlink policy with blocked-symlink diagnostics and preserved hardlink, size, non-regular-file, dangling-link, private-state, and extra-bootstrap protections.

Yes. Source inspection gives a high-confidence path: an external symlinked fixed bootstrap file fails the workspace-root boundary reader, is stored as missing, and renders as `[MISSING] Expected at: ...`.

## Solution Assessment

Unclear until maintainers settle the trust model.
