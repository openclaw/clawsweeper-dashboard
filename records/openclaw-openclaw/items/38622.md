## Summary

Keep open: current main still routes fixed workspace bootstrap files through the boundary reader, which rejects symlink targets outside the workspace root and turns failed reads into missing records that render as `[MISSING]`. There are already open implementation candidates for the same behavior, so this should be handled by reviewing or landing one canonical PR rather than closing the issue or spawning another repair lane.

## What This Changes

The issue asks for workspace bootstrap files such as `SOUL.md` to follow symlinks to shared files outside the workspace instead of being injected as missing.

## Best Possible Solution

Land one canonical, maintainer-reviewed fix that follows explicit fixed workspace bootstrap symlinks to readable regular files, keeps existing safety guards intact, leaves extra/glob bootstrap boundaries unchanged, and adds focused regression coverage in `src/agents/workspace.test.ts` plus a changelog credit if the user-visible behavior changes.
