## Summary

Keep open: current main still routes fixed workspace bootstrap files through the root-bounded guarded reader, so an outside-workspace symlinked `SOUL.md` is still converted into a missing bootstrap entry; the active implementation path is https://github.com/openclaw/openclaw/pull/73401.

## What This Changes

Allow fixed workspace bootstrap files such as `SOUL.md` to load through symlinks instead of being injected as missing.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping any symlink allowance scoped to fixed bootstrap files and preserving the existing boundary-reader protections elsewhere.

## Reproduction Assessment

Yes by source inspection. A sibling-workspace symlinked `SOUL.md` resolves outside the workspace root, the pinned boundary reader rejects that symlink escape, and `loadWorkspaceBootstrapFiles` converts the failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the narrow maintainable direction is to handle explicit fixed bootstrap-file symlinks in `src/agents/workspace.ts` rather than relaxing fs-safe globally. The final allowed target scope still needs maintainer/security review.
