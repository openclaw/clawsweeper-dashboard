## Summary

Keep open. Current main still routes fixed workspace bootstrap files through the boundary-file reader with no symlink fallback, so a symlinked `SOUL.md` whose real target is outside the workspace is still converted into the reported `[MISSING]` record; the open maintainer-labeled #73401 remains the canonical implementation path.

## What This Changes

The issue requests that workspace bootstrap file injection follow symlinks to readable shared files instead of rendering those entries as missing.

## Best Possible Solution

Land one maintainer-reviewed fix that follows explicit fixed bootstrap-file symlinks to readable regular files, preserves existing guards, leaves extra/glob bootstrap boundaries unchanged, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a symlinked fixed bootstrap file escaping the workspace boundary is rejected by `resolveBoundaryPath`, returned as a validation failure, marked missing by `loadWorkspaceBootstrapFiles`, and rendered as `[MISSING] Expected at: ...`.

## Solution Assessment

Yes. The narrow maintainable direction is the scoped workspace-loader fix in #73401 or an equivalent successor, not a broad relaxation of the shared boundary-file reader.
