## Summary

Keep open: current main still funnels fixed workspace bootstrap files through the workspace-root boundary reader, so a valid symlink whose target resolves outside the workspace is treated as a failed guarded read and then surfaced as missing. The active resolution path is the open targeted fix PR #73401, not closing this report early.

## What This Changes

Load workspace bootstrap files such as SOUL.md through user-created symlinks to shared files instead of injecting them as missing.

## Best Possible Solution

Land or close the active targeted bootstrap-reader PR after maintainer review, keeping the global boundary-path policy intact and limiting any fallback to explicit fixed bootstrap files with existing size, type, and hardlink protections.

## Reproduction Assessment

Yes. By source inspection, a sibling-workspace symlinked SOUL.md resolves outside the workspace root, trips the boundary symlink escape path, and is converted by `loadWorkspaceBootstrapFiles` into a missing bootstrap entry.

## Solution Assessment

Yes, the active targeted fix direction is the narrowest maintainable shape: handle explicit fixed bootstrap-file symlinks in `workspace.ts` while preserving the general boundary reader and leaving extra/glob bootstrap paths guarded unless maintainers choose otherwise.
