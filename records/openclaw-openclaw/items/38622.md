Keep this open: current main and the latest release still route fixed workspace bootstrap files through a workspace-root guarded read and render failed reads as missing, while the open maintainer implementation candidate is not merged and is currently conflicted with security/product review blockers.

## What This Changes

The issue reports that workspace bootstrap files such as `SOUL.md` are rendered as missing when they are symlinks to shared files outside the workspace root.

## Best Possible Solution

Keep this canonical issue open while maintainers refresh or replace the scoped bootstrap-reader fix, settle the external-symlink trust policy, and then close duplicate reports after the fix lands.

## Reproduction Assessment

Yes at source level: an external symlinked bootstrap file reaches `openRootFile` with the workspace root as the boundary, and a failed read becomes `missing: true`. The issue also has independent Linux reproduction details, but I did not run a filesystem repro in this read-only review.
