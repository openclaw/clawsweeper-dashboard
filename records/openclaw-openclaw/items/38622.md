## Best Possible Solution

Keep the issue open until the linked maintainer replacement lands with an explicit fixed-bootstrap symlink trust policy, or maintainers choose a different security-preserving design.

No, not as a blanket symlink-following change. The best path is a scoped fixed-bootstrap policy that preserves the shared boundary contract and has explicit maintainer security direction.
