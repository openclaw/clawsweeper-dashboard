## Summary

Keep open: current main still turns outside-workspace symlinked bootstrap files into missing entries, and the active fix path is an open maintainer-labeled PR that needs security/trust-model review.

## What This Changes

The issue requests loading workspace bootstrap files such as `SOUL.md` when they are symlinks to shared files outside the workspace directory.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping the exception scoped to fixed bootstrap files and preserving boundary-reader protections elsewhere.

## Reproduction Assessment

Yes. Source inspection is enough: a sibling-workspace symlinked `SOUL.md` resolves outside the workspace root, the guarded root open rejects it, and `loadWorkspaceBootstrapFiles` converts that failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the best direction is a narrow fixed-bootstrap-file exception in `src/agents/workspace.ts`, not a general relaxation of fs-safe or boundary-path behavior; the exact allowed target set still needs maintainer security judgment.
