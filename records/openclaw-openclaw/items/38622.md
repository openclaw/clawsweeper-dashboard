## Label Justifications

- `P1`: The bug can silently remove bootstrap identity, persona, and memory context from every session for affected symlink-based workspaces.
- `impact:session-state`: The failure changes injected Project Context and agent identity state at session startup.
- `impact:security`: Any fix changes a local-file boundary for content entering the model prompt.

## Summary

Keep open: current main and latest v2026.5.20 still route fixed workspace bootstrap files through the workspace-root guarded reader, so symlinks whose targets resolve outside the workspace become missing Project Context entries. The related open PRs cover plausible fixes, but the remaining decision is the security/product boundary for external local files entering prompt context.

## What This Changes

The issue asks OpenClaw to follow symlinked workspace bootstrap files such as SOUL.md when injecting Project Context instead of rendering them as missing.

## Best Possible Solution

Keep this issue as the canonical user-facing report, then land or reject a scoped symlink policy that preserves strict fs-safe defaults, adds clear blocked-symlink diagnostics, and covers fixed bootstrap files plus relevant guards in tests.

## Reproduction Assessment

Yes for source reproduction. Current main routes bootstrap reads through a workspace-root guard, fs-safe rejects symlink targets outside that root, and the loader renders rejected reads as missing entries.

## Solution Assessment

Unclear until maintainers settle the trust model. The scoped fixed-bootstrap approach in https://github.com/openclaw/openclaw/pull/73401 is narrower than broad boundary relaxation, while an opt-in allowlist may be preferable if external prompt-context reads should remain explicit.
