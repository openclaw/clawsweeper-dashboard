## Summary

Keep open: current main and the latest release still route fixed workspace bootstrap files through a root-bounded reader, and the active fix path is an open maintainer-labeled PR that needs trust-model/security review before this issue can close.

## What This Changes

The issue reports that symlinked workspace bootstrap files such as SOUL.md are injected as missing instead of following the symlink target.

## Best Possible Solution

Review, rebase or replace, and land https://github.com/openclaw/openclaw/pull/73401 with the exception scoped to fixed top-level bootstrap filenames while preserving hardlink, size, non-regular, dangling-link, private OpenClaw state, and extra-bootstrap protections.

## Reproduction Assessment

Yes. Source inspection is high-confidence: a sibling-workspace symlinked SOUL.md resolves outside the workspace root, the guarded root open rejects it, and loadWorkspaceBootstrapFiles converts that failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the best direction is a narrow fixed-bootstrap-file exception in src/agents/workspace.ts rather than relaxing fs-safe or boundary-path behavior globally. The exact allowed target scope still needs maintainer security judgment.
