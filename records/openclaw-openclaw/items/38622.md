## Summary

Keep open: current main still routes fixed workspace bootstrap files through the root-bounded guarded reader, so an outside-workspace symlinked SOUL.md becomes a missing bootstrap entry; the active fix path is the open maintainer PR at https://github.com/openclaw/openclaw/pull/73401.

## What This Changes

The issue requests symlinked workspace bootstrap files such as SOUL.md be loaded as content instead of marked missing.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping the change scoped to fixed bootstrap files and preserving the existing boundary-reader protections elsewhere.

## Reproduction Assessment

Yes. By source inspection, a sibling-workspace symlinked SOUL.md resolves outside the workspace root, the guarded open rejects it under the workspace boundary, and loadWorkspaceBootstrapFiles converts the failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the narrow maintainable direction is to handle explicit fixed bootstrap-file symlinks in src/agents/workspace.ts rather than relaxing fs-safe globally; the final allowed target scope still needs maintainer/security review.
