## Summary

Keep open: current main still marks outside-workspace symlinked bootstrap files as missing, and the active resolution path is the open maintainer PR at https://github.com/openclaw/openclaw/pull/73401 rather than closing this report before a fix merges.

## What This Changes

Workspace bootstrap file injection should follow valid symlinks for files such as SOUL.md instead of marking them as missing.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping the fix scoped to fixed bootstrap files and preserving the existing boundary-reader protections elsewhere.

## Reproduction Assessment

Yes. By source inspection, a sibling-workspace symlinked SOUL.md resolves outside the workspace root, the pinned boundary reader rejects that symlink hop, and `loadWorkspaceBootstrapFiles` converts the failed read into a missing bootstrap entry.

## Solution Assessment

Yes, the active targeted direction is the narrow maintainable fix: handle explicit fixed bootstrap-file symlinks in `src/agents/workspace.ts` while avoiding a global boundary-path relaxation; the final allowed scope still needs maintainer/security review.
