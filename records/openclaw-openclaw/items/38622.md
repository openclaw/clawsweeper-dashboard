## Summary

Keep open: current main and v2026.6.8 still read fixed workspace bootstrap files through a workspace-root fs-safe boundary and render rejected reads as missing, while the active maintainer PR is open, conflicting, and still needs security/product review.

## What This Changes

The issue reports that fixed workspace bootstrap files such as SOUL.md are rendered as missing when they are symlinks to files outside the workspace root.

## Best Possible Solution

Keep this issue open as the canonical report until the maintainer-owned bootstrap symlink fix is rebased, security-reviewed, and merged with tests for allowed shared files and rejected private/OpenClaw state roots.

## Reproduction Assessment

Yes. Source inspection plus a Linux user reproduction show current bootstrap reads use a workspace-root `openRootFile` boundary, and the fs-safe contract rejects symlink hops outside that root before the loader renders the file as missing.

## Solution Assessment

No final fix is on main yet. The best path is a narrow bootstrap-layer fix or approved fs-safe wrapper policy with explicit security tests, not broad boundary relaxation.
