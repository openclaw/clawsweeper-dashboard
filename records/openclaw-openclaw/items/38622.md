## Summary

Keep open: current main and v2026.5.12 still route fixed bootstrap files through workspace-root guarded reads, so a symlinked SOUL.md resolving outside the workspace is still rendered as missing; open implementation PRs should be reviewed rather than closing this report.

## What This Changes

The issue asks for workspace bootstrap file injection to follow symlinks to shared workspace files instead of rendering them as missing.

## Best Possible Solution

Land or explicitly supersede a scoped fixed-bootstrap reader fix with negative security coverage, while preserving the generic fs-safe boundary and extra-bootstrap protections.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a sibling-workspace symlinked SOUL.md reaches openRootFile under the workspace root, fails the root boundary contract, is marked missing by loadWorkspaceBootstrapFiles, and is rendered as [MISSING] Expected at: ... .

## Solution Assessment

Unclear until maintainers settle the trust model. A scoped fixed-bootstrap-reader exception like https://github.com/openclaw/openclaw/pull/73401 looks narrower than the broader allowedExternalPaths or ~/.openclaw boundary-relaxation alternatives, but it still needs security review.
