## Summary

Keep open. Current main still rejects fixed workspace bootstrap symlinks whose targets resolve outside the workspace boundary, and the open maintainer-labeled #73401 is the canonical implementation path to review or land.

## What This Changes

Workspace bootstrap files such as `SOUL.md` should load through symlinks to shared files instead of being injected as missing.

## Best Possible Solution

Land one maintainer-reviewed fix that follows explicit fixed bootstrap-file symlinks to readable regular files, preserves existing hardlink/size/non-regular protections, and keeps extra/glob bootstrap boundaries unchanged.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: an escaped fixed bootstrap-file symlink is rejected by the boundary resolver, returned as a validation failure, marked missing, and rendered as `[MISSING] Expected at: ...`.

## Solution Assessment

Yes. The narrow maintainable direction is #73401 or an equivalent workspace-loader fallback, not a broad relaxation of the shared boundary-file reader.
