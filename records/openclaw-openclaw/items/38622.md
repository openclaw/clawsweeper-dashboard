## Summary

Keep open: current main still routes fixed workspace bootstrap files through a workspace-root guarded reader, so external symlink targets are still recorded and rendered as missing. The issue also has open implementation PRs, including an open PR with closing syntax and a maintainer-labeled scoped replacement, so this should stay open until maintainers settle and land the trust-boundary fix.

## What This Changes

Allow workspace bootstrap files such as SOUL.md and AGENTS.md to load normally when they are symlinks to shared files outside the workspace root.

## Best Possible Solution

Keep this issue open until a reviewed scoped bootstrap-file symlink policy lands, then close it with the merged PR and release provenance while consolidating the related reports.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a sibling-workspace symlinked SOUL.md reaches openRootFile under the workspace root, fails the symlink boundary contract, is recorded as missing, and is rendered as [MISSING] Expected at: ....

## Solution Assessment

Unclear until maintainers settle the trust model. The scoped fixed-bootstrap-reader approach in https://github.com/openclaw/openclaw/pull/73401 is narrower than the broad config or shared-boundary alternatives, but it still changes which external files may feed prompt bootstrap content.
