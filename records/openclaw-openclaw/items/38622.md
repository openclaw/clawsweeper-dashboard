## Summary

Keep open: current main and the latest release still route fixed workspace bootstrap files through the workspace-root guarded reader, so an outside-workspace symlinked SOUL.md is still converted to a missing bootstrap entry. The active fix path is an open maintainer-labeled PR that needs trust-model/security review before this issue can close.

## What This Changes

Load symlinked workspace bootstrap files such as SOUL.md from shared workspace directories instead of injecting them as missing.

## Best Possible Solution

Review and land or supersede the scoped fix in https://github.com/openclaw/openclaw/pull/73401, keeping the exception limited to fixed bootstrap files and preserving negative security tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a sibling-workspace symlinked SOUL.md resolves outside the workspace root, openRootFile rejects it, and loadWorkspaceBootstrapFiles records it as missing; I did not create a filesystem fixture because this review is read-only.

## Solution Assessment

Unclear until the maintainer trust model is decided. A scoped fixed-bootstrap-file exception is safer than relaxing fs-safe or boundary behavior globally, while a config allowlist would be a broader product choice.
