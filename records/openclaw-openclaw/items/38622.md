## Summary

Keep open: current main still reads fixed workspace bootstrap files through the workspace-root guarded reader, so an out-of-workspace symlinked SOUL.md is still converted into a missing Project Context entry. The active implementation path is the open maintainer-labeled PR at https://github.com/openclaw/openclaw/pull/73401, which needs trust-boundary/security review rather than cleanup closure.

## What This Changes

Workspace bootstrap files such as SOUL.md should load through symlinks instead of being injected as missing.

## Best Possible Solution

Review, harden, and either land or supersede https://github.com/openclaw/openclaw/pull/73401 with a scoped fixed-bootstrap-file exception, then consolidate duplicate symlink reports after the fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a sibling-workspace SOUL.md symlink resolves outside the workspace root, openRootFile rejects it under the workspace boundary, loadWorkspaceBootstrapFiles marks it missing, and bootstrap rendering emits `[MISSING] Expected at: ...`.

## Solution Assessment

Yes, if solved narrowly in the workspace bootstrap reader for explicit fixed bootstrap files. A broader relaxation of shared boundary-path or fs-safe behavior is not the best fix because it changes unrelated filesystem containment contracts.
