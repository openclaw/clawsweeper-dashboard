## Summary

Keep open: current main and the latest release still route fixed workspace bootstrap files through the workspace-root guarded reader, so an outside-workspace symlinked SOUL.md is still recorded as missing. The active fix path is the scoped open PR at https://github.com/openclaw/openclaw/pull/73401, plus broader competing approaches that need maintainer trust-model review before this issue can close.

## What This Changes

The issue requests that workspace bootstrap/context files such as SOUL.md be read normally when the workspace path is a symlink to a file outside the workspace root.

## Best Possible Solution

Review and land or explicitly supersede the scoped fix in https://github.com/openclaw/openclaw/pull/73401, keeping the exception limited to fixed bootstrap files and preserving negative security coverage rather than relaxing shared boundary primitives globally.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a sibling-workspace symlinked SOUL.md resolves outside the workspace root, openRootFile rejects it under the root boundary contract, loadWorkspaceBootstrapFiles records it as missing, and the context renderer emits `[MISSING] Expected at: ...`.

## Solution Assessment

Unclear until maintainers settle the trust model. A scoped fixed-bootstrap-reader exception looks safer than a broad allowedExternalPaths feature or a generic boundary-path fallback, but the existing open PR still needs maintainer security review.
