## Summary

Keep open: current main still reads fixed workspace bootstrap files through the root-bounded fs-safe reader, so a symlinked SOUL.md that resolves outside the workspace is still converted into a missing bootstrap entry. The active resolution path is maintainer PR https://github.com/openclaw/openclaw/pull/73401, which still needs explicit security/trust-model review before this issue should close.

## What This Changes

Load workspace bootstrap files such as SOUL.md through valid symlinks instead of injecting them as missing.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/73401, keeping the change scoped to explicit fixed bootstrap files and preserving the existing boundary protections for other reads.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a sibling-workspace SOUL.md symlink resolves outside the workspace root, fs-safe rejects that symlink hop, and loadWorkspaceBootstrapFiles emits a missing entry.

## Solution Assessment

Yes, the targeted direction in the open maintainer PR is the narrow maintainable fix: handle explicit fixed bootstrap-file symlinks in src/agents/workspace.ts instead of weakening the shared boundary reader globally. The final allowed target scope still needs maintainer/security review.
