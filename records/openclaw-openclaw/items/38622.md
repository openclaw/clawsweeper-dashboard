## Summary

Keep open. Current main still routes fixed workspace bootstrap files through the boundary reader with no symlink fallback, so an out-of-workspace `SOUL.md` symlink is rejected and rendered as `[MISSING]`; #73401 is the open maintainer-labeled implementation path.

## What This Changes

Allow workspace bootstrap files such as `SOUL.md` and `AGENTS.md` to follow symlinks to readable shared files instead of rendering `[MISSING]`.

## Best Possible Solution

Review and land #73401 or an equivalent fixed-bootstrap-only symlink loader fix, then close this issue after the behavior is merged and shipped.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a symlinked fixed bootstrap file escaping the workspace boundary is rejected by `resolveBoundaryPath`, returned as a validation failure, marked missing by `loadWorkspaceBootstrapFiles`, and rendered as `[MISSING] Expected at: ...`.

## Solution Assessment

Yes. The narrow maintainable solution is a scoped fixed-bootstrap-file fallback such as #73401, not a broad relaxation of the shared boundary-file reader or extra/glob bootstrap boundaries.
