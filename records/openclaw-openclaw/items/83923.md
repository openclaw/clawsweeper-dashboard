## Label Justifications

- `P2`: Mistyped node-host ports can launch against the wrong gateway address without a diagnostic, but the blast radius is limited to this CLI startup path.

## Summary

Keep open: current main and v2026.5.19 still route invalid explicit `--port` values through the silent fallback path, and the linked open fix PR should be reviewed or landed before this issue closes.

## What This Changes

The issue requests that `openclaw node run --port <bad>` reject invalid explicit ports instead of silently using the configured or default gateway port.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84307, or an equivalent focused fix, so omitted `--port` still falls back while invalid explicit `--port` exits with the existing invalid-port diagnostic.

## Reproduction Assessment

Yes. From source inspection, invalid explicit `opts.port` becomes `null` in `parsePort`, then `parsePortWithFallback` turns that into the configured/default port before `runNodeHost` is called.

## Solution Assessment

Yes. The narrow maintainable fix is to validate only provided `--port` values with the existing invalid-port diagnostic while preserving config/default fallback when the option is omitted.
