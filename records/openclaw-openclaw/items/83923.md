## Label Justifications

- `P2`: This is a normal CLI correctness bug with limited blast radius: a mistyped node-host port can silently use the wrong configured/default gateway port.

## Summary

Keep open: current `main` and v2026.5.18 still have the silent `node run --port` fallback, and open linked PRs with closing syntax should be reviewed and landed before this issue closes.

## What This Changes

Reject invalid explicit `openclaw node run --port` values instead of silently falling back to the configured or default port.

## Best Possible Solution

Land one canonical linked fix PR that rejects invalid explicit `node run --port` values while preserving fallback only when `--port` is omitted, then let GitHub close this issue and retire duplicate fix PRs.

## Reproduction Assessment

Yes. Source inspection on current `main` shows invalid explicit `opts.port` parses to `null`, then `parsePortWithFallback` converts it to the configured/default port before `runNodeHost` is called.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish omitted `--port` from invalid explicit `--port` and reuse the existing invalid-port diagnostic used by adjacent CLI paths.
