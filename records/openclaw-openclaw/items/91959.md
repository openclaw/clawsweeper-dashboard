## Summary

Keep this open: current main still appends each OpenAI Responses `message` item as a separate text block in both the shared LLM consumer and the agent transport consumer, while https://github.com/openclaw/openclaw/pull/92399 is still an open closing PR rather than a merged fix.

## What This Changes

The issue reports Bedrock Mantle GPT-5.x `openai-responses` cumulative message snapshots being concatenated into repeated visible, transcript, and replay text instead of producing one final answer.

## Best Possible Solution

Land a bounded parser/backfill fix that treats same-response cumulative message snapshots as replacements while preserving distinct phase and tool-separated messages, then close this issue through the merged PR.

## Reproduction Assessment

Yes at source level, but not by a live Mantle run in this read-only review. The issue provides exact CLI and raw-provider evidence, and current main’s two Responses consumers still append each completed message item as an independent text block.

## Solution Assessment

Yes, a core parser mitigation is the best repo-side fix. The provider output is already corrupted, but OpenClaw can shield users by making cumulative same-response message snapshots last-wins while preserving legitimate phase/tool boundaries.
