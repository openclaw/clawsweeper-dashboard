The issue reports duplicated final assistant text when Bedrock Mantle GPT-5.x over `openai-responses` emits cumulative message snapshots that OpenClaw appends instead of replacing.

## Solution Assessment

Yes, the best fix direction is a shared `openai-responses` consumer/backfill mitigation that collapses cumulative snapshots while preserving distinct phase, reasoning, and tool boundaries. The issue should stay open until the linked implementation PR merges or maintainers choose a different boundary.
