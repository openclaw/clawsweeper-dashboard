## Summary

This report confirms the same Bedrock Mantle Responses failure already tracked by the older open canonical issue: malformed cumulative final-answer message items are appended as distinct text blocks and can become repeated channel messages or one massively duplicated final text. The newer raw streaming and non-streaming evidence is valuable and is already linked into the canonical discussion, so keeping a second repair thread open would split one root-cause fix.

## What This Changes

Mitigate cumulative duplicate assistant text from Bedrock Mantle Responses outputs, especially for long GPT-5.x replies with reasoning enabled.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/89531 as the single canonical thread and implement a constrained Responses-boundary normalization that replaces same-phase cumulative snapshots while preserving legitimate distinct message items.

## Reproduction Assessment

Yes. The report gives a precise live command and independent raw streaming and non-streaming Mantle evidence, while current main's append-per-message implementation explains the resulting duplicated text.

## Solution Assessment

Yes, consolidation is the best triage outcome because both reports share the same malformed Responses item stream and transport assembly; the eventual fix should use constrained item/phase-aware replacement rather than generic text-prefix deduplication.
