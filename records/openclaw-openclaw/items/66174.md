## Summary

Keep this PR open for repair. Current main still lacks save-time transcript-path rewriting, but the proposed save pass uses the compatibility resolver and can preserve the same structural old-root paths it is meant to canonicalize.

## What This Changes

The branch adds a save-time pass in `saveSessionStoreUnlocked` to rewrite persisted `sessionFile` and `transcriptPath`, types `transcriptPath` on `SessionEntry`, and adds pruning integration coverage.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage.

## Reproduction Assessment

Yes, at source level. A persisted structural old-root `agents/<agent>/sessions/<file>` path can flow through `resolveSessionFilePath`, and this PR's save pass would persist that resolver output instead of rebuilding from the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the target should be built from the active `sessionsDir` and a validated session id rather than from the compatibility resolver.
