## Summary

Keep open: the underlying bug appears real and current main still lacks save-time transcript path canonicalization, but this PR is not ready because it reuses the read-time compatibility resolver and can leave canonical-looking cross-root paths serialized unchanged.

## What This Changes

The PR adds save-time rewriting of persisted session transcript fields, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that derives saved transcript paths from the active sessions directory plus a validated session id, while keeping read-time compatibility and external transcript deletion safeguards intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to save a store entry whose persisted path is a structurally valid absolute path under another root, such as `/old/agents/main/sessions/<sessionId>.jsonl`, then inspect whether the saved JSON is forced into the active sessions directory.

## Solution Assessment

No. The PR direction is right, but using `resolveSessionFilePath` for save-time canonicalization reuses a read-time compatibility fallback; the save pass should compute the active-session target directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:
