## Summary

Keep this PR open. The reported save-time canonicalization bug is still real on current main, but the proposed patch reuses the read-time compatibility resolver and can still persist structural old-root paths instead of forcing the active sessions directory.

## What This Changes

This PR adds save-time normalization of persisted `sessionFile` and `transcriptPath` values, promotes `transcriptPath` onto `SessionEntry`, and adds integration coverage.

## Best Possible Solution

Land a corrected save-time canonicalization patch that builds persisted transcript paths from the active `sessionsDir` and a validated session id, with regression coverage for structural old-root metadata.

## Reproduction Assessment

Yes. Current-main source shows structural old-root absolute session paths can be preserved by the resolver, and the PR’s save pass can serialize that same resolver result instead of forcing the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be derived from `path.dirname(storePath)` plus a validated `sessionId`, not from the compatibility resolver.
