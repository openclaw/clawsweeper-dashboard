## Summary

Keep open: current main still lacks save-time transcript path canonicalization, and this PR uses a compatibility resolver that can preserve structurally valid cross-root session paths, leaving the central bug unfixed. The real behavior proof is adequate, but the patch needs a corrected rebase or replacement before merge.

## What This Changes

The PR adds save-time normalization for persisted `sessionFile` and `transcriptPath` values, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that derives save-time transcript paths from the active store directory plus a validated session id, preserves the legacy alias when present, and keeps external transcript deletion safeguards intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to save an entry whose persisted path is a structurally valid absolute `.../agents/main/sessions/<sessionId>.jsonl` under another root, then inspect whether the saved JSON is forced into the active store directory.

## Solution Assessment

No. The requested direction is right, but this PR uses `resolveSessionFilePath`, a compatibility resolver that intentionally preserves some cross-root paths; save canonicalization should compute the active-session target directly.
