## Summary

Keep open: current main still lacks the requested save-time rewrite, and the PR addresses a real session-store bug but still uses the compatibility resolver in the new save pass, so structurally valid cross-root session paths can survive unchanged.

## What This Changes

The PR adds save-time session transcript path normalization, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected, rebased patch that derives save-time transcript paths from the active store directory plus a validated session id, preserves the legacy alias when present, and keeps external transcript deletion safeguards intact.

## Reproduction Assessment

Yes, at source level. Save a store entry whose persisted path is a structurally valid absolute `.../agents/main/sessions/<sessionId>.jsonl` under another root, then inspect whether the saved JSON is forced into the active store directory.

## Solution Assessment

No. The requested direction is right, but this PR uses `resolveSessionFilePath`, which intentionally preserves compatibility paths; save canonicalization should compute the active-session target directly.

## Review Findings
