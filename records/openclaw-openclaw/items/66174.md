## Summary

Keep open: this PR targets a real session-store bug, but the branch still serializes the read-time resolver result, which can preserve structurally valid old-root session paths instead of rewriting them into the active store directory.

## What This Changes

The PR adds a save-time normalization pass for persisted `sessionFile` and `transcriptPath` values, promotes `transcriptPath` to `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that computes saved transcript paths from the active `sessionsDir` plus a validated session id, while preserving read-time compatibility and external transcript/archive safeguards.

## Reproduction Assessment

Yes. Source-level reproduction is clear: save an entry whose persisted path is shaped like `/old/agents/main/sessions/<sessionId>.jsonl`; this PR's save pass can resolve and serialize that old-root path through the structural fallback.

## Solution Assessment

No as-is. Save-time canonicalization is the right boundary, but the persisted target should be derived from the active `sessionsDir` and validated `sessionId`, not from the read-time compatibility resolver.
