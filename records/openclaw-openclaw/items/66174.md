## Summary

Keep open. Current main still lacks save-time session transcript path canonicalization, and this PR targets a real session-store recovery problem, but the patch is not mergeable as written because it reuses the compatibility resolver that can preserve structurally valid cross-root session paths instead of forcing the active store directory.

## What This Changes

The PR adds save-time normalization of persisted `sessionFile` and `transcriptPath` values, adds `transcriptPath` to `SessionEntry`, and adds an integration test for outside-path canonicalization.

## Best Possible Solution

Land a corrected patch that derives save-time transcript paths from `path.dirname(path.resolve(storePath))` plus a validated session id, persists both `sessionFile` and the legacy `transcriptPath` alias when present, covers structurally valid cross-root paths in tests, and preserves the existing external transcript deletion safeguards.

## Reproduction Assessment

Yes. A source-level reproduction is clear: give the PR helper an entry whose `sessionFile` is a structurally valid absolute `.../agents/main/sessions/<id>.jsonl` path under another root; `resolveSessionFilePath` can return that old path unchanged, so the save pass does not canonicalize it into the active store directory.

## Solution Assessment

No. The requested direction is right, but this implementation should not call the compatibility resolver for forced save canonicalization; it should compute the target directly from the active sessions directory and validated session id.
