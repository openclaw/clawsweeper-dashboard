## Summary

Keep open. The bug is valid and current main still lacks save-time transcript path canonicalization, but this PR can preserve canonical-looking stale paths from another sessions root, so it needs a narrow repair before merge.

## What This Changes

This PR adds a save-time pass to rewrite persisted `sessionFile` and `transcriptPath`, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test for that rewrite.

## Best Possible Solution

Land a corrected current-main patch that derives save-time transcript paths from the active sessions directory plus a validated session id, keeps the legacy alias compatible, and preserves external transcript deletion safeguards.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to save a store entry whose persisted path is a structurally valid absolute path under another root, such as `/old/agents/main/sessions/<sessionId>.jsonl`, then inspect whether the saved JSON is forced into the active sessions directory.

## Solution Assessment

No. The requested direction is right, but using `resolveSessionFilePath` is not the best fix because that resolver intentionally preserves some cross-root compatibility paths; the save pass should compute the active-session target directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
