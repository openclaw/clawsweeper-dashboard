## Summary

Keep open. Current main still lacks save-time transcript-path canonicalization, and this PR's implementation persists `resolveSessionFilePath` output, which can still be an old-root structural `agents/<agent>/sessions/<file>` path instead of the active sessions directory.

## What This Changes

This PR adds a save-time pass that rewrites persisted `sessionFile` and `transcriptPath`, types `transcriptPath` on `SessionEntry`, and adds a generic outside-path pruning integration test.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage and coordination with the resolver-side fix.

## Reproduction Assessment

Yes, at source level. A store entry with an old-root `agents/<agent>/sessions/<id>.jsonl` path can flow through `resolveSessionFilePath`, and this PR's save pass persists that resolver output instead of rebuilding from the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved path should be built from the active `sessionsDir` and a validated session id rather than from the compatibility resolver.
