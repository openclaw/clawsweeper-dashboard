## Summary

Keep this PR open, but it still needs changes before merge. Current main does not have the save-time canonicalization the PR is trying to add, and the proposed branch is directionally useful, but the helper uses the compatibility resolver that can preserve structurally valid cross-root transcript paths instead of forcing them into the active sessions directory.

## What This Changes

The PR adds save-time normalization of persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test for outside-path canonicalization.

## Best Possible Solution

Land a corrected, rebased patch that derives save-time transcript paths directly from `path.dirname(path.resolve(storePath))` plus a validated session id, persists both `sessionFile` and the legacy `transcriptPath` alias when present, covers structurally valid cross-root agent-session paths in regression tests, preserves the existing external transcript deletion safeguards, and adds the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed the PR helper an entry whose persisted path is an absolute `.../agents/main/sessions/<sessionId>.jsonl` under a different root; `resolveSessionFilePath` can return that old path unchanged, so the save pass does not force it into the active store directory.

## Solution Assessment

No. The requested direction is the right maintenance fix, but using `resolveSessionFilePath` is not the narrowest correct implementation because that function intentionally preserves compatibility paths; save canonicalization should compute the target path directly from the active sessions directory.
