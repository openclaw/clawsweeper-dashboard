## Summary

Keep this PR open: the save-time canonicalization is aimed at a real session-store bug, but the current patch still persists compatibility-resolver output that can preserve old-root structural paths outside the active sessions directory.

## What This Changes

The branch adds save-time rewriting of persisted `sessionFile` and `transcriptPath` fields, types `transcriptPath` on `SessionEntry`, and adds pruning integration coverage.

## Best Possible Solution

Land corrected save-time canonicalization that derives persisted transcript paths from the active `sessionsDir` plus a validated session id, with structural old-root regression coverage and coordination with the resolver-side fix.

## Reproduction Assessment

Yes, at source level. Current `resolveSessionFilePath` can return a structural old-root `agents/<agent>/sessions/<file>` path, and this PR's save pass persists that resolver output instead of rebuilding from the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be built from the active `sessionsDir` and validated session id rather than from the compatibility resolver.
