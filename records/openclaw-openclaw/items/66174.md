## Summary

Keep open: this PR targets a real session persistence bug, but the proposed save pass still uses the read-time compatibility resolver, so structural old-root paths can be written back instead of forced into the active sessions directory.

## What This Changes

The PR adds save-time normalization for persisted `sessionFile`/`transcriptPath`, adds `transcriptPath` to `SessionEntry`, and adds an integration test.

## Best Possible Solution

Land corrected save-time canonicalization that validates the session id and builds persisted paths directly under the active sessions directory, with regression coverage for structural old-root metadata.

## Reproduction Assessment

Yes, source-reproducible. Current main can preserve structural old-root absolute session paths, and this PR's save pass can reserialize that resolver result instead of forcing the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be derived from `path.dirname(storePath)` plus a validated `sessionId`, not from the compatibility resolver.
