## Summary

Keep this PR open, but it is not merge-ready: the save-time pass uses the existing compatibility resolver, which can preserve structurally valid old-root `agents/<agent>/sessions/<file>` paths instead of forcing them into the active sessions directory. The linked resolver-side work at https://github.com/openclaw/openclaw/pull/61322 remains relevant, and this PR still needs clearer real behavior proof for the exact stale-path rewrite.

## What This Changes

The PR adds a save-time pass that rewrites persisted `sessionFile` and `transcriptPath` values before session-store writes, adds `transcriptPath` to `SessionEntry`, and adds an integration test.

## Best Possible Solution

Coordinate this with https://github.com/openclaw/openclaw/pull/61322: fix resolver-side same-agent old-root recovery to target the active sessions directory, then make save-time canonicalization derive the persisted path from the active `sessionsDir` and `sessionId` with structural old-root, custom `session.store`, and `transcriptPath` coverage.

## Reproduction Assessment

Yes, at source level. Current main can preserve a structurally valid old-root `agents/<agent>/sessions/<file>` path through the resolver, and the linked issue provides concrete CLI steps and stale metadata for the user-visible failure.

## Solution Assessment

No, not as written. Save-time canonicalization is a reasonable boundary, but it must derive the canonical target from the active sessions directory instead of calling a compatibility resolver that may intentionally preserve old absolute paths.
