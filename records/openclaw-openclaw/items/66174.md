## Summary

Keep this PR open. Current main still lacks save-time canonicalization for persisted transcript paths, but this branch still derives the saved path through the read-time compatibility resolver, which can preserve old-root structural paths instead of forcing the active sessions directory.

## What This Changes

The PR adds save-time normalization of persisted `sessionFile` and `transcriptPath` values, types `transcriptPath` on `SessionEntry`, and adds an integration test.

## Best Possible Solution

Land a corrected save-time canonicalization patch that derives persisted transcript paths from the active `sessionsDir` and a validated session id, with regression coverage for structural old-root metadata.

## Reproduction Assessment

Yes. The source path is clear: current main can preserve structural old-root absolute paths, and this PR's save pass can reserialize that same resolver result instead of forcing the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be built from `path.dirname(storePath)` or the active `sessionsDir` plus a validated session id, not from `resolveSessionFilePath`.
