## Summary

Keep this PR open, but it still needs changes before merge: the save-time helper treats `resolveSessionFilePath` as canonical even though current main and the latest release still allow structurally valid old-root `agents/<agent>/sessions/<file>` paths to be preserved, so the reported stale same-agent path can remain durable.

## What This Changes

The PR adds save-time normalization of persisted `sessionFile` and `transcriptPath` values, promotes `transcriptPath` onto `SessionEntry`, and adds an integration test for generic outside-directory transcript paths.

## Best Possible Solution

Repair the rewrite target so it derives from the active `sessionsDir`, cover old-root same-agent values for both persisted fields, preserve custom-store and cross-agent behavior, then validate on current main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a same-agent old-root absolute `sessionFile` shaped like `agents/<agent>/sessions/<file>.jsonl`, and this PR's `resolveSessionFilePath` call can preserve it instead of rewriting into the active store directory.

## Solution Assessment

No. Save-time canonicalization is a plausible repair point, but this implementation should derive the target from the active `sessionsDir` instead of using the compatibility resolver result as canonical.
