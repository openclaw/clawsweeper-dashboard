## Summary

Keep this PR open: the real behavior proof is useful, but the patch is not merge-ready because its save-time rewrite uses a compatibility resolver that can preserve the stale old-root path it is supposed to canonicalize.

## What This Changes

The PR adds save-time rewriting of persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds an integration test.

## Best Possible Solution

Retarget the save-time repair onto current main, derive rewritten paths directly from the active `sessionsDir` and validated `sessionId`, keep `resolveSessionFilePath` as a read-time compatibility resolver, and coordinate the rule with https://github.com/openclaw/openclaw/pull/61322.

## Reproduction Assessment

Yes, at source level. Current main can preserve an old-root structural `agents/<agent>/sessions/<file>` path, and this PR's save-time rewrite calls that same resolver; I did not run tests because this review is read-only.

## Solution Assessment

No as written. Save-time canonicalization is a reasonable boundary, but it must build the target from the active store directory rather than reusing a compatibility resolver that intentionally preserves some legacy absolute paths.
