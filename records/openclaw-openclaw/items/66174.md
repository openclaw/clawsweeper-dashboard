## Summary

Keep open: the save-time canonicalization is a useful bug fix, but the current patch derives the persisted target through the compatibility resolver, which can preserve structural old-root paths and reserialize the stale value.

## What This Changes

The PR adds a save-time normalization pass for persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds an integration test.

## Best Possible Solution

Land a corrected save-time canonicalization patch that builds persisted transcript paths from the active `sessionsDir` and a validated session id, with regression coverage for structural old-root metadata.

## Reproduction Assessment

Yes, from source inspection. Current main can preserve structural old-root absolute session paths, and this PR's save pass can reserialize that same resolver result instead of forcing the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be derived from `path.dirname(storePath)` plus a validated `sessionId`, not from the compatibility resolver.
