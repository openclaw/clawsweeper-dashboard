## Summary

Keep open: current main still lacks save-time transcript path canonicalization, and this PR is useful, but the patch reuses the read-time resolver so structurally valid old-root paths can still be persisted outside the active sessions directory.

## What This Changes

The PR adds a save-time session-store path normalization pass, adds `transcriptPath` to `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that derives saved transcript paths from the active sessions directory plus a validated session id, while preserving read-time compatibility and external transcript deletion safeguards.

## Reproduction Assessment

Yes, source-level. Save a store entry whose persisted path is shaped like `/old/agents/main/sessions/<sessionId>.jsonl`; the current resolver can preserve that old absolute root, and this PR's save pass would serialize that preserved path.

## Solution Assessment

No as-is. Save-time canonicalization is the right boundary, but the patch should compute the target from the active `sessionsDir` and validated `sessionId` instead of reusing the read-time compatibility resolver.
