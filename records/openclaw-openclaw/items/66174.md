## Summary

Keep this PR open, but do not merge it as-is: the underlying session-path bug is real, current main still lacks a save-time rewrite, and the proposed canonicalization can preserve the stale external root it is meant to eliminate.

## What This Changes

The branch adds a save-time pass that rewrites persisted `sessionFile` and `transcriptPath` fields, adds `transcriptPath` to `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that derives saved transcript paths from the active sessions directory plus a validated session id, while preserving read-time compatibility and external transcript deletion safeguards.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to save a store entry whose persisted path is a structurally valid absolute path under another root, such as `/old/agents/main/sessions/<sessionId>.jsonl`, then inspect whether the saved JSON is forced into the active sessions directory.

## Solution Assessment

No. The PR direction is useful, but using `resolveSessionFilePath` for save-time canonicalization reuses a read-time compatibility fallback; the save pass should compute the active-session target directly and cover the structural cross-root case.

## Review Findings
