## Summary

Keep open: the save-time canonicalization problem is real, but this PR still derives the saved path through the compatibility resolver, so structural old-root session paths can be persisted unchanged.

## What This Changes

This PR adds save-time normalization of persisted `sessionFile` and `transcriptPath` values, promotes `transcriptPath` on `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected save-time canonicalization patch that derives the target from the active sessions directory plus a validated session id, with structural old-root regression coverage.

## Reproduction Assessment

Yes. Seed a store entry with a structural old-root `sessionFile` such as `/old/root/agents/main/sessions/<id>.jsonl` and save it; source inspection shows the PR normalizer can serialize that same old-root resolver result.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved path should be built from the active sessions directory and validated session id rather than from the compatibility resolver.
