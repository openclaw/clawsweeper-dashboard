## Summary

This PR should stay open but needs repair before merge: current main still lacks save-time transcript-path canonicalization, and the proposed implementation can persist the old-root structural path it is meant to replace.

## What This Changes

The PR adds save-time rewriting of persisted `sessionFile` and `transcriptPath` values, types `transcriptPath` on `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage and coordination with the resolver-side fix.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a store entry with an old-root `agents/<agent>/sessions/<id>.jsonl` path can flow through `resolveSessionFilePath`, and this PR would persist that resolver result. I did not execute a live repro because this review is read-only.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the target must be built from the active `sessionsDir` and a validated session id rather than from the compatibility resolver.
