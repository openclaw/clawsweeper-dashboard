## Summary

Keep this PR open, but it still needs changes before merge: the save-time canonicalization is useful, yet the current diff can persist the same structural old-root path because it derives the saved value through the compatibility resolver.

## What This Changes

The PR adds save-time rewriting of persisted `sessionFile` and `transcriptPath`, types `transcriptPath` on `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage and coordination with the resolver-side fix.

## Reproduction Assessment

Yes, at source level. An entry with an old-root `agents/<agent>/sessions/<id>.jsonl` path can flow through `resolveSessionFilePath`, and this PR's save pass persists that resolver output instead of rebuilding from the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved path should be built from the active `sessionsDir` and a validated session id rather than from the compatibility resolver.
