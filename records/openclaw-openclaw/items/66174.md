## Summary

Keep open: current main still lacks save-time transcript path canonicalization, and this PR is a useful bug-fix direction, but the patch uses the compatibility resolver that can preserve the cross-root structural paths it is meant to rewrite.

## What This Changes

The PR adds save-time rewriting of persisted `sessionFile` and `transcriptPath`, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected current-main patch that derives save-time transcript paths from the active store directory plus a validated session id, keeps the legacy alias compatible, and preserves external transcript deletion safeguards.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to save a store entry whose persisted path is a structurally valid absolute path under another root, such as `/old/agents/main/sessions/<sessionId>.jsonl`, then inspect whether the saved JSON is forced into the active sessions directory.

## Solution Assessment

No. The requested direction is right, but this PR uses `resolveSessionFilePath`, which intentionally preserves some cross-root compatibility paths; the save pass should compute the active-session target directly.
