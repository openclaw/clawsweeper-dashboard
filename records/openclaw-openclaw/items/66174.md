## Summary

Keep this PR open. It targets a real session-store bug, but the proposed save-time rewrite still calls the read-time compatibility resolver, so same-agent old-root structural paths can be serialized back outside the active sessions directory.

## What This Changes

This PR adds save-time normalization for persisted `sessionFile` and `transcriptPath`, makes `transcriptPath` a `SessionEntry` field, and adds an integration test.

## Best Possible Solution

Land a corrected patch that derives saved transcript paths from the active `sessionsDir` and a validated session id, while preserving custom-store and cross-agent fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: save an entry whose persisted path is shaped like `/old/agents/main/sessions/<sessionId>.jsonl`, and this PR can reserialize the structural fallback result.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be built from the active `sessionsDir` and validated `sessionId`, not from the read-time compatibility resolver.
