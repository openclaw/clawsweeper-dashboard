## Summary

Keep this PR open: current main still lacks save-time transcript path canonicalization, but the proposed patch does not fully fix the cross-root case because it reuses the compatibility resolver that can preserve old agent-session paths.

## What This Changes

The PR adds save-time normalization of persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test for outside-path canonicalization.

## Best Possible Solution

Land a corrected, rebased patch that derives save-time transcript paths from the active store directory plus a validated session id, persists the legacy alias when present, and preserves external transcript deletion safeguards.

## Reproduction Assessment

Yes. A source-level reproduction is clear: save a store entry whose persisted path is a structurally valid absolute `.../agents/main/sessions/<sessionId>.jsonl` under another root, then inspect whether the saved JSON is forced into the active store directory.

## Solution Assessment

No. The requested direction is right, but the PR uses `resolveSessionFilePath`, which intentionally preserves compatibility paths; save canonicalization should compute the active-session target directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
