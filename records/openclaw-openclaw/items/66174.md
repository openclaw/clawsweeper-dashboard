## Summary

Keep open: current main still lacks the requested save-time transcript metadata rewrite, but this PR persists compatibility resolver output and therefore misses structural old-root paths. The right path is a narrow repair, not cleanup closure.

## What This Changes

The PR adds save-time rewriting of persisted `sessionFile` and `transcriptPath` values, types `transcriptPath` on `SessionEntry`, and adds pruning integration coverage.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage.

## Reproduction Assessment

Yes, at source level. A persisted old-root `agents/<agent>/sessions/<file>` path can flow through `resolveSessionFilePath`, and this PR's save pass persists that resolver output instead of rebuilding from the active sessions directory.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be built from the active `sessionsDir` and a validated session id rather than from the compatibility resolver.
