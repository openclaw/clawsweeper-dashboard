## Summary

Keep open: the storage bug is valid and current main still has no save-time canonicalization, but this PR should not merge as-is because its save pass can reserialize structural old-root paths through the read-time resolver.

## What This Changes

The PR adds save-time canonicalization for persisted session transcript paths, adds `transcriptPath` to `SessionEntry`, and adds an integration test.

## Best Possible Solution

Land a corrected save-time canonicalization patch that derives the persisted path from the active sessions directory and a validated session id, with structural old-root regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main serializes the session store without rewriting transcript paths, and the resolver can still return structural old-root absolute paths that this PR would write back.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but it should build the saved target from the active `sessionsDir` and validated `sessionId`, not from the compatibility resolver.
