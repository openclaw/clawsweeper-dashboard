## Summary

Keep open. The underlying session-path bug is still valid, and this PR is useful, but the patch can persist the same structural old-root path it is meant to clear because the save pass calls the compatibility resolver instead of deriving the target from the active sessions directory.

## What This Changes

The PR adds save-time rewriting of persisted `sessionFile` and `transcriptPath` values, types `transcriptPath` on `SessionEntry`, and adds a pruning integration test.

## Best Possible Solution

Land a corrected save-time canonicalization that derives persisted transcript paths from the active sessions directory plus a validated session id, with structural old-root regression coverage and coordination with the resolver-side fix.

## Reproduction Assessment

Yes. The source-level path is clear: an entry with an old-root `agents/<agent>/sessions/<id>.jsonl` `sessionFile` can flow through `resolveSessionFilePath` and be persisted, although this read-only review did not run a live repro.

## Solution Assessment

No as written. Save-time canonicalization is the right boundary, but the saved target should be built from the active sessions directory and validated session id instead of the compatibility resolver.
