## Summary

Keep this PR open, but it needs changes before merge: the proposed save-time repair uses the compatibility resolver that can preserve old-root structural session paths, so it does not reliably fix the linked session-state bug. The branch is also currently merge-conflicting against main.

## What This Changes

The PR adds save-time canonicalization for persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds an integration test.

## Best Possible Solution

Retarget the save-time repair onto current main, derive rewritten paths directly from the active `sessionsDir` and `sessionId`, preserve the existing compatibility resolver for read-time resolution, and coordinate with https://github.com/openclaw/openclaw/pull/61322 so resolver-side and persistence-time rules do not diverge.

## Solution Assessment

No as written. Save-time canonicalization is a reasonable boundary, but it must use a strict current-store target path rather than the compatibility resolver that intentionally preserves some legacy absolute paths.
