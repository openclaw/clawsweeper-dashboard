## Summary

Keep this PR open, but it still needs changes before merge: current main still lacks the requested save-time transcript path canonicalization, while the PR’s implementation can preserve structurally valid old-root session paths and is missing the required changelog entry.

## What This Changes

The PR adds save-time normalization of persisted session transcript paths, promotes `transcriptPath` onto `SessionEntry`, and adds a pruning integration test for outside-path canonicalization.

## Best Possible Solution

A corrected branch should normalize persisted transcript metadata to the active store directory while preserving compatibility lookup semantics elsewhere and keeping external transcript deletion safeguards intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: save a store entry whose persisted `sessionFile` is an old absolute `agents/main/sessions/<sessionId>.jsonl` path while the active store path is under a different root; the PR’s resolver call can return the old path instead of the active sessions directory.

## Solution Assessment

No. The requested direction is right, but save canonicalization should derive the target from `path.dirname(path.resolve(storePath))` plus a validated session id rather than reusing the compatibility resolver that intentionally preserves some old absolute paths.
