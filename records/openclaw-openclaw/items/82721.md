## Summary

Keep open: the branch is useful for already scoped `NODE_COMPILE_CACHE` paths, but it no longer satisfies the raw cache-root behavior described by the PR body, the linked issue, and current docs. Merging as-is would risk closing the broader launcher issue while the documented raw-root setup still takes the packaged respawn path on each fresh CLI invocation.

## What This Changes

The PR changes the packaged launcher to accept Node's active versioned compile-cache directory when it is under OpenClaw's scoped cache directory, and adds a launcher E2E case plus a live repro script.

## Best Possible Solution

Preserve package-scoped invalidation while making the documented raw cache-root path either avoid repeat respawns or be explicitly moved to a scoped-path contract, with tests and proof matching that contract.

## Reproduction Assessment

Yes from source inspection: current main exact-compares the desired scoped cache root with Node's active versioned child, and the PR proof reproduces that for already scoped paths. The same inspection shows the latest PR still leaves the documented raw-root path outside the compatibility check.

## Solution Assessment

No as written: the scoped-path fix is plausible, but the PR still claims to fix the broader raw-root launcher issue. The safer solution is to either implement the raw-root contract without losing package scoping or narrow the PR so it does not close the linked issue prematurely.
