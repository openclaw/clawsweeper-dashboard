## Summary

Keep open; this PR targets a real packaged-launcher bug and includes real no-respawn output, but the proposed shared-root compatibility check regresses the existing package-scoped compile-cache contract.

## What This Changes

The PR broadens packaged launcher compile-cache directory compatibility, adds an E2E regression for repeated `NODE_COMPILE_CACHE` launches, and adds a live repro script.

## Best Possible Solution

Keep the PR open until the launcher either avoids repeated respawns without dropping package-scoped cache behavior, or explicitly changes that cache contract with owner approval and aligned tests/docs/postinstall behavior.

## Reproduction Assessment

Yes. A Node 22 temp packaged fixture on current main with `NODE_COMPILE_CACHE` set to a root prints the packaged respawn marker on both invocations, matching the linked issue.

## Solution Assessment

No. The patch avoids the respawn by treating sibling directories under the same cache root as compatible, but Node 22 keeps the active cache in the versioned root, so OpenClaw's existing package-scoped cache behavior is lost.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve the package-scoped cache directory:** `openclaw.mjs:88-90`
  - body: This shared-root check treats Node's active `/base/v22...` cache as compatible with OpenClaw's `/base/openclaw/<version>/<marker>` target.
