## Summary

Keep open: the PR now fixes the already-scoped cache-directory case, but the linked bug and current docs use a stable root `NODE_COMPILE_CACHE`; with that root, the patched guard still compares sibling paths and each fresh packaged launch can respawn.

## What This Changes

The PR changes the packaged launcher compile-cache directory comparison, avoids double-appending an already-scoped OpenClaw cache path, and adds a launcher E2E regression plus a live repro script.

## Best Possible Solution

Decide the cache contract first, then either implement and test the documented root-cache behavior or update docs/PR scope so users configure a supported scoped cache path.

## Reproduction Assessment

Yes. The linked issue has a current-main packaged-fixture repro for repeated respawns with a root `NODE_COMPILE_CACHE`, and the current source/docs show the same root-versus-scoped path mismatch.

## Solution Assessment

No. The patch is maintainable for an already-scoped cache path, but it does not resolve the documented root-cache setup without a maintainer decision on the package-scoped cache contract.
