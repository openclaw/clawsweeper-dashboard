## What This Changes

The PR changes the packaged launcher compile-cache comparison to accept active versioned children under the desired scoped cache directory, avoids double-scoping an already scoped cache path, and adds launcher E2E plus live repro coverage.

## Best Possible Solution

Resolve the launcher contract explicitly: support the documented raw cache root while preserving package-version/install-marker scoping, or narrow this PR to scoped env paths and keep https://github.com/openclaw/openclaw/issues/82688 open for raw-root behavior.

Yes. The linked issue's ClawSweeper review records a current-main packaged fixture that respawned on both launches with a raw `NODE_COMPILE_CACHE`, and current main source still exact-compares Node's active versioned child with OpenClaw's desired scoped root.
