## Best Possible Solution

Either expand the launcher fix to handle documented raw cache roots while preserving OpenClaw package-scoped invalidation, or narrow the PR body and closing references so the raw-root issue stays open as the canonical follow-up.

## Reproduction Assessment

Yes. Current `main` and the PR head can be reasoned from source: Node starts with an active cache directory from `NODE_COMPILE_CACHE`, while OpenClaw computes a scoped desired directory, and the PR head only accepts active directories nested under that scoped desired path.
