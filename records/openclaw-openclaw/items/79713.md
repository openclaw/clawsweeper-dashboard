## Summary

Keep open: current main still makes the reseed prompt unreachable for uncompacted CLI sessions, but the existing tests deliberately preserve that safety boundary, so changing it needs maintainer policy direction rather than an automatic close or blind default change.

## What This Changes

The issue requests that invalidated Claude CLI sessions reseed from recent raw transcript messages when no compaction summary exists, so the history prompt fallback is not empty.

## Best Possible Solution

Decide the transcript safety boundary explicitly, then either land a completed opt-in raw-tail reseed path with schema/docs/tests or implement a safer compaction-at-reset/retry strategy while handling the related `missing-transcript` race separately.

## Reproduction Assessment

Yes from source: current main returns no reseed messages for uncompacted sessions, which makes `buildCliSessionHistoryPrompt` return `undefined` and produces `historyPrompt=none` on fresh CLI runs. I did not live-run the idle-reset flow in this read-only pass.

## Solution Assessment

No for the unconditional suggested fallback: it is narrow, but it conflicts with the existing no-raw-pre-compaction safety test. A safer maintainable path is an explicit opt-in flag or reset-time compaction/retry design chosen by maintainers.
