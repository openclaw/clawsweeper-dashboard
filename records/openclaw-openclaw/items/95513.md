## Summary

Keep open: the linked bug is real and source-reproducible, but this PR is not merge-ready because it lets generic failure text past the visible-output guard without returning a fallback classification for the reported non-GPT claude-cli model, broadens visible-output retry behavior, and lacks real exhausted-CLI proof.

## What This Changes

The branch imports the generic external-runner failure text helper and lets matching visible payloads bypass the embedded-agent fallback classifier's visible-output short-circuit.

## Best Possible Solution

Land one canonical fix that returns a non-null fallback classification only for complete generic external-runner failure output, preserves no-retry guards, adds regression coverage, and is backed by redacted real fallback proof.

## Reproduction Assessment

Yes at source level: current main wraps CLI output text as a normal `{ text }` payload and the classifier returns null on visible non-error payloads. The linked issue supplies exhausted-credit evidence, but I did not run a live exhausted Claude CLI account.

## Solution Assessment

No. The classifier boundary is plausible, but this patch should return an explicit classification before the non-GPT guard and require all visible non-error, non-reasoning payloads to be the exact generic failure text.
