## Summary

Keep open: the underlying Control UI bug is still present on current main and this PR targets the right path, but the head diff should not merge until array fragment collection is explicitly bounded.

## What This Changes

The PR updates Control UI tool-card extraction to join array-shaped tool-result `text` and `content` fragments, adds node coverage for array and empty-array cases, and adds a changelog entry.

## Best Possible Solution

Keep the PR open and amend it to cap fragment count and total collected characters, mark truncation, and preserve the existing string and empty-output behavior.

## Reproduction Assessment

Yes from source inspection: current main only accepts string `text` or `content`, so `content: [{ text: "ok" }]` produces no `outputText` and the UI falls back to `No output`. I did not run the UI or tests in this read-only review.

## Solution Assessment

No, not yet. Reading array-shaped tool-result text is the narrow owner-path fix, but the implementation needs explicit size bounds before joining and downstream preview/sidebar/rendering work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound array tool-output collection:** `ui/src/ui/chat/tool-cards.ts:51-60`
  - body: The new array path collects every string fragment and joins the full result before preview extraction, sidebar formatting, and rendering.   - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff still has a concrete client-side resource-exhaustion risk from joining unbounded array-shaped tool-result fragments.

Concerns:

- **[medium] Unbounded tool-result fragment joining:** `ui/src/ui/chat/tool-cards.ts:51`
  - body: `collectTextParts` walks every array entry and the caller joins all collected strings into one `outputText`, which is then parsed for canvas previews, formatted for the sidebar, and rendered. A very large array or large fragments can force large allocations and expensive browser work, so this needs explicit size limits before merge.
  - confidence: 0.86
