## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound array output before joining:** `ui/src/ui/chat/tool-cards.ts:48-61`
  - body: The new array extraction path can collect every accepted fragment and join it into one `outputText` before preview parsing and rendering. Tool-result payloads can be large or provider-influenced, so cap fragments/characters and truncate before returning the joined string.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR broadens provider/tool-influenced tool-result joining without explicit size bounds before browser preview parsing and rendering.

Concerns:

  - body: `collectTextParts()` walks accepted array entries and the callers join the full result into a browser string, creating a client-side resource-exhaustion risk for oversized tool-result payloads.
  - confidence: 0.86
