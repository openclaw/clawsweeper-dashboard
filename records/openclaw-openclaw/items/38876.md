## Summary

Keep open: the linked `content[0].text` bug is already covered on current `main`, but this PR still carries broader branch-only array `text` and string-fragment behavior, lacks real Control UI/WebSocket proof, and has a concrete unbounded-join security finding. A recent member branch update also makes this unsuitable for stale cleanup closure.

## What This Changes

The PR extends Control UI tool-card extraction to join string fragments from array-shaped `text` and `content` tool-result payloads, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Narrow the PR to supported array payload shapes, cap or truncate joined output before preview/rendering, and add redacted real Control UI/WebSocket proof; otherwise keep the shipped content-array fix and close this branch after human review.

## Reproduction Assessment

Partially: current source and tests prove the linked `content[].text` case now renders, and source inspection shows current main would not extract the PR-only `item.text` array shape. There is still no high-confidence real Control UI/WebSocket reproduction for the broader branch-only shapes.

## Solution Assessment

No, not as-is. The maintainable path is bounded extraction for a proven payload contract plus real UI proof, not unbounded joining of every accepted array fragment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound array output before joining:** `ui/src/ui/chat/tool-cards.ts:48-61`
  - body: The new array extraction path can collect every accepted fragment and join it into one `outputText` before preview parsing and rendering. Tool-result payloads can be large or provider-influenced, so cap fragments/characters and truncate before returning the joined string.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR accepts and joins provider/tool-influenced array fragments without explicit size bounds before preview parsing and UI rendering.

Concerns:

- **[medium] Unbounded tool-result fragment joining:** `ui/src/ui/chat/tool-cards.ts:48`
  - body: `collectTextParts()` walks accepted array entries and the callers join the full result into a browser string, creating a client-side resource-exhaustion risk for oversized tool-result payloads.
  - confidence: 0.86
