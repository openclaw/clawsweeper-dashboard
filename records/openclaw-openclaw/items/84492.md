## Summary

Keep open: current main still maps Codex app-server `interrupted` turns to OpenClaw `aborted`, and the same-author closing PR is open rather than merged. The maintainer stack note also names that PR as the active fix and keeps adjacent timeout/truncation reports separate.

## What This Changes

The issue reports that Codex app-server `interrupted` turn completions are treated as OpenClaw aborts, which can suppress no-visible-answer recovery for tool-only user-facing turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84494, or close that PR with a clear alternative, then close this issue only after an equivalent fix reaches main.

## Reproduction Assessment

Yes, at source level: current main still projects app-server `interrupted` status into `aborted`, and the incomplete-turn guard skips aborted attempts. I did not run the Vitest repro because this is a read-only review, but the issue and linked PR provide focused failing and passing test evidence.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path: keep explicit OpenClaw abort/timeout state separate from app-server terminal status and lock it with projector plus incomplete-turn regressions.
