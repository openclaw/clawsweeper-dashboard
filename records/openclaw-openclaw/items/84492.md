## Summary

Keep open: current main and the latest release still contain the reported Codex app-server interrupted-to-aborted projection, and the same-author linked fix PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to stop treating Codex app-server `turn.status: "interrupted"` as an OpenClaw abort unless OpenClaw explicitly cancelled or timed out the attempt.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84494, or an equivalent fix that keeps app-server `interrupted` distinct from explicit OpenClaw aborts/timeouts and preserves the no-visible-answer guard.

## Reproduction Assessment

Yes, at source level: current main still maps app-server `interrupted` to `aborted`, and the incomplete-turn guard skips aborted attempts. I did not run the Vitest repro because this review is read-only, but the issue and linked PR provide focused failing and passing test evidence.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable solution: keep explicit OpenClaw abort/timeout state separate from Codex app-server terminal status, with projector and incomplete-turn regression coverage.
