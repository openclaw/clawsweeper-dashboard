## Label Justifications

- `P1`: The issue describes a broken Codex app-server agent workflow where a user-facing answer can be suppressed after tool output.
- `impact:message-loss`: The affected behavior can leave a user-facing turn without the expected visible assistant response.

## Summary

Keep open: current main still treats a Codex app-server `interrupted` terminal turn as an OpenClaw abort, and the incomplete-turn guard still suppresses no-visible-answer handling when `aborted` is true.

## What This Changes

The issue reports that Codex app-server interrupted turn completions can be treated as OpenClaw aborts, suppressing no-visible-answer handling for tool-only turns.

## Best Possible Solution

Update the Codex app-server projector so only explicit OpenClaw cancellation or timeout marks the attempt aborted, while app-server `interrupted` remains available for separate terminal-outcome classification and incomplete-turn handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send an app-server `turn/completed` event with `status: "interrupted"` and no assistant text, observe `buildResult().aborted === true`, then the incomplete-turn helper returns `null` because `aborted` is true.

## Solution Assessment

Yes. The narrow maintainable fix is to keep explicit `markAborted()` and timeout handling intact while removing app-server `interrupted` as an automatic OpenClaw abort signal, with focused projector and incomplete-turn regression tests.
