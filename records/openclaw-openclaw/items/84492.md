## What This Changes

Codex app-server `interrupted` completions should not suppress no-visible-answer handling unless OpenClaw explicitly cancelled or timed out the run.

## Best Possible Solution

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: remove the blanket interrupted-to-aborted mapping while preserving explicit OpenClaw abort and timeout paths.
