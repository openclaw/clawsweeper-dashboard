## What This Changes

The branch makes the login-shell environment fallback return an empty success on `win32`, threads a platform override for tests, adds a Windows regression test, and adds a changelog entry.

## Best Possible Solution

Land after maintainer confirmation that native Windows should never run the POSIX login-shell env fallback; if Git Bash/MSYS support is still desired, add an explicit opt-in path instead of probing `/bin/sh` by default.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: config loading can enable `loadShellEnvFallback`, and current `main` reaches the login-shell probe without a `win32` guard. I did not run a live Windows repro in this read-only review, but the PR body includes native Windows after-fix proof.

## Solution Assessment

Yes, applying the existing `win32` no-probe boundary to the shared probe path is the narrowest maintainable fix. The only unresolved question is whether maintainers want to preserve an explicit POSIX-shell-on-Windows opt-in.
