## Best Possible Solution

Make required CI run only smoke coverage that is already green with artifact proof, and keep the full `smoke-ci` profile manual or non-required until its scenario failures are fixed.

## Solution Assessment

No: adding the full failing `smoke-ci` profile to a required fast shard is not the best fix while that profile is not green. A safer path is to gate only a proven-green subset or keep full-profile evidence outside required CI until the failures are fixed.
