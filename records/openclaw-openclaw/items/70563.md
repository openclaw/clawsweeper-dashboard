## What This Changes

The issue asks OpenClaw to stop Kimi thinking-enabled subagent or tool-call replay from failing when assistant tool-call messages lack `reasoning_content`.

## Best Possible Solution

Add a Kimi-provider-owned payload/replay repair that makes thinking-enabled Kimi tool-call replays acceptable without adding subagent-specific core logic.

Yes, by high-confidence source inspection plus reporter logs: enable Kimi thinking, trigger a tool-call replay through `sessions_spawn` or a normal follow-up turn, and current `extensions/kimi-coding/stream.ts` still sends the thinking control without supplying `reasoning_content` on replayed assistant messages. I did not run a live Kimi API repro in this read-only review.

## Solution Assessment

Yes on the owner boundary: the fix belongs in the Kimi provider stream/replay payload shaping, not in `sessions_spawn` core. The precise payload strategy should be checked against Kimi Code docs or a redacted live run before choosing between removing Kimi `thinking` and preserving/backfilling `reasoning_content`.
