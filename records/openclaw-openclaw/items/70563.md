## Summary

Keep open; current main still forwards thinking into native subagent runs while the Kimi Coding provider wrapper does not backfill `reasoning_content` on replayed assistant tool-call messages. The related Moonshot PR #74846 is useful context, but it is open and targets a different provider path.

## What This Changes

Report asks for native subagent spawning to work when Kimi/Kimi Coding thinking is enabled instead of failing on missing `reasoning_content` in replayed assistant tool-call history.

## Best Possible Solution

Add a provider-owned Kimi Coding replay repair that preserves existing `reasoning_content` and backfills a Kimi-accepted value when thinking is enabled, with focused stream tests and only a small shared helper if it avoids duplicating the existing provider pattern.

## Reproduction Assessment

Yes. The report gives concrete steps, and current source shows `sessions_spawn` can forward thinking into the child run while the Kimi wrapper lacks any replay repair for assistant tool-call messages.

## Solution Assessment

Yes for a narrow provider-owned fix; no for closing as already fixed. The safer path is to patch the Kimi provider payload wrapper rather than add provider-specific subagent logic in core.
