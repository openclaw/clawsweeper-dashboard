## Summary

Keep open: current main still sends Kimi thinking payloads without adding `reasoning_content` to replayed assistant tool-call messages in the bundled Kimi Coding wrapper, and the focused open fix path is still unmerged.

## What This Changes

Subagent spawning with Kimi thinking enabled should preserve or synthesize `reasoning_content` on replayed assistant tool-call messages so Kimi does not reject the request.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/76073 or an equivalent Kimi provider-wrapper fix that preserves current thinking budget behavior and adds replay coverage.

## Solution Assessment

Yes. Provider-owned Kimi replay normalization is the narrow maintainable fix; generic subagent spawning should not learn Kimi-specific payload rules.
