## Summary

Keep open; current main still sends Kimi `thinking` without repairing replayed assistant tool-call messages, and https://github.com/openclaw/openclaw/pull/76073 remains the focused open fix. The fresh 2026-05-10 comment adds useful release-level reproduction context and says this affects ordinary Kimi tool-call turns too, so this is not cleanup-closeable.

## What This Changes

Make Kimi subagent and follow-up tool-call replays work when thinking is enabled by preserving or backfilling `reasoning_content` on assistant tool-call messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76073 or an equivalent Kimi-provider wrapper fix that preserves existing values and backfills `reasoning_content` only for thinking-enabled assistant tool-call replays, then close this after the fix merges or ships.

## Reproduction Assessment

Yes, source-reproducible. Current `main` still sends `thinking: { type: "enabled" }` for Kimi without repairing replayed assistant tool-call messages, and the issue includes concrete subagent plus current-release tool-call failure reports.

## Solution Assessment

Yes. The provider-local backfill in the open Kimi PR is narrower than deleting supported thinking, because Kimi documents `thinking` as a request parameter and requires `reasoning_content` to be retained for thinking-enabled multi-step tool calls.
