## Summary

Keep open; current main still forwards non-off thinking into native subagent runs, and the Kimi Coding provider wrapper still only patches top-level thinking fields without backfilling `reasoning_content` on replayed assistant messages. The linked Moonshot PR is relevant context, but it is open and targets a different provider path.

## What This Changes

The issue requests that `sessions_spawn` subagents work when Kimi Coding thinking is enabled by preserving or backfilling `reasoning_content` on replayed assistant tool-call messages.

## Best Possible Solution

Add a provider-owned Kimi Coding replay repair that preserves existing `reasoning_content` and fills a Kimi-accepted value when thinking is enabled, using a small shared helper only if it also cleanly serves the Moonshot path.

## Reproduction Assessment

Yes. The report gives concrete steps, and current main shows `sessions_spawn` can forward thinking into the child run while Kimi's wrapper lacks replay repair for assistant messages missing `reasoning_content`.

## Solution Assessment

Yes for a narrow provider-owned fix; no for closing as already fixed. The maintainable path is to patch Kimi Coding payload wrapping or a shared provider helper, not add Kimi-specific logic to core subagent spawning.
