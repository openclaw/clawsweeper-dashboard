## Summary

Keep open; current main still lacks the Kimi replay normalization for thinking-enabled assistant tool-call messages, and the focused fix is tracked by an open, unmerged PR.

## What This Changes

Subagent spawning with Kimi thinking enabled should not fail when replayed assistant tool-call messages lack `reasoning_content`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76073 or an equivalent Kimi provider wrapper fix that preserves Kimi thinking semantics and supplies `reasoning_content` on thinking-enabled assistant tool-call replays.

## Reproduction Assessment

Yes, source-reproducible. The issue gives a concrete Kimi thinking plus `sessions_spawn` path, and current main still enables Kimi thinking without repairing replayed assistant tool-call messages.

## Solution Assessment

Yes, the provider-owned replay repair is the narrow maintainable direction. Deleting Kimi thinking may avoid the rejection, but it would also discard the intended thinking behavior that the current provider surface exposes.
