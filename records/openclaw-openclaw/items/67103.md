## Summary

Keep open. Current main still does not implement Control UI math rendering, and this PR remains a useful implementation attempt, but the branch has blocking correctness, security, dependency, and proof gaps before it can merge.

## What This Changes

This PR adds a KaTeX dependency, imports KaTeX CSS, adds math extraction/restoration around the shared Control UI markdown renderer, and adds markdown tests for inline and display math.

## Best Possible Solution

Rework this PR or a successor so math rendering lands in the shared Control UI markdown renderer while preserving cache and link hardening, using robust delimiter rules, scoped sanitizer allowances, a synced lockfile, focused tests, and real Control UI proof.

## Reproduction Assessment

Yes, source-level. Current main routes chat through toSanitizedMarkdownHtml and has no math-rendering stage, while PR-head blockers are visible because deleted helper definitions are still called.

## Solution Assessment

No. The shared markdown renderer is the right surface, but this patch is not the best implementation because it breaks existing helpers, weakens sanitizer boundaries, omits lockfile proof, lacks real behavior proof, and uses fragile delimiter parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the removed markdown helpers:** `ui/src/ui/markdown.ts:84-89`
  - body: This replacement block removes installHooks, getCachedMarkdown, and setCachedMarkdown, but toSanitizedMarkdownHtml still calls all three. The branch will fail before rendering chat HTML and also drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98
- **[P2] Avoid parsing ordinary prices as math:** `ui/src/ui/markdown.ts:151-153`
  - body: The inline-math detector treats any $...$ pair as KaTeX with no delimiter-boundary rules, so prose like between $5 and $10 is transformed into math and corrupts common chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have $...$ extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9
- **[P2] Add the KaTeX lockfile entry:** `ui/package.json:17`
  - body: The PR adds katex to ui/package.json but does not update pnpm-lock.yaml, so frozen-lockfile installs do not have a reproducible resolution for the new dependency.
  - confidence: 0.86
