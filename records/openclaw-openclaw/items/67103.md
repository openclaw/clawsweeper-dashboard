## Summary

Keep open: current main still lacks Control UI math rendering and this is an active implementation attempt, but the PR is not mergeable as-is because it breaks the shared markdown renderer, weakens sanitizer boundaries, omits lockfile proof, and lacks contributor real behavior proof.

## What This Changes

This PR adds `katex` to the Control UI package and changes the shared markdown renderer and tests to render `$...$` and `$$...$$` chat text as KaTeX HTML.

## Best Possible Solution

Rework this PR or a successor in the shared Control UI markdown renderer so math rendering preserves cache and DOMPurify link hardening, uses robust delimiter rules, scopes KaTeX-specific sanitizer allowances, includes lockfile review, and proves the UI behavior.

## Reproduction Assessment

Yes for source-level current behavior, not as a bug repro: Control UI chat flows through `toSanitizedMarkdownHtml`, and current main has no math-rendering stage or math-rendering dependency.

## Solution Assessment

No. The shared markdown renderer is the right surface, but this patch is not the best implementation because it removes required helpers, broadens sanitizer allowances, omits lockfile proof, lacks real behavior proof, and uses fragile delimiter parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the removed markdown helpers:** `ui/src/ui/markdown.ts:84-89`
  - body: This replacement block removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three. The branch will fail before rendering chat HTML and also drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98
- **[P2] Avoid treating currency text as inline math:** `ui/src/ui/markdown.ts:151-153`
  - body: The inline detector accepts any later `$` as the closing delimiter, so prose such as `between $5 and $10` is transformed into one KaTeX expression and corrupts ordinary chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have `$...$` extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9
- **[P2] Add the KaTeX lockfile entry:** `ui/package.json:17`
  - body: The PR adds `katex` to `ui/package.json` but does not update `pnpm-lock.yaml`, so frozen-lockfile installs do not have a reproducible resolution for the new dependency.
  - confidence: 0.86
