## Summary

Keep open. Current main still lacks Control UI math rendering, and this PR is a relevant implementation attempt, but the branch is not mergeable because it removes existing markdown helpers, weakens link hardening, parses ordinary dollar prose as math, mishandles multi-backtick code spans, and adds KaTeX without lockfile proof.

## What This Changes

This PR adds KaTeX to the Control UI package and changes the shared markdown renderer and tests to render inline and display LaTeX math in chat.

## Best Possible Solution

Rework this PR, or a successor, so math rendering lands in the shared Control UI markdown renderer while preserving cache and link hardening, using robust delimiter rules, narrowing KaTeX-specific style handling, updating the lockfile, and covering currency prose plus code-span/code-fence cases.

## Reproduction Assessment

Yes. The feature gap is reproducible by inspecting current main: there is no math renderer dependency or math pipeline, while chat messages use the shared markdown renderer. The PR-head blockers are statically reproducible from the raw PR file because required helper calls remain after the helper definitions are removed.

## Solution Assessment

No. The shared renderer is the right surface, but this patch is not the best implementation because it breaks existing helper calls, broadens sanitizer attributes, omits the lockfile, and uses delimiter rules that corrupt common non-math text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the removed markdown helpers:** `ui/src/ui/markdown.ts:84-89`
  - body: This replacement block removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three. The PR will fail TypeScript/runtime rendering and drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98
- **[P2] Avoid parsing currency pairs as inline math:** `ui/src/ui/markdown.ts:151-153`
  - body: The inline-math detector treats any `$...$` pair as KaTeX, so normal prose like `between $5 and $10` is transformed into math. That corrupts common chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have `$...$` extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9
- **[P2] Update the lockfile for KaTeX:** `ui/package.json:17`
  - body: The PR adds `katex` to `ui/package.json` but does not update `pnpm-lock.yaml`. Frozen-lockfile installs will not have a reproducible resolution for the new dependency.
  - confidence: 0.86

## Security Review
