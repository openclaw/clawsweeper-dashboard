  - body: The inline scanner accepts any non-space `$...$` pair, so text like `between $5 and $10` is extracted as KaTeX instead of remaining ordinary chat text. That is a visible regression for normal non-math messages.
  - confidence: 0.92

  - body: The code-span bypass skips only until the next single backtick, so code spans such as ``$x$`` and fenced blocks can still have dollar content extracted and rendered as math. Code examples need to stay literal.
  - confidence: 0.9
- **[P2] Keep KaTeX styles scoped to KaTeX output:** `ui/src/ui/markdown.ts:60`
  - body: Adding `style` to the global DOMPurify allowlist permits inline styles on every allowed tag in the shared chat renderer, not just on KaTeX output. Use a narrow KaTeX-specific sanitizer hook or output path instead.
  - confidence: 0.84
