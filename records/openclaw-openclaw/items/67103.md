## Summary

Keep open for maintainer review. The feature is still not implemented on current main, but this branch has blocking renderer, sanitizer, parsing, dependency-lock, merge-conflict, and proof issues that make it unsafe to merge as-is.

## What This Changes

This PR adds a KaTeX dependency and changes the Control UI shared markdown renderer and tests to render `$...$` and `$$...$$` chat text as math.

## Best Possible Solution

Land or replace this with a maintainer-approved shared Control UI markdown renderer implementation that preserves cache/link hardening, scopes KaTeX-specific sanitizer allowances, uses robust delimiter rules, updates dependency lock state, and proves the visible UI behavior.

## Reproduction Assessment

Yes for source-level behavior: Control UI chat flows through `toSanitizedMarkdownHtml`, and current main has no math-rendering stage or KaTeX/MathJax dependency. The PR's blocking defects are also source-reproducible from the diff because removed helpers are still called later in the renderer.

## Solution Assessment

No. The shared renderer is the right surface, but this patch is not the best implementation because it removes required helpers, broadens sanitizer scope, omits lockfile proof, lacks real behavior proof, and uses fragile delimiter parsing.

  - body: This replacement block removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three. The branch will fail before rendering chat HTML and also drops the existing cache plus DOMPurify link-hardening hook.

  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have `$...$` extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9

  - body: The PR adds `katex` to `ui/package.json` but does not update `pnpm-lock.yaml`, so frozen-lockfile installs do not have a reproducible resolution for the new dependency.
  - confidence: 0.86

  - body: Adding `style` to the global allowed attribute list permits inline styles on every allowed element, not only rendered math. Keep the sanitizer surface narrow by allowing the style attribute only for trusted KaTeX output or by using a safer rendering strategy.
