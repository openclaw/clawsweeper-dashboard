## Summary

Keep open. Current main still lacks Control UI math rendering, and this PR targets the right shared markdown surface, but the proposed patch is not mergeable as-is because it removes existing markdown helpers, broadens sanitizer policy, misses dependency lockfile proof, and uses delimiter parsing that would corrupt common prose/code cases.

## What This Changes

This PR adds a KaTeX dependency, changes the shared Control UI markdown renderer to extract and restore `$...$`/`$$...$$` math as KaTeX HTML, and adds math rendering tests.

## Best Possible Solution

Rework this PR, or a successor, against current main so math rendering is added in the shared Control UI markdown renderer while preserving the existing cache and DOMPurify link hardening, using robust delimiter rules, narrowing any KaTeX-specific style allowance, updating the lockfile and changelog, and covering inline math, display math, invalid LaTeX, currency prose, single/multi-backtick code spans, and fenced code blocks.

## Reproduction Assessment

Yes. The feature gap is reproducible by inspecting current main: the Control UI package has no math renderer dependency and the shared markdown renderer has no math pipeline; the PR-head blockers are statically reproducible from the diff because helper definitions are removed while their call sites remain.

## Solution Assessment

No. The shared markdown renderer is the right surface, but this patch is not the best implementation because it deletes required helpers, broadens sanitizer attributes, omits lockfile/changelog updates, and parses common dollar-amount prose and multi-backtick code spans incorrectly.
