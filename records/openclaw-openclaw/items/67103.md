- **[P1] Restore the markdown helpers before calling them:** `ui/src/ui/markdown.ts:86-89`
  - body: This replacement block deleted `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three.   - confidence: 0.97
- **[P1] Add delimiter boundaries before extracting inline math:** `ui/src/ui/markdown.ts:151-158`
  - body: The inline scanner accepts any later `$` as the closing delimiter, so prose like `between $5 and $10` is transformed into KaTeX instead of staying plain text. Math rendering needs delimiter rules that avoid normal currency and dollar-sign chat text.
  - confidence: 0.92
- **[P2] Handle full Markdown code-span delimiters:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass only skips to the next single backtick, so code spans delimited with multiple backticks can still have `$...$` extracted and rendered. That breaks literal code examples that contain dollar signs.
  - confidence: 0.9
- **[P2] Keep style allowance scoped to KaTeX output:** `ui/src/ui/markdown.ts:60`
  - body: Adding `style` to the global DOMPurify allowlist permits inline styles on every sanitized markdown element, not only KaTeX nodes. Preserve the existing sanitizer boundary by allowing KaTeX-required styles only under trusted KaTeX output.
  - confidence: 0.84
