## Summary

Keep open. Current main still lacks Control UI math rendering, but this PR is not mergeable as-is because it breaks the shared markdown renderer helpers, weakens sanitizer handling, omits lockfile proof for a new dependency, and lacks contributor real-behavior proof.

## What This Changes

The PR adds KaTeX-based inline and display math rendering to the Control UI shared markdown renderer, plus a UI package dependency and renderer tests.

## Best Possible Solution

Rework this PR or a successor so math rendering lands in the shared Control UI markdown renderer while preserving cache and link hardening, using robust delimiter rules, scoped KaTeX sanitization, synced dependency metadata, and focused rendering tests.

## Reproduction Assessment

Yes, source-level. Current main has no math dependency or math-rendering stage in the shared renderer, and the PR-head blockers are statically visible because helper calls remain after their definitions are removed.

## Solution Assessment

No. The shared renderer is the right surface, but this patch is not the best implementation because it breaks existing helper calls, weakens sanitizer boundaries, omits the lockfile, and uses delimiter rules that corrupt common non-math text.

- **[P1] Restore the removed markdown helpers:** `ui/src/ui/markdown.ts:84-89`
  - body: This replacement block removes installHooks, getCachedMarkdown, and setCachedMarkdown, but toSanitizedMarkdownHtml still calls all three. The PR will fail TypeScript/runtime rendering and drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98
- **[P2] Avoid parsing currency pairs as inline math:** `ui/src/ui/markdown.ts:151-153`
  - body: The inline-math detector treats ordinary $...$ pairs as KaTeX with no delimiter-boundary rules, so prose like between $5 and $10 is transformed into math and corrupts common chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have $...$ extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9
- **[P2] Update the pnpm lockfile:** `ui/package.json:17`
  - body: The PR adds katex to ui/package.json but does not update pnpm-lock.yaml. Frozen-lockfile installs will not have a reproducible resolution for the new dependency.
