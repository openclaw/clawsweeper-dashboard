## Summary

Keep open. Current main still lacks Control UI math rendering, but this PR is not mergeable because the branch drops required markdown helper definitions, widens sanitizer behavior, misses dependency/changelog bookkeeping, and uses delimiter parsing that would corrupt common chat text.

## What This Changes

This PR adds KaTeX to the Control UI package and changes the shared markdown renderer/tests to render `$...$` and `$$...$$` chat math as KaTeX HTML.

## Best Possible Solution

Rework this PR, or a successor, so math rendering lands in the shared Control UI markdown renderer while preserving cache and link hardening, using robust delimiter rules, narrowing any KaTeX-specific style allowance, updating lockfile/changelog, and covering currency plus code-span/code-fence cases.

## Reproduction Assessment

Yes. The feature gap is reproducible by inspecting current main: chat uses the shared markdown renderer and no math renderer or math pipeline is present. The PR-head blockers are statically reproducible from the raw PR file because removed helper definitions still have call sites.

## Solution Assessment

No. The shared renderer is the right surface, but this patch is not the best implementation because it breaks existing helper calls, broadens sanitizer attributes, misses package bookkeeping, and uses delimiter rules that corrupt common non-math text.

## Review Findings

Full review comments:

- **[P1] Restore the removed markdown helpers:** `ui/src/ui/markdown.ts:84-89`
  - body: This replacement block removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three. The PR will fail TypeScript/runtime rendering and drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98
- **[P2] Avoid parsing currency pairs as inline math:** `ui/src/ui/markdown.ts:151-153`
  - body: The inline-math detector treats any `$...$` pair as KaTeX, so normal prose like `between $5 and $10` is transformed into math. That corrupts common chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span bypass searches only for the next single backtick, so spans delimited with multiple backticks can still have `$...$` extracted and rendered. Literal code examples with dollar signs should remain untouched.
  - confidence: 0.9
- **[P2] Update the lockfile for KaTeX:** `ui/package.json:17`
  - body: The PR adds `katex` to `ui/package.json` but does not update `pnpm-lock.yaml`. Frozen-lockfile installs will not have a reproducible resolution for the new dependency.
  - confidence: 0.88
