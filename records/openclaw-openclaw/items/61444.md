## Summary

Keep open. Current `main` still lacks an `agents import` command, but this PR head has blocking correctness, security-contract, and real-behavior-proof gaps before it is mergeable.

## What This Changes

The branch adds an `openclaw agents import <file>` command, CLI registration, archive import implementation, unit tests, and agents CLI docs.

## Best Possible Solution

Define a safe agents import contract: import existing helpers from the right modules, schema-validate `agent.json`, ignore or tightly constrain workspace metadata, stage filesystem writes around config commits, document the behavior, and prove a real import run.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR head imports non-exported names and gives a crafted `agent.json.workspace` a path to control where extracted `workspace/` contents are copied.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the best merge path until the module imports, archive trust boundary, metadata validation, mutation ordering, and real-behavior proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import file helpers from modules that export them:** `src/commands/agents.commands.import.ts:7`
  - body: `../infra/archive.js` does not export `fileExists` or `readJsonFile` on current `main`, so this command will fail to typecheck or load after rebasing. Import path existence from the fs-safe wrapper and JSON reading from the JSON helper module instead.
  - confidence: 0.94
- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` comes from `agent.json.workspace`, which is controlled by the imported archive, and later receives copied archive contents. Ignore this field by default or constrain it to an approved agent workspace root before copying files.
  - confidence: 0.91
- **[P2] Stage workspace writes around the config commit:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies extracted workspace files before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the import has already modified the filesystem without registering the agent.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be persisted unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the same archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts once to read `agent.json` and again to copy `workspace/`. Keep the staged extraction alive through validation and the final copy so large archives do not pay double extraction cost.
  - confidence: 0.82
