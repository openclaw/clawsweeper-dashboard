## Summary

Keep open. Current main still does not have an `agents import` command, but this PR is not ready to merge because the branch no longer matches current helper exports, trusts archive-controlled workspace destinations, can leave partial filesystem writes before config commit, lacks full metadata validation, and has no real behavior proof.

## What This Changes

The PR adds `openclaw agents import <file>` to import agent config and workspace contents from archives, with CLI registration, docs, and tests.

## Best Possible Solution

Define a safe archive import contract that schema-validates `agent.json`, ignores or tightly constrains workspace metadata, commits config before irreversible file writes or stages rollback safely, documents the format, and includes real CLI import proof.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR head against current main shows the helper-export mismatch, archive-controlled destination, partial-write ordering, and metadata validation gaps without needing a live run.

## Solution Assessment

No, not as submitted. The feature direction is plausible, but the implementation needs a safer archive trust boundary, correct current-main imports, better mutation ordering, full metadata validation, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import file helpers from modules that export them:** `src/commands/agents.commands.import.ts:7`
  - body: `../infra/archive.js` on current main does not export `fileExists` or `readJsonFile`, so this command will fail to typecheck or load after rebasing. Import path existence from the fs-safe wrapper and JSON reading from the JSON helper module instead.
  - confidence: 0.94
- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `agent.json.workspace` is archive-controlled, resolved as a user path, and then used as the destination for copied archive contents. Ignore this field by default or constrain it to an approved agent workspace root before writing files.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies extracted workspace files before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the import has already modified the filesystem without registering the agent.
  - confidence: 0.86
- **[P2] Validate all optional agent metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be persisted unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts once to read `agent.json` and again to copy `workspace/`. Keep the staged extraction alive through validation and final copy so large archives do not pay double extraction cost.
  - confidence: 0.82
