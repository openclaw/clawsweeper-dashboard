## Summary

Keep open: current main still lacks this command, but the PR head is not mergeable as-is because the importer trusts archive metadata for write destinations, can leave partial filesystem changes before config commit, and lacks real behavior proof.

## What This Changes

The branch adds an `openclaw agents import <file>` CLI command, docs, command export/registration, an archive importer, and unit tests.

## Best Possible Solution

Define a safe import contract that treats archives as untrusted: schema-validate `agent.json`, ignore or tightly constrain workspace metadata, stage or order mutations to avoid partial imports, and document the final behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an archive with `agent.json.workspace` set to a user-writable path and a `workspace/` payload is resolved as `targetWorkspace` and copied there before config commit.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the narrowest maintainable solution until the archive trust boundary, metadata schema, and mutation ordering are explicit and tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` comes from `agent.json.workspace`, which is controlled by the imported archive, and later receives copied archive contents. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require an explicit constrained override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` into `targetWorkspace` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be written into config unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts the archive once to read `agent.json` and then extracts it again to copy `workspace/`. Large imports pay the full extraction cost twice; keep the staged extraction alive through validation and copying.
  - confidence: 0.82
