## Summary

Keep open. Current main still does not provide `openclaw agents import`, and the PR is useful direction, but the proposed branch is not merge-ready because it still trusts archive-controlled workspace destinations, can leave partial filesystem changes when config write fails, and does not validate all untrusted `agent.json` fields before using them.

## What This Changes

The PR adds an `openclaw agents import <file>` CLI subcommand, a new archive-based agent config/workspace importer, CLI docs, and unit tests.

## Best Possible Solution

Keep the feature work open, but revise the importer around a safe archive contract: ignore or strictly constrain archive-specified workspace paths by default, validate `agent.json` with a runtime schema, stage extracted files with explicit conflict handling, avoid partial imports on config/write failures, and update docs/tests to cover malformed archives, overwrite behavior, and JSON output.

## Reproduction Assessment

Yes. Static inspection of the PR branch gives a high-confidence reproduction path: a crafted archive with `agent.json.workspace` set to a user-writable path and a `workspace/` directory will resolve that path and copy extracted files there before committing config.

## Solution Assessment

No. Adding an import command is a reasonable feature, but this branch is not the narrowest maintainable solution until it defines the trust boundary for archive metadata and makes import application validate-and-stage before mutating live state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain archive workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` is derived from archive-controlled `agent.json.workspace` and is later used as the destination for extracted files. A crafted import can redirect writes to arbitrary user-writable locations, so the importer should ignore this field by default or require an explicit, constrained operator override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace contents:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered, creating a partial import that is hard to recover from.
  - confidence: 0.86
- **[P2] Validate optional agent metadata before applying config:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but only `workspace` is type-checked. A non-string `name` can crash in `applyAgentConfig` when it calls `.trim()`, and a non-string `model` can be carried into config construction, so parse these fields with a schema before use.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces an archive import path that can write extracted files to a destination controlled by archive metadata.

Concerns:

- **[high] Archive-controlled workspace path:** `src/commands/agents.commands.import.ts:115`
