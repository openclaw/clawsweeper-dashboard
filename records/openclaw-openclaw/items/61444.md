## Summary

Keep open: current main still has no `agents import` command, and the PR is a plausible feature but still has blocking archive trust-boundary and mutation-ordering problems. Earlier docs/test feedback was partly addressed, but the later security findings and missing real behavior proof still block merge.

## What This Changes

The PR adds `openclaw agents import <file>` with archive extraction, agent config import, workspace copying, CLI registration, docs, and unit tests.

## Best Possible Solution

Define a safe import contract for untrusted archives: ignore or tightly constrain workspace metadata, schema-validate `agent.json`, order or stage mutations to avoid partial imports, document the final behavior, and require real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: an archive with `agent.json.workspace` set to a user-writable path and a `workspace/` payload is copied to that resolved destination before the config commit succeeds.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the safest maintainable solution until the archive trust boundary, metadata schema, and mutation ordering are explicit and tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` comes from `agent.json.workspace`, which is controlled by the imported archive, and later receives copied archive contents. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require a constrained explicit override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` into `targetWorkspace` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be written into config unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts the archive once to read `agent.json` and then extracts it again to copy `workspace/`. Large imports pay the full extraction cost twice; keep the staged extraction available through validation and copying after checks pass.
  - confidence: 0.78
