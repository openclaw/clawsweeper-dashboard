## Summary

Keep open. Current main still does not have an `agents import` command, and the PR is useful feature work, but the current branch is not safe to merge because the archive importer trusts archive-controlled metadata for write destinations and can partially mutate workspace state before config commit succeeds.

## What This Changes

The PR adds an `openclaw agents import <file>` CLI subcommand, CLI docs, command registration, an archive-based agent config/workspace importer, and unit tests.

## Best Possible Solution

Keep the feature open, but redesign the importer around a safe archive contract: validate `agent.json` with a runtime schema, ignore or tightly constrain archive-provided workspace paths by default, stage filesystem changes with explicit conflict handling and rollback/backups, then update docs/tests for malformed archives, overwrite behavior, JSON output, and non-interactive runs.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: create an archive with `agent.json.workspace` set to a user-writable destination and include a `workspace/` directory; the PR resolves that path and copies extracted contents there before committing config.

## Solution Assessment

No. Adding an import command is a reasonable direction, but this branch is not the best solution until the archive trust boundary and mutation ordering are made explicit and safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain archive workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` is derived from archive-controlled `agent.json.workspace` and later used as the destination for extracted files. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require an explicit constrained override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace contents:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate optional agent metadata before applying config:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and a non-string `model` can be written into config unexpectedly.
  - confidence: 0.88
