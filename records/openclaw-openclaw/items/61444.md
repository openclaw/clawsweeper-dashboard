## Summary

Keep this PR open: current main still lacks an `agents import` command, and the PR is a plausible feature, but the branch currently has blocking archive trust-boundary and partial-write issues plus no real behavior proof.

## What This Changes

The PR adds an `openclaw agents import <file>` CLI subcommand for archive-based agent import, with docs, command registration, importer implementation, and unit tests.

## Best Possible Solution

Define a safe import archive contract first: schema-validate `agent.json`, ignore or tightly constrain archive-provided workspace paths, stage config and file mutations with clear rollback/conflict behavior, and cover those cases in tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an archive with `agent.json.workspace` pointing to a user-writable directory plus a `workspace/` directory; PR head resolves that path and copies extracted files there before `replaceConfigFile`.

## Solution Assessment

No. Adding an import command is a reasonable feature direction, but this implementation is not the narrowest maintainable solution until archive metadata trust, destination constraints, and partial-write behavior are explicit and tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` is derived from archive-controlled `agent.json.workspace`, and later receives copied archive contents. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require an explicit constrained override.
  - confidence: 0.91
- **[P2] Stage filesystem writes behind a successful config commit:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` into `targetWorkspace` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime schema validation. A non-string `name` can crash in `applyAgentConfig`, and unsupported model shapes can be written unexpectedly.
  - confidence: 0.87
- **[P3] Reuse the validated extraction for workspace copy:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts the archive once to validate `agent.json` and then extracts it again to copy `workspace/`. Large exports pay the full extraction cost twice; keep one validated extraction alive through validation and copying.
  - confidence: 0.8
