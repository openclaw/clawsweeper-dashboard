The PR adds an `openclaw agents import <file>` command with archive parsing, workspace copying, config updates, CLI docs, and unit tests.

## Best Possible Solution

Define a safe importer that schema-validates `agent.json`, ignores or tightly constrains workspace metadata, stages config and file writes consistently, documents the archive format, and lands with real CLI proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import helpers from modules that export them:** `src/commands/agents.commands.import.ts:7`
  - body: `../infra/archive.js` on current main does not export `fileExists` or `readJsonFile`, so this new command will fail to typecheck or load after rebasing. Import path existence from the fs-safe wrapper and JSON reading from the JSON helper module instead.
  - confidence: 0.94
- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `agent.json.workspace` is archive-controlled, resolved as a user path, and then used as the destination for copied archive contents. Ignore this field by default or constrain it to an approved agent workspace root before writing files.
  - confidence: 0.91
- **[P2] Avoid copying files before the config commit:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies extracted workspace files before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the import has already modified the filesystem without registering the agent.
  - confidence: 0.86
- **[P2] Validate optional archive metadata before use:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be persisted unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the same archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts once to read `agent.json` and again to copy `workspace/`. Keep the staged extraction alive through validation and final copy so large archives do not pay double extraction cost.
  - confidence: 0.8
