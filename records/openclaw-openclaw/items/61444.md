## Summary

Keep open: current main still lacks the requested `agents import` command, but this PR remains incorrect because the importer uses non-exported helpers, trusts archive-controlled workspace destinations, can leave partial filesystem writes, and lacks real behavior proof.

## What This Changes

The PR adds an `openclaw agents import <file>` CLI command, archive import implementation, unit tests, and CLI docs for importing agent config and workspace archives.

## Best Possible Solution

Define a safe agents import contract: use exported helper modules, schema-validate `agent.json`, ignore or tightly constrain workspace metadata, stage writes so failed config commits do not leave partial imports, document the behavior, and prove a real import run.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR head and current main shows the bad imports, archive-controlled workspace destination, partial-write ordering, and missing metadata validation without needing a live run.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the best merge path until the module imports, archive trust boundary, metadata validation, mutation ordering, and real-behavior proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import file helpers from modules that export them:** `src/commands/agents.commands.import.ts:7`
  - body: `../infra/archive.js` does not export `fileExists` or `readJsonFile` on current main, so this command will fail to typecheck or load after rebasing. Import path existence from the fs-safe wrapper and JSON reading from the JSON helper module instead.
  - confidence: 0.94
- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` comes from archive-controlled `agent.json.workspace`, then receives copied archive contents. Ignore this field by default or constrain it to an approved agent workspace root before writing files.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies extracted workspace files before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the import has already modified the filesystem without registering the agent.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be persisted unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts once to read `agent.json` and again to copy `workspace/`. Keep the staged extraction alive through validation and the final copy so large archives do not pay double extraction cost.
  - confidence: 0.82
