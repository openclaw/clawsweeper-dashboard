## Summary

Keep this PR open: current main still has no `openclaw agents import` command, but the current branch is not merge-ready because it trusts archive-controlled workspace metadata, can leave partial filesystem writes if config commit fails, lacks full metadata validation, and only reports tests/checks rather than real import proof.

## What This Changes

Adds an `openclaw agents import <file>` CLI subcommand, docs, command registration/export, an archive-based agent config/workspace importer, and unit tests.

## Best Possible Solution

Redesign the importer around a safe archive contract: schema-validate `agent.json`, ignore or tightly constrain imported workspace paths, stage config and file mutations with rollback or safe ordering, then update focused tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to create an archive with `agent.json.workspace` set to a user-writable destination and a `workspace/` payload; the PR resolves that path and copies files there before committing config.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the narrowest maintainable solution until archive metadata validation, destination constraints, and partial-write behavior are explicit and tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` comes from `agent.json.workspace`, which is controlled by the imported archive, and later receives copied archive contents. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require an explicit constrained override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` into `targetWorkspace` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate imported metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and malformed model metadata can be written into config unexpectedly.
  - confidence: 0.87
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts the archive once to read `agent.json` and then extracts it again to copy `workspace/`. Large imports pay the full extraction cost twice; keep the staged extraction alive through validation and copying.
  - confidence: 0.82
