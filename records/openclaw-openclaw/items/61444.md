## Summary

Keep open: current main still lacks `openclaw agents import`, but this PR is not safe to merge because the importer trusts archive-controlled workspace metadata and can leave partial filesystem writes before the config commit.

## What This Changes

This PR adds an `openclaw agents import <file>` CLI subcommand, docs, command registration, an archive-based importer for agent config/workspace files, and unit tests.

## Best Possible Solution

Keep the feature open, but redesign the importer around a safe archive contract: schema-validate `agent.json`, ignore or tightly constrain archive-provided workspace paths, stage config and file mutations with clear conflict/rollback behavior, and update docs, tests, and changelog together.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to create an archive with `agent.json.workspace` set to a user-writable destination and a `workspace/` directory; the PR resolves that archive-chosen path and copies files there before committing config.

## Solution Assessment

No. Adding an import command is a reasonable feature direction, but this implementation is not the narrowest safe solution until archive metadata, destination constraints, and partial-write behavior are explicit and tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain archive-selected workspace destinations:** `src/commands/agents.commands.import.ts:115-117`
  - body: `targetWorkspace` is derived from `agent.json.workspace`, which is controlled by the imported archive, and later receives copied archive contents. A crafted archive can redirect writes outside the intended OpenClaw agent workspace, so ignore this field by default or require an explicit constrained override.
  - confidence: 0.91
- **[P2] Commit config before copying workspace files:** `src/commands/agents.commands.import.ts:173`
  - body: The command copies `workspace/` into `targetWorkspace` before `replaceConfigFile` runs. If the config write fails because of a hash conflict or I/O error, the command has already modified files while leaving the agent unregistered.
  - confidence: 0.86
- **[P2] Validate optional agent metadata before applying it:** `src/commands/agents.commands.import.ts:180-183`
  - body: `agent.json` is untrusted JSON, but `name` and `model` are passed through without runtime validation. A non-string `name` can crash in `applyAgentConfig`, and a non-string `model` can be written into config unexpectedly.
  - confidence: 0.88
- **[P3] Avoid extracting the archive twice:** `src/commands/agents.commands.import.ts:163-168`
  - body: The importer extracts the archive once to read `agent.json` and then extracts it again to copy `workspace/`. Large agent exports pay the full extraction cost twice; keep the temp extraction alive through validation and copying instead.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/cli/program/register.agent.ts:274`
  - body: This adds a user-facing `openclaw agents import` command, but the PR files do not include `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.83
