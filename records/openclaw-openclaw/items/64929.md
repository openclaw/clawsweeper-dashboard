## Summary

Keep open. Current main still sends local-managed browser profiles through raw CDP JSON tab discovery and short-lived CDP socket target creation, while the issue has concrete and corroborated timing reports plus a fresh comment pointing to connection/tab reuse as the likely fix direction.

## What This Changes

The issue asks to reduce local-managed browser CLI latency by avoiding repeated CDP attach, discovery, and target selection overhead, with benchmark or trace coverage for the hot path.

## Best Possible Solution

Add focused browser phase timing or a loopback benchmark, then optimize local-managed tab discovery and action dispatch to reuse prepared CDP/Playwright connection and target state while preserving SSRF policy, tab aliases, lifecycle cleanup, and fallback behavior.

## Reproduction Assessment

Yes. The reported CLI commands against a local-managed Brave or default Chrome profile are concrete, and current main still contains the repeated local-managed CDP discovery/attach path; this review did not run the macOS benchmark.

## Solution Assessment

Unclear for the exact implementation. Persistent Playwright routing, connection reuse, and target-state caching are plausible, but phase timing and browser-owner design should choose the narrowest maintainable fix before this SSRF-sensitive path changes.
