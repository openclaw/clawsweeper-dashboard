## Summary

Keep open. Current main still routes local-managed loopback browser profiles through raw CDP and repeated JSON/tab discovery rather than a reused persistent browser connection, and the issue has concrete, corroborated timing reports with no merged benchmark or lifecycle change proving the slowdown is gone.

## What This Changes

The issue asks for faster local-managed Brave/Chrome browser actions by avoiding repeated CDP attach/tab discovery overhead and adding performance diagnostics or benchmark coverage.

## Best Possible Solution

Add focused browser phase timing or a loopback action benchmark, then optimize local-managed connection and tab reuse while preserving SSRF policy, stable tab aliases, profile lifecycle, and cleanup behavior.

## Reproduction Assessment

Yes. ## Solution Assessment

Unclear for the exact implementation. Persistent Playwright routing, long-lived CDP connection reuse, and target-state caching are plausible, but phase timing should guide the narrowest maintainable fix before changing this SSRF-sensitive browser lifecycle path.
