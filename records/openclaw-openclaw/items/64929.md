## Summary

Keep open. Current main and the latest tagged release still match the reported local-managed browser control path: local-managed profiles avoid persistent Playwright, tab-scoped routes re-enter tab selection, selection re-lists tabs, and raw CDP helpers open short-lived sockets.

## What This Changes

The issue reports slow local-managed browser CLI/tool actions caused by repeated CDP attach, tab discovery, and target-selection overhead.

## Best Possible Solution

Add phase timing or a loopback action benchmark first, then make a targeted browser-plugin lifecycle change that reuses connection and tab state where safe while preserving SSRF policy, tab identity, cleanup, and profile ownership behavior.

## Reproduction Assessment

Yes for a high-confidence reproduction path: run the reported local-managed Brave or default managed Chrome open/wait/snapshot/console actions against https://example.com. I did not run that macOS benchmark here, but current source still matches the repeated discovery and short-lived socket path.

## Solution Assessment

Unclear for the exact implementation. Long-lived CDP or Playwright session reuse is plausible, but phase timing and browser security/lifecycle review should choose the narrowest maintainable change instead of a blanket capability flip.
