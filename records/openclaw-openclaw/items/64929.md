## Summary

Keep open. Current main and the latest shipped release still route local-managed browser profiles through repeated tab discovery, raw CDP tab creation, and short-lived CDP sockets; the related Playwright/SSRF issue is adjacent but does not cover the full performance, connection-reuse, benchmark, and timing-trace request.

## What This Changes

The issue requests faster local-managed Brave/Chrome browser CLI actions by reusing CDP/Playwright connection and tab state, reducing repeated discovery, adding loopback benchmarks, and documenting or tracing action timing.

## Best Possible Solution

Add phase timing or a loopback action benchmark first, then make a targeted browser-plugin lifecycle change that reuses connection and tab state where safe while preserving SSRF policy, tab identity, cleanup, and profile ownership behavior.
