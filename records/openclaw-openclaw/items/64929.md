## Summary

Keep open. Current main and the latest shipped release still keep local-managed browser profiles on raw CDP with repeated tab discovery and short-lived CDP sockets, and the open Playwright/SSRF follow-up does not cover the broader performance and phase-timing request.

## What This Changes

The issue requests faster local-managed browser actions by reusing CDP/Playwright connection and tab state, plus benchmark or timing diagnostics for the slow path.

## Best Possible Solution

Add browser phase timing or a loopback action benchmark, then optimize local-managed connection and tab reuse inside the browser plugin while preserving SSRF policy, tab aliases, profile lifecycle, and cleanup behavior.

## Reproduction Assessment

Yes. The high-confidence path is to run the reported managed Brave or default managed Chrome CLI actions against https://example.com; source inspection on current main still shows the repeated discovery and short-lived CDP socket path, though I did not run that timing benchmark here.

## Solution Assessment

Unclear for the exact implementation. Long-lived CDP or Playwright session reuse is plausible, but phase timing should guide the narrowest maintainable change and it should be coordinated with https://github.com/openclaw/openclaw/issues/67966 before changing lifecycle behavior.
