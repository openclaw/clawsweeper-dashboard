## Summary

Keep open. Current main still matches the reported slow path: local-managed profiles do not use persistent Playwright, ordinary actions still re-enter tab selection, and raw CDP tab creation still does discovery plus short-lived sockets.

## What This Changes

The issue requests faster local-managed browser actions by reducing repeated CDP attach, discovery, tab selection, and related per-action overhead.

## Best Possible Solution

Add focused browser phase timing or a loopback action benchmark, then optimize local-managed selection/open paths to reuse an owned connection or prepared tab state while preserving SSRF policy and cleanup semantics.

## Reproduction Assessment

Yes. Source inspection shows the reported current-main path still exists: local-managed actions resolve tabs through repeated listTabs calls, raw CDP open does /json/version plus withCdpSocket, and the socket is closed after each operation; I did not run the macOS timing benchmark here.

## Solution Assessment

Unclear. Long-lived Playwright/CDP connection reuse and target-state caching are plausible, but phase timing and browser-owner review should choose the narrowest maintainable lifecycle change.
