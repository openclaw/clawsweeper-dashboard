## Summary

Keep open. Current main still routes local-managed browser profiles through repeated tab discovery and raw CDP/json control paths, and the concrete timing reports plus the new long-lived-session suggestion are not superseded by the related local-managed Playwright/SSRF issue.

## What This Changes

Request to speed up local managed browser CLI actions by reusing connection/tab state, reducing repeated CDP discovery, and adding focused timing or benchmark coverage.

## Best Possible Solution

Add browser phase timing or loopback action benchmark coverage, then optimize local-managed connection and tab reuse inside the browser plugin while preserving SSRF policy, stable tab aliases, profile lifecycle, and cleanup semantics.

## Reproduction Assessment

Yes. The reported managed Brave/default Chrome commands against `https://example.com` give a clear benchmark path, and current source still performs per-action tab resolution plus raw CDP/json discovery and short-lived sockets; I did not run the macOS live benchmark here.

## Solution Assessment

Unclear for the exact implementation. Connection/tab reuse is plausible, but the maintainable fix should be guided by launch/connect/discovery/load/snapshot phase timings and coordinated with https://github.com/openclaw/openclaw/issues/67966 before changing lifecycle behavior.
