## Summary

Keep open. Current main still routes local-managed loopback browser profiles through raw CDP, repeated tab discovery, and short-lived CDP sockets, while the issue has concrete and corroborated timing reports plus a later long-lived-session suggestion that is not covered by the related SSRF/Playwright issue.

## What This Changes

The issue requests faster local-managed browser actions by reducing repeated CDP attach, tab discovery, and target-selection overhead, with benchmark/trace coverage for the loopback browser path.

## Best Possible Solution

Add browser phase timing or a loopback action benchmark, then optimize local-managed connection and tab reuse inside the browser plugin while preserving SSRF policy, tab aliases, profile lifecycle, and cleanup behavior.
