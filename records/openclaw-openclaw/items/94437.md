## What This Changes

The PR threads configured agent identity names into agent Runtime prompt metadata so `agent=...` can show `identity.name` while keeping the slot id as `agentId=...`, with focused prompt-param, prompt-rendering, and CLI prompt tests.

## Best Possible Solution

Update the identity resolver to use the same normalized agent-id contract as `resolveAgentConfig`, add a regression case for a configured id that normalizes differently from the runtime id, and then provide direct redacted terminal output showing the Runtime line after the fix.
