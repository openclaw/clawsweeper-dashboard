## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Resolving the existing `identity.name` in `buildSystemPromptParams` is the narrow maintainable fix because embedded, CLI, and command prompt paths already share that helper; adding prompt prefix or suffix config would be a broader feature.
