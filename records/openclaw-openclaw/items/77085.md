## Summary

Keep open: current main still routes doctor’s missing configured plugin repair through the catalog’s bare npm spec, so a beta host can reinstall stable `@openclaw/brave-plugin` even though the update path already has beta-channel selection.

## What This Changes

Requests `doctor --fix` to prefer matching beta prerelease plugin versions for beta OpenClaw hosts instead of reinstalling the stable plugin release.

## Best Possible Solution

Teach doctor’s missing configured plugin install repair to reuse beta-aware default-spec resolution, falling back to stable only when the matching beta tag is unavailable.

## Reproduction Assessment

Yes, by source inspection: a beta host with Brave configured reaches doctor’s missing-install repair, which installs the catalog’s bare `@openclaw/brave-plugin` spec. I did not execute the live npm install in this read-only sweep.

## Solution Assessment

No, current main is not sufficient because only the update path applies beta-channel spec selection. The narrow maintainable fix is to share that selection with doctor’s missing configured plugin install path.
