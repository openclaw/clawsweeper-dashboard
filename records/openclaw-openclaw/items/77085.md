## Summary

Keep open: current main still has a source-visible doctor repair path that installs the catalog's bare npm spec, while the documented plugin contract says beta-channel installs should prefer the npm beta tag.

## What This Changes

Request `doctor --fix` to install matching beta-channel official plugin releases on beta hosts instead of resolving the npm latest stable release.

## Best Possible Solution

Teach doctor’s configured-plugin repair path to reuse the beta-aware official plugin resolution policy, preferring `@beta` for beta hosts and falling back to stable only when no compatible beta release exists.

## Reproduction Assessment

Yes, from source inspection rather than a live `doctor --fix` run: the affected path installs the bare catalog npm spec, and npm currently resolves that spec to stable while `@beta` points to the reported beta version.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing beta-channel plugin update policy to doctor’s missing-install repair path instead of adding a new user setting or Brave-specific exception.
