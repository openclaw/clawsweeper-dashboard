## Summary

Keep open. Current main still starts the classic `openclaw onboard` wizard without an upfront duration expectation, the later setup path can perform channels, workspace/session bootstrap, web search, skills, plugin config, hooks, runtime dependency materialization, and finalization work, and the linked implementation PR is closed unmerged.

## What This Changes

Requested behavior is to show an upfront ETA or long-running setup expectation when `openclaw onboard` starts.

## Best Possible Solution

Add a concise early note in the classic CLI wizard before the main setup choices that explains onboarding can take several minutes or longer depending on auth, channels, daemon setup, skills, plugins, dependencies, and network work; mirror that expectation in the public onboarding docs and add focused wizard coverage without promising an exact universal runtime.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: default `openclaw onboard` reaches `runSetupWizard`, whose startup and QuickStart notes lack duration copy while later setup phases can do long-running work.

## Solution Assessment

Yes, with adjusted wording. A small UX/docs/test update is the narrowest maintainable fix; a fixed 40-minute promise would be less robust than clear variable-duration guidance.

Implement the requested onboarding duration expectation for https://github.com/openclaw/openclaw/issues/74399.
