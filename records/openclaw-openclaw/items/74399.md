## Summary

Keep open. Current main still starts the default classic `openclaw onboard` path without an upfront duration or long-running setup expectation, and the public onboarding docs describe the setup phases without time guidance. The issue is recent, unprotected, not paired with an open closing PR in the provided context, and narrow enough for a focused UX/docs fix.

## What This Changes

The issue requests an upfront ETA or long-running setup warning for `openclaw onboard`, based on a reported onboarding duration of about 40 minutes.

## Best Possible Solution

Add a concise early note to the classic CLI wizard before the main prompts begin, explaining that provider auth, channel setup, daemon install, skills, plugins, optional dependencies, and network operations can take several minutes or longer. Mirror the same expectation in the public CLI onboarding docs and avoid promising a fixed 40-minute duration without timing data.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: default `openclaw onboard` reaches `runSetupWizard`, whose initial header/intro/risk sequence and QuickStart note have no duration warning, while the docs list setup phases without time guidance.

## Solution Assessment

Yes, with wording adjusted. A small UX/docs update is the narrowest maintainable fix; a hard-coded 40-minute ETA should be avoided unless maintainers have platform-specific timing data.
