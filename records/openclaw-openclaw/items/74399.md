## Summary

Keep open. Current main still lacks an upfront ETA or long-running setup expectation in the classic `openclaw onboard` path, and open PR #75622 is already the linked closing implementation candidate, so this issue should remain open until that PR lands or is closed.

## What This Changes

The issue requests an upfront ETA or timeline warning during `openclaw onboard`, noting that first-time setup can take around 40 minutes.

## Best Possible Solution

Review and land PR #75622, or a narrow replacement, that adds concise early expectation-setting copy to the classic wizard and mirrors it in the public onboarding docs. The final wording should describe variable setup duration and major phases rather than asserting a universal fixed ETA.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing UX: default `openclaw onboard` reaches `runSetupWizard`, whose startup and QuickStart note contain no duration warning, while the docs list setup phases without time guidance.

## Solution Assessment

Yes. Keeping the issue open while the linked closing PR is reviewed is the correct repository workflow; the narrow maintainable fix is a small wizard/docs copy change with tests, not closing the issue before the implementation merges.
