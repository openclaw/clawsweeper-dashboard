## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74399 with a focused classic CLI onboarding UX/docs PR. Add an early note to `openclaw onboard` without `--modern`, near the start of `runSetupWizard`, that sets expectations before the main prompts: duration varies, fresh setup may take several minutes or longer, and provider auth, channel login, daemon install, skills, and optional dependencies can extend the run. Do not hard-code 40 minutes as a universal ETA unless maintainers provide measured support. Add or update focused wizard tests that assert the note appears in the default/QuickStart path. Mirror the guidance in the CLI onboarding docs, and add a user-facing CHANGELOG entry. Do not change plugin dependency handling, lockfiles, or unrelated onboarding behavior.
