## Repair Work Prompt

Fix the classic CLI onboarding UX gap for https://github.com/openclaw/openclaw/issues/74399. Add a concise early note in `runSetupWizard` before the main prompts begin that tells users fresh setup can take a while and lists the major phases at a high level, without hard-coding a universal 40-minute promise unless maintainer-approved. Mirror the expectation in the user-facing onboarding docs, and update existing wizard tests or add focused coverage that asserts the note appears. Include a user-facing CHANGELOG.md entry. Treat PR #75622 as prior context only; do not revive broad or over-specific timing copy from it blindly. Do not change provider/channel behavior, install behavior, or modern Crestodian onboarding unless a tiny docs mention is necessary for consistency.
