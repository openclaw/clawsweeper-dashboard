Add an upfront ETA or duration expectation to `openclaw onboard` so new users know setup can take a long time.

Add concise early wizard copy and matching docs that set a variable-duration expectation without promising an exact universal 40-minute runtime.

Implement the onboarding ETA expectation for https://github.com/openclaw/openclaw/issues/74399. Add a concise early note in the classic CLI wizard before the main prompts begin, explaining that model auth, channel setup, daemon install, skills, plugins, and optional dependency/network steps can take several minutes or longer. Mirror the same expectation in the public CLI onboarding docs. Add focused wizard coverage that asserts the note is shown. Do not hard-code a universal 40-minute promise unless the implementation also has measured support for that exact number.
