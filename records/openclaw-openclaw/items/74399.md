## Repair Work Prompt

Add a conservative setup-duration expectation for the classic `openclaw onboard` flow requested by https://github.com/openclaw/openclaw/issues/74399. Put the note early in `runSetupWizard` before the main setup prompts, cover it in `src/wizard/setup.test.ts`, and align public docs/FAQ wording so it does not conflict with the existing 5-15 minute guidance. Avoid promising a universal 40-minute ETA; describe that fresh setups can vary and may take longer when provider auth, channels, daemon install, skills, plugins, or optional dependencies are involved. Coordinate wording with the rate-limit heads-up request at https://github.com/openclaw/openclaw/issues/30067 but do not implement rate-limit-specific warnings unless the maintainer explicitly expands the scope. Do not include unrelated config, dependency, lockfile, WhatsApp, or provider changes. Add a changelog entry because this is user-facing CLI/docs behavior.
