## Summary

Keep open. Current main still lacks an upfront duration or long-running setup expectation in the default classic `openclaw onboard` wizard, and the public onboarding docs do not give users an ETA or time warning. The linked implementation PR #75622 was closed unmerged, so it does not implement or supersede this issue.

## What This Changes

Request to show an upfront ETA or duration expectation when `openclaw onboard` starts so beginners know setup may take a long time.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding wizard before the main prompts begin, explaining that model auth, channels, daemon setup, skills, plugins, and optional dependency/network steps can take several minutes or longer. Mirror the same expectation in the public onboarding docs, add focused wizard coverage, and avoid a universal hard-coded 40-minute promise unless timing data supports it.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: default `openclaw onboard` reaches `runSetupWizard`, whose startup and QuickStart notes lack duration copy while later steps can run long setup work.

## Solution Assessment

Yes, with adjusted wording. A small UX/docs/test update is the narrowest maintainable solution; promising exactly 40 minutes would be less robust than setting a clear variable-duration expectation.

In `src/wizard/setup.ts`, show an early note before the main prompts that fresh setup can take several minutes or longer depending on model auth, channel setup, daemon install, skills, plugins, optional dependencies, and network speed. Do not promise a universal hard-coded 40-minute ETA unless there is timing evidence. Add focused coverage in the existing setup wizard tests for the new startup note. Mirror the user-facing expectation in the public onboarding docs, likely `docs/start/wizard.md`, `docs/cli/onboard.md`, and/or `docs/reference/wizard.md` depending on current docs-list guidance.
