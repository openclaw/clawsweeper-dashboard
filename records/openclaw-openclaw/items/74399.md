## Summary

Keep open. Current main still does not show an upfront duration or timeline expectation when classic `openclaw onboard` starts, the public onboarding docs still omit that expectation, and the linked implementation PR is closed unmerged.

## What This Changes

Request to show beginners an upfront ETA or long-running setup warning during `openclaw onboard`.

## Best Possible Solution

Add concise early classic-wizard copy and matching public docs that set a variable-duration expectation and name the slow phases without promising one universal runtime.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path from `openclaw onboard` to `runSetupWizard`, where startup and QuickStart copy lack duration guidance while later setup phases can perform substantial work.

## Solution Assessment

Yes, with conservative wording. A small UX/docs/test change is the narrowest maintainable solution; exact 40-minute language should be avoided unless maintainers have measured timing data.

Add an early classic `openclaw onboard` note that sets a realistic duration expectation before the main prompts begin; avoid promising a universal hard-coded 40-minute ETA unless maintainers have measured support for it. Mention that fresh setup can take several minutes and may take longer when provider auth, channel setup, daemon install, skills, plugins, or optional dependencies are selected. Mirror the expectation in the public onboarding docs. Add/update focused wizard coverage for the new copy. Include a single-line user-facing CHANGELOG.md entry with reporter credit if appropriate.
