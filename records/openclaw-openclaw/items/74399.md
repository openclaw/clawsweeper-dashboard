## Repair Work Prompt

Fix the onboarding UX gap from https://github.com/openclaw/openclaw/issues/74399. Add a concise upfront note to the classic `openclaw onboard` wizard before the main prompts begin, setting expectations that fresh setup can take a while depending on provider auth, channel setup, daemon install, skills, optional dependencies, and network speed. Do not hard-code a universal 40-minute promise unless there is measured maintainer-backed data; prefer cautious variable-duration wording. Mirror the same expectation in the user-facing CLI onboarding docs, add/update focused wizard coverage, and add a single-line user-facing CHANGELOG.md entry. Do not change the modern Crestodian flow unless the existing code/docs make it necessary for consistency.
