## Summary

Keep open. Current main still routes default `openclaw onboard` to the classic wizard, and that flow plus the public onboarding docs still lack an upfront duration expectation; the related implementation PR #75622 is closed unmerged.

## What This Changes

Request to show an upfront ETA or duration timeline when `openclaw onboard` begins so new users know setup can take a long time.

## Best Possible Solution

Add cautious early classic-wizard copy and matching docs that set a variable-duration expectation, with test and changelog coverage, without promising one universal runtime.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path from default `openclaw onboard` to `runSetupWizard`, where startup and QuickStart copy lack duration guidance while later phases can run substantial setup work.

## Solution Assessment

Yes, with conservative wording. A small wizard/docs/test change is the narrow maintainable path; exact 40-minute language should be avoided unless maintainers have measured timing data.
