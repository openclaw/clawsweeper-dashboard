## Summary

Keep open. Current main still starts the classic `openclaw onboard` flow without an upfront ETA or timeline expectation, the onboarding docs do not mention expected setup duration, and the linked implementation PR #75622 is closed unmerged.

## What This Changes

Request to show an upfront duration or timeline expectation when `openclaw onboard` starts because setup can take a long time.

## Best Possible Solution

Add concise early CLI wizard copy plus matching docs that set a variable-duration expectation and outline the major phases without promising one universal runtime.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path from `openclaw onboard` to `runSetupWizard`, where startup and QuickStart copy lack duration guidance while later phases can run substantial setup work.

## Solution Assessment

Yes, with conservative wording. A small UX, docs, test, and changelog patch is the narrowest maintainable solution; exact 40-minute language should wait for measured timing data.
