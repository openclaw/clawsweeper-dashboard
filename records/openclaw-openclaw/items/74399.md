## Summary

Keep open. Current main still routes plain `openclaw onboard` into the classic wizard, and that path plus the public onboarding docs still lack an upfront duration or timeline expectation; the related implementation PR #75622 is closed unmerged.

## What This Changes

Add an upfront duration or timeline expectation during `openclaw onboard` so new users know setup may take a long time.

## Best Possible Solution

Add concise early classic-wizard copy and matching onboarding docs that set a variable-duration expectation without promising one universal runtime.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path from default `openclaw onboard` to `runSetupWizard`, where startup and QuickStart copy lack duration guidance while later phases can run substantial setup work.

## Solution Assessment

Yes, with cautious wording. A small UX/docs/test change is the narrowest maintainable solution; exact 40-minute language should be avoided unless maintainers have measured timing data.
