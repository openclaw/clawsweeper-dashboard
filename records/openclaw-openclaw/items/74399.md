## Summary

Keep open. Current main still lacks an upfront duration expectation in the default classic `openclaw onboard` flow, and the public onboarding docs still do not tell first-time users that setup can take several minutes or longer. The linked implementation PR #75622 was closed unmerged, so it does not implement or supersede the request.

## What This Changes

Request to show an upfront ETA, timeline, or long-running setup warning when `openclaw onboard` starts.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding wizard before the main setup prompts, setting a variable-duration expectation and naming the phases that can take time. Mirror that expectation in the public onboarding docs, add focused wizard coverage, and avoid promising an exact 40-minute duration unless timing data supports it.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: default `openclaw onboard` reaches `runSetupWizard`, whose startup and QuickStart notes lack ETA/duration copy while later setup phases can run channel, daemon, skills, plugin, and finalization work.

## Solution Assessment

Yes, with conservative wording. A small UX/docs/test update is the narrowest maintainable fix; a fixed 40-minute promise would be less robust than clear copy saying setup can take several minutes or longer depending on selected options and environment.
