## Summary

Keep open. Current main still routes default `openclaw onboard` into the classic interactive wizard without an upfront ETA or long-running setup expectation, and the public onboarding docs still describe the setup phases without time guidance. The linked #75622 was closed unmerged, so it does not resolve the issue.

## What This Changes

Request an upfront ETA or time-expectation note when `openclaw onboard` starts so new users know setup may take a long time.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding wizard before the main prompts begin, explaining that model auth, channels, daemon setup, skills, plugins, and optional dependency/network steps can take several minutes or longer. Mirror the same expectation in the public CLI onboarding docs, add focused wizard coverage, and avoid a universal hard-coded 40-minute ETA unless timing data supports it.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: default `openclaw onboard` reaches `runSetupWizard`, whose startup and QuickStart notes lack duration copy while later steps can run long setup work.

## Solution Assessment

Yes, with adjusted wording. A small UX/docs/test update is the narrowest maintainable solution; promising exactly 40 minutes would be less robust than setting a clear variable-duration expectation.
