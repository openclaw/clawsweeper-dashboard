## What This Changes

The issue reports opencode-go provider streams completing provider-side without propagating stream termination, leaving cron model calls stuck until recovery aborts them.

## Best Possible Solution

Land one provider-owned opencode-go stream termination guard, then decide whether the downstream stuck-recovery classification PR is also needed before closing this issue.

## Reproduction Assessment

## Solution Assessment
