## Summary

Keep open. Current main has fixed the session-store cache/load `structuredClone` hot path, but auth-profile cache, runtime snapshots, and runtime external OAuth overlay still clone through `structuredClone`, and maintainer discussion explicitly left this issue open for that remaining work plus benchmark/soak confirmation.

## What This Changes

Replace `structuredClone`-based session/auth store cloning with JSON-shaped cloning to reduce gateway native-memory growth under large stores.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, preserve JSON-shaped credential semantics and runtime-only OAuth boundaries, add focused no-structuredClone/mutation-isolation coverage, and validate with the reported benchmark or a production-shaped gateway soak before closing.
