## Summary

## What This Changes

The branch adds a topbar/login Control UI locale picker, changes Vietnamese i18n loading/defaults, adds broad inline Vietnamese dashboard/config copy, updates docs/changelog/tests, and carries update/CI release fixes.

## Best Possible Solution

Preserve English/browser-locale fallback behavior, implement Vietnamese through the existing generated i18n pipeline, make locale-dependent labels render-time reactive, drop unrelated release/update/CI churn by rebasing onto current main, and add visible proof of the picker and localized dashboard.

## Reproduction Assessment

Yes, source inspection is enough for the review findings: `translateConfigLabel` runs Vietnamese dictionaries for every non-English locale, and `resolveNavigatorLocale("")` now falls back to Vietnamese on the PR head.

## Solution Assessment

No. The maintainable path is to keep the existing browser-locale/English fallback contract, add the picker and Vietnamese strings through the generated locale system, and avoid a one-off Vietnamese helper spread through render code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate config translations to Vietnamese:** `ui/src/ui/views/config-form.shared.ts:1977`
  - body: This guard treats every non-English locale as Vietnamese. If a user chooses German, Spanish, Japanese, or another supported locale, `translateConfigLabel` and `translateConfigHelp` translate schema copy through the Vietnamese dictionaries, regressing existing locale support. Gate this path on `i18n.getLocale() === "vi"` or move the strings into locale bundles.
  - confidence: 0.93
- **[P1] Keep unsupported browser locales on the English fallback:** `ui/src/i18n/lib/registry.ts:174`
  - body: The fallback now returns `DEFAULT_APP_LOCALE` (`vi`) for blank or unrecognized navigator languages. Current main and the docs use browser-locale detection with English as the fallback, so first-time users with unsupported locales or no navigator will unexpectedly start in Vietnamese. Keep the fallback at `DEFAULT_LOCALE` and load Vietnamese only when selected or detected.
  - confidence: 0.9
- **[P2] Avoid freezing locale labels at module load:** `ui/src/ui/views/config.ts:435`
  - body: These `uiText(...)` calls are evaluated when the module loads, so changing the topbar language later will not update the already-built `SECTION_CATEGORIES` labels. Compute these labels during render or use getters/`t()` so the new picker can actually switch languages.
  - confidence: 0.78
