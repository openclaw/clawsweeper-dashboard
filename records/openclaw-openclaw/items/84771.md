## Summary

Keep open: current main and v2026.6.11 include important mitigations, but the broad startup event-loop saturation report is not implemented end to end. Provider-auth prewarm is still scheduled by default, ACP startup identity reconciliation remains serial, and `sessions.list` still synchronously loads and parses session stores before async row yielding can help.

## What This Changes

The issue reports gateway startup event-loop saturation from model/provider prewarm, session-lock/session-store work, and immediate agent/channel activity causing delayed or lost Discord handling, heap pressure, and restart cascades.

## Best Possible Solution
