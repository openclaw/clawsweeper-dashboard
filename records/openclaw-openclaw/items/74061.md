## Summary

Keep open. Current main still has a source-level reproduction for the Telegram final-reply local media failure: sandboxed reply media normalization drops absolute host-local paths before Telegram receives an upload, while the docs still describe policy-checked absolute local media as supported.

## What This Changes

The issue asks for Telegram final replies to send valid local `MEDIA:` paths as attachments instead of displaying `Media failed`.

## Best Possible Solution

Add a narrow, policy-checked sandbox-to-host media staging bridge for final replies using the existing agent-scoped outbound media access path, and decide explicitly whether cron direct delivery shares that bridge.

## Reproduction Assessment

Yes, at source level. A sandboxed reply with an absolute host-local `MEDIA:` source still goes through sandbox-only normalization and the checked-in test asserts the media is dropped before staging.

## Solution Assessment

No proposed patch is under review. The best fix is not unconditional host-path acceptance; it should reuse the existing policy-checked outbound media access and preserve negative sandbox escape coverage.
