## Summary

Keep open. Current main still drops the reported sandboxed final-reply `MEDIA:` host path before Telegram can upload it, and the maintainer data point about cron delivery appears to be the same missing sandbox-to-host media translation on a separate outbound path.

## What This Changes

The issue reports that Telegram final replies with local `MEDIA:` paths regress to `⚠️ Media failed.` instead of uploading the generated file.

## Best Possible Solution

Add a policy-checked sandbox-to-host media bridge for final-reply media that reuses existing agent-scoped outbound media access, then decide whether cron direct delivery should share that bridge.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: provide an absolute host-local `MEDIA:` path while a sandbox root exists, and current normalization drops it before Telegram delivery; tests were not run because this review is read-only.

## Solution Assessment

No proposed patch is under review. The safest solution is a narrow fallback through existing agent-scoped outbound media access, not unconditional host-path acceptance.
