## Summary

Keep open. Current main still has a source-level reproduction for the documented Telegram final-reply local `MEDIA:` failure, and the open generic artifact-delivery design issue is broader rather than a replacement for this explicit `MEDIA:` regression.

## What This Changes

The issue asks OpenClaw to deliver local files referenced by final-reply `MEDIA:` directives to Telegram instead of replacing them with `Media failed`.

## Best Possible Solution

Add a narrow, policy-checked sandbox-to-host media staging bridge for documented final-reply `MEDIA:` paths using the existing agent-scoped outbound media access path, with explicit negative coverage for sandbox escapes and a maintainer decision on cron scope.

## Reproduction Assessment

Yes, at source level. A sandboxed reply with an absolute host-local `MEDIA:` source still goes through sandbox-only normalization, and the checked-in test asserts the media is dropped before staging.

## Solution Assessment

No patch is under review. The best fix is not unconditional host-path acceptance; it should reuse existing file-read/media policy and preserve sandbox escape protections.
