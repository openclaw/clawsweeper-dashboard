## Summary

Keep open. Current main still contains the sandbox-first final-reply media normalization path that drops absolute host-local `MEDIA:` paths when a sandbox root exists, and the related cron report appears to be an adjacent missing sandbox-to-host media bridge rather than a shipped fix.

## What This Changes

The issue requests restoring Telegram delivery for local `MEDIA:` final-reply paths generated by agents after the April 26 release.

## Best Possible Solution

Add a narrow, policy-checked sandbox-to-host media bridge for final-reply media that preserves existing agent-scoped outbound media access and negative coverage for unrelated host paths; then decide whether cron direct delivery should share that bridge.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: provide an absolute host-local `MEDIA:` path while a sandbox root exists, and current normalization drops it into the `Media failed` fallback; the checked-in test asserts this drop.

## Solution Assessment

No proposed patch is under review. The safest implementation is a narrow policy bridge through existing outbound media access, not unconditional host-path acceptance in final replies or Telegram.
