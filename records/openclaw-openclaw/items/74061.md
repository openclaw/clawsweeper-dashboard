## Summary

Keep open. Current main still has a source-level reproduction for the reported Telegram final-reply media failure: sandboxed final replies route absolute host-local `MEDIA:` paths through sandbox-only normalization and drop them before Telegram receives an upload, while the docs still describe policy-checked host-local media as supported.

## What This Changes

Restore Telegram outbound delivery for agent final replies that include valid local `MEDIA:` file paths after the v2026.4.26 regression.

## Best Possible Solution

Add a narrow, policy-checked sandbox-to-host media bridge for final-reply media, reuse existing agent-scoped outbound media access, preserve negative coverage for unrelated host paths, and decide explicitly whether cron direct delivery should share the same bridge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: provide an absolute host-local `MEDIA:` path while a sandbox root exists, and current final-reply normalization drops it before Telegram upload; tests were not run because this was a read-only review.

## Solution Assessment

No proposed patch is under review. The best fix is not unconditional host-path acceptance; it should route only authorized local media through the existing agent-scoped outbound media access path and keep sandbox escape coverage.
