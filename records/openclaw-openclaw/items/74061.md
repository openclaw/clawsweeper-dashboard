## Summary

Keep open. Current main still preserves the reported failure mode for sandboxed final-reply local media, and the cron follow-up is useful corroboration for the same missing sandbox-to-host media translation on a second delivery path.

## What This Changes

The issue requests that Telegram delivery send valid local files referenced by final-reply `MEDIA:` directives instead of posting `⚠️ Media failed.` after the April 2026 release.

## Best Possible Solution

Add a narrow policy-checked sandbox-to-host media bridge for final-reply `MEDIA:` paths, and deliberately extend it to cron-owned delivery if that surface is in scope, while keeping negative tests for unrelated host paths.

## Reproduction Assessment

Yes. With a sandbox root present, an absolute host-local `MEDIA:` path is routed through `resolveSandboxedMediaSource`, rejected outside sandbox/managed roots, and the checked-in test expects the media to be removed without staging.

## Solution Assessment

No patch is under review. The safest direction is a policy-checked fallback through the existing agent-scoped outbound media access rules, not unconditional host-path acceptance or a Telegram-specific upload change.
