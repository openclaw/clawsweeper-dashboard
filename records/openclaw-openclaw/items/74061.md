## Summary

Keep open. Current main still preserves the reported failure mode: sandboxed final-reply local `MEDIA:` paths are resolved through sandbox-only mapping before agent-scoped host media access can authorize them, and the cron comment corroborates the same missing sandbox-to-host bridge on another delivery path.

## What This Changes

The issue asks OpenClaw to restore Telegram delivery of local files referenced by final-reply `MEDIA:` directives, including sandbox-to-host translation for generated files.

## Best Possible Solution

Add a policy-checked media bridge for final replies and cron outbound delivery that translates sandbox workspace paths to host paths and falls back to existing agent-scoped media access only when filesystem policy allows it.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the checked-in test asserts that a sandboxed final reply with an absolute host-local media path is dropped and not staged, and the current normalizer still follows that path.

## Solution Assessment

No proposed patch is under review. The maintainable direction is a narrow policy-aware bridge or fallback, not unconditional host-path acceptance and not a Telegram adapter change.
