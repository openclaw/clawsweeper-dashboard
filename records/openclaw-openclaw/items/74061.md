## Summary

Keep open. Current main still has a source-level reproduction for the Telegram final-reply local media failure: sandboxed reply-media normalization can drop absolute host-local paths before Telegram receives an upload, while current docs still describe policy-checked absolute local media as supported.

## What This Changes

The issue requests restoring Telegram final-reply delivery of local `MEDIA:` paths from agent workspaces instead of falling back to `⚠️ Media failed.`

## Best Possible Solution

Add a narrow, policy-checked sandbox-to-host media staging bridge for final replies using existing agent-scoped outbound media access, and explicitly decide whether cron direct delivery shares that bridge.

## Reproduction Assessment

Yes, at source level. A sandboxed reply with an absolute host-local `MEDIA:` source still goes through sandbox-only normalization, and the checked-in test asserts that media is dropped before staging.

## Solution Assessment

No proposed patch is under review. The best fix is not unconditional host-path acceptance; it should reuse policy-checked outbound media access and preserve negative sandbox escape coverage.
