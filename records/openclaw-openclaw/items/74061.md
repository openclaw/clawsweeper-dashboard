## Summary

Keep open. Current main still has a source-level path where sandboxed final-reply media normalization drops absolute host-local `MEDIA:` paths before Telegram can upload them, despite docs describing policy-checked host-local outbound media as supported.

## What This Changes

Allow Telegram final replies to deliver supported local `MEDIA:` files instead of falling back to `Media failed`.

## Best Possible Solution

Add a narrow, policy-checked bridge for final-reply local media that reuses agent-scoped outbound media access, keeps sandbox escape tests, and explicitly decides whether cron direct delivery shares that bridge.

## Reproduction Assessment

Yes, as a source-level reproduction. With a sandbox workspace present, an absolute host-local final-reply `MEDIA:` path is sent through sandbox-only resolution, rejected as outside the sandbox root, dropped by the normalizer, and replaced with the warning text.

## Solution Assessment

No proposed patch is under review. The maintainable fix is not unconditional host-path acceptance; it should route only authorized local media through the existing outbound media access policy and preserve negative sandbox coverage.
