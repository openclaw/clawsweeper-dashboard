## Summary

Keep open. Current main still has generic exact-argument tool-loop detection, but it does not normalize exec readback commands by target file or emit the requested in-turn soft warning; this remains a core runtime policy/UX decision.

## What This Changes

The issue proposes a warning-only exec-runtime detector for repeated read-only shell rereads of the same file or file pair within one turn.

## Best Possible Solution

Extend the existing loop-safety surface with a normalized exec-readback detector for obvious read-only commands, privacy-safe diagnostics, focused tests, and docs after maintainer policy decisions.

## Reproduction Assessment

Yes, at source level. Current main hashes full exec params and only compares exact patterns, so different sed/nl/cat readback windows over the same file are not collapsed to a file identity.

## Solution Assessment

Unclear. The warning-only guard is plausible and narrow, but thresholds, default enablement, config shape, synthetic-result behavior, and telemetry need maintainer decisions first.
