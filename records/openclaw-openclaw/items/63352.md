## Summary

Keep open: current main still has generic exact-argument tool-loop detection, but it does not normalize exec readback commands by target file or emit the requested in-turn soft warning; this remains a core runtime policy decision.

## What This Changes

The issue proposes a warning-only per-turn detector for repeated exec-based file rereads, normalized by target file or file pair, to reduce quota burn and timeout failures.

## Best Possible Solution

Extend the existing loop-safety surface with a maintainer-approved exec-readback detector for obvious read-only commands, cwd-relative file normalization, privacy-safe diagnostics, focused tests, and docs aligned with the first-class read-tool sibling.

## Reproduction Assessment

Yes, at source level. Current main hashes full exec params and compares generic tool-call patterns, so sed/nl/cat rereads with different command text over the same file are not collapsed to one target-file identity.

## Solution Assessment

Unclear. The warning-only detector is plausible and narrow, but thresholds, default enablement, config shape, warning visibility, and telemetry privacy need maintainer decisions first.
