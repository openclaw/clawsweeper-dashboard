## Summary

Keep open. Current main still has generic exact-argument tool-loop detection, but not the requested path-normalized exec readback detector or warning-result nudge, and the remaining work needs runtime policy decisions before implementation.

## What This Changes

Request a warning-only exec runtime detector for repeated read-only rereads of the same file or file pair within one turn.

## Best Possible Solution

Design an exec-readback detector under the existing loop-detection surface that normalizes obvious read-only shell commands to target paths, emits a maintainer-approved soft warning/nudge, logs privacy-safe diagnostics, and ships focused tests/docs.

## Reproduction Assessment

Yes, at source level. Current main hashes exact full exec params, so repeated `sed`, `nl | sed`, or `cat` calls over the same file with different line windows remain distinct patterns rather than a normalized file identity.

## Solution Assessment

Unclear. The warning-only exec-side guard is plausible and narrow, but maintainers still need to decide thresholds, config shape, synthetic-result behavior, telemetry, and alignment with the first-class read guard.
