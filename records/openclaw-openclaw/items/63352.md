## Summary

Keep open. Current main has generic tool-loop detection and exec outcome hashing, but it still does not implement the requested exec-side read-only command classifier, cwd/path normalization, line-window collapsing, same-file/file-pair thresholds, warning-only synthetic result, or trigger telemetry before another redundant readback command runs.

## What This Changes

The issue proposes a warning-only exec-runtime detector for repeated read-only shell rereads of the same file or file pair within one turn.

## Best Possible Solution

Add a narrow core exec/tool-loop enhancement: classify only obvious read-only shell readbacks, normalize cwd plus target file paths while collapsing line-window variants, track same-file and same-file-pair counts per run/session, return a warning-only synthetic result before the next redundant read executes, emit structured telemetry, and document the behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: repeated exec calls with slightly different sed/nl/cat command strings for the same file would hash as different params and continue through runExecProcess unless an existing exact-params or generic ping-pong threshold happens to trigger.

## Solution Assessment

Unclear as an implementation path. The proposed warning-only direction is plausible and narrow, but threshold defaults, synthetic-result semantics, and coordination with #64426 need maintainer direction before this should be automated.
