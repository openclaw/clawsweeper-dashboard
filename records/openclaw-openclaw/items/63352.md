## Summary

Keep open. Current main still only detects generic repeated tool calls keyed by full params/outcomes, so it does not implement an exec readback classifier, target-file/file-pair normalization, line-window collapsing, or a warning-result nudge.

## What This Changes

Add a warning-only exec-runtime detector that normalizes repeated read-only shell readbacks by target file or file pair to reduce quota burn and timeout failures.

## Best Possible Solution

Add a maintainer-approved narrow detector for obvious read-only exec readbacks that normalizes target files per run, emits a soft in-turn warning result, logs sanitized trigger metadata, and has focused regression coverage.

## Reproduction Assessment

Yes, at source level. Repeated exec calls that differ only by sed/nl line windows hash as distinct command params, and current detection never collapses them to the same file identity.

## Solution Assessment

Unclear. The requested warning-only guard is plausible, but maintainers still need to choose the read-only command grammar, thresholds, telemetry shape, and synthetic-result behavior.
