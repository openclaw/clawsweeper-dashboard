## Summary

Keep open. Current main still only has generic tool-loop detection and exec outcome hashing; it does not implement the requested exec readback classifier, same-file/file-pair normalization, line-window collapsing, warning-only synthetic tool result, or target-file telemetry before another redundant shell read runs.

## What This Changes

Add a warning-only exec-runtime detector that recognizes repeated read-only shell reads of the same file or file pair within one turn and emits telemetry before executing another redundant read.

## Best Possible Solution

Add a narrow core exec/tool-loop enhancement that classifies only obvious read-only shell readbacks, normalizes cwd plus target file paths while collapsing line-window variants, tracks same-file and same-file-pair counts per run/session, returns a warning-only synthetic result before the next redundant read executes, emits structured telemetry, and updates config/docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: repeat exec calls such as sed, nl piped to sed, and cat against the same file with slightly different ranges, and current main hashes each distinct command string instead of collapsing to a normalized file identity.

## Solution Assessment

Unclear. The warning-only exec-side direction is plausible and narrow, but maintainers still need to choose defaults, config surface, telemetry shape, synthetic-result behavior, and coordination with #64426.
