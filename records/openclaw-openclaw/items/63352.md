## Summary

Keep open. Current main still has only generic exact-argument tool-loop detection around exec, so it does not implement the requested target-file/file-pair classifier, line-window collapsing, warning-only synthetic result, or readback telemetry before another shell read runs.

## What This Changes

Request a warning-only exec runtime detector that normalizes obvious shell readback commands by target file or file pair to interrupt same-turn reread loops.

## Best Possible Solution

Add a narrow maintainer-approved exec/tool-loop enhancement that recognizes only obvious read-only shell readbacks, normalizes target paths, tracks same-file and file-pair counts per run/session, emits a soft in-turn warning result, and covers it with focused tests and docs.

## Reproduction Assessment

Yes, at source level. Repeated exec calls such as sed, nl piped to sed, and cat against the same file with different line windows still hash as distinct full command params rather than a normalized file identity.

## Solution Assessment

Unclear. The proposed warning-only exec-side guard is plausible and narrow, but maintainers still need to choose defaults, config surface, synthetic-result behavior, telemetry shape, and alignment with https://github.com/openclaw/openclaw/issues/64426.
