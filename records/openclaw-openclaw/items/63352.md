## Summary

Keep open. Current main has generic tool-loop detection and exec outcome hashing, but it still lacks the requested exec-side readback classifier, cwd/path normalization, line-window collapsing, same-file/file-pair thresholds, warning-only synthetic tool result, and target-file telemetry before another redundant shell readback runs.

## What This Changes

The issue requests a warning-only exec runtime detector for repeated same-file or same-file-pair read-only shell rereads within one turn, with normalized file identity and telemetry.

## Best Possible Solution

Add a narrow core exec/tool-loop enhancement: classify only obvious read-only shell readbacks, normalize cwd plus target file paths while collapsing line-window variants, track same-file and same-file-pair counts per run/session, return a warning-only synthetic result before the next redundant read executes, emit structured telemetry, and document/configure the behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: repeat exec calls such as sed, nl piped to sed, and cat against the same file with slightly different ranges; current main hashes each command string as distinct params unless the command is exactly identical.

## Solution Assessment

Unclear. The warning-only direction is plausible and narrow, but maintainer choices are still needed for default enablement, thresholds, synthetic-result behavior, telemetry shape, and coordination with #64426.
