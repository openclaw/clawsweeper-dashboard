## Summary

Keep open: current main still has generic exact-argument tool-loop detection, but it does not normalize exec readback commands by target file or emit the requested in-turn soft warning; the remaining work is a core runtime policy choice.

## What This Changes

Requests a warning-only exec-runtime detector that recognizes repeated read-only shell commands against the same file or file pair within one turn.

## Best Possible Solution

Extend the existing loop-safety surface with a maintainer-approved exec-readback detector for obvious read-only commands, cwd-relative file normalization, privacy-safe diagnostics, focused tests, and docs aligned with the first-class read-tool sibling.

## Reproduction Assessment

Yes, at source level. Current main hashes full exec params and only compares generic patterns, so sed/nl/cat rereads with different line windows over the same file are not collapsed to one target identity.

## Solution Assessment

Unclear. The warning-only detector is plausible and narrow, but thresholds, default enablement, config shape, warning visibility, and telemetry need maintainer decisions first.
