## Summary

Keep open. Current main still has generic exact-argument tool-loop detection around exec, but not the requested path-normalized same-file/file-pair readback detector or warning-result nudge, and the remaining work needs maintainer policy choices before implementation.

## What This Changes

Add a warning-only exec-runtime detector for repeated read-only shell rereads of the same file or file pair within one turn.

## Best Possible Solution

Add a maintainer-approved exec readback detector under the existing loop-detection surface that recognizes obvious read-only shell commands, normalizes target paths, emits an in-turn soft warning, logs safe telemetry, and has focused tests/docs.

## Reproduction Assessment

Yes, at source level. Repeated exec calls such as sed, nl piped to sed, and cat against the same file with different line windows still hash as distinct full command params rather than a normalized file identity.

## Solution Assessment

Unclear. The proposed warning-only exec-side guard is plausible and narrow, but maintainers still need to decide thresholds, config shape, synthetic-result behavior, telemetry, and alignment with the first-class read guard.
