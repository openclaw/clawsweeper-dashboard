## Summary

Keep open. The feature direction is useful and related to the open session lifecycle request, but this PR should not merge as-is because it can emit duplicate/mis-keyed failed-spawn `session_end` events and misses cron terminal paths after a run session has already been created.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent completion, updates hook type comments, and adds an Unreleased changelog entry.

## Best Possible Solution

Implement `session_end` as an exactly-once lifecycle signal from the owning lifecycle paths: rely on the existing delete-owned emission for failed provisional subagent sessions, and add a cron terminal/finally-style emission after a run session exists and is persisted, with focused regression tests and current hook-type docs/changelog updates.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available from current main plus the PR diff: failed-spawn cleanup still reaches `sessions.delete`, which already emits `session_end`, while cron abort/error returns can bypass the proposed post-finalize hook block.

## Solution Assessment

No. The proposed approach is not the best fix because it adds an ad hoc failed-spawn emission with the wrong id and duplicate delivery risk, and it does not centralize cron terminal emission across every already-started exit path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the stored session id for failed-spawn session_end:** `src/agents/subagent-spawn.ts:717-718`
  - body: The added payload sets `sessionId` to `childRunId`, but that value is the agent run/idempotency id rather than the stored child session entry id. Plugins correlating `session_start`/`session_end` or looking up session metadata by id would receive an id that does not identify the ended session.
  - confidence: 0.88
- **[P2] Avoid emitting a second failed-spawn session_end:** `src/agents/subagent-spawn.ts:721`
  - body: After this manual emission, the cleanup path still calls `sessions.delete`. On current main, `sessions.delete` emits `session_end` for the deleted session regardless of `emitLifecycleHooks`, so listeners would see two end events for one failed child session.
  - confidence: 0.86
- **[P2] Emit cron session_end from every terminal path:** `src/cron/isolated-agent/run.ts:696-705`
  - body: This block only runs on the post-finalize path. If the cron run aborts before finalize or `executeCronRun` throws after the run session has been prepared, the cron session has ended but no `session_end` hook fires, leaving cleanup plugins polling on failure cases.
  - confidence: 0.84

  - body: The new user-facing hook feature entry lacks the required `Thanks @mitchelldavis44` attribution. Changelog audit policy expects contributor credit on added `feat` entries when the source author is not a forbidden bot or maintainer handle.
