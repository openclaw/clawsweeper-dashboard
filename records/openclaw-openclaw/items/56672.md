## Summary

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent terminal paths, exports hook helpers through runtime barrels, updates cron test-harness mocks, and documents the new hook trigger cases.

## Best Possible Solution

Use one idempotent session lifecycle finalizer: keep delete-owned subagent cleanup, add cron terminal emission exactly once for success, abort, preflight, and thrown paths, then prove it with focused tests and redacted real plugin output.

## Reproduction Assessment

Yes for the PR findings by source inspection: current main's delete handler already emits deleted `session_end`, cron preparation can return before the new helper, and finalize/catch can both emit after a post-persist throw. I did not run tests because this is a read-only review.

## Solution Assessment

No. Extending `session_end` is a reasonable direction, but this patch scatters emissions across multiple owners instead of using one idempotent lifecycle finalization contract.
