## Summary

Keep open. The requested lifecycle-hook direction is still useful and related issues remain open, but this PR branch is not merge-ready: the failed-spawn path would emit an extra `session_end` with the run/idempotency id instead of the stored session id, and the cron hook placement does not cover every terminal cron path on current `main`.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent completion, updates hook type comments, and adds an Unreleased changelog entry.

## Best Possible Solution

Implement `session_end` as an exactly-once lifecycle signal from the owning lifecycle paths: keep relying on `sessions.delete` for failed provisional subagent sessions, and add a cron terminal/finally-style emission after a run session exists and has been persisted, with focused tests, hook docs, and a credited changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from the PR diff and current `main`: failed-spawn cleanup still reaches `sessions.delete`, which already emits `session_end`, while cron abort/error returns can bypass the PR's proposed post-finalize hook block.

## Solution Assessment

No. The proposed patch is not the best way to solve the issue because it adds an ad hoc failed-spawn emission with the wrong id and duplicate-delivery risk, and it does not centralize cron terminal emission across every already-started exit path.
