- **[P1] Restore the current parent-fork export:** `src/agents/subagent-spawn.runtime.ts:9`
  - body: This re-export points at `resolveParentForkMaxTokens`, but current main's `session-fork.ts` does not export that symbol. The PR will fail typechecking/build before any new hook behavior can run; keep `resolveParentForkDecision` or add a real exported helper with coverage.
  - confidence: 0.97
- **[P2] Preserve thread-bound spawn context policy:** `src/agents/subagent-spawn.ts:650`
  - body: Hard-coding omitted `context` to `isolated` bypasses `resolveThreadBindingSpawnPolicy`, so thread-bound subagents no longer default to `fork` or honor `session.threadBindings.defaultSpawnContext`. Existing channel thread workflows would silently lose parent transcript context.
  - confidence: 0.94
- **[P2] Keep oversized forks as isolated fallback:** `src/agents/subagent-spawn.ts:356-364`
  - body: This changes the current parent-fork size policy from a graceful isolated fallback into a spawn error. Current `resolveParentForkDecision` and the context-mode regression test expect an oversized parent context to start the child isolated with a note, not fail the subagent spawn.
  - confidence: 0.91
- **[P2] Use the stored child session id:** `src/agents/subagent-spawn.ts:1146`
  - body: The manual `session_end` event uses `childIdem`, which is the gateway idempotency/run seed, not the session store id assigned to the provisional child session. Plugins correlating `session_start`/`session_end` or loading session metadata by `sessionId` will receive an id that does not identify the ended session.
  - confidence: 0.9
- **[P2] Emit cron session_end for preparation failures:** `src/cron/isolated-agent/run.ts:1090`
  - body: The new hook helper only runs after `prepared.ok`, but model-selection and provider-preflight failures return terminal skipped/error results from preparation after a run session identity is created. Those cron runs still end without `session_end`, which is one of the gaps this PR is meant to close.
  - confidence: 0.86
- **[P2] Preserve lazy-loader retry semantics:** `src/cron/isolated-agent/run.ts:64-104`
  - body: Replacing `createLazyImportLoader` with raw memoized promises makes a rejected runtime import sticky for the process. The shared loader intentionally evicts rejected loads by default so later cron turns can recover from transient load failures; keep the loader or reproduce that behavior.
  - confidence: 0.82
