## Summary

Keep open. This PR targets useful lifecycle-hook work related to https://github.com/openclaw/openclaw/issues/10142 and https://github.com/openclaw/openclaw/issues/55943, but the current branch still has duplicate and omitted `session_end` paths plus no real plugin proof, so it needs contributor proof and maintainer lifecycle/API direction before merge.

## What This Changes

The branch adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent terminal paths, exports hook helpers through runtime barrels, and updates hook JSDoc/test-harness mocks.

## Best Possible Solution

Use a single-owner lifecycle finalizer: let gateway deletion own deleted subagent `session_end`, and add one idempotent cron terminal helper covering success, abort, prepared failures, and thrown finalization paths exactly once, with focused tests and real plugin proof.

## Reproduction Assessment

Yes for the PR review findings by source inspection: current main plus the PR diff show the duplicate delete-owned hook path, the prepared cron failure gap, and the cron double-emission path. I did not run tests because this is a read-only review.

## Solution Assessment

No. Extending the lifecycle hook is a reasonable direction, but this patch adds ad hoc emissions across multiple owners instead of one idempotent lifecycle finalization contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Let sessions.delete own failed-spawn session_end:** `src/agents/subagent-spawn.ts:1187-1204`
  - body: This block emits `session_end` when `emitLifecycleHooks` is false and then still calls `sessions.delete` for the same child session. Current main's delete handler already runs the plugin `session_end` for deleted rows before it applies `emitLifecycleHooks` only to the subagent unbind hook, so listeners can receive duplicate end events; the manual payload also uses the idempotency key instead of the stored session id.
  - confidence: 0.94
- **[P2] Emit cron session_end for prepared failures:** `src/cron/isolated-agent/run.ts:767-783`
  - body: `emitCronSessionEnd` only accepts `PreparedCronRunContext`, but preparation can assign `runSessionId` and `runSessionKey` before returning `ok:false` for model-selection or provider-preflight failures. `runCronIsolatedAgentTurn` returns that prepared result before this helper can run, so terminal cron error/skipped runs still miss the hook this PR is meant to provide.
  - confidence: 0.88
- **[P2] Keep cron session_end one-shot after finalization errors:** `src/cron/isolated-agent/run.ts:1095`
  - body: `finalizeCronRun` emits immediately after persistence, but later payload resolution and delivery dispatch can still throw. The outer catch also emits for the same prepared context, so a post-persist failure can deliver two `session_end` events for one cron run.
  - confidence: 0.9
