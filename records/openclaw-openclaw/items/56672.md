## Summary

Keep open: the hook expansion is useful, but the current patch still has source-reproducible lifecycle defects that can duplicate or miss `session_end` emissions, and the user-facing feature still needs a changelog entry.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawns and cron isolated-agent run endings, plus runtime-barrel exports, hook type docs, and cron test-harness mocks.

## Best Possible Solution

Centralize `session_end` as a one-shot lifecycle event: gateway deletion should own deleted subagent sessions, and cron should use one guarded terminal path that covers success, skip, error, abort, and thrown outcomes exactly once.

## Reproduction Assessment

Yes. Source and diff inspection gives a high-confidence path: failed-spawn cleanup manually emits before `sessions.delete`, while cron prepared failures and post-finalize throws still bypass or duplicate the new cron emission.

## Solution Assessment

No. The requested hook expansion is reasonable, but this implementation should use existing lifecycle owners and an idempotent cron terminal path instead of ad hoc emissions at multiple exits.
