## Summary

Keep open. The hook coverage direction is useful, but the latest PR still adds a failed-spawn `session_end` path that conflicts with current `sessions.delete` behavior and can produce a duplicate, uncorrelatable end event.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent termination, exports hook helpers through runtime barrels, documents the added cases, and updates the changelog.

## Best Possible Solution

Keep the cron `session_end` emission direction, but remove the failed-spawn manual emission or share a helper that emits exactly once with the stored session id, then add focused lifecycle tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the PR emits `session_end` before `sessions.delete`, while current main already emits the plugin hook during deletion using the stored session entry id.

## Solution Assessment

No. The requested lifecycle signal is the right product direction, but failed-spawn cleanup should let the gateway delete path own `session_end` or use the same stored-id helper instead of sending a second event with the idempotency key.
