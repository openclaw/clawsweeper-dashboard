## Summary

Keep open: the feature is useful and not fully implemented on current main, but this PR still has blocking regressions around a non-exported fork helper, thread-bound context policy, duplicate failed-spawn hook emission, and cron prepare-failure paths.

## What This Changes

This PR extends the typed plugin `session_end` hook to failed subagent spawn cleanup and isolated cron run termination, updates runtime barrels/JSDoc, and adds a changelog entry.

## Best Possible Solution

Repair the PR on current main with exactly-once `session_end` emission in the owning lifecycle paths, preserving existing subagent fork policy and covering all terminal isolated cron outcomes with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from the PR diff plus current main: the branch references a missing export, deletes covered context-policy behavior, and places the cron hook after terminal prepare-failure returns.

## Solution Assessment

No. The requested lifecycle signal is the right direction, but this patch is not the best fix because it mixes the hook additions with unrelated fork-policy changes and does not centralize exactly-once cron/subagent hook emission.

- **[P1] Restore the exported fork decision helper:** `src/agents/subagent-spawn.runtime.ts:9`
  - body: This barrel now re-exports `resolveParentForkMaxTokens`, but `session-fork.ts` on current main does not export that symbol. Any typecheck/build that reaches this module will fail before the new hook behavior can run; keep `resolveParentForkDecision` or add a real exported helper with coverage.
  - confidence: 0.97
- **[P2] Preserve thread-bound spawn context policy:** `src/agents/subagent-spawn.ts:650`
  - body: Hard-coding omitted `context` to `isolated` bypasses `resolveThreadBindingSpawnPolicy`, so thread-bound native subagents no longer honor `session.threadBindings.defaultSpawnContext` or the documented fork default. That drops requester transcript context for existing thread-bound workflows.
  - confidence: 0.94
- **[P2] Keep oversized forks as isolated fallback:** `src/agents/subagent-spawn.ts:356-365`
  - body: Current main treats an oversized parent fork as an accepted isolated spawn with a note; this patch turns the same condition into a spawn error. Large thread-bound or explicit-fork requests would fail instead of continuing with clean context.
  - confidence: 0.92
- **[P2] Avoid double-firing failed-spawn session_end:** `src/agents/subagent-spawn.ts:1137-1149`
  - body: `sessions.delete` already emits the plugin `session_end` hook for a deleted session even when `emitLifecycleHooks` is false. This manual emission sends an extra event, and it uses the idempotency key instead of the stored child session id, so hook consumers can see one wrong end event and one delete event for the same failed spawn.
  - confidence: 0.95
- **[P2] Emit cron session_end for prepare failures:** `src/cron/isolated-agent/run.ts:774-784`
  - body: The new helper only accepts a `PreparedCronRunContext`, so it is unreachable when `prepareCronRunContext` returns terminal model-selection or provider-preflight errors after creating the run session. Those skipped/error cron runs still bypass `session_end`, leaving plugins polling on failure paths.
  - confidence: 0.89
