## Summary

Keep open. The feature direction is valid, but the branch is not ready because the failed-spawn path would double-fire `session_end`, and the cron hook still misses a terminal preparation-failure return while also targeting an older cron flow shape.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent-spawn cleanup and cron isolated-agent terminal paths, updates hook type comments, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep generic `sessions.delete` as the owner of failed-spawn `session_end`, then add a single fire-once cron terminal hook in the current cron execution flow with focused regression coverage and updated hook docs/types.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the PR manually fires `session_end` for failed subagent spawn cleanup, and current main's `sessions.delete` already emits `session_end` before the `emitLifecycleHooks:false` gate.

## Solution Assessment

No. The current patch is not the best fix because it duplicates the failed-spawn signal and still leaves at least one cron terminal return without the new hook; the safer path is an exactly-once lifecycle helper wired through all terminal cron exits.
