## Summary

Keep open: the rebased PR still needs changes because failed-spawn cleanup can double-emit `session_end`, and prepare-time cron failures still return before any new `session_end` path.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawns and isolated cron run termination, exports supporting runtime helpers, documents the hook, and adds a changelog entry.

## Best Possible Solution

Centralize exactly-once `session_end` emission in the owning lifecycle paths: leave failed-spawn deletion to `sessions.delete`, and emit cron `session_end` for every terminal cron run, including prepare-time failures, with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the PR manually emits failed-spawn `session_end` when `emitLifecycleHooks:false`, while current main's `sessions.delete` still emits `session_end`; a second static path shows model-selection/preflight cron failures returning before the new cron helper can run.

## Solution Assessment

No. The requested direction is maintainable, but this implementation is not the best fix because it duplicates an existing delete-owned signal and still misses terminal cron runs created during preparation failures.

## Review Findings

Overall confidence: 0.9
