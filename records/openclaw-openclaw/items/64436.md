## What This Changes

This PR adds `api.runtime.usage` pricing helpers to the plugin runtime, updates runtime typings/mocks/docs/tests, tweaks daemon CLI compat type exports, and carries generated protocol artifacts.

## Best Possible Solution

Land a narrow `runtime.usage` API only if maintainers accept the SDK contract, with correct docs, regenerated API baseline, focused runtime coverage, and no unrelated generated artifacts.

## Reproduction Assessment

Yes for the review findings: they reproduce by comparing the PR docs to `src/utils/usage-format.ts`, checking the live PR file list for the missing API baseline hash, and confirming `dist/protocol.schema.json` is ignored generated output. ## Solution Assessment

No as-is. The runtime facade is a plausible narrow direction, but the PR needs an accurate public docs snippet, the generated Plugin SDK API baseline, removal or splitting of generated protocol artifacts, contributor real behavior proof, and maintainer acceptance of the new SDK surface.
