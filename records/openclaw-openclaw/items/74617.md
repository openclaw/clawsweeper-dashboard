## What This Changes

The issue requests fixing a Discord gateway startup race where READY may never be observed, leaving the bot online but unresponsive.

## Best Possible Solution

Keep this open until the linked implementation is reviewed and either landed, replaced by an equivalent target-repo Discord gateway/lifecycle fix, or closed with a clear maintainer rationale. The durable fix should cover both missed startup READY observation and the IDENTIFY limiter/socket-close path with targeted regression tests.
