## Summary

Keep open: the PR is correctly scoped and has supplied logging proof, but it is not merge-ready because the new success log can report the original Telegram topic id after a threadless fallback actually succeeded.

## What This Changes

This PR adds info-level Telegram outbound success logging for text and media sends in `sendMessageTelegram()` plus tests that message body, caption, media URL, and filename are not logged.

## Best Possible Solution

Land the send-level success log after it records only identifiers from the effective successful Telegram request, omitting the thread id when fallback removed it.

## Reproduction Assessment

Yes, source-reproducible. `withTelegramThreadFallback()` removes `message_thread_id` for a successful retry while the PR head logs `threadParams.message_thread_id`, and current tests already cover the media fallback success path.

## Solution Assessment

No, not yet. Adding the log in `extensions/telegram/src/send.ts` is the right boundary, but it must use the effective successful request rather than the original thread params after fallback.

## Review Findings
