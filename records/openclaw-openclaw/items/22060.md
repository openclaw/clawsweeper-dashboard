## Summary

Keep open. Current `main` still treats Discord embed title/description as inbound message text when the visible message body is empty, then carries that text through `RawBody`/`BodyForAgent` into prompt assembly. That means platform-fetched preview metadata can still reach the model as ordinary user input, so the reported vulnerability class is not clearly fixed on `main`.
