## Summary

Keep open: source evidence confirms that the reported Google Play-era Android build advertises Gateway protocol 3 while current gateways require protocol 4, so this is a valid compatibility/release regression rather than stale cleanup.

## What This Changes

The issue reports that the Google Play Android app version 2026.4.5 reaches onboarding and then fails with a Gateway protocol mismatch against newer OpenClaw gateways.

## Best Possible Solution

Publish a current Play Android build with protocol 4 support, then verify a Play-installed app can pair with a current Gateway; only choose a source change if maintainers intentionally want protocol-3 Android clients to remain supported.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a v2026.4.5 Android client sends protocol range 3..3, while current main requires protocol 4 and rejects that handshake with `protocol mismatch`. I did not run a live Android Play install in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to publish the already-updated Android Play build and verify it against the current Gateway. A source-level compatibility change should be a deliberate maintainer choice, not an automatic cleanup patch.
