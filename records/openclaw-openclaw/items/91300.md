## Summary

Core already converts thrown or partial best-effort send failures into `delivered=false`; the remaining phantom-success case is an inaccurate Weixin adapter receipt/liveness problem tracked by the plugin owner at https://github.com/Tencent/openclaw-weixin/issues/191.

## What This Changes

Prevent isolated cron announce runs from reporting successful Weixin delivery when no outbound message was actually accepted.

## Best Possible Solution

Keep core’s generic thrown/partial-failure accounting unchanged and make openclaw-weixin report failure unless its upstream send was actually accepted, with latest-version idle-runtime proof in the canonical plugin issue.

## Reproduction Assessment

Yes at source level: core tests and history reproduce the thrown/partial best-effort failure case, while the plugin report supplies a concrete idle-runtime false-receipt pattern; latest-plugin live confirmation remains pending.

## Solution Assessment

No as additional OpenClaw core logic: core already handles adapter errors correctly, while a plugin returning a non-empty success receipt without sending must be fixed at the adapter boundary.
