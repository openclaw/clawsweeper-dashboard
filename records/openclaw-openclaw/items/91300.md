## Summary

The core false-success path is already fixed and covered by regressions: best-effort payload failures prevent isolated cron runs from recording `delivered=true`, while any remaining stale WeChat runtime or phantom-receipt behavior is tracked in the openclaw-weixin repository.

## What This Changes

Prevent failed best-effort isolated cron announce delivery from being recorded as successfully delivered.

## Best Possible Solution

Keep the shipped core failure accounting and continue any remaining WeChat runtime-liveness investigation in https://github.com/Tencent/openclaw-weixin/issues/191.

## Reproduction Assessment

Yes. Focused regression tests model the exact best-effort partial-failure path and verify that isolated cron delivery remains false after a payload failure.

## Solution Assessment

Yes. Tracking failures inside the shared sender and requiring every cron payload to succeed is narrower and more reliable than depending on optional caller callbacks or WeChat-specific inference.
