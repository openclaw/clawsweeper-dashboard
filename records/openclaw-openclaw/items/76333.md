## Summary

Keep open. Current main and v2026.5.2 still hardcode the Dream Diary narrative wait at 60 seconds, tests confirm timeout leaves `DREAMS.md` unwritten, and the linked closing PR #76342 is open and unmerged, so this issue should stay open until that PR is reviewed and landed or replaced.

## What This Changes

The issue asks for the Dream Diary narrative timeout to be raised and made configurable because the current 60-second subagent wait times out after embedded-agent prep on larger workspace sets.

## Best Possible Solution

Review #76342 and decide the public timeout contract before landing a memory-core fix with aligned schema, docs, regression tests, and changelog coverage.
