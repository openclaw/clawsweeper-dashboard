## What This Changes

The issue asks OpenClaw to stop emitting false-positive plugin id mismatch startup warnings for bundled plugins whose manifest ids are short names but package names include `-plugin` or `-provider` suffixes.

## Best Possible Solution

Land a compatibility-safe registry/discovery fix through https://github.com/openclaw/openclaw/pull/85084 or a replacement, preserving canonical manifest ids while filtering or invalidating obsolete persisted mismatch diagnostics.

## Reproduction Assessment

Yes by source inspection: a persisted installed-index diagnostic with a plugin id and still-existing source path can pass the current stale checks, be copied into the manifest registry, and be formatted as a config warning. I did not run a live gateway repro in this read-only sweep.

## Solution Assessment

Yes for keeping this issue open pending the linked fix path. Renaming manifest ids is not the best solution; the maintainable fix should preserve manifest identity and stop obsolete false-positive diagnostics from being generated or replayed.
