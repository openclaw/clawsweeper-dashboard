## What This Changes

The issue asks OpenClaw to stop emitting misleading bundled-plugin `plugin id mismatch` startup warnings for short manifest ids with suffixed package names such as `xai` vs `@openclaw/xai-plugin`.

## Best Possible Solution

Treat manifest ids as authoritative while invalidating or filtering obsolete persisted `manifest uses ... entry hints ...` diagnostics during registry snapshot loading, without suppressing real runtime export/config id mismatch diagnostics.

## Reproduction Assessment

Yes by source inspection, but not by a live gateway run. A persisted installed-index diagnostic with a plugin id and still-existing source path survives the stale checks, is copied into the manifest registry, and is emitted as a config warning.

## Solution Assessment

Yes for a narrow registry diagnostic lifecycle fix. Renaming manifests or adding suffix-stripping comparison is not the best path because current fresh bundled hint logic already avoids this mismatch and the remaining issue is stale persisted diagnostics.
