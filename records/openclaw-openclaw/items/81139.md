## Summary

Keep open: current main still registers only the bare `openclaw sessions` list handler plus `cleanup` and `export-trajectory`, while the generic parser error path can still build the reported `openclaw sessions list --help` hint. The related closed issue and merged PR at https://github.com/openclaw/openclaw/issues/60905 and https://github.com/openclaw/openclaw/pull/60923 addressed exit code behavior, not the missing alias or misleading recovery path.

## What This Changes

Request `openclaw sessions list` to delegate to the existing bare sessions listing, or at minimum emit clearer guidance consistent with other list-style CLI commands.

## Best Possible Solution

Add a backward-compatible `sessions list` alias that reuses the existing sessions listing handler, with focused registration tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection shows current main has no `sessions list` subcommand and the generic parse-error helper still constructs the reported help hint from `sessions list`; I did not run the CLI under the read-only review rules.

## Solution Assessment
