## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Build claims from effective channel access:** `extensions/discord/src/monitor/instance-claims.ts:213-218`
  - body: `buildClaimEntries` records raw numeric channel keys without applying the same channel resolution used by ingress. Name/slug and wildcard-with-overrides channels can be missed, while `enabled: false` entries can be claimed and make peer instances silently skip traffic that the owner will later reject.
  - confidence: 0.9
- **[P2] Pass the active bot id into native claim checks:** `extensions/discord/src/monitor/native-command.ts:465-472`
  - body: The native ownership lookup does not pass `botId`, so the resolver can scan every bot bucket in the shared file. In multi-bot hosts, another bot's claim can suppress autocomplete or dispatch for this bot; the dispatch path repeats the same omission.
  - confidence: 0.88
- **[P2] Use the resolved preflight channel id:** `extensions/discord/src/monitor/message-handler.preflight.ts:775-783`
  - body: Preflight already normalizes payload fallbacks into `messageChannelId`, but the new ownership lookup queries `message.channelId`. Payloads that rely on `channel_id` or raw-data fallbacks can miss the claim and fall through to the old local allowlist denial.
  - confidence: 0.86
- **[P2] Serialize shared claim updates:** `extensions/discord/src/monitor/instance-claims.ts:238-247`
  - body: `refreshDiscordClaims` performs an unlocked read/modify/write of one shared JSON file. Concurrent Discord instances or accounts can overwrite each other's fresh claims, producing temporary `no-entry` ownership decisions and duplicate or wrong-instance handling.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes Discord access-control routing, and the current branch can blackhole or mis-scope channel ownership decisions.

Concerns:

- **[medium] Claims can blackhole disabled or unmatched channels:** `extensions/discord/src/monitor/instance-claims.ts:213`
  - body: The registry is populated from raw numeric keys instead of effective allowed-channel resolution, so peers may skip channels that the claiming instance will reject, while legitimate name/slug channels may not be claimed correctly.
  - confidence: 0.9
- **[medium] Native claim checks are not bot-scoped:** `extensions/discord/src/monitor/native-command.ts:465`
  - body: Native command ownership checks omit the active bot id, allowing claims for a different Discord bot in the same shared file to suppress this bot's interactions.
  - confidence: 0.88
- **[medium] Shared registry writes can lose peer claims:** `extensions/discord/src/monitor/instance-claims.ts:238`
  - body: The shared claims file is updated through an unlocked read/modify/write cycle, so concurrent instances can overwrite each other's ownership data and route authorization decisions incorrectly.
  - confidence: 0.84
