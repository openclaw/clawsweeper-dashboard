## Summary

Keep this PR open, but it is not merge-ready: the Discord plugin is the right layer for the multi-instance routing bug, yet the current branch still has access-control and message-delivery defects plus missing inspectable real behavior proof.

## What This Changes

This PR adds a shared on-disk Discord instance-claim registry, periodic provider refresh, and preflight/native-command gates to skip Discord events owned by another OpenClaw instance.

## Best Possible Solution

Rework the Discord plugin fix so ownership claims are derived from the same effective per-bot channel access rules as ingress, stored without losing peer claims, and proven with redacted two-instance Discord evidence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Build claims from effective channel access:** `extensions/discord/src/monitor/instance-claims.ts:213-218`
  - body: `buildClaimEntries` records raw numeric channel keys instead of the same effective channel resolution used by ingress. Name/slug allowlists and wildcard-with-overrides can be missed, while disabled channels can still be claimed and make peer instances skip traffic that the owner will reject.
  - confidence: 0.9
- **[P2] Scope native claim checks to the active bot:** `extensions/discord/src/monitor/native-command.ts:465-472`
  - body: The native autocomplete and dispatch lookups do not pass `botId`, so claims for another Discord bot in the shared file can return `claimed-by-other` and silently suppress valid interactions for this bot.
  - confidence: 0.88
- **[P2] Use the resolved preflight channel id:** `extensions/discord/src/monitor/message-handler.preflight.ts:775-783`
  - body: Preflight normalizes fallback payload shapes into `messageChannelId`, but the new ownership lookup uses `message.channelId` directly. Messages that rely on `channel_id` or raw-data fallbacks can miss peer claims and fall through to the old local deny path.
  - confidence: 0.86
- **[P2] Serialize shared claim updates:** `extensions/discord/src/monitor/instance-claims.ts:238-247`
  - body: `refreshDiscordClaims` performs an unlocked read-modify-write of one shared JSON file. Concurrent Discord instances or accounts can overwrite each other's fresh claims, producing temporary `no-entry` decisions and duplicate or wrong-instance handling.
  - confidence: 0.84
