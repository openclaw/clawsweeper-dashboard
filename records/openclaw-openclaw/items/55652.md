## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Build claims from effective Discord channel access:** `extensions/discord/src/monitor/instance-claims.ts:201-218`
  - body: The claim builder records every numeric channel key without checking `allow: false` or `enabled: false`, skips name/slug channel keys that current allowlists support, and misses wildcard-plus-override guild configs. Because the new gate runs before the normal allowlist checks, peers can blackhole denied channels or fail to skip allowed named/wildcard channels; build claims from the same effective channel access resolution used by ingress.
  - confidence: 0.89
- **[P1] Use the resolved preflight channel id:** `extensions/discord/src/monitor/message-handler.preflight.ts:781`
  - body: `preflightDiscordMessage` already normalizes the incoming channel into `messageChannelId` from several payload shapes, but the new ownership lookup passes only `message.channelId`. Messages whose id is only present as `channel_id`, `rawData.channel_id`, or the event fallback will return `no-entry` and can still hit the wrong-instance deny path.
  - confidence: 0.87
- **[P2] Scope native claim lookups to the active bot:** `extensions/discord/src/monitor/native-command.ts:465-472`
  - body: The native autocomplete and dispatch ownership checks omit `botId`, so `resolveDiscordClaimOwnership` can scan every bot key in the shared claims file. On hosts running multiple Discord bots/accounts, another bot's claim can make this bot silently drop valid interactions.
  - confidence: 0.84
- **[P2] Serialize shared claim writes:** `extensions/discord/src/monitor/instance-claims.ts:238-247`
  - body: `refreshDiscordClaims` reads, prunes, rewrites, and renames the whole shared claims file without a lock or compare-and-swap. Two instances refreshing together can overwrite each other's fresh claims, causing temporary `no-entry` decisions and duplicate or wrong-instance handling.
  - confidence: 0.8
