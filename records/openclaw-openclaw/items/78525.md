## Summary

Keep open: this PR targets a source-reproducible tool-policy bypass, but the proposed fix still misses ACP-backed subagent envelopes and lacks the required changelog entry for a user-visible/security policy fix.

## What This Changes

The PR adds an inline skill dispatch resolver that applies configured tool policy before invoking direct tool-dispatch skill commands and adds allow/deny regression coverage.

## Best Possible Solution

Land one canonical fix that reuses the same effective policy semantics as normal tool surfaces, including ACP subagent envelopes, with focused regression coverage and a single-line changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible: a command-dispatch skill builds raw OpenClaw tools and only applies owner-only filtering, so a restrictive tools.allow/tools.deny or subagent policy is bypassed before this PR.

## Solution Assessment

No, not yet. The approach is mostly the right narrow code path, but it must mirror existing subagent-envelope policy detection for ACP child sessions and add the required changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply subagent policy to ACP child sessions:** `src/auto-reply/reply/skill-tool-dispatch.runtime.ts:90-92`
  - body: The new resolver only enables subagentPolicy when the session key literally matches `subagent:`. Existing tool surfaces treat ACP child sessions with stored subagentRole/spawnedBy metadata as subagent envelopes via `isSubagentEnvelopeSession`, so a leaf ACP child can still use an inline skill targeting `sessions_spawn` or other session-control tools and bypass the subagent policy this PR is meant to enforce. Please mirror the existing capability-store based check here.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/auto-reply/reply/get-reply-inline-actions.ts:263`
  - body: This inline-dispatch branch now changes user-visible/security-sensitive skill tool policy behavior: denied or non-allowlisted tools can become unavailable to slash commands. Repo policy requires a CHANGELOG.md entry for user-facing fixes, and this PR currently touches only code/tests.
  - confidence: 0.9
