## Summary

Keep open: this is an active external bug-fix PR for a real memory isolation gap on current main, but the branch still has correctness and contract blockers plus missing real behavior proof.

## What This Changes

This PR adds a public Plugin SDK helper to strip managed Light/REM dreaming blocks, applies it to memory-core daily QMD full and partial reads, and adds focused tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved canonical path, strips managed Light/REM blocks from full and paginated memory_get reads, preserves ordinary content unless a block is removed, and updates the SDK baseline if the helper stays public.

## Reproduction Assessment

Yes for source reproduction. Current main returns raw daily-file text from QMD readFile(), while inline/both storage and old files can still contain managed Light/REM blocks; I did not execute a runtime memory_get flow because this review is read-only.

## Solution Assessment

No, not as currently written. Filtering daily reads is the right direction, but this branch must classify the resolved canonical path, avoid no-op formatting rewrites, include the generated SDK baseline, and provide real behavior proof.
