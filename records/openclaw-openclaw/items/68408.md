## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks the intended daily-memory read filtering, while the PR leaves a normalized-path bypass and misses required public-SDK/release hygiene.

## What This Changes

The PR adds a public `stripDreamingManagedBlocks()` markdown helper, applies it to QMD daily-memory full and partial reads, and adds helper/read-path tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved workspace-relative path, strips only managed Light/REM blocks from both full and paginated `memory_get` reads, preserves ordinary daily-note text when no block is removed, keeps dreaming internals on raw files, and updates the SDK baseline plus changelog if the helper remains public.

## Reproduction Assessment

Yes. On current main, a daily file containing `<!-- openclaw:dreaming:light:start -->` is returned raw by the QMD `readFile()` full and partial paths; on the PR branch, the remaining bypass is a normalized-equivalent request such as `memory/./2026-04-18.md`.

## Solution Assessment

No. The read-path filtering direction is right, but the implementation should classify the resolved canonical path, avoid no-op formatting rewrites, and include the generated SDK baseline and required changelog before merge.
