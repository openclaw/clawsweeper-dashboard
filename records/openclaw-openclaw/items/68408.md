## Summary

Keep open. Current main still returns raw QMD daily-memory reads for explicit inline/both storage and legacy daily files, but this branch has blocking correctness/API-baseline issues and lacks required real behavior proof.

## What This Changes

This PR adds a Plugin SDK markdown helper to strip managed Light/REM dreaming blocks, applies it to memory-core QMD daily full and partial reads, and adds focused tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved canonical path, strips only managed Light/REM blocks from full and paginated memory_get reads, preserves ordinary daily-note text on no-op reads, keeps dreaming internals on raw files, and updates the SDK baseline if the helper remains public.

## Reproduction Assessment

Yes, at source level: memory_get delegates to QmdMemoryManager.readFile, current main returns raw daily-file text, and inline/both or legacy files can still contain managed Light/REM blocks. I did not execute a runtime memory_get flow because this review was read-only.

## Solution Assessment

No, not as currently written. Filtering daily reads is the right direction, but the branch must use the resolved canonical path, avoid no-op formatting rewrites, include the generated SDK baseline, refresh the branch, and provide real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Normalize the resolved daily path before filtering:** `extensions/memory-core/src/memory/qmd-manager.ts:1216`
  - body: `resolveReadPath()` canonicalizes the file that will be read, but this check uses the raw request `relPath`. A request like `memory/./2026-04-18.md` can resolve to the same daily file while skipping the stripping branch, so base the daily-file decision on the resolved workspace-relative path for both full and partial reads.
  - confidence: 0.91
- **[P2] Update the Plugin SDK API baseline:** `src/plugin-sdk/memory-host-markdown.ts:65`
  - body: `memory-host-markdown` is a published Plugin SDK subpath, and this PR adds a new public export from it. Regenerate and commit `docs/.generated/plugin-sdk-api-baseline.sha256`, or keep the helper plugin-local, so the SDK contract check stays aligned.
  - confidence: 0.9
- **[P2] Preserve no-op daily file formatting:** `src/plugin-sdk/memory-host-markdown.ts:75`
  - body: `stripDreamingManagedBlocks()` always collapses runs of blank lines, even when no managed block was removed. `memory_get` should return exact user-authored excerpts for ordinary daily files; only normalize blank lines after an actual strip.
  - confidence: 0.86
