## Summary

Keep open: this is a useful memory-core hardening PR for inline/both or legacy daily dreaming blocks, but it is not merge-ready because the current patch leaves a normalized-path bypass, rewrites ordinary daily-note formatting, lacks the Plugin SDK API contract update, is dirty against main, and still has no real behavior proof.

## What This Changes

This PR adds a public markdown helper to strip managed Light/REM dreaming blocks, applies it to QMD daily-memory full and partial reads, and adds helper/read-path tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved canonical path, strips only managed Light/REM blocks from full and paginated memory_get reads, preserves ordinary content exactly when nothing is stripped, keeps dreaming internals reading raw files, and updates the Plugin SDK baseline if the helper remains public.

## Reproduction Assessment

Yes at source level: current main memory_get reads return raw daily-file content, and inline/both or legacy daily files can contain managed Light/REM blocks. I did not run a live memory_get flow because this review is read-only.

## Solution Assessment

No, the current PR is not the best mergeable form yet. The maintainable fix should key daily detection off the resolved canonical path, preserve no-op reads byte-for-byte, and either update the Plugin SDK API baseline or keep the helper plugin-local.
