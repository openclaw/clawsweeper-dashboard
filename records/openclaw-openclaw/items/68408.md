## Summary

Keep open: this PR targets a real current-main memory isolation bug, but the current head is still not merge-ready because the daily-file filter is bypassable through normalized paths and the public SDK/API housekeeping is incomplete. The author has acknowledged the pending fixes, so this should stay open for source-branch updates or a narrow repair.

## What This Changes

The PR adds a Plugin SDK markdown helper for stripping managed Light Sleep and REM dreaming blocks, applies it to memory-core QMD daily-memory reads, and adds focused tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved workspace-relative path, strips only managed Light/REM blocks from full and paginated `memory_get` reads, preserves ordinary notes when no block is removed, and updates the SDK baseline plus changelog.

## Reproduction Assessment

Yes from source inspection. On current main, inline Light/REM blocks can be written to `memory/YYYY-MM-DD.md`, and `memory_get` reaches `QmdMemoryManager.readFile()`, which returns raw full or partial file text.

## Solution Assessment

No. The read-path filtering direction is right, but this patch should base daily-file detection on the resolved canonical path, avoid no-op formatting rewrites, and include the generated SDK baseline and changelog before merge.
