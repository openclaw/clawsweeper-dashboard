## Summary

Keep open. Current main still has a source-reproducible Telegram announce-delivery bug: a group session key can resolve to `group:<chatId>`, while Telegram send requires the final chat ID to normalize to a numeric value.

## What This Changes

Fix `sessions_send` announce delivery so Telegram group session keys resolve to numeric chat IDs instead of `group:<id>` targets.

## Best Possible Solution

Keep the fix in the Telegram-owned session target normalization or announce-target seam, preserving topic thread IDs without making core parse Telegram-specific `group:` syntax globally.

## Reproduction Assessment

Yes, from source inspection: a session key like `agent:main:telegram:group:-5072149164` flows to `to: "group:-5072149164"`, and the Telegram send path rejects non-numeric final targets. I did not run a live Telegram probe in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to normalize Telegram session-derived announce targets in the Telegram plugin path and add regression coverage. Broadly stripping standalone `group:` in shared/core code would be riskier because other channels also use generic group target syntax.
