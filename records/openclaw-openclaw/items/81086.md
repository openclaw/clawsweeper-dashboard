## Summary

Keep open. Current main still has the source-reproducible Telegram announce-target bug, and the open draft fix at https://github.com/openclaw/openclaw/pull/81229 uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Telegram group announce delivery from `sessions_send` should normalize session-derived group targets to numeric Telegram chat IDs instead of passing `group:<id>` to the send path.

## Best Possible Solution

Land or replace the focused Telegram plugin fix that normalizes session-derived group announce targets before send, with regression coverage and real Telegram group proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a Telegram group session key can become `to: "group:<chatId>"`, and the Telegram send path rejects non-numeric final targets; I did not run a live Telegram probe in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is a Telegram-owned session-target normalizer or equivalent plugin-local fix; teaching core to strip generic `group:` syntax would blur channel ownership.
