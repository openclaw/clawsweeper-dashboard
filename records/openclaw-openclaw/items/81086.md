## Summary

Keep open. Current main still has a source-reproducible Telegram announce-delivery bug, and the linked fix PR is open/draft with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that Telegram `sessions_send` announce delivery resolves group session keys to `group:<chatId>`, which the Telegram send path rejects instead of a numeric chat ID.

## Best Possible Solution

Land the Telegram-owned session target resolver and focused regression coverage from the linked PR after its draft/protected maintainer review clears, while keeping core `sessions_send` generic.

## Reproduction Assessment

Yes. From source inspection, a session key like `agent:main:telegram:group:-5072149164` can flow to `to: "group:-5072149164"`, and the Telegram send path rejects that standalone target before it can become a numeric chat ID.

## Solution Assessment

Yes. The narrow maintainable solution is to normalize session-derived Telegram targets inside the Telegram plugin via `resolveSessionTarget`; broadly stripping standalone `group:` in shared/core target parsing would blur channel ownership.
