## Summary

Keep open. Current main has clearer memory docs, builtin/QMD backend support, and related implementation threads, but it still does not provide the requested unified memory-management contract or config shape across built-in, QMD, mixed/write, and disabled states.

## What This Changes

The issue asks for a strict, consistent memory-management definition and configuration model because users see different SQLite, daily Markdown/QMD, or no-memory behavior.

## Best Possible Solution

Treat this as a memory architecture/product tracker: land or replace the focused per-agent backend and memory-write PRs after review, then either narrow this issue to the remaining unified mode policy or close it against explicit canonical follow-ups.
