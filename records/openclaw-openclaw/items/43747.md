## Summary

Keep open: current main documents and implements separate memory backends, memory-search indexing, and pre-compaction Markdown flush behavior, while the request asks for a new unified memory-management contract and config surface rather than a narrow regression fix.

## What This Changes

Request a unified memory-management mode so team installs consistently choose builtin SQLite, QMD, or combined Markdown-plus-index behavior.

## Best Possible Solution

Define a maintainer-approved memory contract in memory-core/config that states how Markdown writes, builtin indexing, QMD indexing, and index storage paths interact, then align schema, docs, and focused tests to that contract.
