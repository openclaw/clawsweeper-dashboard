## Summary

Keep open for maintainer product follow-up. Current main has clearer builtin/QMD behavior and some QMD isolation improvements, but it still does not provide the requested unified memory mode contract or decide how backend selection, memory flush writes, workspace SQLite placement, and CRUD writes should fit together.

## What This Changes

The issue asks for a stricter, consistent memory-management contract, including a proposed `memory.type` mode for builtin-only, QMD-only, and mixed behavior.

## Best Possible Solution

Decide the memory product contract first: either keep and document the current `memory.backend` plus `memorySearch` plus `memoryFlush` split, or introduce a versioned unified mode with schema, docs, doctor diagnostics/migration, and focused tests across builtin, QMD, and compaction paths.
