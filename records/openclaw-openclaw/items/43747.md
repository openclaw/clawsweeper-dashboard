## Summary

Keep open for maintainer product direction. Current main still exposes separate builtin SQLite search, QMD backend selection, and pre-compaction Markdown memory flush controls; the open memory_add work at https://github.com/openclaw/openclaw/pull/70540 is only a CRUD slice, not the requested unified memory mode contract.

The request asks for a unified memory management contract/config mode so builtin SQLite, QMD Markdown files, and disabled memory behave consistently across users and workspaces.

## Best Possible Solution
