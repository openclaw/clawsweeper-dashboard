## Summary

Keep open for maintainer product follow-up. Current main has clearer builtin/QMD backend behavior and QMD isolation work, but it still does not provide the requested unified `memory.type`/`mixed` contract or decide whether memory flush, backend selection, workspace SQLite placement, and CRUD writes should be one public mode surface.

## What This Changes

The issue asks for a strict, unified memory-management contract so builtin SQLite indexing, QMD Markdown files, and disabled or missing memory states behave predictably across users.

## Best Possible Solution

Decide the memory product contract first: either keep and document the existing `memory.backend` plus `memorySearch` plus `memoryFlush` split, or introduce a versioned unified mode with schema, doctor migration/diagnostics, docs, and focused tests across builtin, QMD, and compaction paths.
