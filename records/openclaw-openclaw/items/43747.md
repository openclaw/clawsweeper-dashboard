## Summary

Keep open for maintainer product follow-up. Current main already has explicit but separate memory backend, memory search, and compaction flush controls; the requested unified `memory.type` or `mixed` contract is a new config/product decision, not a narrow reproducible regression.

## What This Changes

The issue asks for a stricter unified memory-management contract so built-in SQLite memory, QMD daily Markdown memory, and disabled or misconfigured memory do not diverge unexpectedly across users.

## Best Possible Solution

Decide the memory product contract first: either document the existing `memory.backend` plus `memorySearch` plus `memoryFlush` split more clearly, or introduce a versioned unified mode with schema, doctor migration/diagnostics, docs, and tests across builtin, QMD, and compaction paths.
