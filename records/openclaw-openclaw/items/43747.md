## Summary

Keep open. Current main and v2026.6.10 still expose separate builtin/QMD backend selection, memorySearch, compaction memoryFlush, and scalar memory-plugin slot controls; some storage-path cleanup landed, but the requested unified memory mode and migration/product contract remain unresolved.

## What This Changes

The issue asks for a stricter unified memory-management design so builtin SQLite memory, QMD daily Markdown memory, and disabled or misconfigured memory do not vary unpredictably across users.

## Best Possible Solution

Define a maintainer-approved memory mode or role contract that reconciles backend selection, memorySearch, QMD, compaction flush, plugin ownership, workspace scoping, docs, and doctor migration before changing runtime behavior.
