## Summary

Keep open. Current main has corrected some of the report, including `streamTo` and clearer thread-binding key separation, but the sub-agents docs still omit `announceTimeoutMs`, still describe depth 2 as unable to spawn despite the current `callerDepth < maxSpawnDepth` rule, and still use depth-specific auto-archive/tool-policy wording. The issue also has open PRs with closing syntax, so it should remain open until maintainers choose and land one current docs fix.

## What This Changes

The issue asks to correct six sub-agent documentation gaps around config defaults, `sessions_spawn` parameters, thread binding keys, spawn depth, tool policy, and auto-archive scope.

## Best Possible Solution

Land one focused docs PR that updates `docs/tools/subagents.md` and the related config reference to current source facts: document `announceTimeoutMs` with the 120000 ms default, describe spawning as allowed while `current depth < maxSpawnDepth` across the 1-5 range, make auto-archive and cascade wording depth-agnostic, and clarify the current always-denied versus leaf-only sub-agent tool sets while preserving the already-correct `streamTo` and thread-binding sections.

## Reproduction Assessment

Yes. The docs drift is reproducible by comparing `docs/tools/subagents.md` and config docs with the current schema, announce timeout resolver, spawn-depth guard, tool policy helper, and policy tests on main; no runtime execution is needed.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable solution, but it should be refreshed against current main rather than older PR text that still uses the old 90000 ms announce timeout.
