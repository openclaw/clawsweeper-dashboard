## What This Changes

The issue reports that `openclaw doctor` keeps showing a cron-store repair finding for working isolated shell/process cron jobs, including a non-working `doctor --fix` hint and a stale `jobs.json` path after cron moved to SQLite.

## Best Possible Solution

Land one narrow doctor cron PR that separates manual-only shell-prompt advisories from auto-repairable store fixes, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible. A SQLite-backed isolated `agentTurn` cron job whose prompt uses shell tools reaches `unresolvedAgentTurnShellToolPrompt`, and current main routes that issue into a Cron note that still says to run `openclaw doctor --fix`.

## Solution Assessment

Yes in direction, but not resolved yet. The best fix is a narrow doctor-output split that keeps manual-only shell-prompt advisories out of the auto-repair preview; open PRs already attempt that.
