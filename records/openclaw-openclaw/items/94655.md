## What This Changes

The issue reports misleading `openclaw doctor` cron output for SQLite-backed cron jobs where a manual shell-prompt advisory persists after `--fix` and references migrated `jobs.json`.

## Best Possible Solution

Land one narrow fix that separates manual-only cron advisories from auto-repairable doctor output, then close this issue and supersede duplicate PRs.

## Reproduction Assessment

Yes, source-reproducible. An SQLite-backed isolated agentTurn cron job with a shell/process-looking prompt reaches the manual-conversion warning while the same doctor note still promises `openclaw doctor --fix`.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to split manual-only cron warnings from repairable migrations and update the store wording, not to change cron execution or add a new config option.
