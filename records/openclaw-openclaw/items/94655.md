## Summary

Keep open: current main and v2026.6.8 still route the manual-only cron shell-prompt advisory through the auto-repair doctor note, and the candidate fixes are still open or closed-unmerged.

## What This Changes

The issue asks `openclaw doctor` to stop promising an ineffective cron `--fix` and stop pointing SQLite-backed cron users at a migrated `cron/jobs.json` live file.

## Best Possible Solution

Doctor should render unresolved shell/process agentTurn cron jobs as informational/manual-only advisories, leaving repair prompts only for mutations `doctor --fix` can actually perform.

## Reproduction Assessment

Yes. Current main still routes `unresolvedAgentTurnShellToolPrompt` into the cron store preview that carries a repair hint, while the detector does not mutate those jobs; I did not run a live CLI repro because this review is read-only.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to keep the detection but render manual-only shell-prompt jobs outside the auto-repair preview, without changing cron execution or adding configuration.
