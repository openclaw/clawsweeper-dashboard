## Summary

Keep open. Current main still routes the manual-only cron shell-prompt advisory through the auto-repair doctor note, and v2026.6.8 shipped the same behavior; the open candidate PRs should be reviewed or closed before this issue is resolved.

## What This Changes

The issue asks doctor to stop presenting working isolated shell-prompt cron jobs as a repairable `doctor --fix` cron-store problem and to stop pointing users at a migrated `cron/jobs.json` live file.

## Best Possible Solution

Land one narrow fix that separates manual-only shell-prompt cron advisories from auto-repairable cron store issues, avoids stale live-file wording, and leaves cron execution and migration semantics unchanged.

## Reproduction Assessment

Yes, at source level. A current SQLite-backed isolated `agentTurn` cron job with a shell/process-looking prompt reaches `unresolvedAgentTurnShellToolPrompt`, and current main still emits that warning through a Cron note that promises `doctor --fix`; I did not run mutating doctor commands in this read-only review.

## Solution Assessment

Yes in direction. The best fix is a narrow doctor-output change that removes manual-only advisories from the auto-repair preview, rather than changing cron execution, adding config, or auto-converting ambiguous agent prompts.
