## Summary

Keep open. Current main and the latest release still route the manual-only cron shell-prompt warning through doctor output that promises `openclaw doctor --fix`, and the open candidate fixes have not merged.

## What This Changes

The issue asks doctor cron diagnostics to stop promising `--fix` or referencing a non-live `jobs.json` path for working isolated shell-prompt cron jobs.

## Best Possible Solution

Review and land one narrow candidate PR that separates manual-only cron advisories from auto-repairable doctor output, then close the superseded duplicates and this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level. An isolated agent-turn cron job with a shell/process-looking prompt reaches the manual-conversion warning while the same doctor note still promises `openclaw doctor --fix`; I did not run mutating doctor commands in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to split manual-only cron advisories from repairable migrations and update the store wording. Changing cron execution behavior or adding config surface is not needed for this issue.
