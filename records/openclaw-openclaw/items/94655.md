## Summary

Keep open. Current main still routes the manual-only isolated cron shell-prompt advisory through the same doctor note that promises `openclaw doctor --fix`, and an open closing PR now owns the candidate fix.

## What This Changes

The issue asks for doctor cron diagnostics to stop advertising `doctor --fix` and a stale `cron/jobs.json` path for SQLite-backed isolated shell-prompt cron jobs that keep running.

## Best Possible Solution

Land one narrow PR that separates non-auto-fixable shell-prompt advisories from the auto-repair cron-store preview, leaves the supported job unchanged, and removes stale `jobs.json` or `doctor --fix` wording for that case.

## Reproduction Assessment

Yes, at source level. An SQLite-backed isolated `agentTurn` cron job with shell/process-looking prompt text reaches `unresolvedAgentTurnShellToolPrompt`, and current doctor output still appends the `doctor --fix` repair promise to that preview.

## Solution Assessment

Yes. The maintainable fix is to split this manual-only advisory out of the auto-repair preview rather than making `doctor --fix` mutate supported isolated agent-turn jobs.
