## Summary

Keep open. Current main still routes the manual-only cron shell-prompt advisory through the same doctor note that promises `openclaw doctor --fix`, and https://github.com/openclaw/openclaw/pull/94886 is still an open closing PR for the fix.

## What This Changes

The issue asks for `openclaw doctor` to stop showing a persistent, unactionable cron `--fix` finding and stale `cron/jobs.json` wording for SQLite-backed, otherwise-working isolated cron jobs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/94886 or an equivalent narrow fix that separates manual-only cron advisories from auto-repairable doctor output, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. An SQLite-backed isolated `agentTurn` cron job with shell/process-looking prompt text reaches the manual-conversion warning while the same doctor note still promises `openclaw doctor --fix`; I did not execute the CLI because this review is read-only.

## Solution Assessment

Yes in direction. The maintainable fix is to split manual-only cron advisories from repairable migrations and update the store wording, not to change cron execution or add a new config option.
