## What This Changes

The issue asks doctor to stop promising `doctor --fix` and stop implying a live `jobs.json` file for working SQLite-backed isolated cron jobs that only need manual command conversion guidance.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94659, or an equivalent narrow change that separates manual-only cron advisories from auto-repairable doctor output.

## Reproduction Assessment

Yes, at source level. An SQLite-backed isolated agent-turn cron job with a shell/process-looking prompt reaches the manual-conversion warning while the same doctor note still promises `openclaw doctor --fix`; I did not run the mutating command in this read-only review.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to split manual-only cron warnings from repairable migrations and update the store wording, not to change cron execution or add a new config option.
