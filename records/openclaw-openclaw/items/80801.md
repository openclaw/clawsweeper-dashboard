## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Quote provider IDs before rendering re-login commands:** `src/agents/failover-error.ts:517`
  - body: The new recovery hint builds a copy-paste shell command from `FailoverError.provider` without quoting or validating it, and the skip-cache path repeats the same pattern with `candidate.provider`. Provider IDs are plugin/runtime-attributed strings, while `formatCliCommand` does not quote arguments; a malformed provider ID can therefore turn an operator-pasted recovery command into extra shell syntax. Build these commands through a shell-arg helper or reject non-safe IDs in both places, with metacharacter coverage.
  - confidence: 0.88
- **[P2] Keep fallback skipping default-compatible until accepted:** `src/agents/model-fallback.ts:1223`
  - body: This branch skips every non-primary candidate in the same session for the TTL after one auth-class failure. That changes the default retry contract for existing configured fallback chains: after an upgrade, a provider that was retried on the next turn is now suppressed for 60 seconds unless the process restarts or the TTL expires. Because fallback behavior is compatibility-sensitive here, this should either be opt-in/default-compatible or land with explicit maintainer acceptance plus docs and upgrade proof.
  - confidence: 0.82
- **[P2] Document the new auth purge and skip controls:** `src/cli/models-cli.ts:332-335`
  - body: The PR adds the destructive `models auth login --force` flag and `OPENCLAW_FALLBACK_SKIP_TTL_MS`, but no public docs changed. Existing model/auth/failover docs are where operators learn recovery commands and fallback behavior, so this new purge path and retry-suppression override should be documented before merge.
  - confidence: 0.86
