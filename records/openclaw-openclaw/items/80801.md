## Summary

Keep open: the auth recovery direction is useful, but the current patch is not merge-ready because the new fallback skip cache is not wired into the production fallback callers and the new recovery command bypasses the repo’s profile/container-aware command formatter.

## What This Changes

The PR adds a forced model auth re-login flag, auth failover remediation hints, an opt-in session-scoped auth fallback skip cache, docs, tests, a probe script, and adjacent CI drift test fixes.

## Best Possible Solution

Wire the skip cache to the actual production session identity, use the standard profile/container-aware command formatter for re-auth hints, and add focused tests plus refreshed real behavior proof for the second-turn skip and non-default profile recovery paths.

## Reproduction Assessment

Yes for the review findings by source inspection: production callers pass runId/sessionKey but not sessionId, while the new skip cache only marks and checks when params.sessionId is set. The profile/container command issue is also source-provable from the new hand-built command versus existing formatCliCommand usage.

## Solution Assessment

No. The additive auth recovery direction is maintainable, but the skip cache needs production wiring and the re-auth hint should use the existing command-formatting contract before this is the best fix.
