## Summary

Keep open: the PR targets plausible auth/fallback operator pain, but it still has blocking correctness issues in the new `--force` purge ordering and skip-cache lifetime, plus insufficient real behavior proof for the destructive CLI path.

## What This Changes

The branch adds `models auth login --force`, auth-class fallback re-login hints, a session-scoped auth fallback skip cache, tests, and a changelog entry.

## Best Possible Solution

Revise the PR so profile purging occurs only after a valid auth method is selected, stale skip markers are globally bounded or cleared on session teardown, and redacted real CLI proof covers cached-profile purge plus re-auth.

## Reproduction Assessment

Yes for the PR-introduced purge-order bug: source inspection shows the new `--force` deletion block runs before auth-method resolution, and that resolution can still fail. The original operator rough edges have helper/unit proof, but no live CLI `--force` re-auth reproduction in the PR body.

## Solution Assessment

No; the proposed direction is useful, but the current patch is not the best safe implementation because it mutates credential state before later validation and leaves the TTL cache unbounded across one-off sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve the auth method before purging profiles:** `src/commands/models/auth.ts:683-690`
  - body: `--force` deletes cached profiles immediately after provider selection, but `pickProviderAuthMethod` can still return no method for an unknown `--method` or a canceled method prompt. That path removes credentials and then throws before any auth flow runs, so resolve the method first and purge only immediately before `runProviderAuthMethod`.
  - confidence: 0.92
- **[P2] Prune expired skip buckets across sessions:** `src/agents/fallback-skip-cache.ts:65-70`
  - body: The TTL only removes expired entries when the same session bucket is queried. A gateway that sees many one-off sessions with auth fallback failures will keep stale buckets in the global map past the 60s window, so prune globally on writes/checks or clear the cache from session teardown.
  - confidence: 0.86
