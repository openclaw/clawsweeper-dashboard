## Summary

Keep open: the PR addresses a plausible auth/fallback rough edge, but the current patch can delete cached profiles before auth-method validation and can retain expired one-off session skip buckets indefinitely; the live `--force` CLI re-auth path also still needs real behavior proof.

## What This Changes

Adds `openclaw models auth login --force`, auth-failover remediation hints, a session-scoped fallback skip cache, and related tests/changelog coverage.

## Best Possible Solution

Revise the branch so profile deletion happens only after a valid auth method is selected, skip-cache state is bounded or cleared with session lifecycle, and the PR body includes redacted real CLI proof for the `--force` re-auth flow.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: the new purge block precedes method resolution, current tests show unknown methods throw without running auth, and the new cache only prunes queried session buckets. The original operator rough edges have helper/unit proof but not a live CLI `--force` reproduction.

## Solution Assessment

No; the direction is useful, but this implementation needs the destructive purge moved immediately before `runProviderAuthMethod` and the skip cache made globally bounded or tied to session teardown before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve the auth method before purging profiles:** `src/commands/models/auth.ts:683-692`
  - body: `--force` deletes cached profiles before `pickProviderAuthMethod` validates `--method`. With an unknown method, current behavior throws without running any auth flow, so a typo can remove working credentials and leave no re-login attempt; move the purge after method resolution and immediately before `runProviderAuthMethod`.
  - confidence: 0.92
- **[P2] Prune expired skip buckets across sessions:** `src/agents/fallback-skip-cache.ts:65-70`
  - body: The global cache creates one bucket per session, but expired entries are pruned only when that same session is queried. One-off sessions that hit auth fallback failures can leave stale buckets forever; prune all buckets on writes/checks or clear this cache from session teardown.
  - confidence: 0.86
