## Summary

Keep open. The PR is active and useful, with sufficient terminal proof for the targeted seams, but it has a concrete P2 blocker: the new copy-paste re-auth hints interpolate provider IDs into shell commands without validation or quoting.

## What This Changes

The PR adds `openclaw models auth login --force`, auth failover remediation hints, and an in-memory session-scoped fallback skip cache with tests, a probe script, and a changelog entry.

## Best Possible Solution

Sanitize or quote provider IDs in every new re-auth command hint, keep the purge-after-method-validation ordering, and let maintainers decide whether this bounded session skip is the right narrow step while https://github.com/openclaw/openclaw/issues/47910 tracks broader quarantine policy.

## Solution Assessment

No as-is because the new command hints need provider ID sanitization or quoting before they are safe to present as copy-pasteable commands. The rest of the approach is maintainable if maintainers accept the bounded session-scoped skip policy rather than a full provider quarantine.
