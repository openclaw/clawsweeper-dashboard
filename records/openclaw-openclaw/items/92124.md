## Summary

Current main and the latest release still unconditionally remove the provider prefix before the model is used for embedding requests and memory-index identity. The issue should remain open because https://github.com/openclaw/openclaw/pull/92135 is the active closing implementation and has not merged; its existing review also identifies a native-URL classifier defect and missing live router/index-reuse proof.

## What This Changes

Preserve provider-qualified OpenAI embedding model names for non-native OpenAI-compatible routers without changing native OpenAI behavior or invalidating matching memory indexes.

## Best Possible Solution

Land the linked plugin-local fix after it reuses the established semantic native-OpenAI endpoint classification and demonstrates successful live router requests plus reuse of an existing qualified-model index.

## Reproduction Assessment

Yes, from source with high confidence: the supplied configuration reaches unconditional prefix stripping on current main, and the normalized value is used for both the embedding request and index identity. This review did not independently call Requesty, so the status is source-reproducible rather than fully reproduced.

## Solution Assessment

The plugin-local direction is the right ownership boundary, but the proposed PR is not yet the best complete fix because it should use the existing semantic native-host rule and add live router/index-reuse proof.
