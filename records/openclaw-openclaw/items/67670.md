Keep open. Current main still classifies any 403 HTML provider response as `auth_html_403` with re-authentication guidance, and the linked open PR #67717 still uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, merged, or closed.

## What This Changes

The issue asks OpenClaw to handle `openai-codex` Cloudflare browser challenges behind mainland China proxies by adding TLS-fingerprint-aware routing/configuration or at least documenting and classifying the limitation accurately.

## Best Possible Solution

Review PR #67717 as the narrow diagnostic fix. If correct, land the Cloudflare challenge classifier, actionable user copy, raw-HTML suppression, and regression tests; treat browser-like TLS emulation or a generic `request.tlsProfile` as separate maintainer-scoped provider/security work.
