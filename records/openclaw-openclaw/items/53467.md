## Summary

Keep this PR open: current main still lacks Slack `ignoreOtherMentions`, and the branch is a useful focused implementation, but the current patch is not merge-ready because the new drop gate can still false-drop bot-directed Slack ID mentions in regex-only degraded auth mode and the PR lacks inspectable real Slack behavior proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config option with schema/types/docs/generated metadata, tests, and an inbound `prepareSlackMessage` drop gate.

## Best Possible Solution

Keep this as the canonical Slack parity PR, but make the other-mention drop require Slack-ID-specific bot mention detectability, then merge only after inspectable redacted live Slack proof and green checks.

## Reproduction Assessment

Yes for the review finding by source inspection: with `ctx.botUserId` empty and non-empty mention regexes, the new gate can treat an explicit Slack `<@BOT_ID>` token as an other mention. This is not a current-main bug because the PR adds the feature.

## Solution Assessment

No, not as currently written: the schema/resolver/docs shape is reasonable, but regex-only detectability is not enough to distinguish Slack bot-ID mentions from other Slack mentions. The safer fix is to require `ctx.botUserId` or an equivalent Slack-ID-specific explicit mention signal before this drop branch runs.
