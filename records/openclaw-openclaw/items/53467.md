## Summary

Keep this PR open: the Slack parity feature is not implemented on current main and the latest branch is still useful, but the patch still has a merge-blocking degraded-identity false-drop path and lacks inspectable real Slack proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config option with schema, resolver, docs, tests, generated metadata, changelog, and inbound filtering.

## Best Possible Solution

Keep this as the canonical Slack parity implementation, but require Slack-ID-specific bot mention detectability for the drop branch and merge only after redacted live Slack proof and clean checks.

## Reproduction Assessment

Yes for the review finding by source inspection: when Slack `auth.test` leaves `ctx.botUserId` empty but mention regexes exist, the new gate can treat an explicit `<@BOT_ID>` message as an other-mention. This is not a current-main bug reproduction because the PR adds the feature.

## Solution Assessment

No, not as currently written: the schema/resolver/docs shape is maintainable, but regex-only mention detectability is not enough to distinguish Slack bot-id mentions from other Slack user mentions. The safer fix is to require `ctx.botUserId` or an equivalent Slack-ID-specific signal before applying this drop branch.
