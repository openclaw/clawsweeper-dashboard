## Summary

Keep open: current main still lacks Slack `ignoreOtherMentions`, and this PR is the active canonical implementation, but the latest head still has a blocking false-drop edge case and lacks inspectable live Slack proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` boolean across config schema/types/docs/generated metadata, resolves it in Slack channel config, and drops Slack room messages with other user or subteam mentions before normal mention gating.

## Best Possible Solution

Keep this as the canonical Slack parity PR, require Slack-ID-specific certainty for the other-mention drop, align docs/tests with that contract, then merge after redacted live Slack proof and green checks.

## Reproduction Assessment

Yes for the PR review finding by source inspection: with `ignoreOtherMentions`, empty `ctx.botUserId`, non-empty mention regexes, and a Slack `<@BOT_ID>` token, `canDetectMention` can be true while explicit Slack-ID mention resolution is unavailable. This is introduced by the PR, not a current-main bug.

## Solution Assessment

No, not as currently written. The config/schema/resolver shape is reasonable, but the drop branch should require `ctx.botUserId` or an equivalent Slack-ID-specific explicit mention signal before classifying `<@...>` tokens as other mentions.
