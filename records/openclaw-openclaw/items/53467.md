## Summary

Keep open: current main does not implement the Slack `ignoreOtherMentions` option and the other same-scope OpenClaw attempt is closed, but this PR is not merge-ready because the new Slack drop gate can silently discard legitimate messages when bot Slack ID detection is unavailable and real Slack proof is still not inspectable.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config option across schema/types/docs/generated metadata and drops room messages that mention someone else without explicitly mentioning the bot.

## Best Possible Solution

Keep the feature, but gate the other-mention drop on Slack-ID-specific bot mention certainty, add the degraded-identity/non-empty-regex regression test, and require redacted real Slack proof for both drop and pass-through paths.

## Reproduction Assessment

Yes, source-reproducible: on the PR head, set `ignoreOtherMentions: true`, make Slack `ctx.botUserId` unavailable while mention regexes remain non-empty, and send a room message containing a Slack `<@...>` token that the regexes do not match; the new condition can return `null`. I did not run tests because this review is read-only.

## Solution Assessment

No: the option is reasonable, but the current implementation should not use generic `canDetectMention` as sufficient certainty for an other-mention drop that can suppress Slack messages.
