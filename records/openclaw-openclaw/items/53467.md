## Summary

Keep open: the feature is still absent from current main, but this PR should not merge yet because the new Slack drop gate can still false-drop bot-directed Slack ID mentions in degraded identity mode and the PR lacks inspectable real Slack proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` option with config schema/types, resolver wiring, runtime filtering, tests, docs, generated metadata, and a changelog entry.

## Best Possible Solution

Keep the Slack-owned config/schema/resolver/docs/tests shape, but require explicit Slack bot or bot-subteam detectability before the other-mention drop and add redacted real Slack proof before landing.

## Reproduction Assessment

Yes for the review finding by source inspection: with `ctx.botUserId` absent and mention regexes configured, `canDetectMention` is true while Slack ID mention tokens cannot be resolved as the bot. Not applicable as a bug reproduction for the feature request itself.

## Solution Assessment

No, not yet: the owner boundary and tests are mostly the right direction, but the drop predicate should not treat regex availability as sufficient to classify Slack `<@...>` tokens as non-bot mentions. The safer fix is to require Slack bot-id/subteam certainty for this gate or prove the configured regex explicitly matches the bot mention form.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Require bot-id certainty before dropping Slack mention tokens:** `extensions/slack/src/monitor/message-handler/prepare.ts:708`
  - body: This gate treats `mentionRegexes.length > 0` as enough certainty to drop any `<@...>` message where `wasMentioned` is false. If Slack `auth.test` leaves `ctx.botUserId` empty but group mention patterns exist, a real bot ping like `<@BOTID> help` will still have `hasAnyMention` and `!wasMentioned`, so enabling `ignoreOtherMentions` can discard a bot-directed message. Please require explicit Slack bot/subteam detectability for this drop, or add a regression proving regexes cover the Slack ID token before allowing the drop.
  - confidence: 0.83
