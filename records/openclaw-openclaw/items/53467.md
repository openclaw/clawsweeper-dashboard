## Summary

Keep this PR open: current main still lacks the Slack `ignoreOtherMentions` config surface, and the closed duplicate points back here as the canonical implementation. The PR still has a blocking false-drop risk in degraded Slack identity detection and still needs inspectable real Slack proof before merge.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config key across config schema/types/docs/generated metadata, resolves it in Slack channel config, and drops room messages that mention others but not the bot.

## Best Possible Solution

Keep this as the canonical Slack parity PR, require Slack-ID-specific certainty before the other-mention drop, add a degraded-identity regression test, then rebase and merge only after green checks plus redacted live Slack proof.

## Reproduction Assessment

Yes for the PR review finding by source inspection. If `auth.test` leaves `ctx.botUserId` empty while mention regexes are non-empty, a real Slack `<@BOT_ID>` token can produce `hasAnyMention=true`, `wasMentioned=false`, `canDetectMention=true`, and be dropped as `other-mention`.

## Solution Assessment

No, not as currently written. The config/schema/resolver shape is reasonable, but the drop gate should require a resolved Slack bot user id or an equivalent Slack-ID-specific explicit mention signal before treating `<@...>` as another-user mention.
