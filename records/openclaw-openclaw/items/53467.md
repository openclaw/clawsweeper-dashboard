## Summary

Keep open: current main still lacks the Slack `ignoreOtherMentions` option, and the PR is a useful Slack parity feature, but the current head can still false-drop bot-directed Slack messages when bot identity is degraded and no inspectable real Slack proof is attached.

## What This Changes

This PR adds a per-channel Slack `ignoreOtherMentions` config option with schema/docs/generated metadata, resolver wiring, a prepare-time drop gate, and Slack prepare tests.

## Best Possible Solution

Keep the Slack parity feature, but require resolved bot-user-ID certainty or an equivalent Slack-ID-specific explicit mention signal before applying the other-mention drop, add the non-empty-regex degraded-identity regression test, align docs with that contract, and provide redacted live Slack proof.

## Reproduction Assessment

Yes, by source inspection. Configure an agent identity or mentionPatterns so mention regexes are non-empty, let Slack `auth.test` fail so `botUserId` is empty, enable `ignoreOtherMentions`, and a real `<@BOT_ID>` room ping can satisfy `hasAnyMention && !wasMentioned` and be dropped.

## Solution Assessment

No, not as currently written. The config/schema/resolver shape is reasonable, but the drop gate should not treat generic mention regexes as enough certainty to classify Slack `<@...>` tokens as non-bot mentions.
