- **[P2] Require Slack ID certainty before dropping other mentions:** `extensions/slack/src/monitor/message-handler/prepare.ts:986`
  - body: This branch treats any nonempty mention regex list as enough certainty to drop `hasAnyMention && !wasMentioned` messages. But Slack startup can continue with `ctx.botUserId` empty after `auth.test` fails, while mention regexes may still exist from agent identity or config and will not necessarily match native `<@BOT_ID>` tokens. Please gate this drop on resolved Slack bot identity or add a token-aware certainty path with a regression test for empty `botUserId` plus nonempty mention regexes.
