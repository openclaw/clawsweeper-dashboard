The PR adds a Slack per-channel `ignoreOtherMentions` config option with schema/types, resolver wiring, inbound drop logic, docs, tests, and generated config metadata.

## Best Possible Solution

Keep this as the canonical Slack parity PR, require Slack-ID certainty before dropping other mentions, align or implement the documented MPIM scope, then merge only after green checks and redacted live Slack proof.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR head can drop Slack mention messages using generic regex detectability when `ctx.botUserId` is empty, and the documented MPIM scope is excluded by the current `isRoom` config/drop path.

## Solution Assessment

No, not as currently written. The config/schema/resolver direction is reasonable, but the drop condition should require explicit Slack-ID certainty and the MPIM behavior should be implemented or removed from the documented scope.
