## Summary

Keep open: current main does not implement Slack `ignoreOtherMentions`, and this PR is the focused canonical implementation, but the latest head still has a concrete false-drop edge case and lacks inspectable live Slack proof before merge.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config option with schema/type/resolver wiring, inbound filtering, tests, docs, generated config metadata, and changelog text.

## Best Possible Solution

Keep this as the canonical Slack parity implementation, but gate the drop on resolvable Slack bot identity or an equivalent Slack-id-specific signal, add the degraded-identity regression test, then merge only after redacted live Slack proof and clean checks.

## Reproduction Assessment

Yes for the review finding by source inspection: if `auth.test` leaves `ctx.botUserId` empty while mention patterns are configured, a Slack `<@BOT_ID>` message can have `hasAnyMention=true` and `wasMentioned=false`, so the new gate drops it. This is not a current-main bug reproduction because the PR adds a new feature.

## Solution Assessment

No, not as currently written: the schema/resolver/docs shape is maintainable, but regex-only mention detectability is not enough to distinguish Slack bot-id mentions from other Slack user mentions. The safer fix is to require `ctx.botUserId` or an equivalent Slack-id-specific signal for this drop branch.
