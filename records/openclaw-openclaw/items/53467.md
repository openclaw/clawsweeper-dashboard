## Summary

Keep this PR open: Slack `ignoreOtherMentions` is not implemented on current main and this PR is the canonical active attempt, but the latest patch still has a correctness blocker around degraded Slack bot identity plus missing inspectable real Slack proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config key across schema/types/docs/generated metadata, resolves it in Slack channel config, and drops room messages with other Slack mentions before normal mention gating.

## Best Possible Solution

Keep this as the canonical Slack parity PR, require Slack-ID-specific certainty before the other-mention drop, add the degraded-identity regression test, then rebase and merge after green checks plus redacted live Slack proof.

## Reproduction Assessment

Yes for the PR review finding by source inspection: when `ctx.botUserId` is empty but mention regexes exist, a real Slack `<@BOT_ID>` token can leave `wasMentioned=false` while `canDetectMention` and `hasAnyMention` are true. That makes the new drop gate discard a legitimate bot-directed channel message.

## Solution Assessment

No, not as currently written. The config/schema/resolver shape is reasonable, but the drop gate should require `ctx.botUserId` or an equivalent Slack-ID-specific explicit mention signal instead of generic regex detectability.
