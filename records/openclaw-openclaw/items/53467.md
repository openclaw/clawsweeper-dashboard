## Summary

Keep open. Current main still lacks Slack `ignoreOtherMentions`, and this PR is the canonical open attempt after the duplicate https://github.com/openclaw/openclaw/pull/55570 closed, but the latest head still has a false-drop risk in the degraded Slack identity path and lacks inspectable real behavior proof.

## What This Changes

The PR adds a Slack per-channel `ignoreOtherMentions` config option, docs/changelog/generated metadata, resolver wiring, tests, and an inbound message drop gate.

## Best Possible Solution

Keep this as the canonical Slack parity PR, require Slack-ID-specific mention certainty for the other-mention drop, add the regression test/docs alignment, then merge after inspectable redacted live Slack proof and green checks.

## Reproduction Assessment

Yes for the PR review finding by source inspection: when `ctx.botUserId` is empty but mention regexes are present, the new gate still runs while explicit Slack-ID mentions cannot be resolved. This is not a current-main bug because the PR adds the feature.

## Solution Assessment

No, not as currently written. The config/schema/resolver shape is reasonable, but the drop branch should require `ctx.botUserId` or an equivalent Slack-ID-specific explicit mention signal before it can safely classify `<@...>` as another mention.
