## Summary

Current main already hardens the context-engine registry against split bundle chunks and includes a regression test written specifically for issue #40096. The same implementation is present in the tagged release `v2026.4.22`, so the reported `2026.3.7` published-bundle failure is already fixed on shipped code.

## Close Comment

Closing as fixed. Codex review of current `main` found that the context-engine registry is now process-global via `Symbol.for("openclaw.contextEngineRegistryState")` in `src/context-engine/registry.ts:306`, specifically to keep duplicated dist chunks on the same registry map. There is also a dedicated regression test for this exact report in `src/context-engine/context-engine.test.ts:1007` (`Bundle chunk isolation (#40096)`), which asserts that registration in one chunk is visible when resolving from another chunk. The same code is present in the shipped `v2026.4.22` release tag, and the issue timeline already points at the fixing work in PR #40115 / commit `5dc232e2fa4744e7c8852f8e80c6b4e988323137`.
