## Summary

Keep open: the observability goal is useful, but the current patch is not mergeable because it imports private helpers and the synthetic full-drop feedback still conflicts with the existing provider-safe replay repair and synthetic-result policy.

## What This Changes

This PR adds dropped-tool-call diagnostics/logging, exports a report-producing sanitizer, and changes the session write guard/tests to persist synthetic error toolResult entries when an assistant tool-call turn is fully dropped.

## Best Possible Solution

Keep bounded structured drop diagnostics, but make unavailable-tool feedback provider-safe and consistent with the existing synthetic-result policy, then add focused replay/guard coverage and the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction: the PR imports non-exported constants, and a full-drop assistant message would persist only a toolResult while current replay repair drops free-floating toolResults.

## Solution Assessment

No. Logging/report export is a reasonable direction, but orphan toolResult feedback is not the safest transcript shape and the patch also needs export/build, synthetic-policy, logging-bound, and changelog fixes.
