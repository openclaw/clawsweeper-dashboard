The latest diff fixed the earlier mixed-message ordering issue, but the branch still has a build-breaking private import, writes synthetic orphan tool results in a way current replay repair drops, bypasses the existing synthetic-result opt-out, and logs unbounded model-controlled tool names.

## What This Changes

The PR adds dropped-tool-call diagnostics, a report-returning sanitizer export, WARN logging, synthetic error toolResult persistence for fully dropped unknown/malformed assistant tool-call turns, and updated guard tests.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. The logging/report direction is reasonable, but orphan toolResult feedback is not the narrowest maintainable transcript shape and it bypasses the existing synthetic-result policy.
