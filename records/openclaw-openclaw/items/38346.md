## Summary

Keep open: current main and v2026.5.19 still drop Telegram group messages from non-allowlisted senders before normal dispatch or public plugin observation, and the related early-hook work covers mention-gated skips rather than this observe-all/respond-few authorization split.

## What This Changes

The issue requests Telegram group behavior where OpenClaw can passively observe all group messages for sender identification or future reference while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation or unauthorized-audit contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy behavior, and coordinates with the open early-hook and unauthorized-access discussions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive ingest, public hooks, or normal reply processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation or audit contract with privacy and retention boundaries.
