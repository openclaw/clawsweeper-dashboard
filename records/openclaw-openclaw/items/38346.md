## Summary

Keep open: current main and v2026.5.12 still reject non-allowlisted Telegram group senders before passive ingest or public message hooks, and the requested observe-all/respond-few mode is a new privacy-sensitive product/config decision. The adjacent early-hook work only covers mention-skipped messages that already passed access checks, so it does not fully supersede this issue.

## What This Changes

The issue requests a Telegram mode where all group messages can be observed for identity/context while only allowlisted users can trigger bot responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy/visibility behavior, and coordinates with the broader early-hook work in https://github.com/openclaw/openclaw/issues/47167.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main behavior trace: allowlist-denied Telegram group messages return before processInboundMessage, passive ingest, or public plugin hooks.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; a separate opt-in passive-observation hook or config path is the safer maintainable direction.
