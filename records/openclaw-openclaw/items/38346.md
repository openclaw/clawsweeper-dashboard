## Summary

Keep open: current Telegram support has separate group visibility, sender allowlists, mention gating, and a mention-skip ingest hook, but it does not provide the requested mode where the bot observes all group messages/user IDs while only allowlisted users can trigger ordinary replies.

## What This Changes

Request a Telegram group mode that passively sees all group messages for sender identification while only allowlisted users can trigger the bot.

## Best Possible Solution

Design an explicit Telegram group observe/ingest policy that can record non-triggering sender identity without authorizing ordinary replies, with privacy documentation and coordination with the narrower unauthorized-sender ID reply request in #46701.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence behavior trace showing current main lacks the requested observe-all plus allowlisted-trigger split.

## Solution Assessment

No, the proposed `groupPolicy: "open"` configuration is not the best current solution because current main treats open groups as allowed for ordinary mention-triggered replies. A maintainable fix needs an explicit policy/API design instead of overloading existing open/allowlist semantics.
