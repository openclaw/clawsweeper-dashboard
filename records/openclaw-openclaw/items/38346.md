## Summary

Keep open. Current main still treats Telegram group allowlists as an admission and trigger-authorization gate, so allowlist-denied group senders are dropped before passive ingest or normal processing; separating observation from response authorization needs explicit Telegram product and privacy decisions.

## What This Changes

The issue requests a Telegram mode that observes all group messages for user identification while only allowlisted users can trigger bot responses.

## Best Possible Solution

Define an opt-in Telegram passive-observation contract that stores only necessary metadata, documents Bot API privacy-mode requirements, and keeps reply/trigger authorization separate from observation.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main behavior trace: allowlist-denied group messages return before normal processing, while groupPolicy open admits every sender.

## Solution Assessment

No. The proposed groupPolicy open plus groupAllowFrom shape conflates group admission with trigger authorization; a separate opt-in passive-observation setting would be narrower and safer.
