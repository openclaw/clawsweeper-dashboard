## Summary

Keep open: current main and the latest release still drop Telegram group messages from non-allowlisted senders before passive plugin/public processing, and the related early-hook work covers mention-gated skips rather than this observe-all/respond-few authorization split.

## What This Changes

Request to let Telegram observe all group messages for future sender identification while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation or unauthorized-audit contract that separates observation from trigger authorization, minimizes retained payloads, and coordinates with the existing skipped-message and audit threads.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive public hooks, reply-chain recording, or normal inbound processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation or audit contract with privacy and retention boundaries.
