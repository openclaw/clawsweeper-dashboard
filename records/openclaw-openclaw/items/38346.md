## Summary

Keep open: current main still separates Telegram group sender authorization from passive observation, so non-allowlisted group senders are dropped before normal processing or public plugin hooks. The related early-hook work is useful context, but it covers requireMention-skipped messages rather than this observe-all/respond-few authorization split.

## What This Changes

The issue requests a Telegram mode that passively observes all group messages for future user identification while only allowlisted senders can trigger bot responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation or unauthorized-audit contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy behavior, and coordinates with https://github.com/openclaw/openclaw/issues/47167, https://github.com/openclaw/openclaw/pull/83101, and https://github.com/openclaw/openclaw/issues/77546.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive ingest, public hooks, or normal reply processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation or audit contract with privacy and retention boundaries.
