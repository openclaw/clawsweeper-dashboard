## Summary

Keep open. Current Telegram support has group admission, sender allowlists, and post-admission mention-skipped ingest, but it does not provide the requested split where non-allowlisted group senders can be passively observed while only allowlisted users can trigger replies.

## What This Changes

Request a Telegram group mode where the bot passively observes all group messages and records sender IDs/context while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation mode or raw-inbound contract that preserves fail-closed trigger authorization, documents Telegram visibility prerequisites, and coordinates with https://github.com/openclaw/openclaw/issues/39232 and https://github.com/openclaw/openclaw/issues/46701.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence behavior trace: allowlist-rejected Telegram group senders are dropped before the existing mention-skipped history/ingest path, while `groupPolicy: "open"` intentionally bypasses sender allowlists.

## Solution Assessment

No. The proposed `groupPolicy: "open"` plus `groupAllowFrom` config conflicts with the current contract, so the maintainable solution is an explicit observe-only/passive mode with separate trigger authorization.
