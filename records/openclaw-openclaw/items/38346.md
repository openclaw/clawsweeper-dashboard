## Summary

Keep open: current main has passive hooks for mention-skipped Telegram group messages, but it does not satisfy the requested split between listening to everyone and only letting allowlisted users trigger replies. In current policy, `groupPolicy: "open"` lets all group senders pass group policy, while allowlist enforcement blocks unauthorized senders before the passive ingest path can run.

## What This Changes

Request a Telegram group mode where the bot can observe all group messages and record sender IDs while only allowlisted senders can trigger bot responses.

## Best Possible Solution

Define a Telegram plugin policy that explicitly separates observation/identity capture from prompt admission, document the privacy and BotFather visibility requirements, and cover `groupPolicy`, `groupAllowFrom`, per-group `allowFrom`, `requireMention`, and `ingest` interactions with tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Telegram access-policy feature request. Source inspection gives a high-confidence current-behavior check showing the proposed policy is not currently expressible without either allowing all triggers or dropping unauthorized senders before ingest.

## Solution Assessment

No. The proposed `groupPolicy: "open"` plus `groupAllowFrom` direction is not the best current solution because current main does not enforce `groupAllowFrom` as a trigger-only gate in open groups; a deliberate Telegram policy/config design is safer.
