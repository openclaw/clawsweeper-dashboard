## Summary

Keep open. Current main still lacks a Telegram observe-before-trigger mode: unauthorized allowlist group messages are dropped before inbound processing, while `groupPolicy: "open"` makes group senders eligible for the normal trigger path rather than treating `groupAllowFrom` as trigger-only.

## What This Changes

Request Telegram to passively observe all group messages for future user identification while only allowlisted senders can trigger replies or agent behavior.

## Best Possible Solution

If maintainers approve the privacy model, add an explicit opt-in Telegram group/topic observe-before-trigger mode that keeps trigger authorization separate from passive history or hook ingestion.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unauthorized `groupPolicy: "allowlist"` group sender is returned before `processInboundMessage`, while `groupPolicy: "open"` admits normal group messages instead of applying `groupAllowFrom` as trigger-only.

## Solution Assessment

No. Reusing `groupPolicy: "open"` plus `groupAllowFrom` would overload existing documented semantics; a new explicit observe/ingest-before-trigger mode is the narrower maintainable direction after product/security review.
