## Summary

Keep open: current main and the latest release still drop Telegram group messages from non-allowlisted senders before normal processing or public plugin observation, and the request is a new privacy-sensitive observe-all/respond-few feature rather than a stale or already-implemented bug fix.

## What This Changes

The issue requests Telegram group support where the bot can passively observe all group messages for future user identification while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation or unauthorized-audit contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy behavior, and coordinates with the open inbound_observed and unauthorized-access discussions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive ingest, public hooks, or normal reply processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation or audit contract with privacy, retention, and authorization boundaries.
