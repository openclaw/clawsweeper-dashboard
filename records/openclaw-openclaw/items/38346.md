## Summary

Keep open. Current Telegram support has group visibility, group sender allowlists, and mention-skipped ingest for already-admitted messages, but it does not provide the requested split where the bot passively observes non-allowlisted group senders while only allowlisted users can trigger replies.

## What This Changes

Request Telegram group bots to observe all group messages and collect sender identity context while restricting trigger/reply authorization to configured allowlisted users.

## Best Possible Solution

Define an explicit Telegram passive-observation/user-ID discovery mode that preserves fail-closed trigger authorization, documents Telegram privacy-mode prerequisites, and coordinates with #46701 rather than changing `groupPolicy: "open"` semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence behavior trace: allowlist-rejected Telegram group senders are dropped before passive history/ingest, while `groupPolicy: "open"` admits ordinary senders for non-command triggers.

## Solution Assessment

No. The proposed config relies on `groupPolicy: "open"`, which current contracts define as bypassing sender allowlists; the safer solution is an explicit passive-observation design with separate trigger authorization.
