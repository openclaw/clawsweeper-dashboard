## Summary

Keep open: current main still drops allowlist-denied Telegram group senders before passive ingest or room-event handling, and the requested observe-all/respond-few mode needs maintainer product and security review.

## What This Changes

The issue requests Telegram passive group observation for all senders while only allowlisted users can trigger bot replies.

## Best Possible Solution

Define an opt-in Telegram passive-observation contract with payload minimization, retention rules, diagnostics, and trigger authorization kept separate from group admission.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main behavior trace: allowlist-denied group messages return before the existing passive ingest and room-event paths.

## Solution Assessment

No. The proposed groupPolicy open plus groupAllowFrom shape conflates group admission with trigger authorization; a separate opt-in passive-observation setting or hook is the narrower maintainable direction.
