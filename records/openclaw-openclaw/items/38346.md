## Summary

Keep open. Current main still lacks the requested Telegram observe-before-trigger group mode: `groupPolicy: "open"` lets all group messages trigger, while `groupPolicy: "allowlist"` drops unauthorized senders before message context or ingest handling can run.

## What This Changes

Request to let a Telegram bot observe or record all visible group messages and user IDs while only allowlisted users can trigger bot replies, commands, or actions.

## Best Possible Solution

If maintainers approve the product and privacy model, add an explicit opt-in Telegram group/topic configuration that observes or ingests visible group messages before trigger authorization, while preserving separate authorization gates for agent turns, text/native commands, tools, and visible replies. The implementation should update config types/schema, Telegram docs, and regression coverage for unauthorized observed messages versus authorized triggered turns.
