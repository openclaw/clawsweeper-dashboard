## Summary

Keep open: current main and v2026.5.12 still drop Telegram group messages from non-allowlisted senders before passive observation or public plugin hooks, and the related inbound-observed work covers mention-gated skips rather than this observe-all/respond-few authorization split.

## What This Changes

Request a Telegram mode that observes all group messages for future identification/history while allowing only allowlisted users to trigger bot responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy/visibility behavior, and coordinates with https://github.com/openclaw/openclaw/issues/47167 and https://github.com/openclaw/openclaw/pull/83101 without treating them as full replacements.

## Reproduction Assessment

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation contract with privacy and retention boundaries.
