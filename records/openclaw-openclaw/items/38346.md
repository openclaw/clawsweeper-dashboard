## Summary

Keep open: current main and v2026.5.20 still reject Telegram group messages from non-allowlisted senders before passive plugin/public processing, while the requested observe-all/respond-few behavior needs explicit product and security design.

## What This Changes

Request an opt-in Telegram mode where the bot can observe all visible group messages for future identification or memory while only allowlisted senders can trigger replies.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation or unauthorized-audit contract that separates observation from trigger authorization, minimizes retained payloads, and coordinates with the skipped-message and audit threads.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive public hooks, reply-chain recording, or normal inbound processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation or audit contract with privacy and retention boundaries.
