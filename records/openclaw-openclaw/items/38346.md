## Summary

Keep open. Current Telegram behavior still cannot express the requested split: allowlist-rejected group senders are dropped before passive history/hook ingest, while `groupPolicy: "open"` admits ordinary group senders rather than preserving allowlist-only triggering.

## What This Changes

The issue requests Telegram group behavior where the bot can observe all group messages for sender discovery while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit Telegram passive-observation/user-ID discovery mode that preserves fail-closed trigger authorization, documents Telegram privacy-mode prerequisites, and coordinates with #46701.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence behavior trace: allowlist-rejected Telegram group senders are dropped before passive history/ingest, while `groupPolicy: "open"` admits ordinary group senders.

## Solution Assessment

No. The proposed `groupPolicy: "open"` config is not the best solution because current contracts define it as open sender admission; a separate opt-in passive observation mode would be safer and clearer.
