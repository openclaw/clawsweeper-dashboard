## Summary

Keep open. Current main still does not provide a Telegram mode that passively observes non-allowlisted group senders while only allowlisted users can trigger replies; the existing paths either drop allowlist-denied senders before ingest or open the group path to every sender subject only to mention gating.

## What This Changes

Request a Telegram group mode that listens to all group messages for user identification/context while restricting bot-triggering replies to allowlisted users.

## Best Possible Solution

Define a Telegram-owned, opt-in passive-observation contract that is separate from reply authorization, metadata-minimized, documented, and fail-closed by default.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied group messages return before passive ingest or processing, while groupPolicy open admits all senders rather than enforcing a trigger-only allowlist.

## Solution Assessment

No. The proposed groupPolicy open plus groupAllowFrom shape conflates admission with trigger authorization; a separate opt-in passive-observation setting would be narrower and safer.
