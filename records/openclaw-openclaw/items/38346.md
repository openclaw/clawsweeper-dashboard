## Summary

Keep open: current main still rejects non-allowlisted Telegram group senders before passive ingest or public hooks, and the related early-hook work covers requireMention-skipped messages rather than this pre-allowlist observe-all/respond-few mode.

## What This Changes

Request an opt-in Telegram mode where the bot can passively hear group messages for later identification/history while only allowlisted users can trigger responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy/visibility behavior, and coordinates with https://github.com/openclaw/openclaw/issues/47167 and https://github.com/openclaw/openclaw/pull/83101 without treating them as full replacements.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before passive ingest, public hooks, or normal reply processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation contract with privacy and retention boundaries.
