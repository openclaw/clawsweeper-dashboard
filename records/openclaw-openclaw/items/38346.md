## Summary

Keep open: current main and v2026.5.19 still reject non-allowlisted Telegram group senders before normal processing or public plugin hooks, and the linked early-hook work covers mention-gated skips rather than this broader observe-all/respond-few authorization split.

## What This Changes

The issue requests a Telegram mode that passively observes all group messages for later user identification or history while only allowlisted users can trigger bot responses.

## Best Possible Solution

Define an explicit opt-in Telegram passive-observation contract that separates observation from trigger authorization, minimizes retained payloads, documents privacy and visibility behavior, and coordinates with https://github.com/openclaw/openclaw/issues/47167 and https://github.com/openclaw/openclaw/pull/83101 without treating them as complete replacements.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main and latest-release trace: allowlist-denied Telegram group messages return before passive ingest, public hooks, or normal reply processing.

## Solution Assessment

No. The proposed groupPolicy=open plus groupAllowFrom shape conflates group admission with trigger authorization; the safer direction is a separate opt-in passive-observation contract with privacy and retention boundaries.
