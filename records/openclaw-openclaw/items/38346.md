## Summary

Keep open. Current main still cannot passively observe allowlist-denied Telegram group senders while restricting bot triggers to allowlisted users, and the requested split expands privacy- and authorization-sensitive behavior that needs maintainer product/security review.

## What This Changes

Request Telegram group passive observation of all messages or sender IDs while only allowlisted users can trigger bot responses.

## Best Possible Solution

Define an explicit Telegram-owned passive-observation contract that is opt-in, metadata-minimized, documented, and separate from reply/trigger authorization while preserving fail-closed allowlist behavior by default.

## Reproduction Assessment

Not applicable as a feature request. Source inspection still gives a high-confidence behavior trace: allowlist-denied Telegram group messages return before passive ingest, while groupPolicy open admits all senders rather than only observing them.

## Solution Assessment

No. The proposed groupPolicy open plus groupAllowFrom shape conflates group admission with trigger authorization; a separate opt-in passive-observation setting and trigger authorization policy would be narrower and safer.
