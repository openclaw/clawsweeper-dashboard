## What This Changes

The branch moves Codex bounded-turn text collection before failed-turn checks and returns collected text with a warning when a failed terminal turn has non-empty assistant text.

## Best Possible Solution

Land one canonical agent-command lifecycle fix that preserves generated reply delivery while keeping empty/no-reply compaction failures fatal, then close or supersede duplicate collector/helper candidates.

## Reproduction Assessment

Yes. Source inspection shows current main persists the CLI transcript, awaits post-turn compaction, and only then calls final delivery, so a compaction throw can skip delivery after an assistant reply was generated.

## Solution Assessment

No. The bounded-turn collector is not the canonical delivery boundary for the linked issue, and blanket recovery from any failed Codex turn is broader than the Codex app-server failure contract supports.
