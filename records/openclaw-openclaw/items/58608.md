## Summary

Current main still leaves the reported restart-persistence gap open: the session-file repair hook runs before session open/compaction, but it does not apply tool-call/tool-result pairing repair to stored JSONL.

## What This Changes

The issue asks for persisted session-file repair of orphaned or displaced tool-result history after interrupted tool calls.

## Best Possible Solution

Extend repairSessionFileIfNeeded to apply conservative persistent tool-call/tool-result pairing cleanup while preserving headers, custom entries, parent/message metadata, backups, and existing replay-only provider policy boundaries.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a valid session JSONL file with a header and an orphaned or displaced toolResult entry reaches repairSessionFileIfNeeded, is parsed into entries, and returns no-op unless an existing malformed/empty-content counter fires. I did not run a live failing test because this was a read-only review.

## Solution Assessment

Yes. Reusing the existing load-time repair hook is the narrowest maintainable fix; adding an opt-in config option is a separate product choice, not required to fix persisted corruption.

## Repair Work Prompt

Fix the persistent session corruption reported in https://github.com/openclaw/openclaw/issues/58608. Extend repairSessionFileIfNeeded so the on-disk JSONL repair path applies conservative tool-call/tool-result pairing cleanup equivalent to the current replay sanitizer: drop truly orphaned/free-floating toolResult messages, move matching displaced results adjacent to the assistant tool-call turn, deduplicate duplicate results, and synthesize only storage-safe missing results when appropriate. Preserve the session header, non-message/custom entries, message IDs/parent IDs/timestamps where possible, backup and atomic rewrite behavior, file mode preservation, and existing blank-user/empty-assistant repairs. Do not add a new config option unless a maintainer asks. Add focused regression tests for orphaned, displaced, duplicate, and already-valid tool-call/tool-result persisted JSONL, and update transcript hygiene docs/changelog if behavior text changes.
