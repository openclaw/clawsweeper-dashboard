The issue asks OpenClaw to repair and rewrite persisted session JSONL on load when interrupted tool calls leave displaced, duplicate, or orphaned tool_result/toolResult entries.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/81397 or a narrow replacement that preserves provider-specific in-memory synthesis while adding durable move/drop repair and focused regression tests.

## Reproduction Assessment

Yes, by source inspection. A valid session JSONL file with an assistant toolCall, an intervening entry, and a late or orphan toolResult reaches repairSessionFileIfNeeded, but current main only performs malformed-entry cleanup and Codex-specific missing-output insertion before returning unrepaired.

## Solution Assessment

Yes. Extending repairSessionFileIfNeeded is the narrowest maintainable fix because run and compaction already call it before session load; the suggested autoRepair config is optional product work, not required for the bug fix.
