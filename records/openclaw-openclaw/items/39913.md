## Summary

Keep open: current main still has the reported inconsistency, with normal replies applying the effective response-prefix cascade while message-tool sends dispatch the normalized text without a prefix step.

## What This Changes

The issue requests that `messages.responsePrefix` and its channel/account overrides apply to outbound sends made through the shared `message(action="send")` tool.

## Best Possible Solution

Add one shared message-tool send normalization step that resolves the existing effective prefix with agent, channel, and account context, applies it once before plugin/core delivery and mirroring, and avoids adding new channel-specific policy.

## Reproduction Assessment

Yes by source inspection. A focused unit test can call `runMessageAction` for `send` with `messages.responsePrefix` configured and assert that current main delivers unprefixed `params.message` or core `sendMessage` content.

## Solution Assessment

Yes. Reusing the existing prefix resolver/template behavior in the shared `handleSendAction` boundary is narrower and more maintainable than a WhatsApp-only or plugin-by-plugin fix.
