## Summary

Keep open: current main still leaves `message(action="send")` outbound text without `messages.responsePrefix`, and the remaining work is actively represented by an open linked fix PR rather than implemented on main.

## What This Changes

The issue asks for configured `messages.responsePrefix` values to be applied consistently to visible `message(action="send")` outbound messages as well as normal direct replies.

## Best Possible Solution

Review and land or reject the open linked fix PR, then either close this issue as implemented or keep a narrowed follow-up for selected-model prefix template parity.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `messages.responsePrefix`, send visible text through `message(action="send")`, and current main dispatches the raw message text while the normal reply path prepends the prefix.

## Solution Assessment

Unclear as the complete solution until maintainers review the linked PR scope. Reusing the existing prefix cascade in the shared send path is the narrow fix for static and identity prefixes, but selected-model template variables need a product/API decision because tool sends do not perform fresh model selection.
