## Summary

## What This Changes

The PR forwards `hooks.internal.entries.session-memory.model` into LLM slug generation, documents the key, and adds focused handler and slug-generator regression tests.

## Best Possible Solution

Land the narrow pass-through once maintainers accept `session-memory.model` as supported hook config and normal required checks remain green.

## Reproduction Assessment

Yes. Source inspection on current main shows that `llmSlug: true` plus a `session-memory.model` value still calls `generateSlugViaLLM({ sessionContent, cfg })`, and the slug generator resolves the agent default model.

## Solution Assessment

Yes technically, with a config-contract caveat. Reusing `runEmbeddedAgent`'s model-only resolver is the narrowest maintainable fix, but maintainer acceptance of the newly supported hook key is still needed.
