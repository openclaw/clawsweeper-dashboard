## Summary

Current main and the latest release still intentionally keep goal clearing as an operator/session control, and this issue asks maintainers to change that model-authority boundary plus add a new persisted session-history contract. The closed prototype PRs provide useful design evidence, but no merged or open replacement currently owns the remaining product decision.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that clears non-active goals and archives them in bounded canonical session state.

## Best Possible Solution

Maintain the current operator-only behavior until a feature owner approves the authority change; if approved, implement one canonical session-store transition with docs, mutation metadata, catalog tests, and real agent-turn proof.

## Reproduction Assessment

Not applicable as a bug reproduction: current main matches the documented contract that models cannot clear goals. The observable state is source-verifiable, but the issue is asking whether to change that behavior.

## Solution Assessment

Unclear pending maintainer decision: canonical-store archival is the right storage direction if the feature is approved, but model-callable clearing itself is a product and safety boundary decision.
