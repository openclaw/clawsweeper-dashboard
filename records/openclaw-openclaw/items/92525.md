## Summary

Keep open for maintainer product review. Current main and `v2026.6.8` intentionally expose only `get_goal`, `create_goal`, and `update_goal` to models while reserving goal clearing for operator/session controls; the two implementation PRs are closed unmerged and the later one explicitly moved this product question here.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that archives terminal or paused goals into bounded `SessionEntry.clearedGoals` history before clearing the current goal.

## Best Possible Solution

Make the model-authority decision first; if approved, implement the smallest core change with explicit clearable states, canonical-store lifecycle semantics, mutation/catalog/docs coverage, and real agent-turn proof.

## Reproduction Assessment

Not applicable as a feature proposal. Source inspection gives a high-confidence current-main check: models have no `clear_goal` tool, and the docs explicitly reserve clearing for `/goal` and reset controls.

## Solution Assessment

Unclear until maintainers decide the product boundary. The canonical-store archive is a better shape than the discarded JSONL sidecar, but allowed states and archive lifecycle must be approved before implementation is the best fix.
