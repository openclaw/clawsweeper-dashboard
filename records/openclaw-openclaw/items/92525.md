## Label Justifications

- `impact:session-state`: The proposal changes who may clear session goals and would add retained historical goal state to the session store.

## Summary

This should stay open because current main and the latest release still intentionally keep goal clearing as an operator/session control, while the requested `clear_goal` tool would expand model authority and add a durable session-state archive contract that needs maintainer product approval.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that archives terminal goals into bounded canonical session state before clearing the active goal.

## Best Possible Solution

A maintainer should decide the model-authority boundary first; if approved, a new focused PR should define the archive lifecycle, start from the canonical session-store design, update tool metadata/docs/tests, and include real agent-turn proof.

## Reproduction Assessment

Not applicable as a bug reproduction: source and docs show the current behavior is intentional, with model tools limited to get/create/update and clearing reserved for `/goal clear` or reset.

## Solution Assessment

Unclear: canonical-store atomicity is the right direction compared with the discarded JSONL sidecar, but the allowed states, archive consumers, retention behavior, and model authority need maintainer approval before this is the best solution.
