## Summary

Keep open: current main and latest release still intentionally reserve goal clearing/replacement for operator/session controls, and the only `clear_goal` implementation attempts are the two closed unmerged prototypes. This remains the canonical product-decision issue for changing the model authority boundary and adding durable cleared-goal session history.

## What This Changes

Request to add a model-callable `clear_goal` tool that can clear terminal or paused goals while atomically archiving them in bounded canonical session state.

## Best Possible Solution

Decide the model-authority boundary first; if approved, implement a focused terminal-only `clear_goal` tool with canonical session-store archival, docs, mutation/catalog coverage, and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature and product-boundary proposal, not a bug report. Current source and docs clearly show the existing behavior: models can read/create/complete/block goals, while clearing remains an operator/session control.

## Solution Assessment

Unclear: moving archival into canonical session state is preferable to the discarded JSONL sidecar design, but model clear authority, clearable states, archive consumers, and retention semantics need maintainer approval before this is the best solution.

## Risks / Open Questions

- Allowing models to clear paused or budget/usage-limited goals could bypass states that currently require operator control.
- Persisting `clearedGoals` on `SessionEntry` would create reset, export, retention, synchronization, and privacy semantics that are not yet specified.
- A fixed archive of full goal snapshots may grow session state and expose historical objectives or notes to every session-store consumer without a proven retrieval requirement.
