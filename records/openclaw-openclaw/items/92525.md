## Summary

Keep open: current main and the latest release intentionally reserve goal clearing/replacement to operator or session controls, both prototype PRs are closed unmerged, and the requested model authority plus durable archive still needs a maintainer product decision.

## What This Changes

Requests a new model-callable `clear_goal` tool that clears terminal-or-paused goals while atomically archiving them in bounded `SessionEntry.clearedGoals` history.

## Best Possible Solution

Keep the current operator-only contract unless a feature owner explicitly approves model clearing; if approved, implement a terminal-only canonical transition with mutation/catalog/docs/tests and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature/product proposal. Source and docs show current main intentionally requires operator `/goal clear` or session reset before another goal can be created.

## Solution Assessment

Unclear: canonical-store rotation is the right storage direction if the feature is approved, but model authority, allowed states, retention, export, and reset semantics need maintainer decision before it can be called the best solution.
