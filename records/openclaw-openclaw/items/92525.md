## Summary

Current main and the latest release still intentionally reserve goal clearing for operator/session controls, and the only `clear_goal` implementations are closed, unmerged prototypes. This should stay open for an explicit maintainer product decision because it would change model authority and add a durable session-state archive contract.

## What This Changes

Request a model-callable `clear_goal` tool that clears terminal-or-paused goals and archives them into bounded canonical session state.

## Best Possible Solution

Make the model-authority decision first; if approved, land a focused implementation with explicit clearable states, canonical archive lifecycle docs, mutation/catalog wiring, tests, and real agent-turn proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature proposal. Source and docs do confirm the current intentional behavior: models cannot clear or replace goals, and operators must use `/goal clear` or reset the session.

## Solution Assessment

Unclear: canonical-store archival is a better storage direction than the discarded JSONL sidecar, but the model authority boundary, clearable states, and archive lifecycle must be approved before this is the best solution.

## Risks / Open Questions

- Allowing models to clear paused, blocked, or limited goals may weaken an operator-controlled lifecycle boundary that current docs treat as intentional.
- Adding `SessionEntry.clearedGoals` creates durable retention, reset, export, synchronization, and pruning semantics that are not yet specified.
- A bounded archive of full goal snapshots could expose historical objectives and notes to all session-store consumers without a decided retrieval or privacy model.
