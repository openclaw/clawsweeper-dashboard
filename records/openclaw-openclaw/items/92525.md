## Summary

Keep open: current main and the latest release still intentionally keep goal clearing as an operator/session control, and this issue asks maintainers to decide whether to expand model authority and add retained goal-history state. The related implementation attempts are closed unmerged, so no current PR owns the remaining product decision.

## What This Changes

Request to add or approve a model-callable `clear_goal` tool that clears terminal-or-paused goals into a bounded canonical session-store archive before starting new goals.

## Best Possible Solution

A maintainer should decide the model-authority boundary first; if approved, a new focused PR should use the canonical session-store archive design, update tool metadata/docs/tests, and include real agent-turn proof.

## Reproduction Assessment

Not applicable as a bug reproduction: source and docs show the current behavior is intentional, with model tools limited to get/create/update and clearing reserved for `/goal clear` or reset.

## Solution Assessment

Unclear until maintainers decide the authority boundary; the canonical-store archive is a better shape than the discarded JSONL sidecar, but the allowed states, retention behavior, and archive consumers need approval first.
