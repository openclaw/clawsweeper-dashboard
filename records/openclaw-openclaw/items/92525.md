## Summary

Current main intentionally requires operator/session controls to clear or replace goals, and no merged or open implementation owns this requested authority change. The issue should remain open for an explicit product decision because the earlier prototypes were closed and the proposed archive would create a new persistent session-state contract.

## What This Changes

Requests a product decision on adding a terminal-state-only model-callable `clear_goal` tool with bounded archival in the canonical session store.

## Best Possible Solution

Decide the model-authority boundary first; if terminal clearing is approved, start with only `complete` and `blocked`, define archive consumers and lifecycle semantics, and then open a focused implementation with real agent-turn proof.

## Reproduction Assessment

Not applicable: current main reliably reproduces the documented intentional behavior—after a model completes or blocks a goal, operator `/goal clear` or a session reset is required before another goal can be created.

## Solution Assessment

Unclear: canonical-store atomicity is preferable to the discarded JSONL design, but model authority, clearable states, archive consumers, and retention semantics must be decided before this can be called the best solution.

- Allowing models to clear paused or budget/usage-limited goals could bypass states that currently require operator action.
- Persisting `clearedGoals` on `SessionEntry` introduces retention, reset, export, synchronization, and upgrade semantics that are not yet specified.
- A fixed history of 50 full goal snapshots may grow the canonical session store and expose historical objective/note data to every store consumer without a demonstrated retrieval requirement.
