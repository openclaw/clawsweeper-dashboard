## Summary

Keep this open for maintainer/product review: current `main` and latest release still intentionally keep goal clearing as an operator/session control, and the two implementation attempts were closed so this issue is now the canonical decision point.

## What This Changes

Proposal to add a model-callable `clear_goal` tool that clears non-active goals while preserving a bounded canonical session-store archive.

## Best Possible Solution

Have the goal lifecycle owner decide the model-authority boundary first; if approved, implement a focused terminal-only canonical-store transition with catalog, mutation metadata, docs, tests, and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature and product-boundary proposal, not a broken existing behavior report. The premise is source-confirmed because current main and `v2026.6.6` both require `/goal clear` or reset before another goal can be created when one already exists.

## Solution Assessment

Unclear until maintainer review: canonical-store rotation is a better storage shape than the discarded JSONL sidecar, but the model authority change and clearable-state set must be approved before implementation is the best solution.
