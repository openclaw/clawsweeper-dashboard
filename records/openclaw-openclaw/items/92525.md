## Summary

Current main still intentionally keeps goal clearing behind operator/session controls, and no merged or open PR implements the requested `clear_goal` authority change. This issue remains useful as the canonical maintainer product-decision thread after the v01 and v02 implementation PRs were closed unmerged.

## What This Changes

The issue proposes adding a model-callable `clear_goal` tool plus a bounded canonical `SessionEntry.clearedGoals` archive for cleared terminal goals.

## Best Possible Solution

Keep this as the canonical maintainer decision thread; if approved, implement a focused v03 with explicit allowed states, archive lifecycle semantics, metadata/docs updates, and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature and product-boundary proposal. Current main reliably shows the documented intentional behavior: models can create, read, complete, and block goals, while clearing remains an operator/session control.

## Solution Assessment

Unclear until maintainers decide the authority boundary. The canonical-store archive is a better shape than the discarded JSONL sidecar, but the best solution depends on whether model-callable clearing is accepted and which states can be cleared.
