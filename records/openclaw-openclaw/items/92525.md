## Summary

Keep open: current main and the latest release still intentionally keep goal clearing operator/session-only, and the closed prototypes were converted into this issue specifically for a maintainer product decision. The remaining question is whether OpenClaw should change model authority and add durable cleared-goal session history.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that archives terminal or paused goals into bounded canonical session history before clearing the active goal slot.

## Best Possible Solution

Decide the model-authority boundary first; if approved, implement a narrow canonical session-store transition with explicit clearable states, docs, mutation/catalog coverage, focused tests, and real agent-turn proof.

## Reproduction Assessment

Not applicable as a feature/product-direction request. Current-main source and docs reliably show the intentional behavior: models can create, inspect, complete, or block goals, while clearing remains an operator/session control.

## Solution Assessment

Unclear. The canonical-store archive design is better than the earlier JSONL sidecar, but maintainers still need to approve model-callable clearing, the allowed states, and the archive lifecycle before this is the best solution.
