## Summary

Keep open: current `main` and `v2026.6.9` still expose only `get_goal`, `create_goal`, and `update_goal`, while the documented contract reserves clearing/replacement for operator/session controls. This issue is the canonical product-decision thread for whether to expand model authority and add retained cleared-goal session history.

## What This Changes

The issue proposes adding a model-callable `clear_goal` tool plus bounded canonical `SessionEntry.clearedGoals` history.

## Best Possible Solution

A maintainer should decide the model-authority boundary first; if approved, a new focused PR should use canonical session-store history, update metadata/docs/tests, and include real agent-turn proof.

## Reproduction Assessment

Not applicable as a bug reproduction: source and docs show the current behavior is intentional, with model tools limited to get/create/update and clearing reserved for `/goal clear` or reset.

## Solution Assessment

Unclear until maintainers decide the authority boundary; the canonical-store archive is a better shape than the discarded JSONL sidecar, but the allowed states, retention behavior, and archive consumers need approval first.
