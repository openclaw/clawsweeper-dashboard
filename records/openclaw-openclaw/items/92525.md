## Summary

This should stay open for maintainer product review: current main and the latest release intentionally keep goal clearing as an operator/session control, and the only `clear_goal` implementations found are closed, unmerged prototype PRs.

## What This Changes

Proposal to add a model-callable `clear_goal` tool that clears non-active goals while archiving cleared goal history in the canonical session store.

## Best Possible Solution

Maintainers should decide the model-authority boundary first; if approved, implement the smallest accepted clearable-state set and canonical archive semantics in one focused PR with real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature/product proposal, not a bug report. Source inspection confirms current main intentionally requires operator `/goal clear` or session reset before another goal can be created.

## Solution Assessment

Unclear: the canonical-store archive is a better storage shape than the closed JSONL prototype, but the model authority boundary, clearable states, retention, and archive consumers need maintainer decision first.

- A fixed archive of full goal snapshots may grow canonical session state and expose historical objective or note text to every session-store consumer without a defined retrieval requirement.
