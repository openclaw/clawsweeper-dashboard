## Summary

Keep open: current main and the latest release still intentionally expose only `get_goal`, `create_goal`, and `update_goal` to models, while goal clearing remains an operator/session control. The closed prototype PRs are useful design context, but no merged or open implementation owns this product/session-state decision.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that archives cleared terminal-or-paused goals on `SessionEntry` before removing the current goal.

## Best Possible Solution

Have a goal/session owner decide the model authority boundary and archive lifecycle; if approved, implement a narrow canonical transition with metadata, docs, focused tests, and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature/product proposal, not a broken existing contract. Source and docs confirm current main intentionally requires operator `/goal clear` or reset before a new goal can replace an existing one.

## Solution Assessment

Unclear: the canonical-store archive is the right direction compared with the closed JSONL sidecar prototype, but model authority, clearable states, and archive lifecycle must be approved before this is the best solution.
