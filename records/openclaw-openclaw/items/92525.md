## Summary

Keep open. Current main and latest release still intentionally expose only `get_goal`, `create_goal`, and `update_goal` to models while reserving clear/replace behavior for operator or reset controls, and the two related implementation PRs are closed unmerged, so the remaining work is an explicit product decision.

## What This Changes

The issue proposes a model-callable `clear_goal` tool that archives terminal or paused goals into bounded canonical session history before clearing the current goal.

## Best Possible Solution

Decide the model-authority boundary first; if approved, implement a terminal-only canonical session transition with clear retention semantics, complete tool metadata, docs, and real agent-turn proof.

## Reproduction Assessment

Not applicable: this is a feature and authority-boundary proposal. Current source and docs confirm the existing intentional behavior: a model can complete or block a goal, but clearing remains an operator/session control.

## Solution Assessment

Unclear: the canonical single-store archive is a better shape than the earlier JSONL sidecar, but the best solution depends on maintainer approval for model-callable clearing, allowed states, retention, and archive consumers.

## Risks / Open Questions

- Allowing models to clear paused, usage-limited, or budget-limited goals may bypass states that currently require operator/session action.
- A persistent `SessionEntry.clearedGoals` archive needs defined reset, deletion, export, synchronization, pruning, and privacy semantics before it becomes a durable contract.
- A fixed retention count for full historical goal snapshots could grow canonical session state and expose old objectives or notes to every session-store consumer without a proven retrieval requirement.
