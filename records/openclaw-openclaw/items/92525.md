## Summary

Current main intentionally reserves goal clearing and replacement for operator/session controls, while the two implementation attempts are closed unmerged and no other open item owns the decision. This issue should remain the canonical place for maintainers to decide whether to expand model authority and, if so, define the durable-history lifecycle before another implementation PR.

## What This Changes

Allow models to clear non-active session goals while atomically retaining a bounded history in the canonical session store.

## Best Possible Solution

Approve a precise core contract first—preferably limiting model clearing to truly terminal states unless maintainers intentionally include resumable states—then implement one canonical atomic transition with documented retention, reset/export semantics, mutation metadata, focused tests, and a real registered-tool agent proof.

## Reproduction Assessment

Yes at the source-contract level: current main has no model-callable clearing path, `create_goal` rejects any existing goal, and the docs intentionally reserve clearing for `/goal clear` or session reset. This confirms the stated gap, but it is expected behavior rather than a bug.

## Solution Assessment

Unclear pending the product decision. Atomic canonical-state rotation is the right storage shape if history is approved, but the proposed eligible states and retention policy are not yet proven to be the best authority contract.
