## Summary

This should stay open because current main and the latest release still intentionally reserve goal clearing/replacement for operator/session controls, while the requested model-callable clear/archive path changes model authority and durable session-state semantics. The two prototype PRs are closed unmerged, so no merged or viable open implementation currently owns the decision.

## What This Changes

Request a model-callable `clear_goal` tool that moves cleared goals into bounded canonical session history before removing the active `SessionEntry.goal`.

## Best Possible Solution

Decide the model-authority boundary first; if approved, implement a narrow canonical session-store transition with explicit clearable states, archive lifecycle docs, mutation metadata, focused tests, and real agent-turn proof.

## Reproduction Assessment

Yes, at source level: current main exposes only `get_goal`, `create_goal`, and `update_goal` to models, while `/goal clear` remains an operator control. This is not a bug reproduction because the behavior matches the documented contract.

## Solution Assessment

Unclear: canonical-store archival is a safer shape than the closed JSONL sidecar prototype, but model authority, clearable states, and archive lifecycle must be approved before this can be called the best solution.
