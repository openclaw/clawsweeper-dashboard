## Summary

## What This Changes

The branch normalizes cron run-log `jobId` values in `appendCronRunLog` before write-keying, insertion, and pruning, then adds regression tests for padded and unsafe IDs.

## Best Possible Solution

Keep the write-side whitespace normalization, but preserve currently loadable persisted job IDs unless maintainers separately add an explicit job-ID migration or validation policy.

No. Normalizing at `appendCronRunLog` is the right layer for whitespace, but reusing the read-side slash/backslash safety assertion on writes is too broad because current cron job storage can still load and run those IDs.
