## Summary

## What This Changes

## Best Possible Solution

## Reproduction Assessment

No high-confidence way to reproduce the full rejected-RPC zombie incident was established; current main source shows rejected `config.patch`/`config.apply` paths return before commit and restart handling. The narrower no-empty-diff-restart invariant is covered by PR tests and real helper proof.

## Solution Assessment

Yes for the scoped invariant: a shared helper guard is a narrow backstop for future refactors. It is not merge-ready as-is because it must be rebased onto current main and maintainers still need to accept the empty-diff `config.apply` compatibility tradeoff.
