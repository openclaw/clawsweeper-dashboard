## Summary

Keep open: this PR addresses a real WhatsApp duplicate-send bug and includes sufficient live proof, but the current coalescer can drop unrelated buffered attachments when only one media URL overlaps a later captioned payload.

## What This Changes

The PR buffers non-final WhatsApp media-only auto-reply payloads, coalesces them when a later captioned payload uses overlapping media, updates dispatcher coverage, and adds a changelog entry.

## Best Possible Solution

Keep the WhatsApp dispatcher-level coalescing approach, but filter or split only the duplicated media URLs so unrelated buffered attachments still flush normally.

## Reproduction Assessment

Yes. Current main source and the existing dispatcher test show a tool media payload can be delivered before a later captioned block payload with the same URL, and the linked issue plus PR proof include live WhatsApp duplicate-send logs/screenshots.

## Solution Assessment

No, not quite. Dispatcher-level coalescing is the right boundary, but dropping a whole pending payload on partial URL overlap is too broad; the fix should suppress only duplicated URLs and preserve the rest.

## Review Findings

Overall confidence: 0.86
