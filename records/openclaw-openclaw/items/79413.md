## Best Possible Solution

Implement a shared, model-aware reset-preservation guard that drops stale legacy `openai-codex` session pins for native Codex migration resets while preserving explicit user model selections.

## Solution Assessment

Yes in direction: the narrow fix belongs in the shared reset-preserved-selection path used by Gateway reset and chat reset. The closed related PR was not the best implementation because it used the non-model-aware runtime helper that currently returns `auto`.
