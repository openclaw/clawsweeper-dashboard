## Summary

This PR is superseded by the focused `agent-harness` SDK surface now on main and shipped, and its broad `agent-runtime` export is no longer the preferred boundary for active-run control.

## What This Changes

This PR adds five active-run steer/abort helpers from `src/agents/pi-embedded-runner/runs.ts` to the broad `openclaw/plugin-sdk/agent-runtime` barrel.

## Best Possible Solution

Keep active-run steer/abort on the focused trusted `agent-harness` SDK surface; open a new narrow SDK discussion only if plugins still need explicit active/streaming/wait inspection helpers.

## Reproduction Assessment

Not applicable as a feature/API-surface PR rather than a bug report. Source inspection shows the central steer/abort capability is already available through the focused `agent-harness` SDK path.

## Solution Assessment

No. Adding low-level active-run controls to the deprecated broad `agent-runtime` barrel is not the best path when current main already provides a narrower trusted-plugin surface.
