## Summary

Current main and shipped releases already expose the central steer/abort capability through the public `agent-harness` Plugin SDK surface, while this PR's older direct `agent-runtime` re-export would reopen an SDK-boundary and abort-scope decision that maintainers have already routed elsewhere.

## What This Changes

This PR adds five active embedded-run control helpers to the `openclaw/plugin-sdk/agent-runtime` barrel.

## Best Possible Solution

Keep the shipped `agent-harness` SDK surface as the supported steer/abort path, and handle any future state/wait introspection helpers as a narrower SDK design follow-up rather than merging this broad barrel re-export.

## Reproduction Assessment

Yes. Static source inspection verifies that current main does not add these names to `agent-runtime`, but it does expose the central steer/abort capability through the shipped `agent-harness` SDK surface.

## Solution Assessment

No, this PR is not the best path now. The maintainable solution already shipped as a focused `agent-harness` surface; directly re-exporting the raw helper from the deprecated broad `agent-runtime` barrel would publish the process-wide abort overload without a fresh API decision.
