## Summary

Keep open for maintainer review: the PR addresses a real SDK-boundary request, but current main already exposes steer/abort aliases through the trusted agent-harness SDK path, while this direct `agent-runtime` re-export still needs API-baseline/changelog alignment and an explicit decision on the process-wide abort overload.

## What This Changes

The PR adds a named re-export block in `src/plugin-sdk/agent-runtime.ts` for five active embedded-run steer, abort, state, and wait helpers.

## Best Possible Solution

Expose a supported active-run control surface for external channel plugins with session-scoped semantics or explicitly documented trusted capability, then update SDK docs, API baseline/export checks, changelog, and focused contract coverage.

## Reproduction Assessment

Yes. Static inspection on current main shows the exact `agent-runtime` exports are absent while the internal helpers exist; current main also has a partial steer/abort alias path through `agent-harness`.

## Solution Assessment

No, not as proposed. A direct broad-barrel re-export proves the missing surface, but the maintainable fix needs an explicit SDK boundary decision, a constrained or approved abort capability, and public SDK baseline/docs/changelog updates.
