## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested `agent-runtime` steer/abort exports, while the PR promotes a public SDK surface without settling the cross-session abort capability or committing the required SDK API baseline/docs updates.

## What This Changes

The PR adds a named re-export block in `src/plugin-sdk/agent-runtime.ts` for five active embedded-run control helpers from `src/agents/pi-embedded-runner/runs.ts`.

## Best Possible Solution

Expose a supported active-run control surface for external channel plugins, preferably as a narrow SDK subpath or `api.runtime.agent` helper set with a session-scoped abort wrapper unless maintainers explicitly approve the broader overload, then update SDK docs, API baseline/export checks, and focused contract coverage.

## Reproduction Assessment

Yes. The gap is reproducible by static inspection on current main: the five helpers exist internally but are absent from `openclaw/plugin-sdk/agent-runtime`, while `package.json` already exposes that SDK subpath.

## Solution Assessment

No, not as currently proposed. The minimal re-export proves the missing surface, but the maintainable fix needs an explicit SDK boundary choice and an abort-overload policy before this becomes a public contract.
