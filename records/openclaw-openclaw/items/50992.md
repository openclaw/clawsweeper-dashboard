## Summary

Keep open: current main still does not provide the requested public SDK path, and the PR is useful but not ready because it promotes internal active-run controls across the Plugin SDK boundary without the required API-surface follow-through and without maintainer approval for process-wide abort semantics.

## What This Changes

The PR adds five active embedded-run steer, abort, state, and wait helpers to the `openclaw/plugin-sdk/agent-runtime` barrel.

## Best Possible Solution

Expose a supported active-run control SDK surface for external channel plugins with session-scoped semantics or explicitly approved trusted abort authority, then update SDK docs, API baseline/export checks, changelog, and focused contract coverage.

## Reproduction Assessment

Yes. Static inspection on current main shows the published `agent-runtime` subpath exists but lacks the requested helper exports, while the internal helpers and trusted harness aliases do exist.

## Solution Assessment

No, not as proposed. A direct broad re-export demonstrates the missing surface, but the maintainable fix needs an explicit SDK boundary decision, constrained or approved abort semantics, and the required public API follow-through.
