## Summary

Close: current main has removed the vendored `vendor/a2ui` tree and moved A2UI into the Canvas plugin/workspace dependency flow, so this security lockfile PR is obsolete against the shipped architecture.

## What This Changes

The PR updates vendored A2UI manifests and npm/pnpm lockfiles under `vendor/a2ui` to bump markdown-it/Angular packages and pin vulnerable transitive dependencies.

## Best Possible Solution

Keep the plugin-owned A2UI implementation on current main; future A2UI vulnerability fixes should update the Canvas plugin/root workspace dependencies and lockfile rather than restoring `vendor/a2ui`.

## Reproduction Assessment

Not applicable as a current-main bug reproduction. The old reproduction was `trivy fs vendor/a2ui --scanners vuln`, but current main has no `vendor/a2ui` tree to scan.

## Solution Assessment

Yes. Removing the vendored A2UI dependency tree and using plugin/workspace-managed A2UI dependencies is the narrower maintainable solution than updating lockfiles in a deleted tree.
