## What This Changes

The issue asks the web UI to avoid losing typed composer text when the running-state Stop action is tapped in the Send button location.

## Best Possible Solution

Preserve or recover the composer draft on the visible Stop-button abort path, while keeping intentional typed `/stop` command cleanup separate if desired.

## Reproduction Assessment

Yes, at source level: an abortable run renders Stop through `renderChatRunControls`, the click calls `handleAbortChat`, and that handler clears `host.chatMessage` before aborting. I did not run a live iOS Safari session in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to stop the visible Stop action from discarding an unrelated composer draft, then update the abort/control tests. Moving the button left can remain optional UI polish if maintainers want the reporter’s preferred layout too.
