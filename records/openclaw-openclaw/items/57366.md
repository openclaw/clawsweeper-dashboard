## Summary

Keep open. The PR is a useful focused bug fix and it addresses the earlier regex/non-HTML review gaps, but it still only changes the raw-body placeholder while the separate Teams media resolver can download the same emoji sprite as media; the external PR also lacks after-fix real Teams proof.

## What This Changes

The branch adds `extractTeamsEmojiText()`, parser tests, and a changelog entry so MS Teams CDN emoji `<img>` tags can render as Unicode text instead of media-image placeholders.

## Best Possible Solution

Share the pure-emoji classification with Teams inbound media resolution so pure Teams emoji becomes Unicode text and produces no media payload, while mixed or real attachments keep the existing fallback behavior.

## Reproduction Assessment

Yes from source inspection. Current main counts Teams CDN emoji `<img>` tags as inline images, and the separate media resolver still downloads allowed inline image candidates from the same attachment list.

## Solution Assessment

No, not yet. Converting the raw-body placeholder to Unicode is the right direction, but the PR also needs to skip emoji-only CDN images in inbound media resolution so the agent does not still receive an image payload.
