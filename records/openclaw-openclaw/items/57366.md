## Summary

This PR should stay open: the parser edge cases from the early review are addressed, but the branch still only changes the rawBody placeholder path, while current main resolves inbound media from the same attachments separately. A pure Teams emoji CDN image can therefore still be downloaded as media, and the external PR still lacks real Teams after-fix proof.

## What This Changes

The PR adds MS Teams emoji CDN alt-text extraction to attachment placeholder handling, new parser tests, and a changelog entry.

## Best Possible Solution

Use one pure Teams emoji classification path for both placeholder generation and inbound media resolution so emoji becomes Unicode text and produces no sprite media, while mixed or real attachments keep the existing fallback behavior.

## Reproduction Assessment

Yes, source-reproducible: current main builds rawBody from the placeholder helper and separately sends the same attachments to inbound media resolution, whose downloader extracts allowed inline HTML image URLs. The PR changes only the placeholder path, so pure Teams emoji sprites can still be attached as media.

## Solution Assessment

No. Returning Unicode in rawBody is the right direction, but the fix also needs to suppress pure Teams emoji CDN images before media payload construction.
