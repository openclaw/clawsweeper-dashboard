## Summary

## What This Changes

This PR adds MS Teams CDN emoji `img` alt-text extraction to `buildMSTeamsAttachmentPlaceholder()`, plus parser tests and a changelog entry.

## Best Possible Solution

Use one pure Teams emoji classification path for both placeholder generation and inbound media resolution/download so pure emoji becomes Unicode text with no sprite media payload, while mixed or real attachments keep the existing fallback behavior.

## Reproduction Assessment

Yes. Source inspection shows the PR fast path affects only the raw-body placeholder, while current main still resolves the same attachments through the inline media downloader and allows the Teams emoji CDN host.

## Solution Assessment

No. Returning Unicode text is the right direction, but the maintainable fix must also suppress pure Teams emoji sprites before media payload construction and prove that behavior in a real Teams run.
