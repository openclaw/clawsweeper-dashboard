## Summary

Keep open: current main and the latest release still force `preferPersisted: false` in manifest-contract lookup, so standalone migration discovery can still miss globally installed Codex providers. The clean linked fix candidate at https://github.com/openclaw/openclaw/pull/89612 remains open and should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that standalone `openclaw migrate` provider discovery cannot see globally installed Codex migration providers because manifest-contract lookup forces `preferPersisted: false`.

## Best Possible Solution

Land the resolver-level persisted-aware lookup fix in https://github.com/openclaw/openclaw/pull/89612 with regression coverage, then close this issue after it reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows the standalone migrate path still forces `preferPersisted: false`, and a user reproduced the same failure on a packaged macOS install with the Codex plugin loaded.

## Solution Assessment

Yes. The best fix boundary is the generic manifest-contract runtime resolver used by migration-provider discovery; a Codex-specific fallback or new config knob would duplicate plugin discovery policy.
