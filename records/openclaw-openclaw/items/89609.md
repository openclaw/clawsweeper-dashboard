## Summary

Keep open: current main and the latest release still contain the manifest-contract `preferPersisted: false` lookup path, so standalone migration discovery can still miss globally installed Codex migration providers. The linked open PR is the active fix candidate and should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that `openclaw migrate codex` cannot discover globally installed Codex migration providers because standalone manifest-contract lookup disables persisted plugin registry reads.

## Best Possible Solution

Land the resolver-level persisted-aware lookup fix in https://github.com/openclaw/openclaw/pull/89612 with regression coverage, then close this issue after it reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows the standalone migrate path still forces `preferPersisted: false`, and the issue includes packaged Linux plus macOS reproductions with the Codex plugin installed and loaded.

## Solution Assessment

Yes. The best fix boundary is the generic manifest-contract runtime resolver used by migration-provider discovery; a Codex-specific fallback or new config knob would duplicate plugin discovery policy.
