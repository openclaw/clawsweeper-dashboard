## Summary

Keep open: current main still makes the standalone migration-provider lookup demand-only, so it can miss globally installed plugins even though the Codex plugin advertises the documented migration provider contract.

## What This Changes

The issue reports that `openclaw migrate list` and `openclaw migrate codex` cannot discover the globally installed Codex migration provider.

## Best Possible Solution

Make standalone migration-provider discovery include enabled persisted/global plugin records through the generic manifest contract path, then add regression coverage for a global Codex-like provider in list and resolve flows.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the standalone migrate command calls the manifest-contract resolver, which forces `preferPersisted:false` and therefore can omit global plugin install records. I did not run a live global npm install in this read-only pass.

## Solution Assessment

Yes, the best fix is a narrow generic discovery repair for migration-provider contracts, not a Codex-specific fallback or a new configuration knob.
