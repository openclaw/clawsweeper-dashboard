## Summary

Keep open: current main still forces demand-only manifest-contract discovery for migration providers, so globally installed Codex plugins can remain invisible to the standalone migrate CLI; the linked open PR #89612 is the active fix path.

## What This Changes

The issue reports that standalone `openclaw migrate` provider lookup cannot see the globally installed Codex migration provider because manifest-contract discovery disables persisted plugin registry reads.

## Best Possible Solution

Review and land PR #89612, or explicitly choose a different generic registry-discovery fix if maintainers reject using the default persisted snapshot path.

## Reproduction Assessment

Yes; source inspection shows the standalone migrate path forces `preferPersisted:false`, and reporters reproduced the same failure on Linux beta 2026.6.1-beta.1 and macOS OpenClaw 2026.6.6.

## Solution Assessment

Yes; the best fix is a narrow generic discovery repair at the manifest-contract metadata boundary, and PR #89612 is the current implementation path.
