## Summary

Keep open: current `main` still forces `preferPersisted: false` for manifest-contract runtime lookup, so standalone migration discovery can still miss globally installed Codex providers. The open clean fix candidate at https://github.com/openclaw/openclaw/pull/89612 directly references this issue and should be reviewed or landed before closing.

## What This Changes

The issue asks for standalone `openclaw migrate` provider discovery to see enabled global/persisted Codex migration providers instead of throwing `Unknown migration provider`.

## Best Possible Solution

Land or otherwise resolve the persisted-aware manifest-contract fix in the linked open PR, with regression coverage or release smoke for a globally installed migration-provider plugin, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the standalone migrate path still forces `preferPersisted: false`, and a user reproduced the same failure on the latest release with a globally installed Codex plugin.

## Solution Assessment

Yes. The best fix boundary is the generic manifest-contract runtime resolver used by migration-provider discovery; a Codex-specific fallback or new config knob would duplicate plugin discovery policy.
