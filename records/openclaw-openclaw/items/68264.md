Keep open. Current main and the latest release still treat canvas/browser state-changing actions as JSON or node-control operations, while visual output remains an explicit snapshot/screenshot path; changing that default needs maintainer product and security direction rather than cleanup closure.

## What This Changes

Issue requests reliable chat-visible confirmation or rendering for browser/canvas navigation or presentation actions when the chat environment only returns node/context validation errors.

## Best Possible Solution

Define the supported chat-visible contract for browser and canvas controls, then either keep explicit snapshot/screenshot with clearer node guidance or add a narrowly opt-in preview path coordinated with screenshot privacy and dashboard preview decisions.
