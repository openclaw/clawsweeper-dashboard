Keep open. Current main still has JSON-only state-changing canvas/browser controls and separate preview paths, so the reported chat-visibility gap remains and needs a maintainer contract decision rather than cleanup closure.

## What This Changes

## Best Possible Solution

Define the chat-visible contract for canvas/browser state changes, then implement either gated preview delivery after present/navigate or clearer tool/docs/error guidance requiring explicit node selection plus snapshot/screenshot, with focused regression coverage.
