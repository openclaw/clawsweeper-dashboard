Keep open. Current main and v2026.6.10 still keep canvas/browser state-changing actions separate from visual output, and the remaining chat-visible preview behavior needs product and security direction rather than cleanup closure.

## What This Changes

The issue requests reliable chat-visible confirmation or preview output after canvas/browser navigation instead of node/context validation errors or JSON-only tool results.

## Best Possible Solution

Define a security-reviewed chat-visible contract, then either keep explicit snapshot/screenshot with better node guidance or add a narrowly opt-in preview result for canvas/browser state changes.

No high-confidence live reproduction was run for the reporter's LM Studio/Gemma plus paired-node setup. Source inspection does show current main still uses explicit snapshot/screenshot for visual output and can still throw `node required` when no default canvas-capable node is available.
