## Summary

Keep open. Current main and v2026.6.10 still separate canvas/browser state-changing actions from visual output, and the remaining chat-visible preview behavior needs product and security direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to make canvas/browser navigation or presentation reliably produce chat-visible visual confirmation instead of node/validation errors or JSON-only output.

## Best Possible Solution

Define a security-reviewed chat-visible contract, then either keep explicit snapshot/screenshot with better node guidance or add a narrowly opt-in preview result for canvas/browser state changes.

## Reproduction Assessment

No high-confidence live reproduction of the reporter's LM Studio/Gemma plus paired-node setup was run. Source inspection does show current main still returns JSON for canvas/browser navigation, uses explicit snapshot/screenshot for visual output, and can still throw `node required` when no default canvas-capable node exists.

## Solution Assessment

Unclear. Automatic preview after every state-changing action is not obviously the safest fix; maintainers should choose between clearer explicit snapshot guidance, better node errors, or an opt-in preview mechanism.
