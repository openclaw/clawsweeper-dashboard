## Summary

Keep open. Current main still separates canvas/browser state-changing actions from preview-producing snapshot/screenshot actions, and the no-node error path remains reachable, so this is not implemented and still needs a maintainer product decision.

## What This Changes

Requests that canvas/browser navigation or presentation reliably show the current page or canvas state in chat, or otherwise confirm visual state changes instead of surfacing node/validation errors.

## Best Possible Solution

Define the intended chat-visible contract, then update tool behavior, docs, and regression coverage so agents either receive an explicit preview after selected state-changing actions or get clear guidance to call snapshot/screenshot with better node-selection errors.

## Solution Assessment
