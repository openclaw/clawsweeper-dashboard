## Summary

Keep open. Current main and v2026.6.10 still keep canvas/browser state-changing actions separate from visual output, while the remaining chat-visible preview behavior needs product and security direction rather than cleanup closure.

## What This Changes

Request that canvas/browser actions in chat reliably show or confirm the current visual page state instead of returning node/validation errors or JSON-only results.

## Best Possible Solution

Define a security-reviewed chat-visible contract, then either keep explicit snapshot/screenshot with better node guidance or add a narrow opt-in preview result for canvas/browser state changes.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current main still returns JSON for canvas/browser navigation, uses explicit snapshot/screenshot for visual output, and can still throw `node required` when no default canvas-capable node exists.

## Solution Assessment

Unclear. Automatic preview after every state-changing action is not obviously the safest fix; maintainers should choose between clearer explicit snapshot guidance, better node errors, or an opt-in preview mechanism.
