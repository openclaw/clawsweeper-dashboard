## Summary

Keep open. Current main still treats canvas/browser navigation as state-changing controls with explicit snapshot/screenshot preview paths, so the reported chat-visible confirmation gap is not implemented and needs maintainer product judgment.

## What This Changes

Requests that canvas/browser actions in chat reliably show navigated page content or visual-state confirmation instead of returning `node required` or validation-style errors.

## Best Possible Solution

Define the intended canvas/browser chat-visible contract, then implement either preview-producing results for selected state-changing actions or clearer explicit snapshot/screenshot guidance with focused regression coverage.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run. Source inspection does show the current-main paths behind the reported behavior: `present`/`navigate` return JSON, previews require explicit snapshot/screenshot actions, and `node required` remains reachable when no default node can be selected.

## Solution Assessment

Unclear because there is no proposed patch and the correct fix depends on maintainer policy. The narrowest maintainable path is to choose the preview/error contract first, then update tool behavior, docs, and tests together.
