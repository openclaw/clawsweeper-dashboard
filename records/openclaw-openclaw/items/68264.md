## Summary

Keep open. Current main and v2026.6.8 still separate canvas/browser state-changing actions from explicit snapshot/screenshot inspection, and the remaining chat-visible feedback behavior needs maintainer UX/security direction rather than cleanup closure.

## What This Changes

The issue asks for reliable chat-visible confirmation or visualization when canvas/browser navigation or presentation is used, instead of node/validation errors or JSON-only success.

## Best Possible Solution

Define the supported chat-visible contract for canvas and browser actions, then either improve explicit snapshot/screenshot guidance and node errors or add an opt-in preview path coordinated with screenshot privacy and dashboard preview decisions.
