## Summary

Keep open. Current main provides explicit `canvas snapshot`, `browser snapshot`, `browser screenshot`, and Control UI `[embed]` paths, and the older multi-node `node required` canvas bug has a merged fix, but `canvas present`/`canvas navigate` and `browser navigate` still do not automatically attach a chat-visible preview; the remaining behavior needs product direction and a live repro of the reported LM Studio/Gemma failure.

## What This Changes

The issue asks OpenClaw to make canvas/browser navigation or presentation reliably show visible page state in chat instead of returning node-selection, validation, or non-visual feedback.

## Best Possible Solution

Decide the intended UX first: either keep navigation and visual inspection as explicit `snapshot`/`screenshot`/`[embed]` steps with better guidance and errors, or add a narrowly scoped opt-in preview behavior with privacy-safe defaults and live proof.

## Solution Assessment
