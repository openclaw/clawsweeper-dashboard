## Summary

Keep open. Current main still keeps canvas/browser state-changing actions separate from preview-producing actions: `canvas.present`, `canvas.navigate`, and `browser.navigate` return JSON, while chat-visible media/text previews require explicit `snapshot` or `screenshot`; the `node required` resolver path also remains reachable. This needs a maintainer product decision rather than conservative cleanup.

## What This Changes

Request that canvas/browser navigate or present actions reliably render or confirm page state in chat instead of surfacing node or validation errors without a visible preview.

## Best Possible Solution

Define whether state-changing canvas/browser actions should emit preview media automatically or keep explicit snapshot/screenshot as the contract with clearer guidance/errors, then update tool behavior, docs, and focused coverage for that choice.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show the relevant current-main paths: navigate/present return JSON, previews require explicit snapshot/screenshot, and the shared resolver can still throw `node required` when no default node is selected.

## Solution Assessment

Unclear because there is no patch and the right fix depends on the intended product contract. A safe implementation needs maintainer direction on automatic previews versus explicit-preview guidance and improved node-selection errors.
