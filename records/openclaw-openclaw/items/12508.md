## Summary

Keep open. Current main still has mutable sequential plugin hooks without hook-chain integrity/provenance state, per-handler modification audit semantics, or a per-hook permission allowlist, and prior issue discussion explicitly kept this for maintainer security-design work rather than treating the narrower outbound-verification issue as a replacement.

## What This Changes

Request adding hook-chain integrity verification, audit logging, and per-hook permission controls for plugin hooks that can modify messages or context.

## Best Possible Solution

Define a host-owned hook provenance, audit, and permission contract that separates in-process native plugins from any isolated-plugin IPC boundary, then implement it compatibly with docs and focused runtime tests.

## Reproduction Assessment

Yes, at source level: current main shows sequential mutable hook execution, transcript-adjacent message mutation, and broad hook policy fields without chain integrity, per-handler audit, or per-hook allowlist controls. This is a feature and API-contract gap, not a failing documented behavior test.

## Solution Assessment

No, not as a narrow checksum-only patch. The maintainable solution needs a trust-boundary, audit redaction, permission, and compatibility contract that handles in-process plugins and future isolation together.

## Risks / Open Questions

- Checksum-style hook integrity may not provide meaningful protection against in-process native plugins unless maintainers first define the protected boundary and threat model.
- Per-handler audit logging can expose prompts, transcripts, tool outputs, local paths, or credential-adjacent data unless the audit contract is redacted and permission-aware.
- A fail-closed per-hook allowlist or integrity policy could break existing plugins without migration rules, diagnostics, docs, and compatibility tests.
- The narrower outbound verification work at https://github.com/openclaw/openclaw/issues/18571 may inform part of the design but does not cover inbound, tool, session, or transcript-adjacent hook mutation.
