## Summary

Keep open: current main and the latest shipped release still have mutable, sequential plugin hooks without hook-chain integrity state, host-owned modification audit/provenance, or a per-hook permission allowlist. The issue thread already treats this as security-sensitive plugin-runtime design, and the related outbound verification issue is narrower rather than a superseding implementation path.

## What This Changes

The issue requests hook-chain integrity checks, modification audit/provenance logging, and per-hook permission controls for plugin hook execution.

## Best Possible Solution

Define host-owned hook provenance and modification-audit semantics, distinguish native in-process trust from isolated-plugin behavior, then add a backwards-compatible per-hook permission policy with redaction, docs, and focused tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a security hardening and feature-design request. Source inspection gives high-confidence proof that integrity state, audit/provenance records, and per-hook permission allowlists are still missing on current main and v2026.5.12.

## Solution Assessment

No for checksum-only hook-chain integrity as a standalone fix. The maintainable path needs explicit trust-boundary, audit/redaction, config, manifest, and compatibility decisions before implementation.

  - body: Native plugins run in-process with Gateway-level trust, so checksums alone may not defend against a malicious loaded plugin unless the protected boundary and isolated-plugin behavior are specified.

  - body: Current modifying hooks can rewrite hook results or messages sequentially without a host-owned modification ledger, so enforcement needs explicit per-handler audit semantics instead of only a terminal block decision.
  - confidence: 0.88

  - body: The current registry policy only gates broad prompt and conversation hook categories, so a new per-hook allowlist needs compatibility, diagnostics, and docs to avoid silently breaking existing plugins.
  - confidence: 0.8
