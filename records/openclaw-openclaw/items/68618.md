The issue requests a Feishu channel configuration option that maps sender open_ids to custom display names.

## Best Possible Solution

Decide the Feishu-owned alias-map contract, then implement it in the plugin schema/resolver/docs/tests or explicitly reject it in favor of API permission guidance and existing `resolveSenderNames` behavior.

## Reproduction Assessment

Not applicable as a bug repro, but there is a high-confidence source check for the missing capability: current main only exposes `resolveSenderNames` and the resolver only performs Feishu API lookup. The latest release has the same limitation.

## Solution Assessment

Unclear until maintainers choose the config shape and precedence semantics. A native Feishu plugin map is plausible, but it should be a deliberate config contract rather than an automatic repair PR.
