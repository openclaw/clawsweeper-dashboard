Keep open: current main and the latest release still expose Feishu `resolveSenderNames` plus contact lookup only, with no native open_id-to-display-name alias map. ## What This Changes

Request to add a Feishu sender-name alias mapping config for specific open_id values, usable as a fallback or override for API-resolved names.

## Best Possible Solution

Define a Feishu plugin-owned, account-scoped alias-map config contract, apply it in the sender display-name path with documented precedence, and cover API success, API failure, DM, group, and opt-out behavior in docs and tests.
