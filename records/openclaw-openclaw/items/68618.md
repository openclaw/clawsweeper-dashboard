## What This Changes

The issue requests a Feishu sender-name alias mapping config keyed by open_id for fallback or override display names.

## Best Possible Solution

Define a Feishu plugin-owned, account-scoped sender alias-map config contract, apply it in the sender display-name path with documented precedence, and cover API success, API failure, DM, group, multi-account, and opt-out behavior.

## Reproduction Assessment

## Solution Assessment

Yes directionally: Feishu plugin config/runtime/docs are the right boundary, but maintainers should decide alias precedence and coordinate with active Feishu sender-name work before implementation.
