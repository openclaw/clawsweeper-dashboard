## Summary

Keep open: the updated patch addresses the earlier wrong-CODEX_HOME review point, but it still downgrades command approval from a user-layer config even though this run injects `features.hooks=true` through Codex session flags, which can skip the existing native tool policy promotion when hooks are actually enabled.

## What This Changes

The PR adds Codex app-server approval-policy downgrading for disabled Codex hooks, preserves that downgrade through OpenClaw tool-policy promotion, adds warning logs, and expands Codex app-server/native-hook tests.

## Best Possible Solution

Derive the downgrade from Codex's effective runtime hook state after OpenClaw session flags are accounted for, or avoid downgrading when OpenClaw is injecting a native hook relay config that enables hooks.

## Reproduction Assessment

Yes. A focused source reproduction is to set codex-home `features.hooks=false` with an OpenClaw `before_tool_call` policy and observe that PR head sends `approvalPolicy: never` even though the same thread config still injects `features.hooks: true`.

## Solution Assessment

No. The current approach downgrades before accounting for the higher-precedence Codex session flags that OpenClaw itself sends, so the fix should use effective runtime config or keep the policy promotion intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep policy promotion aligned with runtime hooks:** `extensions/codex/src/app-server/run-attempt.ts:556-558`
  - body: This downgrade runs before the native hook relay config is built, but that same config sends `features.hooks=true` as Codex session flags, which have higher precedence than the user codex-home config. With a stale `codex-home/config.toml` setting `hooks=false`, this can force `approvalPolicy="never"` and skip the existing implicit-yolo to `untrusted` promotion even though the spawned thread will have hooks enabled, weakening the `before_tool_call` policy path that the prior fix added.
  - confidence: 0.86
