## Summary

Keep open. The patch is a useful first heuristic, but it falls through to the opaque session id/date fallback after a skipped greeting, so it would not satisfy the linked request for conversation-core session titles; it also lacks after-fix real behavior proof.

## What This Changes

The PR adds a generic-greeting filter to `deriveSessionTitle()` and test coverage so `hi`, `hello`, and `hey` fall back to the session id/date title.

## Best Possible Solution

Define title derivation to prefer existing metadata, then a stored compaction summary or bounded meaningful recent user prompt, while preserving bounded transcript reads and displayName/subject priority.

## Reproduction Assessment

Not applicable as a strict bug reproduction, because this is a feature/UX request. Source inspection does show the current and proposed paths: current main uses only the first user message, and the PR would skip greetings straight to the session id/date fallback.

## Solution Assessment

No. The PR is a plausible small heuristic, but the maintainable fix should add a bounded meaningful title source or re-scope the PR so it does not claim to close the broader feature request.
