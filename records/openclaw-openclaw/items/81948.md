## Summary

Keep open. The PR is a plausible small heuristic, but it does not close the linked feature because skipped greetings still fall through to the opaque session-id/date fallback, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch adds a generic greeting-only filter to `deriveSessionTitle()` and a unit test expecting `hi`, `hello`, and `hey` to fall back to session-id/date titles.

## Best Possible Solution

Define a bounded derived-title contract that keeps display name and subject priority, then uses a compaction summary or a clearly meaningful recent user prompt before the session-id/date fallback, with coverage for gateway listing, tool listing, and TUI picker behavior.

Not applicable as a strict bug reproduction because this is a feature/UX PR. Source inspection clearly shows current `main` uses only `firstUserMessage` for derived titles and the PR would skip greetings into the existing session-id/date fallback.

## Solution Assessment
