## Summary

Keep open. The patch skips greeting-only first messages, but it still has no compaction-summary or later meaningful-message source, so the linked session-title feature would fall back to opaque session id/date titles; it also lacks contributor real behavior proof.

## What This Changes

The PR adds a generic-greeting filter to `deriveSessionTitle()` so greeting-only first user messages fall back to the session id/date title, with a unit test.

## Best Possible Solution

Define a bounded derived-title contract that keeps displayName/subject first, then uses a stored compaction summary or clearly meaningful recent user text, with gateway listing, `sessions_list`, and TUI picker coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction, because this is a UX feature. Source inspection does show the current and proposed paths: current main passes only `firstUserMessage`, and the PR falls through to the session id/date fallback after skipping a greeting.

## Solution Assessment

No. The greeting heuristic may be useful, but the maintainable fix needs a bounded higher-quality title source or the PR should stop claiming to close the broader linked feature.
