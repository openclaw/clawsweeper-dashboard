The PR changes `deriveSessionTitle()` to ignore a small set of generic greeting-only first user messages and adds unit coverage for `hi`, `hello`, and `hey`.

## Best Possible Solution

Repair the title pipeline so skipped greetings are replaced by a bounded meaningful transcript-derived source, or re-scope this PR so it does not close the broader derived-title issue.

## Reproduction Assessment

Yes from source, though this is a feature request rather than a strict bug repro. Current main passes only the first transcript user message into `deriveSessionTitle()`, so greeting-first sessions cannot derive a title from later meaningful content.

## Solution Assessment

No. Skipping greetings inside `deriveSessionTitle()` is only a plausible partial mitigation; the best fix needs the transcript title-field extraction or caller contract to supply a meaningful replacement candidate after the greeting is skipped.
