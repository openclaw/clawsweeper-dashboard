## What This Changes

The PR adds a Gateway helper that treats a small set of greeting-only first user messages as non-title candidates and adds unit assertions that those inputs fall through to the session-id/date fallback.

## Best Possible Solution

Revise this PR so skipped greetings are replaced by a bounded meaningful transcript-derived candidate, or explicitly narrow it so it does not close the broader compaction/later-message title issue.

## Reproduction Assessment

Yes, from source inspection. Current main supplies only the first transcript user message to title derivation, and the PR head skips greeting-only text only after that point, so later meaningful transcript content cannot become the fallback title.

## Solution Assessment

No. Skipping greetings inside deriveSessionTitle() is only a partial mitigation; the better fix belongs in the bounded transcript title-field extraction or caller contract so a skipped greeting can fall through to a meaningful title candidate.
