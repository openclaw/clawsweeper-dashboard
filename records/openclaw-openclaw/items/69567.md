## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after each derived checkout auth header in the CI and Testbox workflow checkout blocks.

## Best Possible Solution

Merge the focused mask registrations after workflow-security review and check rerun or override, while leaving unrelated zizmor findings to separate fixes.

## Reproduction Assessment

Yes, at source level. Current main has 13 custom checkout blocks that derive a Basic-auth header and pass it to Git without first registering it with `add-mask`; the inspected Actions screenshot demonstrates the runner redacts the same value class after registration.

## Solution Assessment

Yes. Registering the already-computed derived header immediately after construction and before the first Git invocation is the narrowest maintainable fix under the GitHub Actions masking contract.
