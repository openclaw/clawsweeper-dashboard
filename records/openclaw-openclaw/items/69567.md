## Summary

Keep open. Current main still derives checkout Basic-auth headers without registering the derived value with the Actions masker, and this PR adds only the missing mask registrations in the right place; merge should wait for workflow-security review and required-check handling.

## What This Changes

Adds `echo "::add-mask::${auth_header}"` after each derived custom checkout auth header in the CI and Testbox workflow files.

## Best Possible Solution

Merge the focused mask registrations after workflow-security review and check rerun or maintainer override, leaving unrelated zizmor findings to separate work.

## Reproduction Assessment

Yes, at source level. Current main has 13 custom checkout blocks that derive a base64 Basic-auth header and pass it to `git` before any `add-mask` registration, and the inspected Actions screenshot demonstrates the same value class is redacted after registration.

## Solution Assessment

Yes. Registering the already-computed derived header immediately after construction and before the first Git invocation is the narrowest maintainable fix under the GitHub Actions masking contract.
