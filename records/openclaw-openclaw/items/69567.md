## Summary

Keep open. The PR is a focused CI token-log hardening change that is not implemented on current main; the patch looks correct and has sufficient real behavior proof, but it still needs maintainer workflow-security review and required check resolution before merge.

## What This Changes

Adds `echo "::add-mask::${auth_header}"` immediately after each derived checkout auth header in `.github/workflows/ci.yml` and the two Testbox checkout workflows.

## Best Possible Solution

Merge the focused mask registrations after maintainer workflow-security review and check resolution, while handling unrelated zizmor findings in other workflows separately.

## Reproduction Assessment

Yes, at source level. Current main derives 13 base64 checkout auth headers and passes them to Git without registering the derived value with the Actions masker, and the forked demo run shows `add-mask` redacts the same shape of value.

## Solution Assessment

Yes. Registering the already-computed base64 header immediately after construction and before any Git invocation is the narrowest maintainable fix under the Actions masking contract.
