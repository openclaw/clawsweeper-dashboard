## Summary

Keep open. Current main still has the unmasked derived checkout headers, and this PR is a narrow, correct hardening patch, but the exact head still has failing workflow security checks that need maintainer resolution before merge.

## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after each derived checkout Basic-auth header in the main CI workflow and both Testbox checkout workflows.

## Best Possible Solution

Merge this focused hardening after the exact-head workflow security failures are green or explicitly accepted as unrelated; keep unrelated workflow audit cleanup out of this PR.

## Reproduction Assessment

Yes. A high-confidence static reproduction is current main: 13 checkout blocks compute a base64 `auth_header` and pass it to Git without any `::add-mask::${auth_header}` registration.

## Solution Assessment

Yes. `add-mask` is the narrow GitHub Actions mechanism for this generated log-sensitive value, and the PR leaves checkout logic, permissions, action refs, dependencies, and job graph unchanged.
