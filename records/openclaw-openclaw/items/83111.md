## Summary

Keep open, but the patch should not land as-is: current main already clears the pre-execution watchdog once process/model/tool execution starts, so the reported silent-tool path is not governed by this helper, while the diff weakens the documented early-stall detector for short timeouts.

## What This Changes

This PR changes the cron pre-execution watchdog ceiling from half the job timeout to the full job timeout in `src/cron/service/timer.ts`.

## Best Possible Solution

Keep the existing split: started process/model/tool execution should rely on the job timeout, and any remaining false abort should be fixed by identifying the last emitted phase or by continuing the separate watchdog tuning discussion.

## Reproduction Assessment

No. Current main clears the pre-execution watchdog on `tool_execution_started`, `process_spawned`, and `model_call_started`, so a silent tool after those milestones should be governed by the job timeout; the PR does not provide phase logs proving a current-main counterexample.

## Solution Assessment

No. Removing the divisor changes pre-execution stall policy rather than the started silent-tool path; the narrower maintainable fix is to patch any missing execution milestone or tune watchdog configuration with focused evidence.
