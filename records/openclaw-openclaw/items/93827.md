## What This Changes

This PR adds `browser.profiles` hot-reload metadata, changes config reload matching to prefer the longest prefix, removes one unused import, and adds planner plus authenticated `config.patch` regression coverage.

## Best Possible Solution

Land the browser-owned hot-prefix and longest-prefix planner change after a narrow docs update and maintainer acceptance of specific-prefix-wins reload semantics.

## Reproduction Assessment

Yes, at source level: current main and the latest release map `browser.profiles.*` through the broad `browser` restart prefix, and the hot-mode `config.patch` path schedules SIGUSR1 when the resulting reload plan requires a gateway restart. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes, with the docs gap fixed. The fix belongs in browser-owned reload metadata plus generic longest-prefix matching; making profile edits no-op would skip hot reload, and making all `browser.*` hot would be too broad while `browser.defaultProfile` is still selected before profile refresh.
