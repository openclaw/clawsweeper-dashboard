## Summary

Keep open: current main has partial Claude CLI continuity safeguards, but source inspection still shows a fresh-session amnesia path when a reusable CLI session hits `session_expired` and the bundled Claude backend has not opted into raw transcript reseeding.

## What This Changes

The issue requests preserving Claude CLI conversation history when a live CLI session rotates and OpenClaw resumes in a fresh process.

## Best Possible Solution

Use the existing bounded reseed machinery for safe Claude CLI fresh-session recovery, with regression coverage for `session_expired` and missing-transcript paths, while preserving native resume when it works.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only sweep. Source inspection does show a concrete path where a `session_expired` Claude CLI resume retries fresh without a prepared OpenClaw history prompt.

## Solution Assessment

Unclear until a focused patch proves the intended Claude path. The narrowest likely fix is to reuse the existing bounded transcript-reseed mechanism for the bundled Claude backend rather than adding a new `resumeHistoryMessages` config surface first.
