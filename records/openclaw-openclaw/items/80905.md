## Summary

Close: current main contains the bounded CLI transcript reseed mechanism, config contract, docs, and regression coverage; the raw transcript replay behavior was deliberately made opt-in for privacy rather than enabled by default.

## What This Changes

The issue asks OpenClaw to preserve Claude CLI conversation context across CLI process rotation by replaying a bounded recent transcript on resume.

## Best Possible Solution

Keep the current default-off, bounded raw transcript reseed and ask for a new report only if a current-main build with the opt-in enabled still loses Claude CLI context.

## Reproduction Assessment

Yes. The related current-main source path and tests show the old failure mode: invalidated CLI sessions without compaction could produce no history prompt, and current main now covers the opt-in raw-tail recovery path.

## Solution Assessment

Yes. The landed opt-in is the narrow maintainable solution because it restores continuity for safe invalidations while preserving the deliberate no-raw default across auth boundaries.
