## Summary

Keep open: current main still has a source-backed Claude CLI amnesia path where a reusable session that fails with `session_expired` retries as a fresh CLI run without a prepared OpenClaw history prompt for the bundled Claude backend.

## What This Changes

The issue reports Claude CLI conversation history being lost when OpenClaw rotates or recreates an underlying Claude CLI session.

## Best Possible Solution

Use the existing bounded reseed machinery for safe Claude CLI `session_expired` fresh-session recovery, with focused regression coverage and no new `resumeHistoryMessages` config surface unless maintainers choose a broader policy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection on current main shows a concrete `session_expired` resume-retry path that can start a fresh Claude CLI run with only the current prompt when no history prompt was prepared.

## Solution Assessment

No for the proposed new `resumeHistoryMessages` option as the first fix. The narrower maintainable path is to reuse the existing bounded transcript-reseed mechanism for the bundled Claude `session_expired` recovery path while preserving native resume when it works.
