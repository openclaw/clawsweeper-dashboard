## Summary

Keep open. Current main still drops the visible `session.message` for suppressed control replies and emits `sessions.changed` without a suppression marker, so the reported client-observability gap remains.

## What This Changes

Request adds a suppression marker and reason to `sessions.changed` `phase: "message"` broadcasts when display projection filters a silent-reply transcript update.

## Best Possible Solution

Define an additive Gateway event payload contract for suppressed transcript updates, then implement it at the broadcast/projection boundary with docs and regression coverage.

## Reproduction Assessment

Yes, by source inspection: a text-only assistant `NO_REPLY` update is filtered by display projection, causing no `session.message`, while the subsequent `sessions.changed` payload has no suppression marker. I did not run a live gateway because this was a read-only review.

## Solution Assessment

Unclear. The proposed broadcast location is likely narrow, but treating every falsy projection as `silent_reply` would be too broad because the same projection layer can hide other non-visible transcript updates.
