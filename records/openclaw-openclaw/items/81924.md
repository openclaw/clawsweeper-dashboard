## Summary

Keep open. Current main and the latest release still drop display-filtered silent-control transcript updates from `session.message` and emit a bare `sessions.changed` `phase: "message"` payload with no explicit suppression marker, so the requested gateway event-contract gap remains.

## What This Changes

The issue requests adding explicit suppression metadata to the existing gateway `sessions.changed` `phase:"message"` payload when a silent-control reply is filtered out of `session.message`.

## Best Possible Solution

Have a maintainer approve the additive `sessions.changed` suppression semantics, then update the broadcaster, protocol docs or schema surface if needed, and focused gateway tests for hidden control replies and visible messages.

## Reproduction Assessment

Yes, from source inspection: an exact silent-control assistant message without non-text content is display-filtered, and the fallback `sessions.changed` message-phase event has no explicit suppression metadata. I did not run a live gateway test because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public event contract. The narrow implementation likely belongs in `src/gateway/server-session-events.ts`, but field names, reason values, docs, and protocol typing need API/product confirmation first.
