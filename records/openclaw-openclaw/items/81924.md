## Summary

Keep open. Current main still omits `session.message` for text-only suppressed control replies and emits an indistinguishable `sessions.changed` frame, but the requested fix is a new Gateway payload contract that needs maintainer API approval before implementation.

## What This Changes

The issue requests a protocol-visible `sessions.changed` suppression marker for silent control replies whose `session.message` event is intentionally omitted.

## Best Possible Solution

Define an additive Gateway event payload contract for suppressed transcript updates, distinguish silent control replies from other hidden projections, then update `server-session-events`, protocol docs, and focused session-event regression coverage.

## Reproduction Assessment

Yes. By source inspection, a text-only assistant `NO_REPLY` update is filtered by display projection, so `session.message` is skipped while the following `sessions.changed` message-phase payload has no suppression marker.

## Solution Assessment

Unclear. The proposed broadcast location is narrow, but the public field names and reason taxonomy need maintainer agreement because `projectChatDisplayMessage` can be falsy for more than silent replies.
