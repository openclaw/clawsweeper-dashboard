## What This Changes

The issue requests an additive suppression marker on `sessions.changed` message-phase broadcasts when a silent control reply is filtered out of `session.message`.

## Best Possible Solution

Add an additive, typed Gateway event payload contract for suppressed transcript updates, document it, and cover the silent-token path with focused session-event regression coverage.

## Reproduction Assessment

Yes. By source inspection, a text-only assistant `NO_REPLY` update is filtered by display projection, causing no `session.message`, while the following `sessions.changed` payload has no suppression marker.

## Solution Assessment

Unclear. Reusing the existing message-phase `sessions.changed` broadcast looks like the narrowest hook, but the protocol-visible marker and reason taxonomy should be confirmed before implementation.
