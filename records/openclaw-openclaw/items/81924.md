## Summary

Keep open. Current main and v2026.6.6 still emit a bare `sessions.changed` `phase: "message"` event when display projection suppresses a transcript update, but they do not expose the requested explicit `suppressed` marker or reason field; this is a Gateway event-contract decision rather than stale cleanup.

## What This Changes

Add explicit suppression metadata to Gateway `sessions.changed` message-phase events when a silent-control transcript update does not produce a `session.message`.

## Best Possible Solution

Have a maintainer approve the additive Gateway payload semantics, then add explicit suppression metadata with focused tests and protocol docs.

## Reproduction Assessment

Yes, from source inspection: a `NO_REPLY`-only assistant row is display-filtered and the broadcaster falls back to `sessions.changed` without an explicit suppression marker. I did not run a live agent turn or tests because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public event contract. The narrow implementation likely belongs in the Gateway session-event broadcaster, but the reason names and docs/schema expectations need product/API confirmation first.
