## Summary

Keep open. Current main still lacks a Control UI per-assistant-message print action: the assistant message toolbar renders only canvas/open and copy-as-markdown controls, there is no Control UI `window.print` or print CSS path, and the related open PR covers a native Swift copy button rather than this webchat print feature.

## What This Changes

Add a Print button beside existing assistant-message actions in the Control UI webchat that prints only the selected assistant message with print-friendly formatting.

## Best Possible Solution

Implement this as a Control UI-only chat feature: add a print button next to the assistant message canvas/copy actions, render only that message through the existing sanitized markdown path into an isolated print view or print-only document, add simple print CSS for tables/code/text, and cover the behavior with focused UI tests plus a user-facing changelog entry crediting the reporter.

## Reproduction Assessment

Yes. Render or inspect an assistant message in Control UI on current main: `renderGroupedMessage` exposes only open-in-canvas and copy-as-markdown controls, and targeted source search finds no single-message print path.

## Solution Assessment

Yes. A narrow Control UI helper that reuses `toSanitizedMarkdownHtml` and isolates printed content is the most maintainable path; this does not require core, plugin API, or native Swift chat changes.
