## Summary

Keep open: this is a broad, security-sensitive Control UI feature request, not something current main already implements or something that can be safely swept into ClawHub today. Current main has PTY internals and exec approval surfaces, but no dashboard terminal and no plugin-rendered Control UI surface that would make this an external plugin-only cleanup close.

## What This Changes

Add a browser-embedded terminal in the Control UI backed by xterm.js, a WebSocket PTY bridge, session tabs, and existing gateway auth.

## Best Possible Solution

Define an opt-in, admin-scoped terminal design in Control UI and Gateway that reuses existing PTY/exec infrastructure where safe, includes audit and lifecycle controls, and documents the security model.

## Reproduction Assessment

Not applicable: this is a new feature request, not a broken existing behavior report. Source and docs inspection show related pieces exist, but no current-main dashboard terminal reproduction path exists because the feature is not implemented.

## Solution Assessment

Unclear: xterm.js plus a PTY bridge may be a plausible implementation primitive, but this is not ready as a narrow cleanup fix. The maintainable path first needs product and security decisions about privilege, opt-in policy, lifecycle, and whether a new plugin/UI API is required.
