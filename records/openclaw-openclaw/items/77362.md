## Summary

Keep open. Current main does not implement a browser terminal, and this is a new privileged Control UI/Gateway capability that needs maintainer product and security design before code.

## What This Changes

Add an embeddable xterm.js-style terminal to Control UI backed by a Gateway PTY bridge, session tabs, existing auth, and optional Canvas embedding.

## Best Possible Solution

Design an explicit opt-in terminal capability or plugin/API seam with Gateway scopes, exec approval/audit behavior, sandbox/node boundaries, session cleanup, and Control UI UX before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection shows current Control UI docs/navigation and Gateway methods omit a browser terminal while PTY support is limited to exec/process tooling.

## Solution Assessment

No, not as a ready cleanup implementation path. xterm.js and node-pty may be plausible building blocks, but the maintainable solution first needs an operator-controlled security model and protocol/API design.
