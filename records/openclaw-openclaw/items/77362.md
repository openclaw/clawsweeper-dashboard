## Summary

Keep open. Current main does not implement an embeddable Control UI web terminal, and the request would add a privileged browser-to-host shell surface that needs maintainer product and security design rather than conservative cleanup.

## What This Changes

Request to add an authenticated xterm.js-style web terminal with a WebSocket PTY bridge and session tabs to the Control UI dashboard.

## Best Possible Solution

Design an explicit, opt-in Control UI/Gateway terminal capability or plugin/UI API expansion that reuses existing exec, process, approval, isolation, and audit contracts before adding a PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not a current behavior bug. Source and docs inspection confirms the requested web terminal is absent today, but that is missing functionality rather than a reproducible regression.

## Solution Assessment

No, not as a ready-to-implement cleanup item. The xterm.js plus PTY direction is plausible, but the maintainable solution needs a maintainer-approved security and product model for browser-to-host shell access first.
