## Summary

Keep open. This is a new privileged Control UI/Gateway feature, not implemented on current main and not safely closeable as ClawHub-only because the current plugin UI surface does not expose an arbitrary embedded terminal/PTY bridge, while browser-to-host shell access needs maintainer security and product design.

## What This Changes

The issue requests an embeddable Control UI web terminal backed by xterm.js and a WebSocket PTY bridge, with session tabs, existing auth integration, and optional Canvas embedding.

## Best Possible Solution

Design an explicit, opt-in Control UI/Gateway terminal capability or plugin UI API expansion that reuses existing exec/process, approval, scope, audit, and session-isolation contracts before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not a current behavior bug. Source and docs inspection show no browser terminal RPC, xterm.js dependency, or dashboard terminal surface, while existing PTY support belongs to agent exec/process tooling.

## Solution Assessment

No, not as a ready cleanup item. The xterm.js plus PTY direction is plausible, but the maintainable solution needs maintainer-approved security, product, and extension-surface design first.
