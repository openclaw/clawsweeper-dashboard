## Summary

Keep open. This is a real new Control UI and Gateway capability, not something already implemented on main or safely closeable as ClawHub-only, because an embedded browser terminal would add a privileged host shell surface that needs maintainer product and security design.

## What This Changes

Requested behavior: add an authenticated browser-based terminal to the Control UI dashboard using xterm.js plus a server-side PTY bridge.

## Best Possible Solution

Design an explicit, opt-in Control UI/Gateway terminal capability or a plugin/UI API expansion that reuses existing exec, process, approval, scope, audit, and session-isolation contracts instead of adding an unscoped PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not a current behavior bug. Source and docs inspection confirms the requested web terminal is absent today, but that is missing functionality rather than a reproducible regression.

## Solution Assessment

No, not as a ready-to-implement cleanup item. The xterm.js plus PTY direction is plausible, but the maintainable solution needs a maintainer-approved security and product model for browser-to-host shell access first.
