## Summary

Keep open. The requested browser terminal is not implemented on current main, and it is not a ClawHub-only cleanup because direct browser-to-host shell access needs new Control UI/Gateway capability plus explicit product and security design.

## What This Changes

Add an xterm.js/WebSocket PTY terminal with session tabs and existing gateway auth to the Control UI dashboard.

## Best Possible Solution

Design an explicit, opt-in terminal capability or plugin/UI API expansion that reuses existing exec, approval, scope, audit, sandbox/node-host, and lifecycle contracts instead of adding an unscoped PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current behavior. Source and docs inspection give high confidence that the requested web terminal is absent today.

## Solution Assessment

No, not as a ready-to-implement cleanup item. xterm.js plus a PTY bridge may be a plausible implementation detail, but the maintainable solution first needs a maintainer-approved security and product model for browser-to-host shell access.
