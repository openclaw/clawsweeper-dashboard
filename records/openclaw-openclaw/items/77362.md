## Summary

Keep open. Current main still has no Control UI web terminal or Gateway terminal-session RPC, and the request would add a new privileged browser-to-host shell capability that needs maintainer product and security design before implementation.

## What This Changes

Requests an embeddable Control UI terminal backed by xterm.js, a WebSocket PTY bridge, session tabs, Gateway auth, and optional Canvas embedding.

## Best Possible Solution

Design an explicit opt-in terminal capability or plugin/API seam that reuses Gateway auth, scopes, approval/audit policy, session cleanup, and sandbox/node boundaries before adding a PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show no Control UI terminal today, while existing PTY support is limited to exec/process tooling.

## Solution Assessment

No, not as a ready implementation path. xterm.js plus a WebSocket PTY bridge may be plausible mechanically, but the maintainable solution first needs an operator-controlled security model and protocol/API design.
