## Summary

Keep open. This is a new privileged Control UI and Gateway capability: current main has PTY-backed agent exec and exec approval controls, but no browser terminal route/RPC, and direct host-shell access needs maintainer product and security design first.

## What This Changes

Add an embeddable xterm.js-style web terminal to the Control UI dashboard backed by a Gateway-authenticated PTY bridge.

## Best Possible Solution

Design an explicit opt-in operator terminal capability with scoped Gateway methods, approval/audit semantics, session cleanup, sandbox/node boundaries, and docs before adding any PTY bridge UI.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current Control UI capabilities omit a terminal while PTY support exists only for agent exec/process tooling.

## Solution Assessment

No, not as a ready implementation path. xterm.js plus node-pty may be plausible components, but the maintainable solution first needs an operator-controlled security model and Gateway protocol/API design.
