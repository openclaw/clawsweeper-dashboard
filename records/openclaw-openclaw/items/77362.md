## Summary

Keep open. This is a new privileged Control UI and Gateway capability, not a bug or already-implemented cleanup item; current main has agent exec/PTY plumbing and approval editing, but no browser terminal, and direct browser-to-host shell access needs maintainer product and security design first.

## What This Changes

Request to add an xterm.js-based web terminal with a WebSocket PTY bridge, session tabs, auth integration, and optional Canvas embedding to the Control UI dashboard.

## Best Possible Solution

Design an explicit opt-in terminal capability or plugin/API seam that reuses Gateway auth, scopes, exec approval/audit, session cleanup, and sandbox/node boundaries before adding any PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection shows current Control UI capabilities omit a terminal while existing PTY support belongs to agent exec/process tooling.

## Solution Assessment

No, not as a ready cleanup implementation path. xterm.js plus PTY may be a plausible component choice, but the maintainable solution first needs an operator-controlled security model and protocol/API design.
