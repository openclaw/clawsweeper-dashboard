## Summary

Keep open. This is a new privileged Control UI and Gateway capability, not a bug or cleanup item; current main has exec PTY plumbing, log tailing, exec approvals, and an OpenShell sandbox backend, but no browser terminal RPC/tab and the direct shell surface needs maintainer product and security design first.

## What This Changes

Request to add an authenticated xterm.js-backed web terminal with a Gateway PTY bridge and session tabs to the Control UI dashboard.

## Best Possible Solution

Design an explicit opt-in Control UI terminal capability or plugin API seam that reuses Gateway auth, scopes, exec approval/audit policy, session cleanup, and sandbox/node boundaries before adding a PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current Control UI capabilities omit a terminal while existing PTY support belongs to agent exec/process tooling.

## Solution Assessment

No, not as a ready cleanup implementation path. xterm.js plus PTY may be plausible, but the maintainable solution first needs an operator-controlled security model and Gateway/UI protocol design.
