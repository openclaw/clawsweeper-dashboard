## Summary

Keep open. This is a new privileged Control UI and Gateway feature request, not cleanup: current main has exec-tool PTY support, logs, and exec-approval UI, but no browser terminal route, xterm dependency, terminal-session RPC contract, or security model for arbitrary browser-to-host shell access.

## What This Changes

The issue requests an embedded xterm.js browser terminal in the Control UI dashboard backed by a WebSocket PTY bridge and authenticated terminal sessions.

## Best Possible Solution

Design an explicit opt-in terminal capability or plugin/API seam with scoped Gateway auth, auditability, approval or policy integration, lifecycle cleanup, and sandbox/node boundaries before implementing any PTY bridge.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection shows current main has no Control UI terminal route or terminal-session RPC surface.

## Solution Assessment

No, not as a ready implementation path. xterm.js plus node-pty may be plausible components, but the maintainable solution first needs a security and Gateway/plugin protocol design.
