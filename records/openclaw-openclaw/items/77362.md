## Summary

Keep open. This is still a new privileged Control UI/Gateway feature request, not conservative cleanup: current main has agent exec PTY support and exec-approval/logging UI, but no browser terminal route, xterm dependency, terminal-session RPC/event contract, or settled security model for arbitrary browser-to-host shell access.

## What This Changes

Add an authenticated xterm.js-based web terminal with WebSocket PTY sessions inside the Control UI dashboard.

## Best Possible Solution

Design an explicit opt-in terminal capability or plugin/API seam with scoped Gateway auth, auditability, approval or policy integration, session lifecycle cleanup, and sandbox/node boundaries before implementing the PTY bridge or UI route.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection shows current main does not expose a Control UI terminal route or terminal-session RPC surface.

## Solution Assessment

No, not as a ready implementation plan. xterm.js and node-pty may be viable components, but the maintainable solution first needs a security, Gateway protocol, and product policy design.
