## Summary

Keep open. The requested browser terminal is a new Control UI and Gateway capability, not an already-implemented feature or a ClawHub-only cleanup item, and it would expose direct host shell access that needs maintainer product and security design first.

## What This Changes

Add an xterm.js-style web terminal with a Gateway PTY bridge and session tabs to the Control UI dashboard.

## Best Possible Solution

Design an explicit, opt-in Control UI/Gateway terminal capability or a plugin/UI API expansion that reuses existing exec, process, approval, scope, audit, and session-isolation contracts.

## Reproduction Assessment

Not applicable: this is a feature request, not a current behavior bug. Source and docs inspection confirm the requested web terminal is absent today, but that is missing functionality rather than a reproducible regression.

## Solution Assessment

No, not as a ready-to-implement cleanup item. The xterm.js plus PTY direction is plausible, but the maintainable solution needs a maintainer-approved security and product model for browser-to-host shell access first.
