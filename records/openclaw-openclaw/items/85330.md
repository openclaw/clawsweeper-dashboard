## Summary

Keep open: the linked docs bug is real for sandboxed `auto` sessions, but this PR overcorrects by documenting all per-call `host=node`/`host=gateway` overrides from `auto` as rejected, which conflicts with current source and shipped tests for no-sandbox sessions.

## What This Changes

The PR changes one note in `docs/tools/exec.md` to describe `tools.exec.host=auto` as sandbox-or-gateway routing that rejects per-call `host=node` and `host=gateway` overrides.

## Best Possible Solution

Revise the docs to state the sandbox-dependent contract: `auto` resolves to sandbox when sandbox is active and gateway otherwise; gateway/node overrides from `auto` are rejected in sandboxed sessions, while the no-sandbox override behavior remains documented as supported.

## Reproduction Assessment

Yes. Source and focused tests reproduce the relevant contract: no-sandbox `auto` allows `node`/`gateway` overrides, while sandboxed `auto` rejects them.

## Solution Assessment

No. The PR is pointed at the right docs mismatch, but the replacement sentence is not the narrowest maintainable fix because it drops the tested no-sandbox behavior.
