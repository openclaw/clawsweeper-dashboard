## Summary

Keep open: current main and the latest release still leave cold-boot runtime-loaded workspace plugins detached from the gateway HTTP-route registry and plugin service lifecycle, and the linked fix PRs are open rather than merged.

## What This Changes

The issue asks OpenClaw to wire runtime-loaded workspace plugin HTTP routes and background services into the gateway during cold boot.

## Best Possible Solution

Land or replace a gateway-owned deferred runtime attachment fix that updates the attached route registry and starts/stops plugin services for runtime-loaded workspace plugins, with focused regression coverage and real curl/service proof.

## Reproduction Assessment

Yes, source-level. Current main shows the cold-boot deferred prewarm path loads runtime plugins without replacing the gateway-attached route registry or starting services; I did not run the live VPS curl/service reproduction.

## Solution Assessment

Yes for the direction: the maintainable fix belongs in gateway-owned deferred runtime attachment and service lifecycle handling. Route-only bootstrap pinning is too narrow because it does not cover the reported service-start failure.
