## Summary

Keep open: current main and the latest release still show a source-visible gap where cold runtime plugin prewarm can load a registry without attaching it to the gateway-owned HTTP route registry or starting that registry's services.

## What This Changes

The issue reports runtime-loaded workspace plugins losing HTTP route dispatch and service startup on cold boot when they are loaded from `plugins.load.paths` without `activation.onStartup`.

## Best Possible Solution

Add a gateway-owned deferred plugin attachment path that updates the server-local route registry and starts/stops services for the same loaded registry, with regression coverage for cold runtime-loaded HTTP routes and services.

## Reproduction Assessment

Yes for a source-level reproduction path: current main and `v2026.6.8` both show cold runtime prewarm can load runtime plugins without attaching that registry to the gateway route/service lifecycle. I did not run the reporter's packaged gateway/curl repro in this read-only review.

## Solution Assessment

Yes, the additive gateway-owned attachment direction is the best fit because it preserves the startup pinned route-registry protection while wiring the deferred registry into the same HTTP and service lifecycle.

## Repair Work Prompt

Fix the cold-boot deferred runtime plugin load gap described in https://github.com/openclaw/openclaw/issues/94572. Runtime-loaded workspace plugins from `plugins.load.paths` that are not `activation.onStartup` should have SDK `registerHttpRoute` routes visible to the running gateway and SDK `registerService` services started by the host lifecycle. Keep the fix gateway-owned: attach the loaded deferred registry through the same server-local route/gateway-method/session/channel surfaces used by startup or reload paths, and start/stop services for that registry with an owned handle. Do not remove the pinned startup HTTP-route registry protection from the earlier registry-churn fix, do not start services from generic tool discovery or CLI registry loads, do not add config flags, and do not edit `CHANGELOG.md`. Add focused regression coverage for cold runtime-loaded HTTP route dispatch and service startup, plus guardrails that generic standalone registry loads do not accidentally start services.
