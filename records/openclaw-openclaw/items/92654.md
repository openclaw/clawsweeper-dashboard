## Best Possible Solution

Land the UI-only quoted-Markdown reply workflow after the reply target is scoped to the selected session or cleared on session changes, with focused tests covering selection, cancel, quoted send, and session-switch behavior.

## Reproduction Assessment

Yes, from source inspection: select a reply in one session, switch sessions, and the reset path clears chat draft/messages but not the new reply target while handleSendChat prepends any existing target. I did not run a browser reproduction for that edge case.

## Solution Assessment

No, not as submitted. The UI-only quoted-Markdown layer is a narrow fit for the linked feature, but the transient reply target must be session-scoped or cleared before this is the best fix.
