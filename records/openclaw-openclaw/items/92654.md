Full review comments:

- **[P1] Guard optional host update callbacks:** `ui/src/ui/app-render.ts:3588-3592`
  - body: `requestHostUpdate` is `(() => void) | undefined`, but these new reply callbacks call it unconditionally. CI fails both prod and test type checks with TS2722 on these lines, so use `requestHostUpdate?.()` or otherwise narrow it before invoking.
  - confidence: 0.99
- **[P2] Focus the composer after selecting Reply:** `ui/src/ui/views/chat.ts:1689-1691`
  - body: The linked workflow expects the user to keep typing immediately after choosing Reply, but the menu click only sets the reply target and removes the body menu. Focus the existing composer textarea after selection so keyboard input goes where the feature promises.
  - confidence: 0.86
- **[P3] Clamp the context menu to the viewport:** `ui/src/ui/views/chat.ts:1687-1688`
  - body: The fixed-position menu is placed directly at `clientX` and `clientY`, so right-clicking near the right or bottom edge can render Reply partly off-screen. Measure after append and clamp the coordinates inside the viewport.
  - confidence: 0.83
