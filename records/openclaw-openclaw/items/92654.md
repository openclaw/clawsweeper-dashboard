## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the native menu when Reply is unavailable:** `ui/src/ui/views/chat.ts:1660-1663`
  - body: This handler calls `preventDefault()` as soon as the target is a chat bubble, then returns without showing any replacement menu when `onSetReply` is absent, the group is missing, the message is streaming, or the bubble has no text. That regresses the browser's existing right-click copy/select behavior for those cases, so only suppress the native menu once the custom menu will actually be shown or provide an equivalent fallback.
  - confidence: 0.88
- **[P2] Compute reply text before submit dedupe:** `ui/src/ui/app-chat.ts:1805-1806`
  - body: The submit guard key is built from the raw draft before `effectiveMessage` includes the selected quote. While a model switch or send is still in flight, replying with the same text to two different messages produces the same guard key and `withChatSubmitGuard` drops the second send even though the queued messages would differ.
  - confidence: 0.82
