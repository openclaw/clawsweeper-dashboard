## Summary

Keep this PR open: current main still has the ClawHub legacy ZIP metadata gap, and this branch is a focused fix with regression coverage and no blocking review findings found. Exact-head CI had no failures in the public check-run data I saw, but two checks were still in progress at review time.

## What This Changes

The branch broadens ClawHub artifact resolver metadata types, falls back to version metadata for summary-shaped legacy ZIP artifacts, updates ClawHub regression coverage and the changelog, and includes one formatting-only test-support change.

## Best Possible Solution

Land the focused fallback after normal maintainer merge handling and exact-head checks, preserving fail-closed archive verification for ClawHub legacy ZIP installs.

## Reproduction Assessment

Yes, source-reproducible: current main accepts top-level legacy ZIP `sha256` as archive integrity and does not fetch version metadata after a successful artifact resolver response. The live ZeroAPI endpoint now includes matching `artifactSha256`, so that package no longer reproduces the older missing-field shape unaided.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it keeps ClawPack and explicit archive-verification paths intact, refuses to trust summary `sha256` for legacy ZIP archive bytes, and only falls back to version metadata when resolver metadata is incomplete.
