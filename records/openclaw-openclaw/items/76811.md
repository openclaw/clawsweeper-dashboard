## Summary

Keep open for normal PR handling. Current main still lacks the fallback/safety rule for summary-shaped legacy ZIP artifacts, and this PR’s focused code, regression test, changelog entry, and exact-head checks show no merge-blocking review findings.

## What This Changes

The branch widens ClawHub artifact resolver typing, avoids treating summary legacy ZIP `sha256` as archive integrity, fetches version metadata as a verification fallback, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land the PR after normal maintainer merge handling, keeping version metadata as the fallback verification source for summary-shaped legacy ZIP artifacts.

## Reproduction Assessment

Yes, by source-modeled reproduction: current main can use a top-level legacy ZIP `sha256` as archive integrity, while the PR regression supplies an intentionally wrong summary `sha256` and expects a version metadata fetch. The live ZeroAPI endpoint now includes `artifactSha256`, so the live endpoint no longer proves the historical mismatch by itself.

## Solution Assessment

Yes. The PR is the narrow maintainable fix: it preserves existing ClawPack and explicit archive-verification paths, and only fetches version metadata when the artifact resolver response lacks usable verification.
