## What This Changes

The PR broadens ClawHub artifact resolver metadata handling, falls back to version metadata for legacy ZIP resolver responses without explicit archive verification, updates regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the PR after normal maintainer merge handling, preserving version metadata as the fallback verification source for summary-shaped legacy ZIP artifacts.

Yes, source-reproducible: current main treats top-level legacy ZIP `sha256` as archive integrity and does not fetch version metadata after a successful artifact resolver response. ## Solution Assessment

Yes. The PR is the narrow maintainable fix because it preserves ClawPack and explicit archive-verification paths, and only fetches version metadata when the artifact resolver response lacks usable verification.
