Keep open for maintainer review: this PR addresses a source-reproducible ClawHub install verification gap, includes sufficient real behavior proof, and I found no blocking functional or security findings.

## What This Changes

The branch updates ClawHub artifact resolver typing and install resolution to fetch version metadata when legacy zip resolver responses lack trusted archive verification, adds regression coverage and a changelog entry, and carries two formatting-only agent file changes.

## Best Possible Solution

Land the version-metadata fallback and regression test after maintainer review and required checks if maintainers agree that legacy zip summary `sha256` is not archive byte integrity.

## Reproduction Assessment

Yes, at source level: current main can treat `artifact.sha256` on a legacy zip summary as archive integrity and has no version-endpoint fallback after a successful artifact resolver response. I did not run the live install path in this read-only review.

## Solution Assessment

Yes. The PR keeps the artifact resolver fast path, narrows legacy zip digest trust to explicit archive metadata, and fetches version metadata only when verification is otherwise missing.
