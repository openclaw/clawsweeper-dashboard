## Summary

Keep open. The PR remains relevant because current main still accepts top-level legacy zip `artifact.sha256` as archive integrity, while this branch routes summary-shaped legacy zip responses through version metadata and adds focused regression coverage plus a changelog entry.

## What This Changes

The PR broadens ClawHub artifact resolver typing, avoids trusting summary legacy zip `sha256` as archive-byte integrity, fetches version metadata as the fallback verification source, updates the regression test, and adds a changelog entry.

## Best Possible Solution

Merge the narrow metadata fallback once CI and maintainer gates pass, keeping version metadata as the legacy zip archive integrity source.

## Reproduction Assessment

Yes. Source inspection shows current main can use top-level legacy zip `artifact.sha256` as archive integrity, and the PR regression test models that digest as intentionally wrong so version metadata must be fetched instead.

## Solution Assessment

Yes. Fetching version metadata only when the artifact resolver lacks usable archive or ClawPack verification is the narrowest maintainable fix and avoids trusting ambiguous summary artifact hashes.
