## Summary

## What This Changes

The PR widens ClawHub artifact resolver typing, avoids treating summary legacy zip `sha256` as archive integrity, fetches version metadata as a fallback, updates regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the PR after the remaining exact-head CI finishes, keeping version metadata as the fallback verification source for summary-shaped legacy zip artifacts.

## Solution Assessment

Yes. The PR uses the narrowest maintainable path: keep ClawPack and explicit archive verification fast paths, and fetch version metadata only when the artifact resolver response lacks usable verification.
