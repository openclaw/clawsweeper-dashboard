## Summary

Keep open for maintainer review: current main still lacks the version-metadata fallback for this ClawHub legacy ZIP resolver shape, and the PR is a narrow supply-chain-safe fix with no actionable review findings found.

## What This Changes

The PR broadens ClawHub artifact resolver typing, avoids trusting summary-shaped legacy ZIP `sha256` as archive integrity, falls back to version metadata for verification, adds regression coverage and a changelog entry, plus a one-line test-support formatting change.

## Best Possible Solution

Land this PR or an equivalent narrow change after normal maintainer review, preserving fail-closed legacy ZIP verification and the regression test for summary-shaped artifact responses.

## Reproduction Assessment

Yes, at source level: current main trusts top-level legacy ZIP `sha256` and does not fetch version metadata after a successful artifact resolver response. The live endpoint has since changed, so the strongest current reproduction is the mocked resolver/version shape covered by the PR test.

## Solution Assessment

Yes. The PR keeps the artifact resolver fast path, falls back only when trusted legacy ZIP verification and ClawPack digests are absent, and avoids widening trust in ambiguous summary `sha256` metadata.
