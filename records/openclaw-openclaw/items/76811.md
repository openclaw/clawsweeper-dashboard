## What This Changes

The branch broadens ClawHub artifact resolver metadata handling, fetches version metadata when legacy ZIP verification is missing, updates regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the PR after maintainer review, keeping version metadata as the fallback verification source for summary-shaped legacy ZIP artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows a summary-shaped legacy ZIP `artifact.sha256` can become archive integrity without consulting the version endpoint, and the PR regression test models that path directly.

## Solution Assessment

Yes. The PR uses the narrowest maintainable fix: preserve explicit archive and ClawPack verification fast paths, and fetch version metadata only when the artifact resolver response lacks usable verification.
