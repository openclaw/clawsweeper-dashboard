## Summary

Keep open: current main still builds the ClawHub install decision from the artifact resolver response without the version-metadata fallback this PR adds, and the PR is not implemented on main. I did not find a functional review blocker, but GitHub currently reports the branch as dirty/unmergeable against the current base.

## What This Changes

The PR widens ClawHub artifact resolver typing, avoids trusting summary legacy ZIP `sha256` as archive integrity, fetches version metadata when verification is missing, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Resolve the current merge conflict/stale base, keep the version-metadata fallback for legacy ZIP verification, and land after exact-head checks remain green.

## Reproduction Assessment

Yes, source-reproducible: current main trusts top-level legacy ZIP `sha256` and does not fetch version metadata after a successful artifact resolver response. Live ZeroAPI now includes `artifactSha256`, so the older missing-verification shape is best covered by the PR's regression mock rather than current live traffic.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it preserves ClawPack and explicit `artifactSha256` paths while only fetching version metadata when the artifact response lacks usable archive verification.
