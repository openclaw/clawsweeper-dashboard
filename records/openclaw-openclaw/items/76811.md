## Summary

Keep open for normal maintainer merge handling: current main can trust ambiguous summary `sha256` for legacy ZIP archive integrity, while this PR narrowly switches that path to version metadata and I found no actionable review or security findings.

## What This Changes

The PR widens ClawHub artifact resolver typing, fetches version metadata when a legacy ZIP resolver response lacks trusted archive verification fields, updates regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the PR after normal maintainer merge handling, keeping version metadata as the fallback verification source for summary-shaped legacy ZIP artifacts.

## Reproduction Assessment

Yes, source-reproducible: current main treats top-level legacy ZIP `sha256` as archive integrity and does not fetch version metadata after a successful artifact resolver response. The live ZeroAPI endpoint now includes `artifactSha256`, so today’s live response no longer proves the historical mismatch by itself.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it preserves ClawPack and explicit archive-verification paths, and only fetches version metadata when the artifact resolver response lacks usable verification.
