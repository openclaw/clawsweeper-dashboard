## Summary

Keep open: current main still lacks the version-metadata fallback for successful ClawHub legacy ZIP resolver responses, and this active PR is a focused fix with regression coverage and no blocking correctness or security finding from this review.

## What This Changes

This PR broadens ClawHub artifact resolver metadata handling and fetches version metadata when a legacy ZIP resolver response lacks archive verification fields.

## Best Possible Solution

Land this fallback, or an equivalent maintainer-owned patch, after syncing with current main and rerunning the focused ClawHub install checks.

## Solution Assessment

Yes. The PR keeps the resolver fast path, avoids trusting summary `sha256` for legacy ZIP archive bytes, and fetches full version metadata only when verification is otherwise missing.
