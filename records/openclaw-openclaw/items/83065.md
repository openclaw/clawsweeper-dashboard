## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/83065. Inbound media prompt notes can expose `media://inbound/<id>`, but structured `read` currently passes that raw value to upstream cwd-relative filesystem resolution. Implement the fix in the actual OpenClaw read execution path, not only in `wrapToolWorkspaceRootGuardWithOptions`; reuse the shared `src/media/media-reference.ts` contract or equivalent shared policy so recognized inbound refs resolve to managed media while malformed/unsupported refs keep the existing error behavior. Preserve workspace-only and sandbox access boundaries. Do not add a new `media_resolve` tool, Signal-only `$MEDIA_PATH` behavior, or arbitrary `exec` command rewriting in this repair. Add focused regression coverage for default host read and any workspace-only/sandbox read behavior touched, update the relevant Signal/media documentation for the supported handle, and add a user-facing changelog fix entry if the branch is prepared for landing.
