## What This Changes

The PR removes the timestamp-presence requirement from trajectory export branch filtering and adds a regression test for timestamp-less legacy v1 sessions.

## Best Possible Solution

Land the narrow exporter fix with the regression test after maintainer approval, preserving the existing owner approval, redaction, and bundle output shape.

## Reproduction Assessment

## Solution Assessment

Yes. Removing the timestamp clause in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization; synthesizing timestamps or adding a second legacy export path would duplicate existing logic.
