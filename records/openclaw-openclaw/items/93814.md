## What This Changes

The PR removes the timestamp requirement from trajectory export session-branch filtering and adds a regression test for legacy v1 session JSONL entries without entry timestamps.

## Best Possible Solution

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable code fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization.
