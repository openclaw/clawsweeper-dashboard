## Summary

Keep open for maintainer review: the patch appears correct and current main plus v2026.6.9 still have the bug, but legacy trajectory exports are sensitive diagnostic bundles and this exact head is paused behind human-review and merge-risk acceptance.

## What This Changes

The PR removes the top-level timestamp requirement from trajectory export branch filtering and adds a regression test for legacy v1 sessions without entry timestamps.

## Best Possible Solution

Land the narrow exporter predicate fix and regression test once maintainers accept that legacy v1 sessions should export redacted transcript events instead of empty bundles.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.6.9 synthesize ids and parentIds for version < 2 entries, then filter timestamp-less canonical entries out before buildTranscriptEvents can normalize them.

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization; synthesizing fake timestamps during migration would add a second owner for the same fallback.
