## Summary

Keep open: current main still drops timestamp-less legacy v1 transcript entries during trajectory export, and this PR is a narrow, correct fix with sufficient copied runtime proof. The remaining blocker is maintainer acceptance that legacy diagnostic bundles should change from empty transcript output to redacted transcript-bearing output.

## What This Changes

The PR removes the trajectory export session-branch timestamp filter and adds regression coverage for legacy v1 session JSONL entries that have ids but no entry timestamps.

## Best Possible Solution

Land the narrow exporter fix and regression test after a maintainer explicitly accepts that legacy v1 trajectory exports should include redacted transcript events.

## Reproduction Assessment

Yes from source inspection: a v1 session is migrated with ids and parentIds but no entry timestamps, then current main filters those entries out before `buildTranscriptEvents` can apply its timestamp fallback. The PR body also supplies before/after copied output showing `transcriptEventCount` changing from 0 to 2.

## Solution Assessment

Yes. Removing the timestamp predicate in `readSessionBranch` is the narrowest maintainable fix because branch selection needs canonical entry identity, while `buildTranscriptEvents` already owns timestamp normalization.
