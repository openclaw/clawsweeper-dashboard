## Summary

Keep open for maintainer review: the patch is a narrow and correct fix for legacy v1 trajectory exports, but it changes old support bundles from empty transcripts to redacted transcript-bearing bundles, so automerge should wait for explicit maintainer acceptance of that sensitive diagnostic-content change.

## What This Changes

The PR removes the top-level timestamp requirement from trajectory session branch filtering and adds a regression test for legacy v1 sessions whose entries lack timestamps.

## Best Possible Solution

Land the narrow exporter fix and regression test after a maintainer explicitly accepts legacy transcript-bearing trajectory exports, or ask for one historical CLI proof run first if that acceptance needs more evidence.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 synthesize ids and parentIds for version < 2 entries but then filter timestamp-less entries before buildTranscriptEvents can normalize them; the PR body also provides before/after copied output changing transcriptEventCount from 0 to 2.

## Solution Assessment

Yes for the code fix. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable repair because branch selection needs canonical entry identity, while timestamp normalization and redaction already happen downstream.
