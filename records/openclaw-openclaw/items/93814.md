## What This Changes

The PR removes the top-level timestamp requirement from trajectory session branch filtering and adds a regression test for legacy v1 session exports without entry timestamps.

## Best Possible Solution

Land the narrow exporter fix and regression test after maintainers accept that legacy v1 trajectory bundles should include redacted transcript events.

## Reproduction Assessment

Yes. Current main and v2026.6.9 synthesize ids/parentIds for version < 2 entries but then filter out timestamp-less entries before `buildTranscriptEvents` can normalize their timestamps; the PR body also supplies copied before/after output.

## Solution Assessment

Yes. Removing the timestamp predicate in `readSessionBranch` is the narrowest maintainable fix because branch selection needs canonical entry identity, while timestamp normalization and export redaction already happen downstream.
