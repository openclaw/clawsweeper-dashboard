## Summary

Keep open for normal review and merge: current main still drops timestamp-less legacy v1 transcript entries, and the PR’s narrow exporter change matches the existing legacy migration and timestamp normalization contracts.

## What This Changes

This PR removes the timestamp requirement from trajectory session branch filtering and adds a regression test for exporting timestamp-less legacy v1 session transcripts.

## Best Possible Solution

Land this narrow exporter fix with the regression test, keeping trajectory redaction and output shape otherwise unchanged.

## Reproduction Assessment

Yes, at source level: current main migrates v1 entries with ids and parentIds but no timestamps, then filters them out before the timestamp normalizer can run. I did not run tests because this review is read-only.

## Solution Assessment

Yes, this is the best narrow fix: the branch reader should select canonical entries by identity, while buildTranscriptEvents already owns timestamp normalization. Synthesizing timestamps in the legacy migration would add a second compatibility path for data the exporter already handles.
