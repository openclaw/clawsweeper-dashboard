The PR removes the timestamp-presence requirement from trajectory branch entry filtering and adds a legacy v1 export regression test.

## Solution Assessment

Yes. Removing the redundant timestamp filter at readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization.
