## Summary

Keep open: this PR is still necessary and appears correct because current main drops timestamp-less legacy v1 transcript entries before the existing timestamp normalizer can handle them, while the patch removes only that redundant gate and adds focused regression coverage.

## What This Changes

The PR removes the timestamp requirement from trajectory branch filtering and adds a regression test for timestamp-less legacy v1 session exports.

## Best Possible Solution

Land this narrow exporter fix with the regression test, keeping trajectory redaction and bundle schema unchanged.

## Reproduction Assessment

Yes. Current main migrates version 1 entries with ids and parentId links but no entry timestamps, then filters those entries out before buildTranscriptEvents can default missing timestamps to the epoch; the PR body also includes before/after runtime output for that path.

## Solution Assessment

Yes. The exporter branch reader should select canonical entries by identity and branch structure, while the existing event builder owns timestamp normalization; synthesizing timestamps in legacy migration would add a second compatibility path for data the exporter already handles.
