## Summary

Keep open: current main and the latest release still require a transcript entry timestamp before export, while this PR narrowly removes that redundant filter for legacy v1 sessions and adds focused regression coverage with no blocking review findings.

## What This Changes

The PR removes the timestamp presence check from trajectory session-branch filtering and adds a regression test for exporting timestamp-less legacy v1 session transcripts.

## Best Possible Solution

Land the narrow exporter fix with the regression test, keeping existing redaction, owner approval, and bundle output shape unchanged.

## Reproduction Assessment

Yes, from source inspection: current main migrates v1 entries without timestamps and then filters those entries out before transcript event construction. The PR body also provides before/after live output showing transcriptEventCount moving from 0 to 2 for the same timestamp-less v1 input.

## Solution Assessment

Yes. Removing the redundant timestamp filter at readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization.
