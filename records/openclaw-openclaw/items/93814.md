## Summary

Keep open for maintainer handling: the patch is a narrow, source-backed bug fix, but the armed automerge is already paused because legacy v1 trajectory exports would change from empty bundles to redacted transcript-bearing diagnostic bundles.

## What This Changes

The PR removes the top-level timestamp requirement from trajectory session branch filtering and adds a regression test for legacy v1 exports without entry timestamps.

## Best Possible Solution

Land the narrow exporter fix and regression test once maintainers accept that legacy v1 trajectory bundles should include redacted transcript events.

## Reproduction Assessment

Yes. Current main and v2026.6.9 synthesize ids and parentIds for version < 2 entries but then filter out timestamp-less entries before buildTranscriptEvents can normalize their timestamps; the PR body also supplies before/after copied output showing transcriptEventCount changing from 0 to 2.

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization and redaction remains on the existing export path.
