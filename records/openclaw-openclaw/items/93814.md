## Summary

Keep open: current main still filters timestamp-less legacy v1 session entries before trajectory event building, and this PR is a focused corrective patch with no blocking code findings. The remaining blocker is maintainer acceptance that legacy trajectory bundles should change from empty output to redacted transcript-bearing diagnostic output.

## What This Changes

The PR removes the top-level timestamp requirement from trajectory session branch export and adds a regression test for timestamp-less legacy v1 session JSONL.

## Best Possible Solution

Land the narrow exporter fix and regression test after a maintainer accepts the sensitive diagnostic-bundle behavior and existing redaction boundary.

## Reproduction Assessment

Yes. A version 1 session JSONL with timestamp-less message entries follows the current-main path that synthesizes ids and parentIds, then filters those entries out before buildTranscriptEvents can normalize timestamps; the PR body supplies before/after output for that path.

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization.
