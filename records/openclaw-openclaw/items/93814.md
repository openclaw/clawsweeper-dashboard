## What This Changes

The PR drops the timestamp-presence filter in trajectory branch export and adds a regression test for timestamp-less legacy v1 session JSONL files.

## Best Possible Solution

Land the narrow exporter fix and regression test after a maintainer accepts the sensitive diagnostic-bundle behavior and existing redaction boundary.

## Reproduction Assessment

Yes. A version 1 session JSONL with timestamp-less message entries follows the current-main path that synthesizes ids and parentIds, then filters those entries out before buildTranscriptEvents can normalize timestamps; the PR body also supplies before/after output for that path.

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization; synthesizing timestamps or adding a second legacy export path would duplicate that contract.
