## What This Changes

The PR removes the timestamp requirement from trajectory session branch filtering and adds a legacy v1 regression test for timestamp-less session exports.

## Best Possible Solution

Land the narrow exporter fix and regression test once maintainers accept that legacy v1 trajectory bundles should include redacted transcript events.

## Reproduction Assessment

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization and the existing redaction path remains unchanged.
