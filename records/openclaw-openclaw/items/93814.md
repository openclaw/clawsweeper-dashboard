Keep open: the PR is a focused bug fix for a real legacy trajectory export regression that is still present on current main and the latest release. The patch has sufficient before/after proof and no blocking findings, but it is already paused for maintainer approval because trajectory bundles are sensitive export artifacts.

## What This Changes

The PR removes the timestamp requirement from trajectory session-branch filtering and adds a regression test for timestamp-less legacy v1 session exports.

## Best Possible Solution

Land the narrow exporter fix with the regression test after maintainer approval, keeping the existing approval, redaction, and bundle output boundaries unchanged.

## Reproduction Assessment

Yes. Current main migrates v1 entries without timestamps and then filters them out before transcript event construction; the PR body also provides before/after live output showing transcriptEventCount changing from 0 to 2.

## Solution Assessment

Yes. Removing the timestamp gate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization; synthesizing timestamps during migration would add another compatibility path.
