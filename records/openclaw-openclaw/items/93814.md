## What This Changes

The PR removes the timestamp requirement from trajectory export branch filtering and adds a regression test for timestamp-less legacy v1 sessions.

## Best Possible Solution

Land this narrow exporter fix with the regression test, keeping redaction, output shape, and command behavior unchanged.

## Reproduction Assessment

Yes, source inspection shows current main migrates v1 entries with ids and parentIds but no timestamps, then filters them out before timestamp normalization. The PR body also includes focused before/after live output for the exporter path.

## Solution Assessment

Yes, this is the best narrow fix because branch selection needs canonical type plus identity, while `buildTranscriptEvents` already owns timestamp normalization. Synthesizing timestamps in migration would add a second compatibility path for data the exporter already handles.
