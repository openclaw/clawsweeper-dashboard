## Summary

## What This Changes

The PR removes the timestamp requirement from trajectory export branch filtering and adds a legacy v1 regression test for timestamp-less sessions.

## Best Possible Solution

Land the narrow exporter fix and regression test once maintainers accept that legacy v1 trajectory bundles should include redacted transcript events.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 synthesize ids and parentIds for version < 2 entries but then filter out timestamp-less entries; the PR body also includes before/after copied output showing transcriptEventCount changing from 0 to 2.

## Solution Assessment

Yes. Removing the timestamp predicate in readSessionBranch is the narrowest maintainable fix because branch selection needs canonical entry identity, while buildTranscriptEvents already owns timestamp normalization and the export path already owns redaction.
