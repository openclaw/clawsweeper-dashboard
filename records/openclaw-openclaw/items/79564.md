## Summary

Keep this PR open: the linked bug is valid and after-fix proof is now supplied, but the patch still needs a caller-intent fix for bare aliases and the required changelog entry before merge.

## What This Changes

The PR changes agent model fallback resolution to skip cross-provider bare-alias matches when a provider is supplied and adds regression tests for the DeepSeek alias collision plus bare-alias behavior.

## Best Possible Solution

Land a narrow fix that honors explicit provider-qualified selections ahead of cross-provider aliases while preserving configured bare-alias selection for callers that did not supply a provider, with regression coverage and release notes.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction through `resolveFallbackCandidates`, and the linked report includes live 2026.5.7 output showing `opencode-go/deepseek-v4-pro` executing as OpenRouter when both refs are configured.

## Solution Assessment

No. The proposed direction is right for explicit provider refs, but the implementation uses the normalized provider argument as a proxy for user intent, which can break bare-alias paths; carry explicit-provider intent from the caller or resolve the raw alias before splitting.
