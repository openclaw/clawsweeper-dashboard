## Summary

Keep open: the reported source-level bug is real, but the patch currently rewrites all ACP-shaped session keys as `acpx` and would mislabel `openclaw acp` bridge sessions; it also still needs after-fix real behavior proof and a changelog entry.

## What This Changes

This PR adds a key-based ACP model/provider overlay to `openclaw sessions` output and a new sessions command test suite for ACP-keyed rows.

## Best Possible Solution

Use persisted ACP runtime metadata to display an acpx sentinel only for acpx-managed child sessions, preserve actual configured/runtime model display for ACP bridge sessions, then add focused tests, changelog, and redacted after-fix terminal proof.

## Reproduction Assessment

Yes at source level: current main’s sessions CLI emits `resolveSessionDisplayModelRef` output for ACP-keyed rows, and that resolver falls back to the configured agent model when the row has no stored runtime model. I did not run a live ACP/Copilot container in this read-only review.

## Solution Assessment

No: the intended overlay is narrow, but key-shape-only detection is too broad because `openclaw acp` bridge sessions also use ACP-shaped keys. The maintainable fix should distinguish acpx child runtime rows using persisted ACP metadata such as `entry.acp.backend` and `entry.acp.agent`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate the overlay on ACP runtime metadata:** `src/commands/sessions.ts:68-72`
  - body: This helper treats every ACP-shaped key as an acpx child runtime. `openclaw acp` bridge sessions also use `acp:` / `agent:<agent>:acp:*` Gateway session keys, but those turns use the normal OpenClaw Gateway model and do not necessarily have `SessionEntry.acp` runtime metadata. After this patch, their CLI JSON/table rows would be rewritten to `acpx` and `<agent>-acp`, hiding the actual model; please gate the sentinel on persisted ACP runtime metadata instead of key shape alone and add a bridge-session regression case.
  - confidence: 0.86
