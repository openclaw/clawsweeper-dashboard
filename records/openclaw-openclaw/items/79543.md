## Summary

Keep open: this is a plausible focused bug fix for misleading ACP session model output, but the PR still needs after-fix real behavior proof and the submitted diff has correctness issues around backend labeling and the changelog reference.

## What This Changes

The PR overlays ACP control-plane session rows in `openclaw sessions` output with a `<agentId>-acp`/`acpx` model sentinel, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the `entry.acp` gate, thread the stored ACP backend into the model-provider overlay, fix the catalog reference, then land after redacted after-fix ACP session output and normal CI/rebase validation.

## Reproduction Assessment

Yes at source level. Current main sends ACP-keyed rows through `resolveSessionDisplayModelRef`, and that resolver falls back to the configured agent model when no stored model fields exist; the contributor also supplied before-fix live output, but I did not live-run ACP here.

## Solution Assessment

No as submitted. The `entry.acp`-gated call-site overlay is the right narrow direction, but it should use the persisted ACP backend instead of hard-coding `acpx`, and the changelog must not link the unrelated public issue.

## Review Findings
