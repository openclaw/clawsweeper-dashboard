## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add an approved reset recovery before failing startup:** `src/agents/workspace.ts:597-598`
  - body: This branch throws `WorkspaceVanishedError` whenever a recently attested workspace path looks brand new. `ensureAgentWorkspace` is used by agent startup, auto-reply, cron, setup, and Gateway agent updates, so an intentional delete/recreate reset can block those flows for 24 hours unless the user discovers and removes a hidden sibling marker. Please add an owner-approved recovery path such as `openclaw doctor --fix` or a setup/reset flow, or preserve the compatible default and make strict refusal explicit.
  - confidence: 0.91
- **[P1] Write the attestation marker without following symlinks:** `src/agents/workspace.ts:369`
  - body: The new marker is a sibling of the workspace and is written with `fs.writeFile`, while freshness is checked with `fs.stat`; both follow symlinks. For custom workspace paths under a shared or writable parent, another process can point `<workspace>.attested` at an unrelated file and have OpenClaw read or overwrite it during startup. Store the marker in an OpenClaw-owned state directory or use no-follow/exclusive safe file handling.
  - confidence: 0.88
