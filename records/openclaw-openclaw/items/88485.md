  - body: When a user intentionally deletes or recreates a workspace inside the 24-hour marker window, every startup caller can now throw `WORKSPACE_VANISHED` until an out-of-band marker is removed.   - body: `attestationPath` is a sibling of the workspace path, and both `fs.stat` and this `fs.writeFile` follow symlinks. For a custom workspace under a shared writable parent, another local user can pre-create `workspace.attested` as a symlink to a writable victim file, causing OpenClaw to clobber it or trust a false freshness signal; use symlink-safe creation/refresh or store the marker under an owned state directory.
  - confidence: 0.91
