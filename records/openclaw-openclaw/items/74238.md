## Summary

Keep open. Current main still has duplicate Web UI page context, so the cleanup target is real, but this PR is not merge-ready because it removes dynamic/session-specific and section-specific labels while leaving empty header wrappers.

## What This Changes

The PR moves Web UI tab subtitles into the dashboard breadcrumb and removes many local page/card header blocks across Control UI views.

## Best Possible Solution

Land a narrower Control UI cleanup that removes only duplicate page-level context while preserving dynamic paths, form modes, subsection labels, and required release-note coverage.

## Reproduction Assessment

Yes, by source inspection. Current main renders shared dashboard context plus local content/page headers for ordinary tabs, and the PR diff shows the introduced regressions without needing a live browser run.

## Solution Assessment

No as-is. Breadcrumb consolidation is a reasonable direction, but the patch should preserve dynamic and section-specific context outside the breadcrumb and omit empty header containers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: This removes the only rendering of `props.result.path`. The breadcrumb subtitle is static, so loaded Sessions pages would no longer show the actual session store location; render that dynamic path elsewhere instead of dropping it with the duplicate title block.
  - confidence: 0.93

  - body: This replaces the New/Edit Job title and subtitle with an empty div. The page breadcrumb only identifies the Cron tab, so users lose the form-mode context; keep a section label or move it to a non-duplicate form header.
  - confidence: 0.86
- **[P3] Skip empty content headers on ordinary tabs:** `ui/src/ui/app-render.ts:1512-1515`
  - body: For non-chat, non-dreams tabs without a global error, this still renders `.content-header` with empty left/meta containers. Because the section has layout styling, the duplicate header becomes blank vertical space instead of disappearing; only render it when it has controls or an error.
  - confidence: 0.86
- **[P3] Remove the empty Manual RPC row:** `ui/src/ui/views/debug.ts:71-74`
  - body: The Manual RPC title/subtitle is replaced by a row containing two empty divs. That leaves unnecessary blank DOM above the form; drop the wrapper or keep a meaningful section label.
  - confidence: 0.9

  - body: This is a user-visible Control UI presentation change, but the PR does not update `CHANGELOG.md`. Add a single-line entry so release notes capture the visible UI cleanup.
  - confidence: 0.88
