## Summary

Keep open. The duplicate-header cleanup target is real on current main, but this PR is not merge-ready because it removes dynamic and section-specific UI context while leaving blank header structures behind.

## What This Changes

The PR moves Control UI tab subtitles into the shared dashboard breadcrumb and removes local page/card header blocks across several Control UI views, with small CSS and test updates.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that keeps only one tab-level header source while preserving dynamic store paths, form-mode labels, section labels, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared topbar breadcrumb plus non-config `.content-header` title/subtitle blocks, and the PR diff gives a clear source-level path for the introduced regressions.

## Solution Assessment

No as-is. Consolidating duplicate tab-level context into the breadcrumb is a reasonable direction, but the patch should remove only duplicate page-level headings and preserve dynamic or section-specific context elsewhere.
