## Summary

Keep open: this PR addresses a real duplicate-header bug and has acceptable screenshot proof, but the branch still loses dynamic Sessions and Cron context and leaves empty header wrappers that need repair before merge.

## What This Changes

The PR moves Control UI page subtitles into the dashboard breadcrumb and removes local page/card header blocks across Web UI views while keeping Chat controls.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that removes duplicated page-level context while preserving dynamic Sessions store path, Cron form context, section labels, and current dashboard breadcrumb behavior.

## Reproduction Assessment

Yes. Current main source renders both shared tab page context and local Usage/Sessions headers, and the PR screenshots show the visual before/after target without needing backend setup.

## Solution Assessment

No as-is. Consolidating duplicated page context is the right direction, but the patch should preserve dynamic and section-specific context and avoid replacing removed headers with empty layout rows.
