## Summary

Keep open. The duplicate-header cleanup is a valid Control UI direction, but the PR is not merge-ready because it removes the dynamic Sessions store path, leaves blank header scaffolding in the rendered UI, and is missing the required user-facing changelog entry.

## What This Changes

The PR updates the Control UI breadcrumb to include tab subtitles, removes local title/subtitle blocks across several views, adjusts Config pending-change presentation, adds breadcrumb subtitle CSS, and updates one cron view test.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that moves static tab context into the breadcrumb, removes only truly duplicate page headers, preserves dynamic per-page details and meaningful section labels, handles Config diff/status changes intentionally, and adds focused render/mobile coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main reproduces the duplicate-header condition, and the provided PR diff shows the introduced regressions without needing a live Gateway.

## Solution Assessment

No as-is. The consolidation approach is maintainable, but the current patch over-removes useful dynamic and section context; the safer fix is a focused cleanup that keeps Sessions store visibility and avoids empty layout placeholders.
