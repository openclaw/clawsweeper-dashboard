## Summary

Keep open. Current main still has the duplicate Control UI page-header pattern this PR targets, but the PR should not merge as-is because it removes dynamic/section-specific UI context and leaves blank header structure behind.

## What This Changes

The PR removes local Control UI page/card header text across several views and moves tab subtitles into the shared dashboard breadcrumb.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that moves duplicated tab-level context into the breadcrumb while preserving dynamic paths, form modes, and subsection labels, then record the user-visible change in the changelog.

## Reproduction Assessment

Yes. Source inspection of current main shows both the shared dashboard breadcrumb and the non-config `.content-header` title/subtitle, and the PR patch shows the introduced Sessions path and blank-header regressions.

## Solution Assessment

No as-is. The consolidation direction is reasonable, but the patch should remove only duplicate page-level headings and keep dynamic or section-specific context outside the breadcrumb.
