## Summary

Keep open: the duplicate Control UI header problem is real and the branch has useful direction, but it still drops dynamic/section-specific context and leaves blank header scaffolding that should be repaired before merge.

## What This Changes

The PR moves tab subtitles into the dashboard breadcrumb and removes local page/card header text across multiple Control UI views, while keeping Chat controls in the content header.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that removes duplicated page-level context while preserving dynamic paths, form-mode labels, section labels, and necessary Chat/Dreams controls.

## Reproduction Assessment

Yes. Current main source shows the shared shell header plus duplicate non-config/content-view headers, and the PR screenshots show the before/after visual target without needing backend setup.

## Solution Assessment

No as-is. The consolidation direction is sound, but the patch should preserve dynamic Sessions store context and Cron form mode context, and omit empty header wrappers instead of leaving blank layout rows.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.86

  - confidence: 0.9
