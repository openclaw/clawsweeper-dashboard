## Summary

Keep open. Current main still has the source-proven hook context mutation bug, and this PR is a narrow, clean fix with regression coverage, changelog entry, no blocking review findings, and clean remote checks.

## What This Changes

The PR updates plugin hook registration to attach `pluginConfig` to the original hook context with restoration, adds loader regression coverage for `agent:bootstrap` context mutation, and adds a changelog entry.

## Best Possible Solution

Land this narrow registry-wrapper fix after maintainer review, preserving mutable hook propagation and scoped `pluginConfig` cleanup with the added regression coverage.

## Reproduction Assessment

Yes. Current main’s plugin wrapper clones `evt.context`, while `applyBootstrapHookOverrides()` reads the original `event.context.bootstrapFiles` after hook dispatch, so a plugin assignment to `bootstrapFiles` is dropped by source inspection.

## Solution Assessment

Yes. Fixing the generic `registerHook` wrapper is the narrowest maintainable solution; special-casing bootstrap would leave the hook contract broken elsewhere, and a return-value hook API would be a larger compatibility change.
