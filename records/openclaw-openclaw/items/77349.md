## What This Changes

The branch changes the plugin registry hook wrapper to preserve the original event context while temporarily scoping `pluginConfig`, adds a loader regression test, and updates the changelog.

## Best Possible Solution

Resolve mergeability and land this PR or an equivalent narrow registry wrapper fix that preserves mutable hook context while restoring or deleting per-handler `pluginConfig` after each plugin hook call.

## Reproduction Assessment

Yes, at source level. A plugin loaded through `api.registerHook('agent:bootstrap')` can assign `event.context.bootstrapFiles`, but current main passes a cloned context while bootstrap resolution later reads the original event context.

## Solution Assessment

Yes. Passing the original context through the generic plugin wrapper and restoring or deleting `pluginConfig` in `finally` is the narrow compatible fix for the documented mutable hook contract.
