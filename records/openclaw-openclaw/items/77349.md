## Summary

Keep open. The PR addresses a source-reproducible plugin hook bug with the narrow wrapper-level fix the related issue requested, but the branch is currently marked dirty/unmergeable and checks were still pending at review time.

## What This Changes

The PR changes the plugin hook wrapper to preserve the original hook context while temporarily injecting `pluginConfig`, adds a loader regression test, and adds a changelog fix entry.

## Best Possible Solution

Land a rebased version of this narrow wrapper fix after checks pass, preserving `pluginConfig` cleanup/restoration and the regression coverage for plugin-loaded `agent:bootstrap` mutations.

## Reproduction Assessment

Yes. Current main’s plugin wrapper clones `event.context`, while `applyBootstrapHookOverrides()` reads the original event context after `triggerInternalHook()`, so a plugin-loaded `agent:bootstrap` assignment is lost by source inspection.

## Solution Assessment

Yes. Fixing the `registerHook` wrapper is the narrowest maintainable solution because special-casing bootstrap would leave the generic mutable hook contract broken, and a return-value hook API would be a larger compatibility change.
