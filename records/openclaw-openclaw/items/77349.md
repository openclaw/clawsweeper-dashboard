## Summary

Keep open. Current main still has the broken hook-wrapper behavior, and this PR is a focused, source-backed fix with sufficient real behavior proof and no blocking review findings.

## What This Changes

The branch changes the plugin registry hook wrapper to pass the original event context with scoped `pluginConfig` cleanup, adds a loader regression test, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow registry wrapper fix so mutable hook context survives while `pluginConfig` remains scoped to each active handler.

## Reproduction Assessment

Yes, at source level. A plugin loaded through `api.registerHook('agent:bootstrap')` can assign `event.context.bootstrapFiles`, but current main passes a cloned context while bootstrap resolution later reads the original event context.

## Solution Assessment

Yes. Reusing the original context and restoring or deleting `pluginConfig` in `finally` is the narrow compatible fix for the documented mutable hook contract; an immutable return-value API would be a larger SDK change.
