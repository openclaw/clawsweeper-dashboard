## Summary

Keep open. Current main still has the cloned plugin hook context that drops assignment-style mutations, and this active PR is the focused fix with sufficient real behavior proof; it is not closeable cleanup, though live metadata now shows the branch is conflicting against main.

## What This Changes

The PR changes the plugin `registerHook` wrapper to preserve the original event context with scoped `pluginConfig` cleanup, adds a loaded-plugin `agent:bootstrap` regression test, and adds a changelog entry.

## Best Possible Solution

Rebase and land this PR or an equivalent narrow registry-wrapper fix so mutable hook context survives while `pluginConfig` remains scoped per active handler.

## Reproduction Assessment

Yes, at source level. A loaded plugin using `api.registerHook('agent:bootstrap')` can assign `event.context.bootstrapFiles`, but current main passes a cloned context while bootstrap resolution later reads the original event context.

## Solution Assessment

Yes. Passing the original context with `finally` cleanup for `pluginConfig` is the narrow compatible fix for the existing mutable hook contract; a return-value hook API or bootstrap-only special case would be broader than needed.
