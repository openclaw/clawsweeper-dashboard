## What This Changes

The PR updates plugin hook registration to inject `pluginConfig` on the original hook context with cleanup, adds a loader regression test for `agent:bootstrap` mutations, and records a changelog fix entry.

## Best Possible Solution

Land this narrow wrapper fix after maintainer review, preserving generic mutable hook propagation plus scoped `pluginConfig` cleanup and the regression coverage.

## Reproduction Assessment

Yes. Current main’s plugin wrapper clones `event.context`, while `applyBootstrapHookOverrides()` reads the original event context after `triggerInternalHook()`, so a plugin-loaded `agent:bootstrap` assignment is lost by source inspection.

## Solution Assessment

Yes. Fixing the generic `registerHook` wrapper is the narrowest maintainable solution; special-casing bootstrap would leave the hook contract broken elsewhere, and a return-value hook API would be a larger compatibility change.
