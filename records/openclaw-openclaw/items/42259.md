## Summary

Keep open: current main still lacks configurable image-tool compression, but this PR has bound-violation bugs, lacks real behavior proof, and needs maintainer agreement on the public config contract before merge.

## What This Changes

The PR adds two `agents.defaults` image compression scalar settings, threads `minQuality` and `maxSide` through image-tool media loading and JPEG optimization, adds tests, and also changes README, provider-doc, doctor, and gateway recovery copy.

## Best Possible Solution

Land a focused compression configuration feature after maintainers settle the config shape, the optimizer preserves configured bounds in every fallback, and real image-tool proof shows the document-quality use case.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows out-of-ladder `minQuality` and `maxSide` values can empty the candidate lists and fall back to `40` or `800`. The original OCR/detail-loss motivation is not reproduced with a real image-tool run.

## Solution Assessment

No. The feature direction may be useful, but this branch needs an agreed config contract, bound-preserving fallback candidates, focused tests/docs, and real behavior proof before it is the maintainable fix.
