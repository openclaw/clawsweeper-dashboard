## Summary

Keep open: current main still lacks configurable image-tool compression, and this PR remains the canonical implementation thread, but the branch has blocking bound-violation bugs and no real behavior proof.

## What This Changes

The PR adds two optional `agents.defaults` image compression knobs, wires them into image-tool media loading/JPEG optimization, adds constraint tests, and includes several unrelated docs/gateway copy edits.

## Best Possible Solution

Settle the public compression config contract, implement bound-preserving optimizer options in the current `src/media/web-media.ts` and image-tool path, add out-of-ladder regression tests and docs, and require redacted real image-tool proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR diff shows out-of-ladder `minQuality` and `maxSide` values can empty candidate lists and fall back to `40` or `800`. No high-confidence real run proves the original OCR/detail-loss improvement.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best fix until the config contract is agreed, fallback candidates preserve configured bounds, and real image-tool behavior is demonstrated.
