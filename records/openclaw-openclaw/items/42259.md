## Summary

Keep open: current main still lacks image-tool compression config, while this PR has source-reproducible bound-violation bugs, missing real behavior proof, a dirty/stale branch, and a public config contract that needs maintainer judgment.

## What This Changes

The branch adds two `agents.defaults` image compression config fields, threads `minQuality`/`maxSide` through image-tool media loading, adds tests, and includes unrelated README, provider-docs, doctor, and gateway recovery-copy edits.

## Best Possible Solution

Land a focused image-tool compression configuration feature only after maintainers settle the config shape, the optimizer preserves configured bounds in all fallbacks, and redacted real image-tool proof demonstrates the document-quality use case.

## Reproduction Assessment

Yes for the PR defects: source inspection of the diff shows out-of-ladder `minQuality` and `maxSide` values can empty the candidate lists and then fall back to `40` or `800`. The original document OCR improvement is not reproduced with a real image-tool run.

## Solution Assessment

No. The feature direction may be useful, but this branch should be reworked around an agreed config shape, bound-preserving fallback candidates, aligned tests/docs, and a focused diff before merge.
