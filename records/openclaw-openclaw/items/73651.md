## Best Possible Solution

Land a narrow Feishu outbound fallback that catches only the known CardKit table-count rejection, resends the same content through the existing plain-text path with reply/thread metadata preserved, and leaves reply-dispatcher or streaming-card scope to the related follow-up work.

## Solution Assessment
