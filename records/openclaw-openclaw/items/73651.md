## What This Changes

The PR adds a Feishu CardKit table-limit error parser, wraps outbound markdown and structured card sends with table-limit-only plain-text fallback, preserves thread metadata on fallback/media paths, and adds parser/outbound tests.

## Best Possible Solution

Land a narrow Feishu outbound fallback that catches only the known CardKit table-count rejection, resends the same content through the existing plain-text Feishu path with reply/thread metadata preserved, and keep reply-dispatcher/streaming-card coverage coordinated through the related open follow-up work.

## Solution Assessment

Yes for this PR's outbound scope. The diff reuses the existing sendMessageFeishu plain-text path, narrows fallback to the table-limit classifier, and rethrows other card failures rather than masking unrelated Feishu errors.
