## Summary

Keep open: current main has the 2026.5.4 SDK helper, but the reported compatibility gap still appears real because Discord advertises a 2026.5.4 plugin API requirement while the npm/local install and non-bundled load paths only enforce the older minHostVersion floor. The invalid visibleReplies portion is already covered by current config validation.

## What This Changes

The issue asks OpenClaw to prevent or gracefully handle Discord plugins installed ahead of the core runtime and to reject invalid visibleReplies values.

## Best Possible Solution

Add generic plugin API compatibility enforcement for non-ClawHub install/load paths and align Discord package/catalog compatibility metadata, while keeping the existing visibleReplies schema behavior.

## Reproduction Assessment

No, not as a live current-main run: this read-only review did not run a 2026.5.3 core with a 2026.5.4 Discord plugin. Source inspection does show the reported mismatch path and the missing non-ClawHub pluginApi enforcement.

## Solution Assessment

Yes: failing closed before installing or loading an incompatible plugin is the narrow maintainable fix. A Discord-only runtime fallback would hide a version contract failure and would not protect other plugins.
