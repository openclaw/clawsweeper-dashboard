## Summary

Keep open: the original Discord helper/export mismatch is no longer the only signal, but current main still lacks the generic non-ClawHub plugin API compatibility gate that would prevent newer external plugins from loading on older cores.

## What This Changes

The issue asks OpenClaw to prevent plugin/core SDK version skew from causing runtime delivery crashes and to reject invalid visible reply config earlier.

## Best Possible Solution

Add a shared plugin API compatibility gate for npm/local install and non-bundled manifest-registry loading, using the existing package metadata and range helper while leaving ClawHub enforcement and visibleReplies validation intact.

## Reproduction Assessment

Yes, by source inspection: a non-bundled plugin whose minHostVersion is satisfied but whose openclaw.compat.pluginApi exceeds the host can still pass npm/local install and registry discovery before runtime load.

## Solution Assessment

Yes. The narrow maintainable fix is a generic plugin API range check in the install and runtime discovery paths, not a Discord-specific fallback that leaves other plugins exposed.
