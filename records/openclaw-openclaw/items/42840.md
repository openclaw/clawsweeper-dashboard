## Summary

Keep open. Current main and the latest release still lack LaTeX/MathJax/KaTeX rendering in the Control UI markdown path, this issue is the canonical tracker for the request, and https://github.com/openclaw/openclaw/pull/67103 remains open and unmerged rather than proving the feature is implemented.

## What This Changes

The issue requests MathJax/LaTeX formula rendering in the Control UI, preferably with an openclaw.json configuration option and inline/display delimiter support.

## Best Possible Solution

Land or replace a maintainer-approved shared Control UI markdown renderer implementation that safely supports inline and display math while preserving cache, link hardening, markdown compatibility, invalid-LaTeX fallback, currency prose, and code-span/code-fence behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a bug repro, but yes as source-level current behavior: chat flows through toSanitizedMarkdownHtml, and that renderer plus package metadata have no math rendering stage or dependency.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact MathJax/config proposal; the shared Control UI markdown renderer is the right surface, but the dependency, default behavior, delimiter policy, and sanitizer strategy need maintainer judgment.
