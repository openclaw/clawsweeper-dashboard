## Summary

Keep open. Current main still lacks MathJax/KaTeX rendering in the Control UI markdown path, and open PR #67103 is an implementation candidate rather than a merged fix or a reason to queue duplicate automation.

## What This Changes

Add MathJax/LaTeX rendering support to Control UI chat so inline and display formulas render as equations instead of raw source text.

## Best Possible Solution

Implement a sanitizer-aware math-rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; close after PR #67103 or an equivalent implementation lands.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a chat message containing `$E=mc^2$` reaches `toSanitizedMarkdownHtml`, which currently runs markdown-it and DOMPurify without any MathJax/KaTeX processing stage.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable path is a shared frontend renderer change with sanitizer-aware tests, while maintainers decide whether to land, revise, or replace PR #67103.
