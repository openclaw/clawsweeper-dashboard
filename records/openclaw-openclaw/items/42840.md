## Summary

Current main and the latest release still lack math rendering, while two open but conflicting implementation PRs cover the feature; keep this issue as the canonical product thread until maintainers select a secure renderer design. Likely related people: @kun9053 for recent renderer work, plus @ImortalMask and @zhangdoudou for focused KaTeX implementation history.

## What This Changes

Requests rendered LaTeX mathematics in Control UI chat messages instead of displaying raw formula markup.

## Best Possible Solution

Keep one canonical issue and land the smallest secure implementation against the current markdown-it renderer, with a maintainer-approved delimiter policy and regression coverage for streaming, code spans and fences, currency prose, malformed input, CJK boundaries, and inline and display math.

## Reproduction Assessment

Yes. Source inspection shows messages containing `$...$` or `$$...$$` pass through the shared markdown-it renderer, which has no math parser, renderer dependency, or delimiter rule on current main or the latest release.

## Solution Assessment

No, the configuration-first proposal should not be accepted as the default design. The narrowest maintainable solution is renderer-local support with explicit sanitizer, delimiter, and streaming contracts; configuration should be added only if maintainers establish a concrete compatibility need.
