Keep open: current main and the latest release still route Control UI chat through the shared markdown-it/DOMPurify renderer with no math-rendering layer, and the active implementation attempts need maintainer decisions on renderer choice, config/default behavior, sanitizer scope, and dependency review.

## What This Changes

The issue requests MathJax/LaTeX-style formula rendering in the Control UI chat so dollar-delimited equations render as math instead of raw source text.

## Best Possible Solution

Keep this issue as the canonical tracker and land a maintainer-approved Control UI markdown/chat renderer change after deciding the renderer, default/config behavior, sanitizer boundary, dependency policy, and browser proof requirements.

## Reproduction Assessment

Yes at source level: Control UI chat text flows through the shared markdown-it plus DOMPurify renderer with no math plugin, math dependency, delimiter extraction, or math restore pass. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear for implementation: adding math rendering in the shared Control UI markdown path is the right surface, but maintainers still need to choose KaTeX, MathJax, native MathML, or another approach and settle default/config and sanitizer constraints.
