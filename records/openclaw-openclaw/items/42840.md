## Summary

Keep open. Current main and v2026.6.1 still lack Control UI math rendering, and the remaining work is a product/security decision around renderer choice, dependency/sanitizer scope, delimiter parsing, and default/config behavior; the existing implementation candidate is https://github.com/openclaw/openclaw/pull/67103.

## What This Changes

Add MathJax/LaTeX, or equivalent KaTeX, rendering for formulas in Control UI chat messages with a proposed config-controlled or auto-detected delimiter mode.

## Best Possible Solution

Review the existing implementation candidate or a narrow replacement that integrates math rendering through the shared Control UI markdown renderer with dependency, CSS, sanitizer, streaming, delimiter, default/config, and regression-test coverage decided up front.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: any Control UI chat message containing inline or display TeX passes through ui/src/ui/markdown.ts, whose current markdown-it setup has no math rule or MathJax/KaTeX dependency.

## Solution Assessment

Unclear. The need is valid, but the proposed config-based MathJax shape is not yet proven to be the best solution versus a KaTeX/default-off/default-on policy in the existing renderer with sanitizer and delimiter tests.
