## Summary

Keep open. Current main still routes Control UI chat markdown through the shared sanitized markdown renderer with no MathJax/KaTeX rendering stage, and #67103 remains an open unmerged implementation candidate rather than a completed fix.

## What This Changes

Request Control UI chat to render LaTeX/MathJax-style formulas instead of displaying raw formula source.

## Best Possible Solution

Add sanitizer-aware math rendering in the shared Control UI markdown renderer, with focused tests for inline/display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and existing markdown compatibility; land #67103 or an equivalent implementation.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a message containing `$E=mc^2$` flows through `toSanitizedMarkdownHtml`, which currently has only markdown-it plus DOMPurify processing and no formula rendering layer.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable path is a shared frontend renderer change with sanitizer-aware tests, while maintainers decide whether to land, revise, or replace #67103.
