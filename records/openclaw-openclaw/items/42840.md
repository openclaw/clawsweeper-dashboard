## Summary

Keep open. Current main still lacks MathJax/KaTeX rendering in the shared Control UI markdown path, and open PR #67103 is an implementation candidate rather than a merged fix or cleanup-close reason.

## What This Changes

The issue requests configurable MathJax/LaTeX rendering support for formulas in the Control UI chat.

## Best Possible Solution

Add a sanitizer-aware math rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and existing markdown behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: a chat message containing `$E=mc^2$` reaches `toSanitizedMarkdownHtml`, which currently runs markdown-it and DOMPurify without any MathJax/KaTeX processing stage.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable solution is a shared, bundled, sanitizer-aware renderer change with regression coverage while maintainers decide whether to land, revise, or replace PR #67103.
