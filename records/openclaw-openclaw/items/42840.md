## Summary

Keep open. Current main and v2026.5.12 still lack Control UI MathJax/KaTeX/LaTeX rendering, this remains the canonical open tracker, and the open implementation attempt at https://github.com/openclaw/openclaw/pull/67103 still needs maintainer decisions around dependency choice, sanitizer scope, delimiter behavior, and default/config policy.

## What This Changes

Add MathJax/LaTeX rendering support to the Control UI so chat messages can display inline and block mathematical formulas instead of raw source.

## Best Possible Solution

Land or replace a maintainer-approved shared Control UI markdown renderer implementation that safely supports inline and display math while preserving cache behavior, DOMPurify link hardening, markdown compatibility, invalid-LaTeX fallback, ordinary dollar amounts, code spans, and code fences.

## Reproduction Assessment

Yes, at source level rather than as a bug repro: chat content flows through toSanitizedMarkdownHtml, and current main plus v2026.5.12 have no math-rendering stage, dependency, or Control UI math config.

## Solution Assessment

Unclear for the exact MathJax/config proposal; the shared Control UI markdown renderer is the right surface, but dependency choice, default behavior, delimiter policy, sanitizer strategy, and optional config need maintainer judgment.
