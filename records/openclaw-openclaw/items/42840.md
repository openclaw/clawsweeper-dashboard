## Summary

Keep open. Current main still lacks LaTeX/MathJax/KaTeX rendering in the Control UI markdown path, and #67103 remains the open implementation candidate rather than a merged fix or a reason to queue duplicate automation.

## What This Changes

Add LaTeX formula rendering, preferably with configurable MathJax-style delimiters, to the Control UI chat and markdown surfaces.

## Best Possible Solution

Review, land, or replace #67103 with a shared sanitizer-aware renderer implementation plus focused coverage for inline math, display math, invalid formulas, code spans/fences, dollar amounts, and normal markdown compatibility.

## Reproduction Assessment

Yes for the current limitation. A high-confidence static path exists: a message containing `$E=mc^2$` reaches `toSanitizedMarkdownHtml`, which currently runs markdown-it and DOMPurify without any MathJax/KaTeX processing stage.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable path is a shared Control UI renderer change with sanitizer-aware tests, while maintainers decide KaTeX vs MathJax and default vs configurable behavior.
