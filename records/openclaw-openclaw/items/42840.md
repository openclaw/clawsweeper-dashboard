## Summary

Keep open. Current main and the latest release still lack Control UI MathJax/KaTeX/LaTeX rendering, this is the canonical open tracker, and the current implementation attempt remains open and unmerged with product, dependency, sanitizer, and delimiter-policy questions unresolved.

## What This Changes

The issue requests MathJax or LaTeX rendering support for mathematical formulas in the Control UI chat surface.

## Best Possible Solution

Land or replace a maintainer-approved shared Control UI markdown renderer implementation that safely supports inline and display math while preserving cache behavior, link hardening, markdown compatibility, invalid-LaTeX fallback, ordinary dollar amounts, and code-span/code-fence behavior.

## Reproduction Assessment

Yes, at source level rather than as a bug repro: chat content flows through toSanitizedMarkdownHtml, and that renderer plus Control UI package metadata have no math rendering stage or dependency.

## Solution Assessment
