## Summary

Keep open. Current main still lacks Control UI LaTeX/MathJax/KaTeX rendering, this remains the canonical tracker for that feature, and the current KaTeX implementation candidate is still open and unmerged rather than proving the request is implemented.

## What This Changes

Add MathJax or LaTeX rendering support to the Control UI so mathematical formulas render in chat instead of appearing as raw markup.

## Best Possible Solution

Land or replace a maintainer-approved shared Control UI markdown renderer implementation that safely supports inline and display math while preserving cache behavior, link hardening, markdown compatibility, invalid-LaTeX fallback, ordinary dollar amounts, and code-span/code-fence behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a bug repro, but yes as source-level current behavior: chat flows through toSanitizedMarkdownHtml, and that renderer plus Control UI package metadata have no math rendering stage or dependency.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact MathJax/config proposal; the shared Control UI markdown renderer is the right surface, but dependency choice, default behavior, delimiter policy, and sanitizer strategy need maintainer judgment.
