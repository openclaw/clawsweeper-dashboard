## What This Changes

The issue requests inline and display LaTeX rendering in Control UI chat, preferably with configurable MathJax-style delimiters.

## Best Possible Solution

Land or replace the open implementation with a maintainer-approved shared Control UI markdown renderer change that safely supports inline and display math while preserving existing markdown, sanitizer, cache, dependency, and fallback behavior.

## Reproduction Assessment

Yes, at source level: Control UI chat content flows through toSanitizedMarkdownHtml, and current main has no math-rendering dependency, config, delimiter parser, or render pass. This is a missing feature rather than broken documented behavior.

## Solution Assessment
