## Summary

Keep open. Current main still lacks LaTeX/MathJax/KaTeX rendering in the shared Control UI markdown path, and the current implementation candidate at https://github.com/openclaw/openclaw/pull/67103 remains open and unmerged with review blockers rather than proving the feature is implemented.

## What This Changes

The issue requests MathJax/LaTeX rendering support in the Control UI so formulas in chat messages render as equations instead of raw delimiter text.

## Best Possible Solution

Land a maintainer-approved shared Control UI markdown renderer implementation with sanitizer-aware coverage for inline math, display math, invalid LaTeX, code spans/fences, currency prose, and normal markdown compatibility.

## Reproduction Assessment

Yes. The current chat path reaches `toSanitizedMarkdownHtml`, and that renderer has no math dependency or processing stage, so LaTeX delimiters remain ordinary markdown text on current main.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The shared markdown renderer is the right surface, but maintainers still need to choose MathJax vs KaTeX, delimiter rules, default behavior, and sanitizer constraints.
