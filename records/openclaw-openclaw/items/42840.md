## Summary

Current main still does not render LaTeX/MathJax/KaTeX in the Control UI chat path, so the feature request remains valid; it should stay open while maintainers decide the renderer/config/sanitizer shape and review the active implementation candidates, especially https://github.com/openclaw/openclaw/pull/87568 and the older https://github.com/openclaw/openclaw/pull/67103.

## What This Changes

Request to add MathJax/LaTeX-style math rendering for Control UI chat messages, preferably with configurable delimiters or auto-detection.

## Best Possible Solution

Land a maintainer-approved implementation in the shared Control UI markdown renderer with explicit renderer choice, delimiter/default policy, sanitizer and dependency review, streaming support, and regression tests for code spans, code fences, currency prose, invalid LaTeX, CJK boundaries, inline math, and display math.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Control UI chat message containing `$...$` or `$$...$$` flows through the shared markdown renderer, which currently has no math-rendering plugin, dependency, or delimiter rule.

## Solution Assessment

Unclear. The feature need is valid, but the best solution needs maintainer choice between MathJax and KaTeX, default-on versus config-gated behavior, and a narrow sanitizer/dependency policy before implementation is accepted.
