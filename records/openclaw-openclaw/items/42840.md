The issue requests MathJax or LaTeX rendering support in the Control UI so chat messages can display inline and block formulas instead of raw delimiters.

## Best Possible Solution

Land or replace the open implementation with a maintainer-approved shared Control UI markdown renderer change that safely supports inline and display math while preserving existing markdown, sanitizer, cache, and fallback behavior.

## Reproduction Assessment

Yes, at source level: Control UI chat content flows through `toSanitizedMarkdownHtml`, and current main has no math-rendering dependency, config, delimiter parser, or render pass. This is a missing feature rather than broken documented behavior.

## Solution Assessment
