## Summary

Keep open: this PR targets a real P1 diagnostic/session-stall bug, but the current patch still misses the default production timing because idle/no-queue zombie recovery is evaluated behind the heartbeat's recent-activity gate.

## What This Changes

Adds diagnostic heartbeat detection for idle sessions that still have active embedded-run activity, plus unit and integration tests for force-clearing leaked embedded-run handles.

## Best Possible Solution

Keep the idle-zombie recovery shape, but move or extend the heartbeat scan so active diagnostic embedded-run activity remains eligible through the default abort threshold, then cover that timing with a regression test.

## Reproduction Assessment

Yes for the PR defect from source: the default abort threshold is at least 300s, while idle/no-queue sessions stop passing the heartbeat scan gate after 120s of no recent diagnostic activity. The original Slack race is supported by the linked logs but was not live-reproduced in this read-only review.

## Solution Assessment

No. The proposed owner boundary is plausible, but the fix is incomplete until the heartbeat can evaluate idle zombie activity through the production abort threshold instead of only in short configured tests.
