## Summary

Keep this PR open for maintainer review. The patch is a focused mitigation for the linked session-stall bug and I did not find a discrete code defect, but it changes session recovery from observing an idle/no-queue active embedded-run marker to aborting or force-clearing it after the stuck-session abort threshold, so the remaining question is maintainer acceptance of that session-state and availability tradeoff.

## What This Changes

The PR adds diagnostics heartbeat detection, recovery routing, and focused tests for idle sessions that still report active embedded-run activity after the normal clear path was skipped.

## Best Possible Solution
