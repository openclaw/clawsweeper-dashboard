Full review comments:

- **[P1] Preserve the shipped inbound callback contract:** `extensions/whatsapp/src/inbound/types.ts:120-123`
  - body: `WebInboundMessage` is exported through the WhatsApp plugin API and the latest release exposes flat fields and methods such as `body`, `chatId`, `reply`, and `mediaPath`. Replacing that shape with only `event`/`payload`/`platform` makes existing callback consumers fail on upgrade, so add deprecated flat aliases/methods with focused tests or land only after explicit maintainer approval for a breaking callback release.
  - confidence: 0.95
- **[P2] Only set media when media facts exist:** `extensions/whatsapp/src/inbound/monitor.ts:1026-1030`
  - body: This always creates `payload.media`, even for text/location/contact messages where `path`, `type`, and `fileName` are all undefined. Since the new type makes `media` optional, callers can reasonably use `if (msg.payload.media)` to detect media; build the object only when a real media fact exists and cover a non-media inbound message.
  - confidence: 0.87
- **[P2] Keep group context absent for direct batches:** `extensions/whatsapp/src/inbound/monitor.ts:364-370`
  - body: The debounce combine path always assigns a `group` object, but direct chats can also be debounced and batched. That makes `msg.group` truthy for `chatType: "direct"`, so only create the group context when the source message is a group message or when group mentions actually exist.
  - confidence: 0.84
