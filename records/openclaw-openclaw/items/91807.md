This PR remains necessary and is ready for normal maintainer review: current main lacks `--file` on `infer image generate`, while the branch follows the existing `image edit` path, forwards loaded inputs through the shared image-generation runtime, adds focused coverage, and supplies convincing real CLI proof. ## What This Changes

Adds repeatable `--file` input support to `infer image generate`, publishes the flag in capability metadata, tests forwarding and inspection output, and updates the CLI guide.

## Best Possible Solution

Land the focused additive CLI change through the existing shared image-generation runtime and keep provider-specific reference-image support and limits owned by each provider's established capability contract.

## Reproduction Assessment

## Solution Assessment

Yes. Extending the existing generate command to reuse `runImageGenerate` and the established provider `inputImages` contract is narrower and more maintainable than adding a separate command or provider-specific CLI branches.
