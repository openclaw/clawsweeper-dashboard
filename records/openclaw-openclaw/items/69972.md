## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve JSON output after RPC restarts:** `src/cli/daemon-cli/lifecycle.ts:386-399`
  - body: When the RPC-first restart succeeds, this branch runs the post-check with `stdout: process.stdout` and `warnings: []`, then returns before the service lifecycle creates and emits daemon action JSON. `openclaw gateway restart --json` can therefore produce no structured success output and can drop diagnostic warnings.
  - confidence: 0.93
- **[P2] Run transaction recovery without requiring the sentinel:** `src/gateway/server-restart-sentinel.ts:160-162`
  - body: The new fallback synthesizes payload from a pending restart transaction only inside `scheduleRestartSentinelWake`, but startup currently calls that function only after the sentinel file exists. If the sentinel write is missing, this recovery path is unreachable and the interrupted-turn follow-up can still be skipped.
  - confidence: 0.9
- **[P2] Schedule restart before aborting the active run:** `src/gateway/restart-transaction.ts:102-119`
  - body: The terminal handoff aborts the active run before `scheduleGatewaySigusr1Restart` and the following transaction write succeed. If scheduling or persistence throws, the user-facing run has already been aborted without a queued restart.
  - confidence: 0.87
- **[P2] Keep the Windows restart wait budget:** `src/cli/daemon-cli/lifecycle.ts:373-374`
  - body: The refactor removes the Windows-specific 180s health wait and uses the default attempt budget for all platforms. That regresses current managed Windows restart behavior, where slow service restarts intentionally get a larger timeout.
  - confidence: 0.82
- **[P2] Keep cwd out of bundled plugin resolution:** `src/plugins/bundled-dir.ts:119-123`
  - body: This prepends a cwd-derived source checkout under Vitest/source execution. Current main intentionally restricts bundled plugin resolution to trusted package roots and tests that VITEST must not add cwd, so this re-opens a plugin-code trust boundary.
  - confidence: 0.86
