## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve daemon JSON output on RPC restart success:** `src/cli/daemon-cli/lifecycle.ts:386-399`
  - body: When the RPC restart path succeeds, this branch returns before the daemon action context emits structured JSON and passes post-check diagnostics into a throwaway warnings array. `openclaw gateway restart --json` can therefore produce no success payload and lose restart diagnostics.
  - confidence: 0.93
- **[P2] Keep the Windows restart wait budget:** `src/cli/daemon-cli/lifecycle.ts:373-374`
  - body: The refactor removes the Windows-specific 180s health wait and uses the default attempt budget for every platform. That regresses the existing managed Windows restart behavior where slow service restarts intentionally get a longer timeout.
  - confidence: 0.82
- **[P2] Run transaction recovery when the sentinel file is missing:** `src/gateway/server-restart-sentinel.ts:160-162`
  - body: The fallback synthesizes payload from a pending restart transaction, but startup only calls `scheduleRestartSentinelWake` after the sentinel file exists. The missing-sentinel recovery path is therefore unreachable in the failure mode it is meant to cover.
  - confidence: 0.9
- **[P2] Schedule the restart before aborting the active run:** `src/gateway/restart-transaction.ts:102-119`
  - body: The terminal handoff aborts the active run before scheduling SIGUSR1 and before the subsequent transaction write succeeds. If scheduling or persistence throws, the user-facing run has already been interrupted without a queued restart.
  - confidence: 0.87
- **[P2] Keep cwd out of bundled plugin resolution:** `src/plugins/bundled-dir.ts:119-123`
  - body: This prepends a cwd-derived source checkout under Vitest/source execution. Current main excludes cwd from bundled plugin candidates and tests that VITEST must not add it, so this reopens a plugin-code trust boundary.
  - confidence: 0.86
