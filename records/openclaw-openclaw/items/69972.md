Full review comments:

- **[P1] Preserve daemon JSON output on RPC restart success:** `src/cli/daemon-cli/lifecycle.ts:386-399`
  - body: When the RPC restart path succeeds, this branch returns before `runServiceRestart` emits the daemon action JSON payload and passes post-check diagnostics into a throwaway warnings array. `openclaw gateway restart --json` can therefore produce no structured success response and lose restart diagnostics.
  - confidence: 0.93
- **[P2] Keep the Windows restart wait budget:** `src/cli/daemon-cli/lifecycle.ts:373-374`
  - body: The refactor removes the Windows-specific 180 second health wait and uses the default attempt budget for every platform. That regresses existing managed Windows restart behavior where slow service restarts intentionally get a longer timeout.
  - confidence: 0.84
- **[P2] Trigger transaction recovery when the sentinel is missing:** `src/gateway/server-restart-sentinel.ts:160-162`
  - body: The fallback synthesizes payload from a pending restart transaction, but startup only calls this function after the sentinel file exists. The missing-sentinel recovery path is therefore unreachable in the failure mode it is meant to cover.
  - confidence: 0.9
- **[P2] Schedule the restart before aborting the active run:** `src/gateway/restart-transaction.ts:102-119`
  - body: The terminal handoff aborts the active run before scheduling SIGUSR1 and before the next transaction write succeeds. If scheduling or persistence throws, user work has already been interrupted without a queued restart.
  - confidence: 0.87
- **[P2] Keep cwd out of bundled plugin resolution:** `src/plugins/bundled-dir.ts:119-123`
  - body: This prepends a cwd-derived source checkout under Vitest/source execution. Current main explicitly tests that VITEST must not add cwd, so an arbitrary OpenClaw-shaped cwd can win over trusted package roots.
  - confidence: 0.86
