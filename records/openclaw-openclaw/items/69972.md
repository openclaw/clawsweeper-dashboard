## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve JSON output after RPC restarts:** `src/cli/daemon-cli/lifecycle.ts:386-399`
  - body: When the RPC-first restart succeeds, this branch runs the post-check with `stdout: process.stdout` and `warnings: []`, then returns before `runServiceRestart` creates and emits the daemon action context. `openclaw gateway restart --json` can therefore produce no structured success output and can drop post-check warnings compared with the existing restart path.
  - confidence: 0.93
- **[P2] Run transaction recovery without requiring the sentinel:** `src/gateway/server-restart-sentinel.ts:160-162`
  - body: The new transaction fallback is inside `scheduleRestartSentinelWake`, but startup only calls that function after the sentinel file exists. If the sentinel write is missing, the recovery path added here is unreachable and the interrupted-turn follow-up can still be skipped.
  - confidence: 0.9
- **[P2] Schedule restart before aborting the active run:** `src/gateway/restart-transaction.ts:102-119`
  - body: The terminal handoff aborts the active run before `scheduleGatewaySigusr1Restart` and the following transaction write succeed. If scheduling or persistence throws, the user-facing run has already been aborted without a queued restart.
  - confidence: 0.87
- **[P2] Keep the Windows restart wait budget:** `src/cli/daemon-cli/lifecycle.ts:373-374`
  - body: The refactor removes the Windows-specific 180s health wait and uses the default attempt budget for all platforms. That regresses current managed Windows restart behavior, where slow service restarts intentionally get a larger timeout.
  - confidence: 0.82
- **[P2] Keep cwd out of bundled plugin resolution:** `src/plugins/bundled-dir.ts:108-121`
  - body: This adds the current working directory as a preferred package root under Vitest/source execution. Current main intentionally restricts bundled plugin resolution to trusted package roots and has regression coverage ensuring VITEST does not add cwd, so this would re-open a plugin-code trust boundary after rebase.
  - confidence: 0.86
