## Summary

Keep open: current `main` and `v2026.6.6` still omit `--file` from `openclaw infer image generate`, and the linked open PR at https://github.com/openclaw/openclaw/pull/91807 is the active implementation path.

## What This Changes

The issue requests repeatable input-file support on `openclaw infer image generate`, mirroring `image edit --file` and failing clearly when selected providers cannot use reference images.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91807 if maintainers accept the additive CLI surface, keeping provider-specific support and limits in the existing image-generation contracts.

## Reproduction Assessment

Yes. Source inspection shows `image generate` has no registered `--file` option or forwarding field, while the shared runner and `image edit` path already support file-backed `inputImages`; I did not run the CLI because this was a read-only review.

## Solution Assessment

Yes. The narrowest maintainable solution is to expose repeatable `--file` on the existing generate command, forward it through `runImageGenerate`, and cover metadata, forwarding, and unsupported-provider behavior.
