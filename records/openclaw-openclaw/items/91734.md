## Summary

Keep open: current main and latest release still omit `--file` from `openclaw infer image generate`, and the linked implementation PR should be reviewed or landed before this issue is closed.

## What This Changes

Request support for passing one or more input/reference image files to the `openclaw infer image generate` CLI path, with clear failure when the selected provider cannot use them.

## Best Possible Solution

Land the linked implementation PR if it preserves no-file generation, forwards repeatable files through the existing `inputImages` path, and keeps unsupported-provider failures explicit.

## Reproduction Assessment

Yes. Source inspection shows `image generate` has no `--file` option or forwarding on current main, while the shared runner and `image edit` already support file-backed `inputImages`; I did not run the CLI in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable solution is to expose repeatable `--file` on the existing `image generate` command, reuse `runImageGenerate`'s input-image path, and cover forwarding plus unsupported-provider behavior in focused tests.
