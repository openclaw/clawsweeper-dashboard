## Summary

Keep open: current main and v2026.6.1 still omit `--file` from `openclaw infer image generate`, while the shared image runner already has an input-image path, so the requested CLI capability remains actionable.

## What This Changes

The issue asks for `openclaw infer image generate` to accept one or more input image files and pass them as provider reference inputs when supported.

## Best Possible Solution

Expose repeatable `--file` on `openclaw infer image generate`, forward it through the existing input-image path, and cover both successful forwarding and unsupported provider/model failure without adding config surfaces.
