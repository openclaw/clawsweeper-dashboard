## What This Changes

The issue requests adding repeatable `--file` support to `openclaw infer image generate` so CLI image generation can use provided reference/input images like the existing image edit command.

## Best Possible Solution

Review and land or explicitly close https://github.com/openclaw/openclaw/pull/91807, keeping CLI metadata, command help, docs, and regression coverage aligned around repeatable `--file` forwarding with explicit unsupported-provider errors.

## Reproduction Assessment

## Solution Assessment

Yes: exposing repeatable `--file` on the existing `image generate` command and forwarding it through the shared runner is the narrowest maintainable fix; provider-specific unsupported behavior should remain explicit.
