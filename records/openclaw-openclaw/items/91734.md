## What This Changes

The issue asks for `openclaw infer image generate` to accept one or more input files and pass them as reference images, matching the existing image edit flow.

## Best Possible Solution

Land a narrow CLI/docs/test update that adds `--file` to `image generate`, passes files through the existing runner/provider request path, and documents the new workflow.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing repeatable file option and `runImageGenerate` file path, with tests/docs, rather than adding a new runtime abstraction.
