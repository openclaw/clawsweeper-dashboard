## Summary

Keep open: the regression and proof are credible, but the patch over-classifies every captured initial session stream as a default fallback and can replace caller-supplied custom streams on the production run/compaction path.

## What This Changes

The branch adds base-stream tracking in embedded stream resolution, tests for captured PI-managed `openai-completions` streams and uncaptured custom streams, and a changelog entry for the linked regression.

## Best Possible Solution

Keep the intended `openai-completions` boundary-aware routing, but narrow PI-default detection so arbitrary custom initial session streams remain custom and add a production-path regression test for that case.

## Reproduction Assessment

Yes. Current main has a clear source-level reproduction path: embedded runs capture a PI-managed stream that is not identity-equal to `streamSimple`, so the resolver bypasses the OpenClaw completions transport that adds `stream_options.include_usage`.

## Solution Assessment

No. The direction is right, but this implementation is not the best fix because it treats every captured initial base stream as a default PI stream; the safer fix should identify PI-managed defaults without swallowing caller-supplied custom streams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve captured custom session streams:** `src/agents/pi-embedded-runner/stream-resolution.ts:40-41`
  - body: Every first-seen session stream is added to `embeddedAgentBaseStreamFns`, and the production runner immediately passes `resolveEmbeddedAgentBaseStreamFn()` into `resolveEmbeddedAgentStreamFn()`. That means an `AgentOptions`/custom `streamFn` present before the first turn is now classified as a default stream and replaced by OpenClaw's transport instead of being preserved. The added uncaptured-custom test does not cover this production path, so please keep PI-managed default detection from swallowing arbitrary captured custom stream functions.
  - confidence: 0.86
