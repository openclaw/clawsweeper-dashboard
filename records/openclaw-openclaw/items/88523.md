## What This Changes

The PR changes subagent completion announce retry accounting so transient parent-busy completion handoffs keep retrying past the normal retry-count cap until the existing completion hard-expiry window.

## Best Possible Solution

Land this scoped retry-accounting fix after the usual CI and maintainer checks, keeping the 30-minute hard expiry and the non-pending/non-completion retry caps unchanged.

## Reproduction Assessment

## Solution Assessment

Yes. Persisting the transient `completion_handoff_pending` outcome and reading it from both guards is narrower than raising the global retry cap or timeout, and it preserves the existing hard-expiry bound.
