## Summary

Keep open for normal maintainer review. The latest head appears to address the prior stale-marker finding by resetting `lastHandoffPending` per attempt and adding a regression, and I did not find a new blocking correctness issue.

## What This Changes

The PR adds a persisted latest-attempt handoff-pending marker and uses it in both subagent completion announce retry guards so transient parent-busy completions retry until the existing hard expiry instead of giving up at the normal retry cap.

## Best Possible Solution

Keep the narrow latest-attempt marker approach, land after relevant checks and maintainer acceptance of the bounded retry semantics, and leave broader session write-queue work to https://github.com/openclaw/openclaw/issues/88383 if still needed.

## Solution Assessment

Yes. The latest persisted-marker approach is narrow, fixes both retry guards, clears stale per-attempt state, and avoids raising retry constants globally.
