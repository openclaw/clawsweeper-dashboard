## Summary

Keep open: the reported bug is still present on current main, but this branch is not merge-ready because it edits legacy `src/memory` files instead of the active memory host SDK and memory-core files, and the dated-path parser still needs the same normalization.

## What This Changes

The PR changes legacy `src/memory` path predicates and tests to treat `memory/` directory prefixes case-insensitively for memory indexing and temporal decay.

## Best Possible Solution

Port the casing fix to `packages/memory-host-sdk/src/host/internal.ts` and `extensions/memory-core/src/memory/temporal-decay.ts`, lowercase before both prefix checks and dated parsing, add focused regression tests, and coordinate with https://github.com/openclaw/openclaw/pull/68155.

## Reproduction Assessment

Yes, at source level. Current main's active predicate rejects `MEMORY/notes.md`, and the dated-memory regex misses `MEMORY/2025-01-01.md` unless the normalized path is lowercased.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the PR edits deleted legacy files and still needs the active dated-path parser normalized before it can safely fix current main.
