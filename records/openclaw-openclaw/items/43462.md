## What This Changes

The PR updates legacy `src/memory` path predicates and tests to treat `memory/` directory prefixes case-insensitively for memory indexing and temporal decay.

## Best Possible Solution

Port the casing fix to the active host SDK and memory-core files, lowercase before both prefix checks and dated parsing, add focused regression coverage, and coordinate with https://github.com/openclaw/openclaw/pull/68155.

## Reproduction Assessment

Yes, at source level. Current main's active predicate rejects case-variant `memory/` directories, and the dated-memory regex fails uppercase paths unless the normalized path is lowercased.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the PR edits deleted legacy files and still needs the active dated-path parser normalized before it can safely fix current main.
