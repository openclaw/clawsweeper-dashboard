Keep this PR open. The heartbeat prompt split is directionally correct, but the latest head still removes the documented explicit `fallback: "pi"` path for missing plugin harnesses; the PR also has the protected `maintainer` label, so it is not a cleanup-close candidate.

## What This Changes

The PR rewrites heartbeat prompt selection for Codex/message-tool heartbeat turns, updates cron/exec/task prompt handling, tests, a prompt snapshot, changelog text, and plugin runtime fallback behavior.

## Best Possible Solution

Keep the heartbeat tool prompt split, but preserve explicit PI fallback semantics and only change runtime fallback policy through a separate maintainer-approved design/docs update.

## Solution Assessment

No. The prompt rewrite is the right direction, but removing explicit PI fallback is broader than necessary; the safer fix is to select heartbeat prompt text from tool availability while preserving the documented runtime fallback contract.
