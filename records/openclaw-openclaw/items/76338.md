## Summary

Keep this maintainer-labeled PR open for review. Current main does have the mixed heartbeat prompt contract, and the branch mostly fixes it for message-tool/Codex-pinned sessions, but the new prompt-mode detection can still miss a supported Codex auto-selection path.

## What This Changes

The branch rewrites heartbeat prompts for structured `heartbeat_respond` turns, updates runner prompt selection, adds regression tests, refreshes the Codex heartbeat snapshot, and adds a changelog entry.

## Best Possible Solution

Keep the structured heartbeat tool contract for every heartbeat turn that actually exposes `heartbeat_respond`, while preserving the text sentinel only for legacy automatic-text paths that do not have the structured tool.

## Reproduction Assessment

Yes. Source inspection on current main shows the default heartbeat prompt asks for `HEARTBEAT_OK` and the runner appends the `heartbeat_respond` instruction to the same turn.

## Solution Assessment

No, not quite. The direction is right, but the patch should align prompt-mode selection with the same harness/default-visible-reply logic used by dispatch so supported Codex auto-selected sessions are not left on the mixed contract.
