## Summary

Keep open. The branch is on the right path and the force-push covers the earlier legacy `codex/*` auto-selection case, but it still misses the documented environment-forced Codex runtime, so some heartbeat turns would keep the mixed `heartbeat_respond`/`HEARTBEAT_OK` contract. The protected `maintainer` label also prevents cleanup-close.

## What This Changes

The PR rewrites heartbeat prompts to use `heartbeat_respond` for message-tool and Codex harness sessions, adds resolver/regression coverage and snapshot updates, and records a changelog fix.

## Best Possible Solution

Reuse the same harness-selection policy inputs as dispatch, including environment overrides and pinned session state, so `HEARTBEAT_OK` remains only on true legacy text-only heartbeat paths.

## Reproduction Assessment

Yes. Source inspection on current main shows the default heartbeat prompt asks for `HEARTBEAT_OK` while the runner appends the structured `heartbeat_respond` instruction to the same heartbeat turn.

## Solution Assessment

No. The direction is correct, but the PR should share dispatch's harness-selection inputs or explicitly account for `OPENCLAW_AGENT_RUNTIME=codex`; otherwise env-forced Codex heartbeats keep the mixed contract.
