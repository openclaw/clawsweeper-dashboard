## Summary

Keep open. Current main still lacks a cron-scoped tool allow/deny policy or explicit inheritance from subagent tool policy, and the unresolved part affects scheduled-job authorization plus message delivery behavior.

## What This Changes

Add cron-scoped tool restrictions, or make cron sessions inherit subagent tool policy, so scheduled jobs cannot use denied tools such as `message`.

## Best Possible Solution

Define a cron-specific policy or explicit inheritance contract, wire it into the shared fail-closed policy pipeline, and update schema/types/help/docs/generated metadata plus focused cron regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main has no cron tool-policy subtree in schema/types/docs, and the runtime subagent-policy branch applies only to subagent envelope sessions while cron relies on per-job `payload.toolsAllow`.

## Solution Assessment

Unclear until maintainers choose the public policy shape. Both `tools.cron.tools.allow/deny` and explicit inheritance from `tools.subagents.tools` are plausible, but the maintainable fix needs a documented precedence contract first.
