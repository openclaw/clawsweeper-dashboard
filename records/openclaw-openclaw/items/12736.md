## Summary

Keep open. Current main still lacks a cron-scoped tool allow/deny policy or explicit inheritance from subagent tool policy; the implemented surfaces are per-job `toolsAllow` and cron delivery safeguards, which do not satisfy the requested config-level enforcement.

## What This Changes

Add a cron-scoped tool restriction config such as `tools.cron.tools.deny`, or make isolated cron sessions inherit the existing subagent tool policy.

## Best Possible Solution

Define a first-class cron tool-policy contract, or explicit cron inheritance from subagent policy, with documented fail-closed precedence plus schema, exported types, docs, metadata/help, and focused cron delivery/tool-policy regression coverage.

## Reproduction Assessment

Yes, by source inspection for the missing capability: current main has no cron-scoped policy subtree in schema/types/docs, and cron sessions do not enter the subagent-policy path. No live run was attempted because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public policy shape. Both `tools.cron.tools.allow/deny` and explicit inheritance from `tools.subagents.tools` are plausible, but the maintainable fix needs a documented precedence contract first.
