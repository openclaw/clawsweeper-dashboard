## Summary

Keep open: current main and v2026.6.8 still lack a cron-scoped config policy or documented inheritance from `tools.subagents.tools`; recent per-job `toolsAllow` and creator-surface work is adjacent but does not solve the requested operator deny layer.

## What This Changes

## Best Possible Solution

Keep this open until maintainers choose a cron tool-policy design, then implement an additive `tools.cron.tools.allow/deny` or explicit subagent-inheritance path with schema/types/docs and regression tests for announce-mode message denial and per-job `toolsAllow` precedence.
