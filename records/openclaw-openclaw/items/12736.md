## Summary

This should stay open: current main still has no cron-scoped tool policy, and the remaining work is a new config/product design rather than an already-defined bug fix.

## What This Changes

The issue requests a cron-scoped tool deny policy so isolated cron sessions can restrict tools like `message` independently from, or by inheriting, subagent tool policy.

## Best Possible Solution

Define a cron-scoped policy contract, add schema/types/help/generated metadata/docs, and cover isolated cron message-tool denial with focused regression tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection confirms current main has no cron-scoped tool policy and cron jobs only use delivery-mode handling plus per-job `toolsAllow`.

## Solution Assessment

No final solution is implemented yet. The maintainable path is either explicit `tools.cron.tools.allow/deny` or documented inheritance from `tools.subagents.tools`, with clear precedence against existing policy layers.
