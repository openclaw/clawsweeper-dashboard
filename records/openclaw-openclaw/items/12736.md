## Summary

Keep open. Current main still lacks a cron-scoped tool policy and cron sessions do not inherit `tools.subagents.tools`, so the requested hard enforcement remains unresolved.

## What This Changes

Request a cron-scoped tool allow/deny policy, or explicit inheritance from subagent tool restrictions, for isolated cron jobs.

## Best Possible Solution

Define and document a cron tool-policy layer, then wire it through schema, exported types, generated config metadata, docs, runtime policy resolution, and regression coverage for isolated cron message-tool denial.

## Reproduction Assessment

Yes for the reported limitation. Source inspection shows cron session keys are distinct from subagent keys, no `tools.cron.tools` schema/type exists, and the runtime only applies subagent policy to subagent envelopes.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to choose between an explicit cron policy and subagent-policy inheritance, then document precedence against existing policy layers.
