## Summary

Keep open. Current main still has per-job cron `toolsAllow` and message-tool delivery handling, but no cron-scoped config policy or documented inheritance from `tools.subagents.tools`, so this remains a feature/config design item.

## What This Changes

Request adding `tools.cron.tools.deny` or equivalent cron inheritance so isolated cron jobs can be restricted from tools such as `message`.

## Best Possible Solution

Define one cron policy contract, document precedence, update schema/types/help/generated metadata/docs, and add focused tests for isolated cron message-tool denial across delivery modes.

## Reproduction Assessment

Not applicable as a feature/config request. Source and docs inspection on current main confirms there is no `tools.cron.tools` contract or cron inheritance path from `tools.subagents.tools`.

## Solution Assessment

Unclear until maintainers choose the policy shape. The maintainable direction is either explicit `tools.cron.tools.allow/deny` or documented inheritance from `tools.subagents.tools`, with precedence specified before implementation.
