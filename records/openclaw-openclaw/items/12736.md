## Summary

Keep open. Current main still lacks a cron-scoped tool allow/deny config or an explicit inheritance rule from subagent tool policy, and the related recent work only covers per-job `toolsAllow` and message delivery behavior, not a system-wide cron tool restriction contract.

## What This Changes

The request is to add `tools.cron.tools.allow/deny` or make cron sessions inherit subagent tool restrictions so isolated cron jobs can be prevented from using tools such as `message`.

## Best Possible Solution

Define a cron-specific policy or explicit inheritance contract, update schema/types/help/docs/generated config metadata, wire it into the shared policy pipeline with fail-closed precedence, and add focused cron regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main has no cron tool-policy subtree in schema/types/docs, and the runtime subagent policy branch only applies to subagent envelope sessions while cron relies on per-job `payload.toolsAllow`.

## Solution Assessment

Unclear until maintainers choose the public policy shape. Both `tools.cron.tools.allow/deny` and explicit inheritance from `tools.subagents.tools` are plausible, but the maintainable fix needs a documented precedence contract first.
