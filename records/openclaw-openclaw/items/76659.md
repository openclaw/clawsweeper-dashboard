## Summary

Keep open. The PR is a valid implementation candidate for the reply-context regression, but the latest head still has a blocking security bug because it scans appended untrusted reply context for image references, and it also lacks the required user-facing changelog entry.

## What This Changes

The PR threads structured reply metadata from auto-reply followup runs into the embedded runner, appends sanitized reply context to the submitted model prompt, and adds focused coverage.

## Best Possible Solution

Keep the explicit reply-context seam, but scan images only from the actual user prompt, add the changelog entry, and rerun focused agent/auto-reply validation before landing this PR or a branch repair for #76419.

## Reproduction Assessment

Yes, source-level. Current main renders `ReplyToBody` into prompt context and then strips that non-transcript text into `runtimeContext`, while the PR diff shows the intended current-turn context seam and the new image-scanning regression.

## Solution Assessment

No, not as submitted. Threading explicit reply context is the right boundary, but image detection should continue to inspect only `promptSubmission.prompt`, and the user-facing fix needs a changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep reply context out of image scanning:** `src/agents/pi-embedded-runner/run/attempt.ts:2810`
  - body: Using `promptForModel` here makes image detection parse the appended replied-message JSON as if it were user-authored prompt text. Since the detector matches local paths, `file://` URLs, and `[media attached: ...]` tokens anywhere in the scanned string, a quoted reply can unexpectedly attach local or workspace media on vision-capable runs; pass only `promptSubmission.prompt` to image detection.
  - confidence: 0.95
- **[P3] Add the required changelog entry:** `src/auto-reply/reply/get-reply-run.ts:341`
  - body: This is a user-facing fix for channel reply context, but the PR does not update `CHANGELOG.md`. Repo policy requires a single-line changelog entry for user-facing fixes before merge.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete data-exposure risk by allowing untrusted replied-message text to drive prompt-local image loading.

Concerns:

- **[high] Untrusted reply text can trigger image attachment:** `src/agents/pi-embedded-runner/run/attempt.ts:2810`
  - body: Passing `promptForModel` into image detection means a replied message body containing `./file.png`, `~/...`, `file://...`, or `[media attached: ...]` can be parsed and loaded as model image input on vision-capable runs, leaking or altering prompt media unexpectedly.
  - confidence: 0.93
