## Summary

Keep open. The PR direction fixes the linked reply-context regression, and the latest force push resolved the earlier TypeScript narrowing issue, but the patch still feeds untrusted replied-message text into prompt image detection and is missing the required changelog entry.

## What This Changes

The PR adds a structured current-turn reply context, threads it through auto-reply followup and embedded runs, appends sanitized reply metadata to the submitted model prompt, and adds focused coverage.

## Best Possible Solution

Keep the explicit current-turn reply context design, but scan only the actual user prompt for image references, add the changelog entry, then rerun the focused agent and auto-reply validation.

## Reproduction Assessment

Yes, source-level. Current main renders `ReplyToBody` and then strips it out of the submitted prompt, while the PR head source shows the appended reply-context suffix being passed into image detection.

## Solution Assessment

No, not as submitted. Threading explicit reply context is the right boundary, but image detection should continue to inspect only `promptSubmission.prompt` so quoted untrusted text cannot attach files or media.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep reply context out of image scanning:** `src/agents/pi-embedded-runner/run/attempt.ts:2810`
  - body: Using `promptForModel` here makes image detection parse the appended replied-message JSON as if it were user instructions. Since `detectImageReferences` matches paths and `[media attached: ...]` tokens anywhere in the scanned prompt, a quoted message can cause workspace/local media to be loaded and attached on vision-capable runs; scan only `promptSubmission.prompt` for image refs.
  - confidence: 0.95
- **[P3] Add the required changelog entry:** `src/auto-reply/reply/get-reply-run.ts:341`
  - body: This is a user-facing fix for channel reply context, but the PR does not update `CHANGELOG.md`. Repo policy requires a single-line changelog entry for user-facing fixes before merge.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete data-exposure risk by allowing untrusted replied-message text to drive prompt-local image loading.

Concerns:

- **[high] Untrusted reply text can trigger image attachment:** `src/agents/pi-embedded-runner/run/attempt.ts:2810`
  - body: Passing `promptForModel` into image detection means a replied message body containing `./file.png`, `~/...`, `file://...`, or `[media attached: ...]` can be parsed and loaded as model image input on vision-capable runs. That can leak local or workspace media and change the model input unexpectedly.
  - confidence: 0.93
