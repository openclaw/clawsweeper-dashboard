## Summary

Keep open: current main already has opt-in `/verbose full`, Control UI tool cards, and compact channel progress previews, but it still does not define the requested default cross-channel policy for automatically pushing full tool results to users. This is a product/security-sensitive feature request, not a broken documented contract.

## What This Changes

Request automatic user-visible delivery of tool and exec results, or a session/channel control for that behavior, without requiring the agent to send a follow-up message.

## Best Possible Solution

Keep the existing `/verbose full`, Control UI cards, and compact progress previews, then define an explicit sanitized channel/session visibility policy before changing defaults for full tool-result delivery.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and tests define opt-in full output plus compact progress previews, not a default full-result push contract. Source inspection verifies the remaining gap is the requested visibility policy, not a failing documented behavior.

## Solution Assessment

No: blanket automatic pushing of every tool result is not the safest solution. The maintainable direction is to reuse the existing event plumbing and `/verbose full` semantics with an explicit sanitized channel/session policy for when full output is user-visible.
