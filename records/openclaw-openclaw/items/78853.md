## Summary

Keep open: current main has structured command-output events, compact channel progress previews, and opt-in `/verbose full` tool-output bubbles, but it still does not define automatic full tool-result delivery to users by default. The remaining work is a product/security policy decision, not a broken documented contract.

## What This Changes

The issue asks OpenClaw to push tool results to users automatically instead of leaving them only in agent context unless the agent sends a follow-up message.

## Best Possible Solution

Define a channel-safe tool-result visibility policy that reuses existing structured events, defaults to compact sanitized progress, and exposes raw or full output only through explicit user/session/channel policy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection on current main confirms normal channel delivery provides compact progress and opt-in `/verbose full`, not automatic full tool-result delivery by default.

## Solution Assessment

No: blanket automatic push of every tool result is not the safest solution. The maintainable path is an explicit sanitized or opt-in visibility policy layered on the existing structured events.
