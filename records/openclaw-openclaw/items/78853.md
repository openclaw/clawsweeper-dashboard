## Summary

Keep open: current main has internal agent events, progress drafts, and opt-in verbose forwarding for tool summaries/output, but it still does not implement default automatic user-visible delivery of full tool results. This needs maintainer product and disclosure-policy direction rather than conservative cleanup.

## What This Changes

Request automatic user-visible delivery of tool command results, or a controllable visibility mode, without requiring the agent to send a follow-up message.

## Best Possible Solution

Define an explicit, channel-safe visibility policy that reuses structured agent events and progress drafts, defaults to compact sanitized progress, and exposes raw or full tool output only through opt-in verbose, session, or config controls.

## Reproduction Assessment

No high-confidence live reproduction was established. Source, docs, and tests do confirm the behavior class: tool results are internal and available through progress/verbose paths, but full results are not automatically posted by default.

## Solution Assessment

No: automatically pushing every tool result is not the safest maintainable solution because tool output can be sensitive and noisy. A bounded, redacted, opt-in visibility policy is the safer direction.
