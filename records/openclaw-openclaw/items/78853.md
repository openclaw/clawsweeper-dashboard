## Summary

Keep open: current main has structured command-output events and channel tool-progress previews, but it still does not define an automatic user-visible push of full tool results. The request is a product/security-sensitive visibility feature rather than a broken existing contract.

## What This Changes

The issue asks for tool execution results to be automatically pushed to users instead of requiring the agent to send a follow-up message.

## Best Possible Solution

Define a channel-safe, redacted tool-result visibility policy that keeps compact progress visible by default and exposes raw/full output only through explicit verbose, session, user, or channel policy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: command and tool events exist internally, while channel delivery defaults to compact progress/verbose-gated output rather than automatic full result messages.

## Solution Assessment

No: automatically pushing every raw tool result is not the best safe default because tool output may contain private data and can be noisy in shared channels. A bounded, redacted, opt-in visibility policy is the maintainable direction.
