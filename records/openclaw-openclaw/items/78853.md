## Summary

Keep open: current `main` has structured agent/tool/command-output events, progress drafts, and `/verbose full` output forwarding, but it still does not implement the requested default automatic user-visible push of full tool results. This remains a product/security-sensitive feature request rather than a broken documented behavior or an already-implemented fix.

## What This Changes

The issue requests automatic user-visible delivery of tool and command results, or an equivalent session/channel visibility control, without requiring the agent to send a follow-up message.

## Best Possible Solution

Define an explicit, channel-safe visibility policy that reuses structured agent events and progress drafts, defaults to compact sanitized progress, and exposes raw or full tool output only through opt-in verbose/session/config controls.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and docs inspection do confirm the current behavior class: tool results are available internally and through opt-in progress/verbose paths, but full results are not automatically pushed to users by default.

## Solution Assessment

No: automatically pushing every tool result is not the safest maintainable solution because tool output can be sensitive and noisy. A bounded, redacted, opt-in visibility policy is the better direction.
