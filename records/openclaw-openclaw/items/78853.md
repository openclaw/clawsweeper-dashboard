## Summary

Keep open: current main gives users compact tool-progress previews by default and opt-in `/verbose full` output bubbles, but it does not implement automatic full tool-result delivery as a default user-visible behavior. That remaining request is a product/security decision, not a narrow bug fix.

## What This Changes

Request automatic or configurable user-visible delivery of tool results after tool execution without relying on the agent’s own follow-up message.

## Best Possible Solution

Keep compact progress previews and `/verbose full` as the current safe behavior; if maintainers want more visibility, define an explicit channel-safe policy for sanitized full-result delivery with opt-in defaults and per-surface UX rules.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms the relevant current behavior: structured tool/command-output events exist and some channels render compact progress, while full output delivery is gated behind `/verbose full` instead of being automatically pushed by default.

## Solution Assessment

No: automatically pushing every full tool result is not clearly the safest solution because tool output can be noisy or sensitive. A maintainer-designed opt-in/sanitized visibility policy is the maintainable direction if the current progress and verbose modes are not enough.
