## Summary

Keep open: current main has internal tool/command-output events and channel tool-progress previews, but it does not implement the requested automatic user-visible push of full tool results. This is a product/security-sensitive feature request, not a stale or already-implemented bug.

## What This Changes

The issue requests automatic user-visible delivery or status indication for tool command results instead of requiring the agent to send a follow-up message.

## Best Possible Solution

Define a channel-safe tool-result visibility policy that reuses the existing structured tool and command_output events, defaults to sanitized progress/status, and only exposes raw output through explicit user or session policy.

## Reproduction Assessment

Source inspection gives a medium-confidence reproduction of the current behavior: tool results and command output are emitted internally, while channel delivery renders compact progress/status rather than automatically sending full results. I did not run a live chat scenario, and the report lacks affected channel/config details.

## Solution Assessment

No: automatically pushing every tool result is not clearly the best safe solution because tool output may contain private data and can be noisy in shared channels. A maintainer-designed sanitized or opt-in visibility policy is the safer direction.
