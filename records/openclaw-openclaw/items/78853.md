## Summary

Keep open: current main has internal command_output events and channel progress drafts that can show compact tool/exec progress, but it still does not define the requested automatic, user-visible delivery policy for full tool results across channels. This is a product/security-sensitive feature request rather than a broken documented contract.

## What This Changes

Request automatic user-visible delivery of tool and exec results, or an equivalent progress/status surface, without requiring the agent to send a follow-up message.

## Best Possible Solution

Define a channel-safe tool-result visibility policy that reuses existing structured tool/item/command_output events, defaults to compact sanitized progress, and exposes raw or full output only through explicit user/session/channel policy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms current main has event and progress-preview plumbing, but not a documented automatic full-result user-delivery contract for all tool results.

## Solution Assessment

No: blanket automatic push of every tool result is not the safest solution because tool output may be sensitive or noisy. The maintainable path is an explicit sanitized/opt-in visibility policy layered on the existing structured events.
