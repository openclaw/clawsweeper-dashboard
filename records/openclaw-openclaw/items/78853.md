## Summary

Keep open: current main has internal `command_output` events, Control UI tool-output cards, and channel progress drafts, but it still does not define the requested automatic user-visible delivery policy for full tool results across channels. This is a product/security-sensitive feature request, not a broken documented contract.

## What This Changes

Request automatic user-visible delivery of tool and command results instead of requiring the agent to send a follow-up message.

## Best Possible Solution

Define a channel-safe tool-result visibility policy that reuses existing structured tool/item/command_output events, defaults to compact sanitized progress, and exposes raw or full output only through explicit user/session/channel policy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms current main has event and progress-preview plumbing, but not a documented automatic full-result user-delivery contract across channels.

## Solution Assessment

No: blanket automatic push of every tool result is not the safest solution because tool output may be sensitive or noisy. The maintainable path is an explicit sanitized or opt-in visibility policy layered on the existing structured events.
