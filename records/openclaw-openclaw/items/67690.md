Keep open: current main has partial documentation, configurable cache-TTL pruning, a persistent chat context usage indicator, and compaction status toasts, but it still lacks pruning-specific preview/history, lifecycle metadata, notifications, and the broader keyword/tag/manual strategy controls requested here.

The issue requests better visibility and user control for context pruning, including pruning previews/logs, notifications, token/context indicators, and richer pruning strategy configuration.

## Best Possible Solution

Define a privacy-safe pruning metadata/event contract first, then layer Control UI/chat notices, public docs, optional redacted history, and approved strategy controls on top while leaving narrower overflow and token-quota work in their canonical threads.
