## Summary

Keep open: current main has session-pruning docs, cache-TTL pruning knobs, inline tool-result trim markers, `/context` diagnostics, and a Control UI context-usage notice, but it still lacks the requested pruning preview/history, pruning notifications, richer strategy controls, and a landed pruning metadata hook.

## What This Changes

Request better user visibility and control for context pruning, including preview/logging, configurable retention strategy, UI usage indicators, and pruning notifications.

## Best Possible Solution

Define a privacy-preserving pruning metadata/event contract first, then layer UI/chat notices, docs, and any strategy controls or split them into the existing narrower issues.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has partial context/pruning visibility but not preview/history, notifications, or richer pruning controls.

## Solution Assessment

Unclear until maintainers choose the product and privacy boundary. The maintainable direction is summary-only pruning metadata/events first, not raw pruned-output previews by default.
