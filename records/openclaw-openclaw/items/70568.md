## Summary

Keep open: the reported Telegram approval fan-out is still source-reproducible on current main, but this PR should not merge as-is because it has blocking routing regressions and still lacks after-fix real Telegram proof.

## What This Changes

The PR changes Telegram exec approval account matching and tests, removes the shared cross-channel approval-account fallback, and adds a local tsgo NODE_OPTIONS default.

## Best Possible Solution

Replace this branch with a narrower Telegram-only ambiguity guard that preserves current target-account scoping and cross-channel session bindings, then validate it with focused tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level: current main still only ambiguity-guards unbound foreign-channel requests, so a `turnSourceChannel: "telegram"` request without a bound account can pass the permissive account match for multiple eligible Telegram accounts. I did not run a live Telegram setup.

## Solution Assessment

No. The narrow ambiguity-guard direction is plausible, but this patch also suppresses `both` mode session delivery, removes shared cross-channel session binding globally, and mixes in unrelated local-check memory policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve session delivery in both mode:** `extensions/telegram/src/exec-approvals.ts:88-89`
  - body: `mode: "both"` is supposed to include the session/origin path, but this gate treats it like pure `targets` mode and can return false before session matching when targets are non-Telegram or for another Telegram account. Limit the target-account gate to the target-forwarding path so valid Telegram-native session approvals still route.
  - confidence: 0.88
- **[P2] Preserve cross-channel session account bindings:** `src/infra/approval-request-account-binding.ts:102`
  - body: Returning `null` for every cross-channel turn source discards the persisted same-channel session binding used to route native approvals back to the expected account. This regresses current tested behavior across native approval channels; keep the fallback or make the ambiguity guard Telegram-local.
  - confidence: 0.84
