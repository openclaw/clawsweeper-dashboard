## Summary

Keep open: the same-channel unbound Telegram approval gap is still source-visible on current main, but this PR is not mergeable as-is because it introduces approval-routing regressions and still lacks real Telegram behavior proof.

## What This Changes

The PR broadens Telegram exec approval ambiguity checks, adds multi-account approval tests, removes the shared cross-channel approval-account fallback, and changes local tsgo NODE_OPTIONS defaults.

## Best Possible Solution

Keep the shipped target-account scoping, replace this branch with a narrower Telegram-only ambiguity guard that preserves `both` mode and shared account bindings, then validate it with focused tests and real Telegram proof.

## Reproduction Assessment

Yes, at source level: current main only ambiguity-guards unbound foreign-channel requests, so a `turnSourceChannel: "telegram"` request without a bound account can still pass the permissive account match. I did not run a live Telegram repro.

## Solution Assessment

No. The narrow ambiguity-guard direction is plausible, but this patch also gates `both` mode session handling, removes cross-channel session binding globally, and mixes in unrelated local-check memory policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve session handling in both mode:** `extensions/telegram/src/exec-approvals.ts:89-92`
  - body: `matchesTelegramExplicitTargetAccount` runs for `mode: "both"` before normal session/origin matching, so configs that forward targets only to another channel or another Telegram account can suppress valid Telegram-native session approvals. Restrict this gate to pure `targets` mode or the target-forwarding path.
  - confidence: 0.88
- **[P2] Preserve cross-channel session account bindings:** `src/infra/approval-request-account-binding.ts:102`
  - body: Replacing the stored session-binding fallback with `return null` removes shared account routing for every native approval channel when the request came from another channel but the session store identifies the expected channel account. Keep the fallback or make the new ambiguity guard Telegram-local.
  - confidence: 0.83
