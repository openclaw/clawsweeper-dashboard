## Summary

Kept open: this PR should not merge as-is because it reintroduces a recently removed Telegram native draft transport, conflicts with current Telegram streaming guidance, and source/dependency inspection shows final delivery can be marked complete without a durable Telegram message.

## What This Changes

The PR adds an opt-in Telegram native `sendMessageDraft` preview transport via `streaming.nativeTransport`, wires DM-only dispatch, broadens config schema/types, and adds unit coverage.

## Best Possible Solution

Keep the current message/edit finalization path unless maintainers reopen the product direction in the linked feature request; any future native draft attempt must finalize through durable `sendMessage` with live Telegram gateway proof and no duplicate final bubble.

## Reproduction Assessment

Yes for the blocking review findings: source inspection of the PR draft path, current lane delivery, and the Telegram Bot API contract shows final native draft delivery can be retained without a durable message. I did not run a live OpenClaw Telegram gateway reproduction.

## Solution Assessment

No. The PR wires an ephemeral preview API into a final-delivery contract that expects a durable message id; the safer maintainable path is to keep edit-in-place behavior or redesign finalization only after explicit Telegram owner approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Persist native drafts before marking finals delivered:** `extensions/telegram/src/draft-stream.ts:203-210`
  - body: `sendMessageDraft` returns success for an ephemeral preview, and Telegram requires a separate `sendMessage` after generation finishes. This path never creates a message id, so current lane delivery can treat the final as `preview-retained` and finish without a permanent Telegram reply.
  - confidence: 0.95

  - confidence: 0.89

  - body: CI reports this expression as an unnecessary assertion, which fails both the main lint shard and bundled-extension lint. Drop the `as "draft" | "message"` assertion or type the function return explicitly so the PR can pass lint.
  - confidence: 0.98
