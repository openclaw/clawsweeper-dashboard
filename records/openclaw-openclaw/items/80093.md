## Summary

Keep open: this is a plausible feature direction but the PR is not merge-ready. ## What This Changes

The PR adds an opt-in Telegram `sendMessageDraft` preview path via `channels.telegram.streaming.transport`, adds draft-stream tests, and also changes Fal image edit routing and payload behavior.

## Best Possible Solution

Resolve the product decision on https://github.com/openclaw/openclaw/issues/79784 first; if maintainers want native drafts back, land a Telegram-only implementation with schema/docs/generated config updates, private-chat fallback, regression coverage, and redacted live Telegram proof.
