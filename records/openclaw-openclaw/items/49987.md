## Summary

Keep open: the requested CLI restart notification is not implemented on current main, but this PR still has blocking lifecycle defects and lacks required real behavior proof.

## What This Changes

The PR adds restart-sentinel imports to `src/cli/daemon-cli/lifecycle.ts` and writes a `Gateway restarted via CLI` payload after `runServiceRestart()` returns true.

## Best Possible Solution

Add a narrow daemon restart lifecycle hook that prepares the sentinel after restart preflight commits to a real restart but before the replacement gateway can pass startup's sentinel gate, using the daemon service environment and cleanup on no-op or failed paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR writes the sentinel after `runServiceRestart()` returns, while startup only schedules delivery if the sentinel file already exists during `sidecars.restart-sentinel`; I did not run a live gateway restart in this read-only review.

## Solution Assessment

No. The requested capability is useful, but this implementation is not the best fix because it writes too late for startup delivery and uses the invoking CLI environment instead of the daemon's persisted state/config environment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move the sentinel write before the startup gate can pass:** `src/cli/daemon-cli/lifecycle.ts:284`
  - body: The new success sentinel is written only after `runServiceRestart()` returns. By then the replacement gateway can already have passed the startup sidecar that checks for `restart-sentinel.json`, so the notification can be missed until a later unrelated start.
  - confidence: 0.92
- **[P2] Write the sentinel with the daemon environment:** `src/cli/daemon-cli/lifecycle.ts:284`
  - body: `writeRestartSentinel(sentinelPayload)` defaults to the CLI process environment. Installed gateways can run with persisted `OPENCLAW_STATE_DIR` or `OPENCLAW_CONFIG_PATH`, so custom-state/profile installs may write the file where the restarted daemon will never read it.
  - confidence: 0.9
