## Summary

Keep open: current main still lacks CLI restart notification support, but this branch has two source-backed correctness blockers and lacks required real behavior proof.

## What This Changes

The PR changes `src/cli/daemon-cli/lifecycle.ts` so `openclaw gateway restart` writes a restart sentinel after `runServiceRestart()` returns true.

## Best Possible Solution

Add a daemon-aware restart hook that records the sentinel in the daemon state after restart preflight commits to a real restart but before replacement startup can pass the sentinel gate, with cleanup for failed or no-op paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the PR defects: the branch writes the sentinel only after `runServiceRestart()` returns, while startup checks for the file during the post-ready sidecar; I did not run a live restart in this read-only review.

## Solution Assessment

No. The late write avoids stale false positives, but it can miss startup delivery and resolves the sentinel path from the CLI process instead of the daemon service environment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Write the sentinel before startup can pass the gate:** `src/cli/daemon-cli/lifecycle.ts:284`
  - body: The new `writeRestartSentinel(...)` call only runs after `runServiceRestart()` returns. That helper can restart the service, run `postRestartCheck`, inspect loaded state, and emit success before returning, while startup only schedules sentinel delivery if `restart-sentinel.json` already exists during its post-ready sidecar check; the replacement gateway can pass that gate before this line writes the file.
  - confidence: 0.92
- **[P2] Use the daemon service environment for the sentinel path:** `src/cli/daemon-cli/lifecycle.ts:284`
  - body: This call does not pass an environment, so `writeRestartSentinel` resolves the file path with `process.env`. Installed gateways can run with persisted `OPENCLAW_STATE_DIR` and `OPENCLAW_CONFIG_PATH` from the service command; a CLI shell using a different profile or state dir can write a sentinel the restarted daemon will never read.
  - confidence: 0.9
