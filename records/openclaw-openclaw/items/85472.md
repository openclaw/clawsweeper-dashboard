## Summary

Keep open: the PR addresses a real WebChat transcript persistence gap with strong proof, but the current patch still adds a gateway durable user write in a helper used by SessionManager-backed agent runs, which can duplicate or mis-chain persisted user turns.

## What This Changes

The branch changes WebChat `chat.send` to run `before_message_write`, append the user message to session JSONL with `appendSessionTranscriptMessage`, emit the appended message, and add parentId/hook fixture tests.

## Best Possible Solution

Route durable WebChat user persistence by transcript owner: gateway appends only when no SessionManager-backed writer owns the prompt, while SessionManager-backed runs keep the canonical guarded write with full history coverage.

## Reproduction Assessment

Yes, for a source-level path. Current main's WebChat helper only emits a live user transcript update while SessionManager owns durable agent turns, and the PR body supplies redacted live JSONL proof of the missing user-row symptom.

## Solution Assessment

No. Persisting a user row is the right direction for gateway-owned paths, but doing it inside a helper invoked for SessionManager-backed agent runs is not the narrow maintainable fix.
