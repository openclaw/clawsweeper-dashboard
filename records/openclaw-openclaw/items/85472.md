## Summary

Keep open: the bug target is valid, but this revision still writes WebChat user turns from the gateway on agent-run paths that current main treats as Pi-owned, and it bypasses normal transcript write hook/provenance semantics.

## What This Changes

The branch adds a gateway-side append of WebChat user messages to session JSONL before transcript update broadcasts and adds fixture-based transcript persistence tests.

## Best Possible Solution

Persist the user turn only on no-agent or otherwise unowned transcript paths, or route all gateway-owned user persistence through a shared guarded writer that preserves SessionManager hook, provenance, redaction, idempotency, and parentId behavior.

## Reproduction Assessment

Yes for a source-level path: current main's no-agent WebChat branch emits the user transcript update but only appends the gateway assistant entry, while the PR-introduced agent-path duplicate risk is also clear from source inspection. I did not run a live WebChat reconnect proof in this read-only review.

## Solution Assessment

No. The maintainable fix needs a single transcript owner per path, plus the same guarded write semantics as SessionManager, instead of adding a broad gateway append inside the shared transcript-update notifier.
