## Summary

Keep open: the bugfix target is valid, but this patch is not merge-ready because it pre-writes user rows in SessionManager-owned agent paths and bypasses the normal transcript hook/provenance contract for user-message persistence.

## What This Changes

The branch adds a gateway-side append of WebChat user messages to session JSONL before broadcasting transcript updates, plus fixture tests for parentId-linked user-message appends.

## Best Possible Solution

Persist WebChat user turns only through the component that owns the durable transcript for that branch: SessionManager for agent-backed sends, and a gateway-owned helper with equivalent hook/provenance/idempotency semantics for no-agent sends.

## Reproduction Assessment

Yes for a source-level path: current main's no-agent WebChat branch emits the user transcript update but only appends the gateway assistant entry to disk. I did not run a live WebChat reconnect reproduction in this read-only review.

## Solution Assessment

No. The patch fixes the right symptom in the no-agent branch, but the shared helper write is too broad and too low-level for agent-backed sends and plugin-governed transcript writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep agent user turns on the SessionManager writer:** `src/gateway/server-methods/chat.ts:2684-2690`
  - body: This append now runs from the shared `emitUserTranscriptUpdate` helper, including normal agent-backed sends. Those sends still go through Pi's `AgentSession.prompt`, which builds a user message and persists it through the guarded `SessionManager.appendMessage` path; pre-writing here makes the new row an orphan/sibling that the run code has to branch around, instead of the single active transcript user turn. Scope this disk append to the no-agent/no-SessionManager branch, or otherwise make one writer own the durable user row.
  - confidence: 0.88
- **[P1] Preserve transcript hooks for gateway user writes:** `src/gateway/server-methods/chat.ts:2684-2690`
  - body: `appendSessionTranscriptMessage` handles parentId and redaction, but it does not run `before_message_write` or apply input provenance like the guarded SessionManager path does. In the no-agent branch this new append becomes the only persisted user row, so plugins that block, redact, or annotate user transcript writes are bypassed. Route gateway-owned user persistence through the same hook/provenance contract or an equivalent shared helper before writing JSONL.
  - confidence: 0.86
