## Summary

Keep open. The WebChat bug target is source-plausible, but the PR is not merge-ready because the new disk append runs through agent-run paths already owned by SessionManager, the branch also carries unrelated prompt-hash CLI logging from a separate open PR, and there is no after-fix WebChat proof.

## What This Changes

The branch appends WebChat user messages to session transcript JSONL before live transcript events and also adds Claude CLI turn-correlation logging with tests.

## Best Possible Solution

Keep the WebChat persistence fix, but scope disk writes to paths not already owned by SessionManager, add JSONL/history regression coverage proving exactly one user turn, split the CLI logging work back to its own PR, and require redacted real WebChat proof before merge.

## Reproduction Assessment

Yes, source-level: current main emits live WebChat user transcript updates while chat.history is rebuilt from durable JSONL, and the non-agent branch appends assistant output without an equivalent user-row disk write. I did not run a live WebChat reconnect proof in this read-only review.

## Solution Assessment

No. The proposed append helper is plausible for no-agent/gateway-injected paths, but applying it inside the shared live-update helper crosses the agent transcript owner boundary and the branch includes unrelated CLI logging changes.
