## Summary

Keep open: the WebChat transcript bug is plausible, but this PR appends the user row on agent-run paths that current code already persists through the agent transcript owner, so it can duplicate user turns; it also bundles unrelated Claude CLI prompt-hash logging and has no real behavior proof.

## What This Changes

The PR adds a direct WebChat user-message transcript append before live transcript notifications and also adds Claude CLI turn-correlation logging/tests.

## Best Possible Solution

Keep the WebChat disk persistence fix, but scope it to paths not already persisted by SessionManager, add JSONL/history regression coverage that proves no duplicate agent-run user turn, and split or drop the unrelated CLI logging changes unless separately approved.

## Reproduction Assessment

Yes for the source-level non-agent WebChat path: current main emits live user transcript updates while chat.history is rebuilt from durable JSONL, and the no-agent branch appends assistant output without an obvious user-row disk write. I did not run a live WebChat reconnect proof in this read-only review.

## Solution Assessment

No. The PR uses a plausible append helper, but applying it to every emitUserTranscriptUpdate path ignores existing agent transcript ownership and bundles unrelated CLI logging, so it is not the narrow maintainable fix yet.
