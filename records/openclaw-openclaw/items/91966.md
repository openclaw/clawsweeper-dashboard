## Summary

This is a focused, correct fix for a current-main regression: indexed or bracket-suffixed consent currently widens into authorization to replace an entire config array, contrary to the documented exact-path contract. The implementation and supplied live Gateway proof support landing after maintainers acknowledge the intentional fail-closed compatibility tightening.

## What This Changes

This PR stops indexed or bracket-suffixed `replacePaths` values from becoming bare whole-array replacement consent and updates the agent-tool normalization test.

## Best Possible Solution

Land the minimal normalization correction while preserving the documented distinction that only bare array paths authorize destructive replacement and indexed input remains entry-scoped.

## Reproduction Assessment

## Solution Assessment

Yes. Removing only the terminal-bracket collapse is the narrowest maintainable fix because it restores exact-path consent while preserving interior index normalization, nested entry paths, and id-keyed merging.
