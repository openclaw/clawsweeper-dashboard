## Summary

The reported consent-widening bug is real on current `main`, and this narrowly scoped patch restores the documented exact-path boundary without disrupting nested index normalization. The supplied live Gateway proof is convincing; the only remaining quality gap is that the exact suffix-bypass matrix is not committed as regression coverage.

## What This Changes

The PR preserves a terminal normalized array bracket in `config.patch` `replacePaths`, preventing index-scoped tokens from authorizing whole-array replacement.

## Best Possible Solution

Keep the one-line normalization correction and add focused regression coverage proving canonical bare-array consent succeeds, terminal index/bracket tokens fail closed, and interior index normalization remains supported.

## Reproduction Assessment

Yes. Current source deterministically collapses terminal index/bracket tokens onto the bare array path, and the supplied real Gateway matrix demonstrates the resulting destructive write before the patch.

## Solution Assessment

Yes. Removing only terminal-bracket stripping is the narrowest maintainable fix because it restores exact consent while preserving the established normalization for indices inside nested paths.
