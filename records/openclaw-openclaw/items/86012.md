## What This Changes

The issue reports LINE channel replies being silently lost when delayed agent responses miss LINE's reply-token window and the push fallback lacks retry, batching, quota, and error-visibility safeguards.

## Best Possible Solution

Merge a reviewed and real-proven LINE plugin fix that keeps reply-token use within LINE's contract or falls back observably with batched, retry-keyed push delivery and quota-aware diagnostics, then close this issue through the linked PR.
