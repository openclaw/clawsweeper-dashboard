- `P1`: The issue reports silent loss of user-facing LINE agent replies, which is a broken channel workflow affecting real users.

## Summary

Keep open: current main and latest release still contain the central LINE reply-token fallback gaps, and the linked closing PR is open rather than merged.

## What This Changes

Fix the LINE channel so replies that miss the one-minute reply-token window do not silently disappear and push fallback has batching, retry, and quota safeguards.

## Best Possible Solution

Review and land a corrected, real-proven LINE plugin fix or narrow replacement for reply-error propagation, batched and retried push fallback, retry keys, and quota diagnostics.

Yes at source level: current main can delay final LINE reply delivery until after agent processing, omits onReplyError from the main text chunk path, and lacks retry/quota safeguards in push fallback. I did not run a live LINE webhook reproduction in this read-only review.
