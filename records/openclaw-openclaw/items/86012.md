## Summary

Keep open: current main and v2026.6.8 still have the LINE reply-token and push-fallback safeguard gaps, while https://github.com/openclaw/openclaw/pull/94680 is the active open candidate fix rather than a merged implementation.

## What This Changes

Fix LINE channel delivery so replies delayed past LINE reply-token expiry fall back to resilient Push delivery instead of silently losing messages when push quota or rate errors occur.

## Best Possible Solution

Land a corrected, real-LINE-proven LINE plugin fix or narrow replacement that adds reply-error propagation, batched and retried Push fallback, retry keys, quota visibility, and loading behavior without creating a competing source of truth.

## Reproduction Assessment

Yes for source-level reproduction: current main still calls the main text sendLineReplyChunks path without onReplyError and still sends Push fallback chunks one at a time with no retry or quota safeguard. I did not run a live LINE account reproduction in this read-only review.
