## Summary

Keep open: current main and latest release still show the LINE reply-token observability, one-at-a-time push fallback, and missing retry/quota safeguards, while the linked closing PR remains open and not merge-ready.

## What This Changes

The issue asks for LINE channel safeguards so delayed agent replies do not disappear when reply tokens expire and push fallback is rate-limited or quota-exhausted.

## Best Possible Solution

Land a reviewed, real-proven LINE plugin fix that propagates reply errors, batches and retries push fallback safely, uses retry keys where appropriate, and adds quota-aware diagnostics, then close this issue through the merged PR.
