Keep open: current main and the latest release still contain the LINE reply-token error-propagation, one-at-a-time push fallback, and missing retry/quota safeguards, while the linked fix PR remains open and not merge-ready.

## What This Changes

The issue reports silent LINE reply loss when long-running agent responses outlive LINE reply tokens and push fallback lacks retry, batching, quota, and observability safeguards.

## Best Possible Solution

Land or replace the linked LINE plugin fix with real LINE proof covering delayed replies and fallback delivery, then close this issue through the merged PR.
