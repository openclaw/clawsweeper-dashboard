## Summary

## What This Changes

The PR scopes `tools.effective` session loading and session-agent resolution with the requested agent id and adds unit plus integration coverage for a global session on a non-default agent.

## Best Possible Solution

Keep the selected-global store scoping fix, but narrow the session-agent override to canonical global keys, add a mismatched non-global regression, and require redacted real Gateway or Control UI proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main loads `tools.effective` sessions without the requested agent, and the patch regression is source-reproducible because `resolveSessionAgentId` prefers explicit `agentId` before parsing the session key.

## Solution Assessment

No. Passing the requested agent into `loadSessionEntry` is the right narrow fix for selected global sessions, but passing it unconditionally into `resolveSessionAgentId` is too broad for non-global session keys.
