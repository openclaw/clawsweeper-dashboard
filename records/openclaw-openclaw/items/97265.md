## Best Possible Solution

Land the focused gateway fix after maintainer review and normal required checks, keeping global store selection tied to the existing session-store resolver and preserving non-global ownership guards.

## Reproduction Assessment

Yes at source level: current main validates the requested agent but loads and resolves `global` without that agent, causing a non-default requested agent to mismatch the default session agent. I did not execute tests in this read-only review.

## Solution Assessment

Yes. Passing the requested agent into existing session-store selection, then allowing the session-agent override only for canonical `global`, is the narrow maintainable fix and avoids a non-global ownership bypass.
