## Summary

Keep open: the useful global-session scoping fix is still paired with an unconditional client-selected agent override that weakens the non-global session ownership guard, and contributor proof remains test-only.

## What This Changes

The PR forwards `agentId` through `tools.effective` session loading/session-agent resolution and adds unit plus integration tests for global sessions scoped to a non-default agent.

## Best Possible Solution

Keep the selected-global store scoping, but only pass an agent override into session-agent resolution after proving the canonical key is global or after validating non-global key/agent consistency; add the mismatched non-global regression and real behavior proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main loads `tools.effective` for `global` without forwarding `agentId`, and the PR regression is source-proven because `resolveSessionAgentId` prefers explicit `agentId` before parsing the session key.

## Solution Assessment

No. Passing the requested agent to `loadSessionEntry` matches the store target contract, but passing it unconditionally to `resolveSessionAgentId` is too broad because that helper treats explicit agent as authoritative.
