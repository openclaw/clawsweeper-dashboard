## What This Changes

The PR edits three browser docs pages to qualify `browser wait --load networkidle` support as managed-profile-only and existing-session-unsupported.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/81577 as the canonical fix, landing wording that says existing-session/Chrome MCP profiles reject `networkidle` while managed and remote-CDP Playwright-backed profiles support it.

## Reproduction Assessment

Yes. Reading `docs/tools/browser-control.md:295` and `docs/tools/browser.md:649` on current main reproduces the contradiction, and source inspection confirms the unsupported case is the existing-session/Chrome MCP path.

## Solution Assessment

No for this PR. The docs-only direction is right, but this patch says managed profile only even though remote-CDP profiles use the Playwright path; the superseding PR is the narrower correct solution.
