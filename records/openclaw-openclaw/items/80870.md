## What This Changes

The PR updates three browser docs pages to qualify `browser wait --load networkidle` as supported for managed `openclaw` profiles and unsupported for existing-session profiles.

## Best Possible Solution

Keep the canonical docs clarification in https://github.com/openclaw/openclaw/pull/80678 and document the unsupported case as existing-session/Chrome MCP while preserving Playwright-backed managed and remote CDP support.

## Reproduction Assessment

Yes. Reading current main reproduces the docs ambiguity, and source plus tests confirm the correct boundary is existing-session/Chrome MCP rather than all non-managed profiles.

## Solution Assessment

No. A docs-only clarification is the right kind of fix, but this PR's managed-only wording is not the narrowest correct boundary and the open canonical PR is a better tracking change.
