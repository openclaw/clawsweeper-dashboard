## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Deep-merge inherited provider settings:** `ui/src/ui/views/agents-panels-overview.ts:63-65`
  - body: `resolveAgentTts` still assigns top-level override keys, so an agent with `tts.providers.elevenlabs.speakerVoiceId` replaces the global `messages.tts.providers.elevenlabs` object instead of preserving inherited `apiKey`/model fields. Runtime uses recursive `resolveEffectiveTtsConfig` semantics for this path, so the card can falsely show a valid global SecretRef or API key as missing for normal per-agent voice overrides.
  - confidence: 0.9
- **[P2] Honor TTS auto mode in the toggle:** `ui/src/ui/app-render.ts:3485-3488`
  - body: The new toggle reads and writes only `enabled`, but `enabled` is deprecated and runtime gives `auto` precedence. With the documented `messages.tts.auto: "always"` config, the checkbox renders off when `enabled` is absent, and toggling off writes `enabled: false` without disabling the inherited `auto`, so users can believe agent TTS is disabled while runtime still speaks.
  - confidence: 0.88
