Full review comments:

- **[P1] Preserve SecretRef-backed TTS API keys:** `ui/src/ui/views/agents-panels-overview.ts:46`
  - body: This casts providers.elevenlabs.apiKey to a string for a password input, but current TTS provider config accepts structured SecretInput objects. A valid SecretRef at this path can be rendered as a non-string value and then replaced by the direct write handler, breaking credential resolution; preserve structured secrets or route credential editing through the existing secret-aware config path.
  - confidence: 0.92
- **[P2] Write canonical TTS auto mode:** `ui/src/ui/app-render.ts:3487-3488`
  - body: The toggle writes agents.list[].tts.enabled, but TtsConfig marks enabled as deprecated and doctor migrates it to auto. New Control UI saves should write agents.list[].tts.auto, and the card should render auto values like always, inbound, tagged, and off instead of treating auto-enabled configs as disabled.
  - confidence: 0.88
- **[P2] Render effective TTS from runtime config:** `ui/src/ui/views/agents-panels-overview.ts:39`
  - body: resolveAgentTts chooses only the agent/default block, but runtime TTS starts from messages.tts and deep-merges per-agent overrides. Existing configs can keep auto mode and provider credentials globally while agents override only voice fields, so this card can show TTS as disabled or incomplete for a valid runtime setup.
  - confidence: 0.9
- **[P2] Write ElevenLabs modelId so the selector affects synthesis:** `ui/src/ui/app-render.ts:3561`
  - body: The model handler writes providers.elevenlabs.model, but the ElevenLabs speech provider reads modelId when building the synthesis request. With this path, changing the UI selector can leave runtime synthesis on the provider default model.
  - confidence: 0.86
