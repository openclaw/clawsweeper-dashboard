## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve inbound and tagged TTS modes:** `ui/src/ui/app-render.ts:3491-3492`
  - body: Current TTS config supports inbound and tagged, and speech-core treats both differently from always. This checkbox renders either mode as enabled but writes always when the user turns it on, so an existing tagged or inbound agent can silently start speaking on every reply after a UI edit.
  - confidence: 0.91
- **[P2] Separate inherited provider state from explicit overrides:** `ui/src/ui/views/agents-panels-overview.ts:101`
  - body: resolveAgentTts returns the merged provider and the select uses that as its value, so a provider inherited from messages.tts appears as an explicit per-agent choice. Keep the stored agent override separate from the effective provider so users can see and restore inheritance without the select snapping back to the inherited value.
  - confidence: 0.82
- **[P2] Respect SecretInput shorthands before editing API keys:** `ui/src/ui/views/agents-panels-overview.ts:116-117`
  - body: TTS apiKey is a SecretInput and the docs use env-template strings such as ${ELEVENLABS_API_KEY}, but this resolver treats every string as an editable plaintext password while only object SecretRefs are read-only. Parse SecretInput shorthands or otherwise show managed env refs distinctly so a normal edit does not turn a reference into a literal per-agent override.
  - confidence: 0.78
- **[P3] Avoid nesting the toggle label:** `ui/src/ui/views/agents-panels-overview.ts:371-373`
  - body: The new label.field wraps another label.toggle-switch, which is invalid HTML and can make click and focus targeting ambiguous. Make the inner toggle wrapper a non-label element while keeping the checkbox associated with visible text.
  - confidence: 0.74
