## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve SecretRef-backed TTS API keys:** `ui/src/ui/views/agents-panels-overview.ts:46`
  - body: `apiKey` is a supported `SecretInput`, but this UI casts it to a string and binds it into a password value. A config that already uses a SecretRef can therefore be rendered through a plain field and replaced or removed by the new write handler, so the card needs a secret-aware display/update path before merge.
  - confidence: 0.91
- **[P2] Render effective TTS instead of only agent/default blocks:** `ui/src/ui/views/agents-panels-overview.ts:39`
  - body: Runtime TTS starts from `messages.tts` and deep-merges per-agent overrides, but this resolver only chooses `agents.list[].tts` or `agents.defaults.tts`. Existing valid setups that keep auto mode or credentials globally can look disabled or incomplete and be overwritten from the new form.
  - confidence: 0.89
- **[P2] Write ElevenLabs modelId so the selector affects synthesis:** `ui/src/ui/app-render.ts:3561`
  - body: The new handler writes `providers.elevenlabs.model`, while the ElevenLabs provider config reads `modelId` before passing the value to synthesis. Selecting a model in this UI can leave runtime speech on the provider default until the field is aligned or a tested migration/compat path is added.
  - confidence: 0.9
