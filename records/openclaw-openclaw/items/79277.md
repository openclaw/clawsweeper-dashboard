## Summary

Keep open: current main still lacks automatic Feishu read receipts, but the repo docs/schema do not define this as existing behavior, so this is a Feishu plugin UX feature/default-policy decision rather than a closeable implemented bug.

## What This Changes

The issue asks for Feishu/Lark inbound messages to be automatically marked read after the bot receives and processes them.

## Best Possible Solution

Add Feishu plugin-owned read-receipt support only after maintainers choose the default/config behavior, then implement it in the inbound receive path with Feishu API error handling, regression coverage, docs, and a changelog entry.

## Reproduction Assessment

Partly. Source inspection on current main shows no mark-read/read-receipt call in the Feishu inbound receive or dispatch path, but I did not establish a live Feishu UI reproduction of the unread indicator.

## Solution Assessment

No. The suggested unconditional API call is not yet the best maintainable solution; a safer path is a Feishu plugin-owned helper with maintainer-approved default/config semantics, permission checks, backoff/error handling, docs, and tests.
