## What This Changes

This PR adds `OPENCLAW_TSDOWN_MAX_OLD_SPACE_MB` to override the tsdown build heap cap and adds resolver tests for precedence, inherited `NODE_OPTIONS`, and malformed values.

## Best Possible Solution

Land the focused resolver and test change only if maintainers accept this build-only env knob as supported override surface; otherwise keep automatic memory detection plus `NODE_OPTIONS` normalization as the canonical path.

## Reproduction Assessment

Not applicable: this PR adds a new build-time override rather than reporting a current-main bug. Source inspection confirms current main and the latest release lack a dedicated tsdown heap override and rely on platform, cgroup/proc, and `NODE_OPTIONS` normalization paths.

## Solution Assessment

Yes technically, if maintainers want the new operator knob: `resolveTsdownMaxOldSpaceMb` feeds both pnpm and no-pnpm tsdown invocations, so this is the narrow owner-boundary change. Product-wise it still needs acceptance because adding an env variable is not purely mechanical.
