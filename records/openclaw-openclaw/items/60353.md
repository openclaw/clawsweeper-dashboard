## Summary

Keep open; current main and the latest release still contain the Slack starter-media fallback, while this branch removes a documented first-turn root-media contract and lacks inspectable Slack proof.

## What This Changes

The PR changes Slack thread preparation so replies never hydrate media from the thread starter, removes the starter-media fallback from prepared media selection, and adds focused tests for starter-media suppression.

## Best Possible Solution

Land a rebased Slack fix that stops repeated starter-media rehydration after thread context exists while preserving reply-owned media and documented first-turn root-file context, or explicitly update the Slack contract and docs if maintainers choose never-hydrate.

## Reproduction Assessment

Yes, at source level. Current main hydrates Slack starter files into `threadStarterMedia` for a text-only thread reply with no direct media, then forwards that fallback as effective inbound media.

## Solution Assessment

No, not as submitted. The broad never-hydrate fix removes a documented first-turn/root-media path; the safer path is to stop repeated hydration after session context exists or make an explicit maintainer-approved contract change.
