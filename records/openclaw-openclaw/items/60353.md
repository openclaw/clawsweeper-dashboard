## Summary

Keep open. Current main still hydrates Slack thread-starter files and then falls back to that media for text-only thread replies, so the PR fixes a real bug, but the submitted branch still needs the matching public Slack docs and active changelog entry before merge.

## What This Changes

The PR changes Slack thread preparation so thread replies stop using thread-starter media as fallback media, and adds regression tests for first-turn, later-turn, and own-media replies.

## Best Possible Solution

Land the Slack-plugin behavior change with updated Slack media docs and an active changelog fix entry, preserving the linked issue and human contributor credit.

## Reproduction Assessment

Yes. On current main, a text-only Slack thread reply with a root message that has files follows `resolveSlackMedia(starter.files)` and then uses `threadStarterMedia` as fallback payload media.

## Solution Assessment

No, not as submitted. The code direction is the narrow owner-plugin fix, but the complete solution must also update public Slack media docs and the active changelog; no core seam is needed.
