## Summary

Keep open. Current main still has the Slack thread-starter media rehydration path, but this PR is not merge-ready because it removes a documented first-turn/file-only root-media contract, is conflicting with current main, has typecheck blockers, and lacks inspectable real behavior proof.

## What This Changes

This PR changes Slack thread preparation to skip thread-starter media hydration for all thread replies, remove the effective-media fallback to starter media, and add tests asserting parent media is never rehydrated.

## Best Possible Solution

Land a rebased Slack fix that stops repeated parent-media rehydration while preserving reply-owned media and either preserving the current first-turn/file-only root-media contract or deliberately updating that contract with docs and maintainer approval.

## Reproduction Assessment

Yes, source-level. Current main hydrates starter.files into threadStarterMedia for a text-only Slack thread reply with no direct media, then forwards that fallback as effective inbound media.

## Solution Assessment

No, not as submitted. The Slack fix belongs in this preparation path, but the unconditional never-hydrate policy is broader than the documented contract unless maintainers intentionally change that behavior.
