## Summary

Keep open. Current main still reproduces the Slack parent-media hydration path, and the PR's behavior fix is useful but incomplete because the public Slack docs still describe the old hydration behavior and the required changelog entry is missing.

## What This Changes

The branch removes Slack thread-starter media hydration for thread replies, switches prepared Slack replies to direct reply media only, and adds targeted regression tests.

## Best Possible Solution

Land the Slack-plugin behavior change after adding the matching Slack media docs and active changelog fix entry, either by updating this branch or by opening a narrow replacement PR that preserves the contributor credit.

Yes. On current main, a Slack thread reply with no direct media and a root message with files follows `resolveSlackMedia(starter.files)` in `prepare-thread-context.ts`, then `prepare.ts` uses `threadStarterMedia` as the effective media.

## Solution Assessment

No, not as submitted. The code direction is the narrow maintainable fix for the media bleed-through, but the complete solution must also update the public Slack media docs and add the required changelog entry.
