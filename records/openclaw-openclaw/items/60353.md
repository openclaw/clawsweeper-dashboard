## Summary

Keep open: this PR targets a real Slack thread-media bug still present on current main, but the branch is conflicting and not mergeable as submitted because it leaves typecheck-breaking unused bindings, misses docs alignment, and lacks inspectable real behavior proof.

## What This Changes

The branch stops Slack thread replies from falling back to thread-starter media and adds thread-context tests for first replies, later replies, and replies with their own media.

## Best Possible Solution

Rebase or replace this with a current-main Slack plugin fix that removes parent-media fallback for replies, preserves reply-owned media, updates the Slack docs contract, and includes focused tests plus redacted live proof.

## Reproduction Assessment

Yes, source-level. On current main, a text-only Slack thread reply with starter files and no direct media hydrates `starter.files` into `threadStarterMedia`, then forwards that fallback through inbound media fields.

## Solution Assessment

No, not as submitted. The direction is narrow and appropriate, but the PR needs a current-main rebase, unused binding/import cleanup, docs alignment, and inspectable live proof before merge.
