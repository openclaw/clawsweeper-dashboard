## Summary

Keep open. Current main still has a source-level Slack thread media bleed-through path, and this PR targets it with focused code and tests, but it should not merge until the Slack media docs are aligned and the external contributor adds inspectable real behavior proof.

## What This Changes

The PR removes Slack thread-starter media hydration for thread replies, makes prepared Slack replies use only direct reply media, and adds targeted regression tests.

## Best Possible Solution

Land this PR or a narrow replacement that stops reattaching parent media while preserving reply-owned media, and update the Slack media docs to the final intended thread-context contract.

## Reproduction Assessment

Yes, source-level. On current main, a text-only Slack thread reply with a starter file and no direct media hydrates `starter.files`, then forwards `threadStarterMedia` through the inbound media fields.

## Solution Assessment

Not yet as submitted. The code direction is a plausible narrow fix for the bleed-through, but the PR also changes documented Slack behavior and needs matching docs plus stronger real behavior proof before merge.
