## Summary

Keep open: the Slack media bug is source-reproducible, but this PR is conflicting, lacks inspectable real Slack proof, and removes a documented shipped thread-root media inheritance path instead of only narrowing repeated hydration.

## What This Changes

The branch changes Slack thread preparation to never hydrate thread-starter media for replies, removes the prepared-message fallback to that media, and adds resolver tests for the never-hydrate policy.

## Best Possible Solution

Land a rebased Slack fix that prevents repeated parent-media rehydration after the intended initial thread context path while preserving reply-owned media and the documented file-only root behavior, or deliberately change that contract with docs and maintainer approval.

## Reproduction Assessment

Yes, source-level. Current main hydrates starter.files for a text-only Slack thread reply with no direct media, then forwards threadStarterMedia as the effective inbound media.

## Solution Assessment

No, not as submitted. The fix belongs in Slack thread message preparation, but the safer implementation should avoid repeated hydration without silently deleting the documented root-media inheritance path unless maintainers choose that product change.
