## Summary

Keep open. The PR targets a real Slack media/context bug, but the branch is conflicting, lacks inspectable real behavior proof, and the submitted never-hydrate approach removes a documented root-message media path.

## What This Changes

The branch changes Slack thread preparation so thread replies never resolve media from the thread starter, removes the starter-media fallback from prepared media selection, and adds tests for starter-media suppression.

## Best Possible Solution

Land a rebased Slack fix that stops repeated starter-media rehydration after thread context exists while preserving reply-owned media and the documented first-turn root-file context, or explicitly update the contract and docs if maintainers choose never-hydrate.

## Reproduction Assessment

Yes, source-level. Current main hydrates Slack starter files into threadStarterMedia for a text-only thread reply with no direct media, then forwards that fallback as effective inbound media.

## Solution Assessment

No, not as submitted. The broad never-hydrate fix removes a documented first-turn/root-media path; the safer path is to stop repeated hydration after session context exists or make an explicit maintainer-approved contract change.
