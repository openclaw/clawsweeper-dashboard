## Summary

Keep open. Current main still hydrates Slack thread-starter media for text-only thread replies, and this PR is a focused functional fix for #60335, but it is not ready as-is because the branch is dirty against current main and the public Slack docs plus required changelog entry are not updated for the changed behavior.

## What This Changes

The PR removes Slack thread-starter media hydration for Slack thread replies, uses only direct reply media in prepared inbound payloads, and adds targeted Slack thread-media regression tests.

## Best Possible Solution

Land the same Slack-plugin behavior change after rebasing or replacing the dirty branch, update the Slack media docs to state that thread replies only attach media from the reply itself while retaining starter text/history context, and add the required Unreleased changelog entry crediting the contributor.

## Reproduction Assessment

Yes. On current main, a Slack thread reply with no direct media and a starter that has files follows the resolveSlackMedia starter-files branch and then prepareSlackMessage uses threadStarterMedia as the effective media.

## Solution Assessment

No, not as-is. The code direction is the narrowest maintainable fix for the observed media bleed-through, but the complete solution needs docs/changelog updates and a clean branch against current main.
