## Summary

Keep open: the linked Slack bug is still source-reproducible on current main, but this PR should not merge as submitted because it removes the shipped first-turn thread-root media inheritance contract, has extension typecheck blockers, lacks inspectable real behavior proof, and is currently conflicting.

## What This Changes

This PR stops Slack thread replies from hydrating thread-starter media, removes the prepared-message fallback to starter media, and adds tests asserting parent media is never rehydrated.

## Best Possible Solution

Land a rebased Slack fix that stops repeated parent-media rehydration after the initial thread context has been recorded while preserving reply-owned media and the documented first-turn/file-only root media path, or deliberately update that contract with maintainer approval.

## Reproduction Assessment

Yes, source-level. Current main hydrates Slack starter files into threadStarterMedia when a text-only thread reply has no direct media, then forwards that fallback as effective inbound media.

## Solution Assessment

No, not as submitted. The narrower maintainable fix should avoid repeated downloads for existing thread sessions without removing the documented first-session root-media inheritance path, unless maintainers intentionally change that Slack behavior.
