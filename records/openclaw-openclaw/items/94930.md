## Best Possible Solution

Reproduce against current main or collect the reporter's sanitized full JSONL, then fix the embedded/WebChat gap-fill dedupe so a cache-ttl tail cannot produce a second assistant row while preserving legitimate repeated replies after user turns.

## Reproduction Assessment

No high-confidence current-main live reproduction was established here.
