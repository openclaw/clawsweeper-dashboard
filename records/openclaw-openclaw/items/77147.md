## Summary

Keep open: current main exposes channel turn helpers and projected `before_dispatch` claim hooks, but not the requested full-fidelity, marker-aware, trust-gated redispatch capability. The linked `mids-neo/klawbi-infra#70` context supports this as a real SDK/API design gap rather than stale support.

## What This Changes

Request a supported trusted Plugin SDK redispatch API that lets a `before_dispatch` handler claim an inbound turn, send an immediate ack, and re-run the full reply path with original context and loop markers.

## Best Possible Solution

Design an additive trusted plugin API around prepared/finalized channel turns or a plugin-callable agent invocation, with explicit replay markers, reply-gate policy, tests, docs, and manifest/trust gating.

## Reproduction Assessment

Yes for the API gap by source inspection: `runtime.channel.turn` lacks redispatch and `before_dispatch` receives only projected fields, while handled results return before model dispatch. This is a new SDK capability request, so a live plugin reproducer is not required to establish the gap.

## Solution Assessment

Unclear: redispatch is plausible, but replaying the full pipeline may be broader than needed. A prepared-turn or plugin-callable agent-invocation API that preserves inbound context and delivery policy may be safer.
