## Summary

Keep open: this PR targets a real doctor/status false warning, but the proposed runtime adds an embedding readiness probe that rejects existing provider-backed auth configurations without a plugin-local API key.

## What This Changes

The PR registers a memory-lancedb memory capability runtime for doctor/status discovery and adds regression coverage.

## Best Possible Solution

Keep the runtime-registration approach, but make readiness probing use the same embedding provider/auth path as memory recall/store and cover API-key-less profile-backed config.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a memory-lancedb memory runtime and doctor warns when no active runtime resolves; the PR body also supplies before-fix live output for the same warning.

## Solution Assessment

No as proposed. Registering the runtime is the narrow maintainable fix, but the embedding probe must honor provider-backed auth instead of requiring plugin-local apiKey.
