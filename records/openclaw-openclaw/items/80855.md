## Summary

Keep open: current main and latest release still have the WebChat session-switch state-loss path, and the open fixing PR at https://github.com/openclaw/openclaw/pull/81037 references this issue with closing syntax, so the issue should wait for that PR to land or be rejected.

## What This Changes

The issue reports WebChat/Control UI clearing visible chat history and subagent reports after switching to another conversation and back.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81037, or close that PR with a maintainer rationale and choose a different per-session WebChat state strategy before closing this issue.

## Reproduction Assessment

Yes at source level, but no live WSL2 multi-channel run was executed. Populate visible WebChat messages, switch sessions through `switchChatSession`, then switch back before `chat.history` returns; current main clears `chatMessages` and has no per-session visible-message cache.

## Solution Assessment

Yes, the open PR's per-session visible-message cache plus switched-away final-event caching is the narrowest maintainable fix shape for this issue. The issue should remain open until that implementation is reviewed and merged or deliberately rejected.
