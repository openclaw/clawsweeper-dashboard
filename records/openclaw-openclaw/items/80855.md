## Summary

Keep open. Current main still has the WebChat session-switch state-loss path, and the linked fixing PR is open but unmerged, so this issue should stay open until that implementation or an equivalent fix lands.

## What This Changes

WebChat should preserve visible conversation history and subagent/report messages when switching away from a conversation and back.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81037 or an equivalent focused Control UI cache fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: populate visible messages in one WebChat session, switch to another session, then switch back before durable `chat.history` can supply those messages. Current main clears `chatMessages` during the switch and has no per-session visible-message cache to restore them.

## Solution Assessment

Yes. The narrow fix belongs in Control UI session-switch state management, preserving per-session visible messages while reconciling with `chat.history`; the linked PR follows that direction and should be reviewed rather than closing the issue early.
