## Label Justifications

- `P1`: The report describes a recent regression that clears active WebChat conversation context during normal session switching.
- `impact:session-state`: The affected behavior loses the visible session transcript/context when the user switches conversations.
- `impact:message-loss`: Switched-away final/subagent WebChat messages can be dropped from the visible UI path instead of being restored.

## Summary

Keep open: current main and the latest release still have the WebChat session-switch state-loss path, and an open closing PR specifically targets this issue with a narrow fix candidate.

## What This Changes

WebChat should preserve active-session messages and subagent reports when switching away from and back to a conversation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81037 if its per-session WebChat message cache still fits current main, or close that PR with a maintainer-chosen alternate UI-state approach.

## Reproduction Assessment

Yes, at source level: current main clears `chatMessages` during `switchChatSession`, then `loadChatHistory` can only preserve the emptied current array, and switched-away events return before caching. I did not run a live WSL2 multi-channel reproduction.

## Solution Assessment

Yes, the narrow fix direction is to preserve per-session WebChat visible-message state while reconciling with durable `chat.history`. The open linked PR is the current implementation candidate and should decide the issue outcome after review.
