Keep open. Current main still has the Control UI/WebChat state-loss path, and the linked fixing PR is open with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that WebChat/Control UI loses visible chat history and subagent reports after switching away from a conversation and back.

## Best Possible Solution

Review and land the linked focused UI cache PR, or replace it with an equivalent per-session message cache plus regression tests before closing this issue.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: put visible messages in session A, switch to session B, switch back before durable history catches up, and current main has already cleared A's `chatMessages`. I did not run the live WSL2 multi-channel setup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in Control UI/WebChat session state handling by caching per-session visible messages and reconciling with `chat.history`, which is exactly the direction of the linked open PR.
