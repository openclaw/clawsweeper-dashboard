## Summary

Keep open: current main still has a source-level WebChat state-loss path, and the open linked fix PR should be reviewed, landed, or rejected before this issue is closed.

## What This Changes

The issue asks WebChat to preserve visible conversation context when switching away from a conversation and back.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81037, or close that PR with rationale if maintainers prefer a different WebChat state approach; close this issue only after the chosen fix is merged.

## Reproduction Assessment

Yes, at source level: switching sessions clears `chatMessages`, then history reload can only preserve messages still present in the current state, and switched-away events are ignored unless they match the active run. I did not run a live WSL2/browser reproduction.

## Solution Assessment

Yes, the narrow maintainable direction is to fix Control UI/WebChat session-switch state handling by preserving per-session visible messages while reconciling with durable history. The open linked PR is the active implementation candidate, so this issue should stay open until that PR is resolved.
