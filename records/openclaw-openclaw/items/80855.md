## What This Changes

The issue reports that WebChat/Control UI loses visible chat history and subagent report context after switching away from a conversation and back.

## Best Possible Solution

Review and land the focused per-session WebChat message-cache fix in https://github.com/openclaw/openclaw/pull/81037, or close that PR with a maintainer-selected alternative state model before closing this issue.

## Reproduction Assessment

Yes, as a source-level path: put visible messages in one WebChat session, switch away, then switch back before durable `chat.history` restores the transcript; current main clears `chatMessages` and has no per-session visible-message cache. I did not run a live WSL2/multi-channel reproduction.

## Solution Assessment

Yes at the triage level: the narrow maintainable fix belongs in Control UI/WebChat session-switch state handling, preserving per-session visible messages while still reconciling with durable `chat.history`. The open linked PR is the active implementation candidate, so a separate repair lane would duplicate it.
