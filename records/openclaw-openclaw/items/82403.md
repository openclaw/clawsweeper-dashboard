## Summary

Keep open: this PR targets a real source-reproducible Codex app-server delivery stall, but the patch appears to release on raw assistant progress that current tests treat as non-terminal, and the external PR only provides mocked harness proof.

## What This Changes

This PR changes Codex app-server run-attempt logic and docs so completed raw assistant `rawResponseItem/completed` notifications can trigger the completed-assistant idle release path, with a changelog entry and focused tests.

## Best Possible Solution

Land a guarded Codex app-server completion fallback that releases only when raw assistant output is known to be terminal, preserves post-tool/status progress waiting, updates docs/tests, and closes https://github.com/openclaw/openclaw/issues/82343 after real app-server proof.

## Reproduction Assessment

Yes, source-reproducible: current main has a focused raw assistant completion case that times out without `turn/completed`, and the linked issue includes channel logs ending at `rawResponseItem/completed`. I did not run live channel credentials in this read-only review.

## Solution Assessment

No. Reusing the completed-assistant release path is plausible, but this PR needs a guard that distinguishes final raw assistant completions from post-tool/status raw assistant progress, plus real app-server proof.
