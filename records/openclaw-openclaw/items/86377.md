Full review comments:

- **[P1] Use the existing message hook surface:** `extensions/discord/src/monitor/message-handler.process.ts:412`
  - body: This adds a Discord-only env command recorder for the same inbound observation role already covered by the documented `message_received` plugin hook, and a sibling channel emits that hook from canonical inbound context. Please wire Discord into that existing hook surface instead of adding a separate hot-path process spawn that will drift in behavior and security review.
  - confidence: 0.88
- **[P2] Record the resolved sender identity:** `extensions/discord/src/monitor/message-handler.process.ts:415`
  - body: For accepted PluralKit messages, `ctx.author.id` is the proxy/bot-side Discord author, while the verified sender used for access checks and `SenderId` is `ctx.sender.id`. The recorder would store the wrong `source_sender_id` for those inbound messages, so use the resolved sender identity or the already-built inbound context sender instead.
  - confidence: 0.9
- **[P2] Remove the hard-coded tenant default:** `extensions/discord/src/monitor/verified-inbound-recorder.ts:10`
  - body: A global default of `johnny` means any operator who sets only the recorder command records every inbound row under another tenant value, and the tests lock in that behavior. Make the tenant explicit, derive it from an OpenClaw-owned account/config value, or omit it rather than shipping a user-specific default.
  - confidence: 0.84
