## Summary

Keep open. Current main and the latest release still have a source-supported path where empty-transcript runtime context becomes the literal runtime-event marker and is submitted through the normal user-message prompt path, while display and memory filters do not prove non-user provenance for that marker.

## What This Changes

The issue asks OpenClaw to stop attributing auto-compaction/runtime event text such as `Continue the OpenClaw runtime event.` to the human user in canonical transcripts.

## Best Possible Solution

Model runtime-only continuation as first-class non-user provenance through prompt submission, transcript persistence, display projection, export, and memory ingestion while preserving the model-visible resumption context.

## Reproduction Assessment

No live high-confidence reproduction was run. The current-main source path is clear: empty runtime context can create the marker, submit it through `AgentSession.prompt()`, and rely on user-role persistence unless a separate suppression flag is active.

## Solution Assessment

No, filtering the literal marker in one consumer would be too narrow. The maintainable fix should carry explicit runtime-only provenance through the runtime, transcript, projection, export, and memory surfaces.
