## Summary

Keep open. Current main and the latest release still have a source-supported path where runtime-only continuation text becomes the normal submitted prompt, then downstream projection/export filters do not prove durable non-user provenance for that prompt.

## What This Changes

The issue reports that auto-compaction can inject `Continue the OpenClaw runtime event.` into canonical transcript history as if the human user authored it.

## Best Possible Solution

Model runtime-only continuation as first-class non-user provenance through prompt submission, transcript persistence, display projection, export, and memory ingestion while preserving the model-visible resumption context.

## Reproduction Assessment

No live current-main WebGUI/TUI reproduction was run in this read-only pass. The issue is source-reproducible with high confidence because current main creates the literal runtime marker, submits it through the normal session prompt path, and lacks a marker-specific provenance filter in WebChat projection or memory export.

## Solution Assessment

The requested outcome is correct, but a durable fix should be a first-class runtime-only provenance contract rather than filtering the literal marker in one consumer. Maintainers should decide the exact audience/provenance shape because it crosses session persistence, UI, export, memory, and security semantics.
