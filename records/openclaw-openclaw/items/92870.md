## What This Changes

## Best Possible Solution

Define and implement one canonical non-user provenance path for runtime-only continuation turns across compaction, transcript persistence, UI/history projection, memory/session export, and model prompt composition.

## Reproduction Assessment

Yes, at source level: current main still emits the literal runtime-only marker prompt unless a caller separately suppresses user-message persistence. I did not run a live WebGUI/direct compaction scenario, so this is not a full live reproduction.

## Solution Assessment

Unclear as submitted because the issue lists several possible treatments. The best fix should be provenance-based at the runtime/transcript boundary, with UI and memory consumers honoring that marker, rather than only prefixing or hiding display text.
