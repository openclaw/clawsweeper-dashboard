## Summary

Keep open. Current main and the latest release still have a source-supported path where runtime-only continuation text is converted into a normal user-role prompt, and related open work only addresses part of the memory-ingestion fallout rather than the full transcript attribution boundary.

## What This Changes

The issue asks OpenClaw to prevent runtime-generated compaction continuation text from being persisted, displayed, exported, or ingested as a user-authored transcript message.

## Best Possible Solution

Model runtime-only continuation as first-class non-user provenance from prompt submission through transcript persistence, display projection, export, and memory ingestion, while preserving the model-visible resumption context and real user-typed marker text.

## Reproduction Assessment

Yes at source level: current main converts runtime-only context with an empty transcript prompt into the marker prompt, submits it through the normal prompt path, and persists it as role=user. No live WebGUI/TUI compaction run was performed in this read-only pass.

## Solution Assessment

The requested outcome is right, but the best fix is not a one-consumer literal marker filter. It should be a durable runtime-only provenance contract with explicit maintainer/security agreement on how transcript, UI, export, and memory consumers handle it.
