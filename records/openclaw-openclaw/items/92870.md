## Summary

Keep open. Current main still has a source-supported path where empty runtime-only context becomes the literal marker prompt, is submitted through the normal user-message path, and lacks a first-class runtime-only provenance filter across WebChat/history and memory export surfaces.

## What This Changes

Prevent auto-compaction/runtime continuation text such as `Continue the OpenClaw runtime event.` from being recorded or displayed as user-authored transcript content.

## Best Possible Solution

Add explicit runtime-only event provenance at the persistence boundary and make WebChat/TUI/export/memory projections honor it, with focused source tests plus a live WebGUI or TUI compaction proof.

## Reproduction Assessment

No live high-confidence repro was run. Source inspection gives a medium-confidence reproduction path: the marker is generated, submitted through `activeSession.prompt`, normalized as a user message, and not filtered by current projection/export rules.

## Solution Assessment

Unclear. The requested outcome is correct, but the best fix needs maintainer product/security direction and should be first-class runtime-only provenance rather than filtering the literal marker text only in one UI path.
