## Summary

Keep open. Current main and the latest release still have a source-supported path where an internal runtime-only continuation marker is submitted and persisted as a normal user prompt, and the display/memory filters I checked do not prove that this exact plain user text is hidden or tagged with non-user provenance.

## What This Changes

The issue asks OpenClaw to stop attributing compaction/runtime system event text such as `Continue the OpenClaw runtime event.` to the human user in canonical transcripts.

## Best Possible Solution

Model runtime-only continuation as first-class hidden runtime/session provenance, then make transcript display, export, and memory ingestion honor that provenance without suppressing real user-authored matching text.

## Reproduction Assessment

Yes, source-reproducible with medium confidence: current main still builds the exact runtime-only marker, submits it through the normal string prompt path, and projection filters do not hide that plain user text. No live WebGUI/TUI compaction reproduction was run.

## Solution Assessment

Unclear as an implementation. The requested outcome is correct, but the best fix should be a runtime-only provenance contract across persistence and projections rather than a UI-only or literal-string filter.
