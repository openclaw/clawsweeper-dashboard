## Summary

Keep open. Current main and the latest release still have a source-supported path where runtime-only continuation text is submitted and persisted as a user-role prompt, while the visible-history and memory filters do not prove that this marker is hidden or de-attributed everywhere.

## What This Changes

Prevent compaction/runtime continuation text from being recorded, exported, or displayed as user-authored transcript content.

## Best Possible Solution

Model runtime-only continuation as first-class hidden runtime/session provenance through prompt submission, persistence, display projection, export, and memory ingestion while preserving model-visible resumption context.

## Reproduction Assessment

No live high-confidence reproduction was run in this pass. The issue is source-reproducible on current main: runtime-only context can create the literal marker, submit it through the normal prompt path, and leave projection/export filters without a marker-specific provenance check.

## Solution Assessment

Unclear. The requested outcome is correct, but the best fix should be a first-class runtime-only provenance contract rather than filtering the literal marker in only one UI or memory path.
