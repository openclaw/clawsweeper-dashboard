## Summary

Keep open. Current main still has a source-backed path where an empty runtime-only compaction/continuation turn becomes the literal user-role prompt `Continue the OpenClaw runtime event.`, and current WebChat/history projection does not hide that marker; the open related fix at https://github.com/openclaw/openclaw/pull/79547 is memory-ingestion scoped and should not be treated as closing this broader transcript-attribution bug.

## What This Changes

The issue asks OpenClaw to prevent runtime/compaction event text from being persisted or displayed as if it were authored by the human user.

## Best Possible Solution

Define durable runtime-only/agent-only provenance for compaction and continuation events, persist it with transcript messages, suppress or system-render those events in WebChat/export, and filter memory ingestion without hiding real user-authored matching text.

## Reproduction Assessment

Yes, from source with high confidence: a runtime-only empty transcript turn resolves to the marker prompt, is submitted through the normal user prompt path, and current history projection does not hide that non-empty user-role message. I did not run a live WebGUI compaction replay in this read-only review.

## Solution Assessment

Unclear as a final implementation. The right direction is provenance-based runtime-event handling across persistence, WebChat/export, and memory ingestion; a UI-only hide or literal-string filter would be too narrow and could suppress real user text.
