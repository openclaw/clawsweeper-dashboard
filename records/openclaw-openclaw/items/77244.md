## Summary

Keep open. Current main still lacks a startup-planning path that loads generation-provider plugins referenced only by explicit media-generation defaults, and open PR #77590 is already tracking the fix with closing syntax for this issue.

## What This Changes

The issue reports generation tools staying catalog-only or not live on v2026.5.3 despite OpenAI/Google API keys, explicit image/video/music generation defaults, and enabled provider plugins.

## Best Possible Solution

Review and land PR #77590 or an equivalent generic startup-planning fix that loads provider plugins referenced by explicit image, video, and music generation defaults while preserving plugin disable, deny, and allowlist policy.

## Reproduction Assessment

Yes at source level, but not from a live current-main gateway run in this read-only review. The reported v2026.5.3 commands and current startup-planner/manifests show a clear path where API-key-only generation provider plugins remain unloaded when chat runtime does not otherwise select them.

## Solution Assessment

Yes. The narrow maintainable fix belongs in generic startup planning for manifest-owned generation providers; changing `models.authStatus` or hard-coding provider imports would not address provider runtime registration correctly.
