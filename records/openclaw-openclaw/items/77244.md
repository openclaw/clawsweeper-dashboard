The report is fresh, detailed, and matches the documented contract that explicit generation defaults or provider API keys should make the shared generation tools available; current main source also intends those factories to be created, so the remaining question is where the v2026.5.3 runtime path loses them.

## What This Changes

The issue reports that image, video, and music generation tools stay absent from effective tools on v2026.5.3 despite explicit generation defaults and provider API keys in the Gateway environment.

## Best Possible Solution

Add a focused regression or diagnostic for tools.effective with explicit OpenAI/Google generation defaults and env-backed provider manifests, then fix the layer that suppresses the media factories or emit a clear policy/config notice.

## Reproduction Assessment

No. The issue includes concrete v2026.5.3 Gateway commands showing the catalog/effective mismatch, but I did not establish a failing current-main live Gateway path; source and tests currently say explicit generation defaults should expose the factories.

## Solution Assessment
