## Summary

Keep open. The report matches the documented contract for configured generation tools, current source still shows a startup-selection gap for media provider plugins, and open PR #77590 is already paired with this issue as the targeted fix.

## What This Changes

Configured OpenAI/Google image, video, and music generation defaults should make the corresponding generation tools live when provider API keys are available.

## Best Possible Solution

Land #77590 or an equivalent generic startup-planning fix that loads provider plugins referenced by configured generation model refs while preserving plugin disable/deny/allow policy.

## Reproduction Assessment

Yes. The source path is high-confidence: configure a generation model for a provider whose plugin is not the chat agent runtime, provide the matching API key, restart the gateway, and current startup selection has no media-model owner path to load that plugin.

## Solution Assessment

Yes in direction. The narrow maintainable fix is startup activation for providers referenced by explicit image/video/music generation defaults, not changing models.authStatus or adding a user config workaround.
