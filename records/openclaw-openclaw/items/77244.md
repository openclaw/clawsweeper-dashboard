## Summary

Keep open. The fresh report matches documented behavior for explicit media generation defaults, and current source shows a plausible policy bug: `tools.alsoAllow` can be converted into a restrictive optional-factory allowlist that excludes `image_generate`, `video_generate`, and `music_generate` before the explicit defaults take effect. `models.authStatus` omitting plain API-key providers is expected and does not prove missing credentials.

## What This Changes

The issue asks OpenClaw to make image, video, and music generation tools live when v2026.5.3 has OpenAI/Google provider keys, explicit generation defaults, and enabled provider plugins.

## Best Possible Solution

Make optional media factory planning distinguish explicit `tools.allow` from additive `tools.alsoAllow` plugin-discovery hints, preserving `tools.deny` and true allowlist behavior while keeping explicit generation defaults live.

## Reproduction Assessment

Yes. The issue provides exact v2026.5.3 gateway outputs, and current source gives a high-confidence path where `tools.alsoAllow: ["group:memory"]` suppresses optional media factories despite explicit defaults; I did not execute the Gateway during this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is in optional media factory policy handling, not in `models.authStatus`, with regression coverage for explicit generation defaults plus `tools.alsoAllow`.
