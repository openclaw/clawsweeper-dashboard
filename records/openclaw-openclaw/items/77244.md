## Summary

Keep open. The release report and follow-up analysis point to a real media provider startup/liveness gap: provider implementations exist, but configured API-key-only media provider plugins can miss every gateway startup eligibility path before their generation providers register.

## What This Changes

The issue reports that image, video, and music generation tools stay absent from `tools.effective` on v2026.5.3 despite explicit generation model defaults, provider API keys, and enabled OpenAI/Google plugins.

## Best Possible Solution

Add a contract-driven startup or activation path for media-generation provider plugins referenced by `agents.defaults.*GenerationModel` and provider env/auth signals, without hardcoding bundled provider IDs, and cover it with a focused regression.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. The issue has two concrete v2026.5.3 runtime repros, and source inspection shows the provider plugin startup path can miss configured media-generation-only providers before registration.

## Solution Assessment

Unclear, but the suggested manifest `onConfigPaths` direction is close. A generic manifest-contract-based startup path for configured media provider model refs may be more maintainable than duplicating the same config paths across every provider manifest.
