## Summary

Keep open. The added config details make this source-reproducible: additive `tools.alsoAllow` can be carried into the optional media factory allowlist as a restrictive allowlist, so explicit generation defaults that should expose `image_generate`, `video_generate`, and `music_generate` can still be suppressed; `models.authStatus` omitting API-key providers is expected behavior, not the root cause.

## What This Changes

The issue requests that generation tools become live in `tools.effective` on v2026.5.3 when OpenAI/Google API keys, explicit generation defaults, and enabled provider plugins are present.

## Best Possible Solution

Keep additive `tools.alsoAllow` from narrowing optional media factory materialization while still honoring explicit `tools.allow` and deny policies.

## Reproduction Assessment

Yes, source-reproducible. The reported config path with `tools.alsoAllow` and no `tools.allow` can produce a restrictive media-factory allowlist before explicit generation defaults are considered; a focused test should instantiate the effective tool path with that config and assert all three generation tools appear.

## Solution Assessment

Yes. The narrow maintainable fix is in tool policy/materialization, not in `models.authStatus`, because API-key providers are intentionally excluded from refreshable auth status while the missing tools are decided earlier by factory gating.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77244. Generation tools disappear from `tools.effective` when config has explicit `agents.defaults.imageGenerationModel`, `videoGenerationModel`, and `musicGenerationModel` plus `tools.alsoAllow` but no `tools.allow`/`tools.deny`. The likely root cause is that implicit allow-all semantics from `alsoAllow` are stripped into `pluginToolAllowlist` before `resolveOptionalMediaToolFactoryPlan`, making optional media factories think `image_generate`, `video_generate`, and `music_generate` are denied. Change only the policy/materialization path so additive `alsoAllow` does not restrict optional media factories; preserve explicit `tools.allow` and `tools.deny` behavior. Add regression coverage in `src/agents/openclaw-tools.media-factory-plan.test.ts` or a nearby effective-inventory test using `tools.alsoAllow: ["group:memory"]` with explicit generation defaults and assert `image_generate`, `video_generate`, and `music_generate` are present. Do not solve this by making `models.authStatus` list API-key-only providers. Add a user-facing `CHANGELOG.md` fix entry.
