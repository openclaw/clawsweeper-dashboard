## Summary

Keep open. The report is fresh, the supplied v2026.5.3 outputs show a catalog/effective mismatch, and current docs/tests/source define explicit image/video/music generation defaults as sufficient to expose the generation tools; `models.authStatus` not showing env-backed API-key providers is expected and does not resolve the missing effective tools.

## What This Changes

The issue reports that `image_generate`, `video_generate`, and `music_generate` are cataloged but absent from `tools.effective` on v2026.5.3 despite explicit generation defaults and provider API keys.

## Best Possible Solution

Add a focused regression around `tools.effective` with explicit image/video/music defaults and env-backed OpenAI/Google provider manifests, then fix the layer that drops the media factories or emit a clear diagnostic when the runtime config is not the expected config.

## Reproduction Assessment

No. The issue has concrete v2026.5.3 gateway calls, but I did not establish the same failure on current main; current source and tests say explicit generation defaults should enable the factories.

## Solution Assessment

Unclear. Changing `models.authStatus` alone is not the best fix because env/API-key providers are intentionally excluded there; the narrow solution is to reproduce `tools.effective` with the reported config and fix the layer suppressing the generation factories.
