## Summary

Keep open. The report matches current source and docs, and an open PR already targets the missing gateway startup path for configured generation providers, so this issue should remain open until that PR lands or maintainers close it.

## What This Changes

Generation tools are reported as cataloged but absent from `tools.effective` on v2026.5.3 despite OpenAI/Google API keys, explicit media generation defaults, and enabled provider plugins.

## Best Possible Solution

Land or close #77590 after review; the desired end state is a generic gateway startup path that loads manifest owners for explicitly configured image, video, and music generation defaults while preserving plugin allow/deny policy.

## Reproduction Assessment

Yes, source-backed. With generation defaults set but no matching `agentRuntime`, current startup selection has no generation-provider activation path, which matches the reported `tools.catalog` versus `tools.effective` mismatch.

## Solution Assessment

Yes, the linked PR direction appears to be the narrow maintainable fix. Updating gateway startup eligibility for configured generation provider owners is safer than treating `models.authStatus` or the catalog output as the root cause.
