## Summary

Keep open. Current main still has the startup activation gap described in the report, and open PR #77590 explicitly targets this issue with a narrow gateway startup fix.

## What This Changes

The issue asks for `image_generate`, `video_generate`, and `music_generate` to become live when provider API keys and explicit generation defaults are configured.

## Best Possible Solution

Land #77590 or an equivalent startup-planning fix that loads configured generation provider owners while preserving plugin disable, deny, and allowlist policy.

## Reproduction Assessment

Yes, source-reproducible: configure explicit OpenAI/Google generation defaults with provider API-key env and no matching provider plugin startup trigger, then gateway startup can omit the provider plugin before `tools.effective` is built. I did not live-run the gateway in this read-only pass.

## Solution Assessment

Yes. Starting manifest owners for explicitly configured generation defaults is the narrow maintainable fix; changing `models.authStatus` would not register the runtime providers.
