## Summary

Close as implemented on current main: explicit generation defaults now count as media-tool availability, so `image_generate`, `video_generate`, and `music_generate` are materialized for effective tools; `models.authStatus` omitting API-key-only providers is intentional dashboard behavior.

## What This Changes

The issue reports that `image_generate`, `video_generate`, and `music_generate` remain absent from `tools.effective` on v2026.5.3 despite explicit generation defaults and provider API keys.

## Best Possible Solution

Keep the current main behavior and ship it in the next release so explicit generation defaults make the shared media tools live; keep `models.authStatus` focused on refreshable credentials rather than API-key env status.

## Reproduction Assessment

Yes. The reported symptom maps directly to the media factory planner used by `tools.effective`; current main source and regression coverage show explicit image, video, and music generation defaults now expose all three tools.

## Solution Assessment

Yes. The narrow maintainable fix is to treat explicit generation defaults as availability evidence in the media-tool factory path, while leaving API-key-only providers out of `models.authStatus` as intended dashboard behavior.
