## Summary

Keep open. The report is fresh and matches the documented contract that explicit generation defaults or provider API keys should expose the media generation tools, while the reported v2026.5.3 runtime output shows a catalog/effective mismatch that current source inspection does not explain away. `models.authStatus` omitting OpenAI/Google API-key providers is not decisive because that endpoint intentionally focuses on refreshable credentials.

## What This Changes

Make `image_generate`, `video_generate`, and `music_generate` live in `tools.effective` when explicit generation defaults and OpenAI/Google provider API keys are present.

## Best Possible Solution

Add a focused `tools.effective` regression or diagnostic for explicit OpenAI/Google generation defaults plus env-backed provider manifests, then fix the layer that drops the generation tools or emit a clear policy/config notice if the runtime is intentionally suppressing them.

## Reproduction Assessment

No. The issue includes concrete v2026.5.3 gateway commands and outputs, but this review did not establish the same failure on current main and current source/tests indicate the explicit defaults should expose the factories.

## Solution Assessment

Unclear. Changing `models.authStatus` would not be the right fix by itself; the maintainable path is to isolate whether `tools.effective` is seeing stale runtime config/plugin metadata, policy filtering, or package shadow state before changing the factory gates.
