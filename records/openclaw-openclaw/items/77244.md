## Summary

Keep open. The report is fresh and matches the documented/source-level contract that explicit generation defaults or provider credentials should expose generation tools, while the reported v2026.5.3 runtime still omits them from `tools.effective`; `models.authStatus` omitting API-key providers is expected and does not explain the liveness failure.

## What This Changes

The issue asks for `image_generate`, `video_generate`, and `music_generate` to be live in `tools.effective` when OpenAI/Google generation defaults and API-key environment variables are configured.

## Best Possible Solution

Add a focused gateway/effective-inventory regression or diagnostic for explicit image/video/music defaults with env-backed OpenAI/Google providers, then fix the layer that suppresses the media factories or emit a precise policy/config notice.

## Reproduction Assessment

No. The issue gives concrete v2026.5.3 commands and outputs for the catalog/effective mismatch, but I did not establish the same failure on current main and source-level coverage says the factories should be available.

## Solution Assessment

Unclear. The current factory design is the right narrow path, but changing `models.authStatus` would not fix tool liveness; the maintainable fix should first identify whether `tools.effective` policy/cache/session state or plugin metadata is suppressing the factories.
