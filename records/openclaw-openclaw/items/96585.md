## Summary

Keep open: this is a credible post-release Node v25+/v26 FileHandle GC crash report. Current main has a post-release session-lock deferred-release improvement, but there is not enough proof that it covers the Feishu workspace `.mp4` path or the active-memory/session `.lock` path, and the pinned fs-safe lock cleanup still has a plausible fire-and-forget close hazard.

## What This Changes

OpenClaw should stop leaking or prematurely dropping FileHandle ownership so Node.js v25+ no longer crashes agent runs during Feishu workspace access or active-memory/session lock cleanup.

## Best Possible Solution

Keep this issue open as the renewed canonical report, reproduce it on Node v25+/v26 with forced GC and stability-bundle evidence, then fix the exact OpenClaw lifecycle or fs-safe sidecar-lock path with coverage across the Feishu/workspace and active-memory/session-lock cases.
