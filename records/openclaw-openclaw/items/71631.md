Keep open. Current main covers important Web UI chat responsiveness pieces, but this umbrella still includes gateway configuration responsiveness, slow-RPC isolation, and end-to-end latency attribution that are not fully implemented or fully superseded by the related narrower issues.

## What This Changes

Improve perceived and actual responsiveness in gateway configuration screens and Web UI chat by adding immediate feedback, streaming/running visibility, slow-RPC isolation, and latency instrumentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella only if a broad performance tracker is desired; otherwise split it into focused tasks for UI-to-gateway timing spans, per-control config pending states, cancellation/stale-response guards for slow panels, and preservation of the existing optimistic streaming chat path.

## Solution Assessment
