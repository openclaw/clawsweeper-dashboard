## Summary

This PR is a focused, still-relevant fix for the current Nextcloud Talk webhook behavior; current main still routes signed non-message Activity Streams payloads through strict message parsing and returns 400, while the PR keeps malformed JSON and malformed message-shaped payloads on the 400 path and acknowledges non-message events.

## What This Changes

The PR updates the Nextcloud Talk webhook parser and replay tests so signed non-message Activity Streams events are acknowledged without invoking the message handler, while malformed JSON and malformed supported message payloads still return 400.

## Best Possible Solution

Merge the focused channel fix after normal maintainer and CI gates; keep optional debug logging as a separate logger-surface follow-up if maintainers want that instrumentation.

## Reproduction Assessment

Yes. Source inspection on current main shows signed payloads still go through strict chat-message schema parsing and return 400 on schema miss, and the linked issue provides concrete Nextcloud/Talk file-share steps for the same failure.

## Solution Assessment

Yes. Parsing signed JSON before strict message validation and acknowledging non-message Activity Streams events is the narrowest maintainable fix; adding debug logging would require a separate logger hook decision.
