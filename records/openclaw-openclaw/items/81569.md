## Summary

Keep open: this is a focused Nextcloud Talk plugin bug fix for a linked, source-reproducible webhook failure, with supplied real behavior proof and no blocking review findings.

## What This Changes

The branch changes Nextcloud Talk webhook parsing to classify signed JSON before strict chat-message schema validation, ignore signed non-message events, and add replay tests for invalid JSON, invalid chat payloads, and ignored Document events.

## Best Possible Solution

Land this focused plugin fix after normal maintainer and CI gating so signed Talk events OpenClaw does not handle are acknowledged without weakening signature, malformed JSON, or invalid supported-message handling.

## Reproduction Assessment

Yes. Current main's post-signature parser returns 400 for any payload outside the strict Create/Note chat schema, and the linked issue supplies Nextcloud 33/Talk 23 file-share steps while the PR supplies after-fix signed-webhook proof.

## Solution Assessment

Yes. Parsing signed JSON before strict message validation is the narrow plugin-owned fix; a fuller Activity Streams union can be added later, but it is not needed to stop benign signed non-message hooks from producing 400s.
