## Summary

Keep open: the PR addresses the main reported Nextcloud Talk 400 response for signed non-message events, and the reporter's coverage feedback was partly handled, but the new classifier can still acknowledge partial chat-message payloads that should remain on the invalid-payload path.

## What This Changes

The PR changes Nextcloud Talk signed webhook parsing so authenticated non-message Activity Streams events are acknowledged instead of treated as invalid payloads.

## Best Possible Solution

Keep the focused plugin fix, but classify partial chat-message-shaped payloads as invalid before ignoring unknown signed Talk events, with regression coverage for the missing required-section case.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.5.7 return 400 for any signed payload outside the strict chat-message schema, and the linked report provides concrete Nextcloud 33/Talk 23 file-share steps.

## Solution Assessment

No, not quite. Parsing signed JSON before strict message validation is the right narrow approach, but the PR should preserve 400 for partial chat-message payloads before acknowledging unknown non-message events.
