Keep open for normal PR review and landing. Current main and the latest release still route signed non-message Nextcloud Talk webhooks through the strict chat-message schema and return 400, while this PR addresses that path with focused code, regression coverage, sufficient terminal proof, and no blocking review findings.

## What This Changes

The PR changes the Nextcloud Talk webhook parser to acknowledge signed non-message Activity Streams events while preserving 400 responses for malformed JSON or malformed message-shaped payloads.

## Best Possible Solution

Merge this focused channel fix after normal maintainer and CI gates; keep optional debug logging and richer structured file-share media parsing in separate follow-up work.

## Solution Assessment

Yes. Parsing signed JSON before strict message validation and acknowledging non-message Activity Streams events is the narrowest maintainable fix; adding debug logging would require a separate logger option/surface decision.
