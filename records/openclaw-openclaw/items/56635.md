- `impact:message-loss`: The issue is about Gmail hook events being delivered and processed for mail labels that should be excluded.

## Summary

Keep open: current main and the latest release still omit the Gmail watcher `--exclude-labels` argv, and the open canonical fix PR still references this issue with closing syntax.

## What This Changes

The issue reports that Gmail webhook watcher startup no longer passes `--exclude-labels SPAM,TRASH,DRAFT,SENT` to `gog gmail watch serve`, causing sent and draft Gmail events to be delivered to agents.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/56720 or a narrow replacement that restores default `SPAM,TRASH,DRAFT,SENT` exclusions, preserves upstream `gogcli` empty-string semantics, and includes redacted watcher/runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared Gmail watcher argv builder cannot emit `--exclude-labels`, and both the gateway auto-start path and current tests use that builder.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable path is to review and land the linked canonical PR or an equivalent replacement, then close this issue after the fix reaches main.
