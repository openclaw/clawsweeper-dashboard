## Summary

Keep open: current main and the latest release still omit `--exclude-labels` from the Gmail watcher argv, and the open canonical fix PR still references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to restore Gmail watcher excluded-label handling so sent, draft, spam, and trash messages do not trigger hook processing.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/56720 or a narrow replacement that preserves default `SPAM,TRASH,DRAFT,SENT` exclusions, keeps empty-list semantics aligned with `gogcli`, and includes redacted real gateway or `gog` proof.
