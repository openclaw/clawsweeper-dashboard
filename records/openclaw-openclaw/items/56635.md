## What This Changes

Restore Gmail watcher excluded-label handling so auto-started `gog gmail watch serve` filters `SPAM,TRASH,DRAFT,SENT` again.

Yes. Source inspection on current main shows the shared Gmail watcher argv builder cannot emit `--exclude-labels`, and both gateway and CLI watcher paths call that builder; I did not run a live Gmail webhook scenario in this read-only review.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix is to restore the excluded-label argument in the watcher argv path, with any configurable override documented and tested before merge.
