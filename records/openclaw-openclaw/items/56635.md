Keep open: current main and the latest release still omit the Gmail watcher `--exclude-labels` argument, and the canonical open fix PR has not landed.

## What This Changes

The issue reports a Gmail watcher regression where startup no longer passes `--exclude-labels SPAM,TRASH,DRAFT,SENT`, allowing sent and draft Gmail events to reach hooks.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/56720 or a narrow replacement that restores default `SPAM,TRASH,DRAFT,SENT` exclusions and proves the real watcher/runtime behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared Gmail watcher argv builder cannot emit `--exclude-labels`, and the gateway watcher path uses that builder; I did not run a live Gmail/gog scenario in this read-only review.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable path is to review and land the linked canonical PR or an equivalent replacement, then close this issue after the fix reaches main.
