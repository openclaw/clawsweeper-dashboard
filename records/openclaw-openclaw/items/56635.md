## Summary

Keep open because current main and the latest release still omit `--exclude-labels` from the Gmail watcher serve arguments, the strict Gmail hook config still lacks an `excludeLabels` field, and an open linked fix PR still uses closing syntax for this issue.

## What This Changes

Restore Gmail watcher excluded-label handling so sent, draft, spam, and trash events are not processed by default, with a typed override if the configurable path is chosen.

## Best Possible Solution

Land one canonical fix, preferably the typed config/docs/test approach in https://github.com/openclaw/openclaw/pull/56720, preserving default `SPAM,TRASH,DRAFT,SENT` exclusions and explicit empty-list semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to inspect the resolved default Gmail hook config and `buildGogWatchServeArgs`; current main has no config field or builder branch that can emit `--exclude-labels`.

## Solution Assessment

Yes. Keeping this issue open while maintainers repair and land the linked canonical PR is the narrow maintainable path; closing before the linked fix merges would lose the regression tracker.
