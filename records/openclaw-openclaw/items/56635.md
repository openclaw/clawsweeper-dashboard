## Summary

Keep open: current main and the latest release still omit the Gmail watcher `--exclude-labels` argument, while the linked canonical fix PR remains open and conflicting rather than merged.

## What This Changes

The issue reports a Gmail watcher regression where `buildGogWatchServeArgs()` stopped passing `--exclude-labels SPAM,TRASH,DRAFT,SENT` to `gog gmail watch serve`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/56720 or a narrow replacement that restores default `SPAM,TRASH,DRAFT,SENT` exclusions and proves the real watcher/runtime behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared Gmail watcher argv builder cannot emit `--exclude-labels`, and the gateway watcher calls that builder; I did not run a live Gmail/gog webhook scenario in this read-only review.

## Solution Assessment

Yes for keeping this issue open. The maintainable fix is to restore default excluded labels in the watcher argv, with any configurable override documented and tested before merge.
