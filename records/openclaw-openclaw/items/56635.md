## Summary

Current main and the latest release still omit the Gmail watcher excluded-label flag, and an open linked PR already owns the fix path, so this issue should remain open rather than be closed or queued for a duplicate repair.

## What This Changes

The issue reports that Gmail watcher startup no longer passes `--exclude-labels SPAM,TRASH,DRAFT,SENT`, allowing sent mail and drafts to reach hook processing.

## Best Possible Solution

Restore `DRAFT` and `SENT` exclusion in the shared Gmail watcher argument builder, then land https://github.com/openclaw/openclaw/pull/56720 or an equivalent refreshed replacement with focused proof.

## Reproduction Assessment

Yes. At source level, current main omits `--exclude-labels` from the shared watcher argv builder while both Gmail watcher entry points use that helper; I did not run a live Gmail/Pub/Sub setup.

## Solution Assessment

Yes for issue routing. The fix belongs in the shared `buildGogWatchServeArgs()` path, and because an open PR already closes this issue, the best next step is to review, refresh, and land that PR or a narrow equivalent.
