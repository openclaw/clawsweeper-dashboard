## Summary

Keep open: current main and the latest release still omit the Gmail watcher excluded-label argument, and https://github.com/openclaw/openclaw/pull/56720 is an open unmerged candidate fix rather than a reason to close this issue.

## What This Changes

The Gmail watcher should exclude SPAM, TRASH, DRAFT, and SENT when building `gog gmail watch serve` args so outbound and draft events do not trigger hook runs.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/56720, or an equivalent narrow patch, so the shared Gmail watcher args restore `DRAFT` and `SENT` exclusion before this issue closes.

## Reproduction Assessment

Yes. Source inspection shows the shared watcher arg builder omits `--exclude-labels`, both watcher entry points use that helper, and gogcli defaults only exclude `SPAM,TRASH`.

## Solution Assessment

Yes for the issue boundary. Restoring the flag in `buildGogWatchServeArgs()` is the narrow shared fix; making the label set configurable is optional maintainer product scope.
