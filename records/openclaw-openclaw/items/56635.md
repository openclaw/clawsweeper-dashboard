## Summary

Keep open: current main and the latest stable release still omit the Gmail watcher `--exclude-labels` argument, and the only open linked fix is still unmerged.

## What This Changes

The issue asks to restore Gmail watcher excluded labels so sent, draft, spam, and trash mailbox changes do not reach hook processing.

## Best Possible Solution

Refresh and land the shared Gmail watcher builder fix, with an explicit maintainer choice on whether configurability belongs in this patch.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` omit `--exclude-labels` from the shared Gmail watcher arg builder, while gogcli defaults only exclude `SPAM,TRASH`.

## Solution Assessment

Yes for the bug boundary. Restoring the flag in `buildGogWatchServeArgs()` is the narrow shared fix; adding `hooks.gmail.excludeLabels` is optional product/config scope.
