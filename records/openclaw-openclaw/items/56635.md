## Summary

Keep open. Current main still cannot pass excluded Gmail labels into `gog gmail watch serve`, the strict Gmail config surface still has no `excludeLabels` field, and upstream `gog` defaults only cover `SPAM,TRASH`; because several open PRs already use closing syntax for this issue, the maintainer path is to choose and land one canonical fix rather than close the issue early.

## What This Changes

The issue requests restoring Gmail watcher excluded-label handling so `SPAM,TRASH,DRAFT,SENT` are excluded by default, with an optional typed `hooks.gmail.excludeLabels` override.

## Best Possible Solution

Land one canonical fix, preferably the fuller schema/config/docs/test direction represented by #56720: restore default excluded labels to `SPAM,TRASH,DRAFT,SENT`, add a typed `hooks.gmail.excludeLabels` override with documented default and intentionally-empty behavior, update generated config metadata/docs, and add regression coverage for default and custom label sets.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to resolve the default Gmail hook config and inspect `buildGogWatchServeArgs` or `buildGogWatchServeLogArgs`; current main has no config field or builder branch that can emit `--exclude-labels`.

## Solution Assessment

Yes. Keeping this issue open while maintainers choose and land one linked fix PR is the best path; a configurable default is more maintainable than only restoring a hardcoded flag because it fixes both the default behavior and the reported config gap.
