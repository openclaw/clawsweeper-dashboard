## Summary

Keep this open: current main and the latest release still omit `--exclude-labels` from the shared Gmail watcher args, while upstream `gog` defaults only exclude `SPAM,TRASH`. The linked fix at https://github.com/openclaw/openclaw/pull/56720 is still open and conflicting, so this issue should stay paired with that implementation path until a refreshed fix lands.

## What This Changes

Restore Gmail watcher excluded-label handling so `gog gmail watch serve` filters SPAM, TRASH, DRAFT, and SENT events by default, optionally via `hooks.gmail.excludeLabels`.

## Best Possible Solution

Carry the excluded-label fix onto current main, preferably through a refreshed or replacement version of https://github.com/openclaw/openclaw/pull/56720, then close this issue from the merged fix.

## Reproduction Assessment

Yes, at source level: current main's shared Gmail watcher argument builder does not append `--exclude-labels`, both watcher callers use that helper, and upstream `gog` defaults only exclude `SPAM,TRASH`. I did not run a live Gmail watcher in this read-only review.

## Solution Assessment

Yes: fixing the shared Gmail watcher arg builder is the narrow maintainable fix because it covers gateway auto-start and `openclaw webhooks gmail run`. The linked PR takes that path, but it needs refresh or replacement before the issue can close.
