## What This Changes

The issue reports that the Gmail watcher no longer passes `--exclude-labels SPAM,TRASH,DRAFT,SENT`, so sent mail and draft saves can trigger Gmail hook processing.

## Best Possible Solution

Refresh and land the linked fix PR, or a narrow replacement, so the shared Gmail watcher builder restores default DRAFT/SENT filtering before this issue closes.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` build the shared Gmail watcher argv without `--exclude-labels`, while gogcli defaults only exclude `SPAM,TRASH`.

## Solution Assessment

Yes for the narrow default restoration in `buildGogWatchServeArgs()`. The broader `hooks.gmail.excludeLabels` config surface in the open PR needs maintainer acceptance, but the issue should stay open until some candidate fix merges.
