## Summary

Keep open: the focused bug fix is valid and has real terminal proof, but the latest PR head still misses generated Claude ACP wrapper commands launched by `node.exe` on Windows, so the unsupported timeout option can still leak there.

## What This Changes

The PR gates ACP timeout config-option emission when advertised keys omit timeout, suppresses Claude ACP timeout forwarding in ACPX, and adds focused regression tests.

## Best Possible Solution

Keep the control-plane advertisement gate and ACPX timeout suppression, then make the Claude command detector cover all supported generated wrapper launchers and add a regression test.

## Reproduction Assessment

Yes. Current main source emits `timeout` from `timeoutSeconds` and rejects/forwards it on the ACPX path, while upstream `claude-agent-acp@0.33.1` rejects unknown config IDs; the PR body also supplies terminal output for the backend rejection and patched behavior.

## Solution Assessment

No, not yet. The proposed direction is the narrow maintainable fix, but the detector must also recognize `node.exe` generated wrapper commands before merge.
