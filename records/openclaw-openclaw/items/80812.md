## What This Changes

The branch gates ACP timeout config-option emission when advertised keys omit timeout, suppresses Claude ACP timeout forwarding in ACPX, and adds regression coverage.

## Best Possible Solution

Land this focused timeout suppression after normal maintainer gating; broader unsupported-option diagnostics or future Claude timeout support can remain follow-up work.

## Reproduction Assessment

Yes. Current main maps `sessions_spawn timeoutSeconds` into ACP `runtimeOptions.timeoutSeconds`, builds a `timeout` config option, and forwards it to non-Codex ACPX commands; `claude-agent-acp@0.33.1` rejects config ids it does not advertise.

## Solution Assessment

Yes. The PR fixes the narrow broken path by omitting timeout config options when advertised keys exclude timeout and by suppressing Claude ACP timeout forwarding, while preserving OpenClaw's in-process turn timeout and non-timeout config options.
