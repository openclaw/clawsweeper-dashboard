## Summary

Keep open. Current main does not implement several intended changes, and the PR branch is not ready to merge: the reset-memory hook bypasses current memory-flush model/config/path contracts, the browser deferral code targets a removed legacy path, and the UI width change omits persisted-settings validation. The related pre-reset memory-flush request remains open in #45608, so this PR should be split, rebased, or replaced with focused work.

## What This Changes

The PR branch increases the macOS wizard connect-challenge timeout, widens the Control UI sidebar limit, adds Brave LLM-context filter wording, counts active browser profiles in restart deferral, adds a reset-triggered memory-flush hook, and documents skill-creator auditing.

## Best Possible Solution

Split or replace the branch with focused changes. Low-risk macOS timeout, Control UI persistence, Brave docs/tool wording, and skill audit docs can be reviewed separately; pre-reset memory flush should reuse the current memory flush plan/model fallback/path helpers and resolve #45608's blocking/background semantics; browser restart deferral should use a current plugin service or generic active-task seam.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection of the PR diff against current main contracts is enough: the proposed reset hook uses the active session model, omits runtime config, falls back to a cwd-relative transcript, and registers append-only hooks from startup.

## Solution Assessment

Is this the best way to solve the issue? No. The goals are valid, but the current patch mixes unrelated work and implements the riskiest piece outside the established memory-flush and current browser-plugin seams.

- **[P1] Honor the memory-flush model override:** `src/gateway/session-reset-memory-flush.ts:43`
  - body: The reset hook resolves provider/model from the active session instead of `agents.defaults.compaction.memoryFlush.model`. Users who configured a local/private housekeeping model can have reset-triggered transcript flushes sent to the conversation provider instead.
  - confidence: 0.9
- **[P1] Pass config into the embedded flush run:** `src/gateway/session-reset-memory-flush.ts:62-77`
  - body: `runEmbeddedPiAgent` is called without `config: cfg`, so provider plugins, model fallback, workspace resolution, and tool policy may not match the gateway runtime. Custom provider or agent settings can make the reset flush fail or run with wrong defaults.
  - confidence: 0.88
- **[P1] Derive the transcript path instead of using cwd:** `src/gateway/session-reset-memory-flush.ts:73`
  - body: `SessionEntry.sessionFile` is optional, and `entry.sessionFile ?? "session.jsonl"` points at the process working directory for legacy or migrated rows. That can flush the wrong transcript immediately before reset.
  - confidence: 0.9
- **[P2] Use the current browser plugin surface:** `src/gateway/server.impl.ts:454-459`
  - body: This wires restart deferral through legacy `src/browser`/`server-browser` code, but current main has moved browser control into `extensions/browser`. As written, the browser-deferral change is stale and will not cover active browser automation on current main.
  - confidence: 0.82
- **[P2] Persist the wider nav width:** `ui/src/ui/app-render.ts:264`
  - body: The renderer allows dragging the sidebar to 600px, but persisted settings still accept only values up to 400px. Widths above 400px will be discarded on reload unless the storage contract and tests are updated too.
  - confidence: 0.86
- **[P2] Make reset hook registration idempotent:** `src/gateway/session-reset-memory-flush.ts:18-19`
  - body: `registerInternalHook` appends to a global handler list, and this wrapper is called from gateway startup. Repeated starts in one process can register the same reset flush multiple times and run duplicate model calls on each `/new` or `/reset`.
  - confidence: 0.82
