## What This Changes

The branch increases the macOS wizard timeout, widens the dashboard nav, updates Brave/skill docs, adds browser restart-deferral counting, and registers a reset-time memory-flush hook.

## Best Possible Solution

Split and rebase the branch; keep https://github.com/openclaw/openclaw/issues/45608 as the canonical reset-memory design thread, and land only narrow changes with focused tests, correct outgoing-session handling, runtime config/sandbox preservation, transcript resolution, idempotent hook registration, warning logs, and real behavior proof.

## Reproduction Assessment

## Solution Assessment

No. The reset-memory direction is plausible, but this implementation bypasses established runtime context, uses an unsafe transcript fallback, and leaves reset-time agent-run policy unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Flush the outgoing reset session:** `src/gateway/session-reset-memory-flush.ts:31-37`
  - body: `command:new` and `command:reset` events carry `previousSessionEntry`, but this hook reads `context.sessionEntry`, which can be the fresh zero-token session after rollover. Use the outgoing entry/transcript so the flush preserves the conversation being reset.
  - confidence: 0.9
- **[P1] Forward runtime config to the flush run:** `src/gateway/session-reset-memory-flush.ts:59-77`
  - body: `runEmbeddedPiAgent` resolves harnesses, model/provider aliases, auth profiles, and policy from `params.config`. Loading `cfg` but not passing `config: cfg` can make reset-time flushes run with the wrong model/auth/sandbox behavior or fail silently.
  - confidence: 0.88
- **[P2] Resolve missing sessionFile through transcript helpers:** `src/gateway/session-reset-memory-flush.ts:71`
  - body: `SessionEntry.sessionFile` is optional, but falling back to literal `session.jsonl` points at the process working directory. Use the existing transcript/session-file resolver so legacy or repaired entries flush the intended session transcript.
  - confidence: 0.86
- **[P2] Guard reset hook registration:** `src/gateway/session-reset-memory-flush.ts:19-21`
  - body: `registerInternalHook` appends handlers to a global list. If gateway startup runs more than once in a process, this registers duplicate reset hooks and can launch multiple concurrent 60-second flushes for one reset.
  - confidence: 0.81
- **[P3] Warn on flush failures:** `src/gateway/session-reset-memory-flush.ts:79-82`
  - body: Provider errors or timeouts are only logged through `logVerbose`, so failures of this preservation path are invisible in normal operator logs. Surface failures at gateway warning level while still allowing reset to continue.
  - confidence: 0.75
