## Summary

Keep open. Current main does not implement the full PR, and the patch remains a broad stale bundle with blocking correctness and security issues in the reset-triggered memory flush plus obsolete browser/UI/tool surfaces.

## What This Changes

The PR branch increases the macOS wizard connection timeout, adds reset-triggered agentic memory flushing, includes browser activity in gateway restart deferral, widens the Control UI nav limit, and updates web-search and skill-creator documentation.

## Best Possible Solution

Split or replace this branch with focused PRs: a correct #45608 reset-flush implementation using the existing memory-flush contracts, a current-surface restart-deferral change if still needed, and independent macOS/UI/docs updates with matching tests and changelog.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection of the PR diff against current main shows the reset hook uses the wrong model/config/path contracts and several changed surfaces no longer match current main.

## Solution Assessment

Is this the best way to solve the issue? No. The goals are plausible, but the reset-memory feature should reuse the established memory-flush plan and current reset/session APIs, while the unrelated timeout/UI/docs changes should be reviewed separately.

Full review comments:

- **[P1] Honor the configured memory-flush model:** `src/gateway/session-reset-memory-flush.ts:43`
  - body: The reset hook resolves provider/model from the active session instead of the configured memory-flush plan. Users who set a local/private housekeeping model can have reset-triggered transcript flushes sent to the conversation provider.
  - confidence: 0.9
- **[P1] Pass runtime config into the reset flush run:** `src/gateway/session-reset-memory-flush.ts:62-77`
  - body: runEmbeddedPiAgent is called without the loaded runtime config, so provider plugins, auth/profile resolution, fallback selection, and tool policy can diverge from the running gateway during the automatic flush.
  - confidence: 0.88
- **[P1] Resolve the actual transcript path before flushing:** `src/gateway/session-reset-memory-flush.ts:73`
  - body: Falling back to entry.sessionFile ?? "session.jsonl" can read a cwd-relative file for legacy or migrated sessions. The flush should use the existing session transcript resolution helpers so it reads the current session transcript.
  - confidence: 0.9
- **[P2] Use the current browser plugin surface:** `src/gateway/server.impl.ts:454-459`
  - body: The restart deferral change is wired through legacy src/browser/server.ts and src/gateway/server-browser.ts paths that are gone on current main, so it cannot protect current browser automation sessions without being ported to the plugin/task surface.
  - confidence: 0.82
- **[P2] Persist the wider nav width:** `ui/src/ui/app-render.ts:264`
  - body: Only the renderer limit is raised to 600px. Persisted settings still accept navWidth only through 400px, so widths above 400px are discarded on reload unless the storage contract and tests are updated.
  - confidence: 0.86
- **[P2] Make reset hook registration idempotent:** `src/gateway/session-reset-memory-flush.ts:19-21`
  - body: registerInternalHook appends to a global handler list, and this wrapper is called from gateway startup. Repeated starts in one process can register duplicate reset flush handlers and run multiple model calls per /new or /reset.
  - confidence: 0.82
- **[P3] Surface reset flush failures in normal logs:** `src/gateway/session-reset-memory-flush.ts:79-82`
  - body: Flush failures are caught and logged only at verbose level. In normal production logging, a failed or timed-out pre-reset memory flush is invisible even though the reset continues without writing the intended memory.
  - confidence: 0.74
