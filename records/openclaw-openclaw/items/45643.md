## Summary

Keep open: the branch contains plausible stability and UX improvements, but the reset-memory implementation still has source-level blocking bugs, lacks contributor real behavior proof, and needs maintainer policy/security review before merge.

## What This Changes

This PR raises the macOS wizard timeout, widens the dashboard nav, updates Brave/skill docs, adds browser restart-deferral counting, and adds a reset-time memory flush hook.

## Best Possible Solution

Split and rebase the branch, keep the reset-memory design anchored to https://github.com/openclaw/openclaw/issues/45608, and land only narrow changes with focused tests, correct outgoing-session handling, runtime config preservation, transcript resolution, idempotent hooks, visible failure logging, and real behavior proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level for the blocking PR bugs: current reset hooks expose previousSessionEntry and cfg, while this patch reads only sessionEntry and starts the embedded flush without forwarding config.

## Solution Assessment

Is this the best way to solve the issue? No. The reset-memory direction is plausible, but this implementation bypasses established runtime context, uses an unsafe transcript fallback, and leaves reset-time agent-run policy unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Flush the outgoing reset session:** `src/gateway/session-reset-memory-flush.ts:31-37`
  - body: command:new and command:reset events carry previousSessionEntry, but this hook reads context.sessionEntry, which can be the fresh or wrong reset-side entry. Use the outgoing entry/transcript so the flush preserves the conversation being reset.
  - confidence: 0.9
- **[P1] Forward runtime config to the flush run:** `src/gateway/session-reset-memory-flush.ts:59-77`
  - body: runEmbeddedPiAgent resolves harnesses, model/provider aliases, auth profiles, plugins, and policy from params.config. Loading cfg but not passing config: cfg can make reset-time flushes run with the wrong model/auth/sandbox behavior or fail silently.
  - confidence: 0.88
- **[P2] Resolve missing sessionFile through transcript helpers:** `src/gateway/session-reset-memory-flush.ts:71`
  - body: SessionEntry.sessionFile is optional, but falling back to literal session.jsonl points at the process working directory. Use the existing session transcript resolver so legacy or repaired entries flush the intended transcript.
  - confidence: 0.86
- **[P2] Guard reset hook registration:** `src/gateway/session-reset-memory-flush.ts:19-21`
  - body: registerInternalHook appends handlers to a global list. If gateway startup runs more than once in a process, this registers duplicate reset hooks and can launch multiple concurrent 60-second flushes for one reset.
  - confidence: 0.81
- **[P3] Warn on flush failures:** `src/gateway/session-reset-memory-flush.ts:79-82`
  - body: Provider errors or timeouts are only logged through logVerbose, so failures of this preservation path are invisible in normal operator logs. Surface failures at gateway warning level while still allowing reset to continue.
  - confidence: 0.75
