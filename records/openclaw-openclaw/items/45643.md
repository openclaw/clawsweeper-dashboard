## Summary

Keep open. This still contains useful work that current main has not fully implemented, but the branch is a stale multi-surface bundle and the reset-memory hook has blocking correctness and privacy-sensitive issues. The browser restart-deferral piece also targets a code shape that has since moved into the browser plugin service.

## What This Changes

The PR branch bundles a macOS wizard timeout increase, a Brave web-search description note, browser restart-deferral counting, a new pre-reset memory-flush hook, a Control UI sidebar width change, and skill-creator audit documentation.

## Best Possible Solution

Split or replace the branch with focused changes. Low-risk macOS timeout, Control UI width/storage, and skill docs changes can be reviewed independently; the reset-memory hook should reuse the existing memory flush plan/model override path, pass runtime config, resolve transcript paths through session helpers, and register idempotently; browser restart deferral should use a current browser plugin service or generic active-task seam.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection of the proposed diff plus current contracts shows the reset hook calls runEmbeddedPiAgent without config, ignores the memoryFlush model override, and uses entry.sessionFile ?? "session.jsonl" even though SessionEntry.sessionFile is optional.

## Solution Assessment

Is this the best way to solve the issue? No. The PR has valid goals, but the current patch mixes unrelated work and implements the riskiest piece outside the existing memory-flush plan/runner and current browser plugin seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor the memory-flush model override:** `src/gateway/session-reset-memory-flush.ts:43`
  - body: The reset flush resolves provider/model from the active session, so `agents.defaults.compaction.memoryFlush.model` is ignored. Users who configured a local/private maintenance model can have `/new` or `/reset` send the transcript to the conversation model instead.
  - confidence: 0.9
- **[P1] Pass config into the embedded flush run:** `src/gateway/session-reset-memory-flush.ts:62-77`
  - body: `runEmbeddedPiAgent` receives no `config`, but the embedded runner uses that config for provider plugins, model fallback, workspace resolution, and tool policy. Custom provider or agent settings can fail during reset and silently skip the intended flush.
  - confidence: 0.88
- **[P1] Derive the missing session transcript path:** `src/gateway/session-reset-memory-flush.ts:73`
  - body: `SessionEntry.sessionFile` is optional, and falling back to `"session.jsonl"` points at the process working directory instead of the stored transcript for that `sessionId`. Legacy or migrated sessions can flush the wrong file right before reset.
  - confidence: 0.9
- **[P2] Make reset hook registration idempotent:** `src/gateway/session-reset-memory-flush.ts:17-19`
  - body: `registerInternalHook` appends to global handlers, and this wrapper is called from gateway startup. If the server starts more than once in a process, each reset will run duplicate LLM flushes.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds reset-triggered model execution and currently bypasses the configured memory-flush model boundary, which is a concrete privacy-sensitive regression.

Concerns:

- **[medium] Reset flush can use the wrong model boundary:** `src/gateway/session-reset-memory-flush.ts:43`
  - body: The proposed hook uses the active session provider/model instead of `agents.defaults.compaction.memoryFlush.model`, so transcripts can be sent to a cloud conversation model even when the operator configured a local/private memory-flush model.
  - confidence: 0.9
- **[medium] Runtime config is not passed to the embedded run:** `src/gateway/session-reset-memory-flush.ts:62`
  - body: Omitting `config: cfg` means provider/plugin/auth/tool-policy behavior may not match the gateway runtime during reset-triggered model execution.
  - confidence: 0.82
