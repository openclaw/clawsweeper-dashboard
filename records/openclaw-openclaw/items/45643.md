## Summary

Keep open. The branch is not implemented on current main and the proposed patch has blocking correctness and security issues in the reset-triggered memory flush path, plus stale browser and UI surfaces.

## What This Changes

The PR branch increases the macOS wizard connection timeout, adds a reset-triggered memory flush hook, includes browser activity in restart deferral, widens the Control UI nav limit, and updates Brave/skill-creator documentation.

## Best Possible Solution

Replace or split the branch into focused PRs. The reset memory flush should reuse the existing memory-flush plan, exact model override, transcript resolution, runtime config, and reset semantics from #45608; browser deferral should use the current browser plugin/task surface; UI width changes should update storage/CSS/tests together; the timeout and docs changes can be reviewed independently.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection of the PR diff against current main contracts is enough to show the reset hook uses the wrong model/config/path, appends duplicate hook registrations, and targets browser/UI files or contracts that no longer match current main.

## Solution Assessment

Is this the best way to solve the issue? No. The goals are plausible, but this broad patch bypasses established memory-flush contracts and current plugin/UI surfaces; focused replacement PRs are the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor the configured memory-flush model:** `src/gateway/session-reset-memory-flush.ts:43`
  - body: The reset hook resolves provider/model from the active session instead of the configured memory-flush plan. Users who set a local/private housekeeping model can have reset-triggered transcript flushes sent to the conversation provider.
  - confidence: 0.9
- **[P1] Pass runtime config into the reset flush run:** `src/gateway/session-reset-memory-flush.ts:62-77`
  - body: runEmbeddedPiAgent is called without config: cfg, so provider plugins, auth/profile resolution, fallback selection, and tool policy can diverge from the running gateway during the automatic flush.
  - confidence: 0.88
- **[P1] Resolve the actual transcript path before flushing:** `src/gateway/session-reset-memory-flush.ts:73`
  - body: Falling back to entry.sessionFile ?? "session.jsonl" can read a cwd-relative file for legacy or migrated sessions. The flush should use the existing session transcript resolution helpers so it reads the current session transcript.
  - confidence: 0.9
- **[P2] Use the current browser plugin surface:** `src/gateway/server.impl.ts:454-459`
  - body: The restart deferral change is wired through legacy src/browser/server-browser paths that are gone on current main. Browser control now lives under extensions/browser, so this does not protect current browser automation sessions.
  - confidence: 0.82
- **[P2] Persist the wider nav width:** `ui/src/ui/app-render.ts:264`
  - body: Only the renderer max is raised to 600px. Persisted settings still accept navWidth only up to 400px, so widths above 400px are discarded on reload unless the storage contract and tests are updated.
  - confidence: 0.86
- **[P2] Make reset hook registration idempotent:** `src/gateway/session-reset-memory-flush.ts:19-21`
  - body: registerInternalHook appends to a global handler list, and this wrapper is called from gateway startup. Repeated starts in one process can register duplicate reset flush handlers and run multiple model calls per /new or /reset.
  - confidence: 0.82
- **[P3] Surface reset flush failures in normal logs:** `src/gateway/session-reset-memory-flush.ts:79-82`
  - body: Flush failures are caught and logged only at verbose level. In normal production logging, a failed or timed-out pre-reset memory flush is invisible even though the reset continues without writing the intended memory.
  - confidence: 0.74
