## Summary

Keep open: the goals are plausible, but the branch is stale against current main and the reset-memory feature bypasses current memory/model/runtime contracts, so it should not merge as-is.

## What This Changes

The PR branch raises the macOS wizard connection timeout, adds reset-triggered agentic memory flushing, includes browser activity in restart deferral, widens the Control UI nav limit, and updates web-search and skill-creator documentation.

## Best Possible Solution

Split or replace the branch with focused current-main PRs: solve #45608 through the existing memory-flush plan and reset/session APIs first, then handle the macOS timeout, UI width, browser deferral, and docs updates independently with matching tests and changelog entries.

## Reproduction Assessment

Yes. Static inspection of the PR diff against current main shows the reset hook uses the wrong model/config/path contracts, and several changed files or assumptions no longer match current main.

## Solution Assessment

No. The central reset-memory behavior should reuse the established memory-flush plan, runtime config, model fallback, and transcript resolution helpers; the unrelated UI, macOS, browser, and docs changes should be reviewed separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor the configured memory-flush model:** `src/gateway/session-reset-memory-flush.ts:43-45`
  - body: The reset hook resolves provider/model from the active session instead of the configured memory-flush plan. Users who configured a local or private housekeeping model could have reset-triggered transcript flushing sent to the conversation provider.
  - confidence: 0.9
- **[P1] Pass runtime config into the reset flush run:** `src/gateway/session-reset-memory-flush.ts:62-77`
  - body: The direct runEmbeddedPiAgent call does not pass the loaded runtime config, so provider plugins, auth/profile resolution, fallback selection, and tool policy can diverge from the running gateway during the automatic flush.
  - confidence: 0.88
- **[P1] Resolve the actual transcript path before flushing:** `src/gateway/session-reset-memory-flush.ts:73`
  - body: Falling back to entry.sessionFile ?? "session.jsonl" can read a cwd-relative file for legacy or migrated sessions. The flush should use the existing session transcript resolution helpers so it reads the current session transcript.
  - confidence: 0.9
- **[P2] Use the current browser plugin surface:** `src/browser/server.ts:97-114`
  - body: The restart-deferral change is wired through legacy src/browser/server.ts and server-browser plumbing, while current main has the browser runtime under extensions/browser and already defers on active task-registry work. This would not reliably protect current browser automation sessions.
  - confidence: 0.82
- **[P2] Persist the wider nav width:** `ui/src/ui/app-render.ts:264`
  - body: Only the renderer limit is raised to 600px. Persisted settings still accept navWidth only through 400px, so widths above 400px are discarded on reload unless the storage contract and tests are updated.
  - confidence: 0.86
- **[P2] Make reset hook registration idempotent:** `src/gateway/session-reset-memory-flush.ts:19-21`
  - body: registerInternalHook appends to a global handler list, and this wrapper is called from gateway startup. Repeated starts in one process can register duplicate reset flush handlers and run multiple model calls per /new or /reset.
  - confidence: 0.82
- **[P2] Do not document supported Brave filters as unsupported:** `src/agents/tools/web-search.ts:1924`
  - body: Current Brave LLM Context support documents freshness and bounded date_after/date_before ranges. The PR's tool description says those filters are unsupported, which would mislead agents and users on current main.
  - confidence: 0.78
- **[P3] Surface reset flush failures in normal logs:** `src/gateway/session-reset-memory-flush.ts:79-82`
  - body: Flush failures are caught and logged only at verbose level. In normal production logging, a failed or timed-out pre-reset memory flush is invisible even though the reset continues without writing the intended memory.
  - confidence: 0.74
