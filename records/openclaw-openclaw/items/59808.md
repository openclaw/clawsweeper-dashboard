## Summary

## What This Changes

The issue reports that the first qmd-backed `memory_search` after a gateway restart can fall back to FTS-only because mcporter stdout makes the JSON result unparsable.

## Best Possible Solution

Keep this issue open until a focused fix proves the contaminated mcporter call output, then make JSON-mode stdout pure at the owner boundary or add a tested structured parser in the qmd mcporter path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows malformed mcporter call stdout would throw at `JSON.parse` and trigger fallback, but maintainer review of the linked patch disproved the specific daemon-start stdout contamination path.

## Solution Assessment

No. The proposed `[mcporter]` line filter was closed because it targets the wrong boundary and misses unprefixed daemon messages; the best fix needs live or dependency-backed proof of the actual contaminated output source.
