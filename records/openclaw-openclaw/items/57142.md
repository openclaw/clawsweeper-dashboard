## Summary

Keep open. Current main and the latest release still only provide slash-command and static welcome suggestions in the WebChat/Control UI composer; the related autocomplete, typo-detection, and dictation issues are adjacent but do not supersede configurable custom-term suggestions, and choosing the storage/API boundary needs maintainer product direction.

## What This Changes

Request configurable WebChat/Control UI chat-input suggestions for project-specific custom terms, especially to help typed or dictated proper nouns.

## Best Possible Solution

Decide a single composer suggestion contract first, then implement accessible custom-term matching in WebChat/Control UI with the chosen persistence, schema/docs, keyboard behavior, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection of the current composer, slash menu, tests, docs, and config schema is a high-confidence way to verify the requested configurable custom-term suggestions are absent.
