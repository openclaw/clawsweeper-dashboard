## Summary

Keep open. Current main still only has slash-command and fixed-argument completion plus static welcome prompts; it does not expose configurable custom-term suggestions in the Control UI/WebChat composer, gateway config, docs, or native chat composers. The remaining blocker is a maintainer product/API choice about where terms live and whether the feature is browser-only or shared across clients.

## What This Changes

Add user-configurable custom-term autocomplete or suggestions to the WebChat/Control UI chat input for project-specific terminology and dictation workflows.

## Best Possible Solution

Define one durable custom-term suggestion contract, then implement partial-match insertion in the Control UI chat composer. If the terms are gateway-configured, update config schema/help/docs and generated docs; if shared across clients, extend the native/mobile composers through the same contract.

## Reproduction Assessment

Yes. For this feature-absence report, source inspection of the composer, slash menu, WebChat config/schema, docs, and native/mobile composers is a high-confidence reproduction path, and those surfaces do not expose configurable custom-term suggestions today.

## Solution Assessment

Unclear. Autocomplete is a reasonable UX direction, but the issue leaves persistence and client scope open, so maintainers should choose the smallest durable product/API shape before implementation.
