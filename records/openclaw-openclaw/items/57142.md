## Summary

Current main still does not implement configurable custom-term autocomplete for the WebChat or Control UI chat input. The report is recent, understandable, and actionable, but the remaining decision is the product/API shape for persistence and cross-client scope, so this should stay open for maintainer direction rather than be swept closed.

## What This Changes

Add autocomplete or suggestions for user-configured project-specific terms in the WebChat or Control UI chat input.

## Best Possible Solution

Define one durable custom-term suggestion contract first, then implement partial-match suggestions in the Control UI chat composer and update schema/help/docs/tests if the terms are gateway-configured. If native apps should share the feature, extend the same contract to the SwiftUI and Android composers rather than creating browser-only behavior by accident.

## Reproduction Assessment

Yes. For this feature-absence report, inspecting current main’s composer, suggestion, config, and docs surfaces is a high-confidence reproduction path; none exposes configurable custom-term autocomplete today.

## Solution Assessment

Unclear. Autocomplete is a reasonable UX direction, but the issue intentionally leaves storage and cross-client scope open, so the best fix depends on a maintainer choice before implementation.
