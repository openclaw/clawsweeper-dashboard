## Summary

Keep open. Current main still fails closed when memory has indexed chunks but no provider and FTS is unavailable, while this PR’s current head still has merge-blocking vector-only, hybrid-ranking, and source-filter regressions plus no real behavior proof.

## What This Changes

The branch adds a memory-core chunk-scan fallback when FTS5 is unavailable, re-enables related memory tests, skips trigram tests without FTS5, and carries stale fixture updates outside memory-core.

## Best Possible Solution

Land a rebased memory-core-only fallback that scans `chunks` only for the no-provider/no-FTS lexical path, preserves vector-only and merged hybrid ranking semantics, and threads current source filters through the fallback.

## Reproduction Assessment

Yes, by source inspection. With indexed `chunks`, no embedding provider, and unavailable or disabled FTS, current main returns `[]` before querying `chunks`, while the schema allows `chunks` to exist without `chunks_fts`.

## Solution Assessment

No. The fallback direction is useful, but this PR head is not the best mergeable fix because it reaches vector-only paths, bypasses merged hybrid ranking, and does not carry current source filters into fallback scans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve vector-only mode when fallback is lexical:** `extensions/memory-core/src/memory/manager.ts:400-401`
  - body: This branch also runs when `hybrid.enabled` is false. If the query vector is zero or vector search returns no hits, vector-only configurations now scan chunks lexically and can return substring matches where the previous vector-only path returned no lexical results.
  - confidence: 0.9
- **[P2] Keep relaxed keyword fallback on merged ranking:** `extensions/memory-core/src/memory/manager.ts:420`
  - body: Returning raw `keywordResults` skips `mergeHybridResults`, including temporal decay and MMR ordering. The relaxed fallback should preserve the merged keyword-backed entries so stale or duplicate chunks do not jump ahead of the configured ranking policy.
  - confidence: 0.88
- **[P2] Thread source filters into fallback scans:** `extensions/memory-core/src/memory/manager.ts:526-535`
  - body: The fallback helper builds a filter from all enabled sources. Current main passes requested source filters into vector and FTS searches before ranking and limiting, so after rebase this fallback can rank and limit against the wrong corpus and suppress requested hits.
  - confidence: 0.84
