The PR adds a chunks-table keyword fallback for memory search when FTS5 is unavailable, re-enables or skips related memory-core tests, and carries nearby test/fixture adjustments.

## Best Possible Solution

Yes, at source level: current main records FTS unavailable when FTS5 table creation fails, and the no-provider search path then returns an empty result set instead of searching indexed chunks. I did not run a live Node build without FTS5 in this read-only review.
