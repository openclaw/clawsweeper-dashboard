## Summary

Keep open: current main still has the no-FTS memory search gap, but this PR is not mergeable because the fallback still leaks into vector-only/provider paths, bypasses hybrid ranking/source-filter semantics, and lacks real FTS-unavailable runtime proof.

## What This Changes

The branch adds a chunks-table keyword fallback for memory search when FTS5 is unavailable, re-enables memory-core tests, skips FTS-dependent helper tests, and adjusts CLI/gateway/doctor test fixtures.

## Best Possible Solution

Land a rebased memory-core fallback that scans chunks only for the no-provider/no-FTS lexical path, threads source filters, preserves vector/hybrid ranking, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main creates chunks before attempting chunks_fts, records FTS as unavailable on create failure, and the no-provider search path returns [] when FTS is unavailable. I did not run a live affected Node build in this read-only review.

## Solution Assessment

No. The fallback direction is appropriate, but this PR head leaks lexical scanning into vector-only/provider paths, bypasses merged hybrid ranking, misses current source filters, and needs runtime proof.

- **[P2] Keep lexical fallback out of vector-only searches:** `extensions/memory-core/src/memory/manager.ts:400-401`
  - body: This branch also runs when `query.hybrid.enabled` is false or FTS is unavailable for a provider-backed search.   - confidence: 0.9

  - body: Returning `keywordResults` directly when `strict` is empty skips `mergeHybridResults`, including temporal decay and optional MMR ordering. The relaxed keyword fallback should keep the already-merged entries so stale or duplicate chunks do not bypass the configured ranking policy.
  - confidence: 0.86

  - body: Current main passes requested source filters into vector and FTS searches before ranking and limiting. This fallback builds the default source filter only, so after rebase it can rank and limit against the wrong corpus and suppress requested hits.
  - confidence: 0.84
