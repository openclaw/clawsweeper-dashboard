## What This Changes

The issue requests an opt-in ACP stream setting that relays Codex assistant `phase: "commentary"` progress text to parent sessions without changing final-answer delivery.

## Best Possible Solution

Add a narrow, default-off ACP parent-stream commentary setting only after maintainer approval, wiring it through config schema/types/docs and the existing bounded relay buffer so commentary can be visible without becoming final-answer text.
