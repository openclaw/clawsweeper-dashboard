## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep signal handling out of the shared gateway helper:** `src/gateway/call.ts:707-708`
  - body: `executeGatewayRequestWithScopes` is shared by gateway callers, but these `process.on` listeners remove Node's default SIGINT/SIGTERM exit behavior while any gateway call is active. A caller that catches the rejected promise can keep running after Ctrl+C or supervisor SIGTERM, so signal handling should live at the CLI boundary and drive the existing AbortSignal path.
  - confidence: 0.86
- **[P2] Validate `--timeout` with the shared strict parser:** `src/cli/program/message/helpers.ts:37-38`
  - body: Using `parseInt` accepts partial values like `1abc`, and invalid or non-positive values silently fall back to the default timeout instead of failing. Existing message CLI numeric options reject malformed values before `messageCommand` runs, so scripts need the same clear error for a bad timeout value.
  - confidence: 0.9
