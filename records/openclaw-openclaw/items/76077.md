## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Handle process signals at the message CLI boundary:** `src/gateway/call.ts:707-708`
  - body: This helper backs every gateway RPC, not only `openclaw message send`. Installing process-wide SIGTERM/SIGINT listeners here suppresses Node's default termination while any gateway call is active and turns the signal into a request error that non-CLI callers may catch and continue from. Use a command-scoped AbortController in the message CLI path, following the existing agent CLI signal bridge pattern, and pass that signal through instead.
  - confidence: 0.9
- **[P2] Reject malformed timeout values strictly:** `src/cli/program/message/helpers.ts:36`
  - body: `parseInt` prefix-parses values such as `1s` as `1` and `1000ms` as `1000`, while invalid values silently fall back to the default. Other CLI numeric flags use strict positive-integer validation and fail before dispatch, so a typo here can either ignore the caller's timeout or make sends time out almost immediately.
  - confidence: 0.88
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:7-9`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message instead, and remove this added Unreleased entry before merge.
  - confidence: 0.92
