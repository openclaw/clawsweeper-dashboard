## Summary

Keep open. The force-push fixed the earlier listener-leak and docs/changelog gaps, but the new timeout parser still accepts invalid values and CI has a lint failure; the external PR also lacks observable real behavior proof.

## What This Changes

This PR adds a `--timeout <ms>` option to `openclaw message send`, forwards it into gateway message dispatch, adds SIGTERM/SIGINT cancellation in gateway calls, and updates docs, changelog, and a listener-cleanup test.

## Best Possible Solution

Merge a repaired version that validates positive timeout input, keeps signal cancellation scoped and disposable, includes docs/changelog/tests, and adds real terminal or log proof for an interrupted stuck `message send`.

## Reproduction Assessment

Yes for the PR-introduced timeout parsing behavior: source inspection shows invalid strings and negative values can be forwarded without a CLI error. The original production CPU-spin/SIGTERM symptom remains source-supported but not live-reproduced here.

## Solution Assessment

No. The direction is reasonable and the earlier listener-cleanup/docs gaps are addressed, but the timeout parsing and proof gaps make this not merge-ready yet.
