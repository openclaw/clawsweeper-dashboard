## Summary

Keep open. The paired bug is still open and current main still lacks the requested command-level timeout/SIGTERM behavior, but this PR needs real behavior proof and two small review fixes before it is mergeable.

## What This Changes

The PR adds a `--timeout <ms>` option to `openclaw message send`, threads it into gateway message dispatch, adds scoped SIGTERM/SIGINT abort handling for gateway calls, and updates tests, docs, and changelog.

## Best Possible Solution

Merge a repaired branch that bounds message-send delivery, keeps signal cancellation scoped and disposable, reports the actual abort signal, passes lint, and includes redacted real terminal/log proof.

## Reproduction Assessment

The original hang is source-supported by current main lacking the requested command timeout and signal wiring, but it was not live-replayed here.

## Solution Assessment

No, not as-is. The timeout and scoped signal-cancellation direction is reasonable, but this branch needs the lint fix, the signal-name correction, and real behavior proof before it is the best merge candidate.

- **[P2] Use Number.parseInt for timeout parsing:** `src/cli/program/message/helpers.ts:38`
  - body: The added timeout parser uses bare `parseInt`, but this repo enables `unicorn/prefer-number-properties` as an error and the exact PR head has a failing `check-lint` status. Replace it with `Number.parseInt` so lint can pass.
