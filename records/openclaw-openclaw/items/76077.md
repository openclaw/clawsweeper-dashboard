## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Move signal handling out of callGateway:** `src/gateway/call.ts:706-707`
  - body: Registering SIGTERM/SIGINT inside `executeGatewayRequestWithScopes` changes process-wide signal behavior for every `callGateway` caller, not only `openclaw message send`. In Node, adding a signal listener prevents default termination for that signal, so callers that catch this rejection can keep running after a supervisor tried to stop the process. Please keep signal handling at the CLI command boundary and pass an AbortSignal into the gateway request instead.
  - confidence: 0.84
- **[P2] Validate timeout as a strict positive integer:** `src/cli/program/message/helpers.ts:36-39`
  - body: `parseInt` accepts partial values such as `1abc` as `1` and silently drops other invalid values to the default, while the neighboring message numeric flags use strict parsers and throw. A typo can make the send timeout almost immediately or ignore the user's requested timeout, so `--timeout` should use the existing strict positive-integer validation path before dispatch.
  - confidence: 0.86
- **[P3] Drop the release-owned changelog entry:** `CHANGELOG.md:8`
  - body: Root policy keeps `CHANGELOG.md` release-owned for normal PRs; release-note context should live in the PR body or squash/commit message. Keeping this manual Unreleased entry creates release-generation drift for an otherwise normal bug-fix PR.
