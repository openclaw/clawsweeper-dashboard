## Summary

Keep open: current main still logs QQ Bot inbound message bodies at info level. The raw WebSocket dispatch payload is already debug-only, but the default message-body log path remains present on current main and in the latest release.

## What This Changes

Request to reduce bundled QQ Bot log verbosity and stop logging full inbound chat payloads at info level by default.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and payloads behind debug or explicit diagnostics, and add focused regression coverage for inbound and urgent-command logging.

## Reproduction Assessment

Yes. Source inspection on current main shows QQ dispatch maps ev.content into QueuedMessage.content for C2C and group messages, and handleMessage logs that value at info level.

## Solution Assessment

Yes, with owner review. A QQ Bot plugin-local logging repair is the narrowest maintainable path; a global redaction layer or new verbosity option is unnecessary unless owners choose a broader policy.
