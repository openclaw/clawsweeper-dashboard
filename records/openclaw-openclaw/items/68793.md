## Summary

Keep open: current main still logs QQ Bot inbound message bodies at info level, and the latest shipped release shows the same default message-content path. The raw WebSocket dispatch payload is debug-only now, so part of the report is already better scoped, but the core privacy/log-verbosity concern remains.

## What This Changes

The issue asks QQ Bot to lower routine log verbosity and stop writing full inbound chat payloads or message bodies at info level by default.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and payload dumps behind debug or explicit diagnostics, and add focused regression coverage for inbound and urgent-command logging.

## Reproduction Assessment

Yes. Source inspection on current main shows C2C and group dispatch copy ev.content into QueuedMessage.content, and handleMessage logs that value at info level.

## Solution Assessment

Yes, with owner review. A QQ Bot plugin-local logging repair is the narrowest maintainable fix; a global redaction layer or new verbosity option is unnecessary unless owners choose a broader policy.
