## Summary

Keep open. Current main still leaves the privacy-relevant part unresolved: QQ Bot dispatch copies raw message content into the queued event and the normal handler logs that content at info level; the raw WebSocket dispatch JSON is already debug-only.

## What This Changes

The issue requests quieter QQ Bot defaults and metadata-only info logging instead of full inbound chat payloads or message bodies.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and dispatch JSON behind debug or explicit diagnostics, and add focused regression coverage for info-level inbound logging.

## Reproduction Assessment

Yes. Source inspection on current main is enough: QQ dispatch maps ev.content into QueuedMessage.content, and handleMessage logs that value at info level.

## Solution Assessment

Yes. The narrow maintainable direction is a QQ Bot plugin-local logging repair; a global redaction change or new verbosity knob is unnecessary unless maintainers explicitly choose that policy.
