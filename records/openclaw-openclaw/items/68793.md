## Summary

Keep this open. Current main still logs the full inbound QQ Bot message body at info level in the normal handler, so the raw-dispatch half is quieter but the default log-content privacy request remains unresolved.

## What This Changes

The issue asks QQ Bot to reduce default info-level log noise and avoid logging full inbound chat or WebSocket payloads by default.

## Best Possible Solution

Fix this inside the QQ Bot plugin so default info logs are metadata-only, while full message bodies and raw payloads stay behind debug or explicit diagnostics, with focused tests covering the normal info logger.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: QQ dispatch events copy raw `content` into `QueuedMessage.content`, and the QQ Bot handler logs that value at info on current main.

## Solution Assessment

Yes. A narrow QQ Bot plugin-local logging repair is the maintainable fix; global redaction changes or a broad verbosity knob are unnecessary for the reported default log leak.
