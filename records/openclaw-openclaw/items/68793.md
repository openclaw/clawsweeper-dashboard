## What This Changes

The issue asks to reduce QQBot default log verbosity and avoid writing full inbound chat payloads to info-level logs.

## Best Possible Solution

Keep QQBot info logs metadata-only by default, with full message bodies available only through explicit debug/verbose behavior and focused regression coverage.

## Solution Assessment

Unclear until maintainers choose the policy. The narrow fix boundary is clear, but the best permanent shape is metadata-only info logging or debug-only full content, not a new verbosity knob unless maintainers explicitly want more config surface.
