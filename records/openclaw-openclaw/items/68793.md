## Summary

Keep open. Current main still logs the full inbound QQ Bot message body at info level; the raw WebSocket dispatch payload is debug-only now, but the default info-log privacy/noise request remains unresolved.

## What This Changes

The issue asks QQ Bot to demote routine logs and stop emitting full inbound chat payloads at info by default.

## Best Possible Solution

Fix this in the QQ Bot plugin by making default info logs metadata-only, moving full chat bodies and raw payloads to debug or explicit diagnostics, and adding focused regression coverage for info-level inbound logging.

## Reproduction Assessment

Yes. Source inspection on current main is enough: QQ dispatch events copy raw `ev.content` into `QueuedMessage.content`, and `handleMessage` logs that value at info level.

## Solution Assessment

Yes. A narrow QQ Bot plugin-local logging repair is the best path; a global redaction change or new channel verbosity knob is unnecessary unless maintainers explicitly choose that policy.
