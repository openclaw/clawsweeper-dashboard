## Summary

Keep open: current main still logs inbound QQ Bot message bodies at info level. The raw WebSocket dispatch payload is already debug-only, but the privacy-relevant chat-text exposure remains in the default log path.

## What This Changes

The issue asks QQ Bot to reduce default info-level log noise and avoid logging full inbound chat payloads or message bodies by default.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and dispatch payloads behind debug or explicit diagnostics, and add focused regression coverage for inbound logging.

## Reproduction Assessment

Yes. Source inspection on current main shows QQ dispatch stores raw ev.content in QueuedMessage.content for C2C and group messages, and handleMessage logs that field at info level.

## Solution Assessment

Yes. A QQ Bot plugin-local logging repair is the narrowest maintainable path; a global redaction layer or new verbosity option is unnecessary unless owners choose that broader policy.
