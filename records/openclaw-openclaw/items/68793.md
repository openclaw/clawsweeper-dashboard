## Summary

Current main and the latest release still log QQBot inbound message bodies at info level, so the issue remains necessary. The related merged QQBot logging PR improved debug-log sanitization, but it did not remove or redact the default info-level inbound content log.

## What This Changes

Request to reduce QQBot default logging noise and stop full inbound chat payloads from appearing in info-level logs.

## Best Possible Solution

Change QQBot inbound logging so default info logs contain only safe metadata or move the line to debug, with focused regression coverage for message-body redaction.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main still formats `event.content` into an info-level QQBot inbound log. I did not perform a live gateway repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the logging policy: the narrow fix is clear, but metadata-at-info versus full demotion to debug is a product/security decision. A new verbosity knob is not required unless maintainers explicitly want that surface.
