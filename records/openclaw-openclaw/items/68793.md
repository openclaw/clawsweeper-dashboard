## Summary

Keep open: current main still logs QQ Bot inbound message content at info level. The raw WebSocket dispatch payload is already debug-only, so the remaining confirmed problem is default info-level exposure of chat text.

## What This Changes

Reduce QQ Bot default log verbosity and avoid logging full inbound chat or dispatch payload content at info level.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and payload dumps behind debug or explicit diagnostics, and add focused regression coverage for inbound and urgent-command logging.

## Reproduction Assessment

Yes. Source inspection on current main shows QQ Bot dispatch copies inbound event content into `QueuedMessage.content`, and `handleMessage` logs that value at info level.

## Solution Assessment

Yes. A QQ Bot plugin-local logging repair is the narrowest maintainable path; a global redaction layer or new channel-specific verbosity knob is not necessary for the confirmed raw-body info log.
