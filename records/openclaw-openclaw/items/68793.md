## Summary

Keep open: current main still logs inbound QQ Bot message text at info level, while the raw WebSocket dispatch payload is already debug-only, so the remaining issue is the default exposure of user-visible chat content in normal logs.

## What This Changes

The issue requests quieter QQ Bot default logs and metadata-only info-level logging for inbound chat payloads.

## Best Possible Solution

Make QQ Bot info logs metadata-only by default, keep raw message text and full payloads behind debug or explicit diagnostics, and add focused regression coverage for inbound and urgent-command logging.

## Reproduction Assessment

Yes. Source inspection on current main shows QQ Bot dispatch copies ev.content into QueuedMessage.content for C2C and group messages, and handleMessage logs that value at info level.

## Solution Assessment

Yes, with owner/security review. A QQ Bot plugin-local logging repair is the narrowest maintainable direction; a global redaction layer or new verbosity option is unnecessary unless owners choose a broader policy.
