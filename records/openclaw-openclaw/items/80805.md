- **[P1] Recompute lane size after clearing interrupts:** `src/auto-reply/reply/get-reply-run.ts:901-905`
  - body: When interrupt mode clears the session lane, this resolver still uses the laneSize captured before clearCommandLane().   - body: In sourceReplyDeliveryMode: "message_tool_only", dispatch suppresses automatic source replies unless the payload metadata sets deliverDespiteSourceReplySuppression. These new queued/steered acknowledgements are returned before a message-tool run and are plain payloads, so Discord/Telegram can still look silent in that mode; mark them with the existing suppression-bypass metadata while preserving sendPolicy denial.
  - confidence: 0.88

  - body: After this patch, raw gateway events only update lastEventAt, but accepted interactions and reactions do not add a replacement lastInboundAt update after their filters. Slash-command or reaction-driven sessions can then look idle even while human input is being processed; update accepted interaction/reaction paths after authorization rather than restoring raw-event inbound activity.
  - confidence: 0.83
