Keep this PR open, but the current patch still has three correctness blockers around interrupt queue state, visible acknowledgements under message-tool-only delivery, and Discord accepted-action activity accounting.

This PR adds busy-run channel status fields, active-turn acknowledgement replies for Discord/Telegram, Discord gateway event status filtering, and focused regression tests.

## Best Possible Solution

Land the responsiveness fix after the branch preserves acknowledgements under source-reply suppression, recomputes queue state after interrupts, and records accepted Discord action activity without counting raw gateway events as human input.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recompute lane size after clearing interrupts:** `src/auto-reply/reply/get-reply-run.ts:901-905`
  - body: When interrupt mode clears the session lane, this resolver still uses the laneSize captured before clearCommandLane(). The new laneSize-based busy check can report the cleared lane as active and return the shutdown/queued path instead of starting the interrupted request; re-read the queue size after clearing or inside the busy-state resolver.
  - confidence: 0.91
- **[P2] Mark busy acknowledgements deliverable:** `src/auto-reply/reply/agent-runner.ts:121-126`
  - body: In sourceReplyDeliveryMode: "message_tool_only", dispatch suppresses automatic source replies unless reply payload metadata sets deliverDespiteSourceReplySuppression. These new queued/steered acknowledgements are returned before a message-tool run and are plain payloads, so Discord/Telegram can still look silent in that mode; mark them with the existing suppression-bypass metadata while preserving sendPolicy denial.
  - confidence: 0.88
- **[P2] Record inbound activity for accepted Discord actions:** `extensions/discord/src/monitor/provider.ts:119`
  - body: After this patch, raw gateway events only update lastEventAt, but accepted interactions and reactions do not add a replacement lastInboundAt update after their filters. Slash-command or reaction-driven sessions can then look idle even while human input is being processed; update accepted interaction/reaction paths after authorization rather than restoring raw-event inbound activity.
  - confidence: 0.83
