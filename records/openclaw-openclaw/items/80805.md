## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recompute lane size after clearing interrupts:** `src/auto-reply/reply/get-reply-run.ts:901-905`
  - body: When interrupt mode clears the session lane, this resolver still uses the `laneSize` captured before `clearCommandLane()`. With the new `laneSize > 0` busy check, an interrupted request can still be treated as active and return the shutdown/queued path instead of starting; re-read the queue size after clearing or inside the busy-state resolver.
  - confidence: 0.91
- **[P2] Mark busy acknowledgements deliverable:** `src/auto-reply/reply/agent-runner.ts:119-126`
  - body: In `sourceReplyDeliveryMode: "message_tool_only"`, dispatch suppresses automatic source replies unless payload metadata sets `deliverDespiteSourceReplySuppression`. These queued/steered acknowledgements are returned before a message-tool run and are plain payloads, so Discord/Telegram can still look silent in that mode; add the existing suppression-bypass metadata while preserving sendPolicy denial.
  - confidence: 0.88
- **[P2] Record inbound activity for accepted Discord actions:** `extensions/discord/src/monitor/provider.ts:119`
  - body: After this change, raw gateway events only update `lastEventAt`, but accepted interactions and reactions do not add a replacement `lastInboundAt` update after their filters. Slash-command or reaction-driven sessions can therefore look idle even while human input is being processed; update accepted interaction/reaction paths after authorization instead of restoring raw-event inbound activity.
  - confidence: 0.83
