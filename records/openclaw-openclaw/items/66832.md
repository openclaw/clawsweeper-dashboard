## Summary

Keep open. Current main partially improves default redaction with a standalone JWT pattern, but the `logs.tail` path still misses raw Basic auth headers and the reported `X-OpenClaw-Token` header shape, and https://github.com/openclaw/openclaw/pull/67041 is still the open closing PR for this issue.

## What This Changes

The issue asks for `logs.tail` to redact Basic auth, JWT/assertion, and custom token-header credentials before returning log lines to `operator.read` clients.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/67041 or an equivalent narrow patch that extends `src/logging/redact.ts` defaults and regression tests for raw Basic auth, `X-OpenClaw-Token`, `x-pomerium-jwt-assertion`, and current JWT behavior.

## Reproduction Assessment

Yes. The current-main source path is clear and a direct regex probe shows the default redactor still misses the raw Basic auth and `X-OpenClaw-Token` examples before `logs.tail` returns redacted lines.

## Solution Assessment

Yes. A bounded update to the shared default redaction patterns with focused redaction and log-tail regression tests is the narrow maintainable fix; the issue should remain open until the linked PR or equivalent fix merges.
