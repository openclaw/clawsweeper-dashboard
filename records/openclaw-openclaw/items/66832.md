## Summary

Keep open. Current main still routes `logs.tail` through the default logging redactor, those defaults still do not cover the reported Basic auth, generic JWT, Pomerium assertion, or `X-OpenClaw-Token` header cases, and open PR #67041 already targets this issue with closing syntax.

## What This Changes

Extend `logs.tail` redaction so Basic auth, generic JWTs, `x-pomerium-jwt-assertion`, `X-OpenClaw-Token`, and similar credential headers are masked before log lines are returned.

## Best Possible Solution

Review and land PR #67041, or an equivalent narrow patch that extends `src/logging/redact.ts` defaults and adds regression tests for each reported credential format.

## Reproduction Assessment

Yes. Put any reported sample value into the configured gateway log file and call `logs.tail`; current main returns lines through `redactSensitiveLines`, and the default pattern list does not include those formats.

## Solution Assessment

Yes. A bounded redaction-pattern change with colocated tests is the maintainable fix boundary; the issue should remain open until the linked PR or an equivalent fix merges.
