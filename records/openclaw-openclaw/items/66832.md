## Summary

Keep open. Current main still misses reported raw Basic auth and `X-OpenClaw-Token` header-style log samples in the shared redactor used by `logs.tail`, and the open closing PR at https://github.com/openclaw/openclaw/pull/67041 remains the implementation candidate.

## What This Changes

The issue asks for `logs.tail` to redact Basic auth, custom gateway/security headers, Pomerium JWT assertions, and generic JWTs before returning lines to `operator.read` clients.

## Best Possible Solution

Review and amend https://github.com/openclaw/openclaw/pull/67041, or land an equivalent narrow patch, so shared defaults and focused tests cover raw Basic auth, `X-OpenClaw-Token`, Pomerium JWT assertions, and standalone JWTs through the `logs.tail` path.

Yes, at source level. `logs.tail` returns `readConfiguredLogTail` output after shared redaction, and the focused current-pattern probe shows the reported Basic auth and custom-header samples are still missed.

## Solution Assessment

Yes, the shared default redactor is the right layer. The issue should stay open until the linked PR or an equivalent patch adds complete patterns plus focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` coverage.
