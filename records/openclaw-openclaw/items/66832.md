## Summary

Keep open. Current main still routes `logs.tail` output through the default logging redactor, those defaults still do not cover the reported Basic auth, generic JWT, Pomerium assertion, or `X-OpenClaw-Token` cases, and open PR #67041 is already the linked fix candidate.

## What This Changes

Expand `logs.tail` redaction so read-scoped clients do not receive Basic auth, generic JWT, Pomerium assertion, or `X-OpenClaw-Token` values from gateway logs.

## Best Possible Solution

Land PR #67041 or an equivalent narrow redaction-default patch with regression tests for each reported credential format.

## Reproduction Assessment

Yes. Put one of the sample header/JWT/Basic-auth values in the configured gateway log file and call `logs.tail`; current main passes returned lines through `redactSensitiveLines`, and the default pattern list lacks those formats.

## Solution Assessment

Yes. A bounded `src/logging/redact.ts` default-pattern change with colocated tests is the narrowest maintainable fix boundary; PR #67041 is the current implementation candidate to review.
