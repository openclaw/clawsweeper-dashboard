## Summary

Keep open. Current main still routes `logs.tail` through the shared logging redactor, the default pattern list still lacks the reported Basic auth, generic JWT, Pomerium assertion, and hyphenated OpenClaw token header cases, and open PR #67041 is already paired to this issue with closing syntax.

## What This Changes

Redact generic JWTs, Basic auth headers, `X-OpenClaw-Token`, and `x-pomerium-jwt-assertion` values from `logs.tail` responses returned to `operator.read` clients.

## Best Possible Solution

Review and land PR #67041, or an equivalent narrow patch that extends `src/logging/redact.ts` defaults and adds regression tests for each reported credential format.

## Reproduction Assessment

Yes, from source inspection. Put any reported sample value into the configured gateway log file and call `logs.tail`; current main returns lines through `redactSensitiveLines`, and the default pattern list does not include those formats.

## Solution Assessment

Yes. A bounded default redaction-pattern change with colocated tests is the narrow maintainable fix; this issue should remain open until #67041 or an equivalent fix merges.
