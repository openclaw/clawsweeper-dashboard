## Summary

Keep open. Current main still misses the reported credential formats in the default `logs.tail` redaction path, and an open contributor PR already targets the same issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `logs.tail` redaction to cover Basic auth, generic JWTs, Pomerium assertions, and hyphenated token header formats before log lines reach read-scoped clients.

## Best Possible Solution

Land PR #67041 after maintainer review, or apply an equivalent bounded patch that extends the default logging redaction patterns and adds focused regression tests for each reported credential format.

## Reproduction Assessment

Yes. Put any reported sample into the configured gateway log file and call `logs.tail`; current main redacts only through `redactSensitiveLines`, and the default patterns do not match those samples.

## Solution Assessment

Yes. A narrow change in `src/logging/redact.ts` with colocated tests in `src/logging/redact.test.ts` is the maintainable fix boundary; the already-linked PR should be reviewed before opening another repair branch.
