## Summary

Keep open. Current main only partially addresses the report with standalone JWT redaction; the shared redactor used by `logs.tail` still misses raw Basic auth and the reported `X-OpenClaw-Token` header shape, and https://github.com/openclaw/openclaw/pull/67041 remains the open closing implementation candidate.

## What This Changes

The issue requests broader `logs.tail` credential redaction so Basic auth, `X-OpenClaw-Token`, `x-pomerium-jwt-assertion`, and generic JWT values are masked before read-scoped clients receive log lines.

## Best Possible Solution

Review and amend https://github.com/openclaw/openclaw/pull/67041, or land an equivalent narrow patch, so the shared defaults and focused tests cover raw Basic auth, `X-OpenClaw-Token`, Pomerium JWT assertions, and standalone JWTs through the `logs.tail` path.

## Reproduction Assessment

Yes, at source level. `readConfiguredLogTail` applies the shared defaults to returned lines, and a focused probe against current patterns shows raw Basic auth and `X-OpenClaw-Token` are still missed while generic JWT coverage is only partially present on main.

## Solution Assessment

Yes. Extending the shared default redaction patterns with focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` regression coverage is the narrow maintainable fix; the issue should remain open until the linked PR or equivalent patch merges.
