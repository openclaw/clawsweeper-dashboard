## Summary

Keep open: current main still lacks Telegram subagent lifecycle hook registration and automatic thread-bound native subagent spawn exposure. The active implementation candidate is https://github.com/openclaw/openclaw/pull/82023, which now supplies live/log proof but remains unmerged and still needs maintainer acceptance of the capability seam and normal landing gates.

## What This Changes

The issue requests Telegram support for `sessions_spawn` with `mode="session"` and `thread=true`, binding spawned subagent sessions to Telegram conversations or forum topics.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82023 as the canonical Telegram implementation, then close this tracker after the fix is on main.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram does not register one or advertise automatic spawn support. I did not run a live Telegram forum reproduction from current main.

## Solution Assessment

Yes, pending maintainer API acceptance: the active PR's direction is narrower than changing Telegram's default placement because it adds explicit per-kind spawn capability plus Telegram hooks while preserving top-level Telegram and ACP behavior.
