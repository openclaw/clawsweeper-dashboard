## Summary

Keep open: current main and v2026.6.6 still do not expose Telegram as a supported `sessions_spawn(thread:true, mode:"session")` channel, and the open implementation candidate needs maintainer direction on the core binding seam before this can close.

## What This Changes

The issue requests Telegram support for thread-bound persistent subagent sessions created with `sessions_spawn(thread:true, mode:"session")`.

## Best Possible Solution

Expose Telegram native subagent child-topic spawning through the current core `SessionBindingService` path with an explicit capability contract, refresh focused tests and live Telegram proof, then close this after the fix lands.

## Reproduction Assessment

Yes, source-level: current main gates thread/session spawn availability on child-placement channels, while Telegram keeps its top-level placement as `current` even though its adapter can create child forum topics.

## Solution Assessment

No for the issue's original hook-based sketch; the maintainable fix is to expose Telegram's explicit child-topic spawn capability through the current core session-binding path rather than adding new deprecated `subagent_spawning` routing.
