## Summary

Keep open: current main still does not expose Telegram thread-bound subagent session spawning, and the direct implementation candidate remains open and conflicting rather than merged.

## What This Changes

The issue requests Telegram support for `sessions_spawn(thread=true, mode="session")` so spawned subagent sessions bind to Telegram conversations or forum topics.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned lifecycle hook and capability design that preserves normal current-topic binding while enabling explicit child-topic subagent session spawns.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, while Telegram still does not register one or advertise automatic child-topic subagent spawns. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance: the requested hook family is the right implementation area, and the active PR's explicit capability seam is safer than changing Telegram's normal top-level placement to `child`. This issue should close only after that PR or a replacement lands on main.
