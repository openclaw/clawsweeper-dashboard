## Label Justifications

- `P2`: This is a concrete Telegram channel capability gap with an active implementation path, but it is not a core runtime outage.
- `impact:message-loss`: The missing thread-bound spawn path can leave expected subagent follow-up or completion messages unavailable in the intended Telegram topic.

## Summary

Keep open: current main still lacks the Telegram subagent lifecycle hook and capability wiring needed for thread-bound native subagent sessions, and the active implementation candidate at https://github.com/openclaw/openclaw/pull/82023 is open rather than merged.

## What This Changes

Request support for Telegram `sessions_spawn(thread=true, mode="session")` to create persistent subagent sessions bound to Telegram conversations or forum topics.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned hook and capability implementation that preserves normal current-topic binding while enabling explicit subagent child-topic spawns.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram does not register that hook or expose automatic child-topic subagent spawn support. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance: the active PR's explicit capability seam is narrower than changing Telegram's normal top-level placement to `child`. This issue should close only after that implementation or a replacement lands on main.
