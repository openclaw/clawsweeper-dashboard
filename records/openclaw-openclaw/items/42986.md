Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram does not register that hook or expose automatic thread-bound subagent spawn support. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance: the linked PR's explicit per-kind capability seam is a narrower fit than changing Telegram's normal top-level placement or adding Telegram-only config keys. The issue should close only after that implementation or a replacement lands on main.
