## Summary

Keep open: current main and the latest release still do not expose Telegram thread-bound subagent session spawning, and the direct implementation candidate remains open and conflicting rather than merged.

## What This Changes

The issue requests Telegram support for persistent `sessions_spawn(thread=true, mode="session")` subagent sessions bound to a Telegram conversation or forum topic.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned hook and capability design that keeps normal Telegram top-level binding as `current` while enabling explicit child forum-topic subagent session spawns.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, while Telegram still does not register that hook or advertise automatic child-topic subagent spawns. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes directionally: Telegram-owned lifecycle hooks plus an explicit spawn capability are the maintainable area to solve this. The issue should close only after https://github.com/openclaw/openclaw/pull/82023 or a replacement lands on main.
