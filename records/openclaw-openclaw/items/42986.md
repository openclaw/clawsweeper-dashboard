## Label Justifications

- `P2`: The missing capability blocks a persistent delegated-session workflow on a major channel, but its blast radius is bounded to thread-bound Telegram spawning.
- `impact:session-state`: The issue concerns keeping a persistent child session correctly associated with its Telegram forum topic across its lifecycle.
- `impact:message-loss`: Without a bound delivery origin and lifecycle routing, child replies or completion announcements can be hidden or delivered outside the intended topic.

## Summary

Current `main` still cannot advertise or execute automatic Telegram child-topic spawning because that capability is inferred from child-default placement, while Telegram intentionally keeps normal bindings in the current conversation. The issue should remain open and paired with the active implementation PR, which has strong live Telegram proof but is currently conflicting and still needs maintainer agreement on the generic channel capability and scope.

## What This Changes

Enable persistent subagent sessions spawned from Telegram to create, bind, route through, and clean up a dedicated forum topic.

## Best Possible Solution

Refresh and narrow the active PR around an explicit per-kind automatic-spawn capability plus Telegram-owned topic creation, delivery-origin routing, completion routing, and cleanup, while preserving Telegram's existing current-context behavior for ordinary bindings and validating sibling channel invariants.

## Reproduction Assessment

Yes at the source-contract level. Current `main` gates automatic thread-bound spawning on child-default placement, and Telegram has no current-main lifecycle hook path that supplies the requested child-topic behavior; no live current-main run was performed during this read-only review.

## Solution Assessment

The linked PR's distinct per-kind automatic-spawn capability is a better direction than changing Telegram's ordinary placement semantics, but the final API and the PR's expanded scope require maintainer judgment and a conflict-free refresh.
