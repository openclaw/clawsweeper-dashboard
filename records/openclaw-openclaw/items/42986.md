## Summary

Keep open: current main and v2026.5.22 still lack Telegram subagent lifecycle hook registration and still document native thread-bound subagent sessions as Discord-only, while the direct Telegram implementation candidate remains open and conflicting.

## What This Changes

The issue requests Telegram support for persistent thread-bound native subagent sessions via `sessions_spawn` with `mode="session"` and `thread=true`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned hook and explicit spawn-capability design that preserves current Telegram binding behavior while enabling child forum-topic native subagent sessions.

## Reproduction Assessment

Yes, at source level: current main still requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram still does not register that hook or advertise automatic child-topic subagent spawns. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes directionally: Telegram-owned lifecycle hooks plus an explicit spawn capability are the maintainable direction. The issue should close only after https://github.com/openclaw/openclaw/pull/82023 or a replacement lands on main.
