## Label Justifications

- `P2`: This is a normal-priority Telegram agent/channel workflow improvement with real session-routing value but bounded blast radius.
- `impact:session-state`: The requested behavior binds persistent spawned subagent sessions to Telegram conversations or topics, so unresolved routing can mis-associate session state.
- `impact:message-loss`: When Telegram child session binding is unavailable or hidden, subagent follow-ups and completion announcements can become invisible or route away from the intended topic.

## Summary

Keep open. Current main has the generic session-binding adapter and Telegram forum-topic primitive, but the model-visible explicit Telegram subagent child-topic spawn path still needs a maintainer-reviewed implementation; https://github.com/openclaw/openclaw/pull/82023 is the active candidate and is still waiting on author/maintainer handling.

## What This Changes

The issue requests Telegram support for persistent thread-bound subagent sessions with `sessions_spawn(mode="session", thread=true)`, ideally binding spawned subagents to Telegram conversations or forum topics.

## Best Possible Solution

Land a rebased or narrowed implementation that advertises explicit Telegram subagent child-topic spawn capability while using the core session-binding adapter path, preserving Telegram's normal current-topic placement and avoiding new deprecated `subagent_spawning` routing.

## Reproduction Assessment

Yes, source-level. Current main keeps Telegram as `defaultTopLevelPlacement: "current"` while `sessions_spawn` exposes thread/session parameters only for automatic child-placement channels, so the requested Telegram child-topic subagent spawn path remains unavailable to the model-visible tool contract.

## Solution Assessment

No, not as originally proposed. Adding Telegram `subagent_spawning` hooks is now the wrong layer because current main routes new thread-bound subagent binding through core session-binding adapters; the best fix is an explicit reviewed capability/placement seam for Telegram subagent child-topic spawns.
