## Summary

Keep this open: current `main` still does not advertise Telegram as a supported model-facing `sessions_spawn(thread:true, mode:"session")` channel, even though Telegram now has binding primitives, and the open implementation candidate still needs maintainer direction.

## What This Changes

The issue requests Telegram support for persistent thread-bound `sessions_spawn(thread:true, mode:"session")` subagent sessions.

## Best Possible Solution

Expose Telegram native subagent child-topic spawning through the current core `SessionBindingService` path with an explicit per-kind capability contract, then refresh focused tests and live Telegram proof before closing this issue.

## Reproduction Assessment

Yes, source-level: current `main` gates model-facing thread/session spawn support on child-placement channels, while Telegram keeps default placement as `current`. The latest release has the same source shape.

## Solution Assessment

No for the issue's original hook-based sketch. The safer current shape is a Telegram-specific automatic spawn capability through core session-binding adapters, not new reliance on deprecated `subagent_spawning` routing.
