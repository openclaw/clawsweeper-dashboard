## Summary

Keep open because current main still lacks Telegram native subagent lifecycle hook registration for thread-bound persistent subagent sessions. Telegram has conversation/topic binding infrastructure and ACP topic-spawn support, but native `sessions_spawn({ mode: "session", thread: true })` still depends on channel `subagent_spawning` hooks that Telegram does not register; PR #42707 remains the focused open implementation candidate.

## What This Changes

Request support for Telegram thread-bound persistent native subagent sessions via `sessions_spawn({ mode: "session", thread: true })`, analogous to Discord.

## Best Possible Solution

Review and land PR #42707 or a narrow equivalent in the Telegram plugin boundary that registers native subagent lifecycle hooks and reuses the existing session-binding adapter and `threadBindings.spawnSessions` policy.

## Reproduction Assessment

Yes. Source inspection shows a Telegram-origin native `sessions_spawn({ mode: "session", thread: true })` reaches the core hook gate, while the Telegram bundled entry never registers the native lifecycle hooks needed to return `threadBindingReady`.

## Solution Assessment

Yes. The maintainable path is Telegram plugin-boundary hook wiring that reuses existing Telegram session-binding infrastructure; adding a core Telegram special case would cut across the channel/plugin boundary.
