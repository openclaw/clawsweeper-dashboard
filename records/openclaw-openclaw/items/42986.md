## Summary

Keep open: current main still lacks the Telegram subagent hook/capability path needed for thread-bound native subagent sessions, and the canonical implementation candidate remains open rather than merged.

## What This Changes

Request support for Telegram `sessions_spawn(thread=true, mode="session")` so persistent spawned subagent sessions can bind to a Telegram conversation or forum topic.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned hook/capability implementation that preserves normal current-topic binding, proves live Telegram behavior, and then close this tracker.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram does not register that hook or expose automatic child-topic subagent spawn support. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance: the active PR's explicit capability seam is a narrower fit than changing Telegram's normal top-level placement to `child`. The issue should close only after that implementation or a replacement lands on main.
