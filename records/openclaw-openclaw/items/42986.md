## Summary

Keep open: current main still lacks native Telegram thread-bound subagent session support. Telegram has session-binding infrastructure, but native `sessions_spawn(..., thread=true, mode="session")` still depends on a registered `subagent_spawning` hook, and Telegram does not register one; the focused open implementation candidate is still under review.

## What This Changes

The issue requests Telegram support for `sessions_spawn` with `mode="session"` and `thread=true`, binding spawned subagent sessions to a Telegram conversation or topic.

## Best Possible Solution

Review, refresh, or replace the open Telegram PR with a plugin-boundary implementation that registers Telegram subagent lifecycle hooks, reuses existing Telegram target parsing and binding managers, covers delivery and cleanup, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes at source level. Current main still gates native thread-bound subagent spawns on `subagent_spawning`, Telegram does not register that hook, and the tool capability path does not advertise Telegram as an automatic native thread-spawn channel.

## Solution Assessment

Yes in direction, but not as a fresh duplicate issue or a direct core import patch. The maintainable fix should use the current channel entry/plugin SDK seams, existing Telegram parsing, and the open PR or a narrow replacement with real Telegram proof.
