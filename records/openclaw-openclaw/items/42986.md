## Summary

Keep open: current main and the latest tagged release still lack Telegram subagent lifecycle hook registration/capability exposure, while the direct implementation candidate is open and conflicting, so this remains active maintainer work rather than cleanup.

## What This Changes

The issue requests Telegram support for `sessions_spawn(thread:true, mode:"session")` so spawned subagents can stay bound to a Telegram conversation or forum topic.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/82023 with Telegram-owned subagent lifecycle hooks and an explicit capability seam, preserving Telegram's current-conversation default while enabling explicit child-topic subagent spawns.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, while Telegram still registers no such hook and is not advertised as automatic child-thread capable. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer approval: the right implementation area is Telegram-owned lifecycle hooks plus an explicit capability seam. The active PR is the best current path, but it needs maintainer review and conflict resolution before the issue should close.
