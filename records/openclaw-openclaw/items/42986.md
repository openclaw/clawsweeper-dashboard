## Summary

Keep this open because current main still lacks Telegram native subagent lifecycle hook wiring for persistent `sessions_spawn(thread: true, mode: "session")` sessions. The request remains valid, and open PR #42707 is already the focused implementation candidate.

## What This Changes

Request Telegram plugin support for `sessions_spawn` with `mode="session"` and `thread=true`, binding persistent subagent sessions to Telegram conversations or forum topics.

## Best Possible Solution

Land PR #42707 or a narrow equivalent in the Telegram plugin boundary that registers native subagent lifecycle hooks and reuses the existing Telegram session-binding adapter and `threadBindings.spawnSessions` policy.

## Reproduction Assessment

Yes. Source inspection shows Telegram native `sessions_spawn({ mode: "session", thread: true })` still reaches the core `subagent_spawning` hook gate, while Telegram registers no such lifecycle hook on current main.

## Solution Assessment

Yes. The maintainable path is Telegram plugin-boundary hook wiring that reuses the existing binding manager and policy controls; adding a core Telegram special case would cut across the plugin boundary.
