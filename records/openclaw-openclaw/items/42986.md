- `P2`: This is a normal-priority Telegram agent/channel workflow improvement with real routing value but bounded blast radius and an active PR candidate.
- `impact:session-state`: The requested behavior binds persistent spawned subagent sessions to Telegram topics, so unresolved routing can mis-associate or hide session state.

## Summary

Keep open. Current main and the latest release still leave the model-visible Telegram subagent child-topic spawn path incomplete, and the active implementation candidate is open but not merged.

## What This Changes

Request Telegram support for persistent `sessions_spawn(thread:true, mode:"session")` subagent sessions bound to Telegram conversations or forum topics.

## Best Possible Solution

Land or replace the active PR with a narrow capability seam that lets Telegram advertise explicit subagent child-topic spawn support while preserving normal current-topic placement and using the core session-binding adapter path.

## Reproduction Assessment

Yes, source-level. Current main keeps Telegram as current-placement while `sessions_spawn` only exposes thread/session parameters for automatic child-placement channels, so the requested Telegram child-topic subagent spawn path remains hidden.

## Solution Assessment

No, not as originally proposed. Adding primary Telegram `subagent_spawning` routing is no longer the best layer; the maintainable fix is an explicit Telegram spawn capability that uses the existing core session-binding adapter path.
