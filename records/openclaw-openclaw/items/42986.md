## Summary

Keep this open: current main still has the core `subagent_spawning` gate for thread-bound native subagent sessions, while Telegram has binding infrastructure but no Telegram lifecycle hook registration to drive it. The duplicate reports already point here as the canonical tracker, and https://github.com/openclaw/openclaw/pull/42707 remains the focused implementation candidate.

## What This Changes

Request Telegram support for persistent native subagent sessions using `sessions_spawn(..., mode:"session", thread:true)`.

## Best Possible Solution

Add Telegram bundled-channel `registerFull` lifecycle hooks that reuse the existing conversation-runtime binding adapter and Telegram target parsing, cover spawn/delivery/end cleanup with focused tests, and collect redacted live Telegram proof before closing this tracker.

## Reproduction Assessment

Yes, at source level. `sessions_spawn(..., mode:"session", thread:true)` still depends on a registered `subagent_spawning` hook, and Telegram currently has no such hook registration even though its binding adapter exists.

## Solution Assessment

Yes for the requested direction, but the implementation should live at the Telegram bundled-channel hook boundary and reuse existing binding/target parsing seams. The current open PR is the right candidate to review, revise, or replace rather than opening another parallel tracker.
