- `P2`: This is a concrete Telegram channel capability gap with a clear implementation path, but it is not a core runtime outage.
- `impact:session-state`: The requested support is about preserving spawned subagent session affinity to a Telegram conversation or forum topic.

## Summary

Keep open: current main and v2026.5.20 still lack Telegram subagent lifecycle hook registration and do not advertise Telegram as supporting automatic thread-bound native subagent session spawns. The direct implementation candidate at https://github.com/openclaw/openclaw/pull/82023 remains open and conflicting, so closing this issue would be premature.

## What This Changes

Request support for Telegram thread-bound native subagent session spawns so `sessions_spawn(..., mode="session", thread=true)` can bind child sessions to Telegram conversations or forum topics.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/82023 with a Telegram-owned hook and explicit spawn-capability design that preserves current Telegram binding behavior while enabling child forum-topic native subagent sessions.

## Reproduction Assessment

Yes, at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, and Telegram still does not register that hook or advertise automatic child-topic subagent spawns. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes directionally: Telegram-owned lifecycle hooks plus an explicit spawn capability are the maintainable area to solve this. The issue should close only after https://github.com/openclaw/openclaw/pull/82023 or a replacement lands on main.
