## What This Changes

Request to add an opt-in `xai-oauth` provider auth method so SuperGrok subscribers can use accounts.x.ai instead of `xai-api-key`/`XAI_API_KEY`.

## Best Possible Solution

Keep the documented API-key path; revisit xAI OAuth only with a public or approved xAI SuperGrok API auth contract, implemented inside the bundled xAI provider plugin if it becomes available.

## Reproduction Assessment

## Solution Assessment
