## Summary

This is a useful provider-specific subscription-auth idea, but it should stay outside OpenClaw core unless xAI publishes a sanctioned API OAuth/subscription contract; current core, docs, and plugin policy point to API-key-only xAI plus plugin distribution for optional integrations.

## What This Changes

Add an opt-in `xai-oauth` auth method so SuperGrok subscribers can use Grok through OpenClaw without provisioning an xAI API key.

## Best Possible Solution

Keep the bundled xAI provider on the documented API-key path, and route any SuperGrok SSO workaround through a separately published plugin until xAI provides official API OAuth/subscription semantics.

## Reproduction Assessment

Not applicable: this is a feature request for a new provider auth path, not a broken existing behavior. Source inspection confirms the current xAI path is API-key-only.

## Solution Assessment

No: adding an unsupported SuperGrok OAuth path to the bundled provider is not the best core solution while xAI documents separate API billing and API-key auth. A plugin is the maintainable place for experimental SSO/proxy behavior.
