## Summary

Keep open: current main still lacks the requested sender/chat fields in the public agent hook context, but this PR is not merge-ready because it leaves live hook propagation incomplete, regresses current channel-context normalization, omits public SDK docs/baseline updates, and still lacks real behavior proof.

## What This Changes

The PR adds optional sender/chat fields to PluginHookAgentContext, threads them into one harness/before_agent_reply path, and adds SenderSessionKey routing to MsgContext/session initialization.

## Best Possible Solution

Land a rebased additive SDK contract that defines the approved sender/chat fields, threads them through every relevant agent hook context while preserving buildAgentHookContextChannelFields semantics, and updates docs, API baseline, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence evidence that current main still lacks sender/chat fields in PluginHookAgentContext and live agent hook contexts while run params already carry sender metadata.

## Solution Assessment

No, not as currently patched. The additive API direction is plausible, but the patch should complete live hook propagation, preserve normalized channel context, and update the public SDK docs/API baseline before merge.
