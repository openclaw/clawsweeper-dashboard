## Summary

Keep this PR open. The proposed API direction is useful, and the system-event session guard from the review thread is addressed, but the patch is not merge-ready because the main agent prompt-hook path still builds its hook context inline without sender/chat identity, and the public SDK change still needs docs, changelog, and focused regression coverage.

## What This Changes

The PR adds optional sender/chat identity fields to `PluginHookAgentContext`, adds `SenderSessionKey` to `MsgContext`, wires sender/chat context into `before_agent_reply`, and lets sender-scoped session keys participate in session resolution.

## Best Possible Solution

Land one additive SDK seam that exposes sender/chat identity consistently on the relevant agent hook contexts, keeps sender-scoped session routing guarded for system events, and documents the stable semantics before release; leave the broader exec-environment injection work to the related open items.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence inspection path: current main lacks the requested fields, and the PR diff shows where it adds them while omitting the main embedded prompt-hook context.

## Solution Assessment

No, not as currently patched. The narrow API approach is reasonable, but the implementation should also populate the main agent hook context and add docs/changelog/tests before maintainers review the final semantics.
