## Summary

Keep open: the additive SDK direction is reasonable, but this PR is stale against current main's channel-context helper and still misses prompt-hook propagation plus public docs/changelog before it can merge.

## What This Changes

This PR adds optional sender/chat identity fields to `PluginHookAgentContext`, introduces `SenderSessionKey` on `MsgContext`, and threads sender-scoped context into before-agent-reply and session-key resolution.

## Best Possible Solution

Land one rebased additive SDK seam that normalizes channel, sender, and chat context in a shared helper, feeds it to before-agent-reply and prompt hooks, preserves system-event guards, and documents the public contract before release.

## Reproduction Assessment

Not applicable as a feature PR. There is a high-confidence inspection path: current main lacks the requested fields, and the PR adds them only to part of the agent-hook flow while missing the embedded prompt-hook context.

## Solution Assessment

No, not as currently patched. The narrow API approach is maintainable, but it should preserve current channel normalization, populate all relevant agent hook contexts, and add docs/changelog/tests before merge.
