## Summary

Keep open. The PR is useful source work and the earlier system-event session-routing concern has been addressed, but it is not ready to merge because the new public agent-hook context fields are not threaded through the main embedded agent hook contexts and the required SDK docs, changelog, and regression coverage are still missing.

## What This Changes

The PR adds optional sender/chat identity fields to plugin agent hook context, passes available sender/chat data into get-reply before_agent_reply context, and adds a guarded SenderSessionKey override in session resolution.

## Best Possible Solution

Land a completed additive SDK context seam that fills sender/chat identity on the agent hook contexts where source data already exists, documents optional field semantics, preserves the system-event session guard, adds focused regression tests, and leaves exec environment injection to the related work.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence inspection path: current main lacks the requested fields, and the PR patch adds them while leaving the main embedded agent hook contexts unpopulated despite available sender data.

## Solution Assessment

No, not as-is. Optional fields and the SenderSessionKey guard are the right compatibility direction, but the implementation should thread the fields through real production hook contexts and add public docs, changelog, and tests before merge.
