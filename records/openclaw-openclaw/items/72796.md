## Summary

Keep open: the PR targets a real SDK feature gap, but the current patch is incomplete for live agent hook contexts, would regress current channel context normalization, and still lacks real behavior proof for an external contribution.

## What This Changes

The PR adds optional sender/chat fields to PluginHookAgentContext, adds SenderSessionKey to MsgContext, and threads part of that context through hook/session setup.

## Best Possible Solution

Land a rebased additive SDK contract that defines the approved sender/chat fields, threads them through every relevant agent hook context while preserving channel normalization, updates docs/API baseline/tests, and leaves exec-env identity to the related dedicated work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence evidence that current main still lacks sender/chat fields in PluginHookAgentContext and live agent hook contexts while run params already carry sender metadata.

## Solution Assessment

No, not as currently patched. The additive API direction is plausible, but the patch should complete live hook propagation, preserve buildAgentHookContextChannelFields semantics, and update the public SDK contract docs/baseline.
