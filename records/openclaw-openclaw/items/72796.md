## Summary

Keep open: current main still lacks the requested public hook context, but this PR is incomplete against current main because it only updates before-agent-reply/harness paths, misses live prompt/turn hook contexts, and needs real behavior proof before merge.

## What This Changes

This PR adds optional sender/chat fields to plugin agent hook contexts, adds a sender-scoped session key to message context, and gives that key priority in session routing.

## Best Possible Solution

Use this PR as source material for a focused, rebased SDK contract that defines the approved sender/chat fields, threads them through before_agent_reply and agent turn hooks, preserves channel normalization, and updates docs/tests while keeping exec-env identity in the separate related work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence feature-gap evidence: current main omits sender/chat fields from PluginHookAgentContext and live hookCtx builders while run params already carry sender metadata.

## Solution Assessment

No, not as currently patched. The additive API direction is reasonable, but the patch should be rebased, complete all agent hook propagation paths, preserve channel normalization, and document the public semantics.
