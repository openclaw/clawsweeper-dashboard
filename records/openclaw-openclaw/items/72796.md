## Summary

This PR should stay open: the latest head still has a TypeScript parse error, one requested agent hook path is still not wired with the new context, public SDK docs are not updated, and the external-contributor real behavior proof gate is still unmet.

The PR adds optional sender/chat fields to `PluginHookAgentContext`, threads them through selected agent hook contexts, and adds `SenderSessionKey` support to reply session resolution.

## Best Possible Solution

Land a complete additive SDK contract that parses, forwards approved sender/chat fields through every relevant agent hook context, documents the field semantics, keeps generated contract hashes aligned, and includes focused coverage plus real runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Plugin SDK capability. Source inspection confirms current main lacks the requested sender/chat fields, while the PR head demonstrates the intended wiring and current blockers.

## Solution Assessment

No, not as currently patched. The direction is plausible, but the branch must parse, wire the early model-selection hook context, document the public SDK contract, and provide real behavior proof before it is the maintainable solution.
