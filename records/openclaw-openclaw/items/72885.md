The issue reports that `memory_search` should surface session transcript hits with `corpus: "sessions"` instead of flattening them to `corpus: "memory"`.

## Best Possible Solution

## Reproduction Assessment

No. Current main no longer reproduces the bug: the mapper now derives `corpus` from `result.source`, and a focused regression test covers a session transcript hit surfacing as `corpus: "sessions"`.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable path: preserve the existing backend `source` field at the tool output boundary and add targeted regression coverage without changing ranking, visibility filtering, or recall behavior.
