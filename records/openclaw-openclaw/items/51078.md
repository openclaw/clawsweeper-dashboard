## Summary

Keep open. Current main still has the Android chat initial-load/session-list mismatch and the Screen tab first-open gap, but this PR still needs contributor-supplied real behavior proof and its Screen-tab fallback introduces a WebView teardown/recreate cycle; the chat hunk was reworked in the right direction after obviyus requested the single-bootstrap path.

## What This Changes

The PR threads an optional Android chat session-list limit through MainViewModel, NodeRuntime, and ChatController so initial chat load can request 200 sessions, and adds a first-open Screen tab render fallback.

## Best Possible Solution
