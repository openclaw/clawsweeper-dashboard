## Summary

Keep open: current main and the latest release support gated Telegram native tool-progress drafts, but the Telegram docs do not yet warn that the native `sendMessageDraft` lane can make Android interactive control unavailable while the draft is active.

## What This Changes

Request to document the Telegram native `sendMessageDraft` tool-progress composer-lock tradeoff and recommend a non-freezing configuration for interactive steering workflows.

## Best Possible Solution

Add a prominent Telegram docs note that native `sendMessageDraft` tool-progress previews are best for passive viewing, may lock the client composer on Telegram Android, and should stay disabled for workflows that need `/steer`, `/stop`, corrections, or mid-run follow-ups.

## Reproduction Assessment

No, not with high confidence from this review alone. Source inspection confirms OpenClaw can keep an active `sendMessageDraft` progress lane in DMs, but the Android composer lock itself was not live-reproduced or backed by attached redacted proof here.

## Solution Assessment

Yes for a docs-first fix: documenting the passive-viewing tradeoff and the `nativeToolProgress: false` interactive workaround is the narrow maintainable path. New presets, option renames, or doctor warnings should be separate maintainer product choices if docs prove insufficient.
