## Summary

Current main and shipped v2026.5.18 intentionally make ambient room events strict message-tool-only even when group visible replies are automatic, so this should stay open as a maintainer product-policy decision rather than cleanup-closeable or a safe automatic bug-fix queue item.

## What This Changes

The issue requests that Discord room/thread room_event turns deliver final assistant text under automatic visible-reply config, or emit a visible fallback when no message tool send occurs.

## Best Possible Solution

Maintainers should decide the room_event contract: either reaffirm/support the current strict ambient-room behavior, or implement a narrow tested fallback that preserves quiet rooms while preventing accidental transcript-only final answers in the intended visible-reply path.

## Reproduction Assessment

Yes, source and tests show the behavior: a Discord room_event with groupChat.visibleReplies set to automatic still resolves to message_tool_only and suppresses normal final delivery. I did not run a live Discord gateway reproduction in this read-only review.

## Solution Assessment

No, removing every room_event hard-force is not clearly the best fix because current docs/tests intentionally keep ambient room events quiet. A safer solution needs maintainer policy input and then a narrow fallback or clarification with Discord-specific regression coverage.
