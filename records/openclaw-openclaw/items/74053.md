## Summary

Keep open. The runtime fix is the right narrow boundary, but the added regression test can false-green because its seeded `lastTo` and wildcard allowlist let current main mirror the remembered recipient into `ctx.To` while still satisfying `not.toBe("heartbeat")`.

## What This Changes

The PR sets heartbeat `ctx.To` from `delivery.to`, adds a target-none regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the runtime context change with the new target-none regression tightened to require `capturedTo` to be undefined, while leaving the distinct #73900 inheritance issue tracked separately.

## Reproduction Assessment

Yes for the field-level bug: current main sets `ctx.To` from `sender`, while target-none delivery has no `delivery.to`. The broader persisted `lastTo` corruption path is less direct on current main because system-event session init now preserves `baseEntry.lastTo`.

## Solution Assessment

Yes for the runtime fix. Deriving `ctx.To` from `delivery.to` preserves the synthetic `From` identity without treating it as a recipient; the safer test assertion is `expect(capturedTo).toBeUndefined()`.
