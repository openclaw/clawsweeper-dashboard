## Summary

Keep this PR open. Current main still builds heartbeat reply context with `ctx.To` copied from `sender`, and the target-none path can still produce a synthetic `heartbeat` sender while no delivery recipient exists. The runtime change is narrow and appropriate; the only concrete review issue is that the new regression test should assert the exact target-none value instead of only excluding the sentinel.

## What This Changes

The PR changes heartbeat runtime context construction so `ctx.To` comes from `delivery.to`, adds a target-none regression test, and adds an Unreleased changelog fix entry.

## Best Possible Solution

Land the heartbeat context correction, tighten the new target-none regression assertion to require `ctx.To` to be undefined, and keep the distinct global `heartbeat.to` inheritance leak tracked separately in #73900.

## Reproduction Assessment

Yes for the narrowed runtime-context issue: configure `heartbeat.target: "none"` so no delivery recipient resolves, let sender resolution fall back to `heartbeat`, and current main passes that sentinel as `ctx.To`. The full persisted `lastTo` corruption path is no longer a high-confidence current-main reproduction because system-event session persistence now preserves the existing route.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` while leaving `From: sender` intact is the narrowest maintainable runtime fix; the safer test assertion is to require `capturedTo` to be undefined in the target-none case.
