## Summary

Keep open. Current main still builds heartbeat context with `ctx.To` from `sender`, and the PR's runtime fix matches the delivery-target contract, but the external contributor still has not provided after-fix real behavior proof and the new regression assertion should be tightened.

## What This Changes

The PR sets heartbeat `ctx.To` from `delivery.to`, adds a target-none regression test, and adds a changelog entry.

## Best Possible Solution

Land the narrow `ctx.To = delivery.to` runtime change after redacted real behavior proof is added and the regression test asserts target-none `ctx.To` is exactly `undefined`.

## Reproduction Assessment

Yes, for a source-level reproduction. Current main sets `ctx.To` to `sender`, target-none delivery leaves `delivery.to` unset, and the sender resolver can produce the literal `heartbeat` sentinel; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Deriving `ctx.To` from the resolved delivery target is the narrow maintainable fix because `To` should describe the outbound recipient while `From` can remain the synthetic heartbeat identity; the safer test shape is to assert `ctx.To` is `undefined` for target-none heartbeats.
