## Summary

Keep open: the runtime change is well-targeted and current main still has the field-level bug, but this external PR needs real behavior proof before merge and the new regression assertion should be tightened.

## What This Changes

The PR changes heartbeat run context to set `ctx.To` from `delivery.to`, adds a target-none regression test, and adds a changelog fix entry.

## Best Possible Solution

Land the runtime context change after tightening the target-none regression to assert `ctx.To` is exactly undefined and after the contributor adds redacted real behavior proof; keep the separate global `heartbeat.to` inheritance issue out of this PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence field-level reproduction: current main sets `ctx.To` to `sender`, while target-none delivery omits `delivery.to` and sender can come from `lastTo` or the `heartbeat` sentinel; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` is the narrow maintainable fix because `To` should describe the actual outbound target while `From` may remain the synthetic heartbeat identity; the test should pin target-none `ctx.To` to undefined.
