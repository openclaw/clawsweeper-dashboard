## Summary

Keep open. Current main still mirrors heartbeat `sender` into `ctx.To`, and the PR's runtime change is the right narrow fix; the only actionable review item is tightening the new regression assertion so it fails when any non-delivery sender is copied into `To`.

## What This Changes

The PR changes heartbeat runtime context construction so `ctx.To` comes from `delivery.to`, adds a target-none regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the runtime context change with the new target-none regression assertion tightened to require `capturedTo` to be undefined, while leaving the distinct #73900 inheritance problem tracked separately.

## Reproduction Assessment

Yes for the narrowed field-level bug: current main sets `ctx.To` from `sender`, while target-none delivery has no `delivery.to`. The broader persisted `lastTo` corruption path is partly mitigated by the system-event guard on current main, so the highest-confidence repro is the heartbeat context value itself.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` while keeping `From: sender` preserves the synthetic prompt identity without treating it as a delivery recipient; the safer test fix is to assert the exact undefined value for target-none delivery.
