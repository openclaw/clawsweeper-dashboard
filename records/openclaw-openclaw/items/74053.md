## Summary

Keep open: the runtime direction is right for current main, but the regression test does not actually fail on the current sender fallback path because it allows the seeded `lastTo` value as a passing `ctx.To`. Tighten that assertion before merge; no security or supply-chain issue was found.

## What This Changes

The PR changes heartbeat context construction to set `ctx.To` from `delivery.to`, adds a target-none regression test, and adds a changelog fix entry.

## Best Possible Solution

Land the runtime context change after tightening the target-none regression to require `ctx.To` to be undefined, while leaving the distinct global `heartbeat.to` inheritance issue to #73900.

## Reproduction Assessment

Yes, for the narrowed field-level bug by source inspection: current main sets `ctx.To` to `sender`, while `target: "none"` produces no `delivery.to` and sender can come from a remembered route or the `heartbeat` sentinel. I did not execute the test suite in this read-only review.

## Solution Assessment

Yes, with one test fix. Deriving `ctx.To` from `delivery.to` keeps recipient context tied to the actual delivery target while preserving `From: sender`; the regression should assert the target-none value is exactly undefined.
