## Summary

Keep this PR open. The runtime fix is well targeted, but the new regression test should be tightened because it can pass on current main when a remembered sender is copied into `ctx.To`; the branch also currently reports as dirty against the base.

## What This Changes

The PR changes heartbeat runtime context to set `ctx.To` from `delivery.to`, adds a target-none regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the runtime context change after tightening the target-none regression assertion to require `capturedTo` to be undefined and refreshing the branch against current main.

## Reproduction Assessment

Yes for the narrowed field-level bug: current main sets `ctx.To` from `sender`, while target-none delivery has no `delivery.to`. The broader persisted `lastTo` corruption path is partly mitigated for system-event turns by the current session guard, so the highest-confidence reproduction is the heartbeat context value itself.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` while keeping `From: sender` preserves the synthetic prompt identity without treating it as a delivery recipient; the test should assert the exact undefined value.
