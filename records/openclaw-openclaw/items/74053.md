## Summary

Keep open. Current main still builds heartbeat `ctx.To` from `sender` even when `target: "none"` has no delivery recipient; the PR's runtime change is the right narrow fix, but the new regression assertion should be tightened because it would pass when current main copies a seeded `lastTo` into `ctx.To`.

## What This Changes

The PR changes heartbeat run context to set `ctx.To` from `delivery.to`, adds a target-none regression test, and records the fix in the changelog.

## Best Possible Solution

Land the runtime context change after tightening the target-none regression to require no `ctx.To` recipient, while leaving the distinct global `heartbeat.to` inheritance issue tracked separately.

## Reproduction Assessment

Yes, for the narrowed field-level bug by source inspection: current main sets `ctx.To` to `sender`, while `target: "none"` returns no `delivery.to`. I did not run tests because this review is read-only; the broader persisted `lastTo` corruption path is partly mitigated by the current system-event route-preservation guard.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` keeps `To` tied to the actual delivery recipient while preserving `From: sender` for the synthetic heartbeat identity; the safer validation is to assert the target-none value is exactly undefined.
