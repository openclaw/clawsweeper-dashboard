## Summary

Keep open: the runtime direction is narrow and correct, but the added regression test is too weak because the seeded `lastTo` path can satisfy `not.toBe("heartbeat")` while `ctx.To` is still wrong.

## What This Changes

The PR changes heartbeat context construction to set `ctx.To` from the resolved delivery target, adds a target-none regression test, and adds a changelog entry.

## Best Possible Solution

Land the runtime context change with the regression assertion tightened to require `ctx.To` to be undefined for target-none delivery, while leaving the distinct #73900 inherited `heartbeat.to` issue tracked separately.

## Reproduction Assessment

Yes for the narrowed field-level bug: current main sets `ctx.To` from `sender` while target-none delivery has no `delivery.to`. The broader persisted `lastTo` corruption path is partially mitigated by the current system-event guard, so the reliable repro is the heartbeat context value itself.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` preserves the synthetic `From` identity without treating it as a recipient; the safer test assertion is to pin the target-none value to `undefined`.
