## Summary

Keep open: current main still sets heartbeat `ctx.To` from `sender`, and this focused PR moves it to the resolved delivery target; it needs contributor real behavior proof before merge, plus a small regression assertion tightening.

## What This Changes

This PR changes heartbeat runner context to set `To` from the resolved delivery target, adds a target-none regression test, and adds a changelog fix entry.

## Best Possible Solution

Land the runtime context change after the contributor adds redacted real behavior proof and the test pins target-none `ctx.To` to exactly `undefined`; keep separate heartbeat routing reports out of this narrow patch.

## Reproduction Assessment

Yes, for a source-level reproduction. Current main sets `ctx.To` to `sender`, while target-none delivery omits `delivery.to` and the sender resolver can return a remembered recipient or the `heartbeat` sentinel; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` is the narrow maintainable fix because `To` should describe the actual outbound target while `From` may remain the synthetic heartbeat identity.
