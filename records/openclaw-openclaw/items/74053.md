## Summary

Keep open: current main still mirrors `sender` into heartbeat `ctx.To`, and the PR’s runtime change is the narrow fix, but the added regression test can pass without the runtime change because it only excludes the literal sentinel instead of asserting no target-none recipient.

## What This Changes

The PR changes heartbeat runtime context to set `To` from `delivery.to`, adds a target-none regression test, and records the heartbeat routing fix in `CHANGELOG.md`.

## Best Possible Solution

Land the runtime `To: delivery.to` change after tightening the target-none regression to require an absent `ctx.To`, while leaving the separate global `heartbeat.to` inheritance leak to #73900.

## Reproduction Assessment

Yes, for the narrowed field-level bug by source inspection: current main sets `ctx.To` to `sender`, target-none delivery has no `delivery.to`, and sender can fall back to `heartbeat`. The broader persisted `lastTo` corruption path is partly mitigated by the current system-event route-preservation guard.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` keeps the prompt context tied to the actual outbound recipient, and the safer test is to assert `ctx.To` is exactly `undefined` when `target: "none"`.
