## Summary

Keep this PR open. Current main still mirrors the heartbeat sender into `ctx.To`, and the proposed runtime change is a narrow fix, but the new regression test should assert that target-none heartbeats have no `ctx.To` rather than only checking it is not the sentinel.

## What This Changes

The PR changes heartbeat run context to set `ctx.To` from `delivery.to`, adds a target-none regression test, and records the fix in the changelog.

## Best Possible Solution

Land the runtime context change after tightening the regression test to require `ctx.To` to be `undefined` for target-none heartbeats, while leaving the separate global `heartbeat.to` inheritance issue tracked in #73900.

## Reproduction Assessment

Yes. Source inspection on current main shows `target: "none"` resolves no `delivery.to`, while `runHeartbeatOnce` still writes `sender` into `ctx.To`; the broader persisted `lastTo` corruption path is partly mitigated by the current system-event route-preservation guard.

## Solution Assessment

Yes. Deriving `ctx.To` from the resolved delivery target is the narrowest maintainable fix because it keeps recipient context tied to an actual delivery route while preserving `From: sender` for heartbeat identity; the test should pin the no-target value to `undefined`.
