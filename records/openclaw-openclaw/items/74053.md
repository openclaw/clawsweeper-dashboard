## Summary

Keep this PR open. Current main still builds the heartbeat runtime context with `ctx.To` copied from `sender`, so the PR’s central runtime-context fix is not implemented on main; however, current main also has a broader session persistence guard that already prevents automated heartbeat/cron/exec turns from rewriting persisted `lastTo`, so the useful remaining change is the narrower in-turn `ctx.To` correction plus a small test assertion tightening and branch repair/replacement.

## What This Changes

The PR changes heartbeat runtime context construction so `ctx.To` uses `delivery.to`, adds a target-none regression test, and adds an Unreleased changelog fix entry.

## Best Possible Solution

Land the narrow heartbeat context correction on current main while keeping the existing system-event session-store guard; the regression test should assert the exact target-none `ctx.To` value and preserve the contributor credit in the changelog.

## Reproduction Assessment

Yes for the narrowed current-main issue: inspect or exercise `runHeartbeatOnce` with `heartbeat.target: "none"` and a queued system event, where `resolveHeartbeatSenderId` can return `heartbeat` and `runHeartbeatOnce` copies that into `ctx.To`. The historical persisted-route corruption is not fully reproducible on current main because `session.ts` now preserves persisted routes for system-event providers.

## Solution Assessment

Yes. Deriving `ctx.To` from `delivery.to` while leaving `From: sender` is the narrowest maintainable runtime-context fix, and the broader session persistence guard should remain because it protects more automated event paths than this sentinel case.
