## Repair Work Prompt

Implement a narrow OpenClaw core skills prompt improvement for https://github.com/openclaw/openclaw/issues/91385. Current main stores `SkillSnapshot.version` and refreshes snapshots when watcher/config/remote changes, but the model-visible `<available_skills>` root in full and compact workspace prompts has no version marker. Add a snapshot-aware workspace prompt rendering path that includes the current snapshot version on the `<available_skills>` root when a version is known, and add concise Skills guidance telling the model to re-read a skill before use when the snapshot differs from a prior turn. Preserve the existing public/default `formatSkillsForPrompt(skills)` output and upstream alignment unless maintainers intentionally update the plugin SDK contract; do not add config/env knobs or unrelated skill discovery changes. Update focused tests for full, compact, prompt resolution, and system prompt guidance, and update `docs/concepts/system-prompt.md` if the user-visible prompt shape changes. Do not edit `CHANGELOG.md`.
