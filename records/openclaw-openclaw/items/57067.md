## Summary

Keep open. Current main and the latest release still use dropdown-first Agents-page navigation; the invalid `agents.defaultId` write was partially fixed, but the visible agent-list regression and refresh-safe default-agent UX remain actionable.

## What This Changes

The issue asks the Control UI Agents page to show agents directly instead of hiding them in a dropdown, and to make default-agent changes persist across refresh.

## Best Possible Solution

Restore an accessible visible Agents list/sidebar as the primary normal-width navigation, keep a compact responsive fallback, and make Default either persist through `agents.list[].default` or clearly remain save-gated until the user saves.

## Reproduction Assessment

Yes, at source level. Rendering `renderAgents()` with multiple agents still puts all choices into one `.agents-select`, and the Default click path only stages a config draft through `stageDefaultAgentConfigEntry()`.

## Solution Assessment

Yes. A focused Control UI repair in the Agents view, styles, default-action wiring, and targeted controller/view tests is the narrowest maintainable path; no new config key or core protocol change is needed.
