## Summary

Keep open. Current main and latest release still use dropdown-first Agents-page navigation; the invalid `agents.defaultId` save path was fixed by a merged PR, but the visible agent-list regression and the refresh-safe/default-save UX remain actionable.

## What This Changes

The issue asks for the Agents page to show agents directly instead of hiding them behind a dropdown, and for the Default action to persist across refresh.

## Best Possible Solution

Restore an accessible visible Agents list/sidebar as the primary normal-width navigation, keep a compact responsive fallback, and make the Default action refresh-safe through the existing `agents.list[].default` config contract or a clearly save-gated UI.

## Reproduction Assessment

Yes, at source level. Rendering `renderAgents()` with multiple agents still puts all choices into one `.agents-select`, and the Default click path only stages the config draft through `stageDefaultAgentConfigEntry()`.

## Solution Assessment

Yes. A focused Control UI repair in the Agents view, styles, default-action wiring, and targeted UI/controller tests is the narrowest maintainable fix; no new config key or core protocol change is required for this issue.
