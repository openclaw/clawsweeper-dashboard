## Label Justifications

- `P1`: Routine idle heartbeats can cause repeated paid model calls, fallback exhaustion, and an unintended persistent model selection.
- `impact:auth-provider`: The bug misclassifies a successful model result as a format failure, alters candidate routing, and can persist an automatic model override.

## Summary

This issue remains necessary on current main: text-only `NO_REPLY` is removed from terminal payloads, but the silent-success classifier rejects the turn because its original assistant text is non-empty, leaving empty-response retry, candidate failure, fallback, and automatic model-override behavior active.

## What This Changes

Treat explicit `NO_REPLY` assistant turns as successful silent terminal outcomes instead of empty incomplete responses that trigger retries and model fallback.

## Best Possible Solution

Carry an explicit recognized-silent-reply fact from payload construction into the canonical terminal outcome so retry decisions, fallback classification, lifecycle metadata, and automatic model-override persistence all agree that the turn succeeded silently.

## Reproduction Assessment

Yes at source level. On current main, exact `NO_REPLY` text is filtered to zero payloads while its non-empty original assistant text prevents the silent-success classifier from firing; the reporter also provides a concrete v2026.6.5 heartbeat trace.

## Solution Assessment

Yes in direction, but the most maintainable fix is to propagate an explicit deliberate-silence terminal fact rather than add a heartbeat- or fallback-specific exception after payload filtering.
