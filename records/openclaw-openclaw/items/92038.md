## What This Changes

The issue reports that `NO_REPLY`-only assistant turns can be filtered to zero payloads and then treated as empty failures, causing retry, fallback, and sticky model override behavior during heartbeat runs.

## Best Possible Solution

Land a repaired version of the linked PR or an equivalent narrow fix that classifies explicit `NO_REPLY` terminal replies as `silent-empty` only under the existing silent-reply policy, preserves post-tool ambiguity safeguards, and proves no heartbeat retry/fallback cascade remains.
