Keep this open: current main still has the source-level `NO_REPLY` silent-turn classification gap, and an open PR with closing syntax is already the active fix path.

## What This Changes

The issue reports that `NO_REPLY`-only embedded assistant turns are treated as empty failures, causing retries, fallback cascade, and a sticky model override during heartbeats.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92073, or replace it with an equivalent narrow patch that treats explicit `NO_REPLY` as successful silence while preserving post-tool safety.

## Reproduction Assessment

Yes at source level: current main filters `NO_REPLY` payloads before terminal completion and then rejects the remaining assistant text before accepting it as silent. I did not run the live provider heartbeat in this read-only review.
