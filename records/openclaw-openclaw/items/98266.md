## Summary

Keep open. Current main still creates a direct Discord Gateway WebSocket agent unless explicit Discord proxy config is set; the broad proxy feature issue and earlier closed PR attempts do not safely own this narrow remaining bug.

## What This Changes

Request Discord Gateway WebSocket setup to honor standard `HTTP_PROXY`/`HTTPS_PROXY` env proxy fallback when no explicit Discord proxy is configured.

## Best Possible Solution

Add a narrow Discord Gateway WebSocket env-proxy fallback using `createNodeProxyAgent` for `wss://gateway.discord.gg`, keep explicit `channels.discord.proxy` precedence, preserve `NO_PROXY` semantics, and cover direct/env/explicit cases in focused Discord proxy tests.

## Reproduction Assessment

Yes. Source inspection shows the Discord Gateway WebSocket path creates a direct `HttpsAgent` unless explicit Discord proxy config is present, while Discord REST and WhatsApp WebSocket already have env proxy fallback.

## Solution Assessment

Yes, with a security review. Reusing the existing plugin-sdk Node proxy agent is the narrow maintainable solution because it avoids new config surface and matches sibling channel behavior.
