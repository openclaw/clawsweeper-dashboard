## Summary

Keep open: current main and latest release still implement only Telegram forum-topic create/edit through the message plugin, while close, reopen, and delete remain missing across the shared action contract, Telegram discovery, dispatch, config schema, docs, and tests.

## What This Changes

The issue asks to expose Telegram forum topic create, edit, close, reopen, and delete lifecycle operations through the message plugin.

## Best Possible Solution

Define one typed Telegram forum-topic lifecycle API in the shared message-action contract, implement it inside the Telegram plugin, and align discovery, config schema, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection on current main shows only `topic-create` and `topic-edit` are advertised and dispatched, while close/reopen/delete topic operations are absent and unsupported.

## Solution Assessment

Yes as a canonical tracker, but not enough as an implementation plan. The best fix needs maintainer-approved action names and gates, then a coherent Telegram plugin implementation rather than another deletion-only branch.
