## Label Justifications

- `P2`: This is a bounded, useful Telegram management enhancement with limited blast radius and no urgent regression.
- `impact:other`: The missing lifecycle actions force users to administer forum topics manually, a concrete workflow limitation outside the more specific impact categories.

## Summary

Current main still supports only Telegram forum-topic creation and editing through the message tool; close, reopen, and delete remain absent, and no viable open or merged PR owns the remaining work. The issue should remain the canonical tracker, with maintainers first choosing the permanent action names and deletion semantics.

## What This Changes

Add Telegram forum-topic close, reopen, and delete actions alongside the existing create and edit message-tool actions.

## Best Possible Solution

Define one typed topic lifecycle API in the shared message-action contract, implement it entirely within the Telegram plugin using grammY-native methods, and align discovery, schemas, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main deterministically advertises and dispatches only topic creation and editing, so close, reopen, and topic deletion are unavailable through the message tool. A live Telegram run is needed to validate a future patch, not to establish the feature gap.

## Solution Assessment

The requested direction is appropriate, but the exact public action names and delete semantics need maintainer approval. Distinct typed lifecycle actions are safer and more maintainable than making generic message deletion carry two unrelated meanings.
