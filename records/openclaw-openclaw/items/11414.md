## Summary

Keep this open. Current main still supports Telegram forum topic creation and editing only; close, reopen, and delete remain absent across the shared message action contract, Telegram discovery/action mapping, runtime dispatch, config schema, docs, and tests, while the prior deletion-only PR was closed unmerged as unsafe/stale.

## What This Changes

The issue requests Telegram forum topic lifecycle management through the message plugin: create, edit, close, reopen, and delete topic actions.

## Best Possible Solution

Keep this as the canonical tracker and land one coherent Telegram plugin implementation for close, reopen, and delete alongside existing create/edit, with explicit action mapping, config gating, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, by source inspection rather than a live Telegram run: current main exposes and tests only `topic-create` and `topic-edit`, while searches found no close/reopen/delete forum-topic action or helper path.

## Solution Assessment
