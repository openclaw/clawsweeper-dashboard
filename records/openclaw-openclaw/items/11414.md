## Summary

Current main implements Telegram forum-topic creation and editing, but still lacks close, reopen, and delete across the shared action contract, Telegram discovery, dispatch, configuration, docs, and tests. The deletion-only pull request was closed unmerged and explicitly routed the remaining lifecycle work back to this canonical issue, so the request remains necessary and needs a maintainer-approved API direction.

## What This Changes

Add complete Telegram forum-topic create, edit, close, reopen, and delete lifecycle management through the message tool.

## Best Possible Solution

Approve one coherent Telegram-owned lifecycle contract with distinct typed close, reopen, and delete actions, then implement discovery, dispatch, grammY-native transport helpers, configuration/schema, docs, and focused tests together.

## Reproduction Assessment

Yes at high confidence from current-main source: the shared action list, Telegram mapping and discovery, runtime dispatch, config schema, docs, and tests cover creation and editing but not closing, reopening, or deletion. This verifies a capability gap rather than reproducing a broken existing contract.

## Solution Assessment

The coherent full-lifecycle direction is the best maintainable approach, but the interface sketch is not ready to implement unchanged. Maintainers should approve distinct action names and configuration semantics rather than reuse the existing message-delete action by implication.
