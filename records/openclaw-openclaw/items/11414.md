Keep open. Current main still exposes only Telegram forum topic creation and editing through the message action surface; close, reopen, and delete remain missing, and the related open deletion PR is only partial prior art for the broader lifecycle request.

Keep this as the canonical tracker and land one coherent Telegram plugin change that adds close, reopen, and delete alongside the existing create/edit support across runtime, discovery, config, docs, and focused tests.

## Solution Assessment

Yes for the general direction, but not by closing this issue. The maintainable path is to extend the current Telegram plugin runtime/action surface and coordinate or replace the partial deletion-only PR rather than treating that PR as a complete fix.
