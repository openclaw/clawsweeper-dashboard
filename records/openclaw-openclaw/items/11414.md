## Summary

Keep open. Current main and the latest release still expose Telegram forum-topic create/edit only; close, reopen, and delete remain missing, and the deletion-only implementation prior art at https://github.com/openclaw/openclaw/pull/41410 was closed unmerged rather than becoming a safe canonical fix path.

## What This Changes

The issue requests Telegram forum topic lifecycle actions in the message plugin: create, edit, close, reopen, and delete.

## Best Possible Solution

Define one typed Telegram forum-topic lifecycle API in the shared message-action contract, implement the remaining operations in the Telegram plugin, and align discovery, config schema, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, from source inspection: current main exposes Telegram forum-topic create/edit but lacks close, reopen, and delete across action names, discovery, runtime dispatch, helpers, config, docs, and tests. This is source-reproducible feature absence rather than a failing bug reproduction.

## Solution Assessment

Unclear until maintainers choose the API shape. The implementation belongs in the Telegram plugin and shared message-action contract, but public action naming, destructive-operation semantics, and config gating need maintainer confirmation before a repair PR is safe.
