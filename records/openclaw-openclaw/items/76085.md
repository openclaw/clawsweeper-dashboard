## Summary

Keep open. Current main still has the runtime bundled-provider allowlist bypass and this PR is the right narrow, default-preserving direction, but it still needs public docs/changelog coverage and the maintainer-requested contract note before merge.

## What This Changes

The branch adds an opt-in `plugins.bundledMode: "respect-allow"` setting with schema/help wiring and tests so bundled provider discovery and activation paths can honor `plugins.allow` while default compat behavior remains unchanged.

## Best Possible Solution

Land the opt-in, default-preserving implementation with public config docs, an Unreleased changelog entry, focused tests, and a small note or test that locks the empty-allow `respect-allow` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a restrictive `plugins.allow`, then bundled provider discovery returns non-workspace plugins before allowlist checks and compat activation can add discovered IDs back into the effective config.

## Solution Assessment

Yes as a direction, but not yet as a shippable patch. The opt-in mode is the narrowest maintainable fix because it preserves legacy defaults; docs/changelog and the requested contract note are the remaining merge blockers.
