## Summary

Keep open. Current main still has the bundled provider allowlist bypass, and the PR is a narrow default-preserving fix, but the patch is not merge-ready because the new user-facing config key is not reflected in generated schema/docs artifacts, public docs, or the changelog, and a maintainer requested one explicit contract note for the empty-allow edge case.

## What This Changes

The PR adds an opt-in `plugins.bundledMode: "respect-allow"` setting so bundled provider runtime discovery and compat activation honor `plugins.allow` while preserving legacy `compat` behavior by default.

## Best Possible Solution

Land the opt-in runtime gating with the compat default unchanged, regenerated config schema/docs baselines, public docs, an Unreleased changelog entry, and a small note locking the empty-allow behavior as intentional.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a restrictive `plugins.allow`, then bundled provider discovery returns non-workspace provider plugins before allowlist checks and activation adds those ids back into the effective config.

## Solution Assessment

Yes as a direction, but not yet as a shippable patch. The opt-in `respect-allow` mode is the narrow maintainable fix because it preserves legacy defaults while using existing activation seams; the remaining work is release/documentation alignment and the requested edge-contract note.
