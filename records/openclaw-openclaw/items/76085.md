## Summary

Keep open. Current main still has the bundled-provider allowlist bypass, and this PR is directionally useful, but the proposed opt-in is not mergeable yet because the new config key is not accepted by the strict runtime schema and the linked issue's web-provider runtime paths remain uncovered.

## What This Changes

This PR adds a `plugins.bundledMode` opt-in, adjusts bundled provider discovery/activation compatibility helpers, and adds provider allowlist tests.

## Best Possible Solution

Land a complete opt-in implementation that preserves default compat behavior, makes `plugins.bundledMode` a validated/documented config key, and applies the mode consistently across LLM, web-search, and web-fetch provider runtime discovery.

## Reproduction Assessment

Yes. A config containing `plugins.bundledMode: "respect-allow"` follows the existing strict `plugins` schema and would be rejected before the new runtime logic can run; the remaining web-provider bypass is reproducible by reading the hard-coded `enablement: "always"` call sites.

## Solution Assessment

No. The opt-in shape is a reasonable narrow approach, but this PR does not yet make the option usable through config validation and does not cover all runtime provider paths described by #75575.
