## Summary

Keep open: current main still has the bundled-provider allowlist bypass, and this PR is a narrow default-preserving direction, but the latest diff still lacks required public docs and changelog coverage for the new user-facing config option.

## What This Changes

The PR adds an opt-in `plugins.bundledMode: "respect-allow"` mode, wires provider discovery and bundled compat helpers to honor restrictive allowlists, and adds schema/help entries plus focused tests.

## Best Possible Solution

Land the opt-in `respect-allow` implementation with public plugin/config documentation, an Unreleased changelog entry, and targeted provider/compat validation.

## Reproduction Assessment

Yes. On current main, configure a restrictive `plugins.allow`; bundled provider discovery returns non-workspace plugins before allowlist checks, and compat activation can add discovered IDs back into `allow` and `entries`.

## Solution Assessment

Yes as a direction, but not yet as a shippable patch. The opt-in mode preserves legacy defaults and uses existing activation seams, but user-facing docs and release notes must be added before merge.
