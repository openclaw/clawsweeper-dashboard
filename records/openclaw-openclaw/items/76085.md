## Summary

Keep open. Current main still has the bundled-provider allowlist/runtime discovery gap from #75575, and this PR is the right narrow direction after the earlier schema and web-path questions, but it still needs public docs and changelog coverage before merge.

## What This Changes

The PR adds an opt-in `plugins.bundledMode: "respect-allow"` setting that filters bundled provider discovery and activation by `plugins.allow` while preserving default compat behavior, with schema/help wiring and focused tests.

## Best Possible Solution

Land the opt-in runtime allowlist mode after adding public plugin/config docs and an Unreleased changelog entry that explain default compat behavior versus `respect-allow`.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: configure a restrictive `plugins.allow`, then bundled provider discovery still includes non-allowlisted bundled plugins and activation can add discovered plugin IDs back into the effective config.

## Solution Assessment

Yes as a direction, but not yet as a shippable patch. The opt-in config preserves backwards compatibility and reuses existing activation seams; the safer remaining step is documentation and changelog alignment before merge.
