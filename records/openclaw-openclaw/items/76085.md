## Summary

Keep open. The PR is directionally aligned with the linked allowlist issue, but it is not merge-ready because the new config key is not accepted by the strict config schema and the web-search/web-fetch runtime paths cited by the report still force-enable bundled provider plugins.

## What This Changes

Adds an opt-in `plugins.bundledMode: "respect-allow"` path for bundled model-provider allowlist filtering in provider discovery/activation and adds focused tests.

## Best Possible Solution

Preserve the default compatibility behavior, but make `plugins.bundledMode: "respect-allow"` a real documented config contract and thread it through all bundled provider runtime surfaces named by the linked issue before landing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: add `plugins.bundledMode` to config and strict validation lacks the key, while the linked K8s comment and current runtime call sites show bundled provider paths still bypassing a restrictive allowlist.

## Solution Assessment

No. The opt-in mode is the right narrow direction, but this patch is incomplete until schema/help/docs/changelog are aligned and the web-search/web-fetch runtime provider paths honor the same mode.
