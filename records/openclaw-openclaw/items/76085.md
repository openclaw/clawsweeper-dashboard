## Summary

Keep open. The PR addresses the main bundled-provider allowlist bypass and now includes schema/help wiring, but it still needs a small policy fix for the new mode plus public docs and changelog coverage before merge.

## What This Changes

The PR adds an opt-in `plugins.bundledMode: "respect-allow"` mode so bundled provider discovery can honor `plugins.allow` while preserving default compatibility behavior.

## Best Possible Solution

Land a default-preserving `respect-allow` mode that honors `plugins.allow`, `plugins.entries.<id>.enabled`, and the master plugin disable switch, with focused tests plus public config docs and an Unreleased changelog entry.

## Reproduction Assessment

Yes. The original issue is source-reproducible on current main by using a restrictive `plugins.allow` and following provider setup discovery into `withActivatedPluginIds`; the PR blocker is also source-reproducible by passing `plugins.enabled: false` with `bundledMode: "respect-allow"` through the changed bundled compat helper.

## Solution Assessment

Not yet as a shippable patch. The opt-in direction is the narrow maintainable fix, but the current diff still leaves one legacy enablement override active and omits required public release/docs coverage.

## Review Findings
