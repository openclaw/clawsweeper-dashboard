- **[P1] Publish the mirrored prompt to selected subscribers:** `extensions/codex/src/app-server/run-attempt.ts:5725-5737`

  - confidence: 0.9
- **[P2] Ignore stale subscription sync completions:** `ui/src/ui/controllers/sessions.ts:503-514`
  - body: syncSelectedSessionMessageSubscription captures nextKey, then awaits network calls while switchChatSession fires it without awaiting. If the user switches A -> B -> C quickly, the B sync can finish after the C sync and set chatSessionMessageSubscriptionKey back to B, leaving the UI subscribed to the wrong session and missing C's live updates. Guard the client/session generation after each await, or serialize/cancel these syncs.
  - confidence: 0.87
