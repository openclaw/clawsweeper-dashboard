## Summary

Keep open. The PR targets a valid observability gap that is still not implemented on current main, but the current head should not merge because the audit attribution can be misleading, sandbox audit logging is inserted in a shared formatter path, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds INFO-level audit logs for tool-policy pipeline filtering and sandbox-denied tools, plus tests for emitted and suppressed pipeline audit logs.

## Best Possible Solution

Carry matched allow/deny rule metadata through policy evaluation and emit one sanitized audit record at enforcement boundaries, with focused tests and redacted real audit-output proof.

## Reproduction Assessment

Yes, source-reproducible. Current main returns filtered tool lists or booleans without matched-rule audit attribution, and the PR diff shows the attribution and duplicate-logging risks without needing a live run.

## Solution Assessment

No. The direction is useful, but the PR should report matched allow/deny rule metadata and move sandbox audit logging to a single enforcement boundary or add explicit per-run/tool dedupe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Report the matched allow or deny rule:** `src/agents/tool-policy-pipeline.ts:161`
  - body: The audit line uses `step.label`, but default global/agent/group policy steps are labeled `*.allow` even when `policy.deny` removes a tool. In an allow+deny policy under `tools.allow`, this points operators at the wrong rule; carry the matched rule type/path from policy evaluation instead.
  - confidence: 0.91
- **[P2] Emit sandbox audit logging once at enforcement:** `src/agents/sandbox/runtime-status.ts:160-162`
  - body: The new `logInfo` call is inside `formatSandboxToolPolicyBlockedMessage`, which is reached through shared assistant-error rendering paths. One blocked tool call can therefore produce repeated identical INFO audit entries, so emit at a single enforcement boundary or dedupe per run/tool.
  - confidence: 0.87
