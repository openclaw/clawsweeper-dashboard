## Summary

Keep open: this PR targets a real observability gap tracked by the linked open issue, but the current patch is not merge-ready because it can misattribute policy decisions, can duplicate sandbox audit logs, is merge-conflicting, and still lacks real behavior proof.

## What This Changes

Adds INFO-level audit logs for tool-policy pipeline removals and sandbox policy blocks, plus focused pipeline logging tests.

## Best Possible Solution

Carry sanitized matched allow/deny rule metadata through policy evaluation and emit one audit record at enforcement boundaries, with focused tests and redacted real-output proof.

## Reproduction Assessment

Yes, source-level. Current main collapses tool policy to filtered lists or boolean allow/deny results, and the PR diff exposes the attribution and duplicate-rendering issues without a live run.

## Solution Assessment

No. The PR is in the right area, but broad step labels are not matched-rule attribution and formatter-side sandbox logging can overcount; a matcher or explanation seam plus enforcement-side logging is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Report the matched policy rule:** `src/agents/tool-policy-pipeline.ts:161`
  - body: The audit line uses `step.label`, but a step labeled `tools.allow` can include both `allow` and `deny`. A deny removal can therefore be reported as an allow-list rule instead of the matched deny pattern, so policy evaluation needs to carry matched allow/deny rule metadata before logging.
  - confidence: 0.91
- **[P2] Move sandbox audit logging out of the formatter:** `src/agents/sandbox/runtime-status.ts:160-162`
  - body: The new `logInfo` call is inside `formatSandboxToolPolicyBlockedMessage`, which is reached through `formatAssistantErrorText` from multiple rendering and failover paths. Rendering the same blocked-tool error more than once can emit duplicate INFO audit records, so emit at enforcement or dedupe per run/tool.
  - confidence: 0.87
