## Summary

Keep open: current main still lacks matched-rule tool-policy audit attribution, and this PR is not merge-ready because it logs broad pipeline labels as rule attribution, emits sandbox audit logs from a shared formatter path, lacks contributor real behavior proof, and is currently conflicting.

## What This Changes

The PR adds INFO-level audit logs for tool-policy pipeline filtering and sandbox-denied tool calls, plus focused pipeline tests for removal logging.

## Best Possible Solution

Carry matched allow/deny rule metadata through policy evaluation and emit one sanitized audit record at enforcement boundaries, with focused tests and redacted real-output proof.

## Reproduction Assessment

Yes, source-level. Current main filters tool policy down to booleans or tool lists without matched-rule audit metadata, and the PR diff exposes the attribution and duplicate-rendering issues without needing a live run.

## Solution Assessment

No. The PR is in the right area, but broad step labels are not matched-rule attribution and formatter-side sandbox logging can overcount; a matcher/pipeline explanation seam plus enforcement-side logging is the safer design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Report the matched policy rule:** `src/agents/tool-policy-pipeline.ts:161`
  - body: The audit line uses `step.label`, but a step labeled `tools.allow` can include both `allow` and `deny`. That means a deny removal can be reported as an allow-list rule instead of the matched deny pattern, so policy evaluation needs to carry the matched allow/deny rule metadata before logging.
  - confidence: 0.91
- **[P2] Move sandbox audit logging out of the formatter:** `src/agents/sandbox/runtime-status.ts:160-162`
  - body: The new `logInfo` call is inside `formatSandboxToolPolicyBlockedMessage`, which is reached through `formatAssistantErrorText` from multiple rendering and failover paths. Rendering the same blocked-tool error more than once can emit duplicate INFO audit records, so emit at enforcement or dedupe per run/tool.
  - confidence: 0.87
