## Summary

Keep open. The audit-logging goal is useful and still not implemented on current main, but this PR does not yet provide true matched-rule attribution, can duplicate sandbox audit lines from formatter paths, and lacks required real behavior proof.

## What This Changes

The PR adds INFO-level audit logs for tool-policy pipeline filtering and sandbox-denied tool calls, plus tests for pipeline removal logging.

## Best Possible Solution

Carry matched allow/deny rule metadata through policy evaluation and emit one sanitized audit record at enforcement boundaries, with focused tests and redacted real audit-output proof.

## Reproduction Assessment

Yes. Source inspection on current main shows policy evaluation returns filtered tools or booleans without matched-rule audit metadata, and the PR diff shows the misattribution and duplicate-rendering paths without needing a live run.

## Solution Assessment

No. The PR is in the right area, but broad step labels are not rule attribution and formatter-side sandbox logging can overcount; a matcher/pipeline explanation seam plus enforcement-side logging is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Report the matched allow or deny rule:** `src/agents/tool-policy-pipeline.ts:161`
  - body: The audit line uses `step.label`, but default steps such as `tools.allow` can also contain `deny` rules. An allow+deny policy can therefore report a deny removal as `tools.allow`, so policy evaluation needs to carry the matched rule type/path instead of only the broad pipeline label.
  - confidence: 0.91
- **[P2] Emit sandbox audit logging once at enforcement:** `src/agents/sandbox/runtime-status.ts:160-162`
  - body: The new `logInfo` call is inside `formatSandboxToolPolicyBlockedMessage`, which is reached through shared error-rendering paths via `formatAssistantErrorText`. Rendering the same blocked tool error more than once can produce duplicate INFO audit lines, so emit at enforcement or dedupe per run/tool.
  - confidence: 0.87
