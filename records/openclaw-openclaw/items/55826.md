## Summary

Keep open. The PR addresses a real operator debugging gap tracked by #55801, and the earlier Greptile test-spy teardown note has been fixed, but the current patch is not merge-ready because the pipeline audit can attribute a deny removal to an allow key and the sandbox audit side effect is placed in shared error formatting.

## What This Changes

The PR adds INFO-level audit logs for tool-policy pipeline removals and sandbox blocked-tool messages, plus unit coverage for pipeline audit logging.

## Best Possible Solution

Implement audit logging at the policy decision boundary with matched allow/deny rule and source-key attribution, while keeping user-facing error formatting side-effect-free or explicitly deduped per blocked call.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a normal deny entry can be filtered through a default step labeled tools.allow, and an unknown-tool sandbox error can reach the formatter-side audit path through multiple error-rendering callers.

## Solution Assessment

No. The direction is useful, but the patch should derive attribution from the matched rule/source and emit sandbox audit logs once at enforcement time or through a deliberate dedupe path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report the matched allow or deny key:** `src/agents/tool-policy-pipeline.ts:161`
  - body: The audit line interpolates `step.label`, but default policy steps are labeled `*.allow` even when a `deny` list removes a tool. With a normal `tools.deny` rule, the new log can point operators at `tools.allow`, so derive or log the matched rule key instead.
  - confidence: 0.91
- **[P2] Move sandbox audit logging out of formatting:** `src/agents/sandbox/runtime-status.ts:160-162`
  - body: `formatSandboxToolPolicyBlockedMessage` is reached from shared error-rendering paths. Logging there can produce repeated identical INFO audit entries for one blocked tool call, so emit the audit at the enforcement boundary or dedupe it per run/tool.
  - confidence: 0.87
