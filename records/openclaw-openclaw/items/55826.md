## Summary

Keep open: current main has not implemented the requested audit capability, and the current patch still has blocking review issues around rule attribution, duplicate logging, and changelog coverage.

## What This Changes

The PR adds INFO-level tool-policy audit logs in the agent filtering pipeline and sandbox blocked-tool formatter, plus pipeline audit-log tests.

## Best Possible Solution

Expose matched rule metadata from tool-policy evaluation, emit one audit log per policy decision at enforcement boundaries, and record the operator-visible feature in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an allow-labeled pipeline step can include `deny`, and sandbox blocked-tool formatting is reachable from several assistant-error rendering paths.

## Solution Assessment

No. The direction is useful, but the current PR logs broad step labels instead of matched rules and puts sandbox audit logging in shared formatting rather than a single audit boundary.
