## What This Changes

The issue reports that memory-core dreaming and sandbox error guidance can silently rewrite `openclaw.json`, enabling sandbox mode and breaking cron jobs on hosts without Docker.

## Best Possible Solution

Define and enforce an operator-approved config mutation boundary: assistant-visible errors should not suggest protected config writes, CLI-through-exec/plugin/cron writes need explicit provenance or guardrails, and `non-main` should remain supported.
