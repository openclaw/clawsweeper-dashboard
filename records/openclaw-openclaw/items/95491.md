## Summary

Keep open: this is a focused candidate for a real WebChat model-display bug, but the current patch recaches a cleared Default selection as an explicit resolved model and still lacks real WebChat/Gateway behavior proof.

## What This Changes

The PR updates WebChat model switching to cache the server-resolved `sessions.patch` model and also changes the legacy real-behavior-proof field parser regex.

## Best Possible Solution

Preserve `null`/Default cache semantics, reconcile only explicit model selections with `sessions.patch.resolved`, add a Default-clearing regression, and split or test the proof-parser change before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: clearing the dropdown sends `model: null`, the gateway returns the current default in `patched.resolved`, and this patch stores that resolved value as an explicit override instead of preserving the Default marker.

## Solution Assessment

No. Reading `sessions.patch.resolved` is the right direction for explicit model switches, but this implementation is not the best fix until it preserves Default semantics and separates or tests the proof-parser automation change.
