## Summary

Keep open: the PR is focused, but it still does not prove or solve the reported runtime fallback drift because it reconciles the dropdown cache from immediate `sessions.patch` selection metadata, not from actual post-run fallback state.

## What This Changes

The PR updates WebChat model switching so the local model dropdown cache is reconciled from the `sessions.patch` response for explicit model selections while leaving Default as a null override.

## Best Possible Solution

Repair the PR so the dropdown reflects refreshed session/effective runtime state, or an explicit resolved-backend contract, while keeping the broader backend identity work tracked in https://github.com/openclaw/openclaw/issues/51441.
