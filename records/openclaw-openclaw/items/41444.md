## Summary

Keep this open. Current main still only applies the Anthropic beta-header wrapper when `resolveAnthropicBetas()` finds extra-param betas, so the model-header-only OAuth path described here remains source-reproducible, and open PR #41461 already references this issue with closing syntax.

## What This Changes

Ensure Anthropic OAuth tokens receive the required `oauth-2025-04-20` beta header even when Anthropic beta headers are supplied through model-level provider headers instead of `params.context1m`.

## Best Possible Solution

Land PR #41461 or an equivalent Anthropic plugin change that always runs the beta-header merge path for direct Anthropic streams while preserving the existing OAuth 1M fallback behavior.

## Reproduction Assessment

Yes, from source inspection. With no `extraParams.anthropicBeta` or `extraParams.context1m`, `resolveAnthropicBetas()` returns undefined and `wrapAnthropicProviderStream()` skips the wrapper that injects `oauth-2025-04-20`.

## Solution Assessment

Yes. Always applying the Anthropic beta-header wrapper, or equivalently separating OAuth beta injection from extra-param beta resolution, is the narrow maintainable fix because the OAuth merge and unsupported-context stripping already live in that wrapper.
