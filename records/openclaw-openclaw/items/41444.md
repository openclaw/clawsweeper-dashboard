## Summary

Keep open: current main still has a source-reproducible Anthropic OAuth header path where model-level `anthropic-beta` can replace the OAuth-required beta header. The same-repo fix PRs are closed unmerged, the copied fork PR is open but conflicting, and the merged 1M GA work only partially overlaps this exact OAuth/model-header case.

## What This Changes

The issue reports that Anthropic OAuth tokens fail with HTTP 401 when `context-1m-2025-08-07` is configured through model-level headers instead of agent `context1m` params.

## Best Possible Solution

Land or replace the linked fix with a current-main patch that merges OAuth-required Anthropic betas with model-level `anthropic-beta` for direct Anthropic endpoints, while preserving custom endpoint suppression and adding regression coverage for that exact configuration.

## Reproduction Assessment

Yes. Source inspection shows direct Anthropic OAuth requests seed `oauth-2025-04-20`, then merge model-level headers afterward with last-wins semantics, so a model-level `anthropic-beta` can replace the required OAuth beta.

## Solution Assessment

The requested direction is right, but the old unconditional-wrapper PR shape is no longer the best exact implementation path. The best fix should target the active direct transport and plugin wrapper behavior on current main while preserving custom-endpoint beta suppression.
