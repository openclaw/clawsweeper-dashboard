## Summary

Keep open: current main and v2026.6.10 still have a source-reproducible direct Anthropic OAuth path where seeded `oauth-2025-04-20` is merged before model-level `anthropic-beta`, so the reported model header can replace the required OAuth beta; same-repo fix attempts are closed and only an open conflicting fork PR remains.

## What This Changes

The issue asks OpenClaw to preserve the Anthropic OAuth beta header when `context-1m-2025-08-07` is supplied through model-level provider headers.

## Best Possible Solution

Land a current-main fix that composes Anthropic beta headers case-insensitively for direct Anthropic OAuth requests before model or option headers can replace required OAuth betas, while preserving custom-endpoint beta suppression.

## Reproduction Assessment

Yes for source reproduction: configure a direct Anthropic OAuth token with model-level `headers["anthropic-beta"]`; current code merges that header after the OAuth beta seed, so the required `oauth-2025-04-20` value can be replaced.

## Solution Assessment

No for the existing linked branch as-is. The best fix should target current main's direct Anthropic transport and plugin wrapper composition rather than only restoring the older unconditional-wrapper shape.
