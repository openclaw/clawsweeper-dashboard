- `P1`: The issue describes an Anthropic OAuth authentication regression that can turn valid provider requests into HTTP 401 failures.
- `impact:auth-provider`: The affected surface is Anthropic OAuth token handling and provider beta-header construction.

## Summary

Keep open. Current main still builds the direct Anthropic OAuth beta header before merging configured model headers, so a model-level `anthropic-beta` can replace `oauth-2025-04-20`; the linked open PR remains the implementation candidate but needs retargeting or replacement.

## What This Changes

The issue requests preserving Anthropic OAuth-required beta headers when `context-1m-2025-08-07` is configured through model-level headers instead of agent extra params.

## Best Possible Solution

Retarget or replace https://github.com/openclaw/openclaw/pull/41461 with a focused direct Anthropic header-composition fix that merges model-level beta headers without dropping OAuth-required betas while preserving custom-endpoint suppression.

## Reproduction Assessment

Yes. A direct Anthropic model using an `sk-ant-oat` token and model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }` follows a source path where the later model header can overwrite the seeded OAuth beta.

## Solution Assessment

No for the older wrapper-only fix shape. The maintainable fix should target active direct Anthropic header composition and keep documented custom-endpoint beta suppression intact.
