## Summary

Keep this issue open: current main and latest release still allow a model-level `anthropic-beta` header to overwrite the OAuth-required Anthropic beta on the active native transport path, and the open linked PR is still the paired but stale/dirty repair candidate rather than a merged fix.

## What This Changes

The issue requests preserving `oauth-2025-04-20` for Anthropic OAuth tokens when legacy 1M context is configured through model-level headers.

## Best Possible Solution

Fix the active native Anthropic transport to semantically merge `anthropic-beta` values from OpenClaw defaults, model headers, and request headers, strip retired `context-1m-*`, preserve OAuth markers, and keep custom-endpoint suppression covered by tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a direct Anthropic OAuth token builds the required beta header, then a model-level `anthropic-beta` later in `model.headers` can overwrite it through last-wins header merging.

## Solution Assessment

No, the currently linked PR is not the best current fix because it targets the older pi-embedded runner gate. The maintainable fix belongs in the active native Anthropic transport beta-header merge while preserving custom endpoint behavior.
