## Summary

Keep open: current main and the latest v2026.6.8 release still have the source-reproducible Anthropic OAuth header path where model-level `anthropic-beta` can overwrite the required OAuth beta before a wrapper re-adds it. The remaining open implementation candidate is a conflicting fork PR, so this issue should stay canonical for maintainer follow-up.

## What This Changes

The issue reports that Anthropic OAuth tokens can receive HTTP 401 when `context-1m` is configured through model-level headers instead of agent extra params because `oauth-2025-04-20` is not preserved.

## Best Possible Solution

Land a current-main fix in the active Anthropic plugin/direct-transport header composition so direct Anthropic OAuth requests always preserve required OAuth betas when model-level `anthropic-beta` is present, while still suppressing implicit betas for custom Anthropic-compatible endpoints and stripping retired 1M beta config.

## Reproduction Assessment

Yes. Source inspection shows direct Anthropic OAuth token requests can lose `oauth-2025-04-20` when model-level `anthropic-beta` overwrites the seeded header and no wrapper runs to merge it back.

## Solution Assessment

Yes, the requested invariant is the right fix direction. The implementation should target the current Anthropic plugin/direct-transport header composition rather than applying the stale fork branch verbatim.
