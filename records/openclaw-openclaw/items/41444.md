## Summary

Keep open: current main still has a source-reproducible Anthropic OAuth header path where model-level `anthropic-beta` can overwrite the OAuth-required beta header. The merged GA 1M work partially reduced legacy `context1m` beta handling, but it does not prove the exact model-header overwrite case is fixed.

## What This Changes

The issue requests preserving `oauth-2025-04-20` for Anthropic OAuth tokens when `context-1m-2025-08-07` is configured through model-level headers.

## Best Possible Solution

Land a current-main fix in Anthropic header composition that merges OAuth-required betas with model-level `anthropic-beta` for direct Anthropic endpoints while preserving custom endpoint beta suppression.

## Reproduction Assessment

Yes. Source inspection shows model-level headers reach `model.headers`, and the direct Anthropic OAuth branch merges those headers after seeding `oauth-2025-04-20`, so a model-level `anthropic-beta` can replace the required OAuth beta.

## Solution Assessment

No for closing. The best solution is a current-main fix in the active Anthropic direct transport or wrapper composition; stale unconditional-wrapper PRs are useful signals but should be retargeted safely.
