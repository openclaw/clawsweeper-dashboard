## Summary

Keep open. Current main still has the reported Anthropic wrapper gate, and this issue is paired with an open closing PR that should be reviewed, landed, or replaced rather than closing the issue early.

## What This Changes

The issue reports that Anthropic OAuth tokens can fail with 401 when `anthropic-beta` is configured through model/provider headers instead of agent extra params.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/41461 or a narrow replacement in the active Anthropic plugin that preserves OAuth/default beta injection for direct Anthropic requests, keeps custom-endpoint suppression, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; the active wrapper is skipped and later headers can replace the seeded OAuth beta.

## Solution Assessment

Partially. Unconditional OAuth beta merging is the right direction, but the maintainable fix should target `extensions/anthropic/stream-wrappers.ts` and preserve direct-endpoint versus custom-endpoint behavior.
