## Summary

Keep open: current main and the latest release still have a source-reproducible path where direct Anthropic OAuth seeds the required beta header before model-level headers can overwrite it. The open fork PR is a useful signal, but it is dirty, not maintainer-modifiable, and targets the older wrapper shape, so this needs maintainer follow-up rather than cleanup closure.

## What This Changes

The issue reports Anthropic OAuth 401 failures when model-level Anthropic beta headers overwrite the required OAuth beta header.

## Best Possible Solution

Land a targeted current-main fix in Anthropic header composition that preserves OAuth-required betas after model-level header overrides, with regression coverage for the direct Anthropic OAuth model-header case.

## Reproduction Assessment

Yes. Source inspection shows model-level headers can become model.headers, and current direct Anthropic OAuth merges those headers after seeding oauth-2025-04-20, so a model-level anthropic-beta can replace the required OAuth beta.

## Solution Assessment

No for closing. The best fix is a current-main repair in the active Anthropic direct/plugin header-composition path; the stale unconditional-wrapper branches are useful clues but not safe landing paths as-is.
