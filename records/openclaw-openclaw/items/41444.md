Keep open. Current main has partially migrated 1M context handling to GA, but the reported model-level `anthropic-beta` overwrite path is still source-visible in the direct Anthropic OAuth transport, and the open closing PR is still unmerged and conflicting.

## What This Changes

The issue reports Anthropic OAuth 401s when a model-level `anthropic-beta` header for retired 1M context overwrites the required `oauth-2025-04-20` beta.

## Best Possible Solution

Retarget or replace https://github.com/openclaw/openclaw/pull/41461 with a current-main fix that preserves required direct Anthropic OAuth betas, strips or merges retired model-level context betas, and keeps custom Anthropic-compatible endpoint beta suppression intact.

## Solution Assessment

No. The currently linked PR changes an older embedded-runner path, while the current fix should target the active Anthropic plugin/direct transport header composition and preserve documented custom-endpoint suppression.
