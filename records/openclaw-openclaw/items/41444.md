## Summary

Keep open: current main and the latest v2026.6.6 release still have a source-reproducible Anthropic OAuth path where model-level `anthropic-beta` headers can override the required `oauth-2025-04-20` beta. The remaining linked implementation candidate is open in a fork but merge-conflicting, so this issue should stay canonical until maintainers retarget, replace, or close that path.

## What This Changes

The issue asks OpenClaw to preserve Anthropic OAuth beta headers when `context1m` or `anthropic-beta` is configured through model-level headers instead of agent extra params.

## Best Possible Solution

Land a current-main fix in the Anthropic direct/plugin header composition that appends required OAuth betas for direct Anthropic OAuth requests even when model-level headers set `anthropic-beta`, while preserving custom endpoint beta suppression and retired 1M beta stripping.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct Anthropic OAuth token plus model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }` can override the seeded OAuth beta before the request is sent.

## Solution Assessment

No for the old unconditional-wrapper shape by itself. The best fix should target the active Anthropic plugin/direct-transport header merge and keep implicit beta suppression for custom Anthropic-compatible endpoints.
