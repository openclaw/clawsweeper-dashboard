## Summary

Keep open. Current main still composes the Anthropic beta wrapper only when extra params produce betas, so model-level `anthropic-beta` headers can bypass OAuth beta injection; open PR #41461 is already the exact closing implementation candidate.

## What This Changes

Inject or preserve the required Anthropic OAuth beta when `context1m` or `anthropic-beta` is supplied through model-level headers instead of agent extra params.

## Best Possible Solution

Land #41461 or a narrow replacement that preserves OAuth beta handling for model-level Anthropic headers, applies the documented OAuth context fallback, and adds regression coverage for that exact configuration.

## Reproduction Assessment

Yes, from source inspection: use an Anthropic model with empty extra params, an OAuth `sk-ant-oat` key, and a configured model-level `anthropic-beta`; current composition skips the beta wrapper, so OAuth beta injection does not run. I did not execute a live provider call in this read-only pass.

## Solution Assessment

Partially. Applying Anthropic beta handling even without extra-param betas is the right direction, but the final patch should also account for existing model-level `context-1m` headers under OAuth according to the current documented fallback behavior.
