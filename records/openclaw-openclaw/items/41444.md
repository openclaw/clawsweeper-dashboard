## Summary

Keep open. Current main still has the reported Anthropic beta-wrapper gate, and this issue is paired with open closing PR https://github.com/openclaw/openclaw/pull/41461, so it should not be closed before that implementation is reviewed, landed, or replaced.

## What This Changes

Anthropic OAuth requests should preserve the required `oauth-2025-04-20` beta when a user supplies `context-1m` through model/provider headers instead of agent extra params.

## Best Possible Solution

Land the open implementation PR or a narrow replacement in the Anthropic plugin that preserves OAuth/default beta injection for direct Anthropic requests, keeps implicit betas suppressed on custom endpoints, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; the active wrapper is skipped and later headers can replace the seeded OAuth beta.

## Solution Assessment

Partially. Always applying OAuth beta merging is the right direction for direct Anthropic OAuth, but the maintainable fix should target `extensions/anthropic/stream-wrappers.ts` and preserve direct-endpoint versus custom-endpoint behavior.
