## Label Justifications

- `P1`: A supported Anthropic OAuth configuration can consistently fail authentication with HTTP 401.
- `impact:auth-provider`: The defect removes required OAuth provider headers and breaks Anthropic model authentication.

## Summary

Current main and release v2026.6.5 still contain the reported overwrite path: direct Anthropic OAuth defaults are created first, then model-level headers replace them by source order. The issue remains necessary, but its linked PR targets an obsolete runner path and needs retargeting or replacement.

## What This Changes

Fix Anthropic OAuth 401 failures when model-level `anthropic-beta` headers overwrite the required OAuth beta markers.

## Best Possible Solution

In the active direct-Anthropic transport, semantically merge `anthropic-beta` values from defaults, model headers, and request headers, remove retired `context-1m-*` values, preserve required OAuth markers, and add regression coverage for direct OAuth plus model-level legacy beta and custom-endpoint suppression.

## Reproduction Assessment

Yes. On current main, direct Anthropic OAuth builds the required beta header and then `mergeTransportHeaders` lets the reported model-level `anthropic-beta` overwrite it; the existing tests establish each component of that path.

## Solution Assessment

No, the linked PR is no longer the best implementation because it changes a retired runner path. The narrow maintainable repair belongs in the active native Anthropic transport's beta-header merge.
