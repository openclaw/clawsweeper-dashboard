- `impact:auth-provider`: The affected behavior is Anthropic OAuth token handling and provider beta-header construction.

## Summary

Keep open. Current main and v2026.5.19 still have a source-reproducible Anthropic OAuth/model-header beta overwrite path, and the linked PR at https://github.com/openclaw/openclaw/pull/41461 should land or be replaced before this issue closes.

## What This Changes

The issue reports that Anthropic OAuth requests fail with HTTP 401 when a model-level `anthropic-beta` header overwrites or bypasses the required OAuth beta.

## Best Possible Solution

Land a fix that composes Anthropic beta headers in the active plugin/direct transport path, preserving `oauth-2025-04-20` for direct Anthropic OAuth while keeping custom endpoint beta suppression.

## Reproduction Assessment

Yes. Use a direct Anthropic model with an `sk-ant-oat` token and model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }`; source shows the required OAuth beta can be skipped or overwritten before request dispatch.

## Solution Assessment

Yes for the reported direction, but not for the stale linked PR shape. The maintainable fix should target active Anthropic plugin/direct transport header composition and cover both direct Anthropic OAuth and custom endpoint suppression.
