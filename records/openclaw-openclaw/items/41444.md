## Summary

Keep open. Current main still skips the Anthropic beta wrapper when no extra-param beta is configured, and this issue is paired with open closing PR https://github.com/openclaw/openclaw/pull/41461 that should be reviewed, landed, or replaced rather than closing the issue early.

## What This Changes

The issue reports that Anthropic OAuth tokens can lose the required `oauth-2025-04-20` beta when `context-1m` is supplied through model-level headers instead of agent extra params.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/41461 or a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that preserves OAuth/default betas for direct Anthropic OAuth requests, respects custom-endpoint beta suppression, and adds a regression test for model-level `anthropic-beta` with empty extra params.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct Anthropic OAuth with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta` header skips the active wrapper and lets the later model header replace the seeded OAuth beta.

## Solution Assessment

Partially. Unconditional beta merging is the right direction for direct Anthropic OAuth, but the maintainable fix needs to target the active Anthropic plugin and preserve documented custom-endpoint suppression and current OAuth/context policy.
