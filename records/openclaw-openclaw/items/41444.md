## Summary

Keep open. Current main still has the source-backed Anthropic OAuth beta-wrapper gap, and this issue is paired with open closing PR https://github.com/openclaw/openclaw/pull/41461, so it should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue reports that Anthropic OAuth tokens fail with HTTP 401 when `context-1m-2025-08-07` is supplied through model-level headers because the OAuth beta header is not injected unless extra-param betas trigger the wrapper.

## Best Possible Solution

Land the open implementation PR or a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that preserves OAuth/default beta injection for direct Anthropic requests, keeps implicit betas suppressed on custom endpoints, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; the active wrapper is skipped and later header merging can replace the seeded OAuth beta.

## Solution Assessment

Partially. Always applying OAuth beta merging is the right direction for direct Anthropic OAuth, but the maintainable fix should target the active Anthropic plugin wrapper and preserve direct-endpoint versus custom-endpoint behavior.
