## Label Justifications

- `P1`: The issue describes a real Anthropic OAuth authentication regression that can turn valid provider requests into HTTP 401 failures.
- `impact:auth-provider`: The affected behavior is provider auth token handling and Anthropic beta-header construction.

## Summary

Keep open. Current main partially addressed the retired 1M beta path, but a model-level `anthropic-beta` header can still overwrite direct Anthropic OAuth beta headers, and the open closing PR remains unmerged and conflicting.

## What This Changes

Ensure Anthropic OAuth requests keep `oauth-2025-04-20` when a model-level `anthropic-beta` header is configured for legacy 1M context.

## Best Possible Solution

Fix Anthropic header composition so direct Anthropic OAuth-required betas are merged with, or protected from, model-level beta headers while preserving implicit-beta suppression for custom Anthropic-compatible endpoints.

## Reproduction Assessment

Yes. A direct Anthropic model using an `sk-ant-oat` token and model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }` follows current source paths where `model.headers` can overwrite the seeded OAuth beta header.

## Solution Assessment

No. Current main fixed the GA `context1m`/`anthropicBeta` path, but the remaining model-header overwrite path still needs a targeted fix in the active Anthropic plugin or direct transport header composition.
