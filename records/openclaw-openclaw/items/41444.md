- `impact:auth-provider`: The affected behavior is Anthropic OAuth token handling and provider request beta-header construction.

## Summary

Keep open. Current main still has a source-reproducible Anthropic OAuth/model-header beta overwrite path, and the open closing PR at https://github.com/openclaw/openclaw/pull/41461 should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to preserve the required Anthropic OAuth beta header when an OAuth token is used with model-level `anthropic-beta` headers.

## Best Possible Solution

Implement one active Anthropic header-composition path that merges model-level beta values with required OAuth/default betas while preserving custom Anthropic-compatible endpoint suppression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token and a model-level `anthropic-beta`; current main can seed the OAuth beta and then replace it through last-wins model-header merging.

## Solution Assessment

Partially. Preserving OAuth betas is the right end state, but the final fix should target the active Anthropic plugin/direct-transport header composition and avoid changing custom endpoint beta suppression.
