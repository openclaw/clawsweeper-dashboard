## Summary

Keep open: current main and latest release still have a source-reproducible Anthropic OAuth path where model-level `anthropic-beta` headers can replace the seeded OAuth beta header. The linked cross-repo closing PR is open but conflicting and targets the older wrapper-gate shape, so this issue should remain the canonical bug until that path is retargeted, replaced, or closed.

## What This Changes

The issue asks OpenClaw to preserve Anthropic OAuth beta headers when Anthropic 1M context is configured through model-level provider headers.

## Best Possible Solution

Fix the active Anthropic plugin/direct-transport header composition so direct Anthropic OAuth requests preserve `oauth-2025-04-20` while custom endpoint beta suppression remains intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token and model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }`; the wrapper does not run from model headers alone and transport header merging is last-wins.

## Solution Assessment

No, the older unconditional wrapper-gate fix is not the best current path. The fix should be retargeted to the active Anthropic plugin/direct transport header composition and preserve the documented custom-endpoint suppression behavior.
