## Summary

Keep open. Current main still has a source-reproducible Anthropic OAuth/model-header beta overwrite path, and an open closing PR already targets the issue, so closure should wait until that PR lands or is replaced.

## What This Changes

Ensure Anthropic OAuth-required beta headers are preserved when `context-1m` is supplied through model-level headers instead of agent extra params.

## Best Possible Solution

Land or replace the linked fix with one active Anthropic header-merging path that preserves OAuth/default betas, merges model-level betas instead of overwriting them, and keeps custom Anthropic-compatible endpoints free of implicit Anthropic betas.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; current main seeds the OAuth beta before model headers and the last-wins merge can replace it.

## Solution Assessment

Partially. Preserving OAuth betas is the right end state, but the final fix should target the active Anthropic transport/plugin header path and preserve the direct-endpoint guard rather than only patching a stale runner-era path.
