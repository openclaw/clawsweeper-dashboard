## Summary

Keep open. Current main still skips the Anthropic beta-header wrapper when no extra-param beta is configured, and this issue is already paired with an open closing PR that should be reviewed, landed, or replaced rather than closing the issue early.

## What This Changes

The issue asks OpenClaw to preserve Anthropic OAuth beta-header injection when `context-1m-2025-08-07` is configured through model-level headers instead of agent extra params.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/41461 or a narrow replacement in the active Anthropic plugin that preserves OAuth beta injection for direct Anthropic model-header cases, keeps custom-endpoint suppression, applies the documented OAuth context fallback, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with empty extra params, an OAuth `sk-ant-oat` token, and a model-level `anthropic-beta`; the beta wrapper is skipped, so OAuth beta merging does not run before later headers can replace it.

## Solution Assessment

Partially. Always applying the Anthropic beta merge for the direct Anthropic OAuth path is the right direction, but the final patch should target the active plugin/current transport and preserve custom-endpoint suppression plus the documented OAuth `context-1m` fallback.
