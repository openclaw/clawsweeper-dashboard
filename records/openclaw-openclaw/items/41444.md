## Summary

Keep open: current main and latest release still have a source-reproducible direct Anthropic OAuth path where model-level headers can overwrite the OAuth-required beta header, despite docs saying OpenClaw preserves those OAuth betas.

## What This Changes

The issue asks OpenClaw to preserve or inject Anthropic OAuth beta headers when an OAuth token is used and Anthropic beta headers are configured at the model level.

## Best Possible Solution

Land a current-main fix that preserves OAuth-required Anthropic betas after model-level headers for direct Anthropic OAuth requests, while keeping custom endpoint beta suppression intact.

## Reproduction Assessment

Yes. Source inspection shows model-level headers survive catalog normalization, the direct Anthropic OAuth path seeds oauth-2025-04-20 before model.headers, and the shared merge helper lets later headers overwrite earlier ones.

## Solution Assessment

No for closing. The active source still needs a fix in the direct Anthropic header-composition path; older unmerged wrapper PRs and the GA 1M migration do not fully solve the reported behavior.
