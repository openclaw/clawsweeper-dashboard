## Summary

Keep open. Current main still source-reproduces the Anthropic OAuth beta/header gap, and the issue has an open closing PR, so closure should wait until that PR lands or is replaced.

## What This Changes

The issue reports that Anthropic OAuth requests lose the required `oauth-2025-04-20` beta when a 1M context beta is supplied through model-level headers instead of agent params.

## Best Possible Solution

Review, land, or replace the linked PR with a narrow fix in `extensions/anthropic/stream-wrappers.ts` that preserves OAuth/default betas for direct Anthropic model-header requests while keeping custom endpoints from receiving implicit betas.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; the wrapper is skipped and model headers can replace the seeded OAuth beta before the request is sent.

## Solution Assessment

Partially. Always running OAuth/default beta merging is the right direction, but the maintainable fix should target the active Anthropic plugin wrapper and preserve the direct-endpoint guard for custom Anthropic-compatible providers.
