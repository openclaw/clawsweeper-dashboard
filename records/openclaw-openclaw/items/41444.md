## Summary

Keep open. Current main still has a source-reproducible Anthropic OAuth/model-header beta overwrite path, and an open closing PR already targets this issue, so closure should wait until that implementation lands or is replaced.

## What This Changes

The issue reports that Anthropic OAuth tokens fail with HTTP 401 when `context-1m-2025-08-07` is configured through model-level headers because the required OAuth beta header is not preserved.

## Best Possible Solution

The end state should be one active Anthropic header-merging path that preserves OAuth/default betas for direct Anthropic requests, merges model-level betas without overwrite, and keeps custom Anthropic-compatible endpoints free of implicit Anthropic betas.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; current main skips the wrapper and the transport merge can replace the seeded OAuth beta.

## Solution Assessment

Partially. Preserving OAuth betas is the right direction, but the maintainable fix should target the active Anthropic plugin/transport header path and preserve the direct-endpoint guard for custom Anthropic-compatible providers.
