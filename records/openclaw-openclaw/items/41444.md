## Summary

Keep open: current main still has a source-reproducible Anthropic OAuth header path where a model-level `anthropic-beta` can replace the OAuth-required beta header. The GA 1M migration reduced the old `context1m` beta path, but it does not prove this model-header overwrite case is fixed.

## What This Changes

The issue asks OpenClaw to preserve Anthropic OAuth beta headers when a model-level `anthropic-beta` header is configured instead of agent `context1m` params.

## Best Possible Solution

Land a current-main fix in Anthropic header composition that merges OAuth-required betas with model-level `anthropic-beta` for direct Anthropic endpoints while preserving custom endpoint beta suppression.

## Reproduction Assessment

Yes. Source inspection shows model-level headers reach `model.headers`, and the direct Anthropic OAuth branch merges those headers after seeding `oauth-2025-04-20`, so a model-level `anthropic-beta` can replace the required OAuth beta.

## Solution Assessment

Yes for the requested direction, but the fix should target the active Anthropic direct/plugin header composition rather than only reviving an older unconditional-wrapper patch.
