## Summary

Keep open: current main and v2026.6.8 still have a source-reproducible path where direct Anthropic OAuth seeds the required beta header before model-level headers can overwrite it. The linked open fork PR is a candidate signal, but it is conflicting and appears to target an older wrapper shape, so this should stay open for maintainer follow-up.

## What This Changes

The issue reports that Anthropic OAuth bearer requests can lose the required oauth-2025-04-20 beta header when a 1M-context beta is configured through model-level headers instead of agent params.

## Best Possible Solution

Land a current-main fix in Anthropic direct/plugin header composition that preserves OAuth-required betas after model-level header overrides, with regression coverage for the direct Anthropic OAuth model-header case.

## Reproduction Assessment

Yes. Source inspection shows model-level headers reach model.headers, and the direct Anthropic OAuth branch merges those headers after seeding oauth-2025-04-20, so a model-level anthropic-beta can replace the required OAuth beta.

## Solution Assessment

No for closing. The best solution is a current-main fix in the active Anthropic direct transport or wrapper composition; the stale unconditional-wrapper branch is only a clue and must be retargeted safely.
