## Summary

Keep open. Current main still source-reproduces the Anthropic OAuth beta/header overwrite path, and an open closing PR already targets this issue, so closure should wait until that PR lands or is replaced.

## What This Changes

The issue reports that Anthropic OAuth tokens fail with HTTP 401 when `context-1m-2025-08-07` is configured through model-level headers instead of Anthropic extra params.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/41461 with a narrow fix in the active Anthropic plugin/transport header path that preserves OAuth/default betas for direct Anthropic model-header requests while keeping custom endpoints from receiving implicit betas.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, empty extra params, and a model-level `anthropic-beta`; current main skips the wrapper and allows model headers to replace the seeded OAuth beta.

## Solution Assessment

Partially. Preserving OAuth betas is the right direction, but the maintainable fix should target the active Anthropic plugin/transport header path and preserve the direct-endpoint guard for custom Anthropic-compatible providers.
