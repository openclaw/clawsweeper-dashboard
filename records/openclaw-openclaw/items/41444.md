## Summary

Keep open. Current main has the GA 1M migration, but the reported direct Anthropic OAuth path can still lose `oauth-2025-04-20` when a model-level `anthropic-beta` header is merged after the seeded OAuth beta, and the open closing PR remains unresolved.

## What This Changes

The issue reports an Anthropic OAuth 401 regression when model-level `anthropic-beta` headers override the required OAuth beta header.

## Best Possible Solution

Retarget or replace https://github.com/openclaw/openclaw/pull/41461 with a focused Anthropic header-composition fix that merges model-level beta headers without dropping OAuth-required betas while preserving custom-endpoint suppression.

## Reproduction Assessment

Yes. A direct Anthropic model using an `sk-ant-oat` token and model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }` still follows a source path where a later model header can overwrite the seeded OAuth beta.

## Solution Assessment

No for the older wrapper-only fix shape. The maintainable fix should target active direct Anthropic header composition and keep documented custom-endpoint beta suppression intact.
