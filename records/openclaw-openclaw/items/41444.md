## Summary

Keep open. Current main still has a source-reproducible Anthropic OAuth/model-header beta overwrite path, and the open closing PR at https://github.com/openclaw/openclaw/pull/41461 should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue reports that Anthropic OAuth bearer requests can lose the required `oauth-2025-04-20` beta when `context-1m-2025-08-07` is supplied through model-level headers.

## Best Possible Solution

Land or replace the linked fix with one active Anthropic header-composition path that merges model-level beta values with required OAuth/default betas while preserving custom-endpoint beta suppression and the documented OAuth/context1m policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token and a model-level `anthropic-beta`; current main can seed the OAuth beta and then replace it through the last-wins model-header merge.

## Solution Assessment

Partially. Preserving OAuth betas is the right end state, but the final fix should target the active Anthropic plugin/direct-transport header path and merge beta values without breaking custom Anthropic-compatible endpoints.
