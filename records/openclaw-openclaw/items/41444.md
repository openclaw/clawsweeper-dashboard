## Summary

Keep open: current main and v2026.6.8 still allow model-level Anthropic headers to override the seeded OAuth beta header in the direct Anthropic OAuth path. An open fork PR references this issue with a proposed fix, but it is merge-conflicting and targets the older wrapper shape, so maintainers need to retarget or replace that path rather than close the report.

## What This Changes

Ensure Anthropic OAuth/subscription tokens keep the required `oauth-2025-04-20` beta when model-level headers configure the retired `context-1m` beta.

## Best Possible Solution

Land a current-main fix in Anthropic direct/plugin header composition that preserves OAuth-required betas after model-level header overrides, with regression coverage for the direct Anthropic OAuth model-header case.

## Reproduction Assessment

Yes. Source inspection shows model-level headers reach `model.headers`, and the direct Anthropic OAuth branch merges those headers after the seeded OAuth beta, so a model-level `anthropic-beta` can replace `oauth-2025-04-20`.

## Solution Assessment

No for the stale unconditional-wrapper branch by itself. The best fix should target the active direct Anthropic transport/header merge while preserving custom endpoint beta suppression and retired 1M beta stripping.
