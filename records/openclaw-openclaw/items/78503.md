## Summary

Keep open: this is a focused, source-backed fix for a slow default `status --json` path, but it intentionally changes the default JSON diagnostics contract and should land only after maintainer compatibility acceptance and a green or rerun broad CI check.

## What This Changes

The PR gates update checks, task summaries, memory inspection, local status-RPC fallback, and the gateway probe budget behind the fast JSON status policy, with deeper diagnostics preserved for `status --json --all`.

## Best Possible Solution

Land the focused gating if maintainers accept default JSON as the lean health-probe contract, with full diagnostics available through `status --json --all`.

## Reproduction Assessment

Yes, source-reproducible: current main still starts update checks, task registry summaries, and possible local status-RPC fallback on the default JSON path outside cold-start skipping. I did not run the Pi-class timing scenario in this read-only review.

## Solution Assessment

Mostly yes: using the existing `--all` boundary is the narrow maintainable fix for making default JSON status usable as a health probe. The main open question is whether maintainers accept the compatibility shift for callers that currently expect default JSON diagnostics.
