Keep this PR open: the patch is a focused status CLI performance fix with sufficient terminal proof, and the remaining blocker is maintainer acceptance of the default JSON compatibility tradeoff rather than an obvious code defect.

## What This Changes

The PR makes default `status --json` skip update, task, memory, and local status-RPC fallback diagnostics unless `--all` is set, caps the default gateway probe, and extracts a cron timeout helper/test.

## Best Possible Solution

Land the focused gating if maintainers accept default `status --json` as the lean health-probe contract, with full diagnostics remaining available through `status --json --all`.

## Reproduction Assessment

Yes for the source-level path: current main still runs update checks, task summary inspection, and local status-RPC fallback work on default JSON outside cold-start skipping. I did not rerun the Pi-class timing repro in this read-only review.

## Solution Assessment

Yes, mostly: using the existing `--all` boundary is the narrow maintainable way to make default JSON status usable as a health probe. The compatibility alternative is an explicit lean mode, but that would not fix the reported default command latency.
