## Summary

Keep open for maintainer review: the patch is focused and appears correct, but it intentionally changes the default `status --json` contract by moving heavier diagnostics behind `--all`.

## What This Changes

The PR changes the status JSON scan so default `status --json` skips update, task, memory, and local status-RPC diagnostics, caps the gateway probe, and keeps deeper diagnostics under `--all`.

## Best Possible Solution

Land the focused gating if maintainers accept default JSON as the lean health-probe contract, with full diagnostics remaining available through `status --json --all`.

## Reproduction Assessment

Yes, source-reproducible: current main still starts update checks, task registry summaries, and possible local status-RPC fallback on the default JSON path outside cold-start skipping. I did not run a Pi-class timing repro in this read-only review.

## Solution Assessment

Mostly yes: using the existing `--all` boundary is the narrow maintainable fix for making default JSON status usable as a health probe. The safer compatibility alternative would be a separate lean flag, but that would not fix the reported default command latency.
