## What This Changes

This PR changes shared auto-reply mention pattern derivation to use Unicode-aware boundaries for CJK identity names and adds regression tests for single-character CJK mentions.

## Best Possible Solution

Keep the shared-helper approach, but fix the regression tests to use `agents.list` plus an agent id and add after-fix proof that runs OpenClaw's helper or a real channel mention path.

## Reproduction Assessment

Yes for the regex root cause: current main's `\b@?包\b` does not match `@包 你好` or `包 你好`, while the proposed boundary shape does. The submitted tests still need correction because they do not reproduce through OpenClaw's actual agent config path.

## Solution Assessment

No as submitted. The production fix is a reasonable shared-layer solution, but the regression tests and proof need to exercise `buildMentionRegexes` with a real `agents.list` entry and agent id before this is the best mergeable fix.
