## What This Changes

The PR changes core derived mention regexes so Han/CJK identity names use explicit Unicode-aware boundaries, with regression tests for one- and two-character Chinese names.

## Best Possible Solution

Land the shared helper fix after normal checks if maintainers accept the intended group-trigger expansion for Han identity names.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the right layer: docs say identity.name derives mentionPatterns, and multiple channels consume the shared buildMentionRegexes helper. Channel-specific parsers would be narrower and would miss SDK users of the same invariant.
