## Best Possible Solution

## Solution Assessment

Yes. Fixing `deriveMentionPatterns` is the right owner boundary because `buildMentionRegexes` is shared through core/channel callers and the plugin SDK; a DingTalk-only parser fix would leave sibling consumers with the same boundary bug.
