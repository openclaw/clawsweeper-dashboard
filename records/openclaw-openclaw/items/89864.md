## What This Changes

The PR changes shared mention pattern derivation to use Unicode-aware boundaries for CJK identity names and adds CJK mention regression tests.

## Best Possible Solution

Land the shared boundary fix after normal CI and maintainer review, keeping mention parsing centralized in the core helper while channel-native explicit mentions stay unchanged.

## Reproduction Assessment

Yes. Current main derives `\b@?包\b`, and a direct Node check of that current pattern does not match `@包 你好` or `包 你好`, which matches the linked report's core failure.

## Solution Assessment

Yes. The shared `deriveMentionPatterns` helper is the right owner boundary because it feeds `buildMentionRegexes` and `stripMentions` across channel gates; channel-specific fixes would duplicate policy and miss other providers using identity-derived mention patterns.
