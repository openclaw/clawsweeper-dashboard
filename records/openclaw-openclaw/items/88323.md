Full review comments:

- **[P1] Only cache generated media after visible delivery:** `src/infra/outbound/outbound-send-service.ts:409-412`
  - body: `sendCoreMessage` can return normally with `deliveryStatus: "suppressed"` and no platform result when durable delivery produced no visible output. This unconditional cache write records the artifact as delivered after that non-visible outcome, so the next real retry of the same artifact will be suppressed before it uploads.
  - confidence: 0.91
- **[P1] Wait for final delivery before caching the artifact:** `src/auto-reply/reply/dispatch-from-config.ts:2013-2017`
  - body: `sendFinalReply` only enqueues the final payload; the dispatcher can still cancel it or record an async delivery failure. Marking the artifact delivered before `waitForIdle()` and outcome checks means a failed final send can poison the cache and suppress the retry that should have delivered the media.
  - confidence: 0.88
