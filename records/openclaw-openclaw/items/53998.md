Keep open: current main still has source-level false-positive and fatal apply paths, and #54141 is an open closing PR, so this should wait for that PR or a replacement instead of being closed.

## What This Changes

The issue asks for `secrets audit` to stop treating env-var/SecretRef references as plaintext and for `secrets configure/apply` to scrub migrations while handling OAuth-profile mismatches non-fatally.

## Best Possible Solution

Keep this open until #54141 is reviewed and either landed, replaced, or closed, with any remaining configure/scrub/rotation work split into focused follow-ups.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: bare `$OPENAI_API_KEY` does not parse as a SecretRef, inline auth-profile env templates can fall through to `PLAINTEXT_FOUND`, and mixed auth-profile types throw during apply.

Unclear for the full issue. #54141 looks like the narrowest fix for the primary audit false positives, but the mixed-profile apply behavior, scrub semantics, and rotation warning still need maintainer scoping.
