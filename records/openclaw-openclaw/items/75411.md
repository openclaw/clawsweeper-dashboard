## Summary

Keep open: current main still has the reported routing gap. The guarded OpenClaw Responses transport handles terminal 429s, but embedded runs with a custom session stream still skip that fallback, and the GitHub Copilot plugin only wraps whichever stream was already selected.

## What This Changes

Ensure GitHub Copilot openai-responses embedded runs fail fast on terminal HTTP 429s even when the recorded stream strategy is session-custom.

## Best Possible Solution

Route every embedded GitHub Copilot openai-responses turn through buildGuardedModelFetch or an equivalent provider-owned guarded transport before idle-timeout handling, while preserving Copilot header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides live v2026.4.29 upstream-429 and session-custom idle-timeout telemetry, and current main plus the locked dependency source still show the same bypass path statically.

## Solution Assessment

Yes. The requested invariant is the right target; the safest repair is a narrow embedded/provider stream-resolution fix with regression tests, not a broad idle-timeout policy change or a Copilot-specific core special case unless the plugin seam proves insufficient.
