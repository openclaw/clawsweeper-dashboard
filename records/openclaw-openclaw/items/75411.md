## Summary

Keep open: current main still has the reported routing gap, because the embedded runner only switches to the guarded boundary-aware Responses transport for an undefined or referentially equal `streamSimple` base stream, while the Pi session default is an async wrapper around `streamSimple`.

## What This Changes

Ensure GitHub Copilot `openai-responses` HTTP 429s fail fast in embedded `session-custom` streams instead of waiting for the idle watchdog.

## Best Possible Solution

Route every embedded GitHub Copilot `openai-responses` turn through `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before idle-timeout handling, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides a concrete live `v2026.4.29` reproduction with immediate upstream 429 and `session-custom` idle-timeout telemetry, and current main plus the locked Pi dependency still show the same bypass route.

## Solution Assessment

Yes. The requested invariant is the right target; the safest implementation is a narrow generic embedded/provider stream-resolution fix with regression tests, not a broad retry-policy change or a hard-coded Copilot branch in core unless the plugin seam proves insufficient.
