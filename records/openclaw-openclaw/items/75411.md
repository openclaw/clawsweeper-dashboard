## Summary

Keep open: current main still has the reported routing gap for embedded Copilot Responses calls, because the guarded OpenClaw transport is only selected for an undefined or referentially equal `streamSimple` base stream while the Pi session default is an async wrapper around `streamSimple`.

## What This Changes

Ensure GitHub Copilot `openai-responses` quota 429s fail fast instead of being misreported as embedded-runner idle timeouts on the session-custom stream path.

## Best Possible Solution

Route embedded `openai-responses` turns that are using the Pi default session stream through OpenClaw's guarded transport, or inject equivalent guarded fetch behavior, while preserving provider `wrapStreamFn` headers and payload hooks.

## Reproduction Assessment

Yes. The issue provides a concrete live `v2026.4.29` reproduction, and current code plus the locked Pi dependency reproduce the routing condition that turns the default stream into `session-custom`.

## Solution Assessment

Yes. The requested invariant is the right target; the maintainable fix is a narrow generic embedded/provider stream-resolution change with regression coverage, not a broad retry-policy change or a Copilot-only core special case unless the plugin seam proves insufficient.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75411. The observable bug is that embedded GitHub Copilot `github-copilot/gpt-5.4` using `openai-responses` can receive an upstream HTTP 429 quickly but OpenClaw records `streamStrategy: "session-custom"` and waits for the idle watchdog because the Pi default session stream is an async wrapper around `streamSimple`, so `resolveEmbeddedAgentStreamFn` bypasses `createBoundaryAwareStreamFnForModel` and `buildGuardedModelFetch`. Implement a narrow generic fix in the embedded/provider stream-resolution path so default Pi session streams for supported OpenAI-family APIs use OpenClaw's guarded transport or equivalent guarded fetch behavior before idle-timeout handling. Preserve explicit custom stream behavior, Copilot `wrapStreamFn` header/payload rewriting, runtime auth injection, abort signal forwarding, and non-OpenAI transports. Do not add a broad retry-policy rewrite or hard-code Copilot in core unless a plugin seam is proven insufficient. Add regression tests for a Pi-style async `streamSimple` wrapper resolving to the guarded boundary-aware path and still allowing provider wrappers.
