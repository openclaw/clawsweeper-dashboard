## Summary

Keep open: current main still has the routing gap reported here. The guarded Responses fetch exists and marks terminal 429 responses as non-retryable, but embedded runs still keep a non-`streamSimple` base stream on the `session-custom` path, and the GitHub Copilot plugin contributes a wrapper rather than a provider-owned stream factory that would force the guarded transport.

## What This Changes

The issue asks OpenClaw to make GitHub Copilot `openai-responses` HTTP 429 quota failures fail fast in embedded `session-custom` streams instead of waiting for the idle watchdog.

## Best Possible Solution

Add a narrow generic embedded/provider stream fix so Copilot and other `openai-responses` embedded runs reach `buildGuardedModelFetch` or an equivalent guarded Responses client before the idle watchdog, while keeping provider wrapper behavior intact and covering the `session-custom` regression with focused tests.

## Reproduction Assessment

Yes. The issue provides live v2026.4.29 telemetry and upstream 429 proof, and current main still has the source-level route that labels and resolves custom embedded streams as `session-custom` instead of selecting the guarded Responses transport.

## Solution Assessment

Yes for the requested invariant. The safest implementation is a narrow stream-resolution or provider-stream registration change for Responses transports, with tests proving Copilot-style wrappers are preserved, rather than a broad retry-policy change or a Copilot-only core branch.

## Repair Work Prompt

Fix openclaw/openclaw issue https://github.com/openclaw/openclaw/issues/75411. The observable bug is that GitHub Copilot `github-copilot/gpt-5.4` using `openai-responses` can receive an immediate upstream HTTP 429, but embedded runner traffic with `streamStrategy: "session-custom"` bypasses `buildGuardedModelFetch` and waits for the LLM idle timeout. Current `src/agents/pi-embedded-runner/stream-resolution.ts` only swaps in `createBoundaryAwareStreamFnForModel` when the saved base stream is undefined or exactly `streamSimple`; `src/agents/provider-stream.ts` also does not return a provider stream for Copilot because the plugin has `wrapStreamFn`, not `createStreamFn`, and there are no proxy/TLS transport overrides. Implement a narrow generic fix so `openai-responses`/`openai-codex-responses`/`azure-openai-responses` embedded runs using the session default custom/pi stream still use `buildGuardedModelFetch` or an equivalent guarded Responses transport before the idle watchdog, while preserving provider `wrapStreamFn` header/payload behavior. Avoid hardcoding Copilot defaults in core unless a generic seam is insufficient, and do not reroute legitimate third-party custom `createStreamFn` transports unexpectedly. Add regression coverage for a Copilot-like wrapper plus custom session stream proving 429/errors surface through the guarded path instead of `session-custom` idle timeout behavior. Add a single-line `CHANGELOG.md` fix entry crediting @zzl360.
