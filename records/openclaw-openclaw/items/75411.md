## Summary

Keep open: current main still has the reported routing gap. The guarded OpenAI Responses transport handles terminal 429s when selected, but embedded runs with a non-`streamSimple` session stream are still classified as `session-custom`, and Copilot's provider hook wraps that existing stream rather than creating the guarded Responses stream.

## What This Changes

Fix Copilot `openai-responses` embedded runs so terminal HTTP 429 quota errors fail fast even when the session starts from a custom stream path.

## Best Possible Solution

Keep this open and land a narrow generic fix at the embedded/provider stream boundary so every Copilot `openai-responses` embedded run uses the guarded Responses fetch or an equivalent provider-owned transport before the idle watchdog, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides a live v2026.4.29 reproduction with immediate upstream HTTP 429 plus `session-custom` timeout telemetry, and current main still has the exact source path that leaves custom session streams outside the guarded Responses transport.

## Solution Assessment

Yes. The requested invariant is the right direction; the maintainable fix should be generic to `openai-responses` transport selection or embedded stream resolution rather than a broad retry-policy rewrite or a Copilot-only core special case unless a generic seam proves insufficient.

## Repair Work Prompt

Fix the Copilot 429 session-custom bypass reported in openclaw/openclaw#75411. Current main only selects `createBoundaryAwareStreamFnForModel` when the embedded session stream is undefined or exactly `streamSimple`, so a Pi/session custom stream remains `session-custom`; the GitHub Copilot plugin then only wraps that stream and never gains `buildGuardedModelFetch`. Make a narrow generic fix so `openai-responses` embedded runs that need boundary-aware transport use the guarded Responses fetch or equivalent even when the initial stream is a session-custom/default Pi closure. Preserve Copilot's header/payload wrapper behavior, avoid hard-coding provider IDs in core unless a generic seam is not viable, and do not broaden unrelated retry policy. Add focused regression coverage for a custom current stream with `github-copilot/gpt-5.4`/`openai-responses` and for the guarded 429 invariant. Add a user-facing changelog fix entry under the active Fixes section with `Fixes #75411` and `Thanks @zzl360`.
