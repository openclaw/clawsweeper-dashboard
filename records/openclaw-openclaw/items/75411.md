## Summary

Keep open. Current `main` still has the routing gap from the report: the guarded Responses fetch exists, but embedded runs only choose the boundary-aware Responses stream when the session stream is undefined or exactly `streamSimple`, so a custom session stream can remain on `session-custom` and miss the Copilot 429 fail-fast path.

## What This Changes

Ensure GitHub Copilot `openai-responses` embedded runs fail fast on upstream HTTP 429 instead of timing out when routed through the session-custom stream path.

## Best Possible Solution

Fix the invariant at the embedded/provider stream boundary: Copilot `openai-responses` turns should use the guarded Responses transport, or an equivalent provider-owned guarded fetch, before the idle watchdog while preserving Copilot's header/payload wrapper and real custom stream behavior.

## Reproduction Assessment

Yes. The issue provides v2026.4.29 telemetry with immediate upstream HTTP 429 plus `streamStrategy: "session-custom"` and idle timeout, and current `main` still contains the source and test paths that preserve that bypass.

## Solution Assessment

Yes. The requested invariant is the right direction; the narrowest maintainable fix is a generic Responses transport-selection change or provider-registration seam, not a Copilot-only retry-policy branch.
