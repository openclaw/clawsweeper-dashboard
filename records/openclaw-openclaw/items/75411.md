## Summary

Keep open: current main still has the reported embedded-runner routing gap. The guarded 429 fetch exists, but Copilot `openai-responses` can still keep the PI session-created stream closure as `session-custom`, so that path does not necessarily reach OpenClaw's guarded Responses transport.

## What This Changes

The issue asks OpenClaw to fail fast on Copilot `openai-responses` HTTP 429s even when embedded runs use the session-created custom stream path.

## Best Possible Solution

Embedded OpenAI Responses turns should compose the OpenClaw guarded transport, or an equivalent guarded provider-owned stream, before idle-timeout handling while preserving Copilot header and payload rewrites.

## Reproduction Assessment

Yes. The issue provides live release telemetry, and current main plus the locked PI 0.71.1 source show a deterministic path where the session-created default stream closure is classified as `session-custom` and returned before the guarded Responses transport is selected.

## Solution Assessment

Yes. A narrow stream-resolution or provider-stream composition fix is the maintainable direction; retry-policy or idle-timeout changes alone would leave Responses traffic split across guarded and unguarded transports.
