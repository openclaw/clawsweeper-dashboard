## Summary

Keep open: current main still has the reported embedded-runner routing gap. The guarded OpenAI Responses transport handles terminal 429s when selected, but the session-custom path is still preserved for non-`streamSimple` session streams, which matches the live `v2026.4.29` telemetry in the report.

## What This Changes

The issue asks OpenClaw to route GitHub Copilot `openai-responses` embedded runs through guarded 429 handling instead of timing out as `session-custom`.

## Best Possible Solution

Make the embedded/provider stream boundary guarantee that Copilot `openai-responses` runs reach `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before the idle watchdog, while preserving the Copilot plugin's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides a concrete live `v2026.4.29` reproduction with immediate upstream 429 plus `session-custom` timeout telemetry, and current main has a deterministic code path and test that preserve non-`streamSimple` session streams as custom.

## Solution Assessment

Yes. The requested invariant is the right target; the safest fix is a narrow generic embedded stream-resolution or provider-stream registration change for Responses transports, not a broad retry-policy change or a core branch hard-coded to Copilot.
