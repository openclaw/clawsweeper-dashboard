## Summary

Keep open: current main still preserves the reported Copilot 429 routing gap because a non-simple cached session stream remains classified as `session-custom` and is returned unchanged before the guarded OpenAI Responses transport can be selected.

## What This Changes

The issue asks for GitHub Copilot `openai-responses` HTTP 429s on embedded session-custom streams to fail fast through the guarded transport instead of waiting for the idle watchdog.

## Best Possible Solution

Embedded OpenAI Responses turns should reach `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before idle-timeout handling, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides live release telemetry for the user-visible failure, and current main still has the deterministic stream-resolution path that classifies non-simple cached session streams as `session-custom`.

## Solution Assessment

Yes. The right fix is a narrow stream-resolution or provider-stream composition change that preserves plugin wrappers while routing OpenAI Responses traffic through the guarded transport; broad retry-policy changes or Copilot-only timeout handling would be less maintainable.
