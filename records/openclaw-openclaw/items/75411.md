## Summary

Keep open: current main still has the reported Copilot 429 routing gap for embedded OpenAI Responses sessions whose cached base stream is neither undefined nor `streamSimple`, so the guarded Responses transport is not guaranteed to run.

## What This Changes

The issue asks OpenClaw to apply the Copilot 429 fail-fast guarded Responses transport to embedded `session-custom` OpenAI Responses streams so quota errors surface immediately instead of idle-timing out.

## Best Possible Solution

Embedded OpenAI Responses calls should reach `buildGuardedModelFetch` or an equivalent guarded provider-owned transport while composing Copilot's wrapper behavior unchanged.

## Reproduction Assessment

Yes, as a source-level reproduction: current main deterministically keeps non-simple cached embedded session streams on the `session-custom` path, and the issue provides live release telemetry for the user-visible Copilot 429 idle timeout.

## Solution Assessment

Yes. A narrow stream-resolution or provider-stream composition fix that routes OpenAI Responses traffic through the guarded transport while preserving provider wrappers is the maintainable fix; Copilot-only timeout handling would be a workaround.
