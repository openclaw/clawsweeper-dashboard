## Summary

Keep open: current main still has the reported routing gap, because the guarded OpenAI Responses transport exists but is only selected for undefined or `streamSimple` base streams, while the PI embedded session default is a custom closure.

## What This Changes

The issue asks OpenClaw to fail fast on GitHub Copilot OpenAI Responses HTTP 429s even when embedded runs use the `session-custom` stream path.

## Best Possible Solution

Embedded OpenAI Responses turns should reach `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before idle-timeout handling, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides live release telemetry, and current main plus the PI 0.71.1 package source show a deterministic path where the default PI session stream is classified as `session-custom` and returned before the guarded OpenAI Responses transport is selected.

## Solution Assessment

Yes. A narrow stream-resolution or provider-stream composition fix is the best direction; retry-policy or idle-timeout tweaks alone would mask the provider error and leave Responses traffic split across inconsistent transports.
