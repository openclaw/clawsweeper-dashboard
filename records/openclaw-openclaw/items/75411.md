## Summary

Keep open: current main still has the reported routing gap. The terminal-429 guard and signal forwarding exist, but embedded runs still classify non-`streamSimple` session base streams as `session-custom`, and the PI default session stream is a custom closure that does not use OpenClaw's guarded Responses fetch.

## What This Changes

The issue reports that Copilot `openai-responses` 429 quota errors bypass the guarded Responses transport on embedded session-custom streams and surface as idle timeouts.

## Best Possible Solution

Embedded OpenAI Responses turns should reach `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before idle-timeout handling, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue provides live release telemetry, and current main plus the PI 0.71.1 package source still show a deterministic path where the default PI session stream is classified as `session-custom` and returned before the guarded OpenAI Responses transport is selected.

## Solution Assessment

Yes. A narrow stream-resolution or provider-stream composition fix is the best direction; retry-policy or idle-timeout tweaks alone would mask the provider error and leave Responses traffic split across inconsistent transports.
