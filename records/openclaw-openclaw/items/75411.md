## Summary

Keep open: current main still has the routing gap described in the report. The guarded Responses fetch exists and handles terminal 429s when selected, but embedded stream resolution still returns a custom session stream whenever the saved base stream is neither undefined nor `streamSimple`, while Copilot contributes a wrapper rather than a provider-owned `createStreamFn`.

## What This Changes

The issue asks OpenClaw to make Copilot `openai-responses` HTTP 429 quota failures fail fast even when embedded runs use the session-custom stream path.

## Best Possible Solution

Fix the generic embedded/provider stream boundary so every `openai-responses` Copilot run reaches `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before the idle watchdog, while preserving Copilot's header and payload wrapper behavior.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 live reproduction with immediate upstream 429 and `session-custom` timeout telemetry, and current main still has the same source-level route that lets a custom embedded stream skip the boundary-aware Responses transport.

## Solution Assessment

Yes. The suggested invariant is the right end state; the safest repair is a narrow generic change in embedded stream resolution or provider stream registration for Responses transports, not a broad retry-policy change or a Copilot-only core branch unless tests prove the generic seam is insufficient.
