## Summary

Keep open: current main still has the reported routing gap. The guarded OpenAI Responses transport exists, but embedded runs only choose it when the cached session stream is undefined or exactly `streamSimple`; a session-custom stream still bypasses that guard and can leave Copilot 429s to the idle watchdog path.

## What This Changes

The issue asks OpenClaw to fail fast on embedded GitHub Copilot `openai-responses` HTTP 429s instead of bypassing the guarded transport and reporting an idle timeout.

## Best Possible Solution

Compose Copilot's stream wrapper around OpenClaw's guarded boundary-aware Responses transport for embedded `openai-responses` turns, with regression coverage for non-`streamSimple` session streams.

## Reproduction Assessment

Yes. The issue provides live `v2026.4.29` telemetry for the user-visible failure, and current main still has the source path that classifies non-simple cached session streams as `session-custom`.

## Solution Assessment

Yes. The maintainable direction is a narrow generic embedded/provider stream-resolution fix that preserves provider wrapper behavior while guaranteeing the guarded Responses fetch path, rather than a broad retry-policy change.

## Repair Work Prompt

Fix the current GitHub Copilot 429 embedded-runner regression. Current `github-copilot/gpt-5.4` `openai-responses` turns can keep `streamStrategy: "session-custom"` and bypass `buildGuardedModelFetch`, so upstream 429 quota errors become 120s idle timeouts. Keep the fix generic at the embedded/provider stream seam: ensure Copilot's `wrapStreamFn` headers and payload rewriting compose around OpenClaw's guarded boundary-aware Responses transport, or otherwise guarantee every embedded Copilot `openai-responses` turn uses equivalent guarded fetch behavior before idle-timeout handling. Do not broaden SDK retry policy globally, do not hard-code only the Copilot weekly-quota body unless a narrow test proves it is required, and preserve third-party provider/plugin contracts. Add regression coverage for a non-`streamSimple` cached session stream plus Copilot `wrapStreamFn` still selecting guarded Responses behavior and surfacing 429 as a provider/model error rather than `idleTimedOut`. Add a single-line user-facing `CHANGELOG.md` fix entry for the active version without bot credit.
