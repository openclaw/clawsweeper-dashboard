## Summary

Keep open: current main still has the reported routing gap for embedded GitHub Copilot Responses runs when the session base stream is a Pi-provided wrapper rather than `streamSimple`, so the guarded 429 transport can still be bypassed.

## What This Changes

Ensure GitHub Copilot `openai-responses` embedded-runner calls preserve HTTP 429 fail-fast handling even when the session uses a Pi session stream wrapper.

## Best Possible Solution

Route every embedded Copilot `openai-responses` turn through `buildGuardedModelFetch` or an equivalent guarded provider-owned transport before idle-timeout handling, while preserving Copilot header and payload wrappers.

## Reproduction Assessment

Yes. The issue gives a concrete live `v2026.4.29` reproduction with immediate upstream 429 and `session-custom` idle-timeout telemetry, and current main still shows the same stream-resolution route for non-`streamSimple` session streams.

## Solution Assessment

Yes. The requested invariant is the right target; the safest fix is a narrow embedded/provider stream-resolution change with regression tests, not a broad retry-policy change or a hard-coded Copilot-only branch unless the plugin seam proves insufficient.

## Repair Work Prompt

Fix openclaw/openclaw#75411. Embedded GitHub Copilot `openai-responses` runs can still bypass `buildGuardedModelFetch` when the cached session base stream is a Pi default wrapper rather than exactly `streamSimple`, producing `streamStrategy: "session-custom"` and idle-timeout telemetry for terminal HTTP 429s. Keep the fix narrow around embedded/provider stream resolution: make the Pi default stream wrapper eligible for the boundary-aware OpenClaw Responses fallback or otherwise ensure Copilot `openai-responses` reaches the guarded transport before idle-timeout handling. Preserve Copilot's `wrapStreamFn` behavior for headers and payload rewriting, preserve true custom session streams, and avoid hard-coding Copilot in core unless the plugin seam is insufficient. Add focused regression tests that simulate an async wrapper around `streamSimple` for `github-copilot/gpt-5.4` with `api: "openai-responses"`, asserting the strategy/selected stream uses the boundary-aware transport and still forwards resolved api key and run signal. Add a user-facing `CHANGELOG.md` fix entry. Do not change dependency versions or broad retry policy.
