## Summary

Keep open: current main still leaves the reported embedded `session-custom` route able to bypass the guarded OpenAI Responses transport, even though the guarded fetch itself now handles terminal Copilot 429s correctly.

## What This Changes

The issue asks OpenClaw to fail fast on GitHub Copilot `openai-responses` HTTP 429 quota errors even when embedded runs use the session-custom stream path.

## Best Possible Solution

Route default embedded OpenAI Responses streams, including the Pi SDK wrapper used by GitHub Copilot, through OpenClaw's guarded transport before idle-timeout handling while retaining provider `wrapStreamFn` behavior.

## Reproduction Assessment

Yes. The issue provides a live `v2026.4.29` reproduction with immediate upstream 429 and `session-custom` idle-timeout telemetry, and current main plus the locked Pi dependency still show the same bypass route in source.

## Solution Assessment

Yes. The narrow maintainable direction is to treat the Pi default stream wrapper as eligible for boundary-aware fallback or otherwise inject the guarded OpenAI Responses transport generically, with regression tests around Copilot's wrapper path.

## Repair Work Prompt

Fix the GitHub Copilot `openai-responses` 429 fail-fast bypass reported in openclaw/openclaw#75411. Current main only selects the boundary-aware OpenAI Responses transport when the embedded session base stream is undefined or exactly `streamSimple`, but the locked Pi SDK default stream is an async wrapper around `streamSimple`; Copilot then wraps that custom stream and 429 quota errors can surface as idle timeouts. Implement a narrow generic fix in embedded stream resolution so default Pi-wrapper OpenAI Responses paths use `buildGuardedModelFetch`/`createOpenAIResponsesTransportStreamFn` before idle-timeout handling, while preserving provider `wrapStreamFn` behavior for Copilot headers/payload rewriting and not forcing arbitrary user custom streams onto OpenClaw transport. Add focused regression coverage for the Pi default-wrapper case, Copilot `openai-responses`, run signal forwarding, and terminal 429 retry bypass. Add a user-facing changelog entry. Do not hard-code a Copilot-only branch in core unless the plugin seam cannot express the behavior safely.
