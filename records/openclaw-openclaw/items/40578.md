The PR adds an opt-in memory-lancedb `agentScoping` config, an `agentId` LanceDB column/migration, and per-agent filtering for memory tools and lifecycle hooks.

## Best Possible Solution

Land a rebased isolation design that scopes auto-recall, auto-capture, duplicate checks, tools, deletes, and operator paths through one trusted scope identity, with migration diagnostics, docs, regression coverage, and redacted real two-agent proof.

## Solution Assessment

No. The direction is plausible, but this branch is not the best mergeable form until it is rebased to current main, scopes the active prompt-build hook, uses one validated scope key, handles migration failure visibly, and includes real behavior proof.
