## Summary

Keep open: current main still lacks per-agent LanceDB memory isolation, and this external draft is not safe to merge while it is dirty, missing real two-agent proof, and has scoped-hook, scope-key, migration-observability, and live-smoke blockers.

## What This Changes

The PR adds an optional `agentScoping` setting to `memory-lancedb`, an `agentId` LanceDB column/migration, and agent-context-aware memory tool and hook behavior.

## Best Possible Solution

Land a rebased memory-lancedb isolation design that scopes the active hooks, tools, duplicate/delete paths, CLI/operator behavior, and migration diagnostics with one validated partition key, plus docs, regression coverage, and live two-agent proof.

## Reproduction Assessment

Yes, source-level. Current main stores and searches memory-lancedb rows without an agent scope and active auto-recall searches the shared table from `before_prompt_build`; I did not establish a live two-agent LanceDB run.

## Solution Assessment

No. The direction is valid, but this patch scopes a stale hook, uses inconsistent scope-key handling, hides migration failures, and lacks live proof for the isolation behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope the active prompt-build recall hook:** `extensions/memory-lancedb/index.ts:591-601`
  - body: Current main runs auto-recall through `before_prompt_build` and has tests/contracts keeping bundled production code off `before_agent_start`. This branch scopes only the legacy hook, so after rebase automatic recall can still search the shared table and inject another agent's memories.
  - confidence: 0.91
- **[P2] Use one validated agent scope key:** `extensions/memory-lancedb/index.ts:428-455`
  - body: The patch stores raw `ctx.agentId` values but reads and deletes with a separately stripped value. Agent IDs containing excluded characters can be written under one key and searched under another, making scoped memories unreachable or colliding after normalization.
  - confidence: 0.88
- **[P3] Report failed agentId migrations:** `extensions/memory-lancedb/index.ts:119-122`
  - body: `migrateAgentIdColumn()` catches schema or `addColumns` failures without warning. Operators can enable scoping while an existing table remains effectively unscoped, with no diagnostic explaining why isolation is not active.
  - confidence: 0.86
- **[P3] Unwrap tool factories in the live smoke:** `extensions/memory-lancedb/index.ts:351`
  - body: The diff changes memory tools to factories so they can receive `ctx.agentId`, but the live smoke still reads `registeredTools[i].tool` and calls `.execute` directly. A live run will throw before exercising store, recall, or forget.
  - confidence: 0.82
