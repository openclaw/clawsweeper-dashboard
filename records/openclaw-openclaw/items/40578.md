## Summary

Keep open. Current main still lacks memory-lancedb agent or partition isolation, and this draft PR is a useful but not mergeable implementation because it targets an older hook path, has scope-key/migration issues, and lacks real two-agent LanceDB proof.

## What This Changes

Adds an opt-in `agentScoping` config to memory-lancedb, an `agentId` LanceDB column/migration, scoped memory tools and hooks, and manifest/config test updates.

## Best Possible Solution

Land a rebased memory-lancedb isolation design that uses one trusted partition key consistently across tools, auto-recall, auto-capture, duplicate checks, forget/delete, CLI/operator paths, migrations, docs, and regression tests.

## Reproduction Assessment

Yes, source-level. Configure two agents or user partitions against the same memory-lancedb `dbPath`, store memory from one, then recall/search from the other; current main has no row field or query filter that can prevent cross-scope results.

## Solution Assessment

No. The requested direction is valid, but this branch is not the best mergeable solution until it is rebased onto current main, scopes the active hook path, derives one validated scope key, reports migration failures, and includes real two-agent proof.

  - body: Current main registers memory-lancedb auto-recall on `before_prompt_build`, and tests assert the legacy hook is not registered. This branch scopes `before_agent_start`, so after rebase the active automatic recall path can still query the shared DB without the new scope key.

  - confidence: 0.88

  - body: `migrateAgentIdColumn()` catches schema or `addColumns` failures without warning. Operators can enable scoping while an existing table remains effectively unscoped, with no diagnostic explaining why isolation is not active.
  - confidence: 0.86

  - body: This patch registers memory tools as factories, but the live smoke still retrieves each registered tool and calls `.execute` directly. Once tools are factories, that path must instantiate the factory with context before exercising store, recall, and forget.
  - confidence: 0.82
