## Summary

Keep open: current main still lacks per-agent memory-lancedb isolation and related reports were consolidated onto this canonical PR, but the draft branch is not mergeable as-is because it is stale against current hook/config code, lacks real behavior proof, and has scoped-access defects.

## What This Changes

The PR adds an opt-in memory-lancedb agentScoping setting, an agentId LanceDB column/migration, scoped memory tool and hook reads/writes, and a small tool-factory test adjustment.

## Best Possible Solution

Land a rebased memory-lancedb isolation design that scopes tools, duplicate checks, before_prompt_build auto-recall, and agent_end capture through one validated scope-key contract, with migration diagnostics, docs/changelog, regression tests, and real two-agent proof.

## Reproduction Assessment

Yes, source-level: current main constructs one shared MemoryDB and the tool/hook paths search/store/delete without an agent scope. I did not run a live multi-agent LanceDB reproduction in this read-only review.

## Solution Assessment

No. The direction is valid, but this branch is stale and should not merge until it targets the current before_prompt_build path and uses one stable validated scope key across storage, search, delete, duplicate checks, and migration diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope the current prompt-build recall path:** `extensions/memory-lancedb/index.ts:591-601`
  - body: Current main moved automatic recall to `before_prompt_build` and tests assert the legacy hook is not registered. This branch scopes only `before_agent_start`, so after rebase the active auto-recall path can still search the shared DB unless the current hook receives the same agent scope.
  - confidence: 0.9
- **[P2] Validate the agent scope before storing it:** `extensions/memory-lancedb/index.ts:428-455`
  - body: The branch writes raw `ctx.agentId` into rows but filters search/delete with a separately stripped value. An ID containing characters like `/` or spaces can be stored under one key and queried under another, making scoped memories unreachable or causing normalized-key collisions.
  - confidence: 0.86
- **[P3] Surface failed agentId migrations:** `extensions/memory-lancedb/index.ts:119-122`
  - body: `migrateAgentIdColumn` catches schema or `addColumns` failures without logging. If an existing table cannot gain the new column, operators can enable scoping with no warning that isolation may not be active.
  - confidence: 0.86
- **[P3] Update live smoke callers for tool factories:** `extensions/memory-lancedb/index.ts:351-362`
  - body: This patch converts registered memory tools into factories, but current live smoke code retrieves `.tool` and calls `.execute` directly. After rebase, that live path needs to instantiate the factory with a context before executing the tool.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR changes memory data isolation semantics and currently has scoped-access defects; no unrelated supply-chain change was found.

Concerns:

- **[high] Automatic recall can remain unscoped:** `extensions/memory-lancedb/index.ts:591`
  - body: The branch scopes the stale `before_agent_start` hook while current main uses `before_prompt_build`; after rebase, the active automatic recall path can still inject memories from the shared table.
  - confidence: 0.88
- **[medium] Agent scope-key mismatch can break isolation:** `extensions/memory-lancedb/index.ts:428`
  - body: Writes use the raw context value while reads/deletes filter on a separately stripped value, so unexpected context values can make scoped memories unreachable or collide with another normalized key.
  - confidence: 0.84
- **[medium] Migration failures can silently disable scoping:** `extensions/memory-lancedb/index.ts:119`
  - body: The agentId column migration catches all errors without warning, leaving operators without a signal that scoped filtering may not be available for existing tables.
  - confidence: 0.84
