Full review comments:

- **[P1] Scope the active prompt-build hook:** `extensions/memory-lancedb/index.ts:591`
  - body: Current main moved memory auto-recall to `before_prompt_build`, and docs mark `before_agent_start` as compatibility-only. This branch scopes the legacy hook path, so a rebase that keeps this shape would leave the active auto-recall path unscoped or drop current timeout/live-config behavior.
  - confidence: 0.9
- **[P1] Validate one scope key before storing:** `extensions/memory-lancedb/index.ts:428`
  - body: This writes the raw `ctx.agentId`, while `search()` and `delete()` strip characters before filtering. Agent IDs like `agent/123` would be stored under one value and queried as `agent123`, so scoped memories can disappear or collide with another scope.
  - confidence: 0.93
- **[P2] Do not cap scoped ANN results before filtering:** `extensions/memory-lancedb/index.ts:146-151`
  - body: The scoped search builds `vectorSearch(vector).limit(limit)` before applying the `agentId` filter. With selective agent scopes, that can return too few rows when the nearest global candidates belong to other agents; use a prefilter-supported query or an inflated probe limit and truncate after filtering.
  - confidence: 0.86
- **[P2] Report failed agentId migrations:** `extensions/memory-lancedb/index.ts:119-122`
  - body: The migration catch block swallows every error, so an operator enabling scoped mode on a table where `schema()` or `addColumns()` fails gets no warning that the isolation column was not added. Surface this through plugin logging or an explicit diagnostic state.
  - confidence: 0.86
