- **[P1] Scope the current prompt-build hook:** `extensions/memory-lancedb/index.ts:588-601`
  - body: Current main moved memory-lancedb auto-recall to `before_prompt_build` and has a regression test that it no longer registers legacy `before_agent_start`.   - confidence: 0.93
- **[P1] Use one validated scope key everywhere:** `extensions/memory-lancedb/index.ts:428-455`
  - body: The scoped paths store `ctx.agentId ?? ""` raw, but search/delete build filters from a stripped version of the agent ID. Agent IDs containing spaces, slashes, or other stripped characters will store memories that cannot be recalled or can collide with another stripped ID, breaking the isolation boundary this PR adds.
  - confidence: 0.9
- **[P2] Surface agentId migration failures:** `extensions/memory-lancedb/index.ts:119-122`
  - body: `migrateAgentIdColumn` catches every error and logs nothing. If an existing table or LanceDB version cannot add the `agentId` column, operators who enable scoping can keep running with a broken isolation setup and no diagnostic telling them why recall is not scoped.
  - confidence: 0.86
- **[P2] Update the live harness for tool factories:** `extensions/memory-lancedb/index.ts:351`
  - body: The branch changes memory tools to factory functions, but the live memory-lancedb test still retrieves `registeredTools[].tool` and calls `.execute` directly. That live path will throw `execute is not a function`; unwrap the factory with the tool context before executing, as the unit test does.
  - confidence: 0.8
