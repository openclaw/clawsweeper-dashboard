## Summary

Keep open: current main and the latest release still use numbered config backup rotation, and this focused PR is not merge-ready because it can delete legacy rollback snapshots, can overwrite same-millisecond backups, is conflicting with main, and still lacks real behavior proof.

## What This Changes

The branch changes config backup rotation from numeric `.bak.1`-`.bak.4` slots to UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files and updates cleanup, permission hardening, and tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation in `src/config/backup-rotation.ts` that preserves or converts legacy numbered snapshots, allocates unique timestamped names, keeps pre-update snapshot support, and proves fresh-install plus upgrade config writes.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path deletes non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the full destination. The timestamp naming request itself is a feature change, not a current-main bug reproduction.

## Solution Assessment

No as submitted. `src/config/backup-rotation.ts` is the right boundary, but the branch needs migration-safe legacy handling, collision-safe naming, current-main conflict resolution, and real behavior proof before it is the maintainable fix.
