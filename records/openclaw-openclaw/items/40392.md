Full review comments:

- **[P1] Preserve legacy backup slots during migration:** `src/config/backup-rotation.ts:143-147`
  - body: Current releases preserve `.bak.1`-`.bak.4` as recovery points, but this cleanup treats those shipped files as orphans and deletes them on the first post-upgrade config write. That can remove most of a user's rollback history before the datetime ring has been populated, so the migration needs to preserve or convert numbered backups and cover that upgrade path in tests.
  - confidence: 0.96
- **[P2] Avoid timestamp collisions before renaming backups:** `src/config/backup-rotation.ts:53-56`
  - body: Using the millisecond timestamp as the entire destination still lets two writes in the same millisecond target the same `.bak.<timestamp>` path. On POSIX the second rename can overwrite the previous backup, so the new format needs a collision-safe suffix or retry before replacing the numeric ring.
  - confidence: 0.88
