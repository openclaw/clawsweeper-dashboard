Full review comments:

- **[P1] Preserve legacy backup slots during migration:** `src/config/backup-rotation.ts:143-147`
  - body: This branch makes every non-datetime `.bak.*` suffix an orphan, so an upgraded user with the shipped `.bak.1` through `.bak.4` ring loses those rollback files on the first config write. Preserve or convert the legacy slots until the datetime set can safely take over.
  - confidence: 0.96
- **[P2] Avoid timestamp collisions when rotating backups:** `src/config/backup-rotation.ts:53-56`
  - body: The destination is only `.bak.<timestamp>`, so two writes in the same millisecond target the same file. On POSIX this overwrites the previous timestamped backup, and on failure paths the following primary copy still drops the prior `.bak`; add a collision-safe suffix or retry before renaming.
  - confidence: 0.87
