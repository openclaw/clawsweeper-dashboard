Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve legacy backup slots during migration:** `src/config/backup-rotation.ts:146-147`
  - body: On upgrade, current releases may already have `openclaw.json.bak.1` through `.bak.4`. This branch treats those numbered slots as non-datetime orphans and unlinks them on the first cleanup pass, before the timestamp ring has replacement history, so users can lose rollback points. Preserve or migrate the legacy slots until the datetime retention window is safely populated.
  - confidence: 0.95
- **[P2] Avoid overwriting same-millisecond backups:** `src/config/backup-rotation.ts:53-55`
  - body: `rotateConfigBackups` uses only one millisecond timestamp as the destination. Both normal `fs.rename` on POSIX and the root-bound move adapter can replace an existing path, so two writes with the same timestamp can silently drop one backup; add a collision fallback or exclusive destination loop.
  - confidence: 0.86
