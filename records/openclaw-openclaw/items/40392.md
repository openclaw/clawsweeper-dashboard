## Summary

Keep open: current main still uses numbered config backup rotation, and this PR has blocking migration and collision defects plus missing real behavior proof before it can be considered mergeable.

## What This Changes

The PR replaces numeric `.bak.N` config backup rotation with UTC millisecond datetime-suffixed backups and updates cleanup, permission hardening, and focused tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, uses collision-safe bounded names, keeps owner-only permissions, updates focused tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the whole destination. The underlying timestamp naming request is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this patch is not the best mergeable solution until legacy migration, collision-safe naming, and real config-write proof are addressed.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:145-147`
  - body: This cleanup path treats every non-datetime `.bak.*` as an orphan, so users upgrading with `openclaw.json.bak.1` through `.bak.4` lose rollback snapshots on the first maintenance cycle. Preserve or migrate numbered slots until timestamped retention has equivalent history.
  - confidence: 0.92

  - confidence: 0.86
