Full review comments:

- **[P1] Preserve legacy backup slots during migration:** `src/config/backup-rotation.ts:146-147`
  - body: Current main treats `.bak.1`-`.bak.4` as managed recovery points, but this branch classifies every non-datetime `.bak.*` as an orphan and unlinks it. On the first post-upgrade config write, users can lose the numbered rollback files before the datetime ring is populated, so preserve or convert those slots during migration.
  - confidence: 0.96
- **[P2] Avoid overwriting same-millisecond backups:** `src/config/backup-rotation.ts:53-55`
  - body: The timestamp is the entire destination name, so two rotations with the same millisecond target the same file. POSIX `rename` replaces the existing destination and the catch hides failures, so add a collision fallback or exclusive destination selection before renaming.
  - confidence: 0.86
