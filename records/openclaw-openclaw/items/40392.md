- **[P1] Preserve legacy numeric backups during migration:** `src/config/backup-rotation.ts:146-147`
  - body: `cleanOrphanBackups` now treats every non-datetime `.bak.*` as an orphan, including existing `.bak.1` through `.bak.4` files. On the first config write after upgrade, this collapses a user's recovery history to the new primary `.bak` plus one timestamped file, while current main and the latest release preserve those numbered slots; migrate or keep them until equivalent timestamped retention exists.
  - confidence: 0.91
- **[P2] Handle timestamp collisions before replacing the ring:** `src/config/backup-rotation.ts:53`
  - body: `rotateConfigBackups` uses the millisecond timestamp as the whole destination name. If two writes land in the same millisecond, POSIX `rename` replaces the earlier timestamped backup and Windows can fail into the catch path, so one recovery point is silently lost; add a unique collision fallback before switching away from the numbered ring.
  - confidence: 0.82
