## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or migrates existing numbered backups, uses collision-safe destination names, keeps bounded retention and owner-only permissions, and updates focused tests with real config-write proof.

## Solution Assessment

No. The datetime naming direction matches the linked feature request, but this patch is not the best mergeable solution until it preserves or migrates legacy numbered backups, uses collision-safe unique destinations, and maintainers choose the canonical scope among the related open backup items.

  - body: On upgrade, existing `openclaw.json.bak.1` through `.bak.4` entries take the non-datetime cleanup path and are unlinked even though only one timestamped backup can exist at that point.   - confidence: 0.9
