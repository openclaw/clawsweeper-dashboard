## Summary

Keep open: current main and the latest release still use numbered config backups, and this PR is not merge-ready because it can delete existing legacy rollback snapshots, can overwrite same-millisecond timestamped backups, lacks real behavior proof, and overlaps another open implementation.

## What This Changes

The PR replaces numeric config backup rotation with UTC millisecond timestamped backup filenames and updates cleanup, permission hardening, and focused tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, allocates collision-safe bounded names, keeps owner-only permissions, updates focused tests, and includes redacted real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the full destination. The underlying datetime naming request is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this branch should not merge until legacy migration, collision-safe naming, real behavior proof, and overlap with https://github.com/openclaw/openclaw/pull/39982 are resolved.

  - body: This branch treats every non-datetime `.bak.*` entry as an orphan, including existing `.bak.1` through `.bak.4`. On upgrade, the first cleanup permanently removes rollback snapshots before timestamped retention has equivalent history; preserve or migrate those numbered backups instead.
