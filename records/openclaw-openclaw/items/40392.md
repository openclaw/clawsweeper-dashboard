## Summary

Keep open: current main still uses the numbered backup ring, and this PR is useful but not mergeable while it deletes legacy numbered backups during migration, can clobber same-millisecond timestamped backups, and lacks the required changelog entry.

## What This Changes

The PR changes config backup rotation and tests from numbered `.bak.1`-`.bak.4` slots to UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files with directory-scanned hardening and cleanup.

## Best Possible Solution

Land one consolidated datetime-backup implementation that preserves or migrates existing numbered backups until equivalent timestamped retention exists, uses collision-safe destination names, keeps bounded retention and owner-only permissions, updates focused tests, and adds the required user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection of the PR path gives a high-confidence reproduction: create existing `openclaw.json.bak.1` through `.bak.4` files and run cleanup or maintenance to hit the unlink branch, and call rotation twice with the same `Date` to target the same timestamped destination.

## Solution Assessment

No. The datetime naming direction matches the linked feature request, but this patch is not the best mergeable form until it preserves or migrates legacy numbered backups, chooses unique collision-safe destinations, and records the user-facing behavior change in the changelog.
