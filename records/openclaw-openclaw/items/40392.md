## Summary

Keep open for maintainer and contributor follow-up: current main still uses numbered config backup rotation, while this branch has blocking backup-retention defects, lacks real behavior proof, and overlaps another open timestamp-backup implementation.

## What This Changes

The PR changes config backup rotation from `.bak.N` slots to UTC millisecond datetime-suffixed backups, updates cleanup and permission hardening, and rewrites focused backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, uses collision-safe bounded names, keeps owner-only permissions, updates focused tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the full destination. The underlying datetime naming request is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this branch should not merge until legacy migration, collision-safe naming, real proof, and overlap with the other open implementation are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:145-147`
  - body: This cleanup path treats every non-datetime `.bak.*` as an orphan, so users upgrading with `openclaw.json.bak.1` through `.bak.4` lose rollback snapshots on the first maintenance cycle. Preserve or migrate numbered slots until timestamped retention has equivalent history.
  - confidence: 0.92
- **[P2] Allocate unique timestamped backup names:** `src/config/backup-rotation.ts:53`
  - body: Rotation renames `.bak` directly to `.bak.<timestamp>`. Two writes in the same millisecond target the same path, so POSIX rename overwrites the previous snapshot and Windows may drop the rotation via the catch path; probe for an available suffix before renaming.
  - confidence: 0.86
