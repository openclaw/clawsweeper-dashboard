Full review comments:

- **[P1] Preserve legacy backup slots during migration:** `src/config/backup-rotation.ts:147`
  - body: This branch treats every non-datetime `.bak.*` file as an orphan, so an existing user with `.bak.1` through `.bak.4` loses those rollback snapshots on the first config write after upgrade. Current main preserves those slots, and the linked issue explicitly calls out preserving or migrating them, so the cleanup needs a migration-safe path before merge.
  - confidence: 0.93
- **[P2] Avoid overwriting backups on timestamp collision:** `src/config/backup-rotation.ts:53`
  - body: `rotateConfigBackups` names the destination only from the millisecond timestamp. If two writes receive the same timestamp, POSIX `rename` replaces the existing backup and Windows failure is swallowed, so one recovery snapshot is lost; add a collision probe or suffix before renaming.
  - confidence: 0.84
