## Summary

Keep this PR open: current main still uses numbered config-backup rotation, but the PR has blocking migration, collision, and changelog issues before it is mergeable.

## What This Changes

The PR replaces numbered config backup slots with UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files and updates backup cleanup, permission hardening, and tests.

## Best Possible Solution

Land one canonical datetime-backup implementation that preserves or migrates existing numbered backups, uses collision-safe destination names, keeps bounded retention and owner-only permissions, updates focused tests, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: create existing `.bak.1` through `.bak.4` files and run cleanup to hit the unlink branch, or call rotation twice with the same `Date` to target the same timestamped path.

## Solution Assessment

No. The datetime naming direction matches #39923, but this patch is not the best mergeable form until legacy backups are preserved or migrated, timestamp collisions get a unique fallback, and the changelog records the user-facing change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve legacy backups during timestamp migration:** `src/config/backup-rotation.ts:147`
  - body: On upgrade, existing `openclaw.json.bak.1` through `.bak.4` entries take the non-datetime cleanup path and are unlinked even though only one timestamped backup can exist at that point. Keep or migrate the numeric ring until equivalent datetime retention exists so users do not lose rollback snapshots.
  - confidence: 0.9
- **[P2] Avoid clobbering backups on timestamp collisions:** `src/config/backup-rotation.ts:53`
  - body: `rotateConfigBackups` uses the timestamp as the whole destination name. Two writes in the same millisecond target the same file, so POSIX rename overwrites the previous backup and Windows can fall into the best-effort catch; choose a unique fallback suffix before renaming.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/config/backup-rotation.ts:46-49`
  - body: This changes user-visible config backup filenames and recovery behavior, but the PR only updates source and tests. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing config changes before merge.
  - confidence: 0.86
