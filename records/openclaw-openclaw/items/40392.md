## Summary

This PR remains necessary as the linked implementation for readable timestamped backups, but it is not mergeable yet: the first post-upgrade maintenance deletes the shipped numbered recovery ring, identical-millisecond rotations can replace a snapshot, and the contributor has not supplied real filesystem proof. The paired issue should remain open until a corrected PR lands.

## What This Changes

The PR replaces the five-slot numeric config-backup ring with UTC millisecond datetime suffixes and updates backup pruning, permission hardening, and focused tests.

## Best Possible Solution

Adopt collision-safe datetime filenames while preserving or migrating legacy numbered backups until the bounded datetime set reaches full capacity, then prove fresh and upgrade flows with redacted real filesystem output.

## Reproduction Assessment

Yes. Create the current `.bak` plus `.bak.1`–`.bak.4` layout, run the patched maintenance flow once, and its datetime-only cleanup removes all numbered files; calling rotation twice with the same `Date` also targets one destination.

## Solution Assessment

No. Datetime names are a reasonable ergonomic improvement, but the current patch is not the best solution because it knowingly sacrifices existing recovery points and collision safety instead of providing a bounded migration.
