  - body: This branch treats every non-datetime `.bak.*` file as an orphan, so existing shipped rollback files like `.bak.1` through `.bak.4` are deleted on the first config write after upgrade. On POSIX that can overwrite a previous backup, and on Windows the swallowed failure drops a history slot; probe or suffix collisions instead of documenting the loss.
  - confidence: 0.84
