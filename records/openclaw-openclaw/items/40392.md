## Summary

Keep open: current main and v2026.5.12 still use numbered config backup rotation, and this external feature PR still has merge blockers: legacy backup deletion, timestamp collision, missing real behavior proof, and an overlapping open implementation at https://github.com/openclaw/openclaw/pull/39982. The linked request at https://github.com/openclaw/openclaw/issues/39923 remains the canonical feature thread for maintainer selection.

## What This Changes

This PR replaces numbered config backup rotation with UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files and updates cleanup, permission hardening, and backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, guarantees collision-safe bounded owner-only backups, updates focused tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR-head cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the whole destination. The underlying request is a feature change, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this patch is not mergeable until legacy migration, collision-safe naming, and real proof are addressed; maintainers also need to choose between this branch and https://github.com/openclaw/openclaw/pull/39982.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:145-147`
  - body: On upgrade, existing `openclaw.json.bak.1` through `.bak.4` files enter this non-datetime cleanup branch and are unlinked even though only one timestamped backup can exist yet. Preserve or convert the numeric ring until equivalent datetime retention exists so users do not lose rollback snapshots.
  - confidence: 0.9
- **[P2] Avoid clobbering backups on timestamp collisions:** `src/config/backup-rotation.ts:53`
  - body: `rotateConfigBackups` uses the timestamp as the whole destination name. Two writes in the same millisecond target the same file, so POSIX rename overwrites the previous backup and Windows can fall into the best-effort catch; allocate a unique fallback suffix before renaming.
  - confidence: 0.84
