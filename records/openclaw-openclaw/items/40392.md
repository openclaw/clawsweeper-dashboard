## Summary

Keep open: the PR targets a real config-backup feature that current main still lacks, but it should not merge until legacy backup migration, timestamp collision handling, missing real behavior proof, and overlap with https://github.com/openclaw/openclaw/pull/39982 are resolved.

## What This Changes

This PR changes config backup rotation from numbered `.bak.N` slots to UTC millisecond datetime-suffixed backups and updates cleanup, permission hardening, and focused tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, uses collision-safe bounded names, keeps owner-only permissions, updates focused tests, and includes redacted real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the full destination. The underlying datetime naming request is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this branch should not merge until legacy migration, collision-safe naming, real proof, and overlap with https://github.com/openclaw/openclaw/pull/39982 are resolved.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:145-147`
  - body: This cleanup path treats every non-datetime `.bak.*` as an orphan, so users upgrading with existing `.bak.1` through `.bak.4` lose rollback snapshots on the first maintenance cycle. Preserve or migrate the numbered ring until timestamped retention has equivalent history.
  - confidence: 0.92
- **[P2] Allocate unique timestamped backup names:** `src/config/backup-rotation.ts:53`
  - body: Rotation renames `.bak` directly to `.bak.<timestamp>`. Two writes in the same millisecond target the same path, so POSIX rename overwrites a snapshot and Windows can drop the rotation through the catch path; probe for an available suffix before renaming.
  - confidence: 0.86
