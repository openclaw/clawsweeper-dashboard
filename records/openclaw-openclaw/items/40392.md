## Summary

Keep open: current main still uses the numbered config backup ring, and this PR is not merge-ready because it can delete legacy rollback backups on upgrade, can collide on same-millisecond writes, lacks real behavior proof, conflicts with main, and overlaps another open implementation.

## What This Changes

This PR changes config backup rotation from numbered `.bak.N` slots to UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files and updates cleanup, permission hardening, and backup tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, allocates collision-safe bounded names, keeps owner-only permissions, and proves fresh-install plus upgrade config writes.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path deletes non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the full destination. The underlying timestamp naming request is a feature change, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this branch should not merge until legacy migration, collision-safe naming, real proof, conflict resolution, and overlap with https://github.com/openclaw/openclaw/pull/39982 are resolved.
