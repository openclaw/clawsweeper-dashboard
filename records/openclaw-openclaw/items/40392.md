## Summary

Keep open: current main and the latest release still use numbered config backup rotation, while this PR is a useful but not-yet-mergeable feature implementation. The PR still deletes legacy numbered backups during migration, can clobber same-millisecond timestamped backups, lacks real behavior proof, and overlaps another open implementation path.

## What This Changes

The PR replaces numbered config backup rotation with UTC millisecond timestamped `.bak.<YYYYMMDD-HHmmssSSS>` files and updates cleanup, permission hardening, and focused backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, uses collision-safe bounded timestamp names, keeps owner-only backup permissions, updates focused tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cleanup path unlinks non-datetime `.bak.*` files including legacy numbered backups, and rotation uses the timestamp as the whole destination. The underlying request is a feature change, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this patch is not mergeable until legacy migration, collision-safe naming, and real proof are addressed; maintainers also need to choose between this branch and https://github.com/openclaw/openclaw/pull/39982.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:145-147`
  - body: This branch treats every non-datetime `.bak.*` file as an orphan, so users upgrading with existing `openclaw.json.bak.1` through `.bak.4` lose rollback snapshots on the first maintenance pass before the datetime set is populated. Preserve or migrate the numbered ring until equivalent timestamped retention exists.
  - confidence: 0.9
- **[P2] Allocate unique names for timestamp collisions:** `src/config/backup-rotation.ts:53`
  - body: The destination is only `.bak.<timestamp>`. Two rotations in the same millisecond target the same path, so POSIX rename overwrites the previous snapshot and Windows can fall into the best-effort catch; probe or allocate a unique suffix before renaming.
  - confidence: 0.84
