## Summary

Keep open. Current main still does not provide a render_svg/show_widget-style SVG or HTML renderer, and the remaining work needs a maintainer product/security decision because external plugin tools can be model-callable but do not have an obvious trusted local-artifact handoff for generated PNGs.

## What This Changes

The issue requests a built-in tool that accepts SVG or simple HTML markup and delivers the rendered result inline across chat channels.

## Best Possible Solution

Define a plugin-first diagram-render artifact contract, with core adding only the missing safe media/artifact handoff if external renderer plugins cannot deliver sanitized generated images through existing channels.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a request for a new rendering capability. Source inspection confirms current main lacks the named render tool while existing paths handle already-produced media references.

## Solution Assessment

No for the request as a built-in core tool. The safer direction is plugin/ClawHub first, with a narrow core plugin API extension only if current media delivery seams cannot support a secure rendered artifact flow.
