## Summary

Keep open. Current main still does not provide an inline SVG/HTML render tool, and the closest plugin/media seams leave product and security decisions around sanitized markup, editable diagram artifacts, and trusted rendered-media delivery.

## What This Changes

The issue requests a built-in tool that turns SVG or simple HTML into inline chat media across OpenClaw channels.

## Best Possible Solution

Define this as a secure plugin-owned renderer or a narrow Plugin SDK/media-trust extension, then publish the capability through ClawHub rather than adding a one-off built-in core tool.

Not applicable; this is a new rendering capability request, not a bug in existing behavior. Source review confirms current main has no matching render_svg/show_widget-style tool to exercise.

## Solution Assessment

Status: needs_attention
