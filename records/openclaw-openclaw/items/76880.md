## Summary

Keep open: current main still lacks the response-header normalization that this regression reports, and the referenced open PR #76791 is the right implementation path but has not merged yet.

## What This Changes

Restore Google Chat inbound webhook auth by normalizing google-auth-library/gaxios response headers after the earlier request-header fix did not resolve `res?.headers.get is not a function`.

## Best Possible Solution

Review and land #76791 if its branch remains correct, keeping the fix plugin-local in the Google Chat auth transport and closing this issue only after merge.

## Reproduction Assessment

Yes at source level: a Google Chat `app-url` webhook reaches `verifyIdToken`, google-auth-library 10.6.2 reads `res?.headers.get('cache-control')`, and current main only normalizes request headers. I did not run a live Google Chat app webhook.

## Solution Assessment

Yes: the narrow maintainable solution is the response-header interceptor and regression coverage already proposed in #76791, not a core gateway change. The issue should remain open until that PR or an equivalent fix merges.
