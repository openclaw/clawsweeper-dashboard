## Summary

Keep open: current main still lacks the Google Chat auth response-header normalization needed for the reported `res?.headers.get is not a function` failure, and open PR #76791 is the active implementation candidate.

## What This Changes

The issue reports a Google Chat inbound auth regression where webhook verification fails because google-auth-library receives gaxios response headers without a `.get()` method.

## Best Possible Solution

Land or replace the response-header normalization in the Google Chat plugin auth transport with focused regression coverage, then close this after it reaches main.

## Reproduction Assessment

Yes, at source level: `audienceType: "app-url"` calls `OAuth2Client.verifyIdToken()` through the shared Gaxios transport, current main only normalizes request headers, and google-auth-library 10.6.2 reads `res?.headers.get('cache-control')`. I did not live-test a Google Chat app.

## Solution Assessment

Yes: a response interceptor in `extensions/googlechat/src/google-auth.runtime.ts` is the narrowest owner-local fix, with #76791 now carrying that approach and matching tests.
