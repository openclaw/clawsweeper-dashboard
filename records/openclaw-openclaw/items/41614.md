## Summary

Keep open: current main still has a source-reproducible stale cron cache path, but this branch is not ready because it misses the current `jobs-state.json` sidecar, uses a stale test dependency, and lacks required real behavior proof.

## What This Changes

The PR adds mtime-based reload logic to `CronService.ensureLoaded()` and unit tests for manual external `jobs.json` edits before list/run operations.

## Best Possible Solution

Rebase the fix so cron cache freshness covers both `jobs.json` and `jobs-state.json`, records freshness after loading, updates tests to the current `requestHeartbeat` dependency, and includes redacted real gateway/manual-edit proof.

## Reproduction Assessment

Yes, source-reproducible by inspection: current main's non-force `ensureLoaded()` returns cached state, `cron.run` checks `runningAtMs` after that path, and runtime fields can live in `jobs-state.json`. I did not run a live gateway because this review is read-only.

## Solution Assessment

No. Mtime invalidation is a maintainable direction, but this branch needs to be sidecar-aware, update its test fixtures for current main, and avoid recording freshness before the load before it is the narrow safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track the state sidecar when checking freshness:** `src/cron/service/store.ts:34-35`
  - body: `loadCronStore()` merges runtime fields from `jobs-state.json`, and `saveCronStore({ stateOnly: true })` can update fields like `runningAtMs` without touching `jobs.json`. Since this fast path only stats `state.deps.storePath`, clearing a stuck marker in the sidecar can still leave `cron.run` returning `already-running` until restart or a primary-file edit.
  - confidence: 0.91
- **[P2] Use the current heartbeat dependency in tests:** `src/cron/service.external-store-reload.test.ts:25`
  - body: Current `CronServiceDeps` requires `requestHeartbeat`, but the added fixtures pass `requestHeartbeatNow`. On current main this test file will not typecheck after rebase; update the constructors to the current dependency shape or shared harness.
  - confidence: 0.92
- **[P3] Record freshness after loading the store:** `src/cron/service/store.ts:39-40`
  - body: The patch makes mtime the cache-validity token, but the load path captures it before reading the file. A concurrent external edit can pair a newer mtime with older loaded content and then make the fast path treat stale state as fresh; stat after `loadCronStore()` instead.
  - confidence: 0.74
