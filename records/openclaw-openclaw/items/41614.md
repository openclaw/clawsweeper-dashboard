## Summary

Keep open: the stale cron-store cache bug is still source-reproducible on current main, but this PR needs focused rebase fixes for sidecar freshness/current deps plus real behavior proof before merge.

## What This Changes

The branch adds mtime-based CronService cache invalidation for external `jobs.json` edits and tests manual clearing/deletion recovery.

## Best Possible Solution

Rebase the fix so cron cache freshness covers both `jobs.json` and `jobs-state.json`, records freshness after loading, updates tests to current dependencies, and includes redacted real gateway/manual-edit proof.

## Reproduction Assessment

Yes, by source inspection: current main’s non-force `ensureLoaded` path returns cached cron state, and `cron.run` checks `runningAtMs` after that path. The current sidecar contract means the stale marker can live in `jobs-state.json`, which this branch does not track.

## Solution Assessment

No. Mtime invalidation is a maintainable direction, but this implementation must be sidecar-aware, use the current dependency names in tests, and record freshness after the load before it is the narrow safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track the state sidecar when checking freshness:** `src/cron/service/store.ts:34-35`
  - body: `loadCronStore()` merges runtime fields from `jobs-state.json`, and `saveCronStore({ stateOnly: true })` can update fields like `runningAtMs` without touching `jobs.json`. Since this fast path only stats `state.deps.storePath`, clearing a stuck marker in the sidecar can still leave `cron.run` returning `already-running` until restart or a config-file edit.
  - confidence: 0.91
- **[P2] Use the current heartbeat dependency in tests:** `src/cron/service.external-store-reload.test.ts:25`
  - body: Current `CronServiceDeps` requires `requestHeartbeat`, but the added fixtures pass `requestHeartbeatNow`. On current main this test file will not typecheck after rebase; update both constructors to the current dependency shape or shared harness.
  - confidence: 0.92
- **[P3] Record freshness after loading the store:** `src/cron/service/store.ts:42-48`
  - body: The patch makes mtime the cache-validity token, but the load path captures it before reading the file. A concurrent external edit can pair a newer mtime with older loaded content and then make the fast path treat stale state as fresh; stat after `loadCronStore()` instead.
  - confidence: 0.74
