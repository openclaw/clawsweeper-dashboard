## Summary

Keep this PR open: it targets a real cron cache bug, but the current branch is not mergeable as-is because it misses the `jobs-state.json` sidecar, uses an outdated test dependency name, records the cache mtime before the read, and lacks after-fix real behavior proof.

## What This Changes

The PR adds mtime-based cron store cache invalidation in `ensureLoaded` and a new regression test file for manual `jobs.json` edits.

## Best Possible Solution

Rebase the fix so cron cache freshness accounts for both `jobs.json` and `jobs-state.json`, records freshness after loading, updates tests to current `CronServiceDeps`, and includes redacted real gateway/manual-edit proof.

## Reproduction Assessment

Yes, by source inspection: current main returns cached cron state for non-force loads, and `cron.run` uses that path before checking `runningAtMs`. The sidecar contract means the stale marker can live in `jobs-state.json`, which the PR’s cache token does not track.

## Solution Assessment

No. Mtime invalidation is a maintainable direction, but this branch needs sidecar-aware freshness, current dependency names in tests, and post-read mtime recording before it is the narrow safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track the state sidecar when checking freshness:** `src/cron/service/store.ts:34-35`
  - body: `loadCronStore()` merges runtime fields from `jobs-state.json`, and `saveCronStore({ stateOnly: true })` can update fields like `runningAtMs` without touching `jobs.json`. Since this fast path only stats `state.deps.storePath`, clearing a stuck marker in the sidecar can still leave `cron.run` returning `already-running` until restart or a config-file edit.
  - confidence: 0.91
- **[P2] Use the current heartbeat dependency in tests:** `src/cron/service.external-store-reload.test.ts:25`
  - body: Current `CronServiceDeps` requires `requestHeartbeat`, but the added fixtures pass `requestHeartbeatNow`. On current main this test file will not typecheck; update both constructors to the current dependency shape or the shared harness.
  - confidence: 0.92
- **[P3] Record the cache mtime after loading:** `src/cron/service/store.ts:42-48`
  - body: The patch makes mtime the cache-validity token, but the load path captures it before reading the file. A concurrent external edit can pair a newer mtime with older loaded content and then make the fast path treat stale state as fresh; stat after `loadCronStore()` instead.
  - confidence: 0.74
