## Summary

## What This Changes

Preserve user-assigned Control UI dashboard session labels across Gateway restarts and browser reconnects.

## Best Possible Solution

Retain the explicit-user-intent boundary so reconnect restores the persisted selected session and UUID dashboard children are created only by deliberate New Chat actions.

## Reproduction Assessment

No current-main failing reproduction remains: source history identifies the former no-intent creation path, and current regression coverage proves it no longer creates or switches sessions.

## Solution Assessment

Yes. Preventing unintended session creation at the shared UI helper is narrower and safer than copying labels between sessions or changing UUID-backed conversation boundaries.
