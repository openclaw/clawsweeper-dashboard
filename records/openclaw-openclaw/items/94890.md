## Summary

Keep this PR open for maintainer review, but it is not merge-ready: the API-layer direction is plausible, yet the branch changes an existing abort default, can reclassify ownerless model markers as live work, and still lacks real CLI/provider recovery proof.

## What This Changes

The PR adds `hasActiveModelCall` to diagnostic session snapshots, uses it for model-call classification and recovery, lowers the unset abort fallback, and updates diagnostics tests plus a test-plan entry.

## Best Possible Solution

Preserve the shipped abort default unless maintainers approve the shorter policy, make non-embedded model-call recovery depend on live owned activity or land equivalent ownerless marker cleanup first, and keep the restart replay remainder tracked separately.

## Reproduction Assessment

Yes for source-level review: current main has a focused diagnostic test where a stale model call without embedded-run ownership emits a stalled diagnostic and does not recover; no live Xiaomi/Codex/provider stalled-call recovery proof is supplied.

## Solution Assessment

No as-is. Adding a model-call activity snapshot field is plausible, but using it without an owner or cleanup boundary and lowering the shipped default is not the narrowest safe fix.
