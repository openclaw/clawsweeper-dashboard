## Label Justifications

- `P1`: The PR targets gateway stuck-session recovery, where bad abort or recovery behavior can block real agent/channel workflows.
- `merge-risk: 🚨 compatibility`: Lowering the unset abort fallback changes upgrade behavior for existing installs with no explicit diagnostic abort setting.
- `merge-risk: 🚨 session-state`: The new marker gate changes how per-session model-call activity is classified and recovered when ownerless markers are present.
- `merge-risk: 🚨 availability`: The watchdog timing and active-abort changes can either leave hung sessions unrecovered or abort slow but healthy model calls too early.

## Summary

Keep this PR open but not merge-ready: the API-layer direction is useful, but the current patch still has mock-only proof, changes the shipped abort fallback for existing installs, and can reclassify ownerless model markers as active long-running work.

## What This Changes

The branch adds `hasActiveModelCall` to diagnostic activity snapshots, uses it for non-embedded model-call stall classification and recovery, lowers the default stalled-run abort fallback, and updates focused diagnostics tests plus one test-plan fixture.

## Best Possible Solution

Preserve the shipped abort default unless maintainers explicitly approve the shorter policy, make non-embedded model-call recovery depend on live owned activity or land ownerless marker cleanup first, and split the restart replay remainder if this PR stays scoped.

## Reproduction Assessment

Yes for source-level review: current main has a focused diagnostic test where a stale model call without embedded-run ownership emits a stalled diagnostic and does not recover. No live Xiaomi/Codex provider stalled-call reproduction is supplied.

## Solution Assessment

No as-is. Adding an activity-layer model-call signal is a plausible direction, but the shorter default abort policy and ownerless-marker semantics need maintainer approval or a narrower ownership-aware repair.
