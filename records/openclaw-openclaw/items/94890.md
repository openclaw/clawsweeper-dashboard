## Summary

Keep this PR open, but it is not merge-ready: the focused diagnostic-recovery direction is useful, while the current branch still changes a shipped abort default, changes ownerless model-call classification, and lacks real CLI/provider stalled-call proof.

## What This Changes

The PR adds `hasActiveModelCall` to diagnostic session activity snapshots, uses it for model-call stall classification and active-abort recovery, lowers the unset diagnostic abort fallback, and updates focused diagnostics tests plus a test-plan entry.

## Best Possible Solution

Preserve the shipped abort default unless maintainers approve the shorter policy, make non-embedded model-call recovery depend on live owned activity or land equivalent ownerless marker cleanup first, and split the restart replay remainder if this PR stays scoped.
