## Summary

Keep open: the PR targets a real diagnostic recovery gap, but the current patch is not merge-ready because it changes an upgrade-sensitive abort default, weakens ownerless model-marker classification, and still lacks real CLI/provider proof.

## What This Changes

The branch adds `hasActiveModelCall` to diagnostic activity snapshots, uses it for non-embedded model-call stall classification and recovery, lowers the unset stalled-run abort fallback, and updates focused diagnostics tests plus one test-plan fixture.

## Best Possible Solution

Preserve the shipped abort default unless maintainers approve the shorter policy, make non-embedded model-call recovery depend on live owned activity or land ownerless marker cleanup first, and split the restart replay remainder if this PR stays scoped.
