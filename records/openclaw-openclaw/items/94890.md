## Summary

Keep open: the PR targets a real source-level recovery gap, but it is not merge-ready because it changes a shipped abort default, conflates live non-embedded model calls with ownerless model markers, and still lacks real runtime proof.

## What This Changes

The branch adds a `hasActiveModelCall` diagnostic snapshot flag, uses it for model-call stall classification and recovery, lowers the default stalled-run abort threshold, and updates focused diagnostic tests.

## Best Possible Solution

Preserve the shipped abort default unless maintainers explicitly approve a shorter policy, add a live non-embedded model-call ownership signal or sequence this after ownerless marker cleanup, and split the restart replay remainder from the linked issue if this PR stays scoped.

## Reproduction Assessment

Yes for source-level review: current main has a focused diagnostic test where a stale model call without embedded-run ownership emits `session.stalled` and does not recover. No live Xiaomi/Codex provider reproduction was supplied for the reported outage.

## Solution Assessment

No as-is. The API-layer direction is plausible, but this implementation conflates live non-embedded model calls with ownerless model markers and changes a shipped default without maintainer-approved upgrade policy.
