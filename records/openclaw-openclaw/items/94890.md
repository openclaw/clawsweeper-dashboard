## Label Justifications

- `P1`: The PR changes active gateway/session recovery behavior for stuck model calls that can block real agent or channel workflows.
- `merge-risk: 🚨 compatibility`: Lowering the unset abort fallback changes upgrade behavior for existing installs with no explicit diagnostics config.
- `merge-risk: 🚨 session-state`: The new snapshot gate changes how per-session model activity markers are classified and recovered when ownership is absent.
- `merge-risk: 🚨 availability`: Earlier active aborts can terminate slow silent providers sooner, while stale ownerless markers can keep sessions in the wrong recovery lane.

## Summary

Keep this PR open, but not merge-ready: the updated branch still changes a shipped diagnostic abort default and treats ownerless model-call markers as long-running/recoverable despite the current documented ownerless-marker contract, and the contributor proof remains unit-test output rather than a real stalled CLI/provider recovery run.

## What This Changes

The PR adds `hasActiveModelCall` to diagnostic activity snapshots, uses it for model-call classification and active-abort recovery, lowers the default stalled-run abort fallback, and updates diagnostic tests.

## Best Possible Solution

Preserve the shipped abort fallback unless maintainers explicitly approve a shorter policy, make non-embedded model-call recovery depend on live owned activity or equivalent ownerless-marker cleanup, and keep replay/deduplication tracked on the canonical issue or a separate maintainer-approved follow-up.

## Reproduction Assessment

Yes for source-level review: current main has a focused diagnostic test where a stale model call without embedded-run ownership emits a stalled event and does not recover. No live Xiaomi/Codex/provider stalled-call recovery path has been proven in the PR.

## Solution Assessment

No as-is. Adding a model-call activity snapshot flag is plausible, but using it without a live owner or cleanup boundary and lowering the shipped default is not the narrowest safe fix.
