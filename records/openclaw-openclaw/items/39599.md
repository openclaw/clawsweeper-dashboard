## Summary

Current main and the latest stable release still accept the reported Feishu/Lark announce cron shape without a target, and live GitHub state shows open closing PRs for the same bug, so this should remain open until one fix is reviewed and merged.

## What This Changes

Require cron create/update validation to reject Feishu/Lark announce jobs without an explicit or deterministically resolvable delivery target.

## Best Possible Solution

Land one narrow current-main fix that enforces Feishu/Lark announce target validation at the cron service/gateway boundary, keeps CLI feedback aligned, and adds regression coverage for missing and valid targets.

## Reproduction Assessment

Yes. Source inspection shows `cron add` can build Feishu/Lark announce delivery without `--to`, the protocol and gateway accept it, and service validation does not reject it before runtime target resolution can fail.

## Solution Assessment

Yes. Fail-fast validation at cron create/update is the narrow maintainable fix; the open linked PRs should be reviewed or consolidated rather than starting another duplicate repair.
