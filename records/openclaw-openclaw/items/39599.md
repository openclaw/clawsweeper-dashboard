Keep this issue open: current main and the latest stable release still accept the reported Feishu/Lark announce cron delivery shape without a target, while open linked fix PRs already cover the same bug and need review or consolidation before the issue can close.

Cron Feishu/Lark announce job creation and update should reject missing `delivery.to` unless a real runtime delivery target can be resolved.

## Best Possible Solution

Review or consolidate one linked current-main fix that enforces Feishu/Lark announce target validation at the cron service/gateway boundary, aligns CLI feedback, and adds regression coverage for missing and valid targets.
