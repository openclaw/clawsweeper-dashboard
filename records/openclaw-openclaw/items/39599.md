The issue requests fail-fast validation for Feishu/Lark announce cron jobs when no explicit or deterministically resolved delivery target exists.

## Best Possible Solution

Review or consolidate one linked fix that enforces Feishu/Lark announce target validation at the cron service/gateway boundary, keeps CLI feedback aligned, and adds regression coverage for missing and valid targets.
