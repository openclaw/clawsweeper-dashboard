## Summary

Keep this open: current main and the latest release still allow the reported Feishu/Lark announce cron shape without an explicit target, and the open linked fix PRs have not merged yet.

## What This Changes

The issue requests create-time validation so Feishu/Lark announce cron jobs without `delivery.to` are rejected unless a deterministic target can be resolved.

## Best Possible Solution

Enforce Feishu/Lark announce target validation at the cron service/gateway boundary, keep CLI feedback aligned, and add focused regression coverage for missing and valid targets.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported `cron add --announce --channel feishu` path can create delivery without `to`, and the runtime target resolver can later return the Feishu missing-target error when no default exists.

## Solution Assessment

Yes. Fail-fast create/update validation, with CLI feedback and regression tests, is the narrow maintainable fix; the issue should stay open until one linked fix is reviewed, consolidated, and merged.
