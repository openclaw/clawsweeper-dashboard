## Summary

Current main still accepts the reported Feishu announce cron shape at create time, and there are already open PRs linked with closing syntax. This should stay open until one narrow fix is reviewed, merged, and closes the issue normally.

## What This Changes

Require Feishu announce cron jobs to fail at creation or update time unless `delivery.to` is explicit or a deterministic target can be resolved.

## Best Possible Solution

Land one narrow validation fix at the cron gateway/service boundary, keep CLI/UI feedback aligned, add regression coverage for Feishu/Lark missing and explicit targets, then let the linked PR close this issue.
