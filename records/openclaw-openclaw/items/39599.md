## What This Changes

The issue requests create/update-time validation so Feishu/Lark announce cron jobs cannot be stored without an explicit or deterministically resolved delivery target.

## Best Possible Solution

Review and land one focused fix that validates Feishu/Lark announce targets at the gateway/service boundary, keeps CLI feedback aligned, and adds regression coverage for missing and valid targets.
