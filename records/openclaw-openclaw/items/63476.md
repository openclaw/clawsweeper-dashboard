## Summary

Keep open. Current main still rewrites `media://inbound/...` into a malformed workspace path in the normal agent-session image-tool path, and the open linked PR remains the right implementation candidate to review or supersede.

## What This Changes

The issue asks for agent-session image tool calls to preserve inbound image references so the image tool receives the actual managed image data.

## Best Possible Solution

Land #63497 or an equivalent narrow patch that preserves canonical `media://inbound` references through image-tool workspace normalization and adds regression coverage for the `workspaceDir` agent-session path.

Yes. A high-confidence source-level reproduction is to create the normal image tool with `workspaceDir` and pass `media://inbound/<id>`; current main resolves it to a malformed `workspace/media:/inbound/...` path before managed media hydration can run.

## Solution Assessment

Yes for the current triage direction. Keeping this open while reviewing, rebasing, or superseding #63497 is the narrowest maintainable path; broad Telegram, gateway, or provider changes are not needed for the observed URI-preservation bug.
