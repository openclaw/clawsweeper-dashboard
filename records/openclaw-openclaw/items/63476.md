## Summary

Keep open. Current main still can rewrite `media://inbound/...` into a malformed workspace path in the normal agent image-tool path, and the issue is already paired with open implementation candidates such as https://github.com/openclaw/openclaw/pull/63497.

## What This Changes

The issue reports that agent-session image-tool calls lose or misresolve inbound image data, causing image understanding to report no image even though direct image model calls work.

## Best Possible Solution

Land a rebased or replacement fix that preserves or resolves `media://` managed-media refs before workspace or sandbox path normalization, covers the `workspaceDir` agent-session path, and then close this issue with the overlapping reports.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create the normal image tool with `workspaceDir` and pass `media://inbound/<id>`; current main resolves it into a workspace `media:/inbound/<id>` path before managed media hydration can run.
