## Summary

## What This Changes

The PR changes `src/auto-reply/thinking.ts` to skip an off-only provider thinking profile when the selected model has no catalog reasoning hint, falling back to generic base thinking levels.

## Best Possible Solution

Land one runtime-catalog/native-menu fix, likely https://github.com/openclaw/openclaw/pull/94067, that passes discovered model reasoning metadata into `/think` menus while preserving provider-owned off-only profiles.

## Reproduction Assessment

Yes. The linked issue gives shipped 2026.6.8 Telegram/Ollama steps, current source explains the configured-catalog fallback, and inspected Mantis screenshots show the before/after menu behavior.

## Solution Assessment

No. The visible behavior improves, but this PR changes the shared resolver too broadly; the narrower maintainable fix is to feed runtime catalog metadata into native `/think` menus or scope the behavior to Ollama.
