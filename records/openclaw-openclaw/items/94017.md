## Summary

Keep open: this PR targets a real live-discovered Ollama `/think` menu bug, but the shared resolver change is too broad and can bypass provider-owned off-only thinking policies when catalog metadata is absent.

## What This Changes

This PR changes `resolveThinkingProfile` to skip an off-only provider profile when catalog reasoning is undefined for a selected model, falling through to the base thinking levels.

## Best Possible Solution

Land one runtime-catalog/native-menu fix, such as https://github.com/openclaw/openclaw/pull/94067, or an equally provider-owned Ollama policy fix that feeds discovered reasoning into `/think` menu resolution while keeping off-only provider profiles authoritative.

## Reproduction Assessment

Yes for source and visual reproduction. The linked issue gives concrete OpenClaw 2026.6.8 Telegram/Ollama steps, current main shows native menu choices falling back to configured-only catalog metadata, and Mantis screenshots show the before/after menu behavior.

## Solution Assessment

No. The visible symptom improves, but this PR changes the shared provider-profile fallback too broadly; the safer fix is to pass runtime catalog metadata into native `/think` menus or scope the exception to an Ollama-owned policy with compatibility coverage.
