## Summary

Keep this PR open, but the patch is not merge-ready: it bypasses provider-owned thinking profiles when catalog metadata is missing, so it would show generic base levels instead of the Ollama profile and can allow unsupported thinking levels for unknown non-thinking models.

## What This Changes

The PR changes `resolveThinkingProfile` so an off-only provider thinking profile can be skipped when the selected model is missing from the catalog.

## Best Possible Solution

Keep provider-owned thinking profiles authoritative and fix the missing-runtime-catalog path so `/think` menus receive the same discovered model capability metadata used by validation/current-level resolution.

## Reproduction Assessment

Yes, source-reproducible: with a selected Ollama model missing from the menu catalog, the current PR falls through from an off-only provider profile to generic base levels. I did not run a live Telegram/Ollama reproduction in this read-only review.

## Solution Assessment

No. The patch is a plausible workaround, but the best fix is to preserve provider-owned profiles and pass or hydrate runtime catalog capability metadata where the `/think` menu is resolved.
