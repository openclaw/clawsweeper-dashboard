## Summary

Keep this PR open, but it is not merge-ready: it addresses a real `/think` menu bug and has useful Telegram proof, but the patch changes the shared provider-profile fallback and can override intentionally off-only provider thinking profiles.

## What This Changes

The PR changes `src/auto-reply/thinking.ts` to skip off-only provider thinking profiles when a selected model lacks catalog reasoning metadata, falling through to generic base thinking levels.

## Best Possible Solution

Land a runtime-catalog native-menu fix, or an equally narrow provider-owned Ollama metadata fix, with regression coverage that preserves provider-owned off-only profiles.

## Reproduction Assessment

Yes. The linked issue gives shipped OpenClaw 2026.6.8 Telegram/Ollama steps, current source shows native menus can fall back to configured-only catalog metadata, and Mantis screenshots show the before/after menu behavior.

## Solution Assessment

No. The visible Ollama menu improves, but this PR changes the shared resolver too broadly; the narrower maintainable solution is to feed runtime catalog metadata into native `/think` menus or scope the behavior to the Ollama provider path.
