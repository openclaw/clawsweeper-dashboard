## Summary

Keep open: the PR improves the supplied Telegram `/think` proof case, but the patch changes the shared thinking resolver too broadly and can bypass provider-owned off-only profiles for existing models.

## What This Changes

The branch changes `src/auto-reply/thinking.ts` to skip off-only provider thinking profiles when a selected model has no catalog reasoning metadata, falling back to generic base levels.

## Best Possible Solution

Land a runtime-catalog native-menu fix, or an equally narrow provider-owned fix, with regression coverage that preserves provider-owned off-only profiles.

## Reproduction Assessment

Yes. The linked issue gives shipped OpenClaw 2026.6.8 Telegram/Ollama steps, current source shows native menus can fall back to configured-only catalog metadata, and Mantis screenshots show the before/after menu behavior.

## Solution Assessment

No. The visible Ollama menu improves, but this PR changes the shared resolver too broadly; the narrower maintainable solution is to feed runtime catalog metadata into native `/think` menus or scope the behavior to the Ollama provider path.
