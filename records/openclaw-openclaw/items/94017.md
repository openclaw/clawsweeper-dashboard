## Summary

Keep open: the PR has convincing visual proof for the reported `/think` menu symptom, but the patch changes the shared provider-profile fallback too broadly and can override provider-owned off-only thinking policies when catalog metadata is absent.

## What This Changes

The PR changes the shared thinking resolver to fall through from off-only provider profiles to base thinking levels when catalog reasoning is absent, and adds regression tests for that behavior.

## Best Possible Solution

Land one runtime-catalog/native-menu fix, such as https://github.com/openclaw/openclaw/pull/94067, or an equally provider-owned Ollama policy fix that feeds discovered reasoning into `/think` menu resolution while keeping off-only provider profiles authoritative.

## Solution Assessment
