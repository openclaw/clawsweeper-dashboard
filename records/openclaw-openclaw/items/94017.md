## Summary

Keep open, but this PR is not merge-ready: it addresses a real linked Ollama `/think` menu bug, but the shared resolver change can expose unsupported thinking levels for providers whose off-only profile is an explicit model contract, and the supplied proof is code-analysis only.

## What This Changes

The PR changes the shared thinking-profile resolver to fall through to base thinking levels when a selected model has no catalog reasoning value and the provider profile normalizes to only `off`.

## Best Possible Solution

Fix the Ollama live-discovery menu gap by passing runtime catalog reasoning metadata into `/think` choices or by narrowing Ollama's provider policy, while preserving provider-authoritative off-only profiles with regression tests.

## Reproduction Assessment

Yes from source: the linked user symptom is supported by the no-arg `/think` path using `formatThinkingLevels(...)` and Ollama returning off-only when reasoning metadata is absent. I did not run a live Ollama plus Telegram reproduction in this read-only review.

## Solution Assessment

No. The requested outcome is valid, but this PR solves it at too broad a layer; the safer fix is to preserve provider-authoritative profiles and feed real Ollama live-discovery metadata into the menu/profile decision.
