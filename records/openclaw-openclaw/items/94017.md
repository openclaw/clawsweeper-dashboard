## What This Changes

The branch adds a guard in `src/auto-reply/thinking.ts` that skips an off-only provider thinking profile when catalog reasoning is absent and falls back to generic base thinking levels.

## Best Possible Solution

Use a runtime-catalog-aware `/think` menu fix that preserves provider-authored off-only profiles, then close or supersede competing candidate PRs for the same issue.

## Reproduction Assessment

Yes from source and visual proof: native `/think` menu choices can resolve without runtime catalog metadata, while Mantis baseline proof shows only `default, off` for the selected Telegram context. I did not run a separate live Ollama reproduction locally.

## Solution Assessment

No. The requested outcome is valid, but this PR solves it at too broad a layer; the safer path is to feed runtime catalog reasoning metadata into native `/think` menu resolution or narrowly adjust Ollama policy while preserving off-only provider profiles.
