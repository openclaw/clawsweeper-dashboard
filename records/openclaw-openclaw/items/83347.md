## Summary

Keep open: current main and the latest stable release still clamp OpenClaw-sandboxed Codex app-server turns to Codex `workspace-write`, and that turn policy still sends `networkAccess: false`. The reporter's timeline correction matches the shipped changelog, and the open fix PR is a candidate to review rather than proof that this issue is resolved.

## What This Changes

The issue reports that OpenClaw-sandboxed Codex app-server turns lose outbound network access because the OpenClaw sandbox clamp maps `danger-full-access` to `workspace-write`, whose Codex turn policy denies network access.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/82452 so Codex app-server turns honor OpenClaw-managed sandbox egress while default Codex `workspace-write` remains network-denied.
