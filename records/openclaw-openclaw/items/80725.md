## Summary

Keep open. The PR addresses a plausible Gateway/SDK protocol gap, and current main still only exposes cumulative chat message snapshots, but the patch is not merge-ready because it misses generated protocol outputs, has an SDK replay edge-case regression, and provides only test/mock proof.

## What This Changes

The PR adds an optional Gateway chat `deltaText` payload field, tracks last broadcast text through Gateway run state cleanup, updates SDK chat projection, and adds docs plus focused tests.

## Best Possible Solution

Land a protocol-compatible PR that keeps `message` cumulative, adds optional `deltaText`, refreshes generated protocol artifacts, protects SDK replay semantics, and includes real Gateway/SDK behavior proof.

## Reproduction Assessment

Yes, source-level. Current main emits cumulative chat `message` snapshots and the SDK infers deltas from previous text; the PR's remaining failures are also source/CI-verifiable through the protocol check log and replay-buffer logic.

## Solution Assessment

No, not as-is. The optional field direction is maintainable, but the patch needs generated protocol outputs, a safer SDK replay fallback, and real runtime behavior proof before merge.

  - body: Adding `deltaText` to `ChatEventSchema` changes the generated protocol surface. The `checks-fast-protocol` job failed because `pnpm protocol:check` generated a new Swift `ChatEvent.deltatext` field that is not committed, so this PR cannot pass protocol validation until the generated model output is included.
  - confidence: 0.96
- **[P2] Preserve the first replayed chat snapshot:** `packages/sdk/src/client.ts:278`
  - body: When `run.events()` starts after earlier chat projection events aged out of the replay buffer, `previousChatProjectionText` is undefined even though Gateway `deltaText` is only the suffix since the last broadcast. Returning that suffix as `data.delta` drops the prefix that is still present in the cumulative `message`; use `deltaText` only after a known previous snapshot or mark the first seen projection as a full replacement.
  - confidence: 0.9
