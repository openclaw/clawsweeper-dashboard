## Summary

Keep open: current main still lacks the additive chat-delta field, while this recent PR is a focused implementation with supplied real Gateway/SDK proof, green relevant checks, and no blocking review findings found.

## What This Changes

The branch adds optional `deltaText` to Gateway chat delta payloads, tracks last broadcast text for safe suffixes and cleanup, updates SDK projection/generated Swift/docs, and adds Gateway/protocol/SDK tests.

## Best Possible Solution

Land the additive protocol field with the guarded SDK projection and focused tests once maintainer review and merge requirements finish.

## Reproduction Assessment

Yes from source inspection, though I did not run current main: current Gateway chat deltas carry only cumulative `message` snapshots, and the SDK derives `assistant.delta.data.delta` from those snapshots. The PR body supplies after-fix live output from a running Gateway and SDK WebSocket client.

## Solution Assessment

Yes. The optional `deltaText` field preserves the cumulative `message` contract, omits unsafe non-prefix replacements, and the SDK guard avoids using suffix-only deltas when replay has no previous snapshot.
