## What This Changes

The PR changes task maintenance so CLI tasks are backed only while their run context is active, updates task audit/regression tests, and adds a changelog entry.

## Best Possible Solution

Update `docs/automation/tasks.md` to document run-context-based liveness for all CLI tasks, then land the task-maintenance fix once the targeted tests and PR checks are green.

## Reproduction Assessment

Yes. Current main can reproduce with a grace-expired CLI task whose `sourceId`/`runId` has no live run context but whose `childSessionKey` is `agent:main:main`; the persistent session row keeps it `running`.

## Solution Assessment

Yes, with a docs update. Returning `hasActiveCliRun(task)` for CLI tasks is the narrow maintainable fix because gateway, agent-command, and media task paths register run contexts and finalize or clear them on completion.
