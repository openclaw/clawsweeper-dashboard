## What This Changes

The PR makes task maintenance treat all CLI task liveness as active run-context liveness, then updates the exec-approval-followup regression coverage, audit expectation, docs, and changelog.

## Best Possible Solution

Land this PR after maintainer acceptance, keeping #76233 as the separate follow-up for the disposed-runtime race.

## Reproduction Assessment

Yes. Current main can reproduce with a grace-expired CLI task whose sourceId/runId has no live run context but whose childSessionKey is agent:main:main with a persistent session-store row; the PR adds that regression path.

## Solution Assessment

Yes. Returning hasActiveCliRun(task) for all CLI tasks is the narrow maintainable fix because the current CLI task producers register and clear run contexts around task lifetime, while session-store row existence is not run-specific.
