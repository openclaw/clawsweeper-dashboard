## Summary

Keep open: the report matches the current shared block-reply path used by WhatsApp, where text block deliveries are only exact-deduped and can still send prefix-growing assistant preambles as separate outbound messages.

## What This Changes

The issue requests that WhatsApp stop sending cumulative assistant tool-call preambles as separate messages during multi-tool turns.

## Best Possible Solution

Keep WhatsApp on the existing shared reply pipeline, but make block-reply projection suppress or delta prefix-growing assistant preamble snapshots across tool-call cycles on non-editable channel sends.

## Reproduction Assessment

Yes, at source level: simulate a WhatsApp block-streamed turn with multiple assistant/tool cycles where each later assistant text is a strict prefix-superset of the prior one; current code would emit each non-identical block. I did not run a live WhatsApp account reproduction.

## Solution Assessment

Yes directionally: fixing shared embedded/block-reply delta suppression is narrower than changing WhatsApp formatting alone, because WhatsApp consumes the shared block dispatcher. The fix should preserve final replies, media/directives, and messaging-tool duplicate suppression.
