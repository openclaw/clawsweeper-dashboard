## Summary

Keep open: the report still matches current main. WhatsApp can route visible block replies through the shared dispatcher, while the embedded subscriber only exact-dedupes per assistant message and resets the relevant prefix/delta state at the next assistant message boundary.

## What This Changes

The issue asks WhatsApp block replies to stop sending cumulative assistant/tool-call preamble snapshots as separate messages during multi-tool turns.

## Best Possible Solution

Keep WhatsApp on the shared block-reply pipeline, but add turn-scoped prefix/delta handling for prefix-growing assistant preamble snapshots on non-editable channel sends while preserving final replies, media/directive metadata, and messaging-tool duplicate suppression.

## Reproduction Assessment

Yes, at source level: simulate a WhatsApp block-streamed turn with multiple assistant/tool cycles where each later assistant text is a strict prefix-superset of the prior preamble. Current main resets cross-message stream state and exact-dedupes only identical block chunks, so each growing snapshot can be sent.

## Solution Assessment

Yes, directionally: fixing shared embedded/block-reply projection is narrower than adding a WhatsApp-only formatter or broad outbound finalizer. The implementation should add focused prefix/delta suppression for this block-reply shape without changing channel configuration policy.
