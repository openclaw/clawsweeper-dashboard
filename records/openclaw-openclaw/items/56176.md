## Summary

Keep open. The underlying ACP compatibility bug is still present on current main, and this PR targets the right boundary, but the branch is conflicting, lacks contributor real behavior proof, and its guard does not fully match the pinned SDK protocolVersion contract.

## What This Changes

The PR wraps the ACP server NDJSON stream to coerce non-numeric initialize `protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Rebase or replace the PR with a current-main ACP initialize stream normalizer before `AgentSideConnection` that matches the SDK uint16 contract, adds focused regression coverage, and includes redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor date-string `protocolVersion` values currently flow from `ndJsonStream` into SDK 0.21.0 `AgentSideConnection`, whose initialize schema requires an integer 0..65535 before OpenClaw's translator runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right seam, but the predicate should match the SDK integer/range contract and the PR still needs tests plus real IDE proof.
