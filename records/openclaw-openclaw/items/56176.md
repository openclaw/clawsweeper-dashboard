## Summary

Keep open. The bug still exists on current main, and this PR is a plausible fix path, but it needs a predicate fix, focused coverage, branch refresh against the current ACP server shape, and redacted VS Code/Cursor proof before merge.

## What This Changes

The PR wraps the ACP server SDK stream in `coerceAcpStream()` to rewrite non-number initialize `protocolVersion` values to the SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Land or replace this PR with a current-main ACP initialize stream normalizer before SDK validation, matching the SDK integer/range contract and adding focused regression coverage plus redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor send `protocolVersion: "2025-11-25"`, current main passes initialize params directly into SDK validation, and SDK 0.21.1 requires a numeric integer in the 0..65535 range before OpenClaw's initialize handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right narrow seam, but the predicate should match the SDK contract and the PR still needs regression coverage plus real IDE proof.
