## What This Changes

The PR adds a `coerceAcpStream()` TransformStream in `src/acp/server.ts` to rewrite non-number `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Refresh or replace this with a current-main stream-boundary normalizer before SDK validation that accepts MCP date-string protocol versions, matches the SDK integer/range contract, adds focused regression tests, and includes redacted VS Code/Cursor proof.

## Reproduction Assessment

Yes, source-level. Current main passes `ndJsonStream` directly into `AgentSideConnection`, SDK 0.21.1 validates initialize params as a uint16 number before OpenClaw's handler runs, and the linked report shows VS Code/Cursor sending the string `2025-11-25`.

## Solution Assessment

No, not as submitted. A pre-`AgentSideConnection` stream normalizer is the right seam, but this branch must be refreshed, the guard should match the SDK integer/range contract, and the fix needs regression coverage plus real IDE proof.
