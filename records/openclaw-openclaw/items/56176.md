## Summary

Keep open. Current main still passes the raw ACP SDK stream into `AgentSideConnection`, so the VS Code/Cursor date-string `protocolVersion` bug remains; this PR is useful source work, but it is dirty against current main and needs a small correctness repair before merge.

## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` to rewrite non-number `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before constructing `AgentSideConnection`.

## Best Possible Solution

Land a rebased ACP server compatibility fix that normalizes incoming `initialize.params.protocolVersion` before SDK validation, preserves valid ACP numeric versions, adds focused regression coverage for date-string input, and credits the bug reporter plus this PR contributor in the changelog.

## Reproduction Assessment

Yes. The linked bug logs provide concrete VS Code/Cursor initialize messages with a string date `protocolVersion`, and current source plus SDK validation show that value reaches a numeric uint16 parser before OpenClaw's handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary transform is the right maintainable shape, but the PR is stale against current server bootstrap and the predicate should match the SDK contract or the code/comment/test boundary should be narrowed to string-only compatibility.

## Review Findings
