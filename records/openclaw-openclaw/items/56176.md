## Summary

## What This Changes

The PR wraps the ACP server stream to rewrite non-numeric `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validates requests.

## Best Possible Solution

Land a current-main ACP server compatibility fix at the `ndJsonStream` boundary that normalizes only SDK-invalid initialize protocol versions, preserves valid ACP numeric versions and all other messages, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The linked bug logs show VS Code/Cursor sending `protocolVersion: '2025-11-25'`, and current main plus the pinned SDK schema show that value reaches numeric uint16 validation before OpenClaw's initialize handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary transform is the narrow maintainable seam, but it should use the SDK-equivalent integer/range predicate and be ported with tests and changelog coverage.
