## Summary

Keep open. The PR targets a valid ACP bridge bug that is still present on current main and latest release, but the submitted patch has a predicate mismatch with the SDK contract and lacks required after-fix IDE proof.

## What This Changes

The PR wraps the ACP server stream with a transform that coerces non-number initialize `protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Refresh or replace the PR with a current-main ACP initialize stream normalizer before `AgentSideConnection` that matches SDK integer/range validation, adds focused regression coverage, and includes redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor send `protocolVersion: "2025-11-25"`, current main passes initialize params directly into SDK validation, and SDK `0.21.1` requires an integer in the 0..65535 range before OpenClaw's initialize handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right narrow seam, but the predicate should match the SDK contract and the PR still needs regression coverage plus real IDE proof.
