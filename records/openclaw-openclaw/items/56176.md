## What This Changes

The PR wraps the ACP server NDJSON stream so initialize messages with non-number `protocolVersion` values are replaced with the ACP SDK `PROTOCOL_VERSION` before SDK validation.

## Best Possible Solution

Rebase or replace the branch with a current-main ACP initialize normalizer before SDK validation that matches the SDK uint16 contract, plus focused tests and redacted VS Code/Cursor proof.

## Reproduction Assessment

Yes, by source inspection. A VS Code/Cursor date-string `protocolVersion` reaches SDK 0.21.0 initialize validation before OpenClaw's translator runs, and that SDK requires an integer in the 0..65535 range.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right seam, but the predicate should match the SDK integer 0..65535 contract and the PR still needs tests plus real IDE proof.
