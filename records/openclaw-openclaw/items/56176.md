## Summary

Keep open. The ACP handshake bug is still present on current main, and this PR is aimed at the right stream boundary, but the submitted patch is conflicting/outdated, misses the SDK numeric range contract, and lacks contributor real behavior proof.

## What This Changes

The PR wraps the ACP NDJSON stream in `src/acp/server.ts` so initialize messages with string `protocolVersion` values are coerced to the ACP SDK `PROTOCOL_VERSION`.

## Best Possible Solution

Rebase or replace the PR with a current-main stream-boundary normalizer before `AgentSideConnection` that coerces SDK-invalid initialize `protocolVersion` values to `PROTOCOL_VERSION`, with focused tests and redacted VS Code/Cursor proof.

## Reproduction Assessment

Yes, by source inspection. A date-string initialize `protocolVersion` currently flows from `ndJsonStream` into SDK 0.21.0 `AgentSideConnection`, whose schema requires an integer 0..65535 before OpenClaw's translator runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right seam, but the predicate should match the SDK integer and 0..65535 contract and the PR still needs tests plus real IDE proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87
