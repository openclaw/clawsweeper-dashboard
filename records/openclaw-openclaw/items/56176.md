## Summary

Keep open. Current main still lacks the ACP stream compatibility layer for date-string initialize protocol versions, and this PR is a plausible fix path, but it is not merge-ready because its guard does not match the pinned SDK contract and the contributor has not provided real after-fix VS Code/Cursor proof.

## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` to rewrite non-numeric `initialize.params.protocolVersion` values before passing ACP stdio messages to the SDK connection.

## Best Possible Solution

Port a current-main ACP initialize stream normalizer before `AgentSideConnection` that only rewrites SDK-invalid `protocolVersion` values, add focused regression coverage, verify with VS Code or Cursor, then close https://github.com/openclaw/openclaw/issues/56102 after merge.

## Reproduction Assessment

Yes, by source inspection. Current main feeds raw `ndJsonStream` into `AgentSideConnection`, while SDK 0.21.0 validates `initialize.params.protocolVersion` as a uint16 number before OpenClaw's translator can return the ACP protocol version.

## Solution Assessment

No, not as submitted. A stream-boundary transform is the right narrow seam, but the predicate should match the SDK integer/range contract and the PR still needs focused tests plus real after-fix proof.
