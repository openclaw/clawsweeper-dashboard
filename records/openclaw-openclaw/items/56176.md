Keep open. The linked bug is source-reproducible on current main, and this PR is the active fix path, but the submitted patch still needs real VS Code/Cursor proof, a current-main rebase, and a small predicate/test correction before merge.

## What This Changes

The PR adds a `coerceAcpStream()` transform in `src/acp/server.ts` and wraps the ACP NDJSON stream so non-number initialize `protocolVersion` values are replaced with the ACP SDK `PROTOCOL_VERSION`.

## Best Possible Solution

Rebase onto current main, normalize SDK-invalid initialize `protocolVersion` values before `AgentSideConnection` using the SDK integer/range contract, add focused tests, and provide redacted IDE proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor send `protocolVersion: "2025-11-25"`, current main passes initialize directly into SDK 0.21.0, and that SDK requires a numeric uint16 before `AcpGatewayAgent.initialize()` runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right seam, but the predicate should match `zProtocolVersion` and the PR still needs regression coverage plus real VS Code/Cursor proof.
