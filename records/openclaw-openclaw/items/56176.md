## Summary

Keep open. Current main still lacks a pre-validation ACP initialize normalizer, and this PR targets the right boundary, but the submitted predicate does not fully match the ACP SDK protocolVersion contract and the external contributor real-behavior proof is still missing.

## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` to rewrite initialize messages with non-number `protocolVersion` values before passing the NDJSON stream to `AgentSideConnection`.

## Best Possible Solution

Use a current-main ACP initialize normalizer before SDK validation that rewrites any `protocolVersion` outside the SDK integer 0..65535 contract, with focused regression coverage and redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor can send `protocolVersion: "2025-11-25"`, current main passes initialize params into SDK 0.21.0 before OpenClaw's translator runs, and that SDK requires a numeric uint16 version.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the narrow maintainable seam, but the predicate should match the SDK integer 0..65535 contract and the PR still needs regression coverage plus real IDE proof.
