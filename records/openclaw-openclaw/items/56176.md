## Summary

Keep open. The linked bug is still valid on current main and this PR targets the right ACP stream boundary, but the submitted branch is conflicting/outdated, lacks after-fix VS Code or Cursor proof, and has a predicate that does not fully match the SDK `protocolVersion` contract.

## What This Changes

The PR adds a `coerceAcpStream()` wrapper around ACP `ndJsonStream()` so `initialize.params.protocolVersion` values that are not numbers are rewritten to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Refresh or replace the PR with a current-main ACP initialize normalizer before `AgentSideConnection`, match the SDK integer/range contract, add focused regression coverage, and include redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor send `protocolVersion: "2025-11-25"`, current main passes initialize params directly into SDK validation, and SDK `0.21.1` requires an integer in the 0..65535 range before OpenClaw's initialize handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right narrow seam, but the predicate should match the SDK integer/range contract and the PR still needs regression coverage plus real IDE proof.
