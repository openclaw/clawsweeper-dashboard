## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` to rewrite non-number `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Refresh or replace the PR on current main with a pre-`AgentSideConnection` initialize normalizer that accepts MCP date-string protocol versions, matches the SDK uint16 contract, adds regression coverage, and includes redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, by source inspection. VS Code/Cursor send `protocolVersion: "2025-11-25"`, current main passes initialize params directly into SDK validation, and SDK `0.21.1` requires a numeric integer in the 0..65535 range before OpenClaw's initialize handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right narrow seam, but the predicate should match the SDK integer/range contract and the branch still needs regression coverage plus real IDE proof.
