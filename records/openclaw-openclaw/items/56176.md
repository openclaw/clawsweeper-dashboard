## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` that rewrites non-number ACP initialize `protocolVersion` values to the SDK `PROTOCOL_VERSION` before `AgentSideConnection` validation.

## Best Possible Solution

Refresh or replace this PR on current main with a pre-`AgentSideConnection` initialize normalizer that accepts MCP date-string protocol versions, matches the SDK integer/range contract, adds focused regression coverage, and includes redacted IDE handshake proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: VS Code/Cursor send a string `protocolVersion`, current main passes initialize params directly into SDK validation, and SDK 0.21.1 requires an integer before OpenClaw's handler runs.

## Solution Assessment

No, not as submitted. A stream-boundary normalizer before `AgentSideConnection` is the right seam, but the guard should match the SDK integer/range contract and the branch still needs regression coverage plus real IDE proof.
