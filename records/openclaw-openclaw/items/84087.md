## Summary

Keep open for changes: the duplicate-Off report is valid, but this patch removes the explicit Off override when the inherited default is non-off and it has only test-level proof rather than real UI proof.

## What This Changes

This branch skips normalized `off` thinking levels in the chat selector when the current override is empty and updates chat UI tests to expect a single Off row.

## Best Possible Solution

Keep the duplicate-Off fix, but make suppression depend on the effective inherited default being Off and preserve explicit Off for non-off defaults, with regression coverage and real UI proof.

## Reproduction Assessment

Yes from source inspection: current main renders the inherited row plus normalized dynamic options, and existing tests show the duplicate Off shape when thinkingLevels include off. I did not run the UI because this review is read-only.

## Solution Assessment

No; filtering solely on `currentOverride` is too broad because an empty override can also mean inheriting High or another non-off default. The safer fix is to pass the effective default into the decision and suppress `off` only when the inherited row already represents Off.
