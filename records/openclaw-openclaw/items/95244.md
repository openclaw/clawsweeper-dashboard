## Best Possible Solution

Keep the shared bounded-reader implementation and land it once maintainers accept the 4 MiB provider discovery limit as the intended compatibility boundary.

## Reproduction Assessment

## Solution Assessment

Yes. The PR uses the existing media-core bounded reader at the two untrusted JSON read sites; the maintainer choice is whether 4 MiB is the right supported cap for legitimate self-hosted catalogs.
