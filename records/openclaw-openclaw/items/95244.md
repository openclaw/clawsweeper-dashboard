## What This Changes

The PR replaces self-hosted provider `/models` and llama.cpp `/props` `response.json()` parsing with a 4 MiB bounded reader and adds overflow/cancel regression tests.

## Best Possible Solution

Land the bounded-reader implementation once maintainers accept 4 MiB as the provider discovery limit, or adjust that cap before merge if supported catalogs can exceed it.

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer-owned limit choice. The PR applies the existing bounded reader to the exact untrusted JSON reads; the remaining decision is whether 4 MiB is the right supported cap.
