## Summary

Keep open for maintainer review: the patch is a focused, source-backed fix for unbounded self-hosted provider discovery body reads, with no concrete correctness finding, but the new 4 MiB fail-closed discovery boundary is compatibility-sensitive.

## What This Changes

The PR replaces self-hosted provider `/models` and llama.cpp `/props` `response.json()` calls with a 4 MiB bounded reader and adds unbounded-stream regression tests.

## Best Possible Solution

Land the shared bounded-reader implementation once maintainers accept 4 MiB as the provider discovery limit, or adjust the cap/docs if supported self-hosted catalogs can exceed it.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits `response.json()` at the `/models` and `/props` paths, and the PR's live loopback proof plus streamed tests model the oversized body failure mode.

## Solution Assessment

Yes, with a maintainer caveat. The PR uses the existing media-core bounded reader at the two untrusted JSON read sites; the remaining decision is whether 4 MiB is the right supported cap for legitimate self-hosted catalogs.
