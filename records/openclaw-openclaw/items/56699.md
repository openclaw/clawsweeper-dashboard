## Summary

Keep open. Current main still routes Telegram native command authorization through the account/default group policy path and skips topic/group `groupPolicy` overrides, so the reported forum-topic bypass remains present. The issue is already paired with open PR #56741 using closing syntax, so it should stay open until that fix or an equivalent replacement lands.

## What This Changes

The issue requests Telegram native slash commands in forum topics to respect per-topic or per-group `groupPolicy: "disabled"` so non-assigned bot accounts ignore commands in disabled topics.

## Best Possible Solution

Land #56741 or an equivalent narrow Telegram plugin fix, and add regression coverage that resolves a forum-topic `topicConfig.groupPolicy: "disabled"` or inherited group-level disabled policy for a native command while the account-level policy would otherwise allow it.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram forum steps, and current main statically reproduces the bypass because native command auth resolves topic config but tells the shared policy helper not to use topic/group overrides.

## Solution Assessment

Yes. Reusing the existing override-aware policy resolver for native commands is the narrow maintainable fix, with focused regression coverage for disabled forum-topic policy.
