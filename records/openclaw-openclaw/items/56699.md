## Summary

Keep open. Current main and the latest release still route Telegram native commands through policy evaluation with topic/group `groupPolicy` overrides disabled, and there is an open fixing PR with closing syntax that should be reviewed or replaced before this issue is closed.

## What This Changes

Make Telegram native slash commands in forum topics respect per-topic and per-group `groupPolicy: "disabled"` so non-owning bots do not process the command.

## Best Possible Solution

Make native Telegram commands use the topic/group-aware policy path, add focused regression coverage for effective disabled topic/group policy, and settle disabled-topic command response semantics intentionally.

## Reproduction Assessment

Yes. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands; source inspection shows current main skips that override for native commands.

## Solution Assessment

Partly. Reusing the existing override-aware resolver is the narrow authorization fix, but the final PR should also cover regression tests, live Telegram proof, and the intended silent-drop versus disabled-reply behavior.
