## Summary

Current main still has the reported source-level behavior: native Telegram commands pass topic/group config into the policy helper but disable topic/group override evaluation. Because an open PR explicitly references this issue with closing syntax, this issue should stay open until that PR is reviewed, repaired, or closed.

## What This Changes

Telegram native commands in forum topics should honor topic-level `groupPolicy: "disabled"` so only the intended bot handles a command.

## Best Possible Solution

Land or replace the paired Telegram PR with a topic/group-aware native-command policy fix, focused forum-topic regression coverage, settled disabled-topic reply semantics, and real Telegram proof.

## Reproduction Assessment

Yes for source-level reproduction: configure a Telegram forum topic with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands, then send a native command; current main passes `useTopicAndGroupOverrides: false` and skips the topic override. I did not run live Telegram, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not fully yet. Flipping the flag is the narrow authorization fix, but the final solution should also decide whether disabled topic commands must be silently dropped like regular messages and include regression plus live Telegram proof.
