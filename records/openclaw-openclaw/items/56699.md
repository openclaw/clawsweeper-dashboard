## Summary

Keep open: current main and v2026.6.6 still skip topic/group `groupPolicy` overrides for Telegram native commands, and the linked fixing PR remains open rather than merged.

## What This Changes

The issue reports that Telegram native slash commands in forum topics bypass effective topic/group `groupPolicy: "disabled"`, causing non-owning bots to reply and create stale topic sessions.

## Best Possible Solution

Land or replace the linked Telegram fix with native-command authorization that uses the same effective topic/group policy as regular messages, silently ignores disabled topics for non-owning bots, and adds focused regression coverage plus real Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main still sets `useTopicAndGroupOverrides: false` for Telegram native-command group policy evaluation while regular group messages set it to true; I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

No merged fix is present yet. The best fix is narrow and belongs in Telegram native-command authorization, but it still needs regression coverage and real Telegram/forum proof before the issue can close.
