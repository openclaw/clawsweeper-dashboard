## Summary

Keep open: current main and the latest release still leave Telegram native slash-command authorization on the account/default policy path instead of the effective topic/group policy path. The closed same-repo fix PR identified the right root cause but was closed unmerged as incomplete, and the open Telegram callback PR covers a different authorization path rather than native slash commands in disabled forum topics.

## What This Changes

The issue reports that Telegram native slash commands in forum topics ignore topic-level `groupPolicy: "disabled"`, causing every bot in the supergroup to process commands for topics it should ignore.

## Best Possible Solution

Land a narrow Telegram plugin fix that makes native slash commands use the same effective topic/group policy as regular messages and silently ignores disabled-topic commands, with focused regression tests and real Telegram forum proof.

## Reproduction Assessment

Yes, source-level: current main passes `useTopicAndGroupOverrides: false` for Telegram native-command policy evaluation while topic `groupPolicy` is only read when that flag is true. I did not run a live Telegram forum repro in this read-only review.

## Solution Assessment

Yes for the fix direction, but not the previously closed one-line PR as-is. The maintainable solution should align native commands with regular message policy resolution and preserve silent ignore behavior for disabled topics.
