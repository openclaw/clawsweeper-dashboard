## Summary

Keep open. Current main still skips topic/group `groupPolicy` overrides for Telegram native commands, and an open linked fix PR with closing syntax should be reviewed or replaced before this issue is closed.

## What This Changes

The issue reports that Telegram native slash commands in forum topics ignore disabled topic policy and let non-owning bots process the command.

## Best Possible Solution

Make Telegram native commands use the topic/group-aware policy path, add regression coverage for disabled effective topic/group policy, and land or replace the linked PR with real Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure a forum topic or inherited group override with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands, then send a native slash command. Current main passes the topic config into the policy helper but disables topic/group override resolution for native commands.

## Solution Assessment

Partly. Reusing the existing override-aware resolver is the narrow fix direction, but the final change should include regression coverage, real Telegram proof, and silent disabled-topic behavior for non-owning bots.
