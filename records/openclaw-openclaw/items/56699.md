## Label Justifications

- `P1`: The issue affects live Telegram multi-bot forum workflows where commands can reach unauthorized or non-owned bots.
- `impact:security`: The report is about a Telegram authorization policy bypass for topic/group `groupPolicy: disabled`.
- `impact:message-loss`: The affected slash commands can be delivered to and answered by bots that should ignore that forum topic.
- `impact:session-state`: The report says wrong-bot command handling creates stale Telegram group-topic sessions.

## Summary

Keep open: current main and v2026.6.6 still skip topic/group `groupPolicy` overrides for Telegram native commands, and the issue is already paired with an open linked fix PR rather than a merged implementation.

## What This Changes

Telegram native slash commands in forum topics should honor topic/group `groupPolicy: "disabled"` so non-owned bots do not process commands or create sessions.

## Best Possible Solution

Land or replace the focused Telegram native-command authorization fix so commands use the effective topic/group policy path, preserve disabled-topic silent-ignore behavior for non-owning bots, and include regression coverage plus real Telegram forum proof.

## Reproduction Assessment

Yes, at source level. Current main still sets `useTopicAndGroupOverrides: false` in Telegram native-command group policy evaluation while the regular message path sets it to true.

## Solution Assessment

The requested direction is correct, but the best implementation is not merged yet. The final fix should use the topic/group-aware policy path and make a maintainer-visible choice about silent ignore versus visible denial for disabled-topic native commands.
