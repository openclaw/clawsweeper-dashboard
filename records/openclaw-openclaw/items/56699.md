- `P1`: The issue reports a concrete Telegram authorization bypass that can trigger wrong bots and sessions in real multi-bot forum groups.

## Summary

Keep open. Current main and v2026.5.22 still skip topic/group groupPolicy overrides for Telegram native commands, and https://github.com/openclaw/openclaw/pull/56741 is open with closing syntax as the active fix path.

## What This Changes

The issue asks Telegram native commands in forum topics to honor per-topic groupPolicy disabled settings so non-owned bots do not respond.

## Best Possible Solution

Land or replace the linked PR with a narrow fix that makes native commands use effective topic/group policy, silently ignores disabled-topic commands for non-owning bots, and adds focused regression coverage plus redacted Telegram forum proof.

## Reproduction Assessment

Yes, at source level. Configure a Telegram forum topic or inherited group override with groupPolicy disabled while the account/default policy would otherwise allow commands; current main passes the topic config into the helper but disables topic/group override resolution for native commands.

## Solution Assessment

Yes, the best direction is a narrow authorization-path fix that reuses the existing override-aware policy helper and preserves silent drops for disabled topics, rather than adding a new config mode.
