- `impact:security`: Native commands can bypass configured topic-level access policy for Telegram forum groups.
- `impact:message-loss`: Commands in one topic can be processed by the wrong bots, producing duplicated or misrouted channel responses.
- `impact:session-state`: The report describes stale topic-qualified sessions being created for bots that should have ignored the topic.

## Summary

Keep open: current main and v2026.6.8 still make Telegram native commands skip topic/group `groupPolicy` overrides, while the duplicate issue is closed and the same-repo fix PR was closed unmerged as incomplete.

## What This Changes

Make Telegram native slash commands in forum topics respect topic/group `groupPolicy: "disabled"` so non-owning bots ignore commands in those topics.

## Best Possible Solution

Land a narrow Telegram plugin fix that applies topic/group policy overrides for native commands and silently drops disabled-topic commands, with regression coverage and real Telegram forum proof.

## Reproduction Assessment

Yes, source-level. Current main passes `useTopicAndGroupOverrides: false` for native command policy evaluation while the helper only honors topic/group `groupPolicy` overrides when that flag is true; I did not run a live Telegram forum repro in this read-only review.

## Solution Assessment

Yes for the fix direction, but not for the closed one-line PR as-is. The maintainable solution is to align native-command policy lookup with regular message gating while preserving silent drops for disabled topics and adding focused proof.
