## Summary

Keep open. Current main still lets Telegram native command authorization skip topic/group `groupPolicy` overrides, and the issue is already paired with an open fixing PR using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Telegram native slash commands in forum topics ignore per-topic or group-level `groupPolicy: "disabled"` and are handled by bots that should be disabled for that topic.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/56741 or an equivalent Telegram plugin fix with regression coverage for effective topic/group `groupPolicy: "disabled"`, then resolve the silent-drop versus disabled-reply behavior intentionally.

## Reproduction Assessment

Yes. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands; current main passes `useTopicAndGroupOverrides: false`, so the resolver skips that override for native commands.

## Solution Assessment

Partly. Reusing the existing override-aware resolver is the narrow fix for the access-control bypass, but the complete user-visible behavior may need a maintainer decision on silent drops versus the current disabled-command reply.
