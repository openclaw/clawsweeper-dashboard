## Summary

Keep open for maintainer handling: the patch is a focused security hardening of the voice-call read-scoped status response, but it intentionally narrows a shipped status payload shape and the PR has the protected `maintainer` label.

## What This Changes

The PR projects live voice-call status responses to a limited lifecycle DTO for `voicecall.status` and `voice_call` tool status, with regression tests for redaction.

## Best Possible Solution

Adopt the status DTO as the read-scoped voice-call contract, and keep any full-record debugging/history access on a separate explicit owner-approved surface if maintainers still want it.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 return raw `CallRecord` objects from read-scoped status paths, and `CallRecord` includes phone numbers, session keys, transcripts, processed event ids, and metadata.

## Solution Assessment

Yes, with a maintainer compatibility call. Projecting at the voice-call plugin boundary is narrower than changing manager storage or active-call lookup semantics, and docs do not promise raw status response fields.
