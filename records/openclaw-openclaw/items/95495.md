## Label Justifications

- `P1`: The latest stable upgrade can leave memory_search paused for users with existing corpora until they notice and pay the full re-embedding cost.
- `impact:data-loss`: The existing indexed memory dataset remains on disk but is silently dropped from the active runtime path after upgrade.
- `impact:session-state`: The active agent memory index state drifts from the user's persisted corpus and reports missing identity in the new per-agent store.

## Summary

Keep open: v2026.6.9 and current main still route built-in memory to the per-agent SQLite database, while the visible migration path removes legacy store config but does not import an existing separate legacy index. The discussion adds useful confirmation from another affected user that old indexes remained on disk and downgrading restored the old path behavior.

## What This Changes

The issue asks OpenClaw to preserve, migrate, or warn about existing memory vector indexes when the v2026.6.9 upgrade moves built-in memory storage into the per-agent SQLite database.

## Best Possible Solution

Add a doctor/update-owned migration or explicit upgrade warning for existing legacy memory DBs before the per-agent DB cutover completes, with regression coverage for an already-indexed corpus and no permanent runtime fallback reader.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: the release and current main resolve built-in memory to the per-agent DB, while the visible migration removes legacy path config and does not import a separate old index DB. The reporter supplied concrete before/after upgrade output and a second commenter confirmed old indexes remained and downgrade restored behavior.

## Solution Assessment

No, current main is not the best solution because it documents and uses the new per-agent DB but does not migrate or warn about an existing separate legacy memory index before the old corpus disappears from active search. The best fix belongs in doctor/update migration or warning code, not a permanent runtime dual-reader.
