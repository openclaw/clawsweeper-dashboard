## Summary

Keep open: current main and shipped v2026.6.9 still route built-in memory to the per-agent SQLite database, while the current schema migration only imports legacy tables inside the database already opened. The linked open fix PR is the right current implementation candidate, but it is not merged, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests a migration or clear upgrade warning for legacy built-in memory sidecar indexes so upgrading to v2026.6.9 does not force a full re-embed.

## Best Possible Solution

Land or replace the linked migration PR with a source-of-truth import path that preserves an existing legacy memory sidecar when the new per-agent memory index is empty, plus regression coverage and clear upgrade messaging.

## Reproduction Assessment

Yes. Current main and v2026.6.9 source show the active memory path moving to the per-agent database while the legacy migration only copies tables in the opened DB; the issue thread adds two field confirmations that old sidecar indexes remain outside the active path.

## Solution Assessment

Unclear for the open fix path. Importing the legacy sidecar at schema initialization appears to be the right owner boundary, but maintainers still need to review the linked PR's upgrade policy before this issue can close.
