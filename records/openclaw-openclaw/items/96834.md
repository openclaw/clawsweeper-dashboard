## Best Possible Solution

Land a focused owner-boundary fix that either clears stale active reply ownership after terminal multimodal run completion or lets recovery reclaim that stale owner without weakening protection for genuinely active runs.

## Reproduction Assessment

Unclear. Both suggested directions are plausible, but the best fix should stay at the reply-owner cleanup or diagnostic recovery liveness boundary and include deterministic regression coverage for terminal multimodal embedded-run completion with queued work.
