## What This Changes

The PR adds compaction identifier extraction from message text, tool-call arguments, and tool-result content, retries strict summaries when lost identifiers exceed a threshold, adds tests, and adds a changelog entry.

## Best Possible Solution

Revise the guard to validate the final returned summary against the full pre-compaction identifier set, exclude stripped toolResult.details, retry strict summaries on any lost identifier, and cover pruning and capping paths with regression tests plus real behavior proof.

## Solution Assessment

No, not yet. The extraction helper is useful, but the maintainable fix needs full pre-compaction source coverage, any-loss strict retries, and final-summary validation before merge.
