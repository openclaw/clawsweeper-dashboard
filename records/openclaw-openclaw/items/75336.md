This PR adds transcript-wide identifier extraction helpers, strict-mode lost-identifier retry hints, focused compaction safeguard tests, and a changelog entry.

## Best Possible Solution

Repair the compaction guard so the survival set comes from the full pre-prune input, strict mode retries on any lost identifier, and the final capped returned summary is validated with regression coverage.

Yes. Source inspection of current main and PR head gives a high-confidence path: put identifiers only in tool calls, pruned messages, one or two strict losses, or final-capped content and the current/PR code paths do not reliably retry or validate them.

## Solution Assessment

No, not yet. The extraction-helper direction is maintainable, but the patch needs full-transcript source selection, any-lost strict retries, and final-return validation before it is the best fix.

Full review comments:

- **[P2] Include pruned history in identifier checks:** `src/agents/pi-hooks/compaction-safeguard.ts:1153-1156`
  - body: Build the survival set before oversized-history pruning and recent-turn splitting. As written, identifiers that exist only in dropped messages depend on the dropped LLM summary, and if that summary omits them the retry path never sees the source value.
  - confidence: 0.91
- **[P2] Retry strict summaries on every lost identifier:** `src/agents/pi-hooks/compaction-safeguard.ts:1262`
  - body: Strict compaction is documented as preserving literal identifiers, but this accepts one or two missing values because `identifierSurvivalFailed` only becomes true when more than two are lost. A single run ID, URL, commit, or session key can be the exact value the next turn needs.
  - confidence: 0.9
- **[P2] Validate the final returned summary:** `src/agents/pi-hooks/compaction-safeguard.ts:1260`
  - body: This checks `summaryWithoutPreservedTurns`, before the returned summary is rebuilt and capped. A long summary can pass the new check and then lose trailing `## Exact identifiers` content during final capping.
  - confidence: 0.84
