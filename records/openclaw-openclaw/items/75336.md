## Summary

Keep this PR open, but it is not merge-ready: the identifier-survival direction is useful, yet the new all-transcript scan still has a credential-redaction gap and the supplied proof is a helper-script terminal run rather than a real OpenClaw compaction run.

## What This Changes

The branch extends compaction safeguards to extract identifiers transcript-wide, retry strict summaries when identifiers are lost, redact credential-shaped extraction inputs, cancel strict compaction on post-cap recovery overflow, and add regression tests.

## Best Possible Solution

Land a version that keeps identifier survival validation, routes every extracted source through a credential-safe parser/redactor before exact preservation, and includes redacted real OpenClaw compaction proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows raw text blocks are scanned, pure hex values are accepted as identifiers, and strict recovery can append missing identifiers. I did not run a live compaction because this review is read-only.

## Solution Assessment

No. The compaction safeguard is the right layer, but the best fix must make identifier extraction credential-safe for structured arguments and raw textual JSON/key-value content before strict retry or recovery can preserve values.
