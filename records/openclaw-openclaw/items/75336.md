## Summary

Keep open. The branch targets a real compaction bug, but it still validates only a post-prune, pre-cap summary and accepts one or two lost identifiers under strict policy, so the central identifier-loss failure remains possible; contributor real behavior proof is also still missing.

## What This Changes

The PR adds compaction identifier extraction from message text, tool-call arguments, and tool-result content, retries strict summaries when more than two identifiers are lost, adds tests, and adds a changelog note.

## Best Possible Solution

Revise the guard to validate the final returned summary against the full pre-compaction source set, retry on any strict-policy lost identifier, exclude stripped toolResult.details, and cover pruning/capping paths with tests and real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main seeds strict identifier auditing from a sampled text-only view, and the PR still has source-reproducible loss paths through pruned dropped messages, the strict threshold, and final capping.

## Solution Assessment

No, not yet. The extraction helper is directionally useful, but the maintainable fix needs full pre-compaction source coverage, any-loss strict retries, and final-summary validation before merge.
