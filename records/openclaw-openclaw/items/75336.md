## Summary

Keep open: current main still lacks transcript-wide identifier survival validation, and this PR is a useful narrow direction, but the patch misses identifiers from pruned dropped-message chunks and is currently dirty against main.

## What This Changes

The PR adds compaction identifier extraction helpers, a strict-policy retry path for lost transcript identifiers, safeguard tests, and a changelog entry.

## Best Possible Solution

Seed identifier survival from the full pre-prune summarization input or explicitly include droppedMessagesList, keep stripped details excluded, add a pruning regression test, then rebase and run the changed gate.

## Reproduction Assessment

Yes. Force pruneHistoryForContextShare with a low maxHistoryShare, put more than two opaque identifiers only in dropped older messages, mock the dropped and main summaries to omit them, and the PR will not retry because it scans only kept messages.

## Solution Assessment

No, not yet. The retry design fits the existing quality guard, but the identifier source must cover pruned dropped messages before it fully solves full-transcript survival.
