## Summary

Keep open: this is a valid compaction/session-state bug, but the current PR still has source-level strict identifier loss paths and only unit/typecheck proof, so it is not ready to merge.

## What This Changes

The PR adds compaction identifier extraction across message text, tool calls, and tool results, checks lost identifiers after summarization, retries strict summaries with hints when more than two are lost, adds tests, and updates the changelog.

## Best Possible Solution

Validate the final returned summary against the full pre-compaction identifier set, exclude stripped toolResult.details, retry strict summaries on any lost identifier, cover pruning and capping regressions, then add redacted real compaction proof.

## Reproduction Assessment

Yes. Source inspection shows current main can miss identifiers in tool-call arguments and older/pruned transcript content, while the PR head still has source-reproducible loss paths through pruned messages, the strict threshold, and final capping.

## Solution Assessment

No, not yet. The helper direction is useful, but the maintainable fix needs full-source coverage, any-loss strict retries, and validation of the final capped summary before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Include pruned history in identifier survival checks:** `src/agents/pi-hooks/compaction-safeguard.ts:1153-1156`
  - body: The survival set is computed from messagesToSummarize after oversized-history pruning and recent-turn splitting, plus turnPrefixMessages. If an identifier only exists in pruned.droppedMessagesList and that dropped-summary pass omits it, lostIdentifiers never sees it, so the retry does not fire.
  - confidence: 0.91
- **[P2] Retry strict summaries on any lost identifier:** `src/agents/pi-hooks/compaction-safeguard.ts:1262`
  - body: Strict identifier preservation should not accept one or two missing opaque identifiers. A single run ID, commit SHA, URL, or session ID can be the exact value the next turn needs, and the existing strict audit fails on any supplied missing identifier.
  - confidence: 0.89
- **[P2] Validate identifiers after final summary capping:** `src/agents/pi-hooks/compaction-safeguard.ts:1305`
  - body: The PR checks summaryWithoutPreservedTurns before the returned summary is rebuilt and capped. A long summary can pass, then lose a trailing Exact identifiers section when capCompactionSummaryPreservingSuffix truncates the body.
  - confidence: 0.82
