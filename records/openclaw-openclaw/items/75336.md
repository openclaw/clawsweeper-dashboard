## What This Changes

The PR adds compaction identifier extraction across transcript messages, compares extracted identifiers against generated summaries, retries strict summarization with preservation hints when more than two identifiers are lost, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the helper direction, but validate the final returned summary against the full pre-compaction identifier set, retry strict mode on any lost identifier, cover pruned history and final capping with tests, and resolve the branch conflicts.
