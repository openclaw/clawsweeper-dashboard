## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Include pruned history in identifier checks:** `src/agents/pi-hooks/compaction-safeguard.ts:1153-1156`
  - body: The survival set is built from `messagesToSummarize` and `turnPrefixMessages` after oversized-history pruning and recent-turn splitting. If an identifier exists only in dropped messages and the dropped-summary pass omits it, this check never sees it and the retry cannot fire.
  - confidence: 0.91
- **[P2] Retry strict summaries on every lost identifier:** `src/agents/pi-hooks/compaction-safeguard.ts:1262`
  - body: Strict compaction is documented as preserving literal identifiers, but this patch accepts one or two missing values because `identifierSurvivalFailed` only becomes true when more than two are lost. A single run ID, URL, commit, or session key can be the exact value the next turn needs.
  - confidence: 0.9
- **[P2] Validate the final returned summary:** `src/agents/pi-hooks/compaction-safeguard.ts:1260`
  - body: This computes lost identifiers against `summaryWithoutPreservedTurns`, before the returned summary is rebuilt and capped. A long summary can pass the new check and then lose trailing `## Exact identifiers` content during final capping.
  - confidence: 0.84
