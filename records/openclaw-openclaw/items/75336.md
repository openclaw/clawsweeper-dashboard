## Summary

Keep open: current main still lacks identifier-survival validation, and the PR is useful, but the patch is not merge-ready because the new survival set misses identifiers moved into pruned dropped-message chunks and the branch is dirty against current main.

## What This Changes

The PR adds compaction identifier extraction helpers, retries strict quality-guard summarization when more than two transcript identifiers are missing, extends safeguard tests, and adds a changelog entry.

## Best Possible Solution

Rebase the PR, seed identifier survival from the full pre-prune summarization input or explicitly include droppedMessagesList identifiers, keep stripped details excluded, add a pruning regression test, and land after the normal changed gate.

## Reproduction Assessment

Yes. The PR’s tests reproduce the base gap where a strict summary can omit more than two identifiers and still pass the older audit; the remaining bug can be reproduced by forcing pruneHistoryForContextShare so identifiers only in pruned.droppedMessagesList are omitted from the final summary without triggering the new retry.

## Solution Assessment

No, not yet. The overall retry approach is narrow and fits the existing quality guard, but it needs to cover pruned dropped messages before it fully solves full-transcript identifier survival.
