The PR changes `sendCompactionNotice` to emit default-visible `console.warn` messages when `onBlockReply` is missing or notice delivery fails.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `agents.defaults.compaction.notifyUser`, hit a compaction event without `opts.onBlockReply`, and the helper returns before delivery; preflight compaction also calls `compactEmbeddedAgentSession` directly without this notice path.

## Solution Assessment

No. Logging makes one skipped-notice branch observable, but the narrow maintainable fix is to reuse canonical session/outbound delivery and cover preflight compaction rather than leaving user notices undelivered.
