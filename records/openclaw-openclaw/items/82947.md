- **[P2] Initialize settle timers before signaling:** `src/hooks/gmail-watcher.ts:151`
  - body: This kill call happens before escalation and finalTimeout are initialized, but the shared settle handler clears both bindings. If the process emits error/exit/close during the signal path, settle() hits the temporal-dead-zone and rejects the stop or re-entry cleanup instead of completing it; Node documents that ChildProcess can emit error when a signal cannot be delivered. Initialize the timeout handles before any signal or settle path can run. ([nodejs.org](https://nodejs.org/download/release/v22.19.0/docs/api/child_process.html))
