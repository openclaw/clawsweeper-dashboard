## What This Changes

The PR adds a Gmail watcher re-entry guard that clears the renewal interval, terminates an existing `gog serve` child before setup, and adds focused unit coverage.

## Best Possible Solution

Track and retire Gmail watcher restart ownership with a stored timeout or generation token, then land with unit coverage plus redacted terminal/log proof of a repeated Gmail watcher restart showing one process and one renewal timer.

## Reproduction Assessment

Yes, source-level: current main overwrites Gmail watcher child and interval handles on repeated successful starts, and the untracked restart timeout gives a concrete duplicate-spawn window. I did not run live or unit tests because this review is read-only.

## Solution Assessment

No, not yet: the active-process guard is the right direction, but it also needs to retire queued restart callbacks before starting the replacement watcher and should be backed by real repeated-start or reload proof.
