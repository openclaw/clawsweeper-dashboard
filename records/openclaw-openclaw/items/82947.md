## What This Changes

The PR tracks the Gmail watcher respawn timer, clears old renewal/process state on direct `startGmailWatcher()` re-entry, clears pending respawn on stop, and adds mocked watcher lifecycle tests.

## Best Possible Solution

Keep the timer cleanup, make re-entry shutdown bounded and already-exited-safe, then require redacted real or scripted gateway proof that repeated start/reload leaves one active `gog serve` process and one renewal/respawn path.

## Solution Assessment

No: the ownership and timer-tracking direction is right, but the re-entry kill wait should remain bounded/already-exited-safe and the changed direct re-entry path still needs real behavior proof.
