## What This Changes

## Best Possible Solution

Bound old-process cleanup on `exit`, `close` or `error`, plus a final timeout after escalation, then add focused regression coverage and redacted live proof that repeated start leaves one active `gog serve` process.

Yes, at source level: current main can overwrite `watcherProcess` and `renewInterval` on successive starts, and the respawn timeout is not stored for later cancellation. I did not run a runtime repro because this review is read-only.
