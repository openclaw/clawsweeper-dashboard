## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Initialize settle timers before signaling:** `src/hooks/gmail-watcher.ts:151`
  - body: `proc.kill("SIGTERM")` runs before `escalation` and `finalTimeout` are initialized, while the registered exit/close/error handlers call `settle()` and clear those handles. If a kill/error/settle path fires synchronously, the promise rejects with a ReferenceError and the new stop/re-entry path fails; initialize nullable handles before registering/signaling, or arm the timers before `kill()`.
  - confidence: 0.84
- **[P2] Prevent old exits from respawning after timeout:** `src/hooks/gmail-watcher.ts:164`
  - body: The final timeout lets re-entry set `shuttingDown` back to false and spawn the replacement even though the old child may still be alive. If that old child exits later, its original `spawnGogServe` exit handler sees `shuttingDown` false with `currentConfig` set and queues another respawn, reintroducing the duplicate watcher leak in the stubborn-process path.
  - confidence: 0.78
