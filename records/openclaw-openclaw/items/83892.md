## What This Changes

The issue requests making `runRestartScript` tolerate spawn failures and child-process error events during the update restart handoff.

Yes, at source level. Current main lacks a child-process `error` listener and a direct Node probe shows a missing executable reaches an unhandled `error` event without one; I did not run a full OpenClaw update on a missing-shell or noexec system.

## Solution Assessment

Yes. Handling both `spawn` throws and child `error` events inside `runRestartScript` is the narrow maintainable fix; the linked PR is the current implementation candidate.
