## What This Changes

The issue requests that `runRestartScript` treat restart-script spawn failures as best-effort by catching synchronous spawn errors and handling ChildProcess `error` events.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84131, or replace it with the same narrow restart-helper fix and regression tests if that PR is abandoned.

## Reproduction Assessment

Yes, at source level. Current main lacks the child-process error listener and spawn guard, and a Node 24.15.0 probe confirms an unhandled child `error` event exits the process without a listener.

## Solution Assessment

Yes, the narrow maintainable fix is to handle both synchronous `spawn` throws and ChildProcess `error` events inside `runRestartScript`, with focused regression tests. The linked PR is the current implementation candidate, so this issue should wait for that PR rather than spawning a duplicate repair job.
