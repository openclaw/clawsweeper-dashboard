## Summary

Keep open. Current main still has the unguarded restart-helper spawn path, and the linked fix PR explicitly closes this issue but is still open and unmerged.

## What This Changes

The issue reports that `runRestartScript` can let `spawn` failures escape during the update restart handoff instead of treating the detached restart script as best-effort.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84131 or an equivalent helper-boundary fix that handles both child `error` events and synchronous spawn validation failures.

## Reproduction Assessment

Yes, at source level. Current main lacks the child-process error listener, and a direct Node probe shows a missing executable emits an unhandled `error` event without one; I did not run a full OpenClaw update end to end.

## Solution Assessment

Yes. The narrow maintainable fix is to make `runRestartScript` best-effort by handling both child `error` events and true synchronous spawn failures at the source, with focused regression coverage.
