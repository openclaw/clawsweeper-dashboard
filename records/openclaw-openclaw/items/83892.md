## What This Changes

The issue requests making `runRestartScript` tolerate spawn failures so the update restart handoff cannot terminate the CLI on missing-shell, noexec, or child-process error cases.

## Best Possible Solution

Keep this issue open as the canonical tracker, review the focused linked PR, and close the issue as implemented only after that fix merges to main.

Yes, at source level. Current main lacks both a synchronous spawn guard and a ChildProcess `error` listener in `runRestartScript`, and the caller awaits that helper during the update restart path; I did not run a real missing-shell or noexec update scenario.

## Solution Assessment

Yes, the narrow maintainable fix is to handle both child `error` events and unexpected synchronous spawn failures inside `runRestartScript` while preserving detached best-effort restart behavior. The open linked PR appears to target that boundary, but the issue stays open until the PR lands.
