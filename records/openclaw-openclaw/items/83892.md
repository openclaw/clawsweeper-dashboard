## Summary

Keep open. Current main and the latest tagged release still spawn the detached restart script without a synchronous `spawn()` guard or `ChildProcess` `error` listener, and https://github.com/openclaw/openclaw/pull/84131 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that `runRestartScript` can crash the update restart handoff when detached process spawning fails because the helper lacks local spawn/error handling.

## Best Possible Solution

Land or replace the linked PR with a focused helper-boundary fix that makes `runRestartScript` best-effort for both synchronous spawn throws and child `error` events, with regression coverage and acceptable proof.

## Reproduction Assessment

Yes, at source level. Current main lacks a child-process `error` listener, and a direct Node 24 probe with a missing executable exits through an unhandled `error` event; I did not run a full OpenClaw update on a missing-shell or noexec system.

## Solution Assessment

Yes. The maintainable fix is local to `runRestartScript`: handle child `error` events and synchronous `spawn` throws at the helper boundary while preserving detached best-effort restart behavior.
