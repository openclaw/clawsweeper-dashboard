## Summary

Keep open. Current main still has the restart handoff error-handling gap, and https://github.com/openclaw/openclaw/pull/85761 is an open linked fix with closing syntax, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests making `runRestartScript` best-effort when `spawn()` throws synchronously or the returned child process later emits an `error` during the update restart handoff.

## Best Possible Solution

Review and land the linked narrow fix PR, then close this issue once current main contains best-effort `spawn` throw and child `error` handling with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main lacks both a synchronous `spawn` guard and a returned-child `error` listener in `runRestartScript`, and a direct Node probe confirms the unhandled child `error` failure mode.

## Solution Assessment

Yes. The best fix is the narrow helper-level guard proposed by the linked PR: catch immediate spawn failures, attach a child `error` handler, preserve detached restart behavior, and add focused regression tests.
