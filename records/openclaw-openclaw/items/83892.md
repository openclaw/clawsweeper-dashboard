- `impact:crash-loop`: An unhandled child-process spawn failure can terminate the CLI during update restart handoff, which is a process availability failure.

## Summary

Keep open. Current main and the latest shipped release still run the detached restart script without guarding `spawn()` or the child `error` event, and the open closing PR is the correct path to resolve it after merge.

## What This Changes

Make `runRestartScript` treat restart-script spawn failures as best-effort so update restart handoff errors do not crash the CLI.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85761 if it remains clean, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level. Current main lacks a child-process `error` listener and a direct Node 24.15.0 probe shows a missing executable raises an uncaught ENOENT without that listener.

## Solution Assessment

Yes. Handling synchronous spawn failures and child `error` events inside `runRestartScript` is the narrow fix, and the linked open PR follows that approach with focused regression coverage.
