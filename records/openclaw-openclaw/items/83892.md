Keep open: current main still lacks best-effort error handling around the detached restart-script spawn, and there is an open closing PR for this issue that has not merged yet.

## What This Changes

The issue asks for `runRestartScript` to handle failed detached restart-script spawning without surfacing unhandled child-process errors during the update restart handoff.

## Best Possible Solution

Land a focused restart-helper fix with regression coverage through the linked PR or an equivalent patch, preserving normal detached restarts while making spawn failures best-effort and non-fatal.

## Reproduction Assessment

Yes, at source level. Current main lacks both a `try/catch` around `spawn` and a ChildProcess `error` listener, and Node 24.15.0 shows missing executables surface through child `error` events that need a listener.

## Solution Assessment

Yes, the narrow maintainable solution is to handle child `error` events and synchronous `spawn` throws inside `runRestartScript`; because an open PR already targets this boundary, the issue should stay open until that fix lands or is replaced.
