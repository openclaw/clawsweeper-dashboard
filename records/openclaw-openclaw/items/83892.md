## What This Changes

The issue requests making `runRestartScript` treat restart-script spawn failures as best-effort so the update restart handoff does not terminate the CLI.

## Best Possible Solution

Land or explicitly reject the linked narrow helper-level fix with regression tests, then close this issue only after the fix is merged or a maintainer chooses a different canonical path.

## Reproduction Assessment

Yes, at source level. Current main lacks a child `error` listener, and a direct Node probe shows a missing subprocess emits an unhandled ENOENT unless the child handles the event; I did not run a full OpenClaw update end-to-end.

## Solution Assessment

Yes, the linked PR's direction is the narrow maintainable fix: handle synchronous spawn validation failures and child `error` events inside `runRestartScript` while preserving detached `unref()` behavior.
