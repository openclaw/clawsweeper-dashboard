## Summary

Keep open. Current main still has a source-level terminal-path gap where pre-tool assistant text can make the run look payload-bearing even if the expected post-tool final assistant message is never persisted, so the reported WebChat and next-turn context loss is plausible and not implemented away.

## What This Changes

The issue reports that final assistant text after a multi-tool agent turn is dropped from WebChat and persisted context.

## Best Possible Solution

Add a runtime-level guard so a tool-use turn cannot finish with only pre-tool text after tool results; it should persist the latest post-tool visible text or surface the existing replay-invalid incomplete-turn warning.

## Reproduction Assessment

No, not as a live current-main reproduction in this read-only pass. We do have a clear source-level path: pre-tool assistant text yields a nonzero payload count, while missing post-tool message_end leaves no final assistant text for persistence and skips incomplete-turn fallback.

## Solution Assessment

Yes, the best direction is a narrow agent runtime/terminal persistence fix rather than a WebChat-only refresh. The durable behavior should make persisted history, streamed UI state, and next-turn context agree.
