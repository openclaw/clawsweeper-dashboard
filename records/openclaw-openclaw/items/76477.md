## Summary

Keep open. Current main still has a source-level gap where pre-tool assistant text can make the terminal path look payload-bearing, so the incomplete-turn guard can skip a missing post-tool final assistant message.

## What This Changes

Ensure final assistant text after tool results in a multi-tool agent turn is persisted, rendered in WebChat, and available to next-turn context.

## Best Possible Solution

Add a regression-backed runtime guard so a tool-use turn cannot finish as successful with only pre-tool text after tool results; continue safely from the latest tool result when replay-safe, otherwise surface an explicit incomplete-turn warning with liveness metadata.

## Reproduction Assessment

Yes at source level, but not as a live current-main WebChat run. Constructing an attempt with pre-tool `assistantTexts`, tool activity, and no post-tool assistant message maps to the current payload-count and incomplete-turn skip path.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the agent terminal/transcript path rather than WebChat-only rendering. WebChat should only need regression confirmation that persisted history and live events agree after the runtime fix.
