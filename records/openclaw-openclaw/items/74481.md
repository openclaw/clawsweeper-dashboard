## Summary

Keep open. Current main still has no configured OpenAI-compatible `/models` catalog refresh path for the Control UI picker, and the related same-author PR is an open first slice rather than a merged implementation.

## What This Changes

Request an opt-in provider-backed dynamic model catalog refresh that fetches configured provider `/models` endpoints and surfaces discovered rows in model pickers.

## Best Possible Solution

Define the provider catalog contract first, then land provider-owned OpenAI-compatible discovery integrated with Gateway/Control UI catalog refresh, deterministic merge or replace behavior, auth/base URL invalidation, tests, docs, and changelog coverage.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is high-confidence from source inspection: configured `baseUrl` and `apiKey` alone do not cause current main to fetch upstream `/models` or add discovered rows to the picker.

## Solution Assessment

Unclear until maintainers choose the seam. The requested behavior is valid, but the narrowest maintainable solution may be extending the existing provider `catalog` path rather than introducing a separate `resolveDynamicCatalog` hook.
