## Best Possible Solution

Decide the OpenAI-compatible live catalog contract first, then implement any approved path through the shared guarded live-catalog SDK with explicit cache, auth, fallback, UI refresh, and security tests.

## Reproduction Assessment

Yes for source-level behavior: current main intentionally skips live OpenAI discovery for custom OpenAI-compatible base URLs and the Control UI configured picker reads the current catalog snapshot.

## Solution Assessment

Unclear until maintainers decide the product/security contract; a standalone `resolveDynamicCatalog` hook is plausible, but current main's shared guarded live-catalog helper is the better implementation boundary if the feature is approved.
