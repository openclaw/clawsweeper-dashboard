## Summary

Keep open: current main and the latest release still intentionally skip live `/models` discovery for custom OpenAI-compatible base URLs, and the remaining work needs maintainer decisions around endpoint trust, cache scope, picker/list semantics, and SecretRef-safe generated catalog refresh.

## What This Changes

The issue requests an opt-in live model catalog refresh path from configured provider model-list endpoints so OpenClaw model pickers and catalog-derived views reflect proxy or gateway model availability.

## Best Possible Solution

Settle the custom OpenAI-compatible live-catalog contract, then implement it through the existing guarded provider-catalog SDK with auth-scoped caching, static fallback, picker/list refresh behavior, and SecretRef-safe generated catalog re-derivation.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 intentionally skip live `/models` discovery for custom OpenAI-compatible base URLs, and a focused test asserts the fetch guard is not called. I did not run a live proxy or Control UI flow in this read-only review.

## Solution Assessment

No, the proposed new hook is not yet proven to be the best solution. The safer path is to extend the existing guarded provider-catalog runtime after maintainers decide endpoint trust, cache scope, merge/replace behavior, refresh lifecycle, and generated-catalog credential handling.
