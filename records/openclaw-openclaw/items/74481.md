## Summary

Keep open. Current main still lacks configured OpenAI-compatible `/v1/models` catalog refresh for the Control UI picker; the repository now has a generic provider catalog seam, but OpenAI does not use it for live catalog discovery and the same-author paired PR #74488 remains the active implementation discussion.

## What This Changes

Request an opt-in dynamic model catalog refresh path that reads configured provider `/v1/models` endpoints and updates OpenAI-compatible model picker entries instead of relying only on the bundled static catalog.

## Best Possible Solution

Design and land a provider-owned catalog discovery path through the approved existing or additive SDK seam, with OpenAI-compatible `/models` fetching, credential/baseUrl-scoped caching, deterministic merge or replace behavior, Gateway and Control UI refresh behavior, focused tests, docs, and changelog coverage.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is high-confidence from source inspection: OpenAI has static/synthetic catalog behavior only, and the UI model picker still calls `models.list` without a provider refresh path.

## Solution Assessment

Unclear. The requested behavior is valid, but the proposed new hook may not be the narrowest maintainable solution now that current main exposes `ProviderPlugin.catalog`; maintainers should choose the seam before implementation continues.
