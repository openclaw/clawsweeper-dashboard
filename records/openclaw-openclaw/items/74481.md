## Summary

Keep open. Current main still lacks a configured OpenAI-compatible `/models` catalog refresh path for the Control UI picker, and related PR #74488 is still open and intentionally stops before catalog/UI integration.

## What This Changes

Request an opt-in dynamic model catalog refresh that fetches configured provider `/models` endpoints and rebuilds model picker entries from discovered models.

## Best Possible Solution

Define the provider catalog contract first, then land provider-owned OpenAI-compatible discovery integrated with Gateway and Control UI catalog refresh, deterministic merge or replace behavior, guarded credentialed fetches, tests, docs, and changelog coverage.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is high-confidence from source inspection: configured provider `baseUrl` and `apiKey` do not currently trigger `/models` discovery, and `models.list` has no refresh/provider parameter.

## Solution Assessment

Unclear until maintainers choose the seam. The requested behavior is valid, but the maintainable path should extend the provider-owned catalog flow with guarded outbound fetches rather than adding UI-only discovery.
