## Summary

Current main already implements the ACP `session/list` fix through a merged lifecycle-handler PR, with a stable `AcpGatewayAgent.listSessions` handler, regression coverage, and docs; the extra legacy alias proposed here is not needed for the pinned SDK and current tests intentionally assert it is absent.

## What This Changes

This PR adds `AcpGatewayAgent.listSessions`, keeps `unstable_listSessions` as an alias, adds ACP list-session tests, and records the fix in the changelog.

## Best Possible Solution

Keep the merged current-main ACP lifecycle implementation and close this older branch as superseded by that implementation; users need a post-`v2026.5.7` build or the next release for the fix.

## Reproduction Assessment

Yes for affected older builds: the reporter's JSON-RPC stdin command reaches a dispatcher that expects `agent.listSessions` while the old translator only had `unstable_listSessions`. Current main no longer reproduces the mismatch because `AcpGatewayAgent.listSessions` exists and is covered by lifecycle tests.

## Solution Assessment

Yes for current main. The merged lifecycle-handler implementation uses the SDK-required stable method name, keeps the implementation Gateway-backed, adds pagination and cwd validation, and intentionally omits the obsolete alias for the pinned SDK 0.21.0 surface.
