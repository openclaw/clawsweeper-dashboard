## Summary

Keep open: current main still has runtime pruning, Gateway/media offload, and chat-history display cleanup, but the session persistence path does not replace user image content blocks with compact media references before JSONL storage.

## What This Changes

Store Discord and other inbound image references in session transcripts instead of persisting inline base64 image data, then hydrate bounded image content when building model context.

## Best Possible Solution

Add a generic, backward-compatible transcript claim-check for image content blocks that persists compact media reference metadata, hydrates only bounded recent references for model context, and keeps legacy inline-base64 transcripts readable.

## Reproduction Assessment

Yes. A focused reproduction is to append a user message containing `{ type: "image", data, mimeType }` through the guarded session manager and inspect the persisted message; current source shows that block is written unchanged, and the Discord comment provides a real-session JSONL example.

## Solution Assessment

Yes. The requested claim-check is the narrow maintainable fix because pruning and chat-history redaction reduce replay or UI bloat but do not solve double storage or raw transcript growth.
