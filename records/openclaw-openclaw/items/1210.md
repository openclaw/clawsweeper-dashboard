## Summary

Keep open: current main still has mitigations for replay pruning, Gateway offload, and chat-history display, but it does not persist inbound user image content as durable media references. The session append guard still writes user `image` blocks after only the existing provenance/text-redaction transforms, and the pinned Pi dependency appends prompt images into the persisted user message.

## What This Changes

Replace inline base64 image blocks in session JSONL transcripts with compact media references and hydrate them when needed.

## Best Possible Solution

Add a backward-compatible transcript claim-check: persist compact media reference metadata instead of raw `image.data`, hydrate only bounded recent references during model-context construction, keep legacy inline-base64 transcripts readable, and coordinate display projection with https://github.com/openclaw/openclaw/pull/43982.

## Reproduction Assessment

Yes, at source level: Discord/media turns can reach `AgentSession.prompt(..., { images })`, the dependency appends those images to the user message, and the guarded append path persists the message without rewriting image blocks. I did not run a live current-main Discord reproduction.

## Solution Assessment

Yes. A persistence-layer claim-check plus bounded replay hydration is the narrow maintainable direction; chat-history redaction and replay pruning reduce symptoms but do not change the durable JSONL boundary.
