## Summary

Keep open: current main still has replay/display/offload mitigations, but the central session persistence path can still write inbound user `image.data` blocks into JSONL when a channel turn is submitted with images. The exact persistence PRs are closed unmerged, and the open Gateway history PR is a display projection change rather than the durable transcript claim-check requested here.

## What This Changes

Store inbound Discord/channel images in session transcripts as durable media references instead of inline base64 content.

## Best Possible Solution

Add a backward-compatible transcript claim-check for inbound image content using the existing media store and `media://inbound`/MediaPath patterns, hydrate only bounded current or recent references during model-context construction, and keep old inline transcripts readable.

## Reproduction Assessment

Yes, at source level: a channel turn with images reaches `AgentSession.prompt(..., { images })`, the dependency appends those images to the user content array, and OpenClaw's guarded append path does not rewrite image blocks before persistence. I did not rerun a live Discord DM.

## Solution Assessment

Yes. A storage-layer claim-check is the narrowest maintainable fix because replay pruning, Gateway attachment offload, and chat history display redaction reduce adjacent symptoms but still leave the durable JSONL boundary unresolved.
