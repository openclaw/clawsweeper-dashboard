## Summary

Keep open. The underlying media optimizer failure is still unresolved on current main, but this PR is not mergeable as-is because the fallback is inside the JPEG optimizer, where it lacks MIME and loader policy context and can catch existing image safety guard failures.

## What This Changes

The PR changes `optimizeImageToJpeg` to return the original buffer with zero resize/quality metadata after all JPEG resize attempts fail.

## Best Possible Solution

Implement fallback at the `loadWebMedia`/`optimizeAndClampImage` boundary: only pass through under-cap non-HEIC originals, preserve MIME and filename, rethrow image guard failures, keep optimizer causes, add regression tests, and include the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-path reproduction exists: make `resizeToJpeg` throw for a non-JPEG or byte-small oversized input, and this PR returns the original buffer from the JPEG optimizer while the caller applies only the byte cap and JPEG content-type mapping.

## Solution Assessment

No. `optimizeImageToJpeg` lacks the original content-type and safety-policy context needed for a safe original-buffer fallback; the narrower maintainable fix belongs at the loader/clamp boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve image guard failures:** `src/media/web-media.ts:657-662`
  - body: The fallback returns `source` after every `resizeToJpeg` attempt throws. `resizeToJpeg` is where pixel-limit and unknown-dimension guards run, so byte-small oversized or unsafe images can be accepted if they are under `maxBytes`; rethrow those guard errors before falling back.
  - confidence: 0.91
- **[P2] Preserve MIME on original fallback:** `src/media/web-media.ts:657-662`
  - body: For non-HEIC PNG/WebP inputs, `source` is still the original bytes, but callers wrap any `optimizeImageToJpeg` result as JPEG and emit `image/jpeg`. Move the fallback to the loader boundary or carry the original content type so consumers do not receive PNG/WebP bytes labeled as JPEG.
  - confidence: 0.92
- **[P3] Keep optimizer causes on failed fallbacks:** `src/media/web-media.ts:657-662`
  - body: When the original is over cap and all resize attempts fail, this return path turns the detailed optimizer failure into a generic cap error from the caller. Preserve the underlying optimizer cause for unsendable originals so missing or broken image backends stay diagnosable.
  - confidence: 0.82
