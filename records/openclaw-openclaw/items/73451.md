## Summary

Keep open, but this patch should not merge as written. It is paired with the still-open https://github.com/openclaw/openclaw/issues/73424, and the branch still has guard, MIME, and diagnostic regressions plus no real after-fix proof; current main has a narrower loader-boundary fallback for missing `sharp` that is safer than returning raw bytes from `optimizeImageToJpeg`.

## What This Changes

The PR changes `optimizeImageToJpeg` to return the original source buffer with zero resize/quality metadata after every JPEG optimization attempt fails.

## Best Possible Solution

Keep the fallback at the media-loader boundary, preserve MIME/filename, fail closed for HEIC, cap, pixel-limit, and unknown-dimension guard errors, preserve optimizer causes, and cover the behavior with focused media tests.

## Reproduction Assessment

Yes. A high-confidence source-path reproduction exists: force `resizeToJpeg` to throw a guard or non-JPEG optimizer error, and this PR returns the original bytes from `optimizeImageToJpeg` before the caller applies only byte-cap and JPEG content-type mapping.

## Solution Assessment

No. `optimizeImageToJpeg` lacks the original MIME and loader policy context needed for a safe passthrough; the maintainable fix belongs at `loadWebMedia`/`optimizeAndClampImage`, where current main already has a narrower missing-`sharp` fallback shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve image input guard failures:** `src/media/web-media.ts:657-662`
  - body: The fallback returns `source` after every `resizeToJpeg` attempt throws. `resizeToJpeg` is where pixel-limit and unknown-dimension guard errors are enforced, so this can accept byte-small but unsafe images instead of failing closed. Re-throw those guard failures or move the passthrough decision to the loader boundary after guard checks are preserved.
  - confidence: 0.91
- **[P2] Preserve original MIME on fallback:** `src/media/web-media.ts:657-662`
  - body: For non-HEIC PNG/WebP inputs, `source` is still the original bytes, but callers wrap any `optimizeImageToJpeg` result as JPEG and emit `image/jpeg`. Carry the original content type or perform fallback at the loader boundary so consumers do not receive PNG/WebP bytes labeled as JPEG.
  - confidence: 0.92
- **[P3] Keep optimizer diagnostics on failed fallbacks:** `src/media/web-media.ts:657-662`
  - body: When the original is still over cap and all resize attempts fail, this return path turns the detailed optimizer failure into a generic cap error from the caller. Preserve the underlying optimizer cause for unsendable originals so broken image backends remain diagnosable.
  - confidence: 0.82
