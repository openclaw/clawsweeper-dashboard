## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:62`
  - body: `validateSystemdServiceName` still accepts names that start with `-`, and the PR passes the resulting unit name to `systemctl` as an operand without a `--` terminator. A value like `OPENCLAW_SYSTEMD_UNIT=-Hhost` can be parsed as a systemctl option instead of a unit name, so reject leading dashes and/or terminate systemctl options consistently.
  - confidence: 0.89
- **[P1] Preserve the missing-unit activation retry:** `src/daemon/systemd.ts:735-740`
  - body: This branch calls `systemctl --user enable` and `restart` only once after `daemon-reload`. Current main fixed the migrated/root user-service race by retrying `daemon-reload` plus the failed action when systemd reports the freshly written unit is missing, so landing this branch without carrying that logic can reopen the install/update outage.
  - confidence: 0.9
- **[P1] Use the override-aware service name on uninstall:** `src/daemon/systemd.ts:794-796`
  - body: Install now targets `resolveSystemdServiceName(env)`, but uninstall still disables `${resolveGatewaySystemdServiceName(...)}`. For `OPENCLAW_SYSTEMD_UNIT=openclaw-node` or another custom unit, uninstall can disable the gateway service while deleting the custom unit file, leaving the intended service state inconsistent.
  - confidence: 0.92
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:637-638`
  - body: The new managed drop-in can contain credential-bearing environment values, but the PR writes the main unit, `.bak`, and drop-in with direct `copyFile`/`writeFile` on predictable HOME-derived paths. A symlinked unit/drop-in path can redirect writes or leak managed env, so the writer needs no-follow or atomic-safe file handling and explicit permission policy before this migration ships.
  - confidence: 0.84
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-273`
  - body: When only some assignments are stripped from an `Environment=` line, the kept assignments are rerendered through `renderEnvLines`, which drops empty or whitespace-only values and trims non-empty values. That changes valid systemd semantics such as `LD_PRELOAD=` or intentional whitespace values during migration, so reconstruct kept assignments without value filtering/trimming.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has concrete security-sensitive concerns around systemctl option parsing and credential-bearing systemd file writes.
