## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the activation retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` and the failed `enable`/`restart` once when systemd says the newly written unit does not exist. This branch's activation path calls each action only once, so refreshing or resolving the stale branch this way would reintroduce the migrated-systemd install failure covered by current tests.
  - confidence: 0.91
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: The branch disables `resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)` during uninstall while deleting the path resolved from `OPENCLAW_SYSTEMD_UNIT`. With `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, uninstall can disable `openclaw-gateway.service` and remove `openclaw-node.service`, leaving the actual node unit running or misreported.
  - confidence: 0.93
- **[P1] Reject option-like unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new service-name validator still allows names beginning with `-`, and the later `systemctl` calls pass the unit name without a `--` terminator. An `OPENCLAW_SYSTEMD_UNIT` value that parses as a systemctl option can change the command meaning instead of being treated as a unit operand.
  - confidence: 0.86
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:627-638`
  - body: The new migration writes backups and managed drop-ins under predictable HOME-derived systemd paths with `copyFile` and `writeFile`. Because the drop-in can contain managed service environment data, this needs a no-follow/atomic-write strategy or explicit accepted threat model before the PR expands the write surface.
  - confidence: 0.82
- **[P1] Preserve kept Environment values when stripping:** `src/daemon/systemd-unit.ts:270-272`
  - body: When stripping only some assignments from an `Environment=` line, this rerenders the kept assignments through `renderEnvLines`, which filters empty/whitespace-only values and trims values. Valid systemd assignments such as `LD_PRELOAD=` can be silently removed during migration, changing the service environment.
  - confidence: 0.84

## Security Review
