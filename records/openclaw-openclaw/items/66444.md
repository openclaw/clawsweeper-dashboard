Full review comments:

- **[P1] Preserve current-main activation retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` plus the failed `enable` or `restart` once when systemd reports `Unit file ... does not exist`. This PR head still calls `enable` and `restart` only once, so refreshing this dirty branch must keep the current-main retry or it can bring back the migrated-install failure.
  - confidence: 0.93
- **[P1] Use the override service name for uninstall:** `src/daemon/systemd.ts:794-796`
  - body: The PR head disables `${resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)}.service` during uninstall even though the unit path is resolved from `OPENCLAW_SYSTEMD_UNIT`. Current main uses `resolveSystemdServiceName(env)`, so a custom node unit such as `openclaw-node.service` would be left enabled if this stale branch is carried forward as-is.
  - confidence: 0.94
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: `validateSystemdServiceName` allows names beginning with `-`, and the same file passes the unit operand to `systemctl` without a `--` terminator. A custom `OPENCLAW_SYSTEMD_UNIT` can therefore be parsed as a systemctl option instead of a unit name.
  - confidence: 0.88
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:637-638`
  - body: This path writes backups, the main unit, and the secret-bearing managed drop-in directly under a HOME-derived systemd directory. Before merge it needs symlink/hardlink-safe writes or explicit maintainer threat-model acceptance for those predictable paths.
  - confidence: 0.82
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-273`
  - body: When only some assignments are stripped, the remaining values are re-rendered through `renderEnvLines`, which filters empty or whitespace-only values and trims values. That can turn a valid operator assignment such as `Environment=LD_PRELOAD=` into no assignment after migration.
  - confidence: 0.86
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:17`
  - body: Root repository policy says normal PRs should not edit `CHANGELOG.md`; release-note context belongs in the PR body or final commit message, with release generation owning the changelog.
  - confidence: 0.91
