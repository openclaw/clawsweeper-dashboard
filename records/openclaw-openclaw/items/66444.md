## Security Review

Status: needs_attention

Summary: The diff changes token-bearing systemd service files and custom unit-name command execution, with concrete option-injection and file-write hardening issues still unresolved.

Concerns:

- **[high] Option-like unit names can reach systemctl:** `src/daemon/systemd.ts:55`
  - body: `validateSystemdServiceName` still permits leading `-`, while `enable`, `restart`, `disable`, `stop`, and `restart` unit operands are passed without a `--` terminator, so an environment-controlled custom unit name can be interpreted as a systemctl option.
  - confidence: 0.84
- **[high] Managed drop-in writes can follow attacker-controlled links:** `src/daemon/systemd.ts:637`
  - body: The new drop-in stores managed environment values under a predictable HOME-derived path using direct `writeFile`, and the backup path uses direct `copyFile`; symlink or hardlink redirection needs to be prevented or explicitly ruled out before merge.
  - confidence: 0.78
- **[medium] Environment rewrite can remove clearing assignments:** `src/daemon/systemd-unit.ts:270`
  - body: Dropping `Environment=FOO=` while stripping managed keys can unintentionally restore inherited environment values for the service, which can affect loader or runtime behavior in security-sensitive ways.
  - confidence: 0.82
