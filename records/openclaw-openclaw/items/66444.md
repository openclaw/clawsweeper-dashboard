## Security Review

Status: needs_attention

Summary: The diff touches systemd command operands and secret-bearing unit/drop-in writes, and the current head still has concrete option-injection, symlink/write, and environment-rewrite concerns.

- **[high] Option-like custom unit names still pass validation:** `src/daemon/systemd.ts:53`
  - body: SYSTEMD_SERVICE_NAME_PATTERN allows a leading dash and systemctl calls do not terminate options with '--', so OPENCLAW_SYSTEMD_UNIT can still be interpreted as a systemctl option.
