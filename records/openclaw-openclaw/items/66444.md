Full review comments:

- **[P1] Preserve current-main activation retry:** `src/daemon/systemd.ts:735-740`
  - body: This branch still calls `enable` and `restart` only once. Current main retries `daemon-reload` plus the failed action when systemd says the just-written unit does not exist, so refreshing this PR without that logic would reintroduce the migrated/root systemd install failure.
  - confidence: 0.92
- **[P1] Use the override service name for uninstall:** `src/daemon/systemd.ts:794-796`
  - body: Uninstall resolves the gateway service name instead of the environment override, so `OPENCLAW_SYSTEMD_UNIT=openclaw-node` disables `openclaw-gateway.service` while removing the node unit file. Current main uses `resolveSystemdServiceName(env)` and has regression coverage for this path.
  - confidence: 0.94
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new validator still accepts names that start with `-`, and systemctl receives the unit operand without an end-of-options marker. A custom unit such as `--user.service` can be parsed as an option rather than a unit name, so the validation needs to reject leading dashes or consistently pass `--` before operands.
  - confidence: 0.86
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:628-637`
  - body: The migration writes backups, main units, and the new secret-bearing drop-in through predictable HOME-derived paths with direct `copyFile`/`writeFile`. Before this lands, the writer needs a clear no-symlink/permission strategy or a maintainer-approved threat model for these paths.
  - confidence: 0.82
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-272`
  - body: When only some assignments are stripped, the kept assignments are re-rendered through `renderEnvLines`, which filters empty/whitespace-only values and trims values. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` after removing a managed key from the same line.
  - confidence: 0.88
- **[P2] Remove release-owned changelog edits:** `CHANGELOG.md:17`
  - body: Root policy says `CHANGELOG.md` is release-owned and normal PRs should not edit it. Move the release-note context into the PR body or final commit message, then let release generation update the changelog.
  - confidence: 0.9
