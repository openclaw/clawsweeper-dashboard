  - body: Current main retries `enable` and `restart` once after a second `daemon-reload` when systemd says the freshly written unit does not exist, which closed the migrated-root install failure.   - confidence: 0.92

  - body: The PR head still computes the uninstall target from `resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)`, ignoring `OPENCLAW_SYSTEMD_UNIT`. For node installs this disables `openclaw-gateway.service` while removing `openclaw-node.service`, which is the custom-unit bug current main already fixed.
  - confidence: 0.91
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new validation still allows names beginning with `-`, and the `systemctl` calls pass the unit operand without `--`. An `OPENCLAW_SYSTEMD_UNIT` value such as `--user` or `-Hhost` can still be parsed as a systemctl option rather than a unit name, so the hardening is incomplete.
  - confidence: 0.9
- **[P1] Avoid direct writes for secret-bearing drop-ins:** `src/daemon/systemd.ts:627-638`
  - body: The new managed drop-in can contain tokens, but it is written directly to a predictable HOME-derived path and the backup path is copied directly as well. Please use a no-symlink/atomic write strategy or otherwise explicitly rule out the overwrite/leakage threat before adding this secret-bearing file surface.
  - confidence: 0.86
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-274`
  - body: When only some assignments are stripped from an `Environment=` line, the kept assignments are re-rendered through `renderEnvLines`, which drops empty values and trims whitespace. That changes valid systemd settings such as `Environment=LD_PRELOAD=` during migration, so the rewrite needs to preserve the original assignment semantics.
  - confidence: 0.88
- **[P2] Remove the release-owned changelog edit:** `CHANGELOG.md:20`
  - body: This normal PR edits `CHANGELOG.md`, but the repository treats that file as release-owned. Move the user-visible release-note context to the PR body or final squash/merge message and leave changelog generation to the release flow.
  - confidence: 0.94
