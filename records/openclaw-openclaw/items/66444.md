  - body: Current main retries `daemon-reload` plus the failed `enable`/`restart` once when systemd reports the freshly written unit as missing.   - body: This line still derives the disable target from the gateway profile instead of `resolveSystemdServiceName(env)`. With `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, uninstall can disable the wrong service or leave the node unit enabled, which current main already fixed for https://github.com/openclaw/openclaw/issues/68287.
  - confidence: 0.95
- **[P1] Carry the env-source contract into the writer:** `src/daemon/systemd.ts:527-533`
  - body: Current main's writer consumes `environmentValueSources` to decide what stays inline, what belongs in the generated gateway/node env file, and what gets removed from sanitized backups. This PR head drops that input from `writeSystemdUnit`, so file-managed tokens and unresolved shell references can be re-inlined, lost, or left in backups during re-stage.
  - confidence: 0.87
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new validation still accepts service names beginning with `-`, and the later `systemctl` calls pass the unit operand without a `--` separator. A custom `OPENCLAW_SYSTEMD_UNIT` that looks like a systemctl option can cross the intended service-name boundary, so reject leading dashes or terminate options consistently.
  - confidence: 0.9
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:637-638`
  - body: The PR writes the managed drop-in and backs up/rewrites the main unit through predictable HOME-derived paths with direct `copyFile`/`writeFile`. Because the new drop-in can carry managed secrets, the refresh should either use no-follow/atomic safe writes or document and test the accepted same-user threat model before landing.
  - confidence: 0.82
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-272`
  - body: When stripping only some assignments from a multi-assignment `Environment=` line, this rewrite path re-renders kept values through `renderEnvLines`, which filters empty or whitespace-only values and trims values. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` and can alter operator-edited units during migration.
  - confidence: 0.88
