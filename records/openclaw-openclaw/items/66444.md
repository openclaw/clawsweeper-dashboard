  - confidence: 0.96
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:60-62`
  - body: The new validation still accepts names beginning with `-`, and the later `systemctl` calls pass the unit operand without a `--` terminator. An `OPENCLAW_SYSTEMD_UNIT` value like `--user` can be parsed as a systemctl option, so the validation needs a leading-dash rejection or consistent operand terminator plus tests.
  - confidence: 0.9

  - body: The new managed drop-in carries service environment values but is written through a predictable HOME-derived path with `writeFile` and `chmod`, and the main-unit backup/rewrite path uses `copyFile`/`writeFile` similarly. A pre-created symlink can redirect token-bearing writes or backups; use no-follow/atomic writes or document and test the threat model before landing.
  - confidence: 0.86
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-273`
  - body: When a mixed `Environment=` line has one assignment stripped, the kept assignments are re-rendered through `renderEnvLines`, which filters out empty values and trims whitespace. systemd treats `FOO=` as a real assignment that clears inherited values, so partial rewrites must preserve the kept values' semantics.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff touches secret-bearing systemd unit/drop-in paths and still has concrete unit-name validation and file-write hardening concerns.

Concerns:

- **[high] Option-like custom unit names remain accepted:** `src/daemon/systemd.ts:60`
  - body: `validateSystemdServiceName` rejects paths and shell fragments but still accepts leading-dash names, while `systemctl` operands are passed without a `--` terminator. A custom unit value can be parsed as a systemctl option rather than a unit name.
  - confidence: 0.9
- **[high] Drop-in writes can follow attacker-controlled links:** `src/daemon/systemd.ts:637`
  - body: The new managed drop-in can contain tokens and is written with `fs.writeFile` to a predictable HOME-derived path, so a pre-created symlink in that tree can redirect the write unless the path is opened no-follow or replaced atomically with directory checks.
  - confidence: 0.86
- **[medium] Sentinel-missing env can land in the main unit:** `src/daemon/systemd.ts:593`
  - body: When the managed-env sentinel is absent, the split treats all provided environment values as user-owned and the fresh main unit write uses default unit-file permissions, so future callers that pass secrets without the sentinel can expose them in the main service file.
  - confidence: 0.78
