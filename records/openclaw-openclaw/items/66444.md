## Review Findings

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve the activation retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` plus the failed `enable` or `restart` once when systemd reports that the freshly written unit does not exist. This branch goes back to one `enable` and one `restart`, so merged as-is it can reintroduce the migrated-systemd install race covered by current main's systemd tests.
  - confidence: 0.94
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: The install path resolves `OPENCLAW_SYSTEMD_UNIT`, but uninstall still derives `openclaw-gateway` from the profile. For node/custom units this disables the wrong service, regressing the current-main behavior that uses `resolveSystemdServiceName(env)`.
  - confidence: 0.96
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: `SYSTEMD_SERVICE_NAME_PATTERN` still accepts names beginning with `-`, and the systemctl calls pass the unit operand after the action without a `--` terminator. A value like `OPENCLAW_SYSTEMD_UNIT=-Hfoo` can be parsed by systemctl as an option, so the new hardening path does not actually reject this unsafe custom unit name.
  - confidence: 0.9
- **[P1] Preserve empty Environment assignments:** `src/daemon/systemd-unit.ts:270-274`
  - body: When only part of an `Environment=` line is stripped, the kept entries are re-rendered through `renderEnvLines`, which drops empty/whitespace-only values and trims values. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration instead of preserving the operator's unit content.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff has concrete systemd security-boundary concerns around option-like unit names, direct writes to HOME-derived paths, and semantic changes to security-sensitive Environment= clearing directives.

Concerns:

- **[high] systemctl unit operand can still be parsed as an option:** `src/daemon/systemd.ts:55`
  - body: The validator permits leading `-` and later systemctl calls do not consistently insert `--` before the unit name, so an environment-provided unit name can alter systemctl option parsing.
  - confidence: 0.9
- **[medium] Unit and drop-in writes follow predictable HOME-derived paths:** `src/daemon/systemd.ts:637`
  - body: The new drop-in writer and rewritten unit path use direct copy/write calls on predictable paths under HOME; the PR needs a safe-write fix or a maintainer-approved threat model for symlink/hardlink overwrite risk.
  - confidence: 0.78
- **[medium] Environment rewrite can remove clearing assignments:** `src/daemon/systemd-unit.ts:271`
  - body: Dropping `Environment=FOO=` while stripping managed keys can re-enable inherited environment variables that the operator intentionally cleared, which is a security-sensitive semantic change for systemd units.
  - confidence: 0.82
