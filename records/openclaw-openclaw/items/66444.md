## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve kept Environment assignments exactly:** `src/daemon/systemd-unit.ts:271-272`
  - body: When this strips only some assignments from a mixed Environment= line, the surviving assignments are passed through renderEnvLines(), which filters out values whose trim() is empty and trims the rest. That changes valid operator-owned systemd state such as Environment="LD_PRELOAD=" or whitespace-bearing values during the migration. Re-render the kept assignments without dropping empty values or trimming, and add mixed-line tests for empty and whitespace values.
  - confidence: 0.91
- **[P1] Reject option-like custom systemd unit names:** `src/daemon/systemd.ts:62`
  - body: The new validator still accepts names beginning with '-', while the lifecycle calls pass the unit name to systemctl without a '--' terminator. An OPENCLAW_SYSTEMD_UNIT such as -Hfoo can be parsed as a systemctl option instead of a unit operand, so the validator needs to reject leading '-' and/or every systemctl call needs a terminator before unit operands.
  - confidence: 0.87
- **[P2] Harden the managed drop-in write path:** `src/daemon/systemd.ts:637-638`
  - body: The new managed drop-in can contain service tokens and is written directly to a predictable HOME-derived path with fs.writeFile() followed by chmod(). That follows a pre-existing symlink at the destination before permissions are corrected, so a hostile or corrupted systemd user directory can redirect the write. Use a no-follow/atomic write strategy or add an explicit threat-model decision before storing managed env in this new file.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff changes secret-bearing systemd file writes and systemctl unit validation, and there are concrete security hardening gaps to resolve or explicitly threat-model before merge.

Concerns:

- **[high] Option-like systemd unit names can reach systemctl:** `src/daemon/systemd.ts:62`
  - body: The PR's new service-name validator permits leading '-' and the systemctl calls do not terminate options before unit operands, so custom unit names can still be interpreted as systemctl flags.
  - confidence: 0.87

  - body: The new managed environment drop-in is written with fs.writeFile() to a predictable HOME-derived path and may contain tokens, so a symlink at that path can redirect the write before chmod is applied.
  - confidence: 0.76

  - body: The new partial Environment= rewrite drops empty kept assignments and trims values, which can unintentionally remove operator clearings such as LD_PRELOAD= during migration.
