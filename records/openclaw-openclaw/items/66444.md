Full review comments:

- **[P1] Carry the env-source writer contract forward:** `src/daemon/systemd.ts:527-533`
  - body: This stale head destructures only `environment` and ignores current main's `environmentValueSources` contract. Current main uses those sources to keep file-backed secrets in `gateway.systemd.env`/`node.systemd.env` instead of rewriting them inline, so carrying this branch as-is would regress existing install/repair behavior.
  - confidence: 0.9
- **[P1] Preserve the missing-unit activation retry:** `src/daemon/systemd.ts:735-740`
  - body: The PR head runs `enable` and `restart` directly after one `daemon-reload`. Current main retries `daemon-reload` plus the failed action when systemd reports `Unit file ... does not exist`, so this needs to be carried forward or migrated installs can fail even after writing the unit.
  - confidence: 0.93
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: This uses the gateway-derived service name for `systemctl disable --now` even when `OPENCLAW_SYSTEMD_UNIT` points at a node/custom unit. Current main resolved this through `resolveSystemdServiceName(env)`, and dropping that behavior leaves custom units enabled while only unlinking the custom file path.
  - confidence: 0.94
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:56-63`
  - body: The new validation still accepts names beginning with `-`, and the systemctl calls pass the unit operand without a `--` terminator. A custom `OPENCLAW_SYSTEMD_UNIT` such as `--user.service` can be parsed as a systemctl option rather than a unit name, so reject leading dashes and/or terminate options consistently.
  - confidence: 0.88
- **[P1] Harden secret-bearing unit and drop-in writes:** `src/daemon/systemd.ts:627-638`
  - body: The new writer copies and writes predictable HOME-derived unit/drop-in paths directly, including the managed drop-in that can contain service tokens. Before moving secrets to this new file, use safe no-follow/atomic writes or capture an explicit maintainer threat-model decision for symlink/hardlink overwrite risk.
  - confidence: 0.82
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When a mixed `Environment=` line is partially stripped, this re-renders the kept assignments through `renderEnvLines`, which filters empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` or intentional trailing-space values during migration.
  - confidence: 0.86
