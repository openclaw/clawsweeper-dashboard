## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: `validateSystemdServiceName` still allows a leading `-`, and this branch passes the resulting unit name to `systemctl` without a `--` operand terminator in enable/restart/disable/show paths. `OPENCLAW_SYSTEMD_UNIT=-Hfoo` can be parsed as a systemctl option, so reject leading dashes and/or terminate operands consistently.
  - confidence: 0.94
- **[P1] Preserve the activation retry from current main:** `src/daemon/systemd.ts:735-741`
  - body: The PR head does one `daemon-reload`, then one `enable` and one `restart`; current main retries `daemon-reload` plus the failed action when systemd reports `Unit file ... does not exist`. Merging this as-is would reintroduce the migrated-install race tracked by https://github.com/openclaw/openclaw/issues/65184.
  - confidence: 0.91
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: `uninstallSystemdService` disables the gateway-derived unit even when `OPENCLAW_SYSTEMD_UNIT` points at a custom/node unit, while the file removal path uses the override-aware path. That can remove `openclaw-node.service` but leave the real node service enabled, regressing https://github.com/openclaw/openclaw/issues/68287.
  - confidence: 0.92
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:628-637`
  - body: The migration writes and backs up predictable HOME-derived unit/drop-in paths with direct `writeFile` and `copyFile`. Because the new drop-in can carry service secrets, this needs no-follow or atomic write handling, or an explicit maintainer-approved threat model, before landing.
  - confidence: 0.86
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When only some assignments are stripped, the code re-renders kept values through `renderEnvLines`, which drops empty values and trims the rest. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration; reconstruct kept assignments without dropping or trimming values and add focused tests.
  - confidence: 0.88

## Security Review
