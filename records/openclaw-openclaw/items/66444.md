  - body: `uninstallSystemdService` still disables `resolveGatewaySystemdServiceName(...)` even though the PR's install path can target `OPENCLAW_SYSTEMD_UNIT`. For node/custom units this removes the custom unit file and drop-in while leaving the actual unit enabled/running under systemd.
  - confidence: 0.93

  - body: The new validator still accepts names beginning with `-`, and the resulting unit name is passed to `systemctl` as an operand without a `--` terminator. An `OPENCLAW_SYSTEMD_UNIT` such as `-Hhost` can be parsed as a systemctl option rather than a unit name.
  - confidence: 0.9

  - body: The managed drop-in can contain gateway tokens, but the writer uses direct `writeFile` on a predictable HOME-derived path. Please use a no-symlink/atomic write pattern or explicitly reject unsafe path components before writing secret-bearing unit artifacts.
  - confidence: 0.82

  - body: The partial rewrite path re-renders kept assignments through `renderEnvLines`, which filters on `value.trim()` and writes `rawValue.trim()`. That drops valid empty assignments like `FOO=` and strips intentional whitespace from user-owned values during migration.
