  - body: The PR head's uninstall path still resolves the gateway service name instead of `resolveSystemdServiceName(env)`, so `OPENCLAW_SYSTEMD_UNIT=openclaw-node` would disable `openclaw-gateway.service` while removing the node unit path.   - confidence: 0.93

  - body: The new validator still accepts names beginning with `-`, and the affected `systemctl` calls pass the unit operand without a `--` terminator. That leaves `OPENCLAW_SYSTEMD_UNIT=--machine=...` style values parseable as systemctl options instead of a unit name, so reject leading dashes and/or terminate operands consistently.
  - confidence: 0.86
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:627-638`
  - body: This writes and backs up predictable HOME-derived unit/drop-in paths with direct `copyFile`/`writeFile` calls, including secret-bearing managed env data. Before landing the new drop-in layout, use a no-follow/atomic write strategy or get an explicit accepted threat model for symlink/hardlink and permission behavior.
  - confidence: 0.84
- **[P1] Preserve kept Environment values when stripping:** `src/daemon/systemd-unit.ts:270-273`
  - body: When a mixed `Environment=` line is partially stripped, the kept assignments are re-rendered through `renderEnvLines`, which filters empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration, so reconstruct kept assignments without dropping or trimming their values.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff changes secret-bearing systemd service files and leaves concrete option-injection, symlink/write-path, and Environment= rewrite concerns unresolved.

Concerns:

- **[high] Option-like unit names can reach systemctl:** `src/daemon/systemd.ts:55`
  - body: `validateSystemdServiceName` permits leading `-`, and subsequent `systemctl` calls pass unit names without `--`, so an environment-provided unit override can be interpreted as an option rather than a unit operand.
  - confidence: 0.86
- **[high] Predictable service-file writes are not link-hardened:** `src/daemon/systemd.ts:627`
  - body: The new drop-in writer and backup path use direct copy/write calls on HOME-derived paths, which can overwrite or leak secret-bearing unit content through symlink or hardlink redirection in a hostile local environment.
  - confidence: 0.83

  - body: The rewrite path can drop empty kept assignments such as `LD_PRELOAD=`, changing the effective service environment during upgrade and potentially re-enabling inherited values the operator intentionally cleared.
