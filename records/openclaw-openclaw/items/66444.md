  - body: Current main retries `systemctl enable`/`restart` once after another `daemon-reload` when systemd says the newly written unit does not exist.   - body: The uninstall path uses `resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)`, so `OPENCLAW_SYSTEMD_UNIT=openclaw-node` disables `openclaw-gateway.service` while deleting `openclaw-node.service`. Current main uses `resolveSystemdServiceName(env)` here and has regression coverage for custom node units.
  - confidence: 0.95
- **[P1] Reject option-like unit names before systemctl:** `src/daemon/systemd.ts:55-63`
  - body: `SYSTEMD_SERVICE_NAME_PATTERN` still allows names beginning with `-`, and the later `systemctl` calls pass the unit name without a `--` separator. An `OPENCLAW_SYSTEMD_UNIT` override such as `--user` or `-Hhost` can be parsed as a systemctl option instead of a unit operand.
  - confidence: 0.88
- **[P1] Carry current env-file source contracts forward:** `src/daemon/systemd.ts:527-533`
  - body: The PR head's writer ignores the current `environmentValueSources` split that decides inline versus file-managed secrets, node-specific `node.systemd.env`, and backup sanitization. Replacing current main with this path can move node/file-backed secrets or stale env-file values back into the wrong service artifact.
  - confidence: 0.9
- **[P1] Keep managed service secret writes symlink-safe:** `src/daemon/systemd.ts:627-638`
  - body: The new drop-in can contain managed tokens, but it is written directly to a predictable HOME-derived path, and the migration backup uses direct `copyFile` to another predictable path. Before this stores secrets in a new file, the writer needs explicit no-symlink/permission handling or a documented threat-model decision.
  - confidence: 0.82
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When only some assignments on an `Environment=` line are stripped, the kept assignments are re-rendered through `renderEnvLines`, which filters empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` or intentionally whitespace-bearing values during migration.
  - confidence: 0.86
