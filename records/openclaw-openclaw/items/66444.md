Full review comments:

- **[P1] Preserve the activation retry:** `src/daemon/systemd.ts:735`
  - body: This PR head goes straight from `daemon-reload` to `enable` and fails immediately on a missing-unit response. Current main retries the failed `enable`/`restart` once after a second reload, with regression coverage for the migrated systemd-user race, so this branch needs to carry that behavior forward before merge.
  - confidence: 0.93
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: `uninstallSystemdService` still resolves the gateway service name from `OPENCLAW_PROFILE`, ignoring `OPENCLAW_SYSTEMD_UNIT`. Current main uses `resolveSystemdServiceName(env)` here, so node/custom unit uninstall would again disable the wrong unit or leave the intended unit behind.
  - confidence: 0.92
- **[P1] Reject option-like unit names:** `src/daemon/systemd.ts:62`
  - body: The new validation still accepts service names that start with `-`, and the later `systemctl` calls pass the unit operand without a `--` terminator. A custom `OPENCLAW_SYSTEMD_UNIT` can still be parsed as a systemctl option, so the hardening needs to reject leading dashes and/or terminate options consistently.
  - confidence: 0.86
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:628-638`
  - body: The new writer copies and writes predictable HOME-derived unit, backup, and drop-in paths directly while the drop-in can contain managed secrets. Before landing a secret-bearing layout migration, the write path needs no-symlink/permission hardening or an explicit maintainer threat-model decision with tests.
  - confidence: 0.8
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-273`
  - body: When stripping only some assignments from an `Environment=` line, the code re-renders kept entries through `renderEnvLines`, which filters empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=`, so partial rewrites need to preserve empty and whitespace-bearing values.
  - confidence: 0.87
