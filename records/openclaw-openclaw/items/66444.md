## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-68`
  - body: `validateSystemdServiceName` still allows values that start with `-`, and this branch passes the resulting unit name to `systemctl` without a `--` operand terminator. A custom `OPENCLAW_SYSTEMD_UNIT=-H...` value can be parsed as a systemctl option instead of a unit name, so reject leading dashes and/or terminate operands consistently.
  - confidence: 0.94
- **[P1] Preserve the activation retry from current main:** `src/daemon/systemd.ts:735-738`
  - body: This activation path fails immediately after the first `enable` or `restart` error. Current main retries `daemon-reload` plus the failed action once when systemd reports `Unit file ... does not exist`; carry that behavior into this branch before merging the systemd migration.
  - confidence: 0.91
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: `uninstallSystemdService` disables `resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)` even though removal uses the override-aware unit path. On `OPENCLAW_SYSTEMD_UNIT=openclaw-node` or other custom units, this disables the gateway unit and removes the custom unit file, leaving the actual custom service enabled.
  - confidence: 0.92
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:593-638`
  - body: The migration writes and backs up predictable HOME-derived unit/drop-in paths with direct `writeFile` and `copyFile`. Because the new drop-in can carry service secrets, this needs a no-follow/atomic write path or an explicit maintainer-approved threat model before landing.
  - confidence: 0.86
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When only some assignments are stripped, the code re-renders kept values through `renderEnvLines`, which filters empty values and trims the rest. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration; reconstruct kept assignments without dropping or trimming values and add focused tests.
  - confidence: 0.88
