## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:56-63`
  - body: `validateSystemdServiceName` still accepts names that begin with `-`, and this PR passes the resulting unit name to `systemctl` without a `--` terminator. A value such as `OPENCLAW_SYSTEMD_UNIT=-Hhost.service` can be parsed as a `systemctl` option instead of a unit operand, so reject leading dashes or terminate operands consistently.
  - confidence: 0.91
- **[P1] Preserve the current missing-unit retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` plus the failed enable/restart when systemd says the freshly written unit does not exist, which closed a real install/update regression. This PR head calls each action once, so the carry needs to preserve that retry before it can land.
  - confidence: 0.9
- **[P1] Use the override-aware service name on uninstall:** `src/daemon/systemd.ts:794-796`
  - body: The uninstall path disables `resolveGatewaySystemdServiceName(...)` while deleting paths resolved from `OPENCLAW_SYSTEMD_UNIT`. For node/custom installs this can disable the gateway unit while removing the custom unit file, leaving the real custom service enabled; use `resolveSystemdServiceName(env)` here like current main.
  - confidence: 0.9
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:628-637`
  - body: The migration writes backups, the main unit, and the managed drop-in through predictable HOME-derived paths with direct `copyFile`/`writeFile`. Because the drop-in can contain service tokens, the PR needs no-symlink/safe-write handling or an explicit accepted threat model before merge.
  - confidence: 0.84
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:41-51`
  - body: When a mixed `Environment=` line is partially stripped, the kept assignments are re-rendered through `renderEnvLines`, which drops empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration, so preserve the original assignment values when rewriting.
  - confidence: 0.88
