Full review comments:

- **[P1] Carry current env-file source contracts forward:** `src/daemon/systemd.ts:527-533`
  - body: This rewrite drops `environmentValueSources` from the systemd writer, but current main uses that contract to keep file-only and inline-and-file secrets in owner-only env files, remove stale shell-reference values, and sanitize backups. Rebase the drop-in migration on top of the current writer instead of replacing those source-aware paths, or upgrades can re-inline or preserve the wrong secret state.
  - confidence: 0.91
- **[P1] Preserve the activation retry on current main:** `src/daemon/systemd.ts:735-740`
  - body: The PR head calls `systemctl enable` and `restart` once after daemon-reload. Current main retries daemon-reload plus the failed action when systemd reports `Unit file ... does not exist`, and the current regression test covers that migrated-root timing failure; dropping it can bring back install failures after update.
  - confidence: 0.9
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: Uninstall still resolves the gateway unit name even though the unit path and drop-in path use `OPENCLAW_SYSTEMD_UNIT`. On node-only or custom-unit installs this disables/removes the wrong unit, regressing the current-main custom-unit uninstall test.
  - confidence: 0.94
- **[P1] Reject option-like custom systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new validator still allows names beginning with `-`, and the later `systemctl` calls pass the unit operand without a `--` terminator. Reject leading dashes and/or terminate systemctl options consistently before claiming unsafe custom unit names are handled.
  - confidence: 0.87
- **[P1] Keep managed service secret writes symlink-safe:** `src/daemon/systemd.ts:627-638`
  - body: The migration writes a new secret-bearing drop-in and backs up the main unit through predictable HOME-derived paths using direct `copyFile`/`writeFile`. Before this layout lands, the write path needs symlink-safe behavior or an explicit accepted threat model, because a controlled service directory can redirect token-bearing writes.
  - confidence: 0.8
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When only some assignments are stripped, the code re-renders the kept assignments through `renderEnvLines`, which filters empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` when a managed key is removed from the same line; preserve the original tokens or use a renderer that does not drop/trim kept values.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff changes token-bearing systemd service storage and command execution paths, and it still has concrete option-injection, symlink/write, and environment-semantics concerns.

Concerns:

- **[high] Option-like systemd unit names remain accepted:** `src/daemon/systemd.ts:55`
  - body: `OPENCLAW_SYSTEMD_UNIT` validation permits leading `-` and systemctl invocations do not terminate options before the unit operand, so a custom unit name can be parsed as a systemctl option instead of a unit.
  - confidence: 0.87
- **[high] Secret-bearing drop-in writes are not symlink-safe:** `src/daemon/systemd.ts:637`
  - body: The PR writes the managed drop-in and backup through predictable HOME-derived paths with direct file operations; if that directory tree is link-controlled, managed env values can be redirected or exposed.
  - confidence: 0.8
- **[medium] Missing sentinel can put secrets in the main unit:** `src/daemon/systemd.ts:593`
  - body: When the managed-env sentinel is absent, the splitter treats all env as user-owned and the fresh main unit is written without an explicit restrictive mode, so caller mistakes can put token values in a less-protected service file.
  - confidence: 0.74
- **[medium] Environment rewrites can remove clearing assignments:** `src/daemon/systemd-unit.ts:271`
  - body: The partial rewrite path can drop `FOO=` or trim whitespace-only values, which may re-enable inherited environment behavior operators intentionally cleared.
  - confidence: 0.84
