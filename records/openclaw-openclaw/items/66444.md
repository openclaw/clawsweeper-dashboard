Full review comments:

- **[P1] Preserve the missing-unit activation retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` plus the failed `enable` or `restart` once when systemd says the freshly written unit does not exist, fixing https://github.com/openclaw/openclaw/issues/65184. This head goes back to one `enable` and one `restart`, so migrated installs can hit the same service recovery failure again.
  - confidence: 0.93
- **[P1] Honor custom systemd units during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: This path ignores `OPENCLAW_SYSTEMD_UNIT` by deriving the disabled unit from the gateway profile, while current main uses `resolveSystemdServiceName(env)` for the fix in https://github.com/openclaw/openclaw/issues/68287. A node-only install can therefore install `openclaw-node.service` but uninstall or disable `openclaw-gateway.service`.
  - confidence: 0.94
- **[P1] Reject option-like systemd unit names:** `src/daemon/systemd.ts:55-63`
  - body: The new validator still allows names beginning with `-`, and the resulting unit name is passed to `systemctl` without a `--` separator in enable/restart/disable/action paths. Reject leading dashes or consistently terminate options before unit operands so an override cannot be parsed as a systemctl option.
  - confidence: 0.86
- **[P1] Avoid direct writes for secret-bearing drop-ins:** `src/daemon/systemd.ts:637-638`
  - body: The new managed drop-in can carry gateway tokens, but this writes the HOME-derived drop-in path directly. Please use an atomic/no-follow write strategy or add a maintainer-approved threat-model note before storing secrets through this new predictable path.
  - confidence: 0.84
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-273`
  - body: When stripping one managed assignment from a multi-assignment `Environment=` line, this re-renders the kept assignments through `renderEnvLines`, which filters empty values and trims whitespace. Valid systemd assignments such as `LD_PRELOAD=` can be silently removed or changed during migration.
  - confidence: 0.88
