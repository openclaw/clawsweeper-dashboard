## Security Review

Status: needs_attention

Summary: The diff changes secret-bearing systemd unit/drop-in storage and systemctl operands, and it still has concrete unit-name and file-write hardening gaps.

Concerns:

- **[high] Option-like systemd unit name:** `src/daemon/systemd.ts:55`
  - body: The validator accepts leading '-' and systemctl receives the unit name as a normal argv element without a '--' separator, so an environment-provided custom unit name can be parsed as an option by systemctl.
  - confidence: 0.94
- **[high] Symlink-sensitive secret-bearing writes:** `src/daemon/systemd.ts:627`
  - body: The PR writes and backs up HOME-derived unit/drop-in paths with copyFile/writeFile, and the new drop-in can contain managed tokens, so an attacker-controlled HOME tree or symlinked path can redirect writes unless the writer is hardened or explicitly out of threat model.
  - confidence: 0.84
- **[medium] Systemd env rewrite changes clearing semantics:** `src/daemon/systemd-unit.ts:270`
  - body: Dropping `Environment=FOO=` during partial migration can re-enable inherited environment values, which is security-sensitive for variables that influence process loading or runtime behavior.
  - confidence: 0.82
