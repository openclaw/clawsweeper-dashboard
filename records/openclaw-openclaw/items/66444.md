Overall confidence: 0.93

Full review comments:

- **[P1] Preserve the activation retry:** `src/daemon/systemd.ts:735-740`
  - body: Current main retries `daemon-reload` plus the failed `enable` or `restart` once when systemd reports the freshly written unit does not exist. This branch goes back to a single enable/restart attempt, so migrated installs can hit the same missing-unit race fixed for https://github.com/openclaw/openclaw/issues/65184.
  - confidence: 0.95
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: Install/activation uses `resolveSystemdServiceName`, but uninstall still disables `${resolveGatewaySystemdServiceName(...)}`. With `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, the branch can remove the node unit file while disabling the gateway unit, regressing the custom-unit fix from current main.
  - confidence: 0.96
- **[P1] Reject option-like unit names:** `src/daemon/systemd.ts:62`
  - body: The new validator still allows service names beginning with `-`, and later `systemctl` calls pass the unit operand without `--`. A value such as `OPENCLAW_SYSTEMD_UNIT=-Hhost` can be parsed by systemctl as an option rather than a unit name, so the hardening is incomplete.
  - confidence: 0.9
- **[P1] Preserve kept Environment values when stripping:** `src/daemon/systemd-unit.ts:270-272`
  - body: When only some assignments are stripped, kept assignments are rebuilt through `renderEnvLines`, which drops empty values and trims whitespace. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration rather than preserving the operator's remaining directive.
  - confidence: 0.9
- **[P1] Harden systemd unit and drop-in writes:** `src/daemon/systemd.ts:628-638`
  - body: The PR writes backups, the main unit, and the managed drop-in through predictable HOME-derived paths with direct `copyFile`/`writeFile`. Because the new drop-in can carry managed environment data, this needs no-follow/atomic-write hardening or an explicit maintainer threat-model decision before merge.
  - confidence: 0.86
- **[P3] Remove the release changelog edit:** `CHANGELOG.md:21`
  - body: `CHANGELOG.md` is release-owned in this repo; normal PRs should keep release-note context in the PR body or commit message. Please drop this manual changelog change from the branch.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff still has concrete systemd command-target and file-write security concerns before merge.

Concerns:

- **[high] Option-like systemd unit names remain accepted:** `src/daemon/systemd.ts:62`
  - body: The added validator allows a leading dash and the systemctl calls do not terminate options before the unit operand, so an environment-controlled unit name can be interpreted as a systemctl option.
