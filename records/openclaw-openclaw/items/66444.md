  - body: Current main retries `daemon-reload` plus the failed `enable`/`restart` once when systemd reports the written unit does not exist. This branch calls `enable` and `restart` directly, so the install race covered by current-main tests can fail again.
  - confidence: 0.96
- **[P1] Honor custom systemd units during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: The branch still derives the disabled unit from `resolveGatewaySystemdServiceName`, ignoring `OPENCLAW_SYSTEMD_UNIT`. A node-only install can disable `openclaw-gateway.service` while deleting `openclaw-node.service`, regressing the current-main custom-unit uninstall path.
  - confidence: 0.95
- **[P1] Reject option-like unit overrides:** `src/daemon/systemd.ts:55-63`
  - body: The new service-name validation still accepts values beginning with `-`, and the `systemctl` calls pass the unit operand without a `--` terminator. An override such as `--user`/`-H...` can be interpreted as a systemctl option rather than a unit name.
  - confidence: 0.9
- **[P1] Avoid direct writes for secret-bearing drop-ins:** `src/daemon/systemd.ts:637-638`
  - body: The managed drop-in carries gateway tokens but is written to a predictable HOME-derived path with direct `writeFile`/`copyFile` behavior. Before adding this token-bearing file, the writer needs no-follow/atomic handling or an explicit accepted threat model for symlink and hardlink overwrite cases.
  - confidence: 0.86
- **[P2] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:270-272`
  - body: When stripping only some assignments from an `Environment=` line, the code re-renders kept entries through `renderEnvLines`, which drops empty values and trims whitespace. Valid systemd assignments like `LD_PRELOAD=` can disappear during migration, changing the operator's unit semantics.
  - confidence: 0.88
- **[P2] Remove the release-owned changelog edit:** `CHANGELOG.md:17`
  - body: `CHANGELOG.md` is release-generated in this repo, and this branch also removes unrelated release entries. Keep this release-note context in the PR body or squash message instead of editing the changelog in a normal PR.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff touches token-bearing systemd service files and has concrete option-injection, file-write, and environment-rewrite concerns that need attention before merge.
