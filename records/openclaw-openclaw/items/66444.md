Full review comments:

- **[P1] Reject option-like unit names before systemctl:** `src/daemon/systemd.ts:55-68`
  - body: `validateSystemdServiceName` still accepts values beginning with `-`, and this branch passes the resulting unit name to `systemctl` without a `--` operand terminator. A custom `OPENCLAW_SYSTEMD_UNIT=--user`/`-H...` style value can be parsed as a systemctl option instead of a unit name, so reject leading dashes and/or terminate operands consistently before merge.
  - confidence: 0.94
- **[P1] Preserve the activation retry from current main:** `src/daemon/systemd.ts:735-738`
  - body: This edited activation path fails immediately after the first `enable` error. Current main retries `daemon-reload` plus the failed `enable`/`restart` once when systemd reports `Unit file ... does not exist`, with regression coverage for migrated user managers; carry that behavior into this branch before merging the systemd migration.
  - confidence: 0.9
- **[P1] Honor custom unit names during uninstall:** `src/daemon/systemd.ts:794-796`
  - body: `uninstallSystemdService` disables `resolveGatewaySystemdServiceName(env.OPENCLAW_PROFILE)` even though the file removal uses `resolveSystemdUnitPath(env)`. On node/custom installs with `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, this disables the gateway unit and then removes the node unit file, leaving the real node service enabled; use `resolveSystemdServiceName(env)` as current main does.
  - confidence: 0.91
- **[P1] Harden managed unit and drop-in writes:** `src/daemon/systemd.ts:593-638`
  - body: The migration writes and backs up predictable HOME-derived unit/drop-in paths with direct `writeFile`/`copyFile`. Because the new drop-in can carry service secrets, this needs a no-follow/atomic or explicitly threat-modeled write path before landing, not just chmod after write.
  - confidence: 0.86
- **[P1] Preserve kept Environment assignment values:** `src/daemon/systemd-unit.ts:269-274`
  - body: When only some assignments are stripped, the code re-renders kept values through `renderEnvLines`, which filters empty or whitespace-only values and trims the rest. That changes valid systemd semantics such as `Environment=LD_PRELOAD=` during migration; reconstruct kept assignments without dropping or trimming values and add focused tests.
  - confidence: 0.88
