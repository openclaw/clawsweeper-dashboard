## Summary

Keep open: current main and v2026.6.9 still use the built-in 8KB WebChat history projection cap with no persisted config key, and https://github.com/openclaw/openclaw/pull/95321 is an open candidate fix rather than a merged implementation.

## What This Changes

Request a supported WebChat/Gateway configuration setting to raise the chat-history display projection cap above the built-in 8KB default.

## Best Possible Solution

Keep this issue open until the linked PR lands, is replaced by a maintainer-sponsored config-contract PR, or maintainers explicitly choose to keep bounded history plus the full-message reader with no new config key.

## Reproduction Assessment

Yes. Source inspection shows `chat.history` display projection uses an 8KB default unless an RPC `maxChars` override is supplied, while ordinary WebChat/Control UI reloads have no accepted persisted config key to raise it.

## Solution Assessment

Unclear. A config knob directly solves the reporter's local/private deployment workflow, but current docs and recent history favor bounded history plus `chat.message.get`, so maintainers need to choose the product direction before merging a new config surface.
