## Summary

Keep open: current main and v2026.6.9 still use the hardcoded 8,000-character history projection cap with no persisted WebChat config section, and https://github.com/openclaw/openclaw/pull/95321 is the open candidate fix rather than a merged implementation.

## What This Changes

The issue requests a supported persisted configuration knob to raise the WebChat/Control UI chat-history text projection cap above the hardcoded default.

## Best Possible Solution

Keep this issue open as the canonical report while maintainers review, revise, land, or close the linked config-surface PR.

## Solution Assessment

Unclear. A config knob would solve the reporter's requested workflow, but maintainers need to choose it over the current bounded-history plus `chat.message.get` full-message-reader design and require schema, docs, bounds, tests, and WebChat proof if they do.
