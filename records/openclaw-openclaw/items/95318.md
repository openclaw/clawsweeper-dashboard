Request to make the WebChat/Control UI history text projection cap configurable instead of using a hardcoded 8 KB limit after history reload.

## Best Possible Solution

Have maintainers review the linked PR and either land a documented schema/test/docs-backed config path or explicitly keep the current bounded-history plus full-message-reader design.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.8` use an 8,000-character default when no RPC `maxChars` is supplied, the HTTP history path hardcodes that default, and the config schema/docs expose no persisted WebChat history-cap setting.

## Solution Assessment

Unclear. A config knob is a plausible narrow fix for users who want larger inline history bubbles, but current docs and prior maintainer discussion intentionally prefer bounded history plus `chat.message.get`, so this needs maintainer product direction.
