## Summary

Keep open: current main and v2026.6.10 still hardcode the WebChat history projection cap at 8,000 characters with no persisted config key, and https://github.com/openclaw/openclaw/pull/95321 is the open linked candidate fix rather than a merged implementation.

## What This Changes

The issue asks for a supported WebChat/Gateway configuration setting to raise the chat-history text projection cap used after history reload.

## Best Possible Solution

Keep this open until the linked config-contract PR lands, is replaced by a maintainer-sponsored alternative, or maintainers explicitly decide the bounded-history plus full-message-reader design should remain non-configurable.

## Reproduction Assessment

Yes. Source inspection shows normal WebChat history reloads take the default projection path, where the resolver ignores config and falls back to 8,000 characters unless an RPC override is supplied.

## Solution Assessment

Unclear. A bounded gateway config key would solve the reporter's requested workflow, but maintainers need to choose that public config surface over the current bounded-history plus full-message-reader contract.
