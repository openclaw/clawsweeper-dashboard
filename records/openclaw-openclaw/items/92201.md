## Summary

The central user problem remains necessary on current main: recovery still ignores structured raw provider details when the user-facing error is genericized, and the newly reported fresh-signature corruption is not covered by the earlier concatenation fix or the later durable transcript repair.

## What This Changes

Investigate intermittent Anthropic thinking-signature corruption in the embedded runner and make replay recovery classify the raw provider validation error.

## Best Possible Solution

Classify the existing structured raw provider error in the one-shot recovery wrapper with regression coverage, while diagnosing the fresh-signature corruption separately from redacted Anthropic event captures and same-model live replay proof.

## Reproduction Assessment

Yes for the recovery gap: source shows that matching provider details retained only in `rawError` are ignored by the current predicate. The intermittent production of invalid fresh signatures is strongly evidenced by the report but not yet deterministically reproduced on current main.

## Solution Assessment

Partly yes: using the structured raw provider error is the narrowest maintainable recovery fix, but stream assembly should change only after a captured real event sequence identifies the violated invariant.
