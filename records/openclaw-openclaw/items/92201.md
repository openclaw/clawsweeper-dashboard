## Summary

Keep open. Current main and latest v2026.6.9 still have remaining Anthropic signed-thinking replay and recovery-carrier gaps, while multiple open PRs only cover separate slices of the broader session-poisoning problem.

## What This Changes

The issue asks OpenClaw to stop corrupting or replaying invalid Anthropic thinking signatures in embedded runner sessions and to make recovery reliably strip thinking blocks and retry when the provider rejects replay.

## Best Possible Solution

Keep this as the canonical issue until the focused recovery-carrier and replay-policy PRs land or are rejected, with final behavior that avoids unrecoverable invalid signed-thinking replay while preserving active tool-turn continuity.

## Reproduction Assessment

Yes, source-level. Current main still has reachable paths where async embedded streams can bypass recovery pumping and native Anthropic replay can preserve signed Claude 4 thinking; I did not run a live Anthropic reproduction in this read-only review.
