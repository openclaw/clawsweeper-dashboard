## What This Changes

The issue requests a new Plugin SDK before_persistence_write hook for blocking, auditing, or classifying durable writes that can cross session boundaries.

## Best Possible Solution

Define one versioned durable-write Plugin SDK contract with explicit coverage, target taxonomy, provenance semantics, veto or mutation behavior, failure policy, docs, and representative tests across durable write paths.

## Solution Assessment

Unclear. A before_persistence_write hook is plausible, but maintainers need to choose the contract shape, target taxonomy, provenance coverage, veto versus mutation semantics, and failure behavior before implementation.
