## Summary

Keep open: current main still lacks a durable persistence-write hook or equivalent pre-write context/veto, while representative memory, file, cron, and config write paths still commit through their owner code without a plugin-facing durable-write policy surface.

## What This Changes

The issue requests a new plugin hook that fires before durable state writes such as memory, file, cron, config, or preference persistence.

## Best Possible Solution

Design a backward-compatible durable-write policy surface with stable target, path, session, and provenance metadata, then wire it consistently through memory, file, cron, config, and preference writes with SDK docs and contract tests.

## Reproduction Assessment

Not applicable as a runtime bug. The high-confidence verification path is source and docs inspection on current main, which shows no durable pre-write hook and representative durable writes that still bypass such a hook.

## Solution Assessment

Unclear. A dedicated hook is a plausible maintainable direction, but the issue leaves product/API decisions open, so maintainers should settle the exact hook shape before implementation.
