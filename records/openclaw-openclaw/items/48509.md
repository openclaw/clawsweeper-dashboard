## Summary

Keep open: current main and v2026.6.10 still lack a durable-state pre-write plugin hook, and the remaining work is a public Plugin SDK/security API decision rather than conservative cleanup.

## What This Changes

The issue requests a new plugin hook that fires before durable writes such as memory, files, cron jobs, config, and preferences.

## Best Possible Solution

Keep this as a maintainer-owned Plugin SDK/security design item and resolve the durable-write taxonomy with the related hook/provenance proposals before adding API.

## Reproduction Assessment

Yes, source-level: current hook types, runtime dispatch, docs, and sampled file/config/cron write paths show generic tool and transcript/message hooks but no durable-state pre-write hook.

## Solution Assessment

Unclear until maintainers choose the public API shape; a dedicated hook is plausible, but it should be coordinated with action classification, provenance, and persistent-state contracts rather than added piecemeal.
