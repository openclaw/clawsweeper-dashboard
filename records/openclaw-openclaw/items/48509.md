## Summary

Current main still lacks a durable persistence-write plugin hook or equivalent pre-write context/veto, so this remains a real plugin-SDK design gap rather than a cleanup close.

## What This Changes

The issue requests a `before_persistence_write` hook so plugins can inspect or block durable memory, file, cron, config, and preference writes before they cross session boundaries.

## Best Possible Solution

Add a backward-compatible durable-write policy surface, or an explicit durable-write context on `before_tool_call`, with stable target/session/path/provenance metadata and focused SDK docs/tests for memory, file/media, cron, config, and preference writes.

## Reproduction Assessment

Not applicable as a runtime bug. The high-confidence verification path is source and docs inspection against current main showing no durable pre-write hook and representative durable writes that still bypass such a hook.

## Solution Assessment

Unclear. The requested hook is a plausible maintainable direction, but the issue intentionally leaves product/API decisions open, so maintainer design should settle the exact hook shape before implementation.
