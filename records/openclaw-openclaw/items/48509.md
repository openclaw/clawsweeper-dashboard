## What This Changes

The issue requests a new Plugin SDK hook that fires before durable state writes such as memory, file, cron, config, and preference writes.

## Best Possible Solution

Define a versioned Plugin SDK durable-write policy contract with explicit target taxonomy, provenance, veto or mutation semantics, trust model, and coverage tests across representative persistent write paths.

Unclear. A before_persistence_write hook is plausible, but maintainers should choose the durable-write contract, target taxonomy, provenance coverage, trust semantics, and block/mutate behavior before landing one narrow SDK event.
