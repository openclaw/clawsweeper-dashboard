## Summary

Keep this open: current main and v2026.6.10 still lack a dedicated durable-state pre-write plugin hook, and the remaining work is a public Plugin SDK/security contract decision rather than conservative cleanup.

## What This Changes

The issue requests an additive `before_persistence_write` plugin hook for memory, file, cron, config, and preference writes that persist across sessions.

## Best Possible Solution

Have maintainers design one additive durable-write policy contract that covers write target, provenance/security exposure, blocking semantics, and tests across the SDK and the durable write sites before implementation.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 expose adjacent generic tool, transcript, message-write, session, and cron hooks, but no durable-state pre-write hook or equivalent event fields.

## Solution Assessment

Unclear until maintainers choose the public SDK shape. A dedicated hook is plausible, but the safer path is a coordinated plugin API/security design that avoids one-sided coverage of only some durable write sites.
