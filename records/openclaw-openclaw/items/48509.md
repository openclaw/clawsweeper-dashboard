## Summary

Keep open: current main still lacks a durable persistence-write hook or equivalent typed pre-write policy context, and the related hook/provenance/security design threads remain open rather than merged implementations.

## What This Changes

Add a `before_persistence_write` plugin hook that can classify and block durable state writes such as memory, file, cron, config, and preference changes.

## Best Possible Solution

Define an additive, versioned Plugin SDK durable-write policy surface with stable target, path, session, provenance, veto, and coverage semantics documented and tested across representative persistent write paths.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The source-verification path is high confidence because current hook types, runtime dispatch, and public docs have no durable write pre-veto hook or equivalent context.

## Solution Assessment

Unclear. A dedicated `before_persistence_write` hook is plausible, but maintainers should first settle the durable-write policy contract, taxonomy, coverage, mutation/read semantics, and trust model.
