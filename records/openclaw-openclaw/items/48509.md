## What This Changes

The issue requests a new Plugin SDK `before_persistence_write` hook for durable writes such as memory, files, cron jobs, config, and preferences.

## Best Possible Solution

Define a versioned Plugin SDK durable-write policy contract with explicit coverage, provenance, trust semantics, veto or mutation behavior, and tests across representative persistent write paths.

Unclear. A `before_persistence_write` hook is plausible, but maintainers need to choose the durable-write contract, target taxonomy, provenance/redaction coverage, veto versus mutation semantics, and failure behavior before implementation.
