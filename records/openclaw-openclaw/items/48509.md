## Summary

Keep open: current main and the latest shipped release still have generic tool, transcript, message, and cron hooks, but no dedicated durable persistence-write hook or equivalent durable-write policy context. The request remains a public Plugin SDK and security-policy design item, not conservative cleanup closure.

## What This Changes

Request a new Plugin SDK hook that fires before durable state writes so plugins can inspect, block, or audit cross-session persistence changes.

## Best Possible Solution

Define a versioned Plugin SDK durable-write policy contract with explicit target taxonomy, provenance, veto or mutation semantics, trust model, and coverage tests across representative persistent write paths.

## Reproduction Assessment

Not applicable as a runtime bug; this is a new Plugin SDK capability request. Source inspection gives high confidence that current main has no durable pre-write hook or equivalent persistence context.

## Solution Assessment

Unclear until maintainers settle the API and trust model. A before_persistence_write hook is plausible, but the durable-write contract, taxonomy, provenance coverage, block versus mutation behavior, and failure semantics need maintainer design first.
