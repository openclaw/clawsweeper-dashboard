## Summary

Keep open: current main and the latest stable release still expose generic tool, transcript/message, cron, and session hooks, but no dedicated pre-write durable-state hook for memory, file, cron, config, or preference writes. This remains a coherent public Plugin SDK and security-policy design item that needs maintainer API and security review.

## What This Changes

Request a new Plugin SDK hook that fires before durable state writes so plugins can block or audit memory, file, cron, config, and preference persistence.

## Best Possible Solution

Define one additive, documented durable-write contract with redacted payloads, timeout/failure semantics, and focused tests across selected writer boundaries before implementation.

## Reproduction Assessment

Yes, at source level: current hook names, event types, runner dispatch, docs, and representative durable write paths lack before_persistence_write or an equivalent durable-write hook.

## Solution Assessment

Unclear pending maintainer design: a dedicated hook is plausible, but maintainers still need to choose taxonomy, block versus mutate semantics, redaction, failure policy, and relation to provenance/read hooks.
