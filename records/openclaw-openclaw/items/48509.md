## Summary

Keep open: current main still lacks a durable persistence-write plugin hook or equivalent pre-write policy surface, and the related CaMeL/provenance issues are open design context rather than merged implementation.

## What This Changes

The issue requests a new Plugin SDK hook that fires before durable writes such as memory, file, cron, config, or preference persistence.

## Best Possible Solution

Define an additive, versioned Plugin SDK policy surface for durable writes, with stable target, path, session, provenance, veto/approval, and coverage semantics documented and tested across representative persistent write paths.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The source-verification path is high confidence because current hook types, dispatch, and docs have no durable write pre-veto hook or equivalent context.

## Solution Assessment

Unclear. A dedicated before_persistence_write hook is plausible, but maintainers should first choose the durable-write policy contract, taxonomy, coverage, and trust semantics rather than landing a narrow one-off hook.
