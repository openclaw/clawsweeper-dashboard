## Summary

Keep open: current main and the latest release still expose only generic tool-call plus transcript/message persistence hooks, with no durable-state pre-write Plugin SDK contract. The requested hook is a security-sensitive SDK feature that needs maintainer product and security review before implementation.

## What This Changes

Requests a new Plugin SDK `before_persistence_write` hook for durable memory, file, cron, config, and preference writes before they cross session boundaries.

## Best Possible Solution

Define one versioned durable-write Plugin SDK contract with explicit coverage, event taxonomy, provenance semantics, veto or mutation behavior, failure policy, and representative tests across durable write paths.

## Reproduction Assessment

Not applicable as a runtime bug; this is a new Plugin SDK API request. Source and docs inspection show current main lacks a durable pre-write hook or equivalent persistence-write context.

## Solution Assessment

Unclear. A dedicated `before_persistence_write` hook is plausible, but maintainers need to choose the contract shape, target taxonomy, provenance coverage, veto versus mutation semantics, and failure behavior before implementation.
