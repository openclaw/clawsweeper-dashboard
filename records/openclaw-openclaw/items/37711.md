## Summary

Keep open. Current main still documents Hostinger Docker Manager as a supported install path, but the Hostinger-owned image/template/proxy path named in the report is not present in the repository, so there is no source-level proof that the reported asset truncation is fixed.

## What This Changes

The issue reports that fresh Hostinger Docker Manager deployments load a black Control UI because `/assets/index-*.js` fails with `ERR_CONTENT_LENGTH_MISMATCH`.

## Best Possible Solution

Keep this open until a maintainer or Hostinger-facing owner verifies a fresh Docker Manager deployment by immutable image digest and compares the failing asset's headers/body size through the Hostinger URL against the file inside the running container. If the image is stale, rebuild or repoint it; if the proxy/template truncates assets, fix that delivery path or document a confirmed workaround in the Hostinger guide.
