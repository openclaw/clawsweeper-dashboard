## Summary

Keep open: the report is source-backed on current main, and the related merged fix does not currently protect the outbound message-tool path that turns an allowlisted bare Discord user snowflake into a channel target.

## What This Changes

Report that outbound Discord `message(action=send)` to an allowlisted DM user ID still returns `Unknown Channel` after v2026.5.7 and https://github.com/openclaw/openclaw/pull/78625.

## Best Possible Solution

Restore the Discord-owned `messaging.targetResolver.resolveTarget` path to delegate to `resolveDiscordTarget` for allowFrom-aware numeric DM targets, then cover the message-tool path so raw allowed user IDs resolve to `user:<id>` before delivery.

## Reproduction Assessment

Yes by source inspection, though not by live Discord execution. On current main, a raw numeric Discord target can normalize to `channel:<id>` because the Discord plugin no longer registers the allowFrom-aware `resolveTarget` hook in `messaging.targetResolver`.

## Solution Assessment

No current-main solution is complete. The narrow maintainable fix is to wire the existing Discord allowFrom-aware resolver back into the plugin messaging target resolver, not to add broad core heuristics for Discord snowflakes.
