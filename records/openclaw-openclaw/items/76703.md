## Summary

Keep open: current main still lets Discord-visible assistant replies reach the shared outbound sender with only session metadata, while durable transcript persistence is gated on mirror data.

## What This Changes

The issue asks OpenClaw to persist every delivered Discord assistant response into the active session JSONL transcript.

## Best Possible Solution

Thread the existing idempotent DeliveryMirror transcript append through Discord normal delivery and preview-finalization paths, with tests proving successful sends and edits persist once.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection gives a current-main reproduction path: Discord delivery can succeed without mirror data, and the shared transcript append only runs when mirror data is present.

## Solution Assessment

Yes for the requested direction. The narrow maintainable solution is to reuse the existing mirror/idempotency transcript path rather than raw-appending JSONL or adding a new configuration mode.
