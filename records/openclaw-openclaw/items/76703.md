## Summary

Keep open. Current main still has a credible source-level gap between Discord-visible final/block delivery and durable assistant transcript persistence, so the report is valid enough for a focused repair rather than cleanup closure.

## What This Changes

The issue reports that a Discord-delivered assistant response in OpenClaw 2026.5.2 was not written to the session JSONL, causing the next turn to lose context.

## Best Possible Solution

Make Discord-visible final/block replies durable by coupling successful delivery, including preview final edits, to the existing idempotent assistant transcript append path.

## Reproduction Assessment

No. There is a medium-confidence source reproduction path because Discord can deliver visible final/block replies without a mirror object, but I did not run a live Discord/vLLM session against current main.

## Solution Assessment

Unclear on the exact trigger, but the maintainable fix direction is clear: reuse the idempotent transcript append helper after successful Discord delivery rather than raw-appending JSONL or changing channel visibility policy.
