## Summary

Keep open. Current main still has a source-backed gap where Discord-visible final/block replies can be delivered without the mirror data that shared outbound transcript persistence requires.

## What This Changes

The issue reports a Discord assistant reply being delivered to chat while the corresponding assistant turn is missing from the session JSONL transcript.

## Best Possible Solution

Make successful Discord-visible assistant delivery durable by threading an idempotent mirror append through the same-surface Discord delivery and preview-finalization paths.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show a credible current-main path: Discord same-surface replies can be delivered without mirror data, and outbound transcript persistence is mirror-gated.

## Solution Assessment

Unclear for the exact patch, but the requested direction is valid. The narrow maintainable fix is to reuse the existing mirror/idempotent transcript append path instead of raw-appending JSONL or adding a new config mode.
