## What This Changes

The issue asks OpenClaw to persist every Discord-delivered assistant response into the session JSONL transcript so later turns keep context.

## Best Possible Solution

Make successful Discord final/block delivery and preview finalization durable by using the existing idempotent assistant transcript append path while preserving Pi parentId transcript structure.

## Solution Assessment

No patch is under review yet. The narrow maintainable fix is to reuse the existing delivery mirror or appendAssistantMessageToSessionTranscript path, not raw JSONL appends or Discord visibility policy changes.
