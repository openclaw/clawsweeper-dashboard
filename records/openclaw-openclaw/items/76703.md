## Summary

Keep open. Current `main` still has a credible Discord-visible delivery path that does not request durable assistant transcript mirroring, while the expected behavior is documented as preserving delivered assistant replies on disk.

## What This Changes

Discord assistant replies can be delivered successfully to a channel while the corresponding assistant turn is missing from the session JSONL transcript.

## Best Possible Solution

Make Discord-visible assistant replies durable by coupling successful final/block delivery and preview finalization to the existing idempotent assistant transcript append path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a medium-confidence path: Discord can deliver final/block replies or finalize a preview without passing the mirror object that shared outbound persistence requires.

## Solution Assessment

Yes for direction, unclear for exact patch because no patch is present. The narrow maintainable fix is to reuse the existing idempotent transcript append helper after successful visible Discord delivery rather than raw-appending JSONL or changing broad outbound policy.
