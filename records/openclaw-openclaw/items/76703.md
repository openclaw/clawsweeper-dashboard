## Summary

Keep open. Current main still has a source-backed path where a Discord-visible final or preview-finalized assistant reply can be delivered without the mirror data required by shared outbound transcript persistence.

## What This Changes

The issue reports that a Discord-delivered assistant reply can be visible in channel output without being appended to the session JSONL transcript.

## Best Possible Solution

Make every successful Discord-visible assistant delivery durable by threading an idempotent mirror append through the Discord same-surface send and preview-finalization paths.

## Reproduction Assessment

No; no high-confidence live Discord/vLLM reproduction was established in this read-only review. Source inspection does show a credible current-main path: Discord final delivery lacks mirror data, and shared persistence is mirror-gated.

## Solution Assessment

Yes for the direction, but the exact patch still needs care. The narrow maintainable fix is to reuse the existing mirror/idempotent transcript append path rather than adding raw JSONL writes or a new configuration mode.
