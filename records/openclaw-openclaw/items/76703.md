## Summary

Keep open. The reported 2026.5.2 behavior still maps to a current-main source gap: Discord delivery can reach the channel without the mirror payload required for shared assistant transcript persistence.

## What This Changes

The issue requests that assistant replies successfully delivered to Discord be persisted in the session JSONL so later turns retain context.

## Best Possible Solution

Persist every successful Discord-visible final reply through the existing idempotent transcript mirror, including preview-finalized edits, while preserving duplicate suppression.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show a credible current-main path: Discord-visible replies can be delivered without mirror data while the shared transcript append path requires mirror data.

## Solution Assessment

Unclear for a specific patch, but the direction is valid. The narrow maintainable solution is to reuse the existing mirror/idempotent transcript append path rather than raw-appending JSONL or adding a new config mode.
