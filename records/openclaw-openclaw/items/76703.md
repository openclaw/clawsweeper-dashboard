## Summary

Keep open. Current main has a credible gap between Discord-visible final/block delivery and durable assistant transcript persistence, and the fresh report is not superseded by the older Discord delivery issues in the provided cluster.

## What This Changes

The issue requests that every assistant reply visible in Discord be persisted to the session JSONL so later turns retain context.

## Best Possible Solution

Make Discord-visible assistant replies durable by coupling successful final/block delivery or preview finalization to an idempotent assistant transcript append/check that preserves the Pi parentId chain.

## Reproduction Assessment

No live Discord/vLLM reproduction was run. Source inspection gives a medium-confidence reproduction path because Discord can successfully deliver final/block replies without passing the mirror object that shared outbound persistence requires.

## Solution Assessment

Unclear for the exact implementation, but the current separation is not the best guarantee. The narrow maintainable fix is to reuse the existing idempotent transcript append helper rather than raw-appending JSONL or changing Discord visibility defaults.
