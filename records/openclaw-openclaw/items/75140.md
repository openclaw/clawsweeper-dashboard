  - body: This public SDK subpath exposes `addTaskRegistryEventListener` over raw task registry events. Any loaded plugin could observe task text, labels, errors, agent IDs, runtimes, and status transitions without a scoped capability, documented stability contract, or security model.
  - confidence: 0.9

  - body: The recorder writes persistent task outcomes to `memory/.dreams/task-outcomes.jsonl` with plain `fs.appendFile`. Current memory events use a symlink-aware append helper and current dreaming state is SQLite-backed plugin state, so this creates an unsafe and divergent persistence path.
  - confidence: 0.91
- **[P2] Attach the listener to plugin lifecycle cleanup:** `extensions/memory-core/index.ts:31`
  - body: `registerTaskOutcomeRecorder(api)` returns an unsubscribe function, but the plugin registration discards it. Since this installs a process-global listener, plugin reload or disable paths can leave stale listeners active and duplicate or stale-write task outcomes.
  - confidence: 0.84
- **[P2] Document the new SDK subpath:** `scripts/lib/plugin-sdk-entrypoints.json:327`
  - body: Adding `task-events` to the package-exported SDK list makes it a plugin authoring contract, but the docs catalog and SDK documentation do not describe its stability or privacy boundary. Plugin authors would get an undocumented public API at the same time the event shape becomes hard to change.
  - confidence: 0.78
