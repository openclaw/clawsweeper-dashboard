- **[P1] Scope task-event subscription before publishing it:** `src/plugin-sdk/task-events.ts:14-17`
  - body: This public SDK subpath exports raw task registry events and `TaskRecord` types to plugins. `TaskRecord` carries task text, labels, errors, agent IDs, runtimes, status, and delivery metadata, so shipping it without a scoped capability, privacy model, and stability contract creates an overly broad third-party API.
  - confidence: 0.9
- **[P1] Use approved memory persistence for task outcomes:** `extensions/memory-core/src/task-outcome-recorder.ts:101-103`
  - body: The recorder writes persistent task outcomes to `memory/.dreams/task-outcomes.jsonl` via plain `fs.appendFile`. Current memory-core dreaming state is SQLite-backed and existing JSONL event logs use symlink-safe `appendRegularFile`, so this creates divergent state and bypasses the storage/security boundary.
  - confidence: 0.88
- **[P2] Wire task listener cleanup into plugin lifecycle:** `extensions/memory-core/index.ts:34`
  - body: `registerTaskOutcomeRecorder(api)` returns an unsubscribe function, but registration discards it. A process-global listener can survive reload, disable, or reset and duplicate stale writes; wire cleanup through `api.lifecycle.registerRuntimeLifecycle` or an equivalent lifecycle-owned path.
  - confidence: 0.84
