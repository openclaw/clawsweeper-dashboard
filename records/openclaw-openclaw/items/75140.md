- **[P1] Gate task-event subscription before publishing it:** `src/plugin-sdk/task-events.ts:14-17`
  - body: This exports a global task-registry listener as a public plugin SDK subpath.   - confidence: 0.89
- **[P1] Avoid direct workspace appends for outcome state:** `extensions/memory-core/src/task-outcome-recorder.ts:91`
  - body: The new recorder persists runtime task outcomes by appending to `<workspace>/memory/.dreams/task-outcomes.jsonl`. Current memory-core dreaming state is SQLite-backed plugin state, root policy rejects new JSONL sidecars for runtime/plugin state, and this direct `fs.appendFile` path follows workspace symlinks instead of using the existing symlink-safe memory event helper. Store these outcomes through plugin state or an approved safe memory event path instead.
