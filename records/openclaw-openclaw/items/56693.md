## Summary

Keep open: current main contains the merged classifier mitigation for `deactivated_workspace`, but it still stores and reuses one token-derived ChatGPT account id without validating, selecting, or recovering an active workspace for multi-workspace OAuth users.

## What This Changes

OpenAI Codex OAuth should avoid or recover from binding to a deactivated ChatGPT workspace when an active workspace exists.

## Best Possible Solution

Add an OpenAI-plugin-owned workspace validation and recovery policy that uses current Codex/OpenAI auth contracts to select and persist a usable ChatGPT accountId with focused tests and live multi-workspace proof.

## Reproduction Assessment

Yes for source reproduction: current main stores one token-derived account id and reuses it in provider discovery and Codex app-server paths, while users reported the same deactivated-workspace failure after OAuth succeeded.

## Solution Assessment

No; the classifier mitigation is useful but not the best complete fix because active workspace validation and recovery need an explicit OpenAI/Codex auth policy.
