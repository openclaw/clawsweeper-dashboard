## Summary

Keep open: current main still has no OpenAI Codex active-workspace validation or recovery path for a deactivated ChatGPT workspace, and the related open PR only covers failover classification for the `deactivated_workspace` marker. The report is corroborated by a second user who reproduced the same runtime error on v2026.4.8.

## What This Changes

Ensure OpenAI Codex OAuth and runtime use an active ChatGPT workspace for multi-workspace accounts, or recover clearly when the selected workspace is deactivated.

## Best Possible Solution

Keep this open and review #55893 as a partial mitigation, then add an OpenAI-plugin-owned recovery path that persists a verified active `accountId` after OAuth/device-code login or detects `deactivated_workspace` at runtime, retries once with a usable active workspace, persists that selection, and documents manual remediation when recovery is impossible.
