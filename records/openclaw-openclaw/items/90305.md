## Summary

Keep open for maintainer review: the patch is a coherent daemon-service bug fix with source-backed proof and no blocking code findings, but it changes generated supervisor stop behavior so maintainers should accept the upgrade and operator-latency tradeoff before merge.

## What This Changes

The PR changes generated systemd and launchd gateway service stop policy, service-audit drift checks, daemon tests, and Linux/gateway docs to preserve the gateway restart drain before supervisor cleanup.

## Best Possible Solution

Adopt the default 330s/mixed generated service policy plus audit-driven repair, with maintainer-visible upgrade guidance for existing services and a follow-up only if custom drain budgets must be rendered into supervisor files.

## Reproduction Assessment

Yes for a source-backed reproduction path, though I did not run a live systemd restart locally. Current main generates KillMode=control-group and 30s stop timeouts while the gateway restart path can wait 300s for active work and Codex stdio treats stdin EOF as connection close.

## Solution Assessment

Yes, this is the best bounded fix for the default supervised restart mismatch: move the generated supervisor policy to main-process-first drain plus cgroup cleanup, and audit stale installed files. The custom deferralTimeoutMs edge is a valid follow-up rather than a blocker for the default-budget bug fix.
