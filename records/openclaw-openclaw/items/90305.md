## Summary

Needs changes before merge: the drain-preserving supervisor defaults are plausible and evidence-backed, but the latest head still treats unsupported systemd time suffixes as healthy in the service audit parser.

## What This Changes

The PR changes generated systemd and launchd gateway supervisor stop policy to a shared 330s stop budget, switches systemd to KillMode=mixed, and extends service audit/tests/docs for stale stop settings.

## Best Possible Solution

Keep the drain-preserving supervisor defaults, but first align the audit duration parser with systemd parse_sec and then let maintainers accept the 330s rollout risk.

## Reproduction Assessment

Yes: current main has a 300s gateway restart drain while generated supervisors use 30s systemd and 20s launchd stop budgets, and the PR body adds a matching live incident timeline.

## Solution Assessment

No: the supervisor fix direction is the right bounded layer, but the current audit parser still diverges from systemd's duration grammar and can produce false healthy results.
