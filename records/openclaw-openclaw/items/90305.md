The PR changes generated systemd/launchd gateway services to use a shared 330s stop budget with systemd KillMode=mixed, updates daemon service-audit checks, and refreshes matching docs/tests.

## Best Possible Solution

Land the bounded supervisor-policy change if maintainers accept the 330s stop budget and gateway-owned child drain; otherwise request live systemd restart proof or a config-derived budget design before merge.
