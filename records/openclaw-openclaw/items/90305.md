## Summary

Keep open for maintainer merge review: the patch looks like a bounded daemon fix with no blocking code findings, but it intentionally changes supervised gateway stop behavior and needs maintainer acceptance of the longer hard-kill window and main-process-first signal policy before merge.

## What This Changes

The PR changes generated gateway supervisor defaults to systemd `KillMode=mixed` with a 330s stop budget, aligns launchd `ExitTimeOut`, audits stale installed stop policy, and updates Linux docs/tests.

## Best Possible Solution

Land the bounded supervisor-policy change after maintainers accept the upgrade impact, keeping service audit/docs as the path for existing installs to repair stale units.

## Solution Assessment

Yes, this appears to be the best bounded fix: the generated supervisor policy and service audit are the owner boundary, while deriving stop budgets from runtime config would be a larger config-plumbing change that the PR correctly leaves out.
