## Label Justifications

- `P1`: The PR targets a real supervised gateway restart path that can interrupt active agent/channel work for existing users.

## Summary

Keep open for maintainer review: the patch appears to address a real supervised gateway restart-drain mismatch with no blocking code findings, but it changes generated and doctor-repaired supervisor stop policy, so maintainers should explicitly accept the compatibility and availability tradeoff before merge.

## What This Changes

This PR changes managed systemd and launchd gateway service defaults to preserve the gateway restart drain, expands service-audit warnings for stale supervisor stop policy, and updates daemon tests plus Linux gateway docs.

## Best Possible Solution

Land this bounded default-policy fix after maintainer acceptance of the supervisor behavior change, and track dynamic service budgets for custom drain timeouts as a separate follow-up if maintainers want that guarantee.

## Reproduction Assessment

Yes. Current main has a 300s gateway restart-drain path, but generated systemd and launchd services use 30s and 20s stop budgets, and systemd currently sends the initial stop signal to the whole cgroup.

## Solution Assessment

Yes, this is the best bounded fix for default managed services: it aligns supervisor stop policy with the existing gateway drain and preserves final cgroup cleanup. The broader alternative is deriving service budgets from runtime config, which is a larger follow-up for non-default drain settings.
