## Summary

Keep the PR open: the drain-preserving supervisor direction is well supported, but the new service-audit duration parser over-accepts time units that systemd does not accept, so doctor can leave an invalid timeout policy marked healthy.

## What This Changes

The PR changes generated systemd and launchd gateway supervisor stop policy to use a shared 330s stop budget, switches systemd units to KillMode=mixed, and updates service audit tests/docs for stale stop settings.

## Best Possible Solution

Keep the drain-preserving supervisor defaults, but first make the audit parser match systemd's accepted time units and land only after maintainers accept the upgrade and stop-timeout behavior.

## Reproduction Assessment

Source-reproducible: current main has a 300s restart drain while generated supervisors use 30s systemd and 20s launchd stop budgets, and the PR body adds a matching live incident timeline.

## Solution Assessment

Not yet: the supervisor fix direction is plausible and dependency-backed, but the audit parser must be aligned with systemd's real duration grammar before this is the best fix.
