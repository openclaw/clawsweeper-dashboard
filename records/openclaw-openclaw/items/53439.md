## Summary

Keep open. Current main still matches the reported Synology Chat incompatibility, and this issue is paired with open implementation PRs using closing syntax, especially #53441, so it should remain open until a focused fix lands or is explicitly superseded.

## What This Changes

The issue requests that the Synology Chat webhook handler accept HEAD probes and return 200 with a {"success":true} JSON body for accepted POST requests.

## Best Possible Solution

Land #53441, or explicitly supersede it with a narrow Synology Chat plugin change that accepts HEAD probes, returns 200 with {"success":true} for successful POST ACKs, preserves existing token/allowlist/rate-limit behavior, and updates the webhook plus integration tests that currently expect 204.

## Solution Assessment
