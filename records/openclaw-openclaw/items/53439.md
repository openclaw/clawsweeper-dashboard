## Summary

Keep open. Current main and v2026.5.20 still reject Synology Chat HEAD probes and acknowledge accepted POST webhooks with 204/no body, while the open closing PR https://github.com/openclaw/openclaw/pull/53441 already owns the narrow fix path.

## What This Changes

The issue requests Synology Chat webhook handling to return 200 OK for HEAD probes and 200 {"success":true} for accepted POSTs instead of 405/204.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/53441, or an equivalent current-main Synology Chat plugin patch that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves token/rate-limit/in-flight guards, and updates webhook plus integration coverage.

## Reproduction Assessment

Yes, at source level. Current main sends HEAD through the non-POST 405 branch and accepted POSTs through the 204/no-body ACK helper while tests still assert 204.

## Solution Assessment
