## Summary

Keep open. Current main and the latest release still reject Synology Chat HEAD probes and still ACK accepted POST webhooks with 204/no body, while the open fixing PR https://github.com/openclaw/openclaw/pull/53441 specifically owns the narrow implementation path.

## What This Changes

Synology Chat webhook handling should accept HEAD probes and return HTTP 200 with {"success":true} for successful POST acknowledgements.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/53441, or an equivalent current-main Synology Chat plugin patch that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves token/rate-limit/in-flight guards, and updates current regression coverage.

## Reproduction Assessment

Yes, at source level. Current main sends HEAD through the non-POST 405 branch and accepted POSTs through the 204/no-body ACK helper while tests still assert 204.

## Solution Assessment

Yes. The maintainable fix is a narrow Synology Chat plugin change with regression coverage; this issue should stay open until the linked implementation lands or is explicitly superseded.
