Keep open: current main and the latest release still reject Synology Chat HEAD probes and still ACK accepted POSTs with 204/no body; the focused fixing PR is open but unmerged, so this issue should remain open until that or an equivalent narrow fix lands.

## Best Possible Solution

Land one focused Synology Chat plugin fix that accepts HEAD, returns the 200 JSON success ACK for accepted POSTs, preserves token/rate-limit/in-flight safeguards, and updates webhook plus integration tests.
