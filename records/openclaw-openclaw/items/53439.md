## Summary

Keep open. Current main and the latest release still reject Synology Chat HEAD probes and acknowledge accepted POST webhooks with 204/no body, and the issue is paired with an open closing implementation PR that maintainers should review or supersede.

## What This Changes

Synology Chat webhooks should accept HEAD probes and acknowledge accepted POSTs with HTTP 200 plus a JSON success body instead of HTTP 204.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/53441 or an equivalent current-main Synology Chat plugin patch that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves token/rate-limit/in-flight guards, and updates webhook plus integration coverage.

## Reproduction Assessment

Yes, at source level. Current main sends HEAD through the non-POST 405 branch and accepted POSTs through the 204/no-body ACK helper while tests still assert 204.

## Solution Assessment

Yes. The maintainable solution is a narrow Synology Chat plugin fix with regression coverage; this issue should remain open until the linked implementation lands or is explicitly superseded.
