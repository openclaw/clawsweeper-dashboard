## Summary

Keep open: current main and the latest release still reject Synology Chat HEAD probes and still ACK accepted POSTs with 204/no body, while the focused linked implementation PR remains open and unmerged.

## What This Changes

Synology Chat outgoing webhooks should accept HEAD probes and return a 200 JSON success ACK for accepted POSTs instead of 405/204 behavior.

## Best Possible Solution

Land one focused Synology Chat plugin fix through https://github.com/openclaw/openclaw/pull/53441 or an equivalent narrow replacement that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves POST token/rate-limit safeguards, and updates the current tests.

## Reproduction Assessment

Yes at source level. Current main's handler sends HEAD through the non-POST 405 path and accepted POSTs through the 204/no-body ACK helper; I did not run a live Synology NAS repro or local tests because this was read-only.

## Solution Assessment

Yes in direction. The fix belongs inside the Synology Chat plugin webhook handler and its focused tests, with the existing open PR rebased/proven or replaced narrowly.
