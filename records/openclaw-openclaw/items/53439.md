## Summary

Keep open. Current main and the latest release still reject Synology Chat HEAD probes and acknowledge accepted POST webhooks with 204/no body, while the focused closing PR at https://github.com/openclaw/openclaw/pull/53441 remains open and needs maintainer review or supersession.

## What This Changes

Accept Synology Chat HEAD webhook probes and respond to successful POST webhooks with 200 plus {"success":true} instead of 204/no body.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/53441 or an equivalent current-main Synology Chat plugin patch that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves token/rate-limit/in-flight guards, and updates current webhook plus integration coverage.

## Reproduction Assessment

Yes, at source level. Current main sends HEAD through the non-POST 405 branch and accepted POSTs through the 204/no-body ACK helper, while current tests still assert 204.

## Solution Assessment

Yes. The maintainable boundary is a narrow Synology Chat plugin fix with regression coverage; this issue should remain open until the linked implementation lands or is explicitly superseded.
