## Summary

Keep open: current main and the latest release still reject Synology Chat HEAD probes and still acknowledge accepted POSTs with 204/no body, and the issue is already paired with an open focused fixing PR that should be reviewed or superseded before this closes.

## What This Changes

Synology Chat webhook handling should accept HEAD probes and acknowledge successful POSTs with 200 plus a success JSON body.

## Best Possible Solution

Land one focused Synology Chat plugin fix that accepts HEAD probes, returns 200 with {"success":true} for accepted POST ACKs, preserves POST token/rate-limit/allowlist safeguards, and updates the webhook plus integration tests; then close this issue through the merged PR.

## Solution Assessment
