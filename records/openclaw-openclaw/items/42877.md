## Summary

Keep open. Current main still provides read-only memory recall plus a Dreaming-only compaction guard; the requested bounded CRUD write tool, hard per-store limits, duplicate handling, frozen snapshot semantics, and secret policy remain unimplemented and need maintainer product/security direction.

## What This Changes

The issue requests a built-in model-callable memory write surface with add/replace/remove, hard configurable store limits, duplicate handling, frozen snapshot semantics, and optional credential scanning.

## Best Possible Solution

Define and approve a memory-core write contract covering CRUD units, budgets, rejection versus eviction, grants, prompt/docs/config semantics, duplicate handling, snapshot behavior, and secret policy; then land a rebased implementation such as https://github.com/openclaw/openclaw/pull/70540 or a successor.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence current main lacks the requested bounded memory CRUD surface: memory-core registers only `memory_search`/`memory_get`, and docs still describe only those two tools.

## Solution Assessment

Unclear. A built-in write API is plausible, but hard rejection versus eviction/consolidation, public budgets, write grants, and secret handling need maintainer approval before implementation is safe.
