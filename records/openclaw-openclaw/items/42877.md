## Summary

Keep open. Current main and the latest release still expose memory recall/read tooling plus adjacent bootstrap and promotion-size safeguards, but they do not provide the requested model-callable bounded memory CRUD surface with hard per-store limits, capacity feedback, duplicate handling, replace/remove, or secret persistence policy.

## What This Changes

The issue requests a built-in bounded memory write tool with hard character limits, add/replace/remove operations, duplicate handling, session-start snapshots, and optional credential-pattern blocking.

## Best Possible Solution

Define the memory-core write contract first, then implement bounded CRUD with tests for budgets, duplicate handling, prompt visibility, and secret rejection or redaction.
