## Summary

Keep open. Current main still provides memory recall/read tooling plus read/injection caps, but it does not implement the requested bounded memory CRUD/write surface with hard per-store character budgets; the KinthAI comment reinforces the same bloat problem, and draft #70540 is only a partial add-only step.

## What This Changes

Add a built-in memory CRUD tool with per-store hard character limits, usage feedback, substring replace/remove, duplicate detection, frozen session snapshots, and optional credential-pattern screening.

## Best Possible Solution

Define a memory-core-owned write API with config-backed per-store budgets, atomic add/list/replace/remove behavior, duplicate handling, credential-pattern screening, prompt/docs updates, and regression coverage coordinated with #70540.
