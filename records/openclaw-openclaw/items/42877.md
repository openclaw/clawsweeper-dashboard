## Summary

Keep open. Current main and the latest release still expose memory recall plus bounded dreaming promotion, but not the requested hard-limited add/replace/remove memory write tool; the remaining work needs maintainer product and security direction rather than cleanup closure.

## What This Changes

## Best Possible Solution

Define one maintainer-approved memory-plugin write design that covers bounded CRUD, hard limits, duplicate/replace/remove behavior, security scanning, and its relationship to memory flush, dreaming promotion, and related open memory durability work.
