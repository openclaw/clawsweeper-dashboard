## Summary

Keep open. Current main and v2026.6.5 bound prompt injection through truncation but still lack the requested hard-limited memory mutation surface, and the related CRUD proposal and implementation PR closed without landing; maintainer product and security decisions remain necessary.

## What This Changes

Add a built-in bounded memory write capability with configurable character limits, capacity reporting, duplicate detection, and add, replace, and remove operations.

## Best Possible Solution

Define a plugin-owned bounded memory CRUD contract with stable entry identity, separate write authorization, explicit capacity accounting, safe full-store recovery, and a documented sensitive-data policy.

## Reproduction Assessment

Yes at source level: current main has unbounded Markdown memory storage, bounded prompt injection, and recall-only memory tools, so the missing hard-bounded CRUD capability is directly verifiable. This is a feature gap, not a failure of an existing contract.

## Solution Assessment

No, not exactly as a generic core-owned substring mutation tool. The narrower maintainable direction is an active-memory-plugin capability with stable entry identity and a distinct write permission boundary.
