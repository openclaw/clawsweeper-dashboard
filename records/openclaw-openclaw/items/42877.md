## Summary

Keep open. Current main now partially bounds one Dreaming promotion path, but memory-core still exposes only `memory_search` and `memory_get`; the requested bounded CRUD write tool with hard per-store limits, rejection/consolidation semantics, replace/remove, duplicate handling, and credential policy remains unresolved.

## What This Changes

Request a built-in bounded memory write tool with add/replace/remove, hard configurable store budgets, over-budget rejection, duplicate detection, frozen session snapshots, and optional credential-pattern blocking.

## Best Possible Solution

Define a memory-core CRUD contract first, including write authority, per-store budgets, over-budget policy, replace/remove targeting, duplicate handling, secret policy, docs, and tests; then land a rebased `memory_add` slice or successor PR inside that approved design.
