## Summary

Keep open. Current main has memory recall/read tooling plus read and bootstrap injection caps, but it still lacks the requested bounded memory CRUD/write surface with hard per-store character budgets; draft #70540 is only a partial add-only step.

Request a built-in bounded memory CRUD tool that enforces configurable per-store character budgets for memory add, replace, and remove writes.

## Best Possible Solution

Define a memory-core-owned write API with config-backed budgets, atomic add/list/replace/remove semantics, duplicate handling, optional credential screening, prompt/docs updates, and regression coverage coordinated with #70540.
