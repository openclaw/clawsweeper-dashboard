## Summary

Keep open. Current main and v2026.5.12 still expose only recall/read memory tools; the only bounded write behavior is Dreaming promotion compaction, while the requested model-callable bounded CRUD store and security/policy contract remain unimplemented and need maintainer product/security review.

## What This Changes

Request a built-in bounded memory write tool with configurable hard character budgets, add/replace/remove operations, duplicate handling, frozen session snapshots, and credential persistence safeguards.

## Best Possible Solution

Approve a memory-core write contract covering CRUD units, per-store budgets, rejection versus eviction, grants, prompt/docs/config semantics, duplicate handling, snapshot behavior, and secret policy; then land a rebased implementation such as the draft memory_add PR or a successor for the approved slice.
