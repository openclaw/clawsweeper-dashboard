## Summary

Keep this PR open, but do not merge it as-is. The underlying gateway history bug is valid, yet the current patch misses current main's bounded async `chat.history` path and introduces regressions around branch filtering, reset/restore privacy, checkpoint boundary duplication, and uncontained transcript reads.

## What This Changes

The branch adds a main-session checkpoint-chain transcript resolver in `src/gateway/session-utils.fs.ts`, plus regression tests and a production-derived fixture for compacted session history.

## Best Possible Solution

Rework the fix around current main by adding a contained, epoch-aware checkpoint-chain reader that preserves active branch selection, trims snapshots at `firstKeptEntryId`, updates both full and bounded async history paths, and documents the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The PR body gives a concrete compacted-main-session reproduction, current main's code confirms single-file reads, and the PR head includes explicit reset/restore fixtures that demonstrate important failure cases for the proposed approach.

## Solution Assessment

No. Walking checkpoint references is the right direction, but this implementation is not the best solution because it bypasses current async history readers, branch filtering, containment, and reset/restore boundary semantics.
