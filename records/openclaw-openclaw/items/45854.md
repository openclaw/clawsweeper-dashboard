## Summary

Keep open. Current main implements the linked call log, SMS search, and per-device session-isolation pieces, but the remaining device-control, Android ADB-style, and dynamic Kotlin execution capabilities are not present and need explicit product/security review before implementation.

## What This Changes

Request Android node support for device-control, ADB-style, and dynamic program execution tools, plus per-device session isolation for multiple Android nodes on one gateway.

## Best Possible Solution

Split the roadmap into narrower designs or PRs: Android device control with explicit permission UX, ADB/node execution aligned to existing exec approvals, and dynamic Kotlin execution only after a security-reviewed sandbox and audit model exist.

## Reproduction Assessment

Not applicable as a remaining feature roadmap rather than a single reproducible bug. Source inspection does give a high-confidence current-main check for what is implemented and what is still missing.

## Solution Assessment

No, closing or implementing this as one umbrella change is not the best path. The maintainable solution is to split the remaining capabilities and review the ADB/dynamic-execution pieces as security-sensitive designs first.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested ADB and dynamic Kotlin execution surfaces need explicit security design before implementation.

Concerns:

- **[high] Dynamic Android code execution needs a sandbox design:**
  - body: Distributing and executing Kotlin/script code on an Android node would let an agent run new code on the device, so it needs a trust model, permission boundary, audit trail, and rollback story before code is accepted.
  - confidence: 0.9
- **[medium] ADB-style tools need exec-policy alignment:**
  - body: ADB/file/process/log access can expose broad device state and user data, so it should be designed against node command allowlists, pairing scopes, and exec approval semantics rather than added as an ordinary Android command.
  - confidence: 0.86
