## Security Review

Status: needs_attention

Summary: No patch is under review, but the remaining ADB-style and dynamic Kotlin execution requests are security-sensitive and need explicit design before implementation.

Concerns:

- **[high] Design a sandbox before dynamic Android code execution:**
  - body: Distributing and executing Kotlin/script code on an Android node would let an agent run new code on a user device, so it needs a trust model, permission boundary, audit trail, and rollback story before implementation.
  - confidence: 0.9
- **[medium] Align ADB-style tools with exec approval policy:** `src/gateway/node-command-policy.ts:81`
  - body: ADB-style file, process, app, and log access can expose broad user data and device state, so it should be designed around node command allowlists, pairing scopes, and approval semantics rather than added as an ordinary Android command.
  - confidence: 0.86
