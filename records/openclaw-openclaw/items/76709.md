## Summary

Keep open. The PR is a useful partial fix for Matrix shared-client shutdown persistence, but it is not merge-ready as written because it marks the broader linked crypto-store issue fixed while the event-driven IDB flush half remains absent on current main.

## What This Changes

The PR adds async Matrix shared-client stop helpers that await `client.stopAndPersist()`, deprecates the synchronous helpers, re-exports two helpers from the Matrix client barrel, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Either narrow this PR to the shared-client shutdown half by removing closing/fixed wording for the broader issue, or expand it to include debounced crypto-event IDB persistence before claiming the full issue is fixed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: the shared stop helpers return after `client.stop()` while `stopAndPersist()` is the only awaitable final-persist path, and crypto snapshots are still persisted only after init, on a 60 second timer, and on shutdown.

## Solution Assessment

No as written. The new async helpers are a good narrow fix for the stop-helper defect, but the PR should not close or changelog the broader crypto-store issue as fixed while the event-driven flush defect remains.
