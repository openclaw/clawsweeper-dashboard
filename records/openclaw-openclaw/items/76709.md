## Summary

Keep open. The async stop-helper work is a valid narrow fix, but this PR still uses closing/fixed wording for #76611, which also tracks the missing event-driven Matrix crypto-store flush that remains unresolved on current main.

## What This Changes

This PR adds async Matrix shared-client stop helpers that await `stopAndPersist()`, deprecates the synchronous helpers, exports two async helpers, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Land the async stop-helper patch with non-closing references to #76611, unless the same PR also implements the remaining debounced crypto-event persistence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level defect: current main has synchronous stop helpers that return after `client.stop()`, while the only awaitable final-persist path is `stopAndPersist()`.

## Solution Assessment

Is this the best way to solve the issue? No as written. The async helpers are a good narrow fix for the shared-client shutdown defect, but the PR should not claim the broader crypto-store issue is fixed while event-driven persistence is still missing.
