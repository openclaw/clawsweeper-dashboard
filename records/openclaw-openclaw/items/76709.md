## Summary

Keep open: current main still has only the synchronous Matrix shared-client stop helpers, while this PR is a narrow source-backed fix with no blocking review findings.

## What This Changes

Adds async Matrix shared-client stop helpers that await `client.stopAndPersist()`, deprecates the synchronous helpers, exports the account and instance variants, adds focused tests, and updates the changelog.

## Best Possible Solution

Land the additive async helper API after targeted Matrix tests and changed-gate proof, while leaving event-driven crypto-store flushing tracked in #76611.

## Reproduction Assessment

Yes, from source inspection: current main's shared stop helpers call `client.stop()` and return before `stopPersistPromise` can be awaited, while `stopAndPersist()` is the existing awaitable path. I did not run a live homeserver restart reproduction.

## Solution Assessment

Yes for this narrowed scope: additive async helpers preserve the existing synchronous API while giving shutdown callers an awaitable persist path. The PR now avoids closing the broader event-driven flush work in #76611.
