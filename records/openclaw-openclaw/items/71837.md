Keep this PR open: current main still lacks the PR’s central REM-to-grounded promotion behavior, but the branch still has source-reproducible correctness issues around REM identity, signal weighting, and meta-tag boosts before it is mergeable.

## What This Changes

The PR changes memory-core REM denoising, short-term promotion scoring, concept vocabulary filters, and memory-host session extraction tests/logic to reduce dreaming scaffolding noise and let selected REM truths reinforce deep promotion.

## Best Possible Solution

Land a rebased version that preserves the REM denoise and REM-to-grounded promotion goal while using stable REM identity, confidence-scaled grounded signal strength, current session sanitization structure, aligned meta vocabulary, and targeted regression coverage.

## Reproduction Assessment

## Solution Assessment

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes candidates with a day-suffixed query key, and grounded recording hashes that query into queryHashes. If the same selected truth appears on multiple days, it can gain distinct synthetic query hashes and satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on dayBucket for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Each selected REM truth records signalCount: 3 even though the per-entry confidence is already computed. That gives a barely passing REM truth the same grounded promotion weight as a high-confidence truth, so derive the count from confidence or lower the floor for borderline candidates.
  - confidence: 0.87
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: Adding system to REM_POSITIVE_TAGS conflicts with current main’s treatment of system as conversation/runtime meta vocabulary in REM reflection tags and concept stop words. This undermines the denoise goal by giving meta language a positive theme boost.
  - confidence: 0.84
