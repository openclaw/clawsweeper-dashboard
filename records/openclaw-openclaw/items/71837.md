## Summary

Keep open: current main still lacks the PR's central REM-candidate-to-grounded-promotion behavior, but the proposed patch is not merge-ready because it can inflate diversity, gives all REM truths the same grounded weight, and reintroduces `system` as a positive REM signal.

## What This Changes

The PR changes memory-core dreaming and memory-host session extraction to strip scaffolding, denoise REM themes/truths, and feed selected REM truths back into grounded short-term promotion candidates.

## Best Possible Solution

Land a rebased version that preserves the REM denoise and REM-to-grounded promotion goal while using stable REM identity, confidence-scaled grounded signal strength, main's current session sanitization structure, aligned meta vocabulary, a changelog entry, and targeted tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes candidates with a day-suffixed query key, and grounded recording hashes that query into `queryHashes`. If the same truth is selected on multiple days, it can gain distinct synthetic query hashes and satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on `dayBucket` for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Each selected REM truth records `signalCount: 3` even though the per-entry confidence is already computed. That gives a barely passing REM truth the same grounded promotion weight as a high-confidence truth, so derive the count from confidence or lower the floor for borderline candidates.
  - confidence: 0.87
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds `system` to `REM_POSITIVE_TAGS`, but current main treats `system` as conversation/runtime meta vocabulary in both REM reflection tags and concept stop words. This works against the denoise goal by giving meta language a positive theme boost.
  - confidence: 0.84
