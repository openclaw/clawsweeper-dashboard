Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes a day-suffixed query key, and grounded candidate recording hashes that query into `queryHashes`. If the same candidate is selected on multiple days, it gains distinct synthetic query hashes and can satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on `dayBucket` for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Each selected REM truth records `signalCount: 3` even though `score` is computed per entry. That gives a barely passing REM truth the same grounded promotion weight as a high-confidence one; derive the count from `calculateCandidateTruthConfidence(entry)` or lower the floor for borderline candidates.
  - confidence: 0.86
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds `system` to `REM_POSITIVE_TAGS` but not to the REM meta set, while current main filters `system` from REM reflection tags and concept vocabulary. This lets transcript/runtime meta language receive a positive REM theme boost, working against the denoise goal.
  - confidence: 0.84
