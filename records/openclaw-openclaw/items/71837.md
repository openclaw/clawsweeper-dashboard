## Summary

Keep open: the PR is still a useful memory-core bug-fix direction, but the current branch has source-reproducible correctness issues in REM grounding identity, REM signal weighting, and meta-vocabulary handling before it should merge.

## What This Changes

The PR denoises memory-core dreaming/session corpus extraction, filters REM meta language, and feeds selected REM truths into grounded short-term promotion.

## Best Possible Solution

Land a rebased version that keeps the REM denoise and REM-to-grounded promotion goal while using stable REM identity, confidence-scaled grounded signal strength, current session sanitization structure, aligned meta vocabulary, and targeted regression coverage.

## Reproduction Assessment

Yes. The review issues are source-reproducible from the PR head and current main: the PR uses a day-suffixed REM query and fixed signalCount, and current main hashes that query and applies that count to promotion state.

## Solution Assessment

No. The direction is useful, but this branch is not the best merge shape until it is rebased onto current main and the REM identity, weighting, and meta vocabulary issues are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes candidates with a day-suffixed query key, and recordGroundedShortTermCandidates hashes that query into queryHashes. If the same selected truth appears on multiple days, it can gain distinct synthetic query hashes and satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on dayBucket for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Each selected REM truth records signalCount: 3 even though the per-entry confidence is already computed. That gives a barely passing REM truth the same grounded promotion weight as a high-confidence truth, so derive the count from confidence or lower the floor for borderline candidates.
  - confidence: 0.87
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: Adding system to REM_POSITIVE_TAGS conflicts with current main's treatment of system as conversation/runtime meta vocabulary in REM reflection tags and concept stop words. This undermines the denoise goal by giving meta language a positive theme boost.
  - confidence: 0.84
