The PR changes memory-core dreaming, short-term promotion, concept vocabulary, session transcript extraction, and focused tests to strip wrapper noise, denoise REM output, and feed selected REM truths into grounded promotion candidates.

Land a rebased version that preserves the denoise and REM-to-grounded promotion direction while using stable synthetic REM identity, confidence-scaled grounded strength, meta-safe REM tag biasing, the current package-owned session extraction structure, a changelog entry, and focused regression coverage.

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes candidates with a day-suffixed query key, and grounded candidate recording hashes that query into `queryHashes`. If the same candidate is selected on multiple days, it gains distinct synthetic query hashes and can satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on `dayBucket` for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Every selected REM truth records `signalCount: 3` even though the per-entry confidence is already computed. That gives a borderline REM truth the same grounded promotion weight as a high-confidence one; derive the count from candidate confidence or lower the floor for weak candidates.
  - confidence: 0.87
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds `system` to `REM_POSITIVE_TAGS`, but current main filters `system` from REM reflection tags and concept vocabulary. This lets transcript/runtime meta language receive a positive REM theme boost, working against the denoise goal.
  - confidence: 0.85
- **[P2] Keep the source shim intact:** `src/memory-host-sdk/host/session-files.ts:1`
  - body: Current main makes `src/memory-host-sdk/host/session-files.ts` a re-export of the package implementation. The branch restores a full copy in `src`, which would reintroduce split session extraction logic on a main rebase; move the scaffold stripping into the package implementation and leave this shim as a re-export.
  - confidence: 0.82
