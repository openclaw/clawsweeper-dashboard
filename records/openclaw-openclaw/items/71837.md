## Summary

Keep open. Current main overlaps with the denoise/backfill direction, but it does not fully implement this PR's live REM-to-grounded promotion path, and the PR still has concrete correctness blockers around synthetic query identity, REM signal weighting, and treating `system` as a positive REM signal.

## What This Changes

The PR changes memory-core REM/session ingestion and short-term promotion so session scaffolding is stripped, REM tags/snippets are filtered and biased, selected REM truths are recorded as grounded short-term candidates, ranking gates are adjusted, and focused tests are added.

## Best Possible Solution

Land a rebased version that keeps the live REM-to-grounded promotion goal, but adapts it to current main's memory-host package layout and backfill helpers, uses stable synthetic REM identity, scales grounded strength from candidate confidence, keeps meta vocabulary out of positive REM boosts, and includes focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. The review issues reproduce from the PR diff: the same REM truth selected on different day buckets gets different synthetic query hashes, all selected REM truths get `signalCount: 3`, and `system` is positively boosted despite current-main meta filtering.

## Solution Assessment

No, not in the current PR shape. The narrow maintainable solution is to rebase onto current main and reuse the existing grounded/backfill conventions while fixing identity, weighting, and meta-tag filtering before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1862`
  - body: The REM grounding path writes a day-suffixed query key and `recordGroundedShortTermCandidates` hashes that query into `queryHashes`. If the same candidate is selected across multiple days, it gains distinct synthetic query hashes and can satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on `dayBucket` for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1854-1855`
  - body: Every selected REM truth records `signalCount: 3` even though the PR computes a per-entry confidence score. This gives a barely passing REM truth the same grounded promotion weight as a high-confidence one; derive the count from `calculateCandidateTruthConfidence(entry)` or use a lower floor for borderline candidates.
  - confidence: 0.86
- **[P2] Keep system out of REM boosts:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds `system` to `REM_POSITIVE_TAGS`, while current main treats `system` as meta vocabulary in REM reflection tags and concept tags. That lets transcript/runtime scaffolding receive a positive REM theme boost, which works against the denoise goal.
  - confidence: 0.84
