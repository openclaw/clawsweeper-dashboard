Full review comments:

- **[P2] Keep phase hits out of recall/diversity gates:** `extensions/memory-core/src/short-term-promotion.ts:426-429`
  - body: This makes light/rem phase hits count toward both signal count and context diversity, so one daily recall can satisfy the deep gates after light+REM review. That turns phase review into synthetic recall/query evidence and can promote weak entries; keep phase hits as scoring boosts or require an explicit maintainer-approved gate policy.
  - confidence: 0.86
- **[P2] Scale REM grounded signal strength by confidence:** `extensions/memory-core/src/dreaming-phases.ts:1855`
  - body: Every selected REM truth writes `signalCount: 3`, and the grounded writer adds that count into `groundedCount` and `totalScore`. A barely eligible truth gets the same promotion weight as high-confidence evidence, so derive or clamp this count from REM confidence instead of using a fixed value.
  - confidence: 0.82
- **[P2] Keep REM grounding query IDs stable:** `extensions/memory-core/src/dreaming-phases.ts:1856`
  - body: The day-suffixed query is hashed into `queryHashes`, so the same REM truth selected on different days can accumulate synthetic query diversity. Use a stable REM grounding query identity and let `dayBucket` handle per-day dedupe.
  - confidence: 0.87
