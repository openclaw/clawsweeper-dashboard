The issue requests a public reusable `@openclaw/gateway-client` package that shares Gateway WebSocket auth, request/response, events, and reconnect logic across CLI, Control UI, and external apps.

## Best Possible Solution

Define a maintainer-approved public Gateway client SDK contract first, including package naming, Node/browser scope, auth and device-storage adapters, versioning, docs, release process, and security review, then publish or deliberately defer it.
