## Summary

Keep open. Current main and v2026.5.22 include useful `@openclaw/sdk` work, but they still do not provide the requested public, platform-agnostic low-level Gateway WebSocket client: the package is private and Node-built, the SDK transport wraps the internal Node client, and Control UI still has its own browser client.

## What This Changes

The issue requests a reusable `@openclaw/gateway-client`-style package with shared WebSocket protocol, auth, request/response, event, reconnect, and adapter surfaces for external clients and future CLI/Control UI reuse.

## Best Possible Solution

Define and ship a supported App SDK/Gateway client contract with generated low-level protocol coverage, adapterized WebSocket and device-identity storage, public docs, parity tests, and a staged CLI/Control UI migration plan.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken existing behavior report. Source, release-tag, docs, and issue-discussion checks give high confidence that current main only partially implements the requested client contract.
