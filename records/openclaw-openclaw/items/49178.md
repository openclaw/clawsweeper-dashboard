## Summary

Keep this open: current main and v2026.6.6 include an internal `@openclaw/gateway-client` extraction, but the requested reusable public SDK is still missing because the package remains private/unpublished, the external-app docs explicitly say there is no public npm client package yet, and Control UI still carries a separate browser WebSocket client. ## What This Changes

The issue requests a reusable public gateway WebSocket client SDK package for third-party OpenClaw gateway consumers.

## Best Possible Solution
