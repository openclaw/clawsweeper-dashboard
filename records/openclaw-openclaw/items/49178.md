## Summary

Keep open: current main and `v2026.6.8` contain useful internal gateway-client/SDK groundwork, but the package is still private, Node-oriented, unpublished, and the Control UI still uses its own browser WebSocket client. The public SDK request remains a maintainer API/security product decision, with positive external demand and a third-party reference implementation already noted in the discussion.

## What This Changes

Request to publish a reusable platform-agnostic `@openclaw/gateway-client` SDK for third-party clients and eventually share protocol client code between the CLI and Control UI.

## Best Possible Solution

A maintainer should either sponsor a versioned, documented, security-reviewed public gateway SDK with clear Node/browser/mobile adapter and storage boundaries, or explicitly keep the packages private and direct external apps to the documented Gateway protocol/RPC surfaces.
