## Summary

Keep open. The protected `maintainer` label prevents cleanup closure, and current main only partially addresses the request: OpenRouter can resolve unknown text models at runtime from cached capabilities, but the normal text catalog/list/picker path is still not a live OpenRouter model catalog and the allowlist/cost-control policy remains unresolved.

## What This Changes

The issue requests fully dynamic runtime model discovery for fast-moving providers, especially OpenRouter, while keeping model picker UX and allowlist/cost controls usable.

## Best Possible Solution

Keep this as the canonical maintainer-tracked feature and implement provider-owned live text catalog discovery for OpenRouter through the existing catalog seams, with bounded picker defaults, cache/refresh behavior, allowlist wildcard semantics, docs, and focused Gateway/CLI/UI coverage.

## Reproduction Assessment

Yes, at source level. Current main resolves unknown OpenRouter text models at runtime, but normal `models.list` and picker paths still use read-only/static/configured catalog rows while OpenRouter only registers live catalog discovery for video generation.

## Solution Assessment

Unclear as a fully specified implementation. Provider-owned live text catalog discovery through the existing catalog seams looks maintainable, but picker defaults, cache/refresh policy, and allowlist/cost-control behavior need maintainer product/API direction first.
