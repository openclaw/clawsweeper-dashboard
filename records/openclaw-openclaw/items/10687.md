## What This Changes

The issue asks OpenClaw to discover fast-moving provider model catalogs dynamically, especially OpenRouter, while preserving picker usability and allowlist/cost controls.

## Best Possible Solution

Keep this as the canonical maintainer-tracked feature and implement provider-owned live text catalog discovery for OpenRouter through the existing catalog seams, with bounded picker defaults, cache/refresh behavior, allowlist wildcard semantics, docs, and focused Gateway/CLI/UI coverage.

## Reproduction Assessment

Yes, at source level. Current main resolves unknown OpenRouter text models at runtime from cached capabilities, but normal `models.list` and picker paths still use static/configured catalog rows while OpenRouter only registers live catalog discovery for video generation.

## Solution Assessment

Unclear as a fully specified implementation. Provider-owned live text catalog discovery through existing catalog seams looks maintainable, but picker defaults, cache/refresh policy, and allowlist/cost-control behavior need maintainer direction first.
