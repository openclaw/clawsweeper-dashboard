## Summary

Keep open. ## What This Changes

Request dynamic provider-backed model discovery so OpenRouter and other fast-moving catalogs can list and select new models without waiting for OpenClaw upgrades while preserving model allowlist controls.

## Best Possible Solution

Keep this as the canonical maintainer-tracked model discovery feature and implement provider-owned live text catalog discovery through existing catalog seams, with bounded picker defaults, cache/refresh behavior, allowlist semantics, docs, and focused Gateway/CLI/UI coverage.

## Reproduction Assessment

Yes, at source level. Current main can synthesize unknown OpenRouter runtime models from cached capabilities, but the normal OpenRouter text catalog remains bounded and live catalog registration is video-only.

## Solution Assessment

Unclear as a fully specified implementation. Provider-owned live text catalog discovery through existing seams looks maintainable, but picker size, cache policy, and allowlist/cost-control semantics need maintainer direction first.
