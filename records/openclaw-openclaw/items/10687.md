## Summary

Keep open: current main partially mitigates selected OpenRouter unknown-model execution with dynamic capability lookup, but the canonical request for provider-backed text catalog/listing/picker behavior and allowlist/cache policy remains open and carries maintainer/product-decision labels.

## What This Changes

Request fully dynamic provider model discovery so OpenRouter and other fast-moving catalogs can list and select current models without waiting for static catalog updates or unsafe allowlist workarounds.

## Best Possible Solution

Keep this as the canonical maintainer/product thread, then implement provider-owned live text catalogs through the existing plugin SDK/catalog seams with explicit picker, cache, offline, and allowlist semantics.

## Reproduction Assessment

Yes by source inspection: current main can dynamically resolve selected unknown OpenRouter text models at runtime, but OpenRouter does not yet register a full live text catalog for /model list or picker surfaces; no live OpenRouter key test was run in this read-only review.

## Solution Assessment

Partially: the existing dynamic fallback is a good narrow execution mitigation, but the complete solution still needs a provider-catalog design for listing, picker scale, cache/offline policy, and allowlist safety.
