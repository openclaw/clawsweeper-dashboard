## What This Changes

The issue requests provider-backed dynamic model catalog discovery for OpenRouter and similar fast-moving providers, including list/picker visibility and allowlist semantics.

## Best Possible Solution

Keep this as the canonical maintainer design item for provider-owned live text catalog discovery through the plugin catalog/augmentation seams, with explicit picker defaults, cache/offline policy, and allowlist semantics.

## Reproduction Assessment

Yes, at source level. Current main has OpenRouter runtime unknown-model synthesis, but the text list/picker catalog remains curated and only the video-generation catalog is registered as a live OpenRouter model catalog.

## Solution Assessment

Unclear as a final implementation. Provider-owned live text catalog discovery through the existing plugin catalog seam looks maintainable, but default UX, cache freshness, offline behavior, and allowlist/cost controls need maintainer direction first.
