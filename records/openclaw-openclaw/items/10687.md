## Summary

Keep open: current main has partial OpenRouter runtime dynamic-model mitigation and a guarded live-catalog SDK, but the requested provider-backed catalog/list/picker/allowlist behavior is still not implemented for OpenRouter and remains a maintainer product/security-policy decision.

## What This Changes

Requests provider-backed dynamic model discovery, starting with OpenRouter, so model lists, pickers, allowlists, and runtime selection can use live catalogs instead of static bundled rows.

## Best Possible Solution

Keep #10687 as the canonical design thread and implement OpenRouter plus broader provider-backed discovery through the existing guarded plugin catalog SDK once maintainers settle picker defaults, wildcard allowlists, cache/offline behavior, and auth/SSRF safety.
