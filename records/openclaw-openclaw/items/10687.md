## Summary

Keep open. Current main and latest release partially handle explicit OpenRouter model execution through provider-owned dynamic runtime hooks, but the requested live catalog/listing, wildcard allowlist, picker-size, cache/offline, and provider API policy work remains unresolved and the issue still carries protected maintainer/product-decision signals.

## What This Changes

Request fully dynamic provider model discovery so OpenRouter and other fast-moving catalogs can list and select current models without waiting for static catalog updates.

## Best Possible Solution

Extend the existing provider-owned live catalog/plugin SDK path so OpenRouter can expose bounded live text-model rows while preserving `provider/*` allowlist, cache/offline, and picker-size policy.

## Reproduction Assessment

Yes from source: OpenRouter returns 336 live model rows while OpenClaw's OpenRouter browse catalog still contains only three rows; explicit runtime resolution is partly fixed, so the remaining repro is list/discovery behavior.

## Solution Assessment

No for the issue's proposed parallel core catalog source as an autonomous fix; best path is extending provider-owned live catalog/SDK surfaces with maintainer-approved picker, cache, offline, and allowlist semantics.
