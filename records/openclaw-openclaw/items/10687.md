## Summary

Keep open. Current main has provider wildcard visibility, self-hosted discovery, and OpenRouter dynamic model execution pieces, but the central OpenRouter/generic live catalog discovery and allowlist/cache/product decisions remain unresolved; the protected maintainer label also requires human handling.

## What This Changes

Request fully dynamic provider model discovery so OpenRouter and other fast-moving catalogs can list and select current models without static catalog updates while preserving allowlist safety.

## Best Possible Solution

Use the existing provider-owned live catalog hooks for OpenRouter and generic OpenAI-compatible discovery, then define explicit allowlist/wildcard and cache/offline semantics before changing runtime defaults.

## Reproduction Assessment

Yes from source inspection: OpenRouter's catalog is still static and generic custom providers require explicit models, while dynamic hooks cover explicit runtime resolution rather than full catalog/picker discovery.

## Solution Assessment

Unclear until maintainers choose the product shape; provider-owned live catalog hooks are the right seam, but broad fallback behavior or a new config mode needs maintainer direction.
