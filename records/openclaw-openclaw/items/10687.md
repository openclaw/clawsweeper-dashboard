## What This Changes

Request fully dynamic provider model discovery, starting with OpenRouter, so list, picker, and validation paths can use current provider catalogs without stale static Pi registry or manual allowlist refresh.

## Best Possible Solution

Keep this as the canonical maintainer-tracked design for provider-owned live text catalog discovery, using existing plugin catalog and visibility seams after cache, picker, and allowlist defaults are decided.
