## Summary

Keep open. Current main partially addresses OpenRouter unknown-model execution through runtime synthesis and cached capabilities, but the provider catalog exposed to model list and picker surfaces is still curated/static for text models and the allowlist/default UX remains a maintainer product decision.

## What This Changes

Request fully dynamic model discovery for OpenRouter and similar fast-moving providers while preserving model allowlist and cost-control behavior.

## Best Possible Solution

Define a provider-owned live text catalog contract for OpenRouter-style providers, then wire it through model list, picker/default views, and allowlist/wildcard semantics without bypassing existing safety controls.

## Reproduction Assessment

Yes, source-level: current main has OpenRouter dynamic runtime resolution, but the text catalog exposed to list/picker surfaces remains curated and the live catalog registration is video-only; I did not run a live OpenRouter key test.

## Solution Assessment

Unclear until maintainers choose the product shape; provider-owned catalog hooks and existing wildcard semantics look like the right boundary, while a broad core allow-any fallback would weaken cost and safety controls.
