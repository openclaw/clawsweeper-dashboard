## Summary

Keep open. The issue has the protected `maintainer` label, and current `main` only partially addresses the request: OpenRouter can synthesize unknown text models at runtime from cached capabilities, but normal model list/picker discovery still lacks a live OpenRouter text catalog and the allowlist/cost-control UX needs maintainer direction.

## What This Changes

The issue asks for provider-backed dynamic model discovery so OpenRouter and similar providers can list and select current models without waiting for static catalog updates while preserving allowlist and cost controls.

## Best Possible Solution

Keep this as the canonical maintainer-tracked feature and implement provider-owned live text catalog discovery for OpenRouter through the existing catalog seams, with bounded picker defaults, cache/TTL behavior, allowlist wildcard semantics, docs, and focused Gateway/CLI/UI coverage.

## Reproduction Assessment

Yes. From source inspection, normal `models.list` and picker paths use read-only/static/configured catalog rows while OpenRouter only registers live catalog discovery for video generation, so current OpenRouter text models will not reliably appear in normal listing without being static or configured.

## Solution Assessment

Unclear as a fully specified implementation. Provider-owned live text catalog discovery through the existing catalog seams looks maintainable, but picker defaults, cache TTL, and allowlist/cost-control behavior need maintainer product/API decisions first.
