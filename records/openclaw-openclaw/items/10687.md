## Summary

Keep open. Current main and v2026.6.5 still expose only a curated OpenRouter text catalog to list and picker surfaces while runtime has partial unknown-model synthesis; the protected `maintainer` label and remaining cache, picker, and allowlist choices make this maintainer-owned product work.

## What This Changes

Request fully dynamic provider model discovery so OpenRouter and similar fast-moving catalogs can list and select new models at runtime while preserving allowlist and cost controls.

## Best Possible Solution

Keep this as the canonical maintainer design item for provider-owned live text catalog discovery, implemented through the plugin model-catalog seam after picker defaults, cache freshness, offline behavior, and allowlist policy are decided.

## Reproduction Assessment

Yes, at source level. Current code has OpenRouter runtime model synthesis and provider wildcard allowlist support, but OpenRouter text listing still depends on curated/static rows and the only live catalog registration is video-scoped.

## Solution Assessment

Unclear as a fully specified implementation. Provider-owned live text catalog discovery through existing plugin seams looks maintainable, but picker size, cache policy, offline behavior, and allowlist/cost controls need maintainer direction first.
