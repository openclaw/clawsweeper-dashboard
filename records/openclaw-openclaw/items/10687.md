## Label Justifications

- `impact:auth-provider`: The requested behavior changes provider model routing, model choice, and configured-model validation semantics.

## Summary

Keep open: current main has partial OpenRouter dynamic model resolution and guarded live-catalog infrastructure, but OpenRouter's catalog/list/picker surface still does not implement the requested provider-backed dynamic discovery behavior, and the remaining cache, allowlist, credential, and UX policy questions are maintainer/product decisions.

## What This Changes

The issue requests fully dynamic provider model discovery, starting with OpenRouter, so model lists, pickers, and configured model validation can track upstream catalogs without static OpenClaw updates.

## Best Possible Solution

Keep this issue as the canonical maintainer design thread and implement provider-owned live catalog discovery through the guarded SDK once picker defaults, caching, fallback, allowlist, and credential-safety policy are settled.

## Reproduction Assessment

Yes at source level: current main still exposes a static three-row OpenRouter catalog while OpenRouter's public model endpoint returns hundreds of models, but no live authenticated OpenClaw picker or agent run was performed.

## Solution Assessment

No current fix is complete: the runtime dynamic-model hooks are useful partial mitigation, but the best fix is a maintainer-approved provider-catalog design using the existing guarded live catalog SDK rather than ad hoc fallback behavior.
