## Summary

Keep open: current main and v2026.6.8 partially mitigate OpenRouter unknown-model execution with provider dynamic model resolution, but this issue still owns the broader provider-backed catalog, picker/list, cache, allowlist, and safety policy decision, and it has a protected maintainer label.

## What This Changes

Request fully dynamic model discovery for OpenRouter and other fast-moving providers so selection, listing, and runtime validation do not depend on stale static catalogs.

## Best Possible Solution

Settle the provider-owned live catalog contract, then implement OpenRouter and generic provider discovery through the existing guarded plugin/catalog SDK with safe caching, browse defaults, and allowlist wildcard behavior.

## Reproduction Assessment

Yes, source-level: current main still exposes only a small static OpenRouter text catalog while runtime dynamic resolution can synthesize explicit OpenRouter models. I did not run a full OpenClaw live model-selection flow.

## Solution Assessment

No, current main is only a partial solution: runtime OpenRouter execution is improved, but fully dynamic catalog/listing/picker and allowlist policy remains a maintainer design decision.
