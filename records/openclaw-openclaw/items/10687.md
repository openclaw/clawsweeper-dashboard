## Summary

Keep open. Current main and v2026.6.6 partially handle explicit OpenRouter model execution through provider-owned dynamic hooks, but full dynamic browse/list discovery remains unresolved and the protected `maintainer` label requires explicit maintainer product/API handling.

## What This Changes

Add fully dynamic model discovery for OpenRouter and other fast-moving provider catalogs so model listing, selection, and validation can use current provider models without manual static catalog refreshes.

## Best Possible Solution

Extend the existing provider-owned live catalog/plugin SDK path so OpenRouter and similar providers can expose bounded live rows after maintainers settle wildcard allowlist, cache, offline, and picker-default semantics.

## Reproduction Assessment

Yes. Source and live endpoint checks show OpenRouter has hundreds of current model rows while the OpenClaw browse catalog still exposes only a small static set; current main partially resolves explicit IDs through dynamic hooks, so the remaining repro is full list/discovery behavior.

## Solution Assessment

No for the issue's proposed parallel core catalog source as an autonomous fix. The better solution is to extend the existing provider/plugin live catalog contract with maintainer-approved picker, cache, offline, and allowlist policy.
