## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so the reported final-line multi-line output remains source-reproducible, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that `isAnnounceSkip` suppress multi-line sub-agent output when a standalone final line is `ANNOUNCE_SKIP`, not only when the whole trimmed output equals the token.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage for final-line `ANNOUNCE_SKIP`, and docs/changelog alignment.

## Reproduction Assessment

Yes. The source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`, which follows the current helper to false because the whole trimmed string is not exactly the token.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line, preserve `REPLY_SKIP` and `NO_REPLY` semantics, and cover that contract with tests and docs.
