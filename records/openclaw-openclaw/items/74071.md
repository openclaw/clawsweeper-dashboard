## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so the reported final-line token case remains source-reproducible; the open closing PR is the active implementation path and should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to treat multi-line sub-agent output whose final standalone line is `ANNOUNCE_SKIP` as silent announce output instead of delivering the preceding summary.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/74136 with the final-line `ANNOUNCE_SKIP` helper contract, focused regression coverage, and docs alignment.

## Reproduction Assessment

Yes. The high-confidence source path is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which remains false while the exact token case is true.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line, preserve strict `REPLY_SKIP` and `NO_REPLY` semantics, and cover the helper plus announce flow with tests and docs.
