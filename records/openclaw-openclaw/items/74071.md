## Summary

Keep open. Current `main` still uses strict trimmed equality for `ANNOUNCE_SKIP`, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that sub-agent announce suppression also recognize `ANNOUNCE_SKIP` when it appears on its own final line after other output.

## Best Possible Solution

The durable end state is a tested helper contract that treats a standalone final-line `ANNOUNCE_SKIP` as silent while preserving existing `REPLY_SKIP` and `NO_REPLY` semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main: the helper compares the whole trimmed string to the token and returns false, and both announce delivery paths rely on that helper.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line, add focused helper and announce-flow regression coverage, and align docs/changelog if maintainers accept that as supported behavior.
