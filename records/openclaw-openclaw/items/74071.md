## Summary

Keep open. Current main still treats `ANNOUNCE_SKIP` as an exact trimmed-token match, so a multi-line sub-agent reply ending with a standalone token still falls through to visible announcement delivery; the issue is also paired with open PR #74136 using closing syntax.

## What This Changes

The issue requests suppressing sub-agent announcements when the output contains a multi-line summary followed by `ANNOUNCE_SKIP` on its own final line.

## Best Possible Solution

Land PR #74136 or an equivalent narrow patch that preserves exact-token behavior, accepts a standalone final-line `ANNOUNCE_SKIP`, adds focused helper and announce-flow regression coverage, and aligns prompt/docs wording with the intended contract.

## Reproduction Assessment

Yes. A source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`, which returns false on current main because the helper compares the whole trimmed string to the token.

## Solution Assessment

Yes. The narrow maintainable fix is to accept only an exact token or a standalone final-line token while leaving `REPLY_SKIP` and `NO_REPLY` semantics unchanged unless separately reviewed.
