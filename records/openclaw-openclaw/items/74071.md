## Summary

Keep open. Current main and the latest tagged release still use strict whole-string equality for `ANNOUNCE_SKIP`, so the reported final-line token case remains source-reproducible; the open closing PR is the active fix path and should be reviewed, landed, or superseded before closing this issue.

## What This Changes

The issue asks for `isAnnounceSkip` to suppress multi-line sub-agent output when a standalone final line is `ANNOUNCE_SKIP`.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/74136 with the final-line `ANNOUNCE_SKIP` contract, focused regression coverage, and docs alignment.

## Reproduction Assessment

Yes. The high-confidence source path is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which evaluates false while the exact padded token evaluates true.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line, preserve strict `REPLY_SKIP` and `NO_REPLY` semantics, and cover the helper plus announce flow with tests and docs.
