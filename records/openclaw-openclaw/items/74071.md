## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so a multi-line reply ending with the token remains a source-reproducible false negative; open PR #74136 is the active implementation candidate and should be reviewed or superseded before this issue closes.

## What This Changes

The issue requests that `ANNOUNCE_SKIP` suppress sub-agent announcements when it appears as a standalone final line after other output.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage, and docs alignment for final-line `ANNOUNCE_SKIP`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `isAnnounceSkip("summary\nANNOUNCE_SKIP")` trims the full string and compares it to the token, returning false on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line while preserving adjacent `REPLY_SKIP` and `NO_REPLY` semantics.
