## Summary

Keep open. Current main still uses exact trimmed-token equality for `ANNOUNCE_SKIP`, the latest release tag shows the same behavior, and open PR #74136 is already the linked implementation candidate for this issue, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

Support treating a standalone final-line `ANNOUNCE_SKIP` in multi-line sub-agent output as an announce-suppression token instead of delivering the preceding summary text.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `isAnnounceSkip` preserves exact-token behavior while also accepting a standalone final token line, with focused helper coverage and announce-flow regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main; the helper compares the full trimmed string to the token and therefore returns false.

## Solution Assessment

Yes. The narrow maintainable solution is to extend only `isAnnounceSkip` for a standalone final-line token while keeping `REPLY_SKIP` strict unless a separate report establishes that behavior needs to change.
