## Summary

Keep open. Current main still uses strict trimmed equality for ANNOUNCE_SKIP, so the reported multi-line final-token path remains source-reproducible; open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests treating a standalone final-line ANNOUNCE_SKIP in multi-line sub-agent output as silence instead of announcing the whole text.

## Best Possible Solution

Land or supersede #74136 with a focused helper contract, regression coverage, and docs/changelog alignment for final-line ANNOUNCE_SKIP while keeping REPLY_SKIP and NO_REPLY strict where intended.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is isAnnounceSkip("summary\nANNOUNCE_SKIP") on current main: the helper trims the whole string and compares strict equality, so it returns false while downstream announce paths would deliver the text.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only isAnnounceSkip for a standalone final token line, preserve the other control-token semantics, and cover that contract with focused tests and docs.
