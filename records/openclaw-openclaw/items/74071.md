## Summary

Keep open. Current main still implements `isAnnounceSkip` as strict trimmed equality, so a multi-line sub-agent output ending with a standalone `ANNOUNCE_SKIP` line is not suppressed; open PR #74136 is the active closing implementation candidate and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue requests that sub-agent announce suppression accept multi-line output whose final standalone line is `ANNOUNCE_SKIP`, not only output containing exactly that token.

## Best Possible Solution

Announce suppression should consistently treat a standalone final `ANNOUNCE_SKIP` line as silent across the helper-driven announce paths, with regression coverage for exact, whitespace-padded, final-line, and negative cases; this issue should close only after PR #74136 or an equivalent patch merges.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main: the helper compares the whole trimmed string to the token and returns false, and delivery/output-reader paths rely on that helper.

## Solution Assessment

Yes. The narrow maintainable fix is to extend only the announce-skip predicate for a standalone final token line while preserving unrelated silent-reply behavior; the linked open PR is the current implementation candidate.
