## Summary

Keep open. Current `main` still uses strict trimmed equality for `ANNOUNCE_SKIP`, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to suppress sub-agent announce output when a multi-line reply ends with `ANNOUNCE_SKIP` on its own final line.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage, and docs/changelog alignment for final-line `ANNOUNCE_SKIP`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`: current main trims and compares the whole string to the token, which returns false, and both announce delivery paths rely on that helper.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line while preserving `REPLY_SKIP` and `NO_REPLY` semantics, then cover that contract with focused tests and docs.
