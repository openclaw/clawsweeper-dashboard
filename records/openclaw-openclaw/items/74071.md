## Summary

Keep open. Current main still has the strict `ANNOUNCE_SKIP` equality check, the announce delivery paths depend on that helper, and open PR #74136 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that `isAnnounceSkip` treat a standalone final-line `ANNOUNCE_SKIP` in multi-line sub-agent output as an announce-suppression token.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage, and docs alignment for final-line `ANNOUNCE_SKIP` suppression.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which returns false while the announce paths depend on that predicate.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line, preserve `REPLY_SKIP` and `NO_REPLY` semantics, and cover the contract with focused tests and docs.
