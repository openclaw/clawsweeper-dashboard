## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so a summary followed by a final standalone token is still delivered instead of suppressed; open PR #74136 is the active closing implementation candidate and should be resolved before this issue closes.

## What This Changes

Sub-agent announce suppression should treat a standalone final-line `ANNOUNCE_SKIP` as silent even when earlier lines contain summary text.

## Best Possible Solution

Land a tested helper-level change that accepts only a standalone final-line `ANNOUNCE_SKIP` and update the user-facing contract if that behavior is accepted.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main: the helper compares the whole trimmed string to the token and both announce delivery paths rely on that helper.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line with focused regression coverage, while preserving existing `REPLY_SKIP` and `NO_REPLY` semantics.
