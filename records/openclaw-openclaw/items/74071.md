## Summary

Keep open: current main and v2026.5.7 still use strict whole-string equality for `isAnnounceSkip`, the reported final-line token case remains source-reproducible, and the open closing PR should be reviewed, landed, or superseded before this issue closes.

## What This Changes

The issue asks OpenClaw to treat a standalone final-line `ANNOUNCE_SKIP` token in multi-line sub-agent output as announce suppression instead of delivering the preceding summary.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/74136 with the final-line `ANNOUNCE_SKIP` contract, focused regression coverage, and docs alignment, then close this issue after the fix merges.

## Reproduction Assessment

Yes. The high-confidence source path is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which evaluates false while the exact padded token evaluates true.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line, preserve strict `REPLY_SKIP` and `NO_REPLY` semantics, and cover the helper plus announce flow with tests and docs.
