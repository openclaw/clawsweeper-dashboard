## Summary

Keep open. Current `main` still only suppresses `ANNOUNCE_SKIP` when the entire trimmed string is the token, and open PR #74136 uses closing syntax for the same bug, so this issue should remain open until that PR or an equivalent narrow fix merges.

## What This Changes

The issue requests that sub-agent announce suppression also accept multi-line output whose final standalone line is `ANNOUNCE_SKIP`.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `isAnnounceSkip` preserves exact-token suppression and also accepts a standalone final `ANNOUNCE_SKIP` line, with helper coverage, announce-flow regression coverage, docs alignment, and any required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current `main`; the helper compares the whole trimmed string to the token and returns false.

## Solution Assessment

Yes. The narrow maintainable fix is to extend only `isAnnounceSkip` for a standalone final-line token while preserving `REPLY_SKIP` and `NO_REPLY` semantics; the open linked PR is the current implementation candidate.
