## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so the reported multi-line final-token case remains source-reproducible, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue reports that strict equality in `isAnnounceSkip` lets multi-line sub-agent summaries ending with `ANNOUNCE_SKIP` leak as announcements.

## Best Possible Solution

Land or supersede PR #74136 with a focused helper contract, regression coverage, and docs/changelog alignment for standalone final-line `ANNOUNCE_SKIP`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which returns false.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line while preserving `REPLY_SKIP` and exact `NO_REPLY` semantics, then cover that contract with focused tests and docs.
