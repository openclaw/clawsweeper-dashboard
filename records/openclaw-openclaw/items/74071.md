## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, which source-reproduces the reported leak path, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Sub-agent output ending with a standalone final-line `ANNOUNCE_SKIP` should suppress inter-session announcement delivery.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage, and docs/changelog alignment for final-line `ANNOUNCE_SKIP`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which returns false while the suggested final-line check would return true.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line while preserving `REPLY_SKIP` and `NO_REPLY` semantics, then cover that contract with focused tests and docs.
