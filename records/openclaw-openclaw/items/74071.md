## Summary

Keep open. Current main still uses an exact trimmed-token comparison for `ANNOUNCE_SKIP`, and the linked implementation PR remains open and unmerged, so the issue should stay open until that PR or an equivalent narrow fix lands.

## What This Changes

The issue asks OpenClaw to treat multi-line sub-agent announcement output ending with a standalone `ANNOUNCE_SKIP` line as silent, instead of only suppressing replies that are exactly the token.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `ANNOUNCE_SKIP` suppression preserves the exact-token case and also accepts a standalone final token line, with focused helper coverage, announce-flow regression coverage, and updated prompt/docs wording if the contract changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main; the helper compares the entire trimmed string to the token and therefore returns false.

## Solution Assessment

Yes. The narrow maintainable fix is to extend only `isAnnounceSkip` for a standalone final-line token while preserving `REPLY_SKIP` and `NO_REPLY` semantics; the open linked PR is the current implementation candidate for that direction.
