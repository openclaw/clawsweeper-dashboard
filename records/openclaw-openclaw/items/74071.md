## Summary

Keep open. Current `main` still implements `isAnnounceSkip` as an exact trimmed-string comparison, the announce and output-reader paths still depend on that helper, and open PR #74136 already uses closing syntax for this same bug, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue requests that `ANNOUNCE_SKIP` suppress sub-agent announcements when it appears as a standalone final line after multi-line output, not only when it is the entire trimmed output.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so the helper preserves exact-token suppression and also accepts a standalone final `ANNOUNCE_SKIP` line, with focused helper coverage, announce-flow regression coverage, docs alignment, and any required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current `main`; the helper compares the whole trimmed string to the token and returns false.

## Solution Assessment

Yes. The narrow maintainable solution is to extend only `isAnnounceSkip` for a standalone final-line token while preserving the stricter `REPLY_SKIP` and `NO_REPLY` behavior; the linked open PR is the current implementation candidate.
