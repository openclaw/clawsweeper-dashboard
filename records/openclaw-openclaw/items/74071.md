## Summary

Keep open. Current main still implements `isAnnounceSkip` as an exact trimmed-token comparison, so a multi-line sub-agent reply ending with a standalone `ANNOUNCE_SKIP` still misses suppression; the issue is also paired with open PR #74136 using closing syntax, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that announce suppression accept multi-line sub-agent replies whose final standalone line is `ANNOUNCE_SKIP`, instead of requiring the whole trimmed reply to equal the token.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so announce suppression keeps the exact-token behavior and also accepts a standalone final-line `ANNOUNCE_SKIP`, with focused helper coverage, announce-flow regression coverage, and aligned prompt/docs wording.

## Reproduction Assessment

Yes. A source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`, which returns false on current main because the helper compares the whole trimmed string to the token; the announce flow then treats the reply as visible content.

## Solution Assessment

Yes. The narrow maintainable direction is to accept only an exact token or a standalone final-line token, while preserving `REPLY_SKIP` and `NO_REPLY` semantics; the active linked PR is the current implementation candidate for that path.
