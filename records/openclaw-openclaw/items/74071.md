## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so a multi-line sub-agent reply ending with a standalone token is not suppressed; the open linked PR #74136 is the active closing implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests that `isAnnounceSkip` also treat multi-line sub-agent output ending with a standalone `ANNOUNCE_SKIP` line as an announce-suppression token.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so only a standalone final `ANNOUNCE_SKIP` line suppresses announces while preserving existing `REPLY_SKIP` and `NO_REPLY` behavior and adding focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main: the helper compares the whole trimmed string to the token and returns false, and both announce-delivery paths rely on that helper.

## Solution Assessment

Yes. The narrow maintainable fix is to extend only the announce-skip predicate for a standalone final token line, keep unrelated reply-suppression semantics strict, and align tests/docs if the contract is accepted.
