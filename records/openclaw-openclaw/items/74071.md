## Summary

Keep open. Current `main` still uses strict trimmed equality for `ANNOUNCE_SKIP`, and open PR #74136 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Allow sub-agent announce suppression to treat a standalone final `ANNOUNCE_SKIP` line in multi-line output as silence instead of delivering the preceding text.

## Best Possible Solution

Review and land or supersede #74136 with a focused helper contract, regression coverage, and docs/changelog alignment for final-line `ANNOUNCE_SKIP`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")`: current main trims and compares the full string to the token, which returns false.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line while preserving `REPLY_SKIP` and `NO_REPLY` semantics unless separately justified.
