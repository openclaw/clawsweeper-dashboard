## Summary

Keep open. Current main still treats `isAnnounceSkip` as trimmed strict equality, so multi-line output ending with `ANNOUNCE_SKIP` remains deliverable; open PR #74136 is the active closing implementation candidate and should be reviewed instead of closing this issue early.

## What This Changes

The issue asks OpenClaw to suppress sub-agent announce output when a standalone final line is `ANNOUNCE_SKIP`, not only when the full output is exactly the token.

## Best Possible Solution

Land a tested helper-level fix that treats final-line `ANNOUNCE_SKIP` as silent for announce delivery while preserving strict `REPLY_SKIP` and `NO_REPLY` behavior and aligning docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `isAnnounceSkip("summary\nANNOUNCE_SKIP")` on current main, which evaluates false because the helper compares the whole trimmed string to the token.

## Solution Assessment

Yes. The narrow maintainable solution is to broaden only `isAnnounceSkip` for a standalone final token line and add helper plus announce-flow regression coverage; `REPLY_SKIP` can remain strict unless separately reported.
