## Summary

## What This Changes

The PR adds non-null/non-array guards in dashboard chat normalization and extraction helpers plus regression tests for malformed content array entries.

## Best Possible Solution

Rebase the guard onto current main by skipping non-record/non-array content entries inside the existing structured normalizer and extractor branches, keep focused regression coverage, and add redacted dashboard/session proof.

## Reproduction Assessment

Yes, at source level. Current main reads `.type` from unguarded content-array entries in `normalizeMessage`, `extractThinking`, and `extractRawText`, so a `null` or `undefined` entry can throw before the dashboard can render the rest of the session.

## Solution Assessment

No, not as currently diffed. The non-null/non-array guard is the right fix boundary, but it needs to be applied around current main's structured branches instead of replaying the older generic mapper.
