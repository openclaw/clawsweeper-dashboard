## Summary

Keep this PR open: the malformed content-block crash path is still source-reproducible on current main, but the branch is stale/conflicting and the normalizer hunk is not safe to apply as-is against the newer chat rendering code. The linked failed-media issue was closed for broader media rendering hardening, but it does not cover the remaining null/undefined helper dereferences this PR targets.

## What This Changes

The PR adds object guards and regression tests so dashboard chat normalization and text/thinking extraction skip malformed content entries.

## Best Possible Solution

Land a rebased UI-only guard that rejects non-object and array-valued content entries before every current normalizer/extractor dereference, while preserving attachment, canvas, assistant media/directive, and tool-argument behavior with focused regression tests.

Yes. Source inspection gives a high-confidence helper-level reproduction path: pass a message with `content` containing `null` or `undefined` to `normalizeMessage`, `extractThinking`, or the raw-text helper path, and current main reads `.type` before proving the block shape.

## Solution Assessment
