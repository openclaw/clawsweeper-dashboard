## Summary

Keep open: the underlying malformed-block crash is still source-reproducible on current main, and the PR direction is useful, but the normalizer hunk is stale against newer chat-content handling and the user-facing fix still needs a changelog entry.

## What This Changes

This PR hardens Control UI chat message normalization and text/thinking extraction to skip malformed `content` array entries and adds colocated regression tests.

## Best Possible Solution

Land a rebased UI-only fix that guards non-object and array-valued content entries in the current normalizer/extract implementations while preserving existing rich chat rendering behavior, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: call `normalizeMessage`, `extractThinking`, or `extractRawText` with `content` containing `null`, `undefined`, or an array beside valid blocks, and current main reads `.type` before validating the entry shape.

## Solution Assessment

No, not as currently diffed. The guard shape is the right fix direction, but it needs to be ported onto the current normalizer branches instead of replacing newer behavior, and the user-facing crash fix needs the required changelog entry.
