## Summary

Keep open: the malformed-content crash is still source-reproducible on current main, but this PR is stale against the current dashboard normalizer and lacks after-fix real behavior proof.

## What This Changes

The PR adds object guards and regression tests so dashboard chat normalization and text/thinking extraction skip null, undefined, and array-valued content entries.

## Best Possible Solution

Rebase the non-record/non-array guard into the current normalizer and extractor branches, preserve structured UI content handling, keep focused regression tests, and add redacted dashboard/session proof.

## Reproduction Assessment

Yes, at source level. Current main reads `.type` from unguarded content-array entries in `normalizeMessage`, `extractThinking`, and `extractRawText`, so a `null` or `undefined` entry can throw before the dashboard can render the rest of the session.

## Solution Assessment

No, not as currently diffed. The guard is the right fix boundary, but it needs to be applied around current main's structured normalizer and extractor branches instead of replaying the older mapper.
