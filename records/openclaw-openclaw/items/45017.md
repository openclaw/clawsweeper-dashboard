## What This Changes

The PR adds non-null, non-array object guards to Control UI chat message normalization and text/thinking extraction, plus regression tests for malformed content entries.

## Best Possible Solution

Land a rebased UI-only fix that skips non-object and array-valued content entries in normalizeMessage, extractThinking, and extractRawText while preserving the current attachment, canvas, media/directive, assistant text, and tool-block behavior; add targeted tests and a CHANGELOG.md Unreleased/Fixes entry. Yes. A high-confidence helper-level reproduction is a message whose content array contains null, undefined, or an array beside a valid text/thinking block; current main still reads x.type or item.type on those malformed entries in the normalizer and extraction helpers.

## Solution Assessment

No, not as currently diffed. The non-null/non-array guard is the right solution shape, but it must be ported onto the current normalizer/extract implementations instead of taking the older generic content mapper.
