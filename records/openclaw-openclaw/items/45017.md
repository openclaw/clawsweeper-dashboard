The PR adds guards and tests so Control UI chat normalization and text/thinking extraction skip malformed null, undefined, and array-valued content blocks.

## Best Possible Solution

Land a rebased UI-only fix that skips non-object and array-valued content entries while preserving current chat content variants and documenting the user-facing crash fix.

## Solution Assessment

No, not as currently diffed. Guarding malformed entries is the right direction, but it needs to be ported onto the current normalizer branches instead of replacing newer chat-rendering behavior, and the fix needs a changelog entry.
