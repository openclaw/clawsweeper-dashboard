This PR updates Control UI chat message normalization and text/thinking extraction to skip malformed content blocks and adds colocated regression tests.

## Best Possible Solution

Land a rebased UI-only fix that skips non-object and array-valued content entries in normalizeMessage, extractThinking, and extractRawText while preserving current chat rendering behavior and adding the missing changelog entry.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is a message whose content array contains null, undefined, or an array beside a valid text/thinking block; current main still dereferences type on those malformed entries in the normalizer and extraction helpers.

## Solution Assessment

No, not as currently diffed. The guard shape is right, but it needs to be ported onto the current normalizer/extract implementations instead of replacing the newer normalizer behavior with the older generic mapper.
