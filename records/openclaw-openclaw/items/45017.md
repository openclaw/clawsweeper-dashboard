## What This Changes

The branch adds record guards and regression tests so dashboard chat normalization and text/thinking extraction skip null, undefined, and array-valued content blocks.

## Best Possible Solution

Apply a non-null, non-array record guard before every content-block property read in current normalizeMessage, extractThinking, and extractRawText, while preserving existing structured content handling and focused tests.

## Reproduction Assessment

Yes, at source level. Current main can read item.type from null or undefined content blocks in normalizeMessage, extractThinking, and extractRawText, and grouped-render reaches those helpers for dashboard chat rendering.

## Solution Assessment

No, not as currently diffed. The guard is the right fix boundary, but it needs to be ported onto current main's richer structured normalizer and extractor paths instead of replaying the older mapper.
