## Summary

Keep open. Current `main` still has the strict model-id equality that the PR targets, the source path matches the linked bug report, and the proposed patch is narrow with no blocking review findings.

## What This Changes

The PR changes agent context-window resolution to match bare and provider-scoped model ids in either direction, with regression tests and a changelog entry.

## Best Possible Solution

Land a narrow agent-owned fix that treats bare and `provider/model` ids as equivalent only under the active provider, preserving configured context limits with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current `main` uses strict equality in `resolveContextWindowInfo`, so a runtime id like `openrouter/tiny` cannot match a configured id `tiny`. The call path from the embedded runner then falls back to runtime model metadata instead of the configured context limit.

## Solution Assessment

Yes. The PR's local symmetric provider-prefix match is the narrowest maintainable fix for this path, and the added tests cover both prefix-mismatch directions without widening core provider behavior.
