Keep open: current main still does not implement `/restore`, and this active feature PR still has blocking source-level issues plus missing real behavior proof before it can be considered for merge.

## What This Changes

The PR adds a built-in `/restore [number]` command, transcript archive listing/restoration helpers, session-key transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the branch to use the current reset-hook seam, restore into `resetResult.entry.sessionFile`, define a fail-closed or explicitly approved legacy archive policy, filter archives after ownership checks, and add real behavior proof.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the PR head imports a removed seam, recomputes a transcript path instead of using the reset result, and accepts keyless or unreadable archive headers for any session key.

## Solution Assessment

No. A built-in restore command may be useful, but this implementation is not the narrow maintainable solution until reset owns the target path and archive ownership fails closed or has an explicit approved legacy policy.
