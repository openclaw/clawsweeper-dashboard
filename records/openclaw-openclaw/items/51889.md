## Summary

Keep open: this is an active external feature PR, current main still lacks the command, and source review finds blocking current-main integration, restore-path, and archive-ownership defects plus missing real behavior proof.

## What This Changes

The PR adds a built-in `/restore [number]` command, transcript archive listing/restoration helpers, session-key transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the branch to use the current reset-hook seam, restore into `resetResult.entry.sessionFile`, define a fail-closed or explicitly approved legacy archive policy, filter archives before applying scan caps, and add real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: the added handler imports a seam removed from current main, the restore helper recomputes a transcript path instead of using the reset result, and the archive filter treats keyless or unreadable headers as visible to any session key.

## Solution Assessment

No. A built-in restore command may be a good direction, but this implementation is not the narrow maintainable solution until reset owns the target path and archive ownership fails closed or has an explicit legacy policy.

  - confidence: 0.94
- **[P1] Fail closed on unreadable or unowned archives:** `src/gateway/session-restore.ts:198-202`
  - body: `readSessionKeyFromHeader` collapses missing keys, parse failures, and read errors to `null`, and this branch accepts `null` for every requested `sessionKey`. That can list and restore another channel's archived transcript.
  - confidence: 0.93
- **[P2] Filter before applying the archive scan cap:** `src/gateway/session-restore.ts:193`
  - body: The scan ceiling is applied before session-key filtering, so newer archives from other channels can consume the whole window and make valid current-channel archives disappear. Bound total work if needed, but collect matches after ownership filtering.
  - confidence: 0.86
- **[P2] Avoid running reset hooks twice:** `src/auto-reply/reply/commands-restore.ts:104-117`
  - body: The handler emits reset command hooks and then calls `performSessionRestore`, whose `performGatewaySessionReset` path triggers the internal command hook again. Hook consumers can see duplicate side effects for one `/restore` invocation.
  - confidence: 0.84
