## Summary

Keep open: current main still lacks a built-in `/restore`, but this PR is not merge-ready because the latest patch has a current-main import break plus restore-path and transcript-ownership defects that affect correctness and confidentiality.

## What This Changes

The PR adds a built-in `/restore [number]` slash command, archive listing/restoration helpers, sessionKey transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the branch to use the current reset-hook seam, restore into `resetResult.entry.sessionFile`, fail closed on archive ownership unless a safe legacy policy is explicitly designed, scan after filtering, and add the required changelog plus regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the new import targets a non-exporting current-main module, and the restore helper accepts `null` archive ownership while recomputing a transcript path that can differ from the reset service's persisted `entry.sessionFile`.

## Solution Assessment

No. A built-in restore command may be a good product direction, but this implementation is not the best fix until it uses the reset service as the single path owner and defines a fail-closed archive ownership contract.

  - body: `performGatewaySessionReset` persists the active transcript path in `resetResult.entry.sessionFile` and may preserve an existing custom path. Recomputing a new path here can move the archive somewhere the session store will not read, so `/restore` can report success while the next turn uses the reset transcript.
  - confidence: 0.94
- **[P1] Require archive ownership before listing or restoring:** `src/gateway/session-restore.ts:198-202`
  - body: `readSessionKeyFromHeader` collapses missing keys, malformed/truncated headers, and read errors to `null`, and this filter accepts `null` for every requested session key. In a multi-channel agent store, that can show and restore another channel's transcript.
  - confidence: 0.92
- **[P2] Let one reset path emit hooks:** `src/auto-reply/reply/commands-restore.ts:103-113`
  - body: This handler emits reset command hooks and then calls `performGatewaySessionReset`, which also triggers reset/internal hook handling. Hook consumers can see duplicate command side effects or conflicting before-reset snapshots; restore should route through one reset owner.
  - confidence: 0.84
- **[P2] Apply the scan cap after ownership filtering:** `src/gateway/session-restore.ts:193`
  - body: The code slices candidates before checking `sessionKey`, so newer archives from other channels can consume the whole scan window and hide valid current-channel archives. Bound total work if needed, but collect enough matching archives after filtering.
  - confidence: 0.86
