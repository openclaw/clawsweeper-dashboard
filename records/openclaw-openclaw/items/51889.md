The branch adds a built-in `/restore [number]` command, archive listing/restoration helpers, `sessionKey` transcript headers, docs, and unit tests for restoring reset/deleted session archives.

## Best Possible Solution

Rebase onto current main and implement archive restore against the current reset-hook and session-file contracts with fail-closed archive ownership, while leaving broader named snapshot/load semantics to the canonical session snapshot/archive design.

Yes. Source inspection of PR head against current main gives high-confidence reproduction paths for the stale import, reset-owned transcript mismatch, fail-open archive filter, pre-filter scan ceiling, and duplicate reset-hook behavior.
