Rework the PR around `resetResult.entry.sessionFile`, the current `commands-reset-hooks` seam, and a fail-closed archive ownership policy, then add redacted real `/restore` list and restore proof while keeping broader snapshot/import semantics aligned with https://github.com/openclaw/openclaw/issues/13700 and https://github.com/openclaw/openclaw/issues/45003.

  - body: `readSessionKeyFromHeader` returns `null` for missing keys, parse failures, and read errors, and this branch treats `null` as visible to every requested session key. In multi-channel deployments, an authorized user in one channel can be offered another channel's archived transcript.
  - confidence: 0.93
- **[P2] Scan until enough owned archives match:** `src/gateway/session-restore.ts:189-193`
  - body: The scan ceiling is applied before the session-key filter, so newer archives from other channels can consume the whole window and hide valid current-channel archives. Apply the limit after ownership filtering or use a bounded but ownership-aware scan.
  - confidence: 0.86
- **[P2] Avoid running reset hooks twice:** `src/auto-reply/reply/commands-restore.ts:101-113`
  - body: The handler emits reset command hooks and then calls `performSessionRestore`, whose reset path emits internal and before-reset lifecycle hooks again. Hook consumers can see duplicate side effects for one restore operation.
  - confidence: 0.84
