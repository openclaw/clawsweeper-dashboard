## Summary

Keep open: current main still lacks a built-in archive restore command, but this PR is not merge-ready because the latest head still has reset-hook, transcript-path, archive-ownership, duplicate-hook, and real-proof blockers.

## What This Changes

The PR adds a built-in `/restore [number]` slash command, archive listing/restoration helpers, `sessionKey` transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the PR around reset-owned `entry.sessionFile`, the current reset-hook seam, and an explicit archive ownership policy, then prove the real `/restore` list and restore flow; keep the broader named snapshot design in https://github.com/openclaw/openclaw/issues/13700.

## Reproduction Assessment

Yes. Source inspection of current main plus the PR head gives high-confidence reproduction paths for the import failure, reset target mismatch, fail-open archive filtering, and duplicate hook behavior.

## Solution Assessment

No. A restore command may be useful, but this implementation is not the safest maintainable path until it uses reset-owned transcript paths and has an explicit archive ownership policy with real proof.
