## Summary

Keep open: current main still lacks a built-in `/restore` command, but this PR is not merge-ready because the latest head has a stale reset-hook import, restores to a path that can diverge from the session store, fails open on keyless or unreadable archives, and lacks real behavior proof.

## What This Changes

The PR adds a built-in `/restore [number]` chat command, session archive listing/restoration helpers, sessionKey transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the branch to use the current reset-hook seam, restore into `resetResult.entry.sessionFile`, define a fail-closed or explicitly approved legacy archive policy, filter archives after ownership checks, and add real behavior proof.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the PR head imports a removed seam, recomputes a transcript path instead of using the reset result, and accepts keyless or unreadable archive headers for any session key.

## Solution Assessment

No. A built-in restore command may be useful, but this implementation is not the narrow maintainable solution until reset owns the target path and archive ownership fails closed or has an explicit approved legacy policy.

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import reset hooks from the current seam:** `src/auto-reply/reply/commands-restore.ts:4`
  - body: Current main no longer exports `emitResetCommandHooks` from `commands-core.ts`; the hook seam is `commands-reset-hooks.ts`, so this added handler will fail to compile or load after rebasing onto main.
  - confidence: 0.9
- **[P1] Restore into the reset entry session file:** `src/gateway/session-restore.ts:248-252`
  - body: `performGatewaySessionReset` persists the active transcript path in `resetResult.entry.sessionFile`. Recomputing from `newSessionId` and `topicId` can move the archive to a file the session store will not read, making restore report success while the next turn uses the fresh reset transcript.
  - confidence: 0.94
- **[P1] Fail closed on unreadable or unowned archives:** `src/gateway/session-restore.ts:198-202`
  - body: `readSessionKeyFromHeader` collapses missing keys, parse failures, and read errors to `null`, and this branch accepts `null` for every requested `sessionKey`. That can list and restore another channel's archived transcript.
  - confidence: 0.93
- **[P2] Filter archives before enforcing the scan ceiling:** `src/gateway/session-restore.ts:193`
  - body: The scan ceiling is applied before session-key filtering, so newer archives from other channels can consume the whole window and make valid current-channel archives disappear. Bound total work if needed, but collect matches after ownership filtering.
  - confidence: 0.86
- **[P2] Avoid firing reset hooks twice:** `src/auto-reply/reply/commands-restore.ts:103-116`
  - body: The handler emits reset command hooks and then calls `performSessionRestore`, whose `performGatewaySessionReset` path triggers the internal command hook again. Hook consumers can see duplicate side effects for one `/restore` invocation.
  - confidence: 0.84
