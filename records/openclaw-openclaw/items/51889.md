## Label Justifications

- `P1`: The PR changes session restoration behavior and currently has blockers that can fail to restore history or expose another channel's transcript.
- `merge-risk: 🚨 session-state`: The feature moves archived transcript files into active session state and can mis-associate the store entry with the wrong transcript file.
- `merge-risk: 🚨 security-boundary`: Fail-open archive ownership filtering can reveal or restore transcripts that belong to a different channel or session key.

## Summary

This PR should stay open: the feature is not on current main, but the branch still has merge-blocking restore correctness and transcript-confidentiality issues, and the PR still lacks real behavior proof for the new command flow.

## What This Changes

The PR adds a built-in `/restore [number]` slash command, transcript archive discovery/restoration helpers, `sessionKey` transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the restore flow to use the current reset-hook seam, restore into the reset result's stored transcript path, define a fail-closed or explicitly approved legacy archive ownership policy, and prove the real list-and-restore flow before merge.

## Reproduction Assessment

Yes. Source inspection of current main plus the PR diff gives a high-confidence path for the blockers: stale reset-hook import, recomputed restore target instead of the stored reset entry path, duplicate reset hook execution, and fail-open archive ownership filtering.

## Solution Assessment

No. A restore command may be useful, but this implementation is not the narrow maintainable solution until reset owns the target transcript path and archive ownership has a safe, explicit policy.

  - confidence: 0.94

  - confidence: 0.93

  - body: The scan ceiling is applied before session-key filtering, so newer archives from other channels can consume the whole window and make valid current-channel archives disappear. Bound total work if needed, but collect matches after ownership filtering.
  - confidence: 0.86

  - confidence: 0.84
