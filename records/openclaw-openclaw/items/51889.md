## Summary

Keep open: current main still lacks a first-party `/restore` command, and this PR is the active partial implementation, but the branch is not mergeable due stale reset-hook imports, restoring into the wrong transcript path, fail-open archive ownership, and missing real behavior proof.

## What This Changes

The PR adds a built-in `/restore [number]` chat command, archive listing/restoration helpers, `sessionKey` transcript headers, docs, and restore-focused tests.

## Best Possible Solution

Revise the branch to use the current reset-hook seam, restore into the reset result's `sessionFile`, fail closed or explicitly design legacy archive ownership, and prove the real list/restore flow.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the PR head imports a helper from a seam current main no longer exports, recomputes the restore target instead of using the reset result's stored transcript path, and treats keyless or unreadable archive headers as visible to any session key.

## Solution Assessment

No. A restore command may be useful, but this implementation is not the narrow maintainable solution until reset owns the target transcript path and archive ownership fails closed or has an explicitly approved legacy policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import reset hooks from the current seam:** `src/auto-reply/reply/commands-restore.ts:4`
  - body: Current main exports `emitResetCommandHooks` from `commands-reset-hooks.ts`, not `commands-core.ts`. This added handler will fail to compile or load after rebasing onto current main.
  - confidence: 0.9
- **[P1] Restore into the reset entry session file:** `src/gateway/session-restore.ts:248-252`
  - body: `performGatewaySessionReset` persists the active transcript path in `resetResult.entry.sessionFile`. Recomputing a path from `newSessionId` and `topicId` can move the archive somewhere the session store will not read, so restore can report success while the next turn uses the fresh reset transcript.
  - confidence: 0.94
- **[P1] Fail closed on unreadable or unowned archives:** `src/gateway/session-restore.ts:198-202`
  - body: `readSessionKeyFromHeader` collapses missing keys, parse failures, and read errors to `null`, and this branch accepts `null` for every requested `sessionKey`. That can list and restore another channel's archived transcript.
  - confidence: 0.93
- **[P2] Filter archives before enforcing the scan ceiling:** `src/gateway/session-restore.ts:193`
  - body: The scan ceiling is applied before session-key filtering, so newer archives from other channels can consume the whole window and make valid current-channel archives disappear. Bound total work if needed, but collect matches after ownership filtering.
  - confidence: 0.86
- **[P2] Avoid firing reset hooks twice:** `src/auto-reply/reply/commands-restore.ts:103-116`
  - body: The handler emits reset command hooks and then calls `performSessionRestore`, whose `performGatewaySessionReset` path triggers the internal command hook again. Hook consumers can see duplicate side effects for one `/restore` invocation.
  - confidence: 0.84
