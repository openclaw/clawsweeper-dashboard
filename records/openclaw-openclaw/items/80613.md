## Summary

Keep open. Current main still has both reported memory-core risks, and two open closing PRs already split the fix work across the CJK dedupe and promotion-rehydration halves, so this issue should stay open until those PRs land or are replaced.

## What This Changes

The issue reports that memory dreaming can promote managed dreaming block scaffolding into `MEMORY.md` and that CJK snippets bypass fuzzy dedupe because dreaming uses ASCII-only tokenization.

## Best Possible Solution

Land or replace the focused memory-core fixes: share the CJK-aware tokenizer for dreaming dedupe and redact managed dreaming ranges before promotion rehydration appends to `MEMORY.md`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the CJK dedupe half: CJK-only snippets tokenize to empty sets and fall back to exact-match similarity. The promotion leak path is also source-reproducible through raw rehydration, with a focused fixture supplied in https://github.com/openclaw/openclaw/pull/80702, but I did not execute it here.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or extract the existing CJK-aware memory tokenizer and sanitize managed Light/REM ranges before relocation, without adding a config or product policy change.
