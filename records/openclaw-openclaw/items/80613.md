## Summary

Keep open: current main still has a source-reproducible CJK dedupe defect, and the promotion rehydration path can still leak managed dreaming block text when a stored candidate range spans the block.

## What This Changes

The issue reports that memory dreaming promotion can include managed candidate metadata in promoted snippets and that CJK text is not deduplicated effectively during dreaming.

## Best Possible Solution

Fix this inside memory-core by using the existing CJK-aware tokenization for dreaming dedupe and by sanitizing or clamping promotion rehydration so managed dreaming blocks can never be appended to MEMORY.md.

## Reproduction Assessment

Yes, at source level: current main's dedupe path returns similarity 0 for related pure CJK snippets, and the promotion relocation logic can select raw managed dreaming lines for a broad stored range. I did not run the full CLI because the review checkout must remain read-only.

## Solution Assessment

Yes: a narrow memory-core fix is the best path, reusing or sharing the existing CJK-aware tokenizer and adding a promotion rehydration guard/regression for managed dreaming blocks. No new config, plugin API, or product decision is needed.
