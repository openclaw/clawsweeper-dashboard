## What This Changes

The issue reports two memory-core dreaming bugs: managed dreaming scaffolding leaking into promoted MEMORY.md snippets and CJK snippets failing fuzzy dedupe because dreaming tokenization is ASCII-only.

## Best Possible Solution

Land or supersede the focused memory-core PRs so dreaming dedupe uses the shared CJK-aware tokenizer and promotion rehydration cannot append managed dreaming ranges to MEMORY.md.

## Reproduction Assessment

Yes for the CJK dedupe path: current source still tokenizes CJK-only snippets to empty sets and the focused command returns similarity 0 for Chinese paraphrases. ## Solution Assessment
