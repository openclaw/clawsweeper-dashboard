## Summary

Keep open. Current main still has the source-reproducible CJK dreaming dedupe bug, and live GitHub shows an open closing PR for that half; the managed-block promotion leak also still needs either a focused sanitizer/regression or an explicit split from this issue before cleanup closure is safe.

## What This Changes

The issue reports two memory-core dreaming regressions: managed dreaming block scaffolding can be promoted into MEMORY.md, and CJK-only snippets do not dedupe because dreaming tokenization is ASCII-only.

## Best Possible Solution

Land or replace the narrow CJK tokenizer fix, then either split the managed-block leak into its own focused memory-core sanitizer task or fix it with a targeted rehydration regression while leaving raw-vs-distilled promotion policy to https://github.com/openclaw/openclaw/issues/67363.

## Reproduction Assessment

Yes for the CJK dedupe half: current main's tokenizer drops CJK-only text and the read-only probe shows similarity 0 for near-duplicate Chinese snippets. The managed-block leak half is credible from source and linked PR proof, but still needs a focused current-main promotion fixture.

## Solution Assessment

Yes for the CJK half: reusing or extracting the existing CJK-aware MMR tokenizer is the narrowest maintainable fix. The promotion leak should be handled at the rehydration/sanitization boundary, not through the broad TTS PR or the separate raw-vs-distilled durable-memory policy thread.
