## What This Changes

The PR moves image, music, and video generation onto a shared async media task lifecycle, adds async image task status and duplicate guarding, and updates tests, docs, and changelog entries.

## Best Possible Solution

Keep the shared async lifecycle, but preserve the existing synchronous image result when the completion path cannot prove delivery to a requester target.

## Solution Assessment

No; the shared lifecycle is the right direction, but the async detach predicate needs to match delivery capability or fall back to the existing synchronous image result.
