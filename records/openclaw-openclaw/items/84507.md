## What This Changes

The PR adds a Commander `InvalidArgumentError` parser to `openclaw skills search --limit` so invalid or non-positive values fail before the ClawHub skills request.

## Best Possible Solution

Use one strict positive-whole-integer parser for both skills and plugins search, add focused command-level regression coverage, and keep the linked issue open until both reported commands fail fast.
