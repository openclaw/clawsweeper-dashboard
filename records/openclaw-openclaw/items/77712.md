## Summary

## What This Changes

Cron delivery preview reports `Unsupported channel: feishu` for configured Feishu announce jobs even though delivery succeeds.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/78809; the desired end state is a generic outbound target resolver bootstrap path with focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main’s cron preview dry-runs `resolveDeliveryTarget`, which reaches `resolveOutboundTarget`; that resolver currently omits `allowBootstrap` and can return the exact unsupported-channel error when the configured channel plugin is not loaded.

## Solution Assessment

Yes, the linked PR’s generic use of the existing `allowBootstrap` seam is the narrowest maintainable direction. It keeps target ownership in channel plugins and avoids hard-coding Feishu in cron preview code.
