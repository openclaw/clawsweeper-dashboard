## Summary

Keep open: the report targets a real cron delivery-preview path, and I did not find a merged issue-specific fix proving Feishu announce previews ignore stale `plugins.entries.feishu`. ## What This Changes

Feishu announce cron jobs configured through `channels.feishu` should not show `Unsupported channel: feishu` in `cron list` or status previews when delivery succeeds.

## Best Possible Solution

Add regression coverage for Feishu announce preview with `channels.feishu` plus stale `plugins.entries.feishu`, then fix the generic preview/target-resolution path so configured extension channels resolve through their channel owner without Feishu-specific core policy.

## Solution Assessment
