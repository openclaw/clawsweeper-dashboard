## What This Changes

The issue asks cron list/status delivery previews to show valid Feishu announce targets instead of `Unsupported channel: feishu` when `channels.feishu` is configured but `plugins.entries.feishu` is stale.

## Best Possible Solution

Make cron preview resolution use the same effective channel-plugin path as successful Feishu delivery, with regression coverage for `channels.feishu` plus a stale `plugins.entries.feishu` entry.
