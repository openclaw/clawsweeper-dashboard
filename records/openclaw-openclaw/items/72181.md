## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised approval mode, and the feature changes private-message delivery, owner command parsing, correction logging, TTL behavior, and agent-origin tagging, so it needs BlueBubbles maintainer/product/privacy review before implementation.

## What This Changes

Request adds optional BlueBubbles trainer/supervised modes that run the agent, capture outbound DM drafts for owner approval or correction through iMessage, log corrections, and optionally tag approved AI drafts.

## Best Possible Solution

Keep this BlueBubbles-owned; after maintainer approval, add schema-backed trainer settings, intercept final DM delivery before send, manage pending drafts with TTL, parse approve/correct commands, define privacy-aware correction logging, update docs, and test config plus delivery paths.
