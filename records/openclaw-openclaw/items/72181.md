## Summary

Keep open. Current main still lacks the requested BlueBubbles owner-gated trainer/supervised reply mode, and the remaining work is a BlueBubbles-owned message-delivery and privacy feature that needs maintainer/product review before implementation.

## What This Changes

Add optional BlueBubbles `trainerMode`/supervised reply behavior that lets the agent draft DM replies, route them to the owner for approval or correction through iMessage, and prevent unapproved delivery to the original sender.

## Best Possible Solution

Keep this BlueBubbles-owned unless a reusable channel approval seam becomes necessary: add schema-backed trainer/supervised config, intercept DM final delivery before text/media sends, persist bounded pending drafts with expiration, parse owner approve/correct commands, deliver only approved supervised replies, define private correction logging semantics, and update BlueBubbles docs plus focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Exact field searches return no trainer config, and the BlueBubbles delivery adapter still sends final replies directly to the resolved target.

## Solution Assessment

Unclear until BlueBubbles maintainer/product review. The proposed plugin-local boundary matches the extension architecture, but owner-command syntax, correction storage, expiration behavior, and AI tagging need sign-off before a fix PR is safe.
