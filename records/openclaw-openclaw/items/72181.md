## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised owner-gated reply mode, and the feature touches private-message delivery, correction logging, and AI-origin tagging, so it needs BlueBubbles maintainer/product/privacy review rather than a cleanup close or autonomous repair job.

## What This Changes

Request a BlueBubbles plugin `trainerMode` with `training` and `supervised` modes that lets the agent draft replies but requires owner approval or correction before any DM reply is delivered.

## Best Possible Solution

Keep this BlueBubbles-owned unless implementation proves a reusable channel seam is necessary: after maintainer approval, add schema-backed trainerMode/trainerNotifyNumber/agentTag config, intercept only DM final delivery before BlueBubbles sends, store pending drafts with TTL, parse owner approve/correct commands, deliver only approved supervised replies, define correction logging privacy rules, and update BlueBubbles docs plus focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Exact source/docs searches show the requested config surface is absent, and the BlueBubbles final delivery path still sends assistant replies directly to the resolved target.

## Solution Assessment

Unclear until BlueBubbles maintainer/product review. The proposed plugin-local boundary is the right starting point, but correction logging, owner command syntax, AI tagging, and private-message safety need explicit decisions before a fix PR is queued.
