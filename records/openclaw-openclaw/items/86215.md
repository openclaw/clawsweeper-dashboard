## Summary

Keep open: current main has meaningful mitigations, but it still does not fully solve the Codex/OpenAI auth incident family because primary auth failures are still retried by default, fallback suppression is opt-in and session-local, API-key-backed Codex routes remain possible, and the issue has fresh live reports of missed cron/channel output.

## What This Changes

The issue requests durable handling for Codex/OpenAI OAuth refresh failures, including clearer operator incident reporting, more aggressive profile quarantine/rotation, session cleanup, and visibility into cost-bearing API-key route drift.

## Best Possible Solution

Define one Codex/OpenAI auth incident model that durably contains repeated or permanent refresh failures, rotates profiles safely, prevents or loudly reports API-key fallback, releases wedged sessions, and exposes redacted status/doctor evidence.

## Reproduction Assessment

Yes, source-reproducible for the remaining default gap: current main still retries primary candidates and leaves fallback skip disabled by default, while issue comments provide multiple live shipped-version repros.

## Solution Assessment

No complete solution exists on main; the timeout increase and cooldown state are useful partial mitigations, but the best fix needs maintainer-approved Codex/OpenAI auth incident and fallback policy.
