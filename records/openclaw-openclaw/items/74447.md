## Summary

This PR should stay open because the intended stale required-allowlist cleanup is not yet on current main, and the proposed patch weakens the unused-file guard by moving the former required entries into the optional list where stale entries are not enforced.

## What This Changes

The PR empties the required Knip unused-file allowlist and moves the previous required entries, plus `src/gateway/test/server-sessions-helpers.ts`, into the optional unused-file allowlist.

## Best Possible Solution

Keep the required allowlist empty only if the current production Knip scan is clean, delete stale paths from both allowlists, preserve only documented platform-dependent optional entries, and merge after deadcode and formatting checks pass.

## Reproduction Assessment

Yes. The helper-level reproduction is high-confidence: an optional-only path is allowed when reported and is not stale when absent, and the existing unit test documents that behavior.

## Solution Assessment

No. Moving stale required entries into the optional allowlist makes the guard pass by bypassing stale enforcement; the safer fix is to remove obsolete entries from both lists except true platform-dependent cases.
