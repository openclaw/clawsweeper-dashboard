## Summary

Keep open. Current main has not implemented the intended cleanup, and the PR diff weakens the unused-file guard by moving the former required Knip findings into the optional allowlist, where stale-entry enforcement no longer applies.

## What This Changes

The PR changes `scripts/deadcode-unused-files.allowlist.mjs` to empty the required Knip unused-file allowlist and move the previous required entries, plus the gateway test helper, into the optional allowlist.

## Best Possible Solution

Keep the required allowlist empty only if the current production Knip scan is clean, delete stale paths from the optional allowlist too, keep optional entries limited to documented platform-dependent findings, and merge after focused deadcode and formatting checks pass.

## Reproduction Assessment

Yes. The focused reproduction is the helper-level optional allowlist behavior: an optional path is accepted whether Knip reports it or not, so moving stale required entries to optional bypasses stale-entry failures.

## Solution Assessment

No. Emptying the required list may be correct, but moving the stale required entries into optional is broader than needed and makes future unused-file regressions easier to miss.
