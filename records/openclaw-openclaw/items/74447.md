## Summary

Keep open: current main has not made the requested cleanup, and the PR weakens the unused-file guard by moving stale required entries into an optional list that is accepted whether or not Knip reports those paths.

## What This Changes

The PR updates `scripts/deadcode-unused-files.allowlist.mjs` to empty the required Knip unused-file allowlist and move the previous required entries plus `src/gateway/test/server-sessions-helpers.ts` into the optional allowlist.

## Best Possible Solution

Delete the stale production entries from both allowlists, keep `KNIP_OPTIONAL_UNUSED_FILE_ALLOWLIST` limited to documented platform-dependent test-only findings, and merge after focused deadcode and formatting checks pass.

## Reproduction Assessment

Yes. The helper-level reproduction is clear: an optional-only path produces no `unexpected` and no `stale` result, and the existing unit test documents that behavior.

## Solution Assessment

No. Emptying the required list may be the right cleanup if the production scan is clean, but moving the stale entries into optional weakens future unused-file detection instead of removing obsolete exemptions.
