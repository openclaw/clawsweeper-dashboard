## Summary

Keep open. The PR is not safe to merge as-is because it moves stale required Knip unused-file exemptions into the optional allowlist, and the current checker/test semantics show optional entries bypass stale enforcement.

## What This Changes

The PR changes `scripts/deadcode-unused-files.allowlist.mjs` to empty `KNIP_UNUSED_FILE_ALLOWLIST` and move the prior required paths plus the gateway test helper into `KNIP_OPTIONAL_UNUSED_FILE_ALLOWLIST`.

## Best Possible Solution

Keep the deadcode allowlists minimal: delete stale production entries from both lists, leave only documented platform-dependent test-only entries optional, and merge after focused deadcode checks pass.

## Reproduction Assessment

Yes. The helper-level reproduction is high confidence: optional-only files produce no `unexpected` and no `stale` result, and the existing unit test asserts that behavior.

## Solution Assessment

No. Emptying the required list may be correct if the production Knip scan is clean, but moving stale required entries into optional weakens future unused-file detection instead of removing obsolete exemptions.
