Keep this PR open, but it is not merge-ready: current main still has required unused-file allowlist entries, and the PR weakens the guard by moving stale required paths into the optional list where stale-entry enforcement does not apply.

## What This Changes

The PR empties the required Knip unused-file allowlist and moves the previous required entries, plus `src/gateway/test/server-sessions-helpers.ts`, into the optional unused-file allowlist.

## Best Possible Solution

Keep the deadcode allowlists minimal: remove stale entries from both lists, leave optional entries only for documented platform-dependent Knip discrepancies, then rebase and validate the PR head.

No. Emptying the required list may be appropriate if the scan is clean, but moving stale production entries into the optional list is not the narrowest maintainable cleanup.
