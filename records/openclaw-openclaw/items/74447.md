## Summary

This PR should stay open because current main has not implemented the requested cleanup, and the proposed patch still weakens the unused-file guard by moving the former required production entries into the optional allowlist. Optional entries are deliberately not stale-checked, so the patch would keep broad exemptions that future `pnpm deadcode:unused-files` runs could no longer catch.

## What This Changes

The PR updates `scripts/deadcode-unused-files.allowlist.mjs` by emptying `KNIP_UNUSED_FILE_ALLOWLIST` and moving the previous required entries, plus `src/gateway/test/server-sessions-helpers.ts`, into `KNIP_OPTIONAL_UNUSED_FILE_ALLOWLIST`.

## Best Possible Solution

Keep the unused-file guard strict: remove stale production entries from both allowlists, reserve `KNIP_OPTIONAL_UNUSED_FILE_ALLOWLIST` for documented platform-dependent test-only findings, and merge only after focused deadcode and formatting checks pass on the PR head.

## Reproduction Assessment

Yes. The helper-level reproduction is clear from `compareUnusedFilesToAllowlist`: with a path in the optional list and not in the required list, the checker reports neither `unexpected` nor `stale`; the existing unit test documents that behavior.

## Solution Assessment

No. Emptying the required list may be the narrow repair if the current Knip production scan is clean, but moving the whole stale required list into optional weakens future detection instead of removing stale exemptions.
