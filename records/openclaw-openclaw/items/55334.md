## Summary

Keep this open. Current main and the latest release still keep `skillsSnapshot.prompt` and `systemPromptReport` inline in session rows while only stripping `resolvedSkills`; pruning and cleanup mitigations exist, but the focused prompt-stripping fix remains in an open PR and the broader OOM/lock-latency picture still needs maintainer sequencing.

## What This Changes

The issue requests bounding session-store growth by pruning ephemeral sessions, deduplicating or externalizing repeated skill/system-prompt payloads, cleaning orphan artifacts, and reducing gateway OOM risk.

## Best Possible Solution

Keep this as the canonical session-store bloat tracker, review or land the focused prompt-stripping fix, then follow with a backward-compatible design for static snapshot/report storage, safe orphan artifact cleanup, and heap-proofed memory fixes.

## Reproduction Assessment

Yes at source level for retained-row bloat: current main still persists `skillsSnapshot.prompt` and `systemPromptReport` while only stripping `resolvedSkills`. I did not reproduce the long-uptime RSS slope live, so the exact OOM mechanism remains partly unproven.

## Solution Assessment

No complete fix is on main yet. The existing retention, entry caps, cron reaper, and `resolvedSkills` stripping are useful mitigations, while the durable fix needs prompt/report externalization or stripping, compatibility-safe rebuild semantics, orphan cleanup policy, and heap proof.
