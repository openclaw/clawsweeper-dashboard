## Summary

Keep this issue open. Current main has useful retention and cleanup mitigations, and it strips `resolvedSkills`, but retained session rows still persist large `skillsSnapshot.prompt` and `systemPromptReport` payloads inline; the latest 2026.5.7 production comment reports the same structural duplication after cleanup.

## What This Changes

The issue asks OpenClaw to stop unbounded `sessions.json` growth by pruning ephemeral sessions, cleaning orphan artifacts, and deduplicating or externalizing repeated skill/system-prompt payloads.

## Best Possible Solution

Keep this as the canonical tracker and land a backward-compatible persistence design that keeps `sessions.json` metadata-light, removes repeated rendered skill/system-prompt payloads from per-session rows, compacts legacy rows, and preserves current retention and cleanup safeguards.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: create or retain many session entries with `skillsSnapshot.prompt` and `systemPromptReport`, then trigger session save/load/list paths that stringify or parse the full store. I did not run a live heap profile in this read-only review.

## Solution Assessment

Yes directionally: the durable store should stop carrying repeated rendered prompt payloads per row. Retention and pruning are necessary mitigations, but the narrow maintainable fix is to strip, dedupe, externalize, or lazily rebuild static prompt payloads with compatibility coverage.
