## Summary

Keep open: current main has real mitigations for the largest persisted `skillsSnapshot.prompt` bloat, but the reported gateway OOM/session-store pressure is not fully resolved because whole-store hydration/cache retention, `systemPromptReport` persistence, disk-budget cleanup defaults, and orphan/archive cleanup policy still leave meaningful work open.

## What This Changes

The issue asks OpenClaw to stop `sessions.json` from growing into gateway OOMs by pruning ephemeral sessions, deduplicating skill/report metadata, cleaning orphan artifacts, and avoiding full-store memory pressure.

## Best Possible Solution

Keep this broad tracker open while maintainers land or revise the focused prompt metadata, cache-retention, and artifact-cleanup paths, then close only after current main or a release has representative proof that session-store size and gateway RSS stay bounded.

## Reproduction Assessment

Yes at source level, but not as a full live OOM reproduction in this read-only pass. Current main still has whole-store hydration/cache paths and persisted `systemPromptReport`, while the issue and comments provide multiple field reports of OOM, lock latency, and artifact buildup.

## Solution Assessment

No final single fix is confirmed yet. The prompt-blob mitigation is the right direction for the largest duplicated payload, but the remaining cache-retention, cleanup-default, and deletion-policy pieces need maintainer-guided convergence and soak proof.
