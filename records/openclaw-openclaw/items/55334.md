## Summary

Keep open: current main and v2026.6.10 have important mitigations for duplicated `skillsSnapshot.prompt`, but the broader gateway OOM/session-store pressure remains open through `systemPromptReport` persistence, whole-store hydration/cache retention, transcript/artifact cleanup, and retention-policy decisions.

## What This Changes

The issue reports gateway memory and availability failures from persistent session-store growth, duplicated per-session snapshots/reports, and incomplete pruning of ephemeral sessions and orphan artifacts.

## Best Possible Solution

Keep this broad tracker open until maintainers land or reject the focused cache, prompt-report, transcript, and artifact-cleanup paths, then close only with current-main or release proof that representative long-running session stores and gateway RSS stay bounded.

## Reproduction Assessment

Partly. Current source still shows durable `systemPromptReport` writes and unbounded parsed object/snapshot cache retention, and the discussion includes fresh v2026.6.10 production evidence, but I did not run a live current-main OOM soak.

## Solution Assessment

No, not as a single completed fix. Prompt-blob externalization is a good partial mitigation, but the best fix still needs maintainer-approved cache caps, prompt-report compaction policy, artifact cleanup, and retention semantics.
