## Summary

Keep open: current main and v2026.6.10 mitigate the largest duplicated skill-prompt payload and enforce session-entry caps, but the broader gateway OOM/session-store pressure remains open across runtime cache retention, durable systemPromptReport data, optional disk-budget cleanup, transcript/artifact cleanup, and missing soak proof.

## What This Changes

The issue asks OpenClaw to bound sessions.json growth by pruning ephemeral sessions, deduplicating per-session skills snapshots, cleaning orphan transcripts, and avoiding full session-store memory pressure.

## Best Possible Solution

Keep this broad tracker open until maintainers land or reject the focused cache, prompt/report metadata, transcript, and artifact-cleanup paths, then close only with current-main or release proof that representative long-running session stores and gateway RSS stay bounded.

## Reproduction Assessment

Yes at source level, but not as a full live OOM reproduction in this read-only pass. Current main still has whole-store hydration/cache paths, durable systemPromptReport storage, and opt-in disk-budget cleanup while the issue discussion supplies repeated field reports.

## Solution Assessment

Unclear as a final fix. Prompt-blob externalization is the right mitigation for the largest duplicated prompt payload, but cache-retention limits, cleanup defaults, transcript/artifact policy, and soak proof still need maintainer-guided convergence.
