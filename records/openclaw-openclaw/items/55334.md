## Summary

Keep this open. Current main and v2026.5.18 include useful mitigations, but they still persist `skillsSnapshot.prompt` and `systemPromptReport` inline in session rows; https://github.com/openclaw/openclaw/pull/78259 is the active partial fix for prompt stripping, not merged yet, and the newer comments raise additional heap/orphan-artifact questions.

## What This Changes

The issue requests fixing unbounded session-store growth and gateway OOMs caused by duplicated skill/system prompt payloads and incomplete pruning of ephemeral session artifacts.

## Best Possible Solution

Keep this as the canonical tracker, review and land a correct prompt-stripping change, then finish a backward-compatible storage/compaction design for remaining static payloads and orphan artifacts using heap evidence to avoid mistaking disk bloat for the whole memory leak.

## Reproduction Assessment

Yes at source level for retained-row bloat: current main still writes `skillsSnapshot.prompt` and `systemPromptReport` into session entries while tests only guarantee `resolvedSkills` is stripped. I did not run a live long-uptime heap reproduction, so the full 50-100 MB/min RSS curve remains unproven here.

## Solution Assessment

No complete fix is on main yet. The current pruning/capping and `resolvedSkills` stripping are good mitigations, but the best remaining solution needs maintainer-approved schema, migration, resume, and artifact-cleanup decisions rather than a separate blind cleanup script.
