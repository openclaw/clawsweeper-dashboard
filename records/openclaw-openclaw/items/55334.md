## Summary

Keep open: current main and v2026.5.18 still retain inline `skillsSnapshot.prompt` and `systemPromptReport` in session rows, while recent production comments and the open prompt-stripping PR show the central bloat/OOM path is only partially mitigated.

## What This Changes

The issue asks OpenClaw to prevent session-store bloat and gateway OOMs by pruning ephemeral sessions, avoiding duplicated static skill/prompt payloads, cleaning orphan artifacts, and reducing whole-store materialization.

## Best Possible Solution

Keep this as the canonical session-store bloat tracker, land a correct prompt-stripping or reference-based fix for static skill payloads, then design backward-compatible compaction for `systemPromptReport`, orphan artifacts, and whole-store materialization with heap/soak proof.
