## Summary

Keep this issue open. Current main and `v2026.5.18` include useful retention/capping and `resolvedSkills` stripping, but retained session rows still carry `skillsSnapshot.prompt` and `systemPromptReport` inline, and the focused prompt-stripping implementation at https://github.com/openclaw/openclaw/pull/78259 is still open.

## What This Changes

The issue requests bounded session-store growth by pruning ephemeral sessions, avoiding repeated `skillsSnapshot` and `systemPromptReport` payloads, and cleaning orphan session artifacts.

## Best Possible Solution

Keep this issue as the canonical session-store bloat tracker, review and land or replace https://github.com/openclaw/openclaw/pull/78259 for prompt stripping, then decide the backward-compatible schema/compaction path for any remaining report payload and artifact cleanup.
