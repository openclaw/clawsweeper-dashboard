## Summary

Keep this issue open. Current main and v2026.5.18 have useful pruning and resolvedSkills stripping, but retained session rows still persist skillsSnapshot.prompt and systemPromptReport inline, and the focused prompt-stripping implementation at https://github.com/openclaw/openclaw/pull/78259 is still open.

## What This Changes

The issue requests bounded session metadata storage by pruning ephemeral sessions, deduplicating or externalizing large skill/system-prompt payloads, and cleaning orphan transcript artifacts.

## Best Possible Solution

Keep this as the broad session-store bloat tracker, land or replace https://github.com/openclaw/openclaw/pull/78259 for prompt stripping, then decide the backward-compatible deduplication or out-of-line storage path for remaining static prompt/report payloads and paired transcript cleanup.

## Reproduction Assessment

Yes, at source level: current main requires and preserves SessionSkillSnapshot.prompt, writes skillsSnapshot/systemPromptReport into session rows, and reads/stringifies the whole sessions.json store. I did not run a live heap/OOM scenario in this read-only review.

## Solution Assessment

Yes, keeping this issue open is the best current handling because the central retained-row payload duplication is not fixed on main or in v2026.5.18. The narrow next step is the open prompt-stripping PR or a compatible replacement, followed by maintainer design for any residual report and artifact cleanup.
