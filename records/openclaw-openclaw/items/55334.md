## Summary

Keep this open: current main has useful retention and resolvedSkills mitigations, but the durable session schema still stores rendered skill prompts and system prompt reports inline, and the focused prompt-stripping PR is still open rather than merged.

## What This Changes

Fix sessions.json growth by pruning ephemeral sessions, deduplicating persisted skill/prompt metadata, cleaning orphan transcripts, and reducing full-store memory pressure.

## Best Possible Solution

Keep this as the canonical session-store bloat tracker, land a correct prompt-stripping fix, then follow with maintainer-approved schema/externalization, orphan-artifact cleanup, and heap-proof work for any remaining RSS growth.

## Solution Assessment

No complete fix is on main yet. The current retention, capping, disk-budget, and resolvedSkills stripping work are useful mitigations, but the remaining solution needs maintainer-approved persisted schema, resume semantics, cleanup defaults, and heap-proof sequencing.
