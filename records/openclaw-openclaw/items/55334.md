## Summary

Keep this open. Current main has useful pruning and maintenance mitigations, but retained session rows can still persist `skillsSnapshot.prompt` and `systemPromptReport` inline, and a fresh v2026.5.7 production report shows the same structural duplication after cleanup.

## What This Changes

The issue asks OpenClaw to prevent gateway OOM and lock latency by pruning ephemeral sessions, cleaning orphan session artifacts, and deduplicating large static session payloads such as `skillsSnapshot`.

## Best Possible Solution

Keep this as the canonical tracker and land a backward-compatible session-store design that keeps retained rows lightweight, stores or reconstructs large skill/system-prompt payloads by reference or hash, compacts legacy rows, and preserves current cleanup and resume semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: retain many sessions with `skillsSnapshot.prompt`, then save, load, list, clean up, or run cron paths that stringify or parse the whole store. I did not run a live heap profile because this review is read-only.

## Solution Assessment

Unclear as a single patch. Stripping or deduplicating the top-level snapshot is the right direction, but the final fix should be schema-aware and preserve nested plugin data, prompt-cache stability, warning dedupe, and cold-resume behavior.
