## Summary

Keep this issue open. Current main and the latest release still persist `skillsSnapshot.prompt` and `systemPromptReport` inline on session entries, and the direct implementation candidate at https://github.com/openclaw/openclaw/pull/78259 is still open rather than shipped.

## What This Changes

The issue asks OpenClaw to stop `sessions.json` growth from duplicated skill/system-prompt payloads, stale ephemeral sessions, orphan transcripts, and whole-store session serialization.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78259 as the prompt-stripping step, then keep any remaining `systemPromptReport`, orphan cleanup, or whole-store loading work tracked explicitly under this canonical session-store bloat thread.

## Reproduction Assessment

Yes for the storage-bloat path: current source still stores `skillsSnapshot.prompt` and `systemPromptReport` per retained session row and serializes the full store. I did not run a live heap/OOM reproduction in this read-only review.

## Solution Assessment

Yes, keeping this open is the best current handling because the core behavior is not fixed on main or in `v2026.5.18`. The best implementation path is the open prompt-stripping PR or a compatible replacement, with follow-up for any remaining static payload and cleanup gaps.
