## Summary

Keep this open. Current main has important mitigations for session-store growth, but the retained-row bloat is still source-reproducible because `skillsSnapshot.prompt` and `systemPromptReport` are still persisted inline, and the related prompt-stripping fix remains an open PR rather than shipped code.

## What This Changes

The issue requests bounded session-store growth by pruning ephemeral sessions, avoiding duplicated per-session skill and system-prompt payloads, and cleaning orphan session artifacts.

## Best Possible Solution

Keep this as the canonical session-store bloat tracker, land or replace the focused `skillsSnapshot.prompt` storage fix, then decide whether `systemPromptReport`, orphan transcript cleanup defaults, and the remaining heap-growth diagnosis need separate narrower follow-ups.

## Reproduction Assessment

Yes, at source level: current main still persists `skillsSnapshot.prompt` and `systemPromptReport` per retained session row, and the focused test suite intentionally preserves `skillsSnapshot.prompt`. I did not live-reproduce the exact 50-100 MB/min RSS slope.

## Solution Assessment

No complete fix is present yet. Existing pruning/capping and `resolvedSkills` stripping are useful mitigations, while the remaining fix needs a maintainer-approved storage/replay design plus review of the open prompt-stripping PR.
