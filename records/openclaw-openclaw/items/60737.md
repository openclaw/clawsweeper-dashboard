## Summary

Keep open. Current main still supports explicit Telegram topic `agentId`, static group-topic ACP bindings, and manual/runtime ACP binding flows, but it does not expose a per-DM/per-chat default ACP config or auto-spawn ACP path for newly created Telegram topics.

## What This Changes

Request a Telegram config surface that auto-spawns and binds a default ACP session for new DM or group forum topics, with explicit per-topic routing taking precedence.

## Best Possible Solution

Design a first-class, opt-in Telegram default ACP binding surface with clear precedence, authorization, allowed-agent/cwd policy, and per-topic session isolation, then implement it through the existing binding/spawn seams with docs, changelog, and focused Telegram/ACP coverage.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from source inspection: strict Telegram schemas reject the requested field, and inbound routing has no branch that materializes a default ACP binding for an unbound new topic.

## Solution Assessment

Unclear until maintainer product/security review. A defaultAcp-style config is plausible, but the safest solution needs explicit opt-in policy, precedence rules, and ACP isolation decisions before implementation.
