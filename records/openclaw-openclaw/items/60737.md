## Summary

Keep open. Current main and the latest release still lack the requested Telegram per-DM/per-chat default ACP binding, and the related broader preset and ACP safety-policy issues remain open rather than superseding this Telegram-specific config and precedence request.

## What This Changes

The issue requests a per-DM or per-chat Telegram config option that auto-spawns and binds an ACP session for new Telegram topics or threads.

## Best Possible Solution

Design an explicit opt-in default ACP binding policy aligned with the broader thread-binding preset work, including precedence rules, allowed agents, cwd constraints, per-topic isolation, docs, changelog, and focused Telegram/ACP coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the surface is absent because Telegram config/schema has no defaultAcp field and routing only uses explicit topic agent IDs plus existing configured/runtime bindings.

## Solution Assessment

Unclear until maintainer product/security review. A defaultAcp-style option is plausible, but the maintainable solution should be an explicit opt-in policy aligned with broader default thread-binding preset and ACP spawn-safety work.
