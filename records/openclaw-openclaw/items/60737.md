## Summary

Keep open. Current main and v2026.5.22 still do not provide a Telegram per-DM/per-chat default ACP config or first-message auto-spawn path, and the related broader default thread-binding preset issue remains open rather than replacing this Telegram-specific precedence and security design work.

## What This Changes

The issue requests a Telegram `defaultAcp`-style config that automatically spawns and binds an ACP session for new DM or group topics, with explicit per-topic `agentId` config taking precedence.

## Best Possible Solution

Design an explicit opt-in Telegram default ACP binding policy aligned with the broader thread-binding preset work, including precedence rules, allowed agents, cwd constraints, per-topic isolation, docs, and focused Telegram/ACP coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the surface is absent because Telegram config/schema has no `defaultAcp` field and routing only uses explicit topic agent IDs plus configured/runtime bindings.

## Solution Assessment

Unclear until maintainer product/security review. A `defaultAcp`-style option is plausible, but the maintainable path should be explicit opt-in policy aligned with broader default thread-binding preset and ACP spawn-safety work.
