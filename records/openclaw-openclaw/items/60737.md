## Summary

Keep open. Current main still lacks a per-DM or per-chat Telegram `defaultAcp` config and has no routing path that auto-spawns ACP for a new unbound topic; this is a feature/config request with ACP security and isolation design implications, not an implemented or stale cleanup item.

## What This Changes

Request to add a Telegram per-DM/per-chat `defaultAcp` binding that automatically spawns and binds ACP sessions for new topics.

## Best Possible Solution

Design an explicit opt-in Telegram default ACP binding surface with precedence rules, allowed-agent and cwd policy, per-topic isolation, docs, changelog, and focused Telegram/ACP regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested surface is absent: `defaultAcp` has no current-main references, Telegram schemas are strict, and routing only covers explicit topic `agentId`, configured bindings, or existing runtime bindings.

## Solution Assessment

Unclear until maintainer product/security review. A `defaultAcp`-style option is plausible, but the maintainable solution needs an explicit opt-in policy and may need to align with the broader thread-binding preset design.
