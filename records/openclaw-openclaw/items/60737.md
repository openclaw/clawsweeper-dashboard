## Summary

Keep open. Current main still lacks a per-DM or per-chat Telegram default ACP binding, and the closest broader preset proposal remains open rather than superseding this Telegram-specific config and precedence request.

## What This Changes

Add a Telegram config surface that auto-spawns and binds ACP sessions for new DM or group topics, with explicit per-topic config taking precedence.

## Best Possible Solution

Design an explicit opt-in default ACP binding policy aligned with the broader thread-binding preset work, including precedence rules, allowed agents, cwd constraints, per-topic isolation, docs, changelog, and focused Telegram/ACP coverage.

## Solution Assessment

Unclear until maintainer product/security review. A defaultAcp-style option is plausible, but the maintainable solution should be an explicit opt-in policy aligned with broader default thread-binding preset work.
