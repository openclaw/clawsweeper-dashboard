## Summary

Keep open. Current main still does not provide a per-DM or per-chat Telegram `defaultAcp` config, and the existing Telegram ACP paths cover explicit topic `agentId`, configured group-topic ACP bindings, or manual/runtime `/acp spawn --thread` binding only.

## What This Changes

Request a Telegram config surface that auto-spawns and binds a default ACP session for new DM or group forum topics, with explicit per-topic routing taking precedence.

## Best Possible Solution

Design a first-class, opt-in Telegram default ACP binding surface with clear precedence, authorization, allowed-agent and cwd policy, and per-topic session isolation, then implement it through existing binding/spawn seams with docs, changelog, and focused Telegram/ACP coverage.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from source inspection: strict Telegram schemas reject the requested field, and inbound routing has no branch that materializes a default ACP binding for an unbound topic.

## Solution Assessment

Unclear until maintainer product/security review. A `defaultAcp`-style config is plausible, but the safest solution needs explicit opt-in policy, precedence rules, and ACP isolation decisions before implementation.
