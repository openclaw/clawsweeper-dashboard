## Summary

Keep open. Current main still lacks a per-DM or per-chat Telegram default ACP config and has no routing path that auto-spawns ACP for a new unbound topic; this remains a security-sensitive feature/config design item.

## What This Changes

Requests a Telegram per-DM/per-chat default ACP binding that automatically spawns and binds an ACP session for new DM or group topics when no explicit topic binding exists.

## Best Possible Solution

Design an explicit opt-in Telegram default ACP binding surface with precedence rules, allowed-agent and cwd policy, per-topic isolation, docs, changelog, and focused Telegram/ACP regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested surface is absent: the config schemas are strict, defaultAcp has no exact matches, and routing only uses explicit topic agentId plus existing configured/runtime bindings.

## Solution Assessment

Unclear until maintainer product/security review. A defaultAcp-style option is plausible, but the maintainable solution needs an explicit opt-in policy and should align with the broader thread-binding preset work.
