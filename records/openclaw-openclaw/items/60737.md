## Summary

Keep open. Current main still lacks the requested per-DM or per-chat Telegram default ACP binding, and the closest related proposal remains open rather than merged or terminally superseding this Telegram-specific request. The feature needs maintainer product/security design because ordinary chat activity would be able to start host-side ACP sessions.

## What This Changes

The issue requests a Telegram direct/group config option that automatically spawns and binds a default ACP session for new unbound topics or threads.

## Best Possible Solution

Design an explicit opt-in default ACP binding surface that aligns with the broader thread-binding preset work, with precedence rules, allowed-agent and cwd policy, per-topic isolation, docs, changelog, and focused Telegram/ACP coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested surface is absent because Telegram config and schemas have no default ACP field and routing only uses explicit topic agent IDs plus existing bindings.

## Solution Assessment

Unclear until maintainer product/security review. A defaultAcp-style option is plausible, but the maintainable solution should be an explicit opt-in policy aligned with the broader default thread-binding preset work.
