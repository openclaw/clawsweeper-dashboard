Full review comments:

- **[P1] Keep reset fallback tied to the requested session:** `src/hooks/bundled/session-memory/transcript.ts:167-172`
  - body: After the requested session's active, canonical reset, and topic reset candidates fail, this falls back to the newest `.jsonl.reset.` file anywhere in the directory. If another session has the newest reset archive, the hook can summarize that conversation into the current session's memory; remove the directory-wide fallback or require a matching session header/id.
  - confidence: 0.9
- **[P1] Gate deleted transcript recall separately:** `skills/session-logs/SKILL.md:78-80`
  - body: The skill now treats `.jsonl.deleted.*Z` as part of normal full recall. `sessions.delete` removes the session from listings while archiving its transcript, so an agent following this helper can resurface deleted conversation content; exclude deleted archives by default or make this a separate explicit user/maintainer-approved path.
  - confidence: 0.86
- **[P3] Handle unset nullglob under set -e:** `skills/session-logs/SKILL.md:76`
  - body: When `nullglob` is unset, `shopt -p nullglob` can return a non-zero status after printing the restore command, so a copied helper running under `set -e` exits before doing any search. Capture it with a non-failing fallback, such as `shopt -p nullglob || true`, before restoring.
  - confidence: 0.78
