## Summary

Keep open: this is a plausible fix for a real archived-transcript gap, but the current PR head still has a live-topic priority regression and a copied shell helper failure mode, and the external real-behavior proof gate is not satisfied.

## What This Changes

The PR adds reset-archive discovery for the bundled session-memory hook, expands session-logs skill examples for reset/deleted archives, adds hook tests, and includes a lockfile importer update.

## Best Possible Solution

Land this PR or an equivalent replacement after live `.jsonl` candidates keep priority over archive fallbacks, the shell helper is robust under `set -e`, stale lockfile churn is rebased away, and redacted real reset/session-log proof is added.

## Reproduction Assessment

Yes, at source level. On current main, a sessions directory with only `<sessionId>.jsonl.reset.*Z` and no live transcript makes `findPreviousSessionFile` return `undefined`, and the current session-logs examples search only active `*.jsonl` files.

## Solution Assessment

No, not as currently ordered. Hook-local archive recovery and session-logs documentation are the right scope, but archive fallbacks should run after all live transcript candidates and the documented shell helper should not fail under `set -e`.
