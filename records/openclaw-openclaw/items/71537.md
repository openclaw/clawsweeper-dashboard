## Summary

Keep open: current main still misses the session-memory reset-only archive discovery and the session-logs skill still documents active `*.jsonl` searches only, so the PR is not obsolete. The branch is not merge-ready because it can choose a reset archive before a live topic transcript, the documented helper exits under `set -e`, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR adds reset-archive fallback discovery for bundled session-memory, tests it, updates the session-logs skill examples to include reset/deleted archives, and syncs a lockfile importer entry.

## Best Possible Solution

Land this PR or an equivalent replacement after live transcript choices retain priority over archive fallbacks, the session-logs helper is robust under `set -e`, stale lockfile churn is rebased away, and redacted real reset-session proof is added.

## Reproduction Assessment

Yes, at source level. Current main returns no previous session file when only reset archives remain and `currentSessionFile` is absent, and the current session-logs examples only search active `*.jsonl`; the PR regression follows from the new reset-archive branch running before live topic variants.

## Solution Assessment

No, not as currently ordered. Hook-local archive recovery and session-logs documentation are the right scope, but archive fallbacks should run after all live transcript candidates and the documented shell helper should not fail under `set -e`.
