## Summary

Keep this PR open: the reported hook and skill gaps still exist on current main, but the branch is not merge-ready because it can prefer an archived canonical transcript over a live topic transcript and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds reset-archive fallback discovery for the bundled session-memory hook, regression tests, session-logs skill guidance for active/reset/deleted transcript searches, and a stale lockfile importer hunk.

## Best Possible Solution

Land this PR or an equivalent replacement after live `.jsonl` candidates keep priority over archive fallbacks, the shell helper is robust under `set -e`, stale lockfile churn is rebased away, and redacted real reset/session-log proof is added.

## Reproduction Assessment

Yes, at source level. Current main can be checked with a sessions directory containing only `<sessionId>.jsonl.reset.*Z`: `findPreviousSessionFile` has no reset-only path, and the session-logs skill examples search only active `*.jsonl` files.

## Solution Assessment

No, not as currently ordered. Hook-local archive recovery and session-logs documentation are the right scope, but reset archive fallbacks should run after all live transcript candidates and the shell helper should survive `set -e`.
