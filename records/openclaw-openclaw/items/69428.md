## Summary

Keep this PR open. Current main still has the one-page MS Teams reply helper, so the underlying bug is real, but the proposed patch has merge-blocking issues: it imports a type that current main no longer exports, and its recent-reply window depends on unverified Graph ordering despite official docs not supporting an order query. The earlier partial-result review concern appears addressed by the later PR commit and regression test.

## What This Changes

The PR changes the MS Teams Graph thread helper to follow `@odata.nextLink` for replies under a 10-page cap, preserve fetched replies after later-page failures, return a bounded recent reply window, and add helper-level unit tests.

## Best Possible Solution

Keep the fix in the MS Teams plugin, but make it current-main compatible and order-safe: follow bounded pagination, preserve partial pages after later failures, select the newest replies by `createdDateTime` with a stable fallback, and add the required changelog entry.

## Reproduction Assessment

Yes. The helper-level bug is reproducible with mocked first-page and next-page Graph responses: current main drops the next page, while the PR exercises pagination; the compile/import regression is also statically reproducible from the current main export surface.

## Solution Assessment

No. The pagination boundary and partial-failure behavior are the right shape, but the current patch is not the best mergeable solution until it fixes the non-exported type import and avoids assuming Graph page order.
