## Label Justifications

- `P1`: The report describes an active QMD query-mode memory-search workflow becoming unusable for Apple Silicon users who opt into hybrid QMD search.
- `impact:crash-loop`: The affected path is triggered by a native QMD/node-llama-cpp Metal subprocess abort.
- `impact:session-state`: Discarding valid QMD search results can remove memory recall context and leave the builtin fallback with no indexed state for that backend.

## Summary

Keep open: current main still rejects a non-zero direct QMD subprocess before parsing captured stdout, so valid query results can be lost and the builtin fallback path can return empty memory results.

## What This Changes

The issue asks OpenClaw to preserve valid QMD query JSON emitted before a macOS Apple Silicon Metal cleanup crash instead of discarding it because the subprocess exits non-zero.

## Best Possible Solution

Accept well-formed JSON from direct QMD search commands that fail after emitting complete stdout, while keeping non-search QMD commands and invalid/truncated outputs fail-closed and leaving broader Apple Silicon Metal policy in the related tracker.

## Reproduction Assessment

Yes at the source level: a mocked QMD child that writes valid JSON to stdout and then closes non-zero exercises the current reject-before-parse path. I did not run a live macOS Apple Silicon QMD repro in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to salvage validated direct-QMD search JSON on non-zero exits, not to change defaults or add a CPU-only config path as the first repair.

## Repair Work Prompt

Fix the QMD direct search result-discard bug reported in https://github.com/openclaw/openclaw/issues/85217. Current main rejects a direct `qmd query/search/vsearch --json` subprocess before parsing stdout when the process exits non-zero after emitting valid JSON, so `FallbackMemoryManager` can switch to builtin search and return empty results. Implement a narrow search-only salvage path that accepts well-formed QMD search JSON emitted before a non-zero direct search exit, while still failing invalid stdout, output truncation, timeouts, spawn errors, and non-search maintenance commands such as update/embed/collection. Keep mcporter behavior separate unless the same direct helper contract is intentionally extended. Add focused regression tests with a mocked child process closing non-zero after valid JSON, plus invalid-output and non-search failure coverage. Do not add a new CPU-only config option or change QMD defaults in this repair. Add a changelog entry because this fixes user-visible memory search behavior.
