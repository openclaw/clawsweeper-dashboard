## Summary

Keep open: current main hardens the initial inter-session injection, but normal A2A announce delivery can still post non-control text through the visible channel send path, and current tests assert that behavior.

## What This Changes

Request that inter-agent `sessions_send` traffic reach the target agent context without rendering in user-facing Slack threads or channels.

## Best Possible Solution

Define an explicit A2A visibility contract, then update the tool/RPC schema, docs, and regression tests so internal coordination reaches target transcripts without surface delivery unless maintainers intentionally preserve visible announces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `sessions_send` to a visible channel session that produces non-control announce text reaches `runSessionsSendA2AFlow`, which calls gateway `send`; an existing regression test asserts that visible send.

## Solution Assessment

No. The current implementation relies on model compliance with `ANNOUNCE_SKIP` and still tests visible announce delivery, so the narrow maintainable solution needs an explicit routing/API contract chosen by maintainers.
