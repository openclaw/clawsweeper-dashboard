## Summary

Keep open: current main fixed the thread-session subset, but normal A2A announce delivery can still call gateway `send` for non-skip replies, and current docs/tests preserve that visible delivery behavior.

## What This Changes

Request that inter-agent `sessions_send` traffic reach the target agent context without posting internal coordination replies into Slack threads or channels.

## Best Possible Solution

Define an enforced internal-only A2A routing contract, or an explicit silent/internal flag if maintainers prefer opt-in behavior, then update the tool schema, protocol/docs, and regression tests so Slack-like surfaces cannot receive coordination text while target transcripts still do.

## Reproduction Assessment

Yes. Source and tests give a high-confidence path: make the A2A announce step return any non-control text, and `runSessionsSendA2AFlow` calls gateway `send` for the resolved visible channel target.

## Solution Assessment

No. The current implementation relies on model compliance with `ANNOUNCE_SKIP` and still documents/tests visible announce delivery; the maintainable fix is an explicit routing/API contract chosen by maintainers.
