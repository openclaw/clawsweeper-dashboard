## Summary

Keep open: current main still has global-only plugin install/uninstall, flat plugin-id install records, and default-workspace plugin activation; the requested per-agent lifecycle remains a real feature with unresolved product and security boundaries.

## What This Changes

Request support for installing and uninstalling plugins per agent, including agent-scoped install directories and independent install records for the same plugin id.

## Best Possible Solution

Define a maintainer-approved per-agent plugin installation contract covering record identity, workspace/global precedence, runtime activation, update/uninstall cleanup, doctor migration, docs, tests, and security policy before implementation.

## Reproduction Assessment

Yes, at source level: current main has no --agent flag for plugins install/uninstall, persists installs by flat plugin id, and uninstalls from the global extensions root.

## Solution Assessment

Unclear. The --agent CLI shape is plausible, but the best fix needs maintainer decisions on install identity, discovery precedence, security policy, update/uninstall isolation, and upgrade behavior.
