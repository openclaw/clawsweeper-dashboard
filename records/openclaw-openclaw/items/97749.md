## Summary

Keep open: current main and v2026.6.10 still manage plugin installs globally, with no `--agent` install/uninstall flag and flat `pluginId` install records. The request is a valid multi-agent isolation feature, but it needs maintainer product/security design before implementation.

## What This Changes

Add `--agent <id>` to plugin install/uninstall so managed plugin records, versions, install directories, and uninstall behavior can be scoped per agent.

## Best Possible Solution

Define a maintainer-approved per-agent plugin installation contract covering record identity, workspace/global precedence, update/uninstall cleanup, discovery/restart behavior, docs, security policy, and upgrade/doctor semantics.

## Reproduction Assessment

Yes, at source level: current main lacks the requested `--agent` CLI flags, installs into the global extensions directory, and persists/removes records by flat `pluginId`.

## Solution Assessment

Unclear. The `--agent` CLI shape is plausible, but the best fix needs maintainer decisions on install identity, discovery precedence, security policy, update/uninstall isolation, and upgrade behavior.
