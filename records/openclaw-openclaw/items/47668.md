## Summary

Keep open. Current main has useful adjacent controls for compaction, tool-result pruning, and context-engine plugins, but it does not fully solve the reported long-session context growth, especially for the reported Ollama/MiniMax path. The related PR #70605 is still open and targets only the strikethrough-context-file slice, so the issue should remain open while maintainers decide the broader default policy and UX.

## What This Changes

The issue requests smoother long-session context handling so users do not need frequent `/new` resets after roughly 80-100 messages.

## Best Possible Solution

The maintainable end state is a measured core policy that keeps long sessions smooth across providers: sanitize noisy bootstrap/context inputs, make pruning and compaction behavior visible and tunable, and add safer defaults only where tests show useful context is preserved.
