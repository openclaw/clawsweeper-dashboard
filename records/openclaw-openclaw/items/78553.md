## Summary

Current main still has the top Skills page default-scoped, while the backend and Agents page already support per-agent skill status; this is a valid UI scope gap, not a closeable duplicate or implemented item.

## What This Changes

Add multi-agent selection to the Web UI Skills page so users can view workspace skills for non-default configured agents.

## Best Possible Solution

Define whether the top Skills page should become agent-scoped or whether the Agents > Skills panel is canonical; if it becomes agent-scoped, status, install, update, and messaging should all use consistent selected-agent semantics.

## Reproduction Assessment

Yes by source inspection: the top Skills page calls `skills.status` with `{}`, which the server resolves to the default agent, while the agent-aware path passes `agentId`. A live UI reproduction was not run in this read-only review.

## Solution Assessment

Unclear: adding `agentId` to the status request is necessary for the requested view, but it is not sufficient while install/update actions remain default/global-scoped. The maintainable fix should decide and implement the whole page's agent scope consistently.
