## What This Changes

The issue reports that `agents.defaults.compaction.model` is ignored during safeguard compaction, causing compaction to use the active session model instead of the configured override.

## Best Possible Solution

Route the remaining work through https://github.com/openclaw/openclaw/issues/57901 and review or land https://github.com/openclaw/openclaw/pull/73704 once proof and checks are acceptable.

## Reproduction Assessment

Yes. Source inspection plus the reporter logs give a high-confidence source reproduction: with safeguard compaction, current main can prefer the session model over the configured compaction runtime model.

## Solution Assessment

Yes. The narrow maintainable path is the canonical safeguard fix, which resolves the configured compaction target before runtime registration and makes the hook prefer that runtime model.
