## Summary

## What This Changes

The issue asks for channel login to handle a missing `channels.<name>` block with an actionable preflight/error instead of saving auth and later reporting `invalid channels.start channel`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83081 so explicit channel login fails before saving auth when the channel block is missing or disabled, and gateway start returns matching actionable guidance.

## Reproduction Assessment

Yes. Current main still performs plugin login before gateway reconciliation and still has the generic `invalid channels.start channel` branch when the running gateway cannot normalize the channel id.

## Solution Assessment

Yes. The fail-before-save preflight direction is the narrowest maintainable fix; auto-writing a minimal config block would be a broader setup behavior change and is not needed to fix the misleading error path.
