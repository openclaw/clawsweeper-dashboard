## Summary

Keep open: current main and the latest release still allow channel login to save auth before gateway reconciliation and can still surface the generic invalid-channel response; https://github.com/openclaw/openclaw/pull/83081 is the open closing PR that owns the repair but has not merged.

## What This Changes

The issue asks for channels login to give clear missing-config guidance and avoid saving auth before a missing `channels.<name>` block is handled.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83081 so explicit channel login/start fails before auth or runtime side effects when the authored channel block is missing or disabled.

## Reproduction Assessment

Yes, at source level: current main still calls plugin login before gateway reconciliation and can still log the generic invalid-channel gateway error when channel registry normalization fails. I did not run the live auth flow because this read-only review must not mutate auth or config state.

## Solution Assessment

Yes: the fail-before-save/start preflight direction in the linked maintainer PR is the narrowest maintainable fix. Auto-writing channel config during auth would be broader setup behavior and could surprise existing operators.
