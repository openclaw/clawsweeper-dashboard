## What This Changes

The issue asks channel login to give actionable missing-config guidance or fail before saving auth when the requested channel config block is absent or disabled.

## Best Possible Solution

Land or replace the linked channel/config repair so explicit channel login fails before saving auth when the channel block is missing or disabled, and gateway start returns the same actionable guidance.

## Reproduction Assessment

Yes, at source level: current main still runs provider login before gateway reconciliation, and the gateway can still answer `invalid channels.start channel` before checking channel config state.

## Solution Assessment

Yes. The fail-before-save preflight direction is the narrowest maintainable fix; auto-writing a minimal config block would be a broader setup behavior change and is not needed to fix the misleading error path.
