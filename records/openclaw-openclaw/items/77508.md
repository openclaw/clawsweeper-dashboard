## What This Changes

The issue asks for channel login/start to give actionable missing-config guidance, preferably failing before auth save when `channels.<name>` is absent or disabled.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main still runs provider login before gateway reconciliation, and `channels.start` can still return the opaque `invalid channels.start channel` path before config-block guidance. I did not run live login because it can mutate local auth state.

## Solution Assessment

Yes: failing explicit login before provider auth/save and sharing the same actionable preflight in `channels.start` is the narrow maintainable fix. Auto-writing `channels.<name>` would be a broader behavior change and is not needed to fix the misleading error path.
