## Summary

Keep open for maintainer review: the patch looks correct for the bounded QQBot error-propagation problem, with sufficient redacted runtime proof, but it intentionally changes shared QQBot text/media fail-fast semantics.

## What This Changes

The PR changes QQBot message result normalization so outbound errors and empty platform message ids reject, and adds adapter regression tests for failed media sends.

## Best Possible Solution

Land the bounded adapter error-propagation fix after maintainer acceptance of shared QQBot text/media fail-fast semantics, while keeping the broader QQ C2C media delivery work on the canonical issue.

## Reproduction Assessment

Yes at source level: current main stores QQBot lower-layer errors in `meta` and can return an empty successful receipt when no platform id exists. I did not run a credentialed live QQBot send in this read-only review, but the PR body includes a redacted credentialed failure-path log.

## Solution Assessment

Yes for the bounded error-propagation problem: the adapter normalizer is the narrowest maintainable layer to reject lower-level send errors and empty receipts. It is not the full solution for QQ C2C media delivery, which remains tracked separately at https://github.com/openclaw/openclaw/issues/92246.
