## Summary

Keep open: the intended media fix is still useful, but this branch is not mergeable as-is because it edits deleted `src/web/*` files, lacks after-fix real behavior proof, and its PNG fast path would bypass image decode validation.

## What This Changes

The branch changes the legacy web media loader/tests to keep under-cap PNGs as PNG and rename JPEG-converted outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, and active shared/WhatsApp regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main's active shared loader still routes non-alpha PNG optimization through the JPEG path, and the active WhatsApp test expects the tiny wrong-extension PNG case to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The intended behavior is reasonable, but the submitted branch edits obsolete files and the PNG preservation path should validate image bytes before returning the original buffer.
