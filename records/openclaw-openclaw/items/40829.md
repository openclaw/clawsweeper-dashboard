## Summary

Keep open: the intended media fix is still useful, but this branch edits deleted legacy files, is conflicting, lacks after-fix real behavior proof, and its PNG fast path would bypass decoder validation.

## What This Changes

The PR adds an under-cap PNG early return in the old web media loader, normalizes JPEG-converted filenames to `.jpg`, and updates old media tests.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, active shared/WhatsApp regression coverage, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.12 shows non-alpha PNGs still go through JPEG optimization, and the active WhatsApp regression expects the tiny wrong-extension PNG case to return `image/jpeg`.

## Solution Assessment

No. The intended fix direction is reasonable, but this branch must target the active shared loader and validate PNG bytes before preserving original data.
