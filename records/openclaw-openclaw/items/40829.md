## Summary

The PR is still aimed at a plausible media bug, but it is not merge-ready: it edits obsolete `src/web` files instead of the active shared media loader, and the PNG fast path would bypass byte-level image validation.

## What This Changes

The branch changes legacy `src/web/media.ts` and its tests to preserve under-cap PNGs and rename JPEG-converted outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, active shared/WhatsApp regression coverage, and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main still routes opaque PNGs through JPEG optimization, and the active WhatsApp regression expects the tiny wrong-extension PNG case to return `image/jpeg`; no tests were run because this review is read-only.

## Solution Assessment

No. The intended behavior is reasonable, but this branch targets obsolete files and must validate PNG bytes before preserving original data.
