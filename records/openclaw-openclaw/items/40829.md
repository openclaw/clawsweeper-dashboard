The PR changes the old `src/web/media` loader/tests to return under-cap PNG buffers unchanged and rename all JPEG-converted outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, shared and WhatsApp regression coverage, and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection at current main and `v2026.5.12` shows opaque PNGs still fall through to JPEG optimization, and the active WhatsApp regression expects the tiny wrong-extension PNG case to return `image/jpeg`; no tests were run because this review was read-only.

## Solution Assessment

No. The intended behavior is reasonable, but this branch must target the active shared loader and validate PNG bytes before preserving original data.
