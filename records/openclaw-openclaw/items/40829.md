## Summary

Keep open: the requested media behavior is still useful, but this PR cannot merge as-is because it edits the deleted legacy `src/web/*` loader and its PNG fast path would skip decode validation for under-cap inputs.

## What This Changes

The PR changes the old web media loader/tests to keep under-cap PNGs as PNGs and rename JPEG-converted outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, and active shared/WhatsApp regression coverage.

## Reproduction Assessment

Yes, by source inspection: current `main` still routes non-alpha PNG optimization through the JPEG path, and the active WhatsApp test expects the tiny wrong-extension PNG case to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The requested behavior is reasonable, but this branch edits obsolete files and the PNG preserve path should validate image bytes before returning the original buffer.

## Review Findings
