## What This Changes

The PR changes the old web media loader/tests to preserve under-cap PNGs as PNG and rename JPEG-converted outputs to `.jpg`.

## Best Possible Solution

## Reproduction Assessment

Yes, by source inspection: current `main` still sends non-alpha PNG optimization through the JPEG path, and the active WhatsApp test expects the tiny wrong-extension PNG case to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The requested behavior is reasonable, but this branch edits obsolete files and the PNG preserve path should validate image bytes before returning the original buffer.
