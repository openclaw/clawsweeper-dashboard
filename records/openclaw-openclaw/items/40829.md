## Summary

Keep open: the bug-fix intent still appears valid, but this branch is not merge-ready because current main uses the shared `src/media/web-media.ts` loader, while the PR edits obsolete `src/web` files, and the PNG fast path would bypass decoder validation if ported directly.

## What This Changes

The PR changes the legacy web media loader and tests to preserve under-cap PNG buffers and rename JPEG-optimized outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` by preserving decoder-validated under-cap PNGs, normalizing every JPEG-optimized filename to `.jpg`, and updating the active media and WhatsApp tests.

## Reproduction Assessment

Yes, by source inspection: the active loader still routes non-alpha PNG images through JPEG optimization, and the active WhatsApp test expects the tiny PNG fixture to return `image/jpeg`. I did not run tests in this read-only review.

## Solution Assessment

No. The desired behavior should be fixed in the active shared loader with decoder validation first; this branch targets obsolete files and its PNG fast path is unsafe if ported as-is.
