## Summary

Keep open: the underlying media fix is useful and still relevant on current main, but this PR is not merge-ready because it targets legacy `src/web` files, conflicts with main, lacks real behavior proof, and would need PNG validation before a direct port.

## What This Changes

The branch edits `src/web/media.ts` and `src/web/media.test.ts` to preserve under-cap PNGs and rename any JPEG-optimized output to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` by preserving decoder-validated under-cap PNGs, normalizing every JPEG-optimized filename to `.jpg`, and updating the active media and WhatsApp tests.

## Reproduction Assessment

Yes, by source inspection: current `loadWebMedia` still sends non-alpha PNGs through JPEG optimization, and the active WhatsApp test expects the tiny PNG fixture to return `image/jpeg`. I did not execute tests in this read-only review.

## Solution Assessment

No. The desired behavior should be fixed in the active shared loader with decoder validation first; this branch targets obsolete files and its PNG fast path is unsafe if ported verbatim.

## Review Findings
