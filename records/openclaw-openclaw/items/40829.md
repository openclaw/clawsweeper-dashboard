## Summary

Keep open: the requested media behavior still looks useful on current main, but this branch is not mergeable as-is because it edits legacy deleted files, has a PNG validation gap, and lacks real behavior proof.

## What This Changes

The PR changes web media loading to preserve under-cap PNG buffers and normalize JPEG-converted filenames to `.jpg`.

## Best Possible Solution

Port the behavior to the active shared web-media loader by preserving decoder-validated under-cap PNGs, normalizing every JPEG-optimized filename to `.jpg`, and updating the active shared/WhatsApp media tests.

## Reproduction Assessment

Yes, by source inspection: current main still routes non-alpha PNG optimization through JPEG conversion in `src/media/web-media.ts`, and the active WhatsApp media test still expects the wrong-extension tiny PNG to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The desired behavior is reasonable, but this branch targets obsolete files and should be replaced or rebased onto `src/media/web-media.ts` with PNG decoder validation before preserving original bytes.
