## Summary

Keep open: the reported media behavior still appears present on current main, but this branch edits deleted legacy files, lacks after-fix real behavior proof, and adds a PNG validation bypass.

## What This Changes

The PR changes the legacy `src/web` media loader/tests to preserve under-cap PNG output and rename JPEG-converted media to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated in-limit PNG preservation, all-JPEG filename normalization, and active shared/WhatsApp regression coverage.

## Reproduction Assessment

Yes, by source inspection: the active shared loader still routes non-alpha PNG optimization through the JPEG path, and the active WhatsApp test still expects the tiny wrong-extension PNG case to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The intended fix direction is reasonable, but it must target the active `src/media/web-media.ts` implementation and validate PNG bytes before returning original data.
