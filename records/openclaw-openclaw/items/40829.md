## Summary

Keep open, but do not merge as-is. The intended PNG preservation bug is still present on current main, but this branch edits legacy `src/web` files that no longer exist on main and its under-cap PNG fast path returns bytes before validating that the payload is a decodable PNG.

## What This Changes

The PR branch changes the legacy web media loader to preserve under-cap PNG images and rename any JPEG-optimized output to `.jpg`, with matching legacy loader tests.

## Best Possible Solution

Port the useful behavior to `src/media/web-media.ts`: preserve under-cap PNG bytes only after validation, normalize filenames for every JPEG output, update shared and WhatsApp-facing tests, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows non-alpha PNG input still reaches JPEG optimization, and the active WhatsApp test still expects the tiny PNG fixture to return `image/jpeg`.

## Solution Assessment

No. The direction is correct, but this PR is not the best current fix because it targets obsolete files and bypasses image validation; the maintainable fix belongs in the shared loader with decode-backed PNG validation.
