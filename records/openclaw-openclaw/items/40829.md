## Summary

Keep open. The requested media behavior still is not on current main, but this branch cannot merge as-is because it edits deleted legacy files, lacks real behavior proof, and its PNG fast path bypasses decoder validation.

## What This Changes

The PR changes the legacy `src/web` media loader/tests to keep under-cap PNGs as PNG and rename JPEG-converted outputs to `.jpg`.

## Best Possible Solution

Port the behavior to `src/media/web-media.ts` with decoder-validated in-limit PNG preservation, all-JPEG filename normalization, and active shared/WhatsApp regression coverage.

## Reproduction Assessment

Yes, by source inspection: the active shared loader still routes non-alpha PNG optimization through the JPEG path, and the active WhatsApp test expects the tiny wrong-extension PNG case to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The intended fix direction is reasonable, but this branch edits obsolete files and the under-cap PNG preservation path should validate image bytes before returning the original buffer.
