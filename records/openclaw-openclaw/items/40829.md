## Summary

Keep this PR open because the media bug is still present on current main, but the branch is not merge-ready: it edits deleted legacy files, needs PNG decode validation, and lacks real behavior proof.

## What This Changes

The branch changes the legacy web media loader to preserve under-cap PNGs and rename any JPEG-converted output to `.jpg`, with tests for both cases.

## Best Possible Solution

Replace or rebase the branch onto `src/media/web-media.ts` with decoder-validated under-cap PNG preservation, all-JPEG filename normalization, and active shared/WhatsApp regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main still sends non-alpha PNG optimization through JPEG conversion, and the active WhatsApp test still expects the tiny wrong-extension PNG to return `image/jpeg`. I did not run tests because this review is read-only.

## Solution Assessment

No. The desired behavior is reasonable, but this branch targets obsolete files and should be ported to the active loader with PNG byte validation before preserving original bytes.
