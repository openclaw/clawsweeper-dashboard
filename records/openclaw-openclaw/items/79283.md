## What This Changes

The issue asks for long-running exec commands to survive session heartbeat timeouts or gain an opt-in detach-style lifecycle.

## Best Possible Solution

Resolve the remaining process-denied case through the related PR or a narrowed follow-up that keeps finite timeout guarantees, synchronizes exec/background-process docs, and preserves the existing abort protection for normal backgrounded exec sessions.
