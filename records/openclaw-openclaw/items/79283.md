## Summary

Keep open: current main and the latest release protect normal backgrounded exec sessions from run-abort kills, but the process-denied path still runs synchronously and can match the remaining heartbeat-timeout failure. The related open PR covers that likely path and needs maintainer policy/security review rather than cleanup closure.

## What This Changes

The issue asks for long-running exec commands to survive session heartbeat timeouts or gain an opt-in detached lifecycle instead of receiving SIGTERM.

## Best Possible Solution

Resolve the process-denied long-exec path through https://github.com/openclaw/openclaw/pull/74951 or a narrower follow-up that keeps finite timeout and management guarantees while preserving existing background-abort protection.
