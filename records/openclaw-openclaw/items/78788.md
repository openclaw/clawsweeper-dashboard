## What This Changes

Ensure MCP tools from configured servers respect `tools.allow` before they are exposed to the model prompt/tool catalog.

## Best Possible Solution

Keep the shipped final bundled-tool policy path; users on 2026.3.x should upgrade to v2026.4.22 or newer.

## Reproduction Assessment

No current-main failing reproduction is available because the source path is already fixed. The historical root cause is source-reproducible from the related report: bundled MCP/LSP tools used to be appended after policy filtering, while current main filters them first.

## Solution Assessment

Yes. Applying the same final policy pipeline to bundled MCP/LSP tools before `effectiveTools` is built is the narrow maintainable fix and matches the documented `tools.allow` contract.
