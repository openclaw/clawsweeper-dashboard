## Summary

Keep open: the linked Telegram reaction wake gap is real and this PR is a useful narrowing, but the latest head still has a security/authority defect and only mock/unit proof rather than real Telegram behavior proof.

## What This Changes

The PR adds stable Telegram standard/custom reaction keys, requests an immediate heartbeat after newly queued reaction events, and updates Telegram docs, changelog, and focused tests.

## Best Possible Solution

Land a Telegram-owned wake patch that enqueues reaction events as untrusted/owner-downgraded, covers that authority flag in focused tests, and includes redacted live Telegram proof of reaction-only wake.

## Reproduction Assessment

Yes for a source-level reproduction: current main enqueues accepted Telegram reaction events and does not request a heartbeat in the message_reaction handler. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, not yet. The stable reaction-key and wake-after-successful-enqueue shape is maintainable, but the PR must owner-downgrade reaction system events before waking and still needs real Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Downgrade reaction system events before waking:** `extensions/telegram/src/bot-handlers.runtime.ts:1541-1544`
  - body: `message_reaction` updates can be accepted from non-owner users under open/all or allowlist configs, but this new wake path queues the reaction without `forceSenderIsOwnerFalse` or `trusted:false`. Heartbeat only clears owner authority when pending events carry that marker, so a reaction-only wake can run with owner privileges; enqueue these events as untrusted before requesting the heartbeat.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff can let accepted non-owner Telegram reactions trigger a heartbeat turn without the system-event owner-downgrade flag.

Concerns:

- **[high] Reaction wake can preserve owner authority:** `extensions/telegram/src/bot-handlers.runtime.ts:1541`
  - body: The PR wakes after queuing a Telegram reaction event, but the enqueue options omit `forceSenderIsOwnerFalse`/`trusted:false`. Heartbeat authority downgrade depends on that flag, so non-owner reaction-only inputs can be processed as an owner-authorized heartbeat turn.
  - confidence: 0.86
