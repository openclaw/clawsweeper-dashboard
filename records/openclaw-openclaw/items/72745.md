## Summary

Keep open. Current main has adjacent diagnostics export, Codex feedback upload, plugin-scope guidance, and optional sandboxing, but not the requested local AI-fix capture/upload, business-impact audit, or sandboxed upstream PR-generation loop; this is broad, security-sensitive product work.

## What This Changes

The issue requests a self-bootstrapping feedback loop that captures local AI fixes, anonymizes and uploads them with opt-in consent, reviews them for business impact, and runs untrusted patches through sandboxed regression before upstream PR generation.

## Best Possible Solution

A maintainer-owned RFC should split the request into privacy/consent/redaction, upstream intake/provenance, untrusted-code sandbox policy, and business-impact review UX before deciding which parts belong in core, plugin APIs, ClawHub, or automation.
