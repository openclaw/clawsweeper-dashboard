## Summary

Keep open. Current main has adjacent diagnostics export, Codex feedback upload, plugin-scope guidance, and optional sandboxing, but not the requested local AI-fix capture/upload, business-impact audit, or sandboxed upstream PR-generation loop; the remaining work is broad and security-sensitive.

## What This Changes

The issue requests a self-bootstrap feedback loop that captures local AI bug fixes, anonymizes and uploads them upstream, audits business impact, and runs untrusted fixes through sandboxed regression before PR generation.

## Best Possible Solution

A maintainer-owned RFC should split the request into privacy/consent/redaction, upstream intake/provenance, untrusted-code sandbox policy, and business-impact review UX before deciding which parts belong in core, plugin APIs, ClawHub, or automation.
