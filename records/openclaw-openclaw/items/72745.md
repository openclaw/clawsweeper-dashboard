## Summary

Keep open. Current main has adjacent diagnostics, Codex feedback upload, ClawSweeper review dispatch, plugin/ClawHub scope boundaries, and optional sandboxing, but it does not implement the requested consented local AI-fix upload, business-impact audit, or sandboxed PR-generation loop; the remaining work is a broad feature and security design.

## What This Changes

The issue requests a self-bootstrap loop that captures local AI-generated fixes after consent, anonymizes and uploads them upstream, produces business-impact review reports, and gates untrusted patches through sandboxed regression before PR creation.

## Best Possible Solution

A maintainer-owned RFC should split this into consented capture/redaction, upstream intake schema and ownership, untrusted-patch sandbox/provenance gates, regression policy, and business-impact review UX, using plugin/API seams where possible.

## Reproduction Assessment

Not applicable; this is a feature and architecture request, not a runtime bug. The high-confidence check is current-main source/docs inspection showing adjacent diagnostics, Codex feedback upload, ClawSweeper dispatch, and sandboxing but no requested upstream AI-fix intake loop.

## Solution Assessment

Unclear; the all-in-one core mechanism is not yet proven to be the narrowest maintainable solution. A safer path is to decompose it across privacy-reviewed local capture, plugin/API design, ClawSweeper or other upstream automation, and explicit sandbox/security policy.
