## Summary

Keep open. Current main has adjacent diagnostics, Codex feedback upload, ClawSweeper forwarding, and optional sandboxing, but it does not implement the requested opt-in local-fix upload, business-impact upstream audit, or sandboxed PR-generation loop; the remaining work is broad and security-sensitive.

## What This Changes

Request to add an opt-in local AI-fix feedback pipeline, upstream business-impact review, and sandboxed anti-poisoning/regression flow for AI-generated fixes.

## Best Possible Solution

A maintainer-owned RFC should split the idea into local capture/redaction, intake schema and ownership, business-impact reporting, and untrusted-patch sandbox/regression policy, using plugin/API or ClawSweeper surfaces where they fit.

## Reproduction Assessment

Not applicable; this is a feature and architecture request, not a runtime bug. The high-confidence check is current-main source/docs inspection showing adjacent diagnostics, feedback, sandbox, and ClawSweeper paths but no requested upstream feedback loop.

## Solution Assessment

Unclear; the proposed all-in-one core mechanism is not proven to be the narrowest maintainable solution. A safer direction is to decompose it across product/privacy/security design, plugin or API seams, and explicit upstream automation policy.
