## Summary

Keep open: current main and the latest release still lack a supported model/provider override for main-session cron jobs using explicit systemEvent payloads. The direct implementation PR is closed unmerged, and the active related UI work intentionally stays on the existing agentTurn model contract.

## What This Changes

The issue requests job-level model/provider selection for cron jobs that run through `sessionTarget: "main"` with `payload.kind: "systemEvent"`.

## Best Possible Solution

Define one maintainer-approved main-session cron override contract and implement it through schema, normalization, SQLite persistence, main wake/enqueue execution, docs/UI, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit systemEvent cron payloads cannot carry `model`, normalization deletes it, persistence does not round-trip it, and main-session execution never passes a model/provider override.

## Solution Assessment

Unclear. The need is valid, but the issue lists multiple possible API shapes and the safe solution depends on maintainer decisions about provider/session precedence and persisted cron configuration semantics.
