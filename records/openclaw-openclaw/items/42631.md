## Summary

Current main still lacks a supported model field for main-session system-event cron jobs, and the linked implementation closed unmerged after revealing a fragile next-heartbeat transport. The request remains useful, but maintainers must first choose a per-run contract that cannot overwrite or leak into persistent main-session model state.

## What This Changes

Support a per-job model override for main-session system-event cron execution while preserving the interactive main session’s default model.

## Best Possible Solution

Add an explicit typed per-run model override carried from the cron payload through wake scheduling into heartbeat execution, validate it with existing model policy, and never persist it to the main-session entry.

## Reproduction Assessment

Yes at source level: `src/cron/types.ts` excludes `model` from `systemEvent`, while the documented model override is limited to isolated agent turns. A live failure is unnecessary to establish that the requested public contract is absent.

## Solution Assessment

No, the prior proposal’s context-key encoding and possible session mutation are not the best solution; a typed, ephemeral execution override is narrower and protects persistent user preferences.
