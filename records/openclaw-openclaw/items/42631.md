## Summary

Keep open: current main and v2026.6.10 still lack a supported model/provider override for main-session cron jobs using explicit systemEvent payloads. Related work covers isolated agentTurn model behavior or UI visibility, while the direct implementation PR is closed unmerged and had unresolved wake-path concerns.

## What This Changes

The issue requests a supported job-level model/provider override for cron jobs that use sessionTarget main with an explicit systemEvent payload.

## Best Possible Solution

Define one maintainer-approved cron override contract, validate it through the existing model-selection policy, and update protocol schema, normalization, persistence, main-session runtime, UI/docs, and regression proof together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence capability reproduction: explicit systemEvent cron payloads cannot carry model, normalization deletes it, persistence does not round-trip it, and main-session execution does not pass a model/provider override.

## Solution Assessment

Unclear. The need is valid, but the safe solution depends on maintainers choosing the supported API shape and provider/session precedence before implementation.
