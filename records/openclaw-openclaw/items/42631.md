## Summary

Keep this open: current main still only supports cron model overrides on isolated `agentTurn` jobs, while main-session `systemEvent` jobs have no schema, type, or execution path for model/provider selection. The related #42708 implementation attempt was closed unmerged, and its review discussion flagged broken `next-heartbeat` handling.

## What This Changes

Support job-level model/provider selection for cron jobs that run as main-session system events instead of only isolated agent turns.

## Best Possible Solution

Introduce an explicit, documented cron model/provider override contract for main `systemEvent` jobs with allowlist validation and non-leaky session semantics, or intentionally document that model overrides remain isolated-agent-only.

## Reproduction Assessment

Yes, from source inspection: current main has no schema/type slot or forwarding path for a model override on `main + systemEvent` cron jobs. I did not run tests because this read-only review cannot create local test artifacts.

## Solution Assessment

Unclear. There is no active patch, and the closed related implementation was not sufficient as-is because its review flagged broken `next-heartbeat` handling; maintainers should choose the API shape before implementation.
