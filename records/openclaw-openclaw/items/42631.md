## Summary

Keep open: current main still lacks a supported job-scoped model/provider override for `sessionTarget: "main"` plus `payload.kind: "systemEvent"` cron jobs, and the linked implementation PR was closed unmerged after review found problems in its approach.

## What This Changes

The issue requests a supported cron job-level model/provider override for main-session system-event cron jobs so scheduled work can use a cheaper model without editing session metadata.

## Best Possible Solution

Add an additive cron protocol, CLI, normalizer, docs, and Gateway execution contract for a validated job-scoped provider/model on main-session system events, with explicit transient-versus-persisted semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `systemEvent` payloads cannot carry model/provider data, normalization strips attempted overrides, and the main-session enqueue/wake path has no override parameter.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public contract first because extending `payload.model`, adding top-level fields, or adding `sessionOverride` have different validation, routing, and persistence semantics.
