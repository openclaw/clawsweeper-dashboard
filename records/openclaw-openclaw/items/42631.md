## Summary

Keep open: current main still lacks a supported provider/model override for `sessionTarget: "main"` plus `payload.kind: "systemEvent"` cron jobs, and the only directly linked implementation PR is closed unmerged. This is an additive cron/Gateway API feature that needs maintainer choice on field shape and session persistence semantics.

## What This Changes

The issue requests a supported job-level provider/model override for cron jobs that run as main-session system events.

## Best Possible Solution

Add an additive cron protocol, CLI, normalizer, docs, and Gateway execution contract for a validated job-scoped provider/model on main-session system events, with explicit transient-vs-persisted semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `systemEvent` payloads cannot carry model/provider data, normalization removes attempted override fields, and the main-session enqueue/wake path has no override parameter.

## Solution Assessment

Unclear. The requested capability is useful, but maintainers should choose the public contract first: extending `payload.model`, adding top-level cron fields, or adding a scoped session override block each has different persistence and provider-routing semantics.
