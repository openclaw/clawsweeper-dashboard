## Summary

Keep open. Current main and v2026.5.20 still lack a supported job-level provider/model override for main-session systemEvent cron jobs; the linked implementation at https://github.com/openclaw/openclaw/pull/42708 closed unmerged after review found runtime design problems.

## What This Changes

The issue requests job-level provider/model override support for cron jobs that use `sessionTarget: "main"` with `payload.kind: "systemEvent"`.

## Best Possible Solution

Approve one additive cron protocol and CLI contract for validated transient job-scoped provider/model overrides on main-session systemEvent jobs, then implement both wake modes without persisting cron-only state into later turns.

## Reproduction Assessment

Yes, by source inspection rather than a runtime failure. Current main rejects or strips model/provider fields from systemEvent cron payloads, and the main-session execution path has no provider/model override carrier.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public field shape and transient-versus-persisted session semantics before implementation; the closed `payload.model` attempt was not safe enough to merge as the permanent contract.
