## Summary

The current main branch still lacks the requested supported model/provider override for `main` + `systemEvent` cron jobs; the source-level behavior is clear, but the fix shape changes cron/Gateway API semantics and override lifetime, so this should remain open for maintainer product review.

## What This Changes

The issue requests a supported cron job-level provider/model override for `sessionTarget: "main"` plus `payload.kind: "systemEvent"` jobs so recurring work can use a cheaper model without editing session metadata.

## Best Possible Solution

Add an additive cron protocol, CLI, normalization, docs, and gateway execution contract for validated job-scoped provider/model overrides on main-session system events, with explicit transient-versus-persisted semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `systemEvent` payloads cannot carry model/provider data, normalization strips attempted overrides, and the main-session enqueue/wake path has no override parameter.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public contract first because the three proposed shapes have different validation, routing, and persistence semantics.
