- `P2`: This is a clear cron/model capability gap with limited blast radius, not an urgent outage or regression.
- `impact:session-state`: The current workaround mutates session model metadata, and the correct fix must avoid leaking cron-only model state into later main-session turns.

## Summary

Keep open. Current main and v2026.5.20 still lack a supported provider/model override for main-session systemEvent cron jobs, and the linked implementation PR was closed unmerged after review found runtime and contract issues.

## What This Changes

Request support for choosing provider/model per cron job when a job runs with `sessionTarget: "main"` and `payload.kind: "systemEvent"`.

## Best Possible Solution

Add an additive cron protocol, CLI, normalization, docs, and Gateway execution contract for validated transient job-scoped provider/model overrides on main-session systemEvent jobs, covering both wake modes without persisting cron-only state into later turns.

## Reproduction Assessment

Yes, by source inspection rather than a runtime failure. Current main accepts job model selection only on agentTurn payloads, rejects or strips it from systemEvent payloads, and has no main-session cron override carrier.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public field shape and transient-versus-persisted session semantics before implementation; the closed `payload.model` attempt was not safe enough to merge as the permanent contract.

## Risks / Open Questions

- Maintainers still need to choose the public field shape: `payload.model`, top-level cron fields, or a dedicated session override block.
- The override lifetime is product-sensitive: persisting a cron model into main-session metadata would change later interactive turns, while a transient carrier must work for both `wakeMode: "now"` and `wakeMode: "next-heartbeat"`.
- Provider/model validation, fallback behavior, and auth-profile selection need to match the existing isolated-agent cron model contract before this becomes a safe implementation task.
