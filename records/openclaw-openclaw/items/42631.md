## Summary

Keep open: current main and the latest release still lack a supported model/provider override for main-session cron jobs using explicit systemEvent payloads. The direct implementation attempt is closed unmerged, and the merged adjacent work only covers implicit text or isolated agentTurn model selection.

## What This Changes

The issue requests a supported per-job model/provider override for cron jobs that run with sessionTarget main and payload.kind systemEvent.

## Best Possible Solution

Land a maintainer-approved cron override contract that validates through existing model-selection policy and updates protocol schema, normalization, persistence, runtime dispatch, UI/docs, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence capability reproduction: explicit systemEvent cron payloads cannot carry model, normalization deletes it, persistence does not round-trip it, and main-session execution does not pass a model/provider override.

## Solution Assessment

Unclear. The user need is valid, but maintainers still need to choose the supported API shape and precedence rules before implementation.
