- `impact:session-state`: The current workaround mutates session model metadata, and the intended fix must avoid leaking cron-only model state into later main-session turns.

## Summary

Keep open. Current main and the latest release still lack a supported job-scoped provider/model override for `sessionTarget="main"` + `payload.kind="systemEvent"` cron jobs, and the only linked implementation PR was closed unmerged after review identified runtime contract issues.

## What This Changes

Allow main-session `systemEvent` cron jobs to choose a per-job provider/model without manually editing session metadata.

## Best Possible Solution

Add an additive cron protocol, CLI, normalization, docs, and Gateway execution contract for validated transient job-scoped provider/model overrides on main-session systemEvent jobs across both wake modes, without persisting cron-only state into later turns.

## Reproduction Assessment

Yes, by source inspection rather than a runtime failure. Current main accepts job model selection only on agentTurn payloads, strips or rejects it for systemEvent payloads, and has no main-session override carrier.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public field shape and transient-versus-persisted session semantics before implementation; the closed `payload.model` attempt was not safe enough to merge as the permanent contract.
