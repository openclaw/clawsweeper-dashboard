## Summary

Keep open: current main and latest release still lack a supported model/provider override for main-session cron jobs using explicit `systemEvent` payloads. The direct implementation PR is closed unmerged, while the merged and open related work only covers implicit text/isolated `agentTurn` behavior or UI visibility.

## What This Changes

Request support for job-level model/provider selection on cron jobs that run with `sessionTarget: "main"` and `payload.kind: "systemEvent"`.

## Best Possible Solution

Define one maintainer-approved cron override contract, validate it through the existing model-selection policy, and update protocol schema, normalization, persistence, main-session runtime, UI/docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence capability reproduction: explicit systemEvent cron payloads cannot carry `model`, normalization deletes it, persistence does not round-trip it, and main-session execution does not pass a model/provider override.

## Solution Assessment

Unclear. The need is valid, but the safe solution depends on maintainers choosing the supported API shape and provider/session precedence before implementation.
