## Summary

Keep open. Current main and v2026.5.12 still lack a supported model/provider override for main-session systemEvent cron jobs, and the linked implementation attempt was closed unmerged after review found runtime design problems; the remaining work is a product/protocol decision, not cleanup.

## What This Changes

Add a supported job-level provider/model override for main-session systemEvent cron jobs so recurring jobs can use a cheaper model without editing internal session metadata.

## Best Possible Solution

Add an additive cron protocol, CLI, normalization, docs, and Gateway execution contract for validated job-scoped provider/model overrides on main-session system events, with explicit transient lifetime and coverage for both wake modes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: systemEvent payloads only accept kind/text, Gateway validation rejects extra fields, normalization strips model/provider, and main-session execution never passes an override.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public field shape and transient-versus-persisted semantics before implementation.
