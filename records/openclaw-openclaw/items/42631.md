## Summary

Keep open. Current main and v2026.5.12 still lack a supported job-level model/provider override for main-session systemEvent cron jobs; the closed linked PR tried one payload.model shape but was not merged, and the remaining work is an API/product decision rather than conservative cleanup.

## What This Changes

Request support for choosing a cheaper model/provider per cron job when the job uses sessionTarget="main" with payload.kind="systemEvent".

## Best Possible Solution

Add an additive cron protocol, CLI, normalization, docs, and Gateway execution contract for validated job-scoped provider/model overrides on main-session system events, with explicit transient override lifetime.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: systemEvent payloads only accept kind/text, the Gateway schema rejects extra fields, normalization deletes model/provider, and main-session execution never passes overrides.

## Solution Assessment

Unclear. The capability is useful, but maintainers should choose the public contract and transient-versus-persisted semantics before implementation.
