## Summary

Keep open: current main and the latest release still lack a native requester-scoped model allowlist for explicit `sessions_spawn.model` overrides, and no canonical PR or duplicate now owns this exact guardrail. The request is valid as a model-routing/cost-control feature, but it adds a new config and enforcement policy surface that needs maintainer product review first.

## What This Changes

Add a native `agents.defaults.subagents.allowedModels` / `agents.list[].subagents.allowedModels` guardrail for explicit `sessions_spawn.model` overrides.

## Best Possible Solution

Maintainers should first decide the native model-override policy; if accepted, implement a requester-scoped allowlist in the canonical config schema, enforce it before child session creation, preserve backward-compatible unset behavior, normalize aliases/provider refs deliberately, and document the upgrade behavior.

## Reproduction Assessment

Yes for the current source behavior, but not for the reported billing incident. Current main still passes `sessions_spawn.model` into native model selection before defaults and has no native `subagents.allowedModels` config key.

## Solution Assessment

Unclear until maintainers choose the product/config policy. The requested shape matches the existing plugin `allowedModels` pattern, but native enforcement needs a deliberate config contract and compatibility decision.
