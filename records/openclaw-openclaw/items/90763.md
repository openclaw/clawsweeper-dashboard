## Summary

Keep open: current main and v2026.6.9 still lack a native `sessions_spawn` model override allowlist, while explicit native model overrides are accepted and applied before configured defaults. This is a valid core model-routing guardrail request, but it adds a new config surface and needs maintainer product approval before implementation.

## What This Changes

The issue requests `agents.defaults.subagents.allowedModels` and/or `agents.list[].subagents.allowedModels` to restrict explicit native `sessions_spawn.model` overrides by requester agent.

## Best Possible Solution

Approve the native requester-scoped model override policy, then implement schema, docs, spawn-plan enforcement, and focused tests while preserving current behavior when the allowlist is unset.

## Reproduction Assessment

Yes, at source level: current schemas have no native `subagents.allowedModels`, and explicit `sessions_spawn.model` overrides are accepted before configured subagent defaults. I did not replay the reporter's billing incident live.

## Solution Assessment

Unclear until maintainers choose the product policy. A requester-scoped allowlist mirrors existing `allowAgents` and plugin `allowedModels` patterns, but adding a native config/default surface needs approval first.
