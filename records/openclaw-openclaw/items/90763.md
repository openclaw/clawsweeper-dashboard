## Summary

Keep open: current main and the latest release still lack a native `subagents.allowedModels` policy, while explicit `sessions_spawn.model` remains the first model-selection input. The request is a plausible cost/model-routing guardrail, but it adds a new config and enforcement surface that needs maintainer product approval before implementation.

## What This Changes

Request adds a requester-scoped native subagent model allowlist under `agents.defaults/list[].subagents` to reject disallowed explicit `sessions_spawn.model` overrides.

## Best Possible Solution

Maintainers should decide the native config policy; if approved, enforce a requester-scoped allowlist before child session creation, normalize provider/model aliases consistently, and document unset or wildcard behavior.

## Reproduction Assessment

Yes for the current source behavior, but not for the reported billing incident. Current main accepts an explicit `sessions_spawn.model` before configured subagent defaults and records it as a user-sourced model override.

## Solution Assessment

Unclear until maintainers choose the config policy. The proposed requester-scoped allowlist matches existing `allowAgents` and plugin `allowedModels` patterns, but the exact native semantics should be approved before code work starts.
