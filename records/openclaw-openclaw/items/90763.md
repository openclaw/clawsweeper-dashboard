## Summary

Keep open: current main and the latest release still lack a native subagent model override allowlist, while explicit native `sessions_spawn.model` overrides are accepted before configured defaults. The related inherit and precedence items are adjacent model-routing work, not the requested cost-control guardrail, so this needs maintainer product approval before implementation.

## What This Changes

Request adds `agents.defaults.subagents.allowedModels` and/or `agents.list[].subagents.allowedModels` to restrict explicit native `sessions_spawn.model` overrides by the dispatching agent.

## Best Possible Solution

Approve the native model-override policy first, then implement requester-scoped allowlist enforcement in schema, spawn planning, docs, and focused tests while preserving current behavior when unset.

## Reproduction Assessment

Yes, at source level: current native schemas have no `subagents.allowedModels`, and explicit `sessions_spawn.model` is resolved before configured subagent defaults. I did not replay the reporter's billing incident live.

## Solution Assessment

Unclear until maintainers choose the product policy. The requester-scoped allowlist mirrors existing `allowAgents` and plugin `allowedModels` patterns, but the native config semantics and enforcement point need approval first.
