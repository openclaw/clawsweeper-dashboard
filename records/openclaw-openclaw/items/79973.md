## Summary

Keep open: current main and the latest shipped release still skip ACP runtime events after the timeout gate closes, and the closed follow-up PR did not land because broad late-event forwarding could leak visible output after timeout.

## What This Changes

The issue reports that `consumeAcpTurnStream` drops runtime events and errors emitted after the ACP turn-timeout gate closes.

## Best Possible Solution

Define a filtered post-timeout ACP stream contract, then preserve late error or terminal diagnostics through a non-visible path while continuing to suppress late visible output, with manager and dispatch/projector regression coverage.

## Reproduction Assessment

Yes. A high-confidence source path is to close `eventGate` from the timeout handler and then yield an ACP `error` or `done` event; current code skips it before delivery or error conversion, though I did not run a failing test in this read-only review.

## Solution Assessment

Unclear until maintainers choose the post-timeout event contract. The broad deliver-then-cancel shape is not the best fix because `onEvent` feeds visible reply projection; a filtered diagnostic path is safer.
