## Summary

Keep open: current main still drops ACP runtime events and errors after the turn-timeout gate closes, and no duplicate or open PR appears to cover this exact post-timeout contract. The issue asks maintainers to choose between late-event delivery and an explicit cancellation signal, so this needs product/owner direction before a repair.

## What This Changes

Request that ACP turn streams preserve or explicitly signal events and errors emitted after a turn-timeout instead of silently discarding them.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update the manager and focused tests so late errors remain observable without allowing late user-visible output after timeout.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate` during a timed-out turn, then have the runtime iterator yield an `error` or other event; current main skips it before delivery or rejection.

## Solution Assessment

Unclear until maintainers choose the contract. The narrow fix should preserve the timeout boundary for output while surfacing or logging late terminal/error information with regression coverage.
