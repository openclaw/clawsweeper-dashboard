## Summary

Keep open: current main still drops ACP events after the timeout gate closes, including runtime error events, and the related fix remains an open PR rather than merged code. The issue is valid, but the post-timeout event contract still needs maintainer review because forwarding all late events can affect user-visible output.

## What This Changes

The issue asks OpenClaw to make ACP runtime events and errors emitted after a turn-timeout observable instead of silently dropping them in `consumeAcpTurnStream`.

## Best Possible Solution

Review and land the linked ACP fix PR or a replacement that keeps late visible output suppressed while still surfacing late terminal/error diagnostics with focused regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: close the ACP event gate during a timed-out turn, then have the runtime iterator yield an `error` or other event afterward. Current main skips that event before delivery, terminal tracking, or error surfacing.

## Solution Assessment

Unclear until maintainers confirm the ACP post-timeout contract. The safest direction is to preserve the timeout boundary for visible output while making late `done`/`error` diagnostics observable through a filtered or non-visible path.
