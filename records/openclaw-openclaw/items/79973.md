## Summary

Keep open: current main still has the reported ACP timeout behavior, and an open linked PR uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue reports that `consumeAcpTurnStream` drops late ACP events and errors after the timeout gate closes and asks maintainers to choose the post-timeout event contract.

## Best Possible Solution

Review and land or adjust https://github.com/openclaw/openclaw/pull/80051, or an equivalent ACP manager fix, so late errors and terminal signals stay observable while late output remains suppressed, with focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate` during `awaitTurnWithTimeout`, then have `runtime.runTurn()` yield an `error` or `done`; current main skips it before delivery or error surfacing.

## Solution Assessment

Unclear until maintainers settle the ACP post-timeout contract. The narrow maintainable direction is to preserve the no-late-output boundary while making late errors and terminal signals observable, and the open linked PR is the current candidate for that review.
