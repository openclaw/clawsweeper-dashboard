## Summary

Keep open: current main and the latest shipped release still have the reported ACP timeout behavior, and the only linked fix PR was closed unmerged after review found that forwarding every late event could leak visible output after timeout.

## What This Changes

The issue asks for `consumeAcpTurnStream` to stop silently dropping ACP events and errors emitted after a turn-timeout closes the event gate.

## Best Possible Solution

Define the ACP post-timeout stream contract, preserve late error or terminal observability through a filtered non-visible path, and add focused manager plus dispatch/projector regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close the gate during a timed-out turn, then have `runtime.runTurn()` yield `error`, `done`, or another event; current main hits the closed-gate `continue` before conversion or delivery.

## Solution Assessment

Unclear. Capturing late diagnostics is the right direction, but the prior deliver-all shape is not the narrowest safe fix because `onEvent` feeds visible reply projection.
