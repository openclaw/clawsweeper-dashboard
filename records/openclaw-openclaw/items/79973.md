## Summary

Keep open: current main still drops ACP turn-stream events after the timeout gate closes, and the related attempted fix was closed unmerged after review identified a visible late-event forwarding risk.

## What This Changes

The issue asks OpenClaw to stop silently losing ACP runtime events and errors emitted after a turn timeout closes the stream gate.

## Best Possible Solution

Define the ACP post-timeout contract, then preserve late error or terminal diagnostics through a filtered non-visible path while keeping late visible output suppressed and covered by focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: close the shared gate in the timeout handler, then have the runtime iterator yield `error`, `done`, or another event; current main skips it before conversion or delivery.

## Solution Assessment

Unclear until maintainers choose the contract. Restoring late error visibility is the right direction, but the closed related PR shows that forwarding every late event through `onEvent` is not the safest implementation.
