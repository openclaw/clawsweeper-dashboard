## Summary

Keep open: current main still drops late ACP turn events and errors after timeout, and the linked fix PR was closed unmerged after review found the all-events-forwarding shape could leak visible output.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP runtime events and errors emitted after a turn timeout closes the event gate.

## Best Possible Solution

Define the post-timeout ACP stream contract, then implement a filtered path that preserves late error or terminal diagnostics while continuing to suppress late visible output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: close `eventGate.open` during a timed-out turn, then have `runtime.runTurn()` yield `error`, `done`, or another event afterward; current main skips it before delivery or error surfacing.

## Solution Assessment

Unclear. The bug is real, but the issue presents competing fix shapes and the closed linked PR showed that forwarding every late event through `onEvent` is not the narrowest maintainable solution.
