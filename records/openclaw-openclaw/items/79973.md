## Summary

Keep open: current main still has the reported ACP timeout stream behavior, and an open PR with closing syntax is already tracking the fix but has not merged.

## What This Changes

The issue reports that `consumeAcpTurnStream` silently drops late ACP events and errors after the turn-timeout gate closes.

## Best Possible Solution

Review and finish the linked PR or a replacement that keeps late output suppressed while surfacing late error/status/terminal information with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close the gate via the ACP turn-timeout path, then have `runtime.runTurn()` yield an `error` or other event afterward; current main skips that event before delivery or error surfacing.

## Solution Assessment

Unclear until the post-timeout contract is settled. The helper is the narrow place to fix this, but the final implementation should filter late output while still making late failure information observable.
