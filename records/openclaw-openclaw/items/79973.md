## Summary

Keep open: current main still drops post-timeout ACP stream events before delivery/error conversion, and https://github.com/openclaw/openclaw/pull/80051 is an open fix PR that explicitly references this issue. The issue should stay open until that PR or an equivalent reviewed fix lands.

## What This Changes

The issue asks ACP to stop silently dropping runtime events and errors emitted after a turn timeout closes the `consumeAcpTurnStream` gate.

## Best Possible Solution

Land a reviewed fix in `consumeAcpTurnStream` that keeps late output behind the timeout boundary while surfacing late errors or non-output terminal information, with targeted manager/projector coverage, then close this issue through the linked PR merge.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate` during a timed-out turn, then have `runtime.runTurn()` yield an event or error; current main continues before delivery or error conversion.

## Solution Assessment

Unclear until the post-timeout contract is settled. The helper is the narrow fix location, but the solution should preserve the no-late-output boundary while making late errors observable.
