## Label Justifications

- `P2`: This is a normal-priority ACP reliability bug with a bounded control-plane surface and no emergency security or crash-loop signal.
- `impact:message-loss`: The reported behavior suppresses runtime events and errors after timeout, which can make ACP turns appear silent to callers.

## Summary

Keep open: current main and v2026.5.22 still skip late ACP stream events before error handling or `onEvent` after the timeout gate closes, and the reporter confirmed the unresolved fix-shape question today.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP runtime events and errors that arrive after the turn-timeout gate closes in `consumeAcpTurnStream`.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update `consumeAcpTurnStream` and focused manager tests so late errors remain observable without leaking late user-visible output after timeout.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: time out an ACP turn so `eventGate.open` becomes false, then have the runtime iterator yield an `error` or other event; the loop skips it before `onEvent` or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. The narrowest maintainable fix should preserve the timeout boundary for user-visible output while still surfacing or recording late terminal/error information.
