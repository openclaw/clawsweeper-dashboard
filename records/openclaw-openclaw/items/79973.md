## Summary

Keep open: current main and the latest release still skip closed-gate ACP runtime events before `onEvent` or error conversion can observe them. The directly related fix PR closed unmerged after review found that forwarding every late event could leak visible output after timeout, so the remaining work needs a contract-aware maintainer fix.

## What This Changes

The issue asks OpenClaw to preserve ACP turn-stream events or errors emitted after a turn-timeout gate closes instead of silently dropping them.

## Best Possible Solution

Define a filtered post-timeout ACP stream contract that suppresses late visible output while preserving terminal/error diagnostics, then cover it with focused manager and ACP dispatch/projector tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate` during a timed-out turn, then have the runtime iterator yield an `error` or other event; current main hits the closed-gate `continue` before error conversion or `onEvent` delivery.

## Solution Assessment

Unclear until maintainers choose the contract. Capturing late errors is the right direction, but the closed PR's deliver-all approach is not the best fix because ACP dispatch projects `onEvent` into visible replies.
