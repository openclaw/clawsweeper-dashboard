## Summary

Keep open: current main and v2026.6.6 still skip closed-gate ACP stream events before terminal/error handling, while the matching fix PR was closed unmerged after review found it could forward visible late output.

## What This Changes

The issue reports that ACP runtime events and errors emitted after a turn-timeout gate closes are silently dropped by `consumeAcpTurnStream`.

## Best Possible Solution

Choose and implement a filtered post-timeout ACP stream contract that preserves late error and terminal diagnostics without rendering late visible output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the timeout callback closes the shared event gate, then late legacy `runTurn` events hit `continue` before terminal/error conversion or `onEvent` delivery.

## Solution Assessment

Unclear. The issue identifies a real bug, but the prior deliver-all fix is not the best shape because dispatch routes `onEvent` into visible projection; maintainers should choose a filtered diagnostic path or explicit timeout/cancel terminal signal.
