## Summary

Keep open: current main and the latest release still skip every ACP turn-stream event once the timeout gate closes, including late error conversion, and the related fix PR was closed unmerged after review found the deliver-all path could leak visible output.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP runtime events and errors emitted after a turn-timeout closes `consumeAcpTurnStream`'s event gate.

## Best Possible Solution

Define the ACP post-timeout event contract, then update the manager and focused tests so late errors or terminal diagnostics remain observable while late visible output stays suppressed.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate.open` during a timed-out turn, then have the runtime iterator yield `error` or `done`; current main skips the event before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the post-timeout contract. The closed related PR shows that forwarding every late event through `onEvent` is not the best fix because it can render visible output after timeout.
