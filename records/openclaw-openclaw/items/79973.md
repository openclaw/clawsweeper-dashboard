## Summary

Keep open: current main still drops ACP events and runtime error events after the timeout gate closes, and the fix needs an explicit post-timeout stream contract decision.

## What This Changes

Ensure ACP turn stream events, especially runtime errors, are not silently lost after the turn timeout closes the event gate.

## Best Possible Solution

Define ACP post-timeout stream semantics, then update the manager and regression tests so late errors or terminal information remain observable while late user-visible output stays blocked.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close the gate through the timeout path, then have the runtime iterator yield an `error` or other event; current main skips it before handlers or error conversion run.

## Solution Assessment

Unclear until maintainers choose the contract. The safest direction is to keep timeout protection for user-visible output while surfacing or logging late error/terminal information with targeted ACP manager and projection coverage.
