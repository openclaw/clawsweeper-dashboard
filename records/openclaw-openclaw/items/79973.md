## Summary

Keep open: current main still skips all ACP runtime events after the timeout gate closes, and the only directly related fix PR was closed unmerged after review found that forwarding every late event could leak visible output.

## What This Changes

The issue asks for ACP turn stream handling to preserve observability of events and errors emitted after a turn-timeout closes the event gate.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update `consumeAcpTurnStream` and focused manager/dispatch coverage so late error or terminal diagnostics remain observable while late visible output stays suppressed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: close `eventGate` during a timed-out turn, then have `runtime.runTurn()` yield `error` or `done`; current code skips that event before delivery, terminal tracking, or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. The closed companion PR confirmed that preserving late errors is the right direction, but forwarding every late event through `onEvent` is not the safest fix because that callback reaches visible reply projection.
