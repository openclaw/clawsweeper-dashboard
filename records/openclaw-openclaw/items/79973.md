## Label Justifications

- `P2`: This is a normal-priority ACP reliability bug with a bounded control-plane surface and no emergency security or crash-loop signal.
- `impact:message-loss`: The reported behavior suppresses runtime events and errors after timeout, which can make ACP turns appear silent to callers.

## Summary

Keep open: current main and the latest release still close the ACP turn event gate on timeout before late stream events reach error handling or `onEvent`, so the central silent-drop behavior remains. The fix should wait for maintainer direction on the post-timeout event contract because delivering all late events could leak user-visible output after the timeout boundary.

## What This Changes

The issue requests ACP timeout handling that keeps late runtime events or errors observable instead of silently discarding them after `consumeAcpTurnStream`'s gate closes.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update `consumeAcpTurnStream` and focused manager tests so late errors remain observable without leaking late user-visible output after timeout.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: time out an ACP turn so `eventGate.open` becomes false, then have the runtime iterator yield an `error` or other event; the loop skips it before `onEvent` or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. The safest implementation should preserve the timeout boundary for user-visible output while still observing or reporting late terminal/error information.
