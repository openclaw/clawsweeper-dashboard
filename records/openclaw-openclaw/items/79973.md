## Summary

Keep open: current main and v2026.6.10 still skip closed-gate ACP stream events before terminal/error handling, and the linked fix PR closed unmerged because the safe post-timeout event contract still needs maintainer selection.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP turn-stream events and errors that arrive after the turn-timeout gate closes.

## Best Possible Solution

Preserve late error and terminal diagnostics after timeout while keeping late visible text/status/tool output suppressed, with regression coverage across the manager and ACP dispatch/projector paths.

## Reproduction Assessment

Yes, source-reproducible. The timeout callback closes the shared gate, then late legacy `runTurn` events hit `continue` before terminal/error conversion or `onEvent` delivery; I did not run live proof in this read-only review.

## Solution Assessment

Unclear. The report identifies a real bug, but the prior deliver-through-`onEvent` shape is not the best fix because visible ACP events can leak after timeout; maintainers should choose a filtered diagnostic path or explicit timeout/cancel terminal signal.
