## Summary

Keep open: live `main` still drops ACP stream events after the timeout gate closes, including late error events, and the linked fix attempt at https://github.com/openclaw/openclaw/pull/80051 is closed unmerged after review found its deliver-all shape could leak visible output after timeout.

## What This Changes

The issue asks OpenClaw to preserve observability for ACP events and errors emitted after `consumeAcpTurnStream` closes its turn-timeout event gate.

## Best Possible Solution

Define and implement a filtered post-timeout ACP stream contract that preserves late error or terminal diagnostics while keeping late visible output suppressed, with focused manager and dispatch/projector coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate` during a timed-out turn, then have `runtime.runTurn()` yield `error`, `done`, or visible events; current main hits the closed-gate `continue` before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. Preserving late errors is the right direction, but forwarding every closed-gate event through `onEvent` is not the narrowest maintainable fix because that callback reaches visible reply projection.
