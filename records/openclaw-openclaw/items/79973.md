## Summary

Keep open: current main still has the reported ACP timeout behavior, and the only focused fix PR was closed unmerged after review found that forwarding every late event could leak visible output after timeout.

## What This Changes

The issue requests a fix for ACP turn-timeout handling so runtime events and errors emitted after `eventGate` closes are not silently lost.

## Best Possible Solution

Define the ACP post-timeout stream contract, then preserve late error/terminal diagnostics through a filtered or non-visible path while keeping late text/status/tool output suppressed, with focused manager and dispatch/projector coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate` from the timeout handler, then have `runtime.runTurn()` yield an `error` or `done`; current main skips it before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. The closed focused PR showed that simply forwarding every late event is not the best fix because ACP dispatch projects `onEvent` into visible reply output.
