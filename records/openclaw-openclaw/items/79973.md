## Summary

Keep open: current main still has the reported ACP timeout behavior, and the related attempted fix at https://github.com/openclaw/openclaw/pull/80051 was closed unmerged after review found that unfiltered late event forwarding could leak visible output after timeout.

## What This Changes

The issue reports that ACP turn-stream events and error events emitted after a turn-timeout gate closes are silently discarded by consumeAcpTurnStream.

## Best Possible Solution

Define the ACP post-timeout stream contract, then preserve late terminal/error diagnostics through a filtered or non-visible path while keeping user-visible output suppressed after timeout.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close eventGate during a timed-out turn, then have the runtime iterator yield an error or terminal event; current main skips it before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. A safe fix should likely keep the timeout boundary for visible output while preserving late error or terminal diagnostics with focused regression coverage.
