## Summary

Keep open: current main still drops post-timeout ACP runtime events and errors in `consumeAcpTurnStream`, and the related open fix PR has not merged and already has review blockers around visible late-event forwarding and proof. The source path is clear, but the final post-timeout event contract needs maintainer direction.

## What This Changes

The issue asks for ACP turn-stream handling to stop silently dropping events and runtime errors that arrive after the turn-timeout gate closes.

## Best Possible Solution

Define the ACP post-timeout contract, then implement filtered late diagnostic/error handling with regression coverage so visible output stays suppressed after timeout while late errors remain observable.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close `eventGate` during a timed-out turn, then have `runtime.runTurn()` yield `error`, `done`, or output afterward; current main skips the event before delivery or error conversion.

## Solution Assessment

Unclear until the contract is chosen. Capturing late errors is the right direction, but forwarding all closed-gate events through `onEvent` is not the narrowest maintainable fix because ACP dispatch routes that callback to visible reply projection.
