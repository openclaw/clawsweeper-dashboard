## Summary

Keep open: current main still skips ACP stream events and error events after the timeout gate closes, and no related item appears to own this exact post-timeout contract decision. Changing delivery could affect user-visible timeout behavior, so this needs maintainer direction before implementation.

## What This Changes

The issue reports that `consumeAcpTurnStream` drains ACP events after a turn timeout but drops late events and errors instead of surfacing them.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update the manager and targeted tests so late errors remain observable without leaking late user-visible output after timeout.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate.open` during a timed-out turn, then have the runtime iterator yield an `error` or other event; current code skips it before delivery or error conversion.

## Solution Assessment

Unclear. Both suggested directions are plausible, but the maintainable fix should preserve the timeout boundary for output while still surfacing or logging late terminal/error information with regression coverage.
