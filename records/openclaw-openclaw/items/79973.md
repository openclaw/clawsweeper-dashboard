## Summary

Keep open: current main still has the reported ACP timeout behavior, and the only related fix PR was closed unmerged after review found it could forward visible late events after timeout.

## What This Changes

The issue requests that ACP turn-stream events and errors remain observable instead of being silently discarded after the turn-timeout gate closes.

## Best Possible Solution

Define the ACP post-timeout stream contract, then update `consumeAcpTurnStream` and focused manager/dispatch/projector coverage so late errors or terminal diagnostics remain observable while late visible output stays suppressed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close the gate through the turn-timeout callback, then have `runtime.runTurn()` yield an `error`, `done`, or visible event; current main skips it before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the contract. Preserving late errors is the right direction, but the prior deliver-all approach is not the best fix unless it filters visible late events or routes diagnostics through a non-visible path.
