## What This Changes

The issue requests a fix for ACP turn-stream events and errors emitted after turn timeout so they are not silently discarded after `eventGate` closes.

## Best Possible Solution

Choose and implement a filtered post-timeout ACP stream contract that preserves late error/terminal diagnostics without rendering late visible output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the timeout callback closes the shared event gate, then late legacy `runTurn` events hit `continue` before terminal/error conversion or `onEvent` delivery.

## Solution Assessment

Unclear. The issue identifies a real bug, but the prior deliver-all fix is not the best shape because dispatch routes `onEvent` into visible projection; maintainers should choose a filtered diagnostic path or explicit timeout/cancel terminal signal.
