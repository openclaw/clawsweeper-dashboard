## Summary

Keep open: current main still drops ACP runtime events after the timeout gate closes, including runtime `error` events, and the linked fix PR was closed unmerged after review found that forwarding all late events could leak visible output after timeout.

## What This Changes

The issue asks OpenClaw to define and fix ACP post-timeout stream handling so late events or errors are not silently discarded after `eventGate` closes.

## Best Possible Solution

Adopt a scoped ACP timeout contract that preserves late error or terminal diagnostics while keeping post-timeout text/status/tool output out of visible reply delivery, then add focused manager and dispatch/projector regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: close the gate during a timed-out turn, then have the runtime iterator yield `error`, `done`, or another event; current main skips it before delivery or error conversion.

## Solution Assessment

Unclear until maintainers choose the ACP post-timeout contract. The prior all-events-forwarding approach is not the best fix because `onEvent` is connected to visible reply projection.
