## Summary

Keep open: current main still drops closed-gate ACP runtime events before `onEvent` or error conversion, and the only linked fix PR is closed unmerged after review found its broad late-event forwarding could leak visible output.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP runtime events and errors emitted after a turn timeout closes the event gate.

## Best Possible Solution

Define the ACP post-timeout stream contract, then implement filtered late diagnostics/error handling that preserves timeout suppression for visible output and add targeted manager plus dispatch/projector regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate` through the timeout handler, then have `runtime.runTurn()` yield `error` or `done`; current main skips the event before delivery or error conversion.

## Solution Assessment

Unclear. The broad deliver-then-cancel shape is not the best fix because it can forward visible output after timeout; maintainers should choose a filtered diagnostic path or explicit cancellation contract before implementation.
