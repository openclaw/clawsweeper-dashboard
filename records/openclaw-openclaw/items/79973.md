## Summary

Keep open: current main and the latest release still have the reported ACP timeout behavior, and the only linked repair PR was closed unmerged after review identified a visible-output leak risk in the proposed fix shape.

## What This Changes

Ensure ACP turn-stream events emitted after a turn timeout, especially error and terminal diagnostics, are not silently discarded while timed-out output remains suppressed.

## Best Possible Solution

Define the ACP post-timeout stream contract and implement a focused manager/test change that observes late error or terminal diagnostics while suppressing visible output after timeout.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: make `AcpSessionManager.runTurn` close `eventGate` through the timeout path, then have `runtime.runTurn()` yield `error` or `done`; current main skips it before delivery or error conversion.

## Solution Assessment

Unclear until maintainers decide the post-timeout contract. The prior deliver-all PR showed why the narrow solution should preserve late error or terminal diagnostics without forwarding visible text/status/tool output through the reply projector.
