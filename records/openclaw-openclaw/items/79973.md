## Summary

Keep open: current main still drops ACP runtime events after the timeout gate closes, and the related fix PR was closed unmerged after review found that forwarding all late events could leak visible output after timeout. The issue is valid, but maintainers still need to choose the post-timeout event contract before a safe repair.

## What This Changes

The issue asks for ACP turn streams to preserve late events and errors after a turn timeout instead of silently dropping them in `consumeAcpTurnStream`.

## Best Possible Solution

Define the ACP post-timeout contract, then preserve late error/terminal diagnostics while keeping late visible output suppressed and cover it with manager plus dispatch/projector tests.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: close `eventGate` in the timeout handler, then have `runtime.runTurn()` yield `error` or `done`; current main skips the event before conversion or delivery. I did not execute a live timeout because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the ACP post-timeout contract. The earlier deliver-after-close PR was not the best final shape because it forwarded all late events through `onEvent`, which reaches visible reply projection.
