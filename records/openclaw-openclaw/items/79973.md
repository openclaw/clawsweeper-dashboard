## Summary

Keep open: current main and the latest release still close the ACP turn event gate on timeout and skip later stream events before `onEvent` or error handling can observe them. A matching fix PR was found, but it was closed unmerged after review identified visible late-output risk, so the remaining work is a maintainer contract choice rather than cleanup.

## What This Changes

Report and preserve ACP runtime events or errors emitted after turn timeout instead of silently dropping them in `consumeAcpTurnStream`.

## Best Possible Solution

Choose a post-timeout ACP stream contract that preserves late error and terminal diagnostics through a filtered or non-visible path while keeping late user-visible output suppressed, then cover both paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: timeout closes `eventGate`, the stream loop keeps draining, and late events hit `continue` before delivery or error conversion; this pass did not run a live ACP timeout.

Unclear. The reported problem is real, but the best fix needs maintainer choice because the direct deliver-then-cancel attempt in https://github.com/openclaw/openclaw/pull/80051 could forward visible output after timeout.
