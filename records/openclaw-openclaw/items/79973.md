## Summary

Keep open: current main still has the reported ACP timeout behavior, the linked fix PR was closed unmerged, and the remaining fix needs an explicit ACP post-timeout delivery contract so late diagnostics do not leak visible output.

## What This Changes

The issue asks OpenClaw to stop silently dropping ACP runtime events and errors emitted after `consumeAcpTurnStream` closes its turn-timeout gate.

## Best Possible Solution

Define the ACP post-timeout stream contract, then preserve late terminal/error diagnostics through a filtered or non-visible path while keeping late visible output suppressed and covered by focused manager plus dispatch/projector tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close the turn timeout gate, let `runtime.runTurn()` yield `done` or `error`, and current main skips that event before `onEvent`, terminal tracking, or error conversion can observe it.

## Solution Assessment

Unclear until maintainers choose the contract. Capturing late errors is necessary, but the prior PR's deliver-all approach is not the narrowest safe solution because `onEvent` feeds visible reply projection.
