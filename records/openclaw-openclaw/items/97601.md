## Summary

Keep open: current main and v2026.6.10 still route fast auto-mode notices through chat-visible progress delivery, and the messages config schema has no knob to suppress only those notices. Because the requested fix adds a new config/default behavior surface, it needs maintainer product review before implementation.

## What This Changes

The issue requests a messages/progress configuration toggle to hide fast auto-mode transition notices from user chat while preserving diagnostics and status visibility.

## Best Possible Solution

Add one narrow fast-auto progress visibility policy under the existing messages/progress surface, with schema/help/docs and tests proving chat suppression while keeping status, logs, and explicit opt-in visibility intact.

## Reproduction Assessment

Yes at source level: current dispatch tests prove fast auto progress is delivered even when normal tool progress is suppressed. I did not run a live Signal/Telegram reproduction because this review is read-only.

## Solution Assessment

Unclear: a narrow messages/progress setting is plausible, but the exact public key, default, and streaming-callback semantics need maintainer decision under the repository's high bar for new config.
