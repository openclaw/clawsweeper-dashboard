## Summary

Keep open. Current main and the latest release still treat `fast-mode-auto` as special progress that can bypass ordinary tool-progress suppression, and there is no existing messages config key for the requested behavior; the exact fix needs maintainer product/API direction because it adds config surface that overlaps existing channel streaming/progress controls.

## What This Changes

Request adds a configuration-controlled way to keep fast auto-mode transition notices out of user-visible chat while preserving diagnostics/status visibility.

## Best Possible Solution

Maintainers should decide whether fast-auto visibility belongs under existing channel progress settings or a new narrow messages flag, then implement one shared dispatch policy with docs and channel regression coverage.

## Reproduction Assessment

Yes at source level: current tests and dispatch code show `fast-mode-auto` payloads are delivered despite default tool-progress suppression. I did not run a live Signal or Telegram turn in this read-only review.

## Solution Assessment

Unclear. Suppressing this by default may be desirable, but the best fix should first decide whether existing channel progress controls should own fast-auto visibility instead of adding another global messages key.
