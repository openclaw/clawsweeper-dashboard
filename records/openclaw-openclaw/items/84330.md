## Summary

Keep open: the current iMessage docs describe SSH/custom `cliPath` wrappers but do not document the incremental JSON-RPC stdio forwarding requirement that explains the reported timeout symptom.

## What This Changes

The issue asks for iMessage docs to warn that `cliPath` SSH/custom wrappers must forward JSON-RPC stdin/stdout incrementally without buffering.

## Best Possible Solution

Add a concise production note near the Remote Mac over SSH wrapper example, and optionally the config reference, stating that wrappers must act as transparent long-lived JSONL stdio pipes.

## Reproduction Assessment

Not applicable as a docs request. Source and upstream protocol inspection establish the line-delimited stdio contract and explain how buffered wrappers can cause the reported `chats.list` timeout symptom.

Fix https://github.com/openclaw/openclaw/issues/84330 with a docs-only PR.
