  - body: When `groupPolicy` is `open`, this returns before checking `channels.<id>.allowFrom`, so a supposedly restricted channel override is ignored and any sender is accepted. Resolve the channel override first and honor its allowlist before falling back to the account policy.
  - confidence: 0.9

  - body: The group send branch always posts to the account `incomingUrl`, so `channel:<name>` or `group:<name>` sends can land in the wrong channel when `channels.<id>.incomingUrl` is configured. Use the same channel-id/name/wildcard lookup as inbound replies for text and media sends.
  - confidence: 0.87

  - body: The target resolver accepts `channel:` and `group:` with a case-insensitive regex, but outbound delivery only checks lowercase `startsWith`, so `CHANNEL:general` is accepted as a target and then treated as a DM. Normalize or use a case-insensitive prefix check before branching.
