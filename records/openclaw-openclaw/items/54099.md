Full review comments:

- **[P1] Restore the sleep import before retry delays:** `extensions/synology-chat/src/client.ts:144`
  - body: `client.ts` calls `sleep()` in both the existing send path and the new channel retry path, but the PR head no longer imports it. TypeScript will fail with `Cannot find name 'sleep'` before the plugin can build or run.
  - confidence: 0.94
- **[P1] Validate channel file URLs before forwarding:** `extensions/synology-chat/src/client.ts:177`
  - body: `sendFileUrlToChannel` forwards the raw `fileUrl`, while the existing DM media path first runs `assertSafeWebhookFileUrl`. Group media sends need the same validation so the NAS cannot be asked to fetch private or otherwise blocked URLs.
  - confidence: 0.93
- **[P2] Apply channel allowlists before accepting open policy:** `extensions/synology-chat/src/group-access.ts:57-58`
  - body: This early return accepts every group sender whenever `groupPolicy` is `open`, so a stricter `channels.<id>.allowFrom` override never takes effect. Resolve the channel override before accepting the open default.
  - confidence: 0.9
- **[P2] Coerce allowlist entries before lowercasing:** `extensions/synology-chat/src/group-access.ts:86-87`
  - body: Raw config can contain numeric Synology IDs in `groupAllowFrom` or `channels.*.allowFrom`. Those entries reach `.toLowerCase()` here and can throw a `TypeError`, turning authorization into a 500 instead of an allow/deny decision.
  - confidence: 0.86
- **[P2] Match group target prefixes case-insensitively:** `extensions/synology-chat/src/channel.ts:353`
  - body: The target resolver accepts `channel:` and `group:` prefixes case-insensitively, but this send branch only checks lowercase prefixes. Inputs like `CHANNEL:general` can fall through to the DM path with `user_ids` instead of the channel webhook.
  - confidence: 0.86
- **[P2] Export the channel media helper from the client mock:** `extensions/synology-chat/src/channel.test-mocks.ts:76-80`
  - body: `channel.ts` imports `sendFileUrlToChannel`, but the shared `./client.js` Vitest mock only exports `sendMessage`, `sendToChannel`, `sendFileUrl`, and name resolution. Tests that load the plugin with this mock can fail during module initialization.
  - confidence: 0.88
- **[P2] Use the current runtime config snapshot:** `extensions/synology-chat/src/inbound-turn.ts:79`
  - body: This new inbound path calls deprecated `rt.config.loadConfig()`, while current main's Synology inbound event path uses `rt.config.current()`. Keep the prepared runtime config snapshot so plugin runtime tests and current architecture do not regress after rebase.
  - confidence: 0.82
