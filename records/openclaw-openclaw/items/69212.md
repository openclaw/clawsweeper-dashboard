## Summary

Keep open. Current main still lacks the PR's central HTTP continuation and OpenAI-compatible encrypted completions replay behavior, but the proposed patch is not ready because it can truncate context for proxy/custom Responses routes and leaves encrypted replay metadata outside the post-tool buffer budget.

## What This Changes

The PR adds HTTP Responses previous_response_id tool-only continuations, OpenAI-compatible completions encrypted_content capture/replay, and payload-shape tests for those paths.

## Best Possible Solution

Land a narrowed revision that enables previous_response_id only for native OpenAI/Codex Responses HTTP routes, keeps full-context payloads for unsupported proxy routes, captures/replays Volcengine completions encrypted_content, accounts encrypted metadata in buffer limits, and includes the required changelog and focused tests.

## Reproduction Assessment

Yes. A context ending with a same-route assistant responseId followed only by toolResult messages exercises the HTTP continuation path, and a completions stream with tool-call buffering plus encrypted-only reasoning metadata exercises the unbudgeted replay path.

## Solution Assessment

No. The direction is useful, but the current PR is not the best fix until continuation is gated to routes known to support previous_response_id and encrypted replay metadata is included in the existing memory budget.
