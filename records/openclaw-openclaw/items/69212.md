## What This Changes

The branch adds OpenAI-compatible completions `encrypted_content` capture/replay, Responses `previous_response_id` tool-only continuations, and payload-shape unit tests.

## Best Possible Solution

Revise the PR to gate stateful Responses continuations to supported native routes, count serialized encrypted replay metadata before buffering, and provide redacted runtime proof for the provider continuation path.

Yes. Source inspection gives high-confidence PR-head paths: a proxy `openai-responses` model with trailing `toolResult` messages exercises the ungated `previous_response_id` truncation, and an encrypted-only reasoning chunk during a tool call exercises the unbudgeted replay metadata path.

## Solution Assessment
