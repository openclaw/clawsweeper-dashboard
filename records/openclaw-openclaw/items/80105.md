## Summary

Keep open, but the patch is not mergeable as-is: it changes icon links to route-relative URLs without fixing the gateway path mapping that returns 404 for prefixed root-level assets, and the external PR is still missing real behavior proof.

## What This Changes

This PR changes Control UI favicon and apple-touch-icon hrefs from root-absolute to relative paths and adds a gateway test for serving favicon.svg under a configured basePath.

## Best Possible Solution

Fix the Control UI gateway/index serving path so known root public assets work under supported prefixes while preserving root-mounted SPA routing, static 404 hardening, and CSP hash behavior.

## Reproduction Assessment

Yes, source-level. With empty `gateway.controlUi.basePath`, `/__openclaw__/favicon.svg` resolves to a nested missing file and hits the static-extension 404 branch; with `basePath: "/mybase"`, the handler strips the prefix and serves `favicon.svg`, so the new test is not the reported failure path.

## Solution Assessment

No. Relative icon hrefs alone fix one configured-basePath symptom but reintroduce route-relative favicon requests for root-mounted nested SPA paths; the safer fix is a gateway-side mapping or CSP-preserving index rewrite with tests for the reported route.
