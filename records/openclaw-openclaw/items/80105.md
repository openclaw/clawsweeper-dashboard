## Summary

Keep open, but this patch should not merge as written: the linked report is the root-mounted prefixed asset path that current main already 404s, while this PR changes the HTML to relative icon URLs and only tests a configured-basePath route that is already served.

## What This Changes

The PR changes Control UI favicon and apple-touch-icon hrefs in `ui/index.html` from root-absolute to document-relative URLs and adds a gateway test for `favicon.svg` under a configured base path.

## Best Possible Solution

Fix root-level Control UI public asset resolution in the gateway, or use a CSP-preserving served-index rewrite, so configured `gateway.controlUi.basePath` and root-mounted prefixed SPA routes both load icons and manifest without weakening static 404 safety.

## Reproduction Assessment

Yes, source-level. Current main maps a no-basePath request like `/__openclaw__/favicon.svg` to a nested file path and then returns the static-asset 404 branch; `src/gateway/gateway-misc.test.ts` already asserts the same behavior for `/webchat/favicon.svg`.

## Solution Assessment

No. The PR makes the icon hrefs relative and tests a configured-basePath path that current main already serves, so the safer fix is a gateway/static-handler mapping or CSP-preserving index rewrite that covers the reported root-mounted prefix.
