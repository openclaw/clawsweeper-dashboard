## Summary

Current main now covers the reported Codex app-server liveness failure: merged progress-aware attempt/outer-lane timeout handling from https://github.com/openclaw/openclaw/pull/82601 combines with the raw-assistant completion release from https://github.com/openclaw/openclaw/pull/82403. The fix is not in the latest published v2026.5.12 release yet, so this should close as implemented on main rather than shipped.

## What This Changes

The issue asks the Codex app-server runner to distinguish live current-turn progress and completed assistant output from dead turns instead of aborting or falling back on a fixed timeout.

## Best Possible Solution

Keep the merged Codex app-server liveness fixes on main and ask for a new report only if the same failure reproduces on a build that includes https://github.com/openclaw/openclaw/pull/82601 and https://github.com/openclaw/openclaw/pull/82403.

## Reproduction Assessment

Yes. The issue has multiple live logs from `v2026.5.12`/beta builds plus a pre-fix source path where accepted Codex app-server turns could be governed by fixed or stale idle timers; current main now has focused regressions for the same progress and completion-stall cases.

## Solution Assessment

Yes. The merged solution is the narrow maintainable fix: only scoped current-turn progress resets the attempt and lane timers, unrelated app-server traffic does not, and completed raw assistant output has a bounded release path when `turn/completed` is missing.
