## Summary

Current main implements the Codex app-server liveness behavior reported here: current-turn activity now extends the attempt and outer lane timeout, while completed assistant output without `turn/completed` releases the session instead of wedging it. The fix is current-main-only relative to the latest released tag, so release users on `v2026.5.12` can still reproduce the older behavior until the next ship.

## What This Changes

The issue reports Codex app-server turns timing out or wedging after current-turn progress, including Telegram-visible stalls around completed items without a final `turn/completed`.

## Best Possible Solution

Keep the current-main Codex app-server watchdog and completed-output release behavior, and use the separate probe/status issue for any remaining operator diagnostics work.

## Reproduction Assessment

Yes. The issue has multiple live log reproductions, and current-main tests model the same source-level paths: progressing turns, non-turn traffic, completed assistant output without `turn/completed`, and silent accepted turns.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: progress is scoped to the active Codex turn, unrelated app-server traffic does not mask dead turns, and completed assistant output can release the session lane without a new config knob.
