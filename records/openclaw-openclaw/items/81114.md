## Summary

Current main now implements the central Codex app-server timeout fixes: active current-turn progress renews the attempt and outer lane watchdogs, non-turn traffic does not mask a stuck turn, and completed raw assistant output can release without waiting for a missing `turn/completed`. These fixes landed after the 2026.5.12 stable reports and shipped in `v2026.5.16-beta.3`.

## What This Changes

The issue asks OpenClaw to make Codex app-server turn timeouts distinguish dead turns from still-progressing or completion-stalled turns, especially under large context or Codex app-server finalization delays.

## Best Possible Solution

Keep the shipped progress-aware Codex app-server watchdog and raw-completion release behavior; open a new focused issue only if a post-`v2026.5.16-beta.3` live run still wedges or if the subagent announce-give-up cascade remains independently reproducible.

## Reproduction Assessment

Yes historically: the issue had live shipped-version logs and current regression tests now encode the old timeout and missing-terminal shapes. It is not reproducible as a current-main bug from source review because the relevant progress and completion paths are now implemented.

## Solution Assessment

Yes. The merged solution is the narrow maintainable path: track active-turn progress, reject unrelated traffic as liveness, carry progress into the lane timeout, and release completed assistant output without requiring a terminal app-server event.
