## Summary

Keep open. Current main still has the stale `chatRunId` queue-blocking path and the linked issue remains open, but this PR should not merge until it handles terminal `agent.wait` timeout snapshots, suppresses transient watchdog probe errors, and includes real browser/gateway behavior proof.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads history, flushes queued messages, and adds focused tests.

## Best Possible Solution

Land a rebased watchdog or replacement that treats authoritative terminal wait snapshots as finished, keeps transient background probe failures out of visible errors, preserves regression coverage, and includes redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, a missed terminal chat event can leave `chatRunId` set; then `isChatBusy` stays true and `flushChatQueue` exits before dequeuing the next message.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is a narrow maintainable fix, but this branch must handle terminal timeout/cancel snapshots and avoid visible errors from transient background probes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return `status: "timeout"` for terminal timeout/cancel snapshots when fields like `endedAt`, `stopReason`, `livenessState`, or `yielded` are present. Retrying every timeout response means an already-ended run can keep the watchdog looping forever and leave queued messages blocked.
  - confidence: 0.86
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog probe runs in the background, so assigning `host.lastError` when the short `agent.wait` request throws can show a visible error banner even though the main connection may still be healthy and the watchdog can retry.
  - confidence: 0.82
