## Summary

Keep open. The linked Control UI busy-state bug remains source-reproducible on current main and this PR is a relevant implementation candidate, but the patch is not merge-ready because it mishandles terminal `agent.wait` timeout snapshots and lacks required real browser/gateway proof.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale local run state, reloads history, flushes queued messages, and adds focused tests.

## Best Possible Solution

Land a rebased Control UI recovery path that treats authoritative `agent.wait` terminal metadata as finished, uses shared run-lifecycle reconciliation, suppresses silent probe noise, preserves regression coverage, and includes redacted live browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and the message is queued while `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

No, not as currently patched. A UI-side `agent.wait` recovery probe is a narrow maintainable direction, but it needs to honor terminal timeout metadata, use the shared run-lifecycle cleanup shape, and avoid visible background probe errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle terminal timeout snapshots as finished:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return an already-ended run as `status: "timeout"` with terminal metadata such as `endedAt`, `stopReason`, `livenessState`, or `yielded`. This branch treats every timeout as still active, so a timed-out or cancelled run can keep looping and leave the queued message blocked.
  - confidence: 0.88
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog is a silent 50 ms background recovery probe that retries automatically. Writing a thrown probe error into `host.lastError` can show a visible error banner even when the main gateway connection and run are otherwise healthy.
  - confidence: 0.82
