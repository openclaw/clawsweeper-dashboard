## Summary

Keep open. Current main still has the stale Control UI busy-latch path, but this patch needs correction before merge because it treats all `agent.wait` timeout results as active runs and lacks real browser/gateway proof.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads history, flushes queued messages, and adds focused regression tests.

## Best Possible Solution

Rebase or replace the watchdog on current main, treat terminal timeout snapshots as finished, keep transient watchdog probe failures out of visible error banners, retain focused regression coverage, and land with redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, leaving `chatRunId` non-null after a missed terminal event makes `isChatBusy` true, normal submit enqueues, and `flushChatQueue` returns early; I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is a narrow maintainable fix, but it must distinguish bare probe timeouts from terminal timeout snapshots and avoid surfacing transient background probe failures as user-visible errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` uses `status: "timeout"` for both a short probe with no terminal snapshot and terminal timeout snapshots that can include `endedAt`, `error`, or `stopReason`. This branch treats every timeout as still active, so a missed terminal event for an aborted or timed-out run can keep the watchdog retrying forever and leave queued messages blocked.
  - confidence: 0.86
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog runs silently in the background, so assigning `host.lastError = String(err)` when the short `agent.wait` probe throws can show a visible error banner even though the primary run may still be healthy and the watchdog will retry. Swallow, log, or store these probe failures in non-visible diagnostics instead.
  - confidence: 0.82
