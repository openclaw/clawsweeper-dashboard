## Summary

Keep open. Current main still has a source-reproducible stale `chatRunId` queue-blocking path, and this PR is a plausible implementation candidate, but the patch is not merge-ready because it treats every `agent.wait` `timeout` as active even though the gateway contract also uses `timeout` for terminal aborted/timeout snapshots.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads chat history, flushes queued messages, and adds focused regression tests.

## Best Possible Solution

Rebase or replace the watchdog on current main, distinguish non-terminal wait probe timeouts from terminal timeout snapshots, suppress transient probe failures from visible banners, keep focused regression coverage, and land with real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and the message stays queued because `isChatBusy` remains true and `flushChatQueue` returns early.

## Solution Assessment

Mostly yes for the direction, but not for the current patch as written. A UI-side `agent.wait` watchdog is narrow and maintainable, but it must respect the gateway timeout contract and avoid surfacing transient background probe failures as user-visible errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` uses `status: "timeout"` for both a short probe with no terminal snapshot and terminal aborted/timeout snapshots, often with metadata such as `endedAt` or `stopReason`. This branch treats every timeout as still active, so a missed terminal event for an aborted/timeout run can keep the watchdog retrying forever and leave queued messages blocked. Distinguish non-terminal probe timeouts from terminal timeout snapshots and clear/flush for the latter.
  - confidence: 0.86
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog runs silently in the background, so assigning `host.lastError = String(err)` when the short `agent.wait` probe throws can show a visible error banner even though the primary run may still be healthy and the watchdog will retry. Swallow, log, or store these probe failures in non-visible diagnostics instead.
  - confidence: 0.82
