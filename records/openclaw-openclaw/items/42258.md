## What This Changes

This PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads history, flushes queued messages, and adds controller tests.

## Best Possible Solution

Land a rebased Control UI recovery path that treats authoritative `agent.wait` terminal metadata as finished, uses current shared run-lifecycle cleanup, keeps silent probe errors out of visible UI errors, preserves regression coverage, and includes redacted live browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

No, not as currently patched. A UI-side `agent.wait` recovery probe is the right narrow direction, but this implementation must handle terminal timeout metadata, avoid visible background probe errors, and be rebased onto current run-lifecycle cleanup.
