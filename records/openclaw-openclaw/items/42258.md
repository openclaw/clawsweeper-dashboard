## Summary

Keep open. The linked bug is source-reproducible on current main and this PR is a plausible focused fix, but the branch is not merge-ready because it lacks real behavior proof and still has a P2 timeout-snapshot correctness gap.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads chat history, flushes queued messages, and adds focused regression tests.

## Best Possible Solution

Land a rebased watchdog or replacement that treats authoritative terminal `agent.wait` snapshots as finished, avoids surfacing transient probe failures as user-visible errors, retains focused regression coverage, and includes redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, leave `chatRunId` non-null after a missed terminal chat event, submit another message, and `flushChatQueue` returns while `isChatBusy` remains true.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is a narrow maintainable fix, but this branch must handle terminal timeout snapshots and keep transient background probe failures out of visible `lastError`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return `status: "timeout"` for terminal timeout snapshots when fields like `endedAt`, `error`, `stopReason`, `livenessState`, or `yielded` are present. Retrying every timeout response means an already-ended timed-out run can keep the watchdog looping forever and leave queued messages blocked.
  - confidence: 0.86
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog probe runs in the background, so assigning `host.lastError` when the short `agent.wait` request throws can show a visible error banner even though the main connection may still be healthy and the watchdog can retry.
  - confidence: 0.82
