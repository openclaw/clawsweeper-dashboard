## Summary

Keep open. The bug remains source-reproducible on current main, and this PR is a plausible narrow fix direction, but the current branch still needs real browser/gateway proof and a correctness fix for terminal `agent.wait` timeout snapshots before merge.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads history, flushes queued messages, and adds focused tests.

## Best Possible Solution

Land a rebased watchdog or equivalent that treats authoritative `agent.wait` terminal metadata as finished, keeps transient background probe failures out of visible UI errors, preserves regression coverage, and includes redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit another message, and current main will keep `isChatBusy` true so `flushChatQueue` returns before sending the queued item.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is a narrow maintainable fix, but this branch must handle terminal timeout/cancel snapshots and avoid surfacing transient background probe errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Treat terminal timeout snapshots as finished:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return an already-ended run as `status: "timeout"` with terminal metadata such as `endedAt`, `stopReason`, `livenessState`, or `yielded`. This watchdog treats every timeout as still active, so a timed-out or cancelled run can keep looping and leave the queued message blocked.
  - confidence: 0.87
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The probe is a silent 50 ms background recovery check that retries automatically. Writing a thrown probe error into `host.lastError` can show a visible error banner even when the main gateway connection and run are otherwise healthy.
  - confidence: 0.82
