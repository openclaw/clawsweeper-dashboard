## Best Possible Solution

Land a rebased Control UI recovery path that uses the existing `agent.wait` contract, treats terminal timeout metadata as finished, uses current run-lifecycle cleanup, suppresses transient probe errors, and includes redacted live browser/gateway proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently patched. A UI-side `agent.wait` recovery probe is the right narrow direction, but this implementation must handle terminal timeout metadata, avoid visible background probe errors, and be rebased onto current run-lifecycle cleanup.
