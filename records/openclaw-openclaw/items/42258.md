## Summary

Keep open. Current main still has the source-reproducible stuck-busy queue path, and this PR is useful but not ready because it misses terminal timeout/cancel wait snapshots and lacks after-fix real browser/gateway proof.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads history, flushes queued messages, and adds focused tests.

## Best Possible Solution

Land a rebased watchdog or replacement that treats authoritative terminal `agent.wait` snapshots as finished when appropriate, suppresses transient probe noise, preserves regression coverage, and includes redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is narrow and maintainable, but it must recover terminal timeout/cancel snapshots and avoid visible errors from transient background probes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return a terminal `status: "timeout"` snapshot with metadata such as `endedAt` or `stopReason`. Treating every timeout as active means an already-ended timeout/cancelled run can keep the watchdog retrying and leave the queued message blocked.
  - confidence: 0.86
