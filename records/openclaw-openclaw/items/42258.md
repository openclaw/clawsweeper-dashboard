## Summary

Keep open: current main still lacks a connected-client recovery path for a stale Control UI `chatRunId`, and the PR remains useful but currently has a correctness blocker plus a missing real-behavior-proof gate.

## What This Changes

The PR adds a Control UI chat-run watchdog that probes `agent.wait` after inactivity, clears stale run state, reloads chat history, flushes queued messages, and adds focused regression tests.

## Best Possible Solution

Land a rebased watchdog or equivalent Control UI recovery path that recognizes authoritative terminal `agent.wait` snapshots, avoids visible errors for transient background probes, keeps focused regression coverage, and includes redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level. On current main, a stale non-null `chatRunId` keeps `isChatBusy` true, and `flushChatQueue` returns before sending queued work when the terminal chat event is missed.

## Solution Assessment

Mostly, but not as currently patched. A UI-side `agent.wait` watchdog is a narrow maintainable fix, but the branch must recover terminal timeout snapshots and keep transient background probe failures out of visible `lastError`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recover terminal timeout wait results:** `ui/src/ui/app-chat.ts:84-85`
  - body: `agent.wait` can return `status: "timeout"` for snapshots that are nevertheless terminal via fields such as `endedAt`, `error`, `stopReason`, `livenessState`, or `yielded`. Retrying every timeout response means an already-ended timed-out run can keep the watchdog looping forever and leave queued messages blocked.
  - confidence: 0.86
- **[P3] Suppress transient watchdog probe errors:** `ui/src/ui/app-chat.ts:97-99`
  - body: The watchdog probe runs in the background, so assigning `host.lastError` when the short `agent.wait` request throws can show a visible error banner even though the main connection may still be healthy and the watchdog can retry.
  - confidence: 0.82
