## Solution Assessment

No, not as currently patched. A UI-side `agent.wait` recovery probe is a reasonable narrow direction, but this branch needs rebase, current wait-contract handling, transient-error cleanup, and real behavior proof before merge.

Full review comments:

- **[P2] Treat terminal timeout snapshots as finished:** `ui/src/ui/app-chat.ts:84`
  - body: `agent.wait` can return a terminal snapshot with `status: "timeout"` plus fields such as `endedAt`, `stopReason`, `livenessState`, or `yielded`. This branch treats every timeout as still active, so a timed-out completed run can keep retrying forever and leave the queued message blocked.
  - confidence: 0.9

  - body: The watchdog is a silent 50 ms background recovery probe that retries automatically. Writing a thrown probe error into `host.lastError` can show a visible error banner even when the main gateway connection and run are otherwise healthy.
  - confidence: 0.82
