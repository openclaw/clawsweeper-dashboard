## What This Changes

The PR records active gateway core method names in plugin runtime state, uses them to reuse compatible explicit gateway-bindable registries when helper/runtime callers omit handler maps, and adds regression tests for mismatched methods and default-mode non-inheritance.

## Best Possible Solution

Land a narrow version of this PR after maintainer review confirms the explicit gateway-bindable compatibility boundary and the regression tests remain green on current main.

## Reproduction Assessment

Yes. The code path is reproducible at the unit seam by setting an active gateway-bindable registry whose cache key includes core gateway handlers, then calling resolveRuntimePluginRegistry or getCompatibleActivePluginRegistry with allowGatewaySubagentBinding but without those handlers; current main cannot reconstruct that cache key.

## Solution Assessment

Yes. Persisting the sorted active gateway method names and using synthetic handlers only for explicit gateway-bindable cache-key comparison is the narrowest maintainable fix, provided the default-mode non-inheritance guard remains in place.
