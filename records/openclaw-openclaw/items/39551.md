## Summary

Keep open. Current main and v2026.4.27 still reject mixed `sessionKey` plus `label` inputs, so the reported user-visible bug remains and this PR is still relevant. The branch is not merge-ready as submitted because it expands mixed-selector/session-id resolution before denied-access response hardening, leaks gateway mock implementations between tests, and overlaps with open replacement path #74009.

## What This Changes

The PR changes `sessions_send` to treat `sessionKey` or `sessionId` as authoritative over redundant `label` and `agentId`, updates the tool description and changelog, and adds mixed-selector regression tests.

## Best Possible Solution

Land one canonical narrow fix after hardening denied session-id responses and fixing test isolation. The final behavior should prefer an explicit `sessionKey` target over redundant `label` and `agentId`, preserve label-only resolution and A2A/visibility checks, and document only the selector surface maintainers want to support.

## Reproduction Assessment

Yes. A `sessions_send` call with both non-empty `sessionKey` and `label` still hits the current-main guard at `src/agents/tools/sessions-send-tool.ts:109`, which is enough to reproduce the reported rejection path by source inspection.

## Solution Assessment

No as submitted. Preferring `sessionKey` is the right narrow direction, but the patch should first avoid resolved-key disclosure on denied mixed session-id sends, reset or restore persistent mocks in the new tests, and settle the `sessionId` alias choice against #74009.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid leaking resolved keys for denied mixed session-id sends:** `src/agents/tools/sessions-send-tool.ts:103-108`
  - body: By making a non-empty `sessionKey` or `sessionId` authoritative, mixed calls with a session-id-shaped selector now bypass the old early rejection and reach session-id resolution. If visibility or A2A policy then denies the send, the existing denial response returns the resolved `displayKey`, letting a caller confirm a session id and learn its canonical session key. Omit `sessionKey` or echo only the original caller-provided identifier on denied access before allowing this mixed-selector path.
  - confidence: 0.84
- **[P2] Reset the gateway mock between send-gating tests:** `src/agents/tools/sessions.test.ts:716`
  - body: The new tests install `callGatewayMock.mockImplementation(...)`, but the `sessions_send gating` `beforeEach` only calls `mockClear()`. That leaves the implementation in place for later tests in the describe block, which can mask or create failures depending on order. Reset or restore the mock implementation in the local `beforeEach` or after each test that overrides it.
  - confidence: 0.86
