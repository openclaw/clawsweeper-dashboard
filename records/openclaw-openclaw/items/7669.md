Keep open. Current main still does not provide a first-class npm `dev` dist-tag publish path, and the remaining work is a release/security policy decision rather than conservative cleanup.

## What This Changes

The issue asks to restore npm `dev` dist-tag publishing for moving `main` builds and verify that the documented development channel works.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit versioning, trigger policy, credential/provenance boundaries, tests, and registry verification, or intentionally retire npm dev snapshots and document `dev` as git-only/source-only.

## Reproduction Assessment

Yes for the source and registry gap: current main's release workflows, release-plan types, publish helper, and live npm dist-tags do not provide `openclaw@dev`. This is a feature/release-policy request rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired end state, but maintainers need to approve snapshot semantics and credential/provenance boundaries before implementation.
