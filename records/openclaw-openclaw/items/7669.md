## Summary

Keep this issue open: current main and the latest shipped release still have npm publish paths for `alpha`, `beta`, and `latest`, but no first-class npm `dev` dist-tag path, and the live registry still returns no `openclaw@dev`.

## What This Changes

The issue asks OpenClaw to re-enable or clarify npm `dev` dist-tag publishing for moving main-branch development releases.

## Best Possible Solution

Maintain one coherent preview contract: either document `dev` as git-only and use npm `alpha` for package previews, or deliberately implement/alias npm `dev` through the approved immutable snapshot release path with workflows, tests, verification, and docs aligned.

## Reproduction Assessment

Yes. The live registry currently has no `openclaw@dev`, and current main plus the latest release expose no workflow, publish-helper, or release-plan path that can publish `dev`.

## Solution Assessment

No, not as a direct auto-fix. Re-enabling an independent `dev` publisher is not clearly the best solution until maintainers decide how it should relate to the existing npm `alpha` channel and git-based `dev` updates.
