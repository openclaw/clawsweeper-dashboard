## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publish lane, the public docs still describe `dev` as an optional npm dist-tag, and the live npm registry only exposes `latest` and `beta` for `openclaw`.

## What This Changes

The issue asks to restore npm `dev` dist-tag publishing for moving-main builds and align CI verification plus docs with that behavior.

## Best Possible Solution

Maintainers should choose one policy and implement it end to end: either a guarded npm `dev` snapshot lane, or docs and release guidance that retire `openclaw@dev` as unsupported.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires an explicit release-policy and supply-chain decision; retiring the docs wording may be safer if moving-main npm snapshots are not intended.
