## Summary

Keep open. Current main still documents an optional npm `dev` dist-tag, but the public release workflows, publish wrapper, release-plan contract, tests, and live npm registry do not provide a first-class moving-main `openclaw@dev` publish path.

## What This Changes

The issue requests re-enabling npm `dev` dist-tag publishing for moving-main OpenClaw builds and verifying that the docs match the release behavior.

## Best Possible Solution

Maintainers should either design a first-class npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or remove npm `dev` from public docs and document the dev channel as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the narrow maintainable path requires a release-policy and supply-chain decision; retiring the npm wording may be safer if moving-main package snapshots are not intended.
