## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publish lane, and the remaining decision is release/supply-chain policy rather than conservative cleanup.

## What This Changes

Request to publish moving `main` builds to npm under the `dev` dist-tag, verify that tag, and align the development-channel docs.

## Best Possible Solution

Maintainers should choose one policy and carry it through: either a guarded npm `dev` snapshot lane with versioning, provenance, verification, and tests, or docs/release guidance that retires `openclaw@dev` as unsupported.

## Reproduction Assessment

Yes for the availability claim: current source inspection shows no `dev` publish path and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path needs an explicit release-policy and supply-chain decision; retiring the npm wording may be safer if moving-main snapshots are not intended.
