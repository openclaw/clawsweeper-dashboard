## Summary

Keep open. Current main still has no npm `dev` dist-tag publish lane and the live npm registry has only `latest` and `beta`; this is a release-policy and supply-chain decision rather than conservative cleanup.

## What This Changes

Request to restore an npm `dev` dist-tag distribution path for moving `main` builds and verify the CI, registry, and docs behavior.

## Best Possible Solution

Maintainers should choose one policy and carry it through: either a guarded npm `dev` snapshot lane with workflow, script, test, provenance, and registry verification support, or docs/release guidance that retires `openclaw@dev` as unsupported.

## Reproduction Assessment

Yes for the availability claim: current source inspection shows no `dev` publish path and `npm view openclaw dist-tags --json` shows no `dev` tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires an explicit release-policy and supply-chain decision; clarifying `dev` as git-only may be safer if moving-main snapshots are not intended.
