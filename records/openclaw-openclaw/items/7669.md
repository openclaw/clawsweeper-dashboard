## Summary

Keep open. Current `main` still lacks a first-class npm `dev` dist-tag publish path, while docs still describe `dev` as an optional npm dist-tag for moving `main`; the live npm registry also has no `dev` tag.

## What This Changes

Request to re-enable a moving-main npm `dev` distribution tag for OpenClaw releases and verify that `openclaw@dev` is published correctly.

## Best Possible Solution

Maintainers should either design and implement a first-class npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or explicitly retire npm `dev` wording and document dev installs as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path, and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the narrow maintainable path requires a release-policy and supply-chain decision; retiring the npm wording may be safer if moving-main package snapshots are not intended.
