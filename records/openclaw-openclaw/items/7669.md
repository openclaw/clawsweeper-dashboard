## Summary

Current main still has no npm `dev` publication path, and the live registry has no `dev` tag. A tagged alpha/nightly channel now exists, but it does not satisfy the requested moving-main behavior and no canonical item owns the remaining release-policy decision, so this issue remains necessary.

## What This Changes

Restore a moving-main npm `dev` dist-tag publication path, add publish verification, and align the development-channel documentation with actual availability.

## Best Possible Solution

Adopt one explicit release-owner-approved snapshot contract: either make tagged alpha/nightly the sole installable prerelease path and remove the npm `dev` promise from all user-facing surfaces, or implement a separately defined secure moving-main channel across release workflows, planning helpers, tests, registry verification, updater/install behavior, and docs.

## Reproduction Assessment

Yes. Current main has no real npm `dev` publish route, and the live npm registry has no `dev` dist-tag while the docs still mention it conditionally.

## Solution Assessment

No, re-enabling `dev` directly is not yet the best established solution. Maintainers first need to decide whether the newer tagged alpha/nightly channel supersedes moving-head npm snapshots or whether `dev` has distinct supported semantics.
