## Summary

Keep this issue open. Current `main` and the live npm registry still lack an npm `dev` snapshot path, no merged or open PR owns the work, and maintainers must decide whether to support moving-main packages or formally make the `dev` channel Git-only.

## What This Changes

Requests restoring an npm `dev` dist-tag that tracks moving `main` builds, with CI verification and accurate channel documentation.

## Best Possible Solution

Define one canonical `dev` contract: either implement a gated, provenance-preserving npm snapshot pipeline with immutable versions and registry verification, or remove the optional npm promise and document `dev` consistently as Git-only.

## Reproduction Assessment

Yes. The live npm registry has no `dev` tag, and current `main` has no workflow input, real-publish route, release-plan state, or helper behavior capable of producing one.

## Solution Assessment

No, not without a product decision. Directly re-enabling publication is not yet the clearly best solution because maintainers may instead choose the narrower and safer contract of making persistent `dev` explicitly Git-only.
