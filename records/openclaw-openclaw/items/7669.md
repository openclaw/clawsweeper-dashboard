## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publish lane, while the docs still describe `dev` as a possible npm dist-tag when published; resolving this requires release-policy and supply-chain ownership, not conservative cleanup.

## What This Changes

The issue asks OpenClaw to re-enable moving-main npm snapshot publishing under the `dev` dist-tag and verify that CI/CD, docs, and registry behavior match.

## Best Possible Solution

Maintainers should choose one policy and carry it through: either a guarded npm `dev` snapshot lane with versioning, provenance, verification, and tests, or docs/release guidance that retire `openclaw@dev` as unsupported.

## Reproduction Assessment

Yes for the availability claim: current source inspection shows no `dev` publish path and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires an explicit release-policy and supply-chain decision; retiring the npm wording may be safer if moving-main snapshots are not intended.
