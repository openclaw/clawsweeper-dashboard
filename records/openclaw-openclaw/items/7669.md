## Summary

Keep open. Current main still lacks an npm `dev` dist-tag publish path, the live npm registry has no `dev` tag, and the remaining decision is release/security policy rather than a narrow source bug.

## What This Changes

The issue requests a first-class npm `dev` dist-tag publish path for moving-main OpenClaw snapshots, plus verification and documentation alignment.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as git/source-only.

## Reproduction Assessment

Yes for the source and registry gap: current main exposes only alpha, beta, and latest in npm publish workflows/helpers, and the live registry has no dev dist-tag. This is a feature/release-policy gap, not a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm dev may be the right direction, but the maintainable solution needs owner approval for snapshot semantics and safe publishing before code should be queued.
