## Summary

Keep this issue open: current main and the live npm registry still lack the documented npm `dev` channel, while the newer Tideclaw `alpha` channel creates a release-policy question rather than resolving the request.

## What This Changes

Restore or explicitly retire the documented npm `dev` distribution channel for moving-main OpenClaw builds.

## Best Possible Solution

Define one canonical development-package policy across `dev` and Tideclaw `alpha`, then either implement securely versioned moving-main publication with end-to-end registry verification or remove npm-dev claims and document dev as a source-only channel.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and release-policy request; however, the reported gap is verified by current source and the live npm E404 for `openclaw@dev`.

## Solution Assessment

No, re-enabling a dist-tag alone is not the best fix; maintainers first need to decide whether `dev` complements or is superseded by `alpha`, then implement or retire the complete release contract consistently.
