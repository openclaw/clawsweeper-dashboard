## Summary

Keep open. Current main still documents an optional npm `dev` dist-tag, but the public release workflows, publish helper, release-plan contract, tests, and live npm registry do not provide a first-class moving-main npm `dev` publish path.

## What This Changes

Request to restore publishing moving-main OpenClaw builds to the npm `dev` dist-tag and add verification/docs alignment for that channel.

## Best Possible Solution

Either design and implement an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or retire npm `dev` snapshots from public docs and document dev as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path, and the live npm registry has no `dev` dist-tag. This is feature/release-infrastructure work rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires a maintainer release-policy and supply-chain decision; retiring the npm tag from docs may be the safer product direction.
