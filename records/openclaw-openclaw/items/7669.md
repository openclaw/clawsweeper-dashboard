## Summary

Keep open. Current main still documents optional npm `dev` snapshots, but the release workflows, publish helper, release-plan contract, tests, and live npm dist-tags do not provide `openclaw@dev`; this remains release-policy and supply-chain design work, not conservative cleanup.

## What This Changes

The issue requests restoring npm `dev` dist-tag publishing for moving main builds and aligning CI verification and docs.

## Best Possible Solution

Maintainers should either design and implement a first-class npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or explicitly retire npm `dev` snapshots and document dev as git-only.

## Reproduction Assessment

Yes. Source inspection on current main shows no workflow, helper, release-plan, or test path that can publish `openclaw@dev`, and live npm dist-tags for `openclaw` do not include `dev`.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires maintainer release-policy and supply-chain design first; retiring the npm dev tag from docs may be the safer product direction.
