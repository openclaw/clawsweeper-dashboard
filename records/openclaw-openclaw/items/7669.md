## Summary

Keep open. Current main still does not provide a first-class npm `dev` dist-tag publishing path, while docs still mention an optional npm `dev` tag and the live npm registry currently exposes only `beta` and `latest`.

## What This Changes

The issue asks to restore an npm `dev` dist-tag channel for moving-main OpenClaw builds and verify that documentation matches the published channel status.

## Best Possible Solution

Either implement a designed npm `dev` snapshot lane with workflow support, release-plan/publish-helper modeling, tests, registry verification, and docs, or deliberately retire npm `dev` from the docs and keep dev as git-only.

## Reproduction Assessment

Yes. Source inspection shows no workflow, helper, release-plan, or test path that can publish `openclaw@dev`, and the live npm registry dist-tags endpoint currently has no `dev` tag.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable solution needs release-policy design first; explicitly documenting dev as git-only may be the safer product direction.
