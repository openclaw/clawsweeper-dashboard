## Summary

Keep open. Current main still has no supported npm `dev` dist-tag publish path, the live registry only exposes `latest` and `beta`, and deciding whether moving-main npm snapshots should exist is release/security policy work.

## What This Changes

Request to re-enable an npm `dev` dist-tag release channel for moving-main OpenClaw builds, with CI/CD publishing and verification.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as source/git-only.

## Reproduction Assessment

Yes for the source and registry gap: current main lacks a `dev` publish path in workflows/helpers/types, and the live npm registry has no `openclaw@dev` dist-tag. This is a feature/release-policy gap rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired outcome, but maintainers need to approve snapshot semantics and safe publishing boundaries before implementation.
