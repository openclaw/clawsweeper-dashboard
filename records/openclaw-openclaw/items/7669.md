## Summary

Keep open. Current main still has no supported npm `dev` publish path: docs mention the optional tag, workflows/helpers/types only support alpha/beta/latest release-tag publishing, and the live npm registry still returns no `openclaw@dev`.

## What This Changes

The issue requests re-enabling an npm `dev` dist-tag for moving-main OpenClaw builds and verifying the release pipeline publishes it.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as source/git-only.

## Reproduction Assessment

Yes for the source and registry gap: current main lacks a `dev` publish path in workflows/helpers/types, and `npm view openclaw@dev` currently returns E404. This is a feature/release-policy gap rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired outcome, but the implementation needs maintainer approval for snapshot semantics and safe publishing boundaries before a fix PR is appropriate.
