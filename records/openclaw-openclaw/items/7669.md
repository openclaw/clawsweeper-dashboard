## Summary

Keep this open. Current main still has no npm `dev` dist-tag publish path, and deciding whether to restore or retire it is release-policy and supply-chain work rather than stale cleanup.

## What This Changes

Re-enable a moving-main npm `dev` dist-tag release channel, or update docs if that channel is no longer supported.

## Best Possible Solution

Release owners should either design and implement a first-class npm `dev` snapshot channel with explicit safety rules, or document `dev` as git/source-only and remove npm `dev` expectations.

## Reproduction Assessment

Yes. Source inspection shows current main exposes only `alpha`, `beta`, and `latest` through the npm release workflows/helpers, and the live npm registry has no `dev` dist-tag.

## Solution Assessment

Unclear. Re-enabling npm `dev` may satisfy the request, but the best maintainable path requires a maintainer decision on whether moving-main npm snapshots should exist and how they are published safely.
