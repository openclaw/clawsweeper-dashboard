## Summary

Keep this issue open: current main and the live npm registry still lack an npm `dev` dist-tag, while the newer `alpha` channel is a related but non-equivalent release path that requires an explicit maintainer policy decision.

## What This Changes

Request a moving-main npm `dev` dist-tag with CI publishing, post-publish verification, and matching documentation.

## Best Possible Solution

Maintain one coherent package-preview contract: either keep `dev` explicitly git-only and document npm `alpha` as the package preview, or deliberately map `dev` to the same approved immutable snapshot artifact, with workflows, installers, updater behavior, tests, and docs aligned.

## Reproduction Assessment

Yes. On June 12, 2026, the live registry lacked `openclaw@dev`, and current main exposes no workflow, publish-helper, or release-plan path capable of creating that tag.

## Solution Assessment

No, restoring an independent `dev` publisher is not yet demonstrably the best solution. Maintainers should first decide whether the shipped npm `alpha` channel replaces, aliases, or intentionally differs from the requested moving-main channel.
