## Summary

Keep open. Current main still has no npm `dev` dist-tag publish path, the live npm registry exposes only `beta` and `latest`, and this remains release-policy and supply-chain work rather than conservative cleanup.

## What This Changes

Request re-enabling npm `dev` dist-tag publishing for moving `main` builds, with CI/CD verification and documentation alignment.

## Best Possible Solution

Either design and implement an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or retire npm `dev` snapshots from public docs and document dev as git-only.

## Reproduction Assessment

Yes. Source inspection on current main shows no workflow, helper, release-plan, or test path that can publish `openclaw@dev`, and the live npm dist-tags endpoint has no `dev` tag.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable solution requires maintainer release-policy and supply-chain design first; documenting dev as git-only may be the safer direction.
