## Summary

Keep open. Current main and the latest shipped release still do not provide a supported npm `dev` dist-tag publish path, while the docs still mention the optional tag and the live npm registry has no `dev` tag for `openclaw`.

## What This Changes

The issue requests re-enabling npm `dev` dist-tag publishing for moving main builds and adding verification/docs alignment for that channel.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as source/git-only.

## Reproduction Assessment

Yes for the source and registry gap: current main lacks a `dev` publish path in workflows/helpers/types, and the live npm registry has no `openclaw@dev` dist-tag. This is a feature/release-policy gap rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired outcome, but the implementation needs maintainer approval for snapshot semantics and safe publishing boundaries before a fix PR is appropriate.
