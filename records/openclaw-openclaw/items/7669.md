## Summary

Keep open. Current main still does not have an npm `dev` dist-tag publish path, live npm has no `dev` tag for `openclaw`, and the remaining work is release/security policy rather than a narrow bug fix.

## What This Changes

The issue requests re-enabling npm `dev` dist-tag publishing for moving main builds and verifying that `openclaw@dev` is available.

## Best Possible Solution

Maintainers should either design a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as git/source-only.

## Reproduction Assessment

Yes for the source and registry gap: current main exposes only alpha, beta, and latest in npm publish workflows/helpers, and live npm has no `dev` dist-tag. This is a feature/release-policy gap, not a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the right product direction, but implementation should wait for maintainer approval of snapshot semantics and safe publishing boundaries.
