## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publish path, the live npm registry has no `dev` tag, and the prior maintainer review already identified this as unresolved release-policy work.

## What This Changes

Re-enable npm `dev` dist-tag publishing for moving `main` builds and add verification plus docs alignment.

## Best Possible Solution

Either design and implement an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or retire npm `dev` from public docs and document dev as git-only.

## Reproduction Assessment

Yes. Source inspection on current main shows no workflow, helper, release-plan, or test path that can publish `openclaw@dev`, and the live npm dist-tags endpoint has no `dev` tag.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable fix needs maintainer release-policy and supply-chain design first; documenting dev as git-only may be the safer direction.
