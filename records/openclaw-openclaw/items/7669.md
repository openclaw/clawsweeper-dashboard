## Summary

Keep open. Current main still documents an optional npm `dev` channel, but release workflows, publish helpers, and release-plan types do not support publishing a moving-main `dev` dist-tag, and the issue is already labeled for maintainer, product, and security review.

## What This Changes

The issue asks OpenClaw to re-enable npm `dev` dist-tag publishing for moving-main builds and verify that the documented development channel works.

## Best Possible Solution

Maintainers should either design and implement a first-class npm dev snapshot channel with explicit release/security rules, or retire npm dev snapshots and document dev as source/git-only.

## Reproduction Assessment

Yes for the source and registry gap: current main still lacks a `dev` publish path in workflows/helpers/types, and the live npm registry currently has no `dev` dist-tag. This is a feature/release-policy gap rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired outcome, but the implementation needs maintainer approval for snapshot semantics and safe publishing boundaries before a fix PR is appropriate.
