## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publish path: the public release workflows only expose `alpha`, `beta`, and `latest`, real OpenClaw npm publish still requires release tags, the publish helper and release-plan contract do not model `dev`, tests cover stable/alpha/beta only, and the live npm registry currently has only `beta` and `latest` for `openclaw`.

## What This Changes

Request re-enabling npm `dev` dist-tag publishing for moving `main` builds, with CI/CD verification and matching documentation.

## Best Possible Solution

Keep this issue open until maintainers either implement a designed npm `dev` snapshot lane for moving `main` builds or deliberately retire npm `dev` from the release-channel docs and keep dev as git-only.

## Reproduction Assessment

Yes. Source inspection shows no current-main workflow, helper, release-plan, or test path that can publish `dev`, and the live npm registry dist-tags endpoint currently returns only `beta` and `latest`.

## Solution Assessment

Unclear. Re-enabling npm `dev` could satisfy the request, but the best maintainable fix needs maintainer release-policy design first; the safer alternative may be to explicitly document dev as git-only.
