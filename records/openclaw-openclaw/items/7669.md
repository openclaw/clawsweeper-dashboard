## What This Changes

## Best Possible Solution

Maintainers should either design a first-class npm dev snapshot channel with explicit versioning, trigger policy, credential/provenance boundaries, tests, and registry verification, or intentionally document `dev` as git/source-only.

## Reproduction Assessment

Yes for the reported gap, though this is a feature request rather than a bug. Source inspection plus a live registry fetch show current main does not provide or publish `openclaw@dev`.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the desired end state, but maintainers need to approve snapshot semantics and credential/provenance boundaries first; documenting `dev` as git/source-only is the safer alternative if snapshots are not intended.
