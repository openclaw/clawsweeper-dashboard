## Summary

Keep open. Current main and the latest release still lack an npm `dev` publishing path, the live registry has no `dev` dist-tag, and choosing whether to publish moving-main snapshots requires an explicit release and security policy decision.

## What This Changes

Add a verified npm `dev` dist-tag publishing path for moving `main` builds and align the release-channel documentation with that behavior.

## Best Possible Solution

Adopt one explicit contract: either implement secured, uniquely versioned moving-main npm snapshots with provenance and registry verification, or retire the npm dev promise and consistently document dev as a git-only update channel.

## Reproduction Assessment

Yes. Current source has no dev publish plan, the latest release has the same limitation, and the live npm registry returns no `dev` dist-tag.

## Solution Assessment

Unclear until maintainers choose the product contract. Publishing snapshots is viable only with explicit versioning and security design; otherwise making dev consistently git-only is the narrower solution.
