## Summary

Keep open. Current main still has no first-class npm `dev` dist-tag publish path, while public docs still describe `dev` as an optional npm dist-tag and the live npm registry has only `latest` and `beta` for `openclaw`.

## What This Changes

Request to re-enable an npm `dev` dist-tag for moving-main OpenClaw development builds and add verification/docs coverage for that channel.

## Best Possible Solution

Maintainers should choose and document one policy: either implement a guarded npm `dev` snapshot lane end to end, or retire npm `dev` wording and keep development updates git/GitHub-main based.

## Reproduction Assessment

Yes for the availability claim: current source inspection shows no `dev` publish path, and the live npm registry has no `dev` dist-tag. As a release-infrastructure feature request, this is not a strict bug reproduction.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path needs an explicit release-policy and supply-chain decision; retiring the docs wording may be safer if moving-main npm snapshots are not intended.
