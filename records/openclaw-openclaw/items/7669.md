## Summary

Keep open. Current main still lacks a first-class npm `dev` dist-tag publishing path, the live npm registry has no `dev` tag, and prior Codex review comments already identified this as release/security policy work rather than a narrow source bug.

## What This Changes

Request to re-enable npm `dev` dist-tag publishing for moving `main` builds and add verification/docs coverage for that channel.

## Best Possible Solution

Maintainership should either design and implement a first-class npm `dev` snapshot channel with explicit versioning, triggers, provenance, verification, and docs, or retire npm `dev` snapshots and document `dev` as git/source-only.

## Reproduction Assessment

Yes for the source-level and registry gap: current main exposes only `alpha`, `beta`, and `latest` in the npm publish workflows/helpers, and the live npm registry has no `dev` dist-tag. This is a feature/release-policy gap, not a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the right product direction, but the maintainable solution needs an owner decision on whether moving-main npm snapshots should exist and how to publish them safely.
