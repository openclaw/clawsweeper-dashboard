## Summary

Keep this issue open for a maintainer release-policy decision. Current `main` and the live npm registry still lack a moving-main `dev` publication path, so the earlier Codex review remains accurate; implementing one requires an explicit supply-chain design rather than a routine release-script change.

## What This Changes

Request a first-class moving-main npm `dev` dist-tag publisher with CI verification and documentation aligned to its availability.

## Best Possible Solution

Define one maintainer-owned contract: either publish immutable, provenance-verified main snapshots under `dev` with registry verification and safe credential boundaries, or remove the optional npm snapshot promise while retaining the supported git-based `dev` channel.

## Reproduction Assessment

Not applicable as a bug reproduction, but the requested capability gap is verified with high confidence: current `main` has no publishable npm `dev` path and the live registry has no `dev` tag.

## Solution Assessment

Unclear until maintainers choose the release contract. Automatically publishing moving `main` is not the best solution unless immutable versions, provenance, credentials, triggers, verification, and rollback are designed together.
