## Summary

Keep this issue open: current main has alpha/beta/latest release publishing and git-based dev updates, but still no npm `dev` dist-tag publisher or live registry tag; the remaining work is a maintainer release-channel and supply-chain policy decision.

## What This Changes

Requests re-enabling an npm `dev` dist-tag publishing path for moving-main builds, plus verification and docs alignment.

## Best Possible Solution

Maintain one coherent preview-channel contract: either retire npm `dev` and document git dev plus npm `alpha`, or implement `dev` as an approved first-class package channel with versioning, workflow permissions, provenance, verification, updater behavior, and docs aligned.

## Reproduction Assessment

Yes. `npm view openclaw@dev version` returns E404, and current main has no workflow, publish-helper, or release-plan path that can create a `dev` npm dist-tag.

## Solution Assessment

No. Re-enabling an independent npm `dev` publisher is not yet proven to be the best fix; maintainers should first decide whether npm `alpha` replaces, aliases, or intentionally differs from the requested moving-main dev channel.
