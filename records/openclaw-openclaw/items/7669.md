## Summary

Keep open. Current main still documents an optional npm `dev` dist-tag, but the release workflows, publish helper, release-plan contract, tests, and live npm registry do not provide a first-class moving-main `openclaw@dev` publish path.

## What This Changes

Re-enable npm `dev` distribution for moving `main` snapshots and add publish verification plus matching release-channel docs.

## Best Possible Solution

Maintainers should either design an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, registry verification, and tests, or remove npm `dev` from public docs and document the dev channel as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path, and live npm checks show no `dev` dist-tag. This is feature/release-infrastructure work rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires a release-policy and supply-chain decision; retiring the npm tag from docs may be safer if snapshots are not intended.
