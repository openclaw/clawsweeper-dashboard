## Summary

Keep open. Current main still has no npm `dev` dist-tag publish path, the live npm registry exposes only `latest` and `beta`, and the remaining work is release-policy and supply-chain design rather than conservative cleanup.

## What This Changes

The request asks OpenClaw to publish moving `main` snapshots under the npm `dev` dist-tag and verify/document that channel.

## Best Possible Solution

Either design and implement an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, tests, registry verification, and docs, or retire npm `dev` snapshots from public docs and document dev as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path, and the live npm dist-tags endpoint has no `dev` tag. This is still a feature/release-infrastructure request rather than a broken existing bug contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the maintainable path requires a maintainer release-policy and supply-chain decision; retiring the npm tag from docs may be the safer product direction.
