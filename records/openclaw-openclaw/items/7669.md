## Summary

Keep open. Current main still documents an optional npm `dev` dist-tag, but the release workflows, publish wrapper, release-plan contract, tests, and live npm registry still do not provide a first-class moving-main `openclaw@dev` publish path.

## What This Changes

The issue requests re-enabling npm `dev` dist-tag publishing for moving `main` builds and verifying that `openclaw@dev` resolves from npm.

## Best Possible Solution

Maintainers should either design an explicit npm `dev` snapshot lane with workflow, release-plan, publish-helper, registry verification, and tests, or remove npm `dev` from public docs and document the dev channel as git-only.

## Reproduction Assessment

Yes for the availability claim: source inspection shows no `dev` publish path, and the live npm registry has no `dev` dist-tag. As a feature/release-infrastructure request, this is not a bug reproduction in the strict sense.

## Solution Assessment

Unclear. Re-enabling npm `dev` would satisfy the request, but the narrow maintainable path requires a release-policy and supply-chain decision; retiring the npm tag from docs may be safer if moving-main snapshots are not intended.
