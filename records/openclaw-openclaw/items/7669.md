## Summary

Keep this issue open. Current main still has no npm `dev` publication path, the live registry has no `dev` tag, and the release-channel docs still ambiguously mention that tag while implementing `dev` as a git-backed update channel; resolving that contract requires an explicit maintainer decision.

## What This Changes

Add a moving-main npm `dev` distribution tag with publishing, verification, and documentation support.

## Best Possible Solution

Establish one canonical channel contract: either securely publish uniquely versioned main snapshots under `dev`, or remove the npm-dev promise throughout the docs and define `dev` exclusively as the existing git-backed update channel.

## Reproduction Assessment

Yes. Current main mentions an npm `dev` tag, its release workflow cannot publish that tag, and the live `openclaw@dev` lookup returns E404.

## Solution Assessment

Unclear pending maintainer policy. An npm snapshot lane would satisfy the request, but making the already-supported git-backed channel the exclusive documented `dev` path is narrower and avoids broadening the supply chain.
