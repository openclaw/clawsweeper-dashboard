## Summary

Keep this issue open: current main still does not publish or verify an npm `dev` dist-tag for moving main builds, while the docs still mention that optional npm tag and the live registry has no `dev` tag.

## What This Changes

The issue asks OpenClaw to re-enable npm `dev` dist-tag publishing for moving main-branch builds and verify the docs match the release behavior.

## Best Possible Solution

Maintainers should either design and implement a secure first-class npm `dev` snapshot channel for moving main builds, or explicitly retire npm `dev` snapshots and document `dev` as git-only.

## Reproduction Assessment

Yes. Current main lacks a `dev` npm publish path in workflow, helper, release-plan, and tests, and the live npm registry has no `dev` dist-tag.

## Solution Assessment

Unclear. Adding npm snapshots may be the right fix if maintainers want that channel, but updating docs to make `dev` git-only is the safer outcome if they do not want a mutable main-branch npm publish path.
