## Summary

Keep open. Current main still has no first-class npm `dev` dist-tag publish path, the live npm registry has no `dev` tag, and the remaining work is release/security policy rather than a narrow bug fix.

## What This Changes

The issue asks to re-enable npm publishing for moving `main` snapshots under the `dev` dist-tag and verify the CI/CD and docs contract.

## Best Possible Solution

Maintainership should either design a first-class npm `dev` snapshot channel with explicit versioning, triggers, provenance, verification, and docs, or retire npm `dev` snapshots and document `dev` as git/source-only.

## Reproduction Assessment

Yes for the source-level gap: current main exposes only `alpha`, `beta`, and `latest` in the npm publish workflows/helpers, and the live npm registry has no `dev` dist-tag. This is a feature/release-policy gap rather than a broken existing command contract.

## Solution Assessment

Unclear. Re-enabling npm `dev` may be the right direction, but the maintainable solution needs an owner decision on whether moving-main npm snapshots should exist and how to publish them safely.
