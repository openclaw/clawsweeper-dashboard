## Summary

Keep open, but the current patch is not merge-ready: it disables Telegram transcript mirroring wholesale, while current source and tests treat that mirror as the final-session update path for Telegram deliveries.

## What This Changes

The PR sets Telegram deliveryBaseOptions.transcriptMirror to undefined so Telegram delivery no longer writes delivery-mirror assistant entries to session transcripts.

## Best Possible Solution

Preserve Telegram transcript mirroring, but make the mirror append dedupe-aware against an existing canonical assistant row and cover both duplicate suppression and mirror preservation in tests.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the duplicate-risk path clearly, but I did not run a real Telegram/WebUI session or a failing regression test in this read-only review.

## Solution Assessment

No. Disabling the whole mirror is too broad; the maintainable fix is to keep the Telegram mirror and dedupe the append when a matching canonical assistant message already exists.
