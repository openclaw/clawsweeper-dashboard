The PR marks native command aliases in the registry, carries that marker into Telegram menu budgeting, prioritizes canonical commands before aliases, strips alias metadata from Telegram payloads, and adds focused registry/menu tests.

## Best Possible Solution

Land one narrow canonical-first alias-budgeting fix after maintainers confirm the alias metadata contract, then close the linked bug and supersede the broader alternate PR.
