## Summary

Keep open. ## What This Changes

The issue requests a provider-aware fix for `github-copilot/claude-opus-4.7` so non-`medium` adaptive `output_config.effort` values do not trigger Copilot 400s and auth-profile cooldown cascades.

## Best Possible Solution

Add an explicit provider/model adaptive-effort policy for `github-copilot/claude-opus-4.7`, preferably table-driven in the Anthropic payload path, with tests covering low/minimal/high/xhigh and preserving direct Anthropic Opus 4.7 behavior.

## Solution Assessment

Unclear. Omitting or forcing `medium` matches this issue's report, but the open related PR makes a conflicting live-capability claim, so maintainers should confirm the current Copilot surface before choosing omit, clamp, or capability-table behavior.
