## Summary

Keep open. Current main still builds Anthropic adaptive-thinking payloads from the model ID alone and emits `output_config.effort` without a GitHub Copilot provider/model compatibility rule; the current Copilot catalog default of `reasoning: false` mitigates bundled defaults but does not fix the reported reasoning-enabled payload path. The open related PR only handles low/minimal and conflicts with this issue's high/xhigh report, so this should stay open for maintainer reconciliation rather than close as implemented or duplicate.

## What This Changes

The issue asks OpenClaw to stop sending unsupported adaptive `output_config.effort` values for `github-copilot/claude-opus-4.7` so non-medium thinking does not 400 and poison Copilot auth profiles.

## Best Possible Solution

Add a provider/model-specific adaptive-effort compatibility rule for `github-copilot/claude-opus-4.7`, ideally as a small capability table consulted by the Anthropic request builder or Copilot wrapper, and cover low/minimal/high/xhigh while preserving direct Anthropic Opus 4.7 behavior.

## Reproduction Assessment

Yes. A source-level reproduction is a reasoning-enabled `github-copilot/claude-opus-4.7` transport request with `/think high` or `xhigh`; current main maps that to `output_config.effort` and emits it without a Copilot provider gate, and the issue provides live 400 logs for that payload.

## Solution Assessment

Unclear. A narrow provider gate or clamp would likely unblock the report, but the safest maintainer path is to reconcile this issue with #70322/#70584 and encode one explicit provider/model effort policy instead of merging a partial low/minimal-only rule.
