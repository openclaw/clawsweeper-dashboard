## Summary

Keep open because current main and the latest release still emit provider-agnostic `output_config.effort` for GitHub Copilot Claude Opus 4.7 adaptive-thinking requests, so the reported high/xhigh rejection is not fixed.

## What This Changes

The issue asks OpenClaw to stop sending unsupported non-medium adaptive-thinking `output_config.effort` values to the GitHub Copilot Claude Opus 4.7 proxy and avoid the resulting fallback/cooldown cascade.

## Best Possible Solution

Add provider-aware Copilot Opus 4.7 adaptive-effort compatibility, ideally from preserved supported-effort capabilities or a small provider/model table, with regression coverage for low, minimal, high, and xhigh while leaving direct Anthropic behavior unchanged.

## Reproduction Assessment

Yes. A `github-copilot/claude-opus-4.7` request with high or xhigh reasoning can be source-traced on current main to `output_config.effort: "high"` or `"xhigh"`, and the issue supplies live Copilot 400 logs for that payload.

## Solution Assessment

Yes for provider-aware transport-boundary normalization or a supported-effort capability table. The open low/minimal-only model-level clamp is not the best complete fix because it can change direct Anthropic behavior and leave this issue's high/xhigh path unresolved.
