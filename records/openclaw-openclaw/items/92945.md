## Summary

Keep open: Mantis now supplies real Telegram proof for the default menu recovery, but the patch is still too narrow because OpenClaw writes default, group, and localized command states while the new skip check verifies only the default command list.

## What This Changes

The PR changes Telegram command-menu sync so a matching local command hash calls `getMyCommands()` and only skips re-registration when Telegram reports existing remote commands.

## Best Possible Solution

Keep the command-hash optimization, but only skip sync after verifying every Telegram command scope and language variant OpenClaw writes, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows that after the process-local hash is cached, this patch can return when default remote commands are non-empty even if the `all_group_chats` scope or localized variants are empty.

## Solution Assessment

No. The patch is in the right helper, but the best fix must query or compare every scope and language variant OpenClaw writes before honoring the cached hash.
