## Summary

Keep open: this is a focused Telegram command-menu bug-fix attempt, but the new remote-state check only proves the default command scope is non-empty while current OpenClaw writes more scopes, and the supplied proof is source-level only.

## What This Changes

The PR changes Telegram command menu sync to call `getMyCommands()` before skipping registration when the local command hash already matches.

## Best Possible Solution

Keep the command-hash optimization, but only skip sync after verifying every Telegram command scope and language variant OpenClaw writes, with focused regression coverage and redacted real Telegram proof.

## Reproduction Assessment

Yes from source, but not from live Telegram proof. Once the process-local hash matches, current main skips remote reconciliation, and the PR's new path can be exercised with a non-empty default command list while another written scope is empty.

## Solution Assessment

No. The patch is in the right helper, but checking only unscoped `getMyCommands()` is too narrow because the helper writes default, group, and localized command entries.
