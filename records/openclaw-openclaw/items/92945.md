## What This Changes

The PR calls `getMyCommands()` before using the cached Telegram command hash to skip command-menu synchronization.

## Best Possible Solution

Keep the command-hash optimization, but only skip sync after verifying every Telegram command scope OpenClaw writes, with focused regression coverage and real Telegram proof.

## Reproduction Assessment

The linked stale-file/restart report is already fixed on main, so this PR is about a narrower residual failure mode.

## Solution Assessment

No. The patch is in the right helper, but checking only the default remote command list is not the best fix because the helper writes multiple Telegram command scopes before caching the hash.
