## Summary

Keep this PR open, but it still needs changes before merge. The patch is in the right Telegram command-sync helper and has useful visual proof for the default menu recovery path, but the new cached-hash skip only verifies the default Telegram command list even though OpenClaw writes group-scope and localized command states too.

## What This Changes

The PR changes Telegram command-menu sync so a matching local command hash calls `getMyCommands()` before deciding whether to skip `setMyCommands()`.

## Best Possible Solution

Keep the command-hash optimization, but only skip sync after verifying every Telegram command scope and language variant OpenClaw writes, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows that after the process-local hash is cached, this patch can return when default remote commands are non-empty even if the `all_group_chats` scope or localized variants are empty.

## Solution Assessment

No. The patch is in the right helper, but the best fix must query or compare every scope and language variant OpenClaw writes before honoring the cached hash.
