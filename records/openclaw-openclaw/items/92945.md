## Summary

Keep this PR open: the latest head fixes the prior default-vs-group scope gap, but the cached-hash shortcut still verifies only base command scopes while current Telegram sync also writes per-language command variants.

## What This Changes

The PR changes Telegram command-menu sync to call `getMyCommands()` for each configured base command scope before honoring the cached local command hash.

## Best Possible Solution

Keep the command-hash optimization, but skip sync only after verifying every command state the helper writes, including base scopes and each supported `language_code`, with focused cached-hash regression tests.

## Reproduction Assessment

Yes from source inspection: populate the cached hash, keep base default and group commands non-empty, clear one `language_code` command variant, and the PR returns after base-scope `getMyCommands()` calls without re-registering the localized menu.

## Solution Assessment

No. The PR is in the right helper and covers the prior group-scope gap, but the best fix must verify the same scope-plus-language command states that `setMyCommands` writes before honoring the cache.
