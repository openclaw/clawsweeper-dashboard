## Best Possible Solution

Land an updated PR or successor that adds a schema-backed per-agent `enabled` opt-out, suppresses fallback work for explicitly disabled agents, updates the current schema/docs surface, covers mixed/all-disabled/schema cases, and splits schedule support if maintainers want it out of scope.
