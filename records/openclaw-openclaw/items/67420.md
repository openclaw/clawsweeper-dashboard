## Summary

Keep open. Current main still lacks per-agent dreaming control, and this PR is the active implementation path, but it is not merge-ready because explicit all-disabled configs can still fall through to default workspace dreaming, real behavior proof is missing, and the linked feature request still needs maintainer scope judgment for per-agent schedules.

## What This Changes

The PR adds `agents.list[].dreaming.enabled` to config types/schema/docs and filters disabled agents out of `resolveMemoryDreamingWorkspaces()`, with a focused mixed-agent unit test.

## Best Possible Solution

Land this PR or a successor after adding a documented, schema-backed per-agent `enabled` contract, preserving no-work semantics for explicitly all-disabled configs, and splitting per-agent schedules into an explicit design or follow-up if they remain in scope.

## Reproduction Assessment

Yes by source inspection. In the PR, filtering all configured agents leaves `agentIds` empty and the existing fallback paths can still enqueue default or primary workspace dreaming work; I did not run tests because this review is read-only.

## Solution Assessment

No. The requested per-agent opt-out is useful, but this patch needs explicit all-disabled semantics and maintainer scope judgment for schedule support before it is the narrow maintainable solution.
