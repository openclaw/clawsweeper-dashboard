## Summary

Keep open. Current main still lacks the per-agent dreaming config surface, and this external PR is relevant but not merge-ready because explicit opt-outs can still fall through to default/primary workspace fallback and no after-fix real behavior proof is present.

## What This Changes

The PR adds `agents.list[].dreaming.enabled` to docs, config types/schema, a generated schema artifact, memory dreaming workspace resolution, and a mixed enabled/disabled unit test.

## Best Possible Solution

Keep this open and land an updated PR or successor with a documented schema-backed `agents.list[].dreaming.enabled` contract, explicit no-work semantics for disabled agents, focused tests for mixed/all-disabled/validation cases, and a separate maintainer decision for per-agent schedules.

## Reproduction Assessment

Yes, by source inspection. Current main lacks `agents.list[].dreaming`, and the PR's new skip guard can still fall through to default-agent or primary-workspace fallback when every configured agent opts out.

## Solution Assessment

No. The enable/disable surface is useful, but the proposed implementation needs explicit all-disabled semantics, current-main schema alignment after rebase, and maintainer scope judgment for schedules before it is the narrow maintainable solution.
