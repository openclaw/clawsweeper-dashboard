## Summary

Keep open. Current main and v2026.6.10 still lack per-agent dreaming control, but this PR is not merge-ready because all-disabled opt-outs can still fall back into dreaming, the new config surface needs maintainer direction, and the external PR has no real behavior proof yet.

## What This Changes

The PR adds `agents.list[].dreaming.enabled` to config types, strict schema, generated schema, docs, and memory dreaming workspace resolution so individual agents can opt out of scheduled dreaming.

## Best Possible Solution

Keep this PR open but require a branch update that makes explicit opt-out suppress fallback dreaming, adds all-disabled and config-validation coverage, and either confirms this config location or rebases onto the maintainer-approved memory override model.

## Reproduction Assessment

## Solution Assessment

No. A per-agent opt-out is a plausible mitigation, but this patch is not the best fix until all-disabled opt-out semantics and the durable config location are resolved.
