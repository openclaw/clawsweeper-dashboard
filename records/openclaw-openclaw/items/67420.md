## What This Changes

The PR adds an `agents.list[].dreaming.enabled` config field to types/schema/docs and filters disabled agents from memory dreaming workspace resolution, with one mixed-agent unit test.

## Best Possible Solution

Land an updated PR or successor that adds a schema-backed per-agent `enabled` opt-out, suppresses fallback work for explicitly disabled agents, updates the current config/schema/docs surface, and splits per-agent schedules unless maintainers choose to include that larger scope now.

## Reproduction Assessment

Yes. By source inspection, a config where `main` or every agent has `dreaming.enabled: false` can empty the PR's filtered list before current default-agent and primary-workspace fallback paths add work again.

## Solution Assessment

No. The schema-backed enable flag is a plausible narrow direction, but the patch needs disabled-agent fallback semantics, current-main schema updates, real behavior proof, and a maintainer decision on schedule scope before it is the best solution.
