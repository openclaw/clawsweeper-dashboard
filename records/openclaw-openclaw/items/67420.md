## Summary

## What This Changes

This PR adds an `agents.list[].dreaming.enabled` opt-out across agent config types, strict schema, generated schema, memory docs, and `resolveMemoryDreamingWorkspaces()` with a mixed-agent unit test.

## Best Possible Solution

Land a refreshed implementation only after maintainers confirm the config shape, fix all-disabled fallback semantics, update the current schema/docs flow, and show real proof that disabled agents do not run dreaming.

## Reproduction Assessment

## Solution Assessment

No, not as submitted. A per-agent opt-out is a plausible narrow mitigation, but the best fix must preserve true all-disabled opt-out semantics, refresh the schema path for current main, and get maintainer agreement on whether schedules belong in this API.
