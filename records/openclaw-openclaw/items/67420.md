## Summary

Keep open. Current main and v2026.6.10 still lack per-agent dreaming control, and this PR is a useful candidate for https://github.com/openclaw/openclaw/issues/67413, but it is not merge-ready because explicit all-agent opt-out can still fall back to the default workspace, the config API is product-sensitive, the branch is conflicting, and real behavior proof is missing.

## What This Changes

Adds an `agents.list[].dreaming.enabled` config key across agent types, strict/generated schemas, docs, and the dreaming workspace resolver, plus a mixed enabled/disabled unit test.

## Best Possible Solution

Keep the PR open but require a refreshed implementation or successor that uses the maintainer-approved memory config path, makes explicit all-agent opt-out produce no dreaming work, preserves default fallback only when no valid agents are configured, and includes schema, docs, tests, and real behavior proof.

## Reproduction Assessment

Yes for the patch defect: source inspection shows that a config where every valid agent has `dreaming.enabled: false` leaves `agentIds` empty and then falls back to `resolveDefaultAgentId(cfg)`. I did not run a live OOM or gateway scenario, and the PR has no after-fix runtime proof.

## Solution Assessment

No. Per-agent opt-out is a plausible mitigation, but this branch must fix all-disabled semantics and get maintainer agreement on the public config path before it is the best solution.
