## Summary

Keep this PR open: current main still lacks per-agent dreaming opt-out, and the PR is on the right surface, but the proposed skip guard can still run dreaming for the default agent when every configured agent opts out. The PR also needs the required changelog entry for a user-facing config feature.

## What This Changes

This PR adds `agents.list[].dreaming.enabled` to config types, strict Zod schema, generated schema metadata, memory docs, and `resolveMemoryDreamingWorkspaces()`, with a mixed enabled/disabled unit test.

## Best Possible Solution

Update this PR or land a narrow successor that keeps the config contract, generated schema, docs, and resolver aligned; preserves default-agent fallback only when no valid agents are configured; treats every explicitly disabled valid agent as a real opt-out; and covers mixed, all-disabled, and schema-validation cases. Per-agent schedules can remain tracked by #67413 if maintainers want this PR scoped to enable/disable only.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a config whose only valid agent, or all valid agents, set `dreaming.enabled: false`; the proposed guard empties `agentIds`, then the resolver fallback adds the default agent and can still run the memory-heavy sweep.

## Solution Assessment

No. The PR direction is maintainable, but the implementation should distinguish `no configured agents` from `configured agents all opted out` before applying default-agent or workspace fallbacks.
