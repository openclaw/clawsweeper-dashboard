## Summary

Keep open. This is the active implementation path for https://github.com/openclaw/openclaw/issues/67413 and current main still lacks per-agent dreaming control, but the patch is not merge-ready because explicit all-disabled configs can still fall through to dreaming work and the external PR lacks real behavior proof.

## What This Changes

Adds `agents.list[].dreaming.enabled` to agent config types/schema/docs/generated schema and filters disabled agents in `resolveMemoryDreamingWorkspaces`, with a mixed enabled/disabled unit test.

## Best Possible Solution

Keep this PR open or land a successor that adds a documented, schema-backed per-agent `enabled` contract, returns no work for explicit all-disabled configs, and splits per-agent scheduling into a deliberate follow-up if needed.

## Reproduction Assessment

Yes by source inspection. In the PR, an all-disabled `agents.list` leaves `agentIds` empty after the new guard, and the existing default-agent plus primary-workspace fallback paths can still enqueue dreaming work; I did not run tests because this review is read-only.

## Solution Assessment

No. The per-agent enable/disable surface is useful, but the implementation needs explicit all-disabled semantics and current-main schema alignment before it is the narrow maintainable solution.
