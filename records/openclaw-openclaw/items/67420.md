## Summary

Keep open for maintainer review. Current main still lacks per-agent dreaming control, and this branch is a plausible narrow attempt, but it has a blocking all-disabled fallback bug, no real behavior proof, a stale/conflicting schema surface, and unresolved public config API direction.

## What This Changes

The branch adds `agents.list[].dreaming.enabled` to the agent config type/schema/docs and filters disabled agents out of memory dreaming workspace resolution with a focused helper test.

## Best Possible Solution

Land a maintainer-approved, schema-backed per-agent dreaming API that treats all-disabled configs as a full opt-out, preserves fallback only when no agents are configured, avoids resurrecting retired generated schema files, and includes validation/runtime tests plus real behavior proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: after the added guard filters every configured agent, the unchanged fallback can still choose the default or primary workspace. I did not run a live multi-agent OOM scenario.

## Solution Assessment

No. The narrow opt-out is plausible, but the best fix must distinguish no configured agents from all configured agents opting out, refresh the branch against current schema generation, and settle the public config API direction.
