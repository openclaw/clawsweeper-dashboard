## What This Changes

The PR adds `agents.list[].dreaming.enabled` to config types/schema/docs and filters disabled agents in `resolveMemoryDreamingWorkspaces`, with one mixed-agent unit test.

## Best Possible Solution

Land an updated PR or successor that adds a schema-backed per-agent `enabled` opt-out, suppresses fallback work for explicitly disabled agents, updates the current config/docs/schema surface, and splits per-agent schedules unless maintainers choose to include that larger scope now.

## Solution Assessment

No. The schema-backed enable flag is a plausible narrow direction, but the patch needs disabled-agent fallback semantics, current-main schema updates, real behavior proof, and a maintainer decision on schedule scope before it is the best solution.
