## Summary

Keep open: current main still has a source-visible gap where channel CLI failures can leave the persisted Claude CLI binding untouched, while existing cleanup only covers successful metadata, selected fresh-retry paths, or the separate agent-command path.

## What This Changes

The issue asks OpenClaw to clear stale `claude-cli` session IDs from persisted session state when a cli-backend FailoverError makes the stored resume session unusable.

## Best Possible Solution

Clear the reused CLI session binding through the shared session-store helper when a channel CLI turn fails after using a stored `claude-cli` binding, while preserving existing retry and missing-transcript reseed behavior.

## Reproduction Assessment

Yes at source level, but not as a live run in this review. The current channel CLI failure path can surface a FailoverError without reaching the successful metadata cleanup that persists cleared `cliSessionIds` and `cliSessionBindings`.

## Solution Assessment

Yes, the requested cleanup is the right shape if scoped to a failed reused CLI binding. The safer implementation is a shared session-state cleanup in the channel CLI failure path, not Telegram-only handling or a new config option.
