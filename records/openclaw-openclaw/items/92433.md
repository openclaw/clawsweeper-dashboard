## Summary

Keep open: current main still treats transcript commit as successful subagent completion delivery, and no merged or open canonical PR owns this exact post-commit/pre-consumption teardown race.

## What This Changes

Issue reports a race where a subagent completion queued into an active requester run can be marked delivered after transcript commit but before the requester actually consumes it.

## Best Possible Solution

Add a durable adoption or teardown-recovery invariant for subagent completion events so a committed but unconsumed internal completion is retried through the existing direct completion path after requester teardown.

## Reproduction Assessment

Yes at source level: current main resolves the active-run delivery wait at queued user message commit, then maps that steered outcome to delivered before any assistant response or adoption acknowledgement. I did not run the intermittent live race in this read-only review.

## Solution Assessment

Unclear on exact fix shape: a consumed/adopted acknowledgement is the clean invariant, while the latest maintainer comment suggests a narrower teardown detector as a first PR-sized slice.
