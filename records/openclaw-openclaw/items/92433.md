## Summary

Keep this open: current main and the latest release still treat transcript commit as successful subagent completion delivery, and the direct linked fix was closed unmerged after maintainer review called for a safer lifecycle contract.

## What This Changes

The issue reports a race where subagent completion delivery is marked successful after transcript commit even if the requester run ends before consuming the queued internal completion event.

## Best Possible Solution

Implement a lifecycle-level delivery outcome contract that distinguishes uncommitted, adopted, terminal-unconsumed, and committed-pending states, with exactly-once fallback only for definitively unconsumed terminal completions.

## Reproduction Assessment

Yes at source level: current main resolves the active queue wait at queued user `message_end`, then dispatch maps that steered outcome to delivered. I did not run a live timing repro because this review is read-only and the report is race-bound.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR was rejected as unsafe, so the best direction is the maintainer-requested closed delivery outcome contract rather than another narrow acknowledgement patch.
