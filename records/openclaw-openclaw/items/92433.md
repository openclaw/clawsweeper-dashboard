## Summary

This issue should stay open: current main and the latest release still treat transcript commit as successful subagent completion delivery, and the only linked fix PR was closed unmerged after maintainer review rejected its acknowledgement design.

## What This Changes

The issue reports an active requester teardown race where a queued subagent completion is written to the transcript, marked delivered, and never processed by the requester run.

## Best Possible Solution

Model completion handoff delivery with closed outcomes such as not-committed, adopted, terminal-unconsumed, and committed-pending, then retry exactly once only for definitively unconsumed terminal completions.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user `message_end`, then let requester `agent_end` occur before any assistant response. Current main has already resolved the queue outcome and dispatch maps that steered outcome to delivered.

## Solution Assessment

Unclear: the report identifies the correct failure boundary, but the closed assistant-ack PR was rejected as unsafe. The best path is the maintainer-requested lifecycle delivery outcome contract rather than another narrow transcript-commit or assistant-ack patch.
