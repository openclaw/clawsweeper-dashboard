## Summary

Keep open: current main and v2026.6.8 still treat transcript commit as successful subagent completion delivery, while the linked attempted fix was closed unmerged after review rejected its acknowledgement design.

## What This Changes

The issue reports a race where a subagent completion is committed into an active requester transcript but marked delivered before the requester run actually consumes it.

## Best Possible Solution

Define and land a closed lifecycle delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending states, with exactly-once fallback only for definitively unconsumed terminal completions.

## Reproduction Assessment

Yes, at source level: current main resolves active-run steering at the queued user message_end and then dispatch maps that steered outcome to delivered. I did not run a live timing race repro in this read-only pass.

## Solution Assessment

Unclear: the bug is real, but the rejected assistant-ack PR shows the final lifecycle contract still needs maintainer choice between durable adoption, committed-pending reconciliation, or teardown recovery.
