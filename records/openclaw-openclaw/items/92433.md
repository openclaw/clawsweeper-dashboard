- `impact:message-loss`: The reported failure loses the completion handoff from the requester's effective message flow with no retry or visible error.

## Summary

Keep open: current main and v2026.6.6 still treat transcript commit as successful subagent completion delivery, and the only linked fix PR was closed unmerged because the acknowledgement design needs a broader lifecycle contract.

## What This Changes

The issue reports a race where subagent completion delivery is marked successful after transcript commit even when the requester run ends before consuming the queued internal completion event.

## Best Possible Solution

Define a closed completion-delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending, then use it to fall back exactly once only when the completion is definitively unconsumed.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user message_end, then let requester agent_end occur before any assistant response; current main has already resolved the queue outcome and mapped it to delivered. The reporter also provided stable-release transcript and SQLite forensics for the same interleaving.

## Solution Assessment

No final implementation path is settled yet. The closed assistant-ack PR is not the best fix as submitted; the maintainable direction is a lifecycle-level delivery outcome contract shared by active-run, fallback, and reconciliation paths.
