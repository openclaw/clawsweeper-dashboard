## Summary

This issue should stay open: current main and the latest release still treat transcript commit as successful subagent completion delivery, and the linked fix PR was closed unmerged after maintainer review rejected its acknowledgement design.

## What This Changes

The issue reports a race where subagent completion delivery is marked successful after transcript commit even if the requester run ends before consuming the queued internal completion event.

## Best Possible Solution

Add the maintainer-requested closed delivery outcome contract that distinguishes uncommitted, adopted, terminal-unconsumed, and committed-pending states, with exactly-once fallback only for definitively unconsumed terminal completions.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user message_end, then let requester agent_end occur before any assistant response. Current main has already resolved the queue outcome and dispatch maps that steered outcome to delivered.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR was rejected as unsafe, so the best direction is the maintainer-requested closed delivery outcome contract rather than another narrow acknowledgement patch.
