## Summary

Keep open: current main and the latest release still credit active-run subagent completion steering at the transcript-commit boundary, not at requester consumption, and the prior assistant-ack PR was closed unmerged after maintainer review requested a safer lifecycle outcome contract.

## What This Changes

The issue reports that subagent completion delivery can be marked delivered after enqueueing into an active requester run even if that run ends before consuming the internal completion event.

## Best Possible Solution

Define and implement the closed delivery outcome contract requested in review, with distinct outcomes such as not_committed, adopted, terminal_unconsumed, and committed_pending so only definitively terminal-unconsumed completions fall back exactly once.

## Reproduction Assessment

Yes, at source level: current main resolves active-run completion steering at the queued user transcript boundary and maps that steer to delivered. I did not run the live timing race because this is a read-only review and the requester-teardown window is intermittent.

## Solution Assessment

No final implementation path is settled. The reporter's requested delivery boundary is directionally correct, but maintainer review rejected the assistant-ack variant and asked for a closed lifecycle delivery outcome contract instead.
