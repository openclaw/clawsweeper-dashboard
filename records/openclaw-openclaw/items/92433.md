## Summary

This issue should stay open: current main and the latest release still resolve subagent completion steering at transcript commit, while the only direct fix PR was closed unmerged after maintainer review rejected its acknowledgement design.

## What This Changes

The issue reports a race where a subagent completion is marked delivered after transcript commit even if the requester run ends before consuming the queued event.

## Best Possible Solution

Define a lifecycle-level completion delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending; only terminal unconsumed should fall back exactly once, while committed pending remains reconciliation-owned.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user message_end, then let requester agent_end occur before any assistant response; current main has already resolved the queue outcome and mapped it to delivered. The reporter also supplied stable-release transcript and SQLite forensics for that interleaving.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR shows why the best fix is a lifecycle delivery outcome contract or teardown reconciliation, not another narrow transcript-wait patch.
