## Summary

Keep open: current main and v2026.6.6 still treat transcript commit as successful subagent completion delivery, and the linked attempted fix was closed unmerged after maintainer review rejected its acknowledgement design.

## What This Changes

The issue asks OpenClaw to stop silently dropping subagent completions when an active requester run accepts the announce but ends before consuming it.

## Best Possible Solution

Define and implement a closed completion-delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending, then retry or fall back exactly once only for terminal unconsumed outcomes.

## Reproduction Assessment

Yes at source level: current main resolves the active-run wait at queued user message_end and maps that steered outcome to delivered before proving requester consumption. I did not run a live timing-race repro in this read-only review.

## Solution Assessment

No final implementation path is settled yet. The best direction is the maintainer-requested lifecycle delivery outcome contract, not the closed assistant-ack PR design.
