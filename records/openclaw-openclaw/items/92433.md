## Summary

This issue should stay open: current main still treats transcript commit as successful subagent completion delivery, and the only linked fix PR was closed unmerged after maintainer review found the acknowledgement design unsafe.

## What This Changes

Prevent subagent completion handoffs from being marked delivered when an active requester only commits the queued internal event but ends before consuming it.

## Best Possible Solution

Add a lifecycle-level delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending; only terminal unconsumed should fall back exactly once, while committed pending remains reconciliation-owned.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user message_end, then let requester agent_end occur before any assistant response; current main has already resolved the queue outcome and mapped it to delivered. The reporter also provided stable-release transcript and SQLite forensics for the same interleaving.

## Solution Assessment

No final implementation path is settled yet. The best direction is the maintainer-requested closed delivery outcome contract, not the closed assistant-ack PR design.
