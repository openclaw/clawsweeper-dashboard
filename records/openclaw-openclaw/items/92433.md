## Summary

Keep open: current main and the latest v2026.6.8 release still treat transcript commit as successful subagent completion delivery, and the only PR linked with closing syntax was closed unmerged after maintainer review rejected its acknowledgement design.

## What This Changes

The issue reports a subagent completion delivery race where a queued internal completion is stamped delivered after transcript commit even if the requester run ends before processing it.

## Best Possible Solution

Add a closed lifecycle delivery outcome contract that distinguishes uncommitted, adopted, terminal-unconsumed, and committed-pending completion states, with exactly-once fallback only for definitively unconsumed terminal completions.

## Reproduction Assessment

Yes, at source level: current main resolves the active queue wait at the matching queued user `message_end`, then maps that steered outcome to delivered before any requester assistant response is required. I did not run a live race repro in this read-only review.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR was rejected as unsafe, so the best direction is the maintainer-requested closed delivery outcome contract rather than another narrow acknowledgement patch.
