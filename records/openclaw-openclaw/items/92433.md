## Summary

Keep open: current `main` and latest release still treat transcript commit as successful subagent completion delivery, while the only linked fix PR was closed unmerged after maintainer review found that acknowledgement design unsafe.

## What This Changes

The issue reports a race where subagent completion steering into an active requester run is recorded as delivered after transcript commit even if the requester run ends before consuming it.

## Best Possible Solution

Define a closed completion-delivery outcome contract that distinguishes not committed, adopted, terminal unconsumed, and committed pending, then fall back exactly once only for terminal unconsumed outcomes.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user `message_end`, then let requester `agent_end` occur before any assistant response; current main has already mapped the queue outcome to delivered. The reporter also supplied stable-release transcript and SQLite forensics for the same interleaving.

## Solution Assessment

No final implementation path is settled. The linked assistant-ack PR was rejected, so the best fix is a maintainer-approved lifecycle delivery outcome contract rather than another narrow acknowledgement patch.
