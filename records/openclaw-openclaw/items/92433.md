## Summary

Keep open: current main and v2026.6.8 still treat a queued internal completion transcript commit as delivered, and the linked fix PR is closed unmerged after maintainer rejection of its acknowledgement design.

## What This Changes

The issue asks OpenClaw to stop silently losing subagent completions when active-requester steering commits the completion event but the requester run ends before consuming it.

## Best Possible Solution

Define a closed completion-delivery outcome contract, then implement exactly-once fallback only for terminal-unconsumed completions while keeping committed-pending events owned by reconciliation.

## Reproduction Assessment

Yes, at source level: emit the queued completion user message_end, then let requester agent_end occur before any assistant response. Current main resolves the wait at message_end and maps the steered outcome to delivered.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR was rejected as unsafe, so the best direction is the maintainer-requested lifecycle delivery outcome contract.
