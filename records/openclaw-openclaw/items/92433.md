## Summary

This issue should stay open: current main and the latest release still treat a transcript-committed subagent completion steer as delivered before requester consumption is proven, and the linked attempted fix was closed unmerged after maintainer review rejected its acknowledgement model.

## What This Changes

The issue reports a race where a subagent completion steered into an active requester run is marked delivered after transcript commit even if the requester run ends before processing it.

## Best Possible Solution

Define and implement a lifecycle-level delivery outcome contract that separates not committed, adopted, terminal unconsumed, and committed pending states, then add focused tests for the teardown interleaving.

## Reproduction Assessment

Yes at source level: emit the queued internal completion user message_end, then let requester agent_end occur before any assistant response; current main has already resolved the queue outcome and mapped it to delivered. The reporter also provided stable-release transcript and SQLite forensics for the same interleaving.

## Solution Assessment

Unclear: the report's fallback or teardown-flush directions are plausible, but the closed assistant-ack PR shows the permanent consumption contract still needs maintainer lifecycle review. The best path is a closed delivery outcome model rather than another one-sided acknowledgement patch.
