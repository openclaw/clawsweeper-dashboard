## Summary

Keep open: current main and v2026.6.10 still credit active-run steering at the transcript-commit boundary, while the attempted assistant-ack fix at https://github.com/openclaw/openclaw/pull/92934 was closed unmerged after maintainer review requested a closed lifecycle delivery outcome contract.

## What This Changes

The issue reports that subagent completion handoffs can be marked delivered after queue/transcript commit even when the requester run ends before consuming the internal completion event.

## Best Possible Solution

Define a closed completion-delivery outcome contract, then implement exactly-once fallback only for terminal-unconsumed completions while keeping committed-pending events owned by reconciliation.

## Reproduction Assessment

Yes, at source level: current main resolves active steering at the queued user message_end and maps that steered outcome to delivered. The reporter also supplied real-run forensics for the teardown window, but this read-only review did not run a fresh live timing reproduction.
