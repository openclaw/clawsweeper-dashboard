## Summary

Keep open: current main and v2026.6.9 still credit active-run steering as delivered at the transcript-commit boundary, while the linked assistant-ack PR was closed unmerged after maintainer rejection of that design.

## What This Changes

The issue reports a subagent completion delivery race where an active requester wake is marked delivered after transcript commit even if the requester run ends before consuming the queued internal completion event.

## Best Possible Solution

Define a closed completion-delivery outcome contract, then implement exactly-once fallback only for terminal-unconsumed completions while keeping committed-pending events owned by reconciliation.

## Reproduction Assessment

Yes, at source level: current main resolves the active queue wait at the matching queued user message_end, then dispatch maps that steered outcome to delivered. I did not run a fresh live reproduction because this review is read-only and the reported timing window is intermittent.

## Solution Assessment

No final implementation path is settled. The closed assistant-ack PR was rejected as unsafe, so the best direction is the maintainer-requested lifecycle delivery outcome contract.
