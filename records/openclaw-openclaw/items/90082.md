## Summary

Keep open: current main and v2026.6.8 have addressed the schema/docs gap, the cooldown is now 60 seconds, and the circuit-breaker-open path skips with an empty summary, but the default threshold is still 3 and generic embedded-agent fallback text is not explicitly filtered when a usable memory-tool result exists.

## What This Changes

The issue reports that Active Memory’s circuit breaker opens too readily and can inject an embedded-agent fallback response into main-session prompt context after provider failures.

## Best Possible Solution

Keep this issue open until maintainers choose the Active Memory degradation policy, add a regression guard that generic embedded-agent fallback text never becomes recalled context, and coordinate runtime failover or latency telemetry with the open fallback issue.

## Reproduction Assessment

Yes at source level for the remaining gaps: the breaker threshold is still 3, and accepted subagent rawReply can become prompt context when memory evidence exists. No live MiniMax/quota-exhaustion run was performed, so the exact reported field reproduction remains medium confidence.

## Solution Assessment

No single mechanical fix is clearly best. Filtering generic fallback payloads is a narrow bug fix, but changing breaker defaults, surfacing health, and adding runtime failover need maintainer policy and should align with the related fallback issue.
