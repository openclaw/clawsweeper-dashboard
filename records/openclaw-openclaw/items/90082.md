## Summary

Keep open: current main and the latest release fixed important pieces by exposing the breaker knobs and avoiding prompt injection when recall has no summary, but the default breaker threshold remains 3 and the provider/failover diagnostic policy still needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to make Active Memory's circuit breaker less aggressive, stop fallback-text prompt pollution, document/surface breaker knobs, and decide degraded-provider failover behavior.

## Best Possible Solution

Keep the current no-summary/no-injection safeguards, then decide one Active Memory degradation policy for breaker defaults, user-visible diagnostics, and provider failover aligned with https://github.com/openclaw/openclaw/issues/79611.

## Reproduction Assessment

Yes at source level for the remaining default/policy gaps: current main still defaults the breaker threshold to 3 and only filters the generic fallback text from search-query text. No live MiniMax/OAuth quota-exhaustion reproduction was run against current main.

## Solution Assessment

No as a single automated fix: the no-summary injection guard is the right narrow fix for open-breaker prompt pollution, but changing defaults, diagnostics, and runtime failover requires maintainer policy and should align with the broader fallback issue.
