## Summary

Keep open: current main partially addresses docs and the simplest completed-output pollution path, but the breaker still defaults to 3 timeouts and Active Memory fallback remains resolution-only, so the remaining degradation behavior needs maintainer policy rather than cleanup close.

## What This Changes

The issue asks OpenClaw to make Active Memory degrade cleanly under repeated model failures by tuning/documenting the circuit breaker, avoiding fallback-text prompt injection, surfacing diagnostics, and deciding runtime failover behavior.

## Best Possible Solution

Keep this issue open until maintainers choose the Active Memory degradation policy: preserve the no-error-payload prompt-context invariant, set intentional breaker defaults/diagnostics, and align runtime failover semantics with https://github.com/openclaw/openclaw/issues/79611.

## Reproduction Assessment

No high-confidence live reproduction was run. The breaker threshold and single-model fallback semantics are source-reproducible, while the exact current fallback-string pollution path needs live or focused regression proof because current main now gates completed summaries on usable memory evidence.

## Solution Assessment

No single mechanical fix is clearly best. Filtering embedded error payloads is the narrow bug invariant, but breaker defaults, doctor/health visibility, and runtime fallback behavior need maintainer-chosen policy before implementation.
