## Summary

Keep open: current main covers explicit reset and successful CLI compaction paths, but it still intentionally preserves provider-owned Claude CLI bindings across the implicit daily default and has no TTL or long-lived invalidation policy for the remaining lifecycle gap.

## What This Changes

The issue requests a lifecycle policy that prevents stale Claude CLI resume bindings and stuck live model switch state from surviving fresh, reset, daily/idle, or long-lived interactive session boundaries unless preservation is explicit.

## Best Possible Solution

Keep the explicit reset and compaction clearing paths, then choose a maintainer-owned policy for provider-owned CLI continuity in implicit daily/idle and long-lived sessions, with tests, docs, and doctor repair behavior for any TTL or invalidation rule.

## Solution Assessment

Is this the best way to solve the issue? Unclear: clearing bindings on explicit reset and compaction is already the narrow implemented boundary, while default TTL or implicit reset invalidation changes documented behavior and needs maintainer policy first.
