## Summary

Keep open. Current main has already fixed explicit non-subagent Gateway reset via https://github.com/openclaw/openclaw/pull/83448, but it still intentionally preserves provider-owned CLI bindings across implicit daily resets, and the requested TTL/long-lived-session invalidation policy is not implemented.

## What This Changes

The issue asks OpenClaw to prevent stale `claude-cli` resume bindings and stuck model-switch state from surviving long-lived session lifecycle boundaries unless backend continuity is explicitly preserved.

## Best Possible Solution

Define a maintainer-approved CLI session lifecycle contract that keeps explicit resets fresh, decides whether implicit daily/idle provider-owned continuity needs a default change or TTL, and aligns docs, schema, doctor repair, and regression tests.

## Reproduction Assessment

Yes at source level: current main intentionally preserves provider-owned CLI bindings across implicit daily reset boundaries, and the CLI runner will resume when that binding survives. I did not live-reproduce the reporter's exact WhatsApp overrun.

## Solution Assessment

Unclear. The explicit reset fixes are good, but the remaining long-lived binding TTL/default-reset behavior needs a maintainer product decision because current docs and tests protect provider-owned CLI continuity.
