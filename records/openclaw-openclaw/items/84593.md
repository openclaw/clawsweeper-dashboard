## Summary

Keep open: current main fixes the explicit gateway reset case and CLI compaction clears bindings after a successful compaction, but the remaining long-lived/default lifecycle behavior is still documented and tested to preserve provider-owned Claude CLI bindings across implicit daily expiry, with no TTL or max-age invalidation policy.

## What This Changes

The issue asks OpenClaw to prevent stale Claude CLI resume bindings and stuck live-model-switch state from surviving new/reset/long-lived session lifecycle boundaries.

## Best Possible Solution

Keep the explicit reset and successful-compaction protections, then make a maintainer decision on implicit/long-lived provider-owned CLI continuity and encode it as a tested policy with docs, diagnostics or doctor guidance, and an opt-in expiry rule if maintainers want TTL behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the full over-budget WhatsApp session was established. Source and tests do reproduce the key disputed lifecycle boundary: provider-owned Claude CLI bindings are preserved across implicit daily expiry on current main.

## Solution Assessment

Unclear as an automatic fix: clearing CLI bindings on explicit reset and after successful compaction is already the right boundary, but default implicit expiry or TTL invalidation conflicts with current docs/tests and needs a maintainer policy choice. The narrower stale `liveModelSwitchPending` reset bug should be handled through https://github.com/openclaw/openclaw/issues/83192 and its linked PRs.
