## Summary

Keep open: current main covers explicit `/reset` and native CLI compaction paths, but manual `/compact` still updates OpenClaw compaction state without clearing Claude CLI resume bindings, matching the 2026-06-10 live report; broader TTL or implicit-reset behavior remains a maintainer policy decision.

## What This Changes

The issue asks OpenClaw to prevent Claude CLI resume bindings from preserving stale provider conversation state after compaction or reset, including optional TTL/config invalidation.

## Best Possible Solution

Land a narrow maintainer-scoped fix that clears or reseeds CLI bindings after successful OpenClaw-owned manual compaction, then separately decide whether implicit daily resets or TTL policy should cut provider-owned continuity.

## Reproduction Assessment

Yes for the manual-compaction slice from source: `/compact` updates OpenClaw compaction metadata without clearing CLI bindings, so the next Claude CLI turn can still resume the old provider conversation; no live current-main repro was run here.

## Solution Assessment

Partly: clearing provider CLI bindings after successful manual/OpenClaw-owned compaction is the narrow bug fix, while TTL and implicit daily reset invalidation conflict with documented provider-owned continuity and need maintainer direction.
