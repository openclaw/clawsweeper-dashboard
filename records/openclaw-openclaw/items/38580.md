## Summary

Keep open: current main and the latest release still lack Telegram pinned-message reads, while the shared message-action vocabulary and two closed unmerged attempts show a clear plugin-owned path that has not landed.

## What This Changes

Add Telegram support for fetching the current pinned chat message via the Bot API, preferably as an on-demand agent/message action.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` action backed by `getChat`, return zero or one current pinned message, gate it with `channels.telegram.actions.pins`, document it, and leave automatic session-start context injection for a separate product decision.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main and `v2026.5.20` gives high confidence that Telegram still does not advertise, gate, or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is the maintainable solution. Automatic session-start context loading should remain out of scope until maintainers explicitly choose that behavior.
