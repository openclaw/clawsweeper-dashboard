## Summary

Keep open: current main still lacks a Telegram pinned-message read path, and the two related implementation PRs were closed unmerged. The remaining work is a narrow Telegram plugin feature because the shared message-action contract already has `list-pins`.

## What This Changes

Request to let Telegram agents fetch the current pinned chat message on demand, likely via Telegram Bot API `getChat.pinned_message`.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` action backed by `getChat.pinned_message`, with a Telegram action gate, tests, docs, and changelog; leave automatic startup context injection for separate discussion.

Implement the Telegram pinned-message read request for https://github.com/openclaw/openclaw/issues/38580 as a narrow plugin-owned feature. Add an on-demand `list-pins` message action for Telegram backed by Bot API `getChat.pinned_message`, returning a `pins` array with zero or one normalized pinned message. Wire discovery in `extensions/telegram/src/channel-actions.ts`, runtime dispatch in `extensions/telegram/src/action-runtime.ts`, any send/API helper in `extensions/telegram/src/send.ts`, and a Telegram action gate such as `channels.telegram.actions.pins` across `src/config/types.telegram.ts` and `src/config/zod-schema.providers-core.ts`. Add focused tests for discovery, runtime success/no-pin/error/gate-disabled behavior, and target normalization. Update `docs/channels/telegram.md` and `CHANGELOG.md`.
