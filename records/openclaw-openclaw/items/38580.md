Add an on-demand way for agents to read the current pinned message from a Telegram chat.

## Best Possible Solution

Add a read-only on-demand Telegram message action that returns the current chat-level pinned message as a zero-or-one result through the existing plugin action runtime, without automatic session injection or a new configuration surface.

## Reproduction Assessment

## Solution Assessment

Yes. A narrow on-demand plugin action is the most maintainable solution; automatic session-start injection would add unnecessary context policy, state coupling, and configuration pressure.
