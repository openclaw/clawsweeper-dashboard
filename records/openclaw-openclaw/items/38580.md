## Summary

Keep open: current main still lacks a Telegram pinned-message read path, while the two related implementation PRs were closed unmerged. The remaining work is a narrow Telegram plugin feature because the shared message-action contract already has `list-pins`.

## What This Changes

Request an on-demand or startup-time Telegram Bot API fetch of a chat's current pinned message for assistant context.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by Bot API `getChat.pinned_message`, gated like other Telegram actions, and leave automatic session-start context injection for later.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main has no Telegram `list-pins`, `getpinned`, or documented pinned-message read path.

## Solution Assessment

Yes: the narrow maintainable solution is an on-demand Telegram `list-pins` action inside the existing Telegram plugin message-action adapter. That avoids core changes and avoids the larger automatic startup-context behavior.

## Repair Work Prompt

Implement the Telegram pinned-message read requested by https://github.com/openclaw/openclaw/issues/38580 as a narrow Telegram plugin feature. Add an on-demand `list-pins` message action backed by Telegram Bot API `getChat` and its `pinned_message` field, returning zero or one pin in the existing message-action result shape for the current chat or an explicit target. Add a `channels.telegram.actions.pins` top-level/per-account gate consistent with other Telegram action gates and Slack's pins pattern. Update Telegram action discovery, runtime handling, config types/schema/tests, docs, and CHANGELOG.md. Do not add automatic session-start context injection, do not move Telegram-specific behavior into core, and do not add a separate slash command unless a maintainer explicitly asks for that scope. Preserve plugin boundary rules and use existing Telegram target/token/runtime helpers.
