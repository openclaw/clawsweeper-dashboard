## Summary

Keep open: current main and latest release still lack Telegram pinned-message reads. The shared `list-pins` action exists, and Telegram has adjacent write-side pin helpers plus `getChat` usage, but the Telegram plugin does not advertise, gate, or dispatch a pinned-message read action; the two related implementation PRs were closed unmerged.

## What This Changes

Request to let Telegram agents fetch the current pinned Telegram chat message via the Bot API, either on demand or during session startup.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by grammY/Bot API `getChat`, return zero or one current pinned message, gate and document it consistently, and defer automatic session-start context injection.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main and `v2026.5.19` gives high confidence that Telegram still does not advertise or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is more maintainable than adding automatic session-start pinned-context loading now.
