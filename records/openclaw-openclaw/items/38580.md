## Summary

Keep open: current main still lacks a Telegram pinned-message read action. The generic `list-pins` action exists and Telegram has write-side pin helpers, but the Telegram plugin does not advertise, gate, or route a read backed by `getChat`; the two related implementation PRs are closed unmerged.

## What This Changes

Request an on-demand Telegram pinned-message read so agents can fetch pinned chat context.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by `getChat`, return zero or one current pinned message, gate and document it consistently, and leave automatic session-start context injection for a separate decision.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that Telegram still does not advertise or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is more maintainable than adding automatic session-start pinned-context loading now.
