## Label Justifications

- `impact:session-state`: The requested feature would pull pinned chat context into agent/session context across turns or sessions.

## Summary

Keep open: current main and the latest release still do not expose Telegram pinned-message reads through the message tool or Telegram command/action surface, and the exact related implementation attempts are closed unmerged. The feature is valid and well-bounded, but it should wait for maintainer confirmation of the Telegram action gate, privacy/access boundary, and whether automatic session-start ingestion is in scope.

## What This Changes

Request to let OpenClaw fetch the current pinned Telegram chat message via the bot API, either on demand or during session startup.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read/list action backed by `getChat.pinned_message`, with an explicit action gate, account/chat/topic-aware authorization, docs, and focused tests; defer automatic session-start ingestion unless maintainers approve that product behavior separately.

## Reproduction Assessment

Yes, at source level. Current main has generic pin/list action concepts and Telegram pin helpers, but Telegram does not advertise, gate, map, alias, dispatch, test, or document a pinned-message read/list action, so the requested behavior is unavailable.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrow maintainable implementation is Telegram-plugin-owned on-demand `list-pins`/read-pinned support using `getChat`, not a core workaround or automatic session-start context injection by default.
