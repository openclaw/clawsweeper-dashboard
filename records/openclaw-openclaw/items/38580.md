## Summary

Keep open: current main and v2026.6.5 still lack a Telegram pinned-message read action. The two linked implementation attempts are closed unmerged, and the related open pin/unpin issue tracks a distinct write capability rather than this read path.

## What This Changes

Request Telegram support for reading the current pinned chat message as persistent session context, preferably through an on-demand command or action.

## Best Possible Solution

Add an on-demand Telegram `list-pins` message action backed by `getChat`, returning zero or one current pinned message without automatic session-start ingestion.

## Reproduction Assessment

Yes, at source level. Current main defines the generic `list-pins` action but Telegram neither advertises nor maps it, and unsupported Telegram actions throw before runtime dispatch.

## Solution Assessment

Yes. The narrow maintainable solution is a Telegram plugin-owned on-demand `list-pins` action; automatic session-start injection would add avoidable prompt, state, and privacy policy decisions.
