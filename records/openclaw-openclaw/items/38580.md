## Summary

Keep open: current main still lacks a Telegram pinned-message read path. The generic channel action name exists, but the Telegram plugin neither advertises nor dispatches `list-pins`, and the two related implementation PRs were closed unmerged.

## What This Changes

Allow Telegram agents to fetch the current chat pinned message via the Bot API, either on demand or during session startup.

## Best Possible Solution

Add a Telegram-owned, on-demand `message` action for `list-pins` backed by Bot API `getChat`, returning a zero-or-one `pins` array and documenting the Telegram API limit without adding startup auto-fetch.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Telegram pinned-message read action or command.

## Solution Assessment

Yes. The narrow maintainable path is an on-demand Telegram plugin action using the existing `list-pins` surface; automatic session-start context injection is a separate product behavior.

## Repair Work Prompt

Implement the on-demand Telegram pinned-message read feature for this issue. Scope it to the existing `message` tool/action surface: advertise and dispatch `action: "list-pins"` for Telegram, call Telegram Bot API `getChat` with the selected account token/apiRoot/proxy/network settings, and return a JSON result with `ok: true` and a `pins` array containing the current `pinned_message` when present or an empty array when absent. Add a `channels.telegram.actions.pins` gate following the Slack `actions.pins` pattern, defaulting consistently with existing Telegram action gates. Add focused unit tests for discovery, dispatch, no-pin, pinned-message, missing token, API failure, account selection, and action gating. Update Telegram docs and `CHANGELOG.md` under Unreleased. Do not add automatic session-start fetching, persistence, prompt injection, native slash-command menu changes, or core action-contract changes unless the existing generic contract is insufficient; stop and ask if the work requires a broader product decision.
