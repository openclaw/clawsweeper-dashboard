## Summary

Keep open: current main still lacks Telegram pinned-message reads. The generic channel action vocabulary includes `list-pins`, but the Telegram plugin only advertises and dispatches send/poll/react/delete/edit/sticker/topic actions, and the linked implementation PRs are closed unmerged.

## What This Changes

Request to let Telegram agents fetch the current pinned chat message on demand, using Telegram Bot API `getChat.pinned_message`.

## Best Possible Solution

Add a scoped Telegram `list-pins` message action backed by `getChat.pinned_message`, document that Telegram exposes at most the current chat-level pin, and leave automatic session-start fetching for a later opt-in design.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence feature gap: Telegram action discovery and dispatch do not expose pinned-message reads on current main.

## Solution Assessment

Yes for the scoped on-demand path. Using the existing Telegram plugin action surface is narrower than adding startup prompt injection, storage, or core behavior.
