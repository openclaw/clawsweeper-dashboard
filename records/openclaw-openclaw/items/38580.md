## Summary

Keep open: current main and v2026.6.5 still lack Telegram pinned-message reads. The existing generic `list-pins` action provides a clean plugin-owned implementation seam, while both linked implementation PRs closed unmerged and the open pin/unpin issue covers distinct write behavior.

## What This Changes

Add an on-demand way for agents to fetch the current pinned message from a Telegram chat.

## Best Possible Solution

Implement Telegram `list-pins` through the existing plugin-owned message action, fetching the current chat pin on demand and returning zero or one normalized pin without automatic session injection, persistence, or a new configuration surface.

## Reproduction Assessment

Yes at the source level: current Telegram action discovery omits `list-pins`, and its dispatcher rejects actions absent from the Telegram action map. No live Telegram call was needed to establish that the OpenClaw capability is missing.

## Solution Assessment

Yes. Reusing the existing generic `list-pins` message action is narrower and more maintainable than adding a Telegram-specific slash command or automatic session-start context injection.
