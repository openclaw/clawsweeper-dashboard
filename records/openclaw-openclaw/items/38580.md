## Summary

Keep open: current main and v2026.6.10 still do not expose an agent-facing Telegram pinned-message read, even though lower-level send pin helpers and pinned delivery exist. The request is valid, but it needs maintainer product/security review before implementation because it would add a new Telegram chat-content read surface.

## What This Changes

Request to let OpenClaw Telegram agents fetch the current pinned message from a chat on demand or at session start via the bot API.

## Best Possible Solution

Approve and implement a Telegram-plugin-owned on-demand pinned-message read/list-pins action backed by getChat, with explicit gating/defaults, docs, focused tests, and live Telegram proof; handle automatic session-start context injection separately.

## Reproduction Assessment

Yes, at source level. Current main has a shared list-pins action name and Telegram getChat plumbing for other lookups, but Telegram does not advertise, map, gate, dispatch, test, or document a pinned-message read action.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The likely best MVP is an explicit on-demand Telegram-plugin action backed by getChat, not automatic session-start context injection or a core workaround.
