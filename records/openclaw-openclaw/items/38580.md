## Summary

Keep open: current main and v2026.6.10 still do not expose a Telegram pinned-message read action, command, config gate, docs, or tests. The related implementation PRs are closed unmerged, and the remaining work needs maintainer product/security direction because it would let agents read persistent Telegram chat content.

## What This Changes

Request to let OpenClaw fetch the current pinned Telegram chat message on demand or at session start via the Telegram Bot API.

## Best Possible Solution

Approve a Telegram-plugin-owned on-demand pinned-message read action first, backed by getChat, explicit action/config gating, chat/topic authorization, docs, focused tests, and live Telegram proof; treat automatic session-start context injection as a separate opt-in decision.

## Reproduction Assessment

Yes, at source level. Current main has shared read/list-pins action names and Telegram getChat support exists, but the Telegram plugin does not advertise, map, gate, dispatch, test, or document pinned-message reads.

## Solution Assessment

Unclear until maintainers choose the product contract. A Telegram-plugin getChat implementation is likely the right layer, but on-demand versus automatic context, default gating, and security boundaries need approval first.
