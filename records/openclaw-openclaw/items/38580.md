## Summary

Keep open: current main and v2026.6.1 still do not expose a Telegram pinned-message read action, and the related implementation PRs were closed unmerged. The narrow remaining path is an on-demand Telegram plugin action backed by getChat, not automatic session-start context ingestion.

## What This Changes

Request to let OpenClaw read the current pinned Telegram chat message on demand so pinned chat context can be surfaced to the assistant.

## Best Possible Solution

Add a Telegram-owned on-demand `list-pins` message action backed by getChat, documented and tested as returning Telegram's current most recent `pinned_message`, while leaving automatic startup injection for a separate product decision.

## Reproduction Assessment

Yes from source inspection, though not from a live Telegram bot run: Telegram does not advertise or dispatch `list-pins`, so an on-demand pinned-message read has no current path on main.

## Solution Assessment

Yes: a narrow explicit `list-pins` action is the maintainable first solution. Automatic session-start fetching should stay out of this repair unless maintainers separately approve that context policy.
