## What This Changes

The issue requests a Telegram Bot API-backed way for agents to read the current pinned chat message on demand.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by `getChat`/`ChatFullInfo.pinned_message`, gate it with `channels.telegram.actions.pins`, document it, and leave automatic session-start context injection for a separate product decision.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action inside the Telegram plugin is the maintainable solution. Automatic session-start context loading should remain out of scope until maintainers explicitly choose it.
