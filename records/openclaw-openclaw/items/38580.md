## Summary

Keep open: current main and v2026.6.8 still do not expose a Telegram pinned-message read through action discovery, runtime dispatch, config gates, tests, or docs. The feature is well scoped, but it adds a Telegram chat-content read surface and should wait for maintainer product/security approval before a fix PR.

## What This Changes

Request to let Telegram agents fetch a chat's current pinned message, on demand or at session start, via Telegram Bot API getChat.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read or list-pins action backed by getChat, with explicit action gating, docs, focused tests, and live Telegram proof; defer automatic session-start context injection until separately approved.

## Reproduction Assessment

Yes, at source level: current main has the shared list-pins action, but Telegram does not advertise, map, gate, or dispatch a pinned-message read action. I did not run a live Telegram repro because there is no current Telegram action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely best MVP is an explicit on-demand Telegram-plugin action backed by getChat, not automatic session-start context injection or a core workaround.
