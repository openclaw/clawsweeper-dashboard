## Summary

Keep open: current main still lacks Telegram pinned-message reads. The shared `list-pins` action exists, Telegram already has write-side pin helpers and Bot API `getChat` support, but the Telegram plugin does not advertise, gate, or dispatch a read action; both related implementation PRs are closed unmerged.

## What This Changes

The issue requests a Telegram bot capability to fetch the current pinned message as persistent chat context.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by Bot API `getChat`, return zero or one current pinned message, gate and document it consistently, and leave automatic session-start context injection for a separate decision.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that Telegram still does not advertise or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is more maintainable than adding automatic session-start pinned-context loading now.
