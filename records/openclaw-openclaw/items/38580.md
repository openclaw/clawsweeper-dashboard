## Summary

Keep open. Current main still has the shared `list-pins` action and `openclaw message pins` command, but the bundled Telegram plugin does not advertise, route, gate, normalize, or execute pinned-message reads, and the two related contributor PRs were closed unmerged.

## What This Changes

Request Telegram support for fetching the current pinned chat message through the Bot API, preferably on demand via a command or shared message action.

## Best Possible Solution

Add bundled Telegram plugin support for the existing `list-pins` action by calling `getChat`, normalizing Telegram's current `pinned_message` into the shared pins result shape, documenting Telegram's zero-or-one pinned-message behavior, and adding a per-action gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: dispatching shared `list-pins` to Telegram reaches `telegramMessageActions.handleAction`, finds no mapped Telegram action, and throws `Unsupported Telegram action`.

## Solution Assessment

Yes. The narrow maintainable solution is to implement this inside the bundled Telegram plugin through the existing shared `list-pins` message action, not through one-off core Telegram logic or automatic session startup ingestion.
