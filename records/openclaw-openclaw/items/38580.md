## What This Changes

Request to let OpenClaw fetch Telegram chat pinned messages on demand or at session start.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read/list-pins action backed by Telegram `getChat`, with explicit action gating, docs, focused tests, and live Telegram proof; defer automatic session-start context injection until separately approved.

## Reproduction Assessment

Yes, at source level: the shared action list has `list-pins`, but Telegram does not advertise, map, gate, or dispatch a pinned-message read action. I did not run a live Telegram repro because current main has no Telegram action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely best MVP is an explicit on-demand Telegram-plugin action backed by `getChat`, not automatic session-start context injection or a core workaround.
