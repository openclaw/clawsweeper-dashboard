## Summary

Keep open. Current main has a shared `list-pins` action and CLI command, but Telegram still does not advertise or handle that action, so the requested pinned-message read remains missing.

## What This Changes

Request to let the bundled Telegram plugin fetch a chat's current pinned message through the Telegram Bot API.

## Best Possible Solution

Add bundled Telegram support for the existing `list-pins` action by calling `getChat`, normalizing Telegram's single `pinned_message` into the shared pins result shape, gating it through Telegram actions config, and documenting the zero-or-one pin behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: dispatching shared `list-pins` to Telegram reaches `telegramMessageActions.handleAction`, finds no mapped Telegram action, and throws `Unsupported Telegram action`; this is a feature gap rather than a regression in documented Telegram behavior.

## Solution Assessment

Yes. The narrow maintainable solution is to implement this inside the bundled Telegram plugin through the existing shared `list-pins` message action, not through one-off core Telegram logic or automatic session-start ingestion.
