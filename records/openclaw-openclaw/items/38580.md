## Summary

Keep open: current main still lacks a Telegram pinned-message read action, while the two related implementation attempts are closed unmerged. The narrow path is clear enough for a focused feature PR, but this is not already implemented on main.

## What This Changes

The issue requests an on-demand Telegram Bot API path for OpenClaw agents to read the current pinned message from a Telegram chat.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by `getChat`, return zero or one current pinned message, gate and document it consistently, and defer automatic session-start context injection.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main does not advertise or execute Telegram pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is more maintainable than adding automatic session-start context loading now.
