## Summary

Keep open: current main and the latest release still do not expose a Telegram pinned-message read through action discovery, runtime dispatch, config gates, tests, or docs. The request is clear and bounded, but it adds a Telegram chat-content read surface, so maintainer product/security approval should happen before implementation.

## What This Changes

The issue requests a Telegram bot/API capability for agents to fetch the current pinned chat message as persistent session context.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read or list-pins action backed by getChat, with explicit action gating, docs, focused tests, and live Telegram proof; defer automatic session-start context injection until separately approved.

## Reproduction Assessment

Yes. Source inspection shows Telegram does not advertise, map, gate, or dispatch a pinned-message read action, and there is no current Telegram action surface to exercise live.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The likely best MVP is an explicit on-demand Telegram-plugin action backed by getChat, not automatic session-start context injection or a core workaround.
