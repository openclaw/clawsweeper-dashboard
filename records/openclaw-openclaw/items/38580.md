## Summary

Keep open: current main still lacks an on-demand Telegram pinned-message read, and the two related PRs that attempted it were closed unmerged. The remaining work is localized to the Telegram plugin because the shared message-action contract already has a `list-pins` action.

## What This Changes

Request to let Telegram agents fetch the current pinned chat message via the Telegram Bot API.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by Bot API `getChat.pinned_message`, gated like other Telegram actions, and leave automatic session-start context injection for a separate decision.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection gives high confidence that current main has no Telegram `list-pins`/`getpinned` action or documented pinned-message read path.

## Solution Assessment

Yes: the narrow maintainable path is to implement an on-demand Telegram `list-pins` action inside the existing Telegram plugin message-action adapter. That avoids core changes and avoids the larger automatic session-start behavior.

## Repair Work Prompt

Implement on-demand pinned-message reads for Telegram for https://github.com/openclaw/openclaw/issues/38580. Add a Telegram `message` tool action `list-pins` in `extensions/telegram/src/channel-actions.ts` and `extensions/telegram/src/action-runtime.ts`, backed by Telegram Bot API `getChat` and its `pinned_message` field. Return a stable JSON result with `ok: true` and `pins: []` when no pin exists or one normalized pin when Telegram returns one; include useful fields such as chatId, messageId, date, sender/from, and text/caption when present. Add top-level and per-account `channels.telegram.actions.pins` config support, defaulting consistently with other safe Telegram actions unless existing action-gate patterns require otherwise. Do not implement automatic session-start fetching, caching, persistence, or broad core changes; use the existing channel message-action seam and existing Telegram token/account/API-root/proxy/network helpers. Add regression coverage for discovery, config gating, no-pin result, pinned-message result, missing token/API failure behavior, and account-scoped gates. Update `docs/channels/telegram.md`, generated/config docs if required by the repo, and `CHANGELOG.md` as a user-facing feature entry. Preserve the closed PRs #38613 and #38718 only as reference, not as patches to copy blindly.
