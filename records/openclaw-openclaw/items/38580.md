## Summary

Keep open: current main still lacks an on-demand Telegram pinned-message read surface, while the related implementation attempts were closed unmerged. The existing Telegram plugin can pin sent messages, but it does not advertise or execute `list-pins`/pinned-message reads.

## What This Changes

Request to let OpenClaw fetch the current pinned message from a Telegram chat via the Telegram Bot API, either on demand or at session start.

## Best Possible Solution

Add a narrow, on-demand Telegram `list-pins` message action backed by `getChat`, document that it returns the current most recent pinned message, and defer automatic session-start context loading until a separate product decision.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main does not advertise or execute Telegram `list-pins`/pinned-message reads, and the related implementation PRs are closed unmerged.

## Solution Assessment

Yes, the best narrow solution is an on-demand Telegram message action rather than automatic session-start fetching. That fits the existing cross-channel `list-pins` action contract and limits the new read surface to an explicit tool invocation.
