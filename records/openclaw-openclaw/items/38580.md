## Summary

Keep open: current main and v2026.6.6 still lack an agent-facing Telegram pinned-message read/list-pins action. The exact prior PRs were closed unmerged, while broader Telegram read and pin/unpin issues are adjacent rather than a safe canonical replacement.

## What This Changes

Requested support for fetching Telegram chat pinned messages as agent-readable context, either on demand or at session start.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand list-pins/read-pinned action using getChat, with explicit action gating, account/target scoping, docs, and tests; treat automatic session-start pinned context as a later opt-in product decision.

## Reproduction Assessment

Yes, at source level. Current main advertises no Telegram list-pins/read-pinned action, has no runtime alias or dispatch branch for it, and v2026.6.6 has the same gap.

## Solution Assessment

No implementation is present yet. The best direction is a narrow Telegram-plugin action around getChat with explicit gating, not a core workaround or broad Telegram message-history read implementation.
