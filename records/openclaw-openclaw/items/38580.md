## Summary

Keep open: current main and the latest release still do not expose a Telegram pinned-message read action, and the two linked implementation PRs were closed unmerged. The request is well-scoped but needs maintainer product and security direction before a fix PR because it adds a new chat-content read surface and likely a new action/config gate.

## What This Changes

Request to let Telegram agents fetch a chat's current pinned message through the Bot API, either on demand or at session start.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read/list-pins action backed by getChat.pinned_message, with explicit authorization, action gating, docs, focused tests, and a separate maintainer decision before any automatic session-start context injection.

## Reproduction Assessment

Yes, at source level. Current main has generic read/list-pins action names and Telegram Bot API plumbing for getChat in other paths, but Telegram does not advertise, map, gate, dispatch, test, or document a pinned-message read action.

## Solution Assessment

Unclear. An on-demand Telegram plugin action is the narrow likely direction, but automatic session-start fetching and any default-enabled config surface need maintainer product/security approval first.
