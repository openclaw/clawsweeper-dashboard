## Summary

Keep open: current main and v2026.6.10 still do not expose a Telegram pinned-message read action, command, config gate, tests, or docs. The request fits Telegram channel improvement work, but it reads persistent chat content and needs maintainer product/security direction before implementation.

## What This Changes

Request to let the Telegram channel fetch and expose the current pinned chat message to agents, either on demand or at session start.

## Best Possible Solution

Approve a Telegram-plugin-owned on-demand pinned-message read action first, backed by getChat, explicit action/config gating, chat/topic authorization, docs, focused tests, and live Telegram proof; treat automatic session-start context injection as a separate opt-in product decision.

## Reproduction Assessment

Yes, at source level. Current main has shared read/list-pins action names and Telegram Bot API support exists, but the Telegram plugin does not advertise, map, gate, dispatch, test, or document pinned-message reads.

## Solution Assessment

Unclear until maintainers choose the product contract. A Telegram-plugin implementation using getChat is likely the right layer, but on-demand versus automatic context, default gating, and security boundaries need approval first.
