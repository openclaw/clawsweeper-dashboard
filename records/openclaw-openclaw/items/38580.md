## Summary

Keep open: current main and v2026.6.6 still do not expose Telegram pinned-message reads through the Telegram message action discovery, runtime dispatch, config gates, or docs. The request is valid and well-scoped, but it needs maintainer product/security approval because it adds a new Telegram data-access path and possibly a new default/config surface.

## What This Changes

Requested support for fetching the current pinned Telegram chat message on demand, with possible later automatic session-start context injection.

## Best Possible Solution

Add a Telegram-plugin-owned on-demand pinned-message read/list-pins action backed by getChat, with explicit action gating, docs, focused tests, and live Telegram proof; defer automatic session-start context injection until separately approved.

## Reproduction Assessment

Yes, at source level: the shared action list already has list-pins, but Telegram does not advertise, map, gate, or dispatch a pinned-message read action. I did not run a live Telegram repro because current main has no action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely best MVP is an on-demand Telegram-plugin action backed by getChat, not automatic session-start context injection or a core workaround.
