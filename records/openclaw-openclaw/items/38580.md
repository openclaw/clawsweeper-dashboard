## Summary

Keep open: current main and v2026.6.6 still lack Telegram agent-facing pinned-message reads. Existing Telegram pin helpers cover delivery/write operations, while the closed PR attempts that targeted this issue were not merged; maintainer product and security review is needed before adding an on-demand `getChat`-backed action or automatic context injection.

## What This Changes

The issue requests Telegram support for agents to fetch pinned chat messages through the Bot API, either on demand or at session start.

## Best Possible Solution

Implement a Telegram-plugin-owned, on-demand pinned-message read action after maintainer approval, with explicit gating, docs, focused tests, and no default automatic session-start injection.

## Reproduction Assessment

Yes, at source level. Current main has shared `list-pins` action vocabulary, but the Telegram plugin does not advertise, map, gate, or dispatch a pinned-message read path.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The likely best direction is an on-demand Telegram plugin action backed by `getChat`, while automatic session-start context injection should remain out of scope unless explicitly approved.
