## Summary

Keep open: current main still lacks a Telegram pinned-message read path, and the related implementation PRs were closed unmerged. The shared `list-pins` message-action contract already exists, so this is a bounded Telegram plugin feature rather than a core API gap.

## What This Changes

Request to let Telegram agents read the current pinned chat message on demand, likely via Bot API `getChat.pinned_message`, so pinned context can be reused across sessions.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by `getChat.pinned_message`, with config/schema/docs/tests, and coordinate with the related Telegram pin/unpin request if a single pins action-family PR is cheaper.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main shows Telegram lacks `list-pins`/`getpinned` discovery, runtime dispatch, config gating, and docs.

## Solution Assessment

Yes: the narrow maintainable solution is an on-demand Telegram `list-pins` action inside the existing Telegram plugin message-action adapter. Automatic session-start context injection should remain a separate product decision.
