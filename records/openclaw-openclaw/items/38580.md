## Summary

Current main still lacks on-demand Telegram pinned-message reads. The shared `list-pins` action and CLI formatting exist, but the bundled Telegram plugin does not advertise, gate, dispatch, fetch, normalize, or document that read capability; the related prior PRs are useful context but were closed unmerged.

## What This Changes

Add an on-demand Telegram pinned-message read so agents can fetch pinned chat context via the Telegram Bot API.

## Best Possible Solution

Implement Telegram plugin support for the existing shared `list-pins` action, backed by Bot API `getChat.pinned_message`, returning a normalized `pins` array with zero or one item plus config/docs/tests for the Telegram limitation.
