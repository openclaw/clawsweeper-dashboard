## Summary

Current main does not implement Telegram pinned-message reads: the bundled Telegram message-action discovery and runtime omit `list-pins`, and the related implementation PRs were closed unmerged. This remains a narrow feature request, with the safest scope being an on-demand action rather than automatic session-start context injection.

## What This Changes

The issue requests a Telegram Bot API-backed way for agents to read a chat's pinned message on demand or at session start.

## Best Possible Solution

Add an on-demand Telegram `list-pins` message action backed by `getChat().pinned_message`, with action gating, docs, changelog, and regression coverage; defer automatic startup ingestion to a separate product discussion.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence absence check because Telegram currently does not advertise or handle `list-pins`.

## Solution Assessment

Yes for the on-demand half of the request: extending the existing channel message-action surface is the narrowest maintainable path. Automatic session-start fetching would be a separate privacy/product behavior and should not be bundled into the first repair.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/38580 as an on-demand Telegram pinned-message read. Add `list-pins` support to the bundled Telegram channel by using the existing Telegram bot token/account resolution and grammY Bot API `getChat` to read `pinned_message`; return `{ ok: true, pins: [...] }` with an empty array when there is no pin and enough normalized fields for the existing CLI formatter to show message id, text/caption, timestamp, chat, and sender when available. Expose the action from `extensions/telegram/src/channel-actions.ts`, handle it in `extensions/telegram/src/action-runtime.ts`, add a `channels.telegram.actions.pins` gate defaulting consistently with other channel pins actions, and update types/zod/generated config metadata as required. Add focused tests for discovery, gating, successful pin read, no-pin result, token/account handling, and unsupported/Telegram API errors. Update `docs/channels/telegram.md` and add a user-facing `CHANGELOG.md` entry. Do not add automatic session-start pinned-context injection, persistence, caching, or unrelated Telegram behavior. Treat closed unmerged PRs #38613 and #38718 as reference only; do not revive obsolete `src/telegram` paths.
