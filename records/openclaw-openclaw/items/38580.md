## Summary

Keep open: current main still lacks a Telegram pinned-message read path, while the shared message-action surface already supports `list-pins` and the related implementation PRs were closed unmerged.

## What This Changes

Request adding an on-demand Telegram pinned-message read capability, using Bot API `getChat.pinned_message`, so agents can pull pinned chat context.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` action backed by `getChat.pinned_message`, returning zero or one normalized pin behind a consistent `actions.pins` gate, while leaving automatic startup context injection for later.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection on current main gives high confidence that Telegram still has no `list-pins`, `listPins`, or `/getpinned` read path.

## Solution Assessment

Yes: the narrow maintainable solution is an on-demand Telegram `list-pins` message action in the Telegram plugin, following the existing Slack/Discord pins pattern. Automatic session-start context injection should remain a separate product decision.
