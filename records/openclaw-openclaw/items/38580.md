## What This Changes

Request to let the Telegram plugin read the current pinned chat message via the Telegram Bot API for persistent chat context.

## Best Possible Solution

Add a Telegram plugin-owned on-demand list-pins message action backed by getChat and ChatFullInfo.pinned_message, gate it with channels.telegram.actions.pins, document it, and leave automatic session-start context injection for a separate product decision.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and latest-release inspection show Telegram still does not advertise, gate, or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel list-pins action inside the Telegram plugin is the maintainable solution. Automatic session-start context loading should stay out of scope until maintainers explicitly choose that behavior.
