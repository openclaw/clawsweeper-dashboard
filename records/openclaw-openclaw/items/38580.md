## Summary

Keep open: current main still lacks a Telegram pinned-message read path, while the shared message-action contract already has `list-pins` and the related implementation PRs were closed unmerged.

## What This Changes

Request to let Telegram agents fetch the current pinned chat message via the Telegram Bot API on demand, with automatic session-start context left as an optional future mode.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` action backed by `getChat.pinned_message`, return zero or one normalized pin, add an `actions.pins` gate plus docs/tests, and leave automatic startup context injection for a separate product decision.

## Solution Assessment

Yes: the narrow maintainable path is an on-demand Telegram `list-pins` message action inside the existing Telegram plugin adapter. That fits the shared action contract and avoids making automatic session-start context injection part of the first implementation.
