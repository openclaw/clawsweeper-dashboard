## Summary

Keep open: current main still has no Telegram pinned-message read action or `/getpinned` path, while the issue discussion and two closed unmerged PRs show a clear plugin-owned implementation shape that has not landed.

## What This Changes

Request to let Telegram agents fetch the current pinned chat message on demand, with automatic session-start context injection discussed as a possible later mode.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` action backed by `getChat`, return zero or one current pinned message, gate it with `channels.telegram.actions.pins`, document it, and leave automatic session-start context injection for separate product review.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main and `v2026.5.20` gives high confidence that Telegram still does not advertise, gate, or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is the maintainable solution. Automatic session-start context loading should remain out of scope until maintainers explicitly choose that behavior.
