## Summary

Keep open: current main still lacks a Telegram pinned-message read action. Telegram has generic `list-pins` action vocabulary and write-side pin helpers, but the Telegram plugin does not advertise, gate, or route a pinned-message read backed by `getChat`, and the two related implementation attempts are closed unmerged.

## What This Changes

The issue requests a Telegram Bot API path for agents to read the current pinned chat message on demand or at session start.

## Best Possible Solution

Add a Telegram plugin-owned on-demand `list-pins` message action backed by `getChat`, return zero or one current pinned message, gate and document it consistently, and leave automatic session-start context injection for a separate decision.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that Telegram does not advertise or execute pinned-message reads.

## Solution Assessment

Yes for the narrow on-demand path: extending the existing cross-channel `list-pins` action in the Telegram plugin is more maintainable than adding automatic session-start pinned-context loading now.
