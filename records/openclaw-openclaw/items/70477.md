## Summary

Keep open. Current main does not implement the central bootstrap-token/chat-activity work, and the PR still has blocking security/correctness findings plus missing real behavior proof, so this needs human review rather than cleanup closure.

## What This Changes

The PR adds loopback Control UI bootstrap-token auth, routes bootstrap tokens through web UI auth helpers, adds chat activity/run-state UI plumbing, updates dashboard messaging/tests, and moves a Telegram temp-dir test helper inside the plugin boundary.

## Best Possible Solution

Keep the PR open, require a server-trusted local provenance design or keep bootstrap config authenticated, fix stale pending-abort terminal handling, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces both review issues: the bootstrap branch can bypass normal auth and include a gateway token, and the pending-abort terminal branch returns a terminal state that downstream code treats as real completion.

## Solution Assessment

No. The intended local auto-connect and run-state UI may be useful, but the current patch is not the best final solution until the token-bearing bypass is backed by server-trusted provenance or removed, and stale abort terminal events are swallowed before cleanup runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind bootstrap bypass to server-trusted local provenance:** `src/gateway/control-ui.ts:690-692`
  - body: This branch skips normal Control UI auth and can include gatewayToken when the socket appears loopback and client-controlled browser headers look local. A same-host reverse proxy that connects to the backend over loopback but omits forwarded headers can let external clients spoof Host, Origin, and Sec-Fetch headers and reach the unauthenticated token response; keep this route authenticated or prove direct local provenance with server-owned data.
  - confidence: 0.9
- **[P2] Swallow pending-abort terminal events:** `ui/src/ui/controllers/chat.ts:443-445`
  - body: When a terminal event arrives for chatAbortPendingRunId, this branch clears the marker but returns the terminal state. handleChatGatewayEvent then runs terminal cleanup and can reset tool streams or reload history while a replacement run is active; return null after clearing the marker so stale abort events do not disturb the new run.
  - confidence: 0.93
