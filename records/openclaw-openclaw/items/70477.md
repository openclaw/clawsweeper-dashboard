## Summary

Keep this PR open. Current main still has the authenticated bootstrap contract, so the intended work is not implemented, and the current PR branch still has blocking review findings: a security-sensitive unauthenticated token path and a pending-abort terminal-event race. Earlier duplicate auth/header-sanitization review comments appear addressed in the latest branch, but these remaining blockers should be fixed before merge.

## What This Changes

The PR adds loopback Control UI bootstrap-token auth, propagates the token through the web UI, adds chat run activity/status plumbing and tests, and includes related dashboard, gateway, and Telegram test-boundary updates.

## Best Possible Solution

Keep the PR open and require changes: either keep bootstrap config behind normal auth or prove direct local provenance without trusting spoofable request headers, return null for stale pending-abort terminal events, and consider splitting or rebasing the branch so bootstrap auth and chat activity can be reviewed independently.

## Reproduction Assessment

Yes. The auth issue is reproducible by source inspection with a same-host reverse proxy that forwards to the Gateway over loopback without forwarded headers while the external caller supplies loopback Host/Origin/Sec-Fetch headers; the pending-abort issue follows the handleChatEvent -> handleTerminalChatEvent path in the PR head.

## Solution Assessment

No. The PR direction addresses a real local bootstrap-auth UX gap, but the current implementation is not the best mergeable solution because it exposes a token through an unauthenticated branch and still lets stale abort terminals drive terminal cleanup for a replacement run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bind bootstrap bypass to non-spoofable local provenance:** `src/gateway/control-ui.ts:690-692`
  - body: This branch treats a request as local when the socket peer is loopback and no forwarding headers are present, then uses client-controlled Host/Origin/Sec-Fetch headers before skipping normal auth and returning gatewayToken. A same-host reverse proxy that omits forwarded headers can make external traffic satisfy the socket check, so unauthenticated callers can spoof those headers and receive the token. Keep bootstrap config authenticated unless the server can prove the original client is local.
  - confidence: 0.9
- **[P2] Swallow terminal events from pending-abort runs:** `ui/src/ui/controllers/chat.ts:443-445`
  - body: When a stale terminal event arrives for chatAbortPendingRunId, this returns payload.state after clearing the marker. handleChatGatewayEvent treats that as a real terminal transition and can reset tool streams or reload history while a replacement run is active; return null after clearing the marker so the old run cannot disturb the new one.
  - confidence: 0.94
