## Summary

This PR should stay open. Current `main` still serves the Control UI bootstrap config through normal gateway auth and does not have the bootstrap-token contract, while the PR is an unmerged security-sensitive gateway/UI change with concrete blocking review findings still present.

## What This Changes

The PR adds loopback Control UI bootstrap-token auth, routes that token through web UI gateway/channel auth helpers, adds chat activity/run-state UI plumbing, adjusts dashboard messaging, adds gateway/UI tests, and moves a Telegram test temp-dir helper inside the plugin boundary.

## Best Possible Solution

Keep this PR open until the bootstrap endpoint either remains normally authenticated or proves direct local provenance using server-trusted data, stale pending-abort terminal events are swallowed before terminal cleanup runs, and the user-facing docs/changelog are updated for the accepted behavior.

## Reproduction Assessment

Yes. The main-vs-PR behavior is reproducible by inspecting the bootstrap contract, auth gate, and HTTP tests; the PR bug is reproducible by following the pending-abort branch where it returns a terminal state that `handleChatGatewayEvent` treats as real terminal completion.

## Solution Assessment

No. The intended local auto-connect and activity UI may be useful, but this patch is not yet the narrowest maintainable solution because it exposes a gateway token through a new unauthenticated path and still lets stale abort terminal events disturb active-run handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind bootstrap token bypass to server-trusted provenance:** `src/gateway/control-ui.ts:690-692`
  - body: This branch skips normal auth when the socket looks loopback and client-controlled browser headers look local, then includes `gatewayToken` in the bootstrap response. A same-host reverse proxy that omits forwarded headers can make external traffic arrive from loopback, and non-browser clients can spoof `Host`, `Origin`, and `Sec-Fetch-*` headers. Keep bootstrap config authenticated unless the server can prove the request came directly from the local Control UI.
  - confidence: 0.9
- **[P2] Swallow terminal events from pending-abort runs:** `ui/src/ui/controllers/chat.ts:443-445`
  - body: When a terminal event arrives for `chatAbortPendingRunId`, this branch clears the marker but still returns the terminal state. `handleChatGatewayEvent` then runs terminal cleanup and may reset tool streams or reload history while a replacement run is active. Return `null` after clearing the marker so stale abort events do not disturb the new run.
  - confidence: 0.93
- **[P3] Add the required changelog entry:** `CHANGELOG.md:7`
  - body: This PR changes user-facing Control UI auth, dashboard messaging, and chat activity behavior, but `CHANGELOG.md` is not touched. Repo policy requires user-facing fix/feat/perf changes to add a single-line entry under `Unreleased` with appropriate non-bot contributor credit.
  - confidence: 0.86
