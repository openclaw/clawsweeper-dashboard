## Summary

Keep this PR open: the central bootstrap-token and chat-activity work is not on current main, but the PR head still has a security-sensitive unauthenticated token path, a pending-abort terminal-event bug, no inspectable real behavior proof, and a dirty merge state.

## What This Changes

This PR adds loopback Control UI bootstrap-token auth, routes the bootstrap token through web UI helpers, adds chat activity/run-state UI plumbing, updates dashboard messaging/tests, and moves a Telegram temp-dir test helper inside the plugin boundary.

## Best Possible Solution

Keep the PR open until the bootstrap path either remains authenticated or uses maintainer-approved server-trusted local provenance, the pending-abort terminal event is swallowed, the branch is rebased, and redacted real behavior proof is added.

## Reproduction Assessment

Yes. Source inspection of PR head f741016c7705aaad3e9a8e10466d5428b1f7d59f shows both remaining review issues: the bootstrap branch can bypass normal auth and include a gateway token, and pending-abort terminal events return a terminal state that downstream code treats as real completion.

## Solution Assessment

No. The local auto-connect and run-state UI may be useful, but this is not the best final solution until the token-bearing bypass has server-trusted provenance or is removed and stale abort terminal events are swallowed before terminal cleanup runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind bootstrap bypass to server-trusted local provenance:** `src/gateway/control-ui.ts:690-692`
  - body: This branch skips normal Control UI auth and can include gatewayToken when the socket appears loopback and browser headers look local. A same-host reverse proxy that connects to the backend over loopback but omits forwarded headers can let external clients spoof Host, Origin, and Sec-Fetch headers and reach the unauthenticated token response; keep this route authenticated or prove direct local provenance with server-owned data.
  - confidence: 0.9
- **[P2] Swallow pending-abort terminal events:** `ui/src/ui/controllers/chat.ts:443-445`
  - body: When a terminal event arrives for chatAbortPendingRunId, this branch clears the marker but returns the terminal state. handleChatGatewayEvent then runs terminal cleanup and can reset tool streams or reload history while a replacement run is active; return null after clearing the marker so stale abort events do not disturb the new run.
  - confidence: 0.93
