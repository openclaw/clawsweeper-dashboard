## Summary

Keep open. This is a draft PR with the protected `maintainer` label, and the patch has a blocking correctness issue: it only discovers the `chrome-linux/chrome` Playwright layout, while OpenClaw pins Playwright 1.59.1 where Linux x64 Chromium lives under `chrome-linux64/chrome`.

## What This Changes

The PR adds Linux detection for Playwright-managed Chromium caches and updates Docker/browser docs and the changelog.

## Best Possible Solution

Update this PR to mirror Playwright's current Linux executable layouts, add regression coverage for both `chrome-linux64/chrome` and `chrome-linux/chrome`, and keep the broader Docker symlink/path policy in the canonical Docker browser thread.

## Reproduction Assessment

Yes. Source inspection shows current main returns null when the only browser is a Playwright cache entry, and Playwright 1.59.1 plus the related Docker report provide a concrete Linux x64 failing path.

## Solution Assessment

No. The direction is narrow, but the implementation must include Playwright's Linux x64 `chrome-linux64/chrome` layout and test it before the Docker/Linux auto-detection claim is correct.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Include the Playwright Linux x64 layout:** `extensions/browser/src/browser/chrome.executables.ts:498`
  - body: This only looks for `<cache>/chromium-*/chrome-linux/chrome`, but OpenClaw pins `playwright-core` 1.59.1, whose registry uses `chrome-linux64/chrome` on Linux x64 and `chrome-linux/chrome` only on Linux arm64. Please add the x64 path and a regression test for it.
