## Summary

Keep open: current main still lacks the package-local absolute-path guardrail, and this PR is directionally useful, but it misses escaped Windows path leaks and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds a shared scanner for package-local absolute paths in dist text artifacts, calls it from release-check and tarball integrity validation, and adds release-check regression tests.

## Best Possible Solution

Keep the shared release/tarball scanner seam, add escaped-backslash Windows coverage with regression tests, and require redacted packed-tarball install output before merge.

## Reproduction Assessment

Yes, source-level: current main's release-check content scan only checks private QA markers, and the tarball checker does not scan dist text for /Users or /home build-path leaks. I did not run package E2E in this read-only review.

## Solution Assessment

No, not yet: a shared scanner is the right maintainable seam, but this implementation should also detect escaped Windows path representations and include inspectable after-fix package proof before merge.
