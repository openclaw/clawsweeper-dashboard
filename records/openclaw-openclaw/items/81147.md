Full review comments:

- **[P2] Detect escaped Windows paths:** `scripts/lib/package-local-path-scan.mjs:6`
  - body: The Windows pattern only matches text containing literal single backslashes. Built JS/JSON commonly records Windows absolute paths as escaped source text like `C:\\Users\\runner\\...` or as `file:///C:/Users/...`, so a Windows-built package can still ship the hard-coded build root while both new checks pass. Add matching and tests for those encoded forms before relying on this as the portability gate.
