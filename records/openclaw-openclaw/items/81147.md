## Summary

Keep open: the package-portability hardening is still useful because current main lacks general dist absolute-path content scanning, but this PR has a concrete scanner gap for escaped Windows paths and still lacks contributor-supplied real behavior proof.

## What This Changes

The branch adds a shared local absolute-path scanner for packaged dist artifacts, wires it into release-check and tarball validation, and adds release-check regression coverage.

## Best Possible Solution

Keep the shared scanner approach, add escaped Windows path coverage with a regression case, and require redacted packed-install output before merge.

## Reproduction Assessment

Yes, source-level: current main only scans private QA content markers and tarball import closure, so a dist text artifact with ordinary local absolute paths is not rejected. I did not run package E2E because this review is read-only.

## Solution Assessment

No, not yet: the shared scanner is the right seam, but it should catch escaped Windows path representations and include inspectable after-fix package proof before merge.
