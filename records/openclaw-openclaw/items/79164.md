## Summary

Keep open. Current main has validation, write-time backup rotation, SecretRef preflight, and invalid-snapshot/doctor recovery, but it still lacks the requested configurable post-apply health check and automatic rollback for schema-valid config changes that leave the gateway unhealthy.

## What This Changes

Add automatic config-write safeguards that back up, health-check after apply, and roll back bad gateway config changes, with opt-in config knobs.

## Best Possible Solution

Define a gateway-owned transactional config write policy that reuses existing validation and backups, then health-gates promotion or restoration for restart-requiring writes with explicit rollback boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested post-apply health-check/rollback path is missing, but the report does not include a concrete schema-valid config that reproduces a current-main gateway failure.

## Solution Assessment

Unclear. The requested direction is plausible, but config knobs, health definition, timeout behavior, and rollback scope need maintainer design because current docs reserve whole-file restore for doctor-managed repair.
