## Summary

Keep open. Current main has validation, write-time backup rotation, SecretRef preflight, and last-known-good recovery for invalid snapshots, but it still lacks the requested configurable post-apply health check and automatic rollback for schema-valid config changes that leave the gateway unhealthy.

## What This Changes

Request automatic safeguards around `config.apply` and `config.patch`: pre-write backup, post-apply gateway health check, automatic rollback, and a configurable config surface.

## Best Possible Solution

Define a gateway-owned transactional config write policy that reuses existing validation and backups, then health-gates promotion or restoration for restart-requiring writes with explicit rollback boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested post-apply health-check and rollback path is absent, but the report does not include a concrete schema-valid config that crashes current main.

## Solution Assessment

Unclear. The requested direction is plausible, but health definition, timeout semantics, rollback scope, default/opt-in behavior, and interaction with doctor-owned whole-file restore need maintainer design before implementation.
