## Summary

Keep open. Current main still has validation, backup rotation, doctor repair, and invalid-snapshot recovery, but it does not provide the requested configurable post-apply health gate or automatic rollback for schema-valid config changes that leave the gateway unhealthy.

## What This Changes

Request an opt-in automatic backup, post-apply health check, and rollback path for `config.apply`/`config.patch` changes that leave the gateway unhealthy.

## Best Possible Solution

Define a gateway-owned transactional config write contract that reuses existing validation/backups and health-gates restart-requiring writes before promotion, with explicit rollback scope, timeout semantics, and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection confirms the requested health-gated rollback path is absent, but the report does not include a concrete schema-valid config that reliably fails current main after restart.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable solution needs maintainer design for health definition, timeout behavior, rollback boundaries, and config surface before code work starts.
