## Summary

Keep open. Current main still validates and writes config, rotates backups, and has doctor/invalid-config recovery, but it does not provide the requested configurable post-apply health gate or automatic rollback for schema-valid config changes that make the gateway unhealthy.

## What This Changes

Request automatic pre-apply backup, post-apply gateway health checking, and rollback for `config.apply` / `config.patch` writes that break the gateway.

## Best Possible Solution

Define a gateway-owned transactional config write contract that reuses existing validation/backups and health-gates restart-requiring writes before promotion, with explicit rollback scope, timeout semantics, and docs.

Not applicable as a feature request. Source inspection confirms the requested health-check/rollback path is absent, but there is no concrete schema-valid config sample that reproduces a current-main gateway failure.

## Solution Assessment
