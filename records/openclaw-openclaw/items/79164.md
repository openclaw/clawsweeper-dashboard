## Summary

Keep open. Current main has schema validation, write-time backup rotation, last-known-good promotion, and doctor-managed recovery, but it still lacks the requested configurable post-apply health check and automatic rollback for schema-valid config changes that leave the Gateway unhealthy.

## What This Changes

Request to add automatic safeguards around `config.apply` and `config.patch`: backup, post-restart health checking, rollback, and timeout/config controls.

## Best Possible Solution

Define a gateway-owned transactional config-write policy that reuses existing backups and last-known-good promotion, then health-gates restart-requiring writes with explicit health, timeout, supervisor, and rollback boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested post-apply health-check and rollback path is missing, but the report does not provide a concrete schema-valid config that reproduces a current-main Gateway failure.

## Solution Assessment

Unclear. The requested direction is plausible, but the health definition, rollback scope, timeout semantics, supervisor behavior, and config surface need maintainer design before implementation.
