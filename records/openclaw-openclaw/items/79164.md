## Summary

Keep open. Current main still validates and writes config, maintains backup rotation, and has invalid-config/doctor recovery, but it does not implement the requested configurable post-apply health gate or automatic rollback for schema-valid config changes that make the gateway unhealthy.

## Best Possible Solution

Define a gateway-owned transactional config write contract that reuses existing validation/backups and health-gates restart-requiring writes before promotion, with explicit rollback scope, timeout semantics, and docs.
