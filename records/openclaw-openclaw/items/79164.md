## Summary

Keep open. Current main has config validation, SecretRef write preflight, backup rotation, last-known-good recovery for invalid snapshots, and restart scheduling, but it still lacks the requested health-gated post-apply promotion and automatic rollback for schema-valid config changes that make the Gateway unhealthy.

## What This Changes

Request automatic safeguards around `config.apply` and `config.patch`: backup before apply, health check after restart, automatic rollback on health failure, and configurable timeout/policy knobs.

## Best Possible Solution

Define a Gateway-owned transactional config write contract that reuses existing validation and backups, then health-gates promotion or restoration for restart-requiring writes with documented rollback boundaries and tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested health-gated rollback path is missing, but there is no concrete schema-valid config sample that reproduces a current-main Gateway failure.

## Solution Assessment

Unclear. The requested direction is plausible, but health definition, timeout behavior, rollback scope, config surface, and failure semantics need maintainer design before implementation.
