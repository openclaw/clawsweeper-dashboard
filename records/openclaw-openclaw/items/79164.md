## Summary

Keep open. Current main has validation, backup rotation, SecretRef preflight, invalid-config last-known-good recovery, and doctor-owned repair, but it still lacks a configurable post-apply health check and automatic rollback for schema-valid config that leaves the Gateway unhealthy.

## What This Changes

Request configurable automatic backup, post-apply health checking, and rollback for `config.apply`/`config.patch` writes that make the Gateway unhealthy after restart.

## Best Possible Solution

Define a gateway-owned transactional config write policy that reuses existing validation and backups, then health-gates promotion or restoration for restart-requiring writes with explicit rollback boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested post-apply health-check and rollback path is missing, but the issue does not include a concrete schema-valid config that reproduces a current-main Gateway failure.

## Solution Assessment

Unclear. The requested direction is plausible, but health definition, observation window, rollback scope, timeout semantics, and the new config surface need maintainer design before code is safe.
