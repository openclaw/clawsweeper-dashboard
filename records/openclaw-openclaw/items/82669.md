## Summary

Keep open. Current main already has active-run model plumbing for `session_status`, but the same path still has no active/resolved thinking-level plumbing, so the tool can still render a persisted or default `Think: off` while the executing run is using `high`.

## What This Changes

The issue reports that `session_status` displays `Think: off` after `/think high` while the UI/runtime shows `high`.

## Best Possible Solution

Thread the active run's resolved thinking level through the existing tool-construction context and use it only for semantic-current or omitted-session lookups, preserving explicit other-session status behavior.

## Reproduction Assessment

Yes, at source level: the runtime computes a resolved thinking level, but `session_status` only receives active model context and persisted/default thinking state. I did not execute the live UI/tool flow in this read-only review.

## Solution Assessment

Yes: mirroring the existing active-model plumbing for active thinking is the narrow maintainable fix. Persisting transient run thinking into session state would be riskier and could regress explicit session/default semantics.
