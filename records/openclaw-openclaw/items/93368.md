Full review comments:

- **[P1] Use realpath-aware containment before reading context files:** `src/agents/sessions/resource-loader.ts:121-124`
  - body: `resolve()` does not collapse symlinks, so a cwd such as `workspace/linked-outside` can pass this textual prefix check while `existsSync` and `readFileSync` follow the symlink and load an outside `AGENTS.md`. Canonicalize both sides with a realpath-aware guard before any ancestor read, and cover the symlink escape.
  - confidence: 0.91
- **[P1] Use platform-safe descendant checks:** `src/agents/sessions/resource-loader.ts:123`
  - body: On Windows, `node:path.resolve()` returns backslash paths, but this check appends `/`, so `C:\ws\project` does not match the `C:\ws/` prefix and legitimate nested workspace context fails closed. Use `path.relative`/`path.sep` or a shared path guard, and cover a win32 descendant path.
  - confidence: 0.88
