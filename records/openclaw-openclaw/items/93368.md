  - body: `resolve()` does not collapse symlinks, so a cwd such as `workspace/linked-outside` can pass this textual prefix check while `existsSync`/`readFileSync` follow the symlink and load an outside `AGENTS.md`.   - body: On Windows, `node:path.resolve()` returns backslash paths, but this check appends `/`, so `C:\ws\project` does not match the `C:\ws/` prefix and legitimate nested workspace context fails closed. Use `path.relative`/`path.sep` or the shared path guard, and cover a win32 descendant path.
  - confidence: 0.88
