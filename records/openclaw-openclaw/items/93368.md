## Summary

Keep open. The linked security problem is real on current main, but this PR uses `cwd` itself as the stop boundary, which removes inherited workspace-root context for runs whose task cwd is below the workspace; the PR also supplies unit-test output rather than real loaded-context proof.

## What This Changes

The PR changes `loadProjectContextFiles()` so `AGENTS.md`/`CLAUDE.md` ancestor discovery stops immediately at `cwd` instead of walking to the filesystem root.

## Best Possible Solution

Use an explicit trusted workspace boundary, walk from the task cwd up to that boundary only, and prove both nested-workspace inheritance and outside-workspace exclusion.

## Reproduction Assessment

Yes, source-reproducible. Current main walks to `/`, while the PR diff breaks immediately at `cwd`; the code and docs show `cwd` can be a task directory distinct from the workspace used for bootstrap context.

## Solution Assessment

No. The narrow, maintainable fix is a separate workspace-boundary walk, not treating the current task cwd as the whole workspace.
