## Summary

Current main already implements the requested cleanup: Dream Diary narrative sessions are deleted after accepted subagent runs, stale dreaming session-store entries and orphan transcripts are scrubbed, and the original leak fix shipped in v2026.4.20.

## What This Changes

Automatically remove memory-core Dream Diary `dreaming-narrative-*` sub-sessions after light, REM, and deep dreaming runs so completed sessions do not accumulate.

## Best Possible Solution

Keep the shipped memory-core and gateway cleanup path; a new report should only track a fresh leak if a current v2026.5.2-or-newer install still creates new completed narrative sessions that survive cleanup.

## Reproduction Assessment

No for current main: the reported leak path is covered by source and regression tests showing accepted narrative sessions are deleted and stale artifacts are scrubbed. The original failure mode is source-backed by PR #67023, but it is not present in the inspected current-main code.

## Solution Assessment

Yes: deleting plugin-owned narrative sessions in `finally`, preserving owner-scoped gateway deletion, and scrubbing stale artifacts is the narrow maintainable fix. A new TTL/config option is unnecessary for the reported cleanup behavior.
