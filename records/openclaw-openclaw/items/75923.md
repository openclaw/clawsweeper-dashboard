## Summary

Keep open. Current main still lacks the requested run-scoped consecutive cross-tool error detector and config/docs surface, and this issue is paired with open closing PR #75924, so cleanup should wait for that PR to merge or be closed.

## What This Changes

Request adding a run-scoped `consecutive_errors` detector and `tools.loopDetection.consecutiveErrorThreshold` config to block cross-tool tails where every completed tool call fails.

## Best Possible Solution

Review and land or close #75924; the accepted path should extend the existing run-scoped detector with schema/help/docs updates and same-run plus cross-run isolation coverage.

## Reproduction Assessment

Yes, from source inspection. With loop detection enabled, same-run completed failed outcomes for distinct tools get `error:` result hashes, but `detectToolCallLoop` has no detector that counts that cross-tool error tail.

## Solution Assessment

Yes at the direction level. Extending the existing run-scoped loop detector is narrower than a turn cap or session retry budget, while #75924 should carry the concrete review for config shape, defaults, docs, and edge cases.
