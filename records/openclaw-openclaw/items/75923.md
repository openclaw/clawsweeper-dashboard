## Summary

Keep open. Current main still lacks the requested detector/config/docs, and the linked closing PR #75924 is the right vehicle to review before this issue is resolved.

## What This Changes

Add a run-scoped `consecutive_errors` loop detector and `tools.loopDetection.consecutiveErrorThreshold` config to stop same-run cross-tool failure cascades.

## Best Possible Solution

Review and land or close #75924; the accepted implementation should extend the existing run-scoped loop detector with schema, docs, and focused detector plus hook-path coverage.

## Reproduction Assessment

Yes, source-reproducible from current main. Record same-run completed outcomes for distinct tools with `error:` result hashes, then check the next distinct tool call; the current detector has no cross-tool error-tail counter.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while #75924 still needs normal review for defaults, schema shape, docs wording, and edge cases.
