## Summary

Keep open. Current main still lacks the requested run-scoped consecutive cross-tool error cascade detector, and this issue is paired with open closing PR #75924.

## What This Changes

Requests a run-scoped loop detector and config threshold that blocks the next tool call after a same-run streak of completed error outcomes across different tools.

## Best Possible Solution

Review and land or close #75924; an accepted implementation should extend the existing run-scoped detector with schema/help/docs updates and same-run plus cross-run isolation coverage.

## Reproduction Assessment

Yes, from source inspection. With loop detection enabled, same-run completed outcomes for distinct tools whose `resultHash` starts with `error:` are recorded, but current `detectToolCallLoop` has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue direction level. Extending the existing run-scoped loop detector is narrower than a turn cap or session retry budget, while #75924 should carry the concrete review for config shape, defaults, docs, and edge cases.
