## Summary

Keep open. Current main still lacks the requested cross-tool consecutive-error detector/config/docs, and the item is paired with open closing PR #75924, so cleanup should wait for that PR to be reviewed and either merged or closed.

## What This Changes

Add run-scoped detection for consecutive failed tool outcomes across different tools, with a configurable threshold under loop detection.

## Best Possible Solution

Review and land or close #75924; the accepted fix should extend the existing run-scoped loop detector with schema, docs, and focused detector plus hook-path coverage.

## Reproduction Assessment

Yes. Source inspection shows that same-run completed outcomes for distinct tools with error-prefixed result hashes can be recorded, then the next distinct tool call reaches detectToolCallLoop with no detector that counts consecutive cross-tool failures.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while the paired PR still needs normal review for defaults, schema shape, docs wording, and edge cases.
