## Summary

Keep open. Current main still lacks the requested run-scoped cross-tool consecutive-error detector and config/docs surface, and the issue is paired with open closing PR #75924.

## What This Changes

Request adding a run-scoped `consecutive_errors` detector and `tools.loopDetection.consecutiveErrorThreshold` to block all-error cross-tool streaks.

## Best Possible Solution

Review and land or close #75924; the accepted implementation should extend the existing run-scoped detector with schema/help/docs updates and same-run plus cross-run isolation coverage.

## Reproduction Assessment

Yes, from source inspection. With loop detection enabled, record same-run completed outcomes for distinct tools whose `resultHash` starts with `error:`, then ask `detectToolCallLoop` about another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the direction level. Extending the existing run-scoped loop detector is narrower than a turn cap or session retry budget, while #75924 should carry the concrete review for config shape, defaults, docs, and edge cases.
