## Summary

Keep open. Current main still lacks the requested run-scoped cross-tool consecutive-error detector and threshold config, and open PR #75924 is the active closing implementation path.

## What This Changes

Add a run-scoped loop detector that blocks the next tool call after a configurable streak of completed failing tool outcomes across different tools in the same run.

## Best Possible Solution

Review and either land or close #75924; an accepted fix should live in the existing loop detector and before-tool-call hook with schema, docs, and focused unit plus hook-path coverage.

## Reproduction Assessment

Yes. Record completed outcomes for distinct tools in one run with error-prefixed result hashes, then call detectToolCallLoop for another distinct tool; current main has no detector that blocks that cross-tool failing tail.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary; the open PR still needs review for the exact implementation choices.
