## Summary

Keep open. Current main still lacks the requested consecutive-error detector/config/docs, and the issue is paired with open closing PR #75924, so cleanup should wait for that PR to be reviewed and merged or closed.

## What This Changes

Add a run-scoped loop detector that blocks the next tool call after a configurable streak of completed cross-tool error outcomes.

## Best Possible Solution

Review and land or close #75924; the accepted fix should extend the existing run-scoped loop detector with schema, docs, and focused detector plus hook-path coverage.

## Reproduction Assessment

Yes, from source inspection. Record same-run completed outcomes for distinct tools with `error:` result hashes, then call `detectToolCallLoop` for another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while #75924 should carry the concrete implementation review for config shape, defaults, docs, and edge cases.
