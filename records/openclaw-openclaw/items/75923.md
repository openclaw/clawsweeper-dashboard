## Summary

Keep open. Current main still lacks a `consecutive_errors` detector or `consecutiveErrorThreshold` config, and this issue is paired with open closing PR #75924, so cleanup should wait for that PR to be reviewed and merged or closed.

## What This Changes

Add a run-scoped loop detector that blocks the next tool call after a configurable streak of completed error outcomes across different tools.

## Best Possible Solution

Review and land or close #75924; the accepted fix should extend the existing run-scoped loop detector with schema, docs, and focused detector plus hook-path coverage.

## Reproduction Assessment

Yes. From source inspection, record same-run completed outcomes for distinct tools with `error:` result hashes, then call `detectToolCallLoop` for another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary; the paired PR still needs review for defaults, schema shape, docs wording, and edge cases.
