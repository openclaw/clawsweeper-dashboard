## What This Changes

The issue requests a run-scoped loop detector that blocks the next tool call after a configurable streak of completed same-run tool failures across different tools.

## Best Possible Solution

Review and land or close #75924; an accepted fix should extend the existing run-scoped loop detector with schema/help/docs updates and focused unit plus hook-path coverage.

## Reproduction Assessment

Yes, from source inspection. Record same-run completed outcomes for distinct tools with `error:` result hashes, then call `detectToolCallLoop` for another distinct tool; current main has no detector that blocks that cross-tool error tail.
