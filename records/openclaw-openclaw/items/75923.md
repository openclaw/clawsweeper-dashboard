## Summary

Keep open. Current main still lacks the requested run-scoped consecutive cross-tool error detector and config/docs surface, and the issue is paired with open closing PR #75924, so cleanup should wait for that PR to merge or be closed.

## What This Changes

Add a run-scoped loop detector that blocks a run after a configurable streak of completed tool outcomes whose result hashes encode errors across different tools.

## Best Possible Solution

Review and land or close #75924; the accepted implementation should extend the existing run-scoped loop detector with focused detector logic, schema/help/docs updates, and same-run plus cross-run isolation coverage.

## Reproduction Assessment

Yes, from source inspection. With loop detection enabled, same-run completed failures for distinct tools get `error:` result hashes, but current detection only evaluates existing repeat, no-progress, polling, ping-pong, and unknown-tool patterns.

## Solution Assessment

Yes at the issue direction level, but the concrete fix should be reviewed in #75924. Extending the existing run-scoped detector is narrower than a turn cap or session retry budget, while still needing maintainer judgment on threshold and docs shape.
