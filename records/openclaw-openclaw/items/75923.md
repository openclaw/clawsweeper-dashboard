## Summary

Keep open. Current main still has only the existing pattern-specific loop detectors and no consecutive-error threshold, and this issue is already paired with open closing PR #75924.

## What This Changes

Add a run-scoped `consecutive_errors` loop detector and `tools.loopDetection.consecutiveErrorThreshold` config to block cross-tool streaks of failed tool outcomes.

## Best Possible Solution

Review and land or close #75924; the accepted implementation should add the run-scoped detector, schema/help/docs updates, and focused unit plus hook-path coverage.

## Reproduction Assessment

Yes, from source inspection. Record same-run completed outcomes for distinct tools with `error:` result hashes, then ask `detectToolCallLoop` about another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue direction level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while #75924 is the right place to review defaults, schema shape, docs wording, and edge cases.
