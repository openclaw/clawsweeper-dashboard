## Summary

Keep open. Current main still lacks the requested run-scoped consecutive-error detector, config field, and docs, and the issue is already paired with open closing PR #75924.

## What This Changes

Add a run-scoped loop detector and threshold that block the next tool call after consecutive completed error outcomes across different tools in the same run.

## Best Possible Solution

Review and land or close #75924; the accepted implementation should extend the existing run-scoped loop detector with schema, docs, and focused detector plus hook-path coverage.

## Reproduction Assessment

Yes, from source inspection. Record same-run completed outcomes for distinct tools with error: result hashes, then ask detectToolCallLoop about another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while #75924 should carry the concrete implementation review for config shape, defaults, docs, and edge cases.
