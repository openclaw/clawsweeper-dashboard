## Summary

Keep open. Current main still lacks the requested run-scoped cross-tool consecutive-error detector and config, and this issue is paired with open closing PR #75924, so cleanup should wait for that PR to be reviewed and merged or closed.

## What This Changes

Request adding a run-scoped `consecutive_errors` loop detector and `tools.loopDetection.consecutiveErrorThreshold` config to block cross-tool failure cascades.

## Best Possible Solution

Review and land or close #75924; the accepted fix should extend the existing run-scoped loop detector and before-tool-call hook with schema, docs, changelog, and focused unit plus hook-path coverage.

## Reproduction Assessment

Yes. Record same-run completed outcomes for distinct tools with `error:` result hashes, then ask `detectToolCallLoop` about another distinct tool; current main has no detector that blocks that cross-tool error tail.

## Solution Assessment

Yes at the issue level. Extending the existing run-scoped loop detector is the narrow maintainable boundary, while the open PR still needs review for exact defaults and edge cases.
