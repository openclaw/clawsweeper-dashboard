## Summary

Keep open: current main has run-scoped tool-loop detection, but it does not implement the requested cross-tool consecutive-error detector or `consecutiveErrorThreshold`, and linked PR #75924 is already the open implementation vehicle.

## What This Changes

Requests a run-scoped loop detector that counts consecutive failed tool outcomes across different tools and blocks once a configurable threshold is reached.

## Best Possible Solution

Review and either land or revise PR #75924 with the detector, config/schema/docs, and regression coverage; close this issue only after that implementation merges or a narrower replacement lands.

## Reproduction Assessment

Yes, by source inspection: current main records error result hashes but only detects same-tool/args, unknown-tool, polling, ping-pong, and global no-progress patterns. A run tail of different failed tools would not hit those branches without the proposed new detector.

## Solution Assessment

Unclear. A run-scoped detector is a narrower direction than turn caps or broad retry budgets, but the exact threshold/config shape and false-positive policy should be settled through the linked PR review.
