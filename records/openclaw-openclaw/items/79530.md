## Summary

Keep open: current main still only partially satisfies this enhancement because Control UI stores the assistant model in collapsed Context metadata, while TUI shows only the session-level model and has no per-message metadata slot.

## What This Changes

Show a visible model-name badge or tag on each assistant response in Control UI WebChat and TUI for multi-model sessions.

## Best Possible Solution

Define a compact visible message-metadata affordance that reuses recorded assistant model data in Control UI and TUI, with graceful fallback when a message lacks model metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main verification path: Control UI model metadata is collapsed by default, and TUI assistant messages are text-only while the footer shows the session model.

## Solution Assessment

Unclear until maintainers choose the intended UX. A badge beside the assistant name is plausible, but the maintainable path should reuse the existing recorded assistant model metadata rather than adding a separate model lookup path.
