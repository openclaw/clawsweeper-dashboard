## Summary

Keep open: current main still only partially satisfies this enhancement. Control UI records assistant model metadata but hides it behind a closed Context disclosure, and TUI still shows model at the session/footer level rather than per assistant message.

## What This Changes

The issue requests visible per-message model attribution in Control UI WebChat and TUI for multi-model sessions.

## Best Possible Solution

Define a compact visible assistant-message model affordance that reuses existing Control UI metadata and adds a TUI message metadata slot with graceful fallback when model data is missing.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI keeps the model chip inside closed Context metadata, while TUI assistant messages accept only text and the footer model is session-level.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the maintainable path should reuse existing metadata extraction and add only the minimum TUI message metadata surface needed.
