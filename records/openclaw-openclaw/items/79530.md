## Summary

Keep open: the prior ClawSweeper review still matches current main. Control UI records assistant model metadata but keeps it behind collapsed Context details, while the TUI shows only the current session model in the footer and has no per-message metadata slot.

## What This Changes

Request to show a visible model-name badge or tag on each assistant message in Control UI WebChat and the TUI.

## Best Possible Solution

Define a small cross-surface message metadata display policy, then render the recorded assistant model in that design for Control UI and TUI with a graceful fallback when model data is absent.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI metadata is collapsed by default, and TUI assistant messages carry only rendered text.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but current Context metadata means the narrowest Control UI change may be a display/discoverability adjustment while TUI needs a new metadata slot.
