## Summary

Keep open: current main still hides WebChat model metadata inside collapsed Context details, and the TUI assistant-message path renders text only, so the requested visible per-message model attribution is not implemented.

## What This Changes

Request to show the model used for each assistant response as a visible badge or tag in Control UI WebChat and TUI multi-model workflows.

## Best Possible Solution

Define a compact cross-surface message metadata design, then render recorded assistant model attribution visibly in WebChat and TUI with graceful fallback when model data is absent.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection is enough to verify WebChat model metadata is collapsed and TUI assistant messages render without a metadata slot.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the intended UX; a visible sender-adjacent badge is plausible, but current collapsed metadata may make this a display-policy change rather than new data plumbing.
