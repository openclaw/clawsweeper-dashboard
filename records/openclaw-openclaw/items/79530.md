## Summary

Keep open: current main only partially covers the request. Control UI has assistant model metadata, but the model chip is inside a collapsed Context disclosure, and the TUI still exposes only the current session model in the footer rather than per assistant message.

## What This Changes

Add visible per-assistant-message model attribution in Control UI WebChat and TUI for model-switching and spawned-session workflows.

## Best Possible Solution

Define a compact message metadata design, then render recorded assistant model data visibly in Control UI and TUI with graceful fallback when a message lacks model metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI model metadata is collapsed behind Context, and TUI assistant message components do not carry per-message metadata.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible assistant-name badge is plausible, but reusing the existing metadata presenter with a clearer default-visible model affordance may be the narrower maintainable path.
