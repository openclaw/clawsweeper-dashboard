## Summary

Keep open: current main still only partially satisfies this enhancement. Control UI stores and renders assistant model metadata, but it is hidden behind a collapsed Context disclosure, and the TUI shows only the current session model in the footer rather than per assistant message.

## What This Changes

The issue requests visible per-assistant-message model attribution in Control UI WebChat and TUI for multi-model workflows.

## Best Possible Solution

Define a compact message metadata design, then render recorded assistant model data visibly in Control UI and TUI with graceful fallback when the message lacks model metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has model metadata only in collapsed Context details, and TUI assistant messages render text without per-message metadata.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the narrowest maintainable path may be to reuse the existing Context metadata presenter with a clearer default-visible model affordance.
