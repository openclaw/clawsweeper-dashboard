## Summary

Keep open: current main only exposes the assistant model in the Control UI inside collapsed Context details, while TUI assistant messages still render as text-only entries with no per-message model attribution. This is a real enhancement request and needs a small UX/product decision before implementation.

## What This Changes

Add visible model-name attribution for each assistant message/task in Control UI WebChat and the TUI.

## Best Possible Solution

Define a compact cross-surface message metadata design, then render the recorded assistant model in Control UI and TUI when available, with a clean fallback when older messages lack model data.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence verification that Control UI model metadata is hidden in a collapsed Context disclosure and TUI assistant messages render without per-message model attribution.

## Solution Assessment

Unclear. A visible badge is plausible, but maintainers should first decide whether this is a default-visible Control UI metadata change, a separate badge, and how dense the TUI representation should be.
