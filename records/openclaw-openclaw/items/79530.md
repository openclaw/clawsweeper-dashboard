## Summary

Keep open: current main still only exposes the WebChat model value inside collapsed Context details, and the TUI renders assistant replies through text-only message components while showing the current model in the footer. This is a feature/UX request, not an implemented bug fix or duplicate cleanup candidate.

## What This Changes

Request to show a visible model-name badge or tag on each assistant message in Control UI WebChat and TUI for multi-model sessions.

## Best Possible Solution

Define a small shared message-metadata presentation and render the recorded assistant model visibly where available, with a clean fallback when model data is absent.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high-confidence current-main verification that WebChat model data is collapsed under Context and TUI assistant messages have no per-message model slot.

## Solution Assessment

Unclear until maintainers choose the UX. A badge next to the assistant name is plausible, but the narrow maintainable path should account for the existing WebChat Context affordance and TUI footer density.
