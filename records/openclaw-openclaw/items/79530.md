## Summary

Keep open: current main only partially satisfies this enhancement. Control UI records assistant-message model metadata, but it is hidden inside a closed Context disclosure, and TUI still exposes model state at the session/footer level rather than per assistant message.

## What This Changes

Request to show a visible model-name badge or tag on each assistant message/task in Control UI WebChat and TUI for multi-model workflows.

## Best Possible Solution

Define a compact visible assistant-message model affordance that reuses existing Control UI message.model metadata and adds a small TUI message metadata slot with graceful fallback when model data is unavailable.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: WebChat keeps the model chip inside closed Context metadata, while TUI assistant messages carry only text and the model appears in session footer state.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the maintainable path should reuse existing metadata extraction and add only the minimum TUI message metadata surface needed.
