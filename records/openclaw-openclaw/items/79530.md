## Summary

Keep open: current main and the latest release still hide Control UI model metadata behind a collapsed Context disclosure, while TUI assistant messages have no per-message metadata slot; this is a feature/UX request rather than a closable implemented bug fix.

## What This Changes

Request to show the model name as a visible per-assistant-message badge or tag in Control UI WebChat and TUI for multi-model workflows.

## Best Possible Solution

Define a compact message-metadata design, then render recorded assistant model/provider data on assistant messages in Control UI and TUI with graceful fallback when the transcript lacks model fields.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection is enough to verify Control UI hides model metadata behind collapsed Context details and TUI assistant messages render only text.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the UX; a visible badge next to the assistant name is plausible, but the narrowest maintainable fix may be a display/default-visibility change plus a small TUI metadata presenter rather than new data plumbing.
