## Summary

Keep open: current main still hides the Control UI model chip inside collapsed Context details and the TUI only exposes the active session model in the footer, so the requested visible per-message attribution remains an enhancement needing UI/product direction.

## What This Changes

Request to show the model that produced each assistant response as visible per-message metadata in Control UI WebChat and TUI.

## Best Possible Solution

Define a compact cross-surface assistant-message metadata design, then render recorded assistant model data in Control UI and TUI with graceful fallback when model metadata is absent.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI model metadata is currently hidden behind collapsed Context details and TUI assistant messages render text without per-message model attribution.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the current collapsed Context affordance means the narrowest maintainable solution depends on the desired default visibility and TUI density.
