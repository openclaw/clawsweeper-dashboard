## Summary

Keep open: current main still only exposes the Control UI model chip inside collapsed Context details, and the TUI assistant message path has no per-message model metadata slot. This is a plausible UI feature, but it needs maintainer UX judgment before implementation.

## What This Changes

Add visible per-message model attribution to assistant responses in Control UI WebChat and the TUI.

## Best Possible Solution

Define a compact cross-surface message metadata presentation that shows the recorded assistant model when present, while preserving graceful fallback for messages without model data.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose badge placement, default visibility, and TUI density; the existing metadata plumbing suggests this is mainly a display/UX decision rather than new model collection.
