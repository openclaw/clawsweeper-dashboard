## Summary

Keep open: current main and v2026.6.8 only partially satisfy this request. WebChat has per-message model metadata, but it is hidden behind a closed Context disclosure, and TUI assistant messages still render only text while the model is shown at session level in the footer.

## What This Changes

Request to add visible per-assistant-message model attribution in Control UI WebChat and TUI for multi-model sessions and spawned tasks.

## Best Possible Solution

Adopt a maintainer-approved compact visible model indicator in WebChat and a TUI message metadata slot, both sourced from per-message metadata with graceful fallback when unavailable.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes at source level: WebChat hides model metadata inside a closed disclosure and TUI assistant messages have no per-message metadata slot.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the intended UX; a visible badge is plausible, but the implementation should reuse existing WebChat metadata extraction and add explicit TUI per-message metadata instead of deriving from current session state.
