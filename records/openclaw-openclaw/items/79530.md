## Summary

Keep open: current main and v2026.6.6 still only partially satisfy the request. WebChat has per-message model metadata, but it is hidden behind a closed Context disclosure, and TUI assistant messages still render text without a per-message model slot.

## What This Changes

The issue requests per-message model attribution badges or metadata in Control UI WebChat and TUI for multi-model workflows.

## Best Possible Solution

Define a compact visible assistant-message model affordance that reuses existing Control UI metadata and adds a TUI message metadata slot with graceful fallback when model data is missing.

## Reproduction Assessment

Not applicable as a bug reproduction; source inspection gives a high-confidence verification path for the feature gap. WebChat keeps model data behind a closed Context disclosure, while TUI assistant messages carry only text and the footer model is session-level.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the maintainable implementation should reuse existing metadata extraction and avoid deriving per-message attribution from current sessionInfo.
