## Summary

Keep open: current main still only partially satisfies the request. WebChat now carries per-message model metadata, but the visible model indicator is inside a closed Context disclosure, and the TUI assistant transcript still has no per-message model slot.

## What This Changes

Show a per-assistant-message model badge/tag in Control UI WebChat and TUI so multi-model sessions can be reviewed without checking session status.

## Best Possible Solution

Define a compact visible assistant-message model affordance that reuses existing message metadata in WebChat and adds a TUI metadata slot with graceful fallback.

## Reproduction Assessment

Yes at source level: WebChat renders the model under a closed Context disclosure, while TUI assistant rows render text only and have no model metadata slot. I did not run a browser/TUI session because this read-only cleanup review relied on current-main source and tests.

## Solution Assessment

Unclear until maintainers choose the UX placement. The maintainable direction is to reuse existing per-message assistant metadata for WebChat and propagate a compact metadata field through the TUI transcript path rather than adding a second source of truth.
