## Summary

Keep open: current main still does not provide the requested always-visible per-message model attribution. Control UI stores and renders model metadata only inside a collapsed Context disclosure, while the TUI exposes the current session model in the footer but not on each assistant message.

## What This Changes

The issue requests visible per-assistant-message model attribution in Control UI WebChat and TUI for multi-model sessions.

## Best Possible Solution

Define a compact message metadata design, then render recorded assistant model data visibly in Control UI and TUI with graceful fallback when a message lacks model metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has model metadata only inside a collapsed Context disclosure, and TUI assistant messages render text without per-message metadata.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the narrowest maintainable path may be to reuse the existing metadata presenter with a default-visible model affordance and add a compact TUI metadata slot.
