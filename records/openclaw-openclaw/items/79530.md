## Summary

Keep open: current main and v2026.6.8 still only partially satisfy the request. WebChat records per-message model metadata, but the model chip is hidden behind a closed Context disclosure, and TUI assistant transcript rows still render text without a per-message model slot.

## What This Changes

Request to add an immediately visible model-name badge or metadata line to each assistant message/task in Control UI WebChat and TUI for multi-model sessions.

## Best Possible Solution

Define a compact visible assistant-message model affordance that reuses existing per-message metadata in WebChat and adds a TUI message metadata slot with graceful fallback when a message lacks model data.

## Reproduction Assessment

Yes at source level: current main carries per-message model metadata in WebChat but hides it behind a closed disclosure, and TUI assistant rows are text-only while the model appears only in the session footer.

## Solution Assessment

Unclear until maintainers choose the intended UX. A visible badge next to the assistant name is plausible, but the maintainable implementation should reuse existing message metadata and avoid deriving per-message attribution from current sessionInfo.
