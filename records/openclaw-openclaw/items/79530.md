## Summary

Keep open: current main only exposes WebChat model metadata inside collapsed Context details, and the TUI assistant path still renders assistant text without per-message model attribution. This is a valid feature request that needs a small UX/product decision before implementation.

## What This Changes

Request to show each assistant response's model name as a visible badge or tag in Control UI WebChat and TUI for multi-model workflows.

## Best Possible Solution

Define a compact cross-surface message metadata presentation, then render the recorded assistant model in WebChat and TUI when available with graceful fallback.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection gives a high-confidence verification path: WebChat model metadata is collapsed by default and TUI assistant messages currently carry only text.

## Solution Assessment

Unclear until maintainers choose the intended UX. A badge next to the assistant name is plausible, but the existing collapsed Context chip means the narrowest maintainable fix could be a visibility/discoverability adjustment plus a TUI metadata slot.
