## Summary

Keep open: current main still does not auto-install or auto-enable hook sidecars from ClawHub skill archives. The existing `before_install` policy covers pre-copy scanning, but the requested executable-hook sidecar behavior still needs maintainer product and security direction for trust, consent, naming, conflict handling, update, and rollback semantics.

## What This Changes

The issue requests ClawHub skill install/update to detect root hook directories, copy them into managed hooks, enable them, and report the hook changes.

## Best Possible Solution

Define a maintainer-approved, security-gated hook sidecar contract for ClawHub skills, or keep runnable hooks on the existing explicit hook-pack/plugin install path if that trust model is not accepted.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main installs ClawHub skills and writes metadata without hook sidecar installation or enablement.

## Solution Assessment

Unclear as proposed: the UX goal is valid, but automatic install-and-enable is not the safest default until maintainers define trust, naming, conflict, update, consent, and rollback rules. A security-gated sidecar contract or the existing explicit hook-pack route is the safer end state.
