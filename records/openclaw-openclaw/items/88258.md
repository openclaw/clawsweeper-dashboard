## Summary

Keep open: current main still does not auto-install or auto-enable hook sidecars from ClawHub skill archives, and the requested behavior would cross an executable-hook trust boundary that needs maintainer product and security direction before implementation.

## What This Changes

The issue requests that ClawHub skill install/update detect bundled `hooks/` directories, copy them into managed hooks, enable them, and report the installed hooks.

## Best Possible Solution

Define a maintainer-approved, security-gated skill hook sidecar contract, or keep hook packs on the existing explicit plugin install path if that trust model is not accepted.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main installs ClawHub skills and writes metadata without hook sidecar installation or enablement.

## Solution Assessment

Unclear as proposed: the UX goal is valid, but automatic install-and-enable is not the safest default until maintainers define trust, naming, conflict, update, and rollback rules. The safer path is an explicit security-gated sidecar contract or the existing hook-pack route.
