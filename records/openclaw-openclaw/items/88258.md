## Summary

Keep open: current main still does not install or enable skill-shipped hook sidecars during ClawHub skill install/update. The existing pre-copy install policy is only a scanner/policy gate, while automatic hook installation and config enablement need maintainer product and security direction.

## What This Changes

Request to make ClawHub skill install/update detect bundled `hooks/` directories, copy them into managed hooks, enable them, and report the result.

## Best Possible Solution

Define a maintainer-approved, security-gated sidecar contract for ClawHub skills, or keep runnable hooks on the existing explicit hook-pack/plugin install path if automatic sidecars are not accepted.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main scans and copies ClawHub skills but does not install or enable hook sidecars.

## Solution Assessment

Unclear as proposed. The UX goal is valid, but automatic install-and-enable is not the safest default until maintainers define trust, consent, naming, conflict, update, and rollback rules.
