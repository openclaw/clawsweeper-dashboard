## Summary

Keep open: current main still installs ClawHub skills as skill roots and records ClawHub metadata, while hook installation and enablement remain explicit hook/plugin flows. The requested automatic hook sidecar install crosses an executable-code and config-mutation trust boundary, so maintainers need product/security direction before implementation.

## What This Changes

Request auto-installing and enabling hook directories shipped inside ClawHub skill archives during skill install/update.

## Best Possible Solution

Define an explicit, operator-confirmed skill hook sidecar contract or keep executable hooks on the existing plugin/hook-pack install path; then add install/update behavior, docs, and tests after the trust model is approved.

## Reproduction Assessment

Yes, at source level for current behavior: ClawHub skill install delegates to installExtractedSkillRoot, which copies the skill root and writes ClawHub metadata without calling hook install or enable logic. As a feature request, there is no failing runtime bug to reproduce.

## Solution Assessment

Unclear as proposed: the UX goal is valid, but automatic install-and-enable is not the safest default without explicit trust prompts, conflict ownership, update/uninstall semantics, and ClawHub metadata support.
