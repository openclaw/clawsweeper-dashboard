## Summary

Keep open: current main still installs ClawHub skill archives as skills only and does not install or enable hook sidecars; the remaining work is a security-sensitive product decision, not conservative cleanup.

## What This Changes

The issue requests automatic detection, installation, enablement, update handling, and CLI reporting for hook directories shipped inside ClawHub skill archives.

## Best Possible Solution

Define an explicit, security-gated skill hook sidecar contract, or keep hook packs on the existing plugin install path if maintainers do not want skill archives to install executable hooks.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes at source level: current main installs ClawHub skill files and writes metadata without installing or enabling hook sidecars. The relevant installer, gateway, UI, and hook-enable paths are clear from source inspection.

## Solution Assessment

Unclear as proposed: the UX goal is valid, but automatic install-and-enable is not the safest default until maintainers define trust, consent, naming, conflicts, update, and rollback semantics. A security-gated sidecar contract or the existing hook-pack install path is safer than unconditional enablement.
