## Summary

Keep open: current main does not install or enable hooks from ClawHub skill archives, but this is a feature request that crosses a security-sensitive install boundary and needs maintainer product/security direction before implementation.

## What This Changes

Request auto-installing and enabling hook directories shipped inside ClawHub-installed skills during skill install and update.

## Best Possible Solution

Define an explicit hook sidecar contract for skills only if maintainers accept the trust model; otherwise route executable hook behavior through plugin or hook-pack packaging with clear prompts, conflict handling, update/uninstall behavior, tests, and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current behavior: ClawHub skill install copies the skill root and records ClawHub metadata without hook installation or enablement.

## Solution Assessment

Unclear as proposed. The UX goal is reasonable, but automatic install-and-enable is not the safest default without an explicit trust prompt, conflict policy, and a maintained packaging contract.
