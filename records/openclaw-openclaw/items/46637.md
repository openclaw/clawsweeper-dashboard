## Summary

Keep open. Current main still captures OpenAI-compatible `reasoning_content` as replayable thinking metadata, but the replay policy only strips historical reasoning for Gemma 4, so the Qwen/oMLX two-turn failure remains a valid source-level regression with clear repair boundaries.

## What This Changes

The issue asks OpenClaw to prevent Qwen/oMLX `reasoning_content` from being replayed into later `openai-completions` requests where it causes strict JSON/backend rejection.

## Best Possible Solution

Add targeted replay-history hygiene for affected Qwen/oMLX-style strict OpenAI-compatible backends so historical provider reasoning blocks are removed or downgraded before follow-up requests, while preserving same-turn tool-call continuations and provider-owned preserved-reasoning contracts.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a second `openai-completions` turn for a Qwen/oMLX custom provider after the prior assistant turn contains a thinking block signed `reasoning_content`; current main captures that field and does not enable `dropReasoningFromHistory` for that model/provider path.

## Solution Assessment

Yes. The best repair direction is a replay-policy fix, not a generic JSON escaping patch: strip or downgrade historical Qwen/oMLX-style provider reasoning before replay while leaving intentional preserved-reasoning providers alone.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/46637 on current main. The observable bug is that Qwen/oMLX-style `openai-completions` models can return `reasoning_content`, OpenClaw stores it as assistant thinking metadata, and later turns replay it unless `dropReasoningFromHistory` is enabled; current policy only enables that for Gemma 4. Add a targeted replay-history policy so affected Qwen/oMLX-style strict OpenAI-compatible history drops or downgrades historical provider reasoning before follow-up requests. Preserve the existing same-turn tool-call continuation exception in `dropReasoningFromHistory`. Do not add a user config option, do not change JSON serialization globally, and do not break providers that intentionally preserve/backfill `reasoning_content` such as DeepSeek V4, Z.AI preserved thinking, Venice, or OpenRouter DeepSeek. Add/update focused tests for transcript policy, shared provider replay helpers, sanitize-session-history replay behavior, and any provider-family SDK helper touched. Update `docs/reference/transcript-hygiene.md` and add a user-facing changelog fix entry if behavior changes.
