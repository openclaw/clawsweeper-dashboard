## What This Changes

This PR adds gzip/Brotli Accept-Encoding negotiation for Gateway-served Control UI static assets and tests gzip, q-value, wildcard, HEAD, and binary passthrough behavior.

## Best Possible Solution

Serve negotiated gzip/Brotli variants from the Control UI handler using cached, async, streamed, or precomputed compression while preserving routing, CSP, file-safety, HEAD behavior, q-values, and binary passthrough.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 do not inspect Accept-Encoding or emit Content-Encoding for Control UI static assets, and the linked issue provides a concrete curl check for the missing header.

## Solution Assessment

No. The Control UI handler is the right owner boundary, but synchronous zlib on the request path is not the best implementation shape for megabyte-scale bundles; cached, async, streamed, or precomputed compression is safer.
