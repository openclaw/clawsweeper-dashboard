## What This Changes

The branch adds gzip/Brotli Accept-Encoding negotiation for Control UI static files and tests q-values, HEAD headers, wildcard fallback, and binary passthrough.

## Best Possible Solution

Serve negotiated gzip/Brotli from the Control UI handler using cached or async/streamed compression while preserving routing, CSP, file-safety, HEAD behavior, q-values, and binary passthrough.

## Reproduction Assessment

Yes, at source level. Current main's static serving helpers send raw file bodies without negotiated Content-Encoding, and the linked issue provides a curl path that observes the missing compression.

## Solution Assessment

No. The Control UI handler is the right boundary, but the current implementation uses sync zlib on the request path; the safer implementation is cached or async/streamed compression with the same negotiation behavior.
