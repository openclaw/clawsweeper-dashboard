## What This Changes

The PR adds gzip and Brotli `Accept-Encoding` negotiation for Control UI static assets plus regression tests for GET/HEAD compression behavior, q-values, wildcard fallback, and binary passthrough.

## Best Possible Solution

Land the narrow Control UI static-asset compression path after normal maintainer merge gates, preserving the existing file-safety and routing boundaries in `handleControlUiHttpRequest`.

## Reproduction Assessment

Yes, by source inspection current main's static serving helpers send raw bodies without negotiated compression, matching the linked user report at https://github.com/openclaw/openclaw/issues/81754. I did not run live curl in this read-only review; the PR supplies live after-fix curl output.

## Solution Assessment

Yes; the Control UI static-file handler is the narrow owner boundary because `src/gateway/server-http.ts` delegates Control UI HTTP requests to `handleControlUiHttpRequest`. A broader gateway-wide wrapper would risk changing unrelated HTTP responses.
