## Summary

Keep this PR open, but it is not merge-ready: the compression behavior is useful and well-proven, while the current patch still runs synchronous gzip/Brotli work on the shared Gateway request path for large Control UI assets.

## What This Changes

The PR adds gzip/Brotli `Accept-Encoding` negotiation for Control UI static files and tests gzip, Brotli, q-values, wildcard fallback, HEAD headers, and binary passthrough.

## Best Possible Solution

Serve negotiated gzip/Brotli from the Control UI handler using cached, async, or streamed compression while preserving routing, CSP, file-safety, HEAD behavior, q-values, and binary passthrough.

## Reproduction Assessment

Yes, at source level. Current main's static serving helpers send raw file bodies without negotiated `Content-Encoding`, and the linked issue provides a concrete curl path that observes the missing compression.

## Solution Assessment

No. The Control UI handler is the right boundary, but the current implementation uses sync zlib on the request path; the safer implementation is cached or async/streamed compression with the same negotiation behavior.
