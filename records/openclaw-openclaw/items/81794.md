## Summary

Keep this PR open: it is the active candidate for the linked Control UI asset-compression request, but the current implementation still has a blocking availability concern because it compresses large static responses with synchronous zlib on the shared Gateway HTTP request path.

## What This Changes

The PR adds gzip and Brotli negotiation for Control UI static asset responses and regression tests for compressed text assets, q-values, HEAD responses, and binary passthrough.

## Best Possible Solution

Serve negotiated gzip/Brotli from the Control UI handler using cached, async, or streamed compression while preserving routing, CSP, file-safety, HEAD behavior, q-values, and binary passthrough.

## Reproduction Assessment

Yes, at source level. Current main's Control UI static response helpers do not inspect Accept-Encoding or emit Content-Encoding, and the linked issue provides a concrete curl check for the missing header.

## Solution Assessment

No. The Control UI handler is the right owner boundary, but the current implementation uses sync zlib on the request path; cached, async, or streamed compression is the safer maintainable shape.
