## Summary

Keep open: current main still serves Control UI static files raw, while this PR adds a focused compression path with sufficient live gateway proof and no blocking review finding from this pass.

## What This Changes

The branch adds q-aware gzip/Brotli `Accept-Encoding` negotiation to Control UI static file serving and updates gateway HTTP tests.

## Best Possible Solution

Review and land this PR if maintainers are comfortable with synchronous per-request compression, then let it close https://github.com/openclaw/openclaw/issues/81754.

## Reproduction Assessment

Yes, for source-level reproduction: on current main, a `curl -H 'Accept-Encoding: gzip'` request to a Control UI JS/CSS/HTML asset follows helpers that never inspect `Accept-Encoding` or set `Content-Encoding`. I did not run a live gateway locally in this read-only pass.

## Solution Assessment

Yes; the PR changes the narrow gateway static-file serving surface, honors q-values and wildcard rules, and adds focused regression coverage. A future optimization could cache precompressed bytes, but that is not required for the reported behavior.
