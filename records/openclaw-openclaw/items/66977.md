## What This Changes

The issue reports macOS sqlite-vec failing to load through OpenClaw memory search, degrading vector recall.

## Best Possible Solution

Keep the shipped node:sqlite `allowExtension` implementation and leave adjacent SQLite capability work, such as FTS5 diagnostics, on their separate trackers.

Yes. The source-level reproduction is a Node SQLite database opened without `allowExtension` before sqlite-vec loading; the issue discussion also includes macOS targeted proof showing the same path succeeds once `{ allowExtension: true }` is used.

## Solution Assessment

Yes. The current constructor-permission path is the narrow dependency-backed solution and avoids the larger better-sqlite3 native dependency swap proposed in the now-closed PR.
