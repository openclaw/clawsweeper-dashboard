  - confidence: 0.9

  - body: The new schema validates periodicExtraction, but the runtime config types and memory-flush planner do not carry or read it, so users can configure a setting that has no effect.
  - confidence: 0.86

  - body: Adding public config fields only to the Zod schemas leaves generated schema/help/label artifacts stale, so schema-driven tooling would not expose the new settings.
