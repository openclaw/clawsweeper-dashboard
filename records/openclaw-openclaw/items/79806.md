## Summary

Keep open: the report is a current, well-evidenced performance bug, and current main still has a source-visible idle health/read-only channel path that can fall back into full plugin metadata and installed-manifest registry reconstruction under the reported compat configuration.

## What This Changes

The issue asks OpenClaw to stop repeatedly scanning bundled plugin manifests at idle when `plugins.bundledDiscovery` is `"compat"` with a restrictive `plugins.allow`.

## Best Possible Solution

Preserve `compat` semantics but make idle gateway health/read-only paths reuse a compatible prepared plugin metadata snapshot or scoped manifest registry so disabled bundled plugins are not rescanned repeatedly.

## Reproduction Assessment

Yes, source-backed rather than locally executed: the reporter supplied exact v2026.5.7 profiling steps and a before/after `compat` versus `allowlist` comparison, and current main still contains the periodic health-to-metadata fallback path that can rebuild the full installed manifest registry.

## Solution Assessment

Unclear. The issue's cache/default-change suggestions are plausible directions, but the safer maintainable fix is to reuse prepared metadata or scope registry reconstruction without adding a broad persistent cache or changing legacy compat semantics.
