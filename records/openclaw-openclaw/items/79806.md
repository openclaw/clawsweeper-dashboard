## Summary

Keep this issue open: current main still has the source-level hot path, and https://github.com/openclaw/openclaw/pull/80207 is an open closing PR with focused after-fix proof. Closing the issue before that PR, or an equivalent fix, merges would be premature.

## What This Changes

The issue reports idle gateway CPU saturation because compat bundled plugin discovery repeatedly re-stats bundled plugin manifests despite no active runs.

## Best Possible Solution

Carry the installed plugin inventory fingerprint on PluginMetadataSnapshot and reuse it in snapshot compatibility checks, with regression coverage preventing repeated manifest/package stat calls.

## Reproduction Assessment

Yes, source-reproducible: current main passes snapshot.index into current-snapshot fingerprint checks, which falls through to resolveInstalledManifestRegistryIndexFingerprint and fs.statSync on manifest/package files. I did not run the live gateway profile in this read-only sweep.

## Solution Assessment

Yes: the linked PR's snapshot-owned inventoryFingerprint handoff is the narrowest maintainable fix path I found, and it preserves explicit snapshot lifecycle invalidation instead of adding a broad persistent discovery cache.
