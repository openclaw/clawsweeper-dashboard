## Summary

Keep open. The report is fresh and well instrumented; current main still contains the compat discovery and metadata-scan paths, while the closest performance fixes are open PRs rather than merged code.

## What This Changes

The issue asks OpenClaw to stop pegging a gateway CPU core at idle when legacy `plugins.bundledDiscovery: "compat"` causes repeated bundled plugin manifest discovery.

## Best Possible Solution

Land a targeted plugin-discovery/runtime-metadata fix that reuses a compatible startup snapshot or otherwise invalidates explicitly, then prove idle gateway CPU with `compat` and a restrictive allowlist no longer repeatedly scans bundled manifests.

## Reproduction Assessment

Yes, with caveat: the issue provides concrete repro steps and before/after CPU profiles, and source inspection shows the same compat and manifest-registry scan paths still exist on current main. I did not live-run the gateway profile in this read-only checkout.

## Solution Assessment

Unclear. Caching or reusing the existing current plugin metadata snapshot looks like the narrowest maintainable direction; changing the compat default again would be a broader migration/product choice and should not be the first fix.
