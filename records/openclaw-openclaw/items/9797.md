## Best Possible Solution

Build a narrow, canonical dispatch-capacity view from the existing lane snapshot and subagent registry state, then expose it through the agent tool catalog with requester-scoped visibility, explicit policy registration, documentation, and focused tests.

## Solution Assessment

No, the broad raw example should not be exposed unchanged; the best path is a requester-scoped capacity contract backed by the existing canonical queue snapshot rather than a second queue model.
