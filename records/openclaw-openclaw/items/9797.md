The issue requests an agent-facing `queue_status` tool so dispatcher agents can inspect queue depth, lane concurrency, and per-agent workload before using `sessions_spawn`.

## Best Possible Solution

Define a shared read-only queue snapshot backend and expose a policy-gated `queue_status` tool through the agent tool catalog/profile surface, with docs and focused tests.
