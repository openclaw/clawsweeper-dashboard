## Summary

Keep open. Current main still has internal command-lane snapshots and diagnostic logging, but no stable agent-facing `queue_status` tool or equivalent dispatcher-oriented queue snapshot contract; the related https://github.com/openclaw/openclaw/pull/43618 remains open and explicitly excludes the in-agent tool.

## What This Changes

Add an agent-facing `queue_status` tool that exposes lane and per-agent workload data so dispatcher agents can route work intelligently.

## Best Possible Solution

Implement one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with permissions, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection against current main gives high-confidence evidence that the requested stable agent-facing queue-status surface is not present.

## Solution Assessment

Yes at the design level: the maintainable path is a shared read-only queue snapshot provider plus an agent-facing tool, not exporting raw internal state or expanding `agents_list` by default.
