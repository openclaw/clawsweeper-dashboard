## Summary

Keep this issue open: current main has the underlying lane snapshot data, but no supported agent-facing tool exposes queue capacity or workload for dispatch, and the narrower capacity request was consolidated into this canonical issue.

## What This Changes

Add agent-visible queue capacity and workload information so agents can dispatch tasks without blindly overloading busy lanes or agents.

## Best Possible Solution

Create one canonical read-only queue snapshot provider, then expose the smallest agent-safe capacity fields through the existing tool catalog and policy system without adding dispatch policy or a parallel orchestration framework.

## Reproduction Assessment

Yes at source level: current main has internal lane snapshots, while the supported agent tool catalog has no queue or capacity status surface. This is a missing capability rather than a failing existing contract.

## Solution Assessment

Unclear: a dedicated queue_status tool is plausible, but maintainers should first choose whether the minimal capacity data belongs in session_status, sessions_spawn results, or a new read-only tool and define its visibility boundary.
