## Summary

Keep open. Current main still has internal lane snapshots and diagnostic log/status surfaces, but no agent-facing `queue_status` tool or stable dispatcher-oriented queue snapshot contract; the related https://github.com/openclaw/openclaw/pull/43618 remains open and explicitly does not add the in-agent tool.

## What This Changes

Request an agent-facing `queue_status` tool that exposes lane queue depth, active/max concurrency, and per-agent workload for smarter `sessions_spawn` dispatch.

## Best Possible Solution

Implement one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with permissions, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection give high-confidence evidence that current main lacks the requested stable agent-facing queue-status surface.

## Solution Assessment

Yes, the maintainable direction is a shared read-only queue snapshot provider plus an agent-facing tool, not raw internal state export or expanding `agents_list` by default.
