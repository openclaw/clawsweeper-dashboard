## What This Changes

Add an agent-facing read-only queue-status capability for dispatch decisions based on live lane and per-agent workload.

## Best Possible Solution

Expose a minimal visibility-filtered projection of the existing lane snapshots through the established agent-tool catalog and policy surfaces, adding per-agent facts only where a canonical runtime owner already exists.

## Reproduction Assessment

Yes at source level: current main has internal queue snapshots, while an exhaustive agent-tool search confirms no agent-facing workload or queue-status projection exists. This is a feature gap rather than a failing existing contract.

## Solution Assessment

Partly: reusing a shared queue snapshot is the narrowest maintainable foundation, but the proposed cross-agent output must be reduced to an explicitly permitted visibility model instead of exposing raw internal state.
