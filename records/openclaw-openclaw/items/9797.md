## Summary

Keep open: current main still has no agent-facing queue_status tool or equivalent stable queue snapshot surface for dispatcher agents, and the related diagnostics PR was closed unmerged with this issue kept as the canonical path.

## What This Changes

Request an agent-facing queue_status tool that exposes lane capacity, active runs, queued work, and per-agent workload so agents can dispatch subagent work intelligently.

## Best Possible Solution

Add one shared read-only queue snapshot provider in core, then expose a policy-gated agent-facing queue_status tool with catalog/profile wiring, docs, and focused tests; optional operator/RPC consumers should reuse the same backend.

## Reproduction Assessment

Yes, for the missing feature via source inspection: current main and v2026.6.5 have internal lane snapshots but no agent-facing queue_status tool, diagnostics.queue RPC, or CLI queue snapshot.

## Solution Assessment

Yes in direction, but not yet as an implementation plan: a shared queue snapshot backend with a policy-gated agent tool is the narrow maintainable shape, while metadata visibility needs maintainer decision first.
