## Summary

Keep open. Current main still does not provide the requested agent-facing `queue_status` tool or equivalent dispatcher-oriented queue snapshot contract; the closest related work is an open operator diagnostics PR that explicitly leaves the in-agent tool out.

## What This Changes

The issue requests an agent-facing `queue_status` tool that exposes lane and per-agent workload so dispatcher agents can route tasks to idle agents.

## Best Possible Solution

Define or reuse one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with docs, permissions, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, live issue, and related-PR inspection give high-confidence proof that the stable agent-facing queue-status surface is absent on current main.

## Solution Assessment

Yes at the design level: a shared read-only snapshot provider plus an agent-facing tool is the maintainable direction. It is not ready for an automated fix lane because the public schema, visibility rules, and coordination with the open diagnostics PR need maintainer decisions.
