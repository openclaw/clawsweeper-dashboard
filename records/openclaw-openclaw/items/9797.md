## What This Changes

The issue requests a new agent-facing `queue_status` tool that exposes lane backlog, active concurrency, and per-agent workload state for smarter `sessions_spawn` dispatch.

## Best Possible Solution

Define a shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with visibility filtering, docs, and focused tests; reuse https://github.com/openclaw/openclaw/pull/43618 if it lands first.

## Solution Assessment
