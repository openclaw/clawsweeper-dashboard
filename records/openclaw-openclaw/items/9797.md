## Summary

Keep open. Current main still lacks the requested agent-facing `queue_status` tool or equivalent dispatcher-oriented queue snapshot contract; the only closely related open work is an operator diagnostics PR that explicitly says it does not add an in-agent tool.

## What This Changes

Add an agent-facing `queue_status` tool so dispatcher agents can inspect lane, queue, and per-agent workload before calling `sessions_spawn`.

## Best Possible Solution

Define or reuse one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with permissions, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and related-PR inspection on current main give high-confidence evidence that the requested stable agent-facing queue-status surface is absent.

## Solution Assessment

Yes at the design level: a shared read-only snapshot provider plus an agent-facing tool is the maintainable direction. The exact JSON schema and visibility policy still need maintainer agreement before implementation.
