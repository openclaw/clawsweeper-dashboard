## Summary

Keep this open: current main still lacks the requested agent-facing `queue_status` tool or equivalent stable queue snapshot surface, and the related operator diagnostics work at https://github.com/openclaw/openclaw/pull/43618 is still open and explicitly excludes an in-agent tool.

## What This Changes

Request an agent-facing `queue_status` tool that exposes queue/lane and per-agent workload snapshots so dispatcher agents can choose idle agents before calling `sessions_spawn`.

## Best Possible Solution

Build a shared read-only queue snapshot provider, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy wiring, docs, and focused tests; reuse https://github.com/openclaw/openclaw/pull/43618 if it lands first.

## Solution Assessment

Unclear until maintainers decide the stable API contract and visibility rules. The best-shaped implementation is a shared snapshot backend plus an agent-facing tool, not asking agents to infer workload from `agents_list`, `sessions_list`, or logs.
