Keep this open: current main still lacks the requested agent-facing `queue_status` tool or equivalent stable queue snapshot surface, and the live related operator diagnostics PR is still open and explicitly excludes the in-agent tool.

## What This Changes

## Best Possible Solution

Build a shared read-only queue snapshot provider, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy wiring, docs, and focused tests; reuse https://github.com/openclaw/openclaw/pull/43618 if it lands first.

## Solution Assessment
