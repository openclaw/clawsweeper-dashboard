## Summary

Keep this open. Current main still has only internal queue snapshots and diagnostic logs/status surfaces; it does not provide the requested agent-facing `queue_status` tool or a stable dispatcher-oriented queue snapshot contract.

## What This Changes

Add an agent-facing `queue_status` tool that exposes lane and per-agent workload so dispatcher agents can route tasks intelligently.

## Best Possible Solution

Implement one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with docs and focused tests, reusing #43618 if it lands first.
