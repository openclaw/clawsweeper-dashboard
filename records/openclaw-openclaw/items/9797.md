## Summary

Keep this open. Current main has internal lane snapshot helpers, but it still lacks the requested agent-facing `queue_status` tool and the stable catalog/profile/policy/docs surface needed for dispatcher agents; the related #43618 work is open and explicitly scoped to operator diagnostics, not an in-agent tool.

## What This Changes

Add an agent-facing `queue_status` tool that reports queue/lane and agent workload so dispatcher agents can route tasks intelligently.

## Best Possible Solution

Keep the issue open and implement one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with docs and focused tests, reusing #43618 if it lands first.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection give high-confidence evidence that current main lacks the requested stable agent-facing queue-status surface.

## Solution Assessment

Yes, the maintainable direction is a shared read-only queue snapshot provider plus an agent-facing tool, not a raw one-off export of internal queue state. #43618 is useful related backend work but is not sufficient by itself because it explicitly excludes the in-agent tool.
