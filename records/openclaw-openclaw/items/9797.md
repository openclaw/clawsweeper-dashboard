## Summary

Keep open. Current main still has internal queue snapshots and diagnostic logging, but it does not expose the requested agent-facing `queue_status` tool or an equivalent dispatcher-oriented queue snapshot contract; the related https://github.com/openclaw/openclaw/pull/43618 is still open and explicitly excludes an in-agent tool.

## What This Changes

Add an agent-facing `queue_status` tool so dispatcher agents can inspect lane and per-agent workload before routing work with `sessions_spawn`.

## Best Possible Solution

Implement one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with permissions, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection against current main gives high-confidence evidence that the requested stable agent-facing queue-status surface is not present.

## Solution Assessment

Yes at the design level: a shared read-only queue snapshot provider plus an agent-facing tool is the narrow maintainable path, preferably reusing the backend from https://github.com/openclaw/openclaw/pull/43618 if it lands.
