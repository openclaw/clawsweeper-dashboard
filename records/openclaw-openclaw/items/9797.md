## Summary

Keep open. Current main still lacks an agent-facing `queue_status` tool or equivalent stable queue snapshot surface, and the open related queue diagnostics PR explicitly does not cover the in-agent dispatch tool.

## What This Changes

Add an agent-facing `queue_status` tool that exposes queue/lane and per-agent workload state for intelligent task dispatch.

## Best Possible Solution

Define or reuse a shared read-only queue snapshot contract, then expose a safe subset through `queue_status` with catalog/profile/policy wiring, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The source verification path is high-confidence: current main lacks the requested tool, catalog entry, Gateway queue RPC, CLI queue diagnostics command, and user-facing docs for a stable queue snapshot.

## Solution Assessment

Yes, the requested direction is plausible, but the best implementation should not dump raw lane internals directly to agents. It should coordinate with #43618, settle the snapshot contract and visibility model, then expose the agent tool through the existing tool policy system.
