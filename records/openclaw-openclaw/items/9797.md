## Summary

Current main still lacks an agent-facing queue or capacity preflight surface, so the requested dispatch capability remains necessary. The shared all-lane snapshot primitive now exists, which narrows the remaining work, but the public contract and authorization scope still require maintainer product judgment; the closed, unmerged diagnostics PR does not supersede this issue.

## What This Changes

Add an agent-facing workload and queue-capacity query so agents can choose available dispatch capacity before calling `sessions_spawn`.

## Best Possible Solution

Expose a minimal, permission-filtered preflight capacity view using the existing lane snapshot and subagent registry primitives, preferably by extending an existing session tool unless maintainers approve a distinct queue-status contract with broader justified consumers.

## Reproduction Assessment

## Solution Assessment

No, not exactly as proposed: a raw all-lane/per-agent status tool is broader than necessary; a narrow permission-filtered capacity surface built from existing primitives is the more maintainable direction.
