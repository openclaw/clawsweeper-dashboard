## Summary

Keep open. Current main still lacks the requested agent-facing `queue_status` tool or equivalent dispatcher-oriented queue snapshot contract, and the related diagnostics PR is still open and explicitly excludes the in-agent tool.

## What This Changes

Request to add an agent-facing `queue_status` tool that reports queue depth, lane concurrency, active runs, and per-agent workload for smarter dispatch decisions.

## Best Possible Solution

Define one shared read-only queue snapshot contract, then expose `queue_status` through the agent tool catalog/profile/policy surface with permissions, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives high-confidence evidence that the requested stable agent-facing queue-status surface is absent.

## Solution Assessment

Yes at the design level: a shared read-only snapshot provider plus an agent-facing tool is the narrow maintainable direction. The open diagnostics PR can supply part of the backend if it lands, but it intentionally does not close the agent-tool request.
