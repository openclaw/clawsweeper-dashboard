## Summary

Keep open: current main still lacks the requested agent-facing queue-status tool or equivalent policy-gated queue snapshot surface, and the related open diagnostics PR explicitly leaves the in-agent tool out of scope.

## What This Changes

Add an agent-facing `queue_status` tool so dispatching agents can inspect lane queue depth, active concurrency, and per-agent workload before calling `sessions_spawn`.

## Best Possible Solution

Define one shared read-only queue snapshot backend, then expose a policy-gated `queue_status` tool through the agent tool catalog/profile surface, reusing the diagnostics backend from https://github.com/openclaw/openclaw/pull/43618 if it lands first.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high confidence that the requested agent-facing tool and stable queue snapshot surface are absent.

## Solution Assessment

Unclear until maintainers approve the agent-visible API shape and workload-metadata visibility rules. The maintainable direction is a shared snapshot provider plus a policy-gated `queue_status` tool, not inference from `agents_list`, `sessions_list`, or logs.
