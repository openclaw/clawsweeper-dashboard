## Summary

Keep this open: current main still lacks an agent-facing `queue_status` tool or equivalent stable queue snapshot surface, and the related operator diagnostics PR at https://github.com/openclaw/openclaw/pull/43618 is still open and explicitly does not add the in-agent dispatch tool.

## What This Changes

Add an agent-facing `queue_status` tool that exposes lane and per-agent workload state so agents can dispatch tasks without blindly spawning onto busy queues.

## Best Possible Solution

Define a shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with visibility filtering, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence proof that no agent-facing `queue_status` or equivalent stable queue snapshot surface exists yet.

## Solution Assessment

Unclear until maintainers approve the API shape and metadata-visibility rules. The maintainable direction is a shared snapshot provider plus an agent-facing tool, not inference from `agents_list`, `sessions_list`, or verbose logs.
