## Summary

Keep open. Current main still lacks the requested agent-facing `queue_status` tool or equivalent dispatcher-oriented queue snapshot contract; the closest related work is an open operator-diagnostics PR that explicitly excludes the in-agent tool.

## What This Changes

The issue requests an agent-facing `queue_status` tool that reports lane queue depth, active concurrency, and per-agent workload for dispatcher-style task routing.

## Best Possible Solution

Define or reuse one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with docs, permissions, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, live issue, and related-PR inspection give high-confidence proof that the stable agent-facing queue-status surface is absent on current main.

## Solution Assessment

Yes at the design level: a shared read-only snapshot provider plus an agent-facing tool is the maintainable direction. It is not ready for an automated fix lane because the public JSON shape, visibility rules, and coordination with the open diagnostics PR need maintainer decisions.
