## Summary

Keep open. Current main still has internal command-lane snapshot helpers and log/diagnostic surfaces, but it lacks the requested agent-facing `queue_status` tool, catalog/profile/policy wiring, or equivalent dispatcher-oriented queue snapshot contract; the related diagnostics PR is still open and explicitly excludes the in-agent tool.

## What This Changes

Request a new agent-facing `queue_status` tool that reports lane depth, active runs, concurrency, and per-agent workload for dispatcher agents.

## Best Possible Solution

Define or reuse one shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with docs, permissions, and focused tests coordinated with https://github.com/openclaw/openclaw/pull/43618.

## Reproduction Assessment

Not applicable as a feature request. Current source, docs, and related-PR inspection give high-confidence proof that the stable agent-facing queue-status surface is absent on current main.

## Solution Assessment

Yes at the design level: a shared read-only snapshot provider plus an agent-facing tool is the maintainable direction. It is not ready for an automated fix lane because the public JSON shape, visibility rules, and coordination with the open diagnostics PR need maintainer decisions.
