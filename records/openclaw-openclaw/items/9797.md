Keep this open: current main still lacks an agent-facing `queue_status` or equivalent stable queue snapshot surface, and the related operator diagnostics PR remains open and explicitly out of scope for in-agent dispatch.

## What This Changes

## Best Possible Solution

Define a shared read-only queue snapshot backend, then expose `queue_status` through the agent tool catalog/profile/policy surface with visibility filtering, docs, and focused tests, reusing the operator diagnostics backend if it lands first.
