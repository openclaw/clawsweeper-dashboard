## Label Justifications

- `P1`: The report describes a current Android chat workflow leaking internal reasoning and tool details into user-visible bubbles.

## Summary

Keep open: current main and v2026.6.8 still have a source-reproducible Android path where text-bearing internal/tool rows from chat.history can become visible OpenClaw chat bubbles, and the two linked fixing PRs are closed unmerged after reviews found they only covered already-filtered content-block cases.

## What This Changes

The issue requests that Android hide assistant thinking/reasoning and tool-call detail blocks instead of rendering them as separate visible chat messages.

## Best Possible Solution

Filter internal tool-result, tool-call, thinking, and reasoning rows from the gateway chat display projection, keep Android role/content filtering as defense in depth, add focused regression coverage, and verify with redacted Android proof before closing.

## Reproduction Assessment

No live high-confidence Android reproduction was run in this read-only review. Source inspection gives a clear reproduction path: a chat.history row with role toolResult/internal and string or top-level text content is parsed into ChatMessage and can render as an OpenClaw bubble.

## Solution Assessment

No, the closed PR approach is not the best fix because it only handled typed content blocks that current main already drops. The maintainable fix should cover the row-level gateway display boundary and keep Android-side filtering as a defensive layer.
