## Summary

Keep open: current main still filters some typed thinking blocks, but text-bearing internal/tool `chat.history` rows can still become Android chat bubbles labeled as OpenClaw.

## What This Changes

The issue requests that Android hide assistant thinking/reasoning and tool-detail content so only user-visible assistant responses render in chat.

## Best Possible Solution

Filter non-user-visible internal/tool/reasoning rows at the gateway display projection, keep Android role/content filtering as defense in depth, and add focused regression plus redacted Android proof.

## Reproduction Assessment

Yes, source-level: current Android history parsing accepts text-bearing non-user/non-system rows from `chat.history`, and current rendering labels those rows as OpenClaw. I did not run a live Android device or emulator in this read-only review.

## Solution Assessment

No, current main is not the best final state because block-level filtering exists but row-level internal/tool content is still not filtered before Android rendering. The best fix should hide those rows at the gateway display boundary and keep Android defense-in-depth coverage.
