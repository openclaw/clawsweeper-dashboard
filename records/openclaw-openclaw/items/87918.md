## Summary

Keep open: current main now strips thinking/tool-call blocks inside assistant content, but the row-level path still lets a text-bearing internal/toolResult history row become an Android chat bubble labeled as OpenClaw. The two linked fix PRs are closed unmerged and only covered the block-level guard, so this issue remains the canonical security-sensitive report.

## What This Changes

The issue reports that Android displays assistant thinking/reasoning and tool-call detail content as separate user-visible chat messages.

## Best Possible Solution

Filter non-user-visible internal/tool/reasoning rows at the gateway display projection, keep Android role/content filtering as defense in depth, add focused regression coverage, and verify with redacted Android proof.

## Reproduction Assessment

Yes, at source level: a chat.history row with a non-user role and text content can pass Android parsing and render with the OpenClaw label. I did not run a live Android device/emulator reproduction in this read-only review.

## Solution Assessment

No, the current partial block-level filtering is not the best final fix because it does not close the row-level display boundary. The narrow maintainable fix should hide internal rows before they reach Android rendering and keep Android-side guards as defense in depth.
