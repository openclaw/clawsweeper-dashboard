## Label Justifications

- `P1`: The report describes a current Android chat leak of internal reasoning and tool details into user-visible conversation bubbles.
- `impact:security`: The affected data can include internal reasoning, tool-call parameters, job/session identifiers, and delivery metadata that should not be exposed as normal chat content.

## Summary

Keep open: current code has some defenses for assistant `thinking` and `toolCall` blocks, but the Android history/render path can still render text-bearing non-chat roles such as `toolResult` as normal OpenClaw bubbles, which matches the reported leak of internal tool details.

## What This Changes

The issue reports that Android renders assistant thinking/reasoning and tool-call details as separate visible chat messages instead of hiding them or showing only final text.

## Best Possible Solution

Filter display history so Android never renders internal tool-result/tool-call/reasoning rows as chat bubbles, preferably at the gateway display projection with an Android defensive guard and regression tests for `toolResult` rows plus assistant `thinking` blocks.

## Reproduction Assessment

Yes from source inspection, but not from a live device run: a `chat.history` row with role `toolResult` and text content is accepted by Android history parsing and rendered as an OpenClaw bubble. Android already drops non-text content parts, so the remaining reproducible gap is row/role-level filtering.

## Solution Assessment

Yes, a narrow source fix is available: keep only display-safe roles/content in gateway history and/or Android parsing, then add focused Android and gateway regression coverage. A collapsible thinking UI can be a later feature; the bug fix should hide internal rows by default.
