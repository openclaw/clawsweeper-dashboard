## Summary

Keep open: current main and v2026.6.9 cover some block-level filtering, but the source path still allows text-bearing tool/internal rows from chat.history to become Android chat bubbles labeled as OpenClaw.

## What This Changes

The issue asks Android chat to hide LLM thinking/reasoning blocks and raw tool-call details so only user-visible assistant text is rendered.

## Best Possible Solution

Filter internal/tool/reasoning rows at the gateway chat display projection, keep Android text/image rendering as defense in depth, add regression coverage, and verify with redacted Android proof.

## Reproduction Assessment

No live Android run was performed, but the source path is high-confidence: chat.history can supply a text-bearing tool/internal row, Android preserves that role/content, and the renderer labels unknown roles as OpenClaw.

## Solution Assessment

No, current main is not the best final state because it filters typed blocks but still lacks a row-level display boundary. The best fix should hide non-user-visible rows before Android renders them and prove the behavior with tests and Android proof.
