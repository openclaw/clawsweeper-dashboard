## Summary

Keep open. Current main and the latest release still have the reported provenance mismatch: official ClawHub trust for a ClawHub-source install still requires a catalog `clawhubSpec`, while the official Microsoft Teams catalog entry is npm-only, so the keyed-store runtime gate can still reject the channel.

## What This Changes

Fix official ClawHub installs of `@openclaw/msteams` so npm-only catalog provenance can satisfy the trusted keyed-store gate or fail early with actionable guidance instead of crash-looping.

## Best Possible Solution

Align official install provenance and runtime trust so msteams official ClawHub installs either pass with URL/package validation plus proof, or fail before startup with actionable npm reinstall or doctor guidance.

## Reproduction Assessment

Yes, at source level. The msteams catalog entry has only `npmSpec`, while the ClawHub trust branch requires `officialInstall?.clawhubSpec`, so `trustedOfficialInstall` stays false and the runtime state gate throws.

## Solution Assessment

Unclear. A narrow predicate fix is plausible, but the best solution still needs maintainer confirmation of the official ClawHub/npm trust boundary and real startup or install-path proof.
