## What This Changes

The issue reports that `openclaw plugins install clawhub:@openclaw/msteams` installs cleanly but cannot receive trusted keyed-store access because the official catalog entry is npmSpec-only, causing the Teams channel to restart repeatedly with an unactionable trust error.

## Best Possible Solution

Keep the issue open until maintainers approve and land a narrow official-ClawHub npmSpec trust rule with package identity, official channel, official URL, install-record, and regression-test coverage while leaving broader self-hosted trust work separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a ClawHub official install record for `msteams` cannot satisfy the current `officialInstall?.clawhubSpec` branch, so `trustedOfficialInstall` remains false and the Teams keyed-state call hits the runtime trust error; I did not run the live K8s/Teams deployment in this read-only review.

## Solution Assessment

Unclear until maintainers approve the trust-boundary rule. The narrow code location is `src/plugins/manifest-registry.ts`, but the best fix must preserve official ClawHub URL, official channel, package identity, plugin id, and matching install-record checks rather than adding a Teams-only bypass or broad trust fallback.
