## Summary

This issue remains necessary: current main still has the same official ClawHub provenance mismatch for Microsoft Teams, and the linked Microsoft tracker plus broader self-hosted trust issue do not replace the msteams-specific install-path bug.

## What This Changes

The issue asks OpenClaw to make official ClawHub installs of `@openclaw/msteams` pass the keyed-store trust gate or fail earlier with an actionable install or doctor diagnostic.

## Best Possible Solution

Preserve strict package identity and official-channel verification while making the official ClawHub install path either trustable by design or refused/diagnosed before the channel starts.

## Reproduction Assessment

Yes, at source level: current main still requires `trustedOfficialInstall` for `openKeyedStore`, the official msteams catalog entry has no `clawhubSpec`, and msteams opens keyed stores. I did not run the reporter's Kubernetes/OCI deployment live.

## Solution Assessment

Unclear until maintainers choose the trust policy. The best fix is a generic provenance-safe rule or pre-start diagnostic, not a broad load-path or ClawHub-source-only trust bypass.
