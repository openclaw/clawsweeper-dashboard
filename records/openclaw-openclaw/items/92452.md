## Summary

Keep open: current main and the latest release still contain the reported provenance mismatch, and the only narrow candidate fix is closed unmerged. The issue remains a valid security-boundary crash-loop bug for official Microsoft Teams ClawHub installs.

## What This Changes

The issue asks OpenClaw to make official ClawHub installs of npmSpec-only official catalog entries such as Microsoft Teams satisfy the trusted keyed-store gate or report an actionable install/provenance error.

## Best Possible Solution

Implement a narrow official-ClawHub/npm-only trust rule that preserves URL, official-channel, package identity, plugin-id, and installed-path checks, with regression coverage and real install/startup proof if maintainers require it.

## Reproduction Assessment

Yes, source-reproducible: current `main` and `v2026.6.8` still require `clawhubSpec` for official ClawHub trust while Microsoft Teams is npmSpec-only and uses `openKeyedStore`. I did not run a live Kubernetes or ClawHub/Microsoft Teams deployment in this read-only review.

## Solution Assessment

Unclear until maintainers accept the trust-boundary rule; the narrow code location is the manifest trust predicate, but the fix must preserve official ClawHub URL, official channel, package, plugin-id, and install-record checks. A Microsoft Teams-only catalog exception or raw trust bypass would be the wrong shape.
