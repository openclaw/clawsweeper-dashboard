## Summary

Keep open: current main and v2026.6.6 still have the reported official-provenance mismatch, so a ClawHub-installed Microsoft Teams plugin can install as official but still fail the keyed-store trust gate.

## What This Changes

The issue asks for the Microsoft Teams ClawHub install path to either satisfy the trusted keyed-state gate or fail/diagnose clearly before the channel crash-loops.

## Best Possible Solution

Keep this in core and fix the generic official-provenance path: verify package id, install record, install path, and official ClawHub channel, then either trust the official install or refuse/doctor it before channel startup.

## Reproduction Assessment

Yes, at source level: an official-channel ClawHub record for Microsoft Teams cannot pass the current predicate because its catalog entry lacks clawhubSpec, and the plugin opens keyed stores during startup. This read-only review did not run the Kubernetes deployment.

## Solution Assessment

Unclear among the listed options: the report identifies the right failure, but the best fix should be a generic provenance-safe trust/diagnostic path rather than a Microsoft Teams-only exception.
