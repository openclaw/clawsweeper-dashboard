## Summary

Keep open: current main and v2026.6.6 still have the reported provenance mismatch, and the related open issues are broader trust-boundary reports rather than a merged fix or safer canonical replacement for this concrete Microsoft Teams ClawHub install failure.

## What This Changes

The issue reports that ClawHub-installed `@openclaw/msteams` cannot satisfy the keyed-store trust gate because the official catalog entry only declares npm install metadata.

## Best Possible Solution

Preserve strict official-plugin provenance while making official ClawHub installs of official npm-first channel plugins either trusted through equivalent catalog proof or rejected with an actionable install/doctor diagnostic before startup.
