## Summary

Keep open: current main and v2026.6.6 still have the reported provenance mismatch, and the related self-hosted issue is broader rather than a replacement for this ClawHub-specific Microsoft Teams install failure.

## What This Changes

The issue reports that installing the official Microsoft Teams plugin through ClawHub leaves it unable to satisfy the keyed-store trusted-plugin gate because the official catalog entry is npm-only.

## Best Possible Solution

Preserve strict official package and channel provenance while making official ClawHub installs of npm-first official packages either trusted or rejected with an actionable install/doctor diagnostic before channel startup.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 still combine an npm-only Microsoft Teams official catalog entry with a ClawHub trust predicate that requires clawhubSpec. I did not run the live Kubernetes/gateway flow in this read-only review.

## Solution Assessment

Unclear which reporter-suggested option is best; the maintainable fix should be a generic provenance-safe rule or actionable pre-start diagnostic, not a Microsoft Teams-only trust exception.
