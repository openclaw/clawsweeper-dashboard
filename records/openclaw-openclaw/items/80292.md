## Summary

Keep open. The extension-local handler is the right product boundary, but the patch deregisters it before shutdown/export flushing and the external PR still has only mocked test proof, so it should not merge as-is.

## What This Changes

The PR changes diagnostics-otel to register a plugin-scoped unhandled-rejection handler for OTLPExporterError and adds focused service tests.

## Best Possible Solution

Keep the diagnostics-otel-owned handler installed until after log provider and SDK shutdown complete, add regression coverage for shutdown-time export failures, and attach redacted real collector-failure proof.

## Reproduction Assessment

Yes, source-reproducible: enable diagnostics-otel against an OTLP endpoint that returns a non-retryable response such as HTTP 410, trigger export or shutdown flush, and current main has no plugin-local handler before the global fatal fallback. I did not run a live collector crash repro in this read-only pass.

## Solution Assessment

No, not as written. The extension-local handler is the narrowest maintainable direction, but deregistration should happen after shutdown/flush in a finally block and the PR still needs real behavior proof.
