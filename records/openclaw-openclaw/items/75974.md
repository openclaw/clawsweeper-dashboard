## Summary

Keep open: the performance problem linked to #75956 and #75290 appears real, but this PR is not mergeable because its global `pluginId` cache crosses request-scoped trusted tool context and has a cache-hit timing type bug.

## What This Changes

The PR adds a global plugin tool factory result cache keyed by `pluginId`, a reset helper/test, and a changelog entry for the Windows plugin-tool slowdown.

## Best Possible Solution

Redesign plugin tool performance around context-safe opt-in memoization or a complete registry/config/request cache key, and add regression coverage for browser, Lobster, and Zalo Personal context-sensitive factories.

## Reproduction Assessment

Yes for the PR regression: call `resolvePluginTools()` twice for the same plugin id with different `sandboxed`, browser, session, or delivery context values and the cache reuses the first tool. The original Windows slowdown has detailed reports in #75956 and #75290 but was not live-run in this read-only pass.

## Solution Assessment

No. Caching may be the right performance direction, but a global `pluginId` cache is not the narrowest maintainable fix because current factory outputs can intentionally depend on trusted request context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Unpack the factory timing result:** `src/plugins/tools.ts:465`
  - body: `describePluginToolFactoryResult()` returns `{ result, resultCount }`, but this cache-hit branch assigns that whole object to `PluginToolFactoryTiming.result`, whose type is the string union consumed by timing formatting. This should fail type checking and would also log the wrong result shape.
  - confidence: 0.97
- **[P1] Preserve request-scoped factory context:** `src/plugins/tools.ts:457`
  - body: This cache is keyed only by `pluginId`, but current factories consume request-scoped context such as browser policy, session key, `sandboxed`, and delivery route. After the first request, later sessions or channels can inherit stale tool behavior from the cached object.
  - confidence: 0.93

## Security Review
