Current main contains a targeted fix for this doctor false warning: doctor now trusts a checked-ready gateway memory probe when CLI-local memory backend resolution is unavailable, with regression coverage. The fix landed after v2026.5.2 and is not in a release tag yet.

## What This Changes

Doctor reported a missing active memory plugin even though memory-core/active-memory were loaded and memory status showed a healthy Ollama-backed memory runtime.

## Best Possible Solution

Keep the current-main guard and ship it in the next release; reopen only if a build containing 1bc97aee still warns when the gateway memory probe is checked and ready.

## Reproduction Assessment

No current-main failing path remains from source inspection: the current code returns before the warning when the gateway memory probe is checked and ready, and the regression test covers that exact condition. I did not run live CLI commands because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The fix is narrow because it trusts runtime evidence only when the gateway has actually checked readiness, while preserving the warning for skipped, unavailable, or not-ready probes.
