## Summary

Current main already fixes this doctor/status false positive by loading the configured memory-slot plugin before reporting that no memory runtime is active; the fix is on main after v2026.5.2 and is covered by focused regression tests.

## What This Changes

Doctor should stop warning that no active memory plugin is registered when bundled memory-core and active-memory are loaded and memory status is healthy.

## Best Possible Solution

Keep the merged memory-runtime slot-loading implementation and let the next release carry it; no further source change is needed for this report.

## Reproduction Assessment

No for current main. I did not run a live Linux systemd/Ollama gateway in this read-only review, but the current source no longer has the reported failing path because doctor resolves the active backend through a resolver that loads the configured memory slot first.

## Solution Assessment

Yes. The merged fix is narrow and maintainable: it scopes standalone loading to the configured memory slot and preserves disabled/denied slot behavior with regression coverage.
