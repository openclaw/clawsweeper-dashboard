## Summary

Keep open: current main still has a doctor path that can emit the reported “No active memory plugin” warning before reconciling healthy gateway or memory-status evidence, and I found no merged fix that clearly resolves this report.

## What This Changes

Report asks doctor to detect a configured, healthy `memory-core` memory setup or give precise remediation instead of saying no active memory plugin is registered.

## Best Possible Solution

Make doctor reconcile the configured memory slot, local runtime resolver, and gateway memory probe, then suppress or replace the warning when the active memory runtime is demonstrably healthy.

## Reproduction Assessment

Yes, source-level: `noteMemorySearchHealth` deterministically emits the reported warning when `resolveActiveMemoryBackendConfig` returns null, and the issue supplies a concrete `memory-core` config plus healthy `memory status --deep` output. I did not run a live CLI reproduction in this read-only checkout.

## Solution Assessment

Unclear until patched, but the narrow maintainable direction is in the doctor/runtime health check rather than a new config option or plugin policy change.

## Repair Work Prompt

Fix the doctor false-positive reported at https://github.com/openclaw/openclaw/issues/76705. When `plugins.slots.memory` selects `memory-core` and the gateway or memory runtime proves memory is healthy, `openclaw doctor --non-interactive` must not emit `No active memory plugin is registered for the current config.` Keep the fix generic to active memory plugins: update `src/commands/doctor-memory-search.ts` and, only if needed, `src/plugins/memory-runtime.ts` or `src/flows/doctor-health-contributions.ts` so doctor reconciles the local backend resolver with `gatewayMemoryProbe`/runtime status before warning. Add focused regression coverage in `src/commands/doctor-memory-search.test.ts` for a configured memory slot with a healthy gateway probe, and keep the existing true no-runtime warning case. Add a single-line user-facing `CHANGELOG.md` fix entry. Do not add a config option, hard-code bundled plugin behavior beyond the existing memory slot contract, or move plugin-owned runtime logic into unrelated core code.
