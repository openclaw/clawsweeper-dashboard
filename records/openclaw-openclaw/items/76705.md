## Summary

Current main already fixes the reported `doctor` false positive by loading the configured memory-slot plugin before doctor/status resolves the active memory backend. The reporter is on `v2026.5.2`, which predates the fix, so the fix is main-only until the next release.

## What This Changes

`openclaw doctor --non-interactive` should not report that no active memory plugin is registered when `plugins.slots.memory` selects a loaded, working `memory-core` runtime.

## Best Possible Solution

Keep the current main implementation and let the next OpenClaw release carry the slot-scoped doctor/status fix.
