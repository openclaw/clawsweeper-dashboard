## Summary

Keep open. Current main still has the doctor warning path, and no post-report change touched the doctor/memory runtime files; the report describes a credible mismatch between healthy gateway memory status and CLI-side doctor runtime detection.

## What This Changes

The issue asks doctor to stop reporting that no active memory plugin is registered when `memory-core` is selected, loaded, and `openclaw memory status --deep` reports healthy memory.

## Best Possible Solution

Update doctor memory health to trust or surface the gateway memory-status result when the running gateway has an active memory manager, and reserve the no-active-plugin warning for cases where both CLI runtime resolution and gateway status cannot find the selected memory plugin.

## Reproduction Assessment

No, not as an executed current-main repro. The issue body gives a credible release reproduction and source inspection identifies the exact warning branch, but the installed-package runtime mismatch still needs a targeted regression test.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the doctor memory health/gateway probe boundary, not in `memory-core` or `active-memory`; `openclaw memory status` already demonstrates the runtime can be found through the gateway path.
