## Summary

Keep open: current main has a compaction indicator and automatic compaction event handling, but the manual Control UI `/compact` path waits for `sessions.compact` and only displays a post-result command message, so the reported missing in-progress state remains valid from source inspection.

## What This Changes

Control UI should show a visible in-progress compaction indicator when a user manually runs `/compact`.

## Best Possible Solution

Reuse the existing Control UI compaction indicator state for manual `/compact`, setting it active before the RPC and completing or clearing it on success, skip, or error without adding a new command or config knob.

## Reproduction Assessment

Yes at source level: run `/compact` through the Control UI local slash-command path and the UI waits on `sessions.compact` without setting `compactionStatus`, while the visible indicator only appears when that state is set. I did not establish a live browser reproduction in this read-only review.

## Solution Assessment

Yes: the narrow fix should reuse `compactionStatus` and `renderCompactionIndicator` for the manual command path rather than creating a separate banner or changing compaction policy.
