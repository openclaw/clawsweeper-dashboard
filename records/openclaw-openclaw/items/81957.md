## What This Changes

The branch hardens release/publish GitHub Actions by pinning actions, tightening permissions and trusted-ref gates, splitting plugin npm trusted publishing from dist-tag mirroring, adjusting cache boundaries, and adding token-boundary tests.

## Best Possible Solution

Repair the ref-validation order, preserve Tideclaw alpha ClawHub compatibility unless release owners explicitly approve its removal, refresh current-head proof, then run maintainer-controlled release/security verification.

## Reproduction Assessment

Yes, for the review blockers by source inspection: manual plugin npm dispatch reaches checkout-local setup before the trusted-ref validation, and manual ClawHub alpha dispatch hits the new main/release-only gate. No live publish was run because those paths mutate external release state.

## Solution Assessment

No; the hardening direction is appropriate, but the current implementation is not the safest merge shape while it leaves a pre-validation local action path and blocks alpha ClawHub publishes. The safer fix is to validate selected refs with trusted workflow logic before local actions and preserve existing alpha compatibility unless release owners approve a removal.
