## Summary

Keep open: latest head appears to fix the earlier npm-token exposure, but the split publish/mirror flow now loses the pre-publish mirror-auth guard, and the posted real behavior proof still points at the older pre-fix head.

## What This Changes

The PR hardens release, publish, live/e2e, and artifact workflows by narrowing permissions, pinning external actions, adding trusted-ref gates, isolating plugin publish auth, and adding token-boundary coverage.

## Best Possible Solution

Keep the hardening direction, preserve the pre-publish mirror-auth availability guard without exposing the token to npm publish, and refresh proof against the latest head before merge.

## Reproduction Assessment

Yes. By source inspection, latest `--publish-package` makes mirror auth optional while the workflow publishes before the authenticated mirror step, so missing `NPM_TOKEN` is detected only after package publish.

## Solution Assessment

No. OIDC publishing and token isolation are the right direction, but the patch should keep the old no-divergence guard and update real behavior proof for the latest commit.
