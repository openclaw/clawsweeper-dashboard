## Summary

Keep open: the hardening direction is useful, but latest head still allows a stable plugin package publish before mirror auth is proven, and the posted terminal proof is for the previous head rather than the token-boundary commit.

## What This Changes

The PR narrows high-risk workflow permissions and trusted refs, pins external GitHub Actions, changes cache boundaries, and splits plugin npm package publishing from dist-tag mirroring.

## Best Possible Solution

Keep the workflow hardening, preserve a trusted pre-publish mirror-auth availability guard without exposing the token to npm publish, then refresh proof against the latest head.

## Reproduction Assessment

Yes. Source inspection shows a stable plugin release with required beta mirroring can run --publish-package without mirror auth, then fail only in the later authenticated mirror step.

## Solution Assessment

No. OIDC package publishing and token isolation are the right direction, but the patch should keep the pre-publish no-divergence guard and refresh proof for the latest head.
