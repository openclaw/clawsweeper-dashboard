## Summary

Keep open. The current head refreshes the token-boundary proof, but it still drops current main’s pre-publish mirror-auth guard: stable plugin packages can be published before a missing npm mirror token is detected in the later dist-tag step.

## What This Changes

The PR pins high-risk workflow actions, narrows release/workflow permissions and trusted refs, removes the ClawHub token fallback, splits plugin npm trusted publishing from dist-tag mirroring, and adds a token-boundary test.

## Best Possible Solution

Keep the hardening direction, but add a separate pre-publish mirror-auth availability preflight so stable npm releases fail before package publication while the publish subprocess remains token-free.

## Reproduction Assessment

Yes. By source inspection, a stable plugin version that needs beta mirroring can run `--publish-package` without mirror auth, publish successfully, and only then fail in the separate `--mirror-dist-tags` step if `NPM_TOKEN` is missing.

## Solution Assessment

No. OIDC publishing and token isolation are the right direction, but the patch should preserve the no-divergence guard with a no-side-effect authenticated preflight before package publish.
