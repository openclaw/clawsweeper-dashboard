## Summary

Keep open. Current main still rejects optional human-readable metadata on group/channel config entries, and this is a public config-contract enhancement that needs maintainer choice on the canonical key and cross-channel scope.

## What This Changes

Add an optional string metadata field such as `name`, `label`, or `description` to group/channel config entries so numeric group IDs are easier for humans and agents to identify.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to relevant group/channel entry types, strict schemas, generated metadata, docs, and validation coverage while runtime ignores it.

## Solution Assessment
