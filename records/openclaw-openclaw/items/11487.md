## What This Changes

The issue requests an optional string metadata field such as `name`, `label`, or `description` on channel group/channel config entries so numeric group IDs are readable to humans and agents.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to group-like config entry types, strict schemas, generated metadata, public docs, and validation coverage while keeping runtime behavior unchanged.
