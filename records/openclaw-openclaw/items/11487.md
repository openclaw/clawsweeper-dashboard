## Summary

Keep open. Current main still lacks a cross-channel optional human-readable group metadata field, and the discussion already identifies the remaining maintainer choice around the canonical key and scope.

## What This Changes

Add an optional string metadata field such as `name`, `label`, or `description` to channel group config entries so numeric group IDs are readable for humans and agents.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to relevant group/channel entry types, strict Zod schemas, generated metadata, docs, and validation coverage while preserving runtime behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-contract request. Source inspection gives high confidence that current main still omits and rejects the proposed field for central group/channel entries.

## Solution Assessment

Yes directionally, but not as an automated cleanup yet. The narrow maintainable path is a deliberate public config-contract change after maintainers choose the canonical key and exact cross-channel scope.
