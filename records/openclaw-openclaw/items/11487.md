## Summary

Keep open. Current main still rejects the requested optional group-entry metadata in the strict config contract, and the live discussion already marks this as a maintainer/product decision rather than cleanup.

## What This Changes

The issue requests an optional human-readable `name`, `label`, or `description` string on channel group config entries so numeric group IDs can be identified without changing runtime behavior.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to group/channel entry types, strict schemas, generated metadata, docs, changelog, and validation tests while preserving and ignoring it at runtime.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-contract request. Source inspection on current main gives high-confidence proof that strict schemas, TypeScript types, generated metadata, and docs still omit the requested metadata key.

## Solution Assessment

Unclear until maintainers choose the public key name and exact cross-channel scope. The maintainable path is a deliberate config-contract addition across schemas, types, generated metadata, docs, and tests rather than an ad hoc single-channel patch.
