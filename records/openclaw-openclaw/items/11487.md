## Summary

Keep open. Current `main` and the latest release still omit the requested human-readable group-entry metadata from representative strict schemas, TypeScript config types, generated channel metadata, and public docs, so this remains a product/config-contract enhancement rather than a cleanup close.

## What This Changes

The issue requests an optional ignored string metadata field such as `name`, `label`, or `description` on channel group config entries so numeric group IDs are readable.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to relevant group/channel entry types, strict schemas, generated metadata, docs, changelog, and validation tests while runtime ignores it.

Not applicable as a bug reproduction because this is a feature/config-contract request. Source inspection on current `main` gives high-confidence proof that representative strict schemas, TypeScript types, generated metadata, and docs still omit the requested metadata key.

## Solution Assessment
