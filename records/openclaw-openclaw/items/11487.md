## Summary

Keep open. Current main still omits the requested human-readable group metadata from strict group/channel schemas, TypeScript config types, generated channel metadata, and public group docs, so this remains a product/config-contract enhancement rather than cleanup that can be closed.

Request an optional string metadata key such as `name`, `label`, or `description` on channel group config entries for human and agent readability.

## Best Possible Solution

Choose one documented optional metadata key and add it consistently to relevant group/channel entry types, strict schemas, generated metadata, docs, changelog, and validation tests while runtime ignores it.
