## Summary

Keep open. Current main still omits optional human-readable metadata on strict group/channel config entries, so the enhancement remains a public config-contract decision rather than cleanup that can be safely closed.

## What This Changes

Add an optional string metadata field such as `name`, `label`, or `description` to group/channel config entries so numeric group IDs can be identified by humans and agents.

## Best Possible Solution

Choose one documented optional string metadata key and add it consistently to the relevant group/channel entry types, strict schemas, generated metadata, docs, changelog, and validation tests while runtime ignores it.

## Solution Assessment
