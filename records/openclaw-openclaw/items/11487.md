## What This Changes

Request to add one optional human-readable `name`, `label`, or `description` string field to channel group config entries while ignoring it at runtime.

## Best Possible Solution

Approve one canonical descriptive field and cross-channel scope, then update the relevant types, strict schemas, generated config metadata, docs, and round-trip validation tests together.

## Reproduction Assessment

Yes, at source level. Current strict schemas and TypeScript types for representative group/channel entries omit the requested descriptive field, so an unknown metadata key remains invalid.

## Solution Assessment

Yes in broad shape, but only after maintainers choose one canonical field name and exact scope. The safer alternative to ad hoc per-channel labels is one coherent config-contract change across schemas, types, metadata, docs, and tests.
