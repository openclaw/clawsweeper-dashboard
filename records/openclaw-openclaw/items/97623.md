## Summary

Keep open: the branch is a plausible mitigation but not merge-ready. Code and dependency review show it caps a shared dispatcher keep-alive hint while the Anthropic path still rethrows the first pre-response transport failure into model fallback, and the new proof shows option injection/startup rather than an affected after-fix gateway run avoiding fallback.

## What This Changes

The PR adds a 25-second `keepAliveMaxTimeout` default to OpenClaw's shared HTTP/1 undici dispatcher helper and updates network tests to expect it.

## Best Possible Solution

Land a bounded same-provider retry for transient pre-response model transport failures in `buildGuardedModelFetch`, or keep the shared cap only with owner approval and affected-gateway proof.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows the first pre-response transport failure, and failover can classify that failure as timeout. I did not live-reproduce the multi-hour stale-socket trigger.

## Solution Assessment

No. The cap is a plausible mitigation for one keep-alive-hint path, but the narrower maintainable fix is a bounded same-provider retry before model fallback unless affected-gateway proof shows this shared cap resolves the real path.
