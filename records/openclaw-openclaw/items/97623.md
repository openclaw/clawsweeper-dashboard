## Summary

Keep open: the PR is a focused attempt at the linked Anthropic fallback bug, but the changed default is a shared dispatcher policy and source/dependency review shows it does not prove the guarded model-fetch failure path is fixed.

## What This Changes

The PR adds a default 25-second undici `keepAliveMaxTimeout` inside OpenClaw's HTTP/1 dispatcher helper and updates related network tests to expect that option.

## Best Possible Solution

Land a bounded same-provider retry for clearly transient pre-response socket/fetch failures in guarded model transport before model fallback, or provide affected-gateway proof that the shared keep-alive cap fixes the reported Anthropic path without broader regressions.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows pre-response transport errors, and can hand those failures to model fallback. I did not live-reproduce the multi-hour stale-socket trigger.

## Solution Assessment

No. The cap is a plausible mitigation only for server keep-alive hints, while the narrower maintainable fix is a bounded same-provider retry before model fallback unless affected-gateway proof shows the cap resolves the real path.
