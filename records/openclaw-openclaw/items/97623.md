## What This Changes

The PR adds a default 25-second `keepAliveMaxTimeout` to OpenClaw’s HTTP/1 undici dispatcher helper and updates network tests to expect that option.

## Best Possible Solution

Fix the guarded provider path with a bounded same-provider retry or equivalent mitigation before model fallback, or provide maintainer-accepted gateway proof that the shared keep-alive cap resolves the reported Anthropic path.

## Reproduction Assessment

## Solution Assessment

No. The cap is a plausible shared-network mitigation, but upstream undici semantics and the current guarded fetch lifecycle make a bounded same-provider retry before fallback the narrower maintainable fix unless gateway proof shows otherwise.
