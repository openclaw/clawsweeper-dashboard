## Summary

Keep this PR open, but it is not merge-ready: the branch adds a shared undici keep-alive hint cap, while the canonical bug still needs the guarded Anthropic model-fetch path to survive transient pre-response socket/fetch failures before model fallback. The new proof shows constructor option injection and gateway startup, but not an affected Anthropic gateway run avoiding fallback after the reported failure window.

## What This Changes

The PR adds a default `keepAliveMaxTimeout: 25000` to OpenClaw’s shared undici HTTP/1 dispatcher helper and updates related network tests to expect that option.

## Best Possible Solution

Rework this into a bounded same-provider retry for transient pre-response socket/fetch failures in `buildGuardedModelFetch`, or provide affected-gateway proof plus maintainer acceptance for the broader shared dispatcher policy.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows the first pre-response transport failure, and the linked issue shows that failure advancing to model fallback. I did not live-reproduce the multi-hour stale-socket trigger in this read-only review.

## Solution Assessment

No. The cap is a plausible mitigation for server keep-alive hints, but the narrow maintainable fix is a bounded same-provider retry in guarded model transport before fallback runs, unless affected-gateway proof shows this shared cap resolves the real path.
