## Summary

Keep open: current main and the latest release still append the generated HEARTBEAT.md workspace-path hint to configured heartbeat prompts despite the documented verbatim contract, and open PR https://github.com/openclaw/openclaw/pull/40756 already owns the direct fix path.

## What This Changes

User-configured heartbeat prompts should remain verbatim and should not receive generated workspace-path guidance.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts.

## Reproduction Assessment

Yes. By source inspection, configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through `appendHeartbeatWorkspacePathHint`.

## Solution Assessment

Yes in direction. The narrow maintainable solution is to track when the base heartbeat prompt came from user config and skip only the generated workspace-path hint for that case, while preserving default/internal prompt guidance.
