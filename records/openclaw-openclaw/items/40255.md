## Summary

Keep open. Current main still appends the workspace HEARTBEAT.md path hint after resolving user-configured heartbeat prompts, which conflicts with the documented verbatim custom-prompt contract; #40756 is still open and unmerged with closing syntax for this issue.

## What This Changes

The issue requests that configured heartbeat prompts remain verbatim instead of receiving an automatically appended workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land or replace #40756 with a focused guard that skips workspace-path hint injection for non-empty configured prompts while preserving it for internal/default prompts, plus regression coverage for default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. From source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that configured text and then appends the workspace hint because the helper only inspects prompt contents.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base prompt is user-configured and skip appendHeartbeatWorkspacePathHint only in that case; #40756 is the current candidate but should not close this issue until it lands or is replaced.
