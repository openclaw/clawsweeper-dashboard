## Summary

Keep open: current main and the latest release still pass configured heartbeat prompt text through the generated HEARTBEAT.md workspace-path hint, despite docs saying custom heartbeat prompts are sent verbatim. The upstream fix PRs are closed; the remaining open closing reference is a fork-local PR, useful as source context but not an upstream landing path.

## What This Changes

User reports that custom heartbeat prompts mentioning HEARTBEAT.md are mutated by generated workspace-path guidance instead of being respected verbatim.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts, with focused regression coverage on the configured-prompt path.

## Reproduction Assessment

Yes, by source inspection. Configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to carry whether the base prompt is user-configured and skip only the generated workspace-path hint for those prompts while preserving default, exec, cron, and task-mode behavior.
