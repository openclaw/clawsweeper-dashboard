## Summary

Keep open. Current main still appends the workspace HEARTBEAT.md path hint after resolving user-configured heartbeat prompts, despite the docs saying custom heartbeat prompts are sent verbatim; #40756 remains the open closing PR for the same bug.

## What This Changes

The issue reports that custom heartbeat prompts are being mutated by unconditional workspace HEARTBEAT.md path-hint injection.

## Best Possible Solution

Land or replace #40756 with a focused guard that skips hint injection for non-empty custom heartbeat prompts while preserving the hint for internal/default prompts, plus regression coverage for default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. Configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that configured text and then appends the workspace hint because the helper only inspects prompt contents.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base prompt is user-configured and skip appendHeartbeatWorkspacePathHint only in that case; the issue should remain open until #40756 lands or is replaced.
