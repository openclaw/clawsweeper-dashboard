## Label Justifications

- `P1`: This is a documented heartbeat configuration regression that can break recurring agent workflows for affected users.
- `impact:session-state`: The bug changes the prompt/context state used by scheduled heartbeat turns instead of preserving the configured user prompt.
- `impact:message-loss`: Reporter evidence shows heartbeat-driven standing programs can silently return empty or HEARTBEAT_OK, preventing expected proactive messages.

## Summary

Keep open: current main still mutates configured heartbeat prompts that mention HEARTBEAT.md, the latest release still has the same unconditional hint path, and the canonical issue is already paired with an open but conflicting cross-repo fix candidate.

## What This Changes

The issue asks OpenClaw to honor user-configured heartbeat prompts instead of appending or replacing them with hardcoded heartbeat guidance.

## Best Possible Solution

Land, refresh, or replace a narrow heartbeat fix that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance only for default or internally generated prompts.

## Reproduction Assessment

Yes, at source level. Configure a heartbeat.prompt containing HEARTBEAT.md and current main resolves that text as the base prompt, then still passes it through the unconditional workspace-path hint helper before dispatch.

## Solution Assessment

No, current main is not the solved state. The narrow maintainable solution is to separate user-authored heartbeat prompt text from generated heartbeat guidance and prove the configured prompt reaches the model-visible body.
