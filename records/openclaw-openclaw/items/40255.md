## Summary

Keep open. Current main still appends the workspace HEARTBEAT.md path hint after resolving configured heartbeat prompts, which conflicts with the documented verbatim custom-prompt contract; #40756 is an open, unmerged fix candidate and should land or be replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to stop appending the workspace HEARTBEAT.md path hint to user-configured heartbeat prompts.

## Best Possible Solution

Land a focused heartbeat-runner change that preserves the workspace-path hint for built-in/internal prompts, skips it for non-empty configured prompts, and adds regression coverage for both default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. From source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that configured text and then appends the workspace hint because the helper only inspects prompt contents.

## Solution Assessment

Yes. The narrow maintainable fix is a custom-prompt/origin guard around hint injection; #40756 is the current open candidate, but this issue should remain open until that PR lands or is replaced.
