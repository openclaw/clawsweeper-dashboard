## Label Justifications

- `P1`: The regression can break active heartbeat-driven workflows for users who rely on configured prompt instructions.
- `impact:session-state`: The issue affects heartbeat run prompt/context behavior and can make the agent session act on the wrong standing instructions.
- `impact:message-loss`: When the configured prompt is contradicted or ignored, expected heartbeat-driven work or notifications can be silently missed.

## Summary

Keep open: current main and the latest release still append the workspace HEARTBEAT.md hint to any resolved heartbeat prompt that mentions HEARTBEAT.md, which conflicts with the documented verbatim custom-prompt contract; newer duplicate evidence is already routed back here and a linked open fix path exists.

## What This Changes

The issue asks OpenClaw to stop overriding or mutating a user-configured heartbeat prompt during heartbeat runs.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while still adding the workspace path hint only for the built-in/default prompt, with regression coverage for both default and custom cases.

## Reproduction Assessment

Yes at source level: configure a heartbeat prompt that mentions HEARTBEAT.md and current main resolves it, then appends the workspace path hint unconditionally. I did not execute a live heartbeat turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to guard the workspace path hint so it applies to built-in/generated default prompts but not user-authored configured prompts. This matches the docs without adding new config or product policy.
