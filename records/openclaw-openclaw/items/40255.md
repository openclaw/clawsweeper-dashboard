- `impact:session-state`: The bug changes the prompt/context state used by scheduled heartbeat turns instead of preserving the configured user prompt.

## Summary

Keep open: current main and the latest stable release still do not honor the documented heartbeat prompt contract. The issue remains canonical, and the linked open cross-repo fix PR is still dirty and unmerged, so this should stay open until maintainers refresh, replace, or deliberately close that fix path.

## What This Changes

The issue reports that user-configured heartbeat prompts are mutated or replaced instead of being sent verbatim as the heartbeat user message.

## Best Possible Solution

Land a narrow heartbeat fix that keeps the transcript placeholder storage-only, passes the configured heartbeat prompt as the model-visible user message, appends the workspace hint only for built-in/internal prompts, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, at source level. Current main resolves configured heartbeat text but still mutates it with the workspace hint and can surface the heartbeat transcript placeholder at the model-bound prompt split; I did not run a live Docker heartbeat in this read-only review.

## Solution Assessment

Yes. The best fix is a narrow runtime repair that separates user prompt text from transcript-only markers and applies workspace guidance only to built-in/internal heartbeat prompts, rather than adding a new config mode.
