## Summary

Keep open: current main and v2026.6.8 still do not satisfy the documented heartbeat prompt contract. Source inspection shows configured heartbeat text is still passed through unconditional workspace-hint mutation, related duplicate evidence was closed back to this issue, and the only linked open fix PR is still unmerged and conflicting.

## What This Changes

The issue asks OpenClaw to honor user-configured heartbeat prompts instead of appending or substituting internal heartbeat prompt text.

## Best Possible Solution

Land a narrow heartbeat fix that sends configured heartbeat prompts as the model-visible user message, keeps the heartbeat poll marker transcript-only, preserves workspace guidance for built-in/internal prompts, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, at source level. Current main resolves configured heartbeat text but still mutates it with the workspace hint, and the prompt/transcript split can surface the heartbeat poll marker; I did not run a live Docker heartbeat in this read-only review.

## Solution Assessment

Partly. Guarding the workspace hint for user-authored prompts is necessary, but the complete fix should also prove the transcript placeholder remains storage-only and the configured prompt is model-visible.
