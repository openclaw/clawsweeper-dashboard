## Summary

The regression remains present on current main: configured heartbeat text is resolved and then passed through a helper that appends the workspace-path instruction whenever it mentions `HEARTBEAT.md`. The issue should remain open because its linked closing PR is still open and currently conflicting, while all earlier OpenClaw fix attempts were closed unmerged.

## What This Changes

Honor user-configured heartbeat prompts without appending OpenClaw’s workspace-path instruction.

## Best Possible Solution

Rebase or replace the linked fix with a focused current-main patch that preserves user-authored heartbeat prompts while applying workspace-path guidance only to OpenClaw-owned default or generated prompts, with tests for custom, default, event, and no-duplicate paths.

## Reproduction Assessment

Yes. Configure a heartbeat prompt containing `HEARTBEAT.md` and inspect the prompt passed to the agent; current main resolves that configured text and then appends the workspace-path sentence.

## Solution Assessment

Yes. The narrowest maintainable fix is to carry prompt ownership through `resolveHeartbeatRunPrompt` and skip the hint for user-authored text, rather than changing the configuration contract or removing guidance from internal prompts.
