## Label Justifications

- `P1`: This is a user-facing regression in core heartbeat prompt construction that can alter scheduled agent behavior for configured prompts.
- `impact:session-state`: The bug changes prompt content used during heartbeat turns, which can drift session instructions and agent behavior.

## Summary

Keep open: current main still routes user-configured heartbeat prompt text through the workspace-path hint helper, and an open linked PR already owns the direct fix path. The report is source-reproducible against current code and should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that user-configured heartbeat prompts are no longer respected because OpenClaw appends a generated HEARTBEAT.md workspace-path instruction even for custom prompt text.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for default and internally generated prompts.

## Reproduction Assessment

Yes by source inspection. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to detect user-configured heartbeat prompt sources and skip only the generated workspace-path hint for those prompts, while preserving default/internal prompt behavior.
