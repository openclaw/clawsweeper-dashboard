## Summary

Keep open. Current main still resolves configured heartbeat prompts and unconditionally passes the resulting text through the workspace-path hint helper, which conflicts with the documented verbatim custom-prompt contract; https://github.com/openclaw/openclaw/pull/40756 remains the open closing fix candidate.

## What This Changes

User reports that custom heartbeat prompts are mutated by the automatic workspace HEARTBEAT.md path hint instead of being honored verbatim.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while retaining the workspace-path hint for default and internally generated prompts.

## Reproduction Assessment

Yes. By source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text and still calls appendHeartbeatWorkspacePathHint on it.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base heartbeat prompt came from user config and skip the hint helper only for that case, while preserving the existing default-prompt hint behavior.
