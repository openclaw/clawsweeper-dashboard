## Summary

Keep open: current main and v2026.6.10 still route configured heartbeat prompts through the workspace-path hint path despite the docs saying custom heartbeat prompts are sent verbatim, and the only linked closing PR is still open and conflicting.

## What This Changes

The issue reports a regression where user-configured heartbeat prompts are mutated or replaced instead of being respected as the heartbeat user message.

## Best Possible Solution

Keep this as the canonical tracker and land a narrow heartbeat-runtime fix that preserves configured prompt text as the model-visible prompt while keeping transcript-only heartbeat markers internal.

## Reproduction Assessment

Yes. Source inspection shows a configured heartbeat prompt mentioning HEARTBEAT.md reaches resolveHeartbeatRunPrompt and is passed through the unconditional workspace hint helper; reporter trajectory evidence also shows the heartbeat transcript marker can become model-visible.

## Solution Assessment

No, current main is not the solved state. The narrow maintainable fix should separate user-authored heartbeat prompt text from generated heartbeat guidance and prove the configured prompt, not the transcript marker, reaches the model-visible body.
