## Summary

Keep open: current main and the latest release still append the workspace HEARTBEAT.md path hint after resolving a custom heartbeat prompt, despite docs saying custom heartbeat prompts are sent verbatim. A linked fix PR exists at https://github.com/BingqingLyu/openclaw/pull/228, but it is open, unmerged, and currently conflicting, so this issue remains necessary as the tracker.

## What This Changes

The issue reports a regression where user-configured heartbeat prompts are mutated by an appended workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land a narrow heartbeat-runner fix that preserves user-authored heartbeat prompt text and only adds the workspace path hint for built-in/default or generated prompt paths, with focused tests for both custom and default behavior.

## Reproduction Assessment

Yes from source inspection: a configured heartbeat prompt can flow through `resolveHeartbeatPrompt`, then `resolveHeartbeatRunPrompt` still appends the workspace path hint whenever the text mentions HEARTBEAT.md. I did not run a live heartbeat scenario in this read-only review.

## Solution Assessment

Yes, the best fix appears narrow: distinguish user-authored heartbeat prompts from default/generated heartbeat prompts before appending the path hint. The linked PR points in that direction, but it needs a clean, mergeable target-repo implementation before this issue can close.
