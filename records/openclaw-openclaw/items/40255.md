## Summary

Keep open. Current main still mutates a resolved custom heartbeat prompt by appending the workspace HEARTBEAT.md hint based only on prompt text, which conflicts with the documented verbatim custom-prompt contract. An open implementation PR, #40756, still uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks OpenClaw to respect user-configured heartbeat prompts without unconditionally appending the workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land a focused heartbeat-runner fix that carries prompt-origin information through resolution, skips the workspace-path hint for non-empty user-authored default and per-agent heartbeat prompts, preserves the hint for built-in/internal prompts, and adds regression coverage for default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that custom text, then `resolveHeartbeatRunPrompt` appends the workspace hint because the helper only inspects prompt contents.

## Solution Assessment

Yes, the narrow maintainable solution is a prompt-origin guard around hint injection while preserving the built-in/default prompt behavior. Because #40756 is already an open implementation candidate with closing syntax, this issue should stay open until that PR is reviewed, landed, or replaced.
