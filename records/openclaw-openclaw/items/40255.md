## Summary

Keep open. Current main still resolves user-configured heartbeat prompts and then unconditionally passes the resulting base prompt through the workspace-path hint helper, which conflicts with the documented custom-prompt verbatim contract; #40756 remains open and unmerged with closing syntax for this issue.

## What This Changes

The report asks OpenClaw to stop appending the workspace HEARTBEAT.md path hint to custom heartbeat prompts configured via agent defaults or per-agent heartbeat settings.

## Best Possible Solution

Land or replace #40756 with a focused guard that skips workspace-path hint injection for non-empty configured heartbeat prompts while preserving the hint for internal/default prompts, with regression coverage for default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. By source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that configured text and still calls appendHeartbeatWorkspacePathHint on it.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base prompt came from user config and skip the hint helper only for that case; #40756 is the current candidate, but this issue should remain open until that PR lands or is replaced.
