## Summary

Keep open: current `main` and the latest `v2026.5.22` release still append the generated `HEARTBEAT.md` workspace-path hint after resolving user-configured heartbeat prompt text, despite the docs saying custom heartbeat prompts are sent verbatim. The upstream fix PRs are closed, and the remaining open reference is fork-local source context rather than a mergeable upstream path.

## What This Changes

The issue asks OpenClaw to stop appending generated `HEARTBEAT.md` workspace-path guidance to user-configured heartbeat prompts.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts, with focused regression coverage for both global and per-agent configured prompts.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to carry whether the base prompt is user-configured and skip only the generated workspace-path hint for those prompts while preserving default, exec, cron, task, and commitment behavior.
