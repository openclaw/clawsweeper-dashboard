## Summary

Keep open. Current main still appends the workspace `HEARTBEAT.md` hint after resolving configured heartbeat prompts, which conflicts with the documented verbatim custom-prompt contract; the open fix PR #40756 uses closing syntax for this issue and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue reports a regression where user-configured heartbeat prompts are mutated by an internal workspace `HEARTBEAT.md` path hint.

## Best Possible Solution

Land a focused heartbeat-runner fix that carries prompt-origin or custom-prompt detection into run-prompt construction, skips the workspace-path hint for non-empty default and per-agent user prompts, preserves internal/default prompt behavior, and adds regression coverage for both default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that custom text and then appends the workspace hint because the helper only inspects prompt contents.

## Solution Assessment

Yes. The narrow maintainable solution is a prompt-origin guard around hint injection; #40756 is the current open implementation candidate, but the issue should stay open until that PR is landed or replaced.
