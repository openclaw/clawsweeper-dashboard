## Summary

Keep open. Current main still resolves configured heartbeat prompts and then unconditionally runs the resulting prompt through the workspace-path hint helper, which conflicts with the documented verbatim custom-prompt contract; #40756 remains open and unmerged as the closing fix candidate.

## What This Changes

Honor custom heartbeat prompts by not appending the workspace HEARTBEAT.md path hint to prompts supplied through heartbeat config.

## Best Possible Solution

Land or replace #40756 with a focused guard that skips workspace-path hint injection for non-empty configured heartbeat prompts while preserving the hint for default/internal prompts and adding regression coverage.

## Reproduction Assessment

Yes. By source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text and still calls appendHeartbeatWorkspacePathHint on it.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base heartbeat prompt came from user config and skip the hint helper only for that case; the issue should remain open until #40756 lands or is replaced.
