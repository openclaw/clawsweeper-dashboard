## Summary

Keep open. Current main still appends the workspace HEARTBEAT.md path hint after resolving configured heartbeat prompts, despite docs saying custom heartbeat prompts are sent verbatim; #40756 is open and unmerged with closing syntax, so it should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to preserve user-configured heartbeat prompts without appending the workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land or replace a focused heartbeat-runner fix that skips workspace-path hint injection for non-empty configured prompts while preserving the hint for built-in/internal prompts, with regression coverage for default and per-agent custom prompts.

## Reproduction Assessment

Yes. Source inspection on current main shows a custom agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt that mentions HEARTBEAT.md flows into basePrompt and then receives the appended workspace hint.

## Solution Assessment

Yes. The narrow maintainable fix is a custom-prompt/origin guard around hint injection; #40756 is the current open candidate, but this issue should stay open until that PR lands or is replaced.
