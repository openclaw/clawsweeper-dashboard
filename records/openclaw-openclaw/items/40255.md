## Label Justifications

- `P1`: This is a regression in core heartbeat prompt construction that can alter agent session behavior for users with custom heartbeat prompts.
- `impact:session-state`: The issue changes the prompt content used for heartbeat runs, which can drift agent behavior and session instructions.

## Summary

Keep open: current main still mutates custom heartbeat prompts that mention `HEARTBEAT.md`, and an open PR with closing syntax already owns the direct fix path. The issue is valid, source-reproducible, and should remain open until the linked fix lands or is replaced.

## What This Changes

The issue reports that user-configured heartbeat prompts are no longer preserved verbatim because the workspace `HEARTBEAT.md` path hint is appended unconditionally.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for default and internally generated prompts.

## Reproduction Assessment

Yes. By source inspection, configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main still routes that text through the workspace-path hint helper.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to detect user-configured heartbeat prompt sources and skip only the generated workspace-path hint for those prompts.
