## Summary

Keep open. Current main still resolves configured heartbeat prompts and then unconditionally runs them through the workspace-path hint helper, which conflicts with the documented verbatim custom-prompt contract; https://github.com/openclaw/openclaw/pull/40756 remains the open closing fix candidate.

## What This Changes

The issue reports that user-configured heartbeat prompts are mutated by an added workspace HEARTBEAT.md path hint instead of being preserved verbatim.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated prompts.

## Reproduction Assessment

Yes. By source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text and still passes it through appendHeartbeatWorkspacePathHint.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base heartbeat prompt came from user config and skip the hint helper only for that case, while preserving default/internal prompt hint behavior.
