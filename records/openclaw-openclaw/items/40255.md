## Summary

Keep open: current main still mutates configured heartbeat prompts that mention HEARTBEAT.md, the latest release has the same unconditional hint path, and the newer duplicate evidence was already closed back to this canonical tracker.

## What This Changes

The issue requests that user-configured heartbeat prompts remain respected and unmodified instead of receiving generated workspace guidance or being replaced by the heartbeat poll marker.

## Best Possible Solution

Land or refresh a narrow heartbeat fix that preserves user-authored heartbeat prompts verbatim, keeps the heartbeat poll marker transcript-only, retains workspace guidance for built-in/internal prompts, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, at source level. Configure a heartbeat.prompt containing HEARTBEAT.md and current main resolves it as the base prompt, then still passes it through the unconditional workspace-path hint helper; I did not run a live heartbeat in this read-only review.

## Solution Assessment

No, current main is not the best solved state. The narrow maintainable fix is to separate user-authored prompt text from generated heartbeat guidance and keep the transcript marker storage-only.
