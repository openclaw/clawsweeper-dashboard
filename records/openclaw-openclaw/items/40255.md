## Summary

Keep open: current main and the latest stable release still apply the workspace-path hint to any heartbeat prompt mentioning HEARTBEAT.md, despite the docs saying configured heartbeat prompts are sent verbatim. The newer duplicate report was closed back to this issue, and the remaining fix path is still unmerged.

## What This Changes

The issue asks OpenClaw to stop mutating user-configured heartbeat prompts with the workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land a narrow heartbeat-runner fix that preserves user-configured prompt bodies verbatim, keeps the workspace-path hint only for built-in/internal heartbeat prompts, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, at source level. Configure a heartbeat prompt containing `HEARTBEAT.md`; current main resolves that custom prompt, then unconditionally runs it through `appendHeartbeatWorkspacePathHint`, which appends the extra workspace-path instruction.

## Solution Assessment

Yes, the best fix is narrow and matches the documented contract: distinguish user-configured prompt bodies from built-in/internal prompts before adding the path hint. The existing linked PR has the right target area but needs refresh, replacement, or deliberate closure because it is currently conflicting and unmerged.
