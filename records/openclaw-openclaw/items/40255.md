## Summary

Keep open: current main and v2026.5.20 still pass user-configured heartbeat prompts through the generated HEARTBEAT.md workspace-path hint, despite the docs saying custom heartbeat prompts are sent verbatim. The target-repo fix PR was closed, so this should remain the canonical tracker and is now a narrow replacement-fix candidate.

## What This Changes

The issue reports that custom heartbeat prompts are modified by appended HEARTBEAT.md workspace-path guidance instead of being sent verbatim.

## Best Possible Solution

Land a current-main fix that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated exec/cron heartbeat prompts, with regression coverage for both config locations.

## Reproduction Assessment

Yes, by source inspection rather than an executed test. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still sends it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes. The narrow maintainable fix is to detect user-configured heartbeat prompt sources and skip only the generated workspace-path hint for those prompts, while preserving default, exec, cron, task, and response-tool heartbeat behavior.
