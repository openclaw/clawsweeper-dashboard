## Summary

Keep open: current main and the latest stable release still violate the documented custom heartbeat prompt contract, and the newly linked cross-version evidence strengthens this issue as the canonical tracker rather than making it closeable.

## What This Changes

The issue asks OpenClaw to honor user-configured heartbeat prompts instead of mutating or replacing them during heartbeat runs.

## Best Possible Solution

Keep this issue as the canonical tracker and land a narrow fix that preserves configured heartbeat prompts as the model-visible prompt while keeping `[OpenClaw heartbeat poll]` transcript-only.

## Reproduction Assessment

Yes. Source inspection shows configured heartbeat prompts enter the runner, are still mutated by the workspace hint/directive path, and can be replaced at the model boundary by the heartbeat transcript placeholder; the issue also has cross-version trajectory evidence.

## Solution Assessment

Yes. The maintainable fix is narrow: preserve user-authored heartbeat prompts as the model-visible prompt, skip generated path guidance for custom prompts, and keep the heartbeat poll placeholder only for transcript/display purposes.
