## Summary

Keep open: current main and v2026.6.6 still route configured heartbeat prompts through the unguarded workspace-path hint appender, despite the docs saying custom heartbeat prompt bodies are sent verbatim. The related target-repo PRs are closed, and the only open linked cross-repo PR is unmerged and conflicting, so this issue remains the canonical tracker.

## What This Changes

The issue asks OpenClaw to honor user-configured heartbeat prompts without appending workspace HEARTBEAT.md path guidance to user-authored prompt text.

## Best Possible Solution

Land a narrow heartbeat-runner fix that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts, with focused coverage for both paths.

## Reproduction Assessment

Yes from source inspection: configure `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text and then passes it through the unguarded workspace-path hint helper. I did not run a live heartbeat scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to distinguish user-authored heartbeat prompts from default/generated prompts before appending the path hint. The linked PR direction matches that shape, but it needs a clean target-repo implementation before this issue can close.
