## Summary

Keep open: current main still violates the documented verbatim contract for custom heartbeat prompts, and an open closing PR already owns the fix path.

## What This Changes

The issue reports that configured heartbeat prompts mentioning HEARTBEAT.md are mutated by an automatically appended workspace-path hint instead of being sent verbatim.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a focused guard that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts.

## Reproduction Assessment

Yes. By source inspection, configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text and still passes it through appendHeartbeatWorkspacePathHint.

## Solution Assessment

Yes. The narrow maintainable fix is to track whether the base heartbeat prompt came from user config and skip the hint helper only for that case, while preserving default/internal prompt guidance.
