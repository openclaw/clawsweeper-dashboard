## Label Justifications

- `P1`: This is a user-facing regression in core heartbeat prompt construction that can alter scheduled agent behavior for configured prompts.
- `impact:session-state`: The bug changes prompt content used during heartbeat turns, which can drift session instructions and agent behavior.

## Summary

Keep open: current main and the latest release still append generated HEARTBEAT.md workspace-path guidance to configured heartbeat prompts, despite docs promising custom prompts are sent verbatim. The linked closing PR remains open and conflicting, so the issue should stay as the canonical tracker until that fix is landed or replaced.

## What This Changes

The issue requests that user-configured heartbeat prompts be honored verbatim without unconditional generated HEARTBEAT.md workspace-path guidance.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/40756 with a current-main guard that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for default and internally generated prompts.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to detect user-configured heartbeat prompt sources and skip only the generated workspace-path hint for those prompts, while preserving default/internal prompt behavior.
