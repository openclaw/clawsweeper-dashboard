## Summary

Keep this PR open: current main still lacks a quota-exhaustion circuit breaker, and the discussion includes useful real incident context, but the patch is not merge-ready because it misses key Gemini quota shapes, bypasses existing lane-suspension behavior, lacks docs and real behavior proof, and carries generated artifacts.

## What This Changes

The PR adds a `quota_exhausted` failover reason, routes it through auth-profile disabled backoff, emits an `agent:provider_tripped` internal hook, updates related tests, and includes unrelated generated protocol and e2e-script changes.

## Best Possible Solution

Land or replace this with a rebased quota circuit breaker that classifies the real Gemini and periodic quota shapes, preserves disabled-lane suspension semantics, documents the new reason and hook contract, removes generated artifacts, and includes redacted real-run proof.

## Reproduction Assessment

Yes at source level: current main maps HTTP 429 and periodic usage-window text to `rate_limit` and has no `quota_exhausted` disabled lane. I did not reproduce it against a live exhausted provider account.

## Solution Assessment

No, not as currently patched. The direction is plausible, but the patch misses the named Gemini quota payloads, bypasses lane suspension for quota disables, leaves docs/proof gaps, and includes unrelated generated artifacts.
