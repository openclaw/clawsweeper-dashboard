## Summary

This PR addresses a real quota-exhaustion problem and has useful follow-up incident context, but it is not merge-ready: the latest branch still drops the current session/lane suspension guardrail for the new reason, exposes an undocumented hook/failure contract, carries unrelated generated artifacts, has failing CI, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `quota_exhausted` failure reason, routes it through long auth-profile disabled backoff, emits an `agent:provider_tripped` internal hook, updates focused tests, and includes generated protocol/script artifacts.

## Best Possible Solution

Land a rebased, narrowed quota circuit breaker that preserves session suspension for the new reason, documents the quota and hook contracts, drops unrelated generated artifacts, includes real after-fix proof, and leaves account-scoped sibling suppression to https://github.com/openclaw/openclaw/issues/80451.

## Reproduction Assessment

Yes at source level: current main still classifies periodic quota text and quota-bearing HTTP 429s as `rate_limit`, and a real incident comment describes the resulting cascade. I did not establish a live exhausted-provider run for this branch's after-fix behavior.

## Solution Assessment

No, not as currently patched. The classification and long-backoff direction is plausible, but the PR needs session-suspension integration, docs, generated-artifact cleanup, green checks, and real behavior proof before it is the maintainable fix.
