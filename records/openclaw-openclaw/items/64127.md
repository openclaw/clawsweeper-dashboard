## Summary

Keep this PR open, but it is not merge-ready: the narrowed quota circuit-breaker idea is useful, yet the current branch still introduces an unaligned shared failure reason, bypasses the existing lane-suspension path for terminal quota-disabled providers, and only supplies local test output rather than after-patch real exhausted-provider proof.

The PR classifies periodic quota exhaustion as `quota_exhausted`, routes that reason into long auth-profile disabled backoff, and emits an `agent:provider_tripped` internal hook.

## Best Possible Solution

## Solution Assessment
