## What This Changes

The branch adds `quota_exhausted` failover classification/backoff, an `agent:provider_tripped` hook, focused tests, generated protocol artifacts, and a small e2e script change.

## Best Possible Solution

Land or replace this with a rebased, narrowed quota circuit breaker that preserves lane suspension, documents the new reason/hook contract, removes generated artifacts, passes checks, and includes redacted after-fix behavior proof.

## Solution Assessment

No, not as currently patched. The direction is plausible, but the branch needs to preserve `suspend_lanes`, update public docs for the new contract, remove generated artifacts, rebase, pass checks, and include real behavior proof.
