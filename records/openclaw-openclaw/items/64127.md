## Summary

Keep this PR open: the quota-circuit-breaker direction is useful and not implemented on current main, but the current patch still needs maintainer/product review, real behavior proof, rebase cleanup, and a few correctness fixes before merge.

## What This Changes

The PR adds a new `quota_exhausted` failover/auth-profile reason, long disabled backoff, model-fallback probe behavior, an `agent:provider_tripped` hook, focused tests, and generated protocol/script artifacts.

## Best Possible Solution

Land a rebased, narrowed quota-circuit-breaker that preserves current session-suspension behavior for the new reason, documents the hook, removes generated artifacts, and keeps account-scoped sibling suppression in the linked follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main still classifies periodic quota-limit text as `rate_limit`, while the PR changes that behavior to `quota_exhausted`; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently patched. The classification/backoff direction is plausible, but the patch needs rebase cleanup, session-suspension preservation, hook docs, artifact removal, and real behavior proof before it is the maintainable solution.
