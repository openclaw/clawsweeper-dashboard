## Summary

Keep open. Current main still creates the active backup archive as an adjacent random `.tmp` file and only cleans it during normal JavaScript unwinding; the issue’s workaround and the KeepMyClaw link are useful mitigations, but they do not fix timeout or cooperative termination cleanup in the core backup tool.

## What This Changes

The issue asks `openclaw backup create` to clean up large temporary archives when timeout or process termination interrupts a backup.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive path, clean it on graceful termination or abort during the write window, preserve no-overwrite final publication, and document hard-kill limits.

## Reproduction Assessment

Yes, by source-backed reproduction: run `openclaw backup create` long enough to create the adjacent temp archive, then terminate it before normal unwinding. I did not run a live timeout because this review had to keep the checkout read-only and artifact-free.

## Solution Assessment

Yes. A narrow backup-owned graceful-interruption cleanup path is the best maintainable fix; broad startup sweeps or deleting arbitrary `*.tmp` files would risk unrelated user files and still miss the active archive lifecycle.

## Repair Work Prompt

Fix the timeout/interruption cleanup bug for `openclaw backup create`. Current main creates `<output>.<uuid>.tmp` in `src/infra/backup-create.ts` and removes it only during normal `finally` cleanup; add backup-owned cooperative interruption cleanup for SIGINT/SIGTERM and/or an explicit abort path while the archive write is active. Track and remove only the exact active temp archive and staged manifest temp directory, preserve no-overwrite publication semantics, preserve normal exit codes such as 130/143, and remove scoped signal handlers after backup completion. Do not add broad startup sweeps or delete arbitrary `*.tmp` files. Add focused regression coverage that simulates interruption during archive creation and asserts temp archive cleanup plus listener cleanup. Update `docs/cli/backup.md` if needed to document graceful cleanup and unavoidable SIGKILL/crash limits; add a changelog entry if the fix is user-visible. Check adjacent backup work in https://github.com/openclaw/openclaw/issues/42273 and https://github.com/openclaw/openclaw/pull/67472, but keep this PR limited to active temp archive lifecycle cleanup.
