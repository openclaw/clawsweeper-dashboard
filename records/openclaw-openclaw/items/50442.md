## Repair Work Prompt

Fix the backup CLI temp-file leak reported in https://github.com/openclaw/openclaw/issues/50442. On current main, `src/infra/backup-create.ts` creates the active archive as `<finalArchive>.<uuid>.tmp`, so killed or timed-out `openclaw backup create` runs can leave large `.tmp` files in the output/current directory. Keep the final archive publish atomic and non-overwriting, but move the active partial archive into a lifecycle-owned staging temp directory or equivalent managed temp root, clean it on normal errors and handled SIGINT/SIGTERM where feasible, and do not add a new config option or broad backup redesign. Add focused regression coverage proving the output directory is not used for the active partial archive and existing success/failure behavior still works. Treat https://github.com/openclaw/openclaw/pull/80598 as useful prior art, but do not edit CHANGELOG.md for this normal fix.
