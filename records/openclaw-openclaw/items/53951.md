## Summary

Keep open: the Defuddle feature may still be useful, but current main moved HTML extraction to plugin-owned webContentExtractors, while this PR adds Defuddle in the obsolete utility/root-dependency path and lacks external real-behavior proof.

## What This Changes

The branch adds Defuddle as the primary web_fetch HTML extractor with Readability fallback, plus dependency, docs, config copy, tests, and a comparison script.

## Best Possible Solution

Implement Defuddle through the webContentExtractors plugin contract, with plugin-local dependency ownership metadata, ordering/fallback coverage, and maintainer-approved default behavior.

## Reproduction Assessment

Not applicable; this is a feature/default-behavior PR. Static source inspection is high-confidence that current main dispatches HTML extraction through manifest-declared webContentExtractors, not the utility helper this branch changes.

## Solution Assessment

No. The maintainable path is a Defuddle webContentExtractors plugin or a reviewed extension of web-readability, with dependency ownership and default ordering handled at the plugin boundary.
