## Summary

Keep this PR open, but it is not mergeable against current main as written: the active web_fetch HTML path is now plugin-backed, while the PR adds Defuddle to the older utility path and adds a new parser dependency outside the current extractor ownership model.

## What This Changes

The PR adds Defuddle as the primary web_fetch HTML extractor with Readability fallback, plus related config copy, docs, tests, a comparison script, and dependency changes.

## Best Possible Solution

Implement Defuddle through the webContentExtractors plugin contract, keep parser dependencies and ownership metadata with the owning extractor plugin, and only make it primary after maintainer approval of the default behavior.

## Reproduction Assessment

Not applicable as a feature PR. The blocking issue is high-confidence from static inspection: current main dispatches HTML extraction through the plugin runtime, while this PR adds Defuddle to a utility module that is no longer the active readable-extraction owner.

## Solution Assessment

No. The narrow maintainable path is a Defuddle webContentExtractors plugin or a reviewed extension of the existing web-readability plugin, with dependency ownership metadata and ordering/fallback tests.
