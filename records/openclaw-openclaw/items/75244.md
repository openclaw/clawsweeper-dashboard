## Summary

Keep open. Current main still enforces native plugin tools through `tool.parameters` only and drops malformed tools with diagnostics; the exact Remnic TypeBox value that allegedly fails this object check is not available in the checkout, so this needs maintainer follow-up rather than cleanup closure.

## What This Changes

The issue reports that TypeBox-backed `openclaw-remnic` memory tools are rejected as missing `parameters` and silently removed from the agent tool list each turn.

## Best Possible Solution

Capture the failing third-party tool shape with safe diagnostics or plugin inspection, then either fix the documented `parameters` path generically or intentionally expand the native schema contract with docs and contract tests.

## Reproduction Assessment

Unclear. Source inspection proves missing-`parameters` and `inputSchema`-only native tools are dropped, but the reported Remnic TypeBox `parameters` object is outside the checkout and not available from npm, so there is no high-confidence current-main reproduction for the exact failure.

## Solution Assessment

Unclear. Better diagnostics are a narrow safe step, but accepting `inputSchema` or adding TypeBox-specific validation should wait until maintainers inspect the failing runtime shape and decide whether to expand the native plugin contract.
