## Summary

Keep open. Current main still has the same native plugin-tool validation boundary: tools are dropped when `parameters` is not a non-array object, with no captured value in the diagnostic and no native `inputSchema` alias, while the exact Remnic TypeBox runtime shape remains outside this checkout.

## What This Changes

The issue asks OpenClaw to stop rejecting/dropping memory plugin tools whose schemas come from TypeBox `parameters`, improve diagnostics, and possibly support `inputSchema` as an alias.

## Best Possible Solution

Capture the failing third-party tool shape with safe diagnostics, then either fix the documented `parameters` path generically or intentionally expand the native plugin schema contract with docs and focused contract tests.

## Reproduction Assessment

Unclear. Source inspection gives a high-confidence path for `inputSchema`-only tools being dropped, but the reported Remnic TypeBox `parameters` object is not present in this checkout and was not reproduced against current main.

## Solution Assessment

Unclear. Better diagnostics are the narrow safe first step, but accepting `inputSchema` or special-casing TypeBox should wait for the captured failing runtime shape and a maintainer decision on the native plugin contract.
