## Summary

Keep open: current main still drops plugin tools whose `parameters` value is missing or not an object, and the exact Remnic TypeBox value that allegedly fails that check is external to this checkout. The newer linked gist adds useful context, but source inspection does not prove the bundled `memory-core` wrapper is currently failing the same way, so this needs a captured failing schema shape and maintainer schema-contract direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop rejecting TypeBox-backed plugin memory tools as malformed and to improve diagnostics or schema compatibility for plugin tool registration.

## Best Possible Solution

Capture the failing Remnic schema/tool object with redacted diagnostics or a fixture, then either fix the documented `parameters` path generically or explicitly expand native Plugin SDK schema aliases with docs and contract tests.

## Reproduction Assessment

No high-confidence exact reproduction for the Remnic TypeBox failure is available. Source inspection does prove that current main drops native plugin tools when `parameters` is missing or non-object, but the failing third-party schema shape has not been captured in this checkout.

## Solution Assessment

Unclear. Richer malformed-schema diagnostics are a narrow safe step, but accepting `inputSchema` or TypeBox-specific exceptions would change the native Plugin SDK contract and should wait for a captured failing shape and maintainer direction.
