## Summary

Keep open. Current main still requires native plugin tools to expose an object-valued `parameters` field, still drops malformed tools with diagnostics, and the exact external Remnic TypeBox shape is not in this checkout, so no conservative cleanup close reason applies.

## What This Changes

The issue asks OpenClaw to stop dropping plugin tools whose registered schemas fail native `parameters` validation and to improve diagnostics around TypeBox or `inputSchema` schema shapes.

## Best Possible Solution

Capture the failing third-party tool shape with safe diagnostics, then either normalize a clearly supported schema alias at plugin registration or document the strict `parameters` contract with focused contract tests.

## Reproduction Assessment

Unclear. Current main has a high-confidence source/test path for `inputSchema`-only tools being skipped, but the exact external TypeBox `parameters` failure requires the Remnic plugin shape or equivalent captured runtime data.

## Solution Assessment

Unclear. Diagnostics are a safe first step, but choosing between `inputSchema` aliasing, TypeBox-specific handling, or stricter docs is an SDK-contract decision until the failing shape is captured.
