## Summary

Keep open. Current main still matches the reported failure surface: native plugin tools are accepted into the registry and later dropped unless `tool.parameters` is a non-array object, while `inputSchema` remains a separate descriptor/MCP boundary concept rather than a native plugin alias.

## What This Changes

The issue asks OpenClaw to stop dropping Remnic memory tools whose TypeBox `parameters` schema is reported as malformed, and to improve diagnostics or schema compatibility handling.

## Best Possible Solution

Capture the failing third-party tool shape with safe diagnostics, then either fix the documented `parameters` path generically or intentionally expand the native plugin schema contract with docs and focused contract tests.

## Reproduction Assessment

Unclear. Source inspection gives a high-confidence path for `inputSchema`-only tools being dropped, but the exact Remnic TypeBox `parameters` value that allegedly fails `isRecord` is not present in this checkout.

## Solution Assessment

Unclear. Better diagnostics are a narrow safe step, but accepting `inputSchema` or special-casing TypeBox should wait until maintainers inspect the failing runtime shape and decide the native plugin contract.
