## Summary

Keep open. Current main still enforces native plugin tools as object-valued `parameters`, drops malformed tools into plugin diagnostics, and does not treat `inputSchema` as a native plugin alias; the exact Remnic TypeBox `parameters` runtime value is outside this checkout, so the reported named TypeBox schema failure still needs capture before choosing a contract change.

## What This Changes

The issue asks OpenClaw to stop dropping Remnic memory plugin tools whose schemas are authored with TypeBox and to improve diagnostics or schema compatibility for plugin tool registration.

## Best Possible Solution

Capture safe diagnostics for the failing external tool shape, then either fix the documented TypeBox `parameters` path generically or deliberately expand the native plugin schema contract with docs and focused contract tests.

## Reproduction Assessment

Unclear. Source inspection gives a high-confidence path for `inputSchema`-only native plugin tools being dropped, but the reported Remnic TypeBox `parameters` object is external and current source would accept any non-array object-valued `parameters`.

## Solution Assessment

Unclear. Better diagnostics are the narrow safe first step, while accepting `inputSchema` or adding TypeBox-specific handling should wait for the captured runtime shape and maintainer plugin-contract direction.
