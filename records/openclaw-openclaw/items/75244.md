## Summary

Keep open. Current main still enforces the documented native plugin-tool contract around `parameters`, and the external TypeBox failure is plausible but not proven from code in this repository; closing would lose a concrete report that likely needs maintainer API-contract direction.

## What This Changes

The issue asks OpenClaw to stop dropping third-party plugin tools whose TypeBox schema under `parameters` is rejected as missing, improve diagnostics, and possibly support `inputSchema` as an alias.

## Best Possible Solution

Keep one explicit native plugin-tool schema boundary: capture the failing third-party tool object, improve diagnostics enough to identify its runtime shape, then either keep `parameters` as the only native contract or add a normalized `inputSchema` alias with SDK docs and contract tests.

## Reproduction Assessment

Unclear. The issue gives concrete external reproduction steps with `openclaw-remnic`, but that plugin object is outside this checkout; current main only gives a high-confidence path for reproducing `inputSchema`-only native tools being skipped.

## Solution Assessment

Unclear. Better malformed-tool diagnostics are a narrow first step, but accepting `inputSchema` or adding TypeBox-specific handling is not clearly the best fix until maintainers inspect the captured failing object and decide whether to expand the native plugin contract.
