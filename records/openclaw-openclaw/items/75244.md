## Summary

Keep open. Current main still rejects native plugin tools unless `tool.parameters` is an object, has no `inputSchema` fallback, and the docs/tests still preserve `parameters` as the native plugin contract, so the reported failure is not implemented, obsolete, or superseded.

## What This Changes

Investigate whether native plugin tools using TypeBox or `inputSchema` schemas are incorrectly rejected as malformed and silently dropped.

## Best Possible Solution

Capture the failing third-party tool shape with safe diagnostics, then either document the existing `parameters`-only contract more clearly or normalize any approved schema aliases at plugin registration with focused contract tests.

Unclear. Current main has a high-confidence path for `inputSchema`-only native tools being skipped, but the exact Remnic TypeBox `parameters` object that reportedly fails `isRecord` is not present in this checkout.

## Solution Assessment

Unclear. Diagnostics are a safe first step, but accepting `inputSchema` or TypeBox-specific handling is not clearly the narrowest maintainable fix until maintainers inspect the failing runtime shape and choose the SDK contract.
