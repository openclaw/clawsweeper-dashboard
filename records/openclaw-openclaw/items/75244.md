## Summary

Current main still has the same native plugin-tool validator behavior: tools are skipped when `parameters` is missing or not an object, and there is no `inputSchema` alias or richer schema-shape diagnostic. The specific Remnic TypeBox object that allegedly fails `isRecord` is external to this checkout, so the item should stay open for a captured failing schema shape and maintainer API-contract direction rather than cleanup closure.

## What This Changes

The issue reports that native plugin tools using TypeBox object schemas are rejected as malformed because their `parameters` value is treated as missing or non-object, causing memory tools to be dropped.

## Best Possible Solution

Capture the failing Remnic tool shape with safe diagnostics or a fixture, then either fix the documented `parameters` path generically or intentionally expand the Plugin SDK schema contract with docs and contract tests.

## Reproduction Assessment

Unclear. Source inspection proves missing-`parameters` and `inputSchema`-only native tools are dropped on current main, but the exact Remnic TypeBox `parameters` object that allegedly fails `isRecord` is outside this checkout and was not reproduced.

## Solution Assessment

Unclear. Better diagnostics are a narrow safe step, but accepting `inputSchema` or adding TypeBox-specific exceptions should wait until maintainers inspect the failing runtime shape and decide whether to expand the native plugin contract.
