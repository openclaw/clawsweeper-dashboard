## Summary

Keep open. Current main still enforces the native plugin-tool contract that returned tools must have object-valued `parameters`, while the reported third-party TypeBox failure remains plausible but is not directly reproducible from this checkout.

## What This Changes

The issue asks OpenClaw to stop silently dropping TypeBox-backed native plugin memory tools, or at least emit diagnostics that reveal the malformed schema shape.

## Best Possible Solution

Capture or safely log the failing third-party tool shape, then choose a documented plugin-boundary path: clearer `parameters` diagnostics only, or normalized `inputSchema` alias support with SDK docs and focused contract tests.

## Reproduction Assessment

Unclear. The issue gives concrete external Remnic steps, but that plugin object is not in this checkout; current main only gives a high-confidence path for reproducing `inputSchema`-only native tools being skipped.

## Solution Assessment

Unclear. Better diagnostics are a safe first step, but accepting `inputSchema` or adding TypeBox-specific handling is not clearly the narrowest maintainable fix until the actual runtime schema value is inspected.
