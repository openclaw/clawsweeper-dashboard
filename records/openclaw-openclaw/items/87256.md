## What This Changes

The issue reports high idle CPU in an idle macOS OpenClaw gateway, suspected to come from Chokidar `awaitWriteFinish` polling in the config reload and skills watchers.

## Best Possible Solution

Keep the issue open and profile current-main macOS gateway idle filesystem activity first; then fix the confirmed watcher path with narrower scope/throttling or one shared asynchronous stability primitive that preserves hot reload semantics without event-loop stat churn.

## Reproduction Assessment

## Solution Assessment

The best fix should start from measured current-main macOS polling pressure and preserve stability through asynchronous, lifecycle-owned code.
