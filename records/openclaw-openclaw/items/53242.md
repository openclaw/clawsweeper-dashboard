## Summary

Keep open. Current main still applies a low character cap before the `chat.history` byte budget, and the open companion PR at https://github.com/openclaw/openclaw/pull/53243 references this issue but has not merged and still needs maintainer review against the moved shared projection path.

## What This Changes

Raise or realign `chat.history` text truncation so long assistant responses that fit payload budgets are returned instead of being cut with `...(truncated)...`.

## Best Possible Solution

Keep this issue open until the companion PR is rebased, replaced, or closed; the durable fix should live in `src/gateway/chat-display-projection.ts`, budget truncation against serialized message size, and align gateway config, docs, and the Control UI request cap.
