Keep open. Current main still truncates chat-history text before the serialized byte budget, and the paired closing PR is still open, conflicting, and not merged.

## What This Changes

The issue reports that `chat.history` truncates long WebChat assistant history text before the configured history and per-message byte budgets require truncation.

## Best Possible Solution

Implement the fix in `src/gateway/chat-display-projection.ts`, budget visible prefixes against serialized message size, align gateway default/config and Control UI cap policy, and add focused gateway/UI regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A 50k ASCII assistant text can fit under the 128 KiB per-message byte guard, but current main applies the 8,000 default projection cap or the Control UI's 4,000 RPC cap before that guard runs.

## Solution Assessment

No, the original one-line 64k bump is not the best complete fix on current main. The durable solution should update the shared projection helper and align the Control UI/config/default policy with serialized-byte-aware coverage.
