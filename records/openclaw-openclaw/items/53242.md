## Summary

Keep open. Current main still truncates chat-history text by a low character cap before the 128 KiB serialized byte guard, and the paired closing PR at https://github.com/openclaw/openclaw/pull/53243 is still open, conflicting, and not merged.

## What This Changes

The issue asks for `chat.history` to preserve long assistant text when it fits the configured history and single-message byte budgets.

## Best Possible Solution

Implement the fix in `src/gateway/chat-display-projection.ts`, budget visible prefixes against serialized message size, align the gateway default/config/UI cap policy, and cover long ASCII plus byte-heavy text with focused gateway/UI tests.

## Reproduction Assessment

Yes, source-reproducible. A 50k ASCII assistant text can fit under the 128 KiB per-message byte guard, but current main applies the 8,000 default cap, or the Control UI's 4,000 RPC cap, before the byte guard runs.

## Solution Assessment

No, the old one-line constant bump is not the best complete fix. The durable solution should update the shared projection helper and measure truncation against the same serialized-byte budget used by the hard placeholder path.
