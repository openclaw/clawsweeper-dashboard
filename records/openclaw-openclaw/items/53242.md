Keep open. Current main still truncates WebChat history text by character projection before the serialized byte budget, and https://github.com/openclaw/openclaw/pull/53243 still has closing syntax for this issue but remains open rather than merged.

## What This Changes

The issue requests that `chat.history` preserve long assistant text that fits the serialized history budgets instead of truncating at the projection or Control UI character caps.

## Best Possible Solution

Review, rebase, replace, or close the linked fix PR on current main, with a solution that aligns shared projection, serialized-byte budgeting, and Control UI history policy.

## Reproduction Assessment

Yes, source-reproducible. A 50k ASCII assistant message fits under the 128 KiB serialized single-message cap, but current main can truncate it first at the 8,000 projection default or the Control UI's 4,000 request cap.

## Solution Assessment

No, the original one-line cap bump is not the best complete fix. The durable path should update the shared projection helper, account for serialized message bytes, and settle the Control UI/config cap policy.
