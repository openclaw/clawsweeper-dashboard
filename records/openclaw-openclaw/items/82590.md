## Summary

Keep open: the reported supervised-node hang is plausible from current source, but this PR changes the shared node-host pause handler too broadly and lacks after-fix real behavior proof for the systemd flow.

## What This Changes

Adds `PAIRING_REQUIRED` to the node-host reconnect-pause exit codes and updates the runner credential test to expect an exit/restart message.

## Best Possible Solution

Make the `PAIRING_REQUIRED` exit supervisor-aware, preserve foreground no-exit behavior, add focused regression coverage for both paths, and include redacted real systemd proof before merge.

## Reproduction Assessment

No real setup reproduction was provided or run. Source inspection shows current main can pause on non-retry-hinted `PAIRING_REQUIRED` without exiting, while managed systemd units would restart only if the process exits.

## Solution Assessment

No. Adding `PAIRING_REQUIRED` to the global exit set is too broad because the same handler is used by foreground `openclaw node run`; a supervisor-aware exit path is the narrower maintainable fix.
