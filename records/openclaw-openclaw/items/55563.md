Keep open: the browser CLI regression is still present on current main, and the WSL2 troubleshooting guide still shows the now-broken post-subcommand `--browser-profile` syntax. The issue asks OpenClaw to restore `openclaw browser tabs --browser-profile <name>` compatibility and handle a gateway systemd restart loop caused by an nvm-managed Node service path.

Restore backward-compatible browser profile parsing for the documented `browser tabs --browser-profile <name>` flow, update the WSL2 guide as needed, and leave durable nvm/system-Node policy decisions to the canonical gateway runtime thread.

## Solution Assessment
