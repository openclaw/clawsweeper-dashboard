## Summary

Keep open. Current main and the latest release still show the browser CLI/docs mismatch for `--browser-profile`, and the gateway half has only partial nvm/service-audit handling without proof that the exact WSL2 systemd cycling case is fixed.

## What This Changes

The issue asks OpenClaw to restore or document browser `--browser-profile` CLI syntax compatibility and make `doctor --fix` repair or clearly diagnose gateway systemd+nvm startup cycling.

## Best Possible Solution

Resolve the browser syntax mismatch deliberately, then add focused doctor/service-audit proof for a stopped or failed WSL2 systemd unit whose `ExecStart` points at an unstable nvm Node path.

## Reproduction Assessment

Yes for the browser regression: current source and docs still show a parent-only option while the WSL2 guide uses after-subcommand placement. Unclear for the gateway regression because current source shows mitigations but not high-confidence proof of the exact WSL2+nvm cycling case.

## Solution Assessment

Unclear. The browser fix should be either compatibility on affected leaf commands with regression tests or an intentional docs/migration update; the gateway half needs focused systemd+nvm proof before treating current mitigations as sufficient.
