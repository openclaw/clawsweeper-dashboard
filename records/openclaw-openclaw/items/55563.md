The gateway cycling half overlaps the open version-manager runtime policy thread, so the safe repair is the browser CLI compatibility regression first.

## What This Changes

The issue asks to restore documented browser CLI profile-flag syntax and address a gateway restart loop after `openclaw doctor --fix` with an NVM-backed systemd unit.

## Best Possible Solution

Restore local backward-compatible browser profile parsing for documented browser subcommand forms with focused CLI coverage, and keep the durable version-manager Gateway runtime policy scoped to https://github.com/openclaw/openclaw/issues/60612 unless a distinct WSL2 service-loop repro remains afterward.
