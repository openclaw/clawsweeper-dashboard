## Summary

Keep open for normal PR handling. The implementation is narrow and matches the existing ClawHub installer/managed-skill loader contracts, but the docs should still add the supported single-skill `update <slug> --global` form before this is fully polished.

## What This Changes

The PR adds a `--global` target for `openclaw skills install` and `openclaw skills update`, plus docs, tests, and a changelog entry.

## Best Possible Solution

Land after the small docs reference fix and exact-head CI completion, keeping the default workspace target unchanged and reusing the existing `CONFIG_DIR/skills` managed-skill root.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is high-confidence: current main has no `--global` install/update option while shared managed skills are already loaded from `CONFIG_DIR/skills`. The PR gives a concrete command-level verification path for install, update-all, single-slug update, and `--agent` conflicts.

## Solution Assessment

Yes, aside from the docs gap. An explicit `--global` flag is the narrowest maintainable solution because it preserves workspace defaults, rejects ambiguous `--agent` combinations, and reuses the existing ClawHub installer target contract.
