## Summary

Keep open for normal PR handling. The implementation is narrow and fits the existing ClawHub installer and managed-skill loader contracts, but a few user-facing CLI/docs references still need cleanup before the feature is polished.

## What This Changes

The PR adds a `--global` target for `openclaw skills install` and `openclaw skills update`, updates skill docs/changelog, and extends CLI unit coverage.

## Best Possible Solution

Land with accurate CLI help/docs for both workspace and shared managed targets, while preserving the default workspace behavior and existing ClawHub installer contract.

## Reproduction Assessment

Not applicable as a bug reproduction; the feature gap is high-confidence because current main lacks `--global` while shared managed skills already load from `CONFIG_DIR/skills`. The PR provides a concrete command-level verification path for install, update-all, single-slug update, and `--agent` conflicts.

## Solution Assessment

Yes, aside from the remaining copy/docs gaps. An explicit `--global` flag is the narrowest maintainable solution because it keeps workspace defaults unchanged, rejects ambiguous `--agent` combinations, and reuses the existing installer target contract.
