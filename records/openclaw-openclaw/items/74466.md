## What This Changes

The PR adds `--global` to `openclaw skills install` and `openclaw skills update`, routes that target through `CONFIG_DIR/skills`, updates docs/changelog, and adds CLI command tests for global install/update plus `--agent` conflicts.

## Best Possible Solution

Land an up-to-date equivalent after maintainer review, preserving workspace installs by default and using `CONFIG_DIR/skills` only for explicit `--global` operations.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new CLI capability. Source inspection gives high confidence in the feature gap because current main has no global install/update target while shared managed skills are already loadable.

## Solution Assessment

Yes. The explicit `--global` flag is the narrowest maintainable shape here because it preserves existing workspace behavior, reuses the existing ClawHub install/update contract, and rejects ambiguous `--agent` combinations.
