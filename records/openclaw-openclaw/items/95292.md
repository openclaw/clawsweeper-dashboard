## Summary

Keep open: current main and v2026.6.9 still provide build-id service-worker registration, cache trimming, strict protocol rejection, and manual protocol-mismatch guidance, but no token-safe in-browser stale-asset recovery path. ## What This Changes

The issue reports cloud-hosted Control UI protocol mismatch lockouts after gateway updates because stale service-worker assets, token auth, and current recovery steps do not provide a reliable non-technical fix.

## Best Possible Solution

Keep strict gateway protocol negotiation and add a token-safe browser recovery path for stale Control UI assets, backed by stale-service-worker upgrade proof.

## Solution Assessment
