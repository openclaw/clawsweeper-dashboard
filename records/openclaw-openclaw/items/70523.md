## What This Changes

The PR adds memory-core logic and tests to remove inline daily Light/REM dreaming blocks after request-scoped narrative fallback, returns fallback status from narrative generation, exports the contamination helper, and adds a MiniMax FAQ entry.

## Best Possible Solution

Land a memory-core-only fix that suppresses raw inline Light/REM blocks after successful request-scoped fallback in both heartbeat and cron detached paths, preserves phase output when fallback writing fails, and keeps unrelated docs separate.

## Reproduction Assessment

Yes, at source level. Current main writes raw inline Light/REM phase blocks before narrative fallback, and PR head only removes them after an awaited fallback result, leaving cron-triggered detached fallback unhandled; I did not execute tests because this review is read-only.

## Solution Assessment
