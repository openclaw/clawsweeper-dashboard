## Summary

Keep open: this PR targets a real memory-core bug that still exists on current main, but PR head still misses request-scoped fallback cleanup for cron/detached narratives and lacks required after-fix real behavior proof.

## What This Changes

The PR changes memory-core dreaming markdown, narrative, phase logic, and tests to remove raw inline Light/REM daily-memory blocks after request-scoped fallback, and also adds a MiniMax FAQ entry.

## Best Possible Solution

Land a memory-core-only fix that suppresses raw inline Light/REM blocks after successful request-scoped fallback in both awaited heartbeat and detached cron paths, preserves inline output when fallback writing fails, and keeps unrelated docs separate.

## Reproduction Assessment

Yes, at source level. Current main writes raw Light/REM phase blocks before narrative fallback, and PR head leaves the cron detached path without a way to observe fallback success and remove the inline block.

## Solution Assessment

No, not yet. The awaited heartbeat cleanup is a reasonable direction, but the detached cron path must carry a fallback-success cleanup result or callback and gain regression coverage before this is the narrow maintainable fix.
