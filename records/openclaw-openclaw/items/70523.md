## What This Changes

The PR changes memory-core dreaming fallback handling to suppress raw inline daily memory dreaming dumps, adds regression coverage, and includes a MiniMax FAQ docs hunk.

## Best Possible Solution

Land a memory-core-only fix that suppresses raw inline light/REM blocks after successful request-scoped fallback in both awaited heartbeat and detached cron paths, preserves phase output when fallback writing fails, and leaves MiniMax docs to a separate change.

## Reproduction Assessment

Yes, by source inspection. Inline light/REM storage writes raw metadata to `memory/YYYY-MM-DD.md` before request-scoped narrative fallback, and current main has no cleanup path for either awaited or detached fallback results.

## Solution Assessment

No. The awaited-path suppression is a reasonable narrow direction, but the proposed patch is incomplete until detached cron fallback can report success and remove the already-written inline block.
