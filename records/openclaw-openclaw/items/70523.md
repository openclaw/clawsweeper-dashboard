## Summary

Keep open. Current main fixed the default daily-file pollution path by changing Dreaming storage to separate, but explicit inline mode is still supported and this PR is the active narrow attempt to handle that path. The current patch still misses cron-detached fallback cleanup and lacks required after-fix real behavior proof, so it is not ready to merge or close as implemented.

## What This Changes

The branch adds fallback reporting to dreaming narrative generation, lets the daily dreaming writer remove inline managed blocks, suppresses inline Light/REM raw metadata after request-scoped fallback, exports a contamination helper, adds regression tests, and adds a MiniMax FAQ entry.

## Best Possible Solution

Land a narrow memory-core fix that suppresses raw inline Light/REM blocks after successful request-scoped fallback in both awaited heartbeat and detached cron paths, preserves inline output when fallback writing fails, and keeps unrelated docs separate.

## Reproduction Assessment

Yes, at source level. Current main still has a supported explicit inline mode that writes raw Light/REM phase bodies, and PR head removes those bodies only after an awaited fallback result, leaving cron-detached fallback uncovered.

## Solution Assessment

No, not yet. The awaited fallback cleanup is a reasonable direction, but the patch needs a detached-path result/callback cleanup and contributor-supplied real behavior proof before it is the maintainable fix.
