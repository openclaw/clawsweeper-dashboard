## Summary

Keep open: current main still has the raw inline dreaming fallback bug, and this PR is useful but incomplete because the cleanup only covers the awaited non-detached path while cron-triggered dreaming detaches narratives and discards the fallback result.

## What This Changes

The PR changes memory-core dreaming markdown/narrative code to report request-scoped fallback status, suppress raw inline daily light/REM blocks after fallback, export contamination detection, add regression tests, and add a MiniMax FAQ section.

## Best Possible Solution

Land a rebased memory-core-only fix that suppresses raw inline light/REM blocks after successful request-scoped fallback in both heartbeat and cron-detached paths, preserves reports and formatting, adds focused coverage and changelog, and splits unrelated docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main writes raw inline light/REM metadata before narrative fallback, and cron-triggered dreaming uses a detached helper that cannot currently feed fallback success back to the inline writer.

## Solution Assessment

No. The proposed direction is narrow for heartbeat-triggered fallback, but it is not complete until detached cron fallback can also remove the raw inline block after a successful fallback write.
