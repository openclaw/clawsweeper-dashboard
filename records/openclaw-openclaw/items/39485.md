## Summary

Keep open: current main still lacks the requested implicit Discord same-name routing, the linked issue remains open, and this PR still needs maintainer policy acceptance, code fixes, and real behavior proof before merge.

## What This Changes

The branch adds a Discord-only fallback in `resolveAgentRoute` that maps unmatched non-default `accountId` values to same-named agents and adds resolver tests.

## Best Possible Solution

Have maintainers decide the Discord routing contract; if accepted, land a guarded fallback or explicit account-binding contract with distinct diagnostics, no phantom agents, docs/changelog alignment, tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, at source level. Current main sends Discord `accountId` through `resolveAgentRoute` and falls through to the default agent after explicit binding tiers; the PR-introduced phantom-agent edge is also visible because `pickFirstExistingAgentId` echoes requested IDs when no agents are configured.

## Solution Assessment

No. The resolver is a plausible boundary if maintainers accept the convention, but this patch should require an existing configured agent, use a distinct implicit match reason, update release-facing docs, and include real behavior proof before merge.

## Review Findings
