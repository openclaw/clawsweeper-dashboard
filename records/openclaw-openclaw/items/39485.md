## Summary

Keep open: current main has not implemented the same-name Discord account routing, and this PR is still an open implementation candidate, but it has blocking review issues. The patch changes permission-sensitive routing, lacks real Discord runtime proof, and can route to phantom agents when no agents are configured.

## What This Changes

The PR adds a Discord-only fallback in `resolveAgentRoute` that maps unmatched non-default `accountId` values to same-named agents and adds two resolver tests.

## Best Possible Solution

Have maintainers accept or reject the same-name account convention; if accepted, land a guarded implicit mapping with distinct diagnostics, docs/tests, and real Discord proof.

## Reproduction Assessment

Yes, source-level. Current main falls to the default route when no explicit binding matches, and the PR-introduced empty-agent regression follows from `pickFirstExistingAgentId` returning sanitized input when the agent lookup is empty.

## Solution Assessment

No. The shared resolver is a plausible boundary if maintainers accept the convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
