## Summary

Keep open: current main still lacks the requested same-name Discord account routing, and this PR is the linked implementation candidate, but it is not mergeable as-is because it changes permission-sensitive routing, has a concrete phantom-agent bug, and lacks real Discord runtime proof.

## What This Changes

The PR adds a Discord-only fallback that maps an unmatched non-default `accountId` to a same-named agent ID and adds resolver tests for the positive path and default-account guard.

## Best Possible Solution

Have maintainers decide whether same-name Discord account routing should become an OpenClaw contract; if accepted, land it with an existence guard, distinct diagnostics, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, source-level. Current main falls through to the default route after explicit binding tiers, and the PR-introduced phantom-agent path follows from `pickFirstExistingAgentId` echoing a sanitized account ID when no agents are configured.

## Solution Assessment

No. The shared resolver is a plausible boundary only if maintainers accept the same-name Discord convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
