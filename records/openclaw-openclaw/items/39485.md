## Summary

Keep open: this external PR is still the active implementation candidate for an open Discord routing issue, but the current diff introduces a permission-sensitive phantom-agent route and still lacks real Discord runtime proof.

## What This Changes

The PR adds a Discord-only fallback that maps an unmatched non-default `accountId` to a same-named agent and adds resolver tests for the match and default-account guard.

## Best Possible Solution

Accept implicit Discord account-to-agent routing only with maintainer approval, a configured-agent guard, distinct implicit-match diagnostics, docs/tests, and redacted real Discord proof.

## Reproduction Assessment

Yes, at source level. Current main falls through to the default route after explicit binding tiers, and the PR-introduced phantom-agent path follows from `pickFirstExistingAgentId` echoing a sanitized account ID when no agents are configured.

## Solution Assessment

No. The shared resolver is a plausible boundary if maintainers accept the convention, but this patch needs a configured-agent existence guard, a distinct implicit match reason, docs alignment, and real Discord proof before merge.

## Review Findings
