## Summary

Keep open: this PR targets an open Discord routing issue, but the patch changes a security-sensitive routing contract, introduces a phantom-agent regression, blurs diagnostics, and still lacks contributor-provided live Discord proof.

## What This Changes

The PR adds a Discord-only implicit accountId-to-agentId fallback in the shared route resolver and adds two resolver tests.

## Best Possible Solution

Have maintainers decide the Discord routing contract; if accepted, land a guarded implicit account mapping with distinct diagnostics, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, source-level. Current main passes Discord accountId into resolveAgentRoute and falls through to the default route when explicit binding tiers miss; the PR-introduced phantom-agent path follows from pickFirstExistingAgentId returning sanitized input when no agents exist.

## Solution Assessment

No. The resolver is a plausible boundary if maintainers accept the convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
