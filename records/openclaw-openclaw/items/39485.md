## Summary

Keep this PR open: current main still uses explicit binding tiers then default routing, so the branch is not obsolete, but the proposed implicit Discord routing has a blocking phantom-agent edge case and lacks real Discord proof.

## What This Changes

The PR adds Discord-only fallback routing from an unmatched non-default `accountId` to a same-named agent ID and adds resolver tests for the positive and default-account cases.

## Best Possible Solution

If maintainers accept the same-name routing convention, update the branch with an existing-agent guard, a distinct implicit match reason, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, at source level. Current main reaches the default fallback after explicit binding tiers, and the PR's phantom-agent edge follows from `pickFirstExistingAgentId` echoing a sanitized requested ID when no agent list exists.

## Solution Assessment

No. The shared resolver is a plausible boundary if maintainers accept the convention, but this patch needs an existing-agent guard, a separate implicit match reason, docs/tests, and real Discord proof before merge.
