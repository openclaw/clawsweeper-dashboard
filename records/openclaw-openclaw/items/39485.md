## What This Changes

The branch adds Discord-only implicit routing from a non-default accountId to a same-named agent plus two resolver regression tests.

## Best Possible Solution

Keep the canonical Discord routing issue open and implement an approved replacement that guards against phantom agents, uses clear diagnostics, documents the routing contract, and proves the behavior in a redacted Discord setup.

## Reproduction Assessment

Yes, at source level: current main passes Discord account IDs into resolveAgentRoute and falls through to the default after explicit binding tiers. The PR-introduced defect also follows from pickFirstExistingAgentId echoing sanitized IDs when no agents are configured.

## Solution Assessment

No. Same-name Discord fallback may be useful, but this branch needs a configured-agent guard, distinct match diagnostics, conflict resolution, docs/tests, real Discord proof, and maintainer product/security approval before merge.

Full review comments:

- **[P1] Verify an agent exists before implicit Discord routing:** `src/routing/resolve-route.ts:179-184`
  - body: The new fallback accepts the result of pickFirstExistingAgentId when it normalizes back to accountId.   - confidence: 0.92

  - confidence: 0.79
