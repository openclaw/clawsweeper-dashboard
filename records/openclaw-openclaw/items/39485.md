## Summary

Keep open: current main and the latest release still use explicit binding/default routing, and this PR is the open implementation for the linked Discord account-routing report. The patch is not merge-ready because it can route unmatched Discord accounts to phantom agents, reuses the explicit-binding match label for an implicit fallback, and lacks contributor real Discord proof.

## What This Changes

The PR adds a Discord-only implicit fallback that maps an unmatched non-default `accountId` to a same-named agent and adds resolver tests for the positive and default-account cases.

## Best Possible Solution

Land this PR or a replacement only after it requires an existing configured agent, reports implicit matches distinctly, updates docs/tests, and includes redacted real Discord proof; explicit bindings remain the supported workaround until then.

## Reproduction Assessment

Yes, at source level. Current main falls through to the default route after explicit binding tiers, and the PR-introduced phantom-agent path follows from `pickFirstExistingAgentId` echoing a sanitized account ID when no agents are configured.

## Solution Assessment

No. The shared resolver is a plausible boundary if maintainers accept the convention, but this patch needs a configured-agent existence guard, a distinct implicit match reason, docs alignment, and real Discord proof before merge.
