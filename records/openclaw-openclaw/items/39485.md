## What This Changes

The PR adds Discord-only implicit account-id-to-agent-id routing after explicit binding tiers, plus two resolver unit tests.

## Best Possible Solution

Land a revised or replacement Discord routing change only after the implicit fallback contract is approved, limited to existing configured agents, distinguishable in diagnostics, documented/tested, and proven with redacted Discord runtime evidence.

## Reproduction Assessment

Yes, at source level. Current main falls through to the default route after explicit binding tiers, and the PR-introduced phantom-agent path follows from pickFirstExistingAgentId echoing a sanitized account ID when no agents are configured.

## Solution Assessment

No. Same-name Discord fallback may be a reasonable direction, but this implementation needs a configured-agent existence guard, distinct diagnostics, docs alignment, real Discord proof, and maintainer product/security approval before merge.
