## Summary

Keep open: current main does not implement the proposed implicit Discord same-name routing, but this PR is not mergeable as written because it changes a permission-sensitive routing contract, lacks real Discord proof, and can route to phantom agents.

## What This Changes

The branch adds a Discord-only implicit accountId-to-agentId fallback in the shared resolver and two resolver tests.

## Best Possible Solution

Have maintainers either keep requiring explicit account bindings or land a guarded implicit Discord mapping with a distinct match reason, docs/tests, and redacted live proof.

## Reproduction Assessment

Yes, source-level: current main passes Discord accountId into resolveAgentRoute and falls through to the default route when explicit binding tiers miss. I did not run live Discord because this review is read-only.

## Solution Assessment

No, not as written. The resolver is a plausible boundary if maintainers accept the convention, but the patch must require a configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
