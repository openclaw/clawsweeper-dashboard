## Summary

## What This Changes

The PR adds a Discord-only implicit fallback that maps unmatched non-default account IDs to same-named agents and adds two resolver tests.

## Best Possible Solution

Land a revised version only after the implicit fallback contract is approved, the match is limited to existing configured agents, diagnostics distinguish implicit matches from explicit bindings, docs/tests are aligned, conflicts are resolved, and redacted Discord runtime proof is provided.

## Reproduction Assessment

Yes, at source level: current main falls through to the default route after explicit binding tiers, and the PR-introduced phantom-agent path follows from pickFirstExistingAgentId echoing a sanitized account ID when no agents are configured. No live Discord repro was run in this read-only review.

## Solution Assessment

No. Same-name Discord fallback may be useful, but this implementation needs a configured-agent guard, distinct diagnostics, docs alignment, conflict resolution, real Discord proof, and maintainer product/security approval before merge.
