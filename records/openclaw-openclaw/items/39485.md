## What This Changes

This PR adds a Discord-only fallback that maps unmatched non-default account IDs to same-named agents and adds resolver unit tests for that behavior.

## Best Possible Solution

Land a revised same-name Discord fallback only after maintainers approve the routing contract, the code verifies the target is a configured agent, diagnostics distinguish implicit matches, docs/tests are updated, conflicts are resolved, and redacted real Discord proof is provided.

Yes, at source level. Current main passes Discord account IDs into `resolveAgentRoute` and falls back to the default after explicit binding tiers; the PR-introduced empty-agent-list defect follows from `pickFirstExistingAgentId` echoing sanitized IDs when no agents are configured.

## Solution Assessment

No. Same-name Discord fallback may be useful, but this implementation needs a configured-agent guard, distinct match diagnostics, docs/tests, conflict resolution, real Discord proof, and maintainer product/security approval before merge.
