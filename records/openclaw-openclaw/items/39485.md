## Summary

Keep open: this is a focused external PR for an open Discord routing issue, but the patch changes permission-sensitive route selection, lacks after-fix Discord runtime proof, and introduces a phantom-agent routing path when no agents are configured.

## What This Changes

The PR adds a Discord-only implicit fallback that maps an unmatched non-default account ID to a same-named agent ID, with two resolver tests.

## Best Possible Solution

Have maintainers decide the Discord same-name routing contract; if accepted, land a guarded implementation with distinct diagnostics, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, source-level. Current main falls through to the default route when no explicit binding matches, and the PR-introduced phantom-agent path follows from pickFirstExistingAgentId echoing a sanitized accountId when no agents are configured; I did not run live Discord or tests in this read-only pass.

## Solution Assessment

No. The shared resolver is a plausible boundary only if maintainers accept the same-name Discord convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
