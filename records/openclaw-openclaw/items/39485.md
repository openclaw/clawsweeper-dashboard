## Summary

Keep this PR open for maintainer review: current main still lacks the proposed implicit Discord account-name routing, but the patch changes a routing permission boundary and has a source-backed phantom-agent regression before merge.

## What This Changes

The PR adds a Discord-only fallback in core route resolution that maps a non-default accountId to a same-named agent when no binding matches, plus resolver tests.

## Best Possible Solution

Land a maintainer-approved routing rule that only maps Discord account IDs to configured agents, reports a distinct implicit match reason, covers configured/default/empty-agent cases, and updates changelog/docs.

## Reproduction Assessment

Yes. The linked issue gives concrete multi-account Discord steps, and current main shows the resolver falls from explicit binding tiers to the default route when no binding matches.

## Solution Assessment

No. The central resolver is a plausible boundary if maintainers accept the convention, but this patch needs a configured-agent guard, a distinct match label, and user-facing release notes before it is the narrow maintainable fix.
