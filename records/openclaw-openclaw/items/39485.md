## Summary

Keep this PR open for maintainer handling: current main still lacks the requested Discord same-name account routing, the paired issue is still open, and this patch has a blocking permission-sensitive routing bug plus missing real Discord proof.

## What This Changes

The PR adds a Discord-only implicit fallback from unmatched non-default account IDs to same-named agent IDs, with two resolver regression tests.

## Best Possible Solution

Have maintainers decide the Discord same-name routing contract; if accepted, land a guarded implementation with distinct diagnostics, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, source-level. Current main falls through to the default route when no binding matches, and the PR-introduced phantom-agent path follows from pickFirstExistingAgentId echoing sanitized accountId when no agents are configured; I did not run live Discord.

## Solution Assessment

No. The resolver is a plausible boundary if maintainers accept the convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
