## Summary

This PR should stay open: current main still falls through from explicit binding tiers to the default route for Discord account traffic, while the proposed implicit same-name fallback has a permission-sensitive correctness blocker and lacks live Discord proof.

## What This Changes

The PR adds a Discord-only implicit route that maps an unmatched non-default `accountId` to a same-named agent and adds resolver tests for the positive and default-account cases.

## Best Possible Solution

Have maintainers decide the Discord routing contract; if accepted, land a guarded implicit account mapping with distinct diagnostics, docs/tests, and redacted live Discord proof.

## Reproduction Assessment

Yes, source-level. Current main passes Discord accountId into `resolveAgentRoute` and falls through to the default route when explicit binding tiers miss; the PR-introduced no-agent phantom route follows from `pickFirstExistingAgentId` returning sanitized input when no agents exist.

## Solution Assessment

No. The resolver is a plausible boundary if maintainers accept the convention, but this patch should require an existing configured agent, distinguish implicit matches from explicit bindings, document the contract, and include real Discord proof.
