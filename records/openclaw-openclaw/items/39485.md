## Summary

Keep open: this PR targets a real source-visible Discord routing gap, but the patch introduces a phantom-agent routing regression when no agents are configured and still lacks required real Discord behavior proof.

## What This Changes

The PR adds a Discord-only same-name `accountId` to `agentId` fallback in `resolveAgentRoute` plus regression tests for the positive match and `default` account guard.

## Best Possible Solution

Update this PR or a replacement to implement a maintainer-approved Discord same-name fallback with a configured-agent guard, distinct diagnostics, focused tests, docs alignment, and redacted live Discord proof.

## Reproduction Assessment

Yes, at source level. Current main falls through to the default route without an explicit binding, and the PR regression follows from `pickFirstExistingAgentId` returning a sanitized account id when the agent list is empty; I did not run live Discord in this read-only pass.

## Solution Assessment

No, not as written. A same-name fallback may be acceptable, but it needs an explicit configured-agent check, a distinct match reason, policy/docs alignment, and real Discord behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require a configured agent before implicit routing:** `src/routing/resolve-route.ts:181-184`
  - body: With `cfg.agents` absent or empty, `pickFirstExistingAgentId` returns a sanitized copy of `accountId`. The new equality guard therefore succeeds and routes Discord traffic to a non-existent agent instead of preserving the prior default-agent fallback, changing session and policy routing for empty-agent configs.
  - confidence: 0.93
- **[P3] Use a distinct match reason for implicit account routes:** `src/routing/resolve-route.ts:831`
  - body: This path is not an explicit binding, but logs and route metadata report `binding.account`. Callers and operators cannot tell whether a configured binding matched or the new naming fallback fired, which weakens diagnostics for security-sensitive routing decisions.
  - confidence: 0.82
