Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve a non-VBS startup fallback:** `src/daemon/schtasks.ts:93`
  - body: This makes the Startup-folder fallback install a `.vbs` login item regardless of `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER`. Hosts with Windows Script Host or `.vbs` execution disabled would lose the previous `.cmd` login path, so keep a tested `.cmd` fallback/opt-out or record an explicit maintainer compatibility decision with coverage.
  - confidence: 0.86
- **[P1] Remove the actual Startup-folder login item:** `docs/install/uninstall.md:117`
  - body: The new manual cleanup command removes `$env:USERPROFILE\.openclaw\gateway.vbs`, but the fallback login item is written under the Windows Startup folder as a task-name file such as `OpenClaw Gateway.vbs`. Users following this after a fallback install would leave the auto-start entry behind, so point cleanup at the Startup-folder `.vbs` and legacy `.cmd` entries.
  - confidence: 0.94
- **[P3] Name the Startup-folder item accurately:** `docs/gateway/index.md:282`
  - body: This says current installs write `gateway.vbs`, but the PR writes a Startup-folder entry named from the task, such as `OpenClaw Gateway.vbs`, which runs the state-dir `gateway.cmd`. Distinguish the login item from the state-dir task script so status and cleanup docs do not point readers at the wrong artifact.
  - confidence: 0.88
