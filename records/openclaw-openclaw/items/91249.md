## Summary

Keep open. The PR addresses a real Windows Startup-folder fallback bug that is still present on current main and v2026.6.6, but it is not merge-ready because real behavior proof is mock-only, the manual uninstall docs remove the wrong VBS path, and the new VBS default still needs maintainer acceptance or mitigation for compatibility-sensitive hosts.

## What This Changes

The PR changes the Windows Startup-folder fallback to write a hidden VBS launcher by default, updates focused fallback tests, and revises gateway/uninstall docs.

## Best Possible Solution

Land a narrow VBS-default Startup fallback only after live Windows proof, accurate Startup-folder cleanup docs, and maintainer acceptance or fallback handling for WSH-blocked hosts.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.6 still render the default Startup-folder fallback with `start "" /min cmd.exe /d /c`, and the PR updates that exact path. I did not establish a live Windows desktop login reproduction in this review.

## Solution Assessment

No, not yet. The runtime change is a plausible narrow fix, but the best merge shape still needs real Windows proof, correct cleanup docs, and a maintainer decision on VBS-only compatibility versus a fallback or opt-out.

  - confidence: 0.86
