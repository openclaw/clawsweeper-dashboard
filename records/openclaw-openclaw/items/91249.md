## Summary

## What This Changes

The PR changes the Windows Startup-folder fallback to write a hidden VBS launcher by default, updates fallback lifecycle tests, and revises gateway/uninstall docs.

## Best Possible Solution

Land a narrow VBS-default Startup fallback only after live Windows proof, accurate cleanup docs for the actual Startup-folder entry, and explicit maintainer acceptance or fallback handling for WSH-blocked hosts.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.6 still render the default Startup-folder fallback as `start "" /min cmd.exe /d /c`, and the current default fallback test asserts that launcher. I did not establish a live Windows desktop reproduction in this review.

## Solution Assessment

No, not yet. The runtime path is a plausible narrow fix, but the best merge shape needs live Windows proof, accurate cleanup docs, and a maintainer decision on VBS-only compatibility versus an explicit fallback.
