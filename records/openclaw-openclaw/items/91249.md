- **[P1] Preserve a non-VBS startup fallback:** `src/daemon/schtasks.ts:93`
  - body: This makes the Startup-folder fallback unconditionally install a `.vbs` login item and always write the VBS launcher. On Windows hosts where WSH or `.vbs` execution is blocked, the previous `.cmd` fallback could still start the gateway but the new default can leave the login item nonfunctional; please keep a tested `.cmd` fallback or record an explicit maintainer acceptance path with upgrade coverage.

Users following this doc after a fallback install would leave the auto-start entry behind, so point the command at the Startup-folder `.vbs` and legacy `.cmd` entries.
  - confidence: 0.93

  - body: This says current installs write `gateway.vbs`, but the PR writes a Startup-folder entry named from the task, such as `OpenClaw Gateway.vbs`, which then runs the state-dir `gateway.cmd`. Please distinguish the login item from the state-dir task script so status and cleanup docs do not point readers at the wrong artifact.
