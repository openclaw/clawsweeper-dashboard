## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep a tested fallback for WSH-blocked hosts:** `src/daemon/schtasks.ts:93`
  - body: By making the Startup-folder entry unconditionally `.vbs`, fallback installs now require Windows Script Host. On hosts where WSH or `.vbs` execution is disabled by policy, Task Scheduler denial no longer leaves a working `.cmd` login fallback, so keep a tested `.cmd` fallback/opt-out or record the maintainer decision before changing the shipped default.
  - confidence: 0.8
- **[P2] Remove the actual Startup-folder launcher in docs:** `docs/install/uninstall.md:117`
  - body: The manual uninstall command removes `$env:USERPROFILE\.openclaw\gateway.vbs`, but `resolveStartupEntryPath()` writes `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\<task name>.vbs`. Users following this doc will leave the login item behind; please align the default, profiled, and gateway docs with the actual Startup-folder path.
  - confidence: 0.91
