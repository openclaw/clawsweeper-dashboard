## Summary

Keep open: current main and v2026.6.9 still use the visible `.cmd` Startup-folder fallback by default, so this PR remains the linked implementation candidate, but it is not merge-ready because it lacks native Windows visual proof and introduces unresolved VBS compatibility plus docs defects.

## What This Changes

This PR changes `src/daemon/schtasks.ts` to default the Windows Startup-folder fallback to a task-name `.vbs` hidden launcher, updates startup fallback tests, and revises gateway/uninstall docs.

## Best Possible Solution

Land a rebased, narrow Startup-folder hidden-launcher fix only after native Windows proof, corrected Startup-folder docs, and either a tested non-VBS fallback or explicit maintainer acceptance of the WSH/VBS default tradeoff.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 still generate a Startup-folder `.cmd` entry using `start "" /min ...cmd.exe /d /c` unless the hidden-launcher env is truthy. I did not establish a native Windows login-session visual reproduction in this read-only review.

## Solution Assessment

No, not yet. The VBS launcher is a plausible narrow fix that reuses existing machinery, but this branch is too unconditional for WSH-blocked hosts, keeps inaccurate cleanup docs, and lacks native Windows visual proof.
