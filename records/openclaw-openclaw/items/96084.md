## What This Changes

The issue asks for `/readyz` to return NotReady when critical PVC-backed workspace or state writes fail with disk-full or unwritable-storage errors.

## Best Possible Solution

Land one gateway-owned cached storage writability probe that makes `/readyz` fail for critical unwritable state/workspace paths, keeps `/healthz` shallow, and closes this issue after the chosen PR merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: make critical workspace writes fail with ENOSPC and current main still computes `/readyz` from startup, draining, event-loop, and channel-health signals only.

## Solution Assessment

Yes. The maintainable fix belongs in readiness rather than liveness: `/readyz` should fail while critical storage is unwritable, while `/healthz` should stay live because restarting does not free a full PVC.
