## What This Changes

The issue asks for gateway `/readyz` to return NotReady when critical PVC-backed workspace or state writes fail with `ENOSPC`, while keeping `/healthz` shallow/live.

## Best Possible Solution

Land one canonical gateway-owned storage writability readiness probe, preferably the active reviewed PR or an equivalent implementation, then close or redirect overlapping candidate PRs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make critical workspace/state writes fail with `ENOSPC` and current `/readyz` still derives readiness from startup, draining, channel, and event-loop signals only.

## Solution Assessment

Yes. Readiness is the right boundary: `/readyz` should fail while critical storage is unwritable, while `/healthz` stays live so a full PVC does not create restart churn.
