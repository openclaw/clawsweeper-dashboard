## What This Changes

The PR changes gateway chat lifecycle finalization so a terminal chat final event is emitted through the existing session/run fallback when the run registry entry disappears between lookup and shift.

## Best Possible Solution

Land the small fallback only after replacing the test with a branch-accurate regression and adding redacted real gateway/UI/TUI proof; keep the linked bug open until this PR or another fix merges.

## Reproduction Assessment

No high-confidence real reproduction was established in this read-only pass. Source inspection confirms the early-return branch if `shift()` misses, but the current registry contract makes the proposed race hard to prove without a branch-specific stub or real runtime evidence.

## Solution Assessment

Mostly yes: falling back to the already resolved session/run identifiers is the narrowest maintainable implementation direction. The proposed test is not the best validation because it exercises the fallback that current main already has.
