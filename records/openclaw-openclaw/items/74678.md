## What This Changes

This PR changes gateway chat lifecycle finalization to emit a terminal chat final through session/run fallback when registry shift misses, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Keep the narrow production fallback, but correct the regression test to force `peek()` success with a `shift()` miss and add redacted real gateway chat proof before landing.

## Reproduction Assessment

Yes for a source-level path: make `registry.peek(evt.runId)` return a chat link while the following `registry.shift(evt.runId)` returns undefined, then send lifecycle `end` or `error`. Not yet as a real runtime race; the PR still needs live gateway/UI/TUI proof.

## Solution Assessment

Partly. The production fallback is the narrow maintainable direction, but the current test is not the best proof because it removes the registry entry before the handler's own `peek()`.
