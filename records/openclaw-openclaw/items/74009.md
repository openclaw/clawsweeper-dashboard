## Summary

The underlying bug is still present on current `main`, and the PR is a useful repair path, but the submitted patch needs changes before merge because it opens a denied-access key disclosure path and leaves a persistent test mock implementation behind.

## What This Changes

The PR removes the mixed `sessionKey`/`label` rejection in `sessions_send`, updates the tool description and changelog, and adds regression tests for explicit `sessionKey` and session-id precedence.

## Best Possible Solution

Land a small updated repair that preserves explicit `sessionKey` or session-id precedence while returning only the caller-provided selector, or no selector, on denied access and keeping the session tests isolated.

## Reproduction Assessment

Yes. Current source still deterministically rejects a `sessions_send` call containing both `sessionKey` and `label`, and the PR discussion includes a production reproduction on `v2026.5.7`.

## Solution Assessment

No as submitted. Preferring explicit `sessionKey` is the right narrow fix, but the patch should avoid echoing a resolved canonical key after denied access and should reset or one-shot the gateway mock in the new test.

## Review Findings

Overall confidence: 0.88

  - body: Removing the mixed-selector guard lets a session-id-shaped `sessionKey` resolve before visibility or A2A checks.   - confidence: 0.84

  - confidence: 0.9
