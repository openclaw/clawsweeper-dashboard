## Summary

Keep open. The PR targets a real `sessions_send` selector bug, but current `main` still rejects mixed selectors and the patch is not mergeable as submitted because it can expose resolved canonical session keys after denied access and its new mock implementation leaks across tests.

## What This Changes

The PR removes the `sessions_send` mixed `sessionKey`/`label` rejection guard, updates the tool description and changelog, and adds regression tests for `sessionKey`/sessionId precedence over redundant `label`/`agentId` hints.

## Best Possible Solution

Land a narrow precedence fix that treats explicit `sessionKey`/sessionId as authoritative while preserving denied-access confidentiality, isolating tests, rebasing onto current main, and keeping the changelog entry.

## Reproduction Assessment

Yes, by source inspection. Current main returns the mutual-exclusion error before resolution when `sessionKey` and `label` are both present; the PR’s denied-path regression follows from sessionId resolution before the existing visibility/A2A denial branch.

## Solution Assessment

No as submitted. Preferring explicit `sessionKey` is the right narrow direction, but the patch should avoid returning resolved canonical keys on denied sends and reset or one-shot the new gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106`
  - body: Removing this guard lets a `sessionKey` value that is really a sessionId resolve before visibility/A2A checks. On denial the existing branch returns `sessionKey: displayKey`, so callers can learn a canonical key they are not allowed to send to; echo the original selector or omit `sessionKey` on denied access.
  - confidence: 0.84
- **[P2] Reset the gateway mock between gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: The new sessionId regression installs a persistent `callGatewayMock.mockImplementation(...)`, but the local `beforeEach` only calls `mockClear()`. Later tests in this describe can inherit the custom gateway responses, so reset the mock or use one-shot implementations.
  - confidence: 0.9
