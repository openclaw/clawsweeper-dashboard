## Summary

Keep this PR open, but do not merge the current head. The linked reports and source PRs support the `sessions_send` mixed-selector bug, and current `main` still rejects `sessionKey` plus `label`, but this patch introduces a denied-access key-disclosure path and a test-isolation bug.

## What This Changes

The PR removes the `sessions_send` `sessionKey`+`label` rejection, updates the tool description and changelog, and adds regression tests for explicit `sessionKey`/sessionId precedence over redundant `label`/`agentId` hints.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit `sessionKey`/sessionId values as authoritative over redundant `label`/`agentId` hints while preserving visibility/A2A denial behavior without leaking canonical keys, with isolated regression tests and a changelog entry.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction: a `sessions_send` call with both `sessionKey` and `label` returns the mutual-exclusion error before resolution, matching #64699 and the `sessions_send` portion of #41199.

## Solution Assessment

No as submitted. Preferring explicit `sessionKey` is the right narrow direction, but the patch should first avoid echoing resolved canonical keys on denied sends and reset or one-shot the new gateway mock implementation.
