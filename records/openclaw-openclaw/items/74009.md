## Summary

Keep this PR open, but do not merge it as submitted: it fixes a live mixed-selector failure, yet it introduces a denied-access canonical-key disclosure path and a regression test mock that can leak into later cases.

## What This Changes

The PR removes `sessions_send`'s mixed `sessionKey`/`label` guard, updates the tool description and changelog, and adds regression tests for `sessionKey`/sessionId precedence over redundant `label`/`agentId` hints.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit `sessionKey`/sessionId as authoritative while preserving denied-access confidentiality, with isolated regression tests and a changelog entry.

## Reproduction Assessment

Yes, by source inspection rather than a local run. Current `main` returns the mutual-exclusion error for `sessions_send` with both `sessionKey` and `label`, and the PR’s denial-path regression follows from sessionId resolution into the existing denied branch.

## Solution Assessment

No as submitted. Preferring explicit `sessionKey` is the right narrow direction, but the patch should avoid returning resolved canonical keys on denied sends and reset or one-shot the new gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106`
  - body: Deleting this guard lets sessionId-shaped `sessionKey` inputs with redundant `label`/`agentId` resolve before visibility/A2A checks. If access is denied, the existing branch returns the resolved `displayKey`, so mixed-selector probes can learn a canonical key they cannot send to; return the original selector or omit `sessionKey` on denial.
  - confidence: 0.84

  - body: The added sessionId regression installs a persistent `callGatewayMock.mockImplementation(...)`, but the local `beforeEach` only calls `mockClear()`. Later tests in the same describe can inherit this gateway behavior, so reset the mock or use one-shot implementations.
  - confidence: 0.9
