## Summary

Keep this PR open, but do not merge it as written. The requested `sessions_send` precedence fix is still valid on current main, but the patch has a security-sensitive denial-path question and a test isolation defect that should be addressed before merge.

## What This Changes

The PR removes the early `sessions_send` rejection for mixed `sessionKey` and `label`, documents `sessionKey` precedence, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the precedence change, but make denied sends avoid revealing resolved canonical keys, isolate the new gateway mock with `mockReset()` or one-shot implementations, then validate the targeted sessions tests and changed-lane gate before merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction: a `sessions_send` payload with both `sessionKey` and `label` hits the early guard in `src/agents/tools/sessions-send-tool.ts` and returns the mutual-exclusion error before resolution.

## Solution Assessment

No as submitted. Preferring `sessionKey` is the narrow maintainable direction, but the final patch should also close the denied-resolution disclosure path and fix the test pollution introduced by the new regression case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid echoing resolved keys after denied sends:** `src/agents/tools/sessions-send-tool.ts:106`
  - body: By deleting the mixed-selector guard, a sessionId-shaped `sessionKey` with redundant `label`/`agentId` can now resolve to a canonical key before visibility/A2A enforcement. The denied branch returns `displayKey`, so a caller can learn the resolved key for a target it cannot send to; return the original input or omit `sessionKey` on denial.
  - confidence: 0.74
- **[P2] Reset the gateway mock between send-gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: The new sessionId test installs a persistent `callGatewayMock.mockImplementation(...)`, but the local `beforeEach` only calls `mockClear()`. Later tests in this describe block can inherit that implementation, so reset the mock or use one-shot implementations for the new test.
  - confidence: 0.9
