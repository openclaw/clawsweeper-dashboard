## Summary

Keep open. The current-main bug still exists and this PR targets the right behavior, but it should not merge as-is because the patch can disclose a resolved session key after access denial and its new test mock can leak into later tests.

## What This Changes

This PR removes the `sessions_send` mixed `sessionKey`/`label` guard, updates the tool description and changelog, and adds regression tests for explicit selector precedence.

## Best Possible Solution

Land an updated current-main repair that treats explicit `sessionKey` or session id input as authoritative while preserving denied-access confidentiality, current send safeguards, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main deterministically rejects a `sessions_send` call with both `sessionKey` and `label`, and a production comment reports the same failure on `v2026.5.7`.

## Solution Assessment

No as submitted. The precedence rule is the right direction, but the patch should avoid returning resolved canonical keys on denied sends and should reset or one-shot the persistent gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106-108`
  - body: Removing the mixed-selector guard lets a session-id-shaped `sessionKey` resolve before visibility or A2A checks. If access is denied, the existing branch returns the resolved `displayKey`, so callers can learn canonical keys they cannot send to; return the original selector or omit `sessionKey` on denial.
  - confidence: 0.84
- **[P2] Reset the gateway mock between send gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: The new session-id regression installs a persistent `callGatewayMock.mockImplementation(...)`, but the enclosing `beforeEach` only clears call history. Later `sessions_send gating` tests can inherit that gateway behavior, so reset the mock or use one-shot implementations.
  - confidence: 0.9
