## Summary

Keep this PR open. Current main still has the mixed-selector `sessions_send` failure, and the PR is a narrow candidate for that bug, but the submitted head has a concrete denied-access key disclosure concern, a test-isolation defect, and failing checks that should be resolved before merge.

## What This Changes

The branch removes the `sessions_send` `sessionKey`+`label` rejection, updates the tool description and changelog, and adds regression tests for `sessionKey`/sessionId precedence over redundant `label`/`agentId` hints.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit `sessionKey` values as authoritative while preserving label-only behavior, but update the denial path so unauthorized sends do not reveal a resolved canonical key, isolate the new tests with reset or one-shot mocks, and rerun the focused sessions test plus changed gate before merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction: a `sessions_send` call with both `sessionKey` and `label` reaches the early guard and returns the mutual-exclusion error before resolution.

## Solution Assessment

Unclear as submitted. Preferring `sessionKey` is the right narrow direction for the reported bug, but the patch should also prevent denied sessionId resolution from echoing canonical keys and fix the mock leakage before it is the best final solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106-109`
  - body: Removing this guard lets sessionId-shaped `sessionKey` values with redundant `label`/`agentId` resolve before visibility/A2A checks. If access is denied, the later denied branch returns the resolved `displayKey`, so mixed-selector probes can learn a canonical session key they cannot send to. Return the caller-provided identifier or omit `sessionKey` on denied access.
  - confidence: 0.74
- **[P2] Reset the gateway mock between gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: This new test installs a persistent `callGatewayMock.mockImplementation(...)`, but the `sessions_send gating` `beforeEach` only clears call history. The following tests can inherit this implementation and pass against polluted gateway behavior, so reset the mock or use one-shot implementations.
  - confidence: 0.9
