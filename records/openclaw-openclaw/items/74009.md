## Summary

Keep open. Current main still has the reported mixed-selector rejection, and the PR is aimed at a real bug confirmed again by recent production feedback, but the patch is not merge-ready because it opens a denied-response key-disclosure path and leaves a leaking test mock.

## What This Changes

The PR removes the sessions_send sessionKey-plus-label early rejection, updates the tool description and changelog, and adds regression tests for sessionKey/sessionId precedence over redundant label and agentId hints.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit sessionKey/sessionId as authoritative, preserves denied-access confidentiality, updates model-facing guidance, and keeps the new regression tests isolated.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: call sessions_send with both sessionKey and label, and the tool returns the mutual-exclusion error before resolution; the latest production comment reports the same behavior on v2026.5.7.

## Solution Assessment

No as submitted. Preferring explicit sessionKey is the right narrow direction, but the patch should not echo resolved canonical keys on denied sends and should reset or one-shot the added gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106`
  - body: Deleting the mixed-selector guard lets sessionId-shaped sessionKey inputs with redundant label/agentId values resolve before visibility and A2A checks. If access is denied, the existing branch returns the resolved displayKey, so mixed-selector probes can learn a canonical key they cannot send to; return the original selector or omit sessionKey on denial.
  - confidence: 0.84
- **[P2] Reset the gateway mock between gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: The added sessionId regression installs a persistent callGatewayMock.mockImplementation, but the local beforeEach only calls mockClear. Later tests in the same describe can inherit that gateway behavior, so reset the mock or use one-shot implementations for this test.
  - confidence: 0.9
