## Summary

Keep this PR open: the reported `sessions_send` mixed-selector bug is real and useful to fix, but the current patch still has a denied-access disclosure concern and a test-isolation bug before it should merge.

## What This Changes

The PR makes `sessions_send` prefer explicit `sessionKey` or session-id inputs over redundant `label`/`agentId` hints, updates the tool description and changelog, and adds focused regression tests.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit `sessionKey` or session-id input as authoritative while preserving denied-access confidentiality and current `main` behavior such as thread-target rejection and session bootstrap.

## Reproduction Assessment

Yes. Source inspection shows current `main` rejects `{ sessionKey, label, message }` before resolution, and a production comment reports the same failure on v2026.5.7.

## Solution Assessment

No as submitted. The precedence rule is the right direction, but the patch should avoid returning resolved canonical keys on denied sends and should reset or one-shot the persistent gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid echoing resolved keys on denied sends:** `src/agents/tools/sessions-send-tool.ts:106-108`
  - body: Removing the mixed-selector guard lets a session-id-shaped `sessionKey` with redundant metadata resolve before visibility/A2A checks. If access is denied, the existing branch returns the resolved `displayKey`, so callers can learn canonical keys they cannot send to; return the original selector or omit `sessionKey` on denial.
  - confidence: 0.84
- **[P2] Reset the gateway mock between gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: This test installs a persistent `callGatewayMock.mockImplementation(...)`, but the enclosing `beforeEach` only clears call history. Later `sessions_send gating` tests can inherit this gateway behavior, so reset the mock or use one-shot implementations.
  - confidence: 0.9
