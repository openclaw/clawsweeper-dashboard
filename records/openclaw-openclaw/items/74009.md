## Summary

Keep open: the underlying mixed-selector bug is still present on current main, and the branch is directionally useful, but this head is not merge-ready because it creates a denied-access disclosure path and leaves test state pollution in the new regression coverage.

## What This Changes

This PR removes the early `sessions_send` rejection for combined `sessionKey`/`label`, documents `sessionKey` precedence, adds changelog credit, and adds focused regression tests for redundant selector inputs.

## Best Possible Solution

Land a narrow sessions tool fix that treats explicit `sessionKey` values as authoritative while keeping label-only resolution unchanged, but first make denied sends echo only the caller-provided selector or omit `sessionKey`, reset or one-shot the new gateway mocks, and rerun the focused sessions test plus changed gate on the final branch.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction: call `sessions_send` with both `sessionKey` and `label`, and the guard at `src/agents/tools/sessions-send-tool.ts:109` returns the mutual-exclusion error before the send can resolve.

## Solution Assessment

No as submitted. Preferring `sessionKey` is the right narrow direction, but the current patch should also harden the denied-resolution response and isolate the new tests before it is the best final solution.
