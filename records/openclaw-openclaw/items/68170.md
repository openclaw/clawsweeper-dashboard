## Summary

Keep open: current main and the latest release still enumerate `/home` or `/Users` and print discovered sibling `.openclaw` paths, while the open linked pull request is the implementation candidate for the scoped fix.

## What This Changes

The issue asks Doctor to stop broadly scanning sibling home directories for `.openclaw` state dirs because it can disclose other users' paths and create backup-directory false positives.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/82495, align the Doctor docs with the final scoped behavior, then close this issue after merge.

Yes. Source inspection shows current main reads `/home` or `/Users`, detects sibling `.openclaw` directories, and prints each discovered path in the Doctor warning; I did not create real OS users in this read-only review.

## Solution Assessment

Yes. The linked scoped-scan PR is the narrowest maintainable fix because it removes sibling-home enumeration while preserving the active-vs-default state-dir warning; an env/config opt-out would add unnecessary surface for this bug.
