## Summary

Keep open. Current main and v2026.5.12 still scan sibling home roots for `.openclaw` and print discovered paths, so the reported privacy/noise issue remains unresolved; related items consolidate under this issue rather than supersede it.

## What This Changes

The issue requests changing Doctor state-directory discovery so it does not enumerate or print sibling users' or backup homes by default, with possible opt-out, scope, or redaction behavior.

## Best Possible Solution

Change Doctor state-integrity defaults to inspect only current/default/explicit state dirs or redact/summarize any sibling-home discoveries, then add focused regression coverage and update the Doctor guide.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on Linux or macOS, any readable sibling home or backup directory containing `.openclaw` is collected by `findOtherStateDirs` and printed by the split-state warning.

## Solution Assessment

No current fix is in place. A default-scoped or redacted diagnostic is narrower and safer than only adding an opt-out or suffix filter, but the exact behavior needs maintainer policy judgment because it changes documented Doctor output.
