## Summary

Keep open. Current main and the latest release still enumerate sibling home directories for `.openclaw` state and print discovered paths in Doctor output; related #68752 corroborates the multi-user noise but is closed and does not supersede the remaining privacy/false-positive concern.

## What This Changes

Request to change Doctor’s multiple-state-dir detection so it does not scan or print sibling users’ `/home` or `/Users` `.openclaw` directories and avoids backup-directory false positives.

## Best Possible Solution

Patch Doctor state integrity so default diagnostics inspect only the current effective home plus explicit/current/default state paths, avoid verbatim sibling-user paths, add focused regression coverage, and update Doctor docs plus changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on Linux or macOS, any readable sibling home directory containing `.openclaw` is collected by `findOtherStateDirs` and printed by `noteStateIntegrity`.

## Solution Assessment

No current fix is in place. A default-scoped or redacted diagnostic is the narrowest maintainable direction because it addresses both cross-user disclosure and backup false positives better than only adding an opt-out or suffix filter.
