## Summary

Keep open. Current main and v2026.5.18 still scan sibling `/home` or `/Users` entries and print discovered `.openclaw` paths, and the linked fix PR is open but not merged, so this issue should close only after that PR or an equivalent maintainer-approved fix lands.

## What This Changes

The issue requests that Doctor stop enumerating sibling Unix home directories for `.openclaw` state dirs because the warning can disclose other users' paths and create backup-dir false positives.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82495 or an equivalent scoped/redacted Doctor fix, with updated Doctor docs, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: a readable sibling or backup home under `/home` or `/Users` with `.openclaw` is discovered and then printed in the multiple-state warning.

## Solution Assessment

Yes, the linked scoped-scan PR is a narrow maintainable direction because it removes sibling-home enumeration while preserving the active-vs-default state warning. The remaining decision is maintainer acceptance of the changed documented diagnostic and the matching docs update.
