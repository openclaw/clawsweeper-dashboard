## What This Changes

The issue asks Doctor state-dir detection to stop scanning sibling OS home directories and avoid leaking or falsely warning on other users' or backup `.openclaw` dirs.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/82495, with Doctor docs updated so the documented diagnostic matches the chosen scoped behavior.

## Reproduction Assessment

Yes from source: current main enumerates `/home` or `/Users` siblings and the Doctor warning prints discovered `.openclaw` paths. I did not create real OS users in this read-only review.

## Solution Assessment

Yes, the best direction is to scope the diagnostic to the current account/default state directory rather than adding an env/config opt-out or backup-name heuristic as the primary fix.
