## Summary

Keep open. Current main and v2026.5.7 still enumerate sibling home directories for `.openclaw` state and print discovered paths, while the related suppressor PR is only an opt-in workaround and does not resolve the default disclosure/noise behavior.

## What This Changes

The issue asks Doctor state-integrity to avoid scanning and printing sibling-user or backup `.openclaw` directories by default, or to provide a scoped/redacted/suppressed alternative.

## Best Possible Solution

Change Doctor state-integrity so default diagnostics do not disclose sibling-user state paths; if cross-home diagnostics stay, make them explicitly scoped, summarized, or redacted, with focused tests and Doctor docs updates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on Linux or macOS, any readable sibling home directory containing `.openclaw` is collected by `findOtherStateDirs` and then printed in the split-state warning.

## Solution Assessment

Unclear. The report's direction is valid, but maintainers need to choose whether the cross-home diagnostic should be restricted, redacted, summarized, opt-in, or configurable because current docs explicitly describe cross-home detection.
