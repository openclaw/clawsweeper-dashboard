## What This Changes

The issue asks Doctor state-integrity checks to stop scanning sibling Unix home directories and leaking or falsely warning on other users' or backup `.openclaw` state paths.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/82495, then update Doctor docs if the scoped behavior is accepted and close this issue after merge.

Yes. Source inspection shows current main reads `/home` or `/Users`, detects sibling `.openclaw` directories, and prints each discovered path in the Doctor warning; I did not create real OS users in this read-only review.
