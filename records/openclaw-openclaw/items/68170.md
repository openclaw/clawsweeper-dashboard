## Summary

Keep open. Current main and the latest release still enumerate sibling `/home` or `/Users` entries and print discovered `.openclaw` paths, so the reported cross-user disclosure and backup-dir false positives remain unresolved.

## What This Changes

The issue requests safer Doctor state-directory discovery so `openclaw doctor` does not expose sibling-user `.openclaw` paths or warn on backup-style home directories by default.

## Best Possible Solution

Change Doctor state-integrity defaults to avoid listing sibling-user paths, add focused regression coverage for multi-user and backup-home layouts, and update the Doctor guide to describe the new privacy-preserving behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on Linux or macOS, any readable sibling home or backup-style directory containing `.openclaw` is collected and printed by the split-state warning.

## Solution Assessment

No current fix is in place. The safest direction is a scoped or redacted default rather than only an opt-out flag, but the exact behavior needs maintainer product/security judgment because current docs intentionally describe cross-home detection.
