## Summary

Keep open. Current main and the latest release still enumerate sibling `/home` or `/Users` entries for `.openclaw` state and print discovered paths, while the related open suppressor PR only covers an opt-in workaround rather than the default cross-user disclosure/noise.

## What This Changes

The issue asks Doctor to stop enumerating and printing sibling-home `.openclaw` state dirs, or to scope, redact, or suppress that scan for managed multi-user hosts.

## Best Possible Solution

Patch Doctor state integrity so default diagnostics inspect only the current effective home plus explicit/current/default state paths, or otherwise redact/summarize sibling-home discoveries; add focused regression coverage and update the Doctor guide.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on Linux or macOS, any readable sibling home directory containing `.openclaw` is collected by `findOtherStateDirs` and then printed by the split-state warning.

## Solution Assessment

No current fix is in place. A default-scoped or redacted diagnostic is the narrowest maintainable direction because it addresses both cross-user disclosure and backup false positives better than only adding an opt-out or suffix filter.
