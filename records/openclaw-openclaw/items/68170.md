## Summary

Keep open: current main and the latest release still enumerate sibling Unix home roots and print discovered `.openclaw` paths, while https://github.com/openclaw/openclaw/pull/82495 is the open closing PR for the scoped fix.

## What This Changes

The issue asks `openclaw doctor` to stop scanning sibling Unix home directories for `.openclaw` state dirs because that can leak other users' paths and cause backup-directory false positives.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/82495, align the Doctor docs with the final scoped behavior, then close this issue after merge.

## Reproduction Assessment

Yes. From source, current main reads `/home` or `/Users`, detects sibling `.openclaw` directories, and prints each discovered path in the Doctor warning; I did not create real OS users in this read-only review.

## Solution Assessment

Yes. The linked scoped-scan PR is the narrowest maintainable fix shape because it removes sibling-home enumeration while preserving the active-vs-default state-dir warning when `OPENCLAW_STATE_DIR` points elsewhere.
