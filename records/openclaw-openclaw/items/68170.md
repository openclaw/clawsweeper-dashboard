## Summary

Keep open. Current main and the latest release still scan sibling `/home` or `/Users` entries and print discovered `.openclaw` paths, while an open linked PR is the active fix path but has not merged yet.

## What This Changes

The issue asks Doctor state-integrity checks to stop enumerating sibling Unix home directories and avoid printing other users' or backup `.openclaw` paths.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/82495 with current-account-scoped state detection, regression coverage, and updated Doctor docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on Linux or macOS, any readable sibling home or backup-style directory containing `.openclaw` is collected and printed by the multiple-state warning.

## Solution Assessment

Yes with one remaining maintainer choice. Scoping discovery to the current account default state dir is the narrow fix direction, but the documented cross-home diagnostic needs to be updated or explicitly preserved by policy.
