## Summary

Keep this PR open, but do not merge as-is: the Moonshot static-catalog fast path is the right direction for https://github.com/openclaw/openclaw/issues/84783, while the current patch bypasses provider normalization for all skip-PI static catalog hits and can break existing static-provider request metadata.

## What This Changes

The PR makes embedded PI model resolution try bundled static catalog rows before dynamic provider hooks when PI discovery is skipped, and opts the embedded runner into that fallback.

## Best Possible Solution

Keep the static catalog lookup before dynamic model preparation, but preserve provider normalization/compat for returned static rows or move the required compat into a lightweight static contract with regression coverage.

## Reproduction Assessment

No, not as a full live high-confidence reproduction in this read-only review. The reporter's trace and current source show the slow `models.json` fallback path, and the PR proof exercises the after-fix model-resolution entrypoint without a live Moonshot dispatch.

## Solution Assessment

No, not as currently patched. Static-before-dynamic is the right fix direction, but the implementation should not bypass provider normalization/compat hooks for every skip-PI static catalog hit.
