## Summary

Keep open: the PR is aimed at the right Moonshot fast-path bug and includes sufficient resolver proof, but the latest patch is not merge-ready because static catalog hits under `skipPiDiscovery` bypass provider model normalization and can regress existing provider compat metadata.

## What This Changes

The PR changes embedded PI model resolution to try bundled static catalog rows before dynamic provider preparation when `skipPiDiscovery` is set, passes that option from the embedded runner, and updates focused resolver tests.

## Best Possible Solution

Keep the static-catalog-before-dynamic-prep fast path, but preserve lightweight target-provider normalization or move the required compat metadata into static catalog rows with regression coverage for Moonshot and Mistral.

No live Discord/Moonshot reproduction was run in this read-only pass. The source path is clear from `runEmbeddedPiAgent` passing `skipPiDiscovery` into `resolveModelAsync`, and the linked issue includes model-resolution timing logs for the affected Moonshot setup.

## Solution Assessment
