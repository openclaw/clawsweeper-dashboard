## Best Possible Solution

Rebase onto current main, pass the resolved Kilocode `discoveryApiKey` into `discoverKilocodeModels()` inside `fetchWithSsrFGuard`, retain safe fallback behavior, cover env/config/profile credentials, and attach redacted live BYOK proof.

No live high-confidence reproduction was established. Source inspection and the Kilo-Org/cloud report support the path: current main sends unauthenticated discovery while BYOK catalog entries require authenticated gateway discovery.

## Solution Assessment

No. Adding discovery auth is the right direction, but the maintainable fix should use the catalog-resolved discovery credential on the current guarded fetch path instead of reading only ambient env on a stale raw-fetch branch.
