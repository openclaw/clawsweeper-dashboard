## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate broker sends to configured platforms:** `extensions/channel-broker/src/outbound.ts:95-99`
  - body: The broker advertises many platform prefixes, but `sendChannelBrokerText` never checks the parsed target platform against the selected account's configured `platforms`. A provider configured for Telegram can still receive `broker:discord:...` or fallback `discord:` sends when native Discord is absent, which violates the documented provider platform contract and can misroute messages; fail before creating the outbound request unless the account intentionally has no platform allowlist.
  - confidence: 0.89
- **[P2] Preserve broker conversation type in canonical routes:** `extensions/channel-broker/src/channel.ts:88-93`
  - body: `buildCanonicalBrokerTarget` drops `conversationType`, so a direct/group broker target is stored as `<platform>:<id>` and later reparses with the default channel type. That can lose DM/group session identity after the first send; fold the route-preservation shape from https://github.com/openclaw/openclaw/pull/86139 or encode the type here before landing this branch.
  - confidence: 0.83
