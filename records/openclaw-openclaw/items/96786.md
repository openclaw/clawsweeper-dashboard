## Best Possible Solution

## Solution Assessment

Yes: reusing `readProviderJsonResponse` is the narrowest owner-boundary fix because the shared provider helper already owns successful provider JSON caps. A custom parser or new config knob would duplicate existing policy unless maintainers reject the 16 MiB default.
