## Summary

Keep this PR open, but not ready to merge: the iOS subscription/mapping direction matches the existing gateway contract, while the new shared echo-adoption branch can still suppress legitimate same-content user transcripts and the live iOS group-chat proof is still missing.

## What This Changes

The PR makes iOS subscribe to `sessions.messages.subscribe`, maps `session.message` frames into the shared chat transport event, and changes `ChatViewModel` to append assistant transcript events while a run is pending with new Swift tests.

## Best Possible Solution

Keep the iOS subscription/mapping approach, but scope user-echo adoption to the actual optimistic local send and then verify the fixed path with live iOS group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes for source reproduction, not by a command I ran: current main’s iOS active-session hook is a no-op and the shared handler drops assistant or pending `session.message` events. The PR body also includes a baseline Swift timeout for the focused view-model case.

## Solution Assessment

No as-is. The subscription and assistant transcript handling are the right direction, but the user-echo adoption needs a narrower identity before this is the maintainable fix.
