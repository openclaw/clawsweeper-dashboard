## Summary

Keep this PR open, but it is not merge-ready: the iOS subscription and `session.message` mapping are the right fix shape, while the new user-echo adoption path can still drop a distinct same-content user transcript and the supplied proof does not show a live iOS Gateway-backed group chat updating in real time.

## What This Changes

The PR makes the iOS chat transport subscribe to per-session transcript events, maps `session.message` frames, and updates shared chat view-model handling and Swift tests for live group-chat transcripts and local user-echo dedupe.

## Best Possible Solution

Land the iOS subscription and `session.message` mapping after constraining user-echo adoption to proven local optimistic sends and adding live iOS group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, at source level: current main leaves iOS unsubscribed from per-session transcripts, ignores `session.message` frames, and drops assistant messages while a run is pending. I did not run the Swift repro locally in this read-only review.

## Solution Assessment

No, not as-is: the subscription and mapping are the maintainable fix shape, but the echo-dedupe path must stop treating every same-content user transcript as a local echo.
