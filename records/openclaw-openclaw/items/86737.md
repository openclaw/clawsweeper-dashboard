## Summary

Keep open: the subscription and event-mapping direction matches the existing gateway/macOS contract, but the new user-echo adoption path can drop distinct repeated user transcript turns, and the supplied proof still does not show a live iOS Gateway group chat updating in real time.

## What This Changes

The PR wires iOS chat to `sessions.messages.subscribe`/`session.message` events and changes the shared chat view model/tests so assistant transcript updates can appear during pending group-chat runs while local user echoes dedupe.

## Best Possible Solution

Land the gateway subscription/mapping fix with user-echo adoption constrained to the active optimistic local row, repeated same-text transcript coverage, and redacted live iOS group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main does not subscribe iOS to per-session transcripts, drops `session.message` in the iOS transport, and filters assistant/pending session messages in the shared view model. I did not run the live iOS group-chat path in this read-only review.

## Solution Assessment

Not as-is: using the existing `sessions.messages.subscribe` and `session.message` contract is the right direction, but the user-echo dedupe needs a narrower local-echo guard and the live iOS group-chat behavior still needs proof.
