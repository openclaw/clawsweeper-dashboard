## Summary

Keep open. The iOS subscription and `session.message` mapping match the existing macOS/gateway contract, but the new user-echo adoption is too broad and can suppress later distinct same-content user messages; the external PR also still needs live iOS Gateway-backed group-chat proof or an explicit maintainer proof override.

## What This Changes

The PR updates iOS/shared Swift chat handling to subscribe to per-session transcript events, map `session.message` frames, append assistant transcripts during pending runs, and add focused Swift tests.

## Best Possible Solution

Constrain echo adoption to the actual pending optimistic local user row, add a repeated same-content user-message regression test, then land after redacted live iOS Gateway proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes for source-level reproduction: current main leaves the iOS active-session hook as a no-op and drops assistant `session.message` events while a run is pending. I did not run tests because this review is read-only, but the PR body includes a baseline failing Swift test for the reported symptom.

## Solution Assessment

No. The subscription and assistant-transcript handling are the right boundary, but the current echo-dedupe repair is too broad because it treats any same-content user event as the local optimistic echo.
