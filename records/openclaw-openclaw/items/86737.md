## Summary

Keep this PR open, but it is not merge-ready: the subscription and `session.message` mapping direction matches the Gateway/macOS contract, while the new user-echo adoption path can still suppress legitimate repeated same-content user transcripts and the live iOS Gateway group-chat proof gate remains unresolved.

## What This Changes

The PR adds iOS `sessions.messages.subscribe` wiring, maps `session.message` frames into shared chat transport events, updates the shared chat view model to accept assistant transcript updates during pending runs, and adds Swift regression coverage.

## Best Possible Solution

Land the iOS transcript subscription fix only after echo adoption is constrained to the actual local optimistic send and live iOS Gateway group-chat behavior is proven or explicitly overridden.

## Reproduction Assessment

Yes from source inspection: current main has a no-op iOS active-session subscription, no iOS `session.message` event mapping, and shared-model guards that drop assistant transcripts while pending. I did not run a live iOS Gateway group-chat reproduction in this read-only review.

## Solution Assessment

No, not yet. The subscription and mapping path is the right owner-boundary fix, but the current echo-adoption implementation is too broad and needs a narrower local-echo match plus live behavior proof or explicit override.
