## What This Changes

The PR wires the iOS chat transport to subscribe to per-session transcript events, maps `session.message` frames, and changes the shared chat view model/tests so active-session assistant transcripts can appear while a run is pending.

## Best Possible Solution

Keep the existing subscription and `session.message` direction, but bound local echo adoption to the actual pending optimistic send and add live iOS Gateway proof or an explicit maintainer proof override before merge.

## Reproduction Assessment

Yes, by source and focused test shape: current main does not subscribe/map iOS `session.message` frames and the shared handler drops assistant or pending session messages. I did not run a live Gateway-backed iOS group-chat repro in this read-only review.

## Solution Assessment

No, not as-is: subscribing/mapping the existing gateway transcript stream is the right fix direction, but the new user-echo adoption is overbroad and needs a narrower local-pending-send guard plus proof for the live iOS flow.
