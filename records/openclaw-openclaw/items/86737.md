## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: current main leaves iOS active-session subscription as a no-op and drops assistant `session.message` events while a run is pending. The PR body also supplies a focused failing/passing Swift test transcript for that pending assistant-message path.

## Solution Assessment

No, not yet: the subscription and decode direction is the right narrow fix, but the content-only echo adoption is too broad and can suppress distinct user transcripts. The safer solution is to identify only the active optimistic local echo, then let all other user messages append through the normal reconciliation path.
