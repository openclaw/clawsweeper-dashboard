## What This Changes

The PR makes the iOS chat transport subscribe to per-session transcript events, maps `session.message` frames, removes shared handler filters that dropped assistant messages during pending runs, and adds Swift regression tests.

## Best Possible Solution

Land this after keeping the iOS subscription and assistant transcript path while explicitly suppressing or reconciling local user transcript echoes, then attach real iOS group-chat proof or a maintainer Mantis override.

## Reproduction Assessment

Yes, from source inspection: current main does not subscribe/map iOS `session.message` frames and the shared handler drops assistant or pending session messages. I did not run the Swift repro locally, but the PR body also includes a failing baseline Swift test transcript for the reported symptom.

## Solution Assessment

No, not yet: subscribing and mapping `session.message` is the right direction, but removing both guards without replacing the local-user echo protection is not the narrowest safe fix. The safer solution is to allow assistant updates during pending runs while preserving or improving user-message reconciliation.
