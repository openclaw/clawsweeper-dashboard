## Summary

Keep open: the iOS real-time group-chat fix is still useful, but the current PR is not merge-ready because the new local-echo adoption path can suppress distinct same-content user transcript events, and live iOS Gateway group-chat proof is still missing.

## What This Changes

The PR subscribes iOS chat transport to per-session transcript events, maps `session.message` frames, relaxes shared chat transcript filtering, and adds Swift regression tests for assistant updates and local echo dedupe.

## Best Possible Solution

Keep the iOS subscription and `session.message` mapping, but scope local echo adoption to a verifiable optimistic local send and add proof that repeated same-content user transcripts are preserved before live iOS proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes from source inspection: current main does not subscribe iOS to per-session messages and drops assistant `session.message` payloads while pending. The PR-side regression is also source-reproducible by emitting a second same-content user transcript after an existing same-content user row.

## Solution Assessment

No, not yet. The subscription and event mapping are the right boundary, but the current echo-dedupe repair needs a narrower identity rule so it does not trade assistant delivery for user-message loss.
