## Summary

Keep this PR open. The central fix direction matches the linked iOS group-chat bug, but the current patch can still drop a distinct same-text user transcript after enabling all per-session messages, and the supplied proof does not yet show the live Gateway-backed iOS group-chat behavior after the fix.

## What This Changes

The PR wires iOS chat to `sessions.messages.subscribe`, maps `session.message` events into the shared chat transport, changes shared transcript handling for assistant events and local user echoes, and adds Swift regression tests.

## Best Possible Solution

Land the iOS session-message subscription only after echo adoption is constrained to the active local optimistic send and a live iOS Gateway group-chat proof or maintainer proof override confirms real-time delivery.

## Reproduction Assessment

Yes. Current main source shows iOS neither subscribes to nor decodes `session.message`, and the shared handler drops assistant messages and pending-run updates; the PR body also includes a baseline Swift test timeout, though I did not run tests in this read-only review.

## Solution Assessment

No, not yet. Subscribing iOS to the existing gateway stream is the right direction, but the echo-dedupe repair needs to be constrained before this is the safest maintainable fix.
