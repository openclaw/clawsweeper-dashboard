## Summary

This PR targets a real iOS group-chat transcript gap, but it is not merge-ready because the new user-echo adoption can drop legitimate repeated same-content user transcript events and the supplied proof still does not show the live Gateway-backed iOS group-chat behavior.

## What This Changes

The PR subscribes iOS chat to per-session transcript events, maps `session.message` frames, and updates shared chat view-model/tests so assistant transcripts can appear while a group-chat run is pending.

## Best Possible Solution

Keep the iOS subscription and `session.message` mapping direction, but restrict user-echo adoption to a known pending local send, preserve same-content external/repeated user transcripts with regression coverage, and add live iOS Gateway group-chat proof or a maintainer proof override before merge.
