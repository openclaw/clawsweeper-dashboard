## Summary

Keep open. ## What This Changes

The PR makes the iOS chat transport subscribe to per-session transcript events, maps `session.message` frames, lets assistant session messages update the shared chat view model during pending runs, and adds Swift regression tests.

## Best Possible Solution

Rebase onto current main, constrain local echo adoption to actual optimistic local sends while preserving assistant session-message streaming, add repeated same-content external-user coverage, then provide live iOS Gateway group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main does not subscribe or decode iOS `session.message` frames, and the shared handler drops assistant messages and all session messages while a run is pending. The PR body also includes a baseline Swift test transcript where the assistant message remains invisible until timeout.

## Solution Assessment

Partly: subscribing iOS to `sessions.messages.subscribe`, mapping `session.message`, and allowing assistant updates through the shared handler is the narrow contract-aligned fix. The local echo dedupe is not safe yet because it matches by content alone instead of a known pending optimistic local send.
