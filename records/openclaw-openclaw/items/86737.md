## What This Changes

The PR wires the iOS chat transport to subscribe to per-session transcript events, maps `session.message` frames into shared chat events, and updates the shared chat view model/tests so group-chat assistant replies can appear while a run is pending.

## Best Possible Solution

Land the iOS subscription/session-message path after echo adoption is constrained to confirmed local optimistic sends and the PR has either live Gateway-backed iOS group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current iOS never subscribes to per-session transcript messages, does not map `session.message`, and the shared view model drops assistant events while a run is pending. I did not run the live iOS group-chat scenario in this read-only review.

## Solution Assessment

No, not quite yet: subscribing and mapping `session.message` is the right fix direction, but the current echo-adoption helper needs to identify only local optimistic sends before the patch is merge-safe.
