## Best Possible Solution

Keep the iOS subscription and assistant transcript mapping, but constrain user-echo adoption to a confirmed local optimistic send and add a regression test showing repeated same-content external user transcripts both remain visible.

## Reproduction Assessment

Yes from source: current main has a no-op iOS active-session subscription and the shared handler drops assistant `session.message` payloads while pending, which matches the linked iOS group-chat symptom. I did not run the Swift tests because this review is read-only.

## Solution Assessment

No, not yet: subscribing and mapping `session.message` is the right fix direction, but the current echo dedupe implementation is too broad and can erase legitimate repeated user messages. The safer solution is to adopt only confirmed local optimistic echoes and let other user transcripts append normally.
