## What This Changes

The PR wires iOS chat transport to subscribe to per-session transcript events, maps `session.message` frames, removes shared filters that dropped assistant session messages during pending runs, and adds Swift regression tests.

## Best Possible Solution

Land the targeted fix after a paired iOS simulator or device group-chat run shows assistant replies arriving in the active chat without leaving and re-entering, with normal Swift and CI checks green.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: current main does not subscribe iOS to per-session transcript events, drops `session.message` frames, and filters assistant messages while a run is pending. I did not run the live iOS group-chat flow locally.

## Solution Assessment

Yes, the patch is the narrow maintainable code path because it uses the existing gateway subscription contract, mirrors macOS event mapping, and keeps the shared session-key/dedupe guards. The remaining blocker is real user-flow proof, not an identified implementation defect.
