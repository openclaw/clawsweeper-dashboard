## What This Changes

The branch makes the iOS chat transport subscribe to `sessions.messages.subscribe`, maps `session.message` frames, and updates shared ChatViewModel/tests so active-session assistant transcripts can appear during pending group-chat runs.

## Best Possible Solution

Keep the iOS transcript subscription/mapping approach, but gate user-echo adoption to the pending local optimistic send, add same-content external-user regression coverage, then prove the live iOS Gateway group-chat flow or record an explicit proof override.

## Solution Assessment

No, not as-is. The subscription and mapping direction is the right narrow fix, but the echo-adoption helper needs a stricter local-pending match and a same-content external-user regression before this is safe to merge.
