## Summary

Keep open; the transport subscription and session-message mapping are the right direction, but the current PR still has a blocking shared-model echo-adoption regression and does not yet provide live iOS Gateway-backed group-chat proof or a maintainer proof override.

## What This Changes

The PR makes iOS subscribe to and map `session.message` frames, then updates the shared chat view model and tests so assistant transcripts can append during pending group-chat runs while local user echoes are deduped.

## Best Possible Solution

Land the iOS subscription and `session.message` mapping after constraining user-echo adoption to proven local optimistic sends and adding live Gateway-backed iOS group-chat proof or an explicit proof override.

## Reproduction Assessment

Yes, at source level: current `main` leaves iOS unsubscribed from per-session transcripts, ignores `session.message` frames, and drops assistant messages while a run is pending. I did not run the Swift repro locally in this read-only review.

## Solution Assessment

No, not as-is: the subscription and mapping are the maintainable fix shape, but the echo-dedupe path must stop treating every same-content user transcript as a local echo.
