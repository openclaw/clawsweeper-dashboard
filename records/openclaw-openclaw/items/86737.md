## Summary

Keep open. The PR targets the right iOS/shared event path for https://github.com/openclaw/openclaw/issues/80231, but current head still has a blocking message-delivery regression: same-content user transcripts can be adopted onto an old row without proving they are the active client's pending local echo, and the real behavior proof still does not show live iOS Gateway group-chat delivery.

## What This Changes

The PR makes iOS subscribe to per-session transcript events, maps `session.message` frames, and updates the shared chat view model plus Swift tests for real-time group-chat transcript updates.

## Best Possible Solution

Land the iOS subscription and `session.message` handling only after echo adoption is limited to a verified pending optimistic local send, a same-content external-user regression is added, and live iOS Gateway proof or an explicit maintainer proof override is recorded.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main does not subscribe/map iOS `session.message` events and the shared handler drops assistant messages while a run is pending. The PR's added Swift test mirrors that path, though I did not run it in this read-only review.

## Solution Assessment

No, not as-is. Subscribing iOS and accepting assistant session messages is the right fix shape, but the echo-deduplication branch needs a narrower guard so repeated or same-content external user transcripts are not lost.
