## Summary

Keep open: the iOS/shared fix is still relevant and uses the right Gateway event path, but the current patch is not merge-ready because its new user-echo adoption can suppress same-content active-session user transcripts, and the supplied proof still does not show a live Gateway-backed iOS group chat updating in real time.

## What This Changes

The PR makes iOS subscribe to per-session transcript events, maps `session.message` frames into the shared transport event, allows assistant transcript updates in the shared chat view model, and adds Swift regression coverage plus local-user echo dedupe.

## Best Possible Solution

Land the existing Gateway subscription/decoding direction after constraining user-echo adoption to a confirmed pending optimistic local send and adding live iOS Gateway group-chat proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes from source inspection and the linked issue steps: current main does not subscribe iOS to per-session transcript messages, drops `session.message` frames, and filters assistant payloads while pending. I did not run a live iOS Gateway group-chat repro in this read-only review.

## Solution Assessment

Partly: using the existing `sessions.messages.subscribe` and `session.message` contract is the narrow maintainable direction. The proposed local-user echo helper needs a stronger pending-send identity check before this is the best mergeable fix.

- **[P1] Constrain user-echo adoption to the pending local send:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:406-407`
