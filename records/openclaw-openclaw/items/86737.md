## Summary

Keep this PR open. ## What This Changes

The PR changes iOS/shared chat event handling so iOS subscribes to per-session transcript events, maps `session.message` frames, and lets assistant transcript updates appear during pending group-chat runs.

## Best Possible Solution

Keep the per-session transcript subscription and assistant-message path, but gate user-echo adoption to actual optimistic local sends and then verify the live iOS group-chat flow or record a maintainer proof override.

## Reproduction Assessment

Yes. Source inspection shows current main does not subscribe iOS to per-session transcript events and drops assistant `session.message` payloads while pending; the PR body also includes a focused failing-before/passing-after Swift test transcript for that path.

## Solution Assessment

No, not yet. The subscription and mapping path matches the existing gateway/macOS contract, but the branch needs a narrower echo-adoption guard so it does not turn distinct same-content user transcript events into message loss.
