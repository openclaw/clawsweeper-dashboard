## Summary

This PR addresses a real iOS group-chat delivery bug, but it is not merge-ready: the current echo-adoption logic can still suppress distinct same-content user transcripts, and the supplied proof does not show the patched iOS app receiving live Gateway-backed group-chat replies.

## What This Changes

The PR makes iOS subscribe to per-session transcript events, maps `session.message` frames into shared chat events, relaxes the shared view model's assistant-message filtering, and adds Swift regression coverage plus local-user echo dedupe.

## Best Possible Solution

Land the existing subscription and `session.message` direction only after echo adoption is constrained to a verified pending optimistic local send and live iOS Gateway group-chat proof, Mantis proof, or an explicit maintainer proof override is attached.

## Reproduction Assessment

Yes. ## Solution Assessment

No, not yet. The transport subscription and `session.message` mapping are the right narrow direction, but the content-only echo adoption is too broad and the proof does not demonstrate the live iOS group-chat behavior.
