## Best Possible Solution

Land after echo adoption is gated to a tracked local optimistic send or server identity with regression coverage, and after redacted live iOS Gateway group-chat proof or an explicit maintainer proof override is added.

## Reproduction Assessment

Yes, from source and supplied focused test shape: current `main` does not subscribe iOS to per-session transcript events, does not map `session.message`, and drops assistant/pending transcript updates in the shared view model. I did not reproduce the full live iOS Gateway group-chat flow locally.

## Solution Assessment

No. The iOS subscription and event-mapping direction is the right contract-aligned fix, but the current user-echo dedupe implementation uses only user content and can suppress another real transcript; it should be restricted to the local optimistic send or canonical server identity.
