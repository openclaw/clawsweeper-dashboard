## Summary

Keep open. The PR is not merge-ready because it changes fallback/UI lists, but current provider-profile resolution still returns exact provider-owned options for real models, and the proposed rank change makes `adaptive` fallback conflict with the updated docs/tests.

## What This Changes

The branch expands backend and Control UI thinking-level lists, ranks, labels, tests, and docs so non-binary selectors would advertise `xhigh`, `adaptive`, and `max` with graceful fallback.

## Best Possible Solution

Decide the thinking-level contract first; if all-level display is intended, apply it after provider-profile normalization or update provider-owned profiles, docs, and Gateway/UI tests consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: OpenAI returns a non-empty provider profile before fallback levels are used, Gateway exports that exact list, and the Sessions UI prefers the Gateway-provided levels.

## Solution Assessment

No. The proposed patch changes fallback and local UI defaults, but the authoritative provider-profile path still bypasses those additions for real providers, and the `adaptive` rank no longer matches the documented fallback intent.

## Review Findings
