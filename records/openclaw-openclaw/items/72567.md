## Summary

Keep open. The PR is a focused attempt, but it does not currently deliver the stated all-eight-option behavior for real provider-profiled models, and one rank change conflicts with the documented fallback behavior.

## What This Changes

The PR expands backend and Control UI fallback thinking-level lists, ranks, labels, tests, and docs to advertise `xhigh`, `adaptive`, and `max` for non-binary providers.

## Best Possible Solution

Decide the thinking-level contract first; if all-level display is intended, apply that policy consistently after provider-profile normalization or update provider-owned profiles, docs, and tests together.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a real provider profile such as OpenAI returns before fallback levels are used, then Gateway passes that exact profile list to the Control UI.

## Solution Assessment

No. The patch changes fallback lists and documentation, but the authoritative provider-profile path remains exact, so the stated UI behavior is not achieved for profiled providers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply all-level policy after provider profiles:** `src/auto-reply/thinking.shared.ts:34-43`
  - body: Expanding `BASE_THINKING_LEVELS` only affects fallback profiles. `resolveThinkingProfile` returns non-empty provider profiles before this list is used, and real providers like OpenAI still declare exact levels without `adaptive`/`max`, so Gateway rows and the Control UI will not show all eight options as the PR claims.
  - confidence: 0.9
- **[P2] Align adaptive rank with the fallback docs:** `src/auto-reply/thinking.shared.ts:50-51`
  - body: With `xhigh` ranked 50 and `adaptive` ranked 55, the existing fallback resolver maps unsupported `adaptive` to `xhigh` on an xhigh-capable profile, not to `high` as the updated docs/tests claim. Put `adaptive` below `xhigh` if `high` is intended, or update the docs/tests for xhigh-capable profiles.
  - confidence: 0.82
