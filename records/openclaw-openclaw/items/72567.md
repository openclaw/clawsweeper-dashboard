## What This Changes

This PR expands backend and Control UI thinking-level option lists, ranks, labels, tests, and docs to advertise `xhigh`, `adaptive`, and `max` for non-binary providers.

## Best Possible Solution

Choose the provider thinking contract first; if all non-binary models should advertise remappable levels, post-process provider profiles or update provider-owned profiles, docs, and tests consistently.

## Reproduction Assessment

Yes. Static source reproduction is high-confidence: OpenAI returns a non-empty provider profile before fallback levels are used, and Gateway/Control UI render that exact `thinkingLevels` list.

## Solution Assessment

No. The patch changes fallback lists and UI defaults without resolving the authoritative provider-profile contract, so it is not the narrowest maintainable implementation yet.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.9

Put `adaptive` below `xhigh` if `high` is intended, or update docs/tests for the xhigh-capable fallback.
  - confidence: 0.84
