## Summary

Keep open. The PR is trying to change a real thinking-level UI contract, but the current patch does not implement its stated behavior for profiled providers: non-empty provider thinking profiles return before the expanded fallback list is used, and the Gateway/Control UI surfaces consume those exact resolved options.

## What This Changes

The PR expands backend and Control UI fallback thinking-level lists to include `xhigh`, `adaptive`, and `max`, adjusts level ranks and labels, and updates thinking tests and docs for all-level dropdown behavior.

## Best Possible Solution

Maintainers should decide the intended thinking-level contract first. If all-level display is desired, implement it once after provider-profile normalization, or update provider-owned profiles consistently, with coverage for a real profiled provider plus Gateway row and Control UI selector paths; if model-accurate pickers remain the contract, this PR should be narrowed to specific provider profile gaps instead.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: for a non-empty provider profile such as OpenAI, `resolveThinkingProfile` returns before the expanded base profile is used, and Gateway rows pass that exact list to the Control UI.

## Solution Assessment

No. The patch changes fallback lists and docs, but the authoritative provider-profile path remains exact and bypasses the new fallback levels, so it is not the narrowest maintainable solution for the stated UI behavior.
