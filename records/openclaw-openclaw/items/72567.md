## What This Changes

The PR expands backend and Control UI thinking fallback lists to include `xhigh`, `adaptive`, and `max`, adjusts ranks and labels, and updates thinking tests and docs for all-level dropdown behavior.

## Best Possible Solution

Maintainers should decide the intended thinking-level contract first. If all-level display is desired, implement it in one central post-normalization path or update provider-owned profiles consistently, with coverage for a real profiled provider plus Gateway and Control UI selector paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: any non-empty provider profile such as OpenAI returns before the expanded fallback profile, then Gateway rows and the Sessions view render that exact profile list.

## Solution Assessment

No. Expanding only fallback lists and docs is not the narrowest maintainable fix because current runtime and UI behavior is provider-profile driven; the safer path is a central provider-profile policy change or explicit provider-owned profile updates.
