## Summary

Keep open. The PR is a plausible attempt at a real thinking-level UI limitation, but the current patch does not deliver the stated all-eight dropdown behavior for profiled providers and it changes the documented provider-profile contract without maintainer product agreement.

## What This Changes

The PR expands backend and Control UI fallback thinking-level lists to include `xhigh`, `adaptive`, and `max`, adjusts ranks and labels, and updates tests/docs to advertise all-level dropdown behavior for non-binary providers.

## Best Possible Solution

Maintainers should decide the intended thinking-level contract first. If all-level display is desired, implement it after provider-profile normalization or update provider-owned profiles consistently, with coverage for a real profiled provider plus Gateway row and Control UI selector paths; if model-accurate pickers remain the contract, narrow the change to specific provider profile gaps.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `resolveThinkingProfile` returns non-empty provider profiles before the expanded fallback list, and Gateway rows pass that exact provider list to the Control UI.

## Solution Assessment

No. The patch changes fallback lists, labels, and docs, but the authoritative provider-profile path bypasses the new fallback levels, and the adaptive rank change does not match the documented fallback behavior for xhigh-capable profiles.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply all-level policy after provider profiles:** `src/auto-reply/thinking.shared.ts:34-43`
  - body: Expanding `BASE_THINKING_LEVELS` only affects fallback profiles. `resolveThinkingProfile` returns non-empty provider profiles before this list is used, and real providers like OpenAI still declare exact levels without `adaptive` or `max`, so Gateway rows and the Control UI will not show all eight options as the PR claims. Apply the policy after provider-profile normalization or update provider-owned profiles and tests consistently.
  - confidence: 0.9
- **[P2] Align adaptive rank with the documented fallback:** `src/auto-reply/thinking.shared.ts:50-51`
  - body: With `xhigh` ranked 50 and `adaptive` ranked 55, the existing fallback resolver maps unsupported `adaptive` to `xhigh` on an xhigh-capable profile, not to `high` as the updated docs and test name claim. Put `adaptive` below `xhigh` if `high` is intended, or update the docs and tests for the xhigh-capable case.
  - confidence: 0.82
