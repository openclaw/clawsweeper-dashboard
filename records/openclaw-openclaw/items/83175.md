## Summary

Keep open: the reported current-main bug is plausible, but this PR cannot merge as-is because it can skip the newest assistant turn when that turn has no visible text and it lacks required real behavior proof.

## What This Changes

The PR updates delivery-mirror transcript dedupe to compare normalized visible assistant text and adds regression tests for whitespace and phased final answers.

## Best Possible Solution

Keep the visible-text/whitespace-normalized comparison, but preserve the first-assistant-turn boundary by aborting when the newest assistant has no visible text, add a regression for that case, and include real delivery-path proof.

## Reproduction Assessment

Yes at source level: current main compares all assistant text exactly, so phased final-answer and whitespace variants can miss dedupe. I did not run a live channel repro because this review is read-only.

## Solution Assessment

No. The visible-text normalization is the right direction, but the implementation should stop at the newest assistant turn even when it has no visible text instead of scanning back to older turns.
