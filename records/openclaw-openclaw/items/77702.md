## Summary

Current main still has a source-reproducible Telegram bug: preview finalization checks media presence before `MEDIA:` directives in the segment text are normalized, so a final reply can be edited/sent as literal text instead of reaching the media delivery path.

## What This Changes

The issue requests that Telegram direct-chat replies containing local-path `MEDIA:` directive lines be delivered as attachments instead of literal text.

## Best Possible Solution

Normalize or parse directive-bearing segment text before Telegram decides a final answer is text-only preview-editable, then keep the existing media loader/file-read checks as the attachment boundary.

## Reproduction Assessment

Yes from source, though not from a live run: with an active preview and a final answer segment containing only `MEDIA:/path`, the lane deliverer computes `hasMedia` from the unparsed payload and can return `preview-finalized` before `deliverReplies` parses directives.

## Solution Assessment

Yes: the narrow fix is to make Telegram's preview/finalization media check use the same normalized reply payload that final delivery uses, rather than changing Telegram upload behavior or adding configuration.
