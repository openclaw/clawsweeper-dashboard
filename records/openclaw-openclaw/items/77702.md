## Summary

Keep open: current `main` still has the reported Telegram finalization bug at source level. The Telegram preview lane decides whether a final answer has media before inline `MEDIA:` text is normalized, while the documented/shared delivery path says those directives should become media attachments.

## What This Changes

The issue asks Telegram direct replies to convert local-path `MEDIA:` directives into photo/document attachments and remove the directive lines from visible text.

## Best Possible Solution

Normalize the final lane segment through the shared outbound payload plan, or an equivalent directive parser, before Telegram decides a reply is text-only preview-finalizable, then keep the existing media loader and file-read policy as the attachment boundary.

## Reproduction Assessment

Yes, at source level: a final answer segment containing only `MEDIA:/path` reaches `deliverLaneText`, whose media check reads only `payload.mediaUrl(s)` and can finalize the preview before `deliverReplies` parses directives.

## Solution Assessment

Yes: the narrow maintainable fix is to make the Telegram preview/finalization media check use the same normalized payload shape as final delivery, instead of changing Telegram upload behavior or adding configuration.
