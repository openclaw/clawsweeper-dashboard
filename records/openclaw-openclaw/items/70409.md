## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain staged paths before writing maintenance files:** `extensions/memory-core/src/dreaming-maintenance.ts:1122-1128`
  - body: `writeFileChange` joins `change.path` from staged or rollback JSON directly under `workspaceDir` and writes it. Because those artifacts are workspace-writable, normalize and constrain every changed path to the expected maintenance files before hashing or writing.
  - confidence: 0.9
- **[P1] Persist rollback metadata before staged writes:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1218`
  - body: The apply loop writes each staged file before saving `last-apply.json`. A crash after a file write but before the record write leaves the workspace modified with no rollback source, so build and persist the restore record before applying file changes.
  - confidence: 0.95
- **[P1] Pass daily signal allowPaths when recording recalls:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: `recordShortTermRecalls` filters non-date memory paths unless `allowPaths` includes them, but this call omits the configured daily-signal paths. Files such as `memory/daily-log.md` can stop refreshing after fallback has already marked the path tracked.
  - confidence: 0.9
