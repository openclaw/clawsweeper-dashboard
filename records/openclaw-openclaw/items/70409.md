  - body: Existing Dreaming configs do not have `maintenance.autoApply`, so resolving that missing option to `false` silently changes upgraded users from automatic `MEMORY.md` promotion to stage-only artifacts.   - confidence: 0.91
- **[P1] Reject daily signal paths outside the workspace:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` accepts arbitrary relative strings and this path is later joined to `workspaceDir` for `stat`/`readFile`; `../` entries can ingest unintended local files into memory evidence. Resolve the path and reject anything outside the workspace before reading it.
  - confidence: 0.9
- **[P1] Keep staged file writes inside the workspace:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: `writeFileChange` trusts `change.path` from staged plan and rollback JSON, so an edited artifact can write or unlink outside the workspace during apply or rollback. Resolve staged paths against the workspace and reject escaped or absolute targets before any read, write, or unlink.
  - confidence: 0.9
- **[P1] Persist rollback metadata before mutating files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1217`
  - body: `applyDreamingMaintenance` writes the managed files before `last-apply.json` exists. A crash in that window leaves `MEMORY.md` or `current.json` changed with no rollback record, and a later apply can conflict on the changed SHA.
  - confidence: 0.88
- **[P2] Pass allowed paths for collected daily signals:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: Configured files that flow through the collected/explicit daily batches call `recordShortTermRecalls` without `allowPaths`, but that recorder filters non-date memory paths unless they are whitelisted. After the fallback seeds the first version and marks the path tracked, later edits to `memory/daily-log.md` can be ignored.
  - confidence: 0.86
