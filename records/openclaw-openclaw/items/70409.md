## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep stage-only from becoming the default upgrade behavior:** `src/memory-host-sdk/dreaming.ts:21`
  - body: Current main and the public Dreaming docs say the deep phase writes promoted candidates into `MEMORY.md`; setting the new maintenance default to stage-only means existing enabled users will silently stop getting those writes until they discover `/dreaming apply` or configure auto-apply. Please preserve the current direct-write behavior as the default, add an explicit opt-in/migration path, or get a maintainer-visible product decision plus upgrade proof for the breaking default change.
  - confidence: 0.9
- **[P1] Record rollback data before mutating workspace files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1217`
  - body: `applyDreamingMaintenance` builds `restoreFiles` and immediately writes each file change, but it does not persist `last-apply.json` until after the writes finish. A crash after `MEMORY.md` or `current.json` is written leaves the workspace modified with no rollback record, which breaks the central apply/rollback guarantee.
  - confidence: 0.93
- **[P1] Reject staged paths that escape the workspace:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: The staged plan and rollback record are JSON files in the workspace, but `writeFileChange` joins `workspaceDir` with `change.path` without checking the resolved path. If a staged plan or rollback record contains `../` or an absolute-looking path, `/dreaming apply` or rollback can write outside the workspace; resolve and validate every staged path before reading or writing it.
  - confidence: 0.9
- **[P1] Pass allowPaths when recording configured daily signals:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: `recordShortTermRecalls` drops non-date files unless they are whitelisted, but this ingestion call records configured files like `memory/daily-log.md` without `allowPaths`. The first sweep can be rescued by the fallback path, but after the path is tracked, later edits are skipped by the fallback and silently fail to refresh the short-term evidence used by deep maintenance.
  - confidence: 0.88
