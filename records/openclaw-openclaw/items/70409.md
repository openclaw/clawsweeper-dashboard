  - body: `writeFileChange` accepts paths from staged and rollback JSON, joins them with the workspace, and writes without rejecting traversal or unexpected targets. Normalize and bound every staged/rollback path to the approved maintenance surface before hashing or writing.
  - confidence: 0.9

  - body: `applyDreamingMaintenance` writes each staged file change before saving `last-apply.json`. A crash after a file write but before the record write leaves changed workspace files with no rollback source, so persist the restore record before applying file changes.
  - confidence: 0.95

  - body: This `recordShortTermRecalls` call omits the configured daily-signal whitelist. Non-date files such as `memory/daily-log.md` are filtered unless allowed, so later edits can stop refreshing the evidence chain after fallback has already tracked the path.
  - confidence: 0.9
