- **[P1] Reject daily signal paths outside the workspace:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` only strips `./` and normalizes slashes before the ingestion path joins it with `workspaceDir`.   - confidence: 0.91
- **[P1] Reject staged apply paths outside the workspace:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: Staged plans and rollback records are mutable JSON under the workspace, but `writeFileChange` joins `change.path` directly with `workspaceDir`. A tampered path can write or unlink outside the workspace during apply or rollback, so apply/rollback must resolve and reject paths outside the allowed root.
  - confidence: 0.9
- **[P1] Write the rollback record before applying files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1217`
  - body: `applyDreamingMaintenance` writes each file change before `last-apply.json` is persisted. If the process exits after a file write but before line 1242, the workspace can be modified with no rollback record and a later apply will conflict on the changed hash.
  - confidence: 0.88
- **[P1] Pass allowed paths for collected daily signals:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: The collected configured-file batches call `recordShortTermRecalls` without `allowPaths`, but `recordShortTermRecalls` drops non-date files unless they are in that allowlist. After the fallback path has tracked `memory/daily-log.md`, later edits can be silently ignored by this main ingestion loop.
  - confidence: 0.86
