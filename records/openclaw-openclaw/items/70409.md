  - body: Current main and v2026.6.9 write deep promotions into `MEMORY.md` during the managed sweep.   - confidence: 0.9
- **[P1] Constrain staged maintenance paths:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: Staged plans and rollback records live in workspace JSON, but their `path` fields are joined directly with `workspaceDir` before apply/rollback reads and writes. A tampered plan can use `../` to touch files outside the workspace; resolve and reject paths outside the allowed workspace roots before every read, write, and unlink.
  - confidence: 0.93
- **[P1] Persist rollback state before applying files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1217`
  - body: The apply loop writes each file before `last-apply.json` is persisted. If the process exits after a write but before the record is saved, the workspace is modified with no rollback path and a later apply can conflict on the changed SHA; write the rollback record first, then apply file changes.
  - confidence: 0.89
- **[P1] Reject escaping daily signal paths:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` is normalized by trimming and slash cleanup only, then `path.join(workspaceDir, signalFile.relativePath)` is used for stat/read. A configured `../` path can ingest unintended local file contents into memory evidence; validate configured paths stay under the workspace before collection and fallback reads.
  - confidence: 0.9
- **[P2] Pass allowed paths for configured daily signals:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: `recordShortTermRecalls` filters non-date files unless `allowPaths` includes them, but this main configured-file ingestion call omits that allowlist. After the fallback records `memory/daily-log.md` once, later edits are skipped by the fallback and ignored here, so deep maintenance can miss updated daily-signal content.
