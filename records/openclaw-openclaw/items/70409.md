## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing Dreaming writes on upgrade:** `src/memory-host-sdk/dreaming.ts:21`
  - body: Current main and docs say the deep sweep writes durable promotions to `MEMORY.md`, but this new default makes maintenance stage-only unless users discover and enable `autoApply` or run apply manually. Existing Dreaming users would silently lose the automatic durable-write behavior after upgrade, so preserve the existing default or add an explicit maintainer-approved migration/compatibility plan.
  - confidence: 0.9
- **[P1] Reject staged paths outside the workspace:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: `writeFileChange` and the apply/rollback checks join `change.path` from mutable staged JSON with `workspaceDir` without verifying the resolved path remains inside the workspace. A tampered staged plan or last-apply record can read, write, or unlink files outside the workspace; resolve and reject paths that escape the allowed root before use.
  - confidence: 0.92
- **[P1] Reject daily signal paths outside the workspace:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` entries are normalized as strings but not constrained before `stat`/`readFile`, so a configured `../` path can ingest arbitrary local files into memory evidence. Keep configured signal files under the workspace, preferably under the intended memory area, before reading them.
  - confidence: 0.88
- **[P1] Pass allowPaths for configured daily batches:** `extensions/memory-core/src/dreaming-phases.ts:1263`
  - body: The main collected-batches path records configured files like `memory/daily-log.md` without `allowPaths`, but `recordShortTermRecalls` now rejects non-date files unless they are explicitly allowed. The fallback can seed the first version, then tracked paths are skipped and later edits go stale, so pass the configured path through this call or consolidate the ingestion path.
  - confidence: 0.86
- **[P1] Record rollback data before writing files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214`
  - body: `applyDreamingMaintenance` writes each file change before persisting `last-apply.json`. A crash after writing `MEMORY.md` or `current.json` but before the record is saved leaves the workspace changed with no rollback path, so persist the restore record first or otherwise make the apply operation crash-recoverable.
  - confidence: 0.86
