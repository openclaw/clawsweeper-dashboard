  - body: Current main writes deep Dreaming promotions to `MEMORY.md` when Dreaming is enabled; this PR defaults the new maintenance mode to stage-only, so existing users can stop receiving durable memory updates until they discover and run `/dreaming apply`.   - confidence: 0.89
- **[P1] Write the rollback record before applying files:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1218`
  - body: `applyDreamingMaintenance` writes each planned file before persisting `last-apply.json`. A crash between those steps can leave `MEMORY.md` or `current.json` changed with no rollback record, so the restore record should be written before mutating the durable files.
  - confidence: 0.93
- **[P1] Constrain staged maintenance paths:** `extensions/memory-core/src/dreaming-maintenance.ts:1121-1128`
  - body: `writeFileChange` joins `change.path` from staged JSON directly with `workspaceDir`, and apply/rollback use the same untrusted paths for conflict checks and writes. Resolve and verify these paths stay inside the intended workspace before any read, write, or unlink.
  - confidence: 0.92
- **[P1] Constrain configured daily-signal paths:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` accepts arbitrary relative strings, then stats and reads them under `workspaceDir` without rejecting `..` traversal. A config entry can ingest unintended local file contents into memory evidence, so normalize to an allowed workspace-relative path before filesystem access.
  - confidence: 0.88
- **[P1] Pass allowPaths for configured daily signals:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: The main configured-file ingestion path calls `recordShortTermRecalls` without `allowPaths`, but `resolveShortTermKind` only accepts non-date paths like `memory/daily-log.md` when explicitly allowed. After the fallback has seeded the file once, later edits can be skipped or fail to become tracked daily-signal evidence.
