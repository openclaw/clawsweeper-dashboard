## Summary

Keep open. Current main still has the Control UI CSS/renderer mismatch described in the report: action buttons can render inside assistant message bubbles, but the bubble never receives the existing padding class that reserves space for those buttons.

## What This Changes

The issue requests that WebChat/Control UI message action icons stop permanently overlaying or truncating message text after the 2026.3.31 upgrade.

## Best Possible Solution

Apply the existing Control UI CSS contract consistently: compute the action state before building the bubble class list, add `has-copy` whenever copy/expand actions render, and cover it with a focused grouped-render test.

## Reproduction Assessment

Yes. Static source reproduction is high-confidence: assistant markdown messages render `.chat-bubble-actions`, while the parent bubble never receives `has-copy`, so the padding rule that prevents overlap cannot apply.

## Solution Assessment

Yes. The narrowest maintainable fix is to wire the existing `has-copy` class to the existing `hasActions` condition in the Control UI chat renderer, rather than changing Gateway/WebChat protocol behavior.
