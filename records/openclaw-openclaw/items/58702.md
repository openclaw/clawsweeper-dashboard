## Summary

Keep this issue open. Current main still renders chat bubble action buttons from a separate action condition while the existing padding rule depends on a `has-copy` class that the renderer does not apply, and an open linked PR is already the active implementation path.

## What This Changes

The issue reports WebChat/Control UI message text being covered or truncated by copy/expand action icons after upgrading to 2026.3.31.

## Best Possible Solution

Land or replace the linked PR with a fix that makes the rendered action state and padding selector match, then add focused grouped-render or browser coverage for action-bearing bubbles.

## Reproduction Assessment

Yes, at source level: assistant messages can render `.chat-bubble-actions` while the bubble class list never includes `has-copy`, and the CSS only reserves action-button space for `.chat-bubble.has-copy`. I did not run a live browser reproduction in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable direction is to fix the renderer/CSS selector contract in the Control UI chat bubble path and validate the hover/coarse-pointer behavior. This issue should remain open while https://github.com/openclaw/openclaw/pull/77417 or a replacement fix is reviewed.
