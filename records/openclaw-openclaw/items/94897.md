Full review comments:

- **[P1] Probe the local tunnel host, not the SSH target host:** `src/infra/ssh-tunnel.ts:125`
  - body: `parsed.host` is the remote machine from the SSH target, but `ensurePortAvailable` uses this value as the local address passed to `server.listen`. For a normal target like `example.com`, this asks the local process to bind a foreign address and can throw before the SSH process starts; the probe should match the `ssh -L`/connect path on `127.0.0.1`.
  - confidence: 0.94
- **[P2] Preserve the busy-port fallback:** `src/infra/ssh-tunnel.ts:125`
  - body: Once the probe is scoped and detects the occupied loopback port, `ensurePortAvailable` throws `PortInUseError`, not an errno-shaped `EADDRINUSE`. This branch leaves the fallback guard unchanged, so the preferred-port collision will still throw instead of selecting an ephemeral port.
  - confidence: 0.9
