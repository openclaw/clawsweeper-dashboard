Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Bind the preflight to the local tunnel interface:** `src/infra/ssh-tunnel.ts:122`
  - body: `parsed.host` is the remote SSH target, but `ensurePortAvailable` uses its host argument as a local bind host. For a normal target such as `user@example.com`, this asks Node to bind the local preflight socket to a non-local hostname before the SSH tunnel starts, so startup can fail instead of falling back or probing the loopback port the tunnel actually owns. The host should match the `-L` local interface, `127.0.0.1`, and the fallback should handle the domain `PortInUseError`.
  - confidence: 0.95
