## Summary

Keep open: current main has a narrow reset transcript replay helper, but that helper already existed in v2026.6.10 and current source does not show the reported abort-settle/takeover path using that lifecycle; the remaining work overlaps existing continuity, takeover, and fallback trackers without being fully owned by one of them.

## What This Changes

The issue requests preserving bounded conversation context when a provider-error abort/takeover resets an embedded session instead of starting the replacement session blank.

## Best Possible Solution

Define one maintainer-approved recovery path for provider-failure abort/takeover that preserves bounded context on the replacement session without crossing auth, channel, thread, or clean-reset boundaries, then cover that exact reset path with regression tests.
