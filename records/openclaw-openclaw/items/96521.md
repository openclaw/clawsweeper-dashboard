## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require authoritative kind before group routing:** `extensions/mattermost/src/session-route.ts:35-36`
  - body: When `peekMattermostChannelKind(rawId)` misses, this falls back to `resolvedTarget.kind`. Current Mattermost directory entries mark both public `O` and private `P` channels as `group`, while inbound `O` still maps to `channel`, so a cold public-channel delivery can be keyed under the group session namespace. Only switch to `group` when an authoritative channel-type lookup/cache proves `P` or `G`, or make directory resolution carry the real O/P distinction.
  - confidence: 0.94
- **[P2] Scope or expire the channel-kind cache:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:13-25`
  - body: The new cache is process-wide, keyed only by channel id, and never expires, while the Mattermost channel lookup feeding it is account/resource-specific and TTL-bound. A reused id across accounts/servers or a public/private conversion can keep returning the wrong chat type after the authoritative lookup would have changed.
  - confidence: 0.88
- **[P2] Use SDK-shaped resolved target fixtures:** `extensions/mattermost/src/session-route.test.ts:99`
  - body: These tests pass `resolvedTarget: { kind, id }`, but the SDK route contract carries `to`, `kind`, and `source`, and the core resolved target also includes `resolutionSource`. Use realistic fixtures so regression coverage matches production input and test typechecking catches contract drift.
  - confidence: 0.9
