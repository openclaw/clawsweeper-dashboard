- **[P1] Keep Mattermost send targets as channel ids:** `extensions/mattermost/src/session-route.ts:42`
  - body: For resolved group targets this now returns `to: group:${rawId}`. The heartbeat route refinement copies `route.to` into the outbound target, but Mattermost sending only treats `channel:<id>` as a channel id; `group:<id>` falls through as a channel name. Keep `to` as `channel:${rawId}` and use `group` only for session identity unless the sender is explicitly extended and proven.
  - confidence: 0.92

  - body: The new process-wide map never expires and has no account scope, while the channel info that feeds it is a TTL-bound account resource. A stale or cross-account entry can keep returning the wrong chat type after the authoritative lookup would have changed. Tie this cache to the same lifecycle/account boundary or avoid a global map for classification.
  - confidence: 0.81
