- **[P1] Keep group routes in the group session peer:** `extensions/mattermost/src/session-route.ts:40`
  - body: This only changes `chatType` when `resolvedKind` is `group`, but `buildChannelOutboundSessionRoute` builds the session key from `peer.kind`.   - body: The new tests pass `resolvedTarget: { kind: "group" }`, but the route parameter contract requires the resolved target shape to include the normalized target and source. Use realistic complete fixtures so check-test-types can pass and the tests exercise the real resolver contract.
  - confidence: 0.84
- **[P2] Bound the channel-kind cache:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:7`
  - body: This process-wide map never expires and is keyed only by channel id, while the channel info that feeds it is per-resource and TTL-bound. A stale or cross-account entry can keep returning the wrong chat type after the authoritative lookup would no longer do so.
  - confidence: 0.87
