Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope the channel-kind cache to the account lookup:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:13-25`
  - body: The new cache is process-wide and keyed only by `channelId`, while Mattermost accounts can point at different servers/tokens and the existing opaque-target cache scopes by baseUrl/token/id. A remembered private channel for one account can make another account's same id resolve as `group`, and stale values never expire, which can split or misroute sessions.
  - confidence: 0.88
- **[P2] Seed the route test's authoritative kind source:** `extensions/mattermost/src/session-route.test.ts:99`
  - body: `resolveMattermostOutboundSessionRoute` now ignores `resolvedTarget.kind` unless the new cache has a remembered `group`, but these regression tests pass only `resolvedTarget.kind: "group"`. With an empty cache, the route returns `chatType: "channel"`, so the test should seed the authoritative cache or the implementation should carry the real channel type through the route contract.
  - confidence: 0.94
- **[P2] Use SDK-shaped resolved target fixtures:** `extensions/mattermost/src/session-route.test.ts:99`
  - body: The route contract requires `resolvedTarget` to include `to`, `kind`, and `source`; this fixture instead supplies `{ kind, id }`. Use a production-shaped resolved target so test typechecking catches contract drift and the regression covers the real caller payload.
  - confidence: 0.9
