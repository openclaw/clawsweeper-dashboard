Full review comments:

- **[P1] Keep public Mattermost channels out of group sessions:** `extensions/mattermost/src/session-route.ts:23`
  - body: `resolvedTarget.kind === "group"` is not an authoritative private-channel signal today: Mattermost directory resolution includes both public `O` and private `P` channels as group entries, and cold `#name`/directory paths can carry that group kind.   - confidence: 0.94
- **[P2] Scope or expire the channel-kind cache:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:7`
  - body: The new cache is process-wide, keyed only by channel id, and never expires, while the monitor lookup feeding it is per-resource and TTL-bound. A stale or cross-account/cross-server entry can keep returning the wrong chat type after the authoritative lookup would no longer do so.
  - confidence: 0.86
- **[P2] Use complete resolved target fixtures:** `extensions/mattermost/src/session-route.test.ts:99`
  - body: The new session-route tests pass `resolvedTarget: { kind, id }`, but the SDK route contract carries `to`, `kind`, and `source` and has no `id` field. Use realistic fixtures so regression coverage matches the production route input.
  - confidence: 0.9
