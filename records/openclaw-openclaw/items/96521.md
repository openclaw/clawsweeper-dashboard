  - body: This changes `chatType` to `group`, but the route still passes `peer.kind: "channel"`. `buildChannelOutboundSessionRoute` derives the base session key from `peer.kind`, while inbound Mattermost private channels route with the trusted `group` peer, so the outbound path can still land in `mattermost:channel:<id>` and leave the split-session bug unresolved.
