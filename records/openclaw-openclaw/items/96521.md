Full review comments:

- **[P1] Keep public Mattermost channels out of group sessions:** `extensions/mattermost/src/session-route.ts:23`
  - body: This treats `resolvedTarget.kind === "group"` as authoritative chat type, but current Mattermost directory entries use `kind: "group"` for both public `O` and private `P` channels. A directory-resolved public channel can therefore get a `group` peer/session while inbound `O` still maps to `channel`, splitting public-channel state.
  - confidence: 0.92
- **[P2] Scope or expire the channel-kind cache:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:7`
  - body: The new cache is process-wide, keyed only by channel id, and never expires, while the monitor lookup feeding it is per-resource and TTL-bound. A stale or cross-account entry can keep returning the wrong chat type after the authoritative lookup would no longer do so.
  - confidence: 0.86
- **[P2] Use complete resolved target fixtures:** `extensions/mattermost/src/session-route.test.ts:99`
  - body: The new tests pass `resolvedTarget: { kind: "group", id: ... }`, but the SDK route contract requires `to`, `kind`, and `source` and has no `id` field. Use realistic fixtures so test type checking and the production route contract stay aligned.
  - confidence: 0.95
