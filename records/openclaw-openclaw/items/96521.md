## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep Mattermost delivery targets as channel ids:** `extensions/mattermost/src/session-route.ts:42`
  - body: For resolved group targets this now returns `to: group:${rawId}`. The heartbeat route refinement copies `route.to` into the outbound target, but `parseMattermostTarget` only supports `channel:`, `user:`, `mattermost:`, `@`, `#`, or bare ids; `group:<id>` is treated as a channel name instead of the channel id. Keep `to` as `channel:${rawId}` and use `group` only for session identity unless the sender is explicitly extended and proven.
  - confidence: 0.91
- **[P2] Bound the Mattermost channel-kind cache:** `extensions/mattermost/src/mattermost/channel-kind-store.ts:7-15`
  - body: The new process-wide map never expires and has no account scope, while the channel info that feeds it is a TTL-bound account resource. A stale or cross-account entry can keep returning the wrong chat type after the authoritative lookup would have changed. Tie this cache to the same lifecycle/account boundary or avoid a global map for classification.
  - confidence: 0.78
