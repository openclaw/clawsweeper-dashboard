## Summary

Keep open for human review: the PR targets a real gateway-agent lock-safety problem, but current main already preserves timeout fallback with a fresh session and this branch would turn that documented recovery path into a hard failure while closed-transport errors still lack request-phase semantics. The external PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw agent` gateway-error classifier that throws instead of embedded fallback for timeout, request-timeout, and closed errors, plus focused tests.

## Best Possible Solution

Preserve fresh-session timeout fallback; if fail-closed close handling is desired, add typed request-phase metadata so only post-request or accepted ambiguous closes fail closed, with focused tests and redacted real CLI/log proof.

## Reproduction Assessment

Yes, at source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert fresh fallback, normal fallback, or fail-closed behavior; I did not run tests during this read-only review.

## Solution Assessment

No. The proposed branch is broader than the maintainable fix because it hard-fails current timeout fallback and uses message matching without request-phase semantics for gateway closes.
