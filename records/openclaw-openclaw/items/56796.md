## Summary

Keep open: the PR targets a real gateway fallback risk, but the current branch is not safe to merge because current main already preserves timeout fallback with a fresh session and this patch would hard-fail those cases while also treating every gateway close as ambiguous.

## What This Changes

The PR adds a message-substring fail-closed classifier to `openclaw agent` gateway fallback handling and tests that timeout, request-timeout, and closed errors skip embedded fallback.

## Best Possible Solution

Keep the fresh-session timeout fallback and implement typed, phase-aware gateway transport classification so only closes after an agent request may be active fail closed, with focused regression tests and real CLI/log proof.

## Reproduction Assessment

Yes, at source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert fresh fallback, normal fallback, or fail-closed behavior; I did not run tests during this read-only review.

## Solution Assessment

No. The proposed branch is broader than the maintainable fix because it hard-fails current timeout fallback and uses message matching without request-phase semantics for gateway closes.
