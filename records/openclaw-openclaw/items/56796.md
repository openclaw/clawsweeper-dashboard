## Summary

Keep open: the reliability problem is plausible, but this branch would now regress current main's fresh timeout fallback and fail closed for gateway closes that may happen before any agent request is active. It also lacks after-fix real behavior proof.

## What This Changes

The PR adds string-based fail-closed handling for `openclaw agent` gateway timeout, request-timeout, and closed errors, plus tests that embedded fallback is skipped for those messages.

## Best Possible Solution

Preserve current fresh-session timeout fallback, add typed phase-aware gateway transport classification for closes after an agent request may be active, and keep pre-request closes on recoverable fallback with regression tests.

## Reproduction Assessment

Yes, at source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert fresh fallback, normal fallback, or fail-closed behavior; I did not run tests during this read-only review.

## Solution Assessment

No. The current PR is broader than the maintainable fix because it uses message matching and treats all closes as ambiguous; the safer path is typed, phase-aware gateway transport classification.
