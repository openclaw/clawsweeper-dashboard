## Summary

Keep this PR open, but it should not merge as-is: the branch now conflicts with current main’s documented fresh-session timeout fallback and still lacks phase-aware close-error metadata plus contributor real behavior proof.

## What This Changes

The PR changes `openclaw agent` gateway fallback behavior so timeout, request-timeout, and closed gateway errors throw instead of falling back to embedded execution, with focused tests for those cases.

## Best Possible Solution

Preserve fresh-session timeout fallback, and only add fail-closed close handling if the gateway call can carry structured request-sent or request-accepted metadata.

## Reproduction Assessment

Yes, at source level: current tests can mock timeout and closed transport errors and assert fallback versus fail-closed behavior. I did not run tests because this review is read-only.

## Solution Assessment

No. The proposed branch is broader than the maintainable fix because it hard-fails current timeout fallback and treats every gateway close as ambiguous without typed request-phase semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: Current main treats typed gateway timeouts and `gateway request timeout for agent` as fresh-session fallback cases. This branch matches those strings first and throws, removing the documented `gateway-fallback-*` path that avoids transcript-lock contention without turning the CLI run into a hard failure.
  - confidence: 0.88
