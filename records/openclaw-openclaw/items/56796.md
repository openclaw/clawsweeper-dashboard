## Summary

Keep open. The PR targets a real gateway-agent lock-contention failure, and current main now covers the timeout part with fresh fallback sessions, but the branch is not merge-ready because it fails closed for every `gateway closed` message instead of distinguishing whether the agent request was actually in flight.

## What This Changes

The PR branch adds a string-based fail-closed guard in `agentCliCommand` for gateway timeout, request-timeout, and closed error messages, plus focused `agent-via-gateway` tests for those cases.

## Best Possible Solution

Rebase or replace the branch with a phase-aware gateway-agent error policy: preserve the fresh-session timeout fallback, preserve embedded fallback for pre-request connection or handshake closes, and only fail closed or isolate fallback once the gateway-side agent request may actually still be active. Cover those phases in `agent-via-gateway` and `callGateway` tests.

## Reproduction Assessment

Yes. The existing unit seams can mock `callGateway` timeout, `gateway request timeout for agent`, pre-request `GatewayTransportError` closes, and pending-request close errors, then assert whether `agentCommand` uses a fresh fallback session, normal fallback, or no fallback.

## Solution Assessment

No. The PR's message-substring classifier is not the best current solution because main already has structured timeout handling and fresh fallback sessions; the safer fix is explicit request-phase classification for closed transports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Distinguish pre-request gateway closes:** `src/commands/agent-via-gateway.ts:189`
  - body: The new classifier treats every `gateway closed` message as an in-flight remote agent and throws before embedded fallback. `callGateway` can also receive `onClose` before `onHelloOk` sends the agent request, so startup or handshake closes would lose recoverable fallback even though no remote turn is active. Please classify the request phase instead of matching all close messages.
  - confidence: 0.86
