## Summary

Keep this PR open, but the current patch should not merge as-is: current main already preserves timeout fallback with a fresh session, while this branch converts those timeouts into hard failures and still lacks request-phase data for `gateway closed` errors.

## What This Changes

The PR adds an `openclaw agent` fail-closed classifier for gateway timeout/request-timeout/closed error messages and focused tests for those cases.

## Best Possible Solution

Keep the fresh-session timeout fallback; if fail-closed closed-transport behavior is still desired, add typed request-phase gateway classification so only post-request ambiguous closes fail closed, with focused tests and redacted real CLI/log proof.

## Reproduction Assessment

Yes, at source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert fresh fallback, normal fallback, or fail-closed behavior; I did not run tests during this read-only review.

## Solution Assessment

No. The proposed branch is broader than the maintainable fix because it hard-fails current timeout fallback and uses message matching without request-phase semantics for gateway closes.
