## Summary

Keep open: current main already shipped a safer fresh-session timeout fallback, but this PR still proposes a broader fail-closed policy for Gateway timeout and close errors that conflicts with shipped fallback behavior and lacks inspectable real behavior proof.

## What This Changes

The PR adds string-matched fail-closed handling for `openclaw agent` Gateway timeout, request-timeout, and closed errors, plus focused command tests that expect embedded fallback to be skipped.

## Best Possible Solution

Preserve the shipped fresh-session timeout fallback, and add fail-closed handling only for close or accepted-run cases once Gateway errors carry request-phase or accepted-ownership metadata with focused tests, docs, and redacted real CLI proof.
