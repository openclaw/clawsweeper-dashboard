## Summary

Keep open. The PR targets a real reliability problem, but current main already shipped timeout isolation with a fresh `gateway-fallback-*` session, while this branch still broadly fails closed on every `gateway closed` message and can remove useful embedded fallback for closes that happen before an agent request is actually active.

## What This Changes

The PR branch makes `openclaw agent` throw without embedded fallback for gateway timeout, gateway request-timeout, and gateway-closed error messages, with focused tests for those fail-closed cases.

## Best Possible Solution

Keep the shipped fresh-session timeout fallback, then rebase or replace this branch only if maintainers want a phase-aware close policy: pre-request gateway closes should keep embedded fallback, while accepted or in-flight agent closes should fail closed or use isolated fallback metadata that cannot race the routed transcript.

## Reproduction Assessment

Yes. A focused unit path can mock `callGateway` to reject with a typed timeout, `gateway request timeout for agent`, a pre-request `closed` transport error, and a pending-request close, then assert whether `agentCommand` receives a fresh fallback session or is not called.

## Solution Assessment

No. The branch’s string classifier is not the best current solution because main now has structured transport errors and shipped fresh-session timeout fallback; the safer direction is explicit request-phase classification rather than matching every `gateway closed` message.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Distinguish pre-request gateway closes:** `src/commands/agent-via-gateway.ts:189`
  - body: The new classifier treats every `gateway closed` message as an in-flight remote agent and throws before embedded fallback. `callGateway` can receive `onClose` before `onHelloOk` sends the agent request, so startup or handshake closes would lose recoverable fallback even though no remote turn is active. Please classify the request phase instead of matching all close messages.
