## Summary

Keep open: the PR is a plausible reliability fix area, but current main already handles gateway timeouts with isolated embedded fallback and this branch would still fail closed for recoverable timeout and pre-request close cases. It also lacks required real behavior proof from the contributor.

## What This Changes

The PR changes `openclaw agent` to throw instead of using embedded fallback for gateway timeout, request-timeout, and closed errors, with focused tests for those fail-closed cases.

## Best Possible Solution

Preserve current fresh-session timeout fallback, add structured phase-aware gateway transport classification for closes after an agent request may be active, and keep pre-request closes on recoverable fallback with regression tests.

## Reproduction Assessment

Yes, at the source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert whether embedded fallback uses a fresh session, normal fallback, or fail-closed behavior.

## Solution Assessment

No. The PR's substring fail-closed policy is broader than the current main contract; a maintainable fix should use typed, phase-aware gateway errors instead of message matching.
