## Summary

Keep open: the timeout-lock problem is now addressed on current main with fresh-session fallback, but this PR would replace that with a hard failure and still treats all gateway closes as ambiguous without request-phase evidence. The external PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds a message-substring classifier that makes `openclaw agent` throw instead of using embedded fallback for gateway timeout, request-timeout, and closed errors, with focused tests.

## Best Possible Solution

Keep the fresh-session timeout fallback and add typed, phase-aware gateway transport classification so only closes after an agent request may be active fail closed, with focused regression tests and real CLI/log proof.

## Reproduction Assessment

Yes, at source level. A focused unit path can mock timeout, request-timeout, pre-request close, and post-request close errors and assert fresh fallback, normal fallback, or fail-closed behavior; I did not run tests during this read-only review.

## Solution Assessment

No. The proposed branch is broader than the maintainable fix because it hard-fails current timeout fallback and uses message matching without request-phase semantics for gateway closes.
