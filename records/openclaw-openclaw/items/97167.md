## What This Changes

The PR defaults internal/WebChat-created message tools to `message_tool_only` when no mode is supplied, passes that mode to the tool description and outbound runner, and adds regression tests for the default and explicit `automatic` override.

## Best Possible Solution

Land one narrow `createMessageTool` defaulting fix after maintainer message-delivery review, then close the sibling candidate and linked issue after the selected branch merges.

## Reproduction Assessment

Yes. Current main leaves internal/WebChat-created message tools without the computed `message_tool_only` mode, while the existing sink requires that mode and target validation throws when the sink is skipped.

## Solution Assessment

Yes. Defaulting after effective current-channel resolution in `createMessageTool` is narrower than making all WebChat room events message-tool-only and preserves explicit `automatic` overrides plus ordinary WebChat final replies.
