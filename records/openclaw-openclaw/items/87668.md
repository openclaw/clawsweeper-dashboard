## Summary

Close: current main and v2026.6.6 implement the central plugin-facing subprocess env contribution need through the supported `resolve_exec_env` hook, with core filtering and exec-tool wiring, so the older BashSpawnHook-specific SDK shape is no longer necessary.

## What This Changes

Request to expose a plugin SDK seam for plugins to contribute filtered session-scoped environment variables to agent subprocess spawns.

## Best Possible Solution

Keep `resolve_exec_env` as the public plugin SDK seam for subprocess env contribution and track any future context-field additions as narrow follow-ups to that hook.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, and tests now show the supported `resolve_exec_env` seam exists and covers the spawn-time env contribution path for the agent `exec` tool.

## Solution Assessment

Yes. The shipped `resolve_exec_env` hook is a narrower and safer public SDK contract than exposing the internal `BashSpawnHook`, because filtering and host/audit behavior stay in core exec runtime.
