## Summary

Keep open. Current main still lacks the environment discovery surface, and this non-draft PR is the active implementation for the open maintainer-labeled API request; the remaining blocker is maintainer review of the public protocol shape, not cleanup closure.

## What This Changes

The PR adds typed read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through/types, `operator.read` scoping, protocol schemas/generated Swift models, docs, changelog, and focused tests while leaving create/delete unsupported.

## Best Possible Solution

Land one additive read-only environment discovery contract after the API owner settles the initial candidate vocabulary and managed-environment representation, keeping create/delete and per-run environment selection unsupported until provisioning semantics are designed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the SDK methods throw unsupported errors, tests assert that behavior, and the Gateway neither advertises nor registers `environments.*` handlers.

## Solution Assessment

Yes, pending maintainer API approval. The PR follows the narrow additive SDK/Gateway/protocol/scope pattern for read-only discovery; the safer alternative is not a code rewrite, but resolving the managed-candidate and capability-vocabulary question before merge.
