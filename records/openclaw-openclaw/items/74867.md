## Summary

Keep open. Current main still lacks the requested SDK/Gateway environment discovery APIs, and this open PR is the active implementation for the maintainer-authored request #74708. The diff is coherent and I found no discrete blocking bug, but the remaining scope/vocabulary question for managed candidates and exact-head CI completion still need maintainer handling before merge.

## What This Changes

The PR adds typed read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through/types, `operator.read` scope classification, protocol schemas/generated Swift models, docs, changelog, and focused tests while keeping create/delete unsupported.

## Best Possible Solution

Land one additive environment discovery contract after maintainer API review: typed schemas, Gateway handlers, method advertisement, least-privilege read scoping, SDK/native model/docs/changelog updates, and tests, while keeping create/delete and per-run environment selection unsupported until provisioning semantics are designed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the SDK methods explicitly throw unsupported errors, the SDK test asserts that behavior, and the Gateway neither advertises nor registers `environments.*` handlers.

## Solution Assessment

Yes, with maintainer API approval. The PR is a narrow additive implementation for read-only discovery, but the environment type/capability vocabulary and managed-candidate representation should be settled before merge.
