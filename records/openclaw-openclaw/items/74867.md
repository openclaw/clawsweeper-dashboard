## Summary

Keep open. Current main still lacks the SDK/Gateway environment discovery surface, and the PR is a focused implementation for the open maintainer request, but the public API vocabulary and managed-candidate shape still need maintainer API approval before merge.

## What This Changes

The branch adds read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through/types, `operator.read` scoping, protocol schemas/generated Swift models, docs, changelog, and focused tests.

## Best Possible Solution

Land an additive read-only discovery contract once the API owner settles the initial environment summary vocabulary, while keeping create/delete/provisioning and per-run environment selection unsupported until those semantics are designed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: SDK `list` and `status` throw unsupported errors, docs/tests mark them unsupported, and Gateway discovery/handler registration has no `environments.*` methods.

## Solution Assessment

Yes, pending maintainer API approval. The PR follows the narrow typed protocol, handler, SDK, scoping, docs, and generated-model pattern; the safer alternative is not a mechanical rewrite but an explicit decision on managed-candidate representation and capability names.
