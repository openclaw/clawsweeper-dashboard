## Summary

Keep open. Current main still lacks SDK/Gateway environment discovery, and this PR is the active implementation for the maintainer-requested #74708 work, but the new public environment summary vocabulary and managed-candidate boundary still need maintainer API review before merge.

## What This Changes

This PR adds read-only Gateway `environments.list` and `environments.status` RPCs with SDK call-through/types, protocol schemas/generated Swift models, `operator.read` scoping, docs/changelog updates, and focused tests while leaving create/delete unsupported.

## Best Possible Solution

Land one additive read-only discovery contract after the API owner confirms the initial environment summary fields, capability vocabulary, and managed-environment representation, while keeping create/delete and per-run environment selection unsupported until provisioning is designed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: SDK `list` and `status` throw unsupported errors, docs mark them unsupported, and Gateway method advertisement/handler registration has no `environments.*` surface.

## Solution Assessment

Yes, pending maintainer API approval. The PR follows the narrow additive Gateway protocol/handler/SDK/scoping pattern for read-only metadata; the safer alternative is to settle vocabulary and managed-candidate representation before merge, not to queue a mechanical rewrite.
