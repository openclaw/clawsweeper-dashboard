## Summary

Keep this open: current main and v2026.6.8 still lack Discord reconnect-triggered outbound delivery recovery, and the linked PR is open candidate work rather than merged implementation provenance.

## What This Changes

The issue requests Discord outbound delivery queueing or retry when the Discord gateway reconnects so messages attempted during short WebSocket disconnect windows are not lost.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94320 with a bounded Discord-owned reconnect drain that reuses the shared delivery queue helper and avoids adding a new config surface.

## Reproduction Assessment

Yes, at source level: current main has durable outbound queue support but the Discord lifecycle never drains Discord-matched pending entries on reconnect, unlike Telegram and WhatsApp. I did not run a live Discord reconnect reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is a Discord-owned reconnect drain using the existing shared delivery queue runtime; adding `channels.discord.delivery.queue` or another new config key is not needed for the core fix.
