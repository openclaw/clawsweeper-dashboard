## Summary

Keep this PR open: current main and the latest shipped release still lack the requested OpenResponses internal tool-call SSE output, but the branch is not merge-ready because it streams raw tool results without a trust/redaction gate and the external contributor has not supplied real after-fix runtime proof. Earlier output-index and completed-snapshot review comments appear addressed in the current head tests and code.

## What This Changes

The PR adds internal tool start/result items to OpenResponses SSE, a non-enumerable raw tool-result side channel, `function_call_output` output schema support, AI SDK input normalization, and gateway/agent-event tests.

## Best Possible Solution

Land this PR or an equivalent Gateway change after defaulting OpenResponses internal tool outputs to sanitized results unless raw output is explicitly authorized, with docs/changelog coverage for the chosen contract and redacted real runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection gives high confidence that current main and `v2026.5.7` do not emit internal tool-call `function_call` or `function_call_output` items in the OpenResponses SSE stream.

## Solution Assessment

No, not as currently proposed. The Gateway SSE listener is the right place for this capability, but raw tool results should be sanitized by default or gated behind an explicit trust contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw tool output before streaming it:** `src/gateway/openresponses-http.ts:1082-1084`
  - body: The `/v1/responses` stream now prefers `getAgentEventToolRawResult(evt)` over the sanitized `evt.data.result` for every internal tool result. Current tool-event code emits a redacted result, and this endpoint is reachable through the OpenAI-compatible HTTP `chat.send` surface, so this can stream API keys, bearer tokens, image payload data, or other raw tool output to callers that previously received sanitized event data. Use the sanitized result by default or require an explicit full-trust gate for raw output.
  - confidence: 0.88

## Security Review

Status: needs_attention
