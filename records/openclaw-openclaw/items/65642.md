## Summary

Keep open: current main still lacks the requested OpenResponses internal tool-event SSE support, but this PR is not safe to merge as-is because it streams raw tool results without a redaction/trust gate and still lacks real after-fix behavior proof.

## What This Changes

The branch adds OpenResponses SSE items for internal tool start/result events, a raw tool-result side channel, `function_call_output` output schema support, AI SDK-compatible input normalization, and gateway/agent-event tests.

## Best Possible Solution

Land this PR or an equivalent Gateway change after defaulting internal tool outputs to sanitized results unless a caller is explicitly allowed to receive raw secret-capable payloads, with docs and changelog updated for the chosen contract.

## Reproduction Assessment

Not applicable as a bug reproduction; the feature gap is source-proven on current main because the OpenResponses SSE listener lacks an internal tool-event branch and the output schema lacks `function_call_output`.

## Solution Assessment

No, not as currently proposed. The Gateway SSE listener is the right place to shape these events, but preferring raw tool results over the existing sanitized event payload is not the safest maintainable contract without an explicit trust gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw tool output before streaming it:** `src/gateway/openresponses-http.ts:1082-1084`
  - body: The `/v1/responses` stream now prefers `getAgentEventToolRawResult(evt)` over the sanitized `evt.data.result` for every internal tool result. Current tool-event code redacts credentials before emitting `data.result`, and this endpoint is reachable through `chat.send`-scoped OpenAI-compatible HTTP auth, so this can stream API keys, bearer tokens, or large raw payloads to callers that previously only received redacted tool-event data. Use the sanitized result by default or gate raw output behind an explicit full-trust/secret-capable contract.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a raw tool-result side channel into OpenResponses SSE without a redaction or authorization gate.

Concerns:

- **[high] Raw tool results bypass existing redaction:** `src/gateway/openresponses-http.ts:1082`
  - body: Current main emits sanitized tool results on shared agent events, but the PR streams the non-enumerable raw result as `function_call_output`; this can expose API keys, bearer tokens, nested credential fields, image payload data, or oversized raw payloads to HTTP clients that should receive the redacted event value by default.
  - confidence: 0.88
