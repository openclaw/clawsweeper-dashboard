## Summary

Keep open. The feature remains useful and is still not implemented on current main, but this PR should not merge as-is because it streams raw internal tool results over the OpenResponses HTTP stream and still lacks contributor-provided real runtime proof; the earlier index/snapshot review points appear addressed in the latest head.

## What This Changes

The PR adds OpenResponses SSE output items for internal tool calls/results, adds function_call_output output schema support, normalizes response-shaped inputs, and carries raw tool results through agent events for HTTP streaming.

## Best Possible Solution

Implement internal tool SSE output using sanitized tool results by default, or add an explicit reviewed trust gate for raw output, then update tests/docs/changelog and require redacted real OpenResponses runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection gives high confidence that current main lacks the requested internal tool-call SSE output behavior.

## Solution Assessment

No, not as currently proposed. The Gateway SSE listener is the right implementation surface, but the PR should default to sanitized results or use an explicit authorized raw-output contract before streaming tool output to HTTP clients.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Gate raw tool output before streaming it:** `src/gateway/openresponses-http.ts:1082-1084`
