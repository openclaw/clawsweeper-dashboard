## Summary

Keep open. Current main and the latest release still have the shared embedded-run media promotion gap: pathless trusted image results are not promoted from original read tool args, and the issue is paired with the open implementation PR at https://github.com/openclaw/openclaw/pull/50359.

## What This Changes

The issue reports that Feishu replies drop media from read image tool results before final outbound payload assembly.

## Best Possible Solution

Review, port, and land https://github.com/openclaw/openclaw/pull/50359 or an equivalent current-main patch that falls back to trusted read start args while preserving local-media trust gates.

## Reproduction Assessment

Yes, by source inspection. Current main still has a path where trusted image content without structured media, `MEDIA:` text, or `details.path` produces no media, while the original read args are available but not passed into extraction.

## Solution Assessment

Yes. The maintainable fix is a shared embedded-run media extractor fallback from trusted read start args; a Feishu-only send change would miss the upstream promotion gap and other channel consumers.
