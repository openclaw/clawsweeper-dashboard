## What This Changes

The issue reports Feishu dropping image media from trusted `read` tool results before the final outbound payload reaches Feishu media delivery.

## Best Possible Solution

Land one shared embedded media-promotion fix that recovers trusted pathless `read` image media into outbound payloads while preserving local-media trust gates, then close this issue with merged commit and release proof.

Yes, source-level. Current main still returns no media for trusted image content without structured media, `MEDIA:` text, or `details.path`, and the checked-in test asserts that pathless shape produces an empty list.
