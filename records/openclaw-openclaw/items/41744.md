## Summary

Keep open: current main and v2026.5.20 still have the source-level pathless read-image media promotion gap, and the narrow repair remains in an open PR rather than merged.

## What This Changes

Preserve media from Feishu read-image tool results into the final outbound payload, including image results that lack `details.path` but still have the original read path in tool start args.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/50359 or an equivalent current-main fix that recovers trusted read start-arg paths into the pending-media queue while preserving local-media trust gates.

## Reproduction Assessment

Yes, source-level. Current main's extractor returns no media for trusted image content without structured media, `MEDIA:` text, or `details.path`, and the checked-in test asserts that pathless shape produces an empty list.

## Solution Assessment

Yes. The maintainable fix is a shared embedded-media fallback from trusted read start args into the existing pending-media queue, not a Feishu-only sender workaround.
