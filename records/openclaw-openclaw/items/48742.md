## Summary

Keep open: the cron portion addresses a valid current-main bug, but this PR is not mergeable as submitted because it also changes provider secret normalization in a way that can weaken the existing plaintext-secret guard.

## What This Changes

The PR removes legacy `notify: true` from no-webhook cron jobs, adds doctor cron tests, changes provider API-key normalization/tests, and updates unrelated WhatsApp and abort tests.

## Best Possible Solution

Land a focused cron doctor fix that removes inert legacy `notify: true` when no `cron.webhook` exists, preserves explicit delivery semantics with clear warnings, and leaves provider secret normalization to current main or a separate security-reviewed proposal.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a stored cron job with `notify: true`, no configured `cron.webhook`, and `delivery.mode: "none"` reaches the warning branch and keeps `notify` in the persisted store.

## Solution Assessment

No, not as submitted. The cron fix is the right narrow direction, but bundling it with provider secret-normalization changes and unrelated tests is not the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep env-secret reverse lookup active without sourceProviders:** `src/agents/models-config.providers.ts:529`
  - body: This guard skips the resolved-env reverse lookup unless `params.sourceProviders` is present, so normalization paths without a source snapshot no longer convert a resolved `OPENAI_API_KEY` value back to the env marker. That can persist plaintext provider secrets into `models.json` and regresses the existing #38757 protection; keep the reverse lookup independent of `sourceProviders` or split this provider change out.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff can weaken provider secret hygiene by allowing resolved env API keys to stay plaintext in generated model state.

  - body: By running the reverse lookup only when `sourceProviders` exists, the patch allows normalization paths without a source snapshot to keep resolved env secret values instead of canonical env markers, which can write provider API keys to generated state.
