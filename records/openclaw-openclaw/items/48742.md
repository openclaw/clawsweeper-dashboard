## What This Changes

The PR removes legacy cron `notify: true` when no webhook is configured, adds cron/provider tests, modifies provider API-key normalization, and touches unrelated WhatsApp and abort tests.

## Best Possible Solution

Land or replace this with a focused cron-only doctor fix and regression coverage while preserving the current provider secret-normalization contract.

Yes. Source inspection on current main shows a job with `notify: true`, `delivery.mode: "none"`, and no `cron.webhook` reaches the warning-only branch without deleting `notify`; no live doctor command was run because this review is read-only.

## Solution Assessment

No as submitted. The cron hunk is the right narrow direction, but the provider secret-normalization hunk should be removed or split, the branch needs a current-main update, and after-fix real behavior proof is still missing.
