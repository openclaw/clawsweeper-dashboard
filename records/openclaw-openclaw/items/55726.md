## Summary

Keep open. Current main and the latest release still leave `collectExposureMatrixFindings()` groupPolicy-only, so this PR addresses a real security-audit gap, but the current head misreads legacy `dm.policy` as open even when modern `dmPolicy` closes it and lacks real audit-output proof.

## What This Changes

The PR adds `dmPolicy` and legacy `dm.policy` paths to the security audit exposure matrix so the existing open-group exposure findings also fire for open DMs.

## Best Possible Solution

Merge a corrected dmPolicy exposure-matrix fix that honors modern-over-legacy policy precedence, carries regression coverage, and preserves existing check IDs.

## Reproduction Assessment

Yes. Source inspection on current main and the latest release shows the exposure matrix exits before elevated/runtime checks whenever `listGroupPolicyOpen()` is empty, while DM policy is only seen by other heuristic/channel audit paths.

## Solution Assessment

No. The direction is right, but the PR should compute the effective DM policy (`dmPolicy ?? dm.policy`) per channel/account, restore focused tests on the live head, and add redacted real `openclaw security audit` output.
