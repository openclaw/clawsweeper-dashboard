## Summary

Keep open: the PR targets a real current-main audit gap, but the patch still needs changes because it treats legacy `dm.policy` as an additional active policy instead of a fallback behind modern `dmPolicy`, which can create false CRITICAL findings during migration.

## What This Changes

The PR adds DM-policy path detection to the security exposure matrix by scanning `dmPolicy` and legacy `dm.policy` and feeding those paths into the existing open-groups audit findings.

## Best Possible Solution

Resolve each effective DM policy using the same modern-before-legacy precedence as provider config validation, then add focused regression coverage plus the required docs and changelog updates while preserving the existing check IDs.

## Reproduction Assessment

Yes. Source inspection shows current main returns before checking DM-only open policies, and the PR-specific false positive follows from the diff plus Discord/Slack `dmPolicy ?? dm.policy` precedence.

## Solution Assessment

No. The direction is correct, but the patch should compute effective DM policy before adding audit paths instead of treating modern and legacy keys as separate active exposures.
