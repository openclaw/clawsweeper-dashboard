## Summary

Keep open: current main still has the DM-policy exposure-matrix gap, but this PR should not merge as-is because the legacy `dm.policy` scan ignores modern `dmPolicy` precedence and can emit false CRITICAL findings.

## What This Changes

The branch modifies `src/security/audit-extra.sync.ts` so modern `dmPolicy` and legacy `dm.policy` can feed the existing open-groups elevated/runtime exposure checks.

## Best Possible Solution

Update this PR to resolve the effective DM policy before adding audit paths, with modern `dmPolicy` winning over legacy `dm.policy`, and add focused current-tree tests for modern, legacy-only, account fallback, and modern-overrides-legacy cases.

## Reproduction Assessment

Yes. Source inspection shows current main returns before checking DM-only open policies, and the PR-specific false positive follows from the PR diff plus Discord/Slack `dmPolicy ?? dm.policy` precedence.

## Solution Assessment

No. The direction is right, but the PR should match effective provider config precedence instead of treating modern and legacy DM policy keys as separate active exposure paths.
