## Summary

Keep open: current main still misses DM-only critical exposure checks, but this PR is not mergeable as-is because it treats legacy `dm.policy` as active even when modern `dmPolicy` overrides it and it only provides unit-test output rather than real audit proof.

## What This Changes

The branch modifies `src/security/audit-extra.sync.ts` to add DM-policy paths to the existing exposure-matrix audit findings.

## Best Possible Solution

Revise the PR to resolve effective DM policy with modern-over-legacy precedence, add focused regression tests for modern and legacy DM policy paths, keep stable check IDs, and add redacted real audit output.

## Reproduction Assessment

Yes. Source inspection on current main shows the exposure matrix exits on DM-only configs, and PR-head source inspection shows the legacy precedence false positive.

## Solution Assessment

No. The direction is right, but the implementation should compute effective DM policy instead of treating modern and legacy keys as independent active exposure paths.
