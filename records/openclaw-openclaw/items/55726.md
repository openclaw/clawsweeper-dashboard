## Summary

Keep open. Current `main` still misses the requested DM-policy exposure-matrix coverage, and this PR is pointed at the right security-audit gap, but the proposed legacy `dm.policy` handling can create false CRITICAL findings when a modern `dmPolicy` value overrides stale legacy config.

## What This Changes

The PR adds DM policy path collection to `collectExposureMatrixFindings()` so modern `dmPolicy` and legacy `dm.policy` are merged into the existing open `groupPolicy` elevated/runtime-fs audit checks while preserving the existing `open_groups` check IDs.

## Best Possible Solution

Resolve the effective DM policy before adding audit paths: modern `dmPolicy` should win, legacy `dm.policy` should be a fallback only when the modern key is absent, and focused tests should cover modern open, legacy-only open, account-level inheritance, and modern-overrides-legacy negative cases.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the original miss because `collectExposureMatrixFindings()` returns before checking DM-only open policies; the PR-specific bug is reproducible with a Discord or Slack config containing `dmPolicy: "allowlist"` plus stale `dm: { policy: "open" }`.

## Solution Assessment

No. The overall direction is correct, but the patch should match provider config precedence instead of treating modern and legacy DM policy keys as separate active exposure paths.
