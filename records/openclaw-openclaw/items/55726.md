## Summary

Keep this PR open, but it is not ready to merge: it targets a real DM exposure-matrix audit gap, yet the current branch can produce false CRITICAL findings for migrated configs, the final diff no longer carries the promised regression tests, and the contributor proof is unit-test output only.

## What This Changes

The PR adds dmPolicy and legacy dm.policy paths to the existing security exposure-matrix findings in src/security/audit-extra.sync.ts while preserving the existing open_groups check IDs.

## Best Possible Solution

Revise the PR to compute effective DM policy with modern-over-legacy precedence, restore focused regression tests, keep stable check IDs, and add redacted real audit output showing the fixed behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns from collectExposureMatrixFindings() when only dmPolicy is open, and PR head misclassifies stale legacy dm.policy because it does not apply the provider schema precedence.

## Solution Assessment

No. The direction is right, but the implementation should resolve the effective DM policy instead of treating modern and legacy keys as independent active exposure paths, and it needs regression tests plus real audit proof.
