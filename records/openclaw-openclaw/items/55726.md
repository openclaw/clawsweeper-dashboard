## Summary

Keep this PR open because it targets a real security-audit gap, but the current patch is not mergeable: it treats legacy `dm.policy` as active even when modern `dmPolicy` overrides it, and the contributor has not provided real behavior proof.

## What This Changes

The branch adds `dmPolicy` and legacy `dm.policy` paths to `collectExposureMatrixFindings()` so the existing open-groups exposure findings also fire for open DMs.

## Best Possible Solution

Resolve effective DM policy with modern-over-legacy precedence, add focused regression coverage for modern, legacy, and account-level paths, keep stable check IDs, and add redacted real audit output.

## Reproduction Assessment

Yes. Source inspection on current main shows the exposure matrix exits on DM-only configs, and PR-head source inspection shows the legacy precedence false positive.

## Solution Assessment

No. The direction is right, but the patch should compute effective DM policy instead of treating modern and legacy keys as independent active exposure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Honor dmPolicy before legacy dm.policy:** `src/security/audit-extra.sync.ts:351-352`
