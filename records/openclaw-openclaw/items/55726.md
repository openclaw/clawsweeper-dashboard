## Summary

Keep open: current main still misses DM-only critical exposure audit coverage, and this PR is useful, but the patch can false-positive when modern `dmPolicy` overrides stale legacy `dm.policy`.

## What This Changes

The PR adds DM-policy path detection to exposure-matrix audit findings so `dmPolicy` and legacy `dm.policy` can trigger the existing elevated and runtime/filesystem exposure checks.

## Best Possible Solution

Resolve the effective DM policy before adding audit paths: modern `dmPolicy` should win, legacy `dm.policy` should be a fallback only when the modern key is absent, with focused tests for modern, legacy-only, account-level, and modern-overrides-legacy cases.

## Reproduction Assessment

Yes. Source inspection shows current main returns before checking DM-only open policies, and the PR-specific false positive follows from the PR diff plus Discord/Slack `dmPolicy ?? dm.policy` precedence.

## Solution Assessment

No. The direction is correct, but the patch should match provider config precedence instead of treating modern and legacy DM policy keys as separate active exposure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor dmPolicy before legacy dm.policy:** `src/security/audit-extra.sync.ts:351-352`
  - body: The new legacy branch records `dm.policy="open"` even when the same section explicitly sets `dmPolicy` to a safer value. Discord and Slack resolve effective policy as `dmPolicy ?? dm.policy`, including account-level fallbacks, so migrated configs such as `dmPolicy: "allowlist"` plus stale `dm.policy: "open"` would get a false CRITICAL exposure finding.
