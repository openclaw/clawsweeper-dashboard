## Summary

Keep open: current main still has the DM exposure-matrix gap, but this PR head remains incorrect because it treats stale legacy `dm.policy` as effective even when modern `dmPolicy` overrides it, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds `dmPolicy` and legacy `dm.policy` detection to `collectExposureMatrixFindings()` and broadens the existing open-group audit messages to include open DMs.

## Best Possible Solution

Merge a corrected exposure-matrix fix that uses effective DM policy resolution, preserves existing check IDs, and adds focused regression coverage plus redacted real audit proof.

## Reproduction Assessment

Yes by source inspection. A config with only `dmPolicy="open"` cannot reach the two `open_groups_*` exposure checks on current main because `collectExposureMatrixFindings()` returns when `listGroupPolicyOpen()` is empty.

## Solution Assessment

No. The direction is right, but the PR should compute the effective DM policy per section (`dmPolicy ?? dm.policy`), restore focused regression tests on the live head, and add redacted real audit output.
