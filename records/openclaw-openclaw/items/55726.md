## Summary

Keep open: current main still has the DM-only exposure-matrix gap, but this PR head has a correctness blocker in legacy DM policy resolution and still lacks after-fix real audit proof.

## What This Changes

The PR adds `listDmPolicyOpen()` in `src/security/audit-extra.sync.ts` and broadens the existing open-group exposure findings to include modern `dmPolicy` and legacy `dm.policy` paths.

## Best Possible Solution

Merge a corrected exposure-matrix fix that uses effective DM policy resolution, preserves stable check IDs, restores regression coverage, and includes redacted real `openclaw security audit` proof.

## Reproduction Assessment

Yes by source inspection. On current main, a config with only `dmPolicy="open"` cannot reach the two `open_groups_*` exposure checks because `collectExposureMatrixFindings()` returns when `listGroupPolicyOpen()` is empty.

## Solution Assessment

No. The direction is right, but the PR should resolve the effective DM policy with modern-before-legacy precedence, add regression coverage on the live head, and include redacted real audit output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor modern dmPolicy before legacy dm.policy:** `src/security/audit-extra.sync.ts:351-352`
