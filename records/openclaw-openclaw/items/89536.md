## Summary

Keep open: current main still has the Google Chat app-auth media-upload message-loss path, and this PR targets the right three send surfaces. It should not merge yet because the fallback matcher overmatches non-scope 403/PERMISSION_DENIED upload errors, and the contributor proof is mock-only rather than a live Google Chat app-auth round trip.

## What This Changes

The branch adds a shared Google Chat media-upload fallback helper and wraps monitor reply delivery, outbound adapter media sends, and message actions to send remote media URLs as text when attachment upload fails with an auth/scope error.

## Best Possible Solution

Keep the three-path Google Chat fallback, but gate it on the actual insufficient-scope upload failure and require redacted live app-auth Google Chat proof before merge.

## Reproduction Assessment

Yes for a source-level reproduction: current main uses chat.bot app auth, and all three Google Chat media paths call the attachment upload endpoint before sending. I did not establish a live failing workspace run in this review.

## Solution Assessment

No: the direction is right and covers the correct owner/plugin boundary, but the helper is too broad for the promised auth-scope-only fallback and the PR has only mock-based proof. The safer solution is the same fallback with a stricter classifier plus live app-auth evidence.
