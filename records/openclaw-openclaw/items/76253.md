## Summary

Close as ClawHub scope: Lobstah is an optional distributed model-provider plugin that uses existing provider-plugin seams, does not demonstrate a missing OpenClaw core API, and the proposed bundled implementation also has merge-blocking network-security issues.

## What This Changes

The PR adds a bundled Lobstah provider plugin plus seven @lobstah workspace packages for a P2P LLM inference grid with worker, router, tracker, CLI, tests, lockfile, and changelog entry.

## Best Possible Solution

Keep Lobstah as an independently maintained provider plugin published through ClawHub, and only add a narrow generic SDK/API change to OpenClaw if that external implementation proves a missing seam.

## Reproduction Assessment

Not applicable as a feature PR, but the merge blockers are statically reproducible by reading the proposed branch: worker startup defaults to 0.0.0.0 and tracker-synced URLs flow into later router fetches.

## Solution Assessment

No for core bundling. The narrower maintainable path is an external ClawHub provider plugin with explicit network-safety controls before any official promotion discussion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind the worker to loopback by default:** `packages/lobstah-worker/src/server.ts:194`
  - body: `startWorker` defaults to `0.0.0.0`, so `lobstah worker start` exposes `/capacity`, `/v1/models`, and `/v1/chat/completions` to the local network without authentication even when the user has not opted into advertising. Default to loopback and require an explicit public bind for contribution mode.
  - confidence: 0.92
- **[P1] Validate synced peer URLs before routing:** `packages/lobstah-cli/src/peers.ts:59`
  - body: A signed announcement only proves key ownership, not that `announcement.url` is safe. Persisting tracker-provided URLs lets later router requests fetch `/capacity` and chat completions from loopback, link-local, or private services on the user's machine, so synced URLs need a network policy or explicit trust gate.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds unauthenticated network services and unguarded tracker-controlled peer URL fetches.

Concerns:

- **[high] Worker binds unauthenticated API to all interfaces:** `packages/lobstah-worker/src/server.ts:194`
  - body: The worker default host is `0.0.0.0`, while its completion, model, and capacity endpoints have no authentication layer. Starting the worker can expose local Ollama-backed inference to the LAN unless the operator supplies a safer host.
  - confidence: 0.92
- **[high] Tracker peer URLs can drive local SSRF:** `packages/lobstah-cli/src/peers.ts:59`
  - body: `peers sync` stores arbitrary signed announcement URLs, and the router later fetches those URLs for capacity and chat forwarding. Signature verification does not prevent private-address, loopback, or metadata-service targets.
  - confidence: 0.88
