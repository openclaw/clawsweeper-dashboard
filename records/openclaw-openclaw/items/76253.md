## What This Changes

The draft PR adds a bundled Lobstah provider plugin, seven @lobstah workspace packages for P2P LLM routing/tracking/ledger/CLI, a changelog entry, and lockfile dependency updates.

## Best Possible Solution

Keep Lobstah as an independently maintained provider plugin/package published through ClawHub, and only add generic SDK seams to OpenClaw core if that external implementation proves a missing API.

## Reproduction Assessment

Not applicable as a feature PR. The review has high-confidence static reproduction for the blockers by inspecting the PR diff paths that bind the worker to all interfaces and persist tracker-supplied peer URLs.

## Solution Assessment

No for core bundling. The narrower maintainable direction is an external ClawHub provider plugin using the existing provider SDK, with security fixes before any official promotion discussion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bind workers to loopback by default:** `packages/lobstah-worker/src/server.ts:194`
  - body: `lobstah worker start` passes no host unless the user supplies `--host`, so `startWorker` binds the unauthenticated worker API to `0.0.0.0`. That makes a default worker reachable from the LAN/public interface even when the user did not opt into advertising, contradicting the default-invisible safety model.
  - confidence: 0.92
- **[P1] Reject unsafe tracker peer URLs:** `packages/lobstah-cli/src/peers.ts:59`
  - body: `peers sync` trusts any signed tracker announcement URL and saves it directly. The router later fetches `${peer.url}/capacity` and chat endpoints, so a public tracker can make the user's router connect to loopback, metadata, or private-network services unless these URLs are validated and routed through an SSRF/private-network policy.
  - confidence: 0.88
- **[P2] Use an API-key env var for auth discovery:** `extensions/lobstah/defaults.ts:3`
  - body: `LOBSTAH_DEFAULT_API_KEY_ENV_VAR` is set to `LOBSTAH_ROUTER_URL`, but the shared self-hosted provider setup treats this value as the API key and sends it as bearer auth; setting a router URL in the environment will not change the base URL and will leak the URL into the auth field.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The proposed bundled plugin opens unauthenticated network surfaces and consumes untrusted tracker URLs without a network guard.

Concerns:

- **[high] Worker binds unauthenticated API to all interfaces:** `packages/lobstah-worker/src/server.ts:194`
  - body: The default worker host is `0.0.0.0`, so starting a worker exposes the OpenAI-compatible completion endpoint to the local network unless the operator notices and overrides `--host`.
  - confidence: 0.92
- **[high] Tracker peer URLs can drive local SSRF:** `packages/lobstah-cli/src/peers.ts:59`
  - body: Signed announcements only prove key ownership, not URL safety; persisting arbitrary tracker URLs lets later router model/capacity requests target private or local services.
  - confidence: 0.88
