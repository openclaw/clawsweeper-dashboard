## What This Changes

This PR adds a Pi embedded-runner helper, prompt-time allowlist re-application, diagnostics, and focused tests so active tools are restored before each non-raw `session.prompt()` dispatch.

## Best Possible Solution

## Reproduction Assessment

Yes. The linked bug provides a high-confidence live reproduction path with version, Telegram channel, direct Anthropic provider, tool policy config, and debug probes showing 17 computed tools becoming provider-facing `tools: []`.

## Solution Assessment

Yes, with a process fix. Re-applying the already policy-filtered Pi allowlist immediately before prompt dispatch is the narrowest maintainable boundary fix I found; the safer alternative, a transport fallback to runner tools, would hide the runner state bug and risks raw-model behavior.
