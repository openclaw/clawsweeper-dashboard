## Summary

Keep open. Current main still leaves the Telegram model picker confirmation ambiguous: it writes a session-scoped override while telling the user only that the model changed, and reset acknowledgements remain generic even though related model-state work is still open.

## What This Changes

The issue asks OpenClaw to make Telegram `/models` and reset/restart behavior distinguish session-scoped model overrides from durable agent defaults.

## Best Possible Solution

Keep core precedence unchanged for now, make Telegram-facing model selection and reset/status copy explicitly identify session override versus agent default, and leave durable-default reconciliation to the broader model-state design work.

## Reproduction Assessment

Yes for the remaining UX mismatch: current source and tests show the Telegram selection writes session override fields while the success text omits the agent-default scope. The exact restart-only fallback still needs a live or targeted regression reproduction before changing model precedence.

## Solution Assessment

Unclear for a full behavior change. The narrowest maintainable first fix is Telegram copy and regression tests; durable default buttons and single-source reconciliation belong with the broader model-state threads.
