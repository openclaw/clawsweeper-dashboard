## Summary

Keep open. Current main still has the reported user-facing mismatch: the Telegram model picker writes a session-scoped override while the success copy does not say it is session-only, and this issue is now paired with an open closing PR that should be reviewed before the issue is resolved.

## What This Changes

The issue asks for Telegram/model-switch acknowledgements and reset/restart behavior to make clear that model changes are session-scoped and do not update the agent's durable default.

## Best Possible Solution

Review and land or revise #75980 for the narrow session-scope copy fix, then leave durable-default reconciliation and reset diagnostics to the broader model-state follow-ups if they remain out of scope.

Yes for the UX mismatch: current source and tests show Telegram picker selection writes session override fields while the confirmation omits session-only and durable-default wording.
