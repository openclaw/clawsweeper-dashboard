## Best Possible Solution

Move the primitive-frame guard to the acpx/ACP SDK stream boundary, or get explicit maintainer acceptance that this PR is a generated-wrapper-only mitigation with a tracked follow-up for the shared ACP path.

## Solution Assessment

No, not as submitted. The wrapper filter is a useful mitigation for generated commands, but the complete fix belongs at the acpx/ACP SDK stream boundary that all adapter commands share.
