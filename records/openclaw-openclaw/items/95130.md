## Summary

## What This Changes

The PR pipes generated ACPX adapter stdout through a line filter that drops top-level JSON primitives, null, and arrays, flushes pending stdout on close, and adds generated-wrapper regression tests.

## Best Possible Solution

Place the object-shape guard at the acpx/ACP SDK parsed-message boundary, or explicitly accept this as wrapper-only mitigation with a tracked follow-up for custom command coverage.

## Reproduction Assessment

## Solution Assessment

No. The wrapper filter is a useful mitigation for generated commands, but the narrowest maintainable fix is at the acpx/ACP SDK parsed-message boundary because current OpenClaw can still pass custom ACP commands around the wrapper.
