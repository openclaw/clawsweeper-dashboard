## Summary

Keep this PR open, but do not merge it as-is. Current main still performs the blocking full transcript read/parse that the PR is trying to fix, while the PR head has correctness gaps around current session-history sequence/cursor semantics, compaction records, and active-branch transcript handling.

## What This Changes

The PR changes `readSessionMessages()` to tail-read large transcript files, cap oversized JSONL records, emit placeholders/warnings, and parse the bounded content for `chat.history`.

## Best Possible Solution

Implement a bounded transcript reader on current main that avoids full hot-path reads/parses while preserving `SessionManager` active-branch behavior, compaction markers, sanitized media-heavy history, and stable absolute `__openclaw.seq` metadata for every emitted entry, including placeholders.

## Reproduction Assessment

Yes. At source level, current main still synchronously reads and parses the whole transcript before `chat.history` limits apply; the PR body also reports a concrete 447 KB single-line JSONL reproduction, though this read-only review did not run a live Gateway reproduction.

## Solution Assessment

No. The PR's direction is appropriate, but the current patch is not the best mergeable solution because it tail-renumbers transcript messages, omits seq metadata on placeholders, and predates current main's active-branch transcript reader.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve absolute seq values after tail reads:** `src/gateway/session-utils.fs.ts:132-135`
  - body: When the tail path starts reading from the middle of a transcript, the parser still initializes `messageSeq` from zero and assigns seq values only within the retained tail. Session-history cursors use `__openclaw.seq`, so the same transcript can expose different seq values as the tail window shifts, causing skipped or duplicated pages. Preserve absolute transcript positions or use a stable cursor before tail-reading this path.
  - confidence: 0.94
- **[P1] Assign seq metadata to oversized placeholders:** `src/gateway/session-utils.fs.ts:260-270`
  - body: The oversized-line branch pushes a synthetic message without incrementing `messageSeq` or adding `__openclaw.seq`. Pagination and SSE snapshots derive cursors from seq metadata, so a page containing this placeholder can have `hasMore` without a usable cursor or can mis-correlate later messages. Give placeholders the same monotonic seq bookkeeping as parsed transcript entries.
  - confidence: 0.92
- **[P2] Parse record type before replacing oversized lines:** `src/gateway/session-utils.fs.ts:195-197`
  - body: The size guard runs before record-type detection, so an oversized non-message record such as a compaction entry becomes a synthetic chat turn instead of the existing compaction divider. That changes history semantics and can push out a real recent message under tight limits. Detect safe record metadata first or skip non-message records without emitting chat placeholders.
  - confidence: 0.87
- **[P2] Preserve active-branch transcript reading:** `src/gateway/session-utils.fs.ts:187-190`
  - body: The PR head builds history directly from bounded JSONL content and lacks current main's `SessionManager.open(filePath).getBranch()` path for Pi transcript DAGs. Merging this approach without rebasing would reopen abandoned parent entries that current tests now exclude. Rebase the fix and keep active-branch selection before applying read bounds.
  - confidence: 0.86
