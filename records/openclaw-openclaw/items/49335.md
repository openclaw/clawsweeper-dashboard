## Summary

Close this PR as implemented on current main: the gateway now uses bounded recent transcript reads, streaming/full transcript readers, and oversized-entry placeholders, while this stale branch still has sequence/cursor regressions and should not merge.

## What This Changes

The PR changes `src/gateway/session-utils.fs.ts` to tail-read large `chat.history` transcript files and replace oversized JSONL records with placeholder messages before parsing.

## Best Possible Solution

Keep the current main implementation and close this stale branch rather than merging its older tail-read approach; any remaining WebUI freeze evidence should continue under the newer related bug report if it still reproduces.

## Reproduction Assessment

Yes for the original problem from source and PR evidence: the old path read and parsed the full JSONL transcript synchronously, and the PR reports a reliable 447 KB single-line transcript freeze. I did not reproduce the freeze live against current main because current main has already replaced that hot path.

## Solution Assessment

No, this PR branch is no longer the best way to solve it. Current main uses bounded recent readers, oversized placeholders, streaming scans, and sequence-aware history stats; merging this branch would reintroduce stale cursor/sequence bugs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve sequence metadata for oversized placeholders:** `src/gateway/session-utils.fs.ts:260-271`
  - body: The oversized-line branch pushes a synthetic message without incrementing `messageSeq` or attaching `__openclaw.seq`. Session history pagination and SSE state derive cursors from those seq values, so an oversized record near a page boundary can produce missing or unstable cursors.
  - confidence: 0.91
- **[P1] Preserve absolute seq values after tail truncation:** `src/gateway/session-utils.fs.ts:279-282`
  - body: When the tail-read path skips the transcript prefix, parsed messages still get `seq: messageSeq` starting from 1 over only the retained tail. Those seq values are no longer stable across reads as the tail window changes, which can make cursor pagination skip or duplicate history entries.
  - confidence: 0.88
