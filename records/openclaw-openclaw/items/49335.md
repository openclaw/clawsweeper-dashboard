## Summary

Keep open: the Gateway history freeze class is valid, but this PR is stale against current `main` and still has merge-blocking history cursor, placeholder, and active-branch regressions.

## What This Changes

The PR changes `src/gateway/session-utils.fs.ts` so `readSessionMessages()` tail-reads large JSONL transcripts, caps oversized records, emits placeholders/warnings, and parses bounded content for `chat.history`.

## Best Possible Solution

Implement the remaining bounded transcript parsing on current `main`, preserving async/index-backed history reads, active-branch selection, compaction markers, sanitized media-heavy messages, and stable absolute seq metadata for every emitted entry.

## Reproduction Assessment

Unclear for current `main`. The original 447 KB JSONL reproduction is concrete, but current `main` now uses bounded async reads for the common first-page `chat.history` path, so a repair should add a focused regression test for oversized retained lines rather than assuming the March freeze still reproduces unchanged.

## Solution Assessment

No. The PR direction is good, but the branch is not the best mergeable solution because it is stale against current transcript-reader semantics and its placeholder/tail logic breaks cursor and record-type behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve absolute seq values after tail reads:** `src/gateway/session-utils.fs.ts:132-135`
  - body: When the tail path starts mid-transcript, this parser rebuilds `messageSeq` from the retained tail instead of the absolute transcript position. History cursors depend on `__openclaw.seq`, so the same transcript can expose shifting cursors as the tail window moves, causing skipped or duplicated pages.
  - confidence: 0.94
- **[P1] Assign seq metadata to oversized placeholders:** `src/gateway/session-utils.fs.ts:260-270`
  - body: The oversized-line branch emits a synthetic message without incrementing the transcript sequence or adding `__openclaw.seq`. Pagination and SSE snapshots derive cursors from that metadata, so placeholders near a page boundary can leave `hasMore` without a usable cursor or misalign later messages.
  - confidence: 0.92
- **[P2] Parse record type before replacing oversized lines:** `src/gateway/session-utils.fs.ts:195-197`
  - body: The size guard runs before record-type detection, so an oversized non-message record such as a compaction entry becomes a synthetic chat turn instead of the existing compaction divider. That changes visible history semantics and can push out a real recent message under tight limits.
  - confidence: 0.87
- **[P2] Preserve active-branch transcript reading:** `src/gateway/session-utils.fs.ts:187-190`
  - body: The PR reads bounded JSONL content directly and does not preserve current main's active-branch transcript selection. Merging this approach would reopen abandoned parent entries that current history readers now exclude.
  - confidence: 0.86
