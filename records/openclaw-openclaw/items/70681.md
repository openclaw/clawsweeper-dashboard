## Summary

Keep open: the PR addresses a real current-main macOS gateway cleanup bug, but the latest head still has a blocking polling regression where both new `ps` verifier calls can use the full 2000ms spawn timeout inside a 2000ms poll budget.

## What This Changes

The PR changes Unix stale-gateway cleanup to parse raw `lsof` listener PIDs, verify candidates with Unix `ps` argv inspection, and update stale-PID/restart tests for macOS `lsof` `p_comm` behavior.

## Best Possible Solution

Land this PR or an equivalent narrow fix after both poll-path verifier calls are bounded by `POLL_SPAWN_TIMEOUT_MS`, while preserving raw `lsof` PID parsing, argv-based gateway verification, self/ancestor exclusions, Windows behavior, focused regression coverage, and the required release-note entry.

## Reproduction Assessment

Yes. The linked issue gives a concrete macOS launchd reproduction, and current main can be source-reproduced by feeding `lsof -Fpc` output with `p<PID>\ncnode` for a gateway process into the current parser; the PR-specific timeout bug is directly visible in the diff.

## Solution Assessment

No, not yet. The Unix argv-verifier direction is the narrowest maintainable fix and mirrors the Windows path, but both poll branches need the short poll timeout before the patch preserves the existing polling invariant.
