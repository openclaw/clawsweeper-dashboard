## Summary

Close as implemented: current main prevents the reported npm global bin disappearance by staging npm installs outside the live prefix, verifying the staged package, swapping the package tree, and replacing the `openclaw` bin shim with rollback handling. The fix shipped in `v2026.4.26`, and the earlier linked implementation PR is now closed unmerged, so it no longer blocks cleanup.

## What This Changes

The issue requests that `openclaw update` on global npm installs verify and restore the public `openclaw` bin link after npm reify can retire it to a hidden temporary path.

## Best Possible Solution

Keep the shipped staged npm update and bin-shim swap as the canonical fix. Treat explicit hidden temp-link cleanup as unnecessary unless a fresh report on `v2026.4.26` or newer shows the public `openclaw` bin still disappearing after update.

## Reproduction Assessment

Yes for the original failure: the report includes concrete global npm install layout, versions, commands, and npm reify log lines showing `bin/openclaw` retired to a hidden `.openclaw-*` name. I did not run a destructive live global npm update in this read-only review; current-main source, tests, and release evidence cover the prevention path.

## Solution Assessment

Yes. Staging npm into a temporary prefix and copying the verified staged bin shim into the real prefix is the narrower maintainable fix because it prevents npm from retiring the live public bin during install, while rollback handling avoids leaving the old shim broken on swap failure.
