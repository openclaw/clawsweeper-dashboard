## Summary

Current main implements the updater-side fix: npm global updates are installed into a verified staged prefix and the staged `openclaw` bin shim is copied into the real global bin directory with rollback, so the live npm reify path no longer removes the public CLI entrypoint. The same-repo repair PR is closed unmerged, and the shipped changelog records the broader fix in `v2026.4.26`.

## What This Changes

Make `openclaw update` on global npm installs leave the public `openclaw` bin link present after npm’s reify/update phase.

## Best Possible Solution

Keep the staged npm install and shim-swap implementation as the canonical fix; any leftover hidden temp-bin cleanup can be handled separately only if it causes an observable problem.

## Reproduction Assessment

Yes. The report gives concrete `openclaw update` steps, npm global layout, versions, and npm reify log lines showing `bin/openclaw` retired to a hidden temp name while the package entry still worked; current source and tests verify the replacement path that prevents that live-bin mutation.

## Solution Assessment

Yes. Staging npm under a temporary prefix and copying the verified shim into the real prefix is the narrower maintainable fix because it avoids npm mutating the live CLI entrypoint rather than trying to repair every possible reify leftover afterward.
