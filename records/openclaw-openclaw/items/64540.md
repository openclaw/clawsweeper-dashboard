Full review comments:

- **[P1] Keep empty overrides aligned with scheduler selection:** `src/infra/heartbeat-summary.ts:36`
  - body: This predicate now ignores `heartbeat: {}`, but `heartbeat-runner.ts` still treats any truthy per-agent heartbeat block as explicit when selecting scheduled agents and then merges defaults into that entry. With defaults configured plus a per-agent `{}`, summary/runOnce can call the agent disabled while the scheduler still selected it under default cadence, so the patch splits one heartbeat contract into two conflicting sources of truth. Please move the empty-object rule into the shared scheduler/config resolution path and cover the mixed defaults plus empty override case.
  - confidence: 0.86
- **[P2] Apply disablement to system-prompt gating:** `src/infra/heartbeat-summary.ts:53-54`
  - body: Returning false for empty defaults here disables scheduling/status, but `shouldIncludeHeartbeatGuidanceForSystemPrompt` still resolves an empty heartbeat config through its own cadence path and falls back to the default interval. That leaves normal agent prompts carrying heartbeat guidance even when this PR says the heartbeat is disabled. The same resolved heartbeat enablement contract needs to drive prompt/bootstrap inclusion too.
  - confidence: 0.83
