## What This Changes

The PR changes `src/infra/heartbeat-summary.ts` so empty heartbeat objects are treated as disabled by explicit-agent detection, enablement, and interval resolution.

## Best Possible Solution

Define one heartbeat disable/absence contract, then implement it through a shared resolver used by scheduler selection, interval resolution, status, prompt/bootstrap, ACP spawn, provider defaulting, docs, and regression tests.

## Reproduction Assessment

Yes, source-level: current main accepts empty heartbeat objects and the helper falls back to the default cadence when `every` is missing, while scheduler selection is still computed separately. I did not run a live gateway or remeasure provider token usage in this read-only review.

## Solution Assessment

No. The one-file patch is too narrow for the behavior it changes; the maintainable fix is a shared heartbeat policy resolver applied consistently to summary, scheduler, prompt, ACP spawn, provider defaults, docs, and tests.
