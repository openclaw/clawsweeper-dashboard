## Summary

Keep open. The reported heartbeat behavior is still source-reproducible on current main, but this PR is not merge-ready because the one-file patch is stale against current heartbeat policy surfaces, lacks after-fix real behavior proof, and conflicts with related open heartbeat contract work.

## What This Changes

The branch modifies `src/infra/heartbeat-summary.ts` to treat empty heartbeat objects as disabled in explicit-agent detection, enablement, and interval resolution.

## Best Possible Solution

Define one heartbeat disable/absence/override contract, then implement it through a shared resolver used by scheduler, status, provider defaults, prompt/ACP, doctor, docs, and regression tests.

## Reproduction Assessment

Yes, source-level: current main accepts `heartbeat: {}`, treats truthy defaults heartbeat as enabled, and falls back to the default cadence when `every` is missing. I did not run a live gateway or remeasure provider token usage in this read-only review.

## Solution Assessment

No. The proposed patch is too narrow and stale because it changes only `heartbeat-summary.ts` while scheduler selection, provider default backfill, system prompt, ACP spawn, docs, doctor diagnostics, and tests still disagree on the heartbeat policy.
