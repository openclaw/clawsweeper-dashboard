## Summary

Keep open. The reported source path still exists on current main, but this PR is stale and incomplete: it changes only heartbeat summary helpers while current-main defaults behavior, provider default backfill, scheduler selection, prompt/ACP gates, docs, and tests remain on different semantics.

## What This Changes

The PR modifies `src/infra/heartbeat-summary.ts` to treat empty heartbeat config objects as disabled for summary enablement and interval resolution.

## Best Possible Solution

Choose one heartbeat config contract, then implement it through a shared resolver used before provider default backfill and by scheduler, status, prompt/bootstrap, ACP spawn, doctor diagnostics, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection shows `agents.defaults.heartbeat: {}` is schema-valid/truthy on current main and resolves to the default cadence, though I did not execute the gateway in this read-only review.

## Solution Assessment

No. A one-off empty-object check in `heartbeat-summary.ts` is not the narrow maintainable fix because runtime scheduling, provider defaults, prompt/ACP gates, docs, and tests would still disagree.
