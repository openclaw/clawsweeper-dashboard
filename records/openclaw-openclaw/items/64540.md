## Summary

Keep open. The reported behavior is source-reproducible on current main, but this PR is too narrow and would create inconsistent heartbeat semantics across summary, scheduler, prompt, and ACP spawn paths.

## What This Changes

The PR changes `src/infra/heartbeat-summary.ts` so empty heartbeat config objects are treated as disabled in summary enablement and interval resolution.

## Best Possible Solution

Choose one heartbeat config contract, then implement it through a shared resolver used by summary, scheduler, system prompt, ACP spawn, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows `agents.defaults.heartbeat: {}` is schema-valid, truthy, and falls through to the default heartbeat cadence on current main; I did not run tests because this review is read-only.

## Solution Assessment

No. The PR introduces the empty-object disable rule in one helper but leaves independent heartbeat gates and docs on different semantics; a shared policy or an explicit decision to keep `0m` as the only disable path is safer.
