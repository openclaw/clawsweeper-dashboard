Keep open. ## What This Changes

The PR changes `src/infra/heartbeat-summary.ts` so empty heartbeat objects are treated as disabled in helper-level enablement and interval resolution.

## Best Possible Solution

Choose one empty-object heartbeat contract and implement it through a shared resolver used by scheduling, status, prompt/bootstrap, ACP spawn, provider defaulting, docs, and tests.

## Reproduction Assessment

Yes, source-level. On current main, `agents.defaults.heartbeat: {}` is schema-valid, helper enablement treats it as truthy, interval resolution falls through to the default cadence, and scheduler setup only skips null intervals.

## Solution Assessment

No. The PR is a partial helper-only fix; the maintainable solution is one shared heartbeat policy path aligned across provider defaults, scheduling, status, prompt/bootstrap, ACP relay, docs, and tests.
