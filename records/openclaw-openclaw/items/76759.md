## Summary

Keep open: current main still lacks public SDK exports for the status payload types, while a related open PR is already carrying the narrow implementation path. The model catalog type is already reachable through the public `plugin-sdk/agent-runtime` subpath, so the remaining decision is the exact supported status-type export surface.

## What This Changes

Request public plugin SDK type exports for status summaries/session rows and model catalog entries so gateway-consuming plugins can bind to canonical host shapes.

## Best Possible Solution

Review and land or revise #77184 so status payload types are exposed on a narrow supported SDK subpath, while keeping model catalog typing on the existing agent-runtime surface unless maintainers choose a broader API.

## Reproduction Assessment

Yes. Source inspection shows the requested status types exist internally but are absent from `plugin-sdk/command-status`, while the reported `recent` session shape is the one current main builds.

## Solution Assessment

Unclear until SDK boundary review finishes. The active PR's focused `plugin-sdk/command-status` export direction fits local guidance better than a broad root barrel, but maintainers should decide the final public surface and ModelCatalogEntry placement.
