## Summary

Keep open: current `main` has the internal status/model catalog shapes, but it still has no intentional public SDK export for `StatusSummary`/`SessionStatus`, and the exact public contract should be chosen with the related gateway-accessor work.

## What This Changes

Request to export `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` through the plugin SDK public surface so plugins can type gateway status and model-list responses.

## Best Possible Solution

Promote the response types through a narrow documented SDK contract alongside the curated gateway accessors, then update SDK export/API checks and docs together.

## Reproduction Assessment

No high-confidence runtime bug reproduction is needed here; the gap is source-reproducible because the internal types exist while the requested public SDK contract is not exported. The reported `entries` versus `recent` mismatch is also confirmed in the current status summary source.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the repo guidance prefers narrow public subpaths and the companion gateway accessors in #76756 should drive where these response types live.
