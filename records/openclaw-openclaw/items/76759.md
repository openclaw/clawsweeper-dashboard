## Summary

Keep open. Current main still lacks the requested status type exports on the public `command-status` SDK subpath, and the associated open fix is https://github.com/openclaw/openclaw/pull/77184, so this issue should not close before that PR is merged or explicitly rejected.

## What This Changes

Request to expose canonical `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` types from the Plugin SDK so plugins can bind gateway status/model data without copying host shapes.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/77184; keep the status payload types on the existing `command-status` subpath and avoid duplicating `ModelCatalogEntry` unless maintainers want a broader SDK barrel change.

## Reproduction Assessment

Yes. Source inspection shows the existing public `command-status` subpath does not export the requested status types while the canonical core types are present; `ModelCatalogEntry` is already reachable through `agent-runtime`.

## Solution Assessment

Yes. The narrow maintainable solution is an additive type-only re-export from the existing focused SDK subpath, with docs/API-baseline coverage; the open linked PR is already following that path.
