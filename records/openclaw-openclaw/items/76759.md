## Summary

Keep open. Current main still lacks a deliberate public SDK contract for the status summary/session row types, and the right import path should be decided with the broader gateway-RPC SDK work instead of closing this as already handled.

## What This Changes

The issue requests public plugin-sdk exports for the gateway `StatusSummary`, per-session `SessionStatus`, and model catalog `ModelCatalogEntry` types.

## Best Possible Solution

Promote a narrow, documented SDK contract for read-only gateway status and model-list results, aligned with #76756 and covered by SDK export/baseline checks.

## Reproduction Assessment

Yes for the type-surface gap: current main defines the status/model shapes internally, and source inspection shows the root SDK does not export the requested status types. This is a feature/API-surface request rather than a runtime bug reproduction.

## Solution Assessment

No, not as a simple root-barrel re-export without maintainer API review. The safer direction is a focused, documented gateway/status SDK contract that keeps the public boundary narrow and matches the eventual RPC accessor shape.
