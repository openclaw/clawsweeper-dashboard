## Summary

Keep open: the type-surface gap is still present for the status response types, while the model catalog type is only reachable through an existing narrow subpath. This is an additive public SDK API decision that should be coordinated with the still-open gateway RPC accessor work rather than closed as cleanup already done.

## What This Changes

Expose `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` as public plugin SDK types for plugins consuming status and model-catalog gateway responses.

## Best Possible Solution

Promote the response types through a narrow documented SDK contract alongside the curated gateway accessors, and update SDK export checks, API baseline, and docs in the same change.

## Reproduction Assessment

Yes for the source-level gap: current main defines the internal status/model catalog types, but the public root and gateway SDK surfaces do not expose `StatusSummary` or `SessionStatus`. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the maintainable solution should align the type contract with the gateway accessor surface and documented SDK subpath policy.
