## Summary

Keep open: current main still has the internal DTOs, but the requested public SDK contract has not been intentionally promoted, and the remaining work is an additive plugin SDK API decision rather than cleanup already implemented.

## What This Changes

The issue requests public plugin SDK exports for StatusSummary, SessionStatus, and ModelCatalogEntry so plugins can type gateway status and model catalog responses.

## Best Possible Solution

Promote the gateway response DTOs through a narrow documented SDK contract, aligned with the #76756 accessor shape, and update SDK export checks, API baseline, and docs together.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The API-surface gap is source-reproducible: the internal DTOs exist, while current public SDK/runtime surfaces do not expose a complete status/list-models contract for plugins.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the repo's SDK guidance favors narrow subpaths and the companion gateway accessor contract in #76756 should drive where these DTOs live.
