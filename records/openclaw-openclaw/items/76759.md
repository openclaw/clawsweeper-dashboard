## Summary

Keep open: current main still lacks public `StatusSummary`/`SessionStatus` exports from the `command-status` Plugin SDK subpath, while `ModelCatalogEntry` is already reachable through `agent-runtime`. The linked implementation PR at https://github.com/openclaw/openclaw/pull/77184 remains open, so this issue should stay open for maintainer review rather than spawning duplicate cleanup work.

## What This Changes

The issue requests public Plugin SDK exports for status summary/session status and model catalog row types so plugins can bind to canonical host gateway shapes instead of copying or guessing them.

## Best Possible Solution

Land or reject a narrow public SDK contract for status payload types, updating SDK docs/API-baseline checks, while keeping `ModelCatalogEntry` on the existing supported surface unless maintainers choose a clearer dedicated model-catalog subpath.

## Reproduction Assessment

Yes. Source inspection shows the canonical status types exist internally while `src/plugin-sdk/command-status.ts` does not export them; I did not run a compile repro because this review is read-only.

## Solution Assessment

Mostly yes. A narrow Plugin SDK type export is the right layer, but the exact subpath/export set needs maintainer approval because it becomes public third-party API.
