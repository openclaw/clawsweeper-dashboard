## Summary

Keep open: current main still defines the requested status and model-catalog shapes internally, but they are not exposed through the plugin SDK public surface, and the related gateway RPC accessor work remains open.

## What This Changes

The issue requests exporting `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` from the plugin SDK public surface for plugin authors consuming host status/model data.

## Best Possible Solution

Add a narrow, documented plugin SDK type surface for these host response shapes, coordinated with the read-only gateway RPC API tracked in #76756.

## Reproduction Assessment

Yes for the source-level gap: current main defines the requested host types internally, while the plugin SDK barrels and runtime type do not expose them. This is a feature/API-surface request rather than a broken existing contract.

## Solution Assessment

Unclear as written: exporting stable types is the right direction, but maintainers should choose the public subpath and coordinate it with #76756 instead of adding an ad hoc root-barrel re-export.
