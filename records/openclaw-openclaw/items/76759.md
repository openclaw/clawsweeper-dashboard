## Summary

Keep open: current main has the internal status and model catalog types, but the public plugin SDK surface does not export them, and making them public is an additive SDK API decision rather than cleanup that is already implemented.

## What This Changes

Request to export `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` from the public plugin SDK so plugins can type host status/model RPC responses.

## Best Possible Solution

Promote the response row types through a narrow documented SDK contract, preferably alongside the curated gateway accessors, and update the SDK export checks and docs in the same change.

## Reproduction Assessment

Not applicable as a runtime bug reproduction, but the type-surface gap is source-reproducible: the internal types exist and the current public SDK exports do not expose them.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the SDK guide favors narrow subpaths over broad barrels and #76756 has not landed yet.
