## Summary

Keep open: current main and `v2026.6.10` still lack the requested public status type exports, while the `ModelCatalogEntry` part is already reachable through `plugin-sdk/agent-runtime`. The linked open PR is the narrow implementation candidate, so this issue should stay open for Plugin SDK API-surface review rather than cleanup close.

## What This Changes

Request to expose canonical status and model catalog TypeScript types through the public Plugin SDK so plugins do not copy or guess gateway payload shapes.

## Best Possible Solution

Review and either land or reject the linked narrow PR; if accepted, expose only the approved status payload types on `plugin-sdk/command-status` and leave `ModelCatalogEntry` on the existing `plugin-sdk/agent-runtime` path.

## Reproduction Assessment

Yes. Source inspection against current main and `v2026.6.10` shows the core status types exist but are not exported from the public `plugin-sdk/command-status` subpath; the model catalog type is already reachable through `plugin-sdk/agent-runtime`.

## Solution Assessment

Unclear until maintainers make the API-surface call. The linked PR is a narrow and plausible implementation for the remaining status-type gap, but promoting these payload types to public SDK contract is the decision that still matters.
