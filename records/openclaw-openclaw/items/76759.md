## Summary

Keep open: current main and the latest release still lack the requested public `StatusSummary`/`SessionStatus` export, while https://github.com/openclaw/openclaw/pull/77184 is an open implementation candidate. The `ModelCatalogEntry` part is already reachable through the existing public `agent-runtime` surface.

## What This Changes

Request to export canonical status and model catalog types from the Plugin SDK public surface so third-party plugins do not copy or guess host payload shapes.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/77184 as the canonical status-type export path; keep `ModelCatalogEntry` on `agent-runtime` unless maintainers choose a focused model-catalog subpath.

## Reproduction Assessment

Yes. Source inspection shows the canonical status types exist in `src/commands/status.types.ts`, while the public `command-status` SDK subpath does not export them on current main or `v2026.6.6`.

## Solution Assessment

Unclear until maintainers accept the public SDK boundary. The linked PR is a narrow plausible fix for the remaining status exports, while `ModelCatalogEntry` is already available through `agent-runtime`.
