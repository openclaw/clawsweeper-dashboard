## Summary

Keep open. Current main still keeps the requested shapes in internal command/agent modules rather than the plugin SDK, and the SDK boundary rules make this an intentional public API decision rather than conservative cleanup.

## What This Changes

The issue requests public plugin-sdk exports for StatusSummary, SessionStatus, and ModelCatalogEntry so plugins can type gateway status and model catalog responses.

## Best Possible Solution

Define a narrow, documented SDK contract for read-only gateway status and model catalog responses alongside the #76756 runtime access decision, then update exports, API baseline/docs, and type-level coverage together.

## Reproduction Assessment

Not applicable as a feature request, but source inspection confirms the gap: the host types exist internally and the plugin-sdk export search does not find them on the public surface.

## Solution Assessment

Unclear. The requested type exposure is reasonable, but the maintainable fix should follow the SDK boundary policy and coordinate with the gateway RPC surface rather than adding an ad hoc broad re-export.
