## Summary

Keep open. Current main still lacks the requested `StatusSummary`/`SessionStatus`/`HeartbeatStatus` exports from the public `command-status` SDK subpath, and the associated closing PR is open, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Export canonical status payload and model catalog types through the public Plugin SDK so plugins can consume gateway status/model payloads without copying host shapes.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/77184; keep the status payload types on `command-status` and leave `ModelCatalogEntry` on `agent-runtime` unless maintainers choose a broader SDK barrel change.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw/plugin-sdk/command-status` exports only builders/options while the canonical status types exist in `src/commands/status.types.ts`, so a public import of those status types would fail.

## Solution Assessment

Yes. The narrow maintainable solution is an additive type-only re-export from the existing focused SDK subpath; the open linked PR follows that path, with normal review/proof/conflict handling still remaining.
