## Summary

Keep open. Current main still lacks the requested public status payload type exports, and the paired closing PR remains open; `ModelCatalogEntry` is already reachable through the existing `agent-runtime` SDK surface.

## What This Changes

Request to expose canonical status summary/session payload and model catalog types through the public Plugin SDK so plugins do not copy or guess host shapes.

## Best Possible Solution

Review and land or explicitly close https://github.com/openclaw/openclaw/pull/77184; keep the status payload types on the existing `command-status` subpath and avoid duplicating `ModelCatalogEntry` unless maintainers choose a broader SDK barrel change.

## Reproduction Assessment

## Solution Assessment

Yes. The narrow maintainable path is an additive type-only re-export from the existing focused `command-status` SDK subpath with API-baseline/docs coverage; the open linked PR is already following that direction.
