## Summary

Keep open: current `main` defines the host status/model types and RPC handlers, but it does not expose `StatusSummary` or `SessionStatus` from any checked SDK entrypoint, and `ModelCatalogEntry` is only indirectly available from `plugin-sdk/agent-runtime`, not the gateway-facing surface requested here.

## What This Changes

Request to export `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` as canonical public Plugin SDK types for plugins consuming status and model catalog data.

## Best Possible Solution

Promote these as documented, type-only Plugin SDK contracts on the narrow gateway-facing surface, update API baseline/docs, and coordinate the final shape with #76756.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API-surface request. Source inspection confirms the host types exist and the checked public SDK surfaces do not currently export the requested status types.

## Solution Assessment

Unclear that the proposed root-barrel re-export is the best final shape. The maintainable direction appears to be a narrow gateway-runtime or dedicated type surface, because existing SDK docs and tests keep the root surface intentionally small.
