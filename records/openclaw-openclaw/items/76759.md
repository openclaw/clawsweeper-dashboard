## Summary

Keep open: current main defines the host status and model catalog shapes, but the gateway-facing SDK surface still does not export `StatusSummary` or `SessionStatus`; `ModelCatalogEntry` is only reachable through the broad `plugin-sdk/agent-runtime` subpath. The related gateway RPC request #76756 remains open and complementary, not a duplicate, because this issue is specifically about stable public types for those RPC results.

## What This Changes

The issue requests exporting `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` from the plugin SDK public surface for plugins consuming gateway status and model catalog data.

## Best Possible Solution

Add a narrow gateway-facing SDK type contract, document it, and cover it with SDK export/API-baseline tests without exposing broad command internals or adding the RPC accessor work from #76756 in the same change.

## Reproduction Assessment

Yes. Current main defines the host response types and gateway RPCs, but source inspection shows the SDK entrypoints plugins would reasonably import from do not expose `StatusSummary` or `SessionStatus`.

## Solution Assessment

Yes for a narrow SDK type facade; the issue's direct root-barrel suggestion is less ideal than exporting these from `plugin-sdk/gateway-runtime` or a dedicated gateway type subpath that matches existing SDK boundary rules.
