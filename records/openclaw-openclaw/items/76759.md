## Summary

Keep open: current main partially supports the request because `ModelCatalogEntry` is already public through `plugin-sdk/agent-runtime`, but the status payload types are still internal and an open PR is already aimed at the missing SDK export.

## What This Changes

Request to expose canonical status and model-catalog TypeScript types through the public plugin SDK so plugins do not copy or guess gateway payload shapes.

## Best Possible Solution

Land a narrow SDK public-surface change that exports status payload types from the existing `command-status` subpath, keeps `ModelCatalogEntry` on `agent-runtime`, and updates the API baseline, docs, changelog, and focused type coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows the missing public status-type export path and confirms the canonical `StatusSummary` shape the issue cites; no runtime test is needed for this type-surface gap.

## Solution Assessment

Yes. A type-only export through existing focused SDK subpaths is the narrowest maintainable direction, and open PR #77184 is the active implementation candidate rather than a reason to close this issue before merge.
