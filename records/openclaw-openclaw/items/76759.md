## Summary

Keep open: current main still lacks the requested status response types on the public plugin SDK surface, and the remaining choice is an additive SDK API contract that should be coordinated with the still-open gateway accessor issue.

## What This Changes

The issue requests exporting `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` through the public plugin SDK type surface for plugins consuming gateway status and model responses.

## Best Possible Solution

Promote the gateway response row types through the narrow SDK subpath chosen for curated gateway accessors, then update the SDK export checks, API baseline, and plugin docs together.

## Reproduction Assessment

Yes for the type-surface gap: current source shows the internal status/model types and gateway methods, while the public runtime/root SDK surfaces do not expose the requested status response contract. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers choose the SDK API shape with #76756. A type-only export is small, but the SDK boundary guide prefers narrow documented subpaths over broad barrel growth.
