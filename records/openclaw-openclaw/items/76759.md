## Summary

Keep open: current main still has the internal status and model catalog types, but the requested SDK exports are not present, and promoting them is an additive public API decision tied to the still-open gateway accessor work.

## What This Changes

Export `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` from the public plugin SDK so plugins can type status and model gateway responses.

## Best Possible Solution

Promote these response row types through a narrow documented SDK contract aligned with #76756, updating SDK docs, export checks, and API baselines together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug reproduction; yes as a source check, because the internal types exist while exact-word searches show they are absent from the public SDK surface.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the SDK API shape; a type-only export is small, but the SDK guide favors intentional narrow public contracts over broad convenience re-exports.
