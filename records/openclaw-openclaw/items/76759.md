## Summary

Keep open: current main and the latest release still lack the requested public `StatusSummary` and `SessionStatus` exports, and https://github.com/openclaw/openclaw/pull/77184 is an open implementation candidate for the remaining status-type surface. The `ModelCatalogEntry` portion is already reachable through the public `plugin-sdk/agent-runtime` surface.

## What This Changes

The issue requests public Plugin SDK type exports for status and model catalog payload shapes so plugins do not copy or guess host gateway response types.

## Best Possible Solution

Review and either land, narrow, or reject https://github.com/openclaw/openclaw/pull/77184 as the canonical status-type export path; keep `ModelCatalogEntry` on `plugin-sdk/agent-runtime` unless maintainers choose a focused model-catalog subpath.

## Reproduction Assessment

Yes, at source level. Current main defines the internal status payload types in `src/commands/status.types.ts`, while `src/plugin-sdk/command-status.ts` and `v2026.6.8` do not export them from the public SDK subpath.

## Solution Assessment

Unclear until maintainers accept the public SDK boundary. The linked PR is a narrow plausible fix for the missing status exports, while `ModelCatalogEntry` is already available through `plugin-sdk/agent-runtime`.
