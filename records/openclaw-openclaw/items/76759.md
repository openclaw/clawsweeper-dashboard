## Summary

Keep open: current main still lacks a public plugin-sdk export for the gateway status types, and the requested SDK type promotion is valid but should be aligned with the narrow subpath convention and the related gateway-RPC work.

## What This Changes

Request to expose `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` as stable plugin-sdk imports for plugins that consume gateway status and model catalog results.

## Best Possible Solution

Promote these as additive plugin-sdk type exports on the narrow gateway/model-facing public surface, document the import path, and coordinate the result shape with the gateway-RPC access tracked in #76756.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API-surface request. Source inspection does confirm the requested status types exist internally and are not exported from the current root or gateway plugin-sdk surfaces.

## Solution Assessment

Partly: exporting the canonical types is the right direction, but a simple root barrel re-export is not clearly the best fit. The safer implementation is a focused public SDK subpath export with API baseline/docs coverage, leaving runtime gateway access to #76756.
