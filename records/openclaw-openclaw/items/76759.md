## Summary

Keep open: current main still lacks public SDK exports for the status response types, and deciding where those types belong is an additive SDK contract choice tied to the open gateway-accessor request in #76756. The report is useful because the internal status shape is concrete and easy to drift from when plugins guess it.

## What This Changes

Request to add public plugin SDK exports for gateway status/session summary types and the model catalog row type so plugins can type host RPC responses.

## Best Possible Solution

Add a narrow, documented SDK contract for gateway status and model-list response types alongside #76756, keeping API baseline, export checks, package exports, and docs aligned.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The source-level gap is high-confidence: the status types exist internally and a focused search finds no `StatusSummary` or `SessionStatus` export under `src/plugin-sdk`.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type export is likely the right end state, but a broad barrel re-export would not be as maintainable as a narrow gateway-aligned contract updated with #76756.
