## Summary

Keep open: current main still has the requested response types only on internal host surfaces, and promoting them into the plugin SDK remains an additive public API decision tied to the open gateway-access work.

## What This Changes

Export `StatusSummary`, `SessionStatus`, and `ModelCatalogEntry` from the public plugin SDK so plugin authors can type gateway status and model-list responses.

## Best Possible Solution

Publish a narrow documented SDK contract for gateway status and model catalog response types, then update SDK export checks, API baseline, package exports, and docs together.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; yes for source reproducibility. Current main defines the internal types and a focused search shows the public SDK surfaces do not export them.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the SDK guide favors narrow documented subpaths and #76756 has not landed yet.
