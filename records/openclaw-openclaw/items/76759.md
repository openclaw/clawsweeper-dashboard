## Summary

Keep open: current main and the latest release still lack the requested public status type exports, while `ModelCatalogEntry` is already reachable through `plugin-sdk/agent-runtime`; https://github.com/openclaw/openclaw/pull/77184 remains the open implementation candidate and needs maintainer Plugin SDK API-surface review before this issue can close.

## What This Changes

Request public Plugin SDK exports for status summary/session status and model catalog row types so plugins can type gateway status/model data without copying internal shapes.

## Best Possible Solution

Review and either land, narrow, or decline https://github.com/openclaw/openclaw/pull/77184 as the canonical status-type export path; keep `ModelCatalogEntry` on `plugin-sdk/agent-runtime` unless maintainers choose a focused model-catalog subpath.

## Reproduction Assessment

Yes, at source level: current main defines the internal status payload types but `plugin-sdk/command-status` exports only command/help builders, and `v2026.6.10` has the same gap.

## Solution Assessment

Unclear as product/API direction: the existing `command-status` subpath is a narrow plausible owner boundary, but maintainers must decide whether full `StatusSummary` is stable public Plugin SDK API or whether a narrower DTO is better.
