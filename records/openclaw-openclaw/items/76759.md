## Summary

Keep open: current main still has the internal status and model catalog types, but the public plugin SDK surface does not export `StatusSummary`, `SessionStatus`, or `ModelCatalogEntry`; promoting them is an additive SDK API decision that should be coordinated with #76756.

## What This Changes

Request to export the gateway status summary/session status and model catalog row types from the public plugin SDK.

## Best Possible Solution

Promote a narrow documented SDK contract for these response types alongside the gateway accessors, with SDK export metadata, API baseline, docs, and checks updated together.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The type-surface gap is source-reproducible: the internal types exist and a focused SDK/export search shows they are not public today.

## Solution Assessment

Unclear until maintainers choose the SDK API shape. A type-only export is small, but the SDK boundary favors narrow documented subpaths and #76756 has not landed yet.
