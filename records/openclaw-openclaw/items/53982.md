## Summary

Keep this open. Current main and the latest shipped release still show an incomplete Nextcloud Talk bot install command and still turn outbound HTTP 401 failures into a secret-only diagnostic, while upstream Nextcloud Talk stable32 requires the response feature for bot message sends.

## What This Changes

The issue asks OpenClaw to document that Nextcloud Talk bots need `--feature response` for outbound replies and/or diagnose that missing feature when sends fail with HTTP 401.

## Best Possible Solution

Update the Nextcloud Talk docs and setup wizard to register bots with `--feature webhook --feature response --feature reaction`, then improve the outbound 401 diagnostic to mention both bot-secret mismatch and missing `--feature response`, with focused regression tests for setup text and send error handling.
