## What This Changes

The issue requests documenting `--feature response` for Nextcloud Talk bot setup or detecting that missing capability when outbound replies fail with HTTP 401.

## Best Possible Solution

Keep the merged Nextcloud Talk plugin-owned fix and direct existing installs to enable the Talk bot response feature with `talk:bot:state` or reinstall using the current setup command.

Yes. The original failure is source-reproducible from the documented old setup command plus Nextcloud Talk stable32 filtering outbound bot messages by `FEATURE_RESPONSE`; current main no longer documents the failing command and has regression coverage for the diagnostic path.

Yes. The merged fix is the narrow maintainable solution because it stays in the Nextcloud Talk plugin/docs surface, preserves the real secret-mismatch hint, and adds doctor/status detection without changing core channel contracts.
