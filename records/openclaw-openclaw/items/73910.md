## Summary

Keep open. The timeout-config rejection is already guarded on main and included in v2026.4.29, but the Codex ACP wrapper still forces an isolated generated CODEX_HOME without auth.json; #73943 is the open closing PR for the remaining auth bridge.

## What This Changes

The issue asks OpenClaw-managed Codex ACP to reuse or document Codex auth for its isolated CODEX_HOME and stop sending unsupported timeout config.

## Best Possible Solution

Land or replace #73943 with a narrowly reviewed auth.json bridge or documented setup flow that keeps generated config isolation and preserves the shipped Codex timeout guard.

## Reproduction Assessment

Yes. Current main sets CODEX_HOME to an isolated generated directory, writes only config.toml there, and has tests asserting auth.json is not copied; the issue and macOS confirmation both report that adding an auth.json bridge gets Codex ACP past authentication.

## Solution Assessment

Unclear until #73943 is reviewed. The timeout guard is the narrow maintainable fix for unsupported config forwarding, while the auth bridge should be limited to required auth.json material and must not import broader Codex config or hooks.
