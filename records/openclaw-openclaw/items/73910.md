## Summary

Keep open. Current main has the Codex ACP timeout guard, but the isolated Codex home still contains only generated config and no auth bridge; open PR #73943 explicitly tracks the remaining auth fix, so this issue should stay open until that PR or an equivalent maintainer-reviewed fix lands.

## What This Changes

The request is to make managed Codex ACP sessions work by safely bridging or documenting auth for the isolated CODEX_HOME and by avoiding unsupported Codex ACP timeout config forwarding.

## Best Possible Solution

Keep the existing timeout guard, then land or replace #73943 with a narrow maintainer-reviewed auth fix: bridge only the minimum Codex auth material or provide a supported isolated-home login flow, preserve generated config isolation, add regression coverage, and update the ACP setup docs.
