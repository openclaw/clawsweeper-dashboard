## Summary

Keep this open: current main still has the reported schema composition bug, and the latest release tag still contains the same qqbot/webhooks manifest shape. The only current fix candidate in the provided context is an open external fork PR, so this issue should stay open until a target-repo fix lands or maintainers choose a different generic direction.

## What This Changes

The issue asks OpenClaw to generate a config schema whose plugin `$defs` and `$ref` targets resolve correctly instead of nesting qqbot definitions under the plugin config node while refs point at the document root.

## Best Possible Solution

Land a target-repo fix that makes schema composition preserve valid JSON Schema references, preferably in `src/config/schema.ts` by hoisting or namespacing plugin/channel root `$defs` and rewriting local refs, with regression coverage in `src/config/schema.test.ts`. Manifest inlining can work for qqbot/webhooks, but it does not protect third-party or future plugin schemas unless maintainers explicitly choose that narrower scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `openclaw config schema` reads manifest metadata, nests each plugin `configSchema` under `plugins.entries.<id>.config`, and current qqbot/webhooks schemas still contain root-relative `#/$defs/...` refs with no generated root definitions.

## Solution Assessment

No current solution is landed on main. The open external PR is a useful candidate, but the safer maintainable direction is a generic schema merger update plus tests unless maintainers intentionally scope the fix to bundled manifest inlining.
