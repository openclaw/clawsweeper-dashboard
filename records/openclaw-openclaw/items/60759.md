## What This Changes

The issue reports that generated `openclaw-schema.json` nests plugin `$defs` under the qqbot config schema while leaving `$ref` pointers rooted at `#/$defs/...`.

## Best Possible Solution

Land a target-repo schema merger fix that hoists or namespaces plugin/channel `$defs` and rewrites local refs, with regression coverage for plugin and channel schemas.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: manifest metadata still contains root-relative `$ref`s, and `openclaw config schema` feeds those schemas through a merger that nests them under plugin/channel config nodes without repair.

## Solution Assessment

No landed solution exists on current main. The most maintainable fix is generic schema-merger handling plus tests; manifest-only inlining is narrower and leaves the underlying plugin/channel schema contract fragile.
