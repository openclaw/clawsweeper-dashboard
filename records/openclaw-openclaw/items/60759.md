## What This Changes

The issue reports that generated `openclaw-schema.json` nests plugin `$defs` under the qqbot config schema while `$ref` values still point at root `$defs`, breaking editor JSON Schema resolution.

## Best Possible Solution

Land a target-repo generic schema merger fix that hoists or collision-safely namespaces plugin/channel `$defs`, rewrites local root refs, and adds regression coverage for plugin and channel schemas.

## Reproduction Assessment

Yes from source inspection: current manifests still contain `$defs` with `#/$defs/...` refs, and current schema merging still inserts those schemas under nested plugin/channel config nodes unchanged. I did not run the CLI/editor flow in this read-only review, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No landed solution exists on current main. The best fix is a generic merger repair with collision-safe hoisting/ref rewriting; manifest-only inlining would fix some bundled plugins but leave the plugin/channel schema contract fragile.
