## Summary

Keep this open: current `main` still has the reported schema composition bug, and the related target-repo fix attempts are closed unmerged while the only open fix candidate in the provided context is external.

## What This Changes

The issue asks for generated `openclaw-schema.json` to keep plugin `$defs` resolvable at the schema root instead of nesting them under `plugins.entries.qqbot.config`.

## Best Possible Solution

Land a target-repo schema composition fix that hoists or namespaces plugin/channel `$defs` and rewrites local `$ref`s, with regression coverage for both plugin and channel schemas.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: manifest metadata still contains root-relative `$ref`s, and `openclaw config schema` feeds those schemas through a merger that nests them without repair.

## Solution Assessment

No landed solution exists on current `main`. The most maintainable fix is generic schema-merger handling plus tests, unless maintainers explicitly choose the narrower bundled-manifest inlining approach.
