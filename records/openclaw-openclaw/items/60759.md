The target-repo fix PRs I found are closed unmerged, and the only open linked PR is in a fork rather than a mergeable target-repo PR.

## What This Changes

The issue asks OpenClaw to generate `openclaw-schema.json` with plugin and channel `$defs` resolvable from the schema root instead of nested under qqbot config.

## Best Possible Solution

Land a target-repo generic schema composer fix that hoists or namespaces plugin/channel `$defs`, rewrites local root refs safely, and adds regression coverage for plugin and channel schemas.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current manifests still declare `$defs` with `#/$defs/...` refs, and current schema assembly nests those schemas unchanged under plugin/channel config nodes. I did not run the CLI because this review had to keep the checkout read-only.

No landed solution exists on current main. The best fix is a generic merger repair with collision-safe hoisting or namespacing and ref rewriting; manifest-only inlining would leave the plugin/channel schema contract fragile.
