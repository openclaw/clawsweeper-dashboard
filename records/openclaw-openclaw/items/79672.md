## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Respect the caller timeout for capability lookup:** `src/video-generation/runtime.ts:165`
  - body: The new capability overlay always gives each candidate a 5s lookup budget before provider execution. For requests with a smaller `timeoutMs`, fallback chains can now wait several seconds per candidate before the actual provider deadline starts, so short-deadline callers no longer get the bounded behavior documented by `GenerateVideoParams.timeoutMs`.
  - confidence: 0.94
- **[P2] Evict expired live-catalog cache entries:** `src/plugin-sdk/provider-catalog-shared.ts:57-64`
  - body: Expired entries are skipped only when the exact same hashed key is requested again. Because keys include provider, auth, and base-url facts, token rotation or many provider contexts leave dead promises in this module-level map for the lifetime of the gateway, producing unbounded memory growth in discovery-heavy processes.
  - confidence: 0.9
