  - confidence: 0.96

  - body: The current managed-image endpoint requires authenticated owner/admin context before serving persisted bytes and explicitly denies trusted-proxy or device-token callers that only self-declare a session. This document handler treats trusted-scope and device-token auth as privileged before the owner check, so mirror the image endpoint contract and add denial coverage.
  - confidence: 0.94
- **[P1] Preserve outgoing-doc URLs in WebChat:** `src/gateway/managed-document-attachments.ts:447-450`
  - body: The new blocks emit `/api/chat/media/outgoing-doc/...`, but current WebChat only exempts `/api/chat/media/outgoing/` from local assistant-media wrapping. Without updating the renderer, these document links can be rewrapped through `/__openclaw__/assistant-media` and lose the headers this PR adds.
  - confidence: 0.91
- **[P2] Read document bytes through the safe file reader:** `src/gateway/managed-document-attachments.ts:914`
  - body: The image sibling reads stored media through `readLocalFileSafely`, but the new document endpoint reads the persisted path with `fs.readFile`. Use the same safe reader boundary so a stale, tampered, or symlinked record path cannot broaden what the HTTP endpoint serves.
  - confidence: 0.86
- **[P2] Strip document blocks when transcript append fails:** `src/gateway/server-methods/chat.ts:469-470`
  - body: Document blocks are added to assistant content, but the existing fallback strip helper only removes managed image blocks. If transcript append fails, the client can receive outgoing-doc URLs that were never promoted to transcript-backed records and then 404.
  - confidence: 0.85
- **[P3] Guard the document route before lazy import:** `src/gateway/server-http.ts:750-753`
  - body: This stage is pushed for every request, so unrelated gateway requests import the document attachment module before the handler can return `false`. Match the current image route guard to keep gateway hot paths from materializing the handler unnecessarily.
  - confidence: 0.82
- **[P3] Remove the direct changelog edit:** `CHANGELOG.md:152`
  - body: `CHANGELOG.md` is release-owned in this repo. Keep the release-note context in the PR body or commit message, but remove the direct changelog entry from this normal PR.
