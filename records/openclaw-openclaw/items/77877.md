Overall confidence: 0.9

Full review comments:

- **[P1] Require owner auth before serving document bytes:** `src/gateway/managed-document-attachments.ts:843-845`
  - body: The new document route treats device-token/trusted-scope requests as privileged and otherwise trusts a client-declared requester-session header before reading `record.original.path`. Current managed images require authenticated owner/admin context for byte fetches because requester-session headers are not an authorization boundary, so this endpoint needs the same owner check and safe file-read behavior before serving documents.
  - confidence: 0.92
- **[P2] Update the media-core import for current main:** `src/gateway/managed-document-attachments.ts:27`
  - body: This new module imports `mediaKindFromMime` and `MAX_DOCUMENT_BYTES` from `../media/constants.js`, but current main moved those exports to `@openclaw/media-core/constants` and no longer has `src/media/constants.ts`. The branch needs a current-main rebase/import repair before it can build or be validated.
  - confidence: 0.91
- **[P1] Preserve outgoing-doc URLs in WebChat:** `src/gateway/server-methods/chat.ts:460-463`
  - body: The PR returns `/api/chat/media/outgoing-doc/...` document URLs, but current WebChat only treats `/api/chat/media/outgoing/` image URLs as already-managed. Without updating the UI preservation path, these document links can be wrapped through `/__openclaw__/assistant-media`, dropping the new attachment response headers that are the user-visible fix.
  - confidence: 0.9
- **[P1] Strip document blocks when transcript append fails:** `src/gateway/server-methods/chat.ts:460-463`
  - body: Document blocks are now added to assistant content, but the append-failure fallback still strips only managed image blocks. If transcript append fails, WebChat can broadcast outgoing-doc links whose records were never promoted into history, so users receive dead 404 document links.
  - confidence: 0.88
- **[P2] Encode non-ASCII download filenames:** `src/gateway/managed-document-attachments.ts:926-929`
  - body: `safeAttachmentFilename` preserves Unicode and interpolates it directly into `filename="..."`. Node HTTP headers must be Latin-1 compatible, so a filename like `報告.pdf` can throw `ERR_INVALID_CHAR`; use an ASCII fallback plus `filename*` encoding before setting Content-Disposition.
  - confidence: 0.86
- **[P3] Guard the document route before importing it:** `src/gateway/server-http.ts:750-753`
  - body: The new request stage is pushed unconditionally, so requests that are not outgoing-doc downloads still import and run the document module before Control UI handling. Mirror the managed-image path guard so ordinary gateway requests do not pay this lazy import or new filesystem module cost.
  - confidence: 0.77
- **[P3] Remove the release-owned CHANGELOG edit:** `CHANGELOG.md:149`
  - body: OpenClaw release generation owns `CHANGELOG.md`; normal PRs should keep release-note context in the PR body or squash message instead of editing the changelog directly. Please drop this line from the branch.
  - confidence: 0.82
