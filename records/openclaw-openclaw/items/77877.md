Full review comments:

- **[P1] Require owner auth before serving document bytes:** `src/gateway/managed-document-attachments.ts:880-895`
  - body: The new document handler accepts `device-token`/trusted-scope bypasses or a client-declared requester session before reading `record.original.path`, but current managed images now require authenticated owner/admin context for byte fetches because requester-session headers are not a trustworthy authorization boundary. Please match the managed image route's owner check before serving local document bytes.
  - confidence: 0.9
- **[P1] Preserve outgoing-doc URLs in WebChat:** `src/gateway/server-methods/chat.ts:460-463`
  - body: This returns `/api/chat/media/outgoing-doc/...` document URLs, but current WebChat only treats `/api/chat/media/outgoing/` as already-managed; `outgoing-doc` paths are still considered local sources and get wrapped through `/__openclaw__/assistant-media`, dropping the new `Content-Disposition` response. The PR needs the UI preservation path and regression coverage before the user-visible download fix holds.
  - confidence: 0.92
- **[P1] Strip document blocks when transcript append fails:** `src/gateway/server-methods/chat.ts:460-463`
  - body: Document blocks are now added to assistant content, but the append-failure fallback stripper still removes only managed image blocks. If the transcript append fails, WebChat can broadcast an outgoing-doc URL that was never persisted into the transcript, and the new handler later 404s it via `recordMatchesTranscriptMessage`.
  - confidence: 0.86
- **[P2] Encode non-ASCII download filenames:** `src/gateway/managed-document-attachments.ts:926-929`
  - body: `safeAttachmentFilename` preserves Unicode and interpolates it into `filename="..."`, but Node rejects non-Latin-1 header content with `ERR_INVALID_CHAR`. Use an ASCII fallback plus `filename*` encoding, and add a regression for a filename such as `報告.pdf`.
  - confidence: 0.95
- **[P2] Rebase the media constants import onto current main:** `src/gateway/managed-document-attachments.ts:27`
  - body: This import points at `../media/constants.js`, but current main has moved `mediaKindFromMime` and `MAX_DOCUMENT_BYTES` to `@openclaw/media-core/constants`. The branch needs a rebase and import repair before review or CI can validate the actual merge result.
  - confidence: 0.88
- **[P3] Guard the document route before importing it:** `src/gateway/server-http.ts:750-763`
  - body: The image stage is only registered for matching `/api/chat/media/outgoing/` paths, but this document stage is pushed for every request and imports the new module before returning false for unrelated paths. Add a path predicate so an import failure or heavier document module cannot break unrelated Control UI/gateway requests.
  - confidence: 0.8
