Full review comments:

- **[P1] Import the current media-core contracts:** `src/gateway/managed-document-attachments.ts:27-30`
  - body: Current main no longer exposes these media constants/helpers from `src/media`, so the new module will not survive a current-main rebase or typecheck. Update it to the current `@openclaw/media-core` and local media helper contracts before merge.
  - confidence: 0.96
- **[P1] Require owner auth before serving document bytes:** `src/gateway/managed-document-attachments.ts:843-844`
  - body: The current managed-image endpoint requires owner/admin semantics before returning stored bytes, but this branch treats trusted declared scopes and device-token auth as privileged for documents. Mirror the image endpoint boundary so chat-history-scoped callers cannot fetch persisted documents by self-declaring a session.
  - confidence: 0.93
- **[P1] Preserve outgoing-doc URLs in the renderer:** `src/gateway/managed-document-attachments.ts:447-450`
  - body: These blocks emit `/api/chat/media/outgoing-doc/...`, but current WebChat only exempts `/api/chat/media/outgoing/` from local attachment wrapping. Without a renderer update, the document link is sent through `/__openclaw__/assistant-media` and loses the new attachment headers.
  - confidence: 0.9
- **[P2] Strip document blocks when transcript append fails:** `src/gateway/server-methods/chat.ts:2735-2737`
  - body: The fallback path still strips only managed image URLs. If transcript append fails after document blocks are created, the client can receive `/outgoing-doc/` links whose records were never transcript-backed and will later 404.
  - confidence: 0.86
- **[P3] Guard the document route before lazy import:** `src/gateway/server-http.ts:750-753`
  - body: The document media stage is registered for every gateway request, unlike the current image route which checks the path prefix before importing the handler. Add the same path guard so unrelated HTTP requests do not load the document module on the hot path.
  - confidence: 0.82
