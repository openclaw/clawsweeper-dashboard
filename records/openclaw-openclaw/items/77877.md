Full review comments:

- **[P1] Require owner auth for document downloads:** `src/gateway/managed-document-attachments.ts:843-844`
  - body: The current managed-image endpoint now requires owner/admin access before bytes are served, but this new document endpoint treats device-token/trusted-scope requests as privileged and otherwise trusts `x-openclaw-requester-session-key`. Mirror `resolveOpenAiCompatibleHttpSenderIsOwner` from the image route and add document denial tests for device-token and header-only access.
  - confidence: 0.91
- **[P2] Strip managed document blocks on append failure:** `src/gateway/server-methods/chat.ts:469-470`
  - body: Document blocks are now added to `assistantContent`, but the append-failure fallback still uses a helper that only removes image blocks. If transcript append fails, the client can receive `/api/chat/media/outgoing-doc/...` URLs for transient records that cannot pass transcript membership and will 404, so extend the strip/detection path to document attachment blocks.
  - confidence: 0.86
- **[P3] Guard the document route before importing it:** `src/gateway/server-http.ts:750-763`
  - body: Unlike the existing image stage, this document stage is pushed for every request, so ordinary non-document requests still import and run the new handler until it returns false. Add an `isManagedOutgoingDocumentPath(scopedRequestPath)` guard like the image route to preserve the gateway hot path.
  - confidence: 0.84
