## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require owner auth for document downloads:** `src/gateway/managed-document-attachments.ts:843-848`
  - body: The current managed-image endpoint was hardened to require owner access before bytes are served, but this new document endpoint treats device-token/trusted scopes as privileged and otherwise trusts `x-openclaw-requester-session-key`. Please mirror the current image endpoint's owner-auth check and add tests that reject device-token or requester-header-only access for document downloads.
  - confidence: 0.91
- **[P2] Strip document URLs when transcript append fails:** `src/gateway/server-methods/chat.ts:460-470`
  - body: Adding document blocks here means the existing append-failure fallback can now broadcast `/api/chat/media/outgoing-doc/...` links, but `stripManagedOutgoingAssistantContentBlocks` still only removes image blocks. Those document records remain transient/unattached, and the handler later returns 404 because transcript membership is required, so extend the managed outgoing detection/strip path to include document attachment blocks.
  - confidence: 0.86
- **[P3] Guard the document route before loading it:** `src/gateway/server-http.ts:750-763`
  - body: This stage is pushed for every request, and `runGatewayHttpRequestStages` runs each stage until one handles the request, so ordinary non-document requests still import and execute the new document handler just to return false. Add an `isManagedOutgoingDocumentPath(scopedRequestPath)` guard like the existing image route to preserve the gateway hot path.
  - confidence: 0.84
