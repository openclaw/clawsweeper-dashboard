## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require owner auth before serving document bytes:** `src/gateway/managed-document-attachments.ts:843-845`
  - body: This endpoint serves persisted local document bytes, but it treats device-token access as privileged and otherwise trusts `x-openclaw-requester-session-key` ownership. Current managed-image serving requires `resolveOpenAiCompatibleHttpSenderIsOwner(...)` before reading bytes, so this new document route would grant broader access than the sibling route it mirrors.
  - confidence: 0.93
- **[P1] Preserve outgoing-doc URLs in WebChat:** `src/gateway/managed-document-attachments.ts:447-448`
  - body: The block URL uses `/api/chat/media/outgoing-doc/...`, but current WebChat only exempts `/api/chat/media/outgoing/` from local-source wrapping. Document attachments from this PR can therefore be routed through `/__openclaw__/assistant-media`, losing the `Content-Disposition: attachment` response this route adds.
  - confidence: 0.9
- **[P1] Strip document blocks when transcript append fails:** `src/gateway/server-methods/chat.ts:460-470`
  - body: Adding document blocks here also needs the append-failure fallback updated. Today `stripManagedOutgoingAssistantContentBlocks` removes managed image blocks only, so an append failure can still broadcast outgoing-doc links whose records remain transient and will fail the transcript-membership check later.
  - confidence: 0.88
- **[P2] Encode non-ASCII download filenames:** `src/gateway/managed-document-attachments.ts:927-928`
  - body: `safeAttachmentFilename` preserves Unicode and the handler interpolates it directly into `filename="..."`. Node rejects non-ASCII header content with `ERR_INVALID_CHAR`, so filenames such as `報告.pdf` can make the new route fail; use an ASCII fallback plus `filename*` encoding.
  - confidence: 0.91
- **[P3] Guard the document stage before importing it:** `src/gateway/server-http.ts:750-753`
  - body: Unlike the managed-image stage above, this stage is pushed for every HTTP request and only returns false after importing the document module. Add the same path guard shape for `/api/chat/media/outgoing-doc/` so unrelated gateway requests do not pay the lazy import/startup cost.
  - confidence: 0.78
