## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P2] Harden the shared state DB open path:** `src/durable/sqlite-store.ts:596-599`
  - body: This durable store can be the first code path to create `state/openclaw.sqlite`, but it only creates the directory and opens `DatabaseSync`. The existing shared state DB opener hardens the DB, WAL, and SHM files because the state DB contains credentials-adjacent metadata; durable should reuse or factor that permission path before/after open so opt-in durable metadata is not left at umask-dependent file modes.
  - confidence: 0.9
- **[P2] Do not advertise unsupported write controls:** `src/durable/coordination-projection.ts:270-285`
  - body: The projection marks cancel/retry/resume/signal as available based only on durable status, but this PR intentionally exposes only read-only `durable.coordination.get` and removed CLI write controls. Operator clients reading the documented supported controls can show dead controls that are not routed through the live agent/task runtime, so keep these false or omit them until matching write contracts exist.
  - confidence: 0.87
