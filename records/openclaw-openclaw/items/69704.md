## Summary

Keep this PR open. Current main already sends Brave API keys via `X-Subscription-Token` in both web and llm-context modes, but it still lacks the focused regression coverage this PR adds, and the patch is a small test-only change with no concrete functional or security finding.

## What This Changes

The PR adds two Brave provider regression tests covering `X-Subscription-Token` header auth and absence of `apikey`/`key` query parameters for web and llm-context modes.

## Best Possible Solution

Land this test-only PR, or add equivalent Brave provider coverage on main, so future refactors cannot regress from header auth back to query-parameter auth. No production runtime or core API change appears needed.

## Reproduction Assessment

Yes for the coverage gap: current main can be checked with `rg` and the Brave test file, which show no header/query-param regression assertions. The runtime bug itself is already fixed, so the remaining verification path is the targeted Brave provider test suite.

## Solution Assessment
