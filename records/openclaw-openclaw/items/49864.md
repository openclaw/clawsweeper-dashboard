## Summary

Keep open. Current main and the latest release still resolve Telegram inbound `requireMention` with persisted activation before explicit topic config, while the public Telegram docs say topic entries override group settings. The linked implementation PR #49980 is still open and unmerged, so this issue should stay open until that PR or a superseding fix lands.

## What This Changes

The issue asks for Telegram forum topic `requireMention` settings to override group-level mention gating so a bot can auto-respond in its configured topic while requiring mentions elsewhere.

## Best Possible Solution

Telegram inbound message handling should make an explicit topic `requireMention` value outrank persisted activation and group defaults, while preserving activation fallback when no topic override exists. The fix should include focused regression coverage for topic false over group true, topic false over `groupActivation: "mention"`, and the no-activation account/topic path raised in the discussion.

## Reproduction Assessment

Yes for the confirmed activation-precedence path: source inspection shows a session `groupActivation: "mention"` resolves to `true`, wins before topic config, and then triggers the `no-mention` skip. The no-activation variant is unclear from current code alone because existing tests already cover the simple topic-over-group case without activation.

## Solution Assessment

Yes, the direction in PR #49980 appears to be the narrowest maintainable fix for the confirmed bug: change the local precedence in the Telegram inbound context and add regression tests. The safer final review should also verify whether the no-activation commenter’s case needs an additional thread-id/config lookup fix.
