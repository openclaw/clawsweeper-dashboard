## Summary

Keep open, but this PR is not merge-ready: it is a coherent Google provider change, yet it moves shipped `google-gemini-cli` runtime/setup registration behind a new env opt-in without approved compatibility direction, upgrade proof, or real behavior proof.

## What This Changes

The PR adds an env-gated Google Gemini CLI harness policy, wraps Google plugin and setup registration in that opt-in, updates Google wizard copy, adds focused tests, and adds a Google provider rebuild doc.

## Best Possible Solution

## Solution Assessment

No. The safer solution is to preserve the existing default until maintainers approve the new opt-in policy, then align manifest, docs, onboarding, and upgrade proof with that decision.
