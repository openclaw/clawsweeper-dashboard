- `P1`: The PR targets and can regress a documented outbound interception hook used by safety, rewrite, and cancellation plugins on primary channel reply paths.

## Summary

Keep this PR open, but it is not merge-ready: it fixes a real shared dispatcher bypass while removing Telegram's delivery-owned `message_sending` hook coverage for paths that still call `deliverReplies` directly.

## What This Changes

The PR adds `message_sending` to configured auto-reply `beforeDeliver` chains, updates one dispatcher test, and removes Telegram `deliverReplies`' direct `message_sending` hook.

## Best Possible Solution

Keep the shared dispatcher repair, but preserve or replace Telegram delivery-owned hook coverage with an explicit exactly-once design covering direct `deliverReplies`, native commands, fallbacks, media, voice fallback, and streamed finalization.

## Reproduction Assessment

Yes. Current-main source shows configured `beforeDeliver` excludes `message_sending`, Telegram supplies configured `beforeDeliver`, and the existing dispatcher test still pins the skipped-hook behavior.

## Solution Assessment

No. Adding the dispatcher hook is a plausible part of the fix, but deleting Telegram's delivery-owned hook is not the best solution because direct delivery, fallback, media, and spokenText paths still need exactly-once `message_sending` coverage.
