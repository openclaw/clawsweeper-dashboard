## Summary

Keep open. The PR is a focused bug fix for a source-reproducible silent-token leak, and the later Mantis Telegram proof now shows the real behavior improving, but merging still needs maintainer acceptance because the shared exported suppression helper changes what token-only text is globally non-deliverable.

## What This Changes

The PR broadens the shared silent-reply exact matcher to suppress token-only replies with wrapping quotes or terminal punctuation and adds unit coverage for those cases.

## Best Possible Solution

Land the focused matcher and tests once maintainers accept quoted and punctuated token-only sentinels as globally silent, while keeping broader mixed-content `NO_REPLY` policy out of this PR.

## Reproduction Assessment

Yes. Current main's source only treats whitespace-wrapped tokens as silent, and the Mantis baseline shows `NO_REPLY.` leaking into Telegram; I did not run a local test because this review is read-only.

## Solution Assessment

Yes. The shared exact silent-token matcher is the narrow owner boundary for token-only punctuation and quote leaks; channel-specific suppression would duplicate policy, while broader mixed-content stripping would reopen the prior over-suppression risk.
