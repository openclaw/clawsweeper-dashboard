## What This Changes

The issue reports that VS Code 1.113 and Cursor send MCP date-string `protocolVersion: "2025-11-25"`, causing OpenClaw ACP initialization to fail with invalid params.

## Best Possible Solution

Refresh, land, or replace https://github.com/openclaw/openclaw/pull/56176 with a current-main stream-boundary normalizer before `AgentSideConnection`, matching the SDK uint16 contract and adding focused regression coverage plus redacted VS Code or Cursor proof.

## Reproduction Assessment

Yes, source-level. VS Code/Cursor logs show a string `protocolVersion`, current main passes it directly into SDK validation, and the SDK requires a uint16 number before OpenClaw's handler runs.

## Solution Assessment

Yes, the maintainable fix is a pre-`AgentSideConnection` stream normalizer that preserves valid ACP numeric clients and coerces MCP date-string initialize versions. The issue should stay open until the linked PR lands or an equivalent tested fix ships.
