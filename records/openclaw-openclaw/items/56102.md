## Summary

Keep open. Current main still feeds raw ACP stdio messages directly into `@agentclientprotocol/sdk` validation, which rejects the reported date-string `protocolVersion`, and the open closing PR has not merged.

## What This Changes

The issue asks OpenClaw's ACP bridge to accept MCP-style date-string `protocolVersion` values from newer VS Code and Cursor clients.

## Best Possible Solution

Land or replace PR #56176 with a current-main ACP initialize compatibility transform before SDK validation, with regression coverage for string and numeric protocol versions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an initialize request carrying `protocolVersion: "2025-11-25"` reaches SDK `zInitializeRequest` validation before OpenClaw can translate it, and SDK 0.21.0 requires a numeric protocol version.

## Solution Assessment

Yes. The narrow maintainable fix is an ACP ingress compatibility layer before `AgentSideConnection` validation that preserves valid numeric ACP requests unchanged.
