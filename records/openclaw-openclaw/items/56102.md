## Summary

Keep open. Current main and the latest release still pass ACP initialize frames directly into `@agentclientprotocol/sdk` 0.21.1, whose schema rejects the reported date-string `protocolVersion` before OpenClaw's initialize handler runs; the linked closing PR is still open and conflicting.

## What This Changes

The issue asks OpenClaw ACP to accept VS Code/Cursor initialize requests that send MCP date-string `protocolVersion` values such as `"2025-11-25"`.

## Best Possible Solution

Review, refresh, land, or replace https://github.com/openclaw/openclaw/pull/56176 with a current-main ACP initialize stream normalizer before SDK validation, matching the SDK integer/range contract and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: VS Code/Cursor send a string `protocolVersion`, current main passes initialize params directly into SDK validation, and SDK 0.21.1 requires an integer before OpenClaw's handler runs.

## Solution Assessment

Yes. A stream-boundary normalizer before `AgentSideConnection` is the narrow maintainable fix, but this issue should stay open until the linked PR lands or is replaced with equivalent tested behavior.
