## Summary

Keep open: this PR targets a real docs-backed heartbeat leak, but the current patch is not merge-ready because it replaces heartbeat-specific sanitation with the generic message branch and still lacks real delivery-path proof.

## What This Changes

The PR changes `buildReplyPayloads` so heartbeat final payloads run through `stripHeartbeatToken` and adds focused regression tests for OK-only, alert, media, and Bun error cases.

## Best Possible Solution

Compose OK-only suppression inside the heartbeat-aware sanitizer path, preserving legacy tool-block cleanup, heartbeat-mode `ackMaxChars`, media retention, and focused regression coverage, then add redacted real channel proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `buildReplyPayloads` maps heartbeat payloads through `sanitizeHeartbeatPayload` without token stripping, so `isHeartbeat: true` plus exact `HEARTBEAT_OK` remains deliverable at the payload layer despite the docs contract.

## Solution Assessment

No, not as written. Removing the raw heartbeat bypass is directionally right, but the fix should compose OK suppression with the heartbeat sanitizer and heartbeat-mode stripping rather than switching heartbeat payloads to generic message-mode stripping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve heartbeat ack-mode stripping:** `src/auto-reply/reply/agent-runner-payloads.ts:123`
  - body: Calling `stripHeartbeatToken` with `mode: "message"` for heartbeat payloads strips the token before heartbeat delivery can apply `ackMaxChars`. A short ack like `HEARTBEAT_OK all good` can become visible `all good` instead of being skipped, so keep heartbeat-mode stripping for heartbeat payloads and only add the exact OK-only suppression this PR needs.
  - confidence: 0.9
- **[P2] Preserve heartbeat tool-block sanitation:** `src/auto-reply/reply/agent-runner-payloads.ts:113`
  - body: Flattening all payloads through the generic branch means heartbeat payloads no longer run through `sanitizeHeartbeatPayload`, which current main uses to strip legacy `[TOOL_CALL]` and `[TOOL_RESULT]` blocks before delivery. Compose OK suppression with that sanitizer instead of replacing the heartbeat-specific path.
  - confidence: 0.85
