## Summary

Keep this PR open: the performance bug and the narrowed fix direction are valid, but the updated cache still ignores plugin metadata snapshot/workspace changes that the setup CLI backend resolver is designed to observe.

## What This Changes

The PR memoizes the setup-manifest branch of `isCliProvider` for status/doctor session-summary performance, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the speedup, but key the setup lookup cache to the same plugin control-plane metadata boundary, or precompute CLI-provider facts inside the status summary flow where the snapshot is stable.

## Reproduction Assessment

Yes, source-reproducible: current main resolves a runtime label for each session row and can re-enter setup CLI backend discovery through `isCliProvider`. The PR body also provides before/after synthetic session-store timings, but I did not rerun them in this read-only pass.

## Solution Assessment

No. The updated patch fixes the stale active-runtime-registry cache from the first review, but the setup-branch cache is still broader than the resolver contract because current plugin metadata snapshots and workspace-scoped discovery can change under the same config/provider key.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Include metadata freshness in the setup cache:** `src/agents/model-selection-cli.ts:47-49`
  - body: The setup-branch cache returns a saved value for the same `(config, provider)` without consulting `resolvePluginSetupCliBackendRuntime` again. That resolver is intentionally sensitive to current plugin metadata snapshots and workspace/env discovery; existing tests expect CLI backend results to change after `setCurrentPluginMetadataSnapshot` changes. A provider probed before a plugin inventory reload can stay cached false or true until the config object changes, so key this cache by the plugin control-plane/snapshot boundary or scope it to one status-summary run.
  - confidence: 0.88
