## Summary

Keep open: this PR fixes a real status/doctor session-summary hot path and now includes live behavior proof, but the current patch still caches setup-manifest answers more broadly than the plugin metadata/workspace freshness contract allows.

## What This Changes

The PR memoizes the setup-manifest branch of `isCliProvider` per config/provider, adds focused regression tests, and updates the changelog.

## Best Possible Solution

Keep the status/doctor speedup, but scope the cache to one stable status-summary scan or key it to the plugin control-plane metadata/workspace/env freshness boundary.

## Reproduction Assessment

Yes, source-reproducible: current main resolves every session row through `resolveSessionRuntimeLabel` and `isCliProvider`, which can re-enter setup CLI backend discovery per row. The PR body also includes before/after benchmark and live container output for the performance symptom.

## Solution Assessment

No. The setup-branch memoization preserves the speedup, but its key omits metadata/workspace/env freshness that the underlying resolver intentionally observes.
