## Summary

Keep open. Current main still has the stale installed-plugin-index recovery gap: disabled persisted records with missing plugin files can be trusted, then rehydrated during metadata/config validation and surfaced as a generic config error instead of being ignored, pruned, or diagnosed as `plugins/installs.json` state.

## What This Changes

The issue asks OpenClaw to tolerate, prune, or clearly diagnose stale disabled bundled plugin records in `plugins/installs.json` after downgrade.

## Best Possible Solution

Add generic installed-index recovery that ignores or prunes missing disabled records before manifest loading, preserves fatal validation for enabled missing plugins, and makes doctor/diagnostics name `plugins/installs.json`, the plugin id, and the missing path.

Yes, from current-main source inspection. Seed `plugins/installs.json` with a disabled record whose root or manifest no longer exists, then run a metadata/config-validation startup path; current code can still include that disabled record with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index freshness and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled missing plugins.
