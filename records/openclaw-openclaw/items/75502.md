## Summary

Keep open. Current main still allows a disabled persisted plugin-index record with missing files to survive snapshot selection, then validation can rehydrate it with disabled plugins included and turn the missing manifest into a fatal generic plugin diagnostic.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin entries in `plugins/installs.json` after downgrade and to point diagnostics at that state file.

## Best Possible Solution

Add a generic registry/doctor recovery path that ignores or prunes missing disabled install-index records before manifest loading, while diagnostics name `plugins/installs.json`, the plugin id, and manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a full generated installed-plugin index containing an `enabled:false` record whose `rootDir` or `manifestPath` no longer exists, then run a validation/status/startup path that loads the plugin registry with `includeDisabled:true`.

## Solution Assessment

Yes. The narrow maintainable solution is generic installed-index validation plus doctor cleanup and better provenance, not a `file-transfer` special case or a broad relaxation for enabled missing plugins.

## Repair Work Prompt

Fix the stale disabled installed-plugin-index downgrade recovery bug reported in https://github.com/openclaw/openclaw/issues/75502. Current main should not fail startup/status/config validation when `plugins/installs.json` contains a disabled persisted plugin record whose `rootDir`, `manifestPath`, `source`, or `setupSource` no longer exists after a package downgrade or bundled plugin removal. Implement this generically in the plugin registry / installed-index / doctor recovery path, not as a `file-transfer` special case. Preserve fail-closed behavior for enabled missing plugins and explicitly configured missing plugins. Improve diagnostics so missing manifest errors coming from the installed index identify `plugins/installs.json`, plugin id, and manifest path instead of only blaming `openclaw.json`/`plugins`. Add focused regression coverage for a disabled missing installed-index record, config validation with `includeDisabled:true`, and doctor/registry cleanup or refresh behavior. Add a user-facing changelog fix entry unless the final change is test-only. Do not broaden plugin discovery, do not load plugin runtime modules on cold validation paths, and do not edit unrelated plugin/provider behavior.
