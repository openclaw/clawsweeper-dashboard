## Summary

Keep open. Current main still has the same disabled stale plugin-index path: stale disabled persisted records are skipped by stale-source fallback, then config validation can rehydrate them with `includeDisabled: true` and surface a missing manifest as a fatal config error.

## What This Changes

The issue asks OpenClaw to tolerate or repair a disabled stale `file-transfer` entry in state-managed `plugins/installs.json` after downgrading, and to point diagnostics at that state file instead of only `openclaw.json`.

## Best Possible Solution

Fix the generic plugin-index recovery path so missing disabled persisted `plugins` cache records are ignored or pruned before config validation and doctor repair, while diagnostics identify `plugins/installs.json` as the stale metadata source.

## Reproduction Assessment

Yes. Seed the state plugin index with a disabled persisted record whose `rootDir` or `manifestPath` no longer exists, then run config validation or a startup/status path that loads plugin metadata with `includeDisabled: true`; current source shows it will attempt to load the missing manifest.

## Solution Assessment

Yes. The narrowest maintainable fix is in shared plugin registry/config-validation and doctor repair code, not a `file-transfer` special case or a startup-only downgrade migration.
