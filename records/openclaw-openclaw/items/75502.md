## Summary

Keep open. Current main still has a boot-blocking stale installed-plugin-index recovery gap for disabled records, and the issue discussion has multiple confirmations of the downgrade path and the misleading config-file diagnostic.

## What This Changes

Downgrading from 2026.4.29 to 2026.4.27 can leave a disabled `file-transfer` record in `plugins/installs.json` that blocks startup and doctor with a missing manifest error attributed to `openclaw.json`.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, and make doctor/config diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, from source inspection rather than a live command. A disabled persisted record with missing files can bypass freshness recovery, then be included with `includeDisabled: true` and produce a missing-manifest config diagnostic.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation/doctor cleanup and better diagnostics, not a `file-transfer` special case or relaxed validation for enabled plugins.
