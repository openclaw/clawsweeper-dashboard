## Summary

Keep open. Current main has a partial doctor cleanup for stale bundled install records, but the disabled persisted-plugin manifest path can still be rehydrated for metadata/config validation and reported as a generic `plugins` config failure.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin entries in `plugins/installs.json` after downgrade and to report that state file accurately.

## Best Possible Solution

Add generic installed-index recovery that skips or prunes missing disabled persisted records before manifest loading, then make diagnostics and doctor name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, at source level. Seed `plugins/installs.json` with a disabled record whose manifest/root no longer exists, then run a metadata/config-validation startup path; current main still includes that record when `includeDisabled: true` is used.

## Solution Assessment

Yes, the narrow maintainable fix is a generic installed-index validation and repair path, not a `file-transfer` special case. It should preserve strict validation for enabled plugins while making disabled stale records non-fatal and diagnosable.
