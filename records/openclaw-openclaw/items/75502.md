## Summary

Keep open. Current main still has the stale installed-plugin-index recovery gap: disabled persisted records can be accepted by the snapshot loader, then rehydrated during config validation and reported as a fatal missing manifest without pointing at `plugins/installs.json`.

## What This Changes

The issue reports that downgrading can fail startup when a disabled stale bundled plugin record remains in `plugins/installs.json`.

## Best Possible Solution

Fix the shared plugin registry, config-validation, and doctor recovery path so missing disabled records from `plugins/installs.json` are ignored or pruned before manifest loading, while diagnostics identify the state file, plugin id, and manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled record whose `rootDir` or `manifestPath` no longer exists, then run a startup/status/config-validation path that loads the registry with `includeDisabled: true`; current source shows that candidate can still reach manifest loading.

## Solution Assessment

Yes. The narrow maintainable fix is generic registry/index validation plus doctor cleanup, not a `file-transfer` special case or a broad relaxation of manifest validation.
