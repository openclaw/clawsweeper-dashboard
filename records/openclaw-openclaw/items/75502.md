## Repair Work Prompt

Fix the stale installed-plugin-index recovery bug from https://github.com/openclaw/openclaw/issues/75502. On current main, a disabled persisted record in state-managed `plugins/installs.json` whose root/manifest/source no longer exists can bypass freshness checks, then be included in metadata/config validation with `includeDisabled: true` and fail as a generic missing manifest. Implement a generic fix, not a `file-transfer` special case: ignore or prune missing disabled installed-index records before manifest loading, keep enabled missing records visible/failing as appropriate, and make doctor/config diagnostics name `plugins/installs.json`, the plugin id, and missing manifest path. Add focused regression coverage for the disabled missing-record path and doctor cleanup. Add a `CHANGELOG.md` fix entry because this changes user-facing downgrade/doctor recovery. Do not add a config option or relax validation for enabled plugins.
