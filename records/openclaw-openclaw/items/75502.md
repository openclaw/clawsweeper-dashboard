## Summary

Keep open. Current main still has the stale installed-plugin-index recovery gap: a disabled persisted record with missing files can survive freshness checks, then metadata/config validation can reload it and report a generic config error instead of repairing or naming `plugins/installs.json`.

## What This Changes

The issue requests downgrade-safe handling and clearer diagnostics for stale disabled bundled plugin records left in `~/.openclaw/plugins/installs.json`.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, and make diagnostics/doctor identify `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, from source inspection. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest no longer exists, then run a metadata/config-validation startup path; current main can still reload that disabled record with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation/doctor cleanup and better diagnostics, not a `file-transfer` special case or relaxed validation for enabled plugins.
