## Summary

Keep open. Current main still appears to allow a disabled stale installed-plugin record with a missing manifest to survive registry freshness checks, re-enter metadata/config validation, and fail with a generic plugin manifest error instead of a repairable `plugins/installs.json` diagnostic.

## What This Changes

Report requests downgrade-safe handling and diagnostics for stale disabled bundled-plugin records persisted in `plugins/installs.json`.

## Best Possible Solution

Add generic installed-index recovery that ignores or prunes missing disabled records before manifest loading, names `plugins/installs.json` with the plugin id and missing path, and lets doctor repair that state before full config validation blocks startup.

Yes. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current main still includes disabled records in that path.

## Solution Assessment
