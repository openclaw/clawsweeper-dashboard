## Summary

Keep open. Current main still appears vulnerable to the reported downgrade/startup failure because disabled persisted plugin records can bypass stale-source invalidation, then be rehydrated with `includeDisabled: true` and fail config validation with a generic missing-manifest diagnostic.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrades, and to report that state file instead of blaming authored config.

## Best Possible Solution

Add a generic installed-index recovery path that skips or prunes missing disabled records before manifest loading, and make diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest no longer exists, then exercise config validation or metadata loading; the current source shows that record can still reach manifest loading through `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable solution is generic persisted-index validation and doctor cleanup, not a `file-transfer` special case or broad suppression of manifest validation.

## Repair Work Prompt

Fix the downgrade/startup recovery bug reported in https://github.com/openclaw/openclaw/issues/75502. Current main can keep a disabled stale bundled record in `plugins/installs.json`, skip missing-source invalidation, then rehydrate it during metadata/config validation with `includeDisabled: true` and fail on a missing manifest. Implement a generic fix, not a `file-transfer` special case: missing disabled persisted records should be ignored or pruned before manifest loading, enabled missing records should still surface real errors, and diagnostics/doctor output should identify `plugins/installs.json`, the plugin id, and the missing manifest path. Keep the fix within plugin registry/index validation plus doctor/repair paths; do not weaken manifest validation for active plugins or unrelated typos. Add focused regression coverage and a user-facing changelog entry.
