## Summary

Keep open. Current main still appears to have the stale installed-plugin-index recovery gap: a disabled persisted record with a missing manifest can survive the persisted-registry freshness check, get rehydrated for metadata/config validation, and surface as a fatal generic plugin manifest error instead of a repairable `plugins/installs.json` problem.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled plugin records in `plugins/installs.json` after downgrades and to point diagnostics at the actual state file.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, while diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current source shows that record can still be loaded with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable solution is generic registry/index validation plus doctor cleanup, not a `file-transfer` special case or a broad relaxation of manifest validation.
