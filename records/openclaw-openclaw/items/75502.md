## Summary

Keep open. Current main still allows a disabled stale installed-plugin registry record to survive snapshot validation, then rehydrates it during config validation and reports the resulting missing manifest against generic plugin config rather than the state-managed plugin index.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrades and to report the real state source.

## Best Possible Solution

Fix the generic installed-plugin registry, config-validation, and doctor recovery path so missing disabled index records are ignored or pruned before manifest loading while diagnostics name `plugins/installs.json`, plugin id, and manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled record whose root or manifest path no longer exists, then run a config validation or startup/status path that loads the plugin registry with `includeDisabled: true`; current source shows that record can still reach manifest loading.

## Solution Assessment

Yes. The narrow maintainable solution is registry/index validation plus doctor cleanup and better diagnostics, not a `file-transfer` special case or a broad relaxation of enabled-plugin manifest validation.
