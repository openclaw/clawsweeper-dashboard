## Summary

Keep open: current main still has the stale installed-plugin-index recovery gap, so a disabled persisted record with missing files can survive freshness checks and then fail metadata/config validation as a generic plugin error.

## What This Changes

The issue asks OpenClaw to survive downgrade/startup when `plugins/installs.json` contains a disabled stale bundled `file-transfer` record, repair that state, and report `installs.json` as the source.

## Best Possible Solution

Add generic installed-index recovery that ignores or prunes missing disabled records before manifest loading, lets doctor/registry refresh repair `plugins/installs.json` without full config validation, and includes the state file, plugin id, and missing manifest path in diagnostics.

## Reproduction Assessment

Yes, from source inspection and reporter confirmations. Seed `plugins/installs.json` with a disabled record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current main still includes that record via `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index freshness and doctor cleanup before manifest loading, not a `file-transfer` special case or relaxed validation for enabled plugins.
