## Summary

Keep open. Current main still has the installed-plugin-index recovery gap: disabled persisted records with missing files can bypass stale-source detection, then metadata/config validation rehydrates disabled entries and reports a generic plugin config failure.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin entries in `plugins/installs.json` after a downgrade, and to report the real state file causing the missing manifest.

## Best Possible Solution

Make persisted-index recovery ignore or prune missing disabled records before metadata/config validation, and surface diagnostics that name `plugins/installs.json`, the plugin id, and missing manifest path.

## Reproduction Assessment

Yes, from source inspection. Seed `plugins/installs.json` with a disabled record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current main can still load that disabled record with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled plugins.
