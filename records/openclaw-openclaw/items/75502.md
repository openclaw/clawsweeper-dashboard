## Summary

Keep open. Current main still has the installed-plugin-index recovery gap: disabled persisted records with missing files bypass stale-source detection, but metadata/config validation can still rehydrate them and surface a generic plugin config failure.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrade, and to report the real state-file source of the missing manifest.

## Best Possible Solution

Make persisted installed-index recovery ignore or prune missing disabled bundled records before metadata validation, and emit diagnostics that name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, from source inspection. Seed `plugins/installs.json` with a disabled record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current main can still load that disabled record with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation plus doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled plugins.
