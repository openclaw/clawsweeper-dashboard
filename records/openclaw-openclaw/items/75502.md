## Summary

Keep open. Current main includes the file-transfer plugin again, but the generic downgrade recovery bug remains: a disabled persisted plugin record with missing files can still bypass stale-source checks, then get rehydrated for metadata/config validation and reported as a generic plugin config error instead of being ignored or repaired from `plugins/installs.json`.

## What This Changes

The issue asks OpenClaw to tolerate or repair disabled stale bundled-plugin records in `plugins/installs.json` after downgrades, and to report that state file accurately.

## Best Possible Solution

Add generic installed-index recovery that ignores or prunes missing disabled records before manifest loading, and make doctor/startup diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, with a source-based reproduction: seed `plugins/installs.json` with a disabled record whose root or manifest no longer exists, then run a metadata/config-validation startup path. Current main still trusts that disabled record and later includes it in manifest loading with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled missing plugins.

## Repair Work Prompt

Fix the downgrade stale installed-plugin-index recovery bug from https://github.com/openclaw/openclaw/issues/75502. Current main can trust a disabled persisted plugin record whose root/manifest/source no longer exists, then later include that disabled record in metadata/config validation with `includeDisabled: true`, causing a missing-manifest startup failure that points at config instead of `plugins/installs.json`. Implement a generic fix: stale disabled records from `plugins/installs.json` should not block startup; doctor/registry repair should be able to prune or ignore missing disabled records; diagnostics should name `plugins/installs.json`, the plugin id, and the missing manifest path. Do not special-case `file-transfer`, do not relax validation for enabled missing plugins, and do not add startup-time core migrations beyond the existing repair/registry seams unless the code already uses that seam. Add focused regression tests and a user-facing changelog entry.
