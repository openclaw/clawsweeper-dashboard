## Summary

Keep open. Current main still has the stale installed-plugin-index recovery gap: disabled persisted records are skipped by the persisted-source freshness check, then included again for metadata/config validation where a missing manifest becomes a fatal generic plugin diagnostic.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrade, and to report the state file that caused the missing-manifest lookup.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, while diagnostics and doctor repair name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled installed-plugin record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current source shows that record can still be loaded with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable solution is generic registry/index validation plus doctor cleanup, not a `file-transfer` special case or a broad relaxation of manifest validation.
