## Summary

Keep open. Current main still has a generic stale installed-plugin-index recovery gap: disabled persisted records can bypass missing-source freshness checks and then be loaded for metadata validation, producing a generic config error instead of a repairable `plugins/installs.json` diagnostic.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrades, and to point errors at that state file.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, and make doctor/diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, from source inspection. Seed `plugins/installs.json` with a disabled persisted record whose root or manifest path no longer exists, then run a metadata/config-validation startup path; current main can still carry that record into manifest loading with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled plugins.
