## Summary

Keep open. Current main still has the stale installed-plugin-index recovery gap for disabled records, and the discussion has multiple confirmations of the same downgrade blocker.

## What This Changes

The issue asks OpenClaw to survive downgrades when a disabled bundled plugin record in `plugins/installs.json` points at a missing manifest, and to repair/report the real state file.

## Best Possible Solution

Add generic installed-index recovery that ignores or prunes missing disabled records before manifest loading, and make diagnostics/doctor name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes, from source inspection. Seed `plugins/installs.json` with a disabled record whose root or manifest path no longer exists, then run metadata/config validation; current main still rehydrates that record with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled plugins.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75502. Current main can keep a disabled persisted record from `plugins/installs.json` even when its root/manifest/source is missing, then metadata/config validation reconstructs the installed-index manifest registry with disabled records included and reports a generic `plugins` config error. Implement a generic recovery path that ignores or prunes missing disabled installed-index records before manifest loading, and make `openclaw doctor --fix`/diagnostics name `plugins/installs.json`, the plugin id, and missing manifest path. Do not special-case `file-transfer`, do not relax validation for enabled plugins, and do not add a new config/env escape hatch. Add regression coverage for disabled missing records in the plugin registry/installed-index path, config validation diagnostics, and doctor cleanup. Add a single-line `CHANGELOG.md` fix entry because this is user-facing downgrade/startup recovery.
