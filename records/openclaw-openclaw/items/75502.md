## Summary

Keep open. Current `main` still has the reported recovery gap: disabled persisted plugin records with missing files are skipped by stale-source detection, but the metadata/config-validation path later rehydrates disabled records and can surface a generic `plugins` config error instead of identifying `plugins/installs.json`.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrades, and to report that state file instead of only `openclaw.json`.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, while doctor and diagnostics name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest no longer exists, then run a metadata/config-validation startup path; current source shows that record bypasses missing-source detection and is later loaded with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic installed-index validation and doctor cleanup, not a `file-transfer` special case or relaxed validation for enabled missing plugins.
