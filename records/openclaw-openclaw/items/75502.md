## Summary

Keep open. Current main still allows a disabled persisted installed-plugin record with missing files to bypass stale-source invalidation, then rehydrates it during metadata/config validation and reports a generic missing manifest error instead of repairing or identifying `plugins/installs.json`.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled plugin records in `plugins/installs.json` after downgrade and to point diagnostics at that state file.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, with diagnostics naming `plugins/installs.json`, the plugin id, and the missing path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled bundled record whose root or manifest path no longer exists, then run config metadata validation; current source shows that record can bypass stale-source invalidation and still be loaded with `includeDisabled: true`.

## Solution Assessment

Yes. The narrow maintainable fix is generic persisted-index validation plus doctor cleanup, not a `file-transfer` special case or a broad relaxation for enabled plugin manifest failures.
