## Summary

Keep open. Current main still has the stale installed-plugin-index gap: missing-path checks skip disabled persisted plugin rows, but metadata/config validation later reconstructs manifests with disabled rows included, so a disabled stale `file-transfer` entry in `plugins/installs.json` can still become a fatal missing-manifest error with generic plugin/config context.

## What This Changes

The issue asks OpenClaw to survive downgrades when `plugins/installs.json` contains a disabled stale bundled plugin record and to point diagnostics at that state file.

## Best Possible Solution

Add a generic installed-index recovery path that ignores or prunes missing disabled records before manifest loading, and make diagnostics/doctor name `plugins/installs.json`, the plugin id, and the missing manifest path.

## Reproduction Assessment

Yes. Seed `plugins/installs.json` with a disabled bundled plugin row whose root or manifest no longer exists, then run a metadata/config-validation startup path; current source shows that row can still be loaded with `includeDisabled: true`.

## Solution Assessment

Yes. The maintainable fix is generic installed-index validation plus doctor cleanup, not a `file-transfer` special case or a broad relaxation of manifest validation.
