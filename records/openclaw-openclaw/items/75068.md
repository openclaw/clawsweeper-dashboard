## What This Changes

Request to make Brave web_search in llm-context mode accept freshness and date filters, and reconsider or clarify ui_lang handling.

## Best Possible Solution

Use #51005 as the canonical fix path for Brave llm-context freshness and date filters, and track any ui_lang policy or copy change separately only if maintainers want that behavior changed.

## Reproduction Assessment

Yes. With Brave configured in llm-context mode and an API key available, the current source deterministically returns unsupported_freshness or unsupported_date_filter before any Brave fetch is attempted.

## Solution Assessment

Yes. Reviewing, rebasing, landing, or replacing #51005 is the narrowest maintainable path for the confirmed freshness/date-range bug; passing ui_lang through is not clearly better because upstream docs do not document it.
