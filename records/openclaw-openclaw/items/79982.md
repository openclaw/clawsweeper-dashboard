## Best Possible Solution

Repair or narrow the all-core contract, keep plugin group handling collision-safe, and land only after maintainers approve the new permission boundary and PDF group membership.

## Reproduction Assessment

Yes, source-level. The PR maps `group:core` to the static catalog, while current runtime conditionally registers `transcripts` outside that catalog, so `tools.allow: ["group:core"]` still filters that built-in tool out.

## Solution Assessment

No, not as-is. A dedicated all-core group is a plausible solution, but it must either cover the actual built-in runtime surface or use a narrower contract that maintainers explicitly approve.
