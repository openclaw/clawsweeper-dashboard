## Summary

Keep open. The `group:core` direction addresses a real source-level naming gap, but the PR currently mixes core-group expansion into generic plugin-tool allowlist matching, which can authorize plugin tools by core tool-name overlap; the external PR also still lacks acceptable after-fix real behavior proof.

## What This Changes

The PR adds a documented `group:core` tool-policy shorthand, revises `group:openclaw` wording, expands plugin tool allowlist group entries, and adds focused policy/plugin tests.

## Best Possible Solution

Keep the `group:core` concept, but implement it as a core-only policy expansion that cannot authorize arbitrary plugin tools; if plugin-backed built-ins need materialization, gate that through an explicit known built-in/plugin-backed set and add collision regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main has no `group:core`, and `group:openclaw` is built from `includeInOpenClawGroup`, which excludes filesystem and shell runtime tools while the docs still describe it broadly.

## Solution Assessment

No. The requested direction is useful, but expanding `group:core` through the generic plugin allowlist is not the narrowest maintainable fix because it can authorize plugin tools by name overlap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep core groups out of plugin-name allowlists:** `src/plugins/tools.ts:86-90`
  - body: Expanding every core group inside `normalizeAllowlist` turns `group:core` into plain names such as `read` and `exec`. The plugin resolver treats plain allowlist names as plugin tool names, and call sites like the standalone plugin-tools MCP server pass no full core `existingToolNames`, so `tools.allow: ["group:core"]` can expose a plugin tool that reuses a core name even though the new contract says plugin tools are excluded. Keep this expansion out of generic plugin matching, or gate it to known built-in plugin-backed tools only.
  - confidence: 0.86
