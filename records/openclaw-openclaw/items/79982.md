## Summary

## What This Changes

## Best Possible Solution

Define a single canonical core-tool catalog for `group:core`, docs, and tests, or deliberately narrow the name/docs before shipping the new public policy shorthand.

## Reproduction Assessment

## Solution Assessment

No: the overall shorthand is useful and the plugin allowlist fix is the right direction, but the current implementation does not satisfy the advertised all-core contract. The safer path is to complete the catalog contract or narrow the public semantics before merge.
