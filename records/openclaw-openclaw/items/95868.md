## What This Changes

The PR adds a lazy hosted external plugin catalog loader that fetches the ClawHub feed through the SSRF guard, validates size/checksum/schema, normalizes hosted entries, and falls back to the bundled catalog.

## Best Possible Solution

Fix live ClawHub integrity normalization, keep the loader fallback-first, and land only after maintainers accept the feed URL, hostname allowlist, schema contract, and staged rollout boundary.

## Reproduction Assessment

## Solution Assessment

No: the guarded lazy loader is a plausible transport slice, but the patch currently drops live feed candidate integrity. The better fix is to reuse the existing ClawHub SHA-256 integrity normalization or explicitly defer and document that install integrity is not carried in this slice.
