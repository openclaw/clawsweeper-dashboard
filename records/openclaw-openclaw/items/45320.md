## Summary

Keep this PR open: current main still hardcodes Discord preview splitting to block mode, but the branch is stale against the current pluginized Discord layout and has a blocking draft-stream race finding to address before merge.

## What This Changes

The PR adds a Discord preview-boundary rotation config option, wires partial-mode preview splitting to it, updates docs/schema metadata, and hardens draft-stream rotation behavior.

## Best Possible Solution

If maintainers want this feature, port it to the current Discord plugin files, choose a current-schema config location, and handle superseded send failures before updating docs and tests.

Yes, by source inspection: current main only rotates Discord preview messages when the resolved mode is `block`, while the PR’s feature asks for that behavior in `partial`. The draft-stream race finding is also source-reproducible from the generation guard plus the catch path that stops the stream.

## Solution Assessment

No, not as-is. The feature direction may be useful, but the branch should be reworked for the current extension paths and current config contract, and the stale-send failure path needs to be fixed before merge.
