## Summary

## What This Changes

The PR branch adds a Discord preview-streaming config option to split partial previews at assistant/reasoning boundaries, plus docs/schema/tests and draft-stream rotation hardening.

## Best Possible Solution

If maintainers want this behavior, port it into the current Discord plugin streaming contract, fix stale in-flight send failures on both success and failure paths, and update focused tests/docs/changelog together.

Full review comments:

- **[P2] Ignore superseded send failures after preview rotation:** `src/discord/draft-stream.ts:99-104`

Check the stale `sendGeneration` in the failure path before stopping the stream.
