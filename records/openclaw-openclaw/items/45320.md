## Summary

Keep open. The feature request is still valid on current main, but this PR is stale against the Discord plugin migration and has a draft-stream race bug that should block merge.

## What This Changes

The PR adds a Discord config option, docs/schema metadata, and tests so partial preview streaming can rotate to a fresh preview message at assistant or reasoning boundaries, plus draft-stream rotation hardening.

## Best Possible Solution

Port the feature into `extensions/discord` using the current nested `streaming.preview` conventions, keep block-mode rotation as the default, add partial-mode opt-in, fix stale-send handling, preserve mention suppression, and update docs/generated metadata/changelog/tests.

## Reproduction Assessment

Yes, from current-main source inspection. With Discord preview streaming in partial mode, partial text sets `hasStreamedMessage`, but assistant/reasoning boundaries reset local progress without rotating messages because the split gate is still block-only.

## Solution Assessment

No, not as proposed. The requested behavior is narrow and maintainable, but the patch targets removed `src/discord` paths and an old flat config key; the safer solution is a current-plugin port with the superseded-send failure path fixed.
