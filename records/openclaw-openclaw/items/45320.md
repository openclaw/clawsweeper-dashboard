## Summary

Keep this PR open. Current main still lacks an independent Discord partial-mode preview boundary rotation option, and the PR remains useful source work, but it is stale against the current `extensions/discord` plugin layout and nested `streaming.preview` config model. The proposed draft-stream hardening also has a concrete failure-path bug that should be fixed before merge.

## What This Changes

The PR adds a Discord preview boundary-rotation option, wires preview splitting away from a hardcoded block-mode check, updates docs/config metadata, and adds draft-stream tests for rotation and in-flight send handling.

## Best Possible Solution

Port the feature into the current Discord plugin as a nested preview-streaming option, with a resolver that keeps block-mode rotation as the default while allowing partial-mode opt-in. The implementation should update the current schema/types/UI hints/generated metadata/docs, add assistant-boundary and reasoning-boundary regression tests, fix superseded-send failure handling, and preserve existing mention-suppression behavior.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence reproduction path: configure Discord preview streaming in partial mode and trigger an assistant boundary after a streamed partial; the handler resets local progress state but cannot call `forceNewMessage()` because the split gate is still `discordStreamMode === "block"`.

## Solution Assessment

No, not as currently proposed. The feature direction is narrow and maintainable, but the branch needs to be ported to `extensions/discord` and the nested `streaming.preview` config surface, and the draft-stream race hardening needs the failure path fixed before merge.
