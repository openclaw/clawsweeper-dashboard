## Summary

Keep open. The feature remains useful and is not implemented on current main, but this PR is stale against the Discord plugin migration and current nested streaming config surface; the branch also leaves the superseded-send failure path unfixed before it can merge safely.

## What This Changes

The PR adds a Discord preview-streaming config option, docs/schema metadata, and tests so partial preview mode can rotate to new preview messages at assistant or reasoning boundaries while block mode keeps its existing default behavior.

## Best Possible Solution

Port the feature into `extensions/discord` as a nested preview-streaming option, with a resolver that preserves block-mode rotation by default and allows partial-mode opt-in. The port should update schema/types/UI hints/generated metadata/docs, add boundary and race regression tests, fix stale-send failure handling, preserve Discord mention suppression, and add a user-facing changelog entry crediting the source contribution.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: set Discord preview streaming to partial, stream text, then trigger an assistant or reasoning boundary; the boundary handler resets local progress state but cannot call `forceNewMessage()` because the split gate is still block-only.

## Solution Assessment

No, not as proposed. The feature direction is narrow and maintainable, but the patch targets removed `src/discord` paths and an old flat config key; the safer solution is a current-plugin port using the nested streaming config conventions plus the draft-stream race fix.
