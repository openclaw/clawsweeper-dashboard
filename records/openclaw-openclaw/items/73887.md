## Summary

Current main does not implement the ACP node replay/store surface, and the branch is not a test-only mock fix: it carries the broad gateway-owned ACP node runtime stack plus a concrete replay recovery bug and security-sensitive gateway/node command surface changes.

## What This Changes

The PR changes ACP replay tests to dynamically install routeReply/TTS mocks before importing replay/delivery modules, while the branch also adds the ACP node runtime, durable store/replay path, gateway/node command plumbing, docs, and extension test-isolation updates.

## Best Possible Solution

Review the ACP node runtime as a coherent stack with #48940, keep the dynamic-import mock ordering fix, and require the replay recovery gap plus gateway/node trust-boundary review to be resolved before landing any runtime slice.

## Reproduction Assessment

## Solution Assessment

No. The dynamic-import mock ordering is a reasonable test fix, but landing it bundled with the full ACP node runtime and an unresolved replay retry gap is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If recordProjectorPreparedSyntheticFinal() throws after resolveSyntheticFinalPayload() builds the final TTS payload, the retry creates a fresh coordinator while already-checkpointed block effects are skipped. That leaves no accumulated block text to rebuild the final payload, so final TTS can be dropped. Persist or reconstruct the payload before this write can fail, and add a regression for that failure path.
  - confidence: 0.83
