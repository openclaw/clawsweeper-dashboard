## Summary

Keep this PR open for maintainer review. Current main does not contain the ACP durable replay/node-runtime surface, and the PR is a broad production ACP gateway-node runtime expansion despite the narrow test-mock title; source inspection confirms a replay correctness bug and a security-sensitive command/event surface that should not be auto-closed or treated as a test-only cleanup.

## What This Changes

The branch adds the ACP node-backed runtime stack with durable gateway store/replay, gateway/node command plumbing, node-host worker tests and architecture docs, plus dynamic-import mock ordering in `dispatch-acp.replay.test.ts` so routeReply/TTS mocks are installed before replay/delivery modules load.

## Best Possible Solution

Review this together with #48940 as one ACP node-backed runtime stack, keep the dynamic-import mock ordering fix, require a focused synthetic-final retry fix, and have maintainers explicitly approve the new gateway-node ACP command/event trust boundary before landing.

## Reproduction Assessment

Yes for the PR-head replay defect: force `store.recordProjectorPreparedSyntheticFinal` to throw after a block delivery checkpoint, then let the projection retry and observe that the fresh coordinator has no accumulated text to rebuild final TTS. Current main cannot reproduce that path because the durable replay module is absent there.

## Solution Assessment

No. The dynamic-import mock ordering is the right test fix, but this PR is not the narrowest way to solve a replay-test mock failure because it bundles the full ACP node-runtime stack and still leaves a concrete final-TTS retry gap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If `recordProjectorPreparedSyntheticFinal()` throws after the final TTS payload is resolved, the runner retries with a fresh coordinator while the already-checkpointed block is skipped. That loses the accumulated text needed to rebuild the synthetic final, so final TTS can be silently skipped; persist or reconstruct the payload before this failure point and add a regression for the write-failure path.
