## Summary

Keep this PR open, but it is not merge-ready: current main does not contain the ACP replay/store/runtime surface that the branch targets, and the PR head still has a replay retry path that can silently drop final TTS after a prepare-write failure. The branch also expands gateway/node ACP command and worker-event surfaces despite the narrow test-mock title, so it needs maintainer review with the canonical ACP node runtime stack.

## What This Changes

The PR adds the ACP node-backed runtime stack, durable ACP store/replay and gateway-node plumbing, architecture docs, test harnesses, extension mock-isolation changes, and a dynamic-import fix for ACP replay test mocks.

## Best Possible Solution

Review this together with #48940 as one ACP node-runtime stack, keep the dynamic-import mock ordering fix, and require a focused replay fix that preserves or reconstructs the synthetic final payload when the prepare write fails before retry.

## Reproduction Assessment

Partially. Current main cannot reproduce the stated ACP replay test failure because the replay module and test do not exist there; on the PR head, the replay bug is reproducible by forcing recordProjectorPreparedSyntheticFinal to throw after block delivery has been checkpointed and observing the retry skip final TTS reconstruction.

## Solution Assessment

No. The dynamic-import mock ordering is the right Vitest direction, but bundling it with a production ACP runtime/control-surface expansion and leaving the synthetic-final retry gap is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If `recordProjectorPreparedSyntheticFinal()` throws after `resolveSyntheticFinalPayload()` builds the final TTS payload, the runner retries with a fresh coordinator while the already-checkpointed block effect is skipped. That leaves no accumulated block text to rebuild the payload, so final TTS can be silently dropped. Persist the payload before this failure point or reconstruct it from durable event text, and add a regression for this write-failure path.
  - confidence: 0.86
