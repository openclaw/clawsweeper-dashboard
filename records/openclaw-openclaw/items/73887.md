## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:426-427`
  - body: ACP worker events and terminals are sent through a helper that catches and ignores node.event request failures. A dropped terminal can make the node settle locally while the gateway never records the durable transition, leaving replay and recovery inconsistent.
  - confidence: 0.93
- **[P1] Reject run ID collisions across sessions:** `src/acp/store/store.ts:532-535`
  - body: startRun returns any existing runId as idempotent without checking sessionKey or requestId. A reused request-derived ID from another session can alias to the wrong durable run and break lease/session validation.
  - confidence: 0.92
- **[P1] Await ACP reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1034`
  - body: The node reconnect path starts ACP lease reconciliation in the background and then continues setup. A reconnecting worker can send progress before the suspect lease is reactivated, causing valid events or terminals to be rejected.
  - confidence: 0.86
- **[P2] Stop the projection runner when run creation fails:** `src/auto-reply/reply/dispatch-acp.ts:394-396`
  - body: The durable dispatch path starts wait-for-run projection before runTurn, then rethrows early failures when no durable run exists. That branch leaves the runner polling for a run that will never be created.
  - confidence: 0.89
- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If recordProjectorPreparedSyntheticFinal throws after block effects were already checkpointed, retry starts with skipped prior effects and an empty coordinator, so final TTS can be silently dropped. Persist resendable payload state earlier or reconstruct it from durable event text.
  - confidence: 0.86
