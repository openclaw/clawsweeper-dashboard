## Summary

Keep this PR open. Current main still does not contain the ACP replay test target or durable ACP node replay stack, but the PR is not merge-ready because it is a stacked ACP runtime/security-surface change with an unresolved replay correctness bug.

## What This Changes

The PR branch changes ACP replay tests to install routeReply/TTS mocks before dynamic replay/delivery imports, while also carrying the gateway-owned ACP node runtime, durable replay/store, gateway/node command plumbing, docs, and extension test-isolation changes.

## Best Possible Solution

Review the ACP node runtime as one coherent stack with #48940, keep the dynamic-import mock ordering fix, and require a targeted replay regression for the prepared-synthetic-final write failure before landing the replay slice.

## Reproduction Assessment

Partially. Current main cannot reproduce the stated replay-test failure because the replay module and test do not exist there; on the PR head, the replay bug is reproducible by forcing recordProjectorPreparedSyntheticFinal to throw after block delivery has been checkpointed, then observing the retry rebuilds no block text and skips final TTS.

## Solution Assessment

No. The dynamic-import mock fix is plausible for Vitest module capture, but bundling it with the ACP node runtime and leaving a replay retry gap is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If `recordProjectorPreparedSyntheticFinal()` throws after `resolveSyntheticFinalPayload()` builds the final TTS payload, the retry starts with a fresh coordinator while already-checkpointed block effects are skipped. That leaves no accumulated block text to reconstruct the payload, so final TTS can be silently dropped. Persist or otherwise reconstruct the payload before this write can fail, and add a regression for the prepare-write failure path.
  - confidence: 0.83
