## Summary

Keep open: current main still lacks the ACP durable replay/node-runtime surface, and the PR branch has a blocking synthetic-final retry gap plus security-sensitive gateway/node command expansion that needs maintainer review alongside #48940.

## What This Changes

The branch adds the ACP node-backed runtime and durable replay stack with docs/tests, plus dynamic-import module-isolation updates for ACP replay and extension mocks.

## Best Possible Solution

Land the ACP node-runtime work only after fixing the durable synthetic-final retry gap and approving the node ACP command/event trust boundary, preferably coordinated with #48940.

## Reproduction Assessment

Yes for the PR-head defect: force recordProjectorPreparedSyntheticFinal() to throw after a block delivery checkpoint, then retry and observe that the fresh coordinator skips the block and cannot rebuild final TTS. The title-level test-mock issue is not reproducible on current main because the durable replay module is absent there.

## Solution Assessment

No. Dynamic imports after vi.resetModules()/vi.doMock() are the right mock-ordering direction, but this branch is not safe to merge while it bundles production ACP runtime expansion and leaves the replay retry gap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If recordProjectorPreparedSyntheticFinal() throws after the final TTS payload is built, retry creates a fresh coordinator, skips the already-checkpointed block, and has no accumulated text to rebuild the payload. Persist the payload before this failure point or reconstruct it from durable event text so final TTS cannot be silently dropped.
