## Summary

Keep open: current main does not contain the ACP node-backed replay stack, and the PR head still has a durable synthetic-final retry gap plus a security-sensitive gateway/node command surface that needs maintainer review.

## What This Changes

The PR adds the ACP node-backed runtime, durable replay/projection tests and docs, plus dynamic ACP replay test imports so routeReply/TTS mocks are installed before replay modules load.

## Best Possible Solution

Land this only as part of the canonical ACP node-runtime effort after fixing synthetic-final recovery and approving the gateway/node trust boundary.

## Reproduction Assessment

Yes for the PR-head bug: force recordProjectorPreparedSyntheticFinal() to throw after a block delivery has been checkpointed, then retry and observe that the fresh coordinator skips the block and cannot rebuild final TTS. Current main cannot reproduce the PR title because the durable replay module does not exist there.

## Solution Assessment

No. Dynamic imports after vi.resetModules()/vi.doMock() are the right test-mock ordering fix, but this PR is not the narrowest safe solution while it bundles the production ACP node-runtime stack and leaves the retry gap.
