## Summary

## What This Changes

The issue reports that ACP `sessions_spawn` for the Claude backend forwards `timeoutSeconds` as an unsupported `session/set_config_option` timeout key, causing an opaque `ACP_TURN_FAILED`.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/issues/81005 as the canonical tracker and review or land https://github.com/openclaw/openclaw/pull/81006, while preserving this issue’s macOS and OpenClaw 2026.5.7 evidence by linking it there.

## Reproduction Assessment

Yes. The issue is source-reproducible: `sessions_spawn` can store `timeoutSeconds`, the ACP manager turns it into a `timeout` config option, the ACPX wrapper forwards it for non-Codex commands, and Claude ACP rejects unknown config IDs.
