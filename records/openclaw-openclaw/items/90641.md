## Summary

Keep open: the fix direction is right, but the PR still treats every non-message entry before the kept user as a hard boundary, so non-context state or metadata can still make the assistant reply disappear after rotation. The external PR also has only focused test output, not real compaction-session proof.

## What This Changes

The branch preserves summarized assistant replies before kept pre-compaction user messages, reanchors the successor compaction boundary to them, strips stale thinking signatures, and adds focused successor-transcript tests.

## Best Possible Solution

Classify boundary entries by replay semantics: skip non-context metadata/state, stop at appendable custom context or non-assistant message boundaries, and prove the fix with focused tests plus a redacted real compaction session.

## Reproduction Assessment

Yes, source-level: put a non-context entry such as `model_change`, `thinking_level_change`, `session_info`, `label`, or `custom` between the summarized assistant and retained user. The PR scan breaks before reaching the assistant even though current replay does not append those entries as conversational messages.

## Solution Assessment

No, not yet. Preserving the assistant boundary is the right layer, but the implementation must match `buildSessionContext` replay semantics instead of treating every non-message entry as a context boundary.
