## Summary

Keep open: the branch targets the right compaction bug, but PR head still treats every non-message entry between a summarized assistant and the retained user as a hard boundary while session replay ignores non-context entries. That can leave the linked disappearing-reply and batched-turn failure unfixed, and the external PR still lacks real compaction-session proof.

The PR changes compaction successor transcript rotation to preserve and reanchor a summarized assistant reply immediately before a retained pre-compaction user, with focused successor-transcript tests.

## Best Possible Solution

Classify boundary candidates by replay effect: skip non-context metadata/state entries, stop on context-producing `custom_message` or non-empty `branch_summary` entries, add regression coverage, then provide redacted real compaction-session proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. Preserving the assistant boundary is the right direction, but the implementation must follow the session replay contract instead of treating every non-message entry as a hard boundary.
