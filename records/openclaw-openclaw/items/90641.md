## Summary

Keep open: the PR targets a real compaction/session-state bug, but the current head still treats every non-message entry before a retained user as a hard boundary, so state-only entries can still make the assistant reply disappear; it also still lacks real end-to-end compaction proof.

## What This Changes

The PR preserves an assistant message immediately before the first surviving pre-compaction user message, reanchors successor compaction replay to that assistant, strips stale thinking signatures from preserved boundary assistants, and adds focused transcript-rotation tests.

## Best Possible Solution

Classify intervening entries by replay effect: skip non-context state/metadata entries, stop at appendable `custom_message`/`branch_summary` or real message boundaries, add the state-marker regression test, and supply redacted real compaction-session proof.

## Reproduction Assessment

Yes at source level. Put a state-only entry such as `model_change`, `thinking_level_change`, or `session_info` between a summarized assistant and the retained user; the PR scan stops at that marker even though replay does not append it as conversational context, so the assistant can still be removed.

## Solution Assessment

No, not yet. Preserving an assistant boundary is the right direction, but the current helper uses entry type shape instead of replay semantics; the maintainable fix should classify entries by whether they affect model-context replay.
