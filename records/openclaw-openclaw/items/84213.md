## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84213 as a narrow CLI completion bug. On current main, direct `openclaw completion -s zsh`, `openclaw completion -s powershell`, `openclaw completion -s zsh --yes`, and `openclaw completion --write-state` eagerly register core/subcommands/plugin CLI commands before output or cache writes; on native Windows the reporter says this hangs indefinitely and leaves missing cache files. Keep the public completion contract intact: explicit user completion should still generate usable zsh/PowerShell scripts and should not silently drop plugin-owned completions unless the fix introduces a clearly bounded internal fast path with tests. Prefer a clean owner-boundary fix in `src/cli/completion-cli.ts` and related completion helpers; if touching doctor/update cache repair, preserve the existing `OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS` contract and do not broaden into the general CLI cold-start issue. Add focused regression coverage for direct completion and write-state behavior, and include a changelog Fixes entry unless maintainers handle release notes separately. Validate with the focused Vitest wrapper plus native Windows/Testbox smoke showing the packaged commands exit promptly and create or print the expected completion output.
