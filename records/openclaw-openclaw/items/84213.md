Keep open. Issue reports that native Windows `openclaw completion` invocations hang instead of printing or writing shell completion cache files.

## Best Possible Solution

Make direct completion generation bounded and Windows-safe while preserving explicit plugin-owned completions, then add focused regression coverage plus a native Windows/Testbox smoke for zsh, PowerShell, and write-state commands.

## Solution Assessment
