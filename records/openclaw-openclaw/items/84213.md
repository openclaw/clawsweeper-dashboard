## Repair Work Prompt

Fix the native Windows hang reported for direct `openclaw completion -s zsh`, `openclaw completion -s powershell`, `openclaw completion --write-state`, and `openclaw completion -s zsh --yes`. Current main registers core commands, subcommands, and plugin CLI commands before writing or printing completion output; update-owned cache refreshes already use a timeout plus `OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS=1`, but direct user commands do not. Implement a bounded Windows-safe completion path that exits promptly and writes/prints the expected files, preserves nested core completion, and avoids permanently dropping plugin-owned completions without a tested fallback or warning. Add focused regression coverage around direct write-state/script generation with slow or failing registration, keep doctor/update behavior aligned, add a changelog entry if the fix is user-visible, and stop if the solution requires a new public config option or product-policy change.
