## Summary

Keep open: current main covers structured HTTP 500 and Codex `server_error` JSON, but the later reported OpenAI-branded raw message still appears unclassified, and assistant fallback only advances from classified failover signals or timeouts.

## What This Changes

The issue asks for Codex/OpenAI HTTP 500 server-side errors to trigger the configured model fallback chain instead of surfacing raw Discord errors.

## Best Possible Solution

Add a narrow classifier path for OpenAI/Codex message-only server failures, with regression coverage for the verbatim branded text and the existing structured 500/server_error paths.
