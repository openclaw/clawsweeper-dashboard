## Summary

Keep this PR open: the Langfuse root-span I/O fix is useful, but the current patch can miss the normal run lifecycle because queued `model.call.completed` events may be processed after synchronous `run.completed` cleanup/finalization.

## What This Changes

The branch mirrors captured model-call input/output onto the owning `openclaw.run` span for Langfuse trace/root-observation display and adds sibling-run regression coverage.

## Best Possible Solution

Preserve run-id isolation, but stamp or buffer model-call I/O before the root run span is ended, add a run-completed-before-async-drain regression test, then merge only after maintainers accept the broader Langfuse content visibility.

## Reproduction Assessment

Yes. Source inspection shows current main lacks root `langfuse.trace.*` I/O, and the PR body provides live Langfuse before/after proof; the new blocker is also source-reproducible from async `model.call.completed` delivery versus synchronous `run.completed` cleanup.

## Solution Assessment

No, not yet. Run-id-keyed projection is the right direction, but the implementation must handle queued model-call completions before root run span finalization; maintainers still need to accept the broader content visibility.
