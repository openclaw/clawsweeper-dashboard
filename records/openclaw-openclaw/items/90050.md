## Summary

Keep this PR open, but it is not merge-ready: the Langfuse I/O fix is useful and the single-run proof is credible, yet the patch keys root-run stamping by OTel trace id, which can attach captured prompt/response content to the wrong sibling run span.

## What This Changes

The PR changes diagnostics-otel to cache root `openclaw.run` spans and mirror model-call input/output onto Langfuse trace-level and root observation attributes.

## Best Possible Solution

Preserve the Langfuse root I/O behavior, but key root-run stamping by the model call's run span/owner and add sibling-run regression coverage before merge.

## Reproduction Assessment

Yes, at source/proof level. Current main only puts model I/O on model-call spans, the PR body shows live Langfuse before/after output for one run, and the sibling-run defect follows from existing same-trace run tests plus the PR's trace-id-only lookup.

## Solution Assessment

No. Mirroring model I/O onto the root run span is the right direction for Langfuse, but the lookup must use the specific run owner/span rather than only the OTel trace id.
