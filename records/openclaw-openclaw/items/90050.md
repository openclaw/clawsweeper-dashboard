## Summary

Keep open: this PR targets a real diagnostics-otel/Langfuse gap and includes live before/after proof, but the current branch can still miss the new root I/O stamping because synchronous run completion clears the root-run lookup before queued model-call completion drains.

## What This Changes

The PR mirrors captured model-call input/output onto diagnostics-otel root run spans for Langfuse trace/root-observation I/O and adds sibling-run isolation coverage.

## Best Possible Solution

Keep the run-id projection, but stamp root I/O before the root span is ended/untracked or defer cleanup until the diagnostic queue drains, with explicit maintainer acceptance of the broader Langfuse content visibility.

## Reproduction Assessment

Yes, source-backed. Current main only attaches captured content to model-call spans, and the PR/source path shows a normal sequence can enqueue `model.call.completed` then synchronously handle `run.completed` before the async drain.

## Solution Assessment

No, not yet. Run-id ownership is the right boundary, but deleting and ending the root span at `run.completed` before queued model-call drain is not the best fix; the safer solution applies root I/O before terminal cleanup and resolves the visibility policy.
