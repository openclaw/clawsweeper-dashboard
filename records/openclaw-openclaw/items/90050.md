## Summary

Keep open: the Langfuse root trace I/O gap is not already solved on current main or the latest release, but this PR is not merge-ready because it drops root-run tracking before queued model-call completions can stamp the root span.

## What This Changes

The PR mirrors captured diagnostics-otel model-call input/output onto the root `openclaw.run` span as Langfuse trace-level and root-observation attributes, with sibling-run isolation coverage.

## Best Possible Solution

Keep the run-id projection, defer root-run cleanup until queued model-call completions have stamped or drained, and merge only after maintainers accept the broader root trace/root observation content visibility.

## Reproduction Assessment

Yes, source-backed. `model.call.completed` is queued through the async diagnostic drain, while `run.completed` dispatches synchronously and the PR head deletes the root-run lookup before the stamping block can run.

## Solution Assessment

No. Run-id keying is the right boundary, but deleting root-run tracking during `run.completed` before queued model-call drain is not the best fix; cleanup should happen after stamping or after the queue drains.
