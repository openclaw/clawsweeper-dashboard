## Summary

Keep open. Current main has provider usage/quota plumbing, provider-owned fetch hooks, and an Overview model-auth hint, but it still lacks the requested manual Control UI decision-point fuel gauge and the requested source/confidence/staleness contract; the reporter also confirmed the linked PoC is reference material for maintainer API/UX discussion.

## What This Changes

Add a manual provider remaining-capacity fuel gauge to the Control UI before starting expensive agent work.

## Best Possible Solution

Define a maintained provider-usage metadata contract with provenance/freshness semantics, keep provider-specific fetch logic in provider hooks, and render a manual refresh capacity affordance near the Chat/model run-start controls.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence check: current usage appears in CLI/status/Overview-style surfaces, while the Chat run controls do not expose a manual provider-capacity gauge.

## Solution Assessment

Unclear. The existing provider-usage hooks and `usage.status` RPC are the right foundation, but the requested normalized metadata shape and Control UI placement need maintainer direction before this is a safe implementation task.
