## Summary

Keep open. Current main has provider usage plumbing and an Overview auth-card hint, but it still lacks the requested manual Control UI capacity gauge at the run/model decision point and lacks the requested source, confidence, freshness, and unavailable-data contract.

## What This Changes

Add a manual Control UI provider-capacity gauge that fetches provider-reported remaining capacity and reset windows before expensive agent work starts.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the model/send decision point once the Control UI slot is chosen.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives a high-confidence absence check: usage data exists in gateway/status and Overview surfaces, but no manual decision-point gauge exists in the chat toolbar.

## Solution Assessment

Unclear. The existing usage hooks and gateway methods are the right foundation, but the metadata contract and UI placement need maintainer direction before a narrow implementation is the best fix.
