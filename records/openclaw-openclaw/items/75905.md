## Summary

Keep open. Current main has provider-usage plumbing and an Overview auth-card hint, but it does not implement the requested manual decision-point fuel gauge with source/confidence/staleness metadata; the reporter also confirmed the PoC is a UX/API reference for maintainer decisions.

## What This Changes

Add a manual Control UI affordance that refreshes provider remaining-capacity and reset-window data before starting expensive agent work.

## Best Possible Solution

Extend the provider-usage contract with provenance/staleness semantics and add a manual refresh control near the chat model/run-start decision point, keeping provider-specific fetch logic behind provider-owned hooks and documenting the new surface.

## Solution Assessment
