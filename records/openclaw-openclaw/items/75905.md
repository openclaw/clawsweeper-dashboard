## Summary

Keep open. Current main has provider usage snapshots, gateway usage RPCs, provider-owned usage hooks, and an Overview auth-card hint, but it still lacks the requested manual Control UI fuel gauge at the run/model decision point and the source, confidence, stale, and unavailable metadata semantics.

## What This Changes

Add a manual Control UI affordance that refreshes and displays provider remaining capacity and reset windows before starting expensive agent work.

## Best Possible Solution

Define a normalized provider-capacity contract with source, confidence, freshness, and unavailable states, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the selected model/send decision point once the Control UI extension boundary is chosen.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence check: current main has provider usage status paths and an Overview hint, but no manual run-decision capacity gauge or requested metadata contract.

## Solution Assessment

Unclear. The existing provider-usage hooks and gateway RPC are the right foundation, but the JSON contract, provider reliability semantics, and Control UI placement need maintainer direction before a narrow implementation is clearly the best fix.
