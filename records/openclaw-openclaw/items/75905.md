## Summary

Keep open. Current main has provider usage snapshots, gateway usage RPCs, and an Overview auth-card hint, but it still lacks the requested manual Control UI fuel gauge at the model/run decision point and the proposed source, confidence, stale, and unavailable semantics.

## What This Changes

Request a manual provider remaining-capacity fuel gauge in the Control UI before expensive agent runs, backed by a normalized provider-capacity JSON shape.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific collection in provider-owned hooks, and render a manual refresh affordance near the selected model/send decision point or through a first-class Control UI plugin slot if that is the chosen product boundary.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence absence check: usage data exists in status/auth/Overview paths, but there is no manual decision-point fuel gauge in the chat/run toolbar.

## Solution Assessment

Unclear. The existing provider-usage hooks and gateway methods are the right foundation, but maintainers still need to choose the JSON metadata contract and Control UI placement before a narrow implementation can be judged best.
