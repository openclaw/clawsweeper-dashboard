## Summary

Keep open. Current main has provider usage snapshots, a gateway `usage.status` method, and an Overview auth-card usage hint, but it still does not provide the requested manual Control UI fuel gauge near the run/model decision point with source, confidence, and stale/unavailable semantics.

## What This Changes

Request a manually refreshed provider remaining-capacity gauge in Control UI before launching expensive agent work, using normalized provider-reported capacity data with source/confidence metadata.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the model/send decision point once the Control UI placement is settled.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence absence check: provider usage exists in gateway/status/Overview surfaces, but not as the requested manual decision-point gauge.

## Solution Assessment

Unclear. The existing usage hooks and `usage.status` RPC are the right foundation, but the UI placement and metadata contract need maintainer direction before a narrow implementation is safe.
