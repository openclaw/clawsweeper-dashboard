## Summary

Keep open. Current main has provider usage snapshots, a gateway usage RPC, and Overview auth-card usage hints, but it still lacks the requested manual Control UI fuel gauge at the run/model decision point plus the source, confidence, stale, and unavailable semantics.

## What This Changes

Add a manual Control UI provider remaining-capacity gauge that fetches provider-reported usage and reset data before expensive agent runs.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the model/send decision point once the Control UI slot is chosen.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence absence check: usage data exists in gateway/status and Overview surfaces, but no manual decision-point gauge exists in the chat/run toolbar.

## Solution Assessment

Unclear. The existing usage hooks and gateway methods are the right foundation, but the data contract and UI placement need maintainer direction before a narrow implementation is the best fix.
