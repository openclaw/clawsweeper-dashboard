## Summary

Keep open. Current main has provider usage snapshots, a gateway usage RPC, docs for usage/quota surfaces, and an Overview auth-card hint, but it still does not provide the requested manual Control UI fuel gauge at the run/model decision point or the requested source, confidence, stale, and unavailable semantics.

## What This Changes

Add a manual provider remaining-capacity fuel gauge to Control UI before starting expensive agent work.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the selected model/send decision point once the Control UI slot is chosen.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence check: usage data exists in gateway/status and Overview surfaces, but no manual decision-point capacity gauge exists in the chat/run toolbar.

## Solution Assessment

Unclear. The existing usage hooks and gateway methods are the right foundation, but the metadata contract, provider reliability semantics, and UI placement need maintainer direction before a narrow implementation is clearly the best fix.
