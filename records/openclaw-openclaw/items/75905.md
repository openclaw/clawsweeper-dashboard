## Summary

Keep open. Current main has provider usage snapshots, gateway/status usage RPCs, plugin usage hooks, and an Overview auth-card usage hint, but it still does not provide the requested manual Control UI fuel gauge at the run/model decision point or the requested source, confidence, stale, and unavailable semantics.

## What This Changes

The issue requests a manual Control UI provider-capacity fuel gauge that fetches remaining usage windows at the run/model decision point with source, confidence, stale, and unavailable semantics.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and render a manual refresh affordance near the selected model/send decision point once maintainers choose the UI slot and stale/unavailable semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence check: usage data exists in gateway/status and Overview surfaces, but no manual decision-point capacity gauge exists in the chat/run toolbar.

## Solution Assessment
