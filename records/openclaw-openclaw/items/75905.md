## Summary

Keep open. Current main has provider usage plumbing, provider-owned usage hooks, and an Overview auth-card hint, but it still lacks the requested manual fuel gauge at the run/model decision point with source, confidence, staleness, and unavailable-data semantics; the reporter's follow-up confirms the PoC is discussion input, not a finished implementation.

## What This Changes

Request a manual Control UI provider-capacity gauge that fetches provider-reported remaining limits and reset windows before expensive agent runs without polling or token spend.

## Best Possible Solution
