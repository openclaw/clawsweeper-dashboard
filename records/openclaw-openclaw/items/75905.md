## Summary

Keep open. Current main has provider usage plumbing and an Overview model-auth hint, but it does not implement the requested manual Control UI decision-point fuel gauge with source, confidence, and staleness metadata; the reporter also confirmed the PoC is discussion/reference material rather than a finished implementation path.

## What This Changes

Add a manually refreshed provider remaining-capacity gauge to the Control UI before expensive agent runs, using provider-reported windows plus source/confidence/staleness metadata.

## Best Possible Solution

Extend the provider-usage contract with explicit freshness/source/confidence semantics, then add a manual refresh affordance near the Chat/model/run-start decision point while keeping provider-specific fetch logic in provider-owned hooks and updating user-facing usage docs.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection of the usage schema, gateway RPC, Overview card, docs, and Chat composer gives a high-confidence absence check for the requested workflow.

## Solution Assessment

Unclear. The existing provider usage hooks and `usage.status` RPC are the right foundation, but the normalized metadata shape and Control UI placement need maintainer direction before a safe implementation PR.
