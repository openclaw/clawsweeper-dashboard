## Summary

Keep open. Current main has provider usage plumbing and an Overview auth-card hint, but it still does not implement the requested manual Control UI fuel gauge at the model/run decision point with source, confidence, and stale/unavailable metadata.

## What This Changes

The issue requests a manual Control UI fuel gauge for provider remaining capacity, reset windows, source, confidence, and stale/unavailable status before expensive runs start.

## Best Possible Solution

Define a normalized provider-capacity contract with source, confidence, and staleness semantics, then render a manual refresh affordance near the chat model/send decision point while keeping provider-specific fetching in provider-owned hooks.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence absence check: usage exists in gateway/status/Overview surfaces, but not as the requested manual decision-point gauge.

## Solution Assessment

Unclear. The existing provider-usage hooks and usage.status RPC are the right foundation, but the specific UI placement and metadata contract need maintainer direction before this is a safe automated implementation.
