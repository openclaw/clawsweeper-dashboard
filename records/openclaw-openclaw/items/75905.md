## Summary

Keep open. Current main has provider usage plumbing and an Overview auth-card hint, but it still lacks the requested manual Control UI capacity gauge at the run/model decision point with source, confidence, and stale/unavailable metadata.

## What This Changes

Request a manual provider remaining-capacity gauge in Control UI before launching expensive agent work.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider plugin hooks, and render a manual refresh affordance near the model/send decision point.

## Solution Assessment
