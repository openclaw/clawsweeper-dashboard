## Summary

Keep open: this focused replacement for https://github.com/openclaw/openclaw/pull/78621 has real behavior proof, but the latest head still lets a new bind-warning suppressor hide invalid trusted-proxy diagnostics and has a test env isolation bug.

## What This Changes

The PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` suppressors for doctor security and state-integrity warnings in managed Docker-out-of-Docker setups, with focused tests.

## Best Possible Solution

Suppress only non-critical bind diagnostics after trusted-proxy validity is established, restore the split-state test env snapshot, and document the operator-owned env knobs if maintainers accept them.

## Reproduction Assessment

Yes, source-level: current main unconditionally emits the relevant exposed-gateway bind and split-state warnings when those conditions are present, and the PR diff changes those branches. I did not run doctor or tests because this review must keep the checkout read-only.

## Solution Assessment

No, not yet. The suppressors may be acceptable for managed Docker deployments, but the bind warning path should only suppress after trusted-proxy `userHeader` and `trustedProxies` validation, and the test must preserve caller env state.
