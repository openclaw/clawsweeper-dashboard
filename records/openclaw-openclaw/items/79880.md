Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep invalid trusted-proxy exposure visible:** `src/commands/doctor-security.ts:238`
  - body: This treats raw `gateway.auth.mode === "trusted-proxy"` as enough to avoid the critical exposure warning. With `OPENCLAW_IGNORE_BIND_WARNING=true`, a LAN bind with missing `gateway.auth.trustedProxy.userHeader`, missing `gateway.trustedProxies`, or a mixed token config can produce no exposure warning even though runtime startup rejects that as a valid proxy auth boundary.
  - confidence: 0.92
- **[P2] Make the split-state suppression test self-contained:** `src/commands/doctor-state-integrity.test.ts:816-827`
  - body: The test creates `.openclaw-other` next to the temp state dir, but current `findOtherStateDirs` scans `/home` or `/Users` and `noteStateIntegrity` only adds the default state dir when it differs from `OPENCLAW_STATE_DIR`. The unsuppressed assertion can therefore fail in a clean environment or pass because of unrelated host state, and the env override should be restored rather than deleted.
  - confidence: 0.86
