## Summary

Keep open: the latest draft still has a blocking security-diagnostic regression and a smaller test environment isolation bug, even though the supplied terminal proof covers the intended managed Docker case.

## What This Changes

This draft PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` suppressors to doctor security/state-integrity checks with focused tests.

## Best Possible Solution

Suppress only non-critical bind diagnostics after trusted-proxy validity is established, preserve caller env in tests, and document the operator-owned env knobs if maintainers accept the policy.

## Reproduction Assessment

Yes, source-level: current main unconditionally emits the relevant exposed-gateway and split-state warnings when those conditions are present, and the PR diff changes those branches. I did not run doctor or tests because this review is read-only.

## Solution Assessment

No, not yet. The suppressors may be acceptable for managed Docker deployments, but the bind-warning path should suppress only after the same trusted-proxy validation required by startup/audit, and the new state-integrity test should restore the caller's env value.
