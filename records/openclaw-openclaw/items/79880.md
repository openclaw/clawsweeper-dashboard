## What This Changes

This PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` doctor opt-outs, plus focused tests for bind-warning and split-state suppression.

## Best Possible Solution

Model bind-warning suppression around a validated trusted-proxy/DooD-safe gateway state, keep critical warnings for invalid proxy config, add focused tests/docs, and attach redacted real doctor output from the managed container.
