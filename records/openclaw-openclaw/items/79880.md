## Summary

Keep open. The narrowed PR addresses a source-verifiable doctor warning-suppression use case and has sufficient terminal proof, but the current patch can suppress critical invalid trusted-proxy diagnostics and still has a focused test env-isolation bug.

## What This Changes

Adds env-variable suppressors for doctor gateway bind and split-state warnings in managed Docker-out-of-Docker deployments, with targeted tests.

## Best Possible Solution

Suppress only non-critical warnings after trusted-proxy validity is established, restore env isolation in the new test, and document the operator-owned knobs if maintainers accept them.

## Reproduction Assessment

Yes, source-level. Current main unconditionally emits the relevant exposed-gateway bind and split-state warnings when those conditions are present, and the PR diff changes those branches.

## Solution Assessment

No, not yet. The suppressors may be acceptable for managed Docker deployments, but the bind warning path should only suppress after trusted-proxy userHeader and trustedProxies validation, and the test must preserve caller env state.
