## Summary

Keep open: this is a focused replacement for the broad closed draft at https://github.com/openclaw/openclaw/pull/78621 and the terminal proof is sufficient, but the current diff still should not merge because the bind suppressor can hide invalid trusted-proxy diagnostics and one new test does not preserve caller env state.

## What This Changes

The PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` doctor suppressors for managed Docker-out-of-Docker setups, with focused tests.

## Best Possible Solution

Validate trusted-proxy completeness before suppressing bind diagnostics, preserve test env snapshots, and document the operator-owned env knobs if maintainers accept the policy.

## Reproduction Assessment

Yes, source-level: current main unconditionally emits the relevant exposed-gateway bind and split-state warnings when those conditions are present, and the PR diff changes those branches. I did not run doctor or tests because this review must keep the checkout read-only.

## Solution Assessment

No, not yet. The suppressors may be acceptable for managed Docker deployments, but the bind warning path should only suppress after trusted-proxy `userHeader` and `trustedProxies` validation, and the new test should preserve caller env state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep invalid trusted-proxy configs visible:** `src/commands/doctor-security.ts:241`
  - body: With `OPENCLAW_IGNORE_BIND_WARNING=true`, this branch treats raw `gateway.auth.mode: "trusted-proxy"` as enough to skip the critical exposed-without-auth diagnostic, then suppresses the remaining bind warning. Startup and the gateway audit still require `gateway.auth.trustedProxy.userHeader` and `gateway.trustedProxies`, so doctor should keep those critical findings visible or suppress only after the same validation passes.
  - confidence: 0.88

  - confidence: 0.76
