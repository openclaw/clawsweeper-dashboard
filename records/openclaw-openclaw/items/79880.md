## Summary

Keep this PR open for review, but it is not merge-ready: the useful Docker-out-of-Docker warning suppressors are still unique, while the current patch can hide invalid trusted-proxy exposure diagnostics and has a small test environment leak.

## What This Changes

The PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` support in doctor security/state-integrity checks, plus focused tests for suppressing managed Docker warning noise.

## Best Possible Solution

If maintainers accept the operator-owned env knobs, suppress only non-critical bind diagnostics after trusted-proxy validity is established, preserve test env state, add focused invalid-config coverage, and document the accepted suppressor contract.

Yes, source-level: current main unconditionally emits the relevant exposed-gateway bind and split-state warnings when those conditions are present, and the PR changes those branches. I did not run doctor or tests because this review is read-only.

## Solution Assessment

No, not yet. The suppressors may be the right shape for managed Docker deployments, but the bind path should only suppress after trusted-proxy validity is established and the test must preserve caller env state.
