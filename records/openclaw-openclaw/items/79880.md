## Summary

Keep open: the PR is focused and includes copied terminal proof, but the current bind-warning change can hide critical trusted-proxy misconfiguration diagnostics by treating raw `trusted-proxy` mode as authenticated before validating proxy and identity-header config.

## What This Changes

The PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` handling plus tests for Docker-out-of-Docker doctor warning suppression.

## Best Possible Solution

If maintainers accept these env knobs, keep critical trusted-proxy misconfiguration diagnostics visible and only suppress the bind warning after the same trusted-proxy validity checks used by startup and security audit pass.

## Reproduction Assessment

Yes, source-level: current main unconditionally emits the relevant bind warning for exposed authenticated gateways and the split-state warning for extra state directories. I did not run doctor or tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not yet. The suppression direction may be acceptable for managed Docker deployments, but the bind suppressor should preserve critical trusted-proxy setup diagnostics or share the validated trusted-proxy contract.
