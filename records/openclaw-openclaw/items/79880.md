## What This Changes

The PR adds `OPENCLAW_IGNORE_BIND_WARNING` and `OPENCLAW_IGNORE_SPLIT_STATE_WARNING` handling plus tests to silence selected doctor bind/state warnings for managed Docker-out-of-Docker deployments.

## Best Possible Solution

Keep the PR open, preserve critical trusted-proxy misconfiguration diagnostics, add focused trusted-proxy coverage, document the accepted env knobs if maintainers want them, and require redacted real `openclaw doctor` before/after proof.

## Reproduction Assessment

Yes, source-level: current main's `noteSecurityWarnings` and `noteStateIntegrity` unconditionally emit the relevant bind/state warnings for the described conditions. I did not run doctor or tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not yet. The suppressor direction may be acceptable for managed Docker deployments, but it should gate bind suppression on an effectively valid trusted-proxy/auth configuration and keep critical diagnostics for invalid proxy setup.
