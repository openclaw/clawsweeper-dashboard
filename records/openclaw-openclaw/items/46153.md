## Summary

Keep open. Current main still generates macOS Gateway LaunchAgent plists without a `StandardInPath` binding, and fresh install, staging, and restart-repair all flow through that same renderer, so the reported launchd-only startup hang is not implemented or superseded.

## What This Changes

Request that generated macOS Gateway LaunchAgent plists bind stdin to `/dev/null` so launchd-started gateways start and listen instead of hanging silently.

## Best Possible Solution

Add `StandardInPath=/dev/null` in the shared LaunchAgent plist renderer, cover fresh install/staging and restart-rewrite behavior with regression tests, and leave the separate external-volume log-path issue to #40207.

## Reproduction Assessment

Yes for source-backed triage: the issue gives concrete launchd steps, and current main still generates the implicated plist without `StandardInPath`. A live macOS smoke should confirm the post-fix behavior.

## Solution Assessment

Yes. The narrowest maintainable fix is in the shared LaunchAgent plist renderer because install, staging, and restart repair all reuse it; a manual plist edit or docs-only workaround would be overwritten.
