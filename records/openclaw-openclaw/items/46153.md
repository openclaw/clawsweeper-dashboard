## Summary

Keep open. Current main still generates macOS Gateway LaunchAgent plists without `StandardInPath` or an equivalent stdin-closing wrapper, and the install, staging, and restart-repair paths all flow through that same renderer. The related context supports this as the focused canonical stdin-hang report rather than something already fixed or superseded.

## What This Changes

The issue requests that macOS launchd-managed Gateway startup stop hanging silently when OpenClaw is launched as a LaunchAgent.

## Best Possible Solution

Add `StandardInPath` pointing to `/dev/null` in the shared macOS LaunchAgent plist renderer, or an equivalent central stdin-closing mechanism that applies to all generated plists. Cover fresh install/staging and restart-rewrite behavior in focused launchd tests, and add a user-facing changelog fix entry without folding in the separate external-volume log-path issue.
