## Summary

Keep open. Current main still renders macOS Gateway LaunchAgent plists without a `StandardInPath`, and fresh install, staging, and restart fallback all reuse that renderer, so the launchd-only startup hang is still a narrow source-backed bug rather than implemented or superseded.

## What This Changes

The issue asks for macOS LaunchAgent Gateway startup to avoid hanging silently under launchd by binding stdin to `/dev/null` in generated service metadata.

## Best Possible Solution

Add `StandardInPath=/dev/null` in the shared LaunchAgent plist renderer, cover install/staging and restart-rewrite paths with regression tests, and leave external-volume log-path handling to #40207.

## Reproduction Assessment

Yes for source-backed triage: the issue provides concrete launchd steps, and current main still generates the implicated LaunchAgent plist without `StandardInPath`. A live macOS launchd smoke is still needed after the patch.

## Solution Assessment

Yes. The narrowest maintainable fix is in the shared LaunchAgent plist renderer because install, staging, and restart fallback all reuse it; a manual plist edit or docs-only workaround would be overwritten.
