## Summary

Keep open: current main still has a source-level path where a configured `channels.feishu` can select the built-in `feishu` id, append it to `plugins.allow`, and then warn when no installed plugin with id `feishu` exists.

## What This Changes

The issue asks Gateway plugin auto-enable to stop adding a missing `feishu` plugin id to `plugins.allow` when `channels.feishu` is handled by an installed `openclaw-lark` plugin.

## Best Possible Solution

Make channel auto-enable respect an installed manifest owner for a configured channel before allowlisting an absent built-in channel id, while preserving `preferOver`, explicit user selection, deny, and disable semantics.

## Reproduction Assessment

Yes, by source inspection: with `channels.feishu`, a manifest registry containing `openclaw-lark` claiming `feishu`, and no installed `feishu` plugin record, current auto-enable can append `feishu` to `plugins.allow` and validation then emits the reported warning.

## Solution Assessment

No to a hardcoded `feishu` to `openclaw-lark` map; the narrow maintainable fix is a generic manifest-owner selection/suppression path that keeps the documented replacement semantics intact.
