## Summary

Keep open: current main still has a source-level path that can auto-add the built-in `feishu` channel id to `plugins.allow` and then warn when no installed plugin with id `feishu` exists. The prior ClawSweeper comment matches the current code, and the narrow fix should be generic channel-owner resolution rather than a third-party plugin special case.

## What This Changes

The issue requests that channel auto-enable stop injecting an unsatisfied `feishu` plugin id into `plugins.allow` when `channels.feishu` is served by `openclaw-lark`.

## Best Possible Solution

Use installed/trusted plugin manifest ownership for configured channels before appending a built-in channel id to `plugins.allow`, while preserving explicit user allow entries and `preferOver` replacement behavior.

## Reproduction Assessment

Yes, by source inspection. A configured `channels.feishu` can select `feishu` during auto-enable, auto-append that id to `plugins.allow`, and validation then emits the reported missing-plugin warning when only an external owner such as `openclaw-lark` is installed.

## Solution Assessment

No to a hardcoded `feishu` to `openclaw-lark` mapping or blanket warning suppression. The maintainable fix is a generic manifest-owner path that recognizes installed plugins declaring the configured channel and keeps existing `preferOver` semantics intact.
