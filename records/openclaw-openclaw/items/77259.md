## Summary

Keep open: current `main` still has a source-level path where `channels.feishu` plus an installed `openclaw-lark` manifest can select the built-in `feishu` plugin id for auto-allowlisting, and validation then warns when `feishu` is not installed.

## What This Changes

The issue requests that Gateway stop injecting the missing `feishu` plugin id into `plugins.allow` when `channels.feishu` is handled by the third-party `openclaw-lark` plugin.

## Best Possible Solution

Teach channel auto-enable to choose an effective trusted installed manifest owner for a configured channel before falling back to the built-in channel id, with a regression fixture for `openclaw-lark`-style ownership.

## Reproduction Assessment

Yes, by source inspection: a restrictive `plugins.allow`, configured `channels.feishu`, and a registry containing `openclaw-lark` with `channels: ["feishu"]` but no installed `feishu` plugin can still select and append `feishu`. I did not run a live Gateway process.

## Solution Assessment

Yes for a generic manifest-owner fix; no for a hardcoded `openclaw-lark` mapping. The narrow maintainable solution is to reuse the existing channel ownership policy or equivalent checks before allowlist materialization.

Fix the `plugins.allow` auto-injection bug for third-party channel owners.
