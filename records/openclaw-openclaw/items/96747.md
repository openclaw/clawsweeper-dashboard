## Summary

Keep open for maintainer review: the patch appears to fix a real memory chunking bug and current exact-head checks are green, but it intentionally adds public SDK surface and forces old memory/session indexes through a full rebuild after upgrade.

## What This Changes

The PR fixes same-line fragment joins in chunkMarkdown, versions chunker-derived hashes and memory index metadata, exports that version through the memory-core storage SDK subpath, updates SDK surface checks, and adds regression tests.

## Best Possible Solution

Land the chunker fix and persisted-index invalidation together after maintainer acceptance of the public SDK export and one-time reindex behavior.

## Reproduction Assessment

Yes by source inspection and contributor fail-before output: current main joins all buffered chunk fragments with "\n", and a single long line with overlap enabled can carry same-line fragments into one chunk. I did not run a local test because this review was read-only.

## Solution Assessment

Yes, mostly: chunkMarkdown is the source of corrupt text, and persisted index identity is the right upgrade boundary. A narrower chunk-only patch would leave old rows serving stale newline-injected snippets.
