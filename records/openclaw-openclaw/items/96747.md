## Summary

Keep this PR open: current main still joins all chunk fragments with a newline, while the PR fixes that at the canonical chunk assembly point and adds focused regression coverage. I found no blocking correctness, security, compatibility, or merge-risk findings.

## What This Changes

The PR changes `chunkMarkdown` to concatenate consecutive fragments from the same physical line without adding a newline, and adds a regression test for overlapping chunks from one long newline-free line.

## Best Possible Solution

Land the chunker-level fix after normal CI finishes, keeping the canonical chunk text faithful before hashing, embedding, FTS indexing, and snippet display.

## Reproduction Assessment

Yes. Current main source shows long-line fragments share `lineNo` but `flush()` joins every entry with `"\n"`, and the PR body gives a concrete `tsx` probe plus fail-before test output for the same path.

## Solution Assessment

Yes. The best fix is in `chunkMarkdown` where canonical chunk text is assembled; downstream indexing, embedding, hashing, and snippet paths correctly consume `chunk.text` and should not need compensating cleanup.
