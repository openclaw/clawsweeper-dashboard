## Summary

This is a valid Xiaomi MiMo native web-search feature request, but current OpenClaw policy and SDK surfaces route optional provider/search capability to a ClawHub or npm plugin rather than a bundled core change; the same request has already been closed on that basis in related threads.

## What This Changes

Add Xiaomi MiMo's native chat-completions web_search tool as an OpenClaw web-search provider with optional Xiaomi plugin configuration.

## Best Possible Solution

Publish a Xiaomi MiMo web-search provider through ClawHub or npm using the existing web-search provider SDK, and open a narrower plugin-API issue only if that implementation exposes a missing seam.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature request, and the source check is high confidence that current main and v2026.5.27 do not ship a Xiaomi web-search provider or webSearch config surface.

## Solution Assessment

Yes, closing the core request in favor of a ClawHub or npm plugin is the best current path. The existing provider plugin SDK can register web search and provider-owned request behavior without adding a one-off bundled Xiaomi path to core.
