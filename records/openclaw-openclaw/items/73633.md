## Summary

This PR should stay open: it is a useful narrow piece of the WebChat code-block copy fix, but current `main` still has the wrong clipboard path and the patch does not address the still-open message-action overlay root cause for the linked bug.

## What This Changes

The PR adds a Control UI helper that copies fenced-code text from the rendered `<pre><code>` element with a `data-code` fallback, wires WebChat’s code-block click handler to use it, and adds focused unit coverage.

## Best Possible Solution

Land one canonical WebChat clipboard fix that combines rendered-code copy text, the bubble-action overlap guard or equivalent layout protection, and focused regression coverage.

## Reproduction Assessment

Yes for source reproduction: the linked bug gives concrete Control UI steps, and current `main` still has the `dataset.code` copy path plus an unactivated `has-copy` overlap guard. I did not run the browser flow in this read-only review.

## Solution Assessment

No. Reading rendered `textContent` is a good part of the fix, but the narrow maintainable solution also needs to prevent message-level overlay interception or coordinate with the related open layout PR.
