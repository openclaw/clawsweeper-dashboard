## Summary

Keep this PR open. The rendered `textContent` resolver is useful, but the branch still does not fully address the linked Control UI bug because clicks can still land on the overlapping message-level copy action before the code-block handler runs; the user-facing fix also still needs the required changelog entry.

## What This Changes

The PR adds a Control UI code-block copy-text helper and unit tests, then updates the chat click handler to copy rendered `<pre><code>` text with a `data-code` fallback and stop event propagation after a code-block button match.

## Best Possible Solution

Land one canonical Control UI fix that keeps the rendered-text copy helper, restores or replaces the `has-copy` layout guard so message-level actions cannot intercept code-block copy clicks, adds regression coverage for both paths, and includes the required single-line changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Control UI steps, and current main shows a source-level reproduction path: the message action bar is absolutely positioned while `has-copy` padding is not applied to the bubble that renders those actions.

## Solution Assessment

No. Reading rendered `textContent` is a good part of the fix, but the narrow maintainable solution also needs the layout or event guard from the related root-cause work so clicks cannot be handled by the message-level markdown copy button.
