## Summary

Keep this PR open. The rendered `textContent` copy path is useful, but the diff does not fully fix the linked WebChat copy bug because the message-level copy overlay can still intercept clicks, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds a Control UI helper that copies WebChat fenced-code text from the rendered `<pre><code>` element with `data-code` as fallback, wires the code-block click handler to use it, and adds focused helper tests.

## Best Possible Solution

Land one canonical WebChat fix that combines rendered-code copy text, the existing bubble-action overlap guard or an equivalent layout fix, regression coverage, and a single-line changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Control UI steps, and source inspection shows the current overlay path: `.chat-bubble-actions` can sit over the code-block copy area because the existing `.has-copy` padding guard is not applied.

## Solution Assessment

No. Reading rendered `textContent` is a good part of the fix, but the narrow maintainable solution also needs to prevent message-level copy overlay interception and record the user-facing change in the changelog.
