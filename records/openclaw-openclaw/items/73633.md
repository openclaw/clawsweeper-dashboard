## Best Possible Solution

Land one canonical Control UI fix that keeps the rendered-text copy helper, restores or replaces the `has-copy` layout guard so message-level actions cannot intercept code-block copy clicks, adds regression coverage for both paths, and includes the required single-line changelog entry.

Yes. The linked bug gives concrete Control UI steps, and current main shows a source-level reproduction path: the floating message action bar can be rendered without the existing `has-copy` padding guard that keeps it away from code-block controls.

## Solution Assessment
