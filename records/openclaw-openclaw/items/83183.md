## Summary

Keep open as a product/security feature request: current main and v2026.6.6 have memory recall tracking, thresholded promotion into MEMORY.md, and manual AGENTS.md lesson guidance, but they do not implement task-type correction accommodation or an automatic instruction update loop.

## What This Changes

The issue requests task-type-level tracking of repeated corrections and a structured update to AGENTS.md or a related instruction surface after the same correction pattern recurs.

## Best Possible Solution

Design an explicit, reviewable accommodation workflow in the memory/instruction boundary that classifies repeated corrections, proposes or applies instruction updates only with clear authorization, and documents when the changed guidance takes effect.
