## Summary

Keep open as a product feature request. Current main has memory recall/promotion and the adjacent Skill Workshop plugin for procedural correction capture, but it does not implement the requested second-correction task-type accommodation loop or an automatic AGENTS.md update trigger.

## What This Changes

The issue asks OpenClaw to detect repeated corrections of the same task type and trigger a durable behavior update after the pattern repeats.

## Best Possible Solution

Define a reviewable procedural-accommodation workflow, likely through Skill Workshop or a closely related plugin surface, with explicit approval before durable instruction or skill updates and tests/docs for fresh and existing workspaces.

No. The issue describes a production pattern but does not provide a replayable transcript, model/channel, memory files, or exact correction wording; source inspection only confirms the requested automatic task-type accommodation loop is not present.

## Solution Assessment
