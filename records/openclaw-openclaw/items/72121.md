## Summary

Keep open. Current main still validates Anthropic `paste-token` input exclusively as a setup-token and would persist only token credentials, while the existing Anthropic API-key path is separate and the related fix PR is closed unmerged.

## What This Changes

The issue requests that Anthropic `models auth paste-token` handle standard `sk-ant-api03-` API keys instead of rejecting them as non-setup tokens, or direct users to the existing API-key auth path.

## Best Possible Solution

Detect Anthropic API-key input in the paste-token flow and route it through existing API-key credential/config handling, while preserving setup-token behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: Anthropic `paste-token` calls the setup-token validator, which rejects `sk-ant-api03-` before any credential write; I did not run the interactive CLI in this read-only review.

## Solution Assessment

Yes for keeping this open and queueing a narrow repair. The closed validator-only PR was not the best fix because the command would still persist API keys as token credentials instead of using the existing API-key auth semantics.

## Repair Work Prompt

Fix the Anthropic `models auth paste-token` handling for standard `sk-ant-api03-` API keys. On current main, `src/commands/models/auth.ts` validates all Anthropic pasted values with `validateAnthropicSetupToken`, then stores them as `credential.type: "token"`; do not solve this by only widening `validateAnthropicSetupToken`. Add narrow handling that detects Anthropic API-key input and persists it through the existing API-key credential/config semantics (`type: "api_key"`, config mode `api_key`, preserving supplied `--profile-id` and `--agent` behavior). Preserve existing `sk-ant-oat01-` setup-token behavior and logs. Add regression coverage for API-key paste, setup-token paste, whitespace normalization if touched, and the requested agent/profile path. Update CLI/provider docs only if user-facing behavior changes, and add a single-line `CHANGELOG.md` fix entry. Do not change provider IDs, introduce a new config option, or move Anthropic-owned auth policy into unrelated core surfaces.
