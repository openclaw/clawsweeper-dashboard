## What This Changes

The issue reports that loading OpenClaw’s undici copy via `createRequire("undici")` overwrites Node’s global dispatcher and breaks channel plugins that use native `globalThis.fetch`.

## Best Possible Solution

Add a focused core network-runtime fix and regression proof that loading OpenClaw’s undici helpers cannot break native `globalThis.fetch` for channel plugins, while preserving existing explicit dispatcher and env-proxy behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction yet. Source and upstream dependency inspection confirm the side-effecting undici load path, but a Node 22.22.3 smoke test did not reproduce the final native-fetch `TypeError`.

## Solution Assessment

Unclear. The suggested fixes are directionally plausible, but the maintainable fix must isolate or avoid the root undici load side effect without breaking OpenClaw’s existing dispatcher, proxy, and runtime-fetch contracts.
