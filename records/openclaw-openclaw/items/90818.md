## Summary

Keep open: current main and the latest release still lack register-time agent identity on the public plugin API, while hook callbacks already expose agent context. This is a valid core Plugin SDK design request rather than a ClawHub/plugin-only item.

## What This Changes

The issue asks for `OpenClawPluginApi` passed to plugin `register()` to include the current `agentId` and ideally `agentName` for per-agent diagnostics and behavior.

## Best Possible Solution

Define a narrow additive SDK registration-context contract with explicit semantics for non-agent-specific registration modes, then update the API type, builder/registry call sites, docs, and contract tests together.

## Reproduction Assessment

Yes, source-reproducible: current `OpenClawPluginApi` and `buildPluginApi` lack register-time agent identity while hook contexts already include `ctx.agentId`. I did not run `openclaw doctor` because this was a read-only review.

## Solution Assessment

Unclear; exposing registration context may be the right direction, but the best shape needs maintainer API design for modes that are not clearly agent-specific.
