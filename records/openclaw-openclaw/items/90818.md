## Summary

Keep open: current main and the latest release still do not expose register-time `agentId` or `agentName` on `OpenClawPluginApi`, and the remaining work is a public Plugin SDK contract decision rather than a stale cleanup candidate.

## What This Changes

The issue requests adding current agent identity, ideally `agentId` and `agentName`, to the plugin `register(api)` surface for per-agent initialization diagnostics and behavior.

## Best Possible Solution

Choose a stable registration-context contract for agent identity, then update `OpenClawPluginApi`, `buildPluginApi`, loader/registry call sites, public docs, API baseline/export checks, and focused registration tests together.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.9` shows `OpenClawPluginApi` and `buildPluginApi` lack register-time `agentId` or `agentName`, while hook contexts already expose `ctx.agentId`.

## Solution Assessment

Unclear. Exposing identity on `OpenClawPluginApi` is plausible, but because this is public SDK shape, maintainers should first choose the permanent contract and naming semantics.
