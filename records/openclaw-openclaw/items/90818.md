## Summary

Keep open: current main and v2026.6.6 still do not expose register-time `agentId` or `agentName` on `OpenClawPluginApi`, while hook-time context already has `ctx.agentId`. This is a real public Plugin SDK design request, not a ClawHub/plugin-content close.

## What This Changes

The issue requests adding current agent identity to the Plugin SDK API object passed into plugin `register(api)` functions.

## Best Possible Solution

Define an additive, documented Plugin SDK registration context that exposes agent identity only where the host can guarantee it, with tests covering full runtime and non-agent registration modes.

## Reproduction Assessment

Yes for a source-level API gap: current `OpenClawPluginApi` and `buildPluginApi` lack `agentId`/`agentName`, while hook docs and types already expose `ctx.agentId` later in the lifecycle. I did not run `openclaw doctor` because this read-only review can prove the missing SDK field directly from source.

## Solution Assessment

Unclear; adding agent identity is plausible, but the best fix needs maintainer API-shape judgment around full runtime versus discovery/setup/CLI metadata registration modes.
