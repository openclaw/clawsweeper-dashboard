## Summary

Keep open: current main and the latest release still expose hook-time `agentId`, but not register-time `agentId` or `agentName` on `OpenClawPluginApi`. This is a valid core plugin SDK/API design request, not a ClawHub move, because the missing surface is the host-provided `register(api)` contract.

## What This Changes

The issue requests register-time agent identity fields on the plugin SDK API so plugins can distinguish per-agent initialization.

## Best Possible Solution

Define a narrow additive SDK registration-context contract, document which registration modes have agent identity, and cover `OpenClawPluginApi`, `buildPluginApi`, plugin loading, SDK docs, and contract tests in one focused change.

## Reproduction Assessment

Yes. Source inspection of current main reproduces the API gap: `OpenClawPluginApi`, `buildPluginApi`, and registry `createApi` do not expose or populate register-time agent identity, while hook context already has optional `agentId`.

## Solution Assessment

Unclear. Exposing current agent identity during `register(api)` is a plausible direction, but maintainers need to choose the public SDK shape and registration-mode semantics before implementation.
