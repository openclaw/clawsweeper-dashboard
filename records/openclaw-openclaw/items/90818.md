## Summary

Keep open: current main and v2026.6.1 still expose plugin identity plus hook-time `ctx.agentId`, but not register-time agent identity. This is a valid plugin SDK/API design request, not a ClawHub-closeable plugin request, because the missing surface is in OpenClaw core.

## What This Changes

The issue requests adding current agent identity, such as `agentId` and possibly `agentName`, to the `OpenClawPluginApi` object passed to plugin `register(api)`.

## Best Possible Solution

Define a narrow additive Plugin SDK contract for per-agent runtime registration context, explicitly documenting setup/CLI-metadata behavior and updating SDK tests/docs with the chosen `agentId` and display-name semantics.

## Reproduction Assessment

Yes for the API gap from source inspection: `OpenClawPluginApi` and `buildPluginApi` lack `agentId` or `agentName`, while hook callbacks already have `ctx.agentId`. I did not run `openclaw doctor`, but the requested missing SDK field is source-proven on current main.

## Solution Assessment

Unclear; exposing current agent identity may be the right direction, but the best shape needs maintainer API design because current plugin loading is workspace/runtime scoped and setup or CLI metadata registration is not clearly agent-specific.
