## Summary

Keep open: current main and the latest release still do not expose register-time agent identity on `OpenClawPluginApi`. This is a valid additive plugin SDK design request, not a ClawHub/core-cleanup close candidate, because the missing capability is in the public core plugin API.

## What This Changes

Request to add current agent identity, ideally `agentId` and `agentName`, to the plugin SDK API object passed into plugin `register()`.

## Best Possible Solution

Define a narrow additive registration-context contract for full agent-scoped plugin loads, document absent/non-agent registration semantics, and update SDK baseline, docs, and focused loader/runtime tests together.

## Reproduction Assessment

Yes. The API gap is source-reproducible: `OpenClawPluginApi` and `buildPluginApi` still lack `agentId`/`agentName`, while hook callbacks already expose `ctx.agentId`.

## Solution Assessment

Unclear. Exposing register-time identity is plausible, but the best shape needs maintainer API design around registration modes and whether identity should be flat fields or a grouped optional context.
