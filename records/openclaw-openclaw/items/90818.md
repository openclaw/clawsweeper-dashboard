## Summary

Keep open: current main and the latest release still do not expose register-time agent identity on `OpenClawPluginApi`; the remaining work is a public Plugin SDK design decision, not stale cleanup.

## What This Changes

The issue requests adding current agent identity to the plugin API object passed into plugin register functions.

## Best Possible Solution

Define an additive, documented Plugin SDK registration context with explicit absence semantics for non-agent-scoped modes, plus focused contract tests and docs.

## Reproduction Assessment

Yes. Source inspection shows the current SDK type and API builder lack register-time `agentId`/`agentName`, while hook contexts already expose `ctx.agentId`.

## Solution Assessment

Unclear. Exposing agent identity may be the right direction, but the best SDK shape needs maintainer design because not every registration mode is agent-specific.
