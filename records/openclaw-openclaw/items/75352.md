## Summary

Keep open. Current main still starts bundled LSP stdio servers with a raw `spawn(config.command, ...)`, and the linked closing PR #75343 is open and unmerged, so this issue should remain open until that fix or an equivalent lands.

## What This Changes

The issue requests Windows `.cmd` shim resolution for bundled LSP server startup so npm-installed language servers can launch on Windows.

## Best Possible Solution

Review and land PR #75343 or a narrow equivalent that reuses the shared Windows spawn resolver in the bundled LSP path, with focused Windows shim coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The report gives a concrete Windows 11, Node 22.14.0, and `typescript-language-server` setup, and current main statically routes that command to raw `child_process.spawn` without `.cmd` wrapper resolution.

## Solution Assessment

Yes as a direction. Reusing the existing shared Windows spawn resolver is the narrow maintainable fix; shell fallback should be limited to explicit, reviewed cases.
