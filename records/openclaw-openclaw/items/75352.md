## Summary

Keep open. Current main still launches bundled LSP stdio servers with a raw `spawn()` call, and the linked closing PR #75343 is open and unmerged, so this issue should remain open until that fix or a narrow equivalent lands.

## What This Changes

The issue asks OpenClaw to resolve Windows `.cmd` npm shims before spawning bundled LSP server commands such as `typescript-language-server`.

## Best Possible Solution

Review and land PR #75343 or a narrow equivalent that reuses the shared Windows spawn resolver in the LSP spawn path with focused regression coverage and the required changelog entry.

## Reproduction Assessment

Yes. The issue gives a concrete Windows 11, Node 22.14.0, and `typescript-language-server` setup, and current main statically reaches the raw `spawn()` path that matches the reported `.cmd` shim failure.

## Solution Assessment

Yes as a direction. Reusing the existing Windows spawn resolver is the narrow maintainable fix; the PR review should still validate the exact shell-fallback policy and test coverage before merge.
