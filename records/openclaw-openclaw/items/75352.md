## What This Changes

The issue asks for bundled LSP server startup on Windows to resolve npm `.cmd` shims instead of directly spawning the bare configured command.

## Best Possible Solution

Land PR #75343 or a narrow equivalent that resolves bundled LSP commands through the shared Windows spawn resolver using the final child environment, adds focused Windows shim coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. The report gives a concrete Windows 11, Node 22.14.0, and `typescript-language-server` setup, and current main still reaches a direct `spawn(config.command, ...)` for bundled LSP servers.

## Solution Assessment

Yes as a direction. Reusing the existing shared Windows spawn resolver is the narrow maintainable fix; broad `shell: true` execution should remain limited to the helper’s explicit fallback policy.
