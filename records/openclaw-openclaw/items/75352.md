## Summary

Keep open. Current main still launches bundled LSP stdio servers with a raw child-process spawn, and the linked closing PR is open and unmerged, so this issue should remain open until that fix path lands.

## What This Changes

Make bundled LSP server startup on Windows resolve npm `.cmd` shims instead of passing bare commands directly to `child_process.spawn()`.

## Best Possible Solution

Review and land PR #75343, or a narrow equivalent, so the LSP spawn path reuses the shared Windows spawn resolver with focused coverage and the required changelog entry.

## Reproduction Assessment

Yes. The report gives Windows 11, Node 22.14.0, and `typescript-language-server`, and current main statically reaches the raw spawn path that matches the reported `.cmd` shim failure.

## Solution Assessment

Yes as a direction. Reusing `resolveWindowsSpawnProgram` in the LSP spawn path is the narrow maintainable fix; this issue should stay open until PR #75343 or an equivalent change merges.
