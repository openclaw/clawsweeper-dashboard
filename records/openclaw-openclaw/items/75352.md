## Summary

Keep open. Current main still starts bundled LSP stdio servers with a raw child-process spawn, and the linked closing PR is open and unmerged, so this issue should stay open until that fix path lands.

## What This Changes

The issue reports that Windows LSP plugin servers fail because the LSP runtime does not resolve npm .cmd shims before spawning server commands.

## Best Possible Solution

Review and land PR #75343 or a narrow equivalent that reuses the shared Windows spawn resolver in the LSP spawn path, with focused coverage and the required changelog entry.

## Reproduction Assessment

Yes. The issue gives a concrete Windows 11, Node 22.14.0, and `typescript-language-server` setup, and current main statically reaches the raw spawn path that matches the reported .cmd shim failure.

## Solution Assessment

Yes as a direction. Reusing the existing Windows resolver is the narrow maintainable fix; the PR review should scrutinize the exact shell fallback policy and test coverage before merge.
