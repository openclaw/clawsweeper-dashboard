## Summary

Keep open. Current main still lacks `iMessageLite` handling and this PR is useful, but the latest diff still misses documented prefixed `chat_guid:` target and allowlist paths plus the required changelog entry.

## What This Changes

The PR adds a BlueBubbles chatGuid service normalizer for `iMessageLite`, applies it during webhook extraction and bare raw GUID parsing, and adds monitor-normalize tests.

## Best Possible Solution

Keep the fix BlueBubbles-local and canonicalize the chatGuid service component anywhere BlueBubbles parses, sends, compares, or formats chat GUIDs.

## Reproduction Assessment

Yes. Static source inspection gives a high-confidence reproduction: `chat_guid:iMessageLite;-;+...` is returned by the prefix parser unchanged, then `resolveChatGuidForTarget` returns it directly; the allowlist mismatch follows from exact `chat_guid` equality after inbound normalization.

## Solution Assessment

No. The PR direction is narrow and appropriate, and the follow-up commit resolved Greptile's duplicate-normalization concern, but the complete fix should also normalize prefixed target parsing and allowlist comparison with the same helper, then add changelog coverage.
