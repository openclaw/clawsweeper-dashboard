## Summary

## What This Changes

The PR changes the Gateway config reloader and skills refresh watcher so `awaitWriteFinish` is only passed to chokidar when `process.env.VITEST` enables polling.

## Best Possible Solution

Remove unnecessary idle watcher overhead with a settled-write replacement for config and skills events, then include redacted macOS before/after Gateway CPU or stat proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection confirms the two current watcher options and the reporter supplied macOS sampling evidence, but the claimed idle CPU drop still needs real before/after proof.

## Solution Assessment

No: the direction is plausible, but dropping `awaitWriteFinish` without targeted write settling is not the safest maintainable fix. The existing memory watcher fix shows a better pattern: remove broad write polling while settling only the changed paths before acting.
