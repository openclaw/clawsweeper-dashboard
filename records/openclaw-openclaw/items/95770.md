## Summary

Keep open: this maintainer PR is useful observability work, but the current patch adds prompt/tool JSON serialization on the model-call hot path before maintainers have accepted or bounded that runtime cost.

## What This Changes

The PR adds safe model-call prompt size metadata and per-call token usage fields to diagnostic events, OTel span attributes, docs, and focused tests.

## Best Possible Solution

Keep the added telemetry shape, but compute prompt component sizes only where the diagnostics consumer needs them or reuse already-built payload/prompt measurements so normal model-call latency is not penalized unnecessarily.

## Reproduction Assessment

Yes for the review concern: source inspection shows the new prompt-stat calculation runs inside the wrapper used by normal embedded and compaction model calls. I did not run a benchmark, so the exact latency impact is not measured.

## Solution Assessment

No: adding the telemetry fields is a good direction, but the current implementation should not add duplicate prompt/tool serialization to the hot path unless that cost is deliberately accepted. A safer solution gates or reuses measurement work.
