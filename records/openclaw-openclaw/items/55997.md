## What This Changes

## Best Possible Solution

## Solution Assessment

No; the current-main session-layer fix is the better solution because it authorizes resets at the point where all configured reset triggers are evaluated. A stricter request rejection could still be added, but it should reuse that boundary rather than adding a narrower regex gate.
