Full review comments:

- **[P2] Reserve a slot for the current active transcript:** `src/gateway/session-history-family.ts:108-123`
  - body: When ancestors have already filled the first 30 family targets, the current session's reset archives are pushed before the active transcript. Two current reset archives can fill the 32-target cap, making the later `activeFile` push a no-op and causing `chat.history(includeFamily)` to omit the active post-reset transcript. Reserve the active current target before current reset archives, or otherwise guarantee it cannot be displaced by the cap.
  - confidence: 0.9
- **[P2] Wire embedded sessions_history to the family resolver:** `src/agents/tools/sessions-history-tool.ts:260-263`
  - body: `sessions_history` now exposes `includeFamily` and `resetAncestors`, and this line forwards `includeFamily` to `chat.history`, but embedded runs use `createEmbeddedCallGateway`, whose `chat.history` implementation ignores that parameter and reads only the current session. An embedded model can ask for reset ancestors and still miss the family history; either wire the shared resolver into the embedded stub with coverage or avoid exposing the option there.
  - confidence: 0.91
