## Summary

Keep open: current main and v2026.6.8 still suppress Discord `isReasoning` payloads even though the shared agent path emits them for `/reasoning on` and docs define a visible separate Thinking message.

## What This Changes

The issue requests that Discord show reasoning-model `type:"thinking"` content as persistent Thinking messages when `/reasoning on` is enabled.

## Best Possible Solution

Make Discord honor the existing `/reasoning on` contract by rendering explicit reasoning payloads as separate Thinking messages while preserving hidden-reasoning suppression for disabled or unsupported modes.

## Reproduction Assessment

Yes, source-level: `/reasoning on` can emit `isReasoning` block replies, and current Discord delivery immediately suppresses those payloads. I did not run a live Discord provider session in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Discord delivery/formatting repair because provider extraction and shared agent reasoning emission already produce the right payload. A new config option or DeepSeek-specific change would be the wrong layer.
