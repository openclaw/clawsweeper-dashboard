## What This Changes

This PR snapshots abort state in the `chat.send` post-dispatch rejection catch path, preserves abort-first terminal dedupe, clears late abort tombstones when the error wins, and adds gateway race regressions.

## Best Possible Solution

Land the narrow gateway fix after adding real Gateway/WebSocket behavior proof that shows abort-first preserves terminal timeout without a late error and error-first retries are not blocked by a stale abort marker.
