## Summary

Keep open. Current main and latest release still promote failed tool cards to visible `Tool error` UI without a turn-success guard, and the direct closing PR remains open and unmerged.

## What This Changes

The issue requests that Control UI/WebChat stop surfacing failed internal tool calls as prominent user-visible error banners when the turn has a successful final assistant reply.

## Best Possible Solution

Review and land the direct Control UI projection fix, or replace it with an equivalent render-boundary change that suppresses only non-terminal internal tool failures.

## Reproduction Assessment

Yes. Current main maps failed tool cards directly to `Tool error` UI without checking whether the same turn later produced a normal assistant reply; I did not run a fresh live browser reproduction in this read-only pass.

## Solution Assessment

Yes. The Control UI chat projection/render boundary has the needed turn-completion context, while changing Codex exec failure semantics or outbound direct-chat dispatch would be broader or the wrong surface.
