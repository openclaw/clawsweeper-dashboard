## Summary

Keep open. Current main and the latest release still render failed tool cards as visible `Tool error` UI without a turn-success guard, and the direct closing PR is open but unmerged.

## What This Changes

Control UI should avoid showing failed internal tool calls as red chat error banners when the same turn later produced a normal assistant reply.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90122 if maintainers accept its UI projection-boundary fix, or replace it with an equivalent change that collapses non-terminal internal tool failures while preserving terminal failure visibility.

## Reproduction Assessment

Yes, source reproduction is high confidence: current main maps failed tool cards directly to `Tool error` UI without checking whether the turn later produced a normal assistant reply. I did not run a fresh live browser reproduction in this read-only pass.

## Solution Assessment

Yes. The Control UI projection/render boundary is the narrow maintainable place to decide whether a failed internal tool is terminal to the visible turn; changing Codex exec semantics or direct-chat dispatch would be broader or the wrong surface.
