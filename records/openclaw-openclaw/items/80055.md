## Summary

Keep open. The draft adds a useful doctor health direction and has sufficient terminal proof for the basic lint output, but the current PR head still violates its read-only lint contract and accepts lint-only selectors in repair modes where they are ignored; the maintainer-labeled doctor/SDK surface also needs explicit maintainer decision.

## What This Changes

This draft adds a structured doctor health-check contract, `openclaw doctor --lint`, structured repair validation, initial migrated core checks, SDK health exports, docs, and focused tests.

## Best Possible Solution

Land this only after maintainer approval, with lint using a no-observe snapshot path and selector flags either rejected outside `--lint` or intentionally wired into scoped repairs.

## Reproduction Assessment

Yes. The no-write blocker is source-reproducible from PR head calling `readConfigFileSnapshot()` and current config IO observing/writing health state during normal snapshot reads; the ignored selector issue follows from the CLI action only forwarding `--skip`/`--only` when `--lint` is true.

## Solution Assessment

No as-is. The direction is coherent, but the implementation must preserve the documented read-only lint contract, validate or wire selector flags consistently, and wait for maintainer approval on the doctor/SDK health surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep doctor lint from writing config observation state:** `src/commands/doctor-lint.ts:44`
  - body: `doctor --lint` is documented as read-only, but this path still calls the normal `readConfigFileSnapshot()`. Current config IO finalizes that read through observation and can persist config health or audit state, so CI/preflight lint can mutate `OPENCLAW_HOME` before reporting findings. Use a no-observe snapshot path or suppress observation for lint, and cover the invariant in tests.
  - confidence: 0.9
- **[P2] Reject lint-only filters outside lint mode:** `src/cli/program/register.maintenance.ts:32`
  - body: Registering `--only` and `--skip` on the doctor command makes Commander accept them for every doctor mode, but the non-lint branch drops those options and runs the full repair doctor. A user running `doctor --fix --only ...` now gets broad mutation instead of a parse error or scoped repair. Reject these flags unless `--lint` is set, or deliberately pass them into a scoped repair implementation.
  - confidence: 0.84
