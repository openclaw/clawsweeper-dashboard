## Summary

Keep open: current main does not have this doctor lint/health-check surface, and the PR is an active draft with a protected maintainer label, explicit API/product questions, and two source-reproducible correctness blockers.

## What This Changes

This PR adds a structured doctor health-check contract, `openclaw doctor --lint` reporting with selectors, structured repair validation, a `plugin-sdk/health` export, docs, and focused tests.

## Best Possible Solution

Land one maintainer-approved diagnostics surface with lint using a no-observe config read path, selector behavior defined for non-lint modes, and SDK/docs aligned to the chosen API boundary.

## Reproduction Assessment

Yes. Source inspection of latest PR head shows the lint path calls the observed config snapshot reader, and `--only`/`--skip` are accepted by non-lint doctor modes without being applied or rejected.

## Solution Assessment

No, not as-is. The direction is coherent, but the implementation should preserve the documented read-only invariant and fail closed on lint-only selectors before maintainers choose this surface over the standalone lint draft.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep lint from writing config observation state:** `src/commands/doctor-lint.ts:44`
  - body: `doctor --lint` is documented as read-only, but this path calls `readConfigFileSnapshot()`. Current main finalizes snapshot reads through `observeConfigSnapshot()`, which can persist config health state or append audit records when the config fingerprint or suspicious signature changes, so a CI lint run can mutate `OPENCLAW_HOME` before returning findings. Use a no-observe snapshot path for lint and cover that invariant.
  - confidence: 0.9
- **[P2] Reject lint-only selectors outside lint mode:** `src/cli/program/register.maintenance.ts:31-32`
  - body: `--skip` and `--only` are registered on the whole `doctor` command, but only consumed in the `opts.lint` branch. `openclaw doctor --fix --only ...` is accepted and then runs the full repair doctor, which is more mutation than the user requested. Reject these flags outside `--lint`, or deliberately wire them into scoped repair.
  - confidence: 0.87
