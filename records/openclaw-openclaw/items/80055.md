## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not preload plugins before doctor repair runs:** `src/cli/command-catalog.ts:265-269`
  - body: This startup policy loads the full plugin registry for `doctor --fix`, `--repair`, and `--yes` before `doctorCommand` can run. Plugin registry startup resolves `getRuntimeConfig()` and uses `throwOnLoadError: true`, so stale plugin paths or invalid plugin config can abort the recovery command before the doctor repair flows that are supposed to fix that state.
  - confidence: 0.87
- **[P2] Return exit 2 for lint runtime failures:** `src/cli/program/register.maintenance.ts:35`
  - body: The new docs reserve exit code 2 for command/runtime failures before lint findings are produced, but this action uses the default `runCommandWithRuntime` error path, which reports uncaught lint errors with exit 1. That makes fatal lint failures indistinguishable from normal findings for automation callers.
  - confidence: 0.9
- **[P2] Reject invalid severity thresholds:** `src/commands/doctor-lint.ts:40`
  - body: Unsupported `--severity-min` values parse to null and then fall back to `info`, so a typo such as `--severity-min warnng` silently runs the broadest lint threshold instead of failing the command. CI-style gates should reject invalid input rather than changing the requested threshold.
  - confidence: 0.86
