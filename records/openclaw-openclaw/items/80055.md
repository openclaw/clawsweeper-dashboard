## Summary

Keep open: this is an active draft PR with the protected `maintainer` label, current `main` does not already contain the doctor lint/health-check contract, and the patch still has two P2 correctness blockers around read-only lint behavior and selector flags.

## What This Changes

The PR adds a doctor health-check contract, `doctor --lint` reporting with selectors, structured repair validation, a `plugin-sdk/health` export, docs, and focused tests.

## Best Possible Solution

Land one maintainer-approved diagnostic surface with lint using a no-observe config read path, selector flags either rejected outside lint or deliberately wired into scoped repair, and focused regression coverage for both invariants.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the lint path uses the normal observed config snapshot reader, and the doctor action accepts `--only`/`--skip` while dropping them outside the lint branch.

## Solution Assessment

No as-is. The direction is coherent, but the implementation should preserve the documented read-only contract and make selector flags fail closed or intentionally scope repair before maintainer product review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep doctor lint from writing config observation state:** `src/commands/doctor-lint.ts:44`
  - body: `doctor --lint` is documented as read-only, but this path uses the normal config snapshot reader. On current main that reader calls `observeConfigSnapshot()`, which can persist config health state or audit records when the config fingerprint or suspicious signature changes, so CI lint can mutate `OPENCLAW_HOME` before reporting findings. Use a no-observe snapshot path or suppress observation for lint, and cover that invariant in tests.
  - confidence: 0.9
- **[P2] Reject lint-only selectors outside lint mode:** `src/cli/program/register.maintenance.ts:31-32`
  - body: `--only` and `--skip` are registered on the whole `doctor` command, but they are only consumed in the `opts.lint` branch. `openclaw doctor --fix --only ...` will be accepted and then run the full repair doctor, which is more mutation than the user asked for. Reject these flags outside `--lint`, or wire them intentionally into scoped repair.
  - confidence: 0.86
