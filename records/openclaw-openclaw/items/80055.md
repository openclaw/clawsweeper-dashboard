## Summary

Keep this draft open. The direction and terminal proof are useful, but the patch still violates its documented read-only lint contract and accepts lint-only selectors in repair mode where they are ignored; the protected maintainer label and explicit doctor-core/SDK questions also require human approval before merge.

## What This Changes

The PR adds structured doctor health checks, a read-only `doctor --lint` mode with JSON/filter options, structured repair validation, SDK health exports, docs, generated SDK metadata, and focused tests.

## Best Possible Solution

Land this only after maintainer approval, with lint using a no-observe config snapshot path and selector flags either rejected outside `--lint` or intentionally wired into scoped repair.

## Reproduction Assessment

Yes. The no-write issue is source-reproducible from PR head calling `readConfigFileSnapshot()` while current main can persist observation state during that read; the selector issue follows from the CLI action only forwarding `--skip` and `--only` inside the lint branch.

## Solution Assessment

No as-is. The feature direction is coherent, but the implementation must preserve the documented read-only lint contract, handle selector flags consistently, and wait for maintainer approval on the doctor and SDK health surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep doctor lint from writing config observation state:** `src/commands/doctor-lint.ts:44`
  - body: `doctor --lint` is documented as read-only, but this path still calls the normal `readConfigFileSnapshot()`. Current config IO finalizes that read through observation and can persist config health or audit state, so CI/preflight lint can mutate `OPENCLAW_HOME` before reporting findings. Use a no-observe snapshot path or suppress observation for lint, and cover the invariant in tests.
  - confidence: 0.9
- **[P2] Reject lint-only filters outside lint mode:** `src/cli/program/register.maintenance.ts:31-32`
  - body: Registering `--only` and `--skip` on the doctor command makes Commander accept them for every doctor mode, but the non-lint branch drops those options and runs the full repair doctor. A user running `doctor --fix --only ...` now gets broad mutation instead of a parse error or scoped repair. Reject these flags unless `--lint` is set, or deliberately pass them into a scoped repair implementation.
  - confidence: 0.84
