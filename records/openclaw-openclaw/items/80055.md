## Summary

Keep open: this is an active draft with a protected maintainer label, current main does not already have the proposed doctor lint contract, and the latest patch still has a source-reproducible read-only violation in the new lint path.

## What This Changes

The PR adds a structured doctor health-check contract, `openclaw doctor --lint` with JSON/filter options, structured repair validation, migrated core checks, SDK health exports, docs, generated SDK metadata, and focused tests.

## Best Possible Solution

Land the structured doctor lint contract only after maintainer approval and after lint uses a snapshot path that cannot observe, audit, migrate, or write state.

## Reproduction Assessment

Yes. The blocking issue is source-reproducible: `runDoctorLintCli` calls `readConfigFileSnapshot()`, and current config IO finalizes that read through `observeConfigSnapshot()`, which can write config health or audit state.

## Solution Assessment

No as-is. The direction is coherent, but the implementation must first preserve the documented no-write lint contract and maintainers still need to approve the doctor-core and SDK health surface choices.
