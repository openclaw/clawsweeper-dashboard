## Summary

Keep open: current main does not contain the doctor health-check contract or `doctor --lint`, and this active draft still has two correctness blockers around the read-only lint promise and lint-only selectors being accepted by repair mode. The PR also carries a protected maintainer label and explicitly asks for CLI/API direction before this surface lands.

## What This Changes

Adds a structured doctor health-check contract, `doctor --lint` reporting and selectors, structured repair validation for `doctor --fix`, a `plugin-sdk/health` export, docs, SDK metadata, and focused tests.

## Best Possible Solution

Land one maintainer-approved diagnostics surface with lint using a no-observe config read path, selector flags either rejected outside lint or intentionally wired into scoped repair, and SDK boundary/docs aligned with the chosen API.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR lint path calls the normal observed config snapshot reader, and the doctor action accepts `--only`/`--skip` while dropping them in the non-lint branch.

## Solution Assessment

No, not as-is. The direction is coherent, but the implementation should preserve the read-only contract and fail closed on lint-only selectors before maintainers decide whether this or the standalone lint PR is the right surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep doctor lint from writing observation state:** `src/commands/doctor-lint.ts:44`
  - body: `doctor --lint` is documented as read-only, but this path calls `readConfigFileSnapshot()`. Current main finalizes every snapshot through `observeConfigSnapshot()`, which can persist config health state or audit records when the config fingerprint or suspicious signature changes, so a CI lint run can mutate `OPENCLAW_HOME` before returning findings. Use a no-observe snapshot path for lint and cover that invariant in tests.
  - confidence: 0.9
- **[P2] Reject lint-only selectors outside lint mode:** `src/cli/program/register.maintenance.ts:31-32`
  - body: `--skip` and `--only` are registered on the whole `doctor` command, but they are only consumed in the `opts.lint` branch. `openclaw doctor --fix --only ...` is accepted and then runs the full repair doctor, which is more mutation than the user asked for. Reject these flags outside `--lint`, or deliberately wire them into scoped repair.
  - confidence: 0.86
