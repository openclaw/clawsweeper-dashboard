## Summary

Keep open: current main still lacks Debug Event Log sticky-scroll handling, but this PR has blocking correctness gaps around shared event-log ordering and test surfaces, plus no inspectable after-fix real behavior proof.

## What This Changes

The PR adds sticky-at-bottom scroll state and tests for the Debug Event Log, changes gateway event-log buffering to append order, and updates Overview to reverse recent entries.

## Best Possible Solution

Repair the branch so Debug uses Logs-like sticky-at-bottom behavior with teardown and tests, keep one shared event-log ordering contract across all producers, then attach redacted runtime proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main renders the Debug Event Log without scroll-state handling and only schedules sticky scrolling for Logs; the focused path is to open Debug, scroll up in Event Log, then receive new gateway events.

## Solution Assessment

Yes, directionally. Reusing the existing Logs-style sticky-at-bottom pattern is the narrow maintainable fix, but this implementation must align the shared event-log buffer contract and test surfaces first.
