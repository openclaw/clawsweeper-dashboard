## What This Changes

This PR adds Debug Event Log sticky-at-bottom scroll state, changes the shared event log toward append order with Overview reversing recent entries, wires Debug scroll callback/state/CSS, and adds focused app-scroll tests.

## Best Possible Solution

Refresh and repair the branch so Debug uses Logs-like sticky-at-bottom behavior while keeping the shared event-log buffer contract consistent across all producers, then attach redacted real-behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Debug renders the event log without scroll-state handling while lifecycle only schedules sticky scrolling for Logs; opening Debug, scrolling up in Event Log, then receiving new gateway events is the focused reproduction path.

## Solution Assessment

Yes, directionally. Mirroring the existing Logs sticky-scroll pattern is the narrow maintainable fix, but this branch needs the shared ordering, test fixture, teardown, and proof gaps fixed first.
