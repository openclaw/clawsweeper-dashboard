## Summary

Keep open: current main still has the source-reproducible Debug Event Log scroll problem, and this PR is a focused fix path, but the branch is not merge-ready because it incompletely changes the shared event-log ordering contract, leaves test surfaces stale, misses scroll-frame cleanup, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds sticky-at-bottom scrolling for the Control UI Debug Event Log, changes the shared event-log buffer to append order, wires Debug scroll callbacks/state/CSS, and adds scroll tests.

## Best Possible Solution

Refresh and repair the branch so Debug uses Logs-like sticky-at-bottom behavior without changing shared event-log ordering inconsistently, then attach redacted real-behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Debug renders `.debug-event-log` without scroll-state handling while lifecycle schedules sticky scrolling only for Logs; open Debug, scroll up in Event Log, then receive new gateway events.

## Solution Assessment

Yes, directionally. Mirroring the existing Logs sticky-scroll pattern is the narrow maintainable fix, but this branch needs the shared ordering, test, teardown, and proof gaps fixed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Update all event log producers to append order:** `ui/src/ui/views/overview-event-log.ts:16-17`
  - body: Changing Overview to `slice(-20).toReversed()` assumes `eventLogBuffer` is oldest-first, but `recordControlUiPerformanceEvent` still prepends into the same buffer. Once performance events mix with gateway events, Debug and Overview can show mixed ordering and retention can keep the wrong side; update that producer and its tests to the same append-order contract.
  - confidence: 0.91
- **[P2] Mock the new scroll helper in lifecycle tests:** `ui/src/ui/app-lifecycle.ts:13`
  - body: Importing `scheduleDebugEventLogScroll` makes the existing `app-lifecycle-connect.node.test.ts` mock for `./app-scroll.ts` incomplete. Vitest will fail when `app-lifecycle.ts` imports a named export the mock does not provide; add the helper to that mock.
  - confidence: 0.93
- **[P2] Add the scroll callback to DebugProps fixtures:** `ui/src/ui/views/debug.ts:22`
  - body: `onEventLogScroll` is now a required `DebugProps` field, but the `createProps` fixture in `ui/src/ui/views/debug.test.ts` still returns props without it. Add a default no-op callback so the test type surface matches the new required prop.
  - confidence: 0.88
- **[P3] Cancel the debug scroll frame on disconnect:** `ui/src/ui/app.ts:515`
  - body: The PR adds a `debugEventLogScrollFrame` requestAnimationFrame handle, but lifecycle teardown only cancels chat and logs frames. If the app disconnects while a Debug log scroll is queued, the callback can still run against a torn-down view; include the new frame in cleanup.
  - confidence: 0.78
