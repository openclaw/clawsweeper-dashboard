## Summary

Keep open: the reported Debug Event Log scroll bug is real and the patch direction is useful, but the current branch still has blocking ordering/test/teardown defects and lacks required inspectable real-behavior proof.

## What This Changes

The PR adds sticky-at-bottom Debug Event Log scroll handling, constrains the log panel, changes event-log ordering toward append order, and adds focused scroll tests.

## Best Possible Solution

Refresh and repair the branch so Debug uses Logs-like sticky-at-bottom behavior, all event-log producers share one ordering contract, teardown/tests are updated, and redacted real-behavior proof is attached before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Debug renders the event log without scroll-state handling while lifecycle only schedules sticky scrolling for Logs; opening Debug, scrolling up in Event Log, then receiving new gateway events is the focused reproduction path.

## Solution Assessment

Yes, directionally. Mirroring the existing Logs sticky-scroll pattern is the narrow maintainable fix, but this branch needs the shared ordering, fixture/mock, teardown, and proof gaps fixed first.
