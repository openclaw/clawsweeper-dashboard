## What This Changes

The PR adds sticky-at-bottom scrolling for the Control UI Debug Event Log, changes event buffer ordering for gateway events, preserves Overview recency, wires a Debug scroll callback through state/rendering, and adds scroll helper tests.

## Best Possible Solution

Refresh the branch onto current main, keep the Debug sticky-bottom approach, align every event-log producer to one ordering contract, repair the test and teardown gaps, and collect redacted real-behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Debug renders the Event Log without scroll state or a scroll handler while lifecycle only schedules sticky scrolling for Logs, so opening Debug, scrolling up, and receiving new gateway events is a high-confidence reproduction path.

## Solution Assessment

Yes, directionally. Mirroring the existing Logs sticky-scroll pattern for Debug is the narrow maintainable fix, but this branch is not merge-ready until the ordering producer, test fixture, teardown, conflict, and proof gaps are handled.
