The PR changes the Matrix decrypt bridge to retain exhausted failed-decryption retry entries, requeue them on Matrix crypto key signals through the SDK event retry path, clear retained retry state on bridge stop, and adds focused Matrix regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes, with source-level confidence: current main and v2026.6.9 show exhausted decrypt entries are removed from the retry map and never requeued by later crypto key signals. The PR body also supplies live Synapse after-fix output showing the late-key recovery path delivering room.message.
