## Summary

Keep open: the reported Feishu topic-session split is valid and the PR includes usable live proof, but the patch is not merge-ready because its normal-group hydration path can re-key existing OpenClaw-created `om_*` root sessions to `omt_*` before routing.

## What This Changes

The PR changes Feishu group-topic session routing to hydrate topic `threadId` from `rootId`, suppress `rootId` when an `omt_*` topic is detected, and updates Feishu docs, changelog, and bot tests.

## Best Possible Solution

Preserve the topic-message-format fix only behind a reliable Feishu topic-context discriminator, while keeping documented `om_*` root-based session keys for OpenClaw-created normal group threads and covering both paths in regression tests.

## Reproduction Assessment

Yes for the reported splitting: current source prefers `rootId` over `threadId` for normal `chat_type: "group"`, and the PR body includes before/after live Feishu logs for both group types. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The proposed direction is right for real Feishu topic contexts, but the implementation suppresses `rootId` for too broad a normal-group path and conflicts with the existing root-based thread contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep root-based sessions for normal Feishu threads:** `extensions/feishu/src/bot.ts:547`
  - body: This condition now hydrates any `chat_type: "group"` topic-scoped message with an `om_*` root before session resolution. Existing normal group threads intentionally keep `rootId` ahead of `threadId`, and current tests model an `om_*` root whose `getMessageFeishu` result has `threadId: "omt_topic_1"` for history while the session remains rooted on `om_*`. With this branch, that same path is re-keyed to `oc-group:topic:omt_topic_1`, splitting OpenClaw-created thread sessions from their first-turn `om_*` key. Please add a reliable discriminator and regression coverage before suppressing `rootId`.
  - confidence: 0.89
