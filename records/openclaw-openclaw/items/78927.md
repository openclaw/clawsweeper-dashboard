## Summary

Keep open: the reported Feishu topic-session split is credible and the PR includes useful real logs, but the patch still has blocking correctness and security issues around topic discrimination and pre-admission Feishu API reads.

## What This Changes

The PR broadens Feishu topic-session hydration to normal groups, queries the replied-to root message for canonical `omt_*` IDs, suppresses root IDs in detected topic contexts, and updates Feishu docs, changelog, and tests.

## Best Possible Solution

Keep the fix Feishu-local: reliably distinguish Feishu topic-message-format groups from OpenClaw-created normal reply threads, hydrate canonical `omt_*` IDs after admission checks, and preserve existing root-keyed normal threads with focused regression coverage.

## Reproduction Assessment

Yes for source/log-level review: current main still prefers `rootId` over `threadId` for normal `chat_type: "group"` topic scopes, and the PR body includes before/after Feishu WebSocket logs. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The Feishu plugin is the right surface, but this patch uses `om_*` root presence as too broad a topic discriminator and performs the new lookup before access checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve normal-group root-keyed sessions:** `extensions/feishu/src/bot.ts:589`
  - body: When `hydratedToNativeTopic` is set for a normal `group` event carrying both `thread_id: omt_*` and `root_id: om_*`, this call suppresses `rootId` and changes the route from the documented/tested `om_*` key to `omt_*`. Add a reliable topic-message-format discriminator or preserve root-keyed normal reply threads.
  - confidence: 0.9
- **[P2] Gate topic hydration behind admission:** `extensions/feishu/src/bot.ts:551`
  - body: The broadened normal-group hydration can call `getMessageFeishu` before disabled-group, group allowlist, sender allowlist, and mention checks return. Move the lookup behind those admission gates while still running before session routing so ignored traffic cannot trigger authenticated Feishu message reads.
  - confidence: 0.84
