## Summary

Keep open: the PR targets a real Feishu session-splitting bug and includes useful live-log proof, but it is not merge-ready because it can re-key existing normal-group topic sessions, expands authenticated Feishu message reads before admission gates, edits release-owned changelog content, and is currently merge-conflicting.

## What This Changes

The PR expands Feishu topic-thread hydration/routing to normal topic-message-format groups, suppresses `rootId` for detected `omt_*` topic contexts, adds Feishu regression tests, and updates Feishu docs plus `CHANGELOG.md`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably distinguishes topic-message-format normal groups or provides a maintainer-approved migration/alias, moves hydration behind admission gates, removes the changelog edit, and rebases cleanly.

Yes, at source level. Current main can choose `rootId` before `threadId` for normal `chat_type: "group"` topic-scoped events, and the PR body includes copied before/after live Feishu logs for the same split shape.

## Solution Assessment

No. The Feishu plugin is the right owner and the branch is a plausible fix, but it is not the safest fix because it changes existing session keys, performs broader pre-admission Feishu message reads, and currently conflicts with `main`.
