## Summary

Keep open: the PR targets a real Feishu session-splitting bug and includes useful live logs, but the current diff is not safe to merge because it can re-key documented normal-group reply threads and broadens Feishu message reads before admission checks.

## What This Changes

The PR changes Feishu topic routing to hydrate canonical `omt_*` topic IDs from the replied-to message, suppress `rootId` when topic context is detected, and updates Feishu docs, tests, and changelog.

## Best Possible Solution

Preserve the topic-message-format fix behind a reliable Feishu topic-context discriminator, keep OpenClaw-created normal group threads rooted on `om_*`, and move any extra Feishu API lookup behind group and sender admission gates.

## Reproduction Assessment

Yes for source/log-level review: current main still prefers `rootId` over `threadId` for normal `chat_type: "group"` topic scopes, and the PR body includes before/after live Feishu WebSocket logs. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The direction is right for real Feishu topic-message-format groups, but this implementation uses `om_*` root presence as too broad a discriminator and should be narrowed before changing routing keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep root-based sessions for normal Feishu threads:** `extensions/feishu/src/bot.ts:582`
  - body: This branch treats normal-group `omt_*` thread plus `om_*` root as a native topic signal and suppresses `rootId`. Existing docs/tests keep `rootId` ahead of `threadId` for normal group reply-created threads, so this can re-key an existing `om_*` session to `omt_*` and split it. Please add a reliable discriminator before suppressing `rootId`.
  - confidence: 0.9
- **[P2] Gate broadened hydration behind group admission:** `extensions/feishu/src/bot.ts:540-547`
  - body: The new normal-group hydration path still runs before disabled-group, group allowlist, and sender allowlist returns. With topic session scope enabled, a non-admitted normal group reply carrying an `om_*` root can now trigger an authenticated Feishu message read, so move this lookup behind admission gates while still running before session routing.
  - confidence: 0.84
