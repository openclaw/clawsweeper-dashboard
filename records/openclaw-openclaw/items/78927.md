## Summary

Keep open: this is a plausible Feishu topic-session bug with real before/after logs, but the PR still changes normal-group routing too broadly and moves extra authenticated Feishu message reads into a pre-admission path.

## What This Changes

The PR changes Feishu topic routing to hydrate canonical `omt_*` topic IDs from replied-to messages, suppress root IDs when topic context is detected, and updates docs, changelog, and tests.

## Best Possible Solution

Keep the fix Feishu-local, but preserve normal root-keyed sessions, add a reliable topic-message-format discriminator, and move extra Feishu API lookups behind admission gates before routing.

## Reproduction Assessment

Yes for source/log-level review: current main still prefers `rootId` over `threadId` for normal `chat_type: "group"` topic scopes, and the PR body includes before/after live Feishu WebSocket logs. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The Feishu plugin is the right surface, but this patch uses `om_*` root presence as too broad a topic discriminator and performs the new lookup before access checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve normal-group root sessions:** `extensions/feishu/src/bot.ts:589`
  - body: This line suppresses `rootId` whenever the new `hydratedToNativeTopic` flag is set, including normal `group` events with an `omt_*` thread and `om_*` root. Current docs and tests require those OpenClaw-created normal group threads to keep the `om_*` root key, so this can re-key an existing session to `omt_*` and split context.
  - confidence: 0.9
- **[P2] Gate topic hydration behind admission:** `extensions/feishu/src/bot.ts:546-551`
  - body: The broadened normal-group hydration can call `getMessageFeishu` before disabled-group, group allowlist, sender allowlist, and mention checks return. Move the lookup behind those admission gates while still running before session routing so ignored traffic cannot trigger authenticated Feishu message reads.
  - confidence: 0.84
