## Summary

Keep open: this is a proof-backed Feishu bug-fix candidate, but the current patch still changes existing Feishu session keys and expands authenticated message reads before group, sender, and mention admission. Current main and v2026.6.10 still have the underlying normal-group topic split shape, so the PR is not obsolete.

## What This Changes

The PR expands Feishu topic thread hydration/routing to normal groups using topic message format, suppresses `rootId` when an `omt_*` topic thread is detected, and updates Feishu tests/docs plus `CHANGELOG.md`.

## Best Possible Solution

Land one Feishu plugin fix that reliably distinguishes Feishu topic-message-format normal groups or preserves existing `om_*` routing by default, moves hydration behind admission gates, and leaves changelog generation to the release process.

## Solution Assessment

No. The Feishu plugin is the right owner and the branch is a plausible fix, but it is not the safest fix because it changes existing session keys and performs broader pre-admission Feishu message reads.
