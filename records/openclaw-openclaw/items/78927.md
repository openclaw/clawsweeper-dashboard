## Summary

This PR should stay open because current main and v2026.6.8 still have the Feishu normal-group topic session split behavior, but the branch is not merge-ready: it can re-key existing normal-group topic sessions and expands authenticated Feishu message reads before admission checks.

## What This Changes

The PR changes Feishu topic-session hydration/routing, adds Feishu regression tests, updates Feishu docs, and edits the release-owned changelog.

## Best Possible Solution

Preserve existing normal-group rootId routing unless Feishu topic-message-format mode is positively identified or maintainers accept a documented migration, move expanded hydration behind admission checks, and keep release-note context out of `CHANGELOG.md`.

## Reproduction Assessment

Yes. Current main can receive both `rootId=om_*` and `threadId=omt_*`, and normal `group` topic scope resolves `rootId` before `threadId`; the PR body also includes live before/after Feishu logs.

## Solution Assessment

No. The Feishu plugin is the right owner, but the implementation still needs a reliable normal-group topic-message-format discriminator or a maintainer-approved session migration path, plus safer hydration ordering.
