## Summary

Keep open: the PR targets a real Feishu topic-session bug and includes usable live-log proof, but the current diff treats `om_*` roots in normal groups as a topic-message-format signal too broadly and can break the existing root-based normal-thread contract.

## What This Changes

The PR changes Feishu topic routing to hydrate thread IDs from `rootId`, suppress `rootId` when an `omt_*` topic is detected, and updates Feishu docs, changelog, and bot tests.

## Best Possible Solution

Preserve the topic-message-format fix behind a reliable Feishu topic-context discriminator, keep OpenClaw-created normal group threads rooted on `om_*`, and move any extra Feishu API lookup behind group/sender admission gates.

## Reproduction Assessment

Yes for source/log-level review: current main still prefers `rootId` over `threadId` for normal `chat_type: "group"` topic scopes, and the PR body includes before/after live Feishu WebSocket logs. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The direction is right for real Feishu topic-message-format groups, but the implementation uses `om_*` root presence as too broad a discriminator and should be narrowed before changing routing keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep root-based sessions for normal Feishu threads:** `extensions/feishu/src/bot.ts:546`
  - body: This branch treats every normal-group `om_*` root as a native topic signal. Existing normal group/topic-mode tests and docs keep `rootId` ahead of `threadId` for OpenClaw-created reply threads, including cases where Feishu history lookup can return an `omt_*` id, so this can re-key an existing `om_*` session to `omt_*` and split it. Please add a reliable discriminator before suppressing `rootId`.
  - confidence: 0.9
- **[P2] Gate broadened hydration behind group admission:** `extensions/feishu/src/bot.ts:546`
  - body: The new normal-group hydration path still runs before disabled-group, group allowlist, and sender allowlist returns. With a global topic session scope, a non-admitted normal group reply carrying an `om_*` root can now trigger an authenticated Feishu message read, so this lookup should move behind admission gates while still running before session routing.
  - confidence: 0.82
