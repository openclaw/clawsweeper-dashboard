## Summary

Keep open: the Feishu session-stability problem is real and the PR has useful live proof, but the patch is not merge-ready because it changes an existing normal-group session-key invariant and expands Feishu message fetching before access checks.

## What This Changes

The PR expands Feishu topic-thread hydration for normal groups, suppresses `rootId` when `omt_*` topic context is detected, and updates Feishu docs, tests, and CHANGELOG.

## Best Possible Solution

Preserve existing rootId-based normal-group sessions unless Feishu topic-message-format mode is positively identified, keep hydration behind group/sender access checks, and leave release-note context in the PR body instead of `CHANGELOG.md`.

## Solution Assessment

No. The patch is plausible for Feishu topic-message-format groups, but using `om_*`/`omt_*` shape as the discriminator also matches existing normal group thread cases, so the best fix needs a reliable signal or an explicit maintainer-approved migration.
