## Summary

Keep open: the Feishu session-splitting bug is still present on current main and the latest release, but this PR is not merge-ready because it can re-key existing normal-group topic sessions and expands an authenticated Feishu message read before group/sender admission completes.

## What This Changes

The PR changes Feishu message handling to hydrate and prefer `omt_*` topic IDs for native topic groups and normal groups using topic message format, adds Feishu regression tests, updates Feishu docs, and edits `CHANGELOG.md`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably distinguishes Feishu topic-message-format normal groups or preserves existing `om_*` routing by default, moves hydration behind admission gates, removes the changelog edit, and keeps focused tests plus redacted Feishu proof.
