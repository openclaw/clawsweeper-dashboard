## Summary

Keep this PR open, but do not merge it as-is: it addresses a real Feishu topic-session bug with useful live-log proof, while the current diff still conflicts with normal-group root-based session routing and broadens authenticated Feishu message reads before admission checks.

## What This Changes

The PR changes Feishu topic routing to hydrate thread IDs from root messages, suppress root IDs when an omt topic is detected, and updates Feishu docs, changelog, and tests.

## Best Possible Solution

Preserve the topic-message-format fix behind a reliable Feishu topic-context discriminator and move any extra Feishu API lookup behind group and sender admission gates.

## Reproduction Assessment

Yes for source/log-level review: current main still prefers rootId over threadId for normal chat_type="group" topic scopes, and the PR body includes before/after live Feishu WebSocket logs. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The direction is plausible for real Feishu topic-message-format groups, but the implementation uses om_* root presence as too broad a discriminator and should be narrowed before changing routing keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep root-based sessions for normal Feishu threads:** `extensions/feishu/src/bot.ts:546`
  - body: This branch treats every normal-group om_* root as a native topic signal. Current docs and tests keep rootId ahead of threadId for normal groups so OpenClaw-created reply threads stay on the initiating message session; suppressing rootId here can re-key an existing om_* session to omt_* and split it. Please add a reliable Feishu topic-message-format discriminator before suppressing rootId.
  - confidence: 0.9
- **[P2] Gate broadened hydration behind group admission:** `extensions/feishu/src/bot.ts:546`
  - body: The new normal-group hydration path still runs before disabled-group, group allowlist, and sender allowlist returns. With topic-scoped routing enabled, an event OpenClaw later ignores can now trigger an authenticated Feishu message read, so move the expanded lookup behind admission checks while preserving routing before dispatch.
  - confidence: 0.82
