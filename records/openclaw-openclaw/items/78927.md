## Summary

Keep open: this is a useful Feishu bug-fix candidate with sufficient real log proof, but it is not merge-ready because the patch can re-key existing normal-group topic sessions and expands authenticated Feishu message reads before admission gates.

## What This Changes

The PR expands Feishu topic-thread hydration/routing to normal groups with topic message format, suppresses rootId when an `omt_*` topic id is present, adds regression tests, updates Feishu docs, and edits the changelog.

## Best Possible Solution

Land one Feishu-plugin fix that reliably distinguishes topic-message-format normal groups or preserves existing `om_*` routing by default, moves new hydration behind admission gates, and leaves changelog generation to the release process.

## Reproduction Assessment

Yes, at source level. Current main can choose `rootId` before `threadId` for normal `chat_type: "group"` topic-scoped events, and the PR body includes copied before/after live Feishu logs for the same split shape.

## Solution Assessment

No. The Feishu plugin is the right owner and the branch is a plausible fix, but it is not the safest fix because it changes existing session keys and performs broader pre-admission Feishu message reads.
