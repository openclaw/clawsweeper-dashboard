## What This Changes

This PR changes Feishu topic-session hydration/routing to hydrate thread IDs from root messages, suppress rootId when native topic context is detected, and update Feishu docs, tests, and the changelog.

## Best Possible Solution

Preserve root-id routing for existing normal-group reply-created threads unless Feishu topic-message-format mode is positively identified, move hydration behind admission gates or prove a metadata-only path, and keep release-note context out of CHANGELOG.md.
