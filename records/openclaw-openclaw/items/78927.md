## Summary

Keep open: this is meaningful Feishu session-state work with strong live-log proof, but the branch is not merge-ready because it can re-key existing normal-group sessions and expands authenticated Feishu message reads before admission gates.

## What This Changes

The PR expands Feishu topic-thread hydration/routing to normal topic-message-format groups, suppresses `rootId` for detected `omt_*` topics, adds Feishu tests, and updates Feishu docs plus `CHANGELOG.md`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably distinguishes topic-message-format normal groups or provides a maintainer-approved migration/alias, moves hydration behind admission gates, and leaves changelog generation to release tooling.
