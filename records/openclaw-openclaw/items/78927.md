## Summary

Keep open: the PR targets a real Feishu topic-session bug with useful live-log proof, but the patch still has blocking issues. Current main does not cover normal groups using Feishu topic message format, while this PR can re-key existing normal-group root sessions and broadens authenticated Feishu message reads before admission gates.

## What This Changes

The PR hydrates canonical Feishu `omt_*` topic IDs from replied-to/root messages for native topic groups and normal groups using topic message format, suppresses root IDs during routing, and updates Feishu tests, docs, and changelog.

## Best Possible Solution

Keep the fix Feishu-local, but hydrate only after group and sender admission and re-key normal groups only when a reliable Feishu topic-message-format signal proves `omt_*` is canonical, preserving OpenClaw-created root-keyed threads.

## Reproduction Assessment

Yes for source/log-level review: current main still only hydrates native `topic_group` events and keeps root precedence for normal groups, while the PR body includes before/after Feishu WebSocket logs showing the reported split and the intended stable `omt_*` sessions. I did not run a live Feishu tenant from this read-only checkout.

## Solution Assessment

No. The Feishu plugin is the right surface, but this patch uses `om_*` root presence as too broad a topic discriminator and performs extra Feishu message reads before admission gates.
