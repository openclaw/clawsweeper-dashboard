## Summary

Keep open: this is a useful, proof-backed Feishu bug-fix attempt, but the branch is not merge-ready because it can silently re-key existing normal-group sessions, expands an authenticated message-read path before group admission checks, edits release-owned changelog content, and is currently conflicting with main.

## What This Changes

The PR expands Feishu topic-thread hydration and routing to normal topic-message-format groups, suppresses `rootId` when an `omt_*` topic is detected, adds Feishu routing tests, and updates Feishu docs plus `CHANGELOG.md`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably distinguishes Feishu topic-message-format normal groups or provides a maintainer-approved migration/alias, moves hydration behind admission gates, and leaves changelog generation to release tooling.

Yes, at source level: current main can choose `rootId` before `threadId` for normal `chat_type: "group"` topic-scoped events, and the PR body includes copied before/after Feishu WebSocket logs for the same split shape. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

No. The Feishu plugin is the right owner and the branch is a plausible fix, but it is not the safest fix because it changes existing session keys and performs broader pre-admission Feishu message reads.
