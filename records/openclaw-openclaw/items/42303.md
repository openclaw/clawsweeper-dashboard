## Summary

Keep open. Current main still lacks a settled fix for #14593, but this PR is not merge-ready because it targets stale Docker setup wiring, has build-breaking installer quoting, and needs maintainer choice among the competing Docker/skills strategies in #55940 and #57553.

## What This Changes

The branch re-adds optional Linuxbrew installation to the Docker image and wires `OPENCLAW_INSTALL_BREW` plus `OPENCLAW_SKIP_BUILD` through the Docker setup script.

## Best Possible Solution

Resolve #14593 with one chosen Docker/skills strategy against the current Docker layout, with verified install behavior, docs, changelog, and supply-chain posture aligned.

## Reproduction Assessment

Yes, from source inspection: current main has no `OPENCLAW_INSTALL_BREW` Docker build/setup path, and brew-backed skill installs return `brew not installed` when no brew executable is present. I did not run a live Docker build in this read-only review.

## Solution Assessment

No. The PR direction may be viable, but this patch targets stale setup wiring and has concrete Dockerfile and recovery-message defects; the safer path is a current-main replacement after maintainers choose the #14593 strategy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire the active Docker setup script:** `docker-setup.sh:219`
  - body: Current main and the Docker docs use `scripts/docker/setup.sh`, but this PR wires `OPENCLAW_INSTALL_BREW` into the old root `docker-setup.sh` path. After rebase, the supported `./scripts/docker/setup.sh` flow would still omit the brew build arg unless the changes move to the active script.
  - confidence: 0.88
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands `$(curl ...)` in the Docker shell and passes the full installer as a single-quoted `bash -c` string. The Homebrew installer contains single quotes, so a brew-enabled build can fail while parsing before `brew` is installed; download to a temporary file or otherwise avoid embedding the script in shell quotes.
  - confidence: 0.92
- **[P3] Scope Homebrew env to brew-enabled images:** `Dockerfile:222-225`
  - body: These `ENV` instructions run regardless of `OPENCLAW_INSTALL_BREW`, so default images get `HOMEBREW_*` values and PATH entries for directories that were never created. Keep the runtime environment scoped to brew-enabled builds, for example with a profile drop-in created inside the conditional install block.
  - confidence: 0.84
- **[P2] Point skip-build failures at a real recovery command:** `docker-setup.sh:418`
  - body: The missing-image path tells users to run `openclaw build`, but current main does not register a top-level or subcommand named `build`. Point users to a real recovery path such as `docker build ...` or rerunning setup without `OPENCLAW_SKIP_BUILD`.
  - confidence: 0.9
- **[P3] Document the new Docker build option:** `docker-setup.sh:426`
  - body: This adds user-facing Docker env/build-arg behavior, but the Docker env-var table and changelog are not updated. If maintainers keep this approach, document `OPENCLAW_INSTALL_BREW` and the skip-build behavior with the same supported script path users are told to run.
  - confidence: 0.76
