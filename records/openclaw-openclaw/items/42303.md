## Summary

Keep open: the linked Docker/skills failure is still present on current main, but this PR is not merge-ready because it targets stale Docker setup wiring, has concrete Dockerfile defects, lacks real behavior proof, and still needs maintainer product/security choice among competing fixes.

## What This Changes

The PR re-adds optional Linuxbrew installation to the Docker runtime image and wires `OPENCLAW_INSTALL_BREW` plus `OPENCLAW_SKIP_BUILD` through a Docker setup script.

## Best Possible Solution

Resolve the linked Docker skill-install problem through one maintainer-approved strategy against the current `scripts/docker/setup.sh` layout, with Docker proof, regression coverage for the touched path, and docs updated for the chosen behavior.

## Reproduction Assessment

Yes, from source inspection: current main has no runtime Docker Linuxbrew build path, the active setup script does not pass a brew build arg, and `installSkill` returns `brew not installed` for brew specs when no brew executable resolves. I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Restoring Linuxbrew may be a viable strategy, but this PR wires the stale setup path and adds Dockerfile and supply-chain problems; the safer solution is a current-main fix after maintainers choose the Docker/package-manager posture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire the supported Docker setup script:** `docker-setup.sh:219`
  - body: Current main and the Docker docs use `scripts/docker/setup.sh`, and the root `docker-setup.sh` path is no longer present. Wiring `OPENCLAW_INSTALL_BREW` here would not make the supported setup flow pass the brew build arg after this PR is brought up to date.
  - confidence: 0.92
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands the installer with `$(curl ...)` and passes the full script as a single-quoted `bash -c` string. The Homebrew installer contains single quotes, so a brew-enabled build can fail during shell parsing before `brew` is installed; download the script to a temporary file or otherwise avoid embedding it in shell quotes.
  - confidence: 0.93
- **[P3] Scope Homebrew environment to brew-enabled images:** `Dockerfile:222-225`
  - body: These `ENV` instructions run for every build, so default images get `HOMEBREW_*` values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled, for example via a profile drop-in created inside the conditional install block.
  - confidence: 0.86
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The missing-image path tells users to run `openclaw build`, but current CLI registration does not provide that command. Use a real recovery path such as rerunning setup without `OPENCLAW_SKIP_BUILD` or an explicit `docker build` command.
  - confidence: 0.9
