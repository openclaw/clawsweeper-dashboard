## Summary

Keep open for maintainer judgment: current main still has the Docker brew-missing failure path, but this branch is conflicting, wires an obsolete setup script, has Dockerfile correctness and supply-chain issues, and lacks inspectable real behavior proof.

## What This Changes

The PR adds an optional `OPENCLAW_INSTALL_BREW` Dockerfile block and wires `OPENCLAW_INSTALL_BREW` plus `OPENCLAW_SKIP_BUILD` through the old root `docker-setup.sh`.

## Best Possible Solution

Keep the canonical Docker skill-install issue open while maintainers choose one strategy; if Linuxbrew-in-Docker is selected, wire `scripts/docker/setup.sh`, harden the Dockerfile installer path, scope runtime env to brew-enabled images, update docs, and provide real Docker proof.

## Reproduction Assessment

Yes from source inspection: current Docker has no Linuxbrew build option, onboarding can offer a brew-only missing skill such as `openai-whisper`, and `installSkill` returns the brew-missing failure when no brew executable resolves. I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Optional Linuxbrew-in-Docker may be a valid strategy, but this patch is not mergeable as-is because it targets an obsolete setup script, has Dockerfile execution/env issues, and needs maintainer package-manager/security direction.

Full review comments:

- **[P2] Wire the supported Docker setup script:** `docker-setup.sh:426`
  - body: Current main and docs use `scripts/docker/setup.sh`, and root `docker-setup.sh` is absent. This branch only persists and passes `OPENCLAW_INSTALL_BREW` in the old root script, so after resolving conflicts the documented setup flow would still build without the brew arg; move the wiring to the supported setup script.
  - confidence: 0.93
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands the installer with `$(curl ...)` and passes the full script as a single-quoted `bash -c` string. Homebrew's installer contains single quotes, so a brew-enabled build can fail during shell parsing; download and execute a temporary file or otherwise avoid embedding the script in quotes.
  - confidence: 0.92
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The skip-build missing-image path tells users to run `openclaw build`, but current CLI registration/search does not expose a top-level `build` command. Point users to rerun setup without `OPENCLAW_SKIP_BUILD` or to an explicit Docker build command.
  - confidence: 0.88
- **[P3] Scope Homebrew env to brew-enabled builds:** `Dockerfile:222-225`
  - body: These `ENV` instructions run for every image, so default builds get `HOMEBREW_*` values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled.
  - confidence: 0.86
