## Summary

Keep open. Current main still has the Docker brew-missing path, but this PR is not safe to merge as-is because it targets a stale setup entrypoint, has Dockerfile correctness and supply-chain issues, and lacks inspectable real Docker proof.

## What This Changes

The PR adds an opt-in `OPENCLAW_INSTALL_BREW` Linuxbrew install path in `Dockerfile`, wires it through root `docker-setup.sh`, and adds an `OPENCLAW_SKIP_BUILD` guard.

## Best Possible Solution

Keep the canonical Docker skill-dependency issue open while maintainers choose one strategy, then land a focused current-main fix with Docker proof, regression coverage, and Docker/skills docs updates.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection shows the current Docker image has no Linuxbrew option, the supported setup script passes no brew build arg, and brew-based installs still return `brew not installed` when no executable resolves; I did not run a live Docker build in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. Restoring Linuxbrew may be one viable product direction, but this branch targets the removed root setup script and introduces Dockerfile correctness plus supply-chain concerns; a current-main fix should start from the chosen Docker/package-manager strategy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire the supported Docker setup script:** `docker-setup.sh:221`
  - body: Current main and the Docker docs use `scripts/docker/setup.sh`, while this branch wires `OPENCLAW_INSTALL_BREW` into root `docker-setup.sh`, which current main no longer has. After rebasing, the supported setup flow would still not pass the brew build arg.
  - confidence: 0.92
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands the installer with `$(curl ...)` and passes the full script as a single-quoted `bash -c` string. The Homebrew installer contains single quotes, so a brew-enabled build can fail during shell parsing; download and execute a temporary file or otherwise avoid embedding the script in quotes.
  - confidence: 0.93
- **[P3] Scope Homebrew env to brew-enabled builds:** `Dockerfile:222-225`
  - body: These `ENV` instructions run for every build, so default images get `HOMEBREW_*` values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled.
  - confidence: 0.86
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The missing-image path tells users to run `openclaw build`, but current CLI registration does not provide that top-level command. Use a real recovery path such as rerunning setup without `OPENCLAW_SKIP_BUILD` or an explicit `docker build` command.
  - confidence: 0.9
