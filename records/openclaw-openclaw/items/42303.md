## Summary

Keep open. Current main still lacks an official Docker/Linuxbrew path for brew-backed skill dependencies, but this branch is not merge-ready because it wires a removed root setup script, has Dockerfile correctness issues, lacks after-fix real Docker proof, and needs maintainer selection of the Docker/Linux package-manager strategy.

## What This Changes

The PR adds an optional Linuxbrew install block to `Dockerfile` and wires `OPENCLAW_INSTALL_BREW` plus `OPENCLAW_SKIP_BUILD` through a root `docker-setup.sh` script.

## Best Possible Solution

Choose one documented Docker/Linux skill-dependency strategy, implement it against current `Dockerfile` plus `scripts/docker/setup.sh` or the installer/onboarding path it actually affects, add focused regression coverage, and include real Docker or Linux proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the current Docker image and setup script do not install or pass Linuxbrew, while brew-backed skill installs still return `brew not installed` when no brew executable resolves; I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Restoring Linuxbrew may be a viable strategy, but this branch targets the stale root setup path and introduces Dockerfile correctness and supply-chain issues; the safer path is a maintainer-selected fix against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire the supported Docker setup script:** `docker-setup.sh:219`
  - body: Current main and the Docker docs use `scripts/docker/setup.sh`, while this PR wires `OPENCLAW_INSTALL_BREW` into the removed root `docker-setup.sh` path. After bringing the branch current, the supported setup flow would still not pass the brew build arg.
  - confidence: 0.92
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands the installer with `$(curl ...)` and passes the full script as a single-quoted `bash -c` string. The Homebrew installer contains single quotes, so a brew-enabled build can fail during shell parsing; download the script to a temporary file or otherwise avoid embedding it in quotes.
  - confidence: 0.93
- **[P3] Scope Homebrew environment to brew-enabled images:** `Dockerfile:222-225`
  - body: These `ENV` instructions run for every build, so default images get `HOMEBREW_*` values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled.
  - confidence: 0.86
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The missing-image path tells users to run `openclaw build`, but current CLI registration does not provide that top-level command. Use a real recovery path such as rerunning setup without `OPENCLAW_SKIP_BUILD` or an explicit `docker build` command.
  - confidence: 0.9
