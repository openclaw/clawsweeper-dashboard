## Summary

Keep open for maintainer follow-up: the linked Docker skill-install problem is still source-reproducible on current main, but this PR is not mergeable as-is because it targets the removed root setup script, adds fragile Dockerfile installer/env behavior, lacks real behavior proof, and crosses a package-manager trust boundary.

## What This Changes

The PR adds an optional Linuxbrew install block to the Dockerfile and wires OPENCLAW_INSTALL_BREW plus OPENCLAW_SKIP_BUILD through the old root docker-setup.sh.

## Best Possible Solution

Keep the canonical Docker skill-install issue open while maintainers choose one package-manager strategy, then land a current-main fix with Docker/Linux proof, focused regression coverage, and Docker plus skills docs updates.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current Docker has no Linuxbrew option, openai-whisper is brew-only, and installSkill returns the brew-missing failure when no brew executable resolves; I did not run a live Docker build in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. Optional Linuxbrew-in-Docker may be viable, but this branch is stale against the supported setup path and introduces correctness, proof, and supply-chain concerns that need maintainer strategy selection first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire the supported Docker setup script:** `docker-setup.sh:426`
  - body: Current main and the Docker docs use scripts/docker/setup.sh, while this branch wires OPENCLAW_INSTALL_BREW into root docker-setup.sh, which current main no longer has. After rebasing, the supported setup flow would still need the brew build arg in scripts/docker/setup.sh.
  - confidence: 0.93
- **[P2] Execute the Homebrew installer from a file:** `Dockerfile:215`
  - body: This expands the installer with $(curl ...) and passes the full script as a single-quoted bash -c string. Homebrew's installer contains single quotes, so a brew-enabled build can fail during shell parsing; download and execute a temporary file or otherwise avoid embedding the script in quotes.
  - confidence: 0.92
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The skip-build missing-image path tells users to run openclaw build, but current CLI registration/search does not expose a top-level build command. Point users to rerun setup without OPENCLAW_SKIP_BUILD or to an explicit docker build command.
  - confidence: 0.88
- **[P3] Scope Homebrew env to brew-enabled builds:** `Dockerfile:222-225`
  - body: These ENV instructions run for every image, so default builds get HOMEBREW_* values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled.
  - confidence: 0.86
