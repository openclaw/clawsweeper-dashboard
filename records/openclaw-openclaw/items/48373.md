## Summary

Keep open: current main and the latest release still return from the Feishu `create` branch after `createDoc` without writing or rejecting supplied content, and the open closing-reference PR is unmerged, so the issue remains necessary.

## What This Changes

The issue asks for `feishu_doc` create to stop silently discarding a supplied `content` value, preferably by creating and writing initial markdown in one call.

## Best Possible Solution

Land a refreshed Feishu plugin fix that either writes initial content through the existing write path with explicit partial-failure behavior and live proof, or rejects unsupported create-time content with a clear error.

## Reproduction Assessment

Yes, at source level. A `feishu_doc` create call with `title` and `content` can reach a create branch that returns after `createDoc`; no live Feishu call was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. Create-plus-write through `writeDoc` is narrow, but the final fix should preserve created metadata on write failure or explicitly reject create-time content.
