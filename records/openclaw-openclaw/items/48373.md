## Summary

Keep open: current main still accepts the reported Feishu create-with-content shape at the tool boundary but the `create` branch neither writes nor rejects `content`, and an open closing PR is already the implementation path.

## What This Changes

The issue requests that Feishu document creation either write supplied `content` during `create` or reject it instead of silently creating an empty document.

## Best Possible Solution

Land a repaired Feishu plugin change that writes create-time content through the existing `writeDoc` path while preserving recovery metadata, or rejects `content` on create with a clear error.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass `action: "create"`, `title`, and `content`; the dispatcher reaches the create branch and returns after `createDoc` without writing or rejecting the content.

## Solution Assessment

Yes. Create-plus-write through the existing `writeDoc` path is a narrow maintainable fix if create-time content remains accepted; explicit validation that rejects `content` on create is the safer fallback.
