## Summary

Keep open: current main and the latest shipped tag still return from the Feishu `create` branch after `createDoc` without writing or rejecting a supplied `content` value, and the linked implementation PR remains open and unmerged.

## What This Changes

The issue asks for `feishu_doc` create calls with `content` to stop silently producing empty documents, either by writing that content or failing clearly.

## Best Possible Solution

Keep one canonical Feishu plugin fix path: review and land or replace https://github.com/openclaw/openclaw/pull/48377, or choose a clear validation error for create-time `content` if maintainers reject create-plus-write.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and returns after `createDoc` without writing or rejecting `content`. I did not run a live Feishu API call in this read-only review.

## Solution Assessment

Yes. Reusing the existing `writeDoc` path after `createDoc` is the narrow maintainable implementation if create-time content is supported; explicit validation that rejects `content` on create is the safer fallback if maintainers do not want that API behavior.
