## Summary

Keep open: current main and the latest shipped release still return from the Feishu `create` branch after `createDoc` without using or rejecting a supplied `content` field, and https://github.com/openclaw/openclaw/pull/48377 remains the open implementation path.

## What This Changes

The issue asks Feishu `feishu_doc` create to stop silently dropping a supplied `content` field, preferably by writing the initial content during create.

## Best Possible Solution

Review, repair, and land or replace the linked Feishu plugin change so create-time content is either written through `writeDoc` with recoverable failure metadata or rejected with a clear validation error.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass `action: "create"`, `title`, and `content`; the dispatcher reaches `create` and returns after `createDoc` without writing or rejecting the content.

## Solution Assessment

Yes for the issue direction. Create-plus-write through the existing `writeDoc` path is maintainable if create-time content remains accepted; explicit rejection of `content` on create is the safer fallback.
