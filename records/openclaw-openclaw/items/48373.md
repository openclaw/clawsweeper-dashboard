## Summary

Keep open: current main and v2026.6.10 still route `feishu_doc` create directly to `createDoc` without writing or rejecting a supplied `content` value, while the only open implementation candidate is still unmerged outside the target repository.

## What This Changes

The issue asks for Feishu document creation to stop silently dropping `content`, preferably by creating and writing initial markdown in one `feishu_doc` call.

## Best Possible Solution

Land or sponsor a refreshed Feishu plugin fix that either writes create-time markdown through the existing write path with explicit partial-failure behavior, or rejects `content` before document creation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: model-facing schema normalization can expose `content`, validated args are passed into tool execution, and the Feishu `create` branch never consumes or rejects `p.content`.

## Solution Assessment

Unclear until maintainers choose the contract. Create-plus-write through `writeDoc` is narrow, but rejecting unsupported `content` before creation is also a safer valid fix if create should remain title-only.
