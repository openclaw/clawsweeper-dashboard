- `impact:data-loss`: The reported behavior can silently drop user-supplied document content while returning a successful create result.

## Summary

Keep open: current main and the latest release still return a successful Feishu document create result without writing or rejecting a supplied `content` value, and https://github.com/openclaw/openclaw/pull/48377 remains the open implementation path for this issue.

## What This Changes

The issue asks OpenClaw to stop silently dropping `content` passed to the Feishu `feishu_doc` create action, either by creating and writing content in one call or by rejecting the unsupported parameter clearly.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/48377; if maintainers reject create-time content, add a clear validation error for `content` on create instead of silently dropping it.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: pass `action: "create"`, `title`, and `content`; current main reaches the create branch and returns after `createDoc` without writing or rejecting `content`.

## Solution Assessment

Yes if maintainers want create-time content: reusing the existing `writeDoc` path after `createDoc` is the narrow maintainable implementation. If they do not want that API behavior, the safer fix is an explicit validation error rather than silent success.
