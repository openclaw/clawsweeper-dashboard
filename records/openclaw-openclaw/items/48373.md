## Summary

Keep open: current main still creates the Feishu document and returns without writing a supplied `content` value, and the linked implementation PR #48377 is still open. The issue should remain open until that PR is reviewed and landed, or maintainers choose the alternative of rejecting `content` on `create` with a clear error.

## What This Changes

Request that `feishu_doc` `create` stop silently discarding provided markdown content, either by writing it during create or by rejecting the unsupported parameter.

## Best Possible Solution

Review and land a narrow Feishu plugin fix that chains `createDoc` and `writeDoc` when `content` is present, with regression coverage and clear partial-failure semantics; otherwise explicitly reject `content` on `create`.

## Reproduction Assessment

Yes, source-reproducible: the current `create` branch returns `createDoc(...)` without any `p.content` check, while markdown population lives in `writeDoc`. I did not run a live Feishu call in this read-only review, but the dispatch path is direct.

## Solution Assessment

Yes, #48377 is the narrow implementation candidate because it reuses the existing `writeDoc` path and adds focused schema/docs/tests. The main thing to verify during PR review is the post-create write-failure behavior called out in the prior related PR.
