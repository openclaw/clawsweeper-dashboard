## What This Changes

The issue asks Feishu `feishu_doc` create to stop silently ignoring `content`, ideally by creating and populating initial markdown in one call.

## Best Possible Solution

Review and either land the open create-with-content PR after partial-failure handling and live proof, or merge a narrow alternative that rejects `content` on create with a clear error.

## Reproduction Assessment

Yes, at source level. A `feishu_doc` create call with `title` and `content` can reach the create branch, which calls only `createDoc`; no live Feishu call was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. Create-plus-write through the existing `writeDoc` path is narrow, but the final fix should preserve created metadata on write failure or explicitly reject create-time content.
