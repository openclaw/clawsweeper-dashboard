## Summary

Keep open: current main and the latest release still do not write or reject create-time `content`; model-facing schema normalization can expose `content`, while the Feishu create branch still returns after `createDoc` only. The same-repo alternatives are closed unmerged and the open cross-repo implementation candidate at https://github.com/BingqingLyu/openclaw/pull/864 is not merged, so this issue remains necessary.

## What This Changes

The issue reports that `feishu_doc` accepts `content` during `create` but silently creates a title-only document instead of writing or rejecting the content.

## Best Possible Solution

Land a refreshed Feishu plugin fix that either writes create-time markdown through the existing write path with explicit partial-failure behavior and proof, or rejects `content` on create before any document is created.

## Reproduction Assessment

Yes, at source level: current schema normalization can expose `content`, validated args are passed into tool execution, and the Feishu create branch never consumes `p.content`. I did not run a live Feishu call in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. Create-plus-write through `writeDoc` is narrow, but rejecting unsupported `content` before creation is also a valid safer fix if create should remain title-only.
