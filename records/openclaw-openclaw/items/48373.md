## Summary

Keep open: current `main` and the latest shipped tag still return from the Feishu `create` branch after `createDoc` without writing or rejecting a supplied `content` value, and the linked closing PR is still open and unmerged.

## What This Changes

The issue requests that `feishu_doc` stop silently dropping `content` passed with `action: "create"`, either by creating and populating the document or rejecting the unsupported parameter clearly.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/48377 with real Feishu proof and a clear partial-write failure contract; if maintainers reject create-time content, reject `content` on create with a clear error instead of silently dropping it.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and returns after `createDoc` without writing or rejecting `content`.

## Solution Assessment

Unclear at the merge-decision level. Reusing `writeDoc` after `createDoc` is the narrow maintainable path if create-time content is supported, but maintainers still need either that repaired/proven contract or an explicit validation error for `content` on create.
