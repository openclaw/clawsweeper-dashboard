## Summary

Keep open: current main and latest release still do not write or reject create-time `content`, and the open closing-reference implementation PR is not merged, so this issue remains necessary.

## What This Changes

Request that `feishu_doc` create stop silently ignoring supplied document content, either by writing it during create or by clearly rejecting the unsupported parameter.

## Best Possible Solution

Land a refreshed Feishu plugin fix that either writes create-time markdown through the existing write path with explicit partial-failure handling and live proof, or rejects `content` on create with a clear error.

## Reproduction Assessment

Yes, at source level. A `feishu_doc` create call with `title` and `content` can reach the create branch, which calls only `createDoc`; no live Feishu call was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. Create-plus-write through the existing `writeDoc` path is narrow, but a final fix should preserve created metadata on write failure or explicitly reject create-time content.
