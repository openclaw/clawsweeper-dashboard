## Summary

Keep open: current `main` and the latest shipped tag still return from the Feishu `create` branch after `createDoc` without writing or rejecting a supplied `content` value, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports that `feishu_doc` accepts `content` with `action: "create"` but creates an empty document instead of writing or rejecting the content.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/48377 with real Feishu proof and a clear partial-write failure contract; if maintainers reject create-time content, reject `content` on create with a clear error instead of silently dropping it.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and returns after `createDoc` without writing or rejecting `content`.

## Solution Assessment

Yes. Reusing the existing `writeDoc` path after `createDoc` is the narrow maintainable implementation if create-time content is supported; explicit validation that rejects `content` on create is the safer fallback if maintainers do not want that API behavior.
