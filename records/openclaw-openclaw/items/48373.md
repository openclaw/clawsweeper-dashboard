## Summary

Keep open: current main and the latest release still route `feishu_doc` create directly to `createDoc` without reading `content`, and the open fix PR at https://github.com/openclaw/openclaw/pull/48377 references this issue as its closing bug fix.

## What This Changes

The issue asks for `feishu_doc` create to stop silently dropping `content`, either by writing it during create or rejecting it clearly.

## Best Possible Solution

Land or replace the open Feishu plugin fix so create either writes supplied content through the existing write path or rejects it clearly, then reconcile the contrary docs-only PR and close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and ignores the supplied content instead of writing or rejecting it.

## Solution Assessment

Yes, create-plus-write is a maintainable fix if `content` remains visible to tool callers; the safer fallback is an explicit validation error, but silent success with dropped content should not remain.
