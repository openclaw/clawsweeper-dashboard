## What This Changes

The issue reports that `feishu_doc` create accepts a `content` field but creates an empty document instead of writing or rejecting that content.

## Best Possible Solution

Land a repaired Feishu plugin change that writes create-time content through `writeDoc` while preserving recovery metadata, or reject `content` on create with a clear error.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass `action: "create"`, `title`, and `content`; the dispatcher reaches the create branch and returns after `createDoc` without writing or rejecting the content.

## Solution Assessment

Yes. Create-plus-write through the existing `writeDoc` path is a narrow maintainable fix if create-time content remains accepted; explicit validation that rejects `content` on create is the safer fallback.
