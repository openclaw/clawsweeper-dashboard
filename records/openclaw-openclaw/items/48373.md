Keep open: current main still returns from the Feishu `create` action without writing or rejecting supplied `content`, and an open closing PR is already the implementation path.

## What This Changes

The issue requests that Feishu document creation stop silently dropping supplied markdown content, either by writing it during create or rejecting it clearly.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/48377 so create-time content is written through the existing markdown path or rejected clearly instead of being silently dropped.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass `action: "create"`, `title`, and `content`; the dispatcher reaches the create branch and returns after `createDoc` without writing or rejecting the content.

## Solution Assessment

Yes. Create-plus-write through the existing `writeDoc` path is a narrow maintainable fix if create-time content remains accepted; explicit validation that rejects `content` on create is the safer fallback.
