## Summary

Keep open: current main still has a source-visible silent-drop path for `content` on `feishu_doc` create, and open PR #48377 already targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `feishu_doc` create either populate provided markdown content during document creation or stop accepting it silently.

## Best Possible Solution

Land or replace #48377 with a current-main Feishu plugin fix that chains `createDoc` and `writeDoc` when `content` is provided, or rejects `content` clearly, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and ignores `p.content` instead of writing or rejecting it.

## Solution Assessment

Yes, create-plus-write is a maintainable fix if `content` remains visible to tool callers; the safer fallback is an explicit validation error, but silent success with dropped content should not remain.
