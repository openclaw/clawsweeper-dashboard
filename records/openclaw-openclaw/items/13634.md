## Summary

Keep open. Current main handles workspace bootstrap files through shared Project Context, but the requested Slack-wide `systemPrompt` config surface is still absent from Slack types, strict schema, docs, and runtime prompt resolution; prior Slack prompt history also shows a similar exposed API was removed, so the remaining work needs maintainer/product approval before implementation.

## What This Changes

The issue requests Slack account/global `systemPrompt` inheritance and Slack workspace file context parity with Telegram.

## Best Possible Solution

Approve the Slack account/global prompt contract, then implement it across Slack config types, schema/metadata, docs, and normal plus slash runtime prompt paths while leaving workspace files on shared Project Context.

## Reproduction Assessment

Yes for the source-level gap, but not as a bug reproduction. Current main shows workspace bootstrap is shared while Slack only types, validates, documents, and consumes `systemPrompt` under per-channel entries, not `channels.slack.systemPrompt`.

## Solution Assessment

Unclear without Slack owner/API confirmation. The need is valid, but the current evidence shows a similar exposed prompt field was removed, so the maintainable path is an approved public config design before implementation.
