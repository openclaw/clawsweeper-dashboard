## Summary

Keep open. Current main already covers the workspace-file half through shared Project Context/bootstrap injection, but the Slack account/global systemPrompt requested here is still absent from the typed config, strict schema, docs, and runtime prompt resolution paths; the related Slack file-access PR is about media hydration, not this prompt/config surface.

## What This Changes

The issue requests Slack account/global systemPrompt support and Slack parity with Telegram-style workspace context behavior.

## Best Possible Solution

Add a documented Slack account/global prompt field in the Slack plugin config that inherits through base-to-account resolution, stacks predictably with per-room overrides, and relies on shared Project Context for workspace files.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap: Slack accepts systemPrompt only under per-channel entries, while the account/global type, strict schema, runtime resolver, docs, and tests do not expose channels.slack.systemPrompt.

## Solution Assessment

Yes. The narrow maintainable fix is Slack plugin config/runtime inheritance for account/global prompts; adding Slack-specific workspace-file injection would duplicate the shared Project Context mechanism that already handles AGENTS.md, SOUL.md, and related files.
