## Summary

Keep open. Current main already handles standard workspace files through shared Project Context, but the Slack-wide `channels.slack.systemPrompt` request is still missing from the typed config, strict schema, account surface, Slack prompt resolution, runtime prompt composition, and docs.

## What This Changes

Add a Slack-wide/account-level `systemPrompt` that applies across Slack conversations, while relying on shared Project Context for workspace file injection.

## Best Possible Solution

Keep workspace-file loading in shared Project Context, and add documented Slack account/global prompt inheritance that composes consistently across room, DM/MPIM, and native slash-command paths.

## Reproduction Assessment

Yes. Source inspection is enough: `channels.slack.systemPrompt` has no account/schema key, and Slack prompt composition only reads per-channel `channelConfig.systemPrompt`.

## Solution Assessment

Yes. The narrow maintainable fix is Slack account/global prompt inheritance; adding a separate Slack-specific workspace-file injector would duplicate the shared Project Context mechanism.
