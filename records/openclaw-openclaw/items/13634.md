## Summary

Keep open. Current main covers workspace-file injection through shared Project Context, but the Slack account/global `systemPrompt` config surface is still absent and prior related prompt work was intentionally closed as the wrong API shape.

## What This Changes

Request Slack account/global `systemPrompt` inheritance plus Slack workspace bootstrap file injection parity with Telegram.

## Best Possible Solution

Add a documented Slack account/global prompt field with explicit inheritance and stacking semantics, while keeping AGENTS/SOUL/MEMORY injection on shared Project Context.

## Reproduction Assessment

Yes for the missing feature surface: source inspection on current main shows Slack accepts `systemPrompt` only under `channels.slack.channels.*`, while account/global types, schema, docs, and prompt paths do not expose `channels.slack.systemPrompt`. Live Slack reproduction is not required to prove this config/API gap.

## Solution Assessment

Partly. Shared Project Context is already the right workspace-file solution, but the remaining global Slack prompt needs maintainer-approved config/API semantics rather than reapplying the previously closed local prompt-stacking patch.
