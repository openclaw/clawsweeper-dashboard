## Summary

The evidence is complete: this remains a valid, partly addressed feature request. Shared Project Context already injects workspace files, but current main still lacks a Slack account-wide prompt spanning rooms and direct peers.

## What This Changes

Requests Slack-wide system-prompt inheritance and automatic workspace-context injection comparable to Telegram.

## Best Possible Solution

Keep workspace bootstrap in generic Project Context, then add one documented Slack account-level prompt that composes predictably with room overrides through shared logic used by normal messages, DMs, MPIMs, and native slash commands.

## Reproduction Assessment

Yes. Current source and tests show room-level and wildcard prompts exist, while direct messages do not inherit them and the Slack account config has no global prompt field.

## Solution Assessment

Yes in narrowed form: Slack-specific workspace-file loading would duplicate generic Project Context, but a single account-level Slack prompt with explicit composition semantics addresses the remaining gap cleanly.
