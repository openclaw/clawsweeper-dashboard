## Summary

Keep open. Current main already covers the workspace-file injection half through shared Project Context, but the requested Slack global/account `systemPrompt` config surface is still absent from types, strict schema, runtime prompt resolution, and docs.

## What This Changes

Add Slack workspace-file parity and a `channels.slack.systemPrompt` setting that applies across Slack conversations unless overridden.

## Best Possible Solution

Keep workspace files on the shared Project Context path, and add a documented Slack account/global prompt field that validates, inherits predictably, and reaches messages, DMs, MPIMs, rooms, and native slash commands.

## Reproduction Assessment

Yes. A current-main config using `channels.slack.systemPrompt` has no typed or strict-schema home, and the Slack runtime prompt path only reads `channelConfig.systemPrompt` from per-channel entries.

## Solution Assessment

Partly. Adding Slack account/global prompt inheritance is the narrow maintainable fix; adding a separate Slack-specific workspace-file injector would duplicate the generic Project Context mechanism already in current main.
