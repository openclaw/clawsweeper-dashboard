## Summary

Keep open. Current main covers the workspace-file part through shared Project Context, but the Slack-wide/account `systemPrompt` surface is still absent from typed config, strict schema, account surface fields, and both normal-message and slash-command prompt composition.

## What This Changes

Add Slack-wide/account-level `systemPrompt` inheritance and workspace-context parity with Telegram, without repeating per-channel Slack prompt config.

## Best Possible Solution

Keep workspace-file loading in shared Project Context, and add a documented Slack account/global prompt field that composes predictably with per-channel overrides across Slack surfaces.

## Reproduction Assessment

Yes. Source inspection on current main shows `systemPrompt` is accepted only under `channels.slack.channels.*`, and Slack prompt composition only reads `channelConfig.systemPrompt`.

## Solution Assessment

Yes. The narrow maintainable solution is Slack account/global prompt inheritance; adding a separate Slack workspace-file injector would duplicate the existing Project Context mechanism.
