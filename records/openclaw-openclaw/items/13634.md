The issue requests Slack parity with Telegram by adding a Slack-wide `systemPrompt` option and automatic workspace-file context injection for Slack sessions.

## Best Possible Solution

Define the supported Slack prompt inheritance contract first; if accepted, implement it through typed config, strict schema, generated metadata, docs, normal-message handling, and native slash-command handling while keeping shared Project Context as the workspace-file mechanism.

## Reproduction Assessment

## Solution Assessment

Unclear. The requested key is plausible, but current main already has shared Project Context plus a per-channel `*` wildcard fallback, and a prior global/DM prompt PR was closed as the wrong shape, so maintainers should choose the public inheritance contract before implementation.
