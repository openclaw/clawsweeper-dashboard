## Summary

Keep open for explicit maintainer product judgment. Shared Project Context already supplies Slack sessions with workspace bootstrap files, but current main intentionally exposes `systemPrompt` only for Slack rooms; the merged scoped-prompt work rejected a new account/global prompt API, while a later maintainer review said the broader inheritance request should remain open.

## What This Changes

Add a Slack account-wide system prompt inherited across channels and direct-message paths while continuing to use shared Project Context for workspace files.

## Best Possible Solution

Keep shared Project Context as the only workspace-file injection path; if maintainers approve broader Slack prompt inheritance, define a single account-level field with documented precedence beneath exact-room and room-wildcard overrides and apply it consistently to rooms, DMs, MPIMs, interactions, and native slash commands.

## Reproduction Assessment

## Solution Assessment

Unclear pending maintainer judgment. An account-level prompt is the narrowest mechanism for DM-inclusive inheritance, but the merged scoped-prompt decision intentionally rejected that API surface, so its permanent scope and precedence must be decided before implementation.
