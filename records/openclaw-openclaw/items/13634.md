## Summary

Keep open. Current main already covers workspace-file bootstrap through shared Project Context, but Slack still exposes `systemPrompt` only on per-channel entries; the top-level/account Slack config, strict schema, docs, and normal/slash runtime prompt paths do not implement `channels.slack.systemPrompt`.

## What This Changes

Request Slack to support an account/global `systemPrompt` and workspace bootstrap context parity so users do not repeat `file:` prompts per Slack channel.

## Best Possible Solution

Add a documented Slack account/global prompt field with explicit inheritance and stacking semantics, while keeping AGENTS/SOUL/MEMORY injection on the shared Project Context mechanism.

## Reproduction Assessment

Yes. Source inspection on current main shows Slack accepts `systemPrompt` only under `channels.slack.channels.*`; the account/global type, strict schema, docs, and prompt composition path do not expose `channels.slack.systemPrompt`.

## Solution Assessment

Partly. Shared Project Context is already the right solution for workspace files, but the global Slack prompt half needs a maintainer-approved config/API design rather than reapplying the previously closed local prompt-stacking patch.
