Keep open. Current main covers the workspace-file half through shared Project Context/bootstrap loading, but Slack still lacks the requested account/global `systemPrompt` in config types, strict schema, runtime prompt resolution, and docs.

## Best Possible Solution

Keep workspace bootstrap files on the shared Project Context path, and add a documented Slack account/global prompt field that inherits through base-to-account config and composes with per-channel overrides across normal and slash-command Slack paths.
