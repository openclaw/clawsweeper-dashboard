## Summary

Keep open. Current main covers the workspace-file half through shared Project Context, but Slack still lacks the requested account/global `systemPrompt`; prior Slack prompt PR history also shows this needs owner/product approval before implementation.

## What This Changes

Request to add Slack-wide `channels.slack.systemPrompt` inheritance and workspace-file context parity so Slack sessions do not require repeated per-channel prompt config.

## Best Possible Solution

Approve or reject the Slack account/global prompt contract explicitly; if approved, implement it through typed config, strict schema, generated metadata, docs, normal-message handling, slash-command handling, and prompt-order tests while leaving workspace files on shared Project Context.

## Reproduction Assessment

Yes for the source-level gap, but not as a bug reproduction. Current main injects workspace files through Project Context, while Slack only types, validates, documents, and consumes `systemPrompt` under per-channel entries, not `channels.slack.systemPrompt`.

## Solution Assessment

Unclear until Slack owner/product approval. Prior PR history rejected the earlier exposed global/DM prompt shape, so the maintainable solution is an approved config and prompt-stacking design rather than a local prompt patch in one Slack path.
