## Summary

Keep open. Current main already handles workspace bootstrap files through shared Project Context, but Slack still lacks a typed, strict-schema, documented account/global `systemPrompt` surface and the runtime prompt path only composes per-channel prompts.

## What This Changes

Request to add Slack-wide `systemPrompt` inheritance and Slack workspace-file context parity with Telegram-style prompt behavior.

## Best Possible Solution

Keep shared Project Context as the workspace-file mechanism, and add a documented Slack account/global prompt field with tested precedence across Slack message surfaces.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap clearly: Slack accepts prompt text only under per-channel config and composes only `channelConfig.systemPrompt`, while workspace bootstrap files are already injected through shared Project Context.

## Solution Assessment

Yes. The narrow maintainable fix is Slack account/global prompt inheritance; adding a separate Slack workspace-file injector would duplicate the existing bootstrap context system.
