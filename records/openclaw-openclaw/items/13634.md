## Summary

Keep this issue open for explicit maintainer product judgment. Current main already supplies Slack sessions with shared workspace bootstrap context, but Slack prompt overrides remain intentionally room-scoped; no open canonical replacement owns the requested account-wide inheritance.

## What This Changes

Add a Slack account/global `systemPrompt` that applies across conversations unless overridden, while matching Telegram-like workspace context behavior.

## Best Possible Solution

Keep shared Project Context as the sole workspace-file mechanism, then have Slack owners explicitly choose either one documented account-level prompt field with deterministic room override semantics or the existing wildcard room entry as the supported global override.

## Reproduction Assessment

Yes at source level: current types, strict schema, Slack resolver, prompt composition, docs, and merged history conclusively show that workspace context is shared while Slack prompts remain room-scoped. This is a feature-gap verification rather than a failing current-main bug reproduction.

## Solution Assessment

No, not as originally bundled: Slack-specific workspace injection would duplicate the existing generic Project Context path. The account-wide prompt portion may be appropriate, but only after maintainers decide whether a new public field is preferable to the existing wildcard-room mechanism and define precedence across all Slack entry paths.
