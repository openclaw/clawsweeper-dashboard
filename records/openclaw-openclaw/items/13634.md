## Summary

Keep open. Current main already covers workspace-file injection through shared Project Context, but the requested Slack account/global `systemPrompt` surface is still absent from types, strict schema, docs, and Slack prompt composition paths.

## What This Changes

The issue requests Slack workspace-file injection and a top-level/account `channels.slack.systemPrompt` that applies across Slack conversations unless overridden.

## Best Possible Solution

Add a documented Slack account/global prompt field with explicit inheritance and stacking semantics, while keeping AGENTS/SOUL/MEMORY injection on the shared Project Context mechanism.

## Reproduction Assessment

Yes for the missing feature surface. Source inspection on current main shows Slack accepts `systemPrompt` only under `channels.slack.channels.*`, while account/global types, schema, docs, and prompt paths do not expose `channels.slack.systemPrompt`.

## Solution Assessment

Partly. Shared Project Context is already the right workspace-file solution, but the remaining global Slack prompt needs a maintainer-approved config/API design rather than reapplying the previously closed local prompt-stacking patch.
