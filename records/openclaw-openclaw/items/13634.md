## Summary

Keep open. Current main covers the workspace-file part through shared Project Context injection, but the requested Slack account/global `systemPrompt` surface is still missing from typed config, strict schema, Slack prompt resolution, runtime prompt composition, and docs.

## What This Changes

Request to add Slack-wide `systemPrompt` support and Telegram-like workspace context behavior without repeating the same prompt in every Slack channel entry.

## Best Possible Solution

Keep workspace-file loading on shared Project Context, and add a documented Slack account/global prompt field that validates, inherits through account resolution, and composes consistently with per-channel overrides across Slack rooms, DMs, MPIM/group DMs, and native slash commands.

## Reproduction Assessment

Yes. On current main, `channels.slack.systemPrompt` has no account/schema key, and Slack prompt composition only reads `channelConfig.systemPrompt` from per-channel entries for roomish contexts.

## Solution Assessment

Yes. The narrow maintainable fix is Slack account/global prompt inheritance; adding a separate Slack-specific workspace-file injector would duplicate the shared Project Context mechanism already in place.
