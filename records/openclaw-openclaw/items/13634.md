## Summary

Keep open. Current main and v2026.6.10 already cover workspace bootstrap files through shared Project Context, but the requested Slack account/global `systemPrompt` contract is still absent from the typed config, strict schema, runtime prompt resolution, and docs; related merged Slack work deliberately avoided exposing that surface.

## What This Changes

The issue requests Slack workspace context injection parity and a Slack-wide `systemPrompt` setting so users do not repeat the same prompt file per Slack channel.

## Best Possible Solution

Keep workspace files in shared Project Context, then add a maintainer-approved Slack account/global prompt contract only if maintainers accept the public config semantics and consistent composition across all Slack conversation paths.

## Reproduction Assessment

Yes, at source level. Current main shows shared Project Context injects workspace files, while Slack account/global prompt config is absent from the account type/schema and normal-message/slash prompt paths.

## Solution Assessment

Unclear until maintainers approve the public config semantics. The maintainable shape is not Slack-specific workspace-file injection, but a documented Slack account/global prompt contract that composes with existing per-room prompts and shared Project Context.
