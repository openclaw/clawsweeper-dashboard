## Summary

Keep open. Current main covers the workspace-file side through shared Project Context injection, but the requested Slack account/global `systemPrompt` is still missing from the config type, strict schema, docs, and runtime prompt resolution path; the related Slack file-access PR is about media hydration, not this prompt/config surface.

## What This Changes

Add Slack workspace-file context parity and a Slack account/global `systemPrompt` that applies across Slack chats unless overridden.

## Best Possible Solution

Add a documented Slack account/global prompt field that inherits through base-to-account resolution, stacks predictably with per-room overrides, and keeps workspace files on the shared Project Context mechanism.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap: Slack accepts `systemPrompt` only under per-channel entries, while the account/global type, strict schema, runtime resolver, docs, and tests do not expose `channels.slack.systemPrompt`.

## Solution Assessment

Yes. The narrow maintainable solution is Slack plugin config/runtime inheritance for account/global prompts; adding Slack-specific workspace-file injection would duplicate the shared Project Context mechanism already used for AGENTS.md, SOUL.md, and related files.
