## Summary

Keep open. Current main covers workspace-file bootstrap through shared Project Context, but the requested Slack top-level/account `systemPrompt` is still absent from the typed config, strict schema, docs, and normal/slash runtime prompt paths.

## What This Changes

The issue requests Slack parity with Telegram by adding a global `channels.slack.systemPrompt` and automatic workspace file context for Slack sessions.

## Best Possible Solution

Add a documented Slack account/global prompt field with explicit inheritance and stacking semantics, while keeping AGENTS/SOUL/MEMORY injection on the shared Project Context mechanism.

## Reproduction Assessment

Yes. Source inspection on current main shows Slack accepts `systemPrompt` only under `channels.slack.channels.*`; the account/global type, strict schema, docs, and prompt composition paths do not expose `channels.slack.systemPrompt`.

## Solution Assessment

Partly. Shared Project Context is already the right workspace-file solution, but the remaining global Slack prompt needs a maintainer-approved config/API design rather than reapplying the previously closed local prompt-stacking patch.
