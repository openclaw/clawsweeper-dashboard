## Summary

Keep open. Current main already covers workspace-file loading through shared Project Context, but the requested Slack account/global `systemPrompt` surface is still absent from Slack config types, strict schema, generated metadata, runtime prompt composition, and docs.

## What This Changes

Add a Slack account/global `systemPrompt` that applies across Slack conversations without repeating the same prompt on every per-channel entry.

## Best Possible Solution

Keep workspace bootstrap on shared Project Context, and implement a documented Slack account/global prompt that inherits through account config and composes consistently with per-channel overrides across rooms, DMs, MPIMs, and slash commands.

## Reproduction Assessment

Yes for the reported missing config surface. Source inspection on current main shows no `channels.slack.systemPrompt` account/global type or schema key, and Slack prompt composition only reads per-channel `systemPrompt`.

## Solution Assessment

Yes. The narrow maintainable path is Slack account/global prompt inheritance; adding a Slack-specific workspace-file injector would duplicate the shared Project Context mechanism already present.
