## Summary

Keep open: current main covers the workspace-file half through shared Project Context injection, but Slack still lacks a typed and documented account/global `systemPrompt` contract. The existing `channels.slack.channels["*"].systemPrompt` fallback may cover room channels, but it is not the requested account-level surface and does not settle DM, MPIM, slash-command, or prompt-stacking policy.

## What This Changes

The issue requests Slack workspace-file context injection plus a Slack account/global `systemPrompt` that applies across Slack conversations unless overridden.

## Best Possible Solution

Decide the supported Slack prompt inheritance contract first; if accepted, implement it through typed config, strict schema, docs, generated config metadata, and all Slack prompt paths while keeping shared Project Context as the workspace-file mechanism.

## Reproduction Assessment

Yes for the missing requested capability: source inspection shows Slack account config/schema and runtime prompt resolution still have no account/global `systemPrompt`. A live Slack repro is not necessary for this feature/config-surface gap.

## Solution Assessment

Unclear. The requested key is plausible, but current code already has shared Project Context and a per-room `*` fallback, so maintainers should choose the public Slack prompt inheritance contract before implementation.
