## Summary

Keep open. Current main covers standard workspace file injection through the shared Project Context path, but the requested Slack account/global `systemPrompt` surface is still missing from typed config, strict schema, runtime prompt resolution, and docs.

## What This Changes

The issue requests Slack parity for automatic workspace-file context plus a global `channels.slack.systemPrompt` that applies across Slack conversations without repeating per-channel config.

## Best Possible Solution

Keep workspace-file loading on the shared Project Context mechanism, and add a documented Slack account/global prompt field that validates in strict config, inherits through root-to-account resolution, stacks predictably with per-channel overrides, and reaches Slack rooms, private channels, DMs, MPIM/group DMs, and native slash commands.

## Reproduction Assessment

Yes. A current-main config using `channels.slack.systemPrompt` has no strict schema key, and the Slack runtime prompt path only reads `channelConfig.systemPrompt` from per-channel entries.

## Solution Assessment

Partly. Adding Slack account/global prompt inheritance is the narrow maintainable fix; adding a separate Slack-specific workspace-file injector would duplicate the generic Project Context behavior that already handles AGENTS.md, SOUL.md, and related files.
