## Summary

Keep open. Current main and the latest release still do not provide the requested automatic sensitive-data output sanitizer; existing logging redaction, transcript redaction, internal-scaffolding cleanup, and plugin hooks are useful partial surfaces but do not settle the product/security decision for default output filtering.

## What This Changes

The issue requests an automatic, configurable sanitizer for agent outputs, tool results, and final responses to prevent accidental disclosure of API keys, tokens, passwords, and PII.

## Best Possible Solution

Keep this open for maintainer/security design, land narrow proven leak fixes separately, and choose whether the long-term sanitizer belongs in core defaults, a bundled plugin, or documented ClawHub/plugin policy using the existing hook seams.

## Reproduction Assessment

Yes, at source level for the capability gap. Current main and v2026.6.6 expose logging/transcript redaction and plugin rewrite hooks, but no automatic outputSanitization-style config or default output filter.

## Solution Assessment

Unclear as a single implementation path. The requested middleware is plausible, but the best maintainable solution needs maintainer/security choice between core defaults, plugin-based policy, and narrower tool-owned redaction fixes.
