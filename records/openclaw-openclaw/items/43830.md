## Risks / Open Questions

- A delivery-only sanitizer could make channel output look safe after sensitive data has already reached an external model provider.
- Broad regex redaction can corrupt legitimate code, quoted evidence, or operational output while still missing unknown secret formats.
- A reversible provider-egress design would create sensitive session state and must avoid cross-turn, cross-tool, cross-channel, and cross-session mis-association.
- New defaults, custom regexes, reporting, and per-agent modes are security-sensitive config surfaces that need maintainer decisions before implementation.
