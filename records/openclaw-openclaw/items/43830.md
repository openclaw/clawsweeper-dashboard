## Summary

Keep open. Current main and v2026.6.10 have log, transcript, tool-payload redaction and final-delivery scaffolding stripping, but not an automatic sensitive-value sanitizer for all agent-visible output or a settled output-sanitization policy surface.

## What This Changes

Add automatic sanitization of sensitive values in agent, tool, and final outputs, with configurable modes, scopes, patterns, exceptions, and redaction reporting.

## Best Possible Solution

Keep this open for a maintainer-approved security design that defines output boundaries, default behavior, core-versus-plugin ownership, config/upgrade policy, and validation while reusing the existing redaction helper where appropriate.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main lacks the requested automatic output-sanitization policy, but no live leak reproduction was run.

## Solution Assessment

No. The proposed middleware/config direction is plausible, but the best fix first needs maintainer/security approval for scope, defaults, false-positive behavior, and whether this belongs in core or a plugin.
