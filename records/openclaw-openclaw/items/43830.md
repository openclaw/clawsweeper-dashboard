## Summary

Keep open: current main still has scoped log/transcript redaction, tool/UI redaction, and internal-scaffolding sanitizers, but it does not provide a universal sensitive-data sanitizer for provider-bound content or all agent outputs. The existing channel hook can support delivery-only filtering, while model-boundary sanitization still needs maintainer security and product direction.

## What This Changes

Request to add configurable automatic sanitization/redaction for sensitive data in agent outputs, tool results, and responses, including modes, scopes, reporting, patterns, and per-agent overrides.

## Best Possible Solution

Separate channel delivery and provider-egress protection: use the existing message_sending plugin seam for delivery-only redaction, and add a narrow model-boundary guardrail or hook only after maintainers define the threat model, state ownership, defaults, and failure behavior.

## Reproduction Assessment

Yes, source-reproducible: current main has scoped logging/transcript redaction and internal-scaffolding sanitizers, but no outputSanitization/securityGuardrail/privacy provider-egress config or universal output filter. No live leak transcript was run in this read-only review.

## Solution Assessment

No. The proposed single middleware/config block mixes channel redaction, provider egress, tool transformation, reversible session state, reporting, and regex policy; the maintainable path is to split those boundaries and design the model-boundary protection explicitly.

## Risks / Open Questions

- A global regex sanitizer can silently corrupt legitimate code, identifiers, quoted evidence, or operational output while still missing unknown secret formats.
- Provider-egress replacement plus inbound restoration changes tool arguments and multi-turn state; incorrect mapping could mis-associate secrets, expose originals, or execute tools with altered values.
- Channel-delivery filtering can make displayed output appear safe even though the sensitive content may already have reached an external model provider.
- User-defined regexes, per-agent modes, reporting, and defaults add a security-sensitive config surface with ReDoS, upgrade, and operator-expectation implications.
