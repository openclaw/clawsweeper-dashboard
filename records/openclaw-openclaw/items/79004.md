## Summary

Keep open: current main imports bridge artifacts and refreshes Memory Wiki indexes, but it does not implement an automated agent synthesis path from new sources into syntheses, entities, or concepts. This is a feature/product gap, not conservative cleanup closure.

## What This Changes

Request automated Memory Wiki synthesis/entity/concept generation from newly imported bridge sources after memory-core dreaming runs.

## Best Possible Solution

Design an opt-in memory-wiki-owned synthesis automation flow that runs after bridge import or dreaming, uses existing wiki search/get/apply primitives, preserves source provenance, and exposes explicit privacy/cost/approval controls.

Not applicable as a strict bug reproduction: this is a requested automation feature. Source inspection confirms bridge imports and index compilation exist today, while structured synthesis still depends on explicit `wiki_apply`/`create_synthesis` calls.

## Solution Assessment

Unclear until maintainers choose the product shape. A cron, hook, or import-integrated agent synthesis flow could work, but the maintainable solution needs explicit policy around autonomy, privacy, provenance, and resource use.
