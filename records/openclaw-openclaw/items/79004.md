## Summary

Keep open: current main and v2026.5.28 still import bridge artifacts as source pages and refresh deterministic indexes, but they do not run an automated synthesis step. `wiki_apply` remains the separate write path for syntheses/metadata, so this is a real feature/product gap rather than stale cleanup.

## What This Changes

Add an automated path from bridge-imported Memory Wiki source pages to model-generated syntheses, entities, or concepts so daily memory artifacts become structured wiki knowledge without manual `wiki_apply`.

## Best Possible Solution

Add an explicitly opt-in Memory Wiki synthesis job or hook in the `memory-wiki` plugin that consumes newly imported bridge sources, writes provenance-rich mutations through the existing wiki apply model, and preserves manual control and auditability.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: bridge import writes source pages, source sync refreshes deterministic indexes, and synthesis is only reachable through the separate manual `wiki_apply`/`wiki.apply` mutation path.

## Solution Assessment

No, closing would not be the best handling because current main has not implemented the requested automation. The maintainable direction is a product-approved, opt-in `memory-wiki` synthesis workflow rather than silently folding model-written synthesis into every bridge import.
