## Summary

Keep open: current main imports bridge artifacts and refreshes wiki indexes, but it does not implement an automated agent synthesis step from new sources into syntheses/entities/concepts. This is a real feature gap that needs maintainer product design rather than cleanup closure.

## What This Changes

Add an automated path from memory-wiki bridge imports to structured syntheses/entities/concepts so daily dreaming can populate the wiki without manual `wiki_apply`.

## Best Possible Solution

Design an opt-in memory-wiki-owned synthesis automation path that triggers after bridge imports or dreaming, uses existing wiki search/get/apply primitives, records source provenance, and has explicit privacy/cost/approval controls.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a requested automation feature. Source inspection confirms current main imports sources and compiles indexes, while structured synthesis still depends on explicit `wiki_apply` calls.

## Solution Assessment

Unclear until maintainers choose the product shape. A cron, hook, or import-integrated agent synthesis flow could work, but the narrow maintainable solution needs policy decisions around autonomy, privacy, provenance, and resource use.
