## Summary

Keep open: current main imports Memory Wiki bridge artifacts and refreshes deterministic indexes, but it still does not automate agent synthesis from new sources into syntheses, entities, or concepts. This is a real feature gap that needs product/privacy/cost design rather than cleanup closure.

## What This Changes

Request automated Memory Wiki synthesis after bridge imports or dreaming so raw sources become structured entities, concepts, and syntheses without manual `wiki_apply`.

## Best Possible Solution

Design an opt-in Memory Wiki synthesis automation path that runs after bridge import or dreaming, uses existing wiki search/get/apply primitives, records source provenance, and exposes explicit privacy, cost, and approval controls.

## Solution Assessment
