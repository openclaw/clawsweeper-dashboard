## Summary

Keep open: current main still imports Memory Wiki bridge artifacts as source pages and refreshes deterministic indexes, but it does not provide an automated agent synthesis path from new sources into syntheses, entities, or concepts. This is a valid feature/product gap rather than cleanup-close material.

## What This Changes

The issue requests an automated Memory Wiki synthesis path from bridge-imported memory-core artifacts into structured syntheses, entities, and concepts.

## Best Possible Solution

Design an opt-in Memory Wiki-owned synthesis automation flow that triggers after bridge imports or dreaming, uses existing wiki search/get/apply primitives, records source provenance, and exposes explicit privacy, cost, and approval controls.
