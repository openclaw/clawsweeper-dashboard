## Summary

## What This Changes

Request an automated Memory Wiki step that turns newly bridged memory-core sources into syntheses, entities, or concepts without manually calling `wiki_apply`.

## Best Possible Solution

Keep bundled Memory Wiki focused on bridge import, compile, and explicit mutation tools, and publish the autonomous source-to-wiki synthesis loop as a ClawHub plugin or skill that uses existing hooks and `wiki_apply`.

## Reproduction Assessment

Yes. Source inspection shows bridge import creates source pages and refreshes indexes, while synthesis/entity/concept creation remains a separate explicit `wiki_apply` mutation path.

## Solution Assessment

No for OpenClaw core. The maintainable solution is an external ClawHub plugin or skill using existing hooks and wiki tools, unless maintainers later choose a bundled high-trust automatic writer with explicit product/security policy.
