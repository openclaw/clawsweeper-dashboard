## What This Changes

The issue reports a Feishu channel regression where streamed replies in OpenClaw 2026.6.1 show only the newest token instead of progressively accumulated text.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main still reduces ordinary non-error streaming final chunks to the latest payload and assigns that value as the whole Feishu streaming-card buffer.

## Solution Assessment
