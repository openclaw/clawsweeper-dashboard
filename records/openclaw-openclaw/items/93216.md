## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid re-running message_sending on suffix chunks:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:431-436`
  - body: After `beforeFinalizePreviewDelivery` runs `message_sending` for the full streamed answer, `sendRemainingChunks` still sends each suffix through `sendPayload`. In Telegram that reaches `deliverReplies`, which runs `message_sending` again per payload, so long replies can be approved once as a whole and then rewritten or cancelled again by tail chunk.
  - confidence: 0.9
- **[P1] Track retained preview messages before final gating:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:373-376`
  - body: When `activeChunkIndex > 0`, earlier preview chunks were retained as separate Telegram messages, but the new gate only receives the current `messageId`. A cancel deletes only that current message and this rewrite branch is skipped, leaving prior visible chunks unchanged after a terminal hook decision.
  - confidence: 0.88
- **[P2] Make the delivery opt-out an explicit private or SDK contract:** `src/auto-reply/dispatch.ts:59-69`
  - body: `dispatchInboundMessageWith*` are exported through `openclaw/plugin-sdk/reply-runtime`, but this patch reads `runsMessageSendingAtDelivery` through a private cast while Telegram supplies it through local casts. If this is a channel contract it needs typed SDK/docs coverage; otherwise keep it behind a private core-to-Telegram adapter.
  - confidence: 0.82
