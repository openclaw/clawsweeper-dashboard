## Summary

Keep this PR open for maintainer review: the patch is a focused candidate fix with sufficient QMD proof and no discrete line-level correctness finding, but it expands the memory runtime contract and changes behavior behind an existing close-named facade, so API ownership needs maintainer acceptance before merge.

## What This Changes

The PR adds an optional scoped memory-search resource-release hook, makes active-memory cleanup prefer that hook before falling back to full manager close, wires memory-core through it, and adds focused regressions plus the Plugin SDK API baseline hash update.

## Best Possible Solution

Land one owner-approved active-memory resource-release contract that preserves explicit full scoped close for real close callers and prevents request-scoped active-memory timeout cleanup from tearing down shared QMD managers.

## Solution Assessment

Yes, conditionally. Separating active-memory resource release from full scoped close is narrower than weakening closeMemorySearchManager globally, but the optional runtime hook and facade semantics need maintainer acceptance before merge.
