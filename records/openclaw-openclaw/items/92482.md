## What This Changes

Reject negative Matrix CLI values by requiring `matrix verify self --timeout-ms` to be positive and `matrix account add --initial-sync-limit` to be non-negative.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92483 or an equivalent Matrix plugin-local validation patch, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Current-main source shows signed negative integers are accepted by the shared Matrix CLI parser and both reported command paths call it without lower-bound validation; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is plugin-local CLI boundary validation with distinct minimums for the two options, and the open canonical PR already implements that shape with focused coverage and proof.
