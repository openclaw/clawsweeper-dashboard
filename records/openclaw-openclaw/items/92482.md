## What This Changes

Validate Matrix CLI numeric options so verification timeouts must be positive and initial sync limits cannot be negative.

## Best Possible Solution

Enforce explicit minimum bounds at the existing Matrix CLI parsing boundary and add focused tests proving invalid input is rejected before verification or account setup actions execute.

Yes. CLI-boundary lower-bound validation using the existing strict integer parser is the narrowest maintainable fix and avoids changing downstream Matrix runtime semantics.
