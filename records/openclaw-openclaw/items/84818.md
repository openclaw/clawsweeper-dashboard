## Summary

## What This Changes

The PR adds precomputed CLI help fast paths for `secrets --help` and `nodes --help`, extends startup metadata generation, and lazy-loads secrets action runtime modules.

## Best Possible Solution

Keep the precomputed help fast path, but make the nodes metadata invalidation match the real help-rendering inputs before merge.

## Reproduction Assessment

No live reproduction was run because this is a read-only review, but the failure mode is source-reproducible: the PR renders real `nodes --help` while hashing only part of the bundled canvas CLI input.

## Solution Assessment

No; the fast-path approach is maintainable, but the nodes metadata cache key needs to cover the actual help-rendering inputs or use a metadata-only render path consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Include real canvas CLI sources in the nodes signature:** `scripts/write-cli-startup-metadata.ts:117-121`
  - body: `writeCliStartupMetadata` now reuses an existing `cli-startup-metadata.json` when the stored signatures match, but `nodesHelpText` is rendered by running the real `openclaw nodes --help` path. That path eagerly registers the bundled canvas node CLI from `extensions/canvas/index.ts` / `extensions/canvas/src/cli.ts`, while this signature only hashes `extensions/canvas/cli-metadata.ts`; changing the actual canvas command help can therefore leave stale precomputed `nodes --help` in incremental builds. Please include the real node CLI registration sources in the signature, or render from the metadata-only descriptor path consistently.
  - confidence: 0.83
