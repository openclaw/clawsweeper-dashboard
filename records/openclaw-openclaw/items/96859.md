## What This Changes

The PR adds a separate Codex app-server `userMcpServersEnabled` gate for wildcard or server-scoped MCP glob allow lists and threads it from run startup into thread lifecycle params.

## Best Possible Solution

Keep the separate MCP-vs-code-mode gate, but pass enough allow-list context to project only the matching user MCP servers/tools and cover separator-containing safe server names with focused tests plus real app-server proof.

## Reproduction Assessment

Yes, at source level: current main ties `userMcpServersEnabled` to the native tool-surface gate, so a restricted `toolsAllow` like `opik__*` does not project user MCP servers, while the PR's new gate would project all eligible user servers instead of just the matched one.

## Solution Assessment

No: separating MCP attachment from native code mode is the right direction, but the implementation needs a scoped user-MCP config projection rather than a boolean that enables the whole user MCP map.

- **[P1] Limit the native MCP projection to the allowed server:** `extensions/codex/src/app-server/run-attempt.ts:681-685`
  - body: With two configured user MCP servers, a run that sets `toolsAllow` to `['opik__*']` now sets `userMcpServersEnabled` to true and `startOrResumeThread` builds the Codex `mcp_servers` patch from every eligible `cfg.mcp.servers` entry. Upstream Codex starts every enabled server in that patch and only applies per-server `enabled_tools`/`disabled_tools`, so this widens a one-server allow list into all user MCP servers. Please pass the allow-list selection into the projection and include only matching servers/tools before enabling native user MCP servers.
  - confidence: 0.9
- **[P2] Handle safe server names that contain the separator:** `extensions/codex/src/app-server/dynamic-tool-build.ts:954-955`
  - body: This predicate uses the first `__`, so a valid safe server prefix like `my__server` will make `my__server__*` fail detection even though current tests already cover exact MCP allow lists with that server-name shape. Use the final separator or an explicit suffix check and cover this glob case so those servers can use the same narrow allow-list path.
  - confidence: 0.88
