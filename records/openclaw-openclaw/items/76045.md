## Summary

Keep open. Current `main` still reproduces the Feishu `message` tool presentation gap, and open PR #76055 is now the paired implementation candidate rather than a reason to close before review and merge.

## What This Changes

The issue reports that Feishu `message` tool presentations with button blocks are delivered as plain fallback text instead of native interactive-card buttons.

## Best Possible Solution

Review and land PR #76055 or a narrow replacement that moves Feishu `send` and `thread-reply` presentation cards onto the native card/button renderer while preserving media, comment-thread, text fallback, and `ocf1` callback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu `message` tool `send` with `presentation.blocks[].type === "buttons"`; current `main` dispatches to the Feishu action handler and the existing test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin action path by sharing or extracting the existing native card renderer, not by adding a core special case or a Feishu-specific shared message field.
