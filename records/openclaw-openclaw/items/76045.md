## Summary

Keep open. Current main still renders Feishu message-tool presentation buttons through the Feishu action fallback card instead of native interactive buttons, and open same-author PR #76055 is already the paired implementation candidate.

## What This Changes

The issue asks Feishu `message` tool presentations with buttons/selects to render as native interactive cards instead of plain text fallback.

## Best Possible Solution

Review and land PR #76055 or a narrow replacement that shares a lightweight Feishu native presentation-card renderer between message actions and outbound payloads while preserving mixed text, media/thread behavior, fallback text, and `ocf1` callbacks.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu `message` tool `send` with `presentation.blocks[].type === "buttons"`; current main dispatches to the Feishu action handler and the existing test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin action card builder by sharing or extracting the existing outbound native card element renderer; the open PR is the right coordination point but still needs review before merge.
