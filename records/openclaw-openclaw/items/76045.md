## Summary

Keep open. Current main has a working Feishu native-card renderer on the lower-level outbound payload path, but the Feishu `message` tool `send` action still takes a separate plugin action path that turns `presentation` buttons into markdown fallback labels before calling `sendCardFeishu`.

## What This Changes

The issue reports that Feishu `message` tool sends with `presentation` buttons degrade to text/fallback output instead of clickable Feishu interactive cards.

## Best Possible Solution

Make the Feishu plugin-owned `send` and likely `thread-reply` message-action path reuse the same native card/button renderer and `ocf1` callback envelope used by `feishuOutbound.sendPayload`, while retaining existing safe fallbacks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu `message` tool `send` with a `presentation.blocks[].type === "buttons"` payload: current main dispatches to `feishuPlugin.actions.handleAction`, whose test expects the button to become markdown text `- Run help`.

## Solution Assessment

Yes. The narrow maintainable fix is in the Feishu plugin action path, preferably by sharing the existing outbound card builder rather than adding core special cases or a Feishu-specific message-tool schema.
