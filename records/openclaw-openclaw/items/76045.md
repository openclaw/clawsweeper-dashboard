## Summary

Keep open. Current main still routes Feishu message-tool `presentation` buttons through a fallback-only card builder, and open same-author PR #76055 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Render Feishu message-tool `presentation` button blocks as native interactive card buttons instead of markdown/plain-text fallback labels.

## Best Possible Solution

Keep the issue open until PR #76055 or a narrow replacement shares the existing Feishu native presentation-card renderer with the message-action path while preserving fallback text, media/thread behavior, and `ocf1` callbacks.

## Reproduction Assessment

Yes. A source-level reproduction is `message action=send` on Feishu with `presentation.blocks[].type === "buttons"`; current main dispatches to the Feishu action handler and the existing test expects markdown fallback `- Run help` instead of a native button.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin by reusing or extracting the existing native card element renderer for the message-action path, with PR #76055 as the current coordination point.
