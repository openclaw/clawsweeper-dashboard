## Summary

Keep open. Current main still reproduces the Feishu message-tool presentation gap; open PR #76055 targets the same path but is not merged and already has review blockers, so this issue should stay open until a corrected implementation lands.

## What This Changes

The issue reports that Feishu message-tool sends with presentation buttons render as plain fallback text instead of native Feishu interactive-card buttons.

## Best Possible Solution

Review and land PR #76055 or a narrow replacement that shares a lightweight Feishu presentation-card renderer between message actions and outbound payloads while preserving mixed text, media/thread behavior, fallback text, and ocf1 callbacks.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu message-tool send with presentation.blocks[].type === "buttons"; current main dispatches to the Feishu action handler and the existing test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin action path by sharing/extracting the existing native card renderer, not by adding a core special case or a Feishu-specific shared field; the open PR currently needs adjustment before merge.
