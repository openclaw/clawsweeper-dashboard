## Summary

Keep open. Current main still has a split Feishu path: the `message` tool `send`/`thread-reply` action builds a markdown-only Feishu card, while the adjacent outbound payload path already renders presentation buttons as native Feishu button actions.

## What This Changes

Feishu message-tool sends with `presentation` buttons should render native interactive card buttons instead of plain text fallback.

## Best Possible Solution

Make the Feishu plugin-owned `send` and `thread-reply` message action path reuse the existing native card/button renderer and `ocf1` callback envelope while preserving safe text/media/comment fallbacks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu `message` tool `send` with `presentation.blocks[].type === "buttons"`; current main dispatches to the Feishu action handler and the test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin action path, preferably by sharing the existing outbound card builder rather than adding a core special case or a Feishu-specific shared tool field.
