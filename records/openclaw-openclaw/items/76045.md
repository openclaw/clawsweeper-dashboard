## Summary

Keep open. Current main still has a split path: Feishu outbound payload delivery can render native card buttons, but the Feishu `message` tool action handler builds a markdown fallback card for `presentation` buttons, matching the report.

## What This Changes

The issue reports that Feishu `message` tool sends with `presentation` buttons are delivered as plain text instead of native interactive cards.

## Best Possible Solution

Reuse the Feishu native presentation card construction for `send` and `thread-reply` message actions, keeping the change inside the Feishu plugin unless a small shared helper is needed.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu `message` tool `send` with `presentation.blocks[].type === "buttons"`; current tests expect the button label to become markdown fallback text.

## Solution Assessment

Yes. The narrow maintainable direction is to share the existing Feishu outbound native-card renderer and `ocf1` callback envelope with the Feishu message-action handler rather than adding core special cases.
