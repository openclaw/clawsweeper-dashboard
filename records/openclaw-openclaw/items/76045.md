## Summary

Keep open. Current main still has a split Feishu path: the shared message-tool send/thread-reply action builds a markdown fallback card for `presentation` buttons, while the adjacent Feishu outbound payload renderer already sends native interactive card buttons.

## What This Changes

The issue asks Feishu message-tool `presentation` button blocks to render as native interactive cards with clickable buttons instead of plain-text fallback.

## Best Possible Solution

Move the Feishu plugin-owned `send` and `thread-reply` message action path onto the existing native card/button renderer and `ocf1` callback envelope while preserving media, comment-thread, and text fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu `message` tool `send` with `presentation.blocks[].type === "buttons"`; current main dispatches to the Feishu action handler and the existing test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin action path, preferably by sharing the existing outbound card builder rather than adding a core special case or a new Feishu-specific shared field.
