## Summary

Keep open. Current main still reproduces the Feishu message-tool presentation gap, and same-author PR #76055 targets this exact path but is still open and already has actionable review blockers.

## What This Changes

The issue reports that Feishu `message` tool sends with presentation buttons render as plain fallback text instead of native Feishu interactive-card buttons.

## Best Possible Solution

Review and land PR #76055 or a narrow replacement that shares a lightweight Feishu native presentation-card renderer between message actions and outbound payloads while preserving mixed text, media/thread behavior, fallback text, and `ocf1` callbacks.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu message-tool `send` with `presentation.blocks[].type === "buttons"`; current main dispatches to the Feishu action handler and the existing test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin action path by sharing or extracting the existing native card renderer; the open PR is the right coordination point but needs adjustment before merge.
