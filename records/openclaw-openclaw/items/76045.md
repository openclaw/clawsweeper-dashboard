## Summary

Keep open. Current main still routes Feishu message-tool presentations through a markdown fallback instead of native card button elements, and the same-author PR at https://github.com/openclaw/openclaw/pull/76055 is the active implementation candidate rather than a reason to close the issue early.

## What This Changes

The issue requests that Feishu `message` tool presentations with buttons render as native interactive Feishu cards instead of plain markdown fallback text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76055 or a narrow replacement that shares a lightweight Feishu native presentation-card renderer between message actions and outbound payloads while preserving mixed text, media/thread behavior, fallback text, and `ocf1` callbacks.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu `message` tool send with `presentation.blocks[].type === "buttons"`; current main routes it through `buildFeishuPresentationCard()`, and the current test expects markdown fallback text instead of a native button.

## Solution Assessment

Yes. The maintainable fix is to reuse or extract the existing Feishu native card element renderer for the message-action presentation path, with focused tests updated to assert native button and select rendering.
