Keep this PR open: it fixes a real SMS command-auth gap, but the patch currently builds `text-slash` command-turn metadata from the broad authorization probe, which can misclassify ordinary SMS text containing inline slash-like tokens as command turns.

The PR adds SMS command authorization resolution, passes command access and text-slash command metadata into inbound SMS context, preserves `SenderE164`, and adds an allowlisted `/status` regression test.

## Best Possible Solution

Keep the SMS command-authorization fix, but split broad authorization probing from exact text-command-turn detection before merge.

- **[P2] Gate command turns on exact SMS command detection:** `extensions/sms/src/inbound.ts:188-194`
