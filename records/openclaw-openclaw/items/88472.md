## Summary

Keep this open: current main has sender, route, command, event, and mention admission gates, but it does not implement the requested content/service-scope admission policy for allowlisted senders before agent dispatch.

## What This Changes

Request adds opt-in channel ingress admission policies, such as service-scope keyword/example matching and default reject, so allowlisted public assistants can filter off-scope messages before agent dispatch.

## Best Possible Solution

Design an opt-in pre-dispatch admission policy that preserves current defaults, has a clear core/plugin ownership boundary, documents the config contract, and proves allow, deny, uncertainty, and admin-bypass behavior before agent dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new policy/config feature request. Source and docs confirm the current supported behavior is sender/route/command/mention admission, not service-scope content admission.

## Solution Assessment

Unclear: pre-dispatch admission is the right layer, but the proposed admissionPolicy/serviceScope schema needs maintainer product and security review before a fix PR should choose the permanent API.
