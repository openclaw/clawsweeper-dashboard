## Summary

Keep this open: current main has sender, group, route, command, event, and mention admission gates, but it does not implement the requested content/service-scope admission policy for allowlisted senders before agent dispatch.

## What This Changes

The issue requests an opt-in channel ingress admission policy that can restrict allowlisted senders to configured services or topics before an agent turn is dispatched.

## Best Possible Solution

Design a generic opt-in channel admission seam that keeps existing sender gates compatible by default, then add WhatsApp coverage, docs, and security review for service-scope matching before considering future model classification.
