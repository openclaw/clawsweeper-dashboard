## Summary

Keep open: current main and the latest release still lack a content/service-scope admission policy before channel messages reach agent dispatch, and the remaining work is a security-sensitive config/API design decision rather than conservative cleanup.

## What This Changes

Request to add opt-in channel admission policies that constrain allowlisted senders to configured business service scopes before agent dispatch.

## Best Possible Solution

Design a generic opt-in channel admission seam that preserves current sender gates by default and lets WhatsApp or other channels reject out-of-service messages before agent dispatch with documented security semantics.

## Reproduction Assessment

No high-confidence bug reproduction is available; this is a feature request. Source inspection does show that current main accepts sender-allowed WhatsApp messages without the requested service-scope content gate.

## Solution Assessment

Unclear. A generic opt-in ingress admission seam is plausible, but the exact config shape, fail-closed behavior, local matcher, admin bypass, and future classifier boundary need product and security review.
