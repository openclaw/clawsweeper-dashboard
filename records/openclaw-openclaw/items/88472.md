## Summary

Keep open: current main and v2026.6.8 still only gate channel ingress by route, sender, command, event, and mention activation; the requested content/service-scope admission layer for allowlisted senders is still missing and needs product/security review.

## What This Changes

Add an opt-in channel admission/service-scope policy that rejects out-of-scope allowlisted sender messages before agent dispatch.

## Best Possible Solution

Design a maintainer-approved, opt-in generic ingress admission seam at the channel or Plugin SDK boundary, keep current sender gates unchanged by default, and add WhatsApp config/schema/docs/tests only after the local matcher, audit, admin-bypass, and failure semantics are settled.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection against current main and v2026.6.8 gives high confidence that the requested service-scope admission mode is absent.

## Solution Assessment

Unclear. A generic opt-in ingress admission seam is plausible, but the config shape, fail-closed default, local matcher semantics, admin bypass, audit/redaction, and Plugin SDK ownership need maintainer security/product design first.
