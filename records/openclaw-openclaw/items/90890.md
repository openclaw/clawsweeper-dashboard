## Summary

Keep open: current main and the latest release still store the browser-local assistant avatar under one global key, so the reported cross-agent overwrite remains source-reproducible. Four open PRs already close this issue, so the next step is to review and select one canonical fix rather than spawn another repair lane.

## What This Changes

The issue reports that Control UI assistant avatar overrides are saved globally instead of per selected agent.

## Best Possible Solution

Land one canonical fix that scopes the browser-local assistant avatar override by the normalized active agent id, preserves existing unscoped avatars through a narrow upgrade path, and adds regression coverage across storage, identity loading, bootstrap, and render paths.

## Reproduction Assessment

Yes. The reported steps map to current main reading and writing one shared `openclaw.control.assistant.v1` localStorage key regardless of `assistantAgentId`; I did not run a live browser repro in this read-only pass.

## Solution Assessment

Yes. The best fix is to scope the existing local override helper and its callers by active agent id, with focused migration/compat coverage for the shipped unscoped key.
