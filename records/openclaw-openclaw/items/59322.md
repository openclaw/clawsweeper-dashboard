## Summary

Keep open. Current main still has the untruncated 160pt session picker, and this draft PR addresses a real UI overflow surface, but the proposed tail truncation is not quite merge-ready because it hides the suffix that distinguishes channel-backed session keys.

## What This Changes

The PR adds single-line tail truncation to the iOS/macOS chat composer session picker label.

## Best Possible Solution

Keep the width cap, but render the selected session label in a way that preserves distinguishing information, such as `.lineLimit(1)` with middle truncation or a compact custom label that keeps the channel/provider and terminal id visible while leaving menu rows readable.

## Reproduction Assessment

Unclear for live reproduction: the code path is easy to exercise with a long key such as `agent:main:discord:channel:<id>`, but this read-only review did not run the iOS/macOS UI. The current source does support the reported overflow shape because the selected label is an uncapped raw key inside a 160pt picker.

## Solution Assessment

No. Adding a line limit is the right direction, but tail truncation is not the best fix for raw session keys because the distinguishing suffix is usually at the end; middle truncation or a custom compact display label is safer.
