## Summary

This draft PR targets a real shared iOS/macOS composer overflow path, but it should stay open because the selected-label fix can hide the distinguishing session suffix and the PR still lacks after-fix iOS/macOS behavior proof.

## What This Changes

The PR adds one-line tail truncation to the shared iOS/macOS chat composer session picker after its 160pt width cap.

## Best Possible Solution

Ship a small composer picker fix that keeps the 160pt cap while preserving the unique session suffix, then verify the long-key case on iOS and macOS.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current main can display raw `agent:main:discord:channel:<id>` keys in a 160pt menu picker, which identifies the affected path but does not prove the fixed UI on device.

## Solution Assessment

No. A one-line cap is the right scope, but tail truncation is not the safest selected-label behavior because the distinguishing session id is normally at the end; middle truncation or a compact label would preserve identity while preventing overflow.
