## Summary

Keep open: current main still has the unbounded native chat composer session picker, but this draft PR lacks after-fix iOS/macOS proof and its tail truncation can hide the distinguishing session suffix.

## What This Changes

The PR adds one-line tail truncation to the shared iOS/macOS chat composer session picker label after the existing 160pt width cap.

## Best Possible Solution

Use a narrow native composer picker label fix that keeps the toolbar within width while preserving the distinguishing session suffix, then attach real iOS and macOS proof for long and short names.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows raw `session.displayName ?? session.key` values can render in the 160pt menu picker, so the overflow path is source-reproducible but not device-proven here.

## Solution Assessment

No. Applying a one-line width cap is the right scope, but `.tail` truncation hides the peer/channel id at the end of raw session keys; middle truncation or a compact label would preserve identity better.
