## Summary

Keep open: this draft PR targets a real native composer overflow path, but the proposed tail truncation can hide the session suffix that distinguishes channel sessions, and the PR still lacks after-fix iOS/macOS proof.

## What This Changes

The PR adds one-line tail truncation to the shared iOS/macOS chat composer session picker after its 160pt width cap.

## Best Possible Solution

Ship a small native composer picker fix that preserves unique session suffixes within the 160pt cap and includes real iOS/macOS proof for long and short session names.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection of current main shows raw `session.displayName ?? session.key` can be shown in a 160pt menu picker, which identifies the overflow path but does not prove it on device.

## Solution Assessment

No. A one-line width cap is the right scope, but tail truncation is not the best selected-label behavior because channel/user identity lives at the end; use middle truncation or compact labels, then verify on iOS and macOS.
