## What This Changes

The PR adds one-line tail truncation to the shared iOS/macOS chat composer session picker label.

## Best Possible Solution

Use a constrained native picker label that keeps both prefix and peer suffix visible, then add after-fix iOS/macOS proof for long and short names.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a raw `session.displayName ?? session.key` menu picker capped at 160pt, which plausibly matches the reported overflow but does not prove it on device.

## Solution Assessment

No. The width-constrained label fix is the right scope, but tail truncation is not the best selected-label behavior because session identity commonly lives at the end; use middle truncation or a compact label that preserves the suffix.
