## Summary

Keep open: current main still renders the raw session label in the 160pt menu picker, and this PR is a focused fix attempt, but it is still draft, lacks real behavior proof, and its tail truncation can hide the suffix that distinguishes channel sessions.

## What This Changes

The PR adds one-line tail truncation to the shared native iOS/macOS chat composer session picker label.

## Best Possible Solution

Constrain the composer picker label with middle truncation or a compact session label that keeps the distinguishing suffix visible, then attach redacted iOS and macOS after-fix proof for long and short names.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows raw long session labels can render inside a 160pt menu picker on current main, so the overflow path is source-reproducible but not device-proven here.

## Solution Assessment

No. The fix boundary is narrow and appropriate, but tail truncation hides the terminal peer or channel id; middle truncation or a compact label would preserve identity while still preventing toolbar overflow.
