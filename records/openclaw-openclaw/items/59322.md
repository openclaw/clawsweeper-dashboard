## Summary

Keep open. The overflow problem is real, but the current branch's tail truncation can hide the unique suffix of raw channel session keys, and the user-facing fix still lacks a required changelog entry and UI validation.

## What This Changes

The PR adds single-line tail truncation to the shared iOS/macOS chat composer session picker label.

## Best Possible Solution

Ship a small composer UI fix that keeps the 160pt width cap while preserving session suffixes, preferably with middle truncation or a compact selected-label formatter, plus a single Unreleased changelog entry and platform UI verification.

## Reproduction Assessment

Unclear for live UI reproduction: this read-only review did not run iOS or macOS UI. The source path does support the reported shape because a long raw key can be the selected label inside a 160pt picker.

## Solution Assessment

No. Limiting the selected label to one line is the right boundary, but tail truncation is not the safest display choice for raw session keys whose unique channel id is usually at the end.
