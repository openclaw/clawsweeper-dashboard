## What This Changes

The PR changes gateway session creation so dashboard New Chat requests with a parent, command hooks, no explicit key, and no initial message create a root session while preserving parent linkage for explicit keyed lifecycle creates.

## Best Possible Solution

Land this narrow gateway-side rootless-create behavior if final CI stays green, keeping lifecycle hooks on the parent and preserving explicit keyed child-session lineage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Control UI New Chat sends an unkeyed `sessions.create` request with `parentSessionKey` and `emitCommandHooks`, and the handler currently writes that parent into the new dashboard entry.

## Solution Assessment

Yes. The patch changes the parent-link write only for the fresh root New Chat shape while leaving hook execution and explicit keyed child-session linkage intact, which is the narrowest maintainable fix I found.
