## What This Changes

The PR updates gateway `sessions.create` so unkeyed, no-message dashboard New Chat requests still run lifecycle hooks against the parent but do not persist `parentSessionKey`, with regression tests for rootless fresh chats and explicit keyed child sessions.

## Best Possible Solution

Land the focused gateway handler change with the added regression tests after normal CI, preserving parent lifecycle hooks while storing dashboard New Chat as a root session.

## Reproduction Assessment

Yes, at source level: current main sends New Chat as `parentSessionKey` plus `emitCommandHooks` with no key/message, and the gateway currently persists that parent onto the new session. I did not execute tests because this review was read-only, but the PR body also reports live Gateway RPC proof.

## Solution Assessment

Yes: the patch keeps the parent hook path intact and changes only the persistence decision for unkeyed, no-message lifecycle creates. The explicit-key regression test covers the main compatibility case that should keep parent linkage.
