## Summary

## What This Changes

The PR adds pre-schema config normalization to infer missing auth profile providers from `<provider>:<name>` profile IDs and adds config regression tests.

## Best Possible Solution

Land a rebased pre-schema normalization that infers provider only when the field is omitted or blank, keeps malformed explicit values schema-fatal, and includes focused regression coverage plus contributor real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main still validates `auth.profiles` against a schema requiring `provider`, with no pre-schema inference for a profile keyed like `qwen-portal:default` and valued as `{ mode: "oauth" }`.

## Solution Assessment

No, not as currently proposed. Pre-schema normalization is the right seam, but the patch should only backfill omitted or blank-string providers and leave malformed explicit provider values for schema rejection.
