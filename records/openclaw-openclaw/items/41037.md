## Summary

Keep open: current main still has the reported config-validation failure, and this PR is in the right area but is not safe to merge as-is because it can mask malformed explicit provider values and lacks the required user-facing changelog entry.

## What This Changes

The PR adds pre-schema auth-profile normalization that infers missing providers from `<provider>:<name>` profile ids, plus regression coverage for inferred and explicit providers.

## Best Possible Solution

Land a rebased narrow fix that infers omitted or blank auth-profile providers before schema validation, preserves malformed explicit provider values for schema rejection, covers the edge cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection against current main shows `auth.profiles["qwen-portal:default"] = { mode: "oauth" }` reaches `OpenClawSchema.safeParse` without a provider backfill while the schema requires `provider: z.string()`.

## Solution Assessment

No, not as currently proposed. Pre-schema normalization is the right seam, but the patch should infer only omitted or blank-string providers, preserve invalid explicit values for schema rejection, add the changelog entry, and rebase onto the current validation pipeline.
