## What This Changes

The branch adds pre-schema config normalization to infer missing auth profile providers from `<provider>:<name>` ids and adds focused regression tests.

## Best Possible Solution

Rebase a narrow fix into the current validation flow that only fills omitted or blank-string providers, preserves malformed explicit providers for schema rejection, and includes regression coverage plus redacted real config-load proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. Pre-schema normalization is the right seam, but the patch should preserve non-string explicit providers so malformed config remains invalid after the rebase.
