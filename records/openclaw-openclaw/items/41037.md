## Summary

Keep open: the bug still exists on current main and the PR uses the right pre-schema normalization seam, but the proposed patch would also mask malformed explicit providers and is missing the required changelog entry.

## What This Changes

The PR adds config validation normalization that infers missing auth profile providers from `<provider>:<name>` profile ids, plus regression tests for inferred and explicit providers.

## Best Possible Solution

Land a rebased narrow fix that infers omitted or blank auth-profile providers before schema validation, preserves malformed explicit provider values for schema rejection, covers the edge cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows `auth.profiles["qwen-portal:default"] = { mode: "oauth" }` reaches schema validation without a provider, while the schema requires `provider: z.string()`.

## Solution Assessment

No, not as currently proposed. Pre-schema normalization is the right seam, but the patch should infer only omitted or blank-string providers, preserve invalid explicit values for schema rejection, add the changelog entry, and rebase onto current main.
