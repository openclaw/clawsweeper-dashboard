## Summary

Keep open. Current main handles the Bonjour/ciao unhandled-rejection class from the reported stack, but the issue is paired with an open closing PR and current main still has raw LAN/tailnet `os.networkInterfaces()` paths that can propagate the sandbox failure.

## What This Changes

The issue reports a gateway crash in restricted Nemoclaw/OpenShell sandboxes when `os.networkInterfaces()` throws `uv_interface_addresses`, preventing TUI connection.

## Best Possible Solution

Land or deliberately retarget the paired PR so gateway-critical LAN/tailnet discovery uses safe or inspection-based interface enumeration, while keeping ciao-specific handling inside the Bonjour plugin.

## Reproduction Assessment

Yes, source-level. Current main directly calls raw interface enumeration in LAN/tailnet helpers, and current tests mock `os.networkInterfaces()` throwing and assert those helpers throw.

## Solution Assessment

Yes for the hardening direction. The narrow maintainable solution is to reuse the existing safe/inspection network-interface boundary at best-effort gateway discovery call sites, then close this issue only after the linked PR lands.
