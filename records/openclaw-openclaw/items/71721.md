## Summary

Keep open. Current main still has gateway LAN and tailnet paths that call raw interface enumeration and can propagate the reported sandbox `uv_interface_addresses` failure, and the issue is already paired with an open closing PR that has not landed.

## What This Changes

Gateway startup should not crash in sandboxes where `os.networkInterfaces()` throws `uv_interface_addresses`; it should degrade and keep the TUI connection usable.

## Best Possible Solution

Gateway-critical LAN and tailnet discovery should use the existing safe or inspection-based interface-enumeration boundary so sandbox failures degrade without crashing while preserving useful diagnostics.

## Reproduction Assessment

Yes, source-level. Current main directly calls raw interface enumeration in the LAN and tailnet helpers, and current tests mock `os.networkInterfaces()` throwing and assert those helpers still throw.

## Solution Assessment

Yes for the fix direction. The narrow maintainable path is to reuse the existing safe/inspection network-interface seam at best-effort discovery call sites, then close this issue only after the linked PR or an equivalent fix lands.
