## Summary

Keep open. Current main has partial Bonjour/ciao hardening for uv_interface_addresses failures, but the core LAN and tailnet discovery helpers still call the raw network interface reader and current tests still assert those paths throw; the linked PR #71950 remains the focused unmerged fix candidate.

## What This Changes

The issue reports that the gateway crashes in a Nemoclaw/OpenShell sandbox when Node's os.networkInterfaces() throws uv_interface_addresses, preventing the TUI from connecting.

## Best Possible Solution

Keep this issue open until PR #71950 or an equivalent narrow fix lands, hardening startup-critical LAN and tailnet discovery callers while preserving the raw readNetworkInterfaces() contract for code that intentionally wants propagation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking os.networkInterfaces() to throw and calling pickPrimaryLanIPv4() or listTailnetAddresses(); current tests still assert both paths throw.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable solution: use the existing safe reader at best-effort gateway/tailnet discovery boundaries instead of globally changing raw helper semantics.
