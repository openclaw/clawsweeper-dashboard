## Summary

Keep open: current main still has LAN and tailnet paths where interface-discovery failures propagate, and PR #71950 is an open closing PR rather than a merged fix.

## What This Changes

The issue asks OpenClaw to tolerate sandbox `os.networkInterfaces()` failures during gateway startup so the TUI can stay connected.

## Best Possible Solution

Land PR #71950 or an equivalent narrow change that uses `safeNetworkInterfaces()` at best-effort gateway LAN and tailnet discovery boundaries while preserving raw `readNetworkInterfaces()` semantics for hard-fail callers.

## Reproduction Assessment

Yes. Mock `os.networkInterfaces()` to throw and call `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert both paths throw.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the best-effort gateway/tailnet callers instead of globally changing the raw helper contract.
