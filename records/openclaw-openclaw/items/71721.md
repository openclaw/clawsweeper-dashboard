## Summary

Keep open. Current main still leaves the reported availability failure reachable through the raw gateway LAN and tailnet network-interface paths, and PR #71950 is already the open closing-syntax implementation candidate for this issue.

## What This Changes

The issue asks the OpenClaw Gateway to tolerate sandbox `os.networkInterfaces()` / `uv_interface_addresses` failures instead of crashing and disconnecting the TUI.

## Best Possible Solution

Keep the issue open until PR #71950 or an equivalent narrow fix lands, so startup-critical LAN and tailnet discovery use the existing safe interface reader while preserving the raw reader for callers that intentionally need propagation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking `os.networkInterfaces()` to throw and calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert that both paths throw.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable solution: harden best-effort gateway and tailnet discovery callers without globally changing `readNetworkInterfaces()` semantics.
