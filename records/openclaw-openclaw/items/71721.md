## Summary

Keep open. Current main partially handles the Bonjour/ciao `uv_interface_addresses` failure path, but raw LAN and tailnet interface discovery paths still propagate a throwing `os.networkInterfaces()`, and the issue is already paired with open PR #71950 as the focused fix candidate.

## What This Changes

Harden gateway network-interface enumeration so a Nemoclaw/OpenShell sandbox `uv_interface_addresses` SystemError degrades gracefully instead of crashing the gateway and disconnecting the TUI.

## Best Possible Solution

Land PR #71950 or an equivalent narrow change that keeps raw `readNetworkInterfaces()` semantics intact while using the safe reader at best-effort gateway LAN, tailnet, and discovery boundaries that should degrade when sandbox interface enumeration is unavailable.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking `os.networkInterfaces()` to throw and calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert both paths throw.

## Solution Assessment

Yes. The best fix direction is the narrow boundary change represented by PR #71950: use the existing safe reader where gateway startup and discovery should degrade, without globally changing the raw helper contract.
