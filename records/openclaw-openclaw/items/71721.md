## Summary

Keep open. Current main still has raw LAN and tailnet network-interface discovery paths that can propagate the reported sandbox `uv_interface_addresses` failure; a related Bonjour crash path was merged separately, but the issue is still paired with the open closing implementation at https://github.com/openclaw/openclaw/pull/71950.

## What This Changes

The issue asks for gateway startup to survive `os.networkInterfaces()` / `uv_interface_addresses` failures in Nemoclaw/OpenShell sandboxes instead of disconnecting the TUI.

## Best Possible Solution

Land or replace the open fix PR with the safe/inspection-helper approach for gateway-critical LAN and tailnet discovery, preserving diagnostics, then close this issue after merge and proof.

## Reproduction Assessment

Yes, source-level. Current main directly calls raw interface enumeration in LAN and tailnet helpers, and current tests already assert those helpers throw when `os.networkInterfaces()` throws.

## Solution Assessment

Yes for the fix direction, but not for closing yet. The narrow maintainable path is to reuse the existing safe/inspection network-interface seam and land https://github.com/openclaw/openclaw/pull/71950 or an equivalent fix before closing.
