## Summary

Keep open. Current main still has source-level paths where `os.networkInterfaces()` failures propagate from LAN and tailnet helpers, and PR #71950 is an open closing PR for this report rather than a merged fix.

## What This Changes

The issue requests that gateway network-interface discovery avoid crashing when `os.networkInterfaces()` throws inside Nemoclaw/OpenShell-style sandbox environments.

## Best Possible Solution

Land PR #71950 or an equivalent narrow change that uses the existing safe interface reader at best-effort gateway LAN, tailnet, and related discovery boundaries while preserving `readNetworkInterfaces()` for callers that intentionally require hard failures.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking `os.networkInterfaces()` to throw and calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests explicitly assert both paths throw.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the gateway/tailnet best-effort callers with `safeNetworkInterfaces()` rather than globally changing the raw helper contract.
