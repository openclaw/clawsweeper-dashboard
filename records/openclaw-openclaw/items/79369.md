## Summary

Keep open: current main intentionally treats per-agent string models as strict and documents that behavior, but it still lacks the requested doctor warning for the confusing defaults-fallback combination.

## What This Changes

Request to add an `openclaw doctor` warning when a per-agent string `model` combines with non-empty `agents.defaults.model.fallbacks`, because that override disables inherited fallbacks.

## Best Possible Solution

Add a warning-only doctor diagnostic with regression coverage, keep the documented strict fallback semantics unchanged, and leave any automatic `--fix` migration for a separate maintainer decision.

## Reproduction Assessment

Yes, by source inspection: the reported config shape is accepted, string-form per-agent models resolve to an empty fallback override, and that override wins over default fallbacks. The current doctor warning collector has no matching diagnostic branch for this pattern.

## Solution Assessment

Yes for the warning-only direction: it catches the footgun without contradicting current docs or tests. Automatic migration should be separate because it changes operator config rather than only surfacing risk.
