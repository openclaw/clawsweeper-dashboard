## Repair Work Prompt

Add a warning-only `openclaw doctor` diagnostic for configs where `agents.defaults.model.fallbacks` resolves to a non-empty list and one or more `agents.list` entries set `model` as a non-empty string. The warning should name the agent id/path and explain that string-form per-agent `model` is strict and does not inherit default fallbacks; suggest object form `{ primary, fallbacks: [...] }` to keep fallback active, or `{ primary, fallbacks: [] }` to make strict behavior explicit. Keep runtime fallback semantics unchanged, and do not add automatic `--fix` migration unless a maintainer explicitly requests it. Add focused unit coverage and a user-facing changelog entry.
