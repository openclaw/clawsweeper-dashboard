## Summary

Keep open: current main still has a legacy/plugin reply dispatcher seam where standalone `MEDIA:` text can reach `deliver` without being promoted to `mediaUrl`/`mediaUrls`. The normal agent-runner, block-reply, and durable outbound paths already parse directives, so this remains a narrow consistency bug rather than a new feature request.

## What This Changes

The issue asks OpenClaw to normalize standalone `MEDIA:` directives into media payload fields before Weixin channel delivery.

## Best Possible Solution

Normalize directive-bearing payloads at the shared legacy/plugin dispatcher boundary before `deliver`, reusing the existing parser and keeping the fix channel-agnostic.

## Reproduction Assessment

Yes. A payload like `{ text: "caption\nMEDIA:/tmp/a.png" }` sent through `createReplyDispatcherWithTyping` reaches `deliver` after `normalizeReplyPayload`, which does not parse `MEDIA:`, so a plugin reading only `mediaUrl`/`mediaUrls` sees no media.

## Solution Assessment

Yes. The narrowest maintainable fix is shared core normalization before legacy/plugin delivery, not a Weixin-specific special case or a plugin-only workaround.
