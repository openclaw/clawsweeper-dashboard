## Summary

Current main and v2026.6.6 already normalize final standalone MEDIA directives into outbound media fields before channel delivery, with regression coverage for the reported text-before-MEDIA shape. The linked contributor PR has useful Weixin proof, but the central issue no longer needs to stay open.

## What This Changes

Report that Weixin text-only delivery can occur when final standalone MEDIA directives are not normalized into payload media fields before channel delivery.

## Best Possible Solution

Keep the shared core final-reply MEDIA normalization in place and handle the now-redundant linked branch as a separate PR review item.

## Reproduction Assessment

No current-main reproduction remains: source and regression tests show final MEDIA directives are parsed into mediaUrls and emitted once before channel delivery. The original 2026.5.5 failure was plausible but is covered by current main and v2026.6.6.

## Solution Assessment

Yes. The shared core final-reply/media normalization is the maintainable fix, rather than adding a Weixin-only fallback parser.
