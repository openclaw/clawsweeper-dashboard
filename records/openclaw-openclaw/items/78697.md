## Summary

Keep open. The report is plausible and distinct from https://github.com/openclaw/openclaw/issues/75182: current source shows the shared parser can normalize standalone MEDIA directives, but the channel-owned turn delivery path can still hand a prepared ReplyPayload directly to an external deliver callback, so the core/plugin boundary needs maintainer follow-up.

## What This Changes

The issue asks OpenClaw and the external Weixin channel to preserve standalone MEDIA directives by normalizing them into mediaUrl/mediaUrls before reply delivery.

## Best Possible Solution

Choose and document one supported boundary, then cover it with a regression: either normalize reply directives before channel-owned delivery or expose/recommend an SDK helper that the Weixin plugin uses for leftover MEDIA text.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue includes concrete Weixin logs and current source shows a plausible unnormalized channel-owned deliver boundary, but the external plugin path was not available for direct current-main verification.

## Solution Assessment
