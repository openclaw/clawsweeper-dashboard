## Summary

Keep open: current main covers final `MEDIA:` parsing when `message_end` carries the final text, but source inspection still shows the active `text_end` streaming path does not parse legacy `MEDIA:` directives from its final accumulator, and the linked candidate PR remains open rather than merged.

## What This Changes

The issue reports that standalone final-reply `MEDIA:` directives can reach the Weixin channel as plain text without `payload.mediaUrl` or `payload.mediaUrls`, causing image delivery to be skipped.

## Best Possible Solution

Keep this issue as the canonical bug until maintainers either land a narrow current-file port for text_end final `MEDIA:` handling or explicitly decide that message_end coverage is sufficient for the supported provider/channel contract.

## Reproduction Assessment

Yes at source level: the active text_end path consumes the streaming directive accumulator, and that accumulator does not parse legacy `MEDIA:` lines into mediaUrls. I did not run a live Weixin reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs at the shared embedded-agent final-reply media boundary rather than as a Weixin-only fallback parser. The linked PR's useful behavior needs to be ported to current `src/agents/embedded-agent-subscribe*` files before it can resolve the issue.
