## Summary

Keep open: current main still has a plugin-facing reply-dispatch path where standalone MEDIA text is passed to a channel deliver callback without being promoted to mediaUrl/mediaUrls. The prior ClawSweeper comment remains accurate, and the reporter's Weixin fallback is useful evidence that this is a shared delivery-contract gap, not the already-closed polluted-path parser bug.

## What This Changes

The issue asks OpenClaw to normalize standalone MEDIA directives in final replies into mediaUrl/mediaUrls before Weixin channel delivery.

## Best Possible Solution

Normalize reply directives in the shared plugin-facing dispatcher path before channel deliver callbacks, reusing the existing directive parser and keeping the fix channel-agnostic.

## Reproduction Assessment

Yes, source-reproducible: a final payload like `{ text: "caption\nMEDIA:/tmp/a.png" }` through `createReplyDispatcherWithTyping` reaches `deliver` after normal reply cleanup but before any MEDIA parsing. The published Weixin plugin then sees no media fields and takes its text-only branch.

## Solution Assessment

Yes, the narrowest maintainable fix is shared core normalization before legacy/plugin delivery. A Weixin-local fallback is useful defense in depth, but core already parses this directive on durable outbound and block-reply paths and should make the plugin callback contract consistent.
