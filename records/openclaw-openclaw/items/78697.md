## Summary

Keep open: current main still has a source-reproducible legacy/channel delivery path where a final payload containing standalone `MEDIA:` text can reach a plugin deliverer without being promoted to `mediaUrl` or `mediaUrls`. The related polluted-path parser fix at https://github.com/openclaw/openclaw/issues/75182 is distinct and does not close this delivery-boundary normalization gap.

## What This Changes

The issue requests that standalone `MEDIA:` directives in assistant final replies be normalized into reply media fields before Weixin channel delivery when `payload.mediaUrl` is absent.

## Best Possible Solution

Normalize reply directives at the shared dispatcher or legacy plugin delivery boundary using `normalizeReplyPayloadDirectives`, preserving already-normalized payload fields and existing delivery ordering.

## Reproduction Assessment

Yes. Source-reproducible: a payload like `{ text: "caption\nMEDIA:/tmp/a.png" }` sent through the dispatcher path is normalized by `normalizeReplyPayload` and can then reach `options.deliver` without directive parsing, while the shared normalizer proves the expected media-field transformation exists elsewhere.

## Solution Assessment

Yes. The narrowest maintainable fix is channel-agnostic normalization before legacy/plugin delivery, not a Weixin-specific parser in OpenClaw core.
