## Summary

Keep open: current main still has a source-reproducible reply-dispatcher path where standalone `MEDIA:` text can reach a channel `deliver` callback without being promoted to `mediaUrl` or `mediaUrls`. The related polluted-path parser issue at https://github.com/openclaw/openclaw/issues/75182 was fixed separately and does not cover this delivery-boundary normalization gap.

## What This Changes

The issue asks OpenClaw to normalize standalone `MEDIA:` directives into media payload fields before Weixin channel delivery.

## Best Possible Solution

Normalize reply directives at the shared dispatcher or legacy plugin delivery boundary using `normalizeReplyPayloadDirectives`, preserving already-normalized payload fields and existing delivery ordering.

## Reproduction Assessment

Yes, source-reproducible: a payload such as `{ text: "caption\nMEDIA:/tmp/a.png" }` sent through `createReplyDispatcherWithTyping` reaches `options.deliver` after `normalizeReplyPayload`, which does not parse `MEDIA:` into media fields. The existing shared normalizer demonstrates the expected media-field transformation but is not called on this dispatcher path.

## Solution Assessment

Yes. The narrowest maintainable fix is channel-agnostic normalization before legacy/plugin delivery, not a Weixin-specific parser in OpenClaw core; existing block and agent-runner paths already use the shared directive normalizer.
