## Label Justifications

- `P2`: The original issue described a normal-priority media delivery bug with limited blast radius in final assistant replies.
- `impact:message-loss`: The reported failure omitted the intended image media payload while still sending text.

## Summary

Current main and the shipped v2026.6.6 release already normalize standalone final `MEDIA:` lines into outbound media fields before channel delivery, so this issue no longer needs a separate core bug track.

## What This Changes

The issue reports that Weixin delivery can receive assistant text containing a standalone `MEDIA:` directive without `payload.mediaUrl` or `payload.mediaUrls`, causing image replies to be sent as text only.

## Best Possible Solution

Keep the shipped shared core final-reply MEDIA normalization and handle the now-redundant linked PR separately.

## Reproduction Assessment

No current-main reproduction remains: source and regression tests show final standalone MEDIA directives are parsed into mediaUrls before block reply delivery. The original 2026.5.5 Weixin failure was plausible, but v2026.6.6/current main cover the reported path.

## Solution Assessment

Yes. The shipped fix is at the shared core final-reply/media boundary, which is the maintainable solution instead of adding a Weixin-only fallback parser.
