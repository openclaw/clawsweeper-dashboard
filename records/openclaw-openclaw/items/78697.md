## Summary

Current main and release v2026.6.6 already normalize final standalone MEDIA lines into outbound media fields before channel delivery, so the reported Weixin text-only failure no longer needs a separate issue-fix path.

## What This Changes

The issue requests that standalone assistant final-reply MEDIA directives be normalized into payload.mediaUrl/mediaUrls before Weixin channel delivery when those fields are initially absent.

## Best Possible Solution

Keep the shared core final-reply MEDIA normalization in place and handle the now-redundant linked PR separately.

## Reproduction Assessment

No current-main reproduction remains: source and regression tests show final MEDIA directives are parsed into mediaUrls before block reply delivery. The original 2026.5.5 failure was plausible, but v2026.6.6/current main cover the reported path.

## Solution Assessment

Yes. The shipped fix is at the shared core final-reply/media boundary, which is the maintainable solution instead of adding a Weixin-only fallback parser.
