## Summary

Keep open: current main still has a source-reproducible delivery-boundary gap where standalone MEDIA text can reach a channel deliver callback without mediaUrl/mediaUrls, and the same reporter has an open companion PR at https://github.com/openclaw/openclaw/pull/79339 that should be updated or replaced rather than closing this issue now. The polluted-path parser report at https://github.com/openclaw/openclaw/issues/75182 is related context but fixed a different parser bug.

## What This Changes

The issue reports that Weixin final replies containing standalone MEDIA directives can be delivered as text-only payloads when mediaUrl/mediaUrls are absent.

## Best Possible Solution

Normalize standalone MEDIA directives at the shared final/legacy delivery boundary using the existing directive parser, preserving silent reply, heartbeat stripping, response prefix, media path normalization, and duplicate-delivery behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a payload such as `{ text: "caption\nMEDIA:/tmp/a.png" }` sent through the dispatcher/legacy deliver path is normalized with normalizeReplyPayload or normalizeOutboundReplyPayload and then delivered without mediaUrl/mediaUrls.

## Solution Assessment

Yes, the narrow maintainable direction is shared channel-agnostic directive normalization before channel delivery; a Weixin-local fallback is useful defense in depth but should not be the core fix. The open PR at https://github.com/openclaw/openclaw/pull/79339 appears to target the right symptom but still needs its existing review blocker resolved before landing.
