## Summary

Current main and shipped release v2026.6.6 already normalize standalone final MEDIA directives into outbound media fields before channel delivery, so the reported Weixin text-only failure no longer needs a separate open issue.

## What This Changes

The issue asks OpenClaw core/channel delivery to preserve standalone final MEDIA directives as payload media fields instead of sending them as text only.

## Best Possible Solution

Keep the shared core final-reply MEDIA normalization and close this now-redundant issue; handle the stale linked PR separately.

## Reproduction Assessment

No current-main reproduction remains: source and regression tests show final MEDIA directives are parsed into mediaUrls before block reply/channel delivery. The original 2026.5.5 failure was plausible, but v2026.6.6 and current main cover the reported path.

## Solution Assessment

Yes. The implemented shared core final-reply MEDIA normalization is the maintainable fix, instead of adding a Weixin-only fallback parser.
