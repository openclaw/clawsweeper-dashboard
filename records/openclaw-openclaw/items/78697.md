## Summary

Keep open: current main still has a legacy/plugin reply delivery path that can pass standalone `MEDIA:` text to a channel callback without promoting it to `mediaUrl` or `mediaUrls`. A same-author PR is open for a related final-media drop, but it is not merged and should be resolved before this issue closes.

## What This Changes

Normalize standalone final `MEDIA:` directives so the Weixin channel receives `mediaUrl`/`mediaUrls` instead of a text-only payload.

## Best Possible Solution

Land a channel-agnostic core fix that promotes standalone `MEDIA:` directives before legacy/plugin deliver callbacks; if the paired PR is the chosen route, ensure it also covers this issue’s `payload.mediaUrl` contract.

## Reproduction Assessment

Yes. Source inspection shows `{ text: "caption\nMEDIA:/tmp/a.png" }` can flow through `createReplyDispatcherWithTyping` to a plugin deliver callback without MEDIA parsing, and the published Weixin plugin sends text-only when media fields are absent.

## Solution Assessment

Yes, the maintainable direction is shared core normalization before plugin delivery rather than a Weixin-specific workaround. The same-author PR should be reviewed as the current implementation candidate, but the final fix should preserve the generic channel contract.
