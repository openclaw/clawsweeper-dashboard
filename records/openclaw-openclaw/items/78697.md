## Summary

Keep open: current main handles final `MEDIA:` parsing when the final text arrives through `message_end`, but the active `text_end` streaming path still depends on an accumulator that withholds `MEDIA:` tails without producing `mediaUrls`, and the linked candidate fix is still open and not landable as submitted.

## What This Changes

Issue reports that standalone `MEDIA:` directives can reach the Weixin channel as text without normalized `mediaUrl` or `mediaUrls`, causing image replies to be sent as text only.

## Best Possible Solution

Keep this issue as the canonical bug and resolve it by porting the relevant final-media handling into the current `src/agents/embedded-agent-subscribe*` path while preserving existing message_end and tool-media delivery behavior.

## Reproduction Assessment

Yes at source level: the active `text_end` path consumes the streaming directive accumulator, and that accumulator does not parse legacy `MEDIA:` lines into `mediaUrls`. I did not run a live Weixin reproduction in this read-only review.

## Solution Assessment

Yes for the issue direction: the best fix belongs at the shared embedded-agent final-reply media boundary, not as a Weixin-only fallback. The open candidate PR still needs a current-file port before it can resolve the issue.
