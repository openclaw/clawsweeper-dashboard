## Summary

Keep open: current main still has a legacy reply-dispatch path used by the published Weixin plugin where standalone `MEDIA:` text is delivered to the plugin without being promoted to `mediaUrl` or `mediaUrls`. The durable outbound path already parses the directive, so this looks like a real contract gap rather than the closed polluted-path parser bug at https://github.com/openclaw/openclaw/issues/75182.

## What This Changes

The report asks OpenClaw to preserve standalone `MEDIA:` directives as `mediaUrl`/`mediaUrls` when final replies are delivered to the external Weixin channel.

## Best Possible Solution

Normalize reply directives before any legacy/plugin deliver callback, reusing the shared MEDIA parser while keeping the fix channel-agnostic.

## Reproduction Assessment

Yes, source-reproducible: a final payload like `{ text: "caption\nMEDIA:/tmp/a.png" }` through `createReplyDispatcherWithTyping` reaches `deliver` after silent/heartbeat normalization but before any MEDIA parsing. The published Weixin plugin then sees no `mediaUrl`/`mediaUrls` and takes its text-only branch.

## Solution Assessment

Yes, the narrowest maintainable fix is shared core normalization before legacy/plugin delivery, not a Weixin-specific core special case. Durable outbound and block-reply paths already parse the directive, so the repair should align this remaining dispatcher path with those contracts.
