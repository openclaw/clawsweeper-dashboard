## Summary

Keep open: current main still has a source-reproducible legacy/plugin delivery path where standalone `MEDIA:` text can reach a channel `deliver` callback without being promoted to `mediaUrl` or `mediaUrls`. The closed polluted-path parser report at https://github.com/openclaw/openclaw/issues/75182 is related context but does not cover this delivery-boundary normalization gap.

## What This Changes

Ensure standalone `MEDIA:` directives in final assistant replies are normalized into `mediaUrl`/`mediaUrls` before Weixin channel delivery.

## Best Possible Solution

Normalize reply directives at the shared dispatcher or legacy plugin delivery boundary, reusing `normalizeReplyPayloadDirectives` while preserving already-normalized payload fields.

## Reproduction Assessment

Yes, source-reproducible: a payload like `{ text: "caption\nMEDIA:/tmp/a.png" }` sent through `createReplyDispatcherWithTyping` reaches `deliver` after `normalizeReplyPayload`, which does not parse `MEDIA:` into media fields. The published Weixin deliver callback then sees no `mediaUrl`/`mediaUrls` and takes its text-only branch.

## Solution Assessment

Yes. The narrowest maintainable fix is channel-agnostic shared normalization before legacy/plugin delivery, not a Weixin-specific parser in core; existing agent-runner and outbound helpers show the parser contract to reuse.
