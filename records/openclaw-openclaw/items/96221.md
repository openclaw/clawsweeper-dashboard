## Summary

Keep open: the generic bounded page read is useful, and the PR has real live-output proof, but it is not merge-ready because the new selector field fails on the top-level `/act` route and the LinkedIn-specific browser API needs maintainer direction.

## What This Changes

The branch adds a browser `readPageState` action, a `linkedin_invitations` page-state classifier, and switches existing-session wait text checks from `innerText` to `textContent`.

## Best Possible Solution

Keep the browser plugin surface generic and internally consistent, then put LinkedIn invitation semantics in the owning skill/plugin unless maintainers explicitly approve a service-specific browser contract.

## Reproduction Assessment

Yes for the patch defect: source inspection shows `/act` rejects `selector` for `readPageState` before Playwright receives the request. The original LinkedIn cron failure has PR-body live-output evidence but was not independently rerun in this read-only review.

## Solution Assessment

No; a generic bounded page-state read is a plausible fix, but adding LinkedIn invitation classification to the generic browser plugin is broader than the owner boundary requires. The safer path is a generic read contract plus LinkedIn parsing in the owning skill/plugin, or explicit maintainer approval for the service-specific API.
