## What This Changes

The PR changes Discord configured ACP parent-channel fallback matching so inherited thread messages use the live thread id for the materialized ACP session key, with related registry and lifecycle tests.

## Best Possible Solution

Land or replace this with a current-main patch that changes only the Discord inherited parent-channel match to materialize the live thread id, preserves direct bindings and other channel-owned canonical remaps, fixes the test helper, and includes redacted real Discord/acpx proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: Discord preflight passes a live thread id plus parent id, the matcher returns the parent id, the registry materializes that match, and the configured ACP key hashes only conversationId.

## Solution Assessment

Yes for the central fix direction, but no for the current branch as merge-ready. The narrow maintainable fix is to return the live thread id only for Discord parent fallback while retargeting the patch to current main and keeping channel-canonical remaps intact.
