## Summary

## What This Changes

This PR changes Discord configured ACP parent-channel fallback matching so inherited threads use the live thread id, adds ACP binding/lifecycle regression tests, and documents reverse-lookup limits.

## Best Possible Solution

Land or replace this with a current-main patch that keys inherited Discord parent-channel ACP bindings by the live thread id, preserves channel-owned canonical remaps, fixes the tests, and includes redacted real Discord/acpx proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by resolving two Discord configured ACP bindings with different live thread conversation ids and the same `parentConversationId`; current main returns the parent id and hashes it into the same configured ACP key.

## Solution Assessment

Yes for the central matcher direction, but not for the current branch as merge-ready. The narrow maintainable fix is to return the live thread id only for Discord parent fallback while preserving direct matches and other channel canonical remaps.
