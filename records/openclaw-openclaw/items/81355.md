## Summary

Keep this issue open. ## What This Changes

This issue asks to reduce cold-start gateway RPC fanout latency by unblocking `tts.status` and deduplicating repeated `applyPluginAutoEnable` work.

## Best Possible Solution

Keep the shipped auto-enable cache and make the remaining TTS status RPC cheap or yielding, with focused coverage that proves prefs/provider configured-state work is not duplicated and does not monopolize sibling gateway RPCs.
