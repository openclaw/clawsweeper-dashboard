## What This Changes

The issue requests reducing first-load gateway RPC fanout latency by making `tts.status` stop monopolizing the event loop and avoiding repeated `applyPluginAutoEnable` recomputation in one fanout.

## Best Possible Solution

Keep this issue as the canonical report, fix or replace the auto-enable cache PR, and land a separate TTS status optimization that reduces sync prefs/provider work without breaking plugin contracts.
