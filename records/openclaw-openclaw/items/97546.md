## What This Changes

The branch extracts Teams Adaptive Card submit-value serialization into a helper and uses it when normal Microsoft Teams message activities have empty text but submit data in `activity.value`.

## Best Possible Solution

Land the focused fallback in the Teams message normalization path, preserving typed text and HTML precedence, or carry the same behavior into the broader multi-account branch if that lands first.

## Reproduction Assessment

Yes. Current-main source shows empty-text Teams message activities ignore `activity.value` and are skipped before dispatch; I did not run the Teams flow locally, but the PR includes live Teams proof for the observed fallback shape.

## Solution Assessment

Yes. Reusing the existing Adaptive Card submit serializer only when raw text and HTML text are absent is the narrowest maintainable fix; a broader Teams interactivity rewrite or a second serializer would add unnecessary drift.
