## Summary

Keep open. Current main still performs one guarded fetch for normal media-understanding attachment URLs, and #74553 is an open unmerged PR with closing syntax that should remain the implementation path.

## What This Changes

Add bounded retries for transient media-understanding remote audio fetch failures before transcription starts.

## Best Possible Solution

Land #74553 or an equivalent narrow retry implementation at the fetchRemoteMedia/MediaAttachmentCache boundary while preserving SSRF, size, 4xx, and abort fail-fast behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a remote Discord audio URL through preflight/media-understanding and make the guarded fetch fail transiently or return 5xx; current main has one cache fetch call before surfacing failure.

## Solution Assessment

Yes. The shared fetch/cache boundary is the narrowest maintainable fix because it covers Discord audio preflight and other media-understanding consumers without Discord-specific retry logic.
