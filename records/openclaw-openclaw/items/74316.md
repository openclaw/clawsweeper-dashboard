## Summary

Keep open. Current main still single-attempts normal media-understanding remote attachment fetches, and open PR #74553 is the linked unmerged fix path.

## What This Changes

Add bounded retry/backoff for transient remote media fetch failures so Discord audio media-understanding does not drop voice notes after one fetch failure.

## Best Possible Solution

Land PR #74553 or an equivalent narrow shared retry implementation at the fetchRemoteMedia/MediaAttachmentCache boundary while preserving SSRF, size, 4xx, and abort fail-fast behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a remote Discord audio URL through preflight/media-understanding and make the guarded fetch fail transiently or return 5xx; current main performs one cache fetch call before surfacing failure.

## Solution Assessment

Yes. A shared retry policy at the remote media fetch/cache boundary is the narrowest maintainable fix because it covers Discord audio preflight and other media-understanding consumers without adding Discord-specific behavior.
