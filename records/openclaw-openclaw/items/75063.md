## What This Changes

The PR changes the Docker attestation verifier and tests to accept attestation manifests that omit artifactType when their layers are in-toto JSON.

## Best Possible Solution

Keep the shipped verifier behavior from v2026.5.7; any future desire for stricter layer mediaType validation should be opened as a separate narrow hardening issue.

## Reproduction Assessment

No for current main. The cited GHCR image verifies with {"errors":[]} on current main, and the attestation manifests from that image omit artifactType rather than carrying a bad value.

## Solution Assessment

Yes. The shipped fix addresses the actual buildx output at the verifier boundary while preserving legacy artifactType rejection for present-but-wrong values and the required predicate checks.
