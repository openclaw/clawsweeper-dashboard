## What This Changes

The PR updates `scripts/verify-docker-attestations.mjs` to recognize Docker attestation manifests by either legacy `artifactType` or all in-toto layer media types, and adds unit tests for the current Buildx shape plus a malformed-manifest rejection case.

## Best Possible Solution

Land this PR or an equivalent focused verifier update that accepts the current Buildx OCI-manifest-with-in-toto-layers shape while preserving legacy support, descriptor prefiltering, required SBOM/provenance predicate checks, and malformed-manifest diagnostics.
