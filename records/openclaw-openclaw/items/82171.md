## Summary

Keep open. Current main still has a clear source path where a final non-assistant current-turn `item/completed` can disarm the short completion-idle watchdog, and the same author has an open paired fix PR that should be reviewed or landed before this issue is closed.

## What This Changes

The issue asks OpenClaw Codex app-server turns to fail fast when the final non-assistant current-turn item completes and no `turn/completed` follows.

## Best Possible Solution

Review and land the paired PR or an equivalent narrow fix that re-arms or preserves the short watchdog after the final non-assistant current-turn item completes, with regression coverage and any needed Codex harness docs alignment.

## Reproduction Assessment

Yes at source level: current main arms the completion-idle watchdog at turn start, then can disarm it on the final non-assistant `item/completed` when no active item remains and no `turn/completed` follows. I did not run the Vitest repro because this review was required to keep the checkout read-only.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: keep or re-arm the short completion-idle guard only for the final non-assistant current-turn completion case, without changing assistant-message release semantics. The paired PR is the right path to review before closing this issue.
