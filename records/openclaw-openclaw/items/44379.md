## Summary

Keep open: current main still lacks the central suppress-hook overflow fallback, and the PR contains useful source work, but the proposed patch should not merge as-is because the retry policy, prompt-hook behavior, stale package metadata, and changelog coverage need a focused rework.

## What This Changes

The PR adds a one-attempt suppress-prompt-hook overflow retry, normalizes context-window stop-reason strings, updates related tests, and changes Google Chat/memory-core package plus lockfile metadata.

## Best Possible Solution

Land a focused rework or replacement that adds one bounded overflow fallback only when hook or queued context can affect prompt size, preserves non-context hook policy effects and compaction_failure semantics, removes package churn, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: mock `runEmbeddedAttempt` to return context overflow while compaction returns nothing and no oversized tool results exist; current main returns `context_overflow` after that path and has no suppress-hook retry.

## Solution Assessment

No. The fallback direction is reasonable, but the current patch is not the narrowest maintainable fix because it skips all prompt-build hook effects, schedules the retry without budget or hook-availability checks, and carries unrelated package metadata changes.
