## Summary

Keep this PR open, but the current patch should not merge as-is: the suppress-retry idea is not already implemented on main, yet the branch still lacks real behavior proof and introduces correctness/security concerns around broad hook suppression, retry budgeting, and unrelated package churn.

## What This Changes

The PR adds a one-attempt prompt-hook-suppressed retry after embedded PI context-overflow recovery fails, normalizes context-window stop-reason errors, updates targeted tests, and changes Google Chat/memory-core peer dependency plus lockfile metadata.

## Best Possible Solution

Rebase the runner fix into a narrow replacement that suppresses only shrinkable hook-provided context when it can help and budget remains, preserves policy/system hook effects, drops package churn, and includes redacted real-run proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main resolves prompt-build hooks before each attempt and returns context_overflow after compaction/tool-result truncation are exhausted, so the reported hook-induced overflow path is plausible but not proven in a real setup.

## Solution Assessment

No. The fallback direction is plausible, but this patch suppresses all prompt-build effects, retries without loop-budget or hook-availability guards, and mixes unrelated dependency churn into the branch.
