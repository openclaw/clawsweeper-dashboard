## Summary

Keep open. Current main already covers the stop-reason normalization part, but the branch still proposes a distinct hook-suppression recovery behavior and has blocking correctness, proof, and security/supply-chain review issues.

## What This Changes

The PR adds a one-attempt prompt-hook-suppressed retry for embedded PI context overflow, context-window stop-reason normalization coverage, and Google Chat/memory-core peer dependency plus lockfile updates.

## Best Possible Solution

Land a rebased replacement that only suppresses shrinkable hook-provided context when it can help, preserves policy/system hook effects, removes unrelated package churn, and includes redacted real-run proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main still resolves prompt-build hooks before each attempt and returns context_overflow after compaction/tool-result truncation without a hook-suppressed retry path.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses all prompt-build hook effects, retries without loop-budget or hook-availability guards, and mixes unrelated dependency churn into the branch.

  - body: This schedules another run-loop iteration after terminal overflow recovery. If overflow happens on the last allowed pass, the next iteration hits the retry-limit guard and returns retry_limit instead of context_overflow guidance and telemetry.
  - confidence: 0.9

  - confidence: 0.88

  - body: When no queued injection or prompt-build hook context can shrink the prompt, suppressPromptHookContext cannot change the constructed request. This adds another full model call that should fail with the same overflow.
  - confidence: 0.84

  - body: This runner recovery PR also changes Google Chat and memory-core package metadata plus a broad pnpm-lock dependency graph. Split or remove that unrelated supply-chain review scope before this branch is considered.
