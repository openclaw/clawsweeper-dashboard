## Summary

Keep open. The retry idea is not implemented on current main and may address a real overflow path, but this branch still has blocking correctness/security findings and lacks external real-behavior proof; only the stop-reason normalization slice is mostly superseded on main.

## What This Changes

The PR adds one PI runner context-overflow retry with prompt-build hooks suppressed, adds context-window stop-reason normalization/tests, and changes Google Chat/memory-core peer metadata plus lockfile entries.

## Best Possible Solution

Land a revised branch or maintainer replacement that adds a bounded hook-context-only overflow fallback, preserves system/policy hook effects and compaction_failure semantics, gates the retry on loop budget and meaningful hook context, and drops unrelated package churn.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main can reach the terminal context_overflow return after compaction and tool-result truncation cannot recover; I did not run tests because this review is read-only.

## Solution Assessment

No. The core fallback direction is plausible, but this implementation is too broad because it suppresses all prompt-build hook effects, can spend an extra retry without loop budget or useful hook context, and includes unrelated dependency churn.
