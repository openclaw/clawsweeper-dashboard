## Summary

Keep open: current main still lacks the proposed suppress-hook overflow fallback, but this branch has blocking correctness, security, proof, and conflict issues that need revision before merge.

## What This Changes

This PR adds a one-attempt prompt-hook-suppressed overflow retry in the embedded PI runner, normalizes context-window stop-reason handling, updates focused tests, and changes Google Chat/memory-core peer metadata plus the lockfile.

## Best Possible Solution

Land a revised or replacement branch that adds a bounded context-only hook fallback, preserves system/policy hook effects and compaction_failure semantics, gates on loop budget and useful hook context, removes unrelated package churn, and includes redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can still reach the terminal context_overflow return after compaction and tool-result truncation are exhausted, but this review did not run a live memory-hook overflow setup.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses the whole prompt-build resolver, spends an extra retry without budget or hook-availability gates, and includes unrelated dependency churn.
