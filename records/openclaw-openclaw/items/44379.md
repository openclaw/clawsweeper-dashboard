## Summary

Keep open. The central suppress-hook overflow retry is not on current main, but this branch is not merge-ready because it lacks real behavior proof, suppresses more hook behavior than the title describes, retries without key guards, and carries unrelated dependency churn.

## What This Changes

The PR adds a final prompt-hook-suppressed retry after context-overflow recovery, normalizes context-window stop-reason strings, updates targeted tests, and changes Google Chat/memory-core package metadata plus the lockfile.

## Best Possible Solution

Land a rebased replacement that filters only size-growing hook context when present, preserves policy/system hook effects, gates the retry by loop budget, removes dependency churn, and proves recovery in a real setup.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main can still reach terminal context_overflow after compaction/tool truncation while prompt-build hooks always run, but this review did not exercise a real memory-hook overflow setup.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses all prompt-build hook effects, retries without loop-budget or hook-availability guards, and mixes unrelated dependency churn into the branch.
