## Summary

Keep open, but do not merge as-is. Current main still lacks the bounded suppress-hook overflow retry, while this branch adds retry behavior that can misclassify the terminal failure, can run when no hook context can shrink the prompt, suppresses broader prompt-build effects than described, and carries unrelated package/lockfile churn.

## What This Changes

This PR adds a suppress-prompt-hook retry after embedded PI context-overflow recovery, expands context-window stop-reason normalization coverage, and changes Google Chat and memory-core plugin package/lockfile metadata.

## Best Possible Solution

Land a focused rework or replacement that adds one bounded overflow fallback only when queued or hook-provided context can plausibly affect prompt size, preserves compaction_failure and final context_overflow semantics, keeps non-context prompt-build effects intact, and removes unrelated package metadata changes.

Yes. A high-confidence unit reproduction is to make runEmbeddedAttempt return a context-overflow promptError while compaction returns nothing to compact and no oversized tool results exist; current main's overflow-compaction loop test covers that one-attempt context_overflow path where the fallback would attach.

## Solution Assessment

No. The submitted branch captures a useful recovery direction, but the implementation should be narrower: check remaining loop budget, gate the fallback on hook/injection availability, preserve systemPrompt and other non-context hook effects, and keep package metadata out of scope.
