## Summary

Keep open. The PR addresses a plausible overflow-recovery bug, but current main only covers part of the stop-reason normalization while the suppress-hook retry remains unmerged and the proposed implementation still has correctness, security, unrelated lockfile churn, and missing real-behavior-proof blockers.

## What This Changes

The branch adds a one-attempt prompt-hook-suppressed retry for embedded PI context overflow, normalizes context-window stop-reason strings, updates tests, and changes Google Chat/memory-core peer metadata plus the lockfile.

## Best Possible Solution

Land a rebased replacement that preserves non-size hook effects, suppresses only shrinkable hook context when present and loop budget remains, removes package/lockfile churn, and proves recovery in a real setup.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main can still reach terminal context_overflow after compaction/tool truncation while prompt-build hooks always run, but this review did not exercise a real memory-hook overflow setup.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses all prompt-build hook effects, retries without loop-budget or hook-availability guards, and mixes unrelated dependency churn into the branch.
