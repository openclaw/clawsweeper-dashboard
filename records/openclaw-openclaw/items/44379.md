## What This Changes

The PR adds a one-attempt prompt-hook-suppressed context-overflow retry, normalizes context-window stop-reason variants, updates targeted tests, and changes Google Chat/memory-core peer metadata plus pnpm lockfile entries.

## Best Possible Solution

Land a rebased or replacement branch that implements a bounded context-only fallback, preserves non-context hook effects, gates the retry on useful hook context and remaining loop budget, removes package churn, and includes redacted real runtime proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main can still reach terminal context_overflow after compaction/tool truncation while always resolving prompt-build hooks, but the PR does not include real memory-hook overflow proof.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses all prompt-build hook effects, retries without hook-availability or loop-budget guards, and mixes unrelated dependency churn into the branch.
