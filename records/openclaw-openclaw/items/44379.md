The PR adds a one-attempt PI runner context-overflow retry with prompt hook context suppressed, extends context-window stop-reason normalization tests, and changes Google Chat/memory-core peer metadata plus the lockfile.

## Best Possible Solution

Land a focused rework or replacement that adds one bounded hook-context-only overflow fallback, preserves system/policy hook effects and compaction_failure semantics, and removes extension/lockfile churn.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: on current main, an overflow after failed compaction and no oversized tool results reaches the context_overflow return directly in src/agents/pi-embedded-runner/run.ts. I did not run tests because this review is read-only.

## Solution Assessment

No. The core idea is plausible, but the branch should gate the retry on loop budget and meaningful hook/context availability, preserve non-context hook effects, and drop unrelated package changes.
