## Summary

Keep open: the overflow-recovery gap is still plausible, but this PR is not mergeable as-is because part of the stop-reason handling is already on main while the new retry path has budget, hook-policy, unrelated dependency-churn, and proof blockers.

## What This Changes

The branch adds a one-attempt prompt-hook suppression retry after exhausted overflow recovery, normalizes context-window stop-reason text, updates targeted tests, and changes Google Chat/memory-core peer plus lockfile metadata.

## Best Possible Solution

Land a revised contributor branch or maintainer replacement that adds a bounded hook-context-only overflow fallback, preserves system/policy hook effects and compaction_failure semantics, checks loop budget and meaningful hook context, and removes unrelated package churn.

## Reproduction Assessment

Yes, source-reproducible. Current main can still reach the terminal `context_overflow` return after compaction and tool-result truncation cannot recover, though this read-only review did not execute a live model run.

## Solution Assessment

No. The fallback direction is plausible, but this implementation is too broad because it suppresses all prompt-build hook effects, can spend an extra retry without loop budget or useful hook context, and includes unrelated dependency churn.
