## Summary

Keep open: the recovery direction is useful and current main still lacks the suppress-hook fallback, but this PR is not mergeable as-is because the retry is too broad, can misclassify terminal overflow as retry_limit, suppresses non-context hook effects, and includes unrelated package/lockfile churn.

## What This Changes

This PR adds a one-shot suppress-prompt-hook retry for embedded PI context-overflow recovery, expands stop-reason normalization tests, and changes Google Chat/memory-core package plus lockfile metadata.

## Best Possible Solution

Land a focused rework or replacement that adds one bounded overflow fallback only when hook/queued context can affect prompt size, preserves non-context hook policy effects and compaction_failure semantics, removes package churn, and adds a changelog entry.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: mock runEmbeddedAttempt to return context overflow while compaction returns nothing and no oversized tool results exist, matching the current overflow-compaction loop test attachment point. I did not run the test because this review is read-only and pnpm is unavailable in the checkout.

## Solution Assessment

No. The branch points at a reasonable fallback, but the maintainable fix should gate the extra retry on remaining loop budget and meaningful hook/injection availability, preserve non-context hook effects, and avoid unrelated package metadata changes.
