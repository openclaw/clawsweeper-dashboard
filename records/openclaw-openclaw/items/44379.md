## Summary

Keep open. Current main still lacks the PR's suppress-hook overflow retry, but the proposed patch has blocking correctness, proof, prompt-policy, and supply-chain review issues before it should merge.

## What This Changes

The PR adds a one-attempt prompt-hook-suppressed overflow retry, context-window stop-reason normalization tests, and Google Chat/memory-core peer plus lockfile updates.

## Best Possible Solution

Land a revised or replacement branch that adds a bounded context-only hook fallback, preserves system/policy hook effects and compaction_failure semantics, gates on loop budget and useful hook context, removes unrelated package churn, and includes redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can still reach the terminal context_overflow return after compaction and tool-result truncation are exhausted, but this review did not run a live memory-hook overflow setup.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses the whole prompt-build resolver, spends an extra retry without budget or hook-availability gates, and includes unrelated dependency churn.

Full review comments:

- **[P2] Gate the retry on remaining loop budget:** `src/agents/pi-embedded-runner/run.ts:1192-1198`
  - body: This branch schedules another loop iteration after terminal overflow recovery. If overflow happens on the final allowed pass, the next iteration hits the retry-limit guard and returns retry_limit instead of context_overflow guidance and telemetry; only continue when loop budget remains.
  - confidence: 0.9

  - confidence: 0.84

  - body: The new flag is described as suppressing prompt-hook context, but this code skips resolvePromptBuildHookResult entirely. That also drops systemPrompt overrides and legacy prompt-build effects, which may remove policy or behavior unrelated to context size.
  - confidence: 0.86

  - body: This runner recovery PR also changes Google Chat and memory-core peer ranges plus a large pnpm-lock dependency graph. Those changes are unrelated to the overflow fix and should be split or removed from this branch.
