## Summary

Keep open: the bug-fix direction is plausible and current main still has a terminal overflow path, but this PR has blocking correctness/security findings, missing real behavior proof, and stale unrelated package churn.

## What This Changes

The PR adds one PI runner context-overflow retry with prompt-build hooks suppressed, normalizes context-window stop-reason handling, updates targeted tests, and changes Google Chat/memory-core package metadata plus the lockfile.

## Best Possible Solution

Land a revised branch or maintainer replacement that adds a bounded hook-context-only overflow fallback, preserves system/policy hook effects and compaction_failure semantics, gates the retry on loop budget and meaningful hook context, and drops unrelated package churn.

## Reproduction Assessment

Yes. Source inspection shows current main can still reach the terminal context_overflow return after compaction and tool-result truncation cannot recover; this read-only review did not run tests.

## Solution Assessment

No. The fallback direction is plausible, but this implementation is too broad because it suppresses all prompt-build hook effects, can spend an extra retry without loop budget or useful hook context, and includes unrelated dependency churn.
