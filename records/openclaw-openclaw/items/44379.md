## Summary

This PR should stay open because the hook-suppressed recovery behavior is not on current main and the branch still proposes a meaningful behavior change, but the current implementation has blocking correctness and security concerns plus no real behavior proof.

## What This Changes

The PR adds a one-attempt embedded PI context-overflow retry with prompt-build hooks suppressed, expands context-window stop-reason normalization tests, and changes Google Chat/memory-core peer dependency metadata plus the lockfile.

## Best Possible Solution

Land a rebased replacement that suppresses only shrinkable hook-provided context when that can reduce the prompt, preserves policy/system hook effects, removes unrelated dependency churn, and includes redacted real-run proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main still returns context overflow after compaction/tool-result truncation without a hook-suppressed recovery path, but the PR body only lists targeted tests rather than real setup proof.

## Solution Assessment

No. The fallback direction is plausible, but this implementation suppresses all prompt-build hook effects, retries without hook-availability and loop-budget guards, and mixes unrelated dependency churn into the branch.

- **[P2] Gate the suppress retry on remaining loop budget:** `src/agents/pi-embedded-runner/run.ts:1192-1200`
  - body: This schedules another run-loop iteration after terminal overflow recovery. If overflow happens on the last allowed pass, the next iteration hits the retry-limit guard and returns `retry_limit` instead of the intended `context_overflow` guidance and telemetry.
  - confidence: 0.9

  - confidence: 0.88
- **[P2] Skip suppress retry without shrinkable hook context:** `src/agents/pi-embedded-runner/run.ts:1192-1200`
  - body: When no queued injection or prompt-build hook context can shrink the prompt, `suppressPromptHookContext` cannot change the constructed request. This adds another full model call that should fail with the same overflow.
  - confidence: 0.84
- **[P2] Drop unrelated dependency churn:** `extensions/googlechat/package.json:11`
  - body: This runner recovery PR also changes Google Chat and memory-core peer dependency metadata plus a broad lockfile dependency graph. Split or remove that unrelated supply-chain review scope before this branch is considered.
