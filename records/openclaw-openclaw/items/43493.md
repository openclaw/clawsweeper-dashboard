## Summary

Keep open: the feature still targets a real custom-provider setup gap and has usable real-behavior proof, but the patch has a P2 correctness issue in the API adapter prompt before it should merge.

## What This Changes

This PR adds a custom-provider-only model metadata wizard step during model configuration, with helpers and tests for updating configured model entries.

## Best Possible Solution

Fix the Auto API-adapter path so it actually removes a model-level `api` override or relabel that choice as keeping the current adapter, then land the setup prompt while leaving broader auto-detection/runtime-warning work to the related issue.

## Reproduction Assessment

Yes for the review finding: source inspection shows an existing custom model with a per-model `api` override will keep that override after the user selects Auto, because the update omits `api` and the merge preserves `entry.model.api`. I did not execute the interactive wizard locally in this read-only review.

## Solution Assessment

No as-is. The setup-time metadata prompt is a good narrow direction, but the API adapter option must either clear the explicit model override when Auto is selected or avoid promising provider inheritance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
