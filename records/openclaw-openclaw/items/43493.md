## Summary

Keep open: current main still lacks the interactive custom-provider metadata prompt, but this PR is not merge-ready because the branch adds a useful flow while missing current model-reference, API-inheritance, and input-modality contracts.

## What This Changes

The PR adds a `model-metadata` command helper, unit tests, and a custom-provider-only call from `promptAuthConfig` so users can edit model metadata during `openclaw configure --section model`.

## Best Possible Solution

Rebase the branch onto current main, integrate the prompt after the current custom-provider setup path, reuse the shared model-ref/model-key contract, preserve the full `ModelDefinitionConfig` metadata surface that the wizard edits, and add focused tests for provider-prefixed IDs, API inheritance/clearing semantics, and media input modalities.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of the configure/custom-provider path, and the PR defects are reproducible with small unit cases for provider-prefixed native model IDs, existing model-level API overrides, and audio/video input metadata.

## Solution Assessment

No. The product direction is useful, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, labels API inheritance in a way that cannot clear an override, and narrows valid model input modalities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key and searches only for the suffix. For a custom provider whose native model id already starts with the provider prefix, the primary model can be stored as `custom/model`, while the configured model id is also `custom/model`; this code searches for `model` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge spreads the existing model before the update, so any previous model-level `api` value survives. The wizard offers provider inheritance but cannot actually remove a stale per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `audio` and `video`, but this prompt only offers `text` and `image` and then overwrites the model input list. Editing a media-capable custom model can drop valid existing modalities when the user only meant to change token metadata.
  - confidence: 0.78
