## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve all existing input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` and the config schema accept `audio` and `video`, but this prompt only offers Text/Image and writes that selected list back to the existing model. Editing an existing custom model with audio/video metadata silently drops those persisted capabilities; preserve unoffered values or expose the full current contract before saving.
  - confidence: 0.94
- **[P2] Clear model API override when Auto is selected:** `src/commands/model-metadata.ts:122-133`
  - body: The Auto branch leaves `updated.api` unset, and the later merge keeps `...entry.model`, so an existing model-level `api` survives even though the prompt says Auto should inherit from the provider. Add explicit clear/delete semantics and a regression test for switching an existing override back to Auto.
  - confidence: 0.9
- **[P3] Use strict decimal parsers for metadata numbers:** `src/commands/model-metadata.ts:93-100`
  - body: These parsers use `Number(raw.trim())`, which accepts forms like `0x100000` and `1e6` even though token-count CLI seams use strict base-10 integer parsing. Persisted model setup metadata should reject surprising numeric coercions before writing context, max-token, or cost values.
  - confidence: 0.78
