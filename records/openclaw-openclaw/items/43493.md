## Summary

Keep open: current main still does not offer setup-time contextWindow/maxTokens metadata editing for custom providers, and this PR contains useful unique work, but the PR head still has a P2 correctness blocker around the Auto API adapter choice.

## What This Changes

The PR adds a custom-provider-only model metadata prompt during `openclaw configure --section model` for display name, API adapter, reasoning, input modalities, context window, max output tokens, and pricing, plus focused tests.

## Best Possible Solution

Fix the Auto adapter inheritance semantics with regression coverage, then land or replace the setup-time metadata prompt while leaving broader auto-detection and runtime-warning work to https://github.com/openclaw/openclaw/issues/48338.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the saved model preserves `entry.model.api`, and current main treats absence as provider inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a narrow maintainable direction, but Auto must clear the model-level adapter override or be relabeled before merge.
