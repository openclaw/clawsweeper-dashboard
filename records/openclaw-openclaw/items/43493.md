## Summary

Keep open: this targets a real unresolved custom-provider setup gap and has after-fix CLI proof, but the PR should not merge until the API adapter `Auto` option can actually clear an existing model-level override.

## What This Changes

This PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for editing configured model metadata.

## Best Possible Solution

Land this setup-time metadata prompt after fixing model-level API override clearing and adding regression coverage; keep broader fallback warnings or auto-detection tracked by https://github.com/openclaw/openclaw/issues/48338 and related follow-ups.

## Reproduction Assessment

Yes for the review finding: source inspection shows choosing `Auto` leaves `updated.api` absent while the merge preserves `entry.model.api`. The PR's screenshot also demonstrates the intended custom-provider prompt path exists after the patch.

## Solution Assessment

No, not as-is. The setup-time metadata prompt is the right narrow direction, but `Auto (inherit from provider)` needs deletion semantics for model-level `api` overrides before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84
