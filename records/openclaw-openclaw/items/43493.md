## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve unoffered input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` and the config schema accept `audio` and `video`, but this prompt only offers Text/Image and writes the selected list back to the model. Editing an existing custom model with audio/video metadata can silently drop those capabilities, so preserve unoffered values or expose the full current contract before saving.
  - confidence: 0.94
- **[P2] Clear the API override when Auto is selected:** `src/commands/model-metadata.ts:122-133`
  - body: The Auto branch leaves `updated.api` unset, and the later merge keeps `...entry.model`, so an existing model-level `api` survives even though the prompt says Auto should inherit from the provider. Add explicit clear/delete semantics and a regression test for switching an existing override back to Auto.
  - confidence: 0.9
- **[P3] Reject non-decimal token counts:** `src/commands/model-metadata.ts:93-100`
  - body: The new token prompts parse with `Number(raw.trim())`, which accepts forms like `0x100000` and `1e6` even though the UI asks for integer token counts and nearby CLI seams use strict decimal integer parsing. Use the shared strict positive-integer parser for `contextWindow` and `maxTokens` before persisting setup metadata.
  - confidence: 0.78
