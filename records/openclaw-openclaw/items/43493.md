## Summary

Keep open: this PR still adds a useful custom-provider metadata prompt that current main does not provide, but the branch has concrete merge-blocking bugs in model lookup and metadata preservation.

## What This Changes

The PR adds a custom-provider-only `openclaw configure --section model` metadata wizard and unit tests for editing configured model display name, API adapter, reasoning, input modalities, context window, max tokens, and pricing.

## Best Possible Solution

Land an updated metadata wizard that reuses shared model-reference handling, preserves the full `ModelDefinitionConfig` metadata surface, accurately clears or represents API inheritance, and has focused regression coverage on current main.

## Reproduction Assessment

Yes for review purposes. Source inspection shows current main lacks the metadata prompt while the PR adds it, and the PR code has concrete failure paths for provider-prefixed model IDs, API override clearing, and audio/video modality preservation.

## Solution Assessment

No. The feature direction is reasonable, but this implementation should not merge until it uses existing model reference semantics and updates metadata without losing supported fields or stale override state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use shared model refs for primary lookup:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key on `/` and searches for only the suffix. For a provider whose native model ID already starts with the provider prefix, current `modelKey` keeps the ID as `provider/model`, so this searches for `model` and skips the metadata prompt; use the shared model-ref/key semantics instead of ad hoc splitting.
  - confidence: 0.88
- **[P2] Clear model API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge keeps `...entry.model`, so an existing per-model `api` override survives. The wizard should either remove the old override when Auto is selected or avoid offering inheritance that cannot be saved.
  - confidence: 0.86
- **[P2] Preserve all existing input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `audio` and `video`, but this prompt only offers `text` and `image` and then overwrites the input list. Editing a media-capable custom model can silently drop valid existing modalities when the user only intended to change token metadata.
  - confidence: 0.82
