## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model` with helper tests for editing configured model limits, pricing, API adapter, reasoning, and input modalities.

## Best Possible Solution

Repair the Auto/inheritance edge case in this PR, then let maintainers decide whether to land this focused setup prompt while leaving broader detection and warning behavior tracked separately.

## Reproduction Assessment

Yes for the review finding: source inspection shows an existing `model.api` survives when the prompt returns no `api` for the Auto choice because the merge preserves `...entry.model`. I did not run the wizard or tests in this read-only review.

## Solution Assessment

No as written. The prompt is a reasonable narrow implementation direction, but selecting Auto must actually clear a model-level API override or the UI cannot deliver its stated provider-inheritance behavior.
