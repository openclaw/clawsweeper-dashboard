## Summary

Keep open: current main still lacks the custom-provider metadata prompt, and the PR remains a useful implementation candidate, but it has a concrete PR-introduced correctness bug around the API adapter Auto option.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, with helper functions and tests for editing configured model definitions.

## Best Possible Solution

Keep the custom-provider metadata prompt direction, fix Auto API inheritance with regression coverage, then let maintainer review decide the final configure/onboard scope.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows Auto writes no `api`, the save path preserves `entry.model.api`, and current main only inherits from the provider when the model-level API field is absent.

## Solution Assessment

No as-is. The narrow fix is to make `Auto (inherit from provider)` remove any existing per-model `api` override before saving, or relabel the option as keep-current behavior and test that explicitly.
