## Summary

Keep open: current main still lacks the requested interactive custom-provider metadata prompt, and this PR is meaningful implementation work, but the patch is not merge-ready because the new Auto API-adapter choice cannot actually clear an existing model-level adapter override.

## What This Changes

The PR adds a `model-metadata` command helper and tests, then calls it after `custom-api-key` setup in `openclaw configure --section model` to edit existing custom-provider model metadata fields.

## Best Possible Solution

Fix the adapter-inheritance bug, add focused regression coverage for existing custom models with a model-level `api`, and have maintainers decide whether this configure-only interactive prompt is the accepted scope for the broader onboarding metadata request.

## Reproduction Assessment

Yes for the PR defect: source inspection shows Auto leaves `result.api` unset while the later merge preserves `entry.model.api`, so an existing override cannot be cleared through the new prompt. I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction is plausible, but the current patch needs the Auto adapter semantics fixed and maintainer confirmation that `configure --section model` is the right scope for the broader custom-provider metadata UX.
