## Summary

Keep open: current main still lacks the setup-time custom-provider metadata prompt, and this PR remains a useful implementation candidate, but it has a concrete P2 correctness bug in the API adapter Auto option before merge.

## What This Changes

The PR adds a custom-provider-only model metadata wizard step to `openclaw configure --section model` and new tests for collecting and updating configured model metadata.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option clear model-level overrides or relabel it so the saved config matches the UX contract.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows Auto leaves `updated.api` absent, the save path preserves `entry.model.api`, and current catalog loading only inherits the provider adapter when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable fit for the custom-provider setup UX, but the API adapter control needs a narrow persistence fix or clearer label before merge.
