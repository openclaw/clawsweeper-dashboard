## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve unoffered input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: The editor only offers Text/Image and then writes `result.input`, so saving an existing model with accepted audio/video metadata rewrites it to the narrower prompted set. Current config types, schema, and catalog tests still accept or preserve audio/video, so this can silently drop model capabilities during reconfigure.
  - confidence: 0.93
- **[P2] Clear the API override when Auto is selected:** `src/commands/model-metadata.ts:122-133`
  - body: Selecting Auto leaves `result.api` unset, and the later merge spreads `...entry.model` before `...updated`, so any existing model-level `api` override remains. The prompt can say the model inherits from the provider while the stored config still routes through the old adapter.
  - confidence: 0.91
- **[P3] Use strict decimal parsing for numeric prompts:** `src/commands/model-metadata.ts:93-100`
  - body: `Number(raw.trim())` still accepts non-decimal forms such as `0x10000` and scientific notation, even though token prompts expect an integer token count and other CLI seams use strict positive-integer parsing. Use the shared strict parser or equivalent validation before persisting context, max-token, and cost values.
  - confidence: 0.78
