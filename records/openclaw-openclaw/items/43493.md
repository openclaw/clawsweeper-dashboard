## Summary

Keep open and request a focused fix: the feature is still useful and not implemented on current main, but the PR's Auto adapter option currently preserves an existing model-level API override instead of restoring provider inheritance.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and tests for editing model definition fields during setup.

## Best Possible Solution

Keep the guarded custom-provider metadata prompt, but make Auto genuinely clear the model-level adapter override and leave broader auto-detection/runtime wording to the open custom-provider context UX issue.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the final merge preserves `entry.model.api`, and current main treats absence as provider-level inheritance.

## Solution Assessment

No as-is. The setup-time metadata prompt is a narrow maintainable direction, but the Auto adapter semantics need a regression-covered fix before merge.
