## Summary

Keep open: current main does not yet implement this metadata prompt, and the PR has sufficient real-behavior proof, but PR head still has a P2 correctness bug where the Auto API choice cannot return an existing model-level API override to provider inheritance.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model` plus helper coverage for editing configured model definitions.

## Best Possible Solution

Keep the custom-provider metadata prompt direction, fix API inheritance semantics with regression coverage, and align the final hook with the current configure/onboarding flow before maintainer review.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows Auto writes no `api`, the save path preserves `entry.model.api`, and current main only inherits from the provider when the model-level API field is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` remove an existing per-model `api` override before saving, or change the label and tests to state that Auto keeps the current override.
