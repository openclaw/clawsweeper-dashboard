## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve existing input modalities:** `src/commands/model-metadata.ts:149`
  - body: This prompt only offers `text` and `image`, then saves the returned choices as the entire `input` array. Existing model config supports `audio` and `video`, so editing any other metadata on a custom model can silently drop those capabilities from persisted config; preserve unsupported existing values unless the user explicitly removes them.
  - confidence: 0.9
- **[P2] Clear model API when Auto is selected:** `src/commands/model-metadata.ts:131-133`
  - body: Choosing `Auto (inherit from provider)` leaves `result.api` unset, but the final merge spreads `entry.model` first, so any existing model-level `api` override survives. That means the new wizard cannot actually return a model to provider-level routing even though the UI says it can.
  - confidence: 0.88
