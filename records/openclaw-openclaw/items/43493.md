## What This Changes

The PR adds a custom-provider-only metadata wizard step to `openclaw configure --section model`, plus helpers and tests for editing model name, API adapter, reasoning, input, context/max tokens, and cost.

## Best Possible Solution

Fix the API Auto clearing bug, add a focused regression test, then land this as the setup-focused improvement while tracking auto-detection and runtime warning copy separately if maintainers want them.

## Reproduction Assessment

Yes. Source inspection of the PR shows the review finding: choosing Auto leaves `result.api` unset and the later merge preserves any existing model-level `api` value.

## Solution Assessment

No, not as-is. The setup-prompt direction is reasonable, but the API adapter Auto option needs an explicit clear/delete path so the UI can actually restore provider inheritance.
