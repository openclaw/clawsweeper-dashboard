## Summary

Keep open: current main still lacks the guided custom-provider metadata prompt, and this PR remains a useful implementation candidate, but it has a blocking API-inheritance bug before merge.

## What This Changes

The PR adds a custom-provider-only model metadata wizard step to `openclaw configure --section model`, with helpers and tests for display name, API adapter, reasoning, input modalities, token limits, and pricing.

## Best Possible Solution

Fix the API-inheritance clearing bug, then land or replace this branch as the implementation path for the custom-provider metadata prompt tracked by https://github.com/openclaw/openclaw/issues/61716.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. The blocking PR behavior is source-reproducible with a configured model that already has a model-level `api` override, then selecting `Auto (inherit from provider)`.

## Solution Assessment

No. The direction is useful, but selecting `Auto (inherit from provider)` must actually remove an existing model-level `api` override before the implementation is mergeable.
