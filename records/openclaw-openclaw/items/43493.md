## Summary

Keep open: current main still lacks the custom-provider metadata prompt, and the PR remains useful but has a localized correctness blocker in the API adapter option before it should merge.

## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for editing model name, API adapter, reasoning, input modalities, context/max token limits, and pricing.

## Best Possible Solution

Keep the metadata prompt direction, fix the API adapter inheritance semantics with regression coverage, then rebase and let maintainers decide whether to extend the same helper into the shared onboarding path.

## Reproduction Assessment

Yes for the review finding. Source inspection shows Auto leaves `updated.api` absent while the save merge preserves `entry.model.api`, and current main treats missing `model.api` as provider inheritance.

## Solution Assessment

No as-is. The feature direction is reasonable, but `Auto (inherit from provider)` must either clear the per-model API override or be renamed to describe keep-current behavior.
