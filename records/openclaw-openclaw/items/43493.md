## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve all supported input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: The prompt only offers `text` and `image`, then writes that returned selection back to `result.input`. Current config and provider catalog contracts also accept audio/video, so editing an existing custom model with those modalities can narrow persisted metadata and change routing behavior.
  - confidence: 0.91
- **[P2] Make Auto clear model API overrides:** `src/commands/model-metadata.ts:131-133`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge starts from `entry.model`, so an existing model-level `api` override survives. That means the UI offers provider inheritance but cannot actually return a model to provider-level routing.
  - confidence: 0.88
- **[P2] Preserve tiered pricing metadata:** `src/commands/model-metadata.ts:210-215`
  - body: When cost editing is enabled, the prompt rewrites `cost` with only the four flat rate fields. Existing model configs can carry `cost.tieredPricing`, so accepting the default pricing flow can silently drop tiered cost metadata and skew usage/cost reporting.
  - confidence: 0.82
