## Summary

Keep open: this PR is still the focused implementation candidate for custom-provider metadata prompts, but it has a correctness bug in the API-adapter Auto path that should block merge.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to model configuration, plus helpers and tests for editing configured model entries.

## Best Possible Solution

Fix or relabel the Auto adapter option so it matches provider-inheritance semantics, add regression coverage, then land the setup prompt while leaving broader context-window UX to the related issue.

## Reproduction Assessment

Yes for the review finding: source inspection shows an existing custom model with a per-model `api` override keeps that override after selecting Auto, because the update omits `api` and the merge preserves `entry.model.api`. I did not run the interactive wizard in this read-only review.

## Solution Assessment

No as-is. The setup-time metadata prompt is a good narrow direction, but the API adapter option must either clear the explicit model override when Auto is selected or avoid promising provider inheritance.
