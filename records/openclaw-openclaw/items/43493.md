## Summary

Keep open: current main still lacks the custom-provider metadata prompt, and the PR remains a useful implementation candidate, but the current patch has a correctness bug in the advertised Auto API inheritance path.

## What This Changes

The PR adds a custom-provider-only interactive model metadata prompt to `promptAuthConfig`, plus helper functions and unit tests for editing model definition fields.

## Best Possible Solution

Repair the API-inheritance semantics, fit the hook into the current custom-provider flow, and keep broader onboard/auto-detection UX decisions coordinated with the related open issues.

## Reproduction Assessment

Yes. Source inspection of the PR head shows that selecting Auto leaves `updated.api` absent while the later merge preserves `entry.model.api`; current main treats absence of `model.api` as provider inheritance.

## Solution Assessment

No as-is. The metadata prompt is a plausible narrow feature, but Auto must actually clear the model-level adapter override and maintainers should align the hook with the current configure/onboard split.
