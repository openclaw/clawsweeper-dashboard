## Summary

Keep open: the feature is still useful and current main does not implement this custom-provider metadata wizard, but the PR still has a P2 data-flow bug where the API adapter Auto option cannot actually return a model to provider-level inheritance.

## What This Changes

The PR adds a custom-provider-only metadata wizard after `openclaw configure --section model`, plus helpers and tests for editing display name, API adapter, reasoning, modalities, token limits, and pricing.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option actually clear model-level overrides for provider inheritance, or relabel it to preserve the current adapter intentionally, with focused regression coverage.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the potential merge commit shows the Auto branch omits `updated.api`, while the save merge preserves `entry.model.api`. No live CLI run is needed to establish that data-flow bug.

## Solution Assessment

No as-is. The feature direction is useful and scoped, but the API adapter control must be corrected so the prompt label and persisted config behavior agree.
