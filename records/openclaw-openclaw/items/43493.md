## Summary

Keep open: current main still lacks this custom-provider metadata prompt, and the PR has useful feature work, but PR head still has a source-proven correctness bug in the API adapter selection path.

## What This Changes

Adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for editing stored `ModelDefinitionConfig` fields.

## Best Possible Solution

Keep the metadata prompt direction, integrate it with the current custom-provider configure path, fix API adapter inheritance semantics, and decide separately whether the same helper belongs in onboarding.

## Reproduction Assessment

Yes for the review finding. Source inspection at PR head shows `Auto` writes no `api`, the merge preserves `entry.model.api`, and current main only inherits from the provider when model-level `api` is absent.

## Solution Assessment

No as-is. The narrow fix is to make `Auto (inherit from provider)` clear the per-model API override before saving, or relabel it as keep-current behavior and add regression coverage.
