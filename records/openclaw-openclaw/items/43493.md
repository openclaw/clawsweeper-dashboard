Keep open: the feature is still useful and not implemented on current main, but this PR has a blocking correctness issue in the new Auto adapter flow before it should merge.

## What This Changes

This PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, plus helper functions and tests for editing model display name, adapter, reasoning, inputs, token limits, and pricing.

## Best Possible Solution

Fix the Auto adapter semantics with regression coverage, then land or replace the guarded custom-provider metadata prompt while leaving broader auto-detection and runtime wording to https://github.com/openclaw/openclaw/issues/48338.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the final merge preserves `entry.model.api`, and current main treats absence as provider-level inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a narrow maintainable direction, but Auto must clear the model-level adapter override or be relabeled before merge.
