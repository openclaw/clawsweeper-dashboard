## Summary

Keep this PR open because current main still lacks a custom-provider metadata prompt and the PR addresses a real setup/DX gap, but it is not mergeable as-is due to a persisted API-adapter override bug.

## What This Changes

The PR adds a custom-provider-only metadata wizard to `openclaw configure --section model`, plus helper functions and tests for editing model display name, API adapter, reasoning, modalities, token limits, and pricing.

## Best Possible Solution

Keep the custom-provider metadata wizard direction, but make the API adapter option either truly clear model-level overrides for provider inheritance or relabel it to preserve the current adapter intentionally, then cover that behavior with a focused regression test.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR diff shows the Auto branch omits `updated.api`, while the save merge preserves `entry.model.api`. No live CLI run was needed to establish this data-flow bug.

## Solution Assessment

No as-is. The feature direction is useful, but the API adapter control needs a narrow correction so the UI label and persisted config behavior agree.
