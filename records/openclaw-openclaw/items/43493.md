## Summary

Keep this PR open: the feature still fills a real gap on current main, but the proposed wizard has a blocking correctness bug in the `Auto (inherit from provider)` API adapter path.

## What This Changes

The PR adds a custom-provider model metadata wizard to `openclaw configure --section model`, with helpers and tests for editing display name, API adapter, reasoning, input modalities, token limits, and pricing.

## Best Possible Solution

Keep the custom-provider metadata flow, but make the API adapter option behave according to its label and cover that behavior with regression tests before merge.

## Reproduction Assessment

Yes for the review finding. Source inspection against the PR head shows `Auto` leaves `updated.api` absent, the save merge preserves `entry.model.api`, and current catalog loading only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` remove an existing per-model `api` override, or change the label and tests to state that Auto keeps the current override.
