## Summary

Keep open: the feature is still not present on current main and addresses the custom-provider metadata setup UX, with usable CLI screenshot proof, but the patch has a blocking adapter-inheritance bug before merge.

## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, updates the configured primary model metadata, and adds unit coverage for the new helpers and skip paths.

## Best Possible Solution

Land a rebased custom-provider metadata setup flow that edits existing model metadata without misleading no-op choices and preserves provider/model API inheritance semantics.

## Reproduction Assessment

Yes for the PR finding: source inspection shows choosing Auto leaves `result.api` undefined, and the later merge preserves an existing `entry.model.api`. For the feature itself, the PR body and inspected screenshot provide after-fix CLI behavior proof.

## Solution Assessment

No as-is: adding the prompt in the custom-provider setup path is the right general surface, but the adapter selection needs explicit clearing semantics before this is the best maintainable implementation.
