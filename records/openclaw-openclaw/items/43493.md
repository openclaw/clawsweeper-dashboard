## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve all existing input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: The prompt only offers `text` and `image`, but the current model config contract also accepts `audio` and `video`. Editing an existing model with those modalities will save back only the offered choices, silently removing capabilities; preserve unoffered values or expose every supported modality before writing `input`.
  - confidence: 0.9
- **[P2] Let Auto clear model-level API overrides:** `src/commands/model-metadata.ts:122-133`
  - body: Selecting Auto omits `result.api`, and the later merge preserves `...entry.model`, so an existing per-model `api` override remains even though the prompt says it will inherit from the provider. Use an explicit clear path or change the UI semantics.
  - confidence: 0.88
- **[P3] Reject non-decimal token counts:** `src/commands/model-metadata.ts:93-96`
  - body: `Number(raw.trim())` accepts inputs such as hex notation for context and max-token fields, which is inconsistent with the strict token-count parsing used elsewhere. Use the strict decimal integer helper so the prompt stores only the format users are asked to enter.
  - confidence: 0.72
