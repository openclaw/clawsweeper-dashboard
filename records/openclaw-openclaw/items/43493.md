## What This Changes

This PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, covering display name, API adapter, reasoning, input modalities, context window, max output tokens, optional pricing, and unit tests.

## Best Possible Solution

Land this only after the API inheritance semantics are fixed or renamed, regression-tested on the current configure flow, and scoped deliberately against the broader onboarding metadata request.

## Reproduction Assessment

Not applicable as an original bug report because this is a feature PR. The blocking defect is high-confidence from source: start with a custom model that has `models.providers.<id>.models[].api`, select `Auto (inherit from provider)`, and the merge keeps the old model-level value.

## Solution Assessment

No, not as currently implemented. The feature direction is reasonable, but the API adapter prompt must either truly clear model-level overrides for provider inheritance or stop labeling that choice as inheritance.
