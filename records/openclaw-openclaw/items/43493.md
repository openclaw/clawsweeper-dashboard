## Summary

Keep open: current main still lacks the custom-provider metadata prompt, but the PR head still has a merge-blocking API adapter inheritance bug. The real behavior proof is sufficient and the security pass is clear, so a narrow repair is appropriate.

## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, a new model-metadata helper, and unit tests for editing model name, API adapter, reasoning, input modalities, token limits, and pricing.

## Best Possible Solution

Keep the metadata prompt direction, fix API inheritance semantics with regression coverage, then let maintainers decide whether the helper should also move into the shared onboarding path.

## Reproduction Assessment

Yes for the review finding. Source inspection shows Auto leaves `updated.api` absent while the save merge preserves `entry.model.api`, and current main only inherits the provider API when `model.api` is absent.

## Solution Assessment

No as-is. The feature direction is reasonable, but `Auto (inherit from provider)` must either clear the per-model API override or be renamed to describe keep-current behavior.
