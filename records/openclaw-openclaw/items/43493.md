## Summary

Keep open: current main still lacks an interactive custom-provider metadata step, and this PR is a plausible implementation candidate, but the PR head still has a source-proven API inheritance bug that should block merge.

## What This Changes

The PR adds a custom-provider model metadata wizard to `openclaw configure --section model`, plus helper functions and tests for editing model limits, API adapter, modalities, reasoning, and pricing.

## Best Possible Solution

Keep this PR open, fix the API inheritance semantics with regression coverage, then decide whether the same metadata collection belongs in the shared custom-provider onboarding path.

## Reproduction Assessment

Yes for the review finding. Source inspection shows `Auto` leaves `updated.api` absent, the save merge preserves `entry.model.api`, and current catalog loading only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` remove an existing per-model `api` override, or change the label and tests to state that Auto keeps the current override.
