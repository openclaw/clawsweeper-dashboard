## Summary

Keep open. Current main still lacks an interactive metadata step for custom-provider setup, and the PR is useful but not mergeable yet because the new helper mishandles provider-prefixed model IDs, stale model-level API overrides, media input modalities, and the required changelog entry.

## What This Changes

The PR adds a custom-provider-only CLI prompt for editing model metadata during `openclaw configure --section model`, plus helper tests for the new metadata flow.

## Best Possible Solution

Rebase the branch onto current main, keep the prompt custom-provider-only, reuse the shared model-ref helpers, preserve the full `ModelDefinitionConfig` surface being edited, allow model-level API inheritance to clear stale overrides, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Current-main source inspection shows the custom-provider configure path writes fallback metadata and has no metadata prompt, while the PR defects are reproducible with small unit cases for provider-prefixed native model IDs, existing model-level API overrides, and audio/video input metadata.

## Solution Assessment

No. The product direction is sound, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, offers inheritance without clearing stale overrides, and can drop valid media modalities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key and searches only for the suffix. For custom providers whose native model ID already starts with the provider prefix, the primary key and stored model ID can both be `custom/model`, so this searches for `model` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge spreads the existing model before the update, so any previous model-level `api` value survives. The wizard offers provider inheritance but cannot actually remove a stale per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `audio` and `video`, but this prompt only offers `text` and `image` and then overwrites the model input list. Editing a media-capable custom model can drop valid existing modalities when the user only meant to change token metadata.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/configure.gateway-auth.ts:135-139`
  - body: This adds a user-facing configure feature, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a single-line active-release changelog entry for user-facing `feat` changes, with contributor credit such as `Thanks @IdrisHanafi`.
  - confidence: 0.9
