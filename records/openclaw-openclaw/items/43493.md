## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve all input modalities when saving metadata:** `src/commands/model-metadata.ts:149`
  - body: The model config contract accepts `audio` and `video`, but this prompt only offers Text/Image and then overwrites `result.input` with that limited selection. Editing an existing custom model that already declares audio or video can silently drop those capabilities, changing routing/media behavior after a setup-only edit. Include every supported modality here or preserve existing values that the editor does not expose.
  - confidence: 0.87
- **[P2] Let Auto remove model API overrides:** `src/commands/model-metadata.ts:131-133`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, but `promptModelMetadataForPrimary` merges it over `entry.model`, so an existing per-model `api` override is retained. The UI promises provider inheritance, yet users cannot clear the override through the new wizard; carry an explicit clear sentinel or delete `api` before saving.
  - confidence: 0.84
