## Summary

Keep open. Current main improved custom-provider defaults but does not implement this PR's interactive metadata prompt, and the PR still has a narrow correctness issue around the API adapter Auto option before it should merge.

## What This Changes

The PR adds a custom-provider-only model metadata wizard step to `openclaw configure --section model`, plus helper tests for updating model definition fields.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option either clear model-level overrides or use wording that matches the persisted behavior.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection of PR head shows Auto leaves `updated.api` unset, the save merge preserves `entry.model.api`, and current main only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable custom-provider setup improvement, but the API adapter control needs a narrow persistence fix or clearer label before merge.
