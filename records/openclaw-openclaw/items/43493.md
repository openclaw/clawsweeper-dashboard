## Summary

Keep open. Current main still has the custom-provider metadata UX gap, and this PR is a useful implementation path, but the proposed patch has merge-blocking correctness issues in model-ref matching, API override clearing, and input modality preservation.

## What This Changes

The PR branch adds a model metadata prompt helper with tests and calls it after custom-provider setup in `promptAuthConfig` so users can edit context window, max tokens, reasoning, input modalities, API adapter, and pricing during `openclaw configure --section model`.

## Best Possible Solution

Rebase the branch onto current main, keep the prompt custom-provider-only, reuse the shared model-ref/model-key helpers, preserve the full `ModelDefinitionConfig` surface being edited, add the required changelog entry, and cover the edge cases with focused command tests.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of the custom-provider configure path, and the PR defects are reproducible with small unit cases for provider-prefixed native model ids, existing model-level API overrides, and audio/video input metadata.

## Solution Assessment

No. The product direction is sound, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, offers provider API inheritance without clearing stale model overrides, and can drop valid media input modalities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key and searches only for the suffix. If a custom provider stores a native model id that already starts with the provider prefix, the primary key can be `custom/model` while the configured model id is also `custom/model`; this searches for `model` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge spreads the existing model before the update, so any previous model-level `api` value survives. The wizard offers provider inheritance but cannot actually remove a stale per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `audio` and `video`, but this prompt only offers `text` and `image` and then overwrites the model input list. Editing a media-capable custom model can drop valid existing modalities when the user only meant to change token metadata.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/configure.gateway-auth.ts:138-140`
  - body: This adds a user-facing configure feature, but the PR does not touch `CHANGELOG.md`. OpenClaw policy requires a single-line active-release changelog entry for user-facing `feat` changes, with contributor credit such as `Thanks @IdrisHanafi`.
  - confidence: 0.9
