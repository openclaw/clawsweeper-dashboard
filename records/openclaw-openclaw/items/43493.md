## Summary

Keep open. The requested custom-provider metadata prompt is not implemented on current main, and this PR is a plausible implementation path, but the latest PR head still has a blocking persistence bug in the API adapter control before it should merge.

## What This Changes

The PR adds a custom-provider-only model metadata prompt after `openclaw configure --section model`, with helpers and unit tests for editing model fields such as API adapter, reasoning, input modalities, context window, max tokens, and pricing.

## Best Possible Solution

Keep the metadata wizard direction, fix the API adapter Auto persistence mismatch with regression coverage, then decide separately whether to adapt the same helper into the broader onboarding flow.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection of PR head shows Auto leaves `updated.api` unset, the save merge preserves `entry.model.api`, and current main only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The feature direction is reasonable, but the proposed API adapter control must either clear model-level overrides when Auto is selected or change the label to match the persisted behavior.
