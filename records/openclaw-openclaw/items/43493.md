## Summary

Keep open. The feature is still useful, but the current PR has a source-reproducible blocker where the API adapter prompt says provider inheritance while preserving an existing model-level adapter override.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model` plus tests for editing existing `ModelDefinitionConfig` fields.

## Best Possible Solution

Land this only after the API inheritance semantics are fixed or relabeled, regression-tested on the current merge result, and deliberately scoped against the broader onboarding metadata request.

## Reproduction Assessment

Not applicable as an original bug report because this is a feature PR. The blocking defect is source-reproducible: start with a custom model that has `models.providers.<id>.models[].api`, select `Auto (inherit from provider)`, and the merge keeps the old model-level value.

## Solution Assessment

No, not as currently implemented. The feature direction is reasonable, but the API adapter prompt must either truly clear model-level overrides for provider inheritance or stop labeling that choice as inheritance.
