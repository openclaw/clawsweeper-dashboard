## Summary

Keep open: current main still lacks this custom-provider metadata prompt, but the PR is not merge-ready because its Auto API adapter option cannot clear an existing model-level override.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, with helper functions and unit coverage for editing configured model definitions.

## Best Possible Solution

Keep the metadata prompt direction, fix Auto API inheritance with regression coverage, then decide whether the helper should be shared with onboarding rather than only `configure --section model`.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows Auto writes no `api`, the merge preserves `entry.model.api`, and current main only inherits from the provider when the model-level API field is absent.

## Solution Assessment

No as-is. The narrow repair is to make `Auto (inherit from provider)` remove a per-model API override before saving, or relabel it as keep-current behavior and cover that choice with a regression test.
