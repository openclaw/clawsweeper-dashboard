## Summary

Keep open: current main still lacks this custom-provider metadata prompt, and the PR remains useful, but the branch has a blocking correctness bug in the API adapter prompt before it should merge.

## What This Changes

The PR adds a custom-provider model metadata wizard to `openclaw configure --section model`, plus tests for editing display name, API adapter, reasoning, input modalities, context window, max output tokens, and pricing.

## Best Possible Solution

Keep the metadata prompt direction, make the Auto API adapter choice either remove model-level overrides or relabel it accurately, then decide whether the same helper should be shared with custom-provider onboarding.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `result.api` unset, the save merge preserves `entry.model.api`, and current catalog loading only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` remove an existing per-model API override, or change the label and tests to state that Auto keeps the current override.
