## Summary

Keep open. Current main still lacks this interactive custom-provider metadata prompt, so the PR is not obsolete, but PR head has a blocking correctness issue where selecting Auto for the API adapter cannot clear an existing model-level API override.

## What This Changes

The branch adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for collecting and updating configured model metadata.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance semantics with regression coverage, then let maintainers decide whether the same helper should be adapted into onboarding.

## Reproduction Assessment

Yes for the review finding. Source inspection at PR head shows Auto writes no `api`, the merge preserves `entry.model.api`, and current main only inherits from the provider when model-level `api` is absent.

## Solution Assessment

No as-is. The narrow repair is to make `Auto (inherit from provider)` remove a per-model API override before saving, or relabel it as keep-current behavior and cover that choice with a regression test.
