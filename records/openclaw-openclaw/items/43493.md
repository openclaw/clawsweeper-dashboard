The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, with helper functions and unit tests for editing model display name, API adapter, reasoning, input modalities, context limits, output limits, and pricing.

## Best Possible Solution

Keep the custom-provider-only metadata prompt direction, fix Auto so it truly restores provider API inheritance or rename it to keep-current behavior, add regression coverage, then leave broader onboarding/context UX to the related open issues.

## Reproduction Assessment

Yes for the PR finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the save merge preserves `entry.model.api`, and current main defines provider inheritance as a missing model-level `api`.

## Solution Assessment

No as-is. The prompt is a reasonable narrow solution, but the Auto API adapter option must either clear the model-level override or stop claiming provider inheritance.
