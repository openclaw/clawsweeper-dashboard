## Summary

Keep this PR open, but do not merge as-is. Current main still lacks interactive custom-provider metadata prompts, and the PR has useful implementation/proof, but one remaining PR-head persistence bug makes the API adapter Auto option misleading.

## What This Changes

The PR adds a custom-provider metadata wizard to `openclaw configure --section model`, with helpers and tests for editing model display name, API adapter, reasoning/input capabilities, context window, max output tokens, and pricing.

## Best Possible Solution

Keep the metadata wizard direction, but make Auto truly restore provider-level API inheritance and keep broader onboarding parity tracked through the canonical issue.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows the PR's Auto branch leaves `updated.api` absent, the save merge preserves `entry.model.api`, and current main only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The feature direction is reasonable, but the Auto adapter option must remove model-level API overrides or be relabeled to match the actual persistence behavior before merge.
