## Summary

Keep open: current main still does not provide an interactive metadata prompt for custom-provider setup, and this PR remains a useful implementation candidate, but the latest diff still has a blocking API adapter Auto persistence bug.

## What This Changes

The PR adds a custom-provider model metadata wizard step to `openclaw configure --section model`, plus helper functions and tests for editing `ModelDefinitionConfig` fields.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option either clear model-level overrides or be relabeled so the persisted config matches the UX contract.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows Auto omits `updated.api` while the save merge preserves `entry.model.api`, and current catalog loading inherits the provider adapter only when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable fit for the custom-provider setup UX, but the API adapter control needs a narrow persistence fix or clearer label before merge.
