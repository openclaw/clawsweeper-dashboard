## Summary

Keep open: this feature is not implemented on current main and remains useful for custom-provider metadata setup, but the PR head has a blocking Auto adapter bug where selecting provider inheritance cannot remove an existing per-model `api` override.

## What This Changes

The PR adds a custom-provider-only model metadata wizard step and tests for editing context window, max output tokens, API adapter, reasoning support, input modalities, and optional pricing during model configuration.

## Best Possible Solution

Fix the Auto adapter semantics with regression coverage, then land or replace the custom-provider metadata prompt while leaving broader auto-detection and runtime warning UX to the related issues.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the final merge preserves the previous `entry.model.api`, and current main’s provider catalog treats absence as provider inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a narrow maintainable direction, but Auto must clear the model-level override or be relabeled to keep the current model override before merge.
