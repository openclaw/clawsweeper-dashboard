## Summary

Keep this PR open: current main still lacks the interactive custom-provider metadata prompt, and the PR remains a useful implementation path, but it has a source-reproducible API-inheritance blocker before merge.

## What This Changes

The PR adds a custom-provider-only model metadata prompt for display name, API adapter, reasoning, input modalities, context window, max output tokens, and pricing during `openclaw configure --section model`, with focused tests.

## Best Possible Solution

Fix the API-inheritance clearing bug, recheck the integration against current main, then use this PR or a narrow replacement as the implementation path for the custom-provider metadata prompt tracked by https://github.com/openclaw/openclaw/issues/61716.

## Reproduction Assessment

Not applicable as an original bug report because this is a feature PR. The blocking PR behavior is source-reproducible by starting with a configured model that has a model-level `api` override, selecting `Auto (inherit from provider)`, and seeing the merge preserve `entry.model.api`.

## Solution Assessment

No. The feature direction is useful, but the implementation must explicitly clear or delete a model-level `api` override when the user selects provider inheritance.
