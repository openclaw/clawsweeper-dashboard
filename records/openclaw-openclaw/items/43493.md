## Summary

Keep open: this PR addresses a real custom-provider metadata prompt gap that current main still lacks, and the submitted proof is sufficient, but the patch still has a P2 correctness bug in the API adapter Auto path.

## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, with helpers and tests for editing stored model metadata fields.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance semantics with regression coverage, then decide whether the helper should also move into shared custom-provider onboarding.

## Reproduction Assessment

Yes for the review finding. Source inspection shows selecting Auto leaves `updated.api` absent while the save merge preserves `entry.model.api`, and current main uses an absent model-level `api` to inherit from the provider.

## Solution Assessment

No as-is. The feature direction is reasonable, but `Auto (inherit from provider)` must clear the per-model API override or be relabeled to avoid promising inheritance it cannot apply.
