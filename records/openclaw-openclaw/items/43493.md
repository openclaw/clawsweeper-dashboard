## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, plus helpers and tests for editing model name, API adapter, reasoning/input capabilities, context window, max tokens, and pricing.

## Best Possible Solution

Keep the metadata wizard direction, fix Auto so it truly restores provider-level API inheritance, and leave broader onboarding parity to the existing follow-up issue.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows the PR's Auto branch leaves `updated.api` absent, the save merge preserves `entry.model.api`, and current main only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The feature direction is reasonable, but the Auto adapter option must remove model-level API overrides or be relabeled to match its actual persistence behavior before merge.
