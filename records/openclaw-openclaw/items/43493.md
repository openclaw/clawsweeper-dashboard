## Summary

Keep open. Current main still lacks the requested custom-provider metadata prompt, and this PR remains a useful implementation candidate, but the patch still has a correctness bug in the new API adapter prompt before it can merge.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for editing configured model definitions.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance with regression coverage, then let maintainers decide whether the helper should also be shared with onboarding.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows Auto writes no `api`, the save path preserves `entry.model.api`, and current main inherits from the provider only when the model-level API field is absent.

## Solution Assessment

No as-is. The narrow repair is to make `Auto (inherit from provider)` remove a per-model API override before saving, or relabel that option as keep-current behavior and cover the choice with a regression test.
