## Summary

Keep open: current main still lacks guided custom-provider metadata prompts, and this PR addresses a real feature gap, but the latest head has a correctness blocker because Auto does not clear an existing model-level API override.

## What This Changes

The PR adds a custom-provider metadata prompt to `openclaw configure --section model`, plus helpers and tests for editing display name, API adapter, reasoning, input modalities, context/max token limits, and pricing.

## Best Possible Solution

Keep the custom-provider metadata prompt direction, fix Auto so it truly restores provider API inheritance or rename it to keep-current behavior, add focused regression coverage, then rebase and validate against current main.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the save merge preserves `entry.model.api`, and current main treats missing `model.api` as provider inheritance.

## Solution Assessment

No as-is. The feature direction is reasonable, but the Auto API adapter option must either clear the model-level override or stop claiming provider inheritance.
