## Summary

Keep open. Current main still lacks the guided custom-provider metadata step, and this PR is useful but not mergeable yet because the patch has source-reproducible model-ref/input preservation blockers plus the required changelog gap.

## What This Changes

The branch adds a custom-provider-only configure wizard step and tests for editing metadata on the selected primary model.

## Best Possible Solution

Rebase the feature onto current main and land the metadata prompt only after it uses canonical model-ref helpers, preserves the full existing model metadata surface, handles API inheritance intentionally, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main and the PR head gives high-confidence reproduction cases for provider-prefixed native model IDs, stale model-level API overrides, and audio/video input metadata loss.

## Solution Assessment

No. The feature direction is sound, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, cannot clear an advertised Auto/inherit API choice, and can drop valid media modalities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: Custom setup stores the primary ref with the shared `modelKey`, which dedupes a native id that already starts with the provider prefix. With provider `custom` and model id `custom/model`, the primary key is `custom/model`, but this split searches for model id `model` and silently skips the metadata prompt.
  - confidence: 0.9
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: The prompt offers `Auto (inherit from provider)`, but this branch leaves `result.api` unset and the later merge spreads the existing model first. If the model already has a stale per-model `api`, choosing Auto preserves that override instead of returning to provider inheritance.
  - confidence: 0.84
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` supports `audio` and `video`, but this prompt only offers `text` and `image` and then overwrites `result.input`. Editing a media-capable custom model can drop existing audio/video metadata even when the user only intended to change token limits.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/commands/configure.gateway-auth.ts:138-140`
  - body: This is a user-facing configure feature, but the PR does not update `CHANGELOG.md`. Add a single-line Unreleased Changes entry with appropriate human contributor credit before merge.
  - confidence: 0.9
