- **[P1] Preserve existing input modalities:** `src/commands/model-metadata.ts:149`
  - body: The wizard always writes `result.input` from a selector that only offers text/image. Existing configured custom models can carry `audio` or `video` in `ModelDefinitionConfig.input`; this prompt can overwrite those values and silently disable media-related routing, so keep unrepresented current modalities or expose the full supported set before saving.
  - confidence: 0.87
- **[P2] Make Auto clear the model API override:** `src/commands/model-metadata.ts:131-133`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the final merge preserves `...entry.model`, so any existing model-level `api` override survives. Runtime resolves the model override before the provider default, so the prompt cannot actually return the model to provider inheritance.
  - confidence: 0.86
