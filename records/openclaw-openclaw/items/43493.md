## Summary

Keep open: current main still lacks an interactive custom-provider metadata prompt, and this PR is the active implementation path for the related custom-provider context-window UX issue. The branch has sufficient real terminal proof, but it needs two narrow correctness fixes before merge.

## What This Changes

The PR adds a custom-provider-only model metadata wizard after `openclaw configure --section model`, with helpers and tests for editing context window, max tokens, API adapter, reasoning, inputs, and cost.

## Best Possible Solution

Revise this PR so the metadata wizard can clear per-model API overrides, preserve all supported input modalities, and then land it as the setup-focused fix for the custom-provider metadata UX.

## Reproduction Assessment

Yes. Source inspection shows selecting `Auto` leaves `updated.api` absent while the merge preserves `entry.model.api`; it also shows the input picker can only return text/image even though config accepts video/audio.

## Solution Assessment

No as written. The setup-time metadata prompt is the right narrow direction, but the patch should repair the two merge/picker semantics before merge.

- **[P2] Clear model API when Auto is selected:** `src/commands/model-metadata.ts:131-133`
  - body: The prompt labels `__auto` as provider inheritance, but this branch omits `api` from `updated`; the final merge spreads `...entry.model` first, so an existing per-model `api` remains and continues overriding the provider-level adapter.   - confidence: 0.91

  - confidence: 0.8
