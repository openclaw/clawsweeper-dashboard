## Summary

Keep open: current main still lacks this custom-provider metadata prompt, and the PR is a useful implementation candidate, but PR-head source shows a blocking API-inheritance bug when `Auto` is selected.

## What This Changes

This PR adds a custom-provider-only metadata prompt to `openclaw configure --section model`, plus helper functions and tests for editing stored model metadata.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance semantics with regression coverage, then decide whether to adapt the helper into the shared onboarding path.

## Reproduction Assessment

Yes for the review finding. Source inspection at PR head shows `Auto` writes no `api`, the save merge preserves `entry.model.api`, and current main only inherits from the provider when `model.api` is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` clear the per-model API override, or relabel it as keep-current behavior, with a focused regression test.
