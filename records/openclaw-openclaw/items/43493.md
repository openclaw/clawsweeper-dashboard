## Summary

## What This Changes

The PR adds a custom-provider model-metadata wizard step during `openclaw configure --section model`, plus helper coverage for collecting, finding, updating, and prompting model definitions.

## Best Possible Solution

Fix the API-inheritance clearing bug with regression coverage, then review this PR or a narrow replacement as the implementation path for https://github.com/openclaw/openclaw/issues/61716.

## Reproduction Assessment

Not applicable as an original bug report because this is a feature PR. The blocking PR behavior is source-reproducible by starting with a configured model that has a model-level `api` override, selecting `Auto (inherit from provider)`, and seeing the merge preserve `entry.model.api`.

## Solution Assessment

No. The feature direction is reasonable, but the implementation must explicitly clear or delete a model-level `api` override when the user selects provider inheritance.
