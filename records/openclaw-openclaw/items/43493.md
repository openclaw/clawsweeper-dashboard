## Summary

Keep open: current main still lacks setup-time metadata collection for custom providers, and this PR is a plausible implementation with real CLI proof, but the current head has a P2 correctness bug in the API adapter Auto path that should be repaired before merge.

## What This Changes

The PR adds a custom-provider-only model metadata wizard step under `openclaw configure --section model` plus helper tests for editing display name, adapter, reasoning, inputs, token limits, and pricing.

## Best Possible Solution

Fix the Auto adapter semantics with regression coverage, then land or replace the setup-time metadata prompt while leaving broader auto-detection and runtime-warning work to the related umbrella issue.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR head shows Auto leaves `updated.api` absent while the final merge preserves `entry.model.api`, and current main treats absence as provider inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a narrow maintainable direction, but Auto must clear the model-level adapter override or be relabeled before merge.
