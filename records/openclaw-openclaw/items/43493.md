## Summary

Keep open: current main still lacks the requested custom-provider metadata prompt, and this PR addresses a real UX gap, but a P2 API inheritance bug still blocks merge.

## What This Changes

The PR adds an interactive metadata prompt for custom provider model setup so users can configure model definition fields during `openclaw configure --section model`.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option either clear model-level overrides or be relabeled so the persisted config matches the UX contract.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows Auto omits updated.api while the save path preserves entry.model.api, and current catalog loading only inherits the provider adapter when model.api is absent.

## Solution Assessment

No as-is. The feature is a reasonable fit for the reported custom-provider setup UX, but the API adapter control needs a narrow persistence fix or clearer label before merge.
