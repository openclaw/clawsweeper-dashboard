## Summary

Keep open: current main still lacks this metadata prompt, and this PR is a useful implementation candidate, but it should not merge until the API adapter Auto persistence mismatch is fixed.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model` plus helper tests for updating model definition fields.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option either clear model-level overrides or relabel it so the saved config matches the UX contract.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows Auto leaves `updated.api` unset, the save path preserves `entry.model.api`, and current catalog loading only inherits the provider adapter when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable fit for the custom-provider setup UX, but the API adapter control needs a narrow persistence fix or clearer label before merge.
