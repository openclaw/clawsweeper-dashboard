## Summary

Keep open: current main still lacks interactive custom-provider model metadata collection, and this PR is a plausible implementation, but it should not merge until the Auto adapter option actually clears per-model overrides.

## What This Changes

The branch adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, with helper functions and tests for editing stored `ModelDefinitionConfig` fields.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance semantics with regression coverage, then decide whether to adapt the helper into the shared onboarding path.

## Reproduction Assessment

Yes for the review finding. Source inspection at PR head shows `Auto` writes no `api`, the save merge preserves `entry.model.api`, and current main only inherits from the provider when `model.api` is absent.

## Solution Assessment

No as-is. The narrow maintainable fix is to make `Auto (inherit from provider)` delete the per-model API override before saving, or relabel it as keep-current behavior, with a focused regression test.
