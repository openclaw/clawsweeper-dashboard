## Summary

Keep open: the PR still targets a real custom-provider metadata UX gap that current main has not implemented, but the patch needs a focused correctness fix before merge because the `Auto (inherit from provider)` API choice cannot actually clear an existing per-model override.

## What This Changes

Adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, with helpers and tests for editing display name, API adapter, reasoning, input modalities, token limits, and pricing.

## Best Possible Solution

Keep the metadata prompt direction, fix API adapter inheritance semantics with regression coverage, then rebase for maintainer review of whether the helper should also move into the shared onboarding path.

## Reproduction Assessment

Yes for the review finding. On PR head, selecting Auto leaves `updated.api` unset while the later save merge keeps `entry.model.api`, and current main represents provider inheritance by leaving `model.api` absent.

## Solution Assessment

No as-is. The feature direction is reasonable, but `Auto (inherit from provider)` must clear the per-model API override or be renamed to describe keep-current behavior.
