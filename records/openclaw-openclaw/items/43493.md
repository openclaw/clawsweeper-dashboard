## Summary

Keep open: current main still does not implement the interactive custom-provider metadata prompt, and the PR has sufficient real CLI proof, but the patch should not merge as-is because the Auto API adapter choice cannot clear an existing per-model adapter override.

## What This Changes

The PR adds a custom-provider-only metadata wizard step to `openclaw configure --section model`, backed by new model metadata helpers and tests.

## Best Possible Solution

Repair the PR branch so Auto truly clears a model-level API override, keep the custom-provider-only guard and existing config schema, and leave broader onboarding parity to the linked tracking issues.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows selecting Auto leaves `updated.api` absent while the later merge preserves `entry.model.api`, and current main uses absence as provider-level inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a reasonable narrow direction, but the Auto adapter option must explicitly remove the model-level override or stop claiming provider inheritance.
