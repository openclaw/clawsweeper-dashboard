## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve all existing input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: This picker only offers `text` and `image`, but the current model metadata contract can carry other input modalities. Editing an existing custom model with those values can save a narrowed `input` list and change routing behavior, so the prompt should expose every supported modality or preserve unoffered existing values.
  - confidence: 0.86
- **[P2] Let Auto clear model-level API overrides:** `src/commands/model-metadata.ts:122-133`
  - body: Choosing `Auto (inherit from provider)` leaves `updated.api` unset, and the later `{ ...entry.model, ...updated }` merge preserves any existing model-level `api`. That means the UI says provider inheritance but cannot actually clear an override, so users remain stuck on the previous adapter.
  - confidence: 0.84
- **[P3] Use strict decimal parsing for token counts:** `src/commands/model-metadata.ts:93-96`
  - body: `Number(raw.trim())` still accepts non-decimal forms such as hex strings for fields labeled as token counts, while the repo already has strict positive-integer parsing helpers for CLI/input seams. Use a strict decimal integer parser here so the wizard does not persist surprising context or max-token values.
  - confidence: 0.72
