## Summary

Keep open: this PR still contains unique useful work that current main does not implement, but it is not merge-ready because the new Auto API-adapter choice can preserve a stale model-level adapter override.

## What This Changes

The branch adds a custom-provider metadata prompt to `openclaw configure --section model`, plus helper and unit-test coverage for editing model display name, API adapter, reasoning, input modalities, context window, max tokens, and optional pricing.

## Best Possible Solution

Land this only after the API inheritance semantics are fixed or renamed, regression-tested on current main, and deliberately scoped against the broader onboarding metadata request.

## Reproduction Assessment

Not applicable as an original bug report because this is a feature PR. The blocking defect is source-reproducible: start with a custom model that has `models.providers.<id>.models[].api`, select `Auto (inherit from provider)`, and the merge keeps the old model-level value.

## Solution Assessment

No, not as currently implemented. The feature direction is reasonable, but the API adapter prompt must either truly clear model-level overrides for provider inheritance or stop labeling that choice as inheritance.
