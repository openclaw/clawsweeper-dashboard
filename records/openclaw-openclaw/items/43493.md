## Summary

Keep open: this is a plausible feature for the custom-provider metadata UX and the PR body includes sufficient terminal screenshot proof, but the patch still has a correctness blocker where selecting `Auto` cannot clear an existing per-model API override.

## What This Changes

The PR adds a custom-provider-only metadata prompt to `openclaw configure --section model` and tests for editing `ModelDefinitionConfig` fields.

## Best Possible Solution

Land this or an equivalent setup-focused change after making `Auto` actually remove model-level `api` overrides, with broader auto-detection and runtime warning work left to https://github.com/openclaw/openclaw/issues/48338 if maintainers want that scope.

## Reproduction Assessment

Yes for the review finding by source inspection: choose `Auto` with an existing model-level `api`, and the merge preserves the old override. I did not run the interactive wizard because this review was read-only.

## Solution Assessment

No as-is: adding the prompt in the custom-provider path is a reasonable narrow solution, but the `Auto` option must match its label by clearing the per-model adapter override before this should merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear the model API override when Auto is selected:** `src/commands/model-metadata.ts:131-133`
