## Summary

Keep open: the feature still addresses a real custom-provider metadata setup gap, but the latest proposed merge keeps a stale model-level API adapter when the wizard says Auto should inherit from the provider.

## What This Changes

The PR adds a custom-provider-only model metadata wizard after `openclaw configure --section model`, with helpers and tests for editing model name, API adapter, reasoning, input, limits, and pricing.

## Best Possible Solution

Keep the metadata wizard direction, but make Auto truly remove model-level API overrides or relabel it to match persistence, then add regression coverage for that case.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection shows Auto leaves `updated.api` unset, the save merge preserves `entry.model.api`, and current main inherits provider API only when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable setup improvement, but the API adapter Auto control needs a narrow persistence fix or clearer wording before merge.
