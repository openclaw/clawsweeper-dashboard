## Repair Work Prompt

Fix the API adapter Auto/inheritance edge case in https://github.com/openclaw/openclaw/pull/43493. In `src/commands/model-metadata.ts`, selecting `__auto` must remove any existing model-level `api` override so the runtime falls back to `providerConfig.api` under the current `model.api ?? providerConfig.api` contract. Add focused regression coverage in `src/commands/model-metadata.test.ts` for an existing custom model with `api: "openai-responses"` where the prompt selects Auto and the saved model no longer has a model-level `api`. Keep the metadata prompt gated to `authChoice === "custom-api-key"`; do not add new config keys, change provider API precedence, broaden the prompt to built-in providers, or touch unrelated docs/changelog in the repair.
