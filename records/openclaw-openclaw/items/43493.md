## Summary

## What This Changes

## Best Possible Solution

Land a scoped metadata-prompt implementation that either clears model-level API overrides when Auto is selected or removes that option, and explicitly ties the chosen configure/onboarding scope to the canonical setup metadata issue.

## Reproduction Assessment

Yes, at source level for the review finding. Selecting Auto on a model with an existing model-level `api` returns no `api` field, the merge preserves `entry.model.api`, and runtime resolution uses the model-level API before provider inheritance.

## Solution Assessment

No, not as-is. Prompting existing model metadata is a plausible direction, but the API inheritance semantics need repair and maintainers need to confirm whether configure-only prompting is the right product scope.
