## Summary

Keep open: current main still lacks the setup-time custom-provider metadata prompt, and this PR remains the focused implementation candidate, but the patch has a blocking Auto adapter regression where selecting provider inheritance does not clear an existing per-model `api` override.

## What This Changes

This PR adds an interactive custom-provider model metadata prompt to `openclaw configure --section model`, with helpers and tests for updating configured model definitions.

## Best Possible Solution

Fix or relabel the Auto adapter option so it matches provider-inheritance semantics, add regression coverage, then land the setup prompt while leaving broader auto-detection/runtime copy to related work.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows the Auto path omits `api` while the final merge preserves the previous `entry.model.api`. I did not run the interactive wizard in this read-only review.

## Solution Assessment

No as-is. The setup-time metadata prompt is a good narrow direction, but the API adapter option must either clear the explicit model override when Auto is selected or stop promising provider inheritance.
