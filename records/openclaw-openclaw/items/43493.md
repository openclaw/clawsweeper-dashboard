## Summary

Keep open: current main still does not implement setup-time custom-provider metadata prompts, and this PR still has a concrete correctness bug where the Auto API adapter option cannot clear an existing per-model override.

## What This Changes

The PR adds a custom-provider-only metadata prompt to the model configuration flow, with helpers and tests for editing model display name, API adapter, reasoning, input modalities, context/max tokens, and token pricing.

## Best Possible Solution

Repair the PR branch so Auto actually clears a model-level API override, keep the custom-provider-only scope, and add focused regression coverage for that merge behavior.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows selecting Auto leaves updated.api absent while the later merge preserves entry.model.api, and current main treats missing model.api as provider-level inheritance.

## Solution Assessment

No as-is. The custom-provider metadata prompt is a reasonable narrow direction, but the Auto adapter option must either clear the model-level override or stop presenting itself as provider inheritance.
