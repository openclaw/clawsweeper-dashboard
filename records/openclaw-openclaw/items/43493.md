## Summary

Keep open: this is a useful custom-provider metadata setup feature with sufficient real behavior proof, but the branch still has a correctness bug where the API adapter Auto option cannot clear an existing model-level `api` override.

## What This Changes

The branch adds a custom-provider-only metadata prompt to `openclaw configure --section model`, plus helpers and tests for editing model display name, API adapter, reasoning, input modalities, context/max tokens, and pricing.

## Best Possible Solution

Fix the API Auto clearing path, add focused regression coverage, then land the setup prompt as the narrow custom-provider metadata improvement while leaving broader detection and runtime-copy work to the related issue.

## Reproduction Assessment

Yes. Source inspection of the PR merge ref shows the review finding: choosing Auto leaves `result.api` unset while the save path preserves `entry.model.api`.

## Solution Assessment

No, not as-is. The setup-prompt direction is reasonable, but the Auto option needs an explicit clear/delete path so it can actually restore provider inheritance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear model API when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` undefined, and `promptModelMetadataForPrimary` later saves `{ ...entry.model, ...updated }`. A model that already has a per-model `api` override therefore cannot revert to provider inheritance even though the prompt offers that option; delete or clear `api` on the Auto path and add a regression test.
