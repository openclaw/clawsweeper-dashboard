## Summary

Keep open: this is a useful custom-provider metadata wizard, and current main still lacks the central prompt, but the PR is not merge-ready because the API adapter Auto option cannot clear an existing model-level override.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to `openclaw configure --section model`, plus helper functions and unit tests for editing configured model metadata.

## Best Possible Solution

Keep the custom-provider metadata wizard direction, but make Auto clear model-level API overrides or intentionally relabel it, with focused regression coverage.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection of the merge result shows Auto omits `updated.api`, while the save merge preserves `entry.model.api`; current catalog loading only inherits the provider adapter when that field is absent.

## Solution Assessment

No as-is. The feature direction is maintainable, but the API adapter control needs a narrow persistence fix so the label and saved config behavior agree.
