## Summary

Keep this PR open: current main still does not provide the requested custom-provider metadata prompt, and the PR remains a plausible implementation path, but it has a blocking P2 correctness bug around returning a model to provider-level API inheritance.

## What This Changes

The PR adds a custom-provider-only `openclaw configure --section model` metadata prompt plus helper tests for editing configured model definitions.

## Best Possible Solution

Keep the metadata prompt direction, fix Auto API inheritance with regression coverage, then let normal maintainer review decide the final prompt scope.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head shows Auto writes no `api`, the save path preserves `entry.model.api`, and current main only inherits from the provider when the model-level API field is absent.

## Solution Assessment

No as-is. The narrow fix is to make `Auto (inherit from provider)` remove any existing per-model `api` override before saving, or to relabel it as keep-current behavior and test that explicitly.
