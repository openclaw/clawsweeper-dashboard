## Summary

Keep open: current main still lacks the custom-provider metadata wizard, and this PR is a plausible implementation, but the PR head still has a P2 correctness bug where the API adapter Auto choice cannot clear an existing model-level override.

## What This Changes

The PR adds a custom-provider-only model metadata prompt after `openclaw configure --section model` custom provider setup, plus helper functions and unit tests for model metadata updates.

## Best Possible Solution

Keep the metadata wizard direction, but make the API adapter Auto option clear model-level overrides or change the label to match the persisted behavior before merge.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection of PR head shows Auto leaves `updated.api` unset, the save merge preserves `entry.model.api`, and current main only inherits provider API when model-level `api` is absent.

## Solution Assessment

No as-is. The feature is a reasonable custom-provider setup improvement, but the API adapter control needs a narrow persistence fix or clearer wording before merge.
