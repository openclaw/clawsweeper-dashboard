## Summary

Keep open. Current main still lacks the requested sandbox-off startup warning, but this PR is not mergeable as-is because its expanded trusted-proxy warning misses the omitted `allowUsers` allow-all case and the branch needs current-main security-warning review.

## What This Changes

The PR adds dangerous startup flag warnings in `src/security/dangerous-config-flags.ts` for explicit `agents.defaults.sandbox.mode="off"`, `gateway.auth.mode="none"`, and explicit empty trusted-proxy `allowUsers`.

## Best Possible Solution

Rebase onto current main, add the explicit sandbox-off startup warning in the core dangerous flag helper with focused tests, and either remove the auth-warning expansion or make trusted-proxy detection match runtime and audit semantics.

## Reproduction Assessment

Yes. A config with `gateway.auth.mode="trusted-proxy"`, a configured `trustedProxy.userHeader`, and omitted `trustedProxy.allowUsers` reproduces the PR bug by static inspection: the new condition skips the warning while runtime and audit treat the omitted value as allow-all.

## Solution Assessment

No. The explicit sandbox-off equality check is the right narrow direction, but the current PR is not the best solution until it is rebased and the trusted-proxy warning either normalizes `allowUsers ?? []` or is split out for separate review.
