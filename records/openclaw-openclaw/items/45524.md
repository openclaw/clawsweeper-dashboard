## Summary

Keep open: the security-warning idea is still useful and not implemented on current main, but this branch targets a stale collector shape, includes incomplete trusted-proxy warning logic, and lacks inspectable after-fix behavior proof.

## What This Changes

The branch adds inline dangerous-config startup warnings for explicit sandbox-off, disabled gateway auth, and empty trusted-proxy allowUsers in src/security/dangerous-config-flags.ts.

## Best Possible Solution

Land a rebased, focused security-warning patch in src/security/core-dangerous-config-flags.ts with regression coverage, keeping sandbox-off separate from or aligned with the dedicated gateway-auth warning work.

## Reproduction Assessment

Yes. Source inspection shows current main still logs dangerous-config collector results at gateway startup, the current core collector lacks the sandbox-off flag, and the PR's trusted-proxy condition skips the omitted allowUsers state that runtime and audit treat as allow-all.

## Solution Assessment

No. The explicit sandbox-off equality check is the right narrow behavior, but the maintainable patch should target src/security/core-dangerous-config-flags.ts on current main and avoid bundling incomplete auth-warning work already tracked by https://github.com/openclaw/openclaw/pull/45523.
