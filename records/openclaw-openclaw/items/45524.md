## Summary

Keep open: current main still lacks the requested sandbox-off startup warning, but this PR is not mergeable or correct as-is because it targets the old collector shape and the added trusted-proxy warning misses the omitted allowUsers allow-all state.

## What This Changes

The PR adds dangerous-config startup warnings for explicit `agents.defaults.sandbox.mode="off"`, `gateway.auth.mode="none"`, and `gateway.auth.trustedProxy.allowUsers=[]` in `src/security/dangerous-config-flags.ts`.

## Best Possible Solution

Add the explicit sandbox-off warning to `src/security/core-dangerous-config-flags.ts` with focused tests, and leave the auth-mode/allowUsers warning work to https://github.com/openclaw/openclaw/pull/45523 or align it there with omitted allowUsers semantics.

## Reproduction Assessment

Yes. Source inspection shows current main’s startup warning path is active but the core collector lacks the sandbox-off flag, and the PR’s trusted-proxy condition skips omitted allowUsers while runtime/audit treat it as allow-all.

## Solution Assessment

No. The explicit sandbox-off equality check is the right narrow direction, but the patch should target the current core collector and either remove the auth-warning expansion or align it with the open auth-warning PR and runtime semantics.
