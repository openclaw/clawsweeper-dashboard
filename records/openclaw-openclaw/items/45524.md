## Summary

Keep open: current main still lacks the sandbox-off startup warning, but this PR is not mergeable as-is because it targets the pre-refactor collector shape, expands into auth-warning work that is already tracked separately, and lacks contributor real behavior proof.

## What This Changes

The PR modifies `src/security/dangerous-config-flags.ts` to add startup dangerous-config warnings for explicit sandbox-off, auth mode none, and empty trusted-proxy allowUsers.

## Best Possible Solution

Land a rebased, focused patch that adds the explicit sandbox-off warning to `src/security/core-dangerous-config-flags.ts` with regression coverage, while leaving the auth-mode and trusted-proxy warning work to https://github.com/openclaw/openclaw/pull/45523 or fixing it there with omitted-allowUsers semantics.

## Reproduction Assessment

Yes. Source inspection shows startup still logs collector results, current main's core collector lacks the sandbox-off flag, and the PR's trusted-proxy condition skips the omitted allowUsers state that runtime and audit treat as allow-all.

## Solution Assessment

No. The explicit sandbox-off equality check is the right narrow behavior, but the patch should target the current core collector and either remove the auth-warning expansion or align it with the dedicated auth-warning PR and runtime semantics.
