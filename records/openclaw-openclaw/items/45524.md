## What This Changes

The branch adds startup dangerous-config warnings for explicit `agents.defaults.sandbox.mode=off`, `gateway.auth.mode=none`, and explicit empty trusted-proxy `allowUsers` in `src/security/dangerous-config-flags.ts`.

## Best Possible Solution

Rebase a focused startup warning into `src/security/core-dangerous-config-flags.ts` with regression coverage for explicit sandbox-off, and either drop or coordinate the auth warnings with the companion auth PR while covering omitted `allowUsers`.

## Reproduction Assessment

Yes, by source inspection. Current startup still logs dangerous-config collector output, current main's core collector lacks the requested sandbox-off startup flag, and trusted-proxy `allowUsers` behavior is traceable through runtime normalization.

## Solution Assessment

No for the current branch shape. The warning direction is sound, but the checks need to move into the current core collector and the trusted-proxy condition must cover omitted `allowUsers` if the auth scope remains.
