## Summary

Keep open. Current main and the latest release still leave Docker sandbox PID, memory, swap, and CPU limits unset unless an operator configures them, and the linked open PR only addresses PID limits.

## What This Changes

Request to add default Docker sandbox CPU, memory, swap, and PID limits while preserving operator overrides.

## Best Possible Solution

Approve explicit Docker sandbox resource budgets and opt-out semantics, then implement them through resolver constants with schema, type/help, docs, changelog, and focused tests proving defaults apply while global and per-agent overrides still win.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve Docker sandbox config with no global or per-agent resource settings, then build Docker create args; the resolved values stay undefined and the resource-limit flags are omitted.

## Solution Assessment

Unclear until maintainers choose the default budgets and opt-out semantics. The narrow implementation point is the Docker sandbox config resolver and create-args coverage, while the existing PID-only PR is useful but incomplete for this broader request.
