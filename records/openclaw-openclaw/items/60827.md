## What This Changes

Add default Docker sandbox CPU, memory, swap, and PID limits so new sandbox containers are capped unless operators override them.

## Best Possible Solution

Land a maintainer-approved Docker sandbox resource-default policy with explicit unlimited opt-out semantics and regression coverage across config resolution, schema/types, docs, and Docker create args.

Yes. Source inspection gives a high-confidence path: resolve Docker sandbox config with no global or per-agent resource settings, then build Docker create args; the resolved values stay undefined and the resource-limit flags are omitted.

## Solution Assessment

Unclear until maintainers choose budgets and opt-out semantics. The likely implementation point is the Docker sandbox resolver plus constants, schema/types, docs, changelog, and tests, while the open PID-only PR is useful but incomplete.
