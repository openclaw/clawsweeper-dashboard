## Summary

Keep open: current main and the latest release still leave Docker sandbox PID, memory, swap, and CPU limits unset unless configured, and the related open PR only covers the PID limit subset.

## What This Changes

Request default Docker sandbox PID, memory, swap, and CPU limits so new sandbox containers are resource-capped without per-agent configuration.

## Best Possible Solution

Approve default Docker sandbox budgets and explicit opt-out semantics, then update resolver constants, schema/types, docs, changelog, and focused tests while coordinating the PID-only PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve Docker sandbox config with no global or per-agent resource settings, then build Docker create args; the resolved values stay undefined and the resource-limit flags are omitted.

## Solution Assessment

Unclear until maintainers choose the default budgets and opt-out semantics. The likely implementation point is the Docker sandbox resolver plus schema/docs/tests, while the open PID-only PR is useful but incomplete for this broader request.
