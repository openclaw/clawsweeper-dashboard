## Summary

Keep open: current main and v2026.5.22 still leave Docker PID, memory, swap, and CPU limits unset unless configured; the related open PR only covers the PID subset and does not settle the broader default-budget policy.

## What This Changes

The issue requests default Docker sandbox CPU, memory, swap, and PID limits so new sandboxed agents are not unlimited unless operators configure overrides.

## Best Possible Solution

Approve explicit Docker sandbox resource budgets and opt-out semantics, then update the resolver, schema/types, docs, changelog, focused tests, and real Docker proof in one coordinated hardening change.

Yes. Source inspection gives a high-confidence path: resolve Docker sandbox config with no global or per-agent resource settings, then build Docker create args; the resolved values stay undefined and the resource-limit flags are omitted.

## Solution Assessment

Unclear until maintainers choose default budgets and compatibility semantics. The resolver is the likely implementation point, but schema, docs, tests, and an explicit opt-out contract need to move with it.
