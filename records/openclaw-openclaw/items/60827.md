## Summary

Keep open. Current main and the latest release still leave Docker sandbox PID, memory, swap, and CPU limits unset unless an operator configures them, and the related open PR only addresses the narrower PID-limit part.

## What This Changes

The issue requests default Docker sandbox CPU, memory, swap, and PID resource limits for newly created sandbox containers.

## Best Possible Solution

Approve explicit Docker sandbox resource budgets, then implement resolver defaults with schema/type/help/docs/changelog updates and tests proving defaults apply while global and per-agent overrides still win.

## Reproduction Assessment

Yes, by source inspection. Resolve Docker sandbox config with no global or per-agent Docker resource settings and `pidsLimit`, `memory`, `memorySwap`, and `cpus` remain undefined; Docker create args then omit the corresponding limit flags.

## Solution Assessment

Unclear until maintainers choose the default budgets and opt-out semantics. The narrow maintainable path is the Docker sandbox config resolver plus schema/types/docs/create-args coverage; the open pids-only PR is useful prior art but insufficient for the broader request.
