## Summary

Keep open. Current `main` and the latest tagged release still leave Docker sandbox PID, memory, swap, and CPU limits unset unless an operator configures global or per-agent values, so the reported host resource-exhaustion hardening gap remains.

## What This Changes

Request default Docker sandbox resource limits so newly enabled sandbox containers are capped without per-agent operator configuration.

## Best Possible Solution

Approve explicit Docker sandbox resource budgets, then implement them through constants/resolver behavior with schema, docs/help, changelog, and tests proving defaults apply while global and per-agent overrides still win.

## Reproduction Assessment

Yes. A source-level reproduction is to resolve Docker sandbox config with no global or per-agent Docker resource settings and observe that `pidsLimit`, `memory`, `memorySwap`, and `cpus` remain undefined, causing Docker create args to omit the corresponding flags.

## Solution Assessment

Unclear until maintainers choose the default budgets. The narrow maintainable path is the Docker sandbox config resolver plus create-args/config tests; the open pids-only PR is useful but insufficient for the broader request.
