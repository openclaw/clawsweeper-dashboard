## Summary

This PR should stay open because the performance problem is valid and the proof is useful, but the proposed global TTL cache can return plugin metadata for the wrong config, workspace, env, or installed-index inputs.

## What This Changes

Adds a process-global five-second TTL cache around `loadPluginMetadataSnapshot()` in `src/plugins/plugin-metadata-snapshot.ts`.

## Best Possible Solution

Keep the performance fix direction, but implement compatible snapshot reuse or a cache keyed to the full metadata scope with invalidation tests and a changelog entry.

## Reproduction Assessment

Yes for the PR regression risk: source inspection shows the cache returns before examining scoped metadata inputs. I did not live-reproduce the original Control UI freeze on current main, but the PR body includes after-fix live output from the reported workflow.

## Solution Assessment

No. A hidden process-global wall-clock cache is not the narrowest maintainable fix; reuse should flow through explicit compatible snapshots or a keyed invalidation strategy that preserves next-read freshness.
