## Summary

Keep open: current main has a global MCP server registry and general tool-policy filtering, but no typed, documented, or runtime-enforced per-server agent ACL. This is a new config/runtime feature that needs maintainer product and security direction.

## What This Changes

The issue requests an optional `agents: string[]` field on `mcp.servers.<name>` to scope a shared MCP server to selected agents and warn on unknown agent IDs.

## Best Possible Solution

If maintainers accept the feature, add a reviewed core MCP config contract with schema/types/docs, server filtering before MCP connection and tool catalog exposure, subagent inheritance rules, and doctor validation against `agents.list`.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Current-main source and docs show no first-class `mcp.servers.<name>.agents` ACL to reproduce as failing.

## Solution Assessment

Unclear until maintainers decide the product contract. A per-server ACL may be the right core seam, but the implementation should define server connection/catalog reuse, subagent inheritance, and interaction with existing tool policies before code changes.
