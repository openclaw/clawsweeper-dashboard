## Risks / Open Questions

- Choosing `service.instance.id` carelessly could expose a stable gateway identity to every configured telemetry backend or create high-cardinality resource labels if derived from run/session/message data.
- Adding a new config key would expand OpenClaw's high-bar config surface and could duplicate standard OTel resource mechanisms unless maintainers explicitly choose that contract.
- `OPENCLAW_OTEL_PRELOADED=1` means the plugin skips its own NodeSDK lifecycle for traces/metrics, so docs and tests need to distinguish plugin-owned resources from host-preloaded SDK resources.
