Add bounded resource attributes inside `diagnostics-otel`, derive `service.version` from the canonical runtime version, use an approved stable non-secret gateway/install identity for `service.instance.id`, leave deployment environment to standard OTel mechanisms unless maintainers choose otherwise, and document/test the final exported resource.
