## Summary

Keep open: current main and the latest release still construct the diagnostics-otel resource with only `service.name`, and the protected `maintainer` label plus the stable instance-identity choice require maintainer review before implementation.

## What This Changes

Request explicit OpenClaw-controlled OpenTelemetry resource identity attributes, such as runtime version and stable gateway or install identity, for diagnostics-otel exports.

## Best Possible Solution

Add a bounded, approved resource-identity seam for plugin services, populate version and stable instance identity from maintainer-approved sources, consume it in diagnostics-otel, and document/test final exported resources without broad new config unless maintainers choose it.

## Reproduction Assessment

Yes. Source inspection on current main shows the plugin-owned resource contains only `service.name`, and docs/config expose only `serviceName`; no collector run is needed to prove the requested OpenClaw-controlled identity attributes are absent.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation depends on maintainer approval for the stable instance identity source and whether the seam belongs in plugin service context, docs-only OTel env guidance, or a config/API surface.
