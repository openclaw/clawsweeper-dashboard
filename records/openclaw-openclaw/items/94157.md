The issue reports `manifest.webmanifest` returning 403 when OpenClaw is accessed through `oauth2-proxy`.

## Best Possible Solution

Keep the current OpenClaw static-asset behavior and handle this as reverse-proxy configuration unless a new report shows an OpenClaw JSON 401/403 or a minimal gateway repro.
