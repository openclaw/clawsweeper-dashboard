## Summary

Keep open: current main and the latest release still serve root-absolute Control UI manifest hrefs, and the issue is already paired with the open runtime fix at https://github.com/openclaw/openclaw/pull/94204.

## What This Changes

The issue reports repeated 403 responses for the Control UI PWA manifest request when OpenClaw is served behind oauth2-proxy or a path-prefixed auth proxy.

## Best Possible Solution

Land or reject https://github.com/openclaw/openclaw/pull/94204 as the canonical runtime fix, then close this issue only after the runtime behavior is resolved.

## Reproduction Assessment

Yes, source-reproducible. Current main serves `index.html` with a host-root manifest link, and configured-basePath routing rejects a host-root `/manifest.webmanifest` request as outside the Control UI path.

## Solution Assessment

Yes, keeping the issue open behind https://github.com/openclaw/openclaw/pull/94204 is the best handling now. A docs-only workaround can help operators, but the narrow runtime fix should prevent the pre-JavaScript host-root asset request that triggers the proxy 403.
