## Summary

Keep open: current main and the latest release still serve root-absolute Control UI manifest/favicon hrefs, and a clean open runtime PR already uses closing syntax for this issue, so the issue should wait for that PR to land or be rejected.

## What This Changes

The issue reports `manifest.webmanifest` returning 403 when the Control UI is accessed behind oauth2-proxy with a configured gateway path.

## Best Possible Solution

Land or reject one canonical runtime fix for configured-basePath public asset hrefs, with docs guidance handled separately once the runtime URL strategy is settled.

## Reproduction Assessment

Yes, source-reproducible. Current main serves `index.html` with a host-root manifest link, and with `gateway.controlUi.basePath` configured the route classifier rejects host-root `/manifest.webmanifest` as outside the Control UI path.

## Solution Assessment

Yes, keeping the issue open behind https://github.com/openclaw/openclaw/pull/94204 is the best handling now. A docs-only workaround can help operators, but the narrow runtime fix should prevent the pre-JavaScript host-root asset request that triggers the proxy 403.
