## Summary

Keep open: current main still lacks the requested transcript session-header check, and the PR is a focused implementation with a direct regression test. I did not find a blocking correctness or security issue in the proposed diff.

## What This Changes

The PR adds first-line transcript session-header validation to `readLatestSessionUsageFromTranscript` and a colocated regression test for mismatched stale transcript usage after reset.

## Best Possible Solution

Land this narrow guard after normal review and CI, preserving legacy/headerless transcript behavior while rejecting parseable mismatched session headers before transcript-backed usage is shown.

## Reproduction Assessment

Yes. A high-confidence reproduction is to create a transcript whose first line has session ID A plus assistant usage, then call `readLatestSessionUsageFromTranscript` for session ID B with that file discoverable; current main would extract A's usage, while this PR expects `null`.

## Solution Assessment

Yes. The first-line header check is the narrow maintainable fix because OpenClaw writes the session header first for new transcripts, and the PR deliberately leaves unparseable or headerless legacy/custom transcripts on the existing fallback path.
