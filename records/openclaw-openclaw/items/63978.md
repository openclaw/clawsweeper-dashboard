The PR adds first-line session-header id validation to `readLatestSessionUsageFromTranscript` and a regression test for stale transcript usage after reset.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. Header-id validation is the right mechanism, but applying it only to one sync helper is not the narrowest maintainable fix because other usage readers remain exposed.
