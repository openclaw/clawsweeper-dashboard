## Summary

Keep open: the linked bug is still present on current main, but this branch is not merge-ready because its raw cron-like detector can suppress normal requester ping-pong delivery.

## What This Changes

The branch adds an isolated-cron requester gate for sessions_send A2A fire-and-forget flows and regression coverage for isolated-cron and normal requesters.

## Best Possible Solution

Use the established cron-run classifier for requester session keys and keep channel-based cron handling only where the runtime actually supplies a cron-only requester channel for isolated runs.

## Reproduction Assessment

Yes from source: current main enters requester ping-pong for cross-session A2A when `maxPingPongTurns > 0`, and the PR's raw detector would zero that value for any requester key containing `:cron:`. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No as written: the direction is right, but the detector should use the existing structural cron-run classifier instead of a raw substring check that over-matches normal session keys.
