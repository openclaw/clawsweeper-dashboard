## Summary

The PR is still necessary and appears to be the stronger canonical fix for the linked isolated-cron `sessions_send` feedback-loop bug; current `main` still forwards positive A2A ping-pong turns to the requester path, while this branch narrows the zero-turn behavior to isolated cron requesters and adds normal-requester regression coverage.

The PR makes isolated-cron `sessions_send` A2A announce flows pass zero ping-pong turns, adds baseline capture for that requester class, and adds regression tests for cron, normal, and cron-like requester keys.

## Best Possible Solution

Land this focused cron-only A2A gate after required checks, keep target announce delivery intact, and resolve the linked bug plus the sibling fix path through this canonical PR.

## Solution Assessment

Yes. Forcing zero ping-pong turns only for canonical isolated cron requesters is narrower than skipping the whole A2A flow, and the added baseline capture protects delayed fire-and-forget reads without changing normal requester behavior.
