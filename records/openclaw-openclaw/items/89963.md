## Summary

## What This Changes

The PR wraps bundled-channel registry-record to runtime-metadata conversion in a per-record try/catch and adds a regression test for a malformed bundled row followed by a healthy row.

## Best Possible Solution

Land this narrow per-record isolation after maintainer review and normal changed-gate proof, keeping the manifest registry as the canonical metadata source.

## Reproduction Assessment

Yes. From source inspection, current main directly maps each registry row through toBundledChannelPluginMetadata, so a throwing record aborts the whole bundled-channel metadata list; the PR test models that failing row followed by a healthy one.

## Solution Assessment

Yes. The patch fixes the failure at the narrow conversion boundary and leaves registry loading, caller filtering, and bundled channel caches unchanged.
