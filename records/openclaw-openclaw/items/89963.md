## Summary

Keep open: current main still converts bundled channel metadata rows directly, so a throwing row can abort discovery, and this protected maintainer draft PR is a focused fix with a matching regression test. I found no actionable patch defect, but it should stay in normal maintainer review and merge-gate flow.

## What This Changes

The PR wraps each bundled channel manifest-record-to-metadata conversion in a per-record try/catch and adds a Vitest regression for skipping one unreadable row while returning a later healthy Slack row.

## Best Possible Solution

Land the narrow per-row metadata isolation after maintainer review and normal gate proof, keeping registry loading canonical and recovery scoped to bundled channel row conversion.

## Reproduction Assessment

Yes from source inspection: current main directly maps each registry record through toBundledChannelPluginMetadata(), so a thrown conversion aborts the full metadata list. I did not execute the regression test because this review is read-only.

## Solution Assessment

Yes, this appears to be the best narrow fix: isolate only per-row metadata conversion while leaving manifest registry loading and downstream channel load diagnostics on their existing boundaries.
