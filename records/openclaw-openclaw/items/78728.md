## What This Changes

This PR records device-token-auth metadata for exec approvals, permits trusted backend gateway-client replays for no-device Control UI/WebChat approvals, requests approval scopes for approved node invokes, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land a narrow approval-binding fix that preserves node, command, cwd, env, session, expiry, and allow-once constraints, after maintainer/security review and real-run proof for the linked Control UI no-device workflow.

## Reproduction Assessment

Yes, source inspection and the linked report give a high-confidence path: a no-device Control UI approval record is bound to the original connId, then the approved node invoke replays through a fresh backend gateway-client connection and current main can reject it with `APPROVAL_CLIENT_MISMATCH`. I did not run the Docker/node scenario live in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: bridge only trusted backend gateway-client replays for no-device approval records while keeping the existing node/context/decision checks. The missing piece is contributor real behavior proof and maintainer/security review, not a different code shape.
