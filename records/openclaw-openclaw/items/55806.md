## Summary

Keep open. Current main has partial observability through diagnostic events, OTel spans, plugin hooks, and redacted UI tool events, but the requested durable sanitized INFO/file log, toolResult errorType/errorRecoverable contract, and generic writeback audit tracking are still absent and need maintainer product/security decisions.

## What This Changes

The issue requests privacy-preserving tool-call observability for sanitized inputs, structured error metadata, and writeback auditing.

## Best Possible Solution

Define one privacy-preserving observability contract across diagnostics, agent events, OTel, persisted transcripts, and mutating-tool audit events, with regression coverage for redacted args, message bodies, and write contents.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is a feature/audit gap. Static inspection of current main gives a high-confidence verification path showing the durable argument log, toolResult error metadata, and generic writeback audit fields are absent.

## Solution Assessment

Unclear. The request is directionally valid, but standalone INFO logs and transcript fields are not clearly the safest solution; the maintainable path is a maintainer-approved redaction and observability contract reused by all durable surfaces.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because new durable observability fields could persist sensitive tool inputs, outputs, message bodies, or write contents.

Concerns:

- **[medium] Centralize redaction before durable audit fields:** `src/agents/pi-embedded-subscribe.tools.ts:147`
  - body: sanitizeToolArgs currently protects the Control UI event path, but durable INFO/file logs or transcript fields would be new exposure points. Any implementation should reuse or strengthen this redaction boundary before persisting parameters.
  - confidence: 0.86
- **[medium] Bound writeback audit payloads:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1204`
  - body: Current patch summaries record target path lists rather than raw contents. A generic file/memory/context writeback event should keep bounded metadata such as target, operation, byte/line counts, and redacted keys instead of storing write values.
  - confidence: 0.82
