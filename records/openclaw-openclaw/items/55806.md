## Summary

Keep open. Current main has partial observability through diagnostic tool execution events, redacted Control UI tool events, OTel export, and apply_patch summaries, but it still lacks the requested INFO/file-log sanitized argument summaries, durable toolResult errorType/errorRecoverable fields, and generic file/memory/context writeback audit tracking; the related #55846 PR is closed and unmerged.

## What This Changes

The issue requests sanitized tool argument observability, structured tool error metadata, and context/file/memory writeback audit tracking.

## Best Possible Solution

Define one privacy-preserving observability contract across diagnostics, agent events, OTel, transcript/toolResult records, and mutating-tool audit events, then implement the missing fields with regression tests proving sensitive args, message bodies, and write contents stay redacted.

## Reproduction Assessment

Not applicable as a feature/audit gap rather than a runtime crash. The high-confidence verification path is static source inspection of current main, which shows the requested durable argument, error metadata, and writeback fields are still missing.

## Solution Assessment

Unclear. The issue direction is valid, but the proposed standalone INFO log and toolResult additions need maintainer product/security schema decisions; the safer path is to extend the existing diagnostics/redaction surfaces under one durable contract.
