## Summary

Keep open. Current main has partial diagnostics and Control UI redaction, but it still lacks the requested durable INFO/file-log sanitized argument summaries, toolResult errorType/errorRecoverable fields, and generic file/memory/context writeback audit tracking; related PR #55846 is closed and unmerged.

## What This Changes

Request privacy-preserving tool-call observability for sanitized input summaries, structured error metadata, and generic context/writeback tracking.

## Best Possible Solution

Define one privacy-preserving observability contract across diagnostics, agent events, OTel, transcript/toolResult records, and mutating-tool audit events, then add bounded/redacted fields with regression coverage.

## Reproduction Assessment

Not applicable; this is a feature/audit gap rather than a crash. Static current-main inspection verifies the requested durable args, error metadata, and writeback tracking are still missing.

## Solution Assessment

Unclear. The direction is valid, but standalone INFO logs and toolResult additions need schema and security decisions; the safer path is to extend existing diagnostics/redaction surfaces under one shared contract.

Status: needs_attention

Summary: This issue is security-sensitive because new durable observability fields could persist secrets or write contents without a central bounded redaction contract.
