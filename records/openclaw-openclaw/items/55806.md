## Summary

Keep open. Current main has partial tool observability through diagnostic events, OTel spans, redacted Control UI tool events, plugin hooks, and apply_patch summaries, but it still lacks the requested durable INFO/file-log sanitized argument summaries, toolResult errorType/errorRecoverable fields, and generic file/memory/context writeback audit tracking.

## What This Changes

Request durable tool-call observability with sanitized argument summaries, structured tool errors, and writeback audit tracking.

## Best Possible Solution

Define one privacy-preserving observability contract across diagnostics, agent events, OTel, transcript/toolResult records, and mutating-tool audit events, then add regression tests proving sensitive args, message bodies, and write contents stay redacted.

## Reproduction Assessment

Not applicable as a feature/audit gap rather than a runtime crash. The high-confidence verification path is static source inspection of current main, which shows the requested durable argument, error metadata, and writeback fields are still missing.

## Solution Assessment

Unclear. The request is directionally valid, but the proposed standalone INFO log and transcript fields need maintainer product/security schema decisions; the safer path is to extend the existing diagnostics/redaction surfaces under one durable contract.

Status: needs_attention

Summary: The request is security-sensitive because any new durable observability surface could persist sensitive tool inputs, outputs, or write contents if it bypasses central redaction.
