## Summary

Keep open. Current main still renders the reported Model Aliases prompt text and still lists configured aliases without provider-overlap guardrails, so the prompt can still encourage an agent to replace an explicit provider/model before the runtime resolver sees it.

## What This Changes

The issue asks OpenClaw to stop model-alias prompt guidance from causing agents to substitute a different provider when an explicit provider/model string was requested.

## Best Possible Solution

Update alias prompt rendering so exact provider/model strings must be passed through verbatim, aliases are only convenience names for unqualified requests, and overlapping-provider cases have focused regression coverage.

## Reproduction Assessment

Yes, for a source-level reproduction: configure overlapping aliases and build the agent system prompt; current main still emits guidance to prefer aliases while the resolver only preserves an exact provider/model if the LLM passes it through. The live substitution itself depends on model behavior, but the production trace is credible and matches the current prompt precondition.

## Solution Assessment

Yes. The narrow maintainable fix is prompt and alias-presentation wording plus targeted tests; provider fallback mechanics and model resolver semantics do not need a new policy knob for this report.
