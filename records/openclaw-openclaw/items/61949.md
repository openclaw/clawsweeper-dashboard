## Summary

Keep open. Current main and the latest release still render model aliases with wording that encourages alias preference, and the linked fix PR is open but not merged, so the issue should remain open until that PR or an equivalent change lands.

## What This Changes

The issue asks OpenClaw to stop prompt guidance from causing explicit provider/model requests to be rewritten to configured aliases for other providers.

## Best Possible Solution

Land or replace the narrow linked prompt/test fix so model-facing guidance says to preserve exact provider/model strings and use aliases only for unqualified model requests.

## Reproduction Assessment

Yes. Configure agents.defaults.models with aliases and build the full agent system prompt; current main source shows the Model Aliases section still tells the model to prefer aliases even though sessions_spawn preserves exact model strings after the tool call.

## Solution Assessment

Yes. The narrow maintainable fix is to change the prompt guidance and regression test rather than adding a new config surface, because the runtime forwarding path already preserves an explicit provider/model once supplied.
