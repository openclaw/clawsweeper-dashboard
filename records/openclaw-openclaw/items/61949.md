## Summary

Keep open. Current main still renders a Model Aliases section that tells agents to prefer aliases and lists every configured alias, so the prompt can still encourage provider substitution before the runtime resolver ever receives the exact provider/model string.

## What This Changes

Prevent Model Aliases prompt guidance from causing an LLM to replace an explicitly requested provider/model with a different provider alias.

## Best Possible Solution

Update alias prompt rendering so exact provider/model strings must be passed through verbatim, aliases are only convenience names for unqualified requests, and overlapping-provider cases have focused regression coverage.

## Reproduction Assessment

Yes, for the source-level precondition: current main still injects alias guidance that prefers aliases while the resolver only preserves an exact provider/model if the LLM passes it through. The live provider substitution itself depends on model behavior, but the production trace in the issue is credible.

## Solution Assessment

Yes. The narrow maintainable fix is prompt and alias-presentation wording plus targeted tests; provider fallback mechanics and model resolver semantics do not need a new policy knob for this report.
