## Summary

Keep open: current main and the latest release still render model-alias guidance that tells agents to prefer aliases, and the open linked fix PR owns the narrow repair but has not merged.

## What This Changes

The issue asks OpenClaw to stop prompt guidance from inviting agents to replace explicit provider/model refs with aliases for other providers.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/85155; the end state should preserve aliases as exact shortcuts while telling agents to pass explicit provider/model strings verbatim.

## Reproduction Assessment

Yes, source-level: current main renders the exact alias guidance that the report identifies, and the test suite still expects that wording. A fully deterministic live provider swap was not established because the final substitution depends on model behavior after reading the prompt.

## Solution Assessment

Yes: the linked PR is the narrow owner-boundary fix because it changes the model-facing prompt contract and locks it with prompt tests. Broader fallback filtering or config behavior changes would be a larger product decision than this report needs.
