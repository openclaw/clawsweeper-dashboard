## Summary

Keep open. Current main still renders all configured model aliases with a "Prefer aliases" instruction, so an LLM can still substitute another configured provider before runtime's exact provider/model pass-through protection runs.

## What This Changes

The issue asks OpenClaw to stop prompt-rendered model aliases from encouraging agents to substitute a different provider when an explicit provider/model string was requested.

## Best Possible Solution

Make alias prompt guidance explicitly preserve full provider/model strings, limit aliases to bare convenience names, and add regression coverage for overlapping providers.

## Reproduction Assessment

Yes for the source-level precondition: a config with `agents.defaults.models` aliases still renders a Model Aliases section that tells agents to prefer aliases. The live provider substitution itself depends on model behavior, but the issue discussion provides a credible production trace.

## Solution Assessment

Yes. The narrowest maintainable fix is prompt and alias-presentation wording plus focused tests, because runtime resolution already preserves qualified refs once the tool receives them.
