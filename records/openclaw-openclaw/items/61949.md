## Summary

Keep open. Current main and v2026.5.19 still expose configured model aliases in the full agent system prompt with wording that prefers aliases, while exact provider/model preservation only applies after the model has already chosen the `sessions_spawn.model` argument.

## What This Changes

The issue asks OpenClaw to stop model alias prompt guidance from causing LLMs to replace explicit provider/model selections with an alias-backed provider.

## Best Possible Solution

Change model-alias prompt rendering so explicit provider/model refs are passed through unchanged and aliases are presented only as convenience names for unqualified model requests, with regression coverage for overlapping providers.

## Reproduction Assessment

Yes for a source-level reproduction: configure overlapping aliases and inspect or build the full agent system prompt; current main still emits the alias-preference guidance. A deterministic live LLM/provider swap was not established.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust alias prompt presentation and regression tests without changing provider fallback policy or adding a new config knob.
