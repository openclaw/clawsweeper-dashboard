## Summary

Keep open. Current main still tells agents to prefer model aliases while the runtime only preserves an exact provider/model after the LLM has already chosen what string to pass to `sessions_spawn`, so the reported provider-substitution path remains plausible and source-visible.

## What This Changes

The issue asks OpenClaw to stop prompt-injected model aliases from encouraging agents to substitute one configured provider/model for another when an explicit provider/model was requested.

## Best Possible Solution

Change alias prompt rendering so exact provider/model strings are treated as routing instructions, aliases remain convenience names for bare requests only, and overlapping-provider cases have focused regression coverage.

## Reproduction Assessment

Yes for the source-level precondition: a config with `agents.defaults.models` aliases still renders a Model Aliases section that tells agents to prefer aliases. The live provider substitution depends on model behavior, but the discussion gives a credible production trace.

## Solution Assessment

Yes. The narrowest maintainable fix is prompt and alias-presentation wording plus focused tests; fallback mechanics and provider runtime behavior do not need broad changes for this report.

## Repair Work Prompt

Fix the model-alias prompt behavior reported at https://github.com/openclaw/openclaw/issues/61949. Current main renders `## Model Aliases` with `Prefer aliases when specifying model overrides; full provider/model is also accepted.`, which can encourage the LLM to substitute a configured fallback provider when a caller explicitly requested a full `provider/model` in `sessions_spawn.model`. Keep the fix narrow: update alias prompt rendering so full `provider/model` strings are passed through verbatim and aliases are described only as convenience names for bare model requests. Do not change provider fallback mechanics, auth selection, or model failover behavior unless a test proves it is necessary. Add regression coverage for the prompt wording and an overlapping-provider alias configuration, and keep/extend coverage that `resolveSubagentSpawnModelSelection` passes already-qualified refs unchanged. Add a user-facing changelog entry because this affects runtime model routing. Likely files: `src/agents/system-prompt.ts`, `src/agents/system-prompt.test.ts`, `src/agents/model-alias-lines.ts`, `src/agents/model-selection.ts`, `src/agents/model-selection.test.ts`, possibly `docs/tools/subagents.md` and `CHANGELOG.md`.
