## Summary

Keep open. Current main still advertises model aliases with “Prefer aliases” guidance while the runtime only preserves provider routing after the LLM passes the exact qualified provider/model string through.

## What This Changes

The issue reports that model-alias prompt guidance can cause an LLM to substitute a different provider/model when a fully-qualified model was requested.

## Best Possible Solution

Update alias prompt rendering so qualified provider/model strings are passed through verbatim, aliases are framed only as convenience names for bare requests, and overlapping-provider cases have regression coverage.

## Reproduction Assessment

Yes for the source-level precondition: configuring `agents.defaults.models` aliases still renders a Model Aliases section that tells agents to prefer aliases. The live substitution itself depends on model behavior, but the discussion provides a credible production trace.

## Solution Assessment

Yes. The narrowest maintainable fix is prompt and alias-presentation wording plus focused tests, because the resolver already preserves qualified refs once it receives them.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/61949 by making model-alias prompt guidance preserve explicit provider/model routing. In current main, `src/agents/system-prompt.ts` tells agents to prefer aliases, `src/agents/model-alias-lines.ts` emits every configured alias, and `resolveSubagentSpawnModelSelection` only preserves exact provider/model strings after the LLM passes them through. Update the prompt/alias presentation so fully qualified `provider/model` values must be copied verbatim when specified, aliases are only convenience names for bare model requests, and overlapping provider aliases cannot imply provider interchangeability. Add focused regression coverage in `src/agents/system-prompt.test.ts` and `src/agents/model-selection.test.ts`; include `src/agents/tools/sessions-spawn-tool.test.ts` only if the tool boundary changes. Do not redesign fallback mechanics or provider auth. Add a single-line `CHANGELOG.md` Fixes entry if this is treated as user-facing prompt behavior.
