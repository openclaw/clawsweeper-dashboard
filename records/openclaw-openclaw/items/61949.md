## Summary

Keep open. Current main and the latest stable release still show alias guidance that can encourage a model to replace an explicit provider/model before the runtime resolver sees it, and no canonical PR or related issue currently owns this exact prompt defect.

## What This Changes

Ensure model aliases in the agent system prompt do not cause explicit provider/model overrides to be substituted across providers.

## Best Possible Solution

Change alias prompt rendering so exact provider/model strings are passed through verbatim, aliases are framed only as convenience names for unqualified requests, and overlapping-provider cases have focused regression coverage.

## Reproduction Assessment

Yes, for source-level reproduction: configure overlapping provider aliases and build the agent system prompt; current main and `v2026.5.18` still emit alias-preference guidance while exact provider/model preservation only applies if the LLM passes the ref through.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust prompt/alias presentation and tests, without changing provider fallback policy or adding a new config knob.
