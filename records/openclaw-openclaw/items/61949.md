## Summary

Keep open: current main and the latest release still render the reported Model Aliases prompt wording, and https://github.com/openclaw/openclaw/pull/85155 is the active linked fix candidate rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to stop prompting LLMs to replace exact provider/model requests with model aliases from other providers.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/85155 so exact provider/model strings remain verbatim and aliases are presented only as exact shortcuts for unqualified requests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 still render the exact Model Aliases guidance reported, and the adjacent test expects it. I did not establish a deterministic live provider-swap run because the final substitution depends on model behavior after reading the prompt.

Yes, the prompt wording fix is the narrow owner-boundary solution for the reported issue. The linked PR is the right active path, with maintainer review needed for its added slash-form alias precedence behavior.
