## Summary

Keep open. Current main still renders the Model Aliases block with alias-preference wording and every configured alias, so the prompt can still invite provider substitution before runtime receives an exact provider/model string.

## What This Changes

The issue asks OpenClaw to prevent model alias prompt guidance from causing agents to substitute a different provider when a full provider/model override was specified.

## Best Possible Solution

Ship a prompt and alias-rendering fix that makes exact provider/model overrides verbatim, treats aliases only as shortcuts for bare requests, and adds overlap/fallback regression coverage.

## Reproduction Assessment

Yes for the source-level precondition: a config with `agents.defaults.models` aliases still renders a Model Aliases section that tells agents to prefer aliases. The live provider substitution depends on model behavior, but the issue discussion provides a credible production trace.

## Solution Assessment

Yes. The narrowest maintainable fix is to adjust alias prompt presentation and tests, because runtime already preserves qualified provider/model refs once it receives them.
