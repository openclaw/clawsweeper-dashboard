## Best Possible Solution

Make gateway busy/free checks socket-probe-first, and keep process attribution bounded, timeout-protected, and limited to OpenClaw-owned listener cleanup.

## Solution Assessment

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75767. Current main still uses `lsof` for gateway `--force`, restart-health port inspection, and stale gateway cleanup; on macOS with slow SMB/Time Machine mounts this can hang because `lsof` traverses mounted filesystems. Implement a socket-probe-first path for routine busy/free checks using existing probe helpers where possible. Keep PID/process attribution only where needed to terminate OpenClaw-owned listeners, and keep any remaining process-inspection calls timeout-bounded. Do not add core migrations or broad restart refactors. Update focused tests for `gateway --force`, `inspectPortUsage`/restart health, and stale cleanup so macOS/Unix paths no longer require `lsof` before bind-probe status where attribution is not needed. Add a `CHANGELOG.md` fix entry because this is user-facing CLI/gateway behavior.
