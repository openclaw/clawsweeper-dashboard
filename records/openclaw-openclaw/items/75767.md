## Summary

Keep open: current main still reaches `lsof` on the reported macOS gateway `--force`, restart diagnostics, and Unix stale-cleanup paths, so the SMB stat-timeout report remains a narrow valid runtime bug.

## What This Changes

The issue asks for macOS gateway restart and force port checks to avoid `lsof`-driven hangs when SMB or Time Machine volumes make `lsof` block during filesystem stat traversal.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep any remaining PID attribution bounded, timeout-protected, and limited to verified OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from `gateway --force`, restart diagnostics, and Unix stale cleanup. A live timing reproduction requires macOS with a slow or unresponsive SMB or Time Machine mount.

## Solution Assessment

Yes: the proposed socket-probe-first direction is the narrowest maintainable fix for availability checks, while process-attribution code should remain only where OpenClaw must identify or terminate a verified listener.
