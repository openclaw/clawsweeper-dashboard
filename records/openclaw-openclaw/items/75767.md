## Summary

Keep open: current main and v2026.6.10 still run per-port Unix `lsof` in the async diagnostics, `--force`, restart-health, and stale-pid cleanup paths, so the reported macOS SMB-mounted-volume hang remains source-backed and not implemented away.

## What This Changes

The issue asks OpenClaw to stop gateway restart/status/force port checks from hanging on macOS when `lsof` blocks on SMB-mounted volumes.

## Best Possible Solution

Keep this issue open and redesign the macOS port-check path so availability uses the existing non-filesystem Node bind probe first, while lsof is reserved for bounded PID/detail lookup only when needed and coordinated with the related macOS lsof pressure work.

## Reproduction Assessment

Partially. Current main source-reproduces the relevant per-port `lsof` call paths and the reporter supplied macOS SMB logs, but this review did not run a live SMB-mounted macOS restart.

## Solution Assessment

Unclear as a final fix. A pure Node bind probe matches an existing helper and is likely the right first step for availability, but PID/owner diagnostics and `--force` behavior still need a bounded design before implementation.
