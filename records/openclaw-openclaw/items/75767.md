## Summary

Keep open: current main still reaches `lsof` on the reported macOS gateway force, restart-health, diagnostics, and stale-cleanup paths, so the SMB-mounted volume hang remains a narrow source-confirmed runtime bug.

## What This Changes

The issue asks OpenClaw to avoid `lsof`-based gateway port checks on macOS because slow SMB or Time Machine mounts can make `openclaw gateway restart` and `openclaw gateway --force` hang.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep PID attribution bounded, timeout-protected, and limited to verified OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from gateway `--force`, restart diagnostics, restart health, and Unix stale cleanup. A live timing reproduction needs macOS with a slow or unresponsive SMB or Time Machine mount.

## Solution Assessment

Yes: the socket-probe-first direction is the narrowest maintainable fix for availability checks. Pure socket probing should not replace the verified-PID path where OpenClaw must identify or terminate its own listener.
