## Summary

Keep open: current main still has `lsof`-first or synchronous gateway port inspection on the reported `--force`, restart-health, and stale-cleanup paths; the related closed macOS restart items address LaunchAgent state rather than SMB mount stat timeouts.

## What This Changes

Request replacing or demoting `lsof`-backed gateway port checks so macOS SMB/Time Machine mounts cannot stall gateway restart or `--force`.

## Best Possible Solution

Make routine gateway port availability checks bind/probe-first, and reserve timeout-bounded process attribution for cleanup paths that actually need listener PIDs.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes the reported `lsof` commands from `gateway --force`, restart health, and stale gateway cleanup. A live timing reproduction requires macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes: socket-probe-first is the narrowest maintainable default for busy/free checks. `lsof` or platform-specific attribution should remain bounded and limited to cases where OpenClaw must identify or terminate a listener.
