## Summary

Keep open: the provided current-main SHA still reaches lsof before any socket probe on the reported `gateway --force`, restart-health, and service-mode stale-cleanup paths, while the related closed reports cover different macOS LaunchAgent restart failures.

## What This Changes

The issue asks OpenClaw to avoid lsof-first gateway port checks on macOS because SMB-mounted volumes can make `openclaw gateway restart` and `gateway --force` hang.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep bounded PID attribution only where OpenClaw must terminate OpenClaw-owned listeners or print process diagnostics.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes the reported lsof commands from `gateway --force`, restart health, and stale gateway cleanup. A live timing reproduction requires macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes, the socket-probe-first direction is the narrow maintainable fix for availability checks. The implementation should preserve process attribution for kill/diagnostic paths without making every health check depend on lsof.
