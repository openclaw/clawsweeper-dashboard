## Summary

Keep open: current main only partially mitigates the report and still reaches `lsof` on the busy-port macOS gateway paths implicated by the SMB stat-timeout hang.

## What This Changes

The issue asks OpenClaw to avoid `lsof`-based gateway port inspection on macOS because SMB-mounted volumes can make `openclaw gateway restart` and `gateway --force` hang.

## Best Possible Solution

Make gateway availability checks socket-probe-first on Unix/macOS, and reserve process enumeration for verified OpenClaw-owned listener termination with bounded timeouts and fallbacks.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from the busy-port `gateway --force`, restart diagnostics, and Unix stale-cleanup paths. A live timing reproduction still needs macOS with a slow or unresponsive SMB or Time Machine mount.

## Solution Assessment

Yes. Socket-probe-first is the narrow maintainable fix for busy/free checks, while process attribution should remain only where OpenClaw must terminate a verified gateway listener.

## Repair Work Prompt

Fix the macOS/Unix gateway port-inspection hang tracked at https://github.com/openclaw/openclaw/issues/75767. Current main still invokes `lsof` from busy-port `gateway --force`, restart diagnostics, and Unix stale gateway cleanup, which can hang on macOS with slow SMB/Time Machine mounts. Make routine busy/free checks socket-probe-first and avoid `lsof` unless OpenClaw must identify or terminate a verified OpenClaw gateway listener. Keep process termination conservative: do not kill unrelated listeners, preserve existing Windows behavior, and do not add a config flag. Add or update focused regression coverage for `src/cli/ports.ts`, `src/infra/ports-inspect.ts`, and `src/infra/restart-stale-pids.ts`; update `CHANGELOG.md` because the fix is user-visible. Stop and ask for maintainer review if the safe fix requires a broader process-attribution redesign.
