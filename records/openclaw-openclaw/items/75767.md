## Summary

Keep open: current `main` still reaches `lsof` from the reported macOS gateway `--force`, restart-health, and service-mode stale-cleanup paths, so the SMB stat-timeout report remains a narrow valid runtime bug.

## What This Changes

The issue requests replacing routine `lsof`-based gateway port checks with socket-probe-first behavior so macOS SMB/Time Machine mounts cannot stall `openclaw gateway restart` or `openclaw gateway --force`.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep any remaining PID attribution bounded, timeout-protected, and limited to verified OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current `main` still invokes `lsof` from `gateway --force`, restart diagnostics, and Unix stale cleanup. A live timing reproduction still requires macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes: socket-probe-first is the narrowest maintainable fix for availability checks, while process-attribution code should remain only where OpenClaw must identify or terminate a listener.
