## Summary

Keep open: current main still reaches `lsof` on the reported gateway `--force`, restart-health, and Unix stale-cleanup paths, so the source-confirmed macOS SMB hang remains actionable.

## What This Changes

The issue requests replacing or avoiding `lsof`-based gateway port checks so SMB/Time Machine mounts cannot stall `openclaw gateway restart` or `openclaw gateway --force`.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep any remaining PID attribution bounded, timeout-protected, and limited to verified OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from `gateway --force`, restart diagnostics, and Unix stale cleanup. A live timing reproduction requires macOS with a slow or unresponsive SMB or Time Machine mount.

## Solution Assessment

Yes: the socket-probe-first direction is the narrowest maintainable fix for availability checks, while process-attribution code should remain only where OpenClaw must identify or terminate a verified gateway listener.
