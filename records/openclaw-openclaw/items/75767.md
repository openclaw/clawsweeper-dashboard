## Summary

Keep open: current main and v2026.6.6 still invoke `lsof` on the Unix/macOS gateway port-inspection paths, and the only related open PR is an explicitly partial status mitigation that does not close the restart or force-path hang.

## What This Changes

The issue requests replacing or reducing macOS gateway port-inspection `lsof` usage because SMB-mounted volumes can make gateway restart, force, and status operations hang.

## Best Possible Solution

Make restart, force, and status busy/free decisions filesystem-independent and non-spawning by default, while keeping `lsof` only as bounded best-effort owner diagnostics that cannot block the user workflow.

## Reproduction Assessment

Partially yes: current source clearly reaches `lsof` on the affected gateway port-inspection paths, and the reporter supplied repeated macOS SMB evidence. I did not run a live SMB-mounted-volume reproduction in this review.

## Solution Assessment

Yes: using Node socket bind/reachability probes for busy/free decisions is the narrow maintainable fix, with `lsof` reserved for bounded diagnostics. The current main implementation and the related partial PR do not yet cover the full restart and force-path problem.
