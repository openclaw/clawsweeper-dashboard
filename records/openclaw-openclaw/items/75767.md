## Summary

Keep open: current main still invokes `lsof` on the reported gateway `--force`, restart-health, and stale-cleanup paths, so the macOS SMB stat-timeout report remains valid and narrowly actionable.

## What This Changes

Avoid macOS gateway restart and `gateway --force` stalls by replacing routine `lsof` port checks with lsof-free, socket-probe-first behavior where process attribution is not required.

## Best Possible Solution

Make gateway busy/free checks lsof-free and socket-probe-first, while keeping any remaining PID attribution bounded, timeout-protected, and limited to OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes the reported `lsof` commands from `gateway --force`, restart health, and stale gateway cleanup. A live stall needs macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes, socket-probe-first is the narrowest maintainable direction for routine port availability checks. PID-based attribution should remain only on paths that actually need to identify or terminate an OpenClaw-owned listener.
