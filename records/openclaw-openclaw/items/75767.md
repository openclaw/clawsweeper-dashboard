## Summary

Keep this open: current main still uses lsof on the Unix/macOS gateway restart, force, and status port-inspection paths, so the SMB-mounted-volume hang is only partially mitigated, not fixed.

## What This Changes

Request replacing or bypassing lsof-based gateway port checks so macOS SMB/NFS mounts cannot hang gateway restart, gateway --force, or related status probes.

## Best Possible Solution

Make busy/free port decisions use filesystem-independent bind probes, and keep lsof/process attribution as bounded best-effort diagnostics that cannot block restart or status workflows.

## Reproduction Assessment

No live SMB repro was run here, but the source path is high-confidence: current main still invokes lsof on the affected Unix/macOS port-inspection paths, and the reporter supplied repeated current-version SMB evidence.

## Solution Assessment

Yes, the pure Node bind-probe direction is the narrowest maintainable default for busy/free decisions; process attribution should stay secondary and bounded rather than driving the restart path.
