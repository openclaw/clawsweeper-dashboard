## Summary

Keep open: current main still reaches `lsof` before the socket probe or fallback on the reported `gateway --force`, restart-health, and stale-cleanup paths, while the related closed reports cover different macOS LaunchAgent restart failures.

## What This Changes

The issue requests replacing or reprioritizing `lsof`-based gateway port checks so macOS SMB/Time Machine mounts cannot stall `openclaw gateway restart` or `--force`.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep any remaining macOS process attribution bounded and limited to paths that actually need to terminate OpenClaw-owned listeners.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes the reported `lsof` commands from `gateway --force`, restart health, and stale gateway cleanup. A live reproduction of the long stall requires macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes, the socket-probe-first direction is the best default for port availability checks. The maintainable complete fix should retain bounded PID attribution only where OpenClaw needs to kill a listener.
