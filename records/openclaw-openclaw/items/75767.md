## Summary

Keep open: current main still invokes `lsof` on the reported gateway `--force`, restart-health, and stale-cleanup paths, so the macOS SMB stat-timeout report remains valid and narrow.

## What This Changes

Avoid gateway restart and `--force` hangs on macOS SMB-mounted volumes by replacing routine `lsof` port checks with socket-probe-first behavior.

## Best Possible Solution

Make gateway busy/free checks socket-probe-first, and keep remaining process attribution bounded, timeout-protected, and limited to OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes the reported `lsof` commands from `gateway --force`, restart health, and stale gateway cleanup. A live stall needs macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes, socket-probe-first is the narrowest maintainable default for routine port availability checks. PID-based `lsof` or platform-specific attribution should remain only on paths that actually need to identify or terminate a listener.
