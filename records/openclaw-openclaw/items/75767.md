## Summary

Keep open: current main still reaches `lsof` on the reported macOS gateway `--force`, launchd restart cleanup, and restart-health diagnostic paths, so the SMB stat-timeout report remains a narrow valid runtime bug.

## What This Changes

Request replacing `lsof`-based gateway port checks with socket-probe-first logic so macOS SMB or Time Machine mount stalls do not hang `gateway restart` or `--force`.

## Best Possible Solution

Make routine gateway busy/free checks socket-probe-first, and keep remaining PID attribution bounded, timeout-protected, and limited to verified OpenClaw-owned listener cleanup.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from `gateway --force`, restart diagnostics, and Unix stale cleanup. Live timing proof requires macOS with a slow or unresponsive SMB or Time Machine mount.

## Solution Assessment

Yes: the socket-probe-first direction is the narrowest maintainable fix for availability checks, while process-attribution code should remain only where OpenClaw must identify or terminate a listener.
