- **[P1] Port location params to the active protocol package:** `src/gateway/protocol/schema/agent.ts:107-115`

  - body: `sendLocation` calls `callMessageGateway` with `method: "location"`, but the gateway registers location handling under the existing `send` handler and has no `location` RPC.   - confidence: 0.91
- **[P2] Return the actual sent JID for location sends:** `extensions/whatsapp/src/send.ts:300`
  - body: This returns `toWhatsappJid(to)` even though `active.sendLocation` resolves phone numbers through the auth-dir LID mapping before sending. For LID-mapped accounts the location is sent to the LID JID, but the gateway payload/session metadata reports the old PN JID, making location delivery identity inconsistent with text sends.
  - confidence: 0.86
