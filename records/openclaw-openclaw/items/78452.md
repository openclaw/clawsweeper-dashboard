Full review comments:

- **[P1] Port location params to the active protocol package:** `src/gateway/protocol/schema/agent.ts:107-115`
  - body: This adds the new send fields to `src/gateway/protocol`, but current main validates gateway send params from `packages/gateway-protocol/src/schema/agent.ts` and the old tree is absent. After rebase, gateway clients will still reject `latitude`/`longitude` as extra properties unless the active package schema and generated clients are updated.
  - confidence: 0.93
- **[P1] Call a registered gateway method for fallback locations:** `src/infra/outbound/message.ts:571`
  - body: `sendLocation` uses `method: "location"`, but the gateway exposes only `message.action`, `send`, and `poll` handlers. Any core fallback path that reaches this helper can fail before a channel adapter sends the pin.
  - confidence: 0.91
- **[P2] Return the actual sent JID for location sends:** `extensions/whatsapp/src/send.ts:300`
  - body: The active listener can resolve phone-number recipients to LID JIDs before sending, but this function returns the pre-resolution `toWhatsappJid(to)` value instead of the remote JID from the send result. That can leave gateway payloads and mirrors with the wrong recipient identity for LID-mapped WhatsApp accounts.
  - confidence: 0.86
