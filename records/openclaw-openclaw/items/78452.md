Land one coordinated native location implementation that uses existing registered Gateway send surfaces, rejects ambiguous mixed payloads consistently, and documents the user-facing contract with the required changelog entry.

## Solution Assessment

No. The feature direction is reasonable, but the patch should route the core fallback through the registered `send` RPC, reject every send payload field before location rerouting, and align docs/changelog/generated surfaces before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Call the registered send RPC for location delivery:** `src/infra/outbound/message.ts:571`
  - body: `sendLocation()` still dispatches Gateway method `location`, but the Gateway has no registered `location` handler and returns `unknown method` for absent handlers. Any non-dry-run core fallback location send that reaches this helper will fail before delivery; send the location params through the existing `send` RPC or add a complete registered RPC surface.
  - confidence: 0.97
- **[P2] Reject all send payload fields before location routing:** `src/infra/outbound/message-action-runner.ts:1246-1250`
  - body: The new guard only checks `message`, `mediaUrl`, and `mediaUrls`, while the normal send path also accepts payload fields such as `media`, `path`, `filePath`, `fileUrl`, `caption`, `presentation`, and `interactive`. A `message.send` call with coordinates plus one of those fields will still succeed as a location-only send and silently drop the other payload.
  - confidence: 0.93
- **[P3] Remove the ignored protocol schema artifact:** `dist/protocol.schema.json:1`
  - body: `dist/protocol.schema.json` is ignored and untracked on current main, but this PR force-adds the generated schema as a large tracked file. Keep the protocol source and tracked generated clients in sync without changing the ignored dist tracking policy unless maintainers explicitly want that policy change.
  - confidence: 0.88
- **[P3] Document the new location send contract:** `src/cli/program/message/register.send.ts:30-40`
  - body: The PR adds user-facing CLI flags, agent tool schema fields, and Gateway send params, but does not update `CHANGELOG.md` or `docs/cli/message.md`. Add the required changelog entry and document latitude/longitude, optional metadata, and the location-only payload rule before merge.
  - confidence: 0.9
