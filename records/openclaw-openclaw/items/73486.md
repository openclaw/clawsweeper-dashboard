## Summary

Keep open. Current `main` still starts the optional model-pricing refresh from initial Gateway runtime setup before the post-attach sidecar/channel readiness path, while this PR narrowly moves that refresh behind scheduled-service activation and sidecar readiness. I did not find a concrete correctness or security finding in the latest PR diff; the remaining action is normal maintainer review and exact-head check handling, with broader pricing resilience tracked separately by #73323 and #75067.

## What This Changes

The PR moves Gateway model-pricing refresh out of initial runtime service startup and into post-attach scheduled-service activation gated on sidecar readiness, while preserving plugin lookup metadata and updating lifecycle tests.

## Best Possible Solution

Land or adapt this PR after maintainer review and required checks, keeping pricing enabled by default while ensuring remote pricing catalog refresh cannot begin until post-attach work and sidecars/channels have reached the ready path. Keep timeout reduction, disk cache, and circuit-breaker behavior in the separate #75067/#73323 follow-up path.
