## Summary

Keep open: the PR is not merge-ready because it changes bare text-channel `/steer` to select the sole active subagent, while the linked Telegram report and current Control UI behavior point to steering the current active session when no subagent target resolves.

## What This Changes

The PR changes text-channel subagent `/steer` parsing to allow `/steer <message>` to target the sole active subagent, adds regression tests and changelog text, and includes unrelated formatting-only edits.

## Best Possible Solution

Settle one `/steer [id] <message>` contract, then align the text handler, native registry, `/subagents` help, public docs, changelog, and regression tests around that contract.

## Reproduction Assessment

Yes. Source inspection shows current main routes text-channel `/steer` into the subagent command path and treats the first word as a subagent target, while Control UI tests already cover current-session steering when no target resolves.

## Solution Assessment

No. The PR implements the sole-active-subagent interpretation, but the linked report and UI behavior support a current-session fallback; it also leaves help/metadata/docs out of sync and adds an unrelated changelog note.
