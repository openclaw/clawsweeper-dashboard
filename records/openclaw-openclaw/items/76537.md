## Summary

Keep open. Current main and the linked Telegram report point to `/steer <message>` needing current-session steering parity, but this PR still routes the fallback to the sole active subagent and leaves the native command metadata/docs on a required subagent target, so it should not merge as-is.

## What This Changes

The PR adds a helper that lets text-channel `/steer` auto-select the sole active subagent when no explicit target resolves, adds focused tests for that path, and adds a changelog entry.

## Best Possible Solution

Make an explicit maintainer decision for text-channel `/steer`, then implement the chosen current-session/subagent fallback consistently across parser, command registry/help, docs, and tests.

## Reproduction Assessment

Yes. Source inspection shows current main routes text/Telegram `/steer check progress` into the subagent handler and treats `check` as the target, while the PR path is source-reproducible as a sole-subagent fallback instead of current-session steering.

## Solution Assessment

No. The branch implements a narrow sole-subagent auto-select behavior, but the linked report and existing Control UI behavior point to current-session steering when no explicit subagent target matches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route bare steer messages to the current session:** `src/auto-reply/reply/commands-subagents/action-send.ts:42-46`
  - body: The linked Telegram repro is `/steer <message>` during the main agent run, and the Control UI already falls back to the current session when the first word is not a subagent. This branch instead selects `activeRuns[0]`, and with no subagents it returns `No active subagent runs to steer.`, so the main-session case remains broken.
  - confidence: 0.93
- **[P2] Preserve usage errors for target-only steer commands:** `src/auto-reply/reply/commands-subagents/action-send.ts:52-55`
  - body: With exactly one active run, `/steer 1` or `/steer last` skips target resolution because there is no second token and sends `1`/`last` as the message. Those tokens are valid explicit targets today, so target-without-message should keep returning usage instead of dispatching accidental steering.
  - confidence: 0.87
- **[P2] Update command metadata with the optional target grammar:** `src/auto-reply/reply/commands-subagents/action-send.ts:29`
  - body: The handler now advertises `/steer [id|#] <message>`, but the command registry, `/subagents` help, and public slash-command docs still require a target. Telegram native commands are generated from the registry, so users will still see the old required-target grammar after this parser change.
  - confidence: 0.86
