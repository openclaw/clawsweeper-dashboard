## Summary

Keep open. The PR addresses a real `/steer` ergonomics report, but the patch is not ready: it targets the sole active subagent instead of the current session fallback used by the Control UI, changes target-only behavior, leaves command help/docs stale, and carries an unrelated contradictory changelog entry.

## What This Changes

The PR changes subagent `/steer` parsing to auto-select the sole active subagent when no explicit target resolves, adds focused tests, and adds changelog entries.

## Best Possible Solution

Settle `/steer [id] <message>` semantics once, then align the text/native parser, command registry, help/docs, changelog, and regression tests with that chosen current-session or subagent fallback.

## Reproduction Assessment

Yes for the parser behavior: source inspection shows Telegram/native text `/steer ...` reaches the subagent command path, and current main treats the first token as the subagent target. The PR behavior is also source-reproducible from its helper selecting the sole active subagent.

## Solution Assessment

No. The branch implements sole-subagent auto-selection, but the linked Telegram report and existing Control UI behavior point to current-session steering when no explicit subagent target resolves; registry and docs should change with the parser if maintainers choose that grammar.
