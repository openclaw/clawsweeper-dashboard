The PR is not ready because it implements a sole-subagent fallback, while the linked Telegram report and existing Control UI behavior point to current-session steering when no explicit subagent target resolves; the branch also leaves command metadata/docs out of sync and carries an unrelated conflicting changelog line.

## What This Changes

The PR changes text-channel subagent `/steer` parsing to auto-select the only active subagent when no explicit target resolves, adds regression tests, adds changelog entries, and includes one formatting-only agent-command change.

## Best Possible Solution

Choose the intended `/steer [id] <message>` contract, then align the text/native parser, Control UI parity, registry/help/docs, regression tests, and changelog around that contract.

Yes for the parser behavior: source inspection shows text-channel `/steer ...` reaches the subagent command path on current main and treats the first word as a subagent target. It is not a strict documented-contract bug because current public docs still define `/steer` as subagent steering.

## Solution Assessment

No. The PR chooses sole-subagent auto-selection, but the linked Telegram report and Control UI behavior point to current-session steering when no explicit target resolves, and the metadata/docs/changelog are not kept consistent.
