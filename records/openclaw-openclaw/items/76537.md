## Summary

Keep open: current main still has the reported text-channel `/steer` parser behavior, but this PR does not fix the linked main-session steering path and introduces parser/help mismatches.

## What This Changes

The PR makes text-channel `/steer` auto-select the sole active subagent when no explicit id resolves, adds subagent steer tests, and adds a changelog entry.

## Best Possible Solution

Choose one `/steer` contract, then implement it consistently across parser behavior, command registry/native menu metadata, `/subagents` help, public docs, and focused text-channel/queue tests.

## Reproduction Assessment

Yes. Source inspection shows current main routes text/Telegram `/steer check progress` into the subagent handler and treats `check` as the target; the PR path is also source-reproducible as a sole-subagent fallback rather than current-session steering.

## Solution Assessment

No. The branch implements a narrow sole-subagent auto-select behavior, but the linked report and existing Control UI behavior point to current-session steering when no explicit subagent target matches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route bare steer messages to the current session:** `src/auto-reply/reply/commands-subagents/action-send.ts:42-46`
  - body: The linked Telegram repro is `/steer <message>` during the main agent run, and the Control UI already falls back to the current session when the first word is not a subagent. This branch instead selects `activeRuns[0]`, and with no subagents it returns `No active subagent runs to steer.`, so the main-session case remains broken.
  - confidence: 0.93
- **[P2] Keep target-only steer commands as usage errors:** `src/auto-reply/reply/commands-subagents/action-send.ts:54-55`
  - body: With exactly one active run, `/steer 1` or `/steer last` skips target resolution because there is no second token and sends `1`/`last` as the message. Those tokens are valid explicit targets today, so target-without-message should keep returning usage instead of dispatching accidental steering.
  - confidence: 0.87
- **[P2] Update all advertised steer grammar:** `src/auto-reply/reply/commands-subagents/action-send.ts:29`
  - body: The handler now advertises `/steer [id|#] <message>`, but the command registry, `/subagents` help, and public slash-command docs still require a target. Telegram native commands are generated from the registry, so users will still see the old required-target grammar after this parser change.
  - confidence: 0.86
