## Summary

Keep open. Current main still has the reported `/steer` parser behavior, and this PR is useful context, but it should not merge as-is because bare `/steer <message>` would steer a sole subagent or fail with no subagents instead of matching the linked main-session active-run shortcut.

## What This Changes

The PR changes subagent `/steer` parsing to auto-select a sole active subagent when no explicit id resolves, adds focused steer tests, and adds a changelog entry.

## Best Possible Solution

Implement the accepted text-channel `/steer` grammar so bare `/steer <message>` steers the current active session when no explicit subagent target matches, preserve explicit subagent targeting, and update registry/help/docs/tests together.

## Reproduction Assessment

Yes, from source inspection: current main routes Telegram/text `/steer check progress` to the subagent steer action and treats `check` as the required target. The PR path is also source-reproducible: with no active subagents it returns a no-subagent message instead of steering the current active session.

## Solution Assessment

No. The implementation is narrow, but it implements a sole-subagent fallback while the linked issue and Control UI behavior point to current-session active-run steering when no subagent target matches.
