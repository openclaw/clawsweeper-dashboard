## Summary

Keep open. The branch still solves a narrower sole-subagent case, while the linked Telegram/main-session shorthand remains unresolved and the public command metadata still advertises a required subagent target.

## What This Changes

The branch changes text-channel subagent `/steer` parsing to choose the sole active subagent when no explicit target resolves, adds focused steer tests, and adds a changelog entry.

## Best Possible Solution

Align text-channel `/steer` with the chosen command contract: either implement current-session fallback while preserving explicit subagent targets and update registry/help/docs/tests, or keep the subagent-only contract and clarify the documentation.

## Reproduction Assessment

Yes, from source inspection: current main routes bare `/steer <message>` to the subagent action and treats the first word as the target. The PR path is also source-reproducible: with no active subagents it returns a no-subagent message instead of steering the current active session.

## Solution Assessment

No. The implementation is narrow, but it implements a sole-subagent fallback while the linked request and Control UI behavior point to current-session active-run steering when no subagent target matches.
