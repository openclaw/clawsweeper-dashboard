## Summary

Keep open. The change is a narrow, plausible fix for the text-channel `/steer` regression, but the PR still needs the user-facing help/docs syntax updated so the newly optional id is discoverable and consistent.

## What This Changes

The PR changes subagent `/steer` command handling so a missing id auto-selects the sole active controlled subagent run, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the handler/test change after aligning command help and public docs with the optional `[id|#]` steer syntax.

## Reproduction Assessment

Yes. Source inspection on current main shows `/steer` always consumes the first token as a subagent target before dispatch, so a message-only steer cannot reach the sole active run.

## Solution Assessment

Yes, with a small documentation/help follow-up. The handler-level fallback is narrow and preserves explicit ids, but the public command surfaces should be updated before merge.
