## Summary

Keep open. The PR is not ready because it changes bare text-channel `/steer` to target the sole active subagent, while the linked reports and current Control UI behavior point to a current-session steer fallback; it also leaves help/metadata/docs mismatched and adds an unrelated changelog entry.

## What This Changes

The PR changes text-channel `/steer` parsing to auto-select the sole active subagent when no explicit target resolves, adds regression tests and changelog text, and includes unrelated formatting-only edits.

## Best Possible Solution

Settle one `/steer [target] <message>` contract, then align the text handler, native registry, `/subagents` help, public docs, changelog, and regression tests around that contract without breaking explicit label/id subagent targets.

## Reproduction Assessment

Yes. Source inspection shows current main routes text-channel `/steer check progress` into the subagent command path and treats `check` as a subagent target; I did not run tests because this was a read-only review and `pnpm` was unavailable.

## Solution Assessment

No. The PR implements a sole-active-subagent fallback, but the linked reports and current UI behavior support current-session steering when no target resolves, and the patch leaves command metadata/docs and changelog content out of sync.
