## Summary

Keep open. The PR touches the right text-command surface, but the current branch routes no-target `/steer <message>` to a sole active subagent while the linked Telegram report and Control UI behavior point to steering the active current session when no explicit subagent target resolves.

## What This Changes

The branch changes text-channel subagent `/steer` parsing to auto-select the sole active subagent when no explicit target resolves, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Define `/steer [target] <message>` semantics once, then align the parser, native-command registry, help/docs, changelog, and regression tests with the chosen current-session or subagent fallback.

## Reproduction Assessment

Yes, source inspection gives a high-confidence parser reproduction: current main maps `/steer text` into the subagent command path and treats the first token as the target. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No. The branch implements sole-subagent auto-selection, but the linked Telegram report and current Control UI behavior point to current-session steering when no explicit subagent target resolves; metadata/docs also need to move with the parser.
