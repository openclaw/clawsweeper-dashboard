## Summary

Keep open. The branch implements a sole-subagent fallback, but the linked Telegram report and Control UI behavior point to bare `/steer <message>` steering the current active session when no subagent target matches; command metadata and docs also still advertise a required target.

## What This Changes

The PR changes `/steer` parsing to auto-select a sole active subagent when no explicit id resolves, adds subagent steer tests, and adds a changelog entry.

## Best Possible Solution

Implement the accepted text-channel `/steer` grammar so bare `/steer <message>` steers the current active session when no explicit subagent target matches, while preserving explicit subagent targeting and updating registry, help, docs, tests, and changelog together.

## Reproduction Assessment

Yes, from source inspection: current main routes text-channel `/steer check progress` to the subagent steer action and treats `check` as the required target. The PR path is also source-reproducible because its no-subagent case returns a subagent error instead of steering the current active session.

## Solution Assessment

No. The PR is narrow, but it fixes a sole-subagent fallback rather than the current-session shorthand shown by the linked Telegram repro and existing Control UI behavior, and it leaves command help/metadata inconsistent.
