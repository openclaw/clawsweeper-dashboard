## Summary

Keep open. The patch does not settle the linked `/steer` command contract: it selects the sole active subagent, while the related Telegram report and Control UI behavior point at current-session steering, and it leaves command metadata/docs plus changelog content inconsistent.

## What This Changes

The PR changes text-channel subagent `/steer` parsing to auto-select the sole active subagent when no explicit target resolves, adds steer tests, updates the changelog, and includes formatting-only edits.

## Best Possible Solution

Settle one documented `/steer [id] <message>` contract, then align the text handler, native registry, `/subagents` help, public docs, changelog, and regression tests around that contract.

## Reproduction Assessment

Yes for the reported parser behavior: current source shows text-channel `/steer check progress` routes through subagent command handling and treats `check` as the subagent target. It is not a strict documented-contract bug because current docs still describe `/steer` as subagent-targeted.

## Solution Assessment

No. The PR implements a sole-active-subagent fallback, but the linked report and Control UI behavior support a current-session fallback; it also leaves help/metadata/docs out of sync and adds unrelated changelog text.
