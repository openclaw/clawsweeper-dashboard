## Summary

Keep open. The PR is not merge-ready because the linked Telegram report and current Control UI behavior point to bare `/steer <message>` steering the current active agent run, while this patch steers a sole active subagent and leaves command metadata/docs/changelog inconsistent.

## What This Changes

The PR changes text-channel `/steer` parsing to auto-select the only active subagent when no explicit target resolves, adds regression tests, adds changelog text, and carries one unrelated formatting-only agent-command change.

## Best Possible Solution

Align the chosen `/steer [id] <message>` contract across the text/native parser, Control UI parity, command registry/help, docs, regression tests, and changelog.

## Reproduction Assessment

Yes for the parser behavior: current source shows text-channel `/steer ...` routes to the subagent command handler and treats the first word as the subagent target. It is not a strict documented-contract bug because current docs still define `/steer` as subagent steering.

## Solution Assessment

No. The proposed fallback selects a sole active subagent, but the linked report and existing Control UI fallback point to current-session steering when no explicit subagent target resolves, and the registry/help/docs are not updated consistently.
