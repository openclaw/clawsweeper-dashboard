## Label Justifications

- `P1`: The PR targets private reasoning leakage in outbound delivery but currently risks breaking real channel message delivery for valid payloads.
- `merge-risk: 🚨 message-delivery`: The diff can suppress or alter outbound payloads before channel send based on broad text heuristics, which can drop intended messages.

## Summary

Keep open: the privacy direction is plausible, but the patch applies assistant-reasoning heuristics to every durable outbound payload and can silently drop valid message-tool or direct-send content. It also has only test output in the PR body, not after-fix real behavior proof.

## What This Changes

The branch adds an outbound assistant-delivery sanitizer in reply dispatcher and durable delivery paths, with regression coverage for stripping thinking tags and dropping reasoning-only payloads.

## Best Possible Solution

Carry payload source or dispatch kind into delivery and reuse the existing code-aware assistant-visible sanitizer only for assistant reply text, while preserving message-tool and prepared-send literals.

## Reproduction Assessment

Yes, source-reproducible. Current main forwards message-tool/prepared payloads through durable delivery, and the PR hard-codes the new sanitizer as assistant-role for that entire path; I did not run a live delivery because this was a read-only review.

## Solution Assessment

No. The safer fix is to sanitize at assistant reply-source boundaries or carry source/kind metadata through delivery, instead of applying reasoning heuristics to every durable outbound payload.
