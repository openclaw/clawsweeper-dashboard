## Summary

Keep open: current main does not contain the PR's deferred streaming-scroll fix, and the PR still hard-codes Adaptive instead of using provider-scoped gateway thinking metadata; it also needs the required user-facing changelog entry.

## What This Changes

The PR adds a fixed Adaptive option to the shared SwiftUI thinking picker and defers streaming auto-scroll into a MainActor task that rechecks the bottom-pin state.

## Best Possible Solution

Render native Swift thinking choices from gateway `thinkingLevels`/`thinkingDefault` metadata, preserve the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

Not fully. No independent iOS runtime reproduction was established, but source inspection shows the current Swift picker ignores gateway thinking option metadata and the PR-introduced Adaptive row can submit unsupported values that `sessions.patch` rejects.

## Solution Assessment

No. The deferred scroll task with an inner pin-state recheck is a narrow fix, but globally hard-coding Adaptive is not; the picker should consume the gateway's provider/model-specific thinking options.
