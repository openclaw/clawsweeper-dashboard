## Summary

Keep open: the branch still has a correctness blocker because the Adaptive picker option is hard-coded while thinking choices are provider/model scoped and rejected by `sessions.patch` when unsupported. It also still needs the required changelog entry for a user-facing iOS chat fix.

## What This Changes

The branch changes the shared SwiftUI chat composer and chat view to add an Adaptive thinking picker row and defer streaming assistant auto-scroll through a MainActor task.

## Best Possible Solution

Render the Swift thinking picker from existing gateway thinking metadata, preserve the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

No independent iOS runtime reproduction was established in this read-only review. Source inspection gives a high-confidence reproduction path for the PR-introduced blocker: choosing a globally rendered Adaptive row on a non-adaptive provider/model submits an unsupported `thinkingLevel` to `sessions.patch`.

## Solution Assessment

No. The deferred streaming-scroll task with an inner pin-state recheck is a narrow fix, but the picker should use the gateway's provider/model-specific `thinkingLevels` metadata instead of hard-coding Adaptive for every session.
