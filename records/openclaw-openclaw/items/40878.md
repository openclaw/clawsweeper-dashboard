## Summary

Keep open: the deferred scroll change is narrow, but the adaptive picker change still hard-codes a provider-scoped option and the PR lacks the required changelog entry.

## What This Changes

The PR changes shared SwiftUI chat UI by adding an Adaptive thinking picker row and deferring streaming assistant auto-scroll into a MainActor task with a rechecked bottom-pin guard.

## Best Possible Solution

Render native Swift thinking choices from gateway `thinkingLevels`/`thinkingDefault` metadata, keep the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

No independent iOS runtime reproduction was established in this read-only review. Source inspection gives a high-confidence reproduction path for the PR-introduced blocker: a global Adaptive row can submit an unsupported explicit value that `sessions.patch` rejects for non-adaptive provider/model profiles.

## Solution Assessment

No. The deferred scroll task with an inner pin-state recheck is a narrow maintainable fix, but hard-coding Adaptive is not; the picker should render the gateway's existing provider/model-specific thinking metadata.
