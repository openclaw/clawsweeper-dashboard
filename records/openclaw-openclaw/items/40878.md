## Summary

Keep open. The hard-coded Adaptive picker change is now superseded by current main's metadata-driven picker and would regress provider/model-specific thinking options, while the remaining streaming-scroll deferral is unique but still lacks inspectable real iOS runtime proof.

## What This Changes

The branch adds a hard-coded Adaptive thinking option to the shared SwiftUI chat composer and defers streaming auto-scroll updates into a MainActor task.

## Best Possible Solution

Preserve current main's metadata-driven thinking picker, drop the hard-coded Adaptive row, and only land a rebased streaming-scroll deferral after visible iOS runtime proof shows it fixes the warning/churn path.

## Solution Assessment

No. The picker solution is superseded by current main's metadata-driven implementation; the streaming-scroll deferral may be narrow, but it needs a clean rebase and real iOS behavior proof before it is a safe fix.
