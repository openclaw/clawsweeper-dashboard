## Summary

Keep open. The PR still addresses behavior not implemented on current main, and the deferred streaming-scroll change is directionally sound, but the hard-coded Adaptive picker row conflicts with the current provider-scoped thinking-level contract and can expose an unsupported explicit setting.

## What This Changes

The PR adds an Adaptive option to the shared SwiftUI thinking picker, includes it in the fixed fallback exclusion list, and defers streaming auto-scroll into a MainActor task that rechecks pin state before scrolling.

## Best Possible Solution

Keep the deferred streaming-scroll fix with the inner pin-state recheck, and change the Swift chat picker to render supported thinking options from gateway session row/default metadata instead of adding Adaptive as a global row. Add the required changelog entry and update the stale OSLog metadata before merge.

## Reproduction Assessment

Yes, partially. Source inspection gives a high-confidence reproduction path for the picker contract issue and current-main synchronous streaming-scroll mutation; the exact SwiftUI warning/loop needs an iOS runtime run, which the contributor reports doing on device and simulator.

## Solution Assessment

No. The deferred scroll task is a narrow maintainable fix, but a hard-coded Adaptive row is not the best picker fix because the gateway already exposes provider/model-specific thinkingLevels for this purpose.
