## Summary

Keep open: the scroll deferral is narrow, but the picker change still advertises Adaptive for every session even though thinking levels are provider/model-specific and unsupported explicit values are rejected. The user-facing iOS fix also needs a changelog entry before merge.

## What This Changes

The PR hard-codes an Adaptive thinking picker row in shared SwiftUI chat UI and defers streaming assistant auto-scroll into a MainActor task with a rechecked bottom-pin guard.

## Best Possible Solution

Render the Swift thinking picker from gateway thinkingLevels/thinkingDefault metadata, keep the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

No independent iOS runtime reproduction was run in this read-only review. Source inspection gives a high-confidence path for the PR-introduced blocker: selecting the new global Adaptive row on a model whose gateway profile does not support adaptive causes sessions.patch to reject the explicit value.

## Solution Assessment

No. Deferring the streaming scroll with an inner pin-state recheck is maintainable, but hard-coding Adaptive is not; the picker should use existing gateway provider/model thinking metadata.
