## Summary

Keep open. Current main still lacks the active shared native chat UI changes, so this PR is not obsolete, but the proposed picker change should not merge as-is because it exposes Adaptive as a fixed option for every model instead of following the provider-scoped thinkingLevels contract.

## What This Changes

The PR adds an Adaptive row to the shared SwiftUI thinking picker and defers streaming assistant auto-scroll into a MainActor task that rechecks pin state before mutating scrollPosition.

## Best Possible Solution

Keep the deferred streaming-scroll fix with the inner pin-state recheck, but render native thinking picker options from the gateway/session thinkingLevels contract or otherwise gate Adaptive to sessions that currently support or already carry that value. Clean the stale OSLog scope from the PR metadata before landing.

## Reproduction Assessment

Yes, partially. Current-main source shows the picker/options mismatch and synchronous streaming scroll update path; the actual SwiftUI warning/loop is runtime-sensitive and was not live-reproduced during this read-only review, though the contributor provided device/simulator verification in the discussion.

## Solution Assessment

No. The deferred scroll task is a narrow maintainable fix, but adding Adaptive as a hard-coded global picker row is not the best way to solve the thinking picker issue because OpenClaw already has provider-scoped thinkingLevels metadata for valid model-specific options.
