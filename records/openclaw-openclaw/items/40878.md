## Summary

Keep open: current main does not already implement the iOS picker/scroll changes, and the live PR still has a blocking correctness issue because the hard-coded Adaptive picker row is not gated by the session/model thinking metadata that the gateway already exposes.

## What This Changes

The PR adds an Adaptive option to the shared SwiftUI thinking picker and defers streaming auto-scroll updates into a MainActor task with a rechecked bottom-pin guard.

## Best Possible Solution

Render the Swift thinking picker from gateway session/default thinking metadata, keep the deferred scroll task with the inside guard, add the active changelog entry, and attach after-fix proof for both picker and streaming-scroll behavior.

## Reproduction Assessment

Yes for the PR-introduced blocker by source inspection: selecting the hard-coded Adaptive row for a provider/model whose gateway options exclude adaptive will send an invalid sessions.patch value while the Swift view model keeps the optimistic local selection. I did not run an independent iOS simulator/device reproduction.

## Solution Assessment

No. Deferring streaming scroll with a rechecked pin guard is narrow, but hard-coding Adaptive is not; the maintainable fix is to decode and consume the gateway's session-specific thinking metadata in OpenClawKit.
