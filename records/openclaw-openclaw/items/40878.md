## Summary

Keep open: the deferred streaming-scroll change is narrow, but the picker change still hard-codes Adaptive for every session despite provider-scoped thinking support, and the user-facing iOS fix lacks the required changelog entry.

## What This Changes

The PR adds a hard-coded Adaptive option to the shared SwiftUI thinking picker and defers streaming auto-scroll into a MainActor task with a rechecked bottom-pin guard.

## Best Possible Solution

Render native Swift thinking choices from gateway `thinkingLevels`/`thinkingDefault` metadata, keep the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

No independent iOS runtime reproduction was established in this read-only review. Source inspection gives a high-confidence path for the PR-introduced issue: a global Adaptive row can submit unsupported explicit values that `sessions.patch` rejects for non-adaptive provider/model profiles.

## Solution Assessment

No. The deferred scroll task with an inner pin-state recheck is a narrow maintainable fix, but hard-coding Adaptive is not; the native picker should render the gateway's existing provider/model-specific thinking metadata.
