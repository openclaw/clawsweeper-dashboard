## Summary

Keep open: the deferred streaming auto-scroll change is narrow, but the fixed Adaptive picker row can expose an unsupported thinking value for some provider/model sessions, and the user-facing iOS fix still needs a changelog entry.

## What This Changes

The PR adds a hard-coded Adaptive option to the shared SwiftUI thinking picker and defers streaming auto-scroll updates into a MainActor task with a rechecked bottom-pin guard.

## Best Possible Solution

Render the Swift thinking picker from gateway session/default thinking metadata, preserve the deferred scroll guard, and add the active changelog entry before merge.

## Reproduction Assessment

Yes for the PR-introduced blocker by source inspection: selecting the fixed Adaptive row on a provider/model that does not support adaptive will send sessions.patch an invalid thinkingLevel. I did not run an independent iOS simulator/device reproduction in this read-only review.

## Solution Assessment

No. Deferring streaming scroll with a rechecked pin guard is a narrow fix, but hard-coding Adaptive is not; the maintainable solution is to render choices from existing gateway thinking metadata.
