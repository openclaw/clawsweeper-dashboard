## Summary

Keep this PR open for review, but it is not merge-ready: it targets a real plugin gateway RPC bug, yet the live-only lookup drops current main's attached-registry dispatch contract and the branch is now conflicting/outdated.

## What This Changes

The PR changes `handleGatewayRequest` to always rebuild the gateway method registry from the live active plugin registry instead of using a supplied `methodRegistry` snapshot.

## Best Possible Solution

Land one conflict-free gateway dispatch fix that uses the supplied registry when it owns the method, falls back to the live registry only for stale plugin registrations, adds regression coverage for both cases, and includes redacted live CLI proof.

## Reproduction Assessment

Yes. Source inspection shows the reported stale-startup registry bug, and current main also has attached-registry coverage that the PR's live-only lookup would violate after rebase; I did not run the live CLI repro.

## Solution Assessment

No. The runtime layer is the right place to fix the bug, but the best fix must preserve the supplied attached registry when it contains the method and fall back to the live registry only for stale misses.
