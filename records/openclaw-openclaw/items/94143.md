## Summary

Keep open: this is a focused fix for a real plugin gateway dispatch bug, but the branch is currently conflicting and its live-only registry lookup would regress current main's attached-registry dispatch and authorization contract.

## What This Changes

The branch changes `src/gateway/server-methods.ts` so `handleGatewayRequest` always rebuilds its method registry from the live plugin registry instead of using `opts.methodRegistry`.

## Best Possible Solution

Land one conflict-free gateway dispatch fix that preserves the supplied attached registry when it owns the method, falls back to live plugin registry data for stale misses, covers both cases, and includes redacted live CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current-main coverage empties global runtime state while dispatching through a supplied registry, and the PR's unconditional live-registry rebuild would ignore that supplied method; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No, not as written. Gateway dispatch is the right layer, but the best fix must compose or refresh registry sources instead of replacing the attached-registry contract from current main.
