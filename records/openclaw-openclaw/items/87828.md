The PR changes plugin registry snapshot validation to treat bundled plugin records under a matching legacy `extensions` root as compatible with the active bundled root.

## Best Possible Solution

Narrow legacy-root acceptance to roots current discovery would derive as valid source-checkout or mounted-overlay bundled roots, add focused accepted/rejected regression tests, then require real CLI proof before merge.

Yes from source inspection: current main rejects persisted bundled records whose rootDir is outside `resolveBundledPluginsDir()`, which explains the reported source-checkout legacy-root mismatch. I did not run the CLI because this review is read-only.

## Solution Assessment

No; accepting the legacy root is the right repair area, but this PR should share discovery's stricter source-checkout or mounted-overlay trust boundary instead of accepting any computed legacy root.
