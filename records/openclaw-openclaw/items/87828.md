## Summary

## What This Changes

The PR changes plugin registry snapshot validation to accept persisted bundled plugin records under a legacy `extensions` source-overlay root and adds a regression test for that path.

## Best Possible Solution

Keep the compatibility goal, but make snapshot stale-source validation accept only legacy roots that discovery would classify as bundled, then prove it with a valid current persisted-index fixture and redacted current-head refresh/list CLI output.

## Reproduction Assessment

Yes, by source inspection and the linked report: current main compares bundled persisted roots only against the active bundled directory, so legacy source-checkout roots can trigger `persisted-registry-stale-source` after refresh. I did not run the CLI repro in this read-only review.

## Solution Assessment

No. The proposed fix is plausible but too broad because it trusts the whole legacy root rather than the narrower source-checkout or mounted-overlay roots discovery would load, and its regression test does not use the current persisted-index shape.
