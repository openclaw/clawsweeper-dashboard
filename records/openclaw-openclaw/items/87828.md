## Summary

Keep open: the compatibility fix is useful, but the current patch is too broad because it can trust stale source-root bundled records after a packaged bundle exists, diverging from current discovery precedence.

## What This Changes

The branch broadens persisted plugin registry bundled-root validation to accept legacy source-checkout bundled roots alongside the packaged bundled root.

## Best Possible Solution

Accept source-root bundled records only when current discovery would still choose that source path, preserve packaged-peer precedence, and add focused registry snapshot tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can persist source-only bundled records and then reject bundled roots outside `resolveBundledPluginsDir`; the PR body also supplies after-fix CLI output, though I did not run the live CLI sequence.

## Solution Assessment

No. The proposed path accepts the whole legacy source root, while current discovery scans packaged bundles first and skips source directories with packaged peers; the safer fix is per-plugin acceptance that mirrors discovery.
