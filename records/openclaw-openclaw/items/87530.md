## Summary

Keep open: this PR addresses a real Discord multi-account voice problem that current main does not solve, but it is not merge-ready because the branch is conflicting and the delayed autoJoin change leaves focused listener tests expecting immediate autoJoin.

## What This Changes

The PR changes the bundled Discord voice manager/provider to use account-scoped `@discordjs/voice` groups and delayed unref'd autoJoin retries.

## Best Possible Solution

Refresh the branch on current main, keep the account-scoped group and unref'd delayed retries if owner review accepts the timing tradeoff, update focused timer tests, and land after single-account and multi-account Discord voice proof.

## Reproduction Assessment

Yes, at source level: current main calls `getVoiceConnection(guildId)` and `joinVoiceChannel` without a group while `@discordjs/voice` defaults connections to group `default`. I did not run a live Discord multi-account reproduction in this read-only review, but the PR body/comment include after-fix real gateway logs.

## Solution Assessment

Mostly yes: account scoping is the right dependency-owned isolation point, and delayed retries match the missed READY/voice-state timing described by the proof. Before merge, the branch needs a clean refresh and focused tests updated for the new delayed listener contract.
