## What This Changes

The branch scopes Discord voice connections by account ID and schedules unref'd delayed autoJoin retries after Discord voice listeners register.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main uses the @discordjs/voice default connection group while the dependency stores connections by group and guild, so multiple bot accounts in one process/guild can collide. I did not run a live Discord multi-account reproduction in this read-only review.

## Solution Assessment

Yes, this is an appropriate owner-boundary fix: the Discord voice manager owns the join/lookup calls and @discordjs/voice directly supports a group key. The remaining blocker is not the fix shape, but refreshing the conflicted branch and validating the merge result.
