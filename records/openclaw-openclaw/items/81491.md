## Summary

Keep open: the report describes a real channel-liveness failure path on current main, but the requested Slack-specific self-SIGTERM threshold is a new process-lifecycle policy and overlaps an open generic channel restart bug rather than a cleanup-close candidate.

## What This Changes

The issue requests a configurable Slack reconnect escalation that SIGTERMs the gateway after repeated failed reconnects so an external supervisor restarts the whole process.

## Best Possible Solution

Fix the generic channel restart path so an unhealthy Slack account can be replaced cleanly, then add a gateway-wide fatal escalation only if maintainers explicitly want supervisor-driven recovery after bounded in-process repair fails.
