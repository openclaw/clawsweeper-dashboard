## Summary

Keep open: this is a valid narrow bug fix area, but the current PR introduces an overbroad fallback that can apply the wrong context cap for slash-containing model IDs. The linked issue is still open, and the earlier replacement PR usefully showed the safer provider-prefix-only shape, but this branch still needs a corrected matcher and valid real behavior proof before merge.

## What This Changes

This PR adds fallback bare/provider-scoped model-id matching in `resolveContextWindowInfo`, two regression tests, and a changelog entry.

## Best Possible Solution

Implement provider-prefix-only bare/scoped matching inside `resolveContextWindowInfo`, preserve nested slash-containing model IDs, add collision regression coverage, and land with valid redacted runtime proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main strict-matches configured model IDs while the embedded runner forwards the runtime `modelId`, so a `provider/model` runtime ref can miss a bare configured context cap.

## Solution Assessment

No. Fixing this resolver is the right location, but the PR should strip or prepend only the active provider prefix instead of comparing final slash segments that can collide.
