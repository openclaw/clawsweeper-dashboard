## Summary

Keep open: the underlying bug is source-reproducible on current main, but this PR’s fallback can overmatch slash-containing model IDs and the supplied real-behavior proof is inconsistent with the current API.

## What This Changes

This PR adds fallback bare/provider-scoped model-id matching in `resolveContextWindowInfo`, two regression tests, and a changelog fix entry.

## Best Possible Solution

Implement provider-prefix-only bare/scoped matching that preserves nested model IDs, add collision regression coverage, and land with valid redacted runtime proof.

## Reproduction Assessment

Yes for source reproduction. Current main exact-matches configured model IDs while the embedded runner forwards the runtime `modelId`, so a provider-scoped runtime ref misses a bare configured context cap.

## Solution Assessment

No. Bare/scoped equivalence belongs in this resolver, but the PR should strip or prepend only the active provider prefix instead of comparing final slash segments that can collide.
