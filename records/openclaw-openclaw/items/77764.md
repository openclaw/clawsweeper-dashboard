## Best Possible Solution

Keep the fix in `resolveContextWindowInfo`, but strip or prepend only the active provider prefix, preserve nested model IDs, add collision coverage, and provide corrected real runtime proof.

## Reproduction Assessment

## Solution Assessment

No. The resolver is the right place to fix the issue, but this implementation should preserve the full nested model ID and only normalize the active provider prefix.
