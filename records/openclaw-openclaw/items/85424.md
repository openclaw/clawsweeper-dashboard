## Summary

Keep open. The runtime fix is plausible and source-backed, but this PR is not merge-ready because it changes the public gateway handshake default without updating the docs/type contract and still needs inspectable after-fix real behavior proof.

## What This Changes

The PR raises the default gateway handshake budget, removes hardcoded 10s CLI wrapper timeouts, adds event-loop drift details to timeout errors, and updates related gateway/CLI tests.

## Best Possible Solution

Land only after the docs/type/protocol defaults are aligned, redacted after-fix slow-host CLI proof is attached, and maintainers accept the longer default timeout tradeoff.

## Reproduction Assessment

Yes, from source inspection: current main hardcodes 10s in the CLI wrappers and gates callGateway's default so the shared 15s default is unreachable when env/config is unset. I did not execute the slow-host reproduction during this read-only review.

## Solution Assessment

Partly. Forwarding undefined from wrappers to callGateway is the right repair shape, but the global 45s default needs aligned docs, real behavior proof, and maintainer acceptance of the timing tradeoff.
