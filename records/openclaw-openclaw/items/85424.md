## Summary

## What This Changes

The PR removes hardcoded 10s gateway timeouts from selected CLI wrappers, makes callGateway use the shared handshake default when unset, and adds event-loop drift details to timeout errors and tests.

## Best Possible Solution

Land the narrowed resolver/wrapper fix after conflict resolution and low-risk wording cleanup, keeping the documented 15s default and treating >15s startup starvation as separate follow-up work.

## Reproduction Assessment

Yes. Source inspection shows current main caps the unconfigured wrapper path at 10s, and the author supplied redacted before/after terminal output showing the same devices command moving from a 10000ms timeout to a successful paired-device listing.

## Solution Assessment

Yes, for the narrowed bug. Forwarding undefined from wrappers and resolving the shared handshake default in callGateway is the maintainable fix; the reverted 45s default should stay out of this PR.
