## Summary

Keep this PR open and require changes before merge: the Codex delivery direction is valid and the supplied live output is useful, but the branch currently exposes task state mutation as a public plugin SDK surface and its gateway reconciler can still skip finalized Codex-native rows after a restart or delayed transcript scan.

## What This Changes

Adds Codex-native subagent monitoring/reconciliation, a generic task-runtime SDK helper, a gateway sidecar, and tests so native `spawn_agent` child completions can wake parent sessions and deliver visible replies.

## Best Possible Solution

Keep the Codex-native completion delivery fix, but make the task-runtime SDK contract explicitly private/reserved or maintainer-approved public API, then update the sidecar to recover terminal undelivered rows with regression coverage.

## Reproduction Assessment

Yes for source-level review: current main mirrors Codex-native child lifecycle into task rows but has no durable path that turns terminal child transcript results into parent completion delivery. The PR body also supplies macOS native gateway plus Discord live-output proof, but I did not rerun the live transport in this read-only review.

## Solution Assessment

No as proposed: the product direction is right, but the patch needs a safer SDK boundary and terminal-row recovery fix before it is the best maintainable solution. The narrow repair is to keep task mutation private or explicitly approved and align the gateway sidecar predicate with the monitor’s terminal-undelivered recovery behavior.
