This PR is a real fix for Codex-native `spawn_agent` completion loss and has useful live Discord proof, but it is not merge-ready because the gateway startup reconciler still skips a restart-loss case and the patch promotes task-state mutation helpers into the public plugin SDK without a settled boundary.

## What This Changes

Adds a Codex-native subagent monitor/reconciler, task-completion handoff delivery, a generic `plugin-sdk/task-runtime` subpath, docs, and regression coverage for native `spawn_agent` completions.

## Best Possible Solution

Land a Codex-owned completion adapter that preserves delivery across restart and keeps task registry mutation behind an explicitly approved SDK or private bundled-plugin boundary.
