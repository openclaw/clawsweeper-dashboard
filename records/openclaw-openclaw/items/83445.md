Full review comments:

- **[P1] Avoid queuing and direct-running the same completion:** `src/agents/subagent-announce-delivery.ts:706`
  - body: This branch is inside `if (params.expectsCompletionMessage && requesterActivity.sessionId)`, so adding `&& !params.expectsCompletionMessage` makes the queued-return path unreachable. A queued wake calls `queueMessage(text, ...)` on the active parent run, then the code continues into the direct agent call with the same `triggerMessage`, so active completions can be processed twice and produce duplicate visible replies.
  - confidence: 0.9
- **[P2] Clear monitors when the app-server client closes:** `extensions/codex/src/app-server/native-subagent-monitor.ts:136-143`
  - body: `CodexNativeSubagentMonitor` starts a reconcile interval in the constructor, but cleanup is only registered when `clearOnClientClose` is true, and the production registration path never sets that option. Retired Codex app-server clients can therefore leave transcript-scanning timers alive after close.
  - confidence: 0.87
- **[P3] Keep the Codex plugin inside the SDK boundary:** `extensions/codex/src/app-server/native-subagent-monitor.ts:13`
  - body: This production extension file imports `TaskRecord` from `../../../../src/...`, crossing out of the plugin package. The scoped extension rules require plugin production code to use `openclaw/plugin-sdk/*` or local barrels, so expose the needed type through the private Codex SDK helper or use a local narrow shape instead.
