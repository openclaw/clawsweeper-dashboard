## What This Changes

The issue requests a deterministic Google/Gmail/Drive OAuth recovery path, better active auth-state UX, reauth loop protection, and immediate Gmail/Drive validation after recovery.

## Best Possible Solution

Define an OpenClaw-owned recovery contract that points to upstream gog auth doctor/remote/manual primitives, adds clear Gmail/gog diagnostics and docs, and optionally validates Gmail read/send plus Drive access after recovery without moving token exchange/storage ownership into OpenClaw core.

## Solution Assessment

Unclear as a full implementation because the issue spans docs, diagnostics, validation UX, and upstream gog behavior. A focused OpenClaw runbook plus auth-failure diagnostics around gog's existing primitives looks like the narrowest maintainable first step.
