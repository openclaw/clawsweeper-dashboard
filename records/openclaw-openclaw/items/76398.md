## Summary

Keep open: current main still has the unsafe omitted-detached Unix group-kill path, but this PR should not merge as-is because it misses a detached LSP cleanup caller and still includes unrelated Claude CLI streaming/changelog work.

## What This Changes

The PR flips `killProcessTree` to direct-pid by default, updates selected gateway/MCP callers and tests, and also adds Claude CLI tool-event streaming plus a release-script lint cleanup.

## Best Possible Solution

Land a focused process fix that defaults omitted `detached` to direct-pid kill, explicitly preserves known detached process-tree cleanup such as MCP and LSP, updates regression tests and the #76259 changelog entry, and leaves #75846 streaming to its own PR.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted `detached` still reaches Unix group-kill, and the PR diff shows the default flip while leaving the detached LSP termination caller with only `{ graceMs }`.

## Solution Assessment

No. The safe default is the right direction, but the patch is incomplete until detached LSP cleanup is preserved and the unrelated #75846 streaming and redundant script cleanup are split out.
