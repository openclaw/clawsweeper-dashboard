## Summary

Keep open. The #76259 process-kill regression is source-backed and the default flip is a plausible direction, but this PR is not merge-ready because it misses at least one detached POSIX caller and carries unrelated Claude CLI tool-streaming/changelog changes.

## What This Changes

The PR flips `killProcessTree` to direct-PID kill by default, updates some process-kill callers/tests, and also includes unrelated Claude CLI tool-event streaming, release-script lint, and changelog changes.

## Best Possible Solution

Land a focused #76259 fix that makes direct-PID kill the safe default, explicitly annotates every known detached process-tree caller, updates regression tests and the correct changelog entry, and moves the Claude CLI tool-streaming work to a separate review path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on current main, any POSIX caller that omits `detached:false` reaches `killProcessTreeUnix(..., true)` and attempts `process.kill(-pid, ...)`. I did not run a live macOS launchd reproduction because this was a read-only review.

## Solution Assessment

No. The proposed default flip is a reasonable narrow fix direction, but the PR must first update all detached callers, at least `src/agents/pi-bundle-lsp-runtime.ts`, and remove or split the unrelated tool-streaming changes.
