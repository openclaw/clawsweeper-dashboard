## Summary

Keep open. The process fix direction is valid, but this PR is not merge-ready because the default flip would leave at least one known detached LSP caller without process-tree cleanup, the branch mixes in unrelated Claude CLI tool-event streaming work, and the changelog entry documents the wrong issue.

## What This Changes

The PR changes `killProcessTree` to require explicit `detached: true` for Unix process-group kills, updates selected MCP/gateway callers and tests, and also adds unrelated Claude CLI tool-event streaming code and a #75846 changelog entry.

## Best Possible Solution

Land a focused process fix that makes Unix group-kill safe only for explicit or verified process-group leaders, preserves known detached process-tree cleanup such as MCP and LSP, and carries the correct #76259 changelog entry.

## Reproduction Assessment

Yes. Source inspection reproduces the current bug path: omitted `detached` still reaches `process.kill(-pid, ...)`, and the PR diff shows the default flip while leaving a detached LSP caller with only `{ graceMs }`.

## Solution Assessment

No. The safe default is the right direction, but this patch is incomplete until it preserves all known detached child cleanup or moves the invariant into `kill-tree`, and the unrelated #75846 streaming changes should be split out.
