## Summary

Keep open: the process-kill direction is valid, but the patch is not merge-ready because the default flip leaves at least one detached process-tree caller relying on the old omitted-option behavior and the branch is scoped dirty.

## What This Changes

The PR changes Unix `killProcessTree` to group-kill only with explicit `detached: true`, updates several callers/tests, and also carries unrelated Claude CLI tool-event streaming plus a release-check lint tweak.

## Best Possible Solution

Land a process-only fix that makes direct-pid kill the safe default, explicitly preserves group-kill for known detached process-tree owners, adds regression coverage and the #76259 changelog entry, and moves unrelated streaming work to its own PR.

## Reproduction Assessment

Yes, from source inspection and the linked macOS report: current main group-kills on Unix whenever `detached` is omitted, and #76259 gives a concrete launchd-managed gateway failure pattern. I did not run a live macOS reproduction in this read-only review.

## Solution Assessment

No, not as proposed. Flipping the default is a maintainable direction, but it must first preserve group-kill for detached owners such as bundled LSP and remove unrelated branch content.
