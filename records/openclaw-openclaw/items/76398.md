## Summary

Keep open. Current main still has the unsafe Unix group-kill default, and the PR moves in the right direction, but it is not merge-ready because it misses a detached LSP caller and carries unrelated Claude CLI streaming/release-script changes.

## What This Changes

The PR flips `killProcessTree` on Unix to direct-PID kill by default, updates selected callers/tests, adds Claude CLI tool-event streaming changes, tweaks a release-script regex, and adds a Claude streaming changelog entry.

## Best Possible Solution

Land a scoped process-only patch that makes direct PID kill the safe default while explicitly preserving group kill for verified detached process trees and documenting the #76259 fix.

## Reproduction Assessment

Yes. The source-level reproduction is current main's Unix `killProcessTree(6666, { graceMs: 10 })` path, which the existing test asserts sends `SIGTERM` to `-6666`; the linked issue also provides a macOS launchd runtime reproduction.

## Solution Assessment

No. The default flip is likely the right direction, but the current PR misses at least one detached caller and should be reduced to the process fix with matching tests and changelog.
