## Summary

Keep open: current main still drops direct Amazon Nova Bedrock cache-retention settings and then routes direct non-Anthropic Bedrock models through a no-cache wrapper, while the current AWS Bedrock contract documents Nova prompt caching support.

## What This Changes

The issue requests native prompt caching support for Amazon Nova models in the bundled Amazon Bedrock provider.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin, with Nova model-family eligibility, Nova-safe cachePoint handling, focused regression tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit cacheRetention for direct amazon.nova-* IDs resolves to undefined and the provider wrapper then forces those direct non-Anthropic Bedrock models to cacheRetention: "none".

## Solution Assessment

Yes. The narrow maintainable path is a bundled Amazon Bedrock provider change that recognizes AWS-supported Nova IDs and injects only Nova-compatible cache checkpoints while leaving unsupported non-Anthropic Bedrock models on the no-cache path.
