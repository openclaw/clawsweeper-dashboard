## Summary

Keep open: current main and v2026.6.8 still encode the Amazon Nova no-cache behavior, while AWS now documents Bedrock prompt caching and AWS’s own launch/blog material includes Nova Micro, Lite, and Pro support. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html?utm_source=openai))

## What This Changes

Request native Amazon Nova prompt-caching support in the bundled Amazon Bedrock provider without sending unsupported Claude-only cache metadata.

## Best Possible Solution

Add Nova-specific prompt-cache handling inside the Amazon Bedrock provider, preserving no-cache behavior for unsupported Bedrock families and avoiding Claude-only TTL assumptions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Nova IDs fail the current cache-support gate, the provider wrapper forces direct non-Anthropic Bedrock calls to `cacheRetention: "none"`, and tests assert that behavior for `amazon.nova-micro-v1:0`.

## Solution Assessment

Yes, but the narrow fix is provider-owned Nova support, not removing the Bedrock guard globally. The implementation should recognize documented Nova cache-capable IDs, inject Bedrock-compatible checkpoints, and keep unsupported families on the no-cache path.
