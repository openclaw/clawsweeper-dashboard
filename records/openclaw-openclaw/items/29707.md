## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/29707 by adding Amazon Nova prompt-caching support in the Amazon Bedrock provider path. Current main drops explicit `cacheRetention` for direct `amazon.nova-*` IDs in `resolveCacheRetention` and wraps direct non-Anthropic Bedrock IDs with `createBedrockNoCacheWrapper`. Recognize only AWS-supported Nova direct, region-prefixed, and global model IDs; pass explicit `cacheRetention` through for Nova; inject only Bedrock/Nova-compatible `cachePoint: { type: "default" }` checkpoints into eligible system/message payload positions; do not add `ttl: "1h"` for Nova unless current AWS docs or live proof confirm support; keep Mistral and other unsupported non-Anthropic Bedrock models on the no-cache path. Keep the policy provider-owned except for small typed shared helpers, and update tests, public docs, and changelog together.
