## Summary

Keep open: current main and the latest release still force direct Amazon Nova Bedrock models through the no-cache path, while current AWS Bedrock docs and the pinned SDK support explicit cache checkpoints. This remains a focused Amazon Bedrock provider enhancement with no open replacement PR.

## What This Changes

The issue requests native Amazon Nova prompt caching support in the Amazon Bedrock provider without sending unsupported cache metadata to other Bedrock models.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin: recognize supported Nova model IDs, preserve valid explicit cache retention, emit only Bedrock/Nova-compatible cache checkpoints, and keep unsupported non-Anthropic Bedrock models on the no-cache path.

## Reproduction Assessment

Yes, source-level: current source and tests show direct Nova is forced to no-cache, and AWS docs/SDK types show Bedrock supports explicit cache checkpoints. I did not run a live AWS Nova turn during this read-only review.

## Solution Assessment

Yes, if implemented narrowly: the maintainable path is provider-local Nova cache eligibility and Bedrock-compatible payload handling, not a broad pass-through for every non-Anthropic Bedrock model.
