## Summary

Keep open: current main still blocks direct Amazon Nova Bedrock prompt caching by dropping explicit cache retention, wrapping direct Nova IDs with `cacheRetention: "none"`, and documenting/tests encoding Nova as disabled while AWS now documents Nova prompt caching support.

## What This Changes

Request native prompt caching for Amazon Nova Bedrock models by preserving `cacheRetention` and using Nova-compatible cache checkpoint handling instead of the non-Anthropic no-cache path.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin and shared cache-retention helpers, limited to AWS-supported Nova IDs, with Nova-safe cache points and tests/docs/changelog updated together.

## Reproduction Assessment

Yes. Source inspection and current tests show direct Nova IDs drop explicit `cacheRetention` and then get forced through the Bedrock no-cache wrapper; no live AWS call is needed to prove OpenClaw currently blocks the setting.

## Solution Assessment

Yes. A narrow provider-owned Nova implementation is the maintainable path, but it should avoid reusing Claude-only `ttl: "1h"` behavior unless current AWS docs or live proof confirms Nova support for that TTL.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/29707 by adding native Amazon Nova prompt-cache support for the bundled Amazon Bedrock provider. Current main drops explicit `cacheRetention` for direct Nova IDs in `resolveCacheRetention`, wraps direct non-Anthropic Bedrock IDs with `cacheRetention: "none"`, and documents/tests Nova as disabled. Implement this inside the provider/cache-retention boundary: recognize AWS-documented Nova model IDs, preserve explicit `cacheRetention` where valid, inject only Bedrock/Nova-compatible `cachePoint` checkpoints for `system` and `messages`, keep unsupported non-Anthropic Bedrock models on the no-cache path, and do not apply Claude-only `ttl: "1h"` to Nova unless AWS docs or a live probe proves that support. Update focused tests, public prompt-caching/Bedrock docs, and the changelog; do not broaden core with provider-specific policy outside the existing Bedrock/provider helper seams.
