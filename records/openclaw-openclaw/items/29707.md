## Summary

Keep open: current main and the latest shipped release still exclude direct Amazon Nova Bedrock model IDs from cache-retention eligibility and force them onto the no-cache path, while current AWS Bedrock docs show Nova prompt caching support and Nova-compatible cachePoint usage. This remains a narrow provider enhancement with no canonical merged fix yet. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html?utm_source=openai))

## What This Changes

Requests native prompt-cache support for Amazon Nova on Bedrock by allowing valid cacheRetention/cachePoint handling instead of forcing all non-Anthropic Bedrock models to no-cache.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported Nova model ID forms, preserve explicit retention only where AWS supports it, inject Bedrock/Nova-compatible cachePoint checkpoints, keep unsupported non-Anthropic Bedrock models on the no-cache path, and update tests, docs, and changelog together.

## Repair Work Prompt

Implement native Amazon Nova prompt caching support for https://github.com/openclaw/openclaw/issues/29707. Current main and v2026.5.12 force direct non-Anthropic Bedrock models such as amazon.nova-* through cacheRetention: "none" and resolver tests expect Nova to be ineligible, while AWS now documents Nova prompt caching and cachePoint support. Keep this provider-owned in the Amazon Bedrock plugin: recognize supported direct, regional, geo/global Nova IDs; inject only Bedrock/Nova-compatible cachePoint markers on supported fields; preserve unsupported non-Anthropic Bedrock models on the no-cache path; do not broadly pass cacheRetention for all amazon-bedrock/openai-completions models; and do not map cacheRetention: "long" to ttl: "1h" for Nova unless current AWS docs or a live probe proves that exact model supports it. Update focused tests, public docs, and changelog; coordinate with the related open cacheRetention PR so its broader resolver change does not bypass the Bedrock guard.
