## Summary

Keep open: current main and v2026.6.5 still classify Bedrock prompt caching as Claude-only and force direct Amazon Nova models through the no-cache wrapper, while current AWS documentation supports explicit prompt caching for Nova. No open replacement PR owns the remaining work, and the correct boundary is a focused change in the existing Amazon Bedrock plugin.

## What This Changes

Allow supported Amazon Nova models on Bedrock to use native prompt caching without re-enabling incompatible caching behavior for unsupported model families.

## Best Possible Solution

Extend the Amazon Bedrock plugin's cache-capability model and payload shaping for AWS-documented Nova models, while retaining the existing no-cache path for unsupported models and preserving Claude behavior.

## Reproduction Assessment

Yes, at source level: current main's Claude-only predicate excludes direct Nova IDs and the selected wrapper forces `cacheRetention: "none"`. A live AWS request is still required for fix verification, not for confirming the current source gap.

## Solution Assessment

Yes, with a narrower boundary than blanket pass-through: add Nova-specific eligibility and AWS-compatible cache-point handling in the Bedrock plugin, while keeping unsupported non-Anthropic models protected.
