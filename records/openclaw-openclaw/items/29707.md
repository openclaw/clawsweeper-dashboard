## Summary

Keep open. Current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and forces those requests through the Bedrock no-cache wrapper, while current AWS Bedrock docs support Nova prompt caching and recommend explicit prompt caching for cost savings.

## What This Changes

Request native prompt-cache support for Amazon Nova models on Bedrock by allowing cache retention and using Nova-compatible cache checkpoints.

## Best Possible Solution

Ship a provider-owned Nova cache implementation in the bundled Amazon Bedrock plugin and shared retention helpers, with tests/docs/changelog updated and unsupported Bedrock families still protected.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: resolveCacheRetention({ cacheRetention: "long" }, "amazon-bedrock", "openai-completions", "amazon.nova-micro-v1:0") resolves to undefined, and direct Nova streams then receive cacheRetention: "none" from the wrapper.

## Solution Assessment

Yes. A focused Amazon Bedrock provider repair is the narrowest maintainable solution, but it should add Nova-specific cache-point handling instead of blindly reusing Claude-only cache semantics or only removing the wrapper.
