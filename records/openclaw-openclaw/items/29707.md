## Summary

Keep open: current main still drops direct Amazon Nova Bedrock cache-retention settings and then forces direct non-Anthropic Bedrock model IDs through a no-cache wrapper, while AWS now documents Nova prompt-caching support.

## What This Changes

Requested native Amazon Nova prompt-caching support on Bedrock using the existing `cacheRetention` surface and Nova-compatible cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin, with model-family eligibility, Nova-safe `cachePoint` handling, focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `cacheRetention` for direct `amazon.nova-*` Bedrock IDs resolves to `undefined`, then the provider wrapper forces those direct non-Anthropic Bedrock models to `cacheRetention: "none"`.

## Solution Assessment

Yes. The narrow maintainable solution is an Amazon Bedrock provider change that recognizes AWS-supported Nova IDs and injects only Nova-compatible cache checkpoints while leaving unsupported non-Anthropic Bedrock models on the no-cache path.
