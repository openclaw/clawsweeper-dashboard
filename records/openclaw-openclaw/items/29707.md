## Repair Work Prompt

Fix native Amazon Nova prompt caching for the Amazon Bedrock provider without enabling cache points for every non-Anthropic Bedrock model. Current main still forces direct non-Anthropic Bedrock IDs through `cacheRetention: "none"` in `extensions/amazon-bedrock/register.sync.runtime.ts`, and `src/agents/embedded-agent-runner/prompt-cache-retention.ts` leaves `amazon-bedrock + amazon.* nova` outside cache eligibility. Add a provider-owned Nova eligibility contract for AWS-documented cache-capable Nova models, preserve explicit `cacheRetention` for those models, shape Bedrock Converse payloads using Nova-supported `cachePoint` placement, avoid sending unsupported `ttl: "1h"` to Nova unless AWS documents it for that model, and keep the existing no-cache behavior for unsupported non-Anthropic Bedrock models. Update focused tests and public docs if behavior changes. Do not edit `CHANGELOG.md`. Validate with the narrow tests and include live Bedrock Nova proof before landing, or clearly state why credentials were unavailable.
