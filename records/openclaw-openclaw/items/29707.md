## Summary

Keep open: current main still treats direct Amazon Nova Bedrock IDs as non-Anthropic and forces them to `cacheRetention: "none"`, while AWS now documents Nova prompt caching and explicit `cachePoint` support.

## What This Changes

Request to add native Amazon Nova prompt caching support on Bedrock by allowing compatible `cacheRetention`/`cachePoint` handling instead of forcing non-Anthropic Bedrock models to no-cache.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin, with explicit Nova model eligibility, Nova-compatible `cachePoint` placement, focused tests, public docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit Nova `cacheRetention` is dropped by `resolveCacheRetention`, then direct Nova model IDs are forced through the Amazon Bedrock no-cache wrapper.

## Solution Assessment

Yes. The narrow maintainable path is a provider-owned Amazon Bedrock change that recognizes AWS-supported Nova IDs and emits only Bedrock/Nova-compatible cache checkpoints while keeping unsupported non-Anthropic Bedrock models on the no-cache path.
