## Summary

Current main still deliberately disables prompt caching for Nova, while current AWS documentation supports explicit prompt caching for Amazon Nova models. The issue remains necessary and has a bounded provider-owned implementation path, but live Bedrock proof should accompany the fix.

## What This Changes

Enable native Amazon Nova prompt caching through the Amazon Bedrock provider without sending incompatible cache metadata.

## Best Possible Solution

Replace the Claude-versus-everything gate with provider-owned model capability handling that emits documented Nova-compatible checkpoints, preserves safe no-cache behavior for unsupported models, and proves cache token accounting in a real two-turn Bedrock run.

## Reproduction Assessment

Yes for the OpenClaw limitation: current main deterministically classifies Nova as non-cacheable and overwrites cacheRetention with none. AWS documentation confirms the missing provider capability, although a live request is still required for end-to-end proof.

## Solution Assessment

No to globally removing the restriction or only passing cacheRetention through. The narrow maintainable solution is a model-family capability policy that creates Nova-compatible cache checkpoints while retaining the existing protection for unsupported Bedrock models.
