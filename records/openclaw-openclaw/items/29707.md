## Summary

Keep open: current main and v2026.6.8 still force direct Amazon Nova Bedrock models off prompt caching, while current AWS Bedrock docs show Nova-compatible prompt caching support. This is a bounded provider repair, not a close candidate.

## What This Changes

The issue requests native Amazon Nova prompt caching support on Bedrock by removing the current non-Anthropic cacheRetention block and avoiding incompatible cachePoint handling for Nova.

## Best Possible Solution

Add provider-owned Nova prompt-cache handling in the Amazon Bedrock plugin, preserving no-cache behavior for unsupported Bedrock families and avoiding Claude-only TTL assumptions for Nova.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: direct Nova cache retention is dropped by the embedded runner gate and the Bedrock wrapper forces non-Anthropic models to cacheRetention: "none". I did not run a live AWS Bedrock request in this read-only review.

## Solution Assessment

Yes, with a narrower shape than removing the guard globally: the maintainable fix belongs in the Amazon Bedrock provider and should recognize only documented Nova cache-capable models while leaving unsupported families on the no-cache path.
