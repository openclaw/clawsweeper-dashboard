## Summary

Keep open: current main and v2026.6.6 still force direct Amazon Nova Bedrock models through `cacheRetention: "none"`, while current AWS Bedrock docs and Nova model cards document Nova prompt caching support.

## What This Changes

Allow Amazon Nova models on Bedrock to use native prompt caching instead of treating all non-Anthropic Bedrock models as cache-ineligible.

## Best Possible Solution

Add provider-owned Nova prompt-cache capability handling in the Amazon Bedrock plugin, with Nova-specific eligibility, `cachePoint` placement, 5-minute TTL semantics, docs, and tests while preserving no-cache behavior for unsupported Bedrock models.

## Reproduction Assessment

Yes, at source level: direct Nova IDs miss the current cache-eligibility checks and then hit the Bedrock no-cache wrapper. I did not run a live Nova request in this read-only review.

## Solution Assessment

Yes, the best path is a narrow Amazon Bedrock provider repair rather than globally removing the non-Anthropic guard. The fix should recognize only supported Nova families and emit Nova-compatible cache checkpoints.
