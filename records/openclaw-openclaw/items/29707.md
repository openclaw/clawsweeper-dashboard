## Summary

Keep open: current main still implements, tests, and documents the direct Amazon Nova Bedrock no-cache path, while the requested Nova prompt-cache support remains a narrow provider-owned enhancement.

## What This Changes

Request to enable native prompt caching for supported Amazon Nova Bedrock models instead of forcing non-Anthropic Bedrock traffic to `cacheRetention: "none"`.

## Best Possible Solution

Implement Nova-specific Bedrock prompt-cache handling inside the Amazon Bedrock plugin and shared cache-retention helpers, limited to documented Nova IDs and Bedrock-compatible checkpoint and TTL semantics, with tests, docs, and changelog updated together.

## Reproduction Assessment

Yes. Source inspection and existing tests show direct Nova IDs currently drop explicit `cacheRetention` and then get forced through the Bedrock no-cache wrapper; no live AWS call is needed to prove current-main behavior.

## Solution Assessment

Yes, with one refinement. The requested provider-owned fix is the right direction, but it should implement Nova's Bedrock-compatible cache-point contract instead of reusing Claude-only long-TTL semantics.
