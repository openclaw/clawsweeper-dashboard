## Summary

Keep open: current main still has no Nova-specific Bedrock prompt-cache support, and the existing guards intentionally preserve the older non-Anthropic no-cache behavior. AWS now documents Nova prompt caching, so this remains a focused Amazon Bedrock provider enhancement rather than a closeable cleanup item.

## What This Changes

The issue requests native Amazon Nova prompt-caching support through the bundled Amazon Bedrock provider instead of forcing Nova requests onto the no-cache path.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin: recognize supported direct, regional, and global Nova model IDs, preserve valid explicit cache settings, inject only AWS/Nova-compatible cache checkpoints, keep unsupported non-Anthropic models on the no-cache path, and update tests, docs, and changelog together.

## Repair Work Prompt

Fix native Amazon Nova prompt-caching support for the bundled Amazon Bedrock provider. Current main drops explicit cache retention for direct Nova model IDs and wraps direct non-Anthropic Bedrock streams with `cacheRetention: "none"`; AWS now documents Nova-compatible prompt caching with `cachePoint`. Implement this inside the Amazon Bedrock plugin/cache-retention path, not as a new config option and not in unrelated providers. Recognize supported direct, regional, and global Nova model IDs such as `amazon.nova-micro-v1:0`, `amazon.nova-lite-v1:0`, `amazon.nova-pro-v1:0`, `amazon.nova-premier-v1:0`, and `amazon.nova-2-lite-v1:0`; preserve explicit cache retention where valid; inject only AWS/Nova-compatible `cachePoint` checkpoints for the Converse payload; keep unsupported non-Anthropic Bedrock models on the existing no-cache path; and do not map `cacheRetention: "long"` to Nova `ttl: "1h"` unless AWS docs or a live call prove that exact Nova TTL support. Update focused tests, public docs (`docs/reference/prompt-caching.md`, `docs/providers/bedrock.md`, and any Anthropic Bedrock note that says all non-Anthropic Bedrock is forced to none), and `CHANGELOG.md` because this is user-visible. Related context: https://github.com/openclaw/openclaw/issues/29707, https://github.com/openclaw/openclaw/issues/20815, https://github.com/openclaw/openclaw/pull/20866, https://github.com/openclaw/openclaw/pull/22303, https://github.com/openclaw/openclaw/pull/69953. Validate with targeted Vitest and formatting; use Testbox/Crabbox for broad changed checks. If AWS credentials are available, run a focused Bedrock Nova ConverseStream live probe proving payload acceptance and cache read/write accounting.
