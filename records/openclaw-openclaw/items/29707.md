## Summary

Keep open: current main and the latest release still force direct Amazon Nova Bedrock models out of prompt-cache retention, while the current AWS Bedrock contract documents Nova prompt caching and recommends explicit prompt caching for cost savings.

## What This Changes

Request native Amazon Nova prompt caching support on Bedrock instead of forcing direct Nova models to `cacheRetention: "none"`.

## Best Possible Solution

Implement Nova prompt-cache capability in the Amazon Bedrock provider with Nova-specific eligibility, default 5-minute `cachePoint` semantics, docs, and tests while preserving no-cache behavior for unsupported Bedrock families.

## Reproduction Assessment

Yes, at source level: direct Nova IDs miss the current cache-eligibility gates and then hit the Bedrock wrapper that forces `cacheRetention: "none"`. I did not run a live Bedrock Nova request in this read-only review.

## Solution Assessment

Yes, the best path is a narrow Amazon Bedrock provider repair rather than globally removing the non-Anthropic guard. The fix should recognize AWS-documented Nova cache support and emit Nova-compatible checkpoints without reusing Claude-only TTL behavior.
