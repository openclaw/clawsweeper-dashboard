## Summary

Keep this open: current main still deliberately disables prompt-cache retention for direct Amazon Nova Bedrock model IDs, while current AWS Bedrock docs now list Nova prompt caching support with 5-minute TTL checkpoints on `system` and `messages`. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html?utm_source=openai))

## What This Changes

Request native Amazon Nova prompt caching support for Bedrock instead of routing Nova through the non-Anthropic no-cache path.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin, limited to documented Nova model refs and Bedrock-compatible checkpoints, while keeping unsupported non-Anthropic Bedrock models on the no-cache path.

## Reproduction Assessment

Yes. Source inspection and existing tests show direct Nova IDs currently drop explicit `cacheRetention` and then get `cacheRetention: "none"` through the Bedrock wrapper; no live AWS call was needed to prove that current-main behavior.

## Solution Assessment

Yes, with one refinement. The right fix belongs in the Amazon Bedrock plugin/provider cache helpers, but it should implement Nova's documented 5-minute `cachePoint` contract instead of reusing Claude-only 1-hour TTL semantics.
