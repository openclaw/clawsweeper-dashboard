## Summary

Keep open: current main and v2026.6.8 still force direct Amazon Nova Bedrock models off prompt caching, while current Amazon Bedrock docs show Nova-compatible prompt caching support.

## What This Changes

Request native prompt-caching support for Amazon Nova models on the Amazon Bedrock provider without reintroducing unsupported Claude-only cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache handling in the Amazon Bedrock plugin, with Nova-specific eligibility, safe 5-minute checkpoint semantics, docs, and tests while preserving no-cache behavior for unsupported Bedrock models.

## Reproduction Assessment

Yes, at source level: direct Nova IDs miss the cache-eligibility checks and then hit the Bedrock no-cache wrapper. I did not run a live Nova request in this read-only review.

## Solution Assessment

Yes, with a narrower shape than globally removing the guard: the maintainable fix belongs in the Amazon Bedrock provider and should recognize only documented Nova cache-capable models.
