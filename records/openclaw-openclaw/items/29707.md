## Summary

Keep open: current main still blocks direct Amazon Nova Bedrock prompt caching even though current AWS Bedrock docs document Nova cache checkpoints and 5-minute TTL support.

## What This Changes

Allow supported Amazon Nova models on Amazon Bedrock to use native prompt caching instead of being forced to `cacheRetention: "none"`.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin with documented Nova eligibility, Nova-safe cachePoint injection, TTL handling, regression tests, docs, and changelog updates.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `cacheRetention` for direct `amazon.nova-*` IDs is dropped and then the Bedrock provider wrapper forces `cacheRetention: "none"`; current tests and docs encode that disabled behavior.

## Solution Assessment

Yes. The narrow maintainable path is an Amazon Bedrock provider change that recognizes AWS-supported Nova IDs and injects only Bedrock/Nova-compatible cache points while leaving unsupported non-Anthropic models on the no-cache path.
