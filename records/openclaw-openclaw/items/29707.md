## Summary

Keep open: current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and forces them through the no-cache wrapper, while AWS now documents Nova explicit prompt caching. This remains a focused Amazon Bedrock provider enhancement with clear source proof and no closing PR on the issue.

## What This Changes

Allow Amazon Nova models on Bedrock to use native prompt caching through existing cache-retention settings without sending invalid Claude-only cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin while keeping unsupported non-Anthropic Bedrock models on the no-cache path.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the unsupported behavior: direct Nova IDs are not cache-retention eligible and are wrapped with `cacheRetention: "none"`; no live AWS call was performed.

## Solution Assessment

Yes. The narrow maintainable direction is an Amazon Bedrock provider enhancement that follows the AWS Nova cache contract and avoids blindly reusing Claude-only TTL semantics.
