## What This Changes

Allow Amazon Nova models on Amazon Bedrock to use native Bedrock prompt caching by replacing the blanket non-Anthropic no-cache routing with Nova-aware cache eligibility and payload shaping.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the Amazon Bedrock plugin with AWS-documented model and TTL gating, while keeping unsupported non-Anthropic Bedrock models on the no-cache path.

Yes, source-level: current main drops direct Nova cacheRetention in the resolver and then forces direct Nova Bedrock requests through the no-cache wrapper. I did not run a live AWS Nova request during this read-only review.
