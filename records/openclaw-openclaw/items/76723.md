## Summary

Close as a support/configuration item: current main intentionally keeps group/channel final replies private by default, and Feishu group dispatch follows that shared visible-reply policy rather than failing in the Feishu transport.

## What This Changes

The issue asks for Feishu group messages that dispatch with `replies=0` to produce visible bot replies like direct messages.

## Best Possible Solution

Keep the shared group visible-reply policy and direct users who want legacy Feishu room auto-replies to set `messages.groupChat.visibleReplies: "automatic"`; a small Feishu docs cross-link would be a separate clarity improvement.

## Reproduction Assessment

Yes. Source inspection shows Feishu group messages reach the shared dispatch path with `ChatType: "group"`, where default group/channel visible replies are `message_tool_only`, so normal final replies can be suppressed and counted as zero without a transport failure.

## Solution Assessment

Yes. The best answer for this report is configuration guidance, not a Feishu transport patch: set `messages.groupChat.visibleReplies: "automatic"` when visible group auto-replies are desired.
