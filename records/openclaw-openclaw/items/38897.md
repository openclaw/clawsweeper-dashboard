## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard dashboard scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: The new callbacks close over `requestContext` before it is assigned. If cron or an agent event fires during the startup awaits, this passes `undefined` into `scheduleDashboardDelta`, whose `WeakMap.has(context)` throws and can crash the callback path. Move context construction before registering these callbacks or guard the closure until the context is available.
  - confidence: 0.94
- **[P2] Classify result failures from isError:** `src/gateway/tool-activity-registry.ts:204-205`
  - body: `handleToolExecutionEnd` emits result events with `data.isError`, but the registry ignores that flag and only inspects `data.result`. Failed tools whose result is a plain string/object will be marked `completed`, so Mission Control underreports recent failures and shows the wrong tool status.
  - confidence: 0.91
