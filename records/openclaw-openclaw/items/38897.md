## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Register dashboard methods in the descriptor registry:** `src/gateway/server-methods.ts:182-183`
  - body: On current main, every core gateway handler must have an entry in `CORE_GATEWAY_METHOD_SPECS`; otherwise `createCoreGatewayMethodDescriptors` throws while building the request registry. This PR adds `dashboard.summary` and `incident.*` handlers but only updates the old method list, so the gateway can fail before serving requests after rebase unless the new methods are added to `src/gateway/methods/core-descriptors.ts` with the right scopes.
  - confidence: 0.93
- **[P1] Guard dashboard scheduling before requestContext exists:** `src/gateway/server.impl.ts:422`
  - body: The new cron and agent-event callbacks are registered before `requestContext` is assigned. If cron startup or an agent event schedules a dashboard delta in that window, `scheduleDashboardDelta(undefined)` reaches the WeakMap lookup and throws, breaking the gateway callback; create the context before these registrations or guard the closures.
  - confidence: 0.91
- **[P2] Use isError when finalizing tool activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Tool result events carry failure status as `data.isError`, but the registry finalizes status by inspecting only `data.result`. A failed tool with a plain string/object result is shown as completed, so Mission Control underreports failed tool calls.
  - confidence: 0.9
