## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep hidden tool events out of the dashboard registry:** `src/gateway/server.impl.ts:516`
  - body: The global registry records every tool event without checking whether the run is Control-UI visible. Current event handling deliberately keeps hidden channel-routed assistant/tool traffic out of Control UI streams; this path would still store tool names, args, and results and expose them via dashboard.summary.
  - confidence: 0.88
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks can run before requestContext is assigned, because startup awaits discovery and delivery recovery after registering them. If a cron or agent event fires in that window, scheduleDashboardDelta(undefined) reaches WeakMap.has and throws, breaking the event path.
  - confidence: 0.9
- **[P2] Honor tool result error flags when finalizing activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as isError: true on a phase result event while the result is plain text or another object without ok: false or error. Finalizing from data.result alone marks those failed calls as completed and underreports failures.
  - confidence: 0.87
