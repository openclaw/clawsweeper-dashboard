Full review comments:

- **[P1] Register new dashboard methods in core descriptors:** `src/gateway/server-methods.ts:185-186`
  - body: Current main builds the gateway registry from CORE_GATEWAY_METHOD_SPECS and throws when a core handler lacks a descriptor. This PR adds dashboard.summary and incident.* handlers to coreGatewayHandlers without matching descriptor specs/scopes, so a rebased gateway can fail registry construction instead of serving the new methods.
  - confidence: 0.91
- **[P1] Guard dashboard scheduling until request context exists:** `src/gateway/server.impl.ts:421`
  - body: The new callbacks call scheduleDashboardDelta(requestContext) before requestContext is assigned later in startup. If cron or agent events fire during the intervening awaits, the WeakMap lookup receives undefined and can throw, so the callback should be moved after context creation or guarded.
  - confidence: 0.88
- **[P2] Classify tool failures from data.isError:** `src/gateway/tool-activity-registry.ts:205`
  - body: handleToolExecutionEnd emits tool result events with failure carried on data.isError; the result payload can be a plain string or object. Finalizing result events only from data.result marks those failed tools as completed and makes Mission Control underreport failures.
  - confidence: 0.9
- **[P1] Remove root prompt files from this feature PR:** `SOUL.md:19`
  - body: SOUL.md, TOOLS.md, and HEARTBEAT.md are recognized bootstrap/context files in OpenClaw. Adding external-authored root copies changes an agent prompt surface unrelated to Mission Control, so these files should be removed or handled through an explicit workspace/docs decision.
  - confidence: 0.86
