## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard dashboard scheduling until context exists:** `src/gateway/server.impl.ts:421-422`
  - body: The new cron and agent-event callbacks close over `requestContext` before it is assigned. Startup awaits occur before assignment, so a cron or agent event can call `scheduleDashboardDelta(undefined)` and throw from the WeakMap-backed scheduler; construct the context before registering these callbacks or guard the closures until it exists.
  - confidence: 0.94
- **[P2] Classify result events with isError:** `src/gateway/tool-activity-registry.ts:205`
  - body: `handleToolExecutionEnd` emits result failures with `data.isError`, but this registry only inspects `data.result`. Failed tools whose result is a plain string or object will be shown as completed, so Mission Control underreports recent failures.
  - confidence: 0.91
- **[P2] Remove root prompt artifacts:** `SOUL.md:1`
  - body: The branch adds root `SOUL.md`, `TOOLS.md`, and `HEARTBEAT.md` files that are not part of the dashboard implementation. These filenames are recognized workspace/bootstrap context in OpenClaw, so shipping local persona/checklist artifacts can change prompt context for source checkouts.
  - confidence: 0.82
