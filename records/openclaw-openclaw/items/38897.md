## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve hidden-run redaction before recording dashboard tools:** `src/gateway/server.impl.ts:516`
  - body: The registry stores every global tool event before applying a dashboard visibility/redaction boundary, then dashboard.summary returns args/output previews. Hidden channel-routed runs can leak tool names, args, and result previews; filter or redact before storing them.
  - confidence: 0.9
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks call scheduleDashboardDelta(requestContext) before requestContext is assigned later in startup. If cron or agent events fire during that window, the scheduler receives undefined and can throw on the WeakMap access.
  - confidence: 0.9
- **[P2] Use isError when finalizing tool activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as isError: true while result is plain text or another object without ok:false or error. Finalizing from data.result alone marks those failed calls completed and underreports failedRecent.
  - confidence: 0.88
- **[P2] Remove workspace bootstrap files from the product branch:** `SOUL.md:1`
  - body: SOUL.md, TOOLS.md, and HEARTBEAT.md are auto-injected workspace bootstrap files. Adding them at repo root changes local agent behavior and is unrelated to the Mission Control feature.
  - confidence: 0.82
