## Summary

Keep open: current main does not implement the requested runtime-aware Mission Control surface, but this PR is not mergeable as-is because source review found blocking privacy, startup, and status-classification bugs, and the external PR lacks real behavior proof.

## What This Changes

The PR adds dashboard runtime summaries, incident APIs, a tool activity registry, execution/autonomy Mission Control panels, overview polling, and several unrelated root/script/test changes.

## Best Possible Solution

Land a smaller rebased Mission Control runtime dashboard that preserves hidden-run redaction, initializes dashboard delta scheduling safely, classifies tool errors from the event contract, and includes real runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection: the PR's event listeners, WeakMap scheduling, and tool status logic expose concrete failure paths against current main contracts. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No. The product direction is useful, but this implementation bypasses an existing privacy guard, can schedule dashboard deltas before context exists, and misclassifies a known tool-error event shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep hidden tool events out of the dashboard registry:** `src/gateway/server.impl.ts:516`
  - body: The global registry records every tool event without checking whether the run is Control-UI visible. Current event handling deliberately keeps hidden channel-routed assistant/tool traffic out of Control UI streams; this path would still store tool names, args, and results and expose them via `dashboard.summary`.
  - confidence: 0.88
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks can run before `requestContext` is assigned, because startup awaits discovery and delivery recovery after registering them. If a cron or agent event fires in that window, `scheduleDashboardDelta(undefined)` reaches `WeakMap.has` and throws, breaking the event path.
  - confidence: 0.9
- **[P2] Honor tool result error flags when finalizing activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as `isError: true` on a `phase: "result"` event while the result is plain text or another object without `ok: false` or `error`. Finalizing from `data.result` alone marks those failed calls as completed and underreports failures.
  - confidence: 0.87
