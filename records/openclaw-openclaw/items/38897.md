## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Register the new methods in core descriptors:** `src/gateway/server-methods.ts:182-183`
  - body: Current main no longer uses the READ_METHODS/WRITE_METHODS sets shown in this branch. Core handlers are built from `CORE_GATEWAY_METHOD_SPECS`, and `createCoreGatewayMethodDescriptors` throws when a handler lacks a descriptor, so adding `dashboardHandlers`/`incidentsHandlers` without matching descriptor entries will break registry construction after rebase.
  - confidence: 0.94
- **[P1] Guard dashboard scheduling until the context exists:** `src/gateway/server.impl.ts:421`
  - body: These closures can run before `requestContext` is assigned, because startup yields through several awaits before the assignment. If a cron or agent event fires in that window, `scheduleDashboardDelta(requestContext)` receives `undefined` and the WeakMap access in the scheduler can throw during startup.
  - confidence: 0.92
- **[P1] Classify tool results with the event error flag:** `src/gateway/tool-activity-registry.ts:195`
  - body: Tool result events already carry failure as `data.isError`; some failed tool results are plain strings or objects without `ok:false` or `error`. Finalizing from `data.result` alone will mark those failed tools as completed, so Mission Control will underreport failures.
  - confidence: 0.93
- **[P1] Remove the root bootstrap prompt files:** `SOUL.md:1`
  - body: `SOUL.md`, `TOOLS.md`, and `HEARTBEAT.md` are recognized workspace bootstrap files and can be injected into agent context. Adding external-authored root persona/tool/heartbeat instructions in a Mission Control PR changes the prompt/security surface and should not ship with this feature.
  - confidence: 0.91
