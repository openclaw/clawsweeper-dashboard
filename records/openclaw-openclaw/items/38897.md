## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Register dashboard methods in the descriptor registry:** `src/gateway/server-methods.ts:185-186`
  - body: On current main, core gateway handlers must have CORE_GATEWAY_METHOD_SPECS entries or createCoreGatewayMethodDescriptors throws while building the registry. This PR adds dashboard.summary and incident.* handlers only to the old handler/list surfaces, so the gateway will fail after rebase unless these methods are added to src/gateway/methods/core-descriptors.ts with the right scopes.
  - confidence: 0.93
- **[P1] Guard dashboard scheduling before requestContext exists:** `src/gateway/server.impl.ts:421`
  - body: The cron and agent-event callbacks are registered before requestContext is assigned. If cron startup or an agent event schedules a dashboard delta during that window, scheduleDashboardDelta receives undefined and reaches the WeakMap lookup, throwing instead of delivering the event.
  - confidence: 0.91
- **[P2] Use isError when finalizing tool activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Tool result events can mark failure with data.isError while result is a plain string or object. Finalizing status from data.result alone shows those failed tool calls as completed, so Mission Control underreports failedRecent and recent tool statuses.
  - confidence: 0.9
