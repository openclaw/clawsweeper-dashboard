## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Guard dashboard scheduling before context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks can run before `requestContext` is assigned at line 575, but `scheduleDashboardDelta` immediately uses the value as a WeakMap key. If a cron or agent event fires during startup, `undefined` reaches `WeakMap.has` and the gateway callback can throw; initialize the context earlier or guard the callback until it exists.
  - confidence: 0.9
- **[P2] Use the tool event error flag when finalizing results:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool result events already carry `data.isError`, but this finalization only inspects `data.result`. Failed tools whose result is a plain string or an object without `ok: false`/`error` are recorded as `completed`, so Mission Control underreports tool failures and shows the wrong recent status.
  - confidence: 0.88
- **[P2] Move new overview copy into the i18n catalog:** `ui/src/ui/views/overview.ts:1942-1943`
  - body: The Control UI keeps user-visible copy in the English locale catalog and generated locale outputs, but the new Mission Control panels add large amounts of hardcoded English. This regresses localized UI behavior and should be moved through the existing i18n source and sync flow.
  - confidence: 0.82
- **[P2] Remove root bootstrap prompt files:** `SOUL.md:1`
  - body: OpenClaw injects root workspace bootstrap files such as `SOUL.md`, `TOOLS.md`, and `HEARTBEAT.md` into agent context. Adding repo-root persona and local-notes files in this dashboard PR changes agent prompt behavior for this checkout and should be removed or handled through the normal workspace/template surfaces.
  - confidence: 0.86
