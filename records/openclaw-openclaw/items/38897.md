## Summary

Keep open: current main still lacks the requested runtime dashboard surface, but the PR is not mergeable as-is because it has concrete privacy, startup-race, tool-status, and proof blockers.

## What This Changes

The branch adds gateway dashboard and incident APIs, a tool activity registry, process/autonomy summaries, overview polling, and Mission Control panels for tool runtime, execution queue, and autonomy posture.

## Best Possible Solution

Land a smaller rebased Mission Control runtime dashboard that preserves hidden-run redaction, initializes dashboard delta scheduling safely, honors tool error flags, removes unrelated workspace churn, and includes real runtime proof.

## Reproduction Assessment

Yes for the review findings via source inspection: the PR registers dashboard callbacks before requestContext assignment, records all global agent events without the current visibility guard, and classifies tool result events without data.isError. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best path until it preserves privacy boundaries, avoids the startup race, classifies failures from the event contract, and provides real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve hidden-run redaction before recording dashboard tools:** `src/gateway/server.impl.ts:516`
  - body: The registry records every global agent event before applying the existing hidden-run visibility boundary. Hidden channel-routed runs can therefore have tool names, args, and result previews retained and exposed through dashboard.summary, so filter or redact before storing them.
  - confidence: 0.88
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks call scheduleDashboardDelta(requestContext) before requestContext is assigned later in startup. If cron or agent events fire during the intervening awaits, scheduleDashboardDelta receives undefined and the WeakMap access can throw.
  - confidence: 0.9
- **[P2] Use isError when finalizing tool activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as isError: true while result is plain text or another object without ok:false or error. Finalizing from data.result alone marks those failed calls completed and underreports failedRecent.
  - confidence: 0.87

## Security Review
