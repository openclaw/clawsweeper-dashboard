Keep this PR open: current main still does not implement the runtime-aware Mission Control surface, but the branch has blocking correctness and privacy issues plus missing real behavior proof.

## What This Changes

The PR adds gateway dashboard and incident APIs, a tool activity registry, process/autonomy summaries, Mission Control UI panels, overview polling, and unrelated root/script/test changes.

## Best Possible Solution

Land a smaller rebased Mission Control runtime dashboard that preserves hidden-run redaction, initializes dashboard delta scheduling safely, honors tool error flags, removes unrelated workspace churn, and includes real runtime proof.

Yes for the review findings via source inspection: the PR registers dashboard callbacks before requestContext assignment, records all global agent events without the current visibility guard, and classifies tool result events without data.isError. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No. The product direction is useful, but this implementation is not the best fix path until it preserves existing privacy boundaries, avoids the startup race, classifies failures from the event contract, and provides real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep hidden tool events out of the dashboard registry:** `src/gateway/server.impl.ts:516`
  - body: The global registry records every agent event before applying the current isControlUiVisible guard used for Control UI tool streams. Hidden channel-routed runs can still have tool names, args, and results retained and exposed through dashboard.summary, so this needs the same visibility/redaction boundary before recording.
  - confidence: 0.88
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks call scheduleDashboardDelta(requestContext) before requestContext is assigned later in startup. If cron or agent events fire during the intervening awaits, scheduleDashboardDelta receives undefined and the WeakMap access can throw, breaking the event path.
  - confidence: 0.9
- **[P2] Honor tool result error flags when finalizing activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as isError: true on a phase result event while result is plain text or another object without ok:false or error. Finalizing from data.result alone marks those failed calls completed and underreports failedRecent.
  - confidence: 0.87
