## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard dashboard scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: The cron and agent-event callbacks close over requestContext before it is assigned later in startup. Because startup awaits and background callbacks can run before line 575, this can call scheduleDashboardDelta(undefined) and throw from the WeakMap-backed scheduler; build the context before registering callbacks or guard them until it exists.
  - confidence: 0.94
- **[P2] Classify result events with isError:** `src/gateway/tool-activity-registry.ts:205`
  - body: handleToolExecutionEnd emits failed tool results with data.isError, but the registry finalizes result events only from data.result. Failed tools with plain output or details.status="error" will be shown as completed, so Mission Control underreports recent failures.
  - confidence: 0.91
- **[P2] Remove root prompt artifacts:** `SOUL.md:1`
  - body: The branch adds SOUL.md, TOOLS.md, and HEARTBEAT.md at the repository root even though those filenames are agent workspace/bootstrap and heartbeat context. Shipping local persona/checklist files with the source checkout can change prompt context and is unrelated to the Mission Control feature.
  - confidence: 0.82
