## What This Changes

The branch adds Mission Control runtime dashboard APIs and panels for tool activity, background processes, incidents, and autonomy posture, plus gateway delta scheduling and overview polling.

## Best Possible Solution

Land a narrower rebased Mission Control runtime dashboard that preserves hidden-run redaction, guards startup scheduler context, honors isError, drops workspace bootstrap files, and includes redacted runtime proof.

## Reproduction Assessment

Yes for the PR review blockers by source inspection: the branch records global tool events before a dashboard visibility filter, registers dashboard callbacks before requestContext assignment, and ignores data.isError when finalizing result events. I did not run a live gateway or browser setup in this read-only review.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best path until it fixes the privacy boundary, startup race, failure classification, unrelated workspace-file additions, conflicts, and proof gap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve hidden-run redaction before recording dashboard tools:** `src/gateway/server.impl.ts:516`
  - body: The registry stores every global tool event before applying a dashboard visibility or redaction boundary, then dashboard.summary exposes args/output previews. Hidden channel-routed runs can leak tool names, arguments, and result previews; filter or redact before storing them.
  - confidence: 0.9
- **[P1] Guard dashboard delta scheduling until context exists:** `src/gateway/server.impl.ts:421`
  - body: These callbacks call scheduleDashboardDelta(requestContext) before requestContext is assigned later in startup. If cron or agent events fire during that window, the WeakMap-backed scheduler receives undefined and can throw.
  - confidence: 0.9
- **[P2] Use isError when finalizing tool activity:** `src/gateway/tool-activity-registry.ts:205`
  - body: Embedded tool completions can report failure as isError: true while result is plain text or another object without ok:false or error. Finalizing from data.result alone marks those failed calls completed and underreports failedRecent.
  - confidence: 0.88
- **[P2] Remove workspace bootstrap files from the product branch:** `SOUL.md:1`
  - body: SOUL.md, TOOLS.md, and HEARTBEAT.md are local workspace bootstrap notes, not Mission Control product files. Adding them at repo root changes local agent behavior and adds unrelated setup content to this feature branch.
  - confidence: 0.82
