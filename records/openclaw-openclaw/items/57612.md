## Summary

Keep open: this PR adds a useful missing DOCX path, but current head still has a security/correctness blocker because DOCX XML is inflated before the byte cap is enforced, public docs need the new contract, and contributor real behavior proof is still absent.

## What This Changes

Adds a JSZip-based DOCX text extractor, wires it into OpenResponses `input_file` and the `read` tool, expands the default MIME allowlist, and adds focused regression tests.

## Best Possible Solution

Land a bounded, maintainer-approved DOCX extraction path with pre-materialization limits, docs, regression coverage, and real behavior proof; route richer document parsing through document-extract plugin contracts if core is not the right owner.

## Reproduction Assessment

Yes for the review finding: PR-head source plus JSZip v3.10.1 source show `word/document.xml` is accumulated before `maxXmlBytes` is checked. Current-main absence of DOCX `input_file` and `read` extraction is also source-checkable.

## Solution Assessment

No, not as-is. The feature direction is useful, but the implementation should enforce limits before XML materialization and needs docs plus a maintainer decision on core versus document-extract plugin ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce DOCX XML limits before inflation:** `src/media/docx-extract.ts:71-76`
  - body: `documentXmlFile.async("uint8array")` inflates and accumulates the whole `word/document.xml` entry before `maxXmlBytes` is checked. A compressed DOCX that passes the outer file-size cap can still expand into a large XML payload in `input_file` or `read`, so enforce an uncompressed entry or streaming byte limit before materializing the XML.
  - confidence: 0.9
- **[P3] Document the DOCX input_file contract:** `src/media/input-files.ts:106`
  - body: Adding DOCX to the default `input_file` allowlist changes the public OpenResponses contract, but the docs still list only text/markdown/html/csv/json/pdf and the config sample omits DOCX. Please document the new MIME type, extraction scope, and limits.
  - confidence: 0.84
