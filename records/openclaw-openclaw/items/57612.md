## Summary

Keep open. This is a useful implementation candidate, but it is not merge-ready: the PR still enforces the DOCX XML byte cap only after JSZip materializes `word/document.xml`, the new public `input_file` MIME contract is undocumented, and contributor proof is test/CI-only while current main still lacks DOCX extraction.

## What This Changes

The PR adds JSZip-based DOCX text extraction, routes DOCX through `input_file` and `read`, expands the default MIME allowlist, and adds focused regression tests.

## Best Possible Solution

Land a bounded, documented DOCX extraction path through the maintainer-approved document-extraction seam, with pre-materialization limits, public docs, and real behavior proof for both `read` and `input_file`.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source plus JSZip 3.10.1 source show `word/document.xml` is accumulated before `maxXmlBytes` is enforced. I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The feature direction is useful, but the implementation should enforce limits before XML materialization, document the new `input_file` contract, and get maintainer agreement on core versus document-extract ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce DOCX XML limits before inflation:** `src/media/docx-extract.ts:71`
  - body: `documentXmlFile.async("uint8array")` inflates and accumulates the whole `word/document.xml` entry before `maxXmlBytes` is checked. A compressed DOCX that passes the outer file-size cap can still expand into a large XML payload in `input_file` or `read`, so enforce an uncompressed entry or streaming byte limit before materializing the XML.
  - confidence: 0.9
- **[P3] Document the DOCX input_file contract:** `src/media/input-files.ts:106`
  - body: Adding the DOCX MIME type to `DEFAULT_INPUT_FILE_MIMES` changes the public OpenResponses `input_file` allowlist, but the docs still list only text/CSV/JSON/PDF and describe only PDF parsing. Please document the DOCX MIME type, extraction scope, and limits.
  - confidence: 0.84
