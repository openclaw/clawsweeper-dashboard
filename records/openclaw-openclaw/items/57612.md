- **[P1] Bound DOCX inflation before allocating XML:** `src/media/docx-extract.ts:71-76`
  - body: `documentXmlFile.async("uint8array")` fully inflates `word/document.xml` before the byte-budget check runs, so a small compressed DOCX can allocate a large XML buffer during `read` or `input_file` handling. Enforce the uncompressed limit before or during inflation, or use a bounded extractor that can reject the entry before accumulation.
  - confidence: 0.91
- **[P1] Route DOCX parsing through the extractor seam:** `src/media/input-files.ts:400-405`
  - body: Current main already has a plugin-backed document extraction contract, but this PR adds a new core JSZip parser and calls it directly from `input_file`. That splits document handling across two mechanisms and bypasses the boundary that now owns parser dependencies and resource policy; implement DOCX as a document-extract plugin entry and call the shared runtime instead.
  - confidence: 0.86

  - body: The PR head is behind current main's input-file byte-sniffing hardening; with DOCX added to the default allowlist, this path can rely on declared `mediaType` before extraction.
