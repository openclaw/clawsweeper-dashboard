## Summary

Keep open. Current main still lacks DOCX extraction, and this PR remains useful, but it needs a ZIP/XML resource-limit fix, public docs, real behavior proof, and maintainer judgment on the core-vs-plugin boundary before merge.

## What This Changes

The PR adds a JSZip-based DOCX text extractor, wires it into OpenResponses `input_file` and the `read` tool, adds DOCX to the default file MIME allowlist, and adds focused tests.

## Best Possible Solution

Add bounded DOCX extraction through the existing document extraction path if maintainers agree, with pre-materialization limits, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the review finding: source and JSZip v3.10.1 code show the PR accumulates `word/document.xml` before checking `maxXmlBytes`. Current-main absence of DOCX extraction is also source-checkable in `input_file` and `read`.

## Solution Assessment

No, not as-is. The feature direction is useful, but the implementation should enforce XML limits before materialization and needs a maintainer decision on core versus document-extract plugin ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce DOCX XML limits before inflating:** `src/media/docx-extract.ts:71-76`
  - body: `documentXmlFile.async("uint8array")` inflates and accumulates the whole `word/document.xml` entry before `maxXmlBytes` is checked. A compressed DOCX that passes the outer file-size cap can still expand into a large XML payload in `input_file` or `read`, so enforce an uncompressed entry or streaming byte limit before materializing the XML.
  - confidence: 0.9
- **[P3] Document the DOCX input_file contract:** `src/media/input-files.ts:106`
  - body: Adding DOCX to the default `input_file` allowlist changes the public OpenResponses contract, but the docs still list only text/markdown/html/csv/json/pdf and describe PDF-only document parsing. Please update the docs with the new MIME, extraction scope, and limits.
  - confidence: 0.84
