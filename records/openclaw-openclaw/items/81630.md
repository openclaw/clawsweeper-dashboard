## Summary

Keep open: current main still has separate live tool-event rendering and final `chat.history` projection paths, so the reported WebChat reflow remains plausible from source and does not meet a close reason.

## What This Changes

The issue asks WebChat to avoid large live tool-call bubbles that visibly shrink when final history projection replaces the streaming view.

## Best Possible Solution

Set a maintainer-approved Control UI contract for live tool progress, then make live rendering and final history projection preserve a stable visual footprint through completion.

## Reproduction Assessment

No high-confidence browser reproduction was run. Source inspection shows the relevant path: live tool messages can include result output, final tool completion reloads `chat.history`, and the final projection uses a different normalized transcript shape.

## Solution Assessment

Unclear until maintainers choose the desired UX. Reusing the existing tool-call visibility/collapse behavior during streaming is likely narrower than adding another setting, but the default visible behavior is a product decision.
