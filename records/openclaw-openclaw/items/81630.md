## Summary

Keep open: current main has mitigations for compact/collapsed tool rows, but the live tool-event path is still distinct from the final `chat.history` reload/reset path, so the reported completion-time structure change remains source-plausible and needs a maintainer UX decision.

## What This Changes

The issue asks WebChat to prevent live tool-call output from inflating a streaming assistant bubble and then shrinking after the final `chat.history` refresh.

## Best Possible Solution

Unify live tool-event display and final history projection semantics so tool-heavy turns keep a compact, stable visible shape through completion without hiding useful progress unexpectedly.

## Reproduction Assessment

No, there is not a high-confidence browser reproduction from this review. Source inspection shows a plausible path: transient live tool messages are rendered during the run, then final `chat.history` reload replaces them and clears live tool state.

## Solution Assessment

Unclear until maintainers choose the desired UX. Applying the same collapse/projection rules during streaming is likely the narrowest direction, while adding or expanding a setting would be broader product surface.
