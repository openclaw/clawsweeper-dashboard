## Summary

Keep open: current main still has separate live tool-message rendering and final `chat.history` projection paths, so the reported WebChat shrink remains source-plausible. The older canonical layout issue at https://github.com/openclaw/openclaw/issues/63956 and active duplicate-text PR at https://github.com/openclaw/openclaw/pull/74498 are adjacent, but they do not fully settle the live tool-output-to-final-history shrink contract for this issue.

## What This Changes

The issue asks WebChat to keep the assistant streaming bubble visually stable when a tool-heavy turn transitions from live tool output to final history rendering.

## Best Possible Solution

Define one Control UI contract for live tool progress, then align live rendering and final history projection so a tool-heavy assistant turn keeps a stable visual footprint through completion.

## Reproduction Assessment

No high-confidence browser reproduction was run. Source inspection shows the path clearly enough to keep this open: live tool messages can include output, final tool runs reload normalized `chat.history`, and then live tool state is reset.

## Solution Assessment

Unclear until maintainers choose the desired UX. A Control UI-only alignment of live tool rendering with final history projection is safer than changing gateway chat protocol or adding a new setting by default.
