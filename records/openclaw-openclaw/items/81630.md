## Summary

Keep open: current main still has distinct live tool-event rendering and final `chat.history` projection/reset paths, and the remaining question is a maintainer UX decision about stable WebChat tool-progress rendering rather than a cleanup-close candidate.

## What This Changes

The issue requests a stable WebChat streaming display so live tool-call output does not inflate an assistant bubble and then shrink after final `chat.history` projection.

## Best Possible Solution

Define the Control UI contract for live tool progress, then make streaming render and final `chat.history` projection preserve a stable visual footprint through completion.

## Reproduction Assessment

No high-confidence browser reproduction was run. Source inspection shows the plausible path: live tool-result messages can include output, final tool completion reloads `chat.history`, and final projection uses a different normalized transcript shape.

## Solution Assessment

Unclear until maintainers choose the desired UX. Reusing the existing collapsed tool-call presentation during streaming is likely narrower than adding another setting, but the default live visibility policy is a product decision.
