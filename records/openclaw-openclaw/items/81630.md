## Summary

Current main and the latest shipped release already address the reported WebChat grow-then-shrink path: live tool output is modeled as collapsible tool-card messages, gated by `chatShowToolCalls`, and gateway ordering flushes assistant text before tool-start cards. The remaining question of whether normal WebChat should hide tool-output rows by default is tracked separately at https://github.com/openclaw/openclaw/issues/89781.

## What This Changes

WebChat should keep assistant streaming content visually stable across multi-tool turns instead of appending raw tool-call/output text and then shrinking after final history projection.

## Best Possible Solution

Keep the shipped separated tool-card/stream-ordering implementation, and use https://github.com/openclaw/openclaw/issues/89781 for any remaining default tool-output visibility decision.

## Reproduction Assessment

No high-confidence current-main repro remains from source inspection. Current WebChat separates live tool output into collapsible/gated tool-card messages and flushes pre-tool text before tool-start events.

## Solution Assessment

Yes. The shipped fix is the maintainable path for this issue: keep assistant text and tool activity on separate render paths, gate tool visibility with `chatShowToolCalls`, and preserve ordering at the gateway.
