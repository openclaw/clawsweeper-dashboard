## Summary

Keep open: current main still has distinct live tool-event rendering and final chat.history projection paths, so the reported WebChat reflow is plausible from source, but the right UX needs maintainer direction before a narrow fix.

## What This Changes

Request a steadier WebChat streaming render so intermediate tool-call output does not visibly expand the assistant bubble and then collapse after the final chat.history refresh.

## Best Possible Solution

Align WebChat live stream rendering with the final history projection, or keep an equivalent collapsed placeholder through final refresh, so tool-heavy turns do not remove large visible content at completion.

## Reproduction Assessment

No high-confidence browser reproduction was run. Source inspection shows the central reflow path is source-reproducible: live tool-event items are rendered during the run, then final chat.history reload replaces them with the normalized transcript view.

## Solution Assessment

Unclear until maintainers choose the desired UX. Applying the same collapse/projection rules to live tool output is likely the narrowest fix, while adding a new setting would be broader product surface.
