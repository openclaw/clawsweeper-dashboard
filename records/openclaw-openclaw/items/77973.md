## Summary

Keep this PR open for maintainer review: the patch is focused and source-backed, but the fixed FIFO cap intentionally changes how many terminal run snapshots remain recoverable for `agent.wait` under high fan-out.

## What This Changes

Adds a 5,000-entry cap to the gateway `agentRunCache`, exposes test-only cache helpers, and adds regression coverage for overflowed lifecycle terminal events.

## Best Possible Solution

Land a bounded `agentRunCache` with the regression test once gateway owners accept the terminal-snapshot retention budget, or adjust the cap or eviction policy before merge.

## Reproduction Assessment

Yes. Source inspection shows unique lifecycle terminal events with fresh run IDs flow into `recordAgentRunSnapshot`, which currently stores snapshots with only TTL pruning; the PR body also supplies live Node/tsx output that drives the real event path past the cap.

## Solution Assessment

Yes for the fix shape. A local hard cap on the existing in-process terminal-snapshot cache is the narrowest maintainable memory bound; only the exact retention budget needs maintainer acceptance.
