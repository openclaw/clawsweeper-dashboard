## Summary

Keep this PR open: it fixes the current-prompt part of the fresh-token undercount, but the new fresh projection can still omit the previous output tokens whenever the prompt-only base is more than the fixed transcript-read buffer below the threshold.

## What This Changes

The PR changes `runPreflightCompactionIfNeeded` to project fresh persisted tokens through the next-input token estimator and updates one adjacent regression expectation.

## Best Possible Solution

Before merge, make fresh preflight projection include a reliable previous-output source, preferably stored `SessionEntry.outputTokens` when transcript output was not read, and add a regression for an output-only threshold crossing.

## Reproduction Assessment

Yes. Source inspection shows current main skips transcript usage reads in the fresh-token path, and the PR proof supplies before/after terminal output for the production preflight decision path.

## Solution Assessment

No. The PR is a good start, but the output-token part is still conditional on an 8192-token read buffer even though stored `outputTokens` are available on normal completed runs.
