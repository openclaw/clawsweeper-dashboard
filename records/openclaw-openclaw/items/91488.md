## Summary

Keep open: the fresh-token preflight projection is needed, but the branch still has a blocker because it treats `SessionEntry.outputTokens` as the previous-output size even though current persistence stores accumulated run output there.

## What This Changes

The PR changes fresh-token preflight compaction to project the next request from persisted prompt tokens plus output and current prompt estimates, with adjacent regression tests.

## Best Possible Solution

Keep the fresh preflight projection, but count previous output only from the latest transcript or another last-call output source and add a regression for accumulated-output sessions.

## Reproduction Assessment

Yes from source: current main uses a prompt-only fresh snapshot for preflight compaction, and `deriveSessionTotalTokens` confirms that snapshot excludes output tokens. I did not run the proof script in this read-only review, but the PR body supplies before/after terminal output for the same decision path.

## Solution Assessment

No: projecting fresh snapshots is the right layer, but using persisted `outputTokens` as the counted previous output is not the best fix because that field stores accumulated run output. The safer repair is to use transcript-derived latest output or a true last-call output source.
