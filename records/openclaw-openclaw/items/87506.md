## Summary

Keep open: current main and the latest release still exclude pairing-store approvals from `dmPolicy="allowlist"`, while public docs still say allowlist can use the paired allow store. That leaves a real Feishu authorization-contract mismatch rather than an implemented cleanup candidate.

## What This Changes

The issue reports that Feishu non-default-account DM approvals stored in `credentials/feishu-lux-allowFrom.json` stop authorizing messages after an npm update.

## Best Possible Solution

Maintainers should decide the intended contract, then align runtime, docs, doctor guidance, and regression tests: either strict allowlist requires config `allowFrom`/doctor migration, or paired-store approvals should authorize allowlist mode.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does reproduce the core store-only allowlist failure on current main, but not the full Windows npm-update sequence or the exact pairing-code response.

## Solution Assessment

Unclear until maintainers choose the contract. A narrow runtime change could honor the docs, but the current strict allowlist behavior was intentionally merged and doctor already has a migration-style repair path.
