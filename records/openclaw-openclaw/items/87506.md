## Summary

## What This Changes

The issue asks OpenClaw to keep honoring Feishu non-default-account DM pairing approvals stored in `credentials/<channel>-<accountId>-allowFrom.json` after npm upgrades and Gateway restarts.

## Best Possible Solution

Maintainers should make one explicit contract and align runtime, doctor/update, and docs: either allowlist mode trusts account-scoped pairing-store approvals, or upgrade/doctor reliably migrates them into canonical account `allowFrom` before Gateway runtime starts.

## Reproduction Assessment
