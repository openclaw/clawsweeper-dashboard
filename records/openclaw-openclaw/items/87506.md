## Summary

Keep open: current main and v2026.6.8 still do not automatically honor account-scoped Feishu pairing-store approvals in allowlist-mode runtime. Doctor can migrate store entries into config, but the reported npm-update plus restart path remains a maintainer contract/security decision.

## What This Changes

The issue asks OpenClaw to preserve Feishu non-default account DM approvals after npm updates when the account-scoped allowFrom credentials file still exists.

## Best Possible Solution

Maintainers should choose one explicit contract: either allowlist mode deliberately trusts approved pairing-store DM entries at runtime, or approvals remain pairing-only and the update/doctor/docs path must reliably migrate store-only senders into config `allowFrom`.
