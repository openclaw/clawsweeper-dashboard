## Summary

Keep open. Current main still gates Anthropic adaptive-thinking payloads on Claude model-id substrings, and the public model compat config/schema still cannot opt an opaque Anthropic-compatible proxy model into adaptive thinking.

## What This Changes

Request support for a config-declared adaptive-thinking capability for custom Anthropic-compatible provider models with opaque IDs.

## Best Possible Solution

Add a typed Anthropic adaptive-thinking compat signal and wire it through config schema, catalog normalization, thinking/session validation, Anthropic payload construction, docs, and focused regression tests while preserving the existing Claude-id heuristic fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a custom anthropic-messages provider with reasoning enabled and an opaque Claude-backed id such as zyb-high; current main cannot expose or emit adaptive thinking for that id.

## Solution Assessment

Yes in direction, but not yet settled as an implementation. An explicit typed opt-in is more maintainable than hardcoding private proxy prefixes or enabling adaptive thinking for every opaque Anthropic-compatible model.
