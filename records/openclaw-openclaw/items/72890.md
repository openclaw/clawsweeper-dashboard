## Summary

Keep open. Current main still has the reported gap: custom `anthropic-messages` model metadata can carry compat data, but adaptive-thinking support is still decided by Claude model-id substrings and cannot be enabled for an opaque proxy id.

## What This Changes

Allow custom Anthropic-compatible provider models to declare adaptive-thinking support so opaque Claude-backed proxy IDs can use adaptive payloads.

## Best Possible Solution

Add a typed model-compat opt-in for Anthropic adaptive thinking and make both thinking-profile validation and Anthropic transport payload resolution honor it before falling back to model-id heuristics.

## Reproduction Assessment

Yes. Configure `models.providers.my-proxy` with `api: "anthropic-messages"`, `reasoning: true`, and an opaque Claude-backed id such as `zyb-high`; static inspection shows `/think adaptive` is not exposed and the transport takes the non-adaptive budget path.

## Solution Assessment

Yes. A config-backed compat opt-in is the narrow maintainable solution because it supports corporate/proxy model IDs without hardcoding private prefixes or requiring a full provider plugin for a simple proxy.
