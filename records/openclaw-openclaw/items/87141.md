Full review comments:

- **[P1] Reject non-plain values in the JSON compatibility guard:** `src/shared/json-compatible.ts:64-66`
  - body: `describeNonJsonCompatibleValue` now accepts any object that can enumerate keys. That means `new Date()`, `new Map()`, `new URL()`, and class instances with no enumerable fields return `undefined` and can still pass the new plugin schema/config/tool metadata gates as JSON-compatible. The existing plugin JSON contract treats Date and Map as invalid, so add a plain-record/prototype check here and cover those object types.
  - confidence: 0.88
- **[P2] Type-tag descriptor cache primitives:** `src/plugins/tool-descriptor-cache.ts:96-97`
  - body: The descriptor context cache still serializes `1n` as `"1"`, which is identical to a string value of `"1"`. If plugin descriptor output depends on that context metadata, the cache can reuse descriptors across distinct contexts and expose stale tool schemas. Prefix primitive kinds here like `createPluginCacheKey` does, and add string-vs-bigint coverage in `tool-descriptor-cache.test.ts`.
  - confidence: 0.86
