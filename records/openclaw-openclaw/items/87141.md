- **[P2] Reject non-plain schema objects in the compatibility walker:** `src/shared/json-compatible.ts:64-83`
  - body: `describeNonJsonCompatibleValue` now treats any readable non-array object as JSON-compatible. Please reject non-plain object prototypes here, or explicitly normalize allowed object types before validation.
  - confidence: 0.86

  - body: The descriptor context key still encodes `1n` as `"1"`, the same fragment used for the string `"1"`. Because this key gates cached plugin tool descriptors, different ambient tool contexts can reuse the wrong descriptor set instead of being recomputed. Please use the type-tagged encoder shape added in `plugin-cache-primitives.ts` or otherwise include primitive type tags here.
  - confidence: 0.89
