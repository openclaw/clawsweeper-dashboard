- **[P2] Reject non-plain JSON values in the compatibility walker:** `src/shared/json-compatible.ts:64-66`
  - body: This walker currently treats any enumerable object as JSON-compatible, so Date, Map, URL, or class instances with no enumerable fields pass. Because schema validation, plugin config metadata, and SDK tool-schema checks now depend on this guard, non-JSON schema/default values can still reach metadata or TypeBox instead of producing deterministic validation errors. Add a plain-record/array check or equivalent and cover those object types.
  - confidence: 0.86
- **[P2] Type-tag descriptor context cache values:** `src/plugins/tool-descriptor-cache.ts:96-97`
  - body: The new descriptor context encoder serializes bigint 1n as the same text as string "1". Plugin tool descriptors can depend on ambient context, so changing a context field from string to bigint can incorrectly reuse the previous descriptor cache entry; provider/loader cache keys use type prefixes, but this path still does not. Reuse createPluginCacheKey or prefix scalar types here and add the missing string-vs-bigint descriptor test.
  - confidence: 0.82
