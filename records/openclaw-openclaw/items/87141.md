Full review comments:

- **[P1] Skip unreadable provider tool array slots per index:** `src/plugin-sdk/provider-tools.ts:650-654`
  - body: This helper wraps the whole array copy in one `try`, so a getter failure for any single tool index returns `undefined`. The provider compatibility normalizers then fall back to the original tool list or lose diagnostics, which means healthy sibling tools can remain unnormalized for Gemini/OpenAI/DeepSeek. Copy each index independently and continue past the bad slot, with a regression where index 0 throws and index 1 still survives.
  - confidence: 0.87
- **[P1] Preserve healthy provider auth rows after one bad slot:** `src/plugin-sdk/provider-entry.ts:127-130`
  - body: `copyArrayEntries` returns an empty list when reading any auth, extraAuth, or env var array slot throws. Since single-provider registration uses this helper before row-level validation, one bad slot can drop all healthy sibling auth methods and env vars. Keep the per-index read best-effort so the PR's sibling-isolation contract holds.
  - confidence: 0.86
- **[P2] Keep catalog model rows after one bad model slot:** `src/plugins/provider-catalog-unified-text.ts:33-35`
  - body: Unified text catalog projection uses the same all-or-nothing array copy for provider models. If one model index getter throws, `copyProviderModels` sees an empty list and erases healthy sibling model rows for that provider, which contradicts the PR's availability-oriented catalog hardening. Continue past the unreadable index instead of returning an empty array.
