Full review comments:

- **[P2] Reject non-plain schema objects in the compatibility walker:** `src/shared/json-compatible.ts:64-83`
  - body: `describeNonJsonCompatibleValue` now treats any readable non-array object as JSON-compatible. That lets `Date`, `Map`, `URL`, class instances, and other non-plain objects with no enumerable schema fields pass the new SDK/schema-validator boundary even though this PR is meant to reject non-JSON-compatible plugin schemas before metadata/runtime registration. Please reject non-plain object prototypes here, or explicitly normalize the allowed object types before validation.
  - confidence: 0.86
- **[P2] Type-tag descriptor context cache values:** `src/plugins/tool-descriptor-cache.ts:96-103`
  - body: The descriptor context key still encodes `1n` as `"1"`, the same fragment used for the string `"1"`. Because this key gates cached plugin tool descriptors, different ambient tool contexts can reuse the wrong descriptor set instead of being recomputed. Please use the type-tagged encoder shape added in `plugin-cache-primitives.ts` or otherwise include primitive type tags here.
  - confidence: 0.89
