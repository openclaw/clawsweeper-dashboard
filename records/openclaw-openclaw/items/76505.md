## Summary

Keep open: current main still has a credible source-level Control UI bug. The standalone PWA path applies safe-area handling at the body level, but the chat shell/content remain height-constrained with `100dvh` and `overflow: hidden`, while the current composer toolbar has no bottom safe-area compensation.

## What This Changes

The issue reports that the Control UI chat composer toolbar is clipped behind the iOS home indicator when opened as a standalone Home Screen PWA.

## Best Possible Solution

Add a narrow Control UI layout fix so the active chat composer toolbar remains above the standalone iOS bottom safe area without changing desktop/Safari behavior or adding configuration.

## Reproduction Assessment

Yes, at source level: the issue steps match the current standalone PWA configuration, and the CSS path shows the toolbar can sit inside a `100dvh`/hidden chat shell without composer-level bottom safe-area spacing. I did not establish a live iPhone failing run, so this is not marked as fully reproduced.

## Solution Assessment

Yes: the narrow maintainable fix is in Control UI layout CSS around the active `.agent-chat__input`/chat shell safe-area behavior. A broader product setting or channel/plugin change is not needed.
