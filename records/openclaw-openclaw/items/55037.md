  - body: `resolveBindingWorkspace` calls `listRouteBindings(cfg)`, but this file does not import `listRouteBindings`, so the PR will not typecheck. Either import the helper or, preferably, avoid this extra scan and pass the resolved route workspace through the existing route path.
  - confidence: 0.98
- **[P1] Do not read a session key from reply options:** `src/auto-reply/reply/get-reply.ts:191`
  - body: `GetReplyOptions` does not define `sessionKey`, and current callers carry the route session on the message context, so `opts.sessionKey` is both a type error and not a reliable data path for this feature. Add a typed route/workspace carrier or use the existing resolved context path instead.
  - confidence: 0.97
- **[P1] Use the matched route instead of suffix-matching peers:** `src/auto-reply/reply/get-reply.ts:63-65`
  - body: The helper selects the first same-agent binding whose peer id suffix matches the session key, ignoring channel, account, guild/team, wildcard, and account/channel binding scopes. In multi-account or non-peer routes this can silently choose the wrong workspace or no workspace even though `resolveAgentRoute` already selected the exact binding.
  - confidence: 0.93
