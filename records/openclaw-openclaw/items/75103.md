## Summary

Keep open: current main still lacks Plugin SDK lock-owner start-time disambiguation, and the PR direction matches the shared PID start-time helper; the remaining narrow gap is the required changelog entry plus green exact-head validation.

## What This Changes

The PR adds optional Linux process start-time metadata to Plugin SDK file-lock payloads, treats live PID/start-time mismatches as stale, and adds unit coverage for writer persistence, mismatch recovery, legacy fallback, and spy cleanup.

## Best Possible Solution

Add the required changelog entry for the Plugin SDK lock recovery, keep the implementation narrow, then land after maintainer review and green exact-head validation.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to precreate a fresh lock file with the current PID and a mismatched Linux starttime; current main treats the live PID as non-stale until the age fallback, while the PR reclaims it.

## Solution Assessment

Yes for the code direction. Reusing the existing raw /proc start-time helper is the narrowest maintainable fix, preserves legacy and non-Linux fallback behavior, and avoids changing public SDK signatures.
