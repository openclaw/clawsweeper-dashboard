- **[P2] Report WebSocket readiness from SDK callbacks:** `extensions/feishu/src/monitor.transport.ts:192-200`
  - body: `WSClient.start()` stores the dispatcher and calls `reConnect(true)` without awaiting the first handshake; the package exposes `onReady` and `onReconnected` for that state. Setting `connected: true` immediately after `start()` can mark Feishu healthy before the socket is ready, so move the connected/lastConnectedAt patch into those callbacks.
  - confidence: 0.92

This branch's WebSocket path is based on older code without that callback/cycle-end path, so resolving conflicts by taking it would drop recovery.
  - confidence: 0.88
