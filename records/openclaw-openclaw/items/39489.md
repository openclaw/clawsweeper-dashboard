- **[P2] Report WebSocket connected from readiness callbacks:** `extensions/feishu/src/monitor.transport.ts:192-200`
  - body: `WSClient.start()` stores the dispatcher and kicks off `reConnect(true)` without awaiting the first handshake; the SDK exposes `onReady` and `onReconnected` for that state. This branch creates the client without that callback path and then waits only for abort, so resolving the conflict this way would drop existing recovery.
