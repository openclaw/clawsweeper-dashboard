## Summary

Keep open: current main still has the hardcoded 25-row text renderer cap, while this PR is a focused implementation with supplied real Discord CLI proof and no blocking correctness or security issue found in the read-only review.

## What This Changes

Threads the message CLI `--limit` value into the text formatter for read, pins, and search output, adds pagination hints, and updates docs, changelog, and formatter tests.

## Best Possible Solution

Land a reviewed version of this PR after normal CI/touched-surface validation, keeping deeper provider pagination work as a separate follow-up if needed.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction path: provider payloads with more than 25 rows reach `renderMessageList`, which slices to 25, and the CLI does not pass the caller's limit into the formatter.

## Solution Assessment

Yes. Passing the existing `--limit` into the shared formatter with focused formatter tests is the narrowest maintainable fix for the display cap; provider-side cursor pagination can stay separate.
