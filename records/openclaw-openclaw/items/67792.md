Keep open. Current main still renders the Telegram model button fallback from the provider-local id, and open PR #72752 explicitly closes this issue, so the right next step is to review and land or close that paired PR rather than close this issue early.

## What This Changes

The issue asks for Telegram model-menu fallback labels to show the full provider/model ref for nested OpenRouter model ids.

## Best Possible Solution

Update the bundled Telegram plugin so nested provider-local fallback labels render as canonical provider/model refs, while preserving display-name overrides, callback data, selection resolution, pagination, truncation, and current-model marking, with focused OpenRouter regression tests.
