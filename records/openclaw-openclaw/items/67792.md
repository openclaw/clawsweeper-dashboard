## Summary

Keep open. Current main still renders the Telegram model menu fallback from the provider-local model id, and the linked fix PR is open and unmerged, so this issue should remain open until maintainers review and land or close that PR.

## What This Changes

The issue requests that Telegram model-selection buttons display the full OpenRouter provider-qualified ref for nested model ids such as `openrouter/openai/gpt-5.4-mini`.

## Best Possible Solution

Keep the issue open and use #72752 as the implementation path; after a reviewed fix lands, close this as implemented by the merged Telegram plugin change.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildModelsKeyboard` with `provider: "openrouter"`, `models: ["openai/gpt-5.4-mini"]`, and no `modelNames` entry; current main falls back to the bare `model` string.

## Solution Assessment

Yes. The narrow maintainable fix is to change only the Telegram fallback display label for nested provider-local ids while preserving `modelNames`, callback data, pagination, truncation, and current-model marking; #72752 appears to follow that shape.
