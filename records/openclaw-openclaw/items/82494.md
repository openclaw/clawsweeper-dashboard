## Summary

Keep open: the normalization change is directionally useful, but it exposes metadata-only entries to a native slash/fast reply path that still assumes every existing entry has a sessionId, and the external PR only provides test proof.

## What This Changes

The PR preserves persisted session-store objects that omit sessionId, adds coverage for that shape, and updates a heartbeat doctor fixture to use a safe session id.

## Best Possible Solution

Keep metadata-only entries loadable, but update every session initialization path to mint a fresh safe ID when sessionId is absent and add regression coverage for the native slash/fast bootstrap path.

## Reproduction Assessment

Yes, by source inspection: after the PR, a persisted metadata-only row can be returned from loadSessionStore, and initFastReplySessionState will reuse its undefined sessionId before resolving a transcript path.

## Solution Assessment

No, not yet; preserving metadata-only entries is the right direction, but the fix needs to carry the fresh-ID fallback into the fast reply/native slash session bootstrap path before merge.
