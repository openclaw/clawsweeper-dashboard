## Summary

Keep open: the latest diff still has a blocking cron correctness issue, and the supplied proof is test-only. The session metadata coverage is compatible with current main, but relaxing persisted `agentTurn.message` validation globally would preserve blank detached jobs that the detached runtime does not skip.

## What This Changes

The PR adds regression coverage for metadata-only session entries and relaxes persisted cron `agentTurn.message` validation so blank main-agentTurn jobs can load into runtime skip handling.

## Best Possible Solution

Keep the session metadata coverage, but scope blank `agentTurn` persistence to main-session rows or add equivalent detached skip handling, then provide redacted persisted-store runtime proof.

## Reproduction Assessment

Yes, by source inspection. Applying the diff lets blank detached `agentTurn` jobs pass persisted-shape validation, and current detached execution forwards `job.payload.message` to isolated execution without a blank-message guard.

## Solution Assessment

No. Preserving repairable main rows is reasonable, but the current PR changes the validation boundary more broadly than the runtime contract supports; the safer fix is target-scoped preservation or matching detached skip handling.
