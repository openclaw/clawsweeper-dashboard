## Summary

Keep open: the session-store coverage is fine, but the cron validation change is too broad. It lets whitespace-only `agentTurn` jobs load for detached targets, and current detached execution forwards that blank message into isolated-agent execution instead of skipping it.

## What This Changes

The PR adds regression coverage for metadata-only session entries and relaxes persisted cron `agentTurn.message` validation so blank main-job rows can reach runtime skip handling.

## Best Possible Solution

Keep the session metadata coverage, but scope blank `agentTurn` preservation to main-session rows or add matching detached skip handling before admitting those persisted jobs.

## Reproduction Assessment

Yes, by source inspection. With the PR diff, whitespace-only `agentTurn` jobs for `isolated`, `current`, or `session:*` targets pass persisted validation and then reach detached execution without a blank-message guard.

## Solution Assessment

No. Preserving repairable main rows is reasonable, but the current patch changes the validation boundary more broadly than the runtime contract supports; target-scoped preservation or equivalent detached skip handling is safer.
