## Summary

Keep open. The PR is focused and plausibly addresses a real persisted-boundary bug, but the latest head still relaxes cron `agentTurn` validation for detached jobs and supplies only test-output proof.

## What This Changes

The PR adds session-store regression coverage and relaxes persisted cron `agentTurn.message` validation so blank main jobs load and can be skipped at runtime.

## Best Possible Solution

Keep the session metadata coverage, but scope blank `agentTurn` preservation to main-session jobs or add matching detached skip handling, then provide redacted persisted-store runtime proof.

## Reproduction Assessment

Yes, by source inspection. A whitespace-only persisted `agentTurn` job with `sessionTarget` `isolated`, `current`, or `session:*` would pass the PR's shape validation and then flow into detached execution without a blank-message guard.

## Solution Assessment

No. Preserving repairable main rows is reasonable, but the current PR changes the validation boundary more broadly than the runtime contract supports; target-scoped preservation or matching detached preflight is safer.
