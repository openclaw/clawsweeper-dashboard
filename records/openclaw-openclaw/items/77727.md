## Summary

Keep open: current `main` still has a source-backed Telegram ordering gap for long/chunked finals after visible non-preview progress, even though the shipped #76529 fix covers short editable finals.

## What This Changes

Telegram streaming should keep the final assistant reply after intermediate tool/progress messages when multiple tool calls occur.

## Best Possible Solution

Telegram finalization should apply the same fresh-final ordering rule to long/chunked finals that already applies to short finals, with focused regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes, by source inspection: create a Telegram preview, deliver a visible non-preview block/tool progress message, then deliver a final answer that splits into multiple chunks. Current `main` can edit the old preview to the first final chunk before sending the rest, placing that first final chunk above the later progress output.

## Solution Assessment

Yes, the narrow fix is to extend the existing fresh-final guard into the long/chunked final path. A new mode, config option, or core channel contract change is not needed.
