## What This Changes

The branch adds skipped-load diagnostics to skill loading, a new `openclaw skills lint` command with text/JSON output, docs, and tests for malformed or incomplete `SKILL.md` files.

## Best Possible Solution

Align the lint scanner with runtime grouped-skill discovery depth and candidate behavior, add a deepest-depth regression test, then let maintainers decide whether `skills lint` alone satisfies the broader diagnostics issue.

Yes, from source inspection: runtime grouped discovery can reach `SKILL.md` at deeper grouped levels than the PR's new lint walker, so a malformed runtime-visible skill at that depth remains unreported. I did not run a filesystem repro because this review was read-only.
