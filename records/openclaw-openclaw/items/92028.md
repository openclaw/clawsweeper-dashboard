## Summary

Keep this PR open for revision: strict malformed-frontmatter diagnostics are useful and the earlier YAML/nested-group findings were addressed, but the lint walker does not match runtime discovery for symlinked skill directories, and the external PR still lacks real CLI proof.

## What This Changes

The PR adds `openclaw skills lint [path]` with strict YAML and required-field diagnostics, recursive scanning, JSON output, nonzero failure exits, and focused loader/status tests.

## Best Possible Solution

Reuse one bounded runtime candidate-discovery primitive for both loading and linting, with lint adding strict YAML and structured failure collection; document the command and prove text/JSON output and exit codes in a real terminal.

## Reproduction Assessment

Yes for the review defect: source inspection shows runtime follows eligible directory symlinks while the proposed lint enumeration excludes them, so the two paths can inspect different skill candidates. The PR’s intended CLI result itself has not been demonstrated in a real environment.

## Solution Assessment

No, not in the current shape. Strict lint-only parsing is appropriate, but a separate filesystem walker is not the best fix because diagnostics should use the same candidate-discovery contract and limits as runtime loading.
