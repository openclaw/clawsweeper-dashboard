## Summary

Keep open. Current main still proves only that invalid-model errors fall back after they surface; it does not prove deterministic 400/unsupported-parameter candidates bypass the request or stream wait before the next model is attempted.

## What This Changes

The issue asks OpenClaw to fail over immediately from deterministic invalid model or unsupported-parameter errors instead of spending about 30 seconds per failed fallback candidate.

## Best Possible Solution

Add an elapsed-time regression for deterministic invalid-model and unsupported-parameter failures, then make that path fail fast without weakening normal slow-provider timeouts.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a delayed current candidate blocks the sequential fallback loop, and current tests only cover immediate invalid-model rejection after the error has already surfaced.

## Solution Assessment

Unclear until the delay source is isolated. The best fix is a narrow fail-fast path at the request/SDK/embedded-run boundary plus elapsed-time coverage, not a broad timeout-policy change.
