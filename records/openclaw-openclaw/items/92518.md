## Summary

The reported delivery failure is valid and the text-link degradation is a maintainable direction, but the fallback trigger is broader than the documented app-auth failure and the claimed production validation is not inspectable. This PR should remain open for a narrow classifier repair and redacted real Google Chat proof.

## What This Changes

The PR adds text-link fallback delivery for remote Google Chat media when attachment upload reports an authorization failure, covering regular replies and outbound adapter sends.

## Best Possible Solution

Keep native uploads when authorized, fall back only for the documented insufficient-authentication-scope response on remote URLs, retain the existing failure path for unrelated errors, and demonstrate both regular reply and outbound-adapter delivery in a real app-auth conversation.

## Reproduction Assessment

Yes at source-contract level: current main always calls Google’s user-auth-only attachment-upload operation for media, so an app-auth token follows a documented unsupported path. This review did not independently run a credentialed Google Chat reproduction.

## Solution Assessment

Not yet in its current form. Text-link degradation is the narrowest solution without adding configuration, but the trigger should identify the documented insufficient-scope response rather than every 403 or generic permission denial.
