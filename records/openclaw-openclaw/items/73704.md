## Summary

Keep open. The PR addresses a real current-main safeguard compaction bug, but the latest diff still resolves configured override models with a raw registry lookup instead of the existing OpenClaw model-resolution contract, so valid custom/inline compaction models can still be dropped.

## What This Changes

The PR passes model registries into safeguard setup, resolves a configured compaction target before registering safeguard runtime state, prefers the runtime model in the safeguard hook, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the runtime-model precedence change, but resolve safeguard runtime targets through the shared model resolver so registry, inline/custom provider, provider fallback, and normalization behavior stay consistent with manual compaction.

## Reproduction Assessment

Yes by source inspection. On current main, set safeguard compaction with `agents.defaults.compaction.model` different from the active session model; runtime registration stores the session model and the hook prefers `ctx.model`.

## Solution Assessment

No as-is. The PR direction is narrow and appropriate, but the raw `modelRegistry.find` path should be replaced with the full model-resolution path used elsewhere before merge.
