## What This Changes

Land a refreshed, narrow version that reuses the existing compaction resolver for safeguard runtime registration while preserving current-main middleware, alias resolution, no-override fallback, and provider/auth regression coverage.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 document the override but register the session model into safeguard runtime and prefer ctx.model, so an active session with a different compaction.model override ignores that config.

## Solution Assessment

Yes, with a branch refresh. Resolving the configured compaction target before safeguard runtime registration is the narrow shared-boundary fix; a hook-only precedence swap is too narrow because runtime.model is currently populated with the session model.
