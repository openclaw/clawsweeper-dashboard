## Summary

Keep open: current main still has the source-level safeguard compaction bug, and this PR targets the right runtime registration path. Before merge, the contributor still needs real non-mocked after-fix behavior proof, and there is one small diagnostic warning mismatch to clean up.

## What This Changes

This PR resolves the configured compaction provider/model before safeguard runtime registration, makes the safeguard hook prefer the registered runtime model, threads registry/workspace context through embedded runners, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the source fix direction, adjust the unresolved-override diagnostic so it matches compact.ts behavior, then merge after real after-fix proof from an actual safeguard auto-compaction run is supplied.

## Reproduction Assessment

Yes, from source inspection. Current main registers the session model in the safeguard runtime and then prefers ctx.model in the hook, so a configured compaction model cannot win in active-session safeguard auto-compaction.

## Solution Assessment

Mostly yes. Resolving the compaction target before runtime registration and preferring runtime.model is the narrow owner-path fix, but the diagnostic wording and real behavior proof should be tightened before merge.
