## Summary

Keep open. Current main still has the safeguard compaction model override gap, and this PR is the right area, but the patch currently misses supported inline/custom provider models and needs a focused resolver fix before merge.

## What This Changes

The PR threads modelRegistry into embedded compaction setup, resolves a safeguard runtime target from agents.defaults.compaction.model, prefers runtime.model in the safeguard hook, and adds tests plus a changelog entry.

## Best Possible Solution

Resolve safeguard runtime targets through the same full model-resolution contract used by manual compaction, add inline/custom provider regression coverage, then rebase and validate the focused agent compaction surface.

## Reproduction Assessment

Yes. On current main, configure safeguard compaction with agents.defaults.compaction.model different from the active session model; source inspection shows runtime registration stores the session model and the hook prefers ctx.model. For the PR finding, use a compaction.model that is defined only in models.providers.*.models[] and the new raw registry lookup cannot resolve it.

## Solution Assessment

No as-is. The PR direction is narrow and maintainable, but the runtime target lookup should not use raw modelRegistry.find because the supported model resolver also covers inline provider config, provider defaults, suppressions, and runtime normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve compaction targets through the full model resolver:** `src/agents/pi-embedded-runner/extensions.ts:166`
  - body: This new lookup only asks modelRegistry.find for a configured compaction model. Valid custom or proxy models from models.providers.*.models[] are resolved separately by the existing inline-provider path, so a safeguard compaction override pointing at one of those models leaves runtime.model undefined and the hook falls back to the session model. Reuse or export the existing model resolver contract here before registering the runtime.
