## Summary

Keep open. Current main still has the reported safeguard model-precedence bug, and this PR is a useful implementation candidate, but the patch still resolves configured safeguard targets with a raw registry lookup that misses supported inline/custom model definitions.

## What This Changes

The PR resolves safeguard auto-compaction runtime provider/model data before registration, prefers `runtime.model` in the safeguard hook, passes `modelRegistry` from embedded call sites, and adds tests plus a changelog entry.

## Best Possible Solution

Reuse the existing model-resolution contract for safeguard runtime targets, including inline/custom provider config and runtime normalization, while keeping the runtime-model precedence change and focused regressions.

## Reproduction Assessment

Yes by source inspection. On current main, configure safeguard compaction with `agents.defaults.compaction.model` different from the active session model; runtime registration stores the session model and the hook prefers `ctx.model`.

## Solution Assessment

No as-is. The PR direction is narrow, but the runtime target lookup should call or share the full model-resolution path used by manual compaction instead of using raw `modelRegistry.find`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve compaction targets through the full model resolver:** `src/agents/pi-embedded-runner/extensions.ts:166-170`
