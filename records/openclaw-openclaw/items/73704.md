## Summary

Keep open: current main still ignores the configured compaction model in safeguard auto-compaction, and this PR is directionally right, but its new resolver still misses supported inline/custom model overrides.

## What This Changes

The PR resolves safeguard auto-compaction runtime model/provider data before registration, prefers `runtime.model` in the safeguard hook, passes `modelRegistry` from embedded call sites, and adds tests plus a changelog entry.

## Best Possible Solution

Reuse the existing model-resolution contract for safeguard runtime targets, including inline/custom provider config and runtime normalization, while keeping the runtime-model precedence change and focused regressions.

## Reproduction Assessment

Yes. On current main, configure `agents.defaults.compaction.mode` as `safeguard` with `agents.defaults.compaction.model` different from the active session model; source shows runtime registration stores the session model and the hook prefers `ctx.model`. For the PR finding, use a compaction override defined only under `models.providers.*.models` and absent from the raw registry.

## Solution Assessment

No as-is. The PR direction is narrow, but the resolver should call or share `resolveModelWithRegistry`/`resolveModelAsync` instead of using raw `modelRegistry.find` so supported custom provider models behave like manual `/compact`.
