## Summary

Keep open. Current main still has the safeguard compaction model-selection bug, and this PR is a focused implementation candidate that addresses the remaining embedded safeguard runtime gap while preserving the linked issue for merge automation. I did not find a discrete functional or security review finding in the latest PR diff; the remaining work is branch health, CI proof, and normal maintainer review.

## What This Changes

The PR threads `ModelRegistry` into embedded safeguard extension setup, resolves `agents.defaults.compaction.model` before registering the safeguard runtime, prefers `runtime.model` in the safeguard hook, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused patch after rebase and green checks: safeguard runtime registration should resolve the effective compaction provider/model through the existing resolver, look up the configured model through the model registry, keep no-override fallback, and preserve manual `/compact` and context-engine behavior.

## Reproduction Assessment

Yes. On current main, configure `agents.defaults.compaction.mode` as `safeguard` with a compaction model different from the active session model; source inspection shows runtime registration stores the session model and the hook prefers `ctx.model`, so the configured model cannot win.

## Solution Assessment

Yes. The PR uses the narrow existing resolver and exact `ModelRegistry.find` contract instead of inventing new parsing, and its tests cover the missing auto-compaction path plus fallback behavior; the safer next step is validation, not a different design.
