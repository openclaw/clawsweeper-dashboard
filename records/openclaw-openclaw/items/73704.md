## Summary

Keep open. The PR fixes the main safeguard precedence path, but the branch still resolves configured compaction overrides with a raw registry lookup, which misses supported inline/custom model configuration handled by the existing model resolver.

## What This Changes

The PR resolves safeguard auto-compaction runtime model data before registration, changes the safeguard hook to prefer `runtime.model`, passes `modelRegistry` into embedded call sites, and adds tests plus a changelog entry.

## Best Possible Solution

Register the safeguard runtime with the same full model-resolution contract used for normal agent model selection, while keeping the runtime-model precedence change and adding inline/custom-provider regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main registers safeguard runtime with the session model and the hook prefers `ctx.model`; the PR branch fixes that path but still has a source-reproducible miss for valid inline/custom configured compaction models.

## Solution Assessment

No as-is. The PR direction is narrow, but the runtime target must use or share `resolveModelWithRegistry`-level behavior instead of raw `modelRegistry.find`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve compaction targets through the full model resolver:** `src/agents/pi-embedded-runner/extensions.ts:166-169`
  - body: This lookup only asks the raw registry for a configured compaction override. Supported inline/custom models under `models.providers.*.models` are resolved by the existing model resolver and may not exist in the registry, so a valid override can leave `runtime.model` undefined and the safeguard hook falls back to the session model or cancels in the direct compact path. Reuse the full model-resolution path before registering the runtime.
  - confidence: 0.88
