## Summary

Keep open. ## What This Changes

The PR resolves safeguard auto-compaction through the existing compaction target resolver, passes model-registry context into embedded runtime registration, flips safeguard model precedence, and adds regression plus terminal proof coverage.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible. Current main documents the override, but the safeguard runtime stores the active session model and the hook prefers ctx.model, so an active session with a different compaction.model override ignores that config; I did not run a live long-session compaction turn.

## Solution Assessment

Yes, with a branch refresh. Reusing the existing compaction target resolver before safeguard runtime registration is the narrow owner-boundary fix, and the precedence swap is necessary so the resolved runtime model can win over ctx.model.
