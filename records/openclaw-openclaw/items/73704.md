## Summary

Keep open. The PR addresses a real current-main safeguard compaction bug, but the proposed resolver still bypasses the supported model-resolution path and can drop valid configured compaction models.

## What This Changes

The PR wires modelRegistry into embedded compaction setup, resolves a configured safeguard compaction target before runtime registration, prefers runtime.model in the safeguard hook, and adds regression tests plus a changelog entry.

## Best Possible Solution

Resolve the safeguard runtime model through the same supported model-resolution contract as manual compaction, with regression coverage for inline/custom provider overrides and the no-override session-model fallback.

## Reproduction Assessment

Yes. On current main, configure safeguard compaction with agents.defaults.compaction.model different from the active session model; source inspection shows runtime registration stores the session model and the hook prefers ctx.model. For the PR finding, configure the compaction model as an inline/custom models.providers.*.models[] entry that is not present in the raw registry.

## Solution Assessment

No as-is. The PR direction is narrow, but the runtime target lookup should reuse or extract the full resolver instead of calling raw modelRegistry.find directly.
