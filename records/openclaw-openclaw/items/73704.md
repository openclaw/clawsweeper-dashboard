## What This Changes

The PR resolves the safeguard auto-compaction provider/model before runtime registration, switches the hook to prefer the resolved runtime model, threads registry/workspace context into callers, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this focused resolver and hook-precedence fix after redacted terminal output or logs show a real safeguard auto-compaction run using the configured compaction model instead of the session model.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main registers the active session model in safeguard runtime and then resolves `ctx.model ?? runtime?.model`, so a different configured compaction model cannot win when `ctx.model` is populated. I did not run a live provider-backed compaction session in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes. Reusing the existing compaction target resolver before safeguard runtime registration and preferring the resolved runtime model is the narrow maintainable fix; merge should wait for real behavior proof, not a replacement implementation.
