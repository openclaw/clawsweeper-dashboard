## Summary

Keep open. Current main still has the safeguard compaction model-precedence bug, and this PR is a useful but incomplete fix because the new resolver still bypasses the full configured-model resolution contract for inline/custom provider models.

## What This Changes

The branch wires the model registry into embedded compaction call sites, adds a safeguard runtime target resolver and warning, flips safeguard model precedence to runtime-first, and adds tests plus a changelog entry.

## Best Possible Solution

Safeguard runtime target resolution should share the existing full model resolver, then keep the runtime-first precedence change and add regression coverage for inline/custom configured provider models.

## Reproduction Assessment

Yes by source inspection. On current main, configure safeguard compaction with `agents.defaults.compaction.model` different from the active session model; runtime registration stores the session model and the hook prefers `ctx.model`.

## Solution Assessment

No as-is. The PR direction is narrow, but the runtime target lookup should reuse the full model-resolution path instead of raw `modelRegistry.find`, which misses supported inline/custom configured models.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the full model resolver for compaction overrides:** `src/agents/pi-embedded-runner/extensions.ts:166-170`
  - body: The helper still resolves an overridden safeguard target with `modelRegistry.find` only. Valid inline/custom models from `models.providers.*.models` are resolved by the existing model resolver, not guaranteed to exist in the raw registry, so those supported overrides leave `runtime.model` undefined and auto-compaction falls back to the session model instead of the configured compaction model.
