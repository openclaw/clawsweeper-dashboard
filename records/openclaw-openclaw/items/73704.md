## Summary

Keep open. Current main still does not honor `agents.defaults.compaction.model` in the safeguard auto-compaction runtime path, and this PR is a focused implementation candidate rather than obsolete work. I did not find a concrete code-level bug in the proposed diff, but the PR is not merge-ready because GitHub reports it as dirty against main and the current head has failing checks.

## What This Changes

The PR threads `modelRegistry` into embedded compaction extension setup, resolves the configured compaction provider/model before safeguard runtime registration, prefers `runtime.model` in the safeguard hook, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused patch after rebasing and clearing CI: safeguard runtime registration should resolve the effective compaction provider/model through the existing resolver, look up the configured runtime model through the model registry, preserve no-override fallback, and keep the explicit `/compact` and context-engine compaction paths unchanged.

## Reproduction Assessment

Yes. On current main, configure safeguard compaction with `agents.defaults.compaction.model` different from the active session model; source inspection shows runtime registration stores the session model and the hook prefers `ctx.model`, so the configured model cannot win in that path.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix because it reuses the existing compaction target resolver and changes only the safeguard runtime model selection surface; the remaining blocker is branch health, not a better product design.
