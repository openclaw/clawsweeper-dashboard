## What This Changes

Read-only CLI commands such as `openclaw health --json`, `openclaw cron list --json`, and `openclaw tasks list` should not trigger a systemd-managed Gateway restart or port takeover.

## Best Possible Solution

Add a narrow regression-backed guard so read-only CLI query commands cannot invoke Gateway startup/restart paths, and make systemd-supervised duplicate lock or EADDRINUSE conflicts fail closed with sysexits 78 instead of retrying into an occupied port. Keep unmanaged duplicate-start behavior intact and document the user-facing fix in the changelog.
