Keep open. Current main routes several listed read-only commands through lean/RPC paths, but it still implements and tests the exact systemd duplicate-start wait/retry branch when the existing Gateway health probe is false, so the v29 restart/takeover report is not fully implemented away.

## What This Changes

The issue asks OpenClaw to stop read-only CLI commands from triggering a systemd-managed Gateway restart or port takeover and to restore immediate exit-78 duplicate-start behavior.

## Best Possible Solution

Make read-only CLI surfaces unable to enter the Gateway runtime, and make systemd-supervised duplicate lock or EADDRINUSE conflicts fail closed with sysexits 78 instead of waiting into a takeover window.

## Solution Assessment
