## Label Justifications

- `P1`: The affected older release consistently returned the wrong agent's data for a core multi-agent cost-accounting workflow.
- `impact:other`: The issue concerns inaccurate per-agent operational cost reporting, which is meaningful but outside the narrower owned impact categories.

## Summary

Current main and the latest release already honor `agentId` for `usage.cost`, isolate the in-memory cache by agent scope, and cover the behavior with a focused regression test; the report was reproduced on older version `2026.5.27`, before the merged fix.

## What This Changes

Make the `usage.cost` RPC return and refresh cost usage for the requested agent without cross-agent memory-cache collisions.

## Best Possible Solution

Keep the shipped canonical agent-aware cache path and advise affected users to upgrade from `2026.5.27` to `v2026.6.6` or later.

## Reproduction Assessment

No on current main or `v2026.6.6`: the handler, memory-cache key, lower cache loader, focused test, and shipped tag all show that the requested agent is honored.

## Solution Assessment

Yes. The merged implementation extends the existing agent-aware cache subsystem and separates cache entries by scope, directly addressing both the wrong cache path and cross-agent collision described in the report.
