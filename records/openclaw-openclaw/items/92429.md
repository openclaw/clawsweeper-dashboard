## Label Justifications

- `P1`: The released regression consistently returns incorrect cost data for secondary agents and breaks a real multi-agent accounting workflow.
- `impact:other`: The issue affects the accuracy of per-agent token and cost reporting through the gateway.

## Summary

Current `main` honors `agentId` in `usage.cost`, separates in-memory cache entries by agent scope, forwards the agent to the on-disk cache loader, and documents the contract. The implementation postdates the latest release, so released users still need a future release or a current-main build.

## What This Changes

Make the `usage.cost` gateway RPC honor an optional agent identifier and return or rebuild usage-cost data for that agent instead of always using the main agent.

## Best Possible Solution

Keep the current agent-scoped RPC and cache implementation and include it in the next OpenClaw release.

## Reproduction Assessment

Yes. The report gives a deterministic multi-agent gateway call, and source inspection confirms the older failure shape and the current agent-aware repair path; no live command was run because this was a read-only review.

## Solution Assessment

Yes. Passing the existing optional agent identifier through the established cache API and including agent scope in the process-cache key is the narrow canonical solution without adding another RPC or configuration surface.
