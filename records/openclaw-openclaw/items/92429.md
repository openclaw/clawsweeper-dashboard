- `P1`: The affected release consistently returned the wrong agent's accounting data for secondary-agent queries, although a shipped fix is now available.
- `impact:other`: The issue concerns cross-agent usage-cost attribution and reporting correctness.

## Summary

Current main and v2026.6.6 already honor the requested agent across the `usage.cost` handler, in-memory cache identity, transcript discovery, and disk cache path; focused regression tests cover the reported failure.

## What This Changes

Make `usage.cost` return and refresh usage-cost data for the requested agent instead of always resolving to the default agent.

## Best Possible Solution

Keep the shipped scope-aware implementation and have installations still using 2026.5.27 upgrade to v2026.6.6 or newer.

## Reproduction Assessment

No on current main or the v2026.6.6 release tree: the handler forwards `agentId`, the in-memory key separates agent scopes, and the disk cache and transcript discovery use the requested agent.

## Solution Assessment

Yes. The merged implementation carries agent identity through the existing canonical RPC and cache pipeline, preserves the omitted-agent fallback, and adds focused isolation tests rather than creating a parallel usage path.
