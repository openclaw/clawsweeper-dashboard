## Label Justifications

- `P2`: The older release consistently returned incorrect cost data for secondary agents, but agent execution remained available and the fix is now shipped.
- `impact:other`: The issue affects the accuracy of operator-visible per-agent usage and cost accounting.

## Summary

Current main and v2026.6.6 already honor `agentId` for `usage.cost`, forward it to the agent-specific disk cache, and isolate in-memory summaries by agent; the report was filed against older version 2026.5.27.

## What This Changes

Make `usage.cost` return cost summaries for the requested agent and prevent cross-agent cache collisions.

## Best Possible Solution

Keep the shipped scope-aware implementation and have installations on 2026.5.27 upgrade to v2026.6.6 or newer.

## Reproduction Assessment

No on current main or v2026.6.6: the handler forwards `agentId`, the disk cache resolves by agent, and the in-memory cache key separates agent scopes.

## Solution Assessment

Yes. The merged implementation carries agent identity through the existing RPC and cache pipeline, covers selected-agent and all-agent views, and includes focused regression tests.
