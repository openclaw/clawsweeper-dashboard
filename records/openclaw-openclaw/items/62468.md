## Summary

Keep open: current main still lacks a Feishu resolver for a bare username such as `jaydenli`, so the target falls through to `Unknown target`, and the related implementation attempt #62510 is closed and unmerged.

## What This Changes

The issue asks Feishu outbound delivery to resolve a bare username such as `jaydenli` to a user `open_id`, or provide a clearer target-format/lookup path.

## Best Possible Solution

Add a Feishu-owned resolver that maps username-like inputs to one unique user `open_id`, reports clear ambiguous or missing-target errors, and documents the supported lookup behavior.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `target: "jaydenli"` is not ID-like, is inferred as a group query, misses Feishu directories, has no Feishu fallback resolver, and throws `Unknown target`.

## Solution Assessment

Yes. Keeping the change in the Feishu plugin is the narrowest maintainable fix because Feishu owns its user fields, API permissions, ID formats, and ambiguity rules; changing generic bare-token behavior in core would risk other channels.
