## Summary

Keep open: current main still cannot resolve a bare Feishu username like `jaydenli` to a user `open_id`, and the only linked implementation attempt, #62510, is closed and unmerged.

## What This Changes

Resolve Feishu message targets like `jaydenli` to a unique user `open_id` or return a clearer Feishu-specific target error.

## Best Possible Solution

Add a Feishu-owned target resolver that maps username-like inputs to one unique user `open_id`, reports clear ambiguous/missing-target errors, and documents the supported lookup behavior.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `target: "jaydenli"` is not ID-like, is inferred as a group query, misses Feishu directories, has no Feishu fallback resolver, and throws `Unknown target`; no Feishu live credentials were visible in env/profile for an API run.

## Solution Assessment

Yes. Keeping the change in the Feishu plugin is the narrowest maintainable fix because Feishu owns its user fields, API permissions, ID formats, and ambiguity rules; changing generic bare-token behavior in core would risk other channels.
