## Summary

This PR should stay open: it is protected by the maintainer label, the central Telegram fixes are plausible, but the patch introduces a code-formatting regression and still needs stronger real Telegram proof before merge.

## What This Changes

The PR caps Telegram native quote reply text/entities, preserves model-emitted Telegram code/pre HTML tags during outbound chunking, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the quote cap, but move code/pre tag promotion into the existing preservation path so escaped tags are only promoted outside generated Markdown code/pre regions, then add regression coverage for literal code tags inside Markdown code spans or fences.

## Reproduction Assessment
