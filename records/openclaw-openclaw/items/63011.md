## Summary

Keep open. Current main still has a source-reproducible gap: durable sends can record hook cancellation as `suppressed`, but gateway `send` still converts non-`sent` outcomes to `No delivery result`, and `message_sending` still lacks the requested outbound identity context.

## What This Changes

Define one canonical outbound `message.send` path that runs `message_sending`, returns delivery success, surfaces guarded denial, and carries audit correlation identity.

## Best Possible Solution

A maintainer-approved outbound send contract should build on the durable lifecycle, thread available agent/session/request identity into `message_sending`, and return a documented delivery-or-denial result for the chosen canonical surface.

## Reproduction Assessment

Yes. A high-confidence source path remains: hook cancellation becomes a durable `suppressed` outcome, gateway `send` discards non-`sent` results and throws `No delivery result`, and the hook context lacks the requested identity fields.

## Solution Assessment

Unclear. The durable lifecycle is the right foundation, but the final fix needs maintainer API judgment on the canonical surface and should coordinate with the open draft PR rather than landing a competing branch.
