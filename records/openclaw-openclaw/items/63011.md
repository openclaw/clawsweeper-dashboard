## Summary

Keep open. Current main still drops hook-suppressed gateway sends into `No delivery result`, does not thread outbound `agentId` or session/request identity into `message_sending`, and the issue is paired with an open same-author draft PR plus overlapping hook-identity PRs that need maintainer coordination.

## What This Changes

The issue requests one canonical outbound `message.send` path that runs `message_sending` hooks, returns delivery results, surfaces guarded denial, and carries identity for audit correlation.

## Best Possible Solution

Review and either land or replace the paired gateway-denial PR together with the hook-identity work so one documented outbound path returns sent-versus-blocked outcomes and carries the agreed correlation fields.

## Reproduction Assessment

Yes. Source inspection is enough: a hook cancellation becomes a durable `suppressed` send, but gateway `send` ignores that status and throws `No delivery result`, while the outbound hook call still omits requested identity.

## Solution Assessment

Unclear. The durable suppression lifecycle is the right base, but maintainers still need to choose whether the canonical contract is gateway-only or must also align direct CLI/plugin-action sends.
