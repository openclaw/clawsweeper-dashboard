## Summary

Keep open. Current main still leaves the requested canonical guarded outbound-send contract incomplete: gateway `send` drops durable `suppressed` outcomes into `No delivery result`, `message_sending` still does not receive outbound agent/session/request identity, and the same-author draft implementation PR remains open for maintainer review.

## What This Changes

Request one canonical outbound `message.send` path that runs through `message_sending`, returns real delivery success, surfaces guarded denial, and preserves identity for audit correlation.

## Best Possible Solution

Review and either land the paired draft PR or replace it with a maintainer-approved canonical outbound send contract that exposes explicit sent-versus-blocked outcomes and carries identity through the hook path.

## Reproduction Assessment

Yes. A high-confidence source path remains: hook cancellation produces a durable `suppressed` outcome, but gateway `send` filters that out and throws `No delivery result`, while the hook call still lacks the requested identity fields.

## Solution Assessment

Unclear. The durable lifecycle is the right foundation, but maintainers still need to decide whether the canonical contract is gateway-only or must also cover direct CLI/plugin-action send boundaries.
