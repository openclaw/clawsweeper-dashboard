## Summary

Keep open. Current main has moved outbound delivery onto the durable message lifecycle and now records suppressed hook outcomes internally, but the requested canonical outbound `message.send` contract is still incomplete: gateway `send` still turns suppressed/non-sent delivery into `No delivery result`, `message_sending` still does not receive agent/session/request identity, and the same-author draft implementation PR remains open.

## What This Changes

The issue requests one canonical outbound `message.send` path that runs `message_sending`, returns real delivery results, exposes guarded denials, and preserves audit correlation identity.

## Best Possible Solution

Build on the durable message lifecycle by having the chosen canonical send surface consume `suppressed` outcomes explicitly, thread outbound identity into `message_sending`, and return a documented delivery-or-denial result with focused gateway/CLI tests.

## Reproduction Assessment

Yes. A high-confidence source path remains: a `message_sending` hook cancellation records a suppressed durable outcome, but gateway `send` maps every non-`sent` result to an empty result and throws `No delivery result`; the hook call also lacks the requested identity fields.

## Solution Assessment

Unclear. The durable message lifecycle is the right foundation, but the final solution still needs maintainer agreement on whether gateway `send` alone is canonical or whether direct CLI/plugin-action sends must expose the same audited contract.
