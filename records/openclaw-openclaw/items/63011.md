## Summary

Keep open. Current main still leaves outbound send semantics split: hook cancellation is skipped as an empty delivery result, gateway `send` still converts that into `No delivery result`, `message_sending` still lacks the requested outbound identity, and the same-author draft PR #63007 remains the active implementation candidate.

## What This Changes

Define a canonical outbound `message.send` contract that runs through `message_sending`, returns provider delivery success, reports guarded denial, and preserves agent/session/request identity for audit correlation.

## Best Possible Solution

Review and either land #63007 or replace it with a maintainer-approved canonical send contract that threads identity into `message_sending`, returns explicit guarded-denial outcomes, preserves provider delivery results, and documents or aligns the CLI, gateway, and plugin-action paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to have a `message_sending` hook return `cancel: true`: `deliverOutboundPayloads` skips delivery and returns no result, and gateway `send` maps that empty result to `No delivery result`.

## Solution Assessment

Unclear. A gateway-only fix may be the narrowest maintainable answer if maintainers choose gateway `send` as canonical, but the issue explicitly calls out direct CLI and plugin-action ambiguity as part of the remaining contract decision.
