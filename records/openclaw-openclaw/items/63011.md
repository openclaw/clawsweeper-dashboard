## Summary

Keep open. Current main still has a source-reproducible guard/audit gap: gateway `send` discards durable `suppressed` outcomes into `No delivery result`, `message_sending` still lacks outbound agent/session/request identity, and an open same-author draft PR already targets the central gap.

## What This Changes

Request one canonical outbound `message.send` path that runs hooks, returns delivery results, surfaces guarded denials, and preserves audit correlation identity.

## Best Possible Solution

Review and either land or replace the open draft PR with a current-main patch that threads outbound identity into `message_sending`, maps hook-cancelled durable suppression to an explicit gateway denial, and documents or aligns the chosen canonical send surface.

## Reproduction Assessment

Yes. A high-confidence source path remains: hook cancellation produces a durable `suppressed` outcome, but gateway `send` maps non-`sent` results to `No delivery result`, and the hook context omits the requested identity fields.

## Solution Assessment

Unclear. The durable message lifecycle is the right foundation, but maintainers still need to approve the canonical public surface and the exact delivery-or-denial response contract.
