## Summary

Keep open; the feature belongs in the acpx plugin, but the PR currently re-flattens argv-style args with a raw space join, which breaks arguments containing whitespace or quotes, and it also misses docs/changelog coverage for a new user-facing config field.

## What This Changes

The PR adds an optional `agents.<id>.args` array to the acpx plugin config schema/types and resolves command plus args into the string override passed to the acpx agent registry.

## Best Possible Solution

Keep the acpx-owned config addition, but only land it after preserving argv element boundaries with quote-safe encoding or a structured upstream override, plus docs and changelog coverage.

## Reproduction Assessment

Yes for the PR-introduced defect: the patch's raw `cmdArgs.join(" ")` feeds a command string into the existing whitespace/quote parser, so an arg like `/path/with space/adapter.mjs` becomes multiple argv elements. I verified this from the PR diff and current parser/spawn path; no runtime test was run in this read-only pass.

## Solution Assessment

No. The acpx plugin is the right boundary for this config shape, but flattening args with a raw whitespace join is not the narrow maintainable implementation; it needs quote-safe encoding or an upstream structured override contract.
