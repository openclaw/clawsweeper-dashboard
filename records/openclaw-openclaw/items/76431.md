## Summary

Keep open: current main still only supports ACPX agent command strings, and this PR is the dedupe parent for a useful plugin-local feature, but it should not merge until empty-argument behavior is fixed and proof covers the ACPX override/flattening path.

## What This Changes

The PR adds an optional `agents.<name>.args` array to the ACPX plugin config/schema/manifest/docs/changelog and flattens those arguments into the existing command-string override path with quote handling and tests.

## Best Possible Solution

Keep the plugin-local `args` config direction, fix or reject empty arg elements with parser coverage, then use this parent PR to supersede https://github.com/openclaw/openclaw/pull/76775 once the proof and checks are complete.

## Reproduction Assessment

Yes for the PR-introduced defect: the current parser probe shows quoted-empty command tokens are dropped, and the PR diff accepts any string in `agents.<id>.args`. The requested `args` capability itself is a feature, not a bug reproduction path.

## Solution Assessment

No. The plugin-local config surface is the right place, but the implementation should define empty-arg behavior and prove the ACPX override path rather than only config storage.
