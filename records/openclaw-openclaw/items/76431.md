## Summary

Keep this PR open, but do not merge it as-is. The acpx config feature belongs in the plugin, but the current branch loses argv boundaries by flattening `args` with a raw space join, and the new user-facing config field still needs docs and changelog coverage.

## What This Changes

The PR adds `agents.<id>.args` to the acpx plugin manifest/schema/types, combines command plus args into an agent override string, and adds config tests.

## Best Possible Solution

Land the acpx config shape after quote-safe encoding for every args element, regression tests for whitespace and quotes, and matching docs plus an active changelog entry.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection and a focused parser probe: a config with `args: ["/path/with space/adapter.mjs"]` would become an unquoted command string and split into two argv entries before spawn.

## Solution Assessment

No. The plugin-local config addition is the right area, but raw space-joining is not the best implementation; reuse or add quote-safe command-string encoding at the existing string boundary.
