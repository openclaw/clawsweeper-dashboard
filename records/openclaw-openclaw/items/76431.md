## Summary

Keep open: this is a plausible acpx plugin feature and the latest commit addressed the earlier quote/docs gaps, but the PR still lacks after-fix real behavior proof and the new args contract accepts empty strings that the current command parser drops.

## What This Changes

The PR adds `agents.<id>.args` to the bundled acpx plugin schema/types/docs/changelog and flattens configured args into the existing command-string override path with quote handling and unit coverage.

## Best Possible Solution

Keep the acpx plugin-local config addition only after defining empty-arg behavior, covering it in tests, adding real ACP override proof, and reconciling the duplicate same-author PR.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection plus the current parser probe show `args: [""]` is accepted but becomes no argv element after command-line parsing. The requested feature itself is not a bug reproduction path.

## Solution Assessment

No. The plugin-local config surface is the right area, but the implementation should either reject empty arg strings or preserve them through parsing, and the contributor still needs real behavior proof before merge.
