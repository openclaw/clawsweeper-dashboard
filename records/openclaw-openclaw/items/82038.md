## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Import the callback type from its defining module:** `src/auto-reply/dispatch.ts:23`
  - body: `AfterSourceReplyDeliveryCallback` is defined in `get-reply-options.types.ts`, but `src/auto-reply/types.ts` does not export it. This import from `./types.js` will fail type checking unless the barrel exports the type or this file imports from the defining module.
  - confidence: 0.95
- **[P2] Preserve hook registration identity in workers:** `src/agents/post-turn/isolated-plugin-hook-worker-entry.ts:59-64`
  - body: The parent starts one isolated worker per hook registration, but the child reloads hooks by only `hookName` and `pluginId`, then runs every matching registration. A plugin with two `agent_end` or `llm_output` handlers would run each handler once per registration, duplicating post-turn work and corrupting job accounting.
  - confidence: 0.88
- **[P2] Remove the proxy compose overlay from this fix:** `docker-compose.ai-proxy.yml:3`
  - body: This root compose overlay adds an unversioned proxy image and routes gateway/CLI HTTP(S) traffic through it, which is outside the stated post-delivery and CLI liveness fix. Move it to a separate reviewed change with pinned images and an explicit trust model.
  - confidence: 0.86
