## Summary

Keep open: the latest head has sufficient liveness proof, but it still regresses established Codex app-server client behavior and cuts the client-level dynamic-tool response timeout below the still-supported tool cap.

## What This Changes

The PR defers post-turn work until reply delivery drains, adds durable/isolated post-turn hook execution, serializes Codex app-server attempts, cleans up one-shot CLI harness resources, adjusts CLI runtime routing, and adds focused regression tests.

## Best Possible Solution

Keep the post-delivery and CLI-liveness direction, but preserve current Codex app-server RPC error detail formatting, outbound string sanitization, and 600s dynamic-tool response timeout alignment.

## Reproduction Assessment

Yes. The PR-introduced regressions are source-reproducible by comparing latest-head client.ts lines 30, 45, and 277 with current-main client tests and the dynamic-tool 600000ms cap; I did not run tests in this read-only review.

## Solution Assessment

No. The liveness and post-delivery fix direction looks plausible, but the patch should not remove established Codex app-server client safeguards or make the transport-level timeout preempt valid long-running dynamic tools.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve relogin details in RPC errors:** `extensions/codex/src/app-server/client.ts:45`
  - body: The constructor now uses only `error.message`, dropping relogin `data.detail` from Auth/cloud-requirements RPC errors. The existing client test expects that detail to be appended so users see actionable sign-in guidance instead of only the generic configuration failure.
  - confidence: 0.94
- **[P2] Preserve outbound JSON-RPC string sanitization:** `extensions/codex/src/app-server/client.ts:277`
  - body: Writing `JSON.stringify(message)` directly removes the replacer that strips unpaired surrogate code units before sending JSON-RPC to the app-server. The existing test still covers this because malformed strings can be written as escaped lone surrogates on the protocol path.
  - confidence: 0.95
- **[P2] Keep the app-server timeout aligned with the tool cap:** `extensions/codex/src/app-server/client.ts:30`
  - body: Lowering the app-server server-request timeout to 240000ms makes it fire before dynamic tools that are still allowed to run up to 600000ms. A configured long-running tool between four and ten minutes will get the fallback timeout response before OpenClaw's own tool watchdog completes.
  - confidence: 0.9
