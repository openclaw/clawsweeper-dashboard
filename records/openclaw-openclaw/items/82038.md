## Summary

Keep this PR open, but it is not ready to merge: the durable post-turn hook isolation path can duplicate plugin hook side effects, the branch mixes unrelated Telegram guest-mode/recovery and host-specific Compose proxy work into a liveness fix, and the supplied proof does not cover the visible Telegram changes.

## What This Changes

The PR defers post-turn hooks until source delivery drains, adds durable/isolated post-turn job handling, cleans up one-shot Codex CLI harness resources, and adds broad Telegram guest-mode/final-delivery recovery plus Docker Compose overlays.

## Best Possible Solution

Split this into a narrow CLI/post-delivery liveness fix with exact single-handler post-turn isolation, and move Telegram guest-mode/recovery plus host-specific Compose overlays into separate reviewed PRs with live Telegram and security proof.

## Reproduction Assessment

No high-confidence runtime reproduction was run in this read-only review. Source inspection does give a clear reproduction path for the PR-introduced hook bug: register two agent_end or llm_output handlers from one plugin and the isolated worker will run all matching handlers for each scheduled registration.

## Solution Assessment

No. The durable worker needs to execute exactly the scheduled handler, and the liveness fix should not be bundled with new Telegram modes or host-specific proxy Compose overlays without separate owner review and real Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Run only the scheduled hook in the worker:** `src/agents/post-turn/isolated-plugin-hook-worker-entry.ts:59-63`
  - body: The main runner now schedules one durable job per hook registration, but the worker request only carries hookName/pluginId and then reloads every matching handler for that plugin. If a plugin registers two agent_end or llm_output handlers, each scheduled worker runs both handlers, so side effects happen four times instead of once per handler. Pass a stable registration id/index or otherwise make the worker execute only the handler it was scheduled for.
  - confidence: 0.89
- **[P2] Remove the host-specific proxy overlay:** `docker-compose.ai-proxy.yml:3`
  - body: This root Compose overlay pulls stack-adguard-vpn and forces gateway/CLI HTTP(S) traffic through it. A clean checkout cannot rely on that local/private image or the /opt proxy config, and adding repo-wide proxy routing is unrelated to the CLI liveness fix. Keep this out of the product PR or move it behind an approved, documented ops path.
  - confidence: 0.84
