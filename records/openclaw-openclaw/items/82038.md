Keep open; the latest head fixes the earlier type-export issue, but it still has blocking Codex client regressions, duplicated post-turn hook execution under worker isolation, an unreviewed proxy deployment surface, and missing real Telegram proof for visible behavior.

## Best Possible Solution

  - confidence: 0.9

  - body: This root overlay adds an unpinned proxy image and routes gateway/CLI HTTP(S) traffic through it in a PR scoped as liveness stabilization. Please remove it from this fix or split it into a separate security-reviewed deployment change with pinned images and a documented trust model.
