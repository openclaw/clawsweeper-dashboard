## Summary

Keep open: the latest PR head still has a blocking post-turn hook isolation bug, adds host-specific Docker/proxy surfaces that need security review, and lacks real Telegram proof for visible Telegram behavior.

## What This Changes

The PR adds post-delivery callback deferral, durable/isolated post-turn hooks, one-shot Codex CLI harness cleanup, Telegram guest/draft/final-delivery recovery behavior, Docker Compose overlays, and regression tests.

## Best Possible Solution

Split this into a narrow post-delivery/CLI liveness fix that preserves one execution per hook registration, and move Telegram guest/recovery behavior plus Docker overlays into separate reviewed PRs with live proof.

## Reproduction Assessment

Yes for the PR-introduced hook bug from source inspection: register two agent_end or llm_output handlers from one plugin and the isolated worker will run all matching handlers for each scheduled registration. I did not run a live runtime repro in this read-only review.

## Solution Assessment

No. The liveness fix should pass a stable scheduled-handler identity into the worker, and the Telegram guest/recovery plus host-specific Compose changes should be split for separate owner, security, and live-proof review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Run only the scheduled hook in the worker:** `src/agents/post-turn/isolated-plugin-hook-worker-entry.ts:59-63`
  - body: The caller schedules one durable job per hook registration, but the worker request only carries hookName/pluginId and then reloads every matching registration for that plugin. If a plugin registers two agent_end or llm_output handlers, each scheduled worker runs both handlers, duplicating side effects. Pass a stable registration id/index or otherwise execute only the handler that was scheduled.
  - confidence: 0.9
- **[P2] Remove the host-specific proxy overlay:** `docker-compose.ai-proxy.yml:3`
  - body: This root Compose overlay pulls stack-adguard-vpn and forces Gateway/CLI egress through it with an absolute /opt config mount. A clean OpenClaw checkout cannot rely on that local image/config, and repo-wide proxy routing is unrelated to the CLI liveness fix. Keep it out of this PR or move it behind an approved documented ops path.
  - confidence: 0.86
