## Summary

Keep open: the PR addresses a real source-backed QMD rebind gap, but it is not merge-ready because the enrichment loop can fan out across unrelated listed collections and the external proof is mock/source-only.

## What This Changes

The PR adds a QMD collection metadata enrichment step that calls `qmd collection show <name>` after listing collections so root path drift can trigger rebind.

## Best Possible Solution

Use QMD `collection show` as the metadata source, but only for OpenClaw-managed and derived legacy collection candidates, with focused regression coverage and redacted live QMD proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main ignores missing path metadata from QMD list output and then refuses rebind when `listed.path` is absent, but no live QMD reproduction was run here.

## Solution Assessment

Is this the best way to solve the issue? Not yet; `collection show` is the right dependency-backed primitive, but the PR should bound the enrichment to managed/legacy candidates and prove the changed path directly.
