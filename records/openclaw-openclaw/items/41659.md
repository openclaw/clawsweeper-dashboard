## Summary

Keep open: the arbitrary `gateway.controlUi.basePath` favicon path remains source-reproducible on current main, and the PR is a focused fix, but it still needs a security repair before merge because it interpolates configurable `basePath` into HTML attributes without escaping or validation.

## What This Changes

The PR rewrites Control UI favicon and apple-touch-icon hrefs under a configured gateway basePath and adds focused gateway tests.

## Best Possible Solution

Port a basePath-safe favicon/apple-touch-icon fix onto current main, escape or reject unsafe basePath text before HTML output, preserve CSP hash handling, and keep regression coverage for arbitrary configured basePath.

## Reproduction Assessment

Yes, source-level. Current main emits root-relative icon hrefs, and basePath routing excludes `/favicon.svg` from a `/webchat` mount, so the browser request bypasses the Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index-serving time is narrow for configured basePath, but the implementation must escape or validate basePath and retain current CSP hash handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape basePath before injecting favicon hrefs:** `src/gateway/control-ui.ts:244`
  - body: The helper writes normalized `gateway.controlUi.basePath` directly inside a double-quoted `href`. The schema accepts arbitrary strings and normalization only trims/slashes, so quote-bearing values can break the HTML attribute context when serving the index; escape the attribute value or reject unsafe basePath input before rewriting.
  - confidence: 0.79

## Security Review
