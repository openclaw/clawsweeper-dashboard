Keep open: current main still emits root-relative Control UI icon hrefs, so a configured `gateway.controlUi.basePath` can still send the browser to `/favicon.svg` outside the mounted UI. The fix direction is valid, but this branch is stale against current main’s CSP-aware index response path and needs repair before merge.

## What This Changes

The PR rewrites Control UI favicon and apple-touch-icon hrefs with the configured basePath at index.html serve time and adds focused gateway tests.

## Best Possible Solution

Yes, source-level. Current `ui/index.html` emits root-absolute icon hrefs, browser URL resolution keeps them outside `/webchat/`, and the basePath router excludes root `/favicon.svg` requests from the mounted Control UI.

No, not the branch as-is. The favicon rewrite is a narrow maintainable fix, but it needs to be rebased into the current CSP-aware `serveResolvedIndexHtml` path and preserve the namespace asset mapping already on main.
