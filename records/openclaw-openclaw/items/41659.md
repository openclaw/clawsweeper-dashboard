## Summary

Keep open: current main still has the configured-basePath favicon gap, and the PR addresses it with real proof, but the branch is stale/conflicting and its index-serving hunk must be reconciled with current CSP hash support before merge.

## What This Changes

The PR adds a gateway-side rewrite for Control UI favicon and apple-touch-icon hrefs when index HTML is served under `gateway.controlUi.basePath`, plus focused gateway tests.

## Best Possible Solution

Port the favicon/apple-touch-icon basePath rewrite into current `serveResolvedIndexHtml`, preserving CSP hash behavior and leaving static routing and config semantics unchanged.

## Reproduction Assessment

Yes, source-level. With basePath `/webchat`, current `ui/index.html` asks for `/favicon.svg` while mounted routing excludes paths outside `/webchat`, so the root favicon request bypasses the Control UI handler.

## Solution Assessment

Yes after porting. A gateway-side rewrite for the known favicon/apple-touch-icon hrefs is narrow, but it must be rebased onto the current CSP-aware index-serving code.
