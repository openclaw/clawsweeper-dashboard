## Summary

Keep open. The bug remains source-reproducible on current main for arbitrary configured Control UI base paths, and this PR is a useful narrow fix, but it needs a current-main/CSP-aware port plus safe basePath attribute handling before merge.

## What This Changes

The PR adds a gateway helper to rewrite Control UI favicon and apple-touch-icon hrefs under a configured basePath and adds focused rewrite/no-op tests.

## Best Possible Solution

Port a small gateway-side icon href rewrite into current main's CSP-aware index HTML serving path, escape or validate the configured basePath before HTML injection, and keep focused regression coverage for configured-basePath icon HTML.

## Reproduction Assessment

Yes, source-level. Current main documents gateway.controlUi.basePath, emits root icon hrefs, serves the index body unchanged, and classifies /favicon.svg as outside a configured /webchat mount.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at serve time is a narrow maintainable fix, but it should be ported onto current main with CSP preservation and attribute-safe basePath handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Escape basePath before injecting hrefs:** `src/gateway/control-ui.ts:244-247`
  - body: The new replacement writes `basePath` straight into an HTML attribute, while the config schema accepts arbitrary strings and normalization only trims/slashes. A configured prefix containing a quote or similar delimiter can break the served index HTML or inject attributes, so escape the attribute value or reject invalid prefixes before rewriting.
  - confidence: 0.78

## Security Review

Status: needs_attention

  - body: `injectFaviconBasePath` builds href attributes with the raw configured basePath. Since current config/schema handling leaves basePath as an arbitrary string, a quote-bearing prefix can break the attribute context; escape the value or validate the prefix before injecting it into HTML.
