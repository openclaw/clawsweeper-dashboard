## Summary

Keep open: current main and the latest release still emit root favicon hrefs that bypass a configured Control UI basePath, and this PR is useful but needs a security fix plus a current-main rebase before merge.

## What This Changes

The PR adds gateway-side favicon/apple-touch-icon href rewriting for Control UI index HTML under a configured basePath and adds focused gateway tests.

## Best Possible Solution

Land a current-main version that rewrites only icon hrefs for configured basePath, escapes or validates the emitted attribute value, and retains Control UI CSP handling.

## Reproduction Assessment

Yes, source-level. Current main and v2026.5.12 emit root icon hrefs, and the basePath classifier rejects `/favicon.svg` outside the configured mount, so the browser request bypasses the Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index-serving time is narrow, but the implementation must escape or reject unsafe basePath text and be rebased without losing CSP hash handling.
