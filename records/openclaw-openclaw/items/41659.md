## Summary

Keep open: current main still leaves the configured `gateway.controlUi.basePath` favicon case unresolved, while the later namespace asset fix only covers `/__openclaw__/...` SPA routes. The PR is useful but needs an attribute-safety fix and a current-main port before merge.

## What This Changes

The PR adds a gateway helper and tests to rewrite Control UI favicon and apple-touch-icon hrefs with the configured basePath when serving index HTML.

## Best Possible Solution

Port the favicon/apple-touch-icon basePath handling to current main, escape or reject unsafe basePath text before HTML output, preserve CSP behavior, and keep focused mounted-Control-UI regression coverage.

## Reproduction Assessment

Yes, source-level. Current main documents `gateway.controlUi.basePath`, emits root icon hrefs, serves index HTML unchanged, and excludes `/favicon.svg` from a configured `/webchat` mount.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index serving time is narrow, but the implementation must escape or reject unsafe basePath values and be ported without losing current CSP handling.
