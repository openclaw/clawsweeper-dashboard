## Summary

Keep open. Current main still emits root-relative Control UI icon links and excludes root paths when the UI is mounted under `gateway.controlUi.basePath`, so the bug remains; the PR is useful but stale against current CSP-aware index serving and should not merge as-is.

## What This Changes

The PR adds a gateway-side rewrite for Control UI favicon and apple-touch-icon hrefs when index HTML is served under `gateway.controlUi.basePath`, plus focused gateway tests.

## Best Possible Solution

Port the icon href rewrite into current `serveResolvedIndexHtml`, preserving CSP hash behavior and leaving routing, config, and non-icon asset handling unchanged.

## Reproduction Assessment

Yes, source-level. With basePath `/webchat`, current `ui/index.html` asks for root icon URLs while mounted routing only handles paths at or below `/webchat`; the prefixed asset path is servable once the HTML points there.

## Solution Assessment

Yes after porting. A gateway-side rewrite for the known favicon/apple-touch-icon hrefs is narrow and works for deep SPA fallback routes, but it must be layered onto the current CSP-aware index-serving code.
