## Summary

Keep open: current main still emits root favicon links that bypass a configured Control UI basePath, and this PR contains useful source work plus real after-fix output, but the branch is conflicting and still needs safe basePath handling before merge.

## What This Changes

The PR adds a gateway helper and tests to rewrite Control UI favicon and apple-touch-icon hrefs under the configured basePath when serving index HTML.

## Best Possible Solution

Port the favicon/apple-touch-icon basePath handling onto current main, escape or reject unsafe basePath text before HTML output, preserve CSP behavior, and keep focused mounted-Control-UI regression coverage.

## Reproduction Assessment

Yes, source-level. Current main emits root-relative icon links while configured basePath routing excludes /favicon.svg from a /webchat mount, so the browser request bypasses the Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index serving time is narrow, but the port must escape or validate basePath and retain current CSP hash handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Escape basePath before injecting favicon hrefs:** `src/gateway/control-ui.ts:244`
  - body: The helper writes normalized config `basePath` directly inside a double-quoted `href`.
