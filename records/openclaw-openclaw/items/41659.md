## Summary

Keep open: this is a valid documented Control UI `basePath` bug, and current `main` plus the latest release still serve root favicon hrefs without rewriting. The PR needs a current-main port over the CSP-aware index-serving path plus attribute-safe `basePath` handling before merge.

## What This Changes

The PR adds a gateway helper that rewrites Control UI favicon and apple-touch-icon hrefs with the configured `basePath` at index serve time and adds focused gateway tests.

## Best Possible Solution

Port a small gateway-side icon href rewrite into current main's CSP-aware index HTML serving path, with attribute-safe `basePath` handling and focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main documents `gateway.controlUi.basePath`, emits root icon hrefs, serves the index body unchanged, and classifies `/favicon.svg` as outside a configured `/webchat` mount.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at serve time is narrow and maintainable, but the PR should be ported over current CSP handling and must escape or validate `basePath` before injecting HTML.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Escape basePath before injecting favicon hrefs:** `src/gateway/control-ui.ts:244-247`
  - body: The helper writes the normalized config `basePath` straight into an HTML `href` attribute. Since the schema accepts any string and normalization only trims/slashes, a quote-bearing prefix can break the attribute context or add attributes to the served index; escape the value or reject invalid prefixes before rewriting.
  - confidence: 0.78
