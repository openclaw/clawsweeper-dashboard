## Summary

Keep open: current main and the latest release still emit root-relative Control UI favicon links while configured basePath routing excludes root /favicon.svg requests. The PR targets a real bug, but the branch needs a small HTML-attribute safety repair before merge.

## What This Changes

The PR adds a gateway index-HTML rewrite helper and tests so Control UI favicon and apple-touch-icon hrefs are prefixed with the configured basePath.

## Best Possible Solution

Port the mounted favicon/apple-touch-icon handling onto current main, escape or reject unsafe basePath text before HTML output, and preserve the existing CSP hash behavior.

## Reproduction Assessment

Yes, source-level. Current main emits root-relative icon hrefs, and basePath routing excludes /favicon.svg from a /webchat mount, so the browser request bypasses the Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index serving time is narrow, but the implementation must escape or validate basePath and retain current CSP hash handling.

## Review Findings
