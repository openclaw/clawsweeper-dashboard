## Summary

Keep open: the PR addresses a real Control UI basePath favicon bug that is still present on current main and latest release, but the proposed helper still injects configurable basePath text into HTML attributes without escaping or validation.

## What This Changes

The PR adds a gateway helper to rewrite Control UI favicon and apple-touch-icon hrefs with `gateway.controlUi.basePath` and adds focused gateway tests.

## Best Possible Solution

Port the favicon/apple-touch-icon rewrite onto current main, escape or reject unsafe basePath text before HTML output, and preserve the existing CSP hash behavior.

## Reproduction Assessment

Yes, source-level. Current main emits root-relative icon hrefs while basePath routing only serves requests under the configured mount, so `/favicon.svg` bypasses a `/webchat` Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index-serving time is narrow, but the implementation must escape or validate basePath and be rebased onto the CSP-aware current gateway code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Escape basePath before injecting favicon hrefs:** `src/gateway/control-ui.ts:244`
  - body: The helper writes `gateway.controlUi.basePath` directly inside double-quoted `href` attributes.
