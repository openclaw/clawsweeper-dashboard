## Summary

Keep open. Current main still leaves configured `gateway.controlUi.basePath` favicon/apple-touch-icon requests outside the mounted Control UI path, and the related `/__openclaw__/` namespace fix plus the closed relative-href PR do not cover this configured-basePath case. The PR is useful and has after-fix proof, but its head is merge-conflicting and needs a current-main rebase or narrow replacement.

## What This Changes

The PR adds a gateway index-HTML helper and tests to prepend the configured Control UI basePath to favicon and apple-touch-icon hrefs.

## Best Possible Solution

Land a current-main-compatible gateway-side favicon/apple-touch-icon href rewrite for configured `gateway.controlUi.basePath`, with focused regression coverage and existing CSP/static-routing behavior preserved.

## Reproduction Assessment

## Solution Assessment

Yes, with rebase care. Serve-time rewriting is a narrow maintainable fix for configured-basePath favicon hrefs, but it must be applied to the current `serveResolvedIndexHtml` path without weakening CSP or static-file safety.
