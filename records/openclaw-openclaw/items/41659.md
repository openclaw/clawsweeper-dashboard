## Summary

Keep open: current main still has the configured-basePath favicon problem, and this PR is directionally useful, but the head still injects configurable basePath text into HTML attributes without escaping or validation.

## What This Changes

The PR adds gateway-side rewriting for Control UI favicon and apple-touch-icon hrefs under a configured basePath, with focused gateway tests.

## Best Possible Solution

Port the favicon/apple-touch-icon basePath handling onto current main with attribute escaping or strict basePath validation, and keep CSP hash handling intact.

## Reproduction Assessment

Yes, source-level. Current main emits root-relative icon hrefs, and basePath routing excludes `/favicon.svg` from a `/webchat` mount, so the browser request bypasses the Control UI handler.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs at index serving time is narrow, but the implementation must escape or reject unsafe basePath text and preserve current CSP hash handling.

## Review Findings
