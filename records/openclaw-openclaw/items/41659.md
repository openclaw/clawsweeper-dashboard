## Summary

Keep open: this is still a valid, narrow Control UI basePath bug. Current main and the latest release still emit root-absolute favicon/apple-touch-icon hrefs and serve index HTML unchanged, while configured `gateway.controlUi.basePath` routes paths outside that prefix away from the Control UI handler; the PR's fix direction is useful but needs current-main integration.

## What This Changes

This PR adds a gateway serve-time rewrite for Control UI favicon and apple-touch-icon hrefs when `gateway.controlUi.basePath` is configured, plus focused gateway tests.

## Best Possible Solution

Land a current-main gateway-side favicon/apple-touch-icon href rewrite for configured Control UI basePath, with CSP-safe integration and focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main still emits root-absolute favicon hrefs, serves index HTML unchanged, and routes `/favicon.svg` outside the Control UI handler when `gateway.controlUi.basePath` is `/webchat`.

## Solution Assessment

Yes, with rebase care. Serve-time rewriting is a narrow maintainable fix for configured-basePath favicon hrefs, but it must be applied to the current `serveResolvedIndexHtml` path without weakening CSP or static-file safety.
