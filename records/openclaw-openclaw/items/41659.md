## Summary

Keep open. The PR addresses a real Control UI basePath favicon bug that is still present on current main and in the latest release, but the branch is conflicting and the proposed helper still writes raw basePath text into an HTML attribute.

## What This Changes

The PR adds a gateway-side helper that rewrites favicon and apple-touch-icon hrefs under a configured Control UI basePath and adds focused rewrite/no-op tests.

## Best Possible Solution

Port a small CSP-aware gateway-side icon href rewrite into current main, escape or validate basePath before emitting HTML, and keep regression coverage for basePath index HTML plus icon asset serving.

## Reproduction Assessment

Yes, source-level. Current main documents gateway.controlUi.basePath, emits root-relative icon hrefs, serves index HTML unchanged, and routes /favicon.svg outside a configured /webchat mount.

## Solution Assessment

Yes, with changes. Rewriting only favicon and apple-touch-icon hrefs for configured basePath is narrow, but the port should preserve current CSP handling and escape or reject unsafe basePath text.
