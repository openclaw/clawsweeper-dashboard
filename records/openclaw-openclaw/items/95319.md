## Summary

Keep this PR open for normal maintainer review: current main and the latest release still use lowercase-only running cron ids, while cron-run session keys use the slugified session-key segment. The PR is a narrow producer-side fix with focused regression coverage and sufficient terminal proof; I found no blocking patch defect.

## What This Changes

The PR changes task maintenance to normalize running cron job ids with the cron-run session-key segment normalizer, updates the session-maintenance contract comment, and adds a regression test for a non-slug cron job id.

## Best Possible Solution

Land the narrow producer-side normalization fix after required CI so session-registry pruning uses one cron-run session-key contract.

## Reproduction Assessment

Yes for source-level reproduction. On current main, a persisted running cron job id `Daily Report` enters the running set as `daily report`, while the session-key parser extracts `daily-report`, so an old live cron-run row is not preserved.

## Solution Assessment

Yes. Reusing `normalizeCronLaneSegment(job.id, "job")` where the running set is produced is the narrowest maintainable fix; changing the parser or adding a fallback would split the session-key contract.
