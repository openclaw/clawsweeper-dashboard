Keep this PR open: current main still has the producer/matcher normalization mismatch, and the branch is a narrow, well-placed fix with focused regression coverage and sufficient terminal proof.

## What This Changes

The PR changes task maintenance to build running cron job ids with the same slugified cron-run session-key segment normalizer used when those session keys are created, and adds a regression test plus a corrected contract comment.

## Best Possible Solution

Land the narrow producer-side normalization fix after normal CI so session-registry pruning uses the existing cron-run session-key contract instead of a second lowercase-only contract.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main lowercases running job ids while cron-run session keys slugify the job segment, so Daily Report becomes daily report in the running set but daily-report in the session key. I did not run tests because this review was read-only.

## Solution Assessment

Yes. Reusing normalizeCronLaneSegment at the producer side is the narrowest maintainable fix; changing the parser or adding another compatibility path would create a second normalization contract.
