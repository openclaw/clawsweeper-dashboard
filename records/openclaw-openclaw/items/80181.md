## Summary

Keep open for maintainer review: current main and the public docs intentionally keep user model selections strict, while this PR introduces a new opt-in public fallback policy and leaves one model docs page contradictory. ## What This Changes

The PR adds an opt-in `userOverrideFallbackPolicy: "resilient"` setting for chat model overrides, with schema/types/help/docs updates and resolver tests.

## Best Possible Solution

If maintainers want resilient user-selected models, land the opt-in policy after resolving current-main conflicts and aligning all model fallback docs; otherwise keep the shipped strict semantics.
