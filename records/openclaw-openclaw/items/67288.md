## Summary

Keep open. Current main still lacks the requested Bedrock Mantle discovery opt-out, and the issue is paired with open fix PR #67341 using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Bedrock Mantle should expose and honor `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled: false` to suppress implicit IAM/model discovery without disabling the plugin.

## Best Possible Solution

Land PR #67341 if it still applies, or replace it with an equivalent focused Mantle-provider fix that adds the schema/UI hint, threads validated plugin config into discovery, short-circuits before IAM token generation and model fetch when disabled, updates docs, and adds regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete steps and logs, and current-main source inspection gives a high-confidence static reproduction path: the config key is rejected, no plugin config reaches the resolver, and IAM token discovery is attempted before returning null.

## Solution Assessment

Yes. A plugin-owned `discovery.enabled === false` gate mirroring `amazon-bedrock` is the narrowest maintainable fix, provided the manifest schema, runtime wiring, docs, and tests are updated together.
