## Summary

Keep open. Current main now supports the explicit Bedrock Mantle `discovery.enabled=false` opt-out, but the default path can still call IAM bearer-token generation and emit the reported diagnostic; open PR https://github.com/openclaw/openclaw/pull/81208 already uses closing syntax for the remaining fix.

## What This Changes

The issue asks Bedrock Mantle to stop unnecessary implicit discovery/IAM-token probing when users are not using Mantle, matching the standard Bedrock discovery gate behavior.

## Best Possible Solution

Review and land, revise, or close https://github.com/openclaw/openclaw/pull/81208; the final state should keep the explicit plugin opt-out and suppress repeated Mantle IAM-unavailable diagnostics without breaking AWS default credential-chain discovery.

Yes. Source inspection gives a high-confidence path: repeated `resolveImplicitMantleProvider` calls without `discovery.enabled=false` or an explicit bearer token still call `generateBearerTokenFromIam`, whose failure path logs the reported diagnostic.
