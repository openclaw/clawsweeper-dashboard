## Summary

## What This Changes

The issue requests a Bedrock Mantle discovery gate so installs without AWS credentials can suppress repeated implicit discovery diagnostics without disabling the whole plugin.

## Best Possible Solution

Review and either land or explicitly close https://github.com/openclaw/openclaw/pull/81208, keeping the shipped explicit disable toggle while settling the default no-credentials diagnostic behavior there.

## Reproduction Assessment

Yes, at source level: repeated calls to `resolveImplicitMantleProvider` with no explicit bearer token and failing IAM generation still reach the diagnostic catch path unless discovery is disabled. I did not run the live Raspberry Pi or Docker scenario in this read-only review.

## Solution Assessment
