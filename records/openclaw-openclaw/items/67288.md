## Summary

Keep open. Current main now supports the explicit Bedrock Mantle `discovery.enabled=false` opt-out, but the default no-credentials path can still reach IAM token generation and the reported diagnostic; an open closing PR already owns that remaining log-spam fix.

## What This Changes

The issue asks for Amazon Bedrock Mantle discovery to be suppressible and to avoid repeated IAM discovery/logging when Mantle is not in use.

## Best Possible Solution

Review and land, revise, or close https://github.com/openclaw/openclaw/pull/81208; the final state should keep the explicit plugin opt-out and suppress repeated Mantle IAM-unavailable diagnostics without disabling AWS default credential-chain discovery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: repeated `resolveImplicitMantleProvider` calls without `discovery.enabled=false` or an explicit bearer token still call `generateBearerTokenFromIam`, whose failure path logs the reported diagnostic.

## Solution Assessment

Yes. A focused Bedrock Mantle-owned fix is the right boundary, and the open PR's log-dedupe direction is safer than a broad no-env skip because AWS default credential-chain auth may work without env markers.
