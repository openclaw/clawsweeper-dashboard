## Summary

Keep this open. Current main and v2026.6.8 include the explicit Bedrock Mantle discovery opt-out, but the default no-credentials path still reaches IAM token generation and the reported diagnostic; the remaining mitigation is already owned by the open linked PR.

## What This Changes

The issue asks Bedrock Mantle to stop per-request IAM discovery/log spam when unused by adding and respecting a discovery disable gate consistent with the standard Bedrock provider.

## Best Possible Solution

Keep the shipped explicit opt-out, resolve the open linked PR by landing or rejecting its IAM-chain-preserving log-spam mitigation, then close this issue.

## Reproduction Assessment

Yes. Source inspection shows the remaining current-main path: with discovery unset and no explicit bearer token, `resolveImplicitMantleProvider` still calls IAM token generation, whose failure path logs the reported diagnostic.

## Solution Assessment

Yes for triage: the provider-local discovery boundary is the right place to solve this. The exact linked PR shape should preserve AWS default credential-chain support while proving the diagnostic is no longer emitted on every failed refresh.
