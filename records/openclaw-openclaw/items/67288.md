## Summary

Keep open. Current main still lacks the Bedrock Mantle discovery opt-out: the manifest rejects `discovery.enabled`, registration does not pass plugin config into discovery, and the resolver still reaches IAM token generation when no bearer token is set; the issue is also paired with open fix PR #67341 using closing syntax.

## What This Changes

Add a Bedrock Mantle plugin config opt-out so implicit Mantle discovery can be disabled without disabling the whole plugin.

## Best Possible Solution

Land #67341 if it still applies, or replace it with an equivalent plugin-owned fix that adds the manifest config surface, gates Mantle discovery before IAM token generation, updates docs/UI hints as appropriate, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the requested config key is not allowed by the Mantle schema, no plugin config reaches the resolver, and the resolver attempts IAM token generation before returning `null`.

## Solution Assessment

Yes. A Mantle-owned `discovery.enabled === false` gate mirroring the standard Bedrock provider is the narrowest maintainable solution, provided schema/docs/tests move together with the runtime gate.
