## Summary

Keep open. Current main still has the reported Bedrock Mantle discovery opt-out gap, and the supplied GitHub context shows open PR #67341 with closing syntax for this exact bug, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests a Bedrock Mantle `config.discovery.enabled` option that suppresses implicit IAM/model discovery without disabling the whole plugin.

## Best Possible Solution

Keep the issue open and resolve it by landing PR #67341 if it still applies, or by replacing it with an equivalent focused Mantle plugin fix that adds the manifest schema/UI hints/docs, passes validated plugin config into discovery, short-circuits before IAM token generation and `/v1/models` when disabled, and adds disabled/default/enabled regression coverage.

## Reproduction Assessment

Yes. The issue provides concrete reproduction steps and logs, and current-main source inspection gives a high-confidence static reproduction path: the requested config key is rejected, registration ignores plugin config, and the resolver attempts IAM token generation without a disable gate.

## Solution Assessment

Yes. A plugin-owned `discovery.enabled === false` short-circuit mirroring `amazon-bedrock` is the narrowest maintainable solution, provided it includes schema, runtime wiring, docs, and tests.
