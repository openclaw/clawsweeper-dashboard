The issue requests a Bedrock Mantle discovery opt-out/noise fix so the plugin does not attempt or log IAM token discovery on every catalog request when Bedrock Mantle is not being used.

## Best Possible Solution

Keep the shipped explicit opt-out, resolve https://github.com/openclaw/openclaw/pull/81208 by landing the accepted log-spam mitigation or closing it as not planned, then close this issue with the final commit/release provenance.
