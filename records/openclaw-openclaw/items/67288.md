## Summary

Keep open. Current main still lacks the Bedrock Mantle discovery opt-out, and open PR #67341 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request adding a Bedrock Mantle `config.discovery.enabled` gate so users can suppress implicit IAM token and model discovery without disabling the plugin.

## Best Possible Solution

Land PR #67341 if it still applies, or replace it with an equivalent focused Mantle fix that adds the schema/UI hint, threads plugin config into discovery, short-circuits before IAM/model discovery when disabled, and adds docs plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the config key is rejected, plugin config is not passed to the resolver, and IAM token discovery is still attempted before returning null.

## Solution Assessment

Yes. A plugin-owned `discovery.enabled === false` gate mirroring `amazon-bedrock` is the narrowest maintainable direction, as long as schema, runtime wiring, docs, and tests are updated together.
