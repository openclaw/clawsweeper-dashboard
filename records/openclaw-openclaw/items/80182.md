## Summary

Keep open: this draft, protected-label PR still removes public Plugin SDK compatibility surfaces that current main exports and documents, and it lacks after-change real behavior proof.

## What This Changes

The PR deprecates many Plugin SDK subpaths, adds SDK surface-report metadata/docs, moves bundled plugin imports to narrower or provider-owned modules, and removes selected provider-owned public SDK exports.

## Best Possible Solution

Keep deprecated compatibility shims and package exports for existing public SDK subpaths and named exports until an explicit major-version SDK removal decision, then add real import/package smoke proof.

## Reproduction Assessment

Yes for the review findings: current main exports, documents, and implements the affected public SDK surfaces, while the PR head removes the package subpath or named exports. An external plugin importing those surfaces would fail module resolution or named-import binding after this patch.

## Solution Assessment

No; deleting public SDK exports is not the safest way to reduce surface under the current compatibility policy. The maintainable path is to keep deprecated shims resolvable, or make the removals an explicit major-version SDK decision.
