## Summary

Keep open. This draft, protected public-SDK cleanup still introduces Plugin SDK compatibility breaks by removing a documented package subpath and public xAI helper exports, and the PR body does not include after-change real behavior proof.

## What This Changes

This PR deprecates and reshapes Plugin SDK subpaths, moves bundled plugin imports to narrower or provider-owned modules, removes selected provider auth/model/stream helpers, and updates SDK surface docs, scripts, tests, and changelog entries.

## Best Possible Solution

Keep bundled plugins off broad or provider-owned SDK seams, but preserve deprecated public package exports and named re-export shims until an explicit major-version SDK removal decision.

## Reproduction Assessment

Yes for the review findings: current main exports, documents, implements, and tests the affected public SDK surfaces, while the PR diff removes the subpath or named exports. An external plugin importing those surfaces would fail module resolution or named-import binding after this patch.

## Solution Assessment

No; deleting public SDK exports is not the safest way to reduce surface under the current compatibility policy. The maintainable path is to keep deprecated shims resolvable, or make the removals part of an explicit major-version SDK decision.
