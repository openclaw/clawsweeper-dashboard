## Summary

Keep open: the reported Ollama Kimi reasoning leak is still plausible on current main and latest release because inline `message.content` is streamed as visible text while only structured `message.thinking`/`message.reasoning` is separated.

## What This Changes

The issue requests an Ollama-provider response sanitizer for Kimi Cloud models so inline reasoning emitted despite thinking-off controls is not delivered to chat.

## Best Possible Solution

Add a narrow Ollama-provider Kimi Cloud response sanitizer after confirming the delimiter variants, keeping unrelated gateway and channel text paths unchanged.

## Reproduction Assessment

No live current-main reproduction was run because it requires an Ollama Cloud Kimi setup. The reporter's screenshot plus source inspection give a high-confidence source reproduction path: inline `message.content` is emitted as visible text unchanged.

## Solution Assessment

Yes. A provider-owned sanitizer in the Ollama stream path is the narrowest maintainable fix; a gateway-wide delimiter split should wait for broader proof.
