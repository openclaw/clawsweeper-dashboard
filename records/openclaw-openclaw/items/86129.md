## Summary

Keep open: current main and v2026.5.22 still separate only structured Ollama `thinking`/`reasoning` fields, while inline `message.content` is streamed and finalized as visible text. The reporter's screenshot and attached local dist hotfix strengthen the report, but the exact delimiter/variant needs maintainer and live-provider confirmation before a safe sanitizer lands.

## What This Changes

The issue requests an Ollama/Kimi response-level sanitizer so reasoning emitted inline despite Think off is not rendered to chat.

## Best Possible Solution

Add a narrow Ollama-provider Kimi Cloud response sanitizer that strips only confirmed inline reasoning before streaming/final chat output, with focused regression coverage and live proof when available.

## Reproduction Assessment

No high-confidence live reproduction was run because it requires an Ollama Cloud Kimi setup, but source inspection gives a clear reproduction shape: any inline reasoning returned in `message.content` is emitted as normal text. The reporter's screenshot and local dist hotfix add useful real-world evidence.

## Solution Assessment

Yes, a provider-owned sanitizer is the best current direction because the failure is specific to Ollama-hosted Kimi output shape. A gateway-wide delimiter stripper should wait for stronger proof because it could corrupt unrelated model output.
