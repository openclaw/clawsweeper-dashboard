Keep open: current main still advertises and selects the Codex OAuth image route, then sends the hosted `image_generation` tool payload that the Windows report and macOS confirmation say the Codex Responses backend rejects.

## What This Changes

The issue asks OpenClaw to either make `openai/gpt-image-2` work through OpenAI Codex OAuth or report clearly when that Codex backend route is unavailable.

## Best Possible Solution

Keep the fix provider-local: use a verified Codex image payload if one exists, otherwise classify this exact hosted-tool rejection as Codex OAuth image generation unavailable and align docs, tests, and changelog.
