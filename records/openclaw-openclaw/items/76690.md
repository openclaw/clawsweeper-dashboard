## Repair Work Prompt

Fix the OpenAI Codex OAuth image-generation failure tracked by https://github.com/openclaw/openclaw/issues/76690. Current main sends `openai/gpt-image-2` through `extensions/openai/image-generation-provider.ts` using the Codex Responses backend with `tools: [{ type: "image_generation", model: "gpt-image-2" }]` and `tool_choice: { type: "image_generation" }`; users on Windows and macOS report HTTP 400 `Tool choice 'image_generation' not found in 'tools' parameter`. First check whether the current Codex backend has a supported image request shape. If not, keep the fix provider-local: detect the exact hosted-tool rejection from the Codex OAuth route, classify it as unavailable for Codex OAuth with a clear message pointing users to configure `OPENAI_API_KEY` or explicit direct OpenAI Images API provider config, and do not silently fall back to an API key when the Codex OAuth route was selected. Update focused regression coverage in `extensions/openai/image-generation-provider.test.ts` and runtime coverage only if the surfaced error/attempt metadata changes. Align `docs/providers/openai.md`, `docs/tools/image-generation.md`, and add a single-line user-facing changelog entry. Do not add a new config option, do not move provider-owned behavior into core, and stop for maintainer review if the solution requires a new core plugin API or a product decision about removing Codex OAuth image support entirely.
