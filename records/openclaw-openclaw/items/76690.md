Fix the OpenAI Codex OAuth image-generation failure tracked by https://github.com/openclaw/openclaw/issues/76690. Current main documents `openai/gpt-image-2` as working with Codex OAuth, but the provider sends a Codex Responses hosted `image_generation` tool request that reports `Tool choice 'image_generation' not found in 'tools' parameter`. Work in the OpenAI plugin boundary. First check whether a supported Codex Responses image request shape exists; if it does, update the provider and tests to use it.
