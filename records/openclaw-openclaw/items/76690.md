## Summary

Keep open: current main and the latest release still document Codex OAuth image generation as supported and still send the hosted `image_generation` tool request shape that both reporters say the Codex Responses backend rejects.

## What This Changes

The issue asks OpenClaw to make `openai/gpt-image-2` work via OpenAI Codex OAuth or report that Codex OAuth image generation is unavailable instead of surfacing the raw hosted-tool 400.

## Best Possible Solution

Keep the direct OpenAI Images API path intact, and either switch the Codex OAuth route to a live-supported request shape or classify the exact hosted-tool rejection as an unavailable Codex image capability with aligned docs and regression tests.
