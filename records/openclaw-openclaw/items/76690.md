In `extensions/openai/image-generation-provider.ts`, either implement a live-verified supported Codex image request shape, or more conservatively classify this exact Codex 400 as image generation unavailable and surface a clear error directing users to the `OPENAI_API_KEY` Images API route. Do not add a new config option, silently fall back between OAuth and API-key credentials, or make core runtime depend on OpenAI-specific error strings beyond a provider-local classification.
