## Repair Work Prompt

Fix the OpenAI Codex OAuth image-generation failure reported in https://github.com/openclaw/openclaw/issues/76690. Current main still routes `openai/gpt-image-2` with only an `openai-codex` OAuth profile to `https://chatgpt.com/backend-api/codex/responses` using a hosted `image_generation` tool and `tool_choice: { type: "image_generation" }`, while reporters on Windows and macOS see HTTP 400 `Tool choice 'image_generation' not found in 'tools' parameter`. Update `extensions/openai/image-generation-provider.test.ts`, `docs/providers/openai.md`, `docs/tools/image-generation.md`, and `CHANGELOG.md` as needed. Do not move provider-specific policy into core, do not remove direct OpenAI API-key/Azure/custom endpoint image routing, and do not broaden the patch into the separate `gpt-5.5-pro` fallback issue unless the same provider-local classifier naturally covers the exact image route.
