Keep open: current main still advertises and selects the Codex OAuth image route, then sends the hosted `image_generation` tool payload that the Windows and macOS reports say the Codex Responses backend rejects.

## What This Changes

The issue asks OpenClaw to make `openai/gpt-image-2` image generation succeed through Codex OAuth or report that this Codex route is unavailable.

## Best Possible Solution

Keep the fix provider-local: use a live-supported Codex image request shape if one exists; otherwise classify this hosted-tool rejection as Codex OAuth image generation unavailable and align docs, tests, and changelog.
