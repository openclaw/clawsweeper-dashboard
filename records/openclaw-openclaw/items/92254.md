## Summary

Keep open: this is a useful fix candidate for the linked auth-profile loss bug, but the patch currently preserves auth overrides unconditionally and can carry a stale provider-specific profile across a cross-provider picker switch.

## What This Changes

The PR updates Telegram and Discord model picker persistence to retain existing auth profile overrides during model-only selections and adds Discord regression tests for fallback and hidden-dispatch paths.

## Best Possible Solution

Apply the existing provider-aware auth-profile preservation rule through an SDK-safe helper or equivalent picker-local predicate on both Telegram and Discord, preserving compatible overrides without changing manual `/model` directive semantics.

## Reproduction Assessment

Yes. Source inspection shows current main's Telegram picker and Discord picker fallback call `applyModelOverrideToSessionEntry` without a profile or preserve flag, and the helper then clears existing auth override fields.

## Solution Assessment

No. The PR fixes the destructive delete directionally, but the best fix should preserve only provider-compatible auth overrides, matching the existing gateway session patch behavior.
