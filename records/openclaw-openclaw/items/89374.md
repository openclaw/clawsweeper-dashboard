## Summary

Keep open: current main and the latest release fix narrower Codex compaction-skip and native-thread pressure paths, but the core timeout-compaction loop can still treat `compacted: true` as success, retry the same channel session, and eventually fall through after repeated timeouts without proving safe headroom or rolling the stuck session over.

## What This Changes

The issue asks OpenClaw to recover Codex-backed channel sessions when timeout-triggered compaction reports success but leaves the session high-context and unrecoverable.

## Best Possible Solution

Keep this as the canonical timeout-compaction recovery tracker, define the maintainer-approved safe reset or rollover policy for unrecoverable Codex channel sessions, and prove recovery on a high-context live or packaged Codex channel flow.
