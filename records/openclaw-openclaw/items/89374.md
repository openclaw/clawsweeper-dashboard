## Summary

Keep open: current main and the latest release fix the narrower Codex automatic-compaction skip path, but the timeout-compaction retry path still accepts successful compactions and can fall through after repeated timeouts without proving the affected channel session has safe headroom or a safe rollover/reset policy.

## What This Changes

The issue asks OpenClaw to recover, reset, or roll over a Codex-backed channel session when repeated timeout-triggered compactions leave the same session stuck near the context window.

## Best Possible Solution

Define a safe automatic reset or rollover policy for unrecoverable Codex channel sessions after repeated timeout compactions, then prove it with focused runner tests and a high-context live channel flow.

## Reproduction Assessment

No live current-main repro was run here, but current source and tests preserve the repeated success/retry/fallthrough path and the issue discussion provides release-era channel logs.

## Solution Assessment

Current main is only a partial solution; the best fix is a maintainer-approved session recovery policy, not another local compaction ownership shim.
