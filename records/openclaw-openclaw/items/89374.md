## Summary

Keep open: current main fixes narrower Codex compaction-skip and native-thread pressure cases, but the timeout-compaction loop can still report a successful compaction, retry, and then fall through without proving the affected channel session recovered or safely rolling it over.

## What This Changes

The issue asks OpenClaw to stop reporting successful timeout compaction while reusing an unrecoverable Codex-backed channel session, and to either prove recovery headroom, report unrecoverable state, or roll over the affected session.

## Best Possible Solution

Keep this as the canonical timeout-compaction recovery tracker, define the safe rollover/reset policy for unrecoverable Codex channel sessions, then prove a high-context Codex channel session recovers without repeated timeout-compaction loops.

## Reproduction Assessment

No full live current-main repro was run, but source and tests still show the repeated timeout-compaction fallthrough path, and the issue has multiple release-era field reports with sanitized logs.

## Solution Assessment

Current main is not the complete solution: it fixes the fatal skip and stale native binding pieces, but it still lacks a maintainer-approved policy for safe session rollover/reset after repeated timeout compactions.
