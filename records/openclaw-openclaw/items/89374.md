## Summary

Keep open: current main fixes the release-era fatal Codex automatic-compaction skip and adds native-thread pressure rotation, but it still does not prove timeout compaction reached safe headroom or automatically roll over the affected channel session after repeated timeout compaction. The remaining behavior is session-recovery policy, not conservative cleanup.

## What This Changes

The issue asks OpenClaw to recover or clearly fail Codex-backed channel sessions when timeout compaction reports success but the session remains near the context limit and continues timing out.

## Best Possible Solution

Define one canonical recovery policy for repeated Codex timeout compactions: verify safe headroom after compaction, and when recovery cannot make the session safe, either surface explicit reset guidance or rotate only the affected channel session with live regression proof.

## Reproduction Assessment

No, not as a high-confidence live current-main repro. Source inspection shows the remaining path: timeout recovery treats `compacted: true` as enough to retry, caps attempts, and lacks a proven headroom or channel-session rollover policy.

## Solution Assessment

No, closing as implemented is not the best outcome. Current main fixed important subpaths, but the maintainable solution still needs an explicit recovery policy and live proof across the affected Codex channel-session surfaces.
