## What This Changes

The PR stops `sessions.create` from copying a parent session’s cached `contextTokens` into new dashboard child sessions and updates the session creation regression test to keep runtime selection while dropping stale context metadata.

## Best Possible Solution

Land the narrow metadata-inheritance fix after protected-label maintainer review and a redacted live or diagnostic after-fix proof that a new dashboard child session derives the current model budget while preserving runtime selection.

## Reproduction Assessment

I did not execute a live Control UI reproduction in this read-only review.

## Solution Assessment

Yes for the code shape: the create helper is the narrow boundary that should preserve runtime selection while not carrying cached budget metadata into a fresh child session. The safer pre-merge addition is live or diagnostic after-fix proof, not a broader code change.
