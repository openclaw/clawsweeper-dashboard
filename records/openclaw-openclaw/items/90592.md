## What This Changes

The PR removes the Synology Chat webhook handler's local `Promise.race` delivery timeout and adds a regression test proving authorized delivery no longer schedules a 120000ms timer.

## Best Possible Solution

Land the narrow timeout-removal patch after rerunning or accepting the failed cron check, keeping agent-turn timeout policy in the shared runtime instead of adding a Synology-specific response-timeout knob.

## Reproduction Assessment

Yes. Current main and `v2026.6.8` show the exact `setTimeout(..., 120_000)` `Promise.race` in the authorized Synology webhook path, and the PR body includes before/after terminal output for that path.

## Solution Assessment

Yes. Removing the Synology-only local race is the narrowest maintainable fix because the shared agent/runtime path already owns run timeout policy; adding a Synology `responseTimeout` would create a competing source of truth.
