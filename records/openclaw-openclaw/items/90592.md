## Summary

Keep open. Current main and the latest release still contain the Synology Chat 120s local delivery timeout, while this PR removes only that wrapper and adds focused regression coverage; I found no patch defect, but merge still needs maintainer and CI gate handling.

## What This Changes

The PR removes Synology Chat webhook handler's hardcoded 120s `Promise.race` around delivery and adds a regression test asserting authorized delivery does not schedule that timeout.

## Best Possible Solution

Land the narrow timeout-removal patch after required maintainer and CI gates, leaving agent timeout policy in the shared runtime rather than adding a Synology-specific timeout knob.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.6.8` both show every authorized Synology webhook delivery racing shared dispatch against `setTimeout(..., 120_000)`, and the linked reports/proof align with that code path; I did not run a live NAS reproduction.

## Solution Assessment

Yes. Removing the Synology-specific `Promise.race` is the best narrow fix because the shared channel lifecycle explicitly avoids run timeouts and the agent runtime already owns `agents.defaults.timeoutSeconds`.
