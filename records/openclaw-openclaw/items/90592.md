## What This Changes

This PR removes Synology Chat webhook handler's hardcoded 120s `Promise.race` around `deliver` and adds a regression test asserting authorized delivery does not schedule that timeout.

## Best Possible Solution

Land the narrow timeout-removal patch after maintainer and CI gates so Synology Chat relies on the shared agent timeout policy.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` still show `Promise.race` with `setTimeout(..., 120_000)` in the Synology webhook handler, and the PR body includes before/after terminal probe output for that exact handler path.

## Solution Assessment

Yes. Removing the Synology-specific local race is the narrowest maintainable fix because the shared channel/core dispatch path already owns agent timeout policy through `resolveAgentTimeoutMs`.
