## What This Changes

The branch removes the Synology Chat webhook handler's hardcoded local 120s `Promise.race` around `deliver` and adds a regression test for the authorized webhook path.

## Best Possible Solution

Land this narrow timeout-removal fix once current-head CI and normal merge gates pass, keeping agent/runtime timeout policy as the single owner of long Synology Chat turns.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 race authorized Synology webhook delivery against `setTimeout(..., 120_000)`, and the PR body supplies before/after terminal probe output for the same handler path.

## Solution Assessment

Yes. Removing the Synology-specific local race is the narrowest maintainable fix because shared channel turn dispatch already owns delivery and documented agent/runtime timeouts own long-running turns; adding a Synology-only timeout knob would create a competing policy.
