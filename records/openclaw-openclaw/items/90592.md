## Best Possible Solution

Land the narrow timeout-removal patch after maintainer and CI gates so Synology Chat relies on the shared agent timeout policy.

## Solution Assessment

Yes. Removing the Synology-only local race is the narrowest maintainable fix because the shared agent runtime already owns run timeout policy; adding a Synology responseTimeout knob would create a competing timeout source.
