## What This Changes

The PR removes the Synology Chat webhook handler's local 120s `Promise.race` around `params.deliver` and adds a regression test asserting authorized webhook delivery no longer schedules a 120000ms timer.

## Best Possible Solution

Land the narrow timeout-removal patch after normal maintainer and CI gates so Synology Chat relies on the shared agent runtime timeout policy.

## Reproduction Assessment

Yes. Current main and v2026.6.8 show the exact `setTimeout(..., 120_000)` `Promise.race` in the authorized Synology webhook path, and the PR body includes before/after terminal output for that path.

## Solution Assessment

Yes. Removing the Synology-only local race is the narrowest maintainable fix because shared agent runtime code already owns configured run deadlines; adding or honoring a Synology-specific response timeout would add a competing config surface.
