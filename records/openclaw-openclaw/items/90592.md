## Summary

Keep this PR open: current main and v2026.6.9 still contain the Synology Chat hardcoded 120s local delivery timeout, and this branch is a narrow, source-backed fix with focused proof and no blocking code/security findings found.

## What This Changes

The PR removes Synology Chat's local `Promise.race` 120s delivery timeout and adds a regression test asserting authorized webhook delivery does not schedule a 120000ms timer.

## Best Possible Solution

Land this narrow timeout removal once current merge gates are green or maintainer-accepted, keeping agent-turn timeout ownership in shared session/runtime policy instead of a Synology-specific wrapper or knob.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.9 still race authorized Synology webhook delivery against `setTimeout(..., 120_000)`, and the related production logs match that 120s boundary; I did not run a live NAS reproduction.

## Solution Assessment

Yes. Removing the Synology-local agent-turn timeout is the cleanest fix because the documented channel contract leaves long-running agent work to shared session/runtime lifecycle after routing.
