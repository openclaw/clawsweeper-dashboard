## Summary

Current main still lacks node-advertised model tool definitions, and this issue is the canonical thread for the missing protocol/runtime capability. Existing plugins can register agent tools and node-host commands, but they do not remove the wrapper-plugin requirement; the requested design also needs explicit product and security decisions because connected nodes would influence the executable model-visible tool surface.

## What This Changes

Allow connected nodes to advertise schemas that become first-class agent tools without a wrapper plugin.

## Best Possible Solution

Define an additive, versioned node command descriptor that is validated by the Gateway and used to synthesize node-provenance agent tools and inventory entries, while execution continues through existing node invocation authorization and operator-controlled tool policy.

## Reproduction Assessment

Not applicable. This is a new capability request, and source inspection establishes the gap: nodes advertise command names but cannot advertise model tool schemas.

## Solution Assessment

Yes at the architectural level: a generic node protocol extension is cleaner than per-integration wrapper plugins. The exact descriptor, lifecycle, inventory, and authorization contract still require maintainer product and security approval before implementation.

## Risks / Open Questions

- Allowing connected nodes to define model-visible tools could expose commands beyond operator intent unless pairing trust, command allowlists, tool allow/deny policy, schema validation, node provenance, naming conflicts, reconnect idempotency, and disconnect cleanup remain Gateway-controlled.
- A parallel node-specific tool contract could drift from existing plugin tool metadata and inventory unless maintainers choose one generic descriptor and explicitly define how it appears in `tools.catalog` and `tools.effective`.
- Dynamic tool availability can alter prompt/tool inventories during a session, so maintainers must decide whether changes apply only to new runs, require stable snapshots, or support an explicit refresh lifecycle.
