## Summary

Keep open. Current main still has node-advertised caps/commands and plugin-registered tools, but it does not let connected nodes advertise first-class LLM tool definitions or synthesize node-backed agent tools; the existing openclaw-tescmd workaround and the closed command-schema duplicate are useful context for the same remaining core protocol gap.

## What This Changes

Allow connected nodes to advertise agent tool definitions that the Gateway exposes to the LLM and routes through node.invoke without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, store validated tool definitions with NodeRegistry sessions, expose node provenance in tool catalog/effective inventory, and route executions through existing node.invoke policy before documenting naming, reconnect, pairing, and allow/deny behavior.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main only carries node command names plus unknown invoke payloads and a static nodes agent tool.

## Solution Assessment

Yes. Keeping this open for a maintainer-approved protocol/runtime design is better than a wrapper plugin or ClawHub-only answer because the missing piece is a core node advertisement and tool synthesis contract.
