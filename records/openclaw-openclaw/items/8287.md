## Summary

Keep open. Current main still accepts node caps and command names, then exposes one static `nodes` tool plus plugin-registered tools; it does not let connected nodes advertise first-class LLM tool definitions or lifecycle-managed node-backed tools.

## What This Changes

Request to let connected nodes register agent tool definitions that the Gateway exposes directly to LLMs without requiring a wrapper plugin.

## Best Possible Solution

Add an additive Gateway protocol and runtime contract for validated node-advertised tools, backed by NodeRegistry lifecycle state, synthesized node tools, inventory provenance, existing node.invoke authorization, and documented policy behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent: connect accepts caps/commands but not tool definitions, and agent tools come from static core tools plus plugins.

## Solution Assessment

Unclear until maintainer design. The requested direction addresses a real core protocol/runtime gap, but the maintainable solution should be an additive, validated node tool contract with policy and provenance semantics rather than a pass-through wrapper around `node.invoke`.
