## Summary

Keep open. Current main still supports node-declared capabilities/commands and plugin tools that can invoke nodes, but it does not provide node-advertised LLM tool definitions, dynamic node-backed tool synthesis, or node provenance in tool inventory; the openclaw-tescmd workaround is useful evidence of the gap, not a reason to close it as ClawHub-only.

## What This Changes

Allow connected nodes to advertise LLM tool definitions that the Gateway dynamically exposes and routes through node.invoke.

## Best Possible Solution

Add an additive Gateway protocol extension for node-advertised tool definitions, validate/store them per NodeRegistry session, synthesize node-backed tools and inventory entries with node provenance, route execution through node.invoke policy gates, and document naming, reconnect, pairing, and allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-confirmed on current main: connect accepts node caps/commands but not tool definitions, and agent tool assembly exposes static core tools plus plugin tools only.

## Solution Assessment

Unclear until maintainer design. The requested direction addresses a real core protocol/runtime gap, but the maintainable path should be an additive contract with policy/provenance semantics rather than a one-off wrapper around existing node.invoke.
