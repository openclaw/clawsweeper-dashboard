## Summary

Keep open. Current main still exposes node capabilities as declared commands plus one static `nodes` tool, while plugin-registered tools remain the only first-class custom LLM tool path; the openclaw-tescmd workaround and prior maintainer comments are useful context for a real core protocol/runtime gap, not a close reason.

## What This Changes

Allow connected nodes to advertise LLM tool definitions that the Gateway registers, routes through `node.invoke`, and unregisters on disconnect.

## Best Possible Solution

Add an additive Gateway protocol and runtime contract for validated node-advertised tools, with NodeRegistry lifecycle state, synthesized node-backed agent tools, inventory provenance, `node.invoke` policy enforcement, and docs for naming, reconnect, pairing, and allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-confirmed on current main: connect accepts node caps/commands but not tool definitions, and agent tool assembly exposes static core tools plus plugin tools only.

## Solution Assessment

Unclear until maintainer design. The requested direction addresses a real core protocol/runtime gap, but the maintainable solution should be an additive, validated contract with policy and provenance semantics rather than a one-off wrapper around existing `node.invoke`.
