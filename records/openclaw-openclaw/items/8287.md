## Summary

Keep open. Current main still has node-advertised caps/commands, the static `nodes` agent tool, and plugin-registered tools, but it does not have a node-advertised first-class LLM tool contract or lifecycle integration.

## What This Changes

Allow connected nodes to advertise first-class LLM tool descriptors that the Gateway registers and routes through `node.invoke` without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway protocol contract for validated node tool advertisements, persist them per live node, expose node provenance in catalog/effective inventories, and execute through existing `node.invoke` authorization and command policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review on current main gives high confidence that node caps/commands and `node.invoke` exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive Gateway protocol/runtime design with inventory, provenance, lifecycle, and policy integration rather than more wrapper-plugin indirection.
