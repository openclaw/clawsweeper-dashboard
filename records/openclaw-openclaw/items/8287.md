## Summary

Keep open. Current main still exposes nodes as capability and command-name hosts, not as first-class node-advertised LLM tools, and prior review context explicitly kept this as a Gateway protocol/runtime design item with security implications.

## What This Changes

The request asks OpenClaw to let connected nodes advertise agent tool definitions that the Gateway registers dynamically and routes through node.invoke without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, and execution through existing node.invoke authorization and command policy.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main only carries node command names and exposes one static nodes agent tool.

## Solution Assessment

Yes as a direction. A core additive protocol/runtime design is the maintainable path because a wrapper plugin or ClawHub-only answer cannot provide the missing node advertisement, lifecycle, and provenance contract.
