## Summary

Keep open. Current main still supports node-declared command names and a static `nodes` tool, but it does not let connected nodes advertise first-class LLM tool definitions or synthesize node-backed agent tools; the remaining work is a security-sensitive Gateway protocol/product design.

## What This Changes

Request connected nodes to register first-class agent tools that the Gateway exposes to LLMs without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, store validated tool definitions with connected node sessions, expose node provenance in catalog/effective tool inventory, and route execution through existing `node.invoke` authorization and command policy with documented naming and allow/deny behavior.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main confirms the existing implementation only carries node command names and exposes one static `nodes` agent tool.

## Solution Assessment

Yes. Keeping this open for maintainer-approved protocol/runtime design is the narrowest maintainable path; a wrapper plugin or ClawHub-only answer cannot provide the missing core node advertisement, lifecycle, and provenance contract.
