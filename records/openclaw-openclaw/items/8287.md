## Summary

Keep open. Current main still exposes node command names through one static `nodes` agent tool, but it does not provide node-advertised LLM tool descriptors, node provenance in tool inventory, or the policy/lifecycle semantics this request needs; the narrower schema request at https://github.com/openclaw/openclaw/issues/33838 was closed as duplicate of this broader design thread.

## What This Changes

Request to let connected nodes advertise first-class agent tools that the Gateway exposes directly to the LLM and routes through `node.invoke`.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing `node.invoke` authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main accepts node command names and exposes one static `nodes` agent tool rather than node-advertised tool definitions.

## Solution Assessment

Yes directionally, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, policy, and security design rather than another wrapper plugin.
