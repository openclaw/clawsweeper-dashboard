## Summary

Keep open. Current main still lacks node-advertised tool schemas, dynamic node-backed agent tool synthesis, and node provenance in catalog/effective inventory, so this remains real Gateway protocol/runtime design work rather than stale cleanup or ClawHub-only work.

## What This Changes

Request to let connected nodes advertise model-visible agent tool definitions that the Gateway registers and routes directly through node invocation without wrapper plugins.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing node.invoke authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature request, and source inspection confirms current main only accepts node command names and exposes a static nodes agent tool.

## Solution Assessment

Yes as a direction, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, config-policy, and security design rather than another wrapper plugin.
