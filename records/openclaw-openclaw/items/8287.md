## Summary

Keep open. Current main still exposes node capabilities and command names plus one static `nodes` tool; it does not support node-advertised first-class LLM tools, and the remaining work changes Gateway protocol, lifecycle, inventory, and authorization policy.

## What This Changes

Allow connected nodes to advertise agent tool definitions that the Gateway exposes to the LLM and routes through `node.invoke` without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, and execution through existing `node.invoke` authorization and command policy.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main confirms the existing implementation only carries node command names and exposes one static `nodes` agent tool.

## Solution Assessment

Yes as a direction. A core additive protocol/runtime design is the maintainable path; a wrapper plugin or ClawHub-only answer cannot provide the missing node advertisement, lifecycle, and provenance contract.
