## Summary

Keep open. Current main still has node-declared command names and plugin-registered tools, but not connected-node advertisement of first-class LLM tool definitions or node-backed dynamic tool lifecycle.

## What This Changes

Request to let connected nodes advertise LLM-facing tools that the Gateway registers, routes through node.invoke, and removes when nodes disconnect.

## Best Possible Solution

Design and implement an additive Gateway node tool-advertisement contract, then wire validated node-backed tools through NodeRegistry, tool inventory, policy, and node.invoke execution with docs and tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main has command-name node declarations plus static/plugin tools rather than node-advertised LLM tool definitions.

## Solution Assessment

Unclear as a concrete implementation because no patch is proposed. The core protocol/runtime direction is the right place for the requested zero-plugin node tool lifecycle, but it needs maintainer design before a safe implementation.
