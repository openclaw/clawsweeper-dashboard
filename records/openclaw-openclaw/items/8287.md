## Summary

Keep open: current main and the latest release still lack node-advertised model-visible tools, and the open implementation candidate has not merged. The issue remains the canonical product/security thread for the Gateway protocol, node provenance, and tool-policy design.

## What This Changes

The issue requests a Gateway/node protocol feature that lets connected nodes advertise first-class LLM agent tools, with direct routing through node invocation instead of wrapper plugins.

## Best Possible Solution

Keep this issue as the canonical product/security design thread until https://github.com/openclaw/openclaw/pull/90431 lands or maintainers choose a different additive Gateway protocol and node tool policy shape.

## Reproduction Assessment

Yes, at source level: current main has no node tool-advertisement schema, no NodeRegistry storage for node-advertised tool definitions, and no node-backed tool source in agent tool assembly. This is a missing feature rather than a failing runtime bug reproduction.

## Solution Assessment

Not yet. The requested core gap is real, but the best implementation needs maintainer-approved protocol, provenance, and trust semantics, with https://github.com/openclaw/openclaw/pull/90431 as the current unmerged candidate.
