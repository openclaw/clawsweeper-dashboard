## Summary

Keep open. Current main still carries node capabilities and command names plus one static `nodes` agent tool, but it does not accept node-advertised first-class LLM tool definitions or expose node-backed tool provenance; the remaining work is a security-sensitive Gateway protocol and product-design change.

## What This Changes

Request to let connected nodes advertise first-class LLM tool definitions that the Gateway exposes dynamically and routes through `node.invoke` without wrapper plugins.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, and execution through existing `node.invoke` authorization and command policy.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main confirms the existing implementation only carries node command names and exposes one static `nodes` agent tool.

## Solution Assessment

Yes. Keeping this open for maintainer-approved protocol/runtime design is the narrowest maintainable path; a wrapper plugin or ClawHub-only answer cannot provide the missing core node advertisement, lifecycle, and provenance contract.
