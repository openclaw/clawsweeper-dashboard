## Summary

Keep open: current main and v2026.6.8 still support node-declared caps/commands plus plugin-owned node-host commands, but they do not let a connected node advertise first-class LLM tool schemas or synthesize node-backed tools. The related narrower command-schema issue was already closed as a duplicate of this issue, so this remains the canonical design thread.

## What This Changes

Request to let connected nodes register LLM-visible agent tools with schemas and direct `node.invoke` routing, without requiring wrapper plugins.

## Best Possible Solution

Keep this as the maintainer-owned Gateway protocol/runtime design thread for additive node tool advertisement, node-provenance inventory, and policy-gated direct execution through existing node authorization.

## Reproduction Assessment

Yes at source level for the missing capability: current connect schema, NodeRegistry, tool assembly, and inventory schemas have no node-advertised tool definition path. This is a feature gap rather than a broken existing runtime behavior.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The requested direction is plausible and not covered by the plugin workaround, but the exact schema, provenance, naming, and policy model should be decided before implementation.
