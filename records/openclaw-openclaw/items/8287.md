## Summary

Keep open: current main and v2026.6.6 still expose node capabilities and command names, plus plugin/MCP tools, but not node-advertised first-class LLM tool definitions, node-owned tool inventory provenance, or a Gateway lifecycle/policy contract for dynamic node-backed tools.

## What This Changes

Allow connected nodes to advertise first-class agent tool definitions that the Gateway validates, exposes to the LLM, routes through `node.invoke`, and removes on disconnect.

## Best Possible Solution

Define an additive, maintainer-approved Gateway node tool/schema advertisement contract with strict validation, node provenance, command-policy enforcement, deterministic lifecycle, catalog/effective inventory support, docs, and focused tests.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 accept node command names but not node-owned tool definitions or schemas, and runtime exposes only the generic `nodes` tool. This is a missing capability rather than a broken existing behavior with a failing runtime repro.

## Solution Assessment

Unclear; first-class dynamic node tools may be the right UX, but maintainers should compare that against a narrower Gateway-owned command-schema contract projected through the existing `nodes` abstraction before expanding the model tool namespace.
