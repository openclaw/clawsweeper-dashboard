## Summary

Keep open: current main and the latest release still model nodes as caps plus command names, expose one static `nodes` agent tool, and only catalog core/plugin/channel/MCP tool sources. This remains a core Gateway protocol, runtime, and security-policy design request rather than a ClawHub-only plugin request.

## What This Changes

Allow connected nodes to advertise LLM-facing tool definitions that the Gateway registers dynamically and routes through node invocation without a wrapper plugin.

## Best Possible Solution

Keep this as the canonical maintainer-approved additive Gateway node tool-advertisement design covering schema validation, dynamic registration/unregistration, node provenance, node.invoke routing, policy, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still lacks node-advertised tool schemas, node-backed synthetic tools, and node provenance in tool inventories.

## Solution Assessment

Yes, keeping this open is the best current path. The maintainable solution is an additive core Gateway protocol/runtime design, not a ClawHub-only workaround, because current plugins can register tools but connected nodes cannot advertise LLM tool schemas directly.
