## Summary

Keep open. Current main supports node-declared command names, `node.invoke`, and plugin-owned tools/policies, but it still has no Gateway protocol or runtime path for connected nodes to advertise first-class LLM tool definitions with lifecycle registration, inventory provenance, and tool policy integration.

## What This Changes

Request an additive Gateway/node capability where connected nodes advertise LLM-facing tool descriptors that are registered while the node is connected and routed through `node.invoke`.

## Best Possible Solution

Design an additive node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, and execute through existing `node.invoke` authorization and command policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review against current main gives high confidence that node caps/commands and `node.invoke` exist, while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive Gateway protocol/runtime design with inventory, provenance, and policy integration rather than more wrapper-plugin indirection.
