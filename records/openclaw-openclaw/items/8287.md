## Summary

Keep open. Current main still supports connect-time node capabilities/command names and plugin-registered tools, but it does not provide a node-advertised first-class LLM tool contract, per-node tool lifecycle, dynamic node-backed tool synthesis, or node provenance in tool inventories; the prior discussion also treats this issue as the canonical thread rather than a ClawHub-only plugin request.

## What This Changes

Request connected nodes to advertise tool descriptors that the Gateway exposes to the LLM and routes directly through node.invoke/system.run.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, route execution through node.invoke and existing command policy, and document lifecycle and allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that node caps/commands and node.invoke exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is core Gateway protocol/runtime work with inventory, provenance, lifecycle, and policy integration; closing as ClawHub would miss the missing core extension point.
