## Summary

Keep open. Current main still has connect-time node capabilities and command names plus plugin-registered agent tools, but it does not have a node-advertised first-class LLM tool contract, per-node tool lifecycle, node-backed tool synthesis, or node provenance in tool inventories.

## What This Changes

The issue requests a Gateway/node protocol feature that lets connected nodes advertise agent tool descriptors and have those tools routed directly through node invocation.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, route execution through node.invoke and existing command policy, and document lifecycle and allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that node caps/commands and node.invoke exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive Gateway protocol/runtime design with inventory, provenance, lifecycle, and policy integration rather than more wrapper-plugin indirection.
