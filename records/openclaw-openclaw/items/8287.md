## Summary

Keep open. Current main still only accepts node caps and command names and exposes one static `nodes` tool; the requested node-advertised tool schema, dynamic lifecycle, inventory provenance, and policy semantics remain missing and need maintainer/security design.

## What This Changes

Request to let connected nodes advertise first-class agent tools that the Gateway exposes to LLMs and routes through `node.invoke` without a wrapper plugin.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing `node.invoke` authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main accepts node command names and exposes one static `nodes` agent tool rather than node-advertised tool definitions.

## Solution Assessment

Yes as a product direction, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, policy, and security design rather than another wrapper plugin.
