## Summary

Keep open. Current main still has node capabilities/command names plus plugin-registered tools, but no protocol, registry, tool assembly, or inventory support for node-advertised first-class LLM tools; this is a valid feature that needs maintainer protocol and security design.

## What This Changes

Request to let connected nodes advertise tool descriptors that the Gateway exposes as first-class LLM tools and routes through node.invoke.

## Best Possible Solution

Add an additive Gateway protocol contract for validated node tool advertisements, persist them per live node, expose node provenance in catalog/effective inventories, execute through existing node.invoke authorization and command policy, and document naming, lifecycle, pairing, and tool-policy behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review on current main gives high confidence that node caps/commands and node.invoke exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive Gateway protocol/runtime design with inventory, provenance, lifecycle, and policy integration rather than more wrapper-plugin indirection.
