## Summary

Keep open. Current main and the latest shipped release still only model node capabilities and command names, while this request needs a new Gateway protocol/runtime contract for node-advertised tool schemas, lifecycle, execution routing, and inventory provenance.

## What This Changes

Request to let connected nodes advertise first-class agent tool definitions that the Gateway exposes to the LLM and routes through node.invoke.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing node.invoke authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main only accepts node command names and exposes a static nodes agent tool, so the requested capability is absent rather than broken existing behavior.

## Solution Assessment

Yes as a product direction, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, and security policy design rather than another wrapper plugin.
