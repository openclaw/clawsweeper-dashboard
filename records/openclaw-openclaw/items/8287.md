## Summary

Keep open. Current main still supports node capabilities, command names, and plugin-registered tools, but it does not let connected nodes advertise first-class model-visible tool definitions or synthesize node-backed agent tools; the live discussion also keeps this as the canonical design thread.

## What This Changes

Request additive Gateway/node protocol support for connected nodes to advertise agent tool definitions that route directly through node invocation without a wrapper plugin.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing node.invoke authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main accepts node command names and exposes one static nodes agent tool rather than node-advertised tool definitions.

## Solution Assessment

Yes as a direction, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, policy, and security design rather than another wrapper plugin.
