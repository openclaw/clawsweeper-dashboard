## Summary

Keep open. Current main still models nodes as capability and command-name hosts, while first-class agent tools and tool inventory come from core/plugin/channel paths; this remains a real Gateway protocol/runtime feature with product and security policy implications.

## What This Changes

Request to let connected nodes advertise validated LLM-facing tool definitions that the Gateway registers dynamically and routes through node.invoke.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, and execution through existing node.invoke authorization and command policy.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main only carries node command names and exposes one static nodes agent tool.

## Solution Assessment

Yes as a direction. A core additive protocol/runtime design is the maintainable path; a wrapper plugin or ClawHub-only answer cannot provide the missing node advertisement, lifecycle, and provenance contract.
