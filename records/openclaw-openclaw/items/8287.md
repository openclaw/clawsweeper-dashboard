## Summary

Keep open. Current main still models nodes as connected command surfaces and exposes the static `nodes` tool plus plugin tools, but it does not support node-advertised first-class LLM tool descriptors, lifecycle registration, inventory provenance, or node-backed tool policy integration.

## What This Changes

Request to let connected nodes advertise agent tool definitions that the Gateway exposes directly to the LLM and routes through `node.invoke`.

## Best Possible Solution

Add an additive Gateway protocol contract for validated node tool advertisements, persist them per live node, expose node provenance in catalog/effective inventories, and execute through existing `node.invoke` authorization and command policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review against current main gives high confidence that node caps/commands and `node.invoke` exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive Gateway protocol/runtime design with inventory, provenance, and policy integration rather than more wrapper-plugin indirection.
