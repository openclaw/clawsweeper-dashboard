## Summary

Keep open: current main and latest release still only accept node caps/commands plus core/plugin/MCP tool sources, so node-advertised LLM tools remain missing and need maintainer product/security design.

## What This Changes

Request to let connected Gateway nodes advertise first-class agent tool definitions that OpenClaw exposes directly to LLM tool selection without a wrapper plugin.

## Best Possible Solution

Keep this as the canonical maintainer-owned Gateway protocol/runtime design thread for additive node tool advertisement, node-provenance inventory, and policy-gated execution through existing node authorization.

## Reproduction Assessment

Yes, at source level: the current connect schema, NodeRegistry, tool assembly, catalog, and effective inventory paths have no node-advertised tool definition path. This is a feature gap rather than a broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The requested direction is plausible and not covered by the wrapper-plugin workaround, but the schema, provenance, naming, and policy model need design first.
