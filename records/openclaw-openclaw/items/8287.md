## Summary

Keep open. Current main still accepts node capabilities and command names, and exposes one static `nodes` tool plus plugin tools, but it does not let connected nodes advertise LLM tool descriptors or add node provenance to tool inventories. The closed https://github.com/openclaw/openclaw/issues/33838 thread was explicitly folded into this broader request, so this remains the canonical design item.

## What This Changes

Allow connected nodes to register LLM-visible agent tool definitions that the Gateway dynamically exposes and routes through `node.invoke`.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, route execution through `node.invoke` and existing command policy, and document lifecycle plus allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that node caps/commands and `node.invoke` exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is core Gateway protocol/runtime work with inventory, lifecycle, provenance, and policy integration; closing as ClawHub would miss the missing core extension point.
