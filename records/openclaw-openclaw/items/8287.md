## Summary

Keep open. Current main still supports node-declared command names plus plugin-registered tools, but it does not provide a Gateway protocol/runtime path for connected nodes to advertise first-class LLM tool definitions or lifecycle-managed node-backed tools.

## What This Changes

Allow connected nodes to advertise agent tool definitions that the Gateway exposes directly to the LLM without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway protocol and runtime contract for validated node-advertised tools, with NodeRegistry lifecycle state, synthesized node-backed tools, inventory provenance, node.invoke authorization, and documented policy behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence the requested feature is absent: nodes declare commands, while agent tools come from static core tools plus plugins.

## Solution Assessment

Unclear until maintainer design. The requested direction addresses a real core protocol/runtime gap, but the maintainable path should be an additive, validated node tool contract rather than a direct pass-through around node.invoke.
