## Summary

Keep open. Current main still lacks node-advertised first-class LLM tools, and the live issue context continues to treat this as a Gateway protocol/runtime feature with product and security implications.

## What This Changes

The issue requests a Gateway protocol/runtime feature that lets connected nodes advertise first-class LLM tool definitions without a wrapper plugin.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, and execution through existing node.invoke authorization and command policy.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main only carries node command names and exposes one static nodes agent tool.

## Solution Assessment

Yes as a direction, but it needs maintainer product and security design before implementation. A wrapper plugin or ClawHub-only answer cannot provide the missing node advertisement, lifecycle, and provenance contract.
