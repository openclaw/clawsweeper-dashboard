## Summary

Keep open: current main and the latest release still lack node-advertised agent tools, while the related implementation is an open, unmerged maintainer PR that still needs the normal product/security review path.

## What This Changes

The issue requests allowing connected nodes to advertise agent-visible tool definitions that the Gateway can expose directly to the LLM without a wrapper plugin.

## Best Possible Solution

Keep this issue as the canonical product/security design thread until https://github.com/openclaw/openclaw/pull/90431 lands or maintainers choose a different additive Gateway protocol and node tool policy shape.

## Reproduction Assessment

Yes, at source level: current main has no node tool-advertisement schema, no NodeRegistry storage for node-advertised tool definitions, and no node-backed tool source in agent tool assembly. This is a feature absence rather than a failing runtime bug repro.

## Solution Assessment

Not yet. The requested core gap is real, but the best implementation should come through a maintainer-approved protocol, provenance, and trust contract, with https://github.com/openclaw/openclaw/pull/90431 as the current unmerged candidate.
