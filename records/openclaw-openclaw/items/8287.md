## Summary

Keep open: current main and v2026.6.8 still only support node-declared caps/commands plus plugin/core/channel/MCP tool inventory, not connected-node advertisement of first-class LLM tool definitions. The narrower command-schema request was closed as a duplicate of this issue, so this remains the canonical design thread.

## What This Changes

Request support for connected nodes to advertise first-class agent tools that the Gateway exposes to the LLM without requiring a wrapper plugin.

## Best Possible Solution

Keep this issue as the maintainer-owned design thread for an additive node tool-advertisement protocol, node-backed tool synthesis, execution routing through node.invoke policy, catalog/effective inventory provenance, and documented security semantics.

## Reproduction Assessment

Yes, at source level: current main has no node tool-advertisement schema, no node tool storage in NodeRegistry, no generated node-backed tools in tool assembly, and no node provenance in tool inventory. This is a feature absence rather than a failing runtime bug repro.

## Solution Assessment

Unclear as an implementation plan, but the requested core design direction is plausible because current plugins can register tools while connected nodes cannot advertise LLM tool schemas directly. Maintainers still need to choose the exact protocol, provenance, naming, lifecycle, and security policy contract before code should be written.
