## Summary

Keep open. Current main still exposes nodes through declared capabilities/commands and plugin-registered tools, but it does not let connected nodes advertise first-class LLM tool definitions that are dynamically registered and removed with the node lifecycle.

## What This Changes

Allow connected nodes to advertise LLM-facing tool definitions that the Gateway registers dynamically and routes through node invocation without wrapper plugins.

## Best Possible Solution

Keep this open for an additive Gateway protocol and runtime design that validates node-advertised tool schemas, tracks them per connected node, exposes node provenance in tool inventories, and routes execution through existing node.invoke policy boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review gives high confidence that current main has node caps/commands and node.invoke, but no node-advertised LLM tool registration path.

## Solution Assessment

Yes as a direction, but it needs maintainer protocol and security design before implementation. The maintainable path is additive node tool advertisement plus inventory/policy integration, not closing this as a ClawHub-only plugin workaround.
