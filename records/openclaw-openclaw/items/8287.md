## Summary

Keep open. Current main and the latest release still expose node commands through the static `nodes` tool and plugin tool paths, but they do not let connected nodes advertise first-class model-visible tool definitions, node provenance, lifecycle handling, or policy semantics.

## What This Changes

Request for connected nodes to advertise model-visible agent tools directly through the Gateway instead of requiring wrapper plugins.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract with validated schemas, node provenance in catalog/effective inventory, lifecycle handling, execution through existing `node.invoke` authorization, and documented policy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main and `v2026.5.22` accept node command names and expose one static `nodes` agent tool rather than node-advertised tool definitions.

## Solution Assessment

Yes as a product direction, but not as an automatic code change. The maintainable path needs maintainer-approved protocol, inventory, lifecycle, policy, and security design rather than another wrapper plugin.
