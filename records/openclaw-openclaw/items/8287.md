## Summary

Keep open. Current main still supports node-declared caps and command names, `node.invoke`, plugin-registered tools, and plugin runtime node helpers, but it does not let connected nodes advertise first-class LLM tool descriptors or expose node-backed tool provenance; recent discussion also keeps this as the canonical design item after the narrower command-schema request was folded into it.

## What This Changes

Request support for connected nodes to register agent tools that the Gateway exposes directly to LLMs without a wrapper plugin.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, route execution through `node.invoke` and existing command policy, and document lifecycle plus allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that node caps/commands and `node.invoke` exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is core Gateway protocol/runtime work with inventory, lifecycle, provenance, and policy integration; a ClawHub wrapper plugin does not solve the missing node advertisement contract.
