## Summary

Keep open: current main and v2026.6.10 still lack node-advertised agent tools, and the matching implementation remains an open, conflicting maintainer PR rather than a merged fix.

## What This Changes

The issue requests a Gateway protocol/runtime feature that lets connected nodes advertise LLM tool definitions directly instead of requiring wrapper plugins around node commands.

## Best Possible Solution

Keep this issue as the canonical product/security design thread until the node-hosted tool PR lands or maintainers choose a different additive Gateway protocol and node tool policy shape.

## Reproduction Assessment

Yes, at source level: current main has no node tool-advertisement schema, no NodeRegistry storage for node-advertised tool definitions, and no node-backed tool source in agent tool assembly. This is a feature absence rather than a failing runtime bug repro.

## Solution Assessment

No final implementation is on main yet. The current best path is maintainer review of the open node-hosted tool PR or a replacement additive protocol design that preserves node.invoke policy, provenance, and inventory behavior.
