## Summary

Keep open. Current main still has node capabilities/commands and plugin-registered tools, but it does not provide a Gateway protocol or runtime path for connected nodes to advertise first-class LLM tool definitions that are registered and removed with node lifecycle.

## What This Changes

Request to let connected nodes advertise LLM tool definitions over the Gateway protocol so the Gateway dynamically exposes node-backed agent tools.

## Best Possible Solution

Design an additive Gateway node tool-advertisement contract, validate and store advertised tools per connected node, expose node provenance in catalog/effective inventories, and route execution through existing node.invoke authorization and command policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review against current main gives high confidence that node caps/commands and node.invoke exist, while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is an additive protocol/runtime design with inventory and policy integration, not a wrapper plugin workaround.
