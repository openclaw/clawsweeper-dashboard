## Summary

Keep open: current main still lacks node-advertised first-class LLM tool descriptors, and the overlapping node-hosted tools PR is open rather than merged. The issue remains the canonical design thread for the Gateway protocol, node lifecycle, tool inventory, and security-policy contract.

## What This Changes

Request to let connected nodes advertise first-class agent tool definitions that the Gateway exposes directly to LLMs without a wrapper plugin.

## Best Possible Solution

Keep this issue open as the canonical design thread and review/land or reject the overlapping node-hosted tools PR after its protocol, plugin SDK, node policy, and upgrade-safety questions are settled.

## Reproduction Assessment

Yes at source level for the missing capability: current main has connect-time node command names and one static nodes tool, but no node-advertised tool descriptor path. This is a feature gap, not a broken existing behavior with a runtime repro.

## Solution Assessment

Unclear until maintainers choose the permanent contract. The open node-hosted plugin/MCP tools PR is a plausible implementation path, but it still needs maintainer acceptance because it changes protocol, plugin SDK, node policy, and effective tool inventory semantics.
