## Summary

Keep open. Current main and the latest release still expose node commands as declared command names plus one static `nodes` agent tool, not node-advertised first-class LLM tool descriptors with lifecycle, provenance, and policy integration.

## What This Changes

Request connected nodes to advertise agent tool definitions that the Gateway registers and routes directly through `node.invoke`.

## Best Possible Solution

Add an additive Gateway node tool-advertisement contract, persist validated descriptors per live node, expose node provenance in catalog/effective inventories, route execution through `node.invoke` and existing command policy, and document lifecycle plus allow/deny behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that node caps/commands and `node.invoke` exist while node-advertised LLM tool registration does not.

## Solution Assessment

Yes as a product direction, but not as an automation-ready patch. The maintainable path is Gateway protocol/runtime work with inventory, lifecycle, provenance, and policy integration; a wrapper plugin does not solve the missing node advertisement contract.
