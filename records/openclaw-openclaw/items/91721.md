## Summary

Keep open: the reported bug is real and the recovery-owned fix direction is close, but this patch is still too broad because it marks every non-mutating recovery outcome idle, including skipped active or in-flight recoveries.

## What This Changes

The branch changes diagnostic recovery to set non-idle diagnostic session entries to idle with `queueDepth = 0` when a recovery outcome does not mutate session state, while keeping pruning idle-only.

## Best Possible Solution

Land a narrow recovery-coordinator fix that only converts genuine failed-recovery ghost states to idle after preserving the existing freshness and active-work invariants.

## Reproduction Assessment

Yes. Source inspection shows current main leaves `status: "failed"` outcomes non-idle because they return through the non-mutating branch; I did not execute tests in this read-only review.

## Solution Assessment

No. The recovery-owned location is the right layer, but the implementation must be narrowed so skipped active/in-flight recoveries keep their existing session state.
