## Summary

Keep this PR open, but it is not merge-ready: the recovery-owned fix direction is useful, yet the new failed-outcome path still reuses a mutating cleanup helper as a predicate and can mark an actually active diagnostic lane idle.

## What This Changes

The PR changes diagnostic recovery handling so failed stuck-session recovery outcomes can transition current, ownerless diagnostic session state back to idle for later pruning.

## Best Possible Solution

Keep the recovery-owned fix shape, but repair the failed-outcome branch so it checks active ownership without clearing it, then add committed coordinator regression tests and refreshed real-behavior proof.

## Reproduction Assessment

Yes from source inspection: a failed recovery with a pre-existing same-session embedded owner reaches the new failed-outcome branch, the helper clears that owner because it predates the recovery snapshot, and the branch can then set the state to idle.

## Solution Assessment

No. The recovery-owned fix location is better than broad TTL pruning, but this implementation needs a non-mutating owner check and committed failed-outcome tests before it is the best fix.
