## Best Possible Solution

Land one canonical recovery-owned cleanup that idles only proven ownerless failed recoveries, preserves current and fresh active owners, and adds focused coordinator regression tests.

## Solution Assessment

No. Recovery-owned cleanup is the right layer, but this patch must not use the owner-clearing helper as proof for failed outcomes because that helper can delete pre-existing activity markers.

Full review comments:

- **[P1] Keep active owners intact on failed recovery:** `src/logging/diagnostic-session-recovery-coordinator.ts:118-128`
  - body: This failed-outcome branch calls `clearDiagnosticEmbeddedRunActivityForSession()` before idling the diagnostic state. That helper removes recovered-owner markers that started before the recovery cutoff and only blocks if newer activity remains, so a thrown recovery can erase an existing active owner and then set `state = "idle"` with `queueDepth = 0`. Failed recovery is not proof that the owner is gone; preserve active state unless ownerlessness is established without clearing it.
