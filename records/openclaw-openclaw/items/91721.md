## Summary

Keep open: the PR targets a real diagnostic ghost-state bug, but the failed-recovery branch can clear pre-existing active diagnostic owners and mark the state idle even when recovery threw before abort/drain succeeded.

## What This Changes

The branch adds a failed-recovery path that attempts to mark current diagnostic session states idle so existing stale-idle pruning can remove them, plus a small prune-loop cleanup.

## Best Possible Solution

Only idle failed-recovery states after proving the owner is gone, preserve active owners on failed outcomes, and use the normal recovery state-transition invariants with committed regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main returns early for failed outcomes before state mutation and prunes only idle empty entries; I did not execute a runtime repro in this read-only review.

## Solution Assessment

No. The ownerless failed-cleanup direction is plausible, but this implementation reuses a successful-recovery activity-clearing helper for failed outcomes and can erase active owner state.
