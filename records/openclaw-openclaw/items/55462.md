## Summary

Keep open. Current `main` and latest release still do not carry the Telegram callback source `message_id` to the generic agent callback context, and an open implementation PR references this issue, so this should stay paired until that PR lands, is replaced, or closes.

## What This Changes

Request to include the source Telegram `message_id` with Telegram `callback_query` turns so agents can call `editMessage` on the originating inline-button message.

## Best Possible Solution

Land a rebased equivalent of https://github.com/openclaw/openclaw/pull/55548 that carries `callbackMessage.message_id` and chat id through every generic Telegram callback turn, preserves `callback.id` for answering and dedupe, and coordinates with https://github.com/openclaw/openclaw/issues/45638.

## Reproduction Assessment

Yes, source-level. A Telegram `callback_query` with `id: "cbq-1"` and `message.message_id: 10` reaches generic callback paths that still set `messageIdOverride` from `callback.id`, with no separate callback source message-id carrier.

## Solution Assessment

Yes, keeping this open is the best current handling. The narrow maintainable path is to review, rebase, land, or replace https://github.com/openclaw/openclaw/pull/55548 rather than closing this issue before the fix reaches `main`.
