## Label Justifications

- `P2`: This is a focused Telegram inline-button workflow gap with real agent usability impact but limited channel-specific blast radius.
- `impact:session-state`: The missing source message identity can mis-associate callback turns, edit targets, and dedupe/session context.

## Summary

Keep open: current main and v2026.5.20 still route generic Telegram callback turns with the callback query id and do not expose a separate source message id to the agent turn, while the open linked implementation PR remains the active fix path.

## What This Changes

Request that Telegram inline-button callback turns expose the originating message id so agents can edit the clicked message.

## Best Possible Solution

Land or replace the linked PR with a rebased fix that carries callback source message id and chat id as explicit metadata across every generic Telegram callback path while preserving callback.id for answering and dedupe.

## Reproduction Assessment

Yes, source-level. Current main has callback fixtures with distinct callback query ids and Telegram message ids, and the generic callback paths pass callback.id as messageIdOverride without a separate source-message-id carrier.

## Solution Assessment

Yes for triage: keeping this issue open is the right path while the linked implementation PR remains unmerged. The implementation should add explicit callback source metadata instead of repurposing the callback query id field.
