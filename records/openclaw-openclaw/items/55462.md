## Label Justifications

- `P2`: This is a focused Telegram inline-button workflow gap with real agent usability impact but limited channel-specific blast radius.
- `impact:session-state`: The missing source message identity can mis-associate callback turns, edit targets, and context metadata.

## Summary

Keep open: current main and v2026.5.22 still route generic Telegram callback turns through `messageIdOverride: callback.id` without a separate source-message-id field, and the open linked implementation PR remains the active fix path.

## What This Changes

Expose the originating Telegram callback message id to agent turns so inline-button callbacks can edit the source message.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/55548 with an additive callback source-message-id path that preserves callback query identity for answering and dedupe.

## Reproduction Assessment

Yes, source-level: current callback fixtures use distinct callback query ids and Telegram message ids, and the generic callback paths pass the callback query id as the message override without a separate source-message-id carrier.

## Solution Assessment

No for a one-line identity swap; the safer solution is an additive callback source metadata field that leaves callback query identity available for answering and dedupe.
