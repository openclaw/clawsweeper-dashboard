## Summary

Keep open. Current main still drops the Telegram callback source message id on the generic agent callback path, and an open implementation PR already references this issue with closing syntax, so the issue should remain paired until that PR lands, is replaced, or closes.

## What This Changes

Expose the source Telegram `callback_query.message.message_id` to the agent so inline-button handlers can edit the originating message.

## Best Possible Solution

Land a rebased equivalent of the linked PR that carries `callbackMessage.message_id` and chat id through every generic Telegram callback turn, preserves `callback.id` for answering and dedupe, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-level. A Telegram `callback_query` with `id: "cbq-1"` and `message.message_id: 10` reaches generic callback routing that still sets `messageIdOverride` from the callback id, leaving no separate callback source message-id carrier.

## Solution Assessment

Yes, keeping this open is the best current handling. The narrow maintainable path is to review, rebase, land, or replace https://github.com/openclaw/openclaw/pull/55548 rather than closing this issue before the fix reaches main.
