## Summary

Keep open. Current main still drops the originating Telegram message ID on generic callback turns, while https://github.com/openclaw/openclaw/pull/55548 remains the open linked implementation path and should land, be replaced, or close before this issue is resolved.

## What This Changes

The issue requests that Telegram inline-button callback turns expose the source Telegram message ID to the agent so `editMessage` can update the original message.

## Best Possible Solution

Land a rebased equivalent of https://github.com/openclaw/openclaw/pull/55548 that carries the callback source message id and chat id through every generic Telegram callback turn while preserving callback query identity for answering and dedupe.

Yes, source-level. A Telegram `callback_query` with distinct `id` and `message.message_id` values reaches current generic callback paths that still use `callback.id` as `messageIdOverride`, and there is no separate callback source-message-id carrier.
