## Summary

Keep open: current main and v2026.6.8 still calculate and persist all-zero per-turn `usage.cost` for all-zero model pricing without a runtime warning; the merged aggregation work is useful mitigation, but it does not solve the reported session JSONL visibility gap.

## What This Changes

The issue asks OpenClaw to stop silently under-reporting cost as zero when a selected provider model has token usage but no known pricing.

## Best Possible Solution

Define a narrow runtime unknown-pricing contract that warns or marks per-turn cost as unknown for hosted paid models with token usage, while preserving explicit free/self-hosted zero-cost models and the aggregate missing-cost behavior already present.

## Reproduction Assessment

Yes, at source level. Any provider path that records nonzero usage and calls `calculateCost` with all-zero model rates will persist zero cost; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution was found for the per-turn warning or unknown-cost marker. The best fix needs maintainer selection of warning versus unknown marker/fallback semantics before implementation.
