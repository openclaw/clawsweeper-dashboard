## Summary

Keep open: current main and the latest release still calculate and persist all-zero runtime `usage.cost` for nonzero usage when the selected model has all-zero/unknown pricing, with no first-use warning. The merged usage-cost aggregation fix is useful adjacent mitigation, but it does not solve the reported session JSONL/runtime visibility gap, and the direct warning PR was closed unmerged.

## What This Changes

The issue reports silent zero-cost usage accounting when a configured provider model is absent from the pricing table.

## Best Possible Solution

Keep the merged aggregate missing-cost behavior and add a maintainer-approved runtime unknown-pricing contract, likely in the shared cost-calculation path, without breaking explicit free/self-hosted zero-cost models.

## Reproduction Assessment

Yes, at source level. Any model with nonzero usage and all-zero cost rates reaches `calculateCost`, and the Anthropic plus sibling provider paths call that helper before usage is persisted.

## Solution Assessment

Unclear until maintainers choose the product contract. A warning or unknown-cost marker is a narrow fix shape, but fallback rates or config-driven pricing would be a broader behavior decision.
