## Summary

Keep open: current main mitigates OpenClaw usage summaries by counting all-zero unpriced turns as missing cost, but the shared runtime cost writer still silently leaves raw `usage.cost` numeric fields at zero when a loaded model has no pricing.

## What This Changes

The issue reports silent zero cost accounting for unpriced model identifiers and asks for either pricing fallback or an operator-visible warning.

## Best Possible Solution

Add a narrow once-per-provider/model warning at the shared `calculateCost` boundary when token usage is nonzero and all loaded pricing rates are zero, while preserving the existing numeric `usage.cost` shape and missing-cost summary behavior.

## Reproduction Assessment

Yes, from source inspection: a loaded model with all-zero `cost` and nonzero token usage will still pass through `calculateCost` as zero without warning. I did not run a live turn because this review is read-only.

## Solution Assessment

No, current main is only a partial mitigation because usage summaries mark unknown costs as missing but the raw runtime write remains silent. The narrow warning shape from https://github.com/openclaw/openclaw/pull/90825 looks like the best low-risk repair path.
