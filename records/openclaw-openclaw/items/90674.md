## Summary

Keep open: current main and the latest release still persist all-zero per-turn `usage.cost` from all-zero model pricing with no runtime warning, while the directly linked warning PR was closed unmerged. The merged usage-cost aggregation work is useful adjacent mitigation, but it does not solve the reported session JSONL/runtime visibility gap.

## What This Changes

The issue reports silent zero-cost accounting when a selected provider model has token usage but no known pricing entry.

## Best Possible Solution

Define a narrow runtime unknown-pricing contract that warns or marks per-turn cost as unknown for hosted paid models with nonzero token usage, while preserving explicit free/self-hosted zero-cost models and the aggregate missing-cost behavior already on main.

## Reproduction Assessment

Yes, at source level. Any model with nonzero usage and all-zero `model.cost` reaches `calculateCost`, and the Anthropic provider calls that helper on streamed usage updates; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution was found for the per-turn warning or unknown-cost marker. The best fix needs a maintainer decision on how to distinguish unknown paid pricing from intentional free pricing before implementing the narrow runtime behavior.
