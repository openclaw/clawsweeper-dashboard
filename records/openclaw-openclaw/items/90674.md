## Summary

Close: current main and v2026.6.1 already stop supported usage/cost summaries from treating unknown-pricing token usage as confident zero spend, and current config/docs provide operator pricing inputs.

## What This Changes

The issue asks OpenClaw to stop silently reporting zero cost for token-using turns when the selected model has no known pricing metadata.

## Best Possible Solution

Keep the shipped missing-pricing signaling plus configured/gateway pricing paths; open a narrower follow-up only if raw JSONL consumers need a separate contract beyond supported usage summaries.

## Reproduction Assessment

Yes. The current regression fixture constructs the reported shape, a token-burning assistant turn whose transcript carries usage.cost.total: 0, and verifies current main reports it as a missing-cost entry.

## Solution Assessment

Yes. Treating unknown pricing as missing/partial in the summary paths and allowing configured or gateway pricing to fill real rates is safer than guessing default model rates.
