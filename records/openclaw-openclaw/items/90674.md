## Summary

Keep open: current main and v2026.6.6 partially mitigate aggregate usage-cost by counting unknown zero-rate pricing as missing, but the shared cost writer still records all-zero `usage.cost` without an operator-visible warning. The related warning PR was closed unmerged, so the specific requested visibility gap remains.

## What This Changes

The issue asks OpenClaw to stop silently writing zero cost for nonzero token usage when a model has no known pricing, either by estimating cost or warning operators.

## Best Possible Solution

Add a shared once-per-model warning for nonzero token usage with all-zero model pricing while preserving the current `missingCostEntries` aggregation and the existing numeric `usage.cost` schema.

## Reproduction Assessment

Yes from source inspection: an all-zero model cost plus nonzero usage makes `calculateCost` write every cost bucket and `total` as zero and return without warning. I did not run a live provider request in this read-only sweep.

## Solution Assessment

Yes: the narrowest maintainable fix is to warn at the shared cost calculation boundary and leave the already-shipped usage-cost aggregation mitigation intact. Adding fallback rates or a new config surface would be broader than necessary for this report.
