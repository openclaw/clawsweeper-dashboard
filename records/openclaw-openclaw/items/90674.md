## Summary

Keep open: current main has a shipped mitigation for transcript-backed `usage.cost` summaries, but the shared runtime calculation still records an all-zero `usage.cost` object with no first-use warning when the resolved model has all-zero pricing. The linked warning PR was closed unmerged, so no canonical open PR currently owns the remaining runtime/transcript visibility gap.

## What This Changes

The issue asks OpenClaw to stop silently treating unknown model pricing as zero cost, either by estimating from fallback/configured rates or by warning operators when `usage.cost` remains zero because pricing is unknown.

## Best Possible Solution

Define one explicit unknown-pricing signal for runtime usage and transcript consumers, while preserving the existing `missingCostEntries` aggregate behavior and avoiding false warnings for intentionally free/local models.

## Reproduction Assessment

Yes from source inspection: an all-zero model cost block plus nonzero token usage flows through `calculateCost` to a zero `usage.cost` object with no warning. I did not run a live provider turn in this read-only review.

## Solution Assessment

Unclear as an implementation choice: the existing aggregate missing-cost fix is useful but does not solve raw transcript/runtime visibility. The best next step needs maintainer agreement on how to distinguish unknown paid pricing from deliberate free/local zero pricing before adding a runtime warning or data-shape signal.
