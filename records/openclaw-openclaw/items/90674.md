## Summary

Keep open: current main has an aggregate-side missing-cost mitigation, but the reported per-turn `message.usage.cost` path still records all-zero cost with no warning when a provider model has token usage and all-zero pricing.

## What This Changes

The issue requests an operator-visible way to distinguish unknown paid-model pricing from real zero-cost usage in persisted per-turn usage data.

## Best Possible Solution

Keep this issue open for a narrow shared runtime warning or explicit missing-pricing signal that preserves current aggregate missingCostEntries behavior and intentional free/local zero-cost models.

## Reproduction Assessment

Yes from source inspection, though no live turn was run in this read-only review. The Anthropic provider records token usage and calls `calculateCost`, which still converts all-zero pricing into all-zero `usage.cost` with no warning.

## Solution Assessment

No, current main is only a partial mitigation. Aggregate `/usage cost` can surface missing cost entries, but the issue's per-turn JSONL/warning behavior remains unresolved.
