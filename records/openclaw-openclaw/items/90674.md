## Summary

Keep open: current main and v2026.6.9 mitigate aggregate `usage-cost` summaries, but the runtime still writes per-turn zero `usage.cost` fields without a warning, so the reported session JSONL visibility gap remains.

## What This Changes

Ensure unknown or unpriced model strings do not silently persist `usage.cost` as confident zero in gateway session usage data.

## Best Possible Solution

Define a narrow runtime/session unknown-pricing contract that makes paid-model missing pricing visible per turn, while preserving the aggregate missing-cost behavior and intentional free/local zero-cost models.

## Reproduction Assessment

Yes, at source level. Any provider path that records nonzero usage and calls `calculateCost` with all-zero model rates still produces a per-turn `usage.cost` object whose fields are all zero; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution fully solves the issue. The aggregate missing-cost mitigation is useful, but the best fix still needs a maintainer-approved per-turn warning or unknown-cost marker contract.
