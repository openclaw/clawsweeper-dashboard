## Summary

Keep open: current main still has the silent all-zero `usage.cost` path for models whose resolved runtime `Model.cost` is all zeros, and the only linked mitigation PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to stop silently writing zero cost totals for priced models that are missing runtime pricing, either by estimating from fallback rates or warning operators clearly.

## Best Possible Solution

Add a provenance-aware or tightly scoped once-per-model warning for unknown hosted-model pricing while preserving the existing numeric `usage.cost` shape and intentional free/local zero-cost behavior.

## Reproduction Assessment

Yes at source level: current main still normalizes missing model pricing to all-zero rates and `calculateCost` writes zero cost fields without warning. I did not run a live agent turn in this read-only review, so this is source-reproducible rather than executed locally.

## Solution Assessment

The closed warning PR is a plausible direction, but the best fix should also preserve intentional free/local zero-cost models. A narrow replacement should make unknown hosted pricing visible without adding configurable fallback rates or changing the persisted `usage.cost` shape.
