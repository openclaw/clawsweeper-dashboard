## Summary

Keep open: current main and v2026.6.8 still compute per-turn `usage.cost` by multiplying token counts against all-zero model pricing with no warning, while the merged aggregate-cost mitigation only marks scanned summaries as missing cost and the direct warning PR closed unmerged.

## What This Changes

The issue requests that OpenClaw stop silently writing all-zero per-turn `usage.cost` for unpriced model strings, either by estimating cost from a real pricing source or warning that pricing is unknown.

## Best Possible Solution

Define a narrow runtime unknown-pricing signal that warns or marks per-turn cost as unknown for hosted paid models with nonzero token usage, while preserving explicit free/self-hosted zero-cost models and the existing aggregate missing-cost behavior.

## Reproduction Assessment

Yes. Source inspection shows that a nonzero-usage model with all-zero `model.cost` reaches `calculateCost`, which writes all-zero `usage.cost` without logging; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution fully solves this issue. The aggregate summary mitigation is useful but incomplete for the per-turn JSONL/runtime warning gap, and the permanent fix needs a maintainer-approved unknown-pricing contract.
