## Summary

Keep open: current main and v2026.6.8 still calculate and persist all-zero per-turn `usage.cost` for all-zero model pricing without a runtime warning; the aggregate `missingCostEntries` mitigation and current local pricing docs help, but they do not solve the reported session JSONL visibility gap.

## What This Changes

The issue reports that unrecognized or unpriced model strings can record nonzero token usage with all-zero `usage.cost` fields and no operator-visible warning.

## Best Possible Solution

Define a narrow runtime unknown-pricing contract that warns or marks per-turn cost as unknown for hosted paid models with token usage, while preserving explicit free/self-hosted zero-cost models and the aggregate `missingCostEntries` behavior already present.

## Reproduction Assessment

Yes, at source level. Any provider path that records nonzero usage and calls `calculateCost` with all-zero model rates will keep a numeric all-zero `usage.cost`; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution was found for the per-turn warning or unknown-cost marker. The best fix needs maintainer selection of warning versus unknown marker/fallback semantics before implementation.
