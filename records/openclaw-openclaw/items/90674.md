## Summary

Keep open: current main and the latest release still compute per-turn `usage.cost` from all-zero model pricing without an operator warning, while the directly linked warning PR was closed unmerged. The merged adjacent usage-cost work helps aggregate summaries count some all-zero pricing as missing, but it does not fix the session JSONL/runtime warning gap reported here.

## What This Changes

The issue reports that unknown model pricing causes `usage.cost` to be recorded as zero without any operator-visible warning.

## Best Possible Solution

Define a narrow runtime unknown-pricing signal that warns or marks per-turn cost as unknown for hosted paid models with nonzero token usage, while preserving intentional free/self-hosted zero-cost models and the existing aggregate missing-cost behavior.

## Reproduction Assessment

Yes. Source inspection shows that any Anthropic/runtime model with nonzero usage and all-zero `model.cost` reaches `calculateCost`, which writes an all-zero `usage.cost` object without logging; I did not run a live provider turn in this read-only review.

## Solution Assessment

No current-main solution was found for the per-turn warning/fallback. The closed related PR was a plausible warning shape, but the best fix still needs a maintainer decision on how to distinguish unknown paid pricing from intentional free pricing.
