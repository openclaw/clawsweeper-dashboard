## Summary

Keep open: current main has staged package swaps, post-core finalization, and managed-service handoff mitigations, but the maintainer-labeled request for a durable external updater that owns package replacement outside the installed OpenClaw package is still not implemented.

## What This Changes

The issue requests moving `openclaw update` package replacement into a small external updater process that runs outside the install root being replaced.

## Best Possible Solution

Define a maintainer-approved external updater contract, then implement a tiny updater outside `node_modules/openclaw` that reads a durable plan, installs and verifies a candidate, swaps with rollback, runs post-update doctor/plugin convergence, and handles service restart across supported platforms.

## Reproduction Assessment

Not applicable: this is an architectural follow-up request, not a bug report with a single failing current-main reproduction path. Source inspection gives high confidence that the requested general external-updater boundary remains absent.

## Solution Assessment

Unclear as an implementation: the requested boundary is coherent and aligned with updater reliability, but the exact plan format, helper packaging, rollback model, and service handoff require maintainer design before code should land.
