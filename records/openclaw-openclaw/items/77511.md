## Summary

Keep open: current main and the latest release have staged package swaps and managed-service handoff mitigations, but not the requested durable external updater that owns package replacement outside the installed OpenClaw package. The live item also carries protected maintainer/product-decision labels, so this remains a maintainer design item rather than conservative cleanup.

## What This Changes

Request to move `openclaw update` package replacement into a small external updater that runs outside the package being replaced and uses a durable update plan.

## Best Possible Solution

Define a maintainer-approved external updater contract, then implement a tiny updater outside `node_modules/openclaw` that reads a durable plan, installs and verifies a candidate, swaps with rollback, runs post-update doctor/plugin convergence, and handles service restart across supported platforms.

## Reproduction Assessment

Not applicable: this is an architectural follow-up request, not a bug report with a single failing current-main reproduction path. Source inspection gives high confidence that the requested general external-updater boundary remains absent.

## Solution Assessment

Unclear as an implementation: the requested boundary is coherent and aligned with updater reliability, but the exact plan format, helper packaging, rollback model, and service handoff need maintainer design before code should land.
