## Summary

Keep open: current main has staged package swaps and managed-service handoff mitigations, but it still does not provide the requested durable external updater that owns package replacement outside the installed OpenClaw package. The live issue also has a protected `maintainer` label and product-decision labels, so conservative cleanup should not close it.

## What This Changes

The issue requests moving `openclaw update` package replacement into a small external updater that runs outside the OpenClaw package being replaced.

## Best Possible Solution

Define a maintainer-approved external updater contract, then implement a small updater outside `node_modules/openclaw` that reads a durable plan, installs and verifies a candidate, swaps with rollback, runs post-update convergence, and handles service restart across supported platforms.

## Solution Assessment
