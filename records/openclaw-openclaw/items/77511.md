## Summary

Keep open: the issue has the protected `maintainer` label, and current main still only has staged swaps plus managed-service handoffs around the normal CLI updater rather than a full external updater that owns package replacement outside the package being replaced.

## What This Changes

Request to move package update execution into a small external updater so package replacement and post-update recovery do not run from the package being replaced.

## Best Possible Solution

Keep this as a maintainer-owned updater architecture tracker until OpenClaw has a durable external updater plan that preserves staged verification, exits the live package process before replacement, runs post-update repair from the right boundary, and has cross-platform rollback proof.

## Reproduction Assessment

Not applicable: this is an architecture follow-up, not a single bug report. Source inspection gives high confidence that current code still runs package replacement through the normal CLI updater path, with managed-service handoff only around Gateway-triggered cases.

## Solution Assessment

Unclear as a one-shot implementation, but the requested boundary is plausible and aligned. The best fix needs maintainer design for helper distribution, update-plan security, rollback, service handoff, and doctor/plugin recovery before code is written.

## Security Review

Status: needs_attention

Summary: Implementing this issue is security-sensitive because an external updater would own package-manager execution, durable plans, temp files, and rollback authority.

Concerns:

- **[medium] Protect update-plan integrity:**
  - body: A durable update plan must be owner-only, non-world-writable, and bound to the intended package root/version so a local attacker cannot swap commands or redirect package replacement.
  - confidence: 0.86
- **[medium] Constrain privileged package replacement:**
  - body: The helper should execute only the intended package-manager and doctor/restart operations, with clear environment and path handling, because it will run outside the live OpenClaw process and mutate executable files.
  - confidence: 0.82
