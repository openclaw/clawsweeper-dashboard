## Summary

Keep open: current main has staged package swaps and managed-service handoff mitigations, but it still does not provide the requested durable external updater that owns package replacement outside the installed OpenClaw package. The live issue also has a protected maintainer label and product-decision labels, so it is not conservative cleanup.

## What This Changes

Request to move openclaw update package replacement into a small external updater that runs outside the package being replaced and can verify, swap, repair, restart, and roll back from a durable plan.

## Best Possible Solution

Define a maintainer-approved external updater contract, then implement a small updater outside node_modules/openclaw that reads a durable plan, installs and verifies a candidate, swaps with rollback, runs post-update convergence, and handles service restart across supported platforms.

## Solution Assessment

Unclear as an implementation: the boundary is coherent and aligned with updater reliability, but the plan format, helper packaging, rollback model, and service handoff need maintainer design before code should land.
