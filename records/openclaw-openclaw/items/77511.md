## Summary

Keep open: current main has staged swaps and a managed-service handoff for Gateway/control-plane updates, but it still does not implement the requested durable external updater for normal `openclaw update` package replacement, and the item has a protected `maintainer` label.

## What This Changes

The issue asks OpenClaw to move package replacement out of the current `openclaw update` process into a small external updater with a durable plan, verification, rollback, doctor, and restart handoff.

## Best Possible Solution

Define and implement a core external-updater boundary where `openclaw update` writes a durable plan, exits before package replacement, and a tiny helper outside `node_modules/openclaw` performs staged install, verification, swap, rollback, post-update doctor/plugin convergence, and restart handoff with cross-platform proof.

## Reproduction Assessment

Not applicable: this is an architectural updater follow-up, not a bug report with a failing current-main reproduction path. Source inspection gives high confidence that the requested external-updater boundary is still absent.

## Solution Assessment

Unclear as an implementation: the direction is coherent and aligned with updater reliability, but the plan format, helper packaging, rollback model, doctor/plugin handoff, and service ownership need maintainer design before code should land.
