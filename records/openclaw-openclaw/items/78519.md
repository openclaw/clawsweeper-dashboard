## Summary

Close as duplicate: the same 2026.5.5 doctor rewrite regression is already tracked in the open canonical issue, including the silent paid-API failure mode when an OpenAI API key is present, and an open reproduction PR is linked there.

## What This Changes

The issue reports that upgrading to 2026.5.5 rewrote an OpenAI Codex OAuth subscription route to an OpenAI API route when an OpenAI API key was also configured.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/78407 as the canonical tracking issue and preserve this API-billing report as duplicate evidence for the eventual doctor/auth-route fix.

## Reproduction Assessment

Yes from source inspection and related reports: current doctor --fix calls the Codex route repair, rewrites openai-codex/* to openai/*, and can set agentRuntime.id to pi. The OpenAI docs then make the reported billing risk clear because openai/* plus pi is the direct OPENAI_API_KEY route.

## Solution Assessment

Yes for cleanup: closing this duplicate and tracking the fix in https://github.com/openclaw/openclaw/issues/78407 is the narrowest maintainer path. The actual product fix should align doctor, docs, auth selection, and regression tests so a subscription route is never silently migrated to paid API billing.
