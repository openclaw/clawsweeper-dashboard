## What This Changes

The PR adds a `gateway.reload.autoRestartOnRequired` hot-mode opt-in, always restarts hot-mode auth/secrets restart-required changes, marks `secrets.providers.*` restart-required, and updates schema/help/docs/tests.

## Best Possible Solution

Unify the hot-mode restart policy across file-watcher and control-plane config writes, document the auth/secrets carve-out on every touched user-facing reload summary, and get maintainer approval for the new config/default split before merge.
