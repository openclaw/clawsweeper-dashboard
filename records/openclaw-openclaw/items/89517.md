## Summary

Keep this PR open for maintainer review: it addresses a real stale-runtime config hole, but the proposed fix changes the compatibility-sensitive `hot` reload contract and can turn a restart-required config edit into a gateway process restart or outage on some supervisors.

## Best Possible Solution

Preserve a distinct no-restart `hot` policy unless maintainers explicitly decide to redefine it; use the existing explicit writer restart intent for doctor/configure repairs that know a restart is required.
