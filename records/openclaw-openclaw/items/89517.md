## Summary

The reported stale-runtime problem is real and the live proof is strong, but the current patch is not merge-ready: its forced-restart security rule targets `gateway.secrets.*`, while OpenClaw’s secret-provider configuration lives under top-level `secrets.*`, so the stated SecretRef rotation protection does not cover that configuration surface. The branch also adds an upgrade-sensitive reload option overlapping `hybrid`; keep it open for a focused correction and maintainer confirmation of the permanent policy.

## What This Changes

The PR adds `gateway.reload.autoRestartOnRequired`, preserves routine hot-mode warn-and-keep behavior by default, and forces hot-mode restarts for changes classified under `gateway.auth.*` or `gateway.secrets.*`, with schema, docs, and tests.

## Best Possible Solution

Fix the credential classifier against the actual top-level secret/provider configuration contract, add focused regression coverage for those paths, and have gateway/config owners decide whether external edits need this new option or should use the existing `hybrid` mode plus explicit restart intent for managed writers.

## Reproduction Assessment

Yes. The contributor supplied production logs showing restart-required hot-mode edits leaving stale runtime state, plus after-fix sandbox and deployed logs; current source confirms the warning-and-return path.

## Solution Assessment

No. The restart concept is plausible, but the current patch’s SecretRef guarantee is wired to the wrong config prefix and the new mode-adjacent option still needs an owner-backed justification over `hybrid` and explicit writer restart intent.
