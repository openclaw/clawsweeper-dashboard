Full review comments:

- **[P2] Apply the hot-mode contract to config RPC writes:** `src/config/types.gateway.ts:276-283`
  - body: The new option says hot mode keeps routine restart-required edits in warn-and-keep unless `gateway.reload.autoRestartOnRequired` is true, but `config.patch` and `config.apply` still schedule a direct restart for `mode === "hot" && plan.restartGateway`. A gateway RPC edit such as `gateway.bind` can therefore restart with the option unset, contradicting the new public contract. Please wire the option/security carve-out through that path or scope the docs/type help to external file-watcher edits only.
  - confidence: 0.9
- **[P2] Document the secrets restart carve-out accurately:** `docs/gateway/configuration-reference.md:697`
  - body: This says `secrets.*` security-critical changes always auto-restart, but the planner only marks `secrets.providers` as restart-required before the general `secrets` no-op rule, and the new tests preserve `secrets.defaults.*` as no-op. Please narrow the docs/help text to the actual restart-required secret provider paths, or change the planner if all `secrets.*` edits are meant to restart.
  - confidence: 0.86
