## Summary

Keep open for normal PR review and landing. The docs-only patch matches current source behavior: `/usage` footer text is assembled and appended only on the normal final-reply payload path, while message-tool sends do not pass through that footer hook.

## What This Changes

Updates usage and gateway docs to state that `/usage tokens|full` footers apply to normal final replies, not explicit message-tool sends such as default tool-only group/channel visible replies.

## Best Possible Solution

Land this docs clarification if maintainers accept documenting current behavior; treat automatic footer injection for message-tool sends as a separate runtime feature decision.
