## Summary

## What This Changes

The issue requests that `plugins.entries.lossless-claw.llm.allowModelOverride` and `allowedModels` be honored by LCM/context-engine runtime LLM completions on normal gateway startup, without relying on config hot reload.

## Best Possible Solution

Review and land one canonical fix, preferably the context-engine authority path that propagates the owning plugin's explicit `llm` policy without bypassing the documented trust gate, then close this issue after merge proof.
