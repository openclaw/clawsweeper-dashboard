## Summary

## What This Changes

The PR changes memory-core corpus supplement search from fail-fast `Promise.all` to `Promise.allSettled` with per-supplement timeout handling, warning logs, and regression tests for rejecting and hung supplements.

## Best Possible Solution

Land a narrow memory-core fix that isolates rejecting or hung supplements, logs only redacted bounded failure details with the plugin id, avoids a hidden env knob unless maintainers explicitly document it, and includes real `memory_search corpus=all` proof.

- **[P2] Redact supplement errors before logging:** `extensions/memory-core/src/tools.shared.ts:249-260`
  - body: `formatSupplementError` reimplements error formatting with raw `Error.message`, raw strings, and `JSON.stringify` for object throws. Corpus supplements are plugin-owned code, so provider failures can carry tokens or private paths; current `formatErrorMessage` redacts before returning, but this new warning path bypasses it. Use the shared redacting formatter, and keep the logged detail bounded, before writing to `console.warn`.
  - confidence: 0.9
- **[P2] Remove or formalize the timeout env override:** `extensions/memory-core/src/tools.shared.ts:157-166`
  - body: `OPENCLAW_MEMORY_SUPPLEMENT_SEARCH_TIMEOUT_MS` adds a new runtime env surface for a hot memory path, but the PR does not document/export it or prove a product need for a new knob. It also feeds the parsed number directly to `setTimeout`, so overflow values can become effectively immediate timeouts. Use a fixed internal deadline, or make this an explicit documented config/env contract with shared timer coercion.
  - confidence: 0.82
