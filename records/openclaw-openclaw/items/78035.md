## Summary

Keep open: this is a focused, source-reproducible memory-core bug fix and the latest diff addresses both rejected and hung supplement searches. Merge should still wait for contributor-supplied real runtime proof, and the public timeout environment override should be documented or made internal.

## What This Changes

The PR changes memory-core supplement search to use per-supplement allSettled/timeout handling with warning logs, adds helper and tool-path regression tests, and adds a changelog fix.

## Best Possible Solution

Land the bounded partial-failure fix after adding real runtime proof and either documenting the timeout override or making it an internal constant.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register one resolving and one rejecting memory corpus supplement, then call memory_search with corpus=all; current main uses Promise.all in searchMemoryCorpusSupplements, so the rejection aborts sibling supplement results before the tool can merge them.

## Solution Assessment

Yes, with one cleanup. Isolating each supplement inside memory-core with allSettled plus a bounded timeout is the narrow owner-module fix; the public timeout override should be documented or internalized before merge.
