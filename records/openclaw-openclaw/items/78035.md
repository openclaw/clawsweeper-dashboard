## Summary

Keep open: the memory-core bug is real and the PR is on the right path, but the current head logs arbitrary supplement rejection details without the repo's redaction helper and the supplied after-fix proof is still test-harness output rather than a live OpenClaw run.

## What This Changes

The PR changes memory-core supplement search to isolate rejecting or hung corpus supplements with `Promise.allSettled` plus a per-supplement timeout, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land a bounded partial-failure fix that preserves healthy supplement results, logs only redacted diagnostics, and includes live OpenClaw `memory_search` proof plus focused regression tests.

## Reproduction Assessment

Yes by source inspection. Register one healthy and one rejecting memory corpus supplement, then call `memory_search` with `corpus="all"`; current main's `Promise.all` path rejects before healthy supplement results can be merged.

## Solution Assessment

No as currently written. `allSettled` plus a bounded timeout is the right shape, but the warning path should use the repo's redacted error formatting and the PR still needs live runtime proof.
