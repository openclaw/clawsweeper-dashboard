## Summary

Keep this PR open because it targets a real memory retrieval bug that still exists on current main and the latest release, but the branch is not merge-ready due to failure-semantics, env-surface, log-redaction, and real-proof blockers.

## What This Changes

The PR changes memory-core supplement search to settled per-supplement aggregation with a timeout, warning logs, and focused regression tests for rejecting or stalled supplements.

## Best Possible Solution

Keep the partial-success direction, but preserve unavailable signaling when no requested supplement succeeds, remove or explicitly document and clamp the timeout env surface, redact warning details through the shared formatter, and require redacted non-test proof before merge.

## Reproduction Assessment

Yes. Current main awaits Promise.all over registered memory corpus supplements, and createMemorySearchTool calls that helper for corpus="wiki" and corpus="all", so one rejecting supplement reproduces the fail-fast behavior by source inspection.

## Solution Assessment

No, not as-is. Isolating supplement failures in searchMemoryCorpusSupplements is the right owner boundary, but this implementation also changes total-failure semantics and adds env/logging surfaces that need tightening.
