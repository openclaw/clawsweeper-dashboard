## Summary

Keep this PR open: it targets a valid, source-reproducible memory retrieval bug, but the branch still has merge-blocking behavior and proof gaps around total supplement failure semantics, a new timeout env surface, and real behavior proof outside tests.

## What This Changes

The PR changes memory-core corpus supplement search to use settled per-supplement execution with a 10s timeout, warning logs, and regression coverage for rejected or hung sibling supplements.

## Best Possible Solution

Keep the partial-success isolation in memory-core, but preserve an explicit failure signal for total requested-supplement failure and either remove or fully document the timeout knob before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main and v2026.6.6 await Promise.all over registered supplement searches, so one rejection rejects the whole fan-out; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. allSettled plus bounded waiting is the right local direction for partial successes, but this branch also hides total supplement failure as empty results and adds an undocumented env knob.
