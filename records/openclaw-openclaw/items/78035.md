## Summary

## What This Changes

The PR changes memory-core supplement search from fail-fast `Promise.all` to settled per-supplement searches with a timeout and warning logs, plus helper and tool-path regression tests.

## Best Possible Solution

Keep the bounded partial-success direction, but preserve unavailable/error behavior when no requested supplement succeeds, remove or formally document and clamp the timeout surface, redact warning details with the shared formatter, and require non-test real behavior proof before merge.

## Reproduction Assessment

Yes. Current main awaits `Promise.all` over registered memory corpus supplements, and `createMemorySearchTool` calls that helper for `corpus="wiki"` and `corpus="all"`, so one rejecting supplement reproduces the fail-fast behavior by source inspection.

## Solution Assessment

No, not as-is. Isolating supplement failures in `searchMemoryCorpusSupplements` is the right owner boundary, but this implementation also changes total-failure semantics and adds env/logging surfaces that need tightening.
