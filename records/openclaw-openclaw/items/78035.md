## What This Changes

The PR changes memory-core supplement search to use `Promise.allSettled` with a per-supplement timeout and warning logs, adds helper and tool-path regression tests, and records a changelog fix.

## Best Possible Solution

Land a bounded memory-core partial-failure fix after live runtime proof is supplied and the timeout override is either documented or kept internal.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register one resolving and one rejecting memory corpus supplement, then call `memory_search` with `corpus="all"`; current main’s `Promise.all` rejects before returning the healthy supplement result.

## Solution Assessment

Yes, with one cleanup. Isolating each supplement inside memory-core with `allSettled` plus a bounded timeout is the narrow maintainable fix; the public timeout override should be documented or internalized before merge.
