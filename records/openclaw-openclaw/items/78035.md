## Summary

Keep this PR open because the linked memory supplement fan-out bug is real and still present on current main/latest release, but this branch is not merge-ready: it changes total supplement failure into a normal empty result, adds an undocumented timeout env surface, logs plugin error details without the shared formatter, and still only supplies test-output proof.

## What This Changes

This PR changes memory-core supplement search from fail-fast aggregation to settled per-supplement searches with timeout warning logs and adds helper/tool-path regression tests.

## Best Possible Solution

Keep the settled partial-success direction, but preserve unavailable output when no requested supplement succeeds, avoid or formally document and clamp the timeout env surface, use shared redacted error formatting, and require redacted non-test memory_search proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two corpus supplements, make one search reject, and current Promise.all rejects the full fan-out through memory_search corpus=wiki or corpus=all.

## Solution Assessment

No, not as-is. allSettled with bounded per-supplement handling is the right direction for partial successes, but this branch also changes total-failure semantics and adds env/logging surfaces that need tightening.
