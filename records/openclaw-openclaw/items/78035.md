## Summary

Keep this PR open: the linked memory-search bug is still present on current main and the latest release, but this branch is not merge-ready because it changes total supplement failure semantics, adds an undocumented timeout env surface, logs unredacted plugin error details, and still has only test-harness proof.

## What This Changes

The branch changes memory-core supplement search to settled per-supplement aggregation with a timeout, warning logs, and regression tests for rejecting or stalled supplements.

## Best Possible Solution

Keep the partial-success direction in searchMemoryCorpusSupplements, but preserve unavailable signaling for total requested-supplement failure, make the timeout policy internal or documented and bounded, redact warning details, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Current main awaits Promise.all over registered memory corpus supplements, and createMemorySearchTool calls that helper for corpus="wiki" and corpus="all", so one rejecting supplement reproduces the fail-fast behavior by source inspection.

## Solution Assessment

No, not as-is. Isolating supplement failures in searchMemoryCorpusSupplements is the right owner boundary, but this implementation also changes total-failure semantics and adds env/logging surfaces that need tightening.
