## Summary

## Best Possible Solution

Keep the Mattermost-local approach, but reset the boundary gate before awaiting detachment, add a regression where an update arrives while the prior boundary is pending, and refresh redacted proof on the final head.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Reset the boundary gate before awaiting detach:** `extensions/mattermost/src/mattermost/draft-stream.ts:78`
  - body: `noteBoundary()` leaves `hasStreamedContent` true while it awaits `forceNewMessage()`. In embedded/fire-and-forget paths, the next block can call `noteUpdate()` before that await finishes, and then this older boundary resumes and sets the flag to false, so the following boundary skips `forceNewMessage()` and edits the later block into the same Mattermost post. Clear the gate before the await and add a regression where an update arrives while the previous boundary is still pending.
  - confidence: 0.87
