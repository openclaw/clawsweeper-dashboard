## What This Changes

The PR changes Mattermost inbound draft preview delivery so existing `streaming.mode="block"` seals draft posts at text/reasoning/tool boundaries, honors `blockStreaming` when preview is off, and adds draft-stream regression tests.

## Best Possible Solution

Land the runtime fix with a small docs update that describes Mattermost block mode as boundary-split preview posts while keeping the default partial mode unchanged.

## Reproduction Assessment

Yes, source-backed: current main and v2026.5.26 force Mattermost inbound preview delivery through one editable draft post and disable block replies on that path. The PR body also provides before/after HTTP logs showing the failing 1 POST + 4 PUT pattern and the fixed 5 POST pattern.

## Solution Assessment

Mostly yes: reusing the existing `streaming.mode` contract and draft lifecycle boundary primitive is the narrow runtime fix. The solution should also update the Mattermost and shared streaming docs so the documented behavior matches the new delivery shape.
