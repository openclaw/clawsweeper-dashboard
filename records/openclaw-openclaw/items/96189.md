## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate pending timeouts through the terminal classifier:** `src/gateway/server-methods/agent-job.ts:486-487`
  - body: The fallback now forwards any pending timeout snapshot. Current terminal-outcome code keeps soft queue/gateway-draining and restart-cancelled provider-started snapshots correctable or cancelled, while the downstream subagent run-manager treats `endedAt` or `stopReason` as terminal wait evidence; forwarding them here can incorrectly wake or complete the parent as a timeout during retry grace. Reuse `terminalOutcomeFromSnapshot(...).reason === "hard_timeout"` and cover soft/restart cases.
  - confidence: 0.88
- **[P1] Keep synthesized notices from echoing statusLabel:** `src/agents/subagent-announce-delivery.ts:979-984`
  - body: The new fallback uses `statusHint` as the visible reason whenever the child result is non-empty and non-placeholder. Upstream, subagent error `statusLabel` is built as `failed: ${outcome.error}`, so an errored child with non-placeholder findings and no visible announce payload can send raw provider/tool error text to the external DM despite this branch's bounded/no-raw-output contract. Use closed reason values or explicitly redacted and capped text before sending this notice.
  - confidence: 0.87
