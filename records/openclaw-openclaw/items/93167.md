## What This Changes

The branch changes Unix process-tree termination to direct-PID kills by default, opts group-kill into `detached: true`, updates one exec timeout caller, and adjusts kill-tree tests.

## Best Possible Solution

Make non-detached kills safe while preserving group cleanup for detached process-group leaders, then add focused regression coverage and redacted real macOS or Unix process-tree proof.

## Reproduction Assessment

Yes at source level: current `main` and `v2026.6.6` default omitted detachment to negative-PID group kill, and the PR flips that default while several detached callers still omit detachment. I did not run the live macOS launchd gateway scenario.

## Solution Assessment

No as submitted: changing the shared default is a plausible core fix, but it is too broad unless all detached process-group callers are migrated or the helper guards group leadership.
