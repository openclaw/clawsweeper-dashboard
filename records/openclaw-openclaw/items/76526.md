## Summary

Close as not actionable for the OpenClaw source repo: the PR is a custom downstream Docker image/workspace seed, duplicates already-closed #76511, and introduces blocking build, startup, and security issues instead of a generic upstream Docker capability.

## What This Changes

The PR adds a custom Docker entrypoint, templates, Python dependency install, and gitlink-backed agent workspace to bake Telegram/Google/Outlook-specific first-run setup into the root image.

## Best Possible Solution

Keep the root Docker image generic; any reusable need should become a reviewed opt-in seeding mechanism with docs, tests, and no embedded personal workspace, unpinned dependencies, or provider-specific defaults.

## Reproduction Assessment

Yes for PR-introduced failures: source inspection shows the entrypoint exits unless custom Telegram, LLM, timezone, and user-name variables are set, while the current Docker path starts with --allow-unconfigured. The patch also calls envsubst without installing gettext-base by default.

## Solution Assessment

No. Baking a downstream Telegram/Gemini/Outlook workspace and startup policy into the upstream Dockerfile is not the narrow maintainable solution; a downstream image or generic opt-in seed hook is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Preserve unconfigured Docker startup:** `scripts/openclaw-entrypoint.sh:15-18`
  - body: The new entrypoint makes every container exit unless TELEGRAM_BOT_TOKEN, LLM_API_KEY, USER_TIMEZONE, and USER_NAME are set. That breaks the current documented Docker path, where the image starts with `--allow-unconfigured` and onboarding/config writes happen separately.
  - confidence: 0.96
- **[P1] Gate Python venv setup behind installed dependencies:** `Dockerfile:262-264`
  - body: This RUN now executes for every root Docker image, but the runtime image does not install python3-venv by default. The documented `docker build -f Dockerfile .` path will fail unless callers know to pass extra apt packages.
  - confidence: 0.94
- **[P1] Do not COPY a gitlink workspace:** `Dockerfile:270`
  - body: `openclaw-team-workspace` is added only as a subproject commit, while this COPY expects actual reviewed files in the build context. Normal checkouts will not have reproducible contents for this layer, and the root image would depend on external workspace state.
  - confidence: 0.96
- **[P2] Install envsubst before using the entrypoint:** `scripts/openclaw-entrypoint.sh:41`
  - body: The entrypoint calls `envsubst`, but the runtime image does not install gettext-base unless a caller supplies a custom apt build arg. Images can build but then fail immediately on first start.
  - confidence: 0.9
- **[P2] Generate seeded config with a JSON encoder:** `scripts/openclaw-entrypoint.sh:41`
  - body: The entrypoint writes JSON templates by substituting raw environment variables. Values containing quotes, backslashes, or newlines can corrupt the JSON or alter its structure; use the config CLI or a JSON-aware generator instead.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces Docker supply-chain and startup-policy concerns in addition to functional build failures.

Concerns:

- **[high] Unreviewed gitlink is copied into the image:** `Dockerfile:270`
  - body: The PR adds `openclaw-team-workspace` as a subproject commit and copies it into `/app/agent-workspace`, making the image depend on external workspace contents that are not present in the reviewed diff.
  - confidence: 0.96
- **[medium] Unpinned PyPI package is installed at build time:** `Dockerfile:263`
  - body: The Dockerfile installs `websockets` from PyPI without a version, hash, lockfile, or repository dependency policy, weakening the reproducibility and supply-chain posture of the root image.
  - confidence: 0.92
- **[medium] Raw envsubst writes secret-bearing config JSON:** `scripts/openclaw-entrypoint.sh:41`
  - body: The entrypoint substitutes environment variables directly into OpenClaw config and auth profile JSON, which can corrupt config structure and stores provider keys as inline plaintext rather than using existing config and SecretRef paths.
  - confidence: 0.86
- **[low] Insecure Control UI flag becomes a baked default:** `templates/openclaw.template.json:47`
  - body: The seeded config sets `gateway.controlUi.allowInsecureAuth` to true even though current audit code warns when that compatibility toggle is enabled.
  - confidence: 0.9
