## Summary

Keep open: current `main` still leaves isolated `deleteAfterRun` cron cleanup outside the run-final path, and the linked open implementation PR is the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue reports that isolated cron jobs with `deleteAfterRun: true` leave run-scoped session `.jsonl` files behind after completion.

## Best Possible Solution

Review and land or rework the linked cleanup PR so isolated cron run sessions are deleted from the run-final path while keeping cleanup scoped to cron run sessions.

## Reproduction Assessment

Yes. Source inspection shows an isolated `agentTurn` with `deleteAfterRun: true` and no direct delivery creates a run-scoped session, then exits through the runner `finally` without a cleanup call; I did not run a live gateway repro because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing `sessions.delete` cleanup from the isolated runner finalizer, scoped to cron session keys, rather than changing global retention or adding another cleanup policy.
