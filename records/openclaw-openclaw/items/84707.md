## What This Changes

The issue reports that isolated cron jobs with `deleteAfterRun: true` leave run session transcripts on disk after completion.

## Best Possible Solution

Review and land or rework https://github.com/openclaw/openclaw/pull/84794 so isolated cron run sessions are cleaned from the run-final path while broader hook/session lifecycle policy stays in its own issue.

## Reproduction Assessment

Yes, at source level. Current main creates run-scoped isolated cron sessions, but delivery-none jobs skip delivery cleanup and the runner finalizer disposes the run context without immediate delete-after-run cleanup.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing cron-scoped session deletion from the isolated runner finalizer, which is exactly the direction taken by the open linked PR.
