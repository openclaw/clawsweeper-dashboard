## What This Changes

The issue asks for `deleteAfterRun: true` on isolated cron `agentTurn` jobs to delete the per-run session transcript after every terminal run.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84794 or an equivalent runner-final cleanup with focused coverage for delivery-none and terminal-error isolated cron runs, then close this issue as fixed.

## Reproduction Assessment

Yes. Source inspection on current main shows isolated cron jobs create fresh run sessions, `delivery.mode: "none"` sets `deliveryRequested: false`, and `deleteAfterRun` deletion is only wired inside delivery-dispatch cleanup paths.

## Solution Assessment

Yes. The maintainable fix is to move or share the existing cleanup into an isolated-run finalization path, rather than changing global session retention or adding a new setting.
