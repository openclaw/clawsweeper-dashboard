## Summary

Keep open. Current main still has manual `/new`/`/reset`, idle/daily reset, context reporting, memory flush, and compaction paths, but not the requested config-backed percent-threshold rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions; the prior ClawSweeper comment reached the same keep-open conclusion.

## What This Changes

Add native, config-backed automatic context/session rollover for long-running Telegram/direct sessions, with threshold stages, durable handoff summaries, and fresh backing-session routing while preserving delivery identity.

## Best Possible Solution

Design an additive generic session lifecycle policy or SDK seam that validates rollover config, computes prompt pressure from fresh context snapshots, writes redacted durable handoffs, atomically rotates the backing `sessionId`, preserves logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, command registration, session initialization, compaction paths, context-engine contracts, tests, and docs.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new core/plugin seam appears safer than a ClawHub-only workaround because current hooks do not own the pre-model backing-session rotation and delivery rebinding requested here.
