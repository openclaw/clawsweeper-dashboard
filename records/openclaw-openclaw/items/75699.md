## Summary

Keep open. Current main has manual reset, status/context reporting, session reset policy, and compaction/successor transcript plumbing, but it still does not implement the requested config-backed context rollover policy, durable handoff command surface, or Telegram/direct automatic fresh-session routing contract.

## What This Changes

The issue requests native automatic context/session rollover with durable handoff summaries for long-running Telegram and other direct messaging sessions.

## Best Possible Solution

Design an additive generic session lifecycle policy that reuses existing compaction and successor-transcript plumbing while adding validated config, redacted durable handoff state, atomic/idempotent session-id rotation, delivery preservation, docs, and focused Telegram/direct tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from exact-name searches plus inspection of the session schema, typed config, command registry, inbound session routing, compaction paths, and docs.

## Solution Assessment

Unclear until maintainer design review. A native generic lifecycle policy appears safer than a Telegram-only workaround because the requested behavior crosses config validation, session-store mutation, delivery identity, transcript retention, and privacy boundaries.
