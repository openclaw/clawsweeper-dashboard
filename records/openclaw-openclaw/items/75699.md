## Summary

Keep open. Current main still has manual reset, daily/idle reset, status/context reporting, and compaction, but not the requested config-backed percent-threshold rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions.

## What This Changes

Request native session context rollover that monitors token pressure, writes durable handoffs, and rotates long-running direct-channel sessions without requiring manual `/new` or `/reset`.

## Best Possible Solution

Design an additive session lifecycle policy or explicit plugin API that validates per-channel/type thresholds, creates redacted durable handoffs, atomically rotates the backing `sessionId` while preserving `sessionKey` and delivery fields, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of the session schema, command registry, inbound session initialization, compaction docs/code, and session update path.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new core plugin seam appears safer than a ClawHub-only workaround because current surfaces do not own pre-model backing-session rotation with delivery-preserving handoff semantics.
