## Summary

Keep open. Current main still has manual `/new`/`/reset`, lazy daily/idle reset, `/compact`, token/context reporting, and compaction pressure handling, but not the requested config-backed percent-threshold rollover with durable handoff creation and fresh backing-session routing for Telegram or other direct sessions.

## What This Changes

The issue requests native automatic context-pressure session rollover with durable handoff summaries and preserved Telegram/direct delivery identity.

## Best Possible Solution

Design an additive core session lifecycle policy or generic plugin API seam that validates typed rollover config, calculates pressure from fresh prompt/context snapshots, creates redacted durable handoff artifacts, atomically rotates the backing `sessionId` while preserving the logical session key and delivery context, and injects the handoff into the next turn with focused generic-direct and fake Telegram tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main source/docs inspection of session config, session initialization, token/compaction docs, command registry coverage, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. The requested outcome likely belongs in a native lifecycle policy or a new generic lifecycle API rather than a ClawHub-only workaround, because the important operation is swapping the backing session before the model call while preserving delivery identity.
