## Summary

Keep open. Current main still has manual reset, lazy daily/idle reset, `/compact`, context/status reporting, and token-pressure compaction, but it does not implement config-backed percent-based session rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions.

## What This Changes

Request native, config-backed context/session rollover with durable handoffs and fresh backing-session routing for long-running Telegram/direct sessions.

## Best Possible Solution

Design an additive session lifecycle policy or plugin API seam that validates rollover thresholds, computes fresh context pressure, writes redacted durable handoffs, atomically rotates the backing `sessionId`, preserves logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, command registration, session initialization, compaction paths, Telegram routing, plugin hooks, and docs.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new core plugin seam appears safer than a ClawHub-only workaround because current public hooks do not own the requested pre-model backing-session rotation.
