## Summary

Keep open. Current main has manual reset, daily/idle reset, compaction, memory flush, and successor-transcript mechanics, but still lacks native percent-based context rollover with durable handoff and automatic fresh backing-session routing for Telegram/direct peers.

## What This Changes

The issue requests config-backed automatic context-usage rollover that creates durable handoffs and routes long-running Telegram/direct sessions into fresh backing sessions before context pressure becomes unsafe.

## Best Possible Solution

Design an additive generic session lifecycle policy or plugin API that validates thresholds, computes fresh context pressure, creates redacted durable handoffs, atomically rotates the backing sessionId while preserving logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, command registration, session initialization, compaction behavior/docs, Telegram session metadata, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. Existing compaction and daily/idle reset paths are not the requested durable handoff rollover, so the narrowest maintainable solution likely needs a core lifecycle policy or a documented plugin seam rather than a channel-local workaround.
