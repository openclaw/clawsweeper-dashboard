## Summary

Keep open. Current main still has manual resets, lazy daily/idle reset, `/compact`, context/status reporting, and token-pressure compaction, but it does not provide config-backed percent-based session rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions. The existing ClawSweeper comment reached the same keep-open conclusion, and the current checkout does not change that product/API gap.

## What This Changes

The issue requests native context-pressure session rollover with durable handoff summaries and automatic fresh backing sessions for Telegram/direct sessions.

## Best Possible Solution

Design an additive session lifecycle policy or a new generic session-lifecycle API that validates typed rollover config, computes pressure from fresh prompt/context snapshots, writes redacted durable handoff artifacts, atomically rotates the backing `sessionId` while preserving the logical session key and delivery context, injects the handoff into the next turn, and covers generic direct sessions plus fake Telegram direct metadata in tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main source/docs inspection of session config, command registry/docs, session initialization, token/compaction helpers, Telegram routing, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or a new generic session-lifecycle API appears safer than a ClawHub-only workaround with the current hooks, because the requested behavior must swap the backing session before the model call while preserving channel delivery identity.
