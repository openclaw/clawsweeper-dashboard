## Summary

Keep open. Current main still provides manual reset, lazy daily/idle reset, status/context reporting, memory flush, and compaction, but not native percent-based session rollover with durable handoff and fresh backing-session routing for Telegram/direct peers.

## What This Changes

Add config-backed automatic context-pressure rollover that creates durable handoff summaries while preserving Telegram/direct session delivery identity.

## Best Possible Solution

Design an additive generic session lifecycle policy or plugin API that validates thresholds, computes fresh prompt pressure, writes redacted durable handoffs, atomically rotates the backing `sessionId`, preserves logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, command registration, session initialization, compaction paths/docs, Telegram session helpers, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new core plugin seam appears safer than a Telegram-only workaround because current hooks do not own the requested pre-model backing-session rotation.
