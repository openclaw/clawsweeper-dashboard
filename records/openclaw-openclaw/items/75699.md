## Summary

Keep open. Current main still has manual reset, daily/idle reset, context/status reporting, and token-pressure compaction, but not config-backed percent-based session rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions.

## What This Changes

The issue requests native config-backed automatic context rollover with durable handoff summaries for Telegram and other long-running direct sessions.

## Best Possible Solution

Design an additive generic session lifecycle policy or plugin API that validates rollover config, computes prompt pressure from fresh context snapshots, writes redacted durable handoffs, atomically rotates the backing `sessionId`, preserves logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, docs, command registration, session initialization, compaction paths, Telegram routing, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new core plugin seam appears safer than a ClawHub-only workaround because current hooks do not own the pre-model backing-session rotation requested here.
