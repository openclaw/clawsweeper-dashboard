## Summary

Keep open. Current main has manual reset, lazy daily/idle reset, status/usage reporting, preflight compaction, and memory flush, but not the requested native percent-based context rollover with durable handoff and fresh backing-session routing for Telegram/direct sessions.

## What This Changes

Request native config-backed automatic context rollover that creates durable handoffs and moves long-running direct-channel sessions to a fresh backing session before context pressure becomes unsafe.

## Best Possible Solution

Design an additive generic session lifecycle policy or plugin API that validates thresholds, computes prompt pressure, writes redacted durable handoffs, atomically rotates the backing `sessionId`, preserves logical session/delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, command registration, session initialization, compaction docs, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new generic plugin seam appears safer than a Telegram-only workaround because current hooks do not own the requested pre-model backing-session rotation.
