## Summary

Keep open. Current main has reset policies, status/context reporting, auto/preflight compaction, memory flush, and successor transcripts, but it still lacks the requested native config-backed session context rollover policy, durable handoff command/config surface, and Telegram/direct fresh-session routing contract.

## What This Changes

Request native config-backed, percent-based context rollover with durable handoff and Telegram/direct fresh-session routing.

## Best Possible Solution

Design an additive generic session lifecycle policy that reuses existing compaction and successor-transcript plumbing while adding validated config, redacted durable handoff state, atomic session-id rotation, delivery preservation, docs, and focused tests.

## Solution Assessment

Unclear until maintainer design review. A native generic lifecycle policy appears safer than a Telegram-only workaround because the requested behavior crosses config validation, session-store mutation, delivery identity, transcript retention, and privacy boundaries.
