## Summary

Keep open. Current main still lacks native config-backed context-pressure rollover with durable handoff creation and automatic fresh backing-session routing for Telegram/direct sessions; the implemented paths cover manual reset, lazy idle/daily reset, context reporting, memory flush, and compaction instead.

## What This Changes

Request native percent-based session context rollover with durable handoff summaries and preserved Telegram/direct delivery identity.

## Best Possible Solution

Design an additive core session lifecycle policy or plugin API that validates rollover config, computes context pressure from reliable snapshots, writes redacted durable handoffs, atomically rotates the backing session id, preserves logical delivery identity, and injects the handoff before the next model turn.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from current-main inspection of config schema, command registration, session initialization, compaction, plugin hooks, Telegram routing, and relevant docs.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or explicit core plugin seam appears safer than a ClawHub-only workaround because current hooks do not own pre-model backing-session rotation and durable handoff injection.
