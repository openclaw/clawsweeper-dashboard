## Summary

Keep open. Current main still has manual/session-age resets, status/context reporting, and auto-compaction, but it does not implement config-backed percent-based context rollover, durable handoff files/state, or automatic fresh backing-session routing for Telegram/direct sessions.

## What This Changes

Request native config-backed automatic context/session rollover with durable handoff summaries and preserved Telegram/direct delivery identity.

## Best Possible Solution

Design an additive session lifecycle policy or explicit plugin SDK seam that validates typed rollover config, computes fresh context pressure before model dispatch, writes redacted durable handoff artifacts, atomically rotates the backing sessionId while preserving logical sessionKey and delivery context, and covers generic direct sessions plus fake Telegram direct metadata in tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main source/docs inspection of session config, command registration, session initialization, token/status reporting, compaction behavior, Telegram routing, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new generic pre-dispatch session lifecycle API appears safer than a ClawHub-only workaround with current hooks, because the requested behavior must rotate the backing session before the model call while preserving channel delivery identity.
