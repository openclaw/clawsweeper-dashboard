## Summary

Keep open. Current main still lacks the requested native percent-based session rollover, durable handoff config/commands, and Telegram/direct fresh-session routing contract; the existing reset, context reporting, and compaction paths are adjacent but do not satisfy the feature request.

## What This Changes

Add native, config-backed automatic context/session rollover for long-running Telegram/direct sessions, including threshold config, durable handoff summaries, commands, and fresh backing-session routing.

## Best Possible Solution

Design an additive generic session lifecycle policy that reuses existing compaction and session-store writer plumbing while adding validated config, redacted durable handoff state, atomic session-id rotation, Telegram/direct delivery preservation, docs, and focused tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainer design review. A native generic lifecycle policy appears safer than a Telegram-only workaround because the request spans config validation, session-store mutation, durable summaries, delivery identity, and privacy boundaries.
