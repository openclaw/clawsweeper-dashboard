## Summary

Keep open. Current main has adjacent reset, status/context reporting, auto/preflight compaction, memory flush, and successor transcript support, but it still lacks the requested native config-backed context-usage rollover policy, durable handoff command/config surface, and Telegram/direct fresh-session routing contract; related #74462 is narrower and still open, not a merged fix.

## What This Changes

Request native config-backed context-usage rollover for long-running Telegram/direct sessions, including threshold config, durable handoff summaries, fresh session-id routing, and status/manual handoff commands.

## Best Possible Solution

Design an additive generic session lifecycle policy that validates config, reuses compaction/successor transcript plumbing where appropriate, preserves delivery identity, redacts durable handoff artifacts, and documents/tests Telegram/direct routing.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection shows the requested config keys, command names, and threshold-based rollover path are absent while adjacent manual reset and compaction paths exist.

## Solution Assessment

Unclear until maintainer design review. A generic session lifecycle policy appears safer than a Telegram-only workaround because the request spans config validation, session-store rotation, durable handoff storage, delivery identity, and privacy redaction.
