## Summary

Keep open. Current main still supports manual reset, lazy daily/idle reset, token/context reporting, and compaction, but it does not implement config-backed percent-threshold session rollover with durable handoff creation and fresh backing-session routing for Telegram/direct sessions.

## What This Changes

The issue requests native automatic context-pressure rollover with durable handoff summaries and preserved Telegram/direct delivery identity.

## Best Possible Solution

Design an additive native lifecycle policy or generic session-lifecycle API that validates typed rollover config, computes pressure from fresh prompt/context snapshots, writes redacted durable handoff artifacts, atomically rotates the backing sessionId while preserving the logical session key and delivery context, and injects the handoff into the next turn with generic direct-session and fake Telegram tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main source/docs inspection of session config, session initialization, token/status reporting, command registration, compaction behavior, and Telegram routing metadata.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or a new generic lifecycle API appears safer than a ClawHub-only workaround because the core operation must swap the backing session before the model call while preserving channel delivery identity.
