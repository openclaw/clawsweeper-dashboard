## Summary

Keep open. Current main has manual `/new` and `/reset`, daily/idle session rollover, `/compact`, context/status visibility, and fresh token snapshots, but it still lacks the requested config-backed percent-threshold rollover with durable handoff summaries and automatic fresh backing-session routing for Telegram/direct sessions.

## What This Changes

The issue requests native automatic context-pressure rollover for long-running Telegram/direct sessions, including config thresholds, durable handoff summaries, dry-run/status commands, and safe preservation of delivery identity.

## Best Possible Solution

Design an additive session lifecycle policy or a first-class plugin API seam that computes rollover from fresh prompt/context snapshots, validates typed threshold config, creates redacted durable handoff artifacts, atomically rotates the backing `sessionId` while retaining the logical session key and delivery context, injects the handoff into the next turn, and covers generic direct sessions plus fake Telegram metadata in tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main inspection of session config, session initialization, token usage/status paths, compaction docs, Telegram context routing, and plugin hook contracts.

## Solution Assessment

Unclear until maintainer design review. A native lifecycle policy or new generic session-rollover API looks safer than a ClawHub-only workaround with current hooks because the feature must swap the active backing session before the model call while preserving delivery identity.
