## Summary

Keep open. Current main has manual reset, status reporting, auto/preflight compaction, and optional successor transcript rotation, but it still lacks the requested native session-level percent rollover policy, durable handoff command/config surface, and Telegram/direct fresh-session routing contract.

## What This Changes

Request native config-backed automatic context rollover that creates durable handoffs and routes Telegram/direct conversations into fresh backing sessions before context pressure becomes unsafe.

## Best Possible Solution

Design an additive generic session lifecycle policy that reuses existing compaction/successor-transcript plumbing while adding validated config, redacted durable handoff state, atomic session-id rotation, delivery preservation, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from exact name searches plus inspection of the session schema, command registry, inbound session routing, compaction paths, and docs.

## Solution Assessment

Unclear until maintainer design review. A native generic lifecycle policy appears safer than a Telegram-only workaround because the requested behavior crosses config validation, session-store mutation, delivery identity, transcript retention, and privacy boundaries.
