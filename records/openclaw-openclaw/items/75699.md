## Summary

Keep open. Current main has reset policies, manual session commands, auto/preflight compaction, successor transcripts, and Telegram delivery-route persistence, but it still lacks the requested config-backed context rollover policy, durable handoff command/config surface, and automatic Telegram/direct fresh-session routing contract.

## What This Changes

Requested native config-backed automatic context rollover with durable handoff and Telegram/direct fresh-session routing for long-running channel sessions.

## Best Possible Solution

Design an additive generic session lifecycle policy that reuses existing compaction and successor-transcript plumbing while adding validated config, redacted durable handoff state, atomic session-id rotation, delivery preservation, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from exact name searches plus inspection of the session schema, command registry, inbound session routing, compaction paths, Telegram routing, and docs.

## Solution Assessment

Unclear until maintainer design review. A native generic lifecycle policy appears safer than a Telegram-only workaround because the requested behavior crosses config validation, session-store mutation, delivery identity, transcript retention, and privacy boundaries.
