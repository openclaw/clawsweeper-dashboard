This PR should stay open for maintainer handling: it is maintainer-labeled and member-authored, and the useful build-log cleanup needs one scoping fix before merge because the new suppression would hide all matching CommonJS-dts warnings from the dts fake-js plugin, not only zod locale noise.

## Best Possible Solution

Keep the existing build graph and zod inlining, but filter only the known zod locale CommonJS-dts noise while forwarding all other dts warnings.

## Reproduction Assessment

Yes, from source inspection: current main bundles `zod/*`, the lockfile uses `rolldown-plugin-dts@0.25.1`, and upstream emits this warning for `export =` declarations such as zod locale `.d.cts` files. I did not run a local build because this review is read-only.

## Solution Assessment

No, not as written. Suppressing the noisy warning is the right direction, but the condition should also verify the known zod locale declaration path so other CommonJS-dts warnings remain visible.
