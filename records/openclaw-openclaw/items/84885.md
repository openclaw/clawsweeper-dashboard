## Summary

Keep open. Current main still contains the non-final Telegram draft overflow path that can rotate into retained visible preview messages, and the `beta-blocker` label requires explicit maintainer handling.

## What This Changes

Telegram partial streaming should keep at most one mutable non-final draft preview per active turn, then finalize or split only the durable final reply.

## Best Possible Solution

Update Telegram draft streaming so non-final partials coalesce or truncate into one editable preview, reserve chaining/retention for finalization, and add focused unit coverage plus live Telegram proof.

## Reproduction Assessment

Yes, source-level: current main still has a non-final oversized-preview rollover path and tests that expect multiple `sendMessage` calls. I did not run a live Telegram repro in this read-only pass.

## Solution Assessment

Yes, the suggested direction is the maintainable fix: gate rollover/retention on finalization and keep non-final partial previews to one mutable message while preserving long final splitting.
