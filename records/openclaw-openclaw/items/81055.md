## Summary

Close as implemented: current main routes the shared SecretRef diagnostics through stderr, so the reported `status --json` stdout pollution is fixed on main. The fix is current-main-only; no release tag contains it yet.

## What This Changes

Make `openclaw status --json` emit only parseable JSON on stdout when SecretRef diagnostics are produced.

## Best Possible Solution

Keep the current shared stderr routing for command SecretRef diagnostics, and handle the now-duplicative open fix PR separately as superseded by main.

## Reproduction Assessment

Yes for the reported pre-fix behavior: the released `v2026.5.12` source still sends `[secrets]` diagnostics through `runtime.log`, while `status --json` writes the JSON payload later. Current main no longer has that failing source path because the shared emitter now uses `runtime.error`.

## Solution Assessment

Yes. Moving the shared SecretRef diagnostic emission from stdout to stderr is the narrowest maintainable fix because it preserves operator diagnostics and keeps JSON stdout parseable for all callers using the shared resolver.
