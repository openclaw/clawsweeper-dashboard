## Summary

Keep open: current main still has a source-proven `status --json` stdout leak, and https://github.com/openclaw/openclaw/pull/81068 is an open closing PR that still needs repair before this issue can be closed.

## What This Changes

The issue reports that `openclaw status --json` can print `[secrets]` diagnostics to stdout before the JSON payload when SecretRef resolution falls back, breaking JSON parsers.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/81068 or a corrected equivalent lands a shared helper fix that preserves JSON-only stdout and keeps diagnostics on stderr or in `secretDiagnostics`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: unresolved SecretRef diagnostics can flow through `runtime.log` during `status --json` scan setup before the JSON payload is written.

## Solution Assessment

Yes, the issue’s requested direction is the right fix boundary: reserve stdout for the JSON document and preserve diagnostics through stderr or structured JSON. The active PR should avoid command-name heuristics and pass output mode or diagnostic stream explicitly.
