## What This Changes

Ensure `openclaw status --json` writes only parseable JSON to stdout when secret resolution emits diagnostics.

## Best Possible Solution

Land one narrow status JSON stdout fix that keeps `[secrets]` diagnostics off stdout while preserving them on stderr or in `secretDiagnostics`, with regression coverage for an unresolved SecretRef.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run `openclaw status --json` with a config containing an unresolved channel SecretRef, then parse stdout with `jq` or `JSON.parse`.

## Solution Assessment

Yes. The narrow maintainable fix is to keep command-secret diagnostics off stdout for JSON mode, preferably through stderr or the existing structured `secretDiagnostics` field, without changing the broader status contract.
