## Summary

Keep open: current main and the latest stable tag still have the source path where `status --json` can emit `[secrets]` diagnostics through `runtime.log` before writing the JSON payload, and open PRs already reference this issue as the fix path.

## What This Changes

Ensure `openclaw status --json` keeps stdout parseable when secret-resolution diagnostics are produced.

## Best Possible Solution

Review and land one focused linked PR that keeps secret diagnostics off `status --json` stdout while preserving them on stderr or in `secretDiagnostics`, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a temp config with an unresolved channel `SecretRef` makes `status --json` call secret resolution, emit `[secrets]` through `runtime.log`, and only later write the JSON payload.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable solution: keep `status --json` stdout JSON-only by routing these diagnostics to stderr or returning them structurally instead of logging them before payload write.
