## Summary

Keep open: current main still has a source-proven path where route-first `status --json` can emit `[secrets]` diagnostics through `runtime.log` before console capture is enabled, while the JSON payload is written later. The closed sibling report only documents the `models status --json` fix, not this status command path.

## What This Changes

The issue asks for `openclaw status --json` to keep stdout JSON-only when SecretRef or gateway diagnostics are produced.

## Best Possible Solution

Keep the route-first JSON stdout contract by ensuring secret diagnostics are routed to stderr or embedded only in `secretDiagnostics` before any `status --json` payload is written.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: route-first `status --json` can resolve secrets, call `runtime.log` for `[secrets]` diagnostics, and return before console capture is installed. A live validation should use a temp config with an unresolved SecretRef and pipe stdout to `jq` or `JSON.parse`.

## Solution Assessment

No, current main is not the best solved state because `routeLogsToStderr()` is set before routed execution but the console patch that honors it is installed only on the fallback path. The narrow fix should move capture/rerouting earlier for routed JSON commands or route these command-secret diagnostics through stderr/structured JSON directly.

## Repair Work Prompt

Fix the `openclaw status --json` stdout contract for unresolved SecretRef/gateway secret diagnostics. Current main can call `resolveCommandConfigWithSecrets(..., runtime)` during `status --json`, which logs `[secrets] ...` through `runtime.log` before the JSON payload; route-first execution returns before `enableConsoleCapture()` is installed, so `routeLogsToStderr()` does not keep those diagnostics off stdout. Make routed JSON command diagnostics stay off stdout, preferably by installing console capture before routed JSON execution or by routing command-secret diagnostics to stderr/structured `secretDiagnostics` without changing text-mode status behavior. Add focused regression coverage for `status --json` with secret diagnostics and for route-first logging/capture order. Do not add a new config option or change the documented `secretDiagnostics` JSON field.
