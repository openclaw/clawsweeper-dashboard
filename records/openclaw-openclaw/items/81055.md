## Summary

Close as implemented on current main: the former shared SecretRef diagnostic leak now writes through stderr, while `status --json` still returns diagnostics structurally in the JSON payload. The fix is current-main-only and is not included in the latest tagged release `v2026.5.12`.

## What This Changes

Ensure `openclaw status --json` keeps `[secrets]` diagnostics off stdout so JSON consumers can parse stdout directly.

## Best Possible Solution

Keep the current shared stderr routing and structured `secretDiagnostics` behavior, and treat the remaining open fix PR as superseded by the main-branch fix.

## Reproduction Assessment

Yes for affected release code: `v2026.5.12` still has the old `runtime.log` path that can precede JSON. Current main no longer has that failing source path because the same helper now writes diagnostics to `runtime.error`.

## Solution Assessment

Yes. Moving the shared command SecretRef diagnostic emission off stdout is the narrow maintainable fix because it preserves diagnostics while keeping JSON stdout parseable across callers using the helper.
