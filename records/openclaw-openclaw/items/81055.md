## Summary

Keep open: current main still has a source-proven `status --json` path where unresolved SecretRef diagnostics are logged before the JSON payload, so the closed sibling https://github.com/openclaw/openclaw/issues/72962 does not fully cover this command path.

## What This Changes

The issue asks `openclaw status --json` to keep stdout parseable when command-secret diagnostics are produced.

## Best Possible Solution

Keep the route-first JSON stdout contract by moving these diagnostics to stderr or retaining them only in structured `secretDiagnostics` before any `status --json` payload is written.

## Reproduction Assessment

Yes at source level: route-first `status --json` can resolve secrets, call `runtime.log` for `[secrets]` diagnostics during the scan, and write JSON only afterward. The reporter also supplied a concrete stdout capture from released builds, but this review did not run a live temp-config reproduction.

## Solution Assessment

Yes, the requested stdout contract is the right fix boundary. The narrow implementation should reroute or structure command-secret diagnostics for routed JSON commands without changing gateway probe semantics or adding a new option.
