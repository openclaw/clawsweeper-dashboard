## Summary

Keep open. Current `main` still has a narrow route-first stdout gap: `runCli` tries routed commands before console capture is enabled, so the existing JSON-mode stderr flag does not protect `runtime.log` diagnostics emitted during `status --json` secret resolution.

## What This Changes

The issue reports that `openclaw status --json` can emit `[secrets]` diagnostics to stdout before the JSON payload when command SecretRef resolution degrades, breaking JSON parsers.

## Best Possible Solution

Keep `status --json` stdout byte-for-byte reserved for the JSON document by making routed JSON command diagnostics honor stderr before route handlers run, while preserving `secretDiagnostics` in the JSON payload.

## Reproduction Assessment

Yes. A config with an unresolved channel SecretRef such as `channels.discord.token` can drive the current `status --json` route into `resolveCommandConfigWithSecrets`, whose diagnostics use `runtime.log` before route-first console capture is installed.

## Solution Assessment

Yes, the best fix is a narrow routing/logging correction for routed JSON commands or command-secret diagnostics. It should not add a config option or change SecretRef resolution semantics.
