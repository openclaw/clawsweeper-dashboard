## Summary

Current main now fixes the reported stdout pollution: command SecretRef diagnostics go through `runtime.error` while the JSON payload still uses the stdout JSON writer. The fix is not in `v2026.5.12`; it is current-main-only as of `e3621f5057f4a53b5fe860bb12af2ffb70054417`.

## What This Changes

The issue reports that `openclaw status --json` can emit `[secrets]` diagnostics on stdout before the JSON document when SecretRef resolution degrades.

## Best Possible Solution

Keep the current-main fix and close the issue as implemented; handle the still-open linked PR as superseded by the direct main commit.

## Reproduction Assessment

Yes for the original bug: the affected release source and issue capture show SecretRef diagnostics went through `runtime.log` before JSON output. Current main no longer has that stdout path because the helper now calls `runtime.error`.

## Solution Assessment

Yes. Routing these command SecretRef diagnostics to stderr while preserving `secretDiagnostics` in the JSON payload is a narrow maintainable fix for parseable `--json` stdout.
