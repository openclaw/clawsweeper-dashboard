## Summary

Current main implements the requested fix: command SecretRef diagnostics now go through stderr, and regression coverage asserts the helper no longer writes those diagnostics through `runtime.log` stdout.

## What This Changes

The issue reports that `openclaw status --json` can emit `[secrets]` diagnostics on stdout before the JSON payload when SecretRef resolution degrades.

## Best Possible Solution

Keep the current-main stderr routing fix and close this issue as fixed on main; handle the remaining linked PR separately as superseded duplicate work.

## Reproduction Assessment

No current-main failing reproduction remains: source inspection shows the original `[secrets]` emission path now calls `runtime.error`, and the regression test asserts `runtime.log` is not called. The original bug was source-reproducible in the latest release tag, which still has the old `runtime.log` path.

## Solution Assessment

Yes. Routing the shared command SecretRef diagnostics to stderr is a narrow maintainable fix for parseable JSON stdout, while the structured `secretDiagnostics` data remains available to JSON consumers.
