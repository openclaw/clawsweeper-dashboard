## Summary

Keep open: current main still has a source-proven path where `status --json` secret diagnostics can be written to stdout before the JSON payload, and multiple open PRs already reference this issue with closing syntax.

## What This Changes

Ensure `openclaw status --json` keeps `[secrets]` diagnostics off stdout so stdout remains parseable JSON for `jq` and `JSON.parse` consumers.

## Best Possible Solution

Land one focused fix that preserves `secretDiagnostics` while routing or suppressing human `[secrets]` diagnostics before JSON stdout, then let the merged PR close this issue.

## Reproduction Assessment

Yes. Source inspection on current main shows `status --json` passes the runtime into secret resolution, diagnostics use `runtime.log`, and JSON is written after the scan; the reporter also supplied a concrete unresolved SecretRef pipe-to-JSON-parser scenario.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: keep diagnostics on stderr or in structured JSON, not as pre-payload stdout text. The remaining maintainer action is to review and land one of the open linked fixes.
