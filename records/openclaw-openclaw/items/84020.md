## Summary

Keep open. Current main still reloads file-backed history on `session.message` instead of applying the live payload, and external-channel assistant/tool progress remains behind an explicit Control UI visibility guard; the new reporter comment adds live release evidence that matches this source path.

## What This Changes

Request immediate Control UI visibility for Telegram inbound user turns and incremental in-progress agent events instead of waiting for the end-of-turn session commit.

## Best Possible Solution

Split the work: make inbound user messages durable or directly consumable by the Control UI immediately via the existing session-message contract, then handle Telegram assistant/tool progress as an explicit visibility and privacy decision before widening hidden-run fanout.
