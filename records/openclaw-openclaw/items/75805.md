## Summary

Keep open: current main still does not cover the reported current-process lock payload with a valid `starttime`, and an open PR already uses closing syntax for this issue, so the issue should remain open until that implementation is reviewed and landed or superseded.

## What This Changes

The issue requests recovery for leaked `.jsonl.lock` session files after gateway shutdown or restart so OpenClaw does not busy-wait and require manual lock deletion.

## Best Possible Solution

Review and land PR #75822 or an equivalent narrow fix that reclaims untracked self-owned locks with matching `starttime` while preserving active locks and existing stale-lock startup cleanup.

## Reproduction Assessment

## Solution Assessment
