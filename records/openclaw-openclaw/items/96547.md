## Summary

Keep open: the force-pushed direction is safer than changing the persisted directory encoding, but the current patch still checks only the single newest file, so a newer mismatched session can make `continueRecent` start fresh even when an older valid session exists for the requested cwd. The PR body proof is also stale because it still describes the removed injective-encoding approach.

## What This Changes

The PR preserves the shipped default session directory encoding, adds cwd verification to `continueRecent`, filters `SessionManager.list(cwd)` by header cwd, and adds session-manager tests.

## Best Possible Solution
