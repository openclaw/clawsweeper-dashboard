## Summary

Keep open. Current main still lacks a per-wrapper cap/backoff/circuit breaker for consecutive LLM idle timeouts, and an open PR already closes this issue with a targeted stream-wrapper fix, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

Request bounded retry behavior for LLM idle timeouts so heartbeat/agent runs cannot repeatedly issue paid model calls after a silent provider stall.

## Best Possible Solution

Land a reviewed wrapper-level circuit breaker or equivalent bounded retry guard that stops repeated paid `baseFn` calls after consecutive idle timeouts while preserving slow streams that keep producing chunks.
