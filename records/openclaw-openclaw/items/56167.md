## Summary

Keep this PR open. Current main still lacks Feishu outbound-level `NO_REPLY` suppression, so the PR is useful, but the proposed patch is incomplete because `sendMedia` can still fall through to the no-media text fallback and send the sentinel; it also needs the required changelog entry for a user-facing Feishu fix.

## What This Changes

The PR imports `isSilentReplyText` into the Feishu outbound adapter, suppresses exact silent tokens in `sendText` and the first `sendMedia` text branch, and adds outbound tests for exact, whitespace-padded, substantive, and media-caption `NO_REPLY` cases.

## Best Possible Solution

Land one Feishu-owned fix that suppresses exact silent tokens at every Feishu outbound text-send boundary while preserving media attachments and substantive text, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `feishuOutbound.sendMedia` with `text: "NO_REPLY"` and no `mediaUrl`; in the proposed patch the new leading text guard skips the first send, but the no-media fallback still sends the raw text.

## Solution Assessment

No. The PR uses the right SDK helper and extension boundary, but it is not the best complete fix until the fallback and relevant caption/comment paths are guarded and tested.
