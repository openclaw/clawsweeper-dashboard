## Summary

Keep open: this PR is not merge-ready because it adds broad Hermes-specific core/protocol plumbing that conflicts with the current media completion contract and still needs a validated metadata trust boundary plus live metadata-delivery proof.

## What This Changes

The PR adds Hermes arbiter metadata plumbing through Gateway send, outbound hook metadata, task/exec/media direct-send paths, generated Swift models, and targeted tests.

## Best Possible Solution

Define a generic, validated hook metadata contract with maintainer-approved caller policy, keep media completions agent-mediated, move Hermes-specific routing into plugin/config-owned code, then add changelog and live metadata-delivery proof.

## Reproduction Assessment

Yes, for the review findings: source inspection shows current docs/tests keep media completions agent-mediated, while the PR patch switches the deprecated direct-send path to immediate direct delivery and adds arbitrary Gateway metadata. I did not run a live Hermes setup.

## Solution Assessment

No. The proposed Hermes-specific core/protocol path is broader than the maintainable boundary; a generic validated metadata contract plus plugin/config-owned Hermes routing is safer.
