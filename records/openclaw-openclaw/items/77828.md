## Summary

This PR should stay open, but the patch is not mergeable as-is because it conflicts with current async media delivery behavior, exposes unvalidated Gateway send metadata to hooks, and hard-codes Hermes routing details in core paths.

## What This Changes

The PR adds Hermes arbiter metadata plumbing through Gateway send, outbound hook metadata, task/exec/media delivery paths, Swift protocol models, and tests.

## Best Possible Solution

Design a generic, validated hook metadata contract with maintainer-approved namespace and caller policy, keep async media completions agent-mediated, then add docs/changelog and live Gateway/Hermes proof.

## Reproduction Assessment

Yes, by source inspection against current main: current docs and tests require async media completions to stay requester-session mediated, while the PR head returns after direct delivery when `directSend` is true. I did not run the PR branch in a live Hermes setup.

## Solution Assessment

No. The proposed Hermes-specific core/protocol plumbing is broader than the maintainable boundary; the safer path is a generic validated metadata contract that preserves current media completion behavior.
