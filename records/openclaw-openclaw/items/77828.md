## Summary

Keep open: this external feature PR is not merge-ready because it revives deprecated direct async media completion delivery, adds a new unvalidated send-metadata hook path, and provides no required real behavior proof.

## What This Changes

The PR adds Hermes arbiter metadata plumbing through gateway send, outbound message hooks, task/media/exec follow-up delivery, Swift protocol models, and related tests.

## Best Possible Solution

Keep async media completions agent-mediated, design Hermes arbiter metadata as an explicit validated contract, and require real gateway/media/task proof before merge.

## Reproduction Assessment

Yes, for the introduced regression by source inspection: the patch returns early through a direct media send when the deprecated `directSend` flag is true, while current docs and types say completions stay requester-session mediated. I did not run the PR branch in a live setup.

## Solution Assessment

No. The PR’s broad core/protocol approach and hard-coded Hermes defaults are not the narrowest maintainable path; a safer fix would validate and document a generic hook metadata contract without re-enabling deprecated direct-send behavior.
