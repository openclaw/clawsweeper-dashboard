## Summary

The PR should stay open because it introduces a broad Hermes arbiter metadata path, but the current patch crosses core/plugin ownership boundaries, changes deprecated media completion behavior, and reroutes direct-mode sends through gateway when metadata is present.

## What This Changes

The PR adds optional outbound metadata to gateway send/protocol models, introduces Hermes arbiter metadata builders, and attaches that metadata to task updates, exec approval followups, and async media completion sends.

## Best Possible Solution

Define a generic, documented metadata hook contract only if maintainers want that surface, keep Hermes topic/bot/routing policy in a Hermes-owned plugin or configuration layer, and preserve agent-mediated async media completion unless an explicit upgrade-safe opt-in is approved.
