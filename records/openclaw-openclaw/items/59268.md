## Summary

Keep open. Current main still defaults macOS WebChat and the session picker to the gateway main session, while existing node identity and node-scoped session paths are available but not used for the local paired Mac default.

## What This Changes

The issue asks for macOS chat picker/session selection to default to the local paired Mac instead of sharing or labeling another Mac’s node session.

## Best Possible Solution

Implement a remote-mode macOS WebChat default that resolves to the local paired node session and local Mac display name, while preserving local-mode WebChat on the selected agent’s main session.

## Reproduction Assessment

Yes, at source level. Current main still routes macOS WebChat startup, history/send canonicalization, and picker defaults through the shared gateway main session instead of a local node-derived session key.

## Solution Assessment

Yes directionally, but not by changing only the static client id. The maintainable fix should use existing paired device/node identity and the gateway’s node-scoped session convention for remote-mode macOS WebChat defaults.
