## Summary

Keep open. Current main still routes macOS WebChat startup and picker defaults through the shared gateway main session instead of the local paired-node session, while existing gateway/node plumbing already has device identity and node-scoped session conventions to build on.

## What This Changes

The issue asks for each paired macOS app to default to its own node chat session and label instead of sharing or swapping sessions across multiple Macs.

## Best Possible Solution

Use the existing device/node identity and node-session convention to make remote-mode macOS WebChat default to the local paired-node session and label, while preserving local-mode main-session behavior.

## Solution Assessment

Yes directionally, but not by changing only the static client id. The maintainable fix should resolve remote-mode macOS WebChat defaults from the paired local node identity and preserve local-mode main-session behavior.
