## Summary

Keep open. Current main still opens macOS WebChat on the shared gateway main session, while node/device identity and node-scoped session conventions exist elsewhere but are not used to choose the macOS chat picker default.

## What This Changes

The issue asks the macOS app to default WebChat to the local paired Mac's own node session and label instead of sharing or swapping sessions across multiple paired Macs.

## Best Possible Solution

Implement a remote-mode macOS WebChat default that resolves to the local paired node session and local Mac display name, while preserving local-mode WebChat on the selected agent's main session.

## Reproduction Assessment

Yes, at source level. Current main still routes macOS WebChat startup and picker defaults through the shared gateway main session, which is enough to reproduce the shared-session half of the reported two-Mac behavior without a live lab run.

## Solution Assessment

Partly. The requested per-node behavior is the right direction, but the implementation should use existing device/node identity and a node-scoped session default rather than adding another static-client-id workaround.
