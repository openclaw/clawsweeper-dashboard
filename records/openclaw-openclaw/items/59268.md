## Summary

Keep open. Current main and the latest v2026.5.18 release still make macOS WebChat open and list sessions through the gateway-wide main session, so the multi-Mac label/session collision remains source-reproducible.

## What This Changes

The issue asks for macOS WebChat to default each paired Mac to its own node-scoped chat session and display the correct node label when multiple Macs share one gateway.

## Best Possible Solution

Use the existing device/node identity and node-session convention so remote-mode macOS WebChat defaults to the local paired node session and label, while local-mode WebChat keeps using the selected agent main session.

## Reproduction Assessment

Yes, at source level. Opening macOS WebChat still resolves the shared gateway main session and the picker rewrites defaults to that same shared key, while existing node-event plumbing has a node-${nodeId} convention the macOS picker does not use.

## Solution Assessment

Yes directionally, but not by changing only the static client id. The maintainable fix should resolve the remote-mode chat default from the paired local node identity and keep local-mode main-session behavior intact.
