## Summary

Keep open. Current main still starts macOS WebChat and orders the session picker from the gateway main session, while paired-device/node identity and node-scoped session conventions exist but are not used for the local paired Mac default.

## What This Changes

The issue asks for macOS WebChat to default each paired Mac to its own node-labeled, isolated chat session instead of sharing or showing another Mac's session label.

## Best Possible Solution

Implement a remote-mode macOS WebChat default that resolves to the local paired node session and local Mac display name, while preserving local-mode WebChat on the selected agent's main session and adding focused Swift/gateway regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still routes macOS WebChat startup, history/send canonicalization, and picker defaults through the shared gateway main session instead of a local paired-node session key.

## Solution Assessment

Yes directionally, but not by changing only the static client id. The maintainable fix should use existing paired device/node identity and the gateway node-scoped session convention for remote-mode macOS WebChat defaults.
