## What This Changes

Request that inbound dispatch avoid reloading and re-registering runtime plugins when the gateway startup registry already satisfies `ensureRuntimePluginsLoaded`.

## Best Possible Solution

Keep the compatibility-checked startup-scope reuse path, then have maintainers reconcile PR #74118 as redundant, incomplete, or still useful before closing this issue.

## Reproduction Assessment

Yes. The source-level reproduction path is clear: inbound dispatch calls `ensureRuntimePluginsLoaded`, which previously reached registry resolution with a narrower option set than gateway startup; current main has focused tests for the compatible reuse path that prevents the rebuild.

## Solution Assessment

Yes for current main's direction. Compatibility-checked reuse is safer than an unconditional active-registry fast path because it still rejects mismatched plugin scope, workspace/config shape, and explicit subagent runtime requests.
