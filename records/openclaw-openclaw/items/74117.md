## What This Changes

The issue asks for inbound dispatch to avoid rebuilding and re-registering runtime plugins when a gateway already has an active plugin registry.

## Best Possible Solution

Keep the current scoped registry compatibility implementation on main and close the still-open linked PR as superseded or review it only for any narrower missing edge case.

## Solution Assessment

Yes. The current main fix is more conservative than an unconditional active-registry fast path because it preserves cache compatibility checks while allowing the valid startup-scoped gateway registry to be reused.
