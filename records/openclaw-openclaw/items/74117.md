## Summary

Keep open. Current main appears to address the reported rebuild path with compatibility-checked startup registry reuse, but the same-author closing PR remains open and should be reconciled before this issue is closed.

## What This Changes

The issue reports that inbound dispatch repeatedly rebuilds and re-registers runtime plugins because `ensureRuntimePluginsLoaded` used load options that did not match the gateway startup registry cache key.

## Best Possible Solution

Keep the compatibility-checked startup registry reuse on main, reconcile PR #74118 as redundant or still-needed, and close this issue only after the maintainer accepts the final implementation path.

## Reproduction Assessment

Yes. The original path is reproducible by source tracing from inbound dispatch through `ensureRuntimePluginsLoaded` into runtime registry resolution, and current tests cover the compatible active-registry reuse that prevents the rebuild.

## Solution Assessment

Yes. Current main's compatibility-checked reuse is safer than an unconditional active-registry short-circuit because it preserves workspace, plugin-scope, gateway-method, and runtime-mode checks before reusing a registry.
