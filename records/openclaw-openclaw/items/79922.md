## Summary

Keep open: this PR targets a valid diagnostics bug, but the live head still skips liveness sampling during the startup grace window and the supplied real-behavior proof is narrative-only.

## What This Changes

The PR adds a diagnostic heartbeat startup grace option, passes a 60 second grace from gateway startup, and adds a regression test for suppressing startup liveness warnings.

## Best Possible Solution

Keep the internal 60 second grace, always run the liveness sampler each heartbeat tick, gate only liveness event and warning emission during grace, and add a regression assertion that sampleLiveness is called during grace.

## Reproduction Assessment

Yes. Source inspection of current main shows gateway startup has no diagnostic liveness grace and the warning path emits when a liveness sample exists with open diagnostic work; a focused fake-timer test with queued work can reproduce the warning path.

## Solution Assessment

No. The PR adds the right internal grace boundary, but the current head gates by skipping sampleLiveness; the maintainable fix is to compute a raw sample every tick and suppress only event/warning emission while grace is active.
