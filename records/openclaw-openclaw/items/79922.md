## Best Possible Solution

Keep the PR open and update it to always call the liveness sampler during startup grace while suppressing only liveness event/WARN emission, with regression coverage and auditable redacted startup output.

## Solution Assessment

No. The requested direction is right, but the current PR implementation skips the sampler during grace; the safer fix is the prior shape that samples every tick and gates only event/WARN emission.
