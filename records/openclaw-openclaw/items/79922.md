## Best Possible Solution

Keep the PR open, change the implementation to sample every heartbeat and suppress only liveness event/WARN emission during grace, then add auditable redacted startup output before merge.

## Solution Assessment

No. The direction is right, but the current PR gates before sampling; the narrower maintainable fix is to always call the sampler and gate only event/WARN emission during the grace window.
