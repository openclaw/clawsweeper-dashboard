## Summary

Keep open: the PR addresses a valid current-main diagnostic false-positive path, but the latest head still skips the liveness sampler during startup grace and the after-fix proof is not auditable.

## What This Changes

This PR adds a `startupGraceMs` diagnostic heartbeat option, passes a 60 second grace window from gateway startup, and adds a focused liveness-warning test.

## Best Possible Solution

Keep the startup-grace direction, but run the liveness sampler every heartbeat and suppress only liveness event/WARN emission during grace, with a test that asserts sampling still occurs during the grace tick.

## Reproduction Assessment

Yes. Current main source shows gateway diagnostics start without startup grace, and the heartbeat path emits liveness warnings from a non-null liveness sample while work is queued; I did not run runtime tests because this review is read-only.

## Solution Assessment

No. The fix direction is right, but this patch gates before sampling; the narrower maintainable fix is to sample every tick and gate only event/WARN emission during the grace window.
