## Summary

Keep this PR open. Current main and the latest release still lack a matcher for Groq's `attempted to call tool ... which was not in request.tools` error, so the proposed formatter change remains useful; the only review issue is that the added test does not prove the captured-tool sandbox rewrite path.

## What This Changes

This PR adds a Groq-specific unknown-tool regex in `formatAssistantErrorText` and a unit test for the reported Groq validation error string.

## Best Possible Solution

Land the narrow formatter matcher with a regression test that passes sandbox config and a sandboxed session where `gog` is blocked, then assert the blocked-tool guidance rather than only the raw fallback text.

## Solution Assessment

Yes, with a test adjustment. Adding a provider-specific matcher in `formatAssistantErrorText` is the narrowest maintainable fix for this error format; the safer test is to assert the sandbox-policy rewrite so the matcher is proven end to end.
