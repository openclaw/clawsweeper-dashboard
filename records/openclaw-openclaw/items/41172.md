## What This Changes

The PR adds a Groq-specific unknown-tool regex to `formatAssistantErrorText` and a formatter test for the bare attempted-call error string.

## Best Possible Solution

Land a narrow matcher that extracts the leading tool name from both bare and argument-bearing Groq validation strings, with sandbox-policy regression coverage and redacted live Groq proof.

## Reproduction Assessment

Yes, at source level. Current main's unknown-tool regex chain cannot match the Groq `attempted to call tool ... which was not in request.tools` format, and the PR regex can be checked directly against known bare and argument-bearing strings.

## Solution Assessment

No, not as written. `formatAssistantErrorText` is the right narrow seam, but the matcher needs to handle inline argument forms and the test needs sandbox config so it proves blocked-tool guidance rather than raw text passthrough.
