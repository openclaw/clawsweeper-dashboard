## Summary

Keep this PR open: it targets a real formatter gap on current main, but the branch is not merge-ready because the matcher misses known argument-bearing Groq validation strings and the included test does not prove the sandbox rewrite behavior.

## What This Changes

The PR adds a Groq-specific unknown-tool regex in `formatAssistantErrorText` and a formatter test for the bare attempted-call error string.

## Best Possible Solution

Land a narrow matcher that extracts the leading tool name from both bare and argument-bearing Groq validation strings, with sandbox-policy regression coverage and redacted live Groq proof.

## Reproduction Assessment

Yes, at source level. Current main's regex chain cannot match the Groq `attempted to call tool ... which was not in request.tools` format, and related Groq reports provide argument-bearing strings that the PR regex still misses.

## Solution Assessment

No, not as written. `formatAssistantErrorText` is the right narrow seam, but the matcher needs to handle inline arguments and the test needs sandbox config so it proves blocked-tool guidance rather than raw text passthrough.
