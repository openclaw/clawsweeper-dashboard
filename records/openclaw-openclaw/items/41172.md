## What This Changes

The PR adds a Groq-specific unknown-tool validation error matcher in assistant error formatting and adds a formatter test for that message.

## Best Possible Solution

Land a narrow current-main-compatible matcher that extracts the leading tool name from Groq attempted-call validation strings, including inline argument forms, with sandbox-policy regression coverage and redacted live Groq proof.

## Reproduction Assessment

Yes, at source level. Current main's regex chain cannot match the Groq `attempted to call tool ... which was not in request.tools` format, and related Groq reports show both bare and argument-bearing forms; I did not run a live Groq request.

## Solution Assessment

No, not yet. `formatAssistantErrorText` is the right seam, but this PR's matcher only handles bare names and the added test would still pass on raw error text without proving sandbox extraction or blocked-tool guidance.
