## What This Changes

The PR changes memory-wiki source ingest to read an existing generated source page, preserve its human Notes block during re-ingest, and add regressions for marker-like content in sources and notes.

## Best Possible Solution

Preserve the human Notes marker pair associated with the generated Notes section regardless of marker comments or headings inside the human note body, with a focused regression before merge.

## Reproduction Assessment

Yes. Re-ingesting the same source after editing the generated Notes block is a high-confidence source reproduction because current main and v2026.6.9 write the regenerated empty block directly; I did not execute tests in this read-only review.

## Solution Assessment

No. `ingestMemoryWikiSource` is the right boundary, but the current helper still anchors to the last `## Notes` heading and can miss the real block when note text contains that heading.
