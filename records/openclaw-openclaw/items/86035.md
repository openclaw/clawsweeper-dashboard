## Summary

Keep open. Current main and the latest release still use a controlled Control UI chat textarea that syncs draft state on every input event while only guarding Enter keydown for IME composition, so the reported CJK composition break remains actionable.

## What This Changes

The issue requests an IME-safe Control UI/WebChat composer so CJK composition is not interrupted by per-keystroke controlled draft syncing.

## Best Possible Solution

Make the WebChat composer IME-aware by suppressing controlled draft sync while composition is active, committing on compositionend, preserving resize/slash-menu behavior, and adding focused tests plus macOS browser proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.22 still bind the textarea value to draft state and call onDraftChange during every input event while only guarding Enter keydown. I did not live-run the macOS Chrome IME scenario in this read-only pass.

## Solution Assessment

Yes, the reported direction is the narrowest maintainable fix: keep the composer controlled for ordinary input, but defer draft-state synchronization during IME composition and commit once composition ends.
