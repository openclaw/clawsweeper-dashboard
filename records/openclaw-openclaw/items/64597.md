## Summary

Keep open: the PR still has a P1 recovery-path regression because the new skip gate can suppress the existing truncation fallback using stale run-loop prompt usage, and the external proof gate is still not met.

## What This Changes

The PR changes embedded-agent overflow recovery to skip tool-result truncation when estimated reducible output cannot clear the overflow, adds loop-harness coverage, and narrows auto-reply reset behavior to embedded compaction failures.

## Best Possible Solution

Revise the overflow skip gate to use current-attempt observed or estimated overflow pressure, or preserve fallback truncation whenever usage provenance is uncertain; coordinate with https://github.com/openclaw/openclaw/pull/81190 and split the auto-reply reset policy if maintainers want it independently.

## Reproduction Assessment

Yes. Source inspection of PR head shows the skip gate derives overflow size from run-loop-scoped `lastRunPromptUsage` and then marks truncation attempted, while current main still attempts fallback truncation when oversized tool output is present.

## Solution Assessment

No. Avoiding futile truncation is reasonable, but this implementation should use attempt-local overflow evidence or keep the fallback when usage provenance is uncertain; the auto-reply reset behavior should not ride along with a blocked recovery change unless maintainers explicitly want that coupling.
