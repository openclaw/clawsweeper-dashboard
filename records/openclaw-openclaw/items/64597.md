## What This Changes

The branch adds a reduction-potential gate around overflow tool-result truncation, a one-shot truncation regression test, and narrows auto-reply session resets to embedded compaction_failure errors.

## Best Possible Solution

Revise the overflow skip gate to use attempt-local prompt pressure or preserve fallback truncation when usage provenance is uncertain; keep the auto-reply reset narrowing only if maintainers still want it.

## Reproduction Assessment

Yes for the review finding. Source inspection shows current main keeps lastRunPromptUsage at run-loop scope while the PR head uses it to skip truncation for the current overflow attempt.

## Solution Assessment

No. Avoiding futile truncation is a reasonable direction, but this implementation should use current-attempt prompt pressure or fall back to truncation when usage provenance is uncertain.
