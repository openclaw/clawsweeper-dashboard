## Summary

Keep open: the branch still introduces a source-reproducible regression because the new truncation skip gate uses loop-level prompt usage that can be stale, and the PR still lacks after-fix real behavior proof.

## What This Changes

This PR gates embedded-runner overflow recovery by estimated tool-result reduction, adds one-shot truncation coverage, and changes auto-reply to reset only on embedded compaction_failure errors.

## Best Possible Solution

Keep the reduction-potential gate only when prompt pressure is known to be attempt-local; otherwise preserve the existing truncation fallback and keep the auto-reply reset narrowing if it remains covered.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: carry a high prior lastRunPromptUsage, then hit a current overflow with oversized tool output and no fresh usage, causing the PR skip gate to bypass truncation that current main would attempt.

## Solution Assessment

No. The direction is narrow and maintainable, but the implementation should use current-attempt prompt pressure or fall back to truncation when usage provenance is uncertain.
