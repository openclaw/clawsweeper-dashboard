The branch adds an estimated tool-result reduction gate for overflow recovery, one-shot truncation coverage, and narrows auto-reply session resets to embedded compaction_failure errors.

## Best Possible Solution

Revise the skip gate to use attempt-local prompt pressure or preserve fallback truncation when usage provenance is uncertain, then provide redacted real overflow-recovery proof before merge.

Yes. Source inspection of PR head and current main shows lastRunPromptUsage is loop-scoped while the PR uses it to skip truncation and mark truncation attempted for the current overflow attempt.

No. Avoiding futile truncation is a reasonable direction, but this implementation should use current-attempt prompt pressure or fall back to truncation when usage provenance is uncertain.
