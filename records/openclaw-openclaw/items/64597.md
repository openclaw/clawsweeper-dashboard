## Summary

Keep open: this PR targets a real embedded-runner overflow recovery problem, but the latest diff can still skip recoverable tool-result truncation because it derives the skip decision from loop-level `lastRunPromptUsage`, and the external real-behavior proof gate is still missing.

## What This Changes

The PR gates embedded-runner overflow recovery by estimated tool-result reduction, adds overflow truncation loop coverage, and narrows auto-reply session resets to embedded compaction failures.

## Best Possible Solution

Keep the reduction-potential gate, but only skip truncation when prompt pressure is attempt-local or otherwise fall back to the existing truncation recovery path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a prior large `lastRunPromptUsage` can survive into a later overflow attempt with oversized tool output and no fresh usage, causing this PR's skip gate to bypass truncation that could recover the current run.

## Solution Assessment

No. The direction is narrow, but the implementation should use current-attempt prompt pressure or preserve truncation when usage provenance is uncertain.
