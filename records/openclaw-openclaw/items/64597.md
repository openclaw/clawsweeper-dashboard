## Summary

Keep open: the PR still has an unresolved P1 correctness issue in the overflow truncation skip gate, lacks after-fix real behavior proof, and has failing CI. A related narrower PR, https://github.com/openclaw/openclaw/pull/81190, tracks a truncation-first approach with proof, but it is still open and does not make this branch safe as-is.

## What This Changes

The branch adds a tool-result reduction gate for overflow recovery, one-shot truncation tests, and changes auto-reply to reset sessions only for embedded compaction_failure payloads.

## Best Possible Solution

Revise the overflow skip gate to use attempt-local prompt pressure or preserve fallback truncation when usage provenance is uncertain; keep any auto-reply compaction_failure narrowing separate if maintainers want it.

## Reproduction Assessment

Yes. Source inspection of PR head and current main shows lastRunPromptUsage is loop-scoped while the PR uses it to skip truncation and mark truncation attempted for the current overflow attempt.

## Solution Assessment

No. Avoiding futile truncation is a reasonable direction, but this implementation should use current-attempt prompt pressure or fall back to truncation when usage provenance is uncertain.
