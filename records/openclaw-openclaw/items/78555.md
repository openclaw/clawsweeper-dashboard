## Summary

This PR should stay open as a focused fix for a real CLI SecretRef bug, but the current diff has a blocking protocol-compatibility issue: it emits a new `secrets.resolve` assignment field under the existing v4 wire contract even though existing TypeScript clients validate that assignment shape with `additionalProperties: false`.

## What This Changes

The PR adds an optional `createIfMissing` hint to `secrets.resolve` assignments and uses it so CLI secret resolution can create missing sibling-ref leaves for legacy ref-only channel configs.

## Best Possible Solution

Keep the narrow sibling-ref apply behavior, but make the wire change compatible by gating `createIfMissing` on client support or versioning the protocol with regenerated clients/docs, then rerun the focused CLI and protocol checks.

## Reproduction Assessment

Yes. Source inspection on current main shows gateway assignments are applied with `setPathExistingStrict`, while Google Chat sibling-ref configs can validly omit `channels.googlechat.serviceAccount`; I did not run live commands because this review is read-only.

## Solution Assessment

No, not as written. The sibling-ref-only create path is the right narrow behavior, but the new ungated v4 result field conflicts with strict existing v4 clients and needs compatibility handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate the new secrets.resolve field by protocol support:** `src/gateway/protocol/schema/secrets.ts:21`
  - body: This adds a new property to a v4 `secrets.resolve` result while existing v4 TypeScript clients validate assignments with `additionalProperties: false`. A newer gateway returning `createIfMissing` to an older CLI will make the client reject the whole payload as invalid, including sibling-ref cases that previously worked when the value leaf already existed. Please either gate this field on client support or version the protocol and regenerate/update the client artifacts and docs.
  - confidence: 0.84
