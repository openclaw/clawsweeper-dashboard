## Summary

Keep this PR open: the reported CLI crash is valid and the branch has strong real-environment proof, but the current patch still adds an ungated field to the v4 gateway result schema, which can break existing v4 clients that validate `secrets.resolve` assignments with `additionalProperties: false`.

The PR changes CLI secret-resolution assignment application for `sibling_ref` SecretRefs by adding a `createIfMissing` assignment flag, generated protocol model updates, regression tests, and a changelog entry.

## Best Possible Solution

Keep the CLI `sibling_ref` create-missing behavior, but derive create eligibility from local secret target metadata or explicitly version/gate any wire-visible protocol change.
