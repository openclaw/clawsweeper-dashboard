## Summary

Keep open: current main still has the source-reproducible CLI apply bug, and this active PR appears to be the focused fix with sufficient real behavior proof and no blocking code findings from this review.

## What This Changes

The PR adds an optional `createIfMissing` flag to `secrets.resolve` assignments, applies flagged sibling-ref CLI secret assignments with `setPathCreateStrict`, updates the Swift protocol model and generated baseline hashes, and adds CLI regression coverage.

## Best Possible Solution

Land this narrow additive protocol-compatible CLI apply fix after normal maintainer review, preserving strict existing-path behavior for unflagged assignments.

## Reproduction Assessment

Yes. Source inspection on current main shows the gateway returns command secret assignments that the CLI applies with `setPathExistingStrict`, while Google Chat validly uses a sibling-ref target whose resolved-value leaf can be absent.

## Solution Assessment

Yes. The PR carries an additive per-assignment hint derived from the existing `secretShape` contract and limits missing-path creation to sibling-ref assignments, which is narrower than weakening strict assignment handling globally.
