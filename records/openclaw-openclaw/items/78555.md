## Summary

Keep open: the PR targets a real source-backed CLI crash and includes credible real behavior proof, but the patch introduces a protocol compatibility blocker for mixed CLI/Gateway versions because older TypeScript clients reject the new assignment property during strict response validation.

## What This Changes

The PR extends `secrets.resolve` assignments with `createIfMissing`, updates CLI SecretRef application to create missing `sibling_ref` leaves, refreshes Swift protocol models, and adds CLI regression coverage plus a changelog entry.

## Best Possible Solution

Keep the sibling-ref create-on-apply behavior, but ship it through a protocol-compatible signal so mixed CLI/Gateway versions either ignore it safely or negotiate/version the new response shape.

## Reproduction Assessment

Yes. Current main source shows the CLI applies gateway assignments with `setPathExistingStrict` while Google Chat declares `serviceAccountRef` as a `sibling_ref` target; the PR body adds before/after terminal proof from a real Pi setup, but I did not run live commands in this read-only checkout.

## Solution Assessment

No, not as-is. The current implementation fixes the narrow apply path, but always emitting `createIfMissing` on `secrets.resolve` is not protocol-compatible with existing strict TypeScript clients; gate or version that field while keeping the current-client apply logic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate createIfMissing for older secrets.resolve clients:** `src/secrets/command-config.ts:82`
  - body: This assignment field is returned over `secrets.resolve`, but existing CLI builds validate `SecretsResolveAssignmentSchema` with `additionalProperties: false` before applying assignments. A newer gateway that emits `createIfMissing` will make an older CLI reject the whole response as an invalid payload for `sibling_ref` assignments, even when the older CLI could have applied an existing target path. Please gate/version this field or otherwise preserve old-client compatibility before emitting it unconditionally.
  - confidence: 0.87
