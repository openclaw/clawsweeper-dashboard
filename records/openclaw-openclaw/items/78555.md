## Summary

Keep open: the PR fixes a real CLI SecretRef crash and has sufficient real behavior proof, but the current diff introduces a `secrets.resolve` version-skew regression that should block merge until the wire contract is made compatible.

## What This Changes

The PR adds a `createIfMissing` secret-resolution assignment hint, applies flagged CLI assignments with `setPathCreateStrict`, updates generated protocol/hash artifacts, adds regression coverage, and documents the CLI/secrets fix in the changelog.

## Best Possible Solution

Keep the sibling-ref CLI apply fix, but avoid an always-emitted strict-schema response field unless it is negotiated; deriving the create behavior from the CLI’s target registry would keep the wire contract compatible.
