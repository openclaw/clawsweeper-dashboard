## Summary

Keep open: this PR fixes a real CLI SecretRef crash and supplies sufficient terminal proof, but the current diff still adds an ungated v4 `secrets.resolve` result field that strict existing clients can reject during CLI/gateway version skew.

## What This Changes

The PR adds a `createIfMissing` hint to `secrets.resolve` assignments, uses it so CLI secret resolution can create missing `sibling_ref` leaves, updates generated protocol/checksum surfaces, and adds CLI regression coverage plus a changelog entry.

## Best Possible Solution

Keep the sibling-ref CLI apply behavior, but avoid an ungated wire-schema change by deriving the create-missing decision from the local secret target shape, or explicitly version/gate the protocol if the field must stay wire-visible.

## Reproduction Assessment

Yes. Current main is source-reproducible: gateway assignments are applied with `setPathExistingStrict`, while Google Chat `sibling_ref` configs can validly omit `channels.googlechat.serviceAccount`; the PR body also supplies real Pi 5 before/after proof.

## Solution Assessment

No, not as written. The sibling-ref-only create path is the right narrow behavior, but adding an ungated v4 result field is not the safest compatibility path; derive it locally or version/gate the wire contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep createIfMissing off the v4 wire payload:** `src/gateway/protocol/schema/secrets.ts:21`
  - body: This adds `createIfMissing` to a v4 `secrets.resolve` result object, but existing v4 CLIs validate assignments with `additionalProperties: false` before applying them. A newer gateway returning this field to an older CLI will make the client reject the whole payload as invalid, including sibling-ref configs that previously worked when the resolved leaf already existed. Please derive the create-missing decision client-side from the local secret target shape, or explicitly version/gate this protocol field.
  - confidence: 0.86
