## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid replaying rich chunks on fallback:** `extensions/telegram/src/send.ts:858`
  - body: The new catch wraps the whole rich chunk helper, but that helper records each successful rich chunk before moving to the next one. If a later chunk matches the unsupported classifier, the fallback resends the full regular chunk plan and can duplicate already-visible content; fall back only before any rich chunk succeeds or resume from the remaining chunks.
  - confidence: 0.9
- **[P2] Cover the direct rich edit path:** `extensions/telegram/src/send.ts:1638-1643`
  - body: The PR wraps sends and draft edits, but `performTextEdit` still calls `richRawApi.editMessageText` directly when rich messages are enabled. Editing an opt-in rich message on an unsupported client can still surface the same failure instead of retrying through the regular HTML edit path.
  - confidence: 0.88
- **[P2] Narrow the rich unsupported matcher:** `extensions/telegram/src/network-errors.ts:276`
  - body: The bare `UNSUPPORTED` alternatives can match unrelated Telegram API errors and turn them into legacy sends or edits. Keep the classifier scoped to known rich-message/client-unsupported responses so validation errors still surface instead of being hidden by fallback delivery.
  - confidence: 0.82
