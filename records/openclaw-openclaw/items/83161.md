## What This Changes

The PR adds Telegram-channel preview final dedup and tests so final text already streamed into an answer preview can be finalized without sending duplicate Telegram messages.

## Best Possible Solution

Keep Telegram-layer ownership, but move the dedup/finalization decision into the current lane-delivery finalizer so it compares against confirmed delivered preview text after stop, preserves buttons and mirroring, and refreshes cleanly against current main.

## Solution Assessment

No, not as currently patched. Telegram channel ownership is the right boundary, but the suppression decision should use the lane deliverer's confirmed-delivery contract instead of finalizing from dispatcher lastPartialText.
