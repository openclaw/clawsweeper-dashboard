## What This Changes

This PR adds Telegram-channel preview final dedup that suppresses text-only finals already streamed into a preview, mirrors the finalized preview, preserves buttons, and adds tests for that path.

## Best Possible Solution

Keep Telegram-layer ownership, but integrate dedup into the current lane-delivery finalization path so suppression happens only after the preview is stopped or materialized and confirmed delivered text matches the final or block.

## Reproduction Assessment

Yes at source/proof level: the PR body includes live Telegram evidence for the disappearing-preview regression, and current source shows lastPartialText can diverge from delivered preview text. I did not run a fresh live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as currently patched. Telegram channel ownership is the right boundary, but the suppression decision should use the lane deliverer's confirmed-delivery contract instead of finalizing from dispatcher lastPartialText.
