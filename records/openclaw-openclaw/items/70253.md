Keep open: current main has resolved the basic Mattermost `streaming` config schema/account-resolution path, but the main inbound Mattermost reply path still forces block streaming off and the bot-authored channel/A2A routing request remains unresolved. ## What This Changes

The issue asks for Mattermost streaming/block delivery config to be honored and for bot-authored channel messages to route between agents when A2A is enabled.

## Best Possible Solution

Split the work: finish the narrow Mattermost block/preview streaming behavior in the Mattermost plugin, coordinate preview UX with https://github.com/openclaw/openclaw/issues/71569 and https://github.com/openclaw/openclaw/issues/75252, and design an explicit Mattermost bot-authored ingress policy with loop protection before changing channel routing.

Yes, source-level. Current main still shows the main Mattermost inbound dispatch forcing block streaming off, no Mattermost `allowBots` ingress surface, mention-gated group drops, and single-target routing; I did not run a live Mattermost/custom WebSocket scenario.
