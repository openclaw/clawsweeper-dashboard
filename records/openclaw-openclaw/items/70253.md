## Summary

Keep open: current main has partially fixed Mattermost `streaming` schema/off-mode support, but this issue still includes unresolved Mattermost block/preview config propagation and bot-authored channel/A2A routing behavior that is not covered by the narrower streaming follow-ups.

## What This Changes

The issue asks OpenClaw to honor Mattermost streaming disable/blocking config and allow configured bot-to-bot or agent-to-agent channel messages to reach other agents.

## Best Possible Solution

Split the work: keep the shipped streaming-off support, finish Mattermost block/final-preview behavior in the narrower streaming follow-ups, and design an explicit Mattermost bot-authored ingress/A2A policy with identity checks, `allowBots` semantics, and loop protection before changing routing.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining paths: inbound Mattermost still forces generic block streaming off, Mattermost has no `allowBots` surface, group mention gating drops unmentioned posts, and channel routing resolves to one agent; I did not run a live Mattermost/custom WebSocket scenario.

## Solution Assessment

No complete current-main solution exists. The narrow streaming-off schema/runtime fix is in place, but the remaining block-preview behavior and bot-authored channel A2A behavior need a split implementation and maintainer product/security review.
