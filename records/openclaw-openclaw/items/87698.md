## Summary

This PR should stay open: current main still lacks registry registration for plugin SDK subagent runs, and the patch is a narrow fit for the reported bug. ## What This Changes

The PR registers `api.runtime.subagent.run()` runs with the subagent registry so `subagent_ended` hooks can fire, keeps child sessions readable until explicit plugin deletion, and adds focused gateway regression tests.

## Best Possible Solution

Land the narrow gateway registration fix only after real plugin/gateway proof is added or maintainers explicitly accept the source-level proof and the plugin lifecycle compatibility change.

## Reproduction Assessment

Yes, source-reproducible: current main's plugin SDK `run()` path returns the gateway run id without registering a `SubagentRunRecord`, while `subagent_ended` emission depends on a registry entry. I did not run a live gateway/plugin reproduction in this read-only review.
