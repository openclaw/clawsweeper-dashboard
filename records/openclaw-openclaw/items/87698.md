## Summary

Keep open: the PR addresses a real source-level gap with a narrow patch and focused regression tests, but the external real-behavior proof is still an in-process Vitest capture with a stubbed execution seam rather than a real plugin running against a gateway.

## What This Changes

The PR registers plugin-SDK subagent runs after `api.runtime.subagent.run()` gateway dispatch so `subagent_ended` can fire and child sessions stay readable until explicit plugin deletion.

## Best Possible Solution

Land this only after redacted real plugin/gateway proof, or explicit maintainer acceptance of the proof gap, while keeping the fix limited to the existing runtime adapter and `cleanup: "keep"` readback contract.
