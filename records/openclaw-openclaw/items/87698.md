## Summary

Keep open. Current main still has the reported plugin SDK gap, while this PR wires `api.runtime.subagent.run()` into the existing subagent registry path with focused regression coverage and no discrete correctness finding from this review.

## What This Changes

The PR registers plugin SDK subagent runs after gateway dispatch, preserves child sessions with `cleanup: "keep"`, and adds gateway regression tests for hook delivery and transcript readback.

## Best Possible Solution

Land the PR after maintainer acceptance of the lifecycle/session-state compatibility change and normal required gates, keeping the SDK signature and config surface unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `api.runtime.subagent.run()` dispatches the gateway `agent` method and returns a run id without creating a `SubagentRunRecord`, so the registry hook path has nothing to complete for plugin SDK runs.

## Solution Assessment

Yes. Registering once at the plugin SDK gateway seam is the narrowest maintainable fix because sibling subagent paths already register on their own side, and the PR keeps transcript readback by using `cleanup: "keep"`.
