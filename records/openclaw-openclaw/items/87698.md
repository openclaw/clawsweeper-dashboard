## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: `dispatchGatewayMethod` can return an already accepted/in-flight Gateway reply for an idempotent retry, but this added registration runs for any payload with a `runId`. Since `registerSubagentRun` unconditionally replaces the existing `runs` entry for that `runId`, a retry can reset lifecycle state for a run that is already tracked; skip registration unless the Gateway response is a fresh accepted run.
  - confidence: 0.9
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler canonicalizes aliases and returns the resolved `sessionKey` in the accepted payload, but the new registry write uses `params.sessionKey`. If a plugin passes an alias such as `main`, the hook context and lifecycle state can point at the wrong child session; register with the returned session key and cover the alias path.
  - confidence: 0.86
