## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: This registers a subagent run for every `agent` response with a `runId`, but current Gateway retries with the same idempotency key can return cached `in_flight` or terminal responses without starting a new run. Because `registerSubagentRun` overwrites the existing record for the same `runId`, a plugin retry can reset lifecycle fields and disturb cleanup or hook state; preserve the raw response status/meta and register only newly accepted runs.
  - confidence: 0.9
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The registry stores `params.sessionKey`, but the `agent` method responds with the canonical `sessionKey` it actually accepted after resolving aliases, global sessions, or selected-agent routing. If a plugin passes one of those forms, `subagent_ended`, cleanup, and transcript lookup can be tied to a different child session than the one that ran; register with the accepted payload session key and cover alias/global cases.
  - confidence: 0.86
