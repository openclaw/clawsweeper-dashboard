Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: The Gateway `agent` method can return cached `accepted`/`in_flight` or terminal payloads for an existing idempotency key instead of starting a new run. This call still registers every payload with a `runId`, and `registerSubagentRun` replaces records by run id, so retries can reset an active/completed registry entry and duplicate or suppress lifecycle state. Only register fresh `status: "accepted"` responses and add retry coverage.
  - confidence: 0.9
- **[P1] Register the Gateway-accepted session key:** `src/gateway/server-plugins.ts:596`
  - body: The agent handler canonicalizes aliases/global sessions and returns the accepted `sessionKey` in the payload, but this registers the caller's requested key. Because `subagent_ended` exposes `entry.childSessionKey`, hook consumers and session-state cleanup can point at an alias or noncanonical key. Use the accepted payload key and add alias/global coverage.
  - confidence: 0.86
