## Summary

## What This Changes

The PR registers `api.runtime.subagent.run()` runs in the subagent registry and adds gateway regression tests for `subagent_ended` hook delivery plus plugin-controlled transcript cleanup.

## Best Possible Solution

Merge the registry-backed plugin SDK fix after redacted installed-plugin proof against a running gateway, or after maintainers explicitly accept the source-level proof and lifecycle compatibility tradeoff.

## Reproduction Assessment

Yes. Current main’s `createGatewaySubagentRuntime().run()` dispatches the gateway `agent` method and returns the `runId` without calling `registerSubagentRun`, while `subagent_ended` emission is registry-record driven.

## Solution Assessment

Yes, with a proof caveat. Registering once in the plugin SDK adapter after a successful gateway dispatch is the narrow owner-boundary fix, and `cleanup: "keep"` preserves the documented readback flow.
