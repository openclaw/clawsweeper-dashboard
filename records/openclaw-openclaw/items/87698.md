Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve idempotent subagent registrations:** `src/gateway/server-plugins.ts:594-600`
  - body: `api.runtime.subagent.run()` accepts an `idempotencyKey`, and the gateway uses that value as the `agent` run id while returning cached accepted or terminal results for duplicate calls. This new unconditional registration runs after every successful dispatch, but `registerSubagentRun` overwrites the record for that run id and starts another completion waiter, resetting `endedHookEmittedAt` if the retry arrives after completion. A retry can therefore mis-associate the record or emit `subagent_ended` twice; skip registration when the registry already has the same run or add an idempotent register path.
  - confidence: 0.88
