## Summary

## What This Changes

The PR adds Gateway-side subagent registry registration and regression tests so plugin SDK `api.runtime.subagent.run()` can emit `subagent_ended` and keep child sessions readable until explicit deletion.

## Best Possible Solution

Keep the hook fix, but reconcile plugin SDK subagent runs to one canonical task record and add redacted installed-plugin live Gateway proof for run-to-hook-to-readback-to-delete before merge.

## Reproduction Assessment

Yes, source-level: current main dispatches plugin `api.runtime.subagent.run()` through Gateway `agent` and returns the runId without registering a `SubagentRunRecord`, while `subagent_ended` is emitted from registry completion paths. I did not run a live plugin harness in this read-only review.

## Solution Assessment

No, not yet: registering after dispatch is a plausible hook-delivery repair, but it is not the best merge shape until the existing Gateway agent task row and the new subagent task row are reconciled into one canonical task-ledger path.
