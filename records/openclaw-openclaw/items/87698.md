- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: The Gateway returns duplicate idempotency-key agent requests as cached `status: "in_flight"` responses with the same `runId`, but this code unwraps only `runId` and registers again. Since `registerSubagentRun` overwrites the record for that runId and clears completion state, a plugin retry can replace lifecycle state for an already running or completed child. Use the raw response/status/meta to register only a fresh accepted run and cover duplicate-idempotency calls.
  - confidence: 0.91
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The agent handler canonicalizes the requested session and returns the accepted `sessionKey`, but this registers the raw `params.sessionKey`. For aliases, global sessions, or other canonicalized routes, `subagent_ended` and cleanup/readback state can point at a different session than the run actually used. Register the accepted payload session key and add alias/global coverage.
  - confidence: 0.89
