## Summary

Keep this PR open: the patch direction is narrow and matches the source-reproduced gap in https://github.com/openclaw/openclaw/issues/59164, but the external-PR proof is still in-process rather than a real installed plugin against a running gateway, and the lifecycle/session-state behavior needs maintainer acceptance before merge.

## What This Changes

The PR registers plugin SDK subagent runs in the subagent registry after the gateway `agent` dispatch, keeps those child sessions until explicit plugin deletion, and adds focused gateway regression tests.

## Best Possible Solution

Land the narrow SDK-runtime registration fix after maintainers either get installed-plugin gateway proof or explicitly accept the lifecycle/session-retention compatibility change.
