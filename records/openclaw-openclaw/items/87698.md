## Summary

Keep open: this PR targets a real plugin SDK lifecycle bug, but the current branch still registers every Gateway response with a runId, including cached or terminal idempotency replies, and records the raw requested child session key instead of the Gateway-accepted one. ## What This Changes

The PR adds best-effort subagent registry registration for `api.runtime.subagent.run()` in the Gateway plugin runtime and adds a focused regression test for `subagent_ended` plus transcript retention.

## Best Possible Solution

Register only fresh accepted Gateway agent runs, use the accepted/canonical child session key from the Gateway response, add regression coverage for idempotent retries and alias resolution, and then provide real plugin-process proof or a maintainer proof override.

## Solution Assessment

No, not yet. Registering at the plugin Gateway seam is the right direction, but the implementation needs to distinguish fresh accepts from cached replies and use the Gateway-accepted session key before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: A repeated `api.runtime.subagent.run()` call with the same `idempotencyKey` can get a cached Gateway `agent` response (`status: "in_flight"` or a terminal payload) instead of spawning a fresh run. This call still registers that runId again, and `registerSubagentRun` overwrites the existing record by runId, which can reset lifecycle state and duplicate hook handling for a run that was already accepted or completed. Gate registration to fresh `status: "accepted"` responses or inspect raw `meta.cached`, and add retry coverage.
  - confidence: 0.9
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler returns the resolved `sessionKey` in the accepted payload after alias/global-session canonicalization, but this registers the raw requested key. When those differ, registry ownership and `subagent_ended` event data can point at a session other than the one the run actually used. Register the accepted payload session key when present and cover alias/global resolution.
  - confidence: 0.88
