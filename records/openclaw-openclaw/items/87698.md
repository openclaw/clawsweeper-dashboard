## Summary

Keep this PR open, but it is not merge-ready yet: the cleanup follow-up fixed the earlier auto-delete regression, but the patch still records the caller-supplied session key instead of the gateway-resolved child session key, and the external real-behavior proof gate remains mock-only.

## What This Changes

The branch adds best-effort subagent registry registration and regression tests for `api.runtime.subagent.run()` so plugin-created subagent runs can emit `subagent_ended` while preserving transcript readback.

## Best Possible Solution

Register plugin subagent runs with the gateway-resolved child session key, keep `cleanup: "keep"`, and land only after redacted live plugin-harness or maintainer/Testbox proof shows hook emission plus transcript readback.

## Reproduction Assessment

Yes. Current main shows `createGatewaySubagentRuntime().run()` dispatching the gateway `agent` method and returning `runId` without `registerSubagentRun`, while the documented plugin flow expects lifecycle hooks and transcript readback.

## Solution Assessment

No. Registering at the plugin runtime adapter is the right boundary and the `cleanup: "keep"` follow-up is directionally correct, but the registration should use the gateway-resolved child session key and still needs real harness proof.
