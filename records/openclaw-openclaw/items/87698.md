## Best Possible Solution

Keep the targeted registry-registration approach, but register only newly accepted runs using the gateway-resolved child `sessionKey`, preserve transcript readback, and keep proof output out of the branch.

## Reproduction Assessment

Yes. Source inspection shows current `api.runtime.subagent.run()` returns the gateway `runId` without registering a `SubagentRunRecord`, while the registry completion path emits `subagent_ended` only for tracked runs.

## Solution Assessment

No, not yet. Registering at the plugin runtime seam is the right narrow fix, but the patch must use the gateway-resolved child session key and avoid re-registering cached in-flight retries.
