## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid re-registering cached subagent runs:** `src/gateway/server-plugins.ts:594-600`
  - body: `runtime.run()` now registers any successful `agent` response, but the gateway can return a cached response for a repeated `idempotencyKey`. Since `registerSubagentRun` overwrites the same `runId`, a retry after completion can clear lifecycle state and allow `subagent_ended` to emit again; only register fresh accepted runs or make this path idempotent.
  - confidence: 0.91
- **[P2] Register the gateway-accepted session key:** `src/gateway/server-plugins.ts:596`
  - body: The gateway accepted payload includes the canonical `sessionKey`, but this code stores the original request `params.sessionKey` as `childSessionKey`. For aliases, global routing, or migrated keys, the registry and `subagent_ended.targetSessionKey` can point at the wrong session.
  - confidence: 0.84
