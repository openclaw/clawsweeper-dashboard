## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Skip cached agent responses before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: `dispatchGatewayMethod` unwraps the Gateway response and drops `meta.cached`. When a plugin retries with the same `idempotencyKey`, `agent` returns a cached in-flight response with the same `runId`; this call still re-registers the run, and `registerSubagentRun` overwrites the existing record keyed by `runId`, resetting lifecycle/cleanup state. Plumb the raw response or status and skip registration for cached/in-flight results.
  - confidence: 0.9
- **[P1] Register the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The `agent` handler canonicalizes requested session keys and replies with the accepted `sessionKey`. Registering `params.sessionKey` can store an alias/raw key while the actual run/transcript lives under the canonical key, so `subagent_ended`, cleanup, and readback can target the wrong session. Use the response `sessionKey` when present and cover alias/global-routed runs.
  - confidence: 0.86
