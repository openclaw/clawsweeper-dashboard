## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Avoid re-registering cached subagent runs:** `src/gateway/server-plugins.ts:594-600`
  - body: `runtime.run()` now registers a subagent record for every successful `agent` response, but the gateway returns cached final responses for repeated `idempotencyKey` calls. Because `registerSubagentRun` overwrites the existing run record with fresh lifecycle state, retrying the same completed call can lose `endedHookEmittedAt`, schedule another wait, and emit `subagent_ended` a second time for the same run. Skip registration for cached responses or make registration idempotent by existing run id before this becomes the public plugin behavior.
  - confidence: 0.86
- **[P2] Register the accepted session key:** `src/gateway/server-plugins.ts:595-597`
  - body: The gateway `agent` handler canonicalizes the requested session and returns the accepted `sessionKey`, but this registration stores `params.sessionKey` instead. For aliases or any route that resolves to a canonical key, the registry and hook payload can point at a different child session than the one the agent actually ran, which breaks ownership/readback assumptions. Capture the returned session key and register that normalized value when available.
  - confidence: 0.78
