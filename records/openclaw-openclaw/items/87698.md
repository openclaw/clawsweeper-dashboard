- **[P1] Avoid creating a second task record for plugin subagent runs:** `src/gateway/server-plugins.ts:594-600`
  - body: The new post-dispatch registration calls `registerSubagentRun` after the Gateway `agent` method has already created a `runtime: "cli"` task for this same session-key run. `registerSubagentRun` creates a separate `runtime: "subagent"` task, and task creation only dedupes within the same runtime, so plugin runs can show duplicate task records or conflicting run-id ownership. Please route this path through one canonical task-state owner and add coverage for task listing and run-id lookup.
  - confidence: 0.9

Please add the build/import-cycle proof or adjust the import shape if build exposes a cycle.
