## Summary

Keep open: the PR addresses a real plugin SDK hook gap, but the current patch can create conflicting CLI and subagent task records for one plugin-created run and still lacks installed-plugin live Gateway proof.

## Best Possible Solution

Keep the hook wiring and transcript-readback behavior, but make plugin subagent runs produce one canonical task record and require redacted installed-plugin live Gateway proof before merge.

## Solution Assessment

No. Registering the missing run is the right direction, but this patch needs one canonical task-state path and installed-plugin proof before it is the safest maintainable fix.
