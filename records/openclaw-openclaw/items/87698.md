## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Skip cached agent responses before registering:** `src/gateway/server-plugins.ts:597-603`
  - body: `dispatchGatewayMethod` hides the Gateway response metadata, so a repeated `api.runtime.subagent.run()` with the same `idempotencyKey` can get the cached `in_flight` response from `agent` and still call `registerSubagentRun` again. Since `registerSubagentRun` unconditionally replaces the entry for that `runId`, the retry resets lifecycle state for an already-running subagent and can break hook/cleanup handling; inspect the raw response or payload status and register only the first accepted dispatch.
  - confidence: 0.92
- **[P1] Register the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:599`
  - body: The Gateway `agent` handler canonicalizes aliases/global routing and returns the accepted `sessionKey`, but this code registers the raw `params.sessionKey` instead. For alias or selected-global runs, the registry can point `childSessionKey` at a key different from the session the agent actually ran in, so `subagent_ended`, cleanup, and readback target the wrong session; use the accepted payload session key when available and cover that case.
  - confidence: 0.9
