## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-596`
  - body: `dispatchGatewayMethod` hides the Gateway response metadata, so this new call runs even when `agent` returned a cached/in-flight reply for a retried idempotency key. `registerSubagentRun` then overwrites the existing record for the same `runId`, resetting lifecycle fields and risking duplicate or stale `subagent_ended` handling; use the raw dispatch response and skip registration when `meta.cached` is true, with regression coverage.
  - confidence: 0.87
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway canonicalizes the requested `sessionKey` before accepting the run and includes that accepted key in the agent payload. Registering `params.sessionKey` here means aliases or legacy keys can leave the registry and hook payload pointing at a different child session than the run actually used; capture the accepted `sessionKey` from the Gateway response and test the canonicalization path.
  - confidence: 0.89
