## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: `agent` retries with the same idempotency key can return a cached/in-flight payload with the same `runId`, but this code still calls `registerSubagentRun`. Since the registry unconditionally replaces the existing run record, a retry can reset lifecycle state for an already-running or completed plugin subagent. Inspect the Gateway response status/meta and skip duplicate registration, or make this registration idempotent with coverage for repeated idempotency keys.
  - confidence: 0.88
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler resolves aliases/global keys and returns the accepted `sessionKey`, but this registration stores the raw `params.sessionKey`. If a plugin passes `main`, `global`, or another alias accepted by Gateway, the registry can attach hook cleanup and transcript state to a different key than the run actually uses. Register the accepted payload session key and add alias/global coverage.
  - confidence: 0.86
