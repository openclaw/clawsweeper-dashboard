## Summary

## What This Changes

The PR adds Gateway plugin-runtime registration for `api.runtime.subagent.run()` responses and regression tests so plugin `subagent_ended` hooks can fire while retaining child-session transcripts.

## Best Possible Solution

Land a narrow fix that registers only fresh accepted Gateway subagent runs using the Gateway-accepted canonical session key, with idempotency/alias regression coverage and real plugin-process Gateway proof or a maintainer proof override.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main dispatches plugin subagent runs through the Gateway and returns runId without registerSubagentRun, while subagent_ended emits only from registry records. I did not run a live plugin harness in this read-only review.

## Solution Assessment

No. The chosen seam is plausible, but the branch must preserve Gateway response metadata and register the accepted canonical sessionKey so idempotency retries and aliases cannot corrupt subagent lifecycle state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: `dispatchGatewayMethod` unwraps only the payload, so a retry with the same idempotency key can get the Gateway's cached `in_flight` response for an already-registered run. This new unconditional registration then calls `registerSubagentRun` again, and the registry replaces the existing record by `runId`, resetting lifecycle state such as hook emission and task/session metadata.
  - confidence: 0.9
- **[P1] Register the Gateway-accepted session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler resolves aliases like `main` and selected-global requests to the accepted canonical `sessionKey`, but this registers the caller's requested key as `childSessionKey`. Because `subagent_ended` exposes that field as `targetSessionKey`, plugins can see or query the wrong child session for alias/global runs.
  - confidence: 0.89
