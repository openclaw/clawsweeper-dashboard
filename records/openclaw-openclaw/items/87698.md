## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent responses before registering the run:** `src/gateway/server-plugins.ts:597`
  - body: The gateway `agent` method returns cached `in_flight` responses for duplicate idempotency keys without starting a new run, but this new call still registers a fresh `SubagentRunRecord` for the returned `runId`. Since `registerSubagentRun` overwrites records by `runId`, a retry can reset the active registry entry with the retry's task/session data and disturb hook/lifecycle state; preserve the raw response metadata or status and skip registration when the response is cached or in-flight.
  - confidence: 0.89
- **[P2] Register the gateway-accepted session key:** `src/gateway/server-plugins.ts:599`
  - body: Gateway agent dispatch canonicalizes the requested session key before accepting and returns that canonical `sessionKey`, but this registration stores the raw `params.sessionKey`. Alias, legacy, or global routes can therefore put the registry's `childSessionKey` on a key different from the actual run/session, which can mis-target `subagent_ended` payloads and cleanup/readback state; use the accepted payload session key and cover canonicalization.
  - confidence: 0.84
