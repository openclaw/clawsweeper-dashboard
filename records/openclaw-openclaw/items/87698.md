## What This Changes

The PR registers `api.runtime.subagent.run()` runs in the subagent registry, keeps their child sessions readable until explicit plugin deletion, and adds focused gateway regression coverage.

## Best Possible Solution

## Solution Assessment

Yes, the adapter-side registration is the narrowest maintainable fix because `api.runtime.subagent.run()` is the missing caller-side site while `sessions_spawn` and ACP paths already register themselves. The remaining question is proof/upgrade acceptance, not an obviously better code shape.
