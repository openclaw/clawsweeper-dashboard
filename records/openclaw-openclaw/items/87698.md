## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: The Gateway `agent` method can return cached `status: "in_flight"` or final replies for repeated idempotency keys. This code registers every successful response with the same run id, and `registerSubagentRun` replaces the run-id entry, so a retry can reset lifecycle state and disturb `subagent_ended` delivery. Only register a fresh accepted run and cover idempotency retries.
  - confidence: 0.9
- **[P1] Register the accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: Gateway `agent` resolves aliases/global routing and returns the accepted `sessionKey`, but this registry record stores `params.sessionKey`. For `main`, `global`, or other canonicalized session keys, hook payloads and registry lookups can point at the requested alias rather than the actual child session. Use the returned session key and add alias/global coverage.
  - confidence: 0.88
