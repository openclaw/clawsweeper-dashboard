## Summary

Keep open: the PR fixes the missing plugin-SDK subagent registry registration in the right area, but the current patch can create duplicate task-ledger records for one plugin-created run and still lacks installed-plugin live Gateway proof.

## What This Changes

Adds Gateway-side registration and tests so `api.runtime.subagent.run()` creates a subagent registry record and can fire `subagent_ended` while keeping the child session readable until explicit deletion.

## Best Possible Solution

Keep the hook wiring and transcript-readback behavior, but make plugin subagent runs produce one canonical task record and require redacted installed-plugin live Gateway proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main's plugin SDK subagent path dispatches `agent` and returns a runId without registering a SubagentRunRecord, while `subagent_ended` emission depends on registry records. I did not execute a live plugin repro in this read-only review.

## Solution Assessment

No. Registering the missing run is the right direction, but this patch needs one canonical task-ledger path and stronger live Gateway proof before it is the safest maintainable implementation.
