## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Skip registration for cached agent responses:** `src/gateway/server-plugins.ts:594-600`
  - body: `dispatchGatewayMethod()` unwraps only the payload, so a retry with the same `idempotencyKey` that the `agent` handler returns as cached `in_flight` still reaches this new registration block. Because `registerSubagentRun()` replaces the record for the same run id, the retry can reset lifecycle state and start another waiter for an already-tracked subagent run; use the raw response meta or an existing-registry check so cached responses do not re-register.
  - confidence: 0.89
- **[P2] Register the accepted canonical session key:** `src/gateway/server-plugins.ts:596`
  - body: The gateway `agent` handler resolves aliases through `loadSessionEntry()` and returns the accepted `sessionKey`, but this code records the caller's original `params.sessionKey`. For aliases or migrated store keys, the subagent registry and `subagent_ended` hook can point at a different child session than the run actually used; register the returned canonical session key and cover that alias case.
  - confidence: 0.86
