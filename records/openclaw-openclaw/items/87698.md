## Summary

Keep open. The patch targets the right gateway/plugin SDK boundary, but it is not merge-ready because cached idempotent `agent` responses can re-register the same run and the registry still stores the caller-provided session key instead of the gateway-accepted canonical key.

## What This Changes

The PR adds plugin SDK subagent registry registration for `api.runtime.subagent.run()` so `subagent_ended` hooks can fire, plus gateway regression tests for hook registration and transcript retention.

## Best Possible Solution

Keep the plugin SDK registration approach, but make it idempotency-safe, register the gateway-accepted session identity, and require real plugin/gateway proof or an explicit maintainer proof override before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid re-registering cached subagent runs:** `src/gateway/server-plugins.ts:594-600`
  - body: `api.runtime.subagent.run()` forwards caller `idempotencyKey`, and the gateway returns a cached accepted/in-flight payload with the same `runId` on retries. This code then calls `registerSubagentRun` again for that same id; the registry currently replaces the entry with `runs.set(runId, entry)` and starts another wait, which can reset completed/cleanup state and make `subagent_ended` fire again for an idempotent retry. Please skip registration for cached `agent` responses or preserve an existing record.
  - confidence: 0.9
- **[P2] Register the gateway-accepted session key:** `src/gateway/server-plugins.ts:594-597`
  - body: The gateway `agent` response includes the canonical `sessionKey` it accepted, but the new registry call stores the original `params.sessionKey`. When the handler canonicalizes or migrates the session key before dispatch, the registry record can point at a different child session than the run that actually executed, so hook/session ownership and cleanup can target stale state. Please read `sessionKey` from the accepted payload and register that value when present.
  - confidence: 0.78
