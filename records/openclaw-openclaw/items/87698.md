## What This Changes

The PR registers plugin SDK subagent runs from `createGatewaySubagentRuntime().run()` in the subagent registry and adds gateway regression coverage for `subagent_ended` delivery plus transcript retention until explicit deletion.

## Best Possible Solution

Merge the registry-backed plugin SDK fix after redacted installed-plugin proof against a running gateway, or after maintainers explicitly accept the in-process proof plus the lifecycle compatibility and session-retention tradeoff.

## Reproduction Assessment

Yes, at source level. Current main dispatches the plugin SDK `agent` run and returns the `runId` without registering a `SubagentRunRecord`, while `subagent_ended` emission is registry-record driven.

## Solution Assessment

Yes, with a proof caveat. Registering once in the plugin SDK adapter after successful gateway dispatch is a narrow owner-boundary fix, and `cleanup: "keep"` preserves the documented plugin readback flow.
