## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent responses before registering:** `src/gateway/server-plugins.ts:594`
  - body: `dispatchGatewayMethod("agent")` can return a cached `status: "in_flight"` or final dedupe payload for a duplicate `idempotencyKey`. This code treats any payload with a `runId` as a fresh spawn and calls `registerSubagentRun` again; the registry then overwrites the existing record for that `runId`, resetting lifecycle/hook state for the active or completed subagent. Please skip cached/non-fresh responses and add an idempotency retry regression test.
  - confidence: 0.9
- **[P1] Register the canonical accepted session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler canonicalizes the requested session and returns the accepted `sessionKey`, but this registration stores `params.sessionKey` as the child key. Alias, legacy, or selected-global routes can therefore emit hooks and track lifecycle against the wrong session key. Please use the gateway-returned canonical session key, with alias/global coverage.
  - confidence: 0.86
