## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:597-603`
  - body: `agent` replies can be cached for idempotency retries and carry `meta.cached: true`; this adapter unwraps only payload and then registers every returned `runId`. Because `registerSubagentRun()` overwrites the existing entry for that run id, a retry can reset lifecycle state and session ownership for an already accepted run. Use the raw response metadata or make registration idempotent, and add a same-idempotency-key retry regression test.
  - confidence: 0.89
- **[P1] Register the accepted child session key:** `src/gateway/server-plugins.ts:599`
  - body: The Gateway `agent` response includes the resolved `sessionKey`, but this call records the raw requested `params.sessionKey`. When the Gateway canonicalizes a global key, alias, or reset route, the subagent registry and `subagent_ended` hook payload can point at the wrong child session. Register the accepted `payload.sessionKey` when present and cover alias/canonicalization behavior.
  - confidence: 0.86
