## Summary

The PR is a plausible fix for the linked plugin SDK hook bug, but it still has two lifecycle correctness blockers: idempotent `agent` retries can re-register and overwrite an existing subagent run record, and the registry uses the raw requested session key instead of the gateway-accepted canonical session key.

## What This Changes

The branch adds best-effort `registerSubagentRun` registration after `createGatewaySubagentRuntime().run()` dispatches `agent`, plus gateway regression tests for hook registration and transcript retention.

## Best Possible Solution

Keep the PR open and repair the registration helper so it registers only newly accepted runs with the gateway-accepted session key, then attach redacted real plugin/Gateway proof or maintainer-provided Crabbox proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the original bug: current main dispatches plugin `runtime.subagent.run()` through `agent` and returns `{ runId }` without creating a `SubagentRunRecord`, while `subagent_ended` emission reads from that registry. I did not run a live plugin harness in this read-only review.

## Solution Assessment

No, not yet. Registering the plugin SDK run in the registry is the right fix direction, but the current patch needs to preserve idempotent retry state and use the gateway-accepted canonical session key before it is the safest maintainable solution.

## Review Findings

Full review comments:

- **[P1] Avoid re-registering cached subagent runs:** `src/gateway/server-plugins.ts:594-600`
  - body: `agent` is idempotent: repeated calls with the same `idempotencyKey` can return the cached accepted/in-flight run instead of spawning a new one. This path still calls `registerPluginSubagentRunQuietly` for every returned `runId`, and `registerSubagentRun` overwrites the existing map entry for that run id, clearing lifecycle state such as `endedHookEmittedAt` and starting another wait. Please carry the cached/accepted signal through this dispatch path and skip registration, or preserve the existing registry record, and add retry coverage.
  - confidence: 0.9
- **[P2] Register the gateway-accepted session key:** `src/gateway/server-plugins.ts:596`
  - body: The `agent` accepted response includes the canonical `sessionKey` that the gateway actually resolved, but this registers the raw `params.sessionKey`. If a plugin passes an alias or otherwise non-canonical key, the registry record and `subagent_ended` payload can point at a different child session than the run uses, breaking transcript readback and lifecycle ownership. Please register the accepted payload's session key when present and cover an alias/canonicalization case.
  - confidence: 0.83
