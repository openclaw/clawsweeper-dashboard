## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip cached agent replies before registering:** `src/gateway/server-plugins.ts:594-600`
  - body: `dispatchGatewayMethod` can return an already accepted/in-flight Gateway reply for an idempotent retry, but this code still registers the run again. Since `registerSubagentRun` unconditionally replaces the existing `runs` entry for that `runId`, a retry can reset `endedAt`, `outcome`, `cleanupHandled`, and hook-emission state for a run that is already tracked; skip registration when the Gateway response is cached/in-flight.
  - confidence: 0.9
- **[P1] Use the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: The Gateway `agent` handler canonicalizes aliases and returns the resolved `sessionKey` in the accepted payload, but the new registry write uses `params.sessionKey`. If a plugin passes an alias such as `main` or another key that the Gateway resolves before dispatch, the registry will attach hook and lifecycle state to the wrong child session; register with the returned session key instead.
  - confidence: 0.86
