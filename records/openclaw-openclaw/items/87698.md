## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip cached agent responses before registering:** `src/gateway/server-plugins.ts:594`
  - body: The Gateway `agent` method returns cached `{ status: "in_flight" }` payloads for repeated `idempotencyKey` calls, but this new code registers every response that contains a `runId`. Because `registerSubagentRun` replaces the existing map entry for that runId, a retry can reset lifecycle fields for the already-running child and disturb hook/cleanup state; only register fresh accepted runs, and add coverage for cached replies.
  - confidence: 0.9
- **[P1] Register the Gateway-accepted child session key:** `src/gateway/server-plugins.ts:596`
  - body: This passes the raw `params.sessionKey` into the registry even though the Gateway canonicalizes aliases and returns the accepted session key in the agent payload. If a plugin passes `main`, `global`, or another alias-like target, the run executes under the canonical key while the subagent registry points hooks, cleanup, and transcript readback at the wrong child session; use the accepted `payload.sessionKey` when present and cover the alias case.
  - confidence: 0.88
