## Summary

Keep open; the bug fix is useful, but the branch still introduces a blocking duplicate task-record path and its external real behavior proof is still only an in-process mocked capture rather than an installed plugin against a live Gateway.

## What This Changes

The PR adds Gateway-side registration for plugin-SDK subagent runs so `api.runtime.subagent.run()` can participate in the subagent registry and fire `subagent_ended` hooks.

## Best Possible Solution

Keep the hook wiring and transcript-readback behavior, but make plugin subagent runs produce one canonical task record and require redacted installed-plugin live Gateway proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main's plugin SDK subagent path dispatches `agent` and returns a runId without registering a SubagentRunRecord, while `subagent_ended` emission depends on registry records. I did not execute a live plugin repro in this read-only review.

## Solution Assessment

No. Registering the missing run is the right direction, but the current patch needs one canonical task-state path and stronger live Gateway proof before it is the safest maintainable fix.
