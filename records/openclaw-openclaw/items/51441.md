## Summary

Keep open. Current main has partial response-model and fallback concepts, but it still does not carry routing-proxy backend identity through OpenAI-compatible agent transport, agent metadata, persisted session rows, Gateway rows, and normal session_status output.

Expose the actual backend model/provider and fallback state for routing-proxy runs in agent runtime metadata, session persistence, Gateway sessions, and session_status output.

## Best Possible Solution

Define a maintainer-approved resolved-backend metadata contract, then propagate it from OpenAI-compatible responses through agent meta, session storage, Gateway rows, and session_status with focused tests.
