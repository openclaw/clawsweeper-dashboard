## Summary

Keep open: current main still has selected/runtime model plumbing, but it does not carry response-side resolved backend identity from OpenAI-compatible transports into agent metadata, session persistence, Gateway session rows, or session_status/runtime output. The related actual-model and OpenRouter cost issues remain open and adjacent rather than replacements.

## What This Changes

The issue requests resolved backend model/provider metadata to be captured from OpenAI-compatible responses and surfaced through session persistence, session_status, CLI status, and agent runtime context.

## Best Possible Solution

Define an additive selected-versus-resolved backend identity contract at the OpenAI-compatible transport boundary, persist it separately from selected runtime model state, and surface it consistently through session_status, Gateway rows, CLI status, and agent runtime context while handling provider tags as a separate policy choice.

## Solution Assessment
