## Summary

Keep open. Current main has partial resolved-model concepts, but it still does not carry routing-proxy backend identity through the OpenAI-compatible agent transport, agent metadata, persisted session rows, and normal session_status output requested here.

## What This Changes

The issue requests resolved backend model/provider and fallback metadata in session status, persisted session usage, and agent runtime context for routing-proxy deployments.

## Best Possible Solution

Add a maintainer-approved resolved backend identity contract that captures upstream response model data in OpenAI-compatible agent transports, carries it through agent metadata and session persistence, and exposes it in session_status without conflating selected model state with actual backend state.

## Reproduction Assessment

Source-reproducible: current main has partial responseModel support but source inspection shows the requested OpenAI-compatible agent transport, agentMeta, session persistence, Gateway row, and session_status fields are still missing; no live proxy repro was run.

## Solution Assessment

Unclear until maintainer direction: capturing upstream response.model is likely part of the fix, but maintainers should choose the durable API/storage shape and how it relates to responseModel, resolvedModel, transport headers, and session rows.
