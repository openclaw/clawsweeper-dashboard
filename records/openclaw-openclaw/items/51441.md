## Summary

Keep open. Current main still drops OpenAI-compatible returned model IDs and persists/exposes only one active model identity; #22021 and #9016 are adjacent visibility/cost requests, not a complete canonical replacement for this session metadata feature.

## What This Changes

Expose proxy-resolved backend model/provider and fallback visibility through agent metadata, session persistence, GatewaySessionRow, session_status, CLI status, and runtime context.

## Best Possible Solution

Add a requested-vs-resolved model metadata pipeline that captures returned backend IDs, keeps the requested alias intact, persists resolved backend fields, exposes them in session/status surfaces, and defers optional LiteLLM request tagging.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an OpenAI-compatible stream whose response.model or chunk.model differs from the requested alias; current main initializes and persists from the requested/runtime model path and has no resolved backend fields to carry that value.

## Solution Assessment

Yes. The requested additive metadata path is the narrow maintainable solution; the optional LiteLLM tagging/correlation work should stay out of the first fix unless needed by the core resolved-model pipeline.
