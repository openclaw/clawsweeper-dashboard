## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Derive opaque cache keys before sending provider payloads:** `src/agents/pi-embedded-runner/llamacpp-cache-key.ts:15`
  - body: `resolveLlamaCppCacheKey` prefixes and forwards the raw OpenClaw session key or session id into `cache_key` and `session_id`. Current session keys can include transport/user identifiers like WhatsApp numbers or Telegram peer ids, and OpenAI-compatible providers can point at arbitrary configured base URLs, so this leaks routing identity into backend requests/logs. Derive a stable opaque key instead, such as a scoped hash, before attaching it to provider payloads.
  - confidence: 0.9
- **[P2] Move vendor payload policy out of the core runner:** `src/agents/pi-embedded-runner/llamacpp-cache-key.ts:3`
  - body: This adds a core allowlist for `llama-cpp-turboquant`, but current main already routes provider-owned request rewrites through `ProviderPlugin.wrapStreamFn` and the PR's `pi-embedded-runner` path no longer exists. Rebase this onto current `embedded-agent-runner` and put the payload mutation behind a provider/plugin hook or a narrow provider context extension so core does not own a one-off vendor policy.
  - confidence: 0.87
