## Best Possible Solution

Add a provider-owned Copilot GPT/Responses replay policy that prevents stale encrypted reasoning/signature replay in channel/direct histories, with focused regression coverage, live or redacted Copilot diagnostic proof, and a repair path for already polluted session JSONL if runtime still reads those entries.
