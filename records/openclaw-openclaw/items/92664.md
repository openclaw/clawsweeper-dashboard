## What This Changes

The issue asks for GBK-encoded text files on Chinese Windows to display correctly in the `read` tool instead of being decoded as UTF-8 mojibake.

## Best Possible Solution

Define one read-tool encoding contract for legacy Windows text files, then implement it in the session read path with regression coverage for GBK bytes and existing UTF-8 behavior.

## Reproduction Assessment

No live high-confidence platform reproduction was run; source inspection is strong because current main always decodes non-image read-tool bytes as UTF-8, which explains GBK mojibake.

## Solution Assessment

Unclear: auto-detection, Windows default-codepage handling, or a model-facing encoding parameter are all plausible, but maintainers should choose the permanent tool contract first.
