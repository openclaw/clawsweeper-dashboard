## Summary

Keep open. Current main still lacks the requested configurable Telegram unknown-group warn/leave behavior, and this PR is paired with the still-open feature issue at https://github.com/openclaw/openclaw/issues/50681, but the branch still has authorization, topic-send, cooldown, and real-proof blockers.

## What This Changes

The PR adds optional Telegram `unknownGroupAction`, `unknownGroupMessage`, and `unknownGroupCooldownMs` config, normal-message ignore/warn/leave handling, cooldown tracking, and focused Telegram tests.

## Best Possible Solution

Land a revised Telegram preflight implementation that precisely identifies truly unknown groups, shares ignore/warn/leave across normal messages and native commands, uses the Telegram thread helper, bounds cooldown state, updates config/docs metadata, and includes redacted real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new feature. The PR defects are source-reproducible from the diff and current-main contracts: group reason semantics, native command authorization, thread send helpers, and cooldown lifecycle.

## Solution Assessment

No. Telegram inbound preflight is the right owner boundary, but this branch needs shared policy handling, safe topic sends, cooldown cleanup, config/docs alignment, and real proof before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Classify only truly unknown groups:** `extensions/telegram/src/unknown-group.ts:11-12`
  - body: `group-policy-allowlist-empty` can come from wildcard-covered groups that are configured but lack sender allowlist entries. Treating that reason as unknown makes `unknownGroupAction: "leave"` remove the bot from a configured wildcard group instead of preserving the existing sender-auth block.
  - confidence: 0.9
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new action branch only runs after `shouldSkipGroupMessage` in normal message handling. On this PR branch, native command auth still sends the static `This group is not allowed.` response, so `/help` and plugin commands bypass the configured ignore/warn/leave behavior.
  - confidence: 0.9
- **[P2] Use the thread params helper for warnings:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1661`
  - body: This passes raw `resolvedThreadId` to `sendMessage`. For Telegram forum General topic, the existing `buildTelegramThreadParams` helper omits id 1 because Telegram rejects it, so warn can fail and leave can happen without the configured warning.
  - confidence: 0.92
- **[P3] Evict expired cooldown entries:** `extensions/telegram/src/unknown-group.ts:27-31`
  - body: The cooldown map keeps every chat id forever; when entries expire or the bot leaves, they are never deleted. Long-running bots added to many unknown groups will retain those ids indefinitely.
  - confidence: 0.86
