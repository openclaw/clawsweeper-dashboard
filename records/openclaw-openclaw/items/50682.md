## Summary

Keep open. The feature is not implemented on current main, but this PR still has source-reproducible gaps in the Telegram command/topic paths and lacks observable real Telegram proof before a security-sensitive auto-leave behavior can merge.

## What This Changes

This PR adds Telegram unknown-group action/message/cooldown config plus runtime and test changes to ignore, warn, or leave unapproved groups before assistant execution.

## Best Possible Solution

Land a revised PR or narrow successor that centralizes Telegram unknown-group preflight across normal messages and native commands, uses existing thread helpers, bounds cooldown state, updates public config docs, and includes redacted live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new Telegram feature. The blocking PR defects are source-reproducible from the PR and current-main code paths: slash commands bypass the new branch, General forum warnings bypass the helper, and cooldown entries are never evicted.

## Solution Assessment

No. Telegram inbound preflight is the right owner boundary, but this diff is incomplete until native commands, forum General-topic sends, cooldown eviction, docs, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply unknown-group actions to slash commands too:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new warn/leave branch only runs in the normal message handler. Native Telegram commands are registered before that handler and command auth still replies with the static group-denied message, so `/help` or plugin slash commands in an unknown group bypass `unknownGroupAction` and never leave.
  - confidence: 0.9
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1661`
  - body: This sends `message_thread_id` directly from `resolvedThreadId`. For a forum General topic that value can be `1`, and the existing helper deliberately omits `1` because Telegram rejects it; `warn` can fail and `leave` can run without the configured warning.
  - confidence: 0.92
- **[P3] Evict expired cooldown entries:** `extensions/telegram/src/unknown-group.ts:19-27`
  - body: `lastActionAt` grows for every unknown chat id and entries are never removed, including after the cooldown elapses or after a successful leave. A long-running bot exposed to many unknown groups will retain those ids indefinitely.
  - confidence: 0.86
