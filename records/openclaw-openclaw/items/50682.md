## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Constrain unknown-group classification before leaving:** `extensions/telegram/src/unknown-group.ts:11-12`
  - body: `group-policy-allowlist-empty` can also describe wildcard-configured groups that still rely on sender filtering. Treating it as unknown means `unknownGroupAction: "leave"` can eject the bot from a configured deployment rather than only from truly unregistered chats.
  - confidence: 0.91
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new warn/leave handling only runs in the normal message handler, while native slash commands are registered first and authorize through their own static-denial path. A command in an unknown group bypasses the configured action and leaves the bot in place.
  - confidence: 0.88
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1660`
  - body: This warning path sends raw `message_thread_id`, but the existing helper intentionally omits forum General-topic id `1` because Telegram rejects it for sends. Warn can fail there, and leave mode can exit without posting the configured warning.
  - confidence: 0.87
- **[P3] Evict expired unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:19-27`
  - body: `lastActionAt` only grows and never deletes expired chat ids. A long-running bot exposed to many unknown groups can accumulate stale cooldown entries, so the tracker should prune expired entries or otherwise bound the map.
  - confidence: 0.82
