## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Separate empty sender allowlists from unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also returned when a wildcard/default group is allowed but no sender allowlist is configured. Treating that as an unknown group means `unknownGroupAction: "leave"` can eject the bot from wildcard-configured groups instead of only unregistered chats; carry an explicit chat-allowlist fact or restrict unknown handling to actual chat allowlist misses.
  - confidence: 0.86
- **[P2] Use the Telegram thread helper for warnings:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: `resolvedThreadId` can be `1` for a forum's General topic, and the existing helper omits that value because Telegram rejects it for sends. Passing the raw id here makes `warn` fail in General topics and makes `leave` skip the configured warning, so this path should build thread params through the shared helper.
  - confidence: 0.9
- **[P2] Apply unknown-group policy to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644`
  - body: This block only runs in the normal message handler, after native command handlers have already consumed slash commands. A first `/help` or plugin command in an unknown group still goes through command auth and returns a static denial without warn/leave, so the configured security action is bypassed for command messages.
  - confidence: 0.87
