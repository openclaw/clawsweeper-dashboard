Full review comments:

- **[P1] Separate unknown chats from sender-allowlist blocks:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also returned for wildcard-configured groups when no sender allowlist entries are present. Treating that as an unknown chat means `unknownGroupAction: "leave"` can eject the bot from a group intentionally covered by `groups["*"]` instead of only rejecting truly unregistered chats.
  - confidence: 0.91
- **[P2] Build warning thread params through the Telegram helper:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This warning path passes `resolvedThreadId` directly, so a General forum topic can send `message_thread_id: 1`. Current Telegram helpers deliberately omit that value because Telegram rejects it, which makes `warn` fail and `leave` proceed without the configured warning.
  - confidence: 0.9
- **[P2] Run unknown-group actions for native commands too:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new action only runs in the generic message handler. Native slash commands are registered before that handler and `resolveTelegramCommandAuth()` consumes unknown-group command updates with the old static denial, so `/help` or `/status` in an unknown group will not warn or leave as configured.
  - confidence: 0.88
- **[P3] Evict expired unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:27-31`
  - body: `lastActionAt` is never pruned, so a long-running bot invited to many unknown groups retains every chat id it has ever warned or left. Drop expired entries when checking cooldown or bound the tracker so the new security path cannot grow memory indefinitely.
  - confidence: 0.78
