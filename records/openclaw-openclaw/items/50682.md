## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not treat sender-policy blocks as unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is not a true unknown-chat decision: current group policy uses it when a wildcard-enabled group has no sender allowlist entries. Classifying it as unknown means `warn`/`leave` can fire in intentionally wildcard-enabled groups instead of only for unapproved chat IDs.
  - confidence: 0.9
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This direct `message_thread_id` payload bypasses `buildTelegramThreadParams`, which intentionally omits General-topic id `1` because Telegram rejects it for message sends. In a General-topic forum, the warning send can fail and `leave` can proceed without the configured warning.
  - confidence: 0.88
- **[P2] Apply unknown-group actions to native commands too:** `extensions/telegram/src/bot-handlers.runtime.ts:1645`
  - body: The new action path only runs in the normal message handler. Telegram native commands are registered and authorized separately, where unknown groups still get fixed denial text, so `/help` or plugin slash commands bypass the configured `warn`/`leave` behavior.
  - confidence: 0.86
- **[P3] Evict expired cooldown entries:** `extensions/telegram/src/unknown-group.ts:27`
  - body: `lastActionAt` is never pruned, so a long-running bot that is added to many unknown groups keeps every chat ID forever even after cooldown expiry or successful leave. Delete expired entries when checking cooldown, or otherwise bound the tracker.
  - confidence: 0.76
