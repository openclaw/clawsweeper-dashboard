## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not treat sender-allowlist misses as unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also returned for wildcard-configured groups when no sender allowlist is configured, so treating it as unknown can warn or eject the bot from groups intentionally covered by `groups["*"]`. Restrict unknown-group actions to true chat-allowlist misses or carry a more precise classifier from `evaluateTelegramGroupPolicyAccess`.
  - confidence: 0.9
- **[P1] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1647`
  - body: This action branch runs only in the normal message handler, while native slash commands are registered and authorized through `resolveTelegramCommandAuth` before that path. If an unknown group's first message is a command, the bot sends the static denial and stays in the group instead of honoring `warn` or `leave`.
  - confidence: 0.88
- **[P2] Use the thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: The warning send passes `message_thread_id` directly when `resolvedThreadId` is truthy. For forum General topic id 1, current Telegram helpers intentionally omit that parameter because Telegram rejects it, so `warn` can fail and `leave` can leave without the configured warning.
  - confidence: 0.9
- **[P2] Bound the cooldown tracker:** `extensions/telegram/src/unknown-group.ts:25-26`
  - body: `lastActionAt` only grows: expired entries are never pruned and `leave` records chats the bot should never hear from again. A long-running public bot exposed to many unknown groups can accumulate unbounded per-chat state; delete expired entries or cap the tracker.
  - confidence: 0.79
