## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Classify only true unknown chats as unknown:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` can mean a wildcard-configured group is allowed at the chat level but lacks sender allowlist entries. Treating that as unknown makes `unknownGroupAction: "leave"` warn or eject the bot from a configured group instead of only unregistered chats.
  - confidence: 0.91
- **[P2] Route unknown-group actions through command auth too:** `extensions/telegram/src/bot-handlers.runtime.ts:1646-1649`
  - body: The new warn/leave behavior only runs from the regular message handler. Native slash commands are registered earlier and resolve `group-chat-not-allowed` inside `resolveTelegramCommandAuth()`, so `/help` or plugin commands in an unknown group still get the fixed denial and never warn or leave.
  - confidence: 0.88
- **[P2] Use thread params for forum warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1661`
  - body: This sends `{ message_thread_id: resolvedThreadId }` directly. For a forum General topic, `resolvedThreadId` is `1`, and current Telegram helpers/docs intentionally omit `message_thread_id=1` because Telegram rejects message sends with that value.
  - confidence: 0.9
- **[P2] Bound the cooldown tracker:** `extensions/telegram/src/unknown-group.ts:19-31`
  - body: `lastActionAt` never evicts expired chat IDs. A long-running bot repeatedly added to unknown groups can accumulate one map entry per group even after `leaveChat`, so the cooldown state should prune expired entries or otherwise stay bounded.
  - confidence: 0.78
