## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not classify empty sender allowlists as unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also returned for configured wildcard groups when no sender allowlist is present; treating it as unknown means `unknownGroupAction: "warn"` or `"leave"` can warn or eject the bot from a group intentionally covered by `groups: {"*": ...}`. Keep the unknown-chat action limited to the precise chat-not-allowed case, or carry a separate explicit unknown-chat signal from the policy layer.
  - confidence: 0.92
- **[P2] Apply unknown-group actions before native command replies:** `extensions/telegram/src/bot-handlers.runtime.ts:1647`
  - body: The new action runs only from the normal message handler, but native `bot.command` handlers are registered first and answer authorization failures through `resolveTelegramCommandAuth()`. A first `/help` or plugin command in an unapproved group will still get the static command denial and will not warn or leave, so the configured action is bypassed for a visible command path.
  - confidence: 0.86
- **[P2] Reuse thread params for unknown-group warnings:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This call passes `resolvedThreadId` directly, so a General forum topic resolves to `message_thread_id: 1`; existing Telegram helpers deliberately omit that value because Telegram rejects message sends to General with thread id 1. Build the warning options through the same thread helper used by normal Telegram sends so `warn` does not 400 before `leave`.
  - confidence: 0.9
- **[P2] Document the new Telegram config keys:** `src/config/types.telegram.ts:203`
  - body: These fields are public operator config and affect authorization plus a `leaveChat` side effect, but the PR does not update the Telegram docs. Add docs for the action values, cooldown, no-message behavior, forum-topic behavior, and the intended allowlist interaction so schema/types/docs stay aligned.
  - confidence: 0.84
- **[P3] Prune expired unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:26`
  - body: `lastActionAt` only grows and entries are never removed, even after the cooldown expires or after a successful leave. Delete expired entries during the cooldown check or otherwise bound the tracker so long-running bots exposed to many unknown chats do not retain every chat id forever.
  - confidence: 0.78
