## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not treat sender-allowlist failures as unknown groups:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` also covers wildcard-matched groups that are allowed by `channels.telegram.groups` but blocked because no sender allowlist is configured. With `unknownGroupAction: "leave"`, this would eject the bot from wildcard-covered groups instead of only acting on truly unlisted groups.
  - confidence: 0.91
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1660`
  - body: This direct `message_thread_id` pass bypasses `buildTelegramThreadParams()`, which omits forum General-topic id `1` because Telegram rejects it. In a General topic, `warn` will fail and `leave` can run without sending the configured warning.
  - confidence: 0.9
- **[P2] Apply unknown-group actions to native slash commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new action path only runs inside the normal message handler. Native slash commands are registered before that handler and return their own group-auth denial, so `/help` or plugin commands in an unknown group will not warn or leave as configured.
  - confidence: 0.86
- **[P2] Evict expired unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:17-27`
  - body: `lastActionAt` is only written and read, never pruned. A long-running bot exposed to many unique unknown groups will retain one map entry per chat even after cooldown expiry or a successful leave.
  - confidence: 0.82
