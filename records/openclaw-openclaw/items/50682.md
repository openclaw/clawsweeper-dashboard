## Summary

Keep open. The feature is useful and paired with https://github.com/openclaw/openclaw/issues/50681, but the current diff misses native slash commands, mishandles Telegram General forum-topic warnings, has unbounded cooldown state, and still lacks observable real Telegram proof.

## What This Changes

The PR adds Telegram unknown-group action/message/cooldown config plus normal-message handling to ignore, warn, or leave denied groups, with Telegram test-harness coverage.

## Best Possible Solution

Land this PR or a narrow successor with a shared Telegram unknown-group preflight that covers normal messages and native commands, uses existing thread helpers, bounds cooldown state, updates public config docs, and includes redacted real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Telegram feature. The review findings are source-reproducible from the PR and current-main code paths: slash commands never reach the new branch, and General forum warnings bypass the existing thread helper.

## Solution Assessment

No. Telegram inbound preflight is the right owner boundary, but the current implementation is incomplete until native commands, forum General-topic sends, cooldown eviction, docs, and real behavior proof are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply unknown-group actions to slash commands too:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1646`
  - body: The new warn/leave branch only runs in the normal message handler. Native Telegram commands are registered before that handler and `resolveTelegramCommandAuth()` still replies with the static group-denied message, so `/help` or plugin slash commands in an unknown group bypass the configured `unknownGroupAction` and never leave.
  - confidence: 0.9
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1661`
  - body: This sends `message_thread_id` directly from `resolvedThreadId`. For a forum's General topic that value can be `1`, and the existing `buildTelegramThreadParams()` helper deliberately omits `1` because Telegram rejects it; `warn` will fail and `leave` will leave without the configured warning.
  - confidence: 0.92
- **[P3] Evict expired unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:17-27`
  - body: `lastActionAt` grows for every unknown chat id and entries are never removed, including after the cooldown has elapsed or after a successful leave. A long-running bot exposed to many unknown groups will retain all of those ids indefinitely; delete expired entries or cap the tracker.
  - confidence: 0.86
