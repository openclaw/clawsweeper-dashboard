## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Do not treat sender allowlist misses as unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is not the same as an unknown Telegram group: current group-access logic uses it when a wildcard group entry enabled the chat but no sender allowlist is configured. With this helper, `unknownGroupAction: "leave"` can eject the bot from groups that were intentionally covered by `groups: {"*": ...}` but sender-gated, so the unknown-group action should be limited to true chat allowlist misses such as `group-chat-not-allowed`.
  - confidence: 0.93
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This direct `message_thread_id` payload bypasses `buildTelegramThreadParams()`, which intentionally omits forum General topic id 1 because Telegram rejects it for message sends. In a General topic, `warn` will fail to deliver and `leave` will leave without the configured warning, so the warning path should build thread params through the existing helper and cover id 1 in tests.
  - confidence: 0.9
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1645-1646`
  - body: The new unknown-group action only runs inside `registerTelegramHandlers`, but native commands are registered first and authorize inside `bot-native-commands.ts`, where `group-chat-not-allowed` still sends a fixed denial. If an unknown group's first activity is `/help` or another native command, the configured warn/leave policy is bypassed, so the command auth path needs to share this decision before responding.
  - confidence: 0.88
- **[P2] Document the new Telegram config fields:** `src/config/zod-schema.providers-core.ts:283-285`
  - body: These strict schema fields introduce user-facing Telegram configuration, but the public Telegram docs still only describe existing group allowlist controls. Add docs for the action, message, cooldown, defaults, and destructive leave behavior so operators can configure and audit the new policy correctly.
  - confidence: 0.84
