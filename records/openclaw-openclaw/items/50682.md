## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not classify wildcard sender misses as unknown groups:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is not the same as an unregistered chat: current `evaluateTelegramGroupPolicyAccess` returns it when `groups` contains `"*"` but no sender allowlist is configured. With this mapping, `unknownGroupAction: "leave"` can warn and eject the bot from wildcard-admitted groups instead of only from chats missing the group allowlist.
  - confidence: 0.9
- **[P2] Use the thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This direct `message_thread_id` payload bypasses `buildTelegramThreadParams`, which intentionally omits `message_thread_id=1` because Telegram rejects sends to the General topic with that parameter. Unknown-group warnings in General-topic forums can fail before the configured warning is delivered.
  - confidence: 0.87
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1645-1646`
  - body: The new warn/leave action only runs in the normal message handler. Telegram native commands use `resolveTelegramCommandAuth` and currently answer an unknown group with fixed denial text, so a first `/help` or plugin command in an unknown group bypasses the configured warn/leave behavior entirely.
  - confidence: 0.86
- **[P3] Bound the cooldown tracker:** `extensions/telegram/src/unknown-group.ts:17-27`
  - body: `lastActionAt` is only ever appended to, so a long-running bot that sees many unique unknown group IDs keeps those entries indefinitely. Prune expired entries or cap the tracker so the new protection does not become unbounded process state.
  - confidence: 0.78
- **[P3] Document the new Telegram config keys:** `src/config/zod-schema.providers-core.ts:283-285`
  - body: These schema/type additions create new public Telegram configuration, but the PR does not update the Telegram docs or config guidance. Add the documented defaults and behavior so operators can understand ignore, warn, leave, message, and cooldown semantics before using them.
  - confidence: 0.82
