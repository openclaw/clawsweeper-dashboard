## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep empty sender allowlists out of unknown chats:** `extensions/telegram/src/unknown-group.ts:11`
  - body: `group-policy-allowlist-empty` is also used when a wildcard `groups["*"]` match enables a group but no sender allowlist entries are configured. Treating that as unknown means `unknownGroupAction: "leave"` can leave configured wildcard groups instead of only chats rejected by the group allowlist.
  - confidence: 0.9
- **[P1] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1641-1642`
  - body: This action is only wired into the normal message handler. Native and plugin slash commands are registered earlier and authorize through `resolveTelegramCommandAuth`, so a first `/status` or plugin command in an unknown group still gets the static denial and never runs the configured warn/leave behavior.
  - confidence: 0.88
- **[P2] Use thread params for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1657`
  - body: The new warning send passes `resolvedThreadId` directly as `message_thread_id`. For a forum General topic this can be `1`, which existing helpers intentionally omit because Telegram rejects it, so `warn` can 400 and `leave` can leave without the configured warning.
  - confidence: 0.9
- **[P2] Prune unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:16-25`
  - body: `lastActionAt` only grows; a long-running bot added to many unknown groups can keep every chat id forever even after cooldown expiry or a successful leave. Evict expired entries or otherwise bound the tracker before using it on attacker-controlled chat ids.
  - confidence: 0.82
- **[P2] Document the new Telegram config keys:** `src/config/types.telegram.ts:203-217`
  - body: The PR adds a new user-facing Telegram config surface but does not update public Telegram docs or generated/help metadata. Operators need the action semantics, defaults, warning text behavior, cooldown, and upgrade notes documented with the schema/types before this ships.
  - confidence: 0.78
