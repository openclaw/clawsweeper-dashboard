## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not treat sender-allowlist misses as unknown chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is not the same as an unregistered chat: wildcard `groups["*"]` admits the group while sender authorization remains separate. With `unknownGroupAction: "leave"`, this branch can eject the bot from wildcard-configured groups that are intentionally covered by config.
  - confidence: 0.89
- **[P2] Use the Telegram thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This sends `message_thread_id` directly, so General forum topic id `1` is included even though `buildTelegramThreadParams()` intentionally omits it for message sends. In that topic, `warn` will fail and `leave` will leave without the configured warning.
  - confidence: 0.9
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1647-1649`
  - body: The new warn/leave path only runs in the normal message handler, but Telegram native and plugin commands are registered earlier and handle auth inside `resolveTelegramCommandAuth()`. If the first unknown-group activity is `/help` or another slash command, the bot replies with the old auth message and never warns or leaves as configured.
  - confidence: 0.86
- **[P2] Document the new Telegram config keys:** `src/config/zod-schema.providers-core.ts:283-285`
  - body: These schema additions expose new user-facing Telegram config, but the Telegram/channel docs still have no `unknownGroup*` guidance. Users need the supported config shape, defaults, and upgrade/security behavior before this ships.
  - confidence: 0.82
