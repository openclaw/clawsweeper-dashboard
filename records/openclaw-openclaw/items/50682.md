## Summary

Keep this PR open. Current main still lacks the proposed Telegram unknown-group config and warn/leave behavior, and the paired open issue #50681 tracks the same user need, but the PR patch has blocking correctness gaps around wildcard group policy, native slash commands, forum General-topic warnings, and cooldown state.

## What This Changes

The PR adds optional Telegram `unknownGroupAction`, `unknownGroupMessage`, and `unknownGroupCooldownMs` config, a new unknown-group cooldown helper, runtime warn/leave handling in the Telegram message handler, `leaveChat` test harness wiring, and focused Telegram tests.

## Best Possible Solution

Keep the PR and paired issue open, then land the feature only after the Telegram plugin has one shared unknown-group enforcement path for normal messages and native commands, precise unknown-chat classification that does not treat wildcard-configured groups as unknown, `buildTelegramThreadParams` for warning sends, bounded cooldown storage, config validation or diagnostics for no-message warn mode, docs/changelog coverage, and targeted tests for wildcard groups, commands, callbacks, forum General topics, and warn/leave cooldown behavior.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure Telegram with `groupPolicy: "allowlist"`, `groups: {"*": {...}}`, no sender allowlist, and `unknownGroupAction: "leave"`; current group policy can return `group-policy-allowlist-empty`, which the PR classifies as unknown. A slash command in an unlisted group also exercises `resolveTelegramCommandAuth`, which the PR does not change.

## Solution Assessment

No. The requested direction is plausible for the Telegram plugin, but this patch is not the best implementation until unknown-group enforcement is shared across message and native command paths and uses the existing thread/group-policy helpers correctly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop treating wildcard allowlists as unknown groups:** `extensions/telegram/src/unknown-group.ts:10`
  - body: `group-policy-allowlist-empty` is not limited to unregistered chats. Current `group-access.ts` intentionally treats wildcard `groups: {"*": ...}` as enabling the group without setting `chatExplicitlyAllowed`, so this PR can warn or `leaveChat` from configured wildcard groups when no sender allowlist is present.
  - confidence: 0.93
- **[P2] Use the thread helper for forum warnings:** `extensions/telegram/src/bot-handlers.runtime.ts:1659-1662`
  - body: The new warning send passes `resolvedThreadId` directly. For forum General topics, `resolvedThreadId` can be `1`, and current `buildTelegramThreadParams()` omits that value because Telegram rejects it; with `unknownGroupAction: "leave"`, the warning can fail and be swallowed before the bot leaves.
  - confidence: 0.9
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1646-1650`
  - body: The PR only runs warn/leave from the regular message-like handler. Telegram native commands are authorized in `resolveTelegramCommandAuth` and reject unknown groups with a fixed message before the regular handler runs, so a first `/help` or plugin command in an unknown group bypasses the configured ignore/warn/leave action.
  - confidence: 0.88
- **[P2] Prune expired cooldown entries:** `extensions/telegram/src/unknown-group.ts:16-27`
  - body: `lastActionAt` is only appended to and never pruned. A long-running bot that is added to many unique unknown groups will retain one map entry per chat even after cooldowns expire or after a successful leave, so the new feature adds unbounded memory growth.
  - confidence: 0.82
- **[P3] Surface warn mode without a message:** `extensions/telegram/src/unknown-group.ts:39-43`
  - body: `unknownGroupAction: "warn"` with no `unknownGroupMessage` records cooldown but sends nothing, making the configured action indistinguishable from ignore. This should be rejected or diagnosed so operators know their fail-closed response is not visible.
  - confidence: 0.75
