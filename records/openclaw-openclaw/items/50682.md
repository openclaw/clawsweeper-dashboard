## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Classify only truly unknown chats as unknown:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also returned for wildcard-matched groups with no sender allowlist entries, which current code treats as distinct from an unregistered chat. With `unknownGroupAction: "leave"`, this can warn or eject the bot from a wildcard-configured group that is intentionally admitted for sender filtering.
  - confidence: 0.9
- **[P2] Reuse thread params for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: The new warning path passes `{ message_thread_id: resolvedThreadId }` directly. For General forum topics, current Telegram send helpers deliberately omit `message_thread_id=1` because Telegram rejects message sends there, so warn/leave can fail to send the configured warning before leaving.
  - confidence: 0.87
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1645-1647`
  - body: Native command middleware is registered before the full message handler, and command auth handles `group-chat-not-allowed` with a static reply. A `/help` or plugin command from an unknown group therefore bypasses this new warn/leave path entirely.
  - confidence: 0.86
- **[P2] Keep the config schema surface aligned:** `src/config/zod-schema.providers-core.ts:280-282`
  - body: Adding accepted `unknownGroup*` keys here without regenerating bundled channel config metadata or documenting the Telegram config leaves `config.schema`, Control UI metadata, and public docs unaware of the new options. Update the generated metadata and docs with the runtime change.
  - confidence: 0.82
- **[P3] Expire unknown-group cooldown entries:** `extensions/telegram/src/unknown-group.ts:27-31`
  - body: `lastActionAt` never removes expired chats, so a long-running bot exposed to many unknown group ids retains one map entry per chat even after the cooldown has passed or after leaving. Prune expired entries in `isOnCooldown` or bound the tracker.
  - confidence: 0.78
