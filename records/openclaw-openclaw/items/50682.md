- **[P1] Narrow unknown-group detection before leaving chats:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `group-policy-allowlist-empty` is also used for configured wildcard groups that are blocked because sender allowlists are missing. Including it here means `unknownGroupAction: "leave"` can eject the bot from a group covered by `groups["*"]` rather than only from an unregistered chat.
  - confidence: 0.9
- **[P2] Use thread helpers for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: `resolvedThreadId` can be `1` for a forum's General topic, and existing Telegram send helpers intentionally omit that value because Telegram rejects `message_thread_id=1`. Build the warning params through the same thread helper used by other sends before calling `sendMessage`.
  - confidence: 0.89
- **[P2] Run unknown-group actions for native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1645`
  - body: This action branch only runs inside the normal message handler, but native command handlers are registered before it and authorize `/help`, plugin commands, and similar messages separately. Unknown groups that first send a slash command still get the static denial path instead of the configured ignore/warn/leave behavior.
  - confidence: 0.87
