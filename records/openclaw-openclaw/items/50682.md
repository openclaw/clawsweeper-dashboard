## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Apply unknown-group actions to command updates:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1645`
  - body: The new warn/leave handling only runs in the normal message handler. Native and plugin slash commands are registered before `registerTelegramHandlers`, and grammY command middleware does not fall through unless it calls `next`, so `/help` or a plugin command in an unknown group still gets the old `This group is not allowed.` reply and never leaves. Move the unknown-group action into the shared command authorization path or an earlier shared preflight so commands obey the same config.
  - confidence: 0.89
- **[P2] Use the thread helper for warning sends:** `extensions/telegram/src/bot-handlers.runtime.ts:1660`
  - body: This direct `message_thread_id` object reintroduces the General-topic case that `buildTelegramThreadParams` already avoids. In a forum's General topic, `resolvedThreadId` can be `1`, and Telegram rejects `sendMessage(...thread_id=1)`, so `warn` fails and `leave` can leave without the configured warning.
  - confidence: 0.91
- **[P2] Do not classify sender-allowlist gaps as unknown chats:** `extensions/telegram/src/unknown-group.ts:11-12`
  - body: `group-policy-allowlist-empty` means sender authorization has no configured allowlist after the chat-level check, not necessarily that the chat is unregistered. Treating it as an unknown group lets `unknownGroupAction: "leave"` eject the bot from wildcard-enabled or otherwise intentionally covered groups that simply need sender auth configuration, creating a second meaning for the new option.
  - confidence: 0.78
- **[P2] Publish the new config contract consistently:** `src/config/zod-schema.providers-core.ts:283-285`
  - body: The branch adds three public Telegram config keys to the runtime schema, but it does not update the Telegram docs or generated channel config metadata. That leaves setup/UI/docs consumers with a stale contract for a security-sensitive option, contrary to the repo's config-surface requirements.
  - confidence: 0.76
