## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Narrow the unknown-group reasons before auto-leave:** `extensions/telegram/src/unknown-group.ts:12`
  - body: `isUnknownGroupPolicyBlock()` treats `group-policy-allowlist-empty` as an unknown chat, but that reason is also reachable for wildcard group configs where the group is configured but no sender allowlist is present. With `unknownGroupAction: "leave"`, this can warn or eject the bot from groups covered by `groups: { "*": ... }` instead of only truly unregistered groups.
  - confidence: 0.88
- **[P2] Apply unknown-group actions to native commands:** `extensions/telegram/src/bot-handlers.runtime.ts:1644-1645`
  - body: The new warn/leave handling only runs in the normal message handler. Telegram native slash commands are registered separately before `registerTelegramHandlers()` and currently answer `group-chat-not-allowed` with fixed text, so `/help` or plugin commands in an unknown group bypass the configured action and leave the bot in the group.
  - confidence: 0.86
- **[P2] Use the shared Telegram thread params for warnings:** `extensions/telegram/src/bot-handlers.runtime.ts:1657-1661`
  - body: This sends `message_thread_id` directly from `resolvedThreadId`. For Telegram forum General topics, id `1` must be omitted by `buildTelegramThreadParams()`, otherwise the warning send can fail before a `leave` action and users will not see the configured message.
  - confidence: 0.9
- **[P2] Publish the new config surface everywhere:** `src/config/zod-schema.providers-core.ts:283-285`
  - body: The PR adds three strict Telegram config fields, but does not update generated channel config metadata or public group/Telegram docs. That leaves setup/status/config tooling and users without the new accepted surface, and the repository has explicit config metadata/docs checks for this contract.
  - confidence: 0.82
