## Summary

Keep this PR open but not merge-ready: it targets a real shipped installer failure, but the patch leaves the later macOS Git installer without a Homebrew bootstrap path and the supplied proof does not run the patched installer.

## What This Changes

The PR moves the macOS Homebrew preflight in `scripts/install.sh` behind the Node version check and loosens the related installer-order test regex.

## Best Possible Solution

Keep the Node-present Homebrew skip, but ensure Homebrew is bootstrapped before every macOS package-install path that needs `brew`, including Git, then add focused installer coverage and real macOS terminal proof.

## Reproduction Assessment

Yes, as a source-level path: current main runs `install_homebrew` before `check_node`, and `install_homebrew` exits for non-admin macOS users without Homebrew. I did not run a live non-admin macOS installer.

## Solution Assessment

No. The Node-first direction is right, but the implementation should ensure Homebrew before every macOS brew-backed package install instead of only when Node is missing.
