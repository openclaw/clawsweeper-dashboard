## Summary

Keep open. The bug is valid and the PR moves in the right direction, but it is not merge-ready because the macOS missing-Git path can now call `brew install git` without first ensuring Homebrew exists, and the supplied proof is static parser output rather than a real patched installer run.

## What This Changes

The PR moves `scripts/install.sh` Homebrew setup from unconditional preparation into the missing/outdated Node branch and updates one installer ordering test regex.

## Best Possible Solution

Keep the focused installer fix, but land it only after Homebrew is required at each macOS package-install point that needs it and the PR body includes redacted real patched-installer output for the adequate-Node/no-Homebrew path.

## Reproduction Assessment

Yes, source-reproducible: current `main()` calls `install_homebrew` before `check_node`, so a no-Homebrew non-admin macOS user exits before the adequate Node check. I did not run a live non-admin macOS installer session in this read-only review.

## Solution Assessment

No, not as written. Moving Homebrew behind the Node check is the right direction, but Homebrew is also required by the macOS Git install path, so the dependency guard should move to each macOS package install point instead of only the Node branch.
