## Summary

Keep open: the PR targets a real installer bug, but the patch still breaks macOS paths that need Git because `install_git` still depends on `brew` and Homebrew is now skipped whenever Node is adequate; the supplied proof also does not run the patched installer.

## What This Changes

The branch moves `scripts/install.sh` Homebrew setup behind the Node version check, loosens one installer-order test regex, and syncs the UI raw-copy baseline.

## Best Possible Solution

Preserve the adequate-Node skip while ensuring every macOS path that may call `install_git` has a Homebrew-aware fallback, add focused installer coverage for adequate-Node/no-Homebrew/no-Git cases, and land with real patched-installer macOS proof.

## Reproduction Assessment

Yes, from source inspection. Current main performs Homebrew setup before Node detection, while the PR makes adequate-Node macOS users skip Homebrew even though later `install_git` calls still require `brew` when Git is missing.

## Solution Assessment

No. Moving Homebrew fully inside the Node-missing branch fixes one path but regresses later Git installation paths; the narrower solution is to skip Homebrew only when no later macOS package-manager operation needs it.
