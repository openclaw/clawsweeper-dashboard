## Summary

Keep open: the proposed Node-present installer fix is directionally right, but the patch only makes Homebrew lazy for the Node path while the later macOS Git install path still assumes `brew` exists. The PR also has test-only proof, so it still needs a code adjustment and real macOS installer evidence before merge.

## What This Changes

The PR moves `scripts/install.sh` Homebrew setup behind the Node version check and updates the installer test's ordering regex.

## Best Possible Solution

Make Homebrew an on-demand prerequisite for every macOS brew-backed install path, then add focused coverage for adequate Node plus missing Git/Homebrew and provide redacted macOS terminal proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: with the PR shape, macOS can skip `install_homebrew` when `check_node` succeeds and then later call `install_git`, which still invokes `brew install git` if Git is missing. I did not execute a live non-admin macOS install in this read-only review.

## Solution Assessment

No, not as submitted. Moving Homebrew only under the Node branch fixes the reported path but misses the sibling Git dependency; the maintainable fix is to guard each macOS brew-backed installer path or centralize the Homebrew prerequisite where those paths use it.
