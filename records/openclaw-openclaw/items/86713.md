The PR moves `scripts/install.sh` Homebrew setup behind the Node version check and updates the installer test's ordering regex.

## Best Possible Solution

Keep the adequate-Node npm path brew-free, but ensure every macOS path that may install Node or Git first has Homebrew available, then add focused tests and redacted live installer proof.

## Reproduction Assessment

Yes for the source-level installer bug: current `main` calls `install_homebrew` before `check_node`, so a no-brew macOS user exits before adequate Node is considered. I did not run the live non-admin macOS account path.

## Solution Assessment

No. Moving Homebrew only inside the Node install branch fixes the npm adequate-Node path but leaves the macOS Git install path without its required brew setup.
