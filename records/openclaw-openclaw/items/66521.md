## Summary

Keep open: the PR targets a real Signal group responsiveness problem, but the current diff still starts a Signal typing side effect before current-main dispatch resolves typing suppression policy, and it lacks observable after-fix proof.

## What This Changes

The PR adds a group-only `typingCallbacks?.onReplyStart()` call in the Signal monitor before dispatching inbound messages to show typing earlier.

## Best Possible Solution

Keep the Signal group UX improvement only through a policy-aware early typing path that honors `sendPolicy`, `typingMode`, cross-channel suppression, and cleanup, with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. Apply this PR and process a Signal group inbound turn whose resolved session has `sendPolicy: "deny"` or `typingMode: "never"`; the added callback can emit typing before dispatch computes `suppressTyping`.

## Solution Assessment

No. Routing through `typingCallbacks` improves cleanup, but calling it before dispatch policy resolution is not the narrowest maintainable solution; the early start needs a policy-aware seam or should be deferred until suppression is known.
