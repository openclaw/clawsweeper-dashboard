## Summary

Keep open. The PR targets a plausible Signal group responsiveness issue, but the current diff starts a real Signal typing side effect before current-main dispatch resolves sendPolicy and typing suppression, and the PR still lacks attached real after-fix proof.

## What This Changes

The PR adds an early Signal group typing callback before inbound dispatch so group chats show typing during pre-dispatch work.

## Best Possible Solution

Move early Signal group typing into a policy-aware dispatch path after sendPolicy and suppressTyping are resolved, then cover denied and disabled-typing sessions with regression tests.

## Reproduction Assessment

Yes, from source inspection. Apply the PR and process a Signal group turn whose resolved session has `sendPolicy: "deny"` or typing disabled; the proposed pre-dispatch callback can emit typing before dispatch computes suppression.

## Solution Assessment

No. Routing through `typingCallbacks` is better than directly calling `sendTypingSignal`, but the early start still needs to be behind the same resolved policy that normal typing indicators obey.
