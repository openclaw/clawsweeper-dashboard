## Summary

Keep open, but do not merge as-is: the revised diff improved the original direct send by gating to groups and using typing callbacks, but on current main that callback can still run before sendPolicy and typing-mode suppression are known. The PR also still lacks attached real Signal proof.

## What This Changes

The PR adds a Signal group-only pre-dispatch call to `typingCallbacks.onReplyStart()` so a typing indicator starts before inbound dispatch.

## Best Possible Solution

Implement a policy-aware early Signal typing cue only after the resolved session sendPolicy and typingMode permit outbound typing, with regression coverage for deny/never cases and real Signal proof.

## Reproduction Assessment

Yes, from source inspection for the review finding. Apply the PR behavior to a Signal group turn whose session has `sendPolicy: "deny"` or `typingMode: "never"`; the pre-dispatch callback can fire before dispatch computes `suppressTyping`.

## Solution Assessment

No. Routing through `typingCallbacks` is better than the original direct send, but starting it before dispatch knows sendPolicy and typing-mode state is not the narrowest maintainable solution.
