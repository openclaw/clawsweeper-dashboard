## Summary

Keep this PR open: the reworked group-only callback is closer than the initial patch, but it still starts Signal typing before current dispatch policy can suppress indicators and it is missing the required changelog entry.

## What This Changes

The PR adds a group-only pre-dispatch `typingCallbacks.onReplyStart()` call in the Signal monitor before inbound dispatch.

## Best Possible Solution

Keep the Signal group UX improvement, but start early typing through a dispatch-resolved policy seam that honors `sendPolicy`, `typingMode`, source delivery mode, and normal teardown, then add focused tests and a changelog entry.

## Reproduction Assessment

Yes. Apply the PR and process a Signal group inbound turn whose resolved session has `sendPolicy: "deny"` or `typingMode: "never"`; the added pre-dispatch callback can emit typing before `dispatch-from-config` computes `suppressTyping`.

## Solution Assessment

No. Reusing `typingCallbacks` improves cleanup, but calling it before policy resolution is not the narrowest maintainable fix; the safer path is to expose or defer to the existing resolved typing policy.
