## Summary

Keep open: the author addressed the DM guard and cleanup path by routing through `typingCallbacks`, but the PR still starts a Signal typing side effect before current-main dispatch policy resolves `sendPolicy` and typing suppression. The external PR also still lacks attached real behavior proof beyond a manual-testing statement.

## What This Changes

The PR adds a Signal group-only `typingCallbacks?.onReplyStart()` call before inbound dispatch so group chats show typing during pre-dispatch work.

## Best Possible Solution

Introduce a policy-aware early typing hook inside the dispatch path after `sendPolicy` and typing suppression are resolved, keep it limited to Signal groups, and cover deny/disabled-typing cases with regression tests.

## Reproduction Assessment

Yes, from source inspection. Apply this PR and process a Signal group turn whose resolved session has `sendPolicy: "deny"` or typing disabled; the added pre-dispatch callback can emit typing before dispatch computes `suppressTyping`.

## Solution Assessment

No. Routing through `typingCallbacks` is an improvement over direct `sendTypingSignal`, but the call needs to move behind the same policy resolution that suppresses normal typing indicators.
