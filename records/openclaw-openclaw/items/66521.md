## Summary

Keep open: this PR targets a real Signal responsiveness problem, but the latest patch starts typing outside the existing typing-policy and delivery-suppression path, lacks inspectable live Signal proof, and is currently conflicting with main.

## What This Changes

The PR adds an early Signal group typing indicator before inbound dispatch by calling the existing typing callback path.

## Best Possible Solution

Make Signal early group typing policy-aware so it preserves `typingMode` and delivery-suppression behavior while still reducing group pre-dispatch silence when policy allows an immediate indicator.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR calls `typingCallbacks.onReplyStart()` before the current `typingMode` resolver and dispatch-suppression path can gate the typing indicator.

## Solution Assessment

No. The useful behavior should be implemented through the same typing policy surface that already owns `typingMode`, delayed typing, and suppression, not by calling the raw lifecycle callback unconditionally from the Signal monitor.
