## Summary

Keep this PR open for focused rework: the Signal group typing UX problem is plausible, but the current diff can start a visible typing indicator before current main has resolved sendPolicy and typing suppression, and the external PR still lacks attached real Signal proof.

## What This Changes

The PR adds a Signal group-only pre-dispatch `typingCallbacks?.onReplyStart()` call so group chats show typing earlier before inbound dispatch completes.

## Best Possible Solution

Resolve sendPolicy and typing-mode suppression before starting any early Signal group typing cue, add regression coverage for deny/never cases, then attach redacted live Signal group proof.

## Reproduction Assessment

Yes. From source inspection, applying the PR to a Signal group turn with `sendPolicy: "deny"` or `typingMode: "never"` can call the Signal typing callback before dispatch computes `suppressTyping`.

## Solution Assessment

No. Routing through `typingCallbacks` is better than the original direct send, but starting it before the policy gate is not the narrowest maintainable solution.
