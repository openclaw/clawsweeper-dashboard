## Summary

The reported rejection matches shipped and current allowlist behavior: adding an Ollama row under `models.providers.ollama.models` registers catalog metadata, but `/model` is gated by `agents.defaults.models` when that allowlist exists.

## What This Changes

The issue asks OpenClaw to select `ollama/qwen3-coder:480b-cloud` after the model is added to `models.providers.ollama.models` and the Docker container restarts.

## Best Possible Solution

Keep the existing allowlist behavior and have the operator add `ollama/qwen3-coder:480b-cloud` or `ollama/*` to `agents.defaults.models`, or remove the allowlist if all configured models should be selectable.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced: with a nonempty `agents.defaults.models` allowlist that lacks `ollama/qwen3-coder:480b-cloud`, `/model` rejects that ref before a normal reply. The issue does not include full config/logs to prove a failure after the documented allowlist entry is present.

## Solution Assessment

Yes, the existing solution is the narrowest maintainable path: add the exact provider-prefixed ref or `ollama/*` to `agents.defaults.models`, or clear the allowlist. Changing model-name validation is not supported by the source evidence because the current Ollama setup tests already preserve this colon/hyphen model id.
