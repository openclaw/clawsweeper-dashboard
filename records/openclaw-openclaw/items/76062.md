## Summary

Keep open. Current main still processes only top-level QQ Bot attachments, while C2C media carried under `msg_elements` remains separate, and the open linked PR is the paired fix candidate rather than merged behavior.

## What This Changes

The issue asks QQ Bot C2C inbound images from QQ `msg_elements` to be surfaced as agent-visible attachments after download.

## Best Possible Solution

Review, land, or replace PR #76084 with a narrow QQ Bot plugin fix that promotes non-quote `msg_elements` media into current-message attachments and covers the dispatcher path with regression tests.

## Reproduction Assessment

Yes. A focused source-level reproduction can dispatch a C2C image event with media under `msg_elements` and no top-level `attachments`, then observe current main queueing `attachments: undefined` and returning an empty attachment result.

## Solution Assessment

Yes, the best direction is a narrow QQ Bot plugin-owned normalization before `processAttachments`, with quote messages kept separate. A core channel contract change is not needed for the reported behavior.
