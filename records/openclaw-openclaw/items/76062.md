## Summary

Keep open. Current main still sends only top-level QQ Bot attachments into the inbound attachment processor, while C2C media can arrive under `msg_elements`; open PR #76084 is already paired with this issue and should be reviewed before cleanup.

## What This Changes

The issue asks QQ Bot C2C inbound images to reach the agent as attachment/media metadata instead of arriving as text-only messages.

## Best Possible Solution

Land or replace #76084 with a QQ Bot plugin-owned normalization that turns current-message `msg_elements` media into inbound attachments before `processAttachments`, while keeping quoted `msg_elements` as quote metadata.

## Reproduction Assessment

Yes. A focused code-level reproduction can dispatch a C2C event with image metadata in `msg_elements` and no top-level `attachments`, then verify current main calls `processAttachments(undefined)` and produces no `MediaPaths`.

## Solution Assessment

Yes, the narrow maintainable direction is to normalize the QQ Bot plugin event shape before inbound attachment processing; a core channel contract change would be broader than needed.
