## Summary

Keep open. Current main still only forwards top-level QQ Bot event attachments into the inbound pipeline, while the report describes C2C media arriving via `msg_elements`; the linked open PR #76084 is the right implementation candidate to review before this issue can close.

## What This Changes

QQ Bot C2C image messages should deliver downloaded image attachment metadata to the agent instead of arriving as text-only user turns.

## Best Possible Solution

Review and land or replace PR #76084 so QQ Bot normalizes current-message C2C and group `msg_elements` media before `processAttachments`, while preserving quoted media as quote metadata and adding regression coverage.

## Reproduction Assessment

Yes. A focused code-level reproduction can dispatch a C2C event with image metadata under `msg_elements` and no top-level `attachments`, then verify current main reaches `processAttachments(undefined)` and produces no agent-visible media paths.

## Solution Assessment

Yes, the narrow plugin-owned normalization before attachment processing is the best direction; this should be resolved by the linked PR or an equivalent QQ Bot plugin patch rather than a core channel contract change.
