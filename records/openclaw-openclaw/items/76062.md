## Summary

Keep open. Current main still forwards only top-level QQ Bot event attachments into the inbound pipeline, while the report and linked discussion point to C2C media arriving through `msg_elements`; the open linked PR is the right candidate to review before this issue can be resolved.

## What This Changes

QQ Bot C2C image messages should preserve downloaded image metadata so the agent receives image attachments instead of a text-only user turn.

## Best Possible Solution

Land or replace the linked QQ Bot plugin fix so current-message C2C media from `msg_elements` is normalized before `processAttachments`, with regression coverage for C2C images and quoted media handling.

Yes. A focused code-level reproduction can construct a C2C event with image metadata in `msgElements` and no top-level `attachments`, then verify current main reaches `processAttachments(undefined)` and produces no `MediaPaths`.
