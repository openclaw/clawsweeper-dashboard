## Summary

Keep open: current main still maps QQ Bot C2C queued attachments from top-level `ev.attachments` while `msg_elements` media remains separate, and the linked open PR is the paired fix candidate rather than merged behavior.

## What This Changes

QQ Bot C2C image messages should deliver downloaded image metadata to the agent as inbound attachments.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76084 with a narrow QQ Bot dispatcher fix that promotes current-message `msg_elements` media into inbound attachments while preserving quote metadata separately.

## Reproduction Assessment

Yes. A source-level reproduction can dispatch a C2C payload with image attachments under `msg_elements` and no top-level `attachments`, then observe current main passing undefined attachments into `processAttachments`.

## Solution Assessment

Yes. Normalizing current-message `msg_elements` inside the QQ Bot dispatcher is the narrowest maintainable fix because the existing inbound pipeline already handles attachments once the queued message carries them.
