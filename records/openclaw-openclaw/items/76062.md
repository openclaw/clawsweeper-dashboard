## Summary

Keep open. Current main still drops QQ Bot C2C media when the raw event carries images only in `msg_elements`: the dispatcher stores `msgElements` separately, but the inbound pipeline classifies only `event.attachments`; PR #76084 remains open and explicitly targets this issue.

## What This Changes

The issue requests QQ Bot C2C inbound images carried through `msg_elements` be surfaced to agents as media attachments instead of arriving as text-only turns.

## Best Possible Solution

Review, land, or replace PR #76084 with a QQ Bot plugin-owned normalization path that promotes current-message `msg_elements` media before `processAttachments` while preserving quoted media as quote metadata.

## Reproduction Assessment

Yes. A focused code-level reproduction can dispatch a C2C event with image metadata only in `msg_elements` and verify current main reaches `processAttachments(event.attachments)` with no attachments, producing no media fields for the agent.

## Solution Assessment

Unclear until the linked PR is reviewed and merged. The best direction is a narrow QQ Bot plugin normalization fix with dispatcher/inbound attachment regression tests, not a core channel contract change.
