## Summary

Keep open. Current main still routes only top-level QQ Bot `attachments` into inbound media processing while C2C `msg_elements` are preserved separately, and the linked open PR #76084 explicitly claims this issue with a candidate fix that should be reviewed before closing.

## What This Changes

Surface QQ Bot C2C image media carried in `msg_elements` as Agent-visible inbound attachments.

## Best Possible Solution

Land or replace PR #76084 with a narrow QQ Bot plugin fix that normalizes current-message C2C/group `msg_elements` media before attachment processing, keeps quoted media as quote metadata, and adds dispatcher regression coverage plus changelog.

## Reproduction Assessment

Yes. A focused code-level reproduction can dispatch a C2C event with image metadata only in `msg_elements` and no top-level `attachments`; current main then reaches `processAttachments(undefined)` and produces no `MediaPaths`.

## Solution Assessment

Current main is not the solution. The best direction is a QQ Bot-owned normalization step before inbound attachment processing, with tests proving current-message media is surfaced and quoted media is not misclassified.
