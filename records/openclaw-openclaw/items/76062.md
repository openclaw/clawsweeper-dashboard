## Summary

Keep open. Current main still sends only top-level QQ Bot attachments into the inbound attachment pipeline, while C2C media preserved under `msg_elements` is only used for quote handling; PR #76084 is the open paired fix candidate and is not merged.

## What This Changes

QQ Bot C2C images carried in `msg_elements` should be surfaced as inbound image attachments to the agent.

## Best Possible Solution

Review, land, or replace PR #76084 with a narrow QQ Bot plugin fix that normalizes non-quote `msg_elements` media into current-message attachments and adds dispatcher regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: dispatch a C2C image event with media under `msg_elements` and no top-level `attachments`, then current main queues `attachments: undefined` and `processAttachments` returns empty media output.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the QQ Bot plugin before `processAttachments`, preserving quote-only `msg_elements` behavior; the linked PR is the right review target rather than starting a parallel repair.
