## Summary

Keep open. Current main has shipped partial operator-managed HTTP proxy support and some provider/env proxy plumbing, but the requested first-class system/custom HTTP plus SOCKS proxy model remains incomplete across top-level config, per-tool web/browser overrides, and reviewed bypass semantics.

## What This Changes

The issue requests first-class system and custom HTTP/SOCKS proxy configuration for OpenClaw, including environment fallback plus per-agent and per-tool override surfaces.

## Best Possible Solution

Keep this as the proxy umbrella and split reviewed follow-ups for SOCKS support, web/provider/browser override surfaces, per-agent environment policy, and bypass semantics while preserving SSRF guardrails.

## Reproduction Assessment

Yes for verifying the missing capability from source inspection. Current main rejects top-level socks5:// proxy URLs, exposes no web/browser proxy override fields, and deliberately blocks explicit browser proxy routing under strict SSRF policy.

## Solution Assessment

No, this is not best handled as one broad automated patch. The maintainable path is staged core and plugin work with explicit review of SSRF, egress, browser routing, credential handling, and bypass policy.
