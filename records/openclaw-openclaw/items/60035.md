## Summary

Keep open. Current main has substantial proxy plumbing, including managed HTTP/HTTPS proxy routing, trusted env proxy helpers, model-provider request proxy overrides, and browser guardrails, but the requested first-class system/custom HTTP plus SOCKS proxy model is still missing across top-level config, SOCKS/noProxy semantics, per-agent/tool/browser overrides, and reviewed bypass policy.

## What This Changes

The issue requests first-class system and custom HTTP/SOCKS proxy configuration, including standard env fallback, per-agent overrides, tool/browser overrides, and no-proxy support.

## Best Possible Solution

Keep this as the proxy umbrella and split maintainer-approved follow-ups for SOCKS support, standard-env semantics, agent/tool/browser override contracts, credential handling, and bypass behavior while preserving current managed proxy guardrails.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a runtime bug, but yes as a source-level missing-surface check: current main still lacks SOCKS/noProxy in managed proxy config and lacks first-class per-agent/tool/browser proxy contracts.

## Solution Assessment

Is this the best way to solve the issue? No as one broad patch; the maintainable direction is staged design and implementation for config precedence, SOCKS, bypass semantics, browser egress, and credential redaction.
