## Summary

Keep open. Current main has useful operator-managed HTTP proxy routing, but the requested first-class system/custom HTTP plus SOCKS proxy model is still missing across SOCKS support, standard env fallback semantics, per-agent/tool overrides, browser profile routing, and reviewed bypass policy.

## What This Changes

The issue requests first-class proxy configuration for OpenClaw, including system proxy variables, custom HTTP/SOCKS URLs, no-proxy rules, and per-agent or per-tool overrides.

## Best Possible Solution

Keep this as the proxy umbrella and split reviewed follow-ups for SOCKS support, standard-env semantics, agent/tool/browser override contracts, credential handling, and bypass behavior while preserving the current managed proxy guardrails.
