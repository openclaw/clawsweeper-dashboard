## Summary

Keep open. Current main has partial HTTP forward-proxy support and some env/provider proxy plumbing, but the requested first-class system/custom HTTP plus SOCKS proxy model is still incomplete across top-level config, web tool overrides, browser routing, and bypass semantics.

## What This Changes

Request first-class OpenClaw proxy configuration for system/env proxies, custom HTTP/SOCKS URLs, per-agent overrides, and per-tool web/browser routing.

## Best Possible Solution

Keep this as the proxy umbrella and split reviewed follow-ups for SOCKS support, web/provider/browser override surfaces, per-agent environment policy, and bypass semantics while preserving SSRF guardrails.

## Reproduction Assessment

Yes for verifying the missing capability. Current main rejects top-level socks5:// proxy URLs, exposes no web/browser proxy override fields, and blocks proxy-routed browser navigation under strict SSRF policy.

## Solution Assessment

No, this should not be solved as one broad automated patch. The maintainable path is staged core and plugin work with explicit SSRF, egress, and bypass-policy review.
