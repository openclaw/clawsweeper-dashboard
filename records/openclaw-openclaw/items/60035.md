## Summary

Keep open. Current main has meaningful partial HTTP proxy support, including managed forward-proxy routing and some provider/media proxy plumbing, but the requested first-class system/custom HTTP plus SOCKS model is still incomplete across top-level SOCKS config, web/browser override surfaces, per-agent proxy policy, and bypass semantics.

## What This Changes

The issue requests first-class system/custom HTTP and SOCKS proxy configuration with env fallback plus agent-level and tool-level overrides for OpenClaw network calls.

## Best Possible Solution

Keep this as the proxy umbrella and split reviewed follow-ups for SOCKS support, web/browser override surfaces, per-agent environment policy, and bypass semantics while preserving SSRF guardrails.

## Reproduction Assessment

Yes for verifying the remaining missing capability. Current main rejects top-level socks5:// proxy URLs, exposes no web/browser proxy override fields, and blocks proxy-routed browser navigation under strict SSRF policy unless private-network access is deliberately enabled.

## Solution Assessment

No, the requested broad one-shot design is not the best immediate implementation path. The maintainable direction is staged core and plugin work with explicit review of SOCKS routing, noProxy behavior, per-tool overrides, and browser SSRF policy.
