## Summary

Keep open. Current main and v2026.6.8 cover important HTTP/HTTPS managed proxy and provider-request proxy paths, but the requested first-class SOCKS top-level config, custom bypass/noProxy policy, per-agent or per-tool overrides, and browser-profile proxy contract remain broader than the shipped surface.

## What This Changes

The issue requests first-class system/custom HTTP and SOCKS proxy support across OpenClaw config, environment fallback, per-agent/per-tool overrides, browser automation, and runtime network clients.

## Best Possible Solution

Keep the shipped HTTP/HTTPS managed proxy and provider request proxy as the baseline, then split a maintainer-approved proxy design into explicit surfaces for SOCKS, env precedence, bypass policy, per-agent/tool scope, browser egress, and credential handling.

No, not as one broad patch. The safer solution is a staged product/security design that preserves existing managed proxy and provider proxy behavior while adding narrow, tested contracts.
