## Summary

Keep open. Current main and the latest release have important HTTP/HTTPS managed proxy, environment-proxy, provider-request proxy, and web_fetch trusted-env-proxy pieces, but the central request still asks for broader first-class SOCKS, noProxy, per-agent, per-tool, and browser-profile proxy contracts that are not implemented.

## What This Changes

The issue requests first-class OpenClaw proxy configuration for HTTP/SOCKS/system proxies, including global config, environment fallback, per-agent overrides, per-tool overrides, and runtime injection for network clients.

## Best Possible Solution

Keep this open for a maintainer-approved proxy design that extends the shipped HTTP/HTTPS managed proxy and provider-request proxy baseline in staged, tested contracts for SOCKS, bypass policy, per-agent/tool scope, browser egress, and credential handling.

## Reproduction Assessment

Yes. This is source-reproducible as a missing-surface check: current main rejects SOCKS for top-level managed proxy config and lacks first-class per-agent, per-tool, noProxy, and browser-profile proxy fields.

## Solution Assessment

No, not as one broad top-level config patch. The safer solution is a staged product and security design that preserves existing managed proxy, provider request proxy, and web_fetch trusted-env-proxy behavior while adding narrow contracts.
