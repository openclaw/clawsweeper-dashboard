Keep open. Current main and v2026.5.18 have managed HTTP/HTTPS proxy routing plus scoped env-proxy helpers, but the requested first-class SOCKS, noProxy, standard env precedence, per-agent/tool/browser override, and reviewed security policy surface is still missing.

## Best Possible Solution

Keep this as the proxy umbrella and split maintainer-approved follow-ups for SOCKS support, standard env semantics, bypass/noProxy policy, per-agent/tool/browser contracts, credential handling, and SSRF-preserving validation.

Is this the best way to solve the issue? No as one broad patch; the maintainable path is staged design and implementation for config precedence, SOCKS, bypass semantics, browser egress, and credential redaction.
