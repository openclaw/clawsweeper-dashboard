## Summary

Keep open. Current main has partial process-level HTTP proxy and environment-proxy support, but the requested first-class SOCKS config, per-agent/tool/browser overrides, and reviewed bypass semantics are still missing.

## What This Changes

Request first-class system/env and custom HTTP/SOCKS proxy configuration across CLI config, agents, web/browser tools, and runtime network clients.

## Best Possible Solution

Keep this umbrella open, then split maintainer-scoped work for top-level SOCKS/env support, web/provider/browser overrides, per-agent environment policy, and SSRF-safe bypass semantics.

## Reproduction Assessment

Yes for verifying the missing capability, though this is a feature request rather than a runtime bug. Current source and tests reject top-level `socks5://` proxy URLs, web tool schemas expose no proxy override, and browser proxy routing is explicitly blocked under strict SSRF policy.

## Solution Assessment

No, this should not be solved as one broad patch. The maintainable path is a staged proxy design with security review for bypass semantics and browser egress behavior.
