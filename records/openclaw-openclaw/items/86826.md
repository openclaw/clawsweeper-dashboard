The PR adds a gateway session-store alias helper and calls it from session-store key migration so raw external keys can be copied alongside canonical agent-scoped keys.

## Best Possible Solution

Use one shared alias-group contract that resolves and mutates the logical raw/canonical session together across migration, patch, delete, reset, compaction, listing/search, and plugin host-hook writers.

## Reproduction Assessment

Yes. Source inspection and the linked issue establish the repro shape: create a raw external key entry, let gateway canonicalization promote or prune it, then mutate through the canonical key or use the raw key again.

## Solution Assessment

No. Preserving aliases during migration is a useful start, but the best fix must make raw and canonical aliases one logical session across every follow-up mutation and listing path.
