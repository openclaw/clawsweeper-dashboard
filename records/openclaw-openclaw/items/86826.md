## Summary

Keep open: this is a coherent fix for the raw/canonical session split bug, but the branch is still a partial implementation with no contributor real-behavior proof and is currently conflicting with main.

## What This Changes

The PR adds a gateway session-store alias helper and calls it during session-store key migration so raw external keys can be copied alongside canonical agent-scoped keys.

## Best Possible Solution

Use one shared alias-group contract that chooses and mutates the logical raw/canonical session together across migration, patch, delete, reset, compaction, list/search, and plugin host-hook writers.

## Reproduction Assessment

Yes, source inspection and the linked issue establish a repro shape: create a raw external key entry, let gateway canonicalization promote or prune it, then mutate through the canonical key or use the raw key again. I did not run a live external-channel replay in this read-only review.

## Solution Assessment

No. Preserving aliases during migration is a useful start, but the best fix must make raw and canonical aliases one logical session across every follow-up mutation and listing path.
