That breaks existing `replyToMode: "off"` and single-use modes by making Google Chat delivery ignore the shared reply policy.
  - confidence: 0.88

  - body: `openclaw/plugin-sdk/inbound-envelope` is a published SDK subpath, and this adds a new `sessionKey` input to its returned builder for a Google Chat-local need without a documented SDK direction. Keep the override local to the Google Chat adapter or make the SDK change explicit with contract docs and tests.
