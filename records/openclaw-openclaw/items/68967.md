## Summary

Keep open: this is useful, real-proven Google Chat session-isolation work, but the branch is not ready because it bypasses existing reply policy, targets stale config-schema structure, and overlaps with active Google Chat thread-delivery repair work.

## What This Changes

The PR adds an opt-in Google Chat `sessionThread` channel option, derives thread-partitioned session keys, routes replies toward inbound Chat threads, updates config/docs/generated metadata, and adds focused session-key tests.

## Best Possible Solution

Rebase and narrow this to a current-main per-thread session feature that preserves existing reply policy, aligns any delivery-threading changes with the active narrower Google Chat repair, ports config/schema/docs together, and leaves `CHANGELOG.md` to release generation.

## Solution Assessment
