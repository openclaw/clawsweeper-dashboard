Overall correctness: patch is incorrect

Full review comments:

- **[P1] Move sessionThread into the active Google Chat schema:** `src/config/zod-schema.providers-core.ts:871`
  - body: `sessionThread` is added to the old provider-core schema, but current Google Chat plugins export `GoogleChatConfigSchema` from `src/config/zod-schema.providers-googlechat.ts`. Because that schema is strict and does not include this key, the documented `channels.googlechat.sessionThread` setting can still be rejected by the active plugin config validation path after merge.
  - confidence: 0.91
- **[P1] Preserve replyToMode before forcing the inbound thread:** `extensions/googlechat/src/monitor-reply-delivery.ts:28`
  - body: Falling back to `forcedThreadName` when `payload.replyToId` is absent bypasses existing reply fanout semantics. For `replyToMode: "off"` or after implicit reply fanout has intentionally cleared the payload target, this sends the reply back into the inbound thread instead of honoring the configured top-level/no-reply behavior.
  - confidence: 0.88
- **[P1] Avoid widening the public inbound-envelope helper for one channel:** `src/plugin-sdk/inbound-envelope.ts:46`
  - body: This adds `sessionKey` to `openclaw/plugin-sdk/inbound-envelope`, a published SDK subpath, to solve one Google Chat timestamp lookup. That creates a new public helper input without the SDK docs/API discussion and gives plugins a second session-key override path; keep the override plugin-local or promote a deliberate SDK contract with matching docs and tests.
  - confidence: 0.84
