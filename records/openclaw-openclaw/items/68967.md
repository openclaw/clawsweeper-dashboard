## Summary

Keep this PR open: the per-thread Google Chat session feature is not on current main and the screenshots are useful real behavior proof, but the branch is not merge-ready because it can bypass the existing reply threading policy and still conflicts with the canonical Google Chat thread-delivery work.

## What This Changes

The PR adds an optional Google Chat `sessionThread` config, per-thread session-key derivation, reply-thread forwarding, config/docs metadata, and focused tests.

## Best Possible Solution

Land a rebased change that keeps Google Chat per-thread session storage opt-in and default-off, preserves space-based route matching, and leaves reply placement to a replyToMode-aware delivery path coordinated with https://github.com/openclaw/openclaw/pull/74235.
