## Summary

Keep open: the per-thread Google Chat session capability is still absent on current main and this PR is the canonical implementation attempt for that request, but it is not merge-ready because the branch is conflicting and the reply-delivery fallback bypasses the existing Google Chat replyToMode policy.

## What This Changes

The PR adds an opt-in Google Chat `sessionThread` config option, derives thread-scoped session keys from inbound thread names, plumbs the derived key into inbound context/envelope handling, forces replies toward inbound threads, updates docs/generated config metadata and CHANGELOG, and adds focused tests.

## Best Possible Solution

Rebase onto current Google Chat code, keep the per-thread session mode as a deliberate opt-in only if maintainers accept the config direction, preserve `replyToMode`, update the current schema/docs/generated surfaces, and leave broader reply-delivery repair to the canonical reply-routing PRs.
