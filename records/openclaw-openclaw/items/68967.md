## Summary

## What This Changes

## Best Possible Solution

Keep the per-thread session work alive, but rebase and narrow it to a maintainer-approved Google Chat session option, move validation to the current schema, preserve `replyToMode`, avoid unapproved SDK expansion, and leave release notes in the PR body or squash message rather than `CHANGELOG.md`.
