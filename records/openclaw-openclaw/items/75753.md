## Summary

Keep open. Current main still has a credible compatibility gap where legacy `claude-cli/*` runtime model refs can be normalized to canonical `anthropic/*` refs in the effective allowlist while cron still validates the raw `payload.model` string against that effective set; the reporter also linked #75763 and #75854 as related silent cron/model-validation regressions, which are useful cluster context rather than a reason to close this report.

## What This Changes

The issue asks OpenClaw to keep `claude-cli/*` entries from `agents.defaults.models` usable for cron `payload.model` validation after the 2026.4.29 regression.

## Best Possible Solution

Keep canonical `anthropic/*` model selection and `agentRuntime.id: "claude-cli"` as the preferred config shape, but preserve legacy `claude-cli/*` refs as accepted aliases in the cron/model allowlist and diagnostics so older cron payloads do not break after migration.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to run a config with a legacy `claude-cli/*` primary and matching `agents.defaults.models` entry through the compatibility normalizer, then validate a cron `payload.model: "claude-cli/..."` through `resolveCronModelSelection`; current tests already assert the legacy allowlist key is removed during normalization.

## Solution Assessment

Unclear because there is no patch yet. The narrowest maintainable fix is to add an alias-preserving allowlist path for legacy runtime refs while leaving canonical provider/runtime migration intact.
