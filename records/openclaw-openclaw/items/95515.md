## Summary

Keep open: current main and latest v2026.6.9 still allow the doctor migration to write `groupAllowFrom` when bundled schema metadata is missing, the narrow fix is closed unmerged, and the broader related PR is still an open draft rather than a proven replacement.

## What This Changes

The issue reports that upgrading from 2026.6.8 to 2026.6.9 writes an invalid `groupAllowFrom` key into an email channel account config, causing gateway startup validation to fail.

## Best Possible Solution

Land a focused doctor migration guard for unknown external channel schemas with regression coverage, or explicitly include and prove that same guard in the broader channel-config protection work.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a reachable current-main doctor migration path that writes `groupAllowFrom` when a channel lacks bundled schema metadata, which matches the reported external email plugin failure mode.

## Solution Assessment

No, current main is not the best final state because it still treats missing bundled schema metadata as permission to write. The narrow maintainable fix is to skip `groupAllowFrom` writeback when the channel schema is unknown and cover that case in the migration test.
