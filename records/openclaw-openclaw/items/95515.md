## Summary

Keep open: current main and the latest release still have the permissive missing-schema doctor migration path, and https://github.com/openclaw/openclaw/pull/95540 is the active open closing PR for this bug, so closing the issue before that PR lands would be premature.

## What This Changes

The issue reports that upgrading from 2026.6.8 to 2026.6.9 writes `groupAllowFrom` into an external email channel account, making the saved config fail validation and blocking Gateway startup.

## Best Possible Solution

Land the focused schema-verified doctor guard with the external email regression test, or explicitly supersede it with broader config-protection work that proves the same email case.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.9 allow missing-schema writes and then assign `groupAllowFrom` onto account records. This review did not run a full macOS ClawMail upgrade reproduction.

## Solution Assessment

Yes, the open focused guard is the narrow maintainable fix if maintainers accept that core doctor must not mutate schemaless external plugin config. The issue should remain open until https://github.com/openclaw/openclaw/pull/95540 lands or is replaced by an equivalent proven fix.
