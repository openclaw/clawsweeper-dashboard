## Summary

Keep open: current main and the latest v2026.6.10 release still contain the permissive missing-schema doctor migration path, the reporter added fresh 2026.6.9 to 2026.6.10 live evidence, and https://github.com/openclaw/openclaw/pull/95540 is the current open candidate fix.

## What This Changes

The issue requests that upgrade/doctor preserve external email channel account config without injecting the Feishu-style `groupAllowFrom` field.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95540, or an equivalent schema-verified doctor guard with external-channel regression coverage, then close this issue after the fix is on main or shipped.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.10 still allow missing-schema doctor writes and then assign `groupAllowFrom` onto account records. The reporter also supplied a fresh 2026.6.9 to 2026.6.10 live upgrade reproduction, though this review did not rerun the packaged ClawMail path.

## Solution Assessment

Yes: the open candidate fix changes the exact write decision so core only writes `groupAllowFrom` when bundled schema metadata verifies that path. If maintainers want external plugins to receive this migration, the safer alternative is an explicit plugin doctor or schema opt-in rather than a core missing-schema default.
