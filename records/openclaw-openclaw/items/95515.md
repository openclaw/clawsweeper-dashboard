## Summary

Keep open: current main and v2026.6.10 still have the permissive missing-schema doctor migration that can write `groupAllowFrom` into external email channel accounts, and https://github.com/openclaw/openclaw/pull/95540 is the active narrow candidate fix. The reporter added live v2026.6.10 upgrade evidence showing the bug still fires.

## What This Changes

Prevent upgrade/doctor repair from injecting `groupAllowFrom` into external email channel account config.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95540 or an equivalent schema-verified doctor guard, then close this issue after merged main or release proof.

## Reproduction Assessment

Yes for a source-level current-main reproduction path: doctor repair calls the migration, missing generated schema currently passes the write guard, and the migration writes `groupAllowFrom` onto account records. I did not run the full macOS ClawMail upgrade in this read-only review.

## Solution Assessment

Yes: the best fix path is the open narrow schema-verified doctor guard or an equivalent change at the same write decision point. Broader post-write validation could be useful defense in depth, but it should not replace preventing the illegal write.
