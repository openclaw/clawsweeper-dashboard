## Summary

Keep open: current main and v2026.6.9 still contain the permissive doctor migration path that can write `groupAllowFrom` into a schema-missing external email channel account, and https://github.com/openclaw/openclaw/pull/95540 is the active closing fix candidate but is not merged yet.

## What This Changes

The issue reports that upgrading from 2026.6.8 to 2026.6.9 corrupts external email channel config by adding an invalid `groupAllowFrom` field under `channels.email.accounts.default`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95540, or an equivalent schema-verified doctor guard with the external email regression test, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes for source-level reproduction: with an external channel such as email absent from bundled metadata, current main allows the migration and writes `groupAllowFrom` onto account config. I did not run the full macOS ClawMail upgrade path in this read-only review.

## Solution Assessment

Yes, the active fix direction is the narrow maintainable path because it changes the exact schema guard before core writes external plugin config. The issue should stay open until that PR or an equivalent fix is merged.
