## What This Changes

The issue reports that upgrading from OpenClaw 2026.6.8 to 2026.6.9 injects an invalid `groupAllowFrom` field into an external email channel account config, causing Gateway startup validation failure.

## Best Possible Solution

Land a focused doctor migration guard, with regression coverage for a schema-missing external channel account, or explicitly fold that fix into the broader protected-config work before closing this issue.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter supplied a concrete corrupted config diff, and current source shows a reachable doctor migration path that writes `groupAllowFrom` when an external channel lacks bundled schema metadata.

## Solution Assessment

No, current main is not the best final state because it still trusts missing bundled schemas. The narrow maintainable fix is to make doctor writebacks conservative for unknown external channel schemas and add focused regression coverage.
