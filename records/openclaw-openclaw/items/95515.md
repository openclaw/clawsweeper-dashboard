## What This Changes

The issue reports that upgrading from OpenClaw 2026.6.8 to 2026.6.9 writes `groupAllowFrom` into an external email channel account config, causing Gateway config validation to fail.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95540 or an equivalent fix that leaves schemaless external channel configs untouched while preserving bundled channel migration behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. The source path is clear: current `main` and v2026.6.9 permit missing-schema writes, and the repair path assigns `groupAllowFrom` when `allowFrom` exists.

## Solution Assessment

Yes, the linked PR is the best current fix path if maintainers accept plugin-owned external repairs. It changes the exact schema-missing write guard and adds focused regression coverage; the alternative is an explicit external plugin doctor/schema opt-in before core writes plugin-owned keys.
