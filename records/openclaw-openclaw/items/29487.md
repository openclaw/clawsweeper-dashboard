## Summary

Keep open. Current main and the latest release handle Feishu quoted context only through `message.parent_id`, while the later issue discussion identifies a separate Feishu quote UI target (`quote.message_id`) that is not modeled, parsed, fetched, or covered by tests; the open quoted-media PR is related but does not close this quote-target gap.

## What This Changes

The issue asks the Feishu plugin to fetch quoted or parent message content and include it in the agent input when a user quote-replies and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the payload field, prefer it before the existing `parentId` fallback, preserve visibility and empty-message safeguards, and add quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a conditional path: a Feishu quote event with a quote-only target and no `message.parent_id` would leave `ReplyToBody` unset on current main.

## Solution Assessment

Yes, if the payload field is verified first. The narrow maintainable solution is inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` quoted-message path rather than adding core behavior.
