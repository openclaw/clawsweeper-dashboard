## Summary

Keep open. Current main fetches Feishu quoted context only from `message.parent_id`; the discussion-identified quote UI target is still not modeled, parsed, or fetched, so the central quote-and-mention workflow is not fully handled.

## What This Changes

Request Feishu include quoted or parent message content in agent context when a user quote-replies while mentioning the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the actual payload field, prefer that target before `parent_id` fallback in the existing fetch path, preserve group visibility checks, and add quote-only regression tests.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a conditional reproduction path: a Feishu event whose quote target is not mirrored into `message.parent_id` will parse no quote target and leave `ReplyToBody` unset.

## Solution Assessment

Yes. The narrow maintainable direction is to handle the Feishu quote target inside the Feishu plugin event/context model and reuse the existing quoted-message fetch path, after verifying the upstream payload field.
