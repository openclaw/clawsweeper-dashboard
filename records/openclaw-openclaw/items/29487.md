## Summary

Keep open. Current main and the latest release handle Feishu quoted context when `message.parent_id` is present, but the later discussion identifies Feishu UI quote replies using a separate quote target that is not modeled, parsed, fetched, or covered by tests.

## What This Changes

The issue requests that Feishu quoted or parent message content be included in the agent input when a quoted message mentions the bot.

## Best Possible Solution

Verify the real Feishu quote payload, then add a plugin-local quote target that falls back to `parentId`, preserves context-visibility filtering, and adds quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available yet. Source inspection shows a quote-only event would be ignored today, but the exact Feishu/Lark payload field still needs official contract or redacted live-event proof.

## Solution Assessment

Mostly yes, if the payload field is verified first. The maintainable path is to reuse the existing Feishu `getMessageFeishu` and context-visibility flow inside the plugin, not add core behavior or guess the event shape.
