## Summary

Keep open. Current main still only models and fetches Feishu quoted/replied context through `message.parent_id`; the issue discussion identifies a separate Feishu quote UI target that is not represented, and the related quoted-media PR does not close that quote-target gap.

## What This Changes

Request Feishu quoted or parent message content to be included in the agent input when a quote-plus-mention event is forwarded.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the payload field, prefer it before the existing parent_id fallback, preserve group visibility and empty-message safeguards, and add quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a conditional path: a Feishu quote event with a quote-only target and no `message.parent_id` would leave `ReplyToBody` unset on current main.

## Solution Assessment

Yes, if the payload field is verified first. The narrow maintainable fix is inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` quoted-message path rather than adding core behavior.
