## Summary

Keep open: current main has shipped support for fetching text context from Feishu `parent_id`, but it still models only `parent_id` and would ignore the separate quote identifier described in the discussion; the broader quoted reply PRs are closed unmerged, so no safe merged fix owns the remaining behavior.

## What This Changes

The issue requests Feishu quoted or parent message content to be included in the agent input when a user quotes a message and mentions the bot.

## Best Possible Solution

Confirm the Feishu UI quote event shape, then keep the fix plugin-local: normalize the supported quoted-message id into Feishu inbound context, reuse the existing safe `getMessageFeishu` and context-visibility path, and add focused parent-id plus quote-id regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main would ignore a payload that supplies only `quote.message_id`, but the pinned Feishu SDK type only documents `parent_id`, so the live payload contract is the missing proof.

## Solution Assessment

Unclear until the Feishu quote event contract is confirmed. The maintainable shape is a narrow Feishu-plugin normalization and fetch-path reuse, not a core change or broad quoted-media/card refactor by default.
