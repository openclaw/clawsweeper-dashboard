## Summary

Keep open. Current main supports Feishu quoted context only when `message.parent_id` is present; it still does not model, parse, or fetch the discussion-identified quote UI target, and the open related PRs found in search are media-focused rather than this target-extraction gap.

## What This Changes

The issue requests Feishu/Lark inbound quote or reply context to be included in the agent-facing message when a user quotes a message and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local support for the explicit quote target, keep `parent_id` as a fallback, preserve existing group context visibility checks, and cover quote-only events with regression tests.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a conditional reproduction path: a Feishu event whose quote target is outside `message.parent_id` will parse no quote target, skip `getMessageFeishu`, and leave `ReplyToBody` unset.

## Solution Assessment

Yes. The narrow maintainable solution is to handle quote-target extraction inside the Feishu plugin’s event/context model and reuse the existing quoted-message fetch path, without adding core behavior or a config option.
