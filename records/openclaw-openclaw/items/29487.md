## Summary

Keep open. Current main and the latest release still only fetch Feishu quoted context through `message.parent_id`; the issue discussion points to a separate Feishu UI quote target that is not modeled, parsed, fetched, or covered by tests yet.

## What This Changes

Request Feishu quote or parent message content to be fetched and included in the agent context when forwarding inbound events.

## Best Possible Solution

Add a Feishu-plugin-local quote-target field after verifying the real payload shape, prefer it before the existing `parentId` fallback, preserve context-visibility filtering, and add quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows that any quote-only event without `message.parent_id` would be ignored by current main, but the exact Feishu quote payload field still needs contract or live-event proof.

## Solution Assessment

Mostly yes, if the payload field is verified first. The maintainable fix belongs inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` path rather than adding core behavior.
