## Summary

Keep open. Current main handles Feishu quoted context when `message.parent_id` is present, but it still does not model a separate UI quote field and does not use `reply_target_message_id` as the fetched quoted-content source; the active canonical fix path is https://github.com/openclaw/openclaw/pull/84588, which needs maintainer judgment before this issue can close.

## What This Changes

The issue requests Feishu/Lark quoted or parent message content to be fetched and included in agent input when a user quote-replies and mentions the bot.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/84588 as the Feishu-owned fix, then close this issue after the accepted implementation is on main.

## Reproduction Assessment

No high-confidence live reproduction is available in this review. Source inspection shows that a Feishu event with only a quote/reply target and no `parent_id` would skip the quoted-content fetch path, but the exact live Feishu payload still needs proof.

## Solution Assessment

Unclear and product-dependent. A plugin-local fix that reuses `getMessageFeishu` and context-visibility filtering is plausible, but the current active PR also changes reply routing semantics and needs maintainer acceptance before it is the best final solution.
