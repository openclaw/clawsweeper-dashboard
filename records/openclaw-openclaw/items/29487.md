## What This Changes

The issue requests that Feishu quote/reply events include the quoted parent message content in the agent input.

## Best Possible Solution

Verify the real Feishu quote payload, then add plugin-local quote-target extraction that falls back to `parentId`, reuses `getMessageFeishu` and context-visibility filtering, and coordinates with the active quoted-reply PR if that becomes the landing path.

## Solution Assessment
