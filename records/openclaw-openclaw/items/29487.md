## Summary

Keep open. Current main still fetches Feishu quoted/replied context only through `parent_id`; it does not model, parse, prefer, or test a distinct UI quote message id, even though the issue discussion identifies that as the remaining gap after the earlier fetched-message parsing work.

## What This Changes

Request that Feishu quote+mention events include the quoted or parent message content in the agent input by fetching the referenced message and prepending it to the new message.

## Best Possible Solution

Feishu should carry a distinct quoted-message id through the inbound event model and use it as the fetch target before falling back to `parent_id`, while preserving existing group `contextVisibility` filtering and reply metadata behavior.
