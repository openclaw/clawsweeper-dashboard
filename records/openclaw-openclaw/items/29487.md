## Summary

## What This Changes

Requests Feishu quoted or parent message content be included in the agent-visible inbound context when a user quotes a message and mentions the bot.

## Best Possible Solution

Confirm the live Feishu UI quote payload, then keep the fix plugin-local by normalizing the supported quoted-message id/body into existing Feishu context-visibility and `getMessageFeishu` paths while preserving `parent_id` behavior.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main handles `parent_id` payloads but would ignore a payload that supplies only a separate quote id/body field.

## Solution Assessment

Unclear until the live Feishu quote event contract is confirmed. The maintainable fix would be a narrow Feishu-plugin normalization and regression tests, not a core change or broad quoted-media/card refactor by default.
