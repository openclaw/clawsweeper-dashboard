## Summary

Keep open. Current main still fetches Feishu quoted context only from `parent_id`; the discussion-identified quote UI path is not modeled, parsed, selected for fetch, or covered by tests, while the related #29501 work only improved fetching/parsing after an ID is already selected.

## What This Changes

Request that Feishu inbound events include quoted or parent message content in the agent input when a user quote-replies and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local support for the explicit quote target, keep existing `parent_id` fallback and context-visibility filtering, and cover quote-only events with regression tests.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows a quote-only event carrying its target outside `message.parent_id` would parse no quote target and skip `getMessageFeishu`, matching the reported failure mode but still needing payload-shape verification.

## Solution Assessment

Yes. The narrow maintainable direction is to handle Feishu quote-target extraction inside the Feishu plugin’s event/context model and fetch selection, without adding core behavior or a config option.
