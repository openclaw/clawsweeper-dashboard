## Summary

Keep open. Current main and v2026.5.12 still fetch Feishu quoted context only through `message.parent_id`, while the discussion identifies a separate Feishu UI quote target that is not modeled, parsed, fetched, or covered by tests.

## What This Changes

The issue asks the Feishu plugin to include quoted or parent message content in agent context when a user quote-replies while mentioning the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the payload field, prefer it before the existing `parentId` fallback, preserve `contextVisibility` filtering, and add quote-only regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a conditional reproduction path: a Feishu quote event with a quote-only target and no `message.parent_id` would leave `ReplyToBody` unset on current main.

## Solution Assessment

Yes, if the payload field is verified first. The maintainable fix belongs inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` quoted-message path rather than adding core behavior.
