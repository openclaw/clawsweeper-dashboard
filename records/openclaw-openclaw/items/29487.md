## Summary

Keep open. Current main handles Feishu `parent_id` reply context and includes the related parsing work from #29501, but the UI quote-id path identified in the discussion still has no event type, parsed context field, fetch selection, reply metadata fallback, or quote-only regression coverage.

## What This Changes

Request Feishu plugin handling that fetches and prepends quoted or parent message content for quote+mention events so the agent receives the user-provided context.

## Best Possible Solution

Add Feishu-plugin-local quote-id handling that preserves parent-id fallback, existing group context visibility filtering, and current fetched-message parsing, then cover quote-only events with regression tests.

## Reproduction Assessment

Yes at source level, with medium confidence: an inbound Feishu quote event whose quoted message id is present only in a separate quote field cannot reach `getMessageFeishu()` because current parsing and dispatch only carry `parentId`. A live Feishu payload should confirm the exact field path before editing.

## Solution Assessment

Yes. The narrow maintainable fix belongs inside the bundled Feishu plugin event/context model and dispatch fetch selection, not in core or behind a new configuration option.
