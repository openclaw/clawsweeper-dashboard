## Summary

Keep open. Current main and the latest release handle Feishu quoted context only when `message.parent_id` is present; the later issue discussion identifies a separate Feishu UI quote target that is still not modeled, parsed, fetched, or covered by tests.

## What This Changes

The issue requests Feishu quote/reply context be included in the agent input when users quote a message and mention the bot.

## Best Possible Solution

Verify the real Feishu quote payload, then add a plugin-local quote target that falls back to `parentId`, preserves context-visibility filtering, and adds quote-only regression coverage.

## Reproduction Assessment

No high-confidence reproduction is available yet. Source inspection shows a quote-only field would be ignored today, but the exact Feishu/Lark payload field still needs official contract or live-event proof.

## Solution Assessment

Mostly yes, if the payload field is verified first. The maintainable path is to reuse the existing Feishu `getMessageFeishu` and context-visibility flow inside the plugin, not add core behavior or guess the event shape.
