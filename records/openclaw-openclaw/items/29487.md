## Summary

Keep open. Current main and the latest release handle Feishu quoted context only when `message.parent_id` is present; the issue discussion identifies a separate Feishu UI quote target that is still not modeled, parsed, fetched, or covered by tests.

## What This Changes

The issue requests Feishu quoted or parent message content to be included in the agent input when a user quotes a message and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the real payload field, prefer it before the existing `parentId` fallback, preserve context-visibility filtering, and add quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows a quote-only event without `message.parent_id` would be ignored by current main, but the exact Feishu quote payload field still needs contract or live-event proof.

## Solution Assessment

Mostly yes, if the payload field is verified first. The maintainable fix belongs inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` path rather than adding core behavior.
