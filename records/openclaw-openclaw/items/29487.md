## Summary

Current main implements quoted context for Feishu events that carry `message.parent_id`, but the issue discussion identifies a remaining Feishu UI quote path and the viable open PR still carries the broader fix with live proof; this issue should remain the product anchor until that PR lands or is replaced.

## What This Changes

Include the content of a quoted or parent Feishu message in the agent context for quote-and-mention interactions.

## Best Possible Solution

Land a focused Feishu path that normalizes the actual UI quote/reply message identifier, safely fetches text/card/media context, and keeps that context identifier independent from explicit thread/topic outbound routing.

## Reproduction Assessment

Yes at the source level with medium confidence: current main only fetches quoted context when `message.parent_id` is present, so a Feishu UI quote delivered through a different field bypasses retrieval. A redacted live event payload for that UI action would raise this to high confidence.

## Solution Assessment

Not fully yet: the open related PR has strong live evidence and the correct plugin ownership, but its broader reply-anchor behavior and outstanding dispatcher defect need resolution before it is the best merge-ready solution.
