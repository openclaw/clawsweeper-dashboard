## Summary

Keep open: current main and the latest release support Feishu quoted context only through `message.parent_id`, while the issue discussion identifies a still-unmodeled UI quote identifier path. The open related Feishu PR has useful live proof for adjacent quoted-context work, but it is broader, currently conflicting, and still needs maintainer compatibility/product review before this issue can close.

## What This Changes

The issue requests Feishu quoted or parent message content be fetched and included in the agent input when a user quotes a message and mentions the bot.

## Best Possible Solution

Land or replace a focused Feishu plugin change that normalizes the actual quote/reply identifier from live events, fetches quoted text/card/media into agent context, and keeps outbound reply-anchor compatibility explicit.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection shows current main only handles `parent_id`, so the commenter-described `quote.message_id` path would miss quoted context, but the exact Feishu UI payload still needs redacted live proof.

## Solution Assessment

Unclear as proposed: the original `parent_id` fetch is already implemented, while the remaining likely fix is to normalize the actual UI quote identifier and keep it separate from outbound reply routing. The open related PR has useful proof but is broader and compatibility-sensitive rather than a clean close-out yet.
