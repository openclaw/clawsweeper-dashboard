## Summary

Keep open. Current main still only fetches Feishu quoted/replied content through `parent_id`; the issue discussion identifies UI quote replies as using a separate quote message id, and that id still has no event type, context field, parser mapping, fetch path, or regression test.

## What This Changes

Feishu quote+mention events should include the quoted or parent message text in the agent input instead of forwarding only the new mention text.

## Best Possible Solution

Add Feishu-plugin-local quote-id handling: type and parse an optional quote message id, prefer it for quoted-content fetching while preserving the `parent_id` fallback and existing group visibility filtering, then cover quote-only events in bot tests.

## Reproduction Assessment

Yes at source level: a Feishu message event carrying the discussed quote id but no `parent_id` cannot reach `getMessageFeishu()` because the parser drops that id and the fetch path only checks `ctx.parentId`. A live Feishu payload should still confirm the exact field path before implementation.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the bundled Feishu plugin event/context model and fetch selection, not in core or a new configuration option.
