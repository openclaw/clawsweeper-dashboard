## Summary

Keep open. Current main handles Feishu quoted/replied context only when `message.parent_id` is present; the later issue discussion identifies a separate quote UI target that is still not modeled or fetched, so the quoted-message workflow is not proven fixed.

## What This Changes

The issue requests Feishu inbound events to include quoted or parent message content in the agent input when a user quotes a message and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the actual event payload field, prefer that target before the existing `parent_id` fallback, preserve group visibility checks, and add quote-only regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a conditional path: a Feishu event whose quote target is not mirrored into `message.parent_id` will parse no quote target and leave `ReplyToBody` unset.

## Solution Assessment

Yes. The narrow maintainable direction is to handle the Feishu quote target inside the Feishu plugin event/context model and reuse the existing quoted-message fetch path after verifying the upstream payload field.
