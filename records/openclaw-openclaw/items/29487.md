## Summary

Keep open. Current main handles Feishu reply context when `parent_id` is present, and related parsing from #29501 appears landed, but the discussion-identified UI quote path still has no quote-id event field, no parsed context field, no fetch selection, no reply metadata fallback, and no quote-only regression coverage.

## What This Changes

The issue requests Feishu quoted or parent message content be included in agent input when a user quote-replies and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-id handling that preserves `parent_id` fallback, existing group context visibility filtering, fetched-message parsing, and graceful fetch failure behavior, then cover quote-only events with regression tests.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection gives a medium-confidence path: a Feishu quote event with only a separate quote id cannot fetch quoted content because current parsing and dispatch only carry `parentId`.

## Solution Assessment

Yes, the narrow maintainable direction is to handle the quote id inside the Feishu plugin’s event/context model and dispatch fetch selection, without adding core behavior or a config option.
