## Label Justifications

- `P2`: Missing quoted content blocks a common Feishu workflow but has limited channel-specific blast radius and a bounded fix shape.
- `impact:session-state`: The agent receives an incomplete conversation turn because user-selected quoted context is omitted.

## Summary

Current main handles quoted context only when Feishu supplies `message.parent_id`; it does not model or normalize the separate quote identifier reported for UI quote replies, and the linked implementation attempt closed unmerged. The issue should remain open until a redacted live payload or official event contract confirms the exact field, after which the narrow plugin-local fix can reuse the existing safe fetch path.

## What This Changes

Include the content of a Feishu message selected through reply or quote UI in the agent turn alongside the new mention text.

## Best Possible Solution

Confirm the Feishu UI quote event contract, add a distinct typed quote target that falls back to `parent_id`, and route the resolved message ID through the existing visibility-checked fetch and supplemental quote-context path with regression tests for both event forms.

## Reproduction Assessment

No, not at high confidence. Source inspection shows the likely gap because only `message.parent_id` is consumed, but a redacted real Feishu UI quote event or official event-schema evidence is still needed to confirm the exact quote field.

## Solution Assessment

Yes in architecture, but the contract detail remains unverified. The narrow maintainable approach is to normalize the documented quote identifier into the existing safe fetch path rather than create parallel quote-enrichment logic.
