## Summary

Keep open. Current main fetches and injects Feishu quoted/replied content only when `parent_id` is present; it still lacks a separate quote-message-id path for UI quote replies, so the reported quote-only workflow remains unfixed despite related fetched-message parsing improvements.

## What This Changes

Request Feishu plugin support for forwarding quoted or parent message content to the agent when a user quotes a message and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-id handling: verify the incoming quote payload shape, type and parse an optional quote message id, prefer it for quoted-content fetching while preserving `parent_id` fallback, and keep the existing group context visibility filtering intact.

## Reproduction Assessment

Yes, at source level: construct a Feishu message event with a quote message id but no `parent_id`; current parsing drops that id and the fetch path is skipped because it only checks `ctx.parentId`. A live Feishu tenant payload would still be useful to confirm the precise field path.

## Solution Assessment

Yes, the narrow maintainable fix is in the bundled Feishu plugin, not core: extend the event/context model and reuse the existing `getMessageFeishu()` fetch path with a quote-id preference and parent-id fallback.
