## Summary

Keep open. Current main and the latest release still carry Feishu quoted context only through `message.parent_id`; the discussion identifies a separate Feishu UI quote target that is not modeled, parsed, fetched, or covered by tests, and the open quoted-media PR does not close that quote-target gap.

## What This Changes

Request Feishu/Lark inbound handling to include quoted or parent message content in the agent input when a user quote-replies and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the payload field, prefer it before the existing `parentId` fallback, preserve `contextVisibility` filtering, and add quote-only regression coverage.

## Reproduction Assessment

No. No live Feishu quote event was exercised; source inspection shows a quote-only event without `message.parent_id` would not fetch or populate `ReplyToBody` on current main.

## Solution Assessment

Yes, if the payload field is verified first. The maintainable solution belongs inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` quoted-message path rather than adding core behavior.
