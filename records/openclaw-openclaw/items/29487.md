## Summary

Keep open. Current main and the latest release still fetch Feishu quoted context only from `message.parent_id`; the issue discussion identifies a separate quote UI target (`quote.message_id`) that is not modeled, parsed, fetched, or covered by tests, and the open quoted-media PR is related but does not close this quote-target gap.

## What This Changes

The issue asks the Feishu plugin to include quoted or parent message content in the agent input when a user quotes a message and mentions the bot.

## Best Possible Solution

Add Feishu-plugin-local quote-target extraction after verifying the payload field, prefer it before the existing `parentId` fallback, preserve `contextVisibility` filtering and empty-message safeguards, and add quote-only regression coverage.

## Reproduction Assessment

No live Feishu reproduction was run. Source inspection gives a conditional reproduction path: a Feishu quote event with a quote-only target and no `message.parent_id` would leave `ReplyToBody` unset on current main.

## Solution Assessment

Yes, if the payload field is verified first. The narrow maintainable solution belongs inside the Feishu plugin event/context model and should reuse the existing `getMessageFeishu` quoted-message path rather than adding core behavior.
