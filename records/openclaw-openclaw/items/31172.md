## Summary

Keep open: the Feishu throttle config feature is not on current main, but this PR is not merge-ready because it changes the established 160 ms default, omits public config surfaces/tests, lacks real Feishu proof, and overlaps the broader open streaming-config PR.

## What This Changes

The PR adds a top-level Feishu `streamingThrottleMs` config field and passes it into `FeishuStreamingSession` as the card update throttle.

## Best Possible Solution

Define one Feishu-owned streaming throttle contract across schema, manifest, docs, runtime, and focused tests, preserving the current 160 ms default unless maintainers intentionally choose a different rate tradeoff.

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a feature/config PR. Source inspection confirms current main lacks the requested knob while keeping a shipped/tested 160 ms throttle default.

## Solution Assessment

No. The plumbing direction is plausible, but this branch should preserve the current default, publish the setting in manifest/docs/tests, reconcile the shape with https://github.com/openclaw/openclaw/pull/42936, and include real Feishu proof before merge.
