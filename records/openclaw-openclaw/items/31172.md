## Summary

Keep this PR open. Current main still lacks a Feishu `streamingThrottleMs` config path, so the contribution is not obsolete, but the branch is not merge-ready because its 100 ms fallback conflicts with current main's 160 ms throttle behavior and it does not update the public plugin metadata/docs/test surface.

## What This Changes

The PR adds a Feishu `streamingThrottleMs` config field, passes it into `FeishuStreamingSession`, and changes the streaming-card throttle from a fixed value to a constructor parameter with a 100 ms fallback.

## Best Possible Solution

A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and plugin manifest metadata, preserve resolved top-level/account config semantics, pass the resolved value into `FeishuStreamingSession`, document it on https://docs.openclaw.ai/channels/feishu, and add focused tests for the default and configured throttle plumbing.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/config PR. The current-main verification path is high-confidence: schema, dispatcher, manifest, docs, and tests all show that `streamingThrottleMs` is not currently implemented.

## Solution Assessment

No, this branch is not the best mergeable form yet. The narrow direction is right, but it should be rebased against current main's 160 ms behavior and complete the manifest/docs/tests config contract before merge.
