## Summary

Keep open: the Feishu throttle config idea is plausible, and the linked discussion shows real cost/control interest, but this branch should not merge as-is because it changes the current shipped 160 ms cadence to 100 ms, does not publish the option in docs/manifest/tests, overlaps the object-shaped streaming config work in https://github.com/openclaw/openclaw/pull/42936, and has no after-fix Feishu/Lark runtime proof.

## What This Changes

The branch adds a top-level Feishu `streamingThrottleMs` setting and passes it into `FeishuStreamingSession` to control Card Kit streaming-card update cadence.

## Best Possible Solution

Define one Feishu-owned streaming throttle contract across schema, manifest, docs, runtime, and focused tests, preserving the current 160 ms default unless maintainers intentionally choose a different rate tradeoff.

## Reproduction Assessment

Not applicable: this is a feature/config PR, not a broken current-main bug report. Source inspection confirms current main has no `streamingThrottleMs` knob and keeps Feishu streaming on the existing boolean `streaming` contract.

## Solution Assessment

No. The throttle-control direction may be useful, but this branch is not the best implementation because it changes the shipped/tested default, omits public config surfaces and tests, overlaps another open config-shape proposal, and lacks real Feishu proof.
