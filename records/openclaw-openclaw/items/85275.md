## What This Changes

## Best Possible Solution

Make the Talk consult result resolve from the canonical displayed source-reply or assistant text for the matched run before submitting realtime function-call output, with regression coverage for message_tool_only plus delivery-mirror.

## Solution Assessment

Yes. The narrowest maintainable fix is to unify Talk's spoken consult result with the same canonical final answer Control UI displays; provider prompt changes would leave the split source of truth intact.
