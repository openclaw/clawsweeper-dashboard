## Summary

Keep open. Current main still trusts the runtime `ContextEngine.assemble()` result shape, so a truthy result with `messages: undefined` can poison session state and later hit the reported `.length` read during prompt assembly.

## What This Changes

The issue reports that enabling `@martian-engineering/lossless-claw` can make OpenClaw crash during prompt assembly when the context-engine assembly result has `messages` undefined.

## Best Possible Solution

Add runtime validation for context-engine `assemble()` results before any runner mutates session or prompt state, and fall back to sanitized pipeline messages with a clear warning when the result is malformed.

## Reproduction Assessment

Yes for the core boundary: a focused regression can use a fake context engine returning a truthy assembly result with `messages: undefined`; the live reporter database is not needed to prove the unsafe assignment path.

## Solution Assessment

Yes. The narrowest maintainable fix is a runtime guard at the shared `assembleHarnessContextEngine()` boundary or the immediate embedded runner call site, while the external plugin can still fix its own malformed return value.
