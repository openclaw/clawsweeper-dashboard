## Summary

Keep open. Current main still trusts a truthy context-engine assemble result without checking that `messages` is an array, so a malformed plugin result can still corrupt session state and crash later during prompt diagnostics.

## What This Changes

The issue reports a prompt assembly TypeError when `lossless-claw` returns malformed assembled context with `messages` missing or undefined.

## Best Possible Solution

Add a generic runtime guard for context-engine assemble results, warn on malformed output, and continue with the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes by source inspection. A fake context engine returning a truthy assemble result with `messages: undefined` can flow into `activeSession.agent.state.messages` and later hit `activeSession.messages.length` on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a shared assemble-result validator or immediate runner guard, not lossless-claw-specific handling.
