## Summary

Keep open. Current main still forwards and assigns context-engine `assemble()` output without validating that `messages` is an array, so the reported plugin-triggered prompt assembly crash remains a valid core hardening bug.

## What This Changes

The issue asks OpenClaw to stop crashing during prompt assembly when `lossless-claw` returns malformed context-engine assembly output.

## Best Possible Solution

Add runtime validation for context-engine `assemble()` results before mutating session or prompt state, fall back to sanitized pipeline messages with a warning when invalid, and cover the malformed result with focused regression tests.

## Reproduction Assessment

Yes for the core boundary: a fake context engine returning a truthy result with `messages: undefined` exercises the current unsafe assignment path and subsequent `.length` read. The live reporter database is not needed to prove the OpenClaw-side bug.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared assemble-boundary guard or immediate embedded-runner guard before state mutation; the external plugin can still fix its own malformed return independently.
