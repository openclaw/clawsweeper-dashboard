## Summary

Keep open. Current main still accepts a truthy context-engine assemble result with non-array or missing `messages`, can assign it into the active session, and later reads `activeSession.messages.length` during prompt diagnostics.

## What This Changes

The issue asks OpenClaw to stop crashing during prompt assembly when `lossless-claw` returns malformed context-engine assemble output with `messages` undefined.

## Best Possible Solution

Add a generic runtime validator for context-engine assemble results at the shared helper or immediate runner boundary, warn on malformed output, keep the existing pipeline messages/session state, and cover it with fake-engine regression tests.

## Reproduction Assessment

Yes. A fake context engine returning a truthy assemble result with `messages: undefined` follows the current code path into `activeSession.agent.state.messages = undefined`, then reaches a later `.length` read.

## Solution Assessment

Yes. The narrow maintainable solution is a generic context-engine result guard, not a `lossless-claw`-specific branch.

## Repair Work Prompt

Fix the prompt-stage crash when a context-engine plugin returns a malformed assemble result, especially `messages: undefined`. Add a generic guard at `assembleHarnessContextEngine()` or the immediate attempt-runner call site so malformed assemble output logs a warning and falls back to the existing pipeline messages without mutating `activeSession.agent.state.messages`. Do not add a lossless-claw-specific branch and do not weaken the public `AssembleResult` type contract. Add focused fake-engine regression coverage for missing/non-array `messages`, preserve well-formed context-engine behavior including `promptAuthority` and `systemPromptAddition`, and add a single-line user-facing `CHANGELOG.md` fix entry.
