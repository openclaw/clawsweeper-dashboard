## Summary

Keep open. Current main still trusts a truthy context-engine assemble result enough to assign `assembled.messages` into session state without checking that it is an array, so the reported prompt-stage `.length` crash remains plausible and narrowly actionable.

## What This Changes

The issue requests hardening prompt assembly so lossless-claw or another third-party context engine cannot crash model calls by returning a malformed assemble result with undefined messages.

## Best Possible Solution

Add runtime validation for context-engine assemble results at the shared assemble boundary or immediate runner call site, warn on malformed output, preserve the existing pipeline messages, and cover it with a focused regression test plus a changelog fix entry.

## Reproduction Assessment

Yes. A focused test can use a fake context engine whose `assemble()` returns a truthy object with `messages: undefined`; current main would assign that value and later read `activeSession.messages.length` during prompt diagnostics.

## Solution Assessment

Yes. The narrowest maintainable fix is a generic runtime guard for the plugin return shape, without special-casing lossless-claw or changing core to depend on third-party plugin internals.
