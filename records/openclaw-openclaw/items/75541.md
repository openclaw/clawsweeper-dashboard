## Summary

Keep open. Current main still accepts a truthy context-engine assemble result without verifying that `messages` is an array, so malformed plugin output can poison runner state and reproduce the reported pre-provider `.length` crash.

## What This Changes

Harden prompt assembly so malformed `lossless-claw` context-engine assemble output cannot crash all model calls before provider submission.

## Best Possible Solution

Add a generic runtime guard at the shared context-engine assemble boundary or immediate runner call site that rejects malformed assemble results, logs a warning, and keeps sanitized pipeline messages without mutating session state.

## Reproduction Assessment

Yes. A focused regression can use a fake `ContextEngine` returning a truthy assemble result with `messages: undefined`; current main would assign it before later reading `activeSession.messages.length`.

## Solution Assessment

Yes. The narrowest maintainable fix is generic assemble-result validation with regression coverage, not a `lossless-claw`-specific core dependency or provider fallback change.
