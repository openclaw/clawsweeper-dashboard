## Summary

Keep open. Current main still validates the transcript before context-engine assembly, but then trusts the plugin's `assemble()` result and can assign a malformed `messages` value into the active session before downstream prompt-build code reads `.length`; the lossless-claw report fits a real plugin-boundary hardening gap rather than the older resolved info-id mismatch.

## What This Changes

The issue reports provider-independent prompt assembly crashes when the lossless-claw context-engine plugin is enabled and returns malformed assembled context.

## Best Possible Solution

Add a shared context-engine assemble-result guard that rejects malformed results before mutating prompt/session state, logs an actionable warning, and falls back to the pre-assembly validated messages. Keep the broader prompt-authority/overflow behavior tracked separately in #74233.

## Reproduction Assessment

Yes for the core bug path: mock a context engine whose `assemble()` returns an object with `messages: undefined` after the normal transcript sanitize/validate pass, then assert current main would reach downstream `.length` reads instead of safely falling back. The exact live lossless-claw database state is not reproducible from the issue alone.

## Solution Assessment

Yes. The narrowest maintainable fix is a core boundary guard around the shared context-engine `assemble()` result, with regression coverage in the embedded runner and Codex harness paths; redesigning lossless-claw or prompt-authority semantics is not needed for this crash.

## Repair Work Prompt

Fix the context-engine assemble boundary for openclaw/openclaw#75541. Current main validates the transcript before context-engine assembly, but `assembleHarnessContextEngine()` returns raw plugin output and the embedded runner can assign `assembled.messages` even when it is undefined, causing downstream prompt assembly to read `.length` on undefined.

Expected fix boundary:
- Add a shared validation/sanitization guard for context-engine `assemble()` results, preferably in `src/agents/harness/context-engine-lifecycle.ts` or a small nearby helper used by both embedded PI and Codex harness callers.
- Require `messages` to be an array before any caller mutates session/history state; validate obvious scalar fields such as `estimatedTokens` and `systemPromptAddition` enough to avoid crashes.
- On malformed results, throw or return a typed failure that existing caller catch blocks convert into fallback to the pre-assembly validated messages; log an actionable warning including the context engine id when available.
- Do not edit the external `@martian-engineering/lossless-claw` package, redesign prompt-authority semantics, or fold in #74233 overflow/precheck behavior.
- Add focused regression coverage for a malformed `assemble()` result with `messages: undefined`; include embedded runner coverage and, if the shared helper affects Codex, a Codex harness/projection regression too.
- Add a user-facing `CHANGELOG.md` fix entry with reporter credit if policy requires it.
