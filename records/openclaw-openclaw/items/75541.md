## Summary

Keep open. Current main still has the source-level plugin-boundary gap reported here: a truthy context-engine `assemble()` result with `messages: undefined` can be assigned into active session state, and the later prompt path reads `.length` on that value before any provider call.

## What This Changes

The issue asks OpenClaw to stop crashing during prompt assembly when the lossless-claw context engine returns a malformed assembly result with an undefined `messages` field.

## Best Possible Solution

Add a generic runtime validation boundary for context-engine assembly results before any session mutation. Malformed results should be logged as a context-engine assemble failure and the runner should keep the already sanitized pipeline messages, without adding lossless-claw-specific policy to core.

## Reproduction Assessment

Yes for the core boundary: a focused regression can mock a context engine whose `assemble()` returns a truthy object with `messages: undefined`, then verify current main would assign it before later `.length` reads. The live reporter database is not available, so the external plugin replay remains indirect.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `ContextEngine.assemble()` output at the shared helper or embedded-runner assignment site and fall back to existing pipeline messages; changing provider fallback or lossless-claw-specific routing would not address the core plugin-boundary hardening gap.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75541. Current main can assign a malformed context-engine assembly result into active session state when `assemble()` returns a truthy object with `messages: undefined`, causing later prompt assembly to read `.length` on undefined. Add a narrow runtime validation boundary for `ContextEngine.assemble()` output, preferably in `src/agents/harness/context-engine-lifecycle.ts` or immediately at the embedded-runner call site in `src/agents/pi-embedded-runner/run/attempt.ts`. Add focused regression coverage for a truthy malformed result with `messages: undefined`, and keep the fix generic rather than special-casing lossless-claw. Add a user-facing `CHANGELOG.md` fix entry with reporter credit if the code path changes runtime behavior.
