## Summary

Keep open. Current main still lacks a runtime guard for the context-engine `assemble()` result before replacing the active session messages; the adjacent main changes cover prompt-authority/precheck behavior and resolved-engine contract validation, but not malformed lifecycle output from a plugin.

## What This Changes

The issue asks OpenClaw to avoid prompt assembly crashes when `lossless-claw` returns malformed assembled context, especially an undefined `messages` field.

## Best Possible Solution

Add a small runtime validator at the context-engine assembly boundary, keep the sanitized pre-assembly pipeline messages when the result is malformed, tolerate malformed optional fields safely, and cover the embedded Pi path with a focused regression test that does not depend on the external `lossless-claw` package.

## Reproduction Assessment

Yes. A focused reproduction is to mock a context engine whose `assemble()` returns a truthy object with `messages: undefined`; current main can assign that value and later hit `.length` reads during prompt assembly.

## Solution Assessment

Yes. The narrow maintainable fix is core boundary hardening around `assemble()` output plus regression coverage; fixing only the external plugin would still leave OpenClaw vulnerable to one malformed context-engine result breaking all provider fallbacks.

## Repair Work Prompt

Fix this issue by hardening the context-engine `assemble()` result boundary. Add runtime validation so a truthy result with missing or non-array `messages` cannot replace `activeSession.agent.state.messages`; fall back to the sanitized pre-assembly pipeline messages and emit an actionable warning/diagnostic that names the context-engine assemble result as malformed. Keep the existing promptAuthority semantics from #74255 intact, and do not require or modify the external `@martian-engineering/lossless-claw` package. Prefer a shared helper in the harness lifecycle path if that keeps both embedded and harness callers consistent. Add focused regression coverage using an in-repo fake context engine whose `assemble()` returns `messages: undefined`, and verify prompt assembly does not crash. Add a user-facing CHANGELOG.md fix entry crediting @tyyim. Do not change unrelated plugin installation, provider fallback, or context-engine prompt-authority behavior.
