## Repair Work Prompt

Fix the context-engine assemble boundary for https://github.com/openclaw/openclaw/issues/75541. A malformed third-party context engine can return a truthy assemble result with `messages` missing or non-array; current runner code assigns that value into active session state and later crashes in prompt assembly. Add a generic runtime guard at `assembleHarnessContextEngine()` or the immediate `runEmbeddedAttempt` call site so malformed assemble results are warned about and ignored/fall back to existing pipeline messages. Do not special-case `lossless-claw`, add core plugin IDs, or change config semantics. Add focused fake-engine regression coverage in the context-engine/embedded-runner surface, and add a single-line `CHANGELOG.md` fix entry because this is user-facing.
