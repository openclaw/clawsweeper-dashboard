## Summary

Keep open. Current main still trusts `ContextEngine.assemble()` return values, so a truthy result with `messages: undefined` can overwrite session messages and then hit the reported `.length` read before any provider call.

## What This Changes

The issue reports a prompt assembly crash when the external lossless-claw context engine returns a malformed assembly result.

## Best Possible Solution

Add runtime validation for context-engine assemble results at the shared assemble boundary or immediate runner call site, warn on malformed results, and fall back to sanitized pipeline messages without mutating session state.

## Reproduction Assessment

Yes. A focused regression can use a fake context engine returning a truthy result with `messages: undefined`, which current main would assign before reading `activeSession.messages.length`.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared runtime guard on the plugin return shape, with regression coverage in the existing context-engine runner tests rather than depending on the external lossless-claw package.

## Repair Work Prompt

Fix the context-engine assemble trust boundary for the prompt-stage `.length` crash reported with lossless-claw. Add runtime validation so a truthy `ContextEngine.assemble()` result must be an object with `messages` as an array before the runner mutates session or prompt state. On malformed output, warn with the engine/context enough for diagnosis and fall back to the already-sanitized pipeline messages; do not assign `undefined` or other malformed values into `activeSession.agent.state.messages`. Keep the public `AssembleResult` contract intact and do not add lossless-claw-specific ids or defaults to core. Add focused regression coverage using an in-repo fake context engine returning `messages: undefined`, plus coverage for the valid assembled path. Add a user-facing `CHANGELOG.md` fix entry because this affects production plugin users. Preserve the existing promptAuthority behavior and existing context-engine tests.
