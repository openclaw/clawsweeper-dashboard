## Summary

Keep open. Current main still trusts a truthy context-engine `assemble()` result and assigns `assembled.messages` into session state without verifying it is an array, so the reported undefined `messages` crash remains a valid core hardening bug.

## What This Changes

A malformed context-engine assembly result from lossless-claw can make prompt assembly crash before any provider call by leaving session messages undefined.

## Best Possible Solution

Add runtime validation for context-engine assembly results before mutating runner/session state, warn and fall back to sanitized pipeline messages on malformed results, and cover the malformed-result path with focused tests.

## Reproduction Assessment

Yes. A focused regression can use a fake ContextEngine that returns a truthy result with `messages: undefined`, which should currently reach the unsafe assignment path without requiring the reporter's live lossless-claw database.

## Solution Assessment

Yes. The narrowest maintainable fix is a generic guard at `assembleHarnessContextEngine()` or the immediate embedded runner boundary, not a lossless-claw special case.
