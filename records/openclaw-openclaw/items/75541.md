## Summary

Keep open. Current main still trusts the runtime `ContextEngine.assemble()` result after checking only for a truthy object, so a third-party engine returning a malformed truthy result with `messages: undefined` can still replace the active session messages and later hit the reported `.length` crash. The existing ClawSweeper comment, related #63222 contract-validation work, and #74233/#74255 prompt-authority work are adjacent but do not close this runtime-result validation gap.

## What This Changes

The issue asks OpenClaw to prevent prompt assembly crashes when lossless-claw or another context-engine plugin returns a malformed assembly result with an undefined `messages` field.

## Best Possible Solution

Add runtime validation at the context-engine assembly boundary, preferably in the shared harness helper or immediately at the embedded runner call site. Malformed assemble results should be logged as a context-engine failure and the runner should continue with the already sanitized pipeline messages instead of mutating `activeSession.messages`; add regression coverage for `messages: undefined` while preserving the valid prompt-authority behavior from #74255.

## Reproduction Assessment

Yes for the core boundary. A focused regression can mock a context engine whose `assemble()` returns a truthy object with `messages: undefined`, then verify current main would assign that value and later reach a `.length` read; the reporter's live lossless-claw database is not needed for that source-level reproduction.

## Solution Assessment

Yes. A narrow guard around `ContextEngine.assemble()` output is the best maintainable fix because OpenClaw should defend the plugin boundary even if the external engine also corrects its own return shape.

## Repair Work Prompt

Fix the context-engine assembly boundary for https://github.com/openclaw/openclaw/issues/75541. Current main validates the resolved ContextEngine object but not the runtime result returned by `assemble()`, so a malformed truthy result with `messages: undefined` can replace `activeSession.agent.state.messages` and later crash at `.length` in prompt assembly. Add a narrow runtime guard, preferably in `src/agents/harness/context-engine-lifecycle.ts` or immediately in `src/agents/pi-embedded-runner/run/attempt.ts`, so malformed assemble results are treated as context-engine assemble failures with a clear warning and the runner continues with the already sanitized pipeline messages. Do not add lossless-claw as a dependency and do not change the #74255 `promptAuthority` semantics for valid results. Add focused regression coverage using an in-repo fake context engine returning `messages: undefined`; preserve existing valid assembled prompt and `preassembly_may_overflow` tests. Add a user-facing `CHANGELOG.md` fix entry thanking @tyyim.
