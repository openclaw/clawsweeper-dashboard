## Summary

Keep open. Current main still accepts a truthy context-engine assemble result without validating that `messages` is an array, so a malformed plugin result can still replace session state and crash later when prompt diagnostics read `.length`.

## What This Changes

The issue asks OpenClaw core to tolerate a malformed context-engine assemble result from lossless-claw instead of crashing before provider calls.

## Best Possible Solution

Add a generic runtime guard for context-engine assemble results, warn on malformed output, and continue with the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes by source inspection. A fake context engine returning a truthy assemble result with `messages: undefined` can flow into `activeSession.agent.state.messages` and later hit `activeSession.messages.length`.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared assemble-result validator or immediate runner guard, not a lossless-claw-specific branch.
