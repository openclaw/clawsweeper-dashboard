## Summary

Keep open. Current main still lacks a runtime guard for malformed context-engine assemble results, so a truthy result with non-array `messages` can replace session state and crash prompt diagnostics before any provider call.

## What This Changes

Fix a prompt-stage TypeError when `@martian-engineering/lossless-claw` is enabled and context-engine assembly yields an undefined `messages` field.

## Best Possible Solution

Add a generic context-engine assemble-result validator, warn on malformed output, and fall back to the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fake context engine returning a truthy assemble result with `messages: undefined` flows into `activeSession.agent.state.messages` and then reaches `activeSession.messages.length`.

## Solution Assessment

Yes. The narrowest maintainable fix is a generic guard at the shared assemble boundary or immediate runner call site, not a lossless-claw-specific branch.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/75541. Current main can accept a truthy context-engine `assemble()` result whose `messages` field is undefined or non-array, assign it into embedded session state, and then crash when prompt diagnostics read `activeSession.messages.length`. Add a generic runtime guard for context-engine assemble results, preferably in the shared harness boundary or the immediate embedded/Codex runner call sites, so malformed output logs a sanitized warning and falls back to the original pipeline messages without mutating session state. Do not add lossless-claw-specific branches, new config, dependency changes, or plugin API breaks. Add regression coverage with a fake context engine returning malformed assemble output. Update `CHANGELOG.md` because this is a user-facing bug fix. Validate with the targeted context-engine runner tests and changed gate.
