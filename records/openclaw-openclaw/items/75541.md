## Summary

Keep open. Current main still accepts the active context engine's post-assembly result without validating that `messages` is an array before replacing session state, and the downstream prompt path still reads `.length` from that state. The report is a plausible plugin-boundary hardening bug, not the older lossless-claw `info.id` mismatch that was already fixed.

## What This Changes

The issue asks OpenClaw to stop crashing during prompt assembly when the lossless-claw context engine returns malformed assembled context.

## Best Possible Solution

Harden the context-engine assemble boundary so OpenClaw validates the returned result shape before mutating session state, warns with actionable context, and falls back to the already-sanitized pre-assembly messages when a plugin returns malformed data.

## Reproduction Assessment

Unclear for the exact live report because the reporter's lossless-claw session state is not available. A high-confidence focused reproduction is clear: mock `contextEngine.assemble()` to return a result with `messages: undefined` and assert the runner falls back instead of reaching `filterHeartbeatPairs()` with invalid state.

## Solution Assessment

Yes. The best narrow fix is a core/shared boundary guard plus regression coverage; changing lossless-claw alone would not protect OpenClaw from other malformed third-party context-engine outputs.

## Repair Work Prompt

Fix the context-engine assemble-result hardening for https://github.com/openclaw/openclaw/issues/75541. Current main sanitizes and validates the transcript before context-engine assembly, then assigns `assembled.messages` directly; a malformed third-party context engine can set session messages to `undefined` and crash downstream `.length` reads before any provider call. Add a shared guard around `assemble()` results, likely in `src/agents/harness/context-engine-lifecycle.ts` or a small context-engine helper, so malformed results are rejected with an actionable warning and the runner keeps using the pre-assembly validated messages. Add focused regression coverage with a mocked context engine returning a malformed result after the normal sanitize/validate pass. Do not redesign context-engine prompt authority, do not edit the external `@martian-engineering/lossless-claw` package, and keep #74233 as the separate broader prompt-authority issue. Add a user-facing `CHANGELOG.md` fix entry crediting `Thanks @tyyim`.
