## Summary

Keep open. Current main still trusts a truthy context-engine assemble result without validating that `messages` is an array, so a malformed plugin result can still replace session history with `undefined` and crash later prompt diagnostics; the linked RFC is related but broader and does not supersede this narrow crash fix.

## What This Changes

Report a prompt-assembly crash when lossless-claw returns a malformed context-engine assemble result with an undefined `messages` field.

## Best Possible Solution

Add a generic assemble-result validator or guard, warn on malformed output, and continue with the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes, by source inspection. A fake context engine returning a truthy assemble result with `messages: undefined` can flow into `activeSession.agent.state.messages` and later hit `activeSession.messages.length` on current main.

## Solution Assessment

Yes. The narrow maintainable fix is shared assemble-result validation or an immediate runner/harness guard, not lossless-claw-specific handling or a new config option.
