## Summary

Keep open. Current main still accepts a truthy context-engine `assemble()` result without validating that `messages` is an array, so malformed plugin output can still poison active session state and crash prompt diagnostics before any provider call.

## What This Changes

The issue asks OpenClaw core to guard prompt assembly when a context-engine plugin such as lossless-claw returns malformed assembled messages.

## Best Possible Solution

Add plugin-agnostic runtime validation for context-engine assemble results, warn on malformed output, and fall back to the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fake context engine returning a truthy result with `messages: undefined` can be assigned into active session state and then fail at `activeSession.messages.length`.

## Solution Assessment

Yes. The narrowest maintainable solution is a generic assemble-result validator at the shared helper or immediate runner boundary, not a lossless-claw-specific branch.

## Repair Work Prompt

Fix issue #75541 by adding a plugin-agnostic runtime guard for context-engine `assemble()` results. If `assemble()` returns no result or returns a result whose `messages` is not an array, log a sanitized warning and continue with the original pipeline/session messages without mutating `activeSession.agent.state.messages`; preserve existing behavior for well-formed engines, `promptAuthority`, and `systemPromptAddition`. Prefer a shared helper in `src/agents/harness/context-engine-lifecycle.ts` or a narrowly scoped guard in `src/agents/pi-embedded-runner/run/attempt.ts`; also cover the Codex harness path if the shared helper is used. Add regression tests with an in-repo fake context engine returning malformed `messages`; do not add a dependency on `@martian-engineering/lossless-claw`, do not add lossless-claw-specific policy, and do not change the public context-engine API unless unavoidable. Add a single-line `CHANGELOG.md` fix entry for the user-facing crash.
