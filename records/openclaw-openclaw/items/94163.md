## What This Changes

The PR adds optional `skillKey` to skill and session snapshots, persists it during workspace snapshot building, uses it for snapshot env override lookup, refreshes old snapshots missing it, and adds focused regression tests.

## Best Possible Solution

Land one canonical fix for https://github.com/openclaw/openclaw/issues/94146, preferably this branch if maintainers accept the optional-field refresh upgrade path, after current-head CI is clean and the overlapping candidate is closed or superseded.

## Reproduction Assessment

Yes from source inspection: current main builds snapshots without `skillKey` and `applySkillEnvOverridesFromSnapshot()` resolves config by `skill.name`, so a `Pretty Weather` skill with `skillKey: weather` misses `skills.entries.weather`. I did not execute tests because this review is read-only.

## Solution Assessment

Yes: persisting the resolved key at snapshot production, using it in snapshot env lookup, mirroring the persisted type, and refreshing old missing-field snapshots is the narrow owner-boundary fix. A doctor migration would be stricter but heavier for this lightweight snapshot state, so it is a maintainer compatibility choice rather than a clear patch defect.
