## Summary

## What This Changes

The PR adds optional `skillKey` to skill/session snapshots, persists the resolved key when building workspace snapshots, uses it for snapshot env override lookup, refreshes old snapshots missing the field, and adds focused regression tests.

## Best Possible Solution

Land one canonical fix, preferably the candidate with producer, consumer, and old-snapshot refresh coverage, after the branch is refreshed against current main and required checks are green.

## Reproduction Assessment

Yes. Current main and v2026.6.9 build snapshots with only the display name and then look up snapshot env overrides by `skill.name`, while the docs require `metadata.openclaw.skillKey` when present.

## Solution Assessment

Yes. Persisting an optional resolved `skillKey`, using `skillKey ?? name`, and forcing refresh for old snapshots missing the field is the narrow owner-boundary fix; the remaining work is rebase/CI and canonical PR selection.
