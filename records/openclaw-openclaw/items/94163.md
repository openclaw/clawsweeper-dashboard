## Best Possible Solution

Land one narrow fix that persists `skillKey`, refreshes old snapshots missing it, and proves both new-snapshot and persisted-upgrade paths with targeted tests plus redacted real runtime proof.

## Solution Assessment

Yes, persisting the resolved key and refreshing old snapshots missing it is the right owner-boundary fix; the remaining issue is proof, not a different implementation layer.
