## Summary

Keep open: the PR is a useful fix for newly built skill snapshots, but it is not merge-ready because older persisted snapshots missing `skills[].skillKey` can still be reused and continue missing `skills.entries.<skillKey>` env/API-key overrides. The external contributor proof is still test-output only, not a real OpenClaw runtime proof.

## What This Changes

The branch adds optional `skillKey` fields to runtime and session skill snapshots, writes `skillKey` when building workspace snapshots, uses it for snapshot env override lookup, and adds targeted regression tests.

## Best Possible Solution

Add a narrow upgrade-path fix that treats snapshots with missing `skills[].skillKey` as stale, or backfills that metadata from a rebuild without changing prompt bytes, then add a session-snapshot test and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection shows the current-main reproduction path: a snapshot containing only `name: "Pretty Weather"` is looked up against config keyed by `weather`, so `applySkillEnvOverridesFromSnapshot()` misses the API key.

## Solution Assessment

No. The proposed change is the right producer/consumer direction for new snapshots, but the best fix also refreshes or hydrates old persisted snapshots missing `skillKey` so upgrades do not keep the broken lookup.
