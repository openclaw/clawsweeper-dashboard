## Summary

Keep this PR open, but it is not merge-ready: it fixes newly built snapshots, yet older persisted snapshots missing `skillKey` can still be reused and keep missing env overrides when `name !== skillKey`; the contributor proof is also still test-runner output only.

## What This Changes

The PR adds optional `skillKey` fields to skill snapshot/session types, writes `skillKey` when building workspace snapshots, uses it for snapshot env override lookup, and adds targeted tests.

## Best Possible Solution

Persist `skillKey` for new snapshots, refresh or backfill reused snapshots that lack it without bumping the prompt format unnecessarily, add a `session-snapshot.test.ts` regression for `name !== skillKey`, and include redacted terminal/live runtime proof of snapshot env injection.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main snapshots omit `skillKey`, snapshot env overrides look up by `skill.name`, and reusable persisted snapshots are not refreshed merely because `skills[].skillKey` is missing. I did not run tests because this review is read-only.
