## Summary

Keep open. The PR fixes newly built skill snapshots, but it is not merge-ready because existing persisted snapshots without `skills[].skillKey` can still be reused and continue missing `skills.entries.<skillKey>` overrides when `name !== skillKey`; it also still has only test-output proof, with the Real behavior proof check failing.

## What This Changes

The PR adds optional `skillKey` to skill and session snapshots, writes it during workspace snapshot building, uses it for snapshot env override lookup, and adds focused regression tests.

## Best Possible Solution

Add a narrow stale-snapshot refresh or metadata backfill for missing `skills[].skillKey`, cover it in `session-snapshot.test.ts`, keep the optional field/backward fallback, and then provide redacted real runtime or CLI proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main builds snapshots without `skillKey` and snapshot env override lookup uses `skill.name`, while config/status paths use the resolved skill key. I did not run a live OpenClaw runtime repro in this read-only review.

## Solution Assessment

No, this is the right layer for newly built snapshots but not yet the best complete fix because the reusable session snapshot path can keep old snapshots without `skillKey`. The safer fix is to refresh or backfill missing `skills[].skillKey` without changing prompt bytes.
