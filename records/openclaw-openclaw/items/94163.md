## What This Changes

## Best Possible Solution

Land one narrow fix that persists `skillKey`, refreshes old snapshots missing it, and proves both new-snapshot and persisted-upgrade paths with targeted tests plus redacted real runtime proof.

## Solution Assessment

Mostly yes. Persisting the resolved key and refreshing old snapshots missing it is the right owner-boundary fix, but the latest upgrade branch needs direct regression coverage and real runtime proof before merge.
