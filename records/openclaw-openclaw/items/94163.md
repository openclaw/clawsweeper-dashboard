## What This Changes

The branch adds optional `skillKey` to skill/session snapshots, writes it from workspace skill metadata, uses it for snapshot env/API-key lookup, refreshes old snapshots missing it, and adds focused regression tests.

## Best Possible Solution

Land one canonical narrow fix for https://github.com/openclaw/openclaw/issues/94146, preferably this broader candidate if maintainers accept the optional-field refresh upgrade path and current-head CI is green, then close the bug report.

## Reproduction Assessment

Yes. Source inspection of current main shows snapshots store only display `name`, while `applySkillEnvOverridesFromSnapshot()` looks up `skills.entries` by that display name; a skill named `Pretty Weather` with `skillKey: weather` misses `skills.entries.weather`.

## Solution Assessment

Yes. Persisting an optional resolved `skillKey` in snapshots, using `skillKey ?? name` for lookup, and refreshing old snapshots missing the field is the narrow owner-boundary fix; the remaining questions are upgrade acceptance, current-head CI, and canonical PR selection.
