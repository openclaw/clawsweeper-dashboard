## Summary

Keep open: this is a valid candidate fix for a current and shipped skills snapshot bug, but it still needs contributor-supplied real runtime proof and a direct regression test for the old persisted-snapshot refresh branch before merge.

## What This Changes

The PR adds optional `skillKey` metadata to skill and session snapshots, writes the resolved key into workspace snapshots, uses it for snapshot env override lookup, refreshes reused snapshots missing it, and adds focused tests.

## Best Possible Solution

Land one canonical fix that persists `skillKey`, uses it for snapshot env lookup, refreshes old persisted snapshots missing it, and proves both the upgrade path and real runtime env injection before merge.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.9` shows snapshots drop `skillKey` and the snapshot env override path resolves config by `skill.name`, so a skill whose display name differs from `metadata.openclaw.skillKey` misses `skills.entries.<skillKey>`.

## Solution Assessment

Mostly yes. Persisting the resolved key and refreshing old snapshots missing it is the right owner-boundary fix, but the latest upgrade branch still needs direct regression coverage and real runtime proof before merge.
