## Summary

Keep open: this targets a real Codex app-server reasoning-summary gap, but the patch currently loses unset-versus-explicit reasoning intent and still lacks PR-path real behavior proof.

## What This Changes

The PR maps session `reasoningLevel` into Codex app-server `model_reasoning_summary` config for thread start/resume and adds focused thread-lifecycle tests.

## Best Possible Solution

Carry explicit/raw reasoning intent into the Codex thread config path so only explicit `stream`/`on` writes `auto`, explicit `off` writes `none`, and unset leaves `model_reasoning_summary` absent, then prove it with focused tests and a real app-server run through `sessions.patch`.

## Reproduction Assessment

Yes, at source level: current main accepts reasoning state, forwards the resolved value into the Codex runtime, and already handles Codex reasoning summary deltas. I did not run the live app-server path in this read-only review.

## Solution Assessment

No, not as written. Mapping `stream`/`on` to `auto` is the right shape, but the implementation must distinguish explicit `off` from unset before writing a Codex config override.
