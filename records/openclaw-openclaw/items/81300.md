## Summary

Keep open: the intended Codex reasoning-summary plumbing is useful and the upstream config key is valid, but the patch currently treats OpenClaw's default resolved `off` the same as an explicit user `off`, so it can override a user's Codex config when no session preference was set. The PR body now has useful live logs for the direct Codex config workaround, but it still does not prove this branch's `sessions.patch` path end to end.

## What This Changes

The PR maps Codex session `reasoningLevel` values into the Codex app-server `model_reasoning_summary` thread config and adds thread-lifecycle tests for that mapping.

## Best Possible Solution

Carry explicit/raw reasoning intent into the Codex app-server config path so only explicit `stream`/`on` writes `auto`, explicit `off` writes `none`, and unset leaves `model_reasoning_summary` absent; then validate with focused tests and a real app-server run through `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection shows current main already forwards resolved reasoning state and projects Codex reasoning deltas, and the PR body gives a live rollout/trajectory recipe showing summaries appear when Codex receives `model_reasoning_summary = "auto"`; I did not run the live app-server path myself.

## Solution Assessment

No, not as written. Mapping `stream`/`on` to `auto` is the right shape, but the implementation should distinguish explicit `off` from the default unset path before writing a Codex config override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unset reasoning before forcing summaries off:** `extensions/codex/src/app-server/thread-lifecycle.ts:303-306`
  - body: This helper uses `params.reasoningLevel`, but normal reply resolution fills a missing directive/session/default with `"off"`, and `run.ts` forwards that resolved value into the Codex harness attempt. With this patch, ordinary sessions with no reasoning preference will send `model_reasoning_summary: "none"`, overriding a user's Codex config even though the PR says unset should be preserved. Please pass explicit/raw reasoning state into this path, or otherwise distinguish explicit `off` from the default before writing the config patch.
  - confidence: 0.9
