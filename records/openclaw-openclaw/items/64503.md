## Summary

## Best Possible Solution

Refresh the branch, remove the manual changelog edit, reuse the existing adaptive-card value serialization and command-body semantics for messageBack, keep focused object/string/empty tests, and add redacted real Teams click proof before merge.

## Reproduction Assessment

## Solution Assessment

No as currently written. Handler-level forwarding is the right owner boundary, but this patch should share the existing adaptive-card value serializer and preserve command-body semantics instead of introducing a divergent `[CARD_ACTION]` path.
