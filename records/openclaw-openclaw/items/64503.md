## Summary

Keep open. Current main still ignores `activity.value` on empty-text Microsoft Teams message activities and drops them before dispatch, so the PR is still useful; however, the branch has a strict/runtime logger bug and is missing the required user-facing changelog entry before merge.

## What This Changes

The PR changes the Microsoft Teams message handler to serialize empty-text messageBack `activity.value` payloads as `[CARD_ACTION] ...` and adds adaptive-card messageBack tests for object, string, and primitive payloads.

## Best Possible Solution

Keep this PR as the active inbound Teams messageBack fix, guard optional logging, add the required changelog entry, and land after targeted Teams validation if maintainers accept the `[CARD_ACTION]` serialization contract.

## Reproduction Assessment

Yes. ## Solution Assessment

No as submitted. The handler-level fix is the narrow right area, but the branch should use `log.warn?.(...)` and include a `CHANGELOG.md` entry before it is merge-ready.
