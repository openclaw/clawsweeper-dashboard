## Summary

This PR should stay open because it is a real candidate for the cron isolated-agent watchdog race, but it is not merge-ready: the patch can mask true startup stalls and the supplied proof is unit/mock output rather than real cron behavior proof.

## What This Changes

The PR changes `src/agents/embedded-agent-runner/run.ts` so cron runs always emit `before_agent_reply` before separately checking whether a `before_agent_reply` hook exists.

## Best Possible Solution

Fix the cron watchdog/progress boundary so the no-hook false abort is prevented while true startup stalls remain under the 60-second guard, then add focused regression coverage and redacted live cron proof.

## Reproduction Assessment

Yes, at source level. Current main gates `before_agent_reply` on hook registration, and the watchdog clears the pre-execution timer when it sees phases mapped as execution; a live slow-provider cron repro is still missing.

## Solution Assessment

No. The PR is a plausible mitigation, but the safer fix should preserve true startup-stall detection instead of clearing the pre-execution watchdog before model-resolution progress is protected.
