Keep open: current main and v2026.6.9 still have the no-hook cron phase gap, but this PR is not merge-ready because it can clear the pre-execution watchdog before real startup progress resumes and only supplies mock/unit proof.

## What This Changes

The PR changes `src/agents/embedded-agent-runner/run.ts` so cron-triggered embedded runs emit `before_agent_reply` even when no `before_agent_reply` hook is registered.

## Best Possible Solution

Keep the no-hook false-abort fix, but implement it at the cron progress/watchdog boundary or otherwise re-arm the guard until real startup progress resumes, with no-hook regression coverage and redacted live cron proof.

## Reproduction Assessment

Yes, by source inspection. Current main gates the `before_agent_reply` phase on hook registration, and the cron watchdog depends on execution-stage phase updates to clear the pre-execution timer.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix should preserve true-stall protection instead of clearing the watchdog before the next protected startup phase is reached.
