## What This Changes

The PR changes cron-triggered embedded agent runs to emit the `before_agent_reply` phase even when no `before_agent_reply` hook is registered.

## Best Possible Solution

Adopt one canonical cron runner/watchdog phase contract that prevents no-hook false aborts while preserving the short guard for true startup stalls, with focused regression coverage and redacted real cron proof.

## Reproduction Assessment

Yes, by source inspection. Current main gates `before_agent_reply` on hook registration while the cron watchdog only clears its pre-execution timer on execution-stage progress or `firstModelCallStarted`.

## Solution Assessment

No. The PR is a plausible targeted mitigation, but the best fix should preserve true startup-stall detection instead of clearing the guard before later startup progress is reached.
