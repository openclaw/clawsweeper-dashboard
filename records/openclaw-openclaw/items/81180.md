## Summary

Keep open: the PR addresses a real source-reproducible Codex attribution mismatch, but the patch currently lets normalized runtime-attribution params affect dynamic tool policy, and the external real behavior proof is still mock-only.

## What This Changes

The PR normalizes Codex app-server local runtime metadata for canonical OpenAI model turns to `openai-codex` / `openai-codex-responses` and adds a focused regression test for transcript and trajectory records.

## Best Possible Solution

Keep the narrow attribution normalization for transcript, trajectory, prompt-report, and context-engine records, but keep tool-policy-sensitive construction on the selected model provider or pass separate attribution metadata; then require real after-fix proof or maintainer override before merge.

## Reproduction Assessment

Yes for a source-level reproduction: current main passes canonical `openai` / `openai-responses` params into the projector and trajectory recorder, so Codex app-server turns with canonical OpenAI metadata are locally stamped as the public OpenAI runtime. I did not establish a live current-main reproduction in this read-only review.

## Solution Assessment

No as written: the attribution direction is narrow, but the implementation should not let normalized runtime-attribution params alter dynamic tool construction and `tools.byProvider` policy matching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep tool policy on the selected model provider:** `extensions/codex/src/app-server/run-attempt.ts:530`
  - body: This call now builds tools with `codexRuntimeParams`, where the helper rewrites `params.model.provider` from `openai` to `openai-codex`. `buildDynamicTools` forwards that value as `modelProvider` to `createOpenClawCodingTools`, and tool construction applies `tools.byProvider` from that provider, so a canonical `openai/gpt-5.5` Codex run will stop honoring existing `tools.byProvider.openai` policy. Keep the attribution rewrite to record/report surfaces, or pass separate attribution metadata that does not change tool-policy inputs.
  - confidence: 0.86
