## Summary

Keep open: the attribution bug is real and the PR direction is useful, but the current patch also routes dynamic tool construction through normalized attribution params, which can change `tools.byProvider` policy matching; external real behavior proof is still mock-only.

## What This Changes

The PR adds Codex app-server runtime-attribution normalization for canonical OpenAI model turns and a regression test covering assistant transcript and trajectory metadata.

## Best Possible Solution

Keep attribution normalization limited to transcript, trajectory, prompt-report, and diagnostics surfaces while preserving the selected model provider for tool-policy-sensitive construction, then require real app-server proof or a maintainer override before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main passes canonical `openai` / `openai-responses` params into the Codex app-server projector and trajectory recorder, so Codex-owned local records are stamped as the public OpenAI runtime. I did not run a live Codex app-server session in this read-only review.

## Solution Assessment

No as written: the attribution fix should not mutate dynamic-tool policy inputs. The safer fix is to pass separate attribution metadata or apply the normalized params only to record/report surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep tool policy on the selected provider:** `extensions/codex/src/app-server/run-attempt.ts:530`
  - body: This call now builds tools with `codexRuntimeParams`, where the helper rewrites `params.model.provider` from `openai` to `openai-codex`. `buildDynamicTools` forwards that value as `modelProvider` to `createOpenClawCodingTools`, and tool construction applies `tools.byProvider` from that provider, so a canonical `openai/gpt-5.5` Codex run will stop honoring existing `tools.byProvider.openai` policy. Keep the attribution rewrite to record/report surfaces, or pass separate attribution metadata that does not change tool-policy inputs.
  - confidence: 0.87
