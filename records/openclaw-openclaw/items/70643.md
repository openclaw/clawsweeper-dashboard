Keep this issue open. Current main still has recovery, validation, and atomic temp-file replacement, but I did not find shared serialization around the full read/merge/validate/write/runtime-refresh config transaction, and the related lock PR is closed unmerged.

## What This Changes

Serialize concurrent writes to `openclaw.json` so simultaneous config mutations cannot concatenate or clobber JSON and drop a gateway/user turn.

## Best Possible Solution

Add core config mutation serialization around the complete OpenClaw-owned transaction, including include writes and runtime refresh/notification ordering, with focused concurrent-writer regression coverage.
