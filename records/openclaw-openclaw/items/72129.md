## Summary

Keep open: the PR follows the smaller existing-plugin shape requested after the earlier Hugging Face extras draft, but its hardcoded `hf-inference` route is paired with a default and docs model list that mostly are not live on `hf-inference`, so the advertised/default path would fail at runtime.

## What This Changes

The branch adds Hugging Face text-to-image support to the existing bundled plugin, with a new image-generation provider, manifest contract metadata, docs, tests, and changelog entry.

## Best Possible Solution

Keep the existing-plugin implementation shape, but align the default, advertised models, tests, docs, and live proof with models that are actually live on `hf-inference`, unless maintainers intentionally choose to add provider-route selection first.

## Reproduction Assessment

Yes. Source inspection shows the provider always uses the `hf-inference` route, and Hugging Face model metadata shows the proposed default and most advertised models lack live `hf-inference` mappings.

## Solution Assessment

No. The owner boundary and SDK helper use are the right direction, but the route/model contract needs to be corrected before merge.
