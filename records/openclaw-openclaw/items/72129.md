## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Forward the caller SSRF policy:** `extensions/huggingface/image-generation-provider.ts:123-130`
  - body: The image runtime passes `req.ssrfPolicy` into providers, and existing HTTP image providers forward it to `postJsonRequest`/`postMultipartRequest`. This new provider drops it, so Hugging Face calls ignore operator hostname, fake-IP, and private-network policy while claiming shared transport semantics. Add `ssrfPolicy: req.ssrfPolicy` and cover it in the provider test.
  - confidence: 0.87
- **[P3] Update the central image-generation guide:** `docs/providers/huggingface.md:220-238`
  - body: Adding only a provider-specific Hugging Face section leaves `docs/tools/image-generation.md` stale: the main guide still omits Hugging Face from common routes, supported-provider and capability tables, provider deep dives, examples, and related links. Users following that page will miss the new `huggingface/black-forest-labs/FLUX.1-Krea-dev` route and its no-edit boundary.
  - confidence: 0.86
