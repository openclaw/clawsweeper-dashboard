## Summary

Keep open: current main has conventional primary/fallback failover, but no task classifier, tiered routing config, or user-defined selector hook, so this is a real feature request rather than cleanup-close material.

## What This Changes

Request automatic task-based model routing so simple requests use cheaper models and complex requests use more capable models.

## Best Possible Solution

Design an additive, opt-in routing policy for model selection, with schema/docs/protocol updates and tests that define how task routing composes with existing primary, fallback, allowlist, auth, and per-agent behavior.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and docs inspection confirms current main only exposes primary/fallback model selection, so there is no failing reproduction to establish.

## Solution Assessment

Unclear; classifier-based routing is plausible, but maintainers need to decide whether it belongs in core, a provider/router integration, or a new plugin/API seam before implementation.
