## Summary

Keep open: current main still only supports configured primary models, manual/per-agent/session routing, and error-driven fallback, so task-classified tier routing is new product/config work rather than cleanup-closeable existing behavior.

## What This Changes

Add a harness-level task classifier and tiered model configuration so simple requests use cheaper models and complex requests use stronger models automatically.

## Best Possible Solution

Define an explicit core model-routing policy/API that preserves strict user intent and existing failover semantics, then add tiered config, safe defaults, tests, and docs if maintainers approve the product direction.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source and docs confirm current main supports static/default model selection, manual/per-agent routing, and error-driven failover rather than task classification.

## Solution Assessment

Unclear; automatic tier routing may be useful, but the classifier, defaults, user hooks, and security model need maintainer product design before implementation. A safer starting point is to preserve explicit per-agent and /model routing while designing a narrow opt-in policy.
