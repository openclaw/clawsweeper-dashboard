## Summary

Keep open: current main and v2026.6.8 still prune missing transcript entries after checking one resolved path, while https://github.com/openclaw/openclaw/pull/69954 is open and unmerged. ## What This Changes

Make `openclaw sessions cleanup --fix-missing` preserve or repair sessions whose persisted `sessionFile` is stale when the canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69954 with a current-main cleanup-service fix that checks the canonical transcript path before pruning, repairs stale generated `sessionFile` metadata, preserves true-missing pruning, and adds regression coverage.

## Solution Assessment

Yes, the narrow cleanup fallback and metadata repair is the best remaining fix. The producer-side rollover fixes on current main are useful but do not protect already-stale store entries.
