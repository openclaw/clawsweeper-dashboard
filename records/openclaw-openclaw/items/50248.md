## What This Changes

The issue reports that session cleanup can trust stale `sessionFile` metadata and prune valid cron or heartbeat session entries whose canonical transcript file still exists.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69954 with a focused current-main cleanup-service fix that checks canonical transcript candidates before pruning, repairs stale generated `sessionFile` metadata, preserves true-missing pruning, and adds regression coverage.

## Reproduction Assessment

Yes. Current main still prunes after checking only the resolved persisted `sessionFile` path, and the issue provides concrete stale `sessionFile` plus live canonical transcript examples.

## Solution Assessment

Yes for the narrow direction, but it is not implemented on main yet. The best fix is a cleanup-service canonical fallback plus stale metadata repair; producer-side rollover mitigation alone does not cover already-stale rows.
