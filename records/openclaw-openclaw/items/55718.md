## Summary

Keep open. Current main has friendlier channel-key fallbacks and existing account `name` fields, but it still does not seed session `label` or `displayName` from channel account configuration, so this remains a real feature/API decision rather than cleanup.

## What This Changes

Request support for configurable channel-account display labels that make new Control UI session entries easier to recognize.

## Best Possible Solution

Add a narrow extension-safe metadata path that resolves the configured channel account display name, seeds only new or unlabeled sessions, and documents/tests the chosen config spelling.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the behavior gap: account display names exist in config, but inbound session metadata and the Control UI selector do not use them for session labels.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing the existing account `name` field is likely cleaner than adding a second display-name concept unless `displayLabel` is intentionally accepted as an alias.
