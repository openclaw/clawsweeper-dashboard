## Summary

Keep open. Current main and the latest release expose an existing channel-account `name` field, but neither inbound session metadata nor the Control UI session surfaces seed `label` or `displayName` from account configuration; the linked channel-icon work does not cover this configurable account-label request.

## What This Changes

Request to add channel-account display label configuration that automatically gives new Control UI sessions a friendly account-derived label.

## Best Possible Solution

Add an extension-safe account display metadata path that uses the chosen account label field to initialize only new or unlabeled sessions, with docs and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the behavior gap: account display names exist in configuration/status surfaces, but session metadata and Control UI session rendering do not use them for session labels.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing existing account `name` is likely the narrowest path unless `displayLabel` is intentionally accepted as a documented alias; either way, the implementation should seed only missing session labels/display names.
