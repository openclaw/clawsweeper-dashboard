## Summary

Keep open. Current main still has no path from channel account configuration to the Control UI session-list label: account `name` exists, sessions can store `label`/`displayName`, and inbound metadata records `accountId`, but the metadata writer and session row builder never resolve the configured account display name.

## What This Changes

Request to support config-driven custom display labels for channel accounts so new channel sessions appear with friendly names in the Control UI selector.

## Best Possible Solution

Implement a shared channel/session metadata bridge that uses the chosen account display field for new or unlabeled inbound sessions, preserves user-edited labels, and updates channel config docs plus focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a channel account with a friendly `name` or proposed `displayLabel`, create an inbound session for that account, and current main records the account id but never resolves the account display field into session `label` or `displayName`.

## Solution Assessment

Unclear. The requested `displayLabel` field may not be the best API because account `name` already exists; the maintainable path is to choose one config spelling, then add a narrow metadata bridge rather than duplicating UI-only fallback logic.
