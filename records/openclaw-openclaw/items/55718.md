## Summary

Keep open. Current main still has the requested feature gap: channel account display names exist as `name`, but inbound session metadata and Gateway session rows do not read channel account config, so new channel-created sessions cannot inherit a friendly Control UI label from account config.

## What This Changes

Request to add channel-account display labels so new channel-created sessions show friendly names in the Control UI session selector without editing `sessions.json`.

## Best Possible Solution

Choose the account label API, then add a generic channel-account lookup that seeds only new or unlabeled inbound sessions and document it with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure an account `name` or proposed `displayLabel`, create an inbound session for that account, and current metadata/row code has no path from account config into stored session `label` or `displayName`.

## Solution Assessment

Unclear until maintainers choose the public config spelling. Reusing existing account `name` as the default session-label source looks narrower than adding a separate `displayLabel`, unless a distinct alias is intentionally desired.
