## What This Changes

The PR changes browser config resolution and tests so a selected existing-session default profile inherits top-level `browser.cdpUrl` when it lacks a profile-local endpoint.

## Best Possible Solution

Land this focused resolver fix if maintainers agree with keeping the selected `user` existing-session profile while honoring the configured endpoint, then let the linked issue close after merge.

## Reproduction Assessment

Yes. Current main can be source-reproduced by resolving `{ defaultProfile: "user", cdpUrl: "http://127.0.0.1:9222" }` and then `resolveProfile`, which selects the built-in existing-session `user` profile with an empty profile-local endpoint.

## Solution Assessment

Yes. The PR's resolver-local inheritance of the normalized top-level endpoint into only the selected existing-session profile is narrow, preserves explicit profile-local `cdpUrl`, and matches the existing Chrome MCP endpoint argument path.
