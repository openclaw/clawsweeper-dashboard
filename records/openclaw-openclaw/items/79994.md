## What This Changes

The PR normalizes browser config so a selected existing-session default profile inherits top-level `browser.cdpUrl` when it lacks a profile-local endpoint, with focused regression tests.

## Best Possible Solution

Land this focused resolver fix if maintainers agree with preserving the selected `user` existing-session profile, then let the linked issue close after merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction on current main is `resolveBrowserConfig({ defaultProfile: "user", cdpUrl: "http://127.0.0.1:9222" })` followed by `resolveProfile`, which selects the built-in existing-session `user` profile with an empty profile-local endpoint and therefore reaches Chrome MCP auto-connect behavior.

## Solution Assessment

Yes. Applying the normalized top-level endpoint only to the selected existing-session default profile without an explicit profile endpoint is a narrow owner-local fix, and upstream Chrome DevTools MCP source confirms `--browserUrl`/`--wsEndpoint` avoid the auto-connect DevToolsActivePort path.
