## Summary

Keep this issue open: current main has partial missing-identity and shadow-publication repairs, but the live gateway publication/reload contract is still not proven complete, and a maintainer explicitly kept the stale cached SQLite handle report open for live gateway proof.

## What This Changes

The issue reports that a running gateway can keep returning `index metadata is missing` after missing metadata or external SQLite reindex publication leaves the live memory manager on stale index state.

## Best Possible Solution

Land one manager-owned publication/reload contract with focused regression coverage and live gateway proof, while preserving the missing-identity and shadow-publication fixes already on main.

## Reproduction Assessment

Yes at source level and from user reports, but not by live current-main execution in this review. The issue and comments provide Linux/macOS stale-handle evidence, and current source still routes search/status availability through the manager's current DB handle without a proven live reload contract.
