Keep this issue open: current main has partial missing-identity and swap-window fixes, but it still lacks the manager-owned live SQLite publication/reload contract requested in the maintainer closeout, and the direct refresh attempt at https://github.com/openclaw/openclaw/pull/92943 is closed unmerged.

## What This Changes

The issue reports that a running gateway can keep `memory_search` disabled with `index metadata is missing` after memory index identity loss or an external SQLite reindex, even when a fresh CLI process sees a healthy rebuilt index.

## Best Possible Solution

Keep this issue open as the canonical stale live memory-manager lifecycle bug until a manager-owned publication/reload contract lands with focused regression coverage and live gateway proof.

## Reproduction Assessment

Yes at source level, but not by live execution in this read-only review: current status/search paths refresh identity through the cached db handle, while safe reindex can publish a replacement SQLite file from another manager/process. The issue also has independent user `lsof` evidence and a maintainer comment confirming the stale cached handle report remains valid.

## Solution Assessment

No, current main is not the complete solution: it only covers FTS-only missing-identity self-heal and the swap absent-window class. The best fix is a manager-owned reload/publication contract that avoids per-observer search-path polling and proves live gateway recovery.
