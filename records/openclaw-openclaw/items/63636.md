The PR adds a guarded `estimateMessageTokens()` fallback for malformed compaction history, routes compaction/pre-prompt token estimates through it, updates compaction metrics typing, and adds malformed-history regression tests.

## Best Possible Solution

Land a rebased guarded-estimator fix that keeps current main's runtime-context exclusion and pre-prompt routing contract, covers the preserved malformed block shapes or links a deliberate follow-up, adds the required user-facing changelog entry, and then lets the linked crash report close after merge.

## Reproduction Assessment

Yes. The linked report gives a concrete pre-provider stack, current main still routes malformed history through direct upstream `estimateTokens()` calls, and the pinned upstream estimator has unchecked `.length` reads on assistant content blocks.

## Solution Assessment

No, not as the PR currently stands. A local guarded estimator is the right boundary for OpenClaw, but this branch must be rebased and repaired to preserve current main's safety and routing behavior before it is the best fix.
