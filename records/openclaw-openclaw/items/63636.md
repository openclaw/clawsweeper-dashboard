## Summary

Keep open. Current main still has the malformed-history crash path through the upstream token estimator, and this PR is the active implementation candidate for the linked crash report, but the branch is stale against current main and would need rebase fixes before it is merge-ready.

## What This Changes

The PR adds a guarded `estimateMessageTokens()` fallback in `src/agents/compaction.ts`, routes compaction and pre-prompt token estimation through it, updates compaction metrics typing, and adds malformed-history regression tests.

## Best Possible Solution

Land a rebased guarded-estimator fix that keeps current main's safety guards and preemptive routing behavior, adds coverage for the remaining malformed block shapes that OpenClaw preserves, includes the required user-facing changelog entry, then close the linked crash report once the fix reaches main.

## Reproduction Assessment

Yes. The linked report gives a concrete pre-provider stack, and current main still routes malformed history through direct upstream `estimateTokens()` calls that read missing `.length` fields.

## Solution Assessment

No, not as the branch currently stands. A local guarded estimator is the right boundary, but this PR needs a rebase that preserves current main behavior and either handles `reasoning` blocks or explicitly links a follow-up before merge.

Full review comments:

- **[P1] Preserve runtime-context stripping in token estimates:** `src/agents/compaction.ts:333-336`
  - body: Current main strips `stripRuntimeContextCustomMessages(messages)` before compaction token accounting, with a security comment saying runtime-context transcript entries must not enter LLM-facing compaction.   - confidence: 0.88

  - body: The PR head still has the older `shouldPreemptivelyCompactBeforePrompt()` contract and route math: it lacks `unwindowedMessages`, `toolResultMaxChars`, `effectiveReserveTokens`, and the minimum prompt-budget cap now present on main. Landing this branch without rebasing would drop current context-engine safeguard compaction and small-context reserve-floor behavior.
  - confidence: 0.86
- **[P2] Count reasoning blocks in the fallback estimator:** `src/agents/compaction.ts:144-145`
  - body: OpenClaw preserves and recognizes `type: "reasoning"` assistant blocks elsewhere, but the fallback only treats `thinking` and `redacted_thinking` as reasoning-bearing content. If a malformed history triggers fallback, reasoning payloads are counted as zero and can misroute pre-prompt compaction decisions. Include `reasoning` in this fallback or add a narrower follow-up before merge.
  - confidence: 0.81
