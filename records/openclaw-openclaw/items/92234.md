## Summary

The fix is still necessary: current main’s RPC wizard can request an OAuth redirect without exposing the authorization URL. The additive protocol seam is directionally sound, but this draft is not merge-ready because it has only mock/test proof, no demonstrated client consumer, a weakly constrained challenge schema, and a shared locality change that affects provider behavior beyond serialization.

## What This Changes

Adds optional OAuth challenge metadata to gateway wizard text steps and propagates authorization URLs through shared OpenAI, Chutes, and Gemini OAuth flows.

## Best Possible Solution

Define a discriminated auth-challenge union with required fields per kind, expose an explicit client-presentation capability without redefining host locality, and land or link a consuming client with a redacted real OAuth round-trip proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Separate auth presentation from remote environment detection:** `src/plugins/provider-auth-choice.ts:288`
  - body: Do not set every RPC-backed provider method’s `isRemote` flag merely because its prompter can render an auth challenge. Providers use this flag for behavior beyond presentation—for example Chutes selects `manual` mode and provider copy/browser/callback paths branch on locality—so a locally hosted gateway controlled through RPC is silently switched to remote OAuth semantics. Add a distinct capability to the OAuth presentation seam and preserve the actual environment value.
  - confidence: 0.93
- **[P2] Make auth challenge states discriminated and actionable:** `packages/gateway-protocol/src/schema/wizard.ts:64-79`
  - body: Require `kind` and the fields needed by that kind instead of making every property optional. The current schema accepts `{ auth: {} }`, `oauth-redirect` without a URL, and mismatched device-code fields, leaving protocol clients unable to know what to render and turning malformed producer output into valid public protocol data.
  - confidence: 0.9
