## Summary

This is a useful optional provider idea, but current OpenClaw policy and SDK boundaries put service-specific web search/fetch integrations in plugins and ClawHub unless a missing generic core API is identified.

## What This Changes

The issue requests adding TinyFish as a native `web_search` provider with optional `web_fetch` support and TinyFish-specific credential configuration.

## Best Possible Solution

Build TinyFish as an external native plugin published through ClawHub, using the existing web search and optional web fetch provider contracts; open a narrower SDK issue only if that plugin exposes a missing generic API seam.

## Reproduction Assessment

Not applicable; this is a request for a new provider integration, not a current-main bug with expected versus actual behavior.

## Solution Assessment

Yes; the current plugin path is the narrowest maintainable solution because OpenClaw already exposes provider contracts for vendor-specific web search and fetch transports, while VISION sets a high bar for adding optional integrations to core.
