## Summary

Keep open. Current main still does not expose fallback-attempt details from `infer model run --json`, and the richer fallback layer records data that the CLI envelope currently drops. The OpenRouter primary-selection path has improved since the reported version, but the reported diagnostics and OpenRouter preset coverage are not complete enough to close as implemented.

## What This Changes

The issue requests that `openclaw infer model run` honor a configured `openrouter/auto` primary, handle OpenRouter preset refs, and show fallback attempts when failover occurs.

## Best Possible Solution

`infer model run` should produce a truthful model-run envelope for both local and gateway transports: use the selected configured or explicit model, preserve raw model-probe behavior, and thread fallback-attempt metadata from the fallback layer into JSON output when failover occurs. OpenRouter `openrouter/auto` and `@preset/...` refs should have focused regression coverage against provider-owned normalization and, where possible, a live or upstream-doc-backed check.

## Reproduction Assessment

Yes for the current diagnostics gap: source inspection shows both local and gateway `model.run` envelopes hard-code empty `attempts` while the fallback layer records attempt details. The exact live OpenRouter preset failure remains only partially reproducible without a provider credential.

## Solution Assessment

No proposed patch is under review. The narrow maintainable fix is to propagate existing fallback diagnostics through the model-run response path and add OpenRouter auto/preset regression coverage, rather than adding OpenRouter-specific fallback policy in core.
