## Summary

Keep open. The branch is a focused attempt at the linked dashboard child-session model bug, but it removes direct child override fields without stopping the existing parent-session fallback path, so the stale parent model can still be selected when it is allowed; the contributor proof is also unit-test-only, not real dashboard/webchat rollover proof.

## What This Changes

The PR removes parent `providerOverride`, `modelOverride`, and `modelOverrideSource` inheritance from gateway child-session creation and updates session-create tests to expect those fields to be absent.

## Best Possible Solution

Fix the stored parent-override selection path, or pass an explicit no-parent-override mode for dashboard rollover, so child session model/provider state resolves as one coherent configured selection and then prove the real dashboard flow.

## Reproduction Assessment
