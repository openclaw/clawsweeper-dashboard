## Summary

Keep open: the PR has useful real Telegram proof and a plausible `/compact` ack fix, but the current branch still pre-resolves every native slash command before dispatcher admission, which can let non-status continuation commands run outside the busy/abort/progress boundary.

## What This Changes

This PR changes Telegram generic/native slash command handling so `/compact` status replies are marked deliverable and native status notices can be delivered directly instead of through the buffered dispatch pipeline.

## Best Possible Solution

Keep the `/compact` terminal status ack path, but make dispatcher admission the boundary for every native command that can continue into agent, tool, directive, or plugin work.

## Reproduction Assessment

Yes, source-reproducible with real behavior evidence: the linked Telegram screenshots show `/compact` silent before the patch and acknowledged after it, and current main still sends native command replies through the dispatcher/fence path. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No. Direct delivery for terminal `/compact` status notices is a good repair direction, but resolving all native command replies before dispatcher admission is not the narrowest maintainable fix.
