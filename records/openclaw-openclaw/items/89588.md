## What This Changes

The PR changes Telegram generic and native slash-command handling so authorized control commands get command metadata and native status-notice replies are resolved and delivered directly, with command reply metadata and regression tests.

## Best Possible Solution

Keep the repair narrow: classify generic control commands and mark terminal command replies for delivery, but resolve non-status native commands inside the buffered dispatcher and attach usable redacted Telegram proof.

## Reproduction Assessment

Source-reproducible for the patch regression: the PR head calls `getReplyFromConfig` before the dispatcher, while current dispatcher code is where resolver delivery mode and progress callbacks are injected. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No. The requested `/compact` fix is useful, but the current implementation is too broad because non-status native commands should keep their resolver inside the buffered dispatcher.
