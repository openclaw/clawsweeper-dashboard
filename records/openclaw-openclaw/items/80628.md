## Summary

Keep open: current main still lacks update-level protected-route baseline/rollback and verbose-off cross-provider fallback notices, and those changes would intentionally alter documented doctor migration and fallback-notice behavior.

## What This Changes

Request that update/doctor avoid silently changing protected Codex routes, fail closed with rollback on route drift, and show route-changing model fallback even when verbose is off.

## Best Possible Solution

Maintainers should define which route transitions are protected during update-mode doctor, then align doctor/update/fallback code, docs, and regression tests around that explicit contract.

Partly: source inspection confirms update invokes `doctor --non-interactive --fix`, doctor repair rewrites legacy Codex refs in repair mode, and fallback chat notices are still verbose-gated. I did not establish a current-main path that loses Codex OAuth after the existing auth-preservation fixes.

## Solution Assessment

Unclear: the requested fail-closed audit and route-changing fallback notice are plausible hardening, but they change current documented behavior and should follow an explicit maintainer route-policy decision.
