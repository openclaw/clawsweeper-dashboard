## What This Changes

The issue asks update-mode doctor to preserve protected Codex OAuth routes, fail closed on protected-route drift, and make cross-provider fallback visibly signaled.

## Best Possible Solution

Define the protected-route policy for update-mode doctor, then add baseline/diff/rollback and targeted fallback visibility behavior only if that policy is approved, with docs and tests updated around the canonical `openai/*` Codex routing contract.
