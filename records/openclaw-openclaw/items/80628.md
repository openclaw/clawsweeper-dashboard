## Summary

Keep open: current main still runs non-interactive doctor repair during update and still rewrites legacy `openai-codex/*` model refs to canonical `openai/*`, but there is no protected-route drift audit/rollback and fallback chat notices remain gated by verbose mode. ## What This Changes

Request that update-mode doctor preserve or fail closed on protected Codex route drift and make cross-provider fallback visible even when verbose is off.

## Best Possible Solution

Decide the protected-route policy first, then either keep canonical `openai/*` only when update can prove Codex OAuth/runtime binding or add a targeted update-mode doctor guard/audit plus matching fallback notice docs and tests.

## Reproduction Assessment

Partly/source-reproducible: current main shows update runs `doctor --non-interactive --fix`, doctor repair can rewrite `openai-codex/*` refs, and fallback chat notices are suppressed when verbose is off. I did not establish a full live current-main config that loses OAuth/billing routing, and recent merged fixes cover adjacent canonical-route auth forwarding.

## Solution Assessment

Unclear: fail-closed route-drift auditing and visible cross-provider fallback notices are plausible hardening, but they intentionally change the current documented doctor migration and fallback UX. Maintainers should choose the route policy before implementation.
