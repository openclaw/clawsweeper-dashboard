## Summary

Keep open: current main and v2026.6.10 still lack structured Control UI reply-target metadata in `chat.send`, and terminal run status remains output-agnostic. Related open PRs/issues cover useful pieces, but none fully supersedes this combined session-protocol and terminal-UX request.

## What This Changes

The issue requests structured reply-to metadata from Control UI chat sends into agent sessions and clearer handling of completed turns with no meaningful visible assistant output.

## Best Possible Solution

Define an additive typed Control UI reply-target contract, propagate it into canonical inbound runtime context, and extend terminal run state with visible-output classification while coordinating final-fallback delivery with the existing channel contract work.
