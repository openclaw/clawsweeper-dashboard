## Summary

Keep open: current main and the latest release still lack a structured Control UI reply target in `chat.send`, and Control UI terminal status remains output-agnostic. The related open PRs and issue cover useful pieces, but none is merged or broad enough to supersede this combined session-protocol and terminal-UX request.

## What This Changes

The issue asks for Control UI reply-to metadata to reach agent sessions and for terminal no-output completion states to avoid misleading success-style notices.

## Best Possible Solution

Define an additive typed reply-target contract for Control UI `chat.send`, propagate it into canonical inbound session context, and extend terminal run state with visible-output classification while coordinating delivery semantics with https://github.com/openclaw/openclaw/issues/87561.

## Reproduction Assessment

Yes. Source inspection on current main shows `chat.send` rejects unknown fields and lacks a reply target, while Control UI terminal state only records `done` or `interrupted`; I did not run a live browser scenario in this read-only review.

## Solution Assessment

Unclear until maintainers choose the protocol and UX contract. A quote-only UI workaround is too narrow for structured reply metadata, and a one-off wording tweak is too narrow for output-aware terminal semantics.
