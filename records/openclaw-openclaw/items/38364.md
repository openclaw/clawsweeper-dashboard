## Summary

Keep open. Current main still lacks a structured Control UI reply-target path from composer/send through `chat.send` into agent `MsgContext`; the no-output terminal-error path has been partly improved by a merged Gateway fix, but the remaining reply-metadata contract is not implemented or fully superseded by the related UI/threading trackers.

## What This Changes

The issue requests structured reply-to metadata from Control UI chat surfaces into agent sessions and clearer handling for no-output completion states.

## Best Possible Solution

Define an additive selected-reply metadata contract from Control UI selection through `ChatSendParamsSchema` into transcript/session semantics and `MsgContext`, while treating https://github.com/openclaw/openclaw/issues/16896 as the UI reply workflow and https://github.com/openclaw/openclaw/issues/77883 as the remaining no-output terminal-state tracker.

## Reproduction Assessment

No, not as a high-confidence end-to-end reproduction. Source inspection on current main does confirm the missing structured reply metadata path from Control UI send through Gateway `chat.send` into `MsgContext`, while the exact no-output notice was not reproduced live here.

## Solution Assessment

Unclear as a direct one-off patch because the durable fix crosses UI selection state, Gateway protocol, transcript/session semantics, and agent context. The maintainable direction is an additive contract coordinated with the existing UI reply and no-output trackers.
