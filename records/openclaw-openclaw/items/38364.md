## Summary

Keep open: current `main` and latest release still lack a structured `chat.send` reply target, and Control UI terminal status remains a generic `done`/`interrupted` presentation without visible-output semantics. Related quote/reply UI issues are narrower and do not supersede this session-protocol plus completion-state request.

## What This Changes

Expose structured Control UI reply-to metadata to agent sessions and make no-output terminal completion feedback less misleading.

## Best Possible Solution

Add an additive typed reply-target field to the `chat.send` contract, propagate it into canonical inbound session context, and extend terminal run state with visible-output classification so Control UI can present precise completion feedback.

## Reproduction Assessment

Yes, source inspection provides a high-confidence path: current `chat.send` rejects unknown fields and exposes no reply target, while Control UI terminal status only records `done` or `interrupted`. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Yes in direction, but the maintainable solution needs a typed end-to-end reply contract and normalized output-aware terminal state rather than a quoted-text-only workaround or one-off wording change.
