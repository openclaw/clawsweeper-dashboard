## Summary

Keep open. Current main still has no structured Control UI reply-target path from composer/send through `chat.send` into agent `MsgContext`, and the no-output terminal UX is only partly covered by existing silent-token handling plus a narrower open recovery tracker.

## What This Changes

The issue asks for Control UI reply-target metadata to reach agent sessions and for terminal no-output states to avoid misleading completion messaging.

## Best Possible Solution

Define an additive selected-reply metadata contract from Control UI selection through `ChatSendParamsSchema`, transcript/session semantics, and `MsgContext`, then coordinate no-output terminal-state UX with https://github.com/openclaw/openclaw/issues/77883.

## Reproduction Assessment

No as a strict end-to-end reproduction. Source inspection gives high confidence that the Control UI `chat.send` path has no reply metadata handoff, while the exact no-output completion notice was not reproduced live.

## Solution Assessment

Unclear as an immediate patch because the right fix crosses UI selection state, Gateway protocol, transcript/session semantics, and agent context. The maintainable direction is an additive contract coordinated with the related reply/threading and no-output trackers.
