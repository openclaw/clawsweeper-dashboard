## Summary

Keep open. Current main still lacks a structured Control UI reply-target path from composer/send through `chat.send` into agent `MsgContext`; silent/no-output display handling is partly covered, but the remaining reply metadata and terminal-state UX need maintainer API and product scoping.

## What This Changes

Request to pass structured Control UI reply-to metadata into agent sessions and replace misleading no-output completion notices with clearer terminal states.

## Best Possible Solution

Define an additive selected-reply metadata contract from Control UI selection through `ChatSendParamsSchema`, transcript/session semantics, and `MsgContext`, then coordinate no-output terminal-state UX with https://github.com/openclaw/openclaw/issues/77883 while preserving existing silent-token suppression.

## Reproduction Assessment

No as a strict bug reproduction; this is mainly an additive metadata and UX feature. Source inspection gives high confidence that current main has no Control UI `chat.send` reply metadata path, while a live repro for the exact no-output notice is still missing.

## Solution Assessment

Unclear as an immediate patch because the change spans UI selection state, Gateway protocol, transcript/session semantics, and agent context. The maintainable direction is an additive contract coordinated with https://github.com/openclaw/openclaw/issues/16896, https://github.com/openclaw/openclaw/issues/7817, https://github.com/openclaw/openclaw/pull/62262, and https://github.com/openclaw/openclaw/issues/77883.
