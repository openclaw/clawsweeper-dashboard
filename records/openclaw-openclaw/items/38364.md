## Summary

Keep open. Current main still lacks the structured reply-target plumbing from Control UI and Gateway send methods into agent-visible context; the silent/no-output display path has defenses, but that only partly addresses the issue.

## What This Changes

The issue requests structured reply-to metadata for Control UI chat sends and clearer handling of no-output completion notices.

## Best Possible Solution

Add a backward-compatible reply-target contract to chat.send and sessions.send, pass sanitized Control UI reply metadata into existing MsgContext reply fields, and preserve the current silent-output filtering behavior.

## Reproduction Assessment

Yes for the source-level capability gap: current schemas, UI send code, sessions forwarding, and chat.send MsgContext construction show no reply-target plumbing. I did not run a live browser flow because this is a protocol/UI capability gap rather than a single failing runtime path.

## Solution Assessment

Yes at the direction level: additive protocol/UI plumbing into existing reply-aware context fields is the narrow maintainable path. The exact wire payload and sanitization rules should be settled by maintainers before implementation.
