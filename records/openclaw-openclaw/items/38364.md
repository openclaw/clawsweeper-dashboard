## Summary

Keep open. Current main still lacks a structured Control UI reply-target field in `chat.send`, the UI send payload, and the Gateway `MsgContext` handoff; the no-output UX portion is only partially covered by a narrower open restart-recovery tracker.

## What This Changes

The issue requests Control UI/Gateway/session support for reply-to metadata plus clearer UX when a run completes without meaningful assistant text.

## Best Possible Solution

Define an additive selected-reply metadata contract from Control UI selection through `ChatSendParamsSchema`, transcript/session semantics, and `MsgContext`, then coordinate no-output terminal-state UX with https://github.com/openclaw/openclaw/issues/77883.

## Reproduction Assessment

No, not as a high-confidence live end-to-end reproduction. Source inspection supports the missing reply metadata path on current main, but the exact no-output completion notice was not reproduced here.

## Solution Assessment

Unclear as a direct patch because the durable fix crosses UI selection state, Gateway protocol, transcript/session semantics, and agent context. The maintainable direction is an additive contract coordinated with the narrower no-output tracker.
