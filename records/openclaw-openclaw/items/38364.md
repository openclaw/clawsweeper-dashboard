## Summary

Keep open. Current main still lacks a structured reply-target path from Control UI or Gateway chat sends into agent-visible context; the silent/no-output display side is partly covered, but the central reply metadata request remains unresolved.

## What This Changes

Request to add structured reply-to metadata through Control UI/Gateway chat sends and avoid misleading no-output completion messaging.

## Best Possible Solution

Add a backward-compatible reply-target field to chat.send and sessions.send, pass sanitized Control UI parent id/body/sender metadata into existing MsgContext reply fields, and preserve current silent-output filtering behavior.

## Reproduction Assessment

Yes for source-level reproduction. Current main has closed chat.send and sessions.send schemas without reply metadata, Control UI does not send a reply target, and chat.send builds MsgContext without existing reply fields.

## Solution Assessment

Yes. Additive protocol/UI plumbing into existing MsgContext reply fields is the narrow maintainable path; encoding reply intent only in message text would not give agents reliable parent-message metadata.
