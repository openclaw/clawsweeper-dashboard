Keep open. Current main still lacks a structured Control UI reply-to contract from `chat.send` into agent session context; the no-output/`NO_REPLY` confusion is partly addressed, but related UI reply and channel-threading work does not supersede the remaining metadata scope.

The issue asks for structured Control UI reply-to metadata to reach agent sessions and for no-output completion states to be less misleading.

## Best Possible Solution

Define an additive Control UI reply metadata contract from selected UI reply target through `chat.send` into `MsgContext` and transcript semantics, while preserving existing silent-output suppression and coordinating with the separate UI reply affordance and channel threading work.

## Reproduction Assessment

## Solution Assessment

Unclear as an immediate patch because the change spans UI, Gateway protocol, and agent-session context. The maintainable direction is an additive contract that does not regress current `NO_REPLY` and heartbeat suppression behavior.
