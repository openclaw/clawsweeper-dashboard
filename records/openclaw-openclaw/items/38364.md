## Summary

Keep open. Current main still lacks structured reply-target plumbing from Control UI or Gateway send methods into agent-visible context; the no-output/silent-token display path is partly covered, but the central reply metadata request remains unresolved.

## What This Changes

Expose structured reply-to metadata from Control UI/chat send paths to agent sessions and make no-output completions less misleading.

## Best Possible Solution

Add a backward-compatible reply-target contract to chat.send and sessions.send, pass sanitized Control UI reply metadata into existing MsgContext ReplyTo fields, and preserve the current silent-output filtering behavior.

## Reproduction Assessment

Yes for the source-level gap: current main has closed chat.send and sessions.send schemas without reply metadata, Control UI does not send a reply target, and chat.send does not populate MsgContext ReplyTo fields. A live browser repro is not necessary to prove this protocol/UI capability gap.

## Solution Assessment

Yes at the direction level: additive protocol/UI plumbing into existing MsgContext ReplyTo fields is the narrow maintainable path. The exact public payload shape still needs maintainer product/API agreement before a repair PR is safe.
