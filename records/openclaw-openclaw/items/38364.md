## Summary

Keep open. Current main has addressed much of the no-output confusion by suppressing `NO_REPLY`/heartbeat payloads and using compact run statuses, but the structured reply metadata path is still absent from the Control UI `chat.send` protocol and sender code; the related right-click reply issue and open reply-threading PR do not close this broader metadata request.

## What This Changes

Request to pass structured reply-to metadata from Control UI/chat surfaces into agent sessions and make no-output completion states less misleading.

## Best Possible Solution

Keep this open as the structured Control UI reply-metadata/threading design tracker, preserve the current silent-output suppression behavior, and coordinate any implementation with https://github.com/openclaw/openclaw/issues/16896 and https://github.com/openclaw/openclaw/pull/62262.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is primarily a new metadata/threading capability request. Source inspection gives high confidence that the Control UI `chat.send` path currently lacks structured reply-to metadata.

## Solution Assessment

Unclear as an immediate implementation because the remaining work crosses UI affordance, Gateway protocol, and session/transcript semantics. The maintainable direction is an additive contract that does not regress the current `NO_REPLY` and heartbeat suppression behavior.
