## Summary

Keep open: current main and v2026.6.6 still do not expose a structured Control UI reply target through `chat.send`, and the Control UI terminal status remains a generic done/interrupted presentation without visible-output semantics. Related quote/reply UI items are narrower and do not supersede this session-protocol plus completion-state request.

## What This Changes

Request to add structured reply-target metadata from Control UI chat sends into agent sessions and to make terminal completion UI distinguish no visible assistant output.

## Best Possible Solution

Add an additive typed reply-target field to the `chat.send` contract, propagate it into canonical inbound session context, and extend terminal run state with visible-output classification so Control UI can present precise completion feedback.

## Reproduction Assessment

Yes. Source inspection shows current main has no `chat.send` reply target and represents terminal success as generic `done` without tracking whether visible assistant output existed.

## Solution Assessment

Yes in direction, but the maintainable solution needs a typed end-to-end reply contract and normalized output-aware terminal state rather than a quoted-text-only workaround or one-off localized wording change.
