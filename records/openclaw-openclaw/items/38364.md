## Summary

Keep open. Current main still lacks structured reply-target plumbing from Control UI or Gateway sends into agent-visible context; the silent-output display side is partly covered, but the central reply metadata request remains unresolved.

## What This Changes

Expose structured reply-to metadata from Control UI/Gateway chat sends to agent sessions and avoid misleading no-output completion notices.

## Best Possible Solution

Define an additive reply-target contract for chat.send and sessions.send, plumb sanitized Control UI reply metadata into existing MsgContext reply fields, and preserve the current silent-output filtering behavior.

## Reproduction Assessment

Yes for the source-level capability gap: closed chat.send and sessions.send schemas omit reply metadata, Control UI sends no reply target, and chat.send builds MsgContext without ReplyTo fields. I did not run a live browser repro because the central issue is a protocol/UI capability gap, and the exact no-output notice appears partly historical on current main.

## Solution Assessment

Yes at the direction level: additive protocol/UI plumbing into existing MsgContext reply fields is the narrow maintainable path. Maintainers should settle the payload shape and coordinate with the related Dashboard reply UI issue before implementation.
