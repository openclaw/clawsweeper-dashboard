## Summary

Keep open. Current main still lacks a structured Control UI reply-target path from composer/send through `chat.send` into agent `MsgContext`; the no-output part has some silent-token suppression and a narrower open tracker at https://github.com/openclaw/openclaw/issues/77883, but the combined reply metadata and terminal-state contract still needs maintainer API/product scoping.

## What This Changes

The issue asks for Control UI reply-to metadata to reach agent sessions and for no-output completions to render as clearer terminal states.

## Best Possible Solution

Define an additive selected-reply metadata contract from Control UI selection through `ChatSendParamsSchema`, transcript/session semantics, and `MsgContext`, then coordinate no-output terminal-state UX with https://github.com/openclaw/openclaw/issues/77883.

## Solution Assessment

Unclear as an immediate patch because the right fix crosses UI selection state, Gateway protocol, transcript/session semantics, and agent context. The maintainable direction is an additive contract coordinated with the related open reply/threading and no-output trackers.
