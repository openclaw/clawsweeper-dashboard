## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop overwriting custom CN-only TLS certificates:** `src/infra/tls/gateway.ts:119`
  - body: This predicate treats any `CN=openclaw-gateway` certificate without SANs as OpenClaw-generated, but `loadGatewayTlsRuntime` reaches it even for explicit `certPath`/`keyPath` deployments when `autoGenerate` is omitted. A user-provided CN-only cert can therefore be deleted and replaced; the repair needs a stronger ownership signal, default-path guard, explicit doctor flow, or equivalent protection plus a regression test.
  - confidence: 0.92
- **[P1] Preserve the old cert/key until regeneration succeeds:** `src/infra/tls/gateway.ts:140-146`
  - body: The upgrade path unlinks the current cert and key before `generateSelfSignedCert` has produced a valid replacement. If OpenSSL or the filesystem fails, the catch returns `false` after the files are gone and the outer function can still start with stale in-memory material, leaving the next startup broken; write to temporary files and swap/rollback only after success.
  - confidence: 0.9
