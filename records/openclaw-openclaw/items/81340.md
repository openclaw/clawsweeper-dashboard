## Summary

Keep open: the linked channel-listing regression is source-reproducible, but this PR is not mergeable because the guard-only change still lets the downstream manifest builder drop the channel, and the branch also carries unrelated session/config and PR-template changes.

## What This Changes

The branch changes read-only channel manifest gating and the missing-channelConfigs warning, adds `session.maintenance.preserveKeys` config plumbing, and adds two GitHub PR templates.

## Best Possible Solution

Land a focused compatibility patch that builds a limited read-only manifest plugin for configured channels declared only in `channels[]`, keeps the warning accurate, preserves session-provider behavior, drops unrelated changes, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a configured external channel declared only in `channels[]` reaches the manifest listing path, but the builder returns `undefined` without `channelCatalogMeta` or `channelConfigs`, and the listing helper skips that falsy plugin.

## Solution Assessment

No. The PR changes the guard but not the object builder that actually creates list entries, so the narrow maintainable fix should add a manifest-only fallback plugin and keep the unrelated session/template work separate.
