Keep this PR open, but it should not merge as written: the linked regression is valid from source inspection, yet the predicate-only fix still lets the builder drop manifest-only channels, and the branch contains unrelated/conflicted changes plus insufficient real behavior proof.

## What This Changes

The branch changes manifest-only channel listing diagnostics, adds PR templates, adds `session.maintenance.preserveKeys` config plumbing, and edits the Windows installer.

## Best Possible Solution

Land a focused channel-only compatibility fix that builds a safe limited-metadata plugin for manifest-only channels, keeps the warning actionable, removes unrelated branch changes, and adds regression coverage.

## Reproduction Assessment

Yes, from source inspection: a configured external channel declared only in `manifest.channels[]` passes through the listing path without metadata and cannot be built into a read-only plugin on current main. I did not run the CLI or gateway because this review was read-only.

## Solution Assessment

No, this is not the best fix as written because returning `true` from the predicate does not change the builder that still returns `undefined`, and the branch mixes unrelated/conflicted changes. The safer solution is a narrow builder-level fallback plus targeted tests.
