## Summary

This PR should stay open because the central channel-listing fix is incomplete and the branch also changes unrelated session-maintenance and PR-template surfaces. Source inspection confirms the reported bug path is real, but the proposed predicate-only change still lets `buildManifestChannelPlugin` return `undefined` for manifest-only channels.

## What This Changes

This PR relaxes manifest channel listing gating and diagnostics for channels without `channelConfigs`, while also adding PR templates and `session.maintenance.preserveKeys` plumbing.

## Best Possible Solution

Land a focused channel compatibility patch that builds a safe limited read-only plugin for configured manifest-only channels, keeps the warning accurate, preserves existing session-maintenance provider behavior, removes unrelated template/config changes, and adds targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a configured external channel declared only in `manifest.channels[]` reaches the manifest listing path, but the builder returns `undefined` without `channelCatalogMeta` or `channelConfigs`.

## Solution Assessment

No. Changing only `canUseManifestChannelPlugin` is not the best fix because the downstream builder still rejects the reported manifest shape; the maintainable fix belongs in the builder/fallback path with regression coverage.
