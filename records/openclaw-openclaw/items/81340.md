## Summary

Keep this PR open, but it is not mergeable as-is: the main channel-listing bug is source-reproducible, the proposed predicate-only fix still does not create a listing plugin, and the branch still contains unrelated changes plus installer conflict markers.

## What This Changes

The branch attempts to show external channel plugins declared only in `manifest.channels[]` by relaxing read-only manifest listing logic and warning text, while also adding PR templates, session `preserveKeys` config plumbing, and a conflicted `install.ps1` edit.

## Best Possible Solution

Land a focused channel compatibility patch that builds a safe limited read-only plugin for configured manifest-only channels, keeps diagnostics actionable, removes unrelated branch changes and conflict markers, and adds targeted regression coverage.

## Reproduction Assessment

Yes, from source inspection: a configured external channel declared only in `manifest.channels[]` can reach the manifest listing path, but the builder returns `undefined` without `channelCatalogMeta` or `channelConfigs`. I did not run the CLI or gateway in this read-only review.

## Solution Assessment

No, this is not the best fix as written because changing only `canUseManifestChannelPlugin` does not change the builder that still drops the channel. The narrower maintainable fix is a builder-level fallback with regression coverage and a scoped branch.
