## What This Changes

The branch attempts to show manifest-declared third-party channel plugins without `channelConfigs` in channel listings and expands the missing-`channelConfigs` warning, while also carrying unrelated installer, session config, and PR template changes.

## Best Possible Solution

Land a scoped fix that creates a safe fallback read-only `ChannelPlugin` for manifest-declared channels without `channelConfigs`, preserves the metadata warning, adds focused channel-listing regression coverage, and removes unrelated branch changes.

## Reproduction Assessment

Yes, source-level: a manifest record with `channels[]` but no `channelConfigs` or `channelCatalogMeta` is blocked on current main, and this PR still reaches a builder that returns `undefined`. I did not run a live gateway/plugin reproduction.

## Solution Assessment

No, this PR is not the best current solution because it relaxes the eligibility check without adding the fallback metadata object that the listing path needs. A narrower patch should fix the builder path and include a regression test for the exact manifest shape.
