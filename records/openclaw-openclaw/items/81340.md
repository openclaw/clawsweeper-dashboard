## Summary

Keep open: the reported regression is source-reproducible, but this PR does not actually restore the manifest-only channel listing path and it also carries unrelated session/config/template changes.

## What This Changes

The PR relaxes read-only channel manifest gating and warning text for channels without channelConfigs, adds session.maintenance.preserveKeys plumbing, and adds two PR templates.

## Best Possible Solution

Land a focused channel compatibility patch that builds a limited read-only plugin for configured manifest-only channels, keeps the warning accurate, removes unrelated changes, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured external channel declared only in manifest channels[] reaches the manifest listing path, but the current builder returns undefined without channelCatalogMeta or channelConfigs.

## Solution Assessment

No. The guard change is not sufficient because the downstream builder still rejects the reported manifest shape; the safer fix belongs in the builder/fallback path with focused tests.

## Review Findings
