## Summary

Keep open: this is a focused fix candidate for the linked prompt-cache bug, but the patch is not merge-ready because it removes already-truncated history from aggregate accounting and can leave prompt history over budget.

## What This Changes

The PR adds truncation-marker detection so aggregate tool-result reduction skips messages that were already truncated, plus two unit tests for cross-turn stability.

## Best Possible Solution

Keep the cache-stability goal, but count stable historical tool results against the aggregate budget while avoiding byte changes to them, then add runtime proof for both cache stability and overflow behavior.

## Reproduction Assessment

Yes, at source level: current main rewrites aggregate tool-result projections from an evolving aggregate total, and the PR’s new filter creates a clear over-budget scenario when many historical entries already contain truncation markers.

## Solution Assessment

No. Skipping replacements for stable historical entries is plausible, but removing them from aggregate accounting is not the best fix because it trades cache stability for unbounded prompt pressure.
