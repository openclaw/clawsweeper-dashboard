## What This Changes

The PR changes plugin skill publishing to lstat existing managed entries before readlink, replace generated Windows directory entries that collide with active plugin skills, and add focused regression coverage.

## Best Possible Solution

Land this narrow lstat-before-readlink fix after normal CI, then let the linked issue close through the PR merge path.

## Reproduction Assessment

Yes. Current main reaches fs.readlinkSync before checking whether the managed entry is a symlink, and readlinkSync on a regular directory returns EINVAL; the linked issue gives the Windows user flow and the PR adds a focused reproduction.

## Solution Assessment

Yes. Inspecting with lstat before readlink and reusing the generated-entry cleanup path is the narrow maintainable fix, while non-generated entries remain untouched.
