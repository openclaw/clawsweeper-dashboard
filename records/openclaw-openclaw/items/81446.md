## Summary

Keep open for normal PR review; I found no blocking correctness or security findings. The patch is a narrow source-backed fix for the Windows regular-directory `readlinkSync` failure path, with supplied terminal proof and focused regression coverage.

## What This Changes

The PR changes `publishPluginSkills` to inspect managed entries with `lstat` before `readlink`, replace generated Windows directory entries, and add a regression test for replacing a stale directory with the current skill link.

## Best Possible Solution

Land the narrow fix after normal maintainer and CI gates so plugin-skill publication can replace owned Windows directory entries without masking unrelated filesystem entries.

## Reproduction Assessment

Yes from source inspection, though I did not execute the full CLI flow locally: current main reads the managed path as a symlink before checking its type, and a real directory produces `EINVAL`. The linked report supplies Windows 10/OpenClaw v2026.5.7 steps, and the PR adds a focused simulated-Windows regression test.

## Solution Assessment

Yes. `lstat` before `readlink` is the narrow maintainable fix, and reusing the existing generated-entry cleanup contract keeps the behavior scoped to owned plugin-skill entries without adding config or product policy.
