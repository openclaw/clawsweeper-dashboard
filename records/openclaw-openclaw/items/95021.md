## Summary

Keep open: this is a plausible bug area, but the PR broadens heartbeat target inheritance at the shared resolver boundary and can route a previous channel's target through a different selected channel; it also lacks after-fix QQ C2C heartbeat proof.

## What This Changes

The PR changes `resolveSessionDeliveryTarget` so heartbeat mode inherits `lastTo` when the selected channel differs, and adds outbound target unit cases.

## Best Possible Solution

Preserve cross-channel target inheritance behind same-channel or explicit turn-source/allow-mismatch contracts, then localize the QQ heartbeat bug with a targeted test and real C2C heartbeat proof.

## Reproduction Assessment

No for the reported QQ C2C failure: the PR body only cites unit tests and provides no live QQ heartbeat run, logs, or screenshot. The PR-introduced cross-channel regression is source-reproducible from `resolveSessionDeliveryTarget` inputs where `mode: "heartbeat"`, selected channel differs, and `lastTo` exists.

## Solution Assessment

No. The proposed fix is too broad because it changes the shared mismatch guard instead of proving why QQ heartbeat delivery needs cross-channel inheritance; a safer fix preserves the guard and localizes routing through `target: "last"`, turn-source context, or QQ-specific delivery evidence.
