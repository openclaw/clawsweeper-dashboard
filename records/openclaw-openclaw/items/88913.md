## Summary

Keep this PR open for maintainer review: the patch looks mechanically coherent and the supplied terminal proof covers the installed-plugin metadata path, but it intentionally broadens the auth-marker trust boundary from bundled manifests to enabled external plugin manifests.

## What This Changes

The PR lets plugin manifest metadata contribute `nonSecretAuthMarkers` beyond bundled plugins, filters disabled installed plugin roots, registers lifecycle-cleared metadata caches, and updates manifest docs/types/tests.

## Best Possible Solution

Land this only after maintainers explicitly accept the external-plugin auth-marker trust boundary; otherwise keep the bundled-only contract or add a narrower explicit trust mechanism.

## Reproduction Assessment

Yes. Current main source filters manifest-owned auth markers to `plugin.origin === "bundled"`, so an installed external/global manifest marker is not accepted without this PR.

## Solution Assessment

Conditionally yes. The implementation follows the manifest-first, lifecycle-cleared metadata pattern, but the best fix depends on maintainer approval of the new external-plugin trust boundary.
