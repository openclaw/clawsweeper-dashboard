## Summary

## What This Changes

The PR lets enabled installed external plugin manifests contribute `nonSecretAuthMarkers`, filters disabled installed plugin roots from fallback scanning, lifecycle-clears marker metadata caches, and updates manifest docs/types.

## Best Possible Solution

Merge only after plugin/auth-security owners explicitly approve enabled external plugin manifests as a trusted source for non-secret auth markers and accept the lifecycle-cleared metadata freshness contract.

## Solution Assessment
