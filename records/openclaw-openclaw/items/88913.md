## Best Possible Solution

Gate external `nonSecretAuthMarkers` to enabled installed plugin owners, preserve bundled behavior, add disabled/unindexed manifest coverage, and include redacted real installed-plugin proof.

## Reproduction Assessment

Yes from source: current main filters manifest markers to bundled plugins, so an `npm` or `global` manifest marker is dropped. I did not run a live installed-plugin repro in this read-only review.

## Solution Assessment

No: removing the bundled guard is a plausible fix, but it is too broad because the scanner includes disabled persisted records and global plugin directories. The better fix is to preserve the external capability while gating marker ownership to active installed plugins.
