## Summary

Keep open. Current main has ja-JP support for the browser Control UI and a separate docs translation pipeline, but the requested core UX localization is still missing for CLI help, onboarding/wizard prompts, workspace templates, and shared runtime labels.

## What This Changes

The issue requests key-based Japanese localization for core OpenClaw UX with English fallback, starting with onboarding, help, templates, and common labels.

## Best Possible Solution

Define a core i18n contract for non-browser OpenClaw surfaces: locale resolution for CLI/wizard contexts, English fallback semantics, ja-JP resource ownership, and focused tests for CLI help, onboarding prompts, fallback behavior, and workspace template generation, while keeping the existing Control UI and docs pipelines aligned but not accidentally conflated.
