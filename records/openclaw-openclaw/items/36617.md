## Summary

Keep open. Current main has ja-JP support for the browser Control UI and a separate docs translation pipeline, but the requested core UX localization for CLI onboarding, help, workspace templates, and shared labels is still not implemented.

## What This Changes

Add key-based i18n and a ja-JP locale with English fallback for core OpenClaw onboarding, help, templates, and shared labels.

## Best Possible Solution

Define a maintainer-approved core i18n contract for non-browser UX, then add ja-JP resources and focused coverage for the agreed CLI onboarding, help rendering, workspace template generation, and common-label surfaces.

## Reproduction Assessment

Not applicable as a runtime reproduction. This is a feature coverage request, and current-main source inspection gives a high-confidence way to verify the gap: only UI/docs localization exists while core CLI onboarding, help, and template text still use English strings directly.

## Solution Assessment

Unclear for implementation details. Key-based i18n with English fallback is a plausible direction, but the narrowest maintainable fix needs maintainer agreement on the core locale contract and localized surface list before moving strings into resources.
