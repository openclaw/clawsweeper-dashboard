## Summary

Current main and the latest release already cover the requested Japanese core UX localization path for the Control UI: ja-JP is in the runtime locale registry, has a generated Japanese bundle with no fallback keys, is selectable from the gateway access UI, and is documented with English fallback behavior.

## What This Changes

The issue requests key-based i18n and a ja-JP locale for core OpenClaw onboarding, help, templates, and common labels.

## Best Possible Solution

Keep the shipped generated Control UI i18n pipeline and handle any future Japanese gaps as targeted missing-string or glossary follow-ups.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the requested ja-JP locale path is now present rather than a bug needing reproduction.

## Solution Assessment

Yes. The current implementation uses a central English source map, generated locale bundles, lazy loading, browser/persisted locale selection, and English fallback, which is the maintainable shape requested here.
