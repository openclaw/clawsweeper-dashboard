## Summary

Keep open: current main has dedicated image/video generation provider surfaces, but generic configured chat models still only carry input metadata and the text-session fallback path has no output-modality contract to reject Poe media-output models clearly.

## What This Changes

Request clearer handling for Poe/custom-provider image and video output models that can be configured as chat models but fail or fall back at runtime.

## Best Possible Solution

Add a versioned output-capability contract for configured/provider catalog models and fail closed with a clear unsupported-media-output message for chat/session selections, while keeping real image/video generation on image_generate/video_generate provider plugins.

## Reproduction Assessment

No high-confidence live Poe reproduction was established. Source inspection on current main confirms the relevant metadata and fallback path still lack an output-modality check, so the core UX is source-reproducible but not live-reproduced.

## Solution Assessment

Unclear as an implementation direction. Full Poe media generation in chat sessions is not the narrowest fix; the safer maintainer path is to define output-capability metadata and fail closed, or route Poe through a dedicated media generation provider/plugin.
