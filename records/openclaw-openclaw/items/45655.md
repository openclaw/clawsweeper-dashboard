## Summary

Keep open. Current main supports image/video generation through dedicated tool/provider paths, but custom `models.providers` chat entries still have no output-modality contract or clear rejection path for media-output Poe models.

## What This Changes

Request to clarify or fix how custom Poe media-output models configured through `models.providers` are handled so image/video generation does not fail through chat completions or silent fallback.

## Best Possible Solution

Define the media-output boundary explicitly: either reject media-output chat models with a clear diagnostic, or add a documented output-capability contract that routes Poe through the dedicated image/video provider surfaces.

## Reproduction Assessment

No for the exact live Poe failure; no credentialed Poe request was run. Source inspection gives a medium-confidence static reproduction that configured chat models lack output modality metadata while image/video generation uses separate provider runtimes.

## Solution Assessment

Unclear. A narrow unsupported-media diagnostic would improve UX, but the maintainable fix needs a maintainer decision on whether output metadata belongs in chat model config or only in dedicated generation providers.
