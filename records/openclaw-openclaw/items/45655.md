## Summary

Keep open. Current main has separate image/video generation tools and stricter user-selected model fallback behavior, but configured chat/session model rows still have no output-modality contract and custom provider rows still enter the LLM stream path, so the reported media-output mismatch is not resolved.

## What This Changes

The issue asks OpenClaw to either support Poe media-output models or reject image/video-output model selections clearly instead of sending malformed text-centric requests or falling back.

## Best Possible Solution

Add a provider-generic output/capability boundary for configured models, reject unsupported image/video-output chat selections clearly, and route supported generation through the existing image_generate/video_generate provider contracts.

## Reproduction Assessment

Yes for the OpenClaw-side mismatch: current source accepts configured provider model rows without output metadata and routes them through LLM streaming. The exact Poe backend 400 was not live-reproduced in this read-only review.

## Solution Assessment

No current-main solution exists yet. The maintainable fix should be a generic output/capability contract and diagnostic, not a Poe-specific name blacklist.
