## Summary

Keep open. Current main and v2026.6.10 still have static Control UI navigation plus the narrower plugin descriptor/action surface, while the closest implementation path remains the open https://github.com/openclaw/openclaw/pull/80388 and still needs maintainer API/security review.

## What This Changes

Request to let plugins declare native Control UI pages or tabs that load plugin UI assets inside the main Control UI shell.

## Best Possible Solution

Keep this as the canonical plugin-native Control UI page design track, then land or replace the open implementation PR only after maintainers choose the public SDK/Gateway/UI contract and browser security model.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks plugin page declarations, dynamic native navigation, and plugin page launch/listing methods.

## Solution Assessment

Unclear until maintainers decide the plugin browser trust model and API contract. The open entry-point PR is the closest plausible path, but the safer final shape may be host-owned descriptors, sandboxed iframes, generic Custom Elements, or a narrower staged API.
