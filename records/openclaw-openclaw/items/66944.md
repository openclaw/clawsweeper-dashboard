Keep open. Current main still lacks a merged plugin Control UI page/entry-point system, and the same remaining work is now being handled by an open implementation PR that should be reviewed and landed or redirected rather than closing this issue early.

## What This Changes

The issue requests a public plugin UI page/entry-point API so plugins can add native Control UI navigation entries instead of standalone plugin routes.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/80388 after its remaining issue is fixed, or redirect it into the broader https://github.com/openclaw/openclaw/issues/71736 design while preserving the native plugin UI entry requirement.
