## Summary

Keep open. Current main and v2026.6.10 still have static Control UI navigation plus only the narrower plugin descriptor/action surface; the closest implementation path is the open https://github.com/openclaw/openclaw/pull/80388, which still needs maintainer API and browser-security review.

## What This Changes

Request to let plugins declare native Control UI pages or tabs that load inside the main UI shell instead of standalone plugin pages.

## Best Possible Solution

Keep this as the canonical native plugin Control UI page design track, then land or replace the open implementation only after maintainers choose the public SDK/Gateway/UI contract and browser security model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that current main lacks plugin page declarations, plugin page bootstrap metadata, dynamic native navigation, and plugin page launch/listing methods.

## Solution Assessment

Unclear until maintainers choose the plugin browser trust model and API contract. The open entry-point PR is the closest plausible path, but the safer final shape may be sandboxed iframes, host-owned descriptors, generic Custom Elements, or a narrower staged API.
