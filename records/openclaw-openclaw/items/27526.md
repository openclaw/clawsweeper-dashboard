## Summary

Keep open. Current main has flat session listing, search, labels, New Chat session creation, a header session picker, and sub-agent primitives, but it still lacks project/topic directories, persistent folder/tag/pin/recommendation semantics, side-by-side multi-model collaboration, committee workflows, task extraction, and project tracking; the discussion also keeps this as the canonical tracker.

## What This Changes

The issue requests first-class WebChat conversation organization plus multi-AI collaboration and project-committee workflows.

## Best Possible Solution

Keep this as the canonical product tracker and split it into durable session-organization metadata/UI milestones plus a separate multi-agent collaboration design that reuses sub-agent primitives where appropriate.

## Reproduction Assessment

Not applicable. This is a broad enhancement request, and current source/docs inspection verifies partial primitives rather than a broken existing documented behavior.

## Solution Assessment

No, not as one monolithic implementation. The maintainable path is to split storage/protocol/UI conversation-management work from multi-agent collaboration semantics and review narrower PRs against this tracker.
