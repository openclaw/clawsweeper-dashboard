## What This Changes

The issue reports that the official Docker image lacks `trash-cli`, causing `openclaw agents delete` to remove agent config while leaving owned workspace or state directories on disk.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84394 if its CI and maintainer review stay clean, then close this issue as fixed while handling the parent-directory residual separately.

## Reproduction Assessment

Yes by source inspection: current main and `v2026.5.20` call a PATH-resolved `trash` binary while the default Docker image does not install one. I did not execute the Docker repro in this read-only review.

## Solution Assessment

Yes for the main bug: reusing the existing fs-safe trash helper is narrower than adding `trash-cli` to the base image or hard-deleting paths, and it aligns CLI cleanup with the Gateway precedent. The parent-directory residual should remain a separate review topic.
