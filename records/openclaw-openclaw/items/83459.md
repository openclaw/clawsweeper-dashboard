Keep open. ## What This Changes

The issue reports that Docker `openclaw agents delete` removes config state but leaves workspace or agent directories behind when the image lacks a trash provider.

## Best Possible Solution

Unify CLI and Gateway deletion on a safe, non-PATH-dependent trash path, add guarded cleanup of the standard parent agent directory, preserve shared-workspace retention, and prove the result in Docker.

## Reproduction Assessment

Yes by high-confidence source inspection: the offline CLI path calls a `trash` binary and latest Docker/runtime package lists do not install one. I did not execute the official Docker image during this read-only review, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No, adding only `trash-cli` would address the official image but would leave CLI/Gateway cleanup split and the parent-directory residual. The best fix is a narrow shared cleanup path plus guarded standard-layout parent cleanup and Docker proof.
