## Summary

Keep this PR open for maintainer follow-up. Current main does not implement file-backed persistence for the stdio ACP bridge session store, and the proposed branch is not merge-ready because the core persistence helper is non-functional, the branch mixes unrelated compaction/shared-context/gateway behavior, and the added shared-context surface has security-sensitive access-boundary concerns.

## What This Changes

The PR branch adds a file-backed ACP stdio bridge session store, adjusts ACP server stdio stream construction, adds enhanced/parallel compaction metrics and shared context configuration, and changes gateway chat handling to auto-abort active runs before new sends.

## Best Possible Solution

A maintainable path is to split a focused ACP bridge persistence change in `src/acp/session.ts` and `src/acp/translator.ts`, add restart/loadSession coverage for bridge session mapping, validate persisted JSON before restoring state, and leave enhanced compaction, shared context, and gateway active-run behavior to separate reviewed PRs with their own docs/changelog and security review.

## Reproduction Assessment

Yes. The core PR defect is reproducible by static review of the PR diff: `_getAllSessions` uses `Record<string, AcpSession>` as a runtime expression, and current-main source search confirms there is no existing file-backed bridge store that makes the PR obsolete.

## Solution Assessment

No. The submitted branch is not the best way to solve the ACP bridge persistence request because it mixes unrelated product surfaces and introduces security-sensitive shared context; the safer alternative is a narrow file-backed bridge store with schema validation and targeted restart tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Return real session data from _getAllSessions:** `src/acp/session.ts:95`
  - body: `_getAllSessions` is implemented as `() => Record<string, AcpSession>`, but `Record` is a TypeScript type, not runtime data. The file-backed store will fail or save `{}` instead of the active sessions, so bridge sessions still will not persist across restarts.
  - confidence: 0.95
- **[P1] Pass enhanced compaction config without global mutation:** `src/agents/pi-extensions/compaction-safeguard.ts:279-295`
  - body: Calling `configureEnhancedCompaction(enhancedConfig.config)` inside the per-compaction path rewrites module-level config shared by all sessions. Concurrent sessions can run with another session's settings; pass the config into `enhancedCompaction` for that invocation instead.
  - confidence: 0.9
- **[P2] Validate persisted session JSON before restoring:** `src/acp/session.ts:119-130`
  - body: `loadFromDisk` reads arbitrary JSON and then accesses `sessionData.sessionKey` and `sessionData.cwd` without proving the entry is an object with string fields. Corrupt or unexpected state should be skipped explicitly before calling `createSession`.
  - confidence: 0.86
- **[P2] Use explicit parent-child context relationships:** `src/agents/shared-context.ts:257-260`
  - body: The shared-context access check treats one agent id being a prefix of another as a parent-child relationship. That can share context from `main` to ids such as `main-attacker`; track explicit parent-child membership instead of inferring it from strings.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:254-264`
  - body: This PR adds user-facing ACP persistence, compaction config, and gateway behavior but does not touch `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-facing feat/fix/perf changes before merge.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces local persistence and cross-agent context sharing; no workflow, lockfile, package publishing, or third-party action supply-chain change is visible, but the new data access boundaries need fixes before merge.

Concerns:

- **[medium] Prefix-based shared-context access can over-share data:** `src/agents/shared-context.ts:257`
  - body: `parent-child` access is inferred from string prefixes, so a crafted or human-readable agent id can match an unrelated agent and receive context that should stay isolated.
  - confidence: 0.88
- **[low] Persisted ACP session data lacks shape validation:** `src/acp/session.ts:119`
  - body: The file-backed session loader trusts parsed JSON entry fields before restoring sessions. A corrupted or attacker-written state file could create confusing session mappings instead of being cleanly rejected.
  - confidence: 0.84
