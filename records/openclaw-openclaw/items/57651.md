## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Return real sessions from `_getAllSessions`:** `src/acp/session.ts:95`
  - body: This function body uses the TypeScript `Record` utility as a runtime value, and `_getAllSessions` is not declared on `AcpSessionStore`. That blocks typechecking or leaves `saveToDisk()` unable to serialize actual sessions, so the advertised file-backed persistence does not work.
  - confidence: 0.97
- **[P2] Keep enhanced compaction config session-local:** `src/agents/pi-extensions/compaction-safeguard.ts:290-292`
  - body: Calling `configureEnhancedCompaction()` here rewrites module-level config during an individual compaction. Concurrent sessions with different settings can race and run under another session's config; pass the config into `enhancedCompaction()` for this call instead.
  - confidence: 0.88
- **[P2] Replace prefix-based shared-context access:** `src/agents/shared-context.ts:371-372`
  - body: `parent-child` access should not be inferred from string prefixes. An agent id like `main` would grant access to unrelated ids such as `main-attacker`, so this needs explicit relationship tracking before shared context is enabled.
  - confidence: 0.86
- **[P2] Validate persisted session JSON before replaying it:** `src/acp/session.ts:119-125`
  - body: `JSON.parse()` data is replayed into `createSession()` without checking that each entry has string `sessionKey` and `cwd` fields. A corrupt or unexpected state file can create malformed bridge sessions instead of being skipped cleanly.
  - confidence: 0.8
