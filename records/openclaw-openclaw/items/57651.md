## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Return actual sessions from the persistence helper:** `src/acp/session.ts:95`
  - body: `_getAllSessions` returns the TypeScript `Record` type expression instead of the in-memory session map. The file-backed store cannot serialize active sessions, so the advertised persistence either fails to compile/run or writes the wrong data.
  - confidence: 0.97
- **[P1] Pass compaction config without mutating global state:** `src/agents/pi-extensions/compaction-safeguard.ts:290-292`
  - body: Calling `configureEnhancedCompaction` inside each compaction run rewrites a module-level config shared by all sessions. Concurrent runs can execute with another session's settings; pass the per-run config into `enhancedCompaction` instead.
  - confidence: 0.91
- **[P2] Validate persisted session entries before restoring:** `src/acp/session.ts:119-125`
  - body: `loadFromDisk` reads arbitrary JSON and dereferences `sessionData.sessionKey` and `sessionData.cwd` without proving the entry shape. Corrupt or unexpected state should be skipped before recreating sessions.
  - confidence: 0.88
- **[P2] Use explicit relationships for shared context access:** `src/agents/shared-context.ts:371`
  - body: The `parent-child` access check treats string prefixes as a relationship, so an ID like `main-attacker` can match `main` and receive context that should remain isolated. Track explicit parent/child membership instead of inferring it from names.
  - confidence: 0.89
- **[P2] Write ACP session state with private permissions:** `src/acp/session.ts:145-146`
  - body: The session file and parent directory are created with default permissions. Persisted session keys and workspace paths should use private file and directory modes, matching the current event-ledger storage pattern.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces local persistence and cross-agent shared context; both have concrete data-boundary issues before merge.

Concerns:

- **[medium] Prefix-based shared-context access can over-share data:** `src/agents/shared-context.ts:371`
  - body: `parent-child` scope is inferred from string prefixes, so unrelated human-readable agent IDs can match and receive shared context.
  - confidence: 0.89
- **[low] Persisted ACP session JSON lacks shape validation:** `src/acp/session.ts:119`
  - body: The loader trusts parsed JSON fields before recreating sessions, which can turn corrupt or attacker-written local state into confusing session mappings.
  - confidence: 0.86
- **[low] ACP session state uses default filesystem permissions:** `src/acp/session.ts:146`
  - body: The PR writes the persisted session file without explicit private mode and creates the parent directory without explicit private mode, exposing session metadata on permissive umasks.
  - confidence: 0.78
