## Summary

Keep this PR open but do not merge as-is. Current main still lacks the proposed file-backed ACP bridge store, but the submitted patch is broad, stale against current agent compaction paths, and has concrete correctness and security-sensitive data-sharing defects already visible in the diff for https://github.com/openclaw/openclaw/pull/57651.

## What This Changes

The PR branch adds file-backed ACP stdio bridge session persistence, enhanced/parallel compaction with metrics and shared context, agent-defaults config schema entries, and gateway active-run auto-abort behavior.

## Best Possible Solution

Split out a narrow ACP bridge persistence change in the current `src/acp/session.ts` and `src/acp/translator.ts` paths, validate persisted JSON before restoring sessions, add restart/loadSession coverage for the bridge mapping, and leave compaction/shared-context/gateway behavior to separate reviewed PRs.

## Reproduction Assessment

Yes. Static review of the PR diff reproduces the primary defect: `_getAllSessions` returns a TypeScript type expression instead of real session data, so the file-backed store cannot persist active sessions. Current-main inspection also confirms there is no existing file-backed ACP bridge store that makes this PR obsolete.

## Solution Assessment

No. The submitted branch is not the best way to solve ACP bridge persistence because it bundles multiple unrelated systems and introduces unsafe shared-context behavior; the maintainable alternative is a focused persistence patch with schema guards and targeted restart tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Return real session data from _getAllSessions:** `src/acp/session.ts:95`
  - body: `_getAllSessions` is implemented as `() => Record<string, AcpSession>`, but `Record` is a TypeScript type, not runtime data. The file-backed store will fail or save `{}` instead of the active sessions, so bridge sessions still will not persist across restarts.
  - confidence: 0.95
- **[P1] Pass enhanced compaction config without global mutation:** `src/agents/pi-extensions/compaction-safeguard.ts:279-295`
  - body: Calling `configureEnhancedCompaction(enhancedConfig.config)` inside the per-compaction path rewrites module-level config shared by all sessions. Concurrent sessions can run with another session's settings; pass the config into that compaction invocation instead.
  - confidence: 0.9
- **[P2] Validate persisted session JSON before restoring:** `src/acp/session.ts:119-130`
  - body: `loadFromDisk` reads arbitrary JSON and then accesses `sessionData.sessionKey` and `sessionData.cwd` without proving the entry is an object with string fields. Corrupt or unexpected state should be skipped explicitly before calling `createSession`.
  - confidence: 0.86
- **[P2] Use explicit parent-child context relationships:** `src/agents/shared-context.ts:257-260`
  - body: The shared-context access check treats one agent ID being a prefix of another as a parent-child relationship. That can share context from `main` to IDs such as `main-attacker`; track explicit parent-child membership instead of inferring it from strings.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:254-264`
  - body: This PR adds user-facing ACP persistence, compaction config, and gateway behavior but does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces local persistence and cross-agent shared context; no supply-chain file change is visible, but the new data access boundaries need fixes before merge.

Concerns:

- **[medium] Prefix-based shared-context access can over-share data:** `src/agents/shared-context.ts:257`
  - body: `parent-child` access is inferred from string prefixes, so a crafted or human-readable agent ID can match an unrelated agent and receive context that should stay isolated.
  - confidence: 0.88
- **[low] Persisted ACP session data lacks shape validation:** `src/acp/session.ts:119`
  - body: The file-backed session loader trusts parsed JSON entry fields before restoring sessions. A corrupted or attacker-written state file could create confusing session mappings instead of being cleanly rejected.
  - confidence: 0.84
