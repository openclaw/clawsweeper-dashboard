## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Return actual sessions from the store:** `src/acp/session.ts:95`
  - body: `Record<string, AcpSession>` is a type utility, not runtime data. This makes the proposed save path throw or write `{}`, so the advertised file-backed session persistence cannot work.
  - confidence: 0.96
- **[P2] Pass enhanced compaction config without mutating globals:** `src/agents/pi-extensions/compaction-safeguard.ts:290-306`
  - body: Calling `configureEnhancedCompaction()` inside each compaction rewrites process-wide state. Concurrent sessions can race and run with another session's settings; pass the config into `enhancedCompaction()` instead.
  - confidence: 0.87
- **[P2] Validate persisted session records before loading:** `src/acp/session.ts:119-126`
  - body: The JSON loader reads arbitrary file contents and then uses `sessionData.sessionKey` and `sessionData.cwd` without a shape check. Corrupt or unexpected state can create invalid sessions instead of being skipped cleanly.
  - confidence: 0.82
- **[P2] Use explicit relationships for parent-child context:** `src/agents/shared-context.ts:370-372`
  - body: Prefix matching treats unrelated ids such as `main` and `main-attacker` as related. That can expose shared context across agents that only have similar names, so parent-child access needs an explicit relationship.
  - confidence: 0.84

## Security Review
