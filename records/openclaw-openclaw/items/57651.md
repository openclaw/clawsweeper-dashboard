## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Return real sessions from the persistence helper:** `src/acp/session.ts:95`
  - body: `_getAllSessions` returns the type-only `Record<string, AcpSession>` expression instead of the in-memory session map. `saveToDisk` calls this helper, so the file-backed store cannot persist active sessions.
  - confidence: 0.97
- **[P1] Pass compaction config without mutating global state:** `src/agents/pi-extensions/compaction-safeguard.ts:290-292`
  - body: `configureEnhancedCompaction` rewrites module-level config from a per-session compaction callback. Concurrent sessions can run with another session's settings; pass the config into `enhancedCompaction` for that call instead.
  - confidence: 0.91
- **[P2] Validate persisted session entries before restoring:** `src/acp/session.ts:119-125`
  - body: `loadFromDisk` parses arbitrary JSON and reads `sessionData.sessionKey` and `sessionData.cwd` without checking each entry is an object with string fields. Corrupt local state should be skipped before recreating sessions.
  - confidence: 0.88
- **[P2] Use explicit relationships for shared-context access:** `src/agents/shared-context.ts:371`
  - body: The `parent-child` scope treats string prefixes as relationships, so an ID such as `main-attacker` can match `main` and receive context that should remain isolated.
  - confidence: 0.89
- **[P2] Write ACP session state with private permissions:** `src/acp/session.ts:145-146`
  - body: The session directory and JSON file are created with default modes even though they contain session keys and workspace paths. Use private directory/file modes like the current ACP event ledger.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces local ACP persistence and cross-agent shared context with concrete data-boundary issues before merge.

Concerns:

- **[medium] Prefix-based shared context can over-share data:** `src/agents/shared-context.ts:371`
  - body: `parent-child` access is inferred from agent ID string prefixes, allowing unrelated human-readable IDs to match and receive shared context.
  - confidence: 0.89
- **[low] Persisted ACP JSON lacks shape validation:** `src/acp/session.ts:119`
  - body: The loader trusts parsed JSON fields before recreating sessions, so corrupt or attacker-written local state can create confusing or unintended session mappings.
  - confidence: 0.86
- **[low] ACP session state uses default filesystem permissions:** `src/acp/session.ts:146`
  - body: The PR writes session metadata without explicit private file and directory modes, exposing session keys and workspace paths on permissive umasks.
