## What This Changes

The PR adds a non-fatal cron diagnostic warning when a job allows `web_search` but no managed web search provider plugin is enabled.

## Best Possible Solution

Gate the warning on actual effective `web_search` availability for the selected cron runtime, including native provider search, and prove it with focused tests plus redacted real cron output.

## Reproduction Assessment

Yes, source-level: the linked issue gives a concrete cron `toolsAllow: ["web_search"]` setup with disabled managed providers, and current main has no corresponding cron preflight warning. I did not run a live cron job in this read-only review.

## Solution Assessment

No. A warning based only on `listWebSearchProviders()` is too narrow because current main also has a provider-native `web_search` path that intentionally suppresses the managed tool.
