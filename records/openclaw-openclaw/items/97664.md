## Summary

Keep open: this PR targets a real cron diagnostics gap, but the patch is not merge-ready because it warns based only on managed web-search provider plugins and can produce a false warning when provider-native OpenAI/Codex `web_search` is active. ## What This Changes

The PR adds a non-fatal cron preflight diagnostic when an agent-turn cron payload allows `web_search` but no managed web search provider plugin is enabled.

## Best Possible Solution

Gate the warning on effective `web_search` availability for the selected cron runtime, including managed providers and active native search, then prove both paths with focused tests and redacted real cron output.

## Solution Assessment

No. The requested warning is the right boundary, but this implementation is too narrow because `listWebSearchProviders()` ignores active provider-native OpenAI/Codex `web_search`.
