## Summary

Keep open. This PR targets a real open gateway auth cluster, but the branch is stale/conflicting, lacks real remote gateway proof, and changes a security-sensitive WebSocket authorization boundary that still needs maintainer policy review.

## What This Changes

The PR threads authMethod into WebSocket missing-device policy so token/password operator auth can allow device-less non-local clients, plus a token-path regression test.

## Best Possible Solution

Settle one WebSocket device-less token/password scope policy, then land a current-main implementation that aligns connect policy, scope clearing, auth tests, protocol/security docs, and redacted real gateway proof.

## Reproduction Assessment

Yes, at source level. Current main tests expect valid shared-token/password operator connects with device null to have scopes cleared and a scoped RPC to fail with missing scope; I did not run a live gateway in this read-only review.

## Solution Assessment

No. The direction may be acceptable only after policy approval, but this patch is stale and does not yet align current-main scope clearing, tests, docs, or real behavior proof.
