## Summary

Keep open. The PR targets a source-reproducible gateway scope-clearing problem, but the branch is not mergeable as-is because current main separately clears token/password scopes after the missing-device allow decision, and the requested remote device-less operator behavior still needs security/product approval plus real gateway proof.

## What This Changes

The PR threads `authMethod` into WebSocket missing-device policy so successful token/password operator auth can return `allow`, and adds a token-path regression assertion.

## Best Possible Solution

## Solution Assessment

No. The PR touches the right area, but on current main it must also update the separate scope-clearing helper and the auth baselines, and the remote device-less operator trust boundary needs explicit maintainer/security approval.
