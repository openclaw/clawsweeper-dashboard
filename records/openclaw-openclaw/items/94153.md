## Label Justifications

- `P1`: The issue describes lost subagent completion announcements in real requester workflows after completed child work.
- `impact:session-state`: The failure leaves parent/requester session state out of sync with completed child runs.
- `impact:message-loss`: The reported symptom is completed subagent output failing to reach the requester after announce retries are exhausted.

## Summary

Keep open: current main fixes the narrower non-terminal pending handoff branch, but it still does not implement the requested durable persistence/queue contract for subagent completion announcements, and related open work only partially overlaps.

## What This Changes

The issue requests durable persistence of subagent completion announcements so a busy requester session can receive completed child output on a later turn instead of losing the announcement after direct dispatch retries.

## Best Possible Solution

Define one maintainer-approved durable parent-visible completion delivery contract in the core agent/session layer, keep the merged direct-pending fix, and coordinate any persistent fallback with the open write-lock and orphan-signal trackers.

## Reproduction Assessment

Yes at source level, but not as a live current-main race reproduction in this read-only review. Current main lacks durable persistence for failed requester handoff paths, and the existing system-event queue is intentionally ephemeral.

## Solution Assessment

No final solution is on main. The merged narrow fix is the right answer for the pending-status branch, but this issue needs a maintainer-chosen durable delivery contract rather than an ad hoc in-memory queue.
