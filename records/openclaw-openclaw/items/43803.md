## Summary

## What This Changes

The issue asks for `config.patch` writes to `browser.profiles.*` to avoid SIGUSR1 restarts when the reload planner can handle them as hot-reloadable browser profile changes.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/93827` if its narrow browser-profile hot-reload classification is accepted; otherwise keep this issue as the canonical report until a replacement PR fixes the reload metadata and regression tests.

## Solution Assessment

Yes: the best fix is a narrow browser-owned reload metadata and planner-classification change with regression coverage, while keeping broader browser settings restart-scoped. The live linked PR is the right place to review that fix before closing this issue.
