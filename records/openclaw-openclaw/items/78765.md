## Summary

Keep open: current main still inserts a synthetic space in mixed Latin/CJK uninterrupted TUI prose, and this PR is a focused fix with regression coverage, real behavior proof, green checks, and no blocking code findings.

## What This Changes

The branch adds a CJK-script guard to TUI long-token sanitization and tests that long pure and mixed CJK prose is preserved without synthetic spaces.

## Best Possible Solution

Land the focused sanitizer change with its regression coverage once ordinary maintainer landing gates remain green.

## Reproduction Assessment

Yes from source: current main routes a >32-character mixed Latin/CJK token through the generic long-token chunker and inserts a visible space. I did not run a live current-main TUI session, so the structured status is source-reproducible rather than live reproduced.

## Solution Assessment

Yes. Preserving tokens that contain CJK script before generic chunking is narrow and aligns with pi-tui's grapheme-level long-word wrapping; the pure-CJK test is redundant on current main, but the mixed CJK test covers the real regression.
