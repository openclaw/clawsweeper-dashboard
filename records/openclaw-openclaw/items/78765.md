## What This Changes

The PR adds a CJK-script guard to TUI long-token sanitization and regression tests so long CJK-containing assistant output is preserved without inserted display spaces.

## Best Possible Solution

Land this narrow TUI sanitizer fix through normal maintainer merge gates, with any user-facing changelog entry handled during landing if maintainers want one.

## Reproduction Assessment

Yes from source: current main preserves pure CJK because it lacks ASCII alphanumerics, but mixed Latin/CJK long prose reaches the 32-character chunker and can receive inserted spaces. The PR's after-fix TUI screenshot covers the real runtime path.

## Solution Assessment

Yes. Preserving CJK-containing prose tokens before generic ASCII chunking is a narrow maintainable fix because pi-tui 0.73.1 already wraps overlong rendered tokens at grapheme boundaries.
