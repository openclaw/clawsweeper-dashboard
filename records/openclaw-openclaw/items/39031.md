## Summary

Keep open. Current main still does not forward legacy iMessage/imsg tapback reactions to the agent, and open PR #39322 plus sibling filtering work in #60274/#60677 need maintainer reconciliation before this can close.

## What This Changes

Request to forward iMessage tapback reactions from legacy imsg monitoring to the agent as inbound reaction events with target message, sender, emoji, and timestamp metadata.

## Best Possible Solution

Define an iMessage-owned reaction event contract and config policy, then land one consolidated implementation that either forwards opted-in tapbacks as typed events or filters them by default.

## Reproduction Assessment

Yes. Source inspection on current main shows legacy imsg subscribes only to normal message notifications and parses only raw.message, so a reaction notification has no path into iMessage inbound context or agent dispatch.

## Solution Assessment

Unclear. Forwarding tapbacks may be the right end state, but maintainers need to reconcile that with sibling filtering work so tapbacks do not become unwanted normal prompts.
