## Summary

Keep open. Current main now preserves imsg reaction metadata and forwards reactions as ordinary inbound text, but it still does not define the requested structured reaction-event contract or settled filtering/group policy, and the same-author implementation PR is still open.

## What This Changes

The issue requests forwarding iMessage tapback reactions to agents as reaction-aware inbound events rather than only as normal text messages.

## Best Possible Solution

Define a plugin-local iMessage reaction contract with explicit DM/group and filtering semantics, then resolve https://github.com/openclaw/openclaw/pull/39322 and the related filtering work against that contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence behavior check: current main can receive imsg reaction metadata, but it converts it into normal message text instead of a structured reaction event.

## Solution Assessment

Unclear. Forwarding tapbacks is directionally useful, but the current main implementation is only partial text synthesis and the open PR path needs a maintainer decision on event shape and filtering policy.
