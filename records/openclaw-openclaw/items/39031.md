## Summary

Keep open. Current main still lacks a legacy imsg path that forwards tapback reactions to the agent, and the open related implementation/filtering work should be reconciled before this is closed or turned into another repair branch.

## What This Changes

Request to deliver iMessage tapback reactions to the agent with target message, reaction, sender, and timestamp metadata.

## Best Possible Solution

Resolve this through a deliberate opt-in legacy imsg reaction event path, preferably by reviewing #39322 together with the tapback-filtering work so reactions are typed or explicitly filtered instead of becoming unwanted normal prompts.

## Reproduction Assessment

Yes. Source inspection on current main shows only message notifications are subscribed/handled and no reaction fields are parsed or included in the inbound context, so legacy imsg tapbacks have no current path to the agent.

## Solution Assessment

Unclear. Forwarding reactions can be the right end state, but maintainers need to settle the opt-in config and event shape against the sibling filtering reports before accepting an implementation.
