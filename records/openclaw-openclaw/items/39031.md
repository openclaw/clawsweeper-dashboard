## Summary

Keep open. Current main still does not forward legacy iMessage/imsg tapback reactions to the agent, and the item is paired with open implementation and filtering work that maintainers should reconcile before closing or spawning another repair branch.

## What This Changes

Forward iMessage tapback reactions from the legacy imsg channel to the agent with target message, reaction, sender, and timestamp metadata.

## Best Possible Solution

Keep this as the canonical legacy imsg tapback-to-agent request and resolve it through a deliberate opt-in implementation that is reaction-aware and coordinated with the open tapback filtering work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: legacy imsg only subscribes for attachments, only handles `message` notifications, and has no reaction payload/config/context path.

## Solution Assessment

Unclear. Forwarding tapbacks can be the right end state, but maintainers need to choose an opt-in reaction event shape and reconcile it with the sibling filtering work so tapbacks do not become unwanted normal prompts.
