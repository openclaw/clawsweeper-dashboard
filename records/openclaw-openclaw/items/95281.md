- **[P1] Make mid-stream finalization idempotent:** `src/llm/providers/openai-completions.ts:404`
  - body: This new transition path calls `finishBlock(thinkingBlock)`, but the same block stays in `blocks` and the existing end-of-stream loop calls `finishBlock` on it again. Add a finished-block guard or otherwise prevent the EOF loop from refinalizing blocks sealed mid-stream.
