## Summary

Keep open: this draft feature PR needs normal maintainer review, real behavior proof, and at least one correctness fix because the reset trigger is placed where no-hook resets skip it.

## What This Changes

The PR adds a built-in training export module with docs, config schema/help, compaction/reset/trajectory triggers, tests, and exported OpenAI conversion helpers.

## Best Possible Solution

Make training export an explicit, privacy-reviewed capability, move the reset trigger outside the hook-only early return, align docs/schema/cleanup behavior, and prove the real compaction/reset/trajectory outputs before merge.

## Reproduction Assessment

For the feature as a whole, runtime reproduction is not applicable yet because this is a draft PR. For the blocking review finding, yes: source inspection of the PR head shows no-hook resets return before the added before_reset training export call.

## Solution Assessment

No. A trajectory-first exporter may be viable, but the current patch is not the best mergeable shape because reset export is coupled to plugin-hook presence and default-on persistent training data needs an explicit maintainer privacy decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Move reset export outside the hook-only early return:** `src/gateway/session-reset-service.ts:456-457`
  - body: The new `before_reset` export is placed after `if (!hookRunner?.hasHooks("before_reset")) return;`, so installs without a registered `before_reset` hook never call `runTrainingExport` during reset. That makes the advertised reset trigger silently miss normal setups; move this export before the hook gate or into reset flow that runs independently of plugin hooks.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The PR introduces default-on persistent training-data writes from sensitive trajectory content, which needs explicit privacy and retention review before merge.

Concerns:

- **[medium] Make training export opt-in before persisting transcripts:** `src/agents/pi-embedded-runner/extensions.ts:165`
  - body: The PR registers export when `trainingExport.enabled !== false` and appends episodes under the global OpenClaw state directory. Existing trajectory docs treat prompts, messages, tool schemas, tool results, runtime events, and local paths as sensitive approval-gated data with best-effort redaction, so a default-on secondary training JSONL can persist private session content without a clear opt-in or cleanup policy.
