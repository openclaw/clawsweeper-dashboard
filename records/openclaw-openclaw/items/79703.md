## Summary

Keep open: this draft external PR proposes useful new feature work, but it is not merge-ready because the new module has a parse-time duplicate export, data correctness bugs, missing real behavior proof, and unresolved privacy/retention policy for sensitive training JSONL output.

## What This Changes

This PR adds an opt-in core training export system with config schema/help, docs, JSONL episode writing, compaction/reset/trajectory triggers, message/tool conversion helpers, and unit tests.

## Best Possible Solution

Continue as a maintainer-reviewed opt-in training export feature only after the parse/correctness bugs are fixed, docs and schema agree, privacy/retention policy is documented, and the contributor supplies redacted real runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the blocking patch defects: source inspection plus a Node ESM check show the duplicate export parse failure, and source/test inspection shows the non-compaction tail-trimming bug; runtime feature reproduction is not applicable to current main because the feature is new and absent.

## Solution Assessment

Is this the best way to solve the issue? No; the trajectory-first direction may be viable, but this patch is not mergeable until the duplicate export, filtering behavior, docs/schema mismatch, privacy policy, and real behavior proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Remove the duplicate named export:** `src/training-export.ts:1181`
  - body: `setCompactionTrainingExportRuntime` is already exported where it is declared, then re-exported here. Native ESM rejects duplicate exported names, so importing this module will fail before the new triggers can run.
  - confidence: 0.95
- **[P2] Restrict trailing-message trimming to compaction snapshots:** `src/training-export.ts:686-691`
  - body: `buildTrainExample` trims trailing non-assistant messages for every trigger. For `before_reset` and `trajectory_export`, this can silently drop an in-progress user/tool tail and export stale previous assistant data; the PR's own trajectory-export test expects those cases to be skipped.
  - confidence: 0.91
- **[P2] Align the compact summary config with the schema:** `docs/training-export.md:136-140`
  - body: The docs advertise `trainingExport.compactSummary.provider/id`, but the strict config schema only accepts `enabled` and `compat`. Users following the new docs will hit config validation failures.
  - confidence: 0.86
- **[P3] Pass summary session metadata into the episode:** `src/training-export.ts:753-757`
  - body: The compaction summary branch builds the payload without `sessionId` or `model`, so compact-summary examples lose attribution even though the helper supports those meta fields.
  - confidence: 0.82
