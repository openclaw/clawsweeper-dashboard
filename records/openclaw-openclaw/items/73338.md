## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Using the existing per-config Gateway lock as the runtime fact and applying it only to default local TUI URL resolution is narrower than adding a new config/env surface and preserves explicit --url plus remote config precedence.
