## What This Changes

The branch writes optional Gateway lock port metadata, lets default local TUI resolution read verified active locks, recognizes titled Gateway processes, and updates focused docs/tests.

## Best Possible Solution

Land the lock-backed TUI fix after preserving documented env-port precedence, or after maintainers explicitly approve and document active-lock precedence with tests.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` resolve the default local TUI URL through config/env/default port data while the active Gateway lock payload does not carry the runtime port.

## Solution Assessment

No, not as-is. Recording the active runtime port in Gateway lock metadata is the right layer, but changing `OPENCLAW_GATEWAY_PORT` precedence needs preservation or explicit maintainer approval with docs and tests.
