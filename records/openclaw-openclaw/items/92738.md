## What This Changes

The branch updates `message_tool_only` prompt/hint wording and shared auto-reply progress forwarding so ClickClack room-event turns can show tool/progress activity while final visible output is sent through the `message` tool.

## Best Possible Solution

Land the narrow `message_tool_only` progress fix while keeping private finals suppressed, and align `commentaryProgressEnabled` with the same predicate that actually permits item-event forwarding.

## Reproduction Assessment

Yes, with medium confidence: the PR body gives a concrete ClickClack before/after path, and current source shows external room events use `message_tool_only` while room-event progress callbacks are suppressed on `main`. I did not run a fresh live ClickClack reproduction in this read-only review.

## Solution Assessment

Yes, mostly: forwarding opted-in room-event progress while preserving `message_tool_only` final-text suppression is the narrow owner-boundary fix. The safer refinement is to make commentary enablement use the same forwarding predicate as the callback itself.
