## Summary

Keep open: current main still has text-only Signal mention gating for this path, and this PR is a focused Signal plugin bug fix. It still needs real Signal after-fix proof and an operator-facing docs update before merge.

## What This Changes

The PR adds Signal native mention metadata matching against the bot account, wires it into Signal group mention gating, and adds tests for uuid, phone, other-participant, and malformed-span cases.

## Best Possible Solution

Land the focused Signal plugin fix after real Signal proof and docs are added, while leaving metadata-only no-regex enforcement as a separate follow-up if maintainers want that policy change.

## Reproduction Assessment

Yes, by source inspection: current main hydrates Signal native mention metadata to @uuid/@phone and then checks only display-name regexes, so a requireMention group with a pattern like @caio skips a native bot mention. I did not run a live Signal account reproduction in this read-only review.

## Solution Assessment

Yes for the targeted bug: matching structured Signal mention metadata against deps.account/deps.accountUuid inside the Signal plugin is the narrowest owner-local fix. The merge-ready path still needs docs and real behavior proof rather than a broader core policy change.
