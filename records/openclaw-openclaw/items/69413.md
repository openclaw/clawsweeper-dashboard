Keep open: current main still has the Signal native-mention gap, and this PR is a focused, plausible fix with no blocking code findings found. It is not merge-ready yet because the external PR only provides tests and explicitly lacks live Signal group proof.

## What This Changes

Adds Signal native mention detection by comparing `dataMessage.mentions` UUID/phone metadata to the bot account identity, ORs that into `wasMentioned`, and adds Signal mention-gating tests.

## Best Possible Solution

Land the focused Signal plugin fix after redacted real Signal proof demonstrates native bot mentions in a `requireMention: true` group; track native-only gating without text patterns separately if maintainers want that expanded behavior.

## Reproduction Assessment

Yes, source-reproducible: current main hydrates Signal native mentions into `@uuid`/`@phone` text and then checks display-name regexes, so a native bot mention whose metadata ID does not match the display pattern is skipped under `requireMention`. I did not run a live Signal account reproduction in this read-only review.

## Solution Assessment

Yes for the scoped bug: comparing structured mention metadata to the bot account identity after validating the text span is a narrow maintainable fix. The no-`mentionPattern` native-only gating behavior is a separate scope choice, not a blocking defect in this focused patch.
