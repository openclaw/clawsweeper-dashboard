## Summary

Keep this PR open: it is a plausible, narrow fix for a source-reproducible Signal mention-gating bug, but it still needs contributor-supplied real Signal behavior proof before merge and the Signal docs would become stale if the behavior lands as-is.

## What This Changes

The PR adds Signal native mention metadata matching against the bot account UUID or phone before group mention gating, plus regression tests for UUID, phone, non-bot, and malformed mention spans.

## Best Possible Solution

Land the focused Signal plugin metadata match after redacted real Signal proof, a Signal docs update, and green checks; track metadata-only enforcement as a separate follow-up if maintainers want that policy.

## Reproduction Assessment

Yes, source-reproducible: current main renders native Signal mentions to @uuid/@phone and then checks only configured text regexes, so a display-name pattern such as @caio drops the native bot mention. I did not run a live Signal account reproduction in this read-only review.

## Solution Assessment

Yes for the targeted bug: matching structured Signal mention metadata against the bot account inside the Signal plugin is the narrow owner-local fix. It still needs real behavior proof, a docs update, and green checks before merge.
