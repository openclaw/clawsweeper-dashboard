## Summary

Keep open. The underlying data-loss problem is real, but this PR is not merge-ready because the replay path still changes queued user work into system events, which loses normal delivery semantics, wakeup behavior, durability, and trust boundaries; it also carries an unrelated runtime log artifact.

## What This Changes

This PR adds disk persistence and startup replay for followup queues and drain-window rejected inbound messages, adds queue persistence tests, adjusts Control UI `/model list` and `/model status` handling, updates the Vitest UI include list, and adds a runtime log file.

## Best Possible Solution

Rework the branch around structured durable replay: persist enough `FollowupRun` and finalized inbound context to resume through the existing followup drain or inbound dispatch machinery, preserve channel/thread routing and trust metadata, wake the intended session or delivery path explicitly, retain persisted data until processing or acknowledgement succeeds, and split the unrelated Control UI model fix into a separate focused change.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main keeps followup queues in process memory, rejects new lane work while gateway draining, and clears runtime state on shutdown; the PR head can be traced from persisted item to session-scoped system event rather than normal delivery. No live restart test was executed under the read-only constraint.

## Solution Assessment

No. System-event replay is not the best way to solve this data-loss bug because it changes role, routing, wakeup, batching, cap, and durability semantics; the safer fix is to replay through the canonical followup/inbound paths with durable acknowledgement.

Overall confidence: 0.92

Full review comments:

- **[P1] Replay recovered items through normal delivery paths:** `src/gateway/replay-persisted-followups.ts:35`
  - body: Recovered followups are converted into session-scoped system events instead of being restored to the followup queue or dispatched through the inbound path. System events do not start the target session and bypass the origin-routing logic used by `runFollowup`, so after restart a reply can sit idle or be delivered with unrelated later traffic.
  - confidence: 0.93
- **[P1] Mark replayed user content untrusted:** `src/gateway/replay-persisted-followups.ts:35`
  - body: This call, and the drain-rejected replay below it, enqueue recovered user text without `trusted: false`. Current system events default to trusted and drain as `System:` lines, so the added prose does not prevent restart-window user input from being presented with system-event authority.
  - confidence: 0.96
- **[P2] Persist complete drain-rejected contexts:** `src/auto-reply/dispatch.ts:43-56`
  - body: The drain-window serializer keeps only text and selected routing fields before replaying as a system event. Media-only messages, command authorization/body fields, and `BodyForAgent` distinctions cannot be faithfully recovered after restart.
  - confidence: 0.88
- **[P2] Keep replay files until processing succeeds:** `src/auto-reply/reply/queue/persist.ts:148`
  - body: `consumePersistedFollowups()` unlinks the pending file before replay has actually processed the items, and replay then stores them only in the in-memory system-event queue. A second restart before the next turn drains those events permanently loses the recovered messages.
  - confidence: 0.9
- **[P2] Write pending followups atomically:** `src/auto-reply/reply/queue/persist.ts:113`
  - body: The shutdown path writes `pending-followups.json` directly, while parse failures on boot delete the file. If the process dies during truncate/write, the next boot can unlink the partial file and lose the queued followups this feature is meant to preserve.
  - confidence: 0.86
- **[P2] Remove the committed runtime log:** `openclaw-2026-03-18.log:1`
  - body: `openclaw-2026-03-18.log` contains local runtime/debug and tool-call metadata unrelated to the fix. Keeping it in source leaks implementation details into clones and release artifacts.
  - confidence: 0.95
