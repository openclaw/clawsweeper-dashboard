## Summary

Keep open. Current main has durable task/lifecycle tracking and accepted run acknowledgements, but it still lacks the requested native receiver-entry ACK taxonomy, track/hop identifiers, dispatch ACK persistence, and plugin-readable landing telemetry across the named dispatch surfaces.

## What This Changes

The issue requests native receiver-entry dispatch telemetry with ACK kinds, stable persistence, and plugin guidance across cron, sessions_spawn, channel inbound, gateway agent, CLI agent, and ACP downstream dispatch paths.

## Best Possible Solution

Define a native receiver-entry telemetry contract that deliberately reuses or extends the task/lifecycle system, then add versioned persistence, bounded non-fatal writes, plugin-facing read/subscribe guidance, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the missing native contract: current-main source, docs, and tests show no requested ACK kinds, dispatch ledger, track/hop IDs, or plugin-readable landing ACK surface. The reported production incident itself was not locally replayed.

## Solution Assessment

Unclear. The proposed ACK taxonomy fits the observability gap, but maintainers need to decide whether the narrowest maintainable solution extends the existing task ledger or introduces a separate dispatch telemetry plane.
