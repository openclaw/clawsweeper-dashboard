## Summary

Keep open. Current main has background task and agent lifecycle tracking, but it does not provide the requested receiver-entry ACK kinds, stable dispatch ledger, track/hop identifiers, or plugin-facing read/watch contract.

## What This Changes

Request native receiver-entry dispatch ACK telemetry and stable persistence across cron, sessions_spawn, channel inbound, gateway-agent, CLI-agent, and ACP downstream dispatch surfaces.

## Best Possible Solution

Define a native dispatch landing telemetry contract that deliberately extends the existing task/lifecycle infrastructure or creates a separate telemetry plane, then add storage, versioning, plugin access, docs, and tests for all six surfaces.

## Reproduction Assessment

Partly. Source inspection on current main gives a high-confidence way to reproduce the missing native ACK/ledger contract, but the reported production false-confidence incident cannot be replayed from the provided local deployment data.

## Solution Assessment

Unclear. The proposed ACK kinds fit the observability gap, but the maintainable implementation needs architecture review around task registry reuse, schema/versioning, and plugin read or subscribe semantics.
