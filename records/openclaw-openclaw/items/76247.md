## Summary

Keep open. Current main and the latest v2026.6.6 release still have accepted/runId/task tracking and adjacent channel/subagent ledgers, but not the requested native receiver-entry ACK taxonomy, dispatch ACK ledger, track/hop identifiers, or plugin-readable landing telemetry across the six named dispatch surfaces.

## What This Changes

Request native receiver-entry dispatch ACK telemetry and durable/readable landing records across cron, sessions_spawn, channel inbound, gateway agent, CLI-agent, and ACP downstream dispatch paths.

## Best Possible Solution

Define one versioned receiver-entry telemetry contract in the approved shared SQLite/telemetry plane, align it with task/subagent/channel state, expose a bounded plugin read or subscribe API, and cover all six dispatch surfaces with docs and regression tests.

## Reproduction Assessment

Yes for the missing native contract: current-main source, schema, docs, and latest-release checks show no requested ACK kinds, track/hop IDs, dispatch ACK ledger, or plugin-readable receiver-entry surface. The reporter's production incidents were not locally replayed.

## Solution Assessment

Unclear. The ACK taxonomy addresses the observed gap, but maintainers need to choose whether the maintainable solution extends the existing task/subagent/channel ledgers or introduces a deliberate telemetry plane instead of a separate sidecar database/env override.
