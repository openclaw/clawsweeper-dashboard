## Summary

Keep open. Current main has accepted run acknowledgements, task/lifecycle tracking, cron execution phases, and plugin hooks, but it still lacks the requested native receiver-entry ACK taxonomy, track/hop identifiers, dispatch ACK persistence, and plugin-readable landing telemetry across the named dispatch surfaces.

## What This Changes

Request native dispatch landing ACK and receiver-entry telemetry across cron, sessions_spawn, channel_inbound, gateway_agent, cli_agent, and acp_downstream paths.

## Best Possible Solution

Define a versioned receiver-entry telemetry contract that fits the existing task/lifecycle system or a deliberate telemetry plane, with bounded non-fatal persistence, plugin read/subscribe guidance, docs, and coverage for the six dispatch surfaces.

## Reproduction Assessment

## Solution Assessment

Unclear. The proposed ACK taxonomy fits the observability gap, but maintainers need to choose whether the narrowest maintainable solution extends the task ledger/lifecycle hooks or introduces a separate dispatch telemetry plane.
