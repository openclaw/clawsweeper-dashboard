## Summary

Keep open. Current main has accepted-run and background-task tracking, but it still lacks the requested durable receiver-entry ACK contract, dispatch ACK taxonomy, track/hop IDs, and plugin-readable dispatch ledger.

## What This Changes

Request native dispatch landing telemetry across cron, sessions_spawn, channel inbound, gateway agent, CLI agent, and ACP downstream paths.

## Best Possible Solution

Define a native receiver-entry telemetry contract that deliberately extends the existing task/lifecycle infrastructure or adds a versioned dispatch ledger, then expose a plugin-facing read or subscribe story with docs and focused tests.

Yes for the missing native contract: current-main source and docs inspection shows no requested ACK kinds, dispatch ledger, or landing-ACK plugin surface. The external production false-confidence incident remains reporter-provided and is not independently replayable from the repo alone.

## Solution Assessment

Unclear. The proposed ACK taxonomy matches the observability gap, but the maintainable fix needs owner direction on task-registry integration versus a new SQLite ledger and on the plugin API boundary.
