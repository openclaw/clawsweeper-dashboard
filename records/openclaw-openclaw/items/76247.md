## Summary

Keep open. Current main has durable background task tracking, but it does not implement the requested receiver-entry ACK taxonomy, track/hop correlation, dispatch ACK ledger, or plugin-readable landing telemetry across the named dispatch paths.

## What This Changes

Add native receiver-entry dispatch ACK telemetry across cron, session spawn, channel inbound, gateway agent, CLI-agent, and ACP downstream dispatch surfaces.

## Best Possible Solution

Define a native telemetry contract that deliberately extends or composes with the existing task/lifecycle system, then add versioned persistence, bounded non-fatal receiver-entry writes, plugin-facing read or subscribe guidance, docs, and focused regression coverage across the six dispatch surfaces.

## Reproduction Assessment

Yes for the missing native contract: current-main source and docs inspection show no requested ACK kinds, dispatch ledger, track/hop IDs, or plugin-readable landing ACK surface. The reported production incident itself was not locally replayed.

## Solution Assessment

Unclear. The proposed ACK taxonomy addresses a real observability gap, but maintainers should decide whether the durable implementation belongs in the existing task registry or in a separate dispatch telemetry plane before code is written.
