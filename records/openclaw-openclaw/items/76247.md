## Summary

Keep open. Current main has durable task tracking and diagnostics events, but it still lacks the requested native receiver-entry ACK taxonomy, track/hop identifiers, dispatch ACK ledger, and plugin-readable persistent landing telemetry across the named dispatch surfaces.

## What This Changes

Request native receiver-entry dispatch acknowledgement telemetry across cron, session spawn, channel inbound, gateway agent, CLI agent, and ACP downstream dispatch paths.

## Best Possible Solution

Define a native receiver-entry telemetry contract that deliberately fits the existing task and diagnostics architecture, then add versioned persistence, bounded non-fatal writes, plugin-facing read/subscribe guidance, docs, and focused coverage across the six dispatch surfaces.

## Reproduction Assessment

Yes for the missing native contract: current-main source and docs show accepted/run/task/diagnostic events but no ack_kind taxonomy, track/hop IDs, dispatches.db ledger, or receiver-entry ACK persistence. The production incident itself remains externally reported rather than locally replayed.

## Solution Assessment

Unclear. The requested ACK taxonomy is directionally aligned with the observability gap, but maintainers need to decide whether the narrowest durable design extends tasks/diagnostics or introduces a separate dispatch telemetry ledger.
