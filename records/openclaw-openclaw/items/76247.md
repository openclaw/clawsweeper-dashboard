## Summary

Keep open. Current main has coarse task and lifecycle tracking, but it still lacks the requested durable receiver-entry ACK contract, track/hop IDs, dispatch ACK taxonomy, and plugin-readable dispatch ledger.

## What This Changes

Request native receiver-entry dispatch ACK telemetry and stable persistence across cron, session spawn, channel inbound, gateway agent, CLI agent, and ACP downstream dispatch paths.

## Best Possible Solution

Define a native receiver-entry telemetry contract that reuses or deliberately extends the existing task/lifecycle infrastructure, then add schema, docs, tests, and a plugin-facing read or subscribe story.

## Reproduction Assessment

Yes for the missing native contract: source inspection on current main shows no requested ACK kinds, dispatch ledger, or plugin-readable landing ACK surface. The production false-confidence incident remains externally reported and cannot be independently replayed from the checkout alone.

## Solution Assessment

Unclear. The proposed ACK taxonomy fits the reported observability gap, but the best implementation needs maintainer design around extending `tasks/runs.sqlite` versus adding a new dispatch ledger.
