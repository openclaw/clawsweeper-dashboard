## Summary

Keep open. Current main has generic task/lifecycle tracking for cron, subagents, ACP, gateway, and CLI agent runs, but it does not implement the requested receiver-entry ACK kinds, stable dispatch ledger, or plugin-author watcher/read contract.

Add native dispatch ACK and receiver-entry telemetry across cron, sessions_spawn, channel inbound, gateway agent, CLI agent, and ACP downstream dispatch paths.

## Best Possible Solution

Define a native receiver-entry telemetry contract that reuses or deliberately extends the existing task and agent lifecycle infrastructure, then add schema, docs, tests, and a plugin-facing read or subscribe story.
