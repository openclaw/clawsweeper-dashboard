## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep bootstrap auth on the node handoff path:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:2652-2655`
  - body: The server only treats setup-code bootstrap as the silent mobile handoff when the connection is a `role == "node"` node client, and the current contract says mobile starts the operator loop after hello-ok returns the bounded operator token. Returning the bootstrap token here starts a separate operator bootstrap connection that cannot satisfy that node-only path, so QR onboarding can race into a manual pairing or invalid-bootstrap failure instead of using the issued operator device token.
  - confidence: 0.9
- **[P1] Deduplicate optimistic chat turns without exact timestamps:** `apps/android/app/src/main/java/ai/openclaw/app/chat/ChatController.kt:631-634`
  - body: `mergeOptimisticMessages` decides whether an optimistic turn is already in history by calling `messageIdentityKey`, but that key includes the exact timestamp. Android creates the optimistic timestamp locally while gateway history persists the user turn with server `Date.now()`, so a normal successful send can reload history and keep both the remote user turn and the optimistic copy; use a run/idempotency-aware marker or a timestamp-tolerant content match and clear the optimistic entry once history catches up.
  - confidence: 0.82
