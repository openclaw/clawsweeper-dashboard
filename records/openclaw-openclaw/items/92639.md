## Summary

Keep open: the PR now closes transient managers and supplies real qmd-backed proof, but it still routes every shared `memory_search` call through transient CLI mode, which changes QMD's current lazy full-manager lifecycle for non-CLI callers.

## What This Changes

The branch changes memory-core `memory_search` to acquire and close a `purpose: "cli"` manager, including the closed-handle retry path, and adds focused regression tests and mock support.

## Best Possible Solution

Thread transient manager mode only to the reported one-shot CLI-local path, keep default manager acquisition for ordinary plugin `memory_search` calls, and keep the new close-safe transient lifecycle coverage.

## Reproduction Assessment

Yes. Source inspection shows the PR routes every shared `memory_search` acquisition through `purpose: "cli"`, while QMD CLI mode skips full-manager lifecycle work; the contributor also supplied a real qmd-backed tool-path proof for the intended exit behavior.

## Solution Assessment

No. The transient manager direction is useful for the reported one-shot CLI-local hang, but the best fix should preserve default QMD lifecycle behavior for non-CLI tool callers instead of changing the shared tool globally.
