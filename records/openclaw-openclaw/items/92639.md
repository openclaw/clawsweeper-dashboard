## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Close the transient QMD manager:** `extensions/memory-core/src/tools.ts:407-412`
  - body: This call now asks for `purpose: "cli"`, and `getMemorySearchManager` deliberately bypasses the cached full QMD manager for CLI requests and returns a transient `QmdMemoryManager`. Unlike the CLI command helper, `memory_search` never closes the manager after search/status work; QMD status/read paths can open the read-only SQLite handle that only `QmdMemoryManager.close()` releases. In a long-running gateway process, every QMD `memory_search` call can leak another manager/DB handle, so wrap the transient manager in a `finally` close or keep the cached full-manager path outside true one-shot CLI use.
  - confidence: 0.87
