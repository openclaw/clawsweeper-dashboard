## Summary

Keep open: current `main` and the latest release still let standalone `openclaw plugins update --all` feed recorded npm specs into the updater without the official-plugin sync/channel context needed to avoid stale exact official targets. The open fix candidate is https://github.com/openclaw/openclaw/pull/94084, but it is not merged and it does not close this issue yet.

## What This Changes

The issue reports that `openclaw plugins update --all` can downgrade or leave official npm plugins behind by reusing stored install specs instead of resolving the intended current package line.

## Best Possible Solution

Review and land a narrow bulk-official sync fix such as https://github.com/openclaw/openclaw/pull/94084, while preserving documented targeted exact-pin behavior and keeping third-party/latest update detection tracked separately.

## Reproduction Assessment

Yes. Source inspection is enough: `--all` selects all install records, the CLI omits official-sync/channel arguments, and the updater only leaves the recorded exact npm spec when that flag is absent.

## Solution Assessment

No for current main. The narrow maintainable direction is to make bulk maintenance sync trusted official records through the existing official catalog path while leaving targeted exact-pin updates unchanged.
