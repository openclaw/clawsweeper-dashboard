## Summary

Keep open: the central bounded-output fix is valid, but the patch has a source-reproducible resumeOutput JSONL regression under the new 1 MiB parse cap, and the required changelog entry is still missing.

## What This Changes

The branch disables supervisor stdout/stderr capture for prepared CLI runs, adds bounded diagnostic and parse buffers, extracts streamed JSONL final output, and adds CLI runner regression coverage.

## Best Possible Solution

Keep the PR open, derive streaming/parser setup from the effective output mode used for this turn, add a focused regression test and changelog entry, then merge after CI remains green.

## Reproduction Assessment

Yes. Source inspection of current main shows supervisor capture defaults on and the prepared runner does not override it; the PR-introduced resumeOutput JSONL edge case is also source-reproducible from the changed parser and parse-cap flow.

## Solution Assessment

No. The main direction is maintainable, but the patch should create/use the streaming parser from the effective outputMode before spawning so resumeOutput:"jsonl" is parsed incrementally too.
