## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wire the advertised timeout into the gateway call:** `src/cli/program/register.status-health-sessions.ts:331`
  - body: This command declares `--timeout <ms>` with a 180000ms default, but the action never passes that option into `sessionsCompactCommand`, and the new command calls `callGateway` without `timeoutMs`. `callGateway` therefore uses its 10s default, so real LLM-backed compactions can time out even when the user passes the documented timeout flag.
  - confidence: 0.91
- **[P2] Remove or implement the compact store option:** `src/cli/program/register.status-health-sessions.ts:329`
  - body: `--store` is accepted here and parent `sessions --store` is read, but the `sessions.compact` protocol only accepts `key`, `agentId`, and `maxLines`, and the new command drops the store path entirely. Users can point the command at an alternate store and still compact whatever the live Gateway resolves from its configured store, so this should be removed/rejected or backed by an explicit protocol change and tests.
  - confidence: 0.9
- **[P2] Make CLI compact rejection script-visible:** `src/auto-reply/reply/commands-compact.ts:211`
  - body: Returning a normal command reply here gives the user text, but the Gateway agent path still treats replies as a successful response and `agentViaGatewayCommand` just logs payloads and returns. That leaves `openclaw agent --message '/compact ...'` with a successful process result, while the linked issue specifically called out exit 0 as the false-positive failure mode; reject this earlier or return a Gateway error so scripts can detect it.
  - confidence: 0.82
- **[P3] Document the new sessions compact command:** `src/cli/program/register.status-health-sessions.ts:325`
  - body: The linked issue is partly a discoverability/docs gap, and this PR adds a new public CLI command without updating `docs/cli/sessions.md`. Please add the compact command syntax, supported parameters, example output, and the relationship to the existing `sessions.compact` RPC so users do not have to inspect source or raw Gateway calls.
  - confidence: 0.86
