## Summary

Keep open. Current main still routes GitHub Copilot Gemini built-in, generated-default, and dynamic model definitions to `openai-responses`, matching the reporter's direct Copilot API evidence that Gemini rejects `/responses` but works via chat completions; the IDE-header side is now covered by the shared OpenAI client header path.

The issue asks OpenClaw to route GitHub Copilot Gemini/chat-completions-only models through `openai-completions` and ensure Copilot IDE headers are present on that path.
