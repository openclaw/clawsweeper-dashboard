Keep open. Current `origin/main` still routes Copilot Gemini built-in, generated-default, and dynamic model definitions to OpenAI Responses, while the reporter supplied direct Copilot API evidence that Gemini works through chat completions instead; the chat-completions IDE-header path appears already covered by the shared OpenAI client header builder.

The issue asks for GitHub Copilot Gemini models to use chat completions instead of Responses and for Copilot IDE headers to apply on the chat-completions transport.

## Best Possible Solution

Land a provider-owned Copilot transport fix that routes verified Gemini ids through `openai-completions`, preserves shared IDE-header injection, updates docs and changelog, and leaves repeated-fallback spend warnings for separate product work.
