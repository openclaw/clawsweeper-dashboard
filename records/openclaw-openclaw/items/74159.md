The issue asks OpenClaw to route GitHub Copilot Gemini models to chat completions and ensure Copilot IDE auth headers are present on that transport path to avoid failing over to paid fallback models.

## Best Possible Solution

Land a focused GitHub Copilot provider fix that routes known and forward-compatible Gemini model ids to `openai-completions`, relies on the shared OpenAI transport for Copilot IDE headers, updates provider docs and changelog, and covers bundled, default-generated, and dynamic Gemini rows with regression tests.
