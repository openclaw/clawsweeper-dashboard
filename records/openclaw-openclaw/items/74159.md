## Summary

Keep open. Current main still routes GitHub Copilot Gemini model IDs through `openai-responses` for both bundled catalog/default rows and dynamic forward-compatible IDs, which matches the reporter's unsupported Responses endpoint failure. The shared OpenAI transport now injects Copilot IDE headers for `github-copilot` OpenAI-family streams, so the remaining actionable bug is the provider-owned Gemini transport selection plus docs/changelog and regression coverage.

## What This Changes

The issue requests routing GitHub Copilot Gemini models to the OpenAI chat-completions transport with Copilot IDE headers, rather than the unsupported Responses endpoint, and suggests a separate warning for repeated primary-to-fallback failures.

## Best Possible Solution

Land a focused GitHub Copilot provider fix that routes known and forward-compatible Gemini model IDs to `openai-completions`, relies on the shared OpenAI transport for Copilot IDE headers, adds regression coverage for bundled/default/dynamic Gemini rows, updates the GitHub Copilot provider docs and changelog, and leaves repeated-fallback cost warnings as a separate follow-up.
