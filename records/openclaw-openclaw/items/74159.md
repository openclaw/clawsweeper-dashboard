Keep open. Current main has fixed the shared Copilot IDE-header path for OpenAI-family transports, but the GitHub Copilot provider still resolves bundled and dynamic Gemini model IDs to `openai-responses`, matching the reporter's unsupported Responses API failure and leaving a narrow provider-owned bug to fix.

The issue requests routing GitHub Copilot Gemini models through chat completions with Copilot IDE headers instead of the unsupported Responses endpoint.

## Best Possible Solution

Land a focused GitHub Copilot provider fix that routes known and forward-compatible Gemini model IDs to `openai-completions`, updates the static manifest catalog and docs to match, adds regression coverage for bundled and dynamic Gemini IDs plus Copilot completions headers, and leaves the broader repeated-fallback cost warning for a separate follow-up.
