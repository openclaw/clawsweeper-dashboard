## Summary

Keep open because current main still source-reproduces the inserted-space rendering for punctuation-wrapped Home Assistant-style entity IDs, and PR #39565 remains the open closing implementation candidate.

## What This Changes

Preserve long dotted entity IDs in assistant/chat output so rendering does not insert copy-breaking spaces inside identifiers.

## Best Possible Solution

Keep the issue open until PR #39565 or an equivalent focused formatter/test fix lands on main.

## Reproduction Assessment

Yes. Current main is source-reproducible: the formatter classifies backtick-wrapped entity IDs as non-copy-sensitive and then inserts a literal space while chunking the long token.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust shared copy-sensitive token classification and add formatter regression coverage, rather than adding channel-specific workarounds.
