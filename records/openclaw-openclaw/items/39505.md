## Summary

Keep open because current main still source-reproduces the inserted-space rendering for punctuation-wrapped Home Assistant-style entity IDs, and PR #39565 remains open as the linked implementation candidate.

## What This Changes

The issue asks OpenClaw to preserve long Home Assistant-style entity IDs exactly in assistant chat output instead of rendering them with inserted mid-token spaces.

## Best Possible Solution

Keep this issue open until PR #39565 or an equivalent focused formatter/test fix lands on main.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust shared copy-sensitive token classification and add formatter regression coverage, not to add channel-specific workarounds.
