## Summary

Keep open. Current main still splits punctuation-wrapped Home Assistant-style entity IDs through the TUI render sanitizer, and PR #39565 is an open unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests preserving long entity IDs in assistant chat output so copy/paste does not introduce spaces inside identifiers.

## Best Possible Solution

Review and land PR #39565 or an equivalent focused fix that classifies punctuation-stripped file-like entity IDs as copy-sensitive, adds regression coverage in `src/tui/tui-formatters.test.ts`, and keeps generic long-token chunking behavior intact.

## Reproduction Assessment

Yes. The current formatter logic can be reproduced deterministically with a read-only probe: backtick-wrapped entity IDs are split with literal spaces, while bare IDs remain intact.

## Solution Assessment

Yes. The narrowest maintainable fix is in the shared formatter classification and its tests, with adjacent note wrapping kept consistent, rather than adding channel-specific workarounds.
