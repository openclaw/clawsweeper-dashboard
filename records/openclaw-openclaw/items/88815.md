## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve selected-agent scope in echo hooks:** `src/infra/outbound/echo-hook.ts:36-38`
  - body: For selected-agent global sessions, current gateway paths carry `SessionKey: "global"` and the selected agent separately. This hook derives the store from `event.sessionKey` alone, so a work-agent global turn will read the default agent's global entry and can echo to the wrong targets or miss the configured targets. Carry `agentId` into the internal message hook context or resolve echo delivery from a caller path that still has selected-agent scope.
  - confidence: 0.9
- **[P2] Include agentId in echo session change events:** `src/gateway/server-methods/sessions.ts:2350-2353`
  - body: The echo handler resolves the selected global store with `p.agentId`, but the `sessions.changed` payload drops that agent id. `emitSessionsChanged` only loads the selected global row when `agentId` is present, so HUD/UI subscribers can refresh the default global session after adding or removing echo targets on another agent.
  - confidence: 0.84
