## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Release handled marks when live rendering fails:** `src/infra/outbound/echo-streaming.ts:230`
  - body: `markHandled` runs before the target renderer has actually delivered anything. If the Telegram draft stream stops after a send/edit failure, the later post-hoc mirror consumes this mark and skips the target, so the pinned channel can receive no assistant response instead of falling back to final-text delivery.
  - confidence: 0.88
- **[P2] Preserve Telegram thread scope for echo renders:** `extensions/telegram/src/echo-renderer-register.ts:52-54`
  - body: This converts every numeric echo `threadId` into a forum thread. Existing Telegram helpers distinguish DM topics, forum topics, and non-forum sends; using `scope: "forum"` can omit DM topic id 1 or otherwise send the native streaming echo to the wrong surface.
  - confidence: 0.82
- **[P1] Bind echo renderer registration to channel ownership:** `src/plugin-sdk/channel-echo.ts:17`
  - body: The new public SDK subpath exports a raw registry function that accepts any channel id, so a plugin can register `telegram` before the Telegram plugin and become the first-wins renderer. Because the fan-out then suppresses fallback for handled targets, this can misroute or drop pinned replies instead of enforcing the documented owner-only contract.
  - confidence: 0.86
