## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Treat mirror DM denial as revocation:** `extensions/telegram/src/bot-message.ts:327-341`
  - body: When the synthetic mirror is a Telegram DM, `enforceTelegramDmAccess` can reject it for disabled DM policy, pairing, or allowlist before `onMirrorAdmissionBlocked` is called. That leaves `revoked` false, so this branch throws and `launchMirrorDispatch` unmarks the target, allowing the raw post-hoc echo path to deliver content that the destination admission path denied.
  - confidence: 0.9
- **[P1] Keep mirror dispatcher registration owner-scoped:** `src/plugin-sdk/channel-outbound.ts:55-59`
  - body: These new public SDK exports let any plugin register or unregister a dispatcher for any `(channel, accountId)`, and the registry replaces the current dispatcher on re-registration. A third-party or wrong channel plugin can intercept or suppress pinned session content for another channel/account, so the registration needs an owner-scoped token/host API or should stay internal until that contract is safe.
  - confidence: 0.86
