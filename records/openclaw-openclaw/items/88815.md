## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Treat Telegram DM denial as mirror revocation:** `extensions/telegram/src/bot-message.ts:327-341`
  - body: `dispatchMirror` only keeps the handled mark when `onMirrorAdmissionBlocked` fires. DM policy, pairing, and allowlist denial happen through `enforceTelegramDmAccess`, which returns `null` without setting that marker; this branch then throws, `launchMirrorDispatch` unmarks the target, and the raw echo path can still send session content to the denied DM because its admission predicate only checks group/topic enablement.
  - confidence: 0.9
- **[P1] Keep mirror registration owner-scoped before exporting it:** `src/plugin-sdk/channel-outbound.ts:55-59`
  - body: This public SDK export exposes a freeform global last-wins registry keyed only by caller-provided channel/account strings. Any plugin using the public API can replace or unregister another channel's mirror/admission handler and receive or suppress mirrored session content, so the seam should be internal or handed out through an owner-scoped registrar before becoming a plugin contract.
  - confidence: 0.84
