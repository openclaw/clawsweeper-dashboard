Full review comments:

- **[P1] Register the gateway method descriptor:** `src/gateway/server-methods.ts:470`
  - body: Adding `sessions.echo` to `coreGatewayHandlers` without also adding it to `CORE_GATEWAY_METHOD_SPECS` breaks the gateway registry path: current main throws for any core handler missing a descriptor, so ordinary gateway requests can fail with `gateway method handler is missing a descriptor: sessions.echo`.   - confidence: 0.95
- **[P2] Echo inbound user turns too:** `src/infra/outbound/echo-hook.ts:68`
  - body: This hook only runs after outbound `message:sent` delivery and then hard-codes `role: "assistant"`. The PR claims pinned channels receive every turn and exposes `echoUser`, but no path calls the echo code for inbound user messages, so pinned transports still get fragmented logs.
  - confidence: 0.9
- **[P2] Include account and thread in target identity:** `src/commands/sessions-echo.ts:65-66`
  - body: The add path treats any existing target with the same `channel` and `to` as a duplicate, ignoring `accountId` and `threadId`. That prevents pinning distinct Telegram topics/accounts that share a chat id and makes remove semantics ambiguous; target identity should match the same fields used by delivery and origin dedupe.
  - confidence: 0.86
