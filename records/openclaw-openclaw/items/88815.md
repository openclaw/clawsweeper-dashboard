## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind registrar operations to the authenticated channel:** `src/infra/outbound/channel-outbound-registrar.ts:41-42`
  - body: `createChannelOutboundRegistrar(owner)` binds the owner but still lets the plugin choose the `channel` key passed into the registry. A channel can therefore call its bound registrar with another channel id, pre-register that `(channel, account)` entry, and the real channel will be refused later as a different owner; the registrar should derive or enforce `channel === owner` instead of forwarding caller input.
  - confidence: 0.94
- **[P1] Parse Telegram topic targets before mirroring:** `extensions/telegram/src/bot-message.ts:265-269`
  - body: Forum sessions use targets like `telegram:-100...:topic:99`, but this mirror parser only strips the prefix before numeric chat detection. That turns a forum target into a string/private chat, so the native mirror can fail and the raw echo/admission path can miss group/topic revocation; parse the topic suffix into `chatId` and `threadId` before dispatch/admission.
  - confidence: 0.9
- **[P2] Keep mirror suppression for the whole origin run:** `src/infra/outbound/mirror-dispatch.ts:116-120`
  - body: `consumeStreamingEchoHandled` deletes the handled mark the first time a `message:sent` hook checks it, but a verbose/tool/final run can produce multiple sent hooks while the native mirror has already rendered the full stream. Later hooks will fall through to raw echo and duplicate the mirrored target; keep suppression scoped through the origin run or per emitted payload batch.
  - confidence: 0.86
