## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Normalize targets before self-excluding echoes:** `src/infra/outbound/echo.ts:42-49`
  - body: `resolveEchoTargets` only compares `target.to` and `originTo` as raw strings. Telegram accepts both raw chat IDs and internal targets like `telegram:999`, and inbound hook origins use the prefixed form, so an echo pin added as `--to 999` will not be recognized as the same origin and can echo back into its own chat/topic. Normalize channel target identities before self-exclusion and cover the raw-vs-prefixed same-topic case.
  - confidence: 0.89
- **[P3] Return a failing status for JSON add-at-limit:** `src/commands/sessions-echo.ts:106-114`
  - body: When `--json` is used and the echo target cap is reached, the command prints `{ ok: false }` but never exits nonzero because the `runtime.exit(1)` path only runs in the non-JSON branch. Scripts using the JSON CLI can miss that the add failed; make the at-limit case exit consistently after writing the JSON result.
  - confidence: 0.78
