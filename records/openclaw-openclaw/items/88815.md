Full review comments:

- **[P1] Gate prompt echo on committed channel turns:** `src/infra/outbound/echo-hook.ts:124-135`
  - body: `message:received` fires before fast-abort/native-command/reply handling, and this hook mirrors every channel input immediately when `echoTargets` exist. A no-reply command or blocked turn can send the user's text to pinned targets without a committed assistant turn. Move channel-originated user echo to the committed dispatch/reply path, like the chat.send acceptance path does.
  - confidence: 0.84
- **[P2] Use native assistant echo options from chat.send fallbacks:** `src/gateway/server-methods/chat.ts:3827-3839`
  - body: The chat.send assistant echo sites still call `fireEchoDeliveries` without `{ prefixed: false }` or the streaming-handled target filter. Stream-off/no-renderer and source-reply paths can therefore still render assistant replies as tagged `[echo]` copies, and they do not share the duplicate-final guard used by the `message:sent` hook.
  - confidence: 0.86
- **[P2] Document the new channel-echo SDK subpath:** `src/plugin-sdk/channel-echo.ts:8-13`
