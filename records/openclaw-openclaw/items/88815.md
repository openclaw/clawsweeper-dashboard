## Summary

Keep this PR open, but it is not merge-ready: the native-dispatch mirror design is promising, yet source review still shows a Telegram denial path that can fall through to raw echo delivery, and the new public dispatcher/persisted contract needs explicit maintainer acceptance.

## What This Changes

## Best Possible Solution

Keep the native-dispatch mirror design, but merge only after all Telegram denial paths fail closed with no raw fallback, dispatcher registration is owner-scoped or internal, maintainers accept the persisted/public contract lifecycle, and denied-destination proof is supplied.

## Reproduction Assessment

Yes for the review blocker from source: a Telegram DM mirror denied by DM policy, pairing, or allowlist returns null without setting the revocation marker, and the launcher then unmarks the target for raw post-hoc echo fallback. I did not run a live Telegram denial path in this read-only review.

## Solution Assessment

No, not yet. Reusing each channel's native dispatch is the right layer, but the merge shape still needs fail-closed DM denial and an owner-scoped or internal dispatcher registration contract before it is the safest maintainable solution.
