## Summary

Keep this PR open, but it still needs changes before merge: the channel-originated user echo path is wired to the pre-admission `message:received` hook, so rejected or fast-abort channel input can still be copied to pinned echo targets.

## What This Changes

The PR adds persistent session echo targets with CLI add/remove/list commands, a `sessions.echo` Gateway method/protocol schema, startup echo-hook registration, outbound fanout, and webchat/channel mirroring.

## Best Possible Solution

Move all user echo delivery to a committed accepted-turn boundary shared by channel and webchat paths, then refresh current-head Telegram proof and get maintainer sign-off on the durable cross-channel recipient API.

## Solution Assessment

No. The webchat side now uses an accepted-dispatch placement, but the channel side still uses a broader pre-admission hook; the best fix is to enforce the same accepted-turn invariant on both surfaces.
