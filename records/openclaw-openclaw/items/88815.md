## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw user echo out of mirror-capable targets:** `src/infra/outbound/echo.ts:158-164`
  - body: `fireEchoDeliveries` sends every target through raw outbound delivery, and the PR calls it directly for user echoes from `chat.send` and `message:received`. For Telegram pins, that bypasses the dispatcher admission path used for group/topic/DM revocation, so a destination disabled after `/pin on` can still receive the user's prompt even if assistant delivery later fails closed. Route user echoes through the same native mirror authority or suppress mirror-capable targets unless their admission path succeeds.
  - confidence: 0.9
- **[P1] Treat mirror DM denial as revocation:** `extensions/telegram/src/bot-message.ts:327-341`
  - body: A Telegram DM mirror still runs DM access with the synthetic sender id `0`. If the pinned DM is disabled or that synthetic sender fails pairing/allowlist, `buildTelegramMessageContext` returns `null` without setting `revoked`, so this branch throws and `launchMirrorDispatch` unmarks the target, allowing raw post-hoc echo to deliver to a destination the native path denied. Mark mirror DM destination denial as revocation, or bypass sender-only DM checks while preserving disabled/requireTopic denial, so no fallback delivery occurs.
  - confidence: 0.91
- **[P1] Keep mirror dispatcher registration owner-scoped:** `src/plugin-sdk/channel-outbound.ts:55-59`
  - body: This public SDK export gives any plugin access to the global last-wins mirror registry, and unregistering also takes only channel/account strings. A plugin can claim or remove another channel/account dispatcher and intercept or suppress pinned session content. Keep the seam host-owned/internal, or issue an owner-scoped registrar tied to the channel runtime instead of exporting raw global mutation functions.
  - confidence: 0.88
