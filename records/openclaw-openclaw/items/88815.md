## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route user echoes through destination admission:** `src/gateway/server-methods/chat.ts:3430-3442`
  - body: This sends every accepted `chat.send` prompt to `echoTargets` via raw `fireEchoDeliveries`, whose delivery helper goes straight to `deliverOutboundPayloadsInternal` without the mirror dispatcher or Telegram admission checks. A pinned Telegram destination that was later disabled or allowlist-revoked can still receive the user's prompt before assistant mirroring fails closed, leaking session content.
  - confidence: 0.93
- **[P1] Treat Telegram DM denial as revocation:** `extensions/telegram/src/bot-message.ts:327-341`
  - body: When `buildTelegramMessageContext` returns null for DM pairing or allowlist denial, `revoked` is still false, so this branch throws and lets `launchMirrorDispatch` unmark the target for raw post-hoc delivery. Mirror DM denial needs to keep the handled mark like group/topic revocation.
  - confidence: 0.9
- **[P1] Keep mirror dispatcher registration owner-scoped:** `src/plugin-sdk/channel-outbound.ts:55-59`
  - body: These public exports expose a global last-wins mirror dispatcher registry by channel/account string. An unrelated plugin can register or unregister another channel's dispatcher and intercept or suppress pinned session content, so the seam should stay host-internal or be owner-scoped before becoming public SDK.
  - confidence: 0.86
