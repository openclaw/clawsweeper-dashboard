## Risks / Open Questions

- The public mirror-dispatcher export is not owner-scoped: a plugin can register or unregister an arbitrary channel/account dispatcher and potentially intercept or suppress pinned session mirror delivery.
- The PR creates new compatibility contracts across persisted `echoTargets`, `sessions.echo`, CLI `sessions echo`, `/pin`, Swift protocol models, and plugin SDK exports; maintainers need to accept and document that lifecycle before merge.
- Destination revocation is security-sensitive because failed revocation can leak session content to a disabled Telegram group/topic; unit tests cover it, but live Telegram proof for the no-native/no-fallback path is still missing.
