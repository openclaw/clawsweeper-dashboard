Full review comments:

- **[P1] Normalize targets before self-excluding echoes:** `src/infra/outbound/echo.ts:42`
  - body: Telegram inbound origins are canonicalized as `telegram:<chat>` and forum topics as `telegram:<chat>:topic:<id>`, but echo targets can be persisted from CLI/Gateway as raw chat IDs. Because this comparison uses exact `target.to === originTo`, a Telegram-origin message pinned to the same raw chat/topic is treated as a different target and gets echoed back into its own conversation instead of being self-excluded; normalize both sides through the channel target contract and add the raw-vs-prefixed regression test.
  - confidence: 0.91
- **[P3] Return a failing status for JSON add-at-limit:** `src/commands/sessions-echo.ts:106-111`
  - body: When `--json` is used and the target cap is already reached, this branch writes `{ ok: false }` but never calls `runtime.exit(1)`, unlike the non-JSON path. Automation that relies on the command status will treat the failed add as successful, so the JSON at-limit path should exit nonzero after emitting the structured error result.
  - confidence: 0.87
