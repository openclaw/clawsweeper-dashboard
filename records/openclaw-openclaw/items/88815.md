## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bind outbound registration to the host channel:** `src/infra/outbound/channel-outbound-registrar.ts:41-46`
  - body: `createChannelOutboundRegistrar(owner)` still lets the plugin choose the `channel` key passed into the global registries. A different channel can pre-register `telegram/default` before Telegram starts, after which the owner check blocks Telegram or routes mirrors/admission through the wrong handler. Bind the channel key in the registrar, or reject `channel !== owner`, and update the SDK shape/docs accordingly.
  - confidence: 0.95
- **[P1] Parse Telegram topic targets before mirroring:** `extensions/telegram/src/bot-message.ts:265-269`
  - body: Forum pins are stored as targets like `telegram:-100123:topic:99` with `threadId` separately, but this parser only strips prefixes before numeric parsing. That leaves `-100123:topic:99` as a non-numeric private/unknown chat, so native mirror dispatch and revocation admission miss the real group/topic and can fall back to raw echo. Reuse `parseTelegramTarget` or otherwise split the topic suffix here and in the admission predicate.
  - confidence: 0.91
- **[P2] Do not consume mirror suppression after one sent hook:** `src/infra/outbound/mirror-dispatch.ts:116-122`
  - body: The handled mark is deleted the first time `consumeStreamingEchoHandled` sees the target, but outbound delivery emits `message:sent` for every delivered payload. Verbose/tool-summary runs can therefore skip raw echo for the first message and then raw-echo later payloads to the same pinned target after native mirror already rendered them. Keep suppression scoped to the origin run until its post-hoc delivery window is complete, or key it by run/payload explicitly.
  - confidence: 0.86
