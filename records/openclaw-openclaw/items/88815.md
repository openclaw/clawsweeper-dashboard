## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate agent scope before echo mutations:** `src/gateway/server-methods/sessions.ts:2278-2282`
  - body: Sibling session mutators reject `agentId` mismatches before writing, but this new handler passes `p.agentId` straight into target resolution. For a keyed session such as `agent:main:main`, the resolver ignores a mismatched `agentId: "work"` and still mutates the main session, so callers can pin echo targets on the wrong agent instead of receiving the expected validation error.
  - confidence: 0.91
- **[P1] Preserve selected-agent scope in echo hooks:** `src/infra/outbound/echo-hook.ts:30-33`
  - body: For selected-global sessions, current inbound/outbound paths can emit the literal `global` session key while the active agent is carried separately. This hook derives the store from `event.sessionKey` alone, so a work-agent global turn reads the default global store and can miss or misroute echo targets. Carry the selected agent id in the hook context or resolve echo delivery from a caller path that still has that scope.
  - confidence: 0.9
- **[P2] Include agentId in echo change broadcasts:** `src/gateway/server-methods/sessions.ts:2368-2370`
  - body: The echo handler resolves selected global stores using `p.agentId`, but its `sessions.changed` payload drops that id. `emitSessionsChanged` only loads the selected global row when `agentId` is present, so UI/HUD subscribers can refresh the default global session after adding or removing echo targets on another agent.
  - confidence: 0.85
