## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep streaming echo targets marked until post-hoc echo skips them:** `src/infra/outbound/echo-streaming.ts:180`
  - body: `unmarkHandled()` runs when the mirror resolver settles on lifecycle end. The normal runner emits that lifecycle end before the final outbound delivery later fires `message:sent`, so the fallback filter can see the target as unhandled and send a second final message to the same pinned channel. Keep the handled marker alive through the post-hoc assistant echo decision or key it to a run-level completion signal that happens after delivery.
  - confidence: 0.87
- **[P1] Gate prompt echo on committed reply turns:** `src/infra/outbound/echo-hook.ts:124`
  - body: This echoes every `message:received` event with content, but the channel dispatch path emits that hook before fast-abort and slash/native command handling. A no-response command or rejected turn can therefore be mirrored to pinned channels with no paired assistant reply, leaking uncommitted prompt text. Move this to the committed dispatch path or add hook metadata that proves the turn will produce a session reply before echoing.
  - confidence: 0.9
