## Summary

Keep open: current main still bypasses command secret resolution for CLI web search, the docs say plugin-scoped webSearch SecretRefs are supported, and an open linked PR with closing syntax is already the implementation path.

## What This Changes

The issue requests CLI web search to resolve selected-provider plugin-scoped `webSearch.apiKey` SecretRefs from the active runtime snapshot before provider execution.

## Best Possible Solution

Review and land one narrow canonical PR that routes `infer web search` and the parallel fetch path through command-specific SecretRef resolution with regression coverage and redacted real CLI proof, then close this issue.

## Reproduction Assessment

Yes. The reporter provided a concrete command, config, and unresolved SecretRef output, and current main still passes raw config from `runWebSearchCommand` into `runWebSearch` without command-time secret resolution.

## Solution Assessment

Yes in direction, but not complete until an open fixing PR lands. The maintainable solution is command-specific SecretRef materialization before local web search/fetch provider execution, without resolving unrelated plugin secrets.
