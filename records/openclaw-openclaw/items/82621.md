## Summary

Keep open: current main still has the reported CLI SecretRef gap, and the active fix path is the open PR that specifically targets CLI web-search SecretRef resolution rather than a shipped main change.

## What This Changes

The issue reports that `openclaw infer web search` reads plugin web-search API key SecretRefs before resolving them from the active gateway runtime snapshot.

## Best Possible Solution

Land a provider-scoped CLI command-secret resolution fix, preferably through the active reviewed PR, then close this issue once the fix is on main.

## Reproduction Assessment

Yes. The issue provides an exact command and unresolved SecretRef error, and current main source shows the CLI web-search path still passes raw config to provider runtime without command SecretRef resolution.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve only the selected web-search provider's command SecretRefs before local CLI execution, reusing the existing command resolver rather than changing provider credential semantics.
