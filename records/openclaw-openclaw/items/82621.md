## What This Changes

Resolve plugin-scoped `webSearch.apiKey` SecretRefs for `openclaw infer web search` before provider config is read, while keeping non-selected provider refs inactive.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/82699; after a merged fix materializes CLI web search/fetch credentials through command SecretRef resolution, close this issue as implemented.

## Solution Assessment
