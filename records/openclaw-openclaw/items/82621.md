## Summary

Keep open. Current main still has a source-visible CLI gap where `openclaw infer web search` runs provider code with the runtime config before command SecretRef resolution, and the related open fix PR has not merged yet.

## What This Changes

The issue requests that CLI web search resolve selected plugin-scoped `webSearch.apiKey` SecretRefs from the active gateway snapshot before local provider execution.

## Best Possible Solution

Land a provider-scoped version of https://github.com/openclaw/openclaw/pull/82598, with regression coverage that proves `infer web search` resolves only the selected provider's active SecretRefs before execution.

## Reproduction Assessment

Yes. The source path is clear: current main passes raw config from `infer web search` into provider execution, and strict provider SecretRef normalization then throws; the related PR body also includes before-error output for the same CLI failure class.

## Solution Assessment

Yes, with scoping. The best fix is the existing PR direction, but target selection must be limited to the selected or auto-detected web-search provider so unrelated provider secrets remain inactive.
