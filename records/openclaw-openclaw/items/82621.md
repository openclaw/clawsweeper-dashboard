## Summary

Keep open. Current main still sends raw CLI config into the local web-search runtime without command-time SecretRef materialization, and this issue is now paired with open fix PRs, including a direct closing PR that should be reviewed and landed or superseded before the issue closes.

## What This Changes

Ensure `openclaw infer web search` resolves selected plugin-scoped `webSearch.apiKey` SecretRefs before running local web search.

## Best Possible Solution

Review and land one focused CLI fix that resolves only the selected web-search command SecretRef targets before provider execution, then close this issue and mark duplicate fix PRs as superseded.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main passes raw config from `runWebSearchCommand` into `runWebSearch`, while Tavily and SecretInput helpers throw when a configured API key remains a raw SecretRef; the reporter also supplied a concrete command and error output.

## Solution Assessment

Yes. The maintainable fix is to reuse the existing command SecretRef resolution path with narrowly scoped web-search targets before local web-search execution; the open fix PRs are implementation candidates, not a reason to close this issue before merge.
