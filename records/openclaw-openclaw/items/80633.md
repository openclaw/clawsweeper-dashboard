## Summary

Close: local/offline Whisper STT is already supported through the shipped `tools.media.audio` CLI path and auto-detected `whisper-cli`; a dedicated bundled provider id/model downloader would be optional packaging rather than remaining core functionality.

## What This Changes

Request to add a built-in `whisper.cpp` audio transcription provider selectable from `tools.media.audio.models` for offline local STT.

## Best Possible Solution

Keep the shipped `tools.media.audio` CLI/auto-detect path for local Whisper, and handle any richer whisper.cpp model-download wrapper as plugin/ClawHub packaging if maintainers want that UX.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source, docs, tests, and release-tag inspection show the local Whisper path is already available on current main and in `v2026.5.7`.

## Solution Assessment

Yes for the central user problem: the existing CLI model surface is the narrowest maintainable core solution for offline local STT. A dedicated `provider: "whisper-cpp"` downloader would be a product/plugin packaging decision, not required for transcription to work.
