## What This Changes

The PR threads `agents.list[].identity.name` into system prompt runtime metadata and renders it as `agent=...`, with the slot id retained as `agent_id=...` when different.

## Best Possible Solution

Land the narrow prompt plumbing that resolves `identity.name` through the canonical agent config resolver, displays it first, and keeps the stable slot id for debugging.

## Reproduction Assessment

## Solution Assessment

Yes. The latest diff uses the existing normalized agent resolver to read `identity.name`, renders it as the visible runtime agent label, and keeps `agent_id` for diagnostics, which is the narrow maintainable fix.
