## Summary

Keep open: this is an active focused bug-fix PR, current main still has the source-reproducible slot-id runtime label, and the latest PR diff addresses that path without introducing a clear regression.

## What This Changes

The PR threads `agents.list[].identity.name` into runtime prompt params, renders it as `agent=...`, preserves the slot id as `agent_id=...` when different, and adds focused prompt tests.

## Best Possible Solution

Land the narrow prompt plumbing that uses the configured identity name for the model-facing runtime label while retaining the stable slot id for diagnostics.

## Reproduction Assessment

Yes, at source level: current main builds the Runtime line from `agentId` and prompt params do not carry `identity.name`, so a config with slot id `main` and identity name `Runt` cannot display `Runt` there. I did not run a live chat path in this read-only review.

## Solution Assessment

Yes. The PR uses the existing normalized agent config resolver, carries the resolved identity name through runtime metadata, and keeps `agent_id` for debugging, which is the narrow maintainable fix.
