## Summary

Keep this PR open: current main still renders the Runtime agent label from the technical slot id, while the PR makes a narrow, source-backed fix with no blocking review findings and sufficient real behavior proof.

## What This Changes

The PR threads `agents.list[].identity.name` into runtime prompt params and renders it as the visible `agent=...` value while retaining a differing slot id as `agent_id=...`.

## Best Possible Solution

Land the narrow prompt plumbing that resolves the configured identity name through the existing agent config resolver, displays it first, and keeps the stable slot id available for debugging.

## Reproduction Assessment

Yes. At source level, current main only carries `agentId` into `buildRuntimeLine`, so a config with `agentId: "main"` and `identity.name: "Runt"` cannot render `Runt` in the Runtime line.

## Solution Assessment

Yes. The PR uses the existing normalized agent resolver, avoids a new prompt-prefix/suffix feature, and preserves the technical slot id separately for diagnostics.
