## Summary

Keep open: current main still renders the runtime agent label from the slot id only, while this PR narrowly threads the configured identity name through the prompt params and Runtime line with focused tests and sufficient live-output proof.

## What This Changes

The PR threads `agents.list[].identity.name` into runtime prompt params, renders it as `agent=...`, keeps the slot id as `agent_id=...` when different, and adds prompt-param/rendering tests.

## Best Possible Solution

Land the narrow prompt plumbing that uses the canonical agent config resolver for `identity.name`, displays it first, and keeps the stable slot id for debugging.

## Reproduction Assessment

Yes, at source level: current main builds the Runtime line from `agentId` only and `buildSystemPromptParams` does not carry `identity.name`, so a config with slot id `main` and identity name `Runt` cannot render `Runt` there.

## Solution Assessment

Yes. The PR reuses the existing normalized agent resolver, trims/coerces the configured identity name, and preserves the technical slot id as diagnostics without adding a competing config surface.
