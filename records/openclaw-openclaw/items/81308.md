## What This Changes

The branch threads `agents.list[].identity.name` into runtime prompt params and renders it as `agent=...`, keeping the technical slot id as `agent_id=...` when different.

## Best Possible Solution

Land the narrow prompt plumbing after maintainer review, keeping the configured identity name first and the stable slot id available for diagnostics.

## Reproduction Assessment

Yes, at source level: current main passes only `agentId` into runtime prompt params and renders that value as `agent=...`, so a config with id `main` and `identity.name` `Runt` cannot show `Runt` there. I did not run an interactive chat path in this read-only review.

## Solution Assessment

Yes. The PR reuses the existing normalized agent config resolver, trims and sanitizes the display fields, and preserves the slot id as `agent_id` only when it differs, which is narrower than adding a new prompt prefix/suffix config surface.
