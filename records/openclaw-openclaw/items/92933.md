## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid re-persisting CLI-owned transcript turns:** `src/auto-reply/reply/agent-runner.ts:1733`
  - body: Current main already asks the CLI runtime to persist normal assistant output and marks those payloads as `assistantTranscriptOwned`, which dispatch/follow-up delivery uses to suppress later mirrors. This new `runner === "cli"` branch calls `persistCliTurnTranscript` again before delivery, so a normal CLI auto-reply can append another user/assistant turn and bypass the existing room-event and suppress flags. Keep the new path to the missing embedded/gap-fill case, or guard it on the absence of runtime transcript ownership.
  - confidence: 0.91
- **[P1] Pass a concrete cwd to transcript persistence:** `src/auto-reply/reply/agent-runner.ts:1753`
  - body: `followupRun.run.cwd` is optional, but `persistCliTurnTranscript` requires `sessionCwd: string`; the current CI type error points at this exact argument. Use the effective runtime cwd or fall back to `followupRun.run.workspaceDir` before calling the transcript writer.
  - confidence: 0.96
- **[P2] Mock the full diagnostic-events contract:** `src/auto-reply/reply/agent-runner.persistence.test.ts:23-26`
  - body: The new test file replaces `../../infra/diagnostic-events.js` with a partial mock that omits `onInternalDiagnosticEvent`, so importing `runReplyAgent` fails before either new test runs. Add the missing export or reuse an existing narrow test helper instead of a broad module mock.
  - confidence: 0.94
