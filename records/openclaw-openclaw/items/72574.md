## Summary

Keep this open because current main still routes spawned Codex app-server startup through the shared client cleanup path, and PR #72812 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks for spawned Codex helper auth failures from `btw` to fail gracefully without tearing down the active gateway/session.

## Best Possible Solution

Land PR #72812, or an equivalent narrow Codex app-server change, so runs with `spawnedBy` use isolated startup clients while normal Codex runs keep shared-client reuse.

## Reproduction Assessment

Yes, source-backed. The issue provides concrete invalid-token steps/logs, and current main still sends spawned Codex startup through the shared client path that can clear the singleton after startup failure.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server clients and limiting shared-client teardown to runs that own the singleton is narrower than changing gateway restart behavior or global session lifecycle.
