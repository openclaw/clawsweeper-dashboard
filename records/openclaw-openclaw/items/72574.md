## Summary

Keep this open. Current main still routes Codex app-server startup through the shared client and clears that shared singleton on startup-path errors, while PR #72812 is the active open fix vehicle linked with closing syntax.

## What This Changes

The issue reports that an invalid or expired Codex bearer token during `btw spawn` returns 401 and also disconnects the active gateway/session, requiring a gateway restart.

## Best Possible Solution

Review and land PR #72812, or an equivalent narrow fix, so helper/spawned Codex app-server startup failures use isolated clients while ordinary runs keep the shared-client contract and cleanup remains explicit.

## Reproduction Assessment

Yes. The report has concrete steps and logs, and current main has a source-level reproduction path where a `thread/start` 401 during spawned/helper startup can flow into the startup catch that clears the shared singleton.

## Solution Assessment

Yes. Isolating only spawned/helper Codex startup is the narrowest maintainable direction because it preserves normal shared-client reuse while preventing auxiliary auth failures from affecting an active session.
