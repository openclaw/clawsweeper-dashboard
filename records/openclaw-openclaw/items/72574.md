## Summary

Keep this open: current main still allows spawned Codex startup to use the shared app-server client, and an open implementation PR already targets the same isolation bug.

## What This Changes

The issue reports that invalid or expired Codex bearer auth during `btw spawn codex 5.5` returns 401 and also disconnects the active gateway/session.

## Best Possible Solution

Review and land PR #72812, or an equivalent narrow Codex app-server change, so helper/spawned runs use isolated clients and cannot clear the live shared client.

## Reproduction Assessment

Yes. The issue gives invalid-token steps and logs, and current main still routes spawned Codex startup through the shared app-server client before `thread/start`, with cleanup that can clear that same singleton on failure.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server clients and scoping shared-client teardown to runs that own the singleton is narrower than changing gateway restart or global session lifecycle behavior.
