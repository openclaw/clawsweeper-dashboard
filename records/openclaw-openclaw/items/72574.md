## Summary

Keep open: the report is still source-reproducible on current main, and the explicit fix candidate at https://github.com/openclaw/openclaw/pull/72812 remains open and unmerged.

## What This Changes

The issue reports that `btw spawn codex 5.5` with an invalid or expired bearer token returns 401 and can disconnect the active gateway/session instead of failing locally.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72812, or an equivalent narrow Codex app-server lifecycle change, so spawned/helper runs use isolated startup clients while normal runs keep shared-client reuse.

## Reproduction Assessment

Yes, source-backed: the issue provides concrete invalid-token steps and logs, and current main still routes spawned Codex startup through shared-client cleanup that can clear the singleton. I did not run a live invalid-token gateway session during this read-only review.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server startup clients by `spawnedBy` is the narrow maintainable fix because it keeps normal shared-client reuse unchanged and avoids changing broader gateway lifecycle behavior.
