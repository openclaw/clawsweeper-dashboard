## Summary

Keep this open because current main still exposes the source-backed spawned Codex app-server shared-client teardown path, and open PR #72812 is the unmerged implementation candidate that explicitly targets this bug.

## What This Changes

The issue reports that invalid or expired Codex bearer auth during `btw spawn codex 5.5` returns 401 and can also disconnect the active gateway/session.

## Best Possible Solution

Land PR #72812, or an equivalent narrow Codex app-server change, so spawned/helper runs use isolated clients while normal Codex runs keep shared-client reuse.

## Reproduction Assessment

Yes, source-backed. The issue provides concrete invalid-token steps and logs, and current main still routes spawned Codex startup through the shared app-server client path that can clear the singleton on startup failure.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server clients and limiting shared-client teardown to runs that own the singleton is narrower than changing gateway restart behavior or global session lifecycle.
