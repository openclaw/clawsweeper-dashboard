## Summary

Keep open: current main still routes spawned Codex startup through the shared app-server client, and startup cleanup can clear that singleton after a `thread/start` failure. PR #72812 is the explicit open implementation candidate, so this issue should remain open until that PR, or an equivalent fix, lands.

## What This Changes

The issue reports that `btw spawn codex 5.5` with an invalid or expired bearer token returns 401 and then disconnects the active gateway/session.

## Best Possible Solution

Land PR #72812, or an equivalent narrow Codex app-server change, so spawned/helper runs use isolated clients while normal Codex runs keep shared-client reuse.

## Reproduction Assessment

Yes, source-backed. The issue provides concrete invalid-token steps and logs, and current main still shows a path where a spawned Codex `thread/start` failure can use and clear the shared app-server client.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server clients and limiting shared-client teardown to runs that own the singleton is narrower than changing gateway restart behavior or global session lifecycle.
