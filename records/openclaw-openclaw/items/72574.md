## Summary

Keep this open: current main still has a source-backed path where a spawned Codex startup uses the shared app-server client and startup cleanup can clear that singleton, while PR #72812 is the open implementation candidate for the same isolation bug.

## What This Changes

Invalid or expired bearer tokens in helper Codex `btw spawn` runs should fail locally without disconnecting the active gateway/session.

## Best Possible Solution

Land PR #72812, or an equivalent narrow Codex app-server change, so spawned/helper runs use isolated clients while normal Codex runs keep shared-client reuse.

## Reproduction Assessment

Yes, source-backed. The issue provides concrete invalid-token steps and logs, and current main still shows a path where spawned Codex `thread/start` failure can use and clear the shared app-server client.

## Solution Assessment

Yes. Isolating spawned/helper Codex app-server clients and limiting shared-client teardown to runs that own the singleton is narrower than changing gateway restart behavior or global session lifecycle.
