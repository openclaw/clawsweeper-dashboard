## What This Changes

The issue asks channel setup to auto-append an explicitly selected Weixin channel plugin to a restrictive `plugins.allow` list so setup can enable and continue instead of failing after install.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/93603; after a fix merges, this issue can close as implemented.

## Reproduction Assessment

Yes from source inspection: with a non-empty `plugins.allow`, the helper returns `blocked by allowlist` for non-ClickClack plugins, and channel setup stops when that result is not enabled. I did not run the interactive Weixin install flow in this read-only review.

## Solution Assessment

Yes, the best fix shape is a helper-level explicit-selection change rather than a Weixin-specific setup branch, because channel setup and the plugin-enable CLI already share that helper and the denylist/global-disable guards can remain centralized.
