## Summary

Keep open: current main and the latest release still limit explicit plugin allowlist auto-append to ClickClack, and https://github.com/openclaw/openclaw/pull/93603 is the open candidate fix for this Weixin setup failure.

## What This Changes

Guided Weixin channel setup should add or prompt for `openclaw-weixin` in a restrictive `plugins.allow` list after installing the selected plugin instead of failing during enablement.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/93603 so explicit setup/install selections update restrictive `plugins.allow` while preserving denylist and global-disable behavior, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection: current main only auto-allowlists `clickclack` after an allowlist block, and the Weixin install/setup path routes through that helper. I did not run the live interactive Weixin flow in this read-only review.

## Solution Assessment

Unclear as a product policy question: the linked PR is the clean central helper fix if maintainers accept explicit selection as the trust gesture. If that is too broad, the safer fix is to scope the behavior to channel setup or prompt before mutating `plugins.allow`.
