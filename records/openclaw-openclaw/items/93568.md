## Summary

Current main and v2026.6.9 still keep explicit plugin allowlist auto-append limited to ClickClack, so the Weixin setup dead end remains. This issue should stay open because https://github.com/openclaw/openclaw/pull/93603 is the live closing PR and the remaining decision is whether maintainers accept broad explicit-selection trust for `plugins.allow`.

## What This Changes

Requests that guided Weixin/channel setup append or prompt for the selected plugin id in `plugins.allow` when a restrictive allowlist is active, instead of installing the plugin and then failing enablement.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/93603; if broad helper-level trust is too much, narrow the mutation to guided channel setup or add a consent prompt before editing `plugins.allow`.

## Reproduction Assessment

Yes, source-reproducible: current main only auto-allowlists `clickclack`, while channel setup and onboarding plugin installation route selected plugins through that helper. I did not run a live interactive Weixin setup in this read-only review.

## Solution Assessment

Unclear until maintainers settle the trust policy: the linked PR is the clean central helper fix if explicit selection should trust any selected plugin. If that is too broad, the better solution is a channel-setup-scoped append or an explicit consent prompt.
