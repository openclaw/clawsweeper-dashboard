## Summary

Keep open: current main and v2026.6.8 still restrict explicit setup auto-allowlisting to ClickClack, so an explicitly selected Weixin/openclaw-weixin path can still hit the reported plugins.allow failure. The open closing PR at https://github.com/openclaw/openclaw/pull/93603 is the right implementation candidate, but it has not merged yet.

## What This Changes

The issue asks channel setup to add an explicitly selected channel plugin to plugins.allow and enable it instead of installing openclaw-weixin and then failing on the restrictive allowlist.

## Best Possible Solution

Review and land an owner-approved version of https://github.com/openclaw/openclaw/pull/93603 that generalizes explicit setup selection auto-allowlisting while preserving denylist and global-disable behavior; then close this issue after merge.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: channel setup calls the explicit-enable helper, and current main returns blocked by allowlist for non-ClickClack selections excluded from plugins.allow. I did not run the interactive Weixin setup flow in this read-only review.

## Solution Assessment

Yes, the best current path is the open PR that removes the ClickClack-only guard from the shared explicit-selection helper and keeps denylist behavior intact. This issue should remain open until that PR, or an equivalent owner-approved fix, merges.
