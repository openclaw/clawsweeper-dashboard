## Summary

Keep open: current main intentionally makes group/channel replies tool-only by default and documents the workaround, but the upgrade behavior remains a maintainer policy question rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to reduce Discord guild/channel reply invisibility after the 2026.5.3 `visibleReplies` default change by warning, preserving legacy automatic replies for upgrades, or adding clearer Discord-specific migration help.

## Best Possible Solution

Maintainers should decide the upgrade policy for existing group/channel configs, then either preserve legacy automatic replies for upgraded installs or add an explicit doctor/release warning that points to the `automatic` opt-out and `didSendViaMessagingTool` diagnostic.

## Reproduction Assessment

Yes, from source inspection: an unset group/channel config resolves to `message_tool_only` and suppresses automatic source delivery, and the dispatch tests assert a Discord channel final reply is not sent. I did not run a live Discord scenario.

## Solution Assessment

Unclear: current docs and doctor support the new default and document the workaround, but they do not settle whether upgraded configs should keep legacy automatic replies or receive a stronger upgrade notice.
