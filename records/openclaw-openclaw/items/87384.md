Keep this open: current main and the latest release still fall back from an empty inferred transcript file to CLI stdout, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that CLI audio transcription can inject progress/status stdout as the transcript when an expected transcript file is empty or missing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/87393 so deterministic file-output CLI STT paths treat transcript files as authoritative and cover empty/missing-file cases with focused regression tests.

## Reproduction Assessment

Yes, at source level: current main reads inferred transcript files only when non-empty and otherwise falls through to stdout. I did not execute a live failing run because this review is read-only.

## Solution Assessment

Yes. Treating deterministic transcript-file outputs as authoritative is the narrow maintainable fix, and the linked PR is the active implementation path to review.
