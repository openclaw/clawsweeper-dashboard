## Summary

Keep this open. Current main and v2026.6.8 still fall back from empty inferred CLI transcript files to stdout, and the linked closing PR is open and conflicting rather than merged.

## What This Changes

The issue requests that CLI audio STT entries with deterministic transcript files treat empty or missing transcript files as no transcript instead of falling back to progress stdout.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/87393 so deterministic file-output CLI STT paths treat empty or missing transcript files as no-transcript results instead of using status stdout.

## Reproduction Assessment

Yes, at source level. Current main reads inferred transcript files only when non-empty, then returns stdout, so an empty or missing file plus progress stdout follows the reported failure path.

## Solution Assessment

Yes. Treating deterministic transcript-file outputs as authoritative is the narrow maintainable fix; the linked PR is the current implementation candidate to refresh, review, and land or replace.
