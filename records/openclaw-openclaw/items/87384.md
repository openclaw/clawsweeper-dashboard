## Summary

Keep this open. Current main and the latest release still allow an empty inferred transcript file to fall through to CLI stdout, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to treat deterministic CLI audio transcript files as the source of truth so empty or missing transcript files do not inject progress stdout as user transcript text.

## Best Possible Solution

Review, refresh, and land or replace the linked media runner fix so deterministic file-output CLI STT paths return no transcript when the transcript file is empty or missing.

## Reproduction Assessment

Yes. Source inspection shows an audio CLI that creates an empty inferred transcript file and prints progress stdout will currently return stdout through runCliEntry.

## Solution Assessment

Yes. Treating deterministic transcript-file outputs as authoritative is the narrow maintainable fix; the linked PR is the current implementation path to review, refresh, or replace.
