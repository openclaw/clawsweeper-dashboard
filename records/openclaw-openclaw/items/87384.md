## Summary

Keep this open. Current main and the latest release still fall back from an empty inferred transcript file to CLI stdout, and the linked closing PR is open and currently conflicting rather than merged.

## What This Changes

The issue reports that CLI audio transcription can use progress stdout as the transcript when an expected transcript file is empty or missing.

## Best Possible Solution

Land or replace the linked media runner fix so deterministic file-output CLI STT paths treat empty or missing transcript files as authoritative no-transcript results instead of falling back to progress stdout.

## Reproduction Assessment

Yes. Source inspection shows an audio CLI that creates an empty inferred transcript file and prints progress stdout will currently return stdout through runCliEntry.

## Solution Assessment

Yes. Treating deterministic transcript-file outputs as authoritative is the narrow maintainable fix; the linked implementation PR is the current path to review or replace.
