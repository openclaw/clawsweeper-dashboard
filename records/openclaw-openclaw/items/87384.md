## Summary

Keep this open. Current main and v2026.6.9 still use stdout when an inferred CLI transcript file is empty, and the linked closing PR is open rather than merged.

## What This Changes

Fix the media CLI audio runner so deterministic transcript-file STT entries do not fall back to progress stdout when expected transcript files are empty or missing.

## Best Possible Solution

Review and land or replace the linked media runner PR so deterministic file-output CLI STT paths treat empty or missing transcript files as no transcript.

## Reproduction Assessment

Yes. Source inspection shows an audio CLI that creates an empty inferred transcript file and prints progress stdout will currently return stdout through runCliEntry.

## Solution Assessment

Yes. Making deterministic transcript-file outputs authoritative is the narrow maintainable fix; the linked PR is the current implementation candidate to review or replace.
