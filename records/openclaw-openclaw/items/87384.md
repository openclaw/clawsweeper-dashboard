## Summary

Keep this open. Current main and v2026.6.6 still fall back from an empty inferred transcript file to CLI stdout, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that CLI audio transcription can treat progress stdout as the transcript when an expected CLI transcript file is empty or missing.

## Best Possible Solution

Land or replace the linked fix so deterministic file-output CLI STT paths treat the transcript file as authoritative and cover empty/missing-file cases with focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows an audio CLI that creates an empty inferred transcript file and prints progress stdout will currently produce stdout as the transcript through runCliEntry.

## Solution Assessment

Yes. Treating deterministic transcript-file outputs as authoritative is the narrowest maintainable fix, and the active linked PR is the implementation path to review.
