## Summary

Keep open. Current main and v2026.6.6 still fall back from an empty inferred transcript file to CLI stdout, and the open linked fix PR is the canonical path rather than a reason to close this issue.

## What This Changes

Request that CLI audio transcription treat inferred transcript-file outputs as the transcript source of truth instead of injecting CLI progress stdout when the expected file is empty or missing.

## Best Possible Solution

Land a focused media runner fix that makes deterministic file-output STT paths authoritative and covers empty or missing transcript-file regressions.

## Reproduction Assessment

Yes at source level: configure a known file-output CLI audio command, have it create an empty inferred transcript file and print progress to stdout, and current main falls through to stdout trimming. I did not execute a live repro because this review is read-only.

## Solution Assessment

Yes for known deterministic file-output commands: making those transcript files authoritative is the narrow repair for the bug. A broader custom-wrapper output policy should stay separate unless maintainers want to expand the contract.
