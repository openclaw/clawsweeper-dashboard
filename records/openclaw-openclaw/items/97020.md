## What This Changes

The issue reports that the Anthropic-family to OpenAI stream-wrapper bridge emits Anthropic custom tool payloads as `type: "custom"`, causing OpenAI Chat Completions requests to fail instead of projecting them to function tools.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/97023, then close this issue only after a fix is merged or a better canonical path replaces it.

## Reproduction Assessment

Yes, at source level: current main preserves `type: "custom"` in the reported bridge path, and the adjacent wrapper test asserts that behavior. I did not run a live OpenAI failover call in this read-only review.

## Solution Assessment

Yes: aligning this bridge with the existing canonical OpenAI function-tool projection is the narrowest maintainable fix. Because https://github.com/openclaw/openclaw/pull/97023 is still open, the issue should stay open until that fix or a replacement merges.
