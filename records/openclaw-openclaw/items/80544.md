## Summary

## What This Changes

The branch adds a native `sessions_spawn` `completionOwner` enum, persists it through subagent run and final-delivery state, alters native completion routing and ACP rejection, updates hook types/tests/prompt snapshots, and changes plugin-disabled queue debounce resolution.

## Best Possible Solution

Land a narrowed owner contract that normalizes `completionOwner` at every raw/persisted boundary, preserves unknown values as legacy-missing owners, covers replay routing, and includes live Gateway/channel proof plus maintainer signoff for native-only and ACP semantics.

## Reproduction Assessment

Yes for the PR-head defect: source inspection shows restored records are raw JSON and the new router treats any defined non-`work-thread-final` owner as explicit. The linked feature request itself is not a current-main bug reproduction.

## Solution Assessment

No, not yet. The direction is plausible, but the durable enum needs normalization at state boundaries, live transport proof, and maintainer signoff on native-only owner semantics before merge.
