## Summary

## What This Changes

The PR adds a `completionOwner` enum to `sessions_spawn`, native subagent spawn records, completion routing, hook payloads, retry persistence, and targeted tests/prompt snapshots.

## Best Possible Solution

Land a native-only completion-owner contract after fixing the rollback gap, documenting or approving the semantics, and proving the real routing paths in a live Gateway/channel setup.

## Reproduction Assessment

Not applicable as a bug reproduction for the linked feature request; it is a design request with no minimal upstream runtime repro. The PR-introduced rollback gap is source-reproducible by following the `completionOwner="work-thread-final"` error branch after context preparation succeeds without a routable thread.

## Solution Assessment

Unclear as a merge decision. The native-only direction is coherent, but the patch should roll back context preparation on the new invalid-owner path and should not merge until maintainers approve the delivery-owner semantics and live proof covers the real routing behavior.
