## Summary

Keep open: the PR addresses an open native subagent delivery-ownership feature, but the current patch can still misroute completions when persisted owner values are stale or unsupported, and the supplied proof is synthetic only.

## What This Changes

Adds native `completionOwner` support across `sessions_spawn`, subagent spawn and announce routing, registry persistence, hook event payloads, prompt snapshots, and regression tests.

## Best Possible Solution

Normalize owner values at restore and routing boundaries, preserve child-binding-first behavior for missing or unknown owners, then land the native-only contract after maintainer approval and live Gateway/channel proof.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature/design PR. For the PR finding, source inspection gives a high-confidence path: persisted records can restore raw owner values, and the router branches on that raw value.

## Solution Assessment

No, not as-is. The implementation direction is reasonable, but persisted and routing owner values should be normalized before changing final-delivery ownership, and the PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize completionOwner before routing:** `src/agents/subagent-announce-delivery.ts:368`
  - body: The new branch treats any defined `completionOwner` other than `work-thread-final` as requester-owned. Persisted run records are restored from JSON by spreading stored fields into `SubagentRunRecord`, so a stale or unsupported owner string survives restore and skips the child binding, contrary to the promised legacy child-binding-first behavior. Normalize before this branch, or at restore, so unknown values behave as missing.
  - confidence: 0.87
