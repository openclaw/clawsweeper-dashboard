## Summary

Keep open: the owner contract is useful, but this PR is not merge-ready because persisted `completionOwner` values are not normalized before retry routing, the proof is synthetic only, and the native/ACP/queue behavior choices still need maintainer signoff.

## What This Changes

The PR adds a native subagent `completionOwner` contract, threads it through spawn, announce delivery, registry persistence, hook types, prompt snapshots, and focused tests.

## Best Possible Solution

Land a narrowed version that normalizes `completionOwner` at every raw and persisted boundary, preserves legacy child-bound fallback for unknown state, includes live Gateway/channel proof, and records maintainer approval for native owner modes and ACP deferral.

## Reproduction Assessment

Yes for the PR-head defect: source inspection shows restored JSON run records can carry an unnormalized `completionOwner` into pending-final retry routing. The linked feature request itself is design/feature work and still lacks a minimal current-main runtime reproduction.

## Solution Assessment

No, not as currently proposed. The contract direction is plausible, but the PR should normalize durable enum state, split or justify the unrelated queue debounce behavior, add live delivery proof, and get maintainer signoff on native-only/ACP semantics.
