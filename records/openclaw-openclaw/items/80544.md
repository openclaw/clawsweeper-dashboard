## Summary

Keep open: this is an active feature PR linked to an open issue, and source inspection still shows a P2 routing bug around unnormalized persisted `completionOwner` values. The supplied proof is structured but synthetic, so it does not satisfy the external-PR real behavior proof gate for Gateway/channel delivery.

## What This Changes

Adds a native subagent `completionOwner` contract through `sessions_spawn`, native spawn, registry persistence, completion announce routing, hook payloads, prompt snapshots, and regression tests.

## Best Possible Solution

Normalize owner values at persisted and routing boundaries, preserve child-binding-first behavior for missing or unknown values, then land the native-only contract after maintainer architecture review and live Gateway/channel proof.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because the linked work is a feature/design request. For the PR finding, yes by source inspection: persisted records can carry raw `completionOwner` values through registry restore into routing, which branches on the raw value.

## Solution Assessment

No, not as-is. The direction is plausible, but owner values should be normalized before routing decisions and the PR still needs live behavior proof for the changed delivery path.
