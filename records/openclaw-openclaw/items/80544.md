## What This Changes

The PR adds a native `completionOwner` option to `sessions_spawn`, propagates it through subagent run records, final-delivery retry payloads, hook payloads, completion routing, prompt snapshots, and native subagent tests while rejecting the option for ACP.

## Best Possible Solution

Normalize owner values at persisted and routing boundaries, preserve the legacy fallback for missing or unknown data, then land the native-only contract after maintainer architecture review and live Gateway/channel proof.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The blocking review finding is source-reproducible from PR-head code: persisted JSON can restore a raw owner value and the new router branches on raw presence before trying the child fallback.

## Solution Assessment

No, not as-is. The implementation direction is plausible, but it should normalize owner values before routing or replay and still needs live behavior proof for the real Gateway/channel path.
