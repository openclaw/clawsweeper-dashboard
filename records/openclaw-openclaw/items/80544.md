The branch adds a native `completionOwner` enum to `sessions_spawn`, persists it through subagent final-delivery state, adjusts completion routing, rejects the option for ACP, and updates focused tests plus prompt snapshots.

## Best Possible Solution

Normalize `completionOwner` at persisted/raw boundaries, preserve legacy child-binding-first behavior for unknown or missing owners, then prove the contract in a real Gateway/channel run before merge.

## Reproduction Assessment

No high-confidence current-main bug reproduction is needed because this is a feature PR. The PR-head defect is source-reproducible: persisted JSON is cast back into typed records and the new router treats any non-undefined owner as explicit.

## Solution Assessment
