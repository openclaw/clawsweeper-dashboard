## Summary

Keep this PR open. The Anthropic onboarding direction is still useful and current main has not implemented it, but the PR head still has a blocking configured-command regression: the new install/login helper hard-codes the PATH binary instead of honoring OpenClaw's existing `claude-cli` backend command override.

## What This Changes

The PR relabels the Anthropic CLI onboarding choice as a Claude subscription path, adds prompted Claude CLI detect/install/login handling, and updates Anthropic docs, changelog, metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding path, but amend it so the wizard resolves the effective `claude-cli` backend command from `ctx.config`, uses that command for detection and login, offers npm global install only for the default PATH command, and adds regression coverage for a custom absolute-command setup.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary and run Anthropic CLI onboarding on a host where plain `claude` is not on PATH; the PR probes `claude --version` and later spawns `claude auth login` instead of the configured command.

## Solution Assessment

No. Plugin-local onboarding is the right product boundary, but this implementation is not the best fix until it reuses the existing CLI backend command-resolution contract rather than introducing a second hard-coded command path.
