## Summary

Keep this PR open: the subscription onboarding direction is still relevant, but the branch still breaks the existing custom Claude CLI command contract by probing and launching the literal `claude` binary.

## What This Changes

The PR relabels Anthropic CLI onboarding as a Claude subscription path and adds interactive Claude Code install/sign-in handling with docs, changelog, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding flow, but make install/login honor the existing CLI backend command contract and cover both default-PATH and custom-command cases in tests.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH, then choose interactive Anthropic CLI onboarding; the PR probes and launches `claude` instead of the configured command.

## Solution Assessment

No. The product boundary is right, but the implementation should resolve the effective command from `ctx.config` for detection and login, and reserve npm global install for the default PATH command case.
