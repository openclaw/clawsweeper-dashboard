## Summary

Keep this PR open for a focused repair. Current main still has the old Anthropic CLI onboarding label, so the contribution is not obsolete, but the PR head still bypasses the existing configured Claude CLI command path by probing and launching the literal `claude` binary during onboarding.

## What This Changes

The PR relabels Anthropic Claude CLI onboarding as a Claude subscription path, adds prompted Claude Code detect/install/login handling, and updates Anthropic docs, changelog, manifest/provider metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned onboarding direction, but resolve the effective `claude-cli` backend command from `ctx.config`, use that command for detection and login, and reserve npm global install for the default PATH command case.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH, then choose the interactive Anthropic CLI auth path; the PR probes `claude --version` before using the configured backend command.

## Solution Assessment

No. The product boundary is right, but the current implementation is not the narrowest maintainable fix because it bypasses the established CLI backend command override instead of reusing it.
