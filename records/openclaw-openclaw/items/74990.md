## Summary

Keep this PR open. Current main still has the older Anthropic CLI onboarding label and manual sign-in path, so the contribution is not obsolete, but the patch still breaks supported custom Claude Code command paths by hard-coding `claude` for detection and login.

## What This Changes

The PR relabels Anthropic Claude CLI onboarding as a Claude subscription path and adds wizard-driven Claude Code install/login handling with docs, changelog, metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding flow, but resolve the effective `claude-cli` backend command from config for detection and login, and reserve npm global install for the default PATH command case.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH, then choose interactive Anthropic CLI onboarding; the PR probes and launches `claude` instead of the configured command.

## Solution Assessment

No. The product boundary is right, but the implementation bypasses the existing CLI backend command override contract instead of reusing it.
