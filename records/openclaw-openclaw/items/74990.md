## Summary

Keep this PR open: the Anthropic subscription onboarding UX is not already on current main and is scoped to the Anthropic plugin, but the current head still bypasses the existing custom Claude CLI command contract by detecting and logging in with the literal `claude` binary.

## What This Changes

The PR relabels Anthropic Claude CLI onboarding as a Claude subscription path and adds interactive Claude Code install/login handling with docs, changelog, metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding flow after it respects configured backend commands and has focused plugin coverage for custom-command hosts.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while plain `claude` is absent from PATH, then run interactive Anthropic CLI onboarding; the PR probes and launches literal `claude` instead of the configured command.

## Solution Assessment

No. The product direction is right, but the implementation should resolve the effective `claude-cli` backend command from config for detection and login, and reserve npm global install for the default PATH command case.
