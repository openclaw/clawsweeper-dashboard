## Summary

Keep this PR open: current main still has the older Anthropic CLI onboarding UX, but the proposed install/login path still bypasses the existing custom Claude CLI command contract and would regress configured binary paths.

## What This Changes

The PR relabels Anthropic CLI auth as a Claude subscription path and adds Anthropic plugin install/login onboarding with docs, changelog, metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding flow, but resolve the effective `claude-cli` backend command from config for detection and login, and offer npm global install only for the default PATH command case.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH, then choose interactive Anthropic CLI onboarding; the PR probes and launches `claude` instead of the configured command.

## Solution Assessment

No. The product boundary is right, but the implementation bypasses the existing CLI backend command override contract instead of reusing or locally honoring it.
