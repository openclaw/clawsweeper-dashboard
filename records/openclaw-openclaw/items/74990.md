## Summary

Keep open. Current main still has the older Anthropic CLI onboarding, but this PR is not merge-ready because it still hard-codes `claude` instead of honoring `agents.defaults.cliBackends.claude-cli.command`; it also lacks required real behavior proof and is dirty against main.

## What This Changes

The PR relabels Anthropic Claude CLI onboarding as a Claude subscription path and adds user-confirmed Claude Code install/sign-in handling, docs, changelog, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding direction, but resolve the effective `claude-cli` command from config for detection/login, offer npm install only for the default PATH command case, then rebase and add real after-fix proof.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH, then choose interactive Anthropic CLI onboarding; source inspection shows the PR still probes and spawns the literal `claude`.

## Solution Assessment

No. The feature direction is plausible, but the current implementation bypasses the existing CLI backend command override contract instead of reusing it in the Anthropic plugin.

## Review Findings

Overall correctness: patch is incorrect
