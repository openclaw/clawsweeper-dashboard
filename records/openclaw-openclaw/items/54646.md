## Summary

Keep open: current main still has the gateway-rpc timeout bug, and this PR is the active contributor path, but the patch is not merge-ready because it still changes a shared parser without validating every affected CLI caller and only provides test/mock proof.

## What This Changes

The PR validates gateway-rpc CLI timeout input, tightens shared positive-integer parsing, validates cron add timeout seconds, and adds focused CLI tests plus a changelog entry.

## Best Possible Solution

Land this branch or a canonical successor after either scoping the strict parser to timeout paths or adding explicit invalid-input errors for every affected shared-helper consumer, with focused regression coverage and real CLI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `--timeout` defaults to `30000` in the CLI wrapper, but malformed explicit input reaches `callGateway` as `NaN` and then falls back to `10000`.

## Solution Assessment

No as-is. CLI-boundary validation is the right direction, but tightening the shared integer helper needs matching validation at remaining explicit callers such as `tasks audit --limit`, or the gateway-rpc fix should use a narrower parser.
