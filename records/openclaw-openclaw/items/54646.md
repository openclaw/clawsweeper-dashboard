## Summary

Keep open: current main still has the gateway-rpc timeout bug, and the PR is useful but still needs one narrow repair because the stricter shared integer parser leaves `tasks audit --limit` silently dropping malformed explicit limits.

## What This Changes

The PR adds gateway-rpc timeout validation/defaulting, tightens shared positive-integer parsing, validates cron add timeout seconds, updates focused CLI tests, and adds a changelog entry.

## Best Possible Solution

Land this branch or a canonical replacement after adding explicit validation for the remaining `tasks audit --limit` parser fallout while preserving the gateway-rpc default and malformed-timeout rejection behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows malformed gateway-rpc `--timeout` input is converted with `Number(...)` and can reach the gateway call fallback instead of failing at the CLI boundary.

## Solution Assessment

No as-is. CLI-boundary validation is the right solution, but the shared helper tightening also needs the remaining caller validation, or the strict parsing should be scoped to timeout-specific paths.
