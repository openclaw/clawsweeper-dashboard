## Summary

Keep open. The report targets the shipped v2026.5.7 update path itself; current and shipped code contain config-clobber safeguards, but they do not explain the reported credentials-directory deletion or prove the packaged macOS/npm auto-update path is safe.

## What This Changes

The issue reports that updating from v2026.5.6 to v2026.5.7 reset `openclaw.json`, removed agents/channels/plugins/bindings, deleted `~/.openclaw/credentials/`, and requests preservation plus pre-change backups.

## Best Possible Solution

Replay the exact packaged update path with representative config and credential files, then either land/backport a validated updater/config-preservation fix or close with proof that the loss came from a separate reset/ownership path.

## Reproduction Assessment

No. The issue has plausible user steps and violates the documented update contract, but I did not establish a high-confidence current-main or packaged v2026.5.7 reproduction, and the credentials deletion path remains unexplained.

## Solution Assessment

Unclear. Existing config write rejection, backup-before-replace, and recovery tests are the right direction, and https://github.com/openclaw/openclaw/pull/79299 may cover one related vector, but the complete fix needs packaged-update validation and a credentials-preservation RCA.
