## Summary

Keep open. Current main has strong config-clobber guards and backup recovery for the update-channel-only shape, but this report targets the shipped v2026.5.7 package-update path and includes unexplained deletion of the credentials directory, so cleanup closure is not safe yet.

## What This Changes

The issue reports that a 2026.5.6 to 2026.5.7 npm auto-update reset `openclaw.json`, removed agents/channels/plugins/bindings, and deleted `~/.openclaw/credentials/`.

## Best Possible Solution

Validate the packaged update path with a realistic macOS npm install, preserve or restore the original config through the guarded write path, and identify or rule out any updater/state-migration path that can remove `~/.openclaw/credentials/`.

## Reproduction Assessment

No, not as a high-confidence current-main reproduction. Source and tests cover the config-clobber shape, but the real v2026.5.6 to v2026.5.7 packaged update and credential-directory loss still need live proof or logs from the affected path.

## Solution Assessment

Unclear as a complete fix. The current-main config write rejection, backup, and recovery behavior are the right direction for config loss, but the best resolution also needs packaged-update validation and a credential-preservation root-cause pass.
