## Summary

Keep open: current main still has a source-level path where restrictive subagent policy can be collapsed to concrete built-in tool names before bundled MCP tools are materialized, so spawned children can skip MCP runtime creation. ## What This Changes

MCP tools registered through `mcp.servers` should be injected into `sessions_spawn` child sessions when allowed by subagent and target-agent tool policy.

## Best Possible Solution

Fix the existing subagent/runtime tool-policy path so inherited subagent policy preserves MCP/plugin intent until the child embedded runner can materialize and then filter bundled MCP tools, with a focused regression proving a spawned child receives allowed MCP schemas and prompt entries.
