## Label Justifications

## Summary

Keep open: current main and latest release still rebuild child inherited allowlists from concrete parent tools before bundled MCP tools are materialized, so configured MCP selector tokens can still be lost for `sessions_spawn` child sessions. The active fix path is the open proof-supplied PR, which needs maintainer review because it changes a subagent tool authorization boundary.

## What This Changes

The issue reports that `sessions_spawn` child sessions receive only built-in tools and miss allowed `bundle-mcp`/MCP tools despite top-level subagent and per-agent allowlists.

## Best Possible Solution

Review and land PR https://github.com/openclaw/openclaw/pull/92521, or an equivalent policy-filtered selector-preservation fix, after maintainer security-boundary approval.

## Reproduction Assessment

Yes. Source inspection shows current main stores a concrete-only inherited allowlist on child sessions, while child MCP runtime creation requires selector tokens that the inherited list can lose; I did not run a fresh live MCP repro in this read-only pass.

## Solution Assessment

Yes, provisionally. The inherited tool-policy path is the right layer, and preserving policy-approved deferred selectors is narrower than removing inherited allow enforcement, but the authorization-boundary design needs maintainer approval.
