The issue requests that MCP tools from `mcp.servers` be injected into spawned native subagent sessions when `bundle-mcp`, per-tool subagent allowlists, and target-agent allowlists permit them.

## Best Possible Solution

Land a maintainer-approved fix that lets the child MCP runtime see permitted `bundle-mcp`/plugin policy or projected MCP tool names while still applying the final effective tool policy before model requests.

## Reproduction Assessment
