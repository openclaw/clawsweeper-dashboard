## Summary

Keep open: current main and the latest release still rebuild spawned-child inherited tool allowlists from concrete materialized tools, while child MCP runtime startup still needs selector tokens such as `bundle-mcp`, `group:plugins`, or `server__tool`. The strongest candidate fix is https://github.com/openclaw/openclaw/pull/92521, but it is open, conflicting, and security-boundary sensitive.

## What This Changes

Report that MCP tools configured through `mcp.servers` and allowed by `bundle-mcp` plus per-agent policies are missing from `sessions_spawn` child sessions.

## Best Possible Solution

Rebase, review, and land https://github.com/openclaw/openclaw/pull/92521 or an equivalent narrow fix that preserves MCP selector tokens through inherited subagent allowlists while enforcing the final effective policy, with focused tests and MCP child-session proof.
