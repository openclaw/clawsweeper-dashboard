## Summary

Current main still does not demonstrate that authorized MCP schemas reach spawned child sessions, and the linked implementation remains open rather than merged. This source-backed bug should stay open for a maintainer-owned policy and security decision.

## What This Changes

Ensure MCP tools permitted by existing subagent and target-agent policies are loaded into native `sessions_spawn` child schemas and prompts.

## Best Possible Solution

Use one canonical child construction pipeline that carries explicit MCP authorization intent through spawn, materializes candidate MCP schemas, and applies the complete target-child effective policy as the final step before schemas and prompt text are exposed.

## Reproduction Assessment

Yes at source level. Current main has a concrete path where gateway inheritance yields a built-in-only allowlist and the child MCP activation check consequently skips schema loading; the attached exported schemas and independent confirmations support the same result.

## Solution Assessment

No, a new public `sessions_spawn.toolsAllow` parameter is not by itself the narrowest solution to this existing configured-policy bug. The canonical inheritance and materialization pipeline should be repaired, with least-privilege filtering proven after MCP schemas load.
