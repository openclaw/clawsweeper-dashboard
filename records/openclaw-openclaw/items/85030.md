## Label Justifications

- `P1`: The bug blocks a documented multi-agent workflow where spawned subagents should receive allowed MCP tools for delegated work.
- `impact:session-state`: The failure is caused by persisted child-session inherited tool state that omits the MCP selectors needed by the spawned runtime.

## Summary

Keep open: current main and the latest release still rebuild the child inherited allowlist from concrete parent tools before bundled MCP tools are materialized, so the reported `sessions_spawn` MCP gap remains real. The active path is the open proof-supplied fix PR, which needs maintainer review because it touches the subagent tool authorization boundary.

## What This Changes

The issue asks for MCP tools allowed by `bundle-mcp`, subagent policy, and target-agent allowlists to be injected into `sessions_spawn` child sessions instead of only built-ins.

## Best Possible Solution

Review and land a bounded selector-preservation fix such as https://github.com/openclaw/openclaw/pull/92521 if maintainers approve its authorization-boundary model, then close this as implemented.

## Reproduction Assessment

Yes. Source inspection shows current main stores a concrete-only inherited allowlist on child sessions, while child MCP runtime creation requires the missing `bundle-mcp` or server/tool selector tokens.

## Solution Assessment

Yes, the issue should be solved in the inherited tool-policy path, but the exact selector-preservation design needs maintainer security-boundary approval. The active PR is narrower than deleting inherited allow enforcement because it keeps the parent-policy gate intact.
