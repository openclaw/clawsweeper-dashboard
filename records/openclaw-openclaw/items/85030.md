## Summary

Keep open: the reporter's attached 2026.5.18 trajectory evidence lines up with a current-main source path where spawned subagents can still lose bundled MCP tools through the inherited parent tool-surface filter.

## What This Changes

The issue asks for MCP tools from `mcp.servers` to appear in `sessions_spawn` child sessions when allowed by subagent policy and the target agent's tool policy.

## Best Possible Solution

Add a focused spawned-subagent MCP regression, then adjust the inheritance/final-policy boundary so allowed child MCP tools survive without weakening the parent-to-child tool restriction model.

## Reproduction Assessment

Yes, for a source-level reproduction: current main stores a parent-derived child allowlist before bundled MCP tools are appended, then applies that allowlist to child bundled MCP tools. I did not run the reporter's live `sessions_spawn` trajectory export on current main.

## Solution Assessment

Unclear until maintainers confirm the security precedence rule. The likely technical fix is narrow, but removing or bypassing inherited policy wholesale would be the wrong shape because it could change delegation authority.
