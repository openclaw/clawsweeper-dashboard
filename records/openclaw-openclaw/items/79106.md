## Summary

The PR adds an owner-only lazy bash `exec` registration to Gateway `/tools/invoke` behind existing `gateway.tools.allow`, factors shared exec config resolution, and updates docs, changelog, and tests.

## Best Possible Solution

Land the narrow exec-only registration after tightening the regression test and rebasing onto current main; keep the broader coding-tools direction in https://github.com/openclaw/openclaw/pull/63919 separate.

## Reproduction Assessment

Yes. Source inspection shows current main can remove `exec` from the HTTP deny list but never registers the agent-side `exec` tool for `/tools/invoke`, and the PR body includes live before/after curl output matching that path.

## Solution Assessment

Yes, with the test correction. Reusing shared exec config resolution plus an owner-only, default-denied HTTP registration is the narrow maintainable exec-only approach; broad coding-tool exposure belongs in the separate broader PR.
