Full review comments:

- **[P1] Guard delayed recovery against stale terminal state:** `src/agents/subagent-registry.ts:376`
  - body: `scheduleLifecycleCatchRecovery` retries the captured completion whenever the run is ended and cleanup flags are clear. Because `completeSubagentRun` overwrites `endedAt`, `outcome`, and `endedReason`, and cleanup can intentionally clear `cleanupHandled` while final delivery is deferred, an older failed timeout/error retry can fire after a newer terminal state and rewrite the run or drive the wrong completion announcement. Compare the current entry to the failed terminal state or resume cleanup from the current entry without replaying stale params.
  - confidence: 0.88
- **[P2] Make the regression test reject completeSubagentRun:** `src/agents/subagent-registry.test.ts:928`
  - body: Mocking `runSubagentAnnounceFlow` does not make `completeSubagentRun` reject: the announce flow is started asynchronously by cleanup and catches its own rejection, and this test also never advances the 1-second recovery timer. As written it can pass without exercising the new catch/recovery path, so inject a rejection before cleanup and advance timers through recovery.
  - confidence: 0.82
