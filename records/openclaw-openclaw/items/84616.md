## What This Changes

The PR adds `.catch()` handlers and delayed recovery for three subagent lifecycle completion fire-and-forget paths, plus warning-focused registry tests.

## Best Possible Solution

Keep the local catch handlers and recovery helper as the code direction, but merge only after redacted terminal or harness proof shows the delayed recovery completes, or after maintainers explicitly accept that proof gap.

## Reproduction Assessment

Yes. Current main has two timer `completeSubagentRun` fire-and-forget calls and one listener async IIFE without `.catch()`, and OpenClaw's global unhandled rejection handler can exit the process.

## Solution Assessment

Yes as a code direction: local `.catch()` handlers plus reuse of existing orphan and announce-cleanup recovery is the narrow maintainable fix. The merge decision should wait for delayed-recovery proof or explicit maintainer acceptance of that gap.
