## Summary

Keep this PR open: the catch-handling direction is useful, but the new recovery branch can bypass part of the normal completion cleanup after the same mid-completion failure that the tests inject, and the supplied proof still does not show registered-run recovery completing.

## What This Changes

This PR attaches catch handlers to three subagent lifecycle completion promises, adds delayed orphan/cleanup recovery, and expands subagent registry tests.

## Best Possible Solution

Keep the catch handlers, but make mid-completion failures resume through the normal completion cleanup path or make the fragile synchronous status event best-effort, then prove registered-run recovery once.

## Reproduction Assessment

Yes, at source level: current main has unhandled fire-and-forget lifecycle completion promises, and the PR tests force completeSubagentRun to reject through lifecycle event emission. I did not run a live registered subagent repro in this read-only review.

## Solution Assessment

No, not yet: adding catch handlers at the fire-and-forget boundaries is the right direction, but the delayed recovery should not bypass the remaining completeSubagentRun cleanup steps after a mid-function rejection.
