## What This Changes

The branch adds warning `.catch()` handlers and delayed recovery to three subagent lifecycle fire-and-forget completion paths, plus registry tests for rejection logging.

## Best Possible Solution

Keep the `.catch()` hardening, but route recovery through a guarded lifecycle-controller path that preserves browser cleanup, run-mode MCP retirement, and existing cleanup bookkeeping, then prove it on a registered run.

## Reproduction Assessment

Yes, at source level: current main has three fire-and-forget lifecycle completion paths without local rejection handling, and `completeSubagentRun` can reject after mutating run state. I did not run a live registered-run crash reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` handlers is the right direction, but the recovery shortcut bypasses required lifecycle cleanup and the delayed recovery callback needs its own guard.
