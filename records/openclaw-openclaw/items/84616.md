## Best Possible Solution

Keep the catch-handler approach, but preserve each branch's original completion payload, align or explicitly extend the orphan-recovery source contract, and then prove latest-head recovery on a registered run.

Yes. Source inspection on current `main` shows the two timer callbacks and listener async IIFE can reject without local `.catch()` handling; I did not run a live fault-injection repro in this read-only review.

## Solution Assessment

No. The narrow maintainable solution is to add catch handling while preserving the original completion payload and keeping the orphan-reconciliation contract type-safe.
