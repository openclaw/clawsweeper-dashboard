## Summary

Keep open: this PR targets a real agents lifecycle bug, but the proposed recovery path is not merge-ready because it can resume announce cleanup after a mid-completion rejection while bypassing the normal browser-session cleanup and run-mode MCP retirement steps.

## What This Changes

The PR adds `.catch()` handlers, delayed lifecycle catch recovery, and subagent registry tests for three fire-and-forget `completeSubagentRun` paths.

## Best Possible Solution

Keep the catch hardening, but route recovery through the same pre-announce cleanup sequence as `completeSubagentRun` or make the best-effort throw sites non-fatal before cleanup resumes.

## Reproduction Assessment

Yes, at source level: current main has three fire-and-forget lifecycle completion paths that can reject without a local `.catch()`. I did not establish a live registered-run crash or cleanup-stall reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` handlers is the right direction, but the recovery helper should not bypass `completeSubagentRun`'s normal pre-announce cleanup steps.
