## Summary

This PR should stay open because the catch handlers are useful, but the new recovery path only handles true orphaned session-store entries and misses the ended-but-not-cleaned state the PR says it is fixing.

## What This Changes

The PR adds `.catch()` handlers to three fire-and-forget subagent lifecycle completion calls, logs failures, schedules orphan reconciliation, and adds focused tests for the rejection paths.

## Best Possible Solution

Keep the catch handlers, but drive the existing resume or cleanup flow for ended runs whose cleanup is incomplete even when the child session is not a true orphan, then prove that path with redacted runtime or harness logs.

## Reproduction Assessment

No high-confidence real-runtime reproduction is supplied. The source path is clear from the unhandled fire-and-forget promises and the PR's mock tests, but the submitted runtime proof does not trigger a rejected lifecycle completion.

## Solution Assessment

No, not yet. Catching the fire-and-forget promises is the right direction, but orphan pruning alone is not the narrowest complete fix because the described partial state can still have a valid child session and need cleanup resumed.
