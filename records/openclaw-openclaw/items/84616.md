## Summary

## What This Changes

The branch adds .catch() handlers plus delayed recovery for three fire-and-forget subagent lifecycle completion paths, with registry tests for warning logs.

## Best Possible Solution

Keep the catch hardening, but move recovery into a lifecycle-controller path that replays browser cleanup and run-mode MCP retirement before announce cleanup, then prove registered-run recovery.

Yes, at source level: current main has three lifecycle completion paths that can reject without local rejection handling. I did not establish a live current-main registered-run crash reproduction in this read-only review.

## Solution Assessment

No, not yet: adding .catch() warning handlers is the right direction, but the recovery shortcut should preserve the normal lifecycle cleanup order before merge.
