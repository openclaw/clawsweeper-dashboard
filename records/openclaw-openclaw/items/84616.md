## Best Possible Solution

Yes, at source level: current main has three fire-and-forget `completeSubagentRun` paths without rejection handlers, and `completeSubagentRun` persists ended state before later lifecycle/cleanup steps that can reject or throw. I did not run fault injection in this read-only review.

## Solution Assessment

No. Catching and logging the rejected promises is the right direction, but recovery should reuse or retry the normal completion cleanup tail instead of calling announce cleanup directly.
