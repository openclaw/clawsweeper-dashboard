## Best Possible Solution

Keep the `.catch()` hardening, but route recovery through the same pre-announce cleanup sequence as `completeSubagentRun` or make the pre-cleanup throw sites non-fatal before announce cleanup resumes.

Yes, at source level: current main has three fire-and-forget lifecycle completion paths that can reject without a local `.catch()`. I did not run a live registered-run crash or cleanup-stall reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` handlers is the right direction, but the recovery helper should not bypass `completeSubagentRun`'s normal pre-announce cleanup steps.
