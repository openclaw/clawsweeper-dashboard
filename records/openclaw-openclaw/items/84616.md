## Best Possible Solution

Keep the catch handlers, but route catch recovery through a shared full post-completion cleanup path that preserves browser cleanup, bundle MCP retirement, and announce cleanup ordering, then prove it on registered run state.

## Solution Assessment

No, not yet: adding `.catch()` handlers is the right direction, but the non-orphan recovery path should preserve the normal post-completion cleanup sequence before merge.
