## What This Changes

The PR modifies `src/agents/subagent-registry.ts` to add catch handlers and delayed recovery around three fire-and-forget subagent lifecycle completion paths.

## Best Possible Solution

Keep the catch-handling direction, but make the recovery helper contract type-safe, retry the exact failed completion payload for every lifecycle branch, remove the unrelated closing reference, and require latest-head registered-run proof.

## Reproduction Assessment

Yes, at source level. Current main has two lifecycle timer callbacks and one lifecycle listener async IIFE that can reject without a `.catch()`, though I did not run fault injection in this read-only review.

## Solution Assessment

No. Adding catches is the right direction, but this implementation should preserve the failed completion payload and either extend or reuse the orphan-recovery source contract before it is safe.
