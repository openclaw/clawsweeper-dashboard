## Summary

## What This Changes

The PR adds catch handlers and delayed retry/orphan recovery around three fire-and-forget subagent lifecycle completion paths in `src/agents/subagent-registry.ts`.

## Best Possible Solution

Keep the catch-handling direction, but make recovery type-safe, preserve the exact failed completion payload for every branch, remove the unrelated closing reference, and require latest-head registered-run proof before merge.

## Reproduction Assessment

Yes, at source level. Current main has two lifecycle timer callbacks and one lifecycle listener async IIFE that can reject without a `.catch()`, though I did not run fault injection in this read-only review.

## Solution Assessment

No. Adding catch handlers is the right direction, but this implementation needs a type-safe orphan source, branch-specific listener recovery payloads, correct closing metadata, and latest-head proof before it is the maintainable fix.
