## What This Changes

The PR adds warning `.catch()` handlers plus delayed recovery for three fire-and-forget subagent lifecycle completion paths and adds focused registry tests.

## Best Possible Solution

Keep the `.catch()` hardening, but route catch recovery through a shared full post-completion cleanup path before announce cleanup and prove it on registered run state.
