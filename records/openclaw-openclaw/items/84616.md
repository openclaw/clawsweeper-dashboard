## What This Changes

The PR adds `.catch()` handlers and delayed recovery for three fire-and-forget `completeSubagentRun` paths in `src/agents/subagent-registry.ts`, with focused subagent registry tests.

## Best Possible Solution

Route catch recovery through the same cleanup sequence as `completeSubagentRun`, or narrow this PR to catch-and-warn hardening and track registered-run recovery separately.

## Reproduction Assessment

Yes, at source level: current main has three `completeSubagentRun` fire-and-forget lifecycle paths without `.catch()` handlers. I did not establish a live failing registered-run reproduction in this read-only review.

## Solution Assessment

No, not yet: adding catch handlers is the right direction, but the non-orphan recovery path should not bypass `completeSubagentRun`'s normal pre-announce cleanup steps.
