## Summary

Keep open: the `.catch()` hardening addresses a real source-reproducible unhandled rejection path, but the added recovery path can bypass normal subagent lifecycle cleanup and the supplied proof does not exercise recovery for a registered run.

## What This Changes

The PR adds warning `.catch()` handlers plus delayed recovery attempts to three subagent lifecycle fire-and-forget completion paths, with registry tests for rejected completions.

## Best Possible Solution

Keep the `.catch()` hardening, but route recovery through a shared lifecycle-controller path that preserves the normal pre-announce cleanup sequence and prove it with a registered-run failure.

## Reproduction Assessment

Yes, at source level: current main has three fire-and-forget subagent lifecycle completion paths without local rejection handling. I did not run a live current-main registered-run crash reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` is the right direction, but the recovery shortcut should preserve the same lifecycle cleanup order as normal completion before it is merge-ready.
