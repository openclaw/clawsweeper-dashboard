## Summary

Keep open: the source bug is real, but the proposed recovery can complete a run through the announce cleanup path without the normal pre-announce browser and run-mode MCP cleanup sequence, and the supplied proof does not exercise recovery for a registered subagent run.

## What This Changes

The branch adds `.catch()` handlers and a delayed recovery helper for three fire-and-forget subagent lifecycle completion paths, with focused registry tests for warning logs.

## Best Possible Solution

Keep the `.catch()` hardening, but route recovery through a retryable post-completion cleanup sequence that preserves browser cleanup and run-mode MCP retirement before announce cleanup, with registered-run proof.

## Reproduction Assessment

Yes, at source level: current main has three fire-and-forget lifecycle completion paths without local `.catch()` handling, and `completeSubagentRun` can reject after mutating/persisting run state. I did not run a live registered-run crash reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` handlers is the right direction, but the recovery helper should not bypass `completeSubagentRun`'s normal pre-announce cleanup steps.
