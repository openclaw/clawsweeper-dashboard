## What This Changes

This PR adds catch handlers and delayed retry/orphan recovery for three fire-and-forget subagent lifecycle completion paths in `src/agents/subagent-registry.ts`.

## Best Possible Solution

Keep the catch handlers, but carry the original completion parameters for each lifecycle branch into recovery, add regression coverage for the blocked-liveness rejection case, remove the unrelated closing reference, and then prove registered-run cleanup or orphan reconciliation on the latest head.

No. Catching these promises is the right direction, but the event-handler recovery must retry the original branch outcome rather than hard-code success, and the latest patch still needs registered-run proof.
