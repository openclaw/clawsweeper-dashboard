## Best Possible Solution

Keep the `.catch()` hardening, but route recovery through a shared lifecycle-controller path that preserves normal pre-announce cleanup ordering and prove it with a registered-run failure.

Yes, at source level: current main has three subagent lifecycle completion paths that can reject without local rejection handling. I did not establish a live current-main registered-run crash reproduction in this read-only review.

## Solution Assessment

No, not yet: adding `.catch()` is the right direction, but the non-orphan recovery path should preserve the same lifecycle cleanup order as normal completion before merge.
