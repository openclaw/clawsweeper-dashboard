## Summary

Keep this PR open because current main still has the reported workspace default behavior and this is a plausible focused fix, but the submitted branch is not mergeable as-is due to an environment-dependent wizard test and missing after-fix real CLI proof.

## What This Changes

This PR adds a peer-workspace suggestion helper, uses it for brand-new `agents add` workspace prompts, re-exports and tests the helper, and adds a changelog entry.

## Best Possible Solution

Land one compatibility-preserving workspace default fix that gives new `agents add` prompts the documented sibling path while preserving explicit workspaces, existing-agent updates, state-dir isolation, and any intentional shared-base contract maintainers keep.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup stores `agents.defaults.workspace`, `agents add` uses the resolver value as the prompt default, and the resolver appends a non-default id while docs expect a sibling workspace.

## Solution Assessment

No, not as submitted. The wizard-only production boundary is plausible, but the branch needs the home environment stubbed in the wizard test and redacted after-fix CLI proof before merge.
