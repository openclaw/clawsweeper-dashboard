## Summary

Keep open. Current main still has the reported wizard default behavior, and this PR is the right narrow direction, but the branch is not merge-ready because one added wizard test is environment-dependent and the changelog attribution needs clarification.

## What This Changes

The branch adds a peer-workspace suggestion helper, uses it for brand-new interactive `agents add` workspace defaults, re-exports and tests the helper, and adds a changelog entry.

## Best Possible Solution

Land a focused wizard-only fix that changes the brand-new interactive default to the documented peer-level workspace path while preserving explicit per-agent workspaces, existing-agent updates, `OPENCLAW_STATE_DIR` isolation, and the #59789 shared-base runtime resolver contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: setup stores `agents.defaults.workspace`, current main’s wizard asks `resolveAgentWorkspaceDir` for a new non-default agent, and that resolver appends the agent id under the main workspace.

## Solution Assessment

No as submitted. The wizard-only strategy is the narrow maintainable fix, but the branch needs the portable test stub and changelog attribution repair before it should be treated as the canonical #71889 fix.
