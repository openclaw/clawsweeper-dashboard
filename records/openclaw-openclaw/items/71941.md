## What This Changes

The PR adds a peer-workspace suggestion helper for brand-new `agents add` wizard prompts, wires it into the add-agent command, re-exports/tests it, and adds a changelog entry.

## Best Possible Solution

Land one compatibility-preserving fix that gives brand-new `agents add` prompts the documented sibling workspace default while preserving explicit workspaces, existing-agent updates, state-dir isolation, and intentional shared-base resolver semantics.

## Reproduction Assessment

Yes. Source inspection shows setup stores `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir` for the prompt `initialValue`, and the resolver joins the fallback with a non-default agent id while the docs expect a peer-level workspace.

## Solution Assessment

No, not as submitted. The wizard-only boundary is a reasonable maintainable direction, but the PR needs a portable `HOME`/`OPENCLAW_HOME` stub in the wizard test and redacted after-fix CLI proof before merge.
