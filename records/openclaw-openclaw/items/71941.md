## Summary

## What This Changes

Adds a peer-workspace suggestion helper for brand-new `agents add` wizard prompts, wires it into the add-agent command, re-exports/tests it, and adds a changelog entry.

## Best Possible Solution

Land one compatibility-preserving fix that gives brand-new `agents add` prompts the documented peer-level workspace default while preserving explicit workspaces, existing-agent updates, state-dir isolation, and intentional shared-base layouts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup stores `agents.defaults.workspace`, the add-agent wizard uses `resolveAgentWorkspaceDir` for the prompt default, and the resolver appends a non-default agent id under that workspace.

## Solution Assessment

No as submitted. The wizard-only approach is a plausible narrow fix, but this branch needs the portable test environment stub and real behavior proof; maintainers should also choose it over the open resolver-level alternatives before landing.
