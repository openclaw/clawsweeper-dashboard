## Summary

Keep open. Current main still seeds `openclaw agents add` from the runtime workspace resolver, so the underlying #71889 bug is real and this PR takes the safer wizard-only direction, but the submitted branch is not merge-ready because the new wizard test is environment-dependent and the changelog attribution is ambiguous.

## What This Changes

This PR adds a `suggestPeerAgentWorkspaceDir` helper, uses it for brand-new `agents add` wizard workspace defaults while preserving existing-agent workspace defaults, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land one focused wizard-only fix that changes the brand-new interactive `agents add` default to the documented peer-level path for the setup-created default workspace, while preserving explicit per-agent workspaces, existing-agent updates, `OPENCLAW_STATE_DIR` isolation, and the #59789 shared-base runtime resolver contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: after setup stores the documented default in `agents.defaults.workspace`, current main’s wizard asks `resolveAgentWorkspaceDir` for a new non-default agent, and that resolver appends the agent id under the main workspace.

## Solution Assessment

No as submitted. The wizard-only approach is the narrow maintainable solution, but the branch needs the environment-dependent test fixed and the changelog attribution clarified before it should be used as the canonical #71889 fix.
