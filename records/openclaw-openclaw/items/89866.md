The PR changes strict-agentic planning-only retry detection and tests so sleep-only bash tool-use turns can retry while ordinary or shell-grouped commands remain non-retryable.

## Best Possible Solution

Classify no-progress sleep using the production metadata or raw-command source at the subscription boundary, then add coverage and proof that exercise the same metadata shape while keeping ordinary commands and shell groups non-retryable.

## Solution Assessment
