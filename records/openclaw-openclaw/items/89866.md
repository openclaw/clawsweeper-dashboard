The PR changes strict-agentic planning-only retry detection and tests so pure sleep bash/exec metadata can be treated as no-progress while ordinary commands and shell groups still count as progress.

## Best Possible Solution

Keep the retry decision tied to the production exec metadata contract by either carrying raw command data into the decision or normalizing all pure-sleep production display suffixes before counting progress.

## Solution Assessment
