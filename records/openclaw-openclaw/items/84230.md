## Summary

Keep this PR open, but it needs changes before merge: the security direction is valid, yet the current patch can silently replace existing models.json-only credentials with an unresolved marker and break provider auth on upgrade.

## What This Changes

The PR redacts plaintext provider apiKey values before serializing generated models.json and adds regression coverage for runtime-config and existing models.json plaintext keys.

## Best Possible Solution

Redact only credentials that have a recoverable runtime/source credential, or migrate existing models.json-only plaintext keys into auth profiles before replacing them with a marker; then prove both redaction and authenticated use after upgrade.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current merge mode preserves existing models.json apiKey values, while the PR redacts those values after merge. I did not run the write path because this review is read-only.

## Solution Assessment

No, not as currently written. The narrow maintainable fix should preserve or migrate models.json-only credentials while still redacting runtime/source-resolved secrets that can be recovered elsewhere.
