## Summary

Keep open: the current-main Nix config mutation bug is source-reproducible and the PR is directionally useful, but the PR is draft/protected, lacks after-fix real Nix gateway proof, and the renamed scope is broader than the Nix-only auto-enable diff.

## What This Changes

The PR makes gateway startup plugin auto-enable runtime-only in Nix mode and adds a focused regression test for that path.

## Best Possible Solution

Keep Nix/declarative startup config read-only across plugin auto-enable, generated auth, and Control UI origin seeding, with runtime-only effective config plus focused regression coverage and real Nix gateway proof.

## Reproduction Assessment

Yes, from source inspection: current main reaches `replaceConfigFile` when plugin auto-enable changes exist even with `OPENCLAW_NIX_MODE=1`. I did not run a live Nix gateway, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No. The Nix-only runtime snapshot branch is a reasonable piece, but it does not satisfy the renamed no-startup-mutation scope because generated auth and Control UI origin seeding can still write config during gateway startup.
