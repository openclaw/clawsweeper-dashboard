## Summary

Keep open: the PR has real behavior proof and green CI, but current head still has a source-proven compatibility regression for legacy plugin install metadata that should block merge.

## What This Changes

The branch makes startup-derived config changes runtime-only, adds a Nix-mode config write guard, blocks mutating CLI/chat/onboarding/doctor flows in Nix mode, and updates docs and changelog.

## Best Possible Solution

Preserve legacy plugin install records for runtime loading without reintroducing automatic openclaw.json writes, then land after owner review of the config/auth policy change.

## Reproduction Assessment

Yes. Source inspection at current head is enough: config reads strip plugins.installs with persist:false, while plugin loading only sees plugins/installs.json or cfg.plugins.installs, and the PR test confirms both are absent after load.

## Solution Assessment

No. The Nix guard and runtime-only startup direction are maintainable, but the patch should keep legacy install records available at runtime before removing the read-time persistence side effect.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
