## Summary

Keep open: the PR has sufficient real behavior proof and the security pass is clear, but it introduces a blocking compatibility regression for legacy config-only plugin install records and still needs maintainer approval for the broad config/auth policy change.

## What This Changes

The PR makes startup/read-derived config changes runtime-only and adds Nix-mode refusal guards for config, plugin lifecycle, update, onboarding, and doctor repair mutators.

## Best Possible Solution

Land the no-automatic-config-write and Nix immutability policy only after preserving legacy plugin install records at runtime without rewriting `openclaw.json`, with focused regression coverage for that compatibility path.

## Reproduction Assessment

Yes. Current main source shows the Nix read-only contract plus startup/auth config writes, and the PR body provides before/after terminal smoke output; the new legacy-install regression is source-reproducible from the PR diff because reads strip `plugins.installs` while plugin loading still depends on the persisted index or `cfg.plugins.installs`.

## Solution Assessment

No. The central Nix guard and runtime-only startup derivations are the right shape, but the PR must keep legacy `plugins.installs` records available in memory or through an explicit repair path before removing the read-time persistence side effect.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve legacy install records at runtime:** `src/config/io.ts:1491`
  - body: By passing `persist: false` here, read-time migration strips `plugins.installs` without writing those records to `plugins/installs.json`. Keep the records available in memory while still avoiding any automatic `openclaw.json` rewrite.
