## What This Changes

This PR adds `warn`, `error`, and `none` agent compaction modes, a Pi pre-prompt token guard, config schema/help updates, Pi auto-compaction disabling for those modes, a changelog entry, and an unrelated `skills/ceospace` placeholder.

## Best Possible Solution

A mergeable version should settle the mode semantics, then implement the modes through the shared prompt-pressure, hook/finalization, Pi auto-compaction, and overflow-recovery lifecycle with focused tests and real redacted behavior proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head shows `warn` appending a warning without setting the skip flag, and `none` is not wired into overflow recovery. I did not run live behavior because this was a read-only review and the PR is conflicting.

## Solution Assessment

No. The proposed direction is useful, but the current implementation duplicates prompt-pressure accounting and does not consistently enforce the new modes across prompt submission, hooks/finalization, and overflow recovery.

  - confidence: 0.9
