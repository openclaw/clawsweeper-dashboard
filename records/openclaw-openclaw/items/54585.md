## Summary

Keep open. Current main still does not implement the requested modes, and the PR remains useful but not merge-ready because the submitted guard and mode handling do not consistently satisfy the feature contract across prompt submission, overflow recovery, and current compaction budget behavior.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt token guard, disables Pi auto-compaction for the new modes, changes the reserve floor, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should define the exact semantics for each mode, implement them through the existing precheck and overflow-recovery lifecycle, preserve current default/safeguard reserve behavior, update the typed config/schema/generated help/docs together, add focused unit coverage for mode resolution and overflow paths, and keep unrelated bundled skill changes out of this PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: force a near-limit `warn` turn and the PR branch emits the warning without setting `skipPromptForCompactionGuard`, so the prompt still reaches `activeSession.prompt`; configure `none` and trigger provider overflow and the current overflow loop still calls `contextEngine.compact()`.

## Solution Assessment

No. The feature direction may be reasonable, but this patch is not the best implementation because it adds a parallel token guard and partial auto-compaction toggle instead of extending the current precheck, hook/finalization, and overflow-recovery machinery.
