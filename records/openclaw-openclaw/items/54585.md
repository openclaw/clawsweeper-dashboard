## Summary

Keep open. Current main still exposes only `default` and `safeguard` compaction modes, while this feature PR remains useful but not merge-ready because its new mode semantics are inconsistent with the runtime paths it touches and it carries unrelated skill scope.

## What This Changes

This PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt token guard, changes Pi auto-compaction and reserve-floor behavior, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should settle the mode semantics, implement them through the existing precheck and overflow-recovery lifecycle, update all config/schema/help/docs/tests consistently, keep the current reserve-floor cap, and remove unrelated skill changes.

## Reproduction Assessment

Yes. Source inspection of PR head shows `warn` emits a notice without setting the skip flag before the prompt block, and `none` only disables Pi internal auto-compaction while current main’s overflow recovery still compacts.

## Solution Assessment

No. The feature direction may be valid, but this patch adds a parallel raw token guard instead of extending the existing precheck/recovery lifecycle and leaves mode behavior inconsistent across runtime, docs, and overflow handling.

  - body: The `warn` branch emits the warning and appends assistant text, but it never sets `skipPromptForCompactionGuard`.   - confidence: 0.93

  - body: Adding `none` here disables only Pi's built-in auto-compaction. OpenClaw's separate overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as the new help text promises.
