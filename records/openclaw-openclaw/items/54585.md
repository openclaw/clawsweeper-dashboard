## Summary

Keep open: current main still has only `default` and `safeguard` compaction modes, and this PR is a real feature branch, but the proposed implementation still has blocking mode-semantics issues and lacks after-fix real behavior proof.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none` modes, adds a Pi pre-prompt context guard and auto-compaction changes, updates the changelog, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

Settle the user-facing semantics for each mode, then implement them consistently through the existing precheck and overflow-recovery lifecycle with focused config/runtime/docs tests and no unrelated bundled skill additions.

## Reproduction Assessment

Yes. Source inspection of PR head is enough to reproduce the review issues: `warn` warnings do not stop prompt submission, and `none` disables only Pi auto-compaction while OpenClaw overflow recovery can still compact.

## Solution Assessment

No. The direction may be useful, but this patch adds a parallel guard and incomplete mode wiring instead of extending the established precheck and overflow-recovery paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2932`
  - body: The PR text and help say `warn` stops before the model, but this branch only sends/appends the warning and leaves `skipPromptForCompactionGuard` false. Execution then reaches `activeSession.prompt(...)`, so near-limit `warn` turns still call the provider.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Adding `none` here disables only Pi's internal auto-compaction. OpenClaw's separate overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit.
  - confidence: 0.9
- **[P2] Route the guard through shared prompt-pressure accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2878-2921`
  - body: The new guard estimates tokens and thresholds beside `shouldPreemptivelyCompactBeforePrompt`, missing the current safety margin, effective reserve cap, unwindowed-message check, and tool-result recovery routing. Extend the shared precheck path so mode behavior uses the same accounting.
  - confidence: 0.86
- **[P3] Remove the unrelated bundled skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds `skills/ceospace/SKILL.md`, which creates a separate bundled skill surface with model invocation enabled. Split or remove it so the PR only changes compaction behavior.
  - confidence: 0.8
