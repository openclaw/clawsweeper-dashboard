## Summary

Keep open: the feature is not implemented on current main, and the PR still has blocking runtime semantics issues plus missing real behavior proof before merge.

## What This Changes

The PR adds `warn`, `error`, and `none` agent compaction modes across config/schema/help, adds a Pi pre-prompt context guard, changes auto-compaction disabling, updates the changelog, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

Settle the user-facing semantics for each mode, then implement them through the existing precheck and overflow-recovery lifecycle with focused config/runtime/docs tests and no unrelated bundled skill additions.

## Reproduction Assessment

Yes. Source inspection of PR head is enough to reproduce the review issues: warn-mode warnings do not stop prompt submission, and `none` only disables Pi auto-compaction while OpenClaw overflow recovery can still compact.

## Solution Assessment

No. The direction may be useful, but this patch adds a parallel guard and incomplete mode wiring instead of extending the established precheck and overflow-recovery paths.
