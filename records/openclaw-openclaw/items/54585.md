## Summary

Keep open: current main has not implemented the new `warn`, `error`, and `none` compaction modes, and the external PR still has blocking runtime semantics issues plus missing real behavior proof. The useful direction needs maintainer product agreement and author rework rather than cleanup close or an autonomous replacement lane.

## What This Changes

This PR adds `warn`, `error`, and `none` agent compaction modes across config/schema/help, adds a Pi pre-prompt context guard, changes Pi auto-compaction disabling/reserve behavior, updates the changelog, and adds a `ceospace` skill placeholder.

## Best Possible Solution

Settle the public semantics for each mode, then implement them consistently through the existing precheck, overflow-recovery, hook, finalization, config/schema/docs, and focused test paths, with unrelated skill work split out.

## Reproduction Assessment

Yes. Source inspection of PR head is enough: warn-mode leaves `skipPromptForCompactionGuard` false before `activeSession.prompt`, and `none` only changes Pi's guard while overflow recovery still calls `contextEngine.compact`.

## Solution Assessment

No. The direction may be useful, but this patch adds parallel and incomplete mode wiring instead of extending the established prompt-pressure and overflow-recovery lifecycle after semantics are agreed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2924-2932`
  - body: The `warn` branch emits the warning and appends it to `assistantTexts`, but it never sets `skipPromptForCompactionGuard`. Execution therefore continues into `activeSession.prompt(...)`, so a near-limit warn turn still calls the model despite the PR describing this mode as stopping before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Adding `none` here disables only Pi's internal auto-compaction. The separate overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as the new help text promises.
  - confidence: 0.9
- **[P2] Route the guard through shared prompt-pressure accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2864-2921`
  - body: The new guard estimates and thresholds tokens beside `shouldPreemptivelyCompactBeforePrompt`, so it can miss the shared safety margin, effective reserve cap, unwindowed-message pressure, and truncation routing. Reuse or extend the existing precheck so warn/error behavior matches current overflow prevention.
  - confidence: 0.86
- **[P3] Remove the unrelated bundled skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds a `skills/ceospace` placeholder with model invocation enabled. That creates a separate bundled skill surface unrelated to compaction and should be split or removed from this branch.
  - confidence: 0.8
