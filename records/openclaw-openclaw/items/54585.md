## Summary

Keep open: current main still does not implement the requested modes, and the PR has blocking mode-semantics issues plus missing real behavior proof.

## What This Changes

The branch adds `warn`, `error`, and `none` compaction modes across config/schema/help, adds a Pi pre-prompt context guard and auto-compaction changes, updates the changelog, and adds a `ceospace` skill placeholder.

## Best Possible Solution

Settle the public semantics for each new mode, then implement them through the existing precheck and overflow-recovery lifecycle with config/docs/tests aligned and no unrelated bundled skill changes.

## Reproduction Assessment

Yes. Source inspection of PR head is enough to reproduce the review issues: `warn` does not stop prompt submission, and `none` only changes Pi auto-compaction while OpenClaw overflow recovery can still compact.

## Solution Assessment

No. The direction may be useful, but this patch adds parallel and incomplete mode wiring instead of extending the established precheck, overflow recovery, hook, and finalization paths after semantics are agreed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The `warn` branch emits the warning via `onBlockReply` and appends it to `assistantTexts`, but it never sets the guard skip flag. Execution therefore continues into the later prompt path, so a near-limit warn turn still calls the model despite the PR describing this mode as stopping before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Adding `none` to the Pi auto-compaction disable check does not disable OpenClaw's separate overflow-recovery loop, which still calls `contextEngine.compact()` after model-limit errors. A session configured with `mode: "none"` can therefore still auto-compact and retry instead of failing at the model limit as documented.
  - confidence: 0.9
- **[P2] Route the guard through shared prompt-pressure accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2861-2921`
  - body: The new guard estimates raw `activeSession.messages` beside the existing precheck path, so it misses the shared safety margin, effective reserve cap, unwindowed-message pressure, and tool-result sanitization/recovery routing. Reuse or extend `shouldPreemptivelyCompactBeforePrompt` so the new modes use the same accounting as current overflow prevention.
  - confidence: 0.86
- **[P3] Remove the unrelated bundled skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds `skills/ceospace/SKILL.md`, creating a separate bundled skill surface with model invocation enabled. Split or remove it so this branch only changes compaction behavior.
  - confidence: 0.8
