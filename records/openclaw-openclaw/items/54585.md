## Summary

Keep this PR open, but it is not merge-ready: current main still has only the existing compaction modes, and the PR head has blocking runtime gaps in the new `warn` and `none` semantics plus missing real behavior proof.

## What This Changes

The PR adds `warn`, `error`, and `none` agent compaction modes, a Pi pre-prompt token guard, schema/help/changelog updates, Pi auto-compaction changes, and an unrelated `ceospace` skill placeholder.

## Best Possible Solution

Settle the public semantics for each mode, then implement them through the existing prompt-pressure, overflow-recovery, hook/finalization, config/schema/docs, and focused test paths, with unrelated skill work removed from this branch.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows `warn` appends a notice without blocking `activeSession.prompt(...)`, and `none` only changes Pi's guard while overflow recovery still calls `contextEngine.compact(...)`. This is a feature PR, so there is no existing-bug reproduction on current main.

## Solution Assessment

No. The direction may be useful, but this patch adds parallel and incomplete mode wiring instead of extending the established prompt-pressure and overflow-recovery lifecycle after the mode semantics are agreed.

  - confidence: 0.86

  - body: This compaction PR also adds a `skills/ceospace` placeholder with model invocation enabled. That creates a separate bundled skill surface unrelated to compaction and should be split or removed from this branch.
