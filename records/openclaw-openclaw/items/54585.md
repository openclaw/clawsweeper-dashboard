## Summary

Keep open: current main still does not implement the requested modes, but the PR has blocking runtime semantics issues, lacks external real behavior proof, and needs maintainer/product direction before merge.

## What This Changes

The PR adds `warn`, `error`, and `none` agent compaction modes across config/schema/help and adds a Pi pre-prompt context guard plus an unrelated bundled skill placeholder.

## Best Possible Solution

Settle the mode semantics, then implement them through the existing precheck and overflow-recovery lifecycle with focused config/runtime/docs tests and no unrelated bundled skill additions.

## Reproduction Assessment

Yes. Source inspection of PR head reproduces the review issues: warn-mode warnings do not stop prompt submission, and current overflow recovery still auto-compacts independently of Pi’s internal auto-compaction flag.

## Solution Assessment

No. The feature direction may be useful, but this patch adds a parallel guard and incomplete `none` semantics instead of extending the established prompt-pressure and overflow-recovery paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2924-2932`

  - confidence: 0.93

  - body: Returning true here disables only Pi’s internal auto-compaction. OpenClaw’s separate overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as documented.
  - confidence: 0.89
- **[P2] Reuse the existing precheck budget path:** `src/agents/pi-embedded-runner/run/attempt.ts:2878-2921`
  - body: The new guard recomputes token pressure and thresholds beside `shouldPreemptivelyCompactBeforePrompt`, so it misses the shared safety margin, effective reserve clamping, unwindowed-message handling, and recovery routing. Implement warn/error behavior through the established precheck path.
  - confidence: 0.84

  - body: This compaction PR also adds a `ceospace` bundled skill placeholder with model invocation enabled. That is unrelated user-visible skill surface and should be removed from this PR or split into separately reviewed work.
