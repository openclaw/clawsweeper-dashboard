## Summary

Keep open: current main still exposes only the existing `default`/`safeguard` compaction modes, and the PR branch is not merge-ready because the new `warn` and `none` behaviors conflict with the proposed docs/runtime contract.

## What This Changes

The PR extends agent compaction config with `warn`, `error`, and `none`, adds a Pi pre-prompt context guard, adjusts auto-compaction settings, updates config help/changelog, and adds a `ceospace` skill placeholder.

## Best Possible Solution

Define the mode semantics first, then implement them through the existing precheck and overflow-recovery lifecycle with focused tests and docs, keeping unrelated skill additions out of this PR.

## Reproduction Assessment

Yes. Source inspection is enough: the PR's latest `warn` branch appends a notice without setting the guard skip flag, and current overflow recovery still calls `contextEngine.compact()` even though the PR only disables Pi internal auto-compaction.

## Solution Assessment

No. The feature direction may be useful, but the patch adds a parallel raw token guard instead of extending the established precheck/recovery path and leaves the new modes inconsistent across runtime and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The latest `warn` branch emits the warning through `onBlockReply` and `assistantTexts.push`, but it no longer sets `skipPromptForCompactionGuard`. Near-limit warn-mode turns therefore continue into the prompt submission path even though this mode is described as stopping before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Disabling Pi auto-compaction here does not disable OpenClaw's separate overflow-recovery loop, which still calls `contextEngine.compact()` after model-limit errors. A session configured with `mode: "none"` can still auto-compact and retry instead of failing at the model limit as the new help text promises.
  - confidence: 0.89
- **[P2] Reuse the existing precheck token accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2904-2916`
  - body: The new guard estimates raw `activeSession.messages` directly, bypassing the existing sanitized accounting that strips `toolResult.details` and applies the safety margin. Large non-model-visible details can false-trigger `warn`/`error`, while estimator undercounts can miss real overflow; thread the mode behavior through the existing precheck path instead.
  - confidence: 0.86
- **[P2] Preserve the reserve floor default:** `src/agents/pi-settings.ts:7`
  - body: Lowering `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` from 20,000 to 1,000 weakens the intended headroom for large-context models. Current main already caps the floor against small context windows, so the default should not be reduced to compensate for the new guard threshold.
  - confidence: 0.84
- **[P3] Remove the unrelated ceospace skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds a bundled `ceospace` skill placeholder with model invocation enabled. That is unrelated user-facing skill surface and should be removed or split into its own reviewed change.
  - confidence: 0.79
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:10`
  - body: The user-facing changelog entry lacks the repo-required `Thanks @...` attribution for added entries. Credit the human contributor or hold the entry until the feature is ready to land.
