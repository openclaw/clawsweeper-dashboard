## Summary

Keep this PR open for maintainer review: current main has not implemented the requested compaction modes, and the PR still has blocking runtime semantics issues plus missing posted real behavior proof.

## What This Changes

The PR adds `warn`, `error`, and `none` agent compaction modes across config/schema/help, adds a Pi pre-prompt context guard, changes auto-compaction settings, updates the changelog, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

Settle the user-facing mode semantics first, then implement them consistently through the existing precheck and overflow-recovery lifecycle with focused config/runtime/docs tests and no unrelated skill additions.

## Reproduction Assessment

Yes. Source inspection of PR head is enough to reproduce the review issues: warn-mode warnings do not stop prompt submission, and current overflow recovery still auto-compacts independently of Pi's internal auto-compaction flag.

## Solution Assessment

No. The feature direction may be useful, but this patch adds a parallel guard and incomplete `none` semantics instead of extending the established precheck and overflow-recovery paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2924-2932`
  - body: The `warn` branch sends the warning through `onBlockReply` and appends it to `assistantTexts`, but it never sets `skipPromptForCompactionGuard`. The later `if (!skipPromptForCompactionGuard)` block still calls `activeSession.prompt(...)`, so a near-limit warn turn can still hit the provider even though this mode is described as stopping before overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Disabling Pi auto-compaction for `mode === "none"` does not disable OpenClaw's separate overflow-recovery loop, which still calls `contextEngine.compact()` after model-limit errors. Thread the resolved mode into overflow recovery and skip that compaction path for `none`.
  - confidence: 0.9
- **[P2] Reuse the existing precheck budget path:** `src/agents/pi-embedded-runner/run/attempt.ts:2878-2921`
  - body: The new guard recomputes tokens and thresholds beside `shouldPreemptivelyCompactBeforePrompt`, so it misses the existing safety margin, effective reserve clamping, unwindowed-message handling, and tool-result recovery routing. Implement warn/error behavior through that precheck path so the modes share the same prompt-pressure accounting as current main.
  - confidence: 0.84
- **[P3] Remove the unrelated ceospace skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds a bundled `ceospace` skill placeholder with model invocation enabled. That is unrelated user-visible skill surface and should be removed from this PR or split into a separate reviewed change.
  - confidence: 0.79
