## Summary

Keep open. Current main does not implement the requested modes, and the PR is still not merge-ready because the latest patch leaves the proposed mode behavior inconsistent with the documented contract and existing overflow recovery.

## What This Changes

This PR adds `warn`, `error`, and `none` compaction modes to agent config/schema/help, adds a Pi pre-prompt token guard, changes Pi auto-compaction handling and reserve defaults, updates the changelog, and adds a `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should settle the mode semantics first, then implement them consistently through exported config types, strict/generated schemas, help/docs, existing precheck and overflow-recovery lifecycles, hooks/finalization, and focused tests, without unrelated skill additions.

## Reproduction Assessment

Yes. Source inspection of the PR patch shows `warn` emits/appends a notice without setting the guard skip flag, and current-main overflow recovery still calls `contextEngine.compact()` while the PR only disables Pi internal auto-compaction for `none`.

## Solution Assessment

No. The feature direction may be valid, but this patch adds a parallel raw token guard instead of extending the existing precheck/recovery lifecycle and leaves mode behavior inconsistent across runtime, docs, and recovery paths.

## Review Findings

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The latest `warn` branch emits the warning through `onBlockReply` and appends assistant text, but it no longer sets `skipPromptForCompactionGuard`. Near-limit warn-mode turns therefore still reach the prompt submission block even though the PR body and help text describe stopping before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Disabling Pi auto-compaction here does not disable OpenClaw's separate overflow-recovery loop, which still calls `contextEngine.compact()` after model-limit errors. A session configured with `mode: "none"` can still auto-compact and retry instead of failing at the model limit as the new help text promises.
  - confidence: 0.89
- **[P2] Preserve the current reserve-floor cap:** `src/agents/pi-settings.ts:7`
  - body: The PR lowers `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` from 20,000 to 1,000, but current main already caps the 20,000 floor for small context windows. Lowering the global default weakens the intended reserve headroom for large-context models.
  - confidence: 0.84
- **[P3] Remove the unrelated ceospace skill placeholder:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds a bundled `ceospace` skill placeholder with model invocation enabled. That creates unrelated user-facing skill surface and should be removed or split into its own reviewed change.
