## Best Possible Solution

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head's `warn` path appends/sends a notice without setting the skip flag before the later prompt block, and `none` does not thread into overflow recovery. This is a feature PR, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but this implementation adds parallel and partial mode wiring instead of extending the existing shared prompt-pressure and overflow-recovery lifecycle after the mode semantics are agreed.
