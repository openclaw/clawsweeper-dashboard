## Summary

Keep open. Current main still exposes only the existing two compaction modes, and the latest PR diff is not mergeable as-is because it leaves new mode semantics inconsistent across prompt submission, overflow recovery, token accounting, reserve defaults, changelog, and unrelated skill surface.

## What This Changes

This PR adds `warn`, `error`, and `none` agent compaction modes, a Pi pre-prompt token guard, Pi auto-compaction disabling for those modes, config schema/help/changelog updates, and a bundled `ceospace` skill placeholder.

## Best Possible Solution

Define the mode semantics first, then rework the branch against current main through the existing precheck and overflow-recovery lifecycle with focused tests/docs, keeping unrelated skill additions out of the compaction PR.

## Reproduction Assessment

Yes. Source inspection is sufficient for the review findings: the latest diff leaves `warn` without setting the skip flag, and current overflow recovery still compacts independently of the PR's Pi auto-compaction toggle.

## Solution Assessment

No. The feature direction may be useful, but the patch adds a parallel guard and partial mode handling instead of extending the established precheck/recovery path consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2932`
  - body: The `warn` branch now sends/appends the warning but never sets `skipPromptForCompactionGuard`. Near-limit warn-mode turns therefore continue into the later `activeSession.prompt(...)` block, contradicting the PR body and help text that say warn mode stops before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:110`
  - body: Adding `none` here disables only Pi's internal auto-compaction. OpenClaw's separate overflow-recovery loop still calls `contextEngine.compact()` after context overflow, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as documented.
  - confidence: 0.9
- **[P2] Use the existing precheck accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2878-2890`
  - body: The new guard estimates its own token total and threshold instead of threading the new modes through `shouldPreemptivelyCompactBeforePrompt`, which already applies sanitized message accounting, safety margin, reserve clamping, unwindowed pressure checks, and recovery routing. This can produce false warnings/errors or missed overflows that the existing path already handles.
  - confidence: 0.86
- **[P2] Preserve the reserve floor default:** `src/agents/pi-settings.ts:4`
  - body: Lowering `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` from 20,000 to 1,000 weakens the default headroom for large-context models. Current main already caps the floor against small context windows, so the default should not be reduced to work around the new guard threshold.
  - confidence: 0.84
- **[P3] Remove the unrelated bundled skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR adds a bundled `ceospace` skill placeholder with model invocation enabled. That is unrelated user-facing skill surface and should be removed or split into a separate reviewed change.
  - confidence: 0.8
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:11`
  - body: The user-facing changelog entry is missing the repo-required human `Thanks @...` attribution for added entries. Credit the contributor or hold the entry until the feature is ready to land.
  - confidence: 0.82
