## Summary

Keep open: current main still has not implemented the new compaction modes, and the PR head is not merge-ready because warn/none semantics still contradict the proposed behavior and the branch carries unrelated skill surface.

## What This Changes

The PR adds `warn`, `error`, and `none` agent compaction modes, a Pi pre-prompt token guard, config/schema/help updates, a changelog entry, and a bundled `ceospace` skill placeholder.

## Best Possible Solution

Settle the mode semantics first, then implement them through the existing precheck and overflow-recovery lifecycle with aligned config schemas, docs/help, tests, and no unrelated skill changes.

## Reproduction Assessment

Yes. Source inspection of PR head shows the warn branch emits a warning without setting the skip flag, and `none` only disables Pi internal auto-compaction while overflow recovery still calls `contextEngine.compact()`.

## Solution Assessment

No. The feature direction may be useful, but this patch adds a parallel guard and leaves behavior inconsistent; the safer path is to extend the existing precheck/recovery lifecycle and public config contract together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2924-2932`
  - body: The `warn` branch emits the warning and appends assistant text, but it never sets `skipPromptForCompactionGuard`. Execution therefore reaches the later prompt block and can still call the model even though the new mode is documented as stopping before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Adding `none` here disables only Pi's built-in auto-compaction. The separate overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit.
  - confidence: 0.89
- **[P2] Preserve the current reserve-floor cap:** `src/agents/pi-settings.ts:4`
  - body: Lowering the default reserve floor from 20,000 to 1,000 weakens the intended headroom for large-context models. Current main already caps the 20,000 floor for small context windows, so this change is no longer the right fix for false warnings on small models.
  - confidence: 0.86
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:10`
  - body: This user-facing feature entry omits the required contributor attribution. Add the credited non-bot GitHub handle for this PR, or remove the entry until the feature is ready to ship.
  - confidence: 0.84
- **[P3] Remove the unrelated bundled skill:** `skills/ceospace/SKILL.md:1-5`
  - body: This compaction PR also adds a `ceospace` bundled skill placeholder with model invocation enabled. That creates unrelated user-facing skill surface and should be split out or removed from this PR.
  - confidence: 0.8
