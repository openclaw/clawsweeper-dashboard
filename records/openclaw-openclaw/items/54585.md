## Summary

Keep this PR open for maintainer/product review rather than cleanup-close: current main still has only the existing two compaction modes, and the proposed branch has runtime semantics blockers plus missing real behavior proof.

## What This Changes

This PR adds `warn`, `error`, and `none` agent compaction modes, config/schema/help updates, a Pi pre-prompt context guard, auto-compaction changes, a changelog entry, and an unrelated bundled skill placeholder.

## Best Possible Solution

Settle the public semantics for each new mode, then implement them through the existing prompt-pressure, overflow-recovery, hook/finalization, config/schema/docs, and focused test paths, with unrelated skill work split out.

## Reproduction Assessment

Yes for the PR defects by source inspection: the branch's final `warn` path appends a notice without blocking prompt submission, and `none` does not thread into overflow recovery. This is a feature PR, so there is no existing-bug reproduction on current main.

## Solution Assessment

No. The direction may be useful, but the branch adds parallel and incomplete mode wiring instead of extending the established prompt-pressure and overflow-recovery lifecycle after the mode semantics are agreed.
