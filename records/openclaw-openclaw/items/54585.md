## Summary

This PR should stay open: current main still only supports `default` and `safeguard`, while the branch’s new `warn`/`error`/`none` feature remains merge-blocked by inconsistent runtime semantics, missing real behavior proof, and unrelated skill scope.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt context guard, disables Pi auto-compaction for those modes, lowers reserve-floor behavior, updates the changelog, and adds a `skills/ceospace` placeholder.

## Best Possible Solution

Define the public semantics for each mode first, then implement them through the existing prompt-pressure accounting, overflow recovery, hooks/finalization, exported config/schema/docs, and focused tests, with unrelated skill work split out.

## Reproduction Assessment

Yes for the PR defects by source inspection: the branch’s `warn` path can emit a notice without stopping prompt submission, and `none` is not wired into the separate overflow recovery loop. This is a feature PR, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but the branch implements a parallel guard and partial mode wiring instead of extending the established prompt-pressure, overflow recovery, hook, and finalization paths after the mode semantics are settled.
