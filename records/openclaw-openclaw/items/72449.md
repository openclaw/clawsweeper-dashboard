- **[P1] Preserve channel changes before skipping updates:** `src/cli/update-cli/update-command.ts:1524`
  - body: This early return fires whenever the installed and target package versions match and no explicit tag was passed. On current main, requested update-channel changes are persisted later via `persistRequestedUpdateChannel`, and the existing no-op detection explicitly avoids treating same-version installs as already-current when `requestedChannel !== storedChannel`. With this branch, `openclaw update --channel beta` can exit before writing the user's channel choice, so the settings change is silently lost.
  - confidence: 0.9
- **[P3] Use the resolved channel label in no-op output:** `src/cli/update-cli/update-command.ts:1527`
  - body: The new user-facing and JSON messages always say `(latest)`, but this path also runs for channel-resolved package updates such as beta when no explicit tag was passed. That makes the CLI report the wrong channel and can mislead users trying to verify a beta or stable channel update.
  - confidence: 0.78
