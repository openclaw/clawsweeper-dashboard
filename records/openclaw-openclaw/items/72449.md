## Summary

Keep open. The cron parser fix direction is useful, but this PR also introduces an unrelated same-version update shortcut that conflicts with the maintained package refresh contract confirmed in current main source, docs, and tests.

## What This Changes

The PR widens `parseAbsoluteTimeMs` for nullish input and adds parser tests, while also adding an update CLI early return that skips same-version package updates.

## Best Possible Solution

Keep the nullish cron parser guard with focused parser/list regression coverage and an Unreleased changelog entry, while preserving current same-version package refresh behavior.

## Reproduction Assessment

Yes. A malformed persisted cron row with `schedule: { kind: "at" }` can reach `printCronList` -> `formatSchedule` -> `parseAbsoluteTimeMs(undefined)` on current main; the PR-introduced update regression is covered by same-version package update tests that expect install and follow-up work.

## Solution Assessment

No. The parser guard is the right narrow fix, but bundling the update early return is not the best solution because it contradicts the current update contract and should be removed or handled in a separate design change.
