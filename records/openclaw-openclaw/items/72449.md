## Best Possible Solution

Land a focused cron parser/formatter guard with regression coverage, remove the unrelated update shortcut, and require redacted terminal or log proof of the cron behavior before merge.

Yes, by source inspection: a malformed cron row with `schedule.kind === "at"` and no string `at` value can reach `formatSchedule`, `formatIsoMinute`, and then `parseAbsoluteTimeMs(undefined)`. I did not execute the CLI path in this read-only review.

## Solution Assessment

No. The cron parser guard is a reasonable narrow fix, but the update-command shortcut is unrelated and conflicts with the current update contract; the safer path is to remove that hunk and prove the cron behavior.
