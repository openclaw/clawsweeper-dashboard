## Summary

Keep open. Current main still drops the subagent task instructions when an agent has `systemPromptOverride`: the task-bearing `extraSystemPrompt` is passed only to prompt builders that are bypassed by the override branches, and the related repair PR was closed unmerged.

## What This Changes

Subagent runs under agents with `systemPromptOverride` should still receive the per-run `extraSystemPrompt` that carries delegated `## Your Role` task instructions.

## Best Possible Solution

Define and document the override composition contract, then preserve subagent `extraSystemPrompt` across embedded attempts, compaction retries, and CLI preparation with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: spawn a native subagent under an agent with `systemPromptOverride`; the delegated task is only in `extraSystemPrompt`, and the override branches skip the builders that append it.

## Solution Assessment

No, current main is not solving the issue. The reporter's append approach is plausible, but the maintainable fix needs maintainer approval on whether runtime task context is allowed to compose with a full prompt override.
