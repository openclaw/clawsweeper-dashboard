## Summary

Keep open. Current main still has the reported override-first prompt assembly in the embedded attempt, compaction retry, and CLI runner paths, so a configured systemPromptOverride can bypass the builder branch that carries the subagent task in extraSystemPrompt. The prior fix PR #73637 is closed unmerged, and #73656 tracks a related but distinct visible-bootstrap fallback rather than this functional task-delivery bug.

## What This Changes

The issue asks OpenClaw to preserve agent-level systemPromptOverride while still delivering task-specific subagent instructions from params.extraSystemPrompt.

## Best Possible Solution

Define and implement a prompt composition contract that preserves delegated subagent task delivery for override-backed agents without accidentally granting unreviewed system-role authority to user task text. The implementation should cover embedded attempts, compaction retry, and CLI runner prompt preparation, with regression tests for no override, override only, and override plus subagent task behavior; docs should change if systemPromptOverride semantics are narrowed or extended.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure an agent with systemPromptOverride and spawn a native subagent; the task is sent as extraSystemPrompt, but all relevant prompt builders return the override before reaching the branch that appends extraSystemPrompt.

## Solution Assessment

No for the reporter's raw concatenation patch as a final answer. It is directionally useful for task delivery, but #73637 surfaced a prompt-authority concern; a safer fix needs maintainer approval on whether the delegated task belongs in system prompt composition or a lower-privilege runtime message contract.
