## Summary

Keep open. Current main still drops the delegated subagent task whenever `systemPromptOverride` resolves, because the override branch bypasses the prompt builders that append `extraSystemPrompt`; the prior fix PR was closed unmerged after a concrete prompt-authority concern, so this still needs maintainer design review rather than cleanup closure.

## What This Changes

Ensure subagent task-specific `extraSystemPrompt` instructions are still delivered when an agent is configured with `systemPromptOverride`.

## Best Possible Solution

Preserve subagent task delivery under `systemPromptOverride` with an explicit prompt-authority contract: preferably route task-bearing runtime instructions through a lower-privilege message path, or deliberately document and test a safe composition rule across embedded attempt, compaction retry, and CLI runner surfaces.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `spawnSubagent` sends the task block as `extraSystemPrompt`, but all three relevant prompt assembly paths return `systemPromptOverride` before reaching the builder that appends it.

## Solution Assessment

No for the reporter's simple append patch as the final solution. It restores task delivery, but the closed PR #73637 discussion showed that raw system-prompt concatenation has prompt-injection/authority concerns; maintainers should choose the prompt contract first.
