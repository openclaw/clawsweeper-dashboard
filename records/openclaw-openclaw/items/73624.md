## Summary

Keep open. Current main still has the reported override-first prompt assembly in the embedded attempt, compaction retry, and CLI runner paths, so a configured `systemPromptOverride` still bypasses the builder branch that carries the delegated subagent task in `extraSystemPrompt`. The prior fix PR #73637 is closed unmerged after a prompt-authority security concern, while #73656 tracks a separate visible-bootstrap fallback issue.

## What This Changes

The issue asks for subagent runs under an agent-level `systemPromptOverride` to preserve task-specific delegated instructions carried through `params.extraSystemPrompt`.

## Best Possible Solution

Preserve delegated subagent task delivery across embedded attempt, compaction retry, and CLI runner prompt assembly while keeping user-controlled task text out of system-role authority unless maintainers explicitly accept and document that composition contract. The final fix should add regression coverage for no override, override-only, and override-plus-task behavior, plus docs if `systemPromptOverride` semantics change.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `spawnSubagent` sends the delegated task through `extraSystemPrompt`, but all three prompt assembly paths return `resolveSystemPromptOverride(...)` before reaching the builder that appends `extraSystemPrompt`.

## Solution Assessment

No for the local raw-concatenation patch as the final answer. It restores task delivery, but #73637 showed a concrete prompt-authority concern; a safer fix needs maintainer agreement on either lower-privilege task delivery or a carefully documented and tested composition rule.
