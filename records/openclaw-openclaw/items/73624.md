## Summary

Keep open. Current main still drops the task-bearing `extraSystemPrompt` whenever `systemPromptOverride` wins the prompt-builder branch, so the reported subagent task loss remains valid.

## What This Changes

The issue requests preserving task-specific subagent instructions when an agent-level `systemPromptOverride` is configured.

## Best Possible Solution

Define the override composition contract, then preserve subagent task context across embedded attempts, compaction retries, and CLI preparation with focused regression coverage and docs alignment.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent under an agent with `systemPromptOverride`; the task is only in `extraSystemPrompt`, and all three prompt assembly paths skip the builder that would append it.

## Solution Assessment

No. Current main still fails this path, and the direct append fix may be correct but needs maintainer approval because the docs currently define `systemPromptOverride` as a full replacement.
