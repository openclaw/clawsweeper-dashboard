## Summary

Keep open. Current main still has the reported prompt assembly gap: native subagent task text is carried through `extraSystemPrompt`, but override-backed runner paths still short-circuit before the builders that append that field. The related fix PR was closed unmerged, and the linked visible-message issue is related but distinct, so there is no implemented or canonical superseding fix to close against.

## What This Changes

The issue reports that subagent runs under an agent-level `systemPromptOverride` lose the delegated task carried in `extraSystemPrompt`, leaving the child with only the bootstrap message.

## Best Possible Solution

Define an explicit prompt composition contract for override-backed runs, then implement it consistently across embedded attempts, compaction retries, and CLI runner preparation with regression coverage proving subagent `## Your Role` task delivery survives `systemPromptOverride`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: spawn a native subagent under an agent with `systemPromptOverride`; `spawnSubagent` sends the task only as `extraSystemPrompt`, while the override-backed runner paths skip the builders that would append it.

## Solution Assessment

No, current main is not solving the issue. The reporter's append approach is plausible, but the best final fix should first settle the prompt-authority contract surfaced by the closed related PR, then encode that contract in one shared helper and tests.
