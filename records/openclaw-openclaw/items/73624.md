## Summary

Keep open. Current main still has the reported prompt assembly bug: native subagent tasks are carried through `extraSystemPrompt`, but every override-backed runner path still short-circuits before the builder that appends that field. The related fix PR was closed unmerged after a concrete prompt-authority concern, so this should remain a maintainer design follow-up rather than cleanup closure.

## What This Changes

The issue requests preserving agent-level `systemPromptOverride` while still delivering task-specific subagent instructions from `params.extraSystemPrompt`.

## Best Possible Solution

Define and implement an explicit prompt-authority contract for runtime task instructions under `systemPromptOverride`, then apply it consistently to embedded attempts, compaction retries, and CLI runner preparation with regression coverage for override plus subagent task delivery.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: spawn a native subagent under an agent with `systemPromptOverride`; `spawnSubagent` sends the task only as `extraSystemPrompt`, while the override-backed runner paths skip the builders that would append it.

## Solution Assessment

No, current main is not solving the issue. The reporter's append approach is a plausible narrow repair but not proven to be the best final design because the prior unmerged PR surfaced prompt-authority concerns that maintainers should settle first.
