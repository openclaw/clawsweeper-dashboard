## Summary

Keep open. Current main still has the reported prompt-composition path: subagent task text is carried in extraSystemPrompt, but systemPromptOverride short-circuits the builders that append it; the only linked fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to preserve task-specific subagent instructions when an agent also has systemPromptOverride configured.

## Best Possible Solution

Define the override composition contract, then preserve task-bearing runtime context across embedded attempts, compaction retries, and CLI preparation with focused regression tests and docs aligned to the chosen semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent under an agent with systemPromptOverride; the task is only in extraSystemPrompt, and the override branch skips every builder that would append it.

## Solution Assessment

No. The local append-helper approach likely fixes the immediate loss, but it is not clearly the best maintainable fix until maintainers decide how full prompt overrides compose with task-bearing runtime context.
