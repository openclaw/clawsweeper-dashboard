## Summary

Keep open. Current main still routes the delegated subagent task through `extraSystemPrompt`, then drops it whenever `systemPromptOverride` short-circuits prompt building; the related fix PR was closed unmerged, and the remaining fix needs maintainer prompt-authority review because the public config docs define overrides as full prompt replacements.

## What This Changes

Ensure subagent runs under an agent-level `systemPromptOverride` still receive the delegated task carried in `params.extraSystemPrompt`.

## Best Possible Solution

Define the override composition contract, then update embedded, compact, and CLI prompt assembly so runtime task context survives overrides with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows native subagent spawn sends the delegated task only via `extraSystemPrompt` while the visible user text points to hidden `Your Role`; with a non-empty `systemPromptOverride`, the runner uses the override and never calls the builders that append `extraSystemPrompt`.

## Solution Assessment

Unclear. A direct append helper would restore task delivery, but because current docs define `systemPromptOverride` as a full replacement, maintainers should first decide whether runtime task context is appended to overrides or carried through a separate contract.
