## Summary

Keep open. Current main still single-sources native subagent task text into `extraSystemPrompt`, and the `systemPromptOverride` prompt-assembly paths do not deliver that text; the related append PR was closed unmerged because the obvious concatenation fix needs prompt-authority review.

## What This Changes

The issue asks OpenClaw to preserve task-specific subagent instructions when the target agent has `systemPromptOverride` configured.

## Best Possible Solution

Preserve override authority while delivering subagent task context through an explicit runtime or message seam across embedded attempt, compaction, and CLI paths, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent for an agent with `systemPromptOverride`; the delegated task is only in `extraSystemPrompt`, while the override branches build the prompt without that field.

## Solution Assessment

No. The local/PR append approach is not clearly the best fix because the docs make the override authoritative; the safer direction is a maintainer-approved separate delivery contract for runtime task context.
