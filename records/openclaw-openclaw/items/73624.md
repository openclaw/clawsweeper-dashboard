## Summary

Keep open. Current main still has a source-reproducible path where native subagent task text is single-sourced into `extraSystemPrompt`, but `systemPromptOverride` paths can still bypass the builder that inserts that prompt; the closed append PR did not land and the remaining fix shape needs maintainer prompt-authority review.

## What This Changes

Ensure subagent task instructions carried through `params.extraSystemPrompt` are still delivered when the target agent has `systemPromptOverride` configured.

## Best Possible Solution

Define a maintainer-approved delivery contract that preserves override authority while reliably delivering subagent role/task context across normal attempts, compaction retries, and CLI-backed runs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent under an agent with `systemPromptOverride`; the task is placed in `extraSystemPrompt`, while override branches bypass the builders that include that field.

## Solution Assessment

No. The local/closed direct-append approach is not clearly the best fix because it changes the documented override authority boundary; a separate or explicitly approved runtime delivery contract is safer.
