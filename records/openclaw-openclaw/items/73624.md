## Summary

Keep open. Current main still drops the subagent role/task context when an agent-level prompt override is present, and the closed append-style PR shows the remaining fix needs maintainer prompt-authority review rather than cleanup closure.

## What This Changes

The issue asks for subagent task instructions carried in `extraSystemPrompt` to survive agent-level `systemPromptOverride` prompt assembly.

## Best Possible Solution

Preserve configured system-prompt overrides as authoritative while delivering subagent task context through a documented runtime/message contract across embedded attempts, compaction, and CLI preparation, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent for an agent with `systemPromptOverride`; the task is placed only in `extraSystemPrompt`, and the override branches skip the builders that would include it.

## Solution Assessment

No. The local append strategy and closed PR #73637 fix task delivery but conflict with the documented full-override contract; a safer fix should route task context separately or explicitly redefine that contract.
