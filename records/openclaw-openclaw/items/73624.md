## Summary

Keep open. Current main still matches the reported failure path: native subagent task text is carried in `extraSystemPrompt`, but override-backed runner paths still choose `systemPromptOverride` before reaching the prompt builders that append that field. The related repair PR was closed unmerged, and the visible-message follow-up is related but does not replace this functional bug.

## What This Changes

Subagent runs under agents with `systemPromptOverride` should preserve the task-specific `extraSystemPrompt` that carries the delegated `## Your Role` instructions.

## Best Possible Solution

Define a maintainer-approved prompt composition contract for override-backed runs, then implement it through a shared helper with regression coverage across embedded attempts, compaction retries, and CLI preparation.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: spawn a native subagent under an agent with `systemPromptOverride`; the task block is only in `extraSystemPrompt`, and the override branches skip the builders that append it.

## Solution Assessment

No, current main is not solving the issue. The reporter's append shape is plausible, but the best final fix needs maintainer approval on prompt-authority semantics before landing.
