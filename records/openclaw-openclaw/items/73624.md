## Summary

Keep open. Current main still drops the delegated subagent task when `systemPromptOverride` is configured, and the only referenced fix PR was closed unmerged because the straightforward append strategy needs prompt-authority and security review.

## What This Changes

Preserve task-specific subagent instructions from `extraSystemPrompt` when an agent-level `systemPromptOverride` is active.

## Best Possible Solution

Define and document the override composition contract, then preserve the task-bearing runtime context across embedded attempts, compaction retries, and CLI preparation with focused regression tests; if system-message concatenation is rejected, add a lower-priority message contract across providers.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: spawn a native subagent under an agent with `systemPromptOverride`; the delegated task is only in `extraSystemPrompt`, and all three prompt assembly paths skip the builders that append it.

## Solution Assessment

No. Current main does not solve the issue, and the reporter's append approach is only a candidate because the safer architecture may be to route task-bearing context as a separate lower-priority message rather than merge it into system prompt authority.
