## Summary

Keep open. Current main still has high-confidence source paths where subagent task context is carried only through `extraSystemPrompt` and then dropped when `systemPromptOverride` wins, while the earlier fix PR was closed because direct concatenation is the wrong prompt-authority shape.

## What This Changes

Requests reliable delivery of subagent task instructions when an agent-level `systemPromptOverride` is configured.

## Best Possible Solution

Keep `systemPromptOverride` authoritative, but deliver the generated subagent role/task context through a separate runtime or kickoff-message contract across embedded, compaction, and CLI paths with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: spawn a native subagent for an agent with `systemPromptOverride`; the delegated task is in `extraSystemPrompt`, and the override branch bypasses the builders that include it.

## Solution Assessment

No. The local direct-append shape is not the best fix because it conflicts with documented full-replacement semantics and the prior PR discussion; a separate delivery contract is safer.
