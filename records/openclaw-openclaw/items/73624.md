## Summary

Keep open. Current main still routes the subagent task through `extraSystemPrompt` and then drops it whenever `systemPromptOverride` short-circuits prompt building; the referenced fix PR was closed unmerged, and the remaining change needs prompt-authority review rather than routine cleanup.

## What This Changes

Preserve agent-level `systemPromptOverride` while still delivering task-specific `params.extraSystemPrompt` for subagent runs.

## Best Possible Solution

Define the override composition contract, then preserve task-bearing runtime context across embedded attempts, compaction retries, and CLI preparation with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent under an agent with `systemPromptOverride`; the task is only in `extraSystemPrompt`, and the override branch skips every builder that would append it.

## Solution Assessment

No. The current implementation still fails this path, and the local append strategy is not clearly the best fix until maintainers decide whether runtime task context may be merged into a full system-prompt override or should be delivered through a separate lower-priority contract.
