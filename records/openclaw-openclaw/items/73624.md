## Summary

Keep open. Current main still has the override short-circuit that drops the task-bearing extra system prompt for subagent runs, and the related fix PR was closed unmerged; the remaining work needs a maintainer decision on prompt-override composition semantics.

## What This Changes

The issue asks for agent-level systemPromptOverride to preserve the override while still delivering subagent task instructions from extraSystemPrompt.

## Best Possible Solution

Define the override composition contract, then preserve task-bearing runtime context across embedded attempts, compaction retries, and CLI preparation with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn a native subagent under an agent with systemPromptOverride; the task is only in extraSystemPrompt, and all three runner composition paths skip the builder that would append it.

## Solution Assessment

No. Current main still fails this path, and the append-helper approach is plausible but should wait for maintainer confirmation that runtime task context may be composed with a full system-prompt override.
