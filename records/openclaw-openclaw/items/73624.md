## Summary

Keep open. Current main still drops the subagent task path when an agent has `systemPromptOverride`: the task-bearing `extraSystemPrompt` is only passed to prompt builders that are skipped by the override branches, and the related repair PR was closed unmerged.

## What This Changes

Subagent runs under agents with `systemPromptOverride` should still receive the per-run `extraSystemPrompt` that carries delegated `## Your Role` task instructions.

## Best Possible Solution

Define the override composition contract, then implement a shared helper across embedded attempts, compaction retries, and CLI preparation with regression tests and docs wording if semantics change.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: spawn a native subagent under an agent with `systemPromptOverride`; the delegated task is only in `extraSystemPrompt`, and the override branches skip the builders that append it.

## Solution Assessment

No, current main is not solving the issue. The reporter's append approach is plausible, but the maintainable fix needs the prompt-override contract settled before landing.
