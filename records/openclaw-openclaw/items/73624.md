## Summary

Keep open. Current main still has a source-reproducible path where native subagent task context is passed through `extraSystemPrompt` and then omitted when `systemPromptOverride` wins prompt assembly; the closed #73637 PR confirms the right area but was closed unmerged after prompt-authority/security concerns, so this needs maintainer contract review rather than cleanup closure.

## What This Changes

Ensure native subagent runs under agents with `systemPromptOverride` still receive task-specific instructions currently carried in `extraSystemPrompt`.

## Best Possible Solution

Define a maintainer-approved prompt delivery contract that preserves the override as authoritative while reliably delivering subagent task context through a separate user/runtime message or equivalent safe seam, with regression coverage across embedded attempt, compaction, and CLI runner paths.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: spawn a native subagent for an agent with `systemPromptOverride`; the delegated task is passed only through `extraSystemPrompt`, and the override branches skip the builders that would append it.

## Solution Assessment

No. The simple append fix from the closed PR is not clearly the best solution because current docs make the override a full replacement and the discussion flagged prompt-authority/security concerns; a separate-message or runtime-context delivery seam is safer.
