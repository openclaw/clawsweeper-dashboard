## Summary

Current main resolves the reported subagent task-loss behavior: native spawns now put the delegated task in the child session's first visible `[Subagent Task]` user message, and the primary attempt prompt builder also preserves runtime extra context under `systemPromptOverride`.

## What This Changes

The issue reports that subagent runs targeting agents with `systemPromptOverride` can drop task-specific `extraSystemPrompt` instructions.

## Best Possible Solution

Keep the current visible task-envelope implementation, and route any remaining hidden runtime-context semantics to the narrower follow-up rather than keeping this older task-loss report open.

## Reproduction Assessment

Yes for the historical bug: the issue describes `sessions_spawn` delivering only a generic kickoff while the task lived in hidden prompt context. Current main no longer reproduces that task-loss path because the delegated task is sent in the child’s first visible `[Subagent Task]` message and is covered by focused tests.

## Solution Assessment

Yes. Moving the delegated task into one visible kickoff message is a narrower and more auditable fix for the reported task-loss behavior than relying on hidden `extraSystemPrompt` composition under every prompt override path.
