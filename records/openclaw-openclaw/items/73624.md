## Summary

Keep open. Current main still has the reported override-first prompt assembly in the embedded attempt, compaction retry, and CLI runner paths, so a configured systemPromptOverride bypasses the builder branch that carries the subagent task in extraSystemPrompt. The prior fix PR #73637 is closed unmerged after a prompt-authority security concern, while #73656 tracks the separate visible-bootstrap fallback rather than this functional delivery bug.

## What This Changes

The issue requests that subagent task instructions carried in extraSystemPrompt remain available when an agent-level systemPromptOverride is configured.

## Best Possible Solution

Preserve delegated subagent task delivery while keeping user-controlled task text out of system-role authority unless maintainers explicitly accept a documented composition contract. The end state should include regression coverage for override-only, no-override, and override-plus-task behavior across embedded attempt, compaction retry, and CLI runner paths, plus config docs if systemPromptOverride semantics change.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: spawnSubagent passes the task block through extraSystemPrompt, but all three prompt assembly paths return the resolved systemPromptOverride before reaching the builder that appends extraSystemPrompt.

## Solution Assessment

No for the simple local append patch as the final answer. It fixes task delivery, but #73637 showed that raw system-prompt concatenation introduces prompt-authority concerns; the best fix needs maintainer approval on either a lower-privilege message contract or a carefully documented and tested composition rule.
