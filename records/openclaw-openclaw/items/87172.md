## Summary

Keep this PR open: the unrelated workflow churn from the earlier review is gone, but the current patch still marks cross-run catalogs as reused after rebuilding and re-registering them, and the reuse equality can keep stale catalog entries when tool metadata or wrapped executables change.

## What This Changes

The PR adds Tool Search catalog fingerprint/reuse flags, suppresses repeated embedded catalog log lines when reuse is reported, and adds helper-level tests for unchanged and changed tool sets.

## Best Possible Solution

Make reuse a real no-op across the embedded session lifecycle only when all catalog-affecting inputs match, then prove the repeated embedded turn no longer rebuilds or re-registers the catalog.

## Reproduction Assessment

Yes. The linked issue includes repeated embedded gateway logs, and current main source shows each embedded attempt applies/logs Tool Search cataloging and clears catalog state during cleanup; I did not run a live Matrix embedded turn in this read-only review.

## Solution Assessment

No. The patch suppresses the log when a fingerprint matches but still rebuilds/registers on the cross-run path, and its same-run equality omits catalog fields that affect describe/call behavior.
