## Summary

Keep open. Current main has official external plugin catalogs and doctor/update repair paths, but the package still omits the affected channel plugins, postinstall explicitly does not install plugin package dependencies, and config validation can still print the misleading stale-config removal warning for missing official channel plugin refs.

## What This Changes

The issue asks OpenClaw to preserve or clearly migrate formerly bundled channel plugins when upgrading from 2026.4.27 to 2026.5.2+ after those channels moved to external `@openclaw/*` packages.

## Best Possible Solution

Keep the externalized plugin model, but make missing official external channel diagnostics and manual update docs point operators to `openclaw doctor --fix` or `openclaw plugins install @openclaw/<id>` instead of telling them to remove valid config.

## Reproduction Assessment

Yes, from source. Current main still omits the affected plugins from the root package and retains a validation warning that tells operators to remove missing official channel plugin config instead of installing or repairing it.

## Solution Assessment

No, current main is not the full fix. The doctor/update repair path helps, but the narrow maintainable follow-up is to make official external channel missing-plugin diagnostics and manual update docs point to the existing install/doctor repair flow.
