## Summary

Keep open: current main still lacks API-call count support, and this PR remains the canonical implementation path, but the branch still has source-level correctness issues and no contributor real-behavior proof.

## What This Changes

The branch adds per-turn API call counting through embedded runner/session usage state and displays it in `/status` plus the response usage footer, with unit tests.

## Best Possible Solution

Rebase onto the current status/footer helper layout, count API/model responses at the response boundary independent of token metadata, deterministically replace or clear latest-run call state, and add focused tests plus redacted real-session proof.

## Reproduction Assessment

Yes for review purposes: current main source shows the feature is absent, and the PR head shows the two remaining incorrect code paths by inspection. I did not run a live OpenClaw session, and contributor after-fix proof is still missing.

## Solution Assessment

No, not as written. The direction is reasonable, but the implementation should count calls outside usage-token guards, update latest-run call state deterministically, and rebase before validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Count calls before usage metadata filtering:** `src/agents/pi-embedded-subscribe.ts:278`
  - body: `callCount` increments only after `hasNonzeroUsage` passes, so a successful model response that omits usage or reports zeros still made an API call but is not counted. Move the call counter to the model-response boundary and keep token aggregation conditional.
  - confidence: 0.88
- **[P2] Replace stale latest-run call counts:** `src/auto-reply/reply/session-usage.ts:104-106`
  - body: This only writes `callCount` when the new value is greater than zero. Because session entries are patched and this field represents latest-run state, a later zero or unknown-count update can leave an older value visible in `/status`; persist numeric zero when known or clear the field when unknown.
  - confidence: 0.9
