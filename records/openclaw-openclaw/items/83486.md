## Summary

Current main still matches the reported gateway-context confusion path, and the issue has an open closing PR that targets the implicated files. Keep this issue open until that PR is reviewed and merged or rejected.

## What This Changes

The issue reports that authenticated admin HTTP RPC requests can execute against the wrong live gateway instance when multiple gateways run in the same Node process.

## Best Possible Solution

Review and land a fix that binds plugin HTTP gateway-method dispatch to the server-local gateway context, with the linked PR as the current implementation candidate: https://github.com/openclaw/openclaw/pull/83487.

## Reproduction Assessment

Yes, source-reproducible: plugin HTTP routes on current main set a scoped client but no gateway context, while in-process gateway dispatch falls back to a process-global context resolver. I did not execute the live two-gateway PoC because this review is read-only.

## Solution Assessment

Yes, the narrow fix shape is to carry the server-local gateway request context into plugin HTTP request and upgrade scopes before gateway-method dispatch. The existing linked PR should be reviewed as that implementation path rather than opening a second repair lane.
