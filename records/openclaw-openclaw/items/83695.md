## Summary

## What This Changes

The PR adds a shared `tryChmodSync` helper and routes exec approvals, task/task-flow registry, and plugin-state permission setup through access-verified chmod fallback tests.

## Best Possible Solution

Merge the helper after maintainers accept the fsGroup permission boundary, preserving the strict private-file guard and adding operator guidance if they want to call out cleanup for too-broad existing files.

## Solution Assessment

Yes, with maintainer acceptance of the boundary. A shared access-verified helper is narrower than scattered best-effort catches and the tests preserve fail-closed behavior for inaccessible paths and unsafe file modes.
