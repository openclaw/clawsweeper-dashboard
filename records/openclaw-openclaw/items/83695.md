## Summary

Keep open for maintainer review: the patch is focused and I found no blocking correctness findings, but it intentionally changes chmod hardening semantics on state paths and needs explicit acceptance of that permission boundary before merge.

## What This Changes

This PR adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state chmod setup through it with focused fsGroup and unsafe-mode tests.

## Best Possible Solution

Land the helper once maintainers accept the fsGroup permission boundary, preserving the private-file fail-closed guard and adding operator guidance only if maintainers want cleanup steps called out.

## Solution Assessment

Yes, pending maintainer acceptance of the boundary. A shared access-verified helper is narrower than scattered best-effort catches, and the tests preserve fail-closed behavior for inaccessible paths and unsafe private-file modes.
