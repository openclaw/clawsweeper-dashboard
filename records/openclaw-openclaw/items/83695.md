## What This Changes

Adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state permission hardening through it, with chmod fallback regression tests.

## Best Possible Solution

Land the centralized helper after explicit maintainer approval of the fsGroup directory/private-file policy and green required checks, then let it resolve https://github.com/openclaw/openclaw/issues/83619.

Yes. Current main has raw `chmodSync` rethrows in exec approvals and SQLite store permission paths, and the PR discussion provides Docker/K8s-style output showing EPERM before the patch and success with the helper afterward.

## Solution Assessment
