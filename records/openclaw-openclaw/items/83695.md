## Summary

## What This Changes

The branch adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state SQLite permission setup through it for accessible Kubernetes fsGroup chmod failures.

## Best Possible Solution

Land the shared helper only after maintainers accept the directory/file permission boundary and the PR body includes redacted current-head Kubernetes startup or affected-store proof.

Yes at source level: current main rethrows non-Windows chmod failures in `src/infra/exec-approvals.ts`, and Kubernetes fsGroup source/docs support the root-owned group-access mode described by the reporter. I did not run a live cluster in this read-only review.
