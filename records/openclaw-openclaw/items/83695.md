## Summary

Keep this PR open: the fix direction is useful, but the live head still rejects the reported fsGroup directory mode, so it is not merge-ready.

## What This Changes

The PR adds a shared `tryChmodSync` helper and routes exec approvals, task/task-flow registries, and plugin-state SQLite permission setup through it.

## Best Possible Solution

Align the helper, tests, and proof around the exact fsGroup state-root modes OpenClaw intends to support, while still rejecting writable public access to private state.

## Reproduction Assessment

Yes, at source level: current main rethrows non-Windows `chmodSync(dir, 0o700)` failures, and the PR proof describes an accessible fsGroup-owned directory where chmod fails. I did not run live K8s proof during this read-only review.

## Solution Assessment

No, not yet: the shared helper is the right shape, but the current guard rejects the reported fsGroup directory mode after access succeeds. The narrow maintainable fix is to encode the maintainer-approved mode boundary and test the exact `2775` case.
