## Summary

Keep open for maintainer review: the patch looks focused and proof is sufficient, but merging intentionally changes state-path chmod hardening from strict chmod success to access-verified tolerance, so maintainers need to accept the compatibility and security-boundary tradeoff.

## What This Changes

Adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state SQLite permission hardening through it with focused fallback tests.

## Best Possible Solution

Land the shared helper once maintainers accept the fsGroup permission boundary, preserving fail-closed checks for inaccessible paths and unsafe private-file modes and adding operator cleanup guidance only if maintainers want it documented.

## Solution Assessment

Yes, pending maintainer acceptance of the boundary. A shared access-verified helper is narrower than scattered catches and keeps fail-closed behavior for inaccessible paths, other-writable directories, and private files with unsafe other bits.
