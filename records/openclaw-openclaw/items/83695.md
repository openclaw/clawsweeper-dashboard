## Summary

## What This Changes

The branch adds a shared `tryChmodSync` helper, tests it, and routes exec approvals, task registry, task-flow registry, and plugin-state SQLite permission hardening through it.

## Best Possible Solution

Land the shared helper after security/compatibility approval and exact-head validation, preserving fail-closed behavior for inaccessible paths and unsafe private-file modes.

Yes, with high confidence from current-main raw chmod calls plus contributor Kubernetes and Docker fsGroup output. I did not rerun the live scenario in this read-only review.

## Solution Assessment

Mostly yes. A shared helper is the maintainable shape for repeated chmod hardening, but the exact directory and private-file tolerance boundary is a maintainer security decision.
