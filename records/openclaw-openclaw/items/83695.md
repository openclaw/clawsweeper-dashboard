## Summary

Keep open: this is a focused bug-fix PR for a real Kubernetes fsGroup chmod failure that current main still has, and the latest PR proof covers the affected state paths. The remaining blocker is maintainer approval of the chmod-failure security boundary plus exact-head validation, not a discrete code defect found in this pass.

## What This Changes

The PR adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state chmod setup through access-verified fallback behavior on expected permission errors.

## Best Possible Solution

Land the shared helper after explicit security/compatibility approval and exact-head fsGroup validation, preserving the private-file guard and documenting the intended directory tolerance if maintainers want that contract public.

## Reproduction Assessment

## Solution Assessment

Mostly yes. A shared helper is the maintainable shape for repeated chmod hardening paths, but the directory permission tolerance is a security/compatibility policy that maintainers should approve explicitly.
