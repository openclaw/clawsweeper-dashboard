## What This Changes

The PR adds a shared `tryChmodSync` helper and routes exec approvals, task registry, task-flow registry, and plugin-state SQLite permission setup through it for accessible chmod failures on managed volumes.

## Best Possible Solution

Land the shared helper after maintainers approve the mode-aware filesystem permission boundary, the proof/status checks are refreshed, and broader macOS/provenance leftovers stay tracked separately.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: current main rethrows non-Windows `chmodSync` failures on the reported state paths, and the linked issues plus PR terminal proof show the root-owned fsGroup mode that triggers EPERM.

## Solution Assessment

Mostly yes: a shared access-verifying helper is the maintainable fix shape, but the mode-aware directory/file permission boundary needs explicit maintainer acceptance before merge.
