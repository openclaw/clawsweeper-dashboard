Overall confidence: 0.92

Full review comments:

- **[P1] Use the unified Talk session RPCs in the native bridge:** `apps/shared/OpenClawKit/Sources/OpenClawKit/CanvasRealtimeTalkBridge.swift:170`
  - body: `CanvasRealtimeTalkBridge.start` still requests `talk.realtime.session`, and the bridge later listens/sends on `talk.realtime.relay*`. Current main exposes the unified `talk.session.create`/`appendAudio`/`submitToolResult`/`close` methods and `talk.event`, so this new native bridge will fail before a Canvas realtime session can start after rebasing.
  - confidence: 0.94
- **[P2] Remove the deprecated Canvas capability refresh method:** `src/gateway/method-scopes.ts:36`
  - body: Current main's protocol contract says native clients must use `pluginSurfaceUrls.canvas` and `node.pluginSurface.refresh`; `node.canvas.capability.refresh` is intentionally unsupported. Re-adding it to node/operator method scopes restores the removed compatibility path instead of the plugin-surface contract.
  - confidence: 0.9
- **[P2] Avoid restarting playback during bridge teardown:** `apps/shared/OpenClawKit/Sources/OpenClawKit/CanvasRealtimeTalkBridge.swift:517`
  - body: `stopInternally` calls `clearOutput` while `self.session` is still set, and `clearOutput` starts a new playback task whenever `session.outputSampleRateHz` exists. Stopping or erroring the bridge can therefore leave fresh playback work alive after teardown; clear the session first or add a non-restarting clear path.
  - confidence: 0.88
