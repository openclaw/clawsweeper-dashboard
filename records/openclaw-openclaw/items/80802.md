Full review comments:

- **[P2] Avoid restarting playback during bridge teardown:** `apps/shared/OpenClawKit/Sources/OpenClawKit/CanvasRealtimeTalkBridge.swift:643`
  - body: `stopInternally` calls `clearOutput()` while `self.session` is still non-nil, and `clearOutput()` starts a new playback stream whenever `session.outputSampleRateHz` exists. Stopping or erroring the bridge can therefore leave a fresh playback handle alive after teardown; clear the session first or add a stop-only clear path.
  - confidence: 0.89
- **[P2] Stop advertising the removed Canvas capability method:** `src/gateway/server-methods-list.ts:163`
  - body: This re-adds `node.canvas.capability.refresh` to advertised gateway methods, but PR head has no core handler for that method and current protocol docs make that compatibility path unsupported. Clients that discover it can get `unknown method`; use `node.pluginSurface.refresh` or add a complete compatible handler and contract update.
  - confidence: 0.84
- **[P2] Validate the app install path before deleting it:** `script/build_and_run.sh:47`
  - body: The run script deletes `$OPENCLAW_APP_BUNDLE` before copying the staged app, but that environment override can point anywhere. A typo or poisoned environment can recursively remove an unrelated directory; reject unsafe targets before `rm -rf`.
  - confidence: 0.82
