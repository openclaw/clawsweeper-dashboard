## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear the session before clearing output during teardown:** `apps/shared/OpenClawKit/Sources/OpenClawKit/CanvasRealtimeTalkBridge.swift:643`
  - body: `stopInternally` calls `clearOutput()` before clearing `self.session`, and `clearOutput()` starts a new playback stream whenever `session.outputSampleRateHz` is still present. Stopping or erroring the bridge can therefore leave a fresh playback handle or continuation alive after teardown; clear the session first or add a stop-only clear path.
  - confidence: 0.89
- **[P2] Stop advertising the unsupported Canvas refresh method:** `src/gateway/server-methods-list.ts:163`
  - body: This adds `node.canvas.capability.refresh` to advertised gateway methods, but the PR head still has no handler for that method, so a client that discovers it will get `unknown method` when calling it. Current protocol docs say this deprecated path is unsupported; use `node.pluginSurface.refresh` or add a complete compatible handler and contract update.
  - confidence: 0.87
- **[P2] Validate the app bundle path before deleting it:** `script/build_and_run.sh:47`
  - body: `INSTALLED_APP` comes from `OPENCLAW_APP_BUNDLE`, but the script deletes it with `rm -rf` before copying the staged app. A typo or poisoned environment can recursively remove an unrelated directory through the new Run action; reject unsafe targets before deletion.
  - confidence: 0.86
