## Summary

Keep open. ## What This Changes

The PR adds a guarded `handleMessageEnd` suppression path and regression tests so Telegram `message_end` block replies are skipped for intermediate `toolUse` assistant turns containing tool-call blocks while preserving normal final replies.

## Best Possible Solution

Land this PR or an equivalent rebased patch with a single-line changelog entry: keep the suppression limited to `message_end` block replies for assistant `toolUse` turns that contain structured tool-call blocks, preserve `text_end`, normal `stop`, and toolUse-without-tool-call behavior, and keep the regression coverage for the suppressed and preserved cases.
