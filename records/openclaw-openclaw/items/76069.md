## What This Changes

The PR adds `/messaging_window` and `/messaging-window` command handlers, parser coverage, registry metadata, and docs for managing `messages.inbound` debounce settings.

## Best Possible Solution

Land the command only after its config reads/writes are aligned with the existing command-enable contract, with focused regression tests, docs that name the chosen gate, and an active changelog entry.

## Reproduction Assessment

Yes. Source inspection is sufficient: with `commands.config` absent or false, an owner command such as `/messaging_window global 3s` can pass the new handler checks and reach `replaceConfigFile`, while existing `/config` and config-backed `/allowlist` write paths stop at `requireCommandFlagEnabled`.

## Solution Assessment

No. The command is a reasonable narrow feature, but the implementation should reuse `commands.config` or add a documented schema-backed flag before it persists `messages.inbound` changes from chat.
