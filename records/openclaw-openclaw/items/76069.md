## Summary

Keep open. The debounce behavior is already available through `messages.inbound` config today; this PR adds a chat/native-command convenience surface for viewing and writing that existing config, but the current head still has an origin-channel config-write authorization gap plus an exact-head lint failure.

## What This Changes

The PR adds `/messaging_window` slash/native command metadata, parser and handler code, tests, docs, and a changelog entry for managing `messages.inbound` debounce settings from chat.

## Best Possible Solution

If maintainers want chat-based debounce management, fix the origin-channel authorization and lint blockers, then land the command as a convenience layer over the existing `messages.inbound` config; otherwise keep the current config/docs path as the supported workflow.

## Reproduction Assessment

Yes. The lint blocker is reproduced by the exact-head `check-lint` annotation, and the authorization issue follows directly from the PR head code path for a non-bundled channel with `command.channelId` unset and `channels.<id>.configWrites=false`.

## Solution Assessment

No. The direction is a reasonable UX shortcut, but the current PR head is not the best merge state until the config-write origin policy and parser lint failure are fixed.
