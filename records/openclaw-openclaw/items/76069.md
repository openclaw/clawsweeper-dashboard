## Summary

Keep open. Current main already supports the underlying inbound debounce/window behavior through `messages.inbound` config, so this PR is a chat/native-command convenience layer over an existing feature, but the current head is not merge-ready because exact-head lint still fails and non-bundled origin channels can bypass `configWrites` on global command writes.

## What This Changes

The PR adds `/messaging_window` and `/messaging-window` to inspect and update existing `messages.inbound` global and per-channel batching windows, with command registry metadata, tests, docs, and a changelog entry.

## Best Possible Solution

Land the command as a convenience wrapper over existing `messages.inbound` settings after the origin authorization and parser lint blockers are fixed, without changing debounce runtime semantics.

## Reproduction Assessment

Yes. The lint blocker is reproduced by exact-head check annotations, and the authorization issue follows from source: a non-bundled origin with no `channelId` reaches config-write policy with a null origin channel and skips the origin-disabled check.

## Solution Assessment

No. The feature direction is narrow and maintainable, but this PR head is not the best merge state until it uses the same safe origin-channel fallback for authorization that it uses for target resolution and clears lint.
