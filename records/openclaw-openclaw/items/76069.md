## Best Possible Solution

Land the feature after mutating `/messaging_window` actions reuse the existing config command enablement policy, tests cover disabled writes, docs name the gate, and the active changelog records the command.

Yes. Source inspection on PR head `a59abd7cbe295967693c820e6494e3d2c55229fc` shows `/messaging_window global 3s` can reach `replaceConfigFile` without `requireCommandFlagEnabled`, while current `/config` and config-backed `/allowlist` writes require `commands.config`.
