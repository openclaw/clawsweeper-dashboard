Keep open. The PR is not implemented on current main and still adds a persistent chat-driven config write path without the existing disabled-by-default config command gate.

## What This Changes

The PR adds `/messaging_window` and `/messaging-window` chat commands, tests, and docs for managing global and per-channel `messages.inbound` debounce settings.

## Best Possible Solution

Land the command only after persistent writes are gated like existing chat config mutations, with tests, docs, and an active changelog entry aligned to the chosen flag.

## Reproduction Assessment

Yes. With `commands.config` absent or false, source inspection shows `/messaging_window global 3s` can still reach `replaceConfigFile`, while `/config` and config-backed `/allowlist` writes stop at `requireCommandFlagEnabled`.

## Solution Assessment

No. The command surface is narrow, but the implementation should reuse `commands.config` or add a documented schema-backed flag before it can persist `messages.inbound` changes.
