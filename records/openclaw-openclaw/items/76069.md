## Summary

Keep open. The draft PR adds a useful command surface, but it should not merge as-is because the new persistent config write path bypasses the existing disabled-by-default config command gate.

## What This Changes

The PR adds `/messaging_window` and `/messaging-window` chat commands, tests, and docs for managing `messages.inbound` global and per-channel debounce settings.

## Best Possible Solution

Land the command only after its persistent writes are gated like existing chat config mutations, with focused tests, docs aligned to the chosen gate, and an active changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `commands.config` absent or false, `/messaging_window global 3s` can still reach `replaceConfigFile`, while current `/config` and `/allowlist` write paths would stop at `requireCommandFlagEnabled`.

## Solution Assessment

No. The command itself is a reasonable narrow surface, but the implementation should reuse the existing config-write enablement contract or add a documented schema-backed flag before persisting changes.
