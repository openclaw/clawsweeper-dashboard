## Summary

Keep open. Current main still resolves completion profile paths with hardcoded HOME-backed zsh/fish defaults and a bash check/install mismatch, and an open target-repo PR already references this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Shell completion should write and check the actual zsh, fish, and bash profile paths when `ZDOTDIR`, `XDG_CONFIG_HOME`, or bash profile fallbacks are in use.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81298 or a narrow replacement that centralizes profile-path resolution for check/install paths and preserves the bash fallback contract with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `ZDOTDIR` or `XDG_CONFIG_HOME`, run completion install/update, and current main still targets HOME-backed zsh/fish files; bash status can also miss an install written to `.bash_profile`.

## Solution Assessment

Yes. A shared resolver used by both status and install paths is the narrowest maintainable fix, and the active linked PR is the right implementation candidate to review before closing this issue.
