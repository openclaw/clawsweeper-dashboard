## Summary

Keep open. Current main still contains the reported shell-completion profile path bug for zsh, fish, and bash checks, and the same-author fix PR #63082 remains open with closing syntax.

## What This Changes

The issue asks shell completion install/check/update flows to respect zsh `ZDOTDIR`, fish `XDG_CONFIG_HOME`, and a consistent bash `.bashrc`/`.bash_profile` fallback.

## Best Possible Solution

Land #63082 or an equivalent focused PR that centralizes profile-path resolution for completion status, install, update, doctor, and setup messaging with unit coverage for the reported shell/env combinations.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: set `ZDOTDIR` or `XDG_CONFIG_HOME`, run completion install/update, and the code still resolves zsh/fish profiles under `$HOME`; bash status can miss a `.bash_profile` install.

## Solution Assessment

Yes. A shared resolver used by both check and install paths is the narrowest maintainable fix; reviewing or replacing #63082 is safer than starting a duplicate repair lane.
