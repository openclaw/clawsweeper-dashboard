## Summary

Keep open. Current main still resolves zsh and fish completion profile paths under `HOME`, keeps the bash check/install mismatch, and the same-author implementation PR #63082 remains open with closing syntax.

## What This Changes

The issue asks shell-completion install and status checks to respect `ZDOTDIR`, `XDG_CONFIG_HOME`, and consistent bash `.bashrc`/`.bash_profile` fallback behavior.

## Best Possible Solution

Land #63082 or an equivalent focused PR that centralizes shell profile path resolution for completion status, install, doctor/update repair, setup messaging, and unit tests for zsh, fish, and bash profile fallback cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: set `ZDOTDIR` or `XDG_CONFIG_HOME`, run completion install/update, and current main still chooses the default `HOME` profile path; bash status can also miss an install written to `.bash_profile`.

## Solution Assessment

Yes. A shared resolver used by both check and install paths is the narrowest maintainable solution; reviewing or replacing #63082 is better than creating a duplicate repair lane.
