## Summary

Keep open. Current main still contains the reported shell-completion profile path bug for zsh, fish, and bash checks, and the same-author PR #63082 remains open as the active implementation path.

## What This Changes

The issue requests shell-completion install/status/update flows that write to and check the profile files actually loaded by zsh, fish, and bash when `ZDOTDIR`, `XDG_CONFIG_HOME`, or macOS bash profile fallbacks are used.

## Best Possible Solution

Land #63082 or an equivalent focused PR that centralizes profile-path resolution for completion status, install, update, doctor, and setup messaging, with unit coverage for zsh `ZDOTDIR`, fish `XDG_CONFIG_HOME`, bash `.bashrc`/`.bash_profile`, and any intentionally included PowerShell behavior.

## Reproduction Assessment

Yes. The source-backed reproduction is high confidence: with `ZDOTDIR` or `XDG_CONFIG_HOME` set, current main still resolves zsh and fish profiles under `$HOME`, and bash status checks can miss a completion that install writes to `.bash_profile`.

## Solution Assessment

Yes. A narrow shared resolver used by both check and install paths is the best maintainable direction; reviewing or replacing the existing focused PR is safer than starting a second repair lane.
