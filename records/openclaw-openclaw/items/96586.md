## Summary

Keep open: current main and v2026.6.10 still return status from active in-memory voice-call manager lookups only, while terminal calls are persisted and then removed from active indexes. An open closing PR and a sibling candidate PR already target this bug, so the issue should remain open until one lands.

## What This Changes

The issue asks for voice-call status lookups to fall back to persisted call records when completed calls are no longer in the active in-memory manager.

## Best Possible Solution

Review and land one linked candidate after it scopes persisted fallback to status reads, returns the newest persisted snapshot, and proves the completed-or-restarted status path.

## Reproduction Assessment

Yes. Current main still checks only active voice-call manager lookups for status requests, while finalization persists terminal calls and removes active indexes.

## Solution Assessment

Yes. The narrow solution is a status-scoped persisted-record fallback by call id or provider call id, with tests that prove terminal persisted calls remain discoverable after active eviction.
