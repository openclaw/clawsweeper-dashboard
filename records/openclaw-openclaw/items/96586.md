## Summary

Keep open: current main still has the voice-call status gap, and the issue is now the canonical tracker for open fix candidates. The status tool/RPC paths still check only active in-memory manager maps, while terminal calls are persisted and removed from those active indexes.

## What This Changes

The issue reports that voice-call status lookup returns not found after completed calls leave the active manager despite persisted call records remaining in plugin state.

## Best Possible Solution

Keep this issue open while one narrow fix PR lands a status-scoped newest/canonical persisted-store fallback for get_status, legacy status mode, and voicecall.status.

## Reproduction Assessment

Yes. Source inspection shows the status paths read active manager maps only, while terminal calls are persisted and then removed from active indexes; I did not place a live voice-provider call in this read-only review.

## Solution Assessment

Yes. A status-scoped persisted-store fallback is the narrowest maintainable fix; adding a new transcript API or broadening all manager lookups is not necessary to repair the existing status contract.
