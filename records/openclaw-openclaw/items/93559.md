The PR moves `retireSessionMcpRuntime()` earlier in gateway session cleanup so MCP runtime retirement happens before waiting for the old embedded agent run to end.

## Best Possible Solution

Use a lease-aware cleanup path that retires stale MCP runtimes only after the embedded run drains, is explicitly force-cleared, or can defer disposal until active MCP leases release, with reset/delete timeout coverage.
