## Summary

Keep open for maintainer review: the linked stale-MCP-process issue is still valid, but this cherry-pick is not safe as submitted because it retires the session MCP runtime before proving the old embedded run has drained or been force-cleared.

## What This Changes

The PR inserts an unconditional `retireSessionMcpRuntime` call before the embedded-run drain wait in `src/gateway/session-reset-service.ts` and adds verbose cleanup logging.

## Best Possible Solution

Adopt a lifecycle-safe cleanup contract that retires MCP runtimes only after the old run has drained, has been explicitly force-cleared, or can defer disposal until the final MCP lease releases, with reset and delete timeout coverage.
