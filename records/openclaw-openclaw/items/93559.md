## Summary

Keep open: the stale MCP process problem is real from source inspection, but this PR is not merge-ready because it retires the session MCP runtime before proving the old embedded run has drained or been safely force-cleared, and the supplied proof is only tests/build/code-diff evidence.

## What This Changes

The branch adds a pre-wait MCP runtime retirement call in the gateway session cleanup helper so cleanup attempts to reap MCP processes before checking whether the embedded run drained.

## Best Possible Solution

Adopt a lifecycle-safe cleanup contract that retires MCP runtimes only after the run has drained, has been explicitly force-cleared, or can defer disposal until the final MCP lease releases, with reset and delete timeout coverage.
