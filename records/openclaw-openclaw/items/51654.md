## Summary

Keep open. Current main still lacks a public `sessions_spawn.env` contract and does not thread session-scoped env values through the ACP spawn, manager, or runtime initialization path; the remaining work is useful but needs maintainer policy because it can carry credentials.

## What This Changes

Request support for per-spawn environment variables on ACP `sessions_spawn` sessions so different ACP child sessions can run with isolated provider keys or endpoints.

## Best Possible Solution

Define a documented ACP-only `sessions_spawn.env` contract with string validation, redaction, and explicit merge/override semantics, then thread it through the tool, ACP spawn, manager, runtime, acpx, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is high-confidence from source inspection. Current main has no `sessions_spawn.env` schema field or forwarding path, while the lower runtime type already has an env seam.

## Solution Assessment

Unclear. Passing env through the ACP pipeline is plausible, but the maintainable solution first needs maintainer policy for redaction and for interaction with provider-auth stripping.
