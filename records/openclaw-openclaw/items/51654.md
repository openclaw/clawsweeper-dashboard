## Summary

Keep open. Current main still has no public `sessions_spawn.env` contract or ACP spawn forwarding path, and the useful feature needs maintainer policy because it can carry credentials.

## What This Changes

Request adding an optional `sessions_spawn.env` parameter that passes string environment variables into ACP/acpx session startup.

## Best Possible Solution

Define an ACP-only `sessions_spawn.env` contract with string validation, explicit env merge semantics, transcript/log redaction, and focused tests before threading it through tool, spawn, manager, runtime, and acpx startup paths.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The feature gap is high-confidence from source inspection: the public tool and ACP spawn/manager path do not accept or forward `env`, while the lower runtime type already has an env seam.

## Solution Assessment

Unclear. Passing env through the ACP pipeline is plausible, but the best maintainable solution depends on maintainer policy for credential redaction and provider-auth stripping semantics.
