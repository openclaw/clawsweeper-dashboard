## Summary

Keep this open: current main and v2026.6.10 still lack a model-callable `sessions_spawn.env` ACP surface, and the safe fix needs maintainer product/security direction because it would route credentials and endpoint overrides into host ACP subprocesses.

## What This Changes

The issue requests an optional per-session environment map for ACP `sessions_spawn` calls so ACP child processes can receive session-specific credentials or endpoint settings.

## Best Possible Solution

Choose one credential-safe ACP/session env contract and wire it through schema, spawn, manager, backend support, redaction, docs, and tests after product/security review.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.10 shows no `sessions_spawn.env` schema field and no ACP spawn/manager forwarding into `runtime.ensureSession`; a live ACP run is not needed to prove the missing surface.

## Solution Assessment

No for the raw issue shape as written. Direct model-callable env passthrough is not the safest boundary; the better path is a maintainer-approved env contract with validation, redaction, backend support, and explicit interaction with credential scrubbing.
