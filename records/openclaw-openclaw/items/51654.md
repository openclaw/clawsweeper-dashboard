## Summary

Keep open: current main and v2026.6.10 still lack a model-callable `sessions_spawn.env` ACP path, and the requested credential-bearing env surface needs maintainer product/security design rather than conservative cleanup.

## What This Changes

Request adds a session-level `env` map to ACP `sessions_spawn` calls so per-session credentials and endpoint settings can be passed to ACPX subprocesses.

## Best Possible Solution

Define one credential-safe ACP session env contract, then wire schema, spawn, manager, ACPX/backend behavior, redaction, docs, and tests through that contract before closing this issue.

## Reproduction Assessment

Yes, source-level. Current schema, ACP spawn params, manager ensure path, and docs all omit a `sessions_spawn.env` path, so a live ACP run is not needed to prove the missing surface.

## Solution Assessment

No for the raw shape as written. A plain model-callable env map is not the best boundary until maintainers define validation, redaction, allowed sources, and interaction with credential scrubbing.
