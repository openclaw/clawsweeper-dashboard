## Summary

Keep this open: current main and the latest release still do not expose or wire a `sessions_spawn.env` ACP surface, and the raw requested shape needs maintainer product/security direction before implementation.

## What This Changes

The issue requests an optional per-session environment map for ACP `sessions_spawn` calls so spawned ACP harnesses can receive session-specific variables.

## Best Possible Solution

Define a credential-safe ACP session environment override contract, then wire approved values through `sessions_spawn` -> `spawnAcpDirect` -> ACP manager -> backend with validation, redaction, tests, docs, and explicit interaction with inherited-env scrubbing.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.8` do not expose `sessions_spawn.env`, and ACP manager initialization does not forward env into `runtime.ensureSession`. A live ACP spawn is not necessary to prove the missing surface.

## Solution Assessment

No for the raw requested shape as written: direct `Record<string,string>` passthrough from a model-facing tool is not the safest boundary. The better path is an explicit environment override contract with validation, redaction, backend support, and credential isolation tests.
