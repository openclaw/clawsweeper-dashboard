## Summary

Keep open. Current main still does not expose or forward a session-level `env` map for ACP `sessions_spawn`; the public tool schema, ACP spawn parameter type, spawn execution path, and ACP manager initialization path all drop that contract even though the lower-level runtime type already has an optional `env` seam.

## What This Changes

The issue requests an optional `env` map on `sessions_spawn` for ACP sessions so per-session environment overrides can be passed to the acpx-backed harness process.

## Best Possible Solution

Add a complete ACP session env contract: expose `env?: Record<string, string>` for `sessions_spawn` when `runtime: "acp"`, validate keys and string values, define merge/override behavior against the inherited gateway environment, redact values from persisted tool inputs/log-visible surfaces, thread it through `SpawnAcpParams` and `AcpInitializeSessionInput` into `runtime.ensureSession`, and document the behavior on `https://docs.openclaw.ai/tools/acp-agents`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested `env` field is absent from the `sessions_spawn` schema and cannot travel through `spawnAcpDirect` or `AcpSessionManager.initializeSession` to the runtime.

## Solution Assessment

Yes, the requested direction is the maintainable path if it is paired with secret-aware validation and redaction. Since `AcpRuntimeEnsureInput` already has an `env` seam, the narrowest fix is propagation and policy documentation rather than global environment mutation or acpx-only special casing.
