## Summary

Keep open: current main and the latest release still lack a public `sessions_spawn.env` contract or ACP forwarding path. The lower ACP runtime type has an `env` seam, but the tool schema, spawn wrapper, manager, acpx reuse policy, docs, and redaction rules remain unwired and need product/security review.

## What This Changes

Request adding a per-spawn env map so `sessions_spawn({ runtime: "acp" })` can launch ACP harness sessions with session-specific credentials or endpoints.

## Best Possible Solution

Define an explicit ACP-only session env or SecretRef-aware harness env contract, including redaction, precedence, allow/deny behavior, and persistent-session reuse semantics, then wire it through `sessions_spawn`, ACP spawn, the manager, and acpx with docs and tests.

## Reproduction Assessment

No, not as a runtime bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested public `sessions_spawn.env` parameter and ACP forwarding path.

## Solution Assessment

Unclear until maintainers choose the credential boundary. Raw env passthrough is not the safest default; the maintainable path is an explicit env or SecretRef-aware contract with redaction, precedence, reuse, and acpx compatibility rules.
