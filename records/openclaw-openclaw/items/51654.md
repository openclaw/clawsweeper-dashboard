## Summary

## What This Changes

Request adding a session-level environment map to ACP `sessions_spawn` so each spawned harness can receive isolated provider keys or endpoints.

## Best Possible Solution

Define an explicit ACP-only session env or SecretRef-aware harness env contract with validation, redaction, merge/override rules, docs, tests, and acpx compatibility proof.

No, not as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks `sessions_spawn.env` and does not forward env through ACP session creation.

## Solution Assessment

Unclear until maintainers define the credential boundary. Passing raw env through the ACP pipeline is plausible, but the safer maintainable direction is an explicit env or SecretRef contract with redaction, precedence, and acpx compatibility rules.
