## Summary

Keep open. Current `main` still lacks the public `sessions_spawn.env` contract and does not thread an env map through ACP spawn or manager initialization, while the credential-bearing behavior needs explicit policy before implementation.

## What This Changes

Add an optional env map to ACP `sessions_spawn` so individual acpx-backed sessions can receive session-specific environment variables.

## Best Possible Solution

Define a documented ACP-only env contract with string validation, credential redaction, and explicit merge/override semantics before threading it through the tool, ACP spawn, manager, and acpx runtime paths.

## Reproduction Assessment

Yes. Static source inspection on current `main` shows the requested env field is absent from the public schema, execution path, ACP spawn params, and manager input, while the lower runtime type already has an env seam.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable fix first needs maintainer policy for redaction and for how session env interacts with provider-auth stripping.

## Security Review

Status: needs_attention

Summary: The requested feature would pass credentials into ACP child processes, so implementation needs explicit redaction and env merge/override policy.

  - body: ACP client startup strips known provider auth variables for protected paths. Session-level env support needs a clear rule for whether caller-supplied env may reintroduce or override those stripped values.
