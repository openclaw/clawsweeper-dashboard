## Summary

Keep open. Current main still does not expose or forward a session-level env map for ACP sessions, and the remaining work is a credential-bearing feature/policy addition rather than an obsolete cleanup item.

## What This Changes

Request adds an optional env map to ACP sessions_spawn so per-session environment variables can be passed to ACP runtime subprocesses.

## Best Possible Solution

Add a documented ACP-only env contract that validates string keys/values, redacts values from all persisted/log-visible tool inputs, threads it through spawn and manager into AcpRuntimeEnsureInput, and defines merge/override behavior against inherited env.

## Reproduction Assessment

Yes. Static inspection of current main shows the requested env field is absent from the sessions_spawn schema, SpawnAcpParams, ACP spawn initialization, and manager input, while the lower-level AcpRuntimeEnsureInput type already has an env seam.

## Solution Assessment

Unclear until maintainer policy is settled. Propagating through AcpRuntimeEnsureInput.env is the narrow implementation seam, but the best fix must first define credential redaction plus merge/override behavior.
