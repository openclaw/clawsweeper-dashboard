## Summary

Keep open: current main still lacks a public `sessions_spawn.env` contract and does not forward per-session env through the ACP spawn/manager/acpx path, while the requested behavior needs credential-boundary policy before implementation.

## What This Changes

Add support for per-session environment variables when spawning ACP sessions through `sessions_spawn`.

## Best Possible Solution

Define an explicit ACP-only `sessions_spawn.env` or SecretRef-aware harness env contract with validation, redaction, merge/override rules, tests, docs, and acpx compatibility proof.

No, not as a bug reproduction because this is a feature request. Source inspection is high-confidence that current main lacks the requested public `sessions_spawn.env` parameter and forwarding path into ACP session creation.

## Solution Assessment

Unclear until maintainers define the credential boundary. Passing raw env through the ACP pipeline is plausible, but the safer maintainable direction is an explicit env or SecretRef contract with redaction, precedence, and acpx compatibility rules.
