## Summary

Keep open. Current main still lacks a public `sessions_spawn` env parameter and does not thread per-session env through ACP spawn or manager initialization, and the credential-bearing behavior needs maintainer policy before implementation.

## What This Changes

Request support for passing session-specific environment variables into `sessions_spawn({ runtime: "acp" })` so ACP child harnesses can use per-session keys or endpoints.

## Best Possible Solution

Define a documented ACP-only env contract, validate string keys and values, redact values from persisted/log-visible tool inputs, then thread the sanitized map through the tool, ACP spawn, manager, and runtime initialization.

## Reproduction Assessment

Yes. Static source inspection on current main shows an `env` object passed to `sessions_spawn` would not be declared in the tool schema, parsed by execution, accepted by `SpawnAcpParams`, or forwarded by the ACP manager, even though the lower runtime type has an env seam.

## Solution Assessment

Unclear until maintainer policy is settled. The narrow code seam is `AcpRuntimeEnsureInput.env`, but the maintainable fix must also cover secret redaction, env validation, docs, tests, and provider-auth stripping behavior.

## Security Review

Status: needs_attention

Summary: The request would carry credential-bearing env values into ACP child processes, so implementation needs explicit redaction and env override policy.

Concerns:

- **[medium] Redact session env values before persistence:** `src/agents/session-transcript-repair.ts:81`
  - body: `sessions_spawn` tool inputs are sanitized before transcript persistence, but the current sanitizer only handles existing sensitive ACP fields and attachments. Adding `env` without extending this path could persist API keys in transcripts or log-visible payloads.
  - confidence: 0.86
- **[medium] Define provider-auth env override policy:** `src/acp/client-helpers.ts:190`
  - body: ACP client startup currently strips known provider auth variables for protected paths. Session-level env support needs an explicit rule for whether caller-supplied env can reintroduce or override those stripped variables.
  - confidence: 0.82
