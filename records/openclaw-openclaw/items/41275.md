## What This Changes

The PR changes the cron Control UI controller so agent-turn `timeoutSeconds` accepts `0`, rejects negative/non-finite values, and preserves explicit zero in the payload.

## Best Possible Solution

Land a refreshed focused PR that changes both validation and payload preservation, updates cron controller tests for the zero-timeout contract, and includes redacted real UI/runtime proof.

## Reproduction Assessment

Yes from source: current main rejects `timeoutSeconds: 0` in `validateCronForm` and also omits explicit zero from the payload builder, while the protocol and runtime accept zero as no-timeout. I did not run the UI because this review is read-only.

## Solution Assessment

Mostly: the Control UI controller is the right boundary because the protocol/runtime contract already supports zero, and the PR correctly touches both validation and payload preservation. It is not merge-ready until the current controller tests and real behavior proof move with the behavior.
