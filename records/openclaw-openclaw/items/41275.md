## What This Changes

The PR changes Control UI cron form validation and payload construction so agent-turn `timeoutSeconds` accepts and submits `0` while rejecting non-finite or negative timeout text.

## Best Possible Solution

Align UI validation, payload building, English copy, and controller tests with the existing gateway contract that `timeoutSeconds: 0` disables the cron wall-clock timeout, then add redacted real Control UI proof before merge.

## Reproduction Assessment

Yes, source-level. The Control UI input writes directly to `form.timeoutSeconds`, current `validateCronForm` rejects `"0"`, current `buildCronPayload` drops zero, and gateway cron schema/runtime tests already accept zero as no-timeout.

## Solution Assessment

No as submitted. The implementation idea is the right layer, but the best mergeable fix must also update the adjacent controller tests and user-facing timeout copy, plus include real Control UI proof.
