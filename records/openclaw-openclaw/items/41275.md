## What This Changes

The PR changes the Control UI cron controller so `timeoutSeconds: 0` is accepted, non-finite or negative timeout input is rejected, and explicit zero is preserved in the submitted cron payload.

## Best Possible Solution

Land this branch or an equivalent narrow fix with controller coverage that accepts and preserves zero, rejects malformed or negative timeout input, and includes redacted Control UI or gateway proof.

## Reproduction Assessment

Yes. Source inspection shows the cron view writes the timeout input into `form.timeoutSeconds`, current validation rejects `"0"`, and the adjacent controller test still asserts that old behavior; I did not run a browser flow in this read-only review.

## Solution Assessment

Yes in direction, but incomplete as submitted. The controller is the right layer because the gateway schema and cron runtime already support zero, but the PR needs test alignment and real behavior proof before it is the best mergeable fix.
