## What This Changes

The PR changes Control UI cron parsing to accept `timeoutSeconds: 0`, reject negative or non-finite timeout text, and include zero in agent-turn payloads.

## Best Possible Solution

Land a focused Control UI fix that accepts zero, preserves `timeoutSeconds: 0` in the payload, keeps malformed values invalid, updates validation copy/locales, and adds controller regression tests while leaving the existing gateway contract unchanged.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main rejects `timeoutSeconds: "0"` in `validateCronForm` and would omit explicit zero in `buildCronPayload`, while schema and runtime code already define zero as valid no-timeout input.

## Solution Assessment

Mostly yes. The Control UI controller is the narrow maintainable boundary, but the PR should also update tests and validation copy before it is merge-ready.
