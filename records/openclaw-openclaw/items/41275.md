## What This Changes

The PR changes the Control UI cron controller to accept `timeoutSeconds: 0`, reject negative or non-finite timeout text, and preserve explicit zero in the cron payload.

## Best Possible Solution

Land a focused Control UI fix that accepts zero, preserves `timeoutSeconds: 0` in submitted payloads, keeps malformed values invalid, updates validation copy/locales, and adds controller regression coverage while leaving the gateway contract unchanged.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main rejects `timeoutSeconds: "0"` in `validateCronForm` and drops explicit zero in `buildCronPayload`, while the schema and runtime timeout policy already allow zero.

## Solution Assessment

Mostly yes. The Control UI controller is the narrow maintainable boundary, but this PR should also update tests and validation copy and provide real behavior proof before merge.
