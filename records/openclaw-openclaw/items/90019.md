## Summary

Keep open: current main and the latest release still leave the periodic memory sync disabled by default, and this PR targets the right stale-index path by defaulting the interval and making interval ticks mark memory dirty. It is not merge-ready yet because the contributor proof is only unit/fake-timer output and the default change is upgrade-sensitive.

## What This Changes

The PR changes memory-search sync resolution to default `sync.intervalMinutes` to 30, makes interval ticks mark memory dirty before syncing, updates config/docs help text, and adds focused regression tests.

## Best Possible Solution

Merge a narrow memory-search fallback fix only after maintainers explicitly accept the default-change tradeoff, the config help text is corrected, and the PR includes redacted real runtime proof of stale content refreshing.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source and release inspection show the fallback is disabled by default and the old interval path is a no-op when `dirty=false`, while the linked report provides the long-running watcher-failure scenario.

## Solution Assessment

Unclear as merge-ready: marking dirty in the interval callback is the right owner-boundary fix for the no-op path, but changing the default to 30 minutes is compatibility-sensitive and still needs real behavior proof.
