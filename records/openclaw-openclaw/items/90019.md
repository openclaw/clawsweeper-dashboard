## Summary

This PR still needs maintainer review, not cleanup closure: the runtime fix is coherent and proof-backed, but changing unconfigured builtin memory search from no periodic sync to a 30-minute background sync is an upgrade-sensitive default change.

## What This Changes

The PR changes builtin memory search to default `memorySearch.sync.intervalMinutes` to 30, forces interval ticks to mark memory dirty before syncing, and updates related docs, config help, and regression tests.

## Best Possible Solution

Land this after a maintainer explicitly accepts the new default background sync behavior, or revise the PR to keep the compatibility default and make periodic fallback an opt-in documented setting.

## Reproduction Assessment

Yes, at source level: current main defaults the interval to `0`, and an interval tick would not rescan memory files unless `dirty` is already true. The PR body also supplies a real-manager short-interval proof showing stale content refresh after the interval tick.

## Solution Assessment

Unclear as a product default, but the runtime repair is the narrow correct path if maintainers accept periodic background sync by default. The safer compatibility alternative is to keep `0` as the default and make the fallback explicitly opt-in.
