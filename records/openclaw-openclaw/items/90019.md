## Summary

Keep this PR open: it addresses the source-level stale builtin memory index failure, but the default-on 30-minute periodic sync is an upgrade-sensitive behavior change that still needs explicit maintainer acceptance before merge.

## What This Changes

The PR changes builtin memory search to default `sync.intervalMinutes` to 30 minutes, marks memory dirty before interval sync, documents the default and opt-out, and adds regression coverage.

## Best Possible Solution

Land only after maintainers explicitly accept the default-on builtin fallback, or keep the old default and make periodic sync an opt-in documented workaround.

## Reproduction Assessment

Yes at source level: current main defaults the interval to 0 and the existing interval tick does not force dirty state, so the reported no-fallback path is clear. I did not reproduce a real multi-hour chokidar outage.

## Solution Assessment

Yes for the narrow code path: resolving the default in memory-search config and forcing dirty before interval sync are the maintainable builtin fixes. The remaining question is whether maintainers want the fallback enabled by default for upgrades.
