## Summary

Keep this PR open: it is the active, focused fix candidate for the linked stale memory index bug, and the remaining blocker is maintainer acceptance of the new default background periodic sync behavior rather than a contributor-facing code defect.

## What This Changes

The PR changes memory search to default unconfigured `memorySearch.sync.intervalMinutes` to 30, forces interval ticks to mark memory dirty before syncing, and updates related docs, schema help, labels, and regression tests.

## Best Possible Solution

Land the narrow fix after maintainer acceptance of the 30-minute default and preserved `0` opt-out; otherwise keep interval sync opt-in and pursue watcher-health fallback design separately.

## Reproduction Assessment

Yes for the source-level path, but not as a live multi-hour chokidar outage in this review. Current main defaults the interval to `0`, and the existing interval callback can call sync while `dirty=false`, so missed watcher events can leave memory files stale.

## Solution Assessment

Mostly yes. Forcing dirty before interval sync is a narrow mitigation for missed watcher events, but changing the shipped default to background periodic indexing still needs maintainer acceptance.
