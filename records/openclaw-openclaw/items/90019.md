## Summary

Keep open for maintainer approval: source review supports the builtin stale-index fix and the supplied runtime proof is sufficient, but the default-on interval changes upgrade behavior for existing unconfigured users.

## What This Changes

This PR defaults builtin memory search interval sync to 30 minutes, forces interval ticks to mark memory dirty before syncing, documents the default and 0 opt-out, and adds regression coverage.

## Best Possible Solution

Land only after maintainers explicitly accept the default-on builtin fallback, or keep the old default and make periodic sync an opt-in or doctor-recommended setting.

## Reproduction Assessment

Yes at source level: current main has no default periodic fallback and the existing interval callback does not force dirty state, so the reported no-fallback failure mode is clear. I did not reproduce a real multi-hour chokidar outage.

## Solution Assessment

Yes for the code path: resolving the default in the memory-search config layer and forcing dirty before interval sync are the narrow builtin fixes, while QMD remains on its separate update interval. The open question is product approval for making that fallback default-on for existing users.
