## Summary

Keep open: this PR addresses a real current-main memory-search freshness gap and has sufficient terminal proof, but it is not merge-ready because the default-on interval changes upgrade behavior and the unconditional interval dirty mark can leave sessions-only managers dirty.

## What This Changes

This PR defaults memory search interval sync to 30 minutes, marks managers dirty before interval sync, documents the default and 0 opt-out, and adds focused regression tests.

## Best Possible Solution

Scope the interval dirty mark to memory-source managers, add non-memory coverage, then merge only if maintainers explicitly accept the 30-minute default-on fallback; otherwise keep the default opt-in and document the workaround.

## Reproduction Assessment

Yes at source level: current main defaults interval sync to 0 and the interval callback does not mark memory dirty before sync. I did not run a multi-hour chokidar outage repro in this read-only review.

## Solution Assessment

No as submitted: the resolver and interval path are the right narrow surface if maintainers accept the upgrade behavior, but the dirty assignment must be scoped to memory-source managers first.
