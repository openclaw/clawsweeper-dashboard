## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Make the interval fallback actually rescan memory files:** `src/agents/memory-search.ts:448`
  - body: Changing the default only arms `ensureIntervalSync()`, but that timer calls `this.sync({ reason: "interval" })` and `runSync` only enters `syncMemoryFiles()` when `force`, a full reindex, or `this.dirty` is true. In the linked failure mode the watcher silently stops, so `markDirty()` is never called and `dirty` stays false; the 30-minute tick will skip memory files and leave the index stale. Please make the interval path mark memory dirty or otherwise run the lightweight hash enumeration without forcing a full reindex, and cover that manager-level behavior with a timer test.
  - confidence: 0.94
- **[P2] Document the new memory sync default and opt-out:** `src/agents/memory-search.ts:448`
  - body: This line changes a user-visible config default from disabled to a 30-minute background sync, but the public memory config reference and schema help still do not expose `sync.intervalMinutes`, and an adjacent runtime comment still says the default is 0. Please update the config/help/docs surface and stale comment so operators can discover the new background work and the explicit `intervalMinutes: 0` opt-out.
  - confidence: 0.87
