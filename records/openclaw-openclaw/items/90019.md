## Summary

Keep this PR open for maintainer review. The patch is a credible fix for the stale memory index path and includes sufficient runtime proof, but it intentionally changes upgrade behavior by turning on a 30-minute background memory sync for unconfigured users, and one generated help string still misstates the opt-out behavior.

## What This Changes

The PR changes memory search to default `memorySearch.sync.intervalMinutes` to 30, forces interval ticks to mark memory dirty before syncing, and updates related docs/config/test coverage.

## Best Possible Solution

Land a narrow memory-core fix that preserves the explicit `0` opt-out, aligns all generated help text with the trigger-based sync behavior, and has maintainer approval for the 30-minute default's upgrade/resource tradeoff.

## Reproduction Assessment

Yes for the source path, but not as a live multi-hour chokidar outage in this review. Current main and `v2026.6.1` default the interval to `0`, and the existing interval callback calls sync without forcing dirty, so a silent watcher miss can leave memory files stale.

## Solution Assessment

Mostly yes. Forcing dirty before interval sync is a narrow mitigation for missed watcher events, but the best merge path still needs maintainer approval for changing the shipped default and a small help-text cleanup.
