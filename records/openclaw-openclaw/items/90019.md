## What This Changes

The PR changes memory-search to default `sync.intervalMinutes` to 30, forces interval ticks to rescan memory by marking the manager dirty, and updates related docs/config/test coverage.

## Best Possible Solution

Land the fallback only if maintainers accept the idle-overhead tradeoff, and first align every opt-out help surface to mention watcher plus trigger-based sync.

## Reproduction Assessment

Yes from source: current main disables the interval fallback by default and interval sync can skip memory-file work when `dirty` is false. I did not recreate a multi-hour chokidar outage in this read-only review.

## Solution Assessment

Mostly yes: forcing dirty before interval sync is the narrow runtime fix, and the default change addresses the missing fallback. The remaining question is whether maintainers accept that default change for existing users.
