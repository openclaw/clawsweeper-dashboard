## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve omitted streaming as enabled:** `extensions/feishu/src/reply-dispatcher.ts:145-146`
  - body: Current Feishu docs and main schema treat `channels.feishu.streaming` as default-enabled, with only `false` disabling it. This predicate only enables explicit `true` or an object, so existing configs with the setting omitted lose streaming cards in `card` or auto-detected card replies unless users add new config.
  - confidence: 0.95
- **[P2] Queue partials while streaming starts:** `extensions/feishu/src/reply-dispatcher.ts:415-416`
  - body: `startStreaming()` is asynchronous, but this callback only queues the partial when `streaming.isActive()` is already true. During Card Kit startup the first auto-mode partial is dropped; short or block-only responses can close with little or no streamed content even though streaming was requested.
  - confidence: 0.88
- **[P2] Align the public Feishu config contract:** `extensions/feishu/src/config-schema.ts:46-52`
  - body: The schema starts accepting an object for `streaming`, but the plugin manifest and docs still advertise boolean-only `channels.feishu.streaming`. That leaves setup/discovery surfaces and user docs with a different public contract than runtime parsing.
  - confidence: 0.86
