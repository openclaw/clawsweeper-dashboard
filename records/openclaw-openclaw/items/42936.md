## Summary

Keep open: this is a plausible Feishu config feature, but the PR still has blocking source-level regressions, leaves public config surfaces inconsistent, and has no contributor-supplied Feishu/Lark runtime proof.

## What This Changes

The branch changes Feishu streaming config parsing to accept a boolean or object form and wires object `throttleMs` and `title` values into streaming card dispatch and tests.

## Best Possible Solution

Define one Feishu-owned streaming config contract across schema, manifest, docs, dispatcher, and tests, preserve current defaults and identity metadata, then prove it in a real Feishu/Lark setup.

## Reproduction Assessment

Yes for source-level PR review: the diff shows omitted `streaming` no longer passes the predicate and the first auto-mode partial is queued only after the stream is already active. No live Feishu/Lark setup was reproduced by the contributor.

## Solution Assessment

No. The object-config direction may be useful, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, bypasses current identity-header behavior, and does not update the public config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve default-enabled streaming:** `extensions/feishu/src/reply-dispatcher.ts:142-146`
  - body: The new predicate only enables streaming for explicit `true` or object config, so an omitted `streaming` setting disables streaming. Existing Feishu configs that rely on the documented default in card or auto mode would stop using streaming cards unless users add new config.
  - confidence: 0.97
- **[P2] Queue the partial that starts auto streaming:** `extensions/feishu/src/reply-dispatcher.ts:408-416`
  - body: `startStreaming()` creates the Card Kit session asynchronously, but this partial is queued only if the session is already active. The first auto-mode chunk can be dropped while startup is in flight, so short replies may show little or no incremental update.
  - confidence: 0.91
- **[P2] Publish object streaming config in public surfaces:** `extensions/feishu/src/config-schema.ts:46-53`
  - body: The runtime schema accepts object values for `streaming`, but the plugin manifest and Feishu docs still advertise only a boolean. Setup/config discovery and users following docs will not see or validate `enabled`, `throttleMs`, or `title`.
  - confidence: 0.88
- **[P2] Preserve the current throttle cadence:** `extensions/feishu/src/reply-dispatcher.ts:198`
  - body: Current main uses and tests a 160 ms streaming-card throttle, but this branch falls back to 150 ms in the dispatcher and 100 ms in the session constructor. Keep the established cadence unless maintainers intentionally choose a different API-rate tradeoff.
  - confidence: 0.84
- **[P2] Merge custom titles with identity metadata:** `extensions/feishu/src/reply-dispatcher.ts:215`
  - body: Passing a hard-coded header for every streaming session bypasses current main's identity-based card header and note path. A title option should flow through the existing resolver instead of replacing configured agent identity metadata.
  - confidence: 0.83
