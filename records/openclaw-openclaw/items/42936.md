## Summary

Keep this PR open for maintainer review, but the current branch should not merge as-is: current main still lacks the requested object-shaped Feishu streaming config, and the PR introduces regressions around default-enabled streaming, early partial updates, and current card metadata.

## What This Changes

The PR changes the Feishu plugin to accept boolean or object-shaped `streaming` config, pass title/throttle options into Card Kit streaming, and update a few streaming test expectations.

## Best Possible Solution

Rebase this into a Feishu-owned streaming config resolver that preserves existing defaults and `false` opt-out behavior, merges title/throttle with current card metadata, and updates schema, manifest, docs, and focused tests together.

## Reproduction Assessment

Yes for source-level PR review: current main and the PR diff show the config/default behavior paths and the async first-partial path. No live Feishu setup was proven by the contributor.

## Solution Assessment

No. The direction is plausible, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, and does not update public config surfaces or preserve current card metadata.
