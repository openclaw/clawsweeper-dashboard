## Summary

Keep open: the PR still proposes a real Feishu config feature, but the current branch is stale against main, has blocking streaming regressions, lacks real Feishu/Lark proof, and needs a maintainer decision on the streaming config shape before it can merge.

## What This Changes

The PR changes Feishu streaming config parsing to accept boolean or object forms, adds object throttle/title options, changes auto-mode streaming startup, and updates streaming-card header test expectations.

## Best Possible Solution

Define one Feishu-owned streaming config contract across schema, manifest, docs, runtime, and focused tests while preserving default streaming, identity metadata, and the current throttle cadence unless maintainers intentionally choose a new rate-limit tradeoff.

## Reproduction Assessment

Yes for source-level PR review: comparing the PR diff with current main shows the default-config regression, async auto-mode partial drop, public metadata gap, throttle mismatch, and identity-header bypass. No live Feishu/Lark runtime reproduction was provided by the contributor.

## Solution Assessment

No. The object-config direction may be useful, but this patch should preserve default semantics, publish one config contract across public surfaces, route title/header behavior through identity metadata, and include real Feishu/Lark proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve default-enabled streaming:** `extensions/feishu/src/reply-dispatcher.ts:143-146`
  - body: The new predicate enables streaming only for explicit `true` or object config, so omitted `streaming` no longer follows current main and docs where Feishu streaming defaults on unless set to `false`. Existing card or auto-card setups would stop streaming unless users add new config.
  - confidence: 0.97
- **[P2] Queue the partial that starts auto streaming:** `extensions/feishu/src/reply-dispatcher.ts:405-416`
  - body: `startStreaming()` begins CardKit creation asynchronously, but this partial is queued only if the session is already active. Early auto-mode chunks emitted while startup is in flight can be dropped, so short replies may show little or no incremental update.
  - confidence: 0.91
- **[P2] Publish object streaming config in public surfaces:** `extensions/feishu/src/config-schema.ts:46-53`
  - body: The runtime schema accepts an object for `streaming`, but the plugin manifest and Feishu docs still advertise only a boolean. Setup/config discovery and users following docs would not see or validate `enabled`, `throttleMs`, or `title`.
  - confidence: 0.88
- **[P2] Preserve the current throttle cadence:** `extensions/feishu/src/reply-dispatcher.ts:199-200`
  - body: Current main uses a 160 ms Feishu streaming-card throttle, but this branch falls back to 150 ms in the dispatcher and 100 ms in the session constructor. Keep the established cadence unless maintainers intentionally choose a new API-rate tradeoff.
  - confidence: 0.84
- **[P2] Merge custom titles with identity metadata:** `extensions/feishu/src/reply-dispatcher.ts:214-215`
  - body: Passing a hard-coded header for every streaming session bypasses current main's identity-based card header and note path. A title option should flow through the existing resolver instead of replacing configured agent identity metadata.
  - confidence: 0.83
