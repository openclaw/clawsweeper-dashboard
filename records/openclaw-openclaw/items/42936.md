## Summary

Keep open, but the latest patch is not merge-ready. Current main still lacks the object-shaped Feishu streaming config, so the PR is not obsolete, but the branch regresses default-enabled streaming, can drop the first auto-mode partial, and does not preserve current card metadata or public config surfaces.

## What This Changes

The PR changes the Feishu plugin to accept boolean or object-shaped streaming config, use that config for throttle/title options, enable auto-mode streaming, and update dispatcher test expectations.

## Best Possible Solution

Rework on current main with a typed Feishu streaming config resolver that preserves `undefined` as enabled and `false` as disabled, keeps current card header/note behavior, updates schema/manifest/docs/tests together, and includes real Feishu proof.

## Reproduction Assessment

Yes for source-level PR review. Current main and docs show boolean/default-true streaming, while the latest PR head shows the omitted-config regression and async first-partial drop path; no live Feishu runtime proof was provided.

## Solution Assessment

No. The direction is useful, but this patch is not the best solution because it changes established defaults, can lose early auto-mode streaming text, hard-codes card identity metadata, and leaves public config surfaces behind.
