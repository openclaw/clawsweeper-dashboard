## Summary

Keep open. Current main still accepts only boolean Feishu `streaming` config, so the PR is not obsolete, but the patch is not ready because it regresses omitted-config defaults and leaves the new public config shape out of manifest/docs/changelog surfaces.

## What This Changes

The PR changes Feishu streaming config parsing to accept boolean or object forms, passes throttle/title/header options into `FeishuStreamingSession`, and updates dispatcher test expectations for the new streaming card header.

## Best Possible Solution

Implement a current-main-based Feishu streaming config resolver that preserves `undefined` as enabled and `false` as disabled, accepts a documented object shape, and updates manifest, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. Source inspection shows current main accepts only boolean `streaming`, and the supplied PR patch shows the omitted-config regression in its `streamingEnabled` predicate.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it changes established defaults, can drop the first auto-mode streaming partial, and leaves public config surfaces incomplete.
