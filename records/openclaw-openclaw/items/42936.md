## Summary

Keep open: this PR still contains unique Feishu streaming-config work that is not on current main, but it is not mergeable as-is because it regresses the existing default-enabled streaming behavior, can miss the first auto-mode partial update, lacks public config/docs coverage, and has no contributor-supplied real Feishu proof.

## What This Changes

The branch changes the Feishu plugin to accept boolean or object-shaped streaming config and pass title/throttle options into Card Kit streaming.

## Best Possible Solution

Rebase and rework this as a Feishu-owned streaming config resolver that preserves `undefined` as enabled and `false` as disabled, updates schema/manifest/docs/tests together, and merges title/throttle options with current card identity metadata.

## Reproduction Assessment

Yes for source-level PR review: current main and the PR diff clearly show the config/default behavior paths and the async first-partial path. No live Feishu setup was proven by the contributor.

## Solution Assessment

No. The direction is plausible, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, and does not update public config surfaces or preserve current card metadata.

  - confidence: 0.86

  - body: Passing a hard-coded header for every streaming session would override current main's identity header behavior and does not preserve the note footer path; the title option needs to merge with the existing metadata contract.
  - confidence: 0.82
