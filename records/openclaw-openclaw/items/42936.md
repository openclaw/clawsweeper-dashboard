## Summary

Keep open, but this PR is not merge-ready. Current main still does not implement object-shaped Feishu streaming config, while the PR head regresses the default-enabled streaming behavior, can drop the first auto-mode partial, leaves public config surfaces boolean-only, and has no after-fix Feishu runtime proof.

## What This Changes

The branch changes Feishu streaming config from boolean-only to boolean-or-object options and passes title/throttle settings into Card Kit streaming.

## Best Possible Solution

Rebase into a Feishu streaming config resolver that preserves default-enabled and `false` opt-out behavior, keeps current header/note/throttle semantics, and updates schema, manifest, docs, and focused tests together.

## Reproduction Assessment

Yes for source-level PR review: current main and the PR diff show the config/default behavior paths and the async first-partial path. No live Feishu setup was proven by the contributor.

## Solution Assessment

No. The direction is plausible, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, and does not update the public config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve default-enabled streaming:** `extensions/feishu/src/reply-dispatcher.ts:143-146`
  - body: The predicate only enables streaming for explicit `true` or object config, so omitted `streaming` no longer follows current main and docs' default-true behavior. Existing Feishu configs using card mode or auto-detected markdown would silently stop streaming unless users add new config.
  - confidence: 0.97
- **[P2] Queue the first partial while streaming starts:** `extensions/feishu/src/reply-dispatcher.ts:408-416`
  - body: In auto mode, `startStreaming()` is asynchronous, but the partial is queued only if `streaming?.isActive()` is already true. The first chunk that creates the Card Kit session can be dropped, so short replies may show no incremental update.
  - confidence: 0.9
- **[P2] Publish the object config in public surfaces:** `extensions/feishu/src/config-schema.ts:46-53`
  - body: The runtime schema accepts an object for `streaming`, but the plugin manifest and Feishu docs still advertise only a boolean. Setup/config UI and users following docs will not see or validate the new `enabled`, `throttleMs`, and `title` shape.
  - confidence: 0.86
- **[P2] Merge custom titles with existing card metadata:** `extensions/feishu/src/reply-dispatcher.ts:214-215`
  - body: Passing a hard-coded header for every streaming session would override current main's identity header behavior and does not preserve the note footer path. The title option needs to merge with the existing card metadata contract.
  - confidence: 0.82
