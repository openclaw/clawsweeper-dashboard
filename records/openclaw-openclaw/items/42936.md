## Summary

Keep open: current main still does not implement the boolean-or-object Feishu streaming config this PR proposes, but the branch is not merge-safe because it regresses default-enabled streaming, can drop the first auto-mode partial, leaves the new config unpublished, and has no contributor Feishu runtime proof. The throttle part also overlaps https://github.com/openclaw/openclaw/pull/31172, so the public config shape needs maintainer direction before repair or merge.

## What This Changes

This PR changes Feishu streaming config to accept boolean-or-object values, starts auto-mode streaming from partial output, and passes custom title/throttle options into streaming cards with test expectation updates.

## Best Possible Solution

Define one Feishu-owned streaming config resolver and public contract across schema, manifest, docs, runtime, and focused tests, preserving current defaults and identity-header behavior before adding live Feishu proof.

## Reproduction Assessment

Yes for source-level PR review: current main and PR head show the default-config regression, async auto-mode partial drop, public metadata gap, and throttle overlap. No live Feishu setup was proven by the contributor.

## Solution Assessment

No. The object-config direction is plausible, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, bypasses current identity header behavior, and does not update the public Feishu config contract.

- **[P1] Preserve default-enabled streaming:** `extensions/feishu/src/reply-dispatcher.ts:142-146`
  - body: The new predicate only enables streaming for explicit `true` or object config, so omitted `streaming` no longer follows current main and docs' default-true behavior. Existing Feishu configs using card mode or auto-detected markdown would stop streaming unless users add new config.

  - confidence: 0.91
- **[P2] Publish the object streaming config in public surfaces:** `extensions/feishu/src/config-schema.ts:46-53`
  - body: The runtime schema accepts an object for `streaming`, but the plugin manifest and Feishu docs still advertise only a boolean. Setup/config discovery and users following docs will not see or validate `enabled`, `throttleMs`, or `title`.
  - confidence: 0.88
- **[P2] Keep the current streaming throttle default:** `extensions/feishu/src/reply-dispatcher.ts:200`
  - body: Current main defines and tests a 160 ms Feishu streaming-card throttle, but this branch falls back to 150 ms in the dispatcher and 100 ms in the constructor. Preserve the existing cadence unless maintainers intentionally choose a new API-rate tradeoff.
  - confidence: 0.84
- **[P2] Merge custom titles with card identity metadata:** `extensions/feishu/src/reply-dispatcher.ts:215`
  - body: Passing a hard-coded header for every streaming session bypasses current main's identity-based card header path. A title option should flow through the existing header resolver instead of replacing configured agent identity metadata.
  - confidence: 0.83
