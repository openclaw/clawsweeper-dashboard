## Summary

Keep open. Current main still does not accept Feishu `streaming` object config, so the PR is not obsolete, but the current patch is not ready to merge: it changes the omitted-config default from enabled to disabled, can drop the first partial that starts auto-mode streaming, and does not update the manifest/docs/changelog for the new user-facing config shape.

## What This Changes

The PR changes Feishu streaming config parsing to accept boolean or object forms, passes throttle/title/header options into `FeishuStreamingSession`, and updates dispatcher test expectations for the new streaming card header.

## Best Possible Solution

Implement a current-main-based Feishu streaming config resolver that accepts legacy booleans and a documented object shape, preserves `undefined` as enabled and `false` as disabled, passes bounded throttle/title/header options through typed streaming-session APIs, and updates the plugin manifest, Feishu docs, changelog, and regression tests without undoing recent streaming lifecycle fixes.

## Reproduction Assessment

Yes. The missing feature is reproducible from source because current main's Zod schema and manifest accept only boolean `streaming`; the PR regression is reproducible by evaluating its `streamingEnabled` predicate with `streaming` omitted, which returns false where current main returns true.

## Solution Assessment

No. The direction is reasonable, but this patch is not the best implementation because it changes established defaults, drops early auto-mode partials while streaming starts asynchronously, and leaves manifest/docs/changelog surfaces out of sync with the runtime schema.
