## Summary

Keep open: current main still lacks the requested object-form Feishu streaming config, but this PR should not merge as-is because it regresses the documented default-enabled streaming behavior and leaves public config surfaces incomplete.

## What This Changes

The PR changes the Feishu plugin to accept boolean or object streaming config, pass throttle/title options into streaming cards, add a streaming card header, and adjust dispatcher tests.

## Best Possible Solution

Implement a current-main Feishu streaming config resolver that accepts the documented object shape, preserves `undefined` as enabled and `false` as disabled, keeps existing card identity metadata unless explicitly overridden, and updates schema, manifest, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes for source-level review. Current main shows boolean-only Feishu streaming config and default-enabled streaming, while the supplied PR patch shows the omitted-config regression in its `streamingEnabled` predicate.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it changes established defaults, can drop early auto-mode streaming partials, and does not expose the new object config through manifest/docs/changelog.
