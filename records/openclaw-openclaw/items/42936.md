## Summary

Keep open: the Feishu config direction is useful, but the current PR would regress default streaming behavior, can drop the first auto-mode partial while streaming starts, and lacks required real Feishu runtime proof.

## What This Changes

The PR changes Feishu streaming config parsing and dispatch so `streaming` can be a boolean or object, adds throttle/title options, and passes a header to streaming cards.

## Best Possible Solution

Define one Feishu-owned streaming config contract across schema, manifest, docs, dispatcher, and tests, preserving default-enabled streaming and the current shipped throttle/header behavior unless maintainers intentionally choose a new default.

## Reproduction Assessment

Yes for the blocking PR regressions by source inspection: the diff drops the omitted-config default path and gates first-partial queueing on an async stream becoming active. The claimed Feishu runtime fix itself has no contributor-provided live proof.

## Solution Assessment

No. The object-config direction may be reasonable, but this branch should preserve current default streaming semantics, reconcile throttle/header behavior with current main and https://github.com/openclaw/openclaw/pull/31172, and include real Feishu proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve default-enabled streaming:** `extensions/feishu/src/reply-dispatcher.ts:145-146`
  - body: The new predicate only enables streaming when `streaming` is explicitly `true` or an object. Omitted config is the documented/default case and currently works via `streaming !== false`, so existing Feishu card/auto setups would silently stop streaming after this change.
  - confidence: 0.96
- **[P2] Queue the first auto-mode partial before start completes:** `extensions/feishu/src/reply-dispatcher.ts:411-417`
  - body: In auto mode this starts the stream asynchronously, then only queues the partial if `streaming.isActive()` is already true. `start()` does not mark the session active until the card is created, so short or partial-only responses can lose their first update and close with no streamed content.
  - confidence: 0.88
- **[P2] Keep the shipped throttle default:** `extensions/feishu/src/reply-dispatcher.ts:197`
  - body: Current main and v2026.5.12 define and test the Feishu streaming throttle at 160 ms. This branch falls back to 150 ms whenever no object value is provided, changing update cadence for existing `streaming: true` users without a documented rate-limit decision.
  - confidence: 0.84
