## Summary

Keep open: the Feishu direction is useful, but the latest patch is not merge-ready because it regresses default-enabled streaming, can drop the first auto-mode partial while Card Kit startup is in flight, leaves manifest/docs config surfaces behind, and lacks contributor-supplied Feishu runtime proof.

## What This Changes

The PR changes Feishu streaming config parsing and streaming-card setup to accept boolean or object-shaped streaming settings with title and throttle options.

## Best Possible Solution

Rework this on current main with a typed Feishu streaming config resolver that preserves `undefined` as enabled and `false` as disabled, updates schema/manifest/docs/tests together, and merges title/throttle options without dropping existing card metadata.

## Reproduction Assessment

Yes for source-level review: current main and the PR diff clearly show the config/default behavior paths and the async first-partial path. I did not run a live Feishu setup because this is a read-only review and the PR lacks runtime proof.

## Solution Assessment

No. The requested direction may be reasonable, but this patch is not the best implementation because it changes established defaults, can lose early streaming text, and does not update public config surfaces or preserve current card identity metadata.
