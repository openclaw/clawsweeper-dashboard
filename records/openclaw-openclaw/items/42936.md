## Summary

Keep open. Current main still exposes Feishu `streaming` as a boolean-only config, but this PR is not mergeable as-is because it regresses omitted/default streaming, can drop the first auto-mode partial, does not publish the new object config through manifest/docs, and has no real Feishu behavior proof.

## What This Changes

The PR changes Feishu streaming config parsing and streaming-card behavior to support boolean or object-shaped streaming settings with title and throttle options.

## Best Possible Solution

Rework this on current main with a typed Feishu streaming config resolver that preserves `undefined` as enabled and `false` as disabled, keeps existing card identity metadata unless explicitly overridden, updates schema/manifest/docs/tests together, and includes real Feishu proof.

## Reproduction Assessment

Yes for source-level review. Current main shows boolean-only Feishu streaming config and a default-enabled predicate, while the PR diff shows the omitted-config regression and async first-partial drop path.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it changes established defaults, can lose early auto-mode streaming text, overrides current card metadata, and leaves public config surfaces behind.
