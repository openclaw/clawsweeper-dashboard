## Summary

Keep open: current main still gates Feishu plugin tools on top-level `channels.feishu.appId/appSecret`, while documented and runtime-supported Feishu multi-account configs can place credentials only under `channels.feishu.accounts.<id>`. The reporter’s follow-up confirms an account-scoped manifest hotpatch made the tools callable in Feishu DM sessions, and the linked packaging PR does not address this account-aware availability gap.

## What This Changes

The issue reports that Feishu plugin tools are omitted from the model-visible tool list in Feishu DM sessions when Feishu uses documented account-scoped credentials.

## Best Possible Solution

Make Feishu plugin tool availability account-aware for top-level and named-account credentials, preserving credential gating and tool policy filtering while avoiding hard-coded account names.

## Reproduction Assessment

Yes. Source inspection shows account-scoped Feishu credentials satisfy docs/setup/runtime account paths but do not satisfy the current top-level-only manifest tool availability check; the reporter also confirmed the same root cause with a local manifest hotpatch.

## Solution Assessment

Yes. The maintainable fix is account-aware plugin tool availability, not another DM-specific injection path or only the packaging guard from the linked PR.
