## Summary

Keep open: current main still gates Feishu tool metadata on top-level `channels.feishu.appId/appSecret`, while documented and runtime-supported named-account credentials remain unable to satisfy that manifest availability check.

## What This Changes

Make Feishu plugin tools visible and callable in Feishu DM sessions when the channel uses documented `channels.feishu.accounts.<id>` credentials.

## Best Possible Solution

Make Feishu tool availability account-aware with a generic account-map manifest signal or equivalent plugin-owned availability predicate, then add regression coverage for top-level and named-account Feishu configs.

## Reproduction Assessment

Yes, from source: account-scoped Feishu credentials are documented and resolved by runtime code, but the current manifest availability check only accepts top-level app credentials. I did not run a live Feishu DM session.

## Solution Assessment

No, the current linked PR is not the best fix for the confirmed root cause as written. The maintainable fix is account-aware manifest availability rather than a DM-specific injection path or only a root `dist` import guard.
