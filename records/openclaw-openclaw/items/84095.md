## Summary

Keep open: current main and the latest release still gate Feishu tool availability on top-level `channels.feishu.appId/appSecret` while the docs and setup code support account-scoped credentials. The reporter’s follow-up confirms a local account-aware manifest hotpatch restores `feishu_app_scopes` and `feishu_doc` in Feishu DM sessions, and the open linked PR at https://github.com/openclaw/openclaw/pull/84133 addresses a packaging guard rather than this source-level availability mismatch.

## What This Changes

The issue reports that Feishu plugin tools are missing from agent tool lists when Feishu is configured with named account credentials instead of top-level credentials.

## Best Possible Solution

Make Feishu plugin tool availability account-aware so top-level credentials and any enabled configured named account both expose Feishu tools, while preserving policy filtering and hiding tools when no usable Feishu credentials exist.

## Reproduction Assessment

Yes, from source: documented `channels.feishu.accounts.<id>.appId/appSecret` credentials do not satisfy the current top-level-only Feishu tool `configSignals`. I did not run a live Feishu DM session in this read-only review.

## Solution Assessment

Yes, the maintainable fix is account-aware Feishu tool availability, not a new DM-specific injection path or only the packaging guard in the linked PR.
