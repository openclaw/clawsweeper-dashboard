## Summary

Keep open: current main still has a source-backed Feishu account-scoped tool availability gap, and the linked open PR is release hardening rather than a proven complete runtime fix for the reported DM tool list failure.

## What This Changes

The issue reports that enabled Feishu plugin tools such as `feishu_doc` do not appear in the agent tool list during Feishu DM sessions.

## Best Possible Solution

Make Feishu plugin tool availability account-aware for both top-level and named-account credentials, while preserving existing tool policy filtering and hiding tools when no usable credentials exist.

## Reproduction Assessment

Yes, from source: documented `channels.feishu.accounts.<id>` credentials do not satisfy the current manifest tool availability check, which only reads top-level `appId` and `appSecret`. I did not run a live Feishu DM session.

## Solution Assessment

The best fix is account-aware Feishu plugin tool availability, not another DM-specific injection path. The linked postpublish guard PR may be useful hardening, but it should not be treated as the complete fix unless maintainers confirm the reporter's runtime case is fully resolved.
