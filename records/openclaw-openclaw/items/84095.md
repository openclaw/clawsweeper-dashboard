## Label Justifications

- `impact:auth-provider`: The failure is tied to authenticated Feishu credential/config detection and whether plugin tools are exposed for that configured integration.

## Summary

Keep open: current main and v2026.5.19 still gate Feishu tool metadata on top-level `channels.feishu.appId/appSecret`, while docs/setup and the reporter's retest support account-scoped credentials under `channels.feishu.accounts.<id>`. The linked open PR is useful release hardening, but it should not close this issue unless the account-aware Feishu tool availability gap is fixed or explicitly superseded.

## What This Changes

Make Feishu plugin tools such as `feishu_doc` visible and callable in Feishu DM sessions when credentials are configured under named Feishu accounts.

## Best Possible Solution

Add account-aware Feishu tool availability so top-level credentials and any usable named account credentials expose the Feishu tools, while preserving policy filtering and hiding tools when no valid credentials exist.

## Reproduction Assessment

Yes. Source inspection shows documented `channels.feishu.accounts.<id>` credentials do not satisfy the current top-level-only Feishu tool metadata, and the reporter confirmed a local account-scoped manifest hotpatch made `feishu_doc` callable in Feishu DM sessions.

## Solution Assessment

Yes. The best fix is account-aware plugin tool availability, not another DM-specific injection path or a release-verifier-only change.
