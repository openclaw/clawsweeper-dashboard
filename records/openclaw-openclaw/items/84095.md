## Label Justifications

- `P1`: A documented Feishu configuration can make authenticated Feishu document/chat tools invisible to real channel sessions, blocking the core Feishu document workflow.

## Summary

Keep open: current main and the latest release still gate Feishu tool availability on top-level credentials, while the documented and runtime-supported named-account config path remains a source-reproducible mismatch. The reporter’s follow-up hotpatch evidence also supports the account-scoped manifest availability diagnosis.

## What This Changes

The issue reports that Feishu plugin tools such as `feishu_doc` are not exposed in Feishu DM sessions when the channel uses documented named-account credentials.

## Best Possible Solution

Make Feishu plugin tool availability account-aware so both top-level and named-account credentials expose Feishu tools, while preserving tool policy filtering and hiding tools when no usable credentials exist.

## Reproduction Assessment

Yes. Source inspection shows `channels.feishu.accounts.<id>.appId/appSecret` is documented and recognized by setup/runtime, but current manifest availability checks only top-level `channels.feishu.appId/appSecret`; the reporter also confirmed a local account-scoped metadata hotpatch restored tools in Feishu DM sessions.

## Solution Assessment

Yes. The maintainable fix is account-aware plugin tool availability rather than a DM-only injection workaround or a postpublish-only guard; current embedded tool construction already passes channel/account context into plugin tool resolution.
