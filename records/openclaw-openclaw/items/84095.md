## Summary

Keep open: current main still has a source-backed Feishu tool-availability gap for named-account configuration, and the open linked PR is not merged and appears to harden packaging rather than fully prove the reported runtime path fixed.

## What This Changes

The issue reports that configured Feishu plugin tools such as `feishu_doc` are not exposed in the model-visible tool list for Feishu DM sessions.

## Best Possible Solution

Make Feishu tool availability account-aware so both top-level and named-account credentials expose Feishu tools, while preserving policy filtering and hiding tools when no usable credentials exist.

## Reproduction Assessment

Yes, source-level: a documented `channels.feishu.accounts.<id>.appId/appSecret` setup can be accepted by Feishu setup/runtime while failing the generic manifest availability check for `feishu_doc`, because current metadata requires top-level credentials. I did not run a live Feishu DM session.

## Solution Assessment

Yes, the maintainable fix is account-aware Feishu tool availability or a generic manifest signal for dynamic account maps, not another DM-specific injection path. The open linked PR may be useful release hardening, but it is not enough by itself to prove this runtime issue fixed.
