## Repair Work Prompt

Fix Feishu plugin tool availability for account-scoped credentials. Current Feishu toolMetadata in extensions/feishu/openclaw.plugin.json only checks channels.feishu.appId/appSecret, while docs and setup support channels.feishu.accounts.<id>.appId/appSecret; this can make feishu_doc, feishu_drive, feishu_chat, and related tools unavailable in Feishu agent sessions even when the channel account works. Update the manifest availability contract or Feishu metadata so tools are available when at least one enabled Feishu account has configured appId/appSecret, while preserving top-level credential support and keeping tools hidden when no usable credentials exist. Do not broaden core plugin loading, bypass tool policy filtering, or add startup migrations. Add regression coverage for account-only and top-level Feishu configs, and include a changelog entry if the fix is user-visible.
