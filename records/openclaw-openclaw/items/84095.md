## Summary

Keep open: current main and the latest release still gate Feishu tool visibility on top-level Feishu credentials, while the documented and runtime-supported multi-account config can put credentials only under named accounts. The linked open PR is useful release hardening, but it only changes package verifier scripts and should not auto-close this source-backed Feishu availability bug.

## What This Changes

The issue reports that Feishu plugin tools are not injected into the model-visible tool list for Feishu DM sessions when the channel uses documented multi-account credentials.

## Best Possible Solution

Make Feishu plugin tool availability account-aware through the manifest/plugin availability contract so top-level credentials and any enabled configured named account can expose Feishu tools, while preserving policy filtering and hiding tools when no usable credentials exist.

## Reproduction Assessment

Yes. Source inspection shows account-scoped Feishu credentials are documented and setup/runtime-recognized, while current tool metadata still checks only top-level appId/appSecret; I did not run a live Feishu DM session.

## Solution Assessment

Yes. The best fix is account-aware Feishu tool availability in the generic manifest/plugin availability path, not another DM-specific tool injection path or a packaging-only guard.
