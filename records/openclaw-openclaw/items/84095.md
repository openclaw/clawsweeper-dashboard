## Summary

Keep open: current main and v2026.5.19 still gate Feishu tools on top-level `channels.feishu.appId/appSecret`, while docs, setup, runtime account resolution, and the reporter's follow-up all support account-scoped credentials. The linked open PR at https://github.com/openclaw/openclaw/pull/84133 adds a postpublish guard, but it does not change the Feishu manifest availability path that the reporter confirmed as the live blocker.

## What This Changes

Feishu plugin tools should be exposed to Feishu DM agents when credentials are configured under documented `channels.feishu.accounts.<id>` entries.

## Best Possible Solution

Add account-map-aware Feishu tool availability so top-level credentials and any enabled configured named account expose Feishu tools, while still hiding them when no usable Feishu credentials exist.

## Reproduction Assessment

Yes. With credentials only under `channels.feishu.accounts.<id>.appId/appSecret`, current Feishu tool metadata still asks the manifest availability gate for top-level `appId/appSecret`, so source inspection shows the tool catalog can drop Feishu tools before runtime registration; I did not run a live Feishu DM session.

## Solution Assessment

Yes for the reporter's narrowed diagnosis: fixing account-aware manifest/tool availability is the maintainable path. A DM-specific injection workaround or the linked postpublish-guard-only PR would not address the confirmed account-scoped availability mismatch.
