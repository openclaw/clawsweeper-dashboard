The issue reports that Feishu plugin tools such as `feishu_doc` are not exposed to Feishu DM agent sessions when credentials are configured under `channels.feishu.accounts.<id>`.

## Best Possible Solution

Make Feishu tool availability account-aware, ideally through a generic manifest availability expression for configured account maps or an equivalent Feishu-owned predicate, while preserving credential gating and tool policy filtering.

## Reproduction Assessment

Yes. Source inspection shows documented `channels.feishu.accounts.<id>.appId/appSecret` credentials do not satisfy the current top-level-only Feishu tool `configSignals`, and the reporter confirmed an account-scoped manifest hotpatch fixed their DM sessions.

## Solution Assessment
