## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main still has a focused test that expects a successful block-streamed turn to drop a text final even when `hasSentPayload()` returns false; I did not run live channel validation in this read-only review.

## Solution Assessment

No as submitted. The direction is narrow, but exact per-payload `hasSentPayload()` matching is not enough to distinguish genuinely new post-tool text from text already covered by a coalesced stream.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve coalesced stream suppression:** `src/auto-reply/reply/agent-runner-payloads.ts:208-212`
  - body: This `hasSentPayload()`-only filter checks each final payload independently. When block coalescing sends chunks such as `A` and `B` as one outbound `A\n\nB` payload, later split final payloads `A` and `B` do not match the coalesced content key or full streamed text, so they can be sent again at turn end. Keep coverage-aware suppression for text already covered by the stream while allowing genuinely new post-tool final text through.
  - confidence: 0.86
