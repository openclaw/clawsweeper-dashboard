## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main returns no final text payload when `buildReplyPayloads` sees block streaming enabled, `didStream() === true`, `isAborted() === false`, `hasSentPayload() === false`, and a text-only final payload.

## Solution Assessment

No as submitted. The direction is right, but a plain per-payload `hasSentPayload()` filter is not coalescing-safe and the branch conflicts with newer current-main safeguards.
