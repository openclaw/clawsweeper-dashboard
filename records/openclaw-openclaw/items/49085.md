  - body: `--server-args` is variadic, but this detector treats it as a single-value flag. For `openclaw acp client --server-args --url ...`, `--url` is not consumed as a server arg, `isAcpClient` stays false, and interactive client `console.log` output is redirected to stderr.
  - confidence: 0.86

  - body: `routeLogsToStderr()` only flips the flag consumed by the patched console wrapper. The standalone server path calls `serveAcpGateway()` directly without first installing console capture, so direct runs can still leave normal console output on stdout.
  - confidence: 0.78
