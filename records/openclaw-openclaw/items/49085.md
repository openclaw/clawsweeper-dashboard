## What This Changes

The PR routes ACP bridge diagnostics to stderr in `serveAcpGateway()` and early CLI startup, with unit coverage for ACP startup log routing and ACP client exceptions.

## Best Possible Solution

Land a refreshed current-main ACP/CLI fix that protects bridge stdout before startup diagnostics, preserves interactive client stdout including variadic `--server-args`, and enables console capture wherever stderr routing is relied on.

## Reproduction Assessment

Yes. Source inspection reproduces the PR-head regressions through `openclaw acp client --server-args --url ...` and the direct `serveAcpGateway()` entrypoint without console capture.

## Solution Assessment

No. Early stderr routing is directionally correct, but this branch mishandles variadic ACP client arguments and relies on stderr routing in a direct entrypoint before installing console capture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

  - confidence: 0.86

  - confidence: 0.78
