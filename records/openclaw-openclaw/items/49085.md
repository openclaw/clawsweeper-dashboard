## Summary

Keep this PR open because it targets a real ACP stdout-safety bug, but the current patch still has correctness blockers and lacks after-fix real behavior proof.

## What This Changes

The PR routes ACP bridge console logging to stderr in the server and CLI startup paths and adds startup regression tests for ACP stdout hygiene.

## Best Possible Solution

Land a focused ACP/CLI fix that routes captured logs to stderr before registration for bridge invocations, preserves all `acp client` stdout behavior, and installs capture for direct ACP server runs before relying on forced stderr routing.

## Reproduction Assessment

Yes, source-reproducible. The current-main gap is visible in `runCli()` registration order, and the PR regression follows directly from `openclaw acp client --server-args --url ...` through the PR's ACP option parser.

## Solution Assessment

No. Early stderr routing is the right direction, but the detector must treat `acp client` as client mode before consuming variadic server args, and the standalone server path needs console capture before `routeLogsToStderr()` can affect console output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve acp client server args on stdout:** `src/cli/run-main.ts:159`
  - body: The detector treats `--server-args` as a normal value flag, but the documented `openclaw acp client --server-args --url ...` form passes an option-shaped value. `getCommandPositionalsWithRootOptions()` returns null, `isAcpClient` stays false, and the interactive client's `console.log` output is redirected to stderr.
  - confidence: 0.86
- **[P2] Enable capture for the standalone ACP server:** `src/acp/server.ts:18`
  - body: `routeLogsToStderr()` only flips the flag used by patched console forwarding; it does not redirect `console.*` unless `enableConsoleCapture()` has already run. The direct server entrypoint calls `serveAcpGateway()` without capture setup, so direct ACP server runs can still write console output to stdout.
  - confidence: 0.78
