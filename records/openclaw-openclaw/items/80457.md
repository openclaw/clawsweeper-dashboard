## Summary

Keep open: the refactor is plausible, but the branch currently fails typecheck because `src/agents/tool-loop-detection.ts` calls `stableStringify` after deleting its local definition without importing the shared helper. The posted proof is copied helper-script output and CI's real-behavior gate is still failing, so this needs contributor follow-up before merge.

## What This Changes

This PR moves three agent-local `stableStringify` implementations into `src/agents/stable-stringify.ts` and updates cache trace, models-config fingerprints, and tool-loop hashing to use it.

## Best Possible Solution

Keep the central helper direction only after importing it in `tool-loop-detection` and adding focused serialization/hash coverage for the call sites whose fingerprint bytes can change.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR regression: CI already reports `TS2304` on the PR branch, and the diff shows `stableStringify` used in `tool-loop-detection` without a replacement import.

## Solution Assessment

Is this the best way to solve the issue? Not yet; centralizing the helper can be maintainable, but this patch needs the missing import and targeted tests before the refactor is safe to land.
