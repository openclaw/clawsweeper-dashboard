## Summary

Keep open: the PR fixed the previous unused-export blocker, but the current cron-parentage implementation still misses subagents whose `spawnedBy` value is itself a cron-run key when the parent cron row is absent from `sessions.json`, which is part of the linked issue shape.

## Best Possible Solution

Keep the Dreaming-only primary transcript filter, but update the SDK classifier to treat a cron-looking spawnedBy key as cron-descended before requiring a parent store entry, with a focused regression test and refreshed proof.

## Solution Assessment

No. The archive boundary and SDK-owned classifier are the right fix shape, but the classifier should recognize cron-run spawnedBy keys even when the parent store entry is absent.
