## Summary

Keep open. The PR addresses a real Dreaming corpus bug, but it currently narrows a shared session listing helper that memory_search and session export also use, so merging as-is would stop reset/deleted archives from being indexed and can remove existing archive rows on a full session sync; external real behavior proof is also missing.

## What This Changes

The PR changes Dreaming/session ingestion to skip archived transcripts, classify cron-descended subagents, persist a cron parent marker, and expose related memory-host SDK helpers.

## Best Possible Solution

Keep listSessionFilesForAgent usage-counted for memory_search/export, add a Dreaming-specific primary-transcript listing or caller-side filter, retain lineage-aware cron classification, and require focused tests plus redacted real corpus proof before merge.

## Reproduction Assessment

Yes at source level for the blocking regression: current main routes memory_search/export through listSessionFilesForAgent, while the PR narrows that helper to primary transcripts and the sync plan deletes omitted paths as stale. I did not run tests or live proof because this review is read-only.

## Solution Assessment

No. The cron lineage part is a plausible fix, but the archive exclusion belongs at a Dreaming-specific ingestion boundary rather than the shared helper that memory_search relies on.
