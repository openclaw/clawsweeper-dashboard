## Summary

Keep open. Current main still exposes Microsoft Teams as one top-level Bot Framework credential/default account, and the closed duplicate plus the open Microsoft tracker show this remains the canonical multi-bot identity request rather than cleanup-closeable noise.

## What This Changes

Request support for multiple Microsoft Teams/Azure Bot Framework bot identities on one OpenClaw gateway, routable to different agents.

## Best Possible Solution

Add bundled msteams multi-account support with channels.msteams.accounts and defaultAccount, preserving the current top-level config as the implicit default and keeping agent selection in bindings[].match.accountId.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection, not as a runtime bug: current config type/schema/setup/runtime code only resolves msteams to a single default account, while multi-agent routing depends on accountId for per-agent identity.

## Solution Assessment

Is this the best way to solve the issue? No for the exact proposed array plus agentBinding shape; the narrower maintainable solution is the existing account-map/defaultAccount contract with routing left in bindings.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because multi-account Teams support would change credential selection, Bot Framework JWT validation, webhook routing, and delegated/SSO token isolation.

Concerns:

- **[medium] Tie account routing to verified bot identity:** `extensions/msteams/src/monitor.ts:304`
  - body: The current runtime builds one Bot Framework JWT validator from one credential set. Multi-account support must validate the incoming token against the selected bot app identity before dispatching under an accountId.
  - confidence: 0.82
- **[medium] Scope Teams delegated and SSO tokens by account:** `extensions/msteams/src/token.ts:145`
  - body: Delegated tokens currently use one file, and SSO tokens are keyed by connectionName and userId without an account or tenant dimension. Multi-account support should isolate these stores per account/tenant.
  - confidence: 0.76
