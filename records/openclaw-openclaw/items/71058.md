## What This Changes

Request support for multiple Microsoft Teams/Azure bot identities on one OpenClaw gateway so different agents can appear as separate Teams bots.

## Best Possible Solution

Land a reviewed Teams accounts/defaultAccount implementation in the msteams plugin, preserving legacy root config and proving account-scoped credentials, routing, auth, and state behavior.

## Solution Assessment

Not yet. The issue's array plus agentBinding sketch is less aligned with existing OpenClaw routing than a Teams-owned accounts/defaultAccount model wired through bindings[].match.accountId, and the open candidate PR still needs maintainer acceptance.
