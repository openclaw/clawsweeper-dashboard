## Summary

Keep open: current main and the latest release still model Microsoft Teams as a single default-account bot credential set, and the only duplicate found was closed back to this issue as the canonical request.

## What This Changes

The issue requests support for configuring multiple Microsoft Teams bot identities on one OpenClaw gateway and binding those identities to agents.

## Best Possible Solution

Add a maintainer-approved Teams-owned account model that preserves the current singleton config as the default account, routes through `bindings[].match.accountId`, and includes upgrade/doctor behavior plus account-scoped credential and token isolation.

## Reproduction Assessment

Yes. Source inspection shows current main resolves one Teams credential set from `channels.msteams` in schema, setup, monitor, and outbound send paths, so multiple Teams bot identities cannot be configured today.

## Solution Assessment

No for the exact proposed array plus `agentBinding` shape. The safer OpenClaw-shaped direction is a named `channels.msteams.accounts` model with explicit `defaultAccount`, existing `bindings[].match.accountId`, migration/doctor handling, and account-scoped auth state.
