## Summary

Keep open: current main still exposes Microsoft Teams as one singleton bot/app credential path, and the duplicate/tracker search did not find a merged or viable canonical implementation that supersedes this issue.

## What This Changes

Request support for multiple Microsoft Teams bot/app identities on one OpenClaw gateway so different agents can appear as distinct Teams bots.

## Best Possible Solution

Keep this open for a Teams-owned `channels.msteams.accounts.<id>` design with default-account migration, account-scoped credentials/webhooks/auth state, bindings, setup/docs, and focused tests.

## Reproduction Assessment

Yes for source-level reproduction: current main's Teams config schema, docs, and inbound routing show only one credential/account path. I did not run a live Teams gateway because this is a feature request and the missing config shape is directly visible in source.

## Solution Assessment

No for the exact array plus `agentBinding` sketch; the more maintainable OpenClaw-shaped path is named `channels.msteams.accounts.<id>` wired through existing `accountId` bindings, with maintainer approval for API and migration details.
