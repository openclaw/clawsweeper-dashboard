## Summary

Keep open: the core stale-deny fix is well motivated and has after-fix live output, but the changed warning path has a concrete diagnostic regression for official external plugin ids. There is also overlapping fuller work at https://github.com/openclaw/openclaw/pull/80471 that maintainers should coordinate before landing either branch.

## What This Changes

The PR changes config validation so missing `plugins.deny` plugin ids become stale-config warnings, adds regression coverage, and updates the changelog.

## Best Possible Solution

Make missing `plugins.deny` ids warn-only without official install hints, add focused coverage for an official external denied id, then choose one implementation path with the broader stale-deny work in https://github.com/openclaw/openclaw/pull/80471.

## Reproduction Assessment

Yes. Source inspection shows current `main` makes missing `plugins.deny` ids fatal, and the PR finding is source-reproducible because the shared warn-only helper emits official install hints before stale-config wording.

## Solution Assessment

Mostly yes, but not as-is. Demoting missing deny ids at validation is the narrow right layer, but denylist warnings should suppress install guidance and have a focused official-external deny test.

## Review Findings
