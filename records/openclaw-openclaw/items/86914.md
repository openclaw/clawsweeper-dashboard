The issue requests a Telegram exec approval option to keep a prior bot message visible before the approval prompt appears.

Keep the shipped draft-rotation behavior as the canonical fix; open a new report only if v2026.6.6 or newer still removes a prior answer bubble before Telegram approval prompts.

## Solution Assessment

Yes. Preserving the existing answer draft when rotating into tool or approval progress is narrower than adding a new config flag, because it fixes the lost-message UX by default while keeping tool-progress-only drafts cleanup intact.
