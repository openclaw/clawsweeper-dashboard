## Summary

Keep open. The PR is the right canonical implementation path for the closed CA-bundle issue, but the proposed marker set still misses default NVS installs and the user-facing fix needs a changelog entry before merge.

## What This Changes

The PR broadens bootstrap CA bundle auto-injection from nvm-only detection to a path-marker list for fnm, volta, asdf, n, nodenv, nodebrew, and nvs, with matching bootstrap tests.

## Best Possible Solution

Update the shared version-manager detection coverage to include default `/.nvs/` paths, keep bootstrap and daemon marker tests aligned, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns false for non-nvm exec paths, and the PR branch still returns false for a default NVS path like `/home/user/.nvs/node/22.14.0/x64/bin/node` because it checks `/nvs/`.

## Solution Assessment

No. The direction is maintainable, but the exact patch copies an incomplete NVS marker; the safer final fix is to cover `/.nvs/` and synchronize the adjacent marker lists/tests before merge.
