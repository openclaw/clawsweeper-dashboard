## Summary

Keep this PR open: the zh-CN screenshots are useful, but the patch currently regresses existing Agents translations in other locales and leaves hardcoded or mispluralized text in the tools/skills panels.

## What This Changes

The PR extracts Agents overview/tools/skills panel strings into i18n keys, adds zh-CN/en translations, updates generated locale metadata, and adjusts one zh-CN Agents tab test.

## Best Possible Solution

Keep the extraction, but regenerate or repair the locale bundles so existing non-zh Agents keys are preserved and the tools/skills labels use complete locale-aware strings.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a localization PR. The supplied screenshots do give high-confidence after-fix proof for the zh-CN overview/tools paths they show, while source review exposes separate regressions.

## Solution Assessment

No. Extracting strings is the right direction, but this branch needs to preserve existing generated locale keys and use locale-aware full sentences/count labels before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the existing non-zh Agents translations:** `ui/src/i18n/locales/de.ts:245`
  - body: The regenerated non-English bundles replace the existing `agents` object with only `overview` and `skills`. Current UI still reads `agents.noAgents`, `agents.tabs.*`, `agents.context.*`, `agents.channels.*`, `agents.cronPanel.*`, and `agents.files.*`, so locales like German now fall back to English across the Agents page. Regenerate or preserve those keys while adding the new ones.
  - confidence: 0.95
- **[P2] Use the full localized all-skills sentence:** `ui/src/ui/views/agents-panels-tools-skills.ts:813-814`
  - body: This branch adds `agents.skills.allEnabled`, but the rendered no-allowlist callout still hardcodes English around `agentTools.enabledSuffix`. In zh-CN this produces mixed English/Chinese instead of the translated sentence, so render the full `agents.skills.allEnabled` string here.
  - confidence: 0.93
- **[P2] Keep English count labels plural:** `ui/src/ui/views/agents-panels-tools-skills.ts:568-586`
  - body: Passing the same translated label as both singular and plural disables pluralization for every locale. English tool groups now render as `4 Tool` / `4 Enabled Tool`, so the zh-CN no-suffix behavior needs a locale-aware count string or per-locale plural handling rather than a global singular-only override.
  - confidence: 0.9
