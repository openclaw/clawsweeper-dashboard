## Summary

Keep open: the i18n direction is useful, but this PR currently regresses existing non-English Agents translations, breaks English count plurals, and leaves one zh-CN skills callout mixed with English; the claimed screenshot proof is also not attached for inspection.

## What This Changes

The PR extracts hardcoded Agents overview and tools/skills panel strings into i18n keys and adds Simplified Chinese translations plus regenerated locale metadata.

## Best Possible Solution

Keep the i18n extraction, but regenerate locale output without deleting existing keys, make count labels locale-aware, use full translated sentences for zh-CN, and then attach inspectable real UI proof.

## Reproduction Assessment

Yes for the review findings: the PR head source shows missing generated locale keys, globally disabled plural labels, and a hardcoded mixed-language skills callout without needing a runtime environment.

## Solution Assessment

No; extracting keys is the right approach, but the current patch should preserve generated locale objects and avoid composing localized prose from English fragments.
