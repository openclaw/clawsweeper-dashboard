## What This Changes

The PR adds a transcript `promptTokensFresh` flag, gates `/status` Context promotion on it, and adds a regression test for cache-heavy legacy transcript usage.

## Best Possible Solution

Separate the status formatter's current Context numerator from aggregate transcript cache/token totals, using a latest/current prompt snapshot for Context and keeping cumulative cache totals on the Cache/Tokens lines.

## Reproduction Assessment

No high-confidence live reproduction of the original issue is present. Source inspection does show the PR's proposed freshness gate is insufficient because the parser sets the same flag for derived usage totals.

## Solution Assessment

No. The patch is close to the right boundary, but it reuses a freshness bit that does not distinguish raw per-turn totals from derived cache-inclusive usage, so the safer fix is to split current prompt context from aggregate transcript totals.
