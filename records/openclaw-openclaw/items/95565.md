## Label Justifications

- `P2`: This is a normal-priority WebChat history improvement affecting long-session continuity without evidence of transcript deletion.
- `impact:session-state`: The issue concerns transcript/session history becoming inaccessible in the active WebChat view after context compaction.

## Summary

Keep open: current main has pieces of this surface, including a compaction divider and a cursor-capable HTTP history endpoint, but WebChat still loads one bounded `chat.history` page and has no active-chat path to fetch older pre-compaction transcript entries.

## What This Changes

Request for WebChat to paginate older transcript messages beyond the active context window and mark archival pre-compaction history in the chat UI.

## Best Possible Solution

Add a bounded WebChat history paging path that reuses the existing Gateway history model where possible and clearly distinguishes active model context from archival pre-compaction transcript entries.

## Reproduction Assessment

Yes, at source level: WebChat requests one fixed-limit `chat.history` page and has no cursor/all-entry request path for older transcript pages before the active branch. I did not run a live browser compaction flow in this read-only review.

## Solution Assessment

Unclear. The existing HTTP cursor endpoint and transcript index all-entry view are promising building blocks, but maintainers should choose the durable WebChat/Gateway contract before implementation.
