## Summary

Keep open for normal PR review: current main still reaches memory-core runtime cron reconciliation before proving an event is managed dreaming, and this PR appears to narrowly quiet only the no-context ordinary heartbeat path while preserving genuine gateway and managed cron warnings.

## What This Changes

The PR adds a memory-core runtime guard, focused dreaming cron tests, and a changelog entry so hook contexts without gateway cron context do not emit managed-dreaming cron-unavailable warnings.

## Best Possible Solution

Land the focused memory-core guard after maintainer review and CI, keeping cron-unavailable warnings tied to managed events or gateway cron-management contexts.

## Reproduction Assessment

Yes, from source inspection. Register memory-core with dreaming enabled but no gateway cron context, then invoke a heartbeat `before_agent_reply` with no managed dreaming token or pending event; current main reaches runtime cron reconciliation before returning.

## Solution Assessment

Yes. The PR checks managed-dreaming intent before reconciliation and only skips when there is no managed event and no cron-management context, which is the narrowest maintainable fix for the noisy warning path.
