Full review comments:

- **[P1] Constrain promoted file-share links:** `extensions/nextcloud-talk/src/monitor.ts:196-198`
  - body: `parseRichObjectContent` accepts any `http(s)` `parameters[*].link` and forwards it as `MediaUrls`. That lets a signed webhook body turn ordinary chat content into arbitrary public remote media fetches; constrain the link to the configured Nextcloud origin or reconstruct a trusted file-download URL before promoting it.
  - confidence: 0.84
- **[P1] Forward MIME hints with media URLs:** `extensions/nextcloud-talk/src/monitor.ts:198`
  - body: The rich-object file parameter includes `mimetype`, but the PR only pushes the link. Downstream context then defaults `MediaTypes` to `application/octet-stream`, and media selection relies on MIME or URL extension, so extensionless Nextcloud shares like `/f/42` can still be skipped by image/audio/video handling.
  - confidence: 0.87
- **[P2] Return parsed text for valid rich objects:** `extensions/nextcloud-talk/src/monitor.ts:211-212`
  - body: When a valid rich-object body has no file URL and no placeholder substitution, this returns `null` and the caller sends the raw JSON envelope to the agent. Real Nextcloud Talk traffic can encode even plain messages as `{message, parameters: []}`, so successful parsing should use the human-readable `message` field instead of falling back to raw JSON.
  - confidence: 0.75
