## What This Changes

The PR adds Nextcloud Talk rich-object parsing for file-share webhook content, forwards parsed media URLs into inbound context, and adds signed webhook parser tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Forward MIME hints with rich-object media:** `extensions/nextcloud-talk/src/inbound.ts:281`
  - body: `parseRichObjectContent` reads each file parameter's `mimetype`, but the inbound context only receives `MediaUrls`. `finalizeInboundContext` fills missing media types with `application/octet-stream`, and media selection then classifies by MIME before falling back to URL extension. Nextcloud share links like `/f/42` are commonly extensionless, so these files can still be skipped by image/audio/video handling despite being forwarded as URLs. Please carry aligned MIME types alongside the URLs.
  - confidence: 0.88
- **[P2] Validate promoted file links against the Nextcloud origin:** `extensions/nextcloud-talk/src/monitor.ts:196`
  - body: The parser accepts any `http(s)` `param.link` and forwards it as media. Downstream fetches have SSRF guards, but this still promotes signed webhook content into arbitrary public media fetch input. File-share links should be validated against the configured Nextcloud backend/account origin, or another explicit trusted-origin rule, before being added to `mediaUrls`.
  - confidence: 0.74
