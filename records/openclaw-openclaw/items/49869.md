## Summary

Keep this PR open: it addresses a real Nextcloud Talk attachment parsing bug that is still present on current main and the latest release, but the patch is not merge-ready because it drops MIME facts, still leaks raw structured JSON for valid bodies without placeholders, and lacks after-fix real file-share proof.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing, forwards extracted file links as inbound MediaUrls, relaxes media-only drops, and adds signed webhook parser tests.

## Best Possible Solution

Land a unified plugin-local structured-body parser that emits display text plus aligned MediaUrls/MediaTypes, coordinates with https://github.com/openclaw/openclaw/pull/79397, and includes focused tests plus redacted real file-share proof.

## Reproduction Assessment

Yes, as a source-level path: a signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file-share body still becomes raw text on current main with no MediaUrls or MediaTypes. I did not run a live webhook reproduction in this read-only review.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward MIME types, return parsed display text for every valid structured body, coordinate with the active structured mention parser, and include real file-share proof before merge.
