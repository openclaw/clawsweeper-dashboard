## Summary

Keep open: current main and the latest release still pass Nextcloud Talk structured webhook content through as raw text, so the PR contains useful bug-fix work, but it is not ready to merge because it drops MIME hints, still exposes raw JSON for valid no-placeholder bodies, and lacks real file-share proof. The overlapping structured mention parser in https://github.com/openclaw/openclaw/pull/79397 also needs coordination before landing divergent parser behavior.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing for file shares, forwards extracted URLs as inbound media, permits media-only inbound messages, and adds signed webhook parser tests.

## Best Possible Solution

Land a unified plugin-local structured-body parser that emits display text plus aligned media URL/MIME facts, composes with the mention parser, and is backed by focused tests plus redacted live file-share proof.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file share; current main and v2026.5.12 pass that JSON through as text and never set MediaUrls or MediaTypes.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward per-file MIME types, return parsed display text for every valid structured body, coordinate with the active mention parser, and include real after-fix file-share proof.

Return displayText for valid structured bodies and reserve null for malformed or non-structured input.
