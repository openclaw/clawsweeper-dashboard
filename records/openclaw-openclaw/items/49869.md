## Summary

Keep this PR open: it addresses a real Nextcloud Talk file-share parsing bug, but the branch still drops MIME facts, has a structured-body fallback edge case, and lacks contributor real-environment proof.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing for webhook file shares, threads extracted media URLs into inbound context, and adds signed-webhook parser tests.

## Best Possible Solution

Land a repaired plugin-local parser that emits display text plus aligned URL/MIME media facts, preserves parsed text for all valid structured bodies, coordinates with the mention parser work, and includes redacted live file-share proof.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk Create webhook whose object.content is the documented rich-object JSON file share; current main exposes that JSON as text and does not add media facts.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward per-file MIME types, return parsed display text for all valid structured bodies, and provide real after-fix proof before merge.

## Review Findings
