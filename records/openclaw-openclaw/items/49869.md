## Summary

Keep open: current main still leaves Nextcloud Talk rich-object file shares as raw webhook text, but this PR is not merge-ready because it lacks successful real attachment proof and still has a concrete MIME propagation bug.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing in the webhook monitor, threads extracted file links into inbound MediaUrls, relaxes media-only drops, and adds signed webhook parser tests.

## Best Possible Solution

Land one plugin-local structured-body parser that emits display text plus aligned MediaUrls and MediaTypes, coordinated with https://github.com/openclaw/openclaw/pull/79397 and backed by focused tests plus redacted real file-share proof.

## Reproduction Assessment

Yes, as a source-level path: a signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file-share body still becomes raw text on current main with no MediaUrls or MediaTypes. I did not run a live webhook reproduction in this read-only review.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward MIME types, return parsed display text for every valid structured body, and coordinate with the active structured mention parser before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Forward MIME types with rich-object media URLs:** `extensions/nextcloud-talk/src/inbound.ts:278`
  - body: This only sends MediaUrls into the inbound context. Without aligned MediaTypes, finalizeInboundContext pads application/octet-stream, and extensionless Nextcloud share URLs like /f/42 can remain unknown before image/audio/video selection. Preserve each rich-object mimetype and pass MediaTypes with MediaUrls.
  - confidence: 0.91
- **[P3] Return parsed text for valid rich objects:** `extensions/nextcloud-talk/src/monitor.ts:211-212`
  - body: When JSON parses as a structured Talk body but has no file URL and no matched placeholder, such as {"message":"hello","parameters":[]}, this returns null and exposes the raw JSON envelope to the agent. Return displayText for valid structured bodies and reserve null for malformed or non-structured input.
  - confidence: 0.87
