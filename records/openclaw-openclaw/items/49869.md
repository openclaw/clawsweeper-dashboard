## Summary

Keep open: current main and the latest release still do not parse Nextcloud Talk rich-object attachments, but this PR is not merge-ready because it forwards URLs without aligned MIME hints, lacks trusted-origin validation, and still needs real file-share proof.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing for inbound file shares, threads extracted links through `mediaUrls`, relaxes media-only inbound dropping, and adds signed webhook tests.

## Best Possible Solution

Land one coordinated Nextcloud Talk structured-content parser that emits clean text plus aligned URL/MIME metadata, validates promoted file links against the configured backend trust boundary, and includes redacted live file-share proof.
