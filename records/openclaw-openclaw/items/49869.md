## Summary

Keep open: current main and v2026.6.10 still do not parse Nextcloud Talk rich-object file attachments, so this PR has unique useful work, but it is not merge-ready because it forwards URL-only media metadata, lacks an explicit backend-origin trust rule, and still lacks live proof of a real file/image share reaching the agent.

## What This Changes

The branch adds Nextcloud Talk rich-object parsing for inbound file shares, threads extracted links into `MediaUrls`, allows media-only inbound messages, and adds signed webhook parsing tests.

## Best Possible Solution

Land one coordinated Nextcloud Talk structured-content parser that emits clean text plus aligned URL/MIME metadata, validates promoted file links against the configured backend trust boundary, and includes redacted live file-share proof.

## Solution Assessment
