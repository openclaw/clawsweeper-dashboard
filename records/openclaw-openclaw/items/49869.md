## Summary

Keep open. The PR targets a real current-main gap and is useful, but it should not merge yet because it drops the rich-object MIME metadata that the media pipeline needs for extensionless Nextcloud share links, and it still surfaces raw JSON for some valid rich-object messages.

## What This Changes

This PR parses Nextcloud Talk rich-object webhook content into display text and media URLs, threads media URLs into inbound context, and adds webhook parsing tests.

## Best Possible Solution

Land a repaired PR that parses signed Nextcloud rich-object file shares, preserves aligned MIME metadata through inbound context, keeps the existing guarded fetch model, and adds regression plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a rich-object `object.content` string is passed through as raw text with no media URLs, and the PR body provides a concrete signed-webhook payload shape for the failing case.

## Solution Assessment

No. Parsing rich objects in the Nextcloud Talk plugin is the right direction, but this patch needs to propagate `mimetype` as `MediaTypes`, handle well-formed rich objects without placeholders, and add the required changelog entry before merge.
