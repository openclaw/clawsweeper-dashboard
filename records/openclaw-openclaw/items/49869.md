## Summary

Keep open: the underlying Nextcloud Talk file-share bug is real on current main, but this PR is not merge-ready because it lacks real file-share proof and still has concrete parser/media-routing defects.

## What This Changes

The PR adds a Nextcloud Talk rich-object parser, forwards extracted file links as inbound media URLs, relaxes the empty-body guard for media-only messages, and adds signed-webhook parser tests.

## Best Possible Solution

Land a unified Nextcloud Talk structured-body parser that emits display text plus aligned media URL/MIME facts, composes with structured mention parsing, and is backed by focused tests plus redacted live file-share proof.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file share; current main passes that JSON through as text and has no media facts for the file.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward per-file MIME types, return parsed display text for every valid structured body, coordinate with the structured mention parser, and include real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Forward MIME types with rich-object media URLs:** `extensions/nextcloud-talk/src/inbound.ts:281`
  - body: This only sends MediaUrls into the inbound context. Without aligned MediaTypes, finalizeInboundContext pads application/octet-stream, and extensionless Nextcloud share URLs like /f/42 can stay unknown before image/audio/video selection. Preserve each rich-object mimetype and pass MediaTypes with MediaUrls.
  - confidence: 0.91
- **[P3] Return parsed text for valid rich objects:** `extensions/nextcloud-talk/src/monitor.ts:211-212`
  - body: When JSON parses as a rich-object body but has no file URL and no matched placeholder, this returns null, so the caller exposes raw JSON instead of the parsed message. Return displayText for valid structured bodies and reserve null for malformed or non-structured input.
  - confidence: 0.86
