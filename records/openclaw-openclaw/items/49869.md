## Summary

Keep open: current main still lacks Nextcloud Talk rich-object file parsing, but this PR is not merge-ready because the central file-share path is not proven in a real setup and the patch still has MIME propagation, structured-body fallback, parser-overlap, and media-link trust blockers.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing for webhook file shares, forwards extracted media URLs into inbound context, permits media-only inbound messages, and adds signed webhook parser tests.

## Best Possible Solution

Use one Nextcloud Talk structured-body parser that returns display text, mention facts, aligned media URLs and MIME types, validates media origin against the trusted Nextcloud source, and is proven with a real file-share webhook.

## Reproduction Assessment

Yes, at source level: current main passes Nextcloud Talk `object.content` through as raw text, drops empty bodies, and never emits `MediaUrls`. A live tester did not reproduce the actual file-share webhook reaching the patched path, so live end-to-end proof is still missing.

## Solution Assessment

No, not as currently proposed. The plugin boundary is right, but the PR should carry MIME types, validate media-link origin, coordinate with the structured mention parser, and include real file-share proof before merge.
