## Summary

Keep this PR open: it targets a real Nextcloud Talk attachment parsing bug, but the branch still has two concrete correctness gaps and the latest live report did not prove the file-share media path end to end.

## What This Changes

The PR adds a Nextcloud Talk rich-object parser, threads extracted file links into inbound media URLs, relaxes the empty-body guard for media-only messages, and adds signed webhook parser tests.

## Best Possible Solution

Land a unified Nextcloud Talk structured-body parser that emits display text plus aligned media URL/MIME facts, composes with the mention parser work, and has focused tests plus redacted live file-share proof.

## Reproduction Assessment

Yes, source-reproducible. A signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file share still becomes raw text on current main and has no MediaUrls/MediaTypes path.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this branch should preserve MIME facts, return parsed text for every valid structured body, and coordinate with the active structured mention parser before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Forward MIME types with rich-object media URLs:** `extensions/nextcloud-talk/src/inbound.ts:278`
  - body: This only sends MediaUrls into the inbound context. Without aligned MediaTypes, finalizeInboundContext pads application/octet-stream, and extensionless Nextcloud share URLs like /f/42 can stay unknown before image/audio/video selection. Preserve each rich-object mimetype and pass MediaTypes with MediaUrls.
  - confidence: 0.91
- **[P3] Return parsed text for valid rich objects:** `extensions/nextcloud-talk/src/monitor.ts:211-212`
  - body: When JSON parses as a rich-object body but has no file URL and no matched placeholder, this returns null, so the caller exposes raw JSON instead of the parsed message. Return displayText for valid structured bodies and reserve null for malformed or non-structured input.
  - confidence: 0.86
