## Summary

Keep open: the Nextcloud Talk file-share bug is still present on current main and in v2026.5.12, but this PR still needs targeted fixes and real file-share proof before merge.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing in the webhook monitor, threads mediaUrls into inbound context, relaxes media-only drops, and adds signed webhook parser tests.

## Best Possible Solution

Land a unified plugin-local structured-body parser that emits display text plus aligned MediaUrls/MediaTypes, composes with the mention parser, and is backed by focused tests plus redacted live file-share proof.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk Create webhook whose object.content is the documented JSON rich-object file-share body; current main passes that string through as text and never sets MediaUrls or MediaTypes.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward MIME types, return parsed display text for every valid structured body, coordinate with the active mention parser, and include real file-share proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Forward MIME types with rich-object media URLs:** `extensions/nextcloud-talk/src/inbound.ts:281`
  - body: This only sends MediaUrls into the inbound context. Without aligned MediaTypes, finalizeInboundContext pads application/octet-stream, and extensionless Nextcloud share URLs like /f/42 can remain unknown before image/audio/video selection. Preserve each rich-object mimetype and pass MediaTypes with MediaUrls.
  - confidence: 0.91
- **[P3] Return parsed text for valid rich objects:** `extensions/nextcloud-talk/src/monitor.ts:211-212`
  - body: When JSON parses as a structured Talk body but has no file URL and no matched placeholder, such as {"message":"hello","parameters":[]}, this returns null and exposes the raw JSON envelope to the agent. Return displayText for valid structured bodies and reserve null for malformed or non-structured input.
  - confidence: 0.87
