## What This Changes

This branch adds Nextcloud Talk rich-object parsing for webhook content, threads extracted file links into inbound context, relaxes media-only message gating, and adds webhook parser tests.

## Best Possible Solution

Land one coordinated Nextcloud Talk structured-body/media parser that emits resolved text plus aligned URL/MIME media facts, preserves malformed/plain-text fallback behavior, and has redacted live file-share proof.

## Reproduction Assessment

Yes at source level: current main passes documented JSON rich-object `object.content` through as raw text and has no inbound media URL path. I did not establish a live file-share webhook repro in this read-only review.

## Solution Assessment

No, not as currently proposed. The parser direction is right, but the mergeable solution should also carry MIME types and coordinate with the overlapping structured-body parser before landing.
