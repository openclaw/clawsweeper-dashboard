## Summary

Keep this PR open: current main still has the reported Nextcloud Talk attachment gap, but the proposed branch is not ready to merge because it drops MIME facts, can still expose raw JSON for valid rich objects, and lacks required real Nextcloud Talk file-share proof. The related open mention-parsing PR covers a neighboring structured-body problem, not this file-share media handoff.

## What This Changes

The PR parses Nextcloud Talk rich-object webhook content into display text and media URLs, relaxes media-only inbound handling, and adds Nextcloud Talk webhook parsing tests.

## Best Possible Solution

Land a repaired plugin-local Nextcloud Talk fix that parses file-share rich objects into human-readable text plus aligned URL and MIME media facts, preserves parsed text for valid non-file rich objects, and includes redacted real file-share proof.

## Reproduction Assessment

Yes. Source inspection on origin/main gives a high-confidence path: send a Nextcloud Talk webhook whose object.content is the documented rich-object JSON file share, and current main passes that JSON through as text without MediaUrls.

## Solution Assessment

No. Plugin-local parsing is the right boundary, but this patch should forward per-file MIME types, return parsed display text for all valid rich objects, and include real after-fix Nextcloud Talk file-share proof before merge.
