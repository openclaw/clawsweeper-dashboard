## Summary

Keep open: current main and the latest release still do not parse Nextcloud Talk rich-object file shares into inbound media, but this PR is not merge-ready because it forwards URL-only attachments without MIME alignment, lacks a plugin-level trusted-origin rule, and still needs live file-share proof.

## What This Changes

The branch adds a Nextcloud Talk webhook rich-object parser, carries file links through inbound media URLs, relaxes the media-only inbound guard, and adds signed webhook tests.

## Best Possible Solution

Land a coordinated Nextcloud Talk structured-content parser that emits clean text plus aligned URL and MIME metadata, validates file links against the configured backend origin, and includes redacted live file-share proof.

## Solution Assessment

No as-is. Plugin-local rich-object parsing is the right layer, but the best fix must preserve MIME metadata, tighten file-link trust, and coordinate with the adjacent structured mention parser instead of landing a partial parallel parser.
