## Summary

Keep this PR open. Current main still does not parse Nextcloud Talk rich-object file shares, but the proposed patch is not merge-ready because it promotes rich-object links without a trusted-origin constraint, omits MIME hints needed by downstream media routing, and still exposes raw JSON for valid rich objects without matching placeholders.

## What This Changes

The PR adds Nextcloud Talk webhook rich-object parsing, forwards extracted file links as inbound media URLs, relaxes the empty-body guard for media-only messages, and adds webhook parsing tests.

## Best Possible Solution

Land a focused revision that parses Nextcloud Talk object.content rich objects in the plugin, resolves display text for all valid rich-object messages, extracts only trusted same-origin Nextcloud file links, forwards aligned MediaUrls and MediaTypes, allows media-only inbound delivery, and adds regression tests for file shares, malformed JSON fallback, origin rejection, extensionless MIME-backed files, no-placeholder rich objects, and mention-placeholder overlap tracked by #66700.

## Reproduction Assessment

Yes. A signed Create webhook whose object.content is a JSON rich object with a file parameter reproduces the current-main behavior: monitor.ts assigns the raw JSON string to text, types.ts has no mediaUrls field, and inbound.ts drops empty text before any media payload can be forwarded.

## Solution Assessment

No. The PR is pointed at the right plugin boundary, but the best fix should also constrain promoted links to the trusted Nextcloud origin, forward MIME hints as MediaTypes, and return parsed message text for every valid rich object instead of relying on the current early-null fallback.

Full review comments:

- **[P1] Constrain promoted file links to the trusted backend:** `extensions/nextcloud-talk/src/monitor.ts:193-198`
  - body: The parser accepts any http(s) parameters[*].link from object.content and forwards it as MediaUrls. That lets an allowed sender craft rich-object JSON that triggers arbitrary outbound media fetch attempts; restrict extracted links to trusted Nextcloud rich-object payloads and the configured/backend origin before promoting them.
  - confidence: 0.82
- **[P2] Forward MIME hints with extracted media URLs:** `extensions/nextcloud-talk/src/monitor.ts:193-198`
  - body: Only mediaUrls are emitted, so extensionless Nextcloud share links like /f/42 have no MIME or extension for media-understanding classification and can be treated as unknown. Collect the file parameter mimetype and pass aligned MediaTypes with MediaUrls.
  - confidence: 0.9
- **[P2] Return parsed text for rich objects without substitutions:** `extensions/nextcloud-talk/src/monitor.ts:209-213`
  - body: For a valid rich object whose message has no matching placeholders, parseRichObjectContent returns null and the caller surfaces the raw JSON string. Once JSON has been validated as a rich object, return the trimmed message text even when no placeholders were replaced.
  - confidence: 0.87
