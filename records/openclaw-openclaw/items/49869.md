Full review comments:

- **[P1] Forward MIME hints with each media URL:** `extensions/nextcloud-talk/src/inbound.ts:278`
  - body: The rich-object parser reads file parameters that include `mimetype`, but the PR only forwards `MediaUrls`. `finalizeInboundContext` then pads the missing `MediaTypes` as `application/octet-stream`, and attachment selection falls back to URL extensions; common Nextcloud share links like `/f/12345` can classify as `unknown` and be skipped by image/audio/video understanding. Extract the file MIME values and pass aligned `MediaTypes` with `MediaUrls`.
  - confidence: 0.86
- **[P2] Constrain rich-object file links to the account origin:** `extensions/nextcloud-talk/src/monitor.ts:193-198`
  - body: This accepts any `http(s)` `param.link` from the signed webhook body and promotes it into media fetching. The webhook signature authenticates the sending backend, not that the rich-object link belongs to the configured Nextcloud account; keep file links on the account `baseUrl` origin or route them through an explicit account network policy before forwarding them as media.
  - confidence: 0.79
