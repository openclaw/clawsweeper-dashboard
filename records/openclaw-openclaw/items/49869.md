## Summary

Keep this PR open. Current main still lacks Nextcloud Talk rich-object attachment parsing, so the contribution is relevant, but the proposed diff has security-sensitive URL promotion and media contract gaps that should be revised before merge.

## What This Changes

The PR adds Nextcloud Talk rich-object parsing, attaches extracted file links as inbound media URLs, allows media-only inbound handling, and adds webhook parsing tests.

## Best Possible Solution

Revise this PR in the Nextcloud Talk plugin boundary: parse valid rich-object content, resolve display text for all rich objects, promote only trusted same-origin/backend file links, forward aligned MediaUrls and MediaTypes, allow media-only inbound delivery, and add regression coverage for origin rejection, MIME-backed extensionless files, no-placeholder rich objects, and the related mention-placeholder behavior tracked by #66700.

## Reproduction Assessment

Yes. A signed Create webhook with object.content set to a JSON rich object containing a file parameter reproduces current-main behavior: the content is delivered as raw text, mediaUrls are unavailable on the inbound message type, and empty-body media-only delivery cannot reach finalizeInboundContext.

## Solution Assessment

No. The PR is aimed at the right plugin boundary, but it is not the best mergeable fix until it constrains promoted links to the trusted Nextcloud origin, forwards MIME hints, and returns parsed message text for every valid rich object.

Full review comments:

- **[P1] Constrain promoted file links to the trusted backend:** `extensions/nextcloud-talk/src/monitor.ts:193-198`
  - body: The parser accepts any http(s) parameters[*].link from object.content and forwards it as MediaUrls. A signed webhook sender can craft rich-object JSON that causes downstream media fetch attempts, so restrict extracted links to trusted Nextcloud rich-object payloads and the configured or verified backend origin before promoting them.
  - confidence: 0.84
- **[P2] Forward MIME hints with extracted media URLs:** `extensions/nextcloud-talk/src/monitor.ts:193-198`
  - body: Only mediaUrls are emitted, so extensionless Nextcloud share links like /f/42 have no MIME or extension for media-understanding classification and can be treated as unknown. Collect each file parameter's mimetype and pass aligned MediaTypes with MediaUrls.
  - confidence: 0.9
- **[P2] Return parsed text for rich objects without substitutions:** `extensions/nextcloud-talk/src/monitor.ts:209-213`
  - body: For a valid rich object whose message has no matching placeholders, parseRichObjectContent returns null and the caller surfaces the raw JSON string. Once JSON has been validated as a rich object, return the trimmed message text even when no placeholders were replaced.
  - confidence: 0.87
