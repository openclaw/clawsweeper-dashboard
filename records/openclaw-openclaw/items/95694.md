## Summary

This is a useful MCP result-handling idea, but it is not necessary as an OpenClaw core item because current main already exposes a plugin middleware seam that can rewrite tool results before the model sees them, and VISION.md routes optional MCP/tool-result behavior to plugins or ClawHub unless a missing core API is proven.

## What This Changes

Request to add optional MCP tool-result caching that writes large MCP outputs into workspace files and tells the agent to read them from disk.

## Best Possible Solution

Publish this as an explicitly enabled ClawHub/plugin middleware that writes large MCP results to a session-scoped workspace artifact and replaces the model-facing result with a short path note; reopen core work only if that plugin proves a missing SDK field.

## Reproduction Assessment

Not applicable: this is a feature/scope request rather than a bug against an established OpenClaw contract. Source inspection confirms the current MCP path returns model-visible tool content and has truncation controls, but no failing current-main repro was run.

## Solution Assessment

No for core: adding a global `mcp.cacheResults` and `.mcp_cache` workspace side path is not the best OpenClaw-core solution while plugin tool-result middleware can rewrite MCP results before the model sees them. A ClawHub plugin is the narrower maintainable path unless it exposes a concrete missing SDK capability.
