- **[P1] Rebase before carrying root package metadata:** `package.json:3`
  - body: The PR head resets the root package to an older release snapshot (`2026.4.26` versus current main's `2026.5.20`) and rewrites package file/export metadata. That makes this more than an Amazon plugin addition and could regress published package contents or dependency resolution on upgrade.
  - confidence: 0.91
- **[P1] Gate Transcribe when the plugin config disables it:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never used to remove capabilities or block the handler, so a disabled Amazon Transcribe provider still advertises audio support and can be auto-selected/invoked. Make the disabled state prevent registration/capability exposure or hard-fail before any AWS call, with coverage for that config.
  - confidence: 0.94
- **[P1] Send Nova Sonic events in the documented sequence:** `extensions/amazon/nova-sonic/bridge.ts:184-197`
  - body: The bridge yields only a custom `sessionStart`, later raw `audioInput`, and `sessionEnd`, but Nova Sonic requires the named `promptStart`/`contentStart` containers and `promptName`/`contentName` on subsequent content events. As written, the Bedrock stream is unlikely to accept or route audio/tool events correctly.
  - confidence: 0.92
- **[P1] Resample telephony audio instead of only changing encoding:** `extensions/amazon/nova-sonic/bridge.ts:67-73`
  - body: Telephony input is 8 kHz mu-law while the Nova session declares 16 kHz PCM input, and Nova output is declared 24 kHz PCM while `onAudio` expects 8 kHz mu-law. This code only converts sample encoding, so calls will send and receive wrong-speed audio unless it reuses the existing resampling helpers or otherwise normalizes rates.
  - confidence: 0.89
- **[P1] Use a valid Polly PCM sample rate for telephony:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: `synthesizeTelephony` requests Polly `outputFormat: "pcm"` at 22050 Hz, but Polly's PCM output only accepts 8000 or 16000. Voice-call use of Amazon Polly will fail with a Polly validation error instead of returning telephony audio.
  - confidence: 0.93
- **[P2] Do not declare a guessed FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC pass-through path hard-codes `MediaSampleRateHertz` to 16000 even though Transcribe requires the declared sample rate to match the input audio. Common FLAC files at 44.1 or 48 kHz will fail or transcribe poorly; detect the true rate or normalize through ffmpeg.
  - confidence: 0.88
