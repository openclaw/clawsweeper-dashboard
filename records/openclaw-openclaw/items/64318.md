Full review comments:

- **[P1] Drop stale root package rewrites:** `package.json:3`
  - body: The branch resets root package metadata to an older 2026.4.26 base and removes current package file/export entries while adding an Amazon plugin. That unrelated root churn can regress published package contents and dependency resolution, so this needs a clean rebase/regeneration from current main and only intentional package changes.
  - confidence: 0.94
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends simplified `{ event: { type }, data }` objects and starts audio without the required `promptStart`/`contentStart` hierarchy and shared `promptName`/`contentName` identifiers. AWS documents a structured bidirectional flow, so this session is unlikely to work against Bedrock as written.
  - confidence: 0.93
- **[P1] Defer error close until reconnect retries fail:** `extensions/amazon/nova-sonic/bridge.ts:289-298`
  - body: `processOutputStream` starts `attemptReconnect()` on stream errors but the `finally` block immediately emits `onClose("error")`. Voice-call handlers treat error close as teardown, so transient Bedrock stream interruptions can end calls before reconnect can succeed.
  - confidence: 0.9
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never applied to provider registration: the provider still advertises audio capability, auto priority, and `transcribeAudio`. Since the current media-understanding contract has no provider `isConfigured` hook, disabled Amazon Transcribe can still be selected and invoked.
  - confidence: 0.95
- **[P2] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-241`
  - body: `synthesizeTelephony` requests `outputFormat: "pcm"` with sample rate `22050`, but Polly only accepts `8000` or `16000` for PCM. Normal telephony synthesis can fail with an AWS validation error instead of returning audio.
  - confidence: 0.94
- **[P2] Declare the real FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC path passes bytes through unchanged while declaring `MediaSampleRateHertz: 16000`. Amazon Transcribe requires the request sample rate to match the input audio, so common FLAC files at 44.1 or 48 kHz can fail or transcribe incorrectly.
  - confidence: 0.9
- **[P2] Resample telephony audio around Nova Sonic:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: The bridge converts between mu-law and PCM but does not resample between OpenClaw telephony's 8 kHz mu-law, Nova's declared 16 kHz input, and its 24 kHz output. That can send time-scaled speech to the model and emit wrong-speed audio back to calls.
  - confidence: 0.88
