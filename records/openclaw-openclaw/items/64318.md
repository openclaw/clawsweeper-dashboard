## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Drop stale root package rewrites:** `package.json:3`
  - body: The branch rewrites the root package version, exports, scripts, dependency pins, and package metadata from an older base while adding an Amazon plugin. That can regress published package contents and dependency resolution unrelated to the feature, so this needs to be rebased/regenerated from current main and limited to intentional package changes.
  - confidence: 0.93
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends simplified `{type, data}` events and starts audio without the required `promptStart`/`contentStart` hierarchy and shared `promptName`/`contentName` identifiers. AWS documents a structured bidirectional flow, so the realtime session is unlikely to work against Bedrock as written. ([docs.aws.amazon.com](https://docs.aws.amazon.com/nova/latest/nova2-userguide/sonic-input-events.html))
  - confidence: 0.92
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never applied to provider registration: the provider still advertises audio capability, auto priority, and `transcribeAudio`. Since the current media-understanding contract has no provider `isConfigured` hook, disabled Amazon Transcribe can still be selected and invoked.
  - confidence: 0.95
- **[P2] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-241`
  - body: `synthesizeTelephony` requests `OutputFormat: "pcm"` with sample rate `22050`, but Polly only accepts `8000` or `16000` for PCM. Normal telephony synthesis can fail with an AWS validation error. ([docs.aws.amazon.com](https://docs.aws.amazon.com/polly/latest/dg/API_SynthesizeSpeech.html))
  - confidence: 0.94
- **[P2] Declare the real FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC path passes bytes through unchanged while declaring `MediaSampleRateHertz: 16000`. Amazon Transcribe requires the request sample rate to match the input audio, so common FLAC files at 44.1 or 48 kHz can fail or transcribe incorrectly. ([docs.aws.amazon.com](https://docs.aws.amazon.com/transcribe/latest/APIReference/API_streaming_StartStreamTranscription.html))
  - confidence: 0.9
- **[P2] Resample realtime audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: The bridge converts telephony mu-law bytes to PCM and 24 kHz PCM output back to mu-law without resampling between OpenClaw's 8 kHz telephony format and Nova Sonic's declared 16 kHz input and 24 kHz output. Calls can get time-scaled recognition input and wrong-speed playback.
  - confidence: 0.88
- **[P2] Defer error close until reconnect retries fail:** `extensions/amazon/nova-sonic/bridge.ts:289-299`
  - body: `processOutputStream` starts `attemptReconnect()` on stream errors and returns, but the `finally` block immediately emits `onClose("error")`. In realtime call flows that treat error close as teardown, a recoverable Bedrock stream interruption can end the call before reconnect has a chance to succeed.
  - confidence: 0.87
