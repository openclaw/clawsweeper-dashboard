## Summary

Keep this PR open, but it is not mergeable as-is. Current main does not already ship a unified Amazon plugin, and the feature direction has useful prior Polly context, but the branch still has blocking package-surface rewrites, missing new-plugin docs/metadata follow-through, Transcribe disable semantics that do not match the current media runner, Nova Sonic protocol/audio/reconnect defects, and a Polly PCM sample-rate bug.

## What This Changes

The PR adds a bundled `extensions/amazon` plugin registering Amazon Polly speech, Amazon Transcribe media-understanding, and Amazon Nova Sonic realtime-voice providers, with AWS SDK dependencies, tests, a plugin manifest/config schema, and root package/lockfile changes.

## Best Possible Solution

Rebase onto current main, narrow the diff to the Amazon plugin plus intentional dependency changes, fix the provider-disable, Nova Sonic protocol/audio/reconnect, Polly telephony, and package-surface issues with contract tests, then add the required docs, changelog, labeler metadata, and changed-surface CI proof. If maintainers decide AWS voice/speech should not be bundled, move this work to an external ClawHub plugin using the existing speech, media-understanding, and realtime-voice SDK seams.

## Reproduction Assessment

Yes. The main blockers are reproducible by static review against current main contracts and the PR diff: current media selection ignores `enabled`, voice-call expects 8 kHz mu-law and hangs up on error close, the Nova stream uses non-documented event shapes, and Polly telephony requests an invalid PCM sample rate.

## Solution Assessment

No. The current PR is not the best way to solve the feature request as-is; the safer path is a clean rebase, focused protocol/audio/config/package fixes, dependency review, and maintainer approval for bundled-core versus ClawHub distribution.

- **[P1] Drop stale root package rewrites:** `package.json:3`
  - body: The PR changes the root package version and package surface back to an older state while adding a plugin. Current main keeps exports, build checks, package-file entries, and dependency versions that this diff removes, so merging would regress published package contents outside Amazon support.
  - confidence: 0.93
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends a custom `sessionStart`/`audioInput` shape and starts audio without opening prompt/content events with stable names. AWS documents a nested bidirectional stream lifecycle, so this bridge will not follow the service protocol.
  - confidence: 0.9
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but the provider still advertises audio capability, auto priority, and `transcribeAudio`. Current media-understanding selection has no `isConfigured` hook, so disabled Amazon Transcribe can still be selected and invoked.
  - confidence: 0.95
- **[P2] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: `synthesizeTelephony` requests `OutputFormat: "pcm"` with sample rate `22050`. Polly PCM synthesis accepts telephony rates such as 8000 or 16000, so normal voice-call TTS through this provider will fail with a provider validation error.
  - confidence: 0.94
- **[P2] Resample realtime telephony audio:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: The bridge converts G.711 mu-law to PCM and PCM back to mu-law but never resamples. Voice-call supplies 8 kHz mu-law while Nova is configured for 16 kHz input and 24 kHz output, so caller audio and assistant playback will run at the wrong rate.
  - confidence: 0.88
- **[P2] Delay error close until reconnect finishes:** `extensions/amazon/nova-sonic/bridge.ts:289-298`
  - body: On stream errors, `processOutputStream` starts reconnect work but the `finally` block immediately calls `onClose("error")`. Voice-call treats an error close as call termination, so transient Bedrock stream failures cannot be recovered by the reconnect path.
  - confidence: 0.91
