## Summary

Keep this PR open, but it is not mergeable as-is. Current main does not already provide the unified Amazon Polly/Transcribe/Nova Sonic plugin, and the feature direction is plausible, but the branch still has concrete runtime, protocol, package-surface, docs/metadata, and supply-chain blockers that need contributor and maintainer review.

## What This Changes

The PR branch adds a bundled `extensions/amazon` plugin registering Amazon Polly speech, Amazon Transcribe media-understanding, and Amazon Nova Sonic realtime-voice providers, with AWS SDK dependencies, tests, a plugin manifest, and root package/lockfile changes.

## Best Possible Solution

Keep the PR active only if it rebases cleanly, narrows the diff to the Amazon plugin plus intentional dependency changes, fixes the provider-disable, Nova Sonic protocol/audio/reconnect, Polly telephony, and package-surface issues, and adds the required docs, changelog, labeler, tests, and changed-surface CI. If maintainers decide AWS voice/speech is optional integration scope rather than bundled-core scope, the best end state is an external ClawHub plugin using the existing speech, media-understanding, and realtime-voice SDK contracts.

## Reproduction Assessment

Yes. The main blockers are reproducible by static review against current main contracts and the PR diff: the media runner selects audio providers by capability/handler presence, AWS docs require different Nova Sonic event shapes, and Polly documents `pcm` sample rates that exclude `22050`; live AWS validation would be useful later but is not needed to reproduce these mismatches.

## Solution Assessment

No. The current PR is not the best way to solve the feature request as-is; the safer path is a clean rebase, focused runtime/protocol fixes with conformance tests, package/security review, and maintainer product approval for bundling versus ClawHub distribution.

Full review comments:

- **[P1] Drop stale root package rewrites:** `package.json:3`
  - body: The PR changes the root package version back to `2026.4.26` and removes current package export/file entries that exist on main. This feature should not regress the published package surface while adding a plugin.
  - confidence: 0.93
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends simplified custom event objects and starts audio without the required `promptStart`/`contentStart` identifiers. AWS documents nested `event.sessionStart`, `event.promptStart`, `event.contentStart`, and matching prompt/content names for input and completion/content events for output, so this stream will not follow the service protocol.
  - confidence: 0.9
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never applied to registration: the provider still advertises audio capability, auto priority, and `transcribeAudio`. Since current media-understanding selection has no `isConfigured` hook, a disabled Amazon Transcribe provider can still be selected and invoked.
  - confidence: 0.95
- **[P2] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: `synthesizeTelephony` requests `OutputFormat: "pcm"` with sample rate `22050`, but Polly only accepts `8000` or `16000` for PCM. Normal voice-call TTS through this provider will fail with an AWS validation error.
  - confidence: 0.94
- **[P2] Resample realtime telephony audio:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: The bridge only converts between mu-law and PCM samples while declaring 16 kHz input and 24 kHz output. Voice-call media streams are 8 kHz mu-law, so calls will get time-scaled input and wrong-speed output unless the bridge also resamples across the declared rates.
  - confidence: 0.88
- **[P2] Delay error close until reconnect finishes:** `extensions/amazon/nova-sonic/bridge.ts:289-298`
  - body: On stream errors, `processOutputStream` starts reconnect work but the `finally` path immediately emits `onClose("error")`. Voice Call treats an error close as call termination, so this prevents the reconnect path from recovering transient Nova stream failures.
  - confidence: 0.91
