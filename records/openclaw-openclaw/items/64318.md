Full review comments:

- **[P1] Rebase before carrying root package metadata:** `package.json:3`
  - body: This patch changes root `package.json` back to `2026.4.26` and drops current package file/export/script changes from main. Merging that stale root metadata would regress package contents and dependency behavior before the Amazon plugin is even exercised.
  - confidence: 0.94
- **[P1] Gate Transcribe when config disables it:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but the provider still always exposes audio capabilities, priority, and `transcribeAudio`. The media-understanding contract has no `isConfigured` hook, so a disabled Amazon Transcribe provider can still be selected and invoked.
  - confidence: 0.93
- **[P1] Follow the Nova Sonic bidirectional event schema:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends a custom `sessionStart` payload and bare `audioInput` events, but AWS' documented flow requires prompt/content identifiers around audio and returns completion/content-scoped output events. This shape is unlikely to establish or parse real Nova Sonic sessions correctly.
  - confidence: 0.9
- **[P1] Resample telephony audio before and after Nova Sonic:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: This path converts mu-law bytes to PCM and back, but it never resamples 8 kHz telephony input to the configured 16 kHz Nova input or 24 kHz Nova output back to 8 kHz mu-law. Voice-call media will play at the wrong speed or be distorted.
  - confidence: 0.89
- **[P1] Defer error close until reconnect fails:** `extensions/amazon/nova-sonic/bridge.ts:289-299`
  - body: `processOutputStream` starts `attemptReconnect()` on stream errors, then the `finally` block immediately sets `connected = false` and calls `onClose("error")`. In voice-call usage that can tear down the call before the reconnect attempt has a chance to recover.
  - confidence: 0.88
- **[P1] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: The telephony path requests `outputFormat: "pcm"` with sample rate `22050`, but Polly PCM output only accepts 8000 or 16000. Normal telephony synthesis will fail with an AWS validation error.
  - confidence: 0.92
- **[P2] Do not hard-code FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: FLAC is passed through unchanged while `MediaSampleRateHertz` is set to 16000. Amazon Transcribe requires the declared sample rate to match the input, so common 44.1/48 kHz FLAC files can fail or transcribe poorly.
  - confidence: 0.86
