## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Drop stale package metadata rewrites:** `package.json:3`
  - body: The PR head changes the root package version back to `2026.4.26` and omits current publish/export entries from main, even though this feature should only need the new plugin and intentional dependency updates. Merging this stale root metadata would regress the published package surface.
  - confidence: 0.93
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never used: the provider always advertises audio capability, auto priority, and `transcribeAudio`. Because the media-understanding contract has no `isConfigured` hook and auto-selection keys off those fields, a disabled Amazon Transcribe provider can still be selected and invoked.
  - confidence: 0.95
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends custom `event.type` objects and starts streaming audio without the required `promptStart`/`contentStart` identifiers. AWS Nova Sonic expects nested events such as `event.sessionStart`, `event.promptStart`, `event.contentStart`, and matching prompt/content names, so this stream will not follow the service protocol.
  - confidence: 0.9
- **[P2] Resample telephony audio for Nova Sonic:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: `sendAudio` only converts 8 kHz G.711 mu-law bytes to PCM samples while the Nova session declares 16 kHz input; output similarly converts 24 kHz PCM straight to mu-law without downsampling to the 8 kHz telephony sink. Calls will get wrong-speed audio even when the codec conversion succeeds.
  - confidence: 0.88
- **[P2] Delay error close until reconnect finishes:** `extensions/amazon/nova-sonic/bridge.ts:289-298`
  - body: On a stream error, `processOutputStream` starts `attemptReconnect()` and returns, but the `finally` block immediately fires `onClose("error")`. The voice-call integration hangs up on that error close, so the reconnect path cannot recover a transient Nova stream interruption.
  - confidence: 0.91
- **[P2] Use a valid Polly PCM telephony sample rate:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: `synthesizeTelephony` requests `OutputFormat: "pcm"` with sample rate `22050`, but Amazon Polly only accepts `8000` or `16000` for PCM output. Normal voice-call TTS through this provider will fail with an AWS validation error.
  - confidence: 0.94
