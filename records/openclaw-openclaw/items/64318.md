## Summary

This PR should remain open. Current main does not include the unified Amazon voice/speech plugin, and the branch still has concrete merge blockers: stale root package rewrites, failing CI, disabled-provider behavior, Nova Sonic protocol/audio issues, missing real behavior proof, and an unresolved product decision about bundling versus external plugin distribution.

## What This Changes

The PR branch adds a bundled `extensions/amazon` plugin for Amazon Polly speech, Amazon Transcribe media understanding, and Amazon Nova Sonic realtime voice, with AWS SDK dependencies, tests, a plugin manifest, and root package/lockfile changes.

## Best Possible Solution

The best end state is either a clean, rebased, maintainer-approved bundled Amazon plugin with fixed protocol/config/audio/package issues, docs, labeler, changelog, green changed-surface CI, and real AWS proof, or an external ClawHub plugin using the existing speech, media-understanding, and realtime-voice SDK contracts.

## Reproduction Assessment

Yes. The blocking defects are source-reproducible from the PR head and current-main contracts: disabled Transcribe still advertises audio handling, the media runner selects by capability/handler presence, Nova Sonic event shapes differ from AWS docs, and Polly telephony asks for an invalid PCM sample rate.

## Solution Assessment

No. This is not the best mergeable shape yet; the safer path is a clean rebase, focused fixes with conformance tests, package/security review, live AWS proof, and an explicit decision on bundled-core placement versus ClawHub distribution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Drop stale root package rewrites:** `package.json:3`
  - body: The branch changes the root package version backward and carries older package `files`/export surface while adding a plugin. That can regress published package contents unrelated to the Amazon feature and needs to be removed by rebasing/regenerating from current main.
  - confidence: 0.93
- **[P1] Send Nova Sonic's documented event shapes:** `extensions/amazon/nova-sonic/bridge.ts:130-168`
  - body: The bridge sends simplified `{type, data}` events and starts audio without the required `promptStart`/`contentStart` hierarchy and shared `promptName`/`contentName` identifiers. This does not match the documented Nova Sonic bidirectional stream protocol, so the realtime session is unlikely to work against Bedrock.
  - confidence: 0.9
- **[P2] Honor the Transcribe enabled flag:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never applied to registration: the provider still advertises audio capability, auto priority, and `transcribeAudio`. Since current media-understanding selection has no provider `isConfigured` hook, a disabled Amazon Transcribe provider can still be selected and invoked.
  - confidence: 0.95
- **[P2] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-242`
  - body: `synthesizeTelephony` requests `OutputFormat: "pcm"` with sample rate `22050`, but Polly only accepts `8000` or `16000` for PCM. Normal telephony synthesis through this provider can fail with an AWS validation error.
  - confidence: 0.94
- **[P2] Declare the real FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC path passes bytes through unchanged while declaring `MediaSampleRateHertz: 16000`. AWS requires the request sample rate to match the input audio, so common FLAC files at 44.1 or 48 kHz can fail or transcribe incorrectly.
  - confidence: 0.88
- **[P2] Resample inbound telephony audio:** `extensions/amazon/nova-sonic/bridge.ts:67-72`
  - body: `sendAudio` converts mu-law bytes to PCM samples but does not resample from telephony 8 kHz to the 16 kHz input rate declared to Nova Sonic. The model will receive time-scaled speech even if the codec conversion succeeds.
  - confidence: 0.88
- **[P2] Downsample Nova output before telephony playback:** `extensions/amazon/nova-sonic/bridge.ts:318-321`
  - body: The session declares 24 kHz PCM output, but `audioOutput` is converted directly to mu-law without downsampling to the 8 kHz telephony stream expected by voice-call output. Callers can hear wrong-speed audio and timing drift.
  - confidence: 0.88
- **[P2] Delay error close until reconnect finishes:** `extensions/amazon/nova-sonic/bridge.ts:289-298`
  - body: On stream errors, `processOutputStream` starts reconnect work but the `finally` path immediately emits an error close. Voice-call treats error close as terminal, so transient Bedrock stream failures can end the call before reconnect can recover.
  - confidence: 0.91
