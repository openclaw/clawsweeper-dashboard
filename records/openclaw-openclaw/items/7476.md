## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending, the related implementation PRs are closed unmerged, and the remaining decision is a new WhatsApp plugin feature with public contract and encoding choices.

## What This Changes

Request outbound WhatsApp sticker sending so WebP sticker media can be delivered as a sticker instead of a normal image attachment.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, scoped WebP preservation or re-encoding semantics, Baileys sticker payload routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes `image/webp` through generic image handling, has no sticker option in the active send options, and does not advertise or dispatch a WhatsApp `sticker` action.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The safer direction is an explicit WhatsApp sticker intent plus encoding handling, rather than only global implicit WebP auto-detection.
