## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is routed as ordinary image media, no sticker intent can reach the active WhatsApp sender, and the related implementation PRs are closed unmerged.

## What This Changes

Add outbound WhatsApp sticker send support to the WhatsApp plugin, such as a `sticker` action or explicit sticker send option.

## Best Possible Solution

Implement plugin-owned WhatsApp outbound sticker support with a documented explicit sticker intent, route intended media through Baileys `{ sticker: buffer }`, preserve the needed WebP media before generic image optimization, and add focused tests plus live redacted WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes `image/webp` through the generic image branch, has no sticker option in `ActiveWebSendOptions`, and does not advertise or handle a WhatsApp `sticker` action; live client rendering proof was not run.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The narrow maintainable direction is a WhatsApp plugin change using an explicit sticker intent and Baileys `{ sticker: buffer }`, while avoiding implicit global WebP auto-detection as the only contract.
