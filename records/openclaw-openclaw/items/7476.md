## Summary

Keep open: current main and latest release still lack outbound WhatsApp sticker sending, the related implementation PRs are closed unmerged, and the newer encoding-policy discussion makes this a maintainer product decision before automation is safe.

## What This Changes

Request to let WhatsApp send outbound stickers instead of delivering qualifying WebP media as normal image attachments.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding policy, Baileys sticker payload delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound `image/webp` through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract and encoding policy. An explicit `action=sticker` or `asSticker`/`sendAsSticker` intent with scoped validation or re-encoding is safer than automatic WebP detection alone.
