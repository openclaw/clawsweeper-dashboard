## Summary

Keep open: current main and the latest release still lack a user-facing WhatsApp sticker send contract; a lower-level `sendSticker` helper exists, but normal outbound sends still route WebP through the generic image path and `message action=sticker` is not supported for WhatsApp.

## What This Changes

The issue requests outbound WhatsApp sticker support through a sticker action, sticker send option, or safe WebP-to-sticker routing.

## Best Possible Solution

Add first-class WhatsApp plugin sticker sending with explicit sticker intent, Baileys sticker payload routing, ordinary WebP image preservation, focused docs/tests, and a documented encoding policy.

## Reproduction Assessment

Yes, at source level: current main still rejects WhatsApp `sticker` actions and routes WebP image media through the generic image send branch. I did not perform a live WhatsApp send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. The implementation belongs in the WhatsApp plugin, and an explicit sticker action or send option is safer than silent WebP heuristics alone.
