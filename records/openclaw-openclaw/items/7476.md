## Summary

Keep open. Current main still lacks an outbound WhatsApp sticker send path, related implementation PRs are closed unmerged, and recent discussion still points to unresolved public-contract and WebP encoding choices.

## What This Changes

Request outbound WhatsApp sticker sending through a dedicated action/option or WebP sticker detection instead of delivering WebP as a regular image.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with a documented explicit intent, scoped WebP preservation or re-encoding behavior, Baileys sticker payload routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes outbound `image/webp` through generic image handling, cannot carry a sticker option in active send options, and does not advertise or dispatch a WhatsApp `sticker` action.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The safer direction is an explicit WhatsApp sticker intent plus encoding handling, rather than only global implicit WebP auto-detection.
