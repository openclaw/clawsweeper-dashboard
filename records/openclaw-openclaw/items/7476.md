## Summary

Keep this issue open. Current main still lacks outbound WhatsApp sticker support, the two related implementation PRs are closed unmerged, and the remaining decision is the product contract plus sticker encoding policy rather than stale cleanup.

## What This Changes

Requested outbound WhatsApp sticker send support so valid WebP media can be delivered as a sticker instead of a regular image attachment.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped media preservation or re-encoding, Baileys `{ sticker: ... }` delivery, aligned action/tool/docs behavior, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option/action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. The reporter's local WebP detection is useful evidence, but an explicit sticker intent plus scoped encoding/validation policy is the safer maintainable direction than autodetection alone.
