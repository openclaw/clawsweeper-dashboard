## Summary

Keep open: current main and the latest release still lack a user-facing WhatsApp sticker send contract, even though a lower-level Baileys sticker helper exists for QA/runtime support.

## What This Changes

Request outbound WhatsApp sticker support so WebP sticker media can be sent as native WhatsApp stickers instead of ordinary image attachments.

## Best Possible Solution

Add a first-class WhatsApp plugin sticker send contract that routes explicit sticker intent through Baileys `{ sticker: buffer }`, preserves ordinary WebP image sends, and documents/tests the chosen encoding behavior.

## Reproduction Assessment

Yes, at source level: current main still rejects non-react/upload-file WhatsApp actions and routes image media through the generic image send branch. I did not perform a live WhatsApp send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. The implementation belongs in the WhatsApp plugin, and an explicit sticker action or send option is safer than silent WebP heuristics alone.
