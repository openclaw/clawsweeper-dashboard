## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP image media follows the generic image path, there is no sticker send option or WhatsApp sticker action, and the related implementation PRs are closed unmerged.

## What This Changes

Request adds WhatsApp outbound sticker sending through a dedicated action, explicit send option, or WebP sticker detection.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, correct WebP preservation or re-encoding semantics, Baileys sticker payload routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes `image/webp` through generic image handling, has no sticker option in `ActiveWebSendOptions`, and does not advertise or dispatch a WhatsApp `sticker` action.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The safer direction is an explicit WhatsApp sticker intent plus encoding handling, rather than only global implicit WebP auto-detection.
