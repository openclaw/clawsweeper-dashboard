## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending, and the useful related PRs are closed unmerged rather than a live implementation path.

## What This Changes

Request outbound WhatsApp sticker send support through a dedicated action, explicit send option, or WebP sticker detection.

## Best Possible Solution

Add plugin-owned WhatsApp outbound sticker support with an explicit documented sticker intent, Baileys `{ sticker: buffer }` routing, media preservation for intended sticker payloads, focused tests, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes `image/webp` through the generic image branch, has no sticker option in `ActiveWebSendOptions`, and does not advertise or dispatch a WhatsApp `sticker` action.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The maintainable direction is a WhatsApp plugin change using an explicit sticker intent rather than relying only on implicit global WebP auto-detection.
