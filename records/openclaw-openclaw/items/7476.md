## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is handled as a generic image, no sticker intent flows through the active send options, and WhatsApp action discovery/dispatch still does not expose a sticker action. The reporter's local patch and closed PR #7502 are useful implementation context, but there is no merged fix to close against.

## What This Changes

Request outbound WhatsApp sticker send support so WebP sticker media can be sent as native WhatsApp stickers instead of normal image attachments.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin with an explicit outbound sticker intent, preserve intended WebP sticker media before generic image optimization, route the active send through Baileys `{ sticker: buffer }`, expose the action or option consistently in discovery/dispatch/docs, and add focused tests for payload selection and media preservation.

## Reproduction Assessment

Yes. A focused unit reproduction can call `createWebSendApi().sendMessage()` with `mediaType: "image/webp"` and observe that current main builds an image payload, not a sticker payload; the source path also shows `message action=sticker` is unsupported for WhatsApp.

## Solution Assessment

Yes, fixing the WhatsApp plugin is the right location because the behavior is channel-specific and Baileys already has a native sticker payload. The closed PR's WebP auto-detection is useful prior art, but the maintainable solution should carry explicit sticker intent rather than silently converting all qualifying WebP sends.
