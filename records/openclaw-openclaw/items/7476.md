## Label Justifications

## Summary

Keep open. Current main has a low-level WhatsApp `sendSticker` helper and QA-driver coverage, but the user-facing message action/upload-file path still does not advertise or dispatch WhatsApp stickers, and the related implementation PRs are closed unmerged.

## What This Changes

Request outbound WhatsApp sticker sending from the message tool, either through a sticker action, an explicit option, or WebP sticker detection.

## Best Possible Solution

Keep this issue open and implement one maintainer-approved outbound WhatsApp sticker contract in the bundled WhatsApp plugin, reusing the existing structured `sendSticker` helper and adding docs plus focused action/send/media tests.

## Reproduction Assessment

Yes, at source level. `message action=sticker` is not supported by the WhatsApp action surface, and normal outbound WebP media still goes through the image branch rather than the sticker payload branch.

## Solution Assessment

No, the current main implementation is only partial. The maintainable fix is to choose one public contract, then wire it through WhatsApp action discovery/handling and outbound send state to the existing `sendSticker` Baileys payload path.
