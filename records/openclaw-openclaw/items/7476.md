## Label Justifications

- `P2`: This is a bounded WhatsApp channel feature gap with clear source evidence and limited blast radius, while ordinary text and media delivery still work.
- `impact:message-loss`: The requested native sticker message semantics are lost because the WhatsApp message tool cannot select WhatsApp's sticker message type.

## Summary

Current main still lacks a user-facing WhatsApp sticker send path. The live message action surface does not advertise or dispatch `sticker`, and ordinary WebP media still reaches the WhatsApp send API as an image payload rather than a sticker payload.

## What This Changes

Request outbound WhatsApp sticker support so the WhatsApp message tool can send sticker media instead of regular image attachments.

## Best Possible Solution

Implement a first-class WhatsApp outbound sticker contract in the bundled WhatsApp plugin, route it through Baileys sticker payloads, keep generic image sends unchanged, and add focused tests plus docs for the chosen validation behavior.

## Reproduction Assessment

Yes, at source level. Current main still rejects or does not advertise WhatsApp `action=sticker`, and ordinary `image/webp` media is routed through the image payload branch rather than a sticker payload.

## Solution Assessment

Unclear until maintainers choose the public contract. The capability belongs in the WhatsApp plugin, but an explicit sticker action or send option is safer than silently treating qualifying WebP images as stickers.
