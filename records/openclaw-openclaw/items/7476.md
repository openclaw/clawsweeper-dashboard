## Summary

Keep open: current main and the latest release still lack an outbound WhatsApp sticker send path, while the related implementation pull requests are closed unmerged. The source gap is clear, but the public sticker intent and static/animated WebP encoding policy need maintainer choice before this is a safe repair lane.

## What This Changes

The issue asks for outbound WhatsApp sticker sending so WebP sticker media can be delivered as WhatsApp stickers rather than ordinary image attachments.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys sticker payload delivery, aligned action/tool/docs behavior, regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound image/webp through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract and encoding policy. An explicit action=sticker or asSticker/sendAsSticker intent with scoped validation or re-encoding is safer than automatic WebP detection alone.
