## Summary

Keep open. Current main still has no outbound WhatsApp sticker send contract or payload path, both related implementation PRs are closed unmerged, and recent discussion identifies unresolved contract and WebP encoding choices.

## What This Changes

Request outbound WhatsApp sticker sending support so WebP sticker media can be sent as stickers instead of ordinary image attachments.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with a documented explicit intent, scoped WebP preservation or re-encoding behavior, Baileys `{ sticker: buffer }` routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main routes outbound `image/webp` through generic image handling, cannot carry a sticker option, and does not advertise or dispatch a WhatsApp `sticker` action. No live WhatsApp send was performed in this read-only review.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The safer direction is an explicit WhatsApp sticker intent plus encoding/metadata handling, rather than only global implicit WebP auto-detection.
