## Label Justifications

## Summary

Keep open: current main and v2026.6.8 have a lower-level WhatsApp sticker payload helper for QA/runtime support, but the user-facing message tool still has no WhatsApp sticker action or send option, and ordinary WebP media still follows the generic image path.

## What This Changes

Request outbound WhatsApp sticker sending through the message tool, either as a sticker action, automatic qualifying WebP detection, or an explicit sticker send option.

## Best Possible Solution

Implement a first-class WhatsApp plugin sticker send contract that routes explicit sticker intent through Baileys `{ sticker: buffer }`, keeps ordinary WebP image sends unchanged, and documents/tests the chosen encoding and validation behavior.

## Reproduction Assessment

Yes, at source level. Current main does not accept or advertise WhatsApp `action=sticker`, and ordinary `image/webp` media still reaches the WhatsApp sender through the image payload branch rather than a sticker payload.

## Solution Assessment

Unclear until maintainers choose the public contract. The implementation belongs in the WhatsApp plugin, and an explicit sticker action or send option is safer than silently treating qualifying WebP images as stickers.
