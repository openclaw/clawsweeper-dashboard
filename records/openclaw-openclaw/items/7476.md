## Summary

Keep open: current main and the latest release still lack a user-facing WhatsApp sticker send path. The code has a lower-level Baileys sticker helper, but normal outbound sends still route WebP as image media, WhatsApp does not advertise or accept the shared sticker action, and the two implementation PRs were closed unmerged.

## What This Changes

Request to support outbound WhatsApp stickers from the message tool through a sticker action, explicit send option, or safe WebP sticker detection.

## Best Possible Solution

Add first-class WhatsApp plugin sticker sending with explicit sticker intent, Baileys `{ sticker: buffer }` routing, scoped media preservation/encoding policy, and focused docs plus tests.

## Reproduction Assessment

Yes, at source level: current main still rejects WhatsApp `sticker` actions and routes WebP image media through the generic image branch. I did not perform a live WhatsApp send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. The implementation belongs in the WhatsApp plugin, and explicit sticker intent is safer than silent WebP heuristics alone.
