## Summary

Keep open. Current main still lacks an outbound WhatsApp sticker contract or Baileys sticker payload path, both related implementation PRs are closed unmerged, and recent maintainer discussion points to unresolved contract and WebP encoding choices.

## What This Changes

Request to add outbound WhatsApp sticker sending support, either through a sticker action, an explicit send option, or scoped WebP sticker detection.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with a documented explicit intent, scoped WebP preservation or re-encoding behavior, Baileys `{ sticker: buffer }` routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for the current limitation, by source inspection: outbound `image/webp` follows generic image handling, no sticker option can be carried, and the WhatsApp action surface does not advertise or handle `sticker`. No live WhatsApp send was performed in this read-only review.

## Solution Assessment

Unclear until a maintainer chooses the public contract and encoding policy. The safer direction is an explicit WhatsApp sticker intent plus encoding/metadata handling, not only global implicit WebP auto-detection.
