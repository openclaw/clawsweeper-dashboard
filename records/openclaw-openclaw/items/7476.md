## Summary

Keep this issue open: current main still lacks an outbound WhatsApp sticker contract, routes WebP through generic image send behavior, and both related sticker PRs are closed without being merged.

## What This Changes

Add support for sending WhatsApp sticker messages instead of delivering qualifying WebP media as ordinary image attachments.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the bundled plugin with explicit sticker intent, scoped WebP preservation or re-encoding policy, Baileys sticker payload delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound `image/webp` through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. An explicit `action=sticker` or `asSticker`/`sendAsSticker` intent with scoped validation or re-encoding is safer than relying on automatic WebP detection alone.
