## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media follows the generic image path, no sticker intent is carried through WhatsApp send options or action discovery, and the related implementation PR is closed unmerged.

## What This Changes

Request outbound WhatsApp sticker support so `.webp` sticker media can be sent as WhatsApp stickers instead of regular image attachments.

## Best Possible Solution

Implement a WhatsApp-plugin-owned outbound sticker path with explicit sticker intent, WebP preservation for intended stickers, Baileys sticker payload delivery, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes for source-level reproduction. On current main, `createWebSendApi().sendMessage()` with `mediaType: "image/webp"` reaches the generic image branch, and WhatsApp action support/discovery does not include `sticker`.

## Solution Assessment

Yes, if the repair uses a plugin-owned explicit sticker intent such as the existing `message action=sticker` surface. The closed PR's WebP auto-detection is useful context, but silent detection should not be the sole contract unless maintainers choose that deliberately.

Implement outbound WhatsApp sticker send support for https://github.com/openclaw/openclaw/issues/7476. Use the WhatsApp plugin boundary, not core channel special-casing. Prefer the existing `message action=sticker` surface accepting `media`/`filePath`/`mediaUrl` WebP input, carry explicit sticker intent through the WhatsApp action/send pipeline, preserve intended WebP sticker media before generic image optimization, and send via Baileys sticker payload rather than the generic image payload.
