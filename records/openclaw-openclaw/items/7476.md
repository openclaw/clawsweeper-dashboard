## Summary

Keep this issue open. Current main and the latest shipped release still route outbound WhatsApp WebP/image media through regular image payloads, the message/action pipeline has no sticker intent, and both known implementation PRs are closed unmerged; the remaining decision is the public contract plus sticker encoding policy.

## What This Changes

Request outbound WhatsApp sticker sending so suitable media can be delivered as WhatsApp stickers instead of regular image attachments.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys sticker delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp`/`image/*` through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. An explicit sticker intent plus scoped validation or re-encoding policy looks safer than relying on automatic WebP detection alone.
