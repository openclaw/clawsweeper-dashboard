- `impact:message-loss`: The requested native sticker message semantics are lost because the WhatsApp message tool cannot select WhatsApp's sticker payload type.

## Summary

Keep open: current main and v2026.6.8 still lack a user-facing WhatsApp sticker send contract, even though a lower-level WhatsApp send helper can now create a Baileys sticker payload for QA/runtime support.

## What This Changes

Request outbound WhatsApp sticker support so the message tool can send explicit sticker intent or qualifying WebP media as native WhatsApp stickers instead of ordinary image media.

## Best Possible Solution

Implement a first-class WhatsApp plugin sticker send contract that routes explicit sticker intent through Baileys { sticker: buffer }, keeps ordinary WebP image sends unchanged, and documents/tests the chosen encoding and validation behavior.

## Reproduction Assessment

Yes, at source level. Current main does not accept or advertise a WhatsApp sticker action, and ordinary image/webp media still reaches the WhatsApp sender through the image payload branch rather than a sticker payload.

## Solution Assessment

Unclear until maintainers choose the public contract. The implementation belongs in the WhatsApp plugin, and an explicit sticker action or send option is safer than silently treating qualifying WebP images as stickers.
