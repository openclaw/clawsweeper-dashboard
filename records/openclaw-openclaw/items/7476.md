## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is routed as a generic image, no sticker intent is carried through WhatsApp send options, and the WhatsApp action surface still does not support `sticker`; the reporter's local patch and closed unmerged PR #7502 remain useful implementation context.

## What This Changes

Request outbound WhatsApp sticker support so WebP sticker media can be sent as a native WhatsApp sticker instead of a regular image attachment.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin with an explicit outbound sticker intent, preserve intended WebP sticker media for that path, route active sends through Baileys sticker payloads, update discovery/dispatch/docs, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `createWebSendApi().sendMessage()` with `mediaType: "image/webp"` enters the generic image branch, and WhatsApp `supportsAction` rejects `sticker`.

## Solution Assessment

Yes. The maintainable direction is plugin-owned WhatsApp support with explicit sticker intent; the closed PR's WebP auto-detection is useful prior art, but silent detection should not be the only contract unless maintainers choose it deliberately.
