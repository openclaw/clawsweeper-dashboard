## Summary

Keep open: current main and v2026.6.5 still lack a user-facing outbound WhatsApp sticker send path. Main now has a low-level WhatsApp `sendSticker` helper, but the message/upload path still sends `image/webp` as generic image media and the WhatsApp action surface does not expose `sticker` or `asSticker`.

## What This Changes

Requested outbound WhatsApp sticker send support so WebP sticker media can be sent as WhatsApp stickers rather than regular image attachments.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys sticker payload delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound `image/webp` through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp send in this read-only review.

## Solution Assessment

No, the current partial implementation is not the best complete solution because the low-level `sendSticker` helper is not exposed through the public message/action path. A maintainable fix should choose an explicit public sticker intent and pair routing with sticker validation or encoding policy.
