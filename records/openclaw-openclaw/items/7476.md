## Summary

Keep open. Current main still lacks an outbound WhatsApp sticker contract and routes WebP/image media through generic image handling; the related implementation PRs are closed unmerged, and recent discussion raises unresolved encoding and metadata policy questions.

## What This Changes

The issue requests outbound WhatsApp sticker sending support from the message tool instead of treating WebP sticker files as ordinary image attachments.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes. Source inspection on current main shows outbound `image/webp` follows the generic image payload, no sticker option can be carried through ActiveWebSendOptions, and the WhatsApp action surface does not support `sticker`.

## Solution Assessment

Unclear. The issue correctly identifies the missing capability, but the safer implementation needs maintainer choice on the public sticker contract and encoding policy instead of adopting broad implicit WebP auto-detection by default.
