## Summary

Keep open: current main and the latest release still lack outbound WhatsApp sticker-send support. WebP outbound media is still sent through the generic image path, no sticker intent/action is exposed, and the related implementation PR is closed unmerged; maintainers still need to choose the public contract and encoding policy.

## What This Changes

Requested outbound WhatsApp sticker-send support through a sticker action, explicit sticker flag, or safe WebP sticker detection.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped media preservation or re-encoding, Baileys sticker payload delivery, aligned action/tool/docs behavior, regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound image/webp through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract and encoding policy. An explicit action=sticker or asSticker/sendAsSticker intent with scoped validation or re-encoding is safer than automatic WebP detection alone.
