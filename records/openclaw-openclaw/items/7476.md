## Summary

Keep open: current main and the latest release still lack outbound WhatsApp sticker sending. WebP media still goes through the generic image path, no explicit sticker intent is exposed, and the two related sticker implementation PRs were closed unmerged, so the remaining work needs a maintainer decision on contract and encoding policy.

## What This Changes

Request outbound WhatsApp sticker sending so WebP sticker media can be delivered as a WhatsApp sticker payload rather than a regular image.

## Best Possible Solution

Implement first-class outbound WhatsApp sticker support in the WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` delivery, aligned action/tool/docs behavior, regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound `image/webp` through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract and encoding policy. An explicit sticker action or send option with scoped validation/re-encoding is safer than automatic WebP detection alone.
