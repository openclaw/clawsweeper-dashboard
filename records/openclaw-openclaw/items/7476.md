## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is routed as a generic image, sticker intent is not carried through WhatsApp send/action paths, and related PR #7502 is closed unmerged.

## What This Changes

Add support for sending WhatsApp stickers from the WhatsApp plugin instead of delivering qualifying WebP media as ordinary image attachments.

## Best Possible Solution

Implement WhatsApp-plugin-owned outbound sticker support through the existing sticker action surface or an explicit send intent, preserve intended WebP sticker media, route Baileys sends through { sticker: buffer }, and update focused tests plus docs.

## Reproduction Assessment

Yes for source-level reproduction. On current main, image/webp follows the generic image branch, while WhatsApp action discovery/support do not include sticker and the active send options have no sticker intent.

## Solution Assessment

Yes. A narrow WhatsApp plugin change is the maintainable direction; the safer contract is an explicit sticker action or send intent rather than relying only on caption-based WebP auto-detection.
