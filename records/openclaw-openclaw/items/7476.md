## Summary

Keep this issue open. Current main still has no outbound WhatsApp sticker send contract or Baileys sticker payload path, and the only known implementation PRs are closed unmerged; the remaining work needs a maintainer decision on the public intent flag/action and sticker encoding policy.

## What This Changes

The issue requests outbound WhatsApp sticker support so a WebP sticker can be sent as a WhatsApp sticker instead of a regular image attachment.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic `image/*` payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. An explicit `action=sticker` or `asSticker` intent with scoped validation/re-encoding is safer than relying on automatic WebP detection alone.
