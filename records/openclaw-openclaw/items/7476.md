## Summary

Keep this issue open: current main still has no outbound WhatsApp sticker send contract, WebP media still follows the generic image path, and the latest discussion leaves a real maintainer decision around explicit sticker intent and encoding policy.

## What This Changes

Request to add outbound WhatsApp sticker sending support so WhatsApp can deliver stickers instead of regular WebP image attachments.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding policy, Baileys sticker delivery, aligned action/tool/docs behavior, focused regression tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes outbound image/webp through the generic image payload and has no WhatsApp sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. An explicit action=sticker or asSticker/sendAsSticker intent with scoped validation or re-encoding is safer than relying on automatic WebP detection alone.
