## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP/image media is routed through the generic image send path, no active send option carries sticker intent, and the two related implementation PRs are closed unmerged while maintainer discussion still needs a contract and encoding decision.

## What This Changes

The issue requests outbound WhatsApp sticker support so WebP sticker media can be sent as a WhatsApp sticker instead of a generic image attachment.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` delivery, action/tool/docs alignment, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option or WhatsApp sticker action to select Baileys sticker media. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an explicit sticker intent plus scoped media encoding/preservation, not only automatic `.webp` detection.
