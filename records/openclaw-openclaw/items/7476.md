## Label Justifications

- `P2`: This is a normal-priority user-facing WhatsApp channel enhancement with a clear source-level gap and limited blast radius.
- `impact:message-loss`: The requested sticker message is currently either rejected as an unsupported action or delivered through the wrong media type instead of a native WhatsApp sticker.

## Summary

Keep open: current main has a low-level WhatsApp sticker payload helper, but it is not exposed through the active listener, message action surface, outbound message tool path, or WhatsApp docs, so the user-facing sticker send request remains unresolved.

## What This Changes

Request to add user-facing WhatsApp sticker sending through the message tool or an equivalent outbound WhatsApp contract.

## Best Possible Solution

Add a public WhatsApp sticker send contract in the bundled plugin, thread it through the message action/outbound listener path, preserve or encode sticker media intentionally, and cover it with focused send-api, action-dispatch, media, and docs tests.

## Reproduction Assessment

Yes. Current source shows WhatsApp `message action=sticker` is not supported and the normal outbound message path cannot pass sticker intent, though I did not run a live WhatsApp send.

## Solution Assessment

Unclear until maintainers choose the public contract. The internal `sendSticker` helper is a useful building block, but it is not sufficient because it is not exposed to the message tool or active listener.
