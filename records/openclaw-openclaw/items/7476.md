## Summary

Keep this issue open. Current main still lacks an outbound WhatsApp sticker send path, both related implementation PRs are closed unmerged, and the remaining choice is a product/contract decision plus sticker encoding policy rather than cleanup.

## What This Changes

The issue requests outbound WhatsApp sticker sending support so valid WebP sticker media can be sent as stickers instead of generic image attachments.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: ... }` delivery, aligned action/tool/docs behavior, and focused regression plus live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option or action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. The reporter's WebP detection patch is useful evidence, but an explicit sticker intent plus scoped encoding or validation policy is safer than autodetection alone.
