## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is routed as a generic image, no sticker intent is carried through WhatsApp send options or action discovery, and the related PR #7502 is closed unmerged.

## What This Changes

Add a way for the WhatsApp plugin to send outbound sticker messages instead of delivering qualifying WebP files as regular image attachments.

## Best Possible Solution

Implement a WhatsApp-plugin-owned outbound sticker path with explicit sticker intent, WebP preservation for intended stickers, Baileys `{ sticker: buffer }` delivery, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes for source-level reproduction. On current main, `createWebSendApi().sendMessage()` with `mediaType: "image/webp"` reaches the generic image branch, and WhatsApp `supportsAction` rejects any action except `react`.

## Solution Assessment

Yes, if the fix uses a plugin-owned explicit sticker intent. The closed PR's WebP auto-detection is useful context, but silent detection should not be the only contract unless maintainers choose it deliberately.
