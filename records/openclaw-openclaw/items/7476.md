## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending, the related implementation PRs are closed unmerged, and recent discussion adds unresolved API/encoding policy questions that need maintainer direction.

## What This Changes

Request outbound WhatsApp sticker support via a sticker action, explicit send option, or WebP sticker detection instead of sending sticker WebP media as ordinary images.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes by source inspection for the current limitation: outbound WebP/image media still follows generic image handling, no sticker option can be carried, and the WhatsApp action surface does not support `sticker`. No live WhatsApp send was performed in this read-only review.

## Solution Assessment

Unclear until a maintainer chooses the public contract and encoding policy. The safer direction is an explicit WhatsApp sticker intent plus scoped encoding/metadata handling, not broad implicit WebP auto-detection.
