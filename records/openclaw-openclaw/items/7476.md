## Summary

Keep this issue open. Current main still lacks an outbound WhatsApp sticker path: WebP media routes through the generic image sender, no active send option carries sticker intent, WhatsApp action discovery does not expose `sticker`, and the related implementation PRs are closed unmerged.

## What This Changes

Add outbound WhatsApp sticker sending support, either as a dedicated sticker action/option or explicit WebP sticker routing.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: ... }` delivery, action/tool/docs alignment, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option/action to select a Baileys sticker payload. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an explicit sticker intent plus scoped media preservation/re-encoding, not only automatic `.webp` detection.
