## Summary

Keep this issue open. Current main has an internal WhatsApp `sendSticker` helper, but the user-facing message tool path still cannot request a WhatsApp sticker: `action=sticker` is not advertised or dispatched for WhatsApp, and ordinary WebP media still goes through the generic image payload branch.

## What This Changes

The issue requests outbound WhatsApp sticker send support so WebP sticker media can be delivered as native stickers instead of regular image attachments.

## Best Possible Solution

Implement a user-facing WhatsApp sticker contract in the bundled WhatsApp plugin, route it through Baileys `{ sticker: buffer }`, keep generic image sends unchanged, and add focused tests plus WhatsApp docs for the chosen contract.

## Reproduction Assessment

Yes, at source level. Current main still rejects WhatsApp `action=sticker` through the action support path, and ordinary `image/webp` media is routed as an image payload rather than a sticker payload.

## Solution Assessment

No, current main is not yet the best solution because the only sticker sender is an internal helper, not the user-facing message action path. A maintainable fix should choose an explicit WhatsApp sticker contract before adding routing, validation, tests, and docs.
