## Summary

Keep this issue open. Current main still lacks outbound WhatsApp sticker sending: WebP media is still sent through the generic image path, no active send option carries sticker intent, WhatsApp action discovery does not expose sticker, and both related implementation PRs are closed unmerged.

## What This Changes

Request first-class outbound WhatsApp sticker support via a sticker action, explicit send flag, or safe WebP sticker detection.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` delivery, action/tool/docs alignment, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option/action to select Baileys sticker media. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an explicit sticker intent plus scoped encoding/media handling, not only automatic `.webp` detection.
