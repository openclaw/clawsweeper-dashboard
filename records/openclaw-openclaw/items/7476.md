## Summary

Keep this issue open. Current main still has no outbound WhatsApp sticker path: WebP media is routed as a generic image, the active send options cannot carry sticker intent, WhatsApp action discovery only exposes react/poll, and the related implementation PRs at https://github.com/openclaw/openclaw/pull/7502 and https://github.com/openclaw/openclaw/pull/19592 are closed unmerged.

## What This Changes

Request outbound WhatsApp sticker sending via a sticker action, explicit sticker flag, or safe WebP sticker detection.

## Best Possible Solution

Implement first-class outbound sticker support in the bundled WhatsApp plugin with explicit sticker intent, scoped media preservation or conversion, Baileys `{ sticker: buffer }` delivery, action/tool/docs alignment, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main routes `image/webp` through the generic image branch and has no sticker option/action to select Baileys sticker media. I did not run a live WhatsApp device send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an explicit sticker intent plus encoding/metadata handling, not only automatic `.webp` detection.
