## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media follows the generic image path, no sticker intent is carried through the WhatsApp send options, and the related implementation PR is closed unmerged.

## What This Changes

Request outbound WhatsApp sticker support so WebP sticker media or a sticker action can send native WhatsApp stickers instead of image attachments.

## Best Possible Solution

Implement plugin-owned WhatsApp outbound sticker support through an explicit sticker action or send intent, preserve intended WebP sticker media before generic image optimization, route sends through the native sticker payload, and document/test the contract.

## Reproduction Assessment

Yes for source-level reproduction. Current main recognizes WebP as image media, optimizes non-GIF images through the generic media finalizer, sends all image/* WhatsApp media as image payloads, and does not advertise or handle a WhatsApp sticker action.

## Solution Assessment

Yes. The narrow maintainable direction is a bundled WhatsApp plugin change using the existing message-action/plugin boundaries, with a safer explicit sticker contract rather than relying only on caption-based WebP auto-detection.
