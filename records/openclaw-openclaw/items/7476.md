## Summary

Keep open. Current main still lacks outbound WhatsApp sticker support, and both related implementation PRs are closed unmerged while the active WhatsApp send path still treats WebP as ordinary image media.

## What This Changes

Add support for sending native WhatsApp stickers from OpenClaw, either through an explicit sticker action/send option or correctly routed WebP sticker media.

## Best Possible Solution

Implement plugin-owned WhatsApp outbound sticker support with an explicit send intent/action, route intended sticker media through Baileys’ sticker payload, and document/test the new contract.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes image/webp through the generic image branch, has no sticker option in ActiveWebSendOptions, and does not advertise or handle a WhatsApp sticker action; live client rendering proof was not run.

## Solution Assessment

Yes. The narrow maintainable solution is a bundled WhatsApp plugin change using an explicit sticker intent/action and Baileys { sticker: buffer }, while avoiding a global image-optimization behavior change for all WebP media.

Keep the change within the bundled WhatsApp plugin and existing channel/message-action boundaries. Prefer an explicit sticker intent such as a WhatsApp-supported sticker action or sendAsSticker/asSticker option; avoid caption-based implicit WebP auto-detection unless the current codebase already has a clear matching pattern.
