## Summary

Keep open. Current main still lacks an outbound WhatsApp sticker contract and still routes WebP/image media through generic image handling; the two related implementation PRs are closed unmerged, and recent discussion adds unresolved encoding and metadata policy concerns.

## What This Changes

Request outbound WhatsApp sticker support through a dedicated action, send option, or WebP sticker detection path.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, scoped WebP preservation or re-encoding, Baileys `{ sticker: buffer }` routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for the current limitation by source inspection: outbound `image/webp` still follows generic image handling, no sticker option can be carried, and the WhatsApp action surface does not support `sticker`. No live WhatsApp send was performed in this read-only review.

## Solution Assessment

Unclear until a maintainer chooses the public contract and encoding policy. The safer direction is an explicit WhatsApp sticker intent plus encoding/metadata handling, not broad implicit WebP auto-detection.
