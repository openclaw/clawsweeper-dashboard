## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is routed through the generic image payload, no sticker intent is carried through WhatsApp send options or action discovery, and the only linked implementation PR is closed unmerged.

## What This Changes

Request support for sending WhatsApp stickers from the WhatsApp message tool via an explicit sticker action/option or WebP sticker routing.

## Best Possible Solution

Add a WhatsApp-plugin-owned outbound sticker path using the existing sticker action surface or an explicit send intent, preserve intended WebP sticker media, route Baileys sends through { sticker: buffer }, and document the behavior.

## Reproduction Assessment

Yes for source-level reproduction. On current main, createWebSendApi().sendMessage() with mediaType "image/webp" reaches the generic image branch, while WhatsApp action discovery and supportsAction do not include sticker.

## Solution Assessment

Yes, a narrow WhatsApp plugin change is the right direction if it uses an explicit sticker action or intent and adds focused tests. The closed PR's WebP auto-detection is useful context, but silent caption-based detection should not be the only contract unless maintainers choose that deliberately.
