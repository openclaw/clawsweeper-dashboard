## Summary

Keep open. Current main still routes outbound WhatsApp WebP media through the generic image path, has no sticker send intent in the active send contract or action surface, and both related implementation PRs are closed unmerged.

## What This Changes

Add outbound WhatsApp sticker sending so a WebP or explicit sticker intent is delivered as a WhatsApp sticker instead of a regular image attachment.

## Best Possible Solution

Add plugin-owned WhatsApp sticker sending with an explicit documented intent, scoped WebP preservation or re-encoding behavior, Baileys sticker payload routing, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main routes `image/webp` through generic image handling, has no sticker option in active send options, and does not advertise or dispatch a WhatsApp `sticker` action.

## Solution Assessment

Unclear until a maintainer chooses the public contract. The safer direction is an explicit WhatsApp sticker intent plus encoding handling, rather than only global implicit WebP auto-detection.
