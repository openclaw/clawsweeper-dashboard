## Summary

Keep open. Current main and the latest shipped tag still flatten non-link Telegram formatting entities before `BodyForAgent`, and the linked closing PR is open rather than merged, so this issue should remain open until that fix lands or is replaced.

## What This Changes

The issue requests preserving Telegram inbound formatting entities such as bold, italic, code, pre, strikethrough, underline, and spoiler as Markdown in the agent-facing prompt body.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83873, or a replacement with equivalent current-head proof, so Telegram-owned inbound entity rendering happens before `rawBody` and `BodyForAgent` are finalized.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with text `Hello world` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, and that `bodyText` becomes `BodyForAgent`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to render supported Telegram inbound entities in the Telegram helper/body layer before `rawBody` and `BodyForAgent` are built; the open linked PR is the current implementation candidate.
