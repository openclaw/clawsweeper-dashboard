## Summary

Keep open. Current main and the latest shipped tag still flatten Telegram formatting entities before the model-facing body is built; the issue remains narrow, source-reproducible, and suitable for a focused Telegram plugin fix.

## What This Changes

Convert Telegram inbound rich-text entities such as bold, italic, code, pre, and strikethrough into model-visible Markdown before building the agent prompt body.

## Best Possible Solution

Add a Telegram-owned inbound entity-to-Markdown renderer before `rawBody` and `bodyText` are finalized, preserving existing `text_link` expansion and covering text plus caption entities with focused offset tests.

## Reproduction Assessment

Yes, source-reproducible: a Telegram update with `text: "Hello world"` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, and that flattened `bodyText` becomes `bodyForAgent`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin body/helper layer and should render supported inbound entities from original Telegram offsets before the prompt body is built.
