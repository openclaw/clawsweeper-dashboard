## Summary

Keep open. Current main and the latest release still flatten non-link Telegram text entities before `bodyForAgent`, so the reported bold/italic/code formatting loss remains a real Telegram plugin bug.

## What This Changes

The issue asks for Telegram formatting entities such as bold, italic, code, pre, and strikethrough to be preserved as Markdown in the agent-facing prompt body.

## Best Possible Solution

Add a Telegram-owned inbound entity-to-Markdown renderer before `rawBody` and `bodyText` are finalized, preserving existing `text_link` expansion and covering text plus caption entities.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with `text: "Hello world"` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, then `bodyText` is used as `bodyForAgent`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin body/helper layer and should render supported entities from original Telegram offsets before the prompt body is built.
