## Summary

Keep open. Current main and v2026.5.20 still only expand Telegram `text_link` entities before `bodyForAgent` is built, and the open linked PR is the correct implementation path to review rather than closing the issue early.

## What This Changes

The issue reports that Telegram inbound formatting entities such as bold, italic, code, and strikethrough are flattened instead of rendered as Markdown in the agent-facing body.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83873, or an equivalent Telegram-owned entity renderer that preserves Telegram UTF-16 offsets before `bodyForAgent` is built.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with `text: "Hello world"` and a `bold` entity at offset 6 length 5 flows through `expandTextLinks`, which ignores non-`text_link` entities, and `bodyText` is then assigned to `bodyForAgent`. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes, the best direction is the linked Telegram helper/body-layer fix that renders supported inbound entities while preserving existing `text_link` behavior and UTF-16 offsets. The issue should stay open until that PR or an equivalent fix lands.
