## Summary

Keep open. Current main still drops Telegram formatting entities before `BodyForAgent`/`RawBody`; the issue discussion identified the same gap, and the current helper/tests still only handle `text_link` expansion. This is a narrow Telegram plugin bug suitable for a focused repair PR.

## What This Changes

The issue requests preserving inbound Telegram formatting entities such as bold, italic, code, pre, strikethrough, underline, and spoiler as Markdown in the agent-visible prompt.

## Best Possible Solution

Implement an inbound Telegram entity-to-Markdown renderer in the Telegram plugin before `rawBody`/`bodyText` are built, preserving existing `text_link` behavior and keeping core unchanged. Add focused tests for text, captions, links plus formatting, adjacent/nested entities, and each supported formatting entity type.

## Reproduction Assessment

Yes. A Telegram message with plain text plus a `bold` entity follows the current `resolveTelegramInboundBody` path into `expandTextLinks`, whose existing test explicitly expects a `bold` entity to leave the text unchanged.

## Solution Assessment

Yes, a narrow Telegram plugin fix is the best direction. The fix should compose link and formatting conversion against original Telegram offsets rather than applying formatting after link expansion without offset adjustment.
