Keep open: current main and the latest release still flatten Telegram formatting entities before the agent sees the message, and the open linked PR is the correct fix path rather than closing this issue early.

## What This Changes

The issue requests rendering Telegram inbound bold, italic, code, pre, strikethrough, and related entities as Markdown in the agent-facing prompt body.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83873, or an equivalent Telegram-owned entity renderer that preserves Telegram UTF-16 offsets before bodyForAgent is built.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with text "Hello world" and a bold entity at offset 6 length 5 flows through expandTextLinks, which ignores non-text_link entities, then bodyText is assigned to bodyForAgent. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes, the best direction is the linked Telegram helper/body-layer fix that renders supported inbound entities while preserving existing text_link behavior and Telegram UTF-16 offsets.
