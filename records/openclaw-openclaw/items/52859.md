## Summary

Keep open. Current main and v2026.5.19 still only expand Telegram `text_link` entities, so formatting entities remain flattened; https://github.com/openclaw/openclaw/pull/83873 is an open closing PR and should remain the fix vehicle rather than closing this issue or queuing another fix.

## What This Changes

Preserve Telegram inbound bold, italic, code, pre, strikethrough, underline, spoiler, and link entities as Markdown in the agent-facing prompt body.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/83873; if it lands, close this issue as implemented by that PR.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with text `Hello world` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities before `bodyText` becomes `bodyForAgent`. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Yes for the implementation shape: rendering Telegram entities in the Telegram helper/body layer before `rawBody` and `bodyText` are assigned is the narrow maintainable fix. Because the linked closing PR is still open, this issue should stay open until that PR lands or is closed.
