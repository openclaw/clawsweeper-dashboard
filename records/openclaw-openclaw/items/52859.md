## Summary

Keep open. Current main still sends Telegram formatting entities through a helper that only expands `text_link`, so bold/italic/code-style emphasis is flattened before `BodyForAgent`/`RawBody` are built.

## What This Changes

The issue requests restoring inbound Telegram formatting entities as Markdown in the agent-facing prompt text.

## Best Possible Solution

Add a Telegram-owned inbound entity-to-Markdown renderer before `rawBody` and `bodyText` are built, preserving existing link expansion and covering text, captions, adjacent entities, and nested or overlapping formatting.

## Reproduction Assessment

Yes. From source inspection, a Telegram message with `text: "Hello world"` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, so `BodyForAgent` receives plain `Hello world`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin body/helper layer and should compose all supported entity rendering from original Telegram offsets rather than applying formatting after link expansion without offset adjustment.
