## Summary

Keep open. Current main still collects Telegram text/caption entities but only expands `text_link` before building the agent-facing body, so bold/italic/code-style entities are flattened before they reach the prompt.

## What This Changes

Telegram inbound formatting entities such as bold, italic, code, pre, strikethrough, and underline should be preserved as Markdown in the agent-facing prompt text.

## Best Possible Solution

Add a Telegram-owned inbound entity-to-Markdown renderer before `rawBody` and `bodyText` are finalized, preserving existing link expansion and covering both text and caption entities.

## Reproduction Assessment

Yes. From source inspection, a Telegram message with `text: "Hello world"` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, so the agent receives plain `Hello world`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin body/helper layer and should render supported entities from original Telegram offsets before the prompt body is built.
