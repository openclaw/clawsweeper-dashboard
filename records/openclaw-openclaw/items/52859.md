## Summary

Keep open. Current main still collects Telegram text/caption entities but only expands `text_link` before constructing the agent-facing body, so bold/italic/code-style Telegram formatting is flattened before it reaches `BodyForAgent`.

## What This Changes

Issue requests preserving Telegram formatting entities as Markdown in Telegram agent-facing prompts.

## Best Possible Solution

Add a Telegram-owned inbound entity-to-Markdown renderer before `rawBody` and `bodyText` are finalized, preserving existing `text_link` expansion and covering text plus caption entities.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with `text: "Hello world"` and a `bold` entity at offset 6 length 5 reaches `expandTextLinks`, which ignores non-`text_link` entities, and then `bodyText` is used as `BodyForAgent`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin body/helper layer and should render supported entities from original Telegram offsets before the prompt body is built.
