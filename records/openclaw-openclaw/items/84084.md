## Summary

Close as implemented: current main now budget-scales both no-context-engine Codex mirrored-history continuity projections and covers the fresh, stale, and forced-fresh paths with regression tests; the fix is main-only and not in v2026.6.8.

## What This Changes

Make Codex no-context-engine legacy mirrored-history continuity projections use a contextTokenBudget-scaled rendered-context cap instead of the 24k default.

## Best Possible Solution

Keep the merged https://github.com/openclaw/openclaw/pull/94093 implementation as canonical and treat overlapping unmerged fix PRs as superseded by current main.

## Reproduction Assessment

Yes. The original failure was source-reproducible on pre-fix code: unset `plugins.slots.contextEngine`, prior user history, a fresh or stale Codex continuity projection, and a high `contextTokenBudget` caused the helper to fall back to 24,000 rendered characters; current main no longer follows that path.

## Solution Assessment

Yes. Reusing the existing budget helper for both no-context-engine continuity call sites, plus fitting only projected context under Codex's input cap, is the narrow maintainable solution; the separate native thread churn investigation remains distinct.
