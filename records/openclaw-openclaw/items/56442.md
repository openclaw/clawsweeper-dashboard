## Summary

Keep this PR open. Current main still does not implement the explicit `parentUpdates: "notify"` API or ACP-specific announce source, and the proposed branch still needs a rebase/design pass because notify-only completion would overlap with current task-registry delivery and notify-mode progress can enter the parent run as live embedded steering.

## What This Changes

The PR adds an optional ACP `sessions_spawn` `parentUpdates` mode, routes ACP run terminal completion through subagent-style announce flow, and updates related docs, tests, and internal event source typing.

## Best Possible Solution

Rebase the feature onto current main and keep the API split precise: progress stays on the existing system-event relay, terminal completion uses a single announce or task-delivery path, notify-only mode suppresses every other user-facing completion path, and docs/tests/protocol types cover the resulting contract.

## Reproduction Assessment

Yes for the review blockers: a static current-main check shows notify-only spawns would still satisfy `streamTo !== "parent"` task tracking, and the PR diff shows progress snippets are queued through `queueEmbeddedPiMessage`. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The explicit API may be a valid direction, but the implementation should first account for current task-registry delivery and should not route untrusted progress updates through embedded parent steering.

Full review comments:

- **[P2] Suppress registry delivery for notify-only ACP spawns:** `src/agents/tools/sessions-spawn-tool.ts:180`
  - body: `parentUpdates: "notify"` can be used without `streamTo: "parent"`, which starts the new parent completion relay. On current main, `sessions_spawn` still creates background-task tracking whenever `streamTo !== "parent"`, so a rebased notify-only spawn can emit both the subagent-style parent announce and the existing `Background task done` delivery. Gate task tracking on notify mode as well, or return an explicit spawn result that tells the tool terminal delivery is already owned by the parent relay.
  - confidence: 0.84
- **[P2] Keep ACP progress out of embedded parent steering:** `src/agents/acp-spawn-parent-stream.ts:291-293`
  - body: In notify mode, progress updates try `queueEmbeddedPiMessage` before falling back to system events. That makes untrusted ACP child output a live steering message for the parent run, even though the PR design says `parentUpdates` controls terminal completion and `streamTo` remains the lightweight progress relay. Keep progress on the system-event path and reserve notify/announce for terminal completion.
  - confidence: 0.78
