## What This Changes

This PR adds a `parentUpdates: "notify"` ACP `sessions_spawn` mode, completion announce routing, event-source typing, docs, and tests.

## Best Possible Solution

Rebase into current main as a maintainer-approved terminal-only ACP completion notify contract that keeps progress on untrusted system events, updates the central event-source list, and proves a live ACP parent-completion flow.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows notify-mode progress using `queueEmbeddedPiMessage`, and current main shows that queue can inject into active parent or reply runs. I did not run a live ACP session in this read-only review.

## Solution Assessment

No, not as currently proposed. The feature direction is plausible, but notify should own terminal completion only, the current central event-source contract must be updated, and the change needs real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep ACP progress on the system-event relay:** `src/agents/acp-spawn-parent-stream.ts:291-293`
  - body: When `parentUpdates: "notify"` is combined with progress relay, progress calls `tryNotifyParentRun` and queues ACP child output into the active parent run before falling back to system events. That contradicts the PR contract that `streamTo` owns lightweight progress while `parentUpdates` owns terminal completion, and it lets untrusted ACP progress steer the parent live.
  - confidence: 0.86
- **[P2] Add ACP to the central event-source list:** `src/agents/subagent-announce.ts:550`
  - body: Current main derives the internal-event type and gateway schema from `AGENT_INTERNAL_EVENT_SOURCES`, but this branch only updates the old local union/schema. After rebase, emitting `source: "acp"` from the announce flow must be backed by the central source list or the new event source will fail type/schema validation.
  - confidence: 0.78
- **[P3] Cache failed parent-session lookups:** `src/agents/acp-spawn-parent-stream.ts:248-249`
  - body: `cachedParentSessionId` only short-circuits truthy session IDs, so a parent entry with a missing or empty `sessionId` repeats config/store loading on every progress update. Track whether resolution was attempted so a known miss does not keep doing disk work on the relay path.
  - confidence: 0.72
