## Summary

Keep open: current main still lacks the opt-in ACP parent completion mode, and the related ACP wake report was closed with this PR as the canonical path. The branch still needs maintainer review because it is dirty against current main and has source-backed blockers around duplicate terminal delivery and progress steering.

## What This Changes

The PR adds an opt-in ACP `sessions_spawn` `parentUpdates: "notify"` mode that routes ACP run terminal completion through subagent-style parent announcements, with related docs, tests, and event source typing updates.

## Best Possible Solution

Rebase onto current main and land a terminal-only ACP completion notify contract that suppresses all other user-facing completion paths, keeps ACP progress on the untrusted system-event relay, and updates the current protocol/event-source contract plus docs and tests.

## Reproduction Assessment

Yes for the review blockers: source inspection shows current main still task-registers non-`streamTo` ACP spawns, and the PR diff routes notify-mode progress through embedded parent steering. I did not run tests because this was a read-only review and `pnpm` is unavailable.

## Solution Assessment

No, not as currently proposed. The API direction may be useful, but the branch should first integrate with current-main task delivery and reserve notify/announce semantics for terminal completion rather than live progress steering.
