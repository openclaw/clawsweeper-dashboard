## Summary

Keep open: the PR addresses a real ACP parent-completion gap and is the canonical path referenced by related triage, but current main does not already implement it and the branch still has blocking prompt-routing/security issues plus missing real behavior proof.

## What This Changes

The PR adds an opt-in ACP `sessions_spawn` `parentUpdates` mode, docs, and tests to route terminal ACP run completion through subagent-style announce handling while leaving default behavior unchanged.

## Best Possible Solution

Rework the branch as a terminal-only ACP completion notify contract: keep progress on the system-event relay, add `acp` to the central internal-event source contract, preserve focused regressions, and require redacted live ACP proof.

## Reproduction Assessment

Yes for the review findings: PR-head source routes notify-mode progress through `tryNotifyParentRun`, and current main's embedded queue defaults to steering-capable active-run input. I did not run a live ACP session in this read-only review.

## Solution Assessment

No, not as currently proposed. The feature direction is plausible, but progress should remain on the untrusted system-event relay, `acp` must be added to the current central event-source contract, and real behavior proof is still required.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep ACP progress on the system-event relay:** `src/agents/acp-spawn-parent-stream.ts:291-294`
  - body: When `parentUpdates: "notify"` is combined with `streamTo: "parent"`, progress calls `tryNotifyParentRun` and queues ACP child output into the active parent run before falling back to system events.   - confidence: 0.86
- **[P2] Add ACP to the central event-source list:** `src/agents/subagent-announce.ts:550`
  - body: Current main derives `AgentInternalEvent.source` and the gateway schema enum from `AGENT_INTERNAL_EVENT_SOURCES`, but this branch only updates the older local union/schema shape. After rebase, emitting `source: "acp"` must be backed by the central source list or the new event source will fail type/schema validation.
  - confidence: 0.8
- **[P3] Cache failed parent-session lookups:** `src/agents/acp-spawn-parent-stream.ts:248-249`
  - body: `cachedParentSessionId` only short-circuits truthy session IDs, so a parent entry with a missing or empty `sessionId` repeats config/store loading on every progress update. Track whether resolution was attempted so a known miss does not keep doing disk work on the relay path.
  - confidence: 0.72
