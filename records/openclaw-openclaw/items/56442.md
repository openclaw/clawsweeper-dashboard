## Summary

Keep open: current main still lacks the proposed `parentUpdates` ACP API, and this PR remains the canonical implementation path discussed by related ACP parent-completion reports. The branch still needs maintainer review because the diff routes notify-mode progress into parent-run steering, misses the current central internal-event source contract, and has no real after-fix ACP behavior proof.

## What This Changes

Adds opt-in ACP `parentUpdates: "notify"` for `sessions_spawn` run-mode completions, routing terminal ACP results through subagent-style announcements with docs and tests.

## Best Possible Solution

Rebase into current main as a terminal-only ACP completion notify contract: keep progress on the system-event relay, add `acp` to the central event-source contract, and provide live ACP parent-completion proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows notify-mode progress calling the parent-run queue, and current main shows that queue can steer active embedded runs. I did not run a live ACP session in this read-only review.

## Solution Assessment

No, not as currently proposed. The feature direction is plausible, but progress should remain on the untrusted system-event relay, `acp` must be added to the current central event-source list, and real behavior proof is still required.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep ACP progress on the system-event relay:** `src/agents/acp-spawn-parent-stream.ts:291-293`
  - body: When `parentUpdates: "notify"` is combined with progress relay, progress calls `tryNotifyParentRun` and queues ACP child output into the active parent run before falling back to system events. That contradicts the PR contract that `streamTo` owns lightweight progress while `parentUpdates` owns terminal completion, and it lets untrusted ACP progress steer the parent live.
  - confidence: 0.86
- **[P2] Add ACP to the central event-source list:** `src/agents/subagent-announce.ts:550`
  - body: Current main derives the internal-event type and gateway schema from `AGENT_INTERNAL_EVENT_SOURCES`, but this branch only updates the old local union/schema. After rebase, emitting `source: "acp"` from the announce flow must be backed by the central source list or the new event source will fail type/schema validation.
  - confidence: 0.8
- **[P3] Cache failed parent-session lookups:** `src/agents/acp-spawn-parent-stream.ts:248-249`
  - body: `cachedParentSessionId` only short-circuits truthy session IDs, so a parent entry with a missing or empty `sessionId` repeats config/store loading on every progress update. Track whether resolution was attempted so a known miss does not keep doing disk work on the relay path.
  - confidence: 0.72
