The PR adds an optional `parentUpdates: "system" | "notify"` mode for ACP `sessions_spawn`, routes notify-mode terminal completion through subagent-style announce flow, updates event schemas/docs, and adds targeted tests.

## Best Possible Solution

Land a rebased opt-in terminal-only ACP completion notify path that keeps progress on the existing untrusted system-event relay, adds ACP to the central event-source contract, and aligns with the parent-wake cluster.
