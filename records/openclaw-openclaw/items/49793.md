## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-10`
  - body: Current `openclaw/plugin-sdk/media-understanding` does not export `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`, so this added file will fail to typecheck after rebase. Import those from `openclaw/plugin-sdk/provider-http` and keep media-only helpers on the media subpath.
  - confidence: 0.95
- **[P1] Forward provider HTTP policy to video uploads:** `extensions/xiaomi/media-understanding-provider.ts:95-101`
  - body: `describeVideo` receives `params.request`, but this `postJsonRequest` call never resolves or forwards provider HTTP policy. That skips configured proxy, dispatcher, and private-network controls for uploaded user video bytes.
  - confidence: 0.94
- **[P1] Use Xiaomi media defaults for video:** `extensions/xiaomi/media-understanding-provider.ts:64`
  - body: The media runner can pass the active Xiaomi text model into `describeVideo`; normal Xiaomi onboarding defaults to `mimo-v2-flash`, which current metadata marks text-only. Preferring `params.model` here can send videos to the wrong model instead of the Omni media model/default.
  - confidence: 0.9
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:61`
  - body: Runtime registration alone does not populate manifest-backed media discovery and defaults before plugin runtime is active. Add `contracts.mediaUnderstandingProviders` plus matching `mediaUnderstandingProviderMetadata` with image/video capabilities and media-capable defaults.
  - confidence: 0.86
