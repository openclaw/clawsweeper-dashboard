## Summary

Keep open. Current main still does not register Xiaomi as a video media-understanding provider, so the feature remains useful, but the submitted patch is not mergeable against the current SDK and media-provider contract because it misses provider HTTP policy, video-safe defaults, and manifest metadata.

## What This Changes

The PR adds and registers a Xiaomi media-understanding provider intended to describe inbound images and videos with MiMo V2 Omni.

## Best Possible Solution

Ship Xiaomi video understanding as plugin-owned Xiaomi code using the shared OpenAI-compatible video helpers, `openclaw/plugin-sdk/provider-http` request-policy plumbing, video-specific MiMo V2 Omni defaults, manifest metadata, focused tests, and matching public docs while preserving current generic image support.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Current-main verification is high confidence: Xiaomi has image-capable catalog metadata but no Xiaomi video media-understanding registration or manifest contract.

## Solution Assessment

No, not as submitted. The direction is right, but the patch needs current SDK imports, request-policy forwarding, manifest-backed defaults, and video-safe model selection before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-9`
  - body: This new provider imports `assertOkOrThrowHttpError`, `normalizeBaseUrl`, and `postJsonRequest` from `openclaw/plugin-sdk/media-understanding`, but current main exposes those helpers from `openclaw/plugin-sdk/provider-http`. As written, the new file will not typecheck against the current SDK surface.
  - confidence: 0.94
- **[P1] Forward request policy to Xiaomi video calls:** `extensions/xiaomi/media-understanding-provider.ts:90-97`
  - body: The video request posts user video bytes with provider credentials but never resolves or forwards `params.request`, `allowPrivateNetwork`, or `dispatcherPolicy`. Existing video providers use `resolveProviderHttpRequestConfig` so operator proxy/private-network policy is honored for provider egress.
  - confidence: 0.91
- **[P1] Keep Xiaomi video off the text-only default model:** `extensions/xiaomi/media-understanding-provider.ts:65`
  - body: `describeXiaomiVideo` prefers `params.model`, and current auto-selection passes the active Xiaomi reply model through for video. A normal Xiaomi install uses `mimo-v2-flash`, which the catalog declares as text-only, so default video understanding would call the wrong model instead of MiMo V2 Omni.
  - confidence: 0.88
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/media-understanding-provider.ts:117-122`
  - body: Runtime registration alone does not update manifest-backed media discovery or bundled fallback/default metadata. Add `contracts.mediaUnderstandingProviders` and `mediaUnderstandingProviderMetadata` for Xiaomi so discovery, defaults, and auto-priority work before plugin runtime loading.
  - confidence: 0.85

## Security Review

Status: needs_attention
