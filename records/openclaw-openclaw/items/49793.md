## Summary

Keep open. Xiaomi video media understanding is still missing on current main, so the feature remains useful, but this PR is stale against the current plugin SDK and misses current transport, model-selection, and manifest requirements.

## What This Changes

The PR adds and registers a Xiaomi media-understanding provider intended to describe inbound images and videos with MiMo V2 Omni.

## Best Possible Solution

Add Xiaomi media understanding inside `extensions/xiaomi` using the current single-provider plugin shape, shared OpenAI-compatible video helpers, `openclaw/plugin-sdk/provider-http` request policy resolution, video-safe Xiaomi model selection, manifest contracts/metadata, focused Xiaomi media tests, and matching public docs.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Current-main verification is high confidence: Xiaomi has no media-understanding registration or manifest contract, and the current docs/provider matrix excludes Xiaomi from video understanding.

## Solution Assessment

No, not as submitted. The direction is right, but the patch needs to be updated to current SDK exports, request-policy forwarding, manifest-backed defaults, and Xiaomi video model selection before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-9`
  - body: This new provider imports `assertOkOrThrowHttpError`, `normalizeBaseUrl`, and `postJsonRequest` from `openclaw/plugin-sdk/media-understanding`, but current main exposes those helpers from `openclaw/plugin-sdk/provider-http`. As written, the new file will not typecheck against the current SDK surface.
  - confidence: 0.94
- **[P1] Forward request policy to Xiaomi video calls:** `extensions/xiaomi/media-understanding-provider.ts:90-97`
  - body: The video request posts user video bytes with provider credentials but never resolves or forwards `params.request`, `allowPrivateNetwork`, or `dispatcherPolicy`. Existing video providers use `resolveProviderHttpRequestConfig` so operator proxy/private-network policy is honored for provider egress.
  - confidence: 0.91
- **[P1] Keep Xiaomi video off the text-only default model:** `extensions/xiaomi/media-understanding-provider.ts:63`
  - body: `describeXiaomiVideo` prefers `params.model`, and current auto-selection passes the active Xiaomi reply model through for video. A normal Xiaomi install uses `mimo-v2-flash`, which the bundled catalog declares as text-only, so default video understanding would call the wrong model instead of MiMo V2 Omni.
  - confidence: 0.88
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/media-understanding-provider.ts:117-122`
  - body: Runtime registration alone does not update the manifest-backed media registry or bundled fallback/default metadata. Add `contracts.mediaUnderstandingProviders` and `mediaUnderstandingProviderMetadata` for Xiaomi so discovery, defaults, and auto-priority work before plugin runtime loading.
  - confidence: 0.85

## Security Review

Status: needs_attention
