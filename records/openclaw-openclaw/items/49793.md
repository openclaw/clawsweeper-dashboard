## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-10`
  - body: On current main, `openclaw/plugin-sdk/media-understanding` does not export `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`, so this file will fail after rebase. Import those helpers from `openclaw/plugin-sdk/provider-http` and keep media-only helpers on the media subpath.
  - confidence: 0.96
- **[P1] Forward provider HTTP policy to video uploads:** `extensions/xiaomi/media-understanding-provider.ts:95-101`
  - body: `describeVideo` receives `params.request`, but this call never resolves or forwards provider HTTP request policy. That skips configured proxy, dispatcher, and private-network controls for uploaded user video bytes.
  - confidence: 0.95
- **[P1] Use a Xiaomi media model for video:** `extensions/xiaomi/media-understanding-provider.ts:64`
  - body: The video auto path can pass the active Xiaomi text model into `describeVideo`; normal Xiaomi onboarding defaults to text-only `mimo-v2-flash`. Preferring `params.model` here can send videos to the wrong model instead of the Omni media model/default.
  - confidence: 0.91
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:61`
  - body: Runtime registration alone does not give manifest-backed media discovery/defaults a Xiaomi provider before runtime loads. Add `contracts.mediaUnderstandingProviders` plus matching media metadata with image/video capabilities and defaults.
  - confidence: 0.87
