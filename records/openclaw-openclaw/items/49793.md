## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-9`
  - body: On current main, `openclaw/plugin-sdk/media-understanding` does not export `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`; those helpers live in `openclaw/plugin-sdk/provider-http`, so this file will fail to typecheck after rebase.
  - confidence: 0.95
- **[P1] Forward request policy to Xiaomi video posts:** `extensions/xiaomi/media-understanding-provider.ts:92-98`
  - body: `runProviderEntry` passes provider request policy into `describeVideo`, but this `postJsonRequest` call never resolves `params.request` with `resolveProviderHttpRequestConfig` or forwards `allowPrivateNetwork`/`dispatcherPolicy`, so configured proxy and private-network policy are ignored for user video bytes.
  - confidence: 0.93
- **[P1] Keep Xiaomi video requests on MiMo V2 Omni:** `extensions/xiaomi/media-understanding-provider.ts:60`
  - body: Video auto-selection can pass Xiaomi's active default model, `mimo-v2-flash`, into `describeVideo`. This line prefers that text-only model over the MiMo V2 Omni fallback, so normal Xiaomi installs can send video requests to the wrong model.
  - confidence: 0.9
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:62`
  - body: Runtime registration alone does not populate manifest-backed media discovery/defaults before plugin runtime is active. Add `contracts.mediaUnderstandingProviders` and `mediaUnderstandingProviderMetadata` for Xiaomi so capability loading and auto-selection can see the provider and its media defaults.
  - confidence: 0.86
