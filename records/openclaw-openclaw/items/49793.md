## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-10`
  - body: On current main, `openclaw/plugin-sdk/media-understanding` does not export `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`, so this file will fail after rebase. Import those helpers from `openclaw/plugin-sdk/provider-http` and keep media-only helpers on the media subpath.
  - confidence: 0.96
- **[P1] Forward provider HTTP policy to video uploads:** `extensions/xiaomi/media-understanding-provider.ts:93-99`
  - body: `describeVideo` receives `params.request`, but this call never resolves or forwards the provider HTTP request policy. That skips configured proxy, dispatcher, and private-network controls for uploaded user video bytes.
  - confidence: 0.95
- **[P1] Use a Xiaomi media model for video:** `extensions/xiaomi/media-understanding-provider.ts:65`
  - body: The media runner can pass the active Xiaomi text model into `describeVideo`; normal Xiaomi onboarding defaults to `mimo-v2-flash`, which current metadata marks text-only. Preferring `params.model` here can send videos to the wrong model instead of the Omni media model/default.
  - confidence: 0.91
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:61`
  - body: Runtime registration alone does not populate manifest-backed media discovery and defaults before plugin runtime is active. Add `contracts.mediaUnderstandingProviders` plus matching `mediaUnderstandingProviderMetadata` with image/video capabilities and media-capable defaults.
  - confidence: 0.88
