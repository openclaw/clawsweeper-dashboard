## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks Xiaomi video media understanding, while the PR is stale against the current provider SDK shape and misses the manifest metadata, model-default, and provider HTTP policy integration needed for a maintainable Xiaomi media provider.

## What This Changes

The PR adds a Xiaomi media-understanding provider, registers it from the Xiaomi plugin, and implements image handling through shared helpers plus a custom MiMo V2 Omni video chat-completions request.

## Best Possible Solution

Rework the contribution on current main: keep Xiaomi behavior inside extensions/xiaomi, add a media-understanding provider that uses the shared OpenAI-compatible video helpers and openclaw/plugin-sdk/provider-http policy helpers, declare contracts.mediaUnderstandingProviders plus mediaUnderstandingProviderMetadata with image/video defaults for mimo-v2-omni, update focused Xiaomi registration/default/request tests, and align the public media docs.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Current-main verification is high confidence: Xiaomi registers only speech, its manifest has no media-understanding contract, and the video runner requires describeVideo.

## Solution Assessment

No, this is not currently the best way to solve it. The right direction is still a Xiaomi media-understanding provider, but the submitted patch needs to be updated to the manifest-first provider shape, current SDK exports, model-default handling, and provider HTTP policy seam.

  - confidence: 0.88

  - body: The runner passes request-policy data into `describeVideo`, but this PR calls `postJsonRequest` without resolving and forwarding `allowPrivateNetwork` or `dispatcherPolicy`. That skips the same proxy/private-network controls used by the existing Moonshot and Qwen video providers.
  - confidence: 0.9
- **[P2] Declare Xiaomi media defaults in the manifest:** `extensions/xiaomi/media-understanding-provider.ts:117-122`
  - body: Registering the provider at runtime is not enough for current manifest-backed discovery and defaults.
