Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-10`
  - body: On current main, `openclaw/plugin-sdk/media-understanding` no longer exports `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`, so this file will fail after rebase. Import HTTP helpers from `openclaw/plugin-sdk/provider-http` and keep media helpers on the media subpath.
  - confidence: 0.95
- **[P1] Forward provider HTTP policy to video uploads:** `extensions/xiaomi/media-understanding-provider.ts:95-101`
  - body: `describeVideo` receives `params.request`, but this `postJsonRequest` call never resolves or forwards provider HTTP request policy. That skips configured proxy, dispatcher, and private-network controls for uploaded user video bytes.
  - confidence: 0.94
- **[P1] Use a media-capable Xiaomi model for video:** `extensions/xiaomi/media-understanding-provider.ts:64`
  - body: The media runner can pass the active Xiaomi chat model into video understanding, and normal Xiaomi onboarding defaults to text-only `mimo-v2-flash`. Because this line prefers `params.model`, the `mimo-v2-omni` fallback may not run for normal installs.
  - confidence: 0.9
- **[P2] Declare Xiaomi media metadata and defaults:** `extensions/xiaomi/media-understanding-provider.ts:116-121`
  - body: Runtime registration alone does not populate manifest-backed media discovery and defaults before plugin runtime is active. Add `contracts.mediaUnderstandingProviders`, matching media metadata, and provider `defaultModels` for Xiaomi image/video capabilities.
  - confidence: 0.86
