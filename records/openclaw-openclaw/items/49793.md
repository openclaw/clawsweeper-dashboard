  - body: `openclaw/plugin-sdk/media-understanding` does not export these HTTP helpers on current main; they are exported from `openclaw/plugin-sdk/provider-http`, so this new file will not compile against the current SDK surface.
  - confidence: 0.95

  - body: This posts user video bytes with provider credentials but never resolves or forwards `params.request`, `allowPrivateNetwork`, or `dispatcherPolicy`. Existing provider media calls use `resolveProviderHttpRequestConfig` so proxy and private-network policy are honored.
  - confidence: 0.93
- **[P1] Select MiMo V2 Omni for video by default:** `extensions/xiaomi/media-understanding-provider.ts:58-59`
  - body: `describeXiaomiVideo` prefers the incoming `params.model`, and the video runner passes the active Xiaomi reply model through. A normal Xiaomi install defaults to text-only `mimo-v2-flash`, so automatic video understanding can call the wrong model instead of MiMo V2 Omni.
  - confidence: 0.9
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:61`
  - body: Runtime registration alone does not populate manifest-backed media discovery or pre-runtime defaults. Add Xiaomi `contracts.mediaUnderstandingProviders` plus `mediaUnderstandingProviderMetadata` and matching provider defaults/priority so automatic media selection can find the provider correctly.
  - confidence: 0.86
