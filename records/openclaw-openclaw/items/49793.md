## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Import HTTP helpers from provider-http:** `extensions/xiaomi/media-understanding-provider.ts:7-9`
  - body: On current main, `openclaw/plugin-sdk/media-understanding` does not export `assertOkOrThrowHttpError`, `normalizeBaseUrl`, or `postJsonRequest`; those helpers are in `openclaw/plugin-sdk/provider-http`, so this file will fail to typecheck after rebase.
  - confidence: 0.95
- **[P1] Forward request policy to video posts:** `extensions/xiaomi/media-understanding-provider.ts:90-97`
  - body: `runProviderEntry` passes provider request policy into `describeVideo`, but this call never resolves it with `resolveProviderHttpRequestConfig` or forwards `allowPrivateNetwork`/`dispatcherPolicy` to `postJsonRequest`, so proxy and private-network policy are bypassed for user video bytes.
  - confidence: 0.93
- **[P1] Keep video requests on MiMo V2 Omni:** `extensions/xiaomi/media-understanding-provider.ts:65`
  - body: Video auto selection passes the active Xiaomi model into `describeVideo`; the default active model is `mimo-v2-flash`, and this line prefers it over the `mimo-v2-omni` fallback, so normal Xiaomi installs can send video requests to a text-only model.
  - confidence: 0.91
- **[P2] Declare Xiaomi media metadata in the manifest:** `extensions/xiaomi/index.ts:61`
  - body: Capability loading and default media selection rely on manifest contracts/metadata before plugin runtime is active. Runtime registration alone leaves Xiaomi video undiscoverable in manifest-backed flows, so add `contracts.mediaUnderstandingProviders` and `mediaUnderstandingProviderMetadata` with defaults/priority.
  - confidence: 0.86
