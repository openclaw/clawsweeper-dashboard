## Summary

Keep open. The branch addresses a source-reproducible DeepInfra catalog/cache problem, but the current diff still changes shipped defaults and has an auth-context mismatch in video capability discovery; real behavior proof remains incomplete.

## What This Changes

The PR refactors the DeepInfra plugin to discover per-surface models live, register image/video catalog providers, inject Anthropic cache markers for DeepInfra `anthropic/*` streams, and update DeepInfra docs, defaults, changelog, and tests.

## Best Possible Solution

Land a narrower DeepInfra catalog/cache update that preserves shipped defaults unless a migration is explicitly approved, uses request-scoped auth for live discovery/capabilities, and includes redacted real-path proof for the changed live behavior.

## Reproduction Assessment

Yes. Source inspection shows current main documents dynamic DeepInfra discovery while `augmentModelCatalog` only returns configured rows, and it advertises `anthropic/*` cache eligibility while using an OpenRouter-gated cache wrapper.

## Solution Assessment

No. The proposed direction is useful, but the patch is not the best merge shape until it preserves or intentionally migrates shipped defaults and uses the same auth context for discovery/capabilities that generation uses.
