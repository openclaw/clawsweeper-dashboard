## Summary

Keep open: the PR is directionally useful, but the proposed discovery fix is still env-only for a key path that OpenClaw already resolves through provider auth, and it also changes shipped implicit DeepInfra defaults without an upgrade-safe maintainer decision.

## What This Changes

The PR updates the DeepInfra plugin to use live model discovery, add image/video model catalog providers, adjust default models, and inject Anthropic cache-control markers for DeepInfra Anthropic streams.

## Best Possible Solution

Use the existing resolved provider-auth contract for all DeepInfra discovery paths, preserve shipped defaults unless maintainers explicitly approve a tested migration, then rebase and prove the catalog and cache behavior on the affected paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence focused reproduction: use a resolved DeepInfra provider API key with `DEEPINFRA_API_KEY` absent from env, then the PR branch's discovery gate returns the static fallback instead of the live catalog.

## Solution Assessment

No. The live discovery and cache-wrapper direction is reasonable, but the implementation should use OpenClaw's resolved auth context and avoid silently changing implicit defaults without maintainer-approved upgrade behavior.
