## Summary

Keep open because current main still has the reported codex-cli observability gap: the CLI runner prepares bundle MCP config but passes an empty tool list into the system prompt report. PR #65409 is an open implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that codex-cli systemPromptReport tool entries and schema character counts reflect injected bundle MCP tools instead of staying empty.

## Best Possible Solution

Review and land PR #65409, or an equivalent narrow fix, so the CLI runner materializes the effective bundle MCP tool metadata for systemPromptReport, disposes any temporary catalog runtime promptly, preserves backend MCP config merging semantics, and adds regression assertions for entries and schemaChars.

Yes. A high-confidence source-level reproduction exists: run a CLI backend with bundleMcp enabled; prepareCliRunContext prepares bundle MCP config, then calls buildSystemPromptReport with tools: [], and the helper derives entries and schemaChars solely from that array.

## Solution Assessment
