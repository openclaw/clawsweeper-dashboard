## Summary

Keep open. Current main still emits the generic verbose-off edit failure warning without a sanitized exact-match mismatch explanation, while https://github.com/openclaw/openclaw/pull/67276 remains the open, conflicting implementation candidate for this exact report.

## What This Changes

The issue asks for clearer user-facing messaging when `edit.oldText` mismatches due to whitespace and a way for reliability-focused deployments to avoid `edit`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/67276 with a sanitized payload-layer exact-match summary and regression coverage; keep broader edit normalization work under https://github.com/openclaw/openclaw/issues/62863 and use existing `tools.deny` for disabling `edit`.

## Reproduction Assessment

Yes, at source level. Build an embedded-run payload with an `edit` exact-match mismatch `lastToolError` and verbose mode off; current main keeps `includeDetails` false and emits the generic failed-tool warning without the mismatch reason.

## Solution Assessment

Yes for the narrow bug. A sanitized payload-layer summary is the narrowest maintainable fix because it improves the visible warning without changing `edit` matching semantics or leaking raw file contents.
