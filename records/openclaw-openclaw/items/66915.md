## Summary

Keep open. Current main still suppresses ordinary verbose-off edit mismatch details into a generic mutating-tool warning, while PR #67276 remains the explicit open implementation candidate; the config-side workaround is already covered by tools.deny, but the actionable warning fix has not landed.

## What This Changes

Improve edit-tool failure surfacing when oldText does not exactly match due to whitespace or newline differences, and provide a reliable way to avoid edit in deployments that prefer write-only behavior.

## Best Possible Solution

A focused payload-layer fix should surface a sanitized exact-match mismatch summary while leaving edit semantics unchanged; broader edit normalization belongs in #62863, and deployments that want to disable edit should use tools.deny.

## Reproduction Assessment

Yes. The source path is directly reproducible by an edit exact-match mismatch with verbose mode off: host-edit can throw richer mismatch context, but the payload policy hides it and emits a generic mutating-tool failure warning.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix for this issue because it summarizes a known edit mismatch case at the final payload layer without changing the edit tool contract or exposing raw file contents.
