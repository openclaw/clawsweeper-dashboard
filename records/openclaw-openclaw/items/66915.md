## Summary

Keep open. Current main still hides the useful `edit` exact-match mismatch reason in normal verbose-off payloads, and the focused closing implementation remains open at https://github.com/openclaw/openclaw/pull/67276 rather than merged.

## What This Changes

Issue requests clearer user-visible feedback when `edit` fails because `oldText` does not exactly match file contents, plus a way to disable `edit` for reliability-focused deployments.

## Best Possible Solution

Review, rebase, and either land or replace https://github.com/openclaw/openclaw/pull/67276 with a sanitized payload-layer summary and regression coverage; keep broader edit normalization work under https://github.com/openclaw/openclaw/issues/62863.

## Reproduction Assessment

Yes, at source level. Build an embedded-run payload with an `edit` exact-match mismatch `lastToolError` and verbose mode off; current main keeps `includeDetails` false and emits the generic failed-tool warning without the mismatch reason.

## Solution Assessment

Yes for the narrow bug. A sanitized payload-layer summary is the narrowest maintainable fix because it improves the visible warning without changing `edit` semantics or leaking raw file contents; existing `tools.deny` covers deployments that want to disable `edit`.
