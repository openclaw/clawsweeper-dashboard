## Summary

Keep open. Current main still collapses ordinary verbose-off edit exact-match mismatch details into a generic mutating-tool warning, while the lower edit layer has richer but raw mismatch context; the linked PR #67276 is still the explicit open implementation candidate, so this issue should not be closed before that PR is reviewed and landed or replaced.

## What This Changes

The issue requests clearer, actionable user-facing errors for edit exact-text mismatches and asks for a way to disable edit in favor of write-only file updates.

## Best Possible Solution

Review and land or replace PR #67276 with a focused sanitized payload-layer summary for edit exact-match mismatches, keep edit semantics unchanged, and use the existing tools.deny policy for deployments that want to disable edit.

## Reproduction Assessment

Yes. The source path is clear: an edit exact-match mismatch produces richer lower-layer context, but with verbose mode off the payload policy omits details and the warning builder falls back to a generic Edit failed payload.

## Solution Assessment

Yes. A narrow sanitized warning in the payload layer is the best fix for this report; changing edit matching semantics or adding an edit-specific disable flag is broader than needed because tools.deny already covers disabling edit.
