## Summary

Keep open. Current main still hides ordinary verbose-off `edit` mismatch details behind a generic mutating-tool failure warning, while the explicit closing PR at https://github.com/openclaw/openclaw/pull/67276 is open, unmerged, and currently conflicting.

## What This Changes

Improve the user-facing `edit` tool failure message for exact-text whitespace/newline mismatches, with the config-side disable-`edit` workaround accounted for.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/67276 with a sanitized payload-layer summary and focused regression coverage; keep broad edit normalization under https://github.com/openclaw/openclaw/issues/62863.

## Reproduction Assessment

Yes, at source level. Build an embedded-run payload with an `edit` exact-match mismatch `lastToolError` and `verboseLevel: "off"`; current main will show the mutating-tool warning while suppressing the mismatch reason.

## Solution Assessment

Yes for the narrow bug. A sanitized payload-layer summary is the best fix because it improves the visible warning without changing `edit` semantics or exposing raw file contents; disabling `edit` is already handled through `tools.deny`.
