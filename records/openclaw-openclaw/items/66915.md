## What This Changes

The issue asks OpenClaw to surface actionable user-facing guidance when the edit tool fails because oldText does not exactly match file whitespace or line endings.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/67276 with a sanitized payload-layer exact-match summary and focused regression coverage; keep broader edit normalization under https://github.com/openclaw/openclaw/issues/62863 and use existing tools.deny for disabling edit.

## Reproduction Assessment

Yes, at source level. Build an embedded-run payload with an edit exact-match mismatch lastToolError and verbose mode off; current main keeps includeDetails false and emits the generic failed-tool warning without the mismatch reason.

## Solution Assessment

Yes for the narrow bug. A sanitized payload-layer summary is the maintainable fix because it improves the visible warning without changing edit matching semantics or exposing raw lower-level file content; disabling edit is already covered by tools.deny.
