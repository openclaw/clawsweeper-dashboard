## Summary

Keep open. Current main still suppresses ordinary verbose-off `edit` mismatch details into a generic mutating-tool warning, and the explicit closing PR is open, unmerged, and currently conflicting.

## What This Changes

Improve `edit` exact-match mismatch UX by surfacing actionable, sanitized warning text, while relying on existing tool policy for deployments that want to block `edit`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/67276 with an edit-only sanitized payload summary and regression coverage; keep broader edit normalization under https://github.com/openclaw/openclaw/issues/62863.

## Reproduction Assessment

Yes. Source inspection on current main shows `edit` mismatch errors can be stored as `lastToolError`, while verbose-off payload generation suppresses the detail and emits only the generic failed-tool warning.

## Solution Assessment

Yes. A narrow payload-layer summary for the known exact-match mismatch is the best fix for this issue because it improves the visible error without changing `edit` semantics or exposing raw lower-level context; disabling `edit` is already covered by `tools.deny`.
