## Summary

Keep open: this PR addresses a real compaction conflict, but it is not merge-ready because it still passes raw config mode instead of the provider-aware safeguard state and it is missing the required changelog entry.

## What This Changes

The PR extends the Pi auto-compaction guard to disable Pi auto-compaction for safeguard mode, threads compaction mode through embedded and CLI compaction paths, reapplies the guard after resource reload, and adds focused pi-settings tests.

## Best Possible Solution

Land a narrow fix that exposes or reuses one provider-aware OpenClaw-owns-compaction resolver, uses it in embedded and CLI guard calls, preserves context-engine ownsCompaction behavior, adds focused provider-backed safeguard coverage, and records the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with agents.defaults.compaction.provider set while mode is default or not materialized: current code enables the safeguard extension, but the PR guard sees only the raw mode and leaves Pi auto-compaction enabled.

## Solution Assessment

No. The PR is close, but the maintainable fix should derive the guard from the same provider-aware safeguard resolution used to load the compaction extension instead of threading only the raw mode string.
