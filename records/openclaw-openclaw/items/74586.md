The issue reports that Active Memory embedded recall aborts `memory_search` tool calls, then records timeout/empty status even when the recall model finishes and leaks prior prompt-error events into later runs.

## Best Possible Solution

Land a focused repair that preserves Active Memory’s hard deadline, avoids treating completed `NONE`/no-recall output as a timeout, prevents stale `openclaw:prompt-error` entries from reaching later LLM input, and validates the embedded `memory_search` abort path with regression coverage.
