## Summary

Keep open: current main still contains the Active Memory embedded-recall abort, timeout-classification, and prompt-error persistence paths that match the report; related PR #74480 is only a setup-grace timeout-budget slice, not the complete fix.

## What This Changes

Active Memory embedded recall should stop aborting memory_search calls, classify completed NONE recalls as empty rather than timeout, and avoid carrying prompt-error entries into later recall context.

## Best Possible Solution

Keep Active Memory bounded by a hard outer deadline, but only abort memory tool dispatch when that deadline actually wins; completed NONE or empty recalls should be classified as empty, and stale prompt-error custom entries should be filtered before later LLM-boundary context.
