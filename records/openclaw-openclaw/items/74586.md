## Summary

Keep open: current main still contains the Active Memory embedded-recall timeout, abort propagation, and prompt-error lifecycle paths that match the reported failure shape; adjacent setup-grace and memory-tool materialization work does not fully cover this issue.

## What This Changes

Fix Active Memory embedded recall so `memory_search` calls are not prematurely aborted, completed `NONE` recalls are not misclassified as timeouts, and stale prompt-error events do not leak into later recall context.

## Best Possible Solution

Keep the recall run bounded by a hard outer deadline, but separate setup/model/tool deadlines so memory tools only receive an aborted signal after the active recall budget actually wins, completed `NONE` recalls classify as empty, and `openclaw:prompt-error` entries are filtered or cleared before the next LLM boundary.
