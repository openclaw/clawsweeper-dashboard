## Summary

Keep open: current main still matches the reported Active Memory embedded-recall failure shape. ## What This Changes

The issue reports Active Memory embedded recall aborting `memory_search` calls and returning timeout/empty status even when the embedded model completed.

## Best Possible Solution

Keep Active Memory bounded by a hard outer deadline, but only abort memory tool dispatch when that deadline actually wins; classify a completed `NONE` recall as empty, and filter stale `openclaw:prompt-error` entries before future LLM-boundary context.
