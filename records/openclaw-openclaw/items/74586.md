## Summary

Keep open: current main still matches the reported Active Memory embedded-recall failure shape. The related setup-grace PR covers an adjacent timeout-budget slice, but current source still forwards the Active Memory abort signal into the embedded run, still maps recovered `NONE` output on the timeout path to `status: "timeout"`, and still persists prompt-error custom entries that are not filtered by the current LLM-boundary custom-message filter.

## What This Changes

The issue reports Active Memory embedded recall returning aborted `memory_search` tool results, treating completed `NONE` recalls as timeouts, and leaking prior prompt-error context into later runs.

## Best Possible Solution

Keep Active Memory bounded by a hard outer deadline while ensuring memory tools only receive an aborted signal after that deadline actually wins, completed `NONE` recalls classify as empty, and stale `openclaw:prompt-error` entries are filtered before later LLM-boundary context.
