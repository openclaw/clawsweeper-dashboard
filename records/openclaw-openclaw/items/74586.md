## Repair Work Prompt

Fix the Active Memory embedded recall bug tracked by https://github.com/openclaw/openclaw/issues/74586. Current main forwards the Active Memory watchdog abort signal into `runEmbeddedPiAgent`, wraps memory tools with that signal, reports empty recovered `NONE` output as `status=timeout`, and persists `openclaw:prompt-error` custom entries that can leak into later embedded runs. Make one focused repair that preserves the hard Active Memory deadline, prevents premature `memory_search` aborts inside healthy embedded recalls, treats completed `NONE`/empty recall as successful empty recall rather than timeout when the model completed within budget, and keeps stale prompt-error diagnostics out of later LLM context. Coordinate with #74480's setup-grace timeout handoff but do not broaden into provider routing, new config knobs, or a core migration. Add regression coverage in the Active Memory plugin and the embedded-runner/runtime-context/tool abort seam, and add a user-facing `CHANGELOG.md` fix entry if behavior changes are committed.
