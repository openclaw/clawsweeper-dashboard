## What This Changes

The issue requests that Active Memory embedded recall stop aborting memory_search tool calls, classify completed NONE/empty recalls correctly, and avoid carrying prior prompt-error entries into later embedded runs.

Keep Active Memory bounded by a hard outer deadline, but only abort memory tool dispatch when that deadline actually wins; classify resolved NONE/empty recalls as empty, filter stale prompt-error custom entries before future LLM-boundary context, and add focused regression tests around the embedded recall path.
