## Summary

Keep open: current main still contains the Active Memory embedded-recall abort, timeout-classification, and prompt-error lifecycle paths that match the report; related #74480 is open and only covers the setup-grace timeout handoff, not the full observed failure.

## What This Changes

Active Memory embedded recall should stop aborting memory tool calls inside hidden runs, classify completed `NONE` replies as empty instead of timeout, and avoid leaking prior prompt-error events into the next run.

## Best Possible Solution

Keep Active Memory bounded by a hard outer deadline while ensuring memory tools only see an aborted signal after that deadline actually wins, completed `NONE` recalls classify as empty, and stale `openclaw:prompt-error` entries are filtered or expired before future LLM-boundary context.
