## Summary

Keep open. Current main still routes new direct messages according to `session.dmScope`, but it does not reconcile, retire, archive, or mark stale peer-keyed direct-session rows after switching back to `main`; the documented cleanup command remains retention/missing-transcript maintenance rather than dmScope-aware repair.

## What This Changes

The issue asks OpenClaw to reconcile, retire, mark inactive, or provide an official cleanup path for stale peer-keyed direct DM sessions after switching `session.dmScope` back to `main`.

## Best Possible Solution

Add a channel-agnostic session maintenance path that detects direct-session rows inconsistent with current main-scoped DM routing, previews them in dry-run output, and preserves history while retiring, archiving, or marking them inactive only under an explicit repair/enforce path; update CLI docs and focused tests for the cleanup behavior.

## Reproduction Assessment

Yes. Seed a session store with `agent:main:main` and `agent:main:telegram:direct:123` while config has `session.dmScope: "main"`; current routing resolves new Telegram DMs to the main key, while current cleanup has no dmScope-specific action for the stale peer-keyed row.

## Solution Assessment

Yes, the narrow maintainable direction is a generic cleanup or doctor-style audit/repair with dry-run and history preservation. Silent startup deletion or a Telegram-specific special case would be less safe.
