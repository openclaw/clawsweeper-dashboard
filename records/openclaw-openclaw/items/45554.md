## Summary

Keep open. Current main still sends new main-scoped DMs to `agent:<agent>:main`, but the session store, session listing, and cleanup command do not reconcile or mark older peer-keyed direct-session rows after `session.dmScope` is switched back to `main`.

## What This Changes

Retire, archive, mark, or provide official cleanup for stale peer-keyed direct DM sessions when `session.dmScope` returns to `main`.

## Best Possible Solution

Add a generic sessions cleanup or doctor repair path that detects peer-keyed direct sessions inconsistent with current main-scoped DM routing, shows them in dry-run output, and preserves history while retiring, archiving, or marking them inactive under an explicit fix/enforce action.

## Reproduction Assessment

Yes, from source inspection. Seed a store with `agent:main:main` plus `agent:main:telegram:direct:123` while config has `session.dmScope: "main"`; route construction returns the main key, but listing and cleanup paths still keep the peer-keyed row as an ordinary session.

## Solution Assessment

Yes. The narrow maintainable path is to extend the existing session cleanup/doctor maintenance surface with a history-preserving stale-direct-session repair, rather than adding Telegram-specific routing behavior or deleting rows silently at startup.
