## Best Possible Solution

Define one maintainer-approved recovery architecture that keeps process supervision isolated from the failing Gateway, reuses existing health and doctor contracts, applies bounded restart escalation, and adds operator notification only through an explicit secure configuration.

## Risks / Open Questions

- The request combines process supervision, model-error retry policy, startup fallback, health telemetry, and notification into one feature; choosing these independently could create competing recovery controllers or unsafe fallback behavior.
- A process watchdog cannot reliably rescue its own crashed process, so maintainers must decide which logic belongs in external service supervision versus an isolated OpenClaw process.
- New restart, degraded-mode, and notification configuration would affect defaults, operator expectations, security boundaries, and cross-platform upgrade behavior.
