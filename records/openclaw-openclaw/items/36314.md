## Summary

Keep open: current main has adjacent Gateway diagnostics, liveness/readiness probes, channel-scoped health restarts, and hardcoded fatal-error handling, but it still lacks the requested coherent Gateway process watchdog, operator-configurable recovery policy, richer `/health` contract, and general degraded-startup mode. Related implementation candidates are still open, including https://github.com/openclaw/openclaw/pull/53716 and https://github.com/openclaw/openclaw/pull/46502, so this needs maintainer scoping rather than cleanup closure.

## What This Changes

Requests a built-in Gateway resilience surface covering a process watchdog, configurable global error-recovery strategies, richer `/health` details, and degraded startup.

## Best Possible Solution
