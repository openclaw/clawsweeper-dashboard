## Summary

Keep open for normal PR review and CI: the patch addresses a real cron/task-maintenance startup race, includes sufficient terminal proof, and I did not find blocking correctness or security issues.

## What This Changes

The branch adds cron active-job liveness authority state, wraps cron startup in reconciling/authoritative transitions, and updates task maintenance/tests to preserve mapped cron tasks during startup.

## Best Possible Solution

Land the narrow startup-liveness guard after CI and maintainer merge gates, while leaving already-terminal lost-row recovery to the separate durable-recovery PR.

## Reproduction Assessment

Yes. Current main has a source-level path where an expired mapped cron task is evaluated while cronRuntimeAuthoritative is true and activeJobIds is empty before cron startup reconciliation completes; the PR body’s isolated terminal proof exercises the fixed before/after behavior.

## Solution Assessment

Yes. The patch is a narrow maintainable fix for the startup window because it changes only the active-job liveness authority boundary and keeps post-start lost-task behavior intact.
