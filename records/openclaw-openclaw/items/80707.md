## Summary

Keep open for normal PR review and merge gating: current main has a source-visible cron startup liveness race, the proposed fix is narrow, real behavior proof is supplied, and I found no blocking correctness or security findings.

## What This Changes

Adds a cron active-job liveness authority flag around cron startup and teaches task maintenance to preserve mapped cron task rows only while that liveness state is non-authoritative.

## Best Possible Solution

Land the narrow startup-liveness guard after required CI and maintainer review, while keeping durable repair of already-terminal lost cron rows tracked separately in https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection on current main shows task maintenance can run as cron-runtime-authoritative before cron.start() reconciles the process-local activeJobIds set after restart.

## Solution Assessment

Yes. The proposed authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores the existing missing-active-job lost behavior once cron start exits.
