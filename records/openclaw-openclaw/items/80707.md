## Summary

Keep open for maintainer review: the PR is a focused cron/task-ledger race fix with sufficient terminal proof and no blocking code findings, but it intentionally changes when mapped cron task rows may remain `running` during startup liveness reconciliation.

## What This Changes

The PR adds a cron active-job liveness authority flag, marks cron startup as non-authoritative until `start()` exits, and teaches task maintenance to preserve mapped cron task rows only during that window.

## Best Possible Solution

Land the narrow liveness-authority fix if maintainers accept the startup contract; otherwise ask for full gateway restart proof or move to a durable cron-state design before merge.

## Reproduction Assessment

Yes. Current main's cron branch trusts the empty process-local `activeJobIds` set after the reconcile grace period, and the supplied terminal proof demonstrates the fixed pre-start and post-start liveness transition.

## Solution Assessment

Yes. The PR is a narrow owner-boundary fix that gates only mapped cron task liveness during cron startup and restores authoritative behavior in `finally`; a durable cron-state design is broader and not required for this specific race.
