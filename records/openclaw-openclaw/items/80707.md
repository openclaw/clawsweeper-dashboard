## Summary

Keep open for normal maintainer review: this is a focused, still-relevant bug fix for a source-reproducible cron startup race, the added proof now satisfies the real-behavior gate, and I found no blocking correctness or security issue.

## What This Changes

The PR adds a cron active-job liveness authority flag, wraps cron startup reconciliation with that state, and updates task maintenance/tests so mapped cron tasks are preserved until startup liveness is authoritative.

## Best Possible Solution

Land the narrow startup-liveness guard after normal CI and maintainer merge gates, while keeping already-terminal lost-row recovery in the separate durable recovery PR.

## Reproduction Assessment

Yes. The source-level path is an expired mapped cron task during the gap where task maintenance is cron-authoritative but activeJobIds is still empty before cron startup reconciliation finishes; the contributor's isolated state-dir terminal proof exercises that path after the fix.

## Solution Assessment

Yes. The patch is the narrow maintainable fix for the startup window because it preserves existing lost-task behavior after cron startup exits and avoids bundling the separate durable terminal-row recovery work.
