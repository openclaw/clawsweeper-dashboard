Exact-head CI is not fully green yet, so merge should wait for maintainer handling of the remaining check state.

## What This Changes

The PR adds cron active-job liveness authority tracking during startup and uses it in task maintenance so mapped cron tasks are not marked lost before cron startup reconciliation finishes.

## Best Possible Solution

Land this narrow startup-liveness guard after exact-head CI and maintainer review, while leaving durable recovery of already-terminal lost cron rows to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection shows current main starts task maintenance as cron-runtime-authoritative before post-ready cron.start, while activeJobIds is empty until startup reconciliation, so a stale mapped cron task can take the false lost path.

## Solution Assessment

Yes. The authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores the existing missing-active-job lost behavior once cron startup exits.
