## Summary

Keep open. Current main still lacks a startup-liveness authority signal, and this fresh PR is a plausible narrow fix with no blocking review findings; it still needs normal maintainer review and completion of exact-head checks.

## What This Changes

The branch adds a cron active-job liveness authority flag, toggles it around `cron.start()`, and uses it in task maintenance plus focused tests to avoid startup false-positive lost cron tasks.

## Best Possible Solution

Land the narrow startup-liveness guard after exact-head CI and maintainer review, while leaving durable repair of already-terminal lost cron rows to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Current main starts task maintenance as cron-runtime-authoritative before post-ready `cron.start()`, while `activeJobIds` is empty until cron startup reconciliation, so a stale mapped cron task can take the false `lost` path.

## Solution Assessment

Yes. The authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores the existing missing-active-job lost behavior once cron startup exits.
