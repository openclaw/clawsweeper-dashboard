## Summary

Keep open. The patch targets a real source-visible cron/task-maintenance race and I found no blocking code finding, but this external PR still only supplies typecheck/unit-test proof rather than after-fix proof from a real gateway or CLI task-maintenance setup.

## What This Changes

The PR adds a cron active-job liveness-authority flag, marks cron startup reconciliation non-authoritative until `start()` exits, and teaches task maintenance to preserve mapped cron tasks during that startup window.

## Best Possible Solution

Land a narrowly scoped startup-liveness guard after contributor real-behavior proof, while keeping durable terminal lost-row recovery as a separate reviewed follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level path is an expired mapped cron task during the gap where task maintenance is cron-authoritative but `activeJobIds` is still empty before cron startup reconciliation finishes; I did not run a live gateway restart reproduction in this read-only review.

## Solution Assessment

Yes for the targeted startup window. The PR keeps the existing lost-task contract after cron startup exits, while avoiding the false-positive pre-start window; broader durable recovery should stay in the linked follow-up rather than be bundled here.
