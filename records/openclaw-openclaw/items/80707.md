## Summary

Keep open for normal maintainer review: current main still has the startup liveness race, the PR is a narrow fix with sufficient real behavior proof, and I found no blocking correctness or security findings.

## What This Changes

The PR adds cron active-job liveness authority state, wraps cron startup with reconciling/authoritative transitions, and updates task-maintenance tests to preserve mapped cron tasks only during startup reconciliation.

## Best Possible Solution

Land the narrow startup-liveness guard after maintainer review and required CI, while leaving durable repair of already-terminal lost cron rows to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection of current main shows task maintenance is made cron-runtime-authoritative before post-ready cron.start(), while activeJobIds is process-local and empty until cron startup/run paths populate it.

## Solution Assessment

Yes. The authority flag is the narrowest maintainable fix because it masks missing active-job liveness only during cron startup reconciliation and restores the existing lost marking after startup exits.
