## Summary

Keep this PR open for normal merge review: the startup race is source-reproducible on current main, the proposed liveness authority flag is narrow, and I found no blocking correctness issue in the diff.

## What This Changes

The branch adds a cron active-job liveness authority flag around cron startup and uses it in task maintenance/tests to avoid false-positive lost cron tasks before startup reconciliation completes.

## Best Possible Solution

Land the narrow startup-liveness guard after required CI and maintainer review, while keeping durable repair of already-terminal lost cron rows tracked in https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection on current main shows task maintenance can run as cron-runtime-authoritative before cron.start() reconciles the process-local activeJobIds set after restart.

## Solution Assessment

Yes. The proposed authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores the existing missing-active-job lost behavior once cron start exits.
