## Summary

Keep open: this is a focused PR for a source-reproducible cron startup race on current main, and I found no blocking correctness issue in the diff. The remaining action is maintainer review plus any needed refresh against the newer main branch before merge.

## What This Changes

The branch adds cron active-job liveness authority state, marks cron startup reconciliation boundaries, and prevents task maintenance from marking mapped cron tasks lost while startup liveness is non-authoritative.

## Best Possible Solution

Land this focused liveness-authority guard after maintainer review and refreshed merge validation, while leaving already-terminal lost cron row recovery to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Current main starts task maintenance as cron-authoritative before cron.start and then hasBackingSession trusts an empty activeJobIds set for mapped stale cron tasks, making the false lost path source-reproducible.

## Solution Assessment

Yes. The PR adds a narrow authority bit owned by cron active-job state and consumes it only in the cron backing check, preserving lost marking after startup reconciliation exits.
