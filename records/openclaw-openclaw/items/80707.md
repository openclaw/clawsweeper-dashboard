## What This Changes

The PR adds cron active-job liveness authority tracking during startup and teaches task maintenance to preserve mapped cron tasks until cron startup reconciliation exits.

## Best Possible Solution

Land this narrow startup-liveness guard after maintainer review and check completion, while leaving recovery of already-terminal lost cron rows to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection of current main shows task maintenance starts as cron-runtime-authoritative before post-ready cron.start runs, while stale mapped cron tasks are checked against an initially empty process-local activeJobIds set.

## Solution Assessment

Yes. The PR’s authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores the existing missing-active-job lost behavior in a finally block.
