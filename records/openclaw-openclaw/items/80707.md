## Summary

Keep open: this PR is a focused fix for a source-reproducible cron/task-registry startup race, with sufficient terminal proof, passing PR checks, and no blocking functional or security findings. It should continue through normal maintainer review and landing rather than cleanup closure.

## What This Changes

The PR adds cron active-job liveness authority state, wraps cron startup reconciliation with reconciling/authoritative markers, and gates task-registry lost marking for mapped cron tasks until startup liveness is authoritative.

## Best Possible Solution

Land this focused liveness-authority guard after maintainer review and exact-head mergeability confirmation, while leaving already-terminal lost cron row recovery to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection shows current main starts authoritative task maintenance before async cron.start and then trusts an empty activeJobIds set for stale mapped cron tasks, making the false lost path reachable during startup.

## Solution Assessment

Yes. The PR adds a narrow authority bit at the cron active-job ownership boundary and consumes it only in the cron backing check, preserving lost marking after startup exits instead of disabling cron lost detection or folding terminal recovery into this patch.
