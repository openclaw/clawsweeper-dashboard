## Summary

Keep open. Current main still has the startup-window false `lost` path, while this PR adds a narrow cron liveness-authority guard with focused tests, sufficient terminal real-behavior proof, and no blocking review findings.

## What This Changes

The PR adds cron active-job liveness authority tracking, marks cron startup reconciliation boundaries, and preserves mapped stale cron task rows while that liveness signal is non-authoritative.

## Best Possible Solution

Land this focused liveness-authority guard after maintainer review, while keeping already-terminal lost-row recovery scoped to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes, source-level. Current main starts task maintenance with cron runtime authority before `cron.start()`, then `hasBackingSession()` trusts an empty process-local `activeJobIds` set for mapped stale cron tasks.

## Solution Assessment

Yes. The PR adds a narrow authority bit owned by cron active-job state and consumes it only in the cron backing check, preserving normal lost marking once startup reconciliation exits.
