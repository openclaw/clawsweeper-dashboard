## Summary

Keep open for normal maintainer review: current main still has the startup liveness gap, and the proposed patch is focused with no blocking correctness finding from this review.

## What This Changes

Adds a cron active-job liveness authority flag, toggles it around cron startup, and updates task maintenance/tests to preserve mapped cron tasks while startup liveness is non-authoritative.

## Best Possible Solution

Land the narrow startup-liveness guard after normal CI and maintainer merge gates, keeping terminal lost-row recovery in the separate durable recovery PR.

## Reproduction Assessment

Yes. Source inspection on current main shows task maintenance can run as cron-runtime-authoritative before cron startup has made the process-local activeJobIds set authoritative.

## Solution Assessment

Yes. The proposed authority flag is a narrow maintainable fix because it preserves mapped cron rows only during startup reconciliation and resumes existing lost-task behavior after cron start exits.
