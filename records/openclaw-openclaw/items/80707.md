## What This Changes

Adds a cron active-job liveness authority flag, toggles it during cron start, and updates task maintenance/tests to preserve mapped cron tasks while startup liveness is non-authoritative.

## Best Possible Solution

Land the narrow startup-liveness guard after normal CI and maintainer merge gates, while leaving durable recovery of already-terminal lost cron rows to https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes. Source inspection on current main shows task maintenance can run as cron-runtime-authoritative while cron liveness is still only the empty process-local activeJobIds set after restart.

## Solution Assessment

Yes. The proposed authority flag is a narrow maintainable fix because it protects only the startup reconciliation window and restores existing lost-task behavior once cron start exits.
