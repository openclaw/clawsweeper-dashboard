## What This Changes

The branch adds cron active-job liveness authority tracking, marks cron startup reconciliation around `cron.start()`, and teaches task maintenance to preserve mapped cron tasks while that liveness is non-authoritative.

## Best Possible Solution

Land this focused startup liveness guard, and keep terminal recovery of already-lost cron rows in https://github.com/openclaw/openclaw/pull/75036.

## Reproduction Assessment

Yes, source-level. Current main starts task maintenance with cron runtime authority before post-ready `cron.start()`, then `hasBackingSession` trusts an empty process-local `activeJobIds` set for stale mapped cron tasks.

## Solution Assessment

Yes. The PR introduces one cron-owned liveness authority bit and consumes it only in the cron backing check, preserving the existing lost behavior once startup exits.
