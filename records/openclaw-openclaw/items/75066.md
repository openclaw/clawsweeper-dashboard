## Summary

Keep open. Current main still has the reported session metadata activity-refresh paths, and this PR is a useful narrow fix, but the PR head still needs two mechanical cleanup items before merge: use the platform temp directory in the new fixture and add the required user-facing changelog entry.

## What This Changes

The PR adds a preserve-activity option to session store entry updates, applies it to compaction and memory-flush metadata writes, removes explicit compaction updatedAt writes, and adds regression coverage for legacy daily/idle reset behavior.

## Best Possible Solution

Land the preserve-activity fix after replacing the hard-coded test temp root with os.tmpdir() and adding a single-line Unreleased Fixes changelog entry. The reset policy itself should remain unchanged, and only metadata-only compaction/flush bookkeeping should opt into activity preservation.

## Reproduction Assessment

Yes. The linked bug gives concrete legacy daily and idle scenarios, and current main source confirms the reproduction path: metadata writes can advance updatedAt while reset freshness falls back to updatedAt when lifecycle timestamps are missing.

## Solution Assessment

Yes, after the two cleanup items. Reusing the existing preserve-activity merge policy for selected metadata-only writers is the narrowest maintainable fix; the remaining safer pre-merge alternative is only fixture portability plus changelog coverage.
