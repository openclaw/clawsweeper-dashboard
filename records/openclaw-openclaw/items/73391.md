## Best Possible Solution

Land the eager-registration fix after the contributor adds redacted real CLI proof and the normal checks are green or confirmed unrelated.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.7 fire-and-forget eager registration while browser command groups await dynamic imports, so `OPENCLAW_DISABLE_LAZY_SUBCOMMANDS=1 openclaw browser status` can parse before handlers attach. I did not run the CLI due the read-only review constraint.

## Solution Assessment

Yes. Awaiting all eager registrations and propagating async through the already-awaited CLI bootstrap is the narrow maintainable fix; the remaining blocker is proof, not a different code design.
