## Summary

Keep open. The PR is still useful because current main retains the eager-registration race and only has the narrower parent-flag repair, but the current PR head still has small merge-blocking lint and changelog issues that should be fixed before landing.

## What This Changes

The PR changes lazy CLI reparsing to walk to the root command with full fallback argv, propagates awaited command registration through core and browser CLI setup, and adds browser/CLI regression tests plus changelog entries.

## Best Possible Solution

Keep the PR direction, repair the small blockers, then land it: root/full-path reparse plus awaited eager registration is the narrow maintainable fix for the remaining lazy/eager CLI runtime gaps.

## Reproduction Assessment

Yes. The source-backed reproduction is `openclaw browser --browser-profile <name> status` for parent-option propagation and `OPENCLAW_DISABLE_LAZY_SUBCOMMANDS=1 openclaw browser status` for eager registration; current main and Commander 14.0.3 source explain both paths without needing speculative assumptions.

## Solution Assessment

Yes for the intended implementation direction, but no for the current head as-is. The branch should await all new async callsites, normalize the eager `Promise.all` mapper, and move the changelog bullets before it is mergeable.
