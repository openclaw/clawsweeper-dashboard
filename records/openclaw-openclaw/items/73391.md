## Summary

Keep open: the production fix direction is sound, but the latest PR head still fails lint because two tests ignore the newly Promise-returning browser CLI registrar, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR makes CLI command-group registration async so eager lazy-runtime registrations are awaited through core/browser CLI bootstrap paths, updates tests, and adds a changelog entry.

## Best Possible Solution

Await the two remaining test calls, rerun affected lint/tests/CI, and land the eager-registration fix after the contributor adds redacted real CLI proof that eager browser dispatch succeeds.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 fire-and-forget eager registrations while browser group registrars await dynamic imports, so parse can run before commands attach. I did not execute the CLI in this read-only review.

## Solution Assessment

Yes for the production approach: awaiting eager registrations and propagating async through already-awaited bootstrap paths is the narrow maintainable fix. This exact PR is not merge-ready until the missed awaits and real-behavior proof are addressed.
