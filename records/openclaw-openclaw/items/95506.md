## Summary

Keep open: current main and the latest release still key bundled MCP runtimes by session id, and the open candidate PR has not merged. The protected maintainer label plus the shared-runtime isolation/config decision make this maintainer-owned rather than cleanup-closeable.

## What This Changes

Request sharing bundled MCP runtime, catalog, and stdio ownership across sessions that have the same workspace and MCP config fingerprint.

## Best Possible Solution

Land one maintainer-approved opt-in shared MCP runtime implementation with session as the default and focused proof for fan-out, leases, idle eviction, config invalidation, disposal, and failure-boundary docs.

## Reproduction Assessment

Yes, at source level. Current main keys MCP runtime ownership and in-flight creation by session id, while each runtime owns its own catalog and MCP server sessions; I did not run a temporary stdio fan-out test because this review is read-only.

## Solution Assessment

Unclear as final product direction. Workspace/config-keyed sharing is the right implementation layer for the reported fan-out, but the safe shape needs maintainer acceptance because shared mode changes the session isolation boundary and likely adds an operator config option.
