## Summary

Keep open: current main and the latest release still key bundled MCP runtimes by session id, so the identical workspace/config fan-out remains source-reproducible. The protected maintainer label and open candidate PRs mean this needs maintainer handling, not cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid duplicating bundled MCP runtime, catalog, and stdio work across sessions that share the same workspace and MCP config fingerprint.

## Best Possible Solution

Land one maintainer-approved opt-in shared-runtime implementation, keep session isolation as the default, document the trusted single-tenant tradeoff, and cover fan-out, leases, idle sweep, config invalidation, disposal, and failure behavior.

## Reproduction Assessment

Yes, at source level: current main keys bundled MCP runtimes by session id, and each runtime owns its own catalog and MCP server sessions. I did not run the temporary stdio fan-out test because this review is read-only.

## Solution Assessment

Unclear as final product direction: workspace/config-keyed sharing is the right implementation layer for the reported fan-out, but the best fix must preserve the default isolation boundary and receive maintainer acceptance for the opt-in shared mode.
