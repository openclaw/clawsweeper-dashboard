## Summary

Keep open: current main and the latest release still key bundled MCP runtimes by session id, so identical workspace/config sessions can still fan out into duplicate runtime, catalog, and stdio work. ## What This Changes

The issue requests reducing duplicate MCP runtime/catalog/stdio work when many sessions share the same workspace and MCP configuration.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/79882 with a maintainer-approved pooling contract, explicit isolation docs, and focused fan-out, lease, idle-sweep, config-invalidation, disposal, and shared-server failure tests.
