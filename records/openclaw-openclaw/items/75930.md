## Summary

Keep open. Current main and the latest release still cap detached Dream Diary narrative work at three concurrent subagent runs, so the strict-provider multi-workspace failure mode described here remains plausible and source-reproducible.

## What This Changes

Request to make cron-triggered memory-core Dream Diary narrative generation provider-safe across multiple workspaces instead of launching concurrent detached LLM calls that skip most diary entries.

## Best Possible Solution

Serialize detached Dream Diary narrative runs by default, or otherwise enforce provider-safe memory-core concurrency while preserving per-workspace diary entries and existing phase outputs.

## Reproduction Assessment

Yes, from source inspection rather than a live provider run. Configure multiple dreaming workspaces on a provider that rejects concurrent same-key requests and trigger the cron path; current main can still start three detached narrative subagent calls at once, and the focused test asserts that behavior.

## Solution Assessment

Yes. A localized memory-core serialization or stricter provider-safe limiter is narrower than batching all workspaces into one diary entry or changing generic provider retry policy.
