## Summary

Keep open. Current main and v2026.5.3 still cap detached Dream Diary narrative runs at three, so the strict-provider multi-workspace failure remains possible even though the earlier unbounded fanout was fixed.

## What This Changes

The issue asks memory-core cron dreaming to avoid concurrent detached Dream Diary narrative API calls across multiple workspaces.

## Best Possible Solution

Serialize detached Dream Diary narrative runs by default, or otherwise enforce provider-safe concurrency of one, while preserving per-workspace and per-phase diary entries.

## Reproduction Assessment

Yes, at source level. Configure multiple memory-core dreaming workspaces and trigger the cron path; current main sets `detachNarratives` for cron and the focused test confirms the detached queue starts three `subagent.run` calls before queuing the rest.

## Solution Assessment

Yes. A localized memory-core serialization of detached narratives is the narrowest maintainable fix; batching all workspaces or adding a new config knob would change more product behavior than needed.
