## Summary

Keep open. Current main no longer has unbounded fan-out, but cron Dream Diary generation can still run three detached narrative subagent calls at once, which leaves the reported strict-provider failure mode unresolved.

## What This Changes

The issue asks memory-core dreaming to avoid concurrent detached Dream Diary narrative API calls across multi-workspace cron sweeps.

## Best Possible Solution

Serialize detached Dream Diary narrative generation by default in memory-core while preserving per-workspace DREAMS.md entries and existing phase outputs.

## Reproduction Assessment

Yes, from source inspection rather than a live provider run. Cron sets detachNarratives, each workspace phase enqueues through a shared cap-three helper, and the focused test asserts three in-flight subagent runs.

## Solution Assessment

Yes. A localized memory-core serialization or provider-safe limiter is narrower than batching workspaces or changing generic provider retry policy, and no new config surface is required for the default-safe fix.
