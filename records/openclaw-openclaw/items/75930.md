## Summary

Keep open. Current main and the latest release still allow three detached Dream Diary narrative subagent runs at once during cron dreaming sweeps, so the strict-provider failure mode reported here is still plausible and not implemented away.

## What This Changes

The issue asks memory-core to prevent cron Dream Diary narrative generation from firing multiple provider subagent calls concurrently across workspaces.

## Best Possible Solution

Serialize detached Dream Diary narrative runs by default, or enforce an equivalent provider-safe single-flight limiter, while preserving per-workspace diary entries and existing phase output semantics.

## Reproduction Assessment

Yes, source-reproducible rather than live reproduced. Configure multiple dreaming workspaces on a provider that rejects concurrent same-key requests and trigger the cron path; current main can still start three detached narrative subagent calls at once, and the focused test asserts that concurrency.

## Solution Assessment

Yes. A localized memory-core serialization or equivalent provider-safe limiter is narrower than batching all workspaces into one diary entry or changing generic provider retry policy.
