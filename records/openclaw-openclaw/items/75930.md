## Summary

Close as implemented on current main. The reported unbounded cron Dream Diary fan-out is now routed through a bounded detached narrative queue, the narrative timeout is 60 seconds, and regression coverage verifies the queue cap.

## What This Changes

The issue reports cron-triggered Dream Diary narrative generation firing all workspace and phase subagent calls concurrently, causing provider rate limits and missing diary entries.

## Best Possible Solution

Keep the current bounded detached narrative queue and release it; open a narrower follow-up only if live provider evidence shows concurrency 3 still needs provider-specific retry or configuration.

## Reproduction Assessment

Yes. The reported path is deterministic in source: cron sets detached narratives, phases enqueue Dream Diary generation, and current tests exercise the bounded detached queue that replaced the unbounded fan-out.

## Solution Assessment

Yes. The current queue implements the suggested limiter approach while preserving background cron behavior; a separate configurable limit would need fresh evidence that the fixed cap is still too high.
