## Summary

Keep open. Current `main` and the latest release still pin `@earendil-works/pi-tui` to a decoder that drops Alt/Ctrl-modified Kitty CSI-u printable sequences, while OpenClaw’s TUI path delegates printable editor input to that dependency.

## What This Changes

German-layout TUI input should insert printable characters once and support AltGr combinations such as `@` and `€`.

## Best Possible Solution

Land a focused AltGr-aware printable input fix at the TUI/pi-tui boundary, with regression coverage for German AltGr `@`/`€` and live proof that plain German printables insert once.

## Reproduction Assessment

Partly yes. The AltGr/Kitty CSI-u failure is source-reproducible through the OpenClaw delegation path and pi-tui decoder source; the duplicated plain German printable examples still need live German-layout terminal proof.

## Solution Assessment

No complete solution is on current main. The maintainable path is a narrow adapter or upstream pi-tui fix that allows AltGr-produced printable text without turning real Alt/Ctrl shortcuts into inserted characters.

Fix https://github.com/openclaw/openclaw/issues/48897. Keep the fix bounded to the TUI/pi-tui boundary or an upstream dependency update/patch; do not add broad keyboard-layout config or change unrelated shortcuts.
