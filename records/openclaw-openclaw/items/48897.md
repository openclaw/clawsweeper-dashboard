## Summary

Keep open. Current main still routes unhandled TUI editor input through pi-tui, and the pinned pi-tui decoder still rejects Ctrl+Alt AltGr printable CSI-u sequences while treating printable release events as insertable text. The linked fix PR uses closing syntax and is still open, so the issue should remain open until that PR, or an equivalent narrow fix, lands.

## What This Changes

The issue requests that the TUI input path insert German-layout printable characters once and correctly handle AltGr-produced `@` and `€`.

## Best Possible Solution

Review and land the linked PR, or an equivalent narrow CustomEditor/pi-tui-boundary fix, then close this issue after the fix is merged and shipped.

## Reproduction Assessment

Yes for the AltGr and printable release-event paths: current main delegates editor input to pi-tui, whose CSI-u printable decoder rejects Ctrl+Alt AltGr sequences and does not filter printable release events. Full physical-keyboard coverage would still benefit from a live German-layout terminal smoke.

## Solution Assessment

Yes, a narrow fix at the CustomEditor/pi-tui boundary is the maintainable path for OpenClaw while preserving existing shortcuts. The linked PR is the active implementation candidate, so the issue should stay open until that PR is reviewed and merged or replaced.
