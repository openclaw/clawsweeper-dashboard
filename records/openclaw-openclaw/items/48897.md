## What This Changes

Fix TUI/gateway-client German keyboard handling so printable characters are inserted once and AltGr combinations such as `@` and `€` work in the editor.

## Best Possible Solution

Keep this open and land a focused AltGr-aware TUI input fix, preferably through an upstream pi-tui release or a narrow OpenClaw adapter, with regressions for German `@`/`€` and duplicate printable pairs.

## Solution Assessment
