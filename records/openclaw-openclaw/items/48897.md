## Summary

Keep open. ## What This Changes

Fix German-layout TUI input so printable characters insert once and AltGr combinations produce characters such as `@` and `€` in the gateway-client editor.

## Best Possible Solution

Review and land the linked PR, or an equivalent narrow fix at the CustomEditor/pi-tui boundary, then close this issue after the fix is merged and shipped.

## Reproduction Assessment

Yes for the source-level raw Kitty CSI-u path: current main delegates editor input to pi-tui, whose decoder rejects Ctrl+Alt AltGr printable sequences and lets printable release events reach insertion. A live German-layout terminal smoke would still improve coverage.

## Solution Assessment

Yes. A narrow CustomEditor/pi-tui-boundary fix that accepts AltGr-produced printable CSI-u input and ignores printable release events is the maintainable path while preserving existing shortcuts.
