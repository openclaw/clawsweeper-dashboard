## Summary

Keep open: current main and the latest release still trust generated host-read HTML only under the OpenClaw temp root, while the reported Telegram path stages the file under media/outbound and reloads it there. The linked closing PR at https://github.com/openclaw/openclaw/pull/90579 remains open, so this issue should stay canonical until that fix lands or is replaced.

## What This Changes

Fix Telegram outbound delivery of trusted generated text/html attachments after outbound staging moves them outside the temp-root trust check.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90579 or an equivalent trust-preserving repair that carries verified generated-HTML provenance through outbound staging without trusting arbitrary staged HTML, then close this issue after merge.
