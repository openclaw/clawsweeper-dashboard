## What This Changes

The issue reports a Telegram regression where generated host-read HTML passes the original temp-root trust check, is staged into outbound media, then is rejected when Telegram reloads the staged copy outside the temp root.

## Best Possible Solution

Keep this issue open as the canonical user report, then close it only after the linked fix PR lands or maintainers replace it with another repair that carries trusted generated-HTML provenance through outbound staging without broadly trusting arbitrary staged HTML.

## Reproduction Assessment

Yes. Source inspection plus the reporter's logs show current main can stage local reply media into media/outbound and then Telegram can reload that staged .html through the host-read path, where only temp-root realpaths are trusted.
