## Label Justifications

- `P2`: The external plugin repeatedly loses semantic vector indexing for affected multilingual text, but remains operational through its keyword-search fallback.
- `impact:other`: The failure degrades semantic memory indexing and recall quality without matching a more specific ClawSweeper-owned impact category.

## Summary

The reported workflow is implemented by the external TencentDB Agent Memory plugin, while OpenClaw current main only supplies a generic sqlite-vec loader and has no memory-tdai indexing or embedding-request code to repair.

## What This Changes

Investigate and fix memory-tdai vector indexing failures on text containing U+2026 ellipsis characters.

## Best Possible Solution

Report and fix the problem in TencentCloud/TencentDB-Agent-Memory by making its remote embedding request path UTF-8-safe and adding a U+2026 regression test without stripping multilingual text.

## Reproduction Assessment

No; OpenClaw current main has no memory-tdai indexer to exercise, although the report provides a credible reproduction for the external TencentDB Agent Memory v0.3.6 plugin.

## Solution Assessment

No; changing OpenClaw or normalizing text before sqlite-vec would target the wrong ownership layer and risk losing multilingual content, while the external plugin should repair its embedding-request encoding directly.
