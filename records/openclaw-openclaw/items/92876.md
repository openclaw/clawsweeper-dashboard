## Summary

Keep open for maintainer review: the PR is a focused memory-wiki lint fix, but it only suppresses two structure errors for raw source pages while the same pages still emit the missing-`updatedAt` freshness warning from the reported frontmatter-noise class, and there is an open docs PR choosing the opposite source-page contract.

## What This Changes

The PR teaches memory-wiki lint to treat unmanaged raw Markdown under `sources/` as raw source evidence while still flagging generated bridge/unsafe-local source pages that appear malformed or truncated.

## Best Possible Solution

Decide the `sources/` contract first; if raw source pages are supported, make the exemption cover all metadata-only lint that raw pages cannot satisfy while preserving errors for generated imported pages, then align the docs and close the conflicting docs-only path.

## Reproduction Assessment

Yes. Source inspection shows current main compiles raw `sources/*.md` pages and then unconditionally emits `missing-id`, `missing-page-type`, and missing-`updatedAt` freshness lint when those pages lack frontmatter.

## Solution Assessment

No. The PR is a plausible narrow repair, but it is not yet the best fix because it leaves the missing-`updatedAt` warning in place for the same accepted raw pages and conflicts with an open docs PR that clarifies `sources/` as structured wrappers.
