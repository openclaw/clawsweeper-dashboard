## Summary

Keep open. The PR targets a real CLI UX bug, but the proposed helper re-checks only generic registry provider config and can misreport configured media-model or CLI attempts as missing provider; it also lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR adds missing-provider error messages for `infer image describe` and `infer audio transcribe` and extends `src/cli/capability-cli.test.ts` to cover missing-provider versus empty-output cases.

## Best Possible Solution

Base CLI error selection on the media-understanding runtime's resolved-entry or attempt decision state. Throw the missing-provider configuration message only when nothing was resolved or attempted, preserve the empty-output error for configured provider/model/CLI attempts, and add the required active-version changelog entry.

## Reproduction Assessment

Yes. Current main is source-confirmed to throw the path-focused empty-result errors on no text, and the PR regression is reproducible by a configured `tools.media` CLI/model entry returning empty output while the new helper still sees no generic provider config.

## Solution Assessment

No. The UX direction is right, but re-deriving configuration from generic provider auth is not the narrowest maintainable fix; the safer solution is to use or expose the runner's existing entry/attempt decision state.
