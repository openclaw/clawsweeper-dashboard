## Summary

Keep open. Current main and the latest release still preserve a /models browse path where authenticated CLI runtime alias provider rows can appear alongside canonical provider rows, and the only targeted dedupe PR is closed unmerged rather than a viable canonical fix path.

## What This Changes

The issue requests that /models browse picker entries not be duplicated under both canonical providers and CLI runtime alias providers when canonical models use agentRuntime aliases.

## Best Possible Solution

Define the shared /models display contract, then land a current-main dedupe that prunes only implicit duplicate runtime-alias rows while preserving explicit CLI-provider browsing and explicit legacy refs.

## Reproduction Assessment

Yes, by source inspection. Current buildModelsProviderData can add canonical visible catalog rows and authenticated CLI runtime alias catalog rows into the same provider map, and current tests assert those provider banners can coexist.

## Solution Assessment

Unclear until maintainers choose the display contract. The closed dedupe PR's preserve-explicit approach is plausible, but current main still encodes the sibling-provider behavior and needs a refreshed implementation if that contract changes.
