## Summary

Keep open. Current main still enforces the 2 MB local avatar file cap and still omits that limit from the relevant avatar docs, and an open closing PR already owns the narrow docs fix.

## What This Changes

Document the 2 MB limit for local workspace-relative avatar image files, or separately reconsider the runtime cap.

## Best Possible Solution

Land the existing docs PR, or replace it with an equivalent docs-only change that scopes the cap to local workspace-relative avatar files and does not imply the same `stat.size` check applies to HTTP(S) URLs or data URIs.

## Reproduction Assessment

Yes. Source inspection shows a local avatar file larger than `AVATAR_MAX_BYTES` returns `too_large`, and current main still lacks the matching docs note in the CLI and gateway agent docs.

## Solution Assessment

Yes. The narrow docs-only path is the best immediate solution because the runtime cap is existing tested behavior and the reported problem is discoverability.
