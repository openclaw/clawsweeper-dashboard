## Summary

Keep open: current main and v2026.6.8 still document avatar source forms without the local 2 MB file limit, while current source and tests enforce that limit and https://github.com/openclaw/openclaw/pull/78884 remains the open closing candidate.

## What This Changes

The issue requests documenting that local workspace-relative agent avatar image files are limited to 2 MB.

## Best Possible Solution

Land a narrow docs-only update in docs/cli/agents.md and docs/gateway/config-agents.md that names the 2 MB local-file limit and clarifies that HTTP(S) and data URI avatars are outside that local file check.

## Reproduction Assessment

Yes. Source inspection shows local avatars over AVATAR_MAX_BYTES resolve as too_large, tests cover that behavior, and current CLI/gateway docs omit the local 2 MB cap.

## Solution Assessment

Yes. A narrow docs clarification is the best fix; removing the limit would be a separate security/product change rather than the docs-gap repair requested here.
