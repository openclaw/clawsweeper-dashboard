## Summary

Keep open. Current main still enforces the 2 MB limit for local avatar files and the relevant CLI/gateway docs still omit that limit; the narrow docs fix is already owned by the open closing PR at https://github.com/openclaw/openclaw/pull/78884, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests documenting the 2 MB limit for local workspace-relative avatar image files.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/78884 or an equivalent docs-only update that scopes the cap to local workspace-relative avatar files and excludes HTTP(S) URLs and `data:` URIs.

## Reproduction Assessment

Yes. Source inspection on current main shows local avatars over `AVATAR_MAX_BYTES` resolve as `none`/`too_large`, and targeted docs inspection shows the CLI and gateway agent docs still omit the local 2 MB cap.

## Solution Assessment

Yes. A narrow docs-only fix is the best path because the limit is existing tested behavior, and changing or removing the cap would be a separate security/product decision.
