## Summary

Keep open. Current main still enforces the 2 MB local avatar file cap and returns oversized local avatars as unavailable, while the CLI and gateway agent docs still describe avatar sources without documenting that limit. The related implementation attempts in the provided context are all closed unmerged, so the docs gap remains actionable.

## What This Changes

The issue requests documenting the 2 MB avatar image limit, or alternatively reconsidering that runtime cap.

## Best Possible Solution

Add a short user-facing note beside the avatar descriptions in the CLI agent identity docs and gateway agent config docs, keeping the existing runtime cap unchanged unless maintainers separately decide to revisit the security policy.
