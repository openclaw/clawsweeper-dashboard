## Summary

Keep open. Current main still enforces the 2 MB local avatar cap and returns oversized local avatars as unavailable, while the CLI and gateway agent docs still describe avatar sources without documenting that local file-size limit. The previously linked closing PRs are closed unmerged, and a live search found no open replacement PR for this docs gap.

## What This Changes

Document the existing 2 MB size limit for local agent avatar image files in the agent identity docs.

## Best Possible Solution

Add a short user-facing note beside the avatar descriptions in `docs/cli/agents.md` and `docs/gateway/config-agents.md`, keeping the existing runtime cap unchanged unless maintainers separately decide to revisit the security policy.

## Reproduction Assessment

Yes. The source-level path is high confidence: create a local workspace avatar larger than `AVATAR_MAX_BYTES` and `resolveAgentAvatar` returns `none` with reason `too_large`; current tests already encode that behavior, and targeted docs search shows the note is absent.

## Solution Assessment

Yes. A narrow docs-only update is the best immediate fix because the cap is intentional security hardening and the user problem is discoverability, not a demonstrated need to relax runtime behavior.
