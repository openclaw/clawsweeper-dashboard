## Summary

Keep open. Current main still enforces the 2 MB local avatar cap while the relevant CLI and gateway docs describe avatar sources without documenting that limit, and the related implementation PRs in the provided context are closed unmerged.

## What This Changes

The issue requests documenting the 2 MB avatar image limit, or otherwise reconsidering that runtime limit, so oversized avatars do not fail silently for users.

## Best Possible Solution

Add a short note beside the avatar descriptions in the CLI agent identity docs and gateway agent config docs stating that local workspace avatar image files must be 2 MB or smaller, while leaving runtime behavior unchanged unless maintainers separately revisit the cap.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a workspace-local avatar file larger than `AVATAR_MAX_BYTES`; current code returns `none` with reason `too_large`, while the current docs omit that limit.

## Solution Assessment

Yes. A narrow docs-only repair is the best immediate solution because the cap is implemented and tested, and the reported user problem is discoverability rather than an unproven runtime defect.
