## Summary

Keep open. Current main parses and stores `expectsCompletionMessage:false`, but the completion lifecycle still enters the announce flow and can queue, steer, or direct-deliver a parent handoff; existing tests currently assert that behavior instead of suppressing it.

## What This Changes

Honor `sessions_spawn` with `expectsCompletionMessage:false` by preventing subagent completion announcements from steering or replacing the parent session context.

## Best Possible Solution

Implement explicit false as a no-completion-handoff path that still performs cleanup and task bookkeeping, preserves default auto-announce when omitted or true, and adds focused regression coverage for active requester queue/steer and direct fallback suppression.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is the current lifecycle plus dispatch path: with `expectsCompletionMessage:false`, completion still reaches queue/direct delivery, and existing tests assert queued/direct delivery for that flag.

## Solution Assessment

Yes, a narrow fix in the registry lifecycle or announce dispatch boundary is the maintainable direction, provided it preserves cleanup and avoids changing the default documented announce behavior.
