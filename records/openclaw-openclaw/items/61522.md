- **[P1] Avoid config-backed logging while config is loading:** `src/config/io.ts:775`
  - body: This catch now logs through `configLogger`, but on this branch file-log enablement reads logging settings by requiring config and calling `loadConfig()` again.   - body: The PR's stated goal is to make config backup IO failures visible, but `hardenBackupPermissions()` and the `maintainConfigBackups()` copy still swallow failures. If the primary backup copy fails, operators still see no signal that the backup ring is stale.
  - confidence: 0.87

  - body: `EIO` means a generic input/output error, not lack of free space. Grouping it with `ENOSPC` will send operators toward disk-space remediation and emit the wrong category for hardware, filesystem, or remote-mount failures.
  - confidence: 0.84
