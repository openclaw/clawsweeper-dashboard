Full review comments:

- **[P1] Avoid config-dependent logging during config bootstrap:** `src/config/io.ts:763`
  - body: This catch can run while config loading is still resolving its own health state. In the PR base, logging through the subsystem logger can fall back through config loading again, so a missing or corrupt health file can recurse during startup; use a bootstrap-safe logger for this path.
  - confidence: 0.86
- **[P2] Classify EIO separately from disk full:** `src/config/io-errors.ts:27`
  - body: EIO is a generic input/output failure, not a no-space condition. Mapping it to disk-full sends operators toward the wrong diagnosis and severity; let it fall through to unknown or add an explicit io-error category.
  - confidence: 0.94
