Full review comments:

- **[P1] Avoid config logger recursion during config load:** `src/config/io.ts:775`
  - body: These config-health read failures can happen while `loadConfig()` is being called to resolve logging settings on this PR's base. Logging them through `configLogger` re-enters `isFileLogLevelEnabled()` before settings are cached, which can recursively load config and stall startup; use the injected `deps.logger` or another bootstrap-safe path for config-load IO failures.
  - confidence: 0.88
- **[P2] Classify EIO separately from disk-full:** `src/config/io-errors.ts:27`
  - body: `EIO` is a generic input/output failure, not a no-space condition. Reporting it as `disk-full` at error level points operators at the wrong fix for hardware, filesystem, or remote-mount failures; let it fall through to `unknown` or add a separate category.
  - confidence: 0.9
