Full review comments:

- **[P1] Avoid config-driven logging in config bootstrap:** `src/config/io.ts:775`
  - body: This catch can run while `loadConfig()` is already being called to resolve logging settings.   - confidence: 0.88
- **[P2] Do not classify EIO as disk-full:** `src/config/io-errors.ts:27-28`
  - body: `EIO` is a generic input/output failure, not a disk-space condition. Mapping it to `disk-full` escalates it to the wrong log level and sends operators toward the wrong remediation; let it fall through or add a distinct IO-error category.
  - confidence: 0.86
