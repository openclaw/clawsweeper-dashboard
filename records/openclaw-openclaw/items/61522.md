Full review comments:

- **[P1] Keep config IO logging out of logger bootstrap:** `src/config/io.ts:763`
  - body: The new config-health read catch logs through `configLogger`, but on this branch subsystem logging checks file-log settings by calling back into `loadConfig()`. Missing or corrupt `config-health.json` is a normal first-run/recovery path, so this can re-enter config loading and recurse or thrash during startup; use an injected/lightweight logger path or avoid logging this bootstrap read failure.
  - confidence: 0.86
- **[P2] Do not classify EIO as disk-full:** `src/config/io-errors.ts:27`
  - body: `EIO` is the generic I/O error errno, while `ENOSPC` is the no-space condition. Grouping both under `disk-full` will send operators toward the wrong diagnosis and escalate unrelated filesystem failures as disk-full errors; let `EIO` fall through or add a separate category.
  - confidence: 0.9
