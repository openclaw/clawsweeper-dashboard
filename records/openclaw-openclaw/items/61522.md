- **[P1] Keep config IO logging out of logger bootstrap:** `src/config/io.ts:763`
  - body: The new config-health read catch logs through `configLogger`, but on this branch subsystem logging checks file-log settings by calling back into `loadConfig()`.   - body: `EIO` is the generic I/O error errno, while `ENOSPC` is the no-space condition. Grouping both under `disk-full` will send operators toward the wrong diagnosis and escalate unrelated filesystem failures as disk-full errors; let `EIO` fall through or add a separate category.
  - confidence: 0.9
