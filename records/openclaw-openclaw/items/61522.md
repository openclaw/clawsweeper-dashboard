  - body: `readConfigHealthStateSync()` is part of `loadConfig()` on this branch.   - body: `EIO` is reported by Node as an `i/o error`, not a no-space condition. Grouping it with `ENOSPC` logs hardware, filesystem, or remote-mount failures as `disk-full`, sending operators to the wrong remediation path.
  - confidence: 0.82
