  - body: `EIO` is a generic input/output failure, not a disk-space condition. Classifying it as `disk-full` sends operators toward the wrong remediation and escalates the message to `error` using the disk-full log level; let it fall through to `unknown`/`warn` or add a separate IO-error category.

  - body: This catch can run during early config/logging bootstrap, but `configLogger` calls subsystem logging, which consults console/file logging settings before emitting.   - confidence: 0.82
