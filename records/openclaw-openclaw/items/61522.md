- **[P1] Avoid subsystem logging during config bootstrap:** `src/config/io.ts:775`
  - body: This catch runs while loading config health state, and at this branch base `readLoggingConfig()` loads the full config module. Calling `configLogger` here can re-enter config loading through logger enablement on a common missing/corrupt health-state path, risking startup recursion or stack overflow; use a bootstrap-safe injected logger or leave this path non-subsystem until logging config is initialized.

  - body: `EIO` is a generic I/O error, while `ENOSPC` is the no-space condition. Grouping both as `disk-full` sends operators toward the wrong remediation and escalates unrelated filesystem or device failures as disk-full errors.
  - confidence: 0.92
