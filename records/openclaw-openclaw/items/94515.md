## Summary

Keep open for maintainer review: the patch is a coherent narrowed mitigation for idle Codex PreToolUse relay subprocesses, but it intentionally changes generated hook default behavior and does not prove full coverage for the linked explicit-loop/concurrency fan-out report.

## What This Changes

The PR changes Codex native hook relay gating so inactive PreToolUse events produce empty Codex hook arrays and no longer emit a no-op fallback command, while tests pin explicit `tools.loopDetection.enabled=true` as the path that keeps PreToolUse installed.

## Best Possible Solution

Land this only as a narrowed idle-relay mitigation after maintainer sign-off on the default change, and keep or create explicit tracking for loop-detection relay fan-out before treating the linked P1 as fully fixed.
