## Summary

Keep open: the PR is a focused and useful security hardening, but it still has merge-blocking behavior and migration gaps around redaction, legacy fallback visibility, and doctor migration safety.

## What This Changes

The PR changes cron webhook URL normalization to reject HTTP(S) URLs whose parsed URL contains username or password userinfo.

## Best Possible Solution

Ship the userinfo rejection together with redacted invalid-target diagnostics, explicit legacy fallback warnings, doctor validation or refusal, focused regression tests, and migration or release notes.

## Reproduction Assessment

Yes, at source level. Current main accepts credential-bearing HTTP(S) webhook URLs after only a protocol check, and the PR proof shows the patched helper returning `null` for one such URL.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch should also handle redacted logging, legacy fallback visibility, doctor migration safety, and regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: When this guard rejects an existing `delivery.mode = "webhook"` target containing userinfo, `dispatchGatewayCronFinishedNotifications` logs `params.job.delivery.to` as `deliveryTo`. That can write the credential-bearing URL to logs on every run, so redact or omit the rejected target before merging.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy `notify: true` jobs resolve `cron.webhook` through this helper, but the invalid-target warning only runs for explicit `delivery.mode = "webhook"`. After this new `null` result, credentialed legacy fallback deliveries become silent no-ops with no migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` for legacy notify jobs without validating it. With this rejection in place, doctor can convert a working legacy fallback into stored metadata the scheduler later rejects.
  - confidence: 0.84
