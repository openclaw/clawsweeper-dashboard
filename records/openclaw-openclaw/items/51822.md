## Summary

Keep open: the security hardening is useful and source-reproducible, but the current patch would route existing credential-bearing cron webhook targets into raw invalid-target logging and leaves legacy fallback/doctor migration paths unsafe or silent.

## What This Changes

Adds a username/password check to `normalizeHttpWebhookUrl` so cron webhook URL normalization rejects HTTP(S) URLs with URL userinfo.

## Best Possible Solution

Ship the shared URL-userinfo rejection together with redacted invalid-target diagnostics, explicit legacy fallback warnings, doctor migration validation/refusal, and focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main returns credential-bearing HTTP(S) URLs after only the protocol check, and the PR proof shows the patched helper rejecting one while accepting a plain HTTPS URL.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch needs the surrounding logging, legacy fallback, doctor migration, and regression-test repairs before it is the safest implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: Rejecting userinfo here makes stored `delivery.mode = "webhook"` values with credentials return `null`, and `dispatchGatewayCronFinishedNotifications` logs `params.job.delivery.to` as `deliveryTo` when normalization fails. That writes the raw credential-bearing URL to logs on every finished run, so redact or omit the rejected target before adding this rejection.
  - confidence: 0.91
- **[P2] Warn when rejecting legacy cron.webhook fallback:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy `notify: true` jobs resolve `cron.webhook` through this helper, but the invalid-target warning only runs for explicit webhook delivery. After this new `null` result, credentialed legacy fallback deliveries become silent no-ops with no migration hint.
  - confidence: 0.86
- **[P2] Validate cron.webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` for legacy notify jobs without validating it. With this rejection in place, doctor can convert a previously delivering legacy fallback into stored metadata the scheduler later rejects.
  - confidence: 0.84
