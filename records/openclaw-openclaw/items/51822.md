## Summary

Keep open: the PR is a focused security hardening change with sufficient helper-level proof, but it changes stored cron webhook delivery from accepted to fail-closed without a maintainer-confirmed upgrade policy or focused coverage for all affected cron paths.

## What This Changes

The branch makes `normalizeHttpWebhookUrl` reject parsed HTTP(S) webhook URLs that contain a username or password.

## Best Possible Solution

Land the credentialed-webhook guard only after maintainers choose the stored-config upgrade behavior and the PR covers validation, runtime delivery, and doctor migration paths.

## Reproduction Assessment

## Solution Assessment

No. The shared helper is the right central location, but unconditional fail-closed behavior is only a plausible hardening step until stored cron configs and doctor migration have an explicit upgrade policy.
