## Summary

Keep open: the security goal is valid, but the current one-helper patch introduces raw credential logging and leaves legacy fallback and doctor migration gaps that need maintainer/security-guided revision before merge.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so cron webhook URLs with URL userinfo are treated as invalid.

## Best Possible Solution

Ship userinfo rejection through the shared cron webhook normalizer together with redacted invalid-target diagnostics, explicit legacy fallback warnings, doctor validation or refusal, regression tests, and migration notes.

## Reproduction Assessment

Yes. Current main source shows the normalizer accepts HTTP(S) URLs without inspecting URL userinfo, and the inspected PR terminal screenshot shows the patched helper rejecting `user:pass@` while accepting a plain HTTPS URL.

## Solution Assessment

No. The shared normalizer is the right seam, but rejecting there alone is incomplete because the resulting invalid explicit delivery path logs raw targets and the legacy/doctor paths are not protected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact stored webhook targets before rejecting them:** `src/cron/webhook-url.ts:20`
  - body: With this new `null` path, an existing stored `delivery.mode="webhook"` URL containing userinfo reaches the invalid-target warning, which logs `params.job.delivery.to` as `deliveryTo`. Redact or omit that value before rejecting credential-bearing URLs.
  - confidence: 0.91
- **[P2] Warn when rejected legacy cron.webhook disables delivery:** `src/cron/webhook-url.ts:20`
  - body: Legacy `notify: true` jobs resolve `cron.webhook` through this helper, but the invalid-target warning only runs for explicit webhook delivery. If credentialed legacy URLs start returning `null`, finished-run callbacks stop with no operator-visible warning.
  - confidence: 0.86
- **[P2] Validate legacy webhooks before doctor migration:** `src/cron/webhook-url.ts:20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` without this normalizer. After this guard lands, doctor can migrate a previously working credentialed fallback into an invalid per-job webhook.
  - confidence: 0.84
