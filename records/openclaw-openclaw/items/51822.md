## Summary

Keep open. The shared cron webhook normalizer is the right place for this guard, but the current diff is incomplete because it can expose rejected stored credential URLs in logs, silently disable legacy notify fallback delivery, and let doctor migrate a legacy credential URL into a newly invalid per-job webhook.

## What This Changes

The PR adds a username/password check to `src/cron/webhook-url.ts` so HTTP(S) cron webhook URLs with URL userinfo return `null`.

## Best Possible Solution

Ship URL userinfo rejection through the shared cron webhook normalizer together with redacted diagnostics, explicit legacy fallback handling, doctor validation or refusal, focused regression tests, and migration notes.

## Reproduction Assessment

Yes. Source inspection shows current main accepts HTTP(S) cron webhook URLs with URL userinfo, and the inspected terminal screenshot shows the patched helper rejecting a credential-bearing URL while accepting a plain HTTPS URL.

## Solution Assessment

No. The shared helper is the right seam, but the current patch is not the best complete fix until it also handles redacted logging, legacy fallback behavior, doctor migration, and regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:20`
  - body: After this guard returns `null` for an existing credential-bearing `delivery.to`, `dispatchGatewayCronFinishedNotifications` falls into the invalid-target warning and logs `params.job.delivery.to` as `deliveryTo`. Redact or omit that value before rejecting URL userinfo so the hardening does not write the secret to logs.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:20`
  - body: Legacy `notify: true` jobs pass `cfg.cron.webhook` through the same normalizer, but the invalid-target warning only runs for `delivery.mode === "webhook"`. Rejecting a credentialed legacy webhook would turn finished-run delivery into a silent no-op, so surface a redacted operator warning or migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:20`
  - body: `openclaw doctor --fix` copies `cfg.cron.webhook` into `delivery.to` and removes `notify` without using the normalizer. Once credentialed URLs are rejected, doctor can rewrite a working legacy fallback into a per-job webhook that the scheduler skips.
  - confidence: 0.84
