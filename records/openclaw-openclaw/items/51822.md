## Summary

Keep open: the security hardening is useful, but the current patch is not merge-ready because the new rejection path interacts badly with existing raw invalid-target logging, legacy fallback behavior, and doctor migration.

## What This Changes

The PR adds a username/password guard to `src/cron/webhook-url.ts` so the shared cron webhook URL normalizer rejects HTTP(S) URLs with URL userinfo.

## Best Possible Solution

Ship userinfo rejection through the shared cron webhook normalizer together with redacted invalid-target diagnostics, explicit legacy fallback handling, doctor migration validation, focused tests, and release or migration notes.

## Reproduction Assessment

Yes. Source inspection shows current main accepts HTTP(S) URLs with userinfo because the normalizer only checks protocol and returns the trimmed value, and the PR body terminal screenshot shows the patched helper rejecting a credential URL while accepting a plain URL.

## Solution Assessment

No. The shared normalizer is the right policy point, but the patch also needs redacted diagnostics, legacy fallback handling, doctor migration validation or refusal, and regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact stored webhook targets before rejecting them:** `src/cron/webhook-url.ts:20`
  - body: With this new `null` path, an existing stored `delivery.to` containing URL userinfo reaches the invalid-target warning in `dispatchGatewayCronFinishedNotifications`, which logs the raw value as `deliveryTo`. Redact or omit that target before making credential-bearing URLs invalid.
  - confidence: 0.91
- **[P2] Warn when rejected legacy cron.webhook disables delivery:** `src/cron/webhook-url.ts:20`
  - body: Legacy `notify: true` jobs resolve the global `cron.webhook` through this helper, but the invalid-target warning only runs for explicit per-job webhook delivery. If this guard rejects a credentialed legacy URL, finished-run callbacks stop without a migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhooks before doctor migration:** `src/cron/webhook-url.ts:20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` and deletes `notify` without using this normalizer. After this guard lands, a credentialed legacy fallback can be migrated into an invalid per-job webhook that the scheduler skips.
  - confidence: 0.84
