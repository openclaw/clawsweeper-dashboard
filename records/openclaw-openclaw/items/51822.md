## Summary

The PR should stay open: rejecting URL userinfo is useful cron webhook hardening, but the current patch creates unsafe and confusing upgrade paths by making stored credential-bearing URLs invalid without redacted diagnostics or doctor migration handling.

## What This Changes

The PR adds a username/password check to the shared cron HTTP webhook URL normalizer.

## Best Possible Solution

Ship the userinfo rejection only with redacted invalid-target diagnostics, explicit legacy fallback warnings, doctor validation or refusal, focused regression tests, and release/migration notes.

## Reproduction Assessment

Yes, at source level. Current main accepts credential-bearing HTTP(S) URLs after only a protocol check, and the PR proof shows the patched helper rejecting one; I did not run an end-to-end cron job in this read-only review.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch is incomplete until rejected targets are redacted, legacy cron.webhook rejection is surfaced, doctor avoids unsafe migration, and regression coverage is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: After this guard returns null for an existing stored delivery.mode="webhook" URL with userinfo, dispatchGatewayCronFinishedNotifications logs params.job.delivery.to as deliveryTo. That can write the credential-bearing URL to logs on the new hardening path.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy notify:true jobs pass cfg.cron.webhook through this normalizer, but the invalid-target warning only covers explicit webhook delivery. A credentialed legacy webhook would stop delivering callbacks without a redacted operator warning or migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: openclaw doctor --fix copies cfg.cron.webhook into delivery.to and removes notify without using this normalizer. Once credentialed URLs are rejected, doctor can migrate a legacy fallback into per-job metadata that the scheduler later skips.
  - confidence: 0.84
