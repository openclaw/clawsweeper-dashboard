## Summary

Keep open: the guard targets a real credential-leak risk at the right shared cron webhook seam, but the branch still introduces unsafe invalid-target logging and misses legacy/doctor migration handling for the new rejection policy.

## What This Changes

The branch adds a username/password check to the shared cron webhook URL normalizer so credential-bearing HTTP(S) webhook URLs return null.

## Best Possible Solution

Reject URL userinfo through the shared cron webhook normalizer together with redacted invalid-target diagnostics, legacy fallback warnings, doctor migration validation, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows normalizeHttpWebhookUrl accepts credential-bearing http(s) URLs, and the PR proof screenshot shows the patched helper rejecting userinfo while accepting a plain webhook URL.

## Solution Assessment

No. The shared normalizer is the right seam, but merging this guard alone is not the best fix because the surrounding logging and migration paths need to be updated with the policy change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-21`
  - body: When this guard makes an existing delivery.to with URL userinfo invalid, finished-run dispatch takes the skipped-delivery branch and logs params.job.delivery.to directly. That writes the credential-bearing URL to cron logs, so redact or omit the target before this guard lands.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-21`
  - body: Legacy notify jobs resolve the global cron.webhook through this normalizer. If userinfo is rejected, the target becomes null, but the invalid-target warning only covers explicit webhook delivery, so legacy callbacks can stop without a migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-21`
  - body: openclaw doctor --fix copies cron.webhook into delivery.to and removes notify. With this normalizer rejecting userinfo, doctor can rewrite a working legacy fallback into an invalid per-job webhook, so it should validate and warn instead.
  - confidence: 0.84
