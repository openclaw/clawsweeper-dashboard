## Summary

Keep open: the guard targets a real credential-leakage issue, but on current main it would send credential-bearing stored URLs into raw logging, silently drop legacy fallback deliveries, and let doctor migrate legacy jobs into invalid webhook metadata.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so HTTP(S) cron webhook URLs with URL userinfo are rejected.

## Best Possible Solution

Ship this hardening only with redacted diagnostics, explicit legacy fallback and doctor handling, focused regression tests, and release or migration notes.

## Reproduction Assessment

Yes, at source level: current main accepts credential-bearing HTTP(S) cron webhook URLs after only a protocol check, and the PR proof shows the patched helper rejecting one. I did not establish an end-to-end cron run in this read-only review.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch needs redacted invalid-target diagnostics, legacy fallback visibility, doctor validation or refusal, and regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: When this guard rejects an existing `delivery.mode = "webhook"` URL containing userinfo, the finished-run notification path logs `params.job.delivery.to` as `deliveryTo`. That writes the credential-bearing URL to logs on each finished run, so redact or omit the rejected target before adding this null case.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy `notify: true` jobs resolve `cron.webhook` through this helper, but the invalid-target warning only runs for explicit webhook delivery. With this new rejection, credentialed legacy fallback deliveries become silent no-ops with no migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` for simple legacy notify jobs without validating it. Under this new policy, doctor can convert a legacy fallback into stored metadata the scheduler later rejects.
  - confidence: 0.84
