## Summary

Keep open: the hardening direction is useful, but the branch is not mergeable because the new rejection path can log stored credential URLs raw and can silently or destructively break legacy cron.webhook migration paths. The PR body terminal proof is sufficient for the helper-level behavior.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so cron HTTP(S) webhook URLs with URL userinfo are rejected.

## Best Possible Solution

Adopt userinfo rejection in the shared cron webhook normalizer together with redacted invalid-target diagnostics, legacy fallback warnings, doctor migration validation, regression coverage, and explicit migration/release handling.

## Reproduction Assessment

Yes. Source inspection of current main shows the helper accepts credential-bearing HTTP(S) URLs, and the inspected PR body terminal screenshot shows the patched helper rejecting one while accepting a plain URL.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch alone is not the best fix until it redacts rejected targets and handles legacy fallback and doctor migration paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-21`
  - body: When this guard makes an existing webhook delivery target containing URL userinfo invalid, `dispatchGatewayCronFinishedNotifications` logs `deliveryTo: params.job.delivery.to` raw. That can expose the credential-bearing URL in gateway logs until the warning redacts or omits the target.
  - confidence: 0.91
- **[P2] Warn on rejected legacy cron.webhook:** `src/cron/webhook-url.ts:19-21`
  - body: Legacy notify jobs run global `cron.webhook` through this helper. If the new guard rejects that legacy URL, `resolveCronWebhookTarget` returns null and no warning fires because the invalid-target warning only checks explicit webhook delivery jobs, so callbacks stop without migration guidance.
  - confidence: 0.86
- **[P2] Validate legacy webhooks before doctor migration:** `src/cron/webhook-url.ts:19-21`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` and deletes `notify` without this normalizer. With the new guard, a credentialed legacy fallback is migrated into an invalid per-job webhook that the scheduler skips; doctor should warn or refuse that migration.
  - confidence: 0.84
