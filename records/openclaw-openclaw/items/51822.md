## Summary

Keep this PR open: the credentialed-URL guard is useful security hardening, but the patch is not merge-ready because the shared cron webhook normalizer also affects stored runtime delivery and doctor migration paths without a maintainer-approved upgrade policy or focused coverage.

## What This Changes

The PR makes `normalizeHttpWebhookUrl` reject HTTP(S) webhook URLs whose parsed URL contains a username or password.

## Best Possible Solution

Land the credentialed-webhook guard only after maintainers choose the stored-config upgrade behavior and the PR covers the affected validation, runtime delivery, and doctor migration paths.
