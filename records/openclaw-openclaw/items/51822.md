## Summary

Keep open: the security hardening targets the right shared cron webhook helper and has terminal proof, but the current diff makes stored credential-bearing webhook URLs invalid before cron logging and migration paths handle them safely.

## What This Changes

The PR adds a username/password userinfo rejection to `normalizeHttpWebhookUrl` for cron webhook URLs.

## Best Possible Solution

Ship userinfo rejection through the shared cron webhook normalizer together with redacted diagnostics, explicit legacy fallback warnings, doctor migration validation or refusal, focused tests, and release/migration notes.

## Reproduction Assessment

Yes. Source inspection on current main shows `normalizeHttpWebhookUrl` only parses and protocol-checks HTTP(S) URLs before returning the original string, and the PR body terminal proof shows the patched helper rejecting a credential-bearing URL while accepting a plain URL.

## Solution Assessment

No. The shared helper is the right seam, but this patch needs redacted invalid-target diagnostics, legacy fallback handling, doctor migration validation or refusal, and regression coverage before the policy is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact stored webhook targets before rejecting them:** `src/cron/webhook-url.ts:20`
  - body: When this guard rejects an existing stored `delivery.to` with URL userinfo, `dispatchGatewayCronFinishedNotifications` logs the raw `params.job.delivery.to` as `deliveryTo` in `src/gateway/server-cron-notifications.ts:223`. That can write the credential-bearing URL to gateway logs on every finished run, so redact or omit the invalid target before shipping the rejection.
  - confidence: 0.91
- **[P2] Warn when rejected legacy cron.webhook disables delivery:** `src/cron/webhook-url.ts:20`
  - body: Legacy `notify: true` jobs run the global `cron.webhook` through this helper, but the invalid-target warning only covers explicit webhook delivery jobs. If the new guard rejects a credentialed legacy URL, finished-run callbacks stop without a migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhooks before doctor migration:** `src/cron/webhook-url.ts:20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` and deletes `notify` without using this normalizer. With this guard, a credentialed legacy fallback can be migrated into an invalid per-job webhook that the scheduler skips.
  - confidence: 0.84
