## Summary

Keep open: the cron URL-userinfo guard is useful security hardening, but current main has not implemented it and the PR is incomplete because the new rejection path can expose raw stored webhook targets, silently drops legacy fallback delivery, and conflicts with doctor migration behavior.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so cron HTTP(S) webhook URLs containing URL userinfo return `null`.

## Best Possible Solution

Ship URL-userinfo rejection through the shared normalizer together with redacted invalid-target diagnostics, explicit legacy fallback handling, doctor validation or refusal, focused regression tests, and migration notes.

## Reproduction Assessment

Yes, at source level. Current main returns credential-bearing HTTP(S) URLs after only a protocol check, and the PR proof shows the patched helper rejecting one; I did not run an end-to-end cron job in this read-only review.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch is not complete until rejected targets are redacted, legacy `cron.webhook` rejection is surfaced, doctor avoids unsafe migration, and regression coverage is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: After this guard returns `null` for an existing stored `delivery.mode="webhook"` URL with userinfo, `dispatchGatewayCronFinishedNotifications` logs `params.job.delivery.to` as `deliveryTo`. That value can contain the credential-bearing URL, so the hardening path can write the secret it is trying to protect.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy `notify: true` jobs pass `cfg.cron.webhook` through this normalizer, but the invalid-target warning only covers explicit webhook delivery. A credentialed legacy webhook would stop delivering finished-run callbacks without a redacted operator warning or migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: `openclaw doctor --fix` copies `cfg.cron.webhook` into `delivery.to` and removes `notify` without using this normalizer. Once credentialed URLs are rejected, doctor can migrate a legacy fallback into per-job webhook metadata that the scheduler later skips.
  - confidence: 0.84
