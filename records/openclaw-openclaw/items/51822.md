## Summary

Keep open: this is plausible security hardening at the right helper, but the branch is not mergeable as-is because the new rejection path can log stored credential URLs raw and can silently break legacy cron.webhook fallback/doctor migration.

## What This Changes

The PR changes the cron HTTP webhook URL normalizer to reject HTTP(S) URLs whose parsed username or password fields are set.

## Best Possible Solution

Adopt userinfo rejection in the shared cron webhook normalizer together with redacted invalid-target diagnostics, legacy fallback warnings, doctor migration validation, regression coverage, and explicit migration/release handling.

## Reproduction Assessment

Yes. Source inspection of current main shows normalizeHttpWebhookUrl only rejects non-http(s) protocols and otherwise returns the trimmed URL, while the PR body terminal screenshot shows the patched helper rejecting a userinfo-bearing URL.

## Solution Assessment

No. The shared normalizer is the right seam, but the patch needs redacted diagnostics, legacy fallback handling, doctor migration validation, tests, and maintainer migration policy before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-21`
  - body: When this guard makes an existing delivery.to with URL userinfo invalid, dispatchGatewayCronFinishedNotifications logs params.job.delivery.to raw in the invalid webhook warning. That can write the credential-bearing URL to logs on every run, so redact or omit the target before landing the guard.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-21`
  - body: Legacy notify jobs resolve global cron.webhook through this helper. With credential-bearing legacy URLs rejected, resolveCronWebhookTarget returns null, but the invalid-target warning only covers explicit webhook delivery jobs, so callbacks can stop with no migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-21`
  - body: openclaw doctor --fix copies cron.webhook into delivery.to and deletes notify without using this normalizer. After this guard, doctor can migrate a legacy fallback into an invalid per-job webhook; validate or warn instead.
  - confidence: 0.84
