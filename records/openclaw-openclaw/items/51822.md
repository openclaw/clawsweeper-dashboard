## Summary

Keep open: the PR is useful security hardening, but the current patch rejects credential-bearing cron webhook URLs without handling the stored-value logging, legacy fallback, and doctor migration paths that become unsafe or silent after the policy change.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so parsed HTTP(S) webhook URLs with URL userinfo are rejected.

## Best Possible Solution

Ship userinfo rejection through the shared cron webhook normalizer together with redacted invalid-target diagnostics, explicit legacy fallback warnings, doctor validation or refusal, focused regression tests, and migration notes.

## Reproduction Assessment

Yes. Source inspection on current main shows the normalizer accepts credential-bearing HTTP(S) webhook URLs, and the Node URL probe confirms the parser fields the PR relies on; the logging and doctor blockers are also source-reproducible from current call paths.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch is incomplete until rejected stored targets are redacted, legacy fallback rejection is surfaced, doctor avoids unsafe migration, and regression coverage is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-20`
  - body: After this guard returns `null` for an existing stored `delivery.mode="webhook"` URL with userinfo, `dispatchGatewayCronFinishedNotifications` logs `params.job.delivery.to` as `deliveryTo`. That value can contain the credential-bearing URL, so the hardening path can write the secret it is trying to protect.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook is rejected:** `src/cron/webhook-url.ts:19-20`
  - body: Legacy `notify: true` jobs pass `cfg.cron.webhook` through this normalizer, but the invalid-target warning only covers explicit webhook delivery. A credentialed legacy webhook would stop delivering finished-run callbacks without a redacted operator warning or migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-20`
  - body: `openclaw doctor --fix` copies `cfg.cron.webhook` into `delivery.to` and removes `notify` without using this normalizer. Once credentialed URLs are rejected, doctor can migrate a legacy fallback into per-job webhook metadata that the scheduler later skips.
  - confidence: 0.84
