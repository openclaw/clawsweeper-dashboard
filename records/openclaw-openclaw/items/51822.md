## Summary

Keep open: the hardening is useful and uses the right shared normalizer, but the current patch still leaves credential-leak and migration regressions for stored and legacy cron webhooks.

## What This Changes

The PR changes cron webhook URL normalization so HTTP(S) URLs with URL userinfo are rejected.

## Best Possible Solution

Reject URL userinfo through the shared normalizer, redact and diagnose all invalid cron webhook paths, guard doctor legacy migration, and add regression tests plus release notes.

## Reproduction Assessment

Yes. Source inspection of current main shows credential-bearing HTTP(S) cron webhook URLs are accepted today, and the PR screenshot shows the proposed helper rejection; the remaining blockers are also source-visible in runtime logging and doctor migration paths.

## Solution Assessment

No. The shared normalizer is the right seam, but this patch is incomplete without redaction, legacy fallback diagnostics, doctor safeguards, focused tests, and a changelog note for the breaking security hardening.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:20-21`
  - body: When this guard makes an existing credential-bearing delivery.to invalid, finished-run dispatch logs params.job.delivery.to as deliveryTo without redaction. That exposes the same raw userinfo URL on every run, so the invalid-target path needs to redact or omit the target before this can merge.
  - confidence: 0.91
- **[P2] Warn when legacy cron.webhook becomes invalid:** `src/cron/webhook-url.ts:20-21`
  - body: Legacy notify jobs resolve the global cron.webhook through this normalizer. With this guard, a credentialed legacy URL becomes null, but the invalid-target warning only covers explicit delivery webhooks, so those callbacks silently stop instead of telling users how to migrate.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:20-21`
  - body: Doctor still migrates legacy notify fallback jobs by copying cron.webhook into delivery.to and deleting notify. If that global webhook contains userinfo, this patch turns the repaired job into an invalid per-job webhook that the scheduler skips.
  - confidence: 0.84
