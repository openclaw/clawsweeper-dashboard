## Summary

## What This Changes

The PR adds a username/password check to the shared cron HTTP(S) webhook URL normalizer so credential-bearing cron webhook URLs are rejected.

## Best Possible Solution

Land the credentialed-webhook guard after maintainers choose the upgrade behavior for existing stored cron configs and the PR covers validation, runtime delivery, and doctor migration paths.

## Solution Assessment

No. The shared helper is a plausible central guard location, but unconditional fail-closed behavior is not the best merge shape until stored webhook configs and doctor migration have an explicit upgrade policy.
