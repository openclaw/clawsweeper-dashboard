## Summary

Keep this PR open: the credential-URL hardening is plausible, but the shared cron webhook helper also controls stored runtime delivery and doctor legacy-notify migration, so merge needs an explicit upgrade policy plus real cron-path proof before it can be treated as safe.

## What This Changes

The branch adds a username/password guard to `src/cron/webhook-url.ts` so `normalizeHttpWebhookUrl` returns `null` for HTTP(S) URLs with parsed URL userinfo.

## Best Possible Solution
