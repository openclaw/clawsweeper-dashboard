## Summary

Keep open: this is a valid, narrow security hardening PR, but the current one-line rejection path still leaves raw credential logging and legacy migration regressions that should block merge.

## What This Changes

The PR changes cron webhook URL normalization so parsed URLs with a username or password are rejected.

## Best Possible Solution

Reject webhook URL userinfo through the shared normalizer while redacting rejected targets, surfacing invalid legacy fallback diagnostics, guarding doctor migration, adding focused regression tests, and documenting the breaking hardening.

## Reproduction Assessment

Yes. Source inspection shows current main accepts credential-bearing http(s) webhook URLs, and the PR screenshot shows the proposed helper returning null for that input; the remaining blockers are source-reproducible runtime and migration side effects.

## Solution Assessment

No. The shared normalizer is the right seam, but the PR is incomplete without redaction, legacy diagnostics, doctor safeguards, tests, and release notes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact rejected stored webhook targets:** `src/cron/webhook-url.ts:19-21`
  - body: When this guard makes an existing credential-bearing delivery.to invalid, finished-run notification dispatch falls into the skipped-delivery warning and logs params.job.delivery.to as deliveryTo. That writes the raw userinfo URL to logs on every run, so the invalid-target path needs to redact or omit the target before this can merge.
  - confidence: 0.91
- **[P2] Warn on invalid legacy cron.webhook fallback:** `src/cron/webhook-url.ts:19-21`
  - body: Legacy notify jobs resolve the global cron.webhook through this normalizer. With this guard, a credentialed legacy webhook becomes null, but the invalid-target warning only covers explicit webhook delivery jobs, so those callbacks stop with no migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhook before doctor migration:** `src/cron/webhook-url.ts:19-21`
  - body: Doctor still migrates notify fallback jobs by copying cron.webhook into delivery.to and deleting notify. If the global webhook contains userinfo, this PR turns the migrated job into an invalid per-job webhook that the scheduler skips.
  - confidence: 0.84
