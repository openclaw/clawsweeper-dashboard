## Summary

Keep open. The shared cron webhook URL normalizer is the right seam and the contributor provided terminal proof for the helper behavior, but the patch is not safe to merge because the new rejection path can expose stored credential-bearing URLs in logs and can break legacy cron webhook migration paths without clear operator guidance.

## What This Changes

The PR adds a username/password check to `normalizeHttpWebhookUrl` so cron HTTP(S) webhook URLs with URL userinfo return `null`.

## Best Possible Solution

Ship userinfo rejection through the shared cron webhook normalizer together with redacted diagnostics, explicit legacy fallback warnings, doctor migration validation or refusal, focused tests, and release or migration notes.

## Reproduction Assessment

Yes. Source inspection on current main shows the normalizer accepts HTTP(S) URLs with userinfo, and the PR body terminal screenshot shows the patched helper rejecting a credential URL while accepting a plain URL.

## Solution Assessment

No. The shared helper is the right place for the policy, but this patch needs redacted invalid-target diagnostics, legacy fallback handling, doctor migration validation or refusal, and regression coverage before the change is safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact stored webhook targets before rejecting them:** `src/cron/webhook-url.ts:20`
  - body: With this new `null` path, an existing stored `delivery.to` containing URL userinfo reaches the invalid-target warning in `dispatchGatewayCronFinishedNotifications`, which logs the raw value as `deliveryTo`. Redact or omit that target before making credential-bearing URLs invalid.
  - confidence: 0.91
- **[P2] Warn when rejected legacy cron.webhook disables delivery:** `src/cron/webhook-url.ts:20`
  - body: Legacy `notify: true` jobs resolve the global `cron.webhook` through this helper, but the invalid-target warning only runs for explicit `delivery.mode === "webhook"` jobs. If this guard rejects a credentialed legacy URL, finished-run callbacks stop without a migration hint.
  - confidence: 0.86
- **[P2] Validate legacy webhooks before doctor migration:** `src/cron/webhook-url.ts:20`
  - body: `openclaw doctor --fix` copies `cron.webhook` into `delivery.to` and deletes `notify` without using this normalizer. After this guard lands, a credentialed legacy fallback can be migrated into an invalid per-job webhook that the scheduler skips.
  - confidence: 0.84
