## Summary

Keep open. Current main and the latest release expose task-flow observer events internally plus plugin task-flow read/mutation APIs, but no plugin-facing task-flow lifecycle hook or subscription; the related lifecycle-hooks issue is broader and does not cover this narrower task-flow contract.

## What This Changes

The issue requests plugin-observable lifecycle events for task-flow creation, status transitions, and deletion.

## Best Possible Solution

Define a narrow plugin SDK task-flow lifecycle subscription contract with sanitized immutable payloads, focused tests for created/transition/deleted delivery, docs, and an optional diagnostics bridge only after the API shape is approved.

## Reproduction Assessment

Yes for the capability gap by source inspection: task-flow writes emit only internal observer events, while plugin task-flow APIs expose read/mutation methods and no lifecycle subscription. This is a missing feature rather than a failing existing behavior path.

## Solution Assessment

Unclear. The direction fits the plugin API roadmap, but the exact `task:flow:*` hook shape needs maintainer approval so OpenClaw does not create a parallel or overbroad observability seam.
