## Summary

Keep open. Current main and v2026.6.6 still lack plugin-visible task-flow lifecycle hooks; https://github.com/openclaw/openclaw/pull/87377 is the open closing implementation candidate, but it remains unmerged and needs maintainer API/security/proof review before this issue can close.

## What This Changes

Request to expose task-flow lifecycle changes as plugin-observable hook events, with optional diagnostics/exporter integration and metadata tagging.

## Best Possible Solution

Review the open implementation candidate, narrow the first landing to a stable sanitized task-flow lifecycle hook contract, and split tags or diagnostics if those need separate API decisions.

## Reproduction Assessment

Yes, by source inspection: task-flow mutations emit only internal observer events, while plugin APIs expose task-flow CRUD and mutation helpers without lifecycle subscription. This is a feature gap rather than a broken existing behavior path.

## Solution Assessment

Unclear. A plugin-facing hook or typed runtime event seam fits the owner boundary, but maintainers still need to choose the stable payload, privacy boundary, delivery semantics, and whether diagnostics/tags belong in the same change.
