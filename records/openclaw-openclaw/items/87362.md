## Summary

Keep open. Current main and v2026.6.9 still do not expose task-flow lifecycle events through plugin hooks, and the open linked PR remains the candidate path for maintainer API, privacy, upgrade, and proof review.

## What This Changes

The issue requests plugin-observable task-flow lifecycle hook events for created, status-transition, and deleted flow records, with optional diagnostics integration.

## Best Possible Solution

Review the linked PR as the canonical candidate, land a maintainer-approved task-flow hook contract with sanitized payloads, and split tags or diagnostics if those surfaces need separate decisions.

## Reproduction Assessment

Yes, by source inspection: task-flow writes emit internal observer events, the hook event union has no task event family, and the plugin task-flow runtime has no listener or subscription API. This is a feature gap rather than broken shipped behavior.

## Solution Assessment

Unclear. A plugin-facing task-flow hook seam fits the project direction, but maintainers still need to choose the stable payload, privacy boundary, delivery semantics, and whether tags or diagnostics belong in the same change.
