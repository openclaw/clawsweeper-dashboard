## Summary

Keep open. Current main and the latest release still expose task-flow mutations only through internal registry observers plus imperative plugin task-flow APIs, while the open linked implementation PR is the right place for maintainer review before this issue can close.

## What This Changes

Request plugin-observable task-flow lifecycle events for created, status-transition, and deleted flows, with optional diagnostics and tagging support.

## Best Possible Solution

Settle the narrow plugin-facing lifecycle event contract, then review or replace https://github.com/openclaw/openclaw/pull/87377 with a focused implementation that keeps diagnostics and tags split if maintainers want smaller steps.

## Reproduction Assessment

Yes for the capability gap by source inspection: task-flow mutations emit internal observer events and plugin APIs expose only read/mutation methods, not lifecycle subscriptions. This is a missing feature rather than a broken existing behavior path.

## Solution Assessment

Unclear. Emitting task-flow lifecycle hooks is a plausible direction, but the public payload shape and whether diagnostics/tags belong in the same change need maintainer API/product review.
