## Summary

Keep open: current main and the latest release still route approved `/hooks/agent` session keys into an isolated cron run with `forceNew: true`, and the same-author closing PR remains open and unmerged.

## What This Changes

Webhook agent runs with an allowed stable `sessionKey` should retain conversation context instead of forcing a new isolated cron session on every call.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70895 or an equivalent narrow fix; if maintainers prefer explicit opt-in persistence, coordinate it with https://github.com/openclaw/openclaw/pull/75918.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: repeated approved `/hooks/agent` calls with the same `sessionKey` enter `dispatchAgentHook`, become `sessionTarget: "isolated"`, and hit `resolveCronSession` with `forceNew: true`.

## Solution Assessment

Yes for the narrow bug boundary: the fix should let approved webhook-sourced isolated runs reuse fresh stored sessions while ordinary isolated cron runs keep fresh-session behavior. The broader explicit `sessionMode` design may be safer if maintainers want persistence to be an opt-in policy rather than automatic for hook-sourced runs.
