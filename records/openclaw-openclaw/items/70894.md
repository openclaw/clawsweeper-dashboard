## Summary

Keep open: current main and v2026.5.12 still route approved stable `/hooks/agent` session keys into isolated cron runs with `forceNew: true`, and the same-author closing PR remains open and unmerged.

## What This Changes

Webhook-triggered agent runs should preserve context across repeated approved calls with the same stable `sessionKey` instead of always starting a fresh session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70895 or the explicit https://github.com/openclaw/openclaw/pull/75918 design, preserving prefix policy checks and ordinary isolated cron fresh-session behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: repeated approved `/hooks/agent` calls with the same `sessionKey` enter `dispatchAgentHook`, become `sessionTarget: "isolated"`, and hit `resolveCronSession` with `forceNew: true`.

## Solution Assessment

Unclear as the final product shape. The narrow repair is to let trusted hook-sourced isolated runs reuse fresh stored sessions, but the explicit `sessionMode` alternative may be safer if maintainers want persistence to require opt-in policy.
