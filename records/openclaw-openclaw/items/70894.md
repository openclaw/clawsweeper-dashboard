## Summary

Keep open: current main still routes approved webhook session keys into an isolated cron run that passes `forceNew: true`, so session reuse is bypassed; the linked closing PR remains open at https://github.com/openclaw/openclaw/pull/70895.

## What This Changes

Webhook-triggered agent runs should reuse an approved stable `sessionKey` instead of starting a fresh isolated session on every call.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70895, or an equivalent narrow fix, so approved webhook-sourced isolated runs can reuse fresh sessions while ordinary isolated cron runs keep fresh-session semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: repeated approved `/hooks/agent` calls with the same `sessionKey` flow into an isolated cron job, and current main passes `forceNew: true` before `resolveCronSession` can reuse the stored session.

## Solution Assessment

Yes. The narrow fix direction is to make the `forceNew` decision account for hook/webhook provenance without changing ordinary scheduled isolated cron behavior or loosening hook session-key policy.
