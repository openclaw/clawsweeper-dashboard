## Summary

Keep open: current main still routes approved stable-key `/hooks/agent` calls through an isolated cron run that passes `forceNew: true`, and an open same-author PR already references this issue with closing syntax. The related `sessionTarget: "current"` transcript-rotation bug was fixed by https://github.com/openclaw/openclaw/pull/82200, but that fix does not remove the webhook-isolated `forceNew` path reported here.

## What This Changes

Webhook-triggered agent runs with an approved stable `sessionKey` should preserve conversation history instead of always creating a fresh isolated session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70895 or the broader explicit `sessionMode` path in https://github.com/openclaw/openclaw/pull/75918, preserving default isolation and hook session-key policy checks.

## Reproduction Assessment

Yes, source-level. Repeated approved `/hooks/agent` calls with the same `sessionKey` enter `dispatchAgentHook`, become `sessionTarget: "isolated"`, and reach `resolveCronSession` with `forceNew: true`, bypassing the existing reuse branch.

## Solution Assessment

Unclear as the final product shape. The linked automatic-reuse PR is narrow, while the explicit `sessionMode` design is safer if maintainers want persistent hook sessions to require opt-in policy.
