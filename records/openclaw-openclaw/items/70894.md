## Summary

Keep open: current main still routes approved stable-key `/hooks/agent` calls into an isolated cron run that forces a fresh session, and an open same-author PR still explicitly closes this issue but is unmerged.

## What This Changes

Webhook agent runs with the same allowed `sessionKey` should preserve conversation history instead of always starting a fresh session.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/70895, or choose the explicit `sessionMode` approach in https://github.com/openclaw/openclaw/pull/75918 while preserving default isolation and session-key prefix gates.

## Reproduction Assessment

Yes, by source inspection. Approved `/hooks/agent` calls with the same stable `sessionKey` become isolated cron jobs, and current main passes `forceNew: true` before the existing reuse branch can run.

## Solution Assessment

Unclear as the final product shape. The narrow automatic-reuse PR may fix the regression, but the explicit `sessionMode` PR is safer if maintainers want persistent hook sessions to require opt-in policy.
