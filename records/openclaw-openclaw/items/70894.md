## Summary

Keep open: current main and v2026.5.12 still route approved `/hooks/agent` session keys into isolated cron runs with `forceNew: true`, and the same-author closing PR remains open and unmerged.

## What This Changes

The issue requests that approved webhook agent calls with the same `sessionKey` reuse existing agent session context instead of always starting a fresh session.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/70895` or the explicit `sessionMode` alternative while preserving prefix policy checks and ordinary isolated cron fresh-session behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: repeated approved `/hooks/agent` calls with the same `sessionKey` enter `dispatchAgentHook`, become `sessionTarget: "isolated"`, and hit `resolveCronSession` with `forceNew: true`.

## Solution Assessment

Unclear as the final product shape. The narrow fix is to let trusted hook-sourced isolated runs reuse fresh stored sessions, but `https://github.com/openclaw/openclaw/pull/75918` may be safer if maintainers want persistence to require explicit opt-in policy.
