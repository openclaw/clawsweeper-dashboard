## Summary

Keep open: current main still routes approved stable-key `/hooks/agent` calls through an isolated cron run that passes `forceNew: true`, and an open closing PR already references this issue.

## What This Changes

The issue requests that webhook-triggered agent runs preserve conversation context across repeated calls with the same approved `sessionKey`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70895 or the explicit `sessionMode` alternative while preserving request-session-key policy checks and ordinary isolated cron fresh-session behavior.

## Reproduction Assessment

Yes, source-level. Repeated approved `/hooks/agent` calls with the same `sessionKey` enter `dispatchAgentHook`, become `sessionTarget: "isolated"`, and reach `resolveCronSession` with `forceNew: true`.

## Solution Assessment

Unclear as the final product shape. The narrow fix is to let trusted hook-sourced isolated runs reuse fresh stored sessions, but the explicit `sessionMode` design may be safer if maintainers want persistence to require opt-in policy.
