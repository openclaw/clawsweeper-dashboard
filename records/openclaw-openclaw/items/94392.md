## Summary

Keep open, but this patch is not merge-ready: it reduces the oversized structured details payload while still putting the reported oversized config snapshot into top-level text content, which the same middleware also caps at 100KB.

## What This Changes

The PR changes the agent gateway tool's config.get result wrapper so structured details become { ok: true } while the full config JSON remains in the text content block.

## Best Possible Solution

Land a fix that keeps both config.get content and details within middleware bounds, either by repairing this PR or by choosing a related safe canonical path once it has proof and mergeability.

## Reproduction Assessment

Yes from source: current main copies the full config snapshot into both details and content, and the middleware caps both details and text content at 100KB. The PR fixes only one of those two oversized surfaces.

## Solution Assessment

No. Reducing details is necessary but not sufficient because the full JSON text remains oversized; the safer fix must also bound or scope the content payload and add regression coverage.
