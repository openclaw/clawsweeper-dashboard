## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats under `visibleReplies: "message_tool"`, and open PR #77325 is the linked implementation candidate that should be reviewed or landed before closing.

## What This Changes

The issue asks for text-prefix slash command replies in group chats to remain visibly delivered even when group visible replies require the message tool.

## Best Possible Solution

Review and land #77325 or an equivalent shared command/system-reply bypass with regression tests, while keeping ordinary model final replies message-tool-only in groups.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Matrix group `/new` becomes a channel text command, resolves to `message_tool_only`, returns a canned reset reply, and then final delivery is skipped under `suppressDelivery`.

## Solution Assessment

Yes. The narrow maintainable fix is a shared command/system-reply visibility bypass with tests, not a Matrix-specific workaround or another config option.
