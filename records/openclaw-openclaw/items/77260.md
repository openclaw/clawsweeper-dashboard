## Summary

Keep open: current main still suppresses visible delivery for text-prefix command replies in group/channel chats under `messages.groupChat.visibleReplies: "message_tool"`, and an open fix PR already references this issue with closing syntax. The Slack, Matrix, and Telegram comments are consistent with the same shared auto-reply delivery policy rather than separate channel-only failures.

## What This Changes

Deliver text-prefix slash command and system replies visibly in group chats even when `messages.groupChat.visibleReplies` is `message_tool`.

## Best Possible Solution

Review and land PR #77325, or an equivalent shared auto-reply change, so text slash command canned replies bypass message-tool-only suppression while ordinary group model replies remain governed by the configured visibility policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Matrix sends group `/new` with `CommandSource: "text"`, current policy resolves `message_tool_only`, `/new` returns a canned confirmation, and final reply dispatch is skipped while `suppressDelivery` is true.

## Solution Assessment

Yes. The best fix shape is a shared command/system-reply bypass in the auto-reply visibility path, not a Matrix-specific workaround or a new config option; the already-open PR is the right item to review for that change.
