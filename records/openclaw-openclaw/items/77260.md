## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats under `visibleReplies: "message_tool"`, and open PR #77325 explicitly targets this issue with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for slash/text command confirmation replies in group chats to remain visible even when group visible replies are configured for message-tool delivery.

## Best Possible Solution

Review and land a shared auto-reply delivery fix that lets text-prefix command/system replies bypass message-tool-only suppression while preserving tool-only behavior for ordinary group assistant replies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Matrix group `/new` enters as `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned reset/new reply, and dispatch skips final delivery while `suppressDelivery` is true.

## Solution Assessment

Yes. The best direction is the narrow shared fix represented by the linked PR, not a Matrix-only workaround or a new configuration knob.
