## What This Changes

Ensure text-prefix slash command/system replies remain visible in group/channel chats when `messages.groupChat.visibleReplies` is `message_tool`, while normal model replies stay tool-only.

## Best Possible Solution

Land or replace #77325 with a shared auto-reply visibility bypass and regression coverage, then let the closing reference resolve this issue.

## Reproduction Assessment

Yes. Current-main source inspection shows a Matrix group `/new` enters with `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned command reply, and then dispatch suppresses final delivery.

## Solution Assessment

Yes. The narrow maintainable fix is a shared command/system-reply visibility bypass in auto-reply delivery rather than a Matrix- or Slack-specific workaround or a new config option.
