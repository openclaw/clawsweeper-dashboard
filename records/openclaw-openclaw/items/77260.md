## Summary

Keep open: current main still suppresses text-prefix slash-command replies in group/channel `message_tool` mode, and open PR #77325 is the implementation candidate but has not merged.

## What This Changes

Text-prefix slash command replies in group/channel chats should remain visibly delivered when `messages.groupChat.visibleReplies` is set to `message_tool`.

## Best Possible Solution

Review and land #77325 or an equivalent shared auto-reply fix that makes text-prefix command/system replies visible while preserving tool-only suppression for normal model replies, with docs adjusted if the command bypass contract changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Matrix group `/new` enters as `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned command reply, and dispatch skips final delivery while `suppressDelivery` is true.

## Solution Assessment

Yes. The narrow maintainable fix is a shared command/system-reply visibility bypass in auto-reply delivery, not a Matrix-specific workaround or a new config option; the open PR is the right review target.
