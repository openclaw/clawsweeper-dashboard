## Summary

Keep open: current main still suppresses visible delivery for text-prefix command replies in group/channel chats under `message_tool`, and linked open PR #77325 is the active fix path.

## What This Changes

Text-prefix slash command/system replies in group chats should remain visible when `messages.groupChat.visibleReplies` is set to `message_tool`.

## Best Possible Solution

Review and land PR #77325 or an equivalent shared fix so command/system replies bypass tool-only suppression while normal model final replies remain governed by `visibleReplies`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Matrix group `/new` enters with `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned reset reply, and then dispatch skips visible delivery while `suppressDelivery` is true.

## Solution Assessment

Yes. The narrowest maintainable fix is shared command/system reply visibility handling in the auto-reply delivery policy, not a Matrix-specific workaround or a new config option.
