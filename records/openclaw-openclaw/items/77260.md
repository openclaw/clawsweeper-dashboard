## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats under `visibleReplies: "message_tool"`, and the provided timeline shows an open PR already targets the focused fix.

## What This Changes

Text slash command replies in group/channel chats should remain visibly delivered when `messages.groupChat.visibleReplies` is set to `message_tool`.

## Best Possible Solution

Land a shared auto-reply policy fix that lets text-prefix command/system replies bypass `message_tool_only` suppression while ordinary group assistant final replies remain governed by the configured visible-reply mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Matrix group `/new` builds `ChatType: "channel"`, `CommandSource: "text"`, and a canned command reply, then the shared policy resolves `message_tool_only` and suppresses final delivery.

## Solution Assessment

Yes. The narrow maintainable fix is a shared command/system-reply bypass in the auto-reply visibility policy, not a Matrix-specific workaround or a new config option; the open linked PR is the current implementation candidate.
