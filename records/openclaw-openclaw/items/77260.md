## Repair Work Prompt

Fix the reported regression where group/channel text slash command replies are silently suppressed when `messages.groupChat.visibleReplies` is `"message_tool"`. In current main, Matrix and other text-command turns use `CommandSource: "text"`, resolve to `sourceReplyDeliveryMode: "message_tool_only"`, and returned command replies such as `/new` are skipped by `suppressDelivery`. Implement a generic command/system-reply bypass in the shared auto-reply delivery path so pre-canned command replies are delivered visibly, while ordinary model final replies remain message-tool-only. Do not add a new config option and do not make the fix Matrix-specific. Add regression coverage for text command replies in group/channel tool-only mode plus guard coverage that normal model final replies are still suppressed and native command behavior remains visible. Add a user-facing changelog entry if the repo's active changelog requires one for this fix.
