## Summary

Keep open: current main still source-reproduces the dropped text slash-command reply path, and open PR #77325 already tracks the narrow fix with closing syntax. The added Slack, Matrix, Telegram, and workaround comments support the same shared `message_tool_only` delivery failure rather than a stale or channel-local report.

## What This Changes

The issue asks OpenClaw to deliver canned text slash-command replies in group/channel chats even when `messages.groupChat.visibleReplies` is `message_tool`.

## Best Possible Solution

Review and land #77325 or an equivalent shared command/system-reply bypass with regression coverage, then let the issue close after merge.

## Reproduction Assessment

Yes. A Matrix group `/new` with `messages.groupChat.visibleReplies: "message_tool"` enters as `ChatType: "channel"` and `CommandSource: "text"`, resolves to `message_tool_only`, produces a plain canned command reply, and is skipped by final delivery suppression.

## Solution Assessment

Yes for the command-reply scope. A shared bypass for validated text slash-command/system replies is narrower and more maintainable than a Matrix-specific workaround or a new config option; broader assistant no-tool fallback behavior should stay in related follow-ups such as #77320 and #66459.
