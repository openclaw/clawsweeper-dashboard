## Summary

Keep open: current `main` still treats text slash commands in group/channel chats as `message_tool_only`, so canned command replies can be produced but skipped by the shared final-reply dispatch path.

## What This Changes

The issue asks for text slash command replies in group chats to remain visible when `messages.groupChat.visibleReplies` is set to `message_tool`.

## Best Possible Solution

Add a shared command/system-reply visibility bypass in auto-reply dispatch, with regression coverage proving text command replies are visible while ordinary model final replies stay message-tool-only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Matrix group `/new` enters as `ChatType: "channel"` and `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned command reply, and then final delivery is skipped under `suppressDelivery`.

## Solution Assessment

Yes. The narrowest maintainable fix is shared command/system reply visibility handling in auto-reply dispatch, not a Matrix-only workaround or a new config option.
