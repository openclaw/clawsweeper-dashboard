## Summary

Keep open: current main still source-reproduces the text-command reply drop, and #77325 is an open implementation PR that explicitly fixes this issue, so the issue should not be closed before that PR or a replacement lands.

## What This Changes

Text-prefixed slash command replies in group chats should remain visibly delivered when `messages.groupChat.visibleReplies` is `"message_tool"`.

## Best Possible Solution

Land #77325 or an equivalent shared auto-reply policy fix that makes command/system replies visible while preserving tool-only delivery for ordinary group chat prose.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Matrix group `/new` enters with `CommandSource: "text"`, resolves to `message_tool_only`, produces a canned reply, and dispatch skips final delivery while `suppressDelivery` is true.

## Solution Assessment

Yes. The narrow maintainable solution is a shared command/system-reply bypass in the auto-reply delivery policy, not a Matrix-only workaround or a new config option.
