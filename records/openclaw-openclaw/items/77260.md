## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats under `message_tool`, and open PR #77325 is already the active implementation candidate with closing syntax for this issue.

## What This Changes

Text-prefix slash command replies in group chats should remain visible when `messages.groupChat.visibleReplies` is `message_tool`.

## Best Possible Solution

Land a narrow shared command/system-reply visibility bypass for text-prefix commands, then leave broader non-command message-tool fallback behavior to the existing related reports.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Matrix group `/new` enters as `ChatType: "channel"` with `CommandSource: "text"`, resolves to `message_tool_only`, produces a canned command reply, and then skips final delivery while `suppressDelivery` is true.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared command/system-reply bypass in the auto-reply delivery policy, not a Matrix-specific workaround or a new config option; open PR #77325 is the active candidate to review.
