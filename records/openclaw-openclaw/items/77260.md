## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats under `messages.groupChat.visibleReplies: "message_tool"`, and open PR #77325 already references this issue with closing syntax as the implementation path.

## What This Changes

Text-prefix slash command replies in group chats should remain visibly delivered even when group visible replies require the message tool.

## Best Possible Solution

Review and land the linked shared auto-reply fix if it preserves normal message-tool-only model replies while making canned command/system replies visible.

## Reproduction Assessment

Yes, by source inspection. A Matrix group `/new` turn uses `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned command reply, and then final delivery is skipped under `suppressDelivery`.

## Solution Assessment

Yes. The narrow maintainable direction is a shared source-reply/command visibility bypass, not a Matrix-specific workaround or a new config option.
