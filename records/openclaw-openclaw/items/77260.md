## Summary

Keep open: current main still suppresses text-prefix command replies in group/channel chats when visible replies resolve to message-tool-only, and open PR #77325 is already paired as the implementation candidate. The added Slack and Matrix comments support the same shared root cause and the `messages.groupChat.visibleReplies: "automatic"` workaround.

## What This Changes

Text-prefix slash command replies in group chats should remain visible when `messages.groupChat.visibleReplies` is set to `message_tool`.

## Best Possible Solution

Add a shared auto-reply visibility rule that delivers canned command/system replies for true text slash commands while preserving message-tool-only suppression for normal model final replies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path on current main: Matrix group `/new` enters with `CommandSource: "text"`, resolves to `message_tool_only`, returns a canned reset reply, and dispatch suppresses final delivery.

## Solution Assessment

Yes. A shared command/system-reply bypass in source reply visibility is the narrow maintainable fix; channel-specific workarounds or a new config option would leave the common dispatch contract inconsistent.
