## Summary

Close as duplicate/superseded, not as fixed. Current main still has the provider-backed hybrid FTS broadening gap, but the same remaining work is already tracked by the open replacement PR at https://github.com/openclaw/openclaw/pull/73976, which superseded the older contributor PR at https://github.com/openclaw/openclaw/pull/39555.

## What This Changes

The issue reports that session transcript chunks with strong exact keyword matches can be filtered out because provider-backed hybrid memory search does not broaden BM25/FTS recall when dialogue-format chunks miss the vector candidate pool.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/73976 or an equivalent current-main patch that adds bounded provider-backed hybrid FTS broadening while preserving existing FTS-only recall behavior.

## Reproduction Assessment

Yes at source level, but not as a replayed live database failure. Current main shows strict all-token FTS in provider-backed hybrid mode and no per-keyword retry when that keyword lookup returns no rows, matching the reporter's empty-result symptom.

## Solution Assessment

Yes, the canonical bounded per-keyword hybrid FTS fallback direction is the narrow maintainable fix. The open replacement PR still needs its review finding addressed before merge, so this issue should close only as duplicate/superseded tracking.
