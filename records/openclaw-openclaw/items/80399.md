## Summary

Keep open. Current main already runs BM25 and vector retrieval as separate paths, so the reported sequential KNN diagnosis does not match the source, but the source still supports a real hybrid recall bug: provider-backed hybrid search does not broaden keyword search when the strict all-token FTS query returns no rows.

## What This Changes

The issue requests reliable memory_search recall for session transcript chunks when vector search misses but FTS can match the transcript text.

## Best Possible Solution

Reuse or adapt the existing FTS-only query-broadening behavior in provider-backed hybrid search so keyword-strong session chunks can enter the merge before minScore filtering, with focused session transcript regression coverage.

## Reproduction Assessment

Yes at source level, but not as a replayed live database failure. Current main shows strict all-token FTS in hybrid mode, no per-keyword broadening when keywordResults is empty, and default minScore filtering that can drop low-vector session chunks.

## Solution Assessment

No, not as the issue diagnoses it: current main already runs BM25 and vector search in parallel. The narrow maintainable fix is hybrid keyword broadening when the exact FTS query misses, without changing sessionMemory defaults or adding a config knob.
