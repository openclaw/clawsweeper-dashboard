## Summary

Keep open. Current main already runs BM25 and vector retrieval in parallel, so the reported sequential diagnosis does not match the current source, but the shipped failure remains plausible because hybrid mode does not broaden the FTS query when the exact all-token BM25 query returns no session chunks.

## What This Changes

The issue asks for session transcript memory_search to surface BM25-strong chunks even when vector KNN misses or underscores dialogue-format chunks.

## Best Possible Solution

Reuse the FTS-only query broadening in provider-backed hybrid search so keyword-strong session chunks participate before minScore filtering, with focused regression coverage.

## Reproduction Assessment

Yes at source level, but not as a replayed live database failure. Current main shows strict all-token FTS in hybrid mode, no per-keyword broadening when keywordResults is empty, and default minScore filtering that can drop low-vector session chunks.

## Solution Assessment

No, the issue's proposed explanation is not the best current fix because parallel FTS already exists. The safer fix is to broaden hybrid keyword recall when the exact FTS query misses, without changing sessionMemory defaults or adding a new config knob.
