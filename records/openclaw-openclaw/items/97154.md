## Summary

Keep open for maintainer/security review: current main does not yet support alternate Slack Web API roots, and the branch uses Slack SDK's intended `slackApiUrl` seam across the main Slack WebClient paths, but the new process/global `SLACK_API_URL` default can redirect bearer-token calls and needs explicit maintainer acceptance.
