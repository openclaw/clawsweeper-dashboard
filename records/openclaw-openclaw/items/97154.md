## Summary

Keep this PR open: current main and the latest release do not yet support alternate Slack Web API roots, and the member-authored PR has a protected maintainer label. ## What This Changes

The PR centralizes Slack WebClient option resolution so explicit `slackApiUrl` or trusted `SLACK_API_URL` can route Slack Web API calls to an alternate root, while blocking workspace `.env` keys ending in `_API_URL`.

## Best Possible Solution

Use the Slack SDK `slackApiUrl` seam, but merge only after maintainers explicitly accept the trusted env and workspace-dotenv boundary or narrow the feature to explicit runtime options.
