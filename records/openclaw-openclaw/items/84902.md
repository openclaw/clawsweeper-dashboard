## What This Changes

Adds an opt-in `overridesCatalogReasoning` provider thinking-profile flag, uses it before stale catalog `reasoning: false` forces off-only thinking, and updates OpenAI/Codex GPT-5 reasoning classification, tests, SDK docs, and API baseline hashes.

## Best Possible Solution

Merge the provider-profile override design if CI passes and maintainers accept the resolver-level proof, with a live OpenAI/Codex subagent proof reserved for higher assurance before release.

## Reproduction Assessment

Yes at source level: current main short-circuits to off-only on catalog `reasoning: false` before provider policy, and the linked issue provides exact live v2026.5.19 `sessions_spawn` repro calls. I did not run a live OpenAI/Codex subagent locally in this read-only review.

## Solution Assessment

Yes, the proposed opt-in provider-profile override is the narrow maintainable fix because it preserves catalog off-only behavior for non-authoritative providers while letting OpenAI/Codex own stale catalog metadata for known reasoning models. The safer alternative, if maintainers want more assurance, is live OAuth subagent proof before merge rather than a different code shape.
