## Summary

## What This Changes

Request to add explicit role/source metadata to conversation history so skills, hooks, and integrations can distinguish user, assistant, and tool-originated messages.

## Best Possible Solution

Define one backward-compatible message provenance contract for plugin-visible runtime surfaces, aligned with the existing ContextEngine provenance RFC and current transcript role shape.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection confirms roles already exist in persisted session messages, while the broader source/provenance metadata contract remains a design gap.

## Solution Assessment

Unclear as a standalone change. Adding only role fields is not the right fix because roles already exist; the maintainable path is to design source/provenance metadata alongside the existing ContextEngine provenance RFC.
