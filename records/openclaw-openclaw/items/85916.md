## What This Changes

## Best Possible Solution

Keep the admin-only direct `browser.proxy` boundary, but update first-party browser proxy callers to use an admin-scoped route and add regression coverage proving both write-scoped direct denial and official browser/Google Meet success before merge.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The admin boundary is the right direction, but the patch needs to update existing first-party browser proxy callers that still use write-scoped `node.invoke` defaults.
