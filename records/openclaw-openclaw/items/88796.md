## Summary

Keep open: the PR addresses a real Discord search bug, but the current patch can silently narrow explicit guild-wide searches to the active Discord channel and still lacks redacted live Discord bot/API proof.

## What This Changes

The PR changes Discord message search handling to infer missing `guildId` from channel context, normalize channel IDs, accept `query`/`content` aliases, and add Discord action regression tests.

## Best Possible Solution

Keep the runtime-side `guildId` inference, but only inject the session channel when `guildId`, `channelId`, and `channelIds` are all absent, then prove both omitted-`guildId` and explicit guild-wide searches in a redacted live Discord setup.

## Reproduction Assessment

Yes. Source inspection of PR head shows `message.search` with an explicit `guildId` and no channel filters receives `ctx.toolContext.currentChannelId`, and the runtime turns that into a `channelIds` search filter.

## Solution Assessment

No. The fallback direction is useful for omitted-`guildId` calls, but the best fix must preserve explicit guild-wide searches and then show live Discord proof.
