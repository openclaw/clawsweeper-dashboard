## Reproduction Assessment

Yes for source-level reproduction: current main and `v2026.6.1` require explicit `guildId` before Discord search can use channel context, so a query-only in-channel search fails. I did not run a live Discord session in this read-only review.

## Solution Assessment

Mostly yes: the fix stays inside the Discord plugin boundary, reuses channel normalization and read-target authorization, and avoids broadening explicit filters. Adding a new protocol `currentGuildId` field would be a broader contract change, so channel-info inference is a reasonable narrow repair if live proof validates the scope.
