## Summary

Keep open: this is a useful Discord search bug fix, but the current PR is not merge-ready because the new session-channel fallback can narrow existing explicit `guildId`-only searches and the supplied proof still does not show a real Discord bot/API search succeeding after the patch.

## What This Changes

The PR changes Discord search action handling to accept `query` or `content`, infer missing `guildId` from channel/session context, normalize Discord channel IDs, and add regression coverage.

## Best Possible Solution

Preserve explicit `guildId`-only guild-wide searches, use the session-channel fallback only when both `guildId` and explicit channel filters are absent, add a wrapper regression test, and require redacted live Discord search proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main requires `guildId` before any fallback, and PR head shows `{ guildId, query }` plus Discord session context getting a `channelId` injected. I did not run a live Discord bot session in this read-only review.

## Solution Assessment

No as written: resolving missing `guildId` from channel info is an acceptable narrow fix, but the wrapper fallback must not alter explicit `guildId`-only searches. A longer-term `currentGuildId` context seam could be cleaner, but it is not required to repair this PR safely.
