## Summary

Keep open: the PR addresses a real Discord search bug, but the current patch still changes explicit guildId-only searches into current-channel searches in Discord sessions, and the supplied proof does not show a real Discord bot/API search succeeding after the fix.

## What This Changes

The PR makes Discord search accept `query`/`content`, infer a missing `guildId` from channel info, normalize channel IDs, and adds Discord action/runtime regression tests.

## Best Possible Solution

Preserve explicit guildId-only guild-wide searches, use the session channel only when `guildId`, `channelId`, and `channelIds` are all absent, add that regression test, and include redacted live Discord proof.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.1 still require explicit `guildId` in the Discord search path, and the linked issue includes live logs for a guild-channel tool call with `query` but no `guildId`.

## Solution Assessment

No, not as currently patched. Inferring `guildId` from channel info is the right owner-local shape, but the wrapper must not inject a channel filter when the caller already supplied `guildId` for a guild-wide search.
