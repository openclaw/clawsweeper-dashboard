## What This Changes

The PR updates Discord message-search handlers and tests to infer `guildId` from current or supplied channel IDs, normalize channel targets, accept `query`/`content` aliases, and avoid session-channel injection when explicit filters or `guildId` are present.

## Best Possible Solution

Land this Discord-plugin fallback after redacted live Discord guild-channel proof, or an explicit maintainer proof exception, confirms omitted-`guildId` search succeeds and explicit filters remain scoped.

## Reproduction Assessment

Yes at source level: current `main` requires `guildId` in the tool adapter and runtime handler before current-channel context can help, and the linked issue includes live failure logs. I did not run a live Discord bot reproduction in this read-only review.

## Solution Assessment

Yes, this appears to be the right narrow fix shape: it stays inside the Discord plugin, keeps Discord REST search guild-scoped, and preserves read-target authorization. Adding a new core `currentGuildId` tool-context API would be broader than needed for this bug.
