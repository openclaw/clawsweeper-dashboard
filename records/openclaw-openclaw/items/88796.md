## Summary

Keep open: this is a useful Discord search bug fix, but PR head still has a blocking compatibility regression in the wrapper fallback and the supplied proof does not show a live Discord search succeeding after the patch.

## What This Changes

The PR changes Discord search actions to accept `query`/`content`, infer missing `guildId` from channel info or the current session channel, normalize Discord channel IDs, and add regression tests.

## Best Possible Solution

Keep the runtime guild-resolution and channel-normalization fix, but only inject the session channel when `guildId`, `channelId`, and `channelIds` are all absent, add a wrapper regression test, and require redacted live Discord proof before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows that `{ guildId, query }` plus Discord session context and no channel filters takes the new current-channel fallback, while current main preserves a guild-wide search path.

## Solution Assessment

No. The runtime fallback is a good repair direction, but the wrapper fallback is too broad; the narrower fix is to preserve explicit guild-wide searches and add current-session fallback only for omitted-`guildId` searches.
