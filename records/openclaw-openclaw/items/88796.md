## Summary

Keep open: this PR addresses a valid Discord search bug, but it is not merge-ready because the session-channel fallback can narrow existing explicit guild-wide searches and the external contributor proof still does not show a live Discord bot/API search succeeding after the patch.

## What This Changes

The PR changes Discord message search handling to infer missing guild IDs from channel context, normalize channel IDs, accept `query` as a search text alias, and add regression tests.

## Best Possible Solution

Gate session-channel injection on missing `guildId` as well as missing explicit channel filters, preserve guild-wide searches, add the regression, and then supply redacted live Discord search proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.1` require `guildId` for Discord search, while the linked issue shows live tool calls naturally omit it. I did not run a live Discord bot session in this read-only review.

## Solution Assessment

No, not as written: resolving `guildId` from channel context is a reasonable fix direction, but the action-layer session fallback is too broad and changes explicit guild-wide search behavior.
