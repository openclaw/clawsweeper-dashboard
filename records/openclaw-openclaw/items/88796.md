## Summary

Keep open: the central Discord search fix is useful, but the current branch still narrows explicit guildId-only searches from a Discord session and lacks live Discord behavior proof.

## What This Changes

The PR makes Discord search accept query/content, infer a missing guildId from channel data, normalize Discord channel IDs, and adds action/runtime regression tests.

## Best Possible Solution

Preserve explicit guildId-only guild-wide searches, use the session channel only when guildId and explicit channel filters are absent, add that regression test, then supply redacted live Discord proof.

## Reproduction Assessment

Yes. Current main source requires guildId in both the message action and runtime search path, and the linked report includes live logs for a guild-channel tool call with query but no guildId.

## Solution Assessment

No, not as currently patched. Inferring guildId from channel info is a good owner-local fix, but the fallback must not inject a channel filter when the caller already supplied guildId for a guild-wide search.
