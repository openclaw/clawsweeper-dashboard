Yes for the code shape: resolving guildId inside the Discord plugin from normalized channel context keeps the Discord REST search API guild-scoped and preserves authorization checks. Adding a new currentGuildId tool-context field would broaden the shared channel contract, so this plugin-local fallback is the narrower fix.
