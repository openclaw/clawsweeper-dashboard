## Summary

Keep this PR open, but not merge-ready: the bug is valid and the runtime direction is useful, yet the current fallback can silently narrow existing explicit guild-wide Discord searches to the current channel, and the supplied proof still does not show a real Discord bot/API search succeeding after the patch.

## What This Changes

The PR makes Discord message search accept `query`/`content`, infer missing `guildId` from channel info or the session channel, normalize channel IDs, and adds focused Discord action/runtime tests.

## Best Possible Solution

Preserve guild-wide searches when `guildId` is explicit, use the session channel only to infer a missing guild, add that regression test, and attach redacted live Discord proof before merge.

## Reproduction Assessment

Yes, at source level: current main requires `guildId` in both the Discord search action entry point and runtime, and the linked issue includes live logs of `ToolInputError: guildId required`. I did not run a live Discord repro in this read-only review.

## Solution Assessment

No, not as written. Runtime guild inference is a reasonable fix direction, but the action fallback must preserve explicit guild-wide searches instead of injecting the session channel whenever `guildId` is already supplied.
