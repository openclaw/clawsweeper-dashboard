## Summary

Keep this PR open: it addresses a real Discord search bug, but the current branch can silently narrow explicit guild-wide searches to the current session channel and still lacks live Discord/API proof of the fixed path.

## What This Changes

The PR makes Discord message search accept `query`/`content`, infer a missing `guildId` from channel context, normalize channel IDs, and adds Discord action/runtime regression tests.

## Best Possible Solution

Keep the guildId inference and query alias, but gate session-channel injection on missing `guildId` as well as missing explicit channel filters, add handle-action coverage for explicit guild-wide search, and require redacted live Discord search proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main requires `guildId` for Discord search while the linked issue shows live tool calls naturally omit it. I did not run a live Discord bot session in this read-only review.

## Solution Assessment

No, not as written: the runtime inference is a reasonable repair direction, but the handle-action fallback is too broad because it changes explicit guild-wide searches. The narrow fix is to inject the session channel only when `guildId` and explicit channel filters are absent.
