The PR makes Discord search accept `query` or `content`, infer a missing `guildId` from current or provided channel IDs, normalize channel IDs before search, and adds focused regression tests.

## Best Possible Solution

Land the narrow Discord-plugin fallback only after redacted live proof confirms the omitted-`guildId`/`query` path searches the intended guild or channel scope.

## Reproduction Assessment

Yes for source-level reproduction: current main requires `guildId` and runtime `content` for Discord search while the linked report shows a live model sent `query` without `guildId`. I did not establish a fresh live Discord failure in this read-only review.

## Solution Assessment

Yes, the PR is a maintainable plugin-local fix because it stays inside Discord action/runtime code and preserves existing authorization before search dispatch. A core `currentGuildId` context seam would avoid the extra channel lookup, but that is a broader API change than this bug needs if live scoping proof passes.
