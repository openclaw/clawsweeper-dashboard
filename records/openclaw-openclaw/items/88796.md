## Summary

Keep open: current main and the latest release still have the reported Discord search failure, but this PR is not merge-ready because the wrapper fallback can narrow explicit guild-wide searches to the current Discord channel and the supplied proof does not show the live Discord search path.

## What This Changes

The PR updates Discord search action handling to accept `query` or `content`, infer a missing `guildId` from channel context, normalize channel IDs before Discord channel-info lookup, and add regression coverage.

## Best Possible Solution

Keep the channel-info `guildId` fallback, guard session-channel injection so explicit `guildId`-only searches stay guild-wide, and merge only after redacted live Discord search proof shows the normal guild-channel path works.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.1 require `guildId` and `content` in the Discord search handler before any fallback, matching the linked report and logs. I did not run a live Discord bot session in this read-only review.

## Solution Assessment

No as written: resolving missing `guildId` from channel info is an acceptable narrow fix, but the wrapper fallback must not alter explicit `guildId`-only searches. A broader `currentGuildId` context seam could be cleaner later, but this PR can be repaired safely with a narrower guard and regression test.
