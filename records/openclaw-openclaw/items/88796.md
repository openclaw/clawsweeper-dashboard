## Summary

Keep open: the PR addresses a real Discord search bug, but the current diff can silently narrow explicit guild-wide searches to the current session channel and the external live Discord proof gate is still unmet.

## What This Changes

The PR makes Discord message search accept `query`/`content`, infer missing `guildId` from channel info, normalize `channel:` IDs, and adds focused Discord action/runtime tests.

## Best Possible Solution

Preserve guild-wide searches when `guildId` is explicit, use the session channel fallback only to infer a missing guild, add that regression test, and then attach redacted live Discord proof.

## Reproduction Assessment

Yes, at source level: current main requires `guildId` in both the Discord search action entry point and runtime, and the linked report includes live logs of `ToolInputError: guildId required`. I did not run a live Discord repro in this read-only review.

## Solution Assessment

No, not as written. The runtime guild inference is a reasonable fix direction, but the action fallback must preserve explicit guild-wide searches instead of injecting the session channel whenever `guildId` is already supplied.
