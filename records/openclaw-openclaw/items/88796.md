This PR makes Discord message search accept `query`/`content`, infer a missing `guildId` from channel info, normalize channel targets, and add regression tests.

## Best Possible Solution

Preserve explicit guild-wide searches, use the session channel fallback only when `guildId` and explicit channel filters are absent, add regression coverage for that boundary, and require redacted live Discord search proof before merge.

## Solution Assessment

No as written. The plugin-local channel lookup is an acceptable narrow fix shape, but the session-channel fallback must be gated to missing `guildId` and the PR still needs live Discord/API proof.
