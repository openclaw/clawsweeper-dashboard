## Summary

Keep open: the PR fixes a real Discord search failure path, but the current patch introduces a blocking scope regression for explicit guild-wide searches and still lacks redacted live Discord proof.

## What This Changes

The PR makes Discord message search accept `query`/`content`, infer missing `guildId` from channel info or the current session channel, normalize channel IDs, and add focused regression tests.

## Best Possible Solution

Keep the missing-`guildId` fallback, but only inject the current session channel when `guildId` is absent, preserve explicit guild-wide searches, and require redacted live Discord proof before merge.

## Reproduction Assessment

Yes, from source and reporter evidence: current shipped/main Discord search requires `guildId`, while the linked issue shows a normal guild-channel model call with only `query` failing with `ToolInputError: guildId required`. I did not run a live Discord repro in this read-only review.

## Solution Assessment

No, this is the right owner area but not the best fix as written. The fallback should be gated on missing `guildId` so explicit guild-wide searches keep their existing scope, with a regression test and live Discord proof.
