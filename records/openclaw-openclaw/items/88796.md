## Summary

Keep open: current main still has the Discord omitted-guildId search bug, but this PR is not merge-ready because the adapter can narrow explicit guild-wide searches to the current channel and the supplied proof does not exercise a real Discord search path.

## What This Changes

The PR changes Discord message search to accept query/content aliases, infer a missing guildId from a channel or session channel, normalize channel ids, and add Discord action regression tests.

## Best Possible Solution

Keep the owner-local Discord fallback, but use the session channel only when guildId and explicit channel filters are both absent; preserve explicit guild-wide searches and require redacted live Discord proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.1 require guildId before Discord search can proceed, matching the linked live logs in https://github.com/openclaw/openclaw/issues/88790. The PR-introduced scope regression is also source-reproducible from the adapter fallback and runtime guild-wide branch.

## Solution Assessment

No as written: inferring guildId from a channel is a reasonable Discord-owned fix, but injecting the session channel when guildId is already explicit changes existing guild-wide searches. The narrow maintainable fix is to gate that fallback on missing guildId too.
