## Summary

Keep open: the Discord search bug is real and the PR is useful, but the current patch still needs a scope decision/fix for omitted-guild searches and stronger live Discord proof before merge.

## What This Changes

The PR updates Discord message search to accept `query`/`content`, infer omitted `guildId` from channel info, normalize channel targets, preserve explicit filters, and add regression tests.

## Best Possible Solution

Derive the session guild without adding an implicit channel filter unless maintainers intentionally choose current-channel fallback, then validate the final behavior with redacted live Discord proof.

## Reproduction Assessment

Yes, at source level. Current main requires `guildId` before Discord search dispatch and the linked report includes live logs showing `ToolInputError: guildId required`, but I did not run a live Discord bot repro in this read-only review.

## Solution Assessment

No, not yet. The runtime channel-info inference is a good narrow direction, but the adapter fallback appears to use the session channel as both guild evidence and a search filter, so maintainers should either preserve guild-wide search or explicitly choose current-channel scoping.
