## What This Changes

The PR relaxes Discord search validation, infers a missing `guildId` from current or explicit Discord channel context, accepts `query`/`content` aliases, normalizes channel IDs, and adds regression tests.

## Best Possible Solution

Land this plugin-owned fallback after redacted live Discord proof, or an explicit maintainer proof override, confirms omitted-`guildId` guild-channel search and explicit channel filters behave correctly.

## Reproduction Assessment

## Solution Assessment

Yes, the proposed code direction is likely the best bounded fix: it keeps guild resolution inside the Discord plugin/runtime instead of expanding core toolContext or gateway protocol surface. The remaining gap is live API proof, not an obvious better code layer.
