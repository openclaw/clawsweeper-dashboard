This PR makes Discord message search accept omitted `guildId` by resolving it from the current or explicit channel, normalizes `channel:` targets, accepts `query`/`content`, and adds focused regression tests.

## Best Possible Solution

Land this patch after redacted live Discord proof, or explicit maintainer acceptance, confirms omitted-`guildId` search succeeds without broadening explicitly filtered searches.

## Solution Assessment

Yes, this appears to be the best narrow fix shape: it stays inside the Discord plugin, avoids expanding the core tool-context protocol, reuses channel lookup and target normalization, and preserves read-target authorization before search dispatch. The remaining gap is live behavior proof, not an identified code-path defect.
