## Summary

Keep open: the PR fixes the right Discord search bug direction, but it still narrows explicit `guildId`-only searches to the current session channel and the contributor proof does not show a live Discord/API search succeeding after the patch.

## What This Changes

This PR makes Discord message search infer a missing `guildId` from channel context, normalize channel filters, accept `query`/`content` aliases, and add regression tests.

## Best Possible Solution

Gate session-channel injection on missing `guildId` as well as missing explicit channel filters, preserve guild-wide search semantics, add the regression, and require redacted live Discord search proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.1` require `guildId` for Discord search, while the linked issue shows live tool calls naturally omit it. I did not run a live Discord bot session in this read-only review.

## Solution Assessment

No, not as written: deriving `guildId` from channel context is the right fix direction, but the current action-layer fallback is too broad and changes explicit guild-wide search behavior. The safer repair is to use the session channel only when no `guildId` or channel filters were supplied.
