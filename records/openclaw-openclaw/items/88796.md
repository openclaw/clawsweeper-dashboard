## Summary

Keep open: current main still has the Discord search `guildId` failure, and this PR is a scoped plugin-side fix with no blocking code findings found. The remaining merge blocker is proof: the supplied terminal simulation and mocked tests do not show a real Discord guild-channel search succeeding after the patch.

## What This Changes

The branch makes Discord message search accept `query` or `content`, infer missing `guildId` from the current or explicit channel, normalize channel targets, and adds regression tests.

## Best Possible Solution

Discord search should infer `guildId` from trusted Discord channel context or explicit channel filters, preserve explicit guild/channel scoping and authorization, and land only with live Discord proof or explicit maintainer acceptance of the proof gap.

## Reproduction Assessment

Yes. Current main source shows both the tool entry point and runtime handler require `guildId`, so a Discord `search` action with only `query` and no `guildId` fails before the search request can run.

## Solution Assessment

Yes, with a proof caveat. Keeping the fix inside the Discord plugin and using existing channel context avoids a broader SDK/protocol expansion; adding `currentGuildId` to core would be a larger API change than this bug needs.
