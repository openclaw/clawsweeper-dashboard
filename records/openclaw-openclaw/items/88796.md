## What This Changes

The branch changes Discord message search to accept `query`/`content`, infer a missing `guildId` from an explicit or current Discord channel, normalize channel targets, and add focused regression tests.

## Best Possible Solution

Land the plugin-local fallback after redacted live Discord proof, or an explicit maintainer proof exception, shows omitted-guildId search succeeds and stays correctly scoped.

## Reproduction Assessment

Yes at source level: current main and v2026.6.1 hard-require `guildId` for Discord search before current-channel context can help. The linked issue also includes live logs, but I did not run a live Discord bot reproduction here.

## Solution Assessment

Yes, this is an acceptable narrow fix: it stays inside the Discord plugin, keeps the REST search guild-scoped, and preserves read-target authorization. Adding `currentGuildId` to tool context could avoid one lookup, but that would expand core/plugin context surface for a narrow bug.
