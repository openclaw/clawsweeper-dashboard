## What This Changes

The issue reports that Windows Claude CLI launches fail because bare npm-style `.cmd` shims are not resolved before child-process spawn.

## Best Possible Solution

Land one canonical Windows supervisor CLI shim fix that materializes bare npm-backed provider commands through the trusted wrapper path, with regression coverage and redacted Windows proof.

Yes at source level: the Anthropic backend declares bare `claude`, the CLI runner forwards it to the supervisor, and the current Windows resolver only suffixes package-manager names. I did not run a live Windows host in this read-only review.
