## What This Changes

The issue reports that Windows supervisor spawning fails for the bundled Claude CLI provider because bare `claude` is not resolved to the npm `.cmd` shim before `child_process.spawn`.

## Best Possible Solution

Land a safe canonical supervisor Windows shim fix from the linked PRs, with regression coverage and `.cmd` execution semantics that avoid unsafe shell parsing unless maintainers explicitly accept that risk.

## Reproduction Assessment

Yes. Current source shows the Anthropic CLI backend passes bare `claude` into the supervisor child adapter, and that adapter only appends `.cmd` for npm/pnpm/yarn/npx; the related Windows 11 report adds live user confirmation, though I did not run Windows in this read-only review.

## Solution Assessment

No, cleanup closure is not the best way to solve the issue. The best path is to review and land the safest linked fix PR, then close this issue from the merged implementation.
