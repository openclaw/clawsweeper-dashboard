## Summary

Keep open: current main and v2026.6.5 still route bare `claude` through the supervisor Windows shim whitelist without `claude`, and the issue is already paired with open fix PRs that should be reviewed rather than replaced.

## What This Changes

The issue asks OpenClaw to resolve Windows npm-style `.cmd` shims for the Claude CLI supervisor spawn path so `claude` does not fail with ENOENT/EINVAL.

## Best Possible Solution

Land one narrow supervisor Windows shim fix with regression coverage and Windows after-fix proof, avoiding `shell:true` unless maintainers explicitly accept the security and argv-parsing tradeoff.

## Reproduction Assessment

Yes, at source level: the bundled Claude CLI backend passes bare `claude` to the supervisor, and current main does not append `.cmd` for that command. I did not run a live Windows reproduction in this read-only review.

## Solution Assessment
