## What This Changes

Fix Windows supervisor launching of bundled Claude and Gemini CLI commands installed as npm `.cmd` shims.

## Best Possible Solution

Reuse the existing trusted system-`cmd.exe` wrapping and escaping contract in the supervisor, cover every bundled npm-installed CLI backend command, and verify a full provider turn on Windows.

## Reproduction Assessment

Yes, at source level. Current main passes bare bundled CLI command names through a supervisor whitelist that omits them, and the related Windows 11 report provides a concrete failing command and output.

## Solution Assessment

No, neither submitted approach is yet the best complete fix: generic `shell: true` is too broad, while suffix-only resolution does not establish the required safe batch-file execution path.
