## Summary

Keep open. Current main still lacks a structured accepted-`sessions_spawn` terminal-success signal for the incomplete-turn classifier, so the reported zero-payload relay turn can still be surfaced as an incomplete response.

## What This Changes

Treat a zero-payload terminal turn with a successful `sessions_spawn` as complete instead of surfacing the incomplete-turn warning.

## Best Possible Solution

Carry a structured terminal-success marker for accepted `sessions_spawn` results and suppress the warning only when `status`, `runId`, and `childSessionKey` prove a child run was created.

## Reproduction Assessment

Yes, source-level. Model an attempt with `payloadCount: 0`, no visible assistant text, `stopReason: "stop"`, and a completed accepted `sessions_spawn`; current main has no accepted-spawn proof available to the classifier, so it can still return the warning.

## Solution Assessment

Yes. A narrow accepted-spawn exemption backed by structured result fields is more maintainable than a relay-agent acknowledgement workaround or a broad empty-turn bypass.

## Repair Work Prompt

Fix the gateway incomplete-turn false-negative for pure relay agents in https://github.com/openclaw/openclaw/issues/72541. Current main records accepted `sessions_spawn` completions only as generic tool metadata, so a zero-payload terminal turn can still surface `Agent couldn't generate a response`. Add a structured terminal-success marker for `sessions_spawn` only when the tool result is non-error and has `status: "accepted"`, a non-empty `childSessionKey`, and a non-empty `runId`; carry that marker through subscribe state / attempt result into `resolveIncompleteTurnPayloadText`, and suppress the incomplete-turn warning only for that proven accepted-spawn case. Add regression coverage for accepted ACP/subagent spawn with zero visible payload, and for failed or malformed spawn results still warning. Do not add a config option, do not suppress generic empty tool turns, and update `CHANGELOG.md` because the fix is user-visible log/behavior noise.
