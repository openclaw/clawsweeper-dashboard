## Summary

Keep open. Current main still has the reported false-negative path: accepted `sessions_spawn` results are documented and returned as terminal background-spawn success, but the incomplete-turn classifier only sees zero payloads plus mutating-tool replay metadata, so it can still surface the incomplete-turn warning for a pure relay agent.

## What This Changes

The issue requests that a zero-text turn ending in a successful `sessions_spawn` call be treated as complete instead of being reported as an incomplete gateway turn.

## Best Possible Solution

Preserve a narrow structured terminal-success signal for completed `sessions_spawn` calls and have the incomplete-turn classifier suppress the warning only when that signal proves an accepted child run was created.

## Reproduction Assessment

Yes. A focused test can construct a zero-payload terminal assistant turn after a successful `sessions_spawn`; current main collapses the tool result to `toolName/meta`, so the classifier still follows the empty-response warning path.

## Solution Assessment

Yes. The issue’s proposed global behavior is appropriate if implemented as a structured accepted-spawn exemption, not as a broad empty-turn disablement or a relay-agent acknowledgement requirement.
