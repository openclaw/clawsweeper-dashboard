## Summary

Keep open. Current main still lacks a structured accepted-`sessions_spawn` terminal-success signal in the incomplete-turn classifier path, so a zero-payload pure-relay turn can still be surfaced as “Agent couldn't generate a response” after a successful child spawn.

## What This Changes

The issue asks the gateway completeness check to treat a successful terminal `sessions_spawn` as a completed turn even when no visible assistant payload is emitted.

## Best Possible Solution

Preserve a structured terminal-success marker for accepted `sessions_spawn` calls and suppress the incomplete-turn warning only when that marker proves a child run was created.

## Reproduction Assessment

Yes, from source. A focused unit test can model `payloadCount: 0`, no assistant text, `stopReason: "stop"` or a reasoning-only final assistant, and a completed accepted `sessions_spawn`; current main records no accepted-spawn fields for the classifier, so the warning path remains reachable.

## Solution Assessment

Yes. A narrow accepted-spawn exemption backed by `status`, `runId`, and `childSessionKey` is the maintainable fix; it is safer than a relay-agent acknowledgement workaround or a broad empty-turn bypass.

## Repair Work Prompt

Fix the gateway incomplete-turn false positive for pure relay agents that successfully call `sessions_spawn` and then emit no visible assistant text. Preserve a structured terminal-success marker for `sessions_spawn` results only when the tool result proves `status: "accepted"` plus non-empty `runId` and `childSessionKey`; use that marker in `resolveIncompleteTurnPayloadText` to suppress the warning for zero-payload terminal turns. Keep the existing warning for empty turns with no accepted spawn, failed/forbidden spawns, missing IDs, tool errors, and unrelated mutating tools. Add focused regression coverage for accepted ACP `sessions_spawn` and a negative case. Update `CHANGELOG.md` with a single-line user-facing fix entry crediting the reporter if appropriate. Do not add a broad config flag or bypass the incomplete-turn heuristic globally.
