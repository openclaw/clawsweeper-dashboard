## Summary

Keep open. Current main still appears to have the reported classifier gap: a zero-payload terminal turn after accepted `sessions_spawn` can still reach the incomplete-turn warning because the accepted spawn result is not preserved as terminal-success evidence for the completeness check.

## What This Changes

The issue asks the gateway completeness check to treat a successful terminal `sessions_spawn` acceptance as completion even when the parent agent emits no text.

## Best Possible Solution

Carry a structured accepted-spawn terminal-success marker into the attempt result and suppress the incomplete-turn warning only when `status`, `runId`, and `childSessionKey` prove that a child run was created.

## Reproduction Assessment

Yes. A focused unit test can model payload count `0`, no visible assistant text, `stopReason: "stop"`, and a completed accepted `sessions_spawn`; source inspection shows current main has no accepted-spawn proof available to the classifier.

## Solution Assessment

Yes. A narrow accepted-spawn terminal-success marker is the maintainable fix; it avoids a relay-agent acknowledgement workaround, avoids a new config option, and preserves warnings for genuinely empty turns.
