## Summary

Keep open. Current main still has the reported false-negative path: a successful `sessions_spawn` can be returned as accepted, but the incomplete-turn classifier does not receive or inspect that accepted result and will still surface the zero-payload incomplete-turn warning for a pure relay agent.

## What This Changes

The issue requests that a zero-text turn ending in a successful `sessions_spawn` result be treated as a completed terminal action instead of an incomplete assistant response.

## Best Possible Solution

Preserve a narrow structured terminal-success signal for completed `sessions_spawn` calls through attempt state, then have the incomplete-turn classifier suppress the warning only when that signal proves an accepted child run was created.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload terminal assistant turn after a successful `sessions_spawn`; current main collapses the tool result before classification, so the warning path remains reachable.

## Solution Assessment

Yes. The maintainable fix is a structured accepted-spawn exemption in the runtime/classifier path, not an agent-level acknowledgement workaround or a broad disablement of empty-turn detection.
