## Summary

Keep open. Current main still has no structured accepted-`sessions_spawn` terminal-success signal in the incomplete-turn classifier, so a zero-payload pure-relay turn can still fall through to the user-visible incomplete-turn warning.

## What This Changes

The issue requests that gateway completeness checks treat an accepted `sessions_spawn` with `runId` and `childSessionKey` as a successful terminal action even when the parent emits no text.

## Best Possible Solution

Carry a structured accepted-spawn terminal-success record through tool execution into the incomplete-turn resolver, and suppress the warning only when `sessions_spawn` completed with `status: "accepted"`, `runId`, and `childSessionKey`.

## Reproduction Assessment

Yes. A focused unit test can model `payloadCount: 0`, final assistant `stop`, no visible text, and a completed accepted `sessions_spawn`; current main has no accepted-spawn signal in the classifier path, so it reaches the incomplete-turn warning.

## Solution Assessment

Yes. A structured accepted-spawn exemption is narrower and more maintainable than requiring relay agents to emit acknowledgement text or disabling empty-turn detection broadly.

## Repair Work Prompt

Fix the gateway/embedded-runner completeness false-negative for pure-relay agents that end with no visible text after a successful `sessions_spawn`. Preserve a structured terminal-success signal for completed `sessions_spawn` calls, including at least `status`, `runId`, and `childSessionKey`, from tool execution into the incomplete-turn resolver. Suppress the `Agent couldn't generate a response` incomplete-turn warning only when that structured result proves an accepted child run was created. Do not suppress warnings for failed, forbidden, malformed, missing, or errored spawn results, and do not broaden generic empty-turn or silent-reply behavior. Add focused regression coverage for zero-payload accepted spawn terminal success and for failed/non-accepted spawn still warning. Add a single-line `CHANGELOG.md` fix entry because this changes user-visible gateway behavior.
