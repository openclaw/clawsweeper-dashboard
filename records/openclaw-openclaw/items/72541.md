## Summary

Keep open. Current main still has the reported false-negative path: a zero-payload terminal turn after `sessions_spawn` is not recognized as successful completion unless there is committed messaging delivery evidence, and the accepted spawn result fields are not preserved for the incomplete-turn classifier.

## What This Changes

The issue asks OpenClaw to stop surfacing the incomplete-turn warning when a pure relay agent successfully ends its turn with an accepted `sessions_spawn` call and no text payload.

## Best Possible Solution

Preserve a structured terminal-success signal for accepted `sessions_spawn` calls and have the incomplete-turn classifier suppress the warning only when that signal proves a child run was created.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload `stop` assistant turn after successful `sessions_spawn`; current main has no accepted-spawn metadata in the classifier path and will still return the incomplete-turn warning.

## Solution Assessment

Yes. The narrow maintainable fix is to carry accepted-spawn result evidence into the attempt result and check it in the classifier, not to add relay-agent acknowledgements or bypass all empty tool-only turns.
