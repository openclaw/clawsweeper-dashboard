## Summary

Keep open. Current main still has the reported false-negative path: zero-payload terminal turns are classified by the generic incomplete-turn heuristic, while successful `sessions_spawn` results are only retained as mutating tool side-effect metadata, not as structured terminal-success evidence.

## What This Changes

The issue asks OpenClaw to treat a successful `sessions_spawn` result as terminal completion for zero-payload pure-relay agent turns instead of surfacing the generic incomplete-turn warning.

## Best Possible Solution

Add a narrow terminal-success signal for accepted `sessions_spawn` results in the embedded run attempt metadata, then have the incomplete-turn classifier skip the warning only when that structured signal proves a successful child session was accepted.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload, no-text terminal assistant turn with a successful `sessions_spawn` result; current main has no place to preserve that success result, so the same shape reaches the incomplete-turn warning path.

## Solution Assessment

Yes. The best fix is to preserve structured spawn-success metadata and classify that as completion, rather than disabling empty-turn detection globally or requiring relay agents to emit artificial acknowledgement text.
