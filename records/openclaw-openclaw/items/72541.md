## Summary

Keep open. Current main still has no accepted-`sessions_spawn` terminal-success signal in the incomplete-turn classifier path, so a zero-payload relay turn can still fall through to the warning.

## What This Changes

The issue asks the gateway to treat an accepted `sessions_spawn` as terminal success for zero-payload pure-relay agent turns.

## Best Possible Solution

Preserve a structured accepted-spawn signal through the embedded tool result path and suppress the incomplete-turn warning only when `status: "accepted"`, `runId`, and `childSessionKey` are present.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload `stop` turn after `sessions_spawn`; current main carries only `toolName/meta` into the classifier, so it still reaches the empty-response warning path.

## Solution Assessment

Yes. The narrow maintainable fix is an accepted-spawn exemption backed by structured result fields, not a relay-agent acknowledgement workaround or a general empty-turn suppression.
