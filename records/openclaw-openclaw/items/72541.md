## Summary

Keep open. Current main still lacks a structured accepted-`sessions_spawn` terminal-success signal in the incomplete-turn classifier, so the reported pure-relay false-negative remains a valid narrow runtime bug.

## What This Changes

The issue requests that the gateway incomplete-turn check treat an accepted `sessions_spawn` terminal tool result as a successful zero-payload turn instead of warning.

## Best Possible Solution

Persist a minimal accepted-spawn terminal-success record and suppress the incomplete-turn warning only when it proves `sessions_spawn` created a child run.

## Reproduction Assessment

Yes. A focused unit test can model `payloadCount: 0`, empty assistant text, `stopReason: "stop"`, and a completed `sessions_spawn` result with `status: "accepted"`, `runId`, and `childSessionKey`; current main has no path for that proof to reach the classifier.

## Solution Assessment

Yes. A structured accepted-spawn exemption is narrower than requiring relay agents to emit acknowledgement text and safer than bypassing all zero-payload tool turns.
