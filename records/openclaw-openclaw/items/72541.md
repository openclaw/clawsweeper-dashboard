## Summary

Keep open. Current main still treats a zero-payload terminal turn as incomplete without checking whether a `sessions_spawn` call successfully accepted a child run, so the reported false-negative remains actionable.

## What This Changes

The issue asks OpenClaw to suppress incomplete-turn warnings when a zero-payload agent turn successfully ends by calling `sessions_spawn`.

## Best Possible Solution

Preserve a structured terminal-success signal for accepted `sessions_spawn` results and let the incomplete-turn classifier treat only accepted spawns with both `runId` and `childSessionKey` as completed work.

## Reproduction Assessment

Yes. A focused unit test can build a zero-payload `stop` attempt after a successful `sessions_spawn`; current main only retains `toolName/meta`, so the classifier still reaches the warning path.

## Solution Assessment

Yes. The narrow maintainable solution is an accepted-spawn exemption backed by structured result data, not a broad empty-turn bypass or requiring relay agents to emit acknowledgement text.
