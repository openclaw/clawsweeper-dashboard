## Summary

Keep open. Current main still has the classifier gap: accepted `sessions_spawn` results are not preserved as structured terminal-success evidence, so a zero-payload pure-relay turn can still fall through to the incomplete-turn warning path.

## What This Changes

The issue requests that successful terminal `sessions_spawn(runtime="acp")` turns be treated as complete even when the parent agent emits no visible text payload.

## Best Possible Solution

Preserve a structured accepted-spawn terminal-success marker from tool completion through the embedded attempt result, then suppress the incomplete-turn warning only when `status`, `runId`, and `childSessionKey` prove a child run was created.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: model a zero-payload stop/reasoning-only terminal turn whose only successful side effect is `sessions_spawn`, and the current classifier has no accepted-spawn signal to return `null`. I did not run a live gateway turn or Vitest in this read-only sweep.

## Solution Assessment

Yes. A narrow accepted-spawn success marker is the maintainable fix; a relay-agent acknowledgement workaround or broad empty-turn bypass would either preserve false warnings or weaken real empty-turn detection.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72541. Current main can surface the incomplete-turn warning when a pure-relay parent agent emits zero visible payloads after a successful `sessions_spawn(runtime="acp")`, because accepted spawn result fields are not preserved for `resolveIncompleteTurnPayloadText`. Add a structured terminal-success marker for successful `sessions_spawn` tool results, carrying at least `status: "accepted"`, `runId`, and `childSessionKey` from tool completion into the embedded attempt result or equivalent classifier input. Make the incomplete-turn classifier return `null` only when that marker proves a child run was created. Preserve existing warnings for no-tool empty turns, failed/forbidden spawn results, tool errors, interrupted tool use, and genuinely incomplete terminal turns. Add focused regression tests for the classifier and the tool/subscription result path. Add a single-line user-facing `CHANGELOG.md` fix entry in the active section. Do not add a config flag or broad zero-payload bypass.
