## Summary

Keep open. Current main still has the reported false-negative path: accepted `sessions_spawn` results are not preserved as terminal success evidence, so a pure-relay turn with no visible payload can still reach the incomplete-turn warning.

## What This Changes

Suppress the gateway incomplete-turn warning for zero-payload terminal turns that successfully accept a `sessions_spawn` child run.

## Best Possible Solution

Add a structured terminal-success signal for accepted `sessions_spawn` results and have the incomplete-turn classifier suppress the warning only when that signal proves a child run was created.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload terminal assistant turn after a successful `sessions_spawn`; current main only carries `toolName/meta`, so the classifier still follows the empty-response warning path.

## Solution Assessment

Yes. The maintainable fix is a narrow accepted-spawn exemption backed by result fields, not a broad empty-turn bypass or a relay-agent acknowledgement requirement.
