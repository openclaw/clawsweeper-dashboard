## Summary

Keep open. Current main still treats a zero-payload terminal turn after `sessions_spawn` as an incomplete response unless it also has committed messaging delivery evidence; accepted spawn status and child session identity are not preserved for the classifier.

## What This Changes

The issue asks the gateway incomplete-turn check to treat a successful terminal `sessions_spawn` as completion for pure-relay agents.

## Best Possible Solution

Preserve a structured terminal-success signal for accepted `sessions_spawn` calls and have the incomplete-turn classifier suppress the warning only when that signal proves a child run was created.

## Reproduction Assessment

Yes. A focused test can construct a zero-payload terminal assistant turn after a successful `sessions_spawn`; current main only carries `toolName/meta`, so the classifier still follows the empty-response warning path.

## Solution Assessment

Yes. The issue’s proposed behavior is best implemented as a narrow accepted-spawn exemption, not as a broad empty-turn bypass or a relay-agent acknowledgement requirement.

## Repair Work Prompt

Fix the gateway false-negative for pure-relay agents that end a turn after a successful `sessions_spawn`. Carry a structured terminal-success signal from tool execution for `sessions_spawn` results only when the sanitized result proves `status: "accepted"` plus nonempty `childSessionKey` and `runId` where available, then have `resolveIncompleteTurnPayloadText` suppress the zero-payload incomplete-turn warning for that accepted-spawn case. Do not suppress failed spawns, tool errors, arbitrary `sessions_spawn` calls without accepted child identity, or unrelated empty turns. Keep replay metadata conservative: accepted spawns can remain replay-unsafe while not being surfaced as an incomplete turn. Add focused coverage in the incomplete-turn helper tests and the tool/subscription metadata tests, update the active `CHANGELOG.md` Fixes section for this user-facing gateway bug, and avoid broader agent-loop or provider behavior changes.
