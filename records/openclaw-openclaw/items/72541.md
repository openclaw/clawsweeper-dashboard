## Summary

Keep open. Current main still lacks a structured accepted-`sessions_spawn` success signal in the incomplete-turn classifier, so the reported zero-payload relay-agent false-negative remains actionable.

## What This Changes

The issue requests suppressing the gateway incomplete-turn warning for pure-relay agents whose terminal successful action is an accepted `sessions_spawn` call.

## Best Possible Solution

Preserve a structured terminal-success signal for accepted `sessions_spawn` results and make the incomplete-turn classifier skip the warning only for that proven handoff.

## Reproduction Assessment

Yes. A focused helper test can feed a zero-payload `stop` attempt after an accepted `sessions_spawn`; current main only preserves tool name/meta, so the classifier still reaches the warning path.

## Solution Assessment

Yes. The narrow maintainable fix is an accepted-spawn exemption backed by structured status, `runId`, and `childSessionKey`, not a broad bypass for all tool calls or a relay-agent acknowledgement workaround.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72541. Current main treats a pure relay turn that ends after a successful `sessions_spawn` as incomplete because the incomplete-turn classifier only sees `toolName/meta`, not the accepted spawn result. Preserve a structured terminal-success signal from `handleToolExecutionEnd` into `EmbeddedRunAttemptResult` for successful `sessions_spawn` results with `status: "accepted"`, nonempty `runId`, and nonempty `childSessionKey`; then update `resolveIncompleteTurnPayloadText` to return `null` for a zero-payload terminal stop only when that accepted-spawn proof exists. Do not suppress arbitrary mutating tools, failed/forbidden/error/timeout spawns, missing run/session keys, or unrelated empty provider stops. Keep the fix generic; do not add Claudette- or deployment-specific ids. Add focused tests for accepted ACP/native spawn suppression and failed/forbidden/missing-key warning preservation, update `CHANGELOG.md` with a user-facing fix entry, and credit @alfredneuworth if adding attribution.
