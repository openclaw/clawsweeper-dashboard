## Summary

Keep open. Current main still classifies zero-payload terminal turns without any structured accepted-`sessions_spawn` success signal, so the pure-relay false-negative remains a valid narrow runtime bug.

## What This Changes

The issue asks the gateway to treat a zero-payload turn ending in a successful `sessions_spawn` acceptance as complete instead of surfacing the incomplete-turn warning.

## Best Possible Solution

Preserve a structured terminal-success record for accepted `sessions_spawn` calls and suppress the incomplete-turn warning only when that record proves a child run was created.

## Reproduction Assessment

Yes. A focused unit test can model payload count `0`, a final `stop` assistant turn, no visible text, and a completed accepted `sessions_spawn`; current main has no accepted-spawn signal in the classifier path, so it reaches the warning branch.

## Solution Assessment

Yes. An accepted-spawn exemption backed by `status`, `runId`, and `childSessionKey` is narrower and more maintainable than a relay-agent acknowledgement workaround or a broad empty-turn bypass.
