## Label Justifications

- `impact:session-state`: The issue is about misclassifying a successful spawned child session as an incomplete parent turn.

## Summary

Keep open. Current main still lacks a structured accepted-sessions_spawn terminal-success signal in the incomplete-turn classifier, so a pure relay can still be reported as an empty failed turn after a successful child-session spawn.

## What This Changes

The issue asks the gateway to treat a zero-payload turn that successfully accepts a sessions_spawn child run as complete instead of surfacing the generic incomplete-turn warning.

## Best Possible Solution

Carry a structured accepted-spawn terminal-success fact through tool completion into attempt metadata, then suppress the incomplete-turn warning only for status "accepted" with non-empty runId and childSessionKey.

## Reproduction Assessment

Yes, at source level. A current-main attempt with payloadCount 0, stopReason "stop", no lastToolError, and a successful sessions_spawn still has no accepted child-run fact available to the classifier and can fall through to the generic warning.

## Solution Assessment

Yes. The maintainable fix is to preserve the structured accepted-spawn result and special-case only that proven terminal success, rather than adding a broad config bypass or treating every successful tool call as a completed response.
