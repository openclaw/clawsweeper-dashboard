## Summary

Keep open. Current main still routes successful zero-payload relay turns through the generic incomplete-turn classifier without preserving enough `sessions_spawn` result metadata to recognize an accepted child session as terminal success.

## What This Changes

The issue requests that the gateway completeness check stop treating successful zero-payload `sessions_spawn` relay turns as incomplete.

## Best Possible Solution

Propagate structured terminal tool-result metadata for successful `sessions_spawn` calls into the embedded attempt result, then treat an accepted child session as completion while preserving warnings for failed spawns and truly empty turns.

## Reproduction Assessment

Yes. A focused unit test can construct `payloadCount: 0`, no assistant text, a non-error `stop`, and successful `sessions_spawn` tool-result metadata; current main lacks the success metadata and reaches the incomplete-turn warning path.

## Solution Assessment

Yes. The narrowest maintainable fix is to preserve the actual accepted spawn result and classify that as terminal success, rather than disabling empty-turn detection globally or requiring relay agents to emit acknowledgement text.

## Repair Work Prompt

Fix the gateway embedded-runner false-negative for pure relay agents that end a turn after a successful `sessions_spawn`. Current main records `sessions_spawn` only as a mutating tool name, so a zero-payload stop still surfaces `Agent couldn't generate a response` even when the tool returned `status: "accepted"` and a real `childSessionKey`. Preserve empty-turn warnings for failed spawns, malformed results, provider errors, and no-tool/no-payload turns. Keep the change generic to terminal tool-result metadata; do not hard-code an agent id or Claudette-specific behavior. Likely implementation: extend attempt/tool metadata types so `handleToolExecutionEnd` retains a small sanitized success summary for terminal tools, teach `resolveIncompleteTurnPayloadText` to treat accepted `sessions_spawn` with non-empty `childSessionKey` as completion, and add focused coverage for accepted spawn, failed spawn, and ordinary empty stop. Add a user-facing CHANGELOG.md fix entry crediting @alfredneuworth. Before handoff run targeted tests locally, then `pnpm check:changed` in Testbox because this touches shared runtime metadata.
