## Summary

Keep open. Current main still has no accepted `sessions_spawn` terminal-success signal in the incomplete-turn classifier; ACP spawn returns structured `status`, `runId`, and `childSessionKey`, but the runner only carries generic tool metadata into the zero-payload stop check.

## What This Changes

The issue requests that zero-text turns ending after a successful `sessions_spawn runtime="acp"` child-run acceptance be treated as complete instead of surfacing the incomplete-turn warning.

## Best Possible Solution

Carry structured accepted child-run creation into attempt metadata and suppress the warning only when accepted status plus non-empty `runId` and `childSessionKey` prove a child run was created.

## Reproduction Assessment

Yes, at source level: model `payloadCount: 0`, no assistant text, `stopReason: "stop"`, and a successful accepted `sessions_spawn`; current main retains only generic tool metadata for the classifier, so the warning path can still fire. I did not run a live ACP dispatch in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve accepted-spawn success facts through tool completion and suppress only that proven terminal-success case, instead of adding a broad empty-turn bypass or a new config option.
