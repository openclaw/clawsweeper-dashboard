## Summary

Keep open. Current main still drops the accepted `sessions_spawn` success fields before the incomplete-turn classifier, so a zero-payload `stop` turn after a successful child spawn can still surface the false incomplete-turn warning.

## What This Changes

A successful accepted `sessions_spawn` with no assistant text should not cause the gateway to emit the incomplete-turn warning.

## Best Possible Solution

Carry structured accepted child-run creation into attempt metadata and treat it as terminal success only when `status`, `runId`, and `childSessionKey` prove a child run was created.

## Reproduction Assessment

Yes, at source level: model `payloadCount: 0`, no assistant text, `stopReason: "stop"`, and a successful accepted `sessions_spawn`; current main retains only generic tool metadata for the classifier, so the warning path can still fire. I did not run a live ACP dispatch because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to preserve accepted-spawn success facts through tool completion and suppress only that proven terminal-success case, rather than adding a broad empty-turn bypass or a new config option.
