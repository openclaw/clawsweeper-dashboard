## Summary

Keep open: the stale-token display bug is still source-reproducible on current main, but this PR is not mergeable as submitted because its TUI guard can retain an old total and it still lacks contributor real behavior proof.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI and Control UI session token display paths and hides stale totals when the flag is false.

## Best Possible Solution

Thread `totalTokensFresh` through the current TUI session row/state contract, clear or ignore retained totals correctly for stale rows while allowing fresh `null` clears, and keep the Control UI formatter defensive.

## Reproduction Assessment

Yes, source-level. Current main emits stale-total freshness metadata, but the TUI state path does not carry it and the Control UI formatter still ignores it when a numeric total reaches the row.

## Solution Assessment

No, not as submitted. The direction is narrow and appropriate, but the TUI guard must handle stale omitted totals and fresh-null clears before this is maintainable.
