## Summary

Keep open. Current main still lacks the TUI freshness plumbing needed to suppress stale footer token totals, and the submitted branch is useful but not merge-ready because its TUI guard can preserve stale counts and it targets an older session-list type shape.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI and Control UI session row/type paths so stale session token totals are not displayed.

## Best Possible Solution

Land a narrow current-main repair that carries `totalTokensFresh` through the active TUI session list/session-info path, suppresses or clears cached TUI footer totals when freshness is false, allows fresh `totalTokens: null` to clear state, keeps Control UI token formatting freshness-aware, and adds focused regressions for both display paths.

## Reproduction Assessment

Yes. Seed TUI state with a numeric `sessionInfo.totalTokens`, then apply a same-session row or patch with `totalTokensFresh: false` and no fresh total; current main has no freshness branch and leaves the old footer value visible. The PR-specific regression is also source-reproducible with a fresh `totalTokens: null` patch, which the submitted non-null guard ignores.

## Solution Assessment

No, not as submitted. The direction is correct, but the TUI fix must handle freshness before value/null checks and be ported to the active current-main `TuiSessionList`/`SessionInfo` path with regression coverage.
