## Summary

Keep open. The reported stale-token display path is still source-reproducible on current main, but the submitted PR is not merge-ready because its TUI guard can retain old totals and the external proof gate is still missing.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI session-list entries and Control UI session-token formatting so stale session totals are not displayed.

## Best Possible Solution

Rebase onto current main, carry `totalTokensFresh` through the TUI row contract, clear retained TUI totals when freshness is false while allowing fresh null clears, keep Control UI formatting defensive, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-level. Gateway rows emit `totalTokensFresh`, current TUI drops or ignores that flag before footer formatting, and Control UI formatting still ignores a stale flag on a direct row; I did not run a live TUI/UI session.

## Solution Assessment

No, not as submitted. The fix boundary is right, but the TUI condition must clear retained stale totals and allow fresh null clears before this is the narrow maintainable solution.
