## Summary

Current main still has the freshness gap: gateway rows can carry `totalTokensFresh=false`, but the TUI session contract and footer state do not carry it, and the Control UI formatter still ignores it. The PR is plausible and narrow, but it remains blocked by missing real behavior proof and a concrete TUI guard bug that can keep stale totals visible.

## What This Changes

The PR threads `totalTokensFresh` into TUI and Control UI session row types and hides session token totals when that flag is false.

## Best Possible Solution

Carry `totalTokensFresh` through the TUI state contract, clear or hide retained totals when freshness is false while still allowing fresh null clears, keep the Control UI formatter defensive, and reconcile with the active TUI footer work before merge.

## Reproduction Assessment

Yes, source-level. Current main emits `totalTokensFresh`, the TUI path does not carry that flag and can retain an older `totalTokens`, and the Control UI formatter ignores a stale flag if a numeric total reaches it.

## Solution Assessment

No, not as submitted. The fix boundary is narrow, but the TUI update guard must handle omitted stale totals and fresh null clears before the patch is correct.
