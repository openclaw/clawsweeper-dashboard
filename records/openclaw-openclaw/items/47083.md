## Summary

Keep open: the reported freshness bug is valid, but this PR still has a blocking TUI stale/null clearing defect and lacks real behavior proof; current main only partially handles freshness through gateway and Control UI event paths.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI and Control UI session row types and gates token display/update paths on that freshness flag.

## Best Possible Solution

Thread freshness through the TUI session row/state contract, clear retained totals when freshness is false while allowing fresh null clears, keep Control UI formatting defensive, and coordinate with the active TUI footer work before merge.

## Reproduction Assessment

Yes, source-level. Current main emits `totalTokensFresh`, the TUI path does not carry that flag and retains old totals when a stale row omits `totalTokens`, and the Control UI formatter still ignores a stale flag if a numeric total reaches it.

## Solution Assessment

No, not as submitted. The fix boundary is narrow, but the TUI guard must handle stale/omitted totals and fresh null clears before this is the maintainable solution.
