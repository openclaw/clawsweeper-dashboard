## Summary

Keep open: the stale-token display bug is still source-reproducible, but this PR is not ready because the submitted TUI guard can retain stale totals, the external contributor has not supplied real behavior proof, and the branch now conflicts with newer session freshness work.

## What This Changes

The branch adds `totalTokensFresh` plumbing to TUI and Control UI session token display paths so stale session totals are not shown.

## Best Possible Solution

Update this PR or a replacement branch against current main to carry `totalTokensFresh` through TUI row/state contracts, clear retained totals when freshness is false while allowing fresh `null` clears, keep the Control UI formatter defensive, add focused tests, and reconcile with the active TUI footer work.

## Reproduction Assessment

Yes, source-level. Current main emits stale rows with `totalTokens` omitted and `totalTokensFresh: false`, while the TUI path does not carry that flag and the Control UI formatter ignores it when a numeric total reaches the presenter.

## Solution Assessment

No, not as submitted. The fix boundary is right, but the TUI update must clear stale retained totals and permit fresh-null clears, and the external PR still needs real behavior proof before merge.
