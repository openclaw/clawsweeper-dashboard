## Summary

Keep open: current main still has the TUI and Control UI token-display gaps, but this PR is not mergeable as submitted because the TUI guard can keep an old total visible and the external PR still lacks real behavior proof.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI and Control UI session display paths so stale total-token values are not shown.

## Best Possible Solution

Rework this narrowly against current main: carry `totalTokensFresh` through TUI session rows, clear retained TUI totals on stale or omitted rows while allowing fresh null clears, keep the Control UI formatter defensive, and reconcile with active footer/token semantics work.

## Reproduction Assessment

Yes, source-level: current main emits `totalTokensFresh`, TUI does not carry it, and both TUI state application and the Control UI formatter can display retained or explicitly stale totals.

## Solution Assessment

No, not as submitted. The target surface is right, but the TUI guard must handle stale omitted totals and fresh-null clears before the branch is maintainable.
