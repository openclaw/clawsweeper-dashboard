Rebase a narrow patch that propagates freshness, clears stale TUI totals, preserves fresh null resets, keeps Control UI formatting defensive, adds regression coverage, and supplies real display proof.

## Reproduction Assessment

Yes, source-level: current main's gateway row contract emits `totalTokensFresh: false` while stale totals are omitted, and TUI state keeps an old `totalTokens` unless explicitly cleared. I did not run a live TUI or Control UI session in this read-only review.

## Solution Assessment

No, not as submitted. The right boundary is plausible, but the patch must handle the freshness flag before the numeric guard, preserve fresh null clears, clear stale existing totals, and rebase over current freshness work.
