## Summary

Keep this PR open, but do not merge it as-is. Current main still has the stale token display gap in TUI and the Control UI formatter, while the PR's TUI guard would leave stale cached counts visible for flag-only updates and block fresh null resets.

## What This Changes

The PR adds `totalTokensFresh` fields and display checks in TUI and Control UI session token paths so stale session totals are hidden.

## Best Possible Solution

Land a narrow repair that carries `totalTokensFresh` through the active TUI session-info path, suppresses or clears stale footer totals, allows fresh `totalTokens: null` clears, makes Control UI token formatting return `n/a` for stale rows, and adds focused regressions plus a changelog entry.

## Reproduction Assessment

Yes. On current main, a TUI session row or patch with `totalTokensFresh: false` and a numeric `totalTokens` is still copied, and `formatSessionTokens` still renders a stale numeric total; the PR-specific regression is visible from the submitted null guard.

## Solution Assessment

No, not as submitted. The direction is correct, but freshness must be checked before value/null guards and the patch needs to be ported to the current TUI type surface with tests.
