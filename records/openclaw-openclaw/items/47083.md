## Summary

Keep open. The bug is real and the PR is pointed at the right surfaces, but the submitted TUI guard can still leave stale totals visible, the branch conflicts with current `main`, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI and Control UI session-token display paths so stale session totals are hidden.

## Best Possible Solution

Land a current-main-based fix that carries `totalTokensFresh` through the TUI session-info path, handles stale flag-only and fresh-null updates correctly, keeps Control UI formatting defensive, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current TUI can retain an old `sessionInfo.totalTokens` when a stale refresh lacks a fresh total, and current Control UI `formatSessionTokens` formats a numeric total even when `totalTokensFresh` is false.

## Solution Assessment

No, not as submitted. The direction is narrow and maintainable, but the TUI guard must evaluate freshness before value/null guards and the branch needs to be ported to the current `TuiSessionList` shape.
