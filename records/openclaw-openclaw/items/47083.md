## Summary

Keep open: current main still has a source-reproducible stale token display path, and this PR is useful but not merge-ready because the submitted TUI guard can still retain stale totals or block fresh-null clears.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI and Control UI session-token display paths so stale session totals are hidden.

## Best Possible Solution

Land a current-main-based fix that carries `totalTokensFresh` through the TUI session-info path, handles stale flag-only and fresh-null updates correctly, keeps Control UI formatting defensive, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a TUI refresh can retain an old `sessionInfo.totalTokens` when freshness metadata is unavailable or stale, and a Control UI row with `totalTokensFresh: false` plus a numeric total still formats as a count.

## Solution Assessment

No, not as submitted. The direction is narrow and maintainable, but the TUI guard must check freshness before value/null guards and the branch should be ported to the current `TuiSessionList` type surface.
